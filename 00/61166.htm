The purpose of copying the files to the tmp folder is to keep a backup copy of your current registry in case you need to get back to where you are.  It is quite possible that a disk problem with that file is what caused your system to fail in the first place.  In that case you may not be able to copy that file.  A good alternate approach at this point is to simply rename the "software" file to "software.xxx" ( ren software software.xxx ).  You might be able to rename the  file whereas you couldn't copy it.

a. user say:

Instead of booting up and shutting down every time, you can sleep or hibernate your computer.

Otherwise, boot time can be shortened by reducing the number of applications that start with Windows to a bare minimum.

Shut down time can be shortened by closing all applications before shutting down.

Those are the only methods achievable by software. You might reduce boot and shutdown times further with a fast SSD, but only if your storage subsystem can take advantage of the speed.

The internet is filled with tips and tricks to reduce boot and shut down time. If they work at all, the number of milliseconds they save will be unnoticeable. And if these techniques are unsupported, you risk damaging your installation.

To answer your specific questions:

Adding RAM doesn't affect startup or shut down by any amount that you'll notice.

A clean install may improve startup and shut down, but only for the reasons described above, which can be achieved without a clean install.

greg carmack - mvp 2010-2020 say:

What happens here most often is that a backup program writes to the System Reserved partition which clogs it so the Update installer cannot edit the boot files to continue. 

One solution is to assign a Drive Letter to System Reserved in Disk Mgmt, browse in to delete the intruding backup file, then either uninstall the errant backup program or redirect its output to  a storage partition. 

Another solution which is more of a band aid is to add space to the System Reserved partition (e.g. 100mb<350mb) following How to extend partition easily with Partition Wizard - video help  which can borrow space from any partition even if not adjacent.

Finally you can delete the System Reserved partition after moving its boot files into C.  To move the Bootmgr to C, download and install EasyBCD (click Download - no Name or Email required). 

1) Choose BCD Backup/Repair Tab.

2) Select Change boot drive

3) Click button to Perform Action

In the popup confirm the New Boot Drive says C, click OK.

Reboot PC, confirm C is now marked System Active Boot.  You can now delete System Reserved in Disk Mgmt, repartition the drive as a data drive in Disk Mgmt by using the Create button.
