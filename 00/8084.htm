Confirmation/correction of my assumptions below would be appreciated, as well as feedback on how well (or not) the approach works for you.

Previously, you used columns CN:CU for the computed results, namely cells(x,79).offset(0,13) through cells(x,79).offset(0,20).  Now it appears that you use CO:CV, namely offset(0,14) through offset(0,21).

I will use the latter.  I also use CX:CZ for new helper cells.

I suggest that you start by prototyping everything using Excel formulas.

You might find that the recalculation time is sufficient, obviating the need to use VBA.  In my case, the full recalculation (ctrl+alt+F9) of about 60,000 rows of data takes less than 1 second; and automatic recalculation time is negligible.

If your experience is different and you still want to put these calculations into VBA, let me know if you need help with that.

First, make a copy of your Excel workbook.  In the copy, set up the new helper cells as follows.

CX2:   =OR(RIGHT(CB2,3)={"ASE","CRU","XID"})+IF(CA2=CA1,CX1)
CY2:   =IF(CA2=CA1,CY1,D2)
CZ2:   =IF(CA2=CA3,CZ3,E2)

Format CX as Number; format CY and CZ as Date.

Copy CX2:CZ2 down through all the rows that comprise the data; that is, about 80,000 rows.

Note that I assume that:  (a) data starts in row 2; (b) row 1 is empty; and (c) the row following the last row of data is empty.  If you cannot live with those assumptions, notably #b and #c, some minor adjustments are needed in the first and last rows.  Let me know if you need help with that.

CX is the cumulative count of all data in CB for the same item ending in ASE, CRU and XID; roughly equivalent to your offset(0,14) formula.

CY is the earliest date in D for the same item; equivalent to Cells(wf.Match(sTemp,rRng,0)+1,4) in your offset(0,15) formula.

CZ is the latest date in E for the same item; equivalent to Cells(wf.CountIf(rRng,sTemp)+wf.Match(sTemp,rRng,0),5) in your offset(0,15) formula.

Finally, set up your original computations as follows.  They are equivalent to your formulas for offset(0,14) through offset(0,21).

CO2:   =IF(CA2=CA3,CO3,CX2)
CP2:   =CZ2-CY2+1
CQ2:   =CP2-CO2
CR2:   =CP2+D2
CS2:   =ROUNDUP(MONTH(CR2)/3,0)
CT2:   =MONTH(CR2)
CU2:  =TEXT(CR2,"ddd")
CV2:   =YEAR(CR2)

Ostensibly, format CO:CQ, CS, CT and CV as Number; format CR as Date; and format CU as General.

However, if you only need to display  month, day and year in CT:CV, change the formulas in CT:CV to simply =CR2, and use the Custom formats m, ddd and yyyy respectively.

On the other hand, if you leave the formulas in CT:CV as-is above, you can change MONTH(CR2)/3 to CT2/3 in CS.

Copy CO2:CV2 down through all the rows that comprise the data; that is, about 80,000 rows.

Note that the formula in CO relies on the same empty-row assumptions noted above.

pamcaswell say:

As Hans suggests,  the cause is the document map.  It applies paragraph outline levels as direct formatting, to short and/or bolded and emphasized paragraphs.  In W2003, the only way to stop the behavior is to stop using the  document map.  

If you must use the document map and if the only thing you object to is the occurrence of these bogus headings in your TOC, you can, after  toggling the  TOC field codes, remove the \u switch--which allows the TOC to gather paragraphs whose outline levels are applied by direct formatting.  (My continued thanks to Stefan Blom for this tip.)

hansv mvp say:

You could add lines to Bernie's code to unprotect/reprotect the sheet:

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim c As Range
   
    'If only one cell at a time should be changed, un-comment the next line
    'If Target.Cells.Count > 1 Then Exit Sub
   
    'To allow deletions to be bypassed without running the code, un-comment the next line
    If Target.Cells(1).Value = "" Then Exit Sub
   
    If Intersect(Target, Range("V:V")) Is Nothing Then Exit Sub 'specific range
   
    'Turn off events to keep out of loops
    Application.EnableEvents = False
   
    ' Unprotect the sheet
    Me.Unprotect 'Password:="secret"
   
    For Each c In Intersect(Target, Range("V:V"))
        If c.Value <> "" Then
            Cells(c.Row, "O").NumberFormat = "m/d/y"
            Cells(c.Row, "O").Value = Date
        End If
    Next c
   
    ' Protect the sheet again
    Me.Protect 'Password:="secret"
   
    'Turn events back on to get ready for the next change
    Application.EnableEvents = True
End Sub

If you protected the sheet with a password, remove the ' before Password:="secret" and change "secret" to the password that you used (both occurrences).
