andreas killer say:
When I double click in the header, in the left ruler, with the mouse I see that there is a bottom and top margin for the header. Is there a way to get both these values?
That might be a way, we can use the range object of the header and footer object to get the position on that page.

Unfortunatelly the InlineShape has no Top property, so we must use the shape object instead. You must convert the InlineShape to a Shape first.

Andreas.

Option Explicit

Sub ConvertInlineShapesToShape()
  Dim I As InlineShape
  Set I = ActiveDocument.InlineShapes(1)
  I.ConvertToShape
End Sub

Sub ResizePictureToPage()
  Dim Header As HeaderFooter
  Dim Footer As HeaderFooter
  Dim R As Range
  Dim S As Shape
  Dim hdTopLine As Single, hdBottomLine As Single, hdLines As Integer
  Dim shTop As Single, shHeight As Single, pgHeight As Single
 
  Set S = ActiveDocument.Shapes(1)
  Set Header = ActiveDocument.Sections(1).Headers(wdHeaderFooterPrimary)
  Set Footer = ActiveDocument.Sections(1).Footers(wdHeaderFooterPrimary)
 
  With ActiveDocument.PageSetup
    'Make sure we can stretch the shape
    S.LockAspectRatio = msoFalse
   
    'Adjust vertical
    S.Left = .LeftMargin + .Gutter
    S.Width = .PageWidth - .LeftMargin - .RightMargin - .Gutter
   
    'Adjust horizontal
    Set R = Header.Range
    'Get number of lines in header
    hdLines = R.Paragraphs.Count
    'Calculate the position of the next line after the header (which doesn't exists)
    If hdLines > 1 Then
      hdTopLine = R.Information(wdVerticalPositionRelativeToPage)
      R.Collapse wdCollapseEnd
      hdBottomLine = R.Information(wdVerticalPositionRelativeToPage)
      shTop = hdBottomLine + (hdBottomLine - hdTopLine) / (hdLines - 1)
    Else
      shTop = R.Information(wdVerticalPositionRelativeToPage) * 2
    End If
    'Make sure we are not over the top
    If shTop < .TopMargin Then shTop = .TopMargin
    S.Top = shTop
   
    'Get the position of the start of the footer on this page
    Set R = Footer.Range
    R.Collapse wdCollapseStart
    'Calculate the possible heights
    shHeight = R.Information(wdVerticalPositionRelativeToPage) - S.Top
    pgHeight = .PageHeight - .BottomMargin - S.Top
    'Make sure we are not below to bottom
    If pgHeight < shHeight Then
      S.Height = pgHeight
    Else
      S.Height = shHeight
    End If
  End With
End Sub

suzanne s. barnhill say:

According to the referenced article:

Scratch File (Temp Directory)

When Word runs out of internal random access memory (RAM), it always creates a single temporary scratch file in the Temp directory to hold information. This scratch file holds information that is swapped out from the Word internal file cache, which is allocated from global system memory. The scratch file varies in size from 64 kilobytes (KB) to 3.5 megabytes (MB).

The default cache size in Word is 64 KB.

In general, temporary files are deleted when the parent file is closed, so the file Word is trying to open may no longer exist. How are you opening the file? From within Word? If so, from the Recent list or the Open dialog? Or from Windows Explorer or a list of recent files on the Start menu?
