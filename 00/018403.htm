Indeed, no official document has provided very detailed steps about upgrading Exchange 2010 hybrid to Exchange 2013/2016. From my experience, the general steps are:

1. Introduce Exchange 2013/2016 in your environment

2. Make Exchange 2013/2016 public facing

3. Point your MX and Name Space to Exchange 2013/2016 from Exchange 2010

4. Update your Hybrid Configuration by running HCW and select Exchange 2013/2016 as hybrid server.

5. Decommission Exchange 2010

 

There an official tool named Exchange Server Deployment Assistant, which has an option about upgrading existing Exchange 2010 hybrid to Exchange 2013/2016.

a. for choose your deployment type, please choose Exchange 2013 hybrid/Exchange 2010 hybrid

b. for current on-premise environment, please choose Exchange server 2010

c. for do you have an existing Exchange 2010-based hybrid deployment configured with Exchange online in Office 365, please choose Yes

d. for do you want to upgrade your existing Exchange 2010-based hybrid deployment, please choose Yes.

 

An example about upgrading Exchange 2010 hybrid to Exchange 2013 hybrid.

suzanne s. barnhill say:

Wrt the indents, I ran across the solution to this in the (free) Smashwords Style Guide:

The Amazon Kindle and its associated apps (Kindle for PC, Kindle for Mac) automatically insert a preceding first-line indent if one isn’t defined, even on block paragraphs. This can create an ugly, unprofessional-looking ebook. To overcome this problem, I configured my Normal to define a “special” first-line indent of 1/100th of an inch (.01″) to trick Kindle apps and devices to not automatically indent my paragraphs. The indent will be imperceptible to most eyes, and it should prevent Kindle and its apps from inserting the first-line indent where you don’t want it.

chris-th say:

I brought in an IT professional and he found the problem to be with the Microsoft facility for saving the Word.doc as an html file.  In doing so, it added spurious font colour definitions.

One solution was to open the html file in Notepad and go through the Styles (ie from <head> to <head>), deleting all references to colour and definition.

The other solution was to download a new Word program (in this case Kingsoft Writer) and use that to save my Microsoft Word.doc as an html file. No spurious colour definitions were added.

A list style only wraps nine levels of numbering/bullets into a single style. The list style is an "entry point" whenever you want to edit the list. To apply numbering to text, apply the appropriate paragraph style assigned to the relevant list level.
