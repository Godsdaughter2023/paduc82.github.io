maxwire say: I had the same problem. I fixed it by deleting all of the entries, except the default entry, in HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\RecentDocs in the registry. There are a bunch of other entries that are numbered for each of the files in the Recent Items list. and MRUListEx entry. Delete all of these. Unfortunately, there are no filenames, but only HEX entries, numbers and letters. After doing this, Windows started listing the files in Recent Items that weren't showing up there before.

I found that the problem followed a couple of files that were present in the Recent Items when I cleared the list a few weeks earlier. After that those files would not show up in Recent Items, but instead the shortcut to the folder appeared. Also, I found that the problem was with the actual files, and not the folder. I opened the 2 files, an Excel and a Word document, and chose Save As, and added a 2 to the document name, saving an exact copy of the documents. Each of these copies showed up in the Recent Items, and opened with no problems. Renaming the copies back to the original names, and overwriting the originals, brought the problem back. I searched the registry for the files to see if there was information about them saved in a special location, but found nothing. They were only listed in the MRUlists for each of their applications, showing the other recently opened Excel and Word documents.

Since I had made changes to the documents and saved the changes in the copies ending in 2, I chose Save As again, and overwrote the original files with the updated copies. Now my original files, with the updated info, show up and open from the Recent Items list.
