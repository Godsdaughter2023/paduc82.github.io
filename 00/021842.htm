ossiemac say:

 

Rick’s last post will always find something because if no data visible then it will still find the Header row.

 

The following solution will tell you if there is visible data in addition to the column header. Also see the next example below with an alternative method of establishing if there are visible rows of data.

 

 Dim rngVisibleData As Range

 

 With Sheets("Sheet1").AutoFilter.Range

   Set rngVisibleData = .Resize(.Rows.Count, 1) _

                        .Cells.SpecialCells(xlCellTypeVisible)

 End With

 

 If rngVisibleData.Cells.Count = 1 Then        'Header row only if 1 cell

   MsgBox "Only column header is visible. No data exists."

 Else

   'Code here if data exists.

 End If

 

Explanation of code:

.AutoFilter.Range is the entire filtered rangeincluding hidden rows.

 

.Resize(.Rows.Count, 1)  resizes to the number of rows in .AutoFilterRange and 1 column wide. (Note: Includes column header and still includes the non visible rows.)

 

.Cells.SpecialCells(xlCellTypeVisible) is to only include visible cells. Still includes column header.

 

You will recall that I said in an earlier post that you cannot count non contiguous rows. However, that does not apply to counting non contiguous cells which can be counted. Therefore if it is only one column wide if you count the cells then that equates to the number of visible rows but it includes the column header. If you only have 1 cell then only the column header is visible so no data is visible.

 

 

A bit of additional information. If you wanted to copy and paste the visible data of an AutoFiltered Range or perform some other processing on the visible data then the following code assigns the visible data, without the column headers, to a range variable. The example copies and pastes the visible data to another worksheet.

 

Note the alternative method of testing for visible data.

 

Sub CopyPasteVisibleData()
'Includes method of testing for visible data
 
  Dim rngCopyData As Range
  Dim rngDestin As Range
 
  With Sheets("Sheet1").AutoFilter.Range
   
    On Error Resume Next   'In case no visible data
   
    Set rngCopyData = .Offset(1, 0) _
                      .Resize(.Rows.Count - 1, .Columns.Count) _
                      .Cells.SpecialCells(xlCellTypeVisible)
    On Error GoTo 0   'Reset error trapping ASAP
   
  End With
 
  If Not rngCopyData Is Nothing Then
    'Assign the destination to a range variable
    With Sheets("Sheet3")
      'Set destination to first blank cell in column A of Sheet3
      Set rngDestin = .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0)
    End With
   
    rngCopyData.Copy Destination:=rngDestin
 
  Else
    MsgBox "No visible data to copy and paste."
  End If
 
End Sub

 

 

 

Explanation of Set rngCopy.

 

.AutoFilter.Range is entire filtered rangeincluding non visible rows.

 

.Offset(1, 0)  moves down one row off the column headers. However, it now includes an extra row at the bottom of the AutoFilter.Range. (Still includes the non visible rows.)

 

.Resize(.Rows.Count - 1, .Columns.Count)  removes the extra row at the bottom caused by Offset. (Still includes the non visible rows.)

 

.Cells.SpecialCells(xlCellTypeVisible)  Only include visible rows.
