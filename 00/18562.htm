teylyn say:
Hello,

the value field buttons on pivot charts are not there to hide or show data. They are used to select certain commands related to the data, like change the value field settings, or the order of the series in the chart.

To show or hide or filter data, you need to use the filter tools, i.e. the page filter, column filer in the pivot table,  or slicers.

But with a data layout where you have the same kind of data (month names) spread out over different columns, the slicers won't work, because they cannot hide or show fields, just filter the values within a field. 

If you want to filter by month, then the data for all months must be in two columns: Month and Value, and not one column for each month. The way your data is laid out, you won't be able to use pivot table filters to hide or show months.

An Excel spreadsheet has over a million rows. If that is not enough to hold your data, then you can use SQL to store more and use a query as the source for the pivot table.

cheers, teylyn

Formula to Add Exponents - Microsoft Community

erins. say:

Formula to Add Exponents

Hi All,

 

Is there any formula in Excel that would help me do the following?

 

A1^(A2-1)+A1^(A2-2)+A1^(A2-3)...+A1^(1)

AKA: x^(n-1)+x^(n-2)...+x^1

 

Thanks.

 

Erin

joeu2004 say: Erin wrote:

Is there any formula in Excel that would help me do the following?
A1^(A2-1)+A1^(A2-2)+A1^(A2-3)...+A1^(1)
AKA: x^(n-1)+x^(n-2)...+x^1


You could use the following formula [2]:

=SUMPRODUCT(A1^ROW($A$1:INDEX($A:$A,A2-1)))

Alternatively, you could use the following formula that is mathematically equivalent to the series [1]:

=(1 - A1^A2) / (1 - A1) - 1



-----
[1] Knuth, "The Art of Computer Programming: Fundamental Algorithms", vol 1, 2nd ed, p. 31, ex 3, eq (14).

[2] [EDIT] Alternatively, we might write:
=SUMPRODUCT(A1^ROW(INDIRECT("A1:A" & A2-1)))
Note that we are using ROW(...) to generate an array of numbers effectively {1,2,...,A2-1}.  An advantage of the INDIRECT form is that it ensures the integrity of the ROW(...) expression even if column A or row 1 is deleted.  A disadvantage is that INDIRECT is a "volatile" function.  Thus, the formula would be recalculated whenever Excel recalculates anything, e.g. each time any cell in any worksheet is modified.  That might not be an issue if you have only a few or a few hundred such formulas.  INDEX is not a "volatile" function.

steven _fyiguy_ hughes say:

Is this Exchange server 2003 or 2007?

One thing to check is to see if items are being deleted and removed from the deleted items folder, which you can manually delete them by selecting the deleted items folder, and under tools recover deleted items, select them and purge.

Another thing could be that you are reaching the maximum size for your Exchange Information store, which by default is 16GB and this Microsoft KB article talks about the Exchange size limits limits and how to increase them as far as 75gb in SP2: http://support.microsoft.com/kb/912375/.

The last thing I can think of is that their email account data store is corrupt. You could create a new dummy account with settings that work from known good and import the data if that works just delete the old account, recreate it and import. If you do have corrupted data you may also try running an ISINTEG -FIX against the database. This will correct any inconsistencies in your database tables and please make a backup before doing this.

Just to make sure adjust the size of the Exchange User storage limit by using Active Directory User & Computers->properties of the user->select Exchange General tab and the Storage Limits button. Next set prohibit send to 2GB or twice the size of what is in their current storage limit.

Also note changes take a while to propogate the AD. For some reason global changes to everyone's account propogates faster, than individual ones.

Hope this Helps.
