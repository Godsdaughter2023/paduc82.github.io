jay freedman say:

From that description, the text is not connected into paragraphs the way it would appear from the screen shot. The first thing you need to do is turn on "nonprinting" characters by clicking the ¶ button on the ribbon. Then you'll see paragraph marks (the ¶ symbol) at the ends of some lines. You may also see hooked left-pointing arrows, which are line breaks that start a new line without starting a new paragraph. Finally, the lines from "Andrew" through "White" are probably in a table; if you click on one of those lines, the Table Tools tab should appear on the ribbon as confirmation.

Now, specifically about the two lines at the bottom of your screen shot: If changing the indents of the first line does not affect the second line, then they must be two different paragraphs -- there must be a paragraph mark at the end of the first line.

The brute-force fix (not very good in general, but OK for one instance) is to adjust the indents of the second line to be the same as those of the first line. A quick way to do that is to click in the first line and press Ctrl+Shift+C (the shortcut for Copy Format), then click in the second line and press Ctrl+Shift+V (the shortcut for Paste Format).

The better way is to delete the paragraph mark at the end of the first line, insert a space if necessary, and then adjust the indents of what is now a single paragraph.

If adjusting those indents also affects everything up to "Andrew", then the table is also part of that same paragraph. Click to the left of "I am..." and press the Enter key to insert a paragraph mark. Then press left-arrow once so the cursor is in the paragraph that contains the table (but not inside the table itself) and set the indents back to the original location (probably the zero point on the ruler).

I think the basic idea here is to FIRST build some kind of prompt form. Just think of any typical application or even web site you ever used.

For example, when you use BingGoole, they don't download the whole internet, and then let you search the whole huge internet page.

And even think of how an instant teller machine works. You FIRST ask when you want, and THEN pull the one record.  In other words, the instant teller machine does not download millions of records and THEN ask you for the one record you want to look at.

So, even with linked tables to SQL server, it is best to simply ask for a user name, an account number etc. I write about and give some UI ideas here:

http://www.kallal.ca/Search/index.html

So I would suggest you build some type of search/list "prompt" form. You give the user a chance to typing in some kind of search for a record, display the results in a pick list that then lets the user launch + view + edit the one record in question in a form that you launch.  As noted, the form you launch to view/edit the record can be bound to the large table, but just make sure you supply a "where" clause when you open the form.

I wasn't sure how to interpret your post... when you change the selection in the drop-down, does anything happen?  e.g. do values of cells update due to the change in the drop-down cell?  (you said "change to the next name")  If so, you'd need the code to actually change the dropdown cell before printing. 

To do this you could modify JLLatham's excellent code thus:

.....
  'change this constant as needed
Const ddCellAddress = "E2" ' change to your dropdown cell address
Dim listRange As Range
Dim anyListEntry As Range

Set listRange = Range(Right(Range(ddCellAddress).Validation.Formula1, _
Len(Range(ddCellAddress).Validation.Formula1) - 1))
For Each anyListEntry In listRange
       Range(ddCellAddress).Value = anyListEntry.Value
       ThisWorkbook.Worksheets(anyListEntry.Value).PrintOut copies:=1
Next
Set listRange = Nothing
End Sub

If changing the dropdown has no affect, I'm a little curious why it's there...
