nuumio say:

System Thread / process scheduling differences in Win 10 Pro vs. Home under heavy CPU load

Hello,

I’ve recently noticed a difference in thread / process scheduling between Windows 10 Pro and Home versions when system is under heavy CPU load. The difference is that in Home version system may slow down a bit and the application causing the load may occasionally get “not responding”. But in Pro version system becomes unusable very quickly after CPU intensive thread count exceeds thread count supported by hardware as everything, including mouse and keyboard input, starts to freeze. These freezes can last from few seconds to over a minute.

I noticed this when I was stress testing my new Windows 10 Pro / Ryzen setup to check overall system stability. The first time I noticed this was when IntelBurnTest was running in very high stress level. Every now and then IntelBurnTest’s own “freeze test” animation stops for few seconds and in the same time mouse cursor stops too. Then I tried opening and closing Chrome when IntelBurnTest is running. After closing Chrome the whole system could freeze for over a minute.

First I thought that it was hardware related. But then I run same tests with my old Window 10 Home / Xeon setup which I then upgraded to Windows 10 Pro. With Home version system was always usable no matter how many CPU intensive threads were running. But with Pro version freezes started quickly after exceeding HW thread limit. Changing process priorities from Task Manager doesn’t seem to have any effect. Process with real time priority freezes when normal priority (or even low priority) process creates enough CPU intensive threads.

To demonstrate the difference I’ve make a couple of videos using self made test software for easy thread count control. Here are links to parts showing the difference between Home and Pro on Xeon platform when 128 threads are running. I ran the test in safe mode to minimize the effect of background applications (this happens in normal mode too):
- Home version / 128 threads: https://youtu.be/rY5XPVgs4d0?t=181
- Pro version / 128 threads: https://youtu.be/Attq3a1Eok8?t=404

So the question is that is there some kind of difference in thread / process scheduling between Windows 10 versions and are there any settings I could change in Pro version to make it work like what I’ve got used to with Home version? I do realize that these kind of tests create extreme load and I don’t expect the system to be 100% responsive. But the difference between Home and Pro is huge and I prefer the way that Home version works in this case.

nuumio say:

Update: I finally had some time to do more testing and I think I figured out what the problem is. It’s the Creators Update.

My old Xeon + Win10 Home setup didn’t have Creators Update installed and my new Ryzen set setup had it as Media Creation Tool automatically used the latest version. After reading about problem people have had after Creators Update I decided to do a little testing.

I downloaded Win10 ISO images for both Anniversary Update and Creators Update. On my old Xeon box I First tried clean install of Creators Update without any further updates and the freezing problem was there. Then I made a clean install of Anniversary Update without any further updates and the problem was gone. No more mouse pointer freezing! Next I installed every update available (excluding Creators Update), all drivers and all software I use. Still no problems. Under extreme load there’s usually little stuttering the first time you minimize a window but it seems to happen only once. Also, changing priorities in task manager now works as expected.

After promising results with my old setup I decided to do a clean install of Anniversary Update on my new Ryzen setup. After installing all updates (no Creator Update) and all software I need the freezing problem was gone on this machine too. So it looks like Creators Update is the one to blame.

"Deleting" a file doesn't actually delete it; it just marks the space
as available to be used. There are third-party programs that can
sometimes recover deleted files. The problem is that the space used by
the file is likely to become overwritten very quickly, and this makes
the file unrecoverable.

So your chances of successfully recovering this file are decent if you
try recovering it immediately after deleting it, and rapidly go
downhill from there if you write other files to the device.

rajesh govind say:

 

Hi,

 

 

It’s great to know that you have chosen Windows 8 and thank you for posting your query on Microsoft Community Forums.

 

As per you description, you would like to remove the login password for your new Windows 8 computer having the Microsoft Account, to avoid the computer from prompting for the password when you restart or power on the computer.

 

Passwords provide the first line of defense against unauthorized access to your computer. The stronger your password, the more protected your computer will be from the unauthorized users. So, it is recommended to have a password for the computer.

 

If you are using the Microsoft Account to log into your computer, then you will not be able to remove the password. The password is mandate when you are using the Microsoft Account to log into your computer.

 

If you want to avoid the computer from asking for the password when you restart or power on the computer and if you have a local account in the computer, then you may have to switch to a local account to remove the password. We can remove the password only when we are using the local account on the computer. If you do not have a local account then you can create one without setting a password for it.

 

To remove the password of the local account on your computer, simply follow these steps and let’s know if this helped.

 

 

a)       Press “Windows Logo” + “C” keys on the keyboard to display the charms bar.

 

b)       Click on “Settings” and then click on “Change PC Settings”.

 

c)       Select “Users” from the left pane of the window.

 

d)       Under “Sign-in options”, click “Change your password”. If you want to remove the password, you can leave it blank and then click “Next”.

 

e)       Follow the instructions, and then click “Finish”.

 

If you don’t have a local account on the computer, then you can create one without setting a password for it. Follow these steps to create a local account.

 

 

To create a local account, follow these steps:

 

a) Press “Windows Logo” + “C” keys on the keyboard to display the charms bar.

 

b) Click on “Settings” and then click on “Change PC Settings”.

 

c) Select “Users” from the left pane of the window.

 

d) Under "Other Users", click on "Add a User".

 

e) On the new windows which opens click on "Sign in without a Microsoft Account" and then click on the button "Local Account".

 

f) Enter all your details and follow the onscreen instructions to create the local account for the computer.

 

 

For more details, you can refer this link on how to create a local account:

 

http://windows.microsoft.com/en-US/windows-8/create-user-account

 

 

Hope this is helpful. Feel free to get back to us if you have any further Windows related queries and issues.
