ossiemac say:

Can be done with VBA code. I suggest the you use Double Click to invoke the code. Could be done by simply selecting the cell but then you might want to select the cell for other purposes and the code would take you to another worksheet. It is not likely that you will inadvertently double click a cell so best to use Double Click.

To install the code:

  1. Right click the worksheet tab name.
  2. Select "View Code". (This will open the VBA editor)
  3. Copy the code below and paste into the VBA editor
  4. Close the VBA editor (Cross very top right of the VBA editor window)
  5. Use "Save as" to save the workbook and select "Excel Macro-enabled workbook (*.xlsm)" from the dropdown "Save as Type:" field.
  6. Ensure that macros are enabled. See Help for how to do this. (Option to "Disable all with notification" should be OK)
  7. Now when you Double Click a cell containing a worksheet name (Must be in the required column - code is for column A but you can change the column as per the comment in green in the code.) it will take you to cell A1 on that worksheet.

Private Sub Worksheet_BeforeDoubleClick(ByVal Target As Range, Cancel As Boolean)
    If Target.EntireColumn.Address = "$A:$A" Then       'Edit "$A:$A" to required column
        Cancel = True   'Cancels Edit mode that is normally invoked with double click
        On Error Resume Next    'In case cell does not contain valid worksheet name
        Application.Goto Worksheets(Target.Value).Range("A1")
        On Error GoTo 0
    End If
End Sub

word 2013 recent documents list has been deleted - Microsoft Community

jgt say:

word 2013 recent documents list has been deleted

I'm using office pro x32 on my Win8.1 x64 system. For the last several months I've noticed that something is deleting the Recent Documents list (click File > Open). It happens rather infrequently and so far I have not been able to determine what is at fault. I use this function in Word a lot thus I notice the issue reasonably soon after it happens. I just checked Excel and PowerPoint and it appears that the Recent list is also being altered. Any suggestions would be appreciated to resolve the issue.
jay freedman say:

Do you use a registry cleaner such as the one in CCleaner? If so, check the settings in that program.

The Recent Documents lists are held in the registry in keys under

HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Word\User MRU

HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Excel\User MRU

HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\PowerPoint\User MRU

In each of those keys you'll find a subkey named "LiveId_" followed by a long string of hex characters and, under that, a subkey named File MRU. Within that last subkey, the values hold the file paths of the recent documents.

Under the key HKEY_USERS there's a key named "S-1-5-21-" followed by another long string of hex characters, which contains all of the same information as the HKEY_CURRENT_USER section. (There may be more than one of these, one for each user identity on the computer.) This HKEY_USERS data is copied to HKEY_CURRENT_USER when you log in, and changes in one section are reflected in the other.

If a registry cleaner is overly aggressive and is allowed to remove values from the File MRU list in either section, you'll see that change in the Recent Documents list in Office programs. There might be other causes, but cleaners are the most common.
