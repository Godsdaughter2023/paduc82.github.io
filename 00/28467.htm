I guess(!) I know what the problem is. It begins with a mistunderstanding:

No OneNote versions are syncing with each other. Not iOS app with OneNote 2016, not OneNote 2016 with the OneNote App.

Instead ALL versions have to access a central storage location where the notebook is stored and sync with that.

Now: ALL OneNote versions EXCEPT OneNote 2010,2013,2016 for windows need that location to be in the cloud, either OneDrive (Consumer) or OneDrive for Business. Some also may use a Sharepoint library.

The reason why your iOS OneNote and the Windows app are in sync is because they both access the same notebook on OneDrive and you are logged in at both version with the correct credentials to access that OneDrive account.

Obviously your OneNote 2016 is NOT logged in to OneDrive or you are using a different notebook with it where you send your mails to.

You got it right: When you save a mail message from Outlook to OneNote, the Outlook Add-In only "talks" to the desktop version of OneNote (that's an old mechanism from times before notebooks could be stored in the cloud). Is usually also asks you where to store that message (notebook, section, page). Make sure that you are using the same notebook on OneNote 2016 that you use with the iOS and WIN 10 app.

Maybe you have to connect your OneNote 2016 to OneDrive first There is an option in the File menu.

This should be enough to get you started. Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module. Paste the following into the new pane titled something like Book1 - Module1 (Code),

Sub mcrSplit_Funds()
    Dim b As Range, lstFnd As String, rw As Long
    With Sheets("Data")
        For Each b In .Range("B1:B" & .Cells(Rows.Count, "B").End(xlUp).Row)
            If UCase(b.Value) <> lstFnd Then
                lstFnd = UCase(b.Value)
                If IsError(Application.Match(lstFnd, Sheets("Element").Range("A:A"), 0)) Then
                    Sheets.Add After:=Sheets(Sheets.Count)
                    Sheets(Sheets.Count).Name = lstFnd
                End If
                With Sheets("Element")
                    .Cells(Application.CountA(.Range("A:A")) + 1, 1) = lstFnd
                    .Cells(Rows.Count, "A").End(xlUp).Offset(0, 1) = b.Offset(0, 2).Value
                End With
                rw = 0
                With Sheets(lstFnd)
                    Do While lstFnd = UCase(b.Offset(rw, 0).Value)
                        b.Offset(rw, 2).Resize(1, 5).Copy _
                          Destination:=.Cells(Application.CountA(.Range("A:A")) + 1, 1)
                        rw = rw + 1
                    Loop
                End With
                With Sheets("Element")
                    .Cells(Rows.Count, "A").End(xlUp).Offset(0, 2) = b.Offset(rw - 1, 2).Value
                End With
            End If
        Next b
    End With
End Sub

Note that is a new fund (e.g. ESFUT1) cannot be located in the Element worksheet, it is assumed to be new and a new worksheet with its name will be added. If the fund names in Element are not harmonious with the collection of worksheets, this will initially cause problems. The routine assumes that the information in Data has already been sorted and uses this assumption in its loop. I had thought about adding a sort routine but I am still concerned about the actual nature of the date/time despite your assurances.

Tap Alt+Q to return to your worksheet. tap Alt+F8 to open the Macros dialog and Run the macro.
