If Word opens without giving you a found errors message, the problem is more likely  faulty formatting.  Some of the following advice is based on your table really being simple, that is, no merged or split cells, no nested tables, and no inserted objects.

Can you edit the table when in draft mode? If yes, are there any section breaks above the table? A year or so ago, we saw one or two cases where a table behaved oddly when a continuous section break was above it.  The fix was to remove the section break.

Make sure only the first few rows (if more than one is necessary) are marked as heading rows.

If  the table is, indeed, simple, you can convert it to text with  tabs as separators, and then convert it back into a table. 

If the other documents with this problem were created from the same template or document, you should fix that template or document as well.
