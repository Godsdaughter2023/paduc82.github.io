·         Tap the down arrow to the right of "Type"

·         Choose  Word 97/2000 Document or Excel 97/2000 Document, as appropriate



Note: This would create copies of the original file in the new format.

john - project say:

DavidUV,

The resource "limit" is called Max Units and it is set by entering a value in the Max Units field on the Resource Sheet. For example, for a normal workday using the Standard calendar, a resource will work 8 hrs per day so the Max Units is set to 100% (1.0). If the resource is assigned such that his/her hours exceeds the Max Units, Project will give visual indicators that the resource is overallocated.

When in the Gantt Chart view, a task with an overallocated resource will show a little red person in the Indicators field (normally the left most column). You will also see the Resource's name in red on the Resource Sheet.

The threshold for the overallocation warning is set under Resource > Level group > Leveling options.

Just for reference, there is no way to set Project so that you are physically prevented from overallocating resources.

Hope this helps.

John

user_100 say:

John,

Thank you for such a timely reply. Perhaps I didn't state my problem clearly. I don't care how long a resource works daily. What I wanted to do is that when a resource (like an unique machine) is in use for one task, it can't be used for some other task at the same time. Is that achievable?

DavidUV

john - project say:

DavidUV,

Yes that is achievable if you use Project's leveling feature. For example, let's say there are four tasks that use the machine resource. Two that last 5 days and two that last 1 day each. As originally scheduled the machine is overallocated since it is being assigned to work on all four tasks simultaneously.

To resolve the overallocation, the four tasks are prioritized and the leveling options are set as shown

When leveling is applied, the four tasks are arranged in priority order such that the machine only works on one task at a time.

Hope this helps.

John
