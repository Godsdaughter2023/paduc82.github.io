djeremyc say:

How to add a primary key to a table made form a Make Table Query?

I have a query that brings in multiple operations of parts.  From this list I sort the values in ascending order as I think it will allow for better use of expressions in this case.  What is needed is to be able to go to a particular operation (which it does), and then give me the operation previous to the current (aka the record above the current as the query is sorted).

DLOOKUP("[FieldName]","Table","[ID]=" & [ID]-1)

So I thought that if making a make table query I could add a primary key and tell the next nested query to give me the operation with the primary key taht is above the current.

So two parts:
1) Is DLOOKUP the proper way to go about getting the previous record?
2) How do you automate adding a primary key to a make table if needed to get this task accomplished?

john w. vinson say:

I certainly would NOT create a new table for this purpose. For one thing, a Table has no order, so "the previous record" is not a meaningful concept; for another, a Primary Key can often be an Autonumber but the terms are certainlynot synonymous.

All you really need to do is put the syntactically required quotes around the criterion in the DLookUp. Assuming that ID is a field of Text datatype which contains numeric data ("123", "124" with no leading zeros), you could use

DLookUp("[FieldName]", "[Table]", "[ID] = '" & CStr(Val([ID]) - 1) & "'")

If the current record has ID equal to the string "124", the Val() function will return a numeric 124; subtracting 1 and using CStr will return a text string "123"; and the DLookUp search argument will become

[ID] = '123'

If there might be gaps in the ID sequence, you could instead use

DLookUp("[FieldName]", "[Table]", "[ID] = DMax("[ID]", "[Table]", "[ID] < '" & [ID] & "'"))

to search for the largest ID preceding the current record's ID.

John W. Vinson/MVP

Dim strCriteria As String
If IsNull(currentId) Then

The first thing I would check is the size of your bottom margin. If that is too small, bits of the footer will disappear.

For a one-line footer, try a bottom margin of 2.5 cm and set the footer to be 1.25 cm from the edge.  That will place the footer across the middle of the margin in the space you have made for it.

Turn on your non-printing characters and check to see you do not have any stray empty paragraphs in the footer.

a. user say:

In Word 2011, if you just provide a file name, Word looks in the root of the startup drive (or something like that). e.g. in a typical installation, Word will look for your file in "Macintosh HD:Source.doc" . It doesn't look in either the same folder as the including document, or the "curdir" that the VBA function Curdir returns.

As a general rule, you have to provide the full pathname and file name, e.g.
