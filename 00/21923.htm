rod gill say: From help: "Unlike the % Complete field, the Physical % Complete field is independent of the total duration or actual duration values used by the % Complete field to calculate BCWP"

So as well as updating Physical % complete, you need to update Percent complete as well. Physical % complete is only there if a more accurate method of tracking for Earned value purposes is required than just Duration % complete.

frustratedpm say: Thanks Rod. 

I understand the independence angle. It makes perfect sense for me to have an alternative mechanism to track the degree of completeness. 

So the % complete is affected by the "duration completeness" (as a concept),. And so to does "duration completeness" adjust the % complete. 

So the benefit of this Physical % Complete is that it is not affected by any other factors. It is purely and simply a number to be entered by the user. Fine with that. 

But now, what does not make sense: 

The Physical % Complete of a summary task is always zero, and not editable. 
  1. If all tasks have some Physical % Complete, then the summary task is logically a certain amount Physical % Complete. There might be a number of different suggestions on how that number should be calculated, but that doesn't change the fact that it must be something greater than 0. 
  2. Seeing as any number of reasonable suggestions could be made about how the summary Physical % Complete should be derived, it should, in the same philosophy, be left upto the user to type in a number. Except that it is not editable.
