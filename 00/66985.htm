ken blake say:

The term "memory" is used to refer to the place where the computer holds the program and data it is working with at the moment. It's normally random access memory (RAM), but that can be extended with the page or swap file on the hard disk. The word "memory" is never used to refer to the space on a drive (whether that's a hard drive, SSD, diskette, CD, DVD, or thumb drive). Space on a disk drive is normally called "storage" or simply "disk space."

Calling disk space "memory" isn't so much wrong as just unconventional. In a sense, disk space is, of course, memory, but the important thing to realize is that the word is never used that way, and if you do so, you do it at the risk of seriously confusing your audience.

Defragging has nothing to do what how much disk space you are using. It just rearranges the used space.

You say you "cleaned out disk space." Exactly how did you do that?

And perhaps most important, you should note that deleting a file does not reduce the amount of used disk space. It moves the file to the recycle bin, where it continues to take up space.

How large is your hard drive (or drives)? How much of its disk space is used and how much is free? Does the amount of used space go up all the time, or only some of the time? When?  If it goes up all the time, it is likely a malware infection.

咲くやこの蚱 say:

Hello.  I had the same problem and managed to solve it.

The 0x81000033 error occurs only when a system image is included in the ongoing backup.

System image backup fails when the size of the Recovery Partition is insufficient.

In a UEFI/GPT partition style disk, usually it is the fourth partition with its default size of 450 MB.

https://docs.microsoft.com/en-us/windows-hardware/manufacture/desktop/configure-uefigpt-based-hard-drive-partitions

You can see it in the Disk Management utility of Windows 10, but it is shown as the third partition because the second MSR partition is hidden.

Windows will show the total size and used space of the Recovery Partition in the “Which drives do you want to include in the backup?” screen displayed while you attempt to create a system image manually from the upper left side of the “Backup and Restore (Windows 7)” applet of the Control Panel.
Windows Disk Management utility is useless to see the free space of the Recovery Partition because it is always shown as 100%.

To expand the Recovery Partition, you should shrink the preceding Windows partition.  This can be done within the Disk Management utility.

Expanding the Recovery Partition cannot be done within the Disk Management utility.  You should use a third-party utility software such as MiniTool’s Partition Wizard Free Edition.

Shrinking the Windows partition 4670 MB and expanding the Recovery Partition to 5 GB was sufficient to me.

If your PC is a branded product, there may be another Recovery Partition with 10 and several GB.  It is for recovering to factory state and has initial data for it.  You would be advised not to touch it.
