timliu_msft1261 say:

hi deepaksb,

in sharepoint online, the users who can view the task list also can view other users’ tasks because the newly created task inherits the list permissions. you can manually remove other users’ permissions for the task after it’s created (click “… “ next to the task > share > shared with > advanced.

meanwhile, i find the following workaround for your reference. it hides the tasks by modifying the view, but users can still view the other tasks directly via the urls.
you can firstly set the my tasks view as the default one for this task list. please follow the steps below (you need to be the tenant admin or sharepoint administrator).
1.navigate to the task list and click the list tab.
2.click list settings in the right of the ribbon.
3.scroll down the page and find the views section.
4.choose the my tasks view and tick the check box before make this the default view (applies to public views only). note: for setting the view as the default one for mobile devices, you can find it in the same page.
 



 
5.click ok.
6.remove all the other views. in the same page as the pictures shows above, you can find the delete button.

after finishing the setting, your users will get the view which only includes their tasks. as an admin, you can create a personal admin view so you can see all tasks. (click the list ribbon > create view > under start from a view template > all tasks > select create a personal view)

i understand that it may not meet your requirement completely, so in the meantime, i would suggest you submit feedback regarding this feature via uservoice. your ideas will be greatly appreciated.

thanks for your understanding.
