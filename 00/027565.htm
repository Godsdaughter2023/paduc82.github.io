ronrosenfeld say: Hans code is tight, but takes a while to run due to repeated reads and writes from the worksheet itself.  Running his code and my code, on the first run, mine runs twice as fast.  (The difference is greater on subsequent runs, which may or may not be an appropriate comparison).  And the absolute magnitude of the difference may not be significant unless you are dealing with thousands or tens of thousands of rows; or if this has to be run many times.

I offer a different approach, which both highlights the affected rows by changing the interior and font coloring; and also adds a comment in the 7th column indicating which are the matching rows, and the sum of the absolute differences.

Furthermore, it will handle instances where there is more than one matching row.

If your data is not in columns A:F, some of the hardcoded values below will need to be altered.

==================================
Sub IdenticalRows()
    Dim SumDiffs As Double
    Const MaxDev As Double = 0.15
    Dim rData As Range
    Dim vData As Variant
    Dim dRes() As Double
    Dim i As Long, j As Long, k As Long, m As Long
Set rData = Range("A1", Cells(Rows.Count, "F").End(xlUp))
    vData = rData
ReDim dRes(0 To UBound(vData, 1), 0 To 2)
For i = 1 To UBound(vData, 1) - 1
    For j = i + 1 To UBound(vData, 1)
        SumDiffs = 0
        For k = 1 To 6
            SumDiffs = SumDiffs + Abs(vData(i, k) - vData(j, k))
        Next k
        If SumDiffs <= MaxDev Then
            dRes(m, 0) = i
            dRes(m, 1) = j
            dRes(m, 2) = SumDiffs
            m = m + 1
        End If
    Next j
Next i

m = 0
Range("G1").EntireColumn.Clear
    With rData.Font
        .ColorIndex = xlAutomatic
        .Bold = False
    End With
    With rData.Interior
        .Pattern = xlNone
        .TintAndShade = 0
        .PatternTintAndShade = 0
    End With
Application.ScreenUpdating = False
Do Until dRes(m, 0) = 0
        With rData.Rows(dRes(m, 0)).Resize(columnsize:=7)
            .Cells(7) = .Cells(7) & rData.Rows(dRes(m, 0)).Address & "/" & _
                rData.Rows(dRes(m, 1)).Address & " " & dRes(m, 2) & "; "
            .Interior.Color = vbRed
            .Font.Color = vbWhite
            .Font.Bold = True
        End With
        With rData.Rows(dRes(m, 1)).Resize(columnsize:=7)
            .Cells(7) = .Cells(7) & rData.Rows(dRes(m, 0)).Address & "/" & _
                rData.Rows(dRes(m, 1)).Address & " " & dRes(m, 2) & "; "
            .Interior.Color = vbRed
            .Font.Color = vbWhite
            .Font.Bold = True
        End With
    m = m + 1
Loop
Application.ScreenUpdating = True
End Sub

Unless this a really new variant, which is possible, the Blaster worm should not have made it past MSE as it is several years old. For it to have affected your PC, that would mean that your PC is not up to date.

Make sure that all important/critical updates, including service packs for the operating system and programs are installed from Microsoft Update (Windows Update).

Make sure Internet Explorer is at version 8 or 9 and updated with all patches.

In Internet Explorer, use the SmartScreen Filter.

Make sure that IE Internet Security settings are at least set to medium-high (default).

Enable the pop-up blocker in IE.

On Vista and Windows 7 make sure that User Account Control (UAC) ON  and not running with elevated privileges.

Make sure that Windows Automatic Updates are set to at least notify, but the preferred setting is to download and install automatically. If you update manually, be sure to update as soon as possible after being notified of available updates.

Make sure that installed applications, especially Adobe Acrobat, Adobe Flash, and Java are at their latest versions. Many vendors are regularly updating and patching for security holes.

Marusca:

 

You wrote:

"When I click on the Search button it opens up a second form called frmSearch (does this make it a sub-form?)."

 

And I just wanted to make sure you got an answer to that.  No, that isn't a subform.  A subform is a form which is actually ON a parent form and it is housed on that parent form by what is known as a subform control.  When you have a subform then it is USUALLY (but not necessarily always) linked to the parent form by means of the Master/Child links in those properties of the subform control.  The subform control is what handles the interface between the parent and child form and the master/child links is the place which defines which field, or fields, are actually the link between them.
