Hi bushva,

 

You can't have two AV/AM/AS security programs providing real-time protection (whether active or not) installed at the same time.  You can't have any two AV programs installed at the same time.  PA Bear's references cover most of this.  But that doesn't mean there aren't more ways to improve your security or increase your backup of the primary real-time security product.  For most people, MSE alone will be just fine, but if you want more, then read on:

 

Here's what I do (and it isn't 100% but nothing is, and in some cases only catches things after they've already gotten into the system, but it's a good process while remaining reasonable and they can all work together - as proven by the fact that I use them).  You can use some, none, or all and you can alter the timeframes to whatever makes you the most comfortable.  As I tend to do some higher risk things due to my job, I feel I need the extra protection and scanning - but if you're an average user, you probably don't need all of them and can probably do them less frequently.  But you should do the bold part as that applies to everyone and even though it's not programs per se, it's every bit if not more important than the programs because while the programs help defend against and remove infections, the real trick is to avoid them in the first place and that section covers good habits which may be more useful than anything else provided.

 

I use MSE for real-time protection and run scheduled daily quick scans with a scheduled bi-weekly full scan.

 

I update and periodically (every couple of weeks or so) run full scans with Malwarebytes: http://www.malwarebytes.org/ and SuperAntiSpyware: http://superantispyware.com/ and ESET online scanner: http://www.eset.com/us/online-scanner Get only the free versions. Any that also have paid versions include features like real-time protection that cannot be used along with MSE anyway (in these cases), so there's no point in spending the money anyway.

 

I use Secunia PSI http://secunia.com/vulnerability_scanning/personal/ to make sure all my programs are updated and current (since updates often plug security holes in the software and thus can be important as many people never check and/or update their software unless prompted - and this way I don't need to check everything since notifications and sometimes updates are automatic).

 

I always follow the following advice from Stephen Boots - MVP:

 

-Make sure that the Windows Firewall is enabled. No other firewall truly offers better protection or is needed. This one is perfectly fine.

 

-Make sure that all important/critical updates, including service packs for the operating system and programs are installed from Microsoft Update (Windows Update).

 

-Make sure Internet Explorer is at version 8 or 9 and updated with all patches.

 

-In Internet Explorer 8 or 9, use the SmartScreen Filter.

 

-Make sure that IE Internet Security settings are at least set to medium-high (default).

 

-Enable the pop-up blocker in IE.

 

-On Vista and Windows 7 make sure that User Account Control (UAC) ON and not running with elevated privileges.

 

-Make sure that Windows Automatic Updates are set to at least notify, but the preferred setting is to download and install automatically. If you update manually, be sure to update as soon as possible after being notified of available updates.

 

-Make sure that installed applications, especially Adobe Acrobat, Adobe Flash, and Java are at their latest versions. Many vendors are regularly updating and patching for security holes.

 

-Never click through links from unknown sources and use caution even if they are from a "trusted" source.

 

-Never open unsolicited email attachments.

 

-Practice safe web browsing.

 

-Always use a standard user account unless you have a specific need for an administrator account.  Then switch, do the administrative task, and return to the standard account.  This greatly limits the access of any type of infection of intrusion into your system since administrative access isn’t in place.

 

I hope this helps.

 

Good luck!

Hi Virfanz, I work on the development team Amy mentioned in her opening post.

To answer your questions, the concern is that we have users who pass links around their companies using office products (Excel and Word from the cases we've seen so far) and they are confused as to why the links work when put into a browser manually but not when followed through from the office doc.

I'm not entirely sure when the issue started, but the first case we have seen for this surfaced around January this year,

and we have replicated this issue in house using both office 2016 for business and office 2010 (I assume this means it will affect all interim versions as well)

From analysing the network traffic generated from Excel (I assume the same process is followed from word) what looks to be happening is that when a hyperlink is clicked in office, it sends out the http request to follow the hyperlink in the background and see's whats returned, it then takes whatever the url that is returned (after any redirects or anything that are done in the first request) and opens that in the browser.

This causes an issue for us as the hyperlink points to a page that authorises the request and redirects to the appropriate home page. So the initial request goes to auto.aspx to authenticate but upon successful authentication the request is redirected to a search page in this instance which is a different url to the one in the hyperlink.

However, because the first request is sent outside of the browser, any session authentication cookie's returned are thrown away and are therefore not present for the browser to attach to the subsequent request for the home page.

This results in a request to access a page that requires an authentication token to access being sent without the token and therefore being bounced back to the login screen.

If you have a browser open that you have recently opened one of the urls in correctly (either by pasting in directly or following from an email link as in Doug Robbins response) this leaves the session cookie in the browser and any further attempts to navigate using the office hyperlinks will succeed as it uses the still active cookie from the previous session.

I hope this helps you figure out a solution for this :)

David

A macro containing the following code will color all of the empty cells Red in all of the tables of the document that is created by executing the merge to a new document (Edit individual documents)

 

Dim i As Long, j As Long, k As Long
With ActiveDocument
    For i = 1 To .Tables.Count
        With .Tables(i)
            For j = 1 To .Rows.Count
                With .Rows(j)
                    For k = 1 To .Cells.Count
                        With .Cells(k)
                            If Len(.Range) < 3 Then
                                .Shading.BackgroundPatternColor = wdColorRed
                            End If
                        End With
                    Next k
                End With
            Next j
        End With
    Next i
End With

steve bobbitt say:

Sorting Error

I'm building a two column table in Word 2016. The first column uses number in the format such a 243-8, 281-29, 5433-1, etc. When I sort on this column I have selected number and descending selected. The table sorts numerically as desired about halfway through the document, then it goes into a brain fade and quits sorting numerically. Here's a link to show the problem. Thanks.

https://1drv.ms/w/s!AsXn-Vm9qsqdgyAv3d8ucLOtVCgT

jay freedman say:

The problem is that you used a hyphen to separate the numbers. When you sorted the column as numbers, Word interpreted the hyphens as minus signs. The table is correctly sorted in descending order by the "sums", such as 248-3 = 245 and 281-29 = 252.

To sort the way you expected, you need to temporarily replace the hyphens with a character that Word won't interpret as part of any kind of calculation, for example the letter Q. After the sort is complete, you can change the character back to a hyphen.

To confine the change to the entries in the first column, use a wildcard search in the Replace dialog. First, click the More button to expand the dialog, and check the box for "Use wildcards". In the Find What box, enter the expression

([0-9]@)-([0-9]@)

which will match any two multiple-digit numbers separated by a hyphen. In the Replace With box, enter the expression

\1Q\2

which will result in the same two numbers separated by the letter Q. Click the Replace All button.

Run the sort on the first column, as numbers, descending.

Reopen the Replace dialog, change the expressions to ([0-9]@)Q([0-9]@) and \1-\2, and click Replace All.
