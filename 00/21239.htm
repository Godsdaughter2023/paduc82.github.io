bill mosca say:

Kim

You can't change the owner of the system tables owned by Engine. As you have seen, not even Admin can own them in an unprotected DB.

But as a member of the Admins group you should have read & update permissions. Updates must be done through code as you cannot directly edit those tables.

You can still change the permissions using 2007 as long as the database is an MDB. If it is secured and coded to disallow all the database properties like the Bypass key you will have to use a utility like my Properties Setter (http://thatlldoit.com/utilsaddins.aspx) to set the Allow Bypass key.

Once you have it open, you should see the Database tools tab on the Ribbon. The User Permissions is on that tab.
