a_user_here say:

There is no such thing as a "universal service pack".  A lot of things have changed in the last 20 years, and there is no way that Microsoft can keep ever program that ever was created to keep working.

One big stop might be that you might be running Windows 10 64-bit (note there is a Windows 10 32-bit version too, but it will be limited to 4 GB of memory).  A 64-bit operating system can run 32-bit programs and 64-bit programs.  A 32-bit version can run 32-bit and 16 bit programs.  Windows 95 was a 32-bit operating system, and it is entirely possible the program you want to run is a 16-bit program.

If you just have to run these old programs, your best bet would be to use some virtual machine software to run say Windows XP, which should be able to run that Windows 95 program.

Phil,

Check in the BIOS to see whether your system has lost time and date. This would indicate that the battery needs to be replaced.

Usually, when that happens the BIOS reloads defaults. If one of those is something you would have disabled like onboard sound for example if you installed your own sound card, then Windows will find the onboard sound when the machine boots up.

Check the BIOS anyway for something similar even if time and date hasn't been lost.

Also, the NVIDIA drivers may not be WHQL-certified. This can cause system instability. To check whether there are any non-WHQL drivers installed, do the following.
  1. Open a command prompt, type CD\ abd hit Enter to take you to the root.
  2. Next, type: sfc /scannow and hit Enter.

When you run that command, the operating system may request the Windows installation CD so have that ready to hand.


If System File Checker finds a corrupt protected system file which it can't retrieve from the C:\WINDOWS\SYSTEM32\dllcache repository, it will extract it from the Windows CD. This may take place a number of times. The process isn't automatic and you'll need to click the "Retry" button each time. The last time I ran it, it took over an hour to complete.


You can hit Enter instead of clicking the Retry button with the mouse, but if you accidentally hit Enter after the "Retry" button disappears, the "Cancel" button which is enabled by default may cancel the operation without prompting you first so do bear that in mind.


If SFC finds non-WHQL graphics drivers, it replaces them with the standard VGA drivers. This means that when you reboot, or shutdown and then power on again sometime later, the screen resolution will revert to 800x600.


To address that issue, you'll need to uninstall and then reinstall the NVIDIA drivers.


The current WHQL-certified drivers for most graphics cards running on Windows XP is version 327.23


As regards the Stop error you're seeing, see this article: http://support.microsoft.com/kb/315335/en-gb


I don't know about the blank one though. You could check in Event Viewer | System menu for clues.

This process builds and updates the Index catalog and typically uses a lot of CPU time. However, cidaemon .exe is also set to run at a low priority, so it usually doesn't degrade the performance of the system even if it's at a high CPU level.

abraham j say:

Hi Blacksheep1,

I checked if the game is compatible with vista, looks like it is.

Ref: http://www.2kgames.com/civ4/support_msr.htm

D3dx9_32.dll errors typically appear when a software program, usually a game, is started. Occasionally, d3dx9_32.dll errors will display after a game is loaded but right before game play begins.

This application has failed to start because d3dx9_32.dll was not found.

These are all optional components to DirectX that may not be installed with the version of DirectX you already have on your system.

The d3dx9_32.dll file is one of many files contained in the DirectX software collection. Since DirectX is utilized by most Windows based games and advanced graphics programs, d3dx9_32.dll errors usually show up only when using these programs.

Usually these will be installed when the game installs DirectX, however these optional components will be provided in the directX SDK, and as such may not be installed with the version of DirectX present on your system.

Follow the steps below to fix this issue:

1.        Restart your computer if you haven't yet done so.
It's unlikely this will fix the problem, but restarting is always a good first troubleshooting step.

2.        Click here to install the latest version of DirectX SDK. Chances are, upgrading to the latest version of DirectX 9 will fix the d3dx9_32.dll not found error.

3.        If latest DirectX SDK doesn't fix the d3dx9_32.dll error, look for a DirectX installation program on your game or application CD or DVD. Usually, the software developers will include a copy of DirectX on the installation disc and is a better fit for the program than the latest version available online.

4.        Uninstall the game or software program and then reinstall it again.

5.        You can also try to restore the file “d3dx9_32.dll “ by extracting the DirectX files to a separate folder.

6.        Copy the DLL file to the system32 folder located in your Windows installation folder. On most computers, that will be C:\Windows\System32.

7.        Restart your computer.

Reply back and let us know if this helped.
