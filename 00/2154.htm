The code below is designed to work with the double click event.

It has an enhancement to ask the user to confirm changing the value if the cell already has a previous value. However, if you don't want this enhancement then delete the code between the asterisk lines (or just comment it out to disable it with a single quote at the beginning of each line and then if you decide later that you do want it then you only need to uncomment it.)

Right-click the sheet tab name and select View Code.

Delete the code provided by Hans or it will interfere with the following code.

Copy the following code into the worksheet module.

Private Sub Worksheet_BeforeDoubleClick(ByVal Target As Range, Cancel As Boolean)
    Dim response As Variant
   
    If Not Intersect(Range("AF23:AF375,AH23:AH375,AJ23:AJ375,AL23:AL375"), Target) Is Nothing Then
        Cancel = True
        Application.EnableEvents = False
        '*****************************************************************************
        If Target.Value <> "" Then
            response = MsgBox("Confirm to change the value of this cell.", vbYesNo)
            If response = vbNo Then     'Do not change
                GoTo ReEnableEvents     'End the processing
            End If
        End If
        '*****************************************************************************
        Target.Value = Date         'Enter today's date
    End If
   
ReEnableEvents:
    Application.EnableEvents = True
           
End Sub

You are going to have do it in stages

First, try typing St for look up and Street for replacement be sure to leave on character blank space just after the T in ST to prevent the issue you are running into.

 

bob jones aka: cybertaz say:

The F&R engine in Mac Excel simply does not have the ability to do an Exact Match for partial cell content. If you have access to a recent PC version you may be able to do that type of replacement. Otherwise, in Mac Excel I believe it would require the use of VBA... Perhaps one of the coders around here will be able to provide the procedure you need.

Another option is to export the data in a Word-compatible file type, Word's F&R is capable of doing this type of Exact Match replacement. Then import the result back into Excel & resave using an Excel file type.
