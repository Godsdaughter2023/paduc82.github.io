wunders say:

There should be nothing random about it.  The key information that you have not provided is whether or not "Simple File Sharing" is enabled or not and whether the "Guest" user is active or disabled and/or whether the Guest user has a password on each of your two machines.

 

If you have XP Home, then Simple File Sharing is force-enabled.  If you have XP Pro, then Folder Options will tell you whether or not it is enabled.  With Simple File Sharing enabled, every connection is forced to authenticate as the "Guest" user.  So the quick answer to get what you want is to make sure that "Simple File Sharing" is enabled, your Guest user is not disabled, and there is no password on the Guest user on both machines.  Then any user connecting to either machine will authenticate as the Guest user and have access to all files and folders accessible to the Guest user.

 

If Simple File Sharing is disabled on the machine that you are connecting to [target machine], then, IIRC, authentication roughly follows this path:

  1. If the target computer has a user name and password that matches the user/pass that you are logged-on to from the source computer, the connection automatically goes through.
  2. Else, If the same user exists on the target machine but the password doesn't match, then the connection is denied.
  3. Else an attempt is made to authenticate as the Guest user with no password.  If the Guest user is active and not passworded, the connection is made.
  4. Else if the Guest user is not active (i.e. is disabled), then it prompts for a username/password

So, to me, (unless my memory skipped a step above), if you are getting propted for a username and password, that tells me that the target machine has Simple File Sharing disabled as well as the Guest user and you are connecting from a username that does not exist on the target machine.

 

"How to use the Simple File Sharing feature to share files in Windows XP"

   < http://support.microsoft.com/kb/304040/en-us >

"How to disable Simple File Sharing and how to  set permissions on a shared folder in Windows XP"

   < http://support.microsoft.com/kb/307874/en-us >

 

To make sure the Guest user is not disabled on a given machine, as an "Administrator" user bring up a command prompt window:

   Start -> Run -> "cmd"

and enter the command:

   net  user  guest  /active:yes
