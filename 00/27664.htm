If you feel that your account has been compromised, the very first thing you should do is visit http://account.live.com and change your password.

 

If you wish to obtain IP login history, we can make that available, but for your security we cannot release that information to anybody that has not been validated as the account owner.

 

To start the request process, please visit http://account.live.com/password/reset

Select I forgot my password from the first screen, and select "Reset Your Password"

Enter the Microsoft account that you wish to obtain the login history for and solve the HIP check then click "next"

From the next screen select "I can't use any of these options"

This will take you into the Automated Validation Process.

Follow the prompts and fill out the information as fully as possible.

Once you submit, you will either receive a response almost immediately if you auto-passed

If not, you will receive an automated e-mail with an issue number, and within the next 24 hours you will receive either notification of a pass or a fail, again with the same issue number.

(Important...in both passed scenarios, there will be a link.  Do not click this link unless you actually want to change your password)

Once you have either the pass message, go to the answers forum:

Hotmail, Messenger & SkyDrive>Microsoft account>Security and Permissions

In that forum post your question, I need to obtain IP log history

In the body of your question, explain that you are concerned that your account has been compromised and you wish to obtain IP log history.  Please do NOT post the message you obtained above.

One of our moderators will respond to your post with a private message and ask you to provide either the e-mail if you auto-passed (make sure and include the headers so we have a date stamp), or the incident number provided in the other mails.

Once the moderator has proof that you are the owner of the account, they will respond with the requested history.

 

Note that IP logs are the only information we have available.  We cannot provide information if mails were read or any other activities that may have occurred.

 

Please also note that the steps above is the only way that we can validate ownership, and without that validation we cannot release any information to anybody.  This is for the protection of all our customers.

karl donaubauer say:

Hi,

the fixed 2 decimal places for numeric fields with potential decimals are an ever lasting problem. There's no way to change the setting.

The usual workaround is to use a query with all the fields you want to export and the use of the Format() function for the problematic fields to cut existing decimals off + cast the number to a string and such turn off the 2 decimal automatism. However in your case this would lead to surrounding quotes in the text file.

If the number type of the field is (Long) Integer then Access exports it without decimals. If you do not want to change the number type in the table this leads to the next workaround. You can use the CLng() function in a query to change the numbers to Long Integers. However if there are records without a value CLng() leads to an error that inhibits the export. So you would end up with an expression like this:

iif(Not IsNull([MyField]),CLng([MyField]))
