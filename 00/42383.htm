a. user say:

Okay - when you boot on your XP installation CD choose to enter the XP Recovery Console, then you should be at this prompt:

C:\WINDOWS

We will replace your missing or potentially afflicted c:\windows\system32\userinit.exe file with a copy that is in another place on your system.  If the userinit.exe file is missing or afflicted you will not be able to login in any mode and will have the symptoms described of the login starting and then being logged out.

From the Recovery Console prompt, type in this command:

copy c:\windows\system32\dllcache\userinit.exe  c:\windows\system32

Respond in the affirmative to overwrite the existing file if asked.

You should see a message that 1 file(s) was copied.

If you have a genuine bootable XP installation CD installed and the CD is drive X, you can expand the file from the i386 folder on the CD into your c:\windows\system32 folder by entering this command:

expand x:\i386\userinit.ex_  c:\windows\system32

Respond in the affirmative if asked to overwrite the existing file and then you should see a message that 1 file(s) was expanded.

Describe how the copy command worked or how it did not work for you when you are done.  There are some situations where we might have to do something else to replace the userinit.exe file, but this is the easiest and will work even if you didn't happen to have an XP installation CD (lucky you!).

Remove the bootable CD and enter 'exit' to restart your system.
