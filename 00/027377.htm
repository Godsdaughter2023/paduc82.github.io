SMTP authentication for outgoing emails via a Linux based Postfix - Microsoft Community

john dammsørensen say:

SMTP authentication for outgoing emails via a Linux based Postfix server

Hi,

I am trying to configure my colleague's Windows 8 phone to use our Postfix SMTP server for outgoing mail.

We use authentication and TSL (SSL) and our server currently supports AUTH PLAIN (and AUTH=PLAIN).

This means that a client that want to use the server for outgoing mail must send the following via SSL:

EHLO <name of client>
AUTH PLAIN <a base64 encoded string of the user name and password>

Apparently the WIndows 8 phone is using another type of authentication, but so far I have been unable to find any documentation decribing what kind of authentication the phone is relying on (LOGIN, PLAIN, CRAM-MD5)


Best
John
john dammsørensen say: In lack of response I will allow myself to provide the answer.

Searching the Internet for possible authentication schemes one very useful site shows up:
http://www.fehcom.de/qmail/smtpauth.html
This page describes in much detail how the various authentication schemes work, but more importantly it also has a table describing what mail clients generally use to authenticate themselves. One thing that sticks out is that Microsoft products seem to mainly rely on LOGIN authentication for outgoing mail.

Reconfiguring our Postfix server to also provide LOGIN as authentication mechanism solved this problem for very many mobile devices.

What people see when the mail server doesn't support the wanted authentication scheme is that it will immediately disconnect after sending the EHLO command to the server and the reception of the server's list of capabilities.

After the initial EHLO xxxxx command the server will respond with:
ehlo xxxxxx.xx
250-yyyy.zzz.www.qq
250-PIPELINING
250-SIZE 151200000
250-VRFY
250-ETRN
250-AUTH PLAIN LOGIN
250-AUTH=PLAIN LOGIN
250-ENHANCEDSTATUSCODES
250-8BITMIME
250 DSN

If the Windows 8 client doesn't see an AUTH line containing LOGIN it will drop the connection and leave the mail in the outgoing folder. Once the LOGIN is enabled people will see that mails long forgotten in the the outbox suddenly goes through.

Log entry from Postfix during EHLO and disconnect of the connection.
Anonymous TLS connection established from 80-62-116-244-mobile.dk.customer.tdc.net[80.62.116.244]: SSLv3 with cipher RC4-SHA (128/128 bits)
 lost connection after EHLO from 80-62-116-244-mobile.dk.customer.tdc.net[80.62.116.244]

Sub ReplaceTextAndKeepHyperlinks(oShp As Shape, FindString As String, ReplaceString As String)

Dim oTxtRng As TextRange
Dim oFRng As TextRange
Dim oRRng As TextRange
Dim bHasLink As Boolean

If oShp.HasTextFrame Then
        If oShp.TextFrame.HasText Then
            Set oTxtRng = oShp.TextFrame.TextRange

20yrexcelvet say:

I'm having trouble getting Excel charts to display the correct data.
With some troubleshooting time I've figured out what is wrong. As most people would expect, it was an operator issue. It turns out that I was using two different ranges, e.g. C2-C24 and D8-D24 for my Primary and Secondary vertical axes respectively. Then, assuming the Horizontal axis would map the data in the rows equally, I restricted the Chart Horizontal axis to the last 12 rows of data (A13-24) and this caused the caused line on the secondary axis to shift Right by 7 or so units.

When I reduced the Primary Vertical axis data to C8-C24 and the Horizontal Axis data to A8-A24, this resolved the misrepresentation.

I think the problem I had was in my assumption that "the Horizontal axis would map the data in the rows equally".
