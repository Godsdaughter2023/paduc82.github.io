chrtod say:

I had the same type of problem and found that just one sheet was "to blame" for all the extra Mbs. I discovered this by making a copy and gradually deleting sheets and saving the newly named version. Alternatively,  copy sheets one by one to a new file and see when that one gets large.

 

Then I tried to find out what was so "special" about this sheet. It was not the contents of the sheet, but the formating. I removed all cell contents from that sheet and it made no difference. Then I copied the formating from another sheet into the "culprit" sheet and the file size returned to what I had. I do not know what was suddenly "wrong" but I guess some formating was covering the whole sheet which are vastly bigger than in my previous excel version (1048576 rows vs 65536).

 

So in my case I could resolved the problem. It would be interesting to hear if you find the same thing.
