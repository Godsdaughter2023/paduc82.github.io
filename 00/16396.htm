Hi, 

 

This is actually strange. What happens is as follows :

 

A subscriber, when he wants to place a voice call, this call is routed from CS (Circuit Switched) networks rather than PS (Packet Switched), as VoLTE(Voice over LTE) is not deployed in most telecom vendors yet. This process is called CS fallback. When the voice call ends, the subscriber is then again connected to the 4G LTE network for packet data.

 

Now, if CS falllback is configured in the network, it means all the services related to voice like call waiting etc should work normally.

 

Since, it is not working for you, that means O2 has a pure LTE network with support for VoLTE. I have no knowledge of O2 network, so it is difficult to comment upon.
