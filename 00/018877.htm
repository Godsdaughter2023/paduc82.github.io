5.6  Why isn't album art displayed when I play a song? There are several reasons why album art might not be displayed when you play on song on your device. In order for album art to be displayed:

bob jones aka: cybertaz say:

It's wholly accurate in response to the question & software to which it applies...

Neither this Community nor this specific conversation addresses Excel 2013 which is a Windows version. This venue pertains to Excel for Mac & the thread refers explicitly to Excel 2011, which does not have any such 'intermittent defect'. There may be such an adversity afflicting the 2013 release but if so it was introduced in that version (long after the release of Office 2011) & exists only there  -- but I don't think it's the defect you believe it to be.

The impression that you can't delete a single cell usually is caused by whether the cell you're trying to delete is within a defined Table. If so, you cannot delete just the one cell because the remainder of the row or column need to be shifted to fill the gap -- that shifting is what I explained in the reply that you chose to contradict. The menu commands & Ribbon tools are contextual, so they change accordingly.

Since Tables are used [almost exclusively] for storage of data records, the shifting of cells would result in scrambling the data. The Table feature is, IMO, well designed to prevent that from happening, so when you attempt to delete a cell in a Table you're provided only with the options to delete the entire table row [i.e., record] or the entire table column [i.e., field] from the table. BTW: Rows & columns outside the range of the table boundary are not affected, the deletions is constrained to the Table. (In earlier versions the operation was virtually identical if the List feature had been applied to a range.)

If the cell to be deleted is not within a Table the Delete> Cells feature works as it always has.
