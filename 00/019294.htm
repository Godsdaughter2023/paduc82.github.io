hansv mvp say:

The usual way is to use the BeforeUpdate event of the text box and to set


Cancel = True


if the value entered by the user doesn't meet your requirements. That will keep the focus set to the text box, but that's what you want, don't you? The user shouldn't be able to move to another control as long as the value isn't correct.


As for e-mail addresses: a simple test would be that the value

- doesn't contain spaces

- contains one @

- contains a . somewhere after the @


For more robust checking, you could use regular expressions, but that's more complicated.


If you want to use the KeyPress event to limit entry to valid numbers, you must specify what type og numbers you want to allow:

- non-negative whole numbers (0, 3, 765, ...) only

- whole numbers, both positive and negative (-25, -1, 0, 37, 218, ...)

- numbers with or without decimal places (-0.25, 3.141592, 912, ...)

In the latter situation, you have to take the decimal separator into account: some people use . and others use ,

There are three things to select when it comes to matching the design:
