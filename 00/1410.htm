graham mayor say:

With a little lateral thinking it is possible to perform this change and toggle the setting using a macro http://www.gmayor.com/installing_macro.htm   

Option Explicit
Sub ToggleWPJustification()
    On Error GoTo err_Handler
    With ActiveDocument
        .Save
        If Len(ActiveDocument.Path) = 0 Then
            MsgBox "Document must be saved before using this function"
            GoTo lbl_Exit
        End If
        If Val(Application.Version) > 14 Then
            If ActiveDocument.CompatibilityMode = 14 Then
                .SaveAs2 FileName:=ActiveDocument.FullName, CompatibilityMode:=Val(Application.Version)
                MsgBox "Justify like Word " & Application.Version
            Else
                .SaveAs2 FileName:=ActiveDocument.FullName, CompatibilityMode:=14
                .Compatibility(wdWPJustification) = True
                MsgBox "Justify like Word Perfect"
            End If
        End If
    End With
lbl_Exit:
    Exit Sub
err_Handler:
    MsgBox Err.Number & vbCr & Err.Description
    Err.Clear
    GoTo lbl_Exit
End Sub

The macro simply saves the document with Word 2010 compatibility and sets the WP justification, or if already in that mode, toggles it back to the default justification (in the following illustration that being Word 2016). dirk goldgar say:

Using VBA, I want to concantenate two strings together, with 2 carriage returns between them, and display this result in a text box on the form. For example:

Jumped Over The Lazy Dog"

 

If the textbox control on the form has its Text Format property set to Rich Text, the output will be:

 

"The Quick Brown Fox  Jumped Over The Lazy Dog" (Instead of two carriage returns between the strings, the result is two spaces between the strings)

 

How can I get this to work properly when using a textbox who's Text Format property is set to Rich Text?


The actual value of a rich text box is more than what is shown.  It uses a limited set of HTML tags to represent the text format.  You could assign your desired three-line value to it in a couple of ways:

(1)

Me.MyTextBox = "<div>" & "The Quick Brown Fox" & "<br><br>" & "Jumped Over The Lazy Dog" & "</div>

(2)

Me.MyTextBox = "<div>" & "The Quick Brown Fox" & "</div><br><div>" & "Jumped Over The Lazy Dog" & "</div>

 

My tests suggest that Access tends to favor creating extra "<div>"s  -- as in (2) above -- over sticking in breaks ("<br>") when it formats the text you type, but method (1) also seems to work.
