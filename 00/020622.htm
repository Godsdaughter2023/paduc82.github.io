jay freedman say:

There are a few changes in the buttons and keystrokes from Word 2007 and earlier, but the same basic procedure still works.

First, Ctrl+F now opens the Navigation pane and puts the cursor in the search box. But that pane doesn't offer a command to Ctrl+H to open the Replace dialog, and then Alt+D to select the Find tab. (There is a down arrow to the right of the search box in the Navigation pane and one on the Find button in the Home ribbon, for a menu that includes Advanced Find, but it's slower to use the mouse to get there.)

It probably doesn't and it wouldn't work if you tried.  That formula, which presumably would be put into some cell on row 2 (but not in column A) should read as:

=SUM($A$2:A2)

As you fill that formula down the column the $A$2 will remain fixed, while A2 will change to A3, A4, A5, etc. down to whatever row you stop filling it into.  Does a nice job of giving a running total for a single column of information.

joeu2004 say: Eric wrote:

One column is gender with letter arguments of either "F" or "M" while the other column has corresponding heights in centimeters (number arguments).  I'm trying to use the ttest function to generate a p-value.  The function I entered is "=TTEST(A2:A65, B2:B65, 2, 3)" with the A column having the categorical variable F or M, and the B column having the numbers.


With your data organized in that fashion, the correct use of TTEST would be the following array-entered formula (press ctrl+shift+Enter instead of just Enter):


=TTEST(IF(A2:A65="f",B2:B65), IF(A2:A65="m",B2:B65), 2, 3)


In the Formula Bar, the entire formula should appear between curly braces; that is, {=....}.  If not, you forgot to press and hold Ctrl and Shift while you press Enter.

It might be easier if the data in column B were separated based on gender.

[ERRATA] Removed misinformation about TTEST requiring same-sized ranges.  That is only for a "paired" test (type=1).

Eric wrote in response to JP:

I tried your method, and it did return a numerical value instead of an error.  [...] Yet the value is 6.07*10^(-84).  -84!  This does not seem reasonable.  P-values are not supposed to be this small.


Commonly selected p-value acceptance and rejection limits are 0.1, 0.05 and 0.01.

But calculated p-values (a probability) can be as small as zero, mathematically.

That said, I would agree that 6E-84 seems unusual.  It's probably just a case of GIGO.  In my experiments with your design using my own random data, I have seen TTEST results as small as about 4E-11 (purposely different distributions for each gender).  But that does not mean it cannot be smaller.

If you want help with using and interpreting TTEST results for your data, you should upload an example Excel file to a file-sharing website, and post the "shared" (public) URL in a response here.
