rob koch say:

I'm not telling you that you shouldn't be as careful as you prefer to be, only that you need to determine the context of what you're doing to see whether it makes any sense in terms of the operation of Windows today and the realities of how much malware you truly experience.

What I've found is that only a small percentage of those worrying about such things ever truly experience malware, while instead they are typically fighting the operation of their antimalware, often due to either past damage to the operating system or antimalware components from a long past installation.

I personally had never performed an operating system upgrade before Windows 10 (from Win7) for this very reason and only did this last upgrade due to the lack of an available clean install process during the free upgrade.  I don't know if this had anything to do with my own maintenance scan issues after the Anniversary Update, since that only affected certain systems, but after testing with Eicar I found no issues with real-time protection so I simply did an occasional manual quick scan until the Creators Update apparently fixed it.

I never experienced any of the other issues you mentioned, but if they'd bothered me at all that would have been sufficient for me to decide to finally perform the clean install I never did.

Windows 10 is specifically designed as a test bed for the aggressive changes required to move Windows from its past into a more secure future, so it was obvious to most technical users that the rate of change due to upgrades would be aggressive.  This means that any long-term legacy issues within an individual system are even more likely to cause problems than in previous operating system updates.  Problems that affect small numbers of systems are typically side issues caused by third-party code, not a problem within the Microsoft code itself.

Please note that whatever Malwarebytes does has no bearing on Microsoft, unless of course they find something within the operating system itself that's truly not working as designed and documented.

I'm not saying that the Microsoft code is perfect and doesn't have issues, since the more recent bugs in the initial release of the Windows Defender Security Center, especially the notification choices made that appear to be annoying many, have been at lest partially caused by design decisions or simply moving too quickly with radical changes.  I'm simply stating that you need to determine whether these are truly causing issues with the protection of your system or only minor annoyances with visual display or other cosmetic issues.

So far all I've seen personally have been cosmetic or minor irritations like needing to perform occasional quick scans myself.  Since as a past MVP I'd personally talked to the original designers of Windows Live OneCare and the later MSE from which Windows Defender was created, I understand that manual or even automatic scanning is virtually unnecessary today, so these irritations never truly bothered me once I'd confirmed the real-time protection was otherwise operating fine.

I understand why someone with no knowledge how to confirm such things might get concerned when the scans, which are the only significant control they appear to have any longer, don't appear to be happening as they should.  But in reality, the user hasn't had any real control over any of the operations of a modern antimalware for many years, as all of these are now background operations performed by the security application and operating system, with only the initial trigger optionally in apparent control (really a request for service) of the user himself.

An afterthought I just had before posting this, are you aware that the Windows Defender client pre-scans every executable within a folder upon browsing into it with any file explorer application and also due to any attempt to execute an application by clicking either its icon or other selection mechanism?  This process is only skipped if the scan cache contained within Windows Defender indicates the file has already been scanned since the last time the system was restarted, since this can't be trusted once the operating system has been shutdown.

This is why I have no real concerns about scans of any sort, since the Windows Defender clients have been operating this way for at least 2 versions now.  To prove this to yourself, download copies of the Eicar test file into various folders while both Defender and SmartScreen are disabled (possibly also MBAM?) since these will both block Eicar, then browse into the folders with them re-enabled.

mike h.. say:

Hi,

 

I'm afraid I got lost in trying to understand all your data layout but it seems clear that what you need is to use the INDIRECT function. Indirect is used to turn a text string in a cell into a range reference.

 

You have this in X3 derived form your formula

 

$R$4:$S$10

 

This formula uses that text like this

 

=VLOOKUP($D6,INDIRECT("'Songs Vs Dates'!" & X3),2,FALSE)

 

and gives exactly the same result as this formula you posted

 

=VLOOKUP($D6,'Songs Vs Dates'!$R$4:$S$10,2,FALSE)

 

You can drag the indirect formula so (say) X3 becomes Y3 or down so X3 becomes X4
