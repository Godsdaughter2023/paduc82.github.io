john - project say:
Bert,
Joe is right, Project isn't really set up to handle time and a half and double time that you typcially find in a normal hourly pay base. However, there is a partial workaround if you only need the two overtime rates for 12 weeks or less. Why 12 weeks? Because for each resource cost rate table (i.e. A through E) a user can enter up to 25 time based pay rates. The "A" table represents what you see as the normal standard rate and overtime rate.

So, here's what you can do. Set up a double time rate in cost rate table "A" to be effective the first Sunday you need double time. On the next line set the overtime rate back to one and a half starting on Monday. Then repeat that sequence for each Sunday and Monday in the ensuing week.

By no means perfect or even convenient, but it might just take care of your issue.

You could extend it even further by using the other 4 cost rate tables, but that will get real messy because you would have to keep track of which cost rate table is associated with each assignment, and . . .  you really don't wanna mess with that.

Hope this helps.

John
