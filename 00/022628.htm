Nope, this functionality was never available in any previous version of Microsoft Project, nor is it available in the 2013 version.

The easiest way to export the Resource Usage data to Excel would be to build your own custom Visual Report for Excel that would export the data for you each time you run the Visual Report.  Create a new Visual Report template for Excel using the Assignment Usage data.  In the Excel PivotTable, add the Resources and Task dimensions to the ROWS drop area, add the Time dimension to the COLUMNS drop area, and add the Work total field to the VALUES drop area.  After you set up your data in Excel, you can save the file as an Excel template (*.xltx) file in your default Templates folder, which will make the report show up in the Visual Reports dialog in Microsoft Project.  From this point forward, you will be able to export your Resource Usage data to Excel using this Visual Report for Excel.

If we assume the subreports are all based on a Contacts table for instance, then base the main report on the same Contacts table and order it by whatever field or fields by which you want the subreports output.  Link each subreport to the parent report on the key, e.g ContactID.  Each set of three 'forms' will then print for each contact.  You can filter the parent report as necessary when you open it to restrict the 'forms' printed to a subset of your contacts.

Leave the parent report without any headers or footers as it is purely a container for the subreports.
