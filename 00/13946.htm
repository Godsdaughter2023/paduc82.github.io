                        net start msiserver

hoss n. feffer say:

Simplest way is to use a monospace font (such as Courier) for those cells.  It will make every letter/character the same width.  Then when you resize the column to some width (say wide enough to accommodate 6 characters) then every cell that has 6 characters will fit on 1 line without increasing the row height, and every cell with 7 or more characters will wrap to the next line and increase the row height.  Note that multiple spaces at the end of a wrapping line are ignored in determining required column width (ditto for spaces at the end of the cell value).

lmac_7 say:

To help anyone else that may find this thread in the future:

1) The adjustment of the column width should work, just make sure that the 'Wrap Text' is on for that column and it will auto fit to the nearest spacing, not to the end of the longest string.

2) If that doesn't work, check to make sure there are no 'ALT-Enter' spaces in your cells wording. 'ALT-Enter' introduces a new line within a cell even if the first line does not go fill the cell width. (This can happen when pasting text into a cell from a source with multiple lines.)

   There are two easy ways to check this;

   A) Widen the column and see if text automatically fills into the upper lines.

   B) Go to the beginning of a line and press 'backspace', if there is still a space between the word in the upper line and the line you just backspaced it's likely there was an artificial 'ALT-Enter' line creation.
