gwyzagain say:

Try downloading the diagnostic utility titled:

WindowsUpdateDiagnostic.diagcab

You can find it by doing a search in the Microsoft Download section.

If the utility fails to fix the problem, you can try the ''subinacl fix''.  You can find the downloadable installer in the Microsoft download section. Search for:

subinacl.msi

Simply follow the instructions in this thread:

http://social.answers.microsoft.com/Forums/en-US/vistawu/thread/a9a2c92d-793b-4050-bdb1-720bb97bd78d/?prof=required

Note:  Please be aware that the ''subinacl.exe'' file can wind up being installed in locations that will not allow the ''reset.bat'' file to work.  The most common _incorrect_ installation location is: Program Files\Windows Resource Kits\Tools\subinacl.exe.  (Or, with Win7 64bit, the incorrect location would be:  Program Files (x86)\Windows Resource Kits\Tools\subinacl.exe).  The _easiest_ remedy for this ''incorrect location'' problem is to perform a search (or just look in the locations stated above), for the ''subinacl.exe'' file (without the quotes). Then, once you have located it, open the location and then copy the file ''subinacl.exe'' (and only that file), then ''paste'' it inside the System32 folder of _any_ version of Windows 7 (both x86 _and_ 64bit).  Then, the ''reset.bat'' file will perform correctly for you.  (In other words, there is no need to alter the commands inside the ''reset.bat'' file, if you simply copy and paste the ''subinacl.exe'' to the System32 Folder of _any_ Windows 7 version). 

After running the ''reset.bat'' file (right click on it and ''Run As Administrator''), if the command prompt window opens and only runs for a second or so, then it has failed.  If so, double check the location of the ''subinacl.exe'' file (as I have stated above), and double check that you have created the ''reset.bat'' _exactly_ as stated in the instructions. (Example: When saving/naming the file, you _must_ have Notepad saving ''file type'' set to ''All Files'', which will show as ''*.*''  Then save as the name ''reset.bat'' (no quotes).  Only in that way is the file created correctly).

Once that ''fix'' has been applied successfully (the command prompt window will run for a very long time processing the ''reset.bat'' command)... when done, restart the computer and then try running Windows Update.  If you _still_ have an update problem, then run the ''WindowsUpdateDiagnostic.diagcab'' utility again.  This time it should report all problems fixed.

If not, you can also check out my thread which covers this in more detail, but it involves ''restoring'' the computer back to before Sp1 was installed.  If you cannot (and/or do not wish to) restore that far back, then I am not aware of any other ''fixes'' other than the ''worst case scenario''...  A clean re-installation of Windows.

http://answers.microsoft.com/en-us/windows/forum/windows_7-update/window-update-error-80070005-various-causes/5d8e3d1a-b035-47e7-a97b-9ea7a400594d

Ps... Yes, the ''SoftwareDistribution'' folder is located in ''C:\Windows

You should delete the old folder.  The new folder will be created correctly, and if there are any ''ownership'' issues with the new folder, they would be corrected by the Diagnostic utility (WindowsUpdateDiagnostic.diagcab)

Best of luck...
