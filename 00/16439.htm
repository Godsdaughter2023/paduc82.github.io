    I'm thinking something like (in the find box) Sheet6*Sheet12  or Sheet6?Sheet12 but I'm thinking that the non-alphanumeric is being treated as part of a larger text string and not a wildcard.

     

    Any thoughts on how to search for this in the IDE? I also considered pasting into Word, but then it will search the whole document, not necessarily a line at a time.

     

    Thanks!

    Keith

    andreas killer say:
    Copy the the code below into your file and run it.

    Andreas.

    Sub Test()
      Dim VbComp As Object 'VBComponent
      Dim Contents() As String
      Dim SubStrings
      Dim i As Long, j As Long
     
      SubStrings = Array("Sheet6", "Sheet12")
      For Each VbComp In ThisWorkbook.VBProject.VBComponents
        With VbComp.CodeModule
          If .CountOfLines > 0 Then
            Contents = Split(.Lines(1, .CountOfLines), vbCrLf)
            For i = 0 To UBound(Contents)
              For j = 0 To UBound(SubStrings)
                If InStr(1, Contents(i), SubStrings(j), vbTextCompare) = 0 Then Exit For
              Next
              If j > UBound(SubStrings) Then
                Debug.Print "Found in " & VbComp.Name & " line " & i + 1
              End If
            Next
          End If
        End With
      Next
    End Sub

    Can only review/repeat how Outlook and IMAP accounts work which does not change.

    #1 - <IF> all your configured email accounts are configured as IMAP - then the calendar info would be stored in a contact folder in one of the OST files in the profile. That .ost data file would have been configured as the "default data file" which is where the default contact/calendar folders are contained.

    #2 - The ONLY exception to the above would be <IF> you had another account configured in the profile either as a POP3 or Outlook.com/Exchange account OR a PST file was created where either the data file for the email account or stand-alone PST file was set to be the default for the profile.

    #3 - The ONLY way you would have an IMAP account in the Outlook Nav pane without contact/calendar folders would be if another data file has already been set as the default data file for the profile

    #4 - <IF> you deleted the email account from the Outlook profile, then the associated .ost file for that account would be deleted immediately so it's not a file that you're going to find after the email account deletion

    There are no exceptions to the above so if there is absolutely no calendar data in the Outlook profile then either the IMAP account where the calendar data was stored was deleted (hence the ost file being deleted) OR it's in a contact folder somewhere in the profile (but you state that's not the case).

    All of this assumes that this issue about missing calendar data is strictly related to the Outlook desktop client and not a case where your gmail calendar is being accessed by the Windows 10 Calendar app which is compleletly separate and distinct from Outlook. If this is the case, deleting the Gmail account from Outlook would have no impact on the Google Calendar accessed by the Win10 Calendar app.
