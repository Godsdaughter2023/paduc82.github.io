For example, one person was trained on 24/09/2014. Their training lasted 3 years before they needed to be trained again, and the expiration date for their training is 23/09/2017, which has passed. How do I use a formula to highlight that in red to show on my spreadsheet that it has passed? And how do I do that for all of the different dates that they were trained on?

Sam 


hansv mvp say:

Select the cell or cells with the training dates.

On the Home tab of the ribbon, select Conditional Formatting > Highlight Cells Rules > Less Than...

Enter the following formula in the box:

=EDATE(TODAY(),-36)

(this returns the date 36 months = 3 years before the current date)

Select a highlight style from the dropdown, then click OK.

bernard liengme say:

Select all the data

Open Conditional Formatting dialog

Select New Rule then Use A Formula, and type this =DATEDIF($C2,TODAY(),"m")>36

The DATEDIF function is undocumented but is available is all version of Excel

Note the $ before the C, this is needed to format the row

best wishes

Continuing the earlier point about DOCX vs DOTX and creating templates in general:

Your design is very elementary, essentially a simple emulation of a typewriter page. It takes advantage of very few Word features.

Using tables to keep pictures and their titles together is something I like to do. However, I like to keep both rows in a single table.  It makes keeping them together a little easier.   Either way, yours layout or mine, I like to select the lines/rows affected then go to the Home tab > Paragraph group > Paragraph dialog launcher > Line and Page Breaks tab.  Then turn on "Keep Lines Together" option.  That way they break across pages as a unit.

Rather than manually numbering each line, and therefor having to create a whole bunch of  you can put auto numbers or the { SEQ } field in them. Then the user simply has to copy and paste to recreate more input areas without worrying about the numbers

FYI: rather than typing the "A&P ... Photo sheet" lines in the body of the text, that is something that absolutely should in the header.  Since you have a header, I assume that you intend these documents to be printed.  On the assumption, I would add a way of uniquely identifying the printed pages, ie the claim number and generated page numbers.  I prefer  Page x of y to make page sorting easier.

The claim number in the header is a "cross reference" to a bookmark I created in the body. They just have to type over "replace" in the body.

The numbers in the tables are done using the { SEQ } field code.

Both the cross reference and the number field codes just need to be updated to display the correct values. <CTL><A> followed by <CTL><F9> will update the sequence numbers.  The header can be set to automatically update when Printing: File menu > Options command > Display option > Printing Options section > turn on "Update Fields Before Printing" and "Updated LInked Data before Printing".

If you have a 32 bit version of Office already present, it will conflict with the 64 bit version; however the 64 bit version is not normally recommended for most users.
