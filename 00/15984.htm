lars h. ellerhorst say:

Office Deployment Tool configuration.xml

Error using the attribute AllowCdnFallback="True" in the configuration.xml for ODT.

I created a small test environment to troubleshoot ODT installation errors. The environment runs on a Windows 2012 R2 host Hyper-V and consists of a domain controller, a file server and some Windows 10 client VMs.

I used the following config.xml to download the installation package for Access 2016 VL Click-to-Run, but the command failed with an error message:

<Configuration>
<Add SourcePath="\\lab-rras-2012\deployment\Office 2016\" OfficeClientEdition="64" AllowCdnFallback="TRUE">
<Product ID="Access2016Volume">
<Language ID="MatchOS " Fallback="en-us" />
</Product>
</Add>
<Updates Enabled="TRUE" />
<Display Level="Full" AcceptEULA="TRUE" />
</Configuration>

According to Microsoft documentation this attribute is optional: https://docs.microsoft.com/en-us/deployoffice/configuration-options-for-the-office-2016-deployment-tool#add-element

Removing the attribute solved the issue, the installation package download succeeded.

<Configuration>
<Add SourcePath="\\lab-rras-2012\deployment\Office 2016\" OfficeClientEdition="64">
<Product ID="Access2016Volume">
<Language ID="MatchOS " Fallback="en-us" />
</Product>
</Add>
<Updates Enabled="TRUE" />
<Display Level="Full" AcceptEULA="TRUE" />
</Configuration>

Same issue happened using the Product IDs HomeBusinessRetail, VisioProXVolume, ProjectProXVolume. Removing the attribute AllowCdnFallback performed well, with this attribute the error message appeared.

Finally I used the config.office.com tool to generate the configuration_ms.xml:

<Configuration ID="976caf98-46c9-40da-ab2c-a5d55140a0eb">
  <Info Description="Test environment" />
  <Add OfficeClientEdition="64" Channel="Broad" SourcePath="\\lab-rras-2012\deployment\Office 2016" AllowCdnFallback="TRUE" ForceUpgrade="TRUE">
    <Product ID="O365BusinessRetail">
      <Language ID="MatchOS" />
      <ExcludeApp ID="Groove" />
      <ExcludeApp ID="OneNote" />
    </Product>
  </Add>
  <Property Name="SharedComputerLicensing" Value="0" />
  <Property Name="PinIconsToTaskbar" Value="TRUE" />
  <Property Name="SCLCacheOverride" Value="0" />
  <Updates Enabled="TRUE" />
  <AppSettings>
    <Setup Name="Company" Value="Contoso" />
  </AppSettings>
  <Display Level="Full" AcceptEULA="TRUE" />
</Configuration>

Surprisingly this one worked as well.

Unfortunately the configuration tool on the website does not list all product ids, therefore I created my own configuration.xml.

OK, so I have an idea of how you could do this.

If you put some code in the Workbook_BeforeSave event you could make a copy of the file by doing the following:

If the user is doing a SaveAs:

1) Cancel the SaveAs
