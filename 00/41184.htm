a. user say:

Even with 64 bit systems the folder is going to be called system32 but inside will be 64 bit programs, DLLs and other files - that is the way it is.

Some Internet searching:

Reveals many examples and reasons why Microsoft chose to leave the folder as system32.  This sounds like a very good reason to keep the folder as system32:

This is to ease the programming purposes and to provide backward compatibility. That is, if a 64-bit program refers to System32 in its code, it will get the 64-bit DLL automatically when called.  This helps developers as they don’t have to change the code in their programs for Windows 64-bit.

If Microsoft used a system64 folder think of all the software in the world that would have to change to accommodate that little adjustment or developers might have to start keeping two versions of their software - not fun.

On 64 bit systems you will also find a SysWOW64 folder:

The SysWOW64 folder (Windows on Windows) is where Windows keeps 32bit copies of applications that might be needed from time to time on your 64bit system (the files may be the same).  It makes sense that there might be files with the same name in bother folders so if a 32bit version of a file is needed for some reason that is where Windows can find it.  There is no SysWOW64 folder on 32bit systems.

Windows and other programs are built in with smarts to get the right files from the right folders when they are needed.

So everything sounds hunky dory on your 64-bit system.
