rob koch say:

Along with everything that Bruce has already correctly stated, I'll point out that the real-time protection of most major AV programs have improved greatly over the years, especially due to the abilities which Microsoft has built into each new version of Windows to make these more effective and in the case of Windows 10, must faster as well.

This means both that it's more likely any true malware might be detected during an initial download or if it did somehow mange to slip by, when that same file might later be opened or even simply "touched" by browsing into the folder containing it.

For example, the most recent Microsoft MSE and Defender clients actually scan any files within a folder the moment a PC user browses into them, assuming they hadn't already been scanned since the last time the system had been shut down, which is tracked by something called the "scan cache" that operates in the background.

On top of that, in many cases the items found by full scans are false positives.  Unlike a quick scan which examines only portions of the operating system which are known to contain malware, the full scan examines every single file, many of which are stored out of context in temporary or personal folders where they have no meaning except as potential "droppers" if, for example a malware executable were clicked by mistake.

The problem here is that the typical consumer PC user doesn't recognize the difference between these active or inactive files, so they panic the moment anything is identified as potential malware, even if its location or the type of file containing it isn't directly executable.  This is why Microsoft only performs a quick scan automatically and also why it's now more difficult with Defender to schedule a full scan, since in most cases these only result in confusion as well as unnecessary system overhead.

Since Microsoft's clients normally perform an initial background full scan shortly after installation and then maintain both real-time and occasional quick scans by default, I only personally perform a manual full scan when either malware has already been detected or for some other reason I feel that malware might actually be present.

Everything else is simply obsessive compulsive behavior we've all learned from years of malware history, whether we've experienced it ourselves or not.  I know, it took me a few years to stop this mostly pointless scanning myself, even with the better understanding of the Microsoft security clients I've gotten.

Rob

Sum across sheets - cells in different locations

Good day!

 

I need to create a YTD summary spreadsheet for 12 month worksheets (Jan:Dec) whereby the location name is always in column B, but may not appear in the same row month by month (locations open or close).  I need to sum the number in column C for the matching location name for the YTD location list.  Will also need to add columns D-G so would love to be able to copy it across/down.

 

In my head, it's this for the first column sum:

=SUMPRODUCT((Jan:Dec!$B2:$B200=YTD!$B2)*(Jan:Dec!$C2:$C200))

 

But that would be too simple, it seems. :)  I don't really want to add 12 sumproducts together in a single equation, but can, if that's the only way to do so.

 

Thank you in advance for your non-VBA (for ease of other users) help. 

hansv mvp say:

There is no built-in support for SUMPRODUCT or SUMIF across multiple worksheets, so if you don't want to resort to VBA, you'll have to add 12 SUMPRODUCT or SUMIF expressions.

jeeped say:
There really isn't a non-VBA/non-Add-in solution for what you are trying to accomplish. A UDF (User Defined Function) as below or an add-in like MOREFUNC will do as you wish.

Public Function udf_SumIf_YTD(fWS As String, eWS As String, ytdWS As String, _
  rCRIT As Range, cCRIT As Range, rSUM As Range) As Variant
    Dim w As Long
    For w = Sheets(fWS).Index To Sheets(eWS).Index
        udf_SumIf_YTD = udf_SumIf_YTD + _
          CCur(Application.SumIf(Sheets(w).Range(rCRIT.Address), _
            Sheets(ytdWS).Range(cCRIT.Address), Sheets(w).Range(rSUM.Address)))
    Next w
End Function

To implement the above, tap Alt+F11 and when the VBE opens immediately use the pull down menus to Insert, Module (Alt+I, M). Paste the above into the new pane titled something like Book1 - Module1 (Code). Tap Alt+Q to return to your worksheet. Syntax as follows,

=udf_SumIf_YTD(<first ws to process>, <last ws to process>, <ws with criteria>, <range to work against criteria>, <cell with criteria>, <range to sum>)

In your case this would be something like,

=udf_SumIf_YTD("Jan", "Dec", "YTD", B2:B200, B2, C2:C200)

There can be no worksheets between Jan and Dec that are not intended to be processed. I've made that sample workbook available from my public DropBox here. SkyDrive would strip the code.

Sum_Across_Worksheets.xlsb

If you plan to go the add-in route, be cautious of the downloads you receive from the internet. The link above is a clean version of MOREFUNC but there are many malformed imitations. Note also that the add-in will have to be installed on every machine using the workbook that references the add-in functions.
