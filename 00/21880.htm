ossiemac say:

I have produced the code below pending your answer to my previous question. (The code is easily edited depending on your answer).

 

With this code you remove your existing Conditional format otherwise one might interfere with the other.

 

Where more than one of the ranges contains TRUE, the last range takes precedence and sets the color.

 

There is a limit on the available color constants so the color constants can be replaced with a color number, colorindex or and RGB code. If you want other colors then let me know and I will give you a method of finding using color index that has more colors.

 

To install the code:

  1. Right click the worksheet tab name
  2. Select View code
  3. Copy the code below and paste it into the VBA editor. (Do not change the sub name)
  4. Close the VBA editor (Cross with red background top right of window)
  5. Save the workbook.
  6. Ensure you have macros enabled. See Help for how to do this. (I think that the option in xl2003 is Disabled with notification)
  7. Now each time a value changes in the ranges that contain TRUE or other, the code runs and updates the colors in A1:AE10.

 

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim rngUnion As Range   'Union of ranges with TRUE in some values
    Dim a As Long           'Each Area in rngUnion
    Dim lngColor As Long
    Dim r As Long           'Rows 1 to 10
    Dim c As Long           'Columns A to AE
   
    Set rngUnion = Union(Range("A51:AE60"), _
                    Range("A101:AE110"), _
                    Range("A151:AE160"), _
                    Range("A201:AE210"), _
                    Range("A251:AE260"), _
                    Range("A301:AE310"))
   
    If Not Intersect(Target, rngUnion) Is Nothing Then
       
        Range("A1:AE10").Interior.ColorIndex = xlNone
       
        For a = 1 To rngUnion.Areas.Count
            Select Case a
                Case 1
                    lngColor = vbBlue
                Case 2

kosh vorlon say:

Hi Michelle,

 

There are several sections in the History tab. To get to the Quarantine section, click the radio button next to Quarantine. If it shows there, then it's in quarantine. You're perfectly safe with it in quarantine as you would be if you removed it. But if it's there and you want to remove it, highlight it and the buttons on the bottom of the screen will become active. Click on Remove and it will remove the item from quarantine.

 

I recommend you do it that way and reset the quarantine time setting to a month or even 3 months in case something that's a false positive ends up there. I assure you it'll be easier to recover from quarantine than if you remove it (where it may not be recoverable at all). This is the safest way. As I said, your computer is just as safe and secure with it in quarantine as it is if it is removed - you just have more options to recover from mistakes.

 

Now click on the radio button for display History. Yes it will still show there because that's just a log of what happened. It has nothing to do with what currently exists or has happened since then. It cannot be changed. Even if you do something to alter what it shows there, what you do will happen but the original entry with what originally occurred will remain exactly the same - it will not "update" because of your changes.

 

If it doesn't show in Quarantine and doesn't appear on another scan, then it was removed and all is well - even if History doesn't show that and still shows it in Quarantine (as will be the case). History remains for 30 days and then disappears. You can make no changes to it except to clear it entirely or leave it be and think of it as "View only" and a record of past events and not of current events. I'd leave it alone as it can come in useful for troublehooting.

 

I hope this helps. In short, it seems to me you're perfectly fine.

 

Good luck!
