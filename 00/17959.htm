dougwells say:

Need help getting excel to auto copy form one sheet to another sheet if a column has a value in it.

I am working on a project and I need excel to do the following automatically for me.

On sheet one of the workbook we will be entering data on a client. there are three columns at the end of the row. lets call them Red, White and Blue.

What we want to happen is when we finish entering the client information into the row and then put a value like an X in the column red, then have excel automatically copy the entire row to sheet 2 (Red).

Also if a value is entered in Blue then to automatically copy the entire row to sheet 3 Blue. 

Also if a value is entered in White then to automatically copy the entire row to sheet 3 White. 

Also if a value is in one or more  of the three *red, white, blue" columns excel will copy to one or more respectively

Thanks in advance for your help 

jeeped say:
I've culled the following together that should compensate for many of the non-specific references you've used. Right-click Sheet1's name tab and choose View Code. When the VBE opens, paste the following into the pane titled something like Book1 - Sheet1 (Code), 

Private Sub Worksheet_Change(ByVal Target As Range)
    On Error GoTo FallThrough
    Dim r As Long, b As Long, w As Long, x As Range
    r = Application.Match("red", Me.Rows(1), 0)
    b = Application.Match("blue", Me.Rows(1), 0)
    w = Application.Match("white", Me.Rows(1), 0)
    If Not Intersect(Target, Union(Me.Columns(r), Me.Columns(b), Me.Columns(w))) Is Nothing Then
        Application.EnableEvents = False
        For Each x In Intersect(Target, Union(Me.Columns(r), Me.Columns(b), Me.Columns(w)))
            If UCase(x.Value) = "X" Then
                With Sheets(Me.Cells(1, x.Column).Value)
                    Me.Cells(x.Row, 1).Resize(1, Me.Cells(x.Row, Columns.Count).End(xlToLeft).Column).Copy _
                      Destination:=.Cells(.UsedRange.Rows.Count + 1, 1)
                End With
            End If
        Next x
    End If
FallThrough:
    Application.EnableEvents = True
End Sub

Tap Alt+Q to return to your worksheet. This assumes that you actually have Red, Blue and White in row 1 as column header labels and that there are worksheets with matching names.

john - project say:

Rajan_Kang,

Unless the predecessor task finishes exactly at 5:00 PM, assuming the standard calendar and work shifts, then yes, a successor task will start as soon as the predecessor task finishes.

To see this more clearly, go to File > Options > General group and pick a date format that includes the time. That allows you to see exactly when a task finishes. A successor task will start on the next available work time so if a task finishes at 4:30PM and has a successor task then that successor task will start at 4:30PM.

If you want to insure a task always starts at the beginning of the day regardless of when the predecessor task finishes, that can be accomplished in a couple of ways but using a lag is not one of them.

Hope this helps.

John

The instructions are either incorrect or not intuitive.  Below are the steps that I had to take with Windows 7 and Excel 2010 to get my files to show up in the My Templates area.

  1. File -> New -> My Templates
  2. Within the Personal Templates window, right click, then choose Include in Library -> Create new library
  3. Windows Explorer will be opened with your new library as the active library.  I chose to rename this library to "Office Templates"
  4. Open your template file in Excel using File -> Open
  5. File -> Save As
  6. Within the Save As dialog, navigate to and click on your newly created library.
  7. Click Save
  8. As an alternative to Save As, simply move your existing template file to your newly created library.

Now your template file should be visible in the My Templates area of Excel.
