Just curious, we use a 2000 db at work and after reading some of the forums if the one we were working on was split.

daniel pineault say:
Are the tables in the active db or in a separate db?  Try editing a table's design, if you can actually change the field properties and save them then the table is in current db and not linked and as such your db is not split.  If you cannot edit them, then the tables are in a back-end database and hence your db is split.

ken sheridan say: Even if there is a back end not every table might be linked to a table in it, so examining one at random is a bit hit and miss.

If you can open the navigation panel/database window linked tables have a small arrow beside the table icon, local tables don't.  If your database is set up so that it doesn't allow you to see the navigation panel/database window, but you can open a new or existing module and add the following procedure to the module (you don't have to save it)

Public Sub Am_I_Split()

     Dim tdf As DAO.TableDef
     
     For Each tdf In CurrentDb.TableDefs
        If tdf.Connect <> "" Then
          MsgBox "Database is split."
          Exit Sub
        End If
    Next tdf
        
     MsgBox "Database is not split."
     
End Sub

With the insertion point in the procedure press the F5 key.

joeu2004 say:

jmay wrote:

I have a pretty simple Macro that was running fine in 2003.  Now that i have upgraded to 10 it seems to be running really slow. Below is the Macro.  Any ideas? 

 

I see no performance difference when I try it in both Excel 2003 and 2010.

It is anyone's guess how your workbook might be more complicated, and why those complications might be exacerbated by the differences in Excel 2010 (probably the 1+ million rows v. 65536 in Excel 2003).

And people will offer their guesses, possibly sending you on wild goose chases.  I might as jump into the fray.

 

1. First, I would add .EnableEvents=False (and True) to the With Application statements, if only because it is usually "good practice" to do so.  But perhaps some poorly-designed event macro is the culprit.

 

2. Second, I would instrument the macro to try to identify the performance bottleneck.  You probably suspect it is the For Each loop.  I suspect it is when .Calculation=xlCalculationAutomatic is executed.  In any case, I would add the following statements:


    Dim st0 As Double, et1 As Double, et2 As Double, et0 As Double
    st0 = Timer
    With Application
        .ScreenUpdating = False
        .Calculation = xlCalculationManual
    End With
   
    Set MyRange = Range("A3:DS3")
   
    MyRange.EntireColumn.Hidden = False
    et1 = Timer
    
    For Each rng In MyRange
        If rng.Value = "Hide Column" Then
            rng.EntireColumn.Hidden = True
        End If
    Next rng
    et2 = Timer
    
    With Application
        .Calculation = xlCalculationAutomatic
        .ScreenUpdating = True
    End With
    et0 = Timer

    MsgBox "total time:  " & Format(et0-st0,"0.000") & " sec" & _
        vbnewline & "unhide time:  " & Format(et1-st0,"0.000") & " sec" & _
        vbnewline & "hide time:  " & Format(et2-et1,"0.000") & " sec" & _
        vbnewline & "auto calc time:  " & format(et0-et2, "0.000") & " sec"

Don't be surprised if some times are zero.  Timer has a resolution of 15.625 milliseconds.  Also, do not execute the code near midnight.  Timer rolls over to zero then, which will screw up the time calculations.  We can adjust for that situation.  But it is tedious to do.  It is not worth the effort for so unlikely a circumstance, one that you can easily avoid.

 

3. If most of the excessive time is charged to "unhide time" or "hide time", look for formulas and features that might depend on whether or not cells are hidden.  For example, SUBTOTAL and auto filtering -- although the latter seems like an unlikely culprit.

 

4. If most of the excessive time is charged to "auto calc time", look for "volatile" formulas and objects that use whole-column references like A:A -- although I do not mean to imply that the reference must be to a potentially hidden column.

By "volatile" formula, I mean any formula that references a "volatile" function like RAND, OFFSET, INDIRECT, TODAY et al, as well as any formula that depends directly or indirectly on cells with such formulas.

Some people say that Conditional Format formulas are also "volatile" calculations.  My experiments suggest otherwise; so perhaps it is only some CF formulas that are "volatile".

I believe that some objects are "volatile" insofar as they are recalculated whenever an auto recalculation cycle is performed.  For example, table objects (like pivot tables) and charts.  That is just my vague recollection; I might be wrong.

 

I believe I had some other ideas; but they have escaped me while I was writing the ones above.  Check back for additional edits later.

    merge labels order top to bottom - Microsoft Community

    eric_gr say:

    merge labels order top to bottom

    My datafile is in EXCEL and my merge layout file is in WORD.  How can I have the labels printed from top to bottom and then across to a second column rather than left column, right column, then row 2 left column, row 2 right column, etc.?

     

    Thanks.

    doug robbins - mvp office apps & services (word) say:

    Hear is a macro that will sort a datasource that is in a Word document so that the labels will be printed in that fashion.

    You can copy and paste your Excel data into a Word document so that you can make use of it.

    ' Macro to assign numbers to data source so that it can be sorted to cause
    labels to print down columns
    Dim Message, Title, Default, labelrows, labelcolumns, i As Integer, j As
    Integer, k As Integer
    Message = "Enter the number of labels in a row"    ' Set prompt.
    Title = "Labels per Row"    ' Set title.
    Default = "3"    ' Set default.
    ' Display message, title, and default value.
    labelcolumns = InputBox(Message, Title, Default)
    Message = "Enter the number of labels in a column"    ' Set prompt.
    Title = "Labels per column"    ' Set title.
    Default = "5"    ' Set default.
    labelrows = InputBox(Message, Title, Default)
    ActiveDocument.Tables(1).Columns.Add
    BeforeColumn:=ActiveDocument.Tables(1).Columns(1)
    ActiveDocument.Tables(1).Rows(1).Range.Cut
    k = 1
    For i = 1 To ActiveDocument.Tables(1).Rows.Count - labelcolumns
       For j = 1 To labelrows
           ActiveDocument.Tables(1).Cell(i, 1).Range.InsertBefore k + (j - 1) *
    labelcolumns
           i = i + 1
       Next j
       k = k + 1
       i = i - 1
       If k Mod labelcolumns = 1 Then k = k - labelcolumns + labelcolumns *
    labelrows
    Next i
    ActiveDocument.Tables(1).Sort FieldNumber:="Column 1"
    ActiveDocument.Tables(1).Rows(1).Select
    Selection.Paste
    ActiveDocument.Tables(1).Columns(1).Delete

    tisky zheng msft say:

    Hi Dirk,

     

    Thanks for your update.

     

    Because OneNote can only decode the hyperlink for one time, while Outlook and Word can decode for several times. When you copy the hyperlink to the OneNote, OneNote can only recognize the %25 as % but it cannot recognize %20 as “Space”. So the link becomes “\\SBS01\data\Invoices\invoice%202016\invoice%20Sample%20216712.pdf”.

     

    Please follow the steps below to check the outcome:

     

    1. Copy the hyperlink to OneNote.

    2. Right click the hyperlink and choose Edit link.

    3. In Address, replace %2520 with “Space”.

    biang_0 say:

    Hi Todd,

    After doing some research, I found this thread:
    http://answers.microsoft.com/en-us/mac/forum/macoffice2016-macword/word-for-mac-2016laptop-office-365/542085c6-31ca-4e71-b559-8d9f01808277

    It claims that Word for Mac 2016 doesn’t take the Return Address info from its own Preferences. It uses the User Contact Record from Outlook or from Apple's Contacts app (formerly known as Address Book in older versions of OS X).

    So, if you have Outlook - regardless of whether you use it or not:
    1.Launch the program
    2.Go to View> People
    3.Go to View> Me Contact (if necessary)
    4.Update your personal record to include address information.
    5.In the Envelope dialog of Word for Mac 2016, clear the check on Use my address, click the Contact Card icon to access the Contacts list, then select Office contacts from the top dropdown menu to pick your Contact record in Outlook.

    If you don't have Outlook:
    1.Launch Apple's Contacts app
    2.Go to Card> Go To My Card.
    3.Update your contact record.
    4.In the Envelope dialog of Word for Mac 2016, clear the check on Use my address, click the Contact Card icon to access the Contacts list, then select Apple contacts to pick your Contact record.

    If both methods don’t work, since this forum focuses on Office 365 for Business online services, I suggest you post at our Office for Mac forum for professional support. Your understanding is highly appreciated.
