I have a Windows 7 box. Today I found that if disable the UAC, then "Run as administrator" will not work. Programs only start with limited privilege instead of Administrator. 

Is this issue by design? If so, is there a workaround for it? I need to use "Run as administrator" with UAC disabled.

Thanks for any suggestions.
frederik long say: Did you test if UAC is off by creating, then deleting a test file such as c:\Test.txt?

If your application makes use of a pre-existing application such as Windows Explorer (Explorer.exe) then launching it with "Run as administrator" won't help you. You would first have to kill Explorer (which you probably can't as an ordinary user), then relaunch it.

You can test elevation like so:
  1. Log on under an ordinary account.
  2. Create a new desktop shortcut. Get it to run diskmgmt.msc.
  3. Right-click it, then launch it with "Run as administrator".
  4. See if it lets you change the drive letter of your CD drive.
  5. Restore the original drive letter.
If you can do all of the above then elevation works as designed.

In summary, if elevation works but your specific program does not then you must run your application under an admin account.
