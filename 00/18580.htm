I am hoping to do this using a macro.

 

Thanks in advance :)

jllatham say:

Does this code with its comments help you any?

 

Sub ListFilesIn1Folder()
  Dim baseFolder As String ' will hold path to a folder
  Dim oneFilename As String ' will hold single filename
 
  Const resultsWSName = "Sheet1" ' name of sheet to put results onto
  Dim anyWS As Worksheet ' worksheet to put results on
 
  'set reference to the worksheet we want to
  'save our results onto - makes typing shorter :-)
  'and will process faster
  Set anyWS = ThisWorkbook.Worksheets(resultsWSName)
  'if you want to keep old info, delete or comment next line
  anyWS.Cells.Clear ' clears contents and formats
  '
  'use Excel built in dialog to get the name of the folder
  'that you want to get files list from
  With Application.FileDialog(msoFileDialogFolderPicker)
      .Show
      On Error Resume Next
      baseFolder = .SelectedItems(1)
      If Err <> 0 Then
        'user cancelled
        Err.Clear
        On Error GoTo 0
        Exit Sub
      End If
  End With
  On Error GoTo 0
  'currently baseFolder will not have the needed last
  'path separator character ( / or : ) so we check that
  'and do test in case Microsoft ever changes its ways.
  If Right(baseFolder, 1) <> Application.PathSeparator Then
    baseFolder = baseFolder & Application.PathSeparator
  End If
  'NOTE: the following does not work in
  'Excel 2011 on an Apple/Mac system - there are better ways
  'to do the same thing on one of those.
  'start getting filenames of Excel type files from the
  'selected folder
  'seed oneFilename to get things rolling
  oneFilename = Dir(baseFolder & "*.xls*")
  'if you want all files in the folder listed, regardless of type,
  'then use this instead
  '     oneFilename = Dir(baseFolder & "*.*")
  'this will make things go MUCH faster - and it
  'turns itself back on when you exit the Sub
  'you won't see results until you click [OK] for the
  'message we give at the end of this routine
  Application.ScreenUpdating = False
  'now we loop through all filenames in the folder
  'and put them on our results sheet
  Do While oneFilename <> ""
    'write the full path + filename to next available
    'blank row in column A.
    anyWS.Range("A" & Rows.Count).End(xlUp).Offset(1, 0) = _
     baseFolder & oneFilename
    'get another filename, when it runs out of filenames
    'it returns an empty string, ""
    'just giving DIR() command uses last provided parameters
    oneFilename = Dir()
  Loop
  'good housekeeping: release assigned resources back to the system
  Set anyWS = Nothing
  'announce completion
  MsgBox "ALL DONE"
End Sub

Before deleting the document look at Stefan Blom's and Pam Caswell's responses. Both have good suggestions.

In general, if you format as much as possible using styles rather than directly applied formatting, you will have a far less complex document. (It will also have a much smaller size.) It sounds like you are trying to use styles appropriately.

Understanding Styles in Microsoft Word

How styles in Microsoft Word cascade by Shauna Kelly

From Feb 2012:

I just had occasion to edit a 100-page document that was created without using styles. It was formatted completely with direct formatting. Each page ended with a page break.

Each time it had to be saved, the save took more than 60 seconds, during which time Word was frozen.

A similar-sized document formatted using Styles takes less than 3 seconds for me to save. When a page break must be forced, instead of using a manual page break, the paragraph to begin the new page is formatted using a style that has "page break before" paragraph formatting.

This difference was due entirely to the document being directly formatted - a much higher level of complexity. Each paragraph mark in a directly-formatted paragraph carries with it up to fifty different formatting commands. When formatted using a style, that paragraph mark will carry with it one command - use this style!
