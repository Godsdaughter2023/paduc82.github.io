hansv mvp say:

These three tables are hidden system tables that are automatically created by Access 2007 and later in a new database.

They keep track of the way database objects are grouped in the Navigation pane on the left hand side of the Access window.

If you display hidden and system objects in the navigation pane, you'll see these and several other system tables. (To do so, right-click the title bar of the navigation pane and select Navigation Options from the context menu. Tick the check boxes for Show Hidden Objects and Show System Objects, then click OK)

I think it's a bug that they are displayed in the Relationships window; you can remove them from the Relationships window without problems. (You can't delete the relationships, and you can't delete the tables themselves either)

Problem is resolved but NOT from your suggestion.

As a last resort I did the following.

  1. Using "Devices and printers" in Control panel, removed the printer from the remote computer (The computer that connects to the printer via Home network)
  2. Shut down the remote computer.
  3. With the master computer (the one with the printer connected via USB) uninstalled all the software associated with the printer via "Programs and features" in control panel.
  4. Re-started the master computer.
  5. Re-installed the printer software on the master computer (the one with the printer connected via USB) using the original disk that came with the printer.
  6. Started the remote computer and installed the printer via "Devices and printers"

Everything now works fine.
