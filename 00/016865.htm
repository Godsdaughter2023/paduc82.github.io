rob koch say:

No that's not a UAC alert, it's an Action Center notification dialog that ends up taking you to exactly the same certificate dialog I was describing if you choose to click its "No..." option.

So if you'd actually do what I requested and click the "Certification Path" tab you will discover that "This certificate is OK" is displayed near the bottom of that dialog box.

This means that the certificate itself is considered fine by Windows and no problem exists.  The only reason the original dialog box is being presented is because you chose to turn off MSE, which is what Action Center is attempting to warn you about, since you are creating a potential risk by disabling your protection.

As I've stated repeatedly and in other threads I'm sure you've seen, the help information abut certificates that you're reading is incomplete and ineffective in its attempt to explain a highly confusing situation that makes Code-Signing Certificates completely different from most other types, especially the Web Identification certificates that most everyday users are aware of.

Though in the case of Web Server Identification a certificate must be within the valid date range, this is untrue for Code-Signing Certificates as long as they were given a timestamp value (date) during the period for which they were valid.  This removes the issue of an older file failing a digital signature check once its certificate has expired.

If this ability to timestamp a file didn't exist, then all files would become invalid when their certificate expired, typically in a year or less and so every signed file on your PC would need to be replaced at least annually.  Obviously this would be a huge problem and in fact impossible in cases where a PC is kept off the Internet or the company that developed a program was no longer in business.

So the timestamp attribute was added to Code-Signing Certificates and is a separate required step that a developer must make in order to set the timestamp date when they compile their code.  If they don't take this step, the certificate expiration date will take precedence and the file's digital signature will be invalid after that date.

So you are simply confused by the appearance of dialogs, which as Steve mentioned most people simply ignore and press YES/OK and so making more out of what you are seeing than is really there.

Rob

georgeogden say:

This sounds like it's possibly another victim of the failed 16288.1 update and is in need of a manual rest of the Windows Update process..... in my case the "NewOS" directory wasn't being created in the update directory C:\$WINDOWS.~BT which may have corrupted the Windows Update process. 

If you've already tried running the Windows 10 Update Troubleshooter and that hasn't fixed the problem (it quite often doesn't) then you'll need to take more drastic action and attempt to reset the Windows Update process manually.

A simple method is to copy the text between the ++++ lines and paste it into Notepad

++++++++++++++++++++++++++++++++++++++++++++++++

net stop wuauserv

net stop cryptSvc

net stop bits

net stop msiserver

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

+++++++++++++++++++++++++++++++++++++++++++++++++

Save as UpdateReset.BAT (note the .BAT and NOT .TXT) to your desktop and close it.

Go to the desktop and right click on the .BAT file and select "Run as Administrator"

That should flush out any corruption in the Windows update files and enable you to move on to download the latest build. If you get a message saying that wuauserv can't be stopped then run the Update Troubleshooter and then try the batch file again.

Once done, go back and check for updates and also check to see if the C:\Windows\SoftwareDistribution.old and C:\Windows\System32\Catroot2.old directories were created and, if they exist, delete them in case you need to re-run the batch file again.

Finally, before trying to update again make sure you detach any non-essential peripherals, USB devices and completely remove any 3rd party anti-virus software as they may interfere with the update process,

Hope that helps and good luck.

Create master spreadsheet. - Microsoft Community

annalouise1990 say:

Create master spreadsheet.

Hi everyone,

I have three separate worksheets. One is a HR spreadsheet showing a table with Name, Agency and Work Area in 3 columns. The second spreadsheet shows in a table; User IDs and Name. The third shows User IDs and Usage.

My aim is to end up with a master spreadsheet showing User ID, Name, Work Area and Usage.

If anyone could help, this would be very much appreciated.

Please ask should you require more details.

Thanks,

Anna

cabgx3 say:

If you want to keep the original sheets intact, Copy and paste Sheet2 over to a new sheet called "Master".

This will fill columns A of the "Master" sheet with "User ID" data and column B of te "Master " sheet with "Name" data.

In cell C1 of the "Master " sheet, type "Work Area".

In cell C2 of the "Master" sheet put this formula and copy down

=VLOOKUP(B2,Sheet1!A:C,2,FALSE)

In cell D1 of the Master sheet, type "Usage"

In cell D2 of the master sheet put this formula and copy down.

=VLOOKUP(A2,Sheet3!A:B,2,FALSE)

 

That should do it.  You can do Copy > PasteSpecial > Values on columns C and D of the Master sheet and delete the other three sheets if you wish.
