winni liang -msft say:

Hi,

Importing Excel data in Visio will not generate Visio shapes automatically. In Visio 2010, we can just import the Excel data to Visio, then drag the data to the drawing page manually to link data.

To import Excel data into Visio 2010, please do:

1. In the drawing, go to Data tab, in the External Data group, click Link Data to Shapes.

2. In the Data Selector window, select Microsoft Excel workbook, Next.

3. Click Browse to select your Excel file, click Open > Next.

4. Select Custom Range or directly use the default Sheet, next.

5. Select Columns or Rows or directly keep the default settings, click Next.

6. Configure Refresh Unique Identifier, click Next.

7. Click Finish to import data.

8. Then an External Data panel would be shown under the drawing page, drag rows onto the page to link data to existing shapes or to add new linked shapes.

For more information about data importing in Visio 2010, please read:

https://support.office.com/en-us/article/Import-data-from-Excel-SQL-Server-SharePoint-sites-and-other-external-sources-a4789a07-46de-4f66-bcaa-4a58d0c41fe1

The problem is that you have the image formatted as 'Tekstin tasossa' which the English command equivalent is 'in line with text'. You need to change that in order to get the other 'text wrapping' options.

Solution 2(for Developers): in the MKV Media Source, parse FlacStreamInfo block from 'CodecPrivate' element, set a MF_MT_AUDIO_BITS_PER_SAMPLE to correct value.

Note: If the key/value is already created, you do not need to create a new one. Simply follow the steps and see if there is a key/value missing that needs to be added.

 

Please try following the steps below to fix the issue for Word:

  1. Run regedit (Windows key + R, type regedit and press Enter), navigate to HKEY_CLASSES_ROOT, CTRL + F -> find .docx key.
  2. Left click the .docx key, there should be three string values on the right: (Default), Content Type, Perceived Type. Double click the (Default) string value and change the Data to "Word.Document.12"
  3. Right click the .docx key > New > Key, name it "Word.Document.12"
  4. Right click the new key Word.Document.12 > New > Key > name it ShellNew
  5. Right click the ShellNew key > New > String Value. Change the Name of the new string value to "NullFile" and press Enter.
  6. Go to your desktop, press F5 (or right-click on your Desktop > Refresh) and check whether the "New Microsoft Word Document" is in the menu.

 

For Excel, the steps are a little bit different:

  1. Run regedit (Windows key + R, type regedit and press Enter), navigate to HKEY_CLASSES_ROOT, CTRL + F -> find .xlsx key.
  2. Left click the .xlsx key, there should be three string values on the right: (Default), Content Type, Perceived Type. Double click the (Default) string value and change the Data to "Excel.Sheet.12"
  3. Right click the .xlsx key > New > Key, name it "Excel.Sheet.12"
  4. Right click the new key Excel.Sheet.12 > Key > name it ShellNew
  5. Right click the ShellNew key > New > String Value. Change the Name of the new string value to "FileName" and the Data to "C:\Program Files\Microsoft Office\root\VFS\Windows\SHELLNEW\EXCEL12.xlsx"
  6. Go to your desktop, press F5 (or right-click on your Desktop > Refresh) and check whether the "Microsoft Excel Worksheet" is in the menu.
