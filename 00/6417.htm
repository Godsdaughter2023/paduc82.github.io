Fix sync errors between Windows Phone and iTunes (Mac) - Microsoft Community

entegy say:

Fix sync errors between Windows Phone and iTunes (Mac)

Technical Level : Intermediate

Summary
Technical Level: Intermediate
Applies to: Windows Phone 7 (Devices Sync and Update), Windows Phone 8 (Music and Videos)

Fix the most common iTunes/Windows Phone sync issues by clearing out bad iTunes settings.
Details

Update May 9 2016: As of iTunes 12.2, the XML file that the Windows Phone app uses is no longer generated by default. However, it can turned back on by following the instructions in this Apple support article.

Most sync issues with iTunes are the result of a certain bad file from iTunes. It’s a file that is safe to delete, so the best thing to do is to delete it and let iTunes make a fresh copy of it!

First, be sure to have disconnected your phone from your Mac, and have closed the Windows Phone Connector and iTunes with Command+Q!

Remove bad iTunes files

1.  Delete the iTunes library.xml file in the iTunes folder. The iTunes folder is found in your Music folder by default

a.       Sometimes, you may also find a file called iTunes Music.xml. Delete this too.

**WARNING**: There are two very similar files in the iTunes folder: iTunes Library.itl and iTunes Library.xml. Sometimes, the "other" file does not have the .itl file extension. Make sure you DO NOT DELETE the .itl file. ONLY delete the XML file. If you can't see any file extensions, make sure they're on in the Finder's advanced Preferences.

Reset the Windows Phone Connector

1.  Open a Finder window and from the menubar, go Go>Go to Folder and put this in: ~/Library/Containers


2.  Find the folder called com.microsoft.Windows-Phone-7-Connector and drag it to the Trash.


Regenerate your settings

1.  Now, open iTunes and give iTunes up to half an hour to fully recreate the XML file. The larger your iTunes library is, the longer it will take to recreate this file. Usually, half an hour is a safe bet.

2.  Now open the Windows Phone Connector, setup a sync relationship with your phone and try syncing again. You should have no more iTunes syncing errors!

    Styles reformatting may be caused by "Automatically update document styles." Turn off that option in the Templates and Add-Ins dialog box. To display the dialog, first display the Developer tab on the ribbon (via File tab | Options | Customize the Ribbon) and then click Document Template on the Developer tab.

    If by "sub docs" you mean the master document feature, then I'd advise against it. It has been buggy for years and should be avoided. See http://word.mvps.org/FAQs/general/RecoverMasterDocs.htm.

    I have never heard of a bug that causes data loss in large documents. Document corruption seems to be a more likely cause of data loss. See http://support.microsoft.com/kb/918429/en-us/ (How to troubleshoot damaged documents in Word 2007 and in Word 2010). To avoid corruption in the future, one thing to watch out for is saving documents directly to removable media (such as USB flash drives). Instead, save to the hard drive and copy to removable media as needed.

    anthony.b say: My suggestion would be to ensure that your document has a very limited set of styles.  For example, my own personal template that I use for my longer Word documents has ONLY styles for Heading 1, Heading 2, Heading 3, Normal, a 36-point main-page Heading style, and a few styles that are auto-generated by the Table of Contents wizard (TOC Heading, TOC 1, etc.).  I think that if the styles are limited to a very small set, then this reduces chances for problems if the document grows over 100 pages.  I used the above-mentioned technique (limited set of styles) for several Word documents that I worked on, and this has worked well for me.  My idea is that if Word has hundreds of styles to keep track of, that bogs it down, especially if the document is over 100 pages.  However, all of the above is just my opinion, and theory on the matter.  Just wanted to throw it out there, in case it might help.

    Mail Merge and any form of restriction of a document are mutually exclusive.

    If none of the mergefields are located in-line with text, one way of preventing changes to the text in the document would be to use copy and paste special to paste it into the document in the form of a picture.

    Another way that you could do it would be to use the Merge with Legacy Formfields utility on my Merge Tools Add-in.  You need not have any formfields in the document, but you would use the Protect Legacy FormFields utility that is included with that add-in to apply the protection to the document and then use the Merge with Legacy Formfields utility to execute the merge.

    You can download the MergeTools – 20150422.dotm Add-in that I created from the following page of my One Drive: http://bit.ly/1hduSCB


    The MergeTools – 20150422.dotm file needs to be saved in the Word Startup folder.  In Windows Vista and Windows 7, 8 or 8.1, and 10, the default location for that folder is

    C:\Users\[User Name]\AppData\Roaming\Microsoft\Word\STARTUP

    If you do not see the AppData folder: -

    In Windows 7, - In Windows Explorer, click on the Organize drop down and then on Folder and search options and in the Folder Options dialog, go to the View tab and select the item "Show hidden files, folders, and drives".  While there, it is a good idea to uncheck the box for "Hide extensions for known file types".

    In Windows 8,  8.1 or 10, in the File Explorer, click on Options on the View tab of the ribbon and then on the View tab in the dialog that appears and select the item "Show hidden files, folders, and drives".  While there, it is a good idea to uncheck the box for "Hide extensions for known file types".
     When that has been done and Word is started\re-started, the tab shown below will be added to the Ribbon:


    The requirements for using the system are:

    1. The mail merge main document must be of the Letter type, though that does not mean that the output cannot be sent as an e-mail message. 

    2. For the Many To One, Merge with Attachments and Merge to Individual Docs utilities, the data source may be either a table or query in an Access database, or in the form of an Excel worksheet and that worksheet must be the first sheet in the Excel workbook. If the data is on some other sheet, you can easily move that sheet so that it is the first sheet in the workbook by clicking on the sheet tab and dragging it to the left.  For the Chart Merge utility, download the Mail Merging with Charts document that is also on that page of my OneDrive for additional requirements of the data source for use with that utility

    3. For a data source in the form of an Excel worksheet, the field names must be in the first row of the worksheet and there must be a field name in all of the cells in that row that are within the range of columns that contain the data.

    4. For both types of data source, the field names must contain only alphanumeric characters (No @,#,$,%,&,(,), etc) and the field names must not start with a numeric character (0-9).

    5. For a data source in the form of an Excel worksheet, the data must start in the second row of the worksheet and there should be no empty rows within the range of data that is to be processed.

    6. If there are multiple addressees in either the To or CC fields of the data source, the email addresses need to be separated by a semi-colon.
