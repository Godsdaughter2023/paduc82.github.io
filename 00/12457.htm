VBA code for Access - compare tables - Microsoft Community

larana626 say:

VBA code for Access - compare tables

I need vba code to compare two tables in an access database. The code will use the primary key "ID" on both tables and will pull data from Table1_field3 to populate table2_field5 in the second table.

Here are more details:

Table1

Id: primary key

field3: data to be "moved" to table 2 if there is a matching record in table 2.

Table2

Id:primary key

field5: field to be updated by data from table 1

The code will also create a table with the "unmatched" IDs between table1 and table2 (as an additional table)

Thank you.

john p spencer say:

You should use queries to do this.  The queries are much more efficient at processing records than using VBA

UPDATE Table2 INNER JOIN Table1
ON Table2.ID = Table1.ID
SET Table2.Field5 = Table1.Field3

To get the unmatched records all in one query you would use a Union query
SELECT Table1.ID, "No Match in Table 2" as Unmatched
FROM Table1 LEFT JOIN Table2
ON Table1.ID = Table2.ID
WHERE Table2.ID is Null
UNION
SELECT Table2.ID, "No Match in Table 1" as Unmatched
FROM Table1 RIGHT JOIN Table2
ON Table1.ID = Table2.ID
WHERE Table1.ID is Null

Union query truncates the field contents of memo fields - Microsoft Community

tonia king say:

Union query truncates the field contents of memo fields

On a previous project I encountered issues when using a union query to combine a number of select statements where one of the fields was formatted as memo.  Essentially when that field contained a lot of text the union query only returned a portion of the contents.  Standalone select statements worked fine. 

Is this a known bug? Or is there something I can do to prevent the field contents being truncated.

I am about to work on a new project which will need a union query to combine a large number of select statements containing a memo field, which may have large amounts of data within, and hoped there was a resolution.

john w. vinson say:

"That's not a bug, that's a feature"  <g>...

A UNION query removes all duplicate records. Since a Memo field could contain billions of bytes, the developers chose to avoid the possibly very timeconsuming task of comparing one billion byte memo field to another billion byte memo field by truncating all memos to 255 bytes.

If you don't need to deduplicate the records - i.e. if you're confidant that the records in the different tables are already different - just use

UNION ALL

instead of UNION in the query. It won't try to deduplicate, it won't truncate, and as a bonus the query will run faster.
