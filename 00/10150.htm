Hi Natasha,

The default calendar permission of a newly created Office 365 user inside the organization is Free/Busy, which cannot be changed automatically upon the account creation for the time being. Thanks for your understanding.

Meanwhile, I understand the inconvenience caused in this case. And we welcome and encourage you to share your ideas through office365.uservoice.com/.../273494-outlook-and-calendar. Understanding your experience helps us to make our product and service better for you and others. Actually, many features came directly from your feedback.

On the other hand, besides asking the newly created user to share his/her calendar manually, we can use remote PowerShell to modify the default permission of all the existing mailboxes (after they are created) as an Office 365 admin. Here are the detailed steps for your reference:
1. Connect to Exchange Online using remote PowerShell
2. Run the following 2 cmdlets separately:
$calendars = Get-Mailbox -RecipientTypeDetails UserMailbox | Get-MailboxFolderStatistics | ? {$_.FolderType -eq "Calendar"} | select @{n="Identity"; e={$_.Identity.Replace("\",":\")}}

$calendars | % {Set-MailboxFolderPermission -Identity $_.Identity -User Default -AccessRights LimitedDetails}
Note: These cmdlets change the default permission to LimitedDetails, you can also replace LimitedDetails with the permission you want if necessary. Here’s an article regarding Calendar permissions that may be helpful: https://support.microsoft.com/en-us/kb/2865291.

Let us know if anything is unclear.

a. user say:

You may be able to do what you need by using hard line returns (shift-Enter instead of Enter) instead of regular paragraph marks.

If you do that, entries in any ToC will still only use 1 line and when you insert a cross-reference to a heading, the complete heading will be treated as a single heading.

However, for a cross-reference, it will only *really* work if the "Chapter 4" part of the heading is inserted using Word's automatic numbering (one of the standard multilevel schemes is set up that way. In that case, to get the complete chapter heading, e.g. Chapter 4 Title of the Chapter, you would need to insert two cross-references to either to the "Heading" or the "Numbered Item", with a space in between, i.e. using a cross reference to the Heading number, followed by a space, followed by a cross reference to the heading text. If you don't use automatic numbering, you only get to include the entire text of the heading, including the hard line ends, which probably isn't what you want.

suzanne s. barnhill say:

See http://wordfaqs.ssbarnhill.com/TOCTips.htm#UnnumberedHeadings

This will do what you want, but you do need to read it carefully. In your case, since you need the numbering for captions, you will use to use the built-in heading style for the chapter number and a custom style for the chapter title.

SOLVED!!!

I figured it out. And I'm a little disappointed in the Microsoft help desk for not realizing this issue. It has to do with two-factor authentication. Custom page sizes always have to be set up in portrait format, with the smallest dimension first.
