MS Project How to track % progress against original baseline programme

Hello,

 

Would anyone know how (On MS Project 2010) would I show percentage completion against my baseline programme (where my baseline programme should be at).

 

I.e. the overall percentage complete for my project is currently 12% but I need info on percentage on track overall. ( 12 % but should be for example 20 % complete as per baseline programme)?

 

john - project say:

EoghanH,

First, I'll assume you set a baseline and that your version of Project 2010 is fully updated.

There is no baseline percent complete field but you can create one using a couple custom fields. First customize a Duration field with the following formula:

Duration1=ProjDateDiff([Baseline Start],[Current Date])

Set the "calculation for task and group summary rows" as a summed rollup

Then customize a Text field with the following formula:

Text1=[Duration1]/[Baseline Duration]*100 & "%"

Set the "calculation for task and group summary rows" to use the formula

The custom text field (Text1) will give equivalent values for "expected complete" as of the current date.

Hope this helps.

John

john - project say:

colin_e_uk,

Yup, you definitely have a corrupt structure. I'm glad your files are only test files and not real files.

I assume that "OTT" means over the top, and yes, it is.

First, do not put the files in different folders. Keep everything in a single folder and ideally not on a server. By moving and renaming the files during your experiment, you effectively created multiple duplicate links. It's a mess to clean all that up but that's what you need to do. You might find the macro below useful. I wrote it some time back to automate unlinking a resource pool/sharer file structure. Just open the pool file and run the macro. Then you can check the pool file to make sure there are no rogue files linked to it. Do the same with each sharer file. When all looks good, you can re-establish sharing.

A static master is created by first opening a new blank file and then going to Project/Insert group/subproject. Select all sharer files and unchecking the "link to project" option in the lower right corner of the Insert Project window. That will create a snapshot in time of all the sharer files. You will need to build a new static master each time you need to do a report but it won't contribute to corruption.

Some users shy away from a single large file but if it's not too large (e.g. less than 10K tasks) filters and/or grouping can be used to make it more manageable. A file larger than that is probably a good candidate for moving to Project Server although a linked structure will work if all users are fully trained and extreme discipline is used to manage the files. It's that last part that is sometimes hard to instill in all users.

John

Sub UnlinkSharerFil()
Dim Shar() As String
Dim sp As Subproject
Dim NumShar As Integer, i As Integer
Dim PoolNam As String
'open all sharer files into temporary master, count them and
' set up array to hold sharer file path names
ResourceSharingPoolAction Action:=pjOpenAllSharers
NumShar = ActiveProject.Subprojects.Count
ReDim Shar(NumShar)
i = 0
'fill array with sharer file path names
For Each sp In ActiveProject.Subprojects
    Shar(i) = sp.SourceProject.FullName
    i = i + 1
Next sp
'close temporary master and get name of pool file
FileCloseEx Save:=pjDoNotSave
PoolNam = ActiveProject.Name

'open sharer files
For i = 0 To NumShar - 1
    FileOpen Name:=Shar(i)
Next i
'activate pool file and unlink all sharer files
Projects(PoolNam).Activate
For i = 0 To NumShar - 1
    ResourceSharingPoolAction Action:=pjUnlinkSharer, FileName:=Shar(i)
Next i
'save changes
FileCloseAllEx Save:=pjSave
End Sub
