karl timmermans [outlook mvp 2012-2018] say:

We are using Pop3/SMTP protocol.  Why would it not be syncing?  Thank you

POP3 accounts do not "sync" any folder. POP3 protocol only "downloads" items from the <Inbox> and if the "Leave Messages on Server" is unchecked, all items from the <Inbox> are removed from the server. If it is checked, then any other computer connecting to the same account will be able to download the items from the <Inbox> (also dependent on how long the items are set to remain on the server)

ronrosenfeld say: Just split up the string using the space delimiter:

to return True or False:

?matchword("75","123 375 93740 14007589 10 454501 789")
======================================
Option Explicit
Function MatchWord(sWord As String, sString As String) As Boolean
    Dim v As Variant
    Dim i As Long
v = Split(sString)
For i = 0 To UBound(v)
    If sWord = v(i) Then
        MatchWord = True
        Exit Function
    End If
Next i
End Function
===============================

To return a count:

countmatch("75","123 375 93740 14007589 10 454501 789")
=================================
Function CountMatch(sWord As String, sString As String) As Long
    Dim v As Variant
    Dim i As Long
v = Split(sString)
For i = 0 To UBound(v)
    If sWord = v(i) Then
        CountMatch = CountMatch + 1
    End If
Next i
End Function
=============================

To use this for alpha words in sentences, one would also have to account for punctuation and line breaks -- probably by removing them or replacing with spaces before processing; decide about case sensitivity, etc.  Or one could just use Regular Expressions.

ronrosenfeld say:

If you want something very flexible in this regard, you could use Regular Expressions.  There's already a .dll for that, distributed with Office.  "Microsoft VBScript Regular Expressions 5.5"  However, for something as simple as your request, the code I provided would be much faster.   A regex pattern, though, could be more flexible.  For example, to detect any single word in a string, where a "word" is a string consisting only of digits, letters and underscores, could be simply:  "\b" & your_word & "\b".  The "\b" is a word boundary token, and takes care of problems in dealing with punctuation at the beginning or end of a line.

 

Another method to solve your particular problem, if speed is an issue because you are executing many loops, would be to split the string into words and add the words to a Collection object.  If you then try to add an identical word, it will result in an error, which can be tested for and handled.

w.m.mitchell say:

Merge to labels from Excel sheet but only selected records, how to?

I have a 2-column spreadsheet; the first column is a selection column where user just types in an "X" if the address is needed in the merged label printout.  Column 2 has the actual addresses.

Is there a way to have the Word merge just pick up the selected records according to the ones that just have an "X" in column A?  We require printing random labels each week according to what comes in the mail to be responded to.  They've been labelling envelopes by hand all this time for each and every piece of correspondence that needs a response which is inefficient and time-consuming.  But having a label "database" in Excel wouldn't work unless we could filter by the entries we actually need at any given moment.  That way we'd print out on a sheet a labels just what was needed that day.

I've done a few merge docts in my time but don't know how to filter through to a merge only a set of specific entries.

w.m.mitchell say: Thank you so much for your help.

I actually just had a very interesting time ... I stumbled upon an automated way completely by accident, so this is an important post.  No need to filter manually at all.  Word can do it!

This morning I created the Word 2003 label file and an  Excel 2003 address data source file.  I chose a handful of the addresses out of all of them by placing an X for some in the A column.

Then I ended up enabling the merge bar as I couldn't remember where the "start merge" command was.  That turns out to have been a godsend.

I then worked and tested and got output labels exactly as needed without doing anything or sorting anything in a special way.  But then panic because I renamed the Word file and had the devil of a time figuring out how to get the merge automatically filtered by selected entries.

But here it is - the magic button is in the 3rd button on my Word 2003 merge bar with tooltip "Mail Merge Recipients".  When you press this button a box with an Excel table-like display comes up and under the label for the column that uses X's, the one with header labelled "Select", I just chose to filter by X (click on the pulldown arrow and choose the cell content option you want, which was "X" in my case, much like when you filter a column by one of the entries in the main Excel interface).

And if you run this several times and change which records you select, the merge comes out by those chosen records one having to filter the Excel file itself.  Word does it behind-the-scenes.

Incredibly awesome!!

Warning, though, every time I renamed even just the Word file, I had to re-select the data source and then select the same filter option via the "Mail Merge Recipients" button on the merge bar and then filtering by the X.  If you don't select this filtering in the recipients, you get a straight mail merge using _all_ the entries, without any filtering.

This is incredible and I can't believe how very, very lucky I was able to stumble upon it quite by accident and then to be able to find it again after renaming the file!

This is great.  I can avoid our department buying a label writer and saving money by making this merge file process available on the shared drive to our 2 teams of 4 people.
