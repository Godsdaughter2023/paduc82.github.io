I offer these crude notes I made after resolving this problem on another system:

The problem is that that .GIF files (images) that get extracted from the installation have become corrupted and must be replaced.  The image files are "built in" to the installation and must be extracted again from the installation of the Linksys program.

Application Error
Exception GIFException in module aviWUSB54Gv4.dll

WUSB54Gv42.exe - Application Error from Linksys wireless adapter

The instruction at "0x0044408d" referenced memory at "0x00000168".  The memory could not be "read".
Click on OK to terminatate the program
Click on CANCEL to debug the program
Click Canel to continue

WUSB54Gv42,exe Application Error

After reading about things to try (that did not work), I did this:

Reinstalled - same problem
Noticed a missing "hole" in Notification Area where I thought the Linksys icon used to be
(although I keep that icon always hidden)

I read that the solution was to remove c:\windows\temp\WUSB*.gif files (they are corrupted), reboot, but I had the same problem.

To really fix it without trying a bunch of things (I already tired them all):

Stop the WUSB service and change it from from Automatic to Manual.

Start, Settings, Control Panel, Administrative Tools, Services)

Find the WUSB... Service, Stop the service and and set the Properties to Manual.

Navigate to this folder:

c:\windows\temp

Delete c:\windows\temp\WUSB*.gif (delete all the .gif files - 4 files I think)

Restart the system (with no WUSB service running since it was set to manual from before).

Wait a few minutes for things to settle down, but no connection to the wireless network was noted since the service is set to manual.

Go back into Services and Start the WUSB service manually and it will extract the GIF files it needs.

Four (or maybe five) WUSB*.gif files then appear in c:\windows\temp but there is no complaining.

The Linksys icon then reappeared in the Notification Area and I was connected.

Let things settle down for a few minutes and test the Internet connection.

Go back into Services and change the WUSB service from Manual to Automatic so it would start on the next reboot.

Reboot and resolve any issues.

Marked the Linksys display icon in the Notification Area as Always hide.

Download drivers from the Linksys page:
http://homesupport.cisco.com/en-us/wireless/linksys/
http://homesupport.cisco.com/en-us/wireless/lbc/WUSB54G
