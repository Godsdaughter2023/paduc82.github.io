Okay, fine.  All the button is doing is opening the query named "Training Report Tracker" as a datasheet.  What you want this new button to do is *export* that query to an Excel workbook.  That's easy enough.

This could be done with either VBA code or a macro.  I'm most comfortable with VBA coding, so I'll give you instructions for that.  However, if you want to use a macro instead for whatever reason, let me know and I'll give you macro instructions.

Here are the steps.

1. In design view of the form, place a new command button on it.  Name the button "cmdExport".  Give it whatever caption and/or image you like.

2. Open the property sheet of the button.  On the Event tab of the property sheet, click on the "On Click" line, so that you see the "build" button at the end of the line.

3. Click the "build" button.  I expect it to show a dialog prompting you to choose "Macro Builder", "Expression Builder", or "Code Builder".  Choose "Code Builder", and click "OK".

4. Access will open the VB Editor to display the form's code module, and create the shell of an event procedure for the button's Click event, looking like this:

    Private Sub cmdExport_Click()

    End Sub

Add code inside that procedure, like this:

    Private Sub cmdExport_Click()

        DoCmd.TransferSpreadsheet acExport, _
            TableName:="Training Report Tracker", _
            FileName:=CurrentProject.Path & "\TrainingReportTracker.xlsx", _
            HasFieldNames:=True

    End Sub

5. Save the module and form.

6. Open the form in form view and test out the button.

The above code is set up to write the file to the folder that the database is in, as returned by CurrentProject.Path .  You can change that to some other folder, if you like.  I believe that if you don't specify the path, and just say

    FileName:="TrainingReportTracker.xlsx", _

it will go into the whatever folder Windows considers to be "current", which may or may not be the one containing the database.

Note: I've assumed you are running an installation of Office 2007 or greater, so that it will create an XLSX file by default.  If that's not the case, we'll need to modify the code slightly.

EDIT:  I notice after I posted that the code line specifying the FileName is wrapped onto two lines.  It should only be one line, from "FileName:=" to the terminal continuation character "_".

  For Each Cell In Selection
    Cell.Hyperlinks.Add Cell, Cell.Text
  Next
End Sub

Just select your cells and then run the macro.

Try the example code below.

The code includes an Input Box to enter the required search criteria.

Note the Prompt in the InputBox indicating that the user is to insert the wild cards in the criteria. This gives the user more control instead of hard coded values to search for and/or adding wild cards after the user enters initial search criteria.

Individual question marks (?) can be used to represent one wild card character and asterisk for any number of characters.

The code attempts to identify the output worksheet. If it does not exist then the code creates it. If it does exist then the code clears any existing data.

The code excludes the output worksheet from the searches.

Sub SearchForData()
    Dim strPrompt As String
    Dim strTitle As String
    Dim varTofind As Variant
    Dim rngToFind As Range
    Dim wsResults As Worksheet
    Dim ws As Worksheet
    Dim rngDestin As Range
    Dim strFirstAddr As String
   
    strPrompt = "Enter value to find." & vbCrLf & _
                "Include wildcards as required." & vbCrLf & _
                "or Cancel to exit."
               
    strTitle = "Enter search value"
   
    Do
        varTofind = Application.InputBox(Prompt:=strPrompt, _
                                        Title:=strTitle, Type:=2)
                                   
        If varTofind = False Then
            MsgBox "User cancelled. Processing terminated."
            Exit Sub
        Else
            Exit Do
        End If
    Loop
   
    On Error Resume Next
    'Test if worksheet "Searched Results" already exists
    Set wsResults = Worksheets("Searched Results")
    On Error GoTo 0
   
    If wsResults Is Nothing Then    'If nothing then sheet does not exist
        Set wsResults = Sheets.Add(After:=Worksheets(Sheets.Count)) 'Add the sheet
        wsResults.Name = "Searched Results"     'Name the sheet
    Else
        wsResults.Cells.Clear      'If already exists then clear the sheet
    End If
   
    'Insert column headers on the Search Results sheet
    With wsResults
        .Cells(1, "A") = "Value Found"
        .Cells(1, "B") = "Sheet Name"
        .Cells(1, "C") = "Cell Address"
        .Range(.Cells(1, "A"), .Cells(1, "C")).Font.Bold = True
    End With
   
    For Each ws In Worksheets   'Loop through all worksheets
        If ws.Name <> wsResults.Name Then   'Do not process sheet containing results
            With ws.Cells
                strFirstAddr = ""     'Initialize to nothing at start of each sheet
               
                Set rngToFind = .Find(What:=varTofind, _
                                LookIn:=xlFormulas, _
                                LookAt:=xlWhole, _
                                SearchOrder:=xlByRows, _
                                SearchDirection:=xlNext, _
                                MatchCase:=False, _
                                SearchFormat:=False)
               
                If Not rngToFind Is Nothing Then    'Not Nothing then is something so found
                    strFirstAddr = rngToFind.Address    'Save address of first found cell on sheet
                    Do
                        With wsResults      'Output results to results sheet
                            Set rngDestin = .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0)
                            rngDestin = rngToFind.Value
                            rngDestin.Offset(0, 1) = ws.Name
                            rngDestin.Offset(0, 2) = rngToFind.Address(0, 0)
                            'Following line is Alternative to previous line with absolute reference
                            'rngDestin.Offset(0, 2) = rngToFind.Address
                        End With
                        Set rngToFind = .FindNext(rngToFind)
                        If rngToFind Is Nothing Then Exit Do
                    Loop While rngToFind.Address <> strFirstAddr
                End If
            End With
        End If
    Next ws
    wsResults.Columns.AutoFit
    Application.Goto wsResults.Cells(1, 1)
    MsgBox "finished"
End Sub

Sub nslookup()

Dim strShellCommand As String
strShellCommand = "C:\Windows\System32\NSLOOKUP.exe " + strCompAddress

'//Create Shell Object in to run Scripts
Set oSh = CreateObject("WScript.Shell")

'//Read output buffer
strBuf = oEx.StdOut.ReadAll
ActiveSheet.Cells(i, 2) = strBuf
i = i + 1
Wend

In Word07 the Quick Parts button is found on the right side of the Insert Tab in the Text group. Below is a quick summary of my understanding of Quick Parts button options.

When you click the Quick Parts button a drop-down or gallery of options pops up including from top to bottom:

  • Document Property: when selected a list of commonly used form fields pops up. When one of these fields is inserted it will appear in the document in a Content Control. 
  • Field: when selected a dialog box will open with a large list of insertable form fields
  • Building Blocks Organizer: when selected opens a dialog box (or gallery) with a list of insertable Building Blocks (collections of fields preformatted in common configurations like a cover page, ToC, etc.)
  • Get More on Office Online: opens a browser allowing you to download content
  • Save Selection to Quick Part Gallery: lets you save back to the Building Blocks Organizer
