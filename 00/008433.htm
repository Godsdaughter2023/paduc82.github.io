You need to go into edit mode to change it. Open message > Actions (Move group) > Edit message.

 

if its something you'll do fairly often, you can use a macro - add it to a button and click it when you want to convert to plain text and reply.

 

This works with open or selected messages -

 

Sub ReplyPlain()
Dim objItem As MailItem
Dim oMail As MailItem
           
    On Error Resume Next
    Select Case TypeName(Application.ActiveWindow)
        Case "Explorer"
            Set objItem = Application.ActiveExplorer.Selection.Item(1)
        Case "Inspector"
            Set objItem = Application.ActiveInspector.currentItem
    End Select
       
    objItem.BodyFormat = olFormatPlain
    Set oMail = objItem.Reply
oMail.Display
End Sub

teylyn say:

Hello,

there are many approaches to this. Formulas, VBA, Power Query. The best approach depends on your comfort level with the technology and whether you need to do this once or repeatedly. 

For an easy approach that works well once, try this:

- ensure the ID column is in column A in both sheets

- make a copy of one of the sheets, let's say copy sheet 2 and call it AllData

- in the AllData sheet, enter this formula into the next empty cell in row 1

= iferror(Vlookup(A1,Sheet1!$A1:$Z1000,Column(B1),false),"no Match")

- copy this formula to the right for as many columns as you have in Sheet 1. You should now see the column names.

- copy the formula down all rows. If you have more than 1000 rows, adjust the formula before copying down. 

- copy all the formulas and use Paste Special > Values to overwrite the formulas with the results. 

- filter or sort and delete the rows with "no Match"

Extra space on the right side of my document? - Microsoft Community

robnanya say:

Extra space on the right side of my document?

Why is there an extra space on my document.  I would like to remove it.

It throws everything off when printing a page, plus looks funny because the page is not centered.

bob jones aka: cybertaz say:

What you are seeing is called the Markup Area. It appears because the document contains Tracked Changes and/or Comments with the preference set to display them in Balloons. Additionally, on the Review tab the viewing option is set for either Final or Original Showing Markup. That combination also causes the Print dialog to default to the Document Showing Markup option. You have several options, including...

  • If there's no need to retain the markup: Turn off Track Changes, Accept/Reject all currently tracked changes & delete all Comments. That will cause the display panel to go away altogether & resolve the printing concern.

If the Changes/Comments must be retained;

  • Turn off the preference to display markup in balloons, or
  • Select Final from the viewing choices in the Tracking group on the Review tab, or
  • Explore other combinations in the Show Markup list in that same group.

As long as the markup is not displayed in the document it will not be included in the printing. If the markup is retained & displayed you can choose Microsoft Word in the Copies & Pages list of the Print dialog, then select Document rather than Document Showing Markup.

robnanya say: I went to View and unchecked Markup.  It worked!
