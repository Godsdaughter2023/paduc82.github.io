As an alternative, here is code that I use to format certain text in a form as double-strike-through, and turn that off. Hidden could be used instead. You would skip the part about bold.

Sub ThirdParty()
'
' ThirdParty Macro
' OnExit Macro for Third-Party Checkbox
' "&chr(10)&"Macro written 05/16/2001 by Charles Kyle Kenyon
'
    Dim rRange As Range
    With ActiveDocument
        .Unprotect
        Set rRange = .Bookmarks("ThirdPartyLanguage").Range
        rRange.Font.DoubleStrikeThrough = _
            Not .FormFields("chkThirdParty").CheckBox.Value
        rRange.Font.Bold = _
            .FormFields("chkThirdParty").CheckBox.Value
        .Protect wdAllowOnlyFormFields, NoReset
    End With
End Sub

If a box is checked, the double strike through is turned off and bold is turned on. If the box is not checked, double strikethrough is turned on and bold is turned off.

Vertical spacing problem between fields in a Report - Microsoft Community

craigbr1 say:

Vertical spacing problem between fields in a Report

Hi,

I got about 10 fields vertically oriented top to bottom in an Access Report with a little even space between them vertically...if there is data in all 10 fields then it looks great, however, if a record only has data in a few fields (e.g. 1st, 4th, 8th fields or 2nd, 5th, 9th fields), then there's too much of a gap between the fields.  (I tried setting shrink property to yes for all 10 field controls as well as the detail section but that did not solve the problem)

If I reduce vertical spacing between all the fields, the gaps do go away for the records with only a few fields filled in, however, it makes the fields too close together, especially for those with data in all fields. 

Any ideas how I can close the vertical spacing gap in my report but only for those records where there is no data in several of the fields?

Thanks very much

Craig

ken sheridan say: Just increase the vertical size of the controls so that they are all vertically contiguous, but leave the font size as is.  When controls shrink any 'white' space above or below them doesn't shrink, so you end up with irregular vertical spacing.  By eliminating the 'white space' in this way the problem is avoided.  If the control's have visible borders then these will be at the full size of the control of course, which may or may not be acceptable.
Ken Sheridan, Stafford, England

marshall barton say:

This problem will yield to a simple but sneaky trick.

First, make the attached labels for each of your text boxes Height so it goes down to the top of the next text box below.  This will maintain the spacing you curently have when there is a value in a text box.

Next, add a new text box below each of your existing text boxes.  Set their CanShrink to Yes and ControlSource expression to =Null so they won't do anything but shrink.  Their Width doesn't matter so you can make them very narrow.  Position them with their Top at the bottom of your text box and set their Height to include the blank space down to your next text box below.

Finally, for each of your text boxes, add a line of code to the detail section's Format event:
   Me.yourtextbox.Visible = Not IsNull(Me.yourtextbox)

When the value of one of your text boxes is null. it  and its attached label will become invisible.  That will allow the section to shrink because the =Null text box will always shrink.
