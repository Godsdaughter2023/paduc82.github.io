The reference count of an object is illegal for the current state of the object. Each time a driver uses a pointer to an object, the driver calls a kernel routine to increase the reference count of the object by one. When the driver is done with the pointer, the driver calls another kernel routine to decrease the reference count by one.

Drivers must match calls to the routines that increase (reference) and decrease (dereference) the reference count. This bug check is caused by an inconsistency in the object's reference count. mmmelaney say:

I posted this in another thread.  Hope it helps.

I'm installing fresh-installing Windows 8.1 into a new Lenovo laptop.  I got into a loop at "Finalizing your settings...."  Here's how I solved it:

For Windows 8 or 7:

*Physically turn off any toggle buttons like wifi, webcam, bluetooth, etc.

*Do  a hard boot -- hold power button for 30 seconds with AC power and battery removed.
*Disconnect all external devices -- even the battery.
*Remove all RAM except for 1 module -- also remove it for 5 minutes or move it to another primary slot to be sure RAM is cleared (techie trick!)

*Reconnect the AC charger only -- NOT the battery.

*Boot into the Bios settings and disable all ports accept Hard Drive (HDD) and Memory (RAM) if possible.

*Set all USB 3.0 ports to 2.0 (or Legacy.)  Disable charging from USB.  Turn off charging in "off" state.

*Save settings (F10) and reboot to Bios again. **See next steps!

***If you have no "Secure Boot"/UEFI/EFI settings in your Bios you're done!  Save and boot to Windows.  If successful, celebrate!  Yay!  Now undo previous changes in the checklist above.  Reboot to Windows and continue with driver installation.

***If you are booting into an MBR type OS from UEFI bios (you'll know if you are) save and boot to Windows. You're done!  Now undo previous changes in the checklist above.  Reboot to Windows and continue with driver installation.

For GPT/UEFI computers:

*After saving settings above, reboot to Windows.  If successful, you're done! Skip the steps below.

*Boot into the Bios again.

*Enable UEFI Only in your Startup or Boot options.

*Disable CSM Support.

*For Windows 8:  Find the settings for Secure Boot.  See if there is a setting to enable Setup Mode or Reset Secure Boot.  Enable it.  (This is not the Bios "Reset" or "Setup Defaults"! Only Secure Boot.)  If you don't see this option, skip this.  (This will put Secure Boot into Custom Mode so that any Windows License key pre-installed in the Bios does not interfere with the operating system setup.  ***Note:  Windows will not be activated!  You will need to Activate Windows with you're own license key, or come back into the Bios later to "Restore Factory Keys" in the Secure Boot options.***

All Done?  Now undo previous changes in the checklist above.  Reboot to Windows and continue with driver installation and OS setup.  Hooray!
