jay freedman say:

The problem here is that VBA, and Word in general, doesn't have a real concept of a "line" except on the screen. The point at which the words wrap to the next line depends on a collection of factors: The individual widths of the characters (which usually changes when you apply Bold to them), the particular font and size of the text, the settings of the margins and paragraph indents, and even the identity of the printer driver that's currently selected.

On the screen, you can use Shift+End to select from the starting cursor position to the end of the line as it exists at the moment, and apply formatting to the selection. If the line is full and there's no room in the existing word spaces to compress it, you'll see that applying Bold to it may make the last bold word go to the next line.

In VBA, the equivalent of pressing Shift+End is the statement

   Selection.MoveEnd Unit:=wdLine, Count:=1

For reasons known only to Microsoft, this doesn't work with a Range object; if you try it, you get a "Bad parameter" runtime error.

Summing up, if you start with the second macro that I posted earlier, it can be modified like this:

        

Sub x()
 Dim rngfound As Range, rngadd As Range
 Set rngfound = ActiveDocument.Range
 With rngfound.Find
     Do While .Execute(FindText:="(\<TYP\>)([0-9]{2})(\<?TYP\>)", MatchWildcards:=True, _

             Forward:=True, Wrap:=wdFindStop) = True
        Set rngadd = rngfound.Paragraphs(1).Range
        rngadd.MoveEnd wdCharacter, -1
        Select Case Mid(rngfound.Text, 6, 2)
            Case "17"
                rngadd.InsertAfter "text for 17"
                Set rngadd = rngfound.Duplicate
                FormatRange myRange:=rngadd
            Case "18"
                rngadd.InsertAfter "text for 18" 'as an example
                Set rngadd = rngfound.Duplicate
                FormatRange myRange:=rngadd
        End Select
        rngfound.Collapse wdCollapseEnd
     Loop
 End With
End Sub

Sub FormatRange(myRange As Range)
    With myRange
        ' extend range to end of current line
        .Select
        Selection.MoveEnd Unit:=wdLine, Count:=1
        Set myRange = Selection.Range
    End With

    ' format
    With myRange.Font
        .ColorIndex = wdBlue
        .Bold = True
        .Underline = wdUnderlineSingle
    End With
End Sub

VBA for Word, I want to get material from file A and paste it in File - Microsoft Community

lewis d. eigen say:

VBA for Word, I want to get material from file A and paste it in File B, but only have file B visible on the screen. How is this done?

VBA for Word, I want to get material from file A and paste it in File B, but only have file B visible on the screen.  How is this done?

Actually, I have a loop and at each cycle of the loop I copy some material from File A and Paste it into File B.  However I want the user to be able to see File B in his/her screen including the dynamics of the material being pasted in.  However, I do not want the user to see File A at all.  Within the loop I go back and forth changing the Active Document to A, copying the material, and then making B the active document and pasting.  Then repeating for new material.

If I do nothing special the screen image changes from File A to File B and back again as each becomes active.  I know how to turn off both of the screens to make the macro run faster, but speed is not a problem here -- visibility is and I want to display B.  How do I show B and suppress A?

jay freedman say:

To answer your main question, you can open a document in Word and prevent it from appearing on screen by setting the Visible argument of the Documents.Open command to False.

For something you didn't ask for, it's much faster (and doesn't kill anything that may already be on the clipboard) to use two Range objects -- one in each document -- and set the .FormattedText property of the destination range equal to the .FormattedText property of the source range. Using .FormattedText instead of just .Text transfers both characters and formatting, including styles.

Here's sample code:

Sub x()
    Dim DocA As Document
    Dim DocB As Document
    Dim p As Paragraph
    Dim rg As Range
    
    Set DocB = ActiveDocument
    Set DocA = Documents.Open("DocA.docx", Visible:=False)
    
    For Each p In DocA.Paragraphs
        Set rg = DocB.Range
        rg.Collapse wdCollapseEnd
        rg.FormattedText = p.Range.FormattedText
'        WasteTime 2   ' slow the macro so user sees one at a time
    Next p
    DocA.Close wdDoNotSaveChanges
End Sub

[The WasteTime routine is at https://www.myonlinetraininghub.com/pausing-or-delaying-vba-using-wait-sleep-or-a-loop, should you want to use it.]

Don't forget to close Document A when the macro ends; otherwise it will hang around invisibly, although you can get to it through View > Switch Windows.

doug robbins - mvp office apps & services (word) say:

As shown in Jay's code, the Set command is used to open a document in the following manner

Set DocA = Documents.Open("DocA.docx", Visible:=False)

though often, in place of "DocA.docx", you may need to specify the fullname (path, filename and extension) of the document that you want to open.  Note that when using the Set command, the argument of the Documents.Open needs to be enclosed in parentheses.  

Having used the Set command, you do not need to use ActiveDocument to refer to the document opened by the use of that command.

The Set command is most useful (and in fact needs to be used) when multiple documents are being used as it provides a way of clearly distinguishing one from the other.
