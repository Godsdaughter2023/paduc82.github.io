Slow movement between cells for data entry - Microsoft Community

shaun mcgregor say:

Slow movement between cells for data entry

There is a pause each time a click on or use the cursor keys to move to a new cell. if I try to enter data during this pause the program says it is not responding. After about 10 seconds I can enter data.

joeu2004 say: Shaun McGregor wrote:
There is a pause each time a click on or use the cursor keys to move to a new cell. if I try to enter data during this pause the program says it is not responding. After about 10 seconds I can enter data.

There are many possible explanations.  For example, you do not say whether or not you might using Excel over a network.  (That is, Excel is running on remote server.)

But a common cause for local applications is having many formulas that access volatile functions.  Every cell that accesses a volatile function and their dependent cells are recalculated every time  any  cell in the workbook is edited.  You might not realize that Excel is busy recalculating when you subsequently try to move the cursor or click on something.

Another common cause:  using large ranges with COUNTIF (and presumably SUMIF), VLOOKUP, MATCH etc.  Even though these are not volatile functions, I have found that Excel recalculates them even when I would not expect it.

Finally (but not exhaustively), another common cause:  poorly-designed event macros like Workbook_SheetChange.

If none of these pan out, you might consider uploading your Excel file to a file-sharing web site and post the URL in a response here.  The following list of free file-sharing web sites have been suggested by others.  I like box.net.  However, because you use XL2007 and I have XL2003, I probably cannot help you.

MediaFire: http://www.mediafire.com

john mcghie say:
Guys:

A "crash on paste" is generally one of two causes: a) Document corruption, b) You pasted something where it doesn't belong, or c) Both :-)

For this particular crash, the most common causes are corrupt bullets and numbering, and corrupt tables/graphics/equations.

Bullets and numbering are often corrupted by Track Changes or by AutoFormat (Auto Format While You Type).

Tables are often corrupted by drag-editing within them.  Equations are often corrupted by going to and from Matthtype.  Graphics seem to corrupt all by themselves.

Once a document has corrupted, it can be very difficult to find the corruption, and even more difficult to cure it.

The first thing I recommend is to save the file in the "other" format.  If you are in .doc, save in .docx, if you are in .docx, save in .doc.  This forces Word to re-encode the entire document,a nd in doing so, it will often either fix the corrupt code, or discard it.  Of course, saving back to the obsolete .doc format will remove some things (generally, graphics) that cannot be stored in .doc format.  If that happens and you lose something valuable, re-save as a new file in .docx, then copy the missing item back in (and hope that THAT was not the corruption!).

Another thing that will work (although it's brutal...) is to save the document as a "web page".  This re-expresses the document in a much simpler language, leaving out a lot of the complex things that can get corrupted.  When you save, use File>Save As and ensure you check the "Save entire file" checkbox.  The other checkbox cuts it back even harder, and you lose much more formatting.

If all else fails, use File>Save As and specify "Plain Text".  Obviously, you will lose all the formatting and graphics.  But for short documents (under 200 pages) it's usually a lot quicker to re-apply the styles than it is to try to find the corruption.

Hope this helps
