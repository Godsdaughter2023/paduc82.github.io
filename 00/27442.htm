Solved this issue on my upgraded Windows 10 running Office 365 by making changes to:

* the 'Outgoing Server (SMTP)' value 

   and

* the '. . . type of encrypted connection' value. 

Note the values in your present account and record them. 

If this fix doesn't work for you, restore your original settings before attempting any other fixes.

* Open 'Outlook'

* click 'File'

* click 'Account Settings'

* click 'Account Settings: Add or remove accounts or change existing connection settings'.

* double click on 'Name of user (user email address account - POP3/SMTP)'

* On 'POP and IMAP Account Settings' page click on 'More Settings' (lower right corner of screen)

* click on 'Advanced' tab of 'Internet Email Settings'

* change 'Outgoing Server (SMTP): to '587'

* change '. . . type of encrypted connection' to 'STARTTLS'

* click 'OK'

* click 'Next' on 'POP and IMAP Account Settings' page which will 'Test Account Settings'

* If these changes correct the '0x800CCC1A error' a message should report '. . . all tests were completed successfully, Click Close to continue'.

* click 'Finish' on the 'Change Account' screen

hansv mvp say:

You could use the following code in the report's module. It requires that your report has a report header (and footer); it doesn't matter if it is empty, and it can have 0 height.

 

Private lngSkip As Long
Private lngCounter As Long
 
Private Sub Detail_Format(Cancel As Integer, FormatCount As Integer)
    lngCounter = lngCounter + 1
    Me.Detail.ForceNewPage = (lngSkip > 0 And lngCounter = lngSkip)
End Sub
 
Private Sub Report_Open(Cancel As Integer)
    lngSkip = Val(InputBox("Number of checks to print on the first page (1, 2 or leave blank)"))
    lngSkip = lngSkip Mod 3
End Sub

 
Private Sub ReportHeader_Format(Cancel As Integer, FormatCount As Integer)
    lngCounter = 0
End Sub

ken sheridan say: Presumably you are passing the number to print on the first page to the report as a parameter.  In which case you can add a hidden text box to the detail section, txtCounter say, with a ControlSource property of =1 and a RunningSum property of 'over all'.  Also add a page break control to the very bottom of the detail section.  If we assume the parameter is Forms!MyForm!txtNumber then in the detail section's Format event procedure activate/deactivate the page break control with:

Me.MyPageBreakControl.Visible =  (txtCounter = Forms!MyForm!txtNumber)
