djsteele, mvp say:
You're absolutely correct that field size isn't an option in Memo fields, because the size of a Memo field is so large...
 
To get the size of a Text field, you can use
 
CurrentDb.TableDefs("NameOfTable").Fields("NameOfField").Size
 
See whether setting the control's ControlTip Text property gives you the effect for which you're looking.

daniel g. grau say:
Word will ask you to determine the type of the file in the following cases:
  • If it's in RTF, HTML or text-only format (this is in spite of the file's extension) and you've enabled Confirm conversions… [Some file-format converters and file-creators (such as computer-assisted translation software) usually create an RTF file but name it with a .doc extension, in which case you will be asked to confirm anyway.]
  • If it cannot determine what the contents of the file are. Generally this happens with Unicode files.
 If the file is a genuine .doc or .docx (i.e., not and RTF masking as such), Word will never ask you to confirm.

Then there are file types and file creators (two four-letter codes that define the file's icon). Sometimes the Finder performs a wrong association and the file acquires the wrong icon, independently of its OS version.

It would be nice if you could post a sample file in a publicly-accessible site.
