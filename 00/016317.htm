    mike h.. say:

    Hi,

    This isn't clear but I assume you want to do this for a particular name. Try this user defined function.

     

    ALT+F11 to open VB editor, right click 'ThisWorkbook' and insert module and paste the code below in. Close VB editor.

     

    Back on the worksheet call with

     

    =count10(A1:I100,B1)

     

    Where A1:I100 is the data range and B1 is a cell containing the name to parse.

     

    Function count10(SrcRange As Range, NameRange As Range) As Long
    Dim c As Range
    For Each c In SrcRange
        If c.Value = NameRange And c.Offset(, 1) = 10 Then
            count10 = count10 + 1
        End If
    Next c
    End Function

    ken sheridan say: The concept of 'last record' is meaningless in the context of a relational database table.  Tables are sets, and by definition a set has no intrinsic order.  You can have the last record in a particular sort order, e.g. with the latest date in a column of date/time data type.  If you mean the last row entered, then you should date/time stamp each row when it is inserted.

    When posting a question like this it always helps if you describe the actual entity types and attributes involved.  This avoids our wasting time by chasing red herrings.  At the risk of doing so, let's assume as an example, however, that you have a report based on a Customers table and within it a subreport based on an Orders table and you wish to return the latest order per customer by OrderDate.  The subreport would be based on a query like this:

    SELECT *
    FROM Orders AS O1
    WHERE OrderDate =
        (SELECT MAX(OrderDate)
          FROM Orders AS O2
          WHERE O2.CustomerID = O1.CustomerID);

    kody_devl say:

    I have found a workaround. 

    It isn't pretty but it is working.

    I found that, on the protected directories, I could not, using explorer, copy and paste these files to my C:\Temp\* drive.

    I also found that if I did a "FileCopy source, destination", my rountine is thrown into error trapping when the directory is protected. 

    Using error trapping to identify protect files, I then mark them for exclusion from my process (from my recordset) and don't open them, because they are protected.

    If I CAN filecopy them to C:\temp\ without error, I can also open them!

    It works
