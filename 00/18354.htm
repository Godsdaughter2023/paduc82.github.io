anna ma msft say:

Hi RockyMondal,

 

Thanks for the information you provided.

 

By checking, we found the emails forwarded from domain 2 to domain 1 both have SCL score of 9, and failed DMARC authentication hens were marked as spam by recipient.

 

The reason that failed DMARC authentication is the ‘From’ address and ‘Mail From’ address didn’t match. You may use Message header analyzer tools to check it->in Authentication-Results entry it shows: dmarc=fail action=oreject header.from=<original_sender_domain>;compauth=fail reason=000.

 

For more about DMARC, here’s for your reference: https://docs.microsoft.com/en-us/office365/securitycompliance/use-dmarc-to-validate-email#inbounddmarcfail

 

According to the two message headers, both ‘From’ addresses are the original sender addresses (who wrote these emails and sent them to domain 2). However, when emails get forwarded, the ‘Mail From’ address changed from the original ‘Mail From’ address (original sender) to domain 2 address.

 

Moreover, after checking these domains, only domain 1 set up DMARC record. Generally, if the DMARC policy of the sending server is p=oreject, EOP marks the message as spam with an action of quarantine.  In your situation, as only recipient domain set up DMARC, it should not trigger DMARC checking. I‘d like to confirm whether this user has set up DMARC for his domain 2?

 

I suggest that you may let this user set up Inbox rule to forward emails to check if it works. You may try to set up a transport rule to change the SCL score for this user incoming forwarded emails: Use mail flow rules to set the spam confidence level (SCL) in messages.

 

For other details, you can check my private messages.

 

Thanks,

Anna

anna ma msft say:

Hi RockyMondal,

 

Yes, it has difference, let me take an example:

 

#1. Use auto-forwarding settings: user1@domain1.com send emails to user2@domain2.com. Auto-forward the email to user3@domain3.com. In message headers, the SMTP From address (user2@domain2.com) is not the same as From address (user1@domain1.com).

 

#2. Use Inbox rule to forward email: user1@domain1.com send emails to user2@domain2.com. Auto-forward the email to user3@domain3.com. In message headers, the SMTP From address (user2@domain2.com) is the same as From address (user2@domain2.com).

 

Please be kindly noted that if this user use Inbox rule, he are not able to directly reply to the original sender. By Inbox rule, forwarded messages appear as messages that are forwarded by you. When you reply, the reply messages are send to the recipient who forward this email, instead of sending to the original sender.

Hi Matt,

Since you already insert the PDF in OneDrive, if you insert the PDF as printout in OneNote 2016, the large margins in the top and bottom is created by printer you use to print the PDFs to OneNote 2016. Because the paper size is certain(set by printer), if the page and the PDF doesn’t match, it can only fit one direction of your PDF. In this case, it fit the width only. 
One way to this is change the size of paper. Use a similar paper size with the original PDF.
Open PDF > Print > Select Printer: Send to OneNote 2016 > More Settings(this depends on the PDF reader you are using) > Use a more proper Paper size.
The same printer can be found in Control Panel: Control Panel \Hardware and Sound\Devices and Printers > right click Sent to OneNote 2016 > Printing Preference, you can set Portrait or Landscape and Paper Size in Advanced.
