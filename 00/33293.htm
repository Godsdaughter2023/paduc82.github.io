The Windows 10 upgrade appears to hang - Microsoft Community

a. user say:

The Windows 10 upgrade appears to hang

What should I do if my Windows 10 upgrade stops making progress at a specific percentage, either at a black screen with a blue circle or in Windows Update, or at the message, “It's taking a bit longer than usual to set up your device, but it should be ready soon”?

a. user say:

Some PCs may need a long time to download Windows 10, depending on the quality of the internet connection, the amount of data to move and the computer’s speed.  In some cases, it may take as long as 24 hours, so please be patient.  If your hard drive is still flashing, there’s a good chance it is still making progress.

 

There are a few points in particular where the progress bar may slow down significantly enough that you may believe that progress has stopped:

  • At the black screen with a blue circle at the 30-39% range, while Windows is downloading all dynamic updates for Windows 10

  • Again at 96%, when Windows is busy backing up your data

  • And at the message “It's taking a bit longer than usual to set up your device, but it should be ready soon”

If the upgrade seems sluggish, try disconnecting any USB devices and extended display cables that may be connected.  This has been seen to speed up the download process significantly.

 

Please contact support or start a new thread if it’s taking longer than 24 hours on the same screen or percentage.  When you do, make note of the CPU’s make and model, the country you are located in, what screen you are stuck on and the percentage shown.  The agent may ask if they can collect the setup logs to investigate the issue.

davidofma say:

Okay, I figured it out. It is indeed a corruption of the built-in Command Prompt menu (which I didn't know existed until I started poking around this problem).  The problem and fix are here:

The issue is a key called "cmd" in HKCR\Directory\shell. Deleting the "cmd" key under "shell" and then adding the following code appears to have fixed the problem.

[HKEY_CLASSES_ROOT\Directory\shell]
@="none"

[HKEY_CLASSES_ROOT\Directory\shell\Command Prompt]
"Extended"=""
"NoWorkingDirectory"=""

[HKEY_CLASSES_ROOT\Directory\shell\Command Prompt\command]
@="@shell32.dll,-8506"
"Extended"=""
"NoWorkingDirectory"=""

[HKEY_CLASSES_ROOT\Directory\shell\Command Prompt\command]
@="cmd.exe /s /k pushd \"%V\""

You wife's computer in this case is the Remote Desktop [RDC] client machine. Any version of Windows 7 can act as a RDC client and connect to a RDC host. In your case your wife's PC can not act as a RDC host because she appears to be running Windows 7 Home or Starter (you don't say what) so the Remote Desktop configuration will not be available on her machine in the Allow remote access window. That is normal in your wife's machine case.

So in her case the general procedure is...

1. Connect to the work VPN server and log on.

http://windows.microsoft.com/en-US/windows7/Set-up-a-remote-connection-to-your-workplace-using-VPN

2. Connect to the work RDC host, ie. your wife's office/work desktop, and log on.

http://windows.microsoft.com/en-US/windows7/Connect-to-another-computer-using-Remote-Desktop-Connection

Now be aware that step 1 depends on what type of VPN the work network administrators have setup and use, ie. a Windows PPTP VPN server or a Cisco VPN server or whatever, and what VPN client she uses on her Windows 7 machine. I would work with her office network administrators on that issue.

http://windows.microsoft.com/en-US/windows7/Why-can-t-I-connect-to-a-network

http://windows.microsoft.com/en-US/windows7/Why-am-I-having-problems-with-my-VPN-connection

Once the VPN connection is made she should be able to use the private LAN address of her work PC to connect to it through the VPN tunnel. Get the IP from the work network administrators.

This illustrates what you want to do. This example is using Secure Shell [SSH] but in general this seems to be what your trying to do, ie. tunnel RDC through a VPN tunnel.

http://theillustratednetwork.mvps.org/Ssh/SSH-RDP.html

Note that Windows 7 Home Premium and Starter may not be able to join her work network if its a domain type environment. She may need to upgrade to Windows 7 Ultimate or Pro in order to fully utilize all network features in a domain environment. I would check with the work network administrators about whether or not she is a domain environment. If she is connecting to a Terminal Server (check with the work network administrators about this) then I also suggest posting or lurking on the Remote Desktop Services forum for help.

http://social.technet.microsoft.com/Forums/en/winserverTS/threads

If her work environment is a simple workgroup or she is NOT using a VPN then she should be able to connect directly to her work RDC host machine. This presumes that TCP Port 3389 is open/forwarded through any firewall/router her work PC is behind. Call using the public IP of her work computer or of the firewall/router the RDC host is behind. If the work network has a fully qualified domain name setup with their ISP use that.

All of this also presumes her RDC host PC has Remote Desktop enabled on it. What OS is that machine running? Keep in mind only Windows 7 Ultimate/Pro, Vista Ultimate/Business or XP Pro/MCE can act as standalone RDC host machines.
