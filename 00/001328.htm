john p spencer say:

Yes it is possible and can work well.

Your criteria would be something like (assuming you were looking for records where one or more fields contain the text you are looking for.

WHERE Field1 Like "*" &  [Find What?] "*" OR  Field2Like "*" &  [Find What?] "*" OR  Field3 Like "*" &  [Find What?] "*" OR  Field4 Like "*" &  [Find What?] "*"

In query design view, you would have to enter

     Like "*" &  [Find What?] "*"

under each field you wanted to search and each entry would have to be on a different criteria line.

An alternative would be to concatenate all the fields together in a calculated column and then search against that calculated string.

WHERE Field1 & " " Field2 & " " & Field3 & " " & Field4 LIke [Find What?]

You could add a control to your form and allow the users to enter there search string in it.

Your query would then use criteria like

Field: Field1

Criteria: LIKE "*" & Forms![NameOfTheForm]![NameOfTheControlWIthSearchString} & "*"

john - project say:

pom15595,

Perhaps it is I who didn't quite understand. Using your method of manually inputting a start date to delay a task, you would need to set a baseline before the edit in order for the variance fields to show that delay. However, if you use the approach I suggested (i.e. add a lag to Predecessor field  of tasks you want to delay), then the lag number itself shows the delay. If you want that lag value shown in it's own field the following macro will write the delay value (assuming it is in days) into the Text1 field for each task that has a delay

:

Sub TaskDelays()
Dim t As Task
For Each t In ActiveProject.Tasks
    If Not t Is Nothing Then
        If t.Summary <> True Then
            If InStr(1, t.Predecessors, "+") > 0 Then
                s1 = InStr(1, t.Predecessors, "+") + 1
                s2 = InStr(1, t.Predecessors, "d")
                t.Text1 = Mid(t.Predecessors, s1, s2 - s1) & " days"
            End If
        End If
    End If
Next t
                
End Sub:

Whether you set the baseline before or after schedule tweaks depends on what you consider the baseline. If the plan, prior to forced delays, is the original plan, then you should set the baseline as soon as the original plan is in place. But it the original plan isn't considered in place until after the delays are inserted, then you would set the baseline after inserting the delays.

macropod say:

The following macro generates a list of all words used in the active document, and outputs them alphabetically sorted, with page #s of occurrence in a two-column table at the end of that document, starting on a new page. Line #s are not included (including them would require a major re-write of the code). It only lists words in the MainTextStory, not headers/footers/footnotes/endnotes etc. The macro also has a pre-populated exclusion list, so that various words and phrases can be excluded. The exclusions list (defined by the words & phrases in the StrExcl string variable) lists the words and phrases to be omitted from the concordance. Any phrases should be listed in the exclusions list before any of the single-word exclusions (so that conflicts don’t occur).

Sub ConcordanceBuilder()
Application.ScreenUpdating = False
Dim StrIn As String, StrOut As String, StrTmp As String, StrExcl As String
Dim i As Long, j As Long, k As Long, l As Long, Rng As Range
'Define the exlusions list
StrExcl = "a,am,an,and,are,as,at,b,be,but,by,c,can,cm,d,did," & _
          "do,does,e,eg,en,eq,etc,f,for,g,get,go,got,h,has,have," & _
          "he,her,him,how,i,ie,if,in,into,is,it,its,j,k,l,m,me," & _
          "mi,mm,my,n,na,nb,no,not,o,of,off,ok,on,one,or,our,out," & _
          "p,q,r,re,s,she,so,t,the,their,them,they,this,t,to,u,v," & _
          "via,vs,w,was,we,were,who,will,with,would,x,y,yd,you,your,z"
With ActiveDocument
  'Get the document's text
  StrIn = .Content.Text
  'Strip out unwanted characters. Amongst others, hyphens and formatted single quotes are retained at this stage
  For i = 1 To 255
    Select Case i
      Case 1 To 35, 37 to 38, 40 To 43, 45, 47, 58 To 64, 91 To 96, 123 To 127, 129 To 144, 147 To 149, 152 To 162, 164, 166 To 171, 174 To 191, 247
      StrIn = Replace(StrIn, Chr(i), " ")
    End Select
  Next
  'Delete any periods or commas at the end of a word. Formatted numbers are thus retained.
  StrIn = Replace(Replace(Replace(Replace(StrIn, Chr(44) & Chr(32), " "), Chr(44) & vbCr, " "), Chr(46) & Chr(32), " "), Chr(46) & vbCr, " ")
  'Convert smart single quotes to plain single quotes & delete any at the start/end of a word
  StrIn = Replace(Replace(Replace(Replace(StrIn, Chr(145), "'"), Chr(146), "'"), "' ", " "), " '", " ")
  'Convert to lowercase
  StrIn = " " & LCase(Trim(StrIn)) & " "
  'Process the exclusions list
  For i = 0 To UBound(Split(StrExcl, ","))
    While InStr(StrIn, " " & Split(StrExcl, ",")(i) & " ") > 0
      StrIn = Replace(StrIn, " " & Split(StrExcl, ",")(i) & " ", " ")
    Wend
  Next
  'Clean up any duplicate spaces
  While InStr(StrIn, "  ") > 0
    StrIn = Replace(StrIn, "  ", " ")
  Wend
  StrIn = " " & Trim(StrIn) & " "
  j = UBound(Split(StrIn, " "))
  l = j
  For i = 1 To j
    'Find how many occurences of each word there are in the document
    StrTmp = Split(StrIn, " ")(1)
    While InStr(StrIn, " " & StrTmp & " ") > 0
      StrIn = Replace(StrIn, " " & StrTmp & " ", " ")
    Wend
    'Calculate the number of words replaced
    k = l - UBound(Split(StrIn, " "))
    'Update the output string
    StrOut = StrOut & StrTmp & vbTab & k & vbCr
    l = UBound(Split(StrIn, " "))
    If l = 1 Then Exit For
    DoEvents
  Next
  StrIn = StrOut
  StrOut = ""
  For i = 0 To UBound(Split(StrIn, vbCr)) - 1
    StrTmp = ""
    With .Range
      With .Find
        .ClearFormatting
        .Text = Split(Split(StrIn, vbCr)(i), vbTab)(0)
        .Replacement.Text = ""
        .Forward = True
        .Wrap = wdFindStop
        .Format = False
        .MatchCase = False
        .MatchWholeWord = True
        .MatchWildcards = False
        .MatchSoundsLike = False
        .MatchAllWordForms = False
        .Execute
      End With
      Do While .Find.Found
        StrTmp = StrTmp & " " & .Information(wdActiveEndPageNumber)
        .Collapse (wdCollapseEnd)
        .Find.Execute
      Loop
    End With
    StrTmp = Replace(Trim(StrTmp), " ", ",")
    StrOut = StrOut & Split(StrIn, vbCr)(i) & vbTab & StrTmp & vbCr
  Next
  'Create the concordance table on a new last page
  Set Rng = .Range.Characters.Last
  With Rng
    .InsertAfter vbCr & Chr(12) & StrOut
    .Start = .Start + 2
    .ConvertToTable Separator:=vbTab, Numcolumns:=3
    .Tables(1).Sort Excludeheader:=False, FieldNumber:=1, _
      SortFieldType:=wdSortFieldAlphanumeric, _
      SortOrder:=wdSortOrderAscending, CaseSensitive:=False
  End With
End With
Application.ScreenUpdating = True
End Sub

The above code strips out trailing apostrophes, with the result that some possessive word forms may look a bit odd.

For installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm
