deanh say:

This is by design, that when you are in the document layer the Header and Footer are greyed out, the same if you are in the Header/Footer layer, the document layer is greyed out.

This cannot be changed.

When printed or PDF'd, this effect does not happen.

If you want the contents of your Header and Footer to remain un-greyed, then you will have to remove them from the H/F layer and contruct them in the document layer. This is easy but can cause you problems espeically if this document is to be distributed.

How is the final documents usually sent out? Are they printed/pdf'd, or do you send them as Word documents?

You will have to consider the pros/cons of leaving the content in the relatively safety of the H/F layer or bring the content into the document layer.

All the best

bernard liengme say:

You should try this first with a copy of your file -- just in case!!

The code below automatically fires when there is a change in any cell in column K

If that change is that the work Sold has been entered, then the Cut and Paste happen

Note this will work with sold, Sold, SOLD etc.

Copy the code below, right click the tab of the Sheet with the data, select View Code and paste the subroutine into the resulting window. Now test it.

Come back and explain about Lost - does the record go to Sheet2 or to another sheet? If it goes to the same sheet then change

If UCase(Target.Value) = "SOLD" Then

to

If UCase(Target.Value) = "SOLD"  Or If UCase(Target.Value) = "LOST" Then

best wishes

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim KeyCells As Range
    Set KeyCells = Range("K:K")
    On Error GoTo fred
    If Not Application.Intersect(KeyCells, Range(Target.Address)) _
           Is Nothing Then
        If UCase(Target.Value) = "SOLD" Then
          n = Target.Row
          Range("a" & n & ":P" & n).Copy Worksheets("Sheet2").Cells(Rows.Count, 1).End(xlUp).Offset(1)
          Cells(n, 1).EntireRow.Delete
        End If
    End If
fred:
End Sub
