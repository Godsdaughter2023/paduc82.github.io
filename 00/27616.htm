site collection logo not displaying - Microsoft Community

jimmy barber say:

site collection logo not displaying

Hi, 

I am currently creating a site collection that will be used to share content with external users, currently, when I share a folder the external end user is not seeing the site logo when viewing the folder. Are there permissions that need to be set for the site logo?

james yan msft say:

Hi Jimmy,

The site logo file is stored in the SiteAssets library if we upload the picture "FROM COMPUTER". If the external user has no permission to the SiteAssets library, it is expected that he won't see the site logo. In other words, if we want the external user to be able to see the site logo, we have to share the SiteAssets library with him (view only permission is enough).

Go to the site home page > Settings > "Site contents", you will find the "Site Assets" library.

Of course we can also choose site logo picture from any other SharePoint location in our tenant (FROM SHAREPOINT option in the above picture), we should also share that location with the external user and choose at least view only permission.

Regards,
James

jack cook - aka help_line say:

If I were going to do that I would follow the following process:

  1. On the MyPhone website, I would remove the connected phone.  All that does is to stop the current relationship.  All of your data is STILL on the MyPhone website and is safe.
  2. On your device DELETE all of the contacts.
  3. Now open the MyPhone application on your device and select the items you want to sync.  Clearly, Contacts will be one of the items.
  4. When you sign in with your Windows Live ID and sync using the MyPhone application, the information stored in your My Phone account will be downloaded to your phone and all of the Contacts you had stored there will now be on your device as well.
