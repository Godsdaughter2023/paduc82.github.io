When I run a vlookup on my key value and search right, the value I am trying to return is in a merged cell from columns AF:AK; The vlookup fails if I try to use the AK column, but AF works, is there a workaround or solution that doesn't require unmerging the cells? TIA for any suggestions.

jeeped say:
You cannot reference the cell using an AK column reference as any merged range of cells is referenced by the top-left cell address. However, you can often manipulate the cell address with OFFSET() and some math based upon the column but this is a bandaid or kludge fix. IMHO, merged cells are primarily for decorative purpose and do not really belong in a data matrix. You would be better off just un-merging the cells.

phillip m jones say:


The are two type of backup or recovery option in Word the is applies whether 2016, 2011, 2008. 

  1. Auto Recover
    Auto-recover works provided you have been periodically saving your work.
    And the time interval should not be set lower 5 or 6 minutes. Auto Recover takes a snapshot of what you typed in that time interval.
    If you crash it saves this file (when set to the correct location) And if you open just after the crash I will ask whether you wish to recover your work choose yes and it will insert the snapshotted information to the end of the document. However, ever thing has to be just right Moon in Phase, Fingers crossed just right. The snapshots save in correct location, and your document did not corrupt in the crash.
  2. The second way is to create backup files.
    You set this option in Word → Preferences  → Save.
    Place checkmark next to Create Backup files
  3. Once set, you must everytime you pause to think, rub your eyer from eye strain or whatever You hit ⌘ - S (Save)  This creates serialized version your file.
  4. If you need to retrieve a previous version you can.

Unfortunately, neither scenario applies to your situation, because, did not take the time to save the file while working. Your file was totally in memory. Once you quit Word, the memory is purged.

The there one other way to recover a file but also depends on saving a copy.

If you have Mac "Time Machine" turned on, and a backup drive attached, it will save any changes made on the computer periodically about every 1 to 2-hour intervals. The backup, too, is serialized by date and time. 

So I hope I haven't wasted your time explaining all this explanation. For any file recovery, depends on you at least saving the file at least once while working. I know it a hard habit to get into, I am even guilty even knowing what I do, off not saving until I finish. However, I don't do long or relevant documents anymore and usually get away with it most times. However, I do have "create backup files" turned on, just in case
