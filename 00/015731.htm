john mcghie say:
Hi Gary:

The problem is that "something" is unzipping the file.

Internally, a Word .docx is a zip file, but it must never be unzipped, or Word can't read it.

If you save the file to your hard drive, then open Word and use File>Open to open the document, chances are it will open fine.  But if you double-click it out of your browser, chances are the browser will look, see the .zip encoding, and "helpfully" unzip it.  After that, it's cactus: Word can't touch it (and neither can any other program).

Of course, it doesn't help that the PC is probably adding the obsolete File Type and Creator Code attributes to the file: these should not be used these days, but if they are, they can cause problems.

It helps greatly in these cases to use a proper email client, rather than WebMail.  The problem with "Free" webmail is that you get what you pay for...  Use an email client such as Outlook or Apple Mail and you will have far less bother.

As to where the damage is occurring: very hard to say: It can be at the sending end, in the middle, or at the receiving end.  If all else fails, Zip the file before sending.  That way, whatever is doing this can unzip it if it wants to, without doing the content any damage.

Hope this helps

smylexbyjoker say:

I experienced this problem and had to go through so many methods of troubleshooting to get the 2010 key to finally redeem. I'm not sure what the exact issue is but this procedure worked for me.

 

Here's my scenario:

 

I have 45 Dell OptiPlex 7010 desktop PCs with Windows 8, preparing them for deployment at my business. Each computer had a Microsoft Office 2010 license. To begin, each PC went through the following procedure:

1. Power on, name PC, setup local administrator account, disable default power save settings, initiate remote desktop connection, activate network sharing, and run windows update. (steps taken to allow for remote administration for convenience)

2. Restart to complete updates, login, update Adobe, install second round of windows updates, connect to domain with appropriate profile, restart and login.

 

Finally I ran the Microsoft Office 2010 installation and entered the product key supplied with the PC.

 

Office 2010 goes through the installation and may require you to restart to complete the process, however, if it doesn't, do so anyway.

 

 

NOTE:

It is necessary for you to open Microsoft Office 2010 post-installation.

 

You must wait 24 hours before attempting redemption.

 

 

 

POSSIBLE FIXES:

 

1. After completing this installation process, run windows update again to apply the updates applicable to the Office packages. (be sure updates associated with other Microsoft products is selected). Restart PC.

 

2. OPEN MICROSOFT OFFICE and select Recommended Settings.

 

3. Wait the necessary 24 hours for activation.

 

4. Go to http://office.microsoft.com/en-us/offer/

 

5. ENABLE COMPATIBILITY VIEW BY CLICKING THE ICON WHICH LOOKS LIKE A TORN SHEET OF PAPER IN THE INTERNET EXPLORER ADDRESS BAR.

 

5. Here, things get tricky. Unfortunately the error code does not explicitly state what the problem is so you'll have to cover all bases.

 

MY PROCEDURE WHICH WORKED:

 

- I tested different ways of filling out this form to see what possible issues could be. One major issue is that you are not entering your product key which CAPITAL LETTERS.

So, before beginning, hit CAPS LOCK and then select your country.

Enter your product key.

Accept the Terms.

Enter the verification code EXACTLY meaning that CAPS LOCK SHOULD STILL BE ENABLED.

 


 

I thought that the issue was associated with the number of licenses allowed for me to redeem since I read that only 25 can be redeemed per organization though my sales representative advised me otherwise.

This debunks that theory.

 

I'd like to note that only after the UPDATES, COMPATIBILITY VIEW, AND UTILIZATION OF CAPS LOCK did the error code stop popping up.

 

 

 

I really hope this helps. I also hope that someone gets furious enough to contact Microsoft about implementing definitive error codes because this error code is absolutely useless - as is Technical Support.
