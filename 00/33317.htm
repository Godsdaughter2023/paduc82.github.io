First, there a several different compressed hal.dlls on the XP CD and the XP installation picks the right one to match your hardware and motherboard at installation time.  Take a look at all the compressed HALXXXX.DL_ files on your installation CD (7?).  One of them will be picked to be your hal.dll at XP installation time (they are not interchangeable), but which one is the right one?  You better know how to figure it out if you think yours needs replacing.

a. user say:

Well if you are running the generic hal.dll right now, chances are good that XP has the wrong impression of your hardware.

When looking around the Internet for other posts with similar problems, try to find those that have happy endings and do not use words like try, might, maybe, could be etc.

The ntbtlog.txt will show show lots of "Did not load driver..." messages because that is the default logging configuration.  It certainly does not mean that there is a problem and things are failing to load.   It does not mean driver files are missing at all.  It means XP looked at a list of potential drivers to load and decided that based on your configuration the driver was not necessary and therefore did not load it and logged it.  It does not necessarily mean the file is missing. You could have Services disabled or turned off, features not installed, disabled...  You certainly don't want XP to load up every single driver on your system - you only want it to load up what it needs to load.

Just because you see a "Did not load driver" message and if you actually look for the file and don't find it, that doesn't mean the file was ever there before or ever even needed.  If you don't want to see those kinds of informational (not error) messages, just tell XP not to display them.

What do you mean "driver files the registry is demanding"?  What driver files is the registry demanding, how do you determine it is demanding them - hopefully not from the messages in the ntbtlog.txt file - that is no indicator.  If you are running the generic hal.dll file now, XP is thinking you have some other kind of hardware than what you really have and now XP is all mixed up (unless you got lucky and chose the right hal.dll), but XP is trying to do the best it can with the information you gave it.

You need to understand what the messages in the boot log mean and then determine if they are part of the problem or not.  A "Did not load driver" message does not mean there is a problem.  Now with your generic hal.dll file, XP might think it needs to load a lot less stuff than is required for your system to run properly.  There is a difference between an informational message and an error message.
