The location of <scanPST.exe> depends on several factors

- OS bit version in use

- Office version in use

- bit level of Office version in use

- type of install in the case of '2013 (C2R vs .msi)


On a Win8 x64 system with Off'2010 (x86) and Off'2013 (x86-C2R installed) - i.e. two versions of Outlook being active - the location for ScanPST is as follows


For O'2013


"Windows Drive:\Program Files\Microsoft Office 15\root\Office15\"


For O'2010 on the same system


"Windows Drive:\Program Files (x86)\Microsoft Office\Office14\


On a Win8 x64 system with only O'2013 x86 installed via the .msi installer - the location is


"Windows Drive:\Program Files (x86)\Microsoft Office 2013\Office15\"


(note: don't know\recall if there any changes made to the default MS Office folder name during the .msi install process so there may be a slight variation but not something that should prevent anyone from finding the folder)

These Tech Support Scam webpages lock up your browser with a modal alert dialog that’s scripted to reopen every time you try to close it (a dialog loop). This is undoubtedly the most prevalent and persistent scam on the Web – just because the browser-locking here isn’t done with file-based malware, but rather with an exploitation of the application-modal dialog. This means that the onus for solving this problem is on browser developers, and unfortunately their countermeasures have been painfully slow to come along, and also largely ineffective.

In the Edge browser, the issue is complicated by the automatic recovery (AutoRecover) of the Tech Support Scam page after the browser process is terminated in Task Manager; so Microsoft belatedly provided us with a Dialog Loop Protection patch for Edge. Once it’s clear that this issue isn’t really a "malware issue" at all (in the conventional sense of the term), it should also be obvious that the key to success with this is actually just learning how to close the Tech Support Scam pages with Task Manager, or in the case of Edge, with the built-in fix that Microsoft has already provided:

rob koch say:

How do you know you truly have a "Recycler Virus"?

In most cases, people asking this have just become confused about the normal Recycle Bin built into Windows, since there are a lot of fake articles on the Internet that call this folder a virus.

Some people will tell their "friends' they have this virus in order to watch them thrash around trying to delete a folder that Windows is designed to automatically recreate, so that just convinces the foolish person even more that this folder is a virus.

In truth it's a Windows self-protection feature that protects the folder to allow the movement of deleted files into this temporary storage until the Recycle Bin is "Emptied", in other words permanently deleted.

In other cases, the installed antivirus program might detect a virus (e.g. malware), in this folder, since this is where all files go first when they are deleted.  However, the fact that malware is detected in a folder doesn't make the folder itself malware, it's just where the file happens to be located.

So again, how do you actually know you have this "virus"?

Rob
