As long as it is in the scanning process and not the repairing process, you can stop it without worries. Otherwise you'll need to restore the backup copy that you or the tool created before the repairing process started. This is a bak-file in the same directory as the pst-file that you are scanning. Rename the pst-file to .old and the bak-file to .pst.

14GB is a large pst-file to scan indeed but 13 hours is quite long indeed, even for that size. To improve the scanning process, make sure that you have no other resource intersive application running in the background and placing your pst-file on a faster hard disk can help as well (if multiple are available in your computer). Note however that not all phases of the scanning process take just as long.

If you can still access (some of your folders in) your pst-file, then I'd recommend to create a new pst-file in Outlook and move the accessible content to that new pst-file. Then either recover the rest of the data from a previous backup or run scanpst.exe again. Hi Ed Hansberry,

 

If you want to install 64-bit Office applications, you can use Office Deployment Tool.

 

According to this article Overview of the Office 2016 Deployment Tool, you can modify the configuration file, for example, you want to install 64-bit Office applications, you can use

 

<Configuration>

    <Add OfficeClientEdition="64" Channel="Monthly">

            <Language ID="en-us" />

        </Product>

    </Add>

You might receive this error message when you're trying to save an Excel workbook if one or more of the following conditions are true:

  • You save an Excel workbook to a network drive on which you have restricted permissions.
  • You save an Excel workbook to a location that does not have sufficient drive space.
  • The connection to the Excel workbook is lost.
  • There is a conflict with an antivirus software program.
  • You save an Excel workbook that is shared.
