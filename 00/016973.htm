    Excel forecasting question - Microsoft Community

    branndonjackson say:

    Excel forecasting question

    Im using the forecast formula in excel and would like to know the actual equation excel is using to forecast the data. I know if you go to the graph and click add trend line or equation it shows you, but I'm looking for the default equation. Does the system automatically forecast a linear regression?

    Moreover, I would like to forecast using a polynomial quadratic equation. Is there a formula not a graph, but formula I can put in to forecast in these terms. I understand that you can change your graph in excel to make the trend lines accommodate the data points. I'm looking for a direct forecast in a poly initially. like ="polyforecast" or something like that... Thank you.

    joeu2004 say:

    BranndonJackson wrote:

    Im using the forecast formula in excel and would like to know the actual equation excel is using to forecast the data.

    Google "excel forecast" without quotes and see the support.office.com help page.

    BranndonJackson wrote:

    Does the system automatically forecast a linear regression?

    Simple linear regression.  Yes.

    BranndonJackson wrote:

    Moreover, I would like to forecast using a polynomial quadratic equation. Is there a formula not a graph, but formula I can put in to forecast in these terms.

    Yes.  One way:  select 5 horizontal cells (e.g. A1:E1) and array-enter the following formula (press

    =LINEST(ydata, xdata^{1,2,3,4})

    Then ypred (aka yhat) can be calculated by the following normal formula (press Enter as usual):

    =SERIESSUM(xcell, 4, -1, $A$1:$E$1)

    or

    =SERIESSUM(xcell, 4, -1, $A$1:$D$1) + $E$1

    Use the latter if xcell might be zero.

    If you would like to see R^2 as well, select a 3x5 range (e.g. A1:E3) and array-enter the following:

    =LINEST(ydata, xdata^{1,2,3,4}, TRUE, TRUE)

    E3 is R^2.

    Caveat:  R^2 might be incorrect if you force a zero-intercept by setting the third parameter ("const") to FALSE.  It's a known defect.  I have a workaround, if you need it.
