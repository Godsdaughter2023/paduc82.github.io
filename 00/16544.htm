vijay a. verma say:

Following approaches can be used -

1. SUMIFS - Mike H. has already responded on this. SUMIF is used for only one condition and for more than one condition, SUMIFS is used.

But I will simplify this further by writing exact formula for you. The syntax for SUMIFS is -

=SUMIFS(Sum_Range, Criteria_Range_1, Criteria_1, ......Criteria_Range_N, Criteria_N....)

In your case, assuming project is in cell A1, SUMIFS formula will be

=SUMIFS(C2:C6, A2:A6, "abc", B2:B6, "labor")

2. Alternatively, you may use SUMPRODUCT which is an array formula but written without { }. Before Excel 2007 when SUMIF(S) and COUNTIF(S) were not there, it was SUMPRODUCT which ruled the Excel kingdom. In fact, many times when I get doubt whether SUMIF(S), COUNTIF(S) are working correctly or not, I check that through SUMPRODUCT. Creating conditions in SUMPRODUCT is more intuitive.

Using SUMPRODUCT will slow down your Excel if lots of SUMPRODUCT are there. Use SUMIF(S) wherever possible or when lot of conditional sums are required.

=SUMPRODUCT(--(E2:E6="abc")*--(F2:F6="labor")*(G2:G6))

3. Another approach can be a simple SUM formula using Array technique (Enter this formula by Control+Shift+Enter approach. Use it only if you know tricks around Array formula)

=SUM(IF(E2:E6="abc",1,0)*IF(F2:F6="labor",1,0)*G2:G6)

OR

=SUM((E2:E6="abc")*(F2:F6="labor")*G2:G6)

hansv mvp say:

The formula becomes more complicated:

 

=IF(ROW(1:1)<=(COUNTA($C$2:$C$99)*COUNTA($E$2:$E$99)*COUNTA($G$2:$G$99)*COUNTA($I$2:$I$99)),CONCATENATE(OFFSET($C$2,INT((ROW(1:1)-1)/(COUNTA($E$2:$E$99)*COUNTA($G$2:$G$99)*COUNTA($I$2:$I$99))),0),OFFSET($E$2,MOD(INT((ROW(1:1)-1)/(COUNTA($G$2:$G$99)*COUNTA($I$2:$I$99))),COUNTA($E$2:$E$99)),0),OFFSET($G$2,MOD((ROW(1:1)-1)/COUNTA($I$2:$I$99),COUNTA($G$2:$G$99)),0),OFFSET($I$2,MOD((ROW(1:1)-1),COUNTA($I$2:$I$99)),0)),"")

 

It will be easier to read if we define some names.

 

Activate the Formulas tab of the ribbon and click Name Manager.

 

Click New.

Enter the name CountC in the Name box.

Enter the formula =COUNTA($C$2:$C$99) in the Refers to box.

Click OK.

 

Do the same with CountE and =COUNTA($E$2:$E$99)

Also with CountG and =COUNTA($G$2:$G$99)

Finally with CountI and =COUNTA($I$2:$I$99)

 

Close the Name Manager.

 

You can now change the formula to

 

=IF(ROW()-1<=CountC*CountE*CountG*CountI, OFFSET($C$2,(ROW()-2)/(CountE*CountG*CountI),0) & OFFSET($E$2,MOD((ROW()-2)/(CountG*CountI),CountE),0) & OFFSET($G$2,MOD((ROW()-2)/CountI,CountG),0) & OFFSET($I$2,MOD((ROW()-2),CountI),0),"")

 

CountC is the number of entries in column C, CountE is the number of entries in column E etc.

CountC*CountE*CountG*CountI is the total number of combinations. We only want to display a combination if the row number minus 1 (since we start in row 2) is less than or equal to the number of combinations. So the general form of the formula is

 

=IF(ROW()-1<=total number of combinations, combination, empty)

 

Each combination is a concatenation of 4 entries. Each of these entries is calculated as an offset from the cell in row 2 of a column. The first one is

 

OFFSET($C$2,(ROW()-2)/(CountE*CountG*CountI),0)

 

We offset from cell C2; the row offset is the row number minus 2 (to start at 0) divided by CountE*CountG*CountI, so that it changes only every CountE*CountG*CountI rows to the next one. The column offset is 0.

 

The second one is

 

OFFSET($E$2,MOD((ROW()-2)/(CountG*CountI),CountE),0)

 

Here, we divide the row number minus 2 by CountG*CountI, so that it changes once every CountG*CountI rows, but we take the remainder modulo CountE so that the value won't become too high.

 

If you need more columns, the formula becomes even more confusing. It might be better to use a macro to compute the combinations.

Creating 2 different indexes from the same document - Microsoft Community

madonna beyrouthy say:

Creating 2 different indexes from the same document

Hello,

I want to create from a big document 2 different indexes, let's say one for the authors and another for other kind of references.

I need them to be on 2 different indexes at the end of my document. Can I do that?

When I am marking the words for indexing I cannot split them in 2 different lists to get 2 different indexes. is there any way to do that?

thanks and regards,

Madonna

jay freedman say:

Yes, you certainly can do that.

The first part involves making two sets of entry markings (XE fields) in the document body. Unfortunately, there's nothing in the Mark Index Entry dialog to help you do that. What you must do is edit the code of each XE field after it's inserted. After the entry's text (between the closing double-quote and the closing field bracket), insert the switch \f followed by one of two "identifiers". An identifier is a single letter enclosed in quotes; you can choose any two letters except "i" to represent your two entry types.

So let's say you choose "a" as the identifier for the author names and "o" for other references. Then you would edit the two types of XE fields to look like these:

{ XE "Rowling:J.K." \f "a" }

{ XE "witchcraft" \f “o” }

The second part consists of inserting two INDEX fields and giving them the same two identifiers. An INDEX field containing the \f "a" identifier will collect entries only from XE fields that contain \f "a", and an INDEX field that contains 

\f "o" will collect the entries marked with \f "o".

Be careful about adding the identifier to every XE field. When the INDEX fields have identifiers, any XE field that has no identifier will not be included in either index.
