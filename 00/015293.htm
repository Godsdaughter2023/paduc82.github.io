jay freedman say:
One thing you can check is whether a group policy has been set in Windows 7 that prevents saving of recent documents.
 
There are two places you can look at this -- they're equivalent (changing one automatically changes the other), so it depends on what makes you more comfortable.
 
1. In the Start menu > Run box, enter  gpedit.msc  to open the Local Group Policy Editor. On the left side, expand
      User Configuration > Administrative Templates > Start Menu and Taskbar
    On the right, look at the policy named "Do not keep history of recently opened documents".
    If its state is "Enabled", double-click its icon and, in the dialog that opens, click "Not Configured" and then OK.
 
2. Alternatively, run  regedit.exe  and navigate to the key
      HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer
    If the right side contains an entry named NoRecentDocsHistory with a value of 0x00000001 (1), click it to
    select it and press the Delete key. Answer 'Yes' to the prompt.
 
If the group policy is already "Not Configured" or "Disabled", or if the NoRecentDocsHistory entry is absent or has a value of 0x00000000 (0), then the problem is elsewhere.

hansv mvp say:

Here is a macro that you can run from the Macros dialog, or from a command button, for example:

 

Sub SendEmail()
    Dim olApp As Object
    Dim olMsg As Object
    Dim nr As Long
    On Error Resume Next
    Set olApp = CreateObject("Outlook.Application")
    If olApp Is Nothing Then
        MsgBox "Can't start Outlook", vbCritical
        Exit Sub
    End If
    On Error GoTo ErrHandler
    nr = ActiveCell.Row
    Set olMsg = olApp.CreateItem(0) ' 0 = olMailItem
    olMsg.To = Range("K" & nr).Value
    olMsg.Subject = "Company welcome"
    olMsg.Body = "Hello " & Range("C" & nr).Value & "," & vbCrLf & vbCrLf & _
        "Welcome to our company. Attached is your employee file and " & _
        "below is your product activation license key:" & vbCrLf & vbCrLf & _
        "key #: " & Range("G" & nr).Value & vbCrLf & vbCrLf & _
        "We welcome you"
    olMsg.Attachments.Add "C:\Users\" & Environ("username") & _
        "\Desktop\Important Files.zip"
    ' Important: use either Send or Display, not both
    ' To send the message directly:
    'olMsg.Send
    ' To display the message so that you can inspect it, then send it manually:
    olMsg.Display
 

ExitHandler:
    Exit Sub

 

ErrHandler:
    MsgBox Err.Description, vbExclamation
    Resume ExitHandler
End Sub

 

hansv mvp say:

vbCrLf is VBA shorthand for "carriage return/line feed", the computer code for starting a new paragraph. In my sample code, I concatenated with vbCrLf & vbCrLf, i.e. two paragraph breaks, to create an empty space between the text. If you want 2 lines below each other, use a single vbCrLf:

 

    "We appreciate your hard work," & vbCrLf & "Welcome!"

 

or, equivalently

 

    "We appreciate your hard work," & vbCrLf & _

        "Welcome!"

 

As Jean-Yves Tfelt pointed out, the _ is used to break a long statement into multiple lines. The _ is called the line continuation character.

 

See Formatting Code for more info.
