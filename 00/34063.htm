I have Windows 7 Ultimate 64-bit and use the Microsoft Bluetooth Notebook Mouse 5000.  I tried the recommended Service.MSC route to select automatic and then local system and then OK and et cetera.  That did not work.  I thought the mouse was defective but then I tried to pair it with three other portables and it successfully paired with all of them.  Why not on the Windows 7 system?  I  figured out the problem.  With all humility wait a couple of days before doing the following fix to give time for some technical genius to say that it is a good fix or if I just opened my system to a host of other issues.  All I can tell you is the failure to connect the mouse has been frustrating but the fix i suggest worked instantly.

Go to START icon and type in SERVICES.MSC and select SERVICES.  The dialog box that opens has a long list of stuff but look for BLUETOOTH SUPPORT SERVICE and double-click or open it.  Select AUTOMATIC.  Go to the LOG on tab.  Choose LOCAL SYSTEM and it is the road to nowhere.  That HAS been the recommended fix but it never worked for me.  Choose THIS ACCOUNT, instead.  Type in LOCAL SERVICE.  NOW; if you click okay it will say select valid password or such.  NO PROBLEM.   Do this; type in Local Service into the dialog box and then delete the password dots (a 15-dot-string) under password and delete the string under confirm (both password boxes are now empty).  Choose okay.  SUCCESS.  It should then tell you to STOP the service and then to RE-START the service before the change takes effect.  It should now work.

The first thing to grasp I suppose is that there is a difference between LOCAL SYSTEM  and LOCAL SERVICE.  The help configuration dialog box says that the password in the LOCAL SERVICE  choice is a null password - that was my clue.  When I selected LOCAL SERVICE (typed it in the dialog box line) I would always get the password box and that is what caused me to pause.  Since it is a NULL password according to the help menu - then you should be able to delete it?  I did.  It accepted my change and after stopping and re-starting the service the Microsoft Bluetooth Notebook Mouse 5000 (uses no transceiver) worked perfectly and CONNECTED!!!

I compared all of my other systems that would accept pairing by checking SERVICES.MSC and each computer was using LOCAL SERVICE AND NOT LOCAL SYSTEM!

zachd123 say:

In the registry, you can create the REG_DWORD value "SlideshowLayout" under:

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Lock Screen

This  is a DWORD value which is parsed as a series of slides, starting least significant number first.  As such, "123" leads to a "3_layout 2_layout 1_layout repeat" scheduling.

Valid values are 1, 2, 3, 4, 5 and 6 for "a horizontally tuned slide of that many images", and 7, 8, and 9 for a "vertically tuned slide of 1, 2, or 3 images".  And of course simply setting it to "1" does exactly what you think it would - and what it sounds like you're asking for. :)

To remove that override, simply delete that specific registry value.
