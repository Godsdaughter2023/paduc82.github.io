Auto populate field on form

I currently am developing a database to move my company from excel to access.  I have a form that, i would like to auto populate the fields.  Here is the scenario, i have two tables, customer list, and work orders.  On the form work order, i want to be able to type in the job address, in the job address field (located in both the customer list and work orders tables) and automatically have access fill in the following fields, owner first name, owner last name, billing address (located in the customer list table).  i am new to access, so please assume i really no nothing

john w. vinson say:

Welcome to Access!

One basic principle of relational databases like Access is what I call the "Grandmother's Pantry Principle": "A place - one place! - for everything, everything in its place". The address information, owner name, etc. should be stored in the Customer table, and noplace else. A Work Order doesn't have a first name!

The Work Order table should contain only a customerID, linked to the primary key of the customer table. You can use this to look up and display the customer's name and address whenever it's needed.

You may want to look at some of these resources. Crystal's tutorials in particular may help:

andrewsmechanical inc. say:

I appreciate the quick response.  However the way my company operates, we would need the address and owner name, will need to be displayed in multiple tables, the way we keep track of our work orders.

john w. vinson say:

I'm sorry, but I must disagree. Don't confuse data storage with data display - they are separate tasks with separate requirements!

You should NOT be looking directly at tables. That's not how databases work! Tables are intended for data storage, not for data display or interaction. If your table contains a foreign key to the Customer table, you can display the name at any time by linking to that table. If you keep multiple copies of the name and address in different tables, and the company moves to a new address or gets a new contact person, you would then have to track down and edit all the many duplicate fields - a real nightmare!

This relational model has worked well for forty years, and is relied on by thousands of businesses. it's not a whim or pedantry to advise proper design; it WORKS. Please reconsider!

hansv mvp say:

The option is not available in the interface for numeric axes, only for axes whose labels are treated as text.

However, you can set the distance in VBA:

  • Select the chart.
  • Press Alt+F11 to activate the Visual Basic Editor.
  • Press Ctrl+G to activate the immediate window.
  • Type or copy/paste the following line:

    ActiveChart.Axes(xlCategory).TickLabels.Offset = 1000

  • The value is a percentage between 0 and 1000. Default is 100.
  • Use xlValue instead of xlCategory for the Y-axis.
  • With the insertion point anywhere in that line, press Enter.
  • Switch back to Excel to inspect the result.
