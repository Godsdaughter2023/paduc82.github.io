There are others in this forum with much more knowledge than myself, so maybe they will correct me and/or be able to give a better explanation, but this is what I THINK I know.

If you use:

DoCmd.Close

it will close the object that has the focus.  If you use it after openning a second form then that second form will be closed.  To close the first form (in which the DoCmd.Close is used) you need to include the name of the object that is to be closed. 

The acform is used to specify what kind of object is to be closed.  I think the code would still work if it were omitted, but you would need to use the comer after it as you would in other code where an argument is omitted.

norman david jones say:

Hi Cheryl,

Hello, I need to figure out how to insert a photo that will automatically size (or resize) based on a predetermined cell(s) area. For example, I choose an image from a cell phone, and during insertion into my Excel spreadsheet, the image will automatically adjust to fit into 2 columns, each column with a width of 33.11 and 27 rows, each row with a height of 13.2. Try as follows:

  • Alt+F11 to open the VBA editor
  • Alt+IM to insert a new code module
  • In the new module, paste the following code      

'=========>>
Option Explicit

'--------->>
Public Sub ResizePicture()
    Dim oPic As Picture
    Dim Rng As Range
    
    Const NumberOfRows = 27
    Const NumberOfColumns = 2

    If TypeName(Selection) = "Picture" Then
        Set oPic = Selection
    Else
        MsgBox "No picture selected!" _
             & vbNewLine & vbNewLine _
             & "Select photo and retry!"
    End If

    If Not oPic Is Nothing Then
        With oPic
            Set Rng = .TopLeftCell.Resize(NumberOfRows, NumberOfColumns)
            Rng.RowHeight = 13.2
            Rng.ColumnWidth = 33.11
            .ShapeRange.LockAspectRatio = msoFalse
            .Top = Rng.Top
            .Left = Rng.Left
            .Height = Rng.Height
            .Width = Rng.Width
        End With
    End If
End Sub
'<<=========

  • Alt+Q to close the VBA editor and return to Excel
  • Save the file as a macro-enabled workbook with an xlsm extension
  • Alt+F8 to open the macro window
  • Select ResizePicture | Run

=IF(X1=#NUM!... - Microsoft Community

lad_310 say:

=IF(X1=#NUM!...

Various functions in Excel can return a "#NUM!" error when a function is erroneous. For example when making an index that goes out of the sheet. I want to clean out such errors by writng a formula such as "=IF(X1=#NUM, "",X1). So if cell X1 contains the #NUM! error, enter nothing (""), otherwise, enter the content of X1. How do I write the "IF #NUM! part"?

Similarly I need to clean up the #N/A error using the an IF statement, when VLOOKUP does not find the value it is looking for.

jeeped say:

... How do I write the "IF #NUM! part"?


You can identify the #NUM! error in X1 with ERROR.TYPE(). The #NUM! error is 6. So if you trap the existence of an error with ISERROR(), then you can identify the #NUM! as 6. Perhaps something like,

=IF(ISERROR(X1),IF(ERROR.TYPE(X1)=6,"","non-#NUM! error"),X1)

... Similarly I need to clean up the #N/A error using the an IF statement, when VLOOKUP does not find the value it is looking for.


You can use IF(ISNA()) or IF(ISERROR()) to catch the #N/A errors. Something like,

=IF(ISNA(VLOOKUP(this, those, that, true)),"returned #N/A",VLOOKUP(this, those, that, true))
... or less specifically,
=IF(ISERROR(VLOOKUP(this, those, that, true)),"returned an error",VLOOKUP(this, those, that, true))

BTW, the #N/A error is identified by ERROR.TYPE() as 7.

Excel 2007/2010 has a new IFERROR() function that greatly reduces the length of IF(ISERROR()) formulas but it is not backwards compatible with Excel 2003.
