basically I just want to use the value on column A to compare to the next cell(s) and find if there is the different.

Tony. 

jllatham say:

See if this works for you.  I've assumed cells A2:F2 are to be tested:

=IF(COUNTIF(A2:F2,A2)=COUNTA(A2:F2),"all equal","not equal")

But if there are empty cells (even just one) in the middle of the range, it does not return "not equal" - so it works as long as there is some kind of entry in all the cells.

Your logic with the one that doesn't work is good, but unfortunately Excel doesn't think like you and I do.  What happens in =(A2=B2=C2,......) is that the result of A2=B2 is either TRUE or FALSE and that is probably not going to be equal to whatever is in C2, thus the failure.  But it was a helluva try!

A couple of other possible ways:

=IF(COUNTIF(A5:F5,A5)=COLUMN(F5),"equal","not equal")

since there are 6 columns in the range A5:F5, we can compare the result of the COUNTIF() to the number of columns that should be involved.  This handles empty cells quite well.

You could even make it more generic to work with a sequence in the middle of a row like this:

=IF(COUNTIF(D5:F5,D5)=(COLUMN(F5)-COLUMN(D5)+1),"equal","not equal")

Hopefully you see the relationship between column references in the left=right side of the test part of the formula.

hansv mvp say:

If you can create an update query that will do what you want, you can run it from VBA:

 

DoCmd.SetWarnings False
DoCmd.OpenQuery "MyUpdateQuery"
DoCmd.SetWarnings False

 

For a more complicated update, you can use DAO:

 

Sub UpdateTable()
    Dim dbs As DAO.Database
    Dim rst As DAO.Recordset
    Set dbs = CurrentDb
    Set rst = dbs.OpenRecordset("MyTable", dbOpenDynaset)
    Do While Not rst.EOF
        rst.Edit
        Select Case rst.Fields("ThisField")
            Case "Value1"
                rst.Fields("ThatField") = 26
            Case "Value2"
                rst.Fields("ThatField") = 37
            ...
        End Select
        rst.Update
        rst.MoveNext
    Loop
    rst.Close
    Set rst = Nothing
    Set dbs = Nothing
End Sub

 

This is just an example, of course.

perform function if reference cell populated - Microsoft Community

dougsaunders say:

perform function if reference cell populated

I have a series of tables in a worksheet where values are added as data is collected with a summary table using values from the various tables above.  Is there a way to have a function or formula return a value only if a reference cell is populated? In other words can I have a cell in the summary table return the results of a formula only if a refernce cell has a value in it?
tom-s say:

DougSaunders,

If the cell is truly not populated i.e. has no content at all, then you can test with ISBLANK function.

However, if the cell appears unpopulated i.e. has an empty string ("") returned to it then a separate test would be needed.

So, to combine both of these, you could test with:

=IF(OR(ISBLANK(A1),A1=""), action if 'blank', action if not 'blank')

 

where action if (not) 'blank' comprises some formula you insert to return to the cell in which the IF function resides.
