john - project say:

Bob,

Just keep in mind that a summary line is NOT a task, rather it is a summary of the performance tasks under it. What is a "performance task"? A task that describes something to be accomplished such that one or more resources are assigned to it (i.e. "perform" work to accomplish it). A summary line does not meet that definition and therefore should not have any resources assigned to it.

My suggestion, if you need to simply identify an "owner" for the summary group, use a text field (e.g. Text1) or the Notes field. However, if that owner also performs effort for the summary group, then create a separate performance task line for that effort just like any other performance task under that summary line.

John
