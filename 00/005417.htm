daniel pineault say:
You'll have to create a bat file, vbscript file to launch your database using the command line using the full path to the appropriate msaccess.exe to use.

daniel pineault say:

Here's a quick example of a vbscript I put together to help you get going in the right direction.

'*******************************************************************************
' Author    : Daniel Pineault, CARDA Consultants Inc.
' Website   : http://www.cardaconsultants.com
' Purpose : Open a database using the specified msaccess.exe 
' Copyright : The following is release as Attribution-ShareAlike 4.0 International
'             (CC BY-SA 4.0) - https://creativecommons.org/licenses/by-sa/4.0/
'
'Revision: 
' 2018-03-07    Initial Release - Forum Help
'*******************************************************************************
 Dim sFrontEnd
 Dim sAccessExe
 Dim sVBSPath
 Dim sDBFullPath
 Dim WshShell
 
 
'User defined parameter
'*******************************************************************************
 'Database FE filename to open
 sFrontEnd  = "YourDatabaseFilename.accde"
 'Path to the msaccess.exe to use to open the database with
 sAccessExe = "C:\Program Files (x86)\Microsoft Office\Office15\msaccess.exe"
 
 
 
 
'===============================================================================
'///////////////////////////////////////////////////////////////////////////////
'    Do NOT edit anything beyond this point
'\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\\
'===============================================================================


'Determine path of this VBScript
'*******************************************************************************
 sVBSPath = Left(WScript.ScriptFullName,(Len(WScript.ScriptFullName) _
       - (Len(WScript.ScriptName) + 1)))
 sDBFullPath = sVBSPath & "\" & sFrontEnd

'Launch the Database for the First Time
'*******************************************************************************
 Set WshShell = CreateObject("WScript.Shell")
 WshShell.Run _
          chr(34) & sAccessExe & _
          chr(34) & " " & chr(34) & sDBFullPath & chr(34)
   
 Set WSHShell = Nothing

Copy the above code and insert it into a blank text file.  Rename the file's extension from .txt to .vbs then edit the 2 User defined parameters, save and then place in the same folder as your database front-end file.  Now, you need only double-click on it to open the database using the specified exe.  You can also now create a shortcut to this vbscript file as you see fit.

ken sheridan say:

A possible reason why all expected rows are not returned is that some dates in the table include non-zero time of day elements.  Unless you have taken specific steps in the table definition to disallow such values you cannot guarantee that such values are not present.  This will cause days on the final day of the defined range to be omitted from the query's result table.

 

The way to ensure that all expected rows are returned, regardless of any time of day element in the value, is to define the range not with the BETWEEN…..AND operator but as follows:

 

[YourDateColumn] >= [Beginning date] AND [YourDateColumn] < [Ending date]+1

 

Another possibility is that the parameter values entered are being misinterpreted as arithmetical expressions.  Parameters of DateTime data type should always be declared as such.  This can be done through the interface in query design view, or by adding a PARAMETERS clause to the query in SQL view:

 

PARAMETERS [Beginning date] DATETIME, [Ending date] DATETIME;

SELECT etc.
