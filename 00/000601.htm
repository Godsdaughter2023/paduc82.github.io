john korchok say:

You have a prefs file that is not saving correctly. Please try the following steps:

  1. Close Word.
  2. Hold down the Alt key while clicking on the Go menu and choose Library. Your hidden user Library folder opens.
  3. Open Library>Preferences.
  4. Find com.microsoft.Word.plist and drag it to your desktop.
  5. Restart Word, then close it. This action writes a new copy of the prefs file.
  6. Open Word and see if saving as PDF preserves your default Word file format setting.

julie fka profjulie say:

Morgof:  I appreciate the link you provided.  That article included alink to a thread on XDA's forum where this issue was discussed at length.  Here are the details I gleaned from reading through the thread:

  • The software in question is called 3D Anti-Terrorist Action
  • The software was downloaded either through Open Market (also referred to as OMarket ), from FreewarePocketPC.com or from 1800PocketPC.  
  • Installing the software introduced a trojan to the windows mobile phone. 
  • The trojan will automatically make a series of international calls and satellite calls (there are 6 different phone numbers that the trojan calls) in the late evening/early morning hours and resets itself to run every 30 days. 
  • Many users were not adversely affected by this, but users whose phones are not restricted from dialing international or satellite calls could potentially incur the cost of these expensive calls.
  • At the time this trojan was being discovered by the users on XDA, none of the mobile antivirus software found the problem.  These users contacted the antivirus software vendors, provided files and information to the vendors and the mobile antivirus software was updated to find this particular trojan.
  • Many users found it easier to perform a hard (factory) reset of their device rather than wait for the mobile antivirus software vendors to update their software. 
  • Apparently the trojan was incorporated into a legitimate shareware application by a hacker or group of hackers in Russia.

In summary, a trojan was introduced into a piece of shareware software that was then distributed through a couple of free mobile software websites.  Running mobile antivirus software on the device did not protect the devices against the trojan because the antivirus software in both the mobile and the computer world follows the paths of virus, trojans and other vermin and can only act in a reactive mode (versus a proactive mode).

In this particular instance, being vigilant (like many of the users on XDA's forum were) by checking the phone's call logs regularly and/or turning the phone OFF at night limited the problem.  Also blocking international calling and satellite calling if you never have a need to make these kinds of calls seems to be a good precautionary measure.

While this issue raises the level of concern, rushing out to install spyware/antivirus software to a mobile device just seems like an over-reaction to one fairly isolated event. As this case demonstrates, installing antivirus software to a mobile device is not the most effective way to avoid such things.

The best case of protection against this sort of thing is:

  • Backup your windows mobile device regularly to a storage card, especially BEFORE you install any questionable software to it
  • Be careful and selective when installing software you download from free windows mobile software websites

biligsaikhan say:

Fixed my problem. Certificate requires key from the computer/browser that was requested from. I used my computer/browser for the first certificate which worked fine. The second certificate was from colleague which we share the email account. I got the key from his computer/browser the certificate now works fine. 

I found out computer/browser generates pair of keys when requesting certificate which is used to validate the user. One key is sent with request other is on hardware. To use certificate for multiple users on multiple PCs everyone needs both Certificate and the Key from the original user who sent the request.
