macropod say:

That suggests either the names were applied after the links were established or that the ranges used for the links were not exactly the same as the named ranges. Regardless, if you expose the field coding for the links (e.g. press Alt-F9), you should be able to change the explicit addresses to the range names.

For example, supposed the LINK field shows:

{ LINK Excel.Sheet.8 "C:\\Users\\Schippers\\Documents\\ReportData.xlsx" "Sheet1!R1C1:R10C10" \a \p }

but you want it to reference a range named 'Table1', you could edit the field code to:

{ LINK Excel.Sheet.8 "C:\\Users\\Schippers\\Documents\\ReportData.xlsx" Table1 \a \p }

When you're done editing, press Alt-F9 again, then press Ctrl-A, F9 to update all the links.

Sub ScratchMacro()
 'A basic Word macro coded by Greg Maxey
 Dim oPar As Paragraph
   For Each oPar In ActiveDocument.Paragraphs
     On Error GoTo BadLayout
     If oPar.Range.Characters.Last.Previous = ":" Then
       oPar.KeepWithNext = True
     End If
ReEntry:
   Next

Named ranges, INDIRECT, and non-consecutive data? - Microsoft Community

theknightmike say:

Named ranges, INDIRECT, and non-consecutive data?

It goes without saying that using named ranges can be very useful. But I recently learned that INDIRECT can make them even more useful, because then the named range itself can become a "variable" in a set of math functions - so that the math functions can e.g. be copied across some columns, then you can enter a named range as the "title" of the column and voila, that column now gives you statistics for the range you just typed in.

...I hope this is all making sense...

Anyway, this worked great for a little while until I came across a non-consecutive set of data. What do I mean by that? I mean that instead of a "simple" range like A1:A10, it is something more like A1:A5,A8:A20.

Either way of saying the ranges works fine if you turn it into a named range, and then use a math function like AVERAGE() on it ... but if you also have an INDIRECT in your argument, the latter case breaks. In case it's not clear, my formulas look like:

   =+AVERAGE(INDIRECT([cell ref]))

where [cell ref] points to a cell that contains the name of the named range I want to use in the function. (And in my case, it's at the top of the column of math functions that are being performed on that range. Nice, huh?)

So, it works fine when the named range points to e.g. A1:A10, but gives a #REF! error when the named range is e.g. A1:A5,A8:A20. Even though those otherwise work fine in the function.

I hope this is making sense. Can anyone help?

   Mike

Note: Please don't ask me to move the non-consecutive cells together. There are reasons why that's a bad idea (I am jumping over bad data points which need to persist for documentation's sake).  :-)


peter atherton say:

Can you think of any way to merge the range on the "other side" of the named range?

You can create a range comprising two blocks if A1:A5,A8:A20 is called data then
=Average(Indirect(F3)) gives the answer when F3 contains the name data.

However you can not use it  with other functions. Say you have a second range in column B using the same rows. =AVERAGE(OFFSET(Data,0,1)) return a Value error.

 

If you split the two initial ranges and call them Data1 and Data2  and place these names in F3 and G3 then  =AVERAGE(INDIRECT([F3]),INDIRECT([G3]))  just as Ronaldo has said. But the offset function will work on this

 

=AVERAGE(OFFSET(INDIRECT(F3),0,1),OFFSET(INDIRECT(G3),0,1))

 

What method you decide will depend on the rest of your spreadsheet.

Peter
