suzanne s. barnhill say:

That it opens in Compatibility Mode even in Word 2010 suggests that it was created for Word 2007 or earlier. You can probably figure this out by looking at the Compatibility Options at the bottom of File | Options | Advanced, which will say, "Lay out this document as if created in [some version]."

 

Yes, there's no need for a text box to enclose a wrapped graphic, and the text box often makes it harder to manipulate the graphic. My guess is that the text box is intended as a placeholder, so that the graphic can be deleted and replaced without changing the layout.

 

The text boxes are obviously linked. To learn how to achieve this, see the video at http://youtu.be/Kuy1GwsGXpo

 (you'd think there would be a text article at Microsoft's site somewhere, but if there is, I haven't found it, and this video is short and sweet).

arzi(2) say: Absolutely fantastic, Suzanne! You're such a gem and I appreciate that link. Who knew!

I actually decided to take my inspiration from the Microsoft template and just start over with a table structure and fills and create my own without all the layers of text boxes. I tried to use the simplest method possible to arrive at a basic template that would work in both Word 2013 and 2010 without compatibility issues. 
 
This might help someone else, so I'll share it. In order to create a gradient background in my table cells, I created a text box with a gradient fill (INSERT tab, Text group, Text box button, Draw Text Box; DRAWING TOOLS, FORMAT tab, Shape Styles group, Shape Fill list, Gradient); then I clicked inside a table cell with all internal margins set to 0 (TABLE TOOLS, LAYOUT tab, Table group, Properties, Cell tab, Options button, uncheck Same as the whole table, change Left and Right boxes to 0, OK twice). I cut the text box, clicked in the desired cell, and pasted In Line with Text. I sized the text box and dragged cell borders until the text box fit perfectly within the cell. I left borders on everything until I was finished so it was easier to see what I was doing. I also like using View Gridlines during the design process. Piece of cake--NOT! 

I liked the gradient fills in the text boxes of the original template, so I cut/paste those into my finished one and played around with them. 

I also find Word crashes with great regularity when I'm working with complex table structures like this, especially when I start scrolling and dragging. Nothing worse than being in the middle of a complex job and have Word check out!

This is a great time to use TABLE TOOLS, LAYOUT tab, Cell Size group, Height and Width boxes and change numbers there using their list arrows to finesse the table size.
