zachary hamilton say:

I just ran into this issue while reimaging some old machines back to factory image in preparation for decommissioning them.  I also sometimes ran into a situation where Windows would be "Checking for updates..." forever.

A lot of material I ran into talked about a DNS virus, but since I had just reimaged them, I knew that was not the case.  I also saw articles about complicated and time-consuming ways of clearing and resetting Windows Updates components.  Those didn't work for me.

I seem to have stumbled across two possible workarounds:

1. Make sure the PC will not go to sleep or hibernate, and then allow the machine to run it's updates automatically.  Assuming you have the machine set to the default, it will check for updates every day at 3am, which of course you can change.  In my case, the machines would error out if I tried to run Windows Update manually, but they would run successfully if I allowed them to run automatically.

2. The other thing I did was change the value for Data Execution Prevention.  I changed it, applied the change, then changed it back to it's original value, and applied the change again.  Reboot, and it works.... mostly.

I would often fix the issue for an update cycle or two, and then have to do it again.  I'd have to reboot and adopt one of the above strategies all over again.

Hope that helps.

File naming using curly brackets - Microsoft Community

t terrific say:

File naming using curly brackets

I notice Windows contains some files whose name consists of HEX strings separated by hyphens and bounded by curly brackets. What are these?
jeeped say:
The files/folders you are seeing are named using a GUID or Globally Unique Identifier. These are the alphanumeric characters grouped in sets separated by hyphens with the collective result wrapped in braces. There are typically used to note something as unique, whether it is a user profile folder or recycle bin for a drive letter. These are usually hidden and often carry the System File attribute as well.

The GUID references are also used extensively within your registry for cross-referencing related configuration settings.

Perform a SFC scan.

Sfc /scannow will inspect all of the important Windows files on your computer, including Windows DLL files. If System File Checker finds an issue with any of these protected files, it will replace it.

  1. Open the Task manager.  Here’s a tip:  Press CTRL+Shift+ESC.
  2. Click File > Run New Task
  3. Make sure you have a check mark beside “Create this task with administrative privileges”
  4. At the Command Prompt, type: sfc/scannow   

If issue still persist after running the SFC scan then you may try to run the DISM commands and see if it helps you to fix the issue.

Deployment Image Servicing and Management (DISM) installs, uninstalls, configures, and updates the features and CBS packages in offline Windows Embedded 8 Standard (Standard 8) images and offline Windows Preinstallation Environment (Windows PE) 4.0 images. The commands and options that are available for servicing an image depend on whether the image is offline or online. All commands work on an offline Standard 8 image. Subsets of the commands are available for servicing a running operating system.

 

I would suggest you to run the below mentioned commands in elevated command prompt and check if it helps.

  1. Press Windows key and X at the same time.
  2. Choose Command Prompt (Admin) from the context menu.
  3. In the Command Prompt, type the following command, and then press ENTER:

          Dism /Online /Cleanup-Image /ScanHealth.

This command is used to check if an image is repairable. Scan the image to check for corruption. This operation will take several minutes.

           Dism /Online /Cleanup-Image /RestoreHealth
