a. user say:

Help me move inventory from one location to another

scott mcdaniel say:

How are you managing stock now?

Generally you would have a Transaction table that would record your items from the point where you get them (either through a PO Receipt, or through a internal Job process) to the point where you dispose of them in some manner (sell them, scrap them, use them in an assembly, etc).

With a transaction table, you'd have 2 records for each movement - one for moving the stock OUT of the first location, and a second for moving the stock INTO the second location.

Note too that inventory can be very, very complex to handle correctly. There are many off-the-shelf products that will manage inventory for you properly. Have you investigated those?

scottgem say:
I'm an independent advisor and I believe I can help.

The foundation of any successful database is a proper table structure. so you need get your tables right first.

The center of an app to track inventory is a transactions table. A transaction table will track each movement of stock in and out. So there are two ways you might handle this.

1) You would add a transaction for each receipt or disbursement of stock from each location. You could then open a form listing all incoming transactions And adjusting the location to increase/decrease the locations.

2) You would add a child table that showed the location listing for each transaction You would then adjust the numbers in that table to move stock around.

ken sheridan say: You have a many-to-many relationship type between Products and Locations which should be modelled by a table like this:

ProductLocations
….ProductID  (FK)
….LocationID  (FK)
….Quantity

The primary key of the table is a composite of the two foreign keys, with Quantity as a non-key attribute of the relationship type.

To move stock from one location to another requires the Quantity value in one row being decreased and simultaneously being increased by the same amount in another row.  To do this I would suggest using an unbound dialogue form in which the product, quantity and the two locations are entered, with a button to then execute two UPDATE queries within a Transaction.

The script message is probably from a Word mail merge. You should expect that message when using Mail Merge.

When a message is stuck in the Outbox it is almost always because there is a malformed email address in the TO, CC or BCC field.

mikemac888 say: im not doing a mail merge.  just simply opening a word file putting the word Test in the document and then trying to send via outlook using the File - Share - Send Document menu option.   i get security error message.

jim gordon mvp say:

It's not an error message. It's a nagging message to alert you to the fact that another application is telling Outlook to do something. In this case, Microsoft Word is telling Outlook to open a new, blank message and fills in the message body. Outlook wants to make sure you know you did what you did just in case you forgot what you just did, or in case some rogue application is trying to take over your email.

In other words, if you see that message when you send an email via Microsoft Word that is normal. If you didn't do anything at all and see that message then Outlook is telling you something is trying to take over your email.

Find Cell Containing Most Characters - Microsoft Community

basilides say:

Find Cell Containing Most Characters

Greetings!

How does one determine which cell in Column B contains the most characters, and ideally indicate the number of characters?

mike h.. say:

Hi,

Both ARRAY formula, see below.

Longest string

=MAX(LEN(B2:B2000))

and it's address

=CELL("address",INDEX(B2:B2000,MATCH(MAX(LEN(B2:B2000)),LEN(B2:B2000),0)))

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.
