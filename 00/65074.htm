e) Now, check if you are able to boot the computer successfully.

Multiple Instances of USB printer

I have an old HP laser printer (Laserjet 4p) that has a parallel port.  To use this with newer computers, I purchased a parallel to USB converter cable.  I'm now using it with my new home computer, one running 64-bit Vista.   The printer works fine in that Windows finds the printer, correctly identifies it, and then installs it.   Once installed everything on the printer works.   The problem is this: I leave the printer connected all the time.  Upon rebooting from a cold state, Vista frequently (maybe always???) reinstalls the printer and gives it a new name.   Instead of "HP Laserjet 4P",  I end up with "HP Laserjet 4P" and "HP Laserjet 4P (Copy 1)" and "HP Laserjet 4P (Copy 2)" etc.    The last copy is usually the active printer, and the other copies are shown as off line or not connected. 

When this happens, I have no trouble deleting the duplicate entries, renaming the active printer to "HP Laserjet 4P', making it the default, etc.    However, my wife also uses the computer, and this frequently trips her up.   She regularly sends documents to the 'off line' copy, then complains when nothing prints.  Then I have to get involved to get the documents reprinted, clean up the extra copies, and so forth.

Is there a way to avoid the creation of multiple copies (without doing something equally aggrevating, like crawing behind the desk every time to disconnect the printer, or having to power down the printer separately from the PC)?

This also used to happen when she used the computer with her XP laptop from work.   I wrote it off as a glich with USB on an older operating system or older hardware.  However, I'm disappointed to find the same behavior in a new computer and new OS.   Is there some deficiency in these parallel to USB converters that can't be dealt with?
