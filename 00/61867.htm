Windows System Error: There is an IP address conflict with another system with the network.How do I eliminate this error message?

I just installed a new internet provider and all my connections are working great but I continually get this "Windows System Error: There is an IP address conflict with another system with the network." I have my internet modem hardwire connected to my Desktop. I also have two wireless devices connected to the Wireless aspect of the Modem. I continually get this error message mentioned earlier and I want to get rid of it.
brian tillman [outlook mvp 2007-2019] say:

It means just what it says.  Your Windows system has an address that already exists on your LAN.  Turn your system off and use a working system to locate that address and to which device it belongs.  In most LANs, the router is the DHCP server.  You may be able to ask the router what addresses are in use and see which device is using that address, but I suspect not.

Duplicate addresses can happen when two systems have been assigned the same address manually or when one has been assigned an address in the DCHP range of the router. It doesn't know when this has happened because it's not in control of the address and hands out the duplicate address to a second system.  You'll need to track this down, making sure that any manually-asigned addresses are out of the DHCP range, that you reserve the address in the router if you want the device to use that address, or configure all devices to request addresses from the router using DHCP.

tryx3 say:

On the friend's computer, go to Computer management [type that in the Start menu search box], Storage, Disk management.  Select the drive, right click & assign it a particular drive letter, say K.  This letter always then applies to the USB device [not the USB port] when used on that computer.

RoboCopy syntax does not look like that for other commands as it works on entire folders not individual files.

ROBOCOPY  "K:\" "C:\USERS\PUBLIC" /MIR

Will make a precise mirror-image copy of the K:\ drive into C:\USERS\PUBLIC, but for other actions such as simple copying see RoboCopy /? [type that in a Command window].  The /MIR switch removes files from C:\USERS\PUBLIC that do not exist in K:\.
