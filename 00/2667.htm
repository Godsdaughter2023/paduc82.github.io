Any idea why Word 2013 ignores a style specification but, Word 2007 - Microsoft Community

m.k.r say:

Any idea why Word 2013 ignores a style specification but, Word 2007 does not do that with identical specification, as demonstrated below?

  • In Office 2007, we notice that the Heading 1, as per specification, leaves 250 spaces above the heading.
  • But then, with same specification in Word 2013, Heading 1 positions itself at the top of the page, ignoring the same spec.
  • Any solution?

suzanne s. barnhill say:

In Word 2013, sadly, you cannot achieve it with Spacing Before alone, even with a page break (and I did see that "Page break before" in the style definition but absent-mindedly thought of it as controlled by the "No" before "widow/orphan control"). I am not the only user to feel that this was a terrible design decision, but for the moment there is nothing that can be done about it.

Your only recourse, therefore, is to add the space some other way. The simplest (but not the best) way is to add an empty paragraph before the heading (it will have to be formatted as "Page break before" instead of the heading). Or you could insert the heading in a single-cell borderless table, which will also preserve the Spacing Before.

The best way, although it is a little more difficult to set up initially, is a lot easier to use in the long run, and that's to include a frame in the Heading 1 formatting.

  1. Right-click Heading 1 in the Quick Styles gallery and choose Modify...
  2. In the Modify Style dialog, click Format and choose Frame... You'll get the dialog shown below.
  3. The "Text wrapping" will be set to Around, and you'll need to change that to None. Both height and width will be set to Auto, and, having changed the wrapping, you can leave those alone, but if you want the frame always to be full margin width, you can set it to that.
  4. Click OK twice to close the Format Frame and Modify Style dialogs. Now whenever you apply Heading 1, it will be in a frame. Frames are considered In Line With Text (even when wrapped), so their content is part of the text flow and is fully visible to Word (all versions, even the older ones that don't see text in a text box) and will be selected when you Ctrl+A to make format changes, update fields, etc. The headings will also appear in the Navigation pane.

suzanne s. barnhill say:
If the frame is not wrapped, that check box is probably not needed, but it can't hurt. A frame, even when it is In Line With Text, is technically anchored to a text paragraph and will stay on the same page with that paragraph. If the anchor is locked, the anchor can't be dragged to a different paragraph (even if you drag the frame to a different location). By default, the framed heading is anchored to the following paragraph, which is what you want, so locking it would probably be a good idea.
