jhl1951 say:

Auto Macro does not work remotely

Using Excel 2007

I have a spreadsheet that contains an auto run macro which executes when you open the spreadsheet.  It runs in about 30-45 seconds.  It minimizes Excel to the bottom of the screen, and closes when it finishes.  It populates another spreadsheet with data stored on a server.

When I am not in the office, I would like to run this process from home.  I can access the file through a secure VPN connection, but when I attempt to open the file to execute the macro, it will minimize the sheet, but it just flashes at the bottom of the screen like it working, but it takes too long.  I've let it sit there flashing for 5-10 minutes before terminating it.  Remember when I'm at work the macro takes 30-45 seconds tops.

What could prevent this from running remotely?  I'm using Excel 2007 at work and home.  The macro was written by someone who is no longer with the company, and I don't know macro codes very well.

Thanks for any help

timjwilliams say:

If the macro is performing a lot of network requests (eg. to the database) then it could be expected to be significantly slower when opened remotely.    To fix this you may need to change how the macro accesses data, for example by having it make fewer but larger requests.

Alternatively, if you have the correct permissions, you could remote desktop into a PC at work and run it from there.  It will then be running locally, and should be as fast as before.
