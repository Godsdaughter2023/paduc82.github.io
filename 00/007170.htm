uk4uk say:

Access: create a new record dependent on a combo box selection - and clone specific fields from the previous record into the new record

Hi - I have an access database that I use for tracking orders.

One field in the form I use is a combo box (cmbStatus) that is used to identify an order status. 

We occasionally have standing orders that recur at odd frequencies and that only need to be entered after the previous order is completed.

When we have one of these recurring orders, what I'd like to do is the following: I have a selection called "Standing order complete" in the cmbStatus combo box. Once selected this would open a new record in the form and clone the entries from several other fields (e.g. Client ID; OrderType) into the same fields of the new record.

In the After Update of cmbStatus I've added the following code that does open a new record appropriately:

Private Sub CmbStatus_AfterUpdate()
If Me.CmbStatus = "Standing order complete" Then
DoCmd.GoToRecord , , acNewRec
End If
End Sub

However, I'm running into problems cloning the fields from the previous record into the new record. I've tried following the suggestions from this source: https://support.microsoft.com/en-us/kb/210236. But I'm having trouble "linking" this approach to the AfterUpdate code above.

Any suggestions about how to accomplish this would be fabulous!

ken sheridan say: I'd suggest that you assign the values from the relevant controls in the form's current record to the DefaultValue property of each of the relevant controls.  When you move to a new record those values will then be in place, ready for additional data to be inserted.  Note, however, that no new record will be created if no additional data is inserted and the values will be lost if the form is closed without doing so:

Private Sub CmbStatus_AfterUpdate()

    If Me.CmbStatus = "Standing order complete" Then
        Me.[Client ID].DefaultValue = """" & Me.[Client ID] & """"
        Me.OrderType.DefaultValue = """" & Me.OrderType & """"
        ' do the same for any other controls whose values
        ' you wish to carry forward     
        DoCmd.GoToRecord , , acNewRec
    End If

End Sub

The default values will apply to all new records you go to in the current session of the form, so if you only want them carried forward to one record you'll need to remove the default values in the form's AfterInsert event procedure:

    Me.[Client ID].DefaultValue = """"
    Me.OrderType.DefaultValue = """"
    ' do the same for other controls whose values
    ' have been carried forward

Note that the DefaultValue property is a string expression regardless of the data type of the column to which the control is bound, so should always be wrapped in literal quotes characters as above.

When you read you begin with ABC, when you sing you begin with do re mi.

When you use MSP you start with TDPRL, which stands for tasks, durations, predecessors, resources, leveling.

You need three resources, "Bob the builder", "Bricks 1" and "Bricks 2". The column next to the resource name column is the type.

Now go back to the Gantt chart view and select the first task.

On the resource ribbon, click on the assign resources button.

In the box/dialog, select Bob and then assign.

Bob is assigned for 8 hours/day for 10 days and costs $50/hour, so there is 80 hours of work and that task costs $4000.

You are currently in the Gantt chart view, and the entry table is the active table.

To see the cost, you need to switch to the cost table.

On the view ribbon, there is a drop down button for tables. Drop it down and select the cost table from the menu.

Go back to the entry table and continue making the list of tasks. You will also want to indent them under headings and sub headings, or summaries as they are called in MSP.

Continue making a list of resources and assigning them to tasks. Continue to check the cost table and the work table as you go.

That's it. Whether there are two tasks or 20000 tasks, TDPR until it's finished. L comes later. And read up on the critical path method.

It's not a problem with .csv files - it also happens if you enter 1.271 and 1.276 in an Excel workbook and subtract them. When you increase the number of decimal places displayed in the cell, you'll see the same result.

It is a rounding error that is caused by the conversion between the decimal numbers that we see and the binary format that Excel uses internally. You can avoid them by rounding the result to a limited number of decimal places, for example

=ROUND(C9-C8, 4)

Windows Defender isn’t supposed to be turned off via Group Policy when a third-party AV app is installed, as I’ve explained in this thread:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-defender-is-off-i-cant-turn-it-on/c1ef0d9b-7940-4108-9578-de2c996b75ea

So when it gets disabled like this on a home PC; we would normally suspect that the Group Policy switch might have been thrown by malware that’s somehow managed to escalate its privilege-level to the rank of  “administrator”. It’s also quite likely that the malware was able to accomplish this because Defender didn’t have a signature/behavioral detection for it – so it would be an excellent idea to scan the PC with this set of malware-removal tools:

Kaspersky Virus Removal Tool:

http://support.kaspersky.com/viruses/kvrt2015

Emsisoft Emergency Kit:

http://www.emsisoft.com/en/software/eek/

Malwarebytes Anti-Malware (free version only):

https://www.malwarebytes.org/antimalware/

Eset Online Scanner:

http://www.eset.com/us/online-scanner/

Some other trusted third-party malware-removal tools are listed here:

https://answers.microsoft.com/en-us/protect/wiki/protect_other-protect_scanning/list-of-malware-removal-tools/d824b9af-ebd8-4c47-94e2-8ee6c544c100

Then turn Defender back on with this REG command: Right-click on the Start button, select Command Prompt (Admin), and then copy, paste, and run (enter) this command line:

REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware 

As I’ve illustrated here:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender-this-app-is-turned-off-by-group/dfa59c2a-b358-4f62-80b6-6b8e4994dc32

https://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

Turn_On_Windows_Defender.reg is a trusted file, and this is all there is to it:
