Add the following function to your code module:

Function findFirstVisibleFilteredCell(StartCell As Range)
    Dim aRng As Range
    Set aRng = StartCell.CurrentRegion  'aRng also contains the header row
    Set aRng = aRng.Offset(1, 0).Resize(aRng.Rows.Count - 1) 'and now it doesn't
    Set findFirstVisibleFilteredCell = aRng.SpecialCells(xlCellTypeVisible).Cells(1, 1)
    End Function

Now, use it as

    findFirstVisibleFilteredCell(Range("H2")).Select

netmc say: I found out what the problem is with this.  I've only seen this after a computer has been infected with malware, then cleaned.  The malware messes with the files and folders under "c:\program files\Microsoft Security Client" folder and makes all the files and folders into junction points (or symlinks).  You can verify this my going into the folder from a command prompt and typing in "dir".  It will list all the files as <JUNCTION> under Windows XP and <SYMLINKD> in Windows 7.

Windows XP cannot handle junction files properly in Windows Explorer (and most command-line utilities).  Attempting to delete the junctions will cause the target file to also be deleted.  Windows Vista and 7 do not have this problem and can work with junction points just fine without affecting the target file.

However, because these are corrupted junction points, windows cannot process them correctly.  I have tried using both junction.exe from sysinternals, and Junction link magic to attempt to delete the corrupted junction points, without any luck.  The actual sub-folders have files in them, so they can't be deleted when the junction point is removed.  (You may be able to delete these from linux.)  You also cannot rename the folder in windows or in safe mode.  The only way around this is to slave the drive to another computer.  then you can rename the c:\program files\Microsoft Security Client folder to something else.  Once that folder is renamed, you can reinstall Microsoft Security Essentials.
