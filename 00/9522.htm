rohn007 say:

What will I lose in the process? 

What do I need to backup/preserve? 

What else do I need to consider?

The Office applications (Word, Excel) were designed from the start, back in the 1980's!, to be uninstalled and re-installed as part of the trouble shooting process.  The idea is you can do a "normal" uninstall / re-install of Office without impacting anything.  When you do a "normal" uninstall it leaves behind application configuration registry entries and files (like Word normal.dotm).  Then when you install (also if you upgrade) Office again, it picks up this remenant configuration information so that the new installation looks exactly like the old one did.

This page lists the 3 ways to uninstall Office:

https://support.office.com/en-us/article/uninstall-office-from-a-pc-9dd49b83-264a-477a-8fcc-2fdf5dbf61d8?ui=en-US&rs=en-US&ad=US

The next Part of the trouble shooting process after normal uninstall / re-install is to do a "full" uninstall, which wipes all of that configuration information on the possibility that the problem exists in the configuration information.  That is when you have to worry about first doing some backups.

FYI:

So the general Office trouble shooting process is:

  • Restart the application
  • restart Windows
  • Start the application in Safe mode (disables all addins which have a long tradition of causing problems)
  • Start Windows and the application in safe mode
  • normal uninstall of Office
  • uninstall Office using the "Fix It" tool

karl timmermans [outlook mvp 2012-2018] say:

Several things

#1 - Are you entering the contact's name in the <FULLNAME> field instead of entering the name <parts> in the appropriate field by clicking on the <FullName> button brings up the window allowing you to individual name components.

#2 - What do you mean exactly by "in the file and on the little name window that pops up"? What is the "file" you are referring to?

#3 - What is the default setting for the contact's FullName setting - option found via File --> Options --Contacts --> "Default FullName Order"?

#4 - Account setting related to the contact's name relates to the AddressBook and does not apply to when a contact is first created. If the name is incorrectly constructed based on items #1 and #3

 

In short - set item #3 to the format you desire and then enter all parts of the contact's name into the appropriate fields. Unfortunately, making these corrections will not correct contacts that have already been created. To do that, you will need to (1) correct the affected contacts manually (2) write code that will do it in bulk or (3) use a 3rd party program like ContactGenie Toolkit - http://www.contactgenie.com/toolkit_4_outlook.htm which has a FullName standardization function. However, one caveat regarding CG Toolkit - there's a limit to what can be accomplished programmatically depending on how messed up the actual names are in terms of where values are stored,

kmob284 say:

VLookup Help - Matching only a piece of text

I am trying to create a lookup to match a vendor from a credit card statement to our AP system's vendor id.  The problem is that it's not a straight match.  For example, the credit card statement says, "Home Depot 0001254987", and our system is simply Home Depot.  How do I do this?  Thank you in advance for your help.
ronrosenfeld say:

OK, Given that information, I put your tables on two different worksheets 'CCStmt' and 'VendorList'.  I then inserted formulas as you can see in the screenshots below:

VendorList worksheet:

 

The formula above must be **array-entered**:

C2:  =MATCH(TRUE,ISNUMBER(FIND(A2,CCSTMT!$A$1:$A$10)),0)

----------------------------------------

To **array-enter** a formula, after entering
the formula into the cell or formula bar, hold down
<ctrl><shift> while hitting <enter>.  If you did this
correctly, Excel will place braces {...} around the formula.

 

And then, on the CCSTMT worksheet:

 

B2: =INDEX(VendorList!$B$1:$B$4,MATCH(ROW(),VendorList!$C$1:$C$13,0))
