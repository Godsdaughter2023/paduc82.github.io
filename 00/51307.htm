a. user say:

Hi,

Using PBR partition, you can recover the PC from a problem or return to the factory state. This partition can come in handy when selling or giving the PC to another member of the family or friend to give a fresh start of the OS.
 

PBR is the drive provided by product manufacturer which acts as a recovery drive. PBR drive will not interfere with any of the system programs/applications performance on the PC.

The partition boot record is the first block of any bootable partition. It is also sometimes referred to as the Volume Boot Record (VBR). The MBR (or the GRUB chainloader) loads the PBR into memory and transfers control to it.

Hence, it should be set as ON as to create restore points to aid in a system restore in case of a system failure to back up all images.

Hi Acemere , it is not easy to remove a file association in Windows 10,

This solution basically points that file type to a dummy program, then you delete the dummy program and reboot and the file association is broken and back yo unknown and hence 'Pick an App' . . .

Open Windows File Explorer

On the view Ribbon (tab) check the box marked 'File Name Extensions'

Close File Explorer

Right click your Desktop and choose New - Text File

Change the file extension on that new file from .txt to .exe

Confirm the change

Right click a one of those files, choose open with - Choose another App

Check the box marked 'Always use this app to open .xxx files' - IMPORTANT!

Scroll to the bottom and click 'More Apps'

Again scroll to the bottom and choose 'Find another app on this PC'

Navigate to the dummy file exe on your Desktop and click OK

The file will not open

Delete the dummy exe file on your Desktop and restart your PC

The file association will then be broken!

Hi,

Launch PowerShell as Administrator, and execute this command (it's a single-line command):

Get-AppxPackage -allusers Microsoft.Windows.Photos | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

This reinstalls the Photos app. [screenshots}

It is unlikely that anything outside of the signed driver or a signed app with admin access to the driver (e.g. coinstalled) can disable an audio device or more specifically an endpoint (headphones, speakers, etc).
