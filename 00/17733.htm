chrisyue8628 say:

HI Kerwin,

The issue has been resolved.

For Desktop Apps such as Outlook when enabled for MFA, a Refresh token and Access token is used.

Following initial UPN/Password authentication, a Refresh token is generated from Azure AD which is then used to obtain an Access token in this case from Exchange online.

This provides for continued authentication and is valid for at least 14 days. Every time the token is used, it gets another 14 days up to 90 days, After this time the token will expire and authentication (and MFA enrolment) will be required.

Removing the token via the Windows Credential Manager control panel applet causes the MFA prompt to appear immediately.

Steps to purge email without any traces and without any possibility to - Microsoft Community

Steps to purge email without any traces and without any possibility to recover it

In office 365 online, what are the steps to purge and email without any traces / without the possibility to recover. 

Thanks

myalo

henry huang msft support say:

Hi Myalo,

To purge an email, you need to delete it from the Deleted Items folder or hard delete it from the inbox.

Then right click the Deleted Items folder and click Recover Deleted Items. Find the email from the recoverable items folder and select it. You will get the option to purge the email. It cannot be recovered if the mailbox is not put on in-Place Hold or Litigation Hold.

If the mailbox is put on hold by the admin, any emails cannot be purged completely. Users will not be able to recover these items. But admin can find it by using in-Place eDiscovery.

In addition, if a journaling rule is configured to copy all the emails to an external email address, the message can also be found by the admin.

Regarding the traces, removing emails by the mailbox owner is not logged by Office 365. And no feature is available to trace this kind of action in O365. If the email is deleted by another user who has the permission to the mailbox, admin can turn on mailbox auditing so that they can run non-owner mailbox access report to check these actions.

Henry

a. user say:

The sources.xml file is supposed to be the "master list". 

Does it contain what you expect it to contain? You should be able to open it in Notepad and look. (Or perhaps almost all of what you expect, except recent additions).

Two reasons that it might not be working are

 a. it could be in the wrong place. For Windows 10/Office 365 I would expect to find it in 

C:\Users\<yourusername>\AppData\Roaming\Microsoft\Bibliography

assuming c:\ is your boot drive. e.g if your username is "uname" it would be in 

C:\Users\uname\AppData\Roaming\Microsoft\Bibliography

 b. it could contain invalid XML. You may be able to check it in an online validator such as https://www.w3schools.com/xml/xml_validator.asp

If it is broken, it may be difficult to tell where, but personally I would have a look at the end of the file first.
