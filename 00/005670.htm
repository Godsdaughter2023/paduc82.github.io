Based on my experiments, these tips may help:

 

1. don't use any punctuation at all in the search string

 

2. use abbreviated street types (e.g. "st" instead of "street")

 

3. the search will use the locality of your last search (including country and city) to provide context for a shorter address (e.g. street name)

 

4. don't make any typos in the address

 

So it could be that the first time you do a search, you need to include the full context (including country name) to set the locality. When I swap between Australian and Swedish addresses, I need to include the country. For consecutive searches in a country or city, I don't need to enter the country or city name.
