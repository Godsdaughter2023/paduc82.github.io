Hi,

 

There are two methods I would suggest you to try and check the status of the issue:

Method 1:

Set the computer to clean boot state. This will disable any active security programs or third party programs running in the background. There have been instances were some programs interfered with the activation process.

http://support.microsoft.com/kb/929135

NOTE: Make sure you reset the computer to start as usual after troubleshooting is complete

 

Method 2:

Insufficient permission to the 'root' directory. The System account by default has Full Control permissions to the registry path: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Enum\Root and any subkeys.

If those permissions have been altered for the 'Root' key or any subkey(s), we would see the error code 0x8007000D.

NOTE: Make sure you backup registry before you make any changes to them. Refer the following article on how to backup registry:

http://support.microsoft.com/kb/322756/

 

Assign the minimum permission of "Enumerate Subkeys" to the System account for the registry path HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Enum\Root and any of its subkeys.

a) Open registry editor window and navigate to the above mentioned path.

b) Right-click on the 'Root' folder and select 'Permission'.

c) Select 'System' under 'Groups or Usernames' and check the permission status. If its changed to only 'Read' then continue to the next step.

d) Click on 'Advanced' button and under 'Permissions' tab, select 'System' and click on 'Edit'.

e) Select 'Full Control' and click on 'ok'.
