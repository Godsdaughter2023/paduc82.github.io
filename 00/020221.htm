jetta48 say:

Unrequested phone calls of this nature are almost always a common scam. Do not give them any info, do  not give them access to your PC, do not give them any money and do  not go to any websites they suggest. Hang up.

 

Please see:

http://www.microsoft.com/en-gb/security/online-privacy/avoid-phone-scams.aspx

 

Microsoft never makes unasked for support or security phone calls.

 

These types of callers may use completely fictitious names or bogusly claim to be from genuine companies other than Microsoft as well.

 

These types of scams are fraud and are almost certainly criminal offences in most countries. You can report them to your local law enforcement/authorities. However these scammers operate worldwide across political/international borders which makes tracking them down and stopping them difficult.

 

Knowledge is power with scammers. Tell your friends & family.  :)

 

One of the things to remember with most scammers is that they initiate the contact with you and not the other way around. So whether its a phone call claiming to be from Microsoft support, or an email saying you've won a lottery that you haven't entered or someone knocking on the door offering to re-surface your driveway, be very suspicious. If in doubt, hang up, don't reply & delete that email or say, "No thanks" and close the door. If you want something, do your own research and find a known reputable company/person to help and make first contact yourself.

 

Some other anti-scam tips are:

  • If something seems to good to be true (like you've won a lottery that you haven't entered) then it is probably a scam.
  • If some email, post, phone call asks for personal info. such as email addresses, passwords, PIN nos. then there's a strong chance that its a scam. Check independently with the company/organization if possible using known good contact details (not anything that's in the email, etc.). For example, if you get an email claiming to be from your bank asking for information, look up your bank's phone no. yourself in the phone book or get it from a bank statement and ask them for confirmation.
  • If an email/phone call says something very bad will happen (like your email account will be closed or your computer is about to crash) then this is another trick used by scammers to try and make you rush into falling into their trap.
  • Having an up to date real-time anti-virus/firewall software, up to date Windows and other programs is a vital part of your computer security. However, don't become complacent just because you have this in place. One scam phone call, etc. in which you give the scammers access to your PC can completely bypass all of these precautions.
  • Use common sense and be suspicious.

ken sheridan say:

I'm looking for a solution for subform? Do you have any, please?

A split form is the easy solution, but using a bound parent and subform, the latter in continuous forms view, is relatively straightforward and gives you more control over appearance.  Both parent and subform have the same RercordSource.  You'll find an example in FindRecord.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

In this little demo the option for a 'simulated split form' illustrates how to build such a form/subform.  The relevant code in the parent form's module is:

Private Sub Form_AfterDelConfirm(Status As Integer)

    Me.sfcAddresses.Form.Requery

End Sub

Private Sub Form_AfterInsert()

    Dim strSQL As String
    Dim lngAddressID As Long
   
    lngAddressID = Me.AddressID
   
    ' requery subform to include new record
    ' and go to parent form's current record in subform
    With Me.sfcAddresses.Form
        .Requery
        .RecordsetClone.FindFirst "AddressID = " & Nz(Me.AddressID, 0)
        If Not .RecordsetClone.NoMatch Then
            .Bookmark = .RecordsetClone.Bookmark
        End If
    End With

    ' insert row into SelectedAddresses table
    strSQL = "INSERT INTO SelectedAddresses(AddressID) " & _
        "VALUES(" & lngAddressID & ")"
       
    CurrentDb.Execute strSQL, dbFailOnError

End Sub

Private Sub Form_Close()

    On Error Resume Next
    Forms("frmOpen").Visible = True

End Sub

Private Sub Form_Current()

    ' go to parent form's current record in subform
    With Me.sfcAddresses.Form
        .RecordsetClone.FindFirst "AddressID = " & Nz(Me.AddressID, 0)
        If Not .RecordsetClone.NoMatch Then
            .Bookmark = .RecordsetClone.Bookmark
            Me.FirstName.SetFocus
        Else
            Me.sfcAddresses.SetFocus
            On Error Resume Next
            DoCmd.GoToRecord Record:=acNewRec
            Me.FirstName.SetFocus
        End If
    End With
           
    ' synchronize unbound combo box with form's current record
    Me.cboGotoContact = Me.AddressID
   
End Sub


The relevant code in the subform's module is similar:

Private Sub Form_AfterDelConfirm(Status As Integer)

    Me.Parent.Requery

End Sub

Private Sub Form_AfterInsert()

    Dim lngAddressID As Long
    Dim strSQL As String
   
  
    lngAddressID = Me.AddressID
   
    ' requery parent form to include new record
    ' and go to subform's current record in parent form
    With Me.Parent
        .Requery
        .RecordsetClone.FindFirst "AddressID = " & Nz(Me.AddressID, 0)
        If Not .RecordsetClone.NoMatch Then
            .Bookmark = .RecordsetClone.Bookmark
        End If
    End With
   
    ' insert row into SelectedAddresses table
    strSQL = "INSERT INTO SelectedAddresses(AddressID) " & _
        "VALUES(" & lngAddressID & ")"
       
    CurrentDb.Execute strSQL, dbFailOnError

End Sub

Private Sub Form_Current()

    ' go to subform's current record in parent form
    With Me.Parent
        .RecordsetClone.FindFirst "AddressID = " & Nz(Me.AddressID, 0)
        If Not .RecordsetClone.NoMatch Then
            .Bookmark = .RecordsetClone.Bookmark
        Else
            On Error Resume Next
            Me.Parent.FirstName.SetFocus
            DoCmd.GoToRecord Record:=acNewRec
        End If
    End With
   
    ' set value of hidden control to current AddressID value
    ' to facilitate conditional formatting
    Me.txtAddressIDHidden = Me.AddressID

End Sub
