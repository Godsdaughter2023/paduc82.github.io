jay freedman say:

Text wrapping around pictures inside a text box is not supported in Word. That is by design. It has not been supported in any version of Word and is not likely to be supported in the future.

In Word, unlike in Publisher, text boxes are not in the main text layer; they float over it. In order to have a picture (or the sort of freehand shape shown in your screen shots) have the box's text wrap around it, Word would have to put the picture in a third layer "above" the text box -- but there is no such layer in Word.

Also unlike Publisher, text boxes are often not necessary in Word documents. You can usually accomplish the layout you want with (maybe borderless) tables.

diana d say:

Hi,

Mixed content situation can happen on HTTPS secure websites such as online stores or financial sites that display images, banners, or scripts that are coming from a server that is not secured. IE has defaulted to prompt user for next course of action when encounter mixed content as displaying mixed content carries risk that non-secure web page or script will be able to access information from the secure content.

Having to click “No” (or “Yes” to block non-secure content) button for every visit to website with mixed content can be troublesome and annoying. Besides, its possible user accidentally clicks “Yes”, which prevents some insecure elements from loading and displaying. For user who confident that he or she does not visit possibly malicious site, it’s possible to instruct IE to display mixed content automatically, instead of asking for input. 

How to Disable and Turn Off Do You Want To View Only the Webpage Content that Was Delivered Securely Security Warning 

a.    Launch Internet Explorer.

b.    Going to Tools->Internet Options->Security

c.    Select the ‘Security’ tab

d.    Click the ‘Custom Level’ button

e.    In the ‘Miscellaneous’ section change “Display mixed content” to Enable

 

Also refer to http://social.answers.microsoft.com/Forums/en-US/InternetExplorer/thread/3f5bc58a-30d2-4dd8-a395-c556d19973f6

hansv mvp say:

This is 'normal' behavior: Formulas is always the only option in the Replace dialog. When you find and replace, you cannot search for and replace characters that only occur in the display value of a cell.

For example if a cell contains the number 1234 and if it displays this as 1,234, the Find dialog with the Values option will find the comma in the display value. The Replace dialog doesn't allow that because the comma is not part of the stored value of the cell, and hence it cannot be replaced.

Hi Ron, I'm an independent adviser and can help.

You would NOT do this in a table. As a generally rule we don't store calculated values. So this would be done in a query. The query would source the other query (or table) with the text column you want to search through. You would then add another column to that query with an expression like:

Answer: IIF(Instr(textfield,1,InputBox(searchstring)>0,True,False)

I'm using an Inptrbox so you can enter the string to search for on the fly, but there are other ways to supply the searchstring. So this expression will return a True if the string is found and false if it is not.

ken sheridan say: You can do this in a number of ways e.g.

SELECT Q1.A,
FORMAT(EXISTS
     (SELECT *
      FROM Q2
      WHERE Q2.B = Q1.A),"Yes/No") AS Matched
FROM Q1;

Or:

SELECT Q1.A,
FORMAT(Q2.B IS NOT NULL,"Yes/No") AS Matched
FROM Q1 LEFT JOIN Q2
ON Q1.A = Q2.B;
