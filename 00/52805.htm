cmdrkeene say:

Sounds great and pretty much expected.  Only servers that use ActiveSync (Outlook, Hotmail, Live, Office 365, etc.) let you create folders in the app and sync backwards to the server.

There's no such thing as "import" into the mail app because it doesn't actually store any data.  Sure it caches it temporarily so you can read your email when disconnected from wifi, but it's like a browser cache:  temporary and volatile. All the messages live on the server, and the Mail app just reads them.  If your computer burns in a fire, you don't lose anything because you can just login to the Mail app on a new computer and instantly see your lifetime of mail there instead. No backups/imports/exports are ever necessary.

Of course, if you have mail stored and orphaned offline (such as mail that got downloaded and removed from a POP server earlier), and you want to see that, you're a bit stuck. You'll need to put that mail back onto the email server if you want it to show up in the Mail app.  There's ways to do that, but perhaps the easiest is to use "real Outlook" (the one that comes with Word/PowerPoint/Excel).  If you have that, it's pretty straightforward and easy: you just import the backup from the old email program into Outlook, then drag them into whatever server folder you want to keep them inside.

    Remove Win 7 (32) from a Dual Boot with Win 7 (64) - Microsoft Community

    aheinlein say:

    Remove Win 7 (32) from a Dual Boot with Win 7 (64)

    I have a Bual Boot set up. Win 7 (64) on drive C: and Win 7 (32) on drive W:. They are on the same physical drive and Win 7 (32) was the OS that was installed first. Now after 2 month running Win 7 (64) only I want to reclaim the space and remove Win 7 (32) completely from the system and Dual Boot. I know I can set the Win 7 (64) as default and time to wait to 0 but that is not the solution I am looking for.
    j w stuart say:

    Note: Make certain you still can locate your product key for the 32-Bit version before you start.

    1) Create an Image Backup.

    2) Run chkdsk on the Windows 7 32-Bit partition (2nd partition). Do this by opening a command prompt (right click on the command prompt option and select ‘Run as Administrator’), from the command prompt enter:  chkdsk  /f  /r

    Chkdsk should not report any errors, if it does let me know immediately.

    3) Download and install EasyBCD:
    http://neosmart.net/dl.php?id=1

    Use EasyBCD to list the boot menu entries. Then use EasyBCD to remove the entry for the 32-Bit version. Use care to correctly identify and remove the correct (32-Bit) version of Windows.Note: If your boot menu list both versions as simply "Windows 7" then you may want to start by using EasyBCD to change the boot menu names from Windows 7 to "Windows 7 64-Bit" and "Windows 7 32-Bit", then verify the menu is correct in that when you select Windows 7 64-Bit from the menu it does in fact boot to the 64-Bit OS. Then remove the unwanted 32-Bit entry from the boot menu.

    4) Open ‘Disk Management’ and perform a “Quick” format of the unwanted Windows 7 32-Bit partition.

    5) Again verify you can still boot to Windows 64-Bit OS.

    6) Open ‘Disk Management’ and delete the now empty partition formally occupied by the 32-Bit OS.

    You should now have a partition map as follows:
    Partition C: Win7 64  Primary (System, Boot, Page File, Active, Crash Dump, Primary Partition)
    Unallocated
    Acronis SZ

    7) Use ‘Disk Management’ (right click on the “Unallocated” space) to ‘Extend’ the C: partition to use as much or as little of the “Unallocated” space you want.

    8) Assuming you used all the unallocated space then your partition map should look like this:

    Partition C: Win7 64  Primary (System, Boot, Page File, Active, Crash Dump, Primary Partition)
    Acronis SZ
