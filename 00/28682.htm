    Sorry, I tuned out loading build 10061.  Ok, my Dell keys were F2 and F12.  F2 duplicates the "volume up-power button" boot bypass to the BIOS boot menu and F12 duplicates the "volume down-power button" leading to the full BIOS menu.

    You also need to research the UEFI vs USB boot aspects of your system.  Meaning, my Dell BIOS actually defaults the USB drive to UEFI if you allow it to.   I found the IOS booted on my Dell only as a USB drive.  I am reading however, that you can work within the UEFI parameters, but I'm glad I did not have to do the work involved.

    Using my Dell M4300 laptop (and so much for the demise of the laptop via the tablet!) the USB drive preparation I followed was:

    1) Format the USB drive with my laptop.

    2) "Mount" the ISO file download by right clicking on the ISO file.  The extracted or expanded boot and repair files appear on a "DVD drive" created on the laptop system drive.  Copy and paste those files onto the USB drive.  I also copied the un-extracted ISO file onto the USB drive as is.

    3) In the tablet BIOS menu I set the boot sequence to USB only, and only after inserting the USB, and after it was recognized by the BIOS menu as an option. This order of USB insertion negated any UEFI conversions.

    4) Apply or ESC, let the system boot or power up again, and follow the install menu accordingly.
