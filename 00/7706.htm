hansv mvp say:

1) Use Excel to create a list of dates in a column, say all dates from 01/01/2015 to 12/31/2024. You can use the Fill > Series feature of Excel for this.

2) In your database, create a table tblDates with a single Date/Time field MyDate.

3) On the Create tab of the ribbon, click Query Design.

Add the MyDate field to the query grid.

Set the sort order of this field to Ascending.

Enter the following condition in the Criteria row:

Between DateSerial(Year(Date())+(Month(Date())<5),12,1) And DateSerial(Year(Date())-(Month(Date())>4),4,30)

In this context, however, you can very easily populate an unbound combo or list box's list with code in the form's Open event procedure:

    Dim dtmDate As Date
    
    dtmDate = VBA.Date
    
    With Me.lstFutureDates
        .RowSourceType = "Value List"
        .AllowValueListEdits = False
        Do Until Format(dtmDate, "mmdd") = "0531"
            dtmDate = dtmDate + 1
            .AddItem Format(dtmDate, "Short date")

You can show the dates in whatever format you wish by amending the .AddItem Format(dtmDate, "Short date") line.  To do this in any form it can easily be made more generic by wrapping the code (suitably amended) in a function in a standard module, with arguments which reference the control object, and the final date for the list in a the format 'mmdd'.  The function could them be called in a form's Open event procedure

What you should be doing is automating Word and then create a new document from the Template that would normally be located in the user's Templates folder.

Dim appWord as Object

Dim docWord as Object

Dim bstartapp as Boolean

On Error Resume Next

Set appWord = GetObject(, "Word.Application")

If Err Then

    bstartApp = True

    Set appWord = CreateObject("Word.Application")

End If

On Error GoTo 0

Set docWord = appWord.Documents.Add("TemplateFullName")With docWord

     'Do your stuff

    .SaveAs

    .Close

End With
Set docWord = Nothing
If bstartApp = True Then
    appWord.Quit
End If
Set appWord = Nothing

All of the code could be in Excel so that the template would not need to be a macro enabled template and could be a .dotx

Sub Create_WordTemplate()

         XXCHECKPROG.Show                                                         ‘initiates progress bar

         XXCHECKPROG.Label2.Width = 0

         XXCHECKPROG.Label3.Caption = 0

         XXCHECKPROG.Repaint

         Dim objWord As Word.Document

         Dim objTemp As Object

         Dim FlName As String

         Const cStrDir As String = "C:\Users\Public\Documents\"

         FlName = cStrDir & "RMA RR TempRep.dotx"             ‘defines target directory for copy template

         ThisWorkbook.Sheets("Templates").Visible = xlSheetVisible

         ThisWorkbook.Sheets("Templates").Select                 ‘defines source of embedded template

         Set objTemp = Sheets("Templates").OLEObjects("Template01")

         objTemp.Activate                                                               ‘activates embedded template

         Set objWord = objTemp.Object                                       ‘instructs Word to grab embedded template

         XXCHECKPROG.Label2.Width = 5                                     ‘advances progress bar

         XXCHECKPROG.Label3.Caption = 3

         XXCHECKPROG.Repaint

         objWord.SaveAs Filename:=FlName                               ‘saves copy of embedded template as new

         objWord.Close SaveChanges = False                               ‘template to target directory

         Set objWord = Nothing

         Set objTemp = Nothing

         Set objOLE = Nothing

         XXCHECKPROG.Label2.Width = 10                                    ‘advances progress bar

         XXCHECKPROG.Label3.Caption = 8

         XXCHECKPROG.Repaint

End Sub

daniel pineault say:

If the database was opened in 2016, then the references get switched over to 2016 references.  Then if you try to run it in 2013, it can't locate the references (it can't downgrade them automatically like it upgrades them) and then you get weird problems and errors.

So knowing this, if this is the case, then you need to open the db in 2013 (full version) and fix the references.  Then distribute it.  It will work on both 2013 and 2016.

All work should always be done in the oldest version that will be used to run the database.
