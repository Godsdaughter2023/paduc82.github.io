This indicates that a kernel-mode driver attempted to access pageable memory at a process IRQL that was too high.

A driver tried to access an address that is pageable (or that is completely invalid) while the IRQL was too high. This bug check is usually caused by drivers that have used improper addresses.

By default, the fault of the crashes is NETIO.sys which is the Network I/O Subsystem. This is not the true cause of the crash, and usually when we have network related crashes like this, it's caused by one of two things:

1. Network drivers themselves need to be updated.

2. 3rd party antivirus or firewall software causing NETBIOS conflicts.

    I was able to resolve the issue. I found that, windows update has corrupted the registry. Following are the steps ->

    1. Press F8 before windows boot. Go into repair windows mode.

    2. You will be asked for username/password.

    3. Open command prompt

    4. Registry backup are stored in C:\Windows\System32\Config\Regback (C: - Windows installation drive)

    5. Take the backup of existing files in C:\Windows\System32\Config\ (system/software/security/default)

    6. Replace these files with corresponding files from C:\Windows\System32\Config\Regback

    After this point when I tried rebooting, I didn't get any BSOD but windows started "configuring updates" and as a part of configuration it restarted and same problem of BSOD appeared. (Yes there is a buggy windows update !) 

    Then I repeated the above steps plus executed following command ->   

    dism /image:C:\ /cleanup-image /revertpendingactions (again C:\ in windows installation drive) (Windows Repair is Windows Pre-installation (PE) environment and hence /online option of dism doesn't work here)

    Execute the above command (dism) from x:/ prompt only. I didn't work for me when I was in c:/

    This command would cancel pending updates in windows (which caused the issue after first boot)

    After reboot, it would again show  "configuring windows" message, but then it displays "Failed to Apply update, reverting". This step takes good amount of time ~5-10 min.  

    Other steps I have performed. Not sure if they have any bearing. Just sharing -> 

    How to remove windows patches from windows PE environment -> 

    Following steps are executed from command prompt in windows repair mode. 

    First search of the packages ->

    dism /image:c:\ /get-packages /Format:table 

    This gives in tabular form. But NOT sorted on installation time. If you know date of patch installation then you can filter the results. 

    dism /image:c:\ /get-packages /Format:table | Find "DATE OF INSTALLATION"

    You would get list of patches installed on that date. Name of the packages would start from "Package_for_"

    Use following command to delete the package ->

    dism /image:c:\/remove-package /PackageName <Package_for_........> (SPECIFY complete name here)

    /PackageName can be specified multiple times in single command to remove multiple packages. 

    dism /image:c:\ /remove-package /PackageName <Package_for_........> /PackageName <Package_for_........> .... 

     Tip -> You can open notepad for text manipulation and prepare the command.
