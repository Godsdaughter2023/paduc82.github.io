dedawson say: Still unable to use VBA to control block/unblock  'automatic picture  download', but have solved the immediate problem.

During code development, I had the following snippet

With oNewItem
    .To = "*** Email address is removed for privacy ***"
    .Display
    .Send
  End With

This let me step through and see the contents of the message I was generating without having to send it.  It was this .Display that was triggering the 'Do you want to download pictures' dialog that brought everything to a screeching halt if the message in question had external pictures.

So, one little comment later, and all is well.

  With oNewItem
    .To = "*** Email address is removed for privacy ***"
'    .Display ' don't need to display it; if it has external pictures, it will hang waiting for OK box click
    .Send
  End With

benoit barabe say:

Normally this happens in this scenario:

1) You print to OneNote a multi-page document
2) In OneNote, each page of the document are a separate object
3) You delete some, but not all, "pages" of the printed document. This is all within the same OneNote page.

The truth is that we keep the whole printout under the cover, so by deleting, you really just hide them. The message is just about letting your know that fact. You do not need to take action, unless you intend to share the OneNote page with someone and you don't want them to potentially extract the other "pages" of the printout.

Sorry this message is overly scary and confusing... Maybe it should read "Some of the printed images in this OneNote page are currently hidden. To remove all traces of the printed document, please delete all printed images contained within this OneNote page". Well, maybe that's still bad, but I understand it better that way.

Hope this helps...
