pamcaswell say:

Although it may seem like  the normal paragraph style is causing your problems, the more likely causes are the  defaults in your template and the pasting behavior of Word. ( See   

When a document is created based on a template other than normal.dot(m), normal.dot(m) does not contribute styles to that document.  Also, templates contribute styles information  to documents only when the docs are created and whenever the Templates and Add-ons dialog's checkbox before "Automatically update document styles" is checked.  That box is clear by default in W2007/10.  

To eliminate changes that may result from differences in the normal paragraph styles and the defaults, go to the template's Styles pane > Manage Styles > Set defaults tab, and change the default font, paragraph spacing, and line spacing to your liking.   

Eliminating changes caused by pasting is more difficult because it depends not only on Word's pasting behavior but also on how the users paste.

One way to get cleaner copy is to encourage users to  use the paste option "Keep text only".  The  paragraph style for the entire passage will be the style in effect at the cursor.   The drawback to this pasting method is that they could lose content formatting such as  superscripts, bullet lists and italics used to denote titles and such. Still, I find that I'm able to use the paste text option most of the time.

Another way is to make it easy for users to apply the correct style.  Change the  styles gallery to make the styles you want them to use more obviously available to them.  The styles gallery in the template will be carried into  new documents based on it.

A global template makes macros, key assignments, and Building Blocks available to all documents. It doesn't affect the styles in other documents (except that it supplies the initial style definitions to documents created from the template).

If you examine the formatting in the user's document, you'd probably find that Times New Roman has been added as direct formatting on top of the style. That would explain why the font is retained when it is copied into another file.
