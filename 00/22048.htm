What Supported Attributes can be used for Azure AD Connect? - Microsoft Community

lourh say:

What Supported Attributes can be used for Azure AD Connect?

I'm working to migrate mailboxes to Office 365 from a multiforest environment where the ObjectGuid cannot be a source Anchor for Azure AD Connect. I'm asking if there is a list of attributes supported by Azure AD connect that will cover all types of Objects (Mailbox, group, sharedMailbox,room, contact,..). Else what recommeended attribute can replace ObjectGUID.

Thanks

neo yu msft say:

Hi lourhma,

In a multi-forest environment with AAD Connect deployed, An example of an attribute that is commonly used to satisfy this requirement is the Employee ID(you can specify the value in the attribute like extensionAttribute1-15,which are available in all the objects). Since an employee will generally retain the same Employee ID throughout their employment with the company (and thus the duration of their presence in the company’s Azure AD tenant).

To avoid any duplication or ambiguity it is usually not recommended to use the Active Directory objectGUID as the source anchor since this value would change if the user is moved between forests or domains within a forest. Instead, a design decision needs to be made in the multi-forest environment to choose an alternate source attribute to determine the sourceAnchor value for the user.

This attribute should be:

  • Unique to that user, across all on-premises forests
  • Very unlikely to change during the lifetime of the on-premises user

In the case where an user/object is moved across forests, it is critical that this attribute value be preserved.

Also, when selecting the Unique Identifier to use, ensure that this identifier does not contain the ‘@’ symbol. The ‘@’ symbol is a reserved character within the Azure Active Directory platform and MUST NOT be used for the Unique Identifier.
