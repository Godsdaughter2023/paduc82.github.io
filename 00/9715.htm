john mcghie say:
Hi Jez:

The most likely scenario is that you have not found all of the section breaks.

Instead of looking and clicking, go into the header or footer and use the "Next Section" button to go to the next section break.

There is no big secret to this: the Page number control is in the section breaks, and the one controlling the page number you can see is the one that occurs above it in the document.

It is "possible" that the document is badly corrupted.  This is extremely rare and thus unlikely.  If that happens, the Master Section Break goes read-only and you can't update it.  The master section break controls all of the document settings.  It is hidden below the last paragraph mark in the document, you cannot see it or edit it.

To fix this, create a new blank document and save it with a new file name.

CAREFULLY copy all EXCEPT the last paragraph mark from the old document and paste into the new document.  That leaves the old section break in the old ,document and replaces it with a new uncorrupt one.

Be careful not to copy the last paragraph mark: if you do, you copy the problem.

onedrive crashes on windows 10 within a few seconds - Microsoft Community

aman naser berhanu say:

onedrive crashes on windows 10 within a few seconds

The problem

I recently had a problem with oneDrive for business. After resetting my windows 10, I started to configure oneDrive using default setup parameters. It would start synchronizing the folders and immediately crash after the folders had been replicated on the computer. I tried all possible solutions from the internet but none of them worked. 

Below is the workaround or solution that finally got it to work. 

The solution

When OneDrive is configured, it creates a folder by default under the user profile "c:\users\username\oneDrive-folder". For some reason the default location or any default directory under the user profiles contradicts with OneDrive and it crashes. In order to avoid this, create any subfolder under the user profiled directory or its subdirectories and then during the configuration screen select the folder which you created as the OneDrive folder. The application will work without any problems. I hope it works for you.

In my case, below are the solutions that didn't work and the one's that worked.

non-working situation. One-drive under the following folders.

"c:\OneDrive-folder"

"c:\users\username\OneDrive-folder" and any default subfolder such as desktop, documents, etc...

"c:\users\username\desktop\OneDrive-folder"

Working situation. Create any new folder under the following folders.

"c:\users\username\newfolder\OneDrive-folder"

"c:\users\username\desktop\newfolder\OneDrive-folder"

Its a bit unorthodox but it worked for me.

I had to ask my fellow MVPs, and there actually is a simple way if you know where it is. One of them wrote:

This was a new feature added for datasheets in 2007. When you have a datasheet open in Datasheet view, click the Home tab and then click the Totals button in the Records group. Internet Explorer stores certain info about the sites you visit on your PC in case there's a crash, hang, or unexpected shutdown.
