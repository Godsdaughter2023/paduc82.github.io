Hi,

1) You can get most builds from this link if you browse... It goes directly to the 1703 build from early 2017...  http://windowsiso.net/windows-10-iso/windows-10-creators-update-1703-download-build-15063/

This is a 3rd party site, but the links redirect to a Microsoft server...

2) Sometimes when you select the ISO in RUFUS the GPT\MBR type changes for some reason, and needs to be reset... Just something to look for...

3) The difference between UEFI and Legacy Bios is primarily that UEFI Bios machines boot to "WindowsBoot Loader" or something similar, that is located on a specific EFI partition...  Legacy Bios boots internally into windows to a boot file...

There are a number of advantages to UEFI Bios and GPT drives...

GPT drives can have a almost unlimited number of partitions... MBR has a max of four..

Legacy Bios was\is getting loaded down with countless settings type issues that are somewhat obsolete... UEFI basically eliminates a lot of that, and streamlines the process, speeding up the boot at the POST screen..

4) It wouldn't hurt to boot through the Boot Options with ESC, F9, F11, F12, whatever your machine uses... And specifically select a UEFI boot device...

I doubt that has anything to do with your problem... You wouldn't get as far as you do if that were the case..

5) It should take nowhere near that long...

6) It might be that you just lucked out with the 32 bit and had everything just right..

I don't know how you are creating your partitions, but UEFI partitioning is finicky...

If you wanted to save a partition at the end of the drive, that would be OK...

Create a partition at the front to take up the space you want to save for windows, then format the one at the end for the "Data"

Then delete the one in front... leave it unallocated... Windows will do what it needs to do... It would be better to not have the "Data" partition at the end, but it should work... That is how I do my boot SSD...

The problem with that is with a UEFI machine, it will take four partition to install windows, and their placement is kind of picky..

There will be a 100-200MB FAT32 partition for boot... it will be probably named "EFI"..

There will be a 500MB or so "Recovery" partition...

There will be the OS partition...

The BIG difference with the GPT type drive is that there will also be a small 16-120MB Hidden Partition... It should appear when using a Custom Install, but will not show up in Disk Management.. It is basically the directory for the GPT partitioning..

The placement of the partitions is somewhat important... That is why I suggest that you install onto a unformatted drive, and let windows do it's thing..

If your problem is not related to a bad ISO, the only other thing I might think is that the machine is having problems determining where to copy the files to...

BTW... The Fall Creator's Update does in fact have some issues with the included WinRe, or windows tools... That may also be part of the problem..

The problem with using the earlier 1703 Creator's Update is that in short order your machine will try to update to it... Then there is another new build coming in a couple of weeks..
