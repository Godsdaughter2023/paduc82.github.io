keratl say:

A few random troubleshooting ideas;

break out the components of your formula for a row where you think you should get a positive result. If you think Row 17 should be a match, then the test formulas might look like:

 =Data3!$B17=$C94

 =Data3!$C17=$C$89

do both return a true value? If not, at least you've narrowed it down and know which part of the formula is failing. My first guess is that you might have leading or lagging spaces in the text, or an uppercase/lowercase/sentence case issue.

Also, go to the bottom right of your data set, look at the Excel scrollbars. Do they indicate that you are already at the bottom right of the used data area, or do they look like they are ready to scroll even further? Depending on how the data was brought in, Excel might think that you are using much more of the sheet, so when you use entire columns ( Data3!$F:$F) it may be calculating a million (mostly blank) rows, not just your 160 rows. If this is the case, you can either delimit your formula to the largest you'll ever need (Data3!$F1:$F200) or manually select the excess rows, delete, select the excess columns, delete, then save the file to reset the used area flag (then do your formulas)

deathbaron2 say:

OneNote on desktop (like Sticky Notes)

I tried to achive something similar to sticky notes with OneNote. Keeping it as a small note on the side surviving restarts and usable in offline mode. The quicknote window was good enough for me, but it lost the compact size after the restart.

After reading the office command line documentation, I came up with the following solution. OneNote.exe has a sidenote argument for compact size and hyperlink to open a specific page/section. To get the hyperlink, right click the page tab of the page you want to link. Select Copy link to page. Paste it somewhere, you need the one starting with "onenote:". This scheme also works offline for me, syncing notes when I get online.

Example:

C:\Program Files (x86)\Microsoft Office\root\Office16\ONENOTE.EXE" /sidenote /hyperlink "onenote:https://d.docs.live.net/2XXXXXXXXa/Documents/DXXXXXXXXXe/Sticky%20Notes.one#section-id={1XXXXXXX-XXXX-XXXX-XXXXXXXXXXX6}&end"

Just put a link in your startup folder or a run key in registry.

Hi Eric,

Please check whether the default language is set to English or German. For this, on the Tools menu, click Language, check which language is selected. Once you made this change, click Review and select Spelling and Grammar, check if it still changes to German.

 If the issue persists, please provide the following information:


  1. Please confirm the version and build of your Word application. To get this information, open Word, go to the Word menu and choose About Word.
  2. Your OS version.
  3. Please check another Word doc created in English to confirm if the issue is with a specific document or all the documents.
  4. Open any PowerPoint document and spell check, see if same thing happens.
