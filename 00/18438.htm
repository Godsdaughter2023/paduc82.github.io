john sr wilson say:
If the broken layouts are content type (7) and if replacing them works (here the text size changes) you MIGHT be able to use a macro like this. Definitely work on a copy!!!

Sub fixPlaceHolders()
Dim ocust As CustomLayout
Dim oplc As Shape
Dim oDes As Design
Dim sngH As Single
Dim sngW As Single
Dim sngL As Single
Dim sngT As Single
Dim lngType As Long
Dim i As Integer
For Each oDes In ActivePresentation.Designs
For Each ocust In oDes.SlideMaster.CustomLayouts
For i = ocust.Shapes.Count To 1 Step -1
Set oplc = ocust.Shapes(i)
If oplc.Type = msoPlaceholder Then
lngType = oplc.PlaceholderFormat.Type
Select Case lngType
Case 7
With oplc
sngH = .Height
sngW = .Width
sngT = .Top
sngL = .Left
oplc.Delete
ocust.Shapes.AddPlaceholder lngType, sngL, sngT, sngW, sngH
End With
Case Else
'do nothing
End Select
End If
Next i
Next ocust
Next oDes
End Sub

Verify that you have removed all other realtime antimalware products that were ever installed on your PC, including free/trial product that were installed when the PC was purchased (Norton, McAfee, TM, AVG, etc).   Then, use the removal/cleanup tools here:

List of anti-malware program cleanup/uninstall tools

Verify that Windows Defender is uninstalled (if XP), or Disabled (if Vista/W7).

Verify that your PC clock is correctly set.

Verify that you are using build 1961 of MSE (as of April 2010).

scottgem say:

If your front end is C# then you should not need the Access Runtime. The Access runtime is for when you need to open the Access file but you aren't, your C# program is connecting to it.

There are several installer programs that will package your C# executable and the data files into an installation file.

However, I would question the use of Access as your datastore. The ACE database engine that comes with Access is the weakest component of the Access toolset. If you are writing your front end in C# I would suggest using SQL Express, MySQL or some other free SQL database instead.
