datanerd say:

I had the same problem. The problem is not with Office itself but with the Microsoft AutoUpdate utility. There is a bug with v3.2.0 (and possibly others, though I have not tested them) where Microsoft AutoUpdate is not able to see newer versions of Office programs because it is too old. That bug also keeps Microsoft AutoUpdate from updating itself to a newer version without that bug.

To fix the problem, download and install Microsoft AutoUpdate v3.4.0 (https://support.microsoft.com/en-us/kb/3133674), then check for updates again. Microsoft AutoUpdate should now find newer versions to install.

I will speculate that on a Mac the data that is contained in the ListGal.dat file on a PC is most likely contained within what is called a pList file... but which one is the question?

However, have you tried just renaming your Normal.dotm file and letting Word generate a new one. It seems to me that if your default list style is corrupted then allowing Word to rebuild the Normal template, which is where the list style is contained, should in theory correct the problem.

To do this shutdown all Office applications and if in doubt I recommend restarting your computer. With absolutely nothing else running open Finder and go to this path location:

~/Library/Application Support/Microsoft/Office/User Templates

In this directory locate the Normat.dotm file and rename it to oldNorma.dotm and then start Word and test the bulleted list function.

If you don't see the Library folder in your Home directory, select Finder > Go and then hold down the Option Key... the Library folder will then display and you can double click it and navigate to the path listed above.

Hope this helps
