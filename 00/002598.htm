aprilzhou say:

Hi Domcharlie,

Before the transfer, please note that we can't transfer an Office 365 domain to another registrar until 60 days after you registered it with Office 365. Reference: Transfer your Office 365 domain to another registrar support.office.com/.../Transfer-your-Office-365-domain-to-another-registrar-951c0268-a8de-4215-806c-5569cde1bd78

As for the email downtime, we may consider the following situations:

1. If GoDaddy allows us to create DNS records for Office 365 Exchange Online before we point the Name Server records to it, we can firstly create all the necessary DNS records at the new DNS hosting provider, and then change the Name Server records to point to it. In this scenario, the downtime of the email service depends on how long it takes for the Name Server records and the DNS records for Exchange Online to take effect. Ideally, the downtime of the email service might only last from a few seconds to several minutes.

2. If it doesn’t allow us to create DNS records for Office 365 Exchange Online before we point the Name Server records to it, we need to create the necessary DNS records after changing the Name Server records. Then the downtime will last for a while until we complete the DNS records set-up process. It will depend on how long it takes for the Name Server records to take effect, and the newly created DNS records for Exchange Online to take effect. The DNS records propagating throughout the whole Internet can take 15 minutes to 72 hours.  

As a result, you may choose the option 1 if it’s possible, and transfer the domain during off-peck hours of email service.
