Outlook 2010 reading pane option - want to permanently adjust the zoom setting.

Greetings, I know that there are a number of ways to adjust the zoom on the reading pane in outlook 2010. But how do i make the change permanent so that it does not go back to 100%.

There are two ways to temporarily increase or decrease the zoom level in the Microsoft OutlookReading Pane(Reading Pane: A window in Outlook where you can preview an item without opening it. To display the item in the Reading Pane, click the item.).

Do one of the following:

  • Use the Zoom control at the bottom of the Outlook window.

  • If you are using a mouse with a wheel control, click in the Reading Pane, press CTRL, and roll the scroll wheel. Rolling the wheel away from you makes the contents larger, rolling the wheel toward you makes the contents smaller.

 Note    The zoom affects only the current item that is previewed. When you select another item or folder, the Reading Pane returns to the default 100% size.

jim gordon mvp say:

The reason why you see a difference when importing from ODBC SQL Server and importing from text is that when you import using ODBC from SQL Server, Excel automatically converts the imported data from a Range object into an Excel table.

When you import using the wizard, the conversion to an Excel table is not automatic. The data is displayed as a Range object.

You need take only one additional step when importing from a text file. After the data imports, go to the Insert tab of the Ribbon and click the Table button. That's all you need to do.

I need to manage multiple projects in MS Project. There is no Master - Microsoft Community

tonyatrytten say:

I need to manage multiple projects in MS Project. There is no Master Project. I don't want a Master Project.

How can I do this?  I have multiple same-priority projects to manage using the same resources and I DON'T want a master project.

john - project say:
Tonya,
You don't need a master project to do this although I'd be interested to know why you don't want to create one.

Your best approach is to create a resource pool file, which is nothing more than a project file with no tasks but all the resources shown on the Resource Sheet. Each of the individual project files is then set up to use the resources from the pool (Resource/Assignments group/Resource pool icon/Share resources). It is very similar to a master/subproject structure in that it is a linked structure.

By the way, which version of Project are you using and is it updated with the latest service pack?

John

rod gill say: Don't spend too much time looking to Project (or any scheduling tool) for the answer. You need senior management to prioritize the projects. Then you need a resource manager who takes the project priorities and assigns resources accordingly. If all projects genuinely have the same priority then put them in a queue and do one at a time.

Don't try to run more than one or two at the same time. Most efficient is one only. Multi-tasking can add up to 30% more work. Only run them in parallel if you have to to utilize your resources.

Good luck!
