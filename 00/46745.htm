Windows actually has a maximum path size of 260 characters, this is determined by a windows API function MAX_PATH. Of the 260 characters this also includes the [Drive Letter][Colon][Backslash]insert rest of path here[backslash] for example everything in bold comes out of the 260 characters C:\insert path here\ which only leaves you with 256 characters left for your path. Now I know none of this answers your questions, but I thought you would be interested in a little background.

There is a work around I can provide which would help you here. Create a drive mapping of the folder where you are trying to back up all of your data too.  But for this to happen the folder needs to be marked as a Shared Drive, this can be done by going to your backup folder -> right click -> properties -> sharing -> Share -> set up to only share with your useraccount -> You should now get a message "Your folder is shared" click OK -> now if you have a look above the Share button you will see a network path eg. \\MyComputer\FolderIjustShared copy this path.

Open up My Computer -> Click on Map Network drive -> Paste the previous address into this path. Now when you want to copy and paste things into you backup folder please use this drive mapping.

Consider the following scenario:

  • You have a computer that is running Windows 8.1, Windows Server 2012 R2, Windows 8, Windows Server 2012, Windows 7 Service Pack 1 (SP1), or Windows Server 2008 R2 Service Pack 1 (SP1).
  • You use Windows Explorer to try to copy files or folders in order to paste them into some other folder.
  • The files or folders that you copy have paths that exceed the maximum allowable path length.


In this scenario, the behavior of the copy operation is unreliable and fails because of the length of the file or folder path. Additionally, you may experience one of the following symptoms:

  • The copy operation fails and generates a message that states that the path (file name) is too long. The message also suggests that you shorten the file name and try to copy again.
  • The copy operation does not start. In this case, no message is generated.
  • The copy operation starts, copies a few files or folders, and then fails without generating a message.


This problem may prevent some files or folders from being copied. You may not realize the the problem occurred if you do not receive an error message. Different symptoms may occur, depending on how the files are selected for copying.

Note This problem can also occur when you try to copy files or folders from Volume Shadow Copy Service snapshots if the file or folder length in the snapshot exceeds the maximum path length.

Windows Key+R > Type services.msc, then press Enter > Right click Wlan AutoConfig > Properties > Select Automatic in Startup type, then click Apply, and click OK > Reboot your computer

4: Restart your devices
Switch off your computer, modem & router > Unplug your wireless router and modem from power source (remove the battery if your modem has a battery backup) > Wait for sometime > Plug your wireless router and modem back into power source again and restart (put the battery back to the modem) > Reboot your computer

Windows Key+X (Or right click on Start) > Click Device Manager > Expand Network Adapter > Right click on your network adapter > Click Update Driver Software > Click Browse my computer for driver software > Click Let my pick from a list of device drivers on my computer

lemp say:

Where can get a Product Key for Windows 7

You can't.

Windows Product Keys only come with a purchase of Windows, and a legitimate copy of Windows 7 is extremely hard to find these days (and correspondingly expensive).

If you have an OEM version of Windows 7 (Windows 7 came pre-installed on the computer when you bought it), the Product Key is on the COA sticker.  On laptops, this will be on the bottom of the case or perhaps inside the battery compartment, under the battery.  On desktop/tower computers, the sticker may be on the side or back of the case.

If you purchased a retail copy or OEM System Builder copy of Windows 7 and your computer is currently functional, you can recover the Product Key using Nirsoft ProduKey (https://www.nirsoft.net/utils/product_cd_key_viewer.html),  MagicalJellyBean KeyFinder (https://www.magicaljellybean.com/keyfinder/), or BelArc Advisor (https://www.belarc.com/products_belarc_advisor). 

Both ProduKey and KeyFinder can be used to recover Product Keys from non-functioning Windows systems if the disk itself is intact, by connecting the disk to a working Windows computer.  I don't believe BelArc Advisor is capable of this.

None of the keyfinder utilities will recover a useful key from an OEM installation.
