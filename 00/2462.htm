bob jones aka: cybertaz say:

Ostensibly, a Fill-In... Field [under Rules on the Mailings tab] prompts for variable input on a record-by-record basis as the merge is being done.

In Word 2016, though, I've found it to be a bit flaky because each document does not display when the prompt appears. IOW, unless you "plan ahead" based on what is to be entered for whom you have to guess which record the variable content is being entered into the prompt for... The final results are not displayed until the merge is completed.

Even if that weren't the case, IMO it slows down the process because the prompt has to be either completed or dismissed for every recipient.

Assuming I correctly understand what you want to do I'd find it easier as well as significantly more efficient to use one of the following approaches:

  • Add a field to the record source that includes the variable information for each recipient, then include that field on a Subject line in the Main Document. The filed in the source could be deleted afterward if necessary, or
  • Include a blank Subject line in the Main Document, output the merge to a new document, then edit each copy to add the variable information.

Collaborative Powerpoint access offline - Microsoft Community

matthewhouser2 say:

Collaborative Powerpoint access offline

If I share a PowerPoint presentation with associates for collaboration, how do they access / present them when they are offline?


john korchok say:

If you're sharing using a cloud service or Sharepoint, they would have to download the file before going off-line.

If you're using PowerPoint Online for collaboration, they can choose FILE>Download As and download a copy to their computer before going offline.

If you're sharing by email, they can download the attachment and go off-line.

With all of these, any changes made to downloaded files would not be part of your collaboration, because there is no easy way to sync the local file with the original version.
