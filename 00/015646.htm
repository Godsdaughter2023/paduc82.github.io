    Any other suggestions about what might be the cause/solution to this issue?

    kathleen orland say:

    If you open Outlook in safe mode, it disables all add ins. If Outlook then works properly, you know one is at fault. What you then do is open Outlook normally, disable all of the add ins and restart Outlook. Add them back one at a time, testing each one until you find the culprit.

     

    If your user can't receive, the send/receive progress window has an error tab. Watch the send/receive progress and click on the error tab for an error message. That specific message will tell you exactly what's wrong. If you don't understand it, post it here. We can decipher it for you.

     

    In any POP3 account, the account holder's emails/calendar/contacts, etc. are all contained in the PST file. (personal folders). Al you have to do when you recreate a mail account is point the new account to the PST file.

     

    Since the PST file is kept on the C drive and likely not backed up, it is advisable to copy it to the user's Home drive on the network on a regular basis. This must be done when Outlook is closed and it's as simple as either doing it manually or even writing a little .bat file to do it for the user so all they have to do after closing Outlook is to double-click the bat file and let the script do the rest. I've written a script for my users and put a shortcut to it on their desktops with the MSN butterfly icon (because it's eye-catching). Whether or not they use it is their choice of course, but in the event of data or hardware loss they get a big, fat "oh well" from me because I've given them the means to back the data up.
