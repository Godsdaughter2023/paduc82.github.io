kergga say:

While this can be done in VBA, I think you can get your desired result using a few formulas.

 

Let's say that your other worksheets are all named based on their CstCtr, e.g. you have a sheet named 501, another named 502, etc.

 

For the first empty Jan cell in your new sheet, you want to get the ACCT=1, Jan value from sheet 501.

 

Let's start with a formula to grab just that value; (aircode)

=sumif('501!$A:$A,$A2,'501!A:A)

if my syntax is correct, that should return the sum of any values on the 501 sheet for account 1

 

Once you have that working, now you just need to make it dynamically pick the correct source sheet, using an INDIRECT formula. Indirect allows you to use text strings that Excel is able to recognize as a reference:

A1 = Indirect ("A1") = Indirect("A" & "1") = Indirect("A" & (3-2))

those will all get you to the same place.

 

So, we want to replace the "501" with a formula to select the right sheet, let's start the process of converting:

=sumif(indirect("'501!$A:$A",$A2,indirect("'501!A:A"))

now we can separate out the sheetname:

=sumif(indirect("'" & "501" & "!$A:$A",$A2,indirect("'" & "501" & "!A:A"))

then we can replace the sheetname with whatever is in column B in our same row:

=sumif(indirect("'" & $B1 & "!$A:$A",$A2,indirect("'" & B1 & "!A:A"))

 

at this point, you should be able to drag the formula down and across (using the cell 'handlebar') and have the formula populate the rest of your sheet.

 

Hope that helps!

Sub DeleteUnusedBlankCellOutsideOfPrintArea()

''' ========================================================
''' PLEASE READ THIS INFORMATION REGARDING THE USE OF THE
'''     MACRO INSTALLED IN THE 'PERSONAL.XLSB 's module.
''' ========================================================

'''     This Macro is designed to remove what appears to be blank cells
'''     below and to the right hand side of data in an Excel Worksheet.
'''     These 'blank cells' can be caused through the deletion & transfering
'''     of data to new workbook or into the 'Recycle Bin'. Excel keeps
'''     those cells as part of the worksheet & due to Version 2007 and later
'''     having one million rows & more columns these extra unneeded cells
'''     can add up to a large number of KB's.
'''     i.e. a last cell in row 245000(using Control+End keys) and a file size
'''     of 2034 kb became a 3000 row of data & a 239 kb file size.

'''
'''     The macro finds the last cell containing a letter or number, this means
'''      if you have made the right most Column a coloured line border for
'''     the last row, the macro will removed both.
'''     To overcome this drawback enter the words 'Keep Clear' in a cell two
'''     rows below and two columns outside of print area.
''      Use this macro with very very extreme caution if you value your data.
''' ========================================================
'''  START ASK USER QUESTION.
        Msg = "DO YOU WISH TO RUN THE MACRO CALLED"
'''  The 2 chr$(13) in lines below creates a empty row before next line of text
        Msg = Msg & Chr$(13) & Chr$(13) & " DELETE-UNUSED-BLANK-CELLS-OUTSIDE-THE-PRINT-AREA?"

        Msg = Msg & Chr$(13) & Chr$(13) & "Affirmative click 'OK' else 'CANCEL' "
            Style = vbOKCancel + vbCritical + vbDefaultButton2
                Title = "WARNING!!"
                    userAnswer = MsgBox(Msg, Style, Title)
            If userAnswer = vbCancel Then
        Exit Sub
    Else

'    Application.ScreenUpdating = False
    End If

Dim myLastRow As Long
Dim myLastCol As Long
Dim wks As Worksheet
Dim dummyRng As Range

For Each wks In ActiveWorkbook.Worksheets
    With wks
        myLastRow = 0
        myLastCol = 0
    Set dummyRng = .UsedRange
    On Error Resume Next
        myLastRow = _
        .Cells.Find("*", After:=.Cells(1), _
        LookIn:=xlFormulas, LookAt:=xlWhole, _
        SearchDirection:=xlPrevious, _
        SearchOrder:=xlByRows).Row
        myLastCol = _
        .Cells.Find("*", After:=.Cells(1), _
        LookIn:=xlFormulas, LookAt:=xlWhole, _
        SearchDirection:=xlPrevious, _
        SearchOrder:=xlByColumns).Column
    On Error GoTo 0

    If myLastRow * myLastCol = 0 Then
    .Columns.Delete
    Else
    .Range(.Cells(myLastRow + 1, 1), .Cells(.Rows.Count, 1)).EntireRow.Delete
    .Range(.Cells(1, myLastCol + 1), .Cells(1, .Columns.Count)).EntireColumn.Delete
    End If
    End With
    Next wks

End Sub

rich007a say:

Quick and dirty way to do this is:

Put this in row 1 in any column:

    ="#RSLINX|Balancer!'Data_U_PCN.CW_Angle[" & ROW()-1 & "],L75,C1'"

(yes, that is a "#" at the start, not an "="!)

Copy that cell. 

Hit F5.  Type:

  D1:D10000

(or whichever column you are in) and click OK.

Paste.

Select the column and Copy.

PasteSpecial As Values.

With the same column selected, hit Ctrl+H.  Replace # with =

 

Or we could stick this in a macro.  Depends on which bits of your formula change each time you have to do this.

Hope that helps.

Cheers
Rich

jeeped say:
You will need a three step process.
  1. First, put this formula into a blank helper column to the right
                  ="=RSLINX|Balancer!'Data_U_PCN.CW_Angle["&ROWS($1:1)-1&"],L75,C1'"
    ... and fill down as necessary.
  2. Next, copy that entire column and paste it into the original column using Paste Special, Values (yes... that is Values not Formulas).
  3. Finally, select all of the original column and select Text to Columns from the Data tab. When the dialog open immediately click Finish.

The end result should be a column of active formulas with sequential numbering from zero to whatever.
