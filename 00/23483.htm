ossiemac say:

The formula is more complex than the previous 2 replies but does NOT need a helper column.

The Match function is used to find the next row containing Environment. Note that it uses a wild card asterisk for the numeric suffix on Environment.

  • Copy the formula to cell B2.
  • Edit $A$13 to the last cell of your data in column A. Can be well below your actual data to allow for expansion of the data. (This Reference is Bolded in the example).
  • Copy the formula to all of the required cells.

=IF(COUNTIFS(Input!$C:$C,Output!B$1,Input!$B:$B,Output!$A2,Input!$A:$A,INDIRECT("Output!$A$"& MATCH("Environment*",INDIRECT("Output!$A"&ROW()&":$A$13"),0)+ROW()-1))>0,"P","")

willy denoyette say:

You should not take the time of the whole preparation phase to calculate the download data size.

The whole preparation time (say 20' total) consists of:

1) download the files needed to build the initial $BT folder contents. < 100MB of data.

2) local and remote processing & transfer of the actual state of your system (Windows + System32 + Program files + ...). This consists of a small chunks of data passed to/from the flighting server(s).

3) Download of the differential data as prepared by the flighting server based on the data from 2)

4) reconstructing the complete system state (differential data + current local state), this is what you'll find in the download subfolder with a long hex name.

 

2) and 4) take ~12' - 15' depending on your system speed (CPU and Disk) and your current system activity *.

3) contains the bulk of the data transfer. The time this takes depends on the size of the diff data and the speed of your internet connection.

For the latest builds I have measured a DL time of 210 sec. (2'30) on a ~150Mbps link.

So, on a 20Mbps link this can take up to 1500 sec. (25').

Notes:

- the link speed may vary, never consider this to be fixed, the internet is a global network!

- 2) above may vary with the load on the flighting server(s).
