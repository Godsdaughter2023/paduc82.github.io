    Date Range Query - Microsoft Community

    dudley james say:

    Date Range Query

    The following query sitll ask me for the parameters even though I have entered the Paramater into a Form. What am I doing wrong?

    What I want is for the query to read the paramater that was entered into the form.

     

    Thank you.

     

    JD

     

    PARAMETERS tbl_Attendance.[Wdate>=tbl_DateRange].SDate DateTime, tbl_Attendance.[WDate<=tbl_DateRange].Edate DateTime;
    SELECT tbl_Attendance.WCode, tbl_Attendance.Type, tbl_Attendance.Credits, tbl_Attendance.WDate, tbl_Attendance.LName, tbl_Attendance.FName, tbl_Attendance.Campus, tbl_Attendance.SNUM, tbl_Attendance.Email
    FROM tbl_Attendance
    WHERE (((tbl_Attendance.WDate)>=[Forms]![Date_Ranges]![SDate] And (tbl_Attendance.WDate)<=[Forms]![Date_Ranges]![EDate]) AND (("where [tbl_Attendance].[WDate]")<=[tbl_DateRange]![SDate]));

    ken sheridan say: You are clearly trying to restrict the query on the basis of a date range defined by two values entered into a form named Date_Ranges.  So far so good if you amend the query as indicated by Vandergahst, though you can use a BETWEEN ....AND operation in this case.  But then you seem to have a reference to a column SDate in a table tbl_DateRange which is not included in the query's FROM clause.  It looks like you may have used the expression builder to include this.  If this table has just one row then you can include it in the FROM clause without specifically joining it to tblAttendance.  This will return the Cartesian product of the two tables, i.e. each row in one is joined to each in the other, but as one only has one row this does not increase the number of rows returned, so:

    PARAMETERS [Forms]![Date_Ranges]![SDate] DATETIME,
    [Forms]![Date_Ranges]![EDate] DATETIME;
    SELECT tbl_Attendance.WCode, tbl_Attendance.Type, tbl_Attendance.Credits, tbl_Attendance.WDate, tbl_Attendance.LName, tbl_Attendance.FName, tbl_Attendance.Campus, tbl_Attendance.SNUM, tbl_Attendance.Email
    FROM tbl_Attendance, tbl_DateRange
    WHERE tbl_Attendance.WDate BETWEEN [Forms]![Date_Ranges]![SDate]
    AND [Forms]![Date_Ranges]![EDate]
    AND tbl_Attendance.WDate<=tbl_DateRange.SDate;

    If tbl_DateRange has more than one row, however, you'd need some criterion to determine the row whose SDate value you wish to use.  Rather than including it in the FROM clause, you'd then use the DLookup function to return the relevant SDate value from the table.

    PARAMETERS [Forms]![Date_Ranges]![SDate] DATETIME,
    [Forms]![Date_Ranges]![EDate] DATETIME;
    SELECT tbl_Attendance.WCode, tbl_Attendance.Type, tbl_Attendance.Credits, tbl_Attendance.WDate, tbl_Attendance.LName, tbl_Attendance.FName, tbl_Attendance.Campus, tbl_Attendance.SNUM, tbl_Attendance.Email
    FROM tbl_Attendance
    WHERE tbl_Attendance.WDate BETWEEN [Forms]![Date_Ranges]![SDate]
    AND [Forms]![Date_Ranges]![EDate]
    AND tbl_Attendance.WDate<=DLOOKUP("SDate","tbl_DateRange",< criterion expression goes here>);

    Assuming that (1) through (4) from my previous post are correct, I believe the code below will do the trick for you.  Paste it into a regular code module (see http://www.contextures.com/xlvba01.html#videoreg for instructions) and give it a try on a copy of your workbook.  I've also uploaded my test workbook, so you can see that it did work with your test

    TransposeData_001.xlsm  24.13 KB
    http://www.mediafire.com/download/441ubjraujudmyf/TransposeData_001.xlsm

     

    Here is the code - change information in bold entries to match sheet names in your workbook.

     

    Sub TransposeTestData()
      'change these Const values as needed
      'this should be the name of the sheet with the source data
      Const srcSheetName = "Sheet1"
      Const srcFirstDataRow = 5
      Const srcTestIDCol = "B"
      Const srcTestValCol = "D"
      Const srcTestGroups = 23 ' not used, could delete this line of code
      Const srcGpStartKeyPhrase = "DATA SOURCE" ' needs all CAPS
     
      'this applies to the sheet we will put the transposed
      'data onto; the "report" (rpt) sheet.
      Const rptSheetName = "Sheet2"
     
      Dim srcWS As Worksheet
      Dim srcLastRow As Long
      Dim srcRowPtr As Long
      Dim srcGpStartRow As Long
      Dim srcGpEndRow As Long

      Dim rptWS As Worksheet
      Dim rptNextRow As Long
      Dim rptColPtr As Integer
     
      'set up to reference the 2 worksheets
      Set srcWS = Worksheets(srcSheetName)
      Set rptWS = Worksheets(rptSheetName)
     
      'clear old entries from the "report" sheet and rebuild headers
      rptWS.Cells.Clear
      For rptColPtr = 2 To 24
        rptWS.Cells(1, rptColPtr) = "Test" & CStr(rptColPtr - 1)
      Next
      '
      'begin processing data on the source sheet
      '
      'find very last row with a test value entry in it
      srcLastRow = srcWS.Range(srcTestValCol & Rows.Count).End(xlUp).Row
      If srcLastRow <= srcFirstDataRow Then
        'there are no values to report on!
        MsgBox "No Test Data Recorded", vbOKOnly + vbExclamation, "Halting"
        GoTo CleanUpAndExit
      End If
      '
      'start hunting down the groups with data in them
      '
      srcGpStartRow = srcFirstDataRow ' seed to start searching
      Do Until srcGpStartRow > srcLastRow
        DoEvents ' allow other events to be dealt with
        If Left(UCase(Trim(srcWS.Range(srcTestIDCol & srcGpStartRow))), _
         Len(srcGpStartKeyPhrase)) = srcGpStartKeyPhrase Then
          'found "data source" entry, test if it has data by looking 2 rows below for non-empty cell
          If Not IsEmpty(srcWS.Range(srcTestIDCol & srcGpStartRow + 2)) Then
            'we have a group with some data in it!
            'record the data source # info on the report sheet
            rptNextRow = rptWS.Range("A" & Rows.Count).End(xlUp).Row + 1 ' never less than 2
            rptWS.Range("A" & rptNextRow) = srcWS.Range(srcTestIDCol & srcGpStartRow)
            'set srcGpStartRow to point to first "Test#" entry for the group
            srcGpStartRow = srcGpStartRow + 2 ' point to Test1 entry
            'find where it ends!
            srcGpEndRow = srcWS.Range(srcTestIDCol & srcGpStartRow).End(xlDown).Row
            If srcGpEndRow > srcLastRow Then
              Exit Do ' out of data!
            End If
            rptColPtr = 2 'start in column B
            For srcRowPtr = srcGpStartRow To srcGpEndRow
              DoEvents ' allow other events to be dealt with
              rptWS.Cells(rptNextRow, rptColPtr) = srcWS.Range(srcTestValCol & srcRowPtr)
              rptColPtr = rptColPtr + 1
            Next
            'bump to next possible data row
            srcGpStartRow = srcGpEndRow + 1
          Else
            'no "Test#" entry found, check next row
            srcGpStartRow = srcGpStartRow + 1
          End If ' end of test for group Test# entries
        Else
          srcGpStartRow = srcGpStartRow + 1
        End If ' end of test for "data source #" entries
      Loop
      'all done, announce it
      MsgBox "Data Transposition Completed", vbOKOnly + vbInformation, "Task Finished"
      'good housekeeping
    CleanUpAndExit:
      Set srcWS = Nothing
      Set rptWS = Nothing
    End Sub
