This is the relevant information from the “help” documentation for MpCmdRun.exe:

Return code is

      0    if no malware is found or malware is successfully remediated and no additional user action is required

      2    if malware is found and not remediated or additional user action is required to complete remediation or there is error in scanning.  Please check History for more information.

 

The logic should essentially discriminate between the conditions of “no user action required” and “user action required” (rather than between the conditions of “malware found” and “no malware found”), if we can assume that the idea is to allow users to script a notification if a threat that requires user action is detected in a command line scan:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_scanning/set-exclusions-using-command-line/f8df9baa-584d-49b2-9d7c-3fb5798e1ebf

 

So I can certainly see your point if you’re saying that MSE is generating a “false positive” by returning an error code of “2” in cases where it simply can’t scan a file rather than where malware has been detected and requires user action. A third error code would be needed to discriminate between cases where the error state is generated by an error in scanning, which doesn’t require user interaction, and where it’s generated by unremediated malware, which does require user interaction.

Here is a TIP for saving your MS Reader Activation (Pocket PC magazine Tips and Tricks)

 

Saving Microsoft Reader activation

Commercial eBooks in MS Reader format contain Digital Rights Management protection. If you want to read them on your Pocket PC, you have to "activate" the MS Reader application, which is accomplished by connecting to the Microsoft Reader Web site (http://www.microsoft.com/rea...) and following a series of activation steps.

Unfortunately, every time you do a hard reset on your Pocket PC, the activation is lost and you have to go through the whole process again. But fortunately, there is a way to back up the activation and restore it to your Pocket PC after the hard reset. To accomplish this, follow these steps:

 

BACKUP

  1. Create a new backup folder somewhere safe on your desktop PC, or on a storage card. Name it "MS Reader Activation."
  2. With your Pocket PC connected to your desktop PC, launch the ActiveSync application.
  3. Click on the "Explore" icon to open the ActiveSync File Explorer.
  4. Go to the My Pocket PC/Windows folder; find and copy the following files:
    • secrepid.dat
    • secrep.dat
    • secrep.dll
    • secrep.xml
    • Microsoft Activation.unload
  5. Navigate to the backup folder you created in step 1 and paste all the above-mentioned files into this folder.

RESTORE

  1. After a hard reset of your Pocket PC, simply copy the above files, from the backup folder on your desktop PC or storage card, to the Windows folder on your Pocket PC. This will restore the activation and you'll be able to read DRM protected eBooks.

NOTE: Notice that in the file list above you have files with the same file name, but different extensions (i.e. secrep.dat, secrep.dll, secrep.xml). To make sure you are copying the correct files, you need to see the file extension. You cannot do this from the File Explorer application built into the Pocket PC, but you can do it from the Explorer associated with ActiveSync by following these steps:

  1. Open the ActiveSync Explorer window.
  2. Go to the Tools > Folder Options > View dialog box.
  3. Uncheck the option labeled "Hide extensions for known file types."
  4. Select the option labeled "Show hidden files and folders" (if it is not already selected
