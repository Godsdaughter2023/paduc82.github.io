frederik long say:

The symbolic link Application Data is replicating itself endlessly to create a too long name that shuts everything down.

This has been a source of irritation for many years. You can deal with it by deleting the symbolic link with this command, executed from a Command Prompt (cmd.exe):

rd  "Application Data" (or whatever the name of the link might be)

This is a safe command. It will only delete a symbolic link or an empty folder but it will never touch a populated folder.

frederik long say:

It seems your imaging process does not quite deliver what it should. I would now do this:

  1. Boot the machine with a Windows 10 repair flash disk.
  2. Open a Command Prompt.
  3. Type this command:
    C:\ESD\Windows\Boot\BOOTSECT.EXE   /NT60   C:

The above command assumes that C: is your system drive while in Repair Mode. This is not always so!

It also assumes that the file C:\ESD\Windows\Boot\BOOTSECT.EXE exists.

Lastly the process assumes that you are familiar with advanced maintenance tasks. If not then I'll exapand on points that are unclear.
