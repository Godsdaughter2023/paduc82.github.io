I suggest you to try the following steps to stop and restart the Windows Update Service and check if it helps.

a. Press Windows + R keys and type Services.msc and press Enter.

b. Search for Windows Update Services.

c. Right click on Windows Update Services and click on Stop.

d. Open Windows Update and check for the updates, the screen will go blank.

e. Press Windows + R keys and type of Services.msc and press Enter.

f. Right click on the Windows Update Service and click on Start.

g. Open Windows Update and check for the update and then try installing the update.

If still the issue persists, then try resetting Windows Update Components and check if it works for you. This will replace the missing or corrupted Windows Update Components.

a. Open a elevated command prompt (Admin). Press Windows + X keys and click on Command Prompt (Admin).

b. Stop the BITS service, the Windows Update service, and the Cryptographic service. To do this, at a command prompt, type the following commands. Make sure that you press Enter after you type each command. 

  • net stop bits
  • net stop wuauserv
  • net stop appidsvc
  • net stop cryptsvc

c. Delete the qmgr*.dat files. To do this, at a command prompt, type the following command, and then press Enter:

Del "%ALLUSERSPROFILE%\Application Data\Microsoft\Network\Downloader\qmgr*.dat"

d. Rename the softare distribution folders backup copies. To do this, at a command prompt, type the following commands. Make sure that you press Enter after you type each command.

  • Ren %systemroot%\SoftwareDistribution SoftwareDistribution.bak
  • Ren %systemroot%\system32\catroot2 catroot2.bak

e. At a command prompt, type the following command, and then press Enter:

cd /d %windir%\system32

f. Reregister the BITS files and the Windows Update files. To do this, at a command prompt, type the following commands. Make sure that you press Enter after you type each command.

  • regsvr32.exe atl.dll
  • regsvr32.exe urlmon.dll
  • regsvr32.exe mshtml.dll
  • regsvr32.exe shdocvw.dll
  • regsvr32.exe browseui.dll
  • regsvr32.exe jscript.dll
  • regsvr32.exe vbscript.dll
  • regsvr32.exe scrrun.dll
  • regsvr32.exe msxml.dll
  • regsvr32.exe msxml3.dll
  • regsvr32.exe msxml6.dll
  • regsvr32.exe actxprxy.dll
  • regsvr32.exe softpub.dll
  • regsvr32.exe wintrust.dll
  • regsvr32.exe dssenh.dll
  • regsvr32.exe rsaenh.dll
  • regsvr3

    g. Reset Winsock. To do this, at a command prompt, type the following command, and then press Enter:

    netsh winsock reset

    h. Restart the BITS service, the Windows Update service, and the Cryptographic service. To do this, at a command prompt, type the following commands. Make sure that you press Enter after you type each command.

    • net start bits
    • net start wuauserv
    • net start appidsvc
    • net start cryptsvc

    i. Restart the computer.

    1. Uninstall Asus PC Probe, as well as any other Asus bloatware.

    2. In your loaded drivers list, dtsoftbus01.sys is listed which is the Daemon Tools driver. Daemon Tools is a very popular cause of BSOD's in 7/8 based systems. Please uninstall Daemon Tools. Alternative imaging programs are: MagicISO, Power ISO, etc.

    3. Enable Driver Verifier if #'s 1 and 2 don't help:

    Driver Verifier:

    What is Driver Verifier?

    Driver Verifier is included in Windows 8/8.1, 7, Windows Server 2008 R2, Windows Vista, Windows Server 2008, Windows 2000, Windows XP, and Windows Server 2003 to promote stability and reliability; you can use this tool to troubleshoot driver issues. Windows kernel-mode components can cause system corruption or system failures as a result of an improperly written driver, such as an earlier version of a Windows Driver Model (WDM) driver.

    Essentially, if there's a 3rd party driver believed to be at issue, enabling Driver Verifier will help flush out the rogue driver if it detects a violation.

    Before enabling Driver Verifier, it is recommended to create a System Restore Point:

    Vista - START | type rstrui - create a restore point
    Windows 7 - START | type create | select "Create a Restore Point"
    Windows 8 - http://www.eightforums.com/tutorials/4690-restore-point-create-windows-8-a.html

    How to enable Driver Verifier:

    Start > type "verifier" without the quotes > Select the following options -

    1. Select - "Create custom settings (for code developers)"
    2. Select - "Select individual settings from a full list"
    3. Check the following boxes -
    - Special Pool
    - Pool Tracking
    - Force IRQL Checking
    - Deadlock Detection
    - Security Checks (Windows 7 & 8)
    - DDI compliance checking (Windows 8)
    - Miscellaneous Checks
    4. Select  - "Select driver names from a list"
    5. Click on the "Provider" tab. This will sort all of the drivers by the provider.
    6. Check EVERY box that is NOT provided by Microsoft / Microsoft Corporation.
    7. Click on Finish.
    8. Restart.


    - If Driver Verifier finds a violation, the system will BSOD. To expand on this a bit more for the interested, specifically what Driver Verifier actually does is it looks for any driver making illegal function calls. When and/if this happens, system corruption occurs if allowed to continue. When Driver Verifier is enabled, it is monitoring all 3rd party drivers (as we have it set that way) and when it catches a driver attempting to do this, it will quickly flag that driver as being a troublemaker, and bring down the system safely before any corruption can occur.

    - After enabling Driver Verifier and restarting the system, depending on the culprit, if for example the driver is on start-up, you may not be able to get back into normal Windows because Driver Verifier will detect it in violation almost straight away, and as stated above, that will cause / force a BSOD.

    If this happens, do not panic, do the following:

    - Boot into Safe Mode by repeatedly tapping the F8 key during boot-up.

    - Once in Safe Mode - Start > Search > type "cmd" without the quotes.

    - To turn off Driver Verifier, type in cmd "verifier /reset" without the quotes.
    ・    Restart and boot into normal Windows.

    If your OS became corrupt or you cannot boot into Windows after disabling verifier via Safe Mode:

    - Boot into Safe Mode by repeatedly tapping the F8 key during boot-up.

    - Once in Safe Mode - Start > type "system restore" without the quotes.

    - Choose the restore point you created earlier.

    -- Note that Safe Mode for Windows 8 is a bit different, and you may need to try different methods: 5 Ways to Boot into Safe Mode in Windows 8 & Windows 8.1

    How long should I keep Driver Verifier enabled for?

    I recommend keeping it enabled for at least 24 hours. If you don't BSOD by then, disable Driver Verifier. I will usually say whether or not I'd like for you to keep it enabled any longer.

    My system BSOD'd with Driver Verifier enabled, where can I find the crash dumps?

    They will be located in %systemroot%\Minidump

    a. user say:

    Hi, 

    One of the possible reason why this issue occurs is due to corrupted system files. You can try these methods in troubleshooting this matter: 

    Method 1: Run the Hardware and Devices Troubleshooter. 

    To run the troubleshooter, follow these steps: 

    1. On the search bar, type Troubleshooting. 

    2. Choose Hardware and Devices. 

    3. Click on Hardware and Devices. 

    Method 2: Run SFC Scan.  

    To run an SFC scan, follow these steps:  

    1. Run Command Prompt as Administrator.  

    2. Type sfc /scannow and hit Enter.  

    This will check and run an auto-repair on the issue.  

    Method 3: Run DISM Tool.  

    After running the SFC scan and the issue still persist, run the DISM to check the system health and to try restoring the files. To run DISM tool, follow these steps:  

    1. Run Command Prompt as Administrator.

    Windows 7: anti-aliasing and Aero - Microsoft Community

    bobtwo say:

    Windows 7: anti-aliasing and Aero

     Is it possible to have anti-aliasing active while using an Aero theme?
    bobtwo say: I have tested your suggestions and concluded: (a) ClearType is not a key element, (b) the magnifier does MORE than simply resize the image, and (c) the DPI setting is not a key element either.

    The magnifier docked and set for 800% enlargement.

    The display is set to enlarge objects by 150% [Control Panel / Display].  This has a
    large smoothing effect because Windows then uses more than twice as many pixels to render each character.

    ClearType is OFF [Control Panel / Display / Adjust ClearType text].

    If I select an Aero theme [Control Panel / Personalization] such as "Windows 7" and  examine a Notepad document prepared with 12 point Lucida Console font, I find that the magnified image is composed of 8x8 "super pixels" rendered in several shades: black to light gray.  The overall appearance is very pixelated/jagged.  Turning ClearType ON produces some change (non-gray pixel colors). The magnified image is still composed of 8x8 "super pixels".

    If I change to a non-Aero theme such as "Windows 7 Basic", the magnified image is rendered with individual pixels and becomes very smooth and clean.  Turning ClearType ON produces a colored halo around the left and upper edges of each character but adds no improvement in legibility.

    Repeating the above tests with display enlargement set to 100% produced the same results.

    I hope this helps.
