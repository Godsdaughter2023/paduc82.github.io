hi saeedsafaeifar,

if you want the search results are shown based on user's relevant permission to the documents accordingly, then the answer will be no. in my experience, we may need to identify the document permission for the users when we create/upload them which can be done by creating a managed metadata site column. then we map this crawled property to a refinablestring managed property, and edit the refinement web part to add the refiners.

please see the detailed steps below:
1. go to site settings > term store management (under site administration) > hover your mouse above site collection - xxxx.sharepoint.com-sites-xxx > click the down arrow and select new term set.
 
2. create a managed metadata site column using the term set created above. one document may be accessed by several users, so please mark allow multiple values, and also select yes under require that this column contains information.
 
3. add this column to a library and select add to default view. upload the files and fill the column value for each file. (please note that the value stands for the user who has related permission to thesedocuments. in my picture below, test1/test2/test3 stand for te1/te2/te3)
 

for steps 4-6, we can also refer to this blog: refiners for managed metadata columns in sharepoint online.
4．map a crawled property – ows_columnname to one of the refinablestring managed property, say refinablestring01. (please go to sharepoint admin center > search > manage search schema)
  
 

5. manually re-index this site (site settings > search and offline availability under search > reindex site) and wait for about 15 minutes.

6. go to the search site (domain.sharepoint.com/search) > settings > edit page > edit the refinement web part and then click on "choose refiners…" button > select refinablestring01 > add > click move up to put this new navigator at the top of the "selected refiners" list in the refinement web part, the users refiners will be displayed for search results and available to filter the results to the appropriate selection > click apply > save.

 

7. after that, when user te1 searches "document", the search results can be filtered through "test1" which lists the documents shared with  te1.

hope it meets your requirements.

If you have used the OCR function to convert the graphic to editable text then be aware that MODI has minimal formatting capabilities. If you want better formatting you need better OCR software. Why must there? The conversion of graphics to text is one of the most complex tasks required of a PC.

As I said in my previous response, no OCR software is perfect. Many (including MODI) will adequately grab the text, but the formatting is a different issue entirely. For example you may not have the fonts used in the original document - and that is only one of the simpler issues to address.

mike h.. say:

Hi,

 

ALT+F11 to open VB editor, right click 'ThisWorkbook' and insert module and paste the code below in and run the code.

 

Sub Sortem()
Dim x As Long, y As Long
For x = 1 To Worksheets.Count
    For y = x To Worksheets.Count
        If UCase(Sheets(y).Name) < UCase(Sheets(x).Name) Then
              Sheets(y).Move before:=Sheets(x)
         End If
    Next
Next
End Sub
