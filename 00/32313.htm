Hi,
 
This issue can be caused by a corrupted state of the .Net framework that currently exists on your computer. This can be caused for various reasons to include but not limited to: other applications overwriting .net files, viruses and corrupt hard drive sectors.
 
To troubleshoot the issue you will need to uninstall multiple versions of the .net framework using a .net cleanup utility, and then reinstall those versions in their correct order.

To repair the .net framework follow the steps below:

 

1. Download .Net Cleanup Utility from the following link:

http://blogs.msdn.com/astebner/attachment/8904493.ashx

2. Using the .net Cleanup Utility, uninstall .Net Framework 1.0, 1.1, 2.0. 3.0 and 3.5 (The cleanup utility won’t allow you to remove .NET Framework 2.0 on Windows Vista and higher because it is installed as an OS component.)

3. Restart the computer

4. Download the following components:

.Net Framework 3.5 SP1 (this will also install .Net Framework 2.0 SP2 and 3.0 SP2)

http://www.microsoft.com/downloads/details.aspx?familyid=AB99342F-5D1A-413D-8319-81DA479AB0D7&displaylang=en 

.Net Framework 1.1

http://www.microsoft.com/downloads/details.aspx?displaylang=en&FamilyID=262d25e3-f589-4842-8157-034d1e7cf3a3 

.Net Framework 1.1 SP1

http://www.microsoft.com/downloads/details.aspx?familyid=A8F5654F-088E-40B2-BBDB-A83353618B38&displaylang=en 

5. Install the downloaded updates in this order (the order is very important):

  • .Net Framework 3.5 SP1

  • .Net Framework 1.1

  • .Net Framework 1.1 SP1


6. Restart the computer

7. Go to Windows Update then choose CUSTOM update

http://update.microsoft.com/microsoftupdate/

8. Under Optional Software Update, check if other versions of .Net Framework is being offered

  • IF NO: Close the case as Resolved

  • IF YES: Install the update using Custom Update then dispatch the case to Non-Tech Callback


9. Once the installation is complete reboot your computer and attempt to install updates again.

Note: If Windows Update is offering .Net Framework and other updates, troubleshoot the other update first before dealing with .Net Framework updates.
 
http://support.microsoft.com/kb/976982

http://support.microsoft.com/kb/923100

thomascrabb say:

Samsung SCX-4100 Printer - Getting Working (Scanning + Printing) with Windows 10

I am writing after working out how to install the obsolete Samsung SCX-4100 Scanner/Printer under Windows 10 and to help others if struggling to do so. Hopefully the search engines will index it appropriately.

For printing use the latest universal printing driver on the Samsung support website. Printing will then work whether old driver/software already (prior to upgrading your old Windows to v10) or not. But scanning does seem to require SCX-4100Twain driver to be on system installed from the original software.

So assuming older printing/scanning software preloaded:-
Install the universal driver "SamsungUniversalPrintDriver2.exe". Now two printers will show under "Devices and Printers" - you can delete older SCX-4100 icon, keep the universal printer icon in "Devices and Printers". If ever need to reinstall universal driver and install program cant see printer turn off/on while install program searching for it (it worked for me).

Scanning software is "SmarThruOffice_V2.10.00.07" from Samsung website and requires extracted from exe self-executing archive, otherwise fails on installation of the exe saying publisher blocked. Right click on the exe and use "extract to..." to empty contents into directories. Once extracted go into TEMP>STO folder and install SmartThru2 by clicking on setup program in this directory..
Then open SmartThru and select SCX-4100Twain driver from scanner choice. Will definitely be there if old software was installed first, otherwise probably not (I have not tested SmartThru2 clean install without the older original software already being on there.
(Cant say for sure as did recent upgrade old software on from window 7 before it)
