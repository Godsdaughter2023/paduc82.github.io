a. user say:

What is a notepad attachment?  Do you mean a .txt file type (or some other file type that is supposed to open with Notepad)?  Go to Start / Default Programs / Associate a File or Protocol with a Program and scroll to the file type of the attachment (the three digit extension following the filename) and make sure Notepad is selected to open that file type.  Does this happen with ALL Notepad attachments?  Try opening a Notepad file on your system that you can read and e-mail it to yourself and see if you can read it from the email - if so then the process is working correctly and the issues is with these attachments.

If Notepad is correctly selected, perhaps these file extensions are incorrect - they do not represent the actual file type of the file (so Vista is using the wrong program to try to open them).  Or maybe the extensions are correct, but they were created by another program that Notepad can't read (that happens with some programs where Notepad is considered the default but can't read such files because they were created in a way that Notepad can't interpret).  Find out from the sender what program was used to create the files - you may need to get some program that can view those file types.

I hope this helps.

Good luck!

First lets disable the services “HomeGroup Listener” and “HomeGroup Provider” and then try to leave the homegroup from both the computers.

To remove Homegroup (and stop Homegroup networking functionality):
a. Go to Control Panel -> Network and Sharing Center -> HomeGroup, and click on Leave the homegroup link to unjoin from any existing home group.
Note: If the home group are shared and hosted from the PC, all HomeGroup connections will be disconnected.
b. Click on Leave the homegroup and confirm. In the future, if you want to use Homegroup networking you can create a homegroup using this same dialog.
c. Go to Control Panel -> System and Security -> Administrative Tools, and double click on Services. Alternatively, type services.msc in Start Search.
d. For each of the following two services:
HomeGroup Listener
HomeGroup Provider

Do the following:
Stop the service, and then double click on the service to open Properties dialog, and set its Startup type to Disabled. Click OK when done. In order to reinstate Homegroup functionality in the future, you will want to set the Startup type back to Manual.
e. The HomeGroup icon and group will no longer be shown in the navigation pane of Windows Explorer in Windows 7. Note that this does not interrupt "traditional" windows networking functionality (e.g., Map Network Drive).
 
Hope this information is helpful. If the issue still persists, please post back for further assistance, we'll be glad to assist you.
