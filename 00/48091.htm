[[[SOLVED]]] Groove music app in Windows 10 Pro (desktop) immediately crashes after starting

Since August of 2015 when I first installed Windows 10 I have been battling with Groove music app. I finally slayed this Microsoft beast and made it behave and this is how I did it.

The main culprit is:
a security permissions issue on the hidden C:\Program Files\WindowsApps folder as root source of the problem with Groove starting and as soon as you click on anything in the window the Groove app just crashes. No warning boxes, no event viewer messages, nothing. The window just closes after any click is made within the border of the window.

Changing the security permission alone will not fix this. Uninstalling Groove without doing the security permissions as well will not fix this. You need to do both and here is how, with SPECIFICITY!:

Step 1

The first problem is that the folder is a hidden C:\Program Files\WindowsApps folder so you have to tell Windows Explorer > View tab to show Hidden Items by selecting the check box. Then this mysterious folder is revealed. Ooh so secretive!!! Shhhh, Microsoft does not want you to find this folder which is why they make it hidden! Since I am running Windows 64-bit I am working in the C:\Program Files folder. If you are running a 32-bit flavor of Windows it may be located in the C:\Program Files (x86) folder. I have no idea, since I run 64-bit, where the 32-bit version of all Windows apps are installed (neener, neener, neener...)

Step 2

The next step it to rip the permission away from TrustedInstaller away and give it your Account name the admin permissions the C:\Program Files\WindowsApps folder deserves. I have no freaking clue who this "TrustedInstaller" fool is but if I am the Admin to my own personal PC and that should be good enough. Ironically, Microsoft does not think the same so we have issues like apps that do not work when started. You must have Full Control access to the entire \WindowsApps folder (I'm not taking any chances, I want it all!!!). There will be a lot of error box messages and nag screens and I am sure the red phone at Microsoft will ring once you have ripped the permission away from TrustedInstaller but since all any MVP from Microsoft care to do to fix this problem (that thousands have with Groove) is post canned responses they won't gripe too much. See Additional Info article link below on how to change folder permissions steps.


Step 3
Uninstall Groove completely. And here is a great article on how you do it. http://www.thewindowsclub.com/uninstall-groove-music-windows-10

Some caveats for the uninstall to know about:
a.) Make sure you use the Admin Windows PowerShell not a plain ole command line. Right-click on Start and select (oddly enough) the choice that shows Windows PowerShell (Admin).

b.) Make sure the entire PackageFullName as displayed on your computer is EXACTLY what comes after you type the remove-AppxPackage command into PowerShell. If you have to open up 2 PowerShell windows and copy and paste from one PowerShell into a different PowerShell window in order to run the command remove-AppxPackage then do it! Specificity is not overkill here. Whatever comes after PackageFullName is what needs to be removed on your computer. The example the article gives you of Microsoft.ZuneMusic_3.6.12711.0_x64__8wekyb3d8bbwe may be different on your machine! The bold part may be different!

c.) Use copy and paste not finger typing. If the PowerShell gives you output A, you copy and paste output A exactly as shown. Do not let fat-fingering a long string of gobbly-gook names of folders screw you up.

d.) Running Get-AppxPackage –AllUsers will give you a very voluminous display. The Microsoft.ZuneMusic with all the gobblygook after it will be towards the top about 2-3 page scrolls worth. In PowerShell, use the icon in the upper left corner of the window and do an Edit > Copy and Edit > Paste to copy and paste between PowerShell windows (if you are using 2 of PowerShell windows) to avoid finger typing and making a simple yet detrimental mistake. Seriously do not use CTRL + C and CTRL + V to copy and paste, go old school and be specific with the PowerShell pull-down flavor of copy and paste.

e.) Do not confuse Microsoft.ZuneMusic with Microsoft.ZuneVideo. They are two different apps as far as Windows is concerned. Do not muck around with Microsoft.ZuneVideo. God knows what that will break. When it comes to Microsoft software, the old adage, if it ain't broke don't fix it applies.


Step 4

Once Groove is uninstalled verify that in the C:\Program Files\WindowsApps folder that any instance of C:\Program Files\WindowsApps\Microsoft.ZuneMusic_?????? is completely gone. You will not have just one folder you can have multiple. Delete them all. Do not delete the folders that say Microsoft.ZuneVideo however.
If you run into a permission problem deleting any \WindowsApps\Microsoft.ZuneMusic_?????? folder then you did not do Step 2 correctly and still do not have Full Control permission to the \WindowsApps folder. I had 4 \Microsoft.ZuneMusic folders (with various names). Here they are comfortably resting in the Recycle Bin ready to be flushed soon:


The 4th folder was removed automatically by the remove-AppxPackage command of Step 3. Check your permissions to the \WindowsApps folder if you can not delete the other 3 folders.


Step 5

Verify that in the C:\Users\[User Name]\AppData\Local\Packages folder there is no Microsoft.ZuneMusic_???? folder as well. If there is, delete it. If you can't, transfer the ownership of the \Packages folder from that bloke TrustedInstaller to your logged Account Name (Admin), with again, Full Control access to that entire \Packages folder.


Step 6

You are now ready to open up Windows Store and reinstall Groove if you so choose. No restart of the computer is necessary since the security permission problem never allowed the process to start correctly anyways. You can, if it makes you feel warm and fuzzy, but a restart is not necessary. Once re-installed you will notice that Groove app now works as it should.
