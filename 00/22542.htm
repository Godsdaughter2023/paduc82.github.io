My hypothesis is that somewhere above the ink, a different font is being used to render text on either the teacher's laptop or the students laptop.

The ink strokes are positioned relative to the page. The image of the cell however, appears to be in the same outline as the text above it, and so it will change position relative to the text above it. If for some reason OneNote picks a different font to render the text above the image, and that font has different width characteristics, the sentences can line wrap differently, which could make the image appear higher on the page relative to the ink strokes (or lower, depending on the characteristics of the font.) Similarly, if the font had different height characteristics, the image could also appear in a different position relative to the ink strokes.

Selecting the font and looking at the font name in the ribbon wont necessarily tell you a different font is being used. If a font isn't on the users system, we will still show the "expected" font name in the UI, but substitute a different font for rendering on the page. So you have to look at the fonts used on the page visually to see if the text is wrapping differently, or looks different, which is why I ask for the screen shot of the text from both the student and teacher pages.

doug robbins - mvp office apps & services (word) say:
Does anyone know how to populate a label sheet using the mail merge?


If you are using a tablet PC, there is a bug which will prevent the labels from propogating correctly.  The following macro can be used as a work-around:

Macro to overcome the problem with propagation of mail merge field fields with a label type mail merge when using a Tablet PC and some other Computers that have Tablet PC Functionality available:

Sub MailMergePropagateLabel()

Dim atable As Table

Dim i As Long, j As Long

Dim source As Cell, target As Cell

Dim myrange As Range

Set atable = ActiveDocument.Tables(1)

Set source = atable.Cell(1, 1)

Set myrange = source.Range

myrange.Collapse wdCollapseStart

ActiveDocument.Fields.Add Range:=myrange, Text:="NEXT", _

PreserveFormatting:=False

source.Range.Copy

For j = 2 To atable.Columns.Count

    Set target = atable.Cell(1, j)

    If target.Range.Fields.Count > 0 Then

        target.Range.Paste

    End If

Next j

For i = 2 To atable.Rows.Count

    For j = 1 To atable.Columns.Count

         Set target = atable.Cell(i, j)

         If target.Range.Fields.Count > 0 Then

              target.Range.Paste

         End If

    Next j

Next i

atable.Cell(1, 1).Range.Fields(1).Delete

End Sub
