a.    Plug the device to the computer via USB.

f.    Run adb -devices again. If it still doesn't show or show as offline, check to see if you have the driver installed. Also check to make sure that the USB connection is set to Charge only (see steps above) and check with the issue.

This depends on what action you to assign the folder pairs in SyncToy. From the SyncToy Help:

 

Synchronize: New and updated files are copied both ways. Renames and deletes on either side are repeated on the other.


Echo: New and updated files are copied left to right. Renames and deletes on the left are repeated on the right.


Contribute: New and updated files are copied left to right. Renames on the left are repeated on the right. No deletions.

 

As you can see above, Contribute will make no deletions, however, it is only going from the left to right in the folder pairs.

So ... I have some good news and some bad news.

First the good news: I have been able to consistently reproduce the issue and mitigate it with a work-around, which works for SOME scenarios, but not all.

The bad news: The work-around involves removing your second (or more) video cameras.

My observation is that this issue comes when there are multiple camera inputs available to Skype for Business. When choosing the camera source in Skype, none of the sources may be in use by another program. This is presumably a limitation in either the OS or the Direct Show libraries running beneath the hood.

However, what happens is this: In the control panel you can see the input from all available sources. If none are in use, you can choose between them, and select your preferred one. When you have selected one, it will be the used source in new Skype conversations.

Then, if another source is present in a multi-source system, and the other source is in use by some other application (not by Skype itself), Skype for Business will determine that your selected source device "isn't set up or you're already using it". The work-around is to disconnect other video sources, or if they are embedded into your device, disabling them in Device Manager.

In other words: I can consistently fix this issue by reducing my configuration to a single video source.

The natural thing to do, is to suspect a bug in the way Skype for Business determines if the selected video source is in use or not. Somehow, when OTHER sources are in use by other applications (or in some cases just present and only previously in use) Skype for Business will incorrectly determine that the SELECTED source is already in use, even though it isn't.

Be aware that the conversation window seems to (perhaps?) cache the state of video sources. When testing you should remember to close conversation windows, so that only the main Skype for Business window is open.

Hope this helps all involved.

Unmerging the Music system folder from the Onedrive Music folder. - Microsoft Community

tdemerse say:

Unmerging the Music system folder from the Onedrive Music folder.

As anyone who has moved their Photos, Music or Documents user folder into their Onedrive folder will know, you are greeted with a prompt stating that that move cannot be reversed as the system folders will merge. I know that reinstalling the operating system will unmerge them, but is there any other advanced way to complete the un-merge? As it turns out, I changed my mind about my set up gearing up for the rumoured Xbox Music Locker, but now am faced with the un-ability to separate the folders. I think at one point either SFC or the OS Refresh tool separated them out, but this no longer seems to be the case.

Thank you for your help with my long shot. If there isn't a way to do it without a reinstall, I'll hang tight until the Music Locker actually materializes and make my decision then.

cmdrkeene say:

System Restore will do it quickly and easily, but only if the change is recent (last week or two). Don't resort to the Reset (those are way too destructive for something like this).

If System Restore isn't possible because it's been too long, you can do it manually in the registry. In fact the only reason I recommend Restore is because it's easier and less advanced (and easy to undo) than a registry change. But as long as you're careful, you can do it in the registry too.  Here's how:

  1. Start regedit

  2. Go to HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders

  3. Find the Music item and set the location to be the right folder again. Use another as a template (like the pictures one for example).  The default music location is %userprofile%/Music

After setting it, just reboot.  The files won't be moved (you'll have to drag them yourself), but the settings will be fixed up.
