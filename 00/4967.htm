Now: Your diagnosis is pretty good: if the Normal template is corrupt, it won't load, and Word can't operate.

So let's fix that one:

  1. Quit Word (must be quit, not minimised)
  2. Also quite Outlook (same, Outlook has a second copy of Word running).
  3. Go to here:
    ~/Library/Group Containers/UBF8T346G9.Office/User Content/Templates
    (In the Finder, on the Go menu, choose Go To Folder, then paste that line in, it will take you straight there).
  4. You will see the file Normal.dotm
  5. Drag it to your desktop.
  6. Restart Word.

If the problem was your Normal template, that will have fixed it.

Word will create a new, perfect, copy of Normal when it restarts.  If there was anything valuable in your Normal template, you can use the Organiser dialog to copy it back to the new Normal without transferring the problem.

If that did NOT fix the problem:

  1. Think carefully: the next procedure is complex and dangerous.  If you are not confident with computing, ring the Microsoft Support line and get them to talk you through it.
  2. Make certain you have a recent backup, and that it works.  Check this by restoring a file to the Group Containers folder mentioned above, to ensure you can get it back.
  3. Quit ALL Microsoft applications.
  4. Go to the following folder:
    ~/Library/Containers
  5. Find the folder com.microsoft.Word
  6. Drag that to your desktop.
  7. Restart your computer (all the way down to power-off, we want to trigger the OS X clean-up tasks that run as part of boot-up)
  8. Re-start Word (you should be prompted to activate again, do that).
  9. Quit Word (don't try to check it works, just Activate it then quit it).
  10. Log out.
  11. When you log back in, the problem should be fixed.  In which case, you can delete the files you moved to the desktop.

If the problem is NOT fixed, ring Microsoft Support.

They need to talk you through a full Remove and Replace. Exactly how you go about that depends on which license you have: the Support Agent can figure that out from your product serial number.

suzanne s. barnhill say:
You can do this with a combination of table and newspaper-style columns. Your best bet is to format the page columns first. On the Page Layout tab, in the Page Setup group, click on Columns and select the desired number. Then insert your two-column table, which will be constrained to the column width. Mark the first row as a header row, and it will be carried over to the next column when the first column is filled.

pamcaswell say: You can:
  • Insert next page section beaks before and after the table.
  • With the cursor in the section that contains the table, change the page orientation to landscape and Set the columns to the number you want.
  • The width  of the gutter between columns (Word ambiguously calls it spacing) is .5"--way too large if you are creating a technical or other formal document and will limit the number of columns you can get on the page.
  • Adjust the spacing  in the Columns dialog to .25 or .3 (if you agree with me).  Note the column width, and be sure that the Equal column width box is checked. OK out of the  dialog.
  • Back in the table,  adjust its width to be equal to or less than the column width.

stefan blom say:

You have to unlink the header (or footer) in the first section, and then you can delete the page numbers.

Note that each section can have up to three different headers (and three different footers), which is the case if you have enabled both "Different odd and even" and "Different first page" in Page Setup. Each type of header (footer) must be separately unlinked from its "neighbor" in the preceding section.

Rather than use the "Remove" option, activate the header/footer view and delete the page numbers directly, as you would delete text.

For more on front matters in Word, see http://wordfaqs.mvps.org/FrontMatterRibbon.htm.
