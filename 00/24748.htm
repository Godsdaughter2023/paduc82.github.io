Hello AussiePlumBob

Answer to your questions depends entirely on what protocol (POP or IMAP) was used to originally configure the account(s),

#1 - <IF> the accounts were originally configure as POP3 accounts, Outlook would have created a .PST file. When you configure the account again in Outlook, you would have to be sure to configure it manually as a POP account and select the option to use an existing data file (as per attached screenshot).

#2 - <IF> the account was configured using IMAP, Outlook would have created an .ost file which then sync's the email folder with the server. The first IMAP account configured (assuming no other account was already configured) would also have folders with the words "This Computer Only" (contact/calendar).

Here's the good and bad news if #2 describes your scenario. The good news is that any time an IMAP account is configured, a new OST file is created and the email folders are sync'd again with the server. An .ost file is a mirror of what is contained on the server in terms of email folders. The bad news is that the contact/calendar folders will be empty. You cannot use an existing .OST file.

OST files can only be opened by the original Outlook profile/email account used to create it. in other words, if your old data files are .ost files, you would need an OST2PST conversion program to access the data which would be required if you need to retrieve contact/calendar data.

If #2 is the case, then the way to avoid losing contact/calendar data in the future would be to create/add a new PST file to the Outlook profile after your accounts are configured and then set this PST file as the default data file.

To add a new PST file to the profile
- from Home Ribbon --> <New Items> --> <More Items> --> <Outlook Data File>

To set the PST file as the default data file
- go to File --> Account Settings --> Data File tab --> select the PST file --> set as default

hansv mvp say:

Copy the following function into a module in the Visual Basic Editor (press Alt+F11, then Alt+I, M to create a module):

Function Extract8Digits(s As String) As String
    Dim re As Object
    Dim mt As Object
    Set re = CreateObject("VBScript.RegExp")
    re.Pattern = "\b([0-9]{8})\b"
    Set mt = re.Execute(s)
    If mt.Count > 0 Then
        Extract8Digits = mt(0)
    End If
End Function

Switch back to Excel.

Let's say the first text value is in A1. Enter the following formula in B1:

=Extract8Digits(A1)

Then fill down.

Make sure that you save the workbook as a macro-enabled workbook (.xlsm) and that you allow macros when you open it.
