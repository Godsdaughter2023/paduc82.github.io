a. user say:

An update to this, as I had a sudden thought, and testing seems to bear this out...

It looks like it's a difference in how the old OneDrive for Business and the NG clients work.

For some reason the old client uploaded these linked folders, and their contents to SharePoint, but hid the contents (no items visible in the name_files folders via the web site, but are visible locally).

After updating to the NG client, it seems the NG client was trying to upload the folders again, and for some reason (presumably due to the folders being hidden on the SharePoint site, but not locally) clashed with the site contents, so refused to upload the files, hence the errors above.

Just moving the files didn't seem to help, but completely deleting the local folder (and the related HTML file), then waiting for OneDrive NG to catch up and re-sync (so deleting the hidden on-line files). I then copied a backup copy of the files back into my local OneDrive again. This time the sync worked file, and viewing the name_files folder in OneDrive online, now shows all the files (rather than an empty folder as before).

So the steps I followed to work around the issue:

  1. Make sure you have a local backup copy of the files you have the issue with. (If not make a copy).
  2. Delete the local files.
  3. Wait for OneDrive NG sync to finish. (I had the SharePoint web site open at the parent folder and hit F5 till I confirmed the files/folder had gone).
  4. Copy your backup files back into the local OneDrive folder.
  5. Again wait for OneDrive NG sync to finish.
  6. Look on the web site, and the files inside the name_files folder should now be visible and no more error in the sync client.

Regards,

Mark
