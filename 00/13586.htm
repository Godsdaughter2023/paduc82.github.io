I regard Microsoft's current implementation of PowerPoint table styles to be incomplete and flawed. There are several important characteristics that can't be set like vertical alignment, cell margins, column widths and row heights.

PowerPoint implements vertical alignment by tagging each cell in the final table. This approach is not amenable to including in a style, because table styles only have row and column parameters, nothing about cells. If they changed this to making vertical alignment a row property, it could be included. Then individual cells could be tagged on the final table if they were different from the row setting.

On the PowerPoint User Voice forum, I've created a suggestion that this be changed in future versions. This is the process I used:  I opened the message contaiing the long string of recipients, clicked (anywhere) in the Bcc field, pressed Ctrl-A to select all of the recipients, then pressed Ctrl+C to copy the list.  I then clicked File>New>Distribution List to open a DL form.  I then clicked Select Members.  I clicked in the Members field at the bottom and pressed Ctrl+V to paste the list of recipients.  I then clicked OK and the recipients all appeared in the DL, but they appeared only once.  Outlook automatically removed the duplicates.

OK, I went through the process one more time and I was correct the first time.  You do use Select Members and you put them in the Members field at the bottom.  The reason Outlook asked you about the name as shown in your image is because it doesn't have a value address.  Outlook only asks you when the address is malformed.  That means that the name about which it asks should be removed from your list because there's no address associated wth that name.  I tested this, using valid addresses and invalid addresses.
