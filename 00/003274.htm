To assign a license

You need not really worry about losing any data or information from IMAP accounts if you happen to delete or lose your OST files regardless of wherever they are saved on the local computer, as recreating the IMAP account on the local computer would resynchronize all the information and bring back all data.

Sorry, but that's not true. Depending on the details you set for keeping information on the server or deleting it, the local file is not necessarily synchronized.

When you use IMAP, the mail is synced in the IMAP ost file. You can't delete it from the server but leave it in the ost file. A POP3 account can delete from the server - but those PST files can be opened in any profile.

Note, if you use Outlook 2013 and keep calendar and contacts in the imap data file, they are not synced or backed up anywhere and there is the potential to lose them if you don't export first. 

On the original question about the file paths - did you set the reg key then create the accounts? The ost is not moved for existing accounts only for new accounts.
