In any cell in the row

1.      When the letter C is entered, all cells (excluding the cell containing the C) to the left change colour.

2.      When the letter L is entered, all cells (excluding the cell containing the L) to the right change colour.

bernard liengme say: In L1:V1 (too lazy to go to GV1) I typed some text with "C" in one of the cells. Copied this down 10 rows and change the position of "C" in each row. Selected L1:V10 and used this rule =COLUMN()<COLUMN($K1)+MATCH("C",$L1:$V1,0) with a blue fill. This gave the required results. I thnk you can see how to add the second rule. best wishes
Bernard Liengme, Nova Scotia, Canada http://people.stfx.ca/bliengme/

pete_uk say:

I set this up using the range C2:R2 for testing. Select that range, with C2 as the active cell, and use this formula for the "C" entry:

=AND(COUNTIF($C2:$R2,"C")>0,COLUMN(C2)<MATCH("C",$2:$2,0))

adjust the references to suit your range, but ensure that you have the $ symbols. Set the format you want for ths condition.

You can use a similar formula for "L" as a second condition, but you will want > rather than < in the second half.

Hope this helps.

Pete

AIP Protected Files - Microsoft Community

kennethbetco say:

AIP Protected Files

Hi All,

Just wanna know if my AIP Protected document will be infected by a workstation that has virus or any malware?

And if the Document is not yet AIP Protected and has virus can I still labeled it or protect it with AIP?

THANKS!

sylvie liang msft say:

Hello KennethBetco,

For your concerns:

1. If my AIP Protected document will be infected by a workstation that has virus or any malware?

Based on my knowledge, Azure Information Protection (AIP) is a cloud-based solution that helps an organization to classify and optionally, protect its documents and emails by applying labels. The administrators of AIP can:

Encrypt the files to a specific set of recipients both inside and outside their organization.

Apply specific set of rights to restrict the usability of the file.

Decrypt content based on the user’s identity and authorization in the rights policy.

Hence, AIP doesn’t provide an ability to prevent virus or malware attack. For more details about AIP, you can refer to: The role of Azure Information Protection in securing data.

2. And if the Document is not yet AIP Protected and has virus can I still labeled it or protect it with AIP?

Based on my knowledge, virus could be a piece of executable code inserted into your document, as I mentioned above, AIP just assigns some labels to protect the access to the content. Thus, you can still label it, but AIP won’t remove the virus code from your documents.

I hope this can help. If you need any further assistance, please feel free to let me know.

Best Regards,

Sylvie
