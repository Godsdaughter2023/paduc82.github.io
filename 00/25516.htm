You can use the OpenArgs argument of the DoCmd.OpenForm method to pass arguments to the form as you open it. Then write code in the form's Open event to populate the controls you want prepopulated.
 
DoCmd.OpenForm "myform", , , , , , CStr(Me!txtProjectID) & "|" & CStr(Me!txtCurrentUser)

This is the code for GetNTUserName():

Private Declare Function GetUserName Lib "advapi32.dll" Alias _
    "GetUserNameA" (ByVal lpBuffer As String, nSize As Long) As Long
 
Public Function GetNTUserName() As String
    'Returns the current username from Windows NT Server.
    'Requires the GetUserName API.
    Dim nSize As Long
    Dim lReturn As Long
    Dim sUsername As String
   
    nSize = 255
    sUsername = String(nSize - 1, 0)
   
    lReturn = GetUserName(sUsername, nSize)
    If (lReturn > 0) Then
        GetNTUserName = Left(sUsername, nSize - 1)
    Else
        GetNTUserName = vbNullString
    End If
End Function

hansv mvp say:

You could create a table with one row and two columns: a narrow left-hand column for the name and a wider right-hand column for the text.

Enter a name in the first cell, then press Tab.

Enter the speech in the second cell. You can create as many paragraphs as you like.

To move to the next character, press Tab again. This will create a new row to enter the name etc. etc.

You can make the table borderless, so that you won't see that there is a table when you print the document.

suzanne s. barnhill say:

While you could use a table as Hans suggests, this is just as easily done with a hanging indent. You can create the indent by dragging the Hanging Indent marker on the ruler or by setting a specific indent in the Paragraph dialog (right-click, choose Paragraph, and then choose Hanging under Special in the dialog). Or you can press Ctrl+T once or twice to move the hanging indent to the built-in default tab stops (watch the ruler to see what happens when you do this).

Ideally, you should define a style that has this formatting and apply it to the dialogue, using a different style for stage directions, scene numbers, etc. The List series of built-in styles have hanging indents; you would need to increase the indent amount, however, as it is just 0.25" for each level.

jay arnott-myers say:

I believe I can fix this problem if UTF-8 encoding is used as I have come accross this before when Japanese characters were used. Hopefully this solution will be of some use to you. This is a common problem when using UTF-8.

First off, ensure that all of your HTML or PHP files are saved in UTF-8 format without BOM (byte order mark)?

You also have to make sure that your server is sending a UTF-8 header in the response headers.  I think (not sure) that ASO's default setup does that, but you will want to examine your response headers either in your browser (use Firefox Inspector if you are using Firefox) to make sure you are or verify using a web header viewer.  The response header character set declaration is much more important than doing it in the HTML (it is a good idea to do both).  If you are not sending a UTF-8 character encoding in the response header, you can do that using PHP's header() function or in htaccess.

Set header character set using PHP header:

header('Content-Type: text/html; charset=UTF-8');

Set character set using .htaccess:

AddDefaultCharset UTF-8

Make sure your MySQL server connection collation is set to UTF-8 (I think it should be by default on ASO).  Then for every text field inside your database tables ensure the collation/chacter set is set to UTF-8.  If you have mangled text inside your database, you will have mangled output.  Take a quick look at it using phpMyAdmin and see if you have character problems.  If you do, you may have to re-import your data or find a way to convert it.

Now that we got that out of the way, make sure you run a "SET NAMES" query before you run any other queries (not tested). 

mysql_query('SET NAMES utf8')

PHP recommends using the mysql_set_charset() function to set the character set instead of a SET NAMES query, but since you are doing things the old way it really doesn't matter.

That last step running the SET NAMES query you may want to try first if you know you are doing everything else mentioned above.
