Selecting Records from multiselect list box to populate a text box or - Microsoft Community

gettin by say:

Selecting Records from multiselect list box to populate a text box or another list box

I've seen this done on other Access applications but I'd like to learn how to do it.  The form has a list of items in a list box that the user can select and then click a button to populate a field next to the list box.  Similar to the way the list box wizard work for selecting fields.  I can figure out a way to have the form create records in a temporary table, but I think it is better to have the items in a virtual recordset.
ken sheridan say: The code to copy the two columns of the selected rows from one list box to another would be:

     Dim ctrl As Control
     Dim varItem As Variant
     
     Set ctrl = Me.lstFirst
     
     ' empty second list box's RowSource
     Me.lstSecond.RowSource = vbNullString
     
     ' loop through first list box's ItemsSelected collection
     ' and copy each selected item to second list box
     For Each varItem In ctrl.ItemsSelected
        Me.lstSecond.AddItem ctrl.ItemData(varItem) & _
            ";" & ctrl.Column(1, varItem)
    Next varItem

To remove a copied row is simply a case of deselecting the row in the first combo box while leaving the others selected and executing the code again.  This will be automatic if it's the first list box's AfterUpdate event procedure, or if it's the Click event procedure of a separate button you'd click the button again after revising the selections in the first list box.

However, as your ultimate aim is to have the selected rows in a table, why not simply add a Boolean (Yes/No) column to the table on which the first list box is based, and update the table on the basis of the selections with, using a Contacts table as an example:

    Dim ctrl As Control
    Dim strSQL  As String
    Dim varItem As Variant
     
    Set ctrl = Me.lstFirst
     
   ' first deselect all rows in table.
   ' if you want the selections to be cumulative, however,
   ' omit this operation
    strSQL = "UPDATE Contacts " & _
        "SET Selected = FALSE"
        
    CurrentDb.Execute strSQL, dbFailOnError
    
    ' update selected rows in table
    For Each varItem In ctrl.ItemsSelected
        strSQL = "UPDATE Contacts " & _
            "SET Selected = TRUE " & _
            "WHERE ContactID = " & ctrl.ItemData(varItem)
            
        CurrentDb.Execute strSQL, dbFailOnError
    Next varItem

Put this in the Click event procedure of a button.  You then can use a query, e.g.

SELECT *
FROM Contacts
WHERE Selected
ORDER BY LastName,FirstName;

rather than filling a separate table.  It is generally not necessary, nor advisable, to do the latter in a relational database, but to use the result set of a query like the above in place of a base table.  You are then guaranteed that there are no inconsistencies.  You can have a second list box with the selections from the first, using a similar query as its RowSource property:

SELECT ContactID,
FirstName & " " & LastName AS FullName
FROM Contacts
WHERE Selected
ORDER BY LastName,FirstName;
