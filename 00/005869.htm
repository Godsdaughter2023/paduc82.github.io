ken sheridan say: If I understand correctly I think what you want to do is not open the second form key of the first form's table?  It seems to be the only candidate for this, and your previous reply suggested this was the case.  If so then the code in the AfterUpdate event procedure of your combo box would be like this:

Select Case Me.YourComboBox
    Case "ENROLLED"
    DoCmd.OpenForm "YourSecondForm", _
        DataMode:=acFormAdd, _
        WindowMode:=acDialog, _
        OpenArgs:=(Me.ID1)
    Case " NOT ENROLLED"
    ' code to open another form goes here
End Select

In the Open event procedures of the second form you can set the DefaultValue property of its LineNo control to the TD1 value which was passed to the form, like so:

If Not IsNull(Me.OpenArgs) Then
    Me.LineNo.DefaultValue = """" & Me.OpenArgs & """"
End If

There must be a control bound to the LineNo field in the second form, but if you don't want it to show it in the form set its Visible property to False (No).  Note that the DefaultValue property is always a string expression regardless of the data type of the field to which the control is bound, so should be wrapped in literal quotes characters as above.

ken sheridan say: Mea culpa!  I missed a colon:

Select Case Me.YourComboBox

If OneDrive.exe cannot be started, please check the following two settings.

    1. Press (Windows key) + R to open the Run box.
    2. Type regedit and click OK.
    3. Navigate to HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Windows\OneDrive. Make sure DisableFileSyncNGSC is set to "0". (It is also OK if the registry doesn’t exist)

In the Local Group Policy Editor, in the folder list under Local Computer Policy, navigate to Local Computer Policy > Computer Configuration > Administrative Templates > Windows Components > OneDrive > Prevent the usage of OneDrive for file storage. Errata....  I wrote:
The YIELD function should take that into account.  But I think it presumes that the maturity date is on the anniversary of a coupon payment.

Consider the following details (please specify your own in a follow-up response):
Settlement date:       Nov 2, 2011
Maturity data:             Oct 5, 2021
Annual rate:               4%
Price:                          872
Redemption value:  1000

=YIELD(DATE(2011,11,2), DATE(2021,10,5), 4%, 87.2, 100, 2, 0)

results in about 5.7065%.

Just use YIELD.

I wrote:
I don't understand why that last term in the PRICE formula is a factor in the first place.

It accounts for interest presumably accrued between the previous coupon period and the settlement date.  This is the so-called "dirty price" model (click here) [1].  In other words, the bond is presumed to have been issued on or before 5 Oct 2011, a previous coupon anniversary date based on the maturity date.

I wrote:
Using IRR, I get about 5.6548%.

Of course, my IRR model is flawed because I do not have equally-spaced cash flows.  The first cash flow is 2 Nov 2011, the second cash flow is 5 Apr 2012, and all subsequent cash flows are 6 months apart.

When I use XIRR as I should have, accounting somewhat for the different models, I do get very close to the same result as YIELD.  But that is really coincidence since XIRR and YIELD use different discounting models.

I wrote:
you could dispense with using YIELD and use IRR instead

Wrong!  Arguably, we would use NPV and IRR instead of PRICE and YIELD when we want the "clean price" model.  But only when the maturity date is an anniversary of the settlement date, and both align with coupon payment dates.  However, in that case, PRICE and YIELD should produce the same results.

john sr wilson say:

The macro (as written) would NOT distort the image and so only set the width to 6.17. If you need to set the height then there will be distortion if the images have different aspect ratios. If this is what you need adapt the code by inserting before the two lines that set the height and width and after with oshp

.LockAspectRatio=False

Sub fix_Album()
Dim oshp As Shape
Dim osld As Slide
Dim iCount As Integer
For Each osld In ActivePresentation.Slides
For Each oshp In osld.Shapes
If oshp.Type = msoPicture Then
iCount = iCount + 1
With oshp
.LockAspectRatio = False
.Height = 72 * 4.63
.Width = 72 * 6.17
.Top = ActivePresentation.PageSetup.SlideHeight - .Height - 20
End With
Select Case iCount
Case Is = 1
oshp.Left = ActivePresentation.PageSetup.SlideWidth / 2 - oshp.Width - 20
Case Is = 2
oshp.Left = ActivePresentation.PageSetup.SlideWidth / 2 + 20
End Select
End If
Next
iCount = 0
Next
End Sub
