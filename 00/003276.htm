Chip Pearson wrote:

A function can return a value to more than one cell; this is called an array function. The functions is entered simultaneously into the cells to which it is to be returned. Suppose you have a function like

Function Swap(X As Long, Y As Long) As Long()
    Dim Arr(1 To 2) As Long
    Arr(1) = Y
    Arr(2) = X
    Swap = Arr
End Function

This returns an array of two elements, just swapping their position. Select, say, C1:D1, type =SWAP(A1,B1) and press CTRL SHIFT ENTER. This array enters the formula into C1:D1 and the result array of the Swap function puts its values into those cells. You cannot, however, have a formula that modifies any cell that does not contain the formula. For example,

Function Swap(X As Long, Y As Long) As Long()
    Dim Arr(1 To 2) As Long
    Range("F1").Value = 123
    Arr(1) = Y
    Arr(2) = X
    Swap = Arr
End Function

will not work because it attempts to change cell F1, and VBA prohibits such action. By default, an array results into an array that is one row spanning several columns, C1:D1 in the example above. To get the results in a column spanning several rows, you have to transpose the array. So, select A4:A5, type =TRANSPOSE(Swap(A1,B1)) and press CTRL SHIFT ENTER.

A VBA procedure called from a worksheet cell cannot change anyting in the Excel environment, including other cellls. It can only return values to the cells from which it came. However, in a broader sense, VBA can modify cells directly:

Range("A1").Value = 1234

Cells(2,"C").Value = MB

puts the value MB in cell C2.

If you want to avoid the TRANSPOSE() in the worksheet formula, something like this seems to work:

Function Swap2(X As Long, Y As Long) As Long()
  Dim Arr(1 To 2, 1) As Long
  Arr(1, 0) = Y
  Arr(2, 0) = X
  Swap2 = Arr
End Function

The formula will now return a 2 rows by 1 column array and =SWAP2(A1,B1) can be array entered directly in A4:A5

If you have a range of cells that you want to loop through you can use a For Each... In... Next loop.  VBA is made up of many collections of objects.  There are, for example, all the Sheet objects in the Workbook.Sheets collection, and all the Shape objects drawn in a worksheet appear in the Worksheet.Shapes collection.

Now a Range is merly a collection of cells.  A cell is itself a range, so a range is really a collection of (single-cell) ranges.  This is how we can use this:

Dim rng As Range
Dim c As Range

Set rng = Range("A1:D1")
For Each c In rng
    'Do stuff with the c object
    c.Value = c.Column
Next c

This will loop through each cell in the range "rng".

You can make use of the "c" object like this:

c.Value = "aString"

or

c.Formula = "=A1+1"

And you can access the current coordinates of the c object by using c.Row and c.Column.

Note, I call it "c" as it represents a cell, but it could be called anything.

It depends.

If you did an upgrade from Windows 8.1 to Windows 10 Beta, you should be able to upgrade to RTM without having to do anything since the key is stored.

If you did a clean install, you have two options:

robxdev say:

You can Create a (Windows) System Image which is a Snapshot of everything on your hard-drive, including hidden partitions and boot information, Windows files, Apps and Programs and all personal folders and files

If it is a clean install, or you don't have much on your hard drive, it should fit on 3 x blank DVDs

If not, an external USB hard drive or 2nd internal drive is used to store the Image Folder (eg F:\WindowsImageBackup)

Go to Control Panel > System and Security > Back up and Restore (Windows 7)

Check W10 is Activated in Control Panel > System and Security > System  before creating the image

[MS decided to get rid of (deprecate) this feature, but I saw somewhere it may be reprieved in W10]

Restoring an image can be done from the Repair Options on a W10 Install DVD or USB Flash

or

Selecting Other > Advanced > Troubleshoot at boot time

or

F8 at Boot Time (hit and miss)

See here

=-=-=

Practically, the above functionality has some limitations (eg It's tricky to keep multiple System Images or Mount a System Image to just grab a few files or folders without doing a Full Restore)

Plenty of 3rd party apps can also make System Images (and a Boot method to Restore in emergency)

Macrium Reflect Free is widely respected, and popular, and has lots of very useful, additional functionality

=-=-=

A scenario for someone wanting to rebuild might go as follows:

Create a System Image of the current system, before doing anything

(check Boot to System Image recovery works and the System Image Folder/File is there, but don't actually proceed)

[Clean install W7(withSP1)/8/8.1 (if desired), Activate and make another System Image]

Update/Upgrade/Install W10. Check Activated and make another System Image

Make sure the Images have sensible Folder/Filenames so they are easily identifiable at a later date

(Note: If using Windows "Create a system image", you would rename WindowsImageBackup before making a new Image, because only one is 'allowed')

The end result could, for example, be 3 folders

F:\WindowsImageBackup-BeforeUpgrade

F:\WindowsImageBackup-W7or8or8.1-Clean

F:\WindowsImageBackup

(the last one in effect would be the W10-Clean Image. The name can't be changed if you ever need to Restore it.

If you want to Restore the others, they have to be renamed first via Command Prompt)

So, that could be

rename   "F:\WindowsImageBackup"   "F:\WindowsImageBackup-W10-Clean"

rename   "F:\WindowsImageBackup-W7or8or8.1-Clean"   "F:\WindowsImageBackup"

3rd Party apps handle Multiple Images without kerfuffle

=-=-=

Regular System Images (weekly, fortnightly, monthly) are very useful as part of a Backup Strategy.

Keep the early ones, and the latest couple, say.

Delete Intermediate ones if the hard drive starts running out of space

=-=-=

I don't know what the final License Key situation is going to be.
