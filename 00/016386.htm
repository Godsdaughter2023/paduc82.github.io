suzanne s. barnhill say:

Perhaps someone else will come up with a brilliant solution, but given the amount of variation you're dealing with here, I think I would settle for this semi-automatic approach.

  1. Use SEQ fields for the numbering, one sequence for each paragraph level. Although SEQ fields don't update dynamically (when you add or remove a list item), they update automatically when fields are updated (with F9 or by printing). They also have the advantage, very difficult to achieve with auto numbering, that you can repeat a number by adding the \c switch to the field.
  2. You can define three paragraph styles, with the desired indents, for the three numbering levels. You'll have to add tabs (if desired) after the numbers manually.
  3. The TOC is going to be very manual as well; you'll have to use TC fields. I'm assuming that only the sidehead text will be included, not the numbering (since it would not be continuous).
  4. Since your paragraph styles will have an outline level of Body Text, they won't be included in the TOC automatically, so that won't be a problem. If you have any built-in headings and don't want them in the TOC, you'll need to edit the TOC options to omit them. If you do want them included and want these sideheads included at a lower level, then you'll create your TC fields accordingly.
  5. Inserting the TC fields is not really that difficult. You select the text you want included and press Alt+Shift+O. Select the level you want, and click Mark. The disadvantage to this approach is that the marked text does NOT appear in the Navigation pane as a heading, but it will appear in the TOC.
  6. A caveat: You can't use either of the "automatic" TOC building blocks in References | Table of Contents. Instead, you have to use Insert Table of Contents and, in the Table of Contents dialog, click Options and check the box for "Table entry fields." If there will be no entries other than the TC fields, you can deselect the boxes for "Styles" and/or "Outline levels."

suzanne s. barnhill say:

SEQ fields are entirely different from autonumbering using multilevel lists. There are ways to easily add a or A to a number in a multilevel list but no easy way to repeat a number in order to add the a/A. That's why, after writing for some time a description of a complex multilevel list format, I realized that that would not work and started over with the SEQ field idea.

 

You can find a Help article about the SEQ field at http://office.microsoft.com/en-us/word-help/field-codes-seq-sequence-field-HP010255520.aspx?CTT=5&origin=HA102110133, but here's a primer on how they work:

 

Each sequence has an identifier. So you could have three sequences with the identifiers L1, L2, L3. Although you can construct the fields manually, you may want initially to go through the Insert | Text | Quick Parts | Field route. Select Seq in the list and click the Options... button to get the General and Field Specific Switches. The General switches allow you to format the field; you won't need one for simple 1, 2, 3 numbering, but you will need one for a, b, c. For the (1), (2), (3), I'm afraid you'll have to add your own parentheses manually. With the numeric picture switch, you can also add your leading zeros as needed (see http://office.microsoft.com/en-us/word-help/insert-and-format-field-codes-in-word-2010-HA101830917.aspx?CTT=5&origin=HA102110133#BM8).

 

The field-specific switches are what allow you to repeat a number or restart the numbering. I would suggest that, for each level, you create two AutoText entries, one to restart the numbering at 1 (or a) and one to repeat the previous number. The advantage of an AutoText entry is that, when you insert it, the number updates automatically (this is not the case if you copy/paste a SEQ field). The AutoText entry can also include the parentheses or other punctuation.

 

Once you've got the numbers, you have complete freedom about content. You'll have to format the sideheads manually (you could use character styles for these if desired), but you have complete freedom about what you include in the TOC using TC fields (see http://office.microsoft.com/en-us/word-help/field-codes-tc-table-of-contents-entry-field-HA102017492.aspx?CTT=5&origin=HA102110133).

 

I will not categorically state that you could not do this with a multilevel list and style separators, but I personally would go crazy trying. Although this method requires a bit of work to set up initially, I think you would find it less trying in execution. And you can certainly combine this with a multilevel list if there are levels that are absolutely consistent (no As and Bs, always or never a heading).

How do I refresh a Pivot Table by clicking on the sheet tab (by writing macro commands)?

Danie_007

hansv mvp say:

Right-click the sheet tab.

Select "View Code" from the context menu.

Copy the following code into the worksheet's code module:

 

Private Sub Worksheet_Activate()
    Dim pvt As PivotTable
    For Each pvt In Me.PivotTables
        pvt.RefreshTable
    Next pvt
End Sub

 

Switch back to Excel. Make sure that you enable macros when you open the workbook.
