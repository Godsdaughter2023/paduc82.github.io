madenna say:

Using VBA to insert new record

I'm trying to increase my VBA knowledge and skills, which at this point are pretty non-existent beyond a few small slivers of code that I use frequently.  So I figured I'd start by trying to alter the SQL I have in an append query into VBA code.  I feel like I'm close, but it's not working.  I don't get any errors, but the record is not created.  Any guidance would be greatly appreciated.  Here's what I have:

     Dim vContactID As Integer
     Dim vCompanyID As Integer
     
     vContactID = Forms!frmContactInfo!ContactID
     vCompanyID = Forms!frmSrchCompany!CompanyID
     
     strSQLa = "INSERT INTO tblCompanyContacts (CompanyID) SELECT tblCompany.CompanyID FROM tblCompany" _
     & "WHERE tblCompany.CompanyID =" & vCompanyID & ""

     & "WHERE tblContacts.ContactID =" & vContactID & ""
     
dirk goldgar say:

1. You need a space in the SQL string between "FROM tblCompany" & "WHERE".

 

2. If you execute both of those SQL statements, you'll be attempting to insert two separate, incomplete records, rather than one complete record.  If I understand what you're trying to do, you need to execute a single INSERT statement that provides values for both CompanyID and ContactID.

 

3. If you have the values to be inserted, you can provide them directly in a VALUES clause; you don't need to look them up with a SELECT clause.

 

4. I don't see statement that actually executes either SQL statement, but maybe you just didn't post it.

 

I think you need something like this:

 

'------ start of example code ------

Dim strSQL As String

Dim lngContactID As Long
Dim lngCompanyID As Long
lngContactID = Forms!frmContactInfo!ContactID
lngCompanyID = Forms!frmSrchCompany!CompanyID
strSQL = _
    "INSERT INTO tblCompanyContacts (CompanyID, ContactID) " & _
    "VALUES(" & lngCompanyID & ", " & lngContactID & ")"
 
CurrentDb.Execute strSQL, dbFailOnError
'------ end of example code ------
 
That assumes that CompanyID and ContactID are numeric values, which you did imply,  I changed them from Integer to Long, because most "ID" fields are originally autonumbers, which are Long integers, not short ones.

Why does 'typed' Date & Time value not equal same Date - Microsoft Community

francis hayes say:

Why does 'typed' Date & Time value not equal same Date & Time value when 'filled' with Fill Handle?

If I enter a 02/15/12 1:00 AM in cell A5 and the same date/time in cell B5 and enter =A5=B5 in another cell I get TRUE as the result. However, if instead, enter I enter 02/14/12 1:00 AM in B4 and drag the fill handle down to give me 02/15/12 1:00 AM in B5, the result of my =A5=B5 formula is now FALSE. Then if I enter the formula =A5-B5, I get 5.78657E-08 as the result. Can someone please explain?
gord dibben say:

Dates and times are no more than numbers in Excel.

 

Dragging numbers with many decimal places causes very tiny changes.

 

The first date/time I entered manually for 02/15/2012 1:00:00 was  40954.0416666667

 

When i did the drag down 02/15/2012 1:00:00   was  40954.0416666088

 

That explains the weird results you see.

 

The reason is floating point arithmetic used by Excel

 

http://blogs.office.com/b/microsoft-excel/archive/2008/04/10/understanding-floating-point-precision-aka-why-does-excel-give-me-seemingly-wrong-answers.aspx

 

 

Gord

sheeloo say:

Excel stores dates as real numbers counting from 1-Jan-1900 which is stored as 1. This means that 24 hours correspond to 1.

 

When you use 02/15/12 1:00 AM the number stored has a fractional part which is not exact so when you drag down you get a slightly different number. You can see this by converting your cells to decimals.

 

A5 would be something like 40954.0416666667 and B5 (after dragging) will be like hence they do not match.

 

If you use

=ROUND(A5,2)=ROUND(B5,2)

you will get TRUE
