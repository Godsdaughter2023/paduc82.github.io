Auto populating tabs in Excel with duplicate dates

·         Create an excel sheet and rename the first sheet as “Temp”

·         Enter start date in cell “B5”

·         Enter end date in cell “A5”

·         Enter the following formula in cell “C5”      =A5-B5+1

·         Enter the following formula in cell “D5”      =A5

·         Enter the following formula in cell “A6”     =TEXT(D5,"dd-mmm-yyyy")

 

Now click on the Developer Tab and click on View Code, then copy paste the code below into the VBA editor.  Close the editor and click on Macro on the Developer Tab and run the Dtpopulatemacro

 

To display the Developer Tab in Excel, do the following

 

·         Click the Microsoft Office Button , and then click Excel Options.

·         Click Popular, and then select the Show Developer tab in the Ribbon check box.


' Code Starts here

Sub Dtpopulate()

 

Dim S As Integer

Dim X As Integer

S = Sheets("Temp").Range("c5").Value

 

For X = 1 To S

newname = Sheets("Temp").Range("a6").Value

Sheets.Add Type:=xlWorksheet

ActiveSheet.Name = newname

Sheets("Temp").Range("d5").Value = Sheets("Temp").Range("a5") - X

Next X

 

End Sub

myall_blues say:

Hi

You cannot have worksheets (tabs) with the same name. Your code also required an additional loop to generate the multiple instances for each date.

Try the following code. It creates four instances of each tab with the same date, but adds a number to the end of each name as (1), (2) etc to distinguish them.

Option Explicit
Sub Dtpopulate()
    Dim S As Integer
    Dim X As Integer, Y As Integer
    Dim newname As String
    S = Sheets("Temp").Range("c5").Value
    For X = 1 To S
        For Y = 1 To 4 '<-- Change to match number of sheets required for each date
            newname = Sheets("Temp").Range("a6").Value & "(" & Y & ")" '<--Adds (1) etc to each sheet name
            Sheets.Add Type:=xlWorksheet
            ActiveSheet.Name = newname
        Next Y
        Sheets("Temp").Range("d5").Value = Sheets("Temp").Range("a5") - X
    Next X
End Sub

Route internal mail through on prem and external through O365 - Microsoft Community

andrew o'young say:

Route internal mail through on prem and external through O365

We have a remote customer who has an O365 tenancy and an Exchange 2016 on premises environment.

Due to their satellite connection we need internal mail to route internally to save bandwidth and for external mail to go out via O365. 

Added to this, they are getting nightly power outages; We would like to see mail route via O365 when the on-prem is not contactable.

a. user say:

Hi Andrew,

 

If you have a solution to your power problems, you may manage your mail flow based on MX record points to Office 365 and Office 365 filters all messages. Under this scenario, emails are sent internally for your on-premise mailboxes, and any emails sent to Internet are using Office 365 by connectors. You need to point your MX record to Office 365 and add both your Office 365 address and on-premises servers to SPF records. Please check the following article for the details: https://technet.microsoft.com/en-us/library/e1da5f2f-c732-4010-85c9-878b2cef3fb3(v=exchg.150)#scenario1

 

But if you have nightly power outages which breaks down your Exchange server, this may not be a suitable method as daily changing mail flow would not be a workable solution considering its complexity. Our suggestion is to using Office 365 instead of a hybrid mode. As an advantage of cloud service, Office 365 can easily deal with the disaster issue with its multiple database around the world.

 

Feel free to let me know if you have further concerns.

 

Regards,

Leo
