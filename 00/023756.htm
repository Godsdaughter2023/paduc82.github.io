In case you want to return the last value from column A where column B = M2, then enter below formula in cell N2:

=LOOKUP(2,1/(B2:B10=M2),A2:A10)

In case you want to return ALL values from column A (ie. multiple values) where column B = M2, then enter below formula as an array formula (Ctrl+Shift+Enter) in cell O2 & copy down:

=IFERROR(INDEX(A$2:A$10,SMALL(IF(B$2:B$10=M$2,ROW(A$2:A$10)-ROW(A$2)+1),ROW(1:1))),"")

For returning the second match, a non-array option:

There are 2 options where you may use a normal (non-array) formula:

Formula in cell N2:

=INDEX(A2:A10,AGGREGATE(15,6,(ROW(A2:A10)-ROW(A2)+1)/(B2:B10=M2),2))

Formula in cell O2:

=INDEX(A2:A10,MATCH(M2,INDEX(B2:B10,MATCH(M2,B2:B10,0)+1):B10,0)+MATCH(M2,B2:B10,0))

Both are non-array formulas and will return the corresponding value from column A for 2nd match of M2 in column B.

poppyyang_msft6299 say:

Hi Ipodtrip,

Generally, when user B have full access to user A, user B can open user A’s mailbox immediately.
User cannot open the mailbox which he has full access to via “open another mailbox”, it means the permission hasn’t been synced to Exchange Online server side, this may due to different reasons such as  network environment or some issue with the permission.
To make sure the permission sync to Exchange Online server side immediately, I would suggest you use the PowerShell to give the permission via the steps below:
a.Connect to Exchange Online using remote PowerShell  
b. Run the command using windows Powershell to check if the permission exists in Exchange Online server:
Get-MailboxPermission –Identity “*** Email address is removed for privacy ***” | ft user,accessrights
c. Run the following cmdlet to add full access permission of the mailbox.
Add-MailboxPermission -Identity "MailboxIdParameter" -User "username" -AccessRights FullAccess


If anything is unclear, please feel free to post back.

Thanks,
Poppy
