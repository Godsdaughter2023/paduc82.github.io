Mailing List Manager (MLM)

ken sheridan say:

I suppose this is the question: might a sort of transitive-type of relationship exist between three tables such that a value in table A can be expressed in a foreign key field in table C (I realize that it can be in table B)?

No, the relationship type in your case is a binary one, i.e. it is modelled by a table which resolves it into two one-to-many relationship types.  A relationship type might also be ternary, where the table which models the relationship type does so by resolving it into three one-to-many relationship types, quaternary etc.  In each case none of the referenced tables, be they two, three, four or more in number, contains a foreign key referencing any other referenced table.  Only the table which models the relationship type contains the foreign keys.

scottgem say:
Lets make the question a bit more general. How does one retrieve the descriptive value when using a coded foreign key?

By descriptive value, I mean the textual value, by coded key, I mean using a numeric representation or alpha numeric code that represents the descriptive value (i.e. an autonumber).

This is standard operating procedure. There are two ways to do it in Access. The first is to include the lookup table in your query joined on the foreign key to the lookup table's primary key. the second is to use a Dlookup() function. A query is preferable. For example:

SELECT EntityName, Lastname & ", " & Firstname As Contact
INNER JOIN ON tblEntity.EntityID = tblEntityContact.EntityID, tblLname.LnameID = tblContact.LnameID, tblFname.FnameID = tblContact.FnameID
FROM tblEntity, tblcontact, tblFname, tblLname;

Note, the above is off the top of my head and may not be syntactically correct, but it should give you the ID. Let me give another more practical example:

SELECT EntityName, OrderNum, OrderDate
INNER JOIN on tblEntity.EntityID = tblOrder.EntityID
FROM tblEntity, tblOrder

The above would be very typical. An Order table would include the EntityID as a foreign key to indicate the entity doing the ordering. But in a query listing orders, you would want to see the Entity name.

Or:
DLookup("[EntityName]","tblEntity","[EntityID=" & EntityID)

pauli o say:

Yes, I use same dism commands, but in loop that patch all desired windows versions.

I try to avoid this kind of 3rd party applications for security.

- Create temporary directory WIM
- Download KBxxxxxx from Windows Update Catalog to folder WIM (renamed Update1.msu)
- Copy \Sources\install.wim from ISO or USB-installer to folder WIM
- Copy/paste Script between **** to notepad and save to folder WIM as ZUP.bat
- Insert "REM" to any first 7 lines if you want to keep that version and change "Replace this" from line: FOR /L %%I IN (1,1,"Replace this")
  to match number of images yo want to save. I save only Home and 10 Pro -> (1,1,2).
- Make sure you have folder WIM with install.wim, ZUP.bat and Update1.msu before start update script!
- Open CMD as Administrator and go to directory WIM and Start ZUP.bat
- When finished see WIM\Log.txt (ServicePack Level should be updated)
- Expected runtime in PC with i5 CPU, 7-10 minutes for each saved image.

*******************************************************************

@echo off
REM Dism /Delete-Image /ImageFile:install.wim /Name:"Windows 10 Home"
        Dism /Delete-Image /ImageFile:install.wim /Name:"Windows 10 Home N"
        Dism /Delete-Image /ImageFile:install.wim /Name:"Windows 10 Home Single Language"
        Dism /Delete-Image /ImageFile:install.wim /Name:"Windows 10 Education"
        Dism /Delete-Image /ImageFile:install.wim /Name:"Windows 10 Education N"
REM Dism /Delete-Image /ImageFile:install.wim /Name:"Windows 10 Pro"
        Dism /Delete-Image /ImageFile:install.wim /Name:"Windows 10 Pro N"

 md MNT
   FOR /L %%I IN (1,1,2) DO ( 
   Dism /Get-ImageInfo /ImageFile:install.wim /index:%%I
   Dism /Mount-Image /ImageFile:install.wim /index:%%I /MountDir:MNT /CheckIntegrity
   Dism /image:MNT /add-package /packagepath:Update1.msu
   Dism /Unmount-Image /MountDir:MNT /commit /CheckIntegrity
   Dism /Export-Image /SourceImageFile:install.wim /SourceIndex:%%I /DestinationImageFile:New_install.wim /CheckIntegrity

   Dism /Get-ImageInfo /ImageFile:install.wim /index:%%I >>Log.txt
 )
 rd MNT

 del install.wim

 ren New_install.wim install.wim

*******************************************************************

If script crach you have to make unmount manually: Dism /Unmount-Image /MountDir:MNT /discard

When script is completed result is New_install.wim with updated images of Home and Pro and it can be copied to USB replacing old install.wim.

To create bootable iso copy all Windows 10 files with new install.wim to folder W10 and run this command in CMD:
(This case UUPtoISO must be installed, note drive letters and paths.)

D:\UUPtoISO\bin\cdimage.exe -m -o -u2 -udfver102 -bootdata:2#p0,e,bD:\W10\boot\etfsboot.com#pEF,e,bD:\W10\efi\microsoft\boot\efisys.bin D:\W10 D:\Win10x64.iso

EDIT 4/8: Added Dism /Export command to reduce size of install.wim.

-g = Uses the Universal Coordinated Time (UCT) for all files instead of the local time

When or where you can see difference /w or /wo -g parameter?

It does not really matter but

With -g the time stamp files in the ISO are dated with UCT Universal Coordinated Time.

Without -g the time stamp files in the ISO are dated with your local Time. (example US time zone - PDT Pacific Daylight Time)

https://www.timeanddate.com/time/aboututc.html

UTC – The World's Time Standard
Coordinated Universal Time (UTC) is the basis for civil time today.

This 24-hour time standard is kept using highly precise atomic clocks combined with the Earth's rotation.

A Standard, Not a Time Zone
UTC is the time standard commonly used across the world.

The world's timing centers have agreed to keep their time scales closely synchronized - or coordinated -

therefore the name Coordinated Universal Time.
