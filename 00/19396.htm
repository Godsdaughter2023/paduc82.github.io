Access- Redefine Rounding function - Microsoft Community

kywilson25 say:

Access- Redefine Rounding function

How can I redefine the Round function to round 5 up.   Round(.5)=1 not 0 or Round(2.5)=3 not 2

This Macro will have to be able to handle a lot of data. I have limited ability with Macro.

Thanks,

ken sheridan say: By default Access uses the Banker's Rounding algorithm (round-half-even).  The following function on the other hand will always round from zero:

Public Function RoundFromZero(dblVal As Double, dblTo As Double) As Double

    Dim intUpDown As Integer
    Dim lngTestValue As Long
    Dim dblTestValue As Double
    Dim dblDenominator As Double
    
    If dblVal < 0 Then
        intUpDown = 1
    Else
        intUpDown = -1
    End If
   
    dblDenominator = intUpDown * dblTo
    dblTestValue = dblVal / dblDenominator
    lngTestValue = Int(dblTestValue)
    RoundFromZero = intUpDown * lngTestValue * dblTo
 
End Function

Paste the function into a standard module in the database, and save the module under a name which does not conflict with that of any other object, e.g. mdlRounding.

To round to an integer, 1 is passed into the function as the dblTo argument, e.g. RoundFromZero(2.5,1) will return 3.  Negative values are also rounded away from zero, so RoundFromZero(-2.5,1) will return -3.

For an outline of the different rounding schemes in common use see:

http://eetimes.com/design/programmable-logic/4014804/An-introduction-to-different-rounding-algorithms

hansv mvp say:

Shouldn't the formula in C1 be =(A1-B1)/B1 ? The formula you mention, =(A1-B1/B1) is equivalent to =A1-1.

You need a bit of VBA code to do what you want. This means that you must save the workbook in a macro-compatible format: .xlsm, .xlsb or .xls, but NOT .xlsx.

And users will have to enable macros when they open the workbook.

Right-click the sheet tab of the sheet with the formula in cell C1.

Select "View Code" from the context menu.

This activates the code module associated with the worksheet.

Copy the following code into the module:

 

Private Sub Worksheet_Calculate()
    Range("D5").EntireRow.Hidden = (Range("C1").Value > -0.05)
End Sub

 

Press Alt+F4 to close the Visual Basic Editor.

That's it!

Yammer SSO

How it works
SSO allows users to sign in with the same set of credentials they use for other applications in your environment. It also lets an enterprise control authentication for Yammer and can enable scenarios such as two-factor authentication. To set up SSO for your Yammer network, also known as federation, you provide various pieces of information to Microsoft.

When you enable SSO, you change the authentication process for Yammer. When users attempt to authenticate at Yammer.com, they are redirected to your IdP. This is a server or service that you use for authentication. When a user is successfully authenticated, that user is redirected back and signed in to Yammer.com.

Although some IdPs might support automatic sign-in so that users don't have to enter credentials, this isn't universal. If you want to avoid users having to type their passwords, select an IdP that supports this capability. Yammer operates independently of your application.

If you wish to implement Yammer SSO, you must submit a service request from your Office 365 Dashboard, Please let me know if you have other concerns.

jonathan lee msft say:

Hi Paul,

Thank you for bringing your concern to our attention. We understand how important for users to be able to access Yammer. We are here to help.

Office 365 sign-in for Yammer is a feature for Microsoft Office 365 and Yammer that allow users to sign in to Yammer using Office 365 credentials. When this feature is enabled, users will see the Office 365 suite navigation bar on the top of Yammer network pages. This will allow users to seamlessly sign in to Microsoft OneDrive, Microsoft SharePoint sites and Microsoft Office Outlook without being prompted for passwords.

Users without License

In this scenario, a user was not assigned a license when it was created. When the user is going to Yammer to login, the user is redirected to Office 365. After being authenticated, the user is redirected to Yammer and gets the following error message “That didn’t work, Something went wrong when we tried to sign you into Yammer with your Office365 Credentials”

In general, users do not need to be assigned a license in order to use their Office 365 credentials to log in to Yammer, except for the first logon. If the user’s first login occurs while the user does not have permissions, he will not be able to log in. Adding the additional licenses does not solve the issue; an additional mapping process is required in order to update the user’s status. 

How to fix it:

1. Check current license status - log in to Office 365 portal with global administrator user, click Admin and choose Office 365. Under the left menu, click Users. In the Users page, locate the user and click on the name. On the user menu that will be shown on the right side of the screen, click license. 
2. Update license to include all services, click Save.
3. Click Details tab in the left menu. On the details page, update the user’s email address (for example, change the domain, the drop-down) and then click Save. This will initiate the mapping process.

4. At this point, the user will be able to login to Yammer. An admin can go back and remove the user’s licenses.
