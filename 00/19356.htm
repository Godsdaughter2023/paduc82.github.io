greginmich say:

I've  just started my computer to find out that Windows Defender did not automatically scan (quick scan) and therefore the laptop was not protected. I thought Defender scanned automatically and I didn't have to worry about the scans any more.  Can anyone shed any light on the subject. ??

Windows Automatic Maintenance triggers the Windows Defender Scheduled Scan task (a Quick Scan) on a daily basis – but since Automatic Maintenance only runs when you’re not using your PC (during system idle states) this scan will only run consistently if you leave the PC unattended for 15 minutes or so every day:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-10-how-do-i-set-windows-defenders-scan/f399bb87-37c9-4a74-bb35-1d7e3d3c2436

The Microsoft method for scheduling a scan (cited above) gets lots of recommendations, and lots of helpful votes from people who’ve never tried it, but adding a trigger to the Windows Defender Scheduled Scan task (as per the instructions) really isn’t very helpful at all – because the WDSS task is already being triggered every day by Automatic Maintenance (without any scheduling). Moreover, adding a trigger to the WDSS task just creates a clone of the run-only-on-idle Automatic Maintenance scan, so the scan will get cancelled (not paused) every time you start using your PC and then have to wait for the next idle state to be restarted (not resumed).

So there's generally no need for scheduling a Quick Scan. Automatic Maintenance does automatic Quick Scans – and if the Automatic Maintenance scan isn't working right, then you probably don't want to clone the Automatic Maintenance scan by adding a trigger to the WDSS task. However, an old-fashioned type of scheduled scan (the kind that actually starts on time and runs without interruptions) might be helpful in this situation, and you can get that by scheduling an independent task with the Create Basic Task Wizard:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-i-set-windows-defender-to-a-custom-auto/ec19aab4-10ce-4f41-9990-343056fd104f

The Create Basic Task Wizard (in Task Scheduler) is the scheduling app for Windows, and it’s the standard way of scheduling anything, including Windows Defender scans. This app creates scheduled scans that run normally, and it also puts the full set of scheduling options at your disposal – including the option to schedule a Full Scan:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02  

If the problem turns out to be deeper than just not taking a coffee break that’s long enough for the Automatic Maintenance scan to finish, then you’ll find some troubleshooting tips and workarounds in this thread:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender/018e1a51-b7c5-4323-b73e-8a498238e914

I believ the issue you are having is not with the format (plain text vs HTML) or source (incoming vs forward) of the email. It is simply with the Zoom factor.

 Just hold the mouse over the body of an incoming mail window and rotate the mouse scroll window while holding the Ctrl key down. The Zoom setting is remembered from use to use so if you change it, it will continue to be changed until changed again.

Are you restarting numbering in each chapter? If you have a section starting with page 1, Word will force an Odd Page break.

When you do need to change a break, don't try to do so by deleting the break and reinserting it. Instead, with the insertion point in the relevant section (the one after the break), go to the Layout tab of Page Setup and change the section start type.
