The gist of what Dana was saying is:  since the third-party calculator uses monthly data, we can use Excel IRR to calculate the monthly IRR, which we can annualize with the formula (1+IRR(...))^12 - 1.

Effectively, I did the same thing using Excel XIRR.  The advantage of using Excel XIRR is:  (1) it returns an annual IRR directly; and (2) it is the function to use if you have the actual dates for external cash flows, so we no longer need to estimate irregular cash flows as monthly events, like the third-party calculator does.

Bill wrote:

My original question should have been:  Since I need to compare returns with mutual funds and indexes, how do you compute time-weighted returns that have deposits and withdrawals through out a year?

Your original question indicated that you want to calculate the compound annual growth rate (CAGR).

Since your original example covers one year, the TWR is the CAGR.  But generally, the CAGR is

(1+TWR)^(12/months) - 1.

For example, if your data covers 2 years, the CAGR for the 2-year TWR is (1+TWR)^(1/2) - 1.  Note that

1/2 = 12/24.

In theory, if your data covers 9 months, the CAGR for the 9-month TWR is (1+TWR)^(12/9) - 1.

However, my recollection (perhaps incorrectly) is:  for less-than-one-year data, it is common practice to report the YTD simple rate of return, not an annualized rate.

ken sheridan say: If the combo box is an unbound control, while you can set the DefaultValue property of an unbound control, it's much the same as assigning an initial value to it, so in the form's Load event procedure you can put:

    Me.Disposition= Me.Disposition.ItemData(0)

If it's a bound control then you could put the following in the form's Load event procedure rather than setting the property in the properties sheet:

    Me.Disposition.DefaultValue = """" & Me.Disposition.ItemData(0) & """"

Note that the DefaultValue property is always a string expression, regardless of the data type of the column to which it is bound, hence the literal quotes.

Excel, using two sheets. References not updating after Sort - Microsoft Community

nigelzz say:

Excel, using two sheets. References not updating after Sort

I have a list of names on sheet 2. I reference these names from sheet 1 using = Sheet2!Xn.  When I sort the list on sheet 2, the references from sheet 1 do not get updated to follow the same name, but now reference a new name which is in the same point in the list.
bob jones aka: cybertaz say:
Well, let me see if I can spell this out a bit more clearly :-)

First, when you sort a column of data it's only the content of the cells that is rearranged. The cells are fixed in place... They do not move, so A1 remains A1 even though the content is different after sorting.

What you're using on sheet 1 begins with an equals sign (=), so Excel considers it to be a formula [specifically, a formula reference or formula link] whether you consider it to be one or not :-). Also, Absolute v. Relative cell referencing has no bearing on this... IOW, it doesn't matter whether the formula is written as =Sheet1!$A$1 or as =Sheet2!A1.

The formula refers to a specific cell on sheet 2, so whatever appears in that cell on sheet 2 is what will appear in the cell containing the formula link on sheet 1 -- IOW, let's say the formula on sheet 1 is =Sheet2!A1 & cell A1 on sheet 2 contains the name John. The formula cell on sheet 1 will display John.

When you sort the list on sheet 2, perhaps the name Allen or Bill will then be in cell A1 on that sheet, so Allen or Bill is what will be displayed in the formula cell on sheet 1. As I indicated before, the formula links to the cell, not the name in the cell.

Your copy/paste experiment is invalid because you copied the data from the list, you didn't create a formula link. By pasting the actual name into sheet 1 it stands to reason that it would not change when you sorted the list on sheet 2 -- there was no connection which would cause it to do so.

The short answer to your ending question is: No, the cell references in the formulas on sheet 1 cannot be made to change [i.e., follow the name] when you sort the data on sheet 2.
