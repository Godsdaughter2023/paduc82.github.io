Macro for auto-scaling both axes - Microsoft Community

pau vb say:

Macro for auto-scaling both axes

Please need macro to auto-scale both axes in chart in sheet2 from table in sheet1
teylyn say:

if the chart is on a chart sheet, then it is a different collection than the objects on the sheet collection. The code needs to change to reference the chart as an object in the ThisWorkbook.Charts collection.

If you want to run the code automatically when the chart sheet is opened, then you need to enter the following code into the Sheet module. Right-click the "Plot" tab and select "View Code". Post the following code into the code window:

Private Sub Chart_Activate()
Dim chtMinX As Double
Dim chtMaxX As Double
Dim chtMinY As Double
Dim chtMaxY As Double
Dim sh1 As Worksheet
Dim cht As Chart
Set sh1 = ThisWorkbook.Sheets("Sheet1")
Set cht = ThisWorkbook.Charts("Plot")
chtMinX = WorksheetFunction.Min(sh1.Range("D32:AY35"))
chtMaxX = WorksheetFunction.Max(sh1.Range("D32:AY35"))
chtMinY = WorksheetFunction.Min(sh1.Range("D28:AY31"))
chtMaxY = WorksheetFunction.Max(sh1.Range("D28:AY31"))
' first set the X axis
cht.Axes(xlCategory).MinimumScale = chtMinX  ' this sets the X axis minimum
cht.Axes(xlCategory).MaximumScale = chtMaxX ' this sets the X axis maximum
' next set the Y axis
cht.Axes(xlValue).MinimumScale = chtMinY ' this sets the Value (Y) axis minimum
cht.Axes(xlValue).MaximumScale = chtMaxY ' this sets the Y axis maximum
End Sub

The code will run when the chart is activated.

kaushikshirodkar say:

To implement validation in MS Word document.

3.I want to check two names (P volume and A volume) are common or not.

Kindly suggest how can I implement the desired validation on the entire ms word document.

jay freedman say:

Validation of any kind requires the use of a macro, because there are no other means of doing it. Given that, there are many details that must be decided before the macro can be written. For example:

  • Will the document (or, more likely, the template used to create many documents) use content controls, legacy form fields, or simply allowing users to type wherever and whatever they want?
  • Will all users have Word versions that support all the features used in the template? Will all users have Word for Windows, or will some use Word for Mac or Word Online? (This also affects the answer to the first point.)
  • What event will cause the validation to begin? Some choices include saving the document, closing the document, or printing the document.
  • What should happen if the document fails one or more of the validation tests?

Simple examples for legacy form fields are shown in http://wordmvp.com/FAQs/TblsFldsFms/ValidateFFields.htm and http://www.gmayor.com/formfieldmacros.htm.

Another approach that could simplify the job is to program a userform to collect all the information needed from the user, and then insert that information in the document. Because the user doesn't get a chance to type in the document, there cannot be any incorrect formatting -- it's all predetermined by the construction of the template and the macro. The page at http://gregmaxey.com/word_tip_pages/validate_userform_text_entry.html shows how to validate the user's responses in the userform.
