marshall barton say:

On a datasheet form (and query/table), down is well defined as the same column in the next row.  OTOH, on a single or continuous form, controls can be scattered all over the various sections and a definition of column doesn't exist.  So, moving the focus down begs the question of which control is "below" the current control.

 

But if you always want the Down arrow to go to the same control in the next record, then you can use the form's KeyPreview (or KeyDown?) event to check for the Down arrow and move to the next record:

    Me.Recordset.MoveNext
By default, the same control in the next record should keep the focus.
