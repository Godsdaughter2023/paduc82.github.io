Microsoft should implement a command-line to Windows 10 as a feature. It should be accessed just like Linux users can. For those who do not know or can't remember, let me tell you how it is accessed. Keyboard Shortcut Interface Description

Cons:

On a Notebook/Netbook:

Use keyboard shortcut "Ctrl + Alt + F1" for Terminal

On a Tablet:

Use app on Start Screen

On a X-Box:

Use app on Start Screen

On a Windows Phone:

Use app on Start Screen

alan wu msft say:

Hi DM,

I'd like to know if you are talking about the image that is inserted in the email body directly or the image attached as an attachment. 

For the image attached as an attachment, it won't be downloaded automatically (but manually we can download it). If you are referring to the image that is inserted in the email body, please try the following methods to give it a shot:

1. Check Internet Options Encrypted Page Setting. 

In Internet Explorer go to (the gear, Tools), Internet Options, Advanced tab. Look under the Security section for Do not save encrypted files to disk. Untick this setting and Apply. Restart Outlook. (If you don't use Internet Explorer, you can go to Control panel, Internet Options or just type Internet Options in the Start menu Search.)

2. For the "not downloading image", whether it shows as "blank" or a "red X". If the former one, you can check the Picture Placeholder setting.

Click the New Email button or press Ctrl+N while viewing the Inbox.
Go to the new message's File, Options dialog.
Click Mail, then Editor Options
Select Advanced and look near the bottom of the dialog for the option to Show Picture Placeholders.
check the picture placeholder setting
Uncheck the box to Show picture placeholders
Close the dialog then close the message.

3. If the "not downloading image" shows as the "red X", you can refer to https://support.microsoft.com/en-us/help/2638687/pictures-cannot-be-displayed-and-are-shown-as-red-x-in-outlook.

Meanwhile, as you mentioned that it works fine if you un-install the new Windows update, I suppose it might be related to the Windows update(s), so you can also try creating a new Windows profile. At the same time, make sure your Outlook is newly updated. 

Regards,
Alan

scottgem say:
DMin is an aggregate function that find the lowest value in a field within a table or query. Min will find the lowest value in a list of values. 

john w. vinson say:

I'm not sure but I think you're treating Access as if it were a spreadsheet. It isn't! A table looks superficially like a spreadsheet but it's VERY different.

For one thing, the DMin() function and the Min() query operator work "down", not "across".  

DMin("[min]", "[query1]") will look at all rows in Query1, and return the lowest value of the field Min in those rows. Since your field named Min (at least in your example) has 11 in all rows, 11 is what you'll get.

Secondly, I would strongly suggest that you not use reserved words such as Min as fieldnames. Access can and will get confused! Similarly, don't use hyphens in fieldnames; Access may interpret d1-n1 as a subtraction operation, and complain when there is no field named d1 or n1.

Finally... just the need to do calculations "across" multiple fields suggests that your table design is incorrect. What other tables do you have? Are the fields [d1-n1] and so on calculated from data in other tables? What real-world information are you depicting in these tables? I strongly suspect that there is a much easier way to do what you're trying to accomplish, but at the moment I'm not clear what that might be!

"Go To" page for Table of Contents and Index

I want to create a Table of Contents and Index in Word 2007 where I can click on the topic in the TOC or Index and that click will take me to the appropriate "link".  How do I do that?  Thanks.
hansv mvp say:

For the TOC: do not use one of the pre-configured tables of contents, but select Table of Contents > Insert Table of Contents on the Review tab of the ribbon. You can then specify that the page numbers will act as hyperlinks (this is, in fact, the default setting).

 

Word doesn't support hyperlinks in Indexes, unfortunately.
