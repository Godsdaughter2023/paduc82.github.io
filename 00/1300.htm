abi99 (2) say:
DennIz wrote:

"Wrong Password"

There are several reasons why that could happen. One reason: Your network changed from WPA2-PSK back to WPA1-PSK. Some Wi-Fi access-points have silly mixed-mode settings. If your network is in mixed mode, first try the old version: WPA-only (WPA-PSK). Because some very old Wi-Fi access-points do not support WPA2 correctly even for multicast, as next step try the more secure version 2: WPA2-PSK. If that does not work:

 

Do you have a Mac OS X computer around?

If yes, please, go to Applications » Utilities » Terminal and enter

/System/Library/PrivateFrameworks/Apple80211.framework/Resources/airport -I -s

The last part (Security) should be ‘WPA2(PSK/AES/AES)’ and nothing else.

 

Do you have a Windows Vista, Windows 7, or Windows 8 computer around?

If yes, please, open a command prompt (as Administrator, keys: Windows-R), and enter

netsh wlan show networks mode=bssid

The network type has to be not Ad-hoc but Infrastructure.

The basic rates should be ‘1 2 5.5 6 11 12 24’ (IEEE 802.11g/n) and nothing else.

 

Do you have a Android device around?

If yes, install the free Wifi Analyzer. Within that tool, please, tap the wrench icon (Settings) » Show full securities. Then go back and have a look at your Wi-Fi.

It must be [WPA2-PSK-CCMP][ESS] and nothing else.

 

If the connection still fails, please go to the configuration interface of your Wi-Fi access point and change your access-point back to the old WPA-PSK (TKIP-only) and 802.11g (only). If not available, avoid 802.11n and go for 802.11b/g. Does it work now?

 

If it still does not work, change your settings back to WPA2-PSK (AES-CCMP) only and 802.11g/n. Then, post the results of the above tools. If you do not like to post them into the public, you can send me a private message. But those information are public to your neighbours anyway.
