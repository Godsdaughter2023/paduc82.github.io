    john john __ say:

    NTFS is the file system, there are no previous versions in the file system, once changes to files are committed and flushed to the disk that is it.  Perhaps you mean the Last Known Good Configuration at the advanced boot menu, that is only good until you reboot and logon to the installation, after a successful boot the Last Known Good Configuration is replace with the current successfully booted configuration.  The Last Known Good Configuration only works on the Control Set, it doesn't do or restore anything else in the registry, it doesn't work for User settings.  If the system was not rebooted after the problem occurred you could reboot the machine to the the advanced boot menu (tap the F8 key before Windows starts to boot) and boot to the Last Known Good Configuration and this would revert the system variables to the state they were when you last logged on to the machine, you would then only need to address your user variables.

    System Restore is really the best way out of these kind of snafus, if you have never used system restore I suggest that you familiarize yourself with it, either that or restore from a system state backup.  If you had a backup of the registry or keys in question you can restore these.   You can copy the variables from another clean Windows 7 installation but any software that you installed which added its own variables will (may) not work properly until you restore those. 

    John
