lewis9177 say:

Adding a hyperlink to a mail merge

macropod say:
Here's how you can do get a mailmerge to display your preferred default 'Text to display' text:
1. Disregarding mergefield issues for the moment, insert a hyperlink into the document in the normal way, choosing whatever 'Click Here' text you want in the 'Text to display' box.
2. Select the inserted hyperlink and press Shift-F9 to expose its field code.
3. Replace everything in the field after 'HYPERLINK' with your mergefield.
4. Select the field and press F9 to update the display.
 
In Word 2007 & later, you can make the display text variable also, by following these additional steps:
5. Position the cursor anywhere within the display text.
6. Insert a mergefield pointing to whatever data field you want to use for the display text (this could even be the same field as used at step 3 above).
7. Delete all of the previous display text either side of your last-inserted mergefield (note that this field will likely have updated already).
8. Execute the merge.
9. After merging to a new document, use Ctrl-A, F9 to update all fields. Without this, the mergefield hover text won’t update to the correct targets.

a. user say:

Can you see and/or copy/paste the link address and see what is wrong with it?

I do not know if this is the problem that you have encountered, but there is a subtle problem here.

There are two different things: the link address, which is set using a nested field in the { HYPERLINK } field, like this:

{ HYPERLINK "{ MERGEFIELD theLinkAddressField}" }

and the display address, which is set by inserting a MERGEFIELD in the "result" of the hyperlink field.

In the case of the mergefield nested within the HYPERLINK field, if you save and re-open your mail merge main document, by default the nested { MERGEFIELD theLinkAddressField } field will not survive, but will be resolved to its current result, e.g.

{ HYPERLINK "http://www.somethingorother.xxx" }

After that, it's useless for merging.

However, when testing simple merges, this is often obscured because you merge before saving, so the field isn't destroeyd, or because the last thing you do before testing is to edit or click inside the HYPERLINK field. This creates an invisible bookmark (which Word uses to "Go back" to a previous selection in the document), and for some reason, the existence of that bookmark prevents the nested field from being resolved. So in test projects, this approach may work perfectly, then fail in a production scenario.

So a simple resolution may be to click in that hyperlink field before saving. Not very reliable, and no good if you have multiple hyperlink fields. So another approach is to insert a bookmark in each nested HYPERLINK field, e.g. immediately after the { MERGEFIELD } field. If you want it to be clear, you could insert a { SET bm1 " " } field in the first, { SET bm2 " " } in the second, etc. Or perhaps  


in each one, and update all the fields before saving.

summer203 say:

How do I bring guides to the front in Publisher 2013?

Publisher 2013 places grid guides behind objects. This makes it difficult to align items, like text blocks, within objects. Is there a way to bring guides to the front?

davidf2 say:

No, sorry there is no way to bring the guides to the top/front, except temporarily.

You could draw a line and overlap the guide with the line and align your objects to that line. When finished delete the line or drag it off into the scratch area.

However you can use the guide lines to help you align objects even with them in the background. Go to Page Design tab > layout group > check Guides and Objects under Align to:. Also go to View > Show group and check Boundaries (along with Guides and Rulers). Finally zoom in to as much as 100% or more of page size to make it easier to see what is happening.

Now when you slowly drag an object (text box, picture etc.) and the boundary or the edge of the object comes in line with your guide line the guide line will change colors, even with it hidden behind other objects. The guide line will turn a darker blue and will actually become visible on top of other objects temporarily to indicate your object is aligned with the guide line. Look at the edge of the page where the guide line is always visible and you can see the line change colors as you move an object in line with it and also when the midpoint on the object box is aligned with the guide line. This allows you to center align objects on that guide line.

Also notice as you drag an object around when the edges align with other objects edges again you will get a temporary pink/red line. That pink or red line also appears when the center of the object aligns with the center of other objects. This helps you align objects without the need of inserting guide lines in the background in most cases.

I think if you zoom in and experiment a bit with how this works you will find this approach actually works well and you won't need to have the guidelines on top all the time.
