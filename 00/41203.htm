a. user say:

When XP reboots, it checks all volumes to see if they are "dirty" and will automatically run chkdsk on the dirty volume(s).  If it wants to run, you should let it run.

A volume could be marked dirty from something like a power interruption, aborted restart or ungraceful shutdown.

If you have scheduled a chkdsk to run on the next reboot, you can "undo" that, but you cannot undo the automatic chkdsk that wants to run on dirty volumes.

If a volume is dirty, you will not be able to run defrag on it until the dirty bit is cleared.  Trying to run defrag on a dirty volume gives a message like this:

Disk Defragmenter has detected that Chkdsk is scheduled to run on this volume: (C:)
 
Please run Chkdsk /f.

You can query the dirty bit on a volume from the command prompt window.

Click Start, Run and in the box enter:

cmd

Click OK to open the command prompt window.

To query the dirty bit on drive C, type:

fsutil dirty query C:

Sample output:
Volume C: is dirty
Volume C: is not dirty

Type 'exit' to close the command prompt window.

Is the volume on which you want to run defrag dirty or is it not dirty?

The only way to clear the dirty bit is to let chkdsk /r (or chkdsk /f) run completely and correct the problems on the volume.  When chkdsk /r runs clean, the dirty bit will be cleared.  That is the only way to clear the dirty bit.

Sometimes it is best to run chkdsk /r from the XP Recovery Console - that way, you can see what it is doing and run it again if needed and keep running it until it runs clean.

Note:  The Recovery Console does not allow you to use fsutil to query the dirty bit (it is not a Recovery Console command).

If you do not have a genuine bootable XP installation CD from which you can run the XP Recovery Console,  make a bootable XP Recovery Console CD (no XP media required), then you can run chkdsk /r from there and see what it has to say.

a. user say:

That is not what it is saying...

/f is not a hard drive - it is an optional parameter to the chkdsk command.  It means "fix" things.

XP is telling you to run chkdsk with the /f option.  If you would comply, and chkdsk is able to fix the volume, then you would not have the issue.

You can read the chkdsk parameters here.

If Windows is installed on your C drive, run the fsutil query command as indicated to see the the C volume is dirty.

If the volume is dirty (or if any volume is dirty), chkdsk will always run at startup until you clear the dirty bit.

You clear the dirty but by running chkdsk /p (or chkdsk /r) on the afflicted volume.  For example:

chkdsk c: /f

You will also not run defrag on any volume/hdd until the dirty bit is cleared.
