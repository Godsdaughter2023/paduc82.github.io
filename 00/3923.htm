jeeped say:

You're a little short on specifics but I'll suppose that the 1 you speak of is hour 1 and that Door Name is inconsequential, just that it needs to be there before a 1 or 0 goes into column E.

Right-click the worksheet's tab name and when the VBE opens, paste the following into the pane titled something like Book1 - Sheet1 (Code),

Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Target, Union(Columns(1), Columns(4))) Is Nothing Then
        On Error GoTo FallThrough
        Dim r As Long, trgt As Range
        Application.EnableEvents = True
        For Each trgt In Intersect(Target, Union(Columns(1), Columns(4)))
            r = trgt.Row
            Cells(r, 5) = 0
            If IsNumeric(Cells(r, 1)) And LCase(Cells(r, 4)) = "door name" Then
                Cells(r, 5) = (Hour(Cells(r, 1).Value) + 1)
            End If
        Next trgt
    End If
FallThrough:
    Application.EnableEvents = True
End Sub

Tap Alt+Q to return to your worksheet. The conditions are rudimentary and you will probably want something like a Select Case ... End Select for different conditions but this should get you started.

jllatham say:

Try this as replacement code for what Jeeped provided.  Not sure why you got an Object required error - I didn't.

To put it in the proper place in the workbook, as he said: choose the sheet with the entries on it  then Right-Click on the sheet's name tab and choose [View Code] copy the code below and paste into the module.  Remember that each sheet can only have one _Change() event code section, so for the time being just delete what Jeeped provided and try this for a 'general' solution. 

Private Sub Worksheet_Change(ByVal Target As Range)
  'change must be below row 1, and in either column A or D
  If Target.Row < 2 Or (Target.Column <> 1 And Target.Column) <> 4 Then
    Exit Sub
  End If
  If IsEmpty(Cells(Target.Row, 1)) Or _
   IsEmpty(Cells(Target.Row, 4)) Then
    'either no time entered or no 'door' entry available
    Exit Sub
  End If
  On Error GoTo FallThrough
  'TimeSerial needs hours in 24 hour format:
  '  Hour: 0 = midnight, 23 = 11 p.m.
  '  minutes and seconds can actually be negative,
  '  but generally use 0-59 for each
  '  TimeSerial(6,22,53) = 6:22:53 AM
  '  TimeSerial(18,35,42) = 6:35:42 PM
  Application.EnableEvents = False
  'initially mark it as either not in correct
  'time or as not a "Door " entry
  Cells(Target.Row, 5) = 0 ' mark it
  'this is testing for doors opened between 1:00:00 am and 1:59:59 am
  If Cells(Target.Row, 1) >= TimeSerial(1, 0, 0) And _
   Cells(Target.Row, 1) <= TimeSerial(1, 59, 59) Then
    If LCase(Left(Cells(Target.Row, 4), 5)) = "door " Then
      Cells(Target.Row, 5) = 1 ' mark it
    End If
  End If
FallThrough:
  Application.EnableEvents = True
  On Error GoTo 0
End Sub

jllatham say:

Macros associated with ThisWorkbook and individual worksheets that are in those object's code modules happen automatically - and because they are defined as Private - they do not appear in the list of macros you can run in normal fashion.  So when you see macros that start with

Private Workbook_ ...

or

Private Worksheet_...

you know they are "special"

Now, the one I wrote is going to automatically run (if macros are enabled) when you make an entry onto the worksheet in a cell (or if it is done by VBA - but not as the result of a recalculation of a formula). 

The first IF statement says that if the change took place in row 1 or not in either column A or D, then don't do anything - just quit.

Then the second IF statement checks to make sure that there are entries in both column A and column D on the row where the change did take place.  If either is empty, again, nothing happens because we don't have either a time to test or we don't have a "Door " phrase to test.

Now IF the change was in column A or D and below row 1 and there are entries in A & D on the row, then it's going to test the time, and test whether or not the entry in column D starts with "door ".  It initially assumes that it's not going to test 'validly' and puts a 0 in column E, then if the tests pass, it rewrites that with a 1.

The Application.EnableEvents = False statement keeps us from jumping back into the routine when the 0/1 is written by VBA onto the sheet. 

So it should work if you actually use the sheet - it worked fine for me.

How to remove/add a space after/before all headings with a - Microsoft Community

derrick80 say:

How to remove/add a space after/before all headings with a macro

I have a document with many level 3 headings. I need a space (paragraph break) added before all of them, and a space (paragraph break) removed after each of them. How can I do this?

Thanks in advance.

macropod say:

Properly formatted, your document shouldn't contain any double paragraphs. All the spacing should be controlled via the paragraph space before & after options, preferably as part of the corresponding Style definitions. Thus, deleting all double paragraphs and fixing the Style definitions will give you a document with a more consistent layout and whose layout is easily changed to suit any further changes in formatting requirements. That said, you can clean up just the heading 3 paragraphs with a macro like:

Sub Demo()
Application.ScreenUpdating = False
Dim i As Long
With ActiveDocument.Range
  With .Find
    .ClearFormatting
    .Replacement.ClearFormatting
    .Text = "[!^13]@^13[^13]{1,}"
    .Replacement.Text = ""
    .Forward = True
    .Wrap = wdFindStop
    .Format = False
    .MatchWildcards = True
    .Execute
  End With
  Do While .Find.Found
    If .Paragraphs.First.Style = "Heading 3" Then
      i = i + 1
      Do While .Paragraphs.Last.Style <> "Heading 3"
        .Characters.Last.Text = vbNullString
      Loop
    End If
    .Collapse wdCollapseEnd
    .Find.Execute
  Loop
End With
Application.ScreenUpdating = True
MsgBox i & " headings updated."
End Sub

Note: The macro doesn't insert an empty paragraph before the heading - you should insert the desired space before via the heading 3 Style definition.

fin_modler say:

I think the iteration switch is global rather than workbook specific.

Macros are typically more stable than circular cell references if you are permitted to use them. After all, loops are normal in most programmes/scripts but somehow Excel has never embraced VBA-free iterations, a pity because they are just so common and so useful. For example, I cannot see how financial forecasting or project scheduling-budgeting could be done reliably without iterations.

If macros are permitted, there is a half-way compromise: you can force iterative calculations on upon opening of the workbook instead of converting all the calculations to UDF or VBA. Going the UDF/VBA route takes away transparency and can make a sheet much harder to correct/change/follow especially after some time has passed since creating it.

Having a breaker is often essential in overcoming that instability. Below is a rather trivial example.

Without breaker (starting from cell A1):

30
=A4*0.1
=-(A1+A2)*0.25
=A1+A2+A3

With breaker:

30 TRUE
=IF(C1,0,A4*0.1)
=-(A1+A2)*0.25
=A1+A2+A3

True in C1 breaks the loop and False turns it on. If you ended up with errors everywhere (unlikely with the above) you would not need to retype formulas first to destroy the loop and then again to reinstate it.

Structuring of the calculations could make a huge difference to robustness. Iterative calculation is one of two situations (from my own experience) where casually replacing a formula or cell contents to facilitate it is not ideal. It is hard to illustrate that with the above example; easier with something having more realistic complexity.

It depends entirely on the specific formulae in the workbook. It is usually the case that formulae that form a circular reference need to recalculate several times to get a set of values that simultaneously satisfy all the formulae. I think this is what you are referring to when you say that your formulae aren't working unless you enable iteration.

So, in your case, yes, you need to have iteration turned on to get your formulae to work correctly.

To determine whether you need iteration in a particular situation, try doing the calculations by hand. Write down all the formulae and work out the result of each formula only once using the display values. If re-calculating any of the formulae a second time will result in a different answer, then your need iteration to get accurate results.
