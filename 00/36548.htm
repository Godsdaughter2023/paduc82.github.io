punchingbag say: I know this is over 3 years late for this thread, but I recently figured out this problem on my dad's home copy of win7 64.

I have successfully been able to install Visio 4.5 standard, Visio 5.0 upgrade, and Visio 2002 upgrade.  All worked successfully on win7, but the earlier older versions had some quirks to fix in the installation process to get 2002 to see them on the harddrive and recognize them as valid upgradeable products.

The main problem is that visio 4.5 and 5.0 used an old 16bit version of installshield installer program.

The apps themselves are 32bit and will run in compat mode for windows NT 4.0 in win7 64bit.

My solution was to get a program that will extract the old installshield cab files used on the cd.

The file to extract from cd is called "data.z" and you need to extract it to a good location on your hd.

I used the program Universal Extractor, from the following website:

http://legroom.net/software/uniextract

the extraction method used was i3comp.

Once the files extracted to HD, I created a shortcut to the visio executable in the extracted folder, in the shortcut properties I went to compatability options and clicked to change settings for all users.  Here I made the program run as Windows NT 4.0 and run as administrator.

I then successfully tested both Visio 4.5 and 5.0 with this method to make sure any registry or ini file stuff needed by 2002 installer would be found easier.

After I verified they were running and worked on win7 64, I put in the 2002 upgrade CD recently purchased brand new from amazon seller.

The installer found the harddrive copy, and I told it to install over the folder of Visio 5.0 on harddrive.

When I launched visio 2002, it asked to activate, I then simply let my father continue the process of activating Visio 2002 upgrade since it was actually him who had bought it and owned it and was installing it on his computer.

He had been working on installing it for a couple of hours and finally had put it on his windows xp virtual machine for win7, but wanted it on his actual win7 64bit installation.

I got the chance to help him for about an hour, and it was enough time to figure out the problem and get Visio 2002 working for him.

I will note that there is a point in the install process when upgrade version wants to find the older copy to upgrade from, it will try searching the hard drive and will seem almost to be frozen, but the hourglass style spinning circle mouse pointer will still be moving.  As long as you wait it out it will eventually find the installed older version.

On my dad's computer there are over 1million files from different programs and old documents to search through, eventually after about 5-6 minutes the installer found the old copy and continued.

After that is done only internet activation is required.

Hope this helps someone.
