ken sheridan say: Normally one would attach a PDF of a report which is restricted to the form's current record by referencing a control in the form as a parameter in the report's query.  You'll find an example as InvoicePDF.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you amend the relevant code snippet in the Click event procedure of the 'Email Invoice' button in my demo to;

    DoCmd.SendObject ObjectType:=acSendForm, _
        ObjectName:="frmInvoice", _
        OutputFormat:=acFormatPDF, _
        To:=strTo, _
        Subject:=strSubject, _
        MESSAGETEXT:=strMessageText, _
        EditMessage:=True

This will attach a PDF of the form, but of all records, not just the current one; to do the latter it would be necessary to ensure that the form's recordset is restricted to the current record.  Even so, forms are usually designed for screen display, so the standard of presentation when output as a PDF file tends to be  poor in comparison with a report specifically designed for the purpose.  I'd recommend spending a little time designing a suitable report which reflects the form's data.

BTW it is not necessary to create a PDF file first before attaching it.
