Memory leak in copy/paste? - Microsoft Community

kergga say:

Memory leak in copy/paste?

Hi all-

 

Win7x64, 8GB RAM, Excel 2010 32-bit

 

So far, moving to Excel x64 is not an option with our IT department.

 

I have a large (100MB+) raw data file (.txt) that I will need to import and analyze frequently (maybe up to once per week). I can open that text file in Excel and convert to the compressed xlsx, and it ends up ranging from 30-40MB.

 

I also have my "main" analytics workbook, into which I need to import this data. I tried manually copying just the used range, and ran into a memory error. So I then decided to try VBA to copy/paste smaller ranges, thinking that as long as I can get a small range to paste once, I should be able to cycle through each of the smaller range copy/pastes without a problem. Unfortunately, I get about halfway through the expected iterations and still run into an out-of-memory error.

 

I have researched and applied all the other fixes I can, including turning off file previews, etc.

 

The key question I have for this group is whether there is a known memory leak with the operations shown below, or if there is something I'm missing that would help clear the clipboard and maybe prevent memory bloat.

 

EDIT: Interestingly, I was able to complete the copy/paste operation by adding a DestWkbk.save and DoEvents within my loop after each 10K rows were copied. So whatever the leak is, it is reset with a filesave. Of course, saving a file this large over a dozen times really slows down my process, but at least I have a kludgy workaround. If anyone has suggestions for what else I could add to my code to avoid using the filesave, I'd appreciate all ideas!

 

Thank you!

 

 

Sub CopyDataInChunks()


    Application.EnableEvents = False
    Application.Calculation = xlCalculationManual

 

    Dim SourceWkbk As Workbook
    Set SourceWkbk = ActiveWorkbook
    Dim SourceSht As Worksheet
    Set SourceSht = ActiveSheet
    Dim DestWkbk As Workbook
    Set DestWkbk = ThisWorkbook
    Dim DestSht As Worksheet
    Set DestSht = ThisWorkbook.Sheets(1)
    
    'Find last row and last column of new data
    Dim LCol As Long
    LCol = LastCol(SourceSht) 'gives column number
    UseCol = ColLetter(LCol) 'converts that number to a letter
    LRow = LastRow(SourceSht)
   
    'Find last row of old data, to make sure it all gets overwritten
    LastOldRow = LastRow(DestSht)
   
    'Turn off page breaks- just in case it helps (from other posts related to the out of memory error). I don't think they are on, but this is just backup
    SourceSht.DisplayPageBreaks = False
    DestSht.DisplayPageBreaks = False
   
    'Copy in chunks
    FinishedCopy = False
    StartRow = 1
    While FinishedCopy = False
        Application.CutCopyMode = False
        EndRow = StartRow + 9999  '1 to 10,000 increments
        Application.StatusBar = "Copying " & StartRow & " to " & EndRow & " of " & LRow
        SourceSht.Range("A" & StartRow & ":" & UseCol & EndRow).Copy
        DestSht.Range("A" & StartRow).PasteSpecial Paste:=xlPasteValues, Operation:=xlNone, SkipBlanks:=False, Transpose:=False
        SourceSht.Range("A1").Copy SourceSht.Range("A1")
        DestSht.Range("A1").Copy DestSht.Range("A1")
        StartRow = StartRow + 10000
        If (EndRow > LRow) And (EndRow > LastOldRow) Then FinishedCopy = True

        DestWkbk.Save

        DoEvents
    Wend

 

    Application.CutCopyMode = False
    Application.StatusBar = False
    Application.EnableEvents = True
    Application.Calculation = xlCalculationAutomatic
   
    MsgBox "Raw data copy is complete." & Chr(13) & Chr(13) & "It is recommended that you save the file before continuing", , "Program finished"

End Sub

Sub Test()
  Dim fso As Object 'Scripting.FileSystemObject
  Dim TS As Object 'Scripting.TextStream
  Dim Contents As String
  Dim Lines
 
  Set fso = CreateObject("Scripting.FileSystemObject")
  Set TS = fso.OpenTextFile("Z:\test.txt")
  Contents = TS.ReadAll
  TS.Close
  Lines = Split(Contents, vbCrLf)
  Range("A1").Resize(UBound(Lines) + 1, 1) = WorksheetFunction.Transpose(Lines)
End Sub

How do I permanently delete a conversation? 

jomel_g say:

Hi

Good day and welcome to Skype Community!

When you delete a conversation, all the messages in the conversation are deleted, and it's removed from the sidebar and your history. The conversation will be deleted from Skype on that specific device, but will still be visible on other devices unless you delete it from there too.

  1. Locate the conversation you want to delete.
  2. Right-click the conversation.
  3. Select Delete conversation.
  4. And select Delete in the confirmation box.

I hope the information above helps.

Regards,
