There are two requirements to obtain a 1366x768 resolution in Win 10 (or any other OS):

1. Your monitor should support that resolution and it should be (preferably) the native resolution for the monitor in question. Any other resolution (than native) may result in a shrunken and/or distorted display.

2. Your graphics arrangement must support the desired resolution. Some older graphics arrangements do not support this resolution; however, most newer rigs can easily accommodate 1366x768.

In summary, if you do not have that option available in your drop-down menu and/or it is not the recommended resolution, one or both of the above requirements have not been met.

a. user say:

Re:  writing to a file using VB.Net


You won't find many at this forum familiar with VB.Net; I certainly don't use it.
However, here are some comments that might help...

GetObject returns a  reference to an existing instance of Excel
If Excel is closed you will need CreateObject to start Excel and provide a reference.

Contrary to your statement, I believe you are accessing an open workbook (not a closed workbook).
Normally (in VBA) one checks to see if a workbook is already open before attemping to open it.
The VBA code can look something like this...
'---
 Function IsItOpen(BookName as String) As Boolean
   Dim WB As Workbook
   On Error Resume Next
   Set WB = Workbooks(BookName)
   IsItOpen = (Err.Number = 0)
   Set WB = Nothing
 End Function
'---
Also, the UsedRange on an Excel worksheet, does not always start in row one.
So  "line = oSheet.UsedRange.Rows.Count + 1"  may or may not provide the first blank row below the data.
"line = oSheet.UsedRange.Rows( oSheet.UsedRange.Rows.Count).Row + 1" would be a safer bet.
'---

fmb501 say: I'm sure there is more than one cause for this problem, but here is a solution that worked for me.

About a week ago, the Vista WMD stopped synchronizing with my WM 6.1 phone for some reason.  I searched for solutions on the Internet for the "Activesync encountered a problem on the desktop" error, but nothing worked.  My best guess was that either the data table on the phone or the synchronization table on the desktop was corrupt or both.

The only solution I could find on the Internet (Microsoft didn't have any information on this problem) was a long set of instructions to uninstall Outlook, do a hard reset on the phone, etc.

I did not want to uninstall Outlook unless is was really the last resort, but since none of the other proposed solutions were working, I decided that I was out of ideas and to give it a try.

In those instructions, one of the steps was to perform a Hard Reset on the phone.  Of course, this deletes all data, files, and 3rd Party software.  That would certainly delete the Activesync data set on the phone, but  I didn't want to lose everything in the process  That got me to thinking about my problem so here is what I did and it solved the problem:

1. Disconnect the phone from the desktop
2. On the desktop running Windows Vista, run the Windows Mobile Device Center (WMDC) and end the partnership with the phone.  When prompted click YES to delete the data for the partnership.  My thought was that this deletes any possible corrupted synchronized 3. On the Phone, still disconnected from the desktop, run Activesycn.  There should be a MENU option and within that there should be an OPTIONS selection.  Click on OPTIONS.
4. In my case, the phone somehow ended up with two sets of data to be synced, and that in itself might have been the problem all along.  I deleted the older data set, but this was just a roll of the dice since I couldn't tell what was in either data set nor could I tell how big they were.

If there is only one data set, I would try deleting it, but that's only what I would do--you have to make that decision yourself.
5. Reconnect the phone to the desktop.  In my case, the WMDC automatically connected to the phone and since I had deleted the partnership, it gave me the option to enter new settings.  I enabled everything because I use all the Outlook and OneNote features.  Once I clicked OK on for the settings, WMDC sat there for a minute or two or three (which is normal) before it started to sync with the phone.

Because the data sets on both the phone and desktop were deleted, the sync took a long time to complete.  This was encouraging for me because I'd normally get the error within a few seconds.

I took a short break and when I returned, the phone and Outlook were synced properly.  There was one odd result.  A few years ago when I would try to do something like this, I could end up with hundreds of duplicate emails, contacts, and notes.  This particular attempt worked very well, but I did end up with one note duplicated 3 times, and two contracts that were duplicated.  I have no rational explanation of how that would happen, but as far as I was concerned, it was a small price to pay to get my phone back in sync with my desktop with more than 500 contacts, a couple hundred emails in the inbox, and about a hundred notes.

I hope this helps someone out there from spending many hours trying to find a solution to this problem.
