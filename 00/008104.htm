It may or may not help, but I use a couple of procedures to check whether a URL is accessible before trying to send posts to it.

Private Declare Function InternetGetConnectedState _
    Lib "wininet.dll" (ByRef dwflags As Long, _
    ByVal dwReserved As Long) As Long

Private Declare Function InternetCheckConnection Lib "wininet.dll" _
  Alias "InternetCheckConnectionA" (ByVal sUrl As String, _
  ByVal lFlags As Long, ByVal lReserved As Long) As Long

Function IsInternetConnected() As Boolean
  Dim L As Long
  Dim R As Long
  R = InternetGetConnectedState(L, 0&)
  IsInternetConnected = (R <> 0) And ((L And 7) <> 0) ' OK if LAN, modem or proxy.
End Function

Function CheckConnection(stURL As String) As Boolean
  If Not IsInternetConnected Then Exit Function
  CheckConnection = InternetCheckConnection(stURL & Chr(0), 1&, 0&)
End Function

Sub Example()

  If Not IsInternetConnected() Then
    MsgBox "You seem not to have an internet connection"
  ElseIf Not CheckConnection("http://www.mysite.co.uk") Then
    MsgBox "I can't get a response from mysite"
  Else
    '... do something
  End If
End Sub

bill manville say:

Clearly if you are able to use the site through a browser it is not blocking you from accessing the site.

If you refresh a query using BackgroundQuery:=False it will wait for the response.

When Excel is busy it will display (Not Responding) in the title bar after a while.

(I wish it just said (Busy) but that's another matter).

If you want to be able to do anything while waiting for the query to return you need to run the query in the background. 

You could do something like this in place of the current .Refresh line:

  .Refresh BackgroundQuery:=True
  Dim iSeconds as Integer

  iSeconds = 0

  Do While .Refreshing
    Application.Wait Now+TimeValue("0:0:1")  ' wait a second
    iSeconds = iSeconds+1
    Application.StatusBar = "I've been waiting for a response for " & iSeconds & " seconds"
  Loop
  Application.StatusBar = False
