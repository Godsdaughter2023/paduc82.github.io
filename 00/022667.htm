I was too far along to quit, when Hans posted, so here is an alternative.
Enter the function to the right of the data and in the row with the data.
Sample cell entry... "=GiveMeAvg()"  no quotes

 '---

Function GiveMeAvg() As Variant
'Jim Cone - Portland, Oregon USA - Feb. 2017
 Dim LastCell As Excel.Range
 Dim FirstCell As Excel.Range
 Dim rngRow As Excel.Range
 Dim N As Long
 Const NumToUse As Long = 5
 
 If TypeName(Application.Caller) <> "Range" Then
   GiveMeAvg = "Not a Range"
   Exit Function
 Else
   Application.Volatile
   Set LastCell = Application.Caller.End(xlToLeft)
   Set FirstCell = LastCell.EntireRow.Cells(1)
 End If
 
 Set rngRow = LastCell.Parent.Range(FirstCell, LastCell).Cells
 If Application.WorksheetFunction.Count(rngRow) < NumToUse Then
   GiveMeAvg = "Insufficient Data"
   Exit Function
 Else
  For N = rngRow.Cells.Count - (NumToUse - 1) To 1 Step -1
    If Application.WorksheetFunction.Count(Range(LastCell, rngRow.Cells(N))) = NumToUse Then
      GiveMeAvg = Application.WorksheetFunction.Average(Range(LastCell, rngRow.Cells(N)))
      Exit Function
    End If
  Next
 End If
End Function

yiyi1215 say:

Ranking a group thru using VBA

Hello,

 

I am using the following formula in a column of 80,00 rows to help rank a group of data based on a condition:

 

= 1 + Sumproduct((CA:CA=CA2)*(E:E<2))

 

I would like to know how I can enter this function in a macro as opposed to entering it in the worksheet's cells. 

 

Thanks!!!

Yiyi1215

joeu2004 say: Yiyi wrote:
I am using the following formula in a column of 80,00 rows to help rank a group of data based on a condition:
= 1 + Sumproduct((CA:CA=CA2)*(E:E<2))

 

I would like to know how I can enter this function in a macro as opposed to entering it in the worksheet's cells.


First, that is probably grossly inefficient in XL2010 because CA:CA and E:E might refer to far more than just the 80,000 rows of interest.

(Einstein also wrote:  "Everything should be made as simple as possible, but not simpler".)

Be that as it may, in VBA, you could write:

Dim n as Long
n = 1 + Evaluate("Sumproduct((CA:CA=CA2)*(E:E<2))")

In either case, instead of SUMPRODUCT, it would be better to use COUNTIFS, to wit:

1 + COUNTIFS(CA:CA,CA2,E:E,"<2")

In VBA:

n = 1 + Evaluate("COUNTIFS(CA:CA,CA2,E:E,""<2"")")

Note the use of double-double-quotes ("") to express one double-quote within a quoted string.

The same caveat of about full-column ranges still applies.

joeu2004 say: Yiyi wrote: 

What do you recommend I do for the range size?  Is it best if I were to create a dynamic named range?


Of course, that is a matter of opinion.  To each his own.

With respect to my comment about the cost of whole-column ranges (A:A), there is little advantage to using a dynamic named range, e.g. OFFSET($A1,0,0,COUNTA($A:$A),1).  First, COUNTA must still look at the entire column.  Second, OFFSET is a volatile function; it is recalculated every time any cell in the entire workbook is modified.  Hopefully that is only one time per edit, even if the named range is reference many places.  But it's the principle of the matter.  Finally, the OFFSET/COUNTA assumes there are no interstitial empty cells (with no constant and no formula).  A more reliable -- and more costly -- expression is COUNTA($A:$A)+COUNTBLANK($A:$A).

Since you said you have "80,000 rows", perhaps CA1:CA8000 and E1:E80000 would suffice.

Alternatively, I usually ensure that there is a "blank" row above and below the data.  (I might use a pattern format to highlight these special rows).  And I insert any new rows between them.  Thus, if the blanks rows are initially A1 and A100, $A$1:$A$100 is the range that I use.  As I insert rows, $A$100 is updated automagically.

There are downsides to that approach.  But it might be a usable low-tech way for you to go.
