OK, we are getting closer, just a matter of closing the communication gap between you and me. Not a problem. Option B had nothing to do with your problem, it is just for geeks and emergencies (you DOC is corrupted and you want to recover the body text only from it).

When you opened the NORMAL.DOTM in Word, as template, you saw exactly what you are supposed to see, a "blank" document. This is the base for every "new" document you see.

While you are in NORMAL.DOTM in edit mode, any change you make (and save when closing the document) will now appear in any NEW document you create.

So, if you modify a style, say "NORMAL" for discussion, and change the font to Times New Roman (like it used to be in 2003), then from now on all of your new documents would show the Normal style uses Times New Roman font.

Changes I typically make (hard to tell what defaults were since I already made them), File tab > Options option >:

Genreral Options: Change user name to my name,

Display: turn on Show All Formatting Marks (equivalent to clicking on the Show/Hide button on the Home tab), turn on Update fields before printing

Save: turn on Save Aurotrecover and reduce time interval to 5 minutes or less, turn on Keep last autosaved version if I close without saving

Language: make sure you have the "right" language selected ie Canadian vs Britsh vs US English

Advanced (ooh there are lots/too many in here to list all, many are specific to how you prefer to work, how much "automatic help" you want from Word ): turn off Show Picture Placeholders, Field Shading:Always, increase number of Recent Documents (I use 25, depends on your screen resolution), turn on show shortcut keys in tooltips, turn on show horizontal scroll bar, turn on vertical scroll bar, turn on show vertical ruler in print layout view, turn on Prompt before saving Normal Template (this one is important so you can keep track of changes made to NORMAL. if you don't know what changes it is asking to save, just say NO. Unfortunately MS still hasn't got around to telling us changes are being saved...) . turn on provide feedback with sound (I like to be beeped at...), turn on Confirm File format conversion on open (I want to always know before Word converts a file from one thing to another), File locations: I've changed the default file save location,

under Advanced > LAYOUT OPTIONS, there are lots of possibilities: my favorite is turn on "Do full justification the way WordPerfect 6.0 for Win does" (this is a more "intelligent" than default MS approach which only inserts full space char between words, WP approach inserts variable size spaces and also increases intra-letter spacing in words). Another one I like to turn on is "Don't expand character spaces on a line that ends with SHIFT RETURN. They are basically "me-too" features that MS added to allow former users of other tools to feel comfortable changing over to Word. nicolasbonzani say: What do you mean by "VBA automate an instance of Excel and push the data directly into the spreadsheet cells?" Isn't that what the DoCmd.TransferSpreadsheet command does? 

dirk goldgar say:
What do you mean by "VBA automate an instance of Excel and push the data directly into the spreadsheet cells?" Isn't that what the DoCmd.TransferSpreadsheet command does? 

No.  TransferSpreadsheet doesn't actually run Excel, but rather writes to the workbook file directly.  I'm talking about something like this:

'------ start of code ------

    Dim rs As DAO.Recordset

    ' Note: the following declarations use early binding, so they require
    ' a reference to be set to the Microsoft Excel? <version> Object Library.
    ' Late binding could be used instead.?
    Dim appExcel    As Excel.Application
    Dim wb          As Excel.Workbook
    Dim ws          As Excel.Worksheet
   
    Dim iCols       As Integer

    Set rs = CurrentDb.OpenRecordset(TableOrQuery)
   
    Set appExcel = New Excel.Application

    Set ws = wb.Sheets(1)

    ' Create a worksheet header row from the recordset's field names.
    With rs
        For iCols = 0 To .Fields.Count - 1
            ws.Cells(1, iCols + 1).Value = .Fields(iCols).Name
        Next
    End With
    ws.Range(ws.Cells(1, 1), ws.Cells(1, rs.Fields.Count)).Font.Bold = True
   
    ' Starting on the next line of the worksheet, copy each row from the
    ' recordset to a new row in the worksheet.  Excel provides a simple
    ' method call to do this.
   
    ws.Range("A2").CopyFromRecordset rs
   
    ' Size columns to fit the data.
    ws.UsedRange.Columns.AutoFit

    ' Name the worksheet.
    If Len(WorksheetName) > 0 Then
        ws.Name = WorksheetName
    End If

    ' Save the workbook.
    wb.SaveAs WorkbookPath
   
    ' Terminate the Excel application.
    appExcel.Quit

End Sub

The above function can can be called to copy a table or query into a new Excel workbook.

jw0914 say: ONLY WAY to alert the Exchange Server Team there's an issue**

This issue has existed since 2012, however until Microsoft re-vamped their website for Windows 10, it was possible to solve it via submitting the Outlook 2013 Sync Error Request Form:

  • http://support.live.com/eform.aspx?productKey=outlooksync&ct=eformts&scrx=1

However it's a broken link that now directs you to:

  • windows.microsoft.com/en-us/windows/support#1TC=windows-10

 The KB Article that applied to this was KB 2802703:

  • http://support.microsoft.com/kb/2802703

However this is now a dead link with KB 2802703 nowhere to be found.

The above solutions came directly from MVPs:

  • I'm getting error “Too many devices syncing with this account” after configuring Hotmail account with Outlook 2013 (Thread 4)
  • I'm getting error “Too many devices syncing with this account” after configuring Hotmail account with Outlook 2013 (Thread 7)
    • There were so many threads over the past three years, they kept getting locked and renamed with the sequential thread number in parenthesis

With the links now broken, there is no way for users to have this fixed as this is an issue that

  • The "too many devices" does not refer to physical devices, as the reason for this error is the exchange server has reached the set threshold for allowed syncs (i.e. send/receives)
    • Even if it did, there's no way for users to delete synced devices from the exchange server.
  • "Syncs" refers to each time the exchange server is contacted for a send/receive on any device and within Outlook 2013. 
    • The only way to fix this is to have the Exchange Server Team manually delete all the thousands of accumulated syncs.
  • The threshold Microsoft has set on their exchange server for public emails (i.e. Hotmail, Live, and Outlook addresses) is in the thousands (probably above 10k), however that is easily surpassed if you use Outlook 2013 for several hours a day over 1 - 2 years
  • This causes issues in other areas due to everything using a Microsoft SSO (Single Sign On) has to sync to the Microsoft account connected to the SSO system.

There is a workaround since Microsoft screwed up their own website & downed the only link to the Outlook 2013 Sync Error Request Form

  • Sign into account.live.com and find the page that lists your devices, then delete the non-gaming devices.
    • Once done, you should be able to sync via Outlook, however for the next 24 - 48 hours you will still see the error in Outlook.  All emails should still be able to be synced however, even though you'll continue to see the error for 1 - 2 days.
  • If emails aren't be pushed to Outlook via auto syncs, you may have to manually click send and receive (or update folder) until Microsoft's Live servers clears the device you deleted from your account.

Perhaps someone at Microsoft thought since you can delete devices associated with your account from account.live.com there's no need for the link to the form, however the devices page isn't meant for this purpose.

  • Case in point:
    • I only had 4 devices registered, an Xbox 360, Xbox One, Windows 10 Pro PC, and a Windows 10 Pro VM. The only device email was set up on was the Windows 10 Pro PC, yet I was still receiving the "too many syncs" error.

If you could please inform the proper departments/individuals the links above are broken and need to be fixed, it would be much appreciated since the Outlook 2013 Sync Error Request Form is the ONLY way to alert the Exchange Server Team there's an issue and have them manually delete all the thousands of accumulated syncs for the user's account.
