jllatham say:

CharAbeuh and Ashish Mathur

I was able to get sample data from ElsaR to test with.  One thing I found almost immediately was that the employee ID entries in the DEFT workbook were numbers-as-text, while the related entries in the history workbook were true numbers. 

She also had an on-again, off-again problem 'seeing' the history workbook from the DEFT workbook; that turned out to be because sometimes she was opening the files in separate instances of Excel.

I ended up writing a UDF for her and then creating a workbook using a bunch of code I blatantly borrowed from Chip Pearson's site to copy the module from that workbook with the UDF in it into each of her 142 .xlsx files and then save the files as .xlsm files.

At this time things appear to be going well for her.  She'd used the UDF in one DEFT workbook after dealing with everything (and she's using it all in Excel 2007 now) and it is giving her results.    The UDF is a bit slow because I didn't know how things were set up in the history workbook when I wrote it, so it examine each and every one of the 238,000+ rows of data in it for each of the 20+ entries on each of the DEFT worksheets.  But at least it seems to be getting the job done for her and she might even make her deadline if the code holds up and she has enough coffee to keep her up for a while this evening.

I found out that the data in the history file is sorted so that all of any given employee ID entries are grouped together, and I've fine tuned the UDF to use .Find to go to the first match quickly and then just examine entries until the IDs no longer match.  Great performance improvement that I'll send to her later if it turns out that this is going to be a routine process for her in the future.

One final hint I provided her with was that she would probably want to use Edit/Copy --> Edit Paste Special w/Values to remove the 'dependence' on the history file after the pay rate values had been returned by the UDF formula on the worksheet(s).

In exchanging emails and files with her, she was most appreciative of all the help that we had all tried to give her in this thread. 

Just to kind of wrap up my end of it here, I'm posting the final UDF code here.

Function FindPayRate(EmpID As Range, RateDate As Range) As Variant
'revision 003
'this version works on the presumption that the data in the 'History' workbook
'is sorted so that the employee (BEMP) numbers are all grouped together rather
'than being mixed within the list of BEMP numbers there.
'
  Const historyWBName = "R140-819 History.xlsx" ' name of the history workbook
  'next is name of the sheet in the History workbook with the data table
  Const historyWSName = "R140-819 History"
  'just to make it easily modified if your table
  'layout changes in the future
  Const historyIDCol = "B"
  Const historyDateCol = "J"
  Const historyRateCol = "I"

  'working variables
  Dim historyWB As Workbook
  Dim historyWS As Worksheet
  Dim histIDList As Range
  Dim anyHistID As Range
  Dim offset2Date As Integer
  Dim offset2Rate As Integer
  Dim leastDaysDiff As Integer
  Dim leastDaysRate As Long
  Dim lastRow As Long
  Dim empIDValue As Long
  Dim historyRowOffset As Long
  '
  Application.Volatile
  If ActiveSheet.Name <> EmpID.Parent.Name Then
    'if you aren't looking at the sheet with formula(s) on it, then
    'no sense wasting time recalculating the formula results, just quit.
    Exit Function
  End If
  On Error Resume Next
  Set historyWB = Workbooks(historyWBName)
  If Err <> 0 Then
    FindPayRate = "WB Closed"
    Err.Clear
    On Error GoTo 0
    Exit Function
  End If
  Set historyWS = historyWB.Worksheets(historyWSName)
  If Err <> 0 Or historyWS Is Nothing Then
    FindPayRate = "WS Missing"
    Err.Clear
    On Error GoTo 0
    Exit Function
  End If
  On Error GoTo 0
  'now to do the real work
  lastRow = historyWS.Range(historyIDCol & Rows.Count).End(xlUp).Row
  If Not IsEmpty(historyWS.Range(historyIDCol & Rows.Count)) Then
    lastRow = Rows.Count
  End If
  Set histIDList = historyWS.Range(historyIDCol & 1 & ":" & _
   historyIDCol & lastRow)
  offset2Date = Range(historyDateCol & 1).Column - _
   Range(historyIDCol & 1).Column
  offset2Rate = Range(historyRateCol & 1).Column - _
   Range(historyIDCol & 1).Column

  empIDValue = Val(EmpID.Value) 'convert numbers-as-text to number
  'assumes that history list is sorted so that all
  'employee IDs are grouped together
  'also assumes all employee IDs on the history sheet
  'are true numbers
  Set anyHistID = histIDList.Find(what:=empIDValue, after:=historyWS.Range(historyIDCol & 1), lookat:=xlWhole)
  If Not anyHistID Is Nothing Then
    historyRowOffset = 0
    Do While anyHistID.Offset(historyRowOffset, 0) = empIDValue
      If anyHistID.Offset(historyRowOffset, offset2Date) <= RateDate Then
        'have to find out if it is closer to date than
        'any other dates for the employee found so far
        If leastDaysRate = 0 Then
          'first possible match, save information as baseline
          leastDaysRate = anyHistID.Offset(historyRowOffset, offset2Rate)
          leastDaysDiff = RateDate - anyHistID.Offset(historyRowOffset, offset2Date)
        Else
          'test if closer date
          If (RateDate - anyHistID.Offset(historyRowOffset, offset2Date)) < leastDaysDiff Then
            'date is closer, save the information
            leastDaysRate = anyHistID.Offset(historyRowOffset, offset2Rate)
            leastDaysDiff = RateDate - anyHistID.Offset(historyRowOffset, offset2Date)
          End If
        End If
      End If
      historyRowOffset = historyRowOffset + 1
      If anyHistID.Row + historyRowOffset > lastRow Or _
       anyHistID.Row + historyRowOffset = Rows.Count Then
        Exit Do
      End If
    Loop
  Else
    leastDaysRate = 0
  End If
  FindPayRate = leastDaysRate

'housekeeping
  Set histIDList = Nothing
  Set historyWS = Nothing
  Set historyWB = Nothing
End Function
