amrita m say:

Hi,

 

·        Was there a force shut down of the computer before you started it today?

 

This process of checking the file system is known as check disk. Check disk is automatically initiated if the computer was not shut down properly or if something had gone wrong while the last shut down. The check disk checks for the consistency of the hard drive. There are a few initial tests that the hardware or software (Operating System) can check at start up that may indicate there is a problem with the "consistency" of the hard drive. Your OS is running ‘chkdsk’ which is a program to fix those programs.

The tool then tries to repair any that it finds. For example, Chkdsk can repair problems related to bad sectors, lost clusters, cross-linked files, and directory errors. Problems such as internal file allocation tables, file integrity, file pointers, and many more can easily get corrupted and sometimes just as easily fixed.

 

If the test starts happening more frequently, then your hard drive could be failing and I would suggest replacing it before it does. If it has happened just for once you need not worry about it as it could also be a routine to check the consistency of the hard drive.

Windows 10 Not Seen on Network Problem - Microsoft Community

douglas545valoro say:

Windows 10 Not Seen on Network Problem

I have 3 computers on my local network.

Two of them are running windows 10 and one is running windows 7.

The windows 7 computer can see both windows 10 computers, and each windows 10 computer can see the windows 7 computer on the local network.  But the windows 10 computers do not see each other. 

a. user say:

Hi Douglas,

Thank you for posting your query in Microsoft Community.

I would suggest you to change to the following options for both the systems on the network in the advanced sharing settings (Right click on the network icon on the taskbar, click Open Network & Sharing Center > Change Advanced Sharing settings):

 

- Turn on Network Discovery

- Turn on file and printer sharing

- Turn on sharing so anyone with network access can read and write files in the public folders

- Turn off password protected sharing

 

Now, try to map the drive following these steps:

1. Press Windows key + E, click Computer and click on Map network drive from the options given under the address bar.

2. In the Drive box, click the drive letter you want to use.

3. In the Folder box, type \\server\share, where you can type the IP address of the drive to be mapped. For example, if the IP address is 192.168.6.xxx, you can type \\192.168.6.xxx.

Finish.

Hope this information is helpful. Do let us know if you need any further assistance, we will be glad to assist you.
