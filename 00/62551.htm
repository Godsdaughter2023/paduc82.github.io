Not unless there is something wrong with the part the drive that the Restore Points are
written on. That it would remove them is possible though unlikely. If you want to see
if the drive has error yet not have them repaired then run CheckDisk (chkdsk) and
do not check the FIX or RECOVER options. Checkdisk will then tell you if there are
any errors and where they are. Then if needed and there are errors you can run
CheckDisk again with the FIX ( /f ) and in needed the RECOVER (  /R ) options.

To just check the drive without affecting it uncheck the Boxes show here in Method 1
#8 and then no data will be affected if there are errors found.

How to Run Check Disk at Startup in Vista
http://www.vistax64.com/tutorials/67612-check-disk-chkdsk.html

======================================================

CheckDisk will show its finding with an on-screen report however if you wanted to
read the log later use these methods.

CHKDSK (short for Checkdisk) - <-- see the Vista log here
http://en.wikipedia.org/wiki/CHKDSK

Checkdisk log is in EventViewer which is easy to read with the utility MyEventViewer.

Look in the Event Viewer to see the log.

right panel - Event Wininit - click Event Properties - new window will open with the log you want. 
Scroll as needed. Event ID is 1001.

should be easy enough.
