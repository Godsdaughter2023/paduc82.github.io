sheenali_msft say:

Hi Karian,

It is not feasible to choose Office for Business as a default program as Windows does not differentiate between the two Office versions.

Based on my test, even if you install Office 365 Pro Plus and Office 365 Business on a computer, there will only be one Office program on your computer. When your Office 365 Pro Plus subscription expires, each user in the organization will get a prompt that their license to Office has expired and they need to log in to prove they have a valid license. After the user logs back in, Office will silently make the necessary changes, and you don't need to do anything.

If you would still like to uninstall Office 365 Pro Plus and you are familiar with Group Policy, it's an easy way to automatically uninstall your Office Pro Plus on multiple computers at once.

You will need to create a script that deploys your offline installer (Office Deployment Tool) in configuration mode (/configure) and configure the configuration.xml to use the Remove element to uninstall a specific product.

This is the only specific element that you need to write inside you configuration file to uninstall Office 365 Pro Plus.

Note that you will need to change the Language ID value to the one that is currently installed, otherwise it won't do anything.

<Configuration>

<Remove>
    <Product ID="O365ProPlusRetail" >
      <Language ID="en-us" />
    </Product>
</Remove>

</Configuration>


There is a dedicated forum for Group Policy that you can explore if you would like to learn more.

There are also several other ways to uninstall your Office product that you can look into from here.

Given those UDF's, you can count all of the cells that have a particular interior color and also contain a certain value with a variation of this formula:

=SUMPRODUCT((GetCellColor(Rng)=GetCellColor(B1))*(Rng=A1))

This formula produces two arrays consisting of TRUE or FALSE

The first:  GetCellColor(Rng)=GetCellColor(B1)

Compares the color in Rng -- the range you are testings; with the color of a reference cell;

The second:    Rng=A1

Compares the contents of each cell in Rng with the contents of a reference cell

Since TRUE = 1 and FALSE = 0, multiplying the two arrays gives an array of 1's for where both conditions are met, or 0's, if both conditions are not met.

We then add the results.

I would suggest, in constructing your references, that you try to limit it to the minimal range size really necessary, as execution time will be faster and it will use fewer resources.

byhamc say:

The code you've supplied simply 'grabs' the colour index for a given cell, but this isn't the same function you have given in your formula.

Given the name of the function 'CountCellsByColor', I'm assuming this just returns a numerical value, in which case, I don't think combining this with your IF statement will yield the results you require.

Instead, I'd go back to basics within VBA.

Use the following VBA Function code:

Function CountColVal(rng As Range, ColCheck As Range, Optional ValCheck As Boolean) As Long
 Dim c As Range
 
 Application.Volatile
 
 For Each c In rng
     If c.Interior.ColorIndex = ColCheck.Interior.ColorIndex Then
        If ValCheck = True Then
            If c.Value = ColCheck.Value Then
                CountColVal = CountColVal + 1
            End If
        Else
            CountColVal = CountColVal + 1
        End If
    
     End If
 Next
 
 End Function

Then, as an example, in cell B1 type =CountColVal(A3:A100,A1,TRUE) where

A3:A100 is the range to be counted

A1 contains both the cell colour AND value parameters to be counted

TRUE indicates Value to be checked (FALSE or omitted and only colour check will be completed)
