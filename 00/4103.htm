fermatd say:

=VLOOKUP(Counties!D71, Counties!A2:B67,2,FALSE)


To start off, add a 'FALSE' at the end of the formula.  That might take care of it.  You need to tell Excel if you want anexact match or a close enough match to your criteria. 

It sounds like you are not bringing back to the proper column number, though you only have two columns in your lookup array, so 2 is really your only option.  Is it possible that your array should extend to column C (or beyond)? =VLOOKUP(Counties!D71, Counties!A2:C67,2,FALSE)

And you may want to make your lookup array an absolute reference, like so: =VLOOKUP(Counties!D71, Counties!$A$2:$C$67,2,FALSE)

See if one of those suggestions works for you.

I'm assuming you have selected the option that right-aligns the page number? In that case, by default, index entries have a hanging indent. Click Modify. Click the Format drop down, and then click Paragraph. You'll find the value for hanging indent on the Indents and Spacing tab. When you are done, click OK twice.

suzanne s. barnhill say:

If you use References | Index | Insert Index, you get the dialog below. If you don't change any of the settings--in particular, don't check the box for "Right align page numbers"--you'll get an index where the page numbers are separated from the entries by a comma and space, as shown. That's why I say it's the default layout.

The syntax of the default INDEX field is { INDEX \c "2" \z "1033" }. The syntax of the field with the "Right align page numbers" box checked is {  INDEX \e "<tab>" \c "2" \z "1033" }. You can press Alt+F9 to display the syntax of your current index field. If you delete the \e switch and the quoted tab character, then update the field, you'll get an index with the default comma and space.
