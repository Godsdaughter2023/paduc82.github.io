tina chen msft say:

Hi Shoaib，

 

Thank you for sharing the experience with us.

 

My understanding of the situation is that you set the mail merge to work when sync the Excel file data to the local computer with OneDrive sync client, however since the mail merge preset document has the setting that uses your local Excel data file, other users cannot get the mail merge to work as they cannot access the Excel file on your computer.

 

So if other users can access your local Excel data file, the mail merge preset document may work for them.

 

If all users use domain added computers, you may try Windows network share to share the Excel data file with those users.

Open the OneDrive sync folder on your computer, find the Excel data file, right click it and select Give access to>Specific people, then add a user, give him Read permission of the file, click Share.

When the user gets the link and open it in File explorer, check it they can access the Excel file, then check if they can run mail merge in the Word file template.

Feel free to share with us the test result.

rich michaels - mvp (greatcirclelearning.com) say:

john mcghie say:

Basically, you can't.  The encryption was upgraded to the latest standard in Word 2011: and the entire document is now encrypted.  The password is NOT stored in the document these days, so you end up having to crack the encryption on the entire document.

Effectively, it's too tough for a home computer: you need a rack of servers and several days or weeks.

As Rich says, if you are entirely comfortable handing your credit card details to someone who cracks passwords for a living, there are several services that will offer to try, if you send them the document to attack on their servers. If they succeed (and most of them won't...) do you really think that they won't sell the content to anyone else?

The .exes you see advertised are generally only capable of retrieving the password from the obsolete .doc format, where it was stored in the document, in a predictable place.  Some of them will attack a .docx, but generally have severe "restrictions" such as only being able to retrieve passwords no longer than 5 characters provided they are the only program running for three or four days.

I have not heard anyone (other than the advertisers) claim that any password cracker will get a complex password from the current format.

a. user say:

Hi,

There is no way to use Outlook Web Access with Windows Live Mail or any email client besides Outlook and Outlook must be version 2003 or later (this is known as Outlook Anywhere or RPC over HTTP).

Outlook Web Access uses a web browser, period. Now, you definitely CAN  connect Windows Live Mail to an Exchange Server, but you must use either POP/SMTP or IMAP, both of which an Exchange Server supports, though the Exchange Administrator must enable them.
