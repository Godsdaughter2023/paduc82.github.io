bob jones aka: cybertaz say:

What you've added to the Ribbon basically is the same as the Styles Pane already included on the Home tab but with fewer features.

The built-in keyboard shortcut to launch the pane is Shift+Option+Command+S. A custom keyboard shortcut can be assigned as well. I don't believe the Style device can have a keyboard shortcut assigned to it, but even if it were possible one major advantage of the Styles Pane is that the keystroke will toggle the display of it regardless of what currently may be the foreground tab of the Ribbon, whereas each time you want to use the device you've added you have to return to the Home tab.

Just in case you aren't aware, keyboard shortcuts can be assigned directly to individual Styles, as well. If you want efficiency & expedience, that might be something to consider.

bob jones aka: cybertaz say:

It's correct that the Styles Pane doesn't support keyboard navigation, but I fail to see how that is overcome by the Styles combo box device. Please don't misunderstand: I'm not trying to be argumentative, nor is it my goal to change your mind, but you have to use the mouse either way & none of the other actions you describe can be done using that list, either. I've found it to be inefficient for a number of reasons, especially when there are multiple Styles with the same starting characters & because it always lists all Styles. At least the Styles Pane provides some degree of control over what it displays.

Another new feature (introduced in Word 2011) is the Quick Style Gallery which displays only the Styles you choose to include. Have you had an opportunity to explore that?

In order to quickly create a new Styles by example, assign a keystroke to the NewStyle command. That makes it a process just as simple as what you described once you select the text:

  1. Execute the keystroke
  2. Type the name
  3. Press return

Your hands never even leave the keyboard.

If you're really want to speed things up a keyboard shortcut also can be assigned to the FormatStyleByExample command which bypasses the naming step & simply names it as StyleN, starting with Style1. The newly created Styles can be renamed &/or revised when time permits.
