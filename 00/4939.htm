don cuthbert say:

First try disabling Secure Boot in your UEFI settings (if you have it enabled). See if that helps.

You might try replacing the EFI System partition, if your device is in UEFI BIOS mode. This would mean booting from install media and then getting a Command Prompt (easiest is SHIFT-F10 from first screen of Windows setup).

Then run Diskpart to delete the 100MB FAT32 EFI/System partition, recreate and format it FAT32. Exit Diskpart and run bootsect and bcdboot to reinstall the System files to the newly replaced System partition.  Of course if you deleted the wrong partition, you could lose everything, so this is not to be attempted unless you are sure that you are working with the right partition in diskpart.

With a Legacy BIOS system, it is a bit more complex, because the Legacy System partition usually also has Windows RE (winre.wim) on it in a hidden folder.

As a test to see if this would even help, you could create a bootable USB stick that has a BCD pointing to boot Windows on the hard drive. The USB stick would basically substitute for your System partition only. You would just need to know how to get your PC to boot from USB.

For example, if USB is F: and your Windows partition is C:, you could format a USB stick as FAT32, then run:

bootsect /nt60 F: /force

bcdboot C:\Windows /s F: /f UEFI

That would make a USB stick on F: bootable, but the BCD on F: would boot Windows on C:.

You might need to initialize the USB as a GPT disk first with diskpart, but I am not sure if it is necessary.

brian tillman [outlook mvp 2007-2019] say:
Again, do NOT export or import. Backing up a PST isn't done in Outlook, it's done by simply copying the original PST using File Manager. Moreover, changing servers doesn't mean you need to change PSTs. With IMAP accounts and a version of Outlook more recent than Outlook 2010, the account uses an OST, not a PST. To get the information in the OST to stick around when changing accounts, you create a new PST while still connected to the old server, the copy the contents of each folder in the IMAP account to the PST. Do not export. You should 1) be subscribed to each folder you will wish to transfer, and 2) open each folder so that Outlook synchronizes the contents of each folder between the server copy and the local copy. The you can copy the folders themselves and get all the messages. Finally, after creating the new account in Outlook, you'll be able to copy the non-default folders to the new server and the contents of the default folders to the new server.

rajkul say:
This problem occurs because the destination worksheet and the range for the new PivotTable are specified in the recorded macro. Therefore, the recorded macro is not dynamic. Additionally, the recorded macro cannot be played repeatedly.

Resolution
To resolve this problem, follow these steps:
  1. In the Microsoft Visual Basic Editor (VBE), locate the code for the recorded macro.
  2. Empty the value of the TableDestination variable as in the following example. 
       TableDestination=""
  3. Remove the following lines from the code. 
       Sheets.add

     Sheets ("sheetname").Select
   Cells (x,y).Select
  4. On the File menu, click Save.
Note The modified code for the example in the Symptoms section resembles the following. 
Sub Macro1()
'
' Macro1 Macro
'
    ActiveWorkbook.PivotCaches.Create(SourceType:=xlDatabase, SourceData:= _
        "Sheet1!R1C1:R12C2", Version:=xlPivotTableVersion12).CreatePivotTable _
        TableDestination:="", TableName:="PivotTable1", DefaultVersion _
        :=xlPivotTableVersion12
End Sub
