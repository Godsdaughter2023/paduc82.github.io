That set of symptoms makes me think that you're using the text and checkbox content controls (the ones that are visible on the ribbon, rather than the ones on the dropdown from the Legacy Tools button).

  • Select the content controls plus the text you want to protect.
  • In the Controls group, click the Group button and choose Group from the menu.

You can group the controls with only part of the remaining text, and you can make several (non-overlapping) groups with unprotected areas between them. macropod say:

A Word document's metadata are not a reliable indicator of any aspect of the document's history. All of it is subject to manipulation, deliberately, or accidentally, innocently or culpably.

The 'Last Printed' date, for example, will only be valid if the document was saved after being printed and before closing. That said, a Last Printed date of 16/12/2014 may only reflect when the document's template was last printed from a template. If so, all one can conclude from this is that the document wasn't saved after printing and before closing and that the Modified date of 02/02/2015 represent a save before the document was printed.

In any event, none of this says anything about whether you received the document. How to Connect

Single Object Data Collection

Use this for instances where you are certain that the object you are collecting is the only one in existence and that there are no conflicts with other accounts. Pick the command according to the object type you are dealing with. 

# Replace <*ID> with the UPN, email address, or GUID of the account you are querying for
# Mailbox
get-mailbox <mailboxID> | fl > mailbox.txt
# Soft-Deleted Mailbox
get-mailbox -softdeletedmailbox MmailboxID> | fl > soft-deleted-mailbox.txt
# Mailuser
get-mailuser <mailuserID> | fl > mailuser.txt
# Soft-Deleted MailUser
get-mailuser -softDeletedMailuser | fl > soft-deleted-mailuser.txt
# Contact
get-mailcontact <mailcontactID> | fl > contact.txt
# Mail-Enabled Public Folder
get-mailpublicfolder <publicFolderID> | fl > pf.txt
# Distribution Group
get-DistributionGroup <DistroGroupID> | fl > DG.txt
# Mail-Enabled Security Group
get-group <SecurityGroupID> | fl > SecGroup.txt
# Dynamic Distribution Group
get-dynamicdistributiongroup <DDGroupID> | fl > DDG.txt
# Normal user account - no mail attributes
get-user <userID> | fl > userAcct.txt

Multiple Object Data Collection

This is for scenarios where you are dealing with a conflicting attribute value on one or more objects. The commands below will query for every type of recipient in Exchange Online for the identifier (UPN, email address, mailbox GUID, archive GUID) that you provide. 

$recipient = get-recipient <recipientID> -ResultSize 10 # Enter the UPN, emailaddress, or GUID for <recipientID>
 
foreach ($recip in $recipient) 
{ switch ($recip.recipientType)
{ "usermailbox" {$filename = "mailbox-" + $recip.identity + ".txt" ; get-mailbox $recip.identity | fl > $filename}
"mailuser" {$filename = "mailuser-" + $recip.identity + ".txt" ; get-mailuser $recip.identity | fl > $filename}
"mailcontact" {$filename = "mailcontact-" + $recip.identity + ".txt" ; get-mailcontact $recip.identity | fl > $filename}
"publicfolder" {$filename = "mailpublicfolder-" + $recip.identity + ".txt" ; get-mailpublicfolder $recip.identity | fl > $filename}
"mailuniversaldistributiongroup" {$filename = "distributionGroup-" + $recip.identity + ".txt" ; get-distributionGroup $recip.identity | fl > $filename}
"mailuniversalsecuritygroup" {$filename = "mailsecuritygroup-" + $recip.identity + ".txt" ; get-group $recip.identity | fl > $filename}
"dynamicdistributiongroup" {$filename = "dynamicDG-" + $recip.identity + ".txt" ; get-dynamicdistributiongroup $recip.identity | fl > $filename}}}
 
$softDeletedMailbox = get-mailbox <recipientID> -softdeletedmailbox # Enter the UPN, emailaddress, or GUID for <recipientID>
if ($softDeletedMailbox -ne $null) {foreach ($item in $softDeletedMailbox){$filename = $item.name + "-softDeletedMBX.txt" ; $item| fl > $filename}}
 
$softDeletedMailuser = get-mailuser <recipientID> -softdeletedmailuser # Enter the UPN, emailaddress, or GUID for <recipientID>
if ($softDeletedMailuser -ne $null) {foreach ($item in $softDeletedMailuser){$filename = $item.name + "-softDeletedmailuser.txt" ; $item| fl > $filename}}
 
$user = get-user <UserPrincipalName> # Enter the UPN for <UserPrincipalName>
if ($user -ne $null) {foreach ($u in $user){$filename = $u.name + "-userAccount.txt" ; $u| fl > $filename}}

$msoluser = get-msoluser -UserPrincipalName <UserUPN> # Replace <UserUPN> with the UserPrincipalName

LDIFDE with DistinguishedName

How to Connect

If the Exchange Management tools are installed in the customer's forest (they should be), then you can use the Exchange Management Shellto easily obtain the DN of the account. You can also run the ldifde tool to create a full dump of the object from Active Directory.

This method works both for collecting single and multiple objects. Since we are invoking a different tool other than Exchange to dump the attribute values, we are not dependent on Exchange Shell and its decision to include/exclude certain attributes based on the account type we are querying for. 

# Any mail-enabled object
$DN = (Get-Recipient <recipientID>).DistinguishedName # Replace <recipientID> with the account's UPN, email-address, exchangeGUID - whichever applies
ldifde -d $DN -f "OnPrem_ADRecipient.txt"
 
# User accounts that are not mail-enabled
$DN = (Get-User <UPN>).DistinguishedName # Replace <UPN> with the account's UserPrincipalName
ldifde -d $DN -f "OnPrem_ADUserDump.txt"

Single/Multiple Object Data Collection

If you know the domain that the account is in, you can specify it directly the get-adobject command with -searchbase. 

# User account - find by UPN
$upn = "<UserUPN>" # Replace <UserUPN> with the UserPrincipalName
$filename = $upn + "-ADSIdump.txt"
Get-ADObject -Filter 'UserPrincipalName -eq $upn' -SearchBase '<DC=domain,DC=com>' -Properties * > $filename # Replace <DC=domain,DC=com> with the actual distinguishedName of the domain
 
# Find account by email-address
$addr = "<*emailAddress>" # For instance, replace "<*emailaddress>" with "*** Email address is removed for privacy ***" and keep the wildcard, that way we query both for primary and secondary smtp addresses
$filename = $addr + "-ADSIdump.txt"
Get-ADObject -Filter 'ProxyAddresses -like $addr -or Mail -like $addr' -SearchBase '<DC=domain,DC=com>' -Properties * > $filename

If you don't know the domain, you'll need to contact a global catalog server.

$upn = "<UPN>" # Replace <UPN> with the UserPrincipalName
$filename = $upn + "-ADSIdump.txt"
$GCList = (Get-ADForest <RootForest.com>).globalcatalogs # Replace <RootForest.com> with the actual root forest's fqdn (fully qualified domain name)
$Server = $GCList[0] + ":3268"
# Replace below <DC=RootForest,DC=com> with the actual distinguishedName of the forest.
Get-ADObject -Filter 'UserPrincipalName -eq $upn' –Server $Server -SearchBase '<DC=RootForest,DC=com>' -Properties * -SearchScope subtree > $filename
