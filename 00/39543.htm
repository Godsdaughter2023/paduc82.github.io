If your PC is updated and the issue still occurs, let's determine the cause of the issue by using the System File Checker (SFC) tool to scan possible corrupted files, and help restore them. Here's how:

  1. Open an elevated prompt. Press Windows key + X, and then right-click on Command Prompt (Admin).
  2. At the command prompt, type the following command, and then press Enter:
                   sfc /scannow

Note: The scan may take some time, so please be patient. Windows will repair any corrupted or missing files that are found. The scan results will be shown after this process is finished. You may refer to this article for additional information.

After performing the scan, let's also run the DISM (Deployment Image Servicing and Management) tool to repair Windows corruption errors, whether the SFC tool found errors or not. Follow these steps:

  1. Open an elevated prompt. Press Windows key + X, and then right-click on Command Prompt (Admin).
  2. At the command prompt, type the following command, and then press Enter:

DISM.exe /Online /Cleanup-image /Restorehealth

See image below.

It may take several minutes for the command operation to be completed. After running DISM, it's still a good idea to re-run SFC /scannow to make sure all the issues were resolved.

To ensure minidumps are enabled:
Go to Start, in the Search Box type: sysdm.cpl, press Enter.
(In win 8 you can use the windows key "win" + "s")

Under the Advanced tab, click on the Startup and Recovery Settings button.

Ensure that Automatically restart is unchecked.

Under the Write Debugging Information header select Small memory dump (256 kB) in the drop down box (the 256kb varies).

Ensure that the Small Dump Directory is listed as %systemroot%\Minidump.

OK your way out.

Reboot if changes have been made.

If minidumps are still not being created check these reasons as to why that may be happening

For dumps to be produced The following should be correct

Page file must be on the same drive as your operating system

Page file base allocation size must be greater than installed physical RAM

Windows Error Reporting (WER) system service should be set to MANUAL

Set page file to system managed on the OS drive

Set system crash/recovery options to "kernel memory dump"

User account control must be running.

SometimesSSD drives with older firmware do not create DMPS (update firmware)

Cleaner applications like Ccleaner delete DMP files so dont run them until you are fixed.
