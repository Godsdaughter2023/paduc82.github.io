I just solved this problem on a computer that was sending and receiving emails in Live Mail with the .txt and .htm attachments instead of displaying them in the message body.  Turned out it was some corrupted file(s) somewhere in the "Temporary Internet Files" folder or sub-folders 

It was suggested elsewhere to move the TIF folder and/or use CCleaner to delete the files.  Neither of those solutions worked in my case.

The solution was to rename the TIF folder and reboot to force Windows to create a new, empty TIF folder.  However, turns out even this is not as straightforward as it sounds.  Windows Vista and Win 7 lock and hide certain folders even with Windows Explorer set to show hidden and system files and folders, and even in Safe Mode.  

I used a third party file manager named Unreal Commander which showed all the files and folders but I still had to go into Safe Mode to rename the folder.

After renaming the TIF folder and rebooting, everything was working properly again in Live Mail, no more ATT###.txt and ATT###.htm email attachments and the emails were displaying as they should.  And CCleaner didn't go into an infinite loop analyzing the TIF folder as it had before.

Location of the TIF folder:  C:\Users\Username\AppData\Local\Microsoft\Windows\Temporary Internet Files
