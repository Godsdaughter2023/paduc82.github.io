albert d. kallal say:

It not at all "clear" what this "copy" of the PK is all about? I don't believe you need that concept here.

Regardless,

You could just use a me.refresh in place of a me.requery in the sub form code – that way after returning from the popup for further editing, you don’t lose your record position.

If you really need the re query to place/show the record in correct sort order after that popup editing, then this code will work in the sub form:

   Dim lngPK      As Long

   If Me.Dirty = True Then Me.Dirty = False

   DoCmd.OpenForm "myPopup", , , "id = " & Me!ID, , acDialog

   ' 0k, user done, grab PK before we re query the form,

   ' since re query loses our current postion

   lngPK = Me!ID

   Me.Requery

   ' move the record pointer and position back to the record we were working on

   Me.Recordset.FindFirst "id = " & lngPK

Sub AddiShipForm()

    ' https://msdn.microsoft.com/EN-US/library/office/ff861028.aspx

    Dim myNamespace As outlook.NameSpace
    Dim myItems As outlook.Items
    Dim myFolder As outlook.Folder
    Dim myItem As outlook.mailitem

    Set myNamespace = Application.GetNamespace("MAPI")

    Set myFolder = Nothing
    Set myItems = Nothing
    Set myItem = Nothing

End Sub
