vista-windows photo gallery won't open .jpg on desktop but will in - Microsoft Community

showerless say:

vista-windows photo gallery won't open .jpg on desktop but will in a folder on the desktop - why?

I'm using vista and if I put a .jpg image on the desktop I can open it with any normal "open with" program except for Windows photo gallery.

If I create a folder on the desktop, and then move the picture to the folder it opens just fine as normal.  I logged in as a different user on the same computer to test if that desktop was experiencing the same wierdness.  It's works fine like it's supposed to.  Therefore, I conclude that there is some misconfigured "thingy" on my main user desktop for automatically opening image files when they are on the desktop.

How do I restore the default double click of the image so that Windows Photo Gallery opens it on the desktop?

showerless say: OK, I found the reason and the solution.  I would like to thank myself for solving this weird behavior of Vista

the problem happens because a URL (web page link) you have on the desktop is too long.  something like a link for google street maps for example or something else that is more then 64 characters long is causing the windows gallery to screw up.  Why those two different files sitting on the desktop can be related I don't know, but they are.

find the link that is the offender and delete or move it somewhere else and off your desktop and bingo!  your desktop image viewer works again.

HOW TO FIX IT:
I found the long url easy because there were only a few links on my desktop so if you've got a whole bunch and you're not sure which one it is, just create a folder called "url junk" and move all the web links to that directory.  Now you'll see that your desktop works again like it should so you know that one of those that you moved is the bad one.  Now, pull each one out, one at a time back to your desktop and keep testing your .jpg picture to see if it views the image.  One of those URL's will cause it to not display and when you check that url's properties you'll see that it's a super long address.  That is the bad one - get rid of it and now your fixed.
