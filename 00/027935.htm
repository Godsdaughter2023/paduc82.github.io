josé caputo say:

Mirroring task between a Master and a subproject

Hi all, I am starting to use subproject in my projects from now. One of those subprojects I will send to a supplier to fill it up, so there are some tasks that I will need to insert it duplicated from the master to the subproject. Thanks in advance
john - project say:
Jose,
First of all let me caution you that using a liked structure (e.g. master/subproject) bring with it a tendency for corruption. I would be especially concerned that you are sending the subproject to another location (e.g. supplier) to have it updated and then bringing it back to your computer.

The cardinal rules for using a linked structure are, never rename, move or "save off" a file in a linked structure. It is also got practice to NOT operate a linked structure over a network. Doing any of these things will most likely result in file corruption.

My second concern is that you want to duplicate a task that resides in the subproject and have it also reside in the master. With a linked structure that is not necessary since the master will contain a pointer to the subproject and all the subproject tasks will be displayed at master level.

I have two suggestions.

First, do not send file that you have already inserted as a subproject into a master to the supplier for update. Go ahead and send him a file and have him update it. THEN you can insert it into your master when he sends it back.

Second, if you expect to get these updates from your supplier on a regular basis the above approach needs to be modified. Each time you need to send the file back to the supplier for update, unlink it from the master by selecting the summary line insertion point of the subproject in the master and then going to Task/Properties group/Information/Advanced tab. Uncheck the "Link to project" option and save both the subproject and master. Then send the unlinked subproject file to the supplier, get his updates, and build a new master when he returns the file.

Or third, don't use a dynamic master at all. Instead create a static master. To do that uncheck the "Link to Project" in the lower right corner of the Insert Project window.

Hope this helps.

John

You need to burn the ISO file to a disc and then boot from the disc in order to choose which partition Windows 10 Technical Preview will be installed on.
