Office 2011 for Mac: why does "Line Input" in vba ignore - Microsoft Community

dougrsm say:

Office 2011 for Mac: why does "Line Input" in vba ignore carriage returns?

In the previous versions of VBA for Word, Line Input performed correctly, reading a file up to the first carriage return. Office 2011 for Mac ignores carriage returns, making it impossible to access a text file line by line.

What's more, the "Get" command as described in VBA literature yields an error (compile error: expected: list separator) when used to replace Line Input.

Anyone have any info?

a. user say:

<<
Office 2011 for Mac ignores carriage returns, making it impossible to access a text file line by line.
>>

What seems to have happened is that previously (e.g. Word 2004) Word recognised ASCII 13 (CR) as a line end, but now it recognises ASCII 10 (LF) instead.

You could argue that 13, or rather 13,10 is the "Windows" standard and 10 is the "Unix" standard on which Mac OS X is based (i.e. if you create a file in TextEdit) and that VBA should use the local standard. Or whatever. Personally, I think VBA should continue to recognise the standard it always recognised, and I suspect this change is accidental. The VBA documentation suggests that because it says the following (and says 13 + 13 where I think it should say 13 + 10 :-( )

The Line Input # statement reads from a file one character at a time until it encounters a carriage return (Chr(13)) or carriage return-linefeed (Chr(13) + Chr(13)) sequence. Carriage return-linefeed sequences are skipped rather than appended to the character string.

But the best you can probably manage is to use the Word->Help-Send feedback... to say you'd like some other option.

<<
What's more, the "Get" command as described in VBA literature yields an error (compile error: expected: list separator) when used to replace Line Input.
>>

What happens with Get depends on the file open mode (there are aspects to this that I never really understood), but the following general approach seems to work:

Sub getData()
Dim fs As String * 1
Dim i As Integer
Dim s As String

' Can't use "FOr Input"
Open "Macintosh HD:a:myfile.txt" For Binary As 1

While Not EOF(1)
  Get 1, , fls
  Debug.Print Asc(fls)
Wend

Close 1

End Sub

        Application.CutCopyMode = False
        On Error Resume Next
        .DrawingObjects.Visible = True
        .DrawingObjects.Delete
        On Error GoTo 0
    End With

    'Publish the sheet to a htm file
    With TempWB.PublishObjects.Add( _
         SourceType:=xlSourceRange, _
         Filename:=TempFile, _
         Sheet:=TempWB.Sheets(1).Name, _
         Source:=TempWB.Sheets(1).UsedRange.Address, _
         HtmlType:=xlHtmlStatic)
        .Publish (True)
    End With

    'Read all data from the htm file into RangetoHTML
    Set fso = CreateObject("Scripting.FileSystemObject")
    Set ts = fso.GetFile(TempFile).OpenAsTextStream(1, -2)
    RangetoHTML = ts.ReadAll
    ts.Close
    RangetoHTML = Replace(RangetoHTML, "align=center x:publishsource=", _
                          "align=left x:publishsource=")

    'Close TempWB
    TempWB.Close savechanges:=False

    'Delete the htm file we used in this function
    Kill TempFile
    Set ts = Nothing
    Set fso = Nothing
    Set TempWB = Nothing
End Function


Sub Send_Row_Or_Rows_2()
'For Tips see: http://www.rondebruin.nl/win/winmail/Outlook/tips.htm
'Don't forget to copy the function RangetoHTML in the module.
'Working in Excel 2000-2013
    Dim OutApp As Object
    Dim OutMail As Object
    Dim rng As Range
    Dim Ash As Worksheet
    Dim Cws As Worksheet
    Dim Rcount As Long
    Dim Rnum As Long
    Dim FilterRange As Range
    Dim FieldNum As Integer
    Dim strBody As String

 

    On Error GoTo cleanup
    Set OutApp = CreateObject("Outlook.Application")

    strBody = "Hello," & "<br>" & _
              "As a courtesy, we are emailing you a list of contracts currently assigned to your office." & "<br>"
   

    With Application
        .EnableEvents = False
        .ScreenUpdating = False
    End With

    'Set filter sheet, you can also use Sheets("MySheet")
    Set Ash = ActiveSheet

    'Set filter range and filter column (column with e-mail addresses)
    Set FilterRange = Ash.Range("A1:K" & Ash.Rows.Count)
    FieldNum = 2    'Filter column = B because the filter range start in column A

    'Add a worksheet for the unique list and copy the unique list in A1
    Set Cws = Worksheets.Add
    FilterRange.Columns(FieldNum).AdvancedFilter _
            Action:=xlFilterCopy, _
            CopyToRange:=Cws.Range("A1"), _
            CriteriaRange:="", Unique:=True

    'Count of the unique values + the header cell
    Rcount = Application.WorksheetFunction.CountA(Cws.Columns(1))

    'If there are unique values start the loop
    If Rcount >= 2 Then
        For Rnum = 2 To Rcount

            'Filter the FilterRange on the FieldNum column
            FilterRange.AutoFilter Field:=FieldNum, _
                                   Criteria1:=Cws.Cells(Rnum, 1).Value

            'If the unique value is a mail addres create a mail
            If Cws.Cells(Rnum, 1).Value Like "?*@?*.?*" Then

                With Ash.AutoFilter.Range
                    On Error Resume Next
                    Set rng = .SpecialCells(xlCellTypeVisible)
                    On Error GoTo 0
                End With

                Set OutMail = OutApp.CreateItem(0)

                On Error Resume Next
                With OutMail
                    .to = Cws.Cells(Rnum, 1).Value
                    .Subject = "Current HomeTelos Contracts"
                    .HTMLBody = strBody & RangetoHTML(rng)
                    .Display  'Or use Send
                End With
                On Error GoTo 0

                Set OutMail = Nothing
            End If

            'Close AutoFilter
            Ash.AutoFilterMode = False

        Next Rnum
    End If

cleanup:
    Set OutApp = Nothing
    Application.DisplayAlerts = False
    Cws.Delete
    Application.DisplayAlerts = True

    With Application
        .EnableEvents = True
        .ScreenUpdating = True
    End With
End Sub
