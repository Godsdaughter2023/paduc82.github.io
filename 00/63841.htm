dax1792 say:

Since Vista, Windows has used the Volume Shadow Copy Service to create Restore Points rather than copying files, so I doubt that the procedure you describe will still work.

 

I believe it is possible to access the Volume Shadow Copies using the following procedure:

List all shadow copies on the system using the command vssadmin list shadows

Use the mklink command to create a symbolic link to the shadow copy. The command will look something like this

mklink /d c:\old \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy2\   

Optionally, assign a drive letter to the directory with the subst command, for example:

subst g: c\old

You can then access the old contents using either the c:\old path or the g:\ path

 

Whether you could easily access the registry hives like this is something I can't comment on. 

 

You could also check if the contents of C:\Windows\System32\config\RegBack are any use to you.
