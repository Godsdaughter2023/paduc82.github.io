I have put the site in trusted sites


Take it out.   That is just as likely to cause problems unnecessarily.

Also, the host name that you gave redirected for me:

http://www.greenerjourneys.com/corporate/

and there was a favicon, so to simplify things try testing with just the favicon:

http://www.greenerjourneys.com/favicon.ico

and to make sure that you aren't just getting a cached copy, the first time you see it press Ctrl-F5.

Then try the above link to avoid the redirect.

Another thing that you could do, in case the problem is with the lookup, is first flush the dnscache.   E.g. in an Administrator Command Prompt (elevated cmd window) enter:

ipconfig /flushdns

After your IE test you could then enter:

ipconfig /displaydns

If there is nothing showing in that display it would suggest a problem with the lookup.  So see what you get with  nslookup   and a   ping  and then another  ipconfig  /displaydns

nslookup www.greenerjourneys.com

ping  -n 1  -w  1  www.greenerjourneys.com

ipconfig  /displaydns


The switches on ping just indicate that we don't care about ICMP connectivity.  We are just using the command to try to get the lookup loaded into the dnscache so that it can be inspected.
