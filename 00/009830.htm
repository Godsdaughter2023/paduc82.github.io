hansv mvp say:

The line

    Worksheets("Template").Copy After:=Worksheets(Worksheets.Count)

duplicates the Template sheet and places the copy after the last worksheet in the workbook: if the workbook has 7 worksheets, Worksheets.Count = 7, so Worksheets(Worksheets.Count) is equivalent to Worksheets(7), i.e. the 7th and last sheet.

You originally wrote that you didn't want the command button to be copied to the new sheet; now you mention that you want to remove the command button from the sheet that calls the code. That is different.

Try this version. You will have to substitute your own bits:

    ' Declare a variable to use for the old sheet
     Dim wshSource As Worksheet
    ' Declare a variable to use for the new sheet
     Dim wshTarget As Worksheet
     Set wshSource = ActiveSheet
     ' Duplicate the active sheet and place the copy before it
     wshSource.Copy Before:=wshSource
     ' Set a reference to the new sheet
     Set wshTarget = ActiveSheet
     ' Rename the new sheet - substitute your own code
     wshTarget.Name = "new name here"
     ' Delete the command button from the old sheet
     ' Change Button 1 to its actual name
     wshSource.Shapes("Button 1").Delete
     ' Copy some data from the old sheet to the new sheet
     ' Fill in the ranges yourself
     wshSource.Range("...").Copy Destination:=wshTarget.Range("...")
     ' Disable cut/copy mode
     Application.CutCopyMode = False

tom ogilvy say:

Worksheets("Template").Copy After:=Worksheets(Worksheets.Count)

worksheets.count counts the number of worksheets.  So this code places the copy of the template worksheet as the last sheet on the right. 

now HansV sets a reference to this new sheet.  He again uses worksheets.count because this will now return the same count incremented by 1 since a new sheet was added.  Again, this will refer to the farthest sheet to the right in the tab order.

Set wsh = Worksheets(Worksheets.Count)

These could be placed to the left.  

In the code module you can refer to the sheet with the code using the predefined reference "me"

me.copy Before:=me

copies the sheet to the left of the template sheet if the template sheet contains the code.

then set a reference

set wsh = me.previous

but then the code deletes the button on the new sheet and you say you want it to delete the button on the template sheet. 

So I put a commandbutton on the template sheet (I assume it is the only activeX control on the sheet).

I put code like this in the sheet module of the template sheet

Private Sub CommandButton1_Click()
Me.Copy Before:=Me
'Set wsh = Me.Previous
Me.Activate
Application.OnTime Now(), "CutButton"
End Sub

Since that is the code that will be running, I want to break contact with that code to actually delete the button in the template sheet.  So I use the Ontime command to run a macro that deletes the button on the activesheet.   I activate the sheet with the code before I call that routine.  Note that the ontime command is told to run that macro now.

So now a copy is made.  I click the button in this sheet and a new copy is made (and the button on the first copy is deleted).   I click the button on the new sheet and a new copy is made (and the botton on the second copy is deleted). 

This is what I understand you say you want.

this bit of code needs to be placed in a general module - not a sheet module.  You get a general module by doing insert=>module in your workbook in the Visual Basic Editor.

Sub CutButton()
ActiveSheet.OLEObjects(1).Delete
Application.ScreenUpdating = False
Application.ScreenUpdating = True
End Sub
