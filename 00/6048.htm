andreas killer say:

the values of these global variables remain unchanged between every event and button pressed during a session?

A variable is physically a part of your RAM, it is only changed when someone "writes there". (The details how the RAM is written and how it is protected doesn't matter here.)

When you open the file, this happens 2 times. (File opens means all the bits from the file are loaded into memory and the spaces for the variables is reserved, resp. provided.)

The first time some code, that is automatically generated from the VBA compiler, writes zeros into the memory block of the variables, in programming languages this means initialization. (The same code is placed into every Sub and Function to initialize the local variables).

The second time your sub writes some values into there.

And there might be a third time, when you reset a VBA project (click on the square button in VBA editor) or when you change a declaration of a global variable or you code crashes, then all variables are initialized again when you run any code.

Counterquestion: What is the reason for your question?

Andreas.

steve rindsberg say:

Generally, the answer is yes, but as Andreas mentions, there are potential situations that can cause problems.  An unhandled error, for example, might cause your variables to lose their values.

Since it only takes a tiny amount of time to initialize the variables, what you can do is handle all the initialization in a single subroutine and call it at the beginning of any other subroutine that might use the variables.  Not strictly necessary but kind of a belt and braces, safety in numbers approach.

  3. In the Open Backup dialog box, note the folder names that appear. Each folder represents a OneNote notebook that’s stored on your computer.

  4. Double-click the notebook folder you want to open.

  5. Select the notebook section backup file you want to open, and then choose Open.

  1. Open the shared notebook where you lost recent notes.

  2. Choose History > Notebook Recycle Bin > Notebook Recycle Bin.

Any notes that were deleted during the past 60 days should appear in the Notebook Recycle Bin, unless the Notebook Recycle Bin was emptied during that time.

Note: To avoid any confusion between backup notes and your normal notes, be sure to close the Recycle Bin when you’re done inspecting any available backups. Click the green, upward-pointing arrow next to the Deleted Notes section tab to return to the previous level.

I'd like to explain that the class notebook created in Classroom will be stored in the "Site Assets" library in the corresponding group site, it will not be listed under the "Manage notebooks" page in Class Notebook.
