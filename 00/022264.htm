% of Row Total returns #N/A in Power Pivot pivot table - Microsoft Community

pjhorst say:

% of Row Total returns #N/A in Power Pivot pivot table after June update

I believe a bug was introduced with the June update re: performance enhancements with Power Pivot sub and grand totals no longer returning results unless required.

Previously, with a Data Model based Pivot Table, you could have value return results as a % of Row Total, have Grand and Sub totals turned off and the % calculations would still display with values.

Now, in the same scenario, you end up with #N/A.  Turning subtotals or grand totals on, enables the calculation to complete.  The updated indicates the subtotal/grand total will only return if required for the pivot table to render.  I believe this is not actually happening.

As a work around, you can write the measure by hand but the impact is you have to know how your pivot table will be formatted so that you can indicate the correct column in your DAX to roll up to ...Calculate (,[countme],allexcept(table1,table1[known_row)) .  This takes away from the flexibility of the original/desired functionality.

There are other minor impacts, like handling the resulting blanks vs previously calculated 0%.

This is very easy to test with a simple dataset.  The original functionality is still present in a standard pivot table or Excel install without the June update applied.

Hi Jim,

While the code base is the same, the operating system environment is completely different.

The extra text for mouse-over of Ribbon buttons is in the code base. I am sure that the folks who make such decisions at Microsoft made a conscious decision not to display that text. The development team is very open to user suggestions, which is why I urged you to post your suggestion to UserVoice. The number of votes for a suggestion gives the Microsoft Office teams a guide as to how many people agree with the suggestion. Then they make their decisions about what the interface will have.

Help is not the same on both platforms partly due to the fact that the operating systems don't have 100% feature parity. If they did, what would be the point of having Mac, Windows and LINUX as OS options? Office for Mac takes advantage of Mac OS to provide a slick feature when searching Help. When using Help, the Help system highlights menu features and pressing Return activates the feature. This is Mac-only.  Here's a screen shot of the feature in action.

It's not.

You don't have to work too hard to find the PivotTable features in Excel for Mac. They're on the Insert tab of the Ribbon, just like the Windows version: If you have enabled Intelligent Services (and maybe even not), you can use the relatively new Recommended PivotTables feature, which relies on artificial intelligence.

You may be thinking of the Power features in Excel for Windows: PowerView, PowerPivot, PowerMap, Data Model, Get and Transform, etc. Those features are currently under development for the Mac.
