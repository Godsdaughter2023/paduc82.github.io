greginmich say:

[Edit for clarification, 5/3/2016]

There isn’t any “Automatic” setting for the MSE scheduled scan, and there’s no such thing as an “automatic” scan for Security Essentials. The MSE scheduled scan will not run automatically at the scheduled time – it will only run at the scheduled time if the PC is in use at that time, or if the power configuration is changed to prevent the PC from entering a sleep state prior to that time. In other words, an automated scheduled scan just isn’t an available option with the built-in MSE scheduled task like it is for an independently scheduled task:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02

That’s because the MSE task can’t be configured to wake up a sleeping PC, or to run-on-restart when a scheduled start in missed. That means that there’s no provision for an unattended scan or for a catch-up scan with the built-in scheduled task – and these two options are the condiciones sine quibus non for an automated scheduled task.  Where these standard options aren't implemented, a task can only be scheduled – it can’t be automated. That might sound like splitting hairs, but if we don’t make this distinction, then it’s completely impossible to understand why there’s been nothing but endless trouble, confusion, and frustration with the MSE scheduled scan. The Microsoft scheduled scan was only made fully automatic with the introduction of Automatic Maintenance for Windows Defender in Windows 8.

[End Edit]

If you can’t be “in attendance” on a schedule, then just run a manual Quick Scan when you’re “in attendance”. When you know that you’re going to be around for a while, then run a Full Scan. If you feel compelled to use the MSE scheduler, then you’ll have to leave the PC on and manipulate the power configuration (Control Panel > Power Options) such that the PC won’t go to sleep, or such that it won’t go to sleep until after the scan is finished. There are some better solutions, but they're more technical:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/why-did-my-computer-shut-down-by-itself-when-i/915f1eee-d185-49e8-840b-5c7fecc52849
