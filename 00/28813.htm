How to create an auto (on-line) backup of an access database - Microsoft Community

wayne-i-m say:

How to create an auto (on-line) backup of an access database

This gets asked quite a lot.  So I thought I would add a simple method here.  There are other methods

Note - the normal use of something like this would be to backup a BackEnd.  In the code below there is "be.accdb".  If you are not backingup a BackEnd then change this to something like .mdb.

A copy of something on the same computer as the original is not a backup.

It is a fact that the vast majority of people do not do backup their data in a way that will actually help them when things go wrong (and they will).

Imagine this:
(Assume that you run a company offering some form of training)

  • You do Backups in the same way that most people do – to some folder on your computer.
  • You have taken some bookings for tomorrow.
  • You can’t find your computer.
  • The clients turn up, but you don’t as you didn’t know there was a class, or the meeting point, how many people you’re teaching, etc.

 A standard Backup now is a total waste of time.

Think about it this way, what is more likely to happen:

  • The code in your database will somehow delete all your records.
  • You will lose your lap top
  • Your lap top will get pinched
  • You will spill beer into your lap top
  • You will drop your lap top
  • For some reason (could be anything) you can’t use your laptop
  • Etc

I would say the 1st choice is the least likely – but this is what most people protect themselves against.

So you should always make a backup that you can get to from anywhere at any time (even when you’ve dropped a pint of beer over your lap top).

Create File/Folder structure

At   www.hotmail.com

If you don’t have a Hotmail account, create one

At   www.hotmail.com
Sign into your hotmail account and click the Down Arrow next to the Outlook logo
Select OneDrive
Select create

In the Drop Down select Folder
Name the folder “Backup”
Log out of your Hotmail Account

Next

Go to this website
http://windows.microsoft.com/en-gb/onedrive/windows-app-faq#1TC=other

Select Download the Free App

Then select Run in the bar at the bottom of the screen


Allow the Setup to run
Select Get Started
Sign in to your Onedrive (Hotmail) account
During the Setup when you see this screen  Select Change

IMPORTANT – Browse to the folder which holds your database

Select the Files folder

Click OK / next

Select Choose a folder to sync
Uncheck all the folders except the Backup folder (which you recently created in OneDrive)

Click Next
Select Done

Check that the folder containing your database has a sub-folder called OneDrive and that this has a sub-folder called Backup.

The Backup folder is where you will place anything you don’t want to lose – including your database backups.


Note – this folder can be used for other files as well as the database backups.
Simply Drag–and–Drop files into the Backup File to upload it to you OneDrive.


To backup your Database

Note – the normal use of something like this would be to backup the BackEnd.  I(n the code below you’ll see this “be.accdb”.  If you are not backingup a BackEnd then this will need to be changed to, for example “.mdb”, etc.

Open you database

Open a form in design view (normally I would use the switchboard but it can be any form)

Add a button to your form (call it cmd_backup)

Add this VBA to the OnClick event of cmd)backup

Private Sub cmd_backup_Click()

On Error GoTo cmd_backup_Click_Err

Dim strNewBEname As String

Dim strOldBEname As String

Dim strDateStamp As String

Dim bckFolder As String

strOldBEname = CurrentProject.Path & "\" & Left(CurrentProject.Name, Len(CurrentProject.Name) - 4) & "_be.accdb"

strDateStamp = Format(Date, "yyyy_mm_dd")

strNewBEname = CurrentProject.Path & "\" & "\OneDrive\Backup\" & "DB_Backup_" & strDateStamp & "_be.accdb"

bckFolder = CurrentProject.Path & "\OneDrive\Backup"

If Dir(bckFolder, vbDirectory) <> vbNullString Then

FileCopy strOldBEname, strNewBEname

MsgBox "The database has been backed up" & Chr(13) + Chr(10) & Chr(13) + Chr(10) & _ "In your Hotmail - Onedrive folder you will see a file called" & Chr(13) + Chr(10) & _ "DB_Backup_" & strDateStamp & "_be.accdb" & Chr(13) + Chr(10) & Chr(13) + Chr(10) & _ "DB = the type of file" & Chr(13) + Chr(10) & _ "Backup = how the files was created" & Chr(13) + Chr(10) & _ strDateStamp & " = the date the file was created" & Chr(13) + Chr(10) & _ "be = Backend - the section of the database where the data is stored" & Chr(13) + Chr(10) & _ "accdb = the file extension" & Chr(13) + Chr(10) & Chr(13) + Chr(10) & _ "PLEASE LEAVE THIS FILE ALONE" & Chr(13) + Chr(10) & _ "It will be required to restore your database if required" & Chr(13) + Chr(10) & Chr(13) + Chr(10) & _ "If the database needs to be restored from the backup" & Chr(13) + Chr(10) & _ "Contact INSERT NAME OF PROGRAMMER.", vbInformation, "Backup complete"

Else

MsgBox " The system could not find the OneDrive Backup folder." & Chr(13) + Chr(10) & Chr(13) + Chr(10) & _ "Go to Windows Explorer and check the folder exists" & Chr(13) + Chr(10) & _ "Your Database Folder > OneDrive > Backup" & Chr(13) + Chr(10) & Chr(13) + Chr(10) & _ "If the folder exists" & Chr(13) + Chr(10) & _ "There is an error with the folder" & Chr(13) + Chr(10) & _ "Go to your control panel and delete the OneDrive program" & Chr(13) + Chr(10) & _ "Then re-download the OneDrive App.  See Backup Instructions" & Chr(13) + Chr(10) & Chr(13) + Chr(10) & _ "If the folder does not exist" & Chr(13) + Chr(10) & _ "Click the help button and see Backup Instructions", vbExclamation, "Error" 'Note the last line assumes you, the prgrammer, have acctually made some help files?

End If

cmd_backup_Click_Exit:

Exit Sub

cmd_backup_Click_Err:

MsgBox Error$

Resume cmd_backup_Click_Exit

End Sub
