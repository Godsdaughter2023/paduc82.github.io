marshall barton say:

There are a few ways to do that, but they are convoluted and difficult to manage as your app grows.  (Nested IIf expression or a custom VBA function are typical ways.)

 

Instead, you should redesign your table so it does not have those repeating fields.  A properly normalized structure would have a second table for your Level values so you can use the standard  query function Max.  This way would also allow for any number of Level values with out concern for fewer or more than 3 values.

 

If you are unable to restructure your table, then you might be able to get by using a union query as a normalized virtual table.  The union query would look something like:

 

SELECT [primary key field], 1 As LevelNum, Level1 As LevelValue FROM [your table]
UNION ALL

SELECT [primary key field], 2, Level2 FROM [your table]
UNION ALL

SELECT [primary key field], 3, Level3 FROM [your table]

 

Then you can calculate the greatest value for each record in your table using a query like:

 

SELECT [primary key field], Max(LevelValue) As GreatesFieldValue 
FROM [the union query]
GROUP BY [primary key field]

 

Note that I presumed the Level fields are a Number data type with a value of 0 to indicate Rejected (you can display "Rejected" instead of 0 simply be specifying a custom format, e.g. 0;;"Rejected", for a form/report text box or query field).  If you made the Level fields the Text data type then they will not compare correctly and the union query would have to be modified to deal with that issue too.

ken sheridan say: Try the following function, which is adapted from one for returning the maximum value of a variable number of numeric values:

Public Function GetMax(ParamArray aVals() As Variant) As Variant

    Dim var As Variant
    Dim varMax As Variant
    
    varMax = -2147483648#
    
    For Each var In aVals
        If IsNumeric(var) Then
            If Val(var) > varMax Then varMax = Val(var)
        End If
    Next var

    GetMax = IIf(varMax = -2147483648#, "Rejected", varMax)
    
End Function

ken sheridan say:
Disregard the question regarding text VS number --- I see what I did wrong.  I changed the false reading from rejected to 0 (zero).

If you are returning a zero rather than 'Rejected', provided that all other values are positive numbers you can use my function in its original form:

Public Function GetMax(ParamArray aVals() As Variant) As Variant

    Dim var As Variant
    Dim varMax As Variant
   
    varMax = aVals(0)
   
    For Each var In aVals
        If var > varMax Then varMax = var
    Next var

    GetMax = varMax
   
End Function

This will return the highest number out of the values passed into it as the parameter array.  If all are zero it will return zero, but if one or more is greater than zero it will return the highest of those numbers.

jay freedman say:
If you're using the Mark Entry dialog (from Insert > Reference > Index and Tables > Mark Entry), just edit the text in the "Main entry" box before you click the Mark button. For your examples, delete the word "The" from the beginning of the entry and add "(The)" at the end of it.

When you click the Mark button, the dialog remains open, and you can further edit the text to make the other entry, and click the Mark button again. It isn't clear to me whether in the second example you want "(The Key To)" to be a subentry that appears on a second line; for poem titles I don't think that's appropriate.

If you prefer, you can ignore the menu/dialog method and insert entries directly into the text. The first thing is to turn on display of nonprinting characters by clicking the ¶ button on the toolbar, because index entry fields are always marked as hidden text. Then:
  • Type the text you want to appear in the index -- either by copying something from the regular text, pasting it and modifying it; or by typing something that may not appear in the text at all.
  • At the beginning of that text, type the letters XE and a space. For the examples, you would type
                    XE Key To Undershore (The)
                    XE Undershore, (The Key To)
             or, if you want the second one to be main-entry-and-subentry, then
                    XE Undershore:(The Key To)
  • Select the text from the XE to the end of the entry and press Ctrl+F9 to make it a field.

ken sheridan say: An ID column will in most cases be a long integer number data type, including where the 'lookup field' wizard has been used.  Number data type values do not need to be delimited with quotes characters, so the syntax would be:

Me.CoursePrice = DLookup("CoursePrice", "tblintschedcourse", "InternalScheduledCourseID = " & Forms!frmaddintbooking.CourseTitle)

Note that the object names do not need to be wrapped in brackets, as they wisely do not include spaces or other special characters.

In this situation the CourseTitle control would usually be a combo box with a RowSource property such as:

SELECT InternalScheduledCourseID, CourseTitle FROM tblintschedcourse ORDER BY CourseTitle;

You can, however, do this more efficiently using a combo box with a RowSource property of:

SELECT InternalScheduledCourseID, CoursePrice, CourseTitle FROM tblintschedcourse ORDER BY CourseTitle;

With the combo box's other properties set as follows:

BoundColumn    1
ColumnCount     3
ColumnWidths:   0cm;0cm;8cm

You can then return the course price in the combo box's AfterUpdate event procedure with:

Me.CoursePrice = Me.CourseTitle.Column(1)

The Column property is zero-based, so Column(1) references the second column, CoursePrice.
