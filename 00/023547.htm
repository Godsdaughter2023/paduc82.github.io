    hansv mvp say:

    Scott's excellent suggestion will work only one field at a time. Here is a variation on the code I posted previously that will handle ALL text fields in a table:

    Sub ChangeCase()
        Dim dbs As DAO.Database
        Dim tdf As DAO.TableDef
        Dim fld As DAO.Field
        Dim strTable As String
        Dim strField As String
        Dim strSQL As String
        strTable = InputBox("Which table do you want to process?")
        Set dbs = CurrentDb
        On Error Resume Next
        Set tdf = dbs.TableDefs(strTable)
        On Error GoTo ErrHandler
        If tdf Is Nothing Then
            MsgBox "Table does not exist", vbExclamation
            Exit Sub
        End If
        For Each fld In tdf.Fields
            If fld.Type = dbText Then
                strField = fld.Name
                strSQL = "UPDATE [" & strTable & "] SET [" & strField & "] = " & _
                    "StrConv([" & strField & "], 3) " & _
                    "WHERE [" & strField & "] Is Not Null"
                dbs.Execute strSQL, dbFailOnError
            End If
        Next fld
    ExitHandler:
        Set fld = Nothing
        Set tdf = Nothing
        Set dbs = Nothing
        Exit Sub
    ErrHandler:
        MsgBox Err.Description, vbExclamation
        Resume ExitHandler
    End Sub

    PAUL MCCARTNEY will become Paul Mccartney. To change it to McCartney would require much more complicated code that checks for exceptions to the "capitalize first letter of each word only" rule.

    keratl say:

    Form checkboxes can appear on sheets or userforms, but either can be linked to a cell (if the cell value is TRUE the checkbox is checked, and visa versa; if the cell value is FALSE the checkbox is unchecked and visa versa).

    Rightclick the checkbox, format control, control tab, add a cell link. The linked cell will be overwritten with either TRUE or FALSE depending on the checkbox state.

    Then replace that TRUE or FALSE in the linked cell with a formula to check if the user entry cell matches your target value. In this example, I linked the checkbox to D2. If desired, hide that helper column so your users don't see the TRUE/FALSE values
