john - project say:

SteveDIY,

First a comment on your duration values. Please note that duration is always expressed in working time as defined by the Project Calendar. Construction projects are normally better described in calendar time (i.e. elapsed time). Duration in hours is unambiguous but days, weeks and particularly months can be problematic. For example, with the default definition of a day (8 hours), a task that runs continuously over 24 hours will show as 3 days duration. Duration in months will never accurately express actual months. For example, a task starting on 6/1/19 with a duration of 2 months will show a finish of 7/25/19, assuming the Standard Project calendar. And with a 7 day Project calendar, that task will finish on 7/10/19. In order for it to truly be a 2 calendar month task, you must enter the duration as 2 emos (elapsed months).

With regard to your missing Gantt bars, I suspect there is a problem with the definition of your Gantt Bar styles (Format > Bar styles group > Format icon > Bat styles). Did you edit the bar styles? If you did not, then I suspect a problem with Project itself and suggest you try a repair through the control panel. If that doesn't help, then I suspect your Global file is corrupt. To fix that, find your working Global, move it to the trash and then re-open Project. A new default Global will be created. Note, you will lose any custom view, tables, filters, etc. with this process.

If you did edit the default Bar styles, take a careful look at the parameters for the various bar styles. Note that if there is a conflict, the bar style that appears latest in the list takes precedence.

Hope this helps.

John

stefan blom say:
Most likely, the digits occupy so much space that the text following the number gets pushed to the next (default) tab stop. To fix this, you'll have to adjust the indentation for level 4 numbers. Click in the first Heading 1 paragraph of your document, and then click Home tab | Multilevel List | Define New Multilevel List (the command, misleading as it is, should be used even when you edit a multilevel list). In the dialog box, adjust the settings for (in particular) "Text indent at" and "Add tab stop at." See also http://shaunakelly.com/word/numbering/numbering20072010.html.

suzanne s. barnhill say:
