ken sheridan say:

The fact that you have a column in the table with a ListRows property suggests that a 'lookup field' is being used.  This may well be a result of the analyze wizard, but in any case 'lookup fields' should be avoided like the plague as they give rise to a number of problems.  You may well have encountered another one.  For more details see:

http://www.mvps.org/access/lookupfields.htm

In table design change the Display Control property of the column to Text Box.  That will show the real values in the column, not the value looked up from another table.  When you want to see the value from the other table either include that table in a query, or use a combo box in a form as the bound control and set it up to show the value from the other table, e.g. to show a city in a form bound to a table of addresses with a CityID foreign key column:

ControlSource:   CityID

RowSource:       SELECT CityID, City FROM Cities ORDER BY City;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column and that the second is at least as wide as the combo box.

Ken Sheridan, Stafford, England
