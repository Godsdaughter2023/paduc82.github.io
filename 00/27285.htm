john w. vinson say:

You're asking for a date that is SIMULTANEOUSLY equal to #4/14/2013# and to #4/29/2013#.

 

There is, of course, no such date!

 

To find records where LeaveDate is between 45 and 60 days ago, you don't need the AgeInDays field at all. Instead, just use a criterion on LeaveDate of

 

>= DateAdd("d", -.[Forms]![frm_reports]![BeginDays], Date()) AND <= DateAdd("d", -[Forms]![frm_reports]![EndDays], Date())

 

Note the minus signs - you're SUBTRACTING BeginDays from today's date to find the beginning of the interval, and the same for EndDays.

 

Note that if you have an index on the LeaveDate field this query will be efficient; if you put a criterion on a calculated field, you don't get any benefit from the indexing.

macropod say:

You put only author names for the relevant chapter in the author field and only editor names in the editor field. If a chapter author is also an editor, the name goes in both fields.

As for adding the a, b, c, etc. suffixes to years, that's going to be problematic. If you don't convert the citation & bibliography entries to static text, the edits will be lost immediately you do anything that causes Word to refresh field codes (e.g. printing the document). The alternative is to add the a, b, c, etc. suffixes to the years in the 'Manage Sources' dialogue itself - which means they'll appear that way in citations and the bibliography alike - but that has implications if you update the master list then want to reference the works in another document but don't reference all of them, since you may end up with unwanted or wrong a, b, c, etc. suffixes there.

macropod say:

Word doesn't support the addition of the a, b, c, etc. suffixes to years. Instead the title & year are both output (though you can suppress either or both).

Referencing a particular chapter does not of itself warrant the addition of Ed. - what you seem to have is a composite work with one of more editors for the work as a whole and authors for individual chapters. In that case, separate citation entries are required for each chapter referenced and you should list the editor(s) in the 'editor' field and only the author(s) of the chapter concerned in the 'author' field. jim gordon mvp say:
The error message  "a version of the software required to install this update was not found" is displayed when an updater can not find the software that you're trying to update.

This could be for a number of reasons. Among the common reasons (in no particular order)

A) One of the main components of Microsoft Office was moved, deleted or renamed. Look in the Applications > Microsoft Office 2011 folder. You should see the following unless you did a custom install:

If anything is missing, then use Spotlight to find the missing component. Then drag it back into the Microsoft Office 2011 folder where it belongs. If this was the problem, the updater will run fine. If you can't find a missing component, you will have to completely remove office and then reinstall it from scratch.

B) You're using the wrong updater for the version of Microsoft Office that you currently have. Updaters must be applied to specific versions and in the correct sequence. The best way to update Office is from within any Office application. Use the Help menu and choose Check for Updates to launch AutoUpdate utility. 

C) You never installed Microsoft Office in the first place. The updaters will not install Office for you.

Try this:

 

Sub AutoSum()
    Dim r As Long
    Dim s As Long
    Dim c As Long
    r = ActiveCell.Row
    If r = 1 Then
        MsgBox "You're in the top row!", vbInformation
        Exit Sub
    End If
    c = ActiveCell.Column
    s = r
    Do While Cells(s - 1, c) = ""
        s = s - 1
        If s = 1 Then Exit Do
    Loop
    Do While Cells(s - 1, c) <> ""
        s = s - 1
        If s = 1 Then Exit Do
    Loop
    ActiveCell.FormulaR1C1 = "=SUM(R[" & (s - r) & "]C:R[-1]C)"
    ActiveCell.Font.Bold = True
End Sub
