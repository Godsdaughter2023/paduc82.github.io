This has been a known shortcoming since first introduced... An Excel Object is restricted to being 5 columns by 10 rows.

The only ways to circumvent it are to;

  1. Open the workbook in Excel & copy the data range
  2. Close the Excel file
  3. Switch to the Word document & use Edit> Paste Special to insert it as something other than an Excel Object. Formatted Text & HTML render a Word Table, Unformatted Text renders a tabular list.

Or...

Use the Edit> Copy to Scrapbook command in Excel so you can then insert from the Office Scrapbook in Word.

doug robbins - mvp office apps & services (word) say:

The way in which Word lays out a document is dictated by the properties of the printer that is set as the active printer and as a result, page breaks may not occur in exactly the same place when different printers are involved.  Normally, that would probably only affect the page count by a few percent.  However, in cases where manual page breaks have been used, the impact could be much larger as such a page break may then be located just a couple of lines from the top of a page.

To prevent the problem, you should use the same printer\printer driver setup active on both computers - you can install the printer driver without having the printer physically attached to the computer, or select the Microsoft Print to PDF as the active printer on both computers. 

You should also use manual page breaks only when really required.
