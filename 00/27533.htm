al edlund say:

OpenXml ( the standards based solution based on MS' original work ) currently only supports spreadsheets, documents, and presentation. AFAIK data diagramming has not been submitted, so

an SDK in the near future is probably not going to happen.

 

Server based Visio is ( practically ) limited to sites implementing Visio Services because the client

version is single threaded and unsupported in that environment. If you compare the xml schemas between visio services and visio client you will notice they are not identical. IMHO for full fidelity of the drawings they should be developed on a client machine ( using the API ) and then mirrored to the host.

 

The pkgVisio implementation is missing ( what I consider ) a necessary option in the area of StyleSheets/Styles which I did not have time to implement. This is necessary for managing master shapes that have any amount of complexity when importing them from stencils. This is where you manage colors, text, and the other visual cues.
