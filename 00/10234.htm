joeu2004 say:
I am looking to know step by step ... what this VBA program is doing?

As you may know, the VBA GetIP procedure attempts to return all IP addresses associated with the local system, displayed in dot-address notation (a.b.c.d).  But in fact, it only returns one of the IP addresses.

The Winsock function gethostname returns the local system name.  Typically, this is the name you see in the Computer Name tab of the My Computer properties.

The Winsock function gethostbyname returns the list of IP addresses associated with the hostname.  For most PCs, there is only one IP address.  But if you execute the command `nslookup google.com` [1], you will see that it is not uncommon for servers to have multiple IP addresses.

However, the comment "get all of the IP address if machine is  multi-homed" is misleading.  For a correct implementation of that functionality, see the Do...Loop in http://support.microsoft.com/kb/160215.

The system procedure RtlMoveMemory does a memory-to-memory move of data.  This works around the strict type compatibility requirements of normal VBA.  In other words, it allows for copying data of one type to a variable of another type without interpretation of the contents.

I presume that the VBA procedures SocketsInitialize and SocketsCleanup are required to use the Winsock functions gethostname and gethostbyname.  I don't know why; I'm not familiar with Winsock.

-----
[1] Click on Start, then Run, and enter "cmd" in the Open field.  In the subsequent command window, enter the command `nslookup google.com` without quotes.
