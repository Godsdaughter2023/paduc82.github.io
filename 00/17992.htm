You're right.

In the process af looking for a solution I discovered a bug - indeed the value of GroupBySummary which is shown as yes for the group summary tasks cannot be read or used (in a filter, a formula or VBA) when it is Yes!

I do not know how litterate you are in VBA but the following is a workaround:

Cpy the following sub in a module:

 

Sub GBS()
SelectBeginning
Do While Not ActiveSelection = 0
On Error Resume Next
Set oldtask = ActiveSelection.Tasks(1)
OLL0 = oldtask.OutlineLevel
SelectRow 1, True
Set newtask = ActiveSelection.Tasks(1)
OLL1 = newtask.OutlineLevel
oldtask.Flag1 = (OLL0 < OLL1)
Loop
End Sub

 

with the grouped view active and all tasks shown. Make sure there is NO EMPTY LINE anywhere.

After the macro has run, the Flag1 field now is a copy of Group By Summary and can be used in a filter which solves your problem.

Note that the values in column C help us to determine what values are summed in the formulas in column D.

The first formula SUMIFS(...,30110) matches both text and numeric values.
