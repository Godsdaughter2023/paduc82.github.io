john w. vinson say:

First off, most of us really dislike the "Lookup Field" datatype. It sounds like you're wanting to use a Combo Box in a Table - this is very limited and often confusing. See The Access Web for a critique. Your Country field in mutationtable should NOT be a Lookup Field; instead it should be a Number, Long Integer field matching the ID of the Countries table. In a Form you can thus store the ID (keeping the computer happy) while displaying the country name (keeping the user happy).

A Combo Box will autocomplete if its autocomplete property is turned on, and the field (Country name in this case) is the first visible column. It works well if you have the combo box's Row Source query sorted by the name field (so it will find Algeria before Azerbaijan for example).

bob jones aka: cybertaz say:

Yes, although it fails to mention it, that article pertains only to Office 2016 for Windows. There is no comparable preference in Office 2016 for Mac.

However, when using the .docx file type Mac Word does not arbitrarily compress images in the first place... A .tif image @ 360 ppi, for example, is stored as such in the document. Some get the wrong impression because the program displays images at a lower resolution in order to avoid drastically bogging things down while editing the document. However, the full res image is used for printing.

In order to compress images in Mac Word the user needs to explicitly use the File> Reduce File Size command. This feature also contributes to the misconception because the Picture Quality setting in that dialog defaults to 220 ppi, but the compression isn't applied unless the OK button is executed - IOW, the value shown in that dialog is not the current res of the image (unless the image just happens to have been inserted at that res or has had compression applied).

BTW: Not to be pedantic :-) but pixels per inch (ppi) is the measurement system used for images. Dots per inch (dpi) is a printing device specification. Although the 2 often are used interchangeably there are significant differences.

The Mail app on the Mac and in the iPhone 5 use pop3 protocol to access the outlook.com server.

To receive emails in both places, messages have to be left on the server after downloading so the emails can be received on both devices.

If you login to the server and delete emails that have already been downloaded to both devices, the emails will still be on both devices.  There is no syncing with the server or the device.

The emails have to be deleted in three places.
