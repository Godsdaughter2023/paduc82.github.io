j377 say:

Open Regedit.

in HKEY_CLASSES_ROOT find the extension you want to unassociate.

(i.e., HKEY_CLASSES_ROOT\.bin)

Right-Click on the Key (.bin) and Export, to save a backup copy of the key in case you want to return it to its current state.

Right-Click again and Rename the Key or Delete the key.

Caution:  Making Registry changes can cause a system to become unstable or unusable.

j377 say:

I'm sure someone back in XP came up with that Registry solution.

In the event that didn't completely remove the association, I have found that the association is also found in the following Key...

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.bin

The listing is almost identical to the HKEY_CLASSES_ROOT. 

I just tested it, and had to delete both key entries to get a null association.

I spoke too soon.  I found 2 more keys that had to be deleted before it was fully disassociated.

HKEY_USERS\S-1-5-21-3721026060-4209833822-3968040267-1001\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.bin

HKEY_USERS\S-1-5-21-3721026060-4209833822-3968040267-1001\SOFTWARE\Microsoft\Windows\Roaming\OpenWith\FileExts\.bin

The user number may be different on your system.

That did remove all association with the extension.  No restart was required.

Might be quicker to do a search in Registry editor for \FileExts\.bin

If you have the cursor in Cell(2,3), after the 3rd character, then place the cursor in Cell(4,5) after the 3rd character, the event does not fire.
