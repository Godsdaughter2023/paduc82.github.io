Your motherboard is probably using UEFI Bios, and you can't install Win 8, 8.1, or 10 properly on a UEFI Bios machine with a DVD...

You need a USB Flash drive formatted FAT32...

Check out a tool called RUFUS... it will create a bootable flash from a ISO, but I'm not sure about a DVD... You may need a tool for that also... There are a lot of free ones...

You might also want to make sure your memory is properly seated in place...

There can be times when you have TOO MUCH memory for the installation to work...

The solution is to remove half, or maybe all but one stick...

Then once Windows is installed, put it back in...

vercingetorix2001 say:

The solution to this may be as simple as disabling "Fast Start".

My new (Jan 2016) Dell XPS 13 laptop suddenly stopped printing via regular apps such as Notepad and MS Word.  It would still print from browsers (which I'm guessing manage their own print jobs).  The symptom was that docs  would very briefly appear in the print queue, then disappear.  There was nothing evidently wrong with the spooler or RPC services, and little mention of similar problems on the web.  Windows diagnostics and fixits were of no use, nor were repeated device and driver removals & reinstalls etc.

I finally saw a comment that "Fast Start" might be the culprit.  The theory is that in order to support the fastest possible return to operation, "Fast Start" lets Windows store a snapshot of the OS state.  When you power back on, this is read into memory instead of going through a normal Windows boot.  There may be a problem with this whole process that occasionally corrupts the print spooler.

There are two factors that confuse the investigation.  First, simply turning off "Fast Start" may not fix the problem if it has already occurred.  You might have to restore your PC back to a condition where the printer works.  In my case that meant resetting to the "factory fresh" state, since System Restore defaults to "Off" under Windows 10 (thanks, Microsoft).  Also, the problem does not recur on every fast start - only sometimes.  So it's hard to tell when you've fixed it.

Once the printer is working eg from Notepad, disable "Fast Start".  The post I found has this in detail:

• Open Power Options in Control Panel
• Control Panel\All Control Panel Items\Power Options
• Click the link on the left side that says "Choose what the power button does"
• Under the section labeled "Shutdown settings"
• Uncheck the "Turn on fast startup (recommended)" checkbox
• You may need to click the link at the top that says "Change settings that are currently unavailable" if the options are grayed out.
• Save changes

Now if you're lucky the printer will continue working after shutdowns, reboots etc.  Mine has been good for two weeks now.


--> So for print spooler problems under Windows 10, try disabling "Fast Start", then do a full power-off power-on cycle.  If that doesn't work and you have no other options, reinstall Windows or reset your machine via its recovery partition - just to get the printer working - then immediately disable "Fast Start".


If this works for anyone, please report it here.  Also I'd like to know if "Fast Start" is enabled by default on new Dell Windows 10 machines.
