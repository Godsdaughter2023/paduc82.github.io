julie sheets say:
No need to use a task calendar on the installation tasks. 

Project uses different calendars when scheduling tasks.  Initially, without resources assigned, the tasks will be scheduled based upon the calendar set as the Project calendar in Project Information.  When you assign resources, the individual resource calendars schedule the resource's work.  If there is only one resource assigned, then the task schedule matches the resource calendar.  If there are multiple resources assigned with different calendars, Project will schedule the work for each resource on the task based upon each individual resource calendar.  For example, if the PM's base calendar is set for 8-12, 1-5 - his/her work will be scheduled only during that time.  If you have the PM assigned to a task where there are installation resources working, they will work on their schedule - working more hours per day and also working on Saturday and the PM will work only M-F 8-5. 

The only time you'd need the installation calendar as a task calendar would be if you needed to override a resource's calendar on a particular task.  For example, on the tasks where the PM and the installation resources are working, if you wanted the PM to work the longer schedule, you would set the installation calendar as the task calendar and check off the option "scheduling ignores resource calendar."  Then all resources assigned to the task would have to work based upon the task calendar.

 I hope this helps.

bernd p. say: A Onenote notebook is realized as a folder within the Windows file system.
After creating a notebook the name of this folder and the notebook's name in Onenote are (naturally) identical.

If you rename a notebook in Onenote (as suggested) only its (display) name in Onenote gets changed. The name of the Windows folder stays the same.
So if you close that notebook its changed name disappears. You'll later think, your notebook is lost.
My suggestion is:
Never rename a notebook in Onenote. To rename a notebook, close it, use Windows explorer to rename it, then re-open it.

Further comment concerning delete:
You have to close the notebook in Onenote before deleting it by using explorer.

Bernd

There's actually a good reason for why OneNote handles this the way it does.  Namely that if it is a Shared notebook, then changing the folder where it lives will break the reference any other user has to it, and will also break any existing hyperlinks pointing to the notebook location.  By keeping the location the same and just changing the display name, any existing links and references continue to work.

Option Explicit

Public Function CheckData() As Boolean
    Dim oCell As Range
    CheckData = True
    With Worksheets("Sheet1") 'Adapt this to match your sheet name!
        For Each oCell In Intersect(.UsedRange, .Range("K:K")).Cells
            If oCell.HasFormula Then
                If oCell.Value < 5 And oCell.Value > 0 Then
                    CheckData = False
                    Application.Goto oCell.EntireRow
                    MsgBox "Data entry for this row is incomplete, please complete this row!", vbExclamation + vbOKOnly
                    Exit Function
                End If
            End If
        Next
    End With
End Function

Option Explicit

Private Sub Workbook_BeforeClose(Cancel As Boolean)
    Cancel = Not CheckData
End Sub
