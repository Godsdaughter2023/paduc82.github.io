Linking a Picture File to a Cell - Microsoft Community

dvdmanuk say:

Linking a Picture File to a Cell

Hi,

I'm looking for some help with inserting a picture file to fill a single cell or merged cells.

         1.   I want to fill a cell with a picture file which is also set as a “Link to File”.

  Note:   If the source files changes then the changes will be reflected in the spread sheet.

         2.   I want it so that if I change the cell height and/or width that the picture will change size with cell.

  Note:   If the picture size is larger or smaller than the cell I wish the picture to fill the cell.

Thank you in advance for all the help you can give me.

shashirekha say:

When you insert a picture into an excel sheet as an object you won’t be able to link it to the same (i.e., If the source files changes then the changes won’t be reflected in the spread sheet).

However you can make the picture change size with the cell accordingly. To achieve that do the following

1. Click on the cell in which you want to insert a picture.

2. Choose the Insert menu and Picture From File to display the Insert Picture dialogue. Browse to the file that you need and double-click. The dialogue closes and the picture appears with its upper-left corner matching the upper-left corner of the cell.

3. Resize the picture as needed by clicking on it until the handles appear. Hold down the Shift key and click on one of the corner handles. While holding the mouse down, drag it to proportionately size the image.

4. Right-click on the picture and choose Format Picture from the pop-up menu. The Format Picture dialogue appears.

5. Click the Properties tab and then choose "Move but don't size with cells." Click the OK button to accept your settings. Your image now moves as though part of the cell.

How do you pull specific information from a table to a report in Access?

Ok.  So I have all the information in my table that I need.  It's a list of all my employees.  I need to pull a report that shows which ones are eligable for a specific award.  I have the report done, i just can't figure out how to get it to show only the eligible employees.  I have a colum of check boxes in the table that I just can just click their name and they're on the list.  Any insight would be much abliged.

 

Thank you!

~Amy

hansv mvp say:

Activate the Create tab of the ribbon.

Click Query Design in the Queries group.

Access will display the Show Table dialog.

Select the table that is currently used as record source for the report.

Click the Add button near the bottom of the dialog, then click Close.

You will see a list of fields in the upper pane of the query design window, and a grid in the lower pane.

Drag the asterisk * from the field list to the Field: row in the first column of the grid and drop it there.

Access will display TableName.* where TableName is the name of the table.

Drag the yes/no field from the field list to the Field: row in the second column of the grid and drop it there.

Clear the check box in the Show: row of the second column of the grid.

Click in the Criteria: row of the second column of the grid.

Enter True in the box.

Click the View button on the left hand side of the Design tab of the ribbon.

This activates datasheet view. You should now see the records for the eligible employees only.

Close the query, and save it with a suitable name (something resembling the name of the table, but not the same as the name of the table).

 

Now open the report in design view.

Activate the Data tab of the Property Sheet.

The Record Source property, listed at the top of the Property Sheet, probably contains the name of the table.

Click in this property.

Select the name of the query that you just saved from the dropdown list in the Record Source property.

Close and save the report.

Next time you open it, it will display eligible employees only.

Which program opens when you double-click a document file is determined by a setting in the registry. Something changed your registry to point to Office 2003 programs instead of Office 2016.

You should be able to change it back by running a repair of Office 2016.

Depending on what version of Windows you have, you may be able to choose Word 2016 and Excel 2016 as the "default programs" for their respective file types.

If that doesn't help, you can use the Windows registry editor (regedit.exe) to point to the 2016 programs. Note: You may have to run the editor "As Administrator" to be allowed to make changes.

Starting at the top of the tree diagram on the left. locate the key

HKEY_CLASSES_ROOT\Word.Document.12\shell\Open\command

and right-click it. Click Export, choose the Desktop as the folder, and give it a name such as docx.reg. That backs up the existing setting so you can restore it if needed (by double-clicking the .reg file).

On the right side of the registry editor, you'll probably see two entries like this, except that the Default entry contains the path to WINWORD.EXE in a folder containing Office 2003 instead of Office 2013:

If the second entry ("command") exists, delete it. Then double-click the first entry ("Default") and change the path to the folder for Office 2016. That is probably

"C:\Program Files (x86)\Microsoft Office\root\Office16\WINWORD.EXE"

but first look in that folder to make sure the WINWORD.EXE file is in there.

That will take care of .docx files (Word.Document.12). You should then repeat the same changes for .doc files (Word.Document.8), .docm files (Word.DocumentMacroEnabled.12), .dotx files (Word.Template.12), .dot files (Word.Template.8), and .dotm files (Word.TemplateMacroEnabled.12).
