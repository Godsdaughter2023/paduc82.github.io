monique tang msft say:

Hi Daniel,

Removing licenses will remove mailboxes and delete emails eventually.

You can use inactive mailboxes to preserve former employees’ emails after they leave your organization. A mailbox becomes inactive when a Litigation Hold or an In-Place Hold is placed on the mailbox before the corresponding Office 365 user account is deleted.

You can access the contents of the inactive mailbox by using In-Place eDiscovery in the Exchange admin center (EAC).

To use the inactive mailbox, the mailbox must be assigned an Exchange Online (Plan 2) license which is a part of an Office 365 Enterprise E3 and E5 subscriptions. If a mailbox is assigned an Exchange Online (Plan 1) license (which is part of an Office 365 365 Enterprise E1 subscription), you would have to assign it a separate Exchange Online Archiving license so that a hold can be placed on the mailbox before it's deleted.

For the detailed information, you can refer to the following article: https://technet.microsoft.com/en-us/library/dn144876(v=exchg.150).aspx

As Rohn007 mentioned, to keep the emails, you can also export the emails to a .pst file before deallocating the license and then import Outlook items from the .pst file as a workaround.

daniel g. grau say:
But you can resort to a macro that will add a bookmark named after the selected text:

Sub AddBookmarkWithSelectedText()

 

If Selection.Type = wdSelectionIP Then

    Message$ = "Enter a name for the bookmark (initial underscore makes it hidden, cannot begin with a number)"

    myBookmarkName = InputBox(Message$, "Define bookmark at cursor position", "")

    If myBookmarkName = "" Then Exit Sub 'Pressed Cancel

    If myBookmarkName = "_" Then Beep: Exit Sub 'Invalid name

End If

myBookmarkName = Selection.Text

'Name cannot start with a digit. Just quit.

If Asc(Left(myBookmarkName, 1)) >= 48 And Asc(Left(myBookmarkName, 1)) <= 57 Then

    Beep

    Exit Sub

Else

    myBookmarkName = Trim(Selection.Text)

End If

 

'There is a maximum length as well

myBookmarkName = Left(myBookmarkName, 40)

'Forbidden characters in bookmark names (there may be others)

'1st one is Space, last one is Option Space

'Cannot start with a number either

Forbidden = " ()¡!¿?„«»“”‘’[]{}+%$#@^&'*•;:,.<>/\|—- " & Chr$(13) & Chr$(9)

 

For Pointer = 1 To Len(Forbidden)

    StartValue = 1

    MustLoop = True

    LookingForThis = Mid(Forbidden, Pointer, 1)

    While MustLoop

        CharacterPosition = InStr(StartValue, myBookmarkName, LookingForThis)

        If CharacterPosition <> 0 Then

            LeftPortion = Left(myBookmarkName, CharacterPosition - 1)

            RightPortion = Right(myBookmarkName, Len(myBookmarkName) - CharacterPosition)

            If CharacterPosition = 1 Then

                myBookmarkName = RightPortion 'Initial underscore makes it hidden

            Else

                myBookmarkName = LeftPortion & "_" & RightPortion

            End If

            StartValue = CharacterPosition + 1

        Else

            MustLoop = False

        End If

    Wend

Next Pointer

ActiveDocument.Bookmarks.Add Range:=Selection.Range, Name:=myBookmarkName

End Sub



Paste the above code in the Visual Basic Editor (Tools > Macro > Visual Basic Editor). Assign it a shortcut (or place it in a toolbar) for convenience. This is what it does:

  • If some text is selected when you call the macro, it will create a bookmark whose name is based on the selected text, minus any invalid characters, which are replaced by underscores. The bookmark will span the selection.
  • If there is no selected text, it will prompt you for a bookmark name and place the bookmark at the cursor's position.
  • If a name starts with an underscore, the bookmark will be hidden.
  • If the proposed name starts with a digit, it will beep and quit.

I think you're asking how to create cross-references. On the References tab of the Ribbon, in the Captions group, click Cross-reference. stefan blom say:

In Word, each cross-reference depends on a hidden bookmark which is added (by Word) around the cross-referenced paragraph.

The problem is that it is very easy to "break" such a bookmark, for example, by adding content (or pressing Enter) at the beginning of the bookmarked paragraph.

You can recreate any failing cross-reference manually. To avoid the problem in the future, always make sure to add new content at the end of paragraphs (not at the beginning).

snowfaerie say: Where does the bookmark get created if the object being referenced is a picture that has thet text set to wrap around it? Still at the beginning of (some? which one?) paragraph?

stefan blom say:

I guess you have created the cross-reference to a figure caption? In that case, the hidden bookmark encloses the paragraph that contains the caption.

Note that the easiest way to learn how bookmarked text behaves is to experiment with a manually created bookmark. You will also have to display bookmarks by selecting that option at File | Options | Advanced. See what happens as you press Enter and/or insert text at the beginning of a bookmarked paragraph (and note that the problem occurs if the whole paragraph or, more specifically, the beginning of the paragraph is included in the bookmark).

Hidden bookmarks are difficult to work with, because those are never shown—not even if you display bookmarks in Word. Recreating a damaged cross-reference will be easier.

snowfaerie say:

Okay, I'll experiment with that, then.

How do I recreate the bookmarks, though? I tried removing and putting the picture back, and adding a new caption for it, but that just removes it from the list of cross references entirely. How come the new pictures do not appear on the list?

stefan blom say:

Newly added figure captions should definitely show up in the list of figures in the Cross-reference dialog box.

Have you updated the figure captions, so that they display propery (they should appear in the logical order in the Cross-reference dialog)?

Also, make sure that you are not tracking changes; if you are, cross-referencing may be extra tricky.

If everything else fails, you can always manually enclose the item in a bookmark and create a cross-reference to bookmark text.
