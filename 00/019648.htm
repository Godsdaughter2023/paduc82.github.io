I've read that some people suggest populating a "calendar" table with all the possible dates in the desired period.  However, I'm hoping that there is a way to do this in sql without having to create and populate a table or recordset.

Well, you either do that or create a table of serial integer numbers from zero upwards, and compute each date in the parameter range by adding its start date to each number.  Either way you return the Cartesian product of the two tables, restricted on the calendar date being within the range defined by the admission and discharge dates and grouped by the date.  But why do you object to an auxiliary calendar table?  It's trivial to set up and cuts out the need for the additional computation.

Open Start, type: CMD
Right click CMD
Click Run as administrator

Type the following command then hit Enter to disable Windows Defender permanently:

REG ADD "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableRoutinelyTakingAction /t REG_DWORD /d 1 /f
