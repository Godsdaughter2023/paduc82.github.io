Networking Issues Persist In Build 10240 - Microsoft Community

terrymuench say:

Networking Issues Persist In Build 10240

I am running W10 Build 10240 as a guest in an Oracle VirtualBox VM using a Windows 8 Host. I have to define two network adapters (NAT, Host-Only) to have both Internet and Host communications.

After upgrading from 10074 to 10130, I had lots of network problems, primarily NetBIOS errors and "... another computer on the network has the same name." For more details, see my post titled "Major Networking Issues After Upgrade from 10074 to 10130".

I am posting again because, while in general Build 10240 seems to be very stable, I am still having the same network problem and must disable/enable the network adapter to get rid of NetBIOS conflicts. I hope this can help others having this problem.

Here is a script I developed to do this from the command line

----------------------------------------------------------------------------------------------------
@Rem
@Rem Disable/Enable the network adapter 'Ethernet 2'
@Rem for some reason NetBIOS no work until this is done
@Rem

@Echo off

@Rem Disable the adapter
Echo.
Echo disabling: 'Ethernet 2'
wmic path win32_networkadapter where NetConnectionID="Ethernet 2" call disable > nul
Pause

@Rem Enable the adapter
Echo.
Echo enabling: 'Ethernet 2'
wmic path win32_networkadapter where NetConnectionID="Ethernet 2" call enable > nul
Pause

@Rem Refresh NetBIOS
Echo.
Echo Refreshing NetBIOS
nbtstat -R

Echo.
----------------------------------------------------------------------------------------------------
Replace 'Ethernet 2' with the name of your Host-Only adapter
It is a good idea to pause for ~5s after each operation

After 'nbtstat -n' should show no conflicts
