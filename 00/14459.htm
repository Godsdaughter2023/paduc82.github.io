Where exactly do you want to switch your Outlook.com? In Outlook desktop program (part of the Office suite)? Which version?

On a generic level, only one Outlook.com account exists regardless of how it was configured to be accessed by any <email client> whether that's Outlook desktop or some other.

In terms of "switching" from one protocol to another (POP to Imap etc) - you don't "switch". Rather you need to configure the email account again using the desired protocol. You would not lose anything that exists on the server but a more important question if this involves the Outlook desktop <email client> is

"Why would you configure the Outlook.com account using IMAP instead of as an Exchange account - the default when you configure the account by entering the email address/password?

Re: #1 - Just for the record, that's the opposite of what was asked in the original post. You also didn't include the version of Office in use

Re: #2 - Configuring the account

Since I'm not sure of how exactly your account is currently configured or the version of Office (Outlook) in use, use the following approach for the sake of simplicity

a) With Outlook closed, open the Control Panel <Mail> app
b) create a <new> Outlook profile and configure the account by entering the email address/password. All that's required to configure the account
c) make sure that you set the newly created profile as the default for the next time it's opened

Some caveats

The significance of knowing how your Outlook.com is currently configured is to guess as to where your contact/calendar info is currently stored.

If the contact/calendar data is currently stored in folders with the additional words "this Computer Only" - the account is configured as IMAP and you need to export those folders to a PST file, taking note of where that PST file has been created.

If the account is configured as a POP3 account, need to take note of where the PST file for the account is located (go to File --> Account Settings --> Data Files tab --> assuming Outlook '2010 or later is being used).

You'll need to add the above PST file - whether exported or from a POP3 account, to the profile for the Outlook.com account to copy the contact/calendar data to the corresponding folders in the Outlook.com account.

pamcaswell say:

I haven't had this problem in a long time (Word 2002 &3), but when I did, it was because there were so many balloons on the page that Word couldn't fit them.  If you only have comments and no other markup, you might try increasing the balloon width.  If there are formatting balloons that you don't need to  keep,  you can run the following macro to delete them:

Sub AcceptFormatChanges()
'
' Macro recorded 5/3/2006 by PPC
'
    WordBasic.ShowComments
    WordBasic.ShowInkAnnotations
    WordBasic.ShowInsertionsAndDeletions
    WordBasic.AcceptAllChangesShown
    WordBasic.ShowComments
    WordBasic.ShowInkAnnotations
    WordBasic.ShowInsertionsAndDeletions
End Sub

 

If there is other markup, consider showing it in line instead of in balloons.

 

HTH,

Pam
