steve rindsberg say:
Corrections:  

Mark As Final in NO WAY protects the presentation.  It simply informs anyone who opens the presentation that it's been marked as final.  It remains editable and people can still copy content from it.

Adding a digital signature doesn't protect anything either; it simply gives the recipient a way of satisfying themselves that the file did indeed come from you and that it's not been changed along the way.

Using a password is one possibility;  Use a MODIFY password to allow anyone to open the file but only those with the password to edit it.  

Another useful trick:  You can save from PPT 2010 as a Picture Presentation.  This is still a PPTX file but it converts each slide to an image.  While entire slides can still be copied, they can't be edited.

Very good, Glenys; if there was an MsMpEng.exe entry in HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\Image File Execution Options, then this is something that you don’t want to revert – because it’s a malware entry. But that also means that you should do some serious scanning with third-party online scanners and malware-removal tools before proceeding:

Kaspersky Virus Removal Tool:

http://www.kaspersky.com/antivirus-removal-tool?form=1

Emsisoft Emergency Kit:

http://www.emsisoft.com/en/software/eek/

Malwarebytes Anti-Malware (free version only):

https://www.malwarebytes.org/antimalware/

Eset Online Scanner:

http://www.eset.com/us/online-scanner/

I installed Avast Free in order to test my hypothesis that the DisableAntiSpyware and DisableAntiVirus DWORDs in HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows Defender are used to turn Windows Defender off when a third-party AV app is installed, and this was the result:

I was also able open both Windows Defender and Avast immediately after the installation until such time as the DWORDs were written to the Microsoft\Windows Defender key. So that’s something that I wasn’t aware of, and something that might be important in other cases where Windows Defender refuses to run. The Farber Service Scanner seems to confirm this:

Windows Defender:

 ==============

 WinDefend Service is not running. Checking service configuration:

 The start type of WinDefend service is set to Demand. The default start type is Auto.

 The ImagePath of WinDefend service is OK.

Windows Defender Disabled Policy:

 ==========================

 [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows Defender]

 "DisableAntiSpyware"=DWORD:1

So once you’re confident that all traces of malware and third-party antimalware have been completely removed, then you can go ahead and attempt to delete the DisableAntiSpyware and DisableAntiVirus DWORDs. I think you’ll probably have to take ownership of the key in order to do that.

If the DWORDs can’t be deleted, or if they get reset, then you might need to repair the WMI Repository. You can follow the steps in this Avast article:

https://www.avast.com/en-gb/faq.php?article=AVKB81

Hi Ivan,

To change default domain in Office 365 tenant, please go to Office 365 admin center > Settings > Domain . select a custom domain > Set as default.

3. Select all users in this filter > Edit domain > Select a custom domain > Save.
