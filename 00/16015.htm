rob koch say:

Phishing protection in Windows 8.x is embedded in both the IE10/11 browsers and Windows 8 itself.

http://windows.microsoft.com/en-US/internet-explorer/use-smartscreen-filter#ie=ie-11

SmartScreen Filter provides both Phishing and general URL protection and since it's extended in Windows 8 to the operating system itself, this protects all applications accessing the Internet, so Windows Defender itself doesn't require separate web page scanning.

E-mail scanning has been pointless for over 10 years, since Microsoft provided its own antivirus API functions in every operating system since Windows XP, which passes all attachments to the operating system to be scanned automatically.  Previously each antivirus application had its own email proxy shim that had done this, so most had dropped these several years ago and I know of none remaining at this time from any modern antivirus vendor.

Microsoft switched to the Windows Filtering Platform (WFP) in Windows Vista to provide security applications access to the protocol filters, so having these embedded in the antimalware application itself is also an outmoded concept since 2007.

http://msdn.microsoft.com/en-us/Library/Windows/Hardware/dn653358(v=vs.85).aspx

"Windows Filtering Platform (WFP) enables TCP/IP packet filtering, inspection, and modification, connection monitoring or authorization, IPsec rules and processing, and RPC filtering."

All modern security applications have been using these filter driver based antivirus and related monitoring systems since Windows Vista, so your basic premise is based on very outdated methods from another era.

Windows Hardware Certification requirements for Windows 8filter drivers

http://download.microsoft.com/download/A/D/F/ADF5BEDE-C0FB-4CC0-A3E1-B38093F50BA1/windows8-hardware-cert-requirements-filter-driver.pdf

"This document contains the Windows Hardware Certification requirements for Windows 8filter drivers. These requirements are Microsoft’s guidelines for designing Windows Filtering Platform Drivers (WFP), File Systems Filter Drivers, Antivirus Filter Drivers, and Early Launch Anti-Malware (ELAM Filter Drivers). Successfully following this guidance allows a partner to receive certification and signing for their filter drivers."

No, you are thinking backwards, the Windows Filtering Platform is part of Windows 10 itself, not Windows Defender.

"Windows Filtering Platform (WFP) is a set of API and system services that provide a platform for creating network filtering applications. The WFP API allows developers to write code that interacts with the packet processing that takes place at several layers in the networking stack of the operating system. https://msdn.microsoft.com/en-us/library/windows/desktop/aa366510%28v=vs.85%29.aspx?f=255&MSPPError=-2147217396

Since Windows Defender is an antivirus or more broadly an anti-malware application, it would include Antivirus Filter Drivers, and Early Launch Anti-Malware (ELAM Filter Drivers) which access the WFP API set and system services to perform its monitoring functions.
