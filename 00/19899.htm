    v_agarwal say:

    You may also go for this alternative,

    Assuming you write any code in the format "Xn.n" where "X" is the any alphabet (Caps or Small) and "n" is a digital number between 1 and 9, then if you write this formula just below that cell and drag it down, then it will give further sequence.

    =IF(RIGHT(A1,1)="9",CHAR(CODE(LEFT(A1,1))+1)&"1.1",LEFT(A1)&RIGHT(A1,3)+1.1)

    Say you have written A1.1 in cell A1 and if you put above formula in A2 and drag then it will give results like A2.2, A3.3 onwards. In the same way if you have written say D4.4 in cell A1 then this formula will start giving next sequence with D5.5 onwards

    joshua83702 say:


    This was a vexing problem, ultimately due to corrupted Office Upload Center Cache files database, causing both the "credentials have expired" and "upload blocked" errors with Office 365 and Office 2016.

    1. The upload cache database is at AppData\Local\Microsoft\Office\16.0\OfficeFileCache1
    2. Therein is CentralTable.accdb database
    3. Clean boot to delete the OfficeFileCache1 folder and with it the corrupted database: use MSCONFIG to hide all Microsoft services then disable all other services and reboot. (Please note I did not find it necessary to disable all startup programs: Services were enough.)
    4. After deleting the OfficeFileCache1 folder, you will see it automatically rebuilt upon launching either the Office Upload Center or starting Word (probably any Office program). This solved the “expired credentials” and “upload blocked” errors.

    This MS article might have worked also, as it stipulates a clean boot under step 1 and the file locking was definitely preventing the cache from being cleared via the Microsoft Office Upload Center Settings Delete cached files option without the clean boot. I further note this is poor coding, as there is no way to know success or fail when clearing the cache because the GUI returns no results.

    See: https://support.office.com/en-us/article/delete-your-office-document-cache-b1d3765e-d71b-4bb8-99ca-acd22c42995d
