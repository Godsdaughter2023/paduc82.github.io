Nokia Lumia 520 MMS problem with Fido (Solution) - Microsoft Community

romaindecarie say:

Nokia Lumia 520 MMS problem with Fido (Solution)

I recently bought a Nokia Lumia 520 to use with my Fido (Rogers) sim card. After some time I discovered the horrible truth : I couldn't open media content from my text messages. I called the Nokia Support team who helped me a little but couldn't fix the problem. Fear not fellow Fido users, I have found the solution.

You will have to manually setup your MMS apn on your device.

1. Go to http://www.fido.ca/web/content/phonewarranty/configure_unlocked_device_guide&lang=en

2. Scroll to the end of the page and select "How to configure other smartphones"

What you will see is the info you need to enter in your phone. Keep the window open or write the info down.

3. Take your Nokia Lumia 520. Go to Settings.

On some phones you will have to go to "Access Point" then "Add a New Profile".

If you don't see this first option, go to "Cellular" instead, then at the bottom of the page click on the button "Add MMS APN".

In either case you will have the same fields. So select the MMS APN.

4. Now you will have to fill up the fields with the info from the Fido website. It is pretty straightforward :
APN
Don't put anything in the Username/Password fields
MMSC
MMSC port

5. If you don't have the "proxy" field, just enter the proxy address in the "WAP gateway" field. Be sure to add "http://" before the proxy since it is a URL.

6. Enter the port number (same port than the MMSC port) in the "WAP gateway port" field.

7. If you have the option at the bottom of the page select IPV4 because the proxy is an IPV4. On my phone it was set to IPV6 and it wasn't working until I changed it.

Forget the maximum size option, your Service Provider is already taking care of that.

There, it should be working properly now !

Enjoy.

Hello again John,

Try the following. It is never a good idea to rearrange the source data in case there is an error and therefore the code adds a new worksheet for the output. it leaves the new worksheet with the default name so you can manually rename it to whatever suits you.

Note that you might need to edit the source data worksheet name if it is not "Sheet1".

Fee free to get back to me it if does not perform as desired.

Sub RearrangeData()
    Dim wsSource As Worksheet
    Dim wsDestin As Worksheet
    Dim rngDestin As Range
    Dim lngLastRow As Long
    Dim r As Long
    Dim i As Long
    Dim cel As Range
    Dim arrSplit() As String
    Dim strCityStateZip As String
    Dim strStateZip As String
   
    Set wsSource = Worksheets("Sheet1")     'Edit "Sheet1" to your source data worksheet name (if required)
    
    Sheets.Add After:=wsSource
    Set wsDestin = ActiveSheet
    With wsDestin
        .Cells(1, "A") = "Name"         'Edit these column headers for the output as desired
        .Cells(1, "B") = "Address"
        .Cells(1, "C") = "City"
        .Cells(1, "D") = "State"
        .Cells(1, "E") = "Zip"
        .Cells(1, "F") = "Phone"
       
        .Range(.Cells(1, "A"), .Cells(1, "F")).Font.Bold = True
        .Columns("E:E").NumberFormat = "@"  'Set to Text so Zip codes retain leading zero
    End With
   
    With wsSource
        lngLastRow = .Cells(.Rows.Count, "A").End(xlUp).Row
       
        For r = 1 To lngLastRow Step 5  'Loop in rows of 5
            With wsDestin
                'Assign next row in Column A of Destination to a range variable
                Set rngDestin = .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0)
            End With
   
            For i = 0 To 3  'Loop through 4 rows of Name/Address/City/Phone
                Set cel = .Cells(r, "A").Offset(i, 0)   'Assign cell to a range variable
                Select Case i
                    Case 0, 1   'Name and Address rows copy as is
                        rngDestin.Offset(0, i) = Trim(cel.Value)
                       
                    Case 2      'City, State, Zip
                        strCityStateZip = Trim(cel.Value)   'Assign City/State/Zip to string variable
                       
                        rngDestin.Offset(0, 2) = Trim(Left(strCityStateZip, _
                            InStr(1, strCityStateZip, ",") - 1))     'City is left of comma (Permits 2 word city names)
                           
                        strStateZip = Trim(Mid(strCityStateZip, _
                            InStrRev(strCityStateZip, ",") + 1))    'Assign State/Zip only to string variable
                           
                        arrSplit = Split(strStateZip, " ")     'Split State and Zip at the space
                       
                        rngDestin.Offset(0, 3) = arrSplit(0)     'Copy State
                        rngDestin.Offset(0, 4) = CStr(arrSplit(1))     'Copy Zip
                       
                       
                    Case 3
                        rngDestin.Offset(0, 5) = Trim(cel.Value)      'Copy Phone
                    Case 4
                End Select
            Next i
        Next r
   
    End With
    wsDestin.Columns("A:F").AutoFit
End Sub

a. user say:

Most common tips & tricks to try if you can.

In your case, I'd try items 4 to 9 first, then 10 to 12 and finally 1 and 2

  1. Run Windows Update troubleshooter or wu10.diagcab
  2. Run Microsoft's Software Repair Tool
  3. Use PowerShell (Admin) to run sfc /scannow and DISM /Online /Cleanup-Image /RestoreHealth
  4. Uninstall any 3rd party antivirus program
  5. Disconnect from the internet once the download is complete
  6. Set Feedback & Diagnostics to FULL
  7. Verify MS account
  8. Unplug any USB devices
  9. Switch Insider Ring to FAST
  10. Reset Windows Update (use the auto-script)
  11. Delete the "Download" folder
  12. Rename the "SoftwareDistribution" folder by running RenameWU.cmd with admin rights

If all failed, re-installing on top of itself or a clean install should be considered (ISO image required).

Read this thread for instructions on how to manually download any Windows ISO image.
