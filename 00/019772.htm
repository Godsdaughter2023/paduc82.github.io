I certainly hope so,
PUIS is not intended to be used on the system / OS drive,
its intended use is for multi disk systems where simultaneous spin up would cause the system to fail to boot due to the high power requirements,
also PUIS is only intended to be used when all components support the full feature set, including the BIOS/EFI/UEFI, HDD controller, and the HDDs installed.

the only controllers I know of that support PUIS or Staged / Staggered Spin Up are enterprise class SCSI/SATA/IDE HBA controllers. 

Enabling PUIS on the system drive is not going to save anyone any amount of power because the system drive must spin before the OS can load or resume from hibernate / standby etc.

Private Sub Worksheet_Change(ByVal Target As Range)
Const myCol As String = "C" '<<< data in column C
Const N As Long = 6 '<<< first row
Dim r As Long
If Not Intersect(Target, Range(myCol & ":" & myCol)) Is Nothing Then
r = Cells(Rows.Count, myCol).End(xlUp).Row
If r < N Then r = N
With ActiveSheet.Sort
.SortFields.Clear
.SortFields.Add Key:=Range(myCol & N), SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal
.SetRange Range(myCol & N & ":" & myCol & r)
.Header = xlNo
.MatchCase = False
.Orientation = xlTopToBottom
.SortMethod = xlPinYin
.Apply
End With
For x = r To N + 1 Step -1
If Cells(x, myCol).Value = Cells(x - 1, myCol).Value Then Cells(x, myCol).Delete shift:=xlUp
Next
End If
End Sub

Are you testing Windows Insider builds?  If so, you must know that there are zero guarantees about whether your system will be usable after a build - we're here to act as the next level of testing.  If you are not testing builds, then this forum is likely not the best one to get your answers.  We're all talking about issues and workarounds for test builds.

If you are a Windows Insider testing builds, then you should know that you need to actually post more than your mouse version.  Best process when you're testing is to: 

1) Give details on your hardware (the mouse info is good, but what's your computer system?  Desktop? Laptop? Make/Model?  Other Bluetooth devices installed and working correctly to the best of your knowledge?)

2) What build(s) are you experiencing problems with?  The most recent build for insiders on the fast ring is 14986.   You already mentioned it was an update, not a clean install.

Hi Lloyd Bedik,

 

For your situation, I suggest you first check the following:

 

Make sure the Outlook Address Book service is added to the profile

 

  1. In Outlook > File > Account Settings > Account Settings > Address books
  2. If the Outlook Address Book service has not been added yet, press the New….
  3. Select “Additional Address Books” and click Next.
  4. Choose “Outlook Address Book” and click Next.
  5. You’ll be prompted that you must close and restart Outlook for changes to take effect.
  6. Press OK and close and restart Outlook.

 

Make sure that the Contacts folder is marked as an Outlook Address Book

 

Right click the Contacts folder and choose Properties-> tab Outlook Address Book-> select option Show this folder as an e-mail Address Book.

 

If the above setting without any problems, to verify whether it is caused by the corrupted data file, you can try using the ScanPST tool to repair your Outlook data file.

 

In addition, if you’re an Office 365 for business user, you can also try using the Microsoft Support and Recovery Assistant for Office 365 which can help you automatically diagnose and fix a range of Outlook problems.

  • Can you successfully send the emails to the missing contacts in address book?
  • What’s the version of your Outlook?
  • How long has this issue occurred.

Huni
