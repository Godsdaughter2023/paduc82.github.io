Auto fill cells - Microsoft Community

marygerhardt say:

Auto fill cells

how do I make a repeated word in an excel column auto fill without having to use the mouse to select what I want?  Excel for Mac 2011, Version 14.3.6 

bob jones aka: cybertaz say:
Well, in order to fill a range of cells you must first select that range... Unless you want to copy the paste from cell to cell to cell. The easiest way for most users is to use the mouse... More specifically, the Fill Handle @ the lower right corner of the starting cell. There are several variations on the technique but most involve using the mouse to select the range.

If you're actually looking for a means of selecting a range of cells without using the mouse you also have at least 2 options;
  • Click in the Name Box [@ the left end of the Formula Bar], type the range as you would in a formula... E.g., B7:B50, then press return, or
  • Use Edit> Go To (Control+G), type the range as above, then either click OK or press return

Both methods, of course, require that you know both the starting & the ending cell reference.


Once the range is selected -- assuming the topmost selected cell contains the word to be used -- you can use Edit> Fill - Down (Control+D) to fill the remaining selected cells.

In case you aren't aware, you don't have to drag in order to select a range of cells. Just select the starting cell then hold Shift while you click the ending cell. This is particularly handy if it's a large range because you can use the scroll bar/wheel to reveal the ending cell rather than having to keep the mouse button held down.

askbal say: The error happened because you have defined or accessed a large number of access points with the device. I tend to access a large number of different AP's with my E65's WiFi. Details of each WiFi or edge or hsdpa access point you connect to is stored in that list. Hence, I got this error once.

I got out of that mess by doing the following:-

1. Go to web browser settings
2. In "access point" select "user defined"
3. Then, the device will provide you with a list of all the access points you have defined and ask you to select the default access point.
4. At this point, you can start deleting some of the access points. (Options -> Delete after selecting any ap in the list)
5. After you have deleted some of the access points, you won't have the memory full error you encountered.

This is certainly an inconvenience. One way to avoid this issue is to manually pare down the access point list once in a while. Delete some of the access points that you use very infrequently or never again.

jay freedman say:

About two dozen lines before the end of the macro is the statement

            .Cells(1).Range.Text = _

This is what you need to modify to get the line number into the table. Unfortunately, VBA makes it harder than it needs to be, because the Information function has a limited number of things it can return. The wdActiveEndPageNumber gives you the page number of the end of the text to which the comment was applied. It would be nice if there was a corresponding wdActiveEndLineNumber, but there isn't -- the only thing that will return any line number is wdFirstCharacterLineNumber, which gives you the line number at the start of the commented text. Also, there is nothing that will return the page number at the start of the text.

If the commented text doesn't cross a page boundary, then using wdFirstCharacterLineNumber and wdActiveEndLineNumber together isn't a problem. If the commented text starts on one page and ends on another, though, they don't match up.

The solution is to use a Range variable as a "helper". The comment's Scope property is a Range, so you can assign it to a Range variable, which you can then manipulate separately. After you get the line number of the start of the Range, you can collapse the Range to its beginning (that is, its start and end are in the same place) and then get the page number of its end.

Near the top of the macro, add the lines 

    Dim pg As String, ln As String
    Dim rg As Range

Then replace the statement  .Cells(1).Range.Text shown above with the following statements:

    With oDoc.Comments(n)
        Set rg = .Scope
        ln = rg.Information(wdFirstCharacterLineNumber)
        rg.Collapse wdCollapseStart
        pg = rg.Information(wdActiveEndAdjustedPageNumber)
    End With
    .Cells(1).Range.Text = "Pg " & pg & " ln " & ln
