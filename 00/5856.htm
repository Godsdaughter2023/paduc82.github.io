  Data = Cells(StartRow, "B").Resize(lastRow - StartRow + 1)
  Data = WorksheetFunction.Transpose(Data)
  BList = Join(Data, "/")

Then add a breakpoint at the first line (select it and press F9) and run your macro. If the code stops mark the word Data and press SHIFT-F9 and click Add. Do the same with BList.

Now you can see the variable Data inside the watch window, press F8 to execute the line. You'll see that the variable inside the watch window is changed, click at the + to explorer it. Press F8 again and again and explore the variables.

This is what we call debugging.

john - project say:

Eric,

Well I think the message says a little more than that but I agree many of the error messages are pretty "unhelpful".

You mentioned that you tried several leveling options but my guess is you stayed with a "day by day" basis and that's the crucial part. Indeed with a day by day leveling basis Project does not know what to do given the assignment level is still at 100% when the resource Max Units is reduced to 40%. The message does give you some suggestions and one of those is to adjust the assignment level.

If you want Project to take some "automatic" leveling action, you can set the leveling basis to week by week or even month by month depending on the task. For example in the screen shot below I set up a simple plan with 4 one-week tasks and four separate resources all initially with 100% max units.

Now I reduce the Max Units for joe to .4 and joe immediately shows up as overallocated. Then with the leveling basis set for week by week the project is leveled. The result is shown below. With a 40% max unit level, joe can only work 16 hours per week, so Project shifts task a to start on Thursday so joe can work 100% for two days on that task. Remember joe's assignment level on task a has not been changed. The task is split such that the following week joe's work is again shifted to the last two days.

Given certain parameters, that is what leveling can do. Remember Project is a tool, it has zero intelligence so it doesn't "know" what to do when several different approaches are possible (i.e. increase work, increase duration, etc.).

Hope this helps.

John

julie sheets say:
Pardon me for bumping in.  Project will not change a resource's assignment units through leveling.  As John notes, the issue is the assignment units are still at 100% and the resource's max units are 40%.  If you want to replace all of the 100% assignment units with 40% assignment units and extend the duration of the tasks, use the Assign resources dialog.  Select the resource, click replace and replace the resource with his/her self and enter the 40%.  As long as the tasks are not effort driven, the duration will increase, leaving the work the same.
