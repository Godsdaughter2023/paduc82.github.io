Please follow the below steps to reset the Windows Updates Components:

1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

5. Type Exit in the Command Prompt to close it and then restart the computer.

6. After restarting the computer, now retry to install Windows Updates.

Hello UMaitin,

For your concerns:

1. Will it go away if I remove SIP from the proxy address?

I tested on my side, after I remove the SIP address from a user, the user still can be searched on Skype for Business. The SIP address is used by Communications Server or Lync Server to route incoming calls and send voice mail to the user. When a SIP address is removed, callers can no longer leave voice mail to the user’s mailbox. Thus, for this concern, the answer is when you remove the SIP address, it won’t “go away” on SFB. According to my knowledge, the short answer is yes. However, when a license is removed, the user who has no SFB license cannot login into SFB either.

farblue say: Just in case anyone was curious, I've managed to solve a lot of my problems :) The fault for the battery life lays with the Imap server - uw-imap.

I found under closer inspection that the Imap server was sending Idle messages every 2 mins and this was triggering the email app to check for headers every 2 mins. Nailing the battery in the process. Having found out why the server was doing this (an addition to the code in 2005 to help get Idle connections past NAT and gateway servers), I disabled the 'feature' and recompiled and everything is now much happier.

I now don't use notification and do use auto-fetching. I have setup to fetch headers every 30 mins which both creates an Idle link for instance email notification and also allows the email to reconnect if the signal is lost (within 30 mins, assuming a signal returns). This could be 15 mins if you wanted but obviously 30 mins reduces battery life.

The only real annoyance left in this otherwise fantastic setup is that if you press the red button it kills the email client and you don't get new mail notifications :(

So if you are listening, Nokia, can I PLEASE have a 'hide' option in email rather than exit and Please Please can I have the option to prevent the red button killing the email?

I already found a workaround.

It is a Windows 7 Bug Features on network.

1. Disable Autotuning
2. Disable Taskoffload
3. Disable Remote Differential Compression

CMD
netsh interface tcp set global autotuning=disabled
nets int ip set global taskoffload=disabled

Windows Features
Uncheck "Remote Differential Compression"
