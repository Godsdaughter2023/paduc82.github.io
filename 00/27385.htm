john korchok say:

Normal.dotm can become corrupt. But it's easy to make Word create a fresh copy:

  1. Close Word.
  2. In the Finder, hold down the Option/Alt key while clicking on the Go menu and choose Library. Your user Library folder opens.
  3. Open ~/Library/Group Containers/UBF8346G9.Office/User Content/Templates.
  4. Drag Normal.dotm to the desktop.
  5. Start Word, then close it. A new Normal.dotm file is created.

If you had a lot of custom AutoText, styles or macros in your old Normal template, you can copy that data back into the new file using Tools>Templates and Add-ins>Organizer.

    jay freedman say:

    Considering that Microsoft has been naming its temp files with ~$ for 20+ years that I know of, I'd say the probability of changing that convention is somewhere between zero and none.

    Some of the temp files probably could be put into a separate hidden folder (such as the Temp folder under AppData\LocalLow in the user's profile), but others are required to be in the same folder as the document's file. One of these is the owner file with the ~$ prefix, as described at http://support.microsoft.com/kb/211632. Since the Office programs now enable simultaneous editing of cloud-based documents, maybe the owner file will eventually be deemed unnecessary and won't be created -- but there's no sign of that yet.

    I know you aren't going to like this, but the best you can do is slow down your trigger finger by a second or two.
