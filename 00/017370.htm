    If Range("A1").Value > 0

        Range("A1").Value = 0

    End If

The End If is accepted but the statement If Range("A1").Value > 0 returns a syntax error

bobgreenblatt say:

The correct syntax is:

if Range("a1")>0 then

   do stuff when it is greater than zero

elseif range("a1")=0

  do stuff when it is equal to zero

end if

Value is implied for a range, so you don't need to specify it. Sometimes the VB editor will give an error on the wrong line if it does not understand what you are trying to do. In this case, the error is probably because "then" is necessary following your test of a1.

ken sheridan say:
In the frmEmployee, there are employeeID and when I click it, it opens frmEmployeeDetail to put new records.

I'd suggest that you open the frmEmployeeDetail form in dialogue mode.  That way code execution in the calling procedure would pauses until the frmEmployeeDetail  form is closed, so you can then requery frmEmployee in the same procedure, e.g.

' open frmEmployeeDetail  to add new records in dialogue mode
DoCmd.OpenForm "frmEmployeeDetail",DataMode:=acFormAdd,WindowMode:=acDialog
' requery current form after frmEmployeeDetail has been closed
Me.Requery

Requerying the current form will move the record pointer to the first record in the sort order.  If you want to stay on the record which was current before the code was executed you'd need to include some further code:

Dim lngEmployeeID As Long

lngEmployeeID = Nz(Me.EmployeeID,0)

' open frmEmployeeDetail  to add new records in dialogue mode
DoCmd.OpenForm "frmEmployeeDetail",DataMode:=acFormAdd,WindowMode:=acDialog
' requery current form after frmEmployeeDetail has been closed
Me.Requery

' navigate back to last current record, if any,
' by finding EmployeeID in form's recordset's clone
' and synchronizing bookmarks
With Me.RecordsetClone
    .FindFirst "Employee ID = " & lngEmployeeID
    If Not .NoMatch Then
        Me.Bookmark = .Bookmark
    End If
End With
