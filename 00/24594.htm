This observation could also support my point about you having trouble with a different server.   None of the pdf files are hosted on the problem server.   Use the Developer Tools  Link Report  to see this.   (F12, Alt-V,r).   I suspect you are only having trouble with links which connect to the  "smartweb"  server, possibly only when using  HTTPS.   The LInk Report allows you to find links to that server which are not php or not HTTPS.  If you can access that server using one of those differences, that could help you further refine your symptom description.

I think you should prove that you have connectivity with that server.  Unfortunately I am not familiar with the troubleshooting tools that Vista provides.   If this was W7 you could use its Connect... to a particular website troubleshooter to test:  smartweb.keralamvd.gov.in

Another option would be to use  telnet  in a cmd window.  To do that you would have to ensure that you had the  telnet client  installed.   It's a  Windows Component which is not installed by default.

Then you could test connectivity to the site's  HTTP  server with

     telnet  smartweb.keralamvd.gov.in  80

Then wait for the screen to clear and enter:

     HEADERS / HTTP/1.1

Note that you will not be able to see your typing but that you could paste via right-click Paste if desired.

Then enter:

     Host: smartweb.keralamvd.gov.in

Then press Enter again.

If the screen cleared to enable you to enter those other lines, it would prove that you were talking to an HTTP server.  That's what the 80 on the telnet command does.  But it would only be getting response from those simulated HTTP request headers which would really prove that you were connected to the server you wanted to be connected to.

You do not need to keep Office 2008. Follow these instructions:

https://support.microsoft.com/en-us/kb/2500821
