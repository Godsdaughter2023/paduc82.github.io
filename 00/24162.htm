davekenroy say:

Save Current Record On Continuous Form

I have a form that uses a continuous subform to perform searches through a table. One of the features I've implemented is a boolean field that is selectable on the subform to mark a record for deletion. While this feature and the deletion works fine, the problem I'm running into is with a feature to set all the "Mark for delete" records that are true to false; a "clear selected" kind of function.

The issue I'm having is when a user selects the toggle button on the subform and then clicks the Clear Selected button on the main form, it cannot update that record as it is still locked for editing. The specific error is

"Microsoft Access was unable to update all the records in the update query.

Microsoft Access didn't update...1 record(s) due to lock violations..."

I found if the user clicks off the current record that is being edited and on to a different record, there's no error. I have already tried through VBA to move to the next record and requery the subform, but neither removed the lock.

Is there any way to programmatically commit the changes made to the record and remove that lock, say when the Clear Selected button is clicked?

Thanks for any help.

Edit: This database is only accessible to me at this time. No other user would be causing a lock.

One more piece of information is the backend of this database is a SharePoint 2010 list.

dirk goldgar say:

That makes sense.  I can think of a few lower-impact things to try, which might work instead of resetting the subform's SourceObject.  In descending order of efficiency, they are:

1. Refresh the subform:

        Me.NameOfYourSubform.Form.Refresh

2. Requery the subform:

        Me.NameOfYourSubform.Form.Requery

3. Reset the subform's RecordSource.

        With Me.NameOfYourSubform.Form

            .RecordSource = .RecordSource
        End With

So if any of those works, it's better than clearing and resetting the subform's SourceObject.
