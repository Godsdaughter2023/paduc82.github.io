Guide to Fixing Critical Error:Start Menu/Cortana Issue - Microsoft Community

zaxzia say:

Guide to Fixing Critical Error:Start Menu/Cortana Issue

I had this issue a couple builds ago.

It specifically occurred as a result of my computer sleeping. However, when I rebooted it as the Error suggested. It returned immediately and was not fixed. I found the solution on this forum, but I don't know where, so I am going to post how to fix it here, since I see a few threads on it, and hopefully the solution will reach you all more easily this way. Instead of getting lost in the threads.

After the error occurs do this to permanently fix it(at least until you update to the next build):

Use Ctrl+Alt+Delete.

Select Task Manager from the list.

When it opens click the File Menu.

Select Run New Task.

It will open a box. Type in msconfig.

Under the General Tab select Normal Startup.

Reboot.

All Fixed!

There is a very, VERY good chance it is set to selective startup. It may be set that way and you have no issues. However if you switch to Normal Startup, your critical error should go away. At least it did in mine, a several other cases.

After you Update your system to a new Build, there is a good chance it will switch back to Selective Startup. (Or even if you don't upgrade.) If it's a new build, the error might reoccur. However I haven't had it reoccur if it switched back to selective on it's own without an update.

For whatever reason it appears that the shellhost for those items is not being started during the selective startup option in some cases. I don't know why, or how. But in Normal startup, all the necessary processes do start, thus no error.

I would do this in VBA, not a macro. 

Dim intRecordCount As Integer

intRecordCount = intRecordCount + DCount("*","query1")

DoCmd.OpenReport acViewPreview,"report1"

intRecordCount = intRecordCount + DCount("*","query2")

DoCmd.OpenReport acViewPreview,"report2"

intRecordCount = intRecordCount + DCount("*","query3")

DoCmd.OpenReport acViewPreview,"report31"

intRecordCount = intRecordCount + DCount("*","query4")

DoCmd.OpenReport acViewPreview,"report4"

If intRecordCount = 0 Then

     msgBox "There was no data!"

End If

2- Just If are you using an office 365 click to Run Product follow the steps: Remove Office 365 license for Subscription based installs (not Shared Computer Licensing scenarios): To remove the Office 365 license, you must run two cscript command lines. The command lines are:
A. Run C:\program files <x86>\Microsoft office\office15>cscript ospp.vbs /dstatus The above command line will generate a report of the currently installed/activated license.
B. Make note of value for “Last 5 characters of installed product key”
C. Run C:\program files <x86>\Microsoft office\office15>cscript ospp.vbs /unpkey:“Last 5 of installed product key” For example: C:\program files <x86>\Microsoft office\office15>cscript ospp.vbs /unpkey:WB222. Repeat the step above if necessary until all keys are removed. After running the /unpkey: command line you will see a “Product Key uninstall successful” message. You can now close the Command Prompt and move onto Step 2.
3- Remove cached identities in HKCU registry:
A. In the Registry Editor navigate to HKCU\Software\Microsoft\Office\15.0 or 16.0\Common\Identity\Identities and remove all of the identities under \Identities.
NOTE: If using Shared Computer Licensing remove the above Identities from HKEYUsers\SID.
