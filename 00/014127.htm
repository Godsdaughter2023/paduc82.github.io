Click somewhere in the left-hand page. Press Ctrl+F to get into the search box on the Navigation pane, and type the code ^b in the box. Click the down arrow once; that should select the next section break after the current cursor position, even though it may be invisible.

jay freedman say:

There's very little else I can suggest.

One thing that may work is to start at the beginning of the document, press Ctrl+G to open the GoTo tab of the Find/Replace dialog, select Section in the list, and enter 9 in the box on the right. When you click the GoTo button, the cursor should land to the left of the first character in Section 9. Press the Shift key and the left arrow key once -- that should select the section break between sections 8 and 9, and pressing the Delete key should remove it.

I used the word "should" so many times in that paragraph because I'm not at all sure this will work. There could be some form of document corruption that makes Word think there's a new section start but won't let you find or select the "break".

If that's the case, the next attempt (and the last, as far as I know) is to look at the XML inside the document file. A docx file is really just a renamed zip file; if you make a copy and change its extension to .zip, you can open the zip file like a folder. Inside it, there are several folders, one of which is named "word". In that folder you'll find many files, one of which is named document.xml. When you extract that file from the zip to a regular folder, you can open it in an editor such as Notepad.

Using that editor's Find function, locate the first word of the unwanted new section. Immediately before that, you should see a bunch of XML tags  starting with

 <w:p w:rsidR="00266C77" w:rsidRDefault="00266C77"><w:pPr><w:sectPr w:rsidR="00266C77">

(but with a different hex number in the quotes) and ending with 

</w:sectPr></w:pPr></w:p>

In between will be specifications of headers and footers, page sizes, and other section-related items.

You can delete that whole chunk of XML, save the document.xml file, and paste it back into the zip container; rename the zip back to docx, and open it in Word -- if it will open rather than giving you an error message. If all that works, the extra section should be gone.

donleverton say:

Hmmm....

Shouldn't a Form have a SubForm, and a Report have a SubReport?

Regardless, you should be able to filter the report so that it displays information relevant to the selected customer only.

This can be done in a variety of ways, with the most common method probably being the use of the CustomerID from the form being passed as criteria to the query that is being used as the Recordsource for the report.

Here's an example of a query's SQL string that does just that:

SELECT tblCustomers.*, tblCustomers.CustID
FROM tblCustomers
WHERE (((tblCustomers.CustID)=[Forms]![frmCustomers]![CustID]));

(This query would also be used as the recordsource for the main report.)

a. user say:

Hi Alura,

 

Welcome to Microsoft Community.

 

I understand that you're unable to open links in your email messages. Let me assist you on this matter and rest assured that I will do my best to help you sort this out.

 

It is possible that a third-party pop-up blocker is blocking the link. The pop-up blocker feature prevents pop-up windows from opening when you click hyperlinks in email messages. To enable pop-up windows to open, please to perform the following steps:

 

Click the Internet Explorer information bar, and then select one of the following options:

  • Temporarily allow pop-ups: This option opens the blocked pop-up window. Additional pop-up windows will not open when you click hyperlinks.
  • Always allow pop-ups from this site: This option adds the hyperlink that you clicked to a list of safe websites, and then opens the blocked pop-up window and any additional pop-up windows on the current site.

Below are the steps to access the pop-up blocker settings.

 

1. Start Internet Explorer

2. On the Tools menu, click Internet Options.

3. On the Privacy tab, click Settings.

4. Under Address of website to allow, add the following sites to the list of websites to allow pop-ups:

 

          a. live.com 

          b. msn.com

          c. hotmail.com

          d. outlook.com

          e. passport.com

 

If you receive a warning message about pop-up window, or nothing happens when you click hyperlinks in email messages, you may need to contact the manufacturer or check the documents of the third-party software to add the websites I enumerated above (.coms) to the their software's list of safe websites.

 

You may also receive the following message at the upper part of the email message indicating that hyperlinks and images in the email message have been disabled. "Pictures and links in this message have been blocked for privacy. Show content." If this is the case, kindly click on the Show Content link to enable the hyperlinks or images in the email message.

 

I really hope that the matter will soon be resolved for you to be able to access/open links without any problem. Please do let us know how you get on and somebody will work with you if the issue continues.
