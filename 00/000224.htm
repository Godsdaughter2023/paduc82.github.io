peter doering say:
Do you really want to access your server through a web address?

Normally, if you are having problems accessing the server from A2007, try following:

Ribbon External Data - Import - More - ODBC Database - Link - Ok. The dialog for DSN selection starts (DSN = data source name). If you don't have a functioning DSN, click New. Select SQL Server as driver, then click Advanced. You will see the first cryptic part of the connection string, e.g. DRIVER={SQL Server}. From here onwards you can type the further connection settings. Syntax:

DRIVER={SQL Server};Server=<ServerAddress>;Database=<DBname>;UID=<UserID>;PWD=<Password>;
or
DRIVER={SQL Server};Server=<ServerAddress>;Database=<DBname>;Trusted_Connection=Yes;

Examples:
DRIVER={SQL Server};Server=192.168.0.1;Database=YourDB;UID=madsmom;PWD=secret;
or
DRIVER={SQL Server};Server=Servername\InstanceName;Database=YourDB;Trusted_Connection=Yes;

For more information on connection strings see: www.connectionstrings.com

Finish the wizard, save the DSN, then the login should work and server tables should be listed.

HTH - Peter

kimi gao msft say:

Hi Diego,

Yes. To delete the OneDrive icon on Windows Explorer, we can change the value of Syetem.IsPinnedToNameSpaceTree file in Registry Editor following the steps below:

  1. Press Windows+R and type regedit in the prompted Run box. Press Enter to open the Registry Editor.
  2. Find Syetem.IsPinnedToNameSpaceTree under HKEY_CLASSES_ROOT\CLSID\{018D5C66-4533-4307-9B53-224DE2ED1FE6}.
  3. Open it and change its Value data to 0.

WARNING: Using Registry Editor incorrectly can cause serious problems that may require you to reinstall your operating system. Microsoft cannot guarantee that problems resulting from the incorrect use of Registry Editor can be solved. Use Registry Edit at your own risk.

Kimi

robert aldwinckle on forums say:

I receive the message, "Internet Explorer cannot display the webpage" when trying to log onto www.google.com. Why and how do you resolve?


You need to do some diagnosis.   E.g. in a cmd window.  First clear the dnscache.

    ipconfig  /flushdns

Then try ping'ing the host just to test for addressability to it:

    ping  -n  1  -w  1   www.google.com

Chances are that that message refers you to a different host name.   You need to do a lookup for it too:

    ping  -n 1  -w 1   www.l.google.com

But in case there are any other aliases involved you should do an nslookup just to check:

    nslookup  www.google.com

FWIW  that shows just the host name that you are trying to use as an alias of the other one I have provided as an example.   Yours may be different and yours may refer to more than one alias.   You should check them all because a break in the chain would ultimately affect the final lookup, which is where you get the IP address from.

In any case once you have ping'ed all the hosts involved you will have a greater chance that IE will be able to use the lookups that you have cached for it.   Use another option of   ipconfig   to see this:

     ipconfig  /displaydns

In the DisplayDNS output from that you can check the  Time To Live  of each element of the lookup that your ping's have cached for IE.   Each value is in seconds, so as long as you have at least 60 on all elements you probably should have no trouble with a lookup by IE.   E.g. that would give you about a minute to do one.

If addressability was the cause of your symptom and if you're lucky, this will be enough of a workaround to avoid the symptom.   If not take careful note of what is happening in all areas of the XP IE8 window, Address bar, Status bar, Title bar, Tab label, as well as the information area.   Remember, your words have to substitute for what our eyes might notice.

A related diagnostic that you could try is the  Diagnose Network Problems  command in the Tools menu.  Unfortunately it does not test the URL that you are having trouble with but if you were having trouble with the hosts that it does test you could use the above procedure to try to assist its performance too.

 

Good luck

Robert Aldwinckle
