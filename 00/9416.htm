I believe what you want is this controlsource expression:

    =IIf([Redundancy - Gross - Total Gross Payment] > [Redundancy - Total Tax Free Portion], [Redundancy - Gross - Total Gross Payment] - [Redundancy - Total Tax Free Portion], 0)

 

As you can see, your use of hyphens in the names of your fields is cumbersome -- it means you have to surround the names with brackets so that Access can parse the expression properly and not confuse the hyphens with minus signs. It's confusing for people, too. :-)

dirk goldgar say:

One way would be to use a calculated control with a controlsource expression like this:

 

    =DLookup("Tax Free Amount - Initial", "TBL_Redundancy_Settings")

 

Expressions using this value would refer to the control name, not the field name.

 

Another way would be to create a subform with TBL_Redundancy_Settings as its recordsource, showing the field [Tax Free Amount - Initial], and place that subform on FRM_Redundancy_Calculator.  However, expressions that want to use this value would have to construct a reference involving both the subform control name and the control name on the subform, so would be a bit more complicated.

regsvr32 /u "C:\Program Files (x86)\Internet Explorer\ieproxy.dll"
