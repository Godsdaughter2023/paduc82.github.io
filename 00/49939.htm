Hello Michael,

Thank you for posting your concern in Microsoft Community and welcome to the Windows 10 Family.

Microsoft Input Configuration Driver is a kernel device driver. This event is triggered only if the user, an application or another service starts it.

If Microsoft Input Configuration Driver fails to start, the failure details are being recorded into Event Log. Then Windows 10 will start up and notify the user that the MTConfig service has failed to start due to the error.

Now, since it is a fresh Windows installation, chances are less that the Windows system files are corrupt.

So as an initial recommendation, I suggest you to boot the computer in safe mode with networking, and check if the issue still persist. This is a diagnostic step towards identifying the cause of the issue.

Safe mode is a troubleshooting option for Windows that starts your computer in a limited state. Only the basic files and drivers necessary to run Windows are started. Refer: Start your PC in safe mode in Windows 10

A clean boot is performed to start Windows by using a minimal set of startup programs. This helps eliminate software conflicts that occur when you install a program.

This is exactly where Microsoft's idea of disk organisation fails. By default Windows puts everything on the same partition which then results in huge image files. The matter is aggravated by the native imaging tool not having a compression option. On the machines I look after I always split things like so:

  • Drive C: - Operating system, programs.
  • Drive D: - User files

I create images of drive C:. Their size is usually between 20 and 30 GBytes. Very easy to restore.

I back up drive D: to an external disk. The size of the backup file depends on the amount of data.
