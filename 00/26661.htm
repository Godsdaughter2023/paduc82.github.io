a. user say:

Hi Norman,

 

By " existing host", do you mean your own email server (on premise server)? So the scenario I assume is: you have set up two email services (Office 365 and your on premise server) to use a single domain and you added Office 365 required DNS records to your existing DNS records which was pointing to your on premise server, then you had two MX records pointing to different servers. If this is the scenario, it's a normal behavior that something was wrong with the mail flow---- Office 365 doesn't support multiple MX records.

 

Given the situation, please check if set up connectors to route mails between your on premise server and Office 365 works: you need to set up two connectors: one from Office 365 to your server and one from your server to Office 365. Office 365 will send emails to the Internet on behalf of your email server and the MX records need to point to Office 365. You can check the detailed steps in this following article: https://technet.microsoft.com/en-us/library/dn751020(v=exchg.150).aspx

 

If this is not the scenario, please provide a little more setup steps so we can provide more targeted help.
