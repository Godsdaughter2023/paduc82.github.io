paul herber say:

I'm afraid that's not how layers work. If a shape master has layers defined then when the shape is dropped on a page then the shape will be allocated to that (those) layer(s).

When a shape without layer information is dropped on a page then if any layer(s) is active then it will have that layer added to it.

In Visio shapes are not contained in layers, instead, layers are a property of a shape.

So, what you are saying is almost right, it is correct for a master without layer information, but not for a master which has pre-defined layer information.

MSO 2011 Mac: Network Path in VBA - Microsoft Community

dariusmacpro say:

MSO 2011 Mac: Network Path in VBA

I have shared a folder called Commons under Documents on another Mac over the LAN. When I record a macro in Word VBA to open a file in Commons, VBA leaves out the part of the file's path preceding "Documents:Commons:".  What is the syntax for accessing a file on another Mac over the network?  All help is appreciated.
daniel g. grau say:

There are two forms of basic syntax:

Documents.Open Filename:="File_Name"

Documents.Open Filename:="Disk_Name:Path_With_Colons:File_Name"

Thus, if you mount the user's home folder on your Desktop, you would use:

Documents.Open Filename:="Home_Folder:Documents:Commons:File_Name"

And if you mount the shared HD at the top level, you'd use:

Documents.Open Filename:="Disk_Name:Path_With_Colons:Documents:Commons:File_Name"

The above path should include "Users:".

When you record, if you are already at the target folder when you select File > Open, Word will only record the file name:

Documents.Open Filename:="File_Name"

But if you first have to navigate to the folder, it will record two statements:

ChangeFileOpenDirectory "Disk_Name:Path_With_Colons:Documents:Commons:"
Documents.Open Filename:="File_Name"
