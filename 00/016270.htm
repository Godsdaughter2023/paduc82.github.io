joeu2004 say: MrGomex wrote:

If the number 41908.999999 is displayed as date, it says 27/10/2014 (I would expect 26/10/2014), but when you want to edit the cell, it actually says "26/10/2014 00:00:00" on the top bar (where you enter the formulas).  Also, why does the DAY function do:

Excel rounds seconds first.  And yes, I find this infuriating, too.

Use the format Custom "dd/yy/mmmm hh:mm:ss.000" without quotes.  You will see:

41908.99999 is 26/09/2014 23:59:59.136
41908.999999 is 26/09/2014 23:59:59.914


These are rounded to 26/09/2014 23:59:59 and 27/09/2014 00:00:00 respectively.

hansv mvp say:

41908.99999 corresponds to 26/09/2014 23:59:59.136

41908.999999 corresponds to 26/09/2014 23:59:59.914

You can see this by applying the custom number format dd/mm/yyyy hh:mm:ss.000 to the cells.

When rounded to the nearest second, the former becomes 26/09/2014 23:59:59, but the latter becomes 27/09/2014 00:00:00.

Apparently functions such as DAY round the time value to the nearest second before computing the day number. You can get around this by using INT: the formula =DAY(INT(41908.999999) returns 26.

ossiemac say:

I am a bit confused. At the top of the thread it indicates "Office 2010 Microsoft Office Programming Excel" and I assumed you are working in Excel. However,  ActiveDocument normally references a Word document so I have now assumed that you have used the term as an object variable and have a worksheet assigned to it so if I have assumed incorrectly then please advise.

If you are working in Excel then the following code does the following.

  • Saves the current width of the active cell.
  • Temporarily AutoFits the column width to the width of the ActiveCell contents.
  • Saves the temporary AutoFit width of the column.
  • Resets the column width to the initial saved column width.
  • Loops through the cells to the right of the active cell and sums the widths until the sum is equal or greater than the temporary width and then sets the interior color to the identified range.

Note that ColumnWidth and Width are 2 different measurements. The Width of a column in pixels can be read but not set. The ColumnWidth is a strange one in that it is apparently an average number of characters and is not uniform because not all characters are the same width in pixels. Therefore ColumnWidth is set but Width is read in this code otherwise it is not possible to accurately determine the number of columns to use for the interior color setting.

The code between the asterisk lines is OssieMac's added code.

Sub FullPath_NameExtension()

    '*********************************************************
    'Dimensioning of variables and code between asterisk lines _
     added by OssieMac
    Dim ActiveDocument As Worksheet
    Dim dblSaveWdth As Double
    Dim dblNewWdth As Double
    Dim dblSumWdth As Double
    Dim c As Long
   
    Set ActiveDocument = ActiveSheet    'OssieMac's assumption
    '********************************************************
   
    With ActiveDocument
         ActiveCell.Select
         ActiveCell.FormulaR1C1 = _
            "=SUBSTITUTE(LEFT(CELL(""filename""),FIND(""]"",CELL(""filename""))-1),""["","""")"

         Selection.Font.Bold = True
         Selection.Font.Size = 12
         Selection.Font.Color = -16776961
         Selection.Font.TintAndShade = 0
        
         '*************************************************************************
         'Following code between asterisk lines added by OssieMac
         With ActiveCell
            dblSaveWdth = .ColumnWidth      'Save initial column width
            .Columns.AutoFit                'AutoFit the width of ActiveCell
            dblNewWdth = .Width             'Save new AutoFit width
            .ColumnWidth = dblSaveWdth      'Return ActiveCell to previous width
           
            Do
                'Sum the column widths until >= to required column width
                dblSumWdth = dblSumWdth + .Offset(0, c).Width
                If dblSumWdth >= dblNewWdth Then Exit Do
                c = c + 1
            Loop
        End With
        Range(ActiveCell, ActiveCell.Offset(0, c)).Interior.Color = 65535
        '**************************************************************************
       
     End With
 End Sub
