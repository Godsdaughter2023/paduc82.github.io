count number of times multiple values occur in an array - Microsoft Community

susanmurphy say:

count number of times multiple values occur in an array

I have 21 parts labeled PP1 through PP21.  I can use 4 (or less) parts per run.  Example:  Run 1 - PP1, PP5, PP7, PP21; Run 2 PP5, PP8, PP10, PP12... etc.  After 25 usages of each part, I need to refurbish it.  I would like to create a counter that flags the operators when a part has been used 25 times.

My current tracking method is in excel where they just enter the parts used as text (1-2-7-9).  Can you tell me an efficient method of data entry and analysis so I can count the part usage?

 

 

rich007a say: In addition to using COUNTIF, you'll also need to know how many are left after the parts bins have been replenished multiple times.  Providing all bins are ALWAYS refilled when 25 parts have been used, you can use the MOD function to return the remainder of your countif function divided by 25:

=MOD(COUNTIF(B2:E1000,"PP1"),25)

if you have your Part1, Part2, Part3, Part4 listed in columns B-E (run number in col A), the above formula will tell you how many of PP1 you have used from a box of 25.

Cheers
Rich

[EDIT: Sorry, I missed the last line in your post about "1-2-3-4", etc.  My example assumed you had entered the parts in separate cells (same number of key presses as using "-" between the numbers; for separate cells just hit the left arrow or the tab key), but anyway, if your data is in n-n-n-n format, the use Ron Coderre's formula, in combination with the MOD formula like this:

=MOD(SUMPRODUCT(--ISNUMBER(FIND("-"&E2&"-","-"&$A$1:$A$100&"-"))),25)

And of course to get the number left in a box use:

=25-MOD(SUMPRODUCT(--ISNUMBER(FIND("-"&E2&"-","-"&$A$1:$A$100&"-"))),25)

Hope that helps.
Cheers
Rich

When cached mode is off, Outlook needs to contacts the server for *everything* - this creates a lot of network traffic and Outlook will be slow and may hang as it contacts the server. If you have a lot of employees working with a remote server in online more, it will slow the network down.

When you had an in-house exchange, you probably had a faster network (100mbit/sec, if no 1000) - your access to fasthosts is slower.

If you are caching the user's mailbox but not caching shared (which is what I intended you to test), they should only experience slowness when they access a shared mailbox.

I would enable caching then clear the cached folders that are not updating correctly. If a lot (more than about 5 ) are sharing a mailbox and actively using it a lot, it can cause the issues you reported where not all mail syncs. I don't know that there is a good solution, other than licensing it and adding it as an account, at least for some users.
