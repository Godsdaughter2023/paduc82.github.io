INDEX + MATCH Breaks When Sorting - Microsoft Community

ekportland say:

INDEX + MATCH Breaks When Sorting

I am seeing some odd behavior with INDEX + MATCH when the sheet name is included in the reference cell for the match. After sorting, the reference cell location does not update. This is not true if the sheet name were not included - this behavior should be the same whether the name is included or not I believe.

  • ISSUE
    • Excel has certain limitations when sorting an INDEX function that includes a MATCH to the current sheet.
    • When the MATCH function uses the Sheet Name to reference the cell (shown in examples below), it may not update the reference cell location if the user were to sort the data.
EXAMPLES

The example below shows the issue in a very simplified manner.

In the first screenshot, you can see that prior to sorting, the INDEX + MATCH formula is referencing the same row that the data is in (e.g., the INDEX in row 2 is matching Tacos in row 2. Note that the tab name is also included in the reference cell (i.e., Data1!). 

Next, if the user were to sort on column A (Food), we can see in the screenshot that the reference cells do not update. That is, they retained the previous location, rather than updating to the reference cell’s new row. In the example below, you can see that Burritos, which is now in row 2, has maintained cell A3 as the reference in the MATCH. This should have been updated to follow the new location of Burritos.

SOLUTION

The fix is simple. When writing the INDEX + MATCH formula, one should never include the Sheet Name in the MATCH if the reference cell is in the same sheet. For example, the example above should remove “Data1!” in the MATCH portion and instead just reference the cell locations. Once you do that, then the sort works fine and the reference cell location updates with column A. 

The real question is why is this behavior different whether the sheet name is included or not? It seems like that is an error in how Excel is handling the sort. In fact, I would expect that the behavior would be the same whether or not the current sheet name is included in the reference cell location or not.

Is the Microsoft Excel team aware of this issue? Is this something that needs to be fixed in the software?
