steve schapel say:

Simian,

As far as I know, you can't sort on a calculated control within a grouping on a report.  I agree with Bill's comments, if we are just talking about the easiest way to get the Group counts on the report.  But if you want them sorted, I think you need a different approach.  I think you will need to make a Totals Query that gives you a count for each Group.  And then, you will add that query to the existing query that the report is based on, suitably joined on your Group ID field.  And then you will have the group count figure right there in the query, which you will be able to use to Sort on in the report, and show on the report in a bound control (i.e. not with the Count(*) function).  Does that make sense?

steve schapel say:

Mike,

I am happy to hear that you like some of the enhancements that Access has seen in the last 10 years! :-)

It is hard to advise explicitly on the query, without knowing some details of your data.  But if you can make a cut-down query, that lists all of the records that will be in the report - but not all of the fields will be required, only the one to identify the Group that you have been referring to, and possibly the primary key field of the report's "core" table, if there is such a thing.

Then, in the design view of the query, select the Totals button in the 'Show/Hide' group on the ribbon's Design tab, which will put a Total row into the query design grid.  Leave it as 'Group By' in the group field, and enter Count in the other.

So the "bare bones" SQL view of such a query will look something like the equivalent of this:

SELECT GroupID, Count([YourMainID]) As GroupCount
FROM YourOtherQuery
GROUP BY GroupID

A variation on this may be necessary to get exactly what you want - as I said, it's hard without seeing it exactly.  But hopefully that will  point you in the right directlion.

Then, you save this query, and then add it to the query the report is based on, join to the existing core table on the equivalent of the GroupID, and add the Count field into the query's output.

Steve Schapel Microsoft Access MVP

Reinstalling Office on the same PC does not legally use another activation, but operationally it might.  Chances are that the activation will be automatic but if you find that reinstalling results on an activation failure this time or any future time when you know that you have not exceeded your legal activation limit, then use the Activate by Phone option and explain to a person what happened.  They should allow the activation.
