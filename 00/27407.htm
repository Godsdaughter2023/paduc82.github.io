a. user say:

You declared rngInfo as a string which limits it to containing text.

When declaring a variable as a Range use...
  Dim rngInfo As Range  (As Variant will also work)

Then you have to tell Excel where the range is located using the "Set" statement.
(for which you used the correct syntax)

Using a named range is a little different. 
Excel (internally) assigns the name of the worksheet and the cell locations to the name when you establish the name.
Assuming that "rngPrintMode" is the named range, then all that is necessary to use the name with VBA is...
  Range("rngPrintMode").Value = "Yes"'

e.j. gun say:

First, please add "Option Explicit" to the top of your code modules if you don't already have it.  Why?  It will help you find errors in mis-using variables.

 

In this case, you define rngInfo as a string, but then you try to set it to a Range object.  That doesn't work.  Perhaps you meant:

 

rngInfo = Worksheets("VBA Control Panel").Range("rngPrintMode").Address  ?

 

That line will return a string, which is what you defined rngInfo to be.

 

For the second line, I don't see where you actually define rngPrintMode.  You try to set its value to "Yes", as if it is a Range, but you have declared it as a Variant, and never set it to anything before that line.  No surprise you are getting an error.

 

The only line you should need is:

 

    Worksheets("VBA Control Panel").Range("rngPrintMode").Value = "Yes"

brian tillman [outlook mvp 2007-2019] say:

The splitting is done ia a rule, so you can simply create your own new folder and rule to accomplish the same thing.

First, click the "New folder" link at the bottom of your folder names in the left-hand pane and create a folder to receive the messages for your new (orl old, if you prefer) address.  Then click the Options gear at the upper right and choose "More mail settings".  In the right-hand column, click the link titled "Rules for sorting new messages".  Click New.  You'll see a dialogue containing a line with two drop-downs and a text field, plus some radio buttons for actions.

In the left drop-down, select "To or Cc line".  In the next drop-down, choose "contains".  In the text field, enter your new (or old) address.  Select the radio button next to "Move to Inbox", click the down-arrow to the right of "Inbox" and pick the folder you created in the first step.  Click Save.  From then on, messages addressed to the that addres will get moved to the folder you selected.

It's not a matter of uninstalling or reinstalling. The software is the same for all license types. It's a matter of logging out of the volume license user and then logging in with your Office 365 credentials.

rich007a say:

Hi,

I'd use some helper columns here.

Insert one helper column between A and B to indicate which cells contain duplicates.  You can use a formula like this in cell B2 and drag down:

 

=IF(COUNTIF(A$2:A5,A5)>1,NA(),"")

 

You will now have cells containing #N/A next to any duplicate cells (but not the first occurances).

You'll be able to select those by doing Home tab > Find & Select > Go To Special > check Formulas and then uncheck all except Errors > OK.

 

But before you delete any, you need to work out the logic for keeping what is now in col C.

Modify the formula as follows:

 

=IF(AND(COUNTIF(A$2:A2,A2)>1,C2<>"Text to keep"),NA(),"")

 

editing the text as required.

Fill the revised formula down and do the Go To Special > Errors again.  Now all the cells in col B containing #N/A will be selected.  Right-click on one of the selected cells > Delete > Entire Row > OK.

 

Hope that helps.

Cheers
Rich
