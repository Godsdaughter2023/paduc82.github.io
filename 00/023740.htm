jllatham say:

OK, here's what I did and have gotten good results:

 

First, I put in a blank CD and dragged an Excel file onto it.  When Windows asked me what type of CD I wanted to create I chose the "Use like a Flash Drive" option, which creates what is known as a multi-session CD.

 

I closed Excel and then opened the file from the CD. Yes, it was there.

 

Then I ran Gord's code as a stand-alone Sub and even removed the .DisplayAlerts = False statement.  Code looks like this:

Sub SaveBackupToCD()
 'Saves the current file to a backup folder and the default folder
 'Note that any previous backup is overwritten
'     Application.DisplayAlerts = False
     ActiveWorkbook.SaveCopyAs Filename:="F:\" & _
             ActiveWorkbook.Name
     ActiveWorkbook.Save
'     Application.DisplayAlerts = True
 End Sub

Now, that works fine and dandy as long as you've opened the file from someplace OTHER than the CD.  If you open it from the CD, then try to use that code, you'll get an error telling you that the file is in use - which it is!  So that's a valid error: you're trying to save a backup copy over the real-deal while it's in use.

Here's the code and some notes I used for the .SaveAs test:

 

Sub JustSaveToCD()
'change the "F:\" to appropriate drive and path
  ActiveWorkbook.SaveAs _
   Filename:="F:\" & ActiveWorkbook.Name
'this time I got the "File Exists..." prompt!
'so if you don't want to see that, put the
' Application.DisplayAlerts=False
'and
' Application.DisplayAlerts=True
'statements before and after the
'.SaveAs command
End Sub

And while working with the copy from the CD, regular Save works just as it would from any other device.

 

The key is the initial formatting of the CD - you have to have set up the CD for multi-session use.  So Gord's initial posted solution is definitely a good one as much as I can see.

 

Remember that CD/DVD writing is much slower than from pretty much any other device, so it could take some time to complete with really large files.

 

 I don't have a CD-RW disk to test with here, but I suspect that you need to make sure that the "session" is not closed on it also.
