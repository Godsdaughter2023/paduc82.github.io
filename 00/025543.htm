jeeped say:
For an Excel 2003 compatible formula you could use SUMPRODUCT() or an array formula with nested SUM(IF(IF())).

      

The standard formula in F2 is,

=SUMPRODUCT(($A$2:$A$999>=D2)*($A$2:$A$999<E2)*($B$2:$B$999))

It's worth mentioning that SUMPRODUCT() chokes on summing anything that isn't a number and I had to remove some spaces from the apparently blank cells in column B to make it work. The following array formula from F3 will happily skip over spaces of text as it is based upon the more forgiving SUM() function.

=SUM(IF($A$2:$A$999>=D2,IF($A$2:$A$999<E2,$B$2:$B$999)))

This requires finalization with Ctrl+Shift+Enter rather than simply Enter to create the array formula.

Both of those should be compatible with your Excel 2003 or 2007. Note that I've used greater than or equal to for the start date but only less than for the end date in these examples.

doug robbins - mvp office apps & services (word) say:

I would not refer to what you want to do as Conditional Formatting.  However, if you were to create an AutoText Entry that contained the required table row and its contents, you could use code such as the following in the This Document object

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
Dim i As Long
With ActiveDocument
    For i = 1 To .ContentControls.Count
        If .ContentControls(i).Tag = "CheckYes" Then
            If .ContentControls(i).Checked Then
                Selection.Tables(1).Range.InsertAfter .AttachedTemplate.AutoTextEntries("Name of AutoText Entry")
                .ContentControls(i).Checked = False
            End If
            Exit For
        End If
    Next i
End With
End Sub

Replicate several columns from a table into multiple worksheets - Microsoft Community

schwarzm say:

Replicate several columns from a table into multiple worksheets

Greetings,

 

I am using Excel to track training tasks and events for personnel in my department.

I would like to have one worksheet that is used for all sorts of member information, such as name, social security, date of arrival, title, office number, phone number, etc.

I want each of the other worksheets to have some of the exact same meber information (Lastname, Firstname, MI), while also having other stuff pertaining to specific training tasks or events.

I have made the worksheet with all the member information and put it into a nice table.

How would you recommend that I replicate just a few of the columns into the other sheets so that the formatting is also replicated, and any changes/additions/deletions on the main member sheet get automatically made in the other sheets?

 

Thank you. I am using Excel 2007 on a Windows box.

jllatham say:

I'll take the last part first: the problem of replicating the formatting.  You simply need to set up the formatting in those other sheets ahead of time for as many rows as you have members plus perhaps a several more rows.  How many more depending on how fast the turnover is in the department.  You can always easily format new unformatted entries later on by using the format painter.

I don't know how your first/master table is set up, but now sounds like a good time to make any changes to it before you get too far along in the project.  I recommend that you set up the social security number in the left-most column of the table, probably table A.  Either it or another equally unique identification such as their employee ID number.

This would allow you to use VLOOKUP() formulas on the other sheets to bring specific information over to them using a match on the unique number (ssan or employee ID).

Let's say your master table looks like this, and is on a sheet named Master, it goes down to row 105.

        A                  B                  C                  D  ........   X
1   SSAN            LName         FName         DOB         Phone
2   000112222

The other sheets would also have SSAN entries in column A.  For this example we'll say you just want a quick phone list to reference.  It looks like this:

          A                        B          C          D
1    SSAN               LName    FName   Phone
2    000112222

In B2 you would put

 =VLOOKUP($A2,Master!$A:$X,2,False)  which will return the value from the 2nd column in the table (last name)

In C2 you would put

=VLOOKUP($A2,Master!$A:$X,3,False) which will return the value from the 3rd column in the table (first name)

and in D2 you would put

=VLOOKUP($A2,Master!$A:$X,24,False) which will return the value from the 24th column in the table (phone #)

Fill the formulas down as far as you want on the other sheets.  Now you may want to fill down for some extra rows to pick up added entries from the Master list later on.  If you do that, you will start getting #NA! errors at some point, to prevent that, you can "wrap" the VLOOKUP() formulas in an IFERROR() statement to return "" (looks empty) when that happens like this:

=IFERROR(VLOOKUP($A2,Master!$A:$X,2,False), "")
