scottgem say:
Yes, don't use Multi-value fields. 

The problem here is a new one to me, but it makes sense if you understand what a multi-value field does. Essentially it creates a hidden table to store the selected values. So, if two users are trying to write to that table at the same time it makes sense that a conflict can be caused. 

So the answer is to use a child table instead of a multi-value field. Create two tables:

tblProjectDept
ProjectDeptID (PK Autonumber)
ProjectID (FK)
DeptID (FK)

tblProjectAnalyst
ProjectAnalystID (PK Autonumber)
ProjectID (FK)
AnalystID (FK)

Then use Subforms to enter the depts and analysts for the project. Each subform would have a combobox to select a dept or analyst (but not a multi-value) the other 2 controls would be hidden.

Aside, to Alan
Using a value list should be rare. Instead you should use a table as the Rowsource of your combos. This makes it very easy to update, by just adding a record to a table.
