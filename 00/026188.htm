ken sheridan say:
In other cases the Category total can be reached by any combination of Sub Category Hours.

Does that imply that each 'class session' can relate to more than one subcategory of a category?  If so you'd need to decompose Scott's tblClassSession table as follows:

tblClassSession
....ClassSessionID (Pk Autonumber)
....ClassID (FK)
....InstructorID (FK)
....ClassDate

tblClassSessionSubCategories
....ClassSessionID (FK)
....SubcategoryID (FK)

The primary key of this table is a composite one of both columns.

Do you actually need an HoursEarned column in either of these tables? You say 'The Categories each have a certain number of Hours associated with them, in a connected basic table. In certain cases the Category stands alone. In other cases the Category total can be reached by any combination of Sub Category Hours'.   If these time-independently determine the HoursEarned values per session then such a column in either of the above tables would be redundant.  The HoursEarned column would be in Scott's tluSubcategory.  Note that if you wish to associate a class session directly with a category rather than via a set of subcategories then, for each category there should be a 'N/A' or similar subcategory for each category, whose HoursEarned value would be that for the category as a whole.

jim gordon mvp say:

You will get that message when the file is open by another user, is open in Excel on your computer, was saved as "read only," or if the file is being used by anyone as a data source for a mail merge.


Quit Word and Excel to release the file. Or better yet, restart your Mac.


If the file was saved as "read only" then use Save As and save it with a new name without the "read only" attribute.
