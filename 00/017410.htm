(1) Why does it lock out .accdb files?

(2) How can I stop it?

Moved from: OneDrive / Working with files and folders on OneDrive / Editing and viewing files and folders / OneDrive in Windows 10

dirk goldgar say:


Dirk - thanks for the detailed commentary. I know that MS Access requires lots of (constant...) read/write to the files. And I know that I could use your solution, which is a complicated way of saying "don't use OneDrive".

Yes.  Don't use OneDrive for Access databases (no matter how much you may want to).

When I used other cloud services to sync/backup files, I didn't have this problem.

Did you use those services with Access databases, while the databases were in use?  And did it work?  If it did, please tell us what services you were using.

My initial questions are:

(1) Why does it lock out .accdb files? I was hoping someone could tell me why OneDrive treats .accdb files differently to other files (which it doesn't lock). While a large .doc file is syncing in OneDrive, I can go open the file, edit it, and save it. Why doesn't this also work with .accdb files?

I don't know the details of how OneDrive locks regular files, much less Access databases, and it's possible that it has special handling for Access databases, designed to avoid accidentally corrupting them.  When Access is performing database operations, certain types of operations, which involve multiple updates to various different pages in the database file, *must* complete as a group, or the database will be left in an inconsistent state.  Even making a manual file copy of a database that is in use may result in a corrupt copy, though other times it may work fine, depending on what exactly is going on in the database.

Access manages the locking of pages within a database, with respect to other Access processes that may be using the database, using the .laccdb or .ldb file.  This is entirely separate from file locking performed by the operating system.  I believe that it also updates the database file's last-modified date/time each time it writes to the database.  So the pattern of usage of an Access database is one of constant updates to the file while the database is in use.

This completely different from the usage pattern of other Office documents, such as Word documents or Excel workbooks.  Word and Excel work on their documents entirely in memory, only updating the file when you explicitly save the file (or an autosave operation runs). And when they save, they update the entire file at one go.  There's no such thing as a "partial save".

That means that Word and Excel can coordinate with OneDrive in a very simple way.  If they try to save a document and OneDrive is currently copying it, then they can simply wait for OneDrive to finish.  On the other hand, OneDrive won't even try to copy a document file until its last-modified date has been changed, which means that OneDrive won't start a copy operation until Word/Excel has saved the document.  I'm sure it's a bit more complicated than that, because OneDrive has to coordinate saves by multiple users.  But ultimately, the difference between Access and other Office programs is that Access databases are not documents that are read into memory, processed in memory, and then saved to disk as a unit.

When you open a large .doc file while it is syncing in OneDrive, it's easy for Word to either wait for the syncing to finish, or else load a shadow copy provided by OneDrive.  I don't know if that last is what happens, but I can imagine the possibility.

(2) How can I stop it? The solution offered so far (Solution #1) is "don't use OneDrive" (but copy the files there when done). However this means that all the files have to sync, every time. This isn't acceptable, nor even sensible. Only the files that are edited need to be synced. And the point of using OneDrive is that if a HDD fails, or power is lost, then the files are not lost. To copy the files over when using them (for days, perhaps a week at a time) and then copy them back, nullifies the point of using OneDrive.

Don't use OneDrive except as a location for backup copies made of closed databases.  Make regular backups.  If you are concerned about data on a minute-by-minute basis, Access is probably not the right type of database to use for your back-end.  You should consider a client-server database instead;  for example, SQL Server.  At the cost of some performance, you could even use SQL Azure, with your back0end hosted in the cloud.

Solution #3: Move the front-end and all the files to a different directory (i.e., one that OneDrive doesn't mess up), and write a script that runs every minute to check LastModified on the files. When it detects a change, then, rename the file (removing .accdb) and copy it to OneDrive. This has an overhead (but not much). If I can implement this solution, then surely there's a fix here that OneDrive itself should be implementing? I.e. an option to "not treat .accdb files in a special way"?

I would never run such a script every minute, but you might run it every hour, or half-hour, if you are so concerned about any possible data loss.  You would want to maintain something like a grandparent-parent-child structure of separate backup files, since no one backup could be guaranteed to be 100% readable (since they would have been taken while Access was active in the database).  That said, if you have that level of concern, then you'd do better with a client-server database.

Both Hotmail & Outlook.com no longer support read receipts.  Window Live Mail does however.

The read receipt feature is currently not available.  To have this feature, you may use Windows Live Mail (email client). Please refer to the steps and links below on how to configure your account in Windows Live Mail to start using the read receipt feature:

1. Download and install Windows Live Mail.

3. How do I add an account to Windows Live Mail.

1. Open Windows Live Mail client.
2. Press Alt + F, O, M.
3. Click on Receipts tab.
4. Under Requesting Read Receipts, check the checkbox Request a read receipt for all sent messages.
5. Under Returning Read Receipts, click the radio button: Notify me for each read receipt request if you want to return read receipts.
6. Click Apply then OK.
