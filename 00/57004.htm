Hello!

Having password set on a user account is a good practice in an office environment.

To set/change a password for the current account:
Once logged on, press CTRL+ALT+DEL, choose "Change Password".

To create a new user account in Windows 8.1:

At the Start Screen ( the one with live tiles ) type PC Settings [ENTER], go to Accounts -> Other Accounts -> Add an account. Click "Sign in without a Microsoft account(not recommended)" - above Next button. Click "Local account". Enter username, password, hint, click Next then Finish.

In "Other accounts" select the user you just created and click Edit. As Account Type choose "Administrator".

But, if you don't use encryption any Administrator user can access your files and  even someone else that can boot from a CD/USB.

You can:

1. Password-protect office files -> in Word at "Save As" dialog, click "Tools" - next to "Save", "Save Options" - set "Password to open"

or

2. Place the sensitive files in a folder, download a ZIP archive manager and create a password protected archive with that folder

For example: http://www.7-zip.org/download.html, 32-bit x86 or 64-bit x64 versions.

or

3. Encrypt the specific files/folder -> being logged on your password-protected user account, right-click the file/folder in File Explorer, Properties->click "Advanced" button, tick "Encrypt contents to secure data". You shall back up your encryption key after doing this.

Risky in case you lose access to your account or you do not have the encryption key.

To export the encryption key -> type certmgr.msc at "Start Screen" [ENTER].

Follow the steps described here - from Step 2:

http://windows.microsoft.com/en-us/windows/back-up-efs-certificate#1TC=windows-7

I recommend you not to use method 3 unless you are an advanced user.

a. user say:

The error 80072EFD means "A connection with the server could not be established".

This is a basic networking error.

Here are a few things to check:

1. Can you browse using IE or Edge to ANY web site?  If not, then that confirms this is a general internet connection problem. 

2. Can you browser using Chrome, Firefox or any other browser to ANY web site?  If you can, then this indicates the problem is a setting shared by IE and Edge, and not the other browsers.

Here are a few things to try:

Check your Proxy

  1. Click Windows Start, and select Settings
  2. Click Network and Internet
  3. Click Internet Options
  4. On the Connections tab, click the LAN Settings button. If "Use a proxy server for you LAN" is checked, ensure you are on a company or school intranet that requires this. If you are not, then try de-selecting this option.

DNS settings

  • Some users have reported that after updating the IP addresses of their DNS servers (responsible for turning a name, like windowsstore.com, into an IP address the computer can use) they were then able to get to the store. DNS configuration is generally configured by whoever manages your network connections, such as your work, or at home, by your internet server provider such as the cable or phone company.  These generally should not be changed.

Other settings that help with other Windows Store issues.

  • On the Advanced tab, scroll to the bottom and ensure Use 1.2 is checked.
  • Ensure the Date/Time on your computer is set.
  • Try removing your internet connection and re-establishing it.
  • If those options do not resolve the issue, try the "Reset" button on the Advanced tab to reset all IE settings.

Let us know if those steps work!

Chkdsk reaches 21% completion the stops n exits. - Microsoft Community

freakmorbidity say:

Chkdsk reaches 21% completion the stops n exits.

Received messages off multiple files being corrupted. Tried to run System Restore, but it says chkdsk needs to be run first. When I run chkdsk, it reaches 21% then stops and exits and solves no problems.  Have done this several times with same resuls.
shinmila h say:

Hey freakmorbidity,

 

The problem could be a logical corruption of the file system or it might even indicate a hardware problem with your hard disk drive.

 

The most likely possibility for this error message is that Chkdsk could not access a file because the file is being used by another program. The solution for this problem is to run the 'CHKDSK /R' in a command window, and then reboot your system so that Chkdsk can have exclusive access to all files by running at startup.

 

Here’s how:

 

Note: To use Chkdsk, you must log on as an administrator or as a member of the Administrators group.

 

a.    Go to Start and type cmd.

b.    Right-Click on cmd and select “Run as administrator”.

This opens command prompt in elevated mode.

c.    To repair errors, locate bad sectors, and recover readable information, at the command prompt, type “chkdsk volume: /r” without the quotes, and then press ENTER. For instance, to run chkdsk on C: drive, type:

Chkdsk C: /r

 

Note If one or more of the files on the hard disk are open, you will receive the following message:

Chkdsk cannot run because the volume is in use by another process. Would you like to schedule this volume to be checked the next time the system restarts? (Y/N)

 

Type Y, and then press ENTER to schedule the disk check, and then restart your computer to start the disk check.

 

See where it leads you.
