You are probably going to need to use a form with a subform.  The problem is going to be that the primary form needs to be a single form (not continuous).

You might be able to use a single form for Table1.

It is important that you import, not open, the CSV file and select Text for the column format in the Import Text File wizard (last dialog box before Finish).

Otherwise, Excel interprets the data as numbers; and as such, Excel interprets only the first 15 significant digits, replacing any digits to the right with zeros.

That is just the way that Excel handles CSV files.  There is nothing that we can do in the CSV file itself to prevent it.  Specifically, prefixing each number with an apostrophe or enclosing each number with double-quotes in the file does not stop Excel from interpreting data as numeric, if it can.

Alternatively, you could write a VBA procedure (macro) to input and interpret the file yourself.

Close document, then open document again and scientific format shows up again. [....] Is there something in the settings of office 365 that I'm missing? 

After you open the CSV file (which you will not do anymore :->), the file is still a CSV file, despite the fact that it looks like a spreadsheet.  Look at the name above the spreadsheet:  it has a ".csv" extension.

So each time you save and close the file, you are creating a CSV file, which is simple text without formatting.  So Excel continues to interpret the data as described above.

Instead, you must do Save As and specify an Excel spreadsheet extension:  xls, xlsx, xlsm or xlsb, whichever applies.
