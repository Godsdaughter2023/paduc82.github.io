pete_uk say:

I agree with Mike that using filters will be the easiest way, but you can do this using formulae in 2 helper columns on Sheet1, and then in Sheet2 select the User and month and the corresponding invoices will be displayed there.

I'm not sure from your example if the date is a proper date formatted to show ddmmyy, or if this is a text value that just looks like a date in that format. If it is a proper date, then put this formula in F2 (for example):

=IF(A2="","-",A2&TEXT(A2,"mm"))

If it is a text representation of a date, then you can use this formula in F2:

=IF(A2="","-",A2&MID(C2,3,2))

Then in G2 you can use this formula:

=IF(A2="","-",F2&"_"&COUNTIF(F$2:F2,F2))

You can copy these 2 formulae down as far as you want to (even beyond your existing data - hypens will show if there is no data in columnA).

Then in a new Sheet2, you could use the top row for appropriate headings, and use A2 to enter the User that you want the details for. You can use B2 to enter the month (as a number), and this could be validated using Data Validation so that only the values 1 to 12 are acceptable. In C2 you can put this formula:

=A2&TEXT(B2,"00")

Columns D, E and F in Sheet2 can be used for the Invoice Number, the Date and the Value respectively, and you will need these formulae:

D2:  =IF(ISNA(MATCH($C$2&"_"&ROW(A1),Sheet1!$G:$G,0)),"",INDEX(Sheet1!B:B,MATCH($C$2&"_"&ROW(A1),Sheet1!$G:$G,0)))

E2:  =IF(ISNA(MATCH($C$2&"_"&ROW(A1),Sheet1!$G:$G,0)),"",INDEX(Sheet1!C:C,MATCH($C$2&"_"&ROW(A1),Sheet1!$G:$G,0)))

F2:  =IF(ISNA(MATCH($C$2&"_"&ROW(A1),Sheet1!$G:$G,0)),"",INDEX(Sheet1!D:D,MATCH($C$2&"_"&ROW(A1),Sheet1!$G:$G,0)))

Note that these are very similar - they just get data from different columns as can be seen in the terms after the INDEX function. You could use a VLOOKUP in the second part if you prefer.

Copy these 3 formulae down as far as you like. Using your example data, this is what you will see:

User Month Combined Inv_No Date Value
AW 5 AW05 123 010510 200
      125 020510 350

Just change the user in A2 or the month in B2 (marked yellow) and the display will automatically adjust. You can hide column C if you wish.

Hope this helps.

Pete

It just that between the

One can access all the objects in the db, The method your propose requires coding each object type individually. 


The CurrentProject and CurrentData objects, along with their various collections of AccessObjects, are properties of an Access Application object, not a DAO Database object.  So the only way you could use those objects with some non-current database is to create another instance of Access and open the database in that instance, then use the objects and collections of that other Access Application.  That's way more overhead than it makes any sense to incur for this purpose, I'd say.

I'm not sure what you are objecting to about using DAO collections instead of those belonging to CurrentProject and CurrentData.  Given an opened database object named "db", you have all of these available to you:

    db.TableDefs                                          (collection of tables)
    db.QueryDefs                                         (collection of queries)
    db.Relations                                           (collection of relationships)
    db.Containers("Forms").Documents      (collection of all forms)
    db.Containers("Reports").Documents    (collection of all reports)
    db.Containers("Scripts").Documents      (collection of all non-embedded macros)
    db.Containers("Modules").Documents   (collection of all modules)

... as well as some others that aren't usually of much interest. There's also this one:

    db.Containers("Tables").Documents

which is a collection of all tables *and* stored queries in the database.

If all you want is the names, and you don't want to Dim a separate variable of the appropriate object type for each collection -- TableDef for the TableDefs collection, QueryDef for QueryDefs, Relation for Relations, Document for the Documents collections -- then just Dim a variable of type Object and use that for looping through all of them, since all of these objects have a Name property.
