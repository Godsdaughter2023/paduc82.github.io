eriq vanbibber say:

Byron2017,

Perhaps telling you what your options are for sending and receiving email may help.

There are 4 object types that can be created and that can have an email address assigned:

1. Mailbox User - this is an object that can receive mail and is stored with the object.

2. Mail User - this is a user account, which means it can be used for logon to O365 services, that has an email address, but also has a forwarding address.  Mail directed at this object will forward to another.

3. Mail Contact - this is a Contact object - cannot be used for logons - that has an email address and a forwarding address.  Mail directed at this object will forward to another.

4. Distribution List - this is a mail-enabled group and will have an email address.  Mail directed at this object will be distributed to the members, based on member type (see 1-3 above).

Of the above objects, a user can be allowed to send "AS" one of the other object types, provided that an admin gives a user the SendAs permission to it.  this allows a user to send email out with the reply-address being a value different than their mailbox.  for example, one may want to send email out as a generic name like sales@ company.com.  The SendAs permission allows this without having to actually logon to a 'sales' mailbox.

Here's an article that provides info on how to set SendAs on MailUser and MailContact objects:

https://support.microsoft.com/en-us/help/2905758/how-to-set-up-send-as-rights-to-a-non-mailbox-object-in-office-365

Distribution lists can also be given the SendAs permission, but they must be a security group.  You would edit the 'Group Delegation' properties of a given group to do this.

It is then also possible to provide SendAs on a mailbox as well.

In Office 365 "User Mailboxes" consume a license.  however, a shared mailbox does not (unless eDiscovery or Legal Hold features are needed) - but check the terms-of-service about them as there are some rules (you're not supposed to use them for terminated users for example).

you could have your alternate email addresses setup on shared mailboxes, and then inbound mail for those email addresses will be held in the related mailbox.

however, you could also use the features mentioned above to send and receive on an address, but the delivery will be determined on the object type.  you could have several mail-contacts that forward to a hidden mailbox internally, or externally.  Same with a group.  You could also have several email addresses on a single object.

as far as the limits, please refer to this page:

https://technet.microsoft.com/en-us/library/exchange-online-limits.aspx

Hope this helps you.
