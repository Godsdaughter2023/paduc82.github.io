How do i sync subforms with the main form? - Microsoft Community

addictedtolivin say:

How do i sync subforms with the main form?

So i have a main form with a subform in it. I would like to be able to sync the subform in a way that every time i open a new form it comes up epmty and every time i open an existing form with data in it the subform that i previously filled in.
Please help

ken sheridan say: The subform control's LinkMasterFields property is set to the name of the primary key of the parent form's recordset, and the LinkChildFields property is set to the name of the corresponding foreign key in the subform's recordset.  Where the key is a composite one of two or more columns the names of each are entered as the properties, separated by a semi-colon.

addictedtolivin say: Hey,
I linked master field with the child field  (both the ID fields in given form/subform) and i saved the the main form with the subform.
When i opened the main form again to see if it worked the was no data in the subform 

ken sheridan say:

If the properties were not set when you originally entered data in the subform, then the value of the primary key from the parent form's recordset will not have been entered into the corresponding foreign key column (field) in the subform's recordset, so the link will not have been made.

 

Note that the LinkChildFields is not the name of the subform's primary key, but a foreign key column which references the primary key of the parent form's table.  Say the subform is based on the following table:

 

Orders

....OrderID (PK)

....CustomerID (FK)

....OrderDate

 

and the subform is based on the following table:

 

OrderDetails

....OrderDetailID (PK)

....OrderID (FK)

....ProductID  (FK)

....Quantity

....UnitPrice

 

Then the LinkMasterFilds and LinkChildFields properties of the subform control would both be OrderID.  This mirrors the relationship between the tables of course, which would have been set up in the relationships window when the database was first designed.

dan elswick say:

Vlookup Errors

I have a small table (200 rows/3 columns) created from a pivot table and then used copy paste as values.  I entered data, on the same sheet, and created a range name.  I am using the formula =vlookup(b?,ar_rc,2,false) - where ?  is the table row number,  to identify each matching row.  Some match, most do not returning #N/A.

However, if I find the matching number in the table, pressing F2 Enter, returns the vlookup answer telling me this is not a formatting or decimal place issue. This is very frustrating and did not happen in previous versions of Excel, only 2007.  Opening and closing the file has no effect, sorting the data to match the table and data entry has no effect.  Calculations are set to auto. 

Is there a setting I am missing in Excel 2007's setup?

jllatham say:

Are the values in the lookup value column B? numeric, and those in first column of ar_rc are also numeric?  I suspect they are based on all that you said ("  not a formatting or decimal place issue").  But I think it may be.

Try testing a couple using a worksheet formula, comparing a B? value against what you believe should be the matching cell in your table, like:

=B2 = R15 or
=B2='Table Sheet Name'!R15

and see if you get True or False.

If you get TRUE but the VLOOKUP() is still returning #N/A, then there is a problem.

It might even be a problem with the RefersTo definition for the table name ar_rc.  Try changing the formula to an absolute reference and see how they hold up,  Let's say your table is on a sheet named "Table Sheet" and occupies A2:C200 then change an ar_rc reference where you're getting a #N/A that you believe should not be there to:

=VLOOKUP($B?,'Table Sheet'!$A$2:$C$200,2,False)

and see what happens (and as with your example $B? would be an actual row number like $B3).
