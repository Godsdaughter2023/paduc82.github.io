Public Function cmdFileDialog() As string

stefan blom say:

This option, as well as all options for nonprinting marks, is "global" in Word; that is, the options apply not to specific files but to the whole program.

Do the following: Make sure only one document is open. Go to File tab | Options | Display. In the "Always show these formatting marks on the screen" section, clear everything that you don't want shown and also clear "Show all formatting marks." Click OK. Restart Word to ensure that the settings are correctly written to the registry.

Note that "Show all..." corresponds to clicking the ¶ button on the Home tab; it toggles any option that isn't specifically selected in the "Always show these formatting marks..." section mentioned above.

See also http://word.mvps.org/faqs/formatting/NonPrintChars.htm.

The link Stefan provided is a good trouble shooting procedure.  Definitely do focus on the question of addins by using the process to start Office in "Safe Mode", which disables all addins.

The "old school" method of saving configuration changes is editing NORMAL.DOTM. 

Make sure you only have one Word window open (that includes closing Outlook, although it shouldn't have an impact).  Then Open your NORMAL.DOTM to edit it. Make the change to turn off the "Show/Hide" option and then add and delete a single character (to force Office to save all changes) and close the file, making sure to save those changes.

Another option is, "... if you can't join 'em, beat 'em (with a baseball bat)! " Create a macro to make the setting you wish.  This tip is written for Word 2003, but should work in 2013 too:

Returning Word to Default Settings

http://word.tips.net/T000642_Returning_Word_to_Default_Settings.html

Summary: Want to set everything back to a pristine state in Word? Doing so is more involved than you may believe.

The specific setting you want is

            .ShowHiddenText = False

You can eliminate all of the other settings if you wish.

You should also use one or more special names for this macro, ie

  • "AutoExec"
  • or "AutoNew" and "AutoOpen"

This article describes the effects of these special names

Auto... macros

stefan blom say:

Following Rohn007's idea, you need two auto macros as follows:

Sub AutoOpen()
ActiveWindow.View.ShowAll = False
ActiveWindow.View.ShowHiddenText = False

End Sub

Sub AutoNew()
ActiveWindow.View.ShowAll = False
ActiveWindow.View.ShowHiddenText = False
End Sub

Add the macros to the Normal template (see http://www.gmayor.com/installing_macro.htm). This will ensure that the display of hidden text is turned off each time a document is created or opened.

However, this is really only a workaround. Troubleshooting as discussed in the Knowledge Base article at http://support.microsoft.com/kb/921541/en-us should help you find the underlying issue.
