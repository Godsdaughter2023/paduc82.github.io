We appreciate your detailed reply, Nick. You might be experiencing this issue because you have enabled two-factor authentication for your iCloud account. Enabling two-factor authentication is a good thing because it provides better security. However, enabling it means that you need to generate an app password to use in Outlook.

To connect your iCloud account to Outlook, we need to generate this special app-specific password. This is needed because you can no longer use your current iCloud password to configure Outlook after enabling two-factor authentication. To generate an app password, kindly follow the steps below:

  1. On the Password and Security page, you’ll now find a Generate an app-specific password section where you can click on the Generate an App-Specific Password link.
  2. Enter a name to label this app-specific password to help you remember where you used it. (For example, Outlook.)
  3. Click the Generate button.
  4. Copy the obtained password code.
  5. Go to Outlook and paste or type the obtained app password when being prompted for your password.
  6. Once you’ve updated your password, Outlook will be able to connect to iCloud again with two-factor authentication enabled for your account.

If you need further assistance, please do not hesitate to post a reply.

=IF(COUNT(PEval!D:D)=COUNTIF(PEval!D:D,53.1),53.1,"Different Values")

Any Empty cells would not be counted, and any text values entered would also be ignored since the COUNT function only counts cells with numerical values.

code chief say:

I solved this by:

1) Install MAGDLDR v2.

2) Install a MAGDLDR custom ROM from a previous version of the phone, i.e. Mango or NoDo..

3) Let Zune take back control of updates, all the way back to the official version.

4) If #3 doesn't work for any reason, then use the Windows Update CAB Sender to manually pick and choose the updates in order to get back to the current state.

This is the same process as the Windows Phone Support Tool (flashing back to an old image so Zune can finally update without errors) but it works. The support tool is currently out of date, it warns you not to use on post-Mango versions and refuses to connect (doesn't find device no matter what you do) if you try it anyway.

So until the support tool is fix the only way for sure to get updates working again is to use the unofficial tools to flash the ROM.

From the log messages I posted the problem is clearly flaky logic in the update package, e.g. throwing an error creating a directory when it already exists, but that should not be a problem.

All of these custom tools are found via (but not always programmed by) the XDA Developers web site. Many of the old download links have expired, which is a pain. We really need an official tool that people easily access and use.
