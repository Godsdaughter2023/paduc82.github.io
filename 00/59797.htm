shawn steele say:

I don't have the background of this change in front of me, however it'd be good to look at "Culture data shouldn't be considered stable." 


Culture data often changes for one reason or another.  Applications requiring stability should either provide an explicit, preferably standard, format, or use the Invariant locale, or use a binary format for interchange.


The suggestion to override your date format settings is one example of why locale data shouldn't be considered to be stable, the user can override those settings.


Another way locale/cultural data can change is with a custom culture, such as one created by the Locale Builder tool.  Custom locales aren't subject to the user default culture limitation, and they apply to the entire machine.   You have to be an administrator to install a custom locale.  Unlike user overrides, the application cannot ignore the user's custom locale.

Windows 7 64 bit comes in handy when you need to address at least 4 GBs or more of RAM. Windows 7 32 bit can utilize up to 3.2 GBs of RAM. Because the memory address space is much larger for 64 bit Windows, that means, you need twice as much memory than 32 bit Windows to accomplish some of the same task, but you are able to do so much more, you can have more applications open, do things like run an Antivirus scan in the background without it affecting your system performance. Windows 7 64 bit is more secure too, malicious code cannot easily infiltrate it, drivers are more reliable since they must be signed before they can work with 64 bit Windows 7.

As for compatibility, you will need 64 bit device drivers for any hardware devices you might have. Also, there is no 16 bit subsystem in Windows 7 64 bit, which means, your applications must be 32 bit only, not 16 bit installer or uninstallers.

Your batch file

should really look like so:

@echo off
xcopy C:\Users\Console\Sample.txt to C:\Windows\System32\Config\ /c /q

The rest is superfluous or makes no sense. If it does not work with your batch compiler then it is really a batch compiler issue, not a Windows problem. Check the batch compiler FAQs or see if you can find a forum for this tool. You should also ask yourself what benefit you derive from running the batch file in compiled form.
