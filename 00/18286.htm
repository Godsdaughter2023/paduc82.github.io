Hi Simon,

 

Thanks for posting on our forum.

According to your description, it is a normal behavior.

 

As you have synced the files to your computer, the files are still syncing between computer and SharePoint site.

When you delete the file on your computer, it will delete from the SharePoint site.

If other users also sync the site to their computer, as the file has been deleted form the site, it will also been deleted form their computer.

So the files appeared on the users’ recycle bin.

 

If you didn’t want to affect other users, I suggest you refer to the following steps to delete the file and it only deleted the file on your computer not affecting SharePoint site:

1. Right click the OneDrive icon.

2. Click Settings>select Account.

3. Select Unlink this PC under the account.

4. Then the SharePoint site’s folder will become a normally folder.

5. Delete the file.

Hi Rick,

 

Office 2016 is automatically set as a default app in Windows 10 once it’s installed on the computer. Here are the steps on how to uninstall Office 365 (2016) trial version and set Office 2010 applications as default apps on the computer:

 

Uninstall Office 365 (2016) using the Control Panel:

  1. In Control Panel, select Programs and click Programs and Features.
  2. Right-click on Microsoft Office 365 and choose Uninstall. Once you’ve successfully uninstalled the program, you need to restart the computer.

 

Set Office 2010 applications as default app on the computer:

  1. Go to Control Panel and select Programs. Afterwards, click Default Programs and select Set Default Programs.
  2. In the list of programs, select the application that you’d like to set as a default program.
  3. Click Set this program as default and click OK. Follow the steps again to set other Office 2010 applications.

rajithr say: Hi,
 
Sometimes this problem is caused by two Windows Registry entries that have become corrupt.  To fix the problem, you have to use Registry Editor to delete the corrupted Registry entries.
However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs.
For more information about how to back up and restore the registry, follow the steps as in the KB Article:
http://support.microsoft.com/kb/322756/
 
 
Check the following registry entry for favorites and modify it accordingly. Normally it should have only one entry on the right side, it should say Default and data should be "none".
To check the registry entry, follow the steps below:
a) Click on “Start”.
b) Click on Run and type “regedit” (Without the quotations) hit “Enter”.
c) You need to find Favorites under the following Registry keys:
d) Navigate to the following location:
 
• HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\Shell folders
• HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell folders
• HKEY_USERS\.default\Software\Microsoft\Windows\Current Version\Explorer\Shell folders
• HKEY_USERS\.default\Software\Microsoft\Windows\Current Version\Explorer\User Shell folders
 
e) Once you find Favorites under the first key “Right Click” on it and select “Modify”.
The “Edit String” dialog will open. The “Value Data” field should contain the correct location of your Favorites folder.
If it does not, enter the right location to restore your missing Favorites folder to Internet Explorer.
The location for XP is: “C:\Documents and Settings\User Name\Favorites”
