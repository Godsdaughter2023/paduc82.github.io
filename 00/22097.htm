Cracked it! Reliable legacy slide pasting. - Microsoft Community

john korchok say:

Cracked it! Reliable legacy slide pasting.

There are 5 details to tick off when creating a PowerPoint template that will allow pasting of legacy slides into a new template or theme:

  1. Identical layout name. This is fairly easy, name the new layout to exactly the same name as the old layout.
  2. Identical layout type. This is a little trickier, but hopefully rarer. Slide layouts have a type set in the XML. If you delete the default Microsoft layouts, then try to recreate them, the slide type will not match and PowerPoint will not paste to your layout. Your alternatives are to recreate the presentation from scratch, using a Microsoft default, or to paste in slides that use those layouts from a Microsoft default. So far, XML text editing doesn't fix this.
  3. Identical number of placeholders. Does the original have 5 placeholders? Then the receiving slide layout  must have the same number.
  4. Identical type of placeholders. If the original has Title, Content, Date, Footer and Slide Number placeholders, so does the receiving layout.
  5. Where there are multiple placeholders on a layout that have the same type, each placeholder will have an idx number. The receiving presentation  must have placeholder idx numbers that match the corresponding placeholders on the source presentation.

As long as you fulfill all 5, legacy paste will work without a hitch. So why didn't Microsoft tell us this 12 years ago? For more details, please see my post on this subject: Legacy Slides - Brandwares Best Practices

Every slide layout will have a type, if it's been created by PowerPoint, or it will have a userDrawn="1" tag in the second line of the XML if it's been created by the user. PowerPoint treats userDrawn layouts as a type. You can paste userDrawn layouts without a problem as long as they follow all 4 of the other criteria (layout name, placeholder types and numbers, and idx numbers).

I can confirm that shapes, pictures, text boxes or logos don't interfere, just placeholders and layout types.

bill manville say:

I suspect you have a mixture of time values (numbers formatted with a time format) and text values.
I would convert them all to time values first.
So, in M3: =IF(ISNUMBER(E3),E3,TIMEVALUE(SUBSTITUTE(E3,".",":"))
and similar in N3.

You can then compute the difference, in O3, by =N3-M3
However, when the difference is negative (ie N3 is earlier than M3) you probably want to add 1 (day):
=IF(N3<M3,1,0)+N3-M3

The ##### is indicative of a negative time value resulting from your formula.
Excel will not display negative times.

The ############# result is because Excel does not accept negative time(s). And 01:00-05:10:15 is negative. If you format the cell as general or number, you will see the negative value.

A way of displaying it in time is:

 

=IF(A6<TIME(F6,G6,H6);TEXT(TIME(F6,G6,H6)-A6;"-hh:mm:ss");A6-TIME(F6,G6,H6))

But this returns a string so you use it anymore to make further calculations on it.

With our latest update to Mail and Calendar (17.7167.4033x.0), Insiders may notice that the feature to pop-out messages to a new window has been temporarily removed from Mail.
