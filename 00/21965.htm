jeeped say:

Add the following routine to the ThisWorkbook sheet in the VBE.

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
    Cancel = SaveAsUI
End Sub

That effectively disables all Save As functionality. You would also work ThisWorkbook.Path into the Private Sub Workbook_Open() function so that if the network path is not ThisWorkbook.Path, then the workbook shuts down.

The following features cannot be saved in macro-free workbooks

VB Project

That is normal, and you took the proper path to save the workbook with the macros preserved in the copy so that their functionality is available.

One thing you might consider trying is Rob Bovey's Code Cleaner add-in.  You can get it from his site here: http://www.appspro.com/Utilities/Utilities.htm

Now it does require that you make a change to the VBA Macro Security settings to use it.  It is a change that potentially leaves your copy of Excel 'exposed' to the ravages of an Excel file with malicious code/macros in it.  But you can change that setting as needed to run the code cleaner, then after it's done it's job, change the setting back and uninstall the add-in (or just disable it).  Once you've done the code cleaning, then save your new .xlsm file and also perhaps try saving it as an Excel 97/2003 compatible .xls file so that further testing can be done with that format's version.

Note: when making the changes to the VB Macro security settings, after making the setting change you have to close Excel and then re-open it for those settings to take effect.  Those settings are Excel settings, not just applied to a single workbook.

What Code Cleaner does is to 'rewrite' all standard code modules in a workbook so that any 'garbage' that may be in them due to continued maintenance of a VBA project is removed.  It is safe to use, I have it installed here on my system and I can vouch that I had one file that exhibited some extreme problems (after having worked for quite some time without any error at all) was 'cured' by using it.

Basically, using Code Cleaner would take a problem with the internal structure of the macros in the workbook pretty much out of the picture as a cause of the problem.

This can be done, but it's a bit tricky. DoCmd.SetFocus Me.MySub.Form

DoCmd.SetFocus Me.MySub.Form!txtGoHere

joeu2004 say: Tony wrote:

For a simple linear regression why is the output of R squared (the correlation coefficient) from the same excel data set varies depending on whether you get it using the Trendline Function (and select to have R squared displayed) or you use the Data Analysis Statistical Tools Regression function.  I Get the same X coefficient but the R squared is markedly different (.995 v's .976)?  Do they use different functions to do the calculation?


I cannot answer your question dispositively, but I suspect it is related to http://support.microsoft.com/kb/829249, which Andre Melenk noted in a discussion at http://answers.microsoft.com/en-us/office/forum/office_2010-excel/help-with-trendlines/5125db26-d6ec-43dd-ab0b-d6bf6873798b.

It appears that you set Set Intercept=0 in Trendline options and the equivalent for Data Analysis.  KB 829249 states that in that case, "the R-squared value is always incorrect in any version of Excel".

Since the KB was written before XL2007 and XL2010, it is questionable whether it truly applies.  But I note that the disparity that you observe with XL2007 is similar to what I observe with XL2003.

In any case, the remedy might be simple:  do not set the Set Intercept=0 option or the equivalent for Data Analysis.

When Set Intercept=0 is not set in Trendline options and the equivalent is not set for Data Analysis, I get similar results for R2, y-intercept and x-coefficient.  In fact, I get similar results with the INTERCEPT, SLOPE and LINEST functions.

Note that I say "similar" results.  They are the same to at least 13 significant digits; to 14 significant digits in some cases.

But the fact that they are not identical to 15 significant digits suggests to me that, yes, the implementations are different in some small way.
