I came here looking for the same answer as you, how to have a machine default to Private network. I still haven't found that answer :)

But! I do have a much simpler way to change to a Private network.

Run a Powershell prompt as Administrator. Execute this:

Get-NetConnectionProfile | Set-NetConnectionProfile -NetworkCategory Private

That will set all the networks on your machine to private. If you want to be pickier than that, use the Powershell Where-Object cmdlet to pick only the ones you want.

For all those who might have the same problem,

Nobody could help me but myself :),  I contacted windows support team, and they got remotely access to my laptop and re-installed windows  2 times, and still I was experiencing the problem with sleep mode (laptop could not go to standby in Idle) , so frustrating, isn't it?

I was lucky I could find some solutions by surfing the net.

There is a command, you should use in your command prompt, while it is running under administrative permissions.

the first command is :

powercfg -requests

this command will show you which programs are running and preventing your system to go to standby.

the other one is :

powercfg /energy

This command will run an observation on your system while it is going to go to standby, and at the end will produce a log file, showing the problems that might have occurred. The log file will be saved in "c:\windows\system32\" as "energy.html".  (before that you should have set your power option settings, so your laptop or system go to standby after one minute being Idle)

For me the problem was with a Driver "srvnet" that prevented my laptop to go to sleep mode, and there are a lot of people that are experiencing the same problem since several years ago.

To solve the problem, I closed and exited the home group in Network and sharing center, and updated my graphic card driver.

Some people may try this command as well, but it has the risk that your system goes to standby, even mid in transferring files, or maybe upgrading !

powercfg -requestsoverride driver srvnet system
