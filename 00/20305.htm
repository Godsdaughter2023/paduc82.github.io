Match function error in Excel - Microsoft Community

kevin_huang say:

Match function error in Excel

If i use function Match(1001,(1000,1001,1002),0), i could get the correct answer 2,

but if i use Match( 1001/10^6*10^6,(1000,1001,1002),0)), It will response an error (#N/A)

 

ps. Match(1001*10^6/10^6, (1000,1001,1002)) , It still work fine. so it's rounding error??

joeu2004 say: Kevin wrote:
If i use function Match(1001,(1000,1001,1002),0), i could get the correct answer 2, but if i use Match( 1001/10^6*10^6,(1000,1001,1002),0)), It will response an error (#N/A)

ps. Match(1001*10^6/10^6, (1000,1001,1002)) , It still work fine. so it's rounding error??

Correct.  The problem is:  most numbers with decimal fractions (e.g. 1001/10^6) cannot be represented exactly by Excel (and most applications).  When you subsequently multiply even by the same divisor, you might propagate that inaccuracy.

For example, 1001/10^6 is exactly 0.00100099999999999,993670340980855826273909769952297210693359375.  When you multiply that by 10^6, the result is exactly 1000.99999999999,98863131622783839702606201171875.

Note that you will not see either inaccuracy even if you format the intermediate results to 17 decimal places.  The reason is:  Excel will format only 15 significant digits -- the digits to the left of the comma in my representation.

For this reason, it is prudent to round computations to the degree of precision that you expect.  For example, ROUND(1001/1E6*1E6,6) results in exactly 1001.
