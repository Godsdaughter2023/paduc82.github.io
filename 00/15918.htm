Mailing Labels - layout order (vertical instead of horizontal) - Microsoft Community

paultukey say:

Mailing Labels - layout order (vertical instead of horizontal) - A simple, universal solution

Sometimes I want to print 10x3 Avery labels that are alphabetized down the page, not across -- especially since Avery labels paper these days is neatly perforated by columns.   In fact, when you tear off a column, the labels on it stick out by 1/8" to make it trivial to peal them off.  Very clever!   I use MS Word mail-merge, with Excel as a data source, which I presume is the most common scenario.

I found a number of discussion threads, some offering ingenious solutions, but they all seemed to me to be unnecessarily complicated and tricky to carry out -- e.g. using a VBA macro, or restructuring an actual MS Word table by hand, or diddling a device driver for a particular printer.

So I worked out a simpler solution.   One contributor to a discussion suggested an approach like this but didn't flesh it out.

The concept is simple:   Alphabetize the list in Excel, then rearrange the rows in the spreadsheet so that even though mail-merge prints them across the page, the final pages of labels end up being alphabetized down the columns. 

The execution is pretty straightforward:

1.  Alphabetize your spreadsheet,

2.  Add an extra column, and put this formula into every cell (this is the magic bullet):

="["&(10000+(1+INT( (ROW()+1-3) /(10*3))))&","&(10000+(3*((1+MOD(ROW()+1-3,10))-1)+(1+INT((ROW()+1-3)/10))))&"]"

A typical entry will look like this: [10003,10025], which means the 25th label on the 3rd page (printed across, but alphabetized down).

3.  Add another extra column, and copy-and-paste VALUES from the first extra column to the 2nd one.   You'll probably clone the column header, too, so alter it slightly (perhaps by putting "v" at the end for "values") else the next step is ambiguous.

4.  Sort the spreadsheet again,this time on the 2nd new column.    And you're done.

5.  Print the labels as usual.  They are magically laid out just way you want them!

NOTE:  This solution easily adapts to other label layouts -- e.g. 11x3 or 10x2 -- or even 4x2 (for name badges, say).   The numbers 10 and 3 appear in the formula (several times).   10 is simply the number of rows and 3 is the number of columns in the labels pages. 

NOTE ALSO: This solution should work equally well for all versions of Word and Excel, and for other MS-Office-like suites (clones), such as OpenOffice (which I've tested) and Google's G-Suite Apps (which I haven't).   Most likely it is the universal solution.   Comments welcome on that!

P.S. The 10000 might need more zeros if you're doing more than 9,999 pages of labels  :-)

Enjoy!
