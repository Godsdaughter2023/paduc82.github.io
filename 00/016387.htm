    I'm helping a friend who has two PCs, both running Windows 10 and both using Office 365 and Outlook 2016.  His email uses POP on both PCs and his POP is set to leave messages on the server. As a result, he sees any incoming mail on both PCs, but if he replies on one PC, he does not see the reply on the other PC. He also has extensive Personal Mail folders in which he stores old mail for future reference.  In addition, he has the same email account set up on Outlook.com, but there, his Personal Mail folders do not show up, just the default ones such as inbox, sent, draft, etc. This Outlook.com account is actually his mail server, since he uses Mail from Bell Canada, which uses Outlook.com as its webmail system.

    In order for him to see all of his mail on both PCs (including replies), I have suggested that he switch from POP to IMAP, but I understand that the only way he can do this is to remove the POP account and then re-add it as IMAP. But if he removes the POP account, I believe that will delete all of his Personal folders, which are all associated with that account.  And when he re-adds the account as IMAP, it will show only the folders that are on the (Outlook.com) server, which does not have the Personal mail folders.

    Finally, two questions:

    1. Am I correct in assuming that removing the POP account will delete the Personal Mail folders?

    2.  If that is correct, is there some other way for him to get complete syncing between his two computers without losing his Personal folders?

    karl timmermans [outlook mvp 2012-2018] say:

    Finally, two questions:

    1. Am I correct in assuming that removing the POP account will delete the Personal Mail folders?

    2.  If that is correct, is there some other way for him to get complete syncing between his two computers without losing his Personal folders?

    #1 - Removing the POP account will not remove the underlying PST file that is associated with the accountt. That folder tree will remain in the Outlook Nav Pane unless you explicitly remove it from the profile. If this is the only email account configured in the profile, the PST file is also the "default data file" for the profile which is significant for reasons as noted in the following points

    #2 - All you need to do is add the same email account using IMAP. When that's done, copy the email folders to the IMAP folder tree. Would strongly suggest that you first set Outlook to "work offline" so that the sync process doesn't kick in until the copy to the IMAP folder tree is complete. When done - set Outlook to "work online" and let it sync.

    #3 - In terms of the contact/calendar data - those folders are stored in the "default data file" for the profile so strongly suggest you leave them in the PST file. If you add the IMAP account with the PST file is still configured in the profile, there will be no contact/calendar folders in the IMAP folder tree. When only an IMAP account is configured in the profile, the contact/calendar folders will have the added description of "This Computer Only" but maintaing data in folders so marked, opens things up for that data to be lost should the IMAP account ever be deleted from the profile. Contact/calendar data doesn't sync either  POP3 or IMAP .

    Some articles that may be of interest

     

    How to change an email account from/to POP/Imap
    http://www.contactgenie.info/how-to-change-outlook-pop-imap-email-account-type

    How to Protect Outlook '2013/'2016 Imap Contact/Calendar Data
    http://www.contactgenie.info/how-to-protect-outlook-this-computer-only-data

     

    karl timmermans [outlook mvp 2012-2018] say:

    #1 - Will things work out OK if we just add an Outlook.com account to his existing Outlook and then make that Outlook.com account his default data file?

    #2 - If we do that and proceed with your step f) above to get his calendar syncing, will he still be able to use his Sympatico account as the default for sending and receiving emails?

    #3 - will they automatically be created in his Outlook.com account, since it is the default data file, or will they exist in his Sympatico IMAP account?

    #1 - Absolutely but just a precaution - in this scenario would suggest

    (a) making a new profile by first making a "COPY" of his existing one

    (b) adding the Outlook.com account to this newly copied profile

    (c) set the new profile as the default for the next time Outlook opens

    - only reason for doing that is <IF> in the effect something doesn't work as expected, you can simply revert to the original profile by making it the default again. Doubt this will be required but always better to be safe.

    #2 - Keep saying this. To stress. the primary significance of the default data file has absolutely nothing to do with <email> data nor does it matter what your <default email account> is - it is purely the container for the "default contact/calendar" folders (which may also be the container for email data depending on the account)

    #3 - Nothing gets automatically created in Outlook.com unless you explicitly move things there and aside from contact/calendar data, don't know why you would want to in this scenario. The email folders will be and remain in the Sympatico IMAP data store.
