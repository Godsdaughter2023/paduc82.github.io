andrew simpson uk say: Hello Deya,

I didn't spot this question first time round, but do have the answer.

Assuming you're using "Oracle Project Management" (OPM) to upload from Project 2007, the timephased data sent up to Oracle isn't as granular as it is in MS Project. Rather than timephasing by day (or even hour), the data sent to OPM is timephased by the financial periods as they are defined in Oracle. The first step in the upload process is to download the start and end dates of all financial periods covered by the project.

So the remaining work for the November financial period will be profiled in a flat line across the whole period in Oracle, while MS Project will show it nicely profiled by day as you have manually entered it.

When you download the project from Oracle, it's then only giving you the information that it has been previously sent, so the detailed timephased information is lost.

On the implementation OPM I worked on previously, we advised users not to download schedules from OPM, for this and various other reasons:
  1. You lose all custom views you've defined
  2. You lose custom formatted you've applied within the view
  3. You lose the values of any custom fields you may have entered
  4. The dates in the downloaded version of the project were inaccurate
Users were therefore trained in how to keep their master copy of the file in a state that allowed it to be uploaded to Oracle.

I've got a lot of experience with this tool, so don't hesitate to get in touch if you need further assistance. I've developed custom integration with OPM that ironed out several bugs in the integration, added additional validation/functionality and also enabled integration with Project 2010 rather than 2007, which I've found to be a more stable product, less inclined to suffer from corruption.

Hope that helps,
Andrew
