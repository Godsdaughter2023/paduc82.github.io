rob koch say:

The registry entries you have listed are all simply usage tracks, not malware, so there's really no reason to worry about them unless you're paranoid about your privacy.

 

http://www.safer-networking.org/faq/usage-tracks/

 

If you've been using Spybot Search and Destroy for a while and don't know what these are, then you likely haven't really learned how to use it effectively.

 

The best method when using security applications is to get to know your primary antimalware application very well and then once you understand that fully, only then bring one more on demand application into the mix as a backup or alternative check against your main protection.  Until you know both of these first two applications well enough to teach someone else how to use them and answer any and all questions they may have about them, you shouldn't be using a third.

 

I personally use only MSE and know it inside out, I've never needed an alternate application because I instead concentrate on the more important issues of keeping both Windows and the very few third-party general applications I use completely up to date, which combined with Windows 7 Pro x64 version and Internet Explorer 9 leaves very few vulnerabilities for malware to exploit.

 

The moral of the story is that more security applications do not result in higher security, but rather less applications in general including security that you know very well and maintain fully with updates from their developers leads to the best security posture.  The more applications you have installed, the more there is to understand and maintain and so the higher the chance that vulnerabilities will exist and so allow malware to enter.

 

The above is proactive security while antimalware are reactive and are always playing catch-up so they serve only as a last resort when all other proactive methods have failed.

If you can't print to OneNote, drag-and-drop elements onto a page, or embed files, then how in the heck are you supposed to even get content into it? If you can't open embedded files, what's the point of using OneNote to collect resources used in a project? If you can't open local files, how are you supposed to manage notebooks that are gigabytes in size? Or notebooks that, as some have pointed out, contain content that can't be uploaded to the cloud (for example, HIPAA-protected information?) Why can't encrypted notebooks be opened?

These features and others represent the defining power of OneNote. This app feels like a port of the iPad app! A glorified viewer, that can't even open local files! I just don't understand Microsoft. scottgem say:
Every field in an Access database has a datatype. The datatype governes the type and size of data that can be entered into the field. Thsi setting is set at the table level by going into the table in Design Mode. You will see a column that governs the datatype.

It sounds like the field is set for Short text which is limited to 255 characters. You would need to select Long Text which allows 64K characters.

Once you correct the data type on the table level, you can edit the form to make the text larger and add scroll bars if needed.
