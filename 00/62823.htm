It's not entirely clear what your problem is. Are you trying to delete messages in the Windows Live Mail store folder in Explorer? That's not a good idea, because the program's message database index will not be aware of the action. It may notice that a file is missing and attempt to recover it. All operations on stored messages should be conducted within the program itself, so that the action can be recorded by the database index.

With a few exceptions, only messages that you move or copy from an account folder will appear in Storage folders. When you delete a message from your inbox or other folder - using the Delete button on the ribbon, the Delete option on the right-click context menu, the Delete key on your keyboard or the keyboard shortcut Ctrl-D - the message is simply moved to the Deleted items folder whence it can be recovered if necessary. It's really a recycle bin. However, if you delete a message from the Deleted items folder, it's gone permanently. You should see a warning before being allowed to complete the action.

If you really want to delete a message permanently, bypassing the Deleted items folder, select it in the message list and press Shift-Delete on the keyboard. You can use these common Windows shortcuts to select multiple files for deletion:

  • Click on an item to select it, hold Ctrl down and click on another to select both;
  • Click on an item to select it, hold Shift down and click on another to select both and all in between;
  • Click on an item to select it, hold Ctrl down and press A to select All items.
