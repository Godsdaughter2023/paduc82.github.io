There is NO option to create groups but there is a work around, using the Mail and People app together.

To create the group:  In Mail, create New Mail, open the cc/bcc field and insert all the addresses for the particular group.

When completed (with the mouse pointer at the end), right click and select 'select all'.  For some reason you can't right click and select copy but you can copy with ctrl+c. Now, open the People app, create new contact, give the group a name and paste the copied names into an email field, I use the 'Other' option and click the SAVE button.

To use the group: Mail app - New mail, click bcc field or whatever, open the People app, snap it left or right, find your group name, click on it, on the right hand side you will see all your group addresses, right click and copy, go back to mail, right click and paste.

I don't know the limitations of this (and there probably is).  None of my groups have more than 10 names in them.

•The website's administrator has to give you permission to view the page or the web server doesn't accept public webpage requests. If this is a website that you should have access to, contact the website administrator.

UPDATE :

So, after trying some new things (nothing worked unfortunately), I think I know where the problem might be.

After reusing the recovery drive and repairing the bcd with bootrec.exe /rebuildbcd the prompt suggested to add C:\Windows to boot list but then said : The media is write protected.

So I checked the state of the disk again but it wasn't. I came to the conclusion that the UEFI boot was set to the X:\ drive. Thing is, I have no idea how to change this setting, would anyone have an idea ?

FINAL UPDATE :

So the solution was finally quite dumb.

Like I thought, somehow the boot list was looked. I thought it was in read only mode.

While I looked for a way to reset the UEFI, I remembered that in the UEFI option there was an option "Enable Boot Configuration Lock". I always thought it was only to lock the order of my current boot sequence but it actually prevent me from  reinstalling any OS on the computer.

So in the end, for other Surface Laptop owners : in order to install a new system, you have to set this option on Off.

Hope it can help someone else.

To troubleshoot your Skype to Skype calling issue, please follow the instructions below:

  1. Uninstall Skype. For the complete instructions, please visit this FAQ link. Once done, do not run the app.
  1. Press the Windows  and R keys on your keyboard at the same time.
  2. Type "%appdata%\skype" into the Run window and click OK.
  3. Find and delete the shared.xml file.
  4. Open the folder that has your Skype name.
  5. Locate and delete config or config.xml file.
  6. Go back to the appdata folder and rename Skype e.g. Skype_old
  7. Launch Skype and select any contact. Type and send /msnp24 (don't worry your contact won't receive it) then restart Skype and check if the steps fix the issue.

Please let us know the result.
