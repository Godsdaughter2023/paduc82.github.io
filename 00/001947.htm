andreas killer say:
Please don't create multiple posts related to the same issue.
http://answers.microsoft.com/en-us/office/forum/office_2013_release-customize/vba-sumif-with-multiple-criteria/11d46c0f-bc9d-417a-b23e-aa32af6491a1

The SUMIFS works in VBA the same way as in the sheet, means if you don't specify the same number of rows, you get an error.

When you use END(XLDOWN) on different columns, it is very easy to run into trouble. A better way is to get the whole data range and use INTERSECT to get the columns.

The benefit is that you can improve the code (later) and use a named range for a heading instead of hard code the column in the code.

Andreas.


Sub Totals()
  Dim AppDB As Worksheet
  Dim DataRange As Range
  Dim AppDBSumNRange As Range
  Dim AppDBSumPRange As Range
  Dim AppDBSumKRange As Range
  Dim AppDBSumMgRange As Range
  Dim AppDBYearRange As Range
  Dim AppDBTargetAreaRange As Range
  Dim AppDBTypeRange As Range

  Dim strYear As String
  strYear = InputBox( _
    "Nutrients Applied and Active Ingredient Totals. Please enter year.", "Totals")
  'Aborted?
  If strYear = "" Then Exit Sub

  Set AppDB = ThisWorkbook.Sheets("Application Database")
  With AppDB
    'Span a range from top-left to bottom-right cell of the data
    Set DataRange = .Range("A4", .Cells.SpecialCells(xlCellTypeLastCell))

    'Get the columns inside the data
    Set AppDBSumNRange = Intersect(DataRange, .Columns("N"))
    Set AppDBSumPRange = Intersect(DataRange, .Columns("O"))
    Set AppDBSumKRange = Intersect(DataRange, .Columns("P"))
    Set AppDBSumMgRange = Intersect(DataRange, .Columns("Q"))
    Set AppDBYearRange = Intersect(DataRange, .Columns("AA"))
    Set AppDBTargetAreaRange = Intersect(DataRange, .Columns("D"))
    Set AppDBTypeRange = Intersect(DataRange, .Columns("F"))
  End With

  With Sheets("Totals")
    'Total Nutrients Applied
    'N & Greens
    .Range("C8") = Application.WorksheetFunction.SumIfs(AppDBSumNRange, _
      AppDBYearRange, strYear, AppDBTargetAreaRange, "Greens", AppDBTypeRange, "Nutrient")

    'N & Tees
    .Range("C9") = Application.WorksheetFunction.SumIfs(AppDBSumNRange, _
      AppDBYearRange, strYear, AppDBTargetAreaRange, "Tees", AppDBTypeRange, "Nutrient")
  End With
End Sub

Comes down to the type of email account that was involved but basically, recovery can be summed up as follows

#1 - If it was a POP3 account, would just need to find the original PST file associated with the account and associate it with the newly created account. #2 - For IMAP and Exchange, email folders are stored on the server and sync'd with the local (OST) data file

karl timmermans [outlook mvp 2012-2018] say:
Don't need the OST file which gets physcially deleted by Outlook when an IMAP account gets deleted from the Outlook profile. Outlook creates a new one when the account is configured again

What you will definitely have lost is any data in any folder with the label "This Computer Only" (contacts/calendar). If the email folders you created were not subscribed and as such not sync'd with the server at least once - then unfortunately all that is lost as well.
