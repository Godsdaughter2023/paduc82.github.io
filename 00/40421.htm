Network ports used by Xbox Live
Port forwarding

If you have a firewall or network hardware, such as a router, you might need to make a configuration change in order for your Xbox One console to communicate with Xbox Live. This configuration change is sometimes called “opening ports” or "port forwarding."

Xbox Live requires the following ports to be open:

  • Port 88 (UDP)
  • Port 3074 (UDP and TCP)
  • Port 53 (UDP and TCP)
  • Port 80 (TCP)
  • Port 500 (UDP)
  • UDP Port 3544 (UDP)
  • UDP Port 4500 (UDP)

If you're connected to a network through your workplace or school, ask the network administrator to open these ports.

How to set up port forwarding

For information about how to configure compatible network hardware, go to the Networking Hardware Information forums.

You may also find information about how to open ports on your router at the Port Forward website. If your router is not listed on the Port Forward website, contact your Internet service provider or router manufacturer for help with opening network ports.

NETGEAR WGR614

Login Information

  • Gateway - 192.168.0.1 (v1-v5) or 192.168.1.1 (v6-v10)
  • Username - admin
  • Password - password

Port Forwarding

  1. Login to router by entering http://routerlogin.net  in to the address bar or by entering the default gateway IP address.
  2. The router will then prompt you to enter the user name and password.the default user name and password are: "admin", "password".
  3. From the menu on the left select Port Forwarding/Port Triggering.
  4. Select the Port Forwarding button at the top of the main screen.
  5. Click the Add Custom Service button at the bottom.
  6. In the following Screen enter a service name that indicates something to do with Xbox Live (XBL).  Set the service type to both TCP and UDP.  Set both the starting port and ending port to 3074. Then enter the IP address that the Xbox console you wish to port forward has.
  7. Click Apply.

 

Disable Port Forwarding

  1. Login to router by entering http://routerlogin.net  in to the address bar or by entering the default gateway IP address.
  2. The router will then prompt you to enter the user name and password.the default user name and password are: "admin", "password".
  3. From the menu on the left select Port Forwarding/Port Triggering.
  4. Select the Port Forwarding button at the top of the main screen.
  5. Select the radio button for any rules needed to be removed.
  6. Select Delete Service.
  7. Select apply.

jerree say: Your problem is most likely your video card memory overlaying on top of protected OS memory space. Yep, it shouldn't happen, but it apparently has. Your video driver is conflicting with the OS kernel memory allocation and causing a serious memory management issue, due most likely to the video driver software extending its boundary (don't expect a fix anytime soon!). Please note: this is note a Microsoft issue! If the BSOD is random (crash address changes each time) slow down your RAM (change T1 to T2) or replace your RAM.

Next you want to configure your system for a single display only with no extended display, multiple display, virtual display, etc. This should minimize the amount of system memory your video card uses and reduce the possibility of a memory conflict with the OS (this should be enforced in hardware as well).

Now disconnect any additional video connectors, if you have more than one connected to your system. You can do this one at a time but to check if its video related do everything noted above together, all at once.

If you have to, cut back on the resolution of your display, if your using very high resolution, as a final option. But most likely configuring for a basic display should do the trick (resolution shouldn't be a issue, except in very rare cases). That's it!

First and foremost, here is my setup.  Please note this has been gone over on many other forums as well, but I want to make sure everyone knows where I'm starting at:

Surface Pro 3 Dock mDP --> DP In on Monitor 1 (With DP 1.2 Enabled)

Monitor 1 DP Out --> mDP In on Monitor 2 (With DP 1.2 Disabled)

Per Dell's instructions for Daisy-chain setup, you should have the last monitor configured with DP 1.2 Disabled to act as the terminating monitor.

Initially this wasn't working, but then I started looking around at monitor settings and noticed that both monitors were automatically detected at 59Hz.  I figured that 60Hz would be the default so I switch both of them to it.  Once I did that, the daisy-chain worked as expected.
