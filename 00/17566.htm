How can I auto populate a table with dates increasing by set intervals - Microsoft Community

john carslake say:

How can I auto populate a table with dates increasing by set intervals such as monthly or four weekly?

I have set up a REPORT to show a bank account. I enter payments and receipts as they arise on a FORM. I want to be able to generate recurring items (such as standing orders) automatically on a TABLE or QUERY as each one becomes due. This will be combined with the data generated from the FORM above.
ken sheridan say: These sort of scheduling operations are not uncommon in various contexts.  I've had occasion to do them for programming regular maintenance operations to engineering structures for instance.  One way to insert a specified number of rows into a table with a date value requires some code which loops through the number of transactions needed and inserts a row into the table, incrementing the date at each iteration of the loop.  This can be wrapped in a little function along the following lines:

Public Sub InsertMonthlyDebits(lngTransactionTypeID As Long, _
                            lngCreditorID As Long, _
                            intNumberOfDebits As Integer, _
                            curAmount As Currency, _
                            dtmStartDate As Date)

    Dim strSQL As String
    Dim n As Integer
    
    For n = 0 To intNumberOfDebits - 1
        strSQL = "INSERT INTO Transactions" & _
            "(TransactionTypeID, CreditorID, TransactionDate, Credit, Debit, Amount) " & _
            "VALUES(" & lngTransactionTypeID & "," & lngCreditorID & ",#" & _
            Format(DateAdd("m", n, dtmStartDate), "yyyy-mm-dd") & _
            "#,0," & curAmount & ")"
    
        CurrentDb.Execute strSQL, dbFailOnError
    Next n
    
End Sub

This assumes a table Transactions with columns TransactionTypeID, CreditorID, TransactionDate, Credit and  Debit.  The table would also have a DebitorID column, but that will be left Null in this case as the transactions are debits from the account payable to a creditor.  TransactionTypeID would be a foreign key referencing the primary key of a TransactionTypes table, CreditorID would be a foreign key referencing the primary key of a Creditors table and Credit and Debit are columns of currency data type.  In this case a zero value is inserted into the former as the amount is being debited from the account.

So to insert rows for 24 standing orders of 10.50 GBP payable on the first of each month, starting next month, to creditor 42 where the key for 'standing order' is 5 would mean calling the procedure like this:

InsertMonthlyDebits 5, 42, 24,10.5, #2012-02-01#

In an application the values would normally be variables of course rather than hard coded, obtained from combo and text boxes on a dialogue form for instance.

I don't think Tom need be too incredulous about using Access in this context.  I've maintained records of all my personal accounts in Access for many years.  This is purely for my own use of course, I'm not running a bank!  I have it set up so that the regular monthly debits from my current accounts (checking accounts in US-speak) are assigned at the beginning of each month.  The actual payments made from the accounts occur on or about the dates assigned of course.

Note that these regular transactions are represented by rows in the same table as other credits and debits.  Your post suggests that you might be considering putting them in a separate table.  That would not be a good idea.  A query on which a report is based can restrict the data for the report in whatever way you wish.  You might want to show only transactions to date for instance, excluding those such as standing orders scheduled for future payment; or you might want to show only regular or non-regular transactions.  Once you have all the required data in the table you can manipulate it however you wish.

scottgem say:
Public Function DisplayHours(dteStart As Date, dteEnd As Date) As String

Dim lngMin As Long, lngHrs As Long

lngMin = DateDiff("n", dteStart, dteEnd)

lngHrs = lngMin / 60

DisplayHours = lngHrs & ":" & Format(lngMin Mod lngHrs, "00")

End Function

Where you want to display the hours and minutes over 24 hours use that function.

Hi, updated. Go to Settings > Update & security > Phone update > Check for updates.

If no one can hear you on the other end while on the call may likely be due to some microphone issues with your phone. Check these easy steps first:

  1. Check that your microphone is not muted, or if there is any background noise, move to a quieter area.
  2. Try again. Sometimes just redialing is enough, but your phone restart can also help.
  3. Check if the issue is happening to a specific phone number on outgoing or incoming calls - can the other person hear other callers?
  4. Make a call, or try to playback the voicemail on your phone. Use your phone in-call options, switch to the speaker mode to check if that resolves the problem (for more info, see the user guide of your phone).
  5. If the other end cannot hear you in the speaker mode, check that no any objects are blocking the internal microphone openings on your phone, such as the accessory case or shell.
  6. If the problem persists, connect a compatible wired headphone. Push and pull the audio plug gently with your fingers, and disconnect it. If this helps, check the audio jack connector for any dust or dirt.
