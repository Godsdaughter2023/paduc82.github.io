doug robbins - mvp office apps & services (word) say:

Do you have a C:\Users\[User Name]\AppData\Roaming\Microsoft\Templates folder?

If you do not see the AppData folder, which is normally hidden, in Windows Explorer, expand the Organize drop down and click on Folder and search options and go to the View tab of the Folder Options dialog and select the radio button for "Show hidden files, folders, or drives".  While the it is also a good idea to uncheck the box for "Hide extensions for known file types"

If you don't have the Templates folder in that location, you could create one.   Then, with Word not running, you could move anything from the C:\Users\[User Name]\Documents\Custom Office Templates folder into the C:\Users\[User Name]\AppData\Roaming\Microsoft\Templates folder and then delete the Custom Office Templates folder.

Then in Word, go to File>Options>Advanced>General>File Locations and click on the User Templates item and make sure that it is pointing to the C:\Users\[User Name]\AppData\Roaming\Microsoft\Templates folder.

graham mayor say:
You should put a filename field in the footer. The simplest way to do this is to do it manually. Open the footer view. Put the cursor where you want the data and then Press CTRL+F9, which will insert a pair of field brackets { }.

Between the brackets type FILENAME /p.

With the cursor in the field, press F9 to update the field.

Press ALT+F9 to toggle the display and show the result.

If you change the location of the document, you can update the field. It should update when you next open the document, or you can force an update either by switching to print preview and back again, or by using a macro such as the sample macro at http://www.gmayor.com/installing_macro.htm
