rich007a say:

Hi,

Try this:

 

 

Sub LoopThroughFiles()
    Dim strFile As String, strFolder As String
    Dim wb As Workbook
   
    strFolder = "C:\temp\"
    strFile = Dir(strFolder & "*.xlsx")
    Do While Len(strFile) > 0
        On Error Resume Next
        Set wb = Workbooks.Open(strFolder & strFile, UpdateLinks:=True)
        On Error GoTo 0
        If Not wb Is Nothing Then
            Application.Calculate
            wb.Close SaveChanges:=True
            Set wb = Nothing
        Else
            Debug.Print "Couldn't open: " & strFolder & strFile
        End If
       
        strFile = Dir
    Loop
End Sub

 

 

Not sure if you need the Application.Calculate there as it will probably calculate it when the workbook is opened....

jun yewgoh say:

How do I set spacing 1.0 between headings and paragraphs and 1.5 as default spacing for all lines paragraphs excluding headings? I am using Word 2013.

doug robbins - mvp office apps & services (word) say:

Modify the Heading 1 Style so that it is based on (none) and the paragraph formatting in that Style so that it has 12 pts after it and single line spacing.  If the other Heading # Styles have been set up properly, Heading 2 should be based on Heading 1 and Heading 3 should be based on Heading 2 and so on, and if the paragraph formatting in them has not previously been changed, they should inherit the paragraph formatting from Heading 1.  If not, you will need to modify each of them in the same way.

Then modify the paragraph formatting in the Style used for the paragraphs (which should be Body Text, rather than Normal) so they have line spacing of 1.5 lines.

Translation of 'Date codes'( 5digits) to - Microsoft Community

horology say:

Translation of 'Date codes'( 5digits) to understandable dates

I have a legacy Excell file (a inventory of a collection) started 20 years ago. 

One of the colums Aquisition date" now is formated  as text (and I believe originally was also text)  which I entered in a YY-MM-DD format  (with records showing  YY-00-00 if exact date was not available).   

Somewhere along the line, as the file migrated  from one comuter to the nex,t and through various versions of software, somehow in  the newer records this  colum starts showing  five digit integers, which I know I did not enter

 I assume they were entered during a time when there was a different format default, but  they now appear in text formated fields as an uniterpretable 5 digit code.  

How can I translate these numers back into dates I can understand.
joeu2004 say: horology wrote:
I have a legacy Excell file (a inventory of a collection) started 20 years ago. 
One of the colums Aquisition date" now is formated  as text (and I believe originally was also text)  which I entered in a YY-MM-DD format  (with records showing  YY-00-00 if exact date was not available).
Somewhere along the line, as the file migrated  from one comuter to the nex,t and through various versions of software, somehow in  the newer records this  colum starts showing  five digit integers, which I know I did not enter
I assume they were entered during a time when there was a different format default, but  they now appear in text formated fields as an uniterpretable 5 digit code.  
How can I translate these numers back into dates I can understand.

Of course, first you need to explain the structure of the 5-digit representation of dates.  I suspect they are ordinal dates of the form yyddd, where ddd is the day of the year numbered from 1 to 366.

In a parallel column, enter the following formula (copy down the column):

=IF(A1="","",IF(LEN(A1)=5,DATE(IF(LEFT(A1,2)<"30",2000,1900)+LEFT(A1,2),1,RIGHT(A1,3)),
IF(RIGHT(A1,5)="00-00",DATE(IF(LEFT(A1,2)<"30",2000,1900)+LEFT(A1,2),1,1),
DATE(IF(LEFT(A1,2)<"30",2000,1900)+LEFT(A1,2),MID(A1,4,2),RIGHT(A1,2)))))

formatted as Date.

After the conversion, you can select the conversion column and copy, then select the original, right-click and click on Paste Special, select "Values and number formats", and click on OK.  Now you can delete conversion column, which is probably full of #VALUE errors now.
