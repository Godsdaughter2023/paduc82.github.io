This issue did not start until after the computer was off for 3 months and all the windows updates were done at once.
vijin v say:

Hi Michaelw,

 

When more than one network connection is available, Windows uses the one with the lowest metric value. By default, it automatically assigns a metric value based on the network connection's rated speed.

 


To force Windows to use a specific network connection, assign a metric value to each one, giving the lowest value to the desired connection:

 

Press the ‘Windows + R’ key on the keyboard.

Type ncpa.cpl and click ‘Ok’.

Right click the desired connection.

Click Properties > Internet Protocol Version 4.

Click Properties > Advanced.

Un-check "Automatic metric".

Enter a number between 1 and 9999 for the "Interface metric".


 

Hope this information is helpful.

Hi,

 

Method 1:

Perform a clean boot and then restart the computer and check. Visit the link and follow the steps.

How to troubleshoot a problem by performing a clean boot in Windows Vista or in Windows 7

http://support.microsoft.com/kb/929135

 

Note: After the troubleshooting is done, follow the step 7 from the article to set the computer to start in Normal startup.

 
Method 2:

Run a SFC scan and check. Follow the steps mentioned below.

 

  1. Click Start.
  2. Type “cmd” in the search bar.
  3. Right click on Command Prompt in the list.
  4. Choose “Run as Administrator”
  5. Type “sfc /scannow” (Note: there is a space after sfc)

 

You can refer to the link provided to understand the logs generated by the SFC scan.

How to use the System File Checker tool to troubleshoot missing or corrupted system files on Windows Vista or on Windows 7

http://support.microsoft.com/kb/929833

 
 
Method 3:
You can perform a Hardware Clean Boot and check if any device is causing the issue. Follow the steps.

1. Unplug all the External Devices (except Mouse and Keyboard)
2 Click on Start and in the start search box type “devmgmt.msc” and press Enter.
 
Note: Devices that can be disabled for doing a Hardware clean boot are:
1. Display Adapters.

2. Sound, video and game controllers.
3. Network Adapters.
4. USB Devices and controller (Please don’t disable this if you are using USB keyboard or Mouse or Wireless Keyboard or mouse)
5. DVD/CD-ROM drives.
6. Modems
7. Ports
Verify which device is causing the issue by enabling one device at a time and restarting the computer. If you find any of the hardware devices causing the issue, you may have to contact the computer manufacturer or the hardware manufacturer to verify if the hardware needs to be replaced or updated.
IMPORTANT: If the issue is not resolved you need to enable the devices that were disabled while performing Hardware clean boot.

Option 1)

Open registry editor and find the following archive:

Reinstall the keyboard driver.

Right click Start > Select Device Manager

Click the > next to Keyboard

Right click driver > select Uninstall

Reboot the computer
