cbarnhorst say:

Microsoft does not offer ISO files for any Fast Ring builds.  Insiders who want an ISO make their own.  Search online for how to do this.  It must be done after a new build downloads but before you hit the Restart button during the installation of a new build.  There are online sources but MSFT advises against using unofficial ISO files from third-parties.  

Microsoft only provides official ISO files for Slow Ring builds.  Typically these are available a few days after a new build is flighted to Slow Ringers.  No Slow Ring build has been flighted in the last several months.  The reason given is that only relatively stable builds are selected for Slow Ring and so far no RS5 build has qualified.  No Slow Ring flight date has been announced and usually isn't until the day it flights.

Use Fast Ring for bug swatting and Slow Ring to test the usability of new features.  In any case, do not use Insider builds on your primary or production devices.  The hassles of recovering valuable data when a build crashes are too great.

john korchok say:

This is just a hunch, but I found a couple of mentions about this issue being related to the Root Certificate not being registered as a trusted root. I run into this occasionally with new certs and have been able to solve it with these steps:

  1. Open Internet Explorer.
  2. Choose Tools>Internet Options, then select the Content tab.
  3. Click on the Certificates button, then select the certificate in question.
  4. Click on Export, then on Next.
  5. Select Yes, export the private key, then click on Next.
  6. Choose Personal Information Exchange - PKCS #12(.PFX).
  7. Check Include all certificates in the certification path if possible, then click Next. (This is the step that fixes the problem)
  8. Enter a password twice, then click on Next.
  9. Enter a path and name or browse for the location to save the exported cert. then click Next.
  10. Click on Finish.
  11. You're taken back to the original Certificates dialog. Now click on Import, import the file you just exported. 
  12. Reboot, then test the new cert to see if the error message is gone.

john korchok say:

The first time you install a digital key, the Import Wizard gives you a choice whether to allow the key to be exported later. That option is unchecked by default, so that's probably why you are unable to export it now. You should be able to fix this by re-importing the original cert. When you get to the pane below (or it's equivalent), check the option Mark this key as exportable.

Then, when you export, choose Cryptographic Message Syntax Standard - PKCS #7 Certificates (.P7B), and check the option Include all certificates in the certification path if possible.

Finally, reimport the cert.
