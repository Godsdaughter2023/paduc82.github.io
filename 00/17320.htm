If you type 1.5 and format it as date and time, you will get

 

January 1, 1900 12:00 pm

 

as the result, because Excel treats whole numbers as days, fractions as hours, and starts counting in 1900.

 

Try adding those numbers, instead of concatenating them, and assign the format once to the column "mm/dd/yyyy hh:mm"

 

If you must concatenate them, use the TEXT formula, as in:

 

=text(a1,"mm/dd/yyyy") &" " & text(b1,"hh:mm")

rich007a say:

Not really sure where you're going with this, but my advice would be to not use the control source feature.  Just let the user populate the textboxes, then when the user hits a button (you usually put an OK command button on your form) your vba code will populate your excel sheets.

So in the Click event of the OK button, you'd have something like:

    Worksheets("Sheet1").Range("A1").Value = Me.TextBox1.Text

or you could set up named ranges on your sheet(s), and access them like this:

    Worksheets("Sheet1").Range("myNamedRange").Value = Me.TextBox1.Text

If you need to change where data goes depending on other things, determine that with code.  For example, you could add data to the next empty cell in column B by using:

   Dim rngLastCell As Range
   Set rngLastCell = Worksheets("Sheet1").Cells(.Rows.Count, "B").End(xlUp)
   rngLastCell.Value = Me.TextBox1.Text

Here I have shown the third way of accessing a range, by declaring a Range object (Dim rngMyRange as Range), then setting it to a range using (Set rngMyRange = a reference to a range), and populating it using (rngMyRange.Value = x).  The range object can be called whatever you like, but when you start using objects and diferent data types, it's good practice to tag the name with the object type, so you know what you are dealing with.  So rngSomething is a Range object, strSomething is a string object, etc.

Also, make sure you delcare all your variables.  You should, in the VBE, go Tools>Options>Editor tab>Require Variable Declaration.  Now all your modules will start with Option Explicit which will force you do declare all variables.  This helps to catch typos, and will give you much better intellisense options (the things that pop up when you hit ".").

Hope that helps.
