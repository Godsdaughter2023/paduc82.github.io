In Windows 8/8.1/10 the Windows Defender scheduled scan (a Quick Scan) is included in the package of scheduled tasks run by Windows Automatic Maintenance, which is totally automatic and doesn’t require any scheduling or configuration whatsoever. Hence the absence of any scheduling controls in the user interface. However, there is an option to change the trigger time for the daily Automatic Maintenance run. But since the Automatic Maintenance tasks are only run when the system is in an idle state (which eliminates the performance hit classically associated with the scheduled scan), the actual run time can be much later than the trigger time:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-10-how-do-i-set-windows-defenders-scan/f399bb87-37c9-4a74-bb35-1d7e3d3c2436

And for those who prefer the old-fashioned type of scheduled scan, which starts on time and runs without interruption, there’s still an option to set up a conventional scheduled scan (including the option for a scheduled Full Scan) with the Create Basic Task Wizard:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-i-set-windows-defender-to-a-custom-auto/ec19aab4-10ce-4f41-9990-343056fd104f

An independently scheduled scan has always been the best option, because this puts the full set of scheduling options at your disposal:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02
