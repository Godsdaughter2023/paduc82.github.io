alexanderbrock say:

Also, I cannot set up the new account with my Microsoft login or map my email to the new account, because there is another account on the computer with the same information (e.g. the corrupted account).    I do not want to erase the old/corrupt account without having the new one completely set up first.  

Is there some way to do this?   

a. user say:

It's not a common issue, and transferring your files to a new user account is easy.

You can transfer all your personal folders, namely:

  • desktop (see below,)
  • documents,
  • music,
  • pictures, and
  • video.

What you see on your desktop is actually a combination of three things:

  • the contents of your user account's desktop folder;
  • the contents of the Public account's desktop folder; and
  • Windows system objects, like the Recycle Bin.

The key to doing this successfully is understanding that your personal files belong to the user account where they are now. Your new user account is completely separate. You know that both the old and the new user account are yours, but Windows doesn't know that, and that's by design.

The new user account needs to take possession of the files from your old user account, so those files will belong to the new user account. Once you understand this, the rest is easy. I'll give you a summary of the steps below. There's more than one way to do this, but this method is easy:

1- Sign in to a user account with Administrator permissions.

2- Navigate to C:\Users\Public

The contents of the Public account are available to every other user account on the computer.

3- Create a new folder in C:\Users\Public

4- Navigate to your old user account and copy the folders described above to the new folder you created under the Public folder.

As soon as you try to open a folder in your 'old' user account, Windows will ask if you want to have permission to access that folder. The real question it's asking is: Do you want the Administrator account to have the same access to this folder as your old account has? And the answer is yes, of course. Your old user account can move those folders anywhere it likes, and now your Administrator account can do that too.

5- As a precaution, try to open some of the files that you moved to the Public account, just to make sure everything transferred correctly.

6- Create a new user account.

7- Sign out of the Administrator account and sign in to your new user account.

8- Move the contents of the folders you copied to the Public account to the corresponding folders in your new account.

9- As a precaution, try to open some of the files that you moved to the new user account, just to make sure everything transferred correctly.

Now you can erase the folders from the Public account and delete the old user account.

balaji kundalam say:

Norman,

There are some legacy features that are present in all the versions of Windows, even in Windows 10. One of them is the C partition label for the HDD partition where Windows is installed. Back then, when MS-DOS was in use and Windows 1.0 came, Windows was available as an add-on to the existing MS-DOS only and Windows can't be installed directly without MS-DOS. During that time, there were two floppy drives present, 'A' and 'B'. 'A' had the operating system files and 'B' was the storage drive for user files. So when HDD came, it had the label 'C'. When Windows 3.1 came, it was a huge installation at that time that it required a HDD for installation. Hence addition of HDD added the label C for HDD. This legacy is being maintained till date. Hence from whichever installation of Windows you see, the Windows installed partition will be labeled as C only. This is also the reason, whenever you add a floppy drive, the driver letter automatically assigned is A or B and nothing else.

Coming to the system reserved partition, it contains the boot files required for booting of Windows (and also BitLocker data, if present), but present in version of Windows where BitLocker is not present (like in the case of Windows 7 Home Premium). Until Windows XP, the boot settings were handled by a file by the name boot.ini, present in C:\ (at the root of C:\). So when you're upgrading or moving from Windows XP, and that no unformatted partition is available, it means a new partition can't be created. In that case, the boot files will be stored in the Windows installation partition only. Hence during fresh installation of Windows Vista or higher, System Reserved partition will be created but if upgraded from Windows XP, the boot files will be stored in the Windows installed partition only.
