ken sheridan say:

Repeating values from a referenced table in non-key columns in a referencing table is bad relational database design.  The columns are transitively dependant on the table's key, with the result that the table is not normalized to Third Normal Form (3NF), which requires all non-key columns to be functionally determined solely by the whole of the key.  The table is consequently open to the risk of update anomalies.

Whenever you need to return the client's full name as a single string expression the tables should either be joined on ClientID in a query, and the values concatenated into a single column in the result table, or the values should be concatenated in a computed control in a form, but not a report, by an expression which references the Column property of a combo box.

To concatenate the values the following function can be used:

Public Function ConcatValues(strSeparator As String, ParamArray arrVals())
 
    ' Pass this function a character or characters
    ' to be used as the separator followed by the values to be combined
    ' For example: strFullName =
    ' ConcatValues(" ",FirstName,MiddleName,LastName)
   
    Dim X As Integer, strLine As String
   
    For X = 0 To UBound(arrVals)
        If Not IsNull(arrVals(X)) And Trim(arrVals(X)) <> "" Then
          strLine = strLine & strSeparator & arrVals(X)
        End If
    Next
     
    ' remove leading separator character(s)
    ConcatValues = Mid(strLine, Len(strSeparator) + 1)
 
End Function

This is a variation of the CanShrinkLines function published by Microsoft and suppresses Nulls.  You'll find it, and other concatenation functions, illustrated in Concat.zip in my public databases folder at:

john - project say:

Sandy,

This is the formula Jan presented in the referenced link (with one misplaced ")" removed so the formula will work)

Iif(([Baseline Duration])>0,([Actual Duration]/[Baseline Duration])*100,0)

I don't "know" exactly what formula you are "looking for" but that's correct, the status date does not enter into the equation. However for duration tracking a more thorough expression of a "planned complete" value would measure the amount of duration between the actual start and the status date. For that case the formula is more complex but could look like this:

IIf(([Baseline Duration])>0,((ProjDateDiff([Baseline Start],[Status Date])+[Minutes Per Day])/[Baseline Duration])*100,0) & "%".

ossiemac say:

Answer byMauro Gamberini requires you to know how many rows are in the filtered range. The code below is generic to whatever number of rows there are in the filtered range.

 

Answer by Andreas will tell you if filters are set but with code it is possible to set filters so that no data is actually displayed so it does not appear to answer your question.

 

A couple of pointers about the code example. Visible rows cannot be counted because the count stops after the first non contiguous row. However, cells can be counted. Therefore the code uses Resize to set the range to one column only of the filtered range and then counts the visible cells in a single column which in turn equates to the visible rows.

 

The number of visible cells includes the column header therefore count must be greater than one for there to be any actual data.

 

The test for AutoFilterMode avoids code errors if AutoFilter is not turned on. Note that AutoFilterMode does not test if any actual filters have been set. You could use the following line in lieu to test if any filters are actually set.

If .FilterMode Then

 

Sub CountFilteredRows()

Dim rngFiltColumn As Range

 

'Edit "Sheet1" to your sheet name

With Sheets("Sheet1")

 'Test if AutoFilter has been turned on

 If .AutoFilterMode Then

   With .AutoFilter.Range

   'Resize to one column only of filtered range

   'so that cells can be counted.

   Set rngFiltColumn = .Resize(.Rows.Count, 1) _

   .SpecialCells(xlVisible)

   End With

 Else

   MsgBox "AutoFilter is not turned on"

   Exit Sub

 End If

End With

 

'Note that headers count as one row of visible data

'therefore must be greater than 1 for actual data.

If rngFiltColumn.Cells.Count > 1 Then

 'Subtract 1 from count due to headers.

 MsgBox "Filtered data contains " & _

   rngFiltColumn.Cells.Count - 1 & " Rows"

Else

 MsgBox "Only headers visible. No filtered data visible"

End If

 

End Sub

 

Just as an added extra. The following code sets a range variable to the visible cells of the AutoFiltered range. The range variable can then be copied and pasted to another location.

 

Sub VisibleDataRange()

 

Dim rngVisible As Range

 

With Sheets("Sheet1").AutoFilter.Range

 Set rngVisible = .Offset(1, 0) _

    .Resize(.Rows.Count - 1, .Columns.Count) _

    .SpecialCells(xlVisible)

End With

 

rngVisible.Copy Destination:=Sheets("Sheet2").Range("A2")

 

End Sub

 

Notes on the code.

.Offset(1,0) moves it down off the column headers but then includes an additional row at the bottom.

.Resize(.Rows.Count - 1, .Columns.Count) removes one row off the bottom and sets to number of columns in the AutoFiltered range.

.SpecialCells(xlVisible) is self explanatory.

 

Where .Rows.Count is used is before the .SpecialCells(xlVisible) and therefore includes all of the rows (visible and non Visible) in the AutoFilted range and it is not until it gets to .SpecialCells(xlVisible) that the non visible cells are excluded.
