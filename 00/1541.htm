joeu2004 say:

Although you say that you want the IRR "for every 12th date", your example actually seems to be a moving 7-month IRR.  I assume that is what you really want.

(Although it is not difficult to adapt the following, if you truly want the IRR "for every 12th date".)

Since the events are about a month apart, it would be easier to use Excel IRR.  In that case, the investment amounts and account values must have

The formula in D8 is:

=(1+IRR((B2:B7,C8)))^12 - 1

Compounding the monthly IRR calculates an annual IRR that is comparable to Excel XIRR (see below).


The formula in E8 calculates the NPV using D8.  It should be zero (F18) or nearly so.  The formula in E8 is:

=B2 + NPV((1+D8)^(1/12)-1, (B3:B7,C8))

Copy D8 into D9:D20, and copy E8 into E9:E20.

The syntax (B2:B7,C8) is called a range union operation.  It is allowed for Excel IRR and NPV, but not for Excel XIRR.

The problem in row 13 demonstrates that it is not reliable to use Excel IRR or XIRR because the default "guess" (10%) is not always sufficient.

The table in columns K:L demonstrates that the annual IRR is between -80% and -70%.  In this case, if we use a "guess" of -75%/12, the formula

=(1+IRR((B7:B12,C13), -75%/12))^12 - 1

returns about -75.19273%.

Unfortunately, it is not always that easy.

Technically, we should use a "guess" of (1-75%)^(1/12)-1.  But even that does not ensure success.

------

If you insist on using Excel XIRR, there are a number of complicating issues, as demonstrated by the following example.  Note that the investment amounts and account values may have the

The array-entered formula (press

=XIRR(IF(ROW(B2:B8)=ROW(B8), -C8, B2:B8), A2:A8)

The normally-entered formula (just press Enter as usual) in E8 is:

=SUMPRODUCT(B2:B7/(1+D8)^((A2:A7-A2)/365)) - C8/(1+D8)^((A8-A2)/365)

As noted above, we cannot use the range union operation with Excel XIRR.  So we must use an IF expression to select C8 instead of B8.

Also, we cannot use Excel XNPV because it does not permit a negative IRR, for no good reason.  (It is a defect, IMHO.)  So we must use SUMPRODUCT to calculate the math formula that appears in the XNPV help page.

Finally, note that in row 13, Excel XIRR returns a bogus numerical result instead of a #NUM error, resulting in an NPV in E13 that is not nearly zero.  With a "guess" of -75%, the array-entered formula

=XIRR(IF(ROW(B7:B13)=ROW(B13), -C13, B7:B13), A7:A13, -75%)

returns about -75.62589% in D13, and the formula in E13 returns about -4.77E-06.
