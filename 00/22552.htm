boblarson say:

A couple of things.  The history would change based on changes you made.  And the history accumulates too, so not sure what you were meaning about the various entries as each individual change type would be a record in history. 

 

Second, I seriously am concerned with changing a key field like this and you seem to indicate this is more than just a single event you would ever do.  Keys should not be changing.  In fact, numeric keys are more efficient than text keys so I would not suggest changing to a text key.  And last, if your keys can change then they aren't good key candidates.  I would use a surrogate key (autonumber) and then use a multi-field index if you need to keep duplicate values from being entered.

joanne li msft say:

Hi Rowden,

Based on my understanding, you have a folder named Management and its sub-folder named Team in a document library, and you'd like to let only two users see the Management folder and everyone be able to access the Team folder.

If so, you may try the steps below to check if it meets your requirement:

1.Create a SharePoint group for the two users who you want to see the management folder.

   a.Go to your SharePoint online site, go to Site settings>Users and Permissions>Site Permissions, click Create Group under the PERMISSIONS tab.

   

   b.On the Create group page, give the group a name, for example, Management. And give proper permission you want to this group, then click Create on the bottom corner.

   c.After the group is created, click the Group name. in my case, click Management, and click New>Add users to this group, then enter the two users name in the Invite people box to add them to this SharePoint group, then click Share.

2.Navigate to your document library in classic experience, select the management folder and click Share button. Click Shared with in the left pane and click ADVANCED, we will go to the permissions page for this folder.

3.click Stop Inheriting permissions under PERMISSIONS tab, then check whether there is anyone who you don't want to let see the management folder in these groups, you need to click into every group to check it. If so, please remove this group by check the box next the group name then click Remove Users Permissions on the PERMISSIONS tab.

4.then click Grant permissions button to grant the group we just created permission to this management folder.

5.go back to document library and click into the sub-folder Team, click Share button and take the steps above to go to the permission page for this folder.

6.click Stop Inheriting Permissions, then click Grant Permissions and enter everyone or everyone's name in the dialog box, then click Share.

Please note by doing this, everyone except the two users cannot see the folder Team when then navigate to this document library, they can access it using the URL. So they need to bookmark this folder to make them easily access it next time.

Besides, a better user practice is to create the folder Team outside the folder Management, Stop inheritance permissions for folder Management and only grant the two users permission to it.

Here are some articles for your reference:

Customize SharePoint site permissions

Edit and manage permissions for a SharePoint list or library

Customize SharePoint site permissions

Feel free to post back if anything is misunderstood.

Regards,

Joanne

In reply to deleted message

The copy and destination in the one line as per you example does not work with values.. However, either of the following 2 methods work.

The advantage of the first method is that it does not leave the destination highlighted after the action like paste special used in the second example does.

Example 1:

Range("D8").Resize(1, 19) = Cells(Target.Row, "D").Resize(1, 19).Value

Example 2:

Cells(Target.Row, "D").Resize(1, 19).Copy

Range("D8").PasteSpecial xlValues
