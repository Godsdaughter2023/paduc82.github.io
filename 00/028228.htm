brian tillman [outlook mvp 2007-2019] say:

I can think of one way this could happen.  Suppose someone two years ago (or longer) had the address "me @live.com" and use that address to log into Facebook or whatever.  Now suppose that person stopped logging into the live.com account for a year.  Because the account has been inactive that long, Microsoft would have deleted the account and released the address for reuse.  All this time, Facebook or any other service (like a bank) that had been configured to use that address would have been sending the messages nowhere becuase no one was using the account any longer.  Now you come along and create the account "me @live.com".  Because the address had been released for reuse, you were able to create the account again.  Now Facebook or the bank or a credit card service or any other service that had been configured to use that address now has a mailbox to send its messages to and you now get them.  You can even then use the address to hijack those other services because you now have control over that address and can place password reset requests.  This demonstrates why releasing addresses for reuse is a BAD idea.

 

Now, I don't know for sure if this is the situation with you, but it would explain what you see.

Me.Text4=Dmin("[PcvNo]","[PettyTab1]","Nz([Status],"""") <> ""CONFIRMED""")

 

Note that there is no need to concatenate the literal string "CONFIRMED" into the expression.  You can simply do as above.  You could of course use single quote characters as the delimiters, rather than representing the literal quotes characters as I've done by contiguous pairs of double quotes characters.  I generally use the latter to allow for apostrophes in a value.

 

I'd also suggest that when you add a control to a form the first thing to do is change its name from the meaningless default value like Text4 to something which reflects its function.  Any reference to the control is then largely self documenting.
