Re #1: - Just out of curiosity, what exactly did you try to do to "fix" your profile?

Re: #2 - Creating a new profile would never result in any emails getting deleted but what steps you need to follow depends on the type (POP3, Imap, Exchange) of email account involved.

*** Before creating a new profile, can you advise what type of email account is involved where you're trying to delete the email message? It may be that there is an error with the data file associated with the account.

(a) for a POP3 account, suggest running ScanPST on the PST file associated with the account. You may need to run it several times to eliminate any potential errors.

(b) for an Exchange account, with Outlook closed, rename the .OST file and open Outlook. A new .ost file will be created and everything will be re-sync'd with the server

(c) For an IMAP account, if there are folders in the IMAP Folder tree labeled with the words "This Computer Only" (i.e. contact/calendar), and these contain data that you need to retain, be sure to export those folders to a PST file using the Outlook export wizard. Once done, with Outlook closed, rename the OST file, start Outlook. The OST file will be recreated and the email folders will be re-sync'd with the server. If you've exported data to a PST file, open that file to the profile by going to Outlook menu File --> Open & Export --> Open Outlook Data File. Once the file has been added, go to Account Settings --> Data Files tab --> highlight the added PST file --> set this file as the default data file for the profile. This will be where the default contact/calendar folders will be located and you won't have to worry about ever losing any data formerly contained in any "This Computer Only" folders

If the issue still persists then

(d) To create a new profile, with Outlook closed, open the Control Panel <Mail> app., add a new profile and configure your email account(s). Once done, be sure to set the new profile as the default for the next time Outlook is opened. (Do not delete the existing profile just in case you need to get back to it for any reason)

Account configuration considerations - during profile creation or after Outlook is started depending on account type

(e) - For POP3, be sure to use the existing PST file that was associated with the account instead of allowing Outlook create a new PST file

(f) - For an Exchange account - just start working normally

(g) - For an IMAP account, add the PST file as described in step 2(c) (if applicable)
