After you set the color, hit Ctrl+G to go to the Immediate window, and enter something like:

?Forms!myForm!myButton.BackColor

It will return a value like this:

-2147483607

That seems like an odd value until you look at the hex value:

?Hex$(Forms!myForm!myButton.BackColor)

80000029


In other words: these theme colors are special values outside of the normal RGB range. You can set them just like any other RGB value:

Const MY_THEME as Long = &H80000029

Forms!myForm!myButton.BackColor = MY_THEME

Sub SplitTxt_01()

Const HelperFile As String = "ABCD" '<<< temp. helper text file Name
Const N As Long = 700000  '<<< split each txt in N rows, CHANGE
Dim myPath
myPath = "c:\Folder1\Folder2\" '<<< folder path, CHANGE
Dim myFile
myFile = "Data File.TXT" '<<< your text file. CHANGE txt file name as needed

Dim WB As Workbook, myWB As Workbook
Set myWB = ThisWorkbook
Dim myWS As Worksheet
Dim t As Long, r As Long
Dim myStr
Application.ScreenUpdating = False

'split text file in separate text files
myFile = Dir(myPath & myFile)
Open myPath & myFile For Input As #1
t = 1
r = 1
Do While Not EOF(1)
Line Input #1, myStr
If r > N Then
t = t + 1
r = 1
End If
Open myPath & HelperFile & t & ".txt" For Append As #2
Print #2, myStr
Close #2
r = r + 1
Loop
Close #1

'copy txt files in separate sheets
For i = t To 1 Step -1
Workbooks.OpenText Filename:=myPath & HelperFile & i & ".txt", DataType:=xlDelimited, Tab:=True
Set WB = ActiveWorkbook
Set rng = ActiveSheet.UsedRange
Set myWS = myWB.Sheets.Add
myWS.Name = HelperFile & i
rng.Copy myWS.Cells(1, 1)
WB.Close False
Next
myWB.Save

'Delete helper txt files
Set Fso = CreateObject("Scripting.FileSystemObject")
Set Fldr = Fso.GetFolder(myPath)
For Each Filename In Fldr.Files
If Filename Like "*" & HelperFile & "*" Then Filename.Delete
Next
Application.ScreenUpdating = True
End Sub

rohn007 say:
No you do not HAVE to uninstall 2003.  The terms of the 2010 license are totally independent of 2003, so you do have the "legal" right to run both at the same time.

I found it handy to have 2003 in place for times when I "just have to get something done" and am P.O.'d hunting for features in 2010. If you are comfortable with 2010, then you might as well get rid of "duplicate" 2003 apps to free up HD space.

If you do decide to keep both, there are a few extra steps you have to take to make the "play well together". Office is designed with the assumption that there is only one version installed. There are 2 main issues: Windows default file associations and "re-installing" office each time you start an app in the "other" version.

The first one is a windows problem. You can only have one app selected as default. So you have to make a choice. The only difference is that for the non-default one you have to use the File Open dialog to open files.

If you want to keep both, here are the tips I used to make them work:
Dual Install - Working with Multiple Versions of Office 

https://office.microsoft.com/en-us/word-help/keep-earlier-versions-of-office-programs-when-installing-office-2010-HA102597134.aspx

This tip describes the specific steps and choices during the 2010 custom install. After you have the multiple versions installed you still have some additional steps to do. They are described in this tip:

http://pubs.logicalexpressions.com/pub0009/LPMArticle.asp?ID=762

With multiple versions installed, there is one issue you can’t avoid, Default Windows File Associations. Only one app can be used to open the files by double clicking in Windows Explorer. You have to pick which one.

How to Create a 300 dpi JPEG from Artwork Created in Word - Microsoft Community

davidbroaden say:

How to Create a 300 dpi JPEG from Artwork Created in Word

I created some artwork in Word and I want to save it as a 300 dpi JPEG picture.  When I save the Word file as a PDF then save the PDF as a JPEG, the JPEG is only 200 dpi.  Suggestions/recommendations?
jay freedman say:

Get a good graphics program, such as the free IrfanView. Copy the artwork in Word, and paste it into the graphics program. Go to the program's Resize dialog (in IrfanView, it's on the Image menu) and set the 300 dpi resolution, as well as the correct size in pixels needed to keep the image at the desired size. When the picture has been resized/resampled, save it.

There is no need to pass through the PDF intermediate file in this method.

If there's no specific requirement for a JPEG file, you'd do better to save the image as a PNG file. The JPEG compression algorithm is "lossy" -- every time you edit and resave the file, it loses some information and can eventually become blurry. In contrast, the PNG format uses a lossless compression algorithm.
