bernd p. say:

The link starting with onenote was originally only created and valid for starting OneNote 201x (or, later, for starting OneNote), depending on the file type association in Windows. Now it works even in OneNote Online.

So it's now compatible with a classic Hyperlink, starting with https, which is created by using OneNote Online.

If you look into the clipboard after copying a link in OneNote (2016) you'll see that you got two links: a https-link and a onenote link. That's some history. In older OneNote versions (e.g. 2010) they had the opposite order. This order is relevant because the first link is used when pasted into OneNote. So your onenote links come from an older OneNote version, at least an older OneNote 2016. 

Bernd

What you can do is either set a scheduled cleanup for specific email address, or create an email rule for the whole domain. When you create an email rule for a domain to automatically move to a specifc folder, then you will lose the option to set a timeframe as to when those emails will be deleted.

 

However, there is a workaround for this. You may create an email rule and move the folders in your Junk folder (all emails in the Junk folder will be deleted automatically after 10 days).

To create an email rule, please follow the simple steps below:

  1. Sign in to your Microsoft account.
  2. Navigate to your Inbox page.
  3. Click the cogwheel icon on the upper right corner of the page, then select More mail settings...
  4. Click Rules for sorting new messages under Customizing Outlook.
  5. Click the New button.
    1. In Step 1, select Senders address on the first drop down menu, then select contains on the next drop down.
    2. In the blank field box, type in the exact domain name (@somecharity.org)
    3. In Step 2, select the first radio button "Move to", then click the down arrow icon for you to select Junk folder.

Additionally, since you have already provided a suggestion on our feedback page, most likely that it will be implemented in the future update of Outlook.com, if Microsoft finds no reason why this feature should not be implemented.
