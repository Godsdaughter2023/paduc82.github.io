ronrosenfeld say:

In your first post you indicated you had only ten columns of data; now you write you have 200 columns.  That makes a difference.

To sum the first N columns starting with the first non-blank cell, with N being a variable, you can use the following formula:

This formula must be **array-entered**:

=SUM(OFFSET(A2,0,MATCH(TRUE,ISNUMBER(2:2),0)-1,1,A1))

  • A1 contains the number of columns to SUM.
  • This formula cannot be entered on the same row as the row you wish to SUM.  If it MUST be entered on that row, then adjustments will need to be made depending on where in the row it is entered.

mike h.. say:

Hi,

So you have a row of numbers and you want to sum the first n non-blank cells. Try this array formula:-

=SUM(A2:INDEX(A2:Z2,SMALL(IF(ISNUMBER(A2:Z2),COLUMN(A2:Z2)-COLUMN(A2)+1),MIN(A1,COUNT(A2:Z2)))))

As written the formula does A2:Z2 but you can extend that range. ARRAY enter the formula; see below on how to do that, and then enter the amount of numbers you want to sum in A1 so if you enter 3 for example in A1 it will sum the first 3 numbers.

Note that if you enter (say) 5 in A1 and there are only 4 numbers the formula will sum the 4 numbers, we can change that if you want.

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

Location shown in Cortana and Maps may not be accurate - Microsoft Community

analy [msft] say:

Location shown in Cortana and Maps may not be accurate

First of all, thank you for all of the feedback and bringing all of the location accuracy issues to our attention.  Even though we’re not able to reply individually to all of the cases, we’re listening and you are helping make Windows better for everyone. Based on your feedback, we’ve investigated and identified some issues we’re working on improving in future builds. We know how frustrating it is when your location is wrong so here’s some more details if you’re interested.

Most apps rely on the location platform in Windows to provide the location information. Depending on the hardware available on each device, the availability of the location information and its accuracy will vary. For instance, a device with GPS will provide better accuracy than a device without it and in general Wi-Fi based information is more accurate than the one based on reverse IP.

What this means is that for most desktop users (where GPS presence is very rare) if the computer doesn’t have a Wi-Fi adapter, the location results will be obtained using the IP address. Using this technique, the results can vary widely in accuracy and can sometimes be incorrect. This has always been the case and we do not expect users to see a major difference here between Windows 10 and previous versions of Windows.

For laptop/tablet users, Wi-Fi based positioning is usually very good, typically in the range of 50m-150m of accuracy. We have uncovered some issues with certain Wi-Fi hardware in Windows 10 that results in a failure getting the positioning from Wi-Fi and thus falling back to the much less accurate IP based positioning. This causes the location shown to users to be incorrect or highly inaccurate even when the device is connected to Wi-Fi and will impact some users after upgrading from a previous version of Windows to Windows 10. We’re actively working on this problem and hope to have it fixed in future builds. *Update* The fix for this issue was released through Windows update at the end of September, users being impacted by it should be getting more accurate locations once they have installed the latest updates.

*Update* The latest version of the Maps app lets you enter the address where you keep your PC and we'll use it as your location when we don't have a precise reading on where you are. Read more on this thread.

- The location and Maps teams
