dirk goldgar say:

The relationship issue would be most likely to arise if you had a table with a lot of fields related to other tables, and it would be magnified if those fields were defined by the lookup wizard, since the lookup wizard creates hidden relationships.

So for example, if you had a table like this:

    Table1

    --------

    ID (pk)

    ID_2 (fk to Table2)
    ID_3 (fk to Table3)
    ID_4 (fk to Table4)
    ID_5 (fk to Table5)
    ID_6 (fk to Table6)
    ID_7 (fk to Table7)
    ID_8 (fk to Table8)
    ... and so on

you could max out your indexes pretty quickly.

Word 2016 is the same in this respect as Word 2013, 2010, and 2007.

Open the File > Options > Save dialog and look at the options that affect AutoRecover. If the document was open for less than the AutoRecover interval set there, or if the Save AutoRecover check box has been cleared, then there won't be any file to recover.

Copy the path to the AutoRecover file location, and open that folder in File Explorer. If there are no files with .asd extensions, there's nothing to recover, so Word won't show the recovery pane.

On Windows 10, you have the ability to set up a File History location. If you've done that, then you should be able to recover some version of the document from that location.
