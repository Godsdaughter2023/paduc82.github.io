dale howard [mvp] say:

DanielNol --

Since you are entering progress using the % Complete field, I believe you need to use the % Complete field rather than the % Work Complete field.  The formula for the % Complete field is Actual Duration/Duration.  So, if a resource has completed 50% of a 10-day task, the Actual Duration will be calculated as 5 days so that this reflects the task is 50% completed.  Also, the formula for % Complete on summary tasks is the same, so you are not going to see the 82.5% that you want to see on the summary task.  The 31% is the correct value and you will need to live with that.  It is just how the software works.

As a side note, I would recommend that you explicitly avoid using Manually Scheduled tasks as they enforce way too many restrictions on your project schedule.  Instead, I would recommend you use Auto Scheduled tasks and that you make sure you have linked every subtask and milestone with at least one Predecessor and at least one Successor.  Doing so will give you a more flexible schedule so that Microsoft Project can do its job, which is to show you the forecast of your current schedule.

Hope these thoughts help.

dale howard [mvp] say:

Daniel --

You cannot change how the values in the % Complete column are calculated for summary tasks.  Using the Physical % Complete field would be no help, as values are not calculated in this column for summary tasks.  Hope this helps.

The Windows Phone OS does not provide a user option that could be used to manually associate file types with apps on the phone.
