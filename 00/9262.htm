michael macayan msft say:

Hi Chris,

 

When you store your Access database file in your SharePoint site, the other user who wants to modify it, will need to download the file and make changes on his local Access application. As real time collaboration for Access is not feasible.

 

To prevent other user from overwriting your changes, you may need to check out the file, to let the user know, that another user is still making some changes on the database file.

 

Even when you share the Access database file to another user in OneDrive, the user will only be able to download a local copy of the database file.

 

With this, I suggest you try network sharing for the Access database file, and then set the open mode to Shared. To configure this, navigate to File > Options and then Client Settings. Go to Advanced and then select Shared as default open mode.

 

Also, you may submit a feedback on our Access UserVoice Forum, as this is the best way to influence our products.
