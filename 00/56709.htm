vijin v say:

Hi Aloke,  

 

This issue is caused if the hardware and drivers are not valid or signatures maintained in the firmware.

 

UEFI Secure Boot:

 

When Secure Boot is activated on a PC, the PC checks each piece of software, including the UEFI drivers (also known as Option ROMs) and the operating system, against databases of known-good signatures maintained in the firmware. If each piece of software is valid, the firmware runs the software and the operating system.

 

The signature database (db) and the revoked signatures database (dbx) list the signers or image hashes of UEFI applications, operating system loaders (such as the Microsoft Operating System Loader, or Boot Manager), and UEFI drivers that can be loaded on the individual PC, and the revoked images for items that are no longer trusted and may not be loaded.

 

If the firmware is not trusted, the UEFI firmware must initiate OEM-specific recovery to restore trusted firmware.

 

What is UEFI?

 

Secure Boot Overview

 

Boot a PC in UEFI Mode or Legacy BIOS-compatibility mode   

 

For more information and update, you may have to contact the System Manufacture’s for the same.

 

Windows Defender:

 

Yes, Windows Defender Offline can help remove such hard-to-find malicious and potentially unwanted programs. Sometimes, malicious and other potentially unwanted software, including rootkits.

 

http://windows.microsoft.com/en-gb/windows/what-is-windows-defender-offline

 

Enable UEFI and Safe mode:

 

Yes, you can enable them from BIOS, incase if you are facing any issue you can disable the driver signature enforcement and check if it helps.

 

Windows Start-up Settings (including safe mode)

 

Note: Modifying BIOS/ complementary metal oxide semiconductor (CMOS) settings incorrectly can cause serious problems that may prevent your computer from booting properly. salvation at the cross say:

The system locale determines the default character set (letters, symbols, and numbers) and font that you use to enter information and that are used to display information in programs that do not use Unicode. This allows non-Unicode programs to run on your computer using the specified language. You might need to change the default system locale when you install additional display languages on your computer. Selecting a different language for the system locale does not affect the language in menus and dialog boxes for Windows or other programs that do use Unicode.

  1. Open Regional and Language Options by clicking the Start button , clickingControl Panel, clicking Clock, Language, and Region, and then clickingRegional and Language Options.

  2. Click the Administrative tab, and then, under Language for non-Unicode programs (System Locale), clickChange system locale.  If you are prompted for an administrator password or confirmation, type the password or provide confirmation.

  3. Select the language, and then click OK.

    To restart your computer, click Restart now.
