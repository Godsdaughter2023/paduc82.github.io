rich michaels - mvp (greatcirclelearning.com) say:
Hi Andy,

Where are you storing the dotm file as you work on it? If you are putting it into the the Startup folder of Word, then that is why you are receiving that message. You cannot edit a template stored in the Startup folder. Only edit "works in process" from the My Templates directory.

Hope this helps

andy1756 say: Rich,

Thank you for your prompt reply.


I had stored the .dotm file in the following directory:

Macintosh HD:Applications:Microsoft Office 2011:Office:Startup:Word

and my VBA code (in Normal.dotm) to install the Add-In was:

Sub AutoExec()

AddIns.Add fileName:="Macintosh HD:Applications:Microsoft Office 2011:Office:Startup:Word:Office Startup Test Add-In.dotm", Install:=True

End Sub



When I stored the Add-In in the MyTemplates directory:

Macintosh HD:Users:Andy:Library:Application Support:Microsoft:Office:User Templates:My Templates

the behavior was exactly the same.


What I discovered is that what was causing the "Project Unviewable" message was having the Add-In INSTALLED, which makes sense, because Word should not allow edits to currently loaded code.  In Word 14.0.0 (prior to installing updates), I WAS able to open the .dotm file even while stored in the Startup folder.  In fact, it made no difference to Word 14.0.0 where the .dotm file was stored; as long as the Add-In was not installed, I could always open its code in VBA.

With Word Update 14.3.7, the restriction you mentioned is now in effect; any Add-In stored in the Startup folder cannot be opened in VBA, because Add-Ins in the Startup folder are now all installed by default.


For what it's worth, I chose to store the Add-In in a logical place:

Macintosh HD:Applications:Microsoft Office 2011:Office:Add-Ins

and to disable loading (and thereby enable editing) of the Add-In, I simply comment out the AutoExec() VBA Code (in Normal.dotm).  If I'm ever in doubt about whether or not I forgot to comment out the VBA code, I can view the Add-In's status in Word's Tools...Templates and Add-Ins... dialog box.


Thank you so much for getting me to explore these dependencies.

Andy
