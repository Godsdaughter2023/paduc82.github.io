a. user say:

The check box in Tools > Options > Library > 'delete files from computer' means to delete them from your hard drive. With the box unchecked, the title will be erased from WMP's inventory list, but the file is still on your computer, so it can be played by WMP or any other player you choose.

And where is the file on your computer? It's wherever you chose to put it when you originally put it on your computer. The default location is C:\Users\<your user name>\Music, but the file could be anywhere. For example, if you originally r-r-ripped the file, it was stored in the folder specified in Tools > Options > Rip Music. If you originally downloaded the file, it was stored in whatever folder is your browser's default for downloads - unless you specified a different folder at the time you downloaded.

a. user say:

Hi Verne,

Thank you for posting in Community. We understand your concern and we will help you resolve the issue.

The error 0x80070002 is caused by the following scenarios:

Scenario 1: This issue occurs if you back up a library that contains custom folders that are located under your user profile. 

Scenario 2: This issue occurs if the following conditions are true:

  • Windows Backup tries to back up files in the LocalLow, Contacts, and Searches folders.
  • These folders are not present on your computer.

Note: These folders are created only if certain applications are installed on the computer.

Method for Scenario 1:

 To resolve this issue, take one of the following actions:

  1. Ignore the error message.
  2. Note The custom library files that are stored in your user profile will not be backed up.
  3. Move the library folder from the user profile path to a location outside the user profile path. For example, move the folder to the C:\MyLibraries folder.
  4. Exclude the library from the list of files that Windows Backup backs up. Then, include the original location of the content in the library in the backup list.
  5. Move the library folder to a location other than the user profile folder. Then, add a link to the moved folder from the library. Make sure that the library is included in the backup. 
    To do this, follow these steps:
  • Log into the computer with the User Account.
  • Right-click the folder that you want to move, and then  click Cut.
  • On the navigation pane, expand Computer, locate the place where you want to store the folder, and then paste the folder in that location.
  • Right-click the folder that you moved to the new location, and then click Copy.
  • Click Start, click your user name, right-click any empty area in the folder, and then click Paste shortcut. 


Note: To store the folder in a location other than a folder in your user profile, you may have to create a new folder. To do this, browse to the root of the drive in Windows Explorer, click New Folder on the Windows Explorer menu, and then edit the folder name. For example, rename the folder "MyLibraries."

Method for Scenario 2:

  1. Go to file history
  2. Left menu pane select Exclude folders, click on Add
  3. From the left navigation pane, click on This PC, then click on c: drive.
  4. Navigate to Users> click on your user account>select AppData folder and click on Select Folder.
  5. Follow the same steps for Searches and Contacts
  6. Click Next, and then click Save settings and exit.
  7. Perform the backup again if desire, and the operation will progress without any error.

Hope this information helps.
