eriqvanbibber say:

Note that although the answer of "yes, Office 365 supports 200gb of Public Folders", it is not a simple 1:1 relationship.

Public folders in Office365 are based on Exchange 2013's "Modern Public Folders" which stores data in one or more special "Public Folder Mailboxes".

In Office365, Public Folder Mailboxes have a default size limit of 50gb.  This value cannot be increased.

As such, when planning a PF migration to O365, work must be done to analyze the source PF data and develop a partitioning strategy to determine how to distribute the source data across multiple PF mailboxes.

This work is not trivial, and currently there are poor scripts that can help do some of this work.  The other part of this is that you DO NOT WANT A PF MAILBOX TO FILL UP!!!  This means that 200gb of on-premises data should likely be split across 6 or 7 (or more) public folder mailboxes, ensuring (as best as possible) that no single PF mailbox is filled more than 50 or 60 percent of the quota, allowing plenty of room for growth.

Also, and even more frustrating is that Microsoft change the way the mathematics are used to report item, folder, and mailbox sizes.  See this post for more details (social.technet.microsoft.com/.../mailbox-sizes-increase-dramatically-after-migration-to-exchange-2013

So, when looking at that 200gb in your source, unless you are getting that value from the disk file for the PF store, the number may need adjustment.  The point is, if you think of taking 39GB of data from a source folder (and it's children), that can exceed 50gb under the new math.

Microsoft allows for up to 100 PF mailboxes, each at 50gb, so that's 5TB of potential use...but don't ever let your org consume the entire 5TB.

Priasoft has some tools to help analyze and restructure PFs as well as an ability to synchronize (not just migrate) PFs between on-prem exchange and O365 (and on-prem to on-prem and cloud-to-cloud too).

N96: Change SMS to MMS or vice-versa on the fly - Microsoft Community

grschinon say:

N96: Change SMS to MMS or vice-versa on the fly

Technical Level : Intermediate

Summary

Details
Once a text message becomes long enough, it becomes cheaper to send it as a text-only MMS message. For example, on T-Mobile UK contracts, an MMS counts as 2 SMS messages, so if your multipart text message requires 3 SMS or more to be sent, then you're better off sending the message as an MMS.

With older Nokia phones you needed to copy the entire text to the clipboard, close the message you were writing, start a new multimedia message and then paste the text into it.

The N96 doesn't have separate "new text message" and "new multimedia message" functions. It just has "new message" and you can change between SMS and MMS quite easily without having to copy/paste the text.

So, assuming you have a long text message that you want to convert into an MMS:

While composing it, go into "Options" / "Message header fields". Mark "Subject" and press "OK". A subject header for you to fill in will appear under the recipient. As soon as you start filling stuff into the subject header, a message box will appear saying "Message type changed to multimedia message". Done!

Alternatively, even if no subject header is displayed, as soon as you insert into the message something that can't be sent as an SMS (a picture, video or audio clip for example), the message type will also switch to multimedia automatically.

Now, if you want to reply to a multimedia message, the chances are that what you're replying to had a subject header, which will be maintained in your reply, thus forcing it to be a multimedia message too. What if you want to reply by text message alone since it's cheaper?

Easy. You either delete everything in the subject header of your reply, which will make the message revert to "text" type automatically, or you remove the subject header altogether: go into "Options" / "Message header fields". Unmark "Subject" and press "OK". If there was something in the subject header you will be asked to confirm that you want to discard that data.

Tip: leave the subject header displayed when composing messages. If you want to send a text message you just leave it blank. If you want to send a multimedia message then you fill it in and/or attach multimedia content. To reply to an MMS by SMS you simply have to delete the subject header's content.

First thing I want to happen is to check  Sheets(“G”).Range (“X1”).Value.  If the value is zero then when someone clicks in the range that cell font should change to black. Then the value within the cell clicked will go into cell T2 and the cell range that was clicked in will go in cell V2.  Next the zero that was in cell X1 will change to 1.

Then the next block of code will be the same except it will check for the value in X1 to be 1.  Then it will run this code (which is the same from above  just the T & V cells are changed.

Then the value within the cell clicked will go into cell T3 and the cell range that was clicked in will go in cell V3.  Next the zero that was in cell X1 will change to 2.

I’m not sure how to write the code, plus the correct location to put it.  I would like for it to go within a module.

rich007a say:

Hi,

Right-click on the sheet's tab (at the bottom) and click View Code.  Paste in this:

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Dim R As Long, C As Long
    Dim rng As Range
   
     'Exit if more than one cell is selected:
    If Not Target.Count = 1 Then Exit Sub
   
    'Create range B,D,F,H,J,L,N,P - 2,4,6,8,10:
    Set rng = Range("B2")
    For R = 2 To 10 Step 2
        For C = 2 To 16 Step 2
            Set rng = Union(rng, Cells(R, C))
        Next C
    Next R
    'Debug.Print rng.Address(False, False)
   
     'If user selects a cell in rng:
    If Not Intersect(Target, rng) Is Nothing Then
        If Range("X1").Value = 0 Then
            Target.Font.Color = RGB(0, 0, 0)
            Range("T2").Value = Target.Value
            Range("V2").Value = Target.Address(False, False)
            Range("X1").Value = 1
       
        ElseIf Range("X1").Value = 1 Then
            Target.Font.Color = RGB(0, 0, 0)
            Range("T3").Value = Target.Value
            Range("V3").Value = Target.Address(False, False)
            Range("X1").Value = 2
        End If
    End If
End Sub

 

Hope that helps (and does what you want).
