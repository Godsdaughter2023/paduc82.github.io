charles kenyon say:

My method is to put the template containing the building blocks on a shared network location (possibly with restricted write access) and to have, as a part of each person's login script, an XCOPY command that copies / updates the file in the local building blocks folder (or startup folder).This means that if you add/change things, you will have to copy your updated file to the network.

You get it onto your network however you put files in shared locations on your network.

You certainly do not want to put a shortcut from your computer onto the network location and then shortcuts from that shortcut on other people's computers.

Every user has a building blocks folder. Every user can have a startup folder, but this folder must be created if it does not exist.

The advantage to doing this is that everyone has access to these building blocks whether or not they are logged into the network and that there is no file lock placed on the master file just because someone has Word open. If, instead, you use a shortcut from that file, people will not have access when off the network and when anyone has Word open, you may not be able to make changes.

I cannot tell you how to add it to a login script. That depends on your network. If you do not have to login to the network, then there is no such animal and a different distribution/update method would be needed. In that case, unless you have some kind of synchronization program going on, I would put the file in a shared folder on the network and put shortcuts from that file in either the startup folder or the building blocks folder.

If a template is placed in the startup folder, more than building blocks can be shared including QAT modifications, macros, and keyboard shortcuts. Such a template will be listed as an Add-In in the templates and add-ins dialog. If it is place in the building blocks folder only building blocks are shared and it does not show up as an Add-In. The only place I know it to show up is in the building blocks organizer.

Again, the AutoComplete feature for AutoText will not work with AutoText stored in either the building blocks folder or the startup folder in Word 2010. (It will in Word 2013.)

I'm afraid I may have added to your confusion. If so, I apologize.
