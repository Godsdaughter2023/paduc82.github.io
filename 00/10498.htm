If you use a macro, you only need to add the two SetWarnings actions, the Visual Basic line will not be used. If the query fails to run, there must be another problem - all that SetWarnings with Warnings on = No does is to suppress the "You are about to run a delete query that ..." and "You are about to delete ..." prompts. The query should be run as if you clicked Yes twice.

To call the VBA function from a command button on a form, you can enter =DeleteRecords() in the On Click event of the command button.

If you want to run it when a form is opened, enter =DeleteRecords() in the On Open event of the form.
