2) note the name of the subkey that represents your default browser, for me it is firefox and the key is Firefox-308046B0AF4A39CB as shown in the picture below

3) Modify (Default) in HKEY_CURRENT_USER\Software\Clients\StartMenuInternet\ and enter the subkey found in step 2

That's all... I found the solution by monitoring registry access of google desktop via the Procmon tool from sysinternal.

Hope this will help other persons who had the same issue.

After having read your question, I found that the default in my startmenuinternet key is now "Avast secure browser" and google desktop still open links in firefox so I dont really know how all this works.

however I also found that there is another location in the registry where all browser are registered :
HKEY_LOCAL_MACHINE\SOFTWARE\Clients\StartMenuInternet

You might find all browser there, then some of the subkeys should be also in HKEY_CURRENT_USER\Software\Clients\StartMenuInternet

In my case I can see only firefox at this place.

You could try copying the subkey for your preferred browser from HKEY_LOCAL_MACHINE to HKEY_CURRENT_USER (export the key of your browser from HKLM\Software\Clients\ to a .reg file, then change paths HKLM to HKCU and reload, then check with regedit)

So it looks like the opening of the browser depends on two keys : HKCU\Software\Clients\StartMenuInternet and HKLM\Software\Clients\StartMenuInternet.

(EDIT) After additional testings, it appears that the results and options are always shown in the default system browser : if I put IEXPLORE.EXE or Google Chrome in the (default) for StartMenuInternet, it always open in Firefox which is my default browser, but if StartMenuInternet is wrong or empty on both HKCU and HKLM then Google Desktop cannot open the browser.
