By default, Visio will paste as a picture. But if you paste it as an "Enhanced Metafile", you can ungroup the resulting paste to get individual shapes. The mileage will vary depending on complexity of flowchart, and it might not remain a connected flowchart.

 

Steps:

4. select the resulting pasted shape, right click and select Group->Ungroup.

 

You should now get individual shapes.

karl timmermans [outlook mvp 2012-2018] say:

As Russ mentioned, how the various fields (FullName and FileAs) get populated depends completely on your default settings.

Also, if you are in the habit of entering the FullName value into the FullName field as opposed to entering the values into the appropriate individual fields (by clicking on the FullName button and then entering the data) - there is a very good chance that your names are incorrectly entered if the default for the FullName value is set to Last/First.

Specifically, if the FullName default setting is set for Last/First and you enter a name of "FName LName" into the FullName field, Outlook will parse the value as

LastName = Fname
FirstName = LName

 

You may find the standardization functions in ContactGenie Toolkit of interest for bulk changes to either of these fields (among many of its other functions)

http://www.contactgenie.com/toolkit_4_outlook.htm

*** one caveat, if the underlying data in the individual <Name> fields is incorrect, there is nothing in CG Toolkit that is automatically going to be able to correct that - i.e. if the individual First and Last Name values contain the incorrect values.

UPDATE: I finally managed to place macro buttons with custom icon images on the QAT (Quick Access Toolbar) by doing the following (but now have a new issue to deal with as explained below) :

 ** IMPORTANT: NEVER HAVE THE DOCUMENT YOU'RE WORKING ON OPEN IN BOTH UI EDITOR AND WORD AT THE SAME TIME! THIS CAN AND WILL CAUSE PROBLEMS SUCH AS LOSING THE CUSTOM XML CODE AND HAVING TO REDO IT. **

 1. Open a new document in Word and save it as a Word macro-enabled document (.docm).

 2. Close Word.

 3. Open the UI Editor (see download link at the end of this post).

 4. Open the Word document that you created in step one.

 5. Click on Insert on the menu and select "Office 2010 Custom UI Part" or "Office 2007 Custom UI Part"
   (choose the one that corresponds to the Office version that you're using).

 6. Click on the "Insert icons" button on the UI Editor toolbar to add the images that you want to use on the macro buttons.
    (In my case, I inserted three images named "Tab", "vertical" and "vertical2". It's best to use images that are 16px x 16px.
    for small buttons and 32px x 32px for large buttons. You can use the following types of images for button icons: PNG, JPG,
    BMP, WMF, ICO and GIF. My attempts to use ICO images have been unsuccessful). 

 7. In the right pane, add the following XML code:


< customUI xmlns="http://schemas.microsoft.com/office/2009/07/customui">
< ribbon>
< tabs>
< tab id="Icons" label="Images" visible="true">
< group id="Images" label="Icon Images">
< button id="Btn1" label="Tab" image="Tab"
onAction="RBX.ButtonOnAction"/>
< button id="Btn2" label="V Line" image="vertical"
onAction="RBX.ButtonOnAction"/>
< button id="Btn3" label="V Lines" image="vertical2"
onAction="RBX.ButtonOnAction"/>

</group>
< /tab>
< /tabs>
< /ribbon>
< /customUI>


The above code creates a custom tab on the Ribbon that has three buttons with custom icons. The custom Ribbon tab is going to be used to "store" the custom buttons/images that will be added to the QAT. The "label" attribute is where you enter the name that you want to appear on the Ribbon tab.

 8. Enter the name (id) of the icon image that you want to use for each button in the buttons "image" attribute. Use the buttons "label" attribute to specify the text that will be located below the button.    

 9. Click on the "Validate" button to verify that the code doesn't contain errors.

10. Click the Save button (or press Ctrl-S) and exit UI Editor.

11. Reopen the document in Word.

12. Press Alt-F11 to open the VBA (Visual Basic for Applications) program.

13. Click on the Project that has the document name next to it in parentheses to select it.

14. Select Insert > Module from the menu and rename the module "RBX".

15. Double-click the "RBX" module.

16. In the right-hand "General" pane, enter the following VBA code:


Sub ButtonOnAction(Control As IRibbonControl)
  Select Case Control.ID
        Case "Btn1"
    NewMacros.Macro10
        Case "Btn2"
    NewMacros.Macro3
        Case "Btn3"
    NewMacros.Macro4
  End Select
End Sub


Enter each buttons id (obtained from the XML code) after the Case statement and enter the name of the macro that you want the button to run on the next line.

Note: You have to include the Project name ("NewMacros" in the example) for the macro buttons to function. The macro names can be obtained by right-clicking on the Ribbon in Word and selecting "Customize the Ribbon". Then select "Macros" from the "Choose Commands" box to see a list of the macros.

17. Save the changes and close VBA.

18. Right-click on the Ribbon or QAT and select "Customize Quick Access Toolbar".

19. Select "All commands" from the "Choose commands from" box.

20. Scroll down the list of commands until you find one of the custom macro buttons that you want to add to the QAT and click on it to select it.

21. Click the "Add" button to add the button to the QAT.

22. Repeat steps 19 and 20 to add additional custom macro buttons to the QAT.

23. Use the arrow buttons on the right side to reposition the buttons on the QAT, if needed.

24. Click "Ok".

Note: If you want to hide the custom Ribbon tab after you're done adding custom buttons to the QAT you can change the "visible" attribute in the XML code to "false". This doesn't affect the custom buttons on the QAT, they'll still remain visible on the QAT.

jay freedman say:

The problem (from Word's point of view) is that the cover page from the template has the page number 0, and the first page of text (after a page break) is page 1. Therefore, if you insert an Odd Page section break at the end of the cover page, Word sees the following page 1 and doesn't add any blank page. If you insert another page with a page break or a section break, that becomes page 1 and forces the text to start on page 2.

To fix it, first undo whatever "fiddling" you've done -- if necessary, by starting a new document from the template and copying your text into it.

Next, switch to Draft mode, and click the ¶ button on the Home tab to display nonprinting characters. The start of the document should look something like this, but with your text:

Select the page break and replace it with an Odd Page section break.

Next, click in the empty paragraph at the top, which says "Header" on the left. Then click the Insert tab, click the Header button, and click Edit Header at the bottom of the gallery. Now that the header pane is open in the document, the Header & Footer Tools tab appears on the ribbon. Click the Page Number button and click Format Page Numbers in the menu. In the dialog that opens, click the Start At box and change it from 0 to 1; then click OK.

Because of the section break you inserted, the first page of text will also be page 1 (but in section 2). Don't change that.

Your report should now print as you want.
