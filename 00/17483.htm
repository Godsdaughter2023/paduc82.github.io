a. user say:

This is either a well-known problem or a new variation on it.

SELECT * FROM "CLIENTS"

to

SELECT % FROM "CLIENTS"

What they probably mean is that if your "Clients" object is a View (in this case, an Access Query) and you have wildcards in it, e.g.

SELECT * FROM Clients WHERE Name Like "Sm*"

then you need to use  the following instead:

SELECT * FROM Clients WHERE Name Like "Sm%"

and/or if you have the single-character wildcard "?" and you have

SELECT * FROM Clients WHERE Name Like "Smit?"

then you need

SELECT * FROM Clients WHERE Name Like "Smit_"

instead.

a. user say:

The SQL *in Word* should just retrieve all the records returned by the query (view) in your Database.

But if the Query in your database uses the old "Jet" wildcards ? and *, it will not return any records via the OLE DB (or ODBC) connection that Word uses.

So if that is the situation, there are at least three possibilities. None of them is particularly straightforward. I suppose I favour (b) if you are in a position to modify your database in Access 2013.
 a. modify the query in your database. As far as I know, to do that you would need to open the database in Access 2013, modify the SQL of the query and save the modifications.
 b. make a copy of the query inyour database, by opening it in Access 2013, copying the query, then modifying the copy. Or create a new one and copy and modify the SQL. Then modify your .docx to get its data from that query.
 c. modifying the .docx so that it has the same SQL as the query (but with the % and _ wildcards instead of the, so that if your database query is called "Clients" and contains

SELECT * FROM Sometable WHERE Name = 'Sm*'

you would change the query in your .docx from

SELECT * FROM Clients

to

SELECT * FROM Sometable WHERE Name = 'Sm%'

You could do that directly in the .docx as you have done (but notice that the query may be in more than one place in the .docx, possibly in two places in the settings.xml part within it), or you could issue some VBA to do it. It may be enough to open the document, open the VB Editor, open the Immediate Window and execute the following (substituting your SQL, of course).

ActiveDocument.MailMerge.DataSource.QueryString = "SELECT * From Sometable WHERE Name = 'Sm%'"

I do not know which of those would be simpler for you.
