ossiemac say:

Hello Renee,

I have uploaded a zipped file to the following link. Have a look and get back to me if you want any changes.

http://1drv.ms/1hPhaNY

I changed the Search box from a drawing text box to an ActiveX textbox otherwise if the user did not exit the textbox box and select another cell on the worksheet before clicking the Search control then the search box remained selected and basically it did not work.

Search criteria is NOT case sensitive so can use either upper or lower case.

It allows for the user to enter a minimum of text for the search and the system automatically adds both leading and trailing wild cards and sets the filter to cells containing the entered text.

The user can also enter further wild cards so say the user wants "SMART Lab - Accredited Nursing" from the Meeting Type field then simply entering "smart*nurs" (without the double quotes) will find all "SMART Lab - Accredited Nursing".

If the user wants only records where "SMART Lab" will match exactly then enclose the search criteria in single quotes like 'smart lab' and then only fields that exactly match "smart lab" will be returned. 

The following code for information so it can help others with similar problems.

Sub SetFilters()
    Dim shp As Shape
    Dim strCaption As String
    Dim rngFiltHead As Range
    Dim lngFiltCol As Long
    Dim strFilter As String

    If Trim(Activesheet.OLEObjects("txtSearch").Object.Value) = "" Then
        Beep
        MsgBox "Please enter a search value." & vbCrLf & _
                "Processing terminated."
        Exit Sub
    End If

    For Each shp In Activesheet.Shapes
        If shp.Type = msoFormControl Then
            If shp.FormControlType = xlOptionButton Then
                'Next line assigns the option button caption to a variable
                If shp.ControlFormat.Value = 1 Then 'Equals 1 when selected
                    strCaption = Trim(shp.TextFrame.Characters.Caption)
                    Exit For
                End If
            End If
        End If
    Next shp

    'Find the column header that matches the Option button caption
    With Activesheet.AutoFilter.Range.Cells
        Set rngFiltHead = .Find(What:=strCaption, _
                        LookIn:=xlFormulas, _
                        LookAt:=xlPart, _
                        SearchOrder:=xlByColumns, _
                        SearchDirection:=xlNext, _
                        MatchCase:=False)

        If Not rngFiltHead Is Nothing Then
            If Activesheet.AutoFilterMode = False Then 'If autoFilter NOT turned on
                rngFiltHead.AutoFilter      'Turn AutoFilter on
            End If
            lngFiltCol = rngFiltHead.Column 'Save the Column number for setting the filter
        Else
            Beep
            MsgBox "Problem. The header indicated by option button caption not found." _
                    & vbCrLf & "Processing terminated."
            Exit Sub
        End If
    End With

    strFilter = Trim(Activesheet.OLEObjects("txtSearch").Object.Value)
    If InStr(1, strFilter, Chr(39)) > 0 Then   'If enclosed in single quotes
        'Set filter to exactly search criteria
        strFilter = "=" & Replace(strFilter, Chr(39), "", 1)
    Else
        'Following line adds leading and trailing asterisk wild cards for search
        strFilter = "=*" & Activesheet.OLEObjects("txtSearch").Object.Value & "*"
    End If

    'In next line "AutoFilter.Range" is the generic range for the AutoFilter so _
     it does not matter if additional rows are added (or rows deleted)
    Activesheet.AutoFilter.Range.AutoFilter Field:=lngFiltCol, Criteria1:=strFilter
    If WorksheetFunction.Subtotal(3, Activesheet.AutoFilter.Range.Columns(lngFiltCol)) = 1 Then
        Beep
        MsgBox "Filter did not find anything. Re-enter search filter." _
                & vbCrLf & strCaption & " filter will be cleared."
        Activesheet.AutoFilter.Range.AutoFilter Field:=lngFiltCol   'removes filter if nothing found
    End If
End Sub

                                           

Sub ClearFilters()
    With Activesheet
        If .AutoFilterMode Then
            If .FilterMode Then
                Activesheet.ShowAllData
            End If
        End If
    End With
    Activesheet.OLEObjects("txtSearch").Object.Value = ""
End Sub

anwu say:

Crazy mouse pointer on ultra HD (4K) monitor and 200% scaling - critical bug

First I have to explain what is "crazy mouse pointer". It depends on what kind of mouse cursor is selected in Windows (mouse pointer) configuration (default, big, black, ...).

  • it can be invisible on white background (pointer border is not drawn)
  • it is green or pink instead of black and very narrow
  • it is very narrow and some crazy graphics is drawn to the right of mouse pointer (half of circle or rectangle with some random filling); this half of circle is from "busy pointer"
  • standard arrow pointer is correct, but other shapes are crazy (for example when you try to resize column in listbox or when mouse pointer is over MS Excel spreadsheet)

Generally pointer is much better when option "mouse pointer shadow" is turned on, but there is no single setting when it is correct!!! I see this strange behavior on MSI GTX 970 Gaming 4G graphic card and ultra HD (4K) monitor connected to HDMI output. Refresh rate is 60Hz. UI scaling 200%. This problem exists in Windows 8, 8.1 and now in 10. I see it on every graphic card driver, starting from initial release, be it from MSI or directly from nVidia.

There are a few solutions to this problem (each one is unacceptable).

  • change screen refresh rate to 30Hz or slower
  • change UI scaling to 300%
  • change screen resolution to 2560x1440 or lower

joeu2004 say: Neil wrote:
For example,

Actually, I think you have the examples reversed.  For me:

1. 65.57 - 58.03 - 7.54 = -7.99361E-15
2. 65.57 - 7.54 - 58.03 = 0
3. 65.57 - 7.54 - 58.03 - 0 = -7.10543E-15

I added #3 to make a point:  in this case, #2 results in exactly zero only because Excel forces it to be zero due to the dubious heuristic poorly described under the misleading title "Example When a Value Reaches Zero" at http://support.microsoft.com/kb/78113.

As I indicated previously, the problem is that none of those numbers are represented exactly.  They are:

65.57:  65.5699999999999,9317878973670303821563720703125
58.03:  58.0300000000000,01136868377216160297393798828125
  7.54:  7.54000000000000,003552713678800500929355621337890625

The order of operations makes a difference only because of the differences in the approximate representations of the intermediate results (pairwise operations).  So:

65.57-58.03:  7.53999999999999,2041921359486877918243408203125
65.57-  7.54:  58.0299999999999,9403144101961515843868255615234375

relative to the last operand, the latter being closer to 58.03 than the former is to 7.54.

Excel forces #2 to zero only because the representation of 65.57-7.54 is "close enough" to the representation of 58.03 that Excel decides
But as #3 demonstrates, Excel does that only if the last subtraction (or addition of operands with opposite signs) causes a change and is "close enough" to zero.  Since "- 0" does not cause a change, Excel does not employ the heuristic.  (Klunk!)

vba macro

Option Explicit

' PGC - AUG 2016
' Permutation with repetition and with restrictions.
' A set of distinguishable objects is given, as well and the number of times each one is repeated
' Ex. Permutations with repetition with 2 "a"'s, 1 "b" and 2 "c"'s
' Input in a nx2 table, first columns the elements and second column how many times they repeat

'

'

Sub PermMultRep()
Dim vIn As Variant, vPerms As Variant, vPerm As Variant
Dim lRow As Long
'
'###tasosk
If Range("A2", Range("A2").End(xlDown)).Rows.Count > 9 Then
MsgBox "wrong, max number of entries is 9"
Exit Sub
End If

Range("A2", Range("A2").End(xlDown)).Offset(, 1) = 1 'qty =1

'###
'
vIn = Range("A2", Range("A2").End(xlDown)).Resize(, 2).Value ' table of elements and number of times they repeat
ReDim vPerm(1 To Application.Sum(Application.Index(vIn, 0, 2))) ' array for the current permutation
ReDim vPerms(1 To Application.MultiNomial(Application.Index(vIn, 0, 2)), 1 To UBound(vPerm)) 'array to store all permutations

PermMultRep1 vIn, vPerm, vPerms, 1, lRow ' calculate all the permutations into the vPerms array

'Columns("D").Resize(, UBound(vPerm) + 1).Clear ' clears columns for the output
Range("D2").CurrentRegion.ClearContents ' tasosk
Range("D2").Resize(UBound(vPerms, 1), UBound(vPerms, 2)).Value = vPerms ' writes the output in D2, down and across
End Sub

'

'

Sub PermMultRep1(ByVal vIn As Variant, vPerm As Variant, vPerms As Variant, ByVal lInd As Long, lRow As Long)
Dim j As Long, lCol As Long
Dim v1 As Variant

For j = LBound(vIn, 1) To UBound(vIn, 1)
    If vIn(j, 2) > 0 Then
        vPerm(lInd) = vIn(j, 1)
        If lInd = UBound(vPerm) Then
            lRow = lRow + 1
            For lCol = 1 To UBound(vPerm)
                vPerms(lRow, lCol) = vPerm(lCol)
            Next lCol
        Else
            v1 = vIn
            v1(j, 2) = v1(j, 2) - 1
            PermMultRep1 v1, vPerm, vPerms, lInd + 1, lRow
        End If
    End If
Next j
End Sub
