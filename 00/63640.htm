3.    Click on “Properties” on the right click menu.

WLMP - XML - save file corrupted mixed, how to correct XML project - Microsoft Community

narcis d. say:

WLMP - XML - save file corrupted mixed, how to correct XML project manually

Hello,

I used more versions of Movie Maker and now I'm using the latest - 2012 (v16.4.3528.0331)

On some projects when apply an split operation, the movie shrinks a lot.

Saving the project before this operation and after that, I can see that in the WLMP file - the problem is in the XML structure of the file.

It seems that some tags are getting wrong values, for example inTime and outTime, more often outTime is smaller than it should be, that is why the movie shrinks.

Sometimes outTime is 0, maybe this means till the end of that media, but I am not sure.

Does anyone encountered such type of problems? And how this can be solved?

Thank you!

considerate_guy say:

It's a known issue and I've fixed numerous corrupted files. I really wish that Microsoft would acknowledge the issue and provide a fix.

I'm not sure of the cause, but I do know the symptoms; instead of writing the outTime, it writes the amount of time left in the clip.

For example, if you have a 10 second clip and you had it going from the 6 second mark to the end, it should write --> inTime="6" outTime="10". But instead, it writes inTime="6" outTime="0".

Likewise, if you had a clip from the 6 second mark to the 9 second mark (a 3 second clip), it ends up writing inTime="6" outTime="1".

To manually fix this, you have to look at the value of mediaItemID and find the corresponding mediaItem near the top of the file. You'll find the name of the file and its duration. Copy this duration into the outTime. Obviously if you plan to attempt this, be sure to work in a copy of the .wlmp file, not the original.
