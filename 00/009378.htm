Declare Function ExtractIcon Lib "shell32.dll" Alias "ExtractIconA" _
                             (ByVal Instance As Long, ByVal ExeFileName As String, ByVal IconIndex As Long) As Long
Declare Function SendMessage Lib "user32" Alias "SendMessageA" _
                             (ByVal hWnd As Long, ByVal Message As Long, ByVal wParam As Integer, ByVal lParam As Long) As Long
Const WM_SETICON = &H80

Public Sub SetAccessAppIcon(ByVal sIcon As String)
    On Error GoTo Error_Handler
    Dim lhWnd                 As Long
    Dim lIcon                 As Long

    lhWnd = Application.hWndAccessApp
    lIcon = ExtractIcon(0, sIcon, 0)

    If lIcon > 1 Then
        Call SendMessage(lhWnd, WM_SETICON, True, lIcon)
        Call SendMessage(lhWnd, WM_SETICON, False, lIcon)
    End If

Error_Handler_Exit:
    On Error Resume Next
    Exit Sub

Error_Handler:
    MsgBox "The following error has occured" & vbCrLf & vbCrLf & _
           "Error Number: " & Err.Number & vbCrLf & _
           "Error Source: SetAccessAppIcon" & vbCrLf & _
           "Error Description: " & Err.Description & _
           Switch(Erl = 0, "", Erl <> 0, vbCrLf & "Line No: " & Erl) _
           , vbOKOnly + vbCritical, "An Error has Occured!"
    Resume Error_Handler_Exit
End Sub

Please stay with this thread and do not re-ask this question.

It sounds like the defrag program you ran deleted or modified the relevant registry key, but sort order of Favorites can change randomly. This has been happening for years.

Solutions:

1. Don't run Micro Defrag

2. Use enough sub-folders in Favorites that sort order is less important. If you can see all Favorites on one screen, an individual Favorite will be easier to find.

3. Use naming tricks to make alphabetical order work for you. Example:

1yahoo.com will come before msn.com. Or  _bing.com will come before apple.com.

4. You can export the relevant registry key every few days. Merging that exported key will restore the previous sort order anytime that gets changed.

To export that key...

Go to Start> Run, type in regedit and hit Enter.  

You navigate the registry just like you would drill down through a folder structure to get to a specific file (key.) Just click the + to open the next level.

You need to get to the User Shell Folders key.

Follow this path:

HKEY_CURRENT_USER

   Software

      Microsoft

         Windows

             CurrentVersion

                 Explorer

                    Menu Order

                       Favorites

Select (left-click) the Favorites key.

Right-click that key and select Export.

Select the directory you want to save it to and Save it.

Right-click > Merge on that REG file will restore that version of Favorites sort order.

    jay freedman say:

    Your recorded macro doesn't contain any executable statements. When you run it from a shortcut or by any other means, it does run but it doesn't do anything.

    There are many things in Word that the macro recorder can't record, and changing the text color by using the Color button on the Home ribbon is one of them. If you had instead changed the font name or size, or entered new text, those changes would have been recorded and could be repeated by running the macro. Most operations related to pictures and graphics cannot be recorded.

    To get the recorder to produce a usable text-color macro, you must record selecting the color in the Font dialog. That records every item in the dialog, including all the ones that you didn't change. That could lead to unintended changes -- for example, it would change the font name and size to whatever was present at the time of the recording, which could be different from those in the selected text when you run the macro later. To make the macro useful, you would have to remove every setting in it except the one for the color.

    If you want a predictable macro to change the text color to green, it would be best to create the necessary line of code manually in the macro editor:

    Sub Green()

        Selection.Font.ColorIndex = wdGreen
    End Sub

    For a longer discussion, see How to Modify a Recorded Macro.

    milagrosm2020 say: Hi,

    It is very common to receive the error message 0x800CCC0E when you set up an email account in Outlook and it is about sending an email, the reason you receive the error is because Outlook does not authenticate your account on the server while sends, so the outgoing mail server rejects the message.

    To solve it, you just have to follow the following steps:

    Open Outlook and go to: Accounts -> Properties -> Servers.

    Once you are in this section, you will have to check the option "My server requires authentication".

    Now you must write your full email account and the password you chose when creating it.

    Finally, press the "Apply" button and the "OK" button, you have now finished configuring the email account in Outlook. Now try to send an email and you should have no problem doing it.
