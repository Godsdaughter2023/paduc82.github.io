how do I get rid of extra page in a document? - Microsoft Community

heatherfrne say:

how do I get rid of extra page in a document?

I have a one page document that had gone onto two but I shortened it to one.  Now I can't remember how to get rid of the extra page.  Thanks in advance!
bob jones aka: cybertaz say:
There are 2 keys to resolving the issue... One is conceptual, the other is physical.

The conceptual is that there are no pages in the structure of a Word document. Microsoft has gone out of their way to show you how the document will appear when printed based on how many sheets of paper will be required because that's what users want to see. That gives the impression that pages are there but what you really have is nothing but content. The volume of content & how it is formatted combined with the Page Setup specifications determine how many sheets will be needed when printing takes place. Long story short, something in the content of your document is giving Word the impression that an additional sheet will be required.

The physical is to become comfortable with the use of what are referred to as non-printing characters in order to better understand what causes a document to flow as it does. This includes such things as carriage returns, tabs, spaces section breaks, etc. Since they do not print [because they are "white space"] they cannot be seen unless you choose to display them. To do so, click the ¶ button on the Standard Toolbar or use the keystroke Command+8 to toggle them On/Off.

Once you have the non-printing characters displayed you'll better be able to determine what is at the bottom of (or elsewhere on) the preceding page that is forcing at least one ¶ [empty paragraph] to display at the top of the 'blank' page. Eliminate that extraneous content and the blank page will evaporate.

Additionally, changing font size, margins & other formatting/layout attributes can be factors.

Const MESSAGETEXT = "Monthly backup has not been undertaken."
Dim strCriteria As String

strCriteria = "Year(BackupDate) = " & Year(VBA.date) & _
    " And Month(BackupDate) = " & Month(VBA.date)

If Nz(DMax("Format(BackUpdate,""yyyymm"")","Backups"),"0") <> Format(VBA.Date,"yyyymm") Then
    MsgBox MESSAGETEXT, vbInformation, "Warning"
End If

Assuming it is the back end to be backed up (backing up the front end before any changes to its design should be the responsibility of the database administrator, presumably you, not individual users) then the following module can be used:

Option Compare Database
Option Explicit

Declare Function CopyFile& Lib "kernel32" Alias "CopyFileA" (ByVal _
lpExistingFilename As String, ByVal lbNewFileName As String, ByVal _
bFailIfExists As Long)

Public Sub MakeFileCopy(strExistingFile As String, _
                   strNewfile As String, _
                   blnDoNotOverWrite As Boolean, _
                   Optional blnShowMessage As Boolean = False)
                  
                  
   Dim strMessage As String
                  
   strExistingFile = strExistingFile
   strNewfile = strNewfile
  
   If CopyFile(strExistingFile, strNewfile, blnDoNotOverWrite) = 1 Then
       strMessage = "File successfully copied."
   Else
       strMessage = "File copy failed."
   End If
  
   If blnShowMessage Then
       MsgBox strMessage, vbInformation, "Copy File"
   End If
  
End Sub

Public Sub BackUp(strBackEnd As String, strBackUp As String)

   Const FILEINUSE = 3356
   Dim strMessage As String
   Dim strBackUpTemp As String
  
   ' if back up file exists get user confirmation
   ' to delete it
   If Dir(strBackUp) <> "" Then
       strMessage = "Delete existing file " & strBackUp & "?"
       If MsgBox(strMessage, vbQuestion + vbYesNo, "Confirm") = vbNo Then
           strMessage = "Back up aborted."
           MsgBox strMessage, vbInformation, "Back up"
           Exit Sub
       Else
           ' make temporary copy of backend file and then delete it
           strBackUpTemp = Left(strBackUp, InStr(strBackUp, ".")) & "bak"
           MakeFileCopy strBackUp, strBackUpTemp, False
           Kill strBackUp
       End If
   End If
  
   On Error Resume Next
   ' attempt to open backend exclusively
   OpenDatabase Name:=strBackEnd, Options:=True

   Select Case Err.Number
       Case 0
       ' no error so proceed
       Application.CompactRepair strBackEnd, strBackUp
       If Err.Number = FILEINUSE Then
           ' file in use by current user
           strMessage = "The file " & strBackEnd & _
               " is currently unavailable. " & _
               " You may have a table in it open."
           MsgBox strMessage
           ' rename temporary copy of back up file
           ' if exists, back to original
           If Dir(strBackUpTemp) <> "" Then
               MakeFileCopy strBackUpTemp, strBackUp, False
               Kill strBackUpTemp
           End If
           Exit Sub
       Else
           On Error GoTo 0
           ' ensure back up file created
           If Dir(strBackUp) = Mid(strBackUp, InStrRev(strBackUp, "\") + 1) Then
               strMessage = "Back up successfully carried out."
               ' delete temporary copy of back up file if exists
               On Error Resume Next
               Kill strBackUpTemp
               On Error GoTo 0
           Else
               strMessage = "Back up failed."
               ' rename temporary copy of back up file
               ' if exists, back to original
               If Dir(strBackUpTemp) <> "" Then
                   MakeFileCopy strBackUpTemp, strBackUp, False
                   Kill strBackUpTemp
               End If
           End If
           MsgBox strMessage, vbInformation, "Back up"
       End If
       Case FILEINUSE
       ' file in use - inform user
       strMessage = "The file " & strBackEnd & _
           " is currently unavailable. " & _
           " It may be in use by another user."
       MsgBox strMessage
       ' rename temporary copy of back up file,
       ' if exists, back to original
       If Dir(strBackUpTemp) <> "" Then
           MakeFileCopy strBackUpTemp, strBackUp, False
           Kill strBackUpTemp
       End If
       Case Else
       ' unknown error - inform user
       MsgBox Err.Description, vbExclamation, "Error"
       ' rename temporary copy of back up file
       ' if exists, back to original
       If Dir(strBackUpTemp) <> "" Then
           MakeFileCopy strBackUpTemp, strBackUp, False
           Kill strBackUpTemp
       End If
   End Select
  
End Sub

To back up the back end you simply pass the path to the back end file and to the backup file to be created, e.g.

Backup "F:\Databases\SomeFolder\SomeFile.mdb", "F:\Databases\SomeFolder\SomeFile_bak.mdb"

Or you might want to automatically name the backup file by incorporating the current year and month in the file name.

If the backup is completed successfully insert a row into the backups table, which you can do with the following code:

Dim strSQL As String

strSQL = INSERT INTO Backups(BackupDate) " & _
    "VALUES(#" & Format(VBA.Date,"yyyy-mm-dd") & "#)"

CurrentDb.Execute strSQL, dbFailOnError

You could put this in the above BackUp procedure immediately after the strMessage = "Back up successfully carried out." line for instance.

A month between backups seems a long time, however.  We backup daily.

marianosilva say:

I was able to solve this issue with a Workaround... it looks like Sharepoint saves the TimeLine configuration somewhere as a flag in each task, but the problem arises when you save your "mpp" back and that synchronizes and overwrites such field.

The workaround is:

A) Finish any edits you were doing on MS Project. Save & Sync ( It's the file menu -> Save ) . This will destroy your timeline.

B) Go into the sharepoint and add all the tasks you want in your timeline as you would normally do

C) Open the Project file FROM THE SHAREPOINT ( not using a previous link, nor from the history in Project. Do this by going to the Sharepoint Task list Ribbon bar , clicking on LIST, then on "Open with Project". 

At this point you will have synced your task list in sharepoint with the "hidden" .mpp file that's stored in the SiteAssets folder

D) Do any changes you want in Project... those will reflect in your sharepoint list, and the TimeLine will not be altered.

You will have to do this each time you add or remove any "Task" from the "Show in Timeline".

Hope that helps.
