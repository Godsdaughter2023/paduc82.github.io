    john w. vinson say:

    Just for what it's worth, converting to .accde doesn't add functionality; it removes functionality - intentionally.

     

    An .accde conceals all the database's VBA code from view and prevents design changes to forms or reports; all of those design tools are simply turned off.

     

    The idea is to "lock down" the structure and function of the database, so that users can use the database as the developer intends, but cannot change (or, some developers would say, trash) the code or design of the user interface. It's a security and "self defense" feature. If you create a .accde you must absolutely keep the .accdb as well, since only it can be enhanced or improved.

    I've taken a short look in the thread you mention and I must say that using a UDF for this is a bad idea.

    Because if the UDF is running, the sheet is locked and normally it is not possible to insert or modify a shape (or something else) in that sheet. That this is possible with XL2010 is either a BUG or a new undocumented feature.

    But as you want to update the pictures in that sheet only from time to time, we can do the same with a normal macro. And if you like to run the macro automatically, we can use the Worksheet_Change event routine for this... later.

    So please make a backup of your file first, then remove your "ShowPicD" completly and run the sub DeleteAllPictures first.

    After that you need only the sub UpdatePictures, this sub imports, renames, moves and resizes the pictures.

    It may possible that a picture can't be renamed, because of an invalid filename, so please check out the cells in column A for a red interior. Try to rename the picture by hand and please rename the file and change the value in column A to get a stable update process.

    If you have a lot of red cells or if you can't rename the pictures for some reason, we can try an other way, e.g. write the name of the shape in the cell in column B and use this to identify the picture.

    Andreas.


    Option Explicit

    Sub DeleteAllPictures()
      Dim S As Shape
      For Each S In ActiveSheet.Shapes
        Select Case S.Type
          Case msoLinkedPicture, msoPicture
            S.Delete
        End Select
      Next
    End Sub

    Sub UpdatePictures()
      Dim R As Range
      Dim S As Shape
      Dim Path As String, FName As String

      'Setup the path
      Path = "J:\Shared\Store Material Projects\Al Mullins\Image Bank\"
      'You can read this value also from a cell, e.g.:
      'Path = Worksheets("Setup").Range("A1")

      'Be sure the path has a trailing backslash
      If Right(Path, 1) <> "\" Then Path = Path & "\"

      'Visit each used cell in column A
      For Each R In Range("A1", Range("A" & Rows.Count).End(xlUp))
        'Try to get the shape
        Set S = GetShapeByName(R)
        'Found?
        If S Is Nothing Then
          'Find the picture e.g. "C:\temp\F500.*"
          FName = Dir(Path & R & ".*")
          'Found?
          If FName <> "" Then
            Set S = InsertPicturePrim(Path & FName, R)
          End If
        End If
        If Not S Is Nothing Then
          'Show the error if the name did not match the cell
          If S.Name <> R Then R.Interior.Color = vbRed
          With R.Offset(0, 1)
            'Move the picture to the cell on the right side
            S.Top = .Top
            S.Left = .Left
            'Resize it
            S.Width = .Width

            'Remove the aspect ratio by default if necessary
            'S.LockAspectRatio = False

            If S.LockAspectRatio Then
              'Make it smaller to fit the cell if necessary
              If S.Height > .Height Then S.Height = .Height
            Else
              'Stretch the picture
              S.Height = .Height
            End If
          End With
          'Move it behind anything else
          S.ZOrder msoSendToBack
        Else
          R.Offset(0, 1) = "No picture available"
        End If
      Next
    End Sub

    Private Function GetShapeByName(ByVal SName As String) As Shape
      'Return the shape with SName, Nothing if not exists
      On Error Resume Next
      Set GetShapeByName = ActiveSheet.Shapes(SName)
    End Function

    Private Function InsertPicturePrim(ByVal FName As String, ByVal SName As String) As Shape
      'Inserts the picture, return the shape, Nothing if failed
      Dim P As Picture
      On Error Resume Next
      'Insert the picture
      Set P = ActiveSheet.Pictures.Insert(FName)
      'Success?
      If Not P Is Nothing Then
        'Return the shape
        Set InsertPicturePrim = P.ShapeRange(1)
        'Rename it, so we can easily find it later
        P.Name = SName
      End If
    End Function

    There are 2 parts that you have to change.

    a.)
      'Visit each used cell in column A
      For Each R In Range("A1", Range("A" & Rows.Count).End(xlUp))

    Change the A to B or D as you like.
      For Each R In Range("D1", Range("D" & Rows.Count).End(xlUp))

    But you can also specify any range as you like
      For Each R In Range("D1:D20")
    to visit each cell in D1:D20 or
      For Each R In Range("A1:A20,D1:D20")
    to visit each cell in A1:A20 and then D1:D20

    b.)
      With R.Offset(0, 1)
      'Move the picture to the cell on the right side
    and
      R.Offset(0, 1) = "No picture available"

    R.Offset(0, 1) means the cell with offset 0 rows (same row) and +1 column. So if the FOR loop visits each cell in column D and you want to have the pictures in column C, use R.Offset(0, -1) instead.

    You can also use an other syntax instead of R.Offset(0, -1)
      Range("C" & R.Row)
