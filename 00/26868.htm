john mcghie say:
Well, allow me to add to the mystery :-)  Since I don't know the answer without seeing the document, I will describe how I would go about looking for it.

"The Open XML file (file name here) cannot be opened because there are problems with the contents. Details: No error detail available" -- That error message indicates that Word has not recognised the document as a WORD document.  It is either attempting to open it as an XML file, or the file is corrupt.

In your case, it is more likely that the file is corrupt.  And the cause is likely to be the graphics.  In order, I would suspect the Equations, the Charts, the Graphics, and the Tables.

One or more of them is corrupt, constituting "a problem with the contents".

Before you start, check the file name extension is exactly correct.  This problem can be produced if the extension is wrong or missing.  And ask yourself if anyone could have saved the document from anything other than Microsoft Word.  The competitors are good, but they're not perfect, especially with the complex stuff.

The quickest way to find out what it is, is to use a binary search: save a copy of the document, then cut it in half and save each half as a separate file.  Which one won't open?  The problem is in there.

Now split the half that won't open in two, and save each half.  Now which one won't open?  Keep at it until you reduce the half that won't open to just ONE of the suspect things.  Then you will know which one it is, go back tio the original, make another copy and chop out the suspect item.  Will it save and open now?

If this is all too much of a fiddle, you can make a copy of the document and simply remove all the Usual Suspects!  For example, rake out all of the Equations, then Charts, then Graphics, and then Tables.  You can use the Browse button (Next command) to find each one in turn.  After deleting all of each type, save AND CLOSE the document, then try it.  Unless you close the document, Word will not clean up the internal code to remove the deleted material.

Once you know what kind of item is corrupt, you can usually form a suspicion about which one, and usually find the culprit in three to five replacements.

Another way you can deal with this is to open the thing in Word 2010, and do a Save As to a different file name.  If Word 2010 has the latest updates, it has a "file fixer" embedded in it which will often deal with this problem.  Avoid Word 2007, particularly if you cannot be sure that it is right up-to-date.  Early versions of Word 2007 contained a bug in the XML writer that would CAUSE this problem.

Note that the problem may not be with the equation, table or graphic itself.  It is often the case that the boundary between the text either side of the object is where the damage occurs.  So it can be the paragraph either side of the table or graphic.

Hope this helps
