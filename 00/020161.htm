Sort Macro Multiple Criteria - Microsoft Community

crewfan say:

Sort Macro Multiple Criteria

I have an excel workbook that I’d like to write a macro to custom sort part of my range based on one criteria and the other part based on different criteria:

 

Columns

Location, File Date, Sale Date

 

Criteria 1

Select rows where location says: “District 1” or “District 2”

Sort these rows by “File Date”

 

Criteria 2

Select rows where location says “District 3” or “District 4”

Sort these rows by “Sale Date”

Thanks!

a. user say:

Assuming that the active sheet contains the data, and that Columns A through C contain the data, the following macro will filter the Location for "District 1" or "District 2", and sort the filtered data by File Date, in ascending order...

Option Explicit

Sub FilterAndSort()

    Dim LastRow As Long
    
    With ActiveSheet
        If .FilterMode Then .ShowAllData
    End With
    
    LastRow = Cells(Rows.Count, "A").End(xlUp).Row

    With Range("A1:C" & LastRow)
        .AutoFilter field:=1, Criteria1:="District 1", Operator:=xlOr, Criteria2:="District 2"
        .Sort key1:=Range("B2"), Order1:=xlAscending, Header:=xlYes, _
            OrderCustom:=1, MatchCase:=False, Orientation:=xlTopToBottom, _
            DataOption1:=xlSortNormal
        '.Copy Worksheets("Sheet2").Range("A1") 'change accordingly
        '.AutoFilter 'clear the filter
    End With
    
End Sub

Note that if you'd like to copy the filtered data to another worksheet and clear the filter, uncomment (remove the first apostrophe's from) the last two lines of code within With/End With.

Hope this helps!

One possible approach is to use the autofilter.  Try the code below.  It assumes you have a header row and 49 data points, and that your data start in A1.  Adjust accordingly.

Eric

'***** BEGIN CODE *****

Sub DoubleSort()

'

    If (Not ActiveSheet.FilterMode) Then ActiveSheet.Range("A1:C50").AutoFilter

    ActiveSheet.Range("$A$1:$C$50").Autofilter _

        Field:=1, Criteria1:="=District 1", Operator:=xlOr, Criteria2:="=District 2"

    ActiveSheet.AutoFilter.Sort.SortFields.Clear

    ActiveSheet.AutoFilter.Sort.SortFields.Add Key:=Range("B2:B50"), _

        SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal

    With ActiveSheet.AutoFilter.Sort

        .Header = xlYes

        .MatchCase = False

        .Orientation = xlTopToBottom

        .SortMethod = xlPinYin

        .Apply

    End With

    ActiveSheet.Range("$A$1:$C$50").AutoFilter _

        Field:=1, Criteria1:="=District 3", Operator:=xlOr, Criteria2:="=District 4"

    ActiveSheet.AutoFilter.Sort.SortFields.Clear

    ActiveSheet.AutoFilter.Sort.SortFields.Add Key:=Range("C2:C50"), _

        SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal

    With ActiveSheet.AutoFilter.Sort

        .Header = xlYes

        .MatchCase = False

        .Orientation = xlTopToBottom

        .SortMethod = xlPinYin

        .Apply

    End With

'

    ActiveSheet.ShowAllData

'

End Sub

Any hints?

Thanks.

pamcaswell say:

In addition to the before and after spacing set to 0, the heading line spacing plus the space before and after it must be a multiple of the body text line spacing. If you are using Word's singe spacing, you must calculate what the line spacing is. I have used 120% of the font size. I don't believe I ever seen this information from MS, but  I have from other Word users.  For me, it has worked for font sizes from 8 to 12 points.  The other option is to set the  line spacing yourself to "exactly".  Also, to stop Word from  adjusting spacing around headings, in layout options (Word options > Advanced > Compatibility options), put a check the box before "Don't use HTML paragraph auto spacing".

This is all doable in Word and only a little more onerous than spec'ing type back in the day.  But, particularly if you have figures, tables, and other displayed objects, it might be easier in a publishing program.
