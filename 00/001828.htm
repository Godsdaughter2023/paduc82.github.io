albert d. kallal say:

I seen this happen - it is sort of a bug.

 

From the debug window, simply delete the menu with the following command:

 

commandbars("YouMenu").Delete

 

The above should fix this. (the add-ins in the ribbon should instant go away).

 

If you launching a form, then REMEMBER check the "other" tab in the properties sheet to ensure you do not have a menu specified for the form.

 

If this is a mdb, then in file->options->current database, you WILL see the extra drop down combo to set the startup menu, but if you convert to accDB, that drop down is gone, and you cannot erase it. - it should show, but it does not ;-)

 

If you deleted the menu bar using above, then you could also attempt to delete the startup property with this:

CurrentDb.Properties.Delete("StartUpMenuBar")  


However, to my knowledge, just deleting the menu bar should do the trick.
