chrisclaessens say:

We finally got this problem figured out with the assistance of Office 365 Escalation Team (Cecil Yan). Just wanted to share the solution here in case anyone else is breaking his head over the same issue.

The issue is caused by O365 integration on a server that is either running Windows Server Essentials, or has the Essentials role installed. After every Server Essentials synchronization cycle it reverts all changes to users made by Powershell or in the O365 online admin centre to the original values again. The solution in this case was to use ADUC on the server to adjust the display name of the users, and wait for the O365 synchronization cycle to propagate the changes to O365. More details can be found in this KB article: support.microsoft.com/.../2992846

winni liang -msft say:

Hi,

I tested in my Visio 2013 Professional application. Followed your steps and I can get the same results as yours. The Pasted page was shown Page-1 instead of Page-2. But this is not an "issue" in Word or Visio. It is expected behavior.

In Word 2013, when you right-click on a blank page and selects 'paste', the Visio drawing is pasted as a Microsoft Visio Drawing Object by default. The whole drawing (Page-1, Page-2 and VBackground-1) is pasted as a Visio drawing and Page-1 is show by default for this drawing in Word. Moreover, you can double-click the pasted object to directly open a Visio window, switch drawing pages to Page-2 manually.

Additionally, if you don't want to paste the whole Visio object and only want to display Page-2 in Word, please paste it with the following actions:

Open a blank document, go to Home tab, in the Clipboard group, click Paste > Paste Special, select paste as Picture instead of Visio drawing object to have a try.

It is not an issue which is caused by any updates. Sorry for the confused information and any inconvenience about it. If you need any further assistance on this issue, feel free to let me know :)

Winni
