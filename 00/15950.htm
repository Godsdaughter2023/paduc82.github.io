I have it working on an Aspire One AOA150.  You need to do a simple registry edit then away you go!

  1. Press the Windows key + R simultaneously to launch Run.
  2.  Type in regedit and press Enter to launch the Registry Editor.
  3. Press Ctrl + F simultaneously to open the Find dialog box.
  4. In the Find dialog box, type in this: Display1_DownScalingSupported then click on Find Next button and Windows will automatically search for you the registry key and high light it for you.
  5. Click on the high lighted key (Display1_DownScalingSupported) and change the Value from 0 to 1.

  7. Once finished, exit the Registry Edit, restart your PC.
  8. After you've restarted, go to your Desktop, right mouse click, choose Screen Resolution and you should be able to choose 1024x768. Please choose this option, save and close.

steve rindsberg say:

Hi Cody,

That article is incomplete.  It mentions passwords and DRM applied to media and is accurate as far as that goes.  But to add to that, Windows versions of PPT since 2003 or perhaps 2007 have been able to apply a form of DRM to PowerPoint files. 

This requires special software (generally a server on the network).  A user whose organization has this software can apply DRM to a presentation to restrict opening the file not just by "Who has the password" but by "Who is this" ... user by user, if desired. 

IIRC, Mac PPT doesn't yet support this in any version, so a file with this form of DRM applied wouldn't be openable in Mac PPT.  Ever.

The Mac user would have to request a non-password protected, non-DRM protected version of the file from the author. 

If they're unwilling to provide that (which they might well be, given that they've protected it in the first place), a PDF might be an acceptable alterntative.  This could be password protected but still openable on Macs.

These normally contain a Microsoft address and are considered extremely suspicious if a different or unknown address is present.

 

Also, an equivalent HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Main key may be present for Search Page and Start Page, ‘Secondary’ keys may be present as well. All these keys should be checked for correctness, i.e. legitimate website addresses that, you, the user have created.

greg maxey say:

This might be easier.  Create your template with all the blah, blah guide text and the content controls for user to enter their data.  Bookmark each block of guide text (stuff you want to eliminate in the finished document).  Use any name for the bookmarks.  When the user is done compile the finished document using:

Sub ScratchMacro()
'A quick macro scratch pad created by Greg Maxey
Dim i As Long
For i = ActiveDocument.Bookmarks.Count To 1 Step -1
  ActiveDocument.Bookmarks(i).Range.Delete
Next i
End Sub

This will remove all the blah, blah text leaving only the text you want to remain.
