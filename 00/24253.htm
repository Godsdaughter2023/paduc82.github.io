How to filter a combo box in a subform based on the selection of a - Microsoft Community

socallmecrazy say:

How to filter a combo box in a subform based on the selection of a combo box in the subform

Access 2010 database.  I have a very basic form that uses a single table for 2 combo boxes that allow the user to select a purchase category combobox (hardware, software, etc.) and then a subcategory combobox (monitor, laptop, desktop if the first selection was hardware; Project, Acrobat, etc if the first selection was software).

 

The form works just fine stand-alone.  But, when I place the form as a subform in my main form, the subcategory combobox doesn't work.  I come up with a blank "list" and a prompt from Access to fill in the criteria for the subcategory criteria.

 

I'm quite sure it has to do with the fact that it is now a subform.  But I'm not clear how to properly reference the criteria for the subcategory combo box (which is a bound text box that contains the results from the category combo box selection).

 

TIA.

 

 

socallmecrazy say:

I finally figured it out! Thought I would share.

 

One thing that may help to make it clearer is to name the subform on the main form something different from it's "form name". Example: when I place my form on a another form (making it a subform), within the properties of the subform, I change it's name to "subNameOfMySubForm" rather than letting it default to "NameOfMyForm".

 

Now I have to change the criteria of the 2nd combobox within the subform while in design mode for the main form (we'll call it "frmMyMainForm"). The 2nd combobox is using as it's criteria the choice from the 1st combox.

 

If you use the Query Builder to assist with the Row Source for the 2nd combobox, the criteria will look something like this:

 

[Forms]![frmMyMainForm]![subNameOfMySubForm].[Form]![1stComboBoxChoiceField]

Voila! It works!!!

jmark@stayintouch.us say: After 20 hours of reading forums on the EXACT SAME issue you have as mine, your so simple explanation worked!  I am SOOOOOOO gratefull!   Thank you!

ken sheridan say: I must have missed this thread when it was first posted last year, but where you have two combo boxes in the same form, whether it's used as a parent form or subform, the RowSource for the second can be a lot simpler as you do not need to reference the parent form at all, so simply using:

[Form]![1stComboBoxChoiceField]

as the criterion is sufficient.  In fact you do not even have to include the Form property, the name of the control on its own will suffice, but I prefer to include the Form property to avoid any ambiguity over just what's being referenced as the parameter.

By doing this it will work whether the form is opened as a parent form, or as the source object of a subform control within another form.
