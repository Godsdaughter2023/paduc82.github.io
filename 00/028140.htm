rohn007 say:
I agree with Charles. Using the Master documents feature is STILL asking for document corruption.  If you insist on using Master / Sub documents, the number one thing you should do is MAKE FREQUENT BACKUPS!  You'll need them, sooner or later.  The "master documents" was originally  added to Word as a "me too" feature to fill out the marketing checklist.  While in WordPerfect the feature worked well (I used to use it in WP, and seriously missed it when I was forced to use MS Word), but MS's implementation was 'fast and dirty', emphasis on dirty.  Even after the functional code was re-written in 2007 it still does not work well (enough).


On a "current" computer (RAM and CPU), 2010 can handle surprisingly large files compared to Office 97 or 2003 on machines that were current back then. I have found that working in a single large file is feasible.


The way you describe the sub document working is FAD, Failing as Designed.  Sequential numbering is context sensitive. When you open a sub document, the page and heading numbering in a sub document is independent, so it resets to 1.  When you open the sub in the master, the numbering is set to match the context.
Understanding Master and Subdocuments

http://word.tips.net/T000221_Understanding_Master_and_Subdocuments.html

Summary: Most people use Word to create regular documents that you edit, view, and print. The program also allows you to create a special type of document relationship that involves master documents and subdocuments. This tip explains what these types of documents are and how they relate to each other.


If you need to edit and especially print sub documents independently, you have 2 option. To use the manual method of setting a starting point for numbering features like heading numbers and page numbers. But that is a PITA. Or, you can switch to numbering systems that use static text to help uniquely identify numbering that restarts at one:

ie Chapter 2 Page #
    Section 3 Chapter #

I understand that this can be a hard sell to users or managers who have need or at least "expectations" of creating large manuals with continuous numbering.  You have to educate them on the limits of Word.

<snip>

Contrary to popular opinion, Word has no integral means of automatically saving the current document, nor of backing-up the current document. What it does have is an option to save AutoRecover information after a specific and configurable interval and an option to save a backup copy of the previously saved version of the document with the default file extension of WBK (and although there is no reason to do so, Word also allows the option to change this extension to the user's choice). This is not a true backup in as much as it is not a copy of the current version of the document.

A function to create a true automatic backup is provided by the Save In Two Places add-in which is detailed in a separate page, There is also available from this web site a complementary add-in which provides numbered versions of a document. Again that has its own web page.
