dirk goldgar say:

That code you found is VB.Net, not VBA.  VBA is the core language of "classic" VB, which doesn't have the same features as VB.Net.  Specifically, it doesn't have the DataObject or Clipboard classes to manipulate as that code does.

You *could* write that code in VB.Net using Visual Studio, and compile it into a .Net assembly that is set to be COM visible. Then you could instantiate that class from your VBA code and call the class method to put the specified file on the clipboard.  However, that's more complicated than doing it all from VBA with Windows API calls, if you can get that to work.  I believe you should be able to get it to work, since whatever VB.Net is doing must ultimately be calling the Windows API to put the file on the clipboard.
