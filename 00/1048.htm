You can run 2010 and 2013 side by side, BUT they MUST be the same "bitness".

Yes, your Office license gives you the right to run 32 or 64 bit office without any extra charge.

You have to uninstall ALL 32 bit Office products (including older versions!) before you can install 64 bit 2013. Installing is the only way to "upgrade" from 32 to 64 bit Office.

Unless you have a specific need for 64 bit, you might as well stay with 32 bit for now. The main reason for going with 64 bit is if you are using large files, mostly Excel online downloads, that are approaching 2GB in size. Or if your files, again typicaly Excel, are using approaching 2GB or RAM.

Other than that there is no significant benefit to running 64 Bit Office. So, 64 bit excel tends to use more RAM than 32 bit.

ronrosenfeld say:

The requirement to not use a criteria header identical to a data column header is ONLY for criteria which consist of formulas.  Since your other columns do not consist of formulas, they MUST have identical headers.

 

That behavior was clearly documented in HELP for Excel 2003 (and earlier) and apparently dropped out of 2007 when the dummied it down. 

 

But the  examples in HELP for 2007 where formulas are used (real formulas, and not the text strings) do show that the headers for those criteria are NOT column data headers.

Hi Paul,

Just one remark that can be important for the OP. The formula makes difference in upper and lower case.

aABBCCDd returns 6

=COUNT(1/FREQUENCY(CODE(MID(UPPER(A1),ROW(INDIRECT("1:"&LEN(A1))),1)),ROW(INDIRECT("1:255"))))

returns 4.

I got this from somewhere (I forget where) for exactly this purpose.  One note capital and lowercase will count as separate characters.

This is an array formula so you have to enter using Ctrl+Shift+Enter

=COUNT(1/FREQUENCY(CODE(MID(A1,ROW(INDIRECT("1:"&LEN(A1))),1)),ROW(INDIRECT("1:255"))))
