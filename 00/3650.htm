Word 2008 bibliography shows full dates in Uzbek with йил - Microsoft Community

sashametro say:

Word 2008 bibliography shows full dates in Uzbek with йил (yil=month), missing "eds." for multiple editors reference

I recently ran into this problem (which apparently still exists with Word 2011) where full date references (with month, day, and year, e.g. for periodicals or internet site date-accessed) show up in the generated bibliography in a weird format (after you save the document and re-open - they show up correctly when you are working with a new document).

There were previous messages on the forums about this, which are no longer accessible except via Google's cache:

http://webcache.googleusercontent.com/search?q=cache:M9ozAwk-2U0J:www.officeformac.com/ms/ProductForums/Word/8040+site:officeformac.com+word+citation+%D0%B9%D0%B8%D0%BB
http://webcache.googleusercontent.com/search?q=cache:sktDeMVLlDgJ:www.officeformac.com/ms/ProductForums/Word/8148+site:officeformac.com+word+citation+%D0%B9%D0%B8%D0%BB

However, nobody was able to find a solution (apart from manually editing the generated references, which is a pain).  I have found a better work-around for this.

First (and best, if it works for you) is to use one of the open-source BibWord citation styles - these are generally free of all the Microsoft cruft in their built-in styles, and generally will work better (and are easier to modify to suit your needs).  You can find these styles at http://bibword.codeplex.com/ and if you use one, you will not have dates in Uzbek.

If you really want to use one of the built-in styles, you can edit the XSL file that defines the citation style, in order to work around the Microsoft Office bugs.

The XSL files are named APA.XSL, CHICAGO.XSL, etcetera, and are stored within the Microsoft Office "package contents" on MacOS.  See the BibWord installation instructions for more details on this http://bibword.codeplex.com/releases/view/15852 but typically the folder containing these styles can be found at:

/Applications/Microsoft Office 2008/Microsoft Word.app/Contents/Resources/Style/

and you can just copy out the style file you want to modify and edit it with TextEdit, putting the modified copy back into the original folder.  The solution to the Uzbek problem is simply to hard-code the locale ID on the full-dates, so the garbage locale ID that Word passes to the templates is ignored.  I have used 1033 [English (US)] as the replacement for $_LCID here but you can use another locale if you want your month names in German or whatever.  The following diff is for the Chicago style, but you can make equivalent changes in the other styles too, just search for b:DMY in the templates ending in _DMY that are actually used in the style (there are many templates, including ones for GOST, SIST, and ISO690 that are not actually supported for MacOS Word) - in Chicago style only templ_prop_APA_DateAccessed_DMY and templ_prop_Chicago_Date_DMY are actually referenced with "call-template" in the XSL style sheet.

         <xsl:with-param name="LCID" select="$LCID"/>
       </xsl:call-template>
     </xsl:variable>
-    <xsl:value-of select="/*/b:Locals/b:Local[@LCID=$_LCID]/b:APA/b:DateAccessed/b:DMY"/>
+    <xsl:value-of select="/*/b:Locals/b:Local[@LCID=1033]/b:APA/b:DateAccessed/b:DMY"/>
   </xsl:template>
 
   <!--
@@ -3581,7 +3583,7 @@
         <xsl:with-param name="LCID" select="$LCID"/>
       </xsl:call-template>
     </xsl:variable>
-    <xsl:value-of select="/*/b:Locals/b:Local[@LCID=$_LCID]/b:Chicago/b:Date/b:DMY"/>
+    <xsl:value-of select="/*/b:Locals/b:Local[@LCID=1033]/b:Chicago/b:Date/b:DMY"/>
   </xsl:template>

While fixing this annoyance, I was also able to fix a problem where the string "eds." was not properly internationalized or localized in Word 2008 for Mac (not sure about Word 2011) causing references to edited volumes with multiple editors (and no author) in Chicago style to look like this:

Editrix, An and Another Editor, . An Edited Volume About Something. Someplace: Some Publisher, 2001.

instead of

Editrix, An and Another Editor, eds. An Edited Volume About Something. Someplace: Some Publisher, 2001.
