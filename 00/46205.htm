DNS (Domain Name System) translates URLs into IP (Internet Protocol) addresses (and vice versa). For example, if you typehttp://www.microsoft.com into the address bar in your web browser, your computer sends a request to a DNS server. The DNS server translates the URL into an IP address so that your computer can find the Microsoft web server.

 

The error "DNS server isn't responding”, wireless internet access point isn't passing DNS to your laptop/computer.

 

You may try these steps in order to troubleshoot this issue.

 

a. Open Command Prompt.

b. At the command prompt, type the following command, and then press ENTER:
ipconfig /flushdns (there is a space between ipconfig and /)
It will show the message successfully flushed the DNS resolver cache.
c. At the command prompt, type the following command, and then press ENTER:
ipconfig /registerdns (there is a space between ipconfig and / )
d. At the command prompt, type the following command, and then press ENTER:
ipconfig /all (there is a space between ipconfig and /)

The ipconfig /all command displays Windows TCP/IP settings for all your network adapters.
If your IP address for your Local Area Connection is either 0.0.0.0 or 169.x.x.x (where x is any number), then your computer is not getting an IP address from the router.

• ipconfig /release in the command prompt and press ENTER

¡firedog say:

This is a problem I've not come across before. I can only make a few comments:

The WLM receive date

You will see two dates in the Windows Live Mail interface: Date and Sent. Sent is the one in the Date: header line, applied by the mail submission server. Date is, I think, the one on the latest (highest) Received: header line. The date when the message is actually received by Windows Live Mail might be much later than these, but it will be recorded as the Date Created of the .eml file it is saved as. The Date Modified of this file will change each time something happens to it. Here are the dates for a message in my inbox when viewed in the Windows 10 File Explorer:


(I had to go back a long way to find a message with an appreciable difference between Date sent and Date received)

So, the date seen in two-line view (View tab > Message list), or in the Date column in one-line view, is the first in the message headers. The syntax of a Received header is clearly laid down in RFC821; it consists at least of a from-host, a by-host, a semi-colon and a date-time group. Any optional information should be given Received line in your example headers is non-standard.  

 Received: from *****mail.ir2.yahoo.com by *****.mail.ir2.yahoo.com; Mon, 06 Mar 2017 13:02:47 +0000; 1488805367.191

It looks like Yahoo is misbehaving and as if Windows Live Mail is trying (and failing) to interpret that number as a date in accepted RFC2822 format.

Can you establish that it's only messages with something like ;1488805367.191 at the end of the highest Received header that demonstrate this weird behaviour?

* 1488805367.191 is the UNIX timestamp representing Monday 6 March 2017 13:02:47 +0000. It's actually the number of seconds since 1 January 1970. See here: Date format converter

lemp say:

There are several issues going on here.

It's quite risky to have only a single account on a Windows computer.  Every now and then Windows accounts become corrupted and if you don't have a "spare" account with computer administrator privileges, you will have a much more difficult time recovering.  Although all Windows computers have a built-in account named "Administrator" that default is disabled by default in Windows 7. 

Go to Control Panel > User Accounts and Family Safety > Add or remove user accounts.  You'll probably see two accounts like this, except your account won't say "Password protected":

It's a "best practice" to not use an "administrator" account for everyday use and to have your account password protected.  However, it's much more important to create a spare administrator account.  CLick the "Create a new account" link and follow the prompts.

If there is only one non-password protected account, Windows 7 automatically logs into that account ... which is what is happening for you now.  As soon as you either password protect your only account or add a second account, you'll get the "Welcome" screen.  To bypass that and automatically log on to your desired "working" account, follow the directions here --> http://www.sevenforums.com/tutorials/377-log-automatically-startup.html

It's often easier to start fresh with a homegroup because the default settings should permit you to share both files and printers.  You need to exit the homegroup on all 3 of the computers in order to make it go away. 

For reference see

http://downloadsquad.switched.com/2011/01/19/how-to-share-files-using-a-homegroup-on-windows-7/

http://windows.microsoft.com/en-us/windows/homegroup-help

If you don't want to do that, at least initially, go to the Network and Sharing center on the computer that has the printer connected to it and click the link to "Choose homegroup and sharing options."  On the "Change homegroup settings" screen --

  • are all the boxes under "Share libraries and printers" checked?  If not, check the appropriate boxes, click "Save changes"

Click the link to "Change advanced sharing settings."  On the next screen, the top section "Home or work (current profile)" should be open.  Examine the options and change as needed.  If any changes are made, click the "Save changes' button when done

  • network discovery should be ON
  • file and printer sharing should be ON
  • the top option under "Public folder sharing" should be selected ("Turn on sharing so anyone with network access can read and write files in the Public folders")
  • Media streaming is ON (although this is less important)
  • the top option under "File sharing connections" should be selected ("Use 128 bit encryption ...")
  • Password protection should be ON (I'm pretty sure that this will work even if you don't have a password; as a test, turn password protection OFF -- and if you are then able to share files and the printer, come back and turn it ON and see if that is what is blocking you
  • Allow Windows to manage homegroup connections should be ON

You should check these same settings on all 3 computers.

If you still have difficulties, see if running the "Homegroup troubleshooter" (at the bottom of the "Change homegroup settings" screen) provides any clues.
