Highlight multiple words - Microsoft Community

kabocca say:

Highlight multiple words

I want to highlight treaty and custom in a word file in one time. Is this possible with Find and Replace?

Thanks in advance

graham mayor say:

The short answer is 'no' - at least not in a single pass; but it is easy enough to do with a macro

Sub Highlight()
Dim vFindText As Variant
Dim oRng As Range
Dim i As Long
    vFindText = Array("custom", "treaty")
    For i = 0 To UBound(vFindText)
        Set oRng = ActiveDocument.Range
        With oRng.Find
            .ClearFormatting
            .Replacement.ClearFormatting
            Do While .Execute(FindText:=vFindText(i), _
                              MatchWholeWord:=True, _
                              Forward:=True, _
                              Wrap:=wdFindStop) = True
                oRng.HighlightColorIndex = wdYellow
                oRng.Collapse wdCollapseEnd
            Loop
        End With
    Next
lbl_Exit:
    Exit Sub
End Sub

I couldn't find a solution, but I came up with the following unelegant solution that is a work-around for me, but will be of limited utility:

 

a) Create all of your section groups on the PC using leading numbers like 1010, 1020, 1030, 1040, 1050, etc..

c) Write down the order in which they appear on SkyDrive notebook. 

 

d) Then, on the PC, rename the section groups as needed so the first one listed on SkyDrive is now called 1010 – whatever on the PC.  The second one listed on SkyDrive is now called 1020 – whatever on the PC.  Sync to SkyDrive (This sync may take a long time, as you’ve potentially re-created all section groups).  In my example, I created section groups 1010-Administration; 1020-Business Contacts; 1030-Clients; all the way up to 1160-Unfiled Notes

 

e) Go to SkyDrive, and all of the sections groups should now appear in alphabetic order, with 1010 – Administration listed first, 1020 - Business Contacts listed second, 1030 – Clients listed third etc.  After that’s confirmed, on the PC, many of your sections will now be in the “wrong” Section Group.  Move all of the sections from within each (now alphabetized by brute-force) section group to the proper section group. 

I have an important OneNote notebook with 16 static Section Groups, that probably won’t change frequently, so I’m making due for the time being, but this will be effective and only last as long as I don’t add a new section group.  Once I do, I'll have to re-do this tedious process.  It’s best to do this after you’re confident you won’t be adding section groups to the notebook. 

 

I hope Microsoft addresses the fact that the OneNote Web App doesn't display Section Groups in alphabetic-order someday soon.
