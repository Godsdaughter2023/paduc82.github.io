transfer autocorrect from 2007 to 2010 - Microsoft Community

abelardobulaclac say:

transfer autocorrect from 2007 to 2010

How do I transf autocorrect file from desktop on Vista/Word 2007 to laptop on Windows 7 ProfessionalWord 2010 
jay freedman say:
The official, complicated way is described at http://support.microsoft.com/kb/926927. It assumes that both computers use Word 2007, but the same technique works between 2007 and 2010 (except that you may have to open the 2007 Normal.dotm template in 2010 and click File > Convert to update the template and get rid of the Compatibility Mode setting).

The easier way is to download http://jay-freedman.info/AutoCorrect2007.zip and unzip it into your Templates folder on the computer with Word 2007. Open that template or make a new document based on it, and click the Backup button. Save the resulting document, and transfer it to the computer with Word 2010. Also put a copy of the template in the Templates folder on the 2010 computer, open it, and click the Restore button. In the dialog, select the document that came from 2007.

jay freedman say:
I've done a small experiment, and I think this will work:

Make copies of the backup document created by my AutoCorrect template, one copy for each language you use, putting the name of the language in the file's name. Delete the rows of the table that contain "True" in the right-hand column (these are the ones that are formatted and therefore stored in the Normal.dotm template, which should not be overwritten).

Open each document in turn, press Ctrl+A to select all the text, and apply the language for that file. If any of the entries need different spellings ("flavor" vs. "flavour", "itemize" vs. "itemise"), edit the text in the Value column. Save and close the document, then repeat for the other documents until they've all been modified.

Then run the restore function of the AutoCorrect template once for each of the backup documents. Because the correct language has been applied to each entry in the backup document, it will be stored in that language's .acl file.

When you've done that, you should find that all the entries work in all the languages; further, where there are per-language differences in the Value text, the proofing language applied at the insertion point governs which value is inserted.
