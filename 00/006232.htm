johnbinsc say:

As this post is over a year old, I'm sure the OP has already found his answer.  But since this thread is the one that comes up first when searching for "reconnect onedrive to existing folders" I thought I would try to answer it, since I just had the question come up today.

My situation:

Regular old OneDrive running under Windows 7 Home Premium (NOT OneDrive Pro as in the original question).

Somehow I disconnected OneDrive without knowing it.  I had a bunch of crud sitting on the keyboard of my laptop.  Sounds far fetched I know, but after that, OneDrive had disconnected.

Click on the OneDrive folder in windows explorer, and this "Set up OneDrive" program automatically starts up. 

Click Next and log into OneDrive with your existing credentials.  Next a couple of times more and it shows you your current OneDrive Folder and asks if you want to use that or use a different location (two buttons).  If you choose "Current Location", you get a warning popup about there being files already in the folder and that the contents of your OneDrive will be merged with the contents of the Folder.  This is what scared the OP and had me also worrying about re-down/uploading duplicate copies of everything, and then needing to manually delete all the duplicates etc.

That worry is unfounded.  If you reaffirm that you want to connect OneDrive with all the same folders in your current OneDrive folder on your PC, it just rechecks the sync status without down/uploading anything.  Like it should.  Wonderful!  Took just a few minutes and the OneDrive icon in the systray said "Up to date".

This re-sync apparently also works (according to another post somewhere out there)  if you need to reinstall Windows, or move your OneDrive Folder to a new drive/computer etc.  So long as the contents of the folders are the same, OneDrive will not down/upload anything or create any duplicates, it will just verify the sync.  I haven't personally tested this aspect however.  I expect that "the contents of the folders are the same" is the trick, in that all the dates attached to the files must also be the same as they were in the previous location... 

I hope this helps someone out there.

ossiemac say:

An alternative is to interrupt the Save and Save as in the Before Save event and call your routine to save the workbook. This same called code can be called from a button. Example below. Edit the Save Routine to your requirements.

 

Place the following code into ThisWorkbook module.

 

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
    On Error GoTo ReEnableEvents
    Application.EnableEvents = False
    Cancel = True
    Call SaveRoutine
ReEnableEvents:
    Application.EnableEvents = True
End Sub

 

Following code into a standard module

 

Sub SaveRoutine()
   
    On Error GoTo ReEnable
    Application.EnableEvents = False
    Dim strPath As String
    Dim strFileName As String
   
    strPath = "C:\Users\User\Documents\Excel\Test Macros\"
   
    strFileName = "Test Before Save Routine " & Format(Date, "yyyy-mm-dd") & ".xlsm"
       
    Application.DisplayAlerts = False    'Optional
   
    ThisWorkbook.SaveAs Filename:= _
            strPath & strFileName, _
            FileFormat:=xlOpenXMLWorkbookMacroEnabled, _
            CreateBackup:=False
           
    MsgBox "Saved as: " & ThisWorkbook.FullName
 
ReEnable:
     Application.DisplayAlerts = True    'Optional
     Application.EnableEvents = True
End Sub
