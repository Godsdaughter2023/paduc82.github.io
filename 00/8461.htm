a. user say:

The first thing I would consider doing is trying to transform the XML with the XSL Transform on the CFR site. It's reasonably easy and quite quick.

An XSL Transform takes an XML document and transforms it into something else, in this case, an HTML document. You can use this to open a formatted document in Word. What I do not know is whether this will help with the chunk of CFR that you want to format.

If you open https://www.gpo.gov/fdsys/bulkdata/CFR/resources you should see a file called cfr.xsl. Download that and make sure you know where to find it on your computer (it may help to copy it to a folder with a name you can remember easily).

Make sure your .xml file is closed.

You have to open this file twice to make this work, so do not be dismayed when you open something that looks like rubbish.

In Word, go to File->Open and locate and select the .xml file you are trying to format. But do not just click the Open button as you would normally do. Click the down arrow to the right of the Open button and select Open with Transform.

You should now se another dialog box. Locate and select the cfr.xml that you just downloaded and click the OK button.

If you see a Convert File dialog, Word will probably have already selected the XML Document option. If not, select that, and click OK.

You will probably see a dialog box telling you about unsupported XML elements. Click through it.

Word applies the transform and opens the file. But at this point, what you actually see is a whole bunch of stylesheet code. 

Close that (don't save anything).

Now go to File->Open again, select Recent, and open the .xml that you just opened. If you see the FIle COnversion dialog again, select XML Document again and click OK. You may also see that warnign dialog - if so, click through it.

This time, you should see something more like a formatted document. My guess is that it will not be formatted very well (it does not use any of Word's style features, which makes it rather difficult to work with the results), but it may be adequate for your purpose.

If that is no use, there are a few of ways you could go.

 - You may be able to find a better Transform that can produce much better Word output.

 - if you open the XML using your Internet browser, it will probably have a default Transform that will lay out the XML rather more neatly than what you see when you open it in Word. (It may vary depending on the browser). Here, I was able to select the entire page of text in the browser and copy it into Word, which resulted in a much neater layout. You would then have to use Find/Replace (or e.g. VBA if you can) to replace all the XML Element tags (the things inside the angle brackets <>). If yo need exta help with that, post back.

 - In principle you could produce your own but you'd have to know the XSL language and lots of other stuff.

 - You could make use of the XML tags in the file to use Word Find/Replace to replace and potentially format chunks of text.

I am trying to highlight a row based on the date in one cell being either approaching this month, over one month after the date.  I am not brilliant at understanding formulas, but I can use them.  Can anyone help?

hansv mvp say:

Let's say the cells with the dates are in column D, in D2:D100

Select rows 2 through 100; I will assume that the active cell within the selection is in row 2.

 

On the Home tab of the ribbon, click Conditional Formatting > New Rule...

Select 'Use a formula to determine which cells to format'.

Enter the formula =AND(D2>TODAY(), D2<EDATE(TODAY(), 1))

This formula will return TRUE if the date in D2 is between today and one month ahead.

Click Format...

Activate the Fill tab and specify the desired fill color.

Click OK twice.

 

Repeat the above steps with the formula =AND(D2>0,D2<EDATE(TODAY(),-1))

This formula will return TRUE if D2 contains a date more than one month ago.

Select another color for this condition.
