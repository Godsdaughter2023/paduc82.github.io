Sub TransposeBlocks()
     Dim i As Long, lnFirstRow As Long, lnLastRow As Long, lnGap As Long, lnLen As Long
 
    Columns("A:A").Replace What:=" : ", Replacement:=":", _
         LookAt:=xlPart, SearchOrder:=xlByRows, MatchCase:=False, _
         SearchFormat:=False, ReplaceFormat:=False
    
    Columns("A:A").TextToColumns _
         Destination:=Range("B1"), _
         DataType:=xlDelimited, _
         TextQualifier:=xlDoubleQuote, _
         ConsecutiveDelimiter:=False, _
         Tab:=False, _
         Semicolon:=False, _
         Comma:=False, _
         Space:=False, _
         Other:=True, _
         OtherChar:=":", _
         FieldInfo:=Array(Array(1, 1), Array(2, 1)), _
         TrailingMinusNumbers:=True
    
    Columns("D:E").ClearContents
    
    For i = 1 To 100
         If Trim(Cells(i, "B").Value) = "Date of experience" Then
             lnFirstRow = i
         ElseIf Trim(Cells(i, "B").Value) = "E mail address" Then
             lnLastRow = i
             Exit For
         End If
     Next i
     For i = lnLastRow + 1 To lnLastRow + 100
         If Trim(Cells(i, "B").Value) = "Date of experience" Then
            lnGap = (i - lnLastRow) - 1
             'lnGap = i - lnLastRow
             Exit For
         End If
     Next i
     If lnFirstRow = 0 Or lnLastRow = 0 Or lnGap = 0 Then Stop
     lnLen = (lnLastRow - lnFirstRow) + 1
    
    Range(Range("B" & lnFirstRow), Range("B" & lnLastRow)).Copy
     Range("D1").PasteSpecial Paste:=xlPasteAll, Operation:=xlNone, _
         SkipBlanks:=False, Transpose:=True
     Application.CutCopyMode = False
    
    i = 2 'destination row
    
    While Range("C" & lnFirstRow).Value <> ""
         Range(Range("C" & lnFirstRow), Range("C" & lnLastRow)).Copy
         Range("D" & i).PasteSpecial Paste:=xlPasteAll, Operation:=xlNone, _
             SkipBlanks:=False, Transpose:=True
    
        lnFirstRow = (lnLastRow + lnGap) + 1
         lnLastRow = (lnFirstRow + lnLen) - 1
         i = i + 1
     Wend
    
    Columns("B:C").Delete Shift:=xlToLeft
     Range("B1").Select
 End Sub

Sub TransposeBlocks()
    Dim i As Long, lnFirstRow As Long, lnLastRow As Long, lnDestRow As Long
    Dim bFirstTime As Boolean
    Dim rng As Range
   
    Application.ScreenUpdating = False
    If InStr(Range("A1").Value, "Date of experience") > 0 Then Range("A1").Insert Shift:=xlDown
   
     'Remove spaces either side of colon:
    Columns("A:A").Replace What:=" : ", Replacement:=":", _
        LookAt:=xlPart, SearchOrder:=xlByRows, MatchCase:=False, _
        SearchFormat:=False, ReplaceFormat:=False
   
     'Split the data into 2 columns:
    Columns("A:A").TextToColumns _
        Destination:=Range("B1"), _
        DataType:=xlDelimited, _
        TextQualifier:=xlDoubleQuote, _
        ConsecutiveDelimiter:=False, _
        Tab:=False, _
        Semicolon:=False, _
        Comma:=False, _
        Space:=False, _
        Other:=True, _
        OtherChar:=":", _
        FieldInfo:=Array(Array(1, 1), Array(2, 1)), _
        TrailingMinusNumbers:=True
   
     'Tidy up:
    Columns("D:E").ClearContents
   
     'Initialize:
    lnDestRow = 2         'destination row
    bFirstTime = True
    lnLastRow = 1
   
    Do
         'Find the next cell containing "Date of experience":
        Set rng = Range("B:B").Find(What:="Date of experience", After:=Range("B" & lnLastRow), _
            LookAt:=xlPart, SearchDirection:=xlNext)
        If rng Is Nothing Then Stop
        If rng.Row <= lnFirstRow Then Exit Do '(this is where the code will end)
       
         'Row at start of next block:
        lnFirstRow = rng.Row
       
         'Find the next cell containing "E mail address":
        Set rng = Range("B:B").Find(What:="E mail address", After:=Range("B" & lnFirstRow), _
            LookAt:=xlPart, SearchDirection:=xlNext)
        If rng Is Nothing Then Stop
        If rng.Row <= lnLastRow Then Stop
       
         'Row at end of next block:
        lnLastRow = rng.Row
       
         'First time only, copy and transpose the column headings:
        If bFirstTime Then
            Range(Range("B" & lnFirstRow), Range("B" & lnLastRow)).Copy
            Range("D1").PasteSpecial Paste:=xlPasteAll, Operation:=xlNone, _
                SkipBlanks:=False, Transpose:=True
            bFirstTime = False
        End If
       
         'Copy the data in the current block:
        Range(Range("C" & lnFirstRow), Range("C" & lnLastRow)).Copy
        Range("D" & lnDestRow).PasteSpecial Paste:=xlPasteAll, Operation:=xlNone, _
            SkipBlanks:=False, Transpose:=True
       
         'Increment the destination row for next time round:
        lnDestRow = lnDestRow + 1
    Loop
   
    Columns("B:C").Delete Shift:=xlToLeft
    Range("B1").Select
    Application.ScreenUpdating = True
End Sub

Access equivalent of Excel's Vlookup - Microsoft Community

mentor_ed say:

Access equivalent of Excel's Vlookup

I have a school code in Table 1 and Table 2 contains the school code and school name.

Data entry is performed using datasheet view. Is there a way to enter the school code in Table 1 and

and populate the school name in table 1?

I'm currently using a lookup wizard in table 1 to return the school name.

marshall barton say:

That would be a bad idea, because the school name would then be in two places so it could potentially be changed in one place and not in the other.

It sort of sounds like you are working (entering/editing/viewing) the data in the table's datasheet.  If you are, it is a very dangerous way to work, because it's just too easy to make mistakes that can make a complete mess of things.  Instead you should always use forms and reports to interact with your data.  For your specific question, a form (record source table1) can use a standard everyday combo box (row source table2) that would save the school code while displaying the school name.

ken sheridan say: As Marshall says that should not be done as it would introduce redundancy and the consequent possibility of update anomalies.  However, there are occasions where you might need to reference the school name, e.g. you might want a form for entering/editing student data from multiple schools, but have the rows in the form sorted alphabetically by school and then by student name.  In this case you'd base the form on a query which joins the tables and orders the rows by the relevant columns, e.g.

SELECT Students.*
FROM Students INNER JOIN Schools
ON Students.SchoolID = Schools.SchoolID
ORDER BY SchoolName, LastName, FirstName;

In the form you would use a combo box bound to the SchoolID column from Students for selecting a school, setting it up like this:

ControlSource:  SchoolID

RowSource:     SELECT SchoolID, SchoolName FROM Schools ORDER BY SchoolName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

Remember that if you've used spaces or other special characters in table or column names (not advised)  the name must be wrapped in square brackets in the SQL statements [like this].

For future reference avoid the 'lookup field' wizard when designing a table.  This feature is generally deprecated by experienced developers.  For reasons why see:

http://www.mvps.org/access/lookupfields.htm

scottgem say:

You are apparently used to using spreadsheets, not databases. So the first thing you need to do is divorce yourself from spreadsheet mindset. Access has no functional equivalent to a VLookup because access doesn't work in columns and rows, but in records and fields. 

Also, as noted you should NOT be working directly into tables. All interaction with the tables should be done in forms. 

One of the principles of relational databases is that data should exist in one place and one place only. So having the School name in multiple tables makes for redundant data. However, as Ken noted, there may be situations where you want to DISPLAY data from a related table on your form after entering in the key data. To do that I've written a blog article (see sig for address) on Displaying Data from related tables. The article discusses several methods to do this.
