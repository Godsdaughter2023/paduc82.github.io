gordon@coldstream say:

Both the network adaptor and its driver must be capable of supporting remote wakeup.  In addition, the feature, if present, must be enabled on the network adaptor.  You must be an administrator to change the configuration of the network adaptor.  Also, your computer's BIOS (accessible during the boot process) must be configured to allow "Wake on LAN".  Refer to my more detailed post from January 2013 for more detailed information.

You might also try the following in this order:  (1) Uncheck "Allow the computer to turn off this device to save power" - and see if the other options are then no longer greyed out; and then (2) Check the box "Allow this device to wake the computer"; and then as the last step (3) Check the box "Allow the computer to turn off this device to save power".

I have been successful in using Remote Desktop from remote locations (half-way around the globe) to access inbound through my firewall and wake-up my Windows 8.1 Pro computer and allow me to login.  As a security tip, you can choose a port other than the default for Remote Desktop (3389) to make it more difficult for hackers.

Please try these steps and check after each step.

A. Repair / Reset Photos App
Windows+X > Settings > Apps > Apps & Features > Locate Photos by Scrolling Down > Click Advanced Options > Click Repair

Windows+X > Click Windows Powershell (Admin) > Copy the below command and right click in Powershell to paste it

get-appxpackage Microsoft.Windows.Photos | remove-appxpackage

Now, you need to reinstall it
Use below Powershell command to install

Get-AppxPackage -allusers Microsoft.Windows.Photos | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”

(If you encounter problem installing it, you can always install it from Microsoft Store)

C. There are three other actions which you can undertake

1. Restore from a system restore point to a point when things were working fine
https://support.microsoft.com/en-us/help/12415/...

2. If the issue has been caused due to recent Windows update - I would suggest that you uninstall the latest KB update and hide it so that it doesn't install it automatically and wait for new update to come which will be installed automatically.

To know which update got installed
Windows Key+i to invoke Settings > Update & Security > Click View installed update history. These KBs should be from the below list. Don't uninstall any other KB
https://support.microsoft.com/en-us/help/4099479
Make note of KB and uninstall and block it.
Uninstall
Windows Key + R > Type: appwiz.cpl > Enter > Click View installed updates > Select the required updates, right click, uninstall.
Block
Microsoft has provided a tool Show/Hide Updates tool to block the update
Download the tool from below link
http://download.microsoft.com/download/f/2/2/f2...
When you run this troubleshooter, it will search for available updates and list them
Select the update which you want to hide. Click Next and follow on-screen directions

3. Perform a Windows 10 repair upgrade. Repair upgrade fixes all Windows errors and retains all files, applications and settings. (You will not lose any data while backup is a good idea) Below is a good guide to perform repair upgrade

 How to Find Firmware Version

  1. Login to the router by entering the default gateway IP address.
  2. The router will then prompt you to enter the user name and password.the default user name and password are: "admin", "admin".

  4. From the sub-menu on the left select status.

schalk.vanwyk say: It's a faulty keyboard, unfortunately MS does not admit it.

I noticed the same issue, thought it was my imagination, until it got the better of me.

I ended opening the keyboard and "fiddled" with the insides a bit. Still some of the keys (k and G) did not work. I then unscrewed the screws at the circuit board containing the LED's and completely removed the top plastic film (be careful not to tear it at the "glue" points), reassembled it with the keypad rubber, tested it, still the "k" and "g" keys did not work. I then noticed what seemed like a small glitch/mistake in the circuit close to those keys, scratched it a bit and retested and all the keys worked again. Not sure how long this will last, but apparently is caused by a static buildup on the plastic films.
