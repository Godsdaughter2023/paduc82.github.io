Sub Schedule(ByVal Task As String, ByVal Procedure As String, _

Sub KillProcess(ByVal EXEName As String)
  Dim ProcessIDs() As Long
  Dim Modules(0 To 200) As Long
  Dim cbNeeded As Long, i As Long, Result As Long
  Dim hProcess As Long
  Dim ProcessName As String

  'Get a list of all running processes
  ReDim ProcessIDs(0 To 1023) As Long
  If EnumProcesses(ProcessIDs(0), 1024 * 4, cbNeeded) = 0 Then Exit Sub
  'Loop thru each process
  For i = 0 To cbNeeded / 4
    'Get a handle to the process
    hProcess = OpenProcess(PROCESS_QUERY_INFORMATION Or PROCESS_VM_READ, 0, ProcessIDs(i))
    If hProcess = 0 Then GoTo NextProcess
    'Get an array of the module handles for the specified process
    If EnumProcessModules(hProcess, Modules(0), UBound(Modules), cbNeeded) = 0 Then _
      GoTo NextProcess
    'Get the name of the process
    ProcessName = Space(MAX_PATH + 1)
    Result = GetModuleFileNameExA(hProcess, Modules(0), ProcessName, MAX_PATH)
    ProcessName = Left$(ProcessName, Result)

    'Found?
    If InStr(1, ProcessName, EXEName, vbTextCompare) > 0 Then
      'Close the process
      CloseHandle hProcess
      'Try to terminate
      hProcess = OpenProcess(PROCESS_TERMINATE, 0&, ProcessIDs(i))
      If hProcess = 0 Then GoTo NextProcess
      TerminateProcess hProcess, 0&
      CloseHandle hProcess
      Exit Sub
    End If

NextProcess:
    If hProcess <> 0 Then CloseHandle hProcess
  Next
End Sub

teylyn say:

Hello,

the #N/A error only has one cause: the lookup value cannot be found in the first column of the lookup table.

Double check:

  • the first column of the lookup table contains the lookup value
  • the lookup value and the values in the lookup table are of the same data type. If numbers are stored as text, they look like numbers, but they are not a match
  • leading or trailing blanks or other invisible characters in either the lookup value or the first column of the lookup table. 

If you think that a match should be found, but you still get an #N/A error,  use a formula like =A1=Sheet2!A1 to compare the two cells. Does that return TRUE? In that case, post a sample file for inspection.

jasles say:

charles kenyon say:
Hi Tony,

Since Word 2013, Word has had the ability to directly open pdf document. However, some pdf documents (from scanners mostly) are really pictures of text rather than the text itself. Just as you can't look under the hood of a picture of a car, you can't edit a picture of a document.

In that case, you need to convert the picture to text. This is a process known as optical character recognition. This is built into Adobe Acrobat and is also in Office OneNote. Most scanner software comes with an OCR component as well. Once you have text, you can edit it with Word.

If you simply want to write on the document (but not in it) you can add a Text Box floating on top of the document layer, whether or not it has been put through the OCR process.

Word documents that have been saved as PDF will not need the OCR process, they retain their text, although not all of their Word structure and formatting.

Finally, documents converted from pdf (or really any other format) to Word can be tough to edit because the conversion process never has a one-to-one matching of how formatting is done. This means that a converted document will seldom be formatted in Word in a way that makes sense. An example is multiple section breaks to change margins, where in Word you would simply change the paragraph indent.

Likewise, a pdf converted to Excel is likely to have mostly text once converted rather than numbers.

I hope this information helps. Please let me know if you have any more questions or require further help.

Regards
