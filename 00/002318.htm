ronrosenfeld say:

The following VBA code consolidates your tables and produces a new table that consists of all of the entries in your original table.  You can then sort and/or filter the results to produce the kind of report you want (or those inputs can be gathered on a user form).

The code assume (which seems to be how your sample is laid out)

  • All of your data is in Tables
  • All of the tables have the same number of columns and the same column headers
  • The worksheets are named for the Classes, and all of the names begin with the word "Class"
      (If that is not the case, we can easily derive another method of being sure we choose the correct worksheet to process)
  • The results will go on a worksheet named Report as a Table, which you can then filter and sort to generate your report
  • The results will include a column naming the Class

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.

From the top menu, select Insert/Class Module

In the Project Explorer, select the class module you just added, probably named Class1; hit <F4> and rename the module cTuition

Then paste the following code into the code window:

==========================================
Option Explicit
Private pClass As String
Private pDataRow As Object

Public Property Get Class() As String
    Class = pClass
End Property
Public Property Let Class(Value As String)
    pClass = Value
End Property

Public Property Get DataRow() As Object
    Set DataRow = pDataRow
End Property
Public Property Set DataRow(Value As Object)
    Set pDataRow = Value
End Property
===================================

Next, from the top menu, select Insert/Module and
paste the code below into the window that opens.

================================

Sub ConsolidateTables()
    Dim cT As cTuition
    Dim colT As Collection
    Dim WS As Worksheet
    Dim LOs As ListObjects, LO As ListObject
    Dim oHeaders As Object
    Dim vRep() As Variant
    Dim R As Range
    Dim I As Long, J As Long
    
Set colT = New Collection
For Each WS In ThisWorkbook.Worksheets
    If WS.Name Like "Class*" Then
        Set LOs = WS.ListObjects
        For Each LO In LOs
        If oHeaders Is Nothing Then Set oHeaders = LO.HeaderRowRange
            For I = 1 To LO.DataBodyRange.Rows.Count
                Set cT = New cTuition
                With cT
                    .Class = WS.Name
                    Set .DataRow = LO.DataBodyRange.Rows(I)
                    colT.Add cT
                End With
            Next I
        Next LO
    End If
Next WS

'Populate report array
ReDim vRep(0 To colT.Count, 0 To oHeaders.Count)
    vRep(0, 0) = "Class"
    For I = 1 To oHeaders.Count
        vRep(0, I) = oHeaders(I)
    Next I

For I = 1 To colT.Count
    With colT(I)
        vRep(I, 0) = .Class
        For J = 1 To .DataRow.Cells.Count
            vRep(I, J) = .DataRow.Cells(J)
        Next J
    End With
Next I

Application.ScreenUpdating = False
On Error Resume Next
Set WS = Worksheets("Report")
    If Err.Number = 9 Then 'Report does not exist, so create it
        Worksheets.Add before:=Sheet1
        ActiveSheet.Name = "Report"
        On Error GoTo 0
        Set WS = Worksheets("Report")
    End If
    
With WS
    .Cells.Clear
    Set R = .Cells(1, 1).Resize(UBound(vRep, 1) + 1, UBound(vRep, 2) + 1)
    With R
        .Value = vRep
        .EntireColumn.AutoFit
    End With
    .ListObjects.Add SourceType:=xlSrcRange, Source:=R, _
        xllistobjecthasheaders:=xlYes
End With
Application.ScreenUpdating = True
                
End Sub

================================

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.

jamesdemers say:

If you change the file type extension of a .pptx or .docx file to .zip (just edit the file name), and then "unzip" the resulting file, you get a folder containing all the bits and pieces that go into the XML file.  The graphic elements should be present in the "media" folder as image1.emf, image2.emf, etc. 

The problem is that they aren't all there, and nobody seems to have figured out yet how or why some of them go missing.  In my tests, the missing graphic elements never get written to the file - they just appear on screen, and you THINK they're being saved when you "save" the file.  It doesn't happen to smaller graphics, under about 100 KB, and it only happens to .emf formatted graphics, which seems to be the default for cut & paste.

Two workarounds, until MS gets their act together, are:

  • "Save as" in .doc or .ppt format, or
  • "Paste special" when inserting graphics, and select png or tiff.

Note: The above observations are for MS Word for Mac 2016, so your mileage may vary.

ok, use as you know the RGB options...

 

Sub CellGradientFill()
  With ActiveCell.Interior
    .Pattern = xlPatternLinearGradient
    With .Gradient
      .Degree = -45
      With .ColorStops
        .Add(0).Color = RGB(192, 192, 192) ' ... for Left-Gray ...
        .Add(0.499999).Color = RGB(192, 192, 192) ' ... for Left-Gray ...
        .Add(0.500001).Color = RGB(51, 102, 255) ' ... for Right-Blue ...

Replace the bottom part of the macro with this:


    Dim olAppt As Outlook.AppointmentItem
    Dim olApptCopy As Outlook.AppointmentItem
        
    'Copy the appointment item
    If objItem.Class = olAppointment Then
        Set olAppt = objItem
        Set olApptCopy = olAppt.Copy

        Exit Sub
    End If


The only downside of this approach is that when you cancel the new item, the copy remains on the Calendar. Deleting the copy won't trigger sending a cancellation though so cleanup is quite painless.
