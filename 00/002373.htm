marshall barton say:

An overflow error is when a calculation results in a number thatis too big to be stored in a field or variable.  For example an Integer field can not store a value greater then 64K, so you will get an over flow when you multiply two numbers (eg. 12345 by 10000) that is greater than 64K.  Another way to get an overfloe is by dividing a number by a really small value such as .00000001 and sometimes even 0.

You may be getting these errors because you are doing something (summing or multiplying?) some numbers that were ok before, but have increased to the point of overflow.  Look at the expression that produces the error and see what the calculation does so you can figure out how it generates a large result, especially check for a divisor of 0.

I don't think it's possible for a 0 numerator to cause an overflow error.  Are you sure you have correctly identified the place where the error occurs?  Overflow is almost always the result of multiplying two fairly large numbers to produce a number that is too big, Or by dividing a very small number into a not small number with the result being a too big number.

Sub UpdateTables()
Application.ScreenUpdating = False
Dim i As Long, oPrev As Range, oNext As Range
With ActiveDocument
  For i = .Tables.Count To 1 Step -1
    With .Tables(i).Range
      Set oPrev = .Characters.First.Previous.Characters.Last
      Set oNext = .Characters.Last.Next.Paragraphs.First.Range
      With oPrev
        .InsertBefore vbCr
        .Start = .End - 1
        .Style = oNext.Style
        .End = .Start
        If Len(oNext.Text) > 1 Then
          With oNext
            .End = .End - 1
            .Cut
            .Delete
          End With
          .Paste
        Else
          oNext.Delete
        End If
      End With
    End With
  Next
End With
Application.ScreenUpdating = True
End Sub

doug robbins - mvp office apps & services (word) say:

Using a macro containing the following code will do what you want:

Dim i As Long

Dim rngcaption As Range

Dim rngFigure As Range

On Error GoTo ExitSub

With ActiveDocument

    For i = 1 To .Fields.Count

        If .Fields(i).Type = wdFieldSequence And InStr(.Fields(i).Code, "Figure") > 1 Then

            Set rngcaption = .Fields(i).Code.Paragraphs(1).Range

            Set rngFigure = .Range

            rngFigure.End = rngcaption.Start - 1

            rngFigure.Collapse wdCollapseEnd

            Set rngFigure = rngFigure.Paragraphs(1).Range

            rngFigure.Cut

            rngcaption.ParagraphFormat.KeepWithNext = True

            rngcaption.Collapse wdCollapseEnd

            rngcaption.Paste

        End If

    Next i

End With

ExitSub:

Hi Kevin,

My apologies for not getting back to you sooner but I had a couple of other commitments. However, install the code below.

Note that if you want to also use the double click on another column for another purpose then just use a case statement to determine in which column the cell was Double Clicked.

The code ignores data that is not numeric so if you use the prefix "M" for manually entered numbers then they will be ignored.

Note where you will need to edit the column id where the numbers will be.

Private Sub Worksheet_BeforeDoubleClick(ByVal Target As Range, Cancel As Boolean)
    Dim rngCol As Range
    Dim lngNextSeq As Long
   
    Set rngCol = Me.Columns("A:A")     'Edit "A:A" to your column reference
   
    If Not Intersect(rngCol, Target) Is Nothing Then    'Not nothing then is within the column
        Cancel = True   'Cancels Edit mode that is normally invoked with double click
        lngNextSeq = WorksheetFunction.Max(rngCol) + 1
        Target.Value = lngNextSeq
    End If
   
End Sub
