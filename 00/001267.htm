Graphics driver Windows Updates failing with error code 0x80070103 - Microsoft Community

adam [msft] say:

Graphics driver Windows Updates failing with error code 0x80070103

Just wanted to give an update on an issue that is likely to be seen on this flight.   For the Graphics driver delivered through Windows Update it may result in error code "0x80070103".  This is basically WU reoffering the same driver.  The problem stems from an issue with how the driver gets installed during setup of the flight.  Note the error doesn't mean you aren't getting the driver you need it just means it's already installed or a better driver exists.

How do I fix this situation?

You can work around this and get the driver to stop being re-offered by uninstalling the driver and letting it re-install.  

  1. Go to device manager.
  2. Go to device manager.
  3. Click on the driver properties of the display adapter.
  4. Click on the driver properties of the display adapter.
  5. Uninstall the driver.
  6. Uninstall the driver.
  7. Let WU scan for an reinstall the driver.
Let WU scan for an reinstall the driver.

Some additional details and how to identify if you want to dig deeper:

What error code do we get when a driver is reoffered?

Incase of better content already installed scenarios,

•In Windows update, you will notice installation to fail with 80070103 error code

•In PnP log, you will notice that installation to exit with 00000103 error code

What does 80070103 error mean?

This just means that the client system has a better ranking driver already installed, and that WU/PnP is trying to force offer/install a driver with low ranking repeatedly. “Driver Ranking” is a very broader topic, but, in short  – WU\PnP will look for Driver Date, Driver Version, and hardware ID match to offer/install a driver to a client system.

Identifying through logs:

In PnP log (setupapi.dev.log or setupapi.setup.log), you will find:

 ndv: {Update Device Driver - exit(00000103)}

!    ndv: No drivers were found that are better matching drivers for device instance PCI\VEN_1002&DEV_674A&SUBSYS_0122103C&REV_00\4&16C9E9F9&0&0010 (Error 0x00000103)

!    ndv: No devices were updated.

….

ken sheridan say: It's not updatable because the BackGroundCheck query is not updatable.  You'll need to call the DLookup function rather than using an INNER JOIN to make the UPDATE query updatable:

UPDATE Adults
SET BackgroundCheck =
DLookup("BkgdCheckExpr", "BackGroundCheck", "AdultID = " & [AdultID])
WHERE DLookup("BkgdCheckExpr", "BackGroundCheck", "AdultID = " & [AdultID]) IS NOT NULL;

This assumes that AdultID is a number data type.  The WHERE clause simulates an INNER JOIN by excluding rows where there is no match.

Hello, Jean Christophe324

Although this FAQ explains the process,

... remember,

-> If you have not already done so, you will be required to fully update a legacy Skype account to a Microsoft account in order to close the Skype account. -> If you need to fully update a legacy Skype account, create a new Microsoft account with its own new email address that you will use as a "throw-away" account, simply to merge/link the new Microsoft account with the legacy Skype account.

-> Creating a new and separate Microsoft account is imperative if you already have a Microsoft account for which you use Microsoft-hosted features such as email, One Drive, X-Box, etc.  Closing the Skype account associated or linked with the Microsoft account automatically closes the entire Microsoft account.
