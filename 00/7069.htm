If you use the export part of Regedit you can make a folder where you save the Regedit copies. Name your copies in a way that you will know when and maybe even a version number if you are making mutiple backups in the same day.

Then if something went wrong you can compare them with command prompt and just see what was changed.

For a test on how it works open Notepad and type in something simple and save it as version 1.

Now add something simple and save as version 2.

Open Command prompt.

In the command line type fc C:\your new folder\version 1.txt C:\your new folder\version 2.txt > C:\your new folder\Result.txt

The Reult.txt will be created in your new folder so you can open it to see what was different between the two versions.

When you compare the Registry files you have to put the .reg in place of the .txt except for what ever you name your Result file because it needs to be saved as a text file.

So this is how it should look if you were to compare 2 versions of your Registry backups.

fc C:\your new folder\version 1.reg C:\your new folder\version 2.reg > C:\your new folder\Result.txt

This helps in ways you can manage the Registry knowing what has been changed especially if you do this before and after you install a Program and you want to remove it later.

So many times the uninstall does not rebuild the registry the way it was before the Program had been installed.

Just remember if you make a Registry change that breaks your OS it can be much harder to repair from outside the OS.:)

jay freedman say:
Before answering your question, I want to point out that Word does not use any file named normal.docx or building blocks.docx. It uses normal.dotm and building blocks.dotx, and it will not use files with any other extension for the same purposes.

A possible reason for the person to see both the customized and the built-in building blocks is that the built-in building blocks may be coming from a file in the same folder (%appdata%\Microsoft\Document Building Blocks\1033\14) with the name Built-In Building Blocks.dotx. [That file is a copy of the installer-created file in the folder C:\Program Files\Microsoft Office\Office14\Document Parts\1033\14.]

To verify whether that's the case, open the Building Blocks Organizer (from the Insert > Quick Parts button) and look at the Template column to see where the built-in building blocks are stored.

If you simply delete the Built-In Building Blocks.dotx file from the folder in the %appdata% path, it will be restored with a copy of the one from the Program Files path the next time Word starts. The way to get the result you want is to delete all the building blocks that are stored in
Built-In Building Blocks.dotx but leave the file in the folder. If you've already deleted the entries from that template on your computer, you can copy that file to the other computer.

*Additional information: If your default language is not English US, then the 1033 in the file paths must be replaced with a different language ID number (a fairly complete list is here). Also, if you're using 64-bit Windows, the installation path is under C:\Program Files (x64) instead of C:\Program Files.
