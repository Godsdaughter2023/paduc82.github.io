Sub AutoF_export_data_new_file()
'Aug 29-2017
Const sCol As String = "B"    '<<< autof in column B
Const sCrit As String = "Name1" '<< criteria name, wb name = Name1.xlsx / change as needed
Dim wb As Workbook
Dim ws As Worksheet
Set ws = ThisWorkbook.Sheets("Sheet1") 'data in sheet1/change source sheet name as needed
Dim rng As Range
Set rng = ws.Range("A1").CurrentRegion
Dim sPath As String
sPath = "c:\users\tasos\desktop\" 'save new workbook on desktop/change username as needed
Dim r As Long
Application.ScreenUpdating = False
ws.AutoFilterMode = False
r = ws.Cells(Rows.Count, sCol).End(xlUp).Row
'
'add new wb and copy filtered data /values-formats only
Set wb = Workbooks.Add(xlWorksheet)
ws.Range(sCol & "1:" & sCol & r).AutoFilter field:=1, Criteria1:=sCrit
rng.SpecialCells(xlCellTypeVisible).Copy
wb.Sheets(1).Range("A1").PasteSpecial xlPasteValues
wb.Sheets(1).Range("A1").PasteSpecial xlFormats
wb.Sheets(1).UsedRange.EntireColumn.AutoFit
Application.CutCopyMode = False
ws.AutoFilterMode = False
'
'save new wbk on desktop
Application.DisplayAlerts = False
wb.SaveAs Filename:=sPath & sCrit & ".xlsx"
Application.DisplayAlerts = True
wb.Close False
Application.ScreenUpdating = True
End Sub

me_5555 say:

Andy David from the other forum provided the details that I was looking for.  He said,

Q1/Q2: They are functionally the same, however generally "receive as " is granted at the database level and not by mailbox. Why? Because if you grant "Full", then the mailboxes are auto-mapped to the Outlook profile by default versus the receive as perms will apply to any mailbox added to that database - even ones created or moved to it in the future but wont auto-map. Full is more explicit and easily managed via EAC/ECP, versus Receive As which would/could be set by an Admin for the entire database.  A typical scenario would be to grant a service account receive as for an entire database versus a user you grant full at the mailbox level.

I found information to Q3 elsewhere.  This is what I found.

Q3: AD Connect will not synchronize Receive As permissions.  Users that have Receive As permissions will need to be assigned Full Access permission manually in Exchange Online if the migration tool doesn't migrate these permissions.
