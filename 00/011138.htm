doug robbins - mvp office apps & services (word) say:

In each comment, where you want the child's name to appear, I would insert a 

{ Docvariable FirstName } 

field.

You must use CTRL+F9 to insert the field delimiters, and you used ALT+F9 to toggle off their display.  I would also run a macro on that document that contains the code

With ActiveDocument
     .Variables("FirstName").Value = "Child's First Name"

     .Range.Fields.Update
End With

Then when you have copied the necessary comments into the report for a child, run a macro containing the following command

With ActiveDocument

     .Variables("FirstName").Value = InputBox("Enter the first name of the child for whom this report is being prepared")

     .Range.Fields.Update
End With

and then the name that you enter into the input box will appear in each comment in the report.

jim gordon mvp say:
Hi John,

As you observed, when you turn on the Macro recorder there's a pop-up menu that lets you choose where you want to save your Macro. The default is "This Workbook" but you can also choose your Personal Macro Workbook or other open workbook.

I think your question is how do you find out where the Macro was stored after you're done recording it. You can see all the Macros in workbooks that are currently open by opening the Visual Basic Editor (go to the Developer Tab of the Ribbon and then click Visual Basic Editor). If the Developer Tab of the Ribbon is not displaying, go to Excel Preferences and in the Ribbon preferences click the check box for Developer.

In the Visual Basic Editor, look in the Project Explorer (View > Project Explorer). Click triangles for any folder called Modules to see the modules that are stored. Each open workbook has its own collection of modules. 

There is no master record of where and when you recorded macros, so it's up to you to remember where you stored your macros.
