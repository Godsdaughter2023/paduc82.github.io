And here some code. Please copy paste it into the code module of your table.

Peter

Private Sub Worksheet_Change(ByVal Target As Range)
   Dim r As Range
   Dim counter As Long

   On Error GoTo error_and_exit
   Application.EnableEvents = False

   If Not Intersect(Target, [B:B]) Is Nothing Then
      For Each r In Range([B2], Cells(Rows.Count, "B").End(xlUp))
         If r = "" Then
            r(1, 0).ClearContents
         Else
            counter = counter + 1
            r(1, 0) = counter
         End If
      Next
   End If

error_and_exit:
   Application.EnableEvents = True
End Sub

rick rothstein, mvp - excel say:

I think this macro does what you requested...

Sub NumberColumnA()
  Dim X As Long, LastRow As Long, Counter As Long
  Counter = 1
  LastRow = Cells(Rows.Count, "B").End(xlUp).Row
  For X = 2 To LastRow
    If Cells(X, "B").Value <> "" Then
      Cells(X, "A").Value = Counter
      Counter = Counter + 1
    End If
  Next
End Sub

t. valko say:

=COUNTIFS($X$6:$X$11,">"&J22,$Z$6:$Z$11,"<"&$AK$14,O6,"=n")

The last criteria which is meant to be text, "y" or "n", is the one causing the problem.

The reference to O6 is not the same size range as the other ranges X6:X11 and Z6:Z11. All ranges must be the same size.

Is it supposed to be O6:O11 ?

=COUNTIFS(X6:X11,">"&J22,Z6:Z11,"<"&AK14,O6:O11,"n")
