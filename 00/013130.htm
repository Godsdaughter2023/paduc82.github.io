superfranky say:

Hi HansAndFranz,

There is no conflict between Litigation hold and Retention Policy.
The end user is not able to know if Litigation Hold is enabled or not.

Retention Policy specifies how long a message remains in a mailbox and the action to be taken when the message reaches the specified retention age. When a message reaches its retention age, it's moved to the user’s In-Place Archive or deleted. The ender user can check and manually change the retention tags for the folders of the mailbox.

Litigation Hold uses the Recoverable Items folder to preserve items. The Recoverable Items folder is hidden from the default view of Outlook, Outlook Web App, and other email clients. By default, when a user deletes a message from a folder other than the Deleted Items folder, the message is moved to the Deleted Items folder. When a user soft deletes an item (by pressing SHIFT + DELETE ) or deletes an item from the Deleted Items folder, the message is moved to the Recoverable Items folder, thereby disappearing from the user’s view.

bob jones aka: cybertaz say:

PMFJI :-) I don't claim to be an expert on the subject, but there appears to be some misconception about what kerning is.

Kerning is an attribute of font design which applies to specific character pairings. It's built into the font metrics based on a Kerning Table & varies greatly from one font to another. It's imposition also is determined to a great extent by the Font Type. This article provides a far more professional explanation: Design Fundamentals: Kerning and Tracking | Graphics.com

All PPT can do is respect the built-in kerning specifications of the font. The check box for Use kerning... simply determines whether the built-in kerning of the font is imposed: On or Off. If kerning is specified for a certain pair of characters it will be imposed. If not specified for that pair, enabling the use of kerning will not have an effect on the spacing of that pair. Further to that, depending on the font design, it's also possible that some spacing will be adjusted but the spacing between other character pairs will not be.

Professional publishing software, such as QuarkXpress & InDesign provide high caliber Kerning & Tracking features as well as a means of actually modifying the Kerning Tables of fonts. Short of adding such capability at a significant cost, there's not much Microsoft can do if a font doesn't include kerning values that cover all the bases.

To help address the issue that arises when certain pairs are not kerned (which often is the case for combinations of upper case characters), PPT provides the option to Expand or Condense the Spacing in .1 pt increments. That adjustment can be applied to individual characters, not just character strings regardless of whether the font includes otherwise acceptable kerning values.

E.g.: In your sample, let's say you want to draw the A & the V a bit closer together... Try this;

  1. Select just the letter A (the leftmost of the 2 characters)
  2. Go to Format> Font - Character Spacing
  3. Choose Condensed from the Spacing options
  4. Set the points value to suit your preference

¡firedog say:
It's not particularly clear from your description what has happened. If I were to guess, I'd say that you've been signing in in Windows Live Mail with two different Microsoft Accounts, one personal, one business. When you sign in with a Microsoft Account, you see the contacts and calendar associated with that account.
