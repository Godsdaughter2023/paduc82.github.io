brookzhang_msft8047 say:

hi randy101,

it’s not feasible to directly have new coming emails to be copied to archive mailboxes immediately or daily. however, after my investigation, i found a workaround that can copy emails to archive mailbox daily although it seems a little complicated. please refer to the following steps:

1. login to exchange admin center with admin credentials.

2. navigate to recipients > mailboxes > choose you own mailbox, in the right panel > in-place archive > enable.

3. navigate to compliance management > retention tags > add a personal tag > create a tag like the following screenshot.

4. login to your owa (outlook web app) and create a new folder.5. options > mail > retention policies > add the newly created one day archive tag > save.

6. in owa > choose the formerly created folder > right click and assign the tag to it.

7. then create an inbox rule to copy all coming emails to the new folder so that all emails will be copied to archive mailbox. owa > options > mail > automatic processing > inbox and sweep rules > create the related rule.

all emails in archive mailboxes will not be automatically deleted. they can be deleted manually. do you mean you would like these emails not to be deleted even when we try to manually delete them?

we can place in-place and litigation hold on a mailbox so that emails will always be saved in serve even we manually delete them. however, both the primary and the archive mailbox are included. it’s not feasible to only place archive mailbox in hold.

if things are a little different from your side, feel free to let me know. further assistance will be provided.

example

If 72<x<90 then = .6 also if 89<x<100 then =.7 and so on.  So I can plug in x   73 and it would equal .6 in the colum below which would then link up to a more simple equation.

scjr say:

If your value for "x" is in cell A1, a formula similar to this (for a range of 72 to 110) should work in cell B1:

=IF(AND(A1>=72,A1<90),0.6,IF(AND(A1>=90,A1<100),0.7,IF(AND(A1>=100,A1<110),0.8)))

The formula says that if the value in A1 is greater than or equal to 72 AND less than 90, insert 0.6 in cell B1; if it is greater than or equal to 90 AND less than 100, insert 0.7; if it is greater than or equal to 100 AND less than 110, insert 0.8.  You can expand the formula and vary for numerical range as needed by using the different elements included in the formula.

jllatham say:

Your "...and so on." leads me to believe that there may be more than just 2 or 3 comparisons to be made.

This could be done using a table of values and a formula to use that table to return the converted value.

For this example, I set up a very short table in columns G and H as:

       G          H
1  MIN       Value
2    73          .6
3    90          .7
4   107         .8

The "MIN" values in column G must be entered from smallest to largest order.  Assume your x value is then entered into cell A1 and you want the converted value to appear in B1, in cell B1 you could put this formula:

=INDEX($H$2:$H$4,MATCH(A1,$G$2:$G$4,1),1)

Values 72 and below will return #N/A
Values 73-89 will return .6
Values 90-106 will return .7
Values 107 and greater would return .8

You can deal with the #N/A problem by modifying the formula like this
For Excel 2007/2010:  =IFERROR(INDEX($H$2:$H$4,MATCH(A1,$G$2:$G$4,1),1),"Below Minimums")

For Excel 2003 and earlier:
=IF(ISNA(INDEX($H$2:$H$4,MATCH(A1,$G$2:$G$4,1),1)),"Below Minimums",INDEX($H$2:$H$4,MATCH(A1,$G$2:$G$4,1),1))

If you need to be alerted that the x value in A1 is larger than the lookup values in the table, then
For Excel 2007/2010
=IF(A1>MAX($G$2:$G$4),"Off the Chart",IFERROR(INDEX($H$2:$H$4,MATCH(A1,$G$2:$G$4,1),1),"Below Minimums"))

For Excel 2003 and earlier:
=IF(A1>MAX($G$2:$G$4),"Off the Chart",IF(ISNA(INDEX($H$2:$H$4,MATCH(A1,$G$2:$G$4,1),1)),"Below Minimums",INDEX($H$2:$H$4,MATCH(A1,$G$2:$G$4,1),1)))

You can have the table on another sheet if you want, you'd just need to include the sheet reference for the $G$2:$G$4 and $H$2:$H$2 ranges.  Naturally, adjust the $G$2:$G$4 and $H$2:$H$2 ranges as needed to include your entire table.
