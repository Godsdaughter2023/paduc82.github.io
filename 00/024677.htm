kirk b. - microsoft support say:

Hi lakesidemom,

 

When I tried setting the AllowEdits property to False in the control's Exit event, I found that it was seemingly ignored because it also looks like Access by default wants to allow one to be able to edit any control on the form.  To get around that issue, I tried adding a DoCmd.RunCommand accmdSaveRecord to the exit event, and that worked great as long as I had edited the record.  If focus just happened to be on the control and I tried to click anywhere else on the form I got a message saying that the Save Record action wasn't available.  The final piece for me was to add a form-level variable and set that on the dirty event of the control.  That left me with the following code on that form:

 

Option Compare Database

Dim stDirtyStatus As String 'Variable created outside procedures

 

Private Sub txtMyControl_Enter()

Me.AllowEdits = True    'Allowing edits on the form

End Sub

 

Private Sub txtMyControl_Exit(Cancel As Integer)

If stDirtyStatus = "Dirty" Then    'We'll run this code only if we make an edit

    DoCmd.RunCommand acCmdSaveRecord

    Me.AllowEdits = False

    stDirtyStatus = "Not Dirty"

End If

End Sub

 

Private Sub txtMyControl_Dirty(Cancel As Integer)

stDirtyStatus = "Dirty" 'Here's the first step of letting the exit event know that we've made a change

End Sub

 

I'm not sure that you'll need the test for whether or not an edit has been made, but I needed it in order to get the save record portion to work.  Hopefully there's something there that will get you on your way.

 

Good luck!

Increase the Cell Reference Numbrer Horizontally

Dear Sir,

How to increase the Cell Reference Number Horizontally By using formula

Regards

bernie deitrick say:

For example, to get B3, B4, B5... when copied across:

=INDIRECT("B" & COLUMN(C1))

or

=INDEX($B:$B, COLUMN(C1))

Or to get B3, C3, D3... when copied down:

=INDIRECT(ADDRESS(3,ROW(A2)))

or

=INDEX($3:$3,ROW(A2))
