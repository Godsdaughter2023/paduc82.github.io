When an Office Click-to-Run application is running, it requires access to the virtual file system. This is why you can access the Q: drive from your File and Save menus.

steven. s say: Hey Lightweight,

If you don't trust the add-on remove it. Most, but not all, cases the add-on will cause a DEP error because the add-on was designed for a previous version of Internet Explorer. For example: an add-on calls for a specific command to be completed within Internet Explorer, but in Internet Explorer 8 that command no longer exists. DEP will see this as a program acting outside its guidelines and shut down the program (Internet Explorer) to prevent anything from happening.

If the add-on is something that you want/use, check to see if there is an upgrade. A good rule is if you don't use it, lose it.

If you are running into DEP kicking in, there isn't a quick and easy way to find which add-on is causing the issue. The simplest is:

  1. Combine the data on the mobile device with the data on the computer:  Choosing this option is your best bet.  If you have data on both devices, you may wind up with some duplication, but you won't lose any data.

If the sheets have the same names in both workbooks, you can use code like this:

 

Sub price_delete_copy_book7()
    Dim wbk6 As Workbook
    Dim wbk7 As Workbook
    Dim varName As Variant
    Set wbk6 = Workbooks("Book6.xlsm")
    Set wbk7 = Workbooks("Book7.xlsm")
    ' Loop through an array of sheet names
    For Each varName In Array("ThisOne", "ThatOne", "Another", "YetAnother", "AndSoOn")
        wbk7.Worksheets(varName).Cells.ClearContents
        wbk6.Worksheets(varName).Cells.Copy _
            Destination:=wbk7.Worksheets(varName).Range("A1")
    Next varName
End Sub

 

Modify the array as needed - alter the names, add other ones, ...
