N8 Tricky Positioning Methods - Microsoft Community

m80116---01 say:

N8 Tricky Positioning Methods

A recent discovery of mine regarding OVI Maps (in future NOKIA Maps) on my newly purchased N8 regards the tricky Positioning Methods.

 

Here is what I found out. There are 5 Positioning methods on the N8 which are to be found in MENU > Settings > Applications settings > Positioning > Positioning methods.
 

 

Assisted GPS

This one acts as a ON or OFF switch of the function. GPS being assisted when a network request is being made to a server to speed up the positioning.

THIS SETTING WILL NOT AFFECT the ability of the device to initiate a network connection automatically (Wi-Fi or GMS/UMTS Network).

If the option is OFF it should be assumed that no A-GPS request is made to the server. Ovi Maps continues to initiate network connection seemingly retrieving the position much faster. Probably other data is used (i.e. cellar data detected via network, but no by the A-GPS server but rather the OVI Maps server that provides maps and services for the application).
 

Integrated GPS

It is the internal GPS of the device, this should be ON for the device to use it's own GPS chipset.

 

Bluetooth GPS

This should allow you to use external GPS accessories like the old NOKIA LD-3W wireless GPS module.

 

Wi-Fi/Network

THIS SETTING INHIBITS AUTOMATIC NETWORK (GPRS/UMTS) CONNECTION for the OVI Maps program. This allows you to save on your network data bill, and still provides you access to the A-GPS function on demand. OVI Maps will request the network connection if the A-GPS setting is ON, often asking you to connect several times (3-4) before letting you go on without. If A-GPS is OFF no user prompt is made and no connection is initiated.

 

Network Based

This is the setting (not verified yet) that should allow the device to extrapolate approximate position from cellar data. This should help positioning less than A-GPS but at least place you in a reasonable range in the mapping program (5-10 Km radius at the most).
 
 

If you have any other info about the PM please report to this thread. I hope this helps clarify how the positioning works in the N8 and ease the migration of users from older N devices (like me coming from N95 8GB).

tim bailen say:

There's another thread on Answers which has some good leads: Unable to open xlsx file by double clicking the file. This helped me after I tried the obvious steps like Unnar did such as trying to set "Open With" and running Office repair.

One commenter talks about removing Notepad from the list of associated apps. I didn't have Notepad associated, but somehow Adobe Reader had become an option for .xlsx files. It seems that this "extra" incorrect association causes Excel to fail to open the file even when you explicitly tell Explorer to "Open With" Excel.

With that as a lead, I was able to fix the problem. This was pretty kludgey, so I hope someone else can follow up with a simpler process (for example, if this were to happen again I think I would try exporting the .xlsx registry key to save a backup and then I'd delete the whole key and see if Excel regenerates it (either when it opens or by running the installer in repair mode)

So, here's what I did:

  1. Locate the HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.XLSX area in the registry. This is where the problem lies.
  2. Notice that some undesirable app (such as Notepad, or in my case Adobe Reader) is associated with .XLSX files in the OpenWithList 
  3. Delete offending app from that list.
  4. Try right-clicking .XLSX file and Open With Excel. Success? No, in my case Adobe Reader got re-added to the OpenWithList in the registry!
  5. Delete offending app in registry again.
  6. Modify registry to match a computer that is working. (Here's a reg export for a computer I have with Office 2010 that is working:

    Windows Registry Editor Version 5.00

    HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.xlsx]

    HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.xlsx\OpenWithList]
    "a"="EXCEL.EXE"
    "MRUList"="caedb"
    "b"="NLNOTES.EXE"
    "c"="Moc.exe"
    "d"="iexplore.exe"
    "e"="chrome.exe"

    [HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.xlsx\OpenWithProgids]
    "Excel.Sheet.12"=hex(0):

  7. Try "Open With" again and choose Excel. Success? No?
  8. Open Excel / Close Excel
  9. Is Excel listed on the "Open With" menu twice? Try the other option (this is what happened for me. Excel got added to the list again and only one of the options worked)
  10. (step 9a) To get Excel on the list twice-- I'm not sure how that happened-- it may have happened when I went to Control Panel > Folder Options > File Types and changed the program used to open .xslx files, and then used the "Restore" button to restore the default program to the default choice. Oddly, this set Adobe Reader as the default, but I believe this is also when it listed Excel as a choice twice.

Good luck!

For those not willing to brave the registry, a commenter mentions http://defaultprogramseditor.com/ I took a quick peek and it looks solid. It has a "Change 'Open With' programs" feature which lets programs be removed from the "Open With" box. Might work for this case.
