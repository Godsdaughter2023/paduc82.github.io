You also have lots of svchost Processes too, right?  Why is that not a problem for you?  We all know that has been explained a million times.

After Windows XP the csrss was changed to two split out into two sessions where the one with Session ID 0 is for Services and the other one (or other two) are for interactive logins.  When you restart your system the one with Session ID 0 always starts first and runs Services and you can see that from the tasklist command (Services)

When you login that will create another process with a Session ID of 1 - you can see that that from the tasklist command (Console).

Prior to Vista and Longhorn, both the Services and console were all under Session ID 0 but MS split them out:

http://blogs.technet.com/b/voy/archive/2007/02/23/services-isolation-in-session-0-of-windows-vista-and-longhorn-server.aspx

You can enable the Session ID column on Task Manager, sort by Image Name and see that there is always going to be one running under Session ID 0 and at least one more running under some other Session ID (usually Session ID 1)

Session ID 0 is for Services.

Session ID 1 is for Console Sessions (your Interactive Login).

There is sometimes panic among user when they see multiple (6, 8, 10) svchost.exe Processes running in Task Manager but that is normal too (usually) as long as one of them is not running amok.  Now that you have Process Explorer you can understand that too.


To that I say:

To help understand your svchost.exe Processes and what is running beneath them, read this article and you will be smarter than the average bear:

http://www.bleepingcomputer.com/tutorials/list-services-running-under-svchostexe-process/

You might be able to get some clues using about what is going with your svchost.exe Processes using Task Manager and maybe figure it out.

You will always be able to figure out what is going with your svchost.exe Processes if you use Process Explorer.

Download Process Explorer so you can see what is "really" running on your system, especially behind those multiple svchosts Processes you see running in Task Manager.

Download Process Explorer from here:

http://technet.microsoft.com/en-us/sysinternals/bb896653.aspx

You'll like Process Explorer when you get the hang of it.  Process Explorer is the Windows Task Manager on steroids.

Process Explorer installs nothing so it will not slow your system down since it only runs on demand.

Process Explorer may look a little intimidating at first since it presents so much information, but you will start to get to like the way it works when you are looking for performance problems.   You can even tell PE that you want it to be your new default "Task Manager" from now on.  You can still run the original Task Manager too.

Once you get Process Explorer running, expand the columns, drag the corners of the display to make it bigger, etc. so you can see the most information as possible in the window.  Now you can really see everything that is running on the system.

Here is a screenshot of my poor system when I use Process Explorer:
http://img222.imageshack.us/img222/2567/processexplorer.png

The CPU column is usually the most interesting to get started with performance issues - who is using the most?

It is okay and normal to have multiple svchost,exe Processes running in Task Manager.  Important Windows Services are actually running under the svchost.exe Processes.  Sometimes there is just one Windows Service running under a svchost.exe Process, sometimes there are several Windows Services running under a svchost.exe Process.

Sometimes malicious software will hide behind a svchost.exe Process since the malicious software knows you will not be able to spot it in Task Manager.  It will hide behind a svchost.exe Process to fool you, but you can outsmart it.

Malicious software can also disguise itself to appear to be a legitimate Windows Process or it could hide under/behind other Processes that you see running in Task Manager so you cannot see it running.

The malicious software would like to fool you into thinking that you need to use a System Restore Point, perform a Repair Install or reinstall your Windows from scratch when you really don't have to.

When looking at the display in Process Explorer, you would like the most CPU to be associated with System Idle Process.  That is the "free time" on your system so the more free time it has, the better.

If you look at the performance graphs and see red spikes (or not) double click the graph in the top left corner to display the usage graph.  Hover the mouse over any spikes to see what causes them.  Even if the spike has already scrolled past in the display, you can still hover the mouse over the spike to see what caused it.  You can also just wait for a spike to occur and then see what caused the spike.

When things are running smoothly, the svchost Processes will usually not be using too much CPU or may just cause a blip in CPU usage.  Hovering the mouse over each svchost Process will show you what is running underneath it.  There may be one process running under it or there could be many processes running underneath them.  

If one of your svchost Processes is running amok and using too much CPU, right click it and choose Properties, Threads.  Drag the CPU column wider so you can see it and double click the column header to sort the processes so you can see which one is using the most CPU and see if you can spot the offending process (it only shows you the DLLs, but that is usually enough to figure it out).

Hello,

Thank you for posting your query in Microsoft Community.

I would suggest you to run System File Checker tool.

Step 1: System File Checker is a utility in Windows that allows users to scan for corruptions in Windows system files and restore corrupted files.

Press Windows key + X and click Command Prompt (Admin)

Type the following command and press Enter

sfc /scannow

Note: Do not close this Command Prompt window until the verification is 100% complete. The scan results will be shown after this process is finished.

Step 2: If the issue persists, try the following steps:

  1. Open the Task manager.  Here’s a tip:  Press CTRL+Shift+ESC.
  2. Click File > Run New Task.
  3. Make sure you have a check mark beside “Create this task with administrative privileges”.
  4. Type PowerShell.
  5. Type the following in the PowerShell prompt:
    $manifest = (Get-AppxPackage Microsoft.WindowsStore).InstallLocation + '\AppxManifest.xml' ; Add-AppxPackage -DisableDevelopmentMode -Register $manifest
  6. Close the PowerShell window.
  7. Reboot.

Hope it helps. Please feel free to reply in case you face any other issues with Windows in future.

Thank you.
