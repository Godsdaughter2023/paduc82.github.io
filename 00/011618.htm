a. user say:

Applications are designed to be installed in the same partition as the operating system. Some people install them in a separate partition, but it's wrong. It's probably the continuation of an old habit from earlier versions of Windows.

When applications are installed they usually (99% of the time) install system files and registry entries, which are blown away when you upgrade an operating system.

But there's more to the story...

There's a difference between what Microsoft calls 'apps' and what you and I understand that term to mean. When Microsoft refers to an app, it means Windows Store apps. It doesn't mean the software that we install with an .exe or .msi installer.

When you upgrade to Windows 10 and choose to keep your apps, that means Windows Store apps, which get re-installed after the upgrade from Windows Store along with all your settings, which are stored in your Windows Store account.

Microsoft has taken to calling all software 'apps' because (1) it sounds modern; and (2) marketing: Microsoft is hoping that all software - even the kind that usually gets installed with an .exe or .msi installer - will be available through the Windows Store.

You need to reinstall Studio. It's probably a good idea to delete its folders in D: and let the installer do its job.

With that layout you can use sort, insert or delete rows, in both files, doesn't matter.

Hi Ed C for school,

For data in Exchange mailboxes, generally, after some emails are deleted, they are moved to Deleted Items folder.

The duration that deleted items are retained in this folder is based on the deleted item retention period configured for the mailbox. Emails will be moved to the “recover deleted items” folder after 30 days by default, during this period, end users can still restore their emails by themselves. Then emails will go to “purge folder” after 14 days, after that, only administrator can restore the messages. And after another 14 days, emails will be permanently deleted and cannot be restored.
We can enable the In-Place hold s service to prevent data loss. If we have enabled the In-Place hold before deleting those emails, we can use the  In-Place eDiscovery  to recover them no matter how long it has been since they were deleted.

For Public Folder, by default, the deleted item retention uses organization retention default which means it is the same as mentioned above. We can also extend the recoverable period by following the steps below:
1. Login to Exchange Admin Center (https://outlook.office365.com/ecp/ ).

2. Select public folder->public folder;
3. Double click your public folder, then click limits;
4. We can set the retain time of the items there.

The recoverable period for the data deleted from SharePoint Online site or OneDrive for Business is the same.
When you delete an item from OneDrive for Business or a team site library, it goes into the recycle bin, where it stays until it is automatically removed approximately 90 days later.

But as a site collection administrator, you can view and manage deleted items across a site collection from the Recycle Bin and Second-Stage Recycle Bin pages.
In the Second-Stage Recycle Bin, all items are automatically deleted in approximately 90 days after they were originally recycled.
That is to say, the deleted items will exist in SharePoint Online for totally 180 days.
Reference:
https://support.office.com/en-us/article/Manage-the-Recycle-Bin-of-a-SharePoint-Online-site-collection-5fa924ee-16d7-487b-9a0a-021b9062d14b?ui=en-US&rs=en-US&ad=US 

As for the azure AD, if we just change an attribute for the user account, this change will be synced to Office 365 directory services. Automatic profile synchronization with the Office 365 directory service occurs at regular predetermined intervals. It may take up to 24 hours before changes appear in a user’s profile.
Reference:
https://support.office.com/en-us/article/Manage-SharePoint-Online-user-profiles-from-the-SharePoint-admin-center-494bec9c-6654-41f0-920f-f7f937ea9723 

For Skype for Business, there is no retention policy assigned for Skype for Business, but we can save the IM conversations in our email Conversation History Folder. Archive Lync conversations and meeting content to Exchange

bob jones aka: cybertaz say:
What you're asking to do cannot be done. Templates are designed based on a specific paper size. You can choose a different paper size to print on but there is nothing that will scale the content accordingly... Essentially, the template needs to be designed based on the preferred paper size.

You might try the Microsoft downloads to see if you can find a comparable template designed for A4. There may also be one available elsewhere on the web.

The best you can do locally is to scale the document as a part of the printing process. You can open the Copies & Pages list, select Paper Handling then check the box & pick the actual paper size. The output will be scaled to fit the paper but it still may not render as you'd like.

Hi,

The orientation is set to allow narrow (portrait) or wide (landscape). Landscape will print with the top of the page being on the left as pages come out of the printer. To me, this is counter-clockwise rotation.

You can rotate text, but that text must be in a table, a textbox, a shape, or a frame. When it is in a shape or a table, you can rotate those with the text in them, as well. (Tables must be floating for this to work.)
