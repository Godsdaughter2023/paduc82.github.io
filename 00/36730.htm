cmdrkeene say:

Holding Ctrl while dragging an icon has always created a copy since the first release of Windows NT 4 back in 1994.  It's not customizable. There are options however that may help:

  1. Take a look at this blog post from a Microsoftie, which explains how you can change your drag threshold (meaning you have to drag a little further before Windows turns your movement into a drag-drop scenario.  This way if you mistakenly dragever so slightly while using Ctrl, it won't cause a copy operation.  You can change this setting yourself in the Registry under the following path:  Current User \ Control Panel \ Desktop (adjust the values for DragWidth and DragHeight).
     
  2. If you do accidentally drag while using Ctrl and copies are created, just press Ctrl-Z to undo - easy since you're already holding Ctrl :)
     
  3. Instead of using Ctrl, consider turning on the checkbox-to-select option. Typefolder into your Start menu to find and open Folder Options. On the View tab, scroll near the bottom and turn on "use check boxes to select items".

Hopefully one of these, or a combination, these help ease the trouble with accidental copy operations.

  1. Right click on the icon for the program
  2. Click on properties
  3. Now click on compatibility TAB
  4. Under compatibility section,  put a check mark on “Run this program in compatibility mode for:”
  5. In the drop down box select windows XP
  6. You can see Input settings section
  7. Under Input settings section, put a check mark on “ Turn off advanced text services for this program’
  8. Click on “Apply”
  9. Click on “OK”

Additional Information: http://windows.microsoft.com/en-us/windows-vista/Make-older-programs-run-in-this-version-of-Windows
