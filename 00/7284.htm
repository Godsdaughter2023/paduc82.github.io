graham mayor say:
You are right the image does look like yours, however the revised version at https://dl.dropbox.com/u/57986755/Doc4.pdf I am sure you will find does not.

The difference is that this file was created using a Windows PostScript printer driver, whereas the other was produced with a non-postscript driver (a Canon inkjet). It seems that the PDF simply translated the document as formatted for the Canon driver, i.e. using the display bitmap and not the EPS instructions.

I don't have a postscript printer available so I setup an HP Laserjet 2300 PS (a standard driver built into Windows) with its output directed to file. This does resolve the Postscript instructions when printed from Word (you can open the PRN file it produces in a text editor and see for yourself). I then used Acrobat to convert the PRN file to PDF.

The bottom line is that you will not be able to do this from Word with the rudimentary PDF converter supplied with Office. You will need Acrobat Distiller to convert the postscript output (which is what the prn file is) to PDF and resolve the EPS commands in the graphic.

Modify repeating table header in subsequant pages - Microsoft Community

c shep say:

Modify repeating table header in subsequant pages

Is there a way to have word modify a repeating header in subsequent insatnces, such as adding the text "(continued)" to all but the first instance of the header?
jay freedman say:
That's a feature that has been requested repeatedly for, oh, about 15 years. No dice.

The most common workaround is to put the "(continued)" in the header row in the first instance and let it repeat on all the other pages. Then, with the cursor in the paragraph above the beginning of the table, insert a text box or shape rectangle (which has an opaque white background and has its border lines turned off) and position it to cover the unwanted "(continued)" in the first instance of the header.

Sweet, huh? NOT. For one thing, floating shapes don't always maintain position with respect to text. If editing of preceding text forces a page break just before the table, the box will be on the wrong page and the table header won't be hidden.
