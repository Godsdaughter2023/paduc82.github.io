jllatham say:

Not sure exactly what you mean by weighted average.  But you can at least start by getting the average of chocolate this way:

This one would include the 0 for chocolate as part of the average:
=SUMIF(A2:A11,A2,B2:B11) / COUNTIF(A2:A11,A2)
This one would give you the average of non-zero Chocolate entries:
=SUMIF(A2:A11,A2,B2:B11) / SUMPRODUCT(--(A2:A11=A2),--(B2:B11>0))

Those assume that the first Chocolate entry is in A2.

In Excel 2007 you could use AVERAGEIFS() to emulate the second formula above:
=AVERAGEIFS(B2:B11,A2:A11,A2,B2:B11,">0")

It turns out that =AVERAGEIF(A2:A11,A2,B2:B11) in Excel 2007 will also give same result as the AVERAGEIFS() formula above.

joeu2004 say:

In Excel, dates are represented by the number of days after 12/31/1899, which Excel displays as 1/0/1900.

In VBA, dates are represented by the number of days after 12/30/1899.  Also, VBA supports zero and (negative) days before 12/30/1899.

Excel treats 1900 as a leap year, in deference to compatibility with Lotus 1-2-3.  Consequently, day 60 (31+29) is displayed as 2/29/1900.

VBA correctly treats 1900 as a non-leap year.  Consequently, day 60 is displayed as 2/28/1900.

They sync up starting with 3/1/1900, which is day 61 in both Excel and VBA.

To demonstrate, run the following code.

Sub doit()
Const n As Long = 31 + 29 + 1
Dim i As Long, d(0 To n, 1 To 3) As Variant
Columns("a:c").Clear
With Range("a1:c1")
    .Value = Array("Excel", "Day#", "VBA")
    .HorizontalAlignment = xlRight
End With
For i = 0 To n
    d(i, 1) = i
    d(i, 2) = i
    d(i, 3) = Chr(39) & Format(i, "m/dd/yyyy")
Next
Range("a2").Resize(n + 1).NumberFormat = "m/dd/yyyy"
Range("b2").Resize(n + 1).NumberFormat = "0"
Range("c2").Resize(n + 1).HorizontalAlignment = xlRight
With Range("a2:c2").Resize(n + 1)
    .Value = d
    .EntireColumn.AutoFit
End With
End Sub

Mail Merge Labels - Microsoft Community

christiewashington say:

Mail Merge Labels

Is there a way to start a mail merge in the middle of the document for when you have a used labels?

macropod say:

The following macro will work with any data source; it does so by modifying the mailmerge output and inserting however many empty rows of labels are required. With this approach, you should never lose more than a few unused labels - from a partly-used row.

Sub MailMergeToDoc()
Application.ScreenUpdating = False
Dim i As Long, j As Long
j = CLng(InputBox("How many rows of labels have been used on the first sheet?", "Skip used labels"))
ActiveDocument.MailMerge.Execute
With ActiveDocument
  For i = .Tables.Count To 2 Step -1
    .Tables(i).Range.Characters.First.Previous.Delete
  Next
  With .Tables(1)
    For i = .Rows.Count To 1 Step -1
      If Len(.Rows(i).Range.Text) = .Rows(i).Cells.Count * 2 + 2 Then
        .Rows(i).Delete
      End If
  Next i
  .Rows.AllowBreakAcrossPages = False
  For i = 1 To j
    .Rows.Add .Rows(1)
  Next
  End With
  '.PrintOut
  '.Close False
End With
Application.ScreenUpdating = True
End Sub

Note: You need to send the merge output to a new document, not direct to printer, but it can be printed from there. To automate that and have the merge perform like a merge to print, un-comment the lines:
  '.PrintOut
  '.Close False
