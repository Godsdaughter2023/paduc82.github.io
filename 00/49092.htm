Windows 10 Storage Setting, Home Folder, and File Explorer

I am more than a little confused by Windows 10 storage and user file management.

I understand that Windows insists that user home folders/profiles remain on the "C:" drive. That trying to move "C:\Users" to any other drive is not supported and unlikely to work correctly if attempted.

First, there are the user settings for storage where I can change the "Save locations" for apps, documents, music, photos and videos, and movies and TV shows. These changes only apply to new files. Old files, if any, must be moved to the new target folders Windows 10 creates on the target drive if all files are to be in the same location.

However, File Explorer is entirely ignorant of the changes in storage settings. In fact, it is entirely ignorant about home folders in general. Instead, it has "Quick Access" and "This PC" entries which provide links to the usual sub-folders in the current user's home folder. These links are unaffected by the user's storage settings. To get them to match, the user has to open a properties dialog for each folder listed for "This PC" which has been moved by changing the associated storage setting and then the "Location" tab must be selected and the "Move..." button activated to change the location. This tedious exercise is required for each moved folder.

Then there are the "Quick Access" links... If they link to the "This PC" links, they should be updated by editing the other link's location -- I need to confirm this on another account because I have already deleted the stale links for "Quick Access" for the current user. Otherwise, each of those links must be replaced with the correct links.

However, in all of this, File Explorer now has no special knowledge of a user's home folder. It is not an option for opening a File Explorer window by default -- "Quick Access" and "This PC" are the only options for a default setting though creating a home folder short-cut on the desktop or pinned to the Start menu or task bar is a possible work-around. Or...the user can simply delete the useless "Quick Access" links added by default and then navigate to their home folder and then right-click "Quick Access" to "Pin current folder to Quick access" to get, well, quick access to their home folder...including the "Contacts," "Desktop," "Favorites," "Links," "One Drive," "Saved Games," and "Searches" standard folders which are not "movable" in storage settings or by editing the location of the "well-known" folder links tied to "This PC" and mostly added to "Quick Access" by default.

What do I not understand about all this or is it really this much of a confused mess? Why is the user's home folder something user's can only find by navigating to "C:\Users?" Why are "Quick Access" and "This PC" links to only "well-known" standard folders in the user's home folder preferred to just opening Explorer up in the home folder by default? Why are storage settings for the various "well-known" folders completely disjoint from File Explorer settings? Why doesn't Windows 10 create symbolic links or, at least, short-cuts (and why are those actually different things?) when "moving" the various "well-known" folders to other locations? And why does all this nonsense presume that only the "well-known" sub-folders are used to organize each user's files -- so that, for example, we end up with nonsense like virtual machine configurations and their associated virtual disks in a sub-folder of the user's "Documents" folder?

I get the idea there is some thought that "Quick Access," "This PC," etc. are somehow "easier" or "more intuitive" than simply having a home folder populated with a default set of sub-folders, but...I'm just not buying it.
