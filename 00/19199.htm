Actually, that is incorrect.  Open the CSV file in Notepad, and you will see that the numbers appear in the file exactly as they appeared in the original Excel file when you saved as CSV.

Liberator17 wrote:

How can I force the .csv to keep all of the numbers I need, either as text or otherwise.

Unfortunately, we cannot -- unless you want to read the CSV file using a VBA procedure.  Of course, you do not have that option if the reader website is not under your control.

By definition, the CSV file contains just ASCII characters.  It is devoid of all type information.

The CSV standard (RFC 4180) allows for enclosing characters between double-quotes.  But the standard does not say how that should be interpreted -- for example, always as text, which seems logical.

Moreover, the standard was written long after Excel implemented its behavior with CSV files.  So technically, the standard does not apply to Excel, except that the standard was heavily influenced by the pre-existing Excel behavior.

Since the CSV file has no type information, Excel interprets all text (between double-quotes and not) as it interprets keyboard data entry.  Consequently:

1. All data that can be interpreted as a number are entered as numbers.  In some cases, the numbers are treated as currency, percentage and dates, depending on appearance in the CSV file.

2. Most numbers are entered will the General format.  That is why we see exponential notation sometimes:  in cells formatted as General, numbers with more than 11 digits to the left of the decimal point are displayed with exponential notation.

(Cells with numbers that interpreted as currency, percentage and dates are assigned corresponding formats.)

karl timmermans (outlook mvp 2012-2018) say:
To review a few things

#1 - The reason you have two folder tree's in the Outlook Nav Pane is because when you deleted the POP account - it did not delete the PST file associated with the account (and you wouldn't want it to be deleted).

The Outlook Nav pane contains the data files configured in your Outlook profile - NOT simply the email accounts.

#2 - Now, the PST file for the POP account would have also contained the default contact/calendar folders since it would have been set as the default data file for the profile. When you added the Imap account, the "default data file" didn't change which means two things
a) the default contact/calendar folders remain in the PST file
b) you have no contact/calendar folders in the Imap folder tree (which is a good thing). Had the Imap account been the first account configured in the Outlook profile, contact/calendar folders would have been created in the Imap folder tree with the added designation of "This Computer Only" which means exactly that, the data is local to the computer only AND if you deleted the Imap account, the OST file would also be summarily deleted resulting in all data in "This Computer Only" folders being lost. The same would be true if Outlook couldn't find the OST file for some reason, it would be automatically recreated with empty contact/calendar folders.

In short, what you have now, with the PST file set as the default for the profile is what you want. Whether you opt to keep any data in any email folders is your choice.

Getting to your question, the reason you have a (1) behind the email address is because the name was already used for the POP account. To change the name f the account as it appears in the Nav Pane, go to Outlook menu --> File --> Account Settings --> Email tab --> highlight the Imap email account --> Change --> click on <More Settings> --> change the Email Account Name.
