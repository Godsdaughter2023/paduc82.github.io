Assuming the computer is fully-patched at Windows Update and Windows Defender is working properly, the behavior you describe suggests that your data may have been infected before you placed it on the flash drive. This suggests that the computer was already-infected when you backed up the data.   • Format the computer's hard-drives & do (another) clean install of Windows; then...

rickcp say:

Win32/Obfuscator is a detection for a threat that has been "obfuscated", which means it has tried to hide its purpose so your security software doesn't detect it. The malware that lies underneath this obfuscation can have almost any purpose.

VirTool:JS/Obfuscator.EK is a detection for these kind of programs/threats that have had their purpose obfuscated to hinder analysis or detection by antivirus or security scanners. It monitors and tracks the websites accessed by the affected computer and attempts to load other scripts from a remote machine. These obfuscations techniques are being used on various kinds of malwares as part of its component.

Error code 0x80508023 means "Threat Not Found" (ERROR_MP_THREAT_NOT_FOUND).
This may occur in some instances when a threat (a malicious JavaScript in this case) was detected (in your Temp folder) by your security software (WD) whilst coming via an infected web page, but when it went to take action, the file was nowhere to be found (as it never saved to the browser cache); i.e., it was located in your temporary Internet files/cache, but it was deleted before WD could act on it.
See a couple of probably related threads [1] & [2].

You may want to Clear the Java cache, and perform a thorough browser clean-up by following the steps (links provided) suggested by user quietman7 in this related thread/post. Then, run a full scan of WD again. You may also want to make sure you're using the latest Java version (test it out here).

Although I would personally think it was probably a 'failed attack attempt', you may wish to make sure no additional malware (not detected by WD) was installed, by running a 'second opinion/on-demand' antimalware scanner such as Malwarebytes Anti-Malware (Free-Edition ONLY!) and/or HitmanPro3 (30 Days - Free License).

Good Luck!

Manual configuration of your Exchange account is not supported in all Exchange configurations (especially when using Exchange 2013).

Outlook requires the AutoDiscover service of Exchange to be properly configured and available. You can then use Auto Account Setup to configure your Exchange account simply by providing your name, email address and password.

=COUNTIFS(Table1[Location],"In-house",Table1[Date Reported],">"&EOMONTH(TODAY(),-1))
