rohn007 say:

If there is only one Product Key sticker on the computer it is almost certainly for Windows.

 

You can use one of these tools to extract the product keys from your computer

 

Tools to Extract Product Key

Normally you have several resources to find your Retail Product Key.

1.      Your original package or purchase confirmation email

2.      Online download account

3.      You can extract the Product Key from the machine itself

4.      Image copy backups of system with Office installed

5.      Activation backup utilities

From what I’ve read, MS uses a file called TOKENS.DAT to store encrypted product activation keys:

C:\ProgramData\Microsoft\OfficeSoftwareProtectionPlatform\Tokens.DAT

siridm say:

Hi Martin,

as I went on combing through the forum, I have finally found a post related to the issue, explaining how to solve this with a simple work around. In my specific case, and probably yours, it works :)

Let me explain in 5 steps:

1. Go to your Microsoft account settings page, in the Alias section.  Primary alias should be set by default to your original @outlook.com/@hotmail.com email address.

2. Then add a new @outlook.com alias.

3. Set this new @outlook.com alias to primary alias.  

4. Now switch back one more time to set your original @outlook.com/@hotmail.com email address as the primary alias.

5. When you go back to your Email settings, in the accounts section, your original @outlook.com/@hotmail.com email address should be set to Send & Receive.

Then, you should NOT be experiencing anymore the "on behalf of" issue when sending mail with your original @outlook.com/@hotmail.com email address (do the test).

If you don't need the new @outlook.com alias you have created, you can delete it without any consequence (except that the name you used will be unavailable for 30 days if you change your mind...)

Have a try and tell me if it works fine for you.

COUNTIFS function and arrays - Microsoft Community

peter hiestand say:

COUNTIFS function and arrays

Using Office 365 and Excel. I need to use the COUNTIFS function with 2 arrays, each of which I have defined as separate Names (e.g., Array1, Array2). Array2 is a column of dates, and I wish to use the MONTH function to extract the month from each date. The criteria for each criteria range (i.e., Array 1, Array2) are in cells A1, A2. So, it looks like the following:

=COUNTIFS(ARRAY1, A1, MONTH(ARRAY2), A2)

However, this expression always generates a generic error message that is not specific to any of the used functions. If I look at the contents of each criteria range, they are clearly express as arrays:   {value1; value2; value3 . . .}. The problem is identifiable to be the use of the MONTH function within the COUNTIFS function. That is, if I create a separate column in the worksheet in which each cell contains the value of MONTH(ARRAY2), which I'll identify here as ARRAY3, then the COUNTIFS function works fine.

So, what am I doing wrong with the inclusion of MONTH(ARRAY2)? Or, is it simply that the COUNTIFS function does not allow doing any internal functions on arrays?

hansv mvp say:

Functions such as COUNTIF, SUMIF, COUNTIFS and SUMIFS only work with ranges, not with arrays that are the result of a function.

You could use the following instead:

=SUMPRODUCT((ARRAY1=A1)*(MONTH(ARRAY2)=A2))
