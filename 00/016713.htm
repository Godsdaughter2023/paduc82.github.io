You do not need any code to do that.

 

The city combo box's RowSource should be:

   SELECT CityID, PersonalCity, PersonalCounty, PersonalState, PersonalZipCode

ken sheridan say: Your City table really needs to be decomposed as it currently contains transitive functional dependencies; city determines county, and county determines state.  I'll come back to zip codes below.

The decomposition would result in the following tables:

Cities
....CityID (PK)
....City
....CountyID (FK)

Counties
....CountyID (PK)
....County
....StateID (FK)

States
....StateID (PK)
....State

I assume that when you say 'I have duplicate Cities with different Zip Codes' this means that, like UK Post Codes, each city can include more than one Zip Code, but a ZipCode does not relate to more than one city.  In which case to model this would require the following table:

ZipCodes
....ZipCode (PK)
....CityID (FK)

i.e. there is a many-to-one relationship type from ZipCodes to Cities.

When it comes to your Churches table this would require a CityID foreign key column, but not a CountyID or StateID column as these are determined by CityID, so to include them would introduce redundancy due to the transitive dependencies, and the consequent risk of update anomalies.  But it would also require a ZipCode column as this is not determined by CityID.

For data entry you can have a combo box bound to the CityID column, whose RowSource is a query which joins the Cities, Counties, and States tables, returning the CityID in the first column and the City, County, and State values in three other columns.  The first column would be hidden with the other three visible in the combo box's list, enabling you to select a city.  Unbound text boxes would reference the Column property of the combo box to show the county and state for the selected city in the way Marshall described.  But not the ZipCode; this would be selected from a second combo box, cboZipCode say, whose RowSource references the CityID combo box, which I'll assume is named cboCity, as a parameter:

    SELECT ZipCode FROM ZipCodes WHERE CityID = Form!cboCity ORDER BY ZipCode;

In the AfterUpdate event procedure of the cboCity control the cboZipCode control would be firstly set to Null, and requeried so that it lists only the available zip codes for the selected city with:

    Me.cboZipCode = Null
    Me.cboZipCode.Requery

You'll find an example of a form which uses a combo box set up as your cboCity control would be in the file ComboDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file uses the local administrative units of civil parish, district, and county in my area, which have the same hierarchical relationship as city, county, and state in your case.  The relevant form is that using a multi-column combo box and text boxes (frmLocations_Cont_SCB).  You'll see that this includes a combo box to select the parish, whose list also includes columns for the district and county.  The form does not include a  control for post codes, however.

andreas killer say:

Am 27.01.2011 20:48, schrieb LNicholson:

I have a number of spreadsheets of variable lengths.  I have a macro that inserts a line every 25 lines and then adds a formula to the cell:

ActiveCell.FormulaR1C1 = "=COUNTA(R[-25]C:R[-1]C )", which results in =COUNTA(G2:G26) and then =COUNTA(G28:G52), in the appropriate cells.

A better way is to use the range object and build the range between to cells and get the address form that range to build the formulas.

Andreas.

Sub Test()
  Const Step = 25
  Dim First As Range, Last As Range, TheEnd As Range
  Dim Finished As Boolean
  'Start on this cell
  Set First = Range("G2")
  Do
    'The last cell of this section is 24 lines below
    Set Last = First.Offset(Step - 1, 0)
    Set TheEnd = Cells(Rows.Count, First.Column).End(xlUp)
    'Are we over the end?
    Finished = Last.Row >= TheEnd.Row
    If Finished Then Set Last = TheEnd
    'Insert a row below
    Last.Offset(1, 0).EntireRow.Insert
    'Fill the formula in this cell
    Last.Offset(1, 0).Formula = _
      "=COUNTA(" & Range(First, Last).Address(0, 0) & ")"
    'Go to the next section (count the row with the formula!)
    Set First = First.Offset(Step + 1, 0)
  Loop Until Finished
End Sub

andreas killer say:

This VBA code uses ADODB connection and recordset. My VBA project references ADO 2.7

That means you use early binding, rewrite your code and use late binding.

https://support.microsoft.com/en-us/kb/245115

E.g. change

  Dim Connection As ADODB.Connection
  Set Connection = New ADODB.Connection
to

  Dim Connection As Object
  Set Connection = CreateObject("ADODB.Connection")

Search the web for "ado late binding vba", there are tons of samples.
