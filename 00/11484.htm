Private Sub Worksheet_Change(ByVal Target As Range)

rich007a say:

Try this, which will only add new items to the list:

First add an ActiveX ComboBox (assumed it's called ComboBox1).

Set up your ListFillRange via the Properties window to: Sheet1!$A$1

Put the initial value for your list in cell A1

Also add a LinkedCell: Sheet1!$D$1

Now right-click on the sheet's tab and paste this code:

Option Explicit

Private Sub ComboBox1_Update()
    Dim c As Range
    Dim nextRow As Long
    Application.EnableEvents = False
    If ComboBox1.Value = "" Then GoTo ExitPoint
   
    For Each c In Range(ComboBox1.ListFillRange)
        If ComboBox1.Value = CStr(c.Value) Then
            GoTo ExitPoint
        End If
    Next c
    
     'Checked all existing values so need to add
    nextRow = Rows(Range(ComboBox1.ListFillRange).Rows.Count).Row + 1
    Sheets("Sheet1").Range("A" & nextRow).Value = ComboBox1.Value
    ComboBox1.ListFillRange = "=Sheet1!$A$1:$A$" & nextRow

ExitPoint:
    Application.EnableEvents = True
End Sub

Private Sub ComboBox1_KeyDown(ByVal KeyCode As MSForms.ReturnInteger, ByVal Shift As Integer)
    If KeyCode = 13 Then Call ComboBox1_Update
End Sub

Private Sub ComboBox1_LostFocus()
    Call ComboBox1_Update
End Sub

 

Back in Excel, exit design mode and select the intial value from your ComboBox.  Now try typing a new value.

With the above code, the list will only be updated when either the user hits Enter, or the user leaves the ComboBox.  If you want to move your ListFillRange elsewhere, I've highlighted in bold the parts of the code you will need to change.  You could change the code so you only have to enter these detail once in the code - I'll leave that to you.

scottgem say:

Yes, do not use parameter prompt queries. Parameter prompts are very limiting. And you cannot do what you want with them.

Instead create a form with two text boxes named txtStart and txtEnd. In the query change the criteria to:

BETWEEN Forms!formname!txtStart AND Forms!formname!txtEnd

On your report add two text boxes to your header. Set the Controlsource of the textboxes respectively to:

=Forms!formname!txtStart

=Forms!formname!txtEnd

Add a button to the form to run the report and you should have the dates n the header.

jay freedman say:

That doesn't appear to be possible. At best you can disable the Ctrl+F1 keyboard shortcut, which runs the built-in command ToggleRibbon, by removing that key assignment in the Word Options > Customize > Keyboard Shortcut Customize dialog. That has no effect on the other two ways the ribbon can be minimized, by double-clicking any ribbon tab or by right-clicking the ribbon or the Quick Access Toolbar and choosing "Minimize the Ribbon".

I think it would be far better to spend a couple of minutes at the beginning of the course explaining all three ways, and demonstrating how to restore the ribbon. If you somehow disabled the option on the computers in your classroom and didn't teach them about the toggle, the students would still be vulnerable to accidental minimization as soon as they go to another computer.

You can use Time Machine to restore your installation. That would be my first choice. I would definitely make sure Time Machine has updated before erasing your drive.

andre for directly say:
Open the Finder in the Dock
Open Applications
Select each Office application (Word, Excel, PowerPoint, Outlook and OneNote).

Right click the icon then click Move to Trash
Thats how you uninstall it.

To reinstall Office for Mac:


Step 1: Sign in and install Office

Go to http://www.office.com and if you're not already signed in, select Sign in.

Note: If you bought Office for personal use through your company's Microsoft HUP benefit, you may not be able to sign in here. Find installation help at Install Office through HUP. https://support.office.com/en-us/article/instal...

Sign in with the account you associated with this version of Office.

Tip: Depending how you got Office, this account can be a Microsoft account or work or school account. If you forgot your account details, see I forgot the account I use with Office. https://support.office.com/en-us/article/i-forg...

On the Office home page, do the following depending on your version of Office.

Select Install Office > Install if you signed in with a Microsoft account.

Select Install Office Apps > Office 2016 if you signed in with a work or school account.


This begins the download of Office. Follow the prompts on your screen to complete the install.


Step 2: Install Office

Once the download has completed, open Finder, go to Downloads, and double-click Microsoft_Office_2016_Installer.pkg (the name might vary slightly).

Tip: If you see an error that says the Microsoft_Office_2016_Installer.pkg can't be opened because it is from an unidentified developer, wait 10 seconds and then try double-clicking the installer package again. If you're stuck at the Verifying…. progress bar, close or cancel the progress bar and try again.

On the first installation screen, select Continue to begin the installation process.
First Mac 2016 installation screen with "Continue" highlighted

Review the software license agreement, and then click Continue.

Select Agree to agree to the terms of the software license agreement.

Choose how you want to install Office and click Continue.

Review the disk space requirements or change your install location, and then click Install.

Note: If you want to only install specific Office apps and not the entire suite, click the Customize button and uncheck the programs you don't want.

Enter your Mac login password, if prompted, and then click Install Software. (This is the password that you use to log in to your Mac.)

The software begins to install. Click Close when the installation is finished. If Office installation fails, see What to try if you can't install or activate Office 2016 for Mac.
