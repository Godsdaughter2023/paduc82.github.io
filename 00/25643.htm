jay freedman say:

Putting my oar in, unasked <grin>...

From a user's point of view, a single userform with multiple check boxes is more friendly because it takes far fewer clicks than one form per question. It can also be easier to program, because all the information needed to make decisions is available in one place, rather than having to pass information from one userform to the next. Finally, it's easier to create (and later to modify) the logic in the VBA that powers the userform than to do the same in nested IF fields in the template body.

Addressing your specific project with the single-userform model, the code that decides which building blocks to insert will reside in the procedure assigned to the Execute button.* The only complex part will be associated with the SBU class of text boxes; the ITAR and EAR logic is just "if the box is checked, insert the text box".

The following shows the kind of logic you can use. Just replace the MsgBox statements with ones that insert the correct building block entry (a.k.a. Quick Part) in the correct location in the document.

Private Sub cmdExecute_Click()
     If cbSBU.Value = True Then
        If cbNASA.Value = True Then
            If cbProprietary.Value = True Then
                If cbLimited.Value = True Then
                    MsgBox "Insert SBU - NASA - Proprietary - Limited text box"
                Else 'cbLimited.Value = False
                    MsgBox "Insert SBU - NASA - Proprietary text box"
                End If
            Else 'cbProprietary.Value = False
                If cbLimited.Value = True Then
                    MsgBox "Insert SBU - NASA - Limited text box"
                Else 'cbLimited.Value = False
                    MsgBox "Insert SBU - NASA text box"
                End If
            End If
        Else 'cbNASA.Value = False
            If cbProprietary.Value = True Then
                If cbLimited.Value = True Then
                    MsgBox "Insert SBU - Proprietary - Limited text box"
                Else 'cbLimited.Value = False
                    MsgBox "Insert SBU - Proprietary text box"
                End If
            Else 'cbProprietary.Value = False
                If cbLimited.Value = True Then
                    MsgBox "Insert SBU - Limited text box"
                Else 'cbLimited.Value = False
                    MsgBox "Insert SBU text box"
                End If
            End If
        End If
     Else ' cbSBU.Value = False
        MsgBox "Insert Not SBU text box"
     End If
     
     If cbITAR.Value = True Then
        MsgBox "Insert ITAR text box"
     End If
     
     If cbEAR.Value = True Then
        MsgBox "Insert EAR text box"
     End If
     
     Unload Me
End Sub

There are ways you could make the code more compact, but this structure makes it easier to read and understand in the inevitable case when you need to add to or modify the template's functions.

As an aside, I count only 9 unique boxes for the SBU and Not SBU choices, not 10 -- did I miss something?

*A bit of general advice: (1) Use the Properties pane to change the names and captions of all the checkboxes and buttons on the form to reflect their functions. For example, change the name of the Execute button to cmdExecute, and change its Default property to True so the Enter key will "press" it. (2) The name of a button or other control will appear in the names of the event procedures that it can run. For example, the click event procedure for a button named cmdExecute will automatically be named cmdExecute_Click(). If you later change the button's name, you'll have to manually change the procedure's name to match. However, the Caption (the visible text on the control) can be anything, and it isn't related to the control's name or the names of its procedures.
