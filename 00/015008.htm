elizabethhirsch say:

Project Professional Subprojects and Master Project

I have a Master Project consisting of two subprojects. The two subprojects represent the scope being performed by two different companies; each subproject has identical WBS's but different resources. Some tasks are being performed by both companies and I was hoping the Master would reflect that but it keeps the two subprojects separate even when I try to sortby start date which should blend the two subprojects...what am I doing wrong?!
john - project say:

ElizabethHirsch,

It sounds like you created a dynamic master instead of a static master. In a dynamic master the subprojects are not actually part of the master, rather the master contains pointers to each of the individual subprojects. That's why common tasks don't combine when you do your sort.

For reference, dynamic masters are prone to corruption so I suggest you first open the existing master, select each subproject insertion point summary line and then go to Task Information > Advanced tab. Un-check the "link to project" option. Repeat this for the second subproject. Then save all files, that will properly break all links. The unlinked master is now a static master that is a single file that you can filter, sort as desired.

Because a static master is a snapshot in time of all the inserted subprojects you will need to create a new one each time you want an update. However, next time un-check the "link to project" option in the lower right corner of the Insert window when you want an updated version. You can automate this process by recording a macro of your creating the static master so the next time you simply run the macro.

Hope this helps.

John
