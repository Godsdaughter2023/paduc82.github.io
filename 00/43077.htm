richardmerriman say:

No, sorry, I've tried everything I can find on the web that even slightly sounds like it might work.  Nothing until now...  

I remember from the old days of a much less stable windows where things went wrong with the page file.  So, I checked and found my page file was way to small.  Probably because my c: drive is to full.  So I changed my pagefile to D: and made it start size, and max size, to 1.5 times my ram.  I have 8 gig ram, so I made the page file 12 gig.  It was listed at just over 4 gig.  That is very small these days.

Results:

All slow downs have stopped.  The whole thing is now smooth as silk!   :-D

How to do this for anybody who thinks this could help:

1. open control panel,

2. open "System",

3. click "Advanced system settings", bottom left side listings,

4. under "Advanced" tab, in "performance box" click "settings" button,

5. Click "Advanced" tab,

6. In "Virtual Memory" box, click "change" button,

7. Select the drive you want to use.  (I had to use my D: drive because my C: was to full.

8. Click "Custom size" radio button, put at least 1.5 times the size of your system ram as value in both the "initial size" and in the "Maximum size".  Boxes are in megabytes.  Ram is in gig.  8 gig ram is 8000 mb ram  so if you have 8 gig ram put 12000 MB in both fields.

9. click "Set" button.

10. Restart when ready for change to take effect.

Note:  The reason I say use the same size for both is the virtual drive this creates as system memory extension will not need to be defragmented if the size does not ever change.  At least, that's what I heard.....   :-D

Another note:  If you let the drive you chose fill to the point the available space is smaller than your settings you will slow down again.

    daniel bardsley say:

    Solved

    This is how we solved the issue for our organization (21+ computers showing the issue). Thanks to @KenTraub for posting the error and location of the repositories

    Determine the Registered vs. unregistered directories 

    1. Click Start > type regedit > launch regedit > expand to the location: HKLM\Software\Microsoft\Windows NT\CurrentVersion\Print\PackageInstallation\Windows NT x86\DriverPackages 
    2. The first entry that starts with ntprint.inf_x86... is the registered driver > make note of the last characters for later. All the other drivers are not registered

     

    Obtain rights to the file repositories that are unregistered

    1. Open a file explorer window to the location: C:\Windows\System32\DriverStore\FileRepository > scroll down to the ntprint.inf_x86... directories > Select the first ntprint.inf_x86... directory that is not the registered one > right click Properties
    2. Click the Security tab > click the Advanced button > click the Change link to change ownership > type the signed in UserID > click Ok > Click the check box for Replace owner on sub-containers and objects > click Ok
    3. Click the Edit button > click Add > type the signed in UserID again > click Ok > click the Allow check boxes for Full control and Modify > click Ok > click Ok 
    4. Repeat the steps above for all the unregistered ntprint.inf_x86... folders

     

    Move the unregistered repositories

    1. Select all the unregistered ntprint.inf_x86... folders by holding the ctrl key while clicking the folders > right click Cut > Browse to c:\windows\temp > right click Paste
    2. You may need to give permission to have the folders moved

    Try the upgrade again
