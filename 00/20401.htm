ken sheridan say: If you nevertheless want to show those clients in the report then you have two options:

1.  Base the report on a query in which the join from Clients to the referencing table is a LEFT OUTER JOIN.

2.  Set up the report so that you have a clients parent report, based solely on the Clients table, and within it a subreport based on the referencing table.  Set the LinkMasterFields and LinkChildFields properties of the subreport control in the parent report to ClientID, or whatever is the name of the keys.

If you are restricting the query on a column in the referencing table you should use the latter method as, if you do this in the query, it will in effect change the LEFT OUTER JOIN to an INNER JOIN and the two clients will disappear again.  There are ways you can build a query which doesn't do this, but it's a little trickier than your normal run of the mill query.

Excel 2010 vs 97-2003 problem - Microsoft Community

starting new @ 47 say:

Excel 2010 vs 97-2003 problem

I have Office 2010.  I have done something where now all my Excel files only save as 97-2003 files.  This is creating problems.  How can I fix this to save as 2010 files?
jllatham say:

Check your default SaveAs setting:

[File] Tab, "Options" choice.

In the Excel Options window, select the "Save" item and look to see what the very first setting on the page is, next to Save files in this format:.  If it happens to be Excel 97-2003 Workbook (*.xls) then change it to something else that you prefer from the list of available formats.

If you make a change, I recommend then closing Excel and reopening it before depending on the setting to take effect for new workbooks.

For existing workbooks, if you open a .xls file, it is probably going to just save it as a .xls type unless you take action to File --> SaveAs to force it to a different type.
