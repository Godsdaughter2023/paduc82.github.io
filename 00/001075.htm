Mac Word 2008 through to 2016 all share a bug that stops images from being properly displayed in outline view. No fixes for about a decade. A big thumbs down.

When will such a basic thing finally be fixed?

john korchok say:

As Bob explained, Outline view is for working with text outlines and is intended to provided a faster image-free display. Images always slow Word down. If images are vital to your organizational work, please switch to Print Layout. It's slower, but you'll see everything.

If you like you can make a suggestion at the Word User Voice forum that Microsoft should change this: Word for Mac User Voice. If enough other users vote for your suggestion, Microsoft might change it's mind.

bob jones aka: cybertaz say:

It isn't a matter of file type. It's a matter of whether the object is inserted as In Line with Text or whether it has Text Wrapping applied to it [i.e., floating].

Outline & Draft views are designed to provide maximum performance for working with the textual content of a document. This is exceptionally significant when dealing with large, complex documents which contain many diagrams, illustrations, etc. Floating graphics are a major detriment to performance, so they are not shown. Since In Line objects are an integral part of the text flow they are displayed. Most experienced users who deal with documents of that type, however, will invoke the Image Placeholders preference so that even In Line objects aren't displayed while using those view. Sadly, that Preference appears to have been excluded from Word 2016.

graham mayor say:
The usual causes of Address labels not lining up are

1. You have added a header/footer to the normal template (rename the normal template)
2. The page size set in the printer properties doesn't match the page size of the label sheet.
3. You have a scaling option set in the zoom section of the print dialog
4. Your printer has poor paper handling. Some printers will slip when faced with label stock and thus are incapable of feeding the label correctly. Cleaning the rollers in the print path may help.

If this is an Avery template you are using, you can probably discount '1'.

8620 labels are the same size as 8920, which is included in the vast list of label sizes included with the Word label tool. Instead of the Avery template, use the built in label configuration.
