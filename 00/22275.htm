graham mayor say:

Storing client addresses in individual files is not the most efficient way of storing addresses, however the problem is not insurmountable. The following macro run from a macrobutton field eg - { Macrobutton InsertClientAddress Click here to insert address } will insert the contents of the selected address document in place of the macrobutton field.

Sub InsertClientAddress()
Dim strFilename As String
Dim fDialog As FileDialog
Set fDialog = Application.FileDialog(msoFileDialogFilePicker)
With fDialog
    .Title = "Select Client Address Document and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , _
        "Client Address"
        Exit Sub
    End If
    strFilename = fDialog.SelectedItems.Item(1)
End With
Selection.InsertFile FileName:=strFilename
End Sub

Is this Project file size normal ? - Microsoft Community

rolando say:

Is this Project file size normal ?

I have a projectfile of 24 Mb. After opening and saving it, allocated space drops to 10Mb probably due to defragmentation. However, is this 10Mb size normal for a file with 3000 lines, of which many tasks are more than 1 level ?

What causes the increase in size the most ? levels of tasks, predecessors ?

joël séguin say:

Hi Rolando,

10 Mb for such a project file is normal.

The deflation you experience is probably due to file reorganization. When you save, Project actually saves just what was updated, appended to the file. That is done to save time. When you open the file and save it right away without applying any change, the whole file is resaved completely, from scratch, thus doing a deflation.

The file size is mainly function of:

  • the number of assignments (resources on tasks) -- of course, the more tasks you have, the more assignments you have.
  • the duration
  • the tracking method (actuals entered on a day-by-day basis is much more expensive than %Complete progress)
  • the number of baselines saved
  • the number of customized fields used
  • the quantity of notes in the Notes field (avoid inserting files and images)
You shouldn't worry about a 10Mb file, unless you need to transfer it often. If so, consider splitting your project into phases or sub-projects.

Hope this helps,
Joel
