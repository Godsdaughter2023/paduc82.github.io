How many records is too many? - Microsoft Community

madenna say:

How many records is too many?

I have a table with nearly 86,000 records.  I inherited the table as part of an older database that we needed to update several years ago.  The table structure was fine, but now due to a change at the state level, the way we track data is changing and the old structure will no longer work.  I can easily transform the existing data to work with the new structure, but it's going to result in nearly 140,000 records to start with.  I know that's not really considered big, but the table will continue to grow at a rapid pace.  At what point do I need to worry about the table growing too large - or do I need to worry at all about the size?  Are there any things I need to take into consideration as the table begins to grow?  I'd appreciate any thoughts.

 

 

john w. vinson say:

There is no explicit limit on the number of records in a table; I have used a table with 1,300,000 (approximately) records successfully. The limit  that may be more critical is that the database (the backend, containing the tables) cannot exceed 2GByte in total - including all the tables, system tables, etc.

As the table(s) get large, it becomes more important to optimize the structure of the databases: proper indexing, good query design, setting up your forms to retrieve as few records as possible rather than basing forms on the entire table, etc. There's some good suggestions at Tony Toews' performance FAQ:

http://www.granite.ab.ca/access/performancefaq.htm

Of course, you also have the option of moving the data into a more robust client-server database system such as SQL/Server, MySQL or the like. You can adapt the Access frontend to link to these tables; you'll often need some tweaking but in general it's pretty straightforward.

Get a link
Choose this option to share items with lots of people you might not even know personally. For example, you can use these links to post to Facebook, Twitter, or LinkedIn, or share in email or IM. terfar say:

Turn on the non-printing characters and make sure that all there is only a single paragraph mark in the H&Fs and check the paragraph formatting to make sure that the Space Before and Space After are set to zero as well as the line spacing is set to zero. Change the font size to 1 pt.

Then go to Page Setup and set the H&F margins zero.

Note 1: It that it is unfortunate that once you create an H or F in a document, it takes up a minimal amount of space even if there is nothing left. The only way to remove the H&Fs completely is to copy the document body into a new blank document and in Page Setup make sure that the H&F margins are set to zero.

Note 2: If your text just exceeds a single page or two pages, if you switch to Print Preview you can execute the shrink to one page command. This command also shrinks three pages to two pages, despite the name of the command! You may need to add the traditional Print Preview command to your QAT.
