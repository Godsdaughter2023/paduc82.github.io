lemp say:

Click F2 instead of F1.  This will open the BIOS configuration utility. 

When you removed the old battery (or if the old battery had completely died), the BIOS settings should have gone to their default values.  It's surprising that the default values do not work properly for you.

When you get to the BIOS configuration utility, you should see something like this:

I can't tell you exactly what the entries should be, but it sounds as if "Auto Power On" is now enabled, so you should disable it (as shown above).

Before you make any changes in the BIOS configuration, write down the setting(s) before you change them so you can restore them if your change doesn't do what you want.

The "diskette seek failure" refers to your floppy disk drive.  If you don't want to see that message, you can enter the "Drive Configuration" section and disable the floppy drive. 

Also look in the "Integrated Devices (LegacySelect Options) and make sure that the interface setting for your hard drive is set to "Auto"

The point of running registry cleaners back in Win 9x days was to stop the registry from growing too large. Windows 9x did not like large registries and the Microsoft registry cleaner of the day used to remove only the safest of stuff.
 
XP, Vista, Windows 7 registry can grow to way larger sizes than any user is ever going to reach, and unlike Windows 9x, these three operating systems don't fall over orphaned entries because they are ignored.
 
So, is there any point in cleaning orphaned items out of registry when it will make zero difference? NO..
 
The conditions which affected the 9x cousins are in the category 'MYTHS' as far as XP, Vista and Windows 7 are  concerned.

My opinion of Registry Cleaners (and Optimisers) is the following
1) If you need to use them, then you don't have enough knowledge to be able to use them safely.
2) If you have enough knowledge to use them safely, you don't need to use them.
 
Yes, under *certain* circumstances, they can do a wonderful job - but ONLY when the machine is absolutely stable, and the user checks each and every modification before allowing it.
too many times, I've seen people with Office-type installs (where some items are install-on-first-use) that are irretrievably broken by using a registry cleaner, because the dependent registry entries are removed by the cleaner. The only solution then, is a complete uninstall/reinstall of the product, and possibly even that doesn't work properly.
The raw numbers of 'fixes' produced by such tools look impressive - but are, in reality, inconsequential in the scheme of things, and make almost no difference to the functioning or speed of the PC.
There is no substitute for knowledge - and a static program cannot know everything about a non-static PC installation.

Entering PPPoE - Microsoft Community

xb1r athena say:

Entering PPPoE

Login Information



  • Gateway: 192.168.0.1
  • Username: admin
  • Password: (blank)





Entering PPPoE



  1. Open a web browser.
  2. Log into the router by entering the default gateway (192.168.0.1) into the address bar of the internet browser.
  3. When prompted, enter "admin" for the user name and leave the password blank.
  4. Select "Setup" from the menu at the top.
  5. Select "Internet" from the menu at the left.
  6. Select "Manual Internet Connection Setup".
  7. Select "PPPoE (Username/Password)" from the "My Internet Connection is" drop down menu.
  8. In the "User Name" and Password" fields enter the PPPoE user name and password that are provided by the internet service provider.
  9. Select Save Settings.
