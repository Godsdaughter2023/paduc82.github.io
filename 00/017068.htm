This template in question does not readily lend itself to the task in  hand, however it is possible to work around it.

Start by creating an Excel worksheet containing the range of numbers you need - see the Excel section near the end of http://www.gmayor.com/Numbered_labels.htm

The template itself comprises a two page table. Delete everything except the first row of the table, which will give you a document containing a single row table with two tickets.

Use the template as the basis of a Directory type mail merge, attaching the Excel worksheet you have just created as a data source - see http://www.gmayor.com/merge_labels_with_word_2007.htm

Immediately before the Mergefield in the counterfoil of the second ticket add a  {NEXT} field.

Edit both the tickets to have the content you require, then merge to printer or a new document.

myveryownself say: Now, the category is in column A and the question string in column B. The method for assigning numerical values to text strings doesn't change.

I approached the "analysis" specification using a table starting in E2001.

I put column labels in row 2001 and row labels in column E. Specifically, E2002:E:2013 contain a, b, c, ..., j, k, l. And assuming at most 9 questions per category, F2001:N2001 contain q1, ..., q9.

Column O will have the category total. Column P will have the percentage of each category questions (including all repeats) to that total.

After entering the row/column labels, put this in O2002 and copy down to O2013:
   =COUNTIF($A:$A,$E2002)

Then put this in F2002 and copy it into the entire array F2002:N2013
   =IF($O2002=0,"",
      TEXT(COUNTIF($B:$B,F$2001&$E2002)/$O2002,"0/"&$O2002&";;"))

Then put this in P2002 and copy down to P2013:
   =TEXT($O2002/SUM(O$2002:O$2013),"0/"&SUM(O$2002:O$2013)&";;")

Though I formatted results as fractions expressed as text, it is straightforward to change the formatting to numeric decimals or percentages (by not using the TEXT function) or to more embellished text (like "2/14% for q3").

Modify to suit.

steve rindsberg say:

Another trick is to go to Home tab | Editing group | click the triangle next to Replace and choose Replace Fonts.

Replace any "mystery" fonts with a very distinctive font (Algerian's a favorite of mine) then look for instances of the new/distinctive/ugly font.

PowerPoint won't let you replace a doublebyte font with a singlebyte one, so you may have to do a little fiddling to find a suitable replacement font to test with.

Your HGSHeisei Minchota W9 is almost certainly a Japanese font.  Has your presentation file paid a visit to Tokyo lately?  Or did it come from a Mac version of PPT?  Sometimes they toss in Japanese fonts because ... um ... they like to mess with Windows users' heads?  Because Think Different?  Don't really know.

PowerPoint used to be able to tell you what fonts the presentation requires AND which of them were missing, but it no longer does this.  Instead, if someone's used HGSHeisei Minchota W9, PPT shows it on the font menu as though it were an installed font and actually lets you CHOOSE it for text, even though it's not installed on your PC.  To find out for sure, quit PPT, restart and create a brand new blank file.  Add some text and see if you can format it in HGSHeisei Minchota W9. If the font no longer appears, you at least know that it's not installed on your computer.
