New OneNote UI on the Mac: fancy, but confusing

For all the enthusiast running Insider Preview who have turned on the "experimental features" like me, you must have noticed that the user interface of OneNote has had a major redesign.

I have to mention that I generally like the direction in which they are going, but there are obviously a few flaws with the new design (at least in my opinion) that make its eventual adoption somewhat worrying.

Forget about where everything was, because we just applied "flip horizontal" to the UI.

This is kind of confusing for me as a long-time user, now that the list of notes is located at the left (previously located at the right) of the window and the list of the sections and notebooks is also located to the left (previously located at the top). The contents of the actual note is now to the right, while previously it used to be to the left of the window. Basically everything is turned around at 180 degrees. This is kind of radical, in my opinion, and although I've gotten used to it after about 10 days, I still think it's not the best idea ever.

Before:

After:

No more freedom to adjust the interface to your liking

Previously, it was possible to adjust the length of the column containing the list of notes. It was also possible to select whether or not you would like to see small picture previews of your notes. Not any more, this has been taken away.

The titles of your stuff don't matter any longer

Good luck with seeing more than just a glimpse of the title of your notes with the new UI - I rarely get to see more than 2-3 words. The same applies to the titles of the sections, which are also extremely truncated. The title bar of the window itself now contains information about the name of the active notebook and (possibly) the syncing status. Previously, that's where the name of the currently opened note used to be.

I'm not sure how usable that is, especially when you sometimes you can't tell which note you're working on. Consider the screenshot below where I have scrolled down in the note and also in the list of notes. There is practically no way of telling which note I'm currently editing. I mean, I could probably tell, but when you're working on a couple of notes (that consist of more than just a few lines) and constantly moving from one note to the other, this becomes an issue.

White, everything is white

Another issue with the new IS is the fact that, while being extremely elegant, it is relatively hard on the eyes (in my opinion). This is because almost everything is in white now, and that applies to the background of the note, the background of the list of notes and the background of the list of sections, which are displayed side-by-side. It is therefore harder to tell where one of these ends and something new starts, the thin grey lines between the three are not that helpful and this is coming from a person with good eyesight. I can imagine how hard it would be for someone who has problems with their eyes.

A suggestion from a user with no background in design

Now, I'm not really a design expert, but I think that the usability of the new UI could be greatly improved in a very simple way - by making it build on the positive sides of the old UI. Below I attach two very amateurish mock-ups of how I see this integrated in the new UI so that it becomes easier for the eye to tell things apart:

Alternative A:

In Alternative A, the list of notes is separated from the list of sections using a strip that matches the color of the section (currently the strip uses a rather desaturated color so it is harder to spot).

Alternative B:

Alternative B makes it even easier to tell things apart by using the same somewhat desaturated color for the list of notes and using white to distinguish the currently opened note from the rest (currently the opposite is done - the background of the other notes is white and the background of the selected note is shaded in the desaturated color). Assuming that:

Names in column A, Dates in column B

expected results in a new sheet

[Edit..]

Sub AutoF_Name_DatesFromUntil()

'Jan 11, 2015

Const myCo1 As String = "A" '<<<names in column A
Const myCo2 As String = "B" '<<<dates in column B
Dim ws As Worksheet
Set ws = Sheets("Data") '<< source sheet name, change
t1 = DateSerial(2014, 10, 5)    '<< start date
t2 = Date - 1    '<< last date until yesterday

Dim r As Long, c As Long
r = ws.Cells(Rows.Count, myCo1).End(xlUp).Row
c = ws.Cells(1, Columns.Count).End(xlToLeft).Column
Dim myName As String
myName = InputBox("please, select a name", , "name1")
If myName = "" Then Exit Sub
If IsError(Application.Match(myName, ws.Range(myCo1 & ":" & myCo1), 0)) Then
MsgBox "wrong name"
Exit Sub
End If
Application.ScreenUpdating = False
ws.AutoFilterMode = False
With ws.Range("A1").Resize(r, c)
.AutoFilter Field:=Cells(1, myCo1).Column, Criteria1:=myName
.AutoFilter Field:=Cells(1, myCo2).Column, Criteria1:=">=" & Format(t1, "m/d/yyyy"), Operator:=xlAnd, Criteria2:="<=" & Format(t2, "m/d/yyyy")
End With
Sheets.Add
ws.Range("A1").Resize(r, c).SpecialCells(xlCellTypeVisible).Copy Range("A1")
ws.AutoFilterMode = False
ActiveSheet.UsedRange.EntireColumn.AutoFit
Application.ScreenUpdating = True
End Sub

When you reset your phone, you erase all personal data and restore your phone to the state it was in when you purchased it. In other words, you are wiping your phone clean.

 

The personal content that gets erased after a reset includes: Purchased and downloaded apps, ringtones, saved games, contact groups, any pictures, music, or videos, and text messages, which can’t be restored after they’re deleted.

 

Items that can be restored to the phone include: Media content that you previously synced with the Zune software, purchased apps, and saved games.

 

While I didn't want to have to recommend a hard reset, I believe that this will resolve the issue.
