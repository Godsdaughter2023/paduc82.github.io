charles kenyon say:

Here is my attempt:

Sub InsertCrossReference()

'  by Charles Kenyon October 3, 2017

'   based on Graham Mayor vba examples
'   http://www.gmayor.com/word_vba_examples.htm
'   Intercepts InsertCrossReference command and selects Numbered Paragraph as type and Full Context for what to display
'
    SendKeys "{Up 6} + {TAB} + {Up 6} + {DOWN 3} + {TAB}"
    Dialogs(wdDialogInsertCrossReference).Show
End Sub

This would need to be placed in your document template or a Global Template. You may want more {Up} commands. Those take you to the top of each drop-down to start where you want regardless of the previous setting.

You could also attach this macro to a QAT button saved in the same template.

Modifying the QAT in Microsoft Word

Remember that SendKeys is old technology and generally regarded as "flaky unreliable."

suzanne s. barnhill say:

I find table styles very confusing (and see http://www.shaunakelly.com/word/tablestyles/index.html). If you create or modify a table style and set it as the default, then it will apply to all new tables you create in a given document (or based on a given template, if you chose that option). You can have more than one table style; you just have to choose the one you want.

 

That said, row height is not one of the properties you can specify in a table style, so it must be that you have modified the paragraph line spacing rather than the row height. Instead of doing that, you might want to define some specific paragraph styles to apply to the text in the table instead of trying to control the text formatting with the table style.

jc_prored say:

Microsoft Word 2016 Calander

Hi

I created a calendar document by using one of the templates word 2016 offers which brought up a pop up that you could change the dates and year which was fab.  However I made this document so that it could be shared with the rest of our team.  However they all have 2016 versions as well but are unable to to change the dates or have not been provided with a pop up when opening the document onto their computers.  I have google but cant find a suitable answer.  Please help!

jay freedman say:

What is the file extension of the template you used? (You can probably find the file in the folder %appdata%\Microsoft\Templates.) If it's ".dotm", the template uses macros to make changes in the dates. Those macros are not in the document that you made from the template, so they aren't available to team members who have only the document and not the template.

The solution is to distribute the template as well as the document, so the macros will be available to the other people.
