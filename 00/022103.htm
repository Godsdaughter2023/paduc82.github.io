jay freedman say:

Ordinarily, you could use Insert > Quick Parts > Document Properties to insert a content control that you could then copy/paste to another place in the document to repeat data entry. Unfortunately, none of the document property controls are check boxes.

Greg Maxey has a tool for creating custom content controls that behave the same way. You'll find a link to download the add-in at the bottom of the page http://gregmaxey.com/word_tip_pages/cc_var_bm_doc_prop_tools_addin.html (get version 4.0). Unzip the downloaded file and store its contents in the folder %appdata%\Microsoft\Word\Startup. When you start Word, you'll find an Add-In tab on the ribbon, with a button to start the tool.

Put the cursor in your template at the location for the first checkbox on the first page. In the add-in, click the Create/Manage Mapped Content Controls tab and click the Add Pre-defined XMLPart button. That will change the display on the right to the basic_XML tree diagram shown below.

Right-click the mappingElement1 item, click the Insert Content Control box that pops up, and click Checkbox in the list. In the document, the Properties dialog box for a checkbox control will appear. At least add a title, change anything else you need, and click OK.

At this point it's important to move the cursor outside the content control's bounding box, to avoid an error at the next step.

Put the cursor at the location for the second checkbox in the document and repeat the steps above, using mappingElement2. (That is, the second checkbox must be mapped to a different XML node than the first checkbox, and likewise for each additional checkbox.)

If you need more than 6 checkboxes, you can add more XML nodes to the tree. Click on the basic_XML item at the top of the tree and click the Append XML Node button on the left. Type a name for the new node in the box at the bottom left of the add-in and click the Append button at the right.

After you've inserted all the checkboxes you need on page 1, copy the entire block and paste it onto page 2. The result is that the first checkbox on page 2 is mapped to the same XML node as the first checkbox on page 1, and any change in one of them will be repeated in the other one. The second checkbox on each page is mapped to the same second XML node, and so on.

The XML nodes are saved in the template's file, and are inherited by each document based on the template.

jay freedman say:

Hi Denise,

You do need a macro. In VBA, a Document_ContentControlOnExit macro runs automatically each time the cursor exits from any content control in the document.

To update all the fields in a document (that's based on your template) each time the user's cursor exits any of the content controls, copy/paste the following macro into the ThisDocument module of the template:

Private Sub Document_ContentControlOnExit(ByVal CC As ContentControl, Cancel As Boolean)
    ActiveDocument.Fields.Update
End Sub

This macro can be made more specific in a number of ways: (1) It can be made to update fields only if the control being exited is a check box or a text control, but not other kinds of controls. (2) It can be made to update fields only if the title of the control being exited is one of a set that you specify. (3) It can be made to update only the REF fields in the document, or (4) only the REF fields within a bookmark that you create in the template. For example, the following macro combines methods (1) and (4):

Private Sub Document_ContentControlOnExit(ByVal CC As ContentControl, Cancel As Boolean)
    Dim fld As Field
    
    If CC.Type = wdContentControlCheckBox Or CC.Type = wdContentControlText Then
        For Each fld In ActiveDocument.Bookmarks("update").Range.Fields
            If fld.Type = wdFieldRef Then
                fld.Update
            End If
        Next fld
    End If
End Sub

This assumes that you've inserted a bookmark named "update" that surrounds all of the Ref fields on page 2 of the template.


You'll need to copy the same code to each of your templates, so users will have access to the macro in each document they create from any of those templates.

Private Sub deleteShapeData(ByVal visShape As Visio.Shape)
    On Error GoTo errHandler
    Dim visCell As Visio.Cell
   
    If visShape.SectionExists(VisSectionIndices.visSectionProp, False) = True Then
        intCt = visShape.RowCount(VisSectionIndices.visSectionProp)
        For intX = intCt - 1 To 0 Step -1
            visShape.DeleteRow VisSectionIndices.visSectionProp, intX
        Next intX ' count backwards through the shape properties
    End If ' test for shape data


    Exit Sub
errHandler:
    Debug.Print Err.Description
End Sub

allan lu msft say:

Hi Wendell,

 

I’d like to explain that the Office 365 license is not associated with the domains in the Office 365 tenant. We can assign the licenses to users with different domain names as long as we have enough licenses.

 

In addition, for the licenses which contains the same services(like Office 365 Business Premium and Office 365 Enterprise E3), we cannot add multiple licenses to the same user, we can only add different plans with different services (like Exchange Online Plan 1 and SharePoint Online Plan 1) to the same user.

Assign or remove licenses for Office 365 for business

Office 365 Plan Options
