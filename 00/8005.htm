daniel g. grau say:

You could insert text after each endnote, but it will disappear as soon as you enter a change affecting the endnotes, because the list would be regenerated.

You can select the endnotes and press Command Shift F9 to convert them to regular text. This will let you move them next to the quotations.

I would use this method, which requires converting paragraphs to tables and combining the tables:

  1. Make a copy of the document.
  2. Delete all extra material and leave just quotations and endnotes.
  3. There should be one paragraph per quotation. If you need a paragraph break, use two line breaks (Shift Return).
  4. Select all quotations and use Table > Convert > Convert Text to Table. Make a single-column table and reduce its width.
  5. Convert all endnotes to text and repeat Step 4 on these paragraphs.
  6. Now you'll have two single-column tables. Select the endnotes table, copy or cut to the clipboard, place the cursor right after the first cell (outside) in the quotations table and paste. You should end up with a two-column table with a quotations column and an endnote column.

You'll probably need to insert or merge cells to align the material.

Hi,

To start with, it is common for Gigabyte as well as other motherboard manufacturers to include a 2nd SATA controller to add a couple more ports... They are usually intended for RAID, but that is not necessary...

This would not apply to your Computer "B", and probably not "C"... it only has Intel, but would appear to apply to Computer "A"...

The 2nd Controller must not have drivers, or the wrong one, and is not functioning...

Gigabyte calls the 2nd Controller GSATA, but it is actually a JMicron... at least on my machine... I had to search for JMicron drivers to resolve the non-functioning ports on my machine...

Part of the problem is that the Windows Installation doesn't recognize a GSATA chip...

Once the JMicron drivers are installed and the chip is identified as JMicron, everything worked...

This is on a Gigabyte board that is about 5 years old... A lot of machines are now using a chip by ASmedia for the 2nd Controller... I have an issue at one time with that one also, on a Asrock board, but all of the latest builds are working...

I think you have two separate problems.... The 2nd SATA Controller on the "A" machine, and a Registry Issue concerning optical drives that has been happening ever since Win 8 was released...

If the drive(s) do not appear in Device Manager at all... or just one of them, you need to run the following regedit... You will need to investigate drivers for the two non-functioning SATA Ports either before of after the regedit, depending on which ports your drives are plugged into...

1. Run cmd as Administrator (right click lower left corner when Start pops up)

2. Type: or copy and paste  ( copy\paste is better)

reg.exe add "HKLM\System\CurrentControlSet\Services\atapi\Controller0" /f /v EnumDevice1 /t REG_DWORD /d 0x00000001

3. Hit Enter

4. Type: or copy and paste

reg.exe add "HKLM\System\CurrentControlSet\Services\atapi\Controller0" /f /v EnumDevice2 /t REG_DWORD /d 0x00000002

5. Hit Enter


6. Reboot
