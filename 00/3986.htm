karl donaubauer say:

Hi,

 

the usual solution is a calculated column with either a SQL subquery:

 

RunNr: (Select Count (*) FROM [YourTable] As Temp WHERE [Temp].[Id] <= [YourTable].[Id])


 

or a domain function (often with better performance):

 

RunNr: DCount("Id","YourTable","Id<=" & [Id])

 

 

In both cases [Id] means a unique field that identifies the record. I don't know if in your case this would be VehicleID or ClassID.

easona say:

The calling phone number may not match the contact name and therefore the name is not shown when you receive a call or SMS in the following situations:

1. The phone number is defined twice in the People hub, the same number is found for two or more contacts (usually the last 6 numbers of the phone number are significant).

2. The phone number is in a different format than the number received in the incoming call. The phone number may have been stored in the local format while the incoming call number is in the international format (including country prefix), or vice versa. The number may not properly match a contact name in this situation, depending on the device variant and the length of the calling number. In order to verify the correct format of the contact phone number (local or international), check the number which is shown when the call is received and edit the contact details accordingly.

If the issue persists, make sure your phone is running the latest software. Click here to visit the software update support page and update your phone.

paul van der heu say: Make sure International Assist is set correctly. Once you switch to a SIM from another country/region you may need to add the international access code to existing numbers for obvious reasons.
