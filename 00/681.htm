Is it an Outlook.com email account? If so, its already on Office 365. 

Are you leaving mail on the server?  That would be the 12000. When you download mail using POP, outlook keeps track of the last message downloaded by saving the message number in a "Mailbox manager" file hidden inside the pst. If something happens to that hidden message, Outlook will need to start over.  Scanpst or making a new pst are typical causes but a bad internet connection could corrupt it or something could have happens on the server that caused outlook to see them all as new messages. 

You can create a rule to move mail received before yesterday to a new folder - this takes care of the duplicates-  but outlook will need to download the mail unless you move or delete it in the mailbox online (using web access). POP only downloads the inbox, so moving it to a new folder is a good option that allows you to keep the mail on the server. If it is an office 365 mailbox, there is an option in outlook on the web to move downloaded mail to a new folder (to remove it from the mailbox).

macropod say:

The preferred method would be to add it to your Excel workbook, that way you'll have a permanent record.

Alternatively, you could add a MERGEREC field or a MERGESEQ field to the mailmerge main document, but these will start at 1 every time, meaning you'd be reusing the numbers over and over. That wouldn't be terribly satisfactory unless you prefixed the number with, say, the mailmerge date formatted as YYYYMMDD. Another way of avoiding duplicates would be to use a field coded as:

{QUOTE{ASK Offset "What is the starting Invoice #?" \o}{SET Val {MERGEREC}}{=Offset+Val}}

This will prompt you for a starting number each time you run the merge.

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues.
