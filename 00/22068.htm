Access 2007: VBA code to retrieve records count for each table - Microsoft Community

larana626 say:

Access 2007: VBA code to retrieve records count for each table

Using VBA, is there a way to get the records count for each table in a Db? I need to retrieve the table name and total number of records (for each table individually).

hansv mvp say:

The following code will produce output in the Immediate window but you can adapt it for your purposes of course.

Sub ListTables()
  Dim dbs As DAO.Database
  Dim tdf As DAO.TableDef
  Dim rst As DAO.Recordset
  Set dbs = CurrentDb
  For Each tdf In dbs.TableDefs
    If Not tdf.Name Like "MSys*" And Not tdf.Name Like "~*" Then
      Set rst = dbs.OpenRecordset(tdf.Name, dbOpenDynaset)
      rst.MoveLast
      Debug.Print tdf.Name, rst.RecordCount
      rst.Close
      Set rst = Nothing
    End If
  Next tdf
  Set dbs = Nothing
End Sub

bernd p. say:

Back to the roots:

A Onenote notebook is realized as a folder (name = notebook's) within the Windows file system. This is the item you open with Onenote. This folder contains one or more sections as .one files.

And a .onetoc2 file as configuration memory.

This is the standard case.

Any additional folders within the notebook's folder, this subtree, will be interpreted as section groups, belonging to that notebook.

So it's not a good idea to create folders under the notebook for using as normal Windows folders for normal files.

You can create such a mess by opening a normal folder as a Onenote notebook accidentally. And create many unwanted .onetoc2 files .....

So I think there is something wrong with your file structure.

BTW: The standard location for Onenote notebooks is a special Onenote Notebooks folder under Documents.

Bernd
