jay freedman say:
Word documents store their creation date internally, separately from when the operating system says the file was created.

The easiest way to get Word's creation date in Windows 7 is to right-click the file, open that same Properties dialog, and click the Details tab. The "Content created" item is the one you want.

In Word 2007, there are two other ways to see the information.
  • Click the Office button, click Prepare, and click Properties. In the Properties pane, click the Document Properties text in the upper left corner and click Advanced Properties. In the dialog that pops up, click the Statistics tab to see the Created date.
  • Anywhere in the document itself, press Ctrl+F9 to insert field brackets, type the word createdate between them, and press F9.

how do you save macro to personal module in PP - Microsoft Community

bill84345 say:

how do you save macro to personal module in PP

How do you store macro in PP so you can use them with any file like in personal module of Excel

 

Bill

rich007a say:

Hi John and Bill,

John, you are of course right, the "real answer" is to create a ppam addin with ribbon menu, but that's possibly too much for many, especially if they are using 2007.

 

How about this for an idea.:

1) Create a new blank presentation and save it as MyPowerPointMacros.pptm in your Documents folder.

2) Press Alt+F11, insert a module and paste the following:

Sub Macro1()
    MsgBox "Hello from Macro1"
End Sub

Paste any other macros you want to use there.

3) Save the file, then right-click on the QAT > Customize > Choose commands from: Macros and add buttons for your macros ( 'MyPowerPointMacros.pptm'!Macro1 ) to the QAT.

4) Close MyPowerPointMacros.pptm 

5) Create a second new blank presentation and save it as MyPowerPointMacrosAddin.pptm in your Documents folder.

6) Press Alt+F11, insert a module and paste the following code:

Sub Auto_Open()
'Note this will only fire automatically when in a loaded Add-in.
'
    Dim strFileName As String
   
     'Path for file with macros:
    strFileName = CStr(Environ("USERPROFILE")) & "\Documents\MyPowerPointMacros.pptm"
   
     'Open the presentation with macros, but keep it hidden:
    Application.Presentations.Open FileName:=strFileName, WithWindow:=msoFalse
End Sub

 

7) Go back to PowerPoint and Save the file.

8) Save As the file as a PowerPoint Add-in (*.ppam).  The default location for add-ins should show automatically.  Save it there.

9) Close that file.

10) Office button > PowerPoint Options > Add-ins > Control: Add-ins > Go

11) Click Browse and open the MyPowerPointMacrosAddin.ppam file, enabling macros if prompted.

12) Close PowerPoint.

13) Restart PowerPoint, your add-in should load, and it should open your MyPowerPointMacros.pptm file but it will be hidden (just like the Personal.xlsb file in Excel).

Because it is open, all of your buttons on the QAT should still function ok.

 

I haven't tested this in "real life", only tested to check that I can stil run Macro1.  Please let us know if it works ok for you.

The advantage of this is that your macro file is fully editable (and saveable via the VBE) all the time.

You might find that a blank presentation no-longer loads when you start powerpoint.  If that's an issue, just add:

    Application.Presentations.Add

to the end of your Auto_Open procedure in the Add-in (unload the add-in, make the change in the pptm version of the add-in, save, then save as an add-in (ppam), over-writing the existing add-in, then reload the add-in via ppt options.

Cheers
Rich

bernie deitrick say:

Jack,

Try this version:

Sub CreateDesktopFileV2()
    Dim WSHShell As Object
    Dim DesktopPath As String
    Dim FName As String
    Dim wb As Workbook

    Set WSHShell = CreateObject("WScript.Shell")
    DesktopPath = WSHShell.SpecialFolders("Desktop")
    
    FName = ActiveSheet.Range("C6").Value
    If FName = "" Then
        MsgBox "There is no name for the file in cell C6"
        Exit Sub
    End If
  
    Application.DisplayAlerts = False
    ActiveSheet.Copy
    Set wb = ActiveWorkbook
    wb.SaveAs Filename:=DesktopPath & "\" & FName & ".xlsx", _
        FileFormat:=xlOpenXMLWorkbook
    wb.Close False
    Application.DisplayAlerts = True

    Set WSHShell = Nothing
    MsgBox "A file has been saved on your desktop."

End Sub

bernie deitrick say:

Try this - You will need to use a reference in your project to Microsoft Scripting Runtime to find the desktop.  I should note that I have never worked with redirected folders - but the code does everything else you wanted, so try it out.

            

Sub CreateDesktopFile()
    Dim WSHShell As Object
    Dim DesktopPath As String
    Dim FName As String

    Set WSHShell = CreateObject("WScript.Shell")
    DesktopPath = WSHShell.SpecialFolders("Desktop")

    FName = "Parts List for " & Application.InputBox("Enter the job number.", Type:=2)
    If FName = "Parts List for " Then Exit Sub

    Application.DisplayAlerts = False
    ThisWorkbook.SaveAs Filename:=DesktopPath & "\" & FName & ".xlsx", _
    FileFormat:=xlOpenXMLWorkbook
    Application.DisplayAlerts = True

    Set WSHShell = Nothing
    MsgBox "A file has been saved on your desktop."

    ThisWorkbook.Close False

End Sub
