a. user say:

Hello,

Thank you for posting your query on Microsoft Community.

 If a file is open to one user it should be locked as read only to another. Even if the 1st user then drops the connection to the file the 2nd user will still hold it read only until they drop and pick it up again. This has nothing to do with Windows 8.1 file sharing, it would apply in a true server/client scenario as well. Network access is at a file level and has to be complete, you can't have one user update one sector and another a different one in the same file.

If this is happening on your systems there is a possibility that some group policy has been applied by your system administrator. I would suggest you to contact your system administrator to check the possible group policy applied on your system.

red264 say:

That's pretty old school, the x360 only supports XviD/AVI, H.264/MP4 and WMV videos.

H.264/MP4 is the standard video codec used everywhere today (was new when Xbox 360 launched 10 years ago) however the x360 has very limited support for it.

The Handbrake video encoder can convert DVD's into H.264/MP4 but you will have to stick to the universal profile if that still works with the x360, it probably will.

Another problem is the x360 only supports FAT32 file system so files/videos must be under 4GB in size, though most of the output in Handbrake would come under 4GB with universal preset.

WMV encoding is a dead end don't even bother.

A far easier option is buy an Xbox One (lots of cheap second hand ones around now the Xbox One S is available), use MakeMKV on your DVD's, this turns them into MPEG-2/MKV videos (no video conversion so it's very fast), copy your movies onto USB HDD formatted in exFAT (replacement for FAT32 with no 4GB limit) and then play them in the Xbox One video player.

Thats the fastest way to digitize your DVD collection and play from the HDD, the x360 is too old with too many hoops to jump through to make it worthwhile.

Hi,

Thank you for posting in Microsoft community.

 

I suggest you to manually reset the Windows Updates Components.

Please follow the below steps to reset the Windows Updates Components:

  1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.
  2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

   3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

   4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

   5. Type Exit in the Command Prompt to close it.

Now you may try running the Windows Updates and check if the above steps resolve the issue.

How do I reset Windows Update components?

net stop msiserver

2. Delete everything within the folder. Do NOT delete the folder itself.

3. Restart your computer and give Windows Update another try to see if the issue still exists.

Now, let’s restart the Windows Update related services by typing the following commands one after the other and hitting on Enter in Command Prompt (Admin):

net start wuauserv
net start cryptSvc
net start bits

To close the Command Prompt (Admin), type Exit and hit on Enter. Restart the computer.

'mike p say:

fwiw;  

Code 43 "Windows has stopped this device because it has reported problems. (Code 43)"

Cause One of the drivers controlling the device notified the operating system that the device failed in some manner.

Recommended Resolution If you have already run the troubleshooters in the "Try these steps first" section, check the hardware documentation or contact the manufacturer for more information about diagnosing the problem.

Reinstall the device driver manually From Start, search for device manager and select Device Manager from the results.

Right-click the device in the list.Select Uninstall from the menu that appears.

After the device is uninstalled, choose Action on the menu bar.

Select Scan for hardware changes to reinstall the driver.

Note You may be prompted to provide the path of the driver.  If you are asked for the driver and you do not have it, you can try to download the latest driver from the hardware vendor’s website.
