Unfortunately master/subproject linked structures in Project are prone to corruption. You can never rename, move, overwrite, or save off to another location any of the files in the structure. All files should reside on a local drive under a single directory and should never be run over a network.

A better option, if you need to view all files together, is to use a static master. That is created by unchecking the "link to project" option in the lower right corner of the Insert Project window. A static master is a snapshot in time of all the subproject files so it needs to be re-created anytime you need to see all files but that process can be easily automated with a recorded macro. A static master is not prone to corruption and it is a single independent file so all elements therein can be directly formatted.

So, let's see what you've got, or need, and go from there.

Hi AE8425,


This problem occurs if Outlook 2016 cannot connect to the Microsoft Exchange Server Autodiscover service because Autodiscover is not configured or is not working correctly. After you upgrade the program or create a new profile, Autodiscover is required to configure an Outlook profile the first time that Outlook is started.

The problem may occur for one of the following reasons:

• The previous version of Outlook was configured to connect to Exchange Server without Autodiscover. This was done by manually specifying the server settings.
• Autodiscover was previously available but is no longer available.
In Outlook 2016, the option to manually configure an Exchange account is no longer available.

To resolve this problem, please check if Autodiscover is functional and can be access by Outlook 2016.
If you are using an Office 365 account, please check if the DNS CNAME record for Autodiscover has been added.
If you are not using an Office 365 account, please contact the email provider to check if Autodiscover is configured correctly.

Okay The Merp 2.0 Folder is Located by:

  1. Open hard Drive (click on the Hard Drive Icon in Finder whatever it is named)
  2. Locate Folder Called Library.
  3. Click Application Support
  4. Click Folder Named Microsoft. 
  5. Locate MERP2.0 Folder
  6. Open MERP2.0 Folder
  7. Click on MERP2.0.app
  8. Locate the MERP Menu and click Preferences
  9. Uncheck the checkbox.
