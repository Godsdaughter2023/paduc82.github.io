andreas killer say:

Am 26.12.2010 05:40, schrieb Gyanprakash:

I am developing an Utility wherein i am supposed to open Excel workbook only once which will Activate Workbook_Activate Event and call a macro . and then i am supposed to minimize the Excel Application and when i again restore it using shortcut key , The same procedure should repeat.

IMHO this is possible, but not with VBA.

Every event is designed to work only inside excel. This means that we do not receive notification of (MS-)Windows events.

The application provides us some events:

--- schnipp ---
Dim WithEvents App As Application

Private Sub Workbook_Open()
  Set App = Application
End Sub

Private Sub App_SheetActivate(ByVal Sh As Object)
  Debug.Print "App_SheetActivate"
End Sub

Private Sub App_WorkbookActivate(ByVal Wb As Workbook)
  Debug.Print "App_WorkbookActivate"
End Sub

Private Sub App_WindowActivate(ByVal Wb As Workbook, ByVal Wn As Window)
  Debug.Print "App_WindowActivate"
End Sub

Private Sub App_WindowResize(ByVal Wb As Workbook, ByVal Wn As Window)
  Debug.Print "App_WindowResize"
End Sub
--- schnapp ---

But they work only if you minimize/resize/maximize/close a sheet/workbook/window inside excel. Open 2 workbooks and execute this macro to see what I mean:

Sub Makro1()
  Windows.Arrange ArrangeStyle:=xlCascade
End Sub

Your event works now if you click inside this windows.

If you want to get noticed if the whole application is minimized/resized/maximized you must use window subclassing, but this is not easy and IMHO it's not possible to subclass the excel application with VBA (Every attempt I made brought the application to crash).

http://www.aaronballman.com/programming/REALbasic/WindowSubclassingArticle.php

If somebody knows if it is possible and how to do, please let me know.

The only other way I know is to call a sub from time to time and detect the windowstate.

Andreas.

Option Explicit

Private Declare Function IsIconic Lib "user32" (ByVal hWnd As Long) As Long
Private Declare Function IsZoomed Lib "user32" (ByVal hWnd As Long) As Long
Private Declare Function GetForegroundWindow Lib "user32" () As Long
Private Declare Function FindWindow Lib "user32" Alias "FindWindowA" ( _
  ByVal lpClassName As String, ByVal lpWindowName As String) As Long

Sub Test()
  Dim hWnd As Long
  hWnd = FindWindow("XLMAIN", Application.Caption)
  If IsIconic(hWnd) Then
    Debug.Print "Minimized"
  ElseIf IsZoomed(hWnd) Then
    Debug.Print "Maximized"
  Else
    If hWnd <> GetForegroundWindow Then
      Debug.Print "Not active"
    Else
      Debug.Print "Normal"
    End If
  End If
  Application.OnTime Now + TimeSerial(0, 0, 1), "Test"
End Sub

Thank you for explaining your issue further. I understand that you are frustrated about this issue and I'm here to assist you. Let me explain how pop settings work. There's a setting in your mail client that enables to "Leave a copy on the server" or "Don't leave a copy on the server". "Leave a copy on the server" means that when you receive your email, you will receive the email on your Inbox and your mail client if it is online. If your mail client is offline, the messages will stay in the pop folder and you need to move it to your inbox so that you can download it once your mail client is online.
