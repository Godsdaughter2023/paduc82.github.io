Mailboxes - Calendaring - Missing Calendar Items - Microsoft Community

alex rizeanu msft say:

Mailboxes - Calendaring - Missing Calendar Items

Technical Level : Intermediate

Summary

This workflow is for diagnosing missing calendar items for users using Outlook for Windows and Outlook for Macs connected to an Exchange Online or Exchange on-premise mailbox. A missing calendar item is defined as one that is not found in Outlook, OWA, devices or any other views of the calendar.


Details

Restore the Data (If Possible)

  • If items were hard-deleted but not purged, they are recoverable by the end user through Outlook, OWA or MFCMapi.
  • If the items were purged but within the deleted item retention period (typically 14 days ), you can run: In-Place eDiscovery search or New-ComplianceSearch on all mailboxes and follow the steps here to recover the items.
  • If the user has an archive mailbox, check for the missing items there as well. It's possible an MRM policy or something else moved them to archive
    • Discovery search will include the archive by default
    • You can also use MFCMapi to connect to the archive mailbox.
  • If you cannot find the item in 'deleted items' or in 'recoverable items', it's possible the item was not deleted but moved to a different folder within the mailbox. An In-Place eDiscovery search or New-ComplianceSearch can be used to locate the misplaced item.
  • It's possible that another user with full access permissions has deleted or moved the item(s). In Outlook, by default, the item will be placed in the deleted items folder of the user who deleted it, not the deleted items folder of the secondary mailbox they deleted it from. This is controlled in Outlook by the DelegateWasteBasketStyle registry value
  • If the customer is journaling email sent or received within the tenant to an external mailbox, you may have a potential source to restore the missing items from there as well.

Prevention

  • Single Item Recovery - this should be enabled on every mailbox and set to 30 days.
  • Litigation or In-Place Hold - if the customer has the required SKU this is an option to halt all further permanent deletion from the mailbox.
  • If Messaging Records Management (MRM) is deleting the items, a potential result of a misconfigured or misunderstood policy, Retention Hold is an option to stop it temporarily until the problem can be resolved.
  • Don't allow unnecessary permissions to be granted to a mailbox.
  • Limit the number of 3rd party addins used with outlook.exe or make sure they are always kept up to date.

Unknown Solution Data Collection

  1. Run CalCheck for the impacted Calendar. Download. You will want to get the output file (.csv) of flagged items to review what may be problematic. This need to be run on a Windows machine (for Mac users, they will need to create a profile on a Windows machine to get the output, there are no Mac calendar tools).
  2. For Windows users, run Outlook client side logging during repro
  3. For Mac users, run Outlook client side logging during repro
  4. Connect to ExO PowerShell as described here or EMS and collect the results of the following commands : 

  1. Get-Mailbox <Mailbox UPN> | fl > "Cloud_Mailbox.txt"
  2. Get-MailboxPermission <Mailbox UPN> | fl > "Cloud_MailboxPermission.txt"
  3. Get-MailboxFolderPermission <Mailbox UPN>:\Calendar | fl > "Cloud_MailboxFolderPermission.txt"

  1. If you can replicate the issue at will by using OWA only (Outlook closed and mobile devices turned off), then collect a Fiddler trace or a F12 Network Trace of the issue.
  2. Connect with MFCMapi and check if the missing items are present in deleted items, recover deleted items or dumpster(deletions and purges) folder of the impacted mailbox as well as the mailbox of all users having permissions on it.

Note: You can skip step 4 and 5 and run instead a single search on all mailboxes (the impacted mailbox + archive as well as all mailboxes that have delegate, full access of any type of editor permissions on the impacted mailbox). For additional information see: In-Place eDiscovery searchor New-ComplianceSearch.

  1. If the issue is reproducible, in order to track what or who or what is removing the calendar items, enable Mailbox Audit on the impacted mailbox.
  2. Run Historical message trace to see if the invites were processed by any rules. 
  1. Start-HistoricalSearch -ReportTitle "MessageTraceDetail for Case xxxxx" -StartDate 10/03/2016  -ReportType MessageTraceDetail -SenderAddress  user@contoso.com -RecipientAddress  user@fabrikam.com  -NotifyAddress admin@fabrikam.com

doug robbins - mvp office apps & services (word) say:

The following code should do what you want.

Dim Counter As Long, Source As Document, Target As Document, fname As String
Set Source = ActiveDocument
Selection.HomeKey Unit:=wdStory
pages = Source.BuiltInDocumentProperties(wdPropertyPages)
Counter = 0
While Counter < pages
    Counter = Counter + 1
    Docname = "Page" & Format(Counter)
    Source.Bookmarks("\Page").range.Cut
    Set Target = Documents.Add
    With Target
        .range.Paste
        fname = Left(.range.Paragraphs(3).range.text, 8)
        .SaveAs filename:=fname
        .Close
    End With
Wend

You may want to include a filepath before fname in the .SaveAs command.

Hello Sean,

Are you in an Admin account?

Try to run a System File Checker (SFC) scan to check for any system files corruption in the system. SFC scan will scan for corrupt system files on the computer and repair them.

  • Press Windows key + X, click Command Prompt (Admin).
  • In the Command Prompt, type the following command, and then press ENTER:
    sfc /scannow

For more information refer to this link:

Use the System File Checker tool to repair missing or corrupted system files

                                                                            

Also try the following steps to run DISM scan to check and restore system file corruption: 

  • Open command prompt by pressing Windows key + X and selecting “Command prompt admin”.

  • In command prompt, type the following commands and press Enter after each command:

               DISM.exe /Online /Cleanup-image /Scanhealth

               DISM.exe /Online /Cleanup-image /Restorehealth

Hope it helps, let us know how it goes, we'll be glad to assist.
