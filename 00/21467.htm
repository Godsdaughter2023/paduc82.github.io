john w. vinson say:

Two problems that I can see with this: one is the "combinatorial explosion" issue. If you have one letter in a seven-letter word there are seven possibilities; with two there are 42; three 210; four 840. That's not bad for this size problem but as the number of letters in the word and in the search go up, the numbers grow rapidly.

A more subtle problem is duplicate letters. Say the name is "Dennis" - your algorithm cannot search for "two n's", because each letter is searched independently. If you enter N twice in the textboxes it will match either Dennis or Denis (finding the single n twice, once for each criterion).

I think you'll need some fairly nontrivial VBA code to do this right! Good luck!
