swischner say:

Yes, the file storage topic of OneNote has gotten a rather complicated one. Let me try to explain a bit:

There are basically two storage places and methods(!) for OneDrive notebooks:

  • On a local harddrive or LAN fileshare the operating file system is used as you would expect and almost every other application does.  For OneNote every notebook has an own folder containing .ONE files which represent the sections. ONEPKG is nothing but a ZIP file using a different file extension (you could rename and open it with an unzipper) containing that file structure for easy archiving or transferring. Important: ONLY the office versions of OneNote can handle notebooks stored in file system format.
  • Cloud storage, introduced with OneNote 2010 (Office 2010 SP2) and limited to OneDrive (formerly Windows Live, SkyDrive). Notebooks which are stored there don't use a visible and usable file structure. Instead they are "hidden away" (as far as I know in someething which is more of a database structure). What you can actually see on OneDrive is just a link. Putting a filestructure on OneDrive (folders and .ONE files) doesn't help in any way! Actually the sync mechanism is a different one.

So the only way to make a notebook composed of actually files (as contained in a ONEPKG archive) availible on OneDrive and thus to all versions of OneNote (including OneNote 2016 free version) is:

  1. Open the ONEPKG with a full (Office) version of OneNote 2010, 2013 or 2016
  2. Create a new empty notebook on OneDrive (still using the Office OneNote). It may well (but doesn't have to) the same name as the ONEPKG notebook
  3. Copy the notebook you opened from ONEPKG section by section to the new notebook.
  4. Wait for the syncing to complete

Steps 2 and 3 may be replaced by the "change location" oder "share to web" function in the file menu of OneNote which does the creating and uploading for you. Don't know the exact naming of that function in the english version of OneNote but you find it near to the name and settings of the just opened notebook.

user 01 say:

Thanks for the reference to Microsoft Support Article 956196. As described in the article, the only recourse a user has is to begin a multistep troubleshooting process. Many of the steps described in the article should not be required as Microsoft Internet Explorer necessarily has more relevant information to provide than it is showing.

A particular URL is: http://www.notarealdomainname.doh/. This is not a valid name in the domain name system (DNS). If this URL is presented to IE, then the first thing it must do is obtain the IP address for the name. With a system configured to use a Comcast DNS server at 75.75.75.75 then the following response would be generated:

>nslookup www.notarealdomainname.doh 75.75.75.75
Server:  cdns01.comcast.net
Address:  75.75.75.75

*** cdns01.comcast.net can't find www.notarealdomainname.doh: Non-existent domain

Presented with this URL then, IE should report a DNS lookup error reported by the DNS server cdns01.comcast.net at 75.75.75.75 reading "Non-existent domain." This is a much more specific and acitonable error message than "Internet Explorer cannot display the webpage." It is notable that in this case, following the recommendations in Microsoft Support Article 956196 would lead the user to perform all sorts of irrelevent and nonsensical tasks.

hansv mvp say:

You could copy the following code into the ThisWorkbook module of the workbook:

 

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
    On Error GoTo ErrHandler
    Application.EnableEvents = False
    With Sh.Range("O1")
        .Value = Date
        .NumberFormat = "mm/dd/yyyy"
    End With

ExitHandler:
    Application.EnableEvents = True
    Exit Sub

ErrHandler:
    MsgBox Err.Description, vbExclamation
    Resume ExitHandler
End Sub

 

Warning: this code will disable the undo capability.
