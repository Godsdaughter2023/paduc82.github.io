bob jones aka: cybertaz say:

Is the recipient using a Windows PC? If so, is your email program set to send 'Windows-friendly' attachments? It's also possible that the recipient's email system is improperly decoding the attachment.

I don't understand why zipping the file wouldn't be acceptable. The document remains a .docx file type when expanded [unzipped] by the recipient. The .zip extension simply indicates that the file has been compressed, primarily in order to:

  • reduce size for archiving or transporting, or
  • to protect the actual file it contains from being damaged [e.g., mangled by email systems]

In no way does 'zipping' a file change the actual file type, its structure, or its content.

If for whatever irrational reason :-) the recipient insists on a naked, unprotected .docx attachment, make certain that s/he is saving the file to disk, then opening the saved copy... Double-clicking attachment icons in an email message easily can result in this type of irregularity because some aspects of the file cannot be handled by the reader used by the recipient's email software. It isn't "a MAC issue" at all.

If you want to see for yourself, just go into a Finder window, select the file in question, then use the Quick Look feature. You'll see that the columnar layout is not displayed. Some email programs still work in much the same, rudimentary way. Also, attach the same file to an email to someone else or to yourself.

Another possible explanation could be the software/version being used by the recipient.
