If above is correct then the first thing that is required is a formula to create a dynamic range for the defined name "Data" so that if you add or delete rows of data then the RowSource for the ListBox will be adjusted.

Following guidelines to create a formula for the dynamic range for the defined name "Data". Note that $E$5 in 2 places in the formula is the top left cell of the actual data range (excludes the column headers).

  • Select sheet1
  • Select Formulas ribbon
  • Select Name Manager
  • Select the name Data in the dialog. (Because there is only one name it should be selected as the default)
  • Click in the "Refers to" field and press F2 once
  • Select and delete the existing range
  • Copy the following formula and paste into the "Refers to" field.

       =OFFSET(sheet1!$E$5,0,0,COUNTA(sheet1!$E$5:$E$1048576),8)

  • Note: Pressing F2 in the Refers to field places it in edit mode so that the arrow keys will work properly and they will not select cells on the worksheet and attempt to place them in the formula.
  • Close the Name Manager dialog and answer Yes to question re save the changes.

Now copy the following code into the Userform module for the command button. Note the comment about forcing the update of the RowSource.

Private Sub CommandButton1_Click()
       
    With Me.lstboxEntry
        .RowSource = .RowSource      'Forces update if data added or deleted from Data range after Userform is shown
        .Selected(.ListCount - 1) = True        'Selects last item in the ListBox
    End With
   
End Sub

john mcghie say:
Doing a play at school and want to create numbered tickets to sell for admittance. Suggestions?

Easiest way is to place a single paragraph in each ticket, formatted with the List Number style.  List Number style by default places a number at the left of the paragraph.  So set the paragraph indent to place the number where you want it.  It's designed for numbering shopping lists, but it will work for this.

This means you would have to copy the first page of tickets multiple times to produce the 400-odd tickets you will need (think BIG -- show night is a very bad time to have to be printing extra tickets...).

Of course, there are many more sophisticated ways of doing this, using mail-merge and counters and what have you.  But for 400 tickets, set them 10 to a page and copy 40 times: you will be done and printed by the time you figure the other ways out.

Hope this helps
