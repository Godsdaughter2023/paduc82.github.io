mccbleue say:

Microsoft is in a situation now that it has never been in before: with its PC operating systems, it has always been either a Software-only provider relying on other companies, referred to as OEMs or Original Equipment Manufacturers, to make hardware (like in the case of Windows) or it has been a manufacturer of a complete product which no other company can make, like in the case of X-Box. With mobile devices, it is now both a Software company and an OEM.

 

However, Microsoft is no different to any other big corporation - despite the same name and logo being on all products, it is not one big organisation but a number of business units. There has always been a Windows unit, an Office unit, an X-box unit, a Skype unit et cetera - now there is a phones unit too manufacturing the Lumia range.

 

So, there are two units within Microsoft that don't necessarily work to the same goals - the unit that produces Windows Phone wants to see its OS on as many phones as possible which is the point of view that you are describing, that unit's survival depends on the distribution of the OS itself; but the unit that produces the Lumia phones has its own goals and its survival depends on reaching certain sales targets, to which end the Lumia unit as a business entity does not want people to buy WP phones made by Samsung, HTC or anyone else - they want you to buy THEIR phones.

 

There are other similar conflicts, like the fact that the Skype business unit brings innovation in the Skype app and service to iOS and Android before WP because it wants its product being used where the largest user base is, and it remains necessarily commercially oblivious to the fact that bringing innovation to WP first might help its colleagues in the Microsoft Lumia business unit to sell more Lumias.

 

Any "Microsoft" app that is produced by the Lumia business unit, and you can identify those because they all changed their names last year from "Nokia blah blah blah" to "Lumia blah blah blah", those are made and maintained by the business unit which depends on selling Lumia devices and so they are used to "differentiate" the Lumia range from Windows Phone models made by other companies; whereas apps produced by other business units are not part of that requirement and so are available to WP phones in general.

Short form.  You don't have to do anything special to use the navigation pane.  Just use the built in heading styles.  You can customize them to look the way you need.

hansv mvp say:

You could use code like this:

Sub Calc()
  Select Case ActiveSheet.Name
    Case "Sheet5"
      Worksheets("Sheet1").Calculate
      Worksheets("Sheet2").Calculate
      Worksheets("Sheet3").Calculate
      Worksheets("Sheet4").Calculate
      Worksheets("Sheet5").Calculate
    Case Else
      ActiveSheet.Calculate
  End Select
End Sub

jim thomlinson say:

Try this... You will want to swap out the message box at the end with a file open line...

Sub Read_Dir()
Dim ofile As Object
Dim dteMax As Date
Dim oFoundFile As Object
Dim oFold As Object
Dim FSO As Object
Set FSO = CreateObject("scripting.filesystemobject")
'change folder to suit
Set oFold = FSO.GetFolder("H:\")
For Each ofile In oFold.Files
With ofile
    If .Type = "Microsoft Excel Worksheet" Then
        If .datelastmodified > dteMax Then
            Set oFoundFile = ofile
            dteMax = .datelastmodified
        End If
    End If
End With
Next ofile
MsgBox oFoundFile.Path
Set FSO = Nothing
End Sub

julie sheets say:
Jan has given a good answer.  If you want to create the filter try:

1. Project > Filtered For > More filters
2. Select the Date Range filter and copy.
3. Give the filter a new name.
4. Insert two rows above the first row in the filter dialog.
5. First blank row: Field Name = Text1, Test = contains, Value = "Enter area"?
6. next row select AND from the And/or
7. Third and fourth rows are the original Date Range filter.
