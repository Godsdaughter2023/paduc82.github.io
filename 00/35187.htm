davem121 say:
The Product Key should be on a Sticker on your computer, alternatively go here for a small app that will show you your Windows 10 Product Key

andre da costa say:

There really is no need to know the product key, since the machine has a digital entitlement.

Here is how you retreive it:

Windows 10 came preinstalled on my computer, how do I find the product key?

Press Windows key + X

Click Command Prompt (admin)

Enter the following command:

wmic path SoftwareLicensingService get OA3xOriginalProductKey

Hit Enter

The product key will be revealed, copy the product key

When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate.

So, there is no need to know or get a product key, if you need to reinstall Windows 10, you can use your Windows 7 or Windows 8 product key or use the reset function in Windows 10.

If you are prompted to enter a product key, click 'I don't have a key' and 'Do this later'

johns-ste say:

I realize this is an old post.  My reply is for those who get here as I did, through a Google search.  I'm on a Dell Precision M6400 laptop which I updated with the Intel 5300 before updating to Windows 10 Pro x64, from Windows 7 Pro x64.  Connecting to an Asus dual band router, I have no problems with the 5300 on the 5 GHz band.  Range isn't great (as typical for the 5 GHz band, due to the laws of physics and increased interference with this wavelength) but speed and stability are fine.  The 2.4 GHz band is another issue entirely.

As of this date, 6/1/17, Intel no longer supports this device.  Their latest 5300 driver supports Windows 8.0 x64, but not Windows 10. 

{Note: Intel's site indicates some of their devices support Windows 10 by using the Windows 8.1 driver and that windows 10 drivers for some of their devices are only available through Windows Update.}

My initial issue with the 2.4 GHz band (SSID) was the inability to connect to DNS (consequently, no Internet).  The Windows troubleshooter indicated the connection was properly configured but DNS was not responding.  As usual, the Windows messages were overly cryptic for a layman such as myself.  Depending on your perspective, the DNS provider IP is either the Router itself (inside the NAT) or the automatically assigned ISP DNS server, if "Auto" is selected within the router configuration.  As a diagnostic, I also selected an independent DNS server within the router.  Windows diagnostics came back with the same result.  At this point, I WAS able to communicate on the 2.4 GHz band within my LAN, with no discernable issues. 

My next step was to install their latest driver, the Windows 8.0 driver.  The driver installed with no error messages.  I AM now able to connect to the Internet on this SSID ... BUT, the connection is unstable both outside and inside the LAN.  The 5 GHz band seems to function the same as before the driver switch.

Hopefully this will save somebody some time.  IF you can get along without the 2.4 GHz band, the 5 GHz seems fine on windows 10, with either the Windows 7 or 8 driver.

IF you want to update your hardware to a device with Windows 10 driver support, the Intel support site provides this information and also breaks out the devices into active and discontinued (no tech support or new drivers) devices.  I'll drop the link here at the bottom.

I'll probable update to the 6300, a similar device with Windows 10 driver support.  One issue to consider, especially if you're updating an older machine, is the increasing number of different slot types.  Be careful to match your existing connection type.  Unfortunately I have not found where Intel breaks out this information.  Note that the Intel site tends to collapse their lists.  Click "see all" to expose group list.  The link:
