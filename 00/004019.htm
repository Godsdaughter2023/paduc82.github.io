virus and spyware definitions update - Microsoft Community

scaron57 say:

virus and spyware definitions update

Virus and spyware definitions: connection failed and virus and unable to complete spyware definitions update due to Internect or network connectivity issue.  But I am able to get on the internet.  Is my computer at risk and how do I complete the update?
greginmich say:

Many update failures are the result of transient problems on the server side and can be ignored. But if you have a persistent problem with MSE updates, there are some diagnostic procedures listed in the “Help and Support for Microsoft Security Essentials” Web page that might help, including a link to a Microsoft site that can potentially direct you to a solution based on the specific error code returned by the failed update. When an automatic update fails, there won’t be any error message displayed, and the only way to access the error code (and error description) will be to examine the update history in the System Event Log. The update history can be accessed with the following steps:

 

1.) Open Event Viewer by typing “eventvwr” (or just “event”) in the search box and hitting the “Enter” key.

 

2.) Expand “Windows Logs” and click on the “System” folder.

 

3.) Click on the “Filter Current Log…” option in the “Actions” pane or “Action” menu.

 

4.) Select “Microsoft Antimalware” as the Event source and enter “2000, 2001” for the Event IDs, (or just “2001” for failed update events).

 

5.) Click on “Save Filter to Custom View” and name the Custom View “MSE Update History”.

The next time you open Event Viewer, simply expand “Custom Views” and click on “MSE Update History”.

 

 

The error codes for manual-update failures will also be included in the history; but when a manual update fails, we normally get a “Virus and spyware definitions update failed” error message with a “Support information” button that displays the error code and error description, and also a “Help” button that opens the “Help and Support for Microsoft Security Essentials” Web page. If we then click on the “Problems and solutions” link on that page, we find these two pages of diagnostic procedures:

 

http://windows.microsoft.com/en-US/windows/error-0x8-encountered-while-virus-and-spyware-definition-updates-or-product-upgrades

 

http://windows.microsoft.com/en-US/windows/troubleshooting-update-issues

 

The second step on the first page directs us to the Microsoft site where the error code should be entered; but searching this forum or the Internet will often turn up additional information. You can select the error code in the Event Viewer information pane and then use Ctrl+C to copy and paste it into the search box.

 

The second page provides some general procedures that might prove helpful. If you decide to try resetting the MSE antivirus update engine, you’ll need to update the file path, and you’ll also need to type in the command line – if you copy and paste the command line, like any sensible person would do, the command will fail, because the command interpreter won’t accept m-dashes.

 

 

GreginMich

    Any help would be greatly appreciated, since my work depends on the templates I have acquired.

    Joshua

    joshua_030 say:

    For anyone looking for the way to revert to a previous version of any Office (2016) application on Mac OS High Sierra (10.13.2) the following worked for me:

    1. Go into one of the office Apps and choose Help > Check for Updates at the top of the screen.  Turn off automatic updates.
    2. Find the previous version of the particular application you want to revert  (Word, Outlook, Powerpoint, Excel) on the Office Update History for 2016 page:  Update History for Office 2016.  You can download the installer package for the individual app and not the whole Office 365 update by clicking the links on the right hand side (Word, Excel, etc).  
    3. Before opening the update pkg, delete the application (Word, Excel, etc) from your Applications folder in the Finder.  It will go to the trash (in case you need to restore it).
    4. Open the installer package and it will re-install the previous version for you.  You may need to open it and force quit 1-2 times before it will function correctly again for some reason.
    5. Wait until a later date and try updating again when hopefully the bugs have been worked out :), and if you have time, post on here about your problem, so that they can work on a fix to it.

    As Tom mentioned, in prior versions of Access, GetObject can "perform the test." In my Access 2003 days I had to trap for that error in this manner which worked well:

    Hello Beccy,

    It's not possible to unlink your Skype and Microsoft accounts, you can move your Skype account to a different Microsoft account following the steps on below Help article:

    Can I move my Skype account to a different Microsoft account if it's already been linked?

    Your Skype account is the same as your Microsoft account. If you close it, you'll no longer be able to sign in to other Microsoft products or services such as Outlook.com, Office Online, OneDrive, Xbox Live, or Windows.

    Also, if you've linked the wrong Microsoft Office account with the wrong Skype account, you'll need to deactivate it first. You may refer to below Help article:

    How do I link my Microsoft Office account with my Skype account?

    IMPORTANT:

    If your One Drive subscription of 60 minutes per month is the Microsoft account that is accidentally linked to your Skype account, we would suggest you to deactivate it first (one-time only) before you moved your Skype account to a different Microsoft account. In that case, you still have it linked to your real Skype account.
