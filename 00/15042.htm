adlionel say:

OK I've finally found a solution!

It seems that IEHosting is disabled when using document compatibility mode higher than 8 (which works flawessly in IE11 in previous Windows versions, WIndows 10 1511 included)

So you can:

- enforce IE8 document compatibility mode (but for all users) using X-UA-Compatible header or meta tag with "IE=8" content (i was using "IE=Edge" mode)

- or you can deploy EnterpriseMode on the Windows 10 1607 computers and specify IE8Enterprise for your website (see https://technet.microsoft.com/en-us/itpro/internet-explorer/ie11-deploy-guide/turn-on-enterprise-mode-and-use-a-site-list)

  1. Try accessing Outlook.com from different browsers, devices or InPrivate.
  2. Update and optimize the browser.
  3. Disable all add-ins and extensions in the browser.
  4. If rules that stopped working involve a connected account, set up the rule in the original provider.
  5. If rules that stopped working involve the primary account, check if the rule has been set up in a mail client / app connected to the account. If rule was set up in a mail client or app, remove the rule from the mail client or app and try to set it up in Outlook.com.
  6. Verify the existence of conflicting rules that override the rule.
  7. Verify the settings of filters (Exclusive filter and Clutter).
  8. Verify if emails not following the correct behaviour have a particular type of attachment.
  9. Verify if emails involved in the rule are being quarantined by an antivirus (in Windows Defender, this is in the History tab > Quarantined items)
  10. For Windows 10 computers, go to Security and Maintenance control panel > Change Windows SmartScreen Settings > temporarily disable the filters.
