punctuation marks produce special characters - Microsoft Community

llightfoot say:

punctuation marks produce special characters

When we receive Word documents in emails from others for our review or use, the punctuation marks are often replaced by special characters.  Why?  How do we make the proper punctuation appear?
suzanne s. barnhill say:
This substitution usually occurs when a document has originated in
WordPerfect and characters such as curly quotes and dashes were entered in
the WP Typographic Symbols font. On a machine that has this font installed,
the problem will not be evident and so will not have been corrected. Only
when the document lands on a computer without the font installed does it
become obvious. The only long-term solution, unfortunately, is to search for
and replace those symbols. (A short-term solution is to install the font,
but that's not really very helpful.)

  2. They look like hyperlinks, but when merged to PDF, they aren't clickable.

{ IF { MERGEFIELD MANAGING } = 0 "It may be time for " }

macropod say:
Regardless of what you do with field coding, a hyperlink field inside an IF field will only ever be rendered as plain text. You would need to do some post-merge processing to convert them back to working hyperlinks. For a discussion, with code you could use to customize both the display text and screen tip for each hyperlink, see: http://www.msofficeforums.com/mail-merge/7803-convert-html-links-into-embedded-hyperlinks.html
