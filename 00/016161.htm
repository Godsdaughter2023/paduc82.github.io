In IE 11, 32-bit program files are in the x86 directory, and 64-bit program files are in the normal program files directory. Starting with IE 10, on 64-bit system, both run simultaneously: a 64-bit host process and isolated 32-bit tabs with better security, regardless of which version of the program you run directly.

 

note that I have only used a beta version of IE 11, not the final release of it, so I will tell you what I know:

 

If you want 64-bit tabs on IE 10 you can simply enabled enhanced protected mode. on IE 11, you must enable a separate option labeled "enable 64-bit processes for enhanced protected mode". These options can be found under tools>internet options>advanced>security>

 

To our disappointment, task manager sometimes will not tell us if a process is 32-bit. If nessecary, you can instead use Sysinternals' process explorer. Once you save it, you may want to make this modification to it:

 

Right-click the column headers at the top and select "select columns". Check "Image type (64 vs 32-bit)". You can also change this option at view>columns. Now you can keep a look out for whether IE is 32 or 64. I would also like to clear something else up that may be confusing: notice that some "iexplore.exe"s seems to be listed underneath an "iexplore.exe". These are typically known as child processes; processes started by another process. This way, you can differentiate between the tabs and windows. 

 

Please note that process explorer is for advanced users, and I do not recommend using it beyond these directions.

With IE10 and IE11, it's normal to have only one IE shortcut in Start, the 64-bit version.

 

IE> Help> About doesn't provide bitness like it used to.

To see whether 32-bit or 64-bit IE10 has opened, open Task Manager (Ctrl+Alt+Delete> Task Manager) and look at the Processes tab.

 

64-bit IE10 will appear as iexplore.exe.

32-bit IE10 will appear as iexplore.exe *32.

 

You may see both processes  with only 1 window and 1 tab open- the 64-bit Manager Process and a 32-bit tab.

 

Your iexplore.exe files are located at:

C:\Program Files (x86)\Internet Explorer\iexplore.exe . . . (32-bit)
and

C:\Program Files\Internet Explorer\iexplore.exe . . . (64-bit)

 

You can create a shortcut to either of those for the desktop or Start Menu.

  

- - - - -

To get the 64-bit Manager Process to open automatically, go to Internet Options> Advanced and check "Enable Enhanced Protected Mode." I've also seen reports that 64-bit IE10 cannot be accessed/run (using any method) if Enable Enhanced Protected Mode is not checked.

 

Also, in Internet Options> Security, verify that Protected Mode is enabled.

 

Also, from Control Panel> Action Center, Change User Account Control Settings, verify that the slider is not set to "Never notify." If User Account Control is set to "Never notify" Protected Mode and Enhanced Protected Mode can't be enabled.

heidisparkes say:

Creating a hyperlink in MS Word to a document location on a shared server

I'm working on a shared office server using Microsoft Office 2013. jay freedman say:

Yes, you can make hyperlinks to documents on a server. You need to know the server's name (or its IP address) and the name of the share. Then the address format is

   \\servername\sharename\folder1\...folderN\documentname.docx

You can type the address in the Address box at the bottom of the Insert Hyperlink dialog, or use the Browse for File button shown here:

Initially the Text to Display box will show the same thing as the Address, but you can change that to something more user-friendly. You can also click the ScreenTip button and enter text that will appear in the mouse-over popup.

If you want to have Windows Explorer open to the folder instead of opening a particular document, omit the "documentname.docx" part and just end the address at the last slash character.

Everyone who uses the document will need to have permission to access the server; otherwise they'll get an error message when they click a link.

I have a Word document that includes several large tables on separate pages. I have used Section Breaks to insert these tables (landscape as opposed to portrait). The problem is that I want the text to flow on the pages before and after the tables, rather than having blank space after the Section Breaks. I hope this makes sense; it is somewhat difficult to explain. 

daniel g. grau say:

Once you insert a section break, you interrupt the text flow so you'll need to fill the gaps manually. For the landscape page, you may need to place your cursor in the first cell of the table and use Table > Split Table to create an empty paragraph above it.

Note that your headers and footers will maintain their position at the top and bottom of your landscape pages, so that they will appear to be rotated if you print the document and square off the pages.

An alternative is to refrain from using section breaks and rotate the table instead (click on the table, select the purple Table Layout tab in the Ribbon, then use the Rotate button):


In the above screenshot, the text also flows around the table. This is accomplished through Table > Table Properties > Around. You can move the table on the page by grabbing it from the four-arrow icon at the top left.

This method will also keep any headers and footers along the short sides of the page, just like in the rest of the document.
