    Method 1: How to fix broken desktop shortcuts.

    http://support.microsoft.com/kb/978980

     

    Method 2: Rebuild the icon cache

     

    To Rebuild icon cache follow the below mentioned steps -

    1. Open Folder Options to select (dot) Show hidden files and folders.

    2. Open a Windows Explorer window.

    3. Go to C:\Users\(User Name)\AppData\Local
    4. Right click on IconCache.db and click on Delete.


    5. Click on Yes to confirm the deletion.
    NOTE: This deletes the file to the Recycle Bin. It is safe to empty the Recycle Bin when finished.

    6. Close the window.

    7. Empty the Recycle Bin.

    8. Restart the computer.

    9. When you go back, you will notice the Size of the IconCache.db file is smaller, and the Date Modified is now the current date. (See screenshot below)
    NOTE: If for some reason IconCache.db is not there or the size has not changed much, then just restart the computer again. You may need to restart a couple of times in some cases.

    10. The icon cache has been rebuilt.

    As for project files, remember that project files contain no actual video or audio.  A project file is just the editing state of your project.  It contains the *name* of each media file and where you want to make cuts, what transitions you want, etc.  So if you are trying to open a project file on another computer, even if you can do that, it won't show a preview unless you have all media files in the same folders as the original computer.

    That aside, assuming you do have both the project files and the media files, you need to have a compatible version of Movie Maker to open them.  If you have a project created in WMM 2.6, it can only be opened in WMM 2.6 or WMM 6.0.  Microsoft chose to only support the prior version being opened in the next version.  So one workaround would be to open the project in WMM 6.0, save that as a new project file (.mswmm) and then open that converted project file in the newest versions.
