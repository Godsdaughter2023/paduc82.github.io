e.j. gun say:

It's best to refer to the tables by name, and then resize the range of each table, as in the sample code line below.  You can associate this line of code to a button for each table.  The bold highlighted part is what increases the number of rows in the table.  You can do the same with columns if you desire.

Be aware that this code will error out if you try to add a row to "Table1", and that row will go into another table below it (i.e. "Table2").  In order to make that work, you would also have to insert a new row into the worksheet between the two tables, and then add a row to "Table1".  Same deal with columns.  If your tables all start on Row 1 and are separated by columns, inserting rows this way won't be a problem.  If your tables are "stacked" one above the other, you will have this problem.

Eric
