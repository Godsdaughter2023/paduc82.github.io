martinheidelberg say: I wrote a macro to solve my problem. I tried testing it thoroughly, but please beware: It is the first Macro I ever wrote.


Sub LocalizeNames()

' This macro makes the scope of all names in the currently active worksheet local to this worksheet, if:
' the current scope of the name is global AND the name refers to the currently active worksheet.


' Some variable declarations
Dim i As Integer
Dim n As Name
Dim bangposition As Integer
Dim oldrefer As String
Dim newname As String

' Set up i as a counter
i = 0

' Loop through all Names in the Workbook (i.e., all worksheets! I fear there is no other way)
For Each n In ActiveWorkbook.Names
  
    ' Is n a global name? Yes, if the "Name of the name" does not contain a "!"
    If InStr(n.Name, "!") = 0 Then
       
        ' Only localize the name if RefersTo points to the acurrently active worksheet
        bangposition = InStr(n.RefersTo, "!")
        ' MsgBox (ActiveSheet.Name & "    " & Mid(n.RefersTo, 2, bangposition - 2))
        If UCase(ActiveSheet.Name) = UCase(Mid(n.RefersTo, 2, bangposition - 2)) Then
            oldrefer = n.RefersTo
            newname = ActiveSheet.Name & "!" & n.Name
            Call n.Delete
            ActiveWorkbook.Names.Add Name:=newname, RefersTo:=oldrefer
            i = i + 1
        End If
    End If

Next n

MsgBox ("Made a total of " & i & " names local to sheet " & ActiveSheet.Name)

End Sub

amahones1 say:

Extract alphanumeric text from a cell when it's located in different places within the cell

Hello,

I have a workbook with two sheets.  The first sheet has a column for Job # and the rest of the columns are blank and need to be populated based on the Job #.  The second sheet has a column where the job number appears in all the cells, but in different locations.  The data I need on the first sheet comes from the second sheet, so I wanted to do a vlookup, where the lookup value was job # from the first sheet, and the lookup array was the data on the second sheet.  

My problems:

  • Job # is not consistent - sometimes it's 100% numeric (1805), sometimes it's numeric with a dash in the middle (1805-17), sometimes it begins with an "M", sometimes an "F" and sometimes "WO"
  • Within the cell itself (on Sheet 2), it is in different locations (e.g. TEXT TEXT 1805 TEXT, M1234 TEXT TEXT, TEXT TEXT K1234)

LEFT() and (RIGHT() sometimes work and sometimes combinations of different data cleansing formulas work, but is there something that could work for all these scenarios?  Even if it just works for a big chunk of them, and the rest have to be manually done, it would be a huge help.

hansv mvp say:

You could use a formula such as

=IFERROR(VLOOKUP("*"&A2&"*",Sheet2!$A$2:$K$200,2,FALSE),"")

where A2 is a cell on the first sheet with a job number, and Sheet2!$A$2:$K$200 is the data range on the second sheet. This formula will return a value from the second column; you can change the column number 2 of course.

The formula can be filled down.

Note: if you have both 1234 and K1234 in the text on the second sheet, a job number 1234 will match either, so the formula will return data from the first matching row, which might not be what you want.

=IFERROR(VLOOKUP("* "&A2&" *",Sheet2!$A$2:$K$200,2,FALSE),"")

might work better, but it expects that there will always be a space before AND after the job number in the text on the second sheet.
