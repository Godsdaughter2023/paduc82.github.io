john w. vinson say:

Not sure why you need InStrRev  rather than InStr unless you might have records with more than one hyphen. The simplest solution would be just

Trim(Mid([Fieldname]), InStr([fieldname], "-" + 1))

The Trim will remove the blank after the hyphen, or you could use + 2 instead of +1 if the hyphen is ALWAYS consistently followed by exactly one blank. 

Note that the Mid() function takes three arguments (the string from which you're extracting, the start position, and the end position); but if you leave off the optional third argument it will give you the rest of the string, however long that might be.

The expression should work using InStrRev instead of InStr if you prefer.

rich michaels - mvp (greatcirclelearning.com) say:

The issue is that often the USB will not respond fast enough and the Office applications interpret this as a errors such as you are seeing.

If you are using the fastest USB technology, which is USB 3.0, and you are using a proper cable capable of delivering the highest speeds, and you are using a single USB connection and not one that is shared on a splitter with other USB devices... then the Office applications will probably work with it okay.

However, as Phillip and John have already told you, editing directly from flash sticks is not recommended. Word in particular uses about 15 hidden temp files to do its work and some of these files are written to the same location as the source document which of course escalates the problem. If your preference setting are also marked to "Always create a backup" and/or you have your AutoRecovery save time set to something less than 10 minutes... this will also intensify the problem as demand for accessing the temporary work files increase.
