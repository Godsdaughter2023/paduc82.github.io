This is what is in the System Reserved partition..

(extracted from this website.. http://helpdeskgeek.com/help-desk/hdg-explains-what-is-the-system-reserved-partition/)

- Boot

- System Volume Information

- bootmgr

- BOOTSECT.bak

- $RECYCLE.BIN

So what is all this for exactly? The partition has two main functions. First, it stores the Boot Manager code and the Boot Configuration Database. The second thing it does is store the startup files needed by BitLocker Drive Encryption. If you end up using BitLocker to encrypt your entire hard drive, you don’t have to repartition your system to enable it. If you don’t plan on ever using BitLocker, then that means you’ll never need this special partition.

In day to day operation, fragmentation of this partition has no bearing on performance at all.

In respect of the problem when backing up, the best solution is to increase the size of the partition. Microsoft was being a little stingy setting the size at 100mb. Increasing it to 200 or 300mb sees any backup issues fade into obscurity..
