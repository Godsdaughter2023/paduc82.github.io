jay freedman say:

The "Filling in forms" protection is not intended to be used with content controls; it's for legacy form fields.

There are two mutually exclusive ways to lock the text outside of the content controls to prevent editing/formatting:

  • Select all the content controls and all surrounding text that should not be editable, click the Group button on the Developer tab, and click Group in the menu.
  • OR: In the Restrict Editing pane, select "No changes (Read only)". Then select each content control and check the "Everyone" box under Exceptions. Don't mark any of the text that should not be editable.

With either of these methods, you don't need a macro to run the spell checker; only the "Filling in forms" protection requires that. If you have no other need for macros, you can set the macro permissions any way you prefer.

leosabatini say: Here is how you can link to SharePoint lists from VBA code:

1. create tables in Access and then export them to your SharePoint site
2. open the Access application to which you want to link the tables created in step 1
3. manually link to the tables created in step 1
4. once the tables are linked, open each one up in design view
5. on the Description line is the information, created by your SharePoint site, that you will need for the vba code link.  For each table, you will need to copy the text from this line.
6. Here is what the VBA code will look like to link each table (you can place this in the open event of your opening form or call it via a command button:
       
DoCmd.TransferDatabase acLink, <paste the text from step 5>

7. create a line like the one above for each table to which you want to link.

8. You can now break all of the links you manually created in step step 3 by simply deleting them

Now, when you run the VBA code, you should get a login prompt from the SharePoint site and the tables (lists) should link.

bob jones aka: cybertaz say:

Regrettably, there appears to have been a misunderstanding somewhere along the line. What I see in the photo appears to be either a Combo Box or Drop Down ListContent Control. As with most Content Controls, neither of those are functional in any version of Mac Word.

"Office 365" makes no difference because the web apps cannot open Protected forms -- they must be opened by the locally installed program, which on a Mac would be Word 2011. In Mac Word the form can be read, but the user cannot open the list. If you have access to a Windows installation of Office 365, the locally installed software would be Office 2013, which would allow full editing of the form.

Assuming you're restricted to access on a Mac, the only suggestion I can offer is to obtain an open source suite such as LibreOffice.
