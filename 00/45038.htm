brian tillman [outlook mvp 2007-2019] say:
This says you're not authenticating properly to the outgoing server.  Apparently, embarqmail requires you to authenticate in order to use their mail server.  On the "Outgoing Server" tab of your mail account properties, check the box that indicates you need to log into the server.  It may also be true that embarqmail requires the use of a port other than 25, like 465 or 587.  I've seen it mentioned in one reference that port 25 with authentication but no encryption should be used and in another that port 587 with authentication and SSL should be used.  Read the documentation that your service provider gave you or visit their Help pages.

gts-nj say:
 Windows 7 can burn an ISO image to a CD or DVD but cannot mount an ISO as a virtual drive.  That requires 3rd party software.  How autorun is handled in that case would depend on the program being used and its settings and options.
 
As far as AutoRun of a inserted CD or DVD, how that is handled depends on the Autoplay settings which you can manage in Control Panel.  In the example you mention, if Autorun for software and games is set to "Ask me every time" you will be prompted.  If you say no to the prompt, the autorun.inf will not be executed.  If "Install or run program from your media" was the setting, it would run without prompting.  That can be a dangerous setting.
