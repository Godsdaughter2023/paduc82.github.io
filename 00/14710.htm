rick song msft say:

Hello Rob,

Based on my experience, this is a built-in retention tag, and it move items that have in the Recoverable Items folder for 14 days to the Recoverable Items folder in the archive mailbox. Here is an article for your reference: https://docs.microsoft.com/en-us/office365/securitycompliance/set-up-an-archive-and-deletion-policy-for-mailboxes

If the archive mailbox isn’t enabled, no action is taken when then mailbox is on hold. It is suggested to turn on the archive for the mailbox, after it succeeds, you can try to move a test item in the Recoverable Items folder. After the 14-day retention period expires for this item, it will be moved to the Recoverable Items folder in the Archive mailbox. Here is an other article for your reference: https://docs.microsoft.com/en-us/office365/securitycompliance/increase-the-recoverable-quota-for-mailboxes-on-hold?redirectSourcePath=%252fen-us%252farticle%252fIncrease-the-Recoverable-Items-quota-for-mailboxes-on-hold-a8bdcbdd-9298-462f-b889-df26037a990c

roland5cox.net say:

Open Access Database Exclusively

I created a MS Access 2010 database application that I want no more than one user at a time to open.  Can I set the application to open exclusively every time regardless of the manner in which users try to open that application?  Are there settings in database options I can use or some code I can write?

mimmo57 (d. laurenza) say:

Hi,

you can create a new module called ShowUsers with Sub ShowUserRosterMultipleUsers to call when starting your DB:

Option Compare Database
Option Explicit
Sub ShowUserRosterMultipleUsers()
    Dim cn As New ADODB.Connection
    Dim rs As New ADODB.Recordset
    Dim wUsers As Integer
    Dim wUser As String
   
    Set cn = CurrentProject.Connection
    Set rs = cn.OpenSchema(adSchemaProviderSpecific, _
    , "{947bb102-5d43-11d1-bdbf-00c04fb92675}")
   
    wUsers = 0
    wUser = rs(0)
    Do While Not rs.EOF
       wUsers = wUsers + 1
       rs.MoveNext
    Loop
   
    If wUsers > 1 Then
       MsgBox "The application is in use......by (" & wUser & ")", vbInformation + vbOKOnly
       DoCmd.Quit
    End If
   
    Set rs = Nothing
    Set cn = Nothing
End Sub
