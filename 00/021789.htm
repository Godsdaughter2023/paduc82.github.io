Since the upgrade from office 2010 on win 8 to office 365 on win 10, I can't sync my contacts to my iphone 5c through itunes and usb cable, as the "syc contacts with" in itues is now missing outlook from the list, it only shows windows contacts.  I dont know what windows contacts are, but I dont believe I have any, which would explain why my phone contacts have now been wiped out.

I have spent so many years trying to find a way to reliably sync contacts with android and now Iphone, I am even considering going back to paper based, as it does not seem to be feasible.  I have spent a small fortune on 3rd party sync SW, but dont want the pain of this any more.

If I login to outlook.com on the web, I see some contacts, and I have no idea where these came from.  I used an old hotmail account as my microsoft ID when I was forced to enter one for the 365 upgrade, but this is a pure dummy account, I dont use it for anything.

I have outlook, and in outlook I have the following accounts setup:

  1. gmail account via IMAP (primary)
  2. work exchange account (work calendar and work emails)
  3. various google apps accounts.
  4. my "windows" hotmail account via IMAP (the outlook connector has never worked - as soon as you install it, indexing dies, with no solution to fix).  Hotmail via IMAP is too unreliable to use for a primary mail account, it is VERY slow, and gives constant errors and timeouts.
  5. local data file, where all my contacts live.

If I go into contacts in outlook, I see:

  1. contacts in local data file (my contacts)
  2. Exchange contacts (work)
  3. this computer only (empty)
  4. icloud (this doesnt work, and never has, it throws an error when it tries to sync, it is confused about my outlook setup)

Does anyone have any suggestions how to:

1) sync contacts with iphone via cable (neither icloud nor itunes work)

2) sync my outlooks contacts with the outlook web version.

Note: I have the 64 bit office and windows 10.

=== UPDATE ===

I came up with a solution, which is not documented anywhere I can see:

  1. I removed my hotmail.com IMAP account from outlook, and added it back in using the hotmail connector now included as part of outlook 2016.  This time it did not stop all indexing, as all previous attempts.  So far so good.
  2. moved all my contacts from outlook data file into hotmail contacts.  I backed up the entire drive first of course.
  3. on the iphone, do not install outlook or office, this does not integrate with the iphone contact list (or it didnt for me anyway). Instead setup your hotmail account in the apple mail application.
  4. now it syncs contacts both ways over wifi

The solution is not perfect, e.g. if you rename a contact, you often end up with both the old and the new (as it syncs on name).  The other problem is that when you create contacts in outlook, it always puts them in the wrong place, even if you set hotmail to be the default.  Every week or so, I troll through my 1000+ contacts looking for ones which are not registered against hotmail, and drag them across so they can sync.  If you edit an contact on both pc and phone, it gets very confused, and I have had the situation where a contact refuses to sync from phone to pc, even though there is no similar contact on the PC.  Renaming their last name  to "lastname 2" fixed the issue

So the advantage of this new method is you dont need a cable.

The disadvantages are contacts which wont sync, problems when you change someones first or last name entry, and problems with outlook always putting new contacts in the wrong place, then its hard to find why they are not syncing to the phone.  I would say about 90% of the time, my contacts sync correctly.  The 10% need manual work.

brian tillman [outlook mvp 2007-2019] say:

> MS renamed the hotmail connector "Exchange Active Sync" or EAS, which is confusing.

No, they didn't. The Connector used a protocol named DeltaSync; the same protocol used by Windows Live Mail with the old Hotmail service. The Outlook.com service that replaced Hotmail uses Exchange ActiveSync, a completely different protocol. The Outlook.com service replacing the current Outlook.com (hosted on the Office 365 backend) uses full Exchange. Since Outlook.com is free, so is your Exchange connection.
