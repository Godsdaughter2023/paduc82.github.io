valuation vision say:

Office365 will not activate in an OSX Server (open directory) environment

Office365 for OSX will not activate if your user's login keychain is located anywhere else except :

/Users/<username>/Library/Keychains

For instance in an OSX Server environment with AFP Home directories a user's login keychain is located at

/Network/Servers/appleserver.notlocal/..../<username>/Library/Keychains

Even attempting to fool the activation program by symlinking the directory doesn't work.

I believe the Microsoft is using a different method to access the Keychain other than the actual Apple OSX Keychain APIs! as no other apps have problems accessing the login keychain.

I was able to work around the problem by turning on Home Sync for that workstation/user. However that increases our risk as the user's home directory is now duplicated on the workstation and is a possible avenue for a data breach.
