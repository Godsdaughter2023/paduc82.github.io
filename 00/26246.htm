albert d. kallal say:

This can be difficult answer, because are so many choices and so many different scenarios here. 

 

There is a rather large slection of mailing list companies and systems that you can subscripbe on the web. These systems allow you to maintain and run the mailing list through their web site.

 

And most of these systems allow people to sign up, to remove themselves (very important). And users can even change their email etc.  And because the systems are specifically set up for doing mailings, and they have all kinds of additional features built in that track how many users actually opened and received the e-mail (often they determine this by sending a tiny imbedded graphic, and when you receive the e-mail, if you open it then tiny grapic is download and tells their server that you actually in fact looked at the e-mail "and" downloaded pictures).

 

 So the systems will not only tally the e-mail to you sent, but can often give you statistics about how many people actually opened and read the e-mail. They also remove bounced emails etc.

At the end of the day you need a pretty robust management system and usually don't want to have some fulltime person in your office to simply deal with removing or adding e-mail names to some list.

 

Also, perhaps you don't really need a direct connection from access to this particular web system or site anyway.  From my experience, the vast majority of these mailing systems sites simply allow you to download the latest or most current list of people in the e-mail system.

 

Considering that the systems are so very cheap to use on a monthly basis, you're probably better off to use one of those systems than attempting to build your own system. On the other hand, I suppose this comes down to if your company already has a few websites and perhaps have been working with some web developers of late. Your web people would be good candidates to ask about these systems. 

 

And in fact, check with your web hosting company you have now. Even the very low-cost ones as part of the package include a number of free Software Systems such as blogging software, database and even some type of email marketing system. Some of these systems would allow MS Access to connect to them, but the ones that don't in the vast majority of cases will allow a download of files that can be imported into Access anyway. 

 

I'm not really sure a lot more advice can (or should) be explained in a forum post, but I would check your current Internet provider if you current have any web site hosting that you are using at this point in time.
