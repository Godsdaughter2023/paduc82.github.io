Backup trying to include non system/active disk in system image - Microsoft Community

demianph say:

Backup trying to include non system/active disk in system image

Computer with Windows 7 Pro.

The computer has a system disk of 230GB and a data disk of 1.5TB.

The OS resides on the 230GB drive and in Disk Management it is the drive that contains the 100MB hidden
partition for windows and the c:\ partition where windows and documents are installed.

The 1.5TB drive contains multimedia, installed games, and other non system files. In Disk Management it has no special attributes listed.

Suddenly the scheduled Windows backup (to a network location big enough to handle several copies of the system drive but not the data drive) fails because it is insisting on adding the 1.5TB drive to the system image part of the backup.

How do I get it to go back to the original way it was working for the last three months?

Just found that another computer set up at the same time has this issue as well, this other computer has 2 1.5 TB drives and only one of them is being targeted.

Both data drives being targeted do have Valve's steam application installed on them.
demianph say: I have fixed the problem.

Dragon Age origins installs an updater service (as in windows service) on the drive that contains the game. This causes Backup to see a service ont he large drive and insist on including it in the backup.

Maybe MS should make backup only care about Microsoft services and not 3rd party ones.

I fixed the problem by making a c:\program files\dragon age updater service directory and then copying the contents of the Dragon Age Origins bin_ship directory into the directory. If you dont do this it will not allow you access to your DLC.
After that I updated the following registry entry to point to the updater.

HKEY_LOCAL_MACHINE\\SYSTEM\\CurrentControlSet\\services\\DAUpdaterSvc

The string you want to edit is under ImagePath.

This instantly solved the issue without needing even a reboot.

Too bad I had to use that restore point. PITA to go and re-install nearly a hundred games again (steam sale plus all my old games that needed to be re-installed due to upgrade to Windows 7).

I think I found the culprit on the other machine as well. It has a service set to run at startup from iwin games called iwintrusted.exe that runs all the time it is installed on a non system drive that is being included against our will.
