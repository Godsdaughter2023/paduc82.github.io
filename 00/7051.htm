josievargas say:

Word Macro to conver date to MM/dd/yy

I would like to convert a written date to MM/dd/yy.  I name my word document based on the date of the case, so I would like to change the date of June 26, 2016 to 06-26-16 but only when it is written in the format of CSD-June 26, 2016.  This way it won't change the date of my letter that I have at the top of the letter as well.  In other words, it needs to change CSD-June 26, 2016 to 06-26-16.  I would no longer need the CSD in front of it once it's converted.  Thank you for your help.
jay freedman say:

Try this on a copy of your document:

Sub ConvertCSDDate()
   Dim strDate As String
   Dim rg As Range

   Set rg = ActiveDocument.Range
   With rg.Find
      .Text = "CSD-[ADFJMNOS][a-z]@ [0-9]{1,2}, [0-9]{4}"
      .MatchWildcards = True
      .Wrap = wdFindStop

      While .Execute
         strDate = rg.Text
         strDate = Format(Replace(strDate, "CSD-", ""), "MM-dd-yy")

         If IsDate(strDate) Then
            rg.Text = strDate
         Else
            rg.HighlightColorIndex = wdYellow
         End If
         rg.Collapse wdCollapseEnd
      Wend
   End With
End Sub

If the text in the original document is not a valid date (for example, "CSD-May 32, 2016"), the macro will apply a yellow highlight instead of changing its format.

a. user say:

Mailmerge fields only ever inserts the "plain text" - none of the formatting (i.e. indents, bold, etc.) comes across.

What you can sometimes do when the data source is a Word document and the formatting has been applied directly, is use a REF field instead of a MERGEFIELD field to insert the text. However, as far as I know this is not "supported".

e.g. if you use alt-F9 to reveal your field codes, and your Motion field is called "Motion" then you will see something like

{ MERGEFIELD Motion }

or

{ MERGEFIELD Motion \* Mergeformat }

Delete the MERGEFIELD and \* Mergeformat so you have

{ Motion }

Then try your merge again.

This will not work with formatting that you have applied using, e.g. paragraph styles, and some types of formatting will probably not come through. In other words, you would have to work in a certain way to make use of this approach, live with its limitations, and accept the possibility that at some point Microsoft might change the behaviour of Word so that it no longer worked.

It relies on the fact that it is still possible to use the merge mechanism in the very first version of Windows Word, which had no MERGEFIELD fields and used "REF" fields such as { Motion } instead.

OK, I think I have it figured out. You MUST use the Wizard, and there are a couple of tricks.

  1. Open a blank document, then select  Mailings > Start Mail Merge > Step-by-Step Mail Merge Wizard...
  2. You're now on Step 1 of 6 in the Wizard. Select the document type. I selected "Labels." Then click "Next: Starting document" at the bottom of the window.
  3. In Step 2 of 6, you get to determine the layout. For my labels, I had "label options..." as a choice. I clicked it and selected "8163 Shipping Labels" for my use. After that, click "Next: Select Recipients"
  4. Before I did anything in this new window, I laid out my return address how I wanted it. I added my logo, sized it down, made the text wrap around it, and then I added my business name & address. I also changed the spacing so it wasn't adding a space before each return (indicating a new paragraph in Word).  After my return address looked right, I hit return a couple of more lines, then hit the "increase indent" tool, so that when I build the address block, it'll be more centered on the label instead of aligned left.
  5. After my return address was built, then and only then did I start the recipient selection in the mail merge wizard (step 3 of 6). I clicked "use an existing list" browsed my way to a spreadsheet I had already. Then I filtered my results as I wanted. Alternately, you can use the check box to select recipients.
  6. Once that's done, click "Next: Arrange your labels"
  7. I selected "address block." but for some reason it didn't pull the state into the address, so instead I had to use "more items" and build my own address block one by one. After you have all the items in place with the appropriate spacing, commas between the city & state, etc., you can highlight the entire block you built and change the font, size, etc. to your liking.
  8. Then click the box reading "Update all labels" This seems to be the piece missing if you don't use the wizard and try to do it on your own, which led to me pulling my hair out. Your welcome. ;-)
  9. Once that's done, click "Next: Preview your labels"
  10. Now, cross your fingers, legs, eyes, and anything else you have and click "Next: Complete the merge"
  11. Ta, da!
  12. Offer your profuse thanks because I hammered through it and saved you the agony.
