Hi Kevin,

I think you have been given the solution to your problem already in this

thread. However, it is covered up in so much minutia, that it is easy to 

miss.

You are right that "DisableAntiSpyware" gets a lot of press in regard to

this problem. However, there is another policy that is sometime used

to disable Windows Defender as well.

You should enter the Registry and navigate through this branch.

Hkey_Local_Machine\Software\POLICIES\Microsoft and select the folder

Windows Defender.

This folder should contain only one Key.   "Default".

Any other Key that you see here is a policy change that has been made

to Windows Defender, by Malware, another Antivirus, or by you.

If you see "DisableAntiVirus" you should disable it by changing its

DWord from 1 to 0. Then click "OK".   Like wise you should do the same

for "DisableAntiSpyware" if it is present. It shouldn't be, you have

previously "Reg Deleted" it. If there are other Policies in the folder, that

you didn't set, you may need to delete the complete Windows Defender

folder... (KEY).   As a precaution, you should "export" it first.  (to  Desktop).

Not to worry, Windows will reconstruct the key.

Best to back out of the Registry by retracing the steps you used to enter.

This keeps the Registry from becoming cluttered. 

Restart your PC and check Windows Defender. If it is not enabled, enter

the following command string in Command Prompt (admin). Enter the

commands one at a time, and hit <enter> after each.

CD\     <enter>

CD Program Files\Windows Defender     <enter>

MpCmdRun -wdenable     <enter>

Exit     <enter>

Restart your PC and check Windows Defender again.

Good luck,  Glen

  1. Close Word.
  2. In the search box on the Windows Start menu, type Programs and Features. When that item appears at the top of the Start menu, click it.
  3. In the Programs and Features dialog, locate "Microsoft Office 2007" in the list and click on it. (It may mention the edition, such as "Standard" or "Professional".)

jay freedman say:
The best practice is to define the tab stops as part of the style you apply to the text that needs them. They'll be present in every paragraph that has that style.

If you insist on using directly applied tab stops, you should get the same tab stops in a new paragraph when you press Enter at the end of the current paragraph that has those stops. That behavior (carry the current paragraph's formatting of all types to the next paragraph) is automatic in Word; in fact, there's no way to turn it off.

But if you already have more than one paragraph in the document, and you apply tabs or other formatting to one paragraph and then just move the cursor to another paragraph with the arrow keys or the mouse, then the formatting isn't carried along -- only the Enter key will do that. (Or you can use the Format Painter button, or the shortcuts Ctrl+Shift+C and Ctrl+Shift+V to copy and paste formatting from one place to another.)

jay freedman say:
One other thing: If you clear all tabs and then want to set tabs for the whole document, you must select the whole document before setting the tabs. That is, newly defined tabs are applied to whatever set of paragraphs is selected at the time, whether one or many.

But using styles is much more efficient!

greginmich say:

A custom scan with MSE will often help reduce the scan time; but something that can potentially be even more effective is disabling Archive Scanning (in the Advanced Settings page) – and this can only be done for a Full Scan. So what I used to do was disable Archive Scanning and then exclude the drives and folders that didn’t need to be scanned:

https://answers.microsoft.com/en-us/protect/forum/mse-protect_scanning-windows_vista/security-essentials-scan-time-lengthy/373e7298-c2ad-4cd4-bf90-6807d345a813

The setting that allows us to disable Archive Scanning was actually included specifically for the purpose of reducing the time for Full Scans, and it basically provides a blanket exclusion for archive files. Archive files don't really pose a threat, but scanning them can consume a huge amount of time and resources because these files can be very large, and also because each container needs to be unpacked in order to scan the contents.
