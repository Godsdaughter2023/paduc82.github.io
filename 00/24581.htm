<rohn007>Excellent article about Win 10 Backup.  It refers to many Win 8 articles, so it is a good reference for Win 8 backup also.

 

Quote from one of the articles, “… telemetry shows that less than 5% of consumer PCs use Windows Backup … adding up all the third party tools … nowhere near half of consumer PCs are backed up”.  Pair that with the additional article linked below and any intelligent computer user is going to want to do a better job of  doing backups. 

 

Using all of the features described in this article gives you a robust backup system. But the absolute minimum you should use is File History. Fortunately File History was designed from the ground up to be user friendly, set and forget. It is designed to specifically focus on user files, ignoring Windows system and application files.

 

Focus on the section about “File History”. EVERYONE should turn it on, tune it and take advantage of it. To take best advantage of File History you should point it to a drive other than the one you have Windows installed on. You can do that several ways (best to worst):

  • Network drive: home network (HomeGroup) or corporate
  • Install a second drive in your computer, dedicated to backups
  • External USB (spinning disk rather than flash drive) hard drive
  • USB Flash Drive
  • Partition the existing drive. Create a significant backup partition
  • Cloud, ie OneDrive

 

If you have a home with multiple computers, and some sort of network connecting them, you could easily define one drive to be the “Backup Drive”.

 

Many desktop computers have “empty bays” that give you room to install a second drive.  Few portable computers have room for extra drives.

 

External USB connected spinning disk drives are now relatively cheap in sizes from 500GB up to 4TB. Cheap enough that it would be reasonable to dedicate one to backup. Pay a little extra for one that supports USB 3, even if your current computer does not support it.  External drives are more appropriate for desktops rather than portable computers since they are relatively large, about the size of a paperback book.

 

USB Flash / “Thumb” drives are both physically smaller and less expensive than External USB drives.  Their small physical size make them more appropriate for portable computers. While their smaller storage space would be less appropriate for desktop computers.  For the price of a 256GB flash drive you can get a much larger external USB spinning disk drive.

 

Splitting a single hard drive into smaller logical drives is known as “Partitioning”.  This has the advantage of separating your backup data from the Windows operating system, and by default your data files. It has the multiprong disadvantage of being located on the same drive.  If the drive fails, you have lost your backup. Sharing the same drive has the potential to create a read/write bottleneck, slowing performance for the user or the backup. 

 

File history cannot be setup to save directly to cloud storage. But you can use the OneDrive “Sync” feature to complement backup done by File History. Actually, the combination of File History and OneDrive Sync meet the requirements of a good backup system as defined by Steve Gibson (a computer storage pioneer) who said

 

Remember 3-2-1 to have good backups:

o at least 3 copies of any file (ie original, File History and Cloud Sync)

o at least 2 different storage mediums (say on your hard drive plus at least one of the following: thumb drive, DVD or the cloud),

o at least 1 copy offsite - in case of fire etc! (ie the “cloud” meets this requirement, so does using a thumb drive moved to a different building, or a server in a different building)

 

To take advantage of this for your specific request you would have to create a script / batch file that would periodically copy your normal.dotm to the onedrive sync location.  Or simply add the templates folder to the onedrive sync.   In the past, I created a batch file that would copy the normal.dotm FROM a "backup" location and only then start word, and finally when word shut down, copy normal.dotm to the backup location. Back then the corporate network login would copy down a "standard" normal.dotm, wiping out all of my customizations.  I applied my customizations to the standard file, then started using the "backup" process.  That way I complied to corporate standards, but still had my extensive customization too.

rory archibald say:

You need to declare your own Namespace and then use that in the idQ attribute for anything you want to share. So your first add-in's CustomUI would be:

<customUI xmlns="http://schemas.microsoft.com/office/2009/07/customui" xmlns:x="MassimoNamespace">
<ribbon>
<tabs>
<tab idQ="x:MassimoTab" label="Massimo">
<group id="CK33group" label="CK33">
<button id="PulsanteCK33" label="CK33 v.1.0" size="large" onAction="CK33_grafico.CK33_grafico" imageMso="Chart3DColumnChart" />
</group>
<group idMso="GroupEnterDataAlignment" />
<group idMso="GroupEnterDataNumber" />
<group idMso="GroupQuickFormatting" />
</tab>
</tabs>
</ribbon>
</customUI>

Note the new x namespace declaration in the CustomUI section, and its use with the idQ attribute for the tab you want to share. You then do the same in the second add-in:

<customUI xmlns="http://schemas.microsoft.com/office/2009/07/customui" xmlns:x="MassimoNamespace">
<ribbon>
<tabs>
<tab idQ="x:MassimoTab" label="Massimo">
<group id="TGSL0group" label="TGSL0">
<button id="PulsanteTGSL0" label="TGSL0 v.0.1" size="large" onAction="TGSL0.TGSL0" imageMso="CacheListData" />
</group>
<group idMso="GroupEnterDataAlignment" />
<group idMso="GroupEnterDataNumber" />
<group idMso="GroupQuickFormatting" />
</tab>
</tabs>
</ribbon>
</customUI>

and you should be all set.
