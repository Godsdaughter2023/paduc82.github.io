Outlook subject decoding problem when subject contains multiple encodings

When receiving an email with a subject line containing two multiple encoded sequences with different charsets (like "utf-8" and "iso-8859-1"), Outlook ignores the second charset and attempts to decoded it as if were the first.

For example, the encoded sequences in following subject will both be decoded in "utf-8":

Subject: RE: =?utf-8?B?SXN0byDDqSB1bWEgcmVzcG9zdGEgYXV0b23DoXRpY2Eu?= #00000003 -

=?iso-8859-1?Q?Ac=E7=E3o:_Isto_=E9_um_teste.?=

However, only the first one should be decoded as "utf-8". The second one should be decoded as "iso-8859-1".

This problem happens regardless of the first charset type. That is, if the first charset is, for instance, "iso-8859-1", all the subsequent are wrongly decoded as "iso-8859-1" and garbled characters are shown.

Outlook should always check the charset for each encoded sequence, instead of blindly using the first one it finds. This happens with Outlook 2010, but also happened in previous versions and also with Outlook Express.

On the other hand, Microsoft Hotmal correctly decodes subjects like the example above.

Is there a solution for this problem?

Thanks in advance,

Paulo
