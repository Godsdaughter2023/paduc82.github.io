 I believe your particular issue is different as I believe you are getting >0% on actual installation.

 In regard of kernel debugging, I was using this guide: 

https://docs.microsoft.com/en-us/windows-hardware/drivers/debugger/setting-up-a-network-debugging-connection

 For WDK setup instruction is here:

https://docs.microsoft.com/en-us/windows-hardware/drivers/download-the-wdk

Please note that debugging USB installation seems to be safer easier because all BCD modifications are done there. This means any change can be undone by booting to working Windows installation. While playing with your main BCD please consider having backup & story how to use it. In my case I have dual boot to Linux. 

Here are commands I used for target  (from Administrative command prompt). g:\ in my case is flash drive. Note if you not provide /store ... parameter you will be modifying current OS BCD. 

rem allow F8 menu

bcdedit /store g:\boot\bcd /set "{default}" bootmenupolicy Legacy
rem turn on debugging

bcdedit /store g:\boot\bcd /set "{default}" debug on
rem debugger parameters
bcdedit /store g:\boot\bcd /dbgsettings net hostip:192.168.111.114 port:50000
rem need to point proper lan adapter

bcdedit /store g:\boot\bcd /set "{dbgsettings}" busparams 4.0.0

I used this command to force breakpoint at kernel initializaition, likely not recommended unless you have setup working.
bcdedit /store g:\boot\bcd /set "{default}" halbreakpoint yes

Hello lesliegs,

There have been many causes of Windows Installer errors.
1) It could possibly be related to the SQM client which is part of the Customer Experience Improvement Program (CEIP). Under the default setting, where MachineThrottling is enabled in the registry, any calls to WinSqmStartSession in ntdll.dll file will cause Explorer to crash, or Windows Installer installation to fail. 

Iin order to fix the issue, just remove and delete the MachineThrottling registry entry from system registry. From the Start Search box, type in Regedit and scroll to the correct entry. Changes are saved when you exit regedit. The MachineThrottling registry entry is located inside the following registry key: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\SQMClient\Windows\DisabledSessions

Be sure and do a system restore point in case there is a problem editing the registry.
Creating a System Restore point manually:Right-click on the Computer icon in the Desktop, and choose Properties
In the left pane, click System Protection
Alternately, to access the System Protection tab directly, click Start and type SystemPropertiesProtection.exe.
If you are prompted for an administrator password or confirmation, type the password or provide confirmation.
Click the System Protection tab, and then click Create.
In the System Protection dialog box, type a description, and then click Create

2) It could be related to 3rd party software. To see if there is a conflict run a clean boot with MSCONFIG.

Go to Start button, and in the start search box, type in MSCONFIG. Then deselect all start up items. Rebooted and see if the error occurs.
Then I continued to run MSCONFIG and checked one program at a time to start on bootup. Rebooted, start over, checking one program at a time.
This is very time consuming but you can select several programs at at time to try and identify the conflict.
