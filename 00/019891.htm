a. user say:

As far as I know there is no particularly good way to do this. Although you can intercept the command "HyperlinkOpen" with your own VBA sub with the same name, that only intercepts the action when the user chooses to follow the link from the right-click context menu, not the ctrl-click action.

A fairly simple workaround that may work for you is to wrap each hyperlink inside a { MACROBUTTON } field like this:

{ MACROBUTTON processHL http://www.mylink.xxx}

and write a macro called processHL that does, e.g.

Sub processHL()
  If Selection.Range.Hyperlinks.Count > 0 Then
    ' do what you want here - e.g....
    MsgBox Selection.Range.Hyperlinks(1).Address
  End If
End Sub

a. user say:

Given this kind of problem, following the link in code may not actually make any difference, but it is probably worth a try.

The macro can be given any (legitimate) name, but obviously you need to use the same name in the macrobutton field and in the VBA editor. SO if there is already a routine called procesHL somewhere in your system, you can call the routine myprocessHL or whatever.

The following macro seems to work in simple cases - it may need some tweaking for specific link types, and so on.

Sub myprocessHL()
If Selection.Range.Hyperlinks.Count > 0 Then
  ActiveDocument.FollowHyperlink Selection.Range.Hyperlinks(1).Address, _
  Selection.Range.Hyperlinks(1).SubAddress
End If
End Sub

The main reasons I suggested that you surrounded the actual hyperlink by a Macrobutton field are that 
 a. I assumed you might already have a lot of hyperlink
 b. you still have something that looks like a link, probably has the same "display text" as your link, but also contains the link address, subaddress, and potentially other info such as the target window.

To turn each hyperlink into a macrobutton manually, you can
 - Select the link (it will probably be easier to see what is happening if you use Alt-F9 to display the field code so you se { HYPERLINK ... } first)
 - press ctrl-F9 to surround it with a new pair of the special field brace codes
 - after the new initial {, type
 MACROBUTTON myprocessHL

and make sure that
 - there is a space after myprocessHL
 - there is no space before the closing } of the MACROBUTTON field

IMO it would be far better to resolve the underlying problem and stick with standard hyperlinks, but I do not know how to do that if you have tried all the suggestions Microsoft's support article describes. If there is no other solution, I'd try one macrobutton set up as above. If the problem is still there, it's obviously a waste of time trying to pursue this approach.

Excel Cell Sizing: Real World Measurements

Does anyone have a conversion formula I can use (what 1mm is in "points"), or better yet, a way to set the cell size unit of measurement to an actual unit of measurement?

ossiemac say:

Change Units of Measurement

  • Select the "View" ribbon
  • Select "Page Layout" on the ribbon.
  • Select Home ribbon and then Format -> Row height/column width settings and they will be in actual length measurement units of inches or metric depending on the default for your region.

  • To change from inches to centimeters or millimeters (or vice versa)
  • Click the "File" menu and select "Options."
  • Select the "Advanced" tab, and then scroll down to the Display section.
  • Click the drop-down button next to "Ruler Unit"
  • Select either "Inches", "Centimeters" or "Millimeters" from the options
  • Click "OK" to return to your worksheet

    Added as an after thought: Unfortunately I don't think there is any way of doing the above from the standard "Normal" view. Must be in "Page Layout". However, you should be able to make your settings in "Page Layout" and then return to "Normal" view.

Add Email Signature on Word-to-Outlook Merge - Microsoft Community

amy baggott say:

Add Email Signature on Word-to-Outlook Merge

I'm creating a Word Merge document that our sales reps are going to use to send personalized emails to our prospects.  Each rep is going to send to a certain group of prospects (which can be done using a filter).  However, I'd like each reps messages to have his own email signature appended so that it matches the "from" address (so I don't have Mike's signature on an email coming from Sarah, for instance).  Is there a way to do that, or would I have to create a separate message for each of them?
graham mayor say:
While you cannot readily change the 'from' account on the fly, you can change the signature according to the data. This would work if you were mailing from a common company account e.g. *** Email address is removed for privacy ***.

You would need to include a field in the data source with the rep's name or uniquely identifying ID. Then conditionally merge that to produce the correct data for that rep. At its most basic it could be something like

This is { MERGEFIELD From }'s Signature

If you want to introduce different telephone extensions etc, you could add those to the data or insert them directly into the message

{ IF { Mergefield From } = "Mike" "This is Mike's Signature block" }{ IF { Mergefield From } = "Sarah" "This is Sarah's Signature block" }{ IF { Mergefield From } = "Bill" "This is Bill's Signature block" }etc.

To see this in action without sending the messages, see http://www.gmayor.com/MailMergeWithAttachments.htm and use the option to merge to the body of an e-mail message (which won't add the Outlook signature). Attachments are not mandatory with this add-in.

3) Choose Language to continue.

4) Click on Options, adjacent to the preferred language.

5) Choose Write characters in freehand under Handwriting section.

6) Click on Save button.
