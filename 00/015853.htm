If you are using Yammer enterprise network associated with office 365 and a verified admin then can navigate to the Yammer network -> Settings -> Network admin -> Export data -> Enter start and end date -> Export.

You will get a file as group.csv and you can find out all group name but to see the group admin there is no such report but by navigating to the groups -> members -> users with star mark can check the group admins.

Groups.csv- You will find the columns as (i) id, (ii) name, (iii) private, (iv)moderated, (v) api_url, (vi) created_by_id, (vii) created_by_type, (viii) created_at, (ix)updated_at, (x) deleted, (xi)external

i. Id:- that is the group id for that group

ii. Name:- This will indicate name of group

iv. Moderated :- If any changed done than value will be there

v. Api_url:- This contains the URL which you can use in the API.

vi. Created_by_id:- ID of the user will be mentioned who created the group, and in the users.csv we can find the same user who created the group

vii. Create_at:- Time of the group creation

viii. Deleted :- if the group is deleted then the value will be true and the group is active than value will be false.

ix. External:- if the group is external then the value will be true and if the value is false than the group is false.

rong# say:

To Change Office Key

It can be done via the command line.  Open up a Command Prompt (Press the Windows Button and type CMD or navigate to All Programs -> Accessories -> Command Prompt).  Then you just need to use the command below that matches your OS.

 

For 32 bit Windows:

cscript "C:\Program Files\Microsoft Office\Office15\OSPP.VBS" /inpkey:yourkeygoeshere 

 

For 64 bit Windows (assuming you are using 32 bit Office):

cscript "C:\Program Files (x86)\Microsoft Office\Office15\OSPP.VBS" /inpkey:yourkeygoeshere

 

Obviously, you need to replace yourkeygoeshere with your product key.

 

Note:  If you are installing a 64 bit version of Office on 64 bit Windows, the command for 32 bit Windows should be used (or just delete the (x86) part since it won't be installing to the C:\Program Files (x86)\ directory).

bernard liengme say:

The trouble with MATCH (and with VLOOKUP) is that unless you are gong fro an exact match, the data needs to be in ascending (or descending) order. Unfortunately the price of stock does not always increase with time. So you need to roll you own function with VBA.

 

I have ignored the 90 day part of the problem (I could not follow that) but have concentrated on find the day (if any) when the stock is worth OriginalValue*(1+percentage in A1).

The table below shows my data; starting in A3 we have the days, B3, C3 and D3 have purcahse prices of stock on day 1.

 

The fromula in B2 is =testme(COLUMN(B2)) and this is copied across giving  =testme(COLUMN(C2)), =testme(COLUMN(D2)) .....

 Note how in B2 we get value 5 - on day 5 the B stock was worth at least 50% more than the starting value in B3.

 

 

50%
9 5 9 0
1 5 100 80
2 5 99 77
3 5 98 76
4 4 97 75
5 11 96 74
6 13 95 73
7 14 110 72
8 15 145 71
9 17 150 70
10 20 155 69

 

Here is my code; you need to cahnge 12 to 1252. If new to VBA have a look at one of the links below

Function testme(mycol)
  For j = 3 To 12
    If Cells(j, mycol) >= Cells(3, mycol) * (1 + Range("A1")) Then
      testme = Cells(j, "A")
      Exit Function
    End If
  Next j
End Function
