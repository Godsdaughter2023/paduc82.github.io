lemp say:

If you are connecting to the printer using wi-fi, the symptom you describe is almost always caused by the printer being assigned a different IP address than the one it had when it was installed. 

If you are not using wi-fi to connect to the printer, ignore the rest of this response.

Using your printer's User Guide, print a test page using the printer's front panel controls.  This will determine whether the problem is in the printer itself or in the communication between the computer and printer.

Assuming that you can print a test page, follow the instructions in the printer's User Guide to display the IP address assigned to the printer (you may be able to print a network configuration page that has this information).

Go to Devices and Printers, right-click on the icon for your printer, select "Printer properties," and click the "Ports" tab.  The port assigned to your printer will be highlighted and checked.  Is the IP address shown here the same as the IP address assigned to the printer?

If the IP addresses are different, print jobs will just "disappear." 

The best solution for this situation is to assign a static IP address to the printer.  For directions, see your printer's User Guide.  The selected IP address must be in the same subnet used by your router (i.e., the first 3 groups of numbers must be the same) and it must be outside the range used by the router's DHCP server. 

If you change the IP address of the printer, you'll have to install it again.  Just use the "Add a printer" wizard in Devices and Printers.  Once the "new" installation is complete, remove the old one (right-click > "Remove device").

Alternatively, your router may be capable of reserving an IP address.  See your router's User Guide.

If you need help with any of the above, provide the make, model, and any hardware version shown on the label on the back or bottom of your wireless router.  Also provide the current IP address of your printer.

As I wrote several days ago, the behavior you describe, including the job "flashing past" in the print queue window, is just what happens when a networked printer has its IP address changed.  That is, it looks like Windows is directing the print job to a different port than the port to which the printer is actually connected.  I've never seen or heard of this behavior with a USB-connected printer, but at this point, it's the only idea I have left ... and for USB ports, the only solution I can imagine is to completely uninstall and reinstall the printer.

I will assume that you have no other printer.  The following procedure will remove all of your printers, so if my assumption is wrong, you'll have to re-install those as well as the Lexmark.

Disconnect the printer from the computer.

Uninstall the printer: Go to  Start / All Programs / Lexmark XXXX Series / Tools / Uninstall Lexmark XXXX Series and run the uninstaller.

If for some reason there is no uninstaller in the "All Programs" list, go to Control Panel > Programs and Features and uninstall any Lexmark software.

Reboot the computer.

Download and run this Microsoft "Fixit" routine --> http://go.microsoft.com/?linkid=9829711  This automated routine will reset the entire Windows printing subsystem to something close to the way it was when Windows was first installed.

Reboot the computer.

Download the Lexmark software here --> http://support.lexmark.com/index?locale=EN&page=product&userlocale=EN&productCode=LEXMARK_X6675&focusedTab=DOWNLOADS#2  If it were me, I'd only download the "Core driver ... minimal print and scan drivers with supporting software" rather than the "Complete" package, but that's your choice.  One package is about 63 MB and the other about 128 MB.  A description of what's included in each package is on the download page.
