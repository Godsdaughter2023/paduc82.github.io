marvin ma msft say:
Hi FrankBranca,

It’s a pleasure to have you participate in our community. As I understand it, if you just want to integrate the email, contacts and calendar of two accounts into one, Outlook provides the feature of importing email, contacts, and calendar from an Outlook Data File (.pst). And you can follow the steps below. Besides, we suggest to making a backup before the steps.
1. Open Outlook desktop client, click File > Open & Export > Import/Export.
2. In the Import and Export Wizard pop-up, select Import from another program or file, and then click Next.
3. Choose Outlook Data File (.pst) and click Next.
4. Click Browse to select the .pst file you want to merge and choose Do not import duplicates under Options.
5. In the next page, select Import items into the same folder in:, and the email address is another account you want to merge.

However, if you attempt to merge the two .pst files into one, please finish the process above to import two .pst files into one account, and then you can carry on Exporting the Outlook Data File to ‘merge’ the two .pst files into one by following the steps below.
1. Click File > Open & Export > Import/Export.
2. In the Import and Export Wizard pop-up, select Export to a file, and the click Next.
3. Choose Outlook Data File (.pst) and click Next.
4. Select the account that has been imported two Outlook data files and click Next.
5. Click Browse to determine the save location of the .pst file.

And if you have any difficulty in the process above or concern on this issue, feel free to tell me. I will help you to check if there is any workaround.

Excel Web App: Get a history of who made what changes? - Microsoft Community

jet shop it say:

Excel Web App: Get a history of who made what changes?

We've got a spreadsheet in the Excel Web App that multiple users update through the day, and we've been having issues with incorrect entries and are trying to find out who is responsible. Is there any way to get a history of who made what changes, for example to find that user John Doe typed 17 into cell C3 at 12:30pm, that sort of thing?

jllatham say:

I actually do not know if this will work or not - no experience using the Web Apps.  But the code below is what I would use in a regular workbook to do this kind of tracking, and so maybe it will work in the Web App, or maybe not.  Absolutely no guarantees.

 

The code goes into the ThisWorkbook code module.  See this page for instructions:

http://www.contextures.com/xlvba01.html#Workbook

 

The code will (presumably) record who made changes, giving you the sheet name that was changed, the addresses of the cells that were changed and what the new values were after the changes were made.

 

Before I tested it, if I were you, I would definitely make a backup of the file in case this turns out to be a case of "...and then things went horribly wrong!".

 


Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
'have a hidden worksheet named "ChangeLog" in the workbook
  Dim nextRow As Long
  Dim specificChanges As String
  Dim logWS As Worksheet
  Dim anyCell As Range

  Set logWS = ThisWorkbook.Worksheets("ChangeLog")
  nextRow = logWS.Range("A" & Rows.Count).End(xlUp).Row
  'always keep room for at least 1000 entries
  If Rows.Count - nextRow < 1000 Then
    'delete old entries at top of the log
    logWS.Range("A2:A1001").EntireRow.Delete
  End If
  'see if nextRow changed because of rows delete action.
  nextRow = logWS.Range("A" & Rows.Count).End(xlUp).Row + 1
  For Each anyCell In Target
    specificChanges = specificChanges & anyCell.Address & " = " & CStr(anyCell.Value)
  Next
  'make the log entry
  logWS.Cells(nextRow, 1) = _
   Application.UserName & " Changed Sheet " & Sh.Name _
   & " cells " & specificChanges
  Set logWS = Nothing
End Sub
