As the following screen shot shows, that should not happen

If you send me a copy of the document, referencing this thread in the covering email message, I will investigate the issue.

The issue with the document that I received was that rather than the Heading 1 and Heading 2 styles being used for the headings, the TOC 1 and TOC 2 styles had been used and the formatting of the Headings had been changed so that the text was center aligned. 

By default, the TOC # styles are set to be automatically updated so that any "direct formatting", such as Center Alignment, applied to the paragraphs to which those styles have been applied causes the definition of the style to be updated and so when the Table of Contents was updated, the entries in it were also Center Aligned.

stefan blom say:

suzanne s. barnhill say:

On the View tab, click on the top part of the Macros split button (far right). Make sure that "Macros in" is set to "All active templates and documents" (note: do NOT look in the list of "Word commands"). Check to see whether there is a macro named FileSave. If so, this macro is intercepting Word's built-in command. You can change the "Macros in" setting to try to figure out where the macro is coming from (which might help you establish whether a defective third-party add-in is causing the problem), but if you can delete it, this should solve the problem.

Another way to establish whether a macro is causing the problem is to start Word in Office Safe Mode (hold down CTRL while loading Word and answer yes when asked if you want to use Safe Mode). If you are able to Save or Save As in Safe Mode, then an add-in or a rogue macro in the Normal template (Normal.dotm) is likely the problem.

Note that using Ctrl+S just runs the FileSave command; if a macro is intercepting it, you should get the same result from Ctrl+S that you get from clicking buttons.
