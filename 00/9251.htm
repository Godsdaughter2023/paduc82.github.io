Sub GetAllHyperlinks()

'Updateby20140214

Dim docCurrent As Document

Dim docNew As Document

Dim oLink As Hyperlink

Dim rng As Range

Application.ScreenUpdating = False

Set docCurrent = ActiveDocument

Set docNew = Documents.Add

For Each oLink In docCurrent.Hyperlinks

Set rng = docNew.Range

rng.Collapse

rng.InsertParagraph

rng.InsertAfter (oLink.Address)

Next

docNew.Activate

Application.ScreenUpdating = True

Application.ScreenRefresh

End Sub

bill manville say:

My preference, for controls on a worksheet, is to use those which are confusingly called Forms controls.  These go back to Excel 5 and are simpler to add and modify that ActiveX controls.

However, they have fewer properties and events.

 

The following code will create a checkbox at B3 on Sheet1 and make its caption match the name in

A2 on Sheet2, names it "chk" followed by that name, links the checkbox to cell B2 on Sheet2 (so that Excel formulas can use the TRUE/FALSE value) and assign a macro "CheckChanged" to the checkbox.

(You may not need all those aspects)

Sheets("Sheet1").Range("B3").Select
With Sheets("Sheet1").CheckBoxes.Add(Selection.Left, Selection.Top, Selection.Width, Selection.Height)
  .Caption = Sheets("Sheet2").Range("A2").Value
  .Name = "chk" & Sheets("Sheet2").Range("A2").Value

  .LinkedCell = Sheets("Sheet2").Range("B2").Address(external:=true)
  .OnAction = "CheckChanged"
End With

 

A common procedure can be written to handle all the checkboxes and put in a standard module:

Sub CheckChanged()
  MsgBox "CheckBox " & Application.Caller & " is now " & IIf(Sheets("Sheet1").CheckBoxes(Application.Caller).Value = xlOn, "On", "Off")
End Sub

 

The procedure to add the checkbox can be modified to delete all the checkboxes on Sheet1 and then add checkboxes for all items in a list starting at A2 in Sheet2.

 

Sub RemakeCheckboxes()
  Dim lRow As Long
  Sheets("Sheet1").CheckBoxes.Delete
  lRow = 2
  Sheets("Sheet1").Range("B3").Select
  Do While sheets("Sheet2").Cells(lRow, "A")<>""
    With Sheets("Sheet1").CheckBoxes.Add(Selection.Left, Selection.Top, Selection.Width, Selection.Height)
      .Caption = Sheets("Sheet2").Cells(lRow, "A").Value
      .Name = "chk" & Sheets("Sheet2").Cells(lRow, "A").Value

      .LinkedCell = Sheets("Sheet2").Cells(lRow, "B").Address(external:=true)
      .OnAction = "CheckChanged"
    End With
    lRow = lRow + 1
    Selection.Offset(1).Select
  Loop

 End Sub

bob jones aka: cybertaz say:

I'm not sure where you're getting this information, but that terminology isn't used by any version of Mac Word that provides Publishing Layout View :-)

First, though, keep in mind that each Content Page has a Master Page associated with it. It may be a unique Master Page or it may simply be an instance of the preceding Master Page, but Master Pages in a Word document are not the same as what you may expect if you're familiar with an actual Desktop Publishing program such as InDesign or QuarkXpress.

"Master Pages" in Word essentially are nothing more than easily exposed Header/Footer layers, they aren't separate entities. That means you can't have one without the other. Soooo, to delete a Master you need do nothing more than delete the associated Content Page by selecting it in the Sidebar & pressing the del [forward delete] button or;

  1. Display the Thumbnails pane [View> Sidebar> Thumbnails]
  2. Right-click the page you want to remove
  3. Select the Remove Page command

Alternatively, you could select the page in the Sidebar, then use either Insert> Remove Page in the main menu or click the Remove button in the Pages group on the Layout tab of the Ribbon.

Another option would be to click the Master Pages tab in the lower left corner of the document window, then do as described in the 3 step process above.

Either way, the page, its Master & all content on each of them will be removed from the document.

john korchok say:

They do use that terminology, but they're really just headers and don't act like master pages in any other program that uses master pages. As one example, you can't apply a Word master arbitrarily to any page, they follow the same limits as headers: every page or first page and/or even and odd. By contrast, in Publisher and PowerPoint, you can apply any master to any page in any order, which is far more useful.

To answer the original question, switch to Page Layout view, find the header you want to get rid of, double-click at the top of the page to open the header and delete the header content. 

You'll also find that section breaks have been added to your document for each "Master" that was added. After a trip to Word>Preferences>View and checking All to show the nonprinting characters, it's easy to see the section breaks as double blue lines, in case you want to delete them.
