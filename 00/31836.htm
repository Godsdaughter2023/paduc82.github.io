steve0319 say:

OK. I found the answer by myself.

Go to "Windows Firewall with Advanced Security" in "Administrator Tools".

Select "Inbound Rules"

Find "File and Printer Sharing (SMB-In)" Local Port 445. There are several these rules. Select the active one which has the green button. The default setting for "Remote Address" is "Local subset". Change the "Remote Address" to "Any". It works.

jose [msft] say:

Hi Meliton,

If the Windows Spotlight image is not working as expected, here are a few tips to try:

  1. If the Windows Spotlight image is not showing on your lock screen or sign-in screen, first check Settings to see if the options are turned on. Open Settings, and select Personalization, then Lockscreen. Make sure the Background is set to Windows spotlight. Then set Show lock screen background picture on the sign-in screen to On.


  2. If the Windows Spotlight option is missing from Settings, it may not be an option for you if the device is managed by an organization and the option is disabled by group policy (for Windows 10 Enterprise or Windows 10 Education).  If so, check with your organization’s system administrator.

  3. Check if the Windows Spotlight image appears if you type the Windows icon key + L on your keyboard.  If the image appears when you lock the screen this way, but not when you reboot, please note that Windows Spotlight can only take effect if there is a user actively signing into the system.  When you reboot, that signs the user out.  Once the user signs in again, their Spotlight settings will be active.  This means Windows Spotlight does not display on reboot (restart or shutdown) or with some modern sleep options, such as hiberboot.

  4. If your Windows Spotlight image is stuck on the same image and is not changing, advanced users may be able to get it working by editing the registry.
    Warning:  Serious problems might occur if you modify the registry incorrectly. For added protection, back up the registry before you modify it.
  • Open the RegistryEditor.  (Press the Windows logo key + R, then type regedit, and select OK.)
  • Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Authentication\LogonUI\Creative
  • Under each subkey (S-1-5-21-… or user SID), there may or may not be more subkeys (131565…). If those subkeys exist, delete them. Don’t delete the “Creative” key.
  • Lock the screen. You will not see anything for up to a week but hopefully it works again after that.

IMAP in Outlook Express - using SUB-folders as special folders - Microsoft Community

l-w-c say:

IMAP in Outlook Express - using SUB-folders as special folders

Servers such as Gmail require you to use a slash ("\") in the path of the special folders.

Alas, OE doesn't accep slashes in the special folders. Some articles about Windows Live state using "#" instead (e.g. "[Gmail]#Drafts") but (at least in OE) it just creates an actual folder called that name.

One working but VERY bad solution is fixing just the symptom. That is, defining "[Gmail]" as the root folder. This avoids the symptom by turning the sub-folders into regular folders. It's bad not just because it bypasses the actual problem, but because it means losing access to each and every custom Gmail folder! It's better off disabling special folders than agreeing to this.

So how can you deal with the real issue and just use sub-folders?

Thanks!

chris visscher say: WARNING: Incorrectly editing the Registry can cause your system to become unstable or crash completely.

In case someone (like me) stumbles upon this old thread:

To use the special IMAP folders from Gmail in OE (Outlook Express):

Step 1:
Go to the IMAP tab in your account properties in OE:

Check:
[x] Store special folders on IMAP server

 Sent item path: [Gmail]#Sent Mail
 Drafts path: [Gmail]#Drafts

(You temporarily use # because the / is not allowed by OE)

Step 2:
Close OE. And open Registry Editor (regedit).
Search the Registry for [Gmail]

If everything goes well, you'll find the IMAP configuration for this account in a subkey of something like this:
HKEY_USERS\S-1-5-21-xxxxxxxx-xxxxxxxxxx-xxxxxxxxx-xxxx\Software\Microsoft\Internet Account Manager\Accounts\0000000x

This location contains 2 Keys type REG_SZ:
IMAP Sent Items Folder
IMAP Drafts Folder

Edit these keys and change the # into /. This should look like this:
 IMAP Sent Items Folder REG_SZ [Gmail]/Sent Mail
 IMAP Drafts Folder REG_SZ [Gmail]/Drafts

Close the Registry

Step 3:
Open OE. Go to Tools > Options > Tab Send.
Uncheck:
[  ] Save copy of sent messages in the "Sent Items" folder

Why? Because Gmail already saves a copy in your Sent Items for you. Why then still go to the trouble to set this as your Special folder Sent Items? Because the display column order is different on this "Special folder" and this way you'll see the To column as the first column of this folder.

The problem with this workaround is that whenever your settings have to be changed, you have to re-apply this trick since OE won't allow these settings to be saved from within OE.

Good luck!

Chris
