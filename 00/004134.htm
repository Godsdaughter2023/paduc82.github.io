jack cook - aka help_line say:

Good, that might well make it easier to resolve.  It is an Outlook error and generaly can be easily resolved.

 

Here is a list of common issues relating to error 0x85010014

List of General Issues

  1. If you are seeing the error 85010014 (a general error) then try uninstalling and reinstalling Outlook. The error in ActiveSync is actually from the Outlook APIs.  You will need to reconfigure Outlook unless you backup your settings.  To uninstall Outlook click on the Control Panel -> Add or remove Programs -> Office 2003 and chose to modify the installation by adding or removing components. Uncheck Outlook and updated it to remove Outlook from Office 2003. Then click on Office 2003 again and re-check the option to install Outlook to add it back to Office 2003. You may need your Office 2003 CD. Then reboot.
  2. If you are seeing the error 85010014 (a general error) and are using Exchange 2003 and folder redirection with Outlook, you may have to turn folder redirection off to sync or upgrade to ActiveSync 4.2.
  3. If you are seeing the error 85010014 and you are using Outlook in  off-line mode with Exchange, you must be online with Exchange to sync.
  4. If you are seeing the error 85010014 and you have Outlook script blocking setup with your antivirus or anti-spyware program enabled you must configure your antivirus or anti-spyware program to allow ActiveSync to send scripts to Outlook.
  5. If you are using an unregistered or illegal copy of Outlook you may see the 85010014 error.

Outlook 2010: macro to reply to sender with greeting - Microsoft Community

mcdude_778 say:

Outlook 2010: macro to reply to sender with greeting

I would like to have a button for a macro in my Outlook 2010.

When clicked, it will generate a greeting:

"Good Morning, [name]"

"Good Afternoon, [name]"

"Good Evening, [name]"

How can I do this in code?

I would specify the time ranges for morning, afternoon, and evening.

Please advise.

graham mayor say:

I think the following should work for you. Select the message you wish to reply to and run the following macro:

Sub TimeOfDayReply()
Dim olItem As Outlook.MailItem
Dim olOutMail As Outlook.MailItem
Dim olInsp As Outlook.Inspector
Dim wdDoc As Object
Dim oRng As Object
Dim sTimeOfDay As String
Dim sName() As String
    If Application.ActiveExplorer.Selection.Count = 0 Then
        MsgBox "No Items selected!", vbCritical, "Error"
        Exit Sub
    End If
    Set olItem = Application.ActiveExplorer.Selection.Item(1)
    Set olOutMail = olItem.Reply
    With olOutMail
        .BodyFormat = olFormatHTML
        Set olInsp = .GetInspector
        Set wdDoc = olInsp.WordEditor
        Set oRng = wdDoc.Range
        oRng.collapse 1
        If Time < "12:00:01" Then
            sTimeOfDay = "Morning"
        ElseIf Time > "12:00:00" And Time < "18:00:01" Then
            sTimeOfDay = "Afternoon"
        Else
            sTimeOfDay = "Evening"
        End If
        sName = Split(olItem.SenderName, Chr(32))
        
        oRng.Text = "Good " & sTimeOfDay & Chr(32) & sName(0) & Chr(44) & vbCr
        oRng.collapse 0
        oRng.Select
        .Display
    End With
End Sub

jay freedman say:

Don't bother uninstalling and reinstalling, as that would be a waste of time. Just do a quick repair of Office, which will restore the registry entries that point to the correct icons.

If you do plan to uninstall and reinstall, it "probably" won't delete the existing add-ins. But it would be best to prepare for the possibility by backing up any templates (*.dotm and *.dotx) in these folders (or your entire drive):

%appdata%\Microsoft\Word\Startup

C:\Program Files (x86)\Microsoft Office\root\Office16\Startup

Just to complicate and confuse, Microsoft also uses the term "add-in" for web-based programs from the button at Insert > Add-ins > Store. The ones you've installed that way are recorded in your Microsoft account and are listed in the "My Add-ins" dialog. Those should not be affected by a reinstall.
