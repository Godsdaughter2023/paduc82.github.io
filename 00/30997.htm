This issue might have occurred because of a corrupted user profile.

You can copy the files from the existing profile to the new profile which you can create. Click on the link below.

http://windows.microsoft.com/en-us/windows-10/getstarted-set-up-accounts 

  1. Log on to the newly created user profile.

  2. Open the My Documents folder. Double-click the hard disk drive that Windows is installed on (it's usually your C: drive), double-click Users, double-click the folder with the name of your account, and then double-click My Documents.

  3. Click the Tools menu, and then click Folder Options.

    If you don't see the Tools menu, press Alt.

  4. Click the View tab, and then click Show hidden files, folders, and drives.

  5. Clear the Hide protected operating system files check box, click Yes to confirm, and then click OK.

  6. Locate the C:\Users\Old_Username folder, where C is the drive that Windows is installed on, and Old Username is the name of the profile you want to copy files from.

  7. Select all of the files and folders in this folder, except the following files:

    • Ntuser.dat

    • Ntuser.dat.log

    • Ntuser.ini

  8. Click the Edit menu, and then click Copy.

    If you don't see the Edit menu, press Alt.

  9. Locate the C:\Users\New_Username folder, where C is the drive that Windows is installed on, and New_Username is the name of the new user profile you created.

  10. Click the Edit menu, and then click Paste.

    If you don't see the Edit menu, press Alt.

  11. Log off, and then log back on as the new user.

dustin harper say:
Wow. That's a HUGE undertaking. Even a text based OS is a huge project. Adding a GUI is exponentially much more difficult.

Windows is built on C, C++, ASM (Assembly Language). Each application could be built on something else, though.

Not sure where you would go to start. Look at Linux. That started and took a lot of time to get where it is today, and it's just the core. The other packages that make up the GUI and others (GNU) are separate.

It's not a weekend project. It's a huge project.

I'm hoping you just want to make your own application and not a full fledged operating system like Windows or Linux. In that case, you can search for which program language you want to use (they all have strengths and weaknesses). For a beginner, I'd use Visual Basic, as it's very easy to learn and use, yet powerful. You can make simple Windows applications pretty fast, and there are actually some pretty decent weekend projects you can do. A lot of open source code out there for various libraries to make your application better and more efficient.

The biggest thing is to find a book or site that starts from the beginning and just start and go to town.

dustin harper say:
As Mike sent above: Linux source code is freely available. You wouldn't be making your OWN OS, but using someone elses source code. You could study it and use it as an example to making your own OS, though.

If you want to use ASM (assembly language), which can also be done through C++ instructions, you can make a very simple OS and build up from there: http://mikeos.berlios.de/write-your-own-os.html

C++ coding for an OS: http://mjc88.0catch.com/

It's not that anyone was implying that you are stupid. It's usually that it is a huge undertaking (many, many people are responsible for the full Linux or Windows or Mac OS) that requires an already deep knowledge of PC hardware and software. It's usually the case of "If you have to ask, you don't know how" type of thing. I know some very skilled programmers that wouldn't even attempt such a thing. That said, I've been interested in it myself, and have even tried making a small OS for an older 386 machine just for fun. It is a very, very fun and worthwhile project, but can get very in depth.
