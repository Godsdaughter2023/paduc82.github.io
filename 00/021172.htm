marshall barton say:

The wizards are very limited in how much they can do for you.  To go beyond those simple helper things, you need to work on the form in design view.

 

To display some fixed text, add a label control to the section (form footer) you want the text and set it's Caption property to the text.  This is very easy via the ribbon's Form Design Tools - Controls section.  Click in the Aa tool and then put the mouse pointer in the form footer section and drag a rectangle to the size you want.  The cursor will then automatically be placed inside the lable where you type the desired text.

 

If/when you want to change the size or location of the label, just click in the label and drag it to a different place or grab a sizing handle and drag it in the direction you want to to make bigger or smaller.

alankleinnyc say:

I am using both the current version of Office for Mac and the current version of OS X.

The issue, it seems, is caused by a bug in the current version of El Capitan 10.11.4 and with the current version of Excel. This is from Apple Discussions and it jives with what I've learned from an AppleCare product specialist:

It's a bug in Quicklook in El Capitan, more specific in /System/Library/Quicklook/Office.qlgenerator version 5.0.
Whenever Quicklook is used to generate an icon preview for an Excel file which contains a formula (which most of them do), it will keep the file open. Even if you close the Finder window which contained the file. Thus making it impossible to delete the file and then empty the trash.
Workaround is to disable Icon preview for al Finder view modes, so Quicklook will never create a preview of the file.
And if you already have this problem with a file, then a reboot will work, but force quitting all QuickLookSatellite processes with Activity Monitor instead will save you some time.

The full discussion is here: Apple Discussions
