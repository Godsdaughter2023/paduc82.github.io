dirk goldgar say:

Terminological note:  one doesn't "run a module".  A module is a container for procedures (Subs and Functions) -- it's these procedures that one runs.

 

Now to what you're trying to do.  It's easy enough to loop through the records in a form's (or subform's) recordset, but one can't do a whole lot to set different properties for the same control on different rows of a continuous form.  The reason is that, the way continuous forms are implemented, there is really only one copy of each control, just "painted" multiple times, once for each displayed row.  If you change a property of a control, for one row, you change it for all rows. 

Basically, if you want controls to be formatted differently on different rows, you are limited to what can be done with Conditional Formatting.  Depending on exactly what sort of "state" you have in mind, it may be possible to do this in the case you're working on, but you won't be able to loop through the rows to do it.  You'll have to set up format conditions that Access can evaluate.  That may not be as limiting as you think, since your format conditions can call your own user-written function, based on values in the row being formatted.

 

dirk goldgar say:
Thanks.  I have actually never used conditional formatting.  I have a couple questions about it.

First, I'm looking at the conditional formatting dialog box right now.  I see bold and italics and such.  Can I set other control properties, such as visible or locked?  I'm guessing I can't, huh?

Second, is there any way to reference a control in a previous row?  One of my states would be set based on the relationship between rows.
 


Unfortunately, you can't set the Visible or Locked properties with Conditional Formatting.  You can set the Enabled property to enable or disable a control, but you can't just lock it.  Many people would love it if you could set the Visible property.  I think I recall reading about people using other properties to hide a control, but I can't remember the details.  You might try a web search on that topic.

It's tricky to refer to a control in a previous row, in a CF condition.  If you write your own function and get CF to call it, I think you can, by working backward from the control to its parent form and that form's recordsetclone.  Stephen Lebans did something like this in his conditional formatting wizardry demo:

 

    http://www.lebans.com/conditionalformatting.htm
