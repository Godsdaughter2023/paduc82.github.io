This assumes that your initial workbook has two worksheets, named "X" and "Y"

The initial sheets should be properly formatted and populated

The macro asks how many new X and Y sheets you need and makes the copies:

 

 

Sub MultiCopy()
n = Application.InputBox(prompt:="Enter number of new X sheets: ", Type:=1)
For i = 1 To n
Sheets("X").Copy after:=Sheets(1)
    ActiveSheet.Name = "X" & i
Next
n = Application.InputBox(prompt:="Enter number of new Y sheets: ", Type:=1)
For i = 1 To n
Sheets("Y").Copy after:=Sheets(1)
    ActiveSheet.Name = "Y" & i
Next
End Sub

jay freedman say:

I agree with Doug (and also see https://support.microsoft.com/en-us/kb/811468).

Regarding the question "How can two files of the same name exist in the same folder. I know they are different file types, but I thought windows referenced the name, not the name/type aggregate.", that is not the case. If you make certain that Windows Explorer is displaying file extensions, you'll see that the shortcut has a .lnk extension. That makes the filename -- taken as a whole -- different from any file with only a .dotm extension. That's the only way Windows will allow files with identical base names (up to the last dot) in the same folder.

I'm surprised that your shortcut worked for as long as it did. It implies that no one using Word on that computer made any changes to styles, macros, AutoText, or other features that would require Word to save the Normal.dotm template. Avoiding all such actions is the only way to prevent Word from creating a new Normal.dotm file in the Templates folder.

charles kenyon say:

First, I agree with previous posters, the Normal Template should not be shared.

Expanding on Doug's response, you make another template into a Global Template by putting it in the Word Startup Folder. This folder can be shared. Such a global template can share AutoText and other building blocks, macros, keyboard shortcuts, and QAT modifications. It can serve as a Global StyleSheet but that requires extra work not required for sharing the other features.

My preference with global templates and Workgroup Templates is to put them in a network location and have a user's login script update a local folder from the network location. This cuts down on network traffic and makes modifications of the files a bit easier. It would not, however, allow adding AutoText to the global by a user. That would have to be done to the prime file on the network.
