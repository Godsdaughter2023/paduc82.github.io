greginmich say:

Web browsers are crawling with these Tech Support Scam pages – but no operating system or antivirus app will ever issue a warning in your web browser, so its easy to identify a fake warning in your web browser, because they’re all fake. And close to 100% of these scam pages lock up your browser with JavaScript tricks that don’t install any malware on your OS (that’s why they aren’t detected as malware by AV apps). So you can usually escape their clutches by either:

1. Cicking on the “OK” button in the popup dialog; putting a check in the “Don’t let this page create more messages” box; and then closing the page.

2. Right-clicking on the taskbar (or pressing Ctrl + Alt + Delete) to start Task Manager, and then ending the browser process.

3. But if you kill the Edge browser process with Task Manager, then you’ll have to restart Edge indirectly with a web search, and then right-click on the search result page tab and select “Close other tabs” in order to remove the scam page.

But after years of trying to educate people on how to escape from these browser lockups, the Tech Support Scammer’s still don’t have any trouble at all finding unsuspecting victims. So minimizing the chances of people encountering these Tech Support Scam sites remains a fundamental precept in protecting the public. This would ideally be done with the Windows Defender SmartScreen filter on Microsoft browsers (and potentially with the new Windows Defender Network Protection feature on non-Microsoft browsers) – and also by keeping high-profile websites free of malvertising. But the scammers are easily able to outpace the SmartScreen filter by simply registering new domains at a lightning pace – and we currently have unresolved malvertising issues on quite a few high-traffic sites. Including the MSN news portal, that are exposing potentially millions of people to malware, crypto mining, and Tech Support Scams.

At the same time, the scammers have pretty well managed to bypass the Edge Dialog Loop Protection feature that was designed to make escaping from these scam pages easy, and this has left lots of Edge users at a loss on how to escape the reloading of Tech Support Scam pages after Edge has been killed with Task Manager. So I think that now would be a good time for shifting the focus from escaping these Tech Support Scam pages, to preventing them.

The current malvertising campaign on the MSN news portal has given me the opportunity to see firsthand how a good ad-blocker extension, like uBlock Origin, will selectively prevent any compromised advertising domains from connecting to a high-profile host site – and will thus prevent these Tech Support Scam redirects from being piped in.

Installing an ad-blocker extension in Edge is safe and easy; because the Settings menu will take you directly to the Microsoft Store, where you’ll find a good selection of ad blockers to choose from. I’m very impressed with the effectiveness of uBlock Origin in mitigating the malvertising menace, and also with the way that it gives us a window into all of the domain connection activity that’s going on behind the scenes at the high-profile sites. But others have reported success with AdBlocker Ultimate, which is also very well-reviewed. And of course, these extensions are all free, so if you’re unhappy with the first one you pick, you can just switch over to another one:

For Microsoft Edge, find an ad-blocker extension in the Microsoft Store:

Settings and more > Extensions > Get Extensions from the Store

The documentation for uBlock Origin (and the instructions for installing it on other browsers) is available here:

https://github.com/gorhill/uBlock

The 10,000 daily recipient rate limit is only applied when sending messages out from the Office 365 side.

james truhlar say:

Current UI Issues and some suggestions for improving them.

So far I'm loving windows 10 but, there are a ton of design 101 mistakes and lots of inconsistencies. Since I'm sure minor design issues will be polished up in the next year, this write up will focus more on design issues that directly impact usability.

Virtual desktops

  • Can't drag apps between desktops.
  • Multiple monitors should not share a single virtual desktop.
  • Full screen apps should behave as their own desktop in task view.

Theme colors, limited options

  • Let power users tinker. It's a little too 'stupid proof' right now.
  • Title bar text color should be white or black depending on the theme brightness, or it just needs to be user customizable.
  • Taskbar color, start menu color, and window color should be separately adjustable. What if I want an electric pink taskbar and neon green windows? Let me choose!
  • The white/black text also applies to desktop icons/wallpaper brightness. I propose something like this. And here's a more extreme example.

Desktop vs. Mobile Issues

  • Side swiping is unintuitive (if even possible) on a desktop. Several apps cut off rather than wrap text. For example: the windows feedback app.
  • On a desktop, apps should not open maximized.

File Explorer

  • Tabbed explorer windows.
  • Simplification is good, until its not... Example (right side is my revision)

If anyone agrees, here are a few suggestions I've found that, more or less, coincide with some of these problems:

Problem solved, although not how I originally wanted it, it will do. New query directly on one of the tables used in EXAMPLE 1 above:

SELECT DateAdd("d",-Weekday(IIf(IsNull([DateStudentNotified]),CDate(#12/31/2999#),[DateStudentNotified]),2)+1,IIf(IsNull([DateStudentNotified]),CDate(#12/31/2999#),[DateStudentNotified])) AS WeekBeginning, Count(ProjectEvents.DateStudentNotified) AS CountOfDateStudentNotified
FROM ProjectEvents
GROUP BY DateAdd("d",-Weekday(IIf(IsNull([DateStudentNotified]),CDate(#12/31/2999#),[DateStudentNotified]),2)+1,IIf(IsNull([DateStudentNotified]),CDate(#12/31/2999#),[DateStudentNotified]))
HAVING (((DateAdd("d",-Weekday(IIf(IsNull([DateStudentNotified]),CDate(#12/31/2999#),[DateStudentNotified]),2)+1,IIf(IsNull([DateStudentNotified]),CDate(#12/31/2999#),[DateStudentNotified])))=DateAdd("d",-Weekday(Date(),2)+1,Date())-7));
