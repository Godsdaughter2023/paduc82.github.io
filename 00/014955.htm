How does AADConnect sync users? - Microsoft Community

ericsnijders say:

How does AADConnect sync users?

Does AADConnect synchronize users in on-premise AD and AAD fully on the "EmailAddress" field?
We have one organisation where users are already created in Office 365. We now want to sync the on-premise Active Directory users to the Office 365 users. The only problem is: the users in Office 365 don't have a mailbox.

Is it still possible to sync/connect them with AADConnect, or do the users need to have a mailbox and the same EmailAddress on both sides?

vasil l. michev say:

Provisioning new users is different than matching already existing ones. For your scenario, you can use two options:

1) Soft match, based on the PrimarySMTPAddress value: support.microsoft.com/.../2641663

2) Hard match, based on the objectGUID: blogs.technet.com/.../how-to-do-hard-match-in-dirsync.aspx

Soft match is easier, but it has some caveats as seen from the article above. Most important for your scenario - soft match will not work if the object has already been synced previously, or soft matched (i.e. if the O365 user has the ImmutableID parameter populated).
