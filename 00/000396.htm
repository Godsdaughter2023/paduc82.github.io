ken sheridan say: Firstly do not store the structured serial number as a single column, but concatenate the formatted values from numeric Style and SerialNumber columns into a single computed column in a query or a computed control in a form or report.  Combining the data into a single column would introduce redundancy and prevent the enforcement of referential integrity.  The two columns in combination are a candidate key of the table so should be defined as such, either by making them the composite primary key (the easiest option) or by including both columns in a single unique index if a surrogate primary key such as an autonumber is employed.

don cuthbert say:

Downloading the ISO might be the simplest and fastest way.  After upgrading to that, you can make sure you are on Fast Ring to get the latest build.

If you have Hyper-V, what I tested the other day was to create a VM and install it with Build 10586 and join it to Insiders and Fast Ring.  I then let that sit for several hours and eventually it downloaded the latest Fast Ring Build.  It will create an ESD file at C:\$Windows.~BT\Sources\install.esd (on the VM).  Copy the ESD file to your main/host PC and use it to create a current ISO for Build 14388.  Then mount the ISO and run setup to upgrade.

There are things to try on your PC like running the Control Panel Troubleshooter for Windows Update, switching between Fast and Slow Rings, Restarting and leaving it sit overnight, etc.  But if none of these are working, using Hyper-V to fetch the next build might be an option.  It will not activate Windows in the VM, but it will still update to the latest Build.
