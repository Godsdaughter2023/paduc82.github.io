Can't say whether it's worth following these steps or not (it's still not working), but I figured I'd show my process.

  1. Perform a failed update. This is the only way to generate a setuperr.log file - used to diagnose where the upgrade is failing.
  2. During the upgrade, record the exact time your device goes to the black screen (and eventually the blue screen). This is important because the log file has a timestamp of every problem it encounters.
  3. Then, open up your log file (located in C:\$Windows.~BT\Sources\Panther). Before doing anything, save this log file to somewhere - it's likely worth titling it appropriately (like "Try 1" or the date - something to keep it organized). Scroll down towards the middle. In mine, I identified a block of errors that actually involve driver errors (in the middle column look for [0x0808fe] MIG).  You may need it later to compare to another failed update to see what worked and what didn't.
  4. Take a moment to look through the log. At first I really wasn’t familiar with it and didn’t understand what it was trying to say. Gradually, through Googling some of the excerpts I learned which were important.
    1. The errors in specific drivers I didn't worry about are ones that end with “is not present.. and configured.”
    2. The errors I did need to worry about are ones that end with a phrase like “configured … and has problem.”
      1. Google these drivers. Oftentimes they’ll be listed in a bit of a cryptic way. For instance, take this line from one of my error reports: “ACPI\VPC2004\0 -> Migrated [0000000000000000], configured [oem33.inf/acpivpc.inf] and has problem (41).” I Googled the phrase “ACPI\VPC2004” and found out that this is the Lenovo Energy Manager.
      2. Disable or uninstall the drivers using Device Manager. Sometimes these errors can arise because of hidden drivers lingering on your system. You can view these by clicking View > Show Hidden Devices. These appear as faded out devices.
      3. For most of the issues I found, disabling the drivers was fine. For some, like my Intel Wireless N 7260 network adapter I needed to uninstall them.
  5. Perform another update. If you're downloading the Windows 10 update from online and your wireless drivers are causing problems, wait until your update downloads completely and is ready to install.

You may find that there are additional errors or different errors.  I actually had 4 separate drivers encounter issues. I needed to uninstall the Lenovo Energy Management Software, uninstall Intel Wireless N 7260 card, uninstall hidden Bluetooth personal networks, and disable both USB ports.

Method 1: Reset Windows update components.

Resetting Windows Update Components will fix corrupt Windows Update Components and help you to install the Windows Updates. Please follow the below steps to reset the Windows Updates Components manually:

  1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.
  2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.
    • net stop wuauserv
    • net stop cryptSvc
    • net stop bits
    • net stop msiserver
  3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.
    • ren C:\Windows\SoftwareDistribution SoftwareDistribution.old
    • ren C:\Windows\System32\catroot2 Catroot2.old
  4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.
    • net start wuauserv
    • net start cryptSvc
    • net start bits
    • net start msiserver

5. Type Exit in the Command Prompt to close it.

Now you may try running the Windows Updates and check if the above steps resolve the issue.
