Difficulties setting up VoIP on Symbian phones - Microsoft Community

poobah---01 say:

Difficulties setting up VoIP on Symbian phones

I have a Nokia E71, and I couldn't get VoIP to work on it for months. The phone produced various generic, unhelpful, or deceptive error messages, including the following:

 

  • No WLAN coverage for making an internet call. Make a regular voice call?
  • Unable to connect to the connection network
  • Unable to register to service
  • Internet telephone service not available. Make a regular voice call?

I've figured out the problem, so I'm intending to post the solution as a reply to this for other people who've run into the same problem. I believe this applies to various Symbian phones rather than just the E71 because I've seen users of other Nokia phone models having the same problems.

poobah---01 say:

Troubleshooting this is difficult because the error messages are too vague. Here's a listing of my findings regarding the causes of each of the different error messages. This isn't exhaustive, but it may help for some of the cases.

 

Unable to connect to the connection network

This seems to occur if you try to register the service from the "Internet tel." / "VoIP settings" section of the phone. I recommend you instead try to make an Internet call. This gives you a slightly more specific error message. To do this from the home screen, do either of the following:

  1. Type in a phone number
  2. Press Options
  3. Go to Call -> Internet Call

or

  1. Type in a contact name
  2. Select the contact from the list
  3. Press the right direction key
  4. Choose Internet call

Registration failed

This seems to appear if you view the status of the SIP profile from SIP settings. I recommend you instead make the call from the home screen as shown in the above section.

 

No WLAN coverage for making an internet call. Make a regular voice call?

  • Public user name is not specified
  • Registrar server address is not specified

Unable to register to service

This can have many different causes. If you are able to monitor the phone's network traffic, I recommend you do so to narrow down the problem. Here are two ideas that could help you get this set up:

  1. Set up a hardware device that can monitor ambient WLAN traffic using its wireless card.
  2. If you have a computer with WLAN support, see if it supports functioning as a "Soft AP" (access point). If it does, set it up for the phone to connect to it for Internet access.

I used the Wireshark program to monitor and analyse the network traffic.

 

If you won't intercept the phone's network traffic to help narrow down the problem, you can just guess at it by ensuring the following problems are addressed.

 

Phone doesn't produce any SIP traffic

  • Public user name is missing a domain part. It should be in the format user@domain.

Phone doesn't initiate authentication; it doesn't respond to the challenge / 401 response

  • User name is missing
  • Password is missing
  • Security is enabled but not supported by the VoIP provider
  • Realm doesn't match the server's. In my case, it needed to be set to asterisk despite my service provider saying it should be the same as the Registrar server address. To find out what it should be, look at Message Header -> WWW-Authenticate -> realm in the 401 Unauthorized response. If you can't intercept your phone's network traffic to do this, you can just install and configure any VoIP software on a computer, such as MicroSIP, and use Wireshark to monitor it to find out this info. See the attached image to see an example of what this looks like.

Server sends 404 Not Found response

  • User name is not recognised by the server

Server sends 403 Forbidden response

  • Password is not correct for User name


Internet telephone service not available. Make a regular voice call?

In my case, the registration was successful but the call just made a dialing sound before showing this error. In my case it was a NAT issue with my LAN. If you're using your phone service provider's Internet service instead, the problem may be something else such as blocked ports on their end.

 

If the problem is NAT-related, one of the following may help:

  • If your router supports SIP ALG, ensure it's enabled.
  • Configure a STUN server in the phone by using the SIP VoIP Settings application, which you might be able to find on the Internet and download.

As Jay said, building blocks don't live in documents, but in templates.

There are several ways to move or organize them. See Building Blocks & AutoText by Greg Maxey.

jay freedman say:
Word never saves building blocks in documents, only in dropdown that shows which templates you can save it in. Note that this dropdown never includes any document.

If you save the building block in Normal.dotm or in Build Blocks.dotx, then that building block will be available on the Quick Part gallery to be inserted in template.

When you close the document (or later close Word, in the case of Normal.dotm or Building Blocks.dotm), you should be asked whether to save changes to the template. If you say no, the building block isn't saved; you must say yes to keep it.

If the building block was never saved to a template, then it isn't a building block -- it's just something (text, graphic, table, etc.) in a document. You can always go back to the document, select that area, and press Alt+F3 and define the building block in a template. Then, following the rules above, you should be able to open another document and insert the building block from the Quick Parts gallery.

andreas killer say:
It is not possible to copy a hidden sheet, here is the proof:

Sub Test()
  Sheets(2).Visible = xlHidden
  Sheets(2).Copy
End Sub

It's not necessary to delete hidden sheets, within my code you can check it very easy:

  For Each ws In ThisWorkbook.Worksheets
if ws.Visible then
    If ws.Cells(8, 8).Text <> "" Then
      'Copy the worksheet to the new workbook
      ws.Copy NewBook.Worksheets(1)
      'remove all other sheets
      For i = NewBook.Worksheets.Count To 2 Step -1
        NewBook.Worksheets(i).Delete
      Next
      'Save a copy
      NewBook.SaveCopyAs "C:\Budget Files\Budget for " & ws.Cells(6, 2).Text & ".xls"
    End If
End If
  Next

But if you want to delete some sheets, it's easier if you count from the end to the beginning.

  Application.DisplayAlerts = False
  For i = Worksheets.Count To 1 Step -1
    'One sheet must remain!
    If Worksheets.Count = 1 Then Exit For
    If Not Worksheets(i).Visible Then Worksheets(i).Delete
  Next
  Application.DisplayAlerts = True

And to trap the errors, you should always add an error handler in your main routine.

Sub Main()
  On Error GoTo ErrorHandler
 
  'rest of the code
 
  Exit Sub
ErrorHandler:
  MsgBox "Error " & Err.Number & ": " & Err.Description
End Sub

hansv mvp say:

The spelling and grammar options are properties of Application.Options:

  • CheckGrammarAsYouType
  • CheckGrammarWithSpelling
  • CheckSpellingAsYouType
  • ContextualSpeller
  • IgnoreInternetAndFileAddresses
  • IgnoreMixedDigits
  • IgnoreUppercase
  • RepeatWord
  • SuggestFromMainDictionaryOnly
  • SuggestSpellingCorrections

(All are True/False)

To set the language of selected text, a range or a style, set its LanguageID to one of the WdLanguageID values:

Selection.LanguageID = wdEnglishUS

hansv mvp say:

You can display the dialog from VBA:

Application.Dialogs(wdDialogToolsGrammarSettings).Show

But the options within this dialog don't seem to be exposed in VBA (the built-in dialog argument list only mentions Options, which is not very useful).
