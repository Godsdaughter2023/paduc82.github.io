van dinh say:

>>Next is where I would Like to add the field Communed      as a checkbox   shows  and allows me to check or uncheck in Datasheet view or form created from this query<<

An APPEND Query cannot be used as RecordSource of the Form!

 

If you actually meant a SELECT Query, you can add a Boolean Field as a Calculated Field in the selection list but this Calculated Field is not updatable.

 

Basically, you need this Field in a Table if you want to be able to check/uncheck individual rows/records in a Form in DatasheetView or CtsFormView.

 

ken sheridan say: An alternative solution, requiring no Boolean column to be added to the Souls table, would be to use an unbound dialogue form with a multi-select list box control whose RowSource property is your existing query.  You can then select a few or as many rows in the list box as required.  The control's bound column should be the primary key ID column.

You might also want an unbound text box control in which the date to be inserted into the CommunionDate column can be entered, unless you want the current date inserted automatically.

Code in a button on the form would then be used to loop through the list box's ItemsSelected collection and at each iteration of the loop build and execute an SQL statement to insert a row into the communion table, e.g. assuming you have a list box lstSouls, and a text box txtCommunionDate into which the date is entered, or which simply defaults to the current date:

    Dim varItem As Variant
    Dim strSQL As String
    Dim ctrl As Control
    
    Set ctrl = Me.lstSouls
    
    If ctrl.ItemsSelected.Count > 0 Then
        For Each varItem In ctrl.ItemsSelected
            strSQL = "INSERT INTO Communion(SoulID, CommunionDate) " & _
                            "VALUES(" & ctrl.ItemData(varItem) & ",#" & _
                            Format(Me.txtCommunionDate,"yyyy-mm-dd") & "#)"
             CurrentDb.Execute strSQL, dbFailOnError
        Next varItem
    Else
        MsgBox "No communicants selected", vbInformation, "Warning"
    End If

d.      Click Disable all in the bottom right and then click OK.


You will be prompted to reboot your computer. After the reboot, test Internet Explorer for connection. Depending on the result, go to the "The connection works" section or to the "The connection still does not work" section.


If the connection works, a third-party service or program may be conflicting with Internet Explorer. To identify the conflicting service or program, follow these steps:

a.       Click Start, click Run, type msconfig, and then click OK.

b.      Click the Services tab and check the box beside Hide all Microsoft services.

c.       Select half of the listed items, and then click OK.

d.      Restart the computer, and then test Internet Explorer.

e.      Repeat steps a through d until you identify the service that is conflicting with Internet Explorer. If you are not using the conflicting service, we recommend that you remove it, or configure it so that it doesn't start when the computer starts.


If you cannot find a service that is causing the issue then open System Configuration agan, click on the Startup tab and repeat steps C thru e.

c.       Click the Startup tab, enable all the programs, and then click OK.

vinod archak say:

Hi Gibson,

 

You are right, the option to "Add to Outlook" is not available when you click on a RSS feed on a webpage. The procedure to get the RSS feeds in your Outlook requires you to synchronize your Outlook to pick the feeds automatically after you have added.

 

In the second Screenshot that you have post, add the feed to a folder of your choice or create a new folder by using the "New Folder" option. Follow the below steps to synchronize your Outlook to receive the feeds.

 

  1. Open Outlook.
  2. Click menu File. 
  3. Click Option from the left panel.
  4. Click on Advanced from the left panel.
  5. Scroll to the RSS feed section.
  6. Check the option "Synchronize RSS feeds"

 

After this whenever you add any feed on an webpage they will automatically start appearing under the RSS feeds folder in your Outlook.
