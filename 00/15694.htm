macropod say:

Documents with corrupt tables can often be 'repaired' by converting the tables to text and back again or by saving the document in RTF format, closing the document then re-opening it and re-saving in the doc(x) format.

 

An alternative approach is to export just the tables to a RTF document, then reimport them. The following macro automates that process.

 

Sub TableRepair()
'Macro to repair damaged tables by saving each table in an RTF-format file, then
' copying & pasting the table from the RTF-format file back to the source document.
Application.ScreenUpdating = False
Dim Rng As Range, i As Long, RTFDoc As Document
With ActiveDocument
  For i = .Tables.Count To 1 Step -1
    MsgBox .Tables(i).Style
    Set Rng = .Tables(i).Range
    Rng.Cut
    Set RTFDoc = Documents.Add(Visible:=False)
    With RTFDoc
      .Range.Paste
      .SaveAs2 FileName:="RTFDoc.RTF", Fileformat:=wdFormatRTF, AddToRecentFiles:=False
      .Close
    End With
    Set RTFDoc = Documents.Open(FileName:="RTFDoc.RTF", AddToRecentFiles:=False, Visible:=False)
    With RTFDoc
      .Tables(1).Range.Copy
      .Close
    End With
    Rng.Paste
  Next
End With
Kill "RTFDoc.RTF"
Application.ScreenUpdating = True
End Sub

a. user say:

FYI

I finally had a chance to research the issue I had with comments disappearing when copied and pasted from an email and found the issue is with the actual WORD "Sent".

So for example if I were to copy and paste the following into a comment section in planner

From: Microsoft Community
Sent: Friday, September 21, 2018 2:00 AM
To: John Doe
Subject: Question has a reply: Microsoft Planner - Comments Disappearing

The entire comment would disappear after closing and re-opening.

However, don't ask me why, but by simply deleting the word "Sent"  (or even changing the spelling), there are no issues.

This work-around is suitable for me but just wanted to let the community know in case somebody runs into the same issue.
