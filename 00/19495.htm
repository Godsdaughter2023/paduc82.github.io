Formatting text box for captions - Microsoft Community

yellowmellow graphics say:

Formatting text box for captions

Hi
I am creating a template for a client. I have formatted the font for the captions and that works fine. But I would like the caption text (figure 1,2 .. etc) to appear in a text box with no fill. As default it comes with a white fill. I have tried changing the style/ border/ shading - no fill, without any luck. The next box comes with white fill again. 
Any suggestions?
Rikke

john mcghie say:
Hi Rikke:

If you apply a caption to a floating graphic, by selecting the graphic and then using Insert>Caption, Word will automatically place the caption in a floating text box. The formatting of such a text box is not adjustable: it will always come in with a white background fill.

This is a pain in the rectangle, so I advise people not to use this "function".

However, if you click after the picture and insert a blank paragraph, then use Insert>Caption, the caption will NOT be inserted in a text box.  You can then use the paragraph style properties to format the position, font, border, and fill for the paragraph.

You could even insert your own text box, and drag the caption into that if you wished.  If you did, you could right-click on your own Text Box and choose "Set as default".  That way, the next text boxes you inserted would be formatted the way you want them.

However, that setting is stored locally on your machine: when you sent the template to the clients, they would each get whatever formatting the individual user had set as their default.  Which is not a saleable solution.

Text boxes do not have a style.  So while you can format the Caption style to change the text within the text box, the style has no effect on the properties of the text box itself: fill, colour, line weight etc.

You could work around this by using a single-cell TABLE instead of a text box.  A Table can have a style, and different tables within the document can have different Table styles.  But trying to teach end users to use single-cell tables instead of the automatic Insert>Caption command is a lost cause.  I wouldn't even attempt it: the results will be wildly inconsistent, and most users will simply ignore it.

There are five types of Styles in Word:  Character, Paragraph, Table, List and Theme.  Respectively they set the properties of the Font; the Paragraph including its font; the Table including rows, columns, cells, and fills and lines and borders; the formatting of Bullets and Numbering paragraphs; and the Document fonts and colours.

It's a massively complex subject which is increasingly falling in to disuse because there is no possibility of teaching today's younger users what they need to know to use these facilities correctly.

You can't really go any further than providing sample documents to the clients and telling them to always begin a new document by copying the sample and typing over your sample text.  Warn the clients to ensure that the samples are read-only to everyone, or someone will save their current project over the template within a week.  And warn the client that they will have to enforce the rule to "Always start with a copy of the sample document, never copy an existing document" otherwise the formatting of the clients documents will become close to random in less than a year.

Sorry, but Word on the Mac really does not have the functionality to lock down templates sufficiently to make this work reliably in the real world at the client's premises.

Hope this helps

don varnau say:

my google chrome was deleted and replaced with Microsoft product...can I get my previous favorites?

Chrome was probably not deleted, but you may have to set it as the default browser again.

Start> Control Panel> (Programs)> Default Programs> set Program access & computer defaults> Custom section> click the down arrows> set Chrome as the default browser.

If you've lost the shortcut that used to open Chrome, navigate to

"C:\Program Files (x86)\Google\Chrome\Application" (64-bit system)

or

"C:\Program Files\Google\Chrome\Application" (32-bit system)

Right-click Chrome.exe and select Send to> Desktop (create shortcut) You can create additional shortcuts in this way for the Star Menu, Taskbar, etc.
