a. user say:

You need to tell us some more details...

Unless your system is afflicted, if you boot XP normally and rename your win32k.sys file to win32k.old Windows File Protection will quickly and silently replace the "missing" file and you would see a message like this in the Event Viewer System log:

Event Type:    Information
Event Source:    Windows File Protection
Event Category:    None
Event ID:    64002
Description:
File replacement was attempted on the protected system file c:\windows\system32\win32k.sys. This file was restored to the original version to maintain system stability. The file version of the system file is 5.1.2600.6257.

So, it would take some extra effort for the file to really be missing - XP will always try to replace it if it comes up missing (if it can).

If you boot into Safe Mode or the XP Recovery Console, you can rename the win32k.sys file and it will not be replaced automatically since Windows File Protection would not be running under those circumstances.

Then if you reboot and there is no win32k.sys file you would see a message like this:

STOP: c000021a {Fatal System Error}
The Session Manager Initialization system process terminated unexpectedly with a status of 0xc0000034 (0x00000000 0x00000000).
The system has been shut down.

Note:  If the win32k.sys is corrupted, XP will look like it is starting to load and then you will see an error like this:

STOP: c000021a {Fatal System Error}
The Session Manager Initialization system process terminated unexpectedly with a status of 0xc000026c (0x00000000 0x00000000).
The system has been shut down.
