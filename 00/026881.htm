charleskenyon (ia) say:

***

I've found the page numbering feature of various editions of Word to be plagued with this sort of problem.  Thanks!

Closing note...

This frustration is not your fault, quite. 

  • Page numbers are fields. { Page } that show the "formatted page number" on any page and in headers and footers.
  • The usual place for page numbers is in headers and footers (even when they are in the side margins). That is because headers and footers are well designed to repeat things on multiple pages.
  • Headers and footers are components of sections.
  • Each section has three headers and three footers whether or not you see them.
  • Each of those six components of a section can be linked to (repeat) the like header/footer in a previous section, or not.
  • Every page can have multiple sections. Word creates new sections when you change margins, columns or orientation in the middle of  document.
  • Word will let you restart page numbering at the beginning of any section. If you do this, and then create a new section in one that you restarted numbering, it will restart numbering by default in the new section as well.

This is how page numbers have worked as long as I've been working with Word. This all tends to work very well for most users most of the time. It is when you try to do something a little different without understanding these principles that you (or I) have trouble, that Word page numbers seem plagued with problems. On these forums we tend to see those situations.

Microsoft dresses everything up in a "fluent interface" and tries to hide the underlying structure as well as it can. Again, most users seem to get by very well without such an understanding. However, if you regularly use large multi-section documents, you may want to learn how sections, headers/footers and page numbers interact.

kosh vorlon say:

Hi Dee,

 

Here's a couple of things about scheduled scans. To run, the PC must be on and connected to the internet (it cannot be shutdown, in sleep mode, in hybrid sleep mode, or in hibernation). The power saving settings must be such that the PC does not shut down or go asleep during the scan (or it will be as if it never happened). Scans do not begin precisely when scheduled - some can start an hour or so earlier or later than the officially set time (depending on Microsoft's servers and the loads at that time). There is a way to use Task Scheduler rather than MSE to awaken sleeping computers, run the scans, and then shut them down when done - or run various types of scans at different times (say a Quick scan daily and a full scan weekly) or several other alternatives, but it is somewhat complex. More on that shortly.

 

Also, go to settings and check scheduled scans and uncheck the box that prevents them from starting unless the system is on and idle (if you're using the system when it's scheduled or it otherwise isn't idle - perhaps downloading Windows Updates or doing something else - then the scan will not start and will be missed). If a scan is missed it does not start at the next available time, but waits for the next scheduled scan.

 

If you schedule the primary scheduled scan to be a daily quick scan at times when you tend to be using the computer, then even if you have it off for a day or two, when you come back and turn it on, then provided you have it scheduled for a time after you turned on the computer, you'll get a quick scan run that day.  If you want to use full scans and find this too intrusive for too long every day, then perhaps you can think of this as a backup to a scheduled weekly scan (we'll talk more about having both shortly).

 

There is a way to use Task Scheduler rather than MSE to awaken sleeping computers, run the scans, and then shut them down when done, but it is somewhat complex. You can also use it to create different scheduled scans of different types since MSE only allows one choice (so you can have daily quick scans and full scans every week or every two weeks).

 

In essence, if you wish to do weekly scans and don't want to consider daily scans and can't predict when the computer will be on or shutdown, then perhaps you may want to consider using sleep or hibernation rather than shutdown the day or two before and the methods described below for separate scheduled tasks in Task Scheduler (bypassing MSE's more limited scheduled scan feature altogether, if you find this to be a more favorable option in your situation). 

 

Please see http://answers.microsoft.com/en-us/protect/forum/protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02 - especially the later corrections from GreginMich.

 

The post from GreginMich here http://answers.microsoft.com/en-us/protect/forum/protect_start/scheduled-scans-not-working/fd769043-f67e-4990-8805-a02427f22dff will also be useful.
