6.  Live titles appear to cache data but eventually this data gets old and when there is no Internet Connection it shouldn't be showing data from days ago.  Weather for example isn't valid for prior days or last week's news.  If there is no internet connection then a message stating so should be seen.

8.  If we backup a PC and Restore it, the Start Menu doesn't work or crashes, etc.  We have to delete the user accounts and rebuilt them.  The Backup/Restore works on prior Windows just fine and we wrote it.   We use the backupread/backupwrite API's and these do not seem to properly restore the Windows Store, Apps, and user accounts any longer.

9.  Troubleshooting the Start Menu or Apps not working is virtually impossible.  Logs show no real or meaningful information and frankly speaking, just closing a Window with NO feedback is very VERY poor and leaves you wondering **** just happened.

10.  Some users complain the Start Menu opens and closes, opens and closes, and eventually works.  When we try to reproduce it, we cannot not.  Nor can the user reproduce it.  It does it randomly or when something starts etc.  No information anywhere and nothing in the logs.

11.  Some machines take a very long time to login and show the Start Menu.  Also, the default pinned applications slowly appear.  If you click on the Start Menu and a Pinned Application populates to the task bar then the Start Menu closes.

12.  Users that launch and stay in one application have less issues, or complaints.  However, employee's that launch and are in and out of a lot of applications complain they cannot find things or the Start Menu is sluggish and/or stops working for them.  Many users, after several days, have either built folders on the desktop or pinned applications to avoid using the Start Menu.

13.  Several users downloaded and installed an alternative Start Menu.  However, we have since blocked internet access to those websites and informed them that this defeated the purpose of our testing.  No harm done, but many requested they be removed from the testing as they found the experience worse than expected and it affected productivity.

14.  Searching has mixed results. We have other issues, but these are just a list of issues with the Start Menu that we have experienced still to this day in Build 10166.

  a.  Heavier loads seem to cause DCOM errors appearing a lot in the logs.  DCOM seems to break and foobar's the Start Menu.  If  DCOM doesn't work under a load or gets corrupted in the registry (CLSID) then it breaks the menu or a Store App and that seems to crash the Start Menu.  Also, this makes since why the menu is not consistent because if it is relying on DCOM which is like CORBA then it will be sluggish just like CORBA and will experience issues when something gets out of sync.

Dynamic creation of controls in vba - Microsoft Community

delipaul say:

Dynamic creation of controls in vba

Hi,

 

I would like to create labels, textboxs, checkboxes dynamically in a form on a click of activex button in excel sheet. How can i do it? Where do i write the method for the same? Please help.

andreas killer say:
Use the Add method of the OLEObjects collection to create the ActiveX controls, this is easy. But you must create a class module with an event variable for each type of control.

Add this code to a class module and name it "MyCommandButton":

--- schnipp ---
Option Explicit

Public WithEvents CommandButton As MSForms.CommandButton

Private Sub CommandButton_Click()
  MsgBox "Hello World"
End Sub
--- schnapp ---

And add this code to the code module of a sheet:

--- schnipp ---
Option Explicit

Dim MyControls As Collection

Sub Test()
  Dim O As OLEObject
  If Not TypeOf Selection Is Range Then Exit Sub
  With Selection
    'Create the control over the selected cells
    Set O = Me.OLEObjects.Add("Forms.Commandbutton.1", _
      Left:=.Left, Top:=.Top, Width:=.Width, Height:=.Height)
  End With
  'Call Worksheet_Activate with OnTime, a direct call did not work!
  Application.OnTime Now, Me.CodeName & ".Worksheet_Activate"
End Sub

Private Sub Worksheet_Activate()
  Dim MCB As MyCommandButton
  Dim O As OLEObject
  'Create a collection for our events
  Set MyControls = New Collection
  For Each O In Me.OLEObjects
    'Create a class for this control
    Set MCB = New MyCommandButton
    'Set the reference to the control
    Set MCB.CommandButton = O.Object
    'Keep it alive in the collection
    MyControls.Add MCB
  Next
End Sub
--- schnapp ---

Select a range and execute Sub Test, select another range and execute Sub Test again. Then try click the buttons.

Andreas.
