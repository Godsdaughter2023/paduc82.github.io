Word 2011 Mac--Updating fields causes pictures to be copied and - Microsoft Community

cameronsh say:

Word 2011 Mac--Updating fields causes pictures to be copied and inserted just prior to their cross-references

Hi all! I appreciate any help on this. 

I am preparing a scientific manuscript in Word 2011 (Mac). In order to include high quality figures without blowing up my file size, I inserted all pictures as "Linked to file" without saving with the document. I have also used Word's cross-referencing system to automatically number and reference figures, tables, etc. 

When I update fields in the document (part of saving to PDF or formatting my references), some of the figures are copied multiple times and appear before their cross-reference in the text. The surrounding text is then converted to the caption style. This doesn't happen with all of the figures, and it happens to different figures each time. 

I have no idea how to fix this. Any thoughts?

Cheers

john mcghie say:

Hi Cameron:

Turn your paragraph marks and field shading on, so you can see what you are doing.

On the Word standard toolbar you will see the "Show all non-printing characters" button (¶).  Click it so that the paragraph marks and spaces appear in the document.

Go to Word>Preferences>View and turn "Field Shading" to "Always".

These two things enable you to see where the boundaries of the items are,

Now you can see the caption numbers have gone grey, showing that they are active fields.  If you select one and toggle the field codes you will see {  SEQ Figure \* ARABIC }.  "SEQ" is the type of field (a numbering field) and "Figure" makes this field a member of the "Figures" collection in the document.

When you insert a cross-reference, Word will retrieve the text of the entire paragraph that contains the SEQ field that's a member of the Figures collection.

Because you can now see where your paragraph marks are, you can now ensure that the caption and the picture are sitting in separate paragraphs.

If you right-click the picture and check that its Wrap Text property is set to  "Inline with text" the picture will now settle in to the paragraph that contains it.  If you want to have text flowing around the picture, you can leave the wrapping style the way it is; in which case a little Anchor symbol will appear at the top left corner of the paragraph to which the picture is anchored.

Either way, you need to ensure that there is a paragraph between the picture and the paragraph that is the caption: otherwise the cross-reference will suck the picture in along with the caption.

Hope this helps

cameronsh say:

Thanks John!

Your reply nearly got me the whole way. The final step was to ensure that the paragraph containing the figure was formatted with the "normal" style instead of the "caption" style. This had been mixed up in the process of deleting figures and replacing them with new ones while keeping the old caption in place.

You are amazing.

* Moved from Education/Forms

* Original title: sharepoint

tina chen msft say:

Hi PoojaSingh3,

 

When we add a document which has the same file name as an existing document in SharePoint library, it is expected the document cannot be added and there will be a notification "A file with this name already exists. Would you like to replace the existing one?" appearing. If we click Replace, the existing document will be replaced but the previous version cannot be saved.

 

We understand you want to keep older versions of the document, as workaround, you may try following:

 

1.Before adding a new version of a same document to the SharePoint library, rename the existing document by selecting the document and click Rename in the ribbon, rename the file name, such as FileName v1, FileName v2.

(using this way, there will be several different versions of a same document in the SharePoint library)

 

2.Use Version history feature of file. (using this way, there will be only one document showing in SharePoint library, you can use Version history to check other versions.)

When you want to update a document, update it by opening it via selecting it>Open in Word online or Open in Word. The new version will be saved. And you can check its previous version via click the 3 dots next to the document>Version history.

 

For detailed information, please refer to these articles:

View the version history of an item or file in a list or library

Enable and configure versioning for a list or library

 

Best Regards,

Tina
