a. user say:

Do NOT export to a csv file. It's completely unnecessary, and will quite probably lose data

 

On the XP machine, close Outlook. Copy the pst file onto whatever media you are using to transfer files.

The default location is here:

C:\Documents and Settings\User Profile\Local Settings\Application Data\Microsoft\Outlook\

 

Paste the pst file into your Documents folder  on the new machine. Do NOT paste the file into the default location.

 

You then have two choices. If the pst file is the new UNICODE format (i.e created with Outlook 2007/2010) then all you need to do is to add the pst file to your Outlook Profile and make it the default  delivery location, and ALL your Outlook data will miraculously appear! (Control Panel-Mail-Data Files-Add)

 

If it's an older ANSI format file (i.e. created in Outlook 2003 or prior) then create a new pst file in Outlook 2010, OPEN the old file in Outlook (File-Open-Outlook Data File) and copy the data you require from the old file to the new.

ken sheridan say: Why are you testing for a Null varItem?  If a row in a multi-select list box is selected it's varItem value will be zero or above, never Null.  Also, after a little debugging, it seems to be the case that a column cannot be Null, even where the underlying column position in the table is Null.  So IsNull will always return False.   Testing the length of the column's value for zero seems to work, however.

So, if it's merely the presence or absence of a value in the fifth column (the Column property is zero-based, so Column(4) is the fifth column) then you only need:

    If Len(.Column(4, varItem)) > 0 Then

As you've already referenced the control with With Me.ROC_LIST you don't need to reference it each time in the code which follows, so just the dot operator is sufficient.
Ken Sheridan, Stafford, England
