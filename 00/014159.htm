I don't know much about saving an InfoPath form to Excel, and I strongly doubt that this is even possible, but since InfoPath forms deliver XML, I guess that Excel is not the most easily available storage place for  IP form data in the first place.

If your goal is to make the IP form data available to a SharePoint list view, you can promote all the fields in your InfoPath form to columns in the SharePoint Forms library. Then you can run list view web parts or DVWPs on the library, using the data that is stored in the InfoPath xml documents in the forms library, since it is also stored as columns in the library.
