julie fka profjulie say: After you've set up an ActiveSync partnership and synced your data for the first time, this is how ActiveSync works:

If you update or add data to your windows mobile device but not your computer, the new or updated data will be synced to the computer

If you update or add data to your computer but not your windows mobile device, the new or updated data will be synced to the windows mobile device.

If you update a record (i.e., a specific contact, task, or appointment) on the windows mobile device AND the computer, ActiveSync will recognize that the data has been updated on both devices.  There is are settings in ActiveSync (the ones I think you are referring to) that you can set to help ActiveSync resolve the conflict).  Your choices are to

  • always replace the item on the computer:  in this case the data for that unique contact, appointment or task on the windows mobile device will prevail and the data on the computer will be overwritten
  • always replace the item on the windows mobile device:  in this case the data for that unique contact, appointment or tasks on the computer will prevail and the data on the windows mobile device will be overwritten
  • leave the item unresolved:  in this case, ActiveSync will sync all data except the unique contact, appointment or task data that has been changed on both the windows mobile device and computer and will prompt you to specify how you wish to proceed in regards to that data.
ActiveSync is not a perfect application, but this part of it works pretty well.
