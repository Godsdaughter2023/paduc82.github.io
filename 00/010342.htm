suzanne s. barnhill say:

Yes, it deals also with images appearing as part of cross-references. The problem is that, when you create a cross-reference, Word inserts a  hidden bookmark around the referenced text. If you later press Enter at the beginning or end of that text and insert an image, it is inserted inside the bookmark and becomes part of the referenced text, so your cross-reference includes it.

To be on the safe side, you need to press Enter at the end of the paragraph preceding the bookmarked text (heading, caption, whatever) or at the beginning of the following paragraph, as appropriate. Once the cross-reference has become "spoiled," it can be corrected only by reinserting it. You can select just the text you want to reference, go to the Bookmark dialog, check the box for "Hidden bookmarks," and select the radio button for "Location." The bookmark that is selected will be the one for your heading or caption. Click Add again to reinsert it around just the desired text.

rrdodge say:

As to using the address function within the indirect function, you can do the following:

=INDIRECT(ADDRESS(ROW()-9,COLUMN()-5))

I use the combination of MATCH, ADDRESS, and INDIRECT for many table type lookups for long-term type solutions as opposed to using pivot tables.  Given issues I have had using pivot tables (they are very restrictive with certain things, thus prevent me from doing them), I think of pivot tables more so as quick and dirty type use, as opposed to using the combination of INDIRECT, ADDRESS, and MATCH for long-term type solutions.  The latter takes more time to set up and they are more fixed, but it has also allowed me to do things that pivot tables haven't allowed me to do.
