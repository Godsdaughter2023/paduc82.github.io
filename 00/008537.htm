swischner say:
Neftali is correct except for the last step.

Also or his instructions you absolutely need OneNote 2016 from the Office package; there is no export function in the OneNote app for Windows 10.

Somehow I am assuming that you are using that Windows 10 app instead of OneNote 2016, because OneNote 2016 lets you assign/add as many Microsoft accounts as you want and even open notebooks from all of them in parallel.

After having created the archive file (ONEPKG) you double click that in the Explorer. You will get asked for a name of a new (locally stored!) notebook. You may use the same name as the original one. When the import in that new local notebook is done, you then use the "Share" function to move that new local notebook to OneDrive again.

The other method without the detour via a ONEPKG file would be to create a new, empty notebook at account #2, open the original notebook from account #1 and use the context menu to copy (or move, but starting with copy is more secure) all sections one by one from the original to the new notebook. If you are not able to add two MS accounts (can't test right now, I'm not sure if there is a limitation in the Win 10 app) you could share your notebook from account #1 with account #2 first. That way you'd only have to login with account #2 to access the notebook from #1.

I'm sorry, but there is no easy way to just move a notebook from one MS account to a different one. Microsoft should (and I am quite sure they will at some point) add a feature for that.
