I would strongly advise against this. Word has a tendency to corrupt documents that contain large numbers of section breaks.

If your purpose is to have a different header and/or footer on each page, there are ways to do that without section breaks. If the page text contains a phrase that should be repeated in the header or footer, apply a unique style to each of those phrases, and then insert a StyleRef field in the header or footer.

If you have some other reason for wanting so many section breaks, please explain, and we'll try to propose a better alternative.

jay freedman say:

Can I do that without use continuous section break?

As I wrote before, if the information (or at least the part of it that changes) exists somewhere on the page, then a style should be applied to the text and a matching StyleRef field should be placed in the footer.

If the information doesn't appear on the page, a better solution is to insert a text box while the cursor is in the page text (not in the footer), and drag the text box to position it in the footer area. You can set the box's Shape Outline to "No Outline" and set its Shape Fill to "No fill" so that the text in the box appears to be part of the footer. To ensure that it stays in the footer area, click the Position button on the Drawing Tools tab of the ribbon, click "More Layout Options" to open the Layout dialog, and set the vertical position to 100% relative to Margin (you may need to adjust the number if the text has to align with other text in the real footer).

hansv mvp say:

If you want to convert the text field "in place", i.e. keeping it as a text field, run an update query on the table that updates the field to

Format([FieldName]/100,"0.00")

where FieldName is the name of the text field.

On the other hand, if you want to convert the text field to a number field, add a new field to the table in design view, set its Data Type to Number and its Field Size to Double (or Single).

Run an update query on the table that updates the new field to

[FieldName]/100

where FieldName is the name of the text field.
