It's impossible for a person to perceive a 4ms delay. So if you're talking about 8ms vs 4ms or even less, it's negligible and therefore doesn't impact the experience whatsoever.

 

Input lag on non-CRT displays is usually somewhere between 15-250ms depending post-processing options and the speed of the video processor. patrick barker say: Hi,

The attached DMP files are of the MEMORY_MANAGEMENT (1a) bugcheck.

This indicates that a severe memory management error occurred.

BugCheck 1A, {5003, fffff70001080000, c14e, 21975209}

The 1st parameter of the bugcheck is 5003 which means that the working set free list is corrupt. This is probably a hardware error.

I ran memtest for 8 passes with both 4 gig sticks in their correct slots 2 and 4.

^^ Did Memtest go ~8 passes with both sticks in their advised slots according to the manual without errors? If so, ensure the rest of your motherboard DIMM slots are functioning properly by testing 1 stick at a time.

For example, test both sticks in slot 2. If they both go at least 8 passes without errors, move on to doing the same for slot 4.

If Memtest rules out okay in all ways, ensure your memory is set correctly in the BIOS. Try the memory settings on Auto. If it's already on Auto, try swapping to Manual settings and manually inputting the factory advised timings... but this time, bump the DRAM voltage just a tad higher. For example, if your memory is factory advised for 1.5v, try 1.55v.

If you do extensive testing of the memory and it's passing with flying colors, I suspect a possible faulty video card as the earliest dump is of a different bugcheck-

PAGE_FAULT_IN_NONPAGED_AREA (50)

This indicates that invalid system memory has been referenced.

Usual causes are a bug in a device driver, hardware related memory issues, corrupt NTFS volume, anti-virus software.

In the call stack for this *50 bugcheck, there are various DirectX Kernel routines being called, which then eventually go directly into the bugcheck.

I would recommend if the motherboard you have has access to integrated video removing your video card drivers, shutting down, removing video card itself, enable integrated video via BIOS, install integrated video drivers, use system as you would regularly.

If no integrated video available, you can settle for running video diagnostics such as Furmark -

Furmark:

FurMark Video Stress Test - free from here:  http://www.ozone3d.net/benchmarks/fur/

FurMark Setup:

- If you have more than one GPU, select Multi-GPU during setup

- In the Run mode box, select "Stability Test" and "Log GPU Temperature"
Click "Go" to start the test  (Looks like it's "BURN-IN test" now)

- Run the test until the GPU temperature maxes out - or until you start having problems (whichever comes first).

- Click "Quit" to exit
