kbp45

considerate_guy say:

There are a couple possibilities:

1) You could have a messed up installation of Windows Live Movie Maker which would require an uninstallation/reinstallation of WLMM.  The latest version is 15.4.3555.0308, so even if this isn't the cause, you may want to install the latest if you are on an earlier version; check the File menu (Alt+F) and then go to About Movie Maker...

2) It could just be a misunderstanding of how transitions work.  A transition happens between two clips. For example, as one video or picture fades out, another fades in.  That would be an example of a Crossfade transition.  If instead you are right at the beginning of your project on the first item, you can't place a transition because you aren't between two clips; there would be no clip to transition from* to display the first clip.  Try selecting the second clip in your project and you should then see the full list of transitions on the Animations tab.

*more accurately, there are a few animations that are designed to work with the first clip (e.g. blur through black, crossfade, pixelate, flip, etc.) but the majority need to transition from a different clip to work. So to see the full set of available animations, place the playback between two clips, not at the beginning of your project.

      Vista backup: access denied 0x80070005 - Microsoft Community

      philip herlihy say:

      Vista backup: access denied 0x80070005

      This post is simply to record that I've found (my) solution to this annoying problem.  Thought it was time to run a new baseline backup, but it failed "Access Denied  0x80070005" after several hours.  Rebooted the PC, same again.  Very cross that the error messge offered no clue about which file or folder (read or write?) was causing the error.

      philip herlihy say:

      SOLVED, in my case (your mileage may vary).

       

      I checked the History tab in my Antivirus, Microsoft Security Essentials.  Choosing to view "all detected items", I saw two malware detections had been quarantined, with date/time corresponding with the failure of the two backup runs.  The cause of the problem is that Security Essentials detected malware in a file, and locked it.  Windows Backup doesn't know how to cope with this, so bombs out (could do better here, Microsoft!).

       

      Solution: I ran a full scan of Security Essentials, and Removed all the detected items.  Backup subsequently ran to completion.  The full scan is recommended, as there may be several items which will trigger a lockout in Security Essentials, and you don't want to find them one backup attempt at a time!

       

      Hope this helps.
