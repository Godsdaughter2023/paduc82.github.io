Reinstall Microsoft XPS Document Writer in Windows 7/8

The Microsoft Document Writer comes pre-installed on Windows 7/8 machines. If you've deleted the Microsoft XPS Document Writer and need to reinstall it, follow the steps below.

Detailed Instructions

To reinstall the Microsoft Document Writer, perform these steps:

  1. Click Start and select Devices and Printers.
  2. Click  Add Printer.
  3. Choose Add a Local Printer.
  4. Select Use an Existing Port.
  5. Select XPS Port and click Next.
  6. In the manufacturer list, choose Microsoft.
  7. On the right side, choose the latest version of the XPS document writer and click Next.
  8. Choose the driver currently installed, click next, and click finish. 

Note: If the XPS Port is not available, perform the following steps:

  1. Click Add a Printer.
  2. Choose Add a Local Printer.
  3. Select Create a New Port.
  4. Select Local Port. then click Next.
  5. In the Enter a Port Name field, type in XPS Port, then click OK.
  6. In the Manufacturer list, choose Microsoft.
  7. On the right side, choose the latest version of the XPS Document Writer, then click Next.
  8. Select Use the Driver that is Currently InstalledClick Next.
  9. Printer name should be Microsoft XPS Document Writer, then click Next.
  10. Select the option Do Not Share Printer, then click Next.
  11. Uncheck the box for Set As Default Printer, then click Finish
If none of that works:

You can go to MS Downloads and get the XML Paper Specification Essentials Pack and install it.  That should get the XPS printer back.

Force-quit Word (you will have to, it has stopped running), then re-start it and cross your fingers.  If Word crashed and managed to detect that, it will bring back the recovered file for you. Usually, it doesn't.

If it doesn't bring it back, try re-opening the file from your disk.  If you can't, sorry, it's gone.

Go to the Word menu and Choose "About Word".

That tells you your update level.  You should see "Latest Installed Update14.3.9".

If you do not, I think you have been bitten by a bug that was cured in an update about two years ago.

If that's the case, the code in the file is destroyed (all the text has turned to what you see on the screen, there's nothing left) so you need to restore from your backup.

If you do not have Time Machine running, you have just learned the most painful lesson of all, about why we should never do computing that matters without a backup.

You're welcome.  I forgot to mention....

If ISNUMBER demonstrates that some/all of the cells contain text that look like numbers, try to use Data > Text To Columns to convert them to numeric data (again, verified with ISNUMBER).

=TRIM(SUBSTITUTE(A1, CHAR(160), ""))
