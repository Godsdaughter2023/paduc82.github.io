    Server Response: -ERR [IN-USE] account is locked by another session or for maintenance, try again.


    There can only be one connection to a POP3 mailbox at once. If you're regularly seeing this message, it would indicate that you're also accessing the account by some other means, e.g. from a smart phone or tablet computer. If you are, try disabling the mail app on the other device for a day or two, then see whether mail starts arriving in Windows Live Mail.

    You should be able to connect from different devices via POP3. Just set the polling intervals to a different (prime) number of minutes (e.g. 13 and 17) to avoid as many collisions as possible.

    lemp say:

    Because you already know it's an *.ini file, you don't have to disable hiding extensions, but for future reference:

    *.ini files are configuration files for programs, although modern Windows programs no longer use this method.  Your guess is correct:  most modern Windows programs include this information in the registry.

    What this means is that it not the AutoIP.ini file in your documents that you're seeing in Task Manager.  If you see something called AutoIP in the "Applications" tab but not in the "Processes" tab, that means that there is some program running that uses "AutoIP" as its "friendly" name but the actual program's executable file is named something else.
