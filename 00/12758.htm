    MS Word Starter, omission of character - Microsoft Community

    snails_232 say:

    MS Word Starter, omission of character

    In the Symbols drop-down box in Word Starter, there are hundreds of alphabetical or written symbols including almost all the ones used in French. But one of the most frequent, the letter a-grave ('a' with a backward-sloping accent above it) is missing. This is very frustrating as I often use French words and phrases. Can I get this letter added, or is there a quick way of getting it via the keyboard?

    jay freedman say:
    I don't have Word Starter to check this, but there should be several ways to get that character. All of these work in the standard editions of Word:
    • In the Symbols dialog, after setting the Font box to "(normal text)" or almost any of the non-symbol fonts and setting the "from" box in the bottom right to "ASCII (decimal)", type 224 in the Character Code box. You should see the à selected, between the ß and á characters.
    • Without using the Symbols dialog, if you have a numeric keypad you can hold down the Alt key while typing 0224 on the keypad; the character will appear when you release the Alt key. (The zero at the start of the number is required.) This method works in all Windows programs, including browsers, but it's a bit difficult on a laptop where you also have to hold the Fn key to get the numeric keypad that's embedded in the regular keys.
    • In Word, there is a built-in shortcut: Hold Ctrl while typing the backtick (`) which usually shares a key with the tilde (~); then release both keys and type an a. The Ctrl+` followed by another key works for all of the vowels. Similarly, Ctrl+' (using the apostrophe) adds the acute accent to vowels.
    • Last, and least convenient, type the four-hex-digit equivalent of the decimal character number -- in this case, 00E0 or 00e0 -- and then press Alt+X.

    Hi K,

     

    For Q1 and Q2, you’re correct that after Exchange hybrid is deployed, messages between on-premises Exchange servers and Exchange Online are treated as “internal" using forced TLS transport. Which means the message will bypass the anti-spam and other services. However, before message is delivered to the Exchange Online mailbox, it will be scanned for viruses first by EOP (Exchange Online Protection).

     

    For Q3, the topology you described is not officially support in an Exchange hybrid deployment. If we want the messages routed to Exchange Online are scanned by EOP, here are two ways we can use to achieve the target:

     

    1. After hybrid is deployed, point the MX record to EOP/Office 365, so that all the messages will be routed to EOP first for scan, then be delivered to on-premises and Exchange Online accordingly.
    2. Don’t deploy hybrid, instead, we can use the simple domain sharing feature as mentioned in: https://support.office.com/en-us/article/Pilot-Office-365-with-a-few-email-addresses-on-your-custom-domain-39cee536-6a03-40cf-b9c1-f301bb6001d7, however, this is not recommended consider that you have so many mailboxes hosted in on-premises servers.

To sideload an app or game, you'll need to have an SD card that contains one or more Windows Phone app files, also known as .XAP files. You can download .XAP files to your PC, then move the files to the phone's SD card.

    Tip

    If you're running Windows Phone 8.1 and your phone supports an SD card, you can install apps directly to your SD card instead of your phone's internal storage. These apps will show up in the App list just like your other apps and games. To learn more, see Using an SD card to add space.   

    To download .XAP files from the Windows Phone Store on the web

    1. Open a web browser and go to www.windowsphone.com.

    2. Click Apps + Games, then click the app you want to download.

    3. Scroll down on the web page, then click Download and install manually (below the app requirements and supported languages).

    4. When prompted, save the .XAP file to a location on your computer, SD card or storage device.

    Note

    If you download the .XAP file to a location other than an SD card supported by your phone, you'll need to move or copy the file to an SD card to install it on your phone.
