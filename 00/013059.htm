    handetech say:

    Answering an old post I know but I came across this issue the other day whilst setting up a new IMAP email account with Outlook 2016.

    Some details re IMAP email account:-

    • https domain is hosted on a shared server with a web hosting provider
    • Logged into the web host via cPanel and created the new IMAP email account
    • Opened Outlook, clicked Add Account and entered email username and password
    • Allowed Outlook to create the account automatically (FWIW I normally manually setup email accounts)
    • Clicked Yes to the certificate warning thinking this would be a once off operation

    Afterwards I continually received the certificate warning and so a web search of the issue was made and I found my way here.

    I too also looked at editing the Registry as per the link within this thread however thought to myself, why edit the Registry when the alert is an advisory warning?

    The warning, to me, was a hint that the email account settings were not quite right. Another hint was that I'd allowed Outlook to auto-configure the settings for this particular account and not used the settings as provided by my web host for email setup.

    So, in the end the fix was to:-

    • In Account Settings/More settings/Advanced tab, ensured that the IMAP and SMTP ports were correct. IMAP 993 and SMTP 465
    • In Account Settings/More Settings/Advanced tab, ensured that the encryption was correct. In my case set to SSL/TLS
    • REMOVED the word "mail" and the "full stop" from the front of the domain name used for incoming and outgoing server settings, i.e. instead of mail.example.com it needed to be just example.com as per my web host settings for IMAP (incoming mail server) and SMTP (outgoing mail server).

    I've not had the warning popup at all since.

    patentssb say:

    OK, I have figured it out.  Doug mentioned styles, so I started there.  Here is how I solved it:

    Each paragraph was already set to the "list paragraph" style.  Even though each paragraph was the same style, the numbering was off (usually due to closing, opening, and inserting a new paragraph in the middle of the document).

    In the styles pop-out menu (on the home ribbon), I went down to "list paragraph," clicked on its drop-down menu, and chose "select all."  Then I scrolled to make sure every numbered paragraph was highlighted.  Then I went back to the list paragraph drop-down and selected "update list paragraph to match selection."  Boom, it "joined" all the errant lists to make one, continuous list.

    If there is a number that is off due to previously manually setting a number value, I can now just click on the number and select "continue numbering."  In one case after updating things got really wonky, and I had to go the first paragraph and set its number value as 1 and go to any following non-sequential number paragraphs and select "continue numbering."

    This will change my life.  Forever.  I can't wait to tell my coworkers.

    Hi Alex,

    The reason you have two OneDrive cloud icons in the notification area is because there are two OneDrive for Business application. One is the legacy sync client, the other is the next generation sync client.

    The legacy sync client supports both SharePoint library and your OneDrive for Business library while the next generation sync client only supports OneDrive for Business for now (SharePoint library support is coming).

    We recommend that you keep using the next generation sync client to sync your OneDrive for Business library, as it removes many of the restrictions of the legacy sync client, and use the legacy sync client to sync your SharePoint library for now.

    If you would like to know more about the sync client you are using, I have included an article for your reference:

    Which OneDrive sync client am I using?

    Considering requirement number 1: Have the files and folders visible in the file exlorer for all users, we'd suggest you keep using the SharePoint library that the local Microsoft partner has set up to collaborate and share files. Your OneDrive for Business library is meant for personal use and is only visible to yourself by default, while files in your SharePoint library will be visible to users who has the permission to the site.

    Your OneDrive for Business sync client that is linked to your SharePoint library should work just like any other cloud storage. You share a file by putting it in the synced folder, and this file will be synced to the cloud, and is replicated on any computer that is also syncing this SharePoint library.

    To learn more about when to save documents to either OneDrive for Business or SharePoint, you may refer to the article below:

    Should I save my documents to OneDrive for Business or a team site?


    For requirement number 2: Have a right-click shortcut to send to OneDrive, please refer to the link below for a guide on how to create a Send To shortcut:

    How to Add OneDrive to the Send To Context Menu in Windows 7 or 8.1.

    Note: The guide is for OneDrive in Windows 7 and 8.1, but it will work with OneDrive for Business and Windows 10 too.
