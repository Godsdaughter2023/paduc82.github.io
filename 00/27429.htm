I'm not sure I have enough information to be precise, but for starters I would save all answers in one table.

Assuming the pre-test questions and the post-test questions are the same, we need to support the notion that the same test is given to the same student multiple times (typically: twice). I'm thinking of a table TestInstances which gets a record each time a test (= a set of questions) is given:

TestInstanceID autonumber PK
TestDate datetime required
TestID long integer, FK, required
IsPreTest yesno, required, default=Yes, Comment=If False, it's a PostTest.

 

Each TestInstance record has a set of answers in the Answers table:
AnswerID autonumber PK
QuestionID long int, required, FK
TestInstanceID long int, required, FK
StudentID long int, required, FK
Answer text, required  (could also be a number 1-5)

ken sheridan say: In database terms a student's response to a test is a many-to-many relationship type between students, questions and answers.  So you'd need tables along these lines;

Tests
....TestID (PK)
....TestName
....etc

Students
....StudentID (PK)
....FirstName
....LastName
....etc

Questions
....QuestionID (PK)
....Question
....TestID (FK)

Answers
....AnswerID (PK)
....Answer

The relationship type between questions and answers would be modelled by a table:

QuestionAnswers
....QuestionID (FK)
....AnswerID (FK)

The primary key of this table is a composite one of both columns.

The relationship type between students, questions and answers is modelled by a table:

StudentResponses
....StudentID (FK)
....QuestionID (FK)
....AnswerID (FK)
....AnsweredWhen

The AsweredWhen column would have possible values Pre-program and Post-program.  This table is again all-key, i.e its primary key is a composite one of all four columns.

You'll appreciate that these are only preliminary thoughts and the model should be tested thoroughly and fine-tuned if necessary.  On the whole I think the structure is about right, though.

tom van stiphout say:

You can tell by the replies that as database designers we are much more interested in storing the data in an efficient way obeying all known business rules, than we are in answering questions such as "by how much did students do better after the test?".  This is because you can ask a good database design ANY logical question and there is a query that can answer that question. For example we can calculate the average score per question of the Pre-tests, and compare it to the average score per question of the Post-tests.

Using my design, the average score would be something like:

select QuestionID, Avg(Answer)
from Answers inner join TestInstances on Answers.TestInstanceID = TestInstances.TestInstanceID

where IsPreTest = True
group by QuestionID

diane poremsky mvp (slipstick.com) say:

There are two things at play here - I think its a windows bug, not Outlook (and its probably technically not a bug) - the problem is the space in the short date. You can only have 1 separator chosen for short dates - with the space you have two (space and /) - the first separator wins and is what Outlook uses as its separator for the format columns view.

The second thing is the column format. Check format columns (view setting) - with the standard short format, you can choose your desired format in outlook but not with the change in windows format, because outlook uses the default date separator set in windows. http://slipstick.me/jing/format_001.png -

Can you live with ddd/M/dd or something similar in windows? If so, you can get your format in Outlook. If you use m/d ddd it will also work in Outlook, since windows uses the first separator- / - as the default separator.

BTW - this exists in outlook 2007 /winxp - i'm pretty sure you'll find its a problem in every os and outlook combo.

Hi Manmohan M,

We can achieve it by using the Azure Rights Management in Office 365 admin center. Please note that not all Office 365 plans have Azure Rights Management, plans have this are for example E3 or E5 plan. So I would like to confirm what Office 365 plan you are using at the moment.

Please make sure that you have activated the Rights Management by the following steps:

1. Sign into Portal (https://portal.office.com) with your Office 365 admin account, Click Go to the old admin center, navigate to Admin – SERVICE SETTINGS – Rights Managements and activate it there.

2. Connect to Exchange Online Using Remote PowerShell.

3. Run the following command. Just select one based on your region: 

For North America:

Set-IRMConfiguration -RMSOnlineKeySharingLocation “https://sp-rms.na.aadrm.com/TenantManagement/ServicePartner.svc”

For European Union:

Set-IRMConfiguration –RMSOnlineKeySharingLocation “https://sp-rms.eu.aadrm.com/TenantManagement/ServicePartner.svc”

For Asia:

Set-IRMConfiguration –RMSOnlineKeySharingLocation “https://sp-rms.ap.aadrm.com/TenantManagement/ServicePartner.svc”

For South America:

Set-IRMConfiguration –RMSOnlineKeySharingLocation “https://sp-rms.sa.aadrm.com/TenantManagement/ServicePartner.svc”

4. Run the command:

Import-RMSTrustedPublishingDomain -RMSOnline -name "RMS Online"

5. Run the command and make sure the result is PASS:

Test-IRMConfiguration -RMSOnline

6. Run the command:

Set-IRMConfiguration -InternalLicensingEnabled $true

Please note: this action will need hours or a day to take effect. For your reference:

https://technet.microsoft.com/en-us/library/dn569291.aspx
