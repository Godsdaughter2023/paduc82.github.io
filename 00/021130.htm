    graham mayor say:

    I am unsure of your definition of 'second page'. The list numbering will span pages in the same document without restarting, unless you include { SET A {=( { DOCPROPERTY "LASTDOCUMENT"} + 1) } } again on the second page. Could that be the problem?

    If there is no LASTDOCUMENT property in the document, then the first number will be 0+1 i.e. 1

    It all revolves around the LASTDOCUMENT property. How do you get the property for the document, other than by checking the last used number in the previous document and adding it?

    You could add that number using a macro e.g.

    Sub AddLastNumProperty()
    Dim oSource As Document
    Dim oTarget As Document
    Dim Fld As Field, Rng As Range
    Dim iLast As Long

        Set oSource = ActiveDocument
        For Each Fld In oSource.Fields
            If Fld.Type = wdFieldListNum Then
                Set Rng = Fld.Code
                If InStr(Rng.Text, "CUSTOMLIST") Then _
                   iLast = Val(Rng.ListFormat.ListString)
            End If
        Next Fld
        Set oTarget = Documents.Add 'identify the next document to be processed - here creating a new
        'document, though I suppose you would be opening the document to be edited.
        With oTarget.CustomDocumentProperties
            .Add name:="LASTDOCUMENT", _
                 LinkToContent:=False, _
                 Type:=msoPropertyTypeNumber, _
                 Value:=iLast
        End With
    End Sub
