As much as you'd like to see it, I believe Doug has provided a realistic interpretation of what MS will do about the request. Nothing.

But there are lots of things you can do today.

.

First: when you see the substitution, you can use <CTL><Z> to undo the "improvement".

.

Second: it sounds like you are trying to setup a table as if you are using a "typewriter".  There are actually 2 fixes for that.  One is to use tabs to consistently space the "columns", ie

.

sfasfsaf<tab> - <tab>dfasfasf 

.

with or without the spaces around the hyphen, should get consistent location when the tabs are set to user specified locations, rather than depending on default tabs. That is closest to your old school approach.

.

The other way is to use a table, with borders not printing. You can use the tab to move from one cell to the next. If you insist, you can still use hyphens to visually delineate the columns, by creating a narrow column dedicated to hyphens.

.

Third: take a look at the Insert tab > Symbol group > Symbol drop down > More Symbols command > Symbol dialog box > Special Characters tab.

It appears to list most of the symbols you want. Some of them already have shortcuts defined. You can use the default shortcuts or define new ones. And you can define shortcuts for the ones that don't have defaults (see #1 in screen cap).

Fourth: For any that are missing, you can create macros to insert them, add the macros to the QAT and/or define shortcut keys for the macros.

Fifth: you can turn off autocorrect for the specific symbols in the symbol dialog.  With the symbol selected in the Symbol dialog, click on #2 / Autocorrect ... button.

Note that the hyphen is already loaded, just turn off the "Replace text as you type" option.

OK to save the change.
Repeat with other symbols.

Sixth: if this change is just for you, you can record macros inserting the special symbols, put the macros on your QAT and/or define shortcut keys for the macros.

Seventh: you can create a "global template" with a new tab containing the special chars to be inserted, ie via macro.

Eighth: if you are willing to do some programming, Dougs suggestion to create an "addin" to define a new tab containing all of the custom characters you want is the most realistic way you can get what you want in a short time frame.  OK doing some more research, it may not be so easy ...
