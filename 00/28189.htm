Hi Ivan

Please find in the link below a copy of your file with a VBA macro as solution to your request.

https://www.dropbox.com/s/muz5p01th20ipp9/Dummy%20data-Answer.xlsm?dl=0

1-Insert your criteria in cell A1

This is the macro

**********************************************************************

Sub TransferData()


Dim SearchRng, TrgRng As Range

Dim Criteria, SCriteria As String

Dim WSh, DSh As Worksheet

Dim StRow As Long


Set WSh = ThisWorkbook.Sheets("2019 Ward-Precinct-CCC-List")  '''Your Data sheet

Set DSh = ThisWorkbook.Sheets("Filtered-Data")  '''Your Destination sheet

Criteria = Trim(DSh.Range("A1").Value)  '''Your criteria search

StRow = 3   '''Starting row in Destination sheet

Application.ScreenUpdating = False


DSh.Activate

'''Delete old filtered  data

DSh.Range(Cells(3, "A"), Cells(Rows.Count, "E")).Clear


WSh.Activate

''''Set your range search

Set SearchRng = WSh.Range(Cells(1, "D"), Cells(Rows.Count, "D").End(xlUp))

'''Loop thru all cells in search range

For Each K In SearchRng

    SCriteria = Trim(K.Value)

     If SCriteria = Criteria Then

            WSh.Cells(K.Row, "B").Copy DSh.Cells(StRow, "A")   '''Transfers the Pct

            WSh.Cells(K.Row, "E").Copy DSh.Cells(StRow, "B")      ''Transfers the Name

            WSh.Cells(K.Row, "G").Copy DSh.Cells(StRow, "C")   '''Transfers the Phone

            WSh.Cells(K.Row, "I").Copy DSh.Cells(StRow, "D")   '''Transfers the Email

            WSh.Cells(K.Row, "H").Copy DSh.Cells(StRow, "E")   '''Transfers the Address

            StRow = StRow + 1

        End If


Next K


DSh.Activate

DSh.Range("A1").Select

Application.ScreenUpdating = True

MsgBox "Done"


End Sub

Migration of Shared Mailbox to 365 - Microsoft Community

won roh say:

Migration of Shared Mailbox to 365

We have large number of Shared Mailbox on Premises

I heard that shared mailbox on Azure cannot be access by on premises users then Can user on Azure access on premises shared mailboxes?

Should shared mailbox migrated last?

what will be best option for not interrupting share mailbox access?

Please advise.   Thanks for your help in an advance. 


alan wu msft say:

Hi Won,

Glad to reply to your case here. 

May I know whether you mean you want the on-premises shared mailbox to be accessed by the Office 365 user? If yes, then you need to migrated the on-premises shared mailbox to Office 365. After the shared mailbox is migrated to Office 365, you can assign the related permission (ie, full access, send as) of the migrated shared mailbox to the target Office 365 user (whom you want to access the shared mailbox). Please note that only the shared mailbox is migrated to Office 365, then the Office 365 user is able to access it after you assign the related permission. 

As for your mentioned "not interruppting shared mailbox access", do you mean the original on-premises user (who has the access to the shared mailbox before the shared mailbox is migrated)? If so, I'd like to say you need to re-assign the related permission after the migration. Please note that if you want the original on-premises users are still be able to access the migrated shared mailbox, you need to deploy a Hybrid mode. And for detailed permission (access) in Exchange Hybrid mode, you can refer to https://docs.microsoft.com/en-us/Exchange/permissions (Mailbox permissions supported in hybrid environments). 

Regards,

Alan
