=SUM(ROUND(L9,2)*V9)

May I know whether the receiving issue happen with every external senders or specific external sender?

If you using custom domain then make sure that whether you have added MX records. If not, please add an MX record so email for your domain will come to Office 365. The MX record for the domain points to the value that's listed in the DNS Settings section of the Office 365 portal for Exchange Online. 

The MX record you need to add includes a value (the Points to address value) that looks something like this: <MX token>.mail.protection.outlook.com. See: Create DNS records at any DNS hosting provider for Office 365

Please try to troubleshoot this issue via changing the custom domain to default domain i.e. "onmicrosoft.com" -based email address (<user>@<domain>.onmicrosoft.com) to see whether the problem still occurs. . (Note: You need to contact your office 365 admin to change your Office 365 email address in the form of <user>@<domain>.onmicrosoft.com.)

Moreover, to make sure the inbound mail flow working correctly from the Office 365 side. Please using the Remote Connectivity Analyzer to Inbound SMTP Email test, follow these steps below: 

1. Browse to the Microsoft Remote Connectivity Analyzer tool at the following website: https://testconnectivity.microsoft.com/
2. Select the Office 365 tab>under Internet Email Tests > run the test for Inbound SMTP Email. Complete all the required fields on the form, and then click Perform Test.
