stevencompton- [o365] say:

So I ended up having to call support this morning.  The issue is that in the cloud would not sync with my local directory.  Typically the error is caused when you create a contact in the cloud then try to create the same contact in your on-premises.  In my case, that was not the case.  My contacts were created in AD on-premises but I got the same type of error.  Not sure what happened to create the error environment.  To resolve, we had to force delete the contacts in Office 365.  We used the command via PowerShell.  For testing, we removed one contact (got the object ID first)

Remove-MsolContact -objectid acfa629b-00db-4f04-9fb1-0ec67114b6fa -force

That worked.  So, we ended up delete all of the contacts.  NOTE:  I had ALL of the contacts on my local AD so deleting them didn't mean anything to me because I simply was going to resync them afterward from my local AD.  If you have cloud contacts in addition to the local, I'm not sure you would run the command.  It just worked with me.

Get-MsolContact -MaxResults 1000 |remove-msolcontact -force

This took a while to finish the command but it removed the Office 365 contacts from the cloud.  Next, we did a resync and the contacts were recreated from my AD OU folder.  I didn't get the error this time because the imported contacts were new and the malformed contacts were gone from the cloud using the command above.  Anyhow, I'm happy this got resolved.
