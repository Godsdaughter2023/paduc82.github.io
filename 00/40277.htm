Has anyone else had blue screens when remote desktop to Windows XP SP3 - Microsoft Community

kimascott say:

Has anyone else had blue screens when remote desktop to Windows XP SP3 (Probably because of JAWS from Freedom Scientific)

Hello All,

This is mostly to document what I think the fix to the problem is, since I haven't found this exact issue listed anywhere else, but if someone has other comments/suggestions it may help someone. :)

When I would use Remote Desktop (RD) to connect to a Windows XP Service Pack 3 (sp3) computer from a Windows 7 64-bit Ultimate (sp1), I would be random Blue Screen Of Death (BSOD) errors upon logging off, disconnecting, or shutting down the RD connection.

Answer(?): Uninstall JAWS from Freedom Scientific; your problems may go away (mine did).

 

History/More Info for those that want it (or are searching on these terms):

 

Stage1: In the beginning...

I started with errors like the following:

0x00000050, parameter 1 0xBF44D216, parameter 2 0x00000008, parameter 3 0xBF44D216, parameter 4 0x00000000

PAGE_FAULT_IN_NONPAGED_AREA

 

Attempted Solution: I tried turning off options, decreasing resolutions, matching color depths (per a lengthy thread I found elsewhere). I tried loading windows hotfixes, and even the latest remote desktop client, on the Windows XP sp3 machine (which is a Sun box with 2 monitors attached). All that didn't help.

 

Stage 2: Get different BSOD, and the critical hint at the problem...

 

So then I carefully match resolutions on a non-widescreen monitor, match color depths, etc. The RD connection seemed to work OK, but still gave the BSOD when I logged off (or maybe tried to shut down the computer, I don't remember now ;-). So I got BSOD errors like the following:

 

Problem in file: fsvidrdp.dll

0x00000050, parameter 1 0xBF44D36C, parameter 2 0x00000008, parameter 3 0xBF44D36C, parameter 4 0x00000000

PAGE_FAULT_IN_NONPAGED_AREA

 

The fsvidrdp.dll was the clue. With a google search, I had a couple of hits on JAWS. I went to http://www.freedomscientific.com/fs_support/BulletinView.asp?QC=874 and it gave me the hint that maybe JAWS was having some troubles with remote desktop.

 

Well, I didn't need JAWS installed anymore on the Windows XP sp3 machine (it was there for some testing I did many moons ago), so I removed it.

 

Voila! The problem is gone; no more bsod's when using remote desktop into the windows xp machine. :)
