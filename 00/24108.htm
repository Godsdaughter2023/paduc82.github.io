    greginmich say:

    You can determine whether or not Windows Defender has actually been disabled in the Policies key by running this command at the Command Prompt (copy, paste, and enter):

    REG QUERY "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware

    A return value of 0x1 indicates that Windows Defender has in fact been disabled in this registry location. Direct editing of the registry key isn’t required in order to turn Windows Defender back on if it’s been disabled here. If you don’t suffer from command line phobia, then you might want to try this: Right-click on the Start button and select Command Prompt (Admin). Then copy, paste, and enter this command:

    REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware /f

    The alternative would be to run the Turn_On_Windows_Defender.reg file found here:

    http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html 

    This is a trusted file which simply deletes the DisableAntiSpyware DWORD in order to turn on Defender. This is the entire contents of the file:

    Windows Registry Editor Version 5.00

    ; Created by: Shawn Brink

    ; Tutorial: http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

    [HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows Defender]

    "DisableAntiSpyware"=-

    These trusted reg files are very easy to run, and are much safer than directly editing the registry – and are therefore the established protocol here in the Virus & Malware forum.
