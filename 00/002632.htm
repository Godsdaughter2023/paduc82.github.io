Finding "Quick Notes" in Adndroid-OneNote app - Microsoft Community

m. tamim say:

Finding "Quick Notes" in Adndroid-OneNote app

Dear Community, 

Suppose I compose a "Quick Note" in "Windows 10" using Windows-Key+N. We know that the "Quick Note" will be save in the "Quick Notes" at the left bottom corner in "OneNote 2016". 

The questions are: 

1. How to find this "Quick Notes" using Android-OneNote app. 

2. How to find this "Quick Notes" using in www.OneNote.com? i.e., in the cloud OneNote? 

Thanks a lot. 

bernd p. say:

An answer to your question depends on the history of your Onenote desktop installations.

Historically  (in older versions of Onenote) the Quick Notes started as Unfiled Notes and were stored locally on your computer. Upgrade installations didn't change that, for compatibility.

Then came the mobile Onenotes (and the UWP App from the Windows Store). They all create Quick Notes solely on Onedrive. If necessary they create a new "personal" notebook with changing names from version to version but always with a section named Quick Notes.

So there is some probability you have two places with Quick Notes.

How to find them:

Where Onenote 2016 stores them you can see in File, Options, Save & Backup:

If they are stored locally then you have to modify this pointer to the Quick Notes on Onedrive.

But how to find them on Onedrive ....

Open your browser and login to https://onedrive.live.com

If you have used a mobile client or the UWP App you should find a "personal" notebook within the folder Documents. Click it to open it in Onenote Online to see whether there is a Quick Notes section. When you have found it then click Edit in Onenote. The notebook should open in Onenote 2016.

Now you have both Quick Notes sections open in Onenote.

Then copy/move all your notes from the local Quick Notes to the Quick Notes on Onedrive.

The last step is to modify the Quick Notes pointer in Options (see above) to point to the Quick Notes on Onedrive.

If all is well, you can delete the old local Quick Notes section with Windows explorer.

Bernd

I lost some hours on debugging this recently and would like to ask if somebody else experienced the same bug with Outlook 2020/Outlook.com.

Previously on hotmail/live accounts you could use Oulook 2010 Hotmail Connector to synchronize mail/calendar/contacts from your Microsoft account. Now as DeltaSync is obsolete and account gets migrated to Outlook.com platform the Hotmail Connector is obsolete and does not sync any more. The proposed fix is to update Outlook 2010 and then just add a new account with auto-discovery (!! important, manual setup does not work !!). On Outlook 2010 it's supposed to use the full-blown Exchange connector and add your Microsoft Account as an exchange account.

This sounds all fine. But when I try to add a new account with a @outlook.de address the automatic detection goes wrong. First you enter a name, the mail-address and password and then on clicking the next button Outlook 2010 will search for the e-mail settings for pretty long time before popping up a window asking to install the Hotmail Connector (!!). I found a solution to this by adding an e-mail alias in the Microsoft Account with an @outlook.com address. Then using this @outlook.com address to add it in the Outlook 2010 automatic mail configuration dialog and immediately Outlook 2010 detects the exchange setup and synchronized mail, calendar and contacts. The drawback of this is of course that you need to add the @outlook.de mail as well as IMAP in order to be able to use your existing sender address.

I think it's a bug or wrong DNS/detection setup on outlook.de which makes it fail on detecting an exchange setup.

Anybody experiencing similar issues? Perhaps with other mail domains like @hotmail.de or Microsoft Accounts with completely different mail-addreses like @web.de, @gmx.*, @gmail.com etc.

As there is no possibility to configure the exchange account manually this is really an issue for users having non-outlook.com addresses.

vincentwu_msft say:

Hi Techsol,

Here is a simple script file for your reference, you may copy the following two lines to an empty new text file and save the file to .bat format.
\\server\odt\setup.exe /download \\server\odt\configuration.xml
\\server\odt\setup.exe /configure \\server\odt\configuration.xml

Note: \\server\odt refers to the UNC path(file share) where setup.exe and xml configuration file are located, replace it with the correct path according to your environment.
