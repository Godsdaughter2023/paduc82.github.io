jllatham say:

More detail may be needed - is the cell to be formatted on a different sheet from the Q & G columns sheet?

If they are on the same sheet, the you'd only need to format column G for the specific value of 1.

But if they are on different sheets, then there are a couple of things that could give you problems:

#1 with VLOOKUP() what you are looking up (value in column Q ?) must be in the 1st/leftmost column of the lookup table.  So you cannot do something like  =VLOOKUP(A5,Sheet2!Q1:Q1000,-10,False)  the -10 is an invalid parameter.

You can overcome this by using a plain VLOOKUP() but for that the entries to be looked up (in column Q) have to be in ascending order, otherwise the results are unreliable.

But to make sure of things, tell us:

The name of the sheet with the cells to be conditionally formatted:

Tell us what column those cells are in:

Tell us what column on that sheet the values to be looked up/matched are in:

Next tell us the name of the sheet with the lookup values are on:

What column the item to be matched is in (I assume column Q)

What column the item to be returned (1 or 0) is in (I assume G)

It may turn out that a form of the INDEX() and MATCH() functions combined may give you better results.

ken sheridan say: Environmental variables can be changed, and very easily with a modicum of knowledge.  It is not beyond the bounds of possibility that there is something at work which changes the username environmental variable to Betsy, though why this would have been done in the first place is difficult to imagine.

The short answer is do not use the environmental variable to get the currently logged in user, which I'd advise in any case due to the ease with which a malicious user can masquerade as another user by changing the value of the variable.  Instead call the Windows API GetUserName function via the following module:

' module basGetUser
Option Compare Database
Option Explicit

Declare Function GetUserName Lib "advapi32.dll" Alias "GetUserNameA" (ByVal _
   lpBuffer As String, nSize As Long) As Long

Public Function GetUser() As String

   Dim strBuffer As String
   Dim lngSize As Long, lngRetVal As Long
   
   lngSize = 199
   strBuffer = String$(200, 0)
   
   lngRetVal = GetUserName(strBuffer, lngSize)
   
   GetUser = Left$(strBuffer, lngSize - 1)

End Function

The currently logged in user's name can then be reliably obtained by calling the GetUser() function.

karl timmermans (outlook mvp 2012-2018) say:
Will answer your questions in a somewhat "combined" approach which hopefully will make things easier.

Re: Your Email Account

You only have ONE email account which is stored on the server. Makes no difference how that account is accessed or to put it another way, if something doesn't exist on the server, you are not going to be able to access it with any email client.

To take this one step further, the significant difference between POP3 and IMAP is that IMAP syncs all the email folders with whatever email client accesses it as an IMAP account. If the account was accessed as a POP3 account (regardless of email client used), only the <Inbox> is downloaded to the client and nothing is sync'd. Again to put this another way, when the account is accessed as a POP3 account, nothing that happens on the local computer/device gets sync'd back to the server

Lastly, you don't "implement" IMAP - it's nothing more then a mechanism for an <email client> to access the <email account>.

So with all of the above - to answer Q1-thru-Q3

Your browser is nothing more then a vehicle to see what already exists on the server. There are times when it's best to create new folders on the server via webmail (your browser) but in general, creating a folder on the desktop will result in the folder being created on the server. If you create a folder in your email client (Outlook desktop in this case) - Outlook needs to sync with the server before you see anything on the server via your browser.

When you configure the account in Outlook using IMAP, a new (ost) data file gets created and once Outlook connects with the server, those folders will be created on the desktop in the IMAP folder tree. It in no way will have any impact on any folder you have stored in any PST file.

Re: Q4 - POP3 vs IMAP

Everyone has their own requirements so there is no one-size fits all answer. In your case, you want to share your email folders among devices, you have little choice in terms of which protocol to use with your att.net account.

What you do have to take into consideration is the amount of space you have to store all your historical on the server and only you know what that limit is.
