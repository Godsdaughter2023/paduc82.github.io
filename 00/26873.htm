rich michaels - mvp (greatcirclelearning.com) say:
To my knowledge there is not a way to do that by any standard method from MS-Word. 

Theoretically you might be able to write a custom macro that applied a custom number value to each page of a finished document and place it into the footer area but I doubt that it would be worth the trouble and aggravation to write.

My current method of dealing with this in our LeaderGuide Pro product is to provide a block, which we call a Participant Workbook Block. This "block" is actually a formatted row in a Word table, that represents the Participant Guide page. In this "block" the writer enters all of the information that will appear on a given page in the Participant Guide. We then have a syncing mechanism between the two documents so that the page content, including the actual page number from the Participant Guide, is included in the LeaderGuide. But this page number appears in the Participant Workbook Block as a reference and has no bearing or relationship to the page numbering of the LeaderGuide document.

BTW, If you were able to make Word duplicate the page numbering in a document it seems to me that it would render the TOC or any document indexing useless... and that would not be good.

jay freedman say:

You cannot construct nested fields (fields inside other field codes) in the Field dialog or by pasting field codes copied from web pages. Each pair of field markers  {  }  must be created by pressing Ctrl+F9; these are not characters that you can type or copy/paste from other programs.

In the document where you inserted the field, press Alt+F9 to display the field codes. Start by selecting one of the expressions DATE \@ "M" and pressing Ctrl+F9 to surround it with real field markers. Then delete the ordinary brace characters before and after the field. Repeat this for the other DATE field. Next, select = { DATE \@ "M" } -1 , press Ctrl+F9, and delete the ordinary braces. Finally, select everything from IF to "MMMM" and press Ctrl+F9, and delete the ordinary braces. If the very first and last braces are also ordinary, select the whole thing and press Ctrl+F9 once more.

One alternative that does allow you to use copy/paste is to use the macro provided by Graham Mayor at http://www.gmayor.com/export_field.htm#TextToField. It will replace the typed braces with real field markers. Follow the link at the bottom of the page if you need instructions for installing the macro.

macropod say:

A simpler field construction, depending on your regional settings, is either:

{QUOTE "01-{=MOD({DATE \@ MM }+10,12)+1}" \@ MMMM"}
or:

{QUOTE "{=MOD({DATE \@ MM }+10,12)+1}-01" \@ MMMM"}

Note: The field brace pairs (i.e. '{ }') for the above example are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practicable to add them via any of the standard Word dialogues.
