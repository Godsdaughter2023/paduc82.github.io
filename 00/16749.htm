Sub removeMarkers()

Dim c As Chart

Dim s As Series

Dim p As Point

Dim i As Long

i = 0

Dim j As Long

j = 250 'interval between markers

Set c = ActiveChart

For Each s In c.SeriesCollection

    For Each p In s.Points

        If i Mod j = 0 Then

            p.MarkerStyle = xlMarkerStyleAutomatic

        Else

            p.MarkerStyle = xlMarkerStyleNone

        End If

        i = i + 1

    Next

Next

End Sub

Preliminary Verdict on Windows 10.9926 - Microsoft Community

a. user say:

Preliminary Verdict on Windows 10.9926

I was really excited to learn that the next build of Windows 10 was coming out early, and I thanked my lucky stars for my 50mbps connection so I could download the ISO and try it out.  The result, however, gives me mixed feelings.

On the one hand, clearly this build is the result of a lot of effort.  I can tell because it seems much more stable and much better-behaved now than it was.  Perhaps more critically, that pesky DRM problem seems to have been fixed, meaning that now I can access and view content purchased through Amazon Unbox on my Windows 10 system.

On the other hand, there's the Start menu.  I knew it was going to change and that change between now and the shipping OS is inevitable.  But, I loved the Start menu as it existed in 9879; all the tiles fit together nicely in the alotted space, and nothing overlapped with the taskbar.  Now, when I open the Start menu, I get something that looks...for lack of a better word, cluttered.  The arrangement of the tiles doesn't look anything like what was there before I installed the new build.  Some are missing entirely, others simply resized and/or moved.  The list of apps on the left looks less organized to me, I can't quite explain why but on an instinctual level I far prefer the 9879 presentation.

As far as technical issues go, despite the fact that it does seem much more stable now, while using Bryce to do multiple renders at once (something that Windows 7 takes in stride), after less than an hour Windows 10 kicked up a Memory Management STOP error.  It's certainly better than it was before--it used to crash almost immediately as soon as I started multiple renders--but the fact that it still can't handle something that, on the exact same machine, Windows 7 handles with no trouble at all is troublesome to me.

I will say that I appreciate the fact that the Start menu can now be toggled to fullscreen and not fullscreen, in effect giving users a choice between a classic feel or a Metro-style feel.  I've said repeatedly that the Metro Start screen does definitely have its uses, even on a mouse-and-keyboard PC, so this is a nice touch.  The only question I have is whether it might be possible to revert the Start menu itself back to the appearance we had in 9879, while leaving that toggle functionality in place?
