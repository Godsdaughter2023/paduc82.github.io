jay freedman say:

The "Microsoft Word" entry in the folders list is a "pseudofolder" like the "Quick Access" link above it. By itself it doesn't point to any real folder. If you try to save a file there, you'll see this error message:

In Word 2010, as you noted, Word placed a link to the Templates folder under it. Starting in Word 2013, though, Microsoft changed the way templates are stored. The original Templates folder, %appdata%\Microsoft\Templates, is now supposed to be used only for the templates that Microsoft supplies. If you save your own template, the Save As dialog automatically switches to the folder %userprofile%\Documents\Custom Office Templates.

The intention of this change is that your personal templates would be synchronized among all the computers on which you use Office. However, I doubt that many people are concerned with this synchronization, and would prefer to have all the templates -- both Microsoft's and their own -- in one place.

Fortunately, you can force Word to do what you wish. In the Options dialog, go to the Save page and insert the path to the Templates folder in the "Default personal templates location" box. That will affect both the Open and the Save As dialogs. At the same time, you can tell Word to default to saving in a local folder instead of OneDrive (check "Save to Computer by default") and specify that folder (in "Default local file location").

One last tip: If you frequently want to go to the hidden Templates folder, add it to the Quick Access group. You can do this in File Explorer and it will appear in the Open and Save As dialogs, or vice versa.
