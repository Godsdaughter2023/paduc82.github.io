Here is another macro that you might like to consider...

Sub Delete_Empty_Rows_In_Active_Sheet()
  Dim UnusedColumn As Long, LastRow As Long
  Const StartRow As Long = 14
  UnusedColumn = Cells.Find(What:="*", SearchOrder:=xlByColumns, _
                 SearchDirection:=xlPrevious, LookIn:=xlFormulas).Column + 1
  LastRow = Cells.Find(What:="*", SearchOrder:=xlRows, _
            SearchDirection:=xlPrevious, LookIn:=xlValues).Row
  Application.ScreenUpdating = False
  With Cells(StartRow, UnusedColumn).Resize(LastRow - StartRow + 1)
    .FormulaR1C1 = "=IF(COUNTA(RC[-" & (UnusedColumn - 1) & "]:RC[-1]),"""",""X"")"
    .Value = .Value
    On Error Resume Next
    .SpecialCells(xlCellTypeConstants).EntireRow.Delete
    On Error GoTo 0
  End With
  Application.ScreenUpdating = True
End Sub

Hi MarkForeman9703,

Based on your description, the issue is a shared mailbox which you have full access permission doesn’t show up in Outlook client.

As far as I know, the Auto-mapping is related to shared mailboxes which show up in Outlook. So about “the shared mailbox disappeared”, if you are an administrator, I recommend you re-set the value of Auto-mapping.
a. Connect to Exchange Online using remote PowerShell   
b. Run the command using windows Powershell:
Remove the full access permission
Remove-mailboxPermission “sharedmailbox” –user your mailbox -AccessRights FullAccess -InheritanceType All

Add the permission again with automapping:
Add-MailboxPermission "sharemailbox" -user your mailbox -AccessRights fullaccess -AutoMapping $true
If the steps above don’t work, you can re-create one new profile:
https://support.microsoft.com/en-us/kb/829918 
Full access users have shared mailbox account folders automatically show up in their Outlook client.

If the shared mailbox still doesn’t show up, please help us collect the information below:
1. About “I can add a shared calendar and load it that way, but with only READ access.”, how do you add the calendar of the problematic shared mailbox in Outlook? Please give more detailed description.
2. Since you mentioned “tried adding a few other users with the same result”, do you mean the other users with full access of the shared mailbox cannot view it in Outlook client?
3. Have you tried to configure your account on Outlook in another computer?
4. Does the problematic shared mailbox show up in Outlook before when you have full access permission?
