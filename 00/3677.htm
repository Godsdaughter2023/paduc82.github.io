How to conditionally hide rows - Microsoft Community

jgang19 say:

How to conditionally hide rows

How can I conditionally hide rows 14 through 475 if the value in that rows column P is greater than 3.1* the value in the same rows column R. 

For example:

P17 has a value of 10. R17 has a value of 32. Thus, the row entire is hidden. 

P18 has a value of 10. R18 has a value of 20. The row remains shown.

gord dibben say:

To hide rows would require VBA.

Here is a toggle hide/unhide macro.

Sub Toggle_Rows()
    Dim rngCell As Range
    Dim Rng As Range
    Set Rng = Range("R14:R475")
    Application.ScreenUpdating = False
    For Each rngCell In Rng.Rows
        If rngCell.Value > 3.1 * rngCell.Offset(, -2).Value Then
           rngCell.EntireRow.Hidden = Not rngCell.EntireRow.Hidden
           Else
           rngCell.EntireRow.Hidden = False
        End If
    Next rngCell
    Application.ScreenUpdating = True
End Sub

The four inactive ones are add-ins that ship with PowerPoint. (They're part of the Document Inspector.) They won't cause this behavior.

I've not heard of the Acrobat PDF Maker add-in causing Ctrl+M to not work, but you might try disabling it just in case.

Actually, before disabling the add-in, try going to Office Button | PPT Options | Resources and run Office Diagnostics there. With any luck, it will fix the problem.

Also, make sure you've rebooted! I know that sounds snarky, but seriously, when keystrokes stop working suddenly for me, a reboot often takes care of it. And if some program running at the same time as PPT is causing a conflict, you might be able to figure it out by starting with a clean boot.
