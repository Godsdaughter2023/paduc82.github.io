john korchok say:

Ctrl + Backspace and Ctrl + Delete both delete entire words.

Ctrl + Shift + Up or Down Arrow selects entire paragraphs.

Pressing F8 repeatedly will expand a current selection from a blinking cursor to a word to a sentence to a paragraph.

Shift + a variety of movement keys will select blocks of text, but the movement keys are off to the right of the QWERTY area, so it's less likely you are combining these accidentally. As an example, Shift + End selects from you cursor position to the end of the line. Shift + Delete deletes all selected text in the same way as Ctrl + X.

suzanne s. barnhill say:

To add to what John said, double-clicking selects a word, triple-clicking a paragraph, and Ctrl+clicking a sentence. If you inadvertently do any of these and then backspace, you'll inadvertently delete a block of text.

The simplest way to produce such a result would be to use a report with a multi-column subreport ordered by R01, with the column follow in the latter across-then-down.  The parent report would return one row per household with a query such as the following as its RecordSource property:

SELECT DISTINCT aacode
FROM People;

The subreport's query would join people and Relatives:

SELECT aacode, relationship
FROM Relatives INNER JOIN People
ON Relatives.ID = People.R01
WHERE R01 <> 0;

The parent report and subreport would then be linked on aacode as the LinkMasterFields and LinkChildFields properties.  You'll find an example of a similar use of a report/subreport as ClubMembership.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

You might have to copy and paste the text of the above (not copy the link location) into your browser; for some reason it sometimes doesn't seem to work as a hyperlink.

If you download the file you'll see it also contains a single report solution, where the layout is manipulated by code in the report's module at runtime.  This is included only because the file originated in response to an enquiry for a code solution by a reader of a magazine column written by a contact of mine.  Ignore this solution though; a subreport is far easier.

If you do wish to go down the concatenation route as suggested by Vanderghast   Jamie Collins once posted  a very efficient method for doing this, using the GetList method of the ADO recordset object.  I've wrapped this in the following function:

Public Function GetList(strTable As String, strColumn As String, strSortColumn As String, strDelim As String, Optional strFilter As String = "True") As String

    Const NOCURRENTRECORD = 3021
    Dim rst As ADODB.Recordset
    Dim strSQL As String
    Dim strList As String
   
    strSQL = "SELECT " & strColumn & " FROM " & strTable & " WHERE " & strFilter & " ORDER BY " & strSortColumn
  
    Set rst = New ADODB.Recordset
   
    With rst
        Set .ActiveConnection = CurrentProject.Connection
        .Open _
            Source:=strSQL, _
            CursorType:=adOpenForwardOnly, _
            Options:=adCmdText
       
        On Error Resume Next
        strList = .GetString(adClipString, , strDelim & " ", strDelim & " ")
        .Close
        Select Case Err.Number
            Case 0
            ' no error so remove trailing space and semi-colon
            ' and return string
            GetList = Left(strList, Len(strList) - 2)
            Case NOCURRENTRECORD
            ' no rows in table so return
            ' zero length string
            Case Else
            ' unknown error
            GetList = "Error"
        End Select
    End With
   
End Function

To see how it works open the sample Northwind database and first create a reference to the Micosoft ActiveX Data Objects library (Tools | References on the VBA menu bar).  Then create the following query and save it as qryProducts:

SELECT Orders.[Customer ID], Products.[Product Name]
FROM Products INNER JOIN (Orders INNER JOIN [Order Details]
ON Orders.[Order ID]=[Order Details].[Order ID])
ON Products.ID=[Order Details].[Product ID]
ORDER BY Products.[Product Name];

The function can then be called in a query, passing the above query's name into it, like so:

SELECT Customers.[Last Name], Customers.[First Name],
GetList("qryProducts","[Product Name]","[Product Name]",",","[Customer ID] = " & Customers.ID) AS ProductsOrdered
FROM Customers;

to return a comma separated list of all products ordered per customer.

hansv mvp say:

If you want to display 1,600,212,000 as 1.6, use a custom format of 0,,,.0 (a zero followed by three commas, a point and another 0).

If you want to round 1,600,212,000 to 1.6 for further processing, you can use a formula of the form

=ROUND(A1/1000000000,1)

where A1 is the cell containing 1,600,212,000.

You cannot select a range on a sheet that is not the active sheet. Try this:

    Dim FindText As String
    Dim ReplaceText As String
    FindText = Sheets("Sheet1").Range("O2").Value
    ReplaceText = Sheets("Sheet1").Range("K4").Value
    Sheets("Sheet2").Range("E1:E999").Replace What:=FindText, _
        Replacement:=ReplaceText, LookAt:=xlPart, _
        SearchOrder:=xlByRows, MatchCase:=False, _
        SearchFormat:=False, ReplaceFormat:=False
