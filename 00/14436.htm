Fieldnames are pretty much irrelevant. If you want to display identifiers on a Form or Report you can use a Label control; you can set the Caption property of the label to one line, or multiple lines of text.

Last <Ctrl-Enter> Name

Last

Outlook must do a better job than it has been. These problems have been around for at least a decade. While most of Office is matured to the point that little feature improvement was necessary, this is one area that is sorely lacking.

 

Problem 1: Attempting to copy a recurring event to a new time results in cloning the event in its original time slot and pattern, rather than creating a new event in a new time slot. Screenshot 

 

What Outlook should do when a user requests to copy an event is to ask whether you want a copy of a single occurrence or to copy the entire event. For simple recurrences, it should easily adjust the pattern to use the new start date. Mondays at 5pm copied to Tuesdays at 4pm? No problem! At the very least, making a single-occurrence copy would at least allow a new recurrence to be manually set from that.

 

 

Problem 2: Outlook clears all exceptions and creates a new copy of the recurring appointment if any edits to the recurrence information are made.

 

Outlook needs to preserve recurrence exceptions as a general rule. Changing the end date to a later time should only add new occurrences. Changing the end date to an earlier time or from "no end" should only remove the occurrences beyond the given time. Changing the start date should only remove past occurrences prior to the start date. The user should be given an option to remove all future exceptions or remove all exceptions.

 

 

Many people value the calendar not just for its present usefulness, but its historical accuracy. If someone creates an indefinite recurring appointment that then changes, why should any modern calendar program force them to lose all past information associated with it? It should be trivially simple to change an indefinite recurrence to a finite one and then to copy that recurrence to a new time and date to start a new one with all the same characteristics.

 

Have I missed anything? I can't be the only one annoyed by these flaws.
