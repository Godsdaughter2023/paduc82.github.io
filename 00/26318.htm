suzanne s. barnhill say:

If you just want a new index, you can delete the INDEX field and start over, but if you are really wanting to delete all the marked entries and start from scratch, here's how:

  1. Display Hidden text. The easiest way to do this is to display all nonprinting characters using the Show/Hide ¶ button in the Paragraph group of the Home tab or press Ctrl+*.
  2. Press Ctrl+H to open the Replace dialog.
  3. In the "Find what" box, type:
    ^d XE
  4. Leave the "Replace with" box empty.
  5. Click Replace All.

The "Find what" text searches for a field beginning with XE. Since this selects the entire field, replacing it with nothing deletes it.

If you want to search for specific XE fields, you can use ^d XE "word" as the "Find what" text, where word is the indexed text.

Sure. There are a couple of approaches.

On the other hand, if you edited the bar styles for the Gantt Chart view but now want to go back to the default set, first change the view to anything other than the Gantt Chart and open the Organizer (File/Info). In the view tab of the Organizer find the Gantt Chart in the selection list on the right side. Hit the "Delete" button and close the Organizer. Now when you select the Gantt Chart view, you will see all the default bar styles restored. A similar approach can be used for other views.

Windows 10 - Adaptec RAID Controller disappearing upon resume from - Microsoft Community

dougguy say:

Windows 10 - Adaptec RAID Controller disappearing upon resume from sleep SOLVED!

OS Windows 10 Home 64bit upgrade from 8.1 OEM.

I have an older Adaptec 3405 RAID controller with 4 drives in RAID 1E.  The problem I was having was that the controller would disappear when the machine resumed from sleep and I would have to reboot to pick up the controller again.

In Windows 8.1 sleep worked correctly and the controller worked after resuming from sleep.  I did an install from Windows 8 DVD on a spare drive and copied all the arcsas.xxx files into a separate folder, then did the same after the 8.1 upgrade, noting the version changes along the way.  Of course Windows 10 would not allow the older unsigned driver from Windows 8.1 to be installed because it did not have a digital signature.

Google search for "disable driver signing Windows 10" turned up the following link, which WORKED PERFECTLY to allow the older driver to install!

http://www.howtogeek.com/167723/how-to-disable-driver-signature-verification-on-64-bit-windows-8.1-so-that-you-can-install-unsigned-drivers/

After running the installer to update the driver, Windows cautioned about using an unsigned driver and installed the driver files I had copied from Windows 8.1 and after rebooting, the older driver is still installed, and the machine will enter sleep mode and then resume from sleep mode with the raid controller intact, and I am able to access files on the array correctly.
