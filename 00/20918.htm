To change DDE functionality in Word after installing the update, follow these steps:

In the Registry Editor navigate to \HKEY_CURRENT_USER\Software\Microsoft\Office\<version>\Word\Security >> AllowDDE(DWORD)

 

     Note: that <version> reflects the highest version shown in registry

     Example: \HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Word\Security >> AllowDDE(DWORD)

 

Set the DWORD value based on your requirements as follows:

AllowDDE(DWORD) = 0: To disable DDE. This is the default setting after you install the update.

AllowDDE(DWORD) = 1: To allow DDE requests to an already running program, but prevent DDE requests that require another executable program to be launched.

AllowDDE(DWORD) = 2: To fully allow DDE requests...'

 

Note: If you need to create a new registry entry

 

1. Right Click >> New >> DWord (32-bit) value

2. Name = AllowDDE

3. Data = 2

a. user say:

Windows 10's activation and licensing comes in two pieces, one involving a license key (those you can buy), and a system hardware hash (which it generates from your installed devices).  Now, if you replace the MBU, it'll read that you're on a completely different system, since in effect you are--the "brain" may be the same, but the "heart" is new, so to speak.

If you replace the MBU, as long as it's compatible with Windows 10, boot cycle should be no problem.  It will probably prompt you for activation though, which may or may not require you to call MSFT.  Once done, that copy will now be linked to your new hardware hash.  That's permissible since you're only using it in one place, as the EULA provides.
