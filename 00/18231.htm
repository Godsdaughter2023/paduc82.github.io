marshall barton say:

Make sure the main report's record source query does not include the subreport's table(s).  If there is no information specific to the main report, the main report should be unbound.  If the main report contains information from the subreport's parent table, remove any joins in the record source query's From clause so the main report's query only refers to that parent  table in its From clause.

OTOH, if both the main report and the sub report are using the same table, then you should consider using Sort/Group instead of subreports.

Does the subreport with the data use multiple columns?

Are you guaranteed that the report, now and forever, will always fit on one page?

You might be able to get by with setting both subreport control's AND the main report section's (that contains the subreports) CanGrow property to Yes.

a. user say:

Hi,

 

There might be a problem with your internet connection and/or the person you're calling. If Skype detects a weak connection, it will reduce the quality to try and stop the call from dropping altogether.

It would be helpful to restart your modem and wireless router. This helps create a new connection to your Internet service provider (ISP). When you do this, everyone that is connected to your Wi-Fi network will be temporarily disconnected. Sometimes antivirus or malware-prevention software might prevent you from getting connected. You can see if the connection issue is caused by antivirus and malware-prevention software by turning it off temporarily and then trying to visit a website you trust. If you find it is causing the connection issues, contact the software manufacturer or visit their website to see if updated software is available.

For the best performance, make sure that you and the person you’re calling are using the latest version of Skype. Download the latest version. I also recommend to Completely uninstall and reinstall Skype.
