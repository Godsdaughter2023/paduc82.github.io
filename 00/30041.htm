Windows Update error 800f0826 is displayed when an update for Windows has failed to install and is preventing you from installing additional updates. Error 800f0826 does not identify the update that has caused the problem. The update that has caused the problem will display a different error. This problem affects updates for Windows only and does not prevent you from installing or automatically receiving updates for other Microsoft programs, such as the Microsoft Office System.

To prevent error 800f0826, you need to identify the update that's causing the problem (it will display a different error message), fix the problem, and then try installing updates again.

4) Posted by : artologik
The solution came to me when I could not recreate
the problem on a different laptop with Vista Home.
Comparing everything that I could, I noticed that the
%PATH% environment variable was different between
the 2 machines.

I guess WLPG relies on DLLs that must be found
on the system path; when not found, it must revert
back to "basic or incomplete" libraries that it finds
on default or user specific paths. The solution for
me was to ensure the overall system path pointed
to the proper folders in the proper order.

To solve, do this:

A. Right-click the COMPUTER icon on the
desktop and choose PROPERTIES

B. Choose ADVANCED SYSTEM SETTINGS

C. If prompted by UAC to authorize, go ahead
and ALLOW

D. Click ENVIRONMENT VARIABLES....

E. In the bottom half, under SYSTEM VARIABLES,
locate the PATH variable and highlight it

F. Click the EDIT button

G. Make sure that whatever is there starts with
%SystemRoot%\system32;%SystemRoot%;

H. Make sure a semi-colon separates  each entry.

I. Click OK three times

J. Restart computer

K. Et voilà.
