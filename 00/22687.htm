hansv mvp say:

By default, all cells in an Excel worksheet are locked, but this only becomes effective when the worksheet is protected. As long as the worksheet is unprotected, it doesn't matter whether the cells are locked or unlocked.

The procedure is as follows:

Step 1: unlock cells

  • While the worksheet is still unlocked, select the cells that the user should be able to edit. You can use Ctrl to select non-adjacent ranges if necessary.
  • Press Ctrl+1 to activate the Format Cells dialog.
  • Activate the Protection tab.
  • Clear the Locked check box.
  • Click OK.

Step 2: protect worksheet

  • Activate the Review tab of the ribbon.
  • Click Protect Sheet.
  • You can tick/clear check boxes to specify what the user should be able to do.
  • At the very least, leave the check box Select unlocked cells ticked; otherwise the user won't be able to select the cells you specified in Step 1.
  • If you want to protect the worksheet with a password, enter it in the box. Do not forget this password - you will need it if you ever want to unprotect the sheet to change its design.
  • Click OK. If you specified a password, you will have to enter it again as confirmation.

There is an option to enable exactly what you have described.  It has been turned off by default because most people do not like that behavior (they start their computer and have to wait 20-30 minutes to use it because we are scanning).  If you want to enable it, you can edit the registry:

Key: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Microsoft Antimalware\Scan
Value Name: DisableCatchupQuickScan
Value Type: DWORD
Value: 0

Key: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Microsoft Antimalware\Scan
Value Name: DisableCatchupFullScan
Value Type: DWORD
Value: 0

You may need to take control of the Microsoft Antimalware Key in order to add these values.  There is always some risk when you make changes to the registry.  You can back-up your registry before making these changes to mitigate some of that risk.

deepa raj say:

Hi CHABRA,
 
This is a very common question, and the answer is usually one of these two scenarios:
 
1.       Did you install Office from a disc that you purchased?  If so, go to Control Panel> Programs>  Programs and Features if its Windows vista or Windows 7 operating system ,if it’s a Windows XP operating system you will have to go to Add or Remove programs and see if there is another copy of Office installed besides the suite you are trying to install. Some pre-installed Trial Office 2007 suites can conflict with the product key validation process.  If you see a duplicate copy of Office installed {(Office 2002/2003 or Office System 2007(Note: you can see Office system on the very top of the list)} uninstall it and try to run your new Office program again.  Chances are your product key will work now.
 
2.       If you are using the 25-digit product key that you found on a sticker on the bottom of your laptop or on the back of your desktop, then you are using the wrong key.  This key is the product key for the Windows operating system, and NOT for the Office Suite.  The correct product key for the Office suite will be in one of these locations:
a.       In the Office 2007 box (if you bought Office from a retail store)
b.       In an e-mail from the online website where you purchased Office from.

 
Note: check if you are typing the key correctly, sometimes the letter "B" and the number "8" looks alike.
 
Try installing office again.
 

Regards,

Deepa Raj
 
 
Microsoft has a knowledge base article that goes into more detail around the invalid product key scenario.  It is located at http://support.microsoft.com/kb/836178 ,and would be a good place to go if these instructions above do not resolve your issue.
