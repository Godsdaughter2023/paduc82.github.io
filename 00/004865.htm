jllatham say:

One thing you might try would be to use the network URL type of address rather than a drive letter as part of the link address. 

But I'm curious as to why the links should be broken or need resetting if you have them set to a mapped drive and that mapping is in effect when you open the workbook later.

But we can explore an alternative:

First thing to do is to find out what the network address is.  Let's say you have drive X:\ mapped to a shared folder on a server or other system that shows up in Network Places as

\\HostMachine\SharedFolderLvl1\SharedFolderLvl2

So your link (with the other file closed) might start out looking like

='X:\[Linked Excel FileName.xls]Worksheet Name'!$A$1078

so you could change the link to

='\\HostMachine\SharedFolderLvl1\SharedFolderLvl2\[Linked Excel FileName.xls]Worksheet Name'!$A$1078

You could make that change with an Edit --> Replace  for each sheet in the workbook.  The advantage here is that drive X doesn't even have to be mapped on your system at the time you use your workbook.  It will work as long as HostMachinen is available on the network.
