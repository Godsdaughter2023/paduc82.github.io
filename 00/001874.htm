    Unable to update this object OnPremiseSecurityIdentifier System.Byte - Microsoft Community

    a. user say:

    Unable to update this object OnPremiseSecurityIdentifier System.Byte

    Install powershell modules on the server Azure AD Connect is configured on.

    Import-Module MSOnline

    $msolcred = get-credential

    connect-msolservice -credential $msolcred

    Install-Module -Name AzureAD

    Connect-AzureAD

    Connect-MsolService

    There was a local AD account needing to sync with O365 as a "Synced with AD" 

    Every sync attempt resulted in the error "Unable to update this object OnPremiseSecurityIdentifier System.Byte" AttributeValue must be unique" etc...

    Extra details of the error: - 

    Conflicting ObjectId was not local but was rather within Azure AD.

    eg. {"Key":"ObjectIdInConflict","Value":["erb5ff1e-3d7b-4a63-b30f-83b625d3ffc9"]},

    PowerShell>

    Get-AzureADObjectByObjectId -ObjectIds erb5ff1e-3d7b-4a63-b30f-83b625d3ffc9

    ObjectId                             DisplayName    Description
    --------                             -----------    -----------
    erb5ff1e-3d7b-4a63-b30f-83b625d3ffc9 FIMSync*****

    erb5ff1e-3d7b-4a63-b30f-83b625d3ffc9 FIMSync*****

    Results showed a FIMSync Group Account as the culprit.

    Deleted the group, do a manual delta sync for AD Connect, monitor the error results.

    This should resuult in the conflicting O365 account showing as synced with local AD.

    -

    Recreate the deleted group if needed which will result in different objectID within Azure AD.

    In this case the group didnt have anything associated with it on-prem or in cloud.
