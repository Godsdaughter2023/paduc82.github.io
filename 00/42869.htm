greg carmack - mvp 2010-2020 say:

If you have a Windows 8.1 retail Pro Product Key it can be used to unlock Windows 10 Pro by the following procedure:

First make your Factory Recovery media.  Then I would also save a Windows 10 Image Backup  so you have the ability to go back at any time to the old OS and therefore have a secondary recovery method. 

Next go to Settings>System>About, confirm you have Version 1511 there.  If so at the link try changing the Product key to the Windows 8.1 Pro Product Key since Version 1511 will accept Windows 7, 8, 8.1 Product Keys to activate the OS. 

If it will not activate it's because your key is for Pro and not Home which is presently installed.  In this case what will work is to insert the default Pro Product Key VK7JG-NPHTM-C97JM-9MPGT-3V66T which will change the OS to Pro version but not yet activate. 

After it changes to Pro version, insert the Windows 8.1 Pro Product Key and it should activate shortly after connecting to the internet. 

You will then have both Home and Pro version licensed on the PC, preinstalled Home version tied to the PC via embedded OEM-BIOS Product Key which can be read at any time using ProduKey, and Pro version via retail Product Key which can also be retrieved to move elsewhere by deleting the Pro version off the PC.

wolverine2009 say:

I had the same issue, except that I had run sysprep on one of my systems, but never actually resealed the system.  I actually wanted to cancel the sysprep, but wasn't sure how.

 

  I found this article about sysprep mini-setup.

 

http://support.microsoft.com/kb/814588

 

That pointed me to the registry setting.

HKLM\System\Setup

 

In that registry folder, CmdLine had the entry to start sysprep at startup.  I cleared the string for CmdLine to an empty string, and I set the AuditInProgress, FactoryPreInstallInProgress, MiniSetupInProgress, SetupType, and SystemSetupInProgress to 0, as well.

 

That fixed it for me.

 

Hope this helps someone else out there.

 

 

NOTE: The windows registry is essentially a database of configuration information for Windows.  Be cautious when editing the registry.

    williamtodd say: Kind of a belated reply, but for what it may be worth:

    System Mechanic is at a minimum being alarmist, since there's certainly nothing wrong with your disk or the file system on it.  Unless your disk is an 'advanced format' traditional disk with 4KB sectors (note that the sector size has nothing to do with whether your operating system is 64-bit) - or a solid-state disk with 4KB sectors - there's nothing even sub-optimal about the actual alignment of its files, and even if you do have an 'advanced format' disk or an SSD the only real advantage to aligning files on 4KB disk boundaries is that small write operations will execute faster (and in the case of the SSD will minimize the number of sectors affected by a write operation, possibly extending the life of the drive a bit).

    Apparently System Mechanic does make some effort to gather related files (e.g., those relevant to a single application) together on the disk to speed up certain operations.  Whether this produces any noticeable performance improvement I have no idea (it could if SM is sufficiently intelligent about this - e.g., if it carefully analyzed the operation of each such application in terms of its disk access and then optimized relative placement on that basis - but forgive me for suspecting that it's nowhere nearly that suave:  if it were, they ought to be touting it in much greater detail).  And whether it does any better a job than Windows itself does at arranging system files to execute efficiently when defragmenting the disk seems questionable.
