sumit (volunteer moderator) say:
This is one of the known issues with the update as Recovery partition should be hidden and is accidentally assigned a drive letter.
Right-click Start>Command Prompt(admin)
Type in the following:
diskpart
list volume
select volume 3
Replace 3 with the number of the volume which is assigned a drive letter. (In hundreds of MBs)
remove letter="F"

Don't worry, removing a drive letter would not cause any harm to your device.

davem121 say:
Hi Hilary, this is a known issue in the 1803 Update,

It is the recovery drive, you are not meant to see it, it is not meant to have a drive letter, remove the drive letter and everything will return to normal:

Click your Start Button, type cmd, then right click Command Prompt and choose 'Run as Administrator'

Run this command and hit Enter

diskpart

Run this command and hit Enter

list volume

Run this command and hit Enter

select volume F

Run this command and hit Enter

remove letter=F

Close Command Prompt and restart your PC

cari [former ms-mvp] say:

You cannot format a Windows partition whilst running Windows.  It will simply refuse to commit suicide.  You will need to boot with the Windows 7 DVD, choose a Custom Install and delete the XP partition from there. 

You will also lose the boot manager at the same time, since Windows has put it on the first partition of the first hard drive (which is probably XP since that was the original operating system.  After you have put Ubuntu on it, you will probably have to reboot from the Windows 7 DVD and "Repair" the installation to restore the boot manager.

100gb is not a very large hard drive these days.  You might want to get a bigger hard drive and take the opportunity to 'start again' with larger partitions.
