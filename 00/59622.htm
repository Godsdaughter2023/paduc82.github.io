Hi Paul

The main problem is that the Application Data 'folder' is not a folder, it is called a Junction.

Some of the system folder names were changed in Windows 7 and to assure compatibility with older programs that are hard coded to look for these former names, Windows 7 uses Junctions with the older name to redirect these older programs to the new location.

The C:\Users\Den\AppData\Local\Application Data junction redirects to: C:\Users\Den\AppData\Local.

These Junctions have the permissions set to Everyone = Access Denied. They cannot be used as a folder to store any files.

This problem occurs when a user changes these permissions in an effort to force access to that junction. Changing the permissions allows any action to 'recursively walk through' the junction and this creates a recursion (Also called an infinite loop) the result of which is what you are describing.

Your best recovery option is to use System Restore to go back to a point before the permissions were changed. If this is not possible, see the following post to see how another member fixed the problem.

http://social.answers.microsoft.com/Forums/en-US/w7files/thread/4ec2c92c-a691-432b-b86a-c76d2b9a3faa/#ef7a1f00-542c-4858-b4ac-1e8a75a30cc8

There are many other Junctions in Windows 7 and these can be recognized by the Shortcut Arrow that is present on the folder icon. See the following post for the instructions on how to create a text file that shows all of the Junctions on Windows 7 and the target folder that they point to.

http://social.answers.microsoft.com/Forums/en-US/w7files/thread/20f32f0c-4cb9-4125-923d-6a57e4d27232/#59862ebd-e54d-4316-bf9a-fd89761ceff8

a. user say:

Often the hal.dll missing message is indicative of a fouled up boot.ini file - pointing to the wrong place.

If you pull the C drive and boot on the D drive, XP will start and the if the boot.ini is telling it to find the hal.dll file on C (which was removed) it will complain in the manner you described.  The hal.dll will be reported as missing because XP is being told to look for it on the C drive.

I suggest you do whatever you have to do to boot, then post up the contents of your c:\boot.ini and then under that post up the contents of your d:\boot.ini and we can help sort it out.  Make it clear which is which.  The boot.ini is just a text file, so open them with notepad or wordpad, select all, copy, paste back here, close the file with no changes.

Sounds like you just need an adjustment in your d:\boot.ini file so it knows there is no longer a C drive.

Incidentally, XP does not even need a boot.ini file to boot in a single partition, single OS installation (it will complain but still boot).

Sadly, the missing hal.dll message sometimes compels people to immediately start trying to figure out how to replace the "missing" hal.dll when it is really not missing at all (as you can see).
