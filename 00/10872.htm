dedawson say:

The function ResolveDisplayNameToSMTP shown below is a slightly modified version of a Sub I found, strangely enough, in the on-line Outlook 2007 VBA Help (search for the function name shown below for the original, no good for 2003 and earlier).   The following code shows both the function and its integration into the solution to the auto-forwarding task I described in my original post.

 

Sub AutoForwardIt(oItem As Outlook.MailItem)
'
'
' This code is called from a Rule one enables when they're out of the office.
' It provides the user outside of the internal Exchange domain with a
' notification that he has received a message within the internal Exchange domain.
'
' This is done by forwarding a copy of the received message to an email address
' outside of the internal Exchange domain after first removing the body and any
' attachments from the received message, thereby avoiding any privacy issues
' associated with the internal Exchange domain.
'
' The original Subject line is retained, and if the received message came from
' an external source, the code writes the Sender's name and SMTP address to the body.
' If the received message came from the internal Exchange domain, the code attempts
' to resolve the sender's name from the Exchange address lists.
' If resolved, the sender's SMTP address and phone number are written to the message body.
' The modified message is then sent to the external address specified in the code.

Dim oNewItem As Outlook.MailItem

  sSubject = oItem.Subject
  sFromName = oItem.SenderName
  sSMTPAddress = ResolveDisplayNameToSMTP(sFromName)  ' use internal internal Exchange domain address if available
  If sSMTPAddress = Empty Then  ' , otherwise use external smtp address
    sSMTPAddress = oItem.SenderEmailAddress
  End If
  Set oNewItem = oItem.Forward
  oNewItem.BodyFormat = olFormatHTML
  oNewItem.Subject = sSubject
  oNewItem.Body = sSMTPAddress + vbCrLf + vbCrLf + "Original message body (and any attachments) deleted from this forwarded copy." + vbCrLf + "Original message may be viewed on internal Exchange domain "
  iNumberOfAttachments = oNewItem.Attachments.Count
  For i = iNumberOfAttachments To 1 Step -1
    oNewItem.Attachments.Remove (i)
  Next
  With oNewItem
    .To = "*** Email address is removed for privacy ***"
'    .Display  ' un-comment and set breakpoint for debug
    .Send
  End With
 
  Set oNewItem = Nothing
End Sub ' AutoForwardIt

 

Function ResolveDisplayNameToSMTP(sFromName)
  Dim oRecip As Outlook.Recipient
  Dim oEU As Outlook.ExchangeUser
  Dim oEDL As Outlook.ExchangeDistributionList
 
  Set oRecip = Application.Session.CreateRecipient(sFromName)
  oRecip.Resolve
  If oRecip.Resolved Then
    Select Case oRecip.AddressEntry.AddressEntryUserType
      Case OlAddressEntryUserType.olExchangeUserAddressEntry
        Set oEU = oRecip.AddressEntry.GetExchangeUser
        If Not (oEU Is Nothing) Then
          ResolveDisplayNameToSMTP = oEU.PrimarySmtpAddress + vbCrLf + oEU.BusinessTelephoneNumber
        End If
      Case OlAddressEntryUserType.olOutlookContactAddressEntry
        Set oEU = oRecip.AddressEntry.GetExchangeUser
        If Not (oEU Is Nothing) Then
          ResolveDisplayNameToSMTP = oEU.PrimarySmtpAddress + vbCrLf + oEU.BusinessTelephoneNumber
        End If
      Case OlAddressEntryUserType.olExchangeDistributionListAddressEntry
        Set oEDL = oRecip.AddressEntry.GetExchangeDistributionList
        If Not (oEDL Is Nothing) Then
          ResolveDisplayNameToSMTP = oEU.PrimarySmtpAddress + vbCrLf + oEU.BusinessTelephoneNumber
        End If
    End Select
  End If
End Function  ' ResolveDisplayNameToSMTP

onenote handwriting personalization - Microsoft Community

onenote tablet user say:

onenote handwriting personalization

I've done a lot of handwriting in OneNote in the past. Recently, I went through Window's handwriting personalization process, writing out 50 sentences it presented me. My question is: will OneNote's recognition of my past handwriting improve, or has OneNote already stored it's translation of my past handwriting somewhere, so that the personalization will have no effect?

If that is the case, is there some way to coax OneNote into re-recognizing all of my past handwriting?
ben m. schorr -bschorr say: I'll answer your second question first - no, you can't force a re-recognize of past handwriting because OneNote doesn't recognize the ink on the page; it recognizes the pen strokes as you write.

As for the first question...my instinct was to say "No" but actually I suspect it might help. OneNote does use the handwriting engine of the operating system so you may actually see some improvement through improving it.
