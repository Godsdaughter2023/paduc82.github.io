Try the below steps:

1.     Follow the steps from the link below: How to use the System File Checker tool to troubleshoot missing or corrupted system files on Windows Vista or on Windows 7: http://support.microsoft.com/kb/929833

 

2.     Side by Side errors can be diagnosed with tool named 'SxSTrace.exe’ It uses ETW model to trace the side-by-side events. It is shipped in Windows Vista and can be found at System32 directory.

 

When required to diagnose side-by-side errors, start the SxSTracing using the below command:

SxsTrace Trace -logfile:SxsTrace.etl

 

Then run your application that is causing the problem (reproduce the problem).

When done, press Enter to stop the tracing and use the below command to generated a text formatted file of the output.

sxstrace Parse -logfile:SxSTrace.etl -outfile:SxSTrace.txt

 

Going through the generated text file should give details of each of the loaded modules and the bindings.

Is there any setting in WLM client to change this behavior and set it the way I want? Any help is appreciated.

¡firedog say:
In WLMail 2009, threads are sorted by the date of the oldest message in the thread. This ensures that threads maintain their position in the message list. To view all unread messages, use Ctrl-U until there are none left.

WLMail 2011 changed this long-standing behaviour. Threads are sorted by the date of the newest message in the thread. This seems to be what you are looking for, although I cannot imagine why you would want threads jumping around every time a new message arrives.

Top/bottom of the list is governed by the sort order in both cases.
