The System Update Readiness Tool creates a log file that captures any issues that the tool found or fixed. The log file is located at the following location:
•%SYSTEMROOT%\Logs\CBS\CheckSUR.log
•%SYSTEMROOT%\Logs\CBS\CheckSUR.persist.log

The system root is normally C. If the files are not there, checkC:\windows\ logs\CBS.

You may need to change your settings in Windows to be able to see the file. To show hidden files type Folder Options in the search box above the Start button and select View, Advanced Settings and verify that the box before "Show hidden files and folders" is checked and "Hide protected operating system files" is unchecked. Expand by clicking the arrowhead pointing right immediately before each folder in turn to see the next folder in the file path until you see the CheckSUR.log and CheckSUR.persist.log (item 3 below).

andre da costa say:

OEM versions of Windows are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system

That means, the computer with the OEM license has not transfer rights.

nitz walsh say: It doesn't look like this scenario was tested that often, I'd say we're in a minority with people with multiple monitor setups and running Modern/Metro apps.  Most of the apps are entertainment/casual focused, so the type of users with multiple monitors are likely focusing on desktop apps.

Win 8.1's multiple monitor support improvements (in terms of multiple taskbars and desktop wallpaper) are long overdue, but it's pretty threadbare despite the amount of time people have been clamoring for this, and the inclusion of the new full-screen apps just make its deficiencies stand out that much more.

While I don't expect the dizzying array of options you have with an add-on app like Display Fusion (excellent utility btw), some basic operations make it appear there wasn't much thought put into it at all <insert snarky comment about Win8's UI in general here>.

For example, with Display Fusion, it's default setting is to move the requested window to the monitor where the request was made from.  For example, I'll have three taskbars on Win7 with DF, just like Win8 default.  However on my Win7 machine with DF, when I click the explorer icon or hit Winkey+E while that monitor has focus, my explorer window will open on the last screen it was opened on as that's default Windows behavior - but then DF will immediately move it to the monitor where the request was made.  It's a small item in a multiple monitor checklist but boy does it make a huge difference in workflow.  With Win8 I'm constantly managing my windows on multiple monitors by moving them around, while I of course want that level of control when I need it, there has to be a little more intelligence on the OS/UI side.

So something like this, or additionally a way to configure a setting whereby a Metro app is "locked" to a screen and desktop apps/windows will automatically be re-directed away from it as to not interfere with it would be most welcome, but again I'd be very surprised if we see any movement on this before Win9, if even that considering how long it took to get multiple taskbars.
