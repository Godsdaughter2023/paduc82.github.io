Looks like fixed this.

It's down to 3 things.

These are what I did in order, but only the last one seemed to tip the balance - but that could be to do with servers updating etc.

1. Make sure you have an SPF record for your domain (on the Google Apps side). Check here: https://support.google.com/a/answer/178723?hl=en

2. Check your MX records are correct. This is a great tool (you can also use it to check for SPF in point 1) http://mxtoolbox.com/ My host set these up automatically with one-click - turns out these are old/wrong settings! They kinda work. Check here: https://support.google.com/a/answer/174125

By this point, it works...kinda...the messages get through to Google Apps (wooo!) but they ALSO bounce. Weird.

3. I was using a child nameserver under my own branding from my hosting reseller account. It was reporting errors (again it mostly works), so I switched the DNS back to the hosting companies nameservers. Check for errors here: http://www.intodns.com/

All seems OK now.

Remember to allow time for server updates to replicate throughout the internet (48hrs).

asillykelley say:

Copy Spreadsheet to another tab monthly

I have a complicated monthly budget spreadsheet.  I am trying to write a Macro (or maybe VBA), so that on the 1st of every month.  The whole sheet in tab one is copied and moved to the next available empty tab.  (Empty Tabs are prenamed, Jan. Feb. Mar.  etc)  After the sheet (with formulas) is copied then I want all the data in certain columns cleared out.   Can anyone help?

ossiemac say:

Hello Lisa,

Rather than copy the data and paste to another existing worksheet, it will be better to simply make a copy of the entire worksheet. Copying and pasting data to another worksheet almost always messes up some of the formatting like column width and row heights etc. However, using Worksheet Copy makes an exact duplicate of the worksheet but to do this the new worksheet cannot pre-exist so suggest deleting the pre-existing sheets "Feb", "Mar" etc.  and use the code below.

Because you say that you want to run the code on the first day of the month (or maybe the first working day of month), I am assuming the new worksheet will be named from the month previous to the first day of the current month. Therefore the code automatically determines the sheet name by the name of the previous month.

The code tests for the existence of a worksheet by the same name and pops up a message box if it exists and instructs the user to delete the sheet and re-run the code.

It is always a good idea to make a backup copy of your workbook before installing and running new code.

The code places the copied worksheet after the last worksheet in the workbook. However, If you have any hidden worksheets then that can affect the position of the new worksheet so if this is a problem then let me know because it can be handled.

Because I do not know what columns etc are to be deleted then I can't write that code but you should be able to record code to do this and insert it where I have placed the comment.

Feel free to get back to me if any problems.

Sub Macro1()
    Dim ws As Worksheet
    Dim dte As Date
    Dim strMth As String
   
    'Following sets dte to date one month before today
    dte = WorksheetFunction.EDate(Date, -1)
    strMth = Format(dte, "mmm")
   
    'Test if worksheet name already exists
    For Each ws In Worksheets
        If ws.Name = strMth Then
            MsgBox "Worksheet " & strMth & " already exists." & vbCrLf & _
                    "Delete the existing worksheet and run code again."
            Exit Sub
        End If
    Next ws
   
    Sheets("Sheet1").Copy After:=Sheets(Sheets.Count)
    Set ws = ActiveSheet
    dte = WorksheetFunction.EDate(Date, -1)
    strMth = Format(dte, "mmm")
    ws.Name = strMth
   
   'Insert code here to remove unwanted data
   'Suggest Record the code and then copy and insert here
  
End Sub
