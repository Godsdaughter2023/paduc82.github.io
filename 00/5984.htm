neo yu msft say:

Hi Daniel,

If you already have DirSync/ADFS/SSO/Hybrid deployed with Office 365, there are several factors we should consider when transferring the domain.

  1. DirSync should be fine since all the modified attributes in the on-premises AD should be synced to Office 365 regularly.
  2. For ADFS/SSO, you will either need to reconfigure a federation server for the new domain (certificate for the new domain is needed) or reconfigure the old federation server to support multiple domains by using the parameter SupportMultipleDomain, here is an article explains how it works for your reference: http://blogs.technet.com/b/abizerh/archive/2013/02/06/supportmultipledomain-switch-when-managing-sso-to-office-365.aspx.
  3. For the hybrid setup part, since you have a new SMTP domain, we will need to re-run the HCW to include the new SMTP domain so that the mail flows for the domain won’t be affected between the on-premises Exchange server and Office 365. You will either need to purchase a new certificate for the domain or use the Autodiscover domain feature introduced in https://community.office365.com/en-us/f/155/t/421590.

Try following one by one and see after every step whether it resolves the issue-

1. Sometimes, add-ins can cause problems. To determine, if add-in is a problem start Powerpoint in safe mode and see if problem has gone away.

To start in Safe Mode -

Hold CTRL key and click on application icon and don't release the CTRL key till it asks you for Safe mode confirmation.

If yes, then start application normally and disable add-ins one by one and start application again every time you disable an add-in to determine the culprit add-in.
To disable add-in > File > Options > Add-ins > Down below you have Manage: Com Add-in : Press Go > Untick the Add-ins to disable and tick to enable them

2. Try repairing your office installation (Quick repair is good enough). Try repairing your Office installation. First try Quick Repair (takes 5 - 10 mins) , if that doesn't help then try Online Repair (Takes more than 30 mins but less than 1 hour in general)

https://support.office.com/en-us/article/Repair...

3. Login to Windows with a new user profile and see if problem is resolved.

4. Start your computer in Safe mode with networking and see whether the problem is solved. If yes, then some program is causing the problem. Many third party applications also cause the problem like Tuneup Utilities, Abby Finereader, Kaspersky / Avast Anti Virus etc. See if you have any of them installed. Close / Disable them one by one and see whether it helps.
https://support.microsoft.com/en-us/help/12376/...
