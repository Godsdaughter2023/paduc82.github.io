hal hostetler - mvp-outlook say:
From TechNet: "When an Outlook 2010 account is configured to use Cached Exchange Mode, Outlook 2010 works from a local copy of a user's Exchange mailbox stored in an Offline Folder (.ost file) on the user's computer, and with the Offline Address Book (OAB). The cached mailbox and OAB are updated periodically from the Exchange Server computer."  What this means is that your mailbox and all email reside on the server AND your local Offline Folders. When you delete something locally, it is NOT deleted on the server until one of these periodic updates. If you want everything to reside locally and be removed from the server as soon as its delivered, you need to turn off Cached mode and reset the delivery location from your Exchange Mailbox to a local .Personal Folders (.PST) file. You do that by selecting your Exchange Server account, and clicking the "Change Folder" button in the section labeled "Selected account delivers new messages to the following location".

cataholic say:

Neiro wrote:

...But these batteries are like this. If you charge too long you will reduce it's lifetime faster and I don't want that as I think 8 hours is too long...

..

That is incorrect.

 

For safety reasons lithium ion batteries are internally protected against overcharging so that means there is no real problem leaving them on the charger past the time when they are fully charged. Charging will stop once the battery is charged even though the charger is still plugged in. There is also no problem with charging them frequently when only partly discharged.

 

Also, because of their  flat voltage vs discharge curve (i.e. there is very little drop in their voltage between fully charged and the point close to discharge) simple battery monitors on devices are typically inaccurate.

macro to create appointment "on behalf of"

Hi. I have a macro which sends an outlook appointment to users. This works fine. However instead of sending from my own account I want the invite sent on behalf of a generic account,  and the appointment added to that calendar instead of mine. Is this possible? 

danielwood1985 say:

The following code works to add the appointment using my own account. However the "SendOnBehalfOf" option is not working. The invite just keeps adding to my own calendar, rather than the sent on behalf of account

 

Sub AddAppointments()
Dim myoutlook As Object
Dim r As Long
Dim myapt As Object

 Const olAppointmentItem = 1
 Const olBusy = 2
 Const olFree = 0
 Const olMeeting = 1
 
 Set myoutlook = CreateObject("Outlook.Application")
 
 r = 2
 
 Do Until Trim$(Cells(r, 1).Value) = ""
 
    Set myapt = myoutlook.CreateItem(olAppointmentItem)
   
    With myapt
        .Subject = Cells(r, 1).Value
        .Location = "Desk"
        .Start = Cells(r, 3).Value
        .Recipients.Add Cells(r, 8).Value
        .MeetingStatus = olMeeting
        .AllDayEvent = True
        .ReminderSet = False
        .BusyStatus = olFree
   
    .Body = Cells(r, 7).Value
    .Save
    r = r + 1
    .Send
   
End With
Loop

End Sub
