brian tillman [outlook mvp 2007-2019] say:

The Lost and Found entry was your Inbox.  The PST internal data structures still pointed to it.  In reality, the name "Inbox" isn't really the important value.  The pointers to it are.  By manually creating a new one and deleting the real one (even though it had a different name) you ruined the PST.  Were this to happen to me, I'd start with a new PST, make it the delivery location, then copy the stuff in the old PST to the new.  Once that's finished, you can close the old one to remove it from Outlook.

Another thing you can try first is to start Outlook once with the /resetfolders command switch.  Press WinKey+R.  In the Open field, enter

outlook.exe /resetfolders

Be sure to include the space.  Press Enter.

Under HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run

Add a New String Value. Called mine 'Defender Icon' set a value of

"C:\Program Files\Windows Defender\MSASCui.exe" -hide

Next time the system is started the Defender icon will appear.

jllatham say:

Here's one 'debugging' method that may help you:  leave a trail of breadcrumbs as your code runs, through the main routine and even into other Subs/Functions called. 

Add a worksheet to your workbook to collect these breadcrumbs, in the example below that is simply "Sheet3".

Add a regular code module and put the breadcrumb routine in it:

'============

Sub Breadcrumb(bcPhrase As String)
ThisWorkbook.Worksheets("Sheet3").Range("A" & _
 Rows.Count).End(xlUp).Offset(1, 0) = bcPhrase
End Sub
'============

Now, back in your main routine, add this line of code near the beginning:

  ThisWorkbook.Worksheets("Sheet3").Cells.Clear
that will clear out old breadcrumbs before beginning a new trail for you.

Throughout your code enter statements like these, making sure that each one provides a unique identification of where it is:

  Run "Breadcrumb", "point 10"
  Run "Breadcrumb", "Point 20"
  Run "Breadcrumb", "Point 30"
  Run "Breadcrumb", "Point 40"

The next time your program breaks, you can examine the contents of Sheet3 (or whatever you named it) to see just how far along in your process you got before failure.  You could put a breakpoint there and start single-stepping through the process at that point to try to find the problem statement.  Or you may need to add more breadcrumbs to the trail - and that's the reason I started out with 10 and incremented by 10 to leave room for more in between "logical" stages of the process.

After all is said and done you can remove all of the Run "Breadcrumb", "phrase" statements, or use Edit Replace to add a single-quote mark in front of them to turn them into comments, or just put Exit Sub as the first statement in the Breadcrumb sub itself to negate its operation.
