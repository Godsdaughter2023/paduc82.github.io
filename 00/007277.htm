joeu2004 say:

It is very likely that the value is not exactly zero, but it looks that only because of formatting.  To confrim, temporarily format L8 as Scientific, and you will probably see something of the form 1.23E-16.

These are actually common off-by-a-little side-effects that are caused by the way that Excel represents values internally, called 64-bit binary floating-point.

The best solution is:  if L8 contains a formula like =expression, change it to =ROUND(expression,2).

It is prudent such explicit rounding whenever you expect a calculation with decimal fractions (or division) to be accurate to some number of decimal places (2 for currency).
