Hi RunAndWrite,

 

This issue happens when the default user profile is corrupted. There is a folder named Default User under Users folder in Windows which is a hidden folder. This folder is responsible for new user profile creation. This folder contains the following files.

o   NTuser.dat

o   NTuser.dat.log1

o   NTuser.dat.log2

 

If any of these files are missing or corrupt in the default user folder, then the creation of new user account is a failure.

 

Adding to what David has suggested, try the steps mentioned below.

 

All you need to do is to copy the Default, Default user folder from a computer which is working fine to your computer which has corrupted user folders. If you don’t have a working computer, then ask any of your friends who has a working computer to get it for you off thier Vista computer and put it on a usb for you.

Note: Make sure NTuser.dat, ntuser.dat.log1 remain unopenable programs (do not set default open to word text)

 

Here are the steps for the same

a.    The Default and the Default User folder contain the above mentioned files which are hidden.

b.    You’ll have to “enable the show hidden files” option to view those files.

c.    Copy them to a flash drive.

d.    Put the USB back to a non-working computer and then copy those folders to the exact same location.

            C:\Users\Default

e.    Restart the computer and then try creating a new user account.

 

You should be able to get it work.
