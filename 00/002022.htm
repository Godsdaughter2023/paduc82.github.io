john369xp say:

Windows Defender full scan- Number of files scanned?

During the Full Scan Windows Defender shows that it is scanning over 13.5 million files.
When the scan has finished it show that it scanned 48.8 thousand files.
Explanation any one?

* Moved From: Windows / Apps on Windows 10 / Other Windows apps / Tools

* Oiginal title: Windows Defender Security Center

rob koch say:

There's a couple possibilities here due to the size and length of that full scan.

First, a few of us have confirmed that the 1703 version of Windows 10 Creators Update released in spring 2017 has a display refresh issue in the new Windows Defender Security Center (WDSC).  This results in the display not properly updating once a full scan is completed, instead returning to the number of files scanned during the previous (typically quick) scan.

The other is that due to the very large number of files and so long scan, the quick scan automatically performed by automatic maintenance is being run sometime after the full scan completes.

I tend to believe the first possibility is still more likely, in which case we've found that simply exiting the WDSC interface and restarting it is enough to force a display refresh, resulting in the more recent full scan results being displayed for our systems.

As I understand it, those who've confirmed this problem on their systems and have already received the Fall Creators Update have reported this had apparently fixed the issue.  I've yet to receive this latest update myself, so I can't personally confirm.

Rob
