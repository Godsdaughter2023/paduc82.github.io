in Access 2010; how can I set up a form to clear all fields and - Microsoft Community

fwoody say:

in Access 2010; how can I set up a form to clear all fields and start with new or is that possible?

I want a form w/in a dBase that would be used for annual tracking of events (dates of completion) at the end of the year I want o clear the dates and begin again for the new year. Is there a way to do that with Access 2010?
john w. vinson say:

A Form is nothing but a window. It doesn't contain any data, so your question as posed is irrelevant.

If you wanted to do this you would need to run a Delete query deleting last year's records from a Table... but I'd really suggest that this would be a Bad Idea, and neither necessary nor beneficial! You'ld be throwing away all of last year's data, destroying any chance of historical comparisons ("attendance up 8% over last year"); and unless your tables have millions of records and your database is pushing 2GByte after compaction, there's no real benefit to doing so. Instead, you can base your forms and reports on Queries using a date criterion (e.g. >= DateSerial(Year(Date()), 1, 1)) to limit the records being displayed to the current year.

 

ken sheridan say: I'd agree entirely with John that deleting each year's data is both unnecessary and limits the potential uses of the database.  You can make the form even more flexible by allowing a choice of year.  To do this add an unbound combo box, cboYears say, to the form header with a RowSourceType property of 'Value List' and a RowSource property which lists a range of years , e.g.

2010;2011;2012;2013;2014;2015

including whatever range you think appropriate.  Base the form on a query which references the combo box, e.g.

SELECT *
FROM Events
WHERE YEAR(CompletionDate) = Forms!frmEvents!cboYears
OR (CompletionDate IS NULL AND Forms!frmEvents!cboYears = YEAR(DATE()))
ORDER BY CompletionDate;

In the form's Open event procedure set the value of the combo box to the current year and requery the form with:

Me.cboYears = Year(VBA.Date)
Me.Requery

This will cause the form to show the rows with completion dates in the current year or where no completion date has yet been entered by default, ordering the rows by completion date.  In the AfterUpdate event procedure of the cboYears control requery the form with:

Me.Requery

The form will then show the rows with completion dates in the year selected.

Hi KungenNorling, 

Accounts are being blocked temporarily if there were any unusual activity that was noticed by the system. This is to protect you from any fraud or abuse that may occur. 

To unblock your account, Microsoft requires you to enter a code that you get via a text message. The phone number you use to receive this text message doesn't need to be associated with your account. It doesn't even need to be your own phone number. If your phone doesn't receive text messages, ask a friend or family member if you can use theirs. 

Note: Microsoft won't save any phone numbers you will use for this process.

You may follow the steps below to unblock your account: 

  1. Go to https://account.microsoft.com and sign in to your suspended account.
  2. Enter a phone number to request a security code be sent to you via text message. This can be any phone that can receive text messages. 
  3. After the text arrives, enter the security code into the web page.
  4. Change your password to complete the unblocking process. Use the tips in Help protect your Microsoft account to create a strong password. 

Note: 

  • If you didn't receive your security code at the phone number you entered, try again or try a different number.
  • If your code doesn't work, make sure you enter the numbers in the body of the message, and not the numbers in the header.

ossiemac say:

I said "I don't want to start writing code if you don't want to use a macro". However, I thought it would be an interesting project so I have done it anyway. Whether or not you use it is up to you.

The code below will set up the DropDowns so that any values not previously selected will be available at the next selection point. It is event driven code and is invoked whenever you select one of the cells requiring the DropDowns.

You do not need to create the DropDowns; they are created from within the code.

There is an example workbook at the following OneDrive link but I will provide all the code and instructions here so it is up to you whether you download it. It is a zipped file to prevent it opening automatically with OnLine Excel which disables macros so download and unzip it.

https://1drv.ms/u/s!ArAXPS2RpafCmygEIKJ33b-utSLt

To install the code in your workbook.

  1. Right click the worksheet tab name
  2. Select "View code" (will open the VBA editor window at worksheet's module)
  3. Copy the VBA code below and paste into the VBA editor.
  4. Do not rename the sub
  5. If necessary, edit the line     lngRow = 2     to the number of the row where you want the Dropdowns to appear. (I have highlighted the comment against it in the code below)
  6. In the previous step you can select the row number but the code is currently set up to commence in column A of the selected row.
  7. Close the VBA editor (Cross very top right of VBA editor window)
  8. Save the workbook as Macro enabled.
  9. Ensure macros are enabled. See help for how to do this. (Option to "Disable all macros with notification" should be OK.)

Copy all of the below code and paste into the VBA editor.

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Dim lngCols As Long
    Dim lngRow As Long
    Dim rngPopulated As Range   'Range in Sheet2 that already has selection from the DropDowns
    Dim rngHeaders As Range     'Range containing headers in Sheet1
    Dim k As Long
    Dim rCel As Range
    Dim i As Long
   
    If Target.Cells.Count <> 1 Then Exit Sub        'Can only process single cell selection
   
    lngRow = 2  'Edit this line to the row number where the DropDowns are required
   
    On Error GoTo ReEnableEvents
    Application.EnableEvents = False
   
    With Worksheets("Sheet1")
        'Assign the range of column headers in Sheet1 to a variable
        Set rngHeaders = .Range(.Cells(1, 1), .Cells(1, .Columns.Count).End(xlToLeft))
        lngCols = .Cells(1, .Columns.Count).End(xlToLeft).Column        'Number of columns used in headers
    End With
   
    With Me     'Me is generic object for the worksheet to which the mocule belongs
        'Test if selected cell is in "lngRow" of sheet where values can be selected using DropDowns
        If Intersect(Target, .Range(.Cells(lngRow, 1), .Cells(lngRow, lngCols))) Is Nothing Then
            GoTo ReEnableEvents
        End If
        'Assign the range from "lngRow", Column A to the Target to a range variable
        Set rngPopulated = .Range(.Cells(lngRow, 1), .Cells(lngRow, lngCols)) 'From column A to last cell to be used
        '.Range(Target, .Cells(Target.Row, .Columns.Count)).ClearContents     'Clear existing values from Target to right
    End With
   
   
    k = 1   'Initialize for the elements in array
    ReDim arrList(1 To k)       'Initialize array with one element so that ReDim Preserve will work
   
    For Each rCel In rngHeaders
        If PrevUsed(rngPopulated, rCel.Value) Is Nothing Then
            ReDim Preserve arrList(1 To k)  'This does not alter array size on the first iteration
            arrList(k) = rCel.Value 'Assign the value to the array if value not already used
            k = k + 1   'Increment so that array can be incremented for next element (If any)
        End If
    Next rCel
   
    Application.CutCopyMode = False
   
    'Create new validation list excluding used values from the list
    With Target.Validation
        .Delete     'Delete existing validation
       
        'Create new validation
        .Add Type:=xlValidateList, _
                    AlertStyle:=xlValidAlertStop, _
                    Operator:=xlBetween, _
                    Formula1:=Join(arrList, ",")  'Creates a single string from array delimited with commas
        .IgnoreBlank = True
        .InCellDropdown = True
        .InputTitle = ""
        .ErrorTitle = ""
        .InputMessage = ""
        .ErrorMessage = ""
        .ShowInput = True
        .ShowError = True
    End With
   
ReEnableEvents:
    If Err.Number <> 0 Then
        MsgBox "Error occurred in module " & Me.Name & ", Private Sub Worksheet_SelectionChange." _
                & vbCrLf & "Report this to the project Administrator."
    End If
   
    Application.EnableEvents = True
   
   
End Sub


Function PrevUsed(rngToSearch As Range, varToFind As Variant) As Range
    'UDF to find if value in Sheet1 column headers has already been selected.
    Dim rngTofind
   
    With rngToSearch
        Set rngTofind = .Find(What:=varToFind, _
                        LookIn:=xlFormulas, _
                        LookAt:=xlWhole, _
                        SearchOrder:=xlByRows, _
                        SearchDirection:=xlNext, _
                        MatchCase:=False, _
                        SearchFormat:=False)
                       
        If Not rngTofind Is Nothing Then
            Set PrevUsed = rngTofind
        End If
    End With
End Function
