parameter query does not show all data within date range

I have a query that I added the criteria 'Between [Beginning Date (mm/dd/yy):] And [Ending Date (mm/dd/yy):]' to the date column.  When I run the query and enter a beginning date and ending date, I do not get the results for all the days in the range.  It leaves out the ending date no matter what date it is. If I take out the criteria, I get all the days.  I have another database in which I use the same criteria on the date and it works giving me all the days in the range.  What is the problem and how do I fix it?
hansv mvp say:

If the date field contains date + time values, you'd miss records with a time after midnight on the ending date. In that case, you could use

 

Between [Beginning Date (mm/dd/yy):] And [Ending Date (mm/dd/yy):]+1

 

or if you want to be very precise

 

Between [Beginning Date (mm/dd/yy):] And [Ending Date (mm/dd/yy):]+TimeSerial(23,59,59)
