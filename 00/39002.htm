The tool Windows driver verifier is commonly used in Microsoftanswers.

There are always problems with this tool....that is there will be a small percentage of users that are unable to turn off or recover from using the tool.

Another method to troubleshoot is to use memory dump files and scan the computer files for problems.

The memory dumps have more information that mini dumps but take longer to upload and download.

There are more commands that can be run on the dump files to find misbehaving drivers.

If the memory dump fails to find either misbehaving drivers or malfunctioning hardware or BIOS there may be information in the computer files:

msinfo32, event viewer, dxdiag, etc.

If there are still needs to perform testing with windows driver verifier the computer files can be backed up to another drive and a backup image can be made.

The memory dump reduces the use of the windows driver verifier tool and fewer OP are exposed to failure to turn off and recover from using the tool.

The scanning of computer files may also reduce the use of the windows driver verifier tool.

There are additional methods to turn off Windows driver verifier using the computer's Windows advanced troubleshooting menu:

a) startup settings > restart > safe mode with command prompt > type:  verifier /reset

b) command prompt > open a black box with title bar:  Administrator: X:\windows\system32\cmd.exe and prompt:  X:\sources> type:

verifier bootmode /resetonbootfail

c) system restore

d) startup settings > restart > disable driver signature enforcement

If none of the above work and if you have important files on the computer you may be able to use Ubuntu to backup files:

https://tutorials.ubuntu.com/tutorial/tutorial-create-a-usb-stick-on-windows#0

In addition you can try to use a Windows 10 iso:

Find a flash drive that you can format ( > or = 8 GB)

https://www.microsoft.com/en-us/software-download/windows10

https://www.tenforums.com/tutorials/9230-download-windows-10-iso-file.html

At times the bootable windows 10 iso can be used to turn off windows drive verifier.

If it is a boot loop then it will fail.

If it is not a boot loop then it may work.

If you are unable to turn off Windows driver verifier and were able to backup files the next step is a clean install:

https://www.tenforums.com/tutorials/1950-clean-install-windows-10-a.html

After a clean install Windows will install Microsoft and many non-Microsoft default drivers.

ron sommer mvp 2009-2017 say:

The Windows 8 Mail app does not have the optional setting to Leave a copy on the server.  That setting only applies to accounts setup using pop3 protocol.  The Mail app does not use pop3 protocol for any accounts.

 

Accounts setup in the Mail app sync with the account's server.  A copy of server messages are synced to your hard drive depending on an accounts settings.  3 days, 7 days, two weeks, a month, or any time are the available sync settings.
