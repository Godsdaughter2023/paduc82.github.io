2.       Double-click Background Intelligent Transfer Service.

3.       If the Startup status lists Stopped, click Start and then click OK.

4.       Double-click Windows Update.

5.       If the Startup status lists Stopped, click Start and then click OK.

6.       Double-click Workstation.

7.       If the Startup status lists Stopped, click Start and then click OK.

Well... I figured it out!

If you have two-factor authentication enabled on your Microsoft Account it causes an issue while adding the GLAS thermostat as a device registered with Cortana.

On the thermostat itself (while signing into Cortana), it appears to support 2FA. It triggers an authentication prompt in the authenticator app you're using (Microsoft Authenticator in my case). I just temporarily turned off 2FA, signed out of Cortana on the GLAS thermostat, rebooted it (just for good measure), signed back into Cortana on in, and followed the normal steps to get it working. Then I went back to my Microsoft Account and re-enabled 2FA.

Why did it initially start malfunctioning? I can only speculate that perhaps my authentication is only going to be valid for a set number of days.

Since Delta update and Cumulative update are available at the same time, it’s important to understand what happens if you deploy both updates in the same month. 

If you approve and deploy the same version of the Delta and Cumulative update, you will not only generate additional network traffic since both will be downloaded to the PC, but you may not be able to reboot your computer to Windows after restart. 

If both Delta and Cumulative updates are inadvertently installed and your computer is no longer booting, you can recover with the following steps: 
1.Boot into WinRE command prompt

2.List the packages in a pending state:

x:\windows\system32\dism.exe /image:<drive letter for windows directory> /Get-Packages >> <path to text file>


Example: x:\windows\system32\dism.exe /image:c:\ /Get-Packages >> c:\temp\packages.txt


3.Open the text file where you piped get-packages. If you see any install pending patches, run remove-package for each package name:

dism.exe /image:<drive letter for windows directory> /remove-package /packagename:<package name>


Example: x:\windows\system32\dism.exe /image:c:\ /remove-package /packagename:Package_for_KB4014329~31bf3856ad364e35~amd64~~10.0.1.0
