Even more strangely when the PPT2010 object file was opened in PPT2007 everything worked fine, except for the loss of features not backward compatible. It worked on a 2003 box too!

Our work around was to add a hyperlink to each inserted object linking it to the placeholder in the same folder. In the past the act of inserting the object wrote that hyperlink. The extra work on 1600 linked objects was not appreciated!

We have pored through the security settings, trust center, DEP, configuration etc etc without finding a solution. I have tried using Pack&Go, saving in different formats etc without any progress. We do many similiar conferences throughout the year and presentation management is a core service offering on most of them.

I could understand lack of compatibility between older versions, but you'd expect 2010 content to run on a different 2010 box!

I would appreciate any insight.

steve rindsberg say:

I'd suggest using links created a bit differently.

Instead of copy/paste/link, create an action setting, Hyperlink to, Other presentation and pick your presentation.

When you click the link, the other presentation will open, leaving your original "menu" presentation open invisibly "behind" it. When the other presentation ends or the user ESCs out, your menu is there waiting to launch the next presentation.

This is a much simpler form of linking than OLE links, and generally more reliable and more easily fixable via a bit of vba or other code.

Other than the method of creating the links, your workflow would be pretty much the same ... link to dummy presentations, then copy in the real ones as you receive them.

As to the way you're doing it now ... yep, it certainly SHOULD work.
