Closing the folder doesn't remove the duplicate data, it only removes the folder from the user interface, correct?

That's correct. Closing a notebook only deletes the locally cached copy.

Before closing any notebooks one should have found the reason for the "duplicates".

My "theory" is:

The original poster started with local notebooks (stored on his computer in the default folder for Onenote notebooks under Documents).

Later he shared these notebooks to Onedrive. He believed that this action moved the notebooks to Onedrive. This is (at least in Onnote 2010) plain wrong. Caused by a design bug in Onenote the notebooks were copied to Onedrive !

The old notebooks were left behind as orphaned children. And never get updated anymore.

During the installation of Onenote 2013 it looks at the default folder for Onenote to find and open existing Onenote notebooks. So it found the old versions of the notebooks now on Onedrive. Only duplicates concerning the name ...
