graham mayor say:
You create new documents from a template - File > New > Select the template. That template then remains attached to the document as long as the document can still see the template at its original location. If the document is moved to (say) another PC or the template is deleted then the document will be attached to the normal template.

If you create a new blank document then that is based on the normal template. There will always be a normal template.

macropod say:

When you create a document, it inherets all of the attributes of it's template - Styles, page layout, etc, though usually not macros. Unless the 'Automatically update document Styles' option is checked, the document need never look at the template again. Aside from macros, that means you can use whatever attributes the template gave the document, even if you move the document to a different PC, where the template isn't available. Conversely, if you later modify the template, those changes won't ordinarily be reflected in any of the documents created from it.

 

If you're trying to test for conformity between a document and its template, the template must be available for testing. Clearly, if ActoveDocument.AttachedTemplate doesn't return the same template name as Dialogs(wdDialogToolsTemplates).Template, you're not going to be able to do that comparison. Even, then, however, it's possible to fool Word if the document has been moved to a different PC with an entirely different template of the same name.

jay freedman say:
That could be an unsolvable problem, depending on how your document is set up.

Text boxes are, by default, "floating" objects that have no fixed position. (In VBA, they're members of the Shapes collection, along with all pictures that are not In Line with Text, all AutoShapes, and WordArt.) The only thing that gives them any ordering -- which one VBA considers to be "next" -- is where their anchors are. The anchor of a floating object is initially the start of the text paragraph that contains the insertion point at the moment the object is created, although the anchor can be moved later to another text paragraph.

It gets further complicated if the anchors of one or more text boxes are in a header or footer, because there is no absolute location associated with headers or footers. They have to show up on every page where that header/footer is visible, so they're sort of painted on from separate storage.

Worst of all is when two or more text boxes are anchored to the same paragraph, because even VBA internally can have trouble deciding what their order is.

So it boils down to the question "What do you mean by 'next', and how is that translated to VBA?" Where is anchor of the text box that you think of as "next" -- in the same header as the insertion point, in another header, or in the body text? If you can describe that, we can discuss the "how".
