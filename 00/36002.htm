skypeforlinux 8.11.0.4 in chroot jail with two skype profiles - Microsoft Community

phaoost say:

skypeforlinux 8.11.0.4 in chroot jail with two skype profiles

Hello,

I've been having troubles to run new skype on Linux in chroot jail and 4.3 version has been turned off by Microsoft.

Here are steps that allowed me to run skype, perhaps someone can find them helpful as I collected them from many sources

I run Devuan GNU/Linux Ascii, but it should be similar in other debian-based distros.

First, setup chroot somewhere. I did that at /src/schroot/amd64/ascii

I use schroot, here is part from /etc/schroot/schroot.conf:

[ascii]
description=Devuan ascii 64
directory=/srv/schroot/amd64/ascii
users=user
root-groups=root

I have added to /etc/fstab the following:

proc            /srv/schroot/amd64/ascii/proc           proc    defaults        0       0
sysfs           /srv/schroot/amd64/ascii/sys            sysfs   defaults        0       0
/dev/snd        /srv/schroot/amd64/ascii/dev/snd        auto    bind    0       0
/tmp            /srv/schroot/amd64/ascii/tmp            auto    bind    0       0
/run            /srv/schroot/amd64/ascii/run            auto    bind    0       0
/run/shm        /srv/schroot/amd64/ascii/run/shm        auto    bind    0       0

Mounted all mounts:

for i in $(grep schroot /etc/fstab|awk '{print $2}'); do mkdir -p $i; mount $i; done

Then went to chroot as root user:

schroot -c chroot:ascii

Installed the following packages: chromium gnome-keyring:

apt-get --no-install-recommends chromium gnome-keyring

This will allow to install all dependencies that aren't included for some reason in skypeforlinux package (perhaps chromium is explicit).

Then installed skype:

wget https://go.skype.com/skypeforlinux-64-preview.deb

dpkg -i skypeforlinux-64-preview.deb

apt-get update

apt-get --no-install-recommends -f install

Added user with the same UID/GID and user name:

groupadd -g 1000 user

useradd -u 1000 -g 1000 user

mkdir -p /home/user/.Skype

chown -R user:user /home/user

chmod 751 /home/user

adduser user audio

On main system I have added /usr/local/bin/skype

#!/bin/sh

xhost +local:
cd / && schroot -c chroot:ascii -u user /usr/local/bin/skype.sh

Then created /srv/schroot/amd64/ascii/usr/local/bin/skype.sh

#!/bin/sh

export DISPLAY=":0"
TZ=UTC skypeforlinux --datapath=/home/user/.Skype/skypeuser1
TZ=UTC skypeforlinux --secondary --datapath=/home/user/.Skype/skypeuser2

This allows me to run two skype instances at one time with 'skype' command (don't forget chmod +x on both scripts).

Replace or remove TZ override if you don't need it. Replace the mentioned user 'user' with your actual username and use appropriate UID/GID if you have other than 1000.

Any enhancements are welcomed.

Hi Louis,

The DISM Tool can only be updated every major update of Windows. It differs versus on your current operating system build because Windows 10 receives cumulative updates every time. Hence, changing the build number.

 

Example:

DISM = KB4043961 (OS Build 16299.19)

OS Build = KB4103714 (OS Build 16299.461)

Wherein, build 16299.19 is the initial version and also start of version 1709.

And, build 16299.461 is the latest cumulative of version 1709.

They will never match.

 

This means that the next DISM version will only be released in the next major updates.

The only changes on this tool is also related to what it can repair on every new features being made on every major Windows update release.
