z_a_s say:

<p>Is there a way to create a VBA Macro that will change a certain character in a spreadsheet to another character? For example, changing all "o" to "&oacute;". Or changing all "8" into "B"
andreas killer say:
You can perform a search & replace with a macro, see code below.

And please read this article for more details
https://docs.microsoft.com/en-us/office/vba/api...

Andreas. 

Sub Test()
  Dim SearchFor, ReplaceWith
  Dim i As Integer
  'All items to search for as array
  SearchFor = Array("o", "8")
  'All items to replace with as array
  ReplaceWith = Array("&oacute;", "B")
  'Execute each
  For i = 0 To UBound(SearchFor)
    Cells.Replace SearchFor(i), ReplaceWith(i), LookAt:=xlPart
  Next
End Sub

ken sheridan say: I'm afraid that is a really bad design.  By having multiple question and score columns data is being 'encoded as column headings'.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.

Study the model for my Questionnaire demo carefully.  You'll see that each question is represented by a separate row in a table (as your Questions table does correctly), as are answers.  The heart of the model is the Responses table which is where the 'chains' of relationship types from two directions meet, thus modelling each respondent's answer(s) to each question in each session of responses to a questionnaire.

While, as I said earlier, this is not directly applicable to your situation, it does illustrate the modelling principles and methodologies of which you need to have a good grasp before embarking on the design of a relational database.  With the current seriously flawed model you would find it very difficult to query the database and return meaningful information from the data.
