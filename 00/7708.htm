steve rindsberg say:

>> I choose actual size since I wasn't sure if most printers have a fit to scale option.

Fit to Scale is a PowerPoint option.  When you select it, PPT asks the printer "How big can I make this thing w/o anything getting cut off?" and scales based on the printer driver's answer.  

Totally independent of printer.

>> I also choose actual size since I thought at least the actual size I knew and this was something that would be a consistent across all platforms and equipment.

Not the case, though.  Since many printers can't print clear to the edges of 8.5x11 paper, setting that as the page size then printing actual size pretty much GUARANTEES that you'll get cut-off printouts on a substantial number of printers.

You might be better off setting the page size to PPT's standard letter size (which is a bit smaller than real letter size and pretty much guarantees that it'll fit on any letter size printer page.)

>>  If it is fit to scale then it depends once again on the printer what it thinks is fit to scale.

And once again, nope.  It depends on what the printer/driver knows about how close to the edges the printer can print, but who'd know better.

>> Even though I doubt it I am going to ask anyways...are there any printers that work better with MS programs and talk the same language more precisely? 

Not really.  Most programs don't really talk any particular printer language;  they send fairly standard output to the printer driver, which converts it to whatever language the printer speaks.

And even if there were a Perfect Printer and you had one, it wouldn't do you a lot of good unless everyone you give your documents to had the same printer and promised to use the same settings.

In a perfect world, eh?
