rick rothstein, mvp - excel say:
You are sort of mixing apples and oranges with your question. Notice that 36526 as a date is not 3/65/26 or any other combination of numbers and slashes and yet you seem to be expecting Excel to figure out that 112000 should become 1/1/2000 by simply inserting slashes. Excel stores dates as an offset from some predetermined "date zero". For Excel, that would be December 31, 1899 so that an offset of 1 is 1/1/1900 (the earliest standard date Excel will let you enter) and an offset of 36526 takes you to 1/1/2000. So, the reason 112000 displays the date as 8/23/2206 is because that is how many days offset from "date zero" that date is. By the way, for VB, "date zero" is December 30, 1899 (this has to do with VB recognizing that 1900 is not a leap year while Excel, apparently mimicking Lotus 1-2-3's mistake, assumes it is). As an aside, Excel could never covert dates written 112000 because, while the interpretation of that date is unique, a number like 1232000 would not be (is it 1/23/2000 or 12/3/2000?).

You can follow the steps below to disable desktop sharing for specific users using windows PowerShell:

 

a. connect to the skype for business PowerShell. (you need to install the skype for business PowerShell module and windows PowerShell 3.0).

b. run the following command, you can get some built-in policies in which the attribute enableappdesktopsharing is set to none.

get-csconferencingpolicy | where-object {$_.enableappdesktopsharing -eq "none"}

grant-csconferencingpolicy -policyname tag:bpossimponlynorec -identity
