ken sheridan say:

How can I build the query that list cell numbers only if there is no home number?

You can use a UNION ALL operation in which the first part returns only those contacts with a home number, and the second those contacts with a cell phone number, restricting the second part by means of a subquery to those who do not have a home phone number:

SELECT FirstName, LastName, PhoneNumber, PhoneType
FROM (tblContacts INNER JOIN tblPhoneNumbers
ON tblContacts.ContactID = tblPhoneNumbers.ContactID)
INNER JOIN tblPhoneTypes
ON tblPhoneNumbers.PhoneTypeID = tblPhoneTypes.PhoneTypeID
WHERE PhoneType = "Home"
UNION ALL
SELECT FirstName, LastName, PhoneNumber, PhoneType
FROM (tblContacts INNER JOIN tblPhoneNumbers
ON tblContacts.ContactID = tblPhoneNumbers.ContactID)
INNER JOIN tblPhoneTypes
ON tblPhoneNumbers.PhoneTypeID = tblPhoneTypes.PhoneTypeID
WHERE PhoneType = "Cell"
AND NOT EXISTS
     (SELECT *
      FROM tblPhoneTypes INNER JOIN tblPhoneNumbers
      ON tblPhoneNumbers.PhoneTypeID = tblPhoneTypes.PhoneTypeID
      WHERE tblPhoneNumbers.ContactID = tblContacts.ContactID
      AND PhoneType = "Home")
ORDER BY LastName,FirstName;

hansv mvp say:

You have to hide the userform before activating the document. And ListIndex starts at 0 while documents are counted starting at 1, so use ListIndex + 1. Or simply the (value of) the list box itself:

 

Private Sub DocumentNames_Click()
    Me.Hide
    Documents(Me.DocumentNames.ListIndex + 1).Activate
    Unload Me
End Sub

 

or

 

Private Sub DocumentNames_Click()
    Me.Hide
    Documents(Me.DocumentNames).Activate
    Unload Me

So now that we know you want to delete objects, I guess the big bug is that unselected objects get deleted unintentionally. Sub Delete()
