bob jones aka: cybertaz say:

If the document was converted from a PDF it's difficult to say from here what sort of artifacts you're having to deal with. Regardless of what they may have been originally, what they were turned into depends on the converter. Actually, the issue is not so much a what  as it is a where :-)

There's little doubt that the lines were converted to graphic objects. Graphic objects can be in only 3 places:

  • The body of the document as In Line or Floating
  • The Endnote/Footnote layer as Separators
  • The Header/Footer layer

If you can't select & delete them in the body of the document & they aren't recognized as endnote/footnote separators (have you tried using the Reset button in the Footnote/Endnote bar?) , the only place left to look is in the Header/Footer.

The anchored box in your last screen shot is typical of a bounding box that contains a floating graphic, but in this case the graphic has no visible Fill or Line. It also has no rotation handle so I'm sure it was generated by the PDF converter. Since you can select it you should be able to delete it. I pressing delete doesn't work, try using the del key if you have a full keyboard: If you have a condensed keyboard try fn+delete.

Don,

Without being able to login myself and without seeing your actual code it is very hard to offer any advice.  All I can say, and I know it doesn't help you, is that I have yet to meet a website I couldn't automate, but you do need to dissect them carefully.

Below is a sample of the type of code I'd use, but I can't take it any further since I don't have the necessary credentials.

'Required References, but could be easily converted to late binding
' - Microsoft HTML Object Library
' - Microsoft Internet Controls
Sub NAPALogin()
    On Error GoTo Error_Handler
    Dim IE                    As InternetExplorer    'Object
    Dim InputCtl              As HTMLInputElement    'Object

    Set IE = CreateObject("InternetExplorer.Application")

    With IE
        .Navigate "http://www.napaprolink.ca/prolink/welcome"
        .Visible = True
        Do While .Busy Or .ReadyState <> 4: DoEvents: Loop

        'UserName
        Set InputCtl = .Document.getElementById("username")
        If (Not InputCtl Is Nothing) Then
            InputCtl.Value = "MyUserName"
        Else
            Debug.Print "UserName input element not found."
        End If
       
        'Password
        Set InputCtl = .Document.getElementById("password")
        If (Not InputCtl Is Nothing) Then
            InputCtl.Value = "MyPassword"
        Else
            Debug.Print "Password input element not found."
        End If
       
        'Login "Button"
        Set InputCtl = .Document.getElementById("login")
        If (Not InputCtl Is Nothing) Then
            InputCtl.Click
        Else
            Debug.Print "Login button input element not found."
        End If
       
        Do While .Busy Or .ReadyState <> 4: DoEvents: Loop
       
    End With

Error_Handler_Exit:
    On Error Resume Next
    Set InputCtl = Nothing
    Set IE = Nothing
    Exit Sub

Error_Handler:
    MsgBox "The following error has occured." & vbCrLf & vbCrLf & _
           "Error Number: " & Err.Number & vbCrLf & _
           "Error Source: NAPALogin" & vbCrLf & _
           "Error Description: " & Err.Description, _
           vbCritical, "An Error has Occured!"
    IE.Visible = True
    Resume Error_Handler_Exit
End Sub

Hopefully, it can help you in some manner.
