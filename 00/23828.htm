When I said "in you[sic] code", that would mean separate from the declaration of the public variable.

I was presenting a concept.   You would change the value of bRun to false in your existing code that would cause the click event for listbox2 (so you would do it before the action that would cause the listbox2 click event to fire).   Then in the click event of listbox2, you would check for the value of bRun at the top of the event and if it was false immediately exit the event  (click event for Listbox2).   So you cannot stop the event from firing but you can avoid having it do anything when you don't want that to happen by using the bRun public variable.   That is what the code snippit was illustrating - setting the bRun variable to false, performing the action that would cause the Listbox2 click event to fire when you don't want it to, then setting bRun back to True. 

So to emphasize the last point, you would need to set bRun back to True when you want to allow the Listbox2 Click event to run.

cortig say:

The easiest is to use the Advanced panel to search for multiple tokens.

It's has a serious limitation though as you hinted in your question: It'll work automatically with AND , but doesn't support either OR or NOT.

There is a trick though (but it's not user-friendly): You can use the advanced option and enter a raw Spotlight query there. It will use the standard Spotlight raw syntax as published by Apple, and will support AND OR and NOT statements in the string.

You can find multiple examples of the syntax on the Web. Here's one: http://hints.macworld.com/article.php?story=20051010171330457
