Hey Matthew Wai,

 

You would have to find an event that only occurs when the computer isn't using a lot of CPU. I can't say this is impossible with 100% certainty, but I think it would be very challenging to find an event that fits that need.

 

The closest that I can find to setting a task to run when your CPU usage is below a certain measure is to set the task to trigger when the computer is idle. Grant the user permissions to start the COM component
