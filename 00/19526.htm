jay freedman say:

Changing the document to use relative paths instead of absolute paths to the linked files is only part of the solution.

If the linked PDFs are only on your computer, then anyone trying to use the hyperlinks will have to be able to access your hard drive. That means you must configure file sharing over the company network and give access permission to the other people. The hyperlinks would have to use the UNC (\\server\folder path\file) convention to address the location on your computer. These would be absolute paths. Also note that this would be useless for anyone who isn't connected to your network, for example a laptop user who has left the building, or a subcontractor who doesn't have access rights.

The alternative is to distribute both the document full of hyperlinks and all the PDF files they point to. In that case, the PDFs would be in the same folder as the document (or maybe in a subfolder of that folder), and the hyperlinks would be relative ones pointing to that location.

bill manville say:

I think there is an error in the macro in that article.
It should be as follows (which I have also renamed as Auto_Open)


Sub Auto_Open()
    Dim c As Range
    Set c = Cells.Find(What:="", LookIn:=xlValues, LookAt:=xlPart)
End Sub

 

This macro can be put in a standard module in any workbook that you open each time you open Excel.  PERSONAL.XLSB would be a suitable choice.


Just Alt+F11 to the visual basic editor, select the project of Personal.xlsb, Insert > Module, copy the code above and paste it into the code window that will have appeared.


File > Save

 

You can run the macro immediately by putting the cursor on it and pressing F5.

On future Excel sessions it should run automatically on startup.

hansv mvp say:

You can't, really. Microsoft hasn't updated the mail merge "engine" which dates from the last century, to work with formatted data.

You can create a query in Access that uses the PlainText function to convert the memo field to unformatted text:

 

Vanilla: PlainText([MemoField])

 

Then use the query as data source for the mail merge. That way, you'll get the text without formatting codes (but also without any formatting).

On Error Resume Next
        Set objWord = GetObject(, "Word.Application")

            Set objWord = CreateObject("Word.Application")
        End If

Function OpenWSP()

Dim objWord As Object
Dim objMergedoc As Object
Dim objDoc As Object
Dim objMergeResult As Object

On Error Resume Next
        Set objWord = GetObject(, "Word.Application")
        If Err.number <> 0 Then
            Set objWord = CreateObject("Word.Application")
        End If
        Err.Clear
    Set objMergedoc = objWord.Documents.Add("\\Bcp\bcp\cmcc\cmcdb\20080922_CMCDB2\Documents\WSP\wsp_title.dotm") 'replace your template here
         objMergedoc.MailMerge.WdMailMergeMainDocType = wdDirectory
         objMergedoc.MailMerge.OpenDataSource "G:\cmcdb\Employees\wsp1.docx", LinkToSource = True, ConfirmConversions = False
         objMergedoc.MailMerge.Execute
         Set objMergeResult = objWord.Documents("Catalog1")
          objMergedoc.Close wdDoNotSaveChanges
        objWord.Documents("G:\cmcdb\Employees\wsp1.docx").Close wdDoNotSaveChanges
        Kill ("G:\cmcdb\Employees\wsp1.docx")
        Kill ("G:\cmcdb\Employees\wsp1.html")
        objMergeResult.SaveAs "G:\cmcdb\Employees\" & ini & "\wsp_temp.docx"
        objMergeResult.Close wdSaveChanges
        Selection.HomeKey wdStory
        Set objMergedoc = Nothing
        Set objMergeResult = Nothing
        Set objWord = Nothing
        objWord.Visible = True
End Function
