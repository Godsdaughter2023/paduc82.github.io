I have found the following works for Excel 2011 (with update to 14.4.8 applied).

SOLUTION:

4. Do this copy and paste of a blank cell to another blank cell a second time. Again a delay.

Having copy and pasted twice, this should have resolved the problem. Check: Before closing the new blank spreadsheet, try repeating a copy and paste a third time, there should be no delay.

5. Click to delete this file, no need to save.

6. Open the excel documents you want to use. Save each one immediately after it has opened.

7. Now try a copy and paste of one cell to another. There should be no significant delay now.

Speculations and further comment:

It seems there is a fundamental coding issue and Excel needs to be cycled through copy and paste twice before the bug is resolved with every fresh opening of the Excel application.

The reason for opening a blank excel sheet first is that the time delay with the two copy and paste's to remove the problem will be much longer for any significant sized spreadsheet file. It seems if the Excel file is big, what ever loop Excel has to go cycle through twice, it trawls through the all the open Excel files to complete the loop.  So for a 150MB Excel file, it can be 10 or 20 minutes for each loop.  Most people interpret this as a permanent hang. Opening a blank Excel sheet makes this workaround a lot quicker. Once done, the issue is resolved until you next quit and reopen Excel.  In which case you have to do this again - each time!

It is possible this issue is related also to other major memory issues. With reading various posts, it does seem that Mac users experience of Excel or Word crashing or having other issues can quite commonly be caused by the downloading of fonts that Office apps like Excel and Word can not handle. Office apps like Excel and Word for most users automatically seek to import all the fonts, including newer user downloaded fonts. Since most people have at some time downloaded some nice creative groovy fonts onto their Mac, this offers explanation for possibly a lot of the crash issues Mac users run into.

The problem I encountered was the time involved with current solutions to test and resolve if 'problem fonts' are the cause of issues in Office apps.   

From what I could find, the current offered solution to test for 'problem fonts' is to re-install Office 2011 so one can select in the instal set up, use MS Office standard fonts only.  Then test to see if the problems continue.

One of the problems with this solution is the time it takes to re-install Office.  If one just re-installs Office over the current installation, I am not sure this gives the user the option to have Office only use its standard fonts to enable a test. 

To achieve this option effectively, I think one has to do a clean install, which requires first removing all Office 2011 files, then doing a fresh install, entering registration code etc. A further problem here re time is that there is no Office 2011 removal tool, like there was for Office 2008. Hence, achieving this kind of full re-install involves identifying all the Office 2011 files in many different places, including some out of the User's account.  This is a potentially very long involved process to do right, and one possibly beyond some new Mac users. I estimated about a days work. This includes re-installing office and restoring user fonts after you have removed the problem fonts. So that is two re-installs to get back to work. 

This brings Mac users struggling with crashes and other issues to a dead end, unless they really have the time to put a day aside to do this, and with no guarantee that 'problem fonts' are in fact the cause of their issue.  So it could be all for nothing!  Very hard to justify when you have deadlines!

thagstrom say:
Hello! I am an Independent Advisor, I would love to help you out!

Here is a link that has the information I think you are looking for:
https://support.office.com/en-ie/article/export...

Specifically this information from there:


Choose File > Open > Browse.

(In Project 2010, choose File > Open.)

In the Files of type box, select the file type that you want to import data from.

Navigate to the folder that contains the file you want to import, and then select the file in the file list.

Choose Open.

Follow the instructions in the Import Wizard to import the data that you want into the proper Project fields.

On the Task Mapping, Resource Mapping, or Assignment Mapping page of the Import Wizard, verify or edit the mapping assumptions of Project:

To import the data from a field in the source file to a different Project field, select the field in the To column, select a new field, and then press ENTER.

To change the name of a field in the destination file, select the field in the To column, and then type a new name.

To delete a field, select it in the From column, and then choose Delete Row.

To insert a new field above another field, select a field in the From column, and then choose Insert Row.

To remove all task, resource, or assignment fields from an import map, choose Clear All.

To add all task, resource, or assignment fields of the destination file to the import map, choose Add All.

Tip: On the Task Mapping, Resource Mapping, or Assignment Mapping page of the Import Wizard, under Preview, you can preview the layout of the import map.

To change the order of the fields in the destination file, select a field in the To column, and then use the Move buttons to move the field to the position that you want.

On the last page of the Import Wizard, choose Finish to import your data.
