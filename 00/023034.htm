Sub SetAllFooters()
  Dim startSheetName As String ' to find our way back to where we started
  startSheetName = ActiveSheet.Name

trevor rabey say:

Kim,

Imagine a river with debris in it, and dotted with small islands and rocks. Some of the debris goes through the gaps but some piles up on the upstream side of the islands and the more it piles up the smaller the gaps for the rest of it to go through. If the islands are large and the gaps between the islands is small and the debris is large as well....

Or you have a bucket of water with sand and rocks in it and you pour it down the sink, but the bigger rocks won't go down the drain. You have to break up the rocks.

Back to reality. You have roadblocks, I mean meetings. I am guessing recurring tasks, 1 day (or even 1 hour) duration each week, with resources assigned. And there are other "real" tasks with long durations with the same resources assigned to those.

To make it work so that the tasks will fit into the gaps between the meetings you need smaller tasks. Long duration tasks limit your options. Long duration tasks are also impractical from a project control point of view.

The joke goes "every project will be late by at least 1/2 the duration of the longest task", and it is not really a joke.

Break a 30 day task into 6 x 5 day stages.

Are you sure that you are making the distinction between duration and work? A 30 day task is 240 working hours, and if you have 60 hours of work then the resource must be assigned at 25% or 2 hours/day.

If the problem is still tight, things get more complicated and you need a bigger hammer.

Set priorities. I would have them decreasing from the earliest to the latest tasks.

If the meetings are recurring tasks then they will automatically get a (SNET) date constraint and a priority= 1000 when they are created, which means for levelling "do not delay this task". Don't change them or they cease to be recurring tasks.

Use the leveling order = priority, standard and check both of the option check boxes "Leveling can adjust..." and "Leveling can create...", but not "Level only within available slack".

After leveling, if there is still residual overallocation, use the resource graph view to see when it is occurring, and the Gantt chart, task usage and resource usage views to see which tasks and which resources.

The easy way is to not assign resources to meetings at all, and leveling will work without the priorities or the option checkboxes, which keeps it simple but you won't capture the resource's meeting hours or the cost of attending meetings if that's what you want. This situation proves what everyone suspects which is that meetings are not "real" tasks and they delay rather than facilitate real task progress.

Another way, to capture the meeting hours, is to create a duplicate of each resource, eg Kim 01, Kim 02 and assign one of them to the meetings and the other one to the real tasks. Then level.

smittypro1 say:

Hi Ben,

You can absolutely use Excel to create a fillable form like that, it just takes some time and planning to make sure everything gets aligned properly. This one of those rare occasions where it's OK to used merged cells, because you use them to create form fields. E.G. you might need to span multiple columns for entering information like Project Name, Location, Inspector Name, etc.

I'd start at the top, working from there down, merging cells and adding columns as needed. Probably the toughest thing is finding the right balance of columns, fortunately, you can have as many as you want. You can also use empty columns with a 1-1.5 width for spacing.

You can also look at using Form or ActiveX Controls, like check boxes.

Finally, I'd use Shapes for your section boxes, specifically rounded rectangles. You can combine 2 to get the effect displayed above, with a large one having just a border and no shape fill, then a smaller one (for the section title) having just shape fill and text. Once you get them aligned, you can select both and group them, so they move as one object. It's much easier to use shapes then try to get rows and columns aligned properly. Just drop a shape in and you're done.

To finish everything off, I'd select all of the input cells, unlock them, and then protect the sheet. That way, no one can mess anything up.

HTH

smittypro1 say:

The top one is simple, but I'd hold off on it until the end, when you know how many columns you'll have, otherwise you'll likely repeat this step.

1) Select row one for the full column width, then change the cell color (I'd start with Home > Styles > Accent1), but you can always modify it later to fit your company colors. Then add your report title, and set that as Title style (again, you can modify the style to fit). Now increase the row height to where you want it.

2) Select the row beneath it and go to Home > Styles > Accent3.

3) Repeat for the third row, just a darker gray.

Note that I mention using Styles several times. Styles are like a template, so you can apply them all over a workbook and if you decide you want to change a style's attributes, you just modify it. Once done, it updates throughout the entire workbook. It keeps you from having to manually change formats all over the place.

For the bottom section, you can repeat the steps for the top. But if you really want the "swoosh" look, you'll need to play around with shapes a bit. I'd get your form working first before going that far.

When you get a crash when using bullets, there are two main causes. Yes, there are other far less likely causes, but the first thing to do is to rule out these two main causes:

1. Font corruption/Font Cache problems

2. File system corruption

Tina's boilerplate reply actually does offer troubleshooting that might help narrow things down. Her #1 suggestion is to start your Mac up in Safe Mode. If your Mac has a current version of Mac OS, starting in safe mode runs Disk Utility in the background and fixes file system corruption. If you have an older version of Mac OS, you should run Disk Utility manually. That takes care of #2 above.

Starting in Safe Mode clears the font cache and enables a limited number of fonts. When you test the bullets in Safe Mode the crash may or may not reproduce. Then again, your bullets may look different if the font you used to make them is not one of the fonts active while in Safe Mode.  Safe Mode is an imperfect diagnostic tool for your purpose.

My advice is to do the Safe Boot to clear the font cache and run Disk Utility. Then do a regular restart and see if the crash is gone. If the crash is still there, then you probably have a bad or duplicate font. Use Font Book app (comes with your Mac) to validate ALL your fonts. Delete bad fonts and old versions of duplicate fonts. Then restart your Mac and empty the trash and test again.

ken sheridan say: I think you might have things the wrong way round.  It's the subreport which contains the names, not the parent report.  The process is:

1.  Create a labels report in the usual way, basing it on a query which returns distinct addresses.   Forget about the names for the moment; just include the address data.

2.  Create a report which returns just the names as a single line, but whose RecordSource includes all the other columns from the table.

3.  Back in the labels report in design view resize the bound control by dragging its top edge down to leave sufficient space for a single line.

4.  In the space above the bound control  embed the report created in 2 as a subreport, linking it to the parent report on all the address columns.  Set the CanGrow property of the subreport control in the parent report to True (yes) to allow it to grow if it returns more than one person at the current address.

Bear in mind that as the subreport control grows to accommodate multiple names it will push the address data below it down, so  if there were too many names at one address the bottom lines of the address will be lost.

At the time of the original post I did add a report, rptAddressLabels, illustrating the above, to my ClubMembership demo which you'll find you'll find in my public databases folder at:

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file was originally produced to illustrate two ways of producing reports with related data (members at each address) laid out horizontally, but it provided a good basis for a report of the type you have in mind.  The members' names in this are just first names, but that's not relevant to the design.

Hyperlinks leading to a file on a shared drive can sometimes cause some problems like the one you're experiencing.
