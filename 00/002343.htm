hansv mvp say:

The Trendline Type is an integer, not a text string.

If you specify the type as a literal string "xlLinear", Excel is smart enough to translate it to the corresponding integer, but that doesn't work if you use a string variable.

Try this:

Sub AddTrendline()
    Dim GraphName As String
    Dim TType As XlTrendlineType
    'Dim PO
   
    GraphName = Sheets("Control").Cells(11, 2).Value
    Select Case Sheets("Control").Cells(9, 2).Value
        Case "Linear"
            TType = xlLinear
        Case "Polynomial"
            TType = xlPolynomial
        Case "Exponential"
            TType = xlExponential
        Case "Logarithmic"
            TType = xlLogarithmic
        Case "Power"
            TType = xlPower
        Case "MovingAvg"
            TType = xlMovingAvg
    End Select
    ' PO doesn't get used...
    'PO = Sheets("Control").Cells(10, 2).Value ' Trendline Parameter
    Call UPS(GraphName, "6785")
    Sheets(GraphName).Select
    ActiveChart.PlotArea.Select
    ActiveChart.SeriesCollection(1).Trendlines.Add(Type:=TType).Select
    Call PS(GraphName, "6785")
End Sub

When you remove the underscore, Excel is reading it as 0 hours and 20 mins and not 0 mins and 20 secs as expected by you.

 

In the formula, repalce A2 by

TEXT(MID(A2,2,LEN(A2)-1)/60,"hh:mm:ss")

 

This will remove the first underscore character and convert the value to 'time'. I am assuming _2:30 means two minutes and 30 seconds.

 

Formula will be;

=ROUNDUP(TEXT(MID(A2,2,LEN(A2)-1)/60,"hh:mm:ss")*(24*60*2),0)*B2

=ROUNDUP(A2*(24*60*2),0)*B2

 

Excel represents dates as number of days from 1-Jan-1900 so 4-Mar-2013 is represented as 41,337. As a result 24 hours is equivalent to 1.

 

So to convert duration in h:mm:ss or h:mm you need to multiply it with 24*60 to get minutes. When you multiply it again by 2 you get the multiples of 30 seconds. Roundup function rounds it up to the next integer to give you number of 30 second pulses used. To get the cost you multiply by 'cost per 30 seconds'.

daniel pineault say:

Figured it out, and it's actually quite slick and at the same time a bit of a PITA because I'd say it is uncommon.

If you look at each toggle button frame, you'll notice that it is bound to QBxAccountNo Control Source, same as the QBxAccountNo combo box!

Then if you look at each toggle button, you'll notice that the Option Value is set to the corresponding AcctNo1 value.

So in fact, when you click on a button, you are in fact updating the QBxAccountNofield value, since it is tied to the QBxAccountNo combo box, it too is updated.  No code whatsoever.
