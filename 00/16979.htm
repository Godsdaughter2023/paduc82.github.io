pwatters say:

So I have discovered what the problem is...

When redirecting a message and both of the accounts (i.e. the sending and receiving accounts) are configured in Outlook, the user must ensure that the sender address is the sending account's address. 

In Outlook 2016 Preferences, I had the option "Reply and forward using the default email account" checked (see image below), so the sender and receiver were the same and this results in the redirect failing.

In previous versions of Outlook this didn't matter. I have now unchecked that preference so that the redirect sender defaults to that account, and it all works fine.

The phone as a device is linked to a Microsoft account for a number of purposes. To change that account you need to do a device reset. In fact, if you received the phone as a gift I would suggest doing a re-flash which will give you an 'out of he box'  and updated phone to start with.
