Connecting a Form to Two Separate Tables - Microsoft Community

chrism_118 say:

Connecting a Form to Two Separate Tables

Dear Access Community,

I'm currently working on an inventory database and I have a problem. I have multiple forms to enter new info into the database. One of these forms is a new inventory item form. When I made it I realized that because one input is connected to another table it will not allow me to complete the record entry. 

This input is a transaction ID which is connected to another table where its respective primary key is also transaction ID. Every time I add a new record the form creates an automatic transaction ID but it will not transfer this ID to the Transaction table so it violates the relationship since the Inventory table has transaction ID's that do not exist in the Transaction table.

Do any of you have any ideas on how to connect these tables or edit the form in a way that will allow me to enter new inventory items while simultaneously creating a Transaction ID for both tables?

Thanks!

 

john w. vinson say:

It would appear that your table design may need reconsideration. One to one relationships are quite rare! An Inventory item is one kind of Entity (real-life person, thing or event); a Transaction is a completely DIFFERENT kind of entity.

Are you asserting that each inventory item must have either zero or exactly one Transaction, never more? That would be very unusual if so. A normal Inventory app will have an Items table (with an ItemID as a primary key) related one to many to a table of Transactions, with a TransactionID primary key and a separate ItemID foreign key, linked to the Items table ItemID. Typically one would use a Form based on Items, with a Subform based on Transactions, using ItemID as the Master/Child Link Field; new TransactionID values will be filled by the Autonumber feature, and each new Transaction will inherit the main form's ItemID by the Master/Child feature.

If this does not fit your real-life business flow, please explain!

chrism_118 say: So what you are saying is that if I create a sub-form for transactions in the item entry form it may eliminate my issue? Also it's a one-many relationship.

john w. vinson say: Yes, if I understand your problem correctly, a subform should solve it. HOWEVER - you absolutely cannot use TransactionID as the Primary Key of both tables. That is the very definition of a one to one relationship. The Transactions table should have a TransactionID primary key, but the related table will have its own primary key.

..............will Access allow me to pick multiple items from the combo box  or do I need to do something in the property sheet to allow this?

To search for multiple items use a multi-select list box, not a combo box.  Unlike a combo box a multi-select list box does not have a value, however.  Instead it has an ItemsSelected collection, which is a collection of variants, each of which points to a selected row in the list.  You need to loop through the ItemsSelected collection in code to process the list.

You'll find examples in MultiSelect.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

You'll also find the use of a multi-select list box illustrated in SearchDemo.zip in the same OneDrive folder.  In this case the form allows you to return records (stores in the demo) which stock either all of the items selected, or any of them.  Again the data is returned as report, but equally well could be as a form.

Try the below steps:
1.      Click the file->print->print options->under print style
2.      Click "Page Setup," and then select the "Paper" tab. In the lower left hand corner of the Page Setup window, you'll see margin measurements for top, bottom, left and right printer margins for the paper.
3.      Change the margin values as needed. The Outlook default is often set at 1/2 inch (0.5") for each of the four margins.
4.      Click "OK" to save new printer margin settings and return to the page you want to print. Click print.
5.      Going directly to "Page Setup" located in the File menu is another option to change margins using a specific printer style.
6.      Select the specific style that comes closest to what you want to print, such as "Table" or "Memo."
7.      Choose the "Paper" tab again to find the margins in the lower left of the window. Change margins as needed. Click "OK" to close, and then "Print."
