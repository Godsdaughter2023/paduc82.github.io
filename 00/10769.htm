And that’s why the EPA publishes a startup script that automatically resets the Sleep timeout:

And that’s why I tried to simplify this whole thing, including the startup reset, by using desktop shortcuts; way back in 2012. This gives you the option to automatically reset the default Sleep setting at startup by dragging the reset shortcut into the Startup folder – just in case somebody forgets about “changing it back”.

To make this a little easier, you can use two desktop shortcuts to toggle between sleep timeout settings: Right-click on the desktop, select “New”, and then “Shortcut”:

For the “Full-Scan Sleep Timeout” shortcut enter this for the “location”:

powercfg -change -standby-timeout-ac 200

For the “Reset Sleep Timeout” shortcut enter this:

powercfg -change -standby-timeout-ac 20

Adjust the sleep timeout (in minutes) as required. You can also drag the “Reset” shortcut into the Startup folder to automate the reset. 

Original source:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/auto-scan-not-working/8041f4c9-1088-4a67-bca4-54c9bd5a99bd

You can also use this batch file to totally automate these changes for a Full Scan in the command line interface:

@echo off

 powercfg -change -standby-timeout-ac 200

 cd C:\Program Files\Windows Defender

 MpCmdRun.exe -Scan -ScanType 2

 powercfg -change -standby-timeout-ac 20

 Pause

But in order to fully automate the changes in the familiar user interface, you first have to copy the reset shortcut to your startup folder – and then use this batch file to extend the Sleep setting and start the scan:

@echo off

powercfg -change -standby-timeout-ac 200

"C:\Program Files\Windows Defender\MSASCui.exe" -FullScan

And last, but certainly not least, the Create Basic Task Wizard makes it a lot easier than you might think to schedule a Full Scan that never sleeps:

Open Task Scheduler (Start button > All apps > Windows Administrative > Task scheduler)

1.) In the right-hand (Actions) panel, click on Create Basic Task and name the task “Weekly WD Full Scan” and click Next

2.) Click on the Weekly option button and click Next, and then pick a time and day of the week and click Next

3.) We’re using the default action (Start a program) so just click Next

4.) Browse to (or just enter) "C:\Program Files\Windows Defender\MSASCui.exe" in the Program/script box (with quotation marks)

5.) Enter -FullScan in the Add arguments box (no quotation marks) and click next

6.) Check the “Open the Properties dialog for this task…” box and click Finish

7.) Change the account to "system" and then click on the Conditions tab, and check the “Wake the computer to run this task” box (this is what prevents the PC from sleeping) and Click OK

Insert a column of formulas into your data table, like

 

=COUNTIFS(A$2:A2,A2,B$2:B2,B2,...........,Z$2:Z2,Z2)=1

 

Expand the COUNTIFS to include as many fields as required to find invalid duplicates - then copy down to match your table.

graham mayor say:

While I agree with Doug that the best way to do this would be to store the data in a worksheet and read it into both documents as required, it would certainly be possible to read data from a document and write that data to another using a macro based process.

To aid that you could write the data to docvariables, bookmarks, content controls or form fields in the estimate document, all of which are easy to read later and write to the contract.

I would choose a fixed number of named content controls to present the data as they are fast to read - much faster than form fields for example. They maintain the document format and their values are easily read by macro.

The following macro and associated function will read the values from content controls in one document (the estimate) and write them to similarly named content controls (if present) in the active document (the contract). The code assumes text content controls in both documents.  For other types of content controls the code may need modification. The aim here is just to point a way.

Option Explicit
'Graham Mayor - http://www.gmayor.com - Last updated - 23 Jan 2018
Sub CreateContract()
Dim oDoc As Document
Dim oSource As Document
Dim strName As String
Dim i As Long, j As Long
    Set oDoc = ActiveDocument
    strName = BrowseForFile("Select the estimate to process")
    Set oSource = Documents.Open(FileName:=strName, AddToRecentFiles:=False, Visible:=False)
    For i = 1 To oDoc.ContentControls.Count
        For j = 1 To oSource.ContentControls.Count
            If oDoc.ContentControls(i).Title = oSource.ContentControls(j).Title Then
                oDoc.ContentControls(i).Range = oSource.ContentControls(j).Range
                Exit For
            End If
        Next j
    Next i
    oSource.Close 0
lbl_Exit:
    Exit Sub
End Sub

Private Function BrowseForFile(Optional strTitle As String, Optional bExcel As Boolean) As String
'Graham Mayor
'strTitle is the title of the dialog box
'Set bExcel value to True to filter the dialog to show Excel files
'The default is to show Word files
Dim fDialog As FileDialog
    On Error GoTo err_Handler
    Set fDialog = Application.FileDialog(msoFileDialogFilePicker)
    With fDialog
        .Title = strTitle
        .AllowMultiSelect = False
        .Filters.Clear
        If bExcel Then
            .Filters.Add "Excel workbooks", "*.xls,*.xlsx,*.xlsm"
        Else
            .Filters.Add "Word documents", "*.doc,*.docx,*.docm"
        End If
        .InitialView = msoFileDialogViewList
        If .Show <> -1 Then GoTo err_Handler:
        BrowseForFile = fDialog.SelectedItems.Item(1)
    End With
lbl_Exit:
    Exit Function
err_Handler:
    BrowseForFile = vbNullString
    Resume lbl_Exit
End Function

You may find - http://www.gmayor.com/BookmarkandVariableEditor.htm useful when creating forms and reviewing values.

{ IF estimate = "Y" "some text appropriate for the estimate" "some text appropriate for the contract" } some shared info { IF estimate = "Y" "more estimate text" "more contract text" } 

{ At the beginning of your document you have a field code like this

{ SET estimate = "Y }

This looks like a lot of steps, but you probably have to do quite a few of them anyway. But, to simplify, your process would consist of...

 a. create a new estimate//contract document with all the necessary text etc.

 b. fill in the values needed for the estimate

 c. save that document as "shared.docx"

 d. save it again as "estimate.docx"

 e. use ctrl-A and F9 to update all the fields in estimate.docx, then ctrl-A and shift-control-F9 to unlink all the fields. Unlinking replaces fields by their current results, so you end up with just the estimate text and none of the contract text.

 f. perhaps modify anything that does not need to be shared which is specific to this estimate

 g. save and close estimate.docx

When you want to produce the contract, you

 a. open shared.docx

 b. save it immediately as contract.docx

 c. change { SET estimate = "Y" } to { SET estimate = "N" }

 d. use ctrl-A and F9 to update all the fields in contract.docx, then ctrl-A and shift-control-F9 to unlink all the fields. 

 e. make any contract-specific modifications

 f. save and close contract.docx

 

(you can insert these fields by using ctrl-F9 to insert a pair of the special field code braces { } then typing everything else as normal between them. Select and f9 to update, Alt-F9 to switch between "code" view and "results" view.

Try it like this:

        

Sub FindDuplicatesInColumnNew()
    Dim lastRow As Long

    lastRow = Cells(Rows.Count, "A").End(xlUp).Row

    'Concatenate A and C in column D

    With Range("D2:D" & lastRow)
        .Formula = "=A2 & "" "" & C2"
        .Value = .Value
    End With
    
    'Find the duplicates and flag them
    With Range("E2:E" & lastRow)
        .Formula = "=IF(COUNTIF(D:D,D2)>1,""Duplicate"","""")"
        .Value = .Value
    End With
End Sub
