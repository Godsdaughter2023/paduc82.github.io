My best guess is that you are previewing the document, rather than "completing the merge," i.e. by merging to a new document then printing (usually the best option IMO), or merging directly to printer. To "complete," you need to click the appropriate button in section 6 ("step 6") in the Mail Merge Manager.

But let's check a few things:
 a. the merge should be a "Mailing Labels" type Merge (step 2 in Mail Merge Manager should tell you which type it is)
 b. for labels the mail merge main document typically consists of a table where each cell corresponds to a label. Each cell should have the fields that you want to print. For some label layouts, the table has what I call "label cells," separated by what I call "spacer cells". In that case, the label cells should be set up as I just described, and the spacer cells should be empty
 c. your Excel data should have 1 row for each address.

synapsis09 say:

Scratch the question, but you should add it to your knowledge base.

 

The answer is to right click on the text and select Remove Content Control.This will then allow you to change the headers for different sections.

 

The complete steps needed are:

  1. Go to Page Layout
  2. Select the appropriate Section Break from the Breaks dropdown menu
  3. Repeat as necessary until you have the desired number of sections (pages)
  4. Double-click on each Header except the first page and do the following steps:
  • De-select Link to Previous
  • Right-click within the text field
  • Select Remove Content Control from the menu
  • Edit your Header however you wish
  • Close your Header by either Close Header and Footer in the Toolbar or double-clicking anywhere in the document
  • Repeat as necessary for each Header

saltgrass say:

Many folks have discovered using a large hard drive for a recovery drive is not a good option.  Hopefully you have reset your external 1 TB drive back for use as data storage and are placing the System Image Backup there.

As David mentions, a small USB drive can be used for the Recovery Drive which can be convenient if you have a Recovery Image to copy over, which you do not.  But any media which can place you into the Recovery environment should be able to recover a System Image Backup.  I keep some of mine on an external drive, but the more recent ones on a second hard drive installed on the system.

You can have more than one WindowsImageBackup folder for different images, but you have to rename the non-current ones to something else so they are not duplicated names.  Only the one with the exact designation will be picked up by the Image Recovery process.

Have you checked whether the box gets checked when you save the document without closing? This would establish whether the problem results from saving or from closing (or, more likely, reopening) the document. This is a long shot, but there are two settings that you might experiment with:

  2. File | Options | Advanced | Web Options | Files: Update links on save.
