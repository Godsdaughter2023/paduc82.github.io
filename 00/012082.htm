charles kenyon say:

Don't bother with Gloria's link. It won't tell you anything you don't already know.

This can get complicated.

Each section of a Word document has three header and three footers, whether you see them or not.

Header/Footer Settings Recap

Every document has at least one section. The headers and footers for that section are in the section break. In the case of a single-section document that section break is the final paragraph mark.

It may be time for the Word Toolkit for Advanced Management of Headers and Footers. This is free and will tell you what is in every header and every footer in your document and take you to them.

First, though, you may want to display Sections in your status bar.

If your status bar is not showing Sections, right-click on it. I suspect you have more sections than you know, especially if you removed them manually rather than using a macro.

doug robbins - mvp office apps & services (word) say:

There maybe Continuous Section Breaks that you missed, on top of which, it can be hard to access the headers and footers of such Section Breaks.

However, running a macro containing the following code will remove all of the headers and footers from a document.

Dim i As Long, j As Long
With ActiveDocument
    For i = 1 To .Sections.Count
        With .Sections(i)
            For j = 1 To .Headers.Count
                .Headers(j).Range.Delete
            Next j
            For j = 1 To .Footers.Count
                .Footers(j).Range.Delete
            Next j
        End With
    Next i
End With
Exit Sub

Sub DeleteEmptyRowsInTable(ByVal T As Table)
  'Delete all empyt rows in the given table, can contain vertical and horizontal merged cells
  Dim C As Cell
  Dim R As Integer
  Dim Stack() As Integer
  ReDim Stack(0 To 0) As Integer
 
  With T
    'The FOR EACH visits the cells in ascending order
    R = 1
    For Each C In .Range.Cells
      'Is the current row greater then our row pointer?
      If C.RowIndex > R Then
        'Yes, we can delete the last row, save the row
        ReDim Preserve Stack(0 To UBound(Stack) + 1)
        Stack(UBound(Stack)) = R
        'Go one with the next row
        R = R + 1
      End If
      'Must we check this row?
      If C.RowIndex = R Then
        'Skip this row if the cell is not empty
        If Len(TrimWhite(C.Range.Text)) > 0 Then R = R + 1
      End If
    Next
    'Delete all empty rows from bottom to top
    For R = UBound(Stack) To 1 Step -1
      'Note:
      '  .Rows(Stack(r)).Delete

    Next
  End With
End Sub

Private Function TrimWhite(ByVal S As String) As String
  'Return a string with white space removed
  Dim i As Long
  For i = 1 To Len(S)
    If Asc(Mid$(S, i, 1)) < 32 Then Mid$(S, i, 1) = vbNullChar
  Next
  TrimWhite = Replace$(S, vbNullChar, "")
End Function

john mcghie say:

It's Mrs Barnhill, she's about 80 in the shade, and she has a very irritating habit of being right :-)

If I may explain: the setting you make for the Margin is in fact the minimum the margin can be. Word will add enough space to that measure to contain your header and footer.  It moves the margins inwards towards the middle of the paper to do that.

If you set a 1 inch margin, that's 72 points.  If you then have two blank lines and a page number in the footer, that's another 36 points (half an inch) that Word will move the margins in, to ensure the header and footer have room to print.  (If you set your Normal style to double-spacing, every other style will inherit that, so each line is 24 points high not 12, so you would add a whole inch.)

If you make sure there is only one paragraph in the footer and put your page number in that, then set the Margin at 1" and the "From edge" distance to 0.5" you should get what you want: the top of your footer will now be 36 + 12 = 48 points from the edge of the sheet and fully within the margin.

But it may not be that at all.  Go into Word>Preferences>View and turn on "Text Boundaries", so you can see where the margins actually are.  You may find that you have paragraphs that are not printing at the top margin because there is a hard page break before them, or not printing at the bottom margin because there is too much space after them.

If you want paragraphs to align correctly at the top of each page, let Word do the pagination for you: don't use page breaks.  Similarly, you may want to check the styles you are using to see how much space has been assigned after the paragraph.  It should normally be 0 for double-spacing, but it pays to check.

It's useful to use the built-in style "Body Text" for your text and leave Normal set at 0 everything.  Word uses the Normal style itself, in graphics and tables.  You can customise the Body Text style to your tastes without having it "infect" every other style in the document.

Hope this helps

ken sheridan say: Reports, like forms, have an OpenArgs property which can be used for passing values into the report.  This means the form does not need to remain open.  Let's say you have a text box named MyTextBox on a form, you can open a report from a button on that form and pass the value in MyTextBox to it, e.g.

DoCmd.OpenReport "MyReport", View:=acViewPreview, OpenArgs:=(Me.MyTextBox )

In the report you can then assign the value to a control, so if you also have a text box control in the report to which you want to assign the value just set its ControlSource property to:

=[Report].[OpenArgs]

You can pass multiple values to a form or report by parsing the OpenArgs property.  You'll find an example as Args_2k.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

This passes values to a form rather than a report, but the method is exactly the same.  It uses a module which was originally developed by Stuart McCall many years ago in the old CompuServe MsDevApps forum, and later extended by me in response to a post there to allow for named or numbered arguments to be passed.
