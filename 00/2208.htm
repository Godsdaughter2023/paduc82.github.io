It sounds like she had the file open when excel closed abnormally, either a crash or the power was turned off on the computer. This leaves the "lock" file behind. It is this lock file that is causing the problem.

On her machine, with no spreadsheets open, in Excel, go to File menu > Options command > Save option.

Make note of the "Default local file location". Then open that folder on her machine.  Look for a spreadsheet file with a name similar to the problem file replacing the first 2 letters with ~$.  Delete it, and any others you may find.

You could even do a global search on her machine for files with that naming convention and clean them all up.  Or you could go old school and use this DOS command to delete all of these temp files:

del /s ~*.*,*.tmp

jay freedman say:

Any sort of "click here to show more" button in a Word document will involve a macro. (The ones you see online are part of the scripting in web pages, and they aren't available in Word.) For people who have no background in Word, getting though the thicket of enabling a macro to run may be even harder than explaining about a right-click.

The fundamental problem is that macros and active controls of all types are commonly abused by evildoers (yes, there is such a thing as evil). Microsoft's response to that is to disable macros unless specific criteria are met (e.g., a certificate from a "trusted publisher"), or the macro-containing document is stored in a "trusted location". These requirements aren't burdensome for users who know what they're doing, but they're even more mysterious to beginners than the idea of a command on a right-click menu.
