Solving the endless Windows Update problem for already installed - Microsoft Community

bogey1962 say:

Solving the endless Windows Update problem for already installed updates (KB979909, KB982168, KB974417) and most likely any other update.

OK, I contacted MS via their 1-800 number regarding the problem I was having with Windows Update.

Basically, Windows Update kept popping the shield up on the taskbar nagging me to install updates that had already been successfully installed (KB979909, KB982168, KB974417 - in my case, but I'll bet that this will work on any updates.)

Disclaimer: I am using Windows XP Pro and this procedure works flawlessy. I take no responsibility for you trying this on a Windows XP machine if you aren't remotely tech savvy with a computer or if you try it with any other MS operating system.

On to the fix...

If Windows keeps bugging you to install updates that have already been installed successfully, you need to rename these directories:

Windows/SoftwareDistribution
Windows/System32/CatRoot
Windows/System32/Catroot2

Just rename them with a .old extension tacked on to the end of their names.

Access will be denied for renaming them, so you need to perform these steps:

From the Start menu select Run

Type SERVICES.MSC and then hit the ENTER key

From the list of services in the newly opened window, Stop these services one at a time by selecting them from the list and clicking on the STOP icon at the top of the window:

Automatic Updates
Background Intelligent Transfer Service
Cryptographic Services

Now you can rename the directories listed above so that in the end you will have these new directories created (which are just the old ones renamed):

Windows/SoftwareDistribution.old
Windows/System32/CatRoot.old
Windows/System32/CatRoot2.old

Go to Windows Update via Internet Explorer and hide the updates that keep recurring by selecting "Don't show this update anymore", or something to that effect, for each of the updates. Windows Update will still warn you that you have hidden critical updates, but the desktop won't bug you about them anymore.

Reboot the computer and the renamed directories will be recreated.

Supposedly, MS is working on this issue, but this is a workaround that solves the issue for now.

If the problem does not occur, repeat step 1 and step 2. In step 2, select only half of the remaining check boxes that are cleared in the Service list. Repeat these steps until you have selected all the check boxes.
