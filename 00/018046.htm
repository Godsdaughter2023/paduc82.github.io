The straight answer to your question is no, what you're experiencing is by design. davidfsi say:

You can setup an "on insert" AND "on update" Data Macro which looks up the values from the originating tables and have the macro "Set Field" of 'Transfer Ref' to the concatenated value.

In your case that could look like:

in your table design view under the design tab, you will see on insert and on update macros buttons.

In your macros, Create 2 parameter that passes the From and To Field Values from [Wire Transfers] Table. Ie -  [PFrom] and [PTo]

Then...

Lookup a record in [From Branch] where PFrom = [From Branch].[ID]

set return Variable = [From Branch].[BranchNumber] (or whatever the field name that has the branch data your looking for)

then...

Lookup a record in [To Branch] where PTo = [To Branch ].[ID]

set return Variable = [To Branch].[BranchNumber]

Finally, Set the macro to set the field [Transer Ref] to concatenate the two return variable values.

Note: Make sure you do this for both On insert AND On Update events. otherwise, any edit to either field will not run the macro that updates the [Transfer Ref] Field.

Hope this was helpful
