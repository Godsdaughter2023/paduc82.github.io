arjay dom say:

Hi,

Windows.old folder contains files that give you the option to go back to your previous version of Windows and deleting this folder cannot be undone.

If you're confident that you will no longer revert back to the previous version of Windows, and you have successfully back up all important data to an external hard drive, you can delete the files manually:

  1. In the search box on the taskbar, type settings, then choose it from the list of results.
  2. Select System > Storage > This PC and then scroll down the list and choose Temporary files.
  3. Under Remove temporary files, select the Previous version of Windows check box and then choose Remove files.

You can also delete the folder through Windows Command Prompt:

  1. Press Windows key + S.
  2. Type CMD.
  3. Right-click on Command Prompt and click Run as administrator.
  4. Enter the administrator credentials.
  5. In the Command Prompt window, enter or copy and paste this command:
    RD /S /Q %SystemDrive%\windows.old
  6. Press Enter.

Some questions on summary tasks and constraints - Microsoft Community

jnbrown say:

Some questions on summary tasks and constraints

-Is there any case when it is practical and useful to define a constraint for a summary task ?
-Why are there only 3 types of contraints available for a summary task ?
-MSP changes the constraint type to As Late As Possible only for summary tasks when I reverse teh
direction of scheduling. Why doesn't MSP change it for subtasks ?

Thanks

jim aksel say:

Although tempting to place constraints (or predecessors) on summary tasks, this quickly leads to logic problems and the inability to properly analyze the schedule. It is generally considered poor technique to use these features on summary tasks.

 

Sometimes, during the very initial stages of a program, predecesors and constraints are applied to very preliminiary schedules.  As the details are flushed out, these tasks end up becoming summary tasks.  At this point the constraints on the "new" summary tasks get removed to observe how the details shift. The scheduler then works with others to determine more proper relationships among the details.

 

The Project Management Insitutre "Practice Standrad for Scheduling" addresses this issue and recommends against any date constraining or logic constraining of sumary tasks.

julie sheets say:
Hello JNBrown,

  1. I can't think of a reason why I would constrain a summary versus a subtask.
  2. The start and finish dates of Summary tasks are determined by the start and finish date of the subtasks.  Therefore the constraints of Must Start On, Must Finish On cannot apply.  In a project scheduled from a start date, the two constraints available are Start no Earlier Than and Finish No earlier than which are the constraints that are applied if you type in a start date or type in a finish date.
  3. It will change default constraints if you switch scheduling "direction" before entering tasks.  You can, apply the same constraint to all subtasks using the Multiple Task Information dialog.
Julie

susansaundersvt say:

When you create a new blank document, is it also in Compatibility Mode? If so, you are most likely using a Normal.dotm template that was saved in 2010 or earlier format. You can fix it following these steps:

  1. In Word, press Alt and F11 at the same time. The VBA editor opens.
  2. Look for the Immediate window at the bottom of the VBA Editor. If it's not open, press Ctrl and G at the same time.
  3. Paste this command into the Immediate windows and press Enter:
    application.NormalTemplate.OpenAsDocument
  4. With Normal.dotm open in Word, choose File>Save As and choose the Current Folder. You will see a dialog:
    Your document will be upgraded to the newest format. Etc., etc.
  5. Click on OK, then close Normal.dotm and create a new blank document. The [Compatibility Mode] warning should be gone.

Older document saved in 2010 or earlier format will still say [Compatibility Mode] until you use File>Save As to resave them in 2013 format.
