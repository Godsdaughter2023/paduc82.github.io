cmdrkeene say:

It's not related to TrustedInstaller, but it is related to directory virtualization that first came into existence in the Vista days.  If an application tries to save data to a place that is supposed to be a system location, such as inside Program Files or the Windows folders, Windows 7 will automatically redirect that to a safe location in the user's profile, but it tricks the program into thinking that the save really worked.  It does this to maintain compatibility, and is mostly for programs that automatically save data, not word processors where the user gets to pick.   But Windows 7 doesn't differentiate, it'll still redirect saves from any program.

The real recommended solution is to only save within non-system locations.

When you install a program, Windows will automatically elevate the installer application to allow it to write to the directory.  The yellow padlock icon isn't indicative of that however... the lock icon only serves to signify that the file is not shared.  You can do this to any file by right-clicking and choosing "Share with: No one".

As for the virtualized directory, yes.  The file is actually being saved to this type of path:  C:\Users\Shawn\AppData\Roaming (replace Shawn with your real username).

cmdrkeene say:

1 - It really shouldn't be exclusive.  In fact if the application is running with administrative rights, it should have full control over the directory.  The only time programs are prevented from writing there is if the program is running in a limited context, such as a limited child account or something.  This is just to protect the integrity of the program files from tampering or attack. You're right that XP didn't do this, Vista was the first.

2 - As the owner, you really do have full control, no real special set up is required.  One thing that you might want to try (and this shouldn't be necessary, but it's worth a shot)... open Task Manager (Ctrl-Shift-Escape) and find the process for the program.  Right-click and un-check the UAC Virtualization option... and then try to save your files there again.

3 - Oh, I think the headache is about it.  This drove me nuts just yesterday trying to change a graphic in program files folder.  I opened the image for editing in my graphics program (Adobe Fireworks) and when I saved the file (not save-as... I mean I really just tried to save the same file I opened), it went somewhere else.  Took me probably 5 minutes to find it in that roaming folder.

 

To be honest I can recognize your frustration and offer these tips, but there's not much I know to change it.
