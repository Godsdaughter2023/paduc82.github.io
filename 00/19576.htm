    Dim strCriteria As String

scottgem say:

This is one of the very few exceptions to the rule against redundant data. When you have a value that is time sensitive you want to preserve the value at the time the record was created. 

First, your tables need some work. Do not accept the default name for your Autonumber fields. You should always use meaningful names for your objects. I always name my primary Key fields as tablenameID  (you might want to use tablename_ID since you seem to prefer underscores to camel notation i.e. Inventory_ID). Then your foreign keys should be similarly named to visually indicate your relationships.  

You should NOT have Product_Name in the tracking table. That is redundant. Instead, you should have Inventory_ID as the foreign key. So the link between the mainform and subform should be Inventory_ID. 

 

Ken gave you one way to do what you want, but I think Ken missed something. Given the way you have your form setup, You would never enter the product name since the FK fills in automatically, so the After Update event wouldn't fire. Since your main form has the current price, I would just set the Default Value of the Price control on the subform to:

=Forms!mainformname!Price

When you add a new record, all you have to enter is the Order_Date. The AutonumberID, Inventory_ID and Price will fill in automatically. Note, the Inventory_Tracking_ID and Inventory_ID controls can be hidden so all the user sees is the Order_Date and Price. They can see the Product name from the main form. 

scott mcdaniel say:

But this isn't redundant data. The price/cost of an item at the time of sale/purchase of that item is an attribute of that transaction, and should be stored with that transaction. Filling that value with a default value is another matter entirely.

doug robbins - mvp office apps & services (word) say:

Go to the Developer tab of the ribbon and from the Controls section, insert either a Rich Text Content Control

or a Plain Text Content Control

Use the former if you want to apply particular formatting (Font colour, bold, italic, etc) to the text that the use inserts into that control.

To learn more about the use of Content Controls, see the following page of Greg Maxey's website:

http://gregmaxey.com/word_tip_pages/content_controls.html
