bob jones aka: cybertaz say:
Those keyboard shortcuts don't work for zooming in Word because they are assigned to other Word features:
Command+= is assigned to Subscript
Shift+Command+= (Command+plus) is assigned to Superscript
Command+- inserts an Optional Hyphen

As for Word, there are 2 separate considerations:
  • Existing documents always open at the Zoom that was in effect the last time the document was Saved & Closed. There is no Preference to cause them to open at a specific Zoom level. If you search the Community there is a macro available to force a specific Zoom if you choose to use it.
  • New documents can be created at a user-specified Zoom level by opening the Normal.dotm template & setting the Zoom there. You also need to type a <space> or or other character, then delete it (don't use Undo). Save & close the template. New documents will then be created at the level you set.

The 1st consideration above applies to PowerPoint as well, but new files is a different story. You cannot access the program's default template. Initial Zoom is determined by the template used to generate the new file but you'd either have to use a macro or create your own template & base new files on it.

To change the Zoom percentage you don't need to rely only on the keystroke. Zoom in a Word or PowerPoint document can be set using:
The Zoom tool on the Standard Toolbar,
The View> Zoom command from the main menu.

In Word you can assign keyboard shortcuts to any of the ViewZoom___ commands, but PowerPoint doesn't support custom keyboard shortcuts.

doug robbins - mvp office apps & services (word) say:

If you send me a copy of the mail merge main document referencing this thread in the covering email message, I will investigate the issue with the use of my Merge Tools Add-in.

Or, the following macro should print the document created by executing the merge to a new document (Edit Individual Documents) as a series of separate print jobs, one for each record so that each would be stapled together.

Sub PrintMailMergeAsSeparateDocuments()

Dim i As Long

With ActiveDocument

    For i = 1 To .Sections.Count 

        .PrintOut Range:=wdPrintFromTo, From:="s" & i, To:="s" & i

    Next i

End With

End Sub

If the mail merge main document contains more than a single section, you would need to use 

Sub PrintMailMergeAsSeparateDocuments()

Dim i As Long

With ActiveDocument

    For i = 1 To .Sections.Count Step #

        .PrintOut Range:=wdPrintFromTo, From:="s" & i, To:="s" & i + # - 1

    Next i

End With

End Sub

where # is the number of sections in the mail merge main document.

erin breaux say:

How do change a file from Publisher layout to regular?

Split from this thread.

John,

How do I reformat "these as a regular Word document in Print Layout"? As in, how do I save them in the regular print layout format so that it doesn't remind me that the file is in old publishing layout every time I open the file in 2016?

Thank you,

Erin 

john korchok say:

This is definitely easier to do if you still have access to Word 2011. Using it, you can see what the original file looked like in Publishing Layout view. Then open another copy in Word 2016 for Mac. 

In Word 2016, choose Word>Preferences>View and in the Show Non-Printing Characters section, check All. This makes the section breaks visible. Publishing Layout documents can have lots of section breaks.

All material that was in a "Master Page" is actually contained in a Header. Separate but adjacent sections of a document can share a header as long as the second and subsequent sections have the Link To Previous attribute turned on. Turn off Link To Previous where you want a header to change. IMO, managing the section breaks and linking or unlinking the headers is the only tricky part in transitioning documents to Print Layout.

All material that was in the All Contents section of the Publishing Layout document is now on the pages of the document. All text is in text boxes. You can leave the text in boxes (that's a more familiar approach for InDesign users), or put some on the main text layer if that works better for your design.
