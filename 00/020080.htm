FIXED! For the benefit of anybody else who sees this, here are the details:

Turns out this is a bug in Samba4 (#11097) that was revealed as a result of a security update in Windows 8.1. If you just want to fix a single client then there's a registry workaround detailed in KB3000850.

Word is definitely backward-compatible. In particular, headers and footers work the same in Word 2019/365 as they did in previous versions.

The fact that you could fix your document by copying content into a new document suggests that the original document had acquired some form of corruption. To be more precise, you can copy the content minus the final paragraph mark (¶) into a new blank document (see also https://support.microsoft.com/en-us/help/918429/how-to-troubleshoot-damaged-documents-in-word).

Note that if "master template" means that you are using Word's master document feature, you should be aware that it has been unstable for years (see https://wordmvp.com/FAQs/General/RecoverMasterDocs.htm). There are other ways to combine different documents into a "master," for example, by making use of INCLUDETEXT fields.

Well, what Rich suggested certainly is an easy approach, but it will not convert color text to grayscale, which is what you said you needed to do. It will cause all text to be rendered in black (or whatever is the default color based on the Theme used in the document).

For Pictures, Text Boxes & other Drawing Objects - provided they are not set to In Line with Text - you can use the Selection Pane [Layout tab] to select multiple objects throughout the document. All then can be recolored at once (which may necessitate a font color change as well).

bob jones aka: cybertaz say:

It's impossible to give a straight answer because there are numerous variables involved, starting with whether the documents are strictly text or contain color images. Other factors how the color was applied to the text, which sets of colors were used, etc. There is no "magic button" to convert a Word document to grayscale :-) Some general points to start with:

  • If text has been colored using Word's Themes, there is a Grayscale Theme which can be selected. That will convert all of the Theme Colors to grays [including Drawing Objects, SmartArt, etc.], but it will not change any text coloring that was not selected from the main palette of shades. Nor will it affect the coloring of photos or other inserted graphic objects.
  • If Styles were used, the Styles can be modified to substitute gray for the original color. Also, you can create a style that uses a shade of gray, then use Find & Replace to replace the colored styles with the gray style.
  • Find & Replace also can be used to replace a font color with gray, but I don't know whether it can be done in one fell swoop... I believe each color would have to be searched for individually.
  • In-Line color images would have to be edited individually unless you have someone capable of writing a macro.
  • Color images which have Text Wrap applied can be multiple-selected in order to reformat the color all at once.

The simplest approach I can suggest right off is to generate a PDF using the printer settings in the Print dialog if the driver supports that capability.
