don varnau say:

Hi,

Your question is more complicated than you think.  ;-)

-- - -- - --

If these conditions are met you will see 64-bit Manager Processes and (mostly) 64-bit Tabs. Some websites will still invoke 32-bit Internet Explorer for add-on compatibility.

1. Control Panel> Action Center, Change User Account Control Settings, verify that the slider is not set to "Never notify." If User Account Control is set to "Never notify" Protected Mode and Enhanced Protected Mode can't be enabled.

2. Internet Options> Security, verify that Protected Mode is enabled.

3.  Internet Options> Advanced> check "Enable Enhanced Protected Mode."

So, a website requiring 32-bit IE should be able to invoke a 32-bit tab.

A program/application that requires 32-bit IE should be able to do the same.

-- - -- - --

> 3.  Internet Options> Advanced> check "Enable Enhanced Protected Mode."

If you uncheck 'Enable Enhanced Protected Mode" you should be, in effect, disabling 64-bit Internet Explorer, so that opening IE from

["C:\Program Files (x86)\Internet Explorer\iexplore.exe"]

instead of

["C:\Program Files\Internet Explorer\iexplore.exe"]

gives you 32-bit Internet Explorer.

-- - -- - --

judeds say:

Windows Defender not Starting

Hi i am using Windows 10 Pro, and my Windows Defender Live threat service wont start, can you some one help me to solve the issue, i am not using any antivirus software. but i am using Advance systemcare 11.1 

aerdian say:

If you have never had another AV program installed, you could possibly have malware that is turning off Windows Defender.

Some recommended secondary scanners you may be able to use without a possible malware infection turning it off would be the following:

Malwarebytes: https://www.malwarebytes.com/free/

Zemana Antimalware: https://www.zemana.com/

HitmanPro: https://www.hitmanpro.com/en-us/hmp.aspx

If you formerly had an AV program installed, try finding any leftover files that may be causing Windows Defender to believe an AV program is still installed. A simple fix may be to look in Task Manager to find any files that are running that are related to that AV and disabling it.

If you can't find an infection and you have never had another program installed, try restarting the service:

  1. Press Windows key and R.
  2. Type "services.msc" and press OK.
  3. Right-click on Security Center and choose Restart.
  4. Right-click on the Windows Defender Antivirus Service and select Start.

You could also try some troubleshooting in the troubleshooting area of settings.

If none of these options work, you can try this: https://www.tenforums.com/tutorials/5918-turn-off-windows-defender-windows-10-a.html#option2

greginmich say:

If you're using Version 1709, then it's possible that this error message means that Windows Defender has been turned off via Group Policy. This is the screenshot that I submitted to the Feedback Hub to demonstrate that the Windows Defender Security Center app wasn't recognizing this setting properly: 

https://aka.ms/AA1aijy

 

So you should run this REG command in order to remove any Group Policy entry that might have disabled Defender. Type “cmd” in the search box; right-click on Command Prompt, and select Run as administrator; and then copy, paste, and enter this command:

REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware 

If that doesn’t resolve the issue, then follow the steps that I’ve provided in this thread:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/windows-defender-the-threat-service-has-stopped/ff4395cf-d93c-4e1c-a6fc-d14cdcaf8bc7

If you can’t get Windows Defender up and running, then you’ll either have to resort to the recovery options, beginning with a System Restore to a point prior to the onset of the issue (if you insist on using Windows Defender):

https://support.microsoft.com/en-us/help/12415/windows-10-recovery-options

Or else install a free third-party AV app for real-time protection so that you’ll be able to connect to the internet safely:

https://www.pcmag.com/article2/0,2817,2388652,00.asp
