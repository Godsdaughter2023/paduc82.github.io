ken sheridan say: The Timesheet demo is, like most files in the OneDrive folder, neither a tutorial nor a template; it is merely an illustration of various methodologies for undertaking common tasks.  Like all my demos it assumes that anyone using it will have sufficient technical knowledge to be able to extract and adapt specific parts of the database for use in their own applications.

In your case the two forms used for the clocking in/out part of the demo contain all the functionality you'd need.  As you apparently want to enter the current actual time, rather than rounding up or down to a time slot of fixed duration, the code can be very easily simplified to insert the return value of the Now function into a column position in the relevant row in the tblAttendance table.  Which of the two columns data is inserted into is determined simply by the presence of a Null in the clock out column for the current employee.  If a Null exists then that row is updated to insert the current date/time into the column.  If no Null exists then a new row is inserted, and the current date/time is inserted into the clock in column.  If an employee is unable to clock in or out as you describe, I would expect the supervisor to manually insert a value into the column for that employee, for which provision can easily be made in the interface.  This is the equivalent of what we used to do in the old days of time cards, when the supervisor would manually enter the time onto the card in the event of the employee being unable to punch their card for some reason.

aainr say:

Assuming that your current setup is a POP e-mail account, and your e-mail is on your computer in a PST, your question should be "how can I import...".  In that case, Nayana is correct. 

 

* Create your account on Outlook.com.

* Now, use Outlook 2013 to connect/use your Outlook.com account (under a new profile, if needed.)

* Using Outlook 2013, 'open a data file'; point to your PST file.

* Now, within Outlook 2013, you can drag e-mail from the PST file to the Outlook.com account.

  (This is what Nayana meant when she said, "move the required folders of PST file into ‘Outlook.com’ data file in Outlook 2013"

 

Once, you've finished dragging what you want in Outlook 2013, it will sync with the Outlook.com account.

 

In my opinion, you're going to lose a lot of functionality by just using Outlook.com, but you can make that determination, later.

 

Good luck, mvbii.
