Hi OneMoreQuestion_FC,

Yes, there is a setting for this policy called BposSAllModality, and you may need to do this one by one.

You can refer to the following two steps methods to turn off the non-archived feature for specific users.

1. In Skype for Business admin center:

a. Login to office 365 admin center with an admin account.


e. Then click save.

This setting turns off the Skype for Business features that cannot be subject to the in-place hold. Enterprises that are required to preserve digital communication can turn this setting on to ensure all **** communication can be captured with the in-place holder features.

The **** online features which are turned off are:

  • File transfer using instant messaging
  • Shared OneNote pages
  • PowerPoint annotations

2. Use the PowerShell commands:

a. Connecting to Skype for Business Online by using Windows PowerShell with an admin credential.

b. The default Conferencing Policy is BposSAllModality. After turning this setting on for a user, the Conferencing policy is set to BposSAllModalityNoFT – notice the “NoFT” in the name = “No File Transfer”.

We can see the exactly settings in each conferencing policy using these two cmdlet:

Get-CsConferencingPolicy -Identity BposSAllModality



Get-CsConferencingPolicy -Identity BposSAllModalityNoFT

Here is the Skype for Business Online PowerShell cmdlet to set a specific user to have a policy (in this case the BposSAllModalityNoFT policy):

Grant-CsConferencingPolicy -identity user@domain.com -PolicyName BposSAllModalityNoFT

Here is the Skype for Business Online PowerShell cmdlet to change all users to use a specific policy (in this case the BposSAllModality policy):

Get-CsOnlineUser | Grant-CsConferencingPolicy -PolicyName BposSAllModality

If anything is unclear, please post back and provide the PowerShell output for us.

Restoring Defaults - Microsoft Community

xbr poseidon say:

Restoring Defaults

Login Information

  • Gateway - 192.168.0.1
  • Username - cusadmin
  • Password - password

 

Restoring Defaults

Through Hardware

  1. Plug the computer into a LAN port via an ethernet cable.
  2. Locate the Reset button.
  3. Using the tip of a ballpoint pen or toothpick press and hold the reset button for 15 seconds.
  4. Release the reset button.
  5. Wait for the device to restart.

Through Firmware

  1. Plug the computer into a LAN port via an ethernet cable.
  2. Open a web browser.
  3. Log into the router by entering the default gateway (192.168.0.1) into the address bar of the internet browser.
