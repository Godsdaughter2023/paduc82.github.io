With Pub 2007 you could turn boundries off totally which removed the dashed border around text boxes, and you did not get the mouseover effect. When you selected the text box you would only get the resize points on each corner and along the side of the actual text box...no visible border. This was changed with Pub 2010.

If you check the boudaries option as Mary described then you will get a dashed boundary around a text box. If you uncheck it that dashed boundary will dissapear until you mouseover the text box. If you check the boudary option then you won't get the on and off of the dashed boudary when you mouseover a text box...it will be there all the time.

If you select a text box a second temporary border line connects the resize points just outside the actual text box until you "deselect" the text box. I also think that is by design so that you can see the actual text box border being resized rather than the line that connects the resize buttons. I don't think there is any way to turn this off. All that you can turn off is the mouseover effect as above.

doug robbins - mvp office apps & services (word) say:
If you are using mailmerge, after inserting the fields into the first label on the sheet and before you use the button to Update all of the labels, select the contents of the first label and use the Format>Paragraph facility to set the line spacing to single and space after (and before if necessary) to 0.

andrea jones - all about resources say: You can also change your styles in Word 2010 so that none of them automatically put the space after paragraph.  To do this, click the arrow on the Change Styles button in the Styles group on the Home ribbon and then click on Style Set and then Word 2003.  To make this stick as your default, click on the Change Styles button again and click on 'Set as default' from the bottom of the menu.
