rich michaels - mvp (greatcirclelearning.com) say:

I don't have a system available to me right now with MS Project loaded on it to verify this but in "theory" if the document file that you have been given is an XML format, meaning it has a "docx" extension on the end of its name, the following should work.

  1. Make a backup copy of the document file and work from the backup
  2. Using Finder add a ".zip" extension onto the end of the file name. For example: MyDocument copy.docx.zip
  3. You will get a message asking are you sure you want to do this... click Yes.
  4. Double click the renamed file and click the subfolder labeled "word"
  5. In this "word" subfolder you should find a folder labeled "embedded" and opening it should reveal the MS Project file you are seeking.
  6. Copy and paste this file to where ever you need it.

Hope this helps

neha singh msft say:

Hi Martin,

From the screenshot you provided, it seems you are on semi-annual channel. I tested on my end with the latest version of semi-annual channel i.e. 1803 (Build 9126.2259) and I see the same behavior.

When opening a workbook in SharePoint using Excel 2016, it opens as read-only. This is a design change from Excel 2013, which opened workbooks directly into edit mode by default.

For your reference: Excel 2016 opens SharePoint workbooks as read-only

This behavior is changed in Monthly channel, since semi-annual channel does not support co-authoring, it continues to open the workbook in read-only mode. If you want workbook to open in edit mode by default, I’ll suggest you switch to monthly channel.

To know more about co-authoring, please see Document collaboration and co-authoring

The following article will help you switch to monthly channel.

How to switch from Semi-Annual Channel to Monthly Channel for the Office 365 suite

If for some reason you are unable to switch, please contact your admin for assistance.
