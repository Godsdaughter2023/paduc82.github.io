jeffreyschlesinger say:

I found that the cause of my problem was an Add-In that was causing "Save Document" to also try to save the template, which I not only did not want, but the users do not have permission to write to the template.

You see the "Save As" dialog was not for the document, but for the template! The document actually did save successfully.

In my case I found the offending Add-In to be the COM Add-In called "Send to Bluetooth". (Other internet sources indicate that poorly written Add-Ins are often the culprit). Disabling the Add-In solved the problem.

But I also made a code change to work around the issue even when this Add-In is installed. The solution is based on this link: I never want to "Save Changes to Template" from a copy .

My code is in VB .NET and forces the template to be considered as already saved, even after the Add-In has caused it to require saving.

The code snippet looks like this:

        If Not (mobWord.ActiveDocument.AttachedTemplate Is Nothing) Then
            mobWord.ActiveDocument.AttachedTemplate.Saved = True
        End If
