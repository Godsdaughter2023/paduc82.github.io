jim gordon mvp say:

Hi Deon,

You are receiving notifications for two different reasons.

The first notification is regarding macros. In Excel preferences you can turn this notification off, but I don't recommend that you turn this off. It displays merely to alert you that the workbook you are opening contains macros. If you're not expecting to have macros in a particular workbook, then don't enable them. Otherwise, simply enable macros to dismiss this dialog.

The second dialog is in regard to features that are not currently supported in Excel for Mac. As Phillip suggested, there could be an Active-X control in the workbook, or perhaps a 3-D map. More and more features that were once Windows-only are now being supported in Excel for Mac. The way to reduce the possibility of getting the unsupported feature dialog is to make sure you keep your install of Office for Mac up to date. Go to the Excel menu and choose About Excel to find your version information. You should have version 16.13.1 or version 16.14 as of this writing.

Ensure your Mac and Office have the latest updates. Here’s how to install updates:

 

Make sure your Mac has a working internet connection, then:

 

  1. Update Mac OS X. Open the App store and click on the Updates button. Install the updates.
  2. After, in any Office application go to the Help menu and choose Check for Updates to launch the AutoUpdate app. Let AutoUpdate install updates and then restart your Mac.

Active-X is being deprecated in Office for Windows and Microsoft discourages its use. New features will be supported eventually, but if your install is up to date, there's nothing you can do if you encounter an unsupported feature that hasn't been hooked up in the Mac version.

vlookup from 2 tables on same worksheet - Microsoft Community

milbourne say:

vlookup from 2 tables on same worksheet

I want to be able to lookup a date from column P within column B Table 1 and return the row value of column F

However, if date P does not appear in that table I want to search Table 2 and likewise return the row value of column F

and so on through Table 3 etc

vijay a. verma say:

If you lookup value is contained in P1, use following formula -

=IFERROR(VLOOKUP(P1,Table1[#All],5,0),IFERROR(VLOOKUP(P1,Table2[#All],5,0),IFERROR(VLOOKUP(P1,Table3[#All],5,0),"")))

If you have more tables than 3, nest them accordingly, hence, in case of 5 tables -

=IFERROR(VLOOKUP(P1,Table1[#All],5,0),IFERROR(VLOOKUP(P1,Table2[#All],5,0),IFERROR(VLOOKUP(P1,Table3[#All],5,0),IFERROR(VLOOKUP(P1,Table4[#All],5,0),IFERROR(VLOOKUP(P1,Table5[#All],5,0),"")))))

ashish mathur say:

Hi,

I have solved a similar problem here - http://www.ashishmathur.com/extract-data-from-unknown-lookup-range/.

If the fifth column is a numeric column, then try this:

=LOOKUP(1E100,CHOOSE({1,2,3},VLOOKUP(P2,Table1,5,FALSE),VLOOKUP(P2,Table2,5,FALSE),VLOOKUP(P2,Table3,5,FALSE)))

If the fifth column is a text column, then try this:

=LOOKUP(REPT("z",99),CHOOSE({1,2,3},VLOOKUP(P2,Table1,5,FALSE),VLOOKUP(P2,Table2,5,FALSE),VLOOKUP(P2,Table3,5,FALSE)))

Hope this helps.
