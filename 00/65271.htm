The way I configured the hidden account, so that they user hardly shows up anywhere in the OS, and most importantly the login screen:

1. Create a new user account using an elevated command prompt by typing the following command:

net user *account name* *password* /add

Then modify the registry entry 

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon

Under the new "SpecialAccounts" key, add another new key, call it "UserLists"

Under "UserLists" add a new DWORD (32-bit). Name the DWORD the same as the new account you made and leave the value at 0x0000000

Now you have a user account that is hidden on the login screen, from Control Panel\All Control Panel Items\User Accounts, and does not by default have a folder in C:\Users.

But it will still show up under the cmd command "net users" and in the registry of course.

To change UAC to prompt for a username and password, instead of selecting a user via picture, modify the following registry entry: 

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\

Create a new key here called "CredUI", and add a DWORD (32-bit) to CredUI called "EnumerateAdministrators" and leave the value at 0x000000

That's it.


As for configuring the user to only be able to run specific programs, I opened "Local Security Policy" under Control Panel\All Control Panel Items\Administrative Tools
