a. user say:

<<
With respect to the missing data, I see it is not reading a number with a letter embedded, but is recogning all the numbers that are pure numbers.
>>

OK, I wondered if you were losing rows, columns or something else.

The way that Word gets data from Excel (and some other types of data source) by default (OLE DB) can cause a lot of problems, because
 a. OLE DB wants to decide a data type (text, numeric, date etc.) for each column, and does so on the content of the first 8 cells in each column. If it decides the column is numeric, texts will come across as 0, for example
 b. some things you do in Word can cause Word to treat fields with text& digits as numeric, e.g. when you use
 { ={ MERGEFIELD myfield} }
or
{ IF { MERGEFIELD myfield } = 0 "zero" "not zero" }

Broadly speaking,
 a. it can be better to connect to Excel using the DDE method, which /tends to/ provide the data to Word as you see it in Excel
 b. it's a good idea to avoid columns with mixed data types in Excel, if you can.

There is a bit more about some of these issues at http://tips.pjmsn.me.uk/t0003.htm
but the page doesn't cover all the possibilities.

Excel Sorting for Birthdays - Microsoft Community

bearyann say:

Excel Sorting for Birthdays

I have an excel worksheet with customer information. One field is birthday which is entered as MM/DD.
When I sort by month, the sort separates out the dates by year entered instead of a just day of month.
For example, when I run the list for May, I have a list May 1-31 with names entered last year followed by May 1-31 posted this year.
How do I get it to sort ALL the data and ONLY on MM/DD?
chip pearson say:

Create a new column next to the column containing your dates. In that column, use the formula

 

=TEXT(MONTH(A1),"00")&TEXT(DAY(A1),"00")

 

where A1 is the cell with the full date. Copy this formula down as far as you need to go. Then, sort the data using this new column as the sort field.

mike h.. say:

Hi,

 

Assuming your birthdays are in Col A put this formula in a helper column

 

 

=MONTH(A2)

 

Drag this down as far as required then select both of the columns and set up a sort of first on the helper column; smallest to largest,  then add a level and sort on this actual birthday column; Oldest to newest, then click OK. You can hide the helper column if you want.

poppyyang_msft6299 say:

Hi David,

Advanced threat protection (ATP) in Exchange Online Protection (EOP) helps you prevent zero-day malware attacks in your email environment. ATP provides a way for you to create policies in the Exchange Admin Center (EAC) that help ensure your users access only links in emails or attachments to emails that are identified as not malicious.
To better protect our mailbox’s data, one mailbox needs one ATP license. As we know, we should pay $2.00 /mailbox/month to use the Advanced threat protection.

To let ATP protect the shared mailbox, we can apply the safe attachments/links policy to the shared mailbox by following the steps below:
a. Login to Exchange admin center.
b. Select advanced threats>> safe attachments>> click “+” to create a new safe attachments policy.
c. Go to Settings, and then choose the options you want to use with this policy.
d. Go to Applied to, and then choose The recipient is>> select the shared mailbox.

Here is a document for your reference:
https://technet.microsoft.com/en-us/library/mt148489(v=exchg.150).aspx
