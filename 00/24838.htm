charles kenyon say:

I have one macro (Launcher) that starts my merge automatically. That macro is always available (in an Add-In) starts the merge by opening the main merge document. It is launched by a custom button in the Add-In's ribbon.

That merge document (read-only) has an AutoOpen macro that links to my data set (a comma-delimited text file) and performs a merge to a new document.

The Launcher macro then attaches that new document to that separate "cleanup" template. (I was surprised to learn that the main merge template is not the attached template for a mail merge result document!)

I suppose if you only had one cleanup macro to run, it could be in your Normal template or an Add-In and be run at the end of your merge macro. In my case, the macro that attaches the result document to the template with the multiple cleanup macros runs at the end of the macro that starts the merge and attaches the data file. You could put your cleanup macro there instead. In my case the merge has an ASK field that pops up for many of the letters, so I know that the other macro doesn't run until the merge is finished.

That macro-holder template has a custom toolbar (designed in Word 2003 because I was lazy) that shows up under the Add-Ins tab and allows a series of cleanup macros to be run.

The person running the merge clicks on toolbar buttons to perform the cleanup. The final macro (Finish) attaches the new merge document to the normal template. The toolbar commands disappear from the ribbon.

In that way, I am not messing up my interface with a bunch of macros or toolbars that are only used during this merge but still have them available for the merge.

So, the way I do it is with at least three macros:
One available generally through an Add-In to open the main merge document.
(2) An AutoOpen in the main merge document to start the merge
(3) a macro to attach the template that holds the cleanup macros which are then run sequentially by hand, with manual intervention. That third macro does not run until after the merge is completed. (I am not sure what would happen if it were part of the AutoOpen macro that starts the merge.) If your cleanup macro will run without manual intervention, you could substitute that for the one attaching the template.

Someone else may have a much simpler and more elegant method. I've been using variations on this for about six years.

Outlook.com has a feature that prevents losing email messages if you accidentally set up POP3 to delete them after reading them on the server. Here’s how it works:

    The first time that Outlook.com receives a POP3 instruction to delete email messages after reading from an email app, Outlook.com won't delete the messages.
