Using an XY (Scatter) chart, if the left column is Y values (vertical axis) and the right column is corresponding X values, the fitted "trendline" quadratic (second-order polynomial) using Excel 2013 is approximately Y = 62.081*X^2 + 4612.7*X + 85718 with R^2 = 0.7574.

To use the fitted equation for calculations, the coefficients should use 15 significant figures. One way to get higher-precision values in worksheet cells is to use the array-entered LINEST worksheet function. To do so, see John Walkenbach's Chart Trendline Formulas web page at

http://spreadsheetpage.com/index.php/tip/chart_trendline_formulas/

Using LINEST results, the calculation of fitted values should use

62.0808578382280
-4612.71329738518
85717.9489475164

Using an XY (Scatter) chart, if the left column is X values (horizontal axis) and the right column is corresponding Y values, a second-order polynomial is not an appropriate fit.

Since you described the data as "coordinates", I interpreted the left column as X and the right column as Y, since we usually write "coordinates" in the form (x,y).  That results in the top chart.

I started by dividng the data "equally" into 3 series, using the following for X:  A1:A70, A70:A140 and A140:A211.  For Y, change A to B.

But as you noted:  second-order polynomial trendlines were not a close fit for each part.

Since it was unclear why you chose only 3 series and only second-order polynomial trendlines, I took some liberties in the top chart.

I divided the data into 4 series, using the following for X:  A1:A71, A71:A106, A106:A140 and A140:A211.

And I used RSQ (R^2) to guide me in choosing polynomial trendlines of varying degrees that resulted in the best fit for each part.

It would be messy to show the trendlines formulas in the chart with the 15-significant-digit precision that we need in order to duplicate the formulas in Excel.  They are:

Part1:

y = 5.13613718000000E+08x5 - 8.94877053201880E+10x4 + 6.23663200965825E+12x3

      - 2.17323589862871E+14x2 + 3.78646219088237E+15x - 2.63888442118933E+16

Part2:

y = -1.06961610644531E+06x3 + 1.11797024482908E+08x2 - 3.89503443164666E+09x

      + 4.52346363547388E+10

Part3:

y = 3.33869148400000E+09x4 - 4.65311071610750E+11x3 + 2.43187781476296E+13x2

      - 5.64881819318653E+14x + 4.92044873450365E+15

Part4:

y = -2.27579566955566E+02x3 + 2.38224325101259E+04x2 - 8.31222190369125E+05x

      + 9.66780776783804E+06

You could copy each set of coefficients into an Excel worksheet.

Generally, it is better to use LINEST to calculate them.  For example, for Part1, select 6 horizontal cells and array-enter (press ctrl+shift+Enter instead of just Enter) the following formula:

=LINEST(B1:B70,A1:A70^{1,2,3,4,5})

But sometimes, as in this case, LINEST fails to produce the same or equivalent coefficients.  That is because the LINEST collinearity check concludes that the high-degree polynomial is redundant.

(However, the trendline algorithm demonstrates that is not the case.  I suspect the disparity arises because of numerical limitations of the LINEST computer arithmetic.)

Indeed, for this data, LINEST works only for second-order polynomial trendlines.  But the estimates are not quite as good.

With the LINEST limitation, perhaps breaking the data into smaller segments would have better results.  (Another exercise for someone else.)

In any case, use SUMPRODUCT or SERIESSUM to appy the set of coefficients to each corresponding part of the data.

But beware:  do not try to extrapolate the data beyond the limits of each part.  Polynomial trendlines rarely produce reasonable extrapolations.
