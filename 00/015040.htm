omer atay say:

To add on what Bernd said...

 

File > Save as in OneNote should really have been called Export. Because it doesn't work like save as does in Word/Excel etc. It just creates a copy of the notebook (in the .onepkg packaged format) in the location you specify, but the notebook you have still is the one from the old location. To complete the creation of a new notebook, you need to open the onepkg file and follow the directions to expand it to a location of your choice.

 

Here is the workflow:

1. You have your notebook ready at c:\notebooks\Client1

2. You save as (export) the notebook to c:\notebooks\template.pkg

Note that at this point you still have your original Client1 notebook.

3. You open c:\notebooks\template.pkg, OneNote asks you to expand it to some location

4. You pick c:\notebooks\Client2

Now you have your new notebook created and you can start working on it. You can close Client1 notebook at this point. Everytime you need to create a new notebook from this template, you can go and open c:\notebooks\template.pkg and it will ask you to expand it to a new location, allowing you to create a new notebook.

 

Compare this to the Word workflow:

1. You have your document at c:\docs\Document1.docx

2. You save it as c:\docs\Document2.docx

Now Word automatically closed Document1.docx and you are working on Document2.docx at this point.

 

This is fundamentally different. The onepkg file is not really a notebook you can directly work on, it is a packaged file that contains the state of notebook at the time you saved it (exported). To work on it you need to double click and expand it to a new location on which you can work on using OneNote. Please let me know if this doesn't make sense and I can explain further.

SELECT DISTINCT Material.YourPrimaryKeyGoesHere

Sub SortWords()
  Dim Where As Range, R As Range
  Dim Contents As String, Delimiter As String, Key As String
  Dim i As Long, j As Long, k As Long, l As Long, m As Long
  Dim Temp
  
  If Not TypeOf Selection Is Range Then
    MsgBox "Select some cells and try again"
    Exit Sub
  End If
  Set Where = Intersect(Selection, ActiveSheet.UsedRange)
  If Where Is Nothing Then
    MsgBox "Select some cells with contents and try again"
    Exit Sub
  End If
  
  Delimiter = vbLf & ","
  
  Application.ScreenUpdating = False
  For Each R In Where
    Contents = R.Value
    
    i = 1
    l = Len(Contents)
    Temp = Array()
    Do While i < l
      For j = i To l
        If InStr(Delimiter, Mid$(Contents, j, 1)) = 0 Then Exit For
      Next
      k = j
      For j = j To l
        If InStr(Delimiter, Mid$(Contents, j, 1)) > 0 Then Exit For
      Next
      Key = Trim$(Mid$(Contents, k, j - k))
      ReDim Preserve Temp(0 To UBound(Temp) + 1)
      Temp(UBound(Temp)) = Key
      i = j
    Loop
    InsertionSort_Prim Temp
    Contents = Join(Temp, vbLf)
    
    R.Value = Contents
  Next
End Sub
Private Sub InsertionSort_Prim(ByRef Arr, _
    Optional ByVal SortOrder As MsoSortOrder = msoSortOrderAscending)
  Dim i As Long, j As Long, Temp
  Dim SortAscending As Boolean
  SortAscending = SortOrder = msoSortOrderAscending
  For i = LBound(Arr) + 1 To UBound(Arr)
    Temp = Arr(i)
    For j = i - 1 To LBound(Arr) Step -1
      If (Arr(j) <= Temp) = SortAscending Then Exit For
      Arr(j + 1) = Arr(j)
    Next
    Arr(j + 1) = Temp
  Next
End Sub
