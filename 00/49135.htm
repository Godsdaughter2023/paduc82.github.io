I believe that the question is about *how to delete the file* or *how to change its permissions*, not about any specific installer or application.  This is a generic question.  I see many reports of people report having this same issue, without resolution.

The point is that it is possible to set the permissions of a file such that the permissions subsequently cannot be read or edited, even by and account with administrator privileges, and even when those administrator privileges have been applied.  The question is, if a file is in that state, how can we take back control of the file?

Here are some things that are not relevant to the problem:  System corruption.  User profile corruption.  Why?  Because this happens on working systems, that otherwise don't have problems.  It is specific files that are in this state.

Here are some things that will not fix the problem:  The GUI for accessing permissions, i.e. the file Properties -> Security tab, cannot do so in this case, so we cannot use that.  The takeown /f <file> command gets "access denied".

Again, this isn't a "what's wrong" question, it is a "how to" question.

Here are some things that may be relevant:  This is under Windows 10.  The Windows 10 upgrade hid the Administrator account, but we can enable it again if needed.  We may have Windows 10 Home.  That doesn't have all the administrative tools that other versions have.  Windows 10 Home may not grant even a user with administrator privileges all access to files.  I'm sure we can all use the command line.  If we need to log in as the Administrator account and run icacls, I'm sure we would all be happy to do so.  If it requires installing Windows Subsystem for Linux, and doing file operations from bash, I'm sure we'd be happy to do that too.
