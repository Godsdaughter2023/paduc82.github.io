Linking specific text in one header to text within another header - Microsoft Community

aaronwoodland say:

Linking specific text in one header to text within another header

Say you have two unique headers, one on page 1 and one on page 2.  Let's say these two headers both have the same reference number contained withing them (like a unique document number - 345345-001).  What is the best method to link the number reference in heading 2 to the number reference in heading 1 so that when this reference number changes in heading 1 it will automatically change in heading 2.
jay freedman say:
As long as the documents will be edited only in Word 2007 or 2010, the simplest method is to use one of the built-in document properties and the content controls that are mapped to them in the document's xml.

Put the cursor in one of the headers. On the Insert ribbon, click Quick Parts > Document Property. Choose one of the items in the menu, such as Subject (unfortunately there is no built-in property named Document Number, but there's no hard rule about what you can put in any property). Go to the other header and do the same, choosing the same property again. You'll now have a content control in each header.

If you enter or change the text in one of the controls, the same text will appear immediately in the other control -- and anywhere else in the document that you've used this feature.

Besides editing the controls directly, you can change the property where it's listed on the File > Info page (in Word 2007 it's on Office button > Prepare > Properties) and the content controls will reflect the change. In macros, it can be written and/or read through the document's BuiltInProperties collection, as in

ActiveDocument.BuiltInDocumentProperties("Subject") = "345345-001".

The one caution is that if the document is opened in Word 2003 or earlier (with the Compatibility Pack to let it read .docx files), the content controls will be converted to plain text. That's OK if the document will be read or printed but not saved. If the document is saved, though, the text in the headers will no longer be connected to the property or to each other.

charles kenyon say:
To go further, you could set a custom document property and use a DOCPROPERTY field to reflect its contents in the headers. This will survive a transition to Word 2003 and back, but is not worth doing unless you need to make such a transition.

For more on content controls see Create Mapped Content Controls.

MP4 is a general format description with many different settings, from very low to very high. The Asha 200, being an entry level device will play the format, but will require videos in this format to be encoded to a lower specification than say your desktop or a higher spec smartphone will.

 

The Nokia Asha 200 support video playback in upto 320x240 resolution at 15fps but a video in this format will requires basically all processing power the Asha 200 has to play smoothly. You might want to install a video transcoding application which will create an mp4 file which can be played on your device. shyam pillai say:

Hi,

The solution will depend on how the text colors were set in the first place.

Quick: Change the textbox color on the slide masters.

1. Click View tab and select Slide Master

2. Select the slide master slide in this view.

3. Select on the title placeholder and from the Format tab select the desired Text fill color.

4. Repeat for the body placeholder.

5. Close master view, if the text box are following the theme colors then all the textboxes will reflect the slide master changes.

The painful method: Individually change the text box colors since they do not follow the theme colors.

You will need to select each text box and change the formatted color.
