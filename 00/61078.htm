a. user say:

If chkdsk is running when Windoes starts it is because the Windows autochk program has found the volume to be dirty.

A volume can be marked dirty from things like an aborted restart, improper shutdown, power interruption, using the reset or power button, battery pulling, etc.  Have any of those events occurred?

You can query the dirty bit on a volume from the command prompt window with Administrator privileges.

Click the Start orb/button and in the box enter the following:

cmd.exe

Right click cmd.exe and choose to Run as Administrator:


To query the dirty bit on drive C, in the Command Prompt window enter the following command:

fsutil  dirty  query  C:

Sample output should be one or the other of:

Volume C: is dirty
Volume C is NOT dirty

You should check each of your volumes or each of your hard drives (C, D, E, etc.).

If a volume is dirty the only thing that can clear it is a chkdsk with error correction and you might need to run it more than one time.  If the drive is failing, you might not ever be able to clear the dirty bit so chkdsk will always run on a restart (annoying and time consuming).

If the dirty bit will never clear you should consider replacing the drive but in the meantime you can tell Windows to NOT check the dirty bit on a volume by issuing a command like this example for the C drive:

chkntfs   /x   c:

That would tell Windows to never check the dirty bit on volume C but that is risky behavior because if the dirty bit will not clear, something is wrong with that drive.

You can undo that command and reset the automatic error checking back to the default value with this command:

chkntfs   /d

It could be that this manual chkdsk you are running will clear the dirty bit - maybe not but at least you know how to check it.   It might take several attempts at chkdsk /r  and if the drive is failing you might not every be able to clear it.

When your system finishes (more than 1-3 hours later) you can check the dirty bit and decide what to do.

I imagine when this chkdsk you are running now finishes you are going to want to know what it found and to do that you have to look in the Event Viewer Application log.  Here's how to do that:

http://www.sevenforums.com/tutorials/96938-check-disk-chkdsk-read-event-viewer-log.html

I predict your troubleshooting will take many reboots so if you can't get the dirty bit to clear and it you don't want to wait hours and hours for the chkdsk that runs every time Windows starts use the chkntfs  /x  c:  command to tell Windows NOT to check the C volume while you troubleshoot other problems.  You can undo that later with the chkntfs  /d  command.

Have you seen how your system behaves if you boot into Safe Mode?

https://support.microsoft.com/en-us/help/17419/windows-7-advanced-startup-options-safe-mode

That will help you decide if some startup program(s) is contributing to the long delay for your desktop to appear.
