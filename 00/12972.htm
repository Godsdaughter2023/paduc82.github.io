pamcaswell say: No. Document templates (unlike normal.dotm and other global templates) can be saved wherever it is convenient for the user or organization. 

charles kenyon say:
Document templates never needed to be stored in any particular folder. It is just that putting them in a user templates or workgroup templates folder gives you the ability to use File > New to give users a choice of templates organized by folders or tabs.

See Templates in Microsoft Word for a discussion of how these can be organized.

It is important that templates be used to create new (unnamed) documents rather than opened as SaveAs to a different name. This second method happens frequently when they are stored in document folders and results in the templates being ruined by inadvertent changes being saved.

rich michaels - mvp (greatcirclelearning.com) say:
I believe the exception to this is if the template has Macros (.dotm) then it must be placed in a designated Trusted Location.

pamcaswell say: What you write is only true if the template must work as a global template--providing its macros to all open documents.  As a document template, it can contribute its macros, keyboard shortcuts, and (I believe) autotext as long as it is attached to a document and Windows can find it.


rich michaels - mvp (greatcirclelearning.com) say:
Now you're jogging my memory... it depends what the macros are doing. I've run into this before and that's why I brought it up. 

The template does not have to be a global template, it could be a document level template. For example, I believe it fails if one of the macros attempts to re-save itself. 

The scenario is... the attached document level template (not stored in the trusted user template area) looks at the attached document styles and then opens itself as a new document, updates the template styles and tries to re-save itself.

Unless something has changed, starting with Word 2007 that used to fail.

A macro in a document template will not run if the template is not in a trusted location or otherwise authorized (signed). If the template is in a trusted location, a macro in it will run.
