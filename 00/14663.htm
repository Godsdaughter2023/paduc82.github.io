Hi OmarMunir,

Here is the answer to your question:

1. It shows in the status it has uploaded 20.5 GB which is the file size, and its been at that state fora long time, should I wait or is here something wrong here? 
I checked the text log file that it created as part of the command I used to run this, just shows the same status. no error.

A] If there is no error message you can wait. However, after the PST files are copied to the Azure storage area, a PST file is imported to an Office 365 mailbox at a rate of at least 24 GB per day.

2. For mapping, if I need to map the pst to a separate folder, I need to specify a folder name, and enable archive, otherwise it will map it to the main mailbox. What if I only specify a folder and not enable archive (because I dont want to enable it), would it still map it to a separate folder?

A] If you set IsArchive parameter FALSE or leave this parameter blank, the PST file is imported to the user's primary mailbox. 

If you specify TargetRootFolder parameter /<foldername>, items in the PST file will be imported to a folder named <foldername> . For example, if you use /ImportedPst, items would be imported to a folder named ImportedPst. This folder will be located in the user's mailbox at the same level as the Inbox folder. 

Tip: Consider running a few test batches to experiment with this parameter so you can determine the best folder location to import PSTs files to.

3. Finally what is the retention period of the temporary storage 'blob' it created as part of the command I ran, for example if I want to map the uploaded pst file to one mailbox first, and then map it to another later on. Can I do that or does it keep it for one 'map operation' only?

A] All PST files are automatically deleted from your Azure storage area. If there are no import jobs in progress, then all PST files in the ** ingestiondata ** container are deleted 30 days after the most recent import job was created.

You may refer to the following articles below:
Use network upload to import your organization PST files to Office 365
FAQ about importing PST files to Office 365

Shyamal

Hi Omar,

Sorry for the late reply. From my understanding, if you'd like to upload the PST file to a specific folder, you can just specify the TargetRootFolder parameter and disable archive.
