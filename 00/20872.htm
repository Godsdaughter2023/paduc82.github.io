pcralidis iii say: Hi Dhenz,

     I appreciate your responding back to my inquiry.

I have managed to resolve the problems mentioned in my OP, after locating an old post in the POPPeeper Forums which also contained a link to information at Microsoft addressing the very problems I was experiencing. 

Description of the solution:

When I first set up both my old Hotmail accounts and the new Outlook email account, the default configuration settings in POPPeeper were for Hotmail/Outlook.  The solution was to change that setting  to "POP3."   I also had to change the Send Mail setting to "SMTP."

The SSL port setting is 995;  The TLS port setting is 587.  In addition, with a Hotmail/Outlook email account, the "Check Mail" timer must be set to check for incoming mail no more frequently than once every 15 minutes.  Setting the timer for less than that will usually cause POPPeeper to generate a "Retrieve Mail" error.

Also, the "Delete Mail Immediately" option box in POPPeeper must be UN-TICKED.  Leaving it checked will subsequently result in an error message in POPPeeper.  Messages marked for deletion will be deleted the next time POPPeeper checks for incoming emails.

s mallett say:

User (archive) migration to Office 365

Hi,

We have a user who has an archive, and whose mailbox and archive are currently on our on-premise Exchange 2013 server. We have a hybrid setup with Office 365 and are slowly migrating users to O365.

However we've never migrated a user with an archive before. In theory, does this just work or are there more steps to follow? The archive itself is almost 8GB. I'm just nervous as I can't lose his emails.

Thank you.

ranmo_msft say:

Hi Mallett,

Generally, to enable the migrated user’s archive mailbox in Hybrid deployment, you can follow the way below.
Before migrating using remote move, enable this user’s archive mailbox in on-premises Exchange. You have done this. It has content (8GB). Then move the user’s primary mailbox along with the archive mailbox using remote move. Note: Verify that the Move primary mailbox along with archive mailbox option is selected, and then click Next.

For the detailed information, you can see the information below:
https://technet.microsoft.com/en-us/library/o365e_hrcmoverequest_fl312271(v=exchg.150).aspx (it includes the scenario moving the archive mailbox along with the primary mailbox)
