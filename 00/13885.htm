Bob,

I believe  you stated you requirement clearly.

Short and sweet, there is no easy built in way to satisfy that requirement with the existing capabilities of data validation.  However, I have outlined a possible approach you could use to achieve your objectives.

What you would need to do is have your list of items but that would not be the source for your data validation.  You would need to set up a second list that is populated with formulas.  

The formulas would reference the choices you have made and build a list for the dropdown that includes no duplicates.

You would dynamically refer to this second list using a named range and the named range would be what you would refer to in your data validation list.  

Once a cell is filled, the choice would be removed from the list.  That would not affect that cell because data validation is only used when an entry is being made (a choice is being made).  

Also, using macros is of course another option.

bob98126 say:

Thanks Mark for your help.

I solved it using vba.

This is what I end up using  on the sheet and it works to prevent duplicate entries, for anyone if interested

Private Sub Worksheet_Change(ByVal Target As Range)

'prevents duplicates

    If Not Intersect(Target, Range("A1:A100")) Is Nothing Then
    With Target
          If WorksheetFunction.CountIf(Columns(.Column), .Value) > 1 Then
            Application.DisplayAlerts = False
            MsgBox "Duplicate data on " & Target.Address & vbNewLine & "No duplicate entries allowed", vbCritical
            .ClearContents
            Application.DisplayAlerts = True
            Target.Select
          End If
    End With
    End If
End Sub
