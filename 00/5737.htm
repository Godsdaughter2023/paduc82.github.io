I find that messages tend to get stuck if you open the Outbox containing mails to be sent. The following macro should work to send any messages in the outbox:

Option Explicit
Sub SendAndReceiveAll()
Dim olNS As NameSpace
Dim olSyncs As SyncObjects
Dim olSync As SyncObject
Dim olItems As Items
Dim olItem As Object
Dim i As Long

    Set olNS = Application.GetNamespace("MAPI")
    Set olSyncs = olNS.SyncObjects
    Set olItems = olNS.GetDefaultFolder(4).Items
    For i = olItems.Count To 1 Step -1
        Set olItem = olItems(i)
        olItem.sEnd
        DoEvents
    Next i
    For i = 1 To olSyncs.Count
        Set olSync = olSyncs.Item(i)
        olSync.Start
        DoEvents
    Next
CleanUp:
    Set olItems = Nothing
    Set olItem = Nothing
    Set olNS = Nothing
    Set olSyncs = Nothing
    Set olSync = Nothing
lbl_Exit:
    Exit Sub
End Sub

    macropod say:
    What you need is a Next Page Section break after the Table of Contents and to unlink the headers/footers for the next Section from the Section containing the Table of Contents. You can then give that Section its own page layout (you probably won't want either the 'different first page' or 'different odd and even' settings). There's nothing you can insert into the field or its code for what you're after.

    charles kenyon say:

    Do not insert section breaks in your table of contents or table of figures, these are fields and shouldn't be split with a section break. The section break will be eliminated if the field is updated (other than for page numbers).

    You could change the formatting for the title paragraph styles to include a borderless Frame which would do the spacing.

    You could have each of them start a new section (before the start of the table) and have that section have a different first page header (not margin) that takes up space. One way of taking up space in such a header is a borderless empty text box. See some of the letterhead samples on my downloads page for ideas on how this can work.

    Tutorials

    suzanne s. barnhill say:
    You don't need a section break to create a different "margin" at the beginning of a section. See http://wordfaqs.mvps.org/DifferentMargins.htm. You may also find http://wordfaqs.mvps.org/FrontMatterRibbon.htm helpful.

    Thanks Doug.  Tried your idea.  Did not work.  I finally got it to work.  Here's how:

    1. Did a complete re-install of MS Office365.
    2. Create net-new Word doc (my business email message) all from scratch starting fresh. (not using existing save as, but net-new).
    3. Created a net-new Excel email address and first name list.  (not using existing save as, but net-new).
    4. Slowly added each element of my message into the Word Doc, and then doing an HTML email mail merge test (added my email address in the doc, then sent test email out; added my company URL, then sent test email out...each test successful using HTML).
    5. Then for the final test, inserted my company logo png image into MS Word doc message.  Did another send as HTML test, and it worked.

    So, I believe the merge HTML issue is once you've updated your MS Office365, for some users, if you try to use your existing MS Word Doc merge and existing MS Excel list doc, there must be some embedded legacy code in the docs that are not compatible with the MS Office365 code update.  So, you have to re-write everything in order to send HTML emails.  At least for me.

    Good to know that the issue is resolved. Here I’d like to write a summary:

    =================


    After re-creating your Office 365 user account, you can’t access your OneDrive for Business.


    The issue is caused by a PUID mismatch between the user account (m***z@c******r) and SharePoint Online user profile data. Below are the scenarios.

    a. A deleted user is re-created with the exact same account name.

    b. A user is synced from Active Directory organizational unit (OU) and then moved to a different OU and resynced with SharePoint Online.

    The above will allow the user to access the SharePoint Online site collection with an incorrect PUID.

    More information can be found from the Scenario 2 under the INTRODUCTION section in the article below:

    How to remove users from SharePoint Online


    We have deployed a fix to make sure all the data is matched.


    It is not recommended to delete and re-create a user with the same user principal name, or move the directory synced user to a different OU.
