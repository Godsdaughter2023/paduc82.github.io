john p spencer say:

I did not use variables but used invisible controls on a form.

My sample code could have been a bit clearer

Docmd.OpenReport "A",acViewPreview,,,acHidden
Me.iReportAPages = Reports("A").Pages  'assign value to control on a form.
Docmd.Close acReport, "A"

If you are going to use global variables, make sure they are declared in a VBA module so they are available throughout the database. 

Option Compare Database
Option Explicit

Global igReportAPages as Integer
Global igReportBPages as Integer
Global igReportCPages as Integer
Global igReportTotalPages as Integer

You would NOT Dim the Global variables in the procedure.

THen you would need to use VBA code to assign the values of the variables to the controls.  You cannot reference the global variables directly. 

So, now the controls would be unbound and you would have to use code like the following in the report's footer section

Me.PageOfPages = "Page " & Me.Page + iReportAPages & " of " & igReportTotalPages

I think that the only way to get the total number of pages would be to open each report and get the pages property.  If you were running the report from a button on a form

Sub PrintButton_Click

'Assign the page counts to hidden controls on the form so they can be referenced on the reports as they print

Docmd.OpenReport "A",acViewPreview,,,acHidden
iReportAPages = Reports("A").Pages

iReportBPages = Reports("B").Pages

iReportCPages = Reports("C").Pages

iReportTotalPages = iReportAPages + iReportBPages  + iReportCPages

'Now open the three reports in sequence to print them

End Sub

Your Page and Pages controls on the reports would read something like

Control Name: txtPagesCount
Control Source: Forms!PrintForm!iReportTotalPages

Control Name: txtPageCount
Control Source: Page   (First Report)

If you don't run the reports from a button on a report, you could do something similar using global variables.

Not sure what you need by spin wheel type of chart, but if you have, say, a header in cell A1 and then the list of names in column A, you could use this formula to randomly draw a winner:

=INDEX(A:A,RANDBETWEEN(2,COUNTA(A:A)-1))

Every time you re-calculate (press F9) you would get a new random name.

If, however, you want to prevent somebody winning two door prizes, you could use a column of formulas to effectively remove their names from the list so that they don't get re-chosen, or randomly sort the names first prior to removing duplicates, or us a macro....  lots of options.

jasoninmelbourne say: I've found a completely different approach that actually works on the HTC Wiki. Although it's prescribed for those who sync via Exchange, the approach also works if you sync via PC.

My particular issue seems to stem from a conflict between the time my phone identifies from getting its automatic time changes from the time on cell network (i.e. seems to be stuck on AEST, therefore all appointments are 1 hour early) and what the time zone is when the appointment is created in Outlook.

The fix for this is (thanks to HTC Wiki):

1. Go to Start > Settings > Phone > Time Zone tab > uncheck "Automatic time change and clock."

2. Go back to the Today screen.

3. Tap the Clock to access the clock options.

4. Change the time zone to "GMT London, Dublin"

5. Synchonrise your device via your normal method (e.g. PC able, via Exchange, etc). This should send your appointments this time, and this will be a clean synchronization.

6. Change the time zone to your local time zone

7. Synchronise your device again. This should send your appointments this time and readjust the appointments to match your time zone. This will resolve the issue.
