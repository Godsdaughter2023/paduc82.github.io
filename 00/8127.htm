timliu_msft1261 say:

Hi Naiche,

I think the issue may related to we sync the same pst file on different devices:
a.When you have a pst file "JobInfo" on Lenovo, and synced it to Surface Pro 3 the first time, there should be no issue.
b.When you make some changes to the pst file on Lenovo and close the computer. The changes are synced to the cloud portal.
c.Now you open Surface Pro 3, the sync client on it finds changes from cloud, so it try syncing the changes to the synced pst file. But the sync client cannot merge the changes between cloud and local synced version. So it create a copy file for the local synced file with the device name and sync back the new cloud version. That's why we get two pst files.
d.After you return to your laptop, the same thing happens, so you get 3 PST files.

It is not suggested to sync the same pst file which always need to be modified on different devices:
1.The pst file cannot be synced in the open status, so we need to close it and Outlook app for syncing.
2.The sync client may not be able to merge the changes on the same pst file between two sync clients.

As a work around, you may create different pst files with a specific period. E.g. create a pst file every 3 months to back up emails or reduce your mail box storage. Sync these pst files for backup rather than always modify the same syncing pst file.

rudy xu msft say:

Hello Bencneretech,

 

For the first question, the short answer is Yes. But it is a new Office 365 tenant, to use your original data, you have to migrate the needed data to this new Office 365 tenant. After you migrate the data, change the MX record points to this new tenant EOP, in this way you can continually use your mails.

 

For your reference:

https://docs.microsoft.com/en-us/exchange/mailbox-migration/migrating-imap-mailboxes/migrate-other-types-of-imap-mailboxes

Note: The easiest way is let end-users export the data from Outlook directly saved as .pst files and then import these .pst files to Outlook client again.

 

For your second concern, actually, you don't have permission to use the Godaddy as your organization name to register your Office 365 tenant, you need create a new one or use your original one for register it. 

 

Hope the above information helpful.
