stefan blom say:

In the Set Numbering Value dialog box, you'll have to choose the "Start new list" option; then you can more reliably set a starting value.

As an alternative, you can copy the numbered item and then paste in plain text format; that way, you'll get a static number. (The downside, of course, is that you lose paragraph formatting.)

pamcaswell say: It did happen in W2007, though you may not have noticed because the numbers up to the ones you designated are hidden and don't show when the nonprinting formatting marks are not showing.  As you've probably noticed, if you delete those hidden numbers, you lose the numbering you want.  Word didn't add the hidden numbers in W2003, but resetting numbering in 2003 was a bit flakier than it is in W2007/10.

When I have reset numbering in docs going out for review or back to the writers, I put a "Do not delete" note in the numbering (therefore also hidden) to avoid deliberate accidents.

navyjax2 say:

Problem with just leaving it, for me, is that these aren't hidden when you go to create a dynamically-created Table of Contents, the way I was doing it.  The lines show up and have to be removed anytime you use the feature to update the TOC.  I wondered:  Why would Microsoft put in this text as hidden instead of just continuing the number from the past style, in accordance with the list?  Like if I have Heading 1 (1. Overview) and Heading 1 (2. Instructions), and I want to place a Heading 2 (2.1 Server Updates), it should auto-set 2.1, not 1.1!  It should be "smart" enough to figure out that there are already two Heading 1s and not start back at the beginning. 

 

Well, I figured out what I was doing wrong.  2010 actually can do this, correctly, too, but you have to play with it immensely to get it to work - at least I did.  You have to use the multi-level list, not the traditional numbering list, and use the one that is dedicated to headings (where each level has "Heading X", where "X" is the number that corresponds to the appropriate level), not the multi-level list that has 1, 1.1, 1.1.1, but does not specify that it corresponds with a Heading.  Then you have to make sure all of your styles are using that list.  You have to make sure that you've updated your Heading 1 to use this multi-level list, and you have to ensure that, before you update your Heading 2 style, that you have a couple of Heading 1s in place and set what you want to be Heading 2 to "Continue from previous list" when you set the number.  You should not have to advance the value manually to get 2.1 - it should do this for you automatically, if it recognizes your Heading 1s.  Otherwise it will say 1.1.  If it does that, you need to make sure both of your "Heading 1s" actually ARE set to Heading 1 and have the multi-level list set.

 

Hope this helps someone....

-Tom

Heading numbering should be done with a multilevel list, yes. This ensures that restarts work correctly. See http://www.shaunakelly.com/word/numbering/numbering20072010.html. (The behavior hasn't changed since, at least, Word 97, although the user interface has changed a lot.)

pamcaswell say:

Word ignores hidden text when formatting marks are not showing, so it would not be caught by the TOC feature.

 

I'm glad you worked your problem out.  The next time it should be simpler if you remember to put the cursor in the first  Heading 1 (or whatever style you intend to link to the first  number level) in the document before you apply the multilevel list.  That way, all of the styles will be updated, and numbered, when you OK out of the multilevel list dialog.  Also note that you can link any paragraph style to a number level, so if some of your numbered levels are not headings but  regular text, you can link them to  text styles such as List and List 1.

It is possible but check this requirements first:

* This feature is only available in Outlook for Office 365 for Mac and Outlook 2019 for Mac and Outlook 2016 for Mac 15.36.1 (170721) and later (select Outlook> About Outlook to check your version).

* An Exchange account such as Outlook.com hosted on Office 365 or on Exchange 2016 CU5 and later.

* It will not work with IMAP or POP accounts such as Gmail, Yahoo, or iCloud.

I still use the old Azure AD module for W indows, but yes, it was deprecated.

You can try to download it from these links though:

https://bposast.vo.msecnd.net/MSOPMW/Current/am...


If your getting connection errors try to download the old version.

If I want to perform Office 365 PowerShell commands like "Restore-MsolUser -UserPrincipalName <account>" , I need to authenticate using Connect-MsolService first.

Yes .It is included in this connection command :

$Creds = Get-Credential
Connect-MsolService –Credential $Creds
$Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri https://outlook.office365.com/powerShell-liveID... -Credential $Creds -Authentication Basic -AllowRedirection
Import-PSSession $Session

This script let you connect to MSOL and Exchange in the same session

If I want to perform AzureAD PowerShell commands I need to authenticate using Connect-AzureAD.

Yes . You can tell me which one and I can help you with it


I also believe even though there is an AzureAD command for removing accounts (soft delete), I do not believe there is its counter part command for restoring?


Soft delete:

remove-MsolUser -UserPrincipalName user@example.com


Hard Delete (delete from deleted)

Remove-MsolUser -UserPrincipalName user@example.com -RemoveFromRecycleBin -Force




Restore User (Soft deleted)

Restore-MsolUser -UserPrincipalName user@example.com

suzanne s. barnhill say:

If each list item is a separate paragraph and the date is the first thing in the item, then you can select the list and sort by date.
