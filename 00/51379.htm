How do you scan your android phone for windows viruses? - Microsoft Community

a. user say:

How do you scan your android phone for windows viruses?

Hi, 

Here's the issue. Several weeks ago I accidentally opened an attachment from my email on my phone, it turned out to be a windows virus, so has no effect to my phone. So I connected my andorid phone to my pc, hoping that I can simply copy all the files on my phone to my pc, and then scan from there. However, these days, Android can only be connected to a PC via MTP (media transfer protocol), it cannot be recognized as a USB mass storage device (MSC), therefore copying system files is a no-go, and I cannot simply point my windows anti-virus software (for e.g. windows defender) to the MTP device, there's no such option.

How do I then scan my android phone for potential windows virus? This is a serious problem, since the virus if left on my phone without being treated, could migrate to my PC eventually some day as I connect my phone to my PC.

Please help !

rob koch say:

You've wound yourself up into worrying about a non-issue here Chris.

Just think for a moment about what you've stated above.  The Windows specific malware file you opened on the Android device couldn't affect it in the first place, so that was a non-issue.

The MTP (media transfer protocol) is designed to move only media files from one device to another, so it wouldn't be transferring random files from other areas of the phone such as an email attachment to another device.  This is especially true if either the file is not a media type (e.g. picture, video or music) or it's not found in the locations that the MTP protocol implementation on the phone is designed to use for such transfers.

Even if all of the above items were a worst case situation and the file was a media type such as a picture or music, the fact that these will be treated by the MTP on the PC as data means that they will be examined by the antivirus there upon transfer and any malware should thus be detected and removed.  Since the MTP transfer itself isn't actually viewing or executing these files, it's not possible for the item to "launch", so the risk is minimal.

So as is typical in such cases, you have combined in your mind all of the worst case scenarios of a malicious file being opened via an email attachment on the PC after transfer from the phone, when in truth that's not what would happening.  In fact even if the file did still exist, it's unlikely it would ever be transferred to the PC for multiple reasons.

The older USB mass storage device system did include some of these problems, since it treated the phone as a filing system, which is exactly why the MTP protocol has replaced it in order to remove some of the most obvious issues that created.
