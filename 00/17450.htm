papersize reports 9999999 - Microsoft Community

rn_1 say:

papersize reports 9999999

I have a macro that assembles a document.  Since I have users from Germany they may create a document on A4 paper.  To prevent this I warn of documents set to other than letter.   However we have a document that was formatted with letter size and starts out as portrait with a change to landscape (via section break).   This document reports 9999999 (not sure of the exact number of 9s).   I checked to IsNull and it reports False.   How can this be?    I have looked at the enumeration and it doesn't have any such value.  
hansv mvp say:

If the range to which PaperSize applies contains multiple page setups, PaperSize will return 9999999, meaning "mixed". This is true even if the actual page sizes are the same but the orientations differ.

 

To avoid this problem you can loop through the sections of the document:

 

Sub CheckLetterSize()
    Dim sec As Section
    For Each sec In ActiveDocument.Sections
        If sec.PageSetup.PaperSize <> wdPaperLetter Then
            MsgBox "At least one section in this document is not set to Letter size!", vbInformation
            Exit For
        End If
    Next sec
End Sub

jay freedman say:
For the obsessive among us:  In VBA the enumeration WdConstants includes the value wdUndefined = 9999999 (&H98967F). That value is returned by any numeric property whose value is not the same throughout the selection or range that owns the property.

dallaswilson1 say:

Access Help Average Monthly Sales by Product and Client

I've up to now been an average Access user.  Now I have to create these reports with code.  


I have to create a report showing client by state by average monthly order by product. I envision this looking something like:

Client 1 | State | Avg Monthly Order | Product 1

Client 1 | State | Avg Monthly Order | Product 2
Client 1 | State | Avg Monthly Order | Product 3
Client 2 | State | Avg Monthly Order | Product 1
Client 3 | State | Avg Monthly Order | Product 1

My average monthly order is a sum of Price, quantity, and shipping, minus any discount applied,  I really don't know how to arrange this so that all the parts work together.

ken sheridan say: This is a matter for a query, rather than VBA code, but you firstly need to define what is meant  by 'average monthly order'.  If it is the average of the number of distinct months in which a customer has ordered products of any type, then you would need to count those district months by means of a subquery, and divide the total amount per month per product by the result of the subquery.  The following is an example using Northwind data:

SELECT Customers.Company, Customers.[State/Province], Products.[Product Name],
SUM([Order Details].[Unit Price]*Quantity)/MonthCount AS [Average Monthly Order]
FROM (Products INNER JOIN ((Customers INNER JOIN Orders
ON Customers.ID = Orders.[Customer ID]) INNER JOIN [Order Details]
ON Orders.[Order ID] = [Order Details].[Order ID])
ON Products.ID = [Order Details].[Product ID])
INNER JOIN (SELECT [Customer ID], COUNT(*) AS MonthCount
                     FROM (SELECT DISTINCT [Customer ID],
                                  YEAR([Order Date]) AS OrderYear,
                                  MONTH([Order Date]) AS OrderMonth
                                  FROM  Orders INNER JOIN [Order Details]
                                  ON Orders.[Order ID] = [Order Details].[Order ID])
                                  GROUP BY [Customer ID]) AS DistinctMonths
ON Customers.ID = DistinctMonths.[Customer ID]
GROUP BY Customers.Company, Customers.[State/Province],
Products.[Product Name],MonthCount;

If, on the other hand, you want the average  monthly order amount per product for those months in which the product was ordered, then you'd have to include the product in the join:

SELECT Customers.Company, Customers.[State/Province], Products.[Product Name],
SUM([Order Details].[Unit Price]*Quantity)/MonthCount AS [Average Monthly Order]
FROM (Products INNER JOIN ((Customers INNER JOIN Orders
ON Customers.ID = Orders.[Customer ID]) INNER JOIN [Order Details]
ON Orders.[Order ID] = [Order Details].[Order ID])
ON Products.ID = [Order Details].[Product ID])
INNER JOIN (SELECT [Customer ID], [Product ID], COUNT(*) AS MonthCount
                     FROM (SELECT DISTINCT [Customer ID], [Product ID],
                                  YEAR([Order Date]) AS OrderYear,
                                  MONTH([Order Date]) AS OrderMonth
                                  FROM  Orders INNER JOIN [Order Details]
                                  ON Orders.[Order ID] = [Order Details].[Order ID])
                                  GROUP BY [Customer ID],[Product ID]) AS DistinctMonths
ON Customers.ID = DistinctMonths.[Customer ID]
AND Products.ID = DistinctMonths.[Product ID]
GROUP BY Customers.Company, Customers.[State/Province],
Products.[Product Name],MonthCount;

Or if you simply want the average monthly order amount per product over a date range, regardless of whether the customer has ordered the product in every month of the range, then you simply need to divide the total per product by the number of months in the date range, e.g. over a year entered as a parameter at runtime:

PARAMETERS [Enter year:] SHORT;
SELECT Customers.Company, Customers.[State/Province], Products.[Product Name],
SUM([Order Details].[Unit Price]*Quantity)/12 AS [Average Monthly Order]
FROM Products INNER JOIN ((Customers INNER JOIN Orders
ON Customers.ID = Orders.[Customer ID]) INNER JOIN [Order Details]
ON Orders.[Order ID] = [Order Details].[Order ID])
ON Products.ID = [Order Details].[Product ID]
WHERE YEAR([Order Date]) = [Enter year:]
GROUP BY Customers.Company, Customers.[State/Province],
Products.[Product Name];

In your case the summation would be of an expression which returns the ((Price*quantity)+shipping)*(1-discount) where discount is expressed as a fractional value, e.g. 0.1 for a 10% discount.
