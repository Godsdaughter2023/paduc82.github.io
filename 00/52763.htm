wunders say: A dead motherboard battery will cause you to lose time when your computer is powered off -- not when powered on.  When the computer is powered up, the system keeps time by counting clock interrupts after setting its time to the BIOS clock.  If a higher priority interrupt occurs and lasts longer than two clock ticks, then time is lost.  Your problem points to some task, process, or interrupt that is consuming too much time in an interrupt-block state. 

The first thing you should try is to boot your computer to Safe Mode (Repeatedly press the "F8" key while booting).  This starts you computer with the bare minimum of programs and processes.  If you find that you don't have this problem in Safe Mode, then the logical assumption is that there is some task, process, or service that starts up with Windows that is causing this problem.  You can start enabling blocks of tasks and startup programs until you find the one that is causing the problem.  Some programs that help you enable and disable tasks are "msconfig" (Start -> Run -> "msconfig") and "autoruns"

Hi Jamie Lang,
 
The error 16500 happens when a child account associated with a parent account, the another parent tries to associate the child account.

Can you please provide us the parent account that you are trying to add the child?

After investigating the child account, I have checked that it is associated with two parent accounts. Have you previously used another account to add the child account?

If you are the owner of the old parent accounts, you can simply remove the child account from the old parent and then associate it with the new parent account. Please follow these steps to remove the child account:

1. Go to the Windows Live Family Safety website, then sign in using the old parent account.
2. Click Manage the list of family members.
3. Check the Child account, then click Remove.
4. Click Remove again to confirm.
