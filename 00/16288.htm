What Ken's referring to is some code s.a. the following:

    Dim ii as Integer, iii as Integer
    Set dbs = CurrentDb()

    Set rst = dbs.OpenRecordset(SQLStmt, dbOpenDynaset)

        ii = ![ACCT]   'get the current highest record number
        iii = .recordcount + 4  'ticker used to add 3 records
            While .RecordCount < iii
                .AddNew

            .UPDATE
            ii = ![ACCT]

john korchok say:

There is a method to PowerPoint's madness. If the placed picture's proportions fill the width, but is short height-wise, the picture will fill the placeholder width, but be not as tall. In this case, there will be extra space above and below, as in your illustration.

I run into clients who think that PowerPoint can perform some kind of magic on mis-proportioned pictures, but it could only fill both the height and the width by distorting the aspect ration of the placed photo.

Instead, prepare you photos for placement in PowerPoint: note the ratio of height to width of the placeholder, then crop the photo in an image editor to match those proportions (warning: this involves math!). Then when you import, the photo will fill the placeholder exactly and the top of the photo will align with the table beside it.

graham mayor say:
The principle is the same for all Word versions. You have added unwanted material to the normal template, which in Word 2010 is Normal.dotm

You can locate this file by typing %appdata%\Microsoft\Templates in the address bar at the top of Windows Explorer. Rename the file to normal.old with Word closed and restart Word. Word will build a new default template.

When Word is again running, from File > Options > Advanced > Save set the options to prompt before saving the normal template and always create backup copy.

Open the normal template in Word. Make your detail changes to it then save and close it.
