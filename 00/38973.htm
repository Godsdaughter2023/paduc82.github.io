cmdrkeene say:

Client OS can't let more than one user access the app interactively over a remote desktop/terminal session at the same time.  If you need simultaneous this way, server is your way to go.

The hack would indeed conflict with your licensing agreement, and would probably cause Windows to become unvalidated and stop receiving updates (and start doing all the things it does to non-activated licenses, like shutting down every hour).  Not good news.

Windows MultiPoint server I think would suite you perfectly, and can absolutely appear to your end user exactly like a client version (they won't know the difference, other than they use Remote Desktop to access it).
