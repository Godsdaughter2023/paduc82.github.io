james yan msft say: Hi Agata82PL,

You can set up unique permissions for this library, give users Edit permission to this library while giving them Read permission to the site.

1. Go to the library > Library settings > Permissions for this document library > Stop Inheriting Permissions.
2. Click Grant Permissions > type the user's name > click SHOW OPTIONS > Select Edit permission level > Share.
3. Go back to the site > Site settings > Site permissions > Grant Permissions > type the user's name > click SHOW OPTIONS > Select Read permission level > Share.

Now, the users should be able to edit the items in the library but they can't modify the site page.

JWH12,

Are you referring to the "auto-complete" feature?  That feature requires you to start typing the first few characters of your username ... it then will show a drop-down list of user names (that you have previously typed) that match those first few characters. The first time you sign in, you will have to type the entire user name so that your browser can remember it. Thereafter, when you start typing, the list of usernames that match what you have started to type will be shown. NOTE: this requires that you do not have your browser set to delete history when it closes (because when your browser deletes history, it deletes the remembered usernames).

Prior to March 2016, the outlook sign on page supported auto-complete. However, during March 2016, Microsoft released a new version of the sign on page, and they have disabled the auto-complete function in the new sign on page!!!

There are many threads in these forums that have been discussing this issue. You can look here and here for more information.

The formulas work provided that the value is found and the adjacent column to the found value (Referenced by Column Index Number) in the lookup array contains data. If the cell referenced by the Column Index Number is blank then zero is returned because that is the value of an empty cell.
