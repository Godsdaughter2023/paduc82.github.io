Function SumByCFColorIndex(Rng As Range, CI As Integer) As Double
    Dim R As Range
    Dim Total As Double
    For Each R In Rng.Cells
        If ColorIndexOfCF(R, False) = CI Then
            Total = Total + R.Value
        End If
    Next R
    SumByCFColorIndex = Total
End Function

Function ColorIndexOfCF(Rng As Range, _
    Optional OfText As Boolean = False) As Integer

Dim AC As Integer
AC = ActiveCondition(Rng)
If AC = 0 Then
    If OfText = True Then
       ColorIndexOfCF = Rng.Font.ColorIndex
    Else
       ColorIndexOfCF = Rng.Interior.ColorIndex
    End If
Else
    If OfText = True Then
       ColorIndexOfCF = Rng.FormatConditions(AC).Font.ColorIndex
    Else
       ColorIndexOfCF = Rng.FormatConditions(AC).Interior.ColorIndex
    End If
End If

End Function

 

Function ActiveCondition(Rng As Range) As Integer
Dim Ndx As Long
Dim FC As FormatCondition
Dim Temp As Variant
Dim Temp2 As Variant

If Rng.FormatConditions.Count = 0 Then
    ActiveCondition = 0
Else
    For Ndx = 1 To Rng.FormatConditions.Count
        Set FC = Rng.FormatConditions(Ndx)
        Select Case FC.Type
            Case xlCellValue
            Select Case FC.Operator
                Case xlBetween
                    Temp = GetStrippedValue(FC.Formula1)
                    Temp2 = GetStrippedValue(FC.Formula2)
                    If IsNumeric(Temp) Then
                       If CDbl(Rng.Value) >= CDbl(FC.Formula1) And _
                           CDbl(Rng.Value) <= CDbl(FC.Formula2) Then
                           ActiveCondition = Ndx
                           Exit Function
                       End If
                   Else

                         Rng.Value <= Temp2 Then
                         ActiveCondition = Ndx
                         Exit Function
                      End If
                   End If

                Case xlGreater
                    Temp = GetStrippedValue(FC.Formula1)
                    If IsNumeric(Temp) Then
                       If CDbl(Rng.Value) > CDbl(FC.Formula1) Then
                          ActiveCondition = Ndx
                          Exit Function
                       End If
                    Else
                       If Rng.Value > Temp Then
                          ActiveCondition = Ndx
                          Exit Function
                       End If
                    End If

Function GetStrippedValue(CF As String) As String
    Dim Temp As String
    If InStr(1, CF, "=", vbTextCompare) Then
       Temp = Mid(CF, 3, Len(CF) - 3)
       If Left(Temp, 1) = "=" Then
           Temp = Mid(Temp, 2)
       End If
    Else
       Temp = CF
    End If
    GetStrippedValue = Temp
End Function

suzanne s. barnhill say:

You can change the respective width of cells in any given row, but if you want the "hanging indent" to be the same for all entries (not necessarily based on word length), then a table solution is not usable.

What should work and would be extremely easy to use is a frame. See the article at http://WordFAQs.mvps.org/MarginalText.htm for an explanation of how to insert and format a frame. The tricky part here is to set the width of the frame, the "Distance from text," and the amount of the hanging indent such that text that wraps around the frame will align with the hanging text. This will require some trial and error, but once you have got it just right, you can define a style that includes this frame. The position of the frame will be defined relative to the text to which it is anchored, and inserting the frame is as easy as applying the style. So you type a paragraph in the style for your entry, then another paragraph in the framed style, in which you put the number. Fine-tuning the frame formatting and defining the style will take some work, but once you have the style defined, the rest is easy.

Shared mailboxes with assigned licenses - Microsoft Community

adowozhere say:

Shared mailboxes with assigned licenses

Our shared mailboxes were migrated to O365 and all were assigned E3 licenses as it was understood that only an E3 license would give litigation hold, which we required.

Subsequently we realized that a Plan 1 license with Exchange Online Archiving would give us the same functionality at a fraction of the cost, and free up those E3 licenses for more productive use.

What do we need to do to change the licenses on the shared mailboxes?

Is there any risk to the data in the mailboxes?  All of the mailboxes we hope to change are less than 20GB in size.

A shared mailbox with a Plan 1 license has a limit of 50GB, right?  Does the addition of the Exchange Online Archiving license to this mailbox increase the limit of the size of the mailbox?

Is there anything else I should be considering in this scenario?

jennifer shi msft say:

Hi AdoWozHere,

 

Thank you for posting your question in our forum. Please kindly check following answers to your questions:

 

1. What do we need to do to change the licenses on the shared mailboxes?

What need to be explained to you is that you do not need to assign licenses to shared mailbox that are under 50GB. After that, if you want to increase the mailbox size to 100GB, an E3 or E5 license must be assigned. So if your shared mailbox don’t need that much size, you can remove the E3 license from it. Detailed steps please refer to this article: Remove a license from a shared mailbox.”

 

2. Is there any risk to the data in the mailboxes?  All of the mailboxes we hope to change are less than 20GB in size.

If you mean that all of the shared mailbox you want to change are less than 20GB in size, then there will no influence with the data in the shared mailbox after you remove the license.

 

3. A shared mailbox with a Plan 1 license has a limit of 50GB, right? 

Yes, you are right. Details please refer to the part “Storage limits across standalone plans” in this article: Exchange Online Limits.

 

4. Does the addition of the Exchange Online Archiving license to this mailbox increase the limit of the size of the mailbox?

First of all, an Exchange Online Plan 1 with Exchange Online Archiving license is used when you want to enable In-Place Hold or a Litigation Hold of a shared mailbox.

 

After you assign an Exchange Online Plan 1 with Exchange Online Archiving license to a shared mailbox and enable both In-Place Archive and auto-expanding archiving for a shared mailbox, additional storage will auto added when the archive mailbox reach to it 100GB storage capacity.

 

So, the answer is yes, it provide a different way let you increase the size of the shared mailbox. Details about this question, you can refer to the “Note” part under the part “Storage limits across standalone plans” in the article I provide in question 3.

 

5. Is there anything else I should be considering in this scenario?

To sum up briefly, if you need Litigation hold on your shared mailbox, then an Exchange Online Plan 1 with Exchange Online Archiving license is available for you. However, if you want to increase the limit size of a shared mailbox with the Exchange Online Plan 1 with Exchange Online Archiving license, you need to enable both In-Place Archive and auto-expanding archiving for a shared mailbox.

 

About how to enable In-Place Archive and auto-expanding archiving, please refer to below articles:

Manage In-Place Archives in Exchange 2016 Note: in Exchange admin center, you may need go to the Recipients > Shared, and then find the Archive option under “mailbox feature”.

Enable unlimited archiving in Office 365 - Admin Help

 

Hope above information helps. Feel free to contact us if you have any other questions.

 

Regards,

Jennifer

To save time hyperlinks are stored in respective cells to be able to open the documents, also stored on SharePoint. These hyperlinks average at 145 characters and work correctly on creation. When the worksheet is saved on closure and reopened the visible text remains the same but the link has been shortened with full stops (..).

How can I keep the correct hyperlinks so that they can then be copy/pasted directly in to an email without additional manual editing?

hansv mvp say:

Activate the File tab of the ribbon.

Click Properties > Advanced Properties.

Enter an x in the Hyperlink base box.

Click OK.

This prevents Excel from using relative hyperlink paths (with ..) in this workbook.

You will have to redo any existing hyperlinks.
