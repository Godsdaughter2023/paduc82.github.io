fran_d say:

All right, I figured this out. On the reinstall, somehow, I opened Outlook with an old profile that included an ANSI PST data file. This is why Outlook jumped to the new Exchange account as the default data file even though I tried to set the PST as the default. The ANSI PST is limited to the number of items that can be stored and is not Unicode compatible. Linking a main form's subform to another main form - Microsoft Community

emerald77 say:

Linking a main form's subform to another main form

I know how to link the data contained in a subform to its main form (Form A) using the Link Master Fields and Link Child Fields properties.

However, I now have a situation where I need to link the aforementioned subform (contained in Form A) to a separate and distinct main form (Form B).

Can this be done, and if so, how?  Thanks in advance for any assistance.
dirk goldgar say:
I know how to link the data contained in a subform to its main form (Form A) using the Link Master Fields and Link Child Fields properties.

However, I now have a situation where I need to link the aforementioned subform (contained in Form A) to a separate and distinct main form (Form B).

Can this be done, and if so, how?  Thanks in advance for any assistance.


Assuming the other form, Form B, is actually open when Form A is open, then you could put a text box on Form A with a controlsource that refers to a control on Form B.  For example, you could have on Form A a text box named "txtIDFromB" with a ControlSource expression like this:

 

    =[Forms]![Form B]![ID]

 

(where "ID" is the name of a control on Form B).

 

Then you could set the Link Master Fields property of the subform to "txtIDFromB".

 

It's not necessary that the text box, "txtIDFromB", be visible.  You can set its Visible property to No, and place it in a convenient, out-of-the-way spot on Form A.

If you have the target Access table ready for import, you can create an import specification file that can be reused. This will remember the cross reference for the Excel No. of payments column against the No of payments Access field as well as specifying a Date format for the long date column. If the long date is not too badly mangled, Access should be able to process it when forced to. An indicator of success would be how well it imports into Excel as a true date and not a textual representation of a date.
