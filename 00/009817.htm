    I find that the easiest way (for me) is to put all of the .csv files that are to be processed into a single folder and then drop in the Excel .xlsm file that will gather up the data into the very same folder.  That way the code can "find" the .csv files very easily (it can determine where it is at during run-time) and then open each one, copy the data and either put it on separate sheets in the Excel file or append it all into one big sheet: your choice.

    jeeped say:
    AFAIK, submitting a second sitemap to Bing via the webmaster tools supersedes the first. In this way only the last one submitted is valid.

    You can reference up to 50,000 URLs which may point to either a URL or a child sitemap in a single master sitemap and each of those child sitemaps can contain 50,000 URLs. This makes the  effective scope of a master sitemap 2.5 billion URLs. This does not mean that you submit the child sitemaps; only that they are references within the master sitemap that is submitted.

    FWIW, I interpret this as "the best practice is one good sitemap". If you have more than 50,000 pages then use a sitemap index (master) that includes references for sectioned sitemap1, sitemap2, etc (child).

    If you are doing research in this yourself, remember that the XML sitemaps are a given standard so topical discussion on this should be very similar between Google and Bing with minor platform differences. Yahoo uses a TXT sitemap standard.

    References:
    Webmaster Blog - Bing enhances support for large Sitemaps
    Sitemap Protocol (sitemaps.org)
