When these "Checking for updates..." hangs occur the Windows Update Agent will saturate an entire CPU core (e.g., 100% CPU activity on a single core CPU, 50% CPU activity on a dual core CPU) and affected users might notice a significant decrease in system performance or hear their cooling fan run constantly when their CPU begins to overheat.  If your Vista OS has Service Pack 2 installed (check at Control Panel | System and Maintenance | System) and you think you might be affected by this problem then see the FAQ How to Fix Vista SP2 "Checking for Updates..." 1. Make sure the WebClient service is running.

10. Type a friendly name for the new network place that will become your SkyDrive folder.


11. Click ‘Next’ and ‘Finish.’ If the box is checked when you click ‘Finish,’ the folder will open.



If this fails to work, open services (WIN key + R, type "services.msc" and press enter) and make sure Webclient is enabled. If it is, restart the service and try the operation again.

Ciao Avijeetshenoy,

I'm Ugo, a Microsoft Independent Advisor willing to help other Microsoft customers.

Regarding your question, as a first troubleshooting option, can you please retrieve your digital Product Key from BIOS:

GET WINDOWS BIOS PRODUCT KEY
Install ProducKey and run it to look for OEM-BIOS key which is the embedded Product Key from factory: http://www.nirsoft.net/utils/produkey-x64.zip.

Then activate Windows with Product Key?

ACTIVATE WINDOWS 10 WITH PRODUCT KEY
Start > Settings > Update & Security > Activation > Change Product Key > Type your Product Key in > Next

ramesh srinivasan say:

Hi,

Graph32.OCX is the VB runtime file for displaying Graph Control in a VB application. If the program vendor doesn't supply this runtime file, you can get it from Microsoft. Simply reinstalling the VB Runtime library should resolve the problem.

 

VB Runtimes:

http://support.microsoft.com/gp/vbruntime
