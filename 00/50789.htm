File Hisitory is making too many PST backups - Microsoft Community

surfer10k say:

File Hisitory is making too many PST backups

I have W10 Pro 64b.

I also have Office 2016 on the PC.

I have been using File history for 3 months now.

My problem is that File History is making way too many PST file backups. In just a few weeks the backup size is 38 Gb while the actual file size is 1gb.

graye say:

Let's start with how File History is supposed to work...

Let's say that you configured File History to perform a backup every hour and to save those backups for 3 months.  So, that means that each hour, it will go find all of the files that have changed during that hour.  If it detects one, it will make a copy of that file (with a date/time stamp appended to the file name).

So, let's assume that you check your email twice a day... that means that your PST file will change twice a day.  So, after a week you'd have:

7 days x 2 times day = 14 copies of your PST file

...and if your PST file was 1 Gb in size, then after a week, your backup would contain 14 copies of that 1 Gb file.   Therefore your backup size would be 14 Gb.

So, what to do about it...

  • You could exclude that file
  • You could change the frequency of the File History
  • You could change the duration period
  • You could just manually go poke around the File History and delete the older versions of the PST file

stevem30 say:

Cannot log into my Windows 7 user account

My Windows 7 user password is not working and I do not have a password reset disk. I've also tried so many passwords that I'm afraid I may have been locked out. Any suggestions? I also cannot get to an Administrator sign-in if there is one.

ramesh.kumar say:

Hi Stevem,

Thank you for posting your query on Microsoft Community.

As per the issue description you are unable to login on your computer as the user account password is not working. In that scenario you need to enable default administrator and then try to change the password of your user account from the default administrator account and see if you get success.

If you have a Windows 7 installation DVD (not a recovery DVD) you can boot the system with it. Select the default language, then choose "Repair your computer". Then select "Command Prompt". At the command prompt type:

net user administrator /active:yes

You should see a message that the command completed successfully. Log out, and you’ll now see the Administrator account as a choice. (Note that the screenshots are from Vista, but this works on Windows 7)

You’ll note that there’s no password for this account, so if you want to leave it enabled you should change the password.


Remove the DVD, reboot the computer, and log into the built-in Administrator account. If the built-in Administrator is truly enabled, you will see an icon for it on the Welcome Screen when you reboot and can get into it.

If you don't have a Windows 7 installation DVD but have the product key then you may refer to the article mentioned below to create an installation media of Windows 7.

Download Windows 7 Disc Images (ISO Files)

After changing the password of your old user account you can Disable Built-in Administrator Account.

Make sure you are logged on as your regular user account, and then open an administrator mode command prompt as above. Type the following command:

net user administrator /active:no

The administrator account will now be disabled, and shouldn’t show up on the login screen anymore.

Hope this would help. If issue still persists post back with current status of your computer and result of proposed suggestion, we will be happy to assist you.

Regards,
