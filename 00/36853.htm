Adding Microsoft account to existing Child Account - Microsoft Community

melissacreech say:

Adding Microsoft account to existing Child Account

When I enter the Windows password, the computer says "Oops, Something went wrong. Whatever happened, it was probably our fault. Please try again." and will not continue. 

I do not wish to create new accounts because the kids would then lose all the apps and games that they already have. What do I do? 

daniel strommen [msft] say:

Hey Melissa,

I'm a software engineer on the family team.  I am pretty sure I know what issue you are hitting and the workaround is a little hairy.

What has happened is that when you went to the "Family and other users" page in the Settings app, it went in the background and created disabled accounts for your kids' Microsoft accounts, in addition to the local accounts that your kids already had.  We call these "stub accounts", and unfortunately once they are created, it won't let you use the same Microsoft accounts for your kids' existing profiles.

What you're going to need to do is, delete these stub accounts and DON'T go to the "Family and other users" page until after you have signed in as each child and connected their Microsoft account.  There are a couple ways to do this.

Solution 1 - Use net user to delete the accounts from the command line

  1. Start by going to "Family and other users" and clicking Block on all the accounts you want to delete.
  2. Close the Settings app and make sure you don't go back to the "Family and other users" page until you're all done, or it will just re-create these stub accounts and you'll have to start over.
  3. Right-click the Start button and click Command Prompt (Admin).
  4. Click Yes in the User Account Control prompt that comes up, to allow Command Prompt to run with admin privileges.  You will need admin privileges to modify the user accounts.
  5. Enter "net user" to list the current accounts on the system.
  6. In that list, try to identify the stub accounts you want to delete.
    • You should ignore accounts with names like Administrator, Guest, and DefaultAccount.
    • The stub accounts are usually named after the first 5 characters of the user's email address.
    • For this example, I'll pretend I found the danie account, because that's how mine would show up.
  7. Make 100% sure it's a stub account, so we don't accidentally delete your kids' real data.
    • Enter: net user danie
    • Look for: Account active: No.
    • If it says Yes, it is active, then don't delete it.
  8. Delete the stub account.
    • Enter: net user danie /delete
  9. If you have more than 1 child in this situation, go through steps 5-8 again to delete any other stub accounts.
  10. Once the stub accounts are deleted, you can sign in as each kid and you should be able to add their Microsoft account successfully.

Solution 2 - Use Local Users & Groups to delete the accounts (Not available on Windows 10 Home)

If you are running Windows 10 Pro, there is a graphical interface for deleting these stub accounts.

  1. Click the Start button and enter: lusrmgr.msc
  2. Go to Users.
  3. Find the stub accounts you need to delete.
    • You should ignore accounts with names like Administrator, Guest, and DefaultAccount.
    • The stub accounts are usually named after the first 5 characters of the user's email address.
    • Any accounts that are blocked should have a little disabled icon on them (arrow pointing down).
    • If an account doesn't have the Disabled icon, then it's active and you shouldn't delete it.
  4. Right-click the stub accounts you want to delete and click Delete.
  5. Once the stub accounts are deleted, you can sign in as each kid and you should be able to add their Microsoft account successfully.

Hi,

Some of the newer Flash Drives have a setting in them that identifies them as a Hard Drive instead of a removable drive... It may be this is the reason that it isn't being recognized. Win 7 Bootable USB Tool won't recognize the newer ones for instance.

You might also check that your Flash drive is formatted properly... Some things still require a FAT32 partition to work..

Part of your Cloning problem is most likely due to the size difference between the HDD and SSD.

Some programs will shrink to fit, and others will not.

I would suggest trying Macrium to clone or create an image based on other peoples experience...

There is a free version...

I myself use Acronis True Image Plus Pack or Premium as it is now called... The regular Acronis should work also... But Acronis doesn't have a free version. It may well be worth the investment in the future though... It's on sale a lot.

I use Acronis to create an Image to a second internal or external HDD... either will work..

I then remove the original HDD, and Recover the Image to the SSD.... Acronis will scale to fit...

You can image the entire HDD, or just the Windows partition... If you do just the Windows partition, you may need to do a boot repair. This is because there are most likely boot files on the other partitions...

Acronis comes on a CD that can either Boot, or install which is nice... Just boot from the CD and your ready to go.

I've recently done this with a HP Laptop without issue... which surprised me a little...

I just Imaged the OS partition, and nothing else. I saved the original HDD in the Laptop box for future use if needed...

You can use a disk management program like Partition Wizard on the HDD to "HIDE" and make "NOT ACTIVE" the OS partition if you still want to use the drive... I've been using version pwhe811

If you wanted, you could use Windows to "SHRINK the OS partition to the size of the SSD first, but that isn't necessary.....

Once your SSD is up and running, make an Image of it also... This is where Acronis earns it's money.

I create a new image of Windows everytime I do any kind of major update or upgrade. Or just something that's a pain, and I don't want to have to do it again....

Acronis also Compresses the image so that it takes up way less space than the actual size of the partition... My 128GB SSD is half full, but the image is only 20GB. I have a couple dozen backup images from different points in history...

It works so well, I even turned off System Restore so it doesn't clutter up my SSD with backup files that I will never need.
