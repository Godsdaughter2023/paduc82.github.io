My Avery labels do not line up since upgrading to Windows 10. How can - Microsoft Community

sarahb159 say:

My Avery labels do not line up since upgrading to Windows 10. How can this be resolved?

Question is in the heading. Has anyone solved this yet?
graham mayor say:

There are four reasons why labels don't align correctly

1. You have added a header/footer to the normal template (rename the normal template)
2. The page size set in the printer properties doesn't match the page size of the label sheet.
3. You have a scaling option set in the zoom section of the print dialog
4. Your printer has poor paper handling. Some printers will slip when faced with label stock and thus are incapable of feeding the label correctly. Cleaning the rollers in the print path may help.

If they were correct before changing the operating system then, everything else being equal, the problem relates to the printer driver, which is from where Word derives its document formatting. Check that the settings in the driver match the page settings of the document.

The root cause of the problem is that Outlook doesn't deal well with large numbers of folders.  I found that when all of the folders are expanded, it takes a long time for the "Move Items" window to open.  However, if I keep the folders "unexpanded", it loads fairly quickly.

After struggling with this issue for a very long time, I finally found a workaround.  I created a folder called "Working Folders" and another folder called "Other Folders".  I put most of my folders into "Other Folders" and put the smaller number of folders I use on a daily basis into "Working Folders".  For most of my work, I leave "Working Folders" expanded and "Other Folders un-expanded.

This workaround is saving me a ton of time and has increased my email productivity, so I'm posting this in case it helps anyone else.  I've seen previous community questions on this subject where many other folks selected "I have this question too", so I know I'm not alone...

Outlook 2003:

C:\Users\<username>\AppData\Local\Microsoft\Outlook

 

Outlook 2007:

C:\Users\<username>\AppData\Local\Microsoft\Outlook

 

Outlook 2010:

If you had upgraded from earlier version of Office then:

C:\Users\<username>\AppData\Local\Microsoft\Outlook

 

Otherwise by default the PST file will be saved in the location:

C:\Users\<username>\Documents\Outlook Files

 

Outlook 2013 preview:

If you had upgraded from earlier version of Office then:

C:\Users\<username>\AppData\Local\Microsoft\Outlook

 

Otherwise by default the PST file will be saved in the location:

C:\Users\<username>\Documents\Outlook Files

 

You may also search for the PST files saved on the system from the explorer:

http://windows.microsoft.com/vi-VN/windows-vista/Find-a-file-or-folder

Note: In the search box you may type *.pst to find all the files with pst extension.

Sub doBeeps(t As Double, b As Long) 'doBeeps (1), (2)    'secs beeps
Dim x As Double, dt As Double
For b = b To 1 Step -1
    Beep
    x = Timer
    Do
        DoEvents

        If dt < 0 Then dt = dt + 86400
