" What's interesting is when I check on the fonts used in both sets of presentations, they are identical in font name ..."

PowerPoint shows you the names of the fonts the presentation was designed with and needs in order to display properly, but it doesn't tell you whether or not those fonts are present on the current computer.  It used to but somebody at MS allowed the feature to be removed.

The simplest thing to do would be to make a list of the fonts that show as needed, then go to Control Panel | Fonts and check to make sure all of the needed fonts are actually there.  

Another thing to check:  PowerPoint can get cranky and make messes of your slides if you don't have a printer driver installed and set as default.  As a test, go to Control Panel | Devices and Printers.  Rightclick the XPS printer driver and set it as default (temporarily only).  Then start PPT, open your presentation and see what it looks like.

joeu2004 say:

Charles wrote:

Is this a workaround for a bug, or is there some logical reason why =B13*B15 won't work?

None at all.  Do not use =SUM(B13*B15).  The use of SUM is superfluous in that context.  The only thing it accomplished is:  entering it caused a recalculation.

In fact, the same thing would happen with the original formula if you simply select B16, press f2, then Enter ("re-enter the formula").

And the same thing would happen with the original formula if you press ctrl+alt+f9 ("Recalculate all formulas in all open workbooks, regardless of whether they have changed since the last recalculation").

See the "Change formula recalculation" help page (click here) [1] for recalculation keyboard shortcuts.

Off-hand, I don't know how you got into the state where the formula in B16 did not track apparent changes in B11:B15.

Wild guess:  you have a chain of circular references in the originating workbook, and that stopped the recalculation cycle "prematurely".

PS.... Since a circle has no beginning or end, it is debatable what "prematurely" might mean.  I mean that the chain of circular-referenced cells might never be consistent since Excel must stop the chain of calculations at some point arbitrarily.  In any case, it is a "wild guess".  The problem might be something else altogether.

greg maxey say:

I don't know why nor do I know of a third party application.  It shouldn't be that difficult using VBA:

Sub ScratchMacro()
'A quick macro scratch pad created by Greg Maxey
Dim oRng As Word.Range
Set oRng = ActiveDocument.Range
With oRng.Find
  .Text = "test"
  While .Execute
    oRng.Select
    If MsgBox("Replace this instance?", vbYesNo) = vbYes Then
      oRng.Text = "replacement text"
      oRng.Select
      oRng.Collapse wdCollapseEnd
      If oRng.End = ActiveDocument.Range.End - 1 Then Exit Sub
      Application.ScreenRefresh
      If MsgBox("Find next instance?", vbYesNo) = vbNo Then Exit Sub
    Else
      oRng.Collapse wdCollapseEnd
      If oRng.End = ActiveDocument.Range.End - 1 Then Exit Sub
      End If
  Wend
End With
End Sub

phillip m jones say:

Arshad' is correct on this one. You can almost doit but you need to use this method if you want to remain on local computer rather than the OneDrive system:

  1. Locate the One Drive Desktop application.
    (Unless you're running a business get the one for Home use.)

  2. Install and set up a local OneDrive Folder.

  3. Now while OneDrive Desktop Application is running,
    (Locate the File you just saved from Word, Excel, or PowerPoint.)

  4. Resize the folder the file is in, so it just wide enough to see name.

  5. Now in Main menu you should see Icon for OneDrive
    (Click on it and click on Open OneDriver folder,
    resize folder window One drive so it is beside the first window.)

  6. Now drag a copy of file (Option Drag) to the OneDrive Folder.

Once this is done first time, all you will need to do is:

  • Save File and go to file location.
  • Open Drive , then open OneDriver folder from drop down menu on main menu bar.
  • Then drag Copy from file location, to OneDrive Folder Location.

In just a few moments to a few minutes (depend on size of file a Copy will be loaded  to one drive).

The folder windows will retain their size and shapes as long as you don't change them .
