2. Save your files and settings on an external hard drive using Windows Easy Transfer (windows.microsoft.com/windows-easy-transfer). arthowe say:

Found this looking for a solution and thought I would update this post.

Actually it is a pretty simple question and answer, but it did take some poking around.

It has to do with the security on Windows 10.

On you Windows 2008 R2 server go into AD for Users and Computers.

Go the the Computers OU

Find the affected Computer and go to properties.

Click the Delegation Tab.

Click the radio button for Trust this computer for delegation to any service.

It will now work.

One last thing is that the policy needs to be updated on the computer.

So you need to log it into the Domain or log into the local admin account on the PC and launch the VPN*.

*The VPN will connect as the local admin.

If using the VPN, Switch users and do a GPUpdate /force.

Good Luck....
