ashish mathur say:

Hi,

The following process allowed me to create a Pivot Table from data in another workbook.  Also, once the Pivot was created, i could simply change the Pivot Table data source to another file.

1. In the Source data file, ensure that the data has a header row with some distinctive formatting such as Bold, Italics etc.

2. Close the source data file

3. Open a new file and go to Insert > Pivot Table > Use an external data source > Choose Connection > Browse for More and now select the Source Excel file > Open

4. Select the relevant worksheet from where you wish to create a Pivot Table > OK

5. Now create your Pivot Table

6. If you now get a new source data file and want to base the Pivot Table on this new file, then select any cell in the Pivot Table and go to Pivot Table > Tools > Options > Change data source > Choose Connections etc. (same as step 3 above)

Hope this helps.

scottgem say:

Well you have the syntax for an INSERT INTO statement incorrect. It is either:

INSERT INTO table (field list)

SELECT field list FROM table

or

INSERT INTO table (field list)

VALUES(field list)

So maybe this is what you need:

strSQL = "INSERT INTO OPITEST (OPI_Id, Prod_Ref, Prod_Ref1, OPI_Order, OPI_DateIssued, " & _

"OPI_DateAssigned, Oper_Id, OPI_DateProcess, OPI_DateComplete, OPI_DateHold, " & _

"Oper_holdoper, OPI_DateResume, OPI_qty, OPI_QtyFinal, OPI_ctry, OPI_comment, " & _

"OPI_Status, OPI_other, OPI_Scan) " & _

"VALUES(" & Me.OPI_Id & ", " & Me.Prod_Ref & ", '" & Me.cboOPIRef.Column(1) & "', " & _
& "', '" & Me.OPI_comment & _

Note that there needs to be a corresponding value for each field in the field list in either the SELECT or VALUES clause. And they have to be in the same order. But an INSERT INTO statement wouldn't be necessary if you were using bound forms.
