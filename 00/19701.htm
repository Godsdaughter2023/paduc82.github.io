My current query:

The table looks somewhat like this (except there are about 50 entries a month):

hansv mvp say:

Start by creating a query to calculate the current and previous month. I've left off tblMachineInformation for simplicity:

 

SELECT tblMachineReading.MachineID, DateSerial(Year([ReadingDate]),Month([ReadingDate]),1) AS ReadingMonth, tblMachineReading.ReadingCount, DateSerial(Year([ReadingDate]),Month([ReadingDate])-1,1) AS PreviousMonth
FROM tblMachineReading;

Save this query as qryMachineReading.

Next, create a new query and add qryMachineReading twice.

Join the two instances on MachineID and on PreviousMonth vs ReadingMonth.

Double click the join lines and specify that you want all records from the first instance and only related records from the second instance.

You can use this to return the beginning and ending readings, and to calculate the difference:

 

PARAMETERS [Start Date] DateTime, [End Date] DateTime;
SELECT qryMachineReading.MachineID, qryMachineReading.ReadingMonth, qryMachineReading_1.ReadingCount AS BeginningCount, qryMachineReading.ReadingCount AS EndingCount, [qryMachineReading].[ReadingCount]-[qryMachineReading_1].[ReadingCount] AS [Usage]
FROM qryMachineReading LEFT JOIN qryMachineReading AS qryMachineReading_1 ON (qryMachineReading.PreviousMonth = qryMachineReading_1.ReadingMonth) AND (qryMachineReading.MachineID = qryMachineReading_1.MachineID)
WHERE (((qryMachineReading.ReadingMonth) Between [Start Date] And [End Date]));

jenicel dri say:

 Hi Nilimesh,


When you synchronize your Exchange mailbox to an Offline Folder file (.ost), the Outlook status bar never shows higher than 3.99 GB when a folder is being updated. Here are some of the several options that you can follow to resolve the issue:

1. Do nothing

Since this problem only occurs during the initial synchronization of the .ost file, if you wait for the synchronization to fully finish, the problem should eventually go away.

If you elect to do nothing, the time to fully synchronize a large folder may take longer than anticipated. If the time to synchronize the large folder is taking longer than desired or if it never fully synchronizes, please consider using one of the following two methods.

2. Reduce the size of the folder contents

Another option is to reduce the size of larger folders by either archiving items or moving them to other (smaller) folders in your mailbox. The best way to clean up your mailbox in this situation is to use either an Online mode profile, Outlook Web Access (OWA), or Outlook Web App (OWA) to reduce the size of a folder.

You can use the following steps to determine the size of a folder in your Exchange mailbox.
1. Right-click the folder in the Navigation Pane and then click Properties.
2. On the General tab, click Folder Size.
3. In the Folder Size dialog box, select the Server Data tab.

The size of the folder is shown as well as the size of any subfolders.

3. Create a synchronization filter for large folders at or near 4 GB

You can instead create a synchronization folder that limits the number of items that are synchronized from your Exchange mailbox to the .ost file. The criteria used for the synchronization folder should be written to ensure the amount of data to synchronize is less than 4 GB. For example, the following steps show how to create a synchronization filter that causes Outlook to only synchronize items received in the last six months.

1. Right-click the folder in the Navigation Pane and then click Properties.
2. On the Synchronization tab, click Filter.
3. On the Advanced tab of the Filter dialog box, click Field, point to Date/Time fields and then click Received.
4. In the Condition box, select on or after.
5. In the Value box, enter 6 months ago.
(Take note that you can use other descriptive text in the Value box for your criteria. For example, you can use 1 year ago to filter on mail that was received within the past year.)
6. Click Add to List, and then click OK.
7. Note the information provided in the prompt advising you what events will take place with the filter applied to the folder, and then click OK.
8. Click OK in the Properties dialog box for the folder.

When Outlook performs the next synchronization of the folder, the following actions will occur due to the synchronization filter:

•Any new or modified items will be synchronized with the folder.
• All items matching the filter criteria will be synchronized to the folder in the .ost file.
• Any items in the folder that do not meet the synchronization filter criteria will be removed from the .ost file (they will, however, remain in your mailbox on the Exchange server).
