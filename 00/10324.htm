You've probably realized by now that almost nothing you learned about using WordPerfect will be applicable to Word. That includes the terminology used to describe parts of the program and parts of documents, as well as the procedures needed to do anything more than just type text.

Let's start with the phrase "insert a template into a document." In Word lingo, a template is a kind of file used as the basis for new documents; each document is associated with exactly one template at any given time. Evidently you have something else in mind, and that is probably what Word calls a "building block". It's a piece of text and/or graphics, possibly containing bookmarks or fields (among other things). I think I'm starting to fill in some of the holes in my understanding of your request. I'll try to give you some best practices for Word. I will say, though, that this project is more complex than I would recommend for anyone who is just getting started with Word and its macros.

First, create a document based on your template. At each place where a surgeon's name should be inserted, type a placeholder such as Surgeon_Name. Also insert placeholders such as Patient_Name (and others as needed).

The text (not including the header or footer) of one complete case report should be stored in your template file as a building block (confusingly, Word also calls these Quick Parts). To do that, select all the text, and click Insert > Quick Parts > Save Selection to Quick Part Gallery. In the dialog, assign a name, change the gallery dropdown to AutoText, and change the Save In dropdown to the template you're using. Click OK.

Then remove the text from the template itself; the macro will insert it, including the placeholders. All that should remain are the headers and footers, which should have the same placeholders where they're needed.

The macro itself can run automatically when you create a new document from the template or when you open an existing document based on the template. To do that, you need two macros named AutoNew and AutoOpen, each of which calls your macro. You can also provide a button on the Quick Access Toolbar or on the ribbon, and/or a keyboard shortcut, to run the macro at other times. The macro should do this:
  • If the document's body is not empty (there is a previous case report), insert a Next Page section break at the end of the document, and unlink the new section's headers and footers from those of the existing preceding section. (There are three of each, for the primary, first-page, and even-page.) Restart the page numbering at 1. Then insert the AutoText entry at the end of the document, creating the "form" for the new report.
  • Open the document full of names, and display the Find dialog. After the find is successful, copy the name, and use it to replace each Surgeon_Name placeholder in the report (including the footer).
  • Request the patient name, and replace each Patient_Name placeholder. Same for record number and other data.
  • Maybe some other steps, such as requesting other names for other placeholders.
  • Put the Selection (the VBA name for the cursor) at the place to start typing.


The "Page x of y" in the footer is created with fields; when the field codes are shown, it should look like Page {PAGE} of {SECTIONPAGES}. That will automatically display the correct page number (since page numbering is restarted for each section) and the number of pages between the section breaks.


There could be other bells and whistles, such as automatically generating a suggested filename for the document during the first Save.

jay freedman say:
Yes, Word on first encounter can remind you of the old computer game with "a maze of twisty passages, all the same". :-( Two things may help. The first is Suzanne S. Barnhill's article about Life after Reveal Codes. It discusses the tools that Word offers to discover the formatting at any particular place in a document. The second is CrossEyes, a third-party paid tool that simulates the WordPerfect code pane in Word.

'Section' is one of those words that has a special and very specific meaning in Word lingo. It refers to the part of a document between two section breaks (which you can insert from the Page Layout > Breaks button). For this purpose, the beginning and end of a document count as invisible section breaks, so every document has at least one section; if you insert a section break somewhere in the middle, you then have two sections. This has nothing to do with formatting (bold, italic) or styles or headings -- only section breaks.

Also, please don't confuse fields with formatting, as they're almost unrelated. A field (in most but not all cases) could be considered to be a tiny program that produces a text result. The {PAGE} field, for example, determines which page it's on and displays that page's number. The {NUMPAGES} field (which is what you usually see in a Page x of y construction) displays the total number of pages in the document. In contrast, a {SECTIONPAGES} field displays the number of pages in the section where the field appears. As a result, if you insert a section break and restart the page numbering at 1 in the section that follows it, the Page x of y construction will work as if each section is a separate document (e.g., Page 1 of 3, Page 2 of 3, Page 3 of 3).

If you want to see the field codes (note these are not formatting codes), you can toggle any single field by clicking on it and pressing Shift+F9, or by right-clicking it and choosing Toggle Field Codes. If you want to see all the field codes in the document, just press Alt+F9. Both of these actions are reversed (show the field results) by repeating the same shortcut or right-click.

The behavior of formatting in pasted text is complicated (unnecessarily so, in my opinion), but it can be tamed somewhat. The first idea is that if you put the cursor immediately after text that has some formatting (bold, italic, whatever) and start to type, the new text will have the same formatting; and if you paste plain text at that location, you get the same behavior. The second idea is that if the source of the new text had some other formatting, you get to decide whether to keep the source formatting or the destination formatting. That's the purpose of the paste options button, which should appear as a little square at the point where you pasted; click it to get buttons for the choices. Also remember that if you don't like the result, you can click Undo and try again.

Also, you can change the default behavior by going to the Options dialog, clicking Advanced, and setting the dropdowns in the "Cut, copy, and paste" group. The Set Default Paste command at the bottom of the Paste Options button's menu is a shortcut to that dialog.
