Excel R squared is Incorrect - Microsoft Community

roryvan tuyl say:

Excel R squared is Incorrect

Excel computes R2 (r squared) incorrectly for the case where a quadratic fit is obtained through a set of data scattered about a parabola.

The correct value of R squared for this case should alway be near zero, due to the definition of R squared.

This raises an important question: how does Excel calculate R squared in the first place?

This result was obtained using the curve fitting feature of a graph in Excel Office 12.

roryvan tuyl say:

So as it turns out, the only problem here is one of communication.

Excel Charts display "R^2" along with the equation of the best fit curve.

I had assumed that this referred to the R^2 of Y vs X, just as the best fit curve refers to Y vs X

But in fact, it refers to the R^2 of Y vs f(X), where f(X) is the best fit function

There should be some clarification in the definition of R^2 and/or the documentation of curve fitting to make users aware of this peculiarity. The numbers are correct, but what they mean was not clear to me, nor would it be to many users.

Thank you for taking this issue seriously. I appreciate your work.

tom_bpc say:

Chris,

Have you tried using <zSubTask.Resources(1).Calendar> instead of <ActiveProject.BaseCalendars.Item(2)>?  That would work for the case when a)There are NO Task Calendars, and b)Every task has exactly 1 resource assigned.  You'd have to add an error trap to handle tasks with no resources.  You'd have to add some more conditions to handle Task calendars (using task.CalendarObject) if that's what your last question is aiming at. 

Keep in mind that no single calendar will work (in a generalized case) if you have more than one resource (with different calendars) assigned to a task or if you have a Task Calendar with "ignore resource calendars" UN-checked.  In these cases, Project computes durations using a blend of the applicable resource and task calendars, so there is no "correct base calendar for a given task."

i can't access my windows defender tool - Microsoft Community

danielmckeown say:

i can't access my windows defender tool

I am experiencing trouble accessing my windows defender tools.
kosh vorlon say:

Hi Daniel,

 

If you installed MSE, then Windows Defender was disabled and should remain disabled or it will cause problems and conflicts.  The same applies to many other AV products that also can't work with Defender (Norton, McAfee,...).  See: http://answers.microsoft.com/en-us/protect/forum/protect_start/windows-defender-and-microsoft-security-essentials/5309cb8d-02e1-40e8-974f-0dcedb9ab9fd.

 

You mention the Defender tools by which I infer you mean Software Explorer (as besides Spyware protection, I'm not sure Defender has anything else of significance - if so, please let me know).   If it's Software Explorer, there are many alternatives that work as well as or or even better anyway (and since it can't be separately enabled without enabling Defender and if Defender is disabled to prevent a conflict, you don't really have a choice).

 

You can't run MSE and Defender at the same time or it will cause conflicts, crashes, freezes, and all sorts of problems, and there's no need to use Defender for Protection and many options for startup-program control.

 

There are at least three ways to manage start-up programs (possibly more, but these are enough). I use all three depending on the circumstances.

 

1. Go to Start / All Programs / Accessories / Run and type msconfig.exe and enter. Go to the Startup Tab and you can check or uncheck the programs you want to run at startup. The two drawbacks are that you get little information about the programs and you can't add programs that aren't already there to the listing. Also, items blocked by Defender sometimes do not appear in msconfig as they are stored elsewhere in the registry. And this one is not as easy on the eyes as Defender either.

 

2. Download the free WinPatrol: http://www.winpatrol.com/download.html and you will not only have control over those programs, but you can also add new ones AND, what I like best, you can allow it to run in the background and it will alert you if anything tries to add itself to the startup menu without your knowledge (as sometimes happens with some downloads and installations). You don't have to use that feature if you don't want, but I find it convenient. It also provides more information than msconfig. I find it better than Defender and would use for startup-program control it even if it were possible to run Defender.

 

3. Download the free Autoruns: http://technet.microsoft.com/en-us/sysinternals/bb963902.aspx. This is the gold standard for controlling startup programs and everything else at startup with loads of details and options and different views. The danger here is that it is so good and powerful, that if you're not careful, you can remove something you really don't want to remove and hose your system (but if you pay attention and show use caution and don't change anything unless you know EXACTLY what you are doing and the ramifications of doing it, you'll be fine).

 

I rarely use msconfig. I mostly use WinPatrol. When I'm really into cleaning things up and need to know exactly what is happening a boot and startup, then I go to Autoruns to get all the details I need.

 

This will cover managing start-up and boot programs and processes and everything related much better than Defender ever did. And if you choose to run WinPatrol only on-demand and not with real-time scanning, then none of them will take up any resources unless you are using them.

 

I hope this helps.

 

Good luck!
