Build 10130 Pro & Enterprise - Install.esd is preserved after - Microsoft Community

don cuthbert say:

Build 10130 Pro & Enterprise - Install.esd is preserved after Windows Update install (It's about time!)

Build 10130 Pro & Enterprise - Install.esd is preserved after Windows Update install.  In the past, this file was deleted once Windows Update proceeded with the install of a Windows 10 Build.

I was happy to discover that now the hidden C:\$Windows.~BT\Sources\Install.esd is preserved (along with the rest of $Windows.~BT contents) even after the Windows Update has rebooted and completed the install.

This allows easy creation of the new ISO from the Install.esd file using the ESD Decrypter script (esd-decrypter-wimlib-4) instead of having to copy the Install.esd file before Windows Update proceeds to automatically reboot and install the upgrade.

Download for ESD Decrypter  - this runs a script that uses Microsoft software to convert the ESD file.  Since the ESD file came from Windows Update, it is genuine.

Download for 7-Zip to extract above file.

don cuthbert say:
This ESD is produced during the processing of the Windows Update install for each Windows 10 Build.  In previous Builds, this file vanished after the initial reboot following the download of the latest Windows 10 Build.

This file is authentic, as it is created during the Windows Update install and is used in that process.

What I did with Build 10130 (as well as previous Builds) was to wait until the Windows Update download portion was completed.  While it was processing, I watched my C: drive until I saw that this esd file had been created (folder C:\$Windows.~BT is a hidden folder) then waited a bit longer to ensure it was done writing to the file.  Then I copied the esd file to another location on my hard disk to preserve it.

I was able to produce a Build 10130 ISO from it, before Windows Update was ready to reboot and I used that ISO to install on my laptop without having to re-download via Windows Update on the laptop.

The point of posting this was that Build 10130 did not delete the esd file, even after Windows 10 was successfully installed on both Enterprise and Pro versions.

I also put in a suggestion via the Windows Feedback App that Microsoft should produce a utility to convert this ESD file into an ISO automatically, during or after the Windows 10 RTM Windows Update installs, which begin July 29th.  This would certainly save on bandwidth, as those who want to have an ISO would not have to download it, except as part of the automatic Windows Update process.

(The bottom line was that I had a working ISO of Build 10130, before Windows Update had even completed on my first computer, on Day 1 of availability).
