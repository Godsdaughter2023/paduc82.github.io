kirk b. - microsoft support say:

Unfortunately, Maury, I think you're running into a catch-22 situation with data collection.  To update data for existing records, you have to specify email addresses from a source in the database.  If the table you're using has the email addresses associated directly in them, that's easy to do--you just select the email field.  In the case of the Project Management template, it's also easy to do because as long as you're using the table as the source, you can select a related table as having the email address.  The problem emerges once you start talking about filtering the results.  The act of using a query so that you can add criteria leaves you in a spot where you can't select that associated table option (Access fails to understand that the Employees table is still associated with the Projects table) and if you add the Employees table and the email address field to your query, you don't have the option to set up data collection to allow updates; it only allows you the option to add records.

 

The only way I could see around this behavior is to use some sort of temp table that includes the records you want to update from the Projects table, and the email address from the Employees table, but we're talking about a de-normalized, really hard to maintain approach here that would require you to also run an update query after the replies were processed.  That's my best guess and I haven't tested it but it might work for you. 

 

Sorry, I know that's probably not the answer you were looking for.

 

Kirk Beller

Microsoft Online Community Support
