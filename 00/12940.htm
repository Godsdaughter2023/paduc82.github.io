jim gordon mvp say:

Hi Jenny,

You are absolutely correct. Excel on the Mac does not support Excel Data Models. 

Sorry I had to call out Joe for jumping in. There isn't a way to send a message directly to someone who is abusing this forum to ask them to stop.

The way around the problem is by doing manually what happens automatically within the data model in the Power tools in Excel for Windows. 

You didn't say which ODBC driver you're using, and in this case it makes a difference. The AcutalTechnologies Access driver lets you query an Excel workbook as a data source, so you have to use this driver.

What you do is have an intermediate workbook with two worksheets in it. One worksheet queries one database (select * from tablename or any subset desired), and the second worksheet queries the other database (select * from tablename or any subset desired). So you have two querytable result sets. 

Because you can use Excel as an ODBC data source, the next thing to do is join two the two datasets. ActualTechnologies driver lets you use the sheet name as a table name in a data source. You make the join query in a new workbook if you need to be able to refresh the query.

If you don't need to refresh the join query, you can make the join query within the same workbook as the two querytable result sets. It may seem odd to run a query against two querytables within a single workbook, but you can do it as long as you don't want the ability to refresh final result set.

And Jenny - you have my admiration for being among the first in the world to know you can do this! I really want to make a YouTube video to show you how to do this.
