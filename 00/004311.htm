My security essentials is red and remains after restarting. I am - Microsoft Community

caystratton say:

My security essentials is red and remains after restarting. I am unprotected - what do i do?

My system crashed.  When i rebooted the security essentials came up red and the dialogue box said "Security Essentials isn't monitoring because program's service has stopped.  Restart the computer."  I have restarted my pc (desktop) repeatedly and the same box reappears and it is still red alert.  It is very scary.  How do i deal with this?
kosh vorlon say:

Hi Cay,

 

As a general rule, when this occurs either click on the icon in the notification area (which will probably be red) or go to Start / All Programs / Microsoft Security Essentials and double-click to open it. You should be on the home page and it should be red and there should be a big button that says something like "Start Program" or "Start" or something like that. Click on that and the program should open and protections should be enabled and the red should turn to green. That should do it. If it continues to happen at every boot, then proceed on.

 

If that doesn't work, go to Start / Control Panel / Administrative Tools / Services and scroll to Microsoft Antimalware Service and click to open it and confirm it is started and automatic. If not, put it on automatic and then start it. If it will not start, click on the Dependencies tab and check the services on which that service depends. Do the same with them if needed and keep going back until you have them all started and automatic and then go back to Microsoft Antimalware Service and try again to put it on automatic and start it. If that works, then reboot and try to start MSE (or see if it starts on its own as it should). If so, the problem is resolved. If not, do the same thing you just did above except this time the target is the Windows Update service. Follow exactly the same procedures and see if that helps resolve the problem.

 

*************************************************************************

 

If that didn't work before getting more complicated, let's see if a simple fix resolves the problem:

 

Go to Start / All Programs / Run and type regedit.exe and enter.

 

Please check out the registry here

 

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\DisallowRun and see if it has msseces.exe in it. If so, delete the msseces.exe entry and see if that resolves the problem.

 

****************************************************************************

 

Next, let's make sure it isn't a simple startup configuration problem.

 

There should be an entry in MSCONFIG (Start / Run / msconfig.exe / Startup Tab) for MSE. Make sure it is checked. If not checked, reboot and see if that did the trick. If not, proceed.

 

If not there or that didn't work, go to the icon you use in the Start menu for Microsoft Security Essentials. Right click on it and click on properties to see the location. it should show the path to a file called msseces.exe. Make a note of that path.

 

Go to Start / Run and type Regedit and enter. Go to HKEY_LOCAL_MACHINE\Software\Microsoft\CurrentVersion\Run or HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
(it may vary based on your version of Windows) and click on run to see the entries. Is there one that references the above file?
