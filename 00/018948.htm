dale howard [mvp] say:

Brian --

Either Fixed Units or Fixed Work as the Task Type setting will yield the results you seek, BUT you must manually enter your progress in the Actual Work row of the timephased grid on the right side of either the Task Usage or Resource Usage view.  For example, a team member was scheduled to work 40 hours during week #1.  You enter 24h in the Actual Work row for that resource on the task in question in the Task Usage view.  If the Task Type for that task is either Fixed Units or Fixed Work, the 16h of uncompleted work will be added at the end of the task, thus extending the Duration of the task and extending the Finish date as well.  Entering the work manually in the timephased grid is the requirement.  Otherwise, if you enter the actual work in the Actual Work column of the Work table, you will need to manually reschedule uncompleted work from the past into the current reporting period.  Hope this helps.

chrispeng_msft6626 say:

Hi Frank,

If you want to keep the email in the sent items folder of the shared mailbox when replying to an email sent to this shared mailbox, we can use PowerShell to enable the shared mailbox to keep a copy of the emails sent as or sent on behalf of it in the sent items folder of it. Follow the steps below:
1. Connect to Exchange Online using remote PowerShell with an admin account.
2. Run two cmdlets:
Set-Mailbox -Identity *** Email address is removed for privacy *** -MessageCopyForSendOnBehalfEnabled $true
Set-Mailbox -Identity *** Email address is removed for privacy *** -MessageCopyForSentAsEnabled $true

About your second concern, for OWA, we can use the cmdlet below to enable the From field as default for all users:
Get-Mailbox | Set-MailboxMessageConfiguration -AlwaysShowFrom $true

For the Outlook client, it’s not feasible to enable the feature from admin side.

nancy dietz say:

Using different fonts in the same family

Hello - I have a font that has installed two versions of the same font (regular and italic) in the same folder.  When using the fonts in Word, I can see the font folder name and access on font (regular) but I can not work out how to access the other font in the same folder.  

rohn007 say:

You get it by applying the italic font attribute to the base font.

In a "proper" font, the font maker is the one to design how the characters look when the italic, or bold attribute is applied. If there is no maker supplied character the printer "simulates" the attribute.

nancy dietz say:

Hi - That's what we thought as well but when we select Italic the font looks different to the font that we can see in the font info page.   Thanks,

rohn007 say:

The display on screen it not always identical to the printed output.  In theory, the OS controls font display, but in an application like Word that is WYSIWYG, the application takes over to a degree. 

Have you tried printing and comparing that to what you see on the font info page?

There are several subtle ways that Word and Windows manipulate character display on screen that are different from how the characters are printed, different media characteristics ...

In Word, go to file menu > Options command > Advanced option > Display section.

Try changing the settings (one at a time) for "Optimize character positioning ..." and "Disable hardware graphics acceleration".

You could also see if you have the option to change the Windows "hardware graphics acceleration" option. Not all video drivers allow you that control
