a. user say:

To put something in the QAT or Ribbon on Mac Word (I'm using version 15.41) the outline is as follows:

1. Create a suitable macro and put it either in Normal.dotm or another .dotm that is loaded automatically at Startup. Let's assume the macro is called "CalcNow" and you put it in a module called Module1 in Normal.dotm

2. Go into Word->Preferences...->Ribbon & Toolbar and and select the appropriate option at the top (Ribbon or Quick Access Toolbar)

3. In the Choose Commands From dropdown near the top left of the dialog box, choose Macros.

4. You should see Normal.Module1.CalcNow in the list. Select it.

5. If you are adding to the QAT, click the right-arrow button in the middle of the dialog box, then close the dialog. I think the macro will then appear with a "circle" icon in the QAT. You can use the "Preferences Wheel" below the list of QAT item to modify the display name of the option (e.g. CalcNow or Calc rather than Normal.Module1.CalcNow.

6. If you are adding to the Ribbon, you will need an existing custom Tab and Custom Group as with the Windows version - you click the "+" icon at the bottom of the list of Tabs to add those things, and the "Preferences Wheel" icon to rename them. Then select the group and click the right-arrow button between the two columns to add the macro. You can then rename the display name as described in (5). I think in this case the tab/group will show a blank button with the display name underneath, but I do not know what the easiest way is to change the button icon.

Test the options, then close and re-open Word to ensure they are still there.

In theory, I think the following VBA code should work for this:

Sub CalcNow()
Selection.Calculate

End Sub

Unfortunately, that does not actually work, so we have to use a technique I really do not like because WordBasic disappeared over 20 years ago:

Sub CalcNow()
WordBasic.ToolsCalculate

End Sub

To insert that in Normal.dotm

1. Use Word's menu Tools->Macro->Visual Basic Editor to start the VB editor (VBE). If you do not see that option, I suggest you use Word->Preferences->Ribbon & Toolbar->Ribbon, check the box next to the Developer tab to enable it. Then in the Developer Tab, click the Visual Basic icon at the left had end of the tab.

2. You should see a list of projects on the left (probably "Normal (Normal.dotm)" and a project that represents a currently open document). Select the Normal Project.

3. At the top of the screen, the menu titled "Word" is now a VBE menu. Click the menu option Insert->Module. the VBE should insert a module called Module1, and show an empty window to the right of the project list.

4. Copy and paste the code I provided above into that module.

Then follow the instructions I provided above for adding that macro into the QAT. I would suggest that you try to test it first - if you are familiar with VBA, that will probably be straightforward for you, but otherwise it is rather too much for me to explain in one post.

Couldn't break links to images in Word 2010 [SOLVED] - Microsoft Community

phoenixofmt say:

Couldn't break links to images in Word 2010 [SOLVED]

I hope this can save someone else some time since I couldn't find anything to directly address my problem.

I was working with a large number of label sheets in Word 2010. I created a single document for each item that needed a label and then collected the individual sheets as needed in master documents. In order to save space, I linked the 700KB logo that appears 27 times in every document.

It quickly became apparent that I shouldn't link like this since my label sheets will need to move to a network drive. Since Word documents don't support relative paths for linked images, IT IS IMPOSSIBLE FOR A DOCUMENT WITH LINKED IMAGES TO EXIST PROPERLY FORMATTED OUTSIDE ITS ORIGINAL DIRECTORY. They cannot be shared, they cannot be moved. (Unless you start with a lot of coordination on the server and network drives that is probably much more expensive than the cost of the storage, and then leave everything in place for all time.)

At some point, I moved a root folder in the structure that held my documents from E:\Users\<user>\Documents\User\<company>\<product line>\Documents\labels\ to E:\Users\<user>\Documents\<company>\<product line>\Documents\labels\ (don't ask).

The images still appeared in my documents, but now that I wanted to break the links, it was impossible to store them in the document (because of the broken link, ironically.) On several documents, I highlighted all the sources in the Links dialog and "Changed Source..."ed them, double-clicking the logo in the correct path 27 times for each document. After this, I could save the image to the document and break the links. (This is a solution, but time-consuming in the exported master documents with a few hundred linked images each.) This was when I noticed that the path displayed didn't quite match the path to the image. I tried recreating the broken path to hold a copy of the image but Word would have none of it.

The fix that saved me some time (and wear on my mouse) was to:

1.  Open the document (.docx file) in an archive program (I used 7-zip) and find

word\_rels\document.xml.rels

2.   Copy this out of the archive and open it in a text editor.

3.   Find the Relationship tag with the incorrect path  in the Target="..." field and fix it. (1 reference shared by each instance in the document.)

4.   Save and then copy the file back into the archive over the old copy.

5.   Open the document in Word and store the image in the document before breaking the links.

Luckily, I only had a single image in my documents. If you have many this could be a bit complicated, depending on how dispersed the images are.

Hope it helps,

PhoenixofMT

Restrict Sharepoint online users to do share with READ only permission "only"

Hi,

Thanks,

sheen valentine ali msft say:

Hi Thomas,

No, when Sharing is allowed (users with Edit permissions and above has permission to share), users also has the option to choose what permissions to grant to the invite (View or Edit). There is no option in SharePoint Online giving users the ability to only share with Read permissions.

As a workaround, if you want to share a site, but you also want to restrict external users from gaining access to some of your organization’s internal content, consider creating a subsite with unique permissions that you use exclusively for the purpose of external sharing.

More information on Sharing best practices can be found in the link below:

Manage external sharing for your SharePoint Online environment.

If you would like there to be an option to restrict users to only sharing Read permissions invites, given the situation, we suggest you submit your idea via UserVoice; this is the best way to influence our products and services to make it better for you and others.
