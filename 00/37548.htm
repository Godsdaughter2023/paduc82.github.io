afzal taher say:

Hi,

This behavior may occur if you install a third-party program that replaces some of the Windows dynamic-link library (DLL) files with those of a different version.

This error occurs when mismatched DLL files try to reference a function in Msvcrt.dll.

I would suggest you to run a System File Checker scan.

SFC scan will make Windows replace corrupt or missing system files on your computer.

Follow these steps:
a. Click Start, click All Programs, click Accessories, right-click Command Prompt, and select Run as Administrator.
b. Click Continue or supply Administrator credentials if prompted.
c. In the Command Prompt window type the following, and press Enter:
sfc /scannow

Refer this KB article for more information:
http://support.microsoft.com/kb/929833
