Well, yes, it probably is a rounding problem.  But because your description is unclear to me, it is difficult for me to say exactly where the rounding problem is and exactly how to fix it.  Perhaps the following general comments will help.

First, formatting alone to 2 decimal places does not remove any rounding problem, unless you have the "Precision as displayed" calculation option set (not recommended).  Formatting only rounds the actual underlying value.  For example, if A1 contains 1.005 formatted to 2 decimal places, it will appear to be 1.01; but A1+A1 will be 2.01 instead of 2.02.

Second, generally, it is prudent to explicitly round arithmetic expressions that you expect to be accurate to a specific number of decimal places, for example dollar and cents.  This avoids infinitesimal anomalies that arise in computer (binary) arithmetic.  For example =ROUND(SUM(D36:D37),2).

I do not think these "anomalies" are the source of your penny-off error.  But explicit rounding also remedies the problem that formatting alone does not fix.  Two fixes for the price of one :-).

From my weak understanding of your problem, I suspect the explicit rounding is needed in D24 and/or D33, whichever contains an arithmetic expression instead of constant.  You might also need to explicitly round the source of -930 in the first column, if that is not a constant.
