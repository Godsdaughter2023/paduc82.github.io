daniel g. grau say:

When Word does not have a font, it substitutes it for another it considers "similar." Thus, Hypatia will appear in font menus, but won't be the real thing.

In W2011, you could verify this behavior and even reassign the substitute font and convert it to something else in Word > Preferences > Compatibility > Font substitution.

W2016 has its own set of fonts now, inside the application's package. However, you can add fonts to your user Fonts folder and they will be listed by Word. To add a font to this folder:

  • Go to the Finder
  • Press the Option key and click on the Finder's Go menu
  • Open Library (as you realize, this is a hidden folder)
  • Open Fonts and place your new fonts here
  • Any unwanted font can be placed in Fonts Disabled, a folder located next to Fonts (if you don't see it, just create it)

You'll probably need to close and reopen Word for it to become aware of changes in available fonts.

vasil l. michev say:

There's no such thing as "password replication only", the corresponding user objects will have to be managed on-prem. As for the other object types, probably the easiest way to exclude them from the sync process is to use OU-based filtering. OU-based filtering however applies to all objects in said OU, so if you are not willing to reorganize your OU structure it might not be the best option. In which case, a "regular" attribute based filtering should be used.

The different filtering methods are listed in details here: https://azure.microsoft.com/en-gb/documentation/articles/active-directory-aadconnectsync-configure-filtering/#configure-attribute-based-filtering

vasil l. michev say:

The precedence should be lower than the default rules. The RecipientType/RecipientTypeDetails attribute value will not be a string, check the values here: https://blogs.technet.microsoft.com/johnbai/2013/09/11/o365-exchange-and-ad-how-msexchrecipientdisplaytype-and-msexchangerecipienttypedetails-relate-to-your-on-premises/

You can use every attribute that is available from the Synchronization Rule Editor dropdowns. Basically, every "regular" attribute available in the local AD Schema (no constructed or backlink attributes).
