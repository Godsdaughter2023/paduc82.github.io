lanista say:

There usually 2 switches. -k is a kernel fault check and -u is a user fault check. Since Microsoft mentioned that dumprep.exe is a non-essential system process, we can disable it. To disable dumprep.exe from startup, to Control Panel > System > Advanced tab > click Settings button for Startup and Recovery. At write debugging information, click the drop down menu, select (none) and click OK.

Next, click the Error Reporting button, and select “Disable error reporting“. You can leave the option “But notify me when critical errors occur” checked. Click OK to close Error Reporting window.

Go to Run and type msconfig. Check if dumprep.exe still exist in the list or not. If it is there, run regedit and navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run. Select KernelFaultCheck name and hit delete. Restart your computer.


Alternatively you can disable any programs that start automatically by using msconfig.

steve makohin say:

THE SOLUTION

 

 

FORWARD:

 

I'm migrating from Windows XP Pro SP3 (32-bit) to Windows 7 Pro SP1 (64-bit). I used Windows Easy Transfer Wizard ("Wizard") to migrate data and settings from the old PC to the new via an external USB disk drive. You can download the Wizard for your old PC here: http://windows.microsoft.com/en-CA/windows7/products/features/windows-easy-transfer

 

As a programmer, I reasoned (contrary to Microsoft's instructions), that it makes more sense to install apps on the new PC before doing a Wizard import. I have over 70 apps to install, and my concern was that a subsequent data and settings import might potentially overwrite or misplace application settings. As it turns out, it's best to install apps first. Details later.

 

 

WHAT THE WIZARD TRANSFERS:

  • It'll create your old User Accounts on your new PC
  • Some Account settings (like open folders in new window), but is misses most of them
  • Everything to do with Internet Explorer (shortcuts, settings, etc.)
  • Outlook's PST file (nothing else)
  • All user data that you've stored where your old version of Windows knows about it (i.e., Desktop, My Documents, My Music, My Pictures, My Videos)
  • Shortcuts: When the Wizard moves shortcuts, it updates them to point at the correct object, providing the target object exists. For example, if you have a Desktop shortcut to a Word document in ...\My Documents\Roger\Accounting\ on your old PC, that shortcut will be updated to point at the same file in ...\Documents\Roger\Accounting\ on your new PC. Also, if you have a shortcut to an app, and that app is already installed on your new PC, the Wizard updates the shortcut to point to the correct app on your new PC. If the Wizard can't automatically resolve what the shortcut points to, is simply does not transfer the shortcut. You may find some of your shortcuts missing on your new PC.

As it turns out, Microsoft's statement that the Wizard transfers "settings" is limited to a few Microsoft apps. Very few.

 

 

INSTALLERS RUNNING ON WINDOWS 7:

 

Windows 7 lets you easily relocate your Personal Folders (e.g., My Documents, My Videos, My Music, etc.) This is useful, for example, if you have a 700 GB iTunes library you'd rather store on a drive other than the default C:. Unfortunately, some installers (and some apps) assume that Personal Folders never change their name or location. This assumption will cause app failures of some sort when the app tries to deal with a folder that doesn't exist.

 

If you decide to relocate or rename any of your Personal Folders:

1.    Take note of the original location and name (File's preferences -> Location tab)

2.    Relocate and/or rename the folder (renaming it in the same directory doesn't make much sense)

3.    Create a folder in the original location, using the original name.

 

 

INTERNET EXPLORER 8 (default):

 

Windows 7 ships with Internet Explorer 8. The Wizard does an outstanding job at Internet Explorer 8 (XP) to Internet Explorer 8 (Windows 7) transfer. Everything is transferred: Settings, favorites (in their correct order!), news feeds, web site User ID's and passwords, cookies, auto-completion entries, menu bar customization. It's all transferred. I don't use custom toolbars, so I don't know if they would get transferred too (I suspect not, as they're usually DLLs). Microsoft has done very nicely here.

 

 

MICROSOFT OFFICE 2010: 

 

I'm migrating from Office 2010 (Word, Excel, PowerPoint, Outlook) on XP to the same version of Office on Windows 7. The Only settings the Wizard transferred were those contained in the Outlook.pst file (details below).

 

 

OUTLOOK 2010:

 

Before you start you Wizard import on your new PC, make sure you launch Outlook and specify where you want your .pst file to be located (you can use the default location if you want). It's important to delete the .pst file on the new PC before you start the Wizard import. After the import, Outlook will contain everything that the .pst file contains, which includes:

  • Email, calendar, contacts, tasks, notes (I don't use news feeds, so I didn't test)
  • Your various folders and their correct organization
  • All your folder views, including conditional formatting
  • Mail rules
  • Categories

What you'll have to do manually:

  • Transfer all email accounts from your old PC. Google is your friend for extracting these registry entries from your old PC and importing them into the new.
  • Toolbar customizations. File (menu) -> Options -> Customize Ribbon -> Import/Export (button). Export from old PC, import to new.
  • Email Send/Receive groups
  • Signatures. Google it.
  • Hide/Show various panes using the View tab
  • Keyboard shortcuts

 

WORD, EXCEL, POWERPOINT: 

 

What you'll have to do manually:

  • Toolbar customizations. File (menu) -> Options -> Customize Ribbon -> Import/Export (button). Export from old PC, import to new.
  • Keyboard shortcuts

 

ALL OTHER APPS' SETTINGS: 

 

Very few apps (e.g., iTunes) have many of their settings in a folder in one of your personal folders (e.g., My Documents, My Music, etc.) Most apps store their settings in the registry or in hidden Application Data directories. Transfer these yourself, but only if you know what you're doing. Otherwise, manually enter the settings yourself in your new PC. This can take some time, depending on the complexity of your apps' settings and how many apps you have.

 

 

CONCLUSIONS: 

 

Windows Easy Transfer Wizard, used for transferring data and some settings from your old PC to a new Windows 7 PC, is a huge step up from previous wizards, which would often wouldn't even complete running. This one runs, at least. It'll transfer all your data, all Internet Explorer's settings, and a handful of User Account settings, but for the most part, it's up to you to migrate your settings from your old PC to your new one, and this can be an arduous task to get your new system configured nicely, like your old one.

 

I rate the Windows Easy Transfer Wizard 3.5 out of 5 stars.

rohit siddegowda say:

Hi Merlyn,

 

Code 43: Windows has stopped this device because it has reported problems. This circumstance can occur if the device hardware fails or if the device driver fails.

 

Method 1:

I would also suggest you to run the Hardware and devices troubleshooter to troubleshoot the device:

a. Press Windows key to launch the Start screen.

b. Type “Troubleshooting” without quotes and click on Troubleshooting.

c. In the new window, click View all on the left pane.

d. In the list of options, click on Hardware and devices.

e. Click Next and follow the on-screen instructions.

 

Method 2:

I would suggest you to uninstall the drivers completely and re-install them:

a.     Swipe in from the right edge of the screen, and then tap Search.

(If you're using a mouse, point to the upper-right corner of the screen, move the mouse pointer down, and then click Search.)

b.    Enter “Device manger” without quotes in the search box.

c.     Tap on Device manager.

d.    Right-click on the device driver and click Uninstall.

e.    Restart the computer.

 

Refer to the link to update the drivers:

Download and install drivers

http://windows.microsoft.com/en-in/windows-8/all-drivers

 

Method 3:

I would suggest you to disable and re-enable the USB controllers. The following steps will uninstall all the USB controllers and then re-install them:

a.     Right click on Start, and then click Run.

b.    Type “devmgmt.msc” without quotes, and then click OK. Device Manager opens.

c.     Expand Universal Serial Bus controllers.
Note: You might have to scroll down the list to find this item.

d.    Right click the first USB controller under Universal Serial Bus controllers, and then click Uninstall to remove it.

e.    Repeat step 4 for each USB controller that is listed under Universal Serial Bus controllers.

 

Note: Restart the computer. After the computer starts, Windows will automatically scan for hardware changes and reinstall all the USB controllers that you uninstalled.

 

Hope it helps. If you have any further issues on the computer, please post your question related to Windows and we will be happy to help you.
