srinivasu guntimadugu say:

Check in safe mode:

Hold the Ctrl key and click on Outlook icon and open in safe mode, now check with Outlook. If you are able to work without any issues then it is due to add-ins you were not able to open. Follow the steps below and disable add-ins in Outlook 2010.

1.   Open Outlook

2.   Click on File> options> Add-ins

3.   Click on ‘GO’ beside Manage – COM Add-ins and uncheck the add-ins and click ok.

4.   Close and open Outlook and check to open add-ins

 

Also check if Outlook is running in the compatibility mode.

 

In Windows 7 64bit -

C:\Program Files(x86)\Microsoft Office\Office12\outlook.exe > right click on it > properties > compatibility > Uncheck “Run this program in compatibility with…” and check if Outlook is working.

In Windows 7 32bit -

C:\Program Files\Microsoft Office\Office12\outlook.exe > right click on it > properties > compatibility > Uncheck “Run this program in compatibility with…” and check if Outlook is working.

 

If the above steps do not fix the issue, create new profile and check.

rohn007 say:

What are the specific customizations you require and are losing?

Lets see if I understand the process (and problem):

  • user logs in to network
  • network login script copies custom normal.dotm to appropriate location on user machine
  • user starts Word
  • Word "replaces" the custom template

If that is the scenario I used to do exactly that with no problem at work.

If that is the problem, the first thing I would do is confirm that the custom template was created using Word 2010. If it was created in an earlier version, then YES, Word will automatically generate a new 2010 specific normal.dotm. BUT, it is supposed to copy customizations over from the older format file so that there should be no visible difference to the user.

There are a couple of options to consider. 

#1 Doug has already suggested putting the tempate into the startup folder

#2 use a name other than normal.dotm, and put it in the user template folder.  Then Create a custom shortcut on desktop or start menu or Word QAT that explicitly loads the custom templae

#3 create a workgroup template folder and put the template there. Again, create a custom shortcut to load word using that template
