ronrosenfeld say:

It is also there in 2007.   And note that the A5:A6 merge has disappeared.

Excel has always treated merged cells in unexpected ways, making them not always intuitive to use.

Here is what is happening, I think.

  •  When you enter A B and C into the merged cells, the contents actually go into A1, A3, A5
  • Note that A2, A4 and A6 are blank and, of course, they are duplicates
  • The RemoveDuplicates function looks at every cell, so it will remove A4, and A6
  • When it removes a cell, those below are moved up.
    • So the contents of A5 has now moved to A4
    • Since A4 is part of the A3:A4 merge, it is no longer visible.
    • You can, however, note that A4 contains "C" by entering a formula =A4 in some cell.

I suppose one way to fix this would be to treat merged cells as a single cell when doing the remove duplicates feature, but I fear that may not be so straightforward.
