Step 1: "The Windows Security Centre Service can't be started" 

To resolve this problem, make sure that the Security Center service is configured correctly, and then start the Security Center service. To do this, follow these steps:

1. Press Windows key + R to open the run command.

2. Type services.msc and press enter 

3. Right-click  Security Center, and then click Properties.

4. Click the "general" tab.

5. In the Startup type list, tap or click Automatic (Delayed Start).

6. Then, try to start the service. In the Service Status area, tap or click Start.

7. Click Apply, tap or click OK, and then close the Services window.

Step 2: "The Audio Service is not running"

1. Press Windows key + R to open the run command.

2. Type services.msc and press enter 

3. Right-click on the Windows Audio service, and then click
     Properties.
 4. On the General tab, next to Startup type, make sure that
    Automatic is selected.
5. Also make sure that the service is started

windows 10 registry

I run an application that uses the registry to store printer and font settings. The appication must be able to update the registry from time to time. I am finding that some Windows 10 systems work fine when others will not update. How do I allow the application to update the registry?

Mike

a. user say:

I think this is a risky process as updating or manipulating the Registry might harm your computer. But if you really want to proceed with it, I would suggest to try adding permission for the Registry and C: drive.

Try the following:

  1. Adding permission for Registry:
    • As you can noticed, the Registry have 5 main Key Hives. You need to provide permission on each Hives.
    • Right click on the main hive and select Permissions....
    • To set permissions for a group or user that does not appear in the Grouporusernames box, click Add. Name it as Everyone, and then click OK.   
    • In the Grouporusernames box, select the name that was added.
    • In the PermissionsforALLAPPLICATIONPACKAGES, put a check on the box for FullControl under Allow column.
    • Then, click on the Advanced button.
    • Change the Owner (upper left part of the window) into Everyone, for Permissions and Auditing tabs. And then click OK.
    • Then click on OK on the Properties window.
    • Do this steps for all the main hives on Registry.

  2. Adding permission for the C:drive:
    • You also need to do this for the C: drive.
    • Right click on the Local Disk (C:) and select Properties.
    • On the Local Disk (C:)Properties window, select Security tab.   
    • To set permissions for a group or user that does not appear in the Grouporusernames box, click Add. Name it as Everyone, and then click OK.   
    • In the Grouporusernames box, select the name that was added.     
    • In the PermissionsforALLAPPLICATIONPACKAGES, put a check on the box for FullControl under Allow column.
    • Then, click on the Advanced button.
    • Change the Owner (upper left part of the window) into Everyone, for Permis
      • Then click on OK on the Properties window.

For more information, check the following links:

  • File and Folder Permissions
  • Set, View, Change, or Remove Permissions on Files and Folders

Please be careful and do this for your own risk. :)
