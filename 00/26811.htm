If you can't find any of those terms, it is possible that it is a "single-voltage" (3.3V range) card and thus does not work with your phone (which requires 1.8V range cards).

confused_in_houston say:

Opening TXT files in Excel and Maintaining Spacing

I am building a file in Excel that ultimately will be used as a load-file in txt format.  Spacing in this load file is crucial since the ERP system that I'm loading to "expects" values to be in a specific location in the file.  I achieve the spacing in Excel by setting the column width for each parameter.  When I'm done loading my data to the Excel file, I save it as a PRN file.  Then I copy the PRN file to a TXT.

I have a need to combine two load-files and each one is formatted differently.  I want to be able to concatenate all my columns of data into a single column.  The concatenation is not the problem.  The problem is that when I open either TXT file or either PFN file in Excel, the orginal spacing is ignored.  Everything is pushed together.  I've tried opening as space delimited and several other options but I'm not finding anything that works.  I appreciate your help if you know how to open a TXT or PRN without losing the spacing.  Thanks 

pete_uk say:

If you add a second sheet to your original Excel file, then you can concatenate the columns from Sheet1 and incorporate the number of spaces needed for each field, doing something like this in A1 of Sheet2:

=Sheet1!A1&REPT(" ",20 - LEN(Sheet1!A1)) & Sheet1!B1&REPT(" ",15-LEN(Sheet1!B1)) & Sheet1!C1&REPT(" ",10-LEN(Sheet!C1))

where I have assumed the first column should have 20 characters, the second column 15 and the third column 10, and so on, and that the spaces are added at the right-hand end of each field. If you have numbers then you might need to use TEXT( ... ) to specify how many decimal places to include, and similarly for dates.

One way of getting your PRN file to be imported into one column in Excel is to rename the file and change the extension, eg to .xyz (or anything else that doesn't exist in your file association list). You might need to use Control Panel and change the option to display file extensions of known file-types, so that you can more-easily change the extension. When you double-click on this renamed file Windows will tell you that it doesn't have an application associated with that extension and will invite you to choose one, so choose Excel, and check against Always use this application. The data from your renamed .PRN file will now appear automatically in column A when you double-click it.

Hope this helps.

Pete
