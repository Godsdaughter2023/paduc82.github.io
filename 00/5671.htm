joeu2004 say: Rita wrote:
That makes sense.  I know he's importing data from somewhere and then, at some point, does a copy>paste special>values operation, so perhaps it's retaining the null string result from earlier.  Is there an easier way to resolve than selecting every row and deleting?

I don't know if there are operation limits (i.e. a limit on the number of cells that Find All will show), but the following procedure worked with a small range of cells.

1. Find all "empty" cells, to wit:  press ctrl+F, do not enter anything in the Find What field, then click on Find All.  Note that that finds cells with constant null strings as well as truly empty cells (no constant and no formula).  It does not find cells with formulas that return the null string, as we intend.

2. Select everything in the Find All results, to wit:  press ctrl+A.

3. Clear Contents, to wit:  click on Clear on Home ribbon, then click on Clear Contents.

Note:  At that point, ctrl+End will still go to the end of the "empty" rows.  Excel does not adjust that location until we save the file or access ActiveSheet.UsedRange in VBA.

change form record source - Microsoft Community

jenapo say:

change form record source

I am trying to change a forms record source when I am opening it from another form.

Form [fPurchaseOrders] has a record source of query [fqPurchaseOrdersCurrent] (showing only current orders). I have form [fProjects] in here I have a table that shows me all purchase orders for that project. I have a button "View" that when I click it it opens form [fPurchaseOrdersAll] but I would like it to open form [fPurchaseOrders] so I need the record source of this formto change to query [fqPurchaseOrdersAll] how can I code this either using an  event procedure or an embeded macro?

jack stockton say:

Try this;

DoCmd.OpenReport "ReportName", , , <WhereCondition>

ken sheridan say: You don't need to change the RecordSource property each time.  If you change it in the form's properties sheet to fqPurchaseOrdersAll  and save the amended design it will return all rows by default.  You can then either open the form with an expression which filters it to the current orders as the WhereCondition argument when you want it filtered, as Jack suggests, or you can use your fqPurchaseOrdersCurrent query as the FilterName argument:

DoCmd.OpenForm "frmOrders", FilterName:="fqPurchaseOrdersCurrent "

when you want to filter it to the current orders.
