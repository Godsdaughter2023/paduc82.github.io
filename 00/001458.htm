gina whipp say:

This database first needs to be converted to a Desktop database then split.  The example below is for Access 2010 but should work for Access 2013.

Yes it is possible to convert the 2010 Northwind Web Database into a client version. It will, however, take some manual work on your part.

Follow these steps:
- You first need to create a new empty client ACCDB file.
- Once that is open, you can import all the web tables, client forms, and client reports from the web copy into the client file. The web tables will be converted into client tables but you cannot import web queries, web forms, web reports, and web macros into the client version.
- Close the client copy now and open the web copy
- You can individually change a web object and save out a copy of the object as a client object.
- To do this, highlight a web query for example in the Navigation Pane. Click File tab, then Save & Publish area on the left, then Save Object As in the middle of this pane, and finally click the Save As Client Object button on the right side. You'll be prompted to give a name for the client object. Since it can't be the same name as the original at the moment, just use the same name and append the word Client at the end.
- You'll see a client copy of that web query in the Navigation Pane. To make sure the application still works, you'll need to delete that web query, and rename the client version to match the original name (remove the appended Client word from the name you gave it earlier.)
- You will need to repeat these steps for each and every web query, web form, web report, and web macro in the application. Yes, it will take some time to do that.
- Once complete, you can import all of these remaining client objects into your client only ACCDB file. Your file should now only have client copies of the Northwind web database.

Saving a document in WORD with text boxes - Microsoft Community

jennifertolleson say:

Saving a document in WORD with text boxes

I've created a document with a picture and I had to insert text boxes on that picture.  After I was done and hit SAVE some of the boxes disappeared and I have no clue why.  It has done this a few times now and it's extremely frustrating.  Any idea as to what I'm doing wrong?  Is this a formatting issue?
john korchok say:

Floating pictures and text boxes are always anchored to text. It can make a big difference which piece of text they're anchored to and what part of the document the item is positioned relative to.

To make anchors visible, choose Word>Preferences>View and check Object anchors. Drag all the anchors to the same paragraph. Then for each text box, right-click and choose Format shape. Select Layout, click on the Advanced button, choose the Position tab and check Lock anchor. Once all objects are anchored to the same paragraph they will all move together.

While you have the Position tab open, check that all objects use the same positioning parameters for both Horizontal and vertical section. I usually prefer Absolute position and change Column or Paragraph to Page. This locks the graphic to a predictable place on the page. If the objects move, they move to a different page but stay in the same lockup. The exception to this rule is when the picture relates to specific text and you want them to move together. Then the default Column and Paragraph settings work better.

A time saver is to set an object with the positioning you prefer, then right-click on it and choose Set as Default Text Box (or AutoShape, or Line, depending on the object type). Then every time you insert a new shape, it is automatically positioned with those parameters already set.

Hi Karen,

Thank you for contacting Microsoft Community. I will surely help you with this issue.

This issue could have occurred due to missing updates. 

You may find a fix for the issue by checking for Windows Updates. To do this without the Start menu:

 

  1. Tap or click the Internet Explorer icon.
  2. Type http://update.microsoft.com into the address bar and hit Enter.
  3. If you are prompted to run a control, click Yes.
  4. This should bring up the Check for Updates window. Install updates.

 

If no update is available, or the available updates don’t resolve the issue, reboot the device. The Start menu should come back. To reboot without using the Start menu:

  1. Hold down the Windows key and press L to Lock the device. Swipe up or press Enter to get the log in screen. Use the Power icon on the log in screen to restart the computer.

    OR
  2. Hold down the Windows key and press R to get a Command Prompt. At the prompt, type shutdown –r to reboot the device.

Hope this information helps. Reply to the post with updated status of the issue so that we can assist you further.

This (install updates) is not a viable solution - as Windows update (and all modern apps inc. START Menu) are 'blocked by parental controls.

It's only the GUI for windows updates that you can't load. The update service itself is still running and will do updates (including new builds) if/when they become available. All updates are mandatory in the TP, so they will eventually apply, however leaving your machine in the current state while waiting for that time is not really desirable.

There have been a lot of different "fixes" that have worked for people, but why some work and others don't is still very much a mystery.

If you can't use the "Revert to previous OS" option, and none of the other fixes work for you, then either waiting, doing a refresh, or a full reinstall are the only other options left at the moment.
