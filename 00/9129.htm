graham mayor say:
Keeping wanted and unwanted material in the deleted folder is asking for trouble, however if you want to permanently delete UNREAD messages from the deleted folder then the following Outlook macro should do the trick ... including a message box telling you what you are deleting.

Sub EmptyUnread()
Dim olDeletedFolder As Outlook.Folder
Dim olItems As Outlook.Items
Dim olItem As Outlook.MailItem
Dim olDelItem As Outlook.MailItem
Dim i As Long, j As Long
    On Error Resume Next
    Set olDeletedFolder = Session.GetDefaultFolder(olFolderDeletedItems)
    Set olItems = olDeletedFolder.Items
    For i = olItems.Count To 1 Step -1
        Set olItem = olItems(i)
        If olItem.UnRead = True Then
            If MsgBox("Delete Message Subject: " & olItem.Subject & vbCr & _
                      "From: " & olItem.Sender & vbCr & _
                      "Dated: " & olItem.SentOn, vbYesNo) = vbYes Then
                olItem.Delete
            End If
        End If
    Next i
    Set olDeletedFolder = Nothing
    Set olItems = Nothing
    Set olItem = Nothing
End Sub

norman david jones say:

Hi Jim,

The function that you are using cannot be used as an Excel worksheet function as the code requires the selection of cells which is not legitimate in the case of a UDF (User Defined Function).

Whilst you could rewrite the function to remove such  cell selections, I would suggest that a much better alternative would be to use the following function written by Charles Williams:

'=========>>
Public Function COUNTU(theRange As Range) As Variant
'--------------------------------------------------------------
' Author     : Charles Williams, Decision Models Limited
' Link         : http://msdn.microsoft.com/en-us/library/ff700515(v=office.14).aspx
' Purpose   : Count unique values
'-------------------------------------------------------------
    Dim colUniques As New Collection
    Dim vArr As Variant
    Dim vCell As Variant
    Dim vLcell As Variant
    Dim oRng As Range
    Dim bCount As Boolean

    Set oRng = Intersect(theRange, theRange.Parent.UsedRange)
    vArr = oRng
    On Error Resume Next
    For Each vCell In vArr
                If vCell <> vLcell Then
            If Len(CStr(vCell)) > 0 Then
                colUniques.Add vCell, CStr(vCell)
            End If
        End If
        vLcell = vCell
    Next vCell

    COUNTU = colUniques.Count
End Function
'<<=========

scottgem say:

First, why are you using Like? Like should only be used when you are trying to compare a part of the contents along with wildcards. Since you are not doing that there is no reason to use the Like operator.

Second, Your query is already dynamic since you are using a parameter prompt. However, I do not recommend using parameter prompts. Instead what you should do is create a Form with comboboboxes to select the Year and Month (I tend to use an Option Group for months.

Then you change the criteria to: =Forms!formname!controlname where formname is the name of the form and controlname the name of the combobox. Then add a button on the form to run the report. 

Some other questions here. Are you actually storing Month and Year separately? Why? If you are not, then what is the expression for your month and year columns? Are you sure you only need to filter for month? What if you want to report on a quarter, the whole year or a custom date range?

ken sheridan say: What is it you wish to restrict, the individual base values on which the aggregations are based, or the aggregated values at the intersections of the row and column headings?  If the former you simply add a WHERE clause to the query, e.g.

If the latter you would need to do so by calling the IIF function in the TRANSFORM clause, e.g.

TRANSFORM IIF(SUM(Amount)>0,SUM(Amount),NULL)
