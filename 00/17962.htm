rdavidhooper say:

Design Goals and Definitions

I believe that Microsoft has misunderstood its main customer base and the reason that they purchase Windows personal computers at home.    I think that their customers buy Windows computers because they want exactly the same experience at home that they have at work.  This applies to desktops, tablets, and if the customers were given the opportunity, cell phones.  The thought is, “If I have to learn a new operating system then I might as well buy a computer that runs a different operating system.  I believe that this is the reason that Microsoft dominates the home PC market.

I am writing this mostly because of the posts lamenting the loss of some Windows 8 features that were put back to the way the main customer base expected them to be.  I myself purchased a tablet computer because I wanted the stylus input.  When I ordered the tablet it was supposed to have Windows 7, but it showed up with Windows 8.  Microsoft has been working on handwriting recognition and touch input for at least 20 years.  My expectation was that they would build on this expertise in Windows 8 not throw it away.

I also feel that the tablet and stylus will replace the laptop in the future.  I know you guys are hung upon your keyboards, but I feel that it is much more natural to hold the tablet in one hand and write on it with the other.  The name, after all, is derived from tablet of paper.

One of Microsoft’s biggest misunderstandings of their customers was when they went from programs to apps.  I think that one of the best things that Microsoft could do to improve their market share would be to make a commercial with Bill Gates in a killer bee suit saying ’’Apps. We don’t need no stinkin’ apps man. We have billions of Windows programs and they are all compatible with the latest version of Windows on any device that it runs on.”  The processors in cell phones are powerful enough to make that happen now.

With all that being said, let’s make a couple of definitions:

No matter if you push it with a finger, a mouse click, or a stylus, an icon is a button with a picture on it.  A menu item is a button with words on it.  One advantage of a menu is that it uses less screen space.  Only the main menu button needs to be displayed.  Another advantage of a menu is that you don’t have to guess what the picture means.  On systems with a mouse or stylus you can hover the pointer above the picture and display the words. How to do this with a finger still needs to be worked out. 

The purpose of the taskbar is to show which programs are running at any given time.  It also shows some system information like battery charge, date and time, unread emails, etc.  Having the taskbar visible at all times allows the switching between running programs without having to have a special multitasking view.  I assume that pining programs/apps to the taskbar was to make up for not having a start menu.  If all of the pinned icons are removed then the taskbar becomes a very good task switcher.

You appear to be under the same misconception as many others - that all electronic devices are equally suitable for all tasks.

All electronic devices - smartphones, tablets, laptops, desktops, etc - are tools.  A screwdriver is immensely useful for screws, but you can't use it on bolts or nails.  Though you can drive or remove a nail with a pair of pliers, that's not what the tool is meant for.  Even in the futuristic worlds of Star Trek: The Next Generation and Ender's Game, computers and mobile devices are used for distinctly different purposes.

For anything which relies upon considerable text entry, a keyboard trumps writing every time.  A person can only write 22-31 words per minute yet generally type 50-120 words per minute.  Typing also ensures accuracy while even the best OCR systems have difficulty with handwriting - particularly if they have poor penmanship (something no longer taught prominently in schools).

For short text, such as simple notes, and diagrams, yes, tablets are superior.  For shorter text, speed is far less relevant and corrections are seldom onerous - and a mouse/trackball/touchpad is awkward for drawing.

I believe that Microsoft has misunderstood its main customer base and the reason that they purchase Windows personal computers at home.  I think that their customers buy Windows computers because they want exactly the same experience at home that they have at work.

This is only partially true.  Even back in the early days of home computing, it was much more a matter of software.  If someone uses Microsoft Office at work, they want/need to use Microsoft Office at home - until fairly recently, that meant Windows.

Software drives hardware and OSes - always has and always will.  In the early days, it was mainly DOS vs CP/M vs OS/2 vs Unix - all of which broke the mold by 'divorcing' the OS from the hardware to deliver the same user experience on multiple platforms.  At one point, there were more than twenty prominent OSes, most of which were platform-specific like iOS/OSX, but they eventually withered and died.

Nowadays, it's mainly Windows vs Mac vs Linux.  The reason Windows dominates is because it has the largest selection of software available.  The reason developers focus on Windows far more than Mac or Linux is because Windows dominates.

The same phenomenon happened with smartphones.  Early on, Blackberry, Sony, Samsung, and others all had their own wedded OSes.  The earliest attempt at a cross-platform OS, Symbian, was a joint venture between Ericsson, Motorola, and Nokia - for a time, it dominated the mobile market but, eventually, being a wedded OS caught up with it and Android took over.

iOS dominated for a long time, owing heavily to both the Disciples of Jobs and simply having the widest selection of apps available.  But once Android caught on as a cross-platform OS and developers had the incentive to create apps for it, it quickly overtook even iOS to now account for nearly 60% of all mobile devices in use today.

Personally, I feel that Windows 8-10 is something of Microsoft making an 'end run' at the mobile market - trying leverage their dominance as a computer OS to gain a foothold in the mobile OS market.  While the idea looks good on paper, trying to unify such a wide range of devices (with a distinct spectrum of utility) under a single OS isn't such a smart idea.  Though it addresses the issue of having a software base, only a tiny fraction of Windows software is applicable to mobile devices - and few mobile apps have the slightest relevance for computer users.

I could be wrong, but I don't see any reason why I should ever want to use Publisher, Maya, or Photoshop on a smartphone.

Yes, operating systems are technically software, but they serve a far different function than programs - which is what most people think of when hearing the word "software".

Two other things:

Back in the 1970s and 1980s, when home computers were transitioning from expensive novelties into affordable tools, operating systems were mostly wedded to their respective hardware.  An Amiga could only run Amiga OS, VMS only ran on VAX hardware, an Apple II could only run AppleDOS, etc.

(Yes, that's a 28-amp [3200-watt] power supply!  And half that price is for the 10 MB hard disk.)

CP/M and UNIX were the earliest cross-platform operating systems, joined soon by DOS and later by OS/2.  These OSes democratized hardware and opened up competition - this and volume made these standardized systems less expensive (and often more powerful) than systems which were wedded to their OSes.  Which OS you used now trumped who manufactured the hardware, and choice of OS was driven largely by what programs were available for it.

PC-DOS (later MS-DOS) gradually emerged as the dominant OS - second only to UNIX in raw power and using a more intuitive system of commands.  For any computer system to remain in production (and manufacturer to stay in business), it had to be able to access this exponentially-expanding software library.  "Trying to be 'IBM compatible'" wasn't what put them out of business - failing to do so did.

Apple was the only closed-platform computer manufacturer to survive this era, for two reasons:

  1. Apple enthusiasts (aka 'Disciples of Jobs') are zealots when it comes to "defending the faith".  (Linux geeks have nothing on these guys!)  Apple has worked hard to build up such levels of consumer loyalty, even donating their systems to schools in order to increase their exposure.
  2. Apple already had a fairly solid foundation of exclusive software - desktop publishing, graphic design, audio/multimedia, etc.  Newspapers, recording studios, and other businesses using this software wanted to avoid the productivity hit they'd take by switching to different software.

By all rights, Apple should have long since been dead and buried.  The fact that they're still in business bears testament to just how powerful software (and OS) loyalty is.

This is really the point that I hoped that Microsoft Marketing would get from my post.  I believe that their core customers will buy a Windows phone or tablet because it runs Windows and Windows programs.  This does not mean that the user would chose to run all of their desktop programs on their cell phone.  But I never thought that I would use an Internet browser when I bought my phone.

And therein lies the rub.

Surfing the web wasn't your motivation for buying that particular phone.  It's a nice feature, sure, but it in no way interfered with the reasons you did choose that phone.

Even while keeping an eye toward the mobile market, Windows is first and foremost a computer operating system.  Where computers (desktops and laptops) are concerned, Microsoft's first priority should be consistency - desktop, Start Menu, etc.

But here's the thing...

There are two parts to any operating system - the kernel and the UI/GUI.  (I'm over-simplifying things a bit here, but bear with me.)  The kernel handles all the "nuts and bolts" stuff while the UI/GUI provides user accessibility to the kernel.

Take a gander at Linux.  No matter which distro you opt for, you can select pretty much any UI/GUI you want.  "It's so simple, a caveman can do it."  A Linux user can even switch between different GUIs pretty much on-the-fly.

Windows 10 is already adopting the multi-desktop approach of most Linux GUIs - it's pretty clear that things are also in line for it to support multiple GUIs.  This makes perfect sense if Microsoft is serious about trying to produce a "one size fits all" operating system, though they also need to keep in mind that all those Linux desktop GUIs include Start Menus.

If software/UI loyalty is powerful enough to pull Apple through the Dark Times, keep Windows 7 at a 55% market share, and doom Windows 8, you can rest assured that Microsoft had better pay attention and made sure Widows 10 (at least where the computer GUI is concerned) conforms to the legacy established by 95, XP, and Windows 7.

Anything less is suicide.

Changing the default format for lists in Word 2010 - Microsoft Community

skipnicholson say:

Changing the default format for lists in Word 2010

Is there a way to permanently kill the "Don’t add space between paragraphs of the same style" option in Word 2010? It's a format I never want, and I spend too much time "unfixing" lists. Each time I start a new list, Word automatically rechecks the box that eliminates the space between the paragraphs. Then it automatically indents the list .25 inch. I've tried modifying the Normal.dot files without any success. Is there a way to have a new list begin with the same paragraph spacing and the indentation as in the preceding paragraph (which is usually the Normal style)?

suzanne s. barnhill say:

If you just click on the Numbering button to start a numbered list, Word applies the List Paragraph style, which includes the "Don't add space..." setting. You can modify that style to omit that setting; be sure to save the change to the Normal template by selecting the radio button for "New documents based on the Normal template."

 

A better solution is to use Define New Multilevel List to set up your list the way you want it, linked to specific styles. Unless you have a good reason for not using them, I recommend the List Number series (List Number for Level 1, List Number 2 for Level 2, and so on). In defining the List Number styles, you can omit the "Don't add space..." setting.
