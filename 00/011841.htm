Tables in publisher - Microsoft Community

bfmuindi say:

Tables in publisher

Hi, How can i make a table outflow to another page, i was preparing a publisher booklet, when i tried inserting a long/large table, the table did not outflow to the next page, instead spreading outside the page. please assist
john korchok say:

Pages in Publisher work differently than Word. In a Word processor, pages are not part of the file, they are a virtual objects that are created as you add content. Publisher can't do that: pages have to be created by the user. The only content link between pages are linked text boxes.

Unfortunately, tables are not part of a text box and can't flow to a new page. You'll either have to manually create extra tables on the other pages, or create the table in a program like Word, copy a page at a time and in Publisher use Home>Paste Special>New Table to paste table portions on each page.

tristanemerson say: I found the same issue, but have been able to replicate my clients' complaints.  We are supporting an older web app, which uses frames heavily, and when clicking a menu link in the menu frame the content frame will sometimes just turn grey.  I am using IE10 installed on Windows 7.  There are lots of ways to get the grey part to disappear (resizing the window, for example), and then the frame contents render correctly.

I opened up the Developer Tools window (F12) and fiddled with the settings, and the only settings where I can reproduce the issue are: 
  • Browser Mode: IE10 
  • Document Mode: IE8 Standards
If I try to change the Document Mode I noticed that it says "(Page default)" next to "IE8 Standards" in the drop-down menu.  A bit more digging and I found the following line in our code:

<meta http-equiv="X-UA-Compatible" content="IE=8"/>

This MSDN article has some information about setting the correct doctype and using the X-UA-Compatible header, but I just removed that line and it seems to have fixed my issue.  Now I just need to make sure it didn't break anything else in the process.
