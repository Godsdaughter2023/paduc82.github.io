demetri m. say:

I've been having this same problem for a long time with Outlook for Android. I finally solved it for me, and hopefully others. IMAP support in Outlook for Android is really bad.

Short answer:

This comes down to a bug in the app that's existed since launch: It does not handle special characters in passwords correctly! If you have any symbols or special characters in your password, remove them and see if you can add the IMAP account again.

Technical details:

This took hours of debugging on a personal mail server to finally figure out. I had a password with a "}" character in it (which is a special reserved character in the IMAP protocol.) Outlook for Android sends the following to the server:

A0 LOGIN username password
when it should be:

A0 LOGIN "username" "password"

According to the IMAP RFC "IMAP4 Implementation Recommendations" located at https://tools.ietf.org/html/rfc2683

"As to special characters in userids and passwords: clients must not
   restrict what a user may type in for a userid or a password.  The
   formal grammar specifies that these are "astrings", and an astring
   can be a literal.  A literal, in turn can contain any 8-bit
   character, and clients must allow users to enter all 8-bit characters
   here, and must pass them, unchanged, to the server (being careful to
   send them as literals when necessary)."

---

A small note regarding port numbers: I guess with a recent update, you can specify port numbers now by adding a ":" after the server name for IMAP and SMTP. Ex: imap.emailprovider.com:993

The buttons can be used to move headings (including any subordinate text) up or down in a document. They're not specifically for master documents, but for Outline view.
