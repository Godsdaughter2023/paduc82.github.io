FIX: Start Screen items erroneously highlighted as NEW. - Microsoft Community

paul coddington say:

FIX: Start Screen items erroneously highlighted as NEW.

I can't respond to previous threads where this problem was raised, as they are locked. But having had more time to address the issue, I think I have found a solution. I couldn't come up with this earlier, as there were a long list of problems to address when upgrading to 8.1 and I could only deal with them in order of urgency one at a time. It was all a bit overwhelming! (;P)

 

PROBLEM

 

Windows 8.1 keeps flagging various items on the Start Screen as NEW, even when the items are not new. An item can lose the NEW flag when used, but have it back again when the system reboots. Merely unhiding Administrative Tools can cause them to be all regarded as NEW.

 

FIX

 

This is a possible fix, which I am still testing. Use this at your own risk.

 

Use Windows Registry Editor to navigate to this key:

 

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\StartPage

 

Rename (or delete after backup) the REG_BINARY value StartMenu_Start_Time.

 

The system will recreate the key with a new time and all the NEW flags on the Start Screen should disappear. They have not reappeared so far, but if they do (which is likely the next time an application is installed or the Start Screen rearranged), the patch can simply be reapplied (it relieves the symptoms, it is not a cure for the bug that causes them).


Edited on 2013.12.06 to add/update:


The data in the UserAssist key is also involved, and I suspect a corruption there may be the problem.


HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\UserAssist


This key also regenerates itself on the next login when it is deleted.

xircal say:

Hi,

net user administrator /active: no disables an enabled Administrator account. But since it hasn't been enabled yet, it generates the error you're seeing.

To enable it, you should change the "no" into "yes" so that it becomes:

net user administrator /active: yes

But that may not be neceessary. Try doing the following first.

  1. Hit Windows key + R to open the Run command, type: netplwiz and click OK.
  2. In the menu which opens, click the user (which should be you) and then click the "Properties" button.
  3. In the next menu, click the "Group Membership" tab.
  4. Next, click "Options" to turn the radio button black and then click the grey block and select "Administrators" from then list.
  5. Click Apply/OK and reboot.

When the computer reboots to the desktop, hit Windows key + R to open Run again, type: msconfig and click OK. You should now be able to remove entries without being denied access.
