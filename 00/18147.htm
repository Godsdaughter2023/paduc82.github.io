Prasad Gopinath

teylyn say:
Hello,

an Excel chart starts from the crossing point of the X and Y axes and shows the values in that order. In a horizontal bar chart, the first value in the data source will be the one displayed closest to the X axis, which is the reverse order of the data table.

To reverse the plot order in the chart, you can format the Y axis and tick the setting "Categories in reverse order" to show the data in the same sequence as the source table.

This will result in the X axis showing at the top of the chart instead of the bottom. To correct that, format the Y axis again and tick the setting for "Horizontal axis crosses" > "At maximum category"

cheers, teylyn

roady [mvp] say:

So why do you believe it would be so difficult to offer the already listed target folder also in the overview of offered folders for the move?

It is based on how things are working now. I'm not saying your idea is bad, but that it will be an expensive implementation change. Currently the Folders MRU is stored in the Registry and lists your 10 last used folders. In that way, it is relatively static throughout your Outlook session and easy to update and maintain.

Conversation View has its own index, which is why it is working that fast even with a large mailbox and a complex folder structure. Dynamically creating the Move to Folder menus (including the right click menu) on itself isn't a hard request but it will probably require additional indexes for it to be fast and it is also less efficient in resource usage. It is a bit of a trade-off and I obviously don't have a complete overview of what is involved for this change but it is definitely touching quite a bit of code. It then comes down to the usage level of this feature and the priority it gets.

Anyway, feel free to post it as a suggestion on Outlook's official UserVoice site;
