' DeepFreezer will hide/unhide frozen cells. Will also freeze Panes if
' called while nothing is frozen. Call via DpFrzCaller or any other sub
'
' by Apostolos55 in 4/10/2012
Sub DeepFreezer(Optional MyCell1 As String = "", Optional AffectRows As Boolean = True, Optional AffectColumns As Boolean = True)
Dim Irw As Long, Icl As Long, myRow As Long, myColumn As Long, myCell As Range
Dim Dum As Variant

If ActiveWindow.FreezePanes = True Then 'Already frozen, do stuff

Application.ScreenUpdating = False

myRow = ActiveWindow.SplitRow
myColumn = ActiveWindow.SplitColumn
If AffectRows = True Then
If Cells(1, 1).EntireRow.Hidden Then
For Irw = 1 To 100
If Cells(Irw, 1).EntireRow.Hidden = True Then
Cells(Irw, 1).EntireRow.Hidden = False
Else
Exit For
End If
Next
Else
For Irw = 1 To myRow
Cells(Irw, 1).EntireRow.Hidden = False
Next
End If
End If
If AffectColumns = True Then
If Cells(1, 1).EntireColumn.Hidden Then
For Icl = 1 To 100
If Cells(1, Icl).EntireColumn.Hidden = True Then
Cells(1, Icl).EntireColumn.Hidden = False
Else
Exit For
End If
Next
Else
For Icl = 1 To myColumn
Cells(1, Icl).EntireColumn.Hidden = True
Next
End If
End If
' Frozen panes exist but don't want to hide/unhide then... Unfreeze
If AffectColumns = False And AffectRows = False Then
ActiveWindow.FreezePanes = False
End If
Application.ScreenUpdating = True

Else ' if no frozen panes exist then Freeze at default or given cell
If MyCell1 = "" Then Set myCell = Selection.Cells(1, 1) Else Set myCell = Range(MyCell1)
Set Dum = Selection
myCell.Select
ActiveWindow.FreezePanes = True
Dum.Select: Set Dum = Nothing
End If

End Sub

I'm of the "if it ain't broke - don't fix it" school of thought.  It's working and you know how/when it works.  If you split it up into 3 or more similar loops, you have to worry about how things are set up in data areas that won't be dealt with this time around (maybe rows that should be made visible will remain hidden).

Your code should already be pretty efficient - you're using object references to get things done.  One thing that might speed things up a little on any of the sheets that happens to be selected when the loop runs and does things on that sheet would be to add:

Application.ScreenUpdating = False just ahead of the For i = LBound().... statement.  No requirement to reset it to =TRUE later because it will be reset to True by Excel when you exit the sub.

If you're going to try to make improvements by, you might consider setting up tests where you declare the contents of AllOfTheRanges to create an array of:

only Data_Set_1 through Data_Set_4

Data_Set_1 through Data_Set_4 plus Data_Set_5 and Data_Set_6

Data_Set_1 through Data_Set_4 plus Data_Set_7 through Data_Set_10

All Data_Set_'s

Although from your description it appears that you NEVER need all 10 sets?  But you always have 1 of 2 combinations:
Data_Set_1 through Data_Set_6  (named range = "YES")
Data_Set_1 through Data_Set_4 plus Data_Set_7 through Data_Set_10

So maybe something like this (but I don't know the name of your YES/NO named range, so I'll just call it "yourNamedRange"

If UCASE(TRIM(Range("yourNamedRange")))="YES" Then

  AllOfTheRanges = Array(Range("Data_Set_1"), Range("Data_Set_2"), Range("Data_Set_3"), Range("Data_Set_4"), Range("Data_Set_5"), Range("Data_Set_6))

Else

AllOfTheRanges = Array(Range("Data_Set_1"), Range("Data_Set_2"), Range("Data_Set_3"), Range("Data_Set_4"),  Range("Data_Set_7"), Range("Data_Set_8"), Range("Data_Set_9"), Range("Data_Set_10"))

End If

Application.ScreenUpdating=False
and on with your For i = LBOUND() .... loop just as you have it now.
