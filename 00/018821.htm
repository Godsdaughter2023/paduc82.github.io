jaxhogan say:

 **Just did another search and found the solution! Thankyou to Maxwell**

This works:

Click on Start > Programs > Accessories > Run > copy and paste the following text 'Regedit' without quotes in the Open field and then press Enter

locate  the registery key  'HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\Session Manager'

Check to see (on the right) if there is an entry named: PendingFileRenameOperations. 
If it is present it might have the value of a file Similar to this

 
PendingFileRen       REG_MULTI_SZ      \‘\C \Users\<username>\AppData\Local\Temp\ose00000 exe

If that key is present then:
1. Back up this registry key:
HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\Session Manager
It can be backed up by Right-Clicking on it and selecting "Export to file".

2. Delete ONLY the PendingFileRenameOperations value - NOT the Session Manager Key
3. Immediately try running BCM Setup again.

PPT 2013 Is embedding fonts permanent? - Microsoft Community

cstrand3m say:

PPT 2013 Is embedding fonts permanent?

When I go to Options>Save>and check Embed fonts in file.... When my client opens the file, makes changes, and saves again - WITHOUT HAVING CHECKED Embed fonts in file on his machine, do the fonts stay embedded? As in, if he sends the file to another person, who makes changes and saves, will the fonts still be embedded in the file?

john korchok say:

The simple answer is yes, but depending on the font chosen your client may not be able to edit the received presentation.

Font foundries can create fonts with different permissions. Most foundries put restrictive permissions on fonts to limit the ability of end users to pirate the fonts from documents. Not that I've ever heard of a single case of anyone stealing a font from a PowerPoint file, but technically it is possible.

You can find out what permissions a font has by double-clicking on it. Initially you'll just see a preview of the font in various sizes. Click on the Properties button in the upper right, then choose the Details tab in the Properties dialog. Look for Font embeddability. Most commercial fonts are set to Preview/Print permissions.

Your client will get the following results depending on the corresponding permission:

  • No Embedding - You simply can't embed these fonts at all.
  • Preview/Print - You can embed these fonts, but your client will not be able to edit the resulting presentation. When they open it, they will see a dialog that says:
    "The presentation cannot be edited because it contains one or more read-only embedded (restricted) fonts. To edit the presentation, you must remove the restricted fonts, or you can open the presentation as read-only." Then the dialog states which fonts are read-only and displays 2 buttons Open Read-only and Remove Restricted Fonts.
  • Editable - Embedding these fonts allows embedding and also allows the end user to edit the document using the embedded font. The fonts that come with Microsoft Office have an editable permission.
  • Installable - The font is not only embeddable but can also be installed on the recipient computer. I've never actually seen any font with this permission.
