Mail retention period of an O365 archive mailbox - Microsoft Community

gayankw say:

Mail retention period of an O365 archive mailbox

Hi, with reference to unlimited archiving in Office 365, how long the emails are kept in an archive mailbox? Or are those mails will be there forever?

Thanks in advance!

lance cai msft say:

Hi GayanKW,

It depends on the retention policy you created/edited.

After the archive mailbox is enabled, the default retention that's assigned to every new mailbox created in your organization does the following:

  • Moves items that are two years or older from a user’s primary mailbox to their archive mailbox.
  • Moves items that are 14 days or older from the Recoverable Items folder in the user's primary mailbox to the Recoverable Items folder in their archive mailbox.

Archive policies are retention tags that use the Move to Archive retention action. They move messages from a user's primary mailbox to their archive mailbox after a specified period.

When an archive policy is assigned to a mailbox, mailbox folder, or specific mailbox items, messages are moved to the archive mailbox after the retention period expires. Messages are moved to a folder in the archive mailbox that has the same name as the source folder in the primary mailbox. If a folder with the same name doesn't exist in the archive mailbox, it’s created when the message is moved.

To learn more about retention policies, retention tags, and the Move to Archive retention action, see Retention tags and retention policies.

sue reins say:

Validation rule - Duplicates exception - Access 2016 table/form

 I need to set my table/form to not allow duplicates, except for three commonly strings.

something like this primitive logic:

if string is not "abc" or "def" or "ghi" then

   No duplicates for this field

end 

*******

is this possible? If so, HOW?

Sue

ken sheridan say: It can be done by applying a CHECK CONSTRAINT to the table.  In Access this cannot be done directly in SQL view, however, but must be executed in code, which can be entered as a single line in the debug (aka 'immediate') window, which can be opened by pressing Ctrl+G.  The statement would be as follows:

CurrentProject.Connection.Execute "ALTER TABLE YourTableName ADD CONSTRAINT SelectiveDuplication CHECK((SELECT YourColumnName FROM YourTableName WHERE NOT (YourColumnName = 'abc' OR YourColumnName = 'def' OR YourColumnName = 'gh1') GROUP BY YourColumnName HAVING COUNT(*)>1)IS NULL)"

To remove the CONSTRAINT you'd enter the following:

CurrentProject.Connection.Execute "ALTER TABLE YourTableName DROP CONSTRAINT SelectiveDuplication"

The table must be closed when either of the above are entered, and, if the database is split the statement must be executed in the back end, not the front end.

talj---01 say: Hey,

I own the phone, isnt it great!?

Yea I have also had a problem with my memory card and the nokia 3250.

Mine was corrupt and had to be sent back.

I am not sure what is the problem with yours though.

Please try this to check if the problem is in the Memory Card:

1.
Insert it into a Card Reader (like Cruzer/Scandisk ect.) (through the adaptor for the micro sd). Then transfer a *multimedia* file (eg a photo) to the card. Safely remove the card from the card reader and plug it in again. If the photo is still in there and you can open it, its most likely that the card is ok.

***Clear the SD Card*** (Not format, just erase the photo you put in)

2.
When you can see the micro sd card through PC Suite, Phone Browser or which any other software you use to view the phones contents through the PC, try transfering a file into the memory card (the micro sd). Preferably a multimedia one. If that works, then you know there is no problem with the Phone reading the card, and that the card is ok.

If the above worked fine, I am tending to think that that is a firmware issue. If I were you, instead of trying things that may not work, like formating the sd card (that can corrupt your sd card, I wouldnt do that), I would go to a service center to check if the memory reader is ok. Bring the Micro SD card with you + its adaptor. Nokia will immediatly say that the card is corrupt, but give it to them with the adaptor (+ if you have a portable card reader)and show them that it works.

They will then check to see if the phones mechanisim is ok.

SD cards are not a reliable type of memory. They have no moving parts in them which makes it very unstable. Unlike a CD which requires movement to be read (a spin) the data is just locked in tiny compartments. Anything can erase that.

So never store anything important to you, without backup somewhere.

For example, Bankcodes, Passwords, Addresses ect.

Good Luck,

Its a phone with a twist..and its awsome!!
