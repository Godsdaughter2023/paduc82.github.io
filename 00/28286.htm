Prevent losing email in Outlook.com when you've set up POP3

Outlook.com has a feature that prevents losing email messages if you accidentally set up POP3 to delete them after reading them on the server. Here’s how it works:

    The first time that Outlook.com receives a POP3 instruction to delete email messages after reading from an email app, Outlook.com won't delete the messages. shane devenshire say:

In addition to the issue Mike raised.  There are a lot of problems with this, not the least of which is you can concatenate a variable with this kind of reference unless you use INDIRECT, but that function does not work against closed workbooks.

Although it doesn't make any difference we would generally use & instead of CONCATENATE:

=VLOOKUP(D:D,"'B:\FolderA\FolderB\FolderC\FolderE\FolderF\FolderG\"&L5&"."&"FolderH\[FileName.xls]TabName'!$D$6:$AQ$52"),VLOOKUP(L5,$C$113:$E$124,3,FALSE),FALSE)

Also the ",L5,"."," suggests you are trying to concatenate a variable folder name with a period and then another folder.   ",L5,".FolderH  would be more appropriate.  However all this is pointless because you can't reference a closed file this way from the spreadsheet.

 So the real question is what are you trying to do.  Reference a closed workbook from a variable location defined in the spreadsheet cell L5.  You might want to consider a VBA custom function.
