rick rothstein, mvp - excel say:

AND is a function and, as such, simply returns a value. That value, like any value (such as TRUE, "Hello", 123) can be used inside of other functions as needed. The syntax for AND is this...

AND(LogicalExpression1,LogicalExpression2,etc.)

where the etc. can be additional logical expressions (delimited from each other and the first two with a comma) as needed. The AND(..) is consider a single unit... the keyword AND followed by a list of arguments enclosed in parentheses. The construction is true of all functions in Excel and is how Excel finds the value returned by each function so it can perform its calculations. Think of functions in more general terms.... FUNC() where this may or may not be any arguments inside the parentheses... each function has its own requirements about whether anything will be inside the parentheses or not, but the parentheses are required (remember, it is this construction, keyword/parenthesis/optional argument list/closing parenthesis, that tells Excel where a function begins and ends so it can retrieve its value). So, to use functions inside other functions, just use it where you would use its value. One possible arrangement...

=FUNC1(FUNC2(..)+FUNC3(..))

Split apart so you can see the arrangement better...

 

=FUNC1(    FUNC2(..)    +    FUNC3(..)    )

The first and last parentheses belong to the FUNC1 function and the sum of FUNC2 and FUNC3 are its (assumed) argument. To sum up, AND is a function no different from any other function when it comes to using it either within other functions or using other functions inside of it. The key is to keep the parentheses straight. Spliting the formula apart like I did above helps when you have a particularly "messy" formula.

rich007a say:

Rick and Joe give very good explanations, but in case it helps, here's an example with an obvious real world meaning behind it:

Let's say you're at the shop and trying to decide what fruit to buy.  Obviously you can only buy what you can afford which today is $1, and obviously you're only going to buy what you like. You could calculate what to do like this:

 

Buy_Apples =AND( I_like_apples, price_of_apples <= $1 )

This equates to:

If the "value" of I_like_apples is TRUE

...AND...

If the price_of_apples IS less than or equal to $1

then the AND function returns TRUE (and I get apples for lunch).

In Excel, comparisons like   =a=b   or like   =x<y   return TRUE or FALSE as a result.  The AND function also returns TRUE or FALSE.  It will return TRUE only when all of the "arguments" (separted by commas) are TRUE.

How do you reset a combobox after a value is selected in access 2010 - Microsoft Community

neil grg say:

How do you reset a combobox after a value is selected in access 2010

I have a combobox in an Access 2010 form which contains a drop down list of values. When the value is selcted to show the record I want the box to revert to blank. In the after Update  action I have the following  where condition in the SearchFor record Action.

="[DFE] = " & Str(Nz([Screen].[ActiveControl],0))

How do i alter this to show a Null value after the record is selected.

johnlv say: Neil-

You need TWO macro commands.  The first should look like:

SearchForRecord
   Object Type:  Form
   Object Name: <name of your form here>
   Record: First
   Where Condition: "[DFE] = " & [Combo79]

To do the second, you need to click the Show All Actions button on the Ribbon.  Where you see the green + under your first command choose SetValue.  It should end up looking like:

SetValue
   Item: [Combo79]
   Expression: Null

When the macro runs, it will first do the search, then clear the combo box.
