young yang msft support say:

Hi LuRe,

Thanks for update.

1. “Is it important on which AD I create the new user if I use dyrsync? (AD on-premise vs AD online) what is the difference?”

If you use DirSync and sync AD users to Office 365, later you need to manage users’ attributes from local AD. And then sync the change to Office 365. We cannot manage or edit users’ attributes in Office 365.

If you directly create users in Office 365, the users have no relationship with local AD. You can directly manage users in cloud.

That’s the difference: manage users in local AD and then sync to cloud, or directly manage users in cloud.

2. “How I understood the AD (e.g. on-premise) replicate itself to the other part (online), if we use DirSync. Does this mean that I can edit the user online/on-premise and have all the options available on both?”

If you use DirSync, you can only edit users in on-premise AD, cannot edit users in cloud.

3. “if the AD is replicated , why is for mailbox impossible to be replicated?”

DirSync only sync user object to Office 365. It does not sync mailbox or mailbox data to cloud.

Detailed information is explained in that thread.

Young

How do I make the references between worksheets relative (i.e. current sheet minus 1)?

haseeb a say: Or you can use INDIRECT.

Assuming sheet names always be Day1, Day2, Day3 etc.... without spaces or leading zeros.


Name: CurrSheet
Refers to: =REPLACE(CELL("filename",!$A$1),1,FIND("]",CELL("filename",!$A$1)),"")

Name: CurrDay
Refers to: =LOOKUP(99999,RIGHT(CurrSheet,{1,2,3,4,5})+0)

Then in D3 on Day2, copy down

=INDIRECT("'Day"&CurrDay-1&"'!"&CELL("address",H3))

G3, then copy down.

=SUMPRODUCT(N(INDIRECT("'Day"&ROW(INDEX(A:A,1):INDEX(A:A,CurrDay))&"'!"&CELL("address",F3))))

Then copy Day2 sheet & change the days.
