ken blake say: "knwetlk" wrote in message news:c2ff3fd1-3e32-46d9-9522-ec05034a2db1...

> How do I manage programs that startup on Windows 7


First, note that you should be concerned with *all* programs that start
automatically, not just with those that go into the system tray. Not all
autostarting programs manifest themselves by an icon in the tray.

On each program you don't want to start automatically, check its Options to
see if it has the choice not to start (make sure you actually choose the
option not to run it, not just a "don't show icon" option). Many can easily
and best be stopped that way. If that doesn't work, run MSCONFIG from the
Start | Run line, and on the Startup tab, uncheck the programs you don't
want to start automatically.

However, if I were you, I wouldn't do this just for the purpose of running
the minimum number of programs. Despite what many people tell you, you
should be concerned, not with how *many* of these programs you run, but
*which*. Some of them can hurt performance severely, but others have no
effect on performance.

Don't just stop programs from running willy-nilly. What you should do is
determine what each program is, what its value is to you, and what the cost
in performance is of its running all the time. You can try google searches
and ask about specifics here.

Once you have that information, you can make an intelligent informed
decision about what you want to keep and what you want to get rid of.

^^^^^^^ There are essentially two "basic" types of RAID: HARDWARE and SOFTWARE.

HARDWARE RAID is controlled and configured by the disk controller.

SOFTWARE RAID is controlled and configured by the operating system.

+Removing a RAID ARRAY must be done from the same "place" it was created.

=To remove a HARDWARE RAID ARRAY you must enter the system's HARDWARE RAID configuration utility (This varies from system to system and a prompt may be displayed shortly after the computer system posts/boots:  To enter the HARDWARE RAID configuration utility strike the <Ctrl>+<R>, <Ctrl>+<G>, <Ctrl>+<I>, <Ctrl>+<E>, F8 or <Ctrl>+<C> keys repeatedly, shortly after the system begins to boot (while there is a black DOS-like screen, depending on your system and follow the menu's to complete the deletion of the HARDWARE RAID ARRAY (You may see a prompt explaining what keys to hit and when to hit them, in order to access the RAID UTILITY during bootup, before the Windows graphics are displayed; If you start to see the Windows bootup graphics then you've gone to far and will have to reboot to retry and/or reconfigure your BIOS/UEFI SETUP to allow you to access the system RAID UTILITY)).  If there is no prompt during your systems booting process and you are unable to access the configuration utility then you may need to enter your systems bios to configure the utility to be accessible during system boot. 

****After you have successfully deleted the HARDWARE RAID ARRAY you will need to configure the underlying hard drives as you normally would, through the operating system using DISK MANAGEMENT (Initialize each disk and then format each disk) FYI-->(GPT for Windows 8 and above, MBR for all other Windows versions (Hard drives over 2TB in size will need to be formatted as 2 or more separate partitions if you choose MBR (Windows 7 and lower will not boot to a GPT formatted disk drive (UEFI systems will not boot in SECURE MODE to MBR formatted disks (BIOS systems may not be able to "recognize" disk partitions larger than 2TB (GPT must be used in order to create a partition larger than 2TB)))))).

=From what I can tell you have the basic idea of how to proceed with regard to SOFTWARE RAID ARRAYS.  These array types are deleted from the DISK MANAGEMENT utility in Windows.  After you have deleted a SOFTWARE RAID ARRAY then you can continue from the **** in the above section (HARDWARE RAID) to prepare and format the disk drives.  That part of the procedure is the same for both "types" of RAID...

=CAUTION: One additional thing to be aware of, If you are using Intel RST and are utilizing an accelerated volume (SSD cache) you must first "turn off" and remove the "acceleration" before removing ANY partitions from any drive on the system.  Failure to do so will more than likely result in BSOD's and an unbootable system.  Before you proceed make certain that you have completely disabled any Intel RST accelerated volumes (SSD caching) and you have successfully rebooted.    Failure to do so may very well result in a memorable disaster of epic proportions.  I CAN'T STRESS ENOUGH FOR YOU TO HAVE A COMPLETE AND CURRENT BACKUP BEFORE YOU ATTEMPT ANY OF THIS.

That's basically the gist of it.  If you have any questions or problems feel free to reply with the specifics and I'll try to help you out...  (Please include your motherboard brand and model and if you have a separate raid controller the brand and model of that as well...).
