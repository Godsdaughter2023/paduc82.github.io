1. Click Start, type msconfig in the Start Search box, and then press ENTER.

6. When you are prompted, click Restart.

7. After the computer starts, check if the issue is fixed.

The error is a SYSTEM_THREAD_EXCEPTION_NOT_HANDLED_M (1000007e) and the driver that appears to be causing the crash is the libusb0.sys.

The libusb0.sys appears to be a driver for the following software:

http://sourceforge.net/projects/libusb-win32/files/

If you have that software installed, try uninstalling that software.

Also, if problems persist you can stop the libusb0.sys from loading by doing the following (a "clean boot" will not stop it):

Open regedit and navigate to the following registry key:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\libusb0

In the right window click on Start and set the Value data to 4

Restart the computer.

andre da costa say:

When I upgrade a preinstalled (OEM) or retail version of Windows 7 or Windows 8/8.1 license to Windows 10, does that license remain OEM or become a retail license?

If you upgrade from a OEM or retail version of Windows 7 or Windows 8/8.1 to the free Windows 10 upgrade this summer, the license is consumed into it. Because the free upgrade is derived from the base qualifying license, Windows 10 will carry that licensing too.

If you upgrade from a retail version, it carries the rights of a retail version.

If you upgrade from a OEM version, it carries the rights of a OEM version.

Full version (Retail):

- Includes transfer rights to another computer.

- Doesn't require a previous qualifying version of Windows.

- Expensive

Upgrade version (Retail):

- Includes transfer rights to another computer.

- require a previous qualifying version of Windows.

- Expensive, but cheaper than full version

OEM :

OEM versions of Windows are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system

What happens if I change my motherboard?

As it pertains to the OEM licenses this will invalidate the Windows 10 upgrade license because it will no longer have a previous base qualifying license which is required for the free upgrade. You will then have to purchase a full retail Windows 10 license. If the base qualifying license (Windows 7 or Windows 8.1) was a full retail version, then yes, you can transfer it.

From the Windows 10 end user license agreement:

b. Stand-alone software. If you acquired the software as stand-alone software (and also if you upgraded from software you acquired as stand-alone software), you may transfer the software to another device that belongs to you. You may also transfer the software to a device owned by someone else if (i) you are the first licensed user of the software and (ii) the new user agrees to the terms of this agreement. You may use the backup copy we allow you to make or the media that the software came on to transfer the software. Every time you transfer the software to a new device, you must remove the software from the prior device. You may not transfer the software to share licenses between devices.

Question 2 isn't important because your printer is connected to your computer via USB not via wifi.

Communication problems when using a USB connection are not as common as when the printer is connected via wifi.  USB connection problems are also more difficult to diagnose.

Start with the easy things.

A.  Connect your printer to a different USB port on your computer.  If you have a desktop or tower computer, try using a USB port on the back of the computer rather than one on the front.

If you have a newer computer that has both USB 2.0 ports (with black inserts) and USB 3.0 ports (with blue inserts), connect the printer to a USB 2.0 port.

B.  Confirm that the USB port to which you are connecting your printer works for another USB device, such as a USB flash drive.

C.  If you have another USB printer cable, try using that.  If you don't have another cable, unplug the cable from the printer and plug it back again, making sure that it is solidly connected.

D.  If none of the above work, given that you were unable to print the printer settings page, it's possible that your printer itself may have a hardware fault.  Did you turn the printer off and then press and hold the power button for at least 15 seconds? 

You may have to take the printer to a local independent computer repair shop.  If you do this, avoid chain stores, consumer electronic stores, and office supply stores. 

On the other hand, this is a 9-year-old printer that when released was "possibly the cheapest Wi-Fi printer in the UK."  It may be time to buy a new printer.

E.  Although you have previously deleted the printer and reinstalled it with an updated driver, you can try doing that again following these directions.  This obviously won't help if the printer does, in fact, have a hardware problem.

  1. Disconnect the printer from the computer
  2. Go to Control Panel > Programs and Features and uninstall all Lexmark software.
  3. Open Devices and Printers, right-click on the icon from the printer and select "Remove device" then answer "Yes" you are sure that you want to remove it.
  4. Click on any other icon in the "Printers and Fax" category.  You should at least have a "Microsoft XPS Document Writer" (if there are no other icons in the "Printers and Fax" category, see Note below)
  5. Click on "Print server properties" in the Tool Bar at the top of the Devices and Printers window
  6. Click the "Drivers" tab
  7. Select the driver for the Lexmark Z1420 and click "Remove."  When asked, select "Remove driver and driver package"
  8. Click Close when done
  9. Reboot the computer
  10. Go here:
    • Open an Elevated Command Prompt (click Start Orb, type cmd.exe, right-click the link to cmd.exe and select "Run as administrator"; respond as needed to the UAC prompt)
    • Type the following in the black Command Prompt window and press Enter    
              printui /s /t2
              exit
    • Continue with step 7
