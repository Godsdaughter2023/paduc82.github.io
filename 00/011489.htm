jay freedman say:

There is a key conceptual difference between the entry/exit macros of legacy form controls and the content control event handlers that Doug mentioned:

For legacy controls, you can write several different macros and attach each control to a specific macro. A given macro can be specific to one control, or it may be used by several controls and determine which one called it by looking at the field's bookmark name.

For content controls, there is only one event handler for each kind of event (Enter, Exit, etc.) that gets called whenever that event occurs in any content control. The handler gets the affected control as an argument to the procedure. Then your code has to determine (from that control's title or tag, or by its type or some other property) what to do with the control. That usually involves a Select Case structure.

john sr wilson say:

AFAIK the answer to all those is no unless you are prepared to write some pretty complex code. Even then the code would need to be present on all PCs with the template. Really it isn't feasible to do unless you already have a fair bit of code knowledge

If you just wanted some code to align the two placeholders and draw a line (and be run manually) then this MIGHT work depending on the layout.

Sub fixme()
'slide would need to have only 2 text placeholders
Dim osld As Slide
Dim opl1 As Shape
Dim opl2 As Shape
Dim oshp As Shape
Set osld = ActiveWindow.Selection.SlideRange(1)
For Each oshp In osld.Shapes
If oshp.Type = msoPlaceholder Then
If oshp.PlaceholderFormat.Type = ppPlaceholderBody Then
If opl1 Is Nothing Then Set opl1 = oshp Else Set opl2 = oshp
End If
End If
Next oshp
If opl1.Top < opl2.Top Then
opl2.Top = opl1.Top + opl1.Height + 20
Call osld.Shapes.AddLine(opl1.Left, opl2.Top - 10, opl1.Left + opl1.Width, opl2.Top - 10)
Else
opl1.Top = opl2.Top + opl2.Height + 20
Call osld.Shapes.AddLine(opl2.Left, opl1.Top - 10, opl2.Left + opl2.Width, opl1.Top - 10)
End If
End Sub
