Hello,

assuming your sheets have headers in row 1 and the formula starts in row 2, use this formula in E2 and copy down:

=IFERROR(INDEX(Sheet1!$E$1:$E$10,MATCH(Sheet2!B2&Sheet2!C2,INDEX(Sheet1!$A$1:$A$10&Sheet1!$B$1:$B$10,0),0)),"not found")

This works well with smaller data sets. For large tables it can get a bit slow. Always use fixed references, not whole columns.

If this gets too slow, a helper column as suggested by Wyman W is advisable.

cpitchford say: Xorg is the X11 server..

X11 is a display / window protocol.. Just like Firefox converts HTML into graphical web pages.. Xorg converts X11 into stuff you see on the screen..

All the apps on Maemo (linux / Unix in general) connect to the Xorg process and talk "X11" to it.. and Xorg, in turn, draws stuff on the screen..

For example, Xterm connects to Xorg and tells it to draw text on the screen.. Xorg, then draws and displays the text.. in reverse, when you press the keyboard or tap on the screen, Xorg sends that information back to the xterminal..

replace x terminal with absolutely anything graphical.. microb, calendar, hildon-desktop.. they ALL talk X11 to xorg.

So.. if something is busy drawing stuff, stuff that is raplidly changing (say like video or games graphics) then the application is busy, but Xorg is busy drawing stuff too..

How do I combine multiple data ranges as sources for a single pivot - Microsoft Community

nofyobu say:

How do I combine multiple data ranges as sources for a single pivot table?

I need to combine multiple data ranges as sources in a single pivot table.  For example, one tab lists buildings with street addresses, square footages, etc. (one row per building).  Another tab lists building operating expenses for maintenance, cleaning, utilities, etc. (dozens of rows for each building).  All data on both tabs is associated with an ID number unique to each building.  How do I pull data from both tabs into a single pivot table that shows the expenses by category for each building and total for all locations?  I recently upgraded to Excel 2010 from 2003.  In 2003, there was a “combine multiple data ranges” feature, but I couldn’t ever make that do what I needed.  I can’t even find that feature in 2010.  Please help.  Thx

rik_uk say:

In Excel2007 (as the link above suggests) pressing Alt-D-P starts the pivot wizard, however, using this method does appear to require all of the ranges to be in an identical crosstab format.

As you are using disimilar ranges / tables, that are linked by an ID number, then I would suggest to create a table using an SQL statement joining the two tables and then returning the data as a pivot.

To do this is fairly straight forward assuming you have Microsoft Query installed...

From the data Ribbon click on 'From Other Sources' in the 'Get External Data' panel.

From the drop down select 'From Microsoft Query', which will open the Microsoft Query 'Choose Data Source' window.

Double click on 'Excel Files*' from the list, which will open the 'Select Workbook' window. You need to locate the file you are working in / where your data is.

Use the directories part on left hand side first to navigate to the required folder (I usually start by double clicking C:\ as enevitably it is not at the right folder). When the correct folder is opened the file name will appear on the right hand side.

Double click the file name and the 'Query Wizard - Choose Columns' window will open.

From the left hand side highlight the first sheet name of your data and click the '>' button (found in the middle) once... then do the same for the second sheet, highlight and click '>'...

Click 'Next >'... :o) Now the fun part... the wizard will complain at this point, asking you to join the tables... no worries!

Click 'OK' in the message box, which will open the 'Microsoft Query' window with a sub-window called 'Query from Excel Files'. At the top of the sub-window are two boxes, which represent your tables of data with their column titles listed.

To join the two tables you will need to click and drag the title of the ID column from one box onto the title of the ID column in the next. If this has been done correctly a little black line should appear between the boxes.

Under the two table boxes is the data area. This displays the resulting data that will be returned to Excel. If you hover the mouse over the column title it will turn into a little down arrow. Clicking once will select the column by highlighting it in black. The column can now be dragged left or right, or deleted. I mention this as you will have two columns for ID in the table and if they have the same names when the data is returned it will get a number appended to it. I usually delete duplicate data columns. Alternately double clicking in the column title will allow you to rename it for when the data is returned.

OK - 101 in MSQuery done...

To return the data to Excel there is a button on the toolbar of the main window, should be fourth from the left, which looks like an open door and an arrow pointing to it. Click 'Return Data' and the MSQuery window will close and an 'Import Data' window will open in Excel.

Select the 'Pivot Table Report' option and select 'New worksheet'. This will open a pivot table in a new sheet and you can click and drag the  fields from the field chooser as normal.

I appreciate this may appear long winded, but once it's been done a couple of times becomes quite easy and quick.
