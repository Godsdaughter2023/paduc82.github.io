You may be able to speed this up by using Replace instead of Find.

If you have a browser other than Internet Explorer set as the default, Internet Explorer will open in the Desktop mode, which looks similar to Internet Explorer 9. Additionally, your Internet Explorer tile will have a smaller “e” and the words “Internet Explorer” will appear over it.

 

If Internet Explorer is set as the default browser, it will open in the new immersive browser, the tile will have a larger “e” and the words “Internet Explorer” will be under it.

 

To set Internet Explorer as the default browser:

 

1.     From the Start screen, type Default Programs, and select the Apps filter.

2.     Tap or click Default Programs in the search results.

3.     Tap or click Set your Default Programs.

4.     Choose Internet Explorer from the list.

5.     Select Set this program as default, and then tap or click OK.

6.     Close Set Default Programs.

I don't see that any incrementing is needed! If you just want to update each of these values to the same value prefixed by 1799-, you would just update the field to

john w. vinson say:

Ok... create a SECOND query:

UPDATE tblDocuments
SET tblDocuments.docDocNo=[Forms[!frmCrtFindNextNumber]![txtPrefix] & "-" & [docDocNo]
WHERE (((tblDocuments.docDocNo) Like ([Forms]![frmCritFindNextNumber]![txtdocDocNo]))) OR (((tblDocuments.docTitle) Like ([Forms]![frmCritFindNextNumber]![txtdocTitle]))) OR (((tblDocuments.docType) Like ([Forms]![frmCritFindNextNumber]![txtdocType]))) OR (((tblDocuments.docDateCreated)>=[Forms]![frmCritFindNextNumber]![txtdocDateCreated] And (tblDocuments.docDateCreated)<[Forms]![frmCritFindNextNumber]![txtdocDateCreated]+1));

This will take whatever text the user has put into the textbox named txtPrefix and append that value, followed by a hyphen, to the current docDocNo value in the table for all records matching the criteria on the form. (I presume you're typing wildcards into the textboxes so the LIKE operator is meaningful). You can do this all on one form; you don't need a second form.

You can then execute this query from a command button on the form after you've verified that you'll be updating the correct records.
