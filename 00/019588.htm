graham mayor say:

The entries in the dropdown field must match  exactly the conditions in the case statements, both in wording and case for the results to be achieved and as Doug stated you must tab out of the dropdown field having made a change. It is also possible to simplify the code a tad given that there are only two possible results.

Sub OnExitDDList()
Dim oFFs As FormFields
Set oFFs = ActiveDocument.FormFields
Select Case oFFs("DropDown1").Result
    Case "None", "Dread Disease"
        oFFs("IntensiveCare").Result = "My current policy does not have a Hospital Intensive Care Benefit."
    Case "Intensive Care", "Dread Disease and Intensive Care Benefit"
        oFFs("IntensiveCare").Result = "My current policy has a $300 daily Hospital Intensive Care Benefit."
End Select
End Sub

Personally I would use the macro to populate a docvariable and reproduce the result in a docvariable field e.g.

Sub OnExitDDList()
Dim oFFs As FormFields
Dim oVars As Variables
Set oFFs = ActiveDocument.FormFields
Set oVars = ActiveDocument.Variables
Select Case oFFs("DropDown1").Result
    Case "None", "Dread Disease"
        oVars("IntensiveCare").Value = "My current policy does not have a Hospital Intensive Care Benefit."
    Case "Intensive Care", "Dread Disease and Intensive Care Benefit"
        oVars("IntensiveCare").Value = "My current policy has a $300 daily Hospital Intensive Care Benefit."
End Select
ActiveDocument.Fields.Update
End Sub

{ DocVariable IntensiveCare }

Ruchi,

I hate breaking bad news, actually I love it, but it's not a problem with a non-Microsoft service. It's a problem with Windows 7. Specifically, the Desktop Window Manager - there are literally thousands of people complaining about this and it needs to get fixed. It's the #1 reason we are not deploying it at our company of over 4000 computers.

The Desktop Window Manager service, if enabled (automatic with the Aero profile) causes issues with every piece of software out there. I've had it freak out with ONLY WINDOWS EXPLORER running. It's not a graphics card issue - I've tried on HP, Toshiba, Acer, and Dell laptops using Intel integrated graphics, Nvidia, Radeon. All of them have this problem - as the system's consumed resources go up - memory, cpu cycles, etc, the desktop window manager starts to freak out. It redraws the windows on screen, causing this "flickering" that people are referring to - the easiest way to drastically reduce the screen flicker is to set all programs you run into compatibility mode: disable visual themes. However, you can't do that for programs that are part of windows, and if you're having screen flickering while, for instance, copying files from one folder to another, it's extremely frustrating. It's a huge flaw in Windows 7 and the #1 complaint I've seen anywhere - ____, talking to the folks at Lockheed Martin, they HATE it because of that issue.

As to the original poster, to fix this issue disable Aero, or disable the Desktop Window Manager service. To disable Aero, set your theme to the Windows 7 Classic theme. To disable just the DWM, go to Start and type "Services.msc" without the quotes. Find the Desktop Window Manager service and right click, properties. Select "Disable service." also, right click, Stop. 

That will immediately stop all screen flickering and allow you to actually use your computer. It is ludicrous that Microsoft is ignoring this gaping issue with the DWM, the same issue that affected early adopters of Vista. It's not the hardware's problem, it's Microsoft's. For the Timeline:

  1. Right click in the timeline and select Copy Timeline>For Presentation from the shortcut menu.
  2. Paste into a PowerPoint slide.

  2. Click the drop-down to the right of the Copy command on the task ribbon.
  3. Select "To GIF image file".
  4. Specify the file name and location.
  5. Insert the picture into the PowerPoint slide.
