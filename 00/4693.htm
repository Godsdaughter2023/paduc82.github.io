scottgem say:

What you are describing is a standard technique called Cascading or Synchronized combos. I'm not sure how they work in Excel (or even if they do work in Excel) but they are easy to do in Access And the technique is well documented. If you search on those terms, you will find examples on how to do it.

Basically, the selection in the first list is used to filter the RowSource of the second list. So the RowSource would look like:

SELECT fieldb FROM tableb WHERE fielda = Forms!formname!cbofielda;

ken sheridan say: While a simple correlation of two or more combo boxes in the way that Scott describes will allow a 'top down' selection by restricting the second combo box's list on the basis of the selection in the first, to also do in the way you wish, and populate the first combo box with the relevant value on the basis of the selection in the second requires a little more work.  With a simple correlation the second combo box's list would be empty until a value is selected in the first, so to allow a 'bottom up' approach it is first necessary for the second combo box's list to show all possible values if no value is selected in the first.

You'll find an example of this in ComboDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file uses the local administrative areas of counties, districts and civil parishes in my local area as an example.  If you take a look at the Open form for 'drilling down' through hierarchies to return records option you'll see that the three unbound combo boxes in the form header include All Counties, All Districts and All Parishes items as the first row in each list.  This allows you to select a parish for instance from a completely comprehensive list, while the first two combo boxes show All Counties and All Districts respectively.  Once the parish is selected the district and county in which the parish is located will be shown in the other two combo boxes.

Take look firstly at the RowSource properties of the three combo boxes, and secondly at the code in their AfterUpdate event procedures to see how this is achieved.

Whether the same thing could be done in Excel I cannot say.
