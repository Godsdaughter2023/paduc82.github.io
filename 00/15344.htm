You must be on the Office 365 Insider build. In October, they released a whole new calculation engine to do the new Dynamic Array functions. You can read about it here. What is happening is Excel thinks you have some functions that are using Implicit Intersection. I had a number of workbooks that also got the SINGLE() function added, though none of mine were broken by it. You can read a bit about that here and what the Excel PM said about it.

Right now you have two choices:

  1. Use the sad face icon to report the issue to Microsoft and see if they get back in touch with you on it. They are being very careful about rolling this out so only the Insider users have it. There are a lot of Excel addins and models that may be using Implicit Intersection, intentionally or unintentionally, so the Excel team wants to handle those carefully in the transition the new calculation engine.

I know upgrading your program status will install the latest, but I don't think downgrading your program status will revert without an uninstall/reinstall. Once you are on a non-Insider build, Excel will remove the SINGLE() function automatically.

If you can, go for option 1, because option 2 is temporary. I think they want to roll out to a wider audience (Monthly Targeted first) in early 2019.

If you haven't already been recording the modification times, there's no way to retrieve that information afterward. The Office programs, and Windows, simply don't keep the information anywhere.

If you want to record the modification times for future documents, you could probably create a macro for each Office program to put the information into an Excel workbook or an Access database each time you close a document. I don't know of any such system that already exists.
