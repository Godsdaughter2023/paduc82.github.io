john mcghie say:

Hi Chris:

Daniel is correct, but the explanation is a bit complex...

What is happening is that the apostrophe is being encoded in a font that expresses it as a "double-width (Chinese) character", because the font you are using does not contain the character.

In TrueType, if the font in use does not have a character, the Mac OS system falls back through its fonts and encodes the character in the font that is the next closest match: in your case, that single character is probably ending up in Lucida Sans Unicode, which is a font of last resort basically intended for the Asian markets.

If you were using Cambria as your body font, you would font this problem would go away, because Cambria has a full set of Unicode characters in it already.  This substitution becomes a problem only if the font in use is a very old one that does not contain a full character set.  Also:  Check that the document does not contain any Chinese or Japanese language settings, because that will trigger a fall-back to these odd fonts also.

Hope this helps

Can I use AverageIfs for discontinuous ranges? - Microsoft Community

kriszza say:

Can I use AverageIfs for discontinuous ranges?

I am trying to average data from two separate ranges that match a condition.  The data ranges contain the same number of cells and the criterion is the same for both ranges.  I've tried to use AverageIfs but can't figure out how to include the two ranges in the expression.  It works fine for each range individually but I need to average over the two ranges.

Thanks in advance for any advice.

Kris

joeu2004 say: Kris wrote originally:

I am trying to average data from two separate ranges that match a condition.  The data ranges contain the same number of cells and the criterion is the same for both ranges.


Kris wrote later:

I was hoping that I could just reference the two lists, rather than create a new one that combines them, but it works.


You certainly do not need a new list that you create manually.  You should be able to use a variation of Bernard's suggestion [1].

However, if you mean that the qualified ranges -- that is, the set of cells selected to be averaged based on the criteria -- have the same number of cells, you can rely on a special case where the average of averages of different sets is equal to the average of the combined sets if the size of each set is the same.

So you might be able to write:

=AVERAGE(AVERAGEIFS(aRange,xRange,123,yRange,456),AVERAGEIFS(bRange,xRange,123,yRange,456))

That selects and separately averages the values in aRange and bRange where the corresponding value in xRange is 123 and the corresponding value in yRange is 456.  Then the separate averages are averaged.


-----
[1] [EDIT] If the size of the qualified ranges might not be the same due to a misunderstanding when you wrote "the criterion is the same", you might need the following:

=(AVERAGEIF(aRange,123)*COUNTIF(aRange,123) + AVERAGEIF(bRange,123)*COUNTIF(bRange,123))
* (COUNTIF(aRange,123)+COUNTIF(bRange,123))
