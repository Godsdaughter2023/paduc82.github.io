Once you set either the Original or the Revised to the currently opened file name, you can then click Browse to browse for the other document (or, if it's listed in the dropdown, choose it from there).

steve rindsberg say:

Working on files stored in the cloud (or even on removable media) is not a good thing to do.  It may work most of the time, but when it doesn't, it can cost you hours of work.

Always copy the files to your local hard drive (to a folder that's NOT synched or automatically backed up) and work on it from there.  Then copy it back to the cloud or external storage when you've saved it.

steve rindsberg say:

Longshot, but try restarting Windows.

And after you do that, open your file by starting PowerPoint then File | Open.  Specifically do not start Windows Explorer to locate and doubleclick the file.

Voodoo?  Not exactly, much though it may sound like it.

PowerPoint sometimes gets into a snit and leaves hidden copies of itself behind. These zombiePPT instances may hold a previous file open, making it read-only.  That could possibly cause some of the problems you're seeing.

Restarting Windows kills off the zombies.  If only the real ones were so easy to dispose of.

And when you click a file in Win Explorer, if the Preview pane is open, Windows opens a hidden instance of PowerPoint to create the preview.  The hidden instance holds the file open, which again can cause PPT to view it as read-only.
