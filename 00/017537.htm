rvnfamily say:

Text Color dependent on value in mail merge

Hi All,

I'm doing a mail merge, creating labels in Word from data in Excel. One of the fields in my Word label layout is <<M_1gr>>, which refers to a student's grade. I want to have each of the possible grade numbers (K, 1, 2, 3, 4, 5) be different (for this instance alone, as I use the same field elsewhere and do not want it to be colored there). For example, I want all of grade 1 to be colored blue, grade 2 to be colored red, etc. Can anyone tell me, in extremely simple terms as this is my first go, how to do this? By the way, my real field name is 1gr, and I don't even know where the M_ preceding it came from. 

Thank you,

Terri

macropod say:

Assuming your «M_1gr» field outputs only K, 1, 2, 3, 4, 5 and you want to change just the colour of those characters, you could use a series of fields coded as:

{IF«M_1gr»= "K" {MERGEFIELD M_1gr \* Charformat}}
{IF«M_1gr»= "1" {MERGEFIELD M_1gr \* Charformat}}
{IF«M_1gr»= "2" {MERGEFIELD M_1gr \* Charformat}}
{IF«M_1gr»= "3" {MERGEFIELD M_1gr \* Charformat}}
{IF«M_1gr»= "4" {MERGEFIELD M_1gr \* Charformat}}
{IF«M_1gr»= "5" {MERGEFIELD M_1gr \* Charformat}}

where the 'M' in 'MERGEFIELD' is formatted the way you want the output to appear. Similarly, to shade some text associated with the grades, you could use a series of fields coded as:

{IF«M_1gr»= "K" "Coloured text for Grade K"}
{IF«M_1gr»= "1" "Coloured text for Grade 1"}
{IF«M_1gr»= "2" "Coloured text for Grade 2"}
{IF«M_1gr»= "3" "Coloured text for Grade 3"}
{IF«M_1gr»= "4" "Coloured text for Grade 4"}
{IF«M_1gr»= "5" "Coloured text for Grade 5"}

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. Likewise, you can't type or copy & paste the chevrons (i.e. '« »') - they're part of the actual mergefields, which you can insert from the 'Insert Merge Field' dropdown. The spaces represented in the field construction are all required.

If you want to shade the actual labels, that's somewhat more involved - see Conditionally Shade Table Cells in the Mailmerge Tips and Tricks thread at:
http://www.msofficeforums.com/mail-merge/21803-mailmerge-tips-tricks.html
or:
http://windowssecrets.com/forums/showthread.php/163017-Word-Mailmerge-Tips-amp-Tricks

Ah good, that means your system is working normally.

To understand what I mean you need to realize there is a difference between how data is stored versus how it is shown. Thus I can store -1, and show it as "True" or "Yes" or a checked checkbox. Each of these representations store the value as -1.

Same with dates: they are stored as some complicated 8-byte value, and you can show it as "2/9/2015" or "Monday February 9, 2015, or several other formats.

Typically you use the Format property of your field (textbox) to tell Access how to display it.

Set filter to just black and if any visible rows then assign to rng variable.

Add the pattern to the Criteria1 filter and if any visible rows then add that to the rng variable. karen lorr say:

It's OK.  This is the simplest I could come up with.

 

Shown here in case anyone has the same question.

 

 

Private Sub txtOne_AfterUpdate()
Dim strCount As Long
strCount = DCount("ID", "TableName", "FiedName = Forms![FormName]![txtOne]")
If strCount > 0 Then
MsgBox "Found " & strCount & " records", , "Success"
Else
MsgBox "No records found", , "Failure"
End If
End Sub

ken sheridan say: When calling the DCount function you should concatenate the value of the parameter into the criteria expression rather than reference it:

strCount = DCount("*", "TableName", "FieldName = """ & Me.txtOne & """")

Or you could create a query:

SELECT COUNT(*) AS NumberFound
FROM TableName
WHERE FieldName = Forms!FormName!txtOne;

You can then look up the value with:

strCount = DLookup("NumberFound", "QueryName")
