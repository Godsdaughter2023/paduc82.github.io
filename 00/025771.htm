jay freedman say:

The QAT is stored in the file C:\Users\<login name>\AppData\Local\Microsoft\Office\Word.officeUI. (Each Office program has a separate *.officeUI file in that folder.)

There are two kinds of AutoCorrect entries: "Plain Text" and "Formatted Text".

  • The Formatted entries created in Word are stored in the Normal.dotm template and are available only in Word. Those created in Outlook are stored in the NormalEmail.dotm template and are available only in the Outlook editor.
  • The Plain Text entries, which are shared with all other Office programs, are stored in a *.acl file whose name depends on the proofing language applied to the text. For example, entries made with English (US) are in C:\Users\<login name>\AppData\Roaming\Microsoft\Office\MSO1033.acl, where 1033 is the Locale ID for that language.

macropod say:

Assuming:

• your document consists of a single table;

• you have formatted content in the cells; and

• you need to keep the content in the same order (across then down),

you could use a macro like the following, but don't expect a fast result - on my laptop, it took about 3 minutes to process a 1,000 cell table with only 3 empty cells (the first one being 8th in the table).

 

Sub Demo()
Dim eTime As Single, oCellTgt As Cell, oCellSrc As Cell, i As Long, j As Long
' Start Timing
eTime = Timer
Application.ScreenUpdating = False
With ActiveDocument.Range.Tables(1).Range.Duplicate
  j = .Cells.Count
  For Each oCellTgt In .Cells
    i = i + 1
    StatusBar = "Processing cell " & i & " of " & j
    If i Mod 40 = 0 Then DoEvents
    If Len(oCellTgt.Range.Text) = 2 Then
      If oCellSrc Is Nothing Then Set oCellSrc = oCellTgt.Next
      While Len(oCellSrc.Range.Text) = 2
        If oCellSrc.Range = .Cells(.Cells.Count).Range Then GoTo Done
        Set oCellSrc = oCellSrc.Next
        i = i + 1
        If i Mod 40 = 0 Then DoEvents
      Wend
      oCellSrc.Range.Cut
      oCellTgt.Range.Paste
      If oCellSrc.Range = .Cells(.Cells.Count).Range Then GoTo Done
      Set oCellSrc = oCellSrc.Next
    End If
  Next
End With
Done:
StatusBar = "Finished!!"
Application.ScreenUpdating = True
' Calculate elapsed time
eTime = (Timer - eTime + 86400) Mod 86400 ' Just in case execution time spans midnight
MsgBox "Execution took " & Format(eTime / 86400, "hh:mm:ss")
End Sub

 

If you weren't worried about preserving formatting, or were content to simply shift cells up (so that there might be some empty cells at the bottom of each column), or you didn't really care about the data order, a different approach could be taken, with much faster processing.

Just a thought - a simpler way of creating the new table would be to add the required # of rows to the existing one, then split it after the last original row:

 

Sub ScratchMacroII()
Dim oColl As New Collection
Dim oCell As Word.Cell
Dim lngIndex As Long
Dim oTbl As Word.Table
Dim sngTime As Single
  'Start Timer
  sngTime = Timer
  Application.ScreenUpdating = False
  System.Cursor = wdCursorWait
  Set oTbl = ActiveDocument.Tables(1)
  Set oCell = oTbl.Range.Cells(1)
  lngIndex = oTbl.Rows.Count
  'Store Cells with content.
  Do
    If Len(oCell.Range.Text) > 2 Then oColl.Add oCell
    Set oCell = oCell.Next
  Loop Until oCell.Range.Start = oTbl.Range.Cells(oTbl.Range.Cells.Count).Range.Start
  'Make a duplicate table below the original table, with the required # of rows.
  oTbl.Rows.Last.Select
  Selection.InsertRowsBelow -Int(-1 * oColl.Count / oTbl.Columns.Count)
  oTbl.Split lngIndex + 1
  Set oTbl = ActiveDocument.Tables(2)
  'Repopulate cell content in duplicate table.
  For lngIndex = 1 To oColl.Count
    If lngIndex Mod 50 = 0 Then
      DoEvents
      StatusBar = "Processing " & lngIndex & " of " & oColl.Count
    End If
    Set oCell = oTbl.Range.Cells(lngIndex)
    oColl.Item(lngIndex).Range.Copy
    oCell.Range.Paste
  Next lngIndex
  'Delete the original table.
  ActiveDocument.Tables(1).Delete
  ActiveDocument.Tables(1).Range.Characters.First.Previous.Delete
  System.Cursor = wdCursorNormal
  StatusBar = "Finished!!"
  Application.ScreenUpdating = True
  'Calculate elapsed time
  sngTime = (Timer - sngTime + 86400) Mod 86400 ' Just in case execution time spans midnight
  MsgBox "Execution took " & Format(sngTime / 86400, "hh:mm:ss")
End Sub

hansv mvp say:

You can shorten

 

     Set MyRangePP = ActiveDocument.Range(Start:=ActiveDocument.Paragraphs(1).Range.Start, _
                                                                           End:=ActiveDocument.Paragraphs(1).Range.End)

 

to

 

     Set MyRangePP = ActiveDocument.Paragraphs(1).Range

 

The text of this range will include the paragraph marker at the end (vbCr = character with code 13).

Instead of a string with 20 spaces (hard to count, so easy to make a mistake), you can use Space(20).

To delete the paragraph, you don't have to select it. Simply use MyRangePP.Delete.

So the code becomes:

 

Sub Test()
    Dim MyText As String
    Dim MyRangePP As Range

    Set MyRangePP = ActiveDocument.Paragraphs(1).Range
    MyText = MyRangePP.Text
    If MyText = Space(20) & vbCr Then
        MyRangePP.Delete
    End If
End Sub
