daniel g. grau say:
I think that you are getting this message for the following reasons:

  1. In Word > Preferences > Spelling and Grammar you have enabled "Check spelling as you type" and/or "Check grammar as you type"; AND
  2. The language you are using in your document is not installed (or Word cannot find it)

So, as soon as you start typing, Word tries to check your text and gives you the error message. First thing to do:

  • Close all open documents, then create a blank document (Command N). Since there is no text, Word should not give you any message.
  • Disable the options mentioned in item 1 above. This should get rid of the messages. (However, if you open a document with text and try to spell check it, you will see the message once again, so don't do any spell checking until you've completed the following step.)

You now need to install the missing files
If you still have the Office installer around, open it. If not, download it from here.

  1. Open the installer (if this is a .dmg image, it will mount as a virtual disk in your Desktop).
  2. Press the following sequence of buttons: Continue, Continue, Agree, Customize.
  3. You will see the Custom Install options.
  4. Deselect everything, except Proofing Tools. You can install only the languages you will be using, or all of them.
  5. Click on the Install button.

After installing the language proofing tools, it is a good idea to check for updates in Word's Help menu, in the main menu bar.

bernie deitrick say:

    1) Copy this code.
    2) Right-Click the first sheet tab
    3) Select "View Code"
    4) Paste the code into the window that appears.
    5) Save the file as a macro-enabled .xlsm file.
    6) Enter values as you described - I have assumed that the entry of the title is into column A, in the first row of each range.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim rngC As Range

    If Target.Cells.Count > 1 Then Exit Sub
    If Target.Value = "" Then Exit Sub

    Set rngC = Range("A10,A19,A28,A37")

    If Intersect(Target, rngC) Is Nothing Then Exit Sub
    Worksheets((Target.Row - 1) / 9 + 1).Name = Target.Value
 End Sub

  • Run few instances / small parts of the code at both versions, raising slowly the instance number. Meanwhile monitor speeds and system resources (especially cache and disks)

andreas killer say:

Create a root node before you load the data from the sheet, e.g.

 

Set MyRootNode = Me.Market_TV.Nodes.Add(Text:="MyRootNode")
LoadTreeSheet Sheet2, Me.Market_TV, MyRootNode

 

Note: The code below uses column A for the contents and column B for the level.

Andreas.

 

Function GetNodesDepth(ByVal Root As Node) As Long
  'Return depth of this node
  Do While Not Root Is Nothing
    GetNodesDepth = GetNodesDepth + 1
    Set Root = Root.Parent
  Loop
End Function

Sub SaveTreeSheet(ByVal WS As Worksheet, ByVal TreeView As MSComctlLib.TreeView, _
    Optional Root As MSComctlLib.Node = Nothing)
  'Save all child nodes of root, or whole tree if no root is given, into a sheet
  Dim Stack() As Node
  Dim StackPtr As Long
  Dim Line As Long, DecLevel As Long
  On Error GoTo ExitPoint
  'Start by the root node if no node is given
  If Root Is Nothing Then
    Set Root = TreeView.Nodes(1)
  Else
    DecLevel = GetNodesDepth(Root)
    Set Root = Root.Child
  End If
  ReDim Stack(1 To 64) As Node
  'Prepare sheet
  WS.Columns(1).Resize(, 2).ClearContents
  WS.Columns(1).NumberFormat = "@"
  WS.Columns(2).NumberFormat = "0"
  Do While Not Root Is Nothing
    Line = Line + 1
    WS.Cells(Line, 1) = Root.Text
    WS.Cells(Line, 2) = GetNodesDepth(Root) - DecLevel
    'Are there subnodes?
    If Root.Children > 0 Then
      'Yes, save this node
      StackPtr = StackPtr + 1
      If StackPtr > UBound(Stack) Then _
        ReDim Preserve Stack(1 To UBound(Stack) * 2) As Node
      Set Stack(StackPtr) = Root
      'Go on with the child node
      Set Root = Root.Child
    Else
      'Go on with the next node
      Set Root = Root.Next
      'Get the next node from stack if there isn't a node
      Do While Root Is Nothing And StackPtr > 0
        Set Root = Stack(StackPtr).Next
        StackPtr = StackPtr - 1
      Loop
    End If
  Loop
ExitPoint:
End Sub

Sub LoadTreeSheet(ByVal WS As Worksheet, ByVal TreeView As MSComctlLib.TreeView, _
    Optional Root As MSComctlLib.Node = Nothing)
  'Load all nodes from a sheet into a tree, optional as child to an existing node
  Dim Contents As String, Level As Long, IncLevel As Long
  Dim Stack() As Node
  Dim StackPtr As Long
  Dim Line As Long

  On Error Resume Next
  If Not Root Is Nothing Then IncLevel = GetNodesDepth(Root)
  StackPtr = 64
  Do While StackPtr < IncLevel
    StackPtr = StackPtr * 2
  Loop
  ReDim Stack(1 To StackPtr) As Node
  If Not Root Is Nothing Then Set Stack(IncLevel) = Root

  'Read the sheet line by line
  On Error GoTo ExitPoint
  Line = 1
  Do While Not IsEmpty(Cells(Line, 1))
    'Get the contents of the node
    Contents = Cells(Line, 1)
    Level = Cells(Line, 2)
    If Level + IncLevel > UBound(Stack) Then _
      ReDim Preserve Stack(1 To UBound(Stack) * 2) As Node
    If Level = 1 And IncLevel = 0 Then
      'This is a root node
      Set Stack(Level) = TreeView.Nodes.Add(Text:=Contents)
    Else
      'This is a child node
      Set Stack(Level + IncLevel) = TreeView.Nodes.Add( _
        Relative:=Stack(Level + IncLevel - 1), RelationShip:=tvwChild, Text:=Contents)
    End If
    Line = Line + 1
  Loop
ExitPoint:
End Sub

andreas killer say:

I've tried a little bit around with the TreeView control and can now understand why you have problems.

The main problem is that TreeView.HitTest did not work properly in VBA, because the coordinates in X and Y must be in TWIPS.

I was a little bit confused about the coordinates, because in mouse events we get them in pixels, but in drag events they are in points. The function HitNode works now accurately.

You need a ListBox and a TreeView in a Userform for the code below.

Andreas.

Option Explicit

Private Const TWIPSPERINCH = 1440
Private Const LOGPIXELSX = 88
Private Const LOGPIXELSY = 90
Private Declare Function GetDC Lib "user32" ( _
  ByVal hwnd As Long) As Long
Private Declare Function GetDeviceCaps Lib "gdi32" ( _
  ByVal hdc As Long, ByVal nIndex As Long) As Long
Private Declare Function ReleaseDC Lib "user32" ( _
  ByVal hwnd As Long, ByVal hdc As Long) As Long

'The node that we drag over
Private DragNode As MSComctlLib.Node
'A continous key generator
Private DragKeyCount As Long

Private Function HitNode(TV As TreeView, _
    ByVal X As Single, ByVal Y As Single, _
    Optional InMouseEvent As Boolean = False) As MSComctlLib.Node
  'Returns the node that is hit when dragging

  Static PixelsPerInchX As Long, PixelsPerInchY As Long
  Dim hdc As Long
  If PixelsPerInchX = 0 Then
    'Get number of pixels per logical inch along the screen
    hdc = GetDC(0)
    PixelsPerInchX = GetDeviceCaps(hdc, LOGPIXELSX)
    PixelsPerInchY = GetDeviceCaps(hdc, LOGPIXELSY)
    ReleaseDC 0, hdc
  End If
  'Note: Coordinates in mouse events are already in Pixels!
  If Not InMouseEvent Then
    'Calculate Points to Pixels
    X = X / (TWIPSPERINCH / 20) * PixelsPerInchX
    Y = Y / (TWIPSPERINCH / 20) * PixelsPerInchY
  End If
  'Ask Treeview which node is hit
  Set HitNode = TV.HitTest( _
      X * TWIPSPERINCH / PixelsPerInchX, _
      Y * TWIPSPERINCH / PixelsPerInchY)
End Function

Private Sub ListBox1_MouseMove(ByVal Button As Integer, _
    ByVal Shift As Integer, ByVal X As Single, ByVal Y As Single)
  'Take up selected element, starts dragging
 
  Dim MyDataObject As DataObject
  Dim Effect As Integer
  If Button = vbKeyLButton Then
    Set MyDataObject = New DataObject
    MyDataObject.SetText ListBox1.Value
    Effect = MyDataObject.StartDrag
  End If
End Sub

Private Sub TreeView1_Click()
  'Highlight the selected item
  Set TreeView1.DropHighlight = TreeView1.SelectedItem
End Sub

Private Sub TreeView1_Enter()
  'Turn highlight on, select last element
  Set TreeView1.DropHighlight = DragNode
End Sub

Private Sub TreeView1_Exit(ByVal Cancel As MSForms.ReturnBoolean)
  'Turn highlight off on exit, remember last element
  Set DragNode = TreeView1.DropHighlight
  Set TreeView1.DropHighlight = Nothing
End Sub

Private Sub TreeView1_OLEDragOver(Data As MSComctlLib.DataObject, _
    Effect As Long, Button As Integer, Shift As Integer, _
    X As Single, Y As Single, State As Integer)
  'Higlight the element that we drag over
 
  'Get the node where we are over
  Set DragNode = HitNode(TreeView1, X, Y)
  'Enable or disable highlighting
  Set TreeView1.DropHighlight = DragNode
  'Expand tree if necessary
  If Not DragNode Is Nothing Then DragNode.Expanded = True
  'Show that we are in copy mode
  Effect = fmDropEffectCopy
End Sub

Private Sub TreeView1_OLEDragDrop(Data As MSComctlLib.DataObject, _
    Effect As Long, Button As Integer, Shift As Integer, _
    X As Single, Y As Single)
  'Drop to last highlighted item
  Dim Item As String
  'Get item to drop
  Item = Data.GetData(1)
  'Do nothing if there is nothing
  If Len(Item) = 0 Then Exit Sub
  'Add a node
  With TreeView1
    If DragNode Is Nothing Then
      'Add to root
      Set DragNode = .Nodes.Add(, , "N" & DragKeyCount, Item)
    Else
      'Add to existing node
      Set DragNode = .Nodes.Add(DragNode.Key, tvwChild, "N" & DragKeyCount, Item)
    End If
    'Increment the key
    DragKeyCount = DragKeyCount + 1
    'Expand the node
    DragNode.Expanded = True
  End With
End Sub

Private Sub UserForm_Activate()
  'Initialize key generator with number of items in Treeview
  DragKeyCount = TreeView1.Nodes.Count
End Sub

Private Sub UserForm_Initialize()
  'Enable drag&drop in Treeview
  TreeView1.OLEDropMode = ccOLEDropManual
  TreeView1.OLEDragMode = ccOLEDropManual
 
  'Just for testing
  Dim I As Integer
  For I = 1 To 10
    ListBox1.AddItem I
  Next
End Sub

japanzaman say: As far as I know there isn't any way to "move" a node on a userform. You simply have to create an identical one and then remove the other one. Also be aware than any child nodes will be deleted as well so it can be rather tricky to do what you are talking about depending on how many children the node you want to copy has. The best advice I can give is start by creating a new node at the desired location, then loop through all the nodes and if the Parent of any node matches the node you want to copy, then copy that node as a child.
