john w. vinson say:

You certainly don't need (and should not have!) any code to open the Subforms. A Subform object is a child of its parent form, and opens when the parent form is opened, using its Master Link Field and Child Link Field properties to display the appropriate records.

 

That said... the symptom you're seeing occurs when two conditions hold:

 

1. The form's Recordsource returns no records (so you don't see any existing data) and

2. The form's Recordsource, or the form itself, is not updateable (so you don't see the blank new record).

 

What are the subform's Recordsources? What are their Master/Child Link Fields? Is there any code in the form that runs when the form is opened?

I was experiencing a similar problem.  I solved it by enclosing the identifier found in the "Control Source" property with brackets [].

SELECT ..., [Training Due], ... FROM ...

 

and the form control for that column had ...

 

Control Source=Training Due

 

when I did this,

 

Control Source=[Training Due]

 

the problem went away.
