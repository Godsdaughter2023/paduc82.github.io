I want to be able to change the values in A1 and C200 all day long and the other cell show the value of its sister cell. How can this be done?

nuculerman say:

Right click on the worksheet in question and click "view code."  Copy and paste the following into the module

Private Sub Worksheet_Change(ByVal Target As Range)

If Target.Address = "$A$1" Then
    If Target.Value <> Range("C200").Value Then Range("C200").Value = Target.Value
ElseIf Target.Address = "$C$200" Then
    If Target.Value <> Range("A1").Value Then Range("A1").Value = Target.Value
End If

End Sub

Private Sub Worksheet_Change(ByVal Target As Range)

ken sheridan say: It is the reference as shown in your deleted post, but you also need to set it in Access.  The image you posted was of the dialogue in Word.

If you merely want to open a Word file from Access you can do this in a number of ways.  One is by means of the FollowHyperlink method, but I normally call the Windows API ShellExecute function which you can do by adding the following module to the database:

' basShellExecute Module
Option Compare Database
Option Explicit

Public Const SW_HIDE = 0
Public Const SW_SHOWNORMAL = 1
Public Const SW_SHOWMINIMIZED = 2
Public Const SW_SHOWMAXIMIZED = 3
Public Const OP_OPEN = "Open"
Public Const OP_PRINT = "Print"


Declare Function ShellExecute& Lib "shell32.dll" Alias "ShellExecuteA" (ByVal _
hwnd As Long, ByVal lpOperation As String, ByVal lpFile As String, ByVal _
lpParameters As String, ByVal lpDirectory As String, ByVal nshowcm As Long)

Sub ShellToFile(strPath As String, _
            Optional strOperation As String = OP_OPEN, _
            Optional lngShow As Long = SW_SHOWNORMAL)

    Dim lngRetVal As Long
    Dim lngHwnd As Long
    
    lngHwnd = Application.hWndAccessApp
    
    lngRetVal = ShellExecute(lngHwnd, strOperation, strPath, _
        vbNullString, CurDir, lngShow)
        
    If lngRetVal < 32 Then
        MsgBox "Unable to open file " & strPath, vbInformation, "Warning"
    End If
    
End Sub

Then call the ShellToFile function, passing the path to your Word file to it, e.g.

ShellToFile "F:\MyWordFiles\MyFile.docx"

ken sheridan say: First create your Word letter file as a mail merge document in word adding fields in the appropriate places for each of the columns returned by your query.  Place the Word file in the same folder as your database file.

Then import the four modules from my demo file into your own database.  To execute the merge you need to amend the code from the Click event of the relevant button in my main form, so for the basic merge without saving option the code is:

    On Error GoTo Err_Handler

    Dim strDataDoc As String, strFormLetterDoc As String
    Dim strExtension As String
    Dim RetVal
    
    ' insert list of AddressID values into hidden text box
    BuildAddressList
    
    Select Case Int(SysCmd(acSysCmdAccessVer))
        Case 8
        strExtension = ".doc"
        Case Else
        strExtension = ".txt"
    End Select
    
    strDataDoc = CurrentFolder() & "AccWordAddressList" & strExtension
    strFormLetterDoc = CurrentFolder() & "AccWordMergeLetter.Doc"
    
    ' ensure current record is saved
    RunCommand acCmdSaveRecord
        
    RetVal = WordMerge("qryAddressList", strDataDoc, strFormLetterDoc)

Exit_here:
    Exit Sub
    
Err_Handler:
    MsgBox Err.Description & " (" & Err.Number & ")", vbExclamation, "Error"
    Resume Exit_here

The amendments you need to make are:

1.  Change AccWordAddressList to whatever you want to call the text file which the procedure creates.  This can be any name of your choice.

2. Change AccWordMergeLetter.Doc to the name of your Word mail merge file.

Note that my code is in a bound form, so needs to ensure the current record is saved.  If you run it from a button on an unbound dialogue form take out the following lines:

    ' ensure current record is saved
    RunCommand acCmdSaveRecord

Clearing chat history on the Skype for Windows Desktop will not have any effect on the chat history saved in Skype UWP.

Clearing chat history with Skype Desktop deletes the history from

%appdata%\Skype\<Skype Name>\main.db

while the chat history on Skype UWP is saved in

%localappdata%\Packages\Microsoft.SkypeApp_kzf8qxf38zg5c\LocalState\<Skype Name>\main.db
