guessed say:

The TOC uses a series of styles TOC 1, TOC 2 etc to style the paragraphs. You can edit those styles to achieve the look you want. There shouldn't be a need to go back to the Insert TOC dialog once you have one in your document.

Sometimes it is hard to actually get at those styles to edit them. Firstly, the TOC styles may be hidden and also in a TOC with the \h switch, the style list may be displaying the Hyperlink character style instead of the underlying paragraph style applied. On Windows you can temporarily remove the character style by pressing Ctrl-Q and Ctrl-Space. Often the TOC styles have the style setting to update automatically so you can just change one paragraph directly and the other instances automatically update to reflect that change.

I wouldn't be using Page Break before on a TOC style unless you want a TOC that takes up lots of pages. The Keep Lines Together setting shouldn't be useful unless your TOC entries wrap to multiple lines - it keeps all the lines of a paragraph together on a single page. This shouldn't be the case in your TOC because a TOC entry longer than one line should be an exception.

john korchok say:

It's not hard to format the TOC styles:

  1. On the Home tab, click on the Styles Pane icon. The Style Pane opens.
  2. At the bottom, change List to Styles in Use.
  3. Click at the right end of TOC 1, 2 or 3 styles names and choose Modify Style.
  4. If the change should apply to the template so future documents have the formatting, check Add to template.
  5. Make any style changes, then click on OK to save.

joeu2004 say:

Thelma wrote:

I'm having problems to combine the following three nested functions into one[....]

IF(F2="LOW PRESSURE",IF(E2<=12,"26",IF(E2<=30,"24",IF(E2<=54,"22",IF(E2<=84,"20",IF(E2>=85,"18",""))))))
IF(F2="MEDIUM PRESSURE",IF(E2<=18,"24",IF(E2<=48,"22",IF(E2<=72,"20",IF(E2>=73,"18","")))))
IF(F2="HIGH PRESSURE",IF(E2<=48,"22",IF(E2<=72,"20",IF(E2<=96,"18",IF(E2>=97,"16","")))))

Remember that the form of the function is IF(condition, value_if_true, value_if_false).  So the form of the nested function is:

IF(F2="low pressure", stuff, IF(F2="medium pressure", stuff, IF(F2="high pressure", stuff, "")))

where "stuff" is the relevant nested IF(E2<=...) function for each value in F2.

But before we get to a turnkey formula, there are some things that might require your attention.

(I use the "low pressure" formulas as an example.  Everything below applies to the other formulas as well.)

1. The "low pressure" formula returns the null string ("") only for 84<E2 and E2<85.  I suspect that is not your intention.  Instead, I suspect that you want to return "18" for E2>84.  Right?

2. The "stuff" formula returns strings like "26", "24", etc.  I suspect that is not your intention; or it shouldn't be.  Instead, I suspect that you intend to use the results as numbers.  Right?

In that case, you should return numbers; that is, 26, 24, etc without quotes.

3. The "low pressure" formula returns FALSE for F2<>"low pressure".  I suspect that is not your intention.  Instead, I suspect you want to return the null string ("").  Right?

In that case, I already corrected that in the paradigm above ("so the form ... is").

Applying my assumptions, your single formula might be:

=IF(F2="LOW PRESSURE",

      IF(E2<=12, 26, IF(E2<=30, 24, IF(E2<=54, 22, IF(E2<=84, 20, 18)))),

  IF(F2="MEDIUM PRESSURE",

      IF(E2<=18, 24, IF(E2<=48, 22, IF(E2<=72, 20, 18))),

  IF(F2="HIGH PRESSURE",

      IF(E2<=48, 22, IF(E2<=72, 20, IF(E2<=96, 18, 16))), "")))

Finally, we might be able to simplify the "stuff" formulas if we knew something about the magnitude and precision of the values in E2.  For example, if E2 is always an integer not less than zero, you might consider:

=IF(F2="LOW PRESSURE", LOOKUP(E2, {0, 12.1, 30.1, 54.1, 84.1}, {26, 24, 22, 20, 18}),

  IF(F2="MEDIUM PRESSURE", LOOKUP(E2, {0, 18.1, 48.1, 72.1}, {24, 22, 20, 18}),

  IF(F2="HIGH PRESSURE", LOOKUP(E2, {0, 48.1, 72.1, 96.1}, {22, 20, 18, 16}), "")))

[EDIT] If you want to allow for E2<0, replace 0 with -1E300 or some other minimum value.

But be careful with your assumptions.  If E2 is calculated and you do not explicitly round or truncate to an integer (or 1 decimal place), the value that appears in E2 might be different from the actual value in E2.

Usually, formatting alone is not sufficient to round an actual value to the way that it appears.

jay freedman say:

<Moved from Windows 7 | Music, Photos & Videos>

There are many images to be entered into a report.

The goal is to have all the images the same size on all of the pages.

Four images are to be spaced evenly on each page.

Is there a way to automatically insert the images?

Bonus question:

Is it possible to automatically caption each image with its file name?

Create a table in a document, with the cells positioned the way you want the pictures to appear and with the width you want them to be (at this point the height doesn't matter). On the Table Tools | Layout tab of the ribbon, click the AutoFit button and choose Fixed Column Width in the dropdown.

Select the whole table, press Alt+F3, and enter a name (for example, PicTable). Open the Gallery dropdown and choose Table. Open the Category dropdown and click Create New Category, then give a category name (preferably one that starts with a letter near the beginning of the alphabet, or an underscore, so it sorts to the top of the list). Click OK -- you've now created a custom building block that you can insert wherever you want pictures.

To use this, you can click Insert > Table > Quick Tables and click the building block's picture in the gallery:

Alternatively, you can start to type the name of the building block in the document. As soon as the typed part is unique among the names of building blocks, a tooltip will suggest the full name, and pressing Enter will insert the block.

Insert or drag/drop a picture into a cell. If the picture is larger than the cell, it will be automatically resized to fit the width. Smaller pictures won't be enlarged, though.

Automatic captioning is not available as a built-in feature of Word. It may be possible to write a macro to do it.
