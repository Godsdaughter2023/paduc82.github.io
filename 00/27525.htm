cristhianescobarpq say: Hello USA9854.

You can not add an alias that is already associated with another Microsoft account. Aliases can not be used to share information between two existing Microsoft accounts.

Existing @ hotmail.com, @ live.com, Outlook.com and @ msn.com addresses can not be added as aliases.

If you remove an email address from a non-Microsoft domain (for example, @ gmail.com), it will be immediately available so you can add it as an alias to another Microsoft account. If you remove an email address from the domain @ outlook.com, you can recreate it as a new account or alias after 30 days. As regards email addresses from other Microsoft domains (for example, @ hotmail.com, @ live.com, @ outlook.com or @ msn.com), they can not be re-added as aliases to any another Microsoft account once they have been removed.

If you remove an email address from @hotmail, @MSN or @Live, it can not be recovered.

You can see a lot more about aliases in: Manage aliases in your Microsoft account.

https://support.microsoft.com/en-us/help/12407/...

For the message that shows you the process of creating accounts you are using a domain that is part of those that are reserved by Microsoft, in order to keep running the accounts that are still active. These reserved domains are @Live, @Msn and @Hotmail with extension of countries such as .es, .cl, among others; the addresses with these domains can no longer be retrieved.

Currently, the only domain available to recover or create new accounts are: @ Hotmail.com, @ Outlook.com and @ Outlook.es.

Source: https://answers.microsoft.com/es-es/outlook_com...

I hope this information is useful.

The developers eventually gave up their defeatist attitude toward AV self-protection and hardened the Microsoft Antimalware Service against shutdowns:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_scanning/dose-microsoft-security-essentials-have-strong/5d7a4d3f-1316-4de6-ab21-de918562737a

 

Because this is by design, please note that posting methods for defeating the self-protection of the Antimalware Service might be construed as a violation of the COC in this forum, as it was in the thread cited below – although this is less likely now that we're effectively without moderators in the Virus and Malware forum:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/why-cant-i-control-the-msmpsvc-service/1c7cad27-f7c8-4951-a583-fc4d899d3c61?page=1 

 

Even if you're able to break MSE's self-protection in order to follow the steps in this fix, I think it will most likely prove to be an exercise in futility – for these reasons:

 

1.) The error normally doesn’t seem to have any impact at all on MSE’s functionality.

2.) The fix that you’re trying to achieve is frequently reported to have a short-lived success, only to be undone whenever there’s a new manifestation of the underlying error condition.

3.) There are other free AV solutions that don’t seem to exhibit these error conditions; or the registry corruptions that often lead to installation/uninstallation problems; and that also get better protection ratings – so you’re not getting much in return for the time that you invest in trying to fix MSE.

Changing the default to "no markup" - Microsoft Community

aimaral say:

Changing the default to "no markup"

Can the default in Word 2013 be changed so that documents open to "no markup" instead of "all markup" or "simple markup"? Thanks.

doug robbins - mvp office apps & services (word) say:

A document will open in the mode in which it was saved.


However, if you had an AutoOpen macro in an Add-in that contained the following code, it should turn off the tracking of Revisions and display of Revisions


With ActiveDocument
    If .ProtectionType = wdAllowOnlyRevisions Then
        .Unprotect
    End If
    .TrackRevisions = False
    .ShowRevisions = False
End With



It would not work however if the document was protected for Tracked Changes with a password applied unless you also supplied the password by replacing the line


         .Unprotect


with


         .Unprotect password


where in place of password, you supply the actual password.

The display options on the Review tab are usually not saved with a document. Switching between "All Markup" and "Simple Markup" seems to be a global setting for some users, that is, it applies to all documents, though.

In theory, (some of) the options at Review tab | Show Markup can be saved with the document if you clear "Make hidden markup visible when opening or saving" at File | Options | Trust Center | Trust Center Settings | Privacy Options. But note that the "Make hidden markup visible..." option itself applies to your computer and your Word installation, only. You can't control what other people see if you send them the file.
