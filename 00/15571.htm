It sounds to me like you are trying to use Project Online with no training, trying to figure it out through trial and error.  As you have already discovered, the software is definitely not intuitive.  There are several things you need to do to try and resolve this problem:

First, ask your Project Online application administrator to log into Project Web App and to navigate to the PWA Settings page.  On the PWA settings page, he/she should click the Timesheet Settings and Defaults link.  In the Default Timesheet Creation Mode section of the Timesheet Settings and Defaults page, he/she MUST select the Current Task Assignments option and then click the Save button.  From your description of the problem, it sounds like he/she erroneously selected the Current Projects option instead.

Second, you need to launch Microsoft Project Professional and connect to Project Online.  Open the enterprise project in question.  On the Resource ribbon, click the Build Team from Enterprise button.  In the Build Team dialog, add all your team members to the project and then click the OK button.  By the way, your team members MUST be enterprise resources that were added to the Enterprise Resource Pool by your application administrator.

On the Resource ribbon, click the Assign Resources button, then select the first regular task in the project.  In the Assign Resources dialog, select the resources you want to assign to the task, and then click the Assign button.  Complete this step for every regular task in the project, but not for summary tasks or milestone tasks.  When finished assigning resources to tasks, save the project, and then PUBLISH the project.  You must publish the project in order for tasks to show up on the timesheets of your project's team members.

Third, you need to log into PWA and navigate to the PWA Settings page.  In the Time and Task Management section of the page, click the Manage Timesheets link.  On the Manage Timesheets page, click anywhere on the line for the current timesheet period, but DO NOT click the My Timesheet link.  This will select the current timesheet row.  In the Timesheets ribbon at the top of the page, click the Delete button to delete the erroneously created timesheet.  After the deletion is completed, click the Click to Create link for the current timesheet period to recreate the timesheet.

You should now see tasks scheduled for the current week in your timesheet.  Do you?  Let us know and we will try to help you.

Need to Categorize data passing multiple keyword search.

Hello all,

my question is suppose there are 1000 rows containing different sentences, i want to categorize them using common key words,

For Example: if a sentence containts "un aware", "not aware", "was made aware" it should categorize it to "AWARENESS" , if a sentence contains "made changes", "make changes" it should categorize it to "Changes".

here is my Algorithm for it but its not working accurately

=IF(ISERROR(SEARCH({"not aware","unaware","un aware"},B2))=FALSE,"Awareness",IF(ISERROR(SEARCH({"Itemkey","Item key"},B2))=FALSE,"Itemkeys",IF(ISERROR(SEARCH({"workflow","workflow"},B2))=FALSE,"WorkFlow",IF(ISERROR(SEARCH("educat"},B2))=FALSE,"UserEducation",IF(ISERROR(SEARCH({"cache","cacheing"},B2))=FALSE,"Cache",IF(ISERROR(SEARCH({"wanted to know","wants to know","Client wanted to know"},B2))=FALSE,"CWTK" ,"NA"))))))

please help, kindly suggest if there is a better way to put this up.

Thanks and regards

Malav shelat

bernie deitrick say:

The error is probably due to your named ranges not being properly defined, or equally matched.

Also, you can have as many semi-unique values in your keyvalues list:

unaware    AWARENESS

not aware  AWARENESS

Note though, if you used just this

aware        AWARENESS

then unaware would not be found unless the formula is modified to remove the " " &     and  & " " - those are used to require that whole words be found, so that

bad         BADNESS

would not return BADNESS for badminton, for example.


bernie deitrick say:

Try looking for common substrings for each, like so, using ISNUMBER to get rid of the =FALSE requirement:

=IF(ISNUMBER(SEARCH("aware",B2)),"Awareness",IF(ISNUMBER(SEARCH("key",B2)),"Itemkeys",IF(ISNUMBER(SEARCH("flow",B2)),"WorkFlow",IF(ISNUMBER(SEARCH("educat",B2)),"UserEducation",IF(ISNUMBER(SEARCH({"cache"},B2)),"Cache",IF(ISNUMBER(SEARCH("know",B2)),"CWTK",IF(ISNUMBER(SEARCH("change",B2)),"Changes","NA")))))))

Or, create a list of all the strings that you want to use, and another of the return values.  Name the first list KeyValues and the second Returns, then use this array formula - entered using Ctrl-Shift-Enter

=IFERROR(INDEX(Returns,MATCH(TRUE,ISNUMBER(SEARCH(" "&KeyWords&" "," "&B2&" ")),0)),"Not Found")

If you enter it correctly, Excel will enclose the formula you entered in {  } (see the picture)

In the attached picture, KeyWords is filled tan, Returns is filled green - these lists can be as long as you want, as long as they are both the same number of rows.
