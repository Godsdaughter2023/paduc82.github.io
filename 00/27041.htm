First Cell in a Range - Microsoft Community

gary's student say:

First Cell in a Range

Usually if I define a range, I can get the first cell in that range with (1).  For example:

 

Sub qwerty()
Dim r As Range
Set r = Range("A:A")
MsgBox r(1).Address
End Sub

 

will report:

$A$1

the first cell in the column (the first cell in the range).  However if I define the range with EntireColumn, this does not hold:

 

Sub qwerty2()
Dim r As Range
Set r = Range("A:A").EntireColumn
MsgBox r(1).Address
End Sub


will report:

$A:$A

 

Why don't both subs display the same result??

hansv mvp say:

Another example:

 

Sub qwerty3()
    Dim r As Range
    Set r = Range("A:A")
    MsgBox r.Count
End Sub

 

will result in 1,048,576 in an .xlsx/.xlsb/.xlsm workbook. Here, r.Count is equivalent to r.Cells.Count, the number of cells in the range.

But

 

Sub qwerty4()
    Dim r As Range
    Set r = Range("A:A").EntireColumn
    MsgBox r.Count
End Sub

 

will result in 1. Now, r.Count is equivalent to r.Columns.Count, the number of columns in the range.

hansv mvp say:

By using r(1), you refer to the 1st element of the default property of the range.

Normally, the Cells property is the default property of a Range object, so that r(1) is equivalent with r.Cells(1).

But if you apply EntireColumn to a range, the Columns property becomes the default property, so that r(1) is equivalent to r.Columns(1).

And if you apply EntireRow, the Rows property becomes the default property, so that r(1) is equivalent to r.Rows(1).

This feature only works when you do a SHUTDOWN and then Boot.

It doesn't effect a RESTART.

To turn it off or on...

Lower down on the window, uncheck Fast Startup.

This may or may not help your problem, but is easy to check.

jay freedman say:
Try starting Word in safe mode by holding down the Ctrl key while you click the program's icon or Start menu item, or by opening the Run box and entering winword.exe  /safe . Then try inserting tables.

If that works, then the most likely cause of the problem is an add-in that's interfering. Try disabling any add-ins, one at a time, restarting Word, and testing the table button. If you find one whose removal makes the problem disappear, you can either update that add-in if possible, or leave it disabled/uninstalled.

Other possible causes are corruption of the Normal.dotm template and incorrect entries in the registry. See http://support.microsoft.com/kb/921541 if needed.

marvin ma msft say:
Hi SJ Law,

Based on your description, your Office 365 account is unable to send emails. According to the error code ” 550 5.1.8 Access denied, bad outbound sender.” It indicates that your account probably has been blocked by Office 365. please contact your organization admin and try to delist your account via following steps:

1. Use the admin account to login Admin portal, click Admin icon to Admin Center.
2. Navigate to Admin centers > Exchange > protections > action center, then release the banned account. And then, since the performance may delay, please wait for 24 hours to check if it works.

If the issue persists, please provide the following information.

1. The entire unmodified Non-Delivery Report (NDR) , to protect your privacy, you may send it to me via Private Message.
2. Please check whether you can send emails to internal users, or can’t send emails to both internal and external recipients? 
3. Did other users in your organization experience the same issue?
