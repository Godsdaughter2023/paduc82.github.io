Hi Paul,

For the question of security groups:
If you have on-premises security groups which need to be migrated, you have to do it as the official document instructs. Please be aware that almost all kinds of on-premises mailbox permissions CANNOT be migrated to Exchange Online. greginmich say:

All web-based malware alerts are fakes. Microsoft actually does have an ongoing conversation with your PC (via telemetry) regarding updates, system errors, malware issues, and so forth – but it never responds with a warning in your web browser. Any legitimate alert from your AV app, or from your OS, will always appear in either a notification area pop-up, or an independent window (outside of your browser window) – never as a pop-up in your web browser.

So whenever you see one of these security alerts pop up in your browser, it always means that somebody out there is looking to scam you, and that they’re using this fake alert just to lure you into calling their “tech support” number. If you call the number, they’ll just try to scam you out of some money by selling you worthless software or services at an exorbitant price, and maybe by stealing your personal information. If you refuse to pay up, there’s also a chance that they'll punish you by installing some malware or by locking you out of your computer. These Tech Support Scam pop-up messages (dialogs) come in lots of familiar themes: virus detections; hacker alerts; firewall intrusions; hardware error reports; system error reports; and what have you – but all of these web-based warnings are fakes, and you just have to ignore the content and close the page.

And of course closing the malicious page is usually the problem, because most of these scam sites try to coerce you into calling their “tech support” number by preventing you from closing your browser. They sometimes do that with a dialog loop, which is just a script that reloads a fake pop-up alert (dialog) every time you try to close it. So in order to escape from one of these dialog-loop based malicious pages, the first thing that you have to do is try to close the alert. That allows the browser to detect the message loop – and the browser will then immediately add a Dialog Loop Protection checkbox (e.g., “Don’t let this page create more messages”) to the fake alert popup. And then once you’ve checked that option, you should be able to close the scam page right along with the popup:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/microsoft-virus/5335f80f-4cfc-49e0-bbcf-8f575730779a?auth=1 

Unfortunately, many of the newer versions of the Tech Support Scam are able to either hide the dialog loop, or to avoid using it altogether by replacing it with a page loop and shifting to full-screen mode. For these scam pages, you can toggle Edge’s full-screen mode with the F11 key, and just use the old familiar Ctrl + Alt + Delete to start Task Manager and then kill Edge. But since the issue with AutoRecover was never addressed, killing Edge with Task Manager means that we now have to revert to using the old “indirect startup” method in order to get rid of the recycled scam page:

1. First, right-click on the taskbar and select Task Manager – or use Ctrl + Alt + Delete if the taskbar is hidden.

2. Then select the browser app and click on the End task button.

3. Now, restart Edge indirectly by entering a search topic (like “tesla”) for a Cortana web search.

4. Then right-click on the search-result tab and choose “Close other tabs”.

If Edge will then close and reopen without bringing up the browser-locking scam page, then the issue was probably just caused by the malicious behavior of the webpage itself, rather than by any real malware. Of course there are a few Tech Support Scam pages out there that won’t be removed by these procedures because they actually lockup the browser with binary malware that’s installed on the system – and these will need to be removed with specialized malware-removal tools:

https://malwaretips.com/blogs/remove-tech-support-scam-popups/ 

Also note that using an ad-blocker might help with this issue: AdBlock, Adblock Plus, Adguard AdBlocker, uBlock Origen, and Ghostery extensions are available for Edge:

Settings and more (…) > Extensions > Get extensions from the Store

kmaage say: On my 6233 (320x240 screen), I've done two movies so far, and got good results with the settings below:

I use "ffmpeg" a command line tool
http://www.videohelp.com/tools/ffmpeg

If you are handy enough to create batch files (.bat) below is what I use. I stuck in a few REMark lines as a kind of documentation. Change things after the = to suit your movie. Make sure the ffmpeg command is one one long line.



REM set variables, all times are in seconds
set start=0

REM specific to your movie:
set input_movielength=7200
set input_moviefile=mymovie.avi
set output_bitrate=80000

set output_framerate=15
REM If you notice a slight "stutter" in the motion
REM you can read up on the -r setting for ffmpeg
REM If your movie's original framerate is 24.97
REM the correct framerate setting is actually
REM a little complicated
REM set output_framerate=15000/1001

REM these control how your files will be named
REM extrazero and zero pad the names with the
REM correct number of zeros, so files sort
REM properly
set extrazero=0
set zero=0
set output_prefix=mymov


REM loop through making all the movie chunks
:beginloop
set /A time=start
if /I %time% GEQ 10 set extrazero=
if /I %time% GEQ 100 set zero=

ffmpeg -i %input_moviefile% -r %output_framerate% -s 320x240 -padtop 16 -padbottom 16 -padcolor 000000 -ar 16000 -ac 2 -vol 600 -acodec libfaac -ab 32000 -ss %start% -t %output_chunksize% -vcodec mpeg4 -vb %output_bitrate% %output_prefix%%extrazero%%zero%%time%.3gp

set /A start += output_chunksize
set /A end += output_chunksize
set /A time = start / 60
if /I %start% GEQ %input_movielength% goto theend
goto :beginloop

:theend
