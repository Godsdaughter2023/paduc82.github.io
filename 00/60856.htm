a. user say:

Sometimes queued of backed up print jobs get stuck and in order to manually delete them you must first stop the Print Spooler Service, the empty the print queue and restart the Print Spooler Service.

Here's how:

From the Star orb, in the box enter:

services.msc

Click services.msc to open the Services applet and locate the Print Spooler Service.

Right click the Print Spooler Service choose Properties them click the Stop button:

With the Print Spooler Service Stopped, navigate to the print spooler folder which is usually here (assuming Windows is installed on your C drive:

C:\Windows\system32\spool\printers

Select all the files in the folder and delete them so the folder is empty:

Close that window, return to the Services applet and choose to Start the Print Spooler Service:

Click OK, then close the Services applet and resubmit any failed print jobs.

 Microsoft must have updated the original PATCH as the numbers were higher on the update I have been receiving the last few days. The installation results were the same as the original Patch.

However my problem became apparent from the replys. In my Windows file I had a "Faultykeybord" text document with a list of files ending with .kbd. I could not find these files anywhere on my computer but there were many keys in my registry " HKEY_LOCAL_MACHINE\Control Set\Control\Keyboard Layouts"  (note the s on the end). I opened each of the 100 or so Keys anf found many that refered to the .kbd files. Most of these Keys address .dll files.

I exported the Keyboard Layout key so it could be restored if errored then I deleted all the Keys refering to .kbd files. I went to the Windows file and deleted the "Faultykeybord" text file and rebooted.

I then went to the windows update site and installed the AM_DELTA_PATCH. It installed flawlessly. The next time I restarted my computer the original PATCH appeared for installation. Upon clicking the button to install it also installed quickly.

I have not noticed any ill results from deleting the Keys that  refered to the .kbd files so I did not restore the Keyboard Layout Key and everything seems to be running great.

Maybe this will help someone else but I assume my problem was unique.

I thank Vinod N  for steering me in the right direction.

Problem solved.
