joeu2004 say: Thomas wrote:

I am trying to calculate the XIRR for the below series of cash flows. [....] I believe the value calculated by the XIRR function is not accurate though - when I remove many of the smaller flows, I am getting around -2%, but the formula inclusive of the entire list of flows is giving me 0.000000002980. Am I overloading the capabilities of the function?


Well, yes.  But not because of any arbitrary limit on the number of cash flows, AFAIK.

One correct IRR for your original cash flows is -9.22701240948999%.  I found that using Solver.  Actually, Solver only came close using default options.  Then I modified the result interatively until the NPV became close to zero.  (Actually, exactly zero!)

And Excel XIRR will return about the same result if we pass a "guess" of -1%.  XIRR(B1:B55,A1:A55,-1%) returns -9.22701240913011%.

We can only speculate about why Excel XIRR does not find that result without a "guess" parameter.  The derivation of IRR is an interative algorithm that works best when there is only one sign change in the net cash flows.  When there are multiple sign changes, as in your example, it is well-known that the algorithm does not work well.  Also, there might be multiple solutions.  These flaws of the algorithm are not limited to the Excel XIRR implementation.  See the IRR wiki page.

That said, my own XIRR implementation does return -9.22701240948997% without a "guess" parameter.  I have long suspected that the Excel implementation does not truly use a Newton-Raphson algorithm, despite Microsoft documentation that says it does, or Excel's N-R implementation is flawed in some subtle way.

In any case, the value 2.98023223876953E-09 that XIRR returns without a "guess" is wrong.  With that value, Excel XNPV returns about -19,932,757.54, which is nowhere near zero.  That is one of the flaws of Excel XIRR:  it should return a #NUM error when it fails to find an IRR such that the NPV is close to zero.

Ironically, Excel XNPV returns a #NUM error when we use the correct IRR value -9.22701240948999%.  I believe that is due to an undocumented limitation of Excel XNPV.

How do I reprogram the function keys? - Microsoft Community

johndouthit say:

How do I reprogram the function keys?

I want to may asterisk bigger for WORD to highlight stuff in my reports. How do I program function key 8 to get asterisk to automatically be entered at font of c. 18 or 20?

suzanne s. barnhill say:

Although it is possible to assign custom functions to most key combinations, changing the action of function keys is not recommended. Instead of using a function key, have you considered using AutoCorrect?

  1. Insert an asterisk, change the font size and style as desired, and select it.
  2. Go to File | Options | Proofing and click AutoCorrect Options.
  3. In the AutoCorrect Options dialog, under "Replace text as you type," you will see an asterisk in the "With" box. The radio button for "Plain text" will be selected; select "Formatted text" instead, and you will see your large asterisk.
  4. In the "Replace" box type some text you want replaced by this large asterisk. I would suggest not using an actual asterisk (you may sometimes want it the normal size), but you could use an asterisk preceded by some character unlikely to be followed by an asterisk. I often use / or \ as the first character in AutoCorrect keys, so you might try /* or \*.
  5. Whenever you type the combination you have selected, you will get the large asterisk.

If you do this you will be bypassing the Nokia Messaging push email. That's okay but just be aware that your phone will be polling for mail on regular intervals which may take a lot more data than with push. If you're not paying for data then it's not an issue, but if you are, just watch out what the data consumption is.

 

Alternatively if your email provider supports IMAP, try manually configuring an IMAP account, but beware also here as IMAP IDLE is broken is some Nokia Email clients and the phone instead is constantly sending polling data. IMAP IDLE is required for the phone to just sit there until the server tells it that there is mail available. This makes a big reduction in data traffic.

 

Nokia Messaging is intended to make all the configuration simple (ha!) and fetches the mail from all your accounts on the NM server, and then the phone just connects to NM which only sends mail when there is any (so low data whilst it's connected).
