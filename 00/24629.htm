jp ronse say:
My concern would be some situation where you returned a row that gave a total just under the 80,000 limit, say 79,995 but the rows following that one only had 1 minute in each of them, meaning you'd then need to work on down another 6 rows before getting to the true end.  I think a combination of methods will work incredibly well, and incredibly fast.  I have already turned your Sub into a Function which gets the 80000, source sheet name and column to examine passed to it, and returns the high row number as its result.

 

If I knew with certainty that your routine always returned a row number that was either =80000 or that had just caused the sub total to go over that mark, then it would be a piece of cake.

 

JL,

 

I did some extended tests with values whereby the limit is nearly reached, 79995 .... 80000, in case last steps increase the sum by 1 - and you make a point - that there was a gap in the code when the result was exactly 80000. See the updated code hereafter. I didn't handle that case. (I tried also some "0" as next values and now it returns the correct values.

 

lngTmpLow returns always the latest value before the sum was above 80000. To catch equal to 80000, see the bold part in the code.

 

By this lngTmpLow will always be row just below 80000, no matter if the next increase is greater or equal then 80000.

 

To make lngSum equal to 80000, you have to take "credits" from lngTmpLow+1 and decrease the "credits" there.

 

There is another exceptional condition. Let say, the routine comes out on 80000. Let say, the range upto lngTmpLow returns 79990 as sum and lngTmpLow + 1 contains 10. Taking the "credits" has to put a zero in that cell. What to do with it? I would keep it and update to zero.

 

Talking about the 80's, think we must have about the same age. My first programming disasters were on a TRS 80, do you remember? And Fortran and ...

 

Wkr,

 

JP

 

Sub test()
   Dim sngSum As Single
   Dim lngRows As Long
   Dim intCount As Integer
   Dim lngTmpLow As Long, lngTmpHigh  As Long, lngTmp As Long
   Dim intDummy As Integer
  
   lngRows = Cells(2, 1).End(xlDown).Row: lngTmpHigh = lngRows: lngTmpLow = 2: lngTmp = lngTmpHigh
   sngSum = WorksheetFunction.Sum(Range(Cells(2, 1), Cells(lngRows, 1)))
  
   Do
      sngSum = WorksheetFunction.Sum(Range(Cells(2, 1), Cells(lngTmp, 1)))
     
      If sngSum > 80000 Then
         lngTmpHigh = lngTmp
         lngTmp = Int(lngTmpHigh / 2 + 0.5)
      ElseIf sngSum < 80000 Then
         lngTmpLow = lngTmp
         lngTmp = Int((lngTmpHigh + lngTmp) / 2)
      Else
         lngTmpLow = lngTmp - 1
         lngTmpHigh = lngTmpLow + 1
      End If
      intDummy = intDummy + 1
      Debug.Print sngSum, lngTmpLow, lngTmpHigh, intDummy
     
      If lngTmpHigh - lngTmpLow <= 1 Then Exit Do
   Loop
  
End Sub
