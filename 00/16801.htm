ken sheridan say:
User uses drop down, selects business unit A, once selected a report is generated that shows all information related to business unit A FROM table 1, 2, and 3.

Firstly design a report based on a query which joins the three tables and returns the data for all business units.  The query should return the relevant primary key value which identifies a business unit in addition to whatever other columns you wish to show in the report.  This could be the name of the unit as a text value, but more usually will be an autonumber numeric value which is the primary key of a BusinessUnits table.  The combo box on the dialogue form would then be set up as follows:

RowSource:     SELECT BusinessUnitID, BusinessUnit FROM BusinessUnits ORDER BY BusinessUnit;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

To open the report in print preview the following code would go in the AfterUpdate event procedure of the combo box:

Dim strCriteria As String
Dim ctrl As Control

Set ctrl = Me.ActiveControl

strCriteria = "BusinessUnitID = " & ctrl

If Not IsNull(ctrl) Then
    DoCmd.OpenReport "YourReportNameGoesHere", _
        View:= acViewPreview, _
        WhereCondition:=strCriteria
End If

If the text name of the business unit is used as the key, rather than a numeric 'surrogate key', the combo box's RowSource property would be:

SELECT BusinessUnit FROM BusinessUnits ORDER BY BusinessUnit;

and its other properties would be left as the defaults.  The expression for the OpenReport method's WhereCondition argument would be amended as follows:

strCriteria = "BusinessUnitID = """ & ctrl & """"

For more flexible means of opening a report from a dialogue form, including the ability to select one or more values in a multi-select list box you might like to take a look at MultiSelect.zip in my public databases folder at:

The second parameter can be a range, but the first can only be a single text value.
