jesper holmberg say:

Hi, 

I'm Jesper and I'm one of the developers on Skype for Windows 10. There's a technical reason why Skype doesn't have a setting for auto-starting. It's a Windows Store app, and Windows places some constraints on what apps are allowed to do. Starting themselves is one of those things. I suppose that this is to make sure that the Windows apps don't show a bunch of popup ads or fake login screens to trick people to give up their passwords. 

There is a way to make Skype auto-start though. Windows has a Task Scheduler console. Start that thing and select 'Create a basic task' in the top right corner. Pick 'When I log on' as the trigger. Let the program be explorer.exe, and set the arguments to skype: (note, include that colon, it's important).

This will do a protocol launch of Skype every time you log in. One of my machines does a lot of stuff just when I log in and I had to add a 30 second delay to the task for it to work. If you need that too, you can do that on the task properties page, by editing the trigger.

Hope this helps,

Jesper

charles kenyon say:

"nomacro" tells me that this is a MacroButton prompt. It is not supposed to run a macro; it is the forerunner of the Content Control and dates back to at least Word 97. The purpose is to provide placeholder text which is all selected to be replaced by typing. These work in any version of Word from Word 97-2016 including on Macs.

This can be confusing because it is using a MacroButton field to do something that has nothing to do with macros.
