Excel feature - Seems to be causing incorrect reproduction of cells - Microsoft Community

sdehall say:

Excel feature - Seems to be causing incorrect reproduction of cells

issue = under specific conditions Excel will increment data instead of copying data when using the auto fill

researched issue
appears to be a designed feature in Excel
Excel auto fill will attempt to extrapolate a pattern and either copy or increment data based on the pattern
informed user

 regarding the issue with the way Excel auto fills cells in a table.

Unfortunately, it doesn’t appear like there is any way to change the way Excel behaves.
It looks like Excel will always try to extrapolate a pattern when using the auto fill and there is no way to change the behavior and force it to always copy cells.  Microsoft designed the auto fill feature to look for a pattern and guess what the data should be.
The result is that under certain conditions it will copy the data and under others it will increment the data.

The only way around it is to force Excel into copying the data using one of the following methods:

When auto filling data by dragging the auto fill icon, hold ctrl while dragging the auto fill or use the right click button on the mouse and then select copy from the pop-up menu.

When auto filling using the double-click, the only way to copy the cells is to double click the auto fill icon and after Excel auto fills the data, click the auto fill icon again at the bottom of the data range and select copy from the menu.
Excel will then change all of the data that was auto filled to the copy mode instead of series.

From what I can tell, the auto fill in Excel has behaved like this as far back as Excel 2003, so it doesn’t appear that Microsoft will modify the way the auto fill works.

The issue is not that Excel extrapolates patterns when auto-filling!  I use that feature regularly.  The issue is that it extrapolates two columns, with exactly the same starting value differently. And that the difference arbitrarily depends on the cell position relative to the edge of the range being copied, and it’s adjacency to a formula or text.  This arbitrary behavior is very dangerous, and exposes our work products to risk.

    Suppress part of a TIME value in an Excel cell - Microsoft Community

    roger1475 say:

    Suppress part of a TIME value in an Excel cell

    When the cell returns the TIME value, say 10:14:22, I want to display just the hours and minutes, with the seconds rounded up or down to the minutes according to the number of seconds.  Can anyone help please.


    joeu2004 say:

    As I understand it, you want to affect just how it appears, not change the actual value.  In that case, right-click the cell(s), click Format Cells > Number (tab) > Custom (category), enter h:mm into the Type field, then click OK.

    You might prefer hh:mm if you want leading zeros, or [h]:mm if the time is calculated elapse time that might exceed 23:59, or or m/dd/yyyy hh:mm if the cell includes a date.

    Excel rounds seconds, but truncates minutes.  In other words, if the actual time is 10:14:31, it will appear to be 10:14.  But if the actual (calculated?) time is 10:14:59.6, it will appear to be 10:15.

    PS.... Beware that if the format is h:mm or hh:mm and the (calculated?) time is 23:59:59.6, it will appear to be 0:00 or 00:00.  That's the benefit of [h]:mm :  it will appear to be 24:00 .
