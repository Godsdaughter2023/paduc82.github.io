srimadhwa b say:

Hi Raiden Raccoon,

Thank you for posting your question in the Microsoft Community.

The svchost.dmp (5.93MB) and svchost-full.dmp (5.29GB) files, are the Windows dump files, which is created when a computer program fails.

Svchost.exe is a process on your computer that hosts or contains other individual services that Windows uses to perform various functions. For example Windows Defender uses a service that is hosted by a svchost.exe process.

Article for reference.

What is svchost.exe?:

When an operating system encounters a problem that forces it to shut down immediately, it is commonly called a "crash”. When a crash takes place in Windows, the operating system creates a file containing the contents of the computer's memory at the time of the crash. This file is called a "dump file”. Dump files can be analyzed by experts to help determine the cause of the crash.

You can use the Windows Disk Cleanup tool to delete dump files and error reports, reclaiming the hard drive space consumed by these files.

There can be multiple instances of svchost.exe running on your computer, with each instance containing different services. One instance of svchost.exe might host a single service for a program, and another instance might host several services related to Windows. You can use Task Manager to view which services are running under each instance of svchost.exe.

Article for reference.

How to read the small memory dump file that is created by Windows if a crash occurs: https://support.microsoft.com/kb/315263?wa=wsignin1.0

Without knowing specifically why the updates failed your best hope is an educated guess. If you see an error message or code you should post it here (verbatim.)

It's an unfortunate fact that some updates just don't install properly (or at all) on some computers, while there's no trouble at all on other computers. It's easy to say "Microsoft is stupid" but without solid technical information to back that up it doesn't mean much.

In general: Updates usually install properly when all the hardware and installed software are compatible with Windows 10, and all configuration settings are made in accordance with Microsoft's instructions. The further away your computer is from that state, the more unstable it becomes.

Here's something else, and it's got nothing to do with Windows per se. A computer can work just fine today and get killed by an update tomorrow. If any of the hardware or installed software is not being supported on Windows 10 by the hardware manufacturer or software developer it's usually just a matter of time before it fails and that can bring down the whole computer. An update makes changes to Windows, and if the hardware or installed software is not being regularly updated to keep pace with the changes to Windows, that's trouble.

To get an answer in your situation - a real answer, not something that works only for a month or two - you need to find out why your computers failed. That may be involve a visit to a professional.

Back up the registry

http://windows.microsoft.com/en-us/windows7/Back-up-the-registry

 

a.    Go to start and type regedit.exe.

b.    check the following registry key:

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\NetBT\Parameters]

 

c.    Please find if the "TransportBindName" value exists. If it does not exist, create it. If it is incorrect, change the data. The values should be like this:

                                                 i.    Name: TransportBindName
Type: REGSZ
Value: \Device\

d.    Under the same registry key, if you find the "SMBDeviceEnabled" value exists and is 0, it means Direct Hosting is disabled, (again, if it doesn't show up there, create as the above, just make sure its type is REG_DWORD). You may change its value to 1 to turn on it unless there are requirements that this should be disabled.

                                                 i.    Name: SMBDeviceEnabled
Type: REG_DWORD
Value: 1

 

Close registry editor and verify if you can successfully log into your account.
