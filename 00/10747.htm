andreas killer say:
A short test shows that the MouseMove event is fired even if the forms hasn't the focus.

We can use it to show a label with the tip text similar as the original tip text.

Create a new class module, name it "MyControlTipTextHandler" and paste in this code:

--- schnipp ---
Option Explicit

Public ControlTipText As String
Public ControlTipTextControl As MSForms.Label
Public WithEvents Label As MSForms.Label
Public WithEvents Textbox As MSForms.Textbox
Public WithEvents UserForm As MSForms.UserForm

Private Sub Label_MouseMove(ByVal Button As Integer, ByVal Shift As Integer, _
    ByVal X As Single, ByVal Y As Single)
  ShowControlTipText Label
End Sub

Private Sub Textbox_MouseMove(ByVal Button As Integer, ByVal Shift As Integer, _
    ByVal X As Single, ByVal Y As Single)
  ShowControlTipText Textbox
End Sub

Private Sub ShowControlTipText(ByVal C As MSForms.Control)
  'Enlarge the control
  ControlTipTextControl.AutoSize = False
  ControlTipTextControl.Width = 999
  'Store the tip text
  ControlTipTextControl.Caption = ControlTipText
  'Let it shrink
  ControlTipTextControl.AutoSize = True
  'Move it to the control
  ControlTipTextControl.Left = C.Left + 5
  ControlTipTextControl.Top = C.Top + C.Height + 5
  'Show it
  ControlTipTextControl.Visible = True
End Sub

Private Sub UserForm_MouseMove(ByVal Button As Integer, ByVal Shift As Integer, _
    ByVal X As Single, ByVal Y As Single)
  'Hide the tip text
  ControlTipTextControl.Visible = False
End Sub
--- schnapp ---

Then add this code in your form, resp. combine the code with your existing:

--- schnipp ---
Option Explicit

Dim MyControls As New Collection
Dim MyControlTipText As MSForms.Label

Private Sub UserForm_Initialize()
  Dim C As Control
  Dim H As MyControlTipTextHandler
 
  'Add a label that shows the tip text
  Set MyControlTipText = Me.Controls.Add("Forms.Label.1", "MyControlTipText", False)
  MyControlTipText.BackColor = &H80000018
  MyControlTipText.SpecialEffect = fmSpecialEffectRaised
 
  'Create a handler for the userform (that hides the tip text control)
  Set H = New MyControlTipTextHandler
  Set H.UserForm = Me
  Set H.ControlTipTextControl = MyControlTipText
  'Keep it alive in the collection
  MyControls.Add H
 
  'Visit each control in this form
  For Each C In Me.Controls
    If C.ControlTipText <> "" Then
      'Create a handler for the control
      Set H = New MyControlTipTextHandler
      Set H.ControlTipTextControl = MyControlTipText
      'Move the tip text into the handler
      H.ControlTipText = C.ControlTipText
      C.ControlTipText = ""
      'Use the right event control
      Select Case TypeName(C)
        Case "Label"
          Set H.Label = C
        Case "TextBox"
          Set H.Textbox = C
      End Select
      'Keep it alive in the collection
      MyControls.Add H
    End If
Skip:
  Next
End Sub
--- schnapp ---

That is the basic for labels and text boxes, IMHO it should not be difficult to write the very similar code for the other controls.

Question about VBA ReDim Preserve - Microsoft Community

keith171 say:

Question about VBA ReDim Preserve

in VBA I have a 4 by 4 array
defined as...

a()
ReDim a(0 to 3, 0 to 3)
The array is fully functional with no errors (although there may be a typo in this note)


When I try either of the following....

  ReDim Preserve a(2,2)  or
  ReDim Preserve a(5,5)

I receive an out of range error.  

What am I doing wrong with the Preserve statement?



jeeped say:
Only the upper boundary of the latter array dimension can be changed when you are applying the optional Preserve parameter.

From ReDim Statement (Visual Basic)

  • Resizing with Preserve. If you use Preserve, you can resize only the last dimension of the array, and for every other dimension you must specify the same bound it already has in the existing array.

    For example, if your array has only one dimension, you can resize that dimension and still preserve all the contents of the array, because you are changing the last and only dimension. However, if your array has two or more dimensions, you can change the size of only the last dimension if you use Preserve.


hansv mvp say:

When you specify the keyword Reserve, you can only change the last dimension. So the following will be valid:


ReDim Preserve a(0 To 3, 0 To 2)


or


ReDim Preserve a(0 To 3, 0 To 5)


But the following will cause an error:


ReDim Preserve a(0 To 2, 0 To 3)


or


ReDim Preserve a(0 To 5, 0 To 3)


or


ReDim Preserve a(0 To 4, 0 To 4)


since you try to change the first dimension in each of these.
