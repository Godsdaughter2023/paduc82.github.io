This error occurs because the Explorer.exe process accesses memory that has already been freed during the shutdown process.

Normally, this issue occurs when you right-click the Start button or press the Windows key+X keyboard shortcut, and then you use the Shut down or sign out option to shut down or restart Windows.

If that is the case, the next time you want to restart or shut down the computer, left click the start button and click the power button.

Thank you for writing to Microsoft Community Forums.

Are you getting any error message when you try to access any folder on your PC?

I suggest you to follow the steps mentioned below to take ownership using the following method and check if that helps.

  1. Type CMD in the Windows search bar.

  2. Right click and Run as administrator.

  3. At the command prompt, type “takeown /f <foldername> /r /d y” without quotes and then press ENTER. (Replace <foldername> with the path of the folder with quotes).

  4. Then, type “icacls <foldername> /grant administrators:F /T” without quotes and hit enter. (Replace <foldername> with the path of the folder with quotes).

If the issue still persists, create a new user account and check if the issue is specific to user account. Refer the following article for steps to create a
