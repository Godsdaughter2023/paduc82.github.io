suzanne s. barnhill say:

I assume you are aware that displaying a document in Final view does not remove the markup, just hides it. To remove the markup, you need to Accept All Changes in Document and Delete All Comments in Document.

 

If you want to preserve the markup and still open in Final view rather than Final Showing Markup, then you can try this:

 

  1. Go to File | Options | Trust Center.
  2. Click Trust Center Settings.
  3. Select the Privacy Options tab.
  4. Clear the check box for "Make hidden markup visible when opening or saving."

This may or may not do what you want. Just be aware of the liability that you may inadvertently send out a document containing markup you don't want others to see.

If I may expand on Dale's response a bit.

The reason you can do it with Excel and Word and not with Project is that Excel and Word allow multiple instances of the application to be opened at the same time whereas Project only allows one instance. Why? I don't know, but that's how it has always been.

As far as comparing two projects, besides using Dale's suggestion, if the two projects are genetically related (i.e. one is generated from the other), you can use Project's Compare Projects utility which is found under, Project > Reports group > Compare Projects icon.
