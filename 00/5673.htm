greginmich say:

Many of us are seeing these fake Adobe Flash Player updates on the MSN news portal, and it looks like they're being piped in via a compromised advertizing domain – but that wouldn’t explain what you’re seeing there unless there’s an advertising channel attached to the site that you’re visiting when the popup hits. In my case, all of the Trojan:JS/Flafisi.D detections have been on MSN news pages and the correlation is very obvious:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/im-seeing-trojanjsflafisid-detections-and-tech/8fbe8eaf-1af0-4e76-9ab0-57828f631a5f

These recurring detections are easy to explain if we assume that the attack vector is a compromised advertising domain, because a compromised domain gets another chance to deliver a malware-site redirect every time it gets reconnected to the host site, and we can thus explain the issue without postulating the presence of any persistent malware.

And in fact many of us have been able to manage this issue by just installing an ad-blocker extension like uBlock Origin, which selectively prevents any suspect advertising domains from connecting to the host site. Although uBlock Origin includes some granular script-blocking features, I’ve been able to manage the issue with just the default domain-blocking functionality. Ad-blocker extensions for Edge are available directly from the Edge Settings menu:

Settings and more > Extensions > Get Extensions from the Store

https://github.com/gorhill/uBlock

But of course if the fake update pop-ups that you’re seeing don’t fit this “malvertising model”, then I’m not sure that an ad-blocker will actually resolve the issue.

How do I insert a paragraph right a after a table by VBA? - Microsoft Community

fritesmodern say:

How do I insert a paragraph right a after a table by VBA?

How do I insert a paragraph right a after a table by VBA? Thanks
jay freedman say:

The old "imitate a typist" style of programming would do something like this:

Sub x1()

    ActiveDocument.Tables(1).Select

    With Selection

        .MoveDown unit:=wdLine, Count:=1

        .Text = vbCr

        .Style = ActiveDocument.Styles("Normal")

    End With

End Sub

The table may not be the first one in the document, in which case you'd have to replace Tables(1) with something that will identify the correct table. The setting of the style may or may not be necessary -- the new paragraph will initially take the same style as the paragraph that originally followed the table -- and you may want it to be in a style other than Normal.

A better programming method is to declare a Range object and use it instead of the Selection object. That way, the user doesn't lose the original cursor position, and the screen doesn't have to redraw.

Sub x2()

    Dim rg As Range

    Set rg = ActiveDocument.Tables(1).Range

    With rg

        .Collapse wdCollapseEnd

        .Text = vbCr

        .Style = ActiveDocument.Styles("Normal")

    End With

End Sub

twyla naythias fox say:

A fair number of people have reported corrupted/incomplete downloads of the ISO(s), so this is the first thing I would check.  The downloads page lists both the correct size and the hash/checksum for each ISO - if either doesn't match, download a fresh copy using a different filename.  (Some have argued that the browser you use has an effect, but this is not true - corrupted downloads can happen regardless of browser.)

Once you're certain the ISO is correct and intact, try burning another disc.  No media is foolproof and write errors can occur even on the best systems.

Another possibility is that this system's optical drive may need cleaning or may otherwise be defective.  Try using Explorer (or a third-party utility such as Rufus) to create a bootable USB.  (Most of the ISOs will fit on a 4GB USB drive.)

Two other things to note:

One, double-check that your system is sufficient to run Windows 10:

  • 1 GHz (or faster) CPU
  • 1 GB RAM (for 32-bit) / 2 GB RAM (for 64-bit)
  • 16 GB (or more) of space on system drive
  • Graphics system capable of using DirectX 9
  • Display resolution of 1024x768 (or greater)

Two, even if your system meets all the above requirements, some older CPUs and hardware aren't supported by the Windows 10 Technical Preview.

If none of these factors addresses your issue, it is likely that the Technical Preview doesn't support this particular system.

nathan li msft say:

Hi dzulkhairuddindzulkefli,

On iPad, you pull down the Notebooks list (or the Recent Notes list) to force a sync of all open notebooks. Alternately, you can pull down the section list or page list to force a sync of the current notebook.

Another way to solve this issue is to create a new notebook that could sync, the long press on the section tab, and tap the move/copy icon in the toolbar to copy the section to the new notebook.

If the above steps doesn't work, you may also go to OneDrive.com to open the notebook in OneNote Online, and check the contents of the notebook is already synced to the cloud. If not, you may copy the new contents to the notebook in OneNote Online; if the notebook is already fully synced, you may close the notebook in the OneNote application then reopen it.

Please let us know if there is any updates.
