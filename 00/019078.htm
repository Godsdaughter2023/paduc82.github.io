julie fka profjulie say:
This is a limitation with some POP3 email accounts.  Here are a few workarounds to consider:
  1. If you can change the accounts to IMAP accounts (some email services allow you to use either POP3 or IMAP services), then the deleted email will not be deleted from the server (I was able to do this with one of my POP3 email accounts).
  2. You could forward all your POP3 email to your Microsoft account and then create a rule to move those emails to a designated folder.  Then when you retrieve your email from your POP3 accounts you will still have a backup of that email in the forwarded email folder on within your Microsoft account (I do this with one of my POP3 email accounts that I have been having issues with).
  3. Rather than delete email on your phone, don't delete it. When you check email on your phone, select the "unread email" view to read just the new email (that requires one swipe to the next view).

jllatham say:

You are correct to look at macros - you cannot hide/unhide rows/columns based on the contents of cells on a sheet directly.  Normally we would use a Worksheet_Change() event handler to deal with this and when you typed a new value into F3, we would see what to do.

BUT - I am betting that F3 has a formula in it because if it did not, then once you put a 1 into F3 and it becomes hidden, there's no way to type a 2 into it because it's hidden!

So we need to see the formula that is in F3 right now.  And we need to know which cell that is used in the formula is one where you actually type entries into.

Also you've used "etc" in your description of what to hide and unhide.  I assume that if we give you an example of code showing what to do with 1 or 2 columns when F3 = 1 and F3 = 2 that you'd be able to add more to those sections to cover the "etc" ?

Here is 'generic' code that hopefully you can adapt and use, or at least that will provide us a starting point.  As I said, we need to watch for changes in some cell that the user makes entries into that would affect the result of the F3 cell.  That cell might or might not be on the same sheet with the F3 cell.  The code below needs to go into the worksheet's code of that "other" cell; and it is alright if they happen to both be on the same sheet.  All you need to do is change the 2 Const values that are bold in the code below.

To put it into a worksheet's code module, choose that worksheet and right-click on its name tab and choose [View Code] from the list that appears.  Copy the code, paste it into the module, make any needed edits and press [Alt]+[Q] to close the code editor.  Save the workbook as a macro enabled workbook, type .xlsm or .xlsb.

Private Sub Worksheet_Change(ByVal Target As Range)
  'watchCellAddress needs to be the address of a cell on
  'this sheet that the user makes entries into and that
  'affects the results of the formula in cell F3 on the sheet
  'This code needs to be associated with the worksheet that
  'this cell is on even if it is not on the same sheet
  'as F3 and the columns to be hidden/made visible
  Const watchCellAddress = "C3" ' change as required
 
  'these all deal with the sheet with the F3 cell on it
  'this can be the same sheet that the watchCellAddress
  'is on or it can be a different one if F3 is on
  'a different one
  Const F3SheetName = "Sheet1" ' change as required
  Const testCellAddress = "F3"
  Dim TheF3Sheet As Worksheet
  Dim testValue As Variant
 
  If Not Application.Intersect(Target, Range(watchCellAddress)) _
   Is Nothing Then
    'our watchCellAddress value was changed by the user
    'get prepared to make things happen
    'set up to work with the sheet that F3 is on
    Set TheF3Sheet = ThisWorkbook.Worksheets(F3SheetName)
    'get the value in F3 on that sheet
    testValue = TheF3Sheet.Range(testCellAddress)
    Select Case testValue
      Case Is = 1
        'what to do when F3 = 1
        'we can reference any cell on any row in a column
        'to get the job done: row doesn't matter, column letter does.
        TheF3Sheet.Range("F1").EntireColumn.Hidden = True
        TheF3Sheet.Range("AB1").EntireColumn.Hidden = True
        'and here is how you would unhide a column
        TheF3Sheet.Range("C1").EntireColumn.Hidden = False
       
      Case Is = 2
        'what to do when F3 = 2
        'hide column X
        TheF3Sheet.Range("X1").EntireColumn.Hidden = True
        'unhide column F
        TheF3Sheet.Range("F1").EntireColumn.Hidden = False
     
      Case Else
        'what to do when F3 doesn't = 1 or 2
        'we do not care, don't do anything!
       
    End Select
    'release the F3Sheet resource back to the system
    Set TheF3Sheet = Nothing
  End If
End Sub

So if you get this error I did the following...then made sure to wait till my changes were replicated to all AD servers (most importantly the primary AD server that our Exchange Active Sync talks to).

5. Select the "Security" tab

7. Checked the "Include inheritable permissions from this object's parent" box

8. Clicked "OK" to close the "Advanced" box

9. Clicked "OK" to close the "[USER] Properties" box

10. Closed ADUC

11. Wait 

12. Re-sync phone (At this point I had just looked at phone and had already synced)

13. win

Good Morning Sven

Currently with the Office 365 Home edition, you only get 1TB of free storage, and there is no way to purchase any more. Each  users only gets 1TB of storage and no way to upgrade that (at this time)

I do not know what the future holds, but as of right now, the answer is no.
