martinbannister say:

I know this is a bit old but in case anyone else comes across it; in Office 2013 at least the File MRU and Place MRU keys have moved to a User MRU key:

HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Excel\User MRU\

This is followed by a unique key for the user under which the File & Place MRU keys can be found.  I don't have an office 2016 install to check but I suspect that this will have continued.

In a one off instance it's a simple case of editing the Reg file you exported in notepad and changing the references to the Excel\File MRU to match the 2016 location.  

I would suggest exporting the whole User MRU key from your Office 2016 install and using that to update the original export.

cividan. say:

Thanks, this is working for office 2016.

To export them from office 2013 to office 2016.

1- open word/excel at least once and open a file (This is to create the MRU)

2- export the current registry value for the user MRU for office 2016:

HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Word\User MRU\%RandomValueHere%\File MRU

HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Word\User MRU\%RandomValueHere%\Place MRU

3- export the old one from office 2013:

HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Word\User MRU\%RandomValueHere%\File MRU

HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Word\User MRU\%RandomValueHere%\Place MRU

4- replace the item list of the new one with those from the old one (The actual MRU list start with "Item 1"="xxxxxxxxxx up to "Item 50"="yyyyyy)

This step is important because the random value change with different installation.

5- import back into the registry the new modified file that contain all your MRU item.

convert from word to access. - Microsoft Community

danabug1029 say:

convert from word to access.

Someone sent me a word label document and I want to convert the labels to an access table.  Is there any easy way to do this?
patrick rossiter say: I will assume that the labels are in a multi column table in Word.

I will further assume that not all labels have exactly the same number of lines

Start by adding additional blank lines to gain a common format for all addresses. So take the longest as your guide and fill others 

OR

If you have only a few addresses with excessive lines, reduce to fewer by omitting redundant lines.

In turn, highlight each column starting with the rightmost column, cut the column, jump to end of document and paste each column to the end for the first column.  You are trying to convert the multiple column labels to a single column document.

Once you have the single column document, select all, then on the Layout Ribbon, convert to text, check the commas button. 
Add some labels to the top line of your file to help distinguish each of the fields, e.g. House, Street, Town, County etc

You now have a CSV
With a comma separated values file (CSV) you can save the document as other formats, select a plain text file and change the .txt file extension to .csv.

You are now ready to  import the file straight into a database table
