chrish.bananas say:

Hi there,

I feel your pain. I was working on my thesis proposal last night for HOURS and suddenly Word decides to crash. This is a fix for people who; 1) Did not "Save as...", 2) Did not have AutoRecovery/Save features turned on, 3) Did not have a recent Time Machine Backup OR SnapShots. This is also a fix for people who remember what time their Word application crashed.

I exhausted all forms of recovery (looking in library folders under AutoRecovery, looking under "Open Recent...", ect.). I finally found something interesting.

Follow this path:
- "Users",
- "<Your Account>" (In this case, it was my name)
- "Library"
- "Containers"
- "com.microsoft.Word".... by this point you're probably saying to yourself, "I've done this already..." but keep up with me here......next, go to...
- "Data"
- "Library"

- "Application Support"
- "Microsoft" and finally.....

- "Temp"....

This is it. This is where my mind was blown and I figured out the meaning of life and existence. This is also where I found my .tmp (temporary) word document file that I thought I had lost. All you need to do is look for the most recent file before the crash. At this point there a few things you should do. First, if you find a .tmp file that resembles a recognizable time-stamp (for me, I saw that a .tmp file was modified at 2:56 am which was right before word crashed, so this signaled to me that this is probably the file I was looking for), make a copy of it on your desktop. Next, open that copied document in Text Edit. Now this is part is bittersweet. At this point, the good news is that the file you've opened SHOULD contain all of your changes up until the application crashed, the bad news is that this .tmp file records every single keystroke in ORDER, meaning your file will look like a Jackson Pollock painting (if you don't know what his art looked like, look at it and you will understand) except with letters. In essence, Sentences will be out of order. Words will be cut off. Portions of sentences will be out of place as well.

This is, however, as best as I have been able to recover my file and the only method that has worked for me. It has been difficult to decipher, but I have my work back. It will save crucial time and energy that you already put into the document when you typed it to begin with.

I'm hoping this will work for you. Please let me know and Good luck!

- Chris

don cuthbert say:

Was your last attempt at upgrading done by running setup.exe from the 14393 public build ISO?  That is the method that is recommended.

I searched for your exact upgrade error 0xc1900101 - 0x20004 and found this:

http://answers.microsoft.com/en-us/windows/forum/windows_10-windows_install/windows-10-installation-error-0xc1900101-0x20004/e0b26cca-23e7-4e7e-9660-5a00aba9a747 

In the end, one user with this error upgraded his BIOS from the computer manufacturer (HP Pavilion G6-2244ca) and then his upgrade succeeded.  So check on that first to see if you have the latest BIOS version.

Here is another one that goes back over a year ago.  Hard to say what worked, as he changed many things, like disabling some of his SATA controllers in BIOS, all other peripherals that he could that way, but in the end, he did say that he disabled his network adapters too.  The original poster said that it worked after "resetting my BIOS" but he never explained what he meant by that.  Maybe he just hit the key to return BIOS to Factory Defaults, which is normally an option.

http://answers.microsoft.com/en-us/windows/forum/windows_10-windows_install/solved-windows-10-upgrade-error-0xc1900101-0x20004/749299b3-2520-41cc-8849-7e008a7d60ec 

Here is another one with this exact error, where it is claimed that disabling network adapters was the answer:

https://social.technet.microsoft.com/Forums/en-US/a582dc91-70e7-411a-acba-1cfe8c64b960/0xc1900101-0x20004-when-upgrading-to-10159-or-10162?forum=WinPreview2014Setup 

So if you have the option in BIOS settings to disable all network adapters (as you say you have already done in Windows) you could try that also.  After the upgrade is complete, you can re-enable all of these things (including the network adapters) so that you can download the Cumulative updates that have been released to date.  If network adapters are enabled, you have the option to get updates during the upgrade, but it can be done later once the upgrade has succeeded, especially since they may be causing the issue.

I did have the problem with my network adapters and Windows 10 Insider Builds and had to disable them in order to upgrade, but that was long ago, before the first Windows 10 was released to the public.  For me at that time, it dramatically sped up the upgrade to new builds also.

EDIT: Here is one more article about this error that was linked in one of the others above, I think.

http://borncity.com/win/2015/07/31/windows-10-upgrade-error-0xc1900101-0x20004/

Changing your Skype ID name - Microsoft Community

jobibby say:

Changing your Skype ID name

Hi, can you rename your skype ID name or do you have to delete and redo?

susan_s. say:

Hello

Please be informed that we do not have the option to change/customize Skype Name. What we can only modify is your display name in Skype (the name that is displayed on your contact’s end) or creating a new account in Skype.

To change display name in Skype:

  1. Sign in to Skype on the website.
  2. On the left pane, click on Edit profile.
  3. Edit your First Name and/or Last Name.
  4. Click on Save.

Note: Make sure to log out from the Skype app before changing your display name on the website. This is to sync the changes in your profile.

You can also edit your display name in Skype on the client, however, if you're using Skype for Windows 10 (Skype UWP) v12.xx, the option to edit your display name is not possible.

Let us know if you need further assistance.

derekmclean say:

I've been trying lots of suggestions that have been posted on other forums, and I've found one that works for me.

 

Go to Control Panel > Network and Internet > Internet Options and in the box that appears, select the Advanced tab, scroll down to Security and uncheck the 'Enable Enhanced Protected Mode' option.

 

It says you have to reboot, but you should be able to test it before rebooting. It has to be worth a try.

 

By the way, it was Graham Wright in a different forum that suggested this one. I've just modified the method of accessing the Internet Options. You can also get there from IE, but of course if IE isn't working, ...!

 

Oddly, my laptop (also just upgraded to W8.1) has this box turned on but doesn't suffer the problem. But the default status in W7 seems to be that it is turned off, so I guess it's not desperately important.

 

Another oddity is that the same PC has had W8.1 Preview in place a for a few months without this problem appearing. Suddenly, after installing the 'proper' version, I get this problem. Thanks to Graham for offering me the trick that worked. But should we have to find these things out in this way?
