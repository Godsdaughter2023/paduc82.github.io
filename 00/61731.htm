frederik long say:

I tried System Restore first; it did not help.

Your report does not say if you had no restore points, or if System Restore failed to run to completion, or if System Restore was unable to undo the damage.

If you have any suitable System Restore points then you can use them to restore your registry files (those in C:\Windows\System32\config) and your tasks. Both activities require SystemRestoreExplorer. They also require considerable skills.

Restoring the registry files:

  1. Use SystemRestoreExplorer to mount a sufficiently old Restore Point.
  2. Extract the registry files. Store them in a temporary folder.
  3. Boot the machine into Windows Repair Mode.
  4. Save the current registry files on an external disk.
  5. Put the restored files into the folder c:\Windows\System32\config.

Restoring tasks:

  1. Use SystemRestoreExplorer to mount a sufficiently old Restore Point.
  2. Extract the missing task files. They reside in \Windows\System32\Tasks. Store them in a temporary folder.
  3. Rename the files by giving them an .xml extension.
  4. Import them into the Task Scheduler

I do not know if the above methods will work, because I never had any need to try them myself. When I modify system settings (e.g the registry, or the tasks library) then I always make sure that I have a tried and proven backout path. In the case of tasks this would mean that I deactivate them instead of deleting them.
