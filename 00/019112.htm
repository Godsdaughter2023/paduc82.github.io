john mcghie say:

Hi Scott:

Rich is correct: it can't be done.  It's a "feature": the message bar will always appear, by design.

The reason is that so many people ran into trouble with this: Most would work on the document in their email, and then wonder why they "lost their changes" when they re-opened the document.

Others would get hammered by whichever Internet Nasty was doing the rounds, since their antivirus solution had not had a chance to look at the attachment since they did not save it.

To prevent both scenarios, Microsoft changed Word so that telltale bar always appears. It's the same as Tracked Changes: if a document has any, they will always be forced visible at document open.

Sometimes, people who know better have to suffer to protect people who should know better from themselves :-)

    com.microsoft.DatabaseDaemon.plist

Outlook.com: Troubleshoot Send/Receive Mail issues via Command Prompt

Technical Level : Intermediate

Summary
These procedures are design to equip Outlook.com users with the ability to quickly diagnose and determine the root cause of NDR scenarios and troubleshoot send/receive issues.
Details

Step 1:  Use NSLOOKUP to verify the MX Record Configuration

  1. On your computer hit the “Windows Key + R” to load the Run Dialog box

  2. In the Run Dialog box type “CMD” to launch Command Prompt Window

  3. In the Command Prompt window type “NSLOOKUP”

  4. (Next Line) > “set type=MX”, press Enter key

  5. (Next Line) > “My Domain.com” – Note: This is the domain of the Sender

    1. e.g. Outlook.com, press Enter key

    2. e.g. Gmail.com, press Enter key

    3. e.g. Yahoo.com, press Enter key

Notice the Non-authoritative answers

The command will provide you with the Mail Exchanger (MX) record used to message transmission

  1. Document the MX record configuration(mx1.mydomain.com)

Note:  You will need the MX record to properly use the Telnet Command to test Send/Receive Mail issues.

Step 2:  Test Send/Receive Mail issues via Telnet

  1. On your computer hit the “Windows Key + R” to activate the Run Dialog box

  2. In the Run Dialog box type “CMD” to launch Command Prompt window

  3. In the Command Prompt window type “TELNET MX1.DomainName.com 25”, then press the Enter key

Notice the 220 mx.domain.com.  This is an indication that you have connected successfully. (See Below)

Telnet smtp.xxxxx.com 25

220 xxxxx.com Microsoft ESMTP MAIL Service

  1. You will see a flashing cursor.  Please type the following:

  2. Type helo DomainName.com

Note: Successfully connectivity will display “250 MX.domain.com at your service or a (Hello) response”

  1. Type MAIL FROM:*** Email address is removed for privacy *** (press Enter key)

  2. Type RCPT TO:*** Email address is removed for privacy *** (press Enter key)

  3. Type DATA (press Enter key)

  4. Type To: *** Email address is removed for privacy *** (press Enter key) – Recipients Email Address

  5. Type From: *** Email address is removed for privacy *** (press Enter key) – Senders Email Address

  6. Type Subject: This is a test! (Press Enter key)

  7. Type Testing (Message Body)

  8. Type a “.” (Press Enter Key) -  To send the test email

Note:  You will receive a similar response 250 <Mail Server Domain> Queued mail for delivery

  1. Check the Inbox and Junk Email folder of the Recipient and verify if a test email was received successfully.

  2. If the test email was received successfully, check with the recipient(s) for any rules and/or filters enabled on their account
