This code:

What does it do?

I was not kidding saying it was dangerous. Translated into English:

As long as True is True   'Forever

You may already know messages (such as myButton.OnClick, which raises the WM_CLICK event) are at the foundation of not just Access apps but all Windows applications.

macropod say:

The following code will relocate the content of all footnotes whose references are in tables to the document body, immediately after the tables concerned, and delete the footnotes. The relocated notes will be numbered (using a SEQ field) and will occur in the same sequence as in the tables.

Sub Demo()
Application.ScreenUpdating = False
Dim Tbl As Table, i As Long, FtNt As Footnote, Rng As Range
With ActiveDocument
  For Each Tbl In .Tables
    With Tbl.Range
      For i = .Footnotes.Count To 1 Step -1
        Set FtNt = .Footnotes(i)
        Set Rng = .Characters.Last.Next
        With Rng
          .InsertBefore vbCr
          .Collapse wdCollapseStart
          .FormattedText = FtNt.Range.FormattedText
          .Collapse wdCollapseStart
          .InsertBefore "x:" & vbTab
          .Fields.Add Rng.Characters.First, wdFieldEmpty, "SEQ TblNt", False
          .InsertBefore "Note "
          .Font.Bold = True
        End With
        .Footnotes(i).Delete
      Next
    End With
  Next
  .Fields.Update
End With
Set FtNt = Nothing: Set Rng = Nothing
Application.ScreenUpdating = True
End Sub
