hansv mvp say:

That is because Access doesn't load the entire list immediately, for performance reasons. Once you have scrolled to the bottom of the list, the entire list is loaded so Access knows how to scroll it when you drag the "thumb".

You can force Access to load the entire list by adding two lines like this to the Form's On Load event procedure, or - in particular if you change the Row Source of the combo box dynamically - in the On Got Focus event of the combo box itself:

    Dim lngDummy As Long
    lngDummy = Me.NameCombo.ListCount

where NameCombo again is the name of the combo box.

hansv mvp say:

You could use the On Change event of the combo box:

Private Sub NameCombo_Change()
    Me.NameCombo.Dropdown
End Sub

where NameCombo is the name of the combo box. Alternatively, you could have the dropdown list display the moment the user clicks or tabs into the combo box, by using the On Got Focus event:

Private Sub NameCombo_GotFocus()
    Me.NameCombo.Dropdown
End Sub

Option Explicit
'Main Function
Function SpellNumber(ByVal MyNumber)
    Dim Dollars, Cents, Temp
    Dim DecimalPlace, Count
    ReDim Place(9) As String
    Place(2) = " Thousand "
    Place(3) = " Million "
    Place(4) = " Billion "
    Place(5) = " Trillion "
    ' String representation of amount.
    MyNumber = Trim(Str(MyNumber))
    ' Position of decimal place 0 if none.
    DecimalPlace = InStr(MyNumber, ".")
    ' Convert cents and set MyNumber to dollar amount.
    If DecimalPlace > 0 Then
        Cents = GetTens(Left(Mid(MyNumber, DecimalPlace + 1) & _
                  "00", 2))
        MyNumber = Trim(Left(MyNumber, DecimalPlace - 1))
    End If
    Count = 1
    Do While MyNumber <> ""
        Temp = GetHundreds(Right(MyNumber, 3))
        If Temp <> "" Then Dollars = Temp & Place(Count) & Dollars
        If Len(MyNumber) > 3 Then
            MyNumber = Left(MyNumber, Len(MyNumber) - 3)
        Else
            MyNumber = ""
        End If
        Count = Count + 1
    Loop
    Select Case Dollars
        Case ""
            Dollars = "No Dollars"
        Case "One"
            Dollars = "One Dollar"
         Case Else
            Dollars = Dollars & " Dollars"
    End Select
    Select Case Cents
        Case ""
            Cents = " and No Cents"
        Case "One"
            Cents = " and One Cent"
              Case Else
            Cents = " and " & Cents & " Cents"
    End Select
    SpellNumber = Dollars & Cents
End Function
      
' Converts a number from 100-999 into text 
Function GetHundreds(ByVal MyNumber)
    Dim Result As String
    If Val(MyNumber) = 0 Then Exit Function
    MyNumber = Right("000" & MyNumber, 3)
    ' Convert the hundreds place.
    If Mid(MyNumber, 1, 1) <> "0" Then
        Result = GetDigit(Mid(MyNumber, 1, 1)) & " Hundred "
    End If
    ' Convert the tens and ones place.
    If Mid(MyNumber, 2, 1) <> "0" Then
        Result = Result & GetTens(Mid(MyNumber, 2))
    Else
        Result = Result & GetDigit(Mid(MyNumber, 3))
    End If
    GetHundreds = Result
End Function
      
' Converts a number from 10 to 99 into text. 
Function GetTens(TensText)
    Dim Result As String
    Result = ""           ' Null out the temporary function value.
    If Val(Left(TensText, 1)) = 1 Then   ' If value between 10-19...
        Select Case Val(TensText)
            Case 10: Result = "Ten"
            Case 11: Result = "Eleven"
            Case 12: Result = "Twelve"
            Case 13: Result = "Thirteen"
            Case 14: Result = "Fourteen"
            Case 15: Result = "Fifteen"
            Case 16: Result = "Sixteen"
            Case 17: Result = "Seventeen"
            Case 18: Result = "Eighteen"
            Case 19: Result = "Nineteen"
            Case Else
        End Select
    Else                                 ' If value between 20-99...
        Select Case Val(Left(TensText, 1))
            Case 2: Result = "Twenty "
            Case 3: Result = "Thirty "
            Case 4: Result = "Forty "
            Case 5: Result = "Fifty "
            Case 6: Result = "Sixty "
            Case 7: Result = "Seventy "
            Case 8: Result = "Eighty "
            Case 9: Result = "Ninety "
            Case Else
        End Select

It's not part of Defender, it's a new version of the ClickOnce Security Checks at Launch dialog box when both the issuer of the certificate and the publisher represented by the certificate are unknown on the client machine.

What it's telling you is that the file mentioned is trying to install an application named Restream Chat, which is from the domain listed and published (created) by Restream, Inc., which is determined from it's digital signature.

If you know what the file is and trust both the location you downloaded it from as well as the organization that created it, then the notification is simply a confirmation of that information, nothing more.  Otherwise, it's meant to warn you of the information it can determine about the file and will only indicate it's malware if that's a known fact, in which case the notification box would be entirely different and from Defender as you mentioned.

diane poremsky mvp (slipstick.com) say:

Did you create a Run a Script rule for it? After adding the code, create a rule with the run a script action and select the macro.

Screenshots: http://www.slipstick.com/outlook/rules/outlooks-rules-and-alerts-run-a-script/

 

You could use the rule you used to move the messages - but instead of moving them, the rule will run the script that moves the messages and deletes the messages. If you do this, you can remove the If line that checks the subject (and the end if).

 

Use this in thisoutlooksession, - the rule identifies the messages then use the run a script action with this macro.

Sub MoveAndClearFolder(olItem As Outlook.MailItem)
Dim i As Long
Dim olOldItem As MailItem
Dim olFolder As Folder
Const strFolder As String = "Currency"
     With olItem
             Set olFolder = Session.GetDefaultFolder(olFolderInbox).Folders(strFolder)
            For Each olOldItem In olFolder.Items
                olOldItem.Delete
            Next olOldItem
            olItem.Move olFolder
     End With
End Sub
