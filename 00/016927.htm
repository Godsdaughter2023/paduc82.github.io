a. user say:

OK, I thought again about how that date could be retrieved from Word. It looks as if Word always stores the date in a standard format "internally" regardless of the date format, language, etc., but I think we can only extract that date by extracting the XML of the open document. Then we have to use the "MS XML library" to extract the first date content control.

The code is as follows. I haven't tested it much at all, and even if it functions, e.g. with a multiple Word document selection, obviously it may not do exactly what you need. Also, Word seems to get stuck with a "you have put a large amount of data on the clipboard" type message at the end.

But I think it is probably quite close. Please notice that
 a. I have assumed you are using Excel VBA
 b. I have used my own naming conventions (such as they are) for variable names
 c. you need to make the two references in Tools->References... that I have indicated in the code
 d. I'm much more familiar with Word VBA than Excel VBA. There may be better ways to work with Excel objects than the ones currently in the code.
 e. A lot of things to do with dates can be affected by the locale on your Windows system. The date comparisons seem to work OK here in the UK and I would hope it would be the same everywhere, but if necessary, a string comparison would be clearer.
 f. There could well be a problem with longer documents because the XML may be too large for MS XML to cope with. In that case, I think more would have to be done to identify the first Date content control, then extract the XML just for that control.

If you are still interested, please try to check that it does what you need with some small test documents, and identify any problems. If you can get something going, but have problems with (f), I can suggest some changes.

Sub copyPasteWordLast4Weeks()
CopyPasteWord DateAdd("ww", -4, Date), Date
End Sub
Sub CopyPasteWord(Startdate As Date, Enddate As Date)

' VBA variables
Dim datCC As Date
Dim intFile As Integer
Dim lngNextRow As Long
Dim varFiles As Variant

' Word objects
' Use Tools->References... to
' include a reference to the
' Microsoft Word Object Library
Dim objWord As Word.Application
Dim objDocument As Word.Document

' MSXML DOM objects
' Use Tools->References... to
' include a reference to the
' Microsoft XML, v6.0 library
Dim objFlatOPCDocument As MSXML2.DOMDocument60
Dim objXMLNode As MSXML2.IXMLDOMNode

' Excel objects
Dim rng As Excel.Range
Dim wb As Excel.Workbook
Dim ws As Excel.Worksheet

' Clear our work area
Set wb = Application.ActiveWorkbook
Set ws = ActiveWorkbook.ActiveSheet
ws.Rows("8:6240").ClearContents

' Get the list of Word files
varFiles = Application.GetOpenFilename("Word files (*.doc*),*.doc*", _
  Title:="Please select the files you want to copy from", _
  MultiSelect:=True)
  
' Quit if none selected
If TypeName(varFiles) = "Boolean" Then
  Exit Sub ' Cancelled
Else
  ' Start a Word instance
  Set objWord = CreateObject("Word.Application")
  objWord.Visible = True
  'objWord.DisplayAlerts = wdAlertsNone
  ' for each file...
  For intFile = LBound(varFiles) To UBound(varFiles)
    ' open the document
    Set objDocument = objWord.Documents.Open(varFiles(intFile))
    
    ' Create an XML DOM Document
    Set objFlatOPCDocument = New MSXML2.DOMDocument60
    objFlatOPCDocument.async = False
    objFlatOPCDocument.validateOnParse = True
    
    ' Load the Word Open XML text of the document
    objFlatOPCDocument.LoadXML objDocument.Range.WordOpenXML
    If Not objFlatOPCDocument.parseError Then
      ' Need to set up a namespace prefix for searching
      objFlatOPCDocument.setProperty _
        "SelectionNamespaces", _
        "xmlns:w='http://schemas.openxmlformats.org/wordprocessingml/2006/main'"
        
      ' Look for the first w:date node with a w:fullDate attribute.
      ' As far as I can see, non-date Content controls will not have a w:date node,
      ' and a date content control with no date wil not have a fullDate attribute
      Set objXMLNode = objFlatOPCDocument.SelectSingleNode("//w:date[1]/@w:fullDate")
      
      ' Only process this file if we find one of those
      If Not (objXMLNode Is Nothing) Then
        ' You should see a standard date time format no matter what the
        ' date format, language, calendar type, etc.
        ' We need the first 10 characters, which are in YYYY-MM-DD format
        'Debug.Print objXMLNode.Text
        datCC = DateValue(Left(objXMLNode.Text, 10))
        'Debug.Print datCC
        If (datCC >= Startdate) And _
           (datCC <= Enddate) Then
          objDocument.Content.Copy
          wb.Sheets("MacroSheet").Select
          ' Find the next row of data
          lngNextRow = Range("A1000000").End(xlUp).Row + 1
          Range("A" & lngNextRow).Select
          wb.Sheets("MacroSheet").PasteSpecial _
            Format:="HTML", _
            Link:=False, _
            DisplayAsIcon:=False, _
            NoHTMLFormatting:=True
        End If
      End If
      Set objXMLNode = Nothing
    End If
    Set objFlatOPCDocument = Nothing
    objDocument.Close savechanges:=wdDoNotSaveChanges
    Set objDocument = Nothing
  Next
  objWord.Quit
  Set objWord = Nothing
End If

Set ws = Nothing
Set wb = Nothing

End Sub

So 290203 would be interpreted as 3 Feb 1929; but 10203 or 010203 would be interpreted as 3 Feb 2001?

That is not the usual interpretation of 2-digit years.  The default interpretation is that years 00 through 29 are in the 2000s, and years 30 through 99 are in the 1900s.  But the dividing line can be changed in the Regional and Language Options control panel.

Did you intend to interpret 2-digit years differently?  Or did you simply misspeak?

-----

I assume that you want numeric dates with the format Custom d-m-yyyy.

For the examples that you offer, you could use the Text To Columns feature.  Select a vertical range of cells, click on Data, then Text To Data.  The key is in the last menu:  under Column Data Format, select Date and YMD.

After clicking on Finish, right-click the still-selected cells, click on Format Cells, then the Number tab, click on Custom and enter d-m-yyyy into the Type field.

However, T-to-C does not work well (IMHO) with 2-digit years less than 10.  For example, numeric 10203 or 010203 is interpreted as 3 Feb 2010 (!).  (On the other hand, text "010203" is interpreted as 3 Feb 2001.)

-----

An alternative is to use DATE, which is independent of regional conventions.  You might write the following in a parallel column:

=DATE(INT(A1/10000),MOD(INT(A1/100),100),MOD(A1,100))

Again, format as Custom d-m-yyyy.

Note the use of INT(A1/10000) instead of LEFT(A1,2).  The latter does not work if A1 contains
However, DATE interprets all 2-digit years as in the 1900s.  So 10203 or 010203 is interpreted as 3 Feb 1901.

If that is what you want, fine!

Alternatively, you might work around this DATE behavior by writing:

=DATE(INT(A1/10000)+2000*(INT(A1/10000)<30),MOD(INT(A1/100),100),MOD(A1,100))

formatted as Custom d-m-yyyy.

But that presumes only the default dividing line for 2-digit years.  It would not work if someone changes the dividing line in the Regional and Language Options control panel.

------

A final alternative depends on regional conventions for writing "short dates".  That, too, is determined by the Regional and Language Options control panel.

For example, if the regional "short date" is d-m-yyyy, you might write:

=--(MOD(A1,100) & "-" & MOD(INT(A1/100),100) & "-" & INT(A1/10000))

formatted as Custom d-m-yyyy.

An advantage of that form is that Excel will interpret 2-digit years according to the normal convention; that is, based on the dividing line between 1900s and 2000s defined by the Regional and Language Options control panel.

But again, the huge disadvantage is that the string form depends on regional conventions.  That does not bode well if you share the file with people in various countries.

Sub AutoF_DefineName_Criteria()
Const sName As String = "myList" '<< define name
On Error Resume Next
Dim ws As Worksheet
Set ws = ActiveSheet
ws.AutoFilterMode = False
Dim rng As Range, rr As Range
Set rng = ws.Range("A1").CurrentRegion
Dim n As Integer, t As Integer
n = Range(sName).Rows.Count
Dim v As Variant
ReDim v(1 To n)
t = 1
For Each rr In Range(sName)
v(t) = rr.Value
t = t + 1
Next
rng.AutoFilter Field:=1, Criteria1:=v, Operator:=xlFilterValues
End Sub
