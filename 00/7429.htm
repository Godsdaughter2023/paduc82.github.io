greginmich say:

Real-time protection is the primary defense against malware, and it’s always running, even when a scheduled or manual scan is also running. So the value of scheduled scans is debatable; but the question is really moot with Windows 8 Defender, because a daily Quick Scan is a built-in feature. That’s why there aren’t any settings for a scheduled scan in the Windows Defender interface. The new Automatic Maintenance feature triggers a daily Windows Defender Quick Scan, and for the vast majority of users that’s entirely adequate. The built-in scheduled scan only runs when the system is idle, so it shouldn’t interfere with your work at all; but if you want to reschedule it for some reason, you can do that in the Action Center with the “Change maintenance settings” control.

 

I think the Eight Forums “scheduled scan” solution that’s frequently recommended in this forum is totally bogus: In the first place, there’s no need to schedule a duplicate of the built-in Quick Scan; and if you schedule a Full Scan with the Eight Forums method, it also coverts the daily Automatic Maintenance scan into a Full Scan, which doesn’t make any sense at all. If you want to schedule a Full Scan in addition to the built-in Quick Scan, then follow the instructions in this thread:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02
