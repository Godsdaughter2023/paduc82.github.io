seanmckeon- [o365] say:

To close the loop:

After I figured out how to open an incident I was able to get some help from a MS Engineer.

He was able to recreate the issue with Office 365.  He confirmed that ISO-8859-1 did not work with Office 365 but did work with on-premise servers.  So the issue is confirmed.

My app is using HttpClient to connect to Exchange Web Services (SoapUI is also using HttpClient).

It looks like I need to specify UTF-8 when creating the post body (which I was not doing).

I was specifying UTF-8 as the "Content-Type" header, but that was not enough.

When you set the post entity you need to make sure it's in UTF-8 format, otherwise special characters will show up as question marks "?" or the sync may fail due to invalid schema.

Before:

HttpClient client = builder.build();

URL url = new URL(postUrl);

HttpPost post = new HttpPost(url.toURI());

post.setEntity(new StringEntity(postBody));

...

After:

HttpClient client = builder.build();

URL url = new URL(postUrl);

HttpPost post = new HttpPost(url.toURI());

post.setEntity(new StringEntity(postBody, "UTF-8"));

...

After making the change I am able to use a ü character.  I verified that it appears in OWA just fine.

So it looks like I have my solution.  I can use UTF-8 instead of ISO-8859-1.  The ISO-8859-1 issue remains for Office 365, but I am no longer impacted.  Thanks for your help!

I can't see a way without code but I'm sure someone will know just in case:-

If you have a table selected this code would give you the color of the top border in the first cell

Sub table_bord()
Dim otbl As Table
Dim lngcol As Long
Dim myRED As Long
Dim myGREEN As Long
Dim myBLUE As Long
If ActiveWindow.Selection.Type <> ppSelectionShapes Then Exit Sub
If ActiveWindow.Selection.ShapeRange(1).Type <> msoTable Then Exit Sub
Set otbl = ActiveWindow.Selection.ShapeRange(1).Table
lngcol = otbl.Cell(1, 1).Borders(1).ForeColor.RGB
myRED = lngcol And &HFF
myGREEN = (lngcol And &HFF00&) \ 256
myBLUE = lngcol \ 65536
MsgBox "R = " & myRED & vbCrLf _
& "G = " & myGREEN & vbCrLf _
& "B = " & myBLUE
End Sub

I Just figured it out.

I added an Extra Column with a conditional formula:

=IF(A2="Paid", C2*-1, C2))

if cells are blank you may see zero, and to avoid that, you may use the following combination:

=IF(IF(A2="Paid", C2*-1, C2))=0, "", IF(A2="Paid", C2*-1, C2)))

Finally, use this new column in the pivot for Amounts.
