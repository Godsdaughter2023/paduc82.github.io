trevor rabey say:

You are using the zig-zag progress line. Don't use it. Various reasons, mainly it is useless.

Instead, right click on the chart, gridlines, and use the status date (2nd last from the bottom), make it a vertical, red, solid line on the chart.

Make sure that everything which is actual, ie all of the actual duration days, is to the left of the status date.

Make sure that everything which is scheduled, ie all of the scheduled duration days, is to the right of the status date.

Your progress line, if you have set it to display on the status date, is way too early because it shows actual duration later that the progress line. But really the progress line position doesn't matter because you are not going to use it, but your status date is too early. Either that, or the actual start date of those tasks must be earlier, ie at least early enough so that there is no actual duration to the right of the status date.

Where did you set the status date? Best way is not on the ribbon but via project, project information.

And in file, options choose a date/time format which displays the time and make sure that the status date time is the end of that day, not the beginning (this was a bug, the default should always be the end of that day, 17:00 or whatever)

Ignore the 43 % complete in the project overview report. It is meaningless.

Hello Oldhasbeen:

MS Project does not use the status date while calculating the %Complete value at the summary level.

MS Project calculates a summary progress date but it is not available as a date value to access anywhere in a project plan. It is available graphically by using the SummaryProgress field in the Bar Styles dialog box; so you can draw a bar on the summary bars to represent the progress by using this field. If you update the schedule based on the Status Date - as Rod and Trevor suggest -, then the summary progress (and the progress on the tasks as well) will be aligned with the status date automatically. 

Updates are important as Julie always says, and I use the Project Information dialog box to input the status date to the Status date field as Julie's mentioned.

the goss family say:

Our house ran into this stupid error today, but thankfully we got it fixed.

Here's the details:

Xbox One

USA (California)

Trying to play a movie in Xbox Video

DirecTV receiver --HDMI--> Xbox --HDMI--> Smart TV 

Originally we had the HDMI cable from the Xbox box from receiver to Xbox and a custom one from Xbox to TV, and had to set HDMI on the Xbox Video settings (Instead of Auto-Detect, which kept it at 640x480).

What I changed to fix the problem:

(1) Change the Xbox's video source setting from HDMI to Auto-Detect (which for us made it go 640x480).

(2) Power off TV and Xbox. Disconnect power cord from Xbox and HMDI cable(s) from Xbox & TV.

(3) Changed connection from TV to Xbox to use the Xbox's HDMI cable it came with:

DirecTV receiver --personal HDMI cable--> Xbox --Xbox's HDMI cable--> TV

(4) Reconnect power to Xbox, power up the TV, then retry (We were successful, and back on 1080p automatically thank goodness).

I hope this helps! It's absolutely silly that Microsoft is so huge but can't even make a support page for this error, much less a technical-detail version of the error message that can actually describe the problem.
