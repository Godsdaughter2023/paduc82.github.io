hansv mvp say:

You can't remove rows and columns that you don't need, but you can hide them.

 

Select the first column that you don't need by clicking on its column header.

Press End, then Shift+right-arrow. This selects all columns up to and including the last one (XFD).

On the Home tab of the ribbon, click Format > Hide & Unhide > Hide Columns.

 

Select the first row that you don't need by clicking on its row header.

Press End, then Shift+down-arrow. This selects all rows up to and including the last one (1048576).

On the Home tab of the ribbon, click Format > Hide & Unhide > Hide Rows.

marshall barton say:

First and foremost, except for a superficial visual resemblance, a database table is nothing like a spreadsheet.

 

Second, users (including yourself when adding/editing data), should never see a table's datasheet.

 

What those statements mean is that you need an extraordinary reason to ever even think about using a calculated field in a table.  Instead, you should create a form, probably one for each table, for viewing/adding/editing data.  At this point you can use form control (text/combo box, etc) features (or maybe calculated fields in a form's record source query) to calculate and format the data convenient for humans to see.

 

The expression you posted could not work because the Sum function totals all the records.  I suspect that you should be using a form text box in the form's detail section with the expression:

 

   =[price/ac] * [tac/hr]

 

And if you want to see the total of all the records in the form's record source table/query, use a text box in the form's Header/Footer section with the expression:

 

   =sum([price/ac] * [tac/hr])

    Hi, this is how you can block this type of message:

     

    For Outlook (Hotmail) users there is a very simple solution which will be completely successful in preventing these ever reaching your inbox or junk box if they contain the word ‘Adriana’ or ‘Adriana22’

    Follow these steps:

    1 Log in to your Outlook email

    2 Click on the gear icon (a wheel shape on the upper right side of the mailbox bar).

    3 Select ‘More mail settings’

    4 Select ‘Customizing Outlook’ and click on ‘Rules for sorting new messages’.

    5 Click on ‘new’ to create a new rule.

    6 Under Step 1 – select : ‘Sender’s address’ option, then ‘contains’ and type ‘Adriana’ in the third box so the line reads: ‘Sender’s address>contains>adriana’

    7 Under Step 2 – select : ‘Delete these messages –don’t even put them in my Deleted folder’.

    8 Click on ‘Save’.

    This will entirely solve the problem and you will note that you can use this method, not just for the irritating ‘Adriana’ but for any name, word or line you wish to ban from you incoming email.
