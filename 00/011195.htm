The Calendar view for a week shows the current calendar week, depending on (as you say) your setting for the starting day of the week. This is entirely by design and what most people would expect that view to do. At the top it displays the exact date range which is currently being shown, so mine says "30 August - 05 September 2010" for the current week at the moment.

On the Home ribbon is a button to show the next 7 days, which you might prefer to use instead.

You can also select a specific range of dates in the Date Navigator on the To-Do bar (or in the Navigation pane of calendar view if you have To-Do bar turned off). To select up to eight days, just click and drag with the mouse.

If you go beyond 8 days it will switch to whole weeks by default, but you can choose a specific period up to 14 days by clicking on a single date, then hold Shift and click another date to get all the dates between and including those.

When the documents are pinned in Word, they are saved to the following location:

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Office\16.0\Word\User MRU\ ADAL_XXXX>File MRU.

The key contains some values with names "Item 1", "Item 2", etc. Each value contains a string like [F00000000] [T01CFA783D052DEC0][O00000000]*C:\test1\test.docx

The file name is the part after the asterisk. The first part is [F00000000] if the document is not pinned on the list and it is [F00000001] if it is pinned.

If you see [F00000000], this means items are not pinned, in this case I’ll suggest you pin the documents manually.

.Resize(rows, columns) is not the same as .Offset(rows, columns) in that you can .Offset() something zero rows but you cannot resize to something zero rows high.

I want to select activecell down to end of data and 2 columns to right ...


This description is a little ambiguous to me. If you mean you want to copy a range starting with the active cell down to the last cell in the active cell column and include 2 columns to the right (total three columns) then try,

    ActiveCell.Resize(ActiveCell.End(xlDown).Row - ActiveCell.Row + 1, 3).Copy
... or,
    Range(ActiveCell, Cells(Rows.Count, ActiveCell.Offset(0, 2).Column).End(xlUp)).Copy

Part of my confusion was whether you wanted a copied range two or three wide. I've made the above three wide as per your statement "select activecell down to end of data and 2 columns to right".
