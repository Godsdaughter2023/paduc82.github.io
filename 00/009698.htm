How do I unprotect the workbook in windows 8?  

HOW TO UNPROTECT A WORKBOOK

Take the following actions:-

1. Open the Workbook using the password to open it.

2. Office Button (possibly File in your version)

Save As

The:-

Save As

 - window should open.

3. In the:-

Save As

 - window in the lower right hand corner click on the drop down arrow beneath:-

Tools

 - then click on:-

General Options . . .

The:-

General Options

- window should open.

4. In the:-

General Options

 - window remove the password(s) from the following fields:-

Password to open:

Password to modify:

 - then click:-

OK

5. EXCEL should now return to the:-

Save As

 - window.

Click:-

Save

6. A:-

Confirm Save As

 - window should open.

Click:-

Yes

 - if you want to save the change.

7. Save the file (if required).

Close the file.

8. Re-open the file and you should see that a password is no longer required.

Your Workbook is now unprotected as you required.

HOW TO UNPROTECT A WORKSHEET

Take the following actions:-

1. Open the (un-protected) Workbook.

Click on the Worksheet that's protected (Sheet2 for example) then:-

2. Home tab

Cells group

Click on the drop down arrow beneath:-

Format

 - then click on:-

Unprotect Sheet . . .

The:-

Unprotect Sheet

 - window should open.

3. In the:-

Unprotect Sheet

 - window in the box called:-

Password:

 - type in the password then click on:-

OK

4. EXCEL should now return to the open Worksheet.

5. Save the file at this point if required.

Close the file at this point if required.

6. Re-open the (un-protected) file.

7. Click in (for example) Sheet2 (the Worksheet that you un-protected) and you should now find that that Worksheet is indeed un-protected.

 

Your Worksheet is now unprotected as you required.

john w. vinson say:

Ok, that's simpler. Your earlier examples seemed to be involving another table. 

To replace ALL [Date Received] records with a CBA code, in the entire table, just run an UPDATE query. Create a new query based on the table; select only the [Date Received] and [Code] fields.

Change the query to an UPDATE query using the query type tool on the ribbon.

Put 

"CBA" 

on the Criteria line under Code, and type

[Date Create Order]

on the Update To line under [Date Received]. Be sure to include the brackets, otherwise it will try to update to the text string "Date create order" - which is not what you want, and will give a type conversion error anyway!

Note that this will overwrite anything in the [Date Received] field for every single CBA record in the table, even if the existing data is correct. You may need some other criteria (such as a date or date range on the Date Create Order field.
