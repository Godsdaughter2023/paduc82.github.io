    frederik long say:

    It is rare that a poster speaks so frankly about his error . . .

    Windows Backup maintains a catalog of the zip files and what is inside each of them. To do a proper job one would need to know the exact structure of the catalog (which I don't). To make matters worse, the catalog is a hexadecimal file which makes it hard to guess what the designer had in mind.

    However, there could be an easier option. Backup files are numbered sequentially like so:

    Backup files 1.zip
    Backup files 2.zip
    Backup files 3.zip
    Backup files 4.zip

    It is not unreasonable to assume that file 1 was created first and file 4 was created last. You could therefore rename the zip files so that the number sequence agrees with the file time sequence. Windows Explorer lets you sort file listings in order of date.

    There is trap here. Windows Explorer displays the date stamp only down to minutes. On my machine several .zip files were created within the same minute. You therefore need a resolution of seconds instead of minutes. If you are unable to find an explorer setting that gives you seconds then you must use a console method. Here it is briefly. Post again if you need more details.

    1. Save these lines as E:\GetDate.vbs (E: is my backup drive letter).
      Set oFSO = CreateObject("Scripting.Filesystemobject")
      Set oFolder = oFSO.GetFolder(WScript.Arguments(0))
      For Each oFile In oFolder.Files
          WScript.Echo oFile.Name, oFile.DateCreated
      next
    2. Open a Command Prompt, then get into this folder:
      E:\WorkPC\Backup Set 2015-01-06 000832
    3. Type this command:
      cscript.exe  E:\GetDate.vbs
      then press Space bar.
    4. Press the Tab key until "Auto-Complete" gives you the desired backup folder name. Now press Enter. You will get a listing like so:
      Microsoft (R) Windows Script Host Version 5.8
      Copyright (C) Microsoft Corporation. All rights reserved.
      Backup files 1.zip 06.01.2015 00:11:38
      Backup files 2.zip 06.01.2015 00:12:16
      Backup files 3.zip 06.01.2015 00:12:53
      Backup files 4.zip 06.01.2015 00:12:57

    Since you now get the file times down to seconds, you can give the zip files appropriate sequential numbers.

    Good luck!
