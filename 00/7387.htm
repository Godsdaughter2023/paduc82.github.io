Notice before anyone's Hackles go up. This is the normal way I, (me/myself) does installs of new Software.

  1. Make absolutely sure you have a good copy downloaded.
  2. Now once item is downloaded quit all applications.
  3. Next restart computer holding shift key down (safe mode).
  4. Continue holding down shift until progress bar get almost half way through.
  5. Release shift key and wait until a sign in screen appears.
    supply username (if necessary) and password
  6. Allow to finish loading.
  7. Locate the installer .pkg (package).
  8. Start install.

At this point if you come up with same messages, you have a bad download. (Because all the round-robin switching from one connection to another I would not be surprised. Took me 5 hours on a 60 MB cable connection – from 9pm to 2am –  to get the install a couple of nights ago for me.)

So re-download and try again. Once install is complete:

  1. run disk utility and run repair permissions.
    After repairs are done or none are found.
  2. Next run Font Book and check for defective and duplicate fonts
    Go to this Website (click here) and follow directions, located page on: 
    FontBook and OXS 10  Lion and above
    Once duplicates are resolved then quit Font Book.

would someone explain in plain english what is a dump file?

I have 300 mgb of "dump files" do I need to delete these?  In plain English please, I am not that fluent...I have Windows 7.
jay freedman say:

As Doug said, a dump file is a record of everything that was in the computer's memory when a crash occurred, and it's meant to help diagnose what went wrong. Each file represents a different crash.

If you are not planning to use the dump files to investigate the cause of the crashes, then they are useless and can be deleted. I would not even bother to back them up.

Interpreting a dump file is a task for a technician with adequate tools (usually Microsoft Visual Studio and a set of "symbol files" that can identify the memory addresses listed in the dump file). It isn't something that just anyone can do. If you want to hire someone to examine one or a few of the files and try to fix the computer, save only the latest one(s) on a writeable DVD.

The activation service uses multiple factors to determine whether or not a license is active.  Among those factors are time since last activation. software on the PC, and hardware on the PC.  If you change the motherboard, for example, the activation service considers that you are using a different PC.  Changing the hard drive has at times (but not always) also affected the activation service.  Perhaps changing graphics cards is also one of the hardware factors, although I don't know if it is.  Microsoft doesn't publish the factors.  All they say is "Some changes to your computer
components or the software may require you to reactivate the software."
