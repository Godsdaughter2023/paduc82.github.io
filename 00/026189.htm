Month function not working - Microsoft Community

ordnance1 say:

Month function not working

Below is the SQL for my query but it returns no records. I checked the table CalDueDate is formatted as Date/Time.  If I remove Month(Now()) all records display. As a test I also tried Year(Now()) and no records displayed.

 

SELECT tblCalibrationData.CalDueDate
FROM tblCalibrationData
WHERE (((tblCalibrationData.CalDueDate)=Month(Now())));

 

john w. vinson say:

The Month() function returns an integer number between 1 and 12; a Date/Time value (like CalDueDate) is a double float number, a count of days since midnight, December 30, 1899.

The current Month(Now()) is 4, so this would match a duedate of January 2, 1900... which is probably a bit out of date!

Similarly, the Year() function returns an integer year number, such as 2012. This is not equal to a date that you care about, but rather a date back in 1905.

I'd use

WHERE CalDueDate >= DateSerial(Year(Date()), Month(Date()), 1) AND CalDueDate < DateSerial(Year(Date()), Month(Date()) + 1, 1)

to get the RANGE OF DATES (rather than one specific date) during the current month. You can also use

Month([CalDueDate]) = Month(Date())

but that's less than ideal for two reasons: one is that it will match any CalDueDate in April, whether that's April 2012, April 2016 or April 1938; and second, it will not use any index on CalDueDate.

Note also that Now() does not return today's date. It returns the current date and time, accurate to the second; #04/04/2012 4:29:18pm# is not in fact equal to April 4, 2012, it's several hours later.

Possible to protect whole sheet but still allow users to add rows and - Microsoft Community

a. user say:

Possible to protect whole sheet but still allow users to add rows and edit them?

Is it possible to protect worksheet in a way that users can still add rows and edit rows added by them in current session but cannot edit existing rows?

ossiemac say:

"Are macros only way to achieve this?"

I don't think there is any other way of achieving this but it can be done with VBA as per the following guidelines and VBA code. Feel free to get back to me if any problems

Important: Ensure that you keep a record of any passwords used because we cannot help you if you forget them (It is against the forum policy to assist with this).

  • Ensure you always have a backup of your workbook in case something goes wrong.
  • Select a range of cells in a single column (Suggest column A) which extends down past any possible number of rows that you are likely to use. (Don't select the entire column or it won't work).
  • On Formulas ribbon select Define a name and name the selected range RowCount.
  • Right click the worksheet tab name
  • Select "View code" (will open the VBA editor window at worksheet's module)
  • Copy the VBA code below and paste into the VBA editor.
  • Do not rename the sub
  • Edit the password "ossie" in 2 places (Unprotect and Protect lines) to a password of your choice (if the sheet is already protected then you must use the same password.)
  • Close the VBA editor (Cross very top right of VBA editor window)
  • Save the workbook as Macro enabled.
  • Ensure macros are enabled. See help for how to do this. (Option to "Disable all

To lock the VBA Project against viewing.

  • Open the VBA Editor.
  • Right click the project name in the Project Explorer (left column)
  • Select VBA Project properties
  • Select Protection tab (at top of dialog)
  • Check box against "Lock project for viewing"
  • Enter a password of choice and then confirm password
  • Click OK


The project is not locked until after you Save and Close the workbook and re-open and then the VBA cannot be viewed.

  • To unlock the VBA Project
  • Double click the Project name in the Project Explorer (or click plus sign to expand tree)
    Enter the password.
  • Expand the Project tree by clicking each of the plus signs in the Project Explorer column.
  • After unlocking the project, the project remains unlocked until after save and close the workbook.

To remove the VBA password

  • Same as for locking but uncheck the box against "Lock project for viewing"
  • Delete the password from the fields.
  • Ensure that you do not forget the password.

Copy all of the code below into the VBA editor.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim strTargAddr As String
    Dim lngPreInsert As Long
    Dim lngPostInsert As Long
   
    On Error GoTo ReEnableEvents
    Application.EnableEvents = False
    'Following line only applies for entire row inserted
    If Target.Columns.Count = Me.Columns.Count Then         'If True then entire row
        strTargAddr = Target.Address                        'Save the Target Address (Target object lost after undo)
        lngPostInsert = Range("RowCount").Cells.Count       'Rows count after sheet change
        Application.Undo                                    'Undo the change
        lngPreInsert = Range("RowCount").Cells.Count        'Rows count before the change
        If lngPostInsert > lngPreInsert Then                'If greater than row inserted
            ActiveSheet.Unprotect Password:="ossie"
            Range(strTargAddr).Insert Shift:=xlDown         'After worksheet unprotected
            Range(strTargAddr).Locked = False               'Turn off locking on inserted row
            ActiveSheet.Protect Password:="ossie", _
                                DrawingObjects:=True, _
                                Contents:=True, _
                                Scenarios:=True, _
                                AllowInsertingRows:=True
        End If
    End If
ReEnableEvents:
    If Err.Number <> 0 Then
        MsgBox "Error occurred in Sub Worksheet_Change."
    End If
   
    Application.EnableEvents = True

End Sub
