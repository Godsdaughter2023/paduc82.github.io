IF and True/False merge field - Microsoft Community

gordonprice1967 say:

IF and True/False merge field

Is it possible to evaluate a merge field against TRUE? Of course the RESULT of an evaluation can be true, but I have a merge field that actually has a value of True or False, and I want to place some dumb text depending on which state that field is in. I will never actually show the merge field, just use it for evaluation. Basically
{ IF {HasMinutes} = TRUE "Attendees" "Invitees"}
I have tried TRUE and YES, in quotes and not. Always results in a false condition.

Any help is greatly appreciated.

Thanks!
Gordon

suzanne s. barnhill say:

You'd need this syntax:

 

     { IF { MERGEFIELD "HasMinutes" } = TRUE "Attendees" "Invitees" }

 

You shouldn't need quotes around TRUE (quotes are usually needed only when there's more than one word"), but you do need to match the case, so the field must actually be "TRUE," not "True" or "true." Both IF and MERGEFIELD fields must be inserted using Ctrl+F9 (though you can use the Insert Merge Field button to insert the merge field).

 

Sometimes it will work to reverse the condition:

 

     { IF { MERGEFIELD "HasMinutes" } <> TRUE "Invitees" "Attendees" }

a. user say:

> How can I find out what connection is used to make sure Word does not use US formatting as default?

If you have a column full of dates (or even cells with data that looks to the OLE DB provider to be dates, the provider will /probably/ decide that every value in the column is a date. In that case, you are OK, because 

 a. OLE DB will transfer the date cells as dates in US DD/MM/YYYY format, not text strings

 b. when you use date format switches in Word, Word should get it right

I say /probably/ because in this case I would have to go back and check in this particular case. There's an explanation of how OLE DB chooses the column format in many cases at http://tips.pjmsn.me.uk/t0003.htm , but that page does not yet cover all possibilities.

You can choose the connection method by checking Word->File->Options->Advanced->General->"Confirm file format conversion on open", going through the connection process, and selecting DDE, ODBC (or OLE DB) in the additional "Confirm Data Source" dialog box that then appears. If your Excel sheet is a .xslx or .xlsm, you have to check the "Show All" box before you get to see DDE or ODBC. Generally speaking it is better to choose DDE at this point, but that does mean that Excel and your workbook will open if they are not already open, and that you will need to put your data in the first sheet in the book.

To see what method Word used to open your data source, with the Mail Merge Main Document open, you can go into VBA and type the following in the Immediate Window:

? activedocument.mailmerge.datasource.connectstring

If it starts with

"Provider="

it's OLE DB (it will probably be

Provider=Microsoft.ACE.OLEDB.12.0;

or possibly

Provider=Microsoft.Jet.OLEDB.4.0;

If it starts with something like 

DSN=

(e.g. DSN=Excel files

it's ODBC.

Anything else (other than blank) and it's DDE - probably:

"Entire Spreadsheet"

or a range or range name.
