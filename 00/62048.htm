a_user_here say:

I see the article you posted is from June, which predates the release, and when Microsoft clarified things, and most of it is just wrong.

1) The Insider builds are not a path to a release license.  They are only a path to Insider builds.  Now since Microsoft didn't terminate the Insider program, you can pretty much just keep using them provided that you accept the fact the every once in a while (different for the Slow/Fast ring) you will have to download and update to the builds they push.  If you don't then the Insider build will expire.  And also note that people that got stuck on an Insider build and didn't get the 10240 build downloaded and installed in the period of time Microsoft gave for it, they might be totally locked out and have to reinstall their old Windows and upgrade to the free Windows 10 RTM to do anything including rejoining the Insider Program.

2) It has never been possible to use an old Windows version product key to activate the newer version Windows.  What has been done in the past is to issue an update key for the new version of Windows.  And those update keys would only work if the old version of Windows was installed.  You could never take for instance a Windows 7 key and install Windows 8 and then use the Windows 7 key to activate it.

3) Windows 10 update is really only different so far in the handling of update keys.  So far it doesn't have any.  Instead what they have done is record the "fingerprint" that is used for activating your Windows 10 upgrade on their servers.  And if a request for activation with the same "fingerprint" comes up again it will automatically activate.  Basically there is the requirement that the upgrade be done from the running Windows 7/8/8.1, which means that is the way they can ensure it is valid.  Then they allow the upgrade and when it finishes do the automatic activation and store of the "fingerprint".  If someone does a clean install of Windows 10 without first doing the upgrade, then it just won't activate.

4) After the upgrade the user is free to do a reinstall of Windows 10 and can skip the requests for the product key, and once they get to the point where they log in, it will automatically activate again based on the "fingerprint".

5) So for this process the ending of the free upgrade doesn't matter. But what if you change something in your hardware that makes the "fingerprint" different?  Well it isn't going to be something like your drive or graphics card.  Changing network cards or motherboards might trigger it.  If that happens  Then the next big question is what kind of license you had with your old version of Windows.  If it is an OEM license (one you got with a new machine), well as far as Microsoft is concerned, that is a new device, and your license was only valid "for the life of the device".  So you would either have to purchase a new Windows 10 license/product key or some how get your old Windows installed/restored, and then if the free upgrade is still open, upgrade to Windows 10.  If your license is "retail" as in you bought the DVD/product id.  Then it would be the same process reinstall/upgrade during the free upgrade period.  After that period Microsoft is going to have to do something, but as of yet they have not said what that will be.
