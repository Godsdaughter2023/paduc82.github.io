If you only have bound controls, you can use

Private Sub Form_Current()
    Me.AllowEdits = Me.NewRecord

Me!Part_Number.Locked = Not Me.NewRecord

within the Form_Current event procedure as you want.

COMMENTS:

In Outlook 2013, you can change your port number by doing the following:

  1. Start Outlook.
  2. Click File > Account Settings and click Account Settings again.
  3. Highlight your Comcast account and click Change.
  4. In the window that opens, click More settings…
  5. Select the Advanced tab and change the Outgoing server (SMTP) field to 465.
  6. Close the windows and try send/receive again.

passwords for multiple users in multiple workbooks - Microsoft Community

becka831 say:

passwords for multiple users in multiple workbooks

I have a series of workbooks structured exactly the same.  Each workbook will be accessible and editable by one manager and his or her reports only, except that one manager needs to add material to more than one book.  Is there a way to assign that manager a password that will allow her access to all books so she doesn't have to keep track of 6 or 7 passwords?  Everyone else is restricted from accessing any workbook other than their own business line's, and each line has their own unique password. 

grant j stephens say:

Hi Becka,

Yes, there are ways to give one manager access to all workbooks.

1. Use File System permissions to control access instead of 'password to open'

This is by far the simplest method, but does involve your local I.T. (or you, if your account has sufficient privileges).

This is assuming the files reside on a standard network share, and assumes some basic knowledge of Windows File System permissions:

- Put each workbook into a separate folder

- Set the permissions for each respective folder so that only the intended network username and the network username of the manager appear under 'Group or User Names'

(you could also consider creating a group for each folder, adding members to the group, then adding the group to 'Group or User Names')

The advantages of using the FS permissions over workbook passwords are numerous, including streamlining the process of accessing the information and taking the heartache out of password management.

2. Use Microsoft's Information Rights Management (IRM) service (Office 2010 - not sure it's available in previous versions of Office)

A great concept, IRM allows you to create permissions for content that you create and provides a way to manage the permissions of restricted content. This may be more relevant if you were expecting to publish a workbook (send via email or place on the web) as the 'permisssions' are managed by Microsoft's IRM servers in the cloud.

To access IRM (from within Excel) go to File --> Protect Workbook --> Restrict Access by People

You will need to sign in using your Microsoft account to access the service, and your computer will need to be configured to use the service (this happens automatically when you sign in for the first time). Therein lies the problem, however, in that each user who accesses your restricted content must also sign in and their computer (any computer they use to access the file - an issue in 'hot-desk' and 'BYOD' environments) will need IRM configured in order to access the content.

There's also another catch: the service is free, and Microsoft can and may at any time decide to terminate the service (as does happen from time to time - Outlook.com used to allow you to create and manage customised email addresses for your own domains for free, which has been discontinued in favour of their Office 365 'software as a service' model for which you pay a subscription).

Additionally, I am not sure if simply omitting a username from the Read field means that only users explicitly added to the field (say, your manager and the business line manager) can read the file, but this would make sense (haven't had a chance to play around with it much because nobody else in my organisation has IRM credentials). Try mucking around with it yourself...

Personally, and if your corporate version of Office doesn't support IRM, I'd go down the file system route.

-Grant
