recyclebinquestions say:

C:\$Recycle.Bin is blocked

I am trying to access C:\$Recycle.Bin to retrieve a file that I had accidentally deleted, but when I try to open 3 folders inside all it says is access is denied. I am on the main admin account so I don't know why it's blocked from any and all access. Is there any way to get into these folders?

a. user say:

To see your permissions, right click the folder/file, click on properties, and check out the security tab.  Check to see what permissions you have by clicking on your username (or user group).  Here are the types of permissions you can have: http://windows.microsoft.com/en-US/windows-vista/What-are-permissions.  You need to be an administrator or the owner to change permissions (and sometimes being an administrator or even an owner isn't enough - there are ways to block even them from access (though a smart administrator knows these ways and can get around them - but usually shouldn't because they were denied access usually for a very good reason).  Here's how to change folder permissions in Vista: http://www.online-tech-tips.com/windows-vista/set-file-folder-permissions-vista/.
 
To fix this problem with the folders, take ownership of the folders (as an administrator) and give yourself full rights.  Right click the folder, click on properties, go to the security tab and click on Advanced and then click on the Owner tab.  Click edit and then click the name of the person you want to give ownership to (you may need to add it if it isn't there - or it may be yourself). If you want it to apply to sub-folders and files within that folder/drive, then check the box for replace owner on sub-containers and objects and click OK.  Back out and now there's a new owner for the folders and files who can change permissions as needed.  You can now change them from read-only or access denied (even if the main folder says they still are read only - you can have access yourself as the owner).  You can keep them as read-only for other users and guests and even administrators (though they can take ownership themselves and gain access if they so desire and there really isn't much you can do to stop this except protect the folder with a password using a 3rd party product.  Here's more information on taking ownership of a file or folder: http://www.vistax64.com/tutorials/67717-take-ownership-file.html.

These procedures were written with normal folders in mind, but I suspect they'll work equally well in the Recycle Bin.  Frankly, I've never heard of anyone having a permissions issue in the recycle bin before - but I can see how it might be possible because some people may not want even their deleted files read by other people and so the permissions that applied to them when they were still good files might stay with them when they are sent to the Recycle Bin and Vista may have taken that into account (it's not a bad idea when you think about it).  Still, as an administrator, you should have been able to gain access (though perhaps only by taking ownership).

You'll have to do this with each of the folders unless you know which one contains your accidentally deleted file - and again, I don't understand why it put the file into folders when it should just have the file sitting separate in the recycle bin - unless you deleted an entire folder at the same time and one file wasn't supposed to be deleted but you forgot to remove it first.  But the fact that you were able to delete the files suggests you have adequate permissions to view them so the access is denied message really doesn't make any sense to me - but hopefully the above procedures will give you the access you need.

Good luck and I hope this helps!  Please post back in any case so we'll know how it turns out (and try to come up with another solution if this one doesn't work).

Do you have a genuine Vista Installation Disk?  If not, make a Recovery Diskusinghttp://neosmart.net/blog/2008/windows-vista-recovery-disc-download/ along with burning software like: http://www.snapfiles.com/get/active-isoburner.html and, of course, a blank CD (perhaps made on a different computer).

Boot to the disk. To boot to the CD you may need to change the BIOS to make the CD-drive first in the boot sequence.  To do that, wait for the screen that tells you the F key to push to access the boot menu or boot setup.  Push it quickly.
