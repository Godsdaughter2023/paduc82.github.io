rudy xu msft say:

Hi Andrew,

 

The information Karl mentioned is right.

 

Here are the requirements for SMTP client submission:

Authentication: you need to configure a user name and password to send email on the device.
Mailbox: you need a licensed Office 365 mailbox to send email from.
Transport layer security (TLS): your device needs to use TLS version 1.0 and above.
Port: port 587 (recommended) or port 25 is required and needs to be unblocked on your network. Some network firewalls or ISPs block ports—especially port 25.

 

If it is essential to use SMTP client submission and your customer only supports SSL 3.0, you can set up an alternative configuration called Indirect SMTP client submission. This uses a local SMTP relay server to connect to Office 365. Instructions can be found here: How to configure IIS for relay with Office 365.

 

To know detailed information about it, please click the link below:

How to set up a multifunction device or application to send email using Office 365

Excel 2010. Inaccuracy in calculation - Microsoft Community

dmitrymff say:

Excel 2010. Inaccuracy in calculation

Please help!
0.8049-0.7999=0.00499999999999989, and not 0.005.
Why?
hansv mvp say:

Excel stores numbers in binary format with a finite precision. Most decimal numbers cannot be represented exactly that way, so very small rounding errors occur.

You can either format the cell with the formula as a number with 3 decimal places: the underlying value will still contain the rounding error, but the display will be correct. Or you can use a formula such as

=ROUND(A1-B1,3)

to round the result to 3 decimal places.

rick rothstein, mvp - excel say:

There is a long history behind this problem (it applies to computer math in general and is not an Excel only problem). You can get started on finding out what the problem is by Googling this...

excel floating point number precision

and reading some of the articles that get listed.
