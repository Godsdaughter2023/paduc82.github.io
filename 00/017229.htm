Memo field in query - Microsoft Community

grimwadec say:

Memo field in query

I have a report based on a multi table query that includes a Memo type field. I have to use the aggregate function in the query and this limits the Memo field output to 256 characters. Please, does anyone know a way to circumvent this limitation?

djsteele, mvp say:

Unfortunately, since aggregate functions require sorting, and Access converts memo fields to 255 characters to make the sorts more manageable, it's difficult.

One approach is to create a query that aggregates everything else, and join that query to the original table to pick up the memo field. Something like:

SELECT SubQ.Field1, SubQ.Field2, Table1.MemoField, SubQ.SummedValue
FROM
Table1 INNER JOIN
(SELECT Table1.Field1, Table2.Field2, Sum(Table2.NumericValue) As SummedValue
FROM Table1 INNER JOIN Table2
ON Table1.Field1 = Table2.Field2
GROUP BY Table1.Field1, Table2.Field2) AS SubQ
ON Table1.Field1 = SubQ.Field1

That meant press (and hold) the Windows Logo key with R (which is a universal keyboard shortcut for bringing up the Run... dialog, even if the user does not have it in their Start Menu.)   In W7 an alternative to reaching the Run... dialog apparently is to just type run and press Enter.   Who knew?   (Not in any other OS, e.g. when there was no Start Menu Search box.)

An alternative would be use an Explorer window's Address bar as a command line input area.   E.g. press Win-E  and then Alt-d  to start that.

You need some kind of input area in order to add the switches to the program name.   Run... and Windows Explorer can find the path to the program for you but a cmd window cannot.   But if you would prefer to use a cmd window you could do it this way:


1.   Open a Windows Explorer window.   (E.g. press Win-E)

2.   Use the Address bar to navigate to  Internet Explorer's  home directory.   E.g. press Alt-d and enter there:    %ProgramFiles(x86)%\Internet Explorer

3.   Use the Address bar to launch a cmd window with its current directory open at that path.   E.g.  press Alt-d and enter:  cmd.exe

4.    Assuming that opened a cmd window with the current directory set to the directory that the Explorer window was opened at, in the cmd window's command line enter the command line I gave you for typing in the Run... input box:

       iexplore.exe  -nohome  -extoff

5.    If the cmd window is not showing that path type instead:

      cd  /d  %ProgramFiles(x86)%\Internet Explorer  &  iexplore.exe  -nohome  -extoff

6.   Launch the IE window from the cmd window with those options.   E.g. press Enter.

Unfortunately, these type of malware attacks are difficult to keep up with because they trick you into letting them install. They usually come from an infected web site, and usually through an advertisement. You get a pop-up from the infection and you click it to close the pop-up - which allows the infection to install.  They can also be delivered in a "drive-by" fashion with no action needed by the user due to the system being unpatched, no matter what security software is running.

-If you used task manager to close browser instances, reboot the machine.

-Perform a full scan with your antimalware program.
