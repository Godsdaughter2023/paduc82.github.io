Hi Nath,

 

This could occur because IE is being used by some startup programs to do things that require IE (perhaps I should be more specific and say an internet connection using browser functionality but in the background) but do not require IE to open a window to do them.  Examples might be startup entries to check for updates to certain programs which are using a "hidden" IE to access the site to check and/or update rather than Chrome. 

 

In any event, this sounds much more like a startup entry configuration problem than anything to do with malware.  It may even be something as simple as a time synchronization which might be harder to identify without the tools listed below.  Keep in mind that it's quite possible that what's happening is normal (given your configuration and in spite of having Chrome as the default) rather than a problem that needs to be fixed - though it could be something that is neither a problem nor normal but a matter of configuration (depending on what you have installed). 

 

The trick is to find out what is calling the process and why.  Once we know that, we can decide if it's a problem or normal (even desired) or something you want to eliminate (recognizing that doing so may have consequences).

 

Let's start by trying a clean boot to see if it still occurs.  Try a clean boot (Vista and W7) http://support.microsoft.com/kb/929135.  If the problem goes away then it's just a matter of tracking down the culprit causing the problem. Follow the procedures in the article. Once found, delete, remove, deactivate, uncheck, or uninstall it. Once done be sure to reset your system back to normal status as explained in the procedures. If the problem occurs in clean mode then just restore the system to normal status and reboot - this solution is not going to work. If using XP, see http://support.microsoft.com/kb/310353 and http://support.microsoft.com/kb/316434.

 

Note that whatever you uncheck will not happen, so if it is a check for a program update, you need to consider running that check manually from time-to-time to make sure it stays current and remains securely patched.

 

Though I'm fairly certain that will identify the problem and allow you to either take action or understand the reasoning behind it, excellent tools to help you with this (if the above doesn't work or to help you with the above since I suspect some things occur which may not show up as entries in such basic tools) are the following (all free from Microsoft):

 

Autoruns (a better version of MSCONFIG): http://technet.microsoft.com/en-us/sysinternals/bb963902.aspx.

 

Process Explorer (a better version of Task Manager): http://technet.microsoft.com/en-us/sysinternals/bb896653.aspx.

 

Process Monitor (a way to track the process history so you don't have to try to catch it all on the fly as it's occurring): http://technet.microsoft.com/en-us/sysinternals/bb896645.aspx.

 

Again, I do not think this is an infection but a process that is configured to run for reasons that we simply do not fully understand.

 

I hope this helps.

saltgrass say:

Attach another Disk Management picture, maybe it will help.

When you install an UEFI system over an older one, it needs the prior Critical partitions replaced since it will not usually overwrite them.  If you were installing as UEFI I will normally delete the four Critical partitions and then install in the vacated space.

If Diskpart will not delete a partition, there may be a good reason, such as it thinks it is still being used.  The Override switch will usually do the trick and is normally indicated with the first attempt fails.  Be aware, deleting the partition, depending on your configuration, may stop the boot.

I have 2 EFI system partitions, one on each drive where an independent OS install was done.  When I boot, I choose which OS I want to boot into using the Boot Device Menu.

rohn007 say:

"Document locked" can happen when you don't close Word properly.  When you open a file, Word creates a temporary "lock" file that it deletes when Word closes normally.

Look in the same folder as the file is saved in for files with the first 2 letters replaced by ~W.  Delete all of them.

The file being locked should not affect printing, to the best of my knowledge.

Go to Control Panel > Devices and Printers. Double click on the printer name you are using to open it. If there are any entries in it, delete them all. These are old print jobs that appear to be bottlenecked behind a "stuck" print job. Also turn off the Printer, leave it off for a minute, then power it back on.
