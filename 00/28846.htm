Anyway, I found some of my old code that did this.

This is a little simpler.  We use FormatNumber to get the correct number of decimal places, and we use a simple IF statement to add the "+" sign to the imaginary number if it's needed.

Function FormatComplex(N, Optional Digits = 2)
     Dim Re As String
     Dim Im As String

         Re = FormatNumber(.ImReal(N), Digits)
         Im = FormatNumber(.Imaginary(N), Digits)
         If .Imaginary(N) >= 0 Then Im = "+" & Im
     End With
     FormatComplex = Re & Im & "i"
 End Function

There are no decimal points in the inputs, but it appears to work.

I'm not sure what you mean by having to have a decimal point for negative numbers.

Outlook records meeting responses on a per-recipient basis. When you send to an Exchange GAL DL, you're essentially sending to a single recipient, the DL, not to individual recipients. So, in your scenario, when the responses come in, Outlook has no recipients with which to match them.

Hi,

Sorry for the delay in response.

This issue may arise when some of the Windows components or applications gets corrupted.

I suggest you to manually reset the Windows update components and check if it helps.

Reset Windows update components.
Resetting Windows Update Components will fix corrupt Windows Update Components and help you to install the Windows Updates.

Please follow the below steps to reset the Windows Updates Components manually:
1. Press Windows Key + X on the keyboard and then select Command Prompt (Admin) from the menu.
2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following 

    commands at a command prompt. Press the “ENTER” key after you type each command.
    net stop wuauserv
    net stop cryptSvc
    net stop bits
    net stop msiserver
3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands 

    in the Command Prompt. Press the ENTER key after you type each command.
    ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

    ren C:\Windows\System32\catroot2 Catroot2.old
4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the   

    following commands in the Command Prompt for this. Press the ENTER key after you type each command.
    net start wuauserv
    net start cryptSvc
    net start bits
    net start msiserver
5. Type Exit in the Command Prompt to close it.

 

Now you may try running the Windows Updates and check if the above steps resolve the issue.

Hope it helps. Revert to us with the results, will be glad to assist you.
