This resolved the issue for me.

For those of you who, like me, are wary of just running PowerShell commands from the web willy-nilly:

This script uses the Get-AppxPackage cmdlet to get all the installed app packages in the system (for all users). It then pipes the installation directory of the apps (via the InstallLocation property) to a ForEach loop. The loop runs the Add-AppxPackage cmdlet for all the listed app packages, registering them from the local files.

Essentially it reinstalls all loaded Universal apps.

Filtering subform in Tab Control - Microsoft Community

tamras72 say:

Filtering subform in Tab Control

I have a tab control [tabCtl17] with 4 pages (Name / Page Index):

- Details [0]

- Monthly Activity [1]

- Increased Activity [2]

- Yearly Activity [3]

 

Each page has a subform.  I want to be able to filter the subform based on the tab. For example, if I click the Yearly Activity tab, I want to filter the [Yearly Activity Subform] control [rptYear] by the current year.

 

I tried the onClick event of the [Yearly Activity] page using:

SetFilter

Where = [rptYear] = Year(Date())

ControlName = [Yearly Activity Subform]

 

It didn't do anything :(

 

Any help is appreciated.  Thank you.

 

 

ken sheridan say: 1.  In the page of the tab control which contains the yearly subreport add an unbound text box, txtYear, above or below the subform as preferred.

2.  For the subform's RecordSource, instead of using the Year function to restrict the result set use a parameter which references the text box in the parent form:

WHERE rptYear = Forms![YourParentFormName]!txtYear

3.  In the parent form's Load event procedure set the value of the text box to the current year with:

    Me.txtYear = Year(VBA.Date)
    Me.[Monthly Activity Subform].Form.Requery

Note that Monthly Activity Subform must be the name of the subform control, i.e. the control which houses the subform, not the name of the underlying form object, unless both have the same name.

4.  Requery the subform in the AfterUpdate event procedure of the txtYear control with:

    Me.[Monthly Activity Subform].Form.Requery

When the parent form loads the subform will be restricted to the current year.  When a user changes the value in the txtYear text box it will be requeried to the year entered in the text box.  No further code is needed to filter the subform.

You can do similarly on other pages of the tab control to allow the user to change the rows returned in a subform, e.g for a monthly subform you'd have text controls for the year (named something other than txtYear of course) and month, and these would be referenced in the subform's RecordSource:

WHERE rptYear = Forms![YourParentFormName]!txtYear_2
AND rptMonth = Forms![YourParentFormName]!txtMonth
