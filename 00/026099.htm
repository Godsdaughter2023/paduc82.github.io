You can certainly store all the "root" values needed to create a Job Number like "A300" or "A300-1". I'd think you'd store a JobType (the "A"), the Job Number (the "300") and the JobSequence (the "1").

randy ebs say:

Auto numbering in excel

I am trying to create a project list that contains a project number (column A and a project name (column B). How do I get cell A1 to automatically create the project number? The number should always start with the year that the project was added to the list.

For example:

 

When I type a project name into B1, I need A1 to display 131. This would indicate the year 2013, project 1. And when I type a project name into B2, I need A2 to display 132, which would indicate the second project of the year 2013. And so on....

jllatham say:

Try the following worksheet level event processing code.  Note that I've set it up to NOT work in row 1 - assuming you have labels in that row even though you did say A1 and B1 in your request.  If you really need entries in row 1 also, just say so and I'll show you how to edit the code to work there also.  (actually, simple, just delete the line of code that reads:

   Or Target.Row = 1 _
and make sure after you delete it that you don't leave a blank row there.

 

Caveat:  if you delete a project in the middle of a list and make a new entry there, the new entry will then contain the new "largestToDate" value.  Doing so may leave a gap in your project number series.

 

I don't know if you want this part of the code or not:

'if user deleted project name entry, delete project number!
If IsEmpty(Target) Then
   Cells(Target.Row, 1).ClearContents
End If

If you don't, just delete it from the code below.  After adding the code to your workbook, be sure to save it as a 'macro enabled workbook, type .xlsm', or the code will get removed.

 

 

Private Sub Worksheet_Change(ByVal Target As Range)
' to put this code into a Worksheet code module see
' the instructions on this web page
' http://www.contextures.com/xlvba01.html#Worksheet

  'only works when 1 cell in column B changes
  'and only assigns project number if cell
  'in column A is empty when entry is made into B
  'change must also be in row 2 or greater
  Dim largestToDate As Long
  Dim testToday As Long
  If Target.Cells.Count > 1 _
   Or Target.Column <> 2 _
   Or Target.Row = 1 _
   Or Not IsEmpty(Cells(Target.Row, 1)) Then
    Exit Sub
  End If
  'if user deleted project name entry, delete project number!
  If IsEmpty(Target) Then
    Cells(Target.Row, 1).ClearContents
  End If
 
  'simple logic used here:
  ' each project number is going to be greater
  ' than any before it.  Only thing to watch
  ' for is difference in year
  largestToDate = Application.WorksheetFunction.Max(Range("A1:" & _
   Range("A" & Rows.Count).End(xlUp).Address))
  'test if largestToDate is in current year
  If Val(Left(CStr(largestToDate), 2)) < Right(Year(Now()), 2) Then
    'time to start a new year's sequence!
    Cells(Target.Row, 1) = Val(Right(Year(Now()), 2) & "1")
  Else
    'still in same year, just bump up by one
    Cells(Target.Row, 1) = largestToDate + 1
  End If
End Sub

 

 

jllatham say:

Turns out the above code DOES NOT WORK!

Revised to work with an added sheet in the workbook (named SysUse which may be hidden).

 

The new code - a bit of a kludge, but hammered together in a hurry late at night after a long day:

 

Private Sub Worksheet_Change(ByVal Target As Range)
 ' to put this code into a Worksheet code module see
 ' the instructions on this web page
 ' http://www.contextures.com/xlvba01.html#Worksheet
 
  'only works when 1 cell in column B changes
  'and only assigns project number if cell
  'in column A is empty when entry is made into B
  'change must also be in row 4 or greater
  Const pnumCol = "A"
  Const pnameCol = "D"
  Const pnumFirstRow = 4
 
  Dim largestToDate As Long
  Dim testToday As Long
  Dim sysWS As Worksheet
  Dim pNumList As Range
  Dim lastRow As Long
  Dim workList As Range
  Dim anyWorkCell As Range
  Dim nextPNumber As Long
  
  If Target.Cells.Count > 1 _
   Or Target.Column <> Range(pnameCol & 1).Column _
   Or Target.Row < pnumFirstRow _
   Or Not IsEmpty(Cells(Target.Row, 1)) Then
    Exit Sub
  End If
  'if user deleted project name entry, delete project number!
  If IsEmpty(Target) Then
    Cells(Target.Row, 1).ClearContents
  End If
   
  lastRow = Range(pnumCol & Rows.Count).End(xlUp).Row
  If lastRow < pnumFirstRow Then
   lastRow = pnumFirstRow
  End If
  Set pNumList = Range(pnumCol & pnumFirstRow & ":" & pnumCol & lastRow)
  Set sysWS = Worksheets("SysUse")
  sysWS.Cells.Clear
  Application.ScreenUpdating = False
  pNumList.Copy Destination:=sysWS.Range("A1")
  Application.CutCopyMode = False
 
  lastRow = sysWS.Range("A" & Rows.Count).End(xlUp).Row
  Set workList = sysWS.Range("A1:" & sysWS.Range("A" & Rows.Count).End(xlUp).Address)
  For Each anyWorkCell In workList
    If IsEmpty(anyWorkCell) Then
      'this handles the first project number on a clean sheet.
      Cells(Target.Row, pnumCol) = Val(Right(Year(Now()), 2) & "1")
      Exit Sub
    End If
    anyWorkCell.Offset(0, 1) = Val(Left(anyWorkCell, 2))
    anyWorkCell.Offset(0, 2) = Val(Right(anyWorkCell, Len(anyWorkCell) - 2))
    If CStr(anyWorkCell.Offset(0, 1)) <> Right(Year(Now()), 2) Then
      anyWorkCell.Offset(0, 1).Clear
      anyWorkCell.Offset(0, 2).Clear
    End If
  Next
  'all we should have are cells with this year's project numbers in them
  nextPNumber = Application.WorksheetFunction.Max(sysWS.Columns("C:C")) + 1
  Cells(Target.Row, pnumCol) = Val(Right(Year(Now()), 2) & CStr(nextPNumber))
  'a 'best practice' - explicitly release assigned resources back to the system.
  Set sysWS = Nothing
  Set workList = Nothing
  Set pNumList = Nothing
End Sub
