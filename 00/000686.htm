The HP rep was probably confusing windows phone with its older, rather distant cousin windows mobile.

 

There is no doubt in my mind that the HP printer could be used to print via bluetooth connection with an old windows mobile phone, but it is not possible to print from a windows phone.  These are two different mobile operating systems that do not share the same functionality.

julie fka profjulie say: Unfortunately, you won't be able to connect your phone to a bluetooth printer and print.  The phone does not have any bluetooth printing profiles (the only bluetooth profiles that a windows phone has are audio profiles).  There also isn't a way to connect a windows phone to a printer via usb connection. 

You might be able to find a way to print via some kind of internet or network connection, but this would require finding a print driver for windows phone that is distributed by the printer OEMs and pretty limited to the makes/models of printers supported, something like this Brother one.  Print drivers for windows phone are pretty rare, and I am not aware of any HP drivers for windows phone 7.

Microsoft has always had an interest in steering users toward the Windows operating system. In addition, the OS X market share is relatively small with less potential revenue, while the costs of programming are the same.

jim gordon mvp say:

Triggers are supported in that if they exist within a presentation PowerPoint for Mac will honor their settings and work the same as if the presentation is playing in Windows PowerPoint.

Triggers are supported in Visual Basic for Applications (VBA) programming language in PowerPoint for Mac. You can make and modify triggers using VBA.

The user interface for triggers is not available on the Mac at this time. If you follow the link John provided you can vote for having the interface built, as well as see Microsoft's statement about the feature.

Personally, I am surprised that someone hasn't built an add-in that provides a GUI for triggers on the Mac.
