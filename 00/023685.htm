Let me fix it myself To unregister the mswrd632 converter yourself, edit the registry as follows:

1. Click Start, click Run, type regedit, and then click OK.

2. Locate and then click the following registry subkey:

You seem to be confusing two very different scenarios.

Both Vista and Windows 7 are operating systems. You usually¹ have only one of these, so installing one will overwrite the other. To avoid loss of data, you need to upgrade from one to the other which should² retain all of your old programs, files and settings if performed correctly.

Office 2010 is a suite (e.g. collection) of software applications that work to create and edit documents, spreadsheets, databases and presentations. It is designed to work under an operating system like Windows 7.

So the answer is yes, Office 2010 products are compatible with Windows 7 operating systems.

Microsoft Office 2010 system requirements

ken sheridan say: If each contact can have one or more phone numbers, and each phone number applies to only one contact, then the telephone table needs a foreign key ContactID column referencing the primary key of ContactDetails, not vice versa.  Data entry would normally be via a telephones subform within a contacts parent form, the subform being linked to the parent form on ContactID.

If on the other hand each phone number can apply to one or more contacts then the relationship type between ContactDetails and Telephones is many-to-many.  To model this neither table has a foreign key, and the relationship type is modelled by a third table as follows:

ContactTelephones
....ContactID (FK)
....TelephoneID (FK)

The primary key of this table is a composite one composed of both columns.  The table resolves the many-to-many relationship type into two one-to-many relationship types as follows:

Contacts---<ContactTelephones>---Telephones

Data entry would again be by means of a form/subform, but in this case the subform is based on the ContactTelephones table, not the Telephones table, and the control in the subform would be a combo box which lists the available phone numbers.  You'll find a simple example of such a many-to-many relationship type and how to represent it as ParentActivities.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

To enter a new phone number not yet included in the Telephones table the NotInList event procedure of the combo box can be used to enable the new number to be typed into the combo box, and a new row inserted into the Telephones table by the code in the event procedure.  My demo also illustrates this; try typing in a new activity in the activities subform in the parents form to see how it works.
