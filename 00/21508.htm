Hello Avrisan,

We have been hitting exactly the same problem which we believe it is a bug of Win10. The same still occurred in 1803 as well.
The observation from our side is that the problem occurs with non-XPS printer drivers (meaning that XPS was not the PDL used, e.g. PCL), regardless of printer brands/models, and is also dependent on font types.

In addition, the problem is that font-weight was not able to change after the first character of an affected font. That means if you have a character of the affected font set to bold at first, the rest of the characters using the same font would become bold regardless of the style set for them. (For your reference, before MS fix the problem, we are currently trying to workaround this problem for example with PMingLiU for bold characters and MingLiU for normal characters - taking the advantage that PMingLiU and MingLiU are different font types but identical style for Chinese glyphs. It is not so desirable though.)

It seems to be a bug of IE/Win10 with the universal printer driver at the generation of print spool file?
