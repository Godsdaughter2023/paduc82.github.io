quietman7 say:

Most security experts will advise against paying the ransom demands of the malware writers because doing so only helps to finance their criminal enterprise and keep them in business. One of the reasons that folks get infected is because someone before them paid the bad guys to decrypt their data. The more people that pay the ransom, the more cyber-criminals are encouraged to keep creating ransomware for financial gain. Further, there is never a guarantee that paying the ransom will actually result in the successful restoration (decryption) of your files or that the decrypter provided by the cyber-criminals will work as they claim. Using a faulty or incorrect decryptor may damage or corrupt the files even further. The criminals may even send you something containing more malware...so why should you trust anything provided by those who infected you in the first place.

With that said, some ransomware victims have reported they paid the ransom and were successful in decrypting their data. Other victims have reported paying the ransom only to discover the criminals wanted more money...demanding additional payments with threats the data would be destroyed or exposed. Still others have reported they paid but the cyber-criminals did not provide a decryptor or a key to decrypt the files, while others reported the decryption software and/or key they received did not work, resulted in errors and in some cases caused damage to the files. Most cyber-criminals provide instructions in the ransom note that allow their victims to submit one or two limited size files for free decryption as proof they can decrypt the files. However, decryption in bulk may not always work properly or work at all and decryption of very large files may be unsuccessful even with the criminal's decryption tool. In some cases victims may actually be dealing with no intention or capability of decrypting files after the ransom is paid.

Now to answer your question on what you should do with your encrypted data files due Stop Ransomware.

In cases where there is no free decryption tool, restoring from back up is not a viable option and backup/save your encrypted data as is and wait for a possible solution...meaning, what seems like an impossibility at the moment (decryption of your data), there is always hope someday there may be a potential solution.

Law enforcement authorities have had some success arresting cyber-criminals, seizing C2 servers and releasing private RSA decryption keys to the public. In some cases, the cyber-criminals, for whatever reason, choose to release the master keys after a period of time.

Note: Imaging the drive backs up everything related to the infection including encrypted files, ransom notes, key data files (if applicable) and registry entries containing possible information which may be needed if a solution is ever discovered. The encrypted files do not contain malicious code so they are safe. Even if a decryption tool is available, there is no guarantee it will work properly or that the malware developer will not release a new variant to defeat the efforts of security researchers so keeping a backup of the original encrypted files and related information is a good practice.

Will MSE scan if computer is turned off ?? - Microsoft Community

Will MSE scan if computer is turned off ??

Will Microsoft Security Essentials run a scheduled scan if computer is turned off ??
kosh vorlon say:

Hi Philip,

 

Here's a couple of things about scheduled scans. To run, the PC must be on and connected to the internet (it cannot be shutdown, in sleep mode, in hybrid sleep mode, or in hibernation). The power saving settings must be such that the PC does not shut down or go asleep during the scan (or it will be as if it never happened). Scans do not begin precisely when scheduled - some can start an hour or so earlier or later than the officially set time (depending on Microsoft's servers and the loads at that time). There is a way to use Task Scheduler rather than MSE to awaken sleeping computers, run the scans, and then shut them down when done, but it is somewhat complex. More on that shortly.

 

Also, go to settings and check scheduled scans and uncheck the box that prevents them from starting unless the system is on and idle (if you're using the system when it's scheduled or it otherwise isn't idle - perhaps downloading Windows Updates or doing something else - then the scan will not start and will be missed). If a scan is missed it does not start at the next available time, but waits for the next scheduled scan.

 

Go to Start / Control Panel / Administrative Tools / Services / scroll to Task Scheduler and make sure it is started and automatic. If not set it to automatic and restart it. If it won't start, click on the dependencies tab and do the same thing for each of the ones listed. If any of those won't start, continue with it's dependencies and do the same thing. When you get to the end, reverse course and Start the ones you couldn't because of the dependencies as you go until you get to Task Scheduler and hopefully it will now go to automatic and start. If you changed nothing read on. If you changed anything, reboot and see if that helps.

 

There is a way to use Task Scheduler rather than MSE to awaken sleeping computers, run the scans, and then shut them down when done, but it is somewhat complex. You can also use it to create different scheduled scans of different types since MSE only allows one choice.

 

Please see http://answers.microsoft.com/en-us/protect/forum/protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02 - especially the later corrections from GreginMich.

 

The post from GreginMich here http://answers.microsoft.com/en-us/protect/forum/protect_start/scheduled-scans-not-working/fd769043-f67e-4990-8805-a02427f22dff will also be useful.

 

I'd also suggest the post from GreginMich here http://answers.microsoft.com/en-us/protect/forum/protect_start/i-would-like-to-schedule-mse-to-scan-pc-with/58474560-4cca-49c1-a51a-76ca9bc3b9f2

 

I hope this helps.

 

Good luck!

peter n roth say:
  1. Launch your database.
  2. Near the top of the application window are the menus. In Access 2010, they are File, Home, Create, External Data, Database Tools, and perhaps another one or two. Click the database tools menu tab to open that ribbon.
  3. Second from the left is Visual Basic. Click that. A whole new world opens before you.
  4. In the upper right hand corner is the help search window. In that window, type docmd.outputto. The Help window will open, but may say “nothing available”. That’s when you go to the lower right corner of the help window and click on “Connected to Office.com” to open yet another dropdown. Choose “Show content only from this computer”. (Are we having fun yet?)
  5. Now at last, you should have some helpful content in the help window. You might find that the 4th parameter “Forms!ExcelExport!ReportTitle” is the name of the file to be (over)written.
  6. Now you can do an internet search on “VBA file exists” and get some code you can use to prevent overwriting. It will look something like this:
             Public Function ShallIOverwrite(fname As String) As Boolean
    Dim exists As Boolean
    exists = FileExists(fname) '  find on internet

    Dim result As Integer
    result = vbNo
    If exists Then
        result = MsgBox(fname & " exists. Overwrite?", vbYesNo, "OVERWRITE FILE?")
    End If

    If Not exists Or (result = vbYes) Then
        DoCmd.OutputTo acOutputReport, "ESR", "MicrosoftExcelBiff8(*.xls)", fname, True, "", 0, acExportQualityPrint
    End If

End Function
