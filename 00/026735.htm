Sub HideRows()
On Error Resume Next

ack_ say:

I was having the same issue where after updating from Office 2007 to 2013.  All of my external links to mapped network drives wouldn't update.  I have hundreds of spreadsheets with external links on our company's network drive and it worked fine in office 2003 and 2007.

It turns out after installing office 2013, Microsoft resets all of the security settings by default.  You need to do the following to get your external links to update properly:

1. Go to File, Options, and select "Trust Center" from the left hand column of options.

2. Click on "Trust Center Setting..."

3. On the left side of the screen, click on "External Content"

4. Under security setting for Workbook Links, make sure "Enable automatic update for all Workbook Links (not recommended)" is selected.

5. Click "OK".

I also made sure that the security settings for "data connections" was also enabled, and enabled all ActiveX settings.  While not recommended by Microsoft, for my purposes it was necessary to have these enabled.  I don't believe these are necessary to have external links update.

I hope this works for anyone having this issue.  I spent most of my morning trying to find answers on the forums to this problem, and other solutions weren't working (updating network links from "'X:\folder to '\\Fileserver name\folder).

Good luck!

lumia 2520 windows rt 8.1 updates say:

I have performed the following on my Nokia Lumia 2520 32 Bit OS - ARM  Windows RT 8.1 to get it up to date.

1.)  You need to download to a USB Stick the following:

      "Update for Windows 8.1 KB 2919442" from www.microsoft.com downloads.

      You will install this update after you have performed a full restore of your system.

2.)  Go to Settings on your tablet, then select PC Settings, then select Update and Recovery, then select Recovery,

      then select Remove Everything and Reinstall Windows.  Wait for the system to recover.

3.)  At this point when I checked update history, there were 10 updates installed.  DO NOT CHECK FOR UPDATES

4.)  It is now time to install KB 2919442 which you should have downloaded to a USB Stick.  When you click on it, it

      should ask you if you want to download the update and install it.  Proceed with the install.  KB 2919442 performs

      the magic that should result in you receiving the update that updates your machine to the new interface we are all

      missing, KB 2919355.  Check update history and make sure KB 2919442 installed.

5.)  At this point I went to the Store and downloaded and installed Apps that I had on my tablet prior to the reinstall.

      Really do not know if this is required at this point but I felt more comfortable doing it here.  I also added accounts

      at this point for other family members whose accounts had been deleted because of the reinstall.

6.)  It is now time to check for windows updates.  I received error code 80244010 because of the volume of updates I

      attempting to retrieve.  I just hit retry and it worked.  If your machine is like mine (ATT) you probably will be notified

      that a System Firmware is available and needs to be installed along with the other updates.  Check the updates

      and make sure KB 2919355 is one of the updates that is going to be installed.  After the System Firmware and 

      Updates were installed my tablet now had the famous search icon that meant I had upgraded.

7.)  There are lots of updates to be installed (I believe my installed count is 298).  Be patient as it is going to be very

      time consuming.

8.)  And yes, KB 3033055 was installed which is the new start screen.  You will enjoy the new interface and store after all

      updates have been installed.

9.)  Best of luck and no guarantees.  All I know is it worked for me.
