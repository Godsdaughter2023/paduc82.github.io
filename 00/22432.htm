3. The purpose for "strSQL" is to create a recordset of unique CL_NAME's. This recordset is then used as a source for code that loops through it to create a concatenated list of CL_NAME's. The control source for a field on the repot is then set to the concatenated list.


Now -- finally -- I understand!  I'll bet you are (or were) using code like this:

 

    Dim rs As DAO.Recordset

    Dim strSQL As String

 

    strSQL = "SELECT DISTINCT CL_NAME FROM [Int Std1]"

    Set rs = CurrentDb.OpenRecordset(strSQL)

 

But that won't work when the query has a reference to a form control, as DAO is resolving this query, not Access, and DAO doesn't know about forms and controls; it just sees an unresolved parameter in the query.  You can make it work, though, by changing the code like this:

 

    Dim db As DAO.Database

    Dim qdf As DAO.QueryDef

    Dim prm As DAO.Parameter

    Dim rs As DAO.Recordset

    Dim strSQL As String

    strSQL = "SELECT DISTINCT CL_NAME FROM [Int Std1]"

 

    Set db = CurrentDb

    Set qdf = db..CreateQueryDef("", strSQL)

 

    ' Resolve parameters in the query.

    For Each prm In qdf.Parameters

        prm.Value = Eval(prm.Name)

    Next prm

 

    Set rs = qdf.OpenRecordset()
