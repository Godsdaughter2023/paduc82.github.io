What you call "sort" is actually "filter". Sort means ordering items alphabetically or numerically.

The Rami Radi Daily Wallpaper Viewer is a free third party app available from the Windows store. The currently available version is intended to be installed on Windows 8.1 systems. 2) Is it serious?

The software is causing faults in the correct running of COMBASE.DLL, a dynamic link library which provides API (application programming interface) functionality to at least 16 related .DLLs. The seriousness of the fault you are experiencing is a subjective call. If your system (or an application like FireFox) is crashing and the timing of the crashes matches the reports in Event Viewer, then yes... it is serious. If you are not experiencing any program conflicts beyond an occasional hiccup that you can live with then no... it is not serious.

Keep your system up-to-date with Windows Update and revisit the Microsoft Store and/or the developer's web site for updates and bug fixes.

I would guess that the Daily Wallpaper viewer is conflicting with something else on your system. Run FireFox with no add-ons (FireFox, Help, Restart with add-ons disabled) to see if that clears things up. If it does and you really want to keep the wallpaper viewer, try disabling other add-ons that you can live without.

james yan msft say:

Hi Nghdawg,

We can use "Copy Link to Notebook" function to link to the thirty notebooks from one notebook. ("Copy Link to Notebook" is only available in OneNote client.)

1. Image these notebooks are all stored in your OneDrive for Business library or your team site library. Create the Sales Activity notebook in the same library. Make sure all the notebooks are shared with your boss (right click the notebook > Share).

2. Open all the thirty notebooks in OneNote client. (right click the notebook in the library > "Open in OneNote")

3. In OneNote client, locate the section which you want to link to, right click the section tab > click "Copy Link to Notebook". Locate the Sales Activity notebook, open a new section, paste the link to the content area.

Note: After we finish these steps, the section will be opened in OneNote client by default if we click on the links. If we want the link to open in OneNote Online each time we click on it, please remove the onenote:xxxxxx part in the link.

(right click the link in the notebook > Edit Link... > in the Address part, remove the onenote:https://....&end part in the link. So the link looks like https://****-my.sharepoint.com/personal/james_****_onmicrosoft_com/_layouts/OneNote.aspx?id=%2Fpersonal%2Fjames_****_onmicrosoft_com%2FDocuments%2F12&wd=target%28sec3.one%7CFFB8315B-7960-45FC-B0DF-330B72E93AF6%2F%29)

Thanks,
