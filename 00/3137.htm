stefan blom say:

Turn off the option to update fields on print (File tab | Options | Display: "Update fields before printing").

To make sure that the TOCs are actually up to date, you'll need a macro that updates them in one step. For example:

Sub UpdateAllTOCFields()
Dim toc As TableOfContents
For Each toc In ActiveDocument.TablesOfContents
toc.Update
Next toc
End Sub

For assistance, see http://www.gmayor.com/installing_macro.htm.

knightryder2 say:

Microsoft is really screwing the pooch with these updates lately...some of our users are having this index issue, others can't even attach word documents to send in emails.

Here is the fix for your search problem (worked for us anyways):

1. Quit Outlook.

2. Open Terminal by going to Applications --> Utilities --> Terminal

3. Type "mdimport -L"

4. If you see more than one instance of "Microsoft Outlook Spotlight Importer.mdimporter", delete the Outlook application that you are NOT using, empty it from the Trash, restart your Mac, and go back to Step 1.

5. Now re-index your Outlook database using the following command, substituting your own <user_name>:

mdimport -g "/Applications/Microsoft Outlook.app/Contents/Library/Spotlight/Microsoft Outlook Spotlight Importer.mdimporter" -d1 "/Users/<user_name>/Library/Group Containers/UBF8T346G9.Office/Outlook/Outlook 15 Profiles/Main Profile"

Note in the command above the path after "-g" is the default path where your Outlook is installed. The path after "-d1" is the default path to your profile. You will have to substitute your actual paths if you have renamed your profile or installed Outlook in a different location. Also please do change the <user_name> in the above command.

6. After starting the command in Step 4 it will take some time to complete re-indexing. Please wait until the re-indexing completes.

After following the above steps, your search should start working again.
