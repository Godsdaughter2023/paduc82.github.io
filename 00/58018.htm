a. user say:

Hi Arifeen,

As per your description, I understand that when your PC goes to Sleep the two external monitors that are attached to your PC do not wake. Upon reboot, the setting personalized in the other two monitors are lost which enforces you to re-set the settings.

The issue is most likely with the graphics driver.  Whenever a single monitor is disconnected from a multimonitor setup in Windows, Windows will attempt to move everything from that screen onto the remaining screen(s).  Usually, this will not occur when suspending/hibernating/shutting down the computer.  Thus, it is probably because of the graphics driver doing one of the following:

  • 'Disconnecting' the monitors when you put the computer to sleep, so when Windows awakens, it must re-determine where to place everything.
  • Recognizing the monitors in a different order each time you bring the computer out of sleep, forcing Windows to rearrange your application windows.

To rectify the above, I suggest that you open Windows Update and check if there are any pending updates related to your PC as well as your graphic card driver. If so, disconnect all external devices other than mouse and keyboard, install the updates and check.

If the above did not help, then I suggest that you uninstall your graphic card driver and re-install the latest driver update available from the manufacturer's website. To learn how to perform the same, refer to the steps outlined in the link below

How to: Install and Update drivers in Windows 10

andre da costa say:

To install Windows 8 on your PC, the processor (CPU) must support the following features: Physical Address Extension (PAE), NX, and SSE2. Most CPUs have support for these features, so if you receive this error, it is likely because the NX feature is not enabled on your system.

-------------------------

To resolve this error, follow manufacturer guidelines to enable NX ("No eXecute bit"), or the equivalent XD ("eXecute Disabled"), feature within the BIOS settings. This feature is typically found in the Advanced or Security tabs within the BIOS settings, and can be referred to by a variety of names, including but not limited to:

· No Execute Memory Protect

· Execute Disabled Memory Protection

· EDB (Execute Disabled Bit)

· EVP (Enhanced Virus Protection)

If the BIOS setting for the NX (XD, EDB, or EVP) support option is not available on your system, you may need to contact the manufacturer to update the BIOS. Note that some very old processors may not contain these features and will be incompatible with Windows 8 Release Candidate.

A whitepaper has been published with further details about the PAE/NX/SSE2 requirement for Windows 8, error cases and scenarios that customers encounter when machines fail to meet the requirement, and what to do to install Windows 8 on their PC’s. You can download the whitepaper at http://msdn.microsoft.com/en-us/library/windows/hardware/hh975398.aspx.

DeviceDisplayObjectProvider.exe  is part of the Microsoft .NET framework and as long as it is running from the System32 directory under Windows it should be fine.

It is primarily used to update the status of icons in the task bar and/or list removable devices.
