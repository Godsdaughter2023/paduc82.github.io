chet814 say:
, , , if installed a hard firewall will it protect me from a pc in my network if they are hacked?


I'll use my home network as an example to answer your above question.

 

My network consists of my cable modem then the router then my computers and devices. My network consists of:

 

2 Windows 7 Professional x64 desktops - wired

1 Windows XP Professional x86 desktop - wired

1 Windows XP Home Edition x86 desktop - wireless

2 Windows 7 Home Premium x64 laptops - wireless

1 TV - wired

1 Blu-ray home threatre - wireless

1 Gaming console - wireless

Plus several virtual machines on the 2 Win7 Pro desktops (MS Virtual PC and VirtualBox), the WinXP Pro desktop (MS Virtual PC 2007) and one of the Win7 Home Premium laptops (VirtualBox). All of the virtual machines (when running) are accessible on the network. The virtual machines include 1-Windows 98, 1-Windows 98 SE, 2-Windows XP Mode x86, 3-Windows Vista all x86, 3-Windows 7 1-x64 and 2-x86, and the Windows 8 Developers Preview x64.

 

All of the above are protected by the hardware firewall in the router plus all the computers and virtual machines have their individual software firewalls (I use the built-in Windows firewall).

 

Should a hacker break-through the router's firewall into a single computer, it would make the other computers more vulnerable as only their software firewalls would remain, but it would not automatically grant them access to the other computers.

 

So, to answer your question: No - once a hacker broke through the router into a single computer it would no longer protect the other computers in the network, the other computer's protection would then fall onto their individual software firewalls.

 

With all that said, if a hacker is smart enough, they can break through any firewall, but for a home network having both a hardware and software firewall is about the best you can do. Large corporations and governments spent countless $$ on protecting themselves from hackers and then, occasionally, this is still not enough as someone will still break through.

Why do Windows programmers have such a problem with log files? - Microsoft Community

stephen_ford say:

Why do Windows programmers have such a problem with log files?

As I understand it the log file is there to help solve problems by providing information. I suppose it could have legal overtone too in various scenarios. However, when a file is in a temporary folder, by its very location it's expendable. Therefore when a log file is in a temporary folder it's telling us that there is probably something wrong. If there is nothing wrong and it's in a temporary folder, then the Bl**dy software should tidy it up...

I would have been mortified in my programming days if a user had levelled the complaint at me that my code had left lots of log files scattered around temp folders.

There have been plenty of issues posted where OneDrive goes into overdrive in filling up its log files. But I suggest that when OneDrive creates the aria-debug-xxxx log files it should delete them when the PC shuts down, or at the beginning of a new day, or at a user-configurable milestone. Leaving log files around is in my view and indication of an untidy mind. Do we want programmers like this working on Windows? For me it doesn't bode well...

If there is any chance that a log file may be required then it's not temporary. And at that point there needs to be some explanation of it's purpose.

And then there are other programmers who create systems (eg drivers) where it would be really handy to have a log file for debugging.

So in the Window's fraternity we get log files when they are of no use, and we don't get them when we need them. What sort of mindset operates like this? Nanny McPhee?
