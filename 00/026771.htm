willy denoyette say:

The timings are important.

If you put a SF to sleep, it enters what is called modern sleep which is not a real deep sleep.

After a while, the system may transition to hibernate state, provided hibernate is configured.

So there are a number of things you'll need to check.

1. Is hibernation configured?

you can check this from the command prompt (or powershell) by running next command

powercfg /a

Hibernation is enabled when this command returns Hibernate as a  possible sleep state.

2. If Hibernate is enabled, check when the system enters hibernate when asleep.

powercfg /q SCHEME_BALANCED SUB_SLEEP  HIBERNATEIDLE

this returns some info from which next is the most important. (note that this assumes your power scheme is set to balanced)

....

    Current AC Power Setting Index: 0xxxxxxxxxx
    Current DC Power Setting Index: 0xyyyyyyyyy

xxxxxx and yyyyy are the time values (in hex) after which the system hibernates from sleep.

f.i when yyyyyy is set to 0x00000100 , the system will hibernate after 0x258 = 600 seconds being asleep.

With this information at hand, it should be easy for you the determine whether your issue is return from sleep or hibernate related.
