scottgem say:

The code I suggested was designed to pop up a messages whenever you view a record within 2 days of expiration on your form. It means you actually have to open that record. 

If, instead, you want a list of all records that will expire and have that list open when you first open the database, then you need to do it differently. 

You would need to create a query from the T_Package table. You would set the criteria for the ExpirationDate column to <DateAdd('d",-1,Date()). This query should return all the expiring records. You could then create a report base on that query and set that report to run when you open the database. Either with an AutoExec macro or in the On open event of your default form.
