This issue is mainly caused When display Dots-Per-Inch (DPI) gets sufficiently dense this “shrinking effect” can make content hard to see and UI targets difficult to click/tap.

 

By default, Windows 8.1 will choose the optimum DPI scaling value for each display based on the value chosen for overall scaling in the control panel.

To adjusted the screen resolution for each of the displays connected to your Windows system to exaggerate DPI scaling. This resulted in the following physical display DPI values:

• 23” display: 64 DPI
• 27” display: 81 DPI
• 32” display: 140 DPI (native)

 

To check the UI scales configuration let’s try to open Calculator app side-by-side on each display using 100% scaling in the control panel, If we overlay screenshots of these calc.exe app instances, we can see how they are scaled differently at the pixel level.
