¡firedog say:

When you import contacts, each one will be added as a new contact regardless of whether any of them were already present in the list. If a contact was already present, you'll end up with duplicates. Either delete the contacts in WLMail you know to be duplicated in the CSV file before importing, or (probably safer) remove the duplicates once the import is complete.

If the import starts and progresses some way before stalling, then the CSV file is in the right format but there are one or more records (lines) in the file that contain corrupt data. You would have to examine the CSV file, for example in Excel, to look for dodgy data in the first row that failed to import properly. You could try simply deleting the row before trying again. Don't leave any blank lines in the CSV file.

If you're not confident about doing this yourself, I'll happily have a look at the CSV file for you. You could upload it to SkyDrive and then use the Share function to email a link to jnllb at outlook dot com. No promises, though.

It's impossible to say what the problem might be, but a common cause is having too much data in the Notes field. WLMail can only handle 1024 charactes in this field; if there are more in the CSV record, the import will fail.
