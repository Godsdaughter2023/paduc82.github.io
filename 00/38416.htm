Falls obige Tipps nicht funktionieren, musst Du dem Hotel weismachen, dass die Xbox Dein Laptop ist. Das geschieht über MAC Address Cloning oder Spoofing.

Starte Deinen Laptop, und gehe in den Netzwerkeinstellungen auf Deine WLAN Karte. Wähle dort im Rechtsklickmenü "Status". Du solltest dort in den Details eine Zeile sehen die "Physikalische Adresse" (oder ähnlich) lautet. Dort sollte die MAC Adresse der WLAN Karte stehen. Sieht in etwa so aus: 12-A3-B4-C5-9F-8E

Notiere Dir die MAC Adresse. Dann starte die Xbox und gehe in dem Einstellungs-Hub auf die System Kachel oben links und dann in die Netzwerkeinstellungen.

Verbinde Dich mit dem Hotel WLAN. Gehe dann in die erweiterten Netzwerkeinstellungen, indem Du einmal weiter nach rechts gehst.

Dort kannst Du für den Xbox WLAN Adapter eine eigene MAC Adresse eingeben. Trage dort die WLAN MAC Adresse des Laptops ein.

Damit sollte die Xbox dann online gehen können.

ABER ACHTUNG! Du darfst jetzt NICHT den Laptop und die Xbox gleichzeitig verwenden!!

Hi Alex,

What we can do is rebuild the Windows Icon Cache Database, this is where files and icon apps are stored, it makes easier for the system to retrieve the icons faster.

Please follow these steps:

  1. Right click Windows icon then select Command Prompt (Admin)
  2. Type cd %homepath%\AppData\Local\Microsoft\Windows\Explorer then, type dir iconcache* to verify the icon cache database are at the location.
  3. Stop File Explorer by typing taskill /f /im explorer.exe then press enter.
  4. Type the del iconcache* then press enter to delete icon cache files.
  5. Type explorer.exe then press enter to Start File Explorer.
  6. Close the command Prompt then, restart your computer.

Mickey,

You ought to be able to restore a system image by using either of the following procedures.  I have used both procedures but my normal habit is to use procedure 2 because it operates independently of the current Windows installation so it seems to me to be less likely to suffer from whatever system problems have made me decide to restore from an image [I know I am probably being over-cautious in this respect].

During either procedure, you will see that your OS drive is referred to by a drive letter that might not be C:  Write down the drive letter that the system is using so you do not get confused later on.  The processes controlling system image recovery have their own rules about how to allocate drive letters and they can make different decisions from those that Windows makes in its normal operating mode.  This is not a problem and will not affect how the system drive is referred to after system image restoration is completed or how Windows will refer to the system drive if you end up not running the system image restoration after all.

0 Preparations

0.1  Backup your data files, any downloaded application installation files, any product keys, ... even if they are on a separate drive to the operating system**.  The system image will restore the system drive to exactly what its state was when the image was made so any of your own files on the system drive will also be overwritten with their state when the image was made so newer files would be lost and changes to older files would also be lost.  ** I have never had a system image restoration go wrong and corrupt other drives but I update my backups each time just in case.

0.2  Disconnect all non-essential external devices to keep things as simple as possible.

0.3  Connect the drive containing the system image & look at it in File explorer just to make sure it has been recognised by the computer.  The system image is in the WindowsImageBackup folder, as you already know, and the system image restoration process will only recognise images in that folder.  If you have tampered with any of the contents of that image, the system image restoration process will refuse to recognise the image at all.  Using File explorer to just browse through the folders within the image would make you add extra access permissions but doing so would not corrupt the image.

shinmila h say:

Hi prtprrt,

 

The time limit feature of Parental Control is such that it prevents particular user from logging on during specified hours. Also, for logged on users, when the allotted time ends  they should be automatically logged off.

 

More information here: What can I control with Parental Controls?

 

Since this isn’t happening here, it could be due to third party software interference. You can see one such instance here: Parental control does not lock computer after time limit expire

 

To check if any such third party software is intervening with the feature, put the computer in clean boot state and verify. Clean boot eliminates any third party software conflicts. This helps in easily determining if the problem is related to third party service or startup item.

 

Refer:

 

How to troubleshoot a problem by performing a clean boot in Windows Vista or in Windows 7

http://support.microsoft.com/kb/929135

 

Note: Ensure that you put your computer back to normal startup mode after completing the troubleshooting steps. Steps are available in the article given above.
