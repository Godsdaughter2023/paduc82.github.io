what is the difference between quick parts and macros?

I dont really get the difference between using quick parts(the building blocks feature) and macros . What additional things can macros do that the quick parts feature cannot? I use Word 2010. 
suzanne s. barnhill say:

Quick Parts and building blocks are chunks of content to be inserted in a document. They can include any type of content that can be found in a document.

Macros are a quick way to execute a series of repetitive steps--actions you would normally take manually. While they can be used to insert content, that is not their primary purpose. See http://www.addictivetips.com/microsoft-office/create-macros-in-word-2010/ for tips on recording a macro.

There are limitations to recorded macros, but they can be edited in the VBA editor, and this can give you a little familiarity with VBA that can lead to more expertise. If you are interested in editing or writing macros, see these articles:

http://word.mvps.org/FAQs/MacrosVBA/ModifyRecordedMacro.htm

wynhopkins say:

Hi Kelly

On each sheet in the workbook you should create a named range that is scoped to that sheet 

e.g. wksShowHideRange

Then this is the code to hide the rows...

--------------------------------------------------------------------------

Sub ShowHideRows()

Dim c As Range



'show all rows in range
Range("wksShowHideRange").EntireRow.Hidden = False



'hide rows with Yes
For Each c In Range("wksShowHideRange")


If c.Value = "Yes" Then

    c.EntireRow.Hidden = True

End If


Next c


End Sub

-----------------------------------

To apply this to all worksheets

------------------------------------

Sub HideOnAll()

Dim wks As Worksheet


For Each wks In Worksheets

wks.Activate


On Error Resume Next
Call ShowHideRows

Next wks


End Sub

---------------------------------------

The on Error Resume next isn't very elegant but it's there to handle the error if the named range doesn't exist on a particular sheet.
