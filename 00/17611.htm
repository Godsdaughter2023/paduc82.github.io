Can't have both and input mask and a validation rule. - Microsoft Community

bazsl1 say:

Can't have both and input mask and a validation rule. Why???

I have a short text field with the input mask 000\-000 and the validation rule LIKE "###-###". When I enter six digits I always get an error dialog with the validation text message. If I delete the input mask and enter the same six digits I do not get the validation text error dialog. Why can't I have both an input mask and a validation rule? Thanks.

dirk goldgar say:

I have a short text field with the input mask 000\-000 and the validation rule LIKE "###-###". When I enter six digits I always get an error dialog with the validation text message. If I delete the input mask and enter the same six digits I do not get the validation text error dialog. Why can't I have both an input mask and a validation rule? Thanks.

It's not that you can't have both, it's that your input mask as specified doesn't store the hyphen in the value of the field.  By default, when you use an input mask, Access only stores the values actually typed into the field, not any formatting characters specified by the mask.  So, if you leave your mask as currently specified, then your validation rule would be:

    LIKE "######"

If you really want to store the hyphen in the field, you can specify this as your input mask:

    000\-000;0

If you do that, your original validation rule will work.

For more information on how input masks work, see the help topic on "Textbox.InputMask Property".
