Excel - set default text setting of select cells in a spread sheet to - Microsoft Community

prasad gopinath say:

Excel - set default text setting of select cells in a spread sheet to Capitalize Each Word

Is there a way I can set default text setting of select cells in a spread sheet to Capitalize each word
hansv mvp say:

If it is OK to use VBA code:

- Right-click the sheet tab.

- Copy the following code into the worksheet module that appears:

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim rngText As Range
    Dim rngCell As Range
    ' Change the range as desired
    Set rngText = Intersect(Range("A2:A20"), Target)
    If Not rngText Is Nothing Then
        On Error Resume Next
        Set rngText = rngText.SpecialCells(xlCellTypeConstants, xlTextValues)
        On Error GoTo 0
        If Not rngText Is Nothing Then
            Application.EnableEvents = False
            For Each rngCell In rngText
                rngCell.Value = Application.Proper(rngCell.Value)
            Next rngCell
            Application.EnableEvents = True
        End If
    End If
End Sub

- Change A2:A20 to the range within which you want to capitalize each word.

- Switch back to Excel.

- Make sure that you save the workbook in a macro-enabled format (.xlsm, .xlsb or .xls)

- Users will have to enable macros for the code to work.

scottgem say:

First, it sounds like your database is not deployed properly. EVERY multi-user database NEEDS to be split between a back end (the tables) and a front end (everything else). The back end would be stored on a network shared drive and the front ends on each local PC. 

If this is not how its deployed then you need to change this ASAP. Sharing a single, not split file is an invitation to corruption. 

I don't know if properly deploying will fix your problem (It may or may not, but it needs to be done), but it certainly does sound like a permissions issue. Every user needs Read/Write/Modify (basically everything but Full Control) on the folder where the app resides. You can check the permissions, by going to the folder in Windows Explorer, right clicking and selecting properties, then select the Sharing tab. The list of people (or groups) that have access to the folder and what their access is will be listed. 

One other point. You said your team has admin rights. How is that accomplished? You indicate you are using Access 2013. Is the app an MDB file still using User Level Security (ULS). ULS was deprecated with the accdb format. 

daniel pineault say:
Are you saying you all use the same database file?  If so, this is the source of your headaches, and you need to split the database, relink the tables and then distribute a copy of the front-end to each user to run locally on their PC.  You never want to share an Access database it will normally lead to corruption, and other issues.
