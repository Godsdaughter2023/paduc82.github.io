daniel g. grau say:
It actually is possible to pass parameters back via AppleScripts return statement. This is how you do it:

By including the AppleScript directly within Word
You can include the whole script in Word. However, note that you'll need to consider that quotation marks passed on to AppleScript need to be double escaped. It can get complex. This an AppleScript that in turn calls some Unix utilities to access an interactive web page, get the results, clean them up and pass them to the script and then back to Word:


MyScript = "set textToEncode to do shell script ""python -c 'import sys, urllib; print urllib.q

uote(sys.argv[1])' "" & quoted form of (""" & TranslateThis$ & """)" & vbCr & "return do shell script ""curl -s -A \""Mozilla/5.0 (Macintosh; U; PPC Mac OS X 10_4_11; en-us)\"" -d \""source="" & textToEncode & ""\"" \""translation2.paralink.com/newbot.asp?dir=" & SourceLang$ & "%2F" & TargetLang$ & "\"" | egrep -oE '<textarea[^<>]*>[^<>]+</textarea>' | sed 's/<.*>\\(.*\\)<\\/.*>/\\1/' """

In AppleScript Editor it looks like this:

    set textToEncode to do shell script "python ... etc.    [i.e. do something]

    return do shell script "curl -A ... etc.                        [i.e. return some value]

Once the script code is defined in a variant or string as above, you use it as follows:

TypeThis$ = MacScript(MyScript)

The result returned by the script is read in VBA's TypeThis$ string.

You can also use a file

Sub ChooseFile()
Dim scriptToRun As String
scriptToRun = MacScript("run script ""Macintosh HD:Users:daniel:Documents:NF Daniel:Choose File.scpt""")
MsgBox scriptToRun
End Sub

where "Choose File.scpt" contains the actual Applescript code.

April 2017 and the update still not out yet!

We are facing the same issue which we solved 8 months ago, but now we have it again.... 

To those that recommend to disable "Download Public Folder Favorites" or the whole Cached Exchange, THAT IS NOT A SOLUTION. We are aware of this (read the main post). "Download Public Folder Favorites" option is required to make Public Folders acceptable to the users. Otherwise the system is too slow for a day to day environment.

Allow me to elaborate our issue and findings:

About 8 months ago, we started seeing this issue with 1 client that has 4 workstations. However, it was only happening in our client's machine. Meaning, if I added their accounts to my workstation, the issue did not happen. After a long troubleshooting, we concluded that the issue was related to MS Office only.

Our client has Office 365 Education which only includes email, SharePoint and other cloud services, but they didn't have a license for MS Office (software). How did they install MS Office?

We spoke to the client and found out that they had installed MS Office which was acquired by purchasing Office 365 Home and it installs MS Office Home and Student (Or similar version). Our license (my workstation) uses Office 365 E3 and it installs MS Office ProPlus. 

As part of our troubleshooting, we went ahead and uninstalled MS Office on one of the client's workstations, installed our license and tested the issue. The issue was gone....

Our solution at the time was to cancel the Office 365 Home and add the subscription "Office 365 ProPlus for faculty" to the Education package. Once we did this, we uninstalled MS Office from all workstations and installed the new one (ProPlus). Issue solved.

Two days ago, 4/4/2017, we start facing this issue again. It seems that the client's MS Office software updated to version 1609 (Build 7369.2120) and the issue started happening again. My workstation also has this version and is presenting the same issue. However, one of the workstations is still using version 16.0.6962.2092 and still works!!!!

For the love of god, we got updated to make things worse!!!!

For the love of god, did you read the original question and/or my post?

For everyone else, I solved the issue on my end.

It seems that an update for Outlook was actually release in January. However, those marked as "Deferred Channel" will not get this update for a while. Meaning, you have to switch your channel to "Current Channel". MS Office version 1702 (Build 7870.2031) is actually working in 6 workstations that I tested. The workstations are a mix of Windows 7 Pro and 10 Pro.

After switching to Current Channel and updating MS Office.

How to switch to a different channel?

Important: Make a backup of your registry before proceeding.

I follow the steps outlined on this Microsoft document: https://support.microsoft.com/en-us/help/3185078/how-to-switch-from-deferred-channel-to-current-channel-for-the-office-365-suite

Once you switch to the new channel, do not go back. If you do, the system reverts the updates and you will have the issue again. So, wait until this update is release to the "Deferred Channel".

Please be aware that I did not run the auto-fix application nor the VB script. I simply followed the locations mentioned on the script and made all the changes manually. Then, I ran the command "%CommonProgramFiles%\microsoft shared\ClickToRun\OfficeC2RClient.exe" /update user from search bar on Windows.

I hope this helps someone out there.
