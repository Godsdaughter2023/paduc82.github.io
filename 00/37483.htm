tryx3 say:

I recommend this approach for hardware that does not work in your new Windows OS but did work in earlier Windows versions.

[Updated 5th June 2016]

Where to get hardware drivers

1.1  Windows Update or Device manager, Update driver.  This is not a waste of time; Windows 10, just for example, is likely to install a basic display driver and not attempt to install a decent one until you run Windows update.

1.2  The PC maker for hardware included with the PC at purchase.  Normally, you would use that part of their support site that deals with your computer model but, because they might have used the same component in later models, a general search of their support site for the device name without mentioning your model might reveal the availability of updated drivers that could be worth trying.  In addition, search any user forums for the PC make / model as they might identify solutions that have worked for other users.

1.3  The OEM for additional hardware [for example, Intel® Driver Update Utility  - this only assists with currently-supported hardware and, for older hardware, it ends with a link to the in this way.]  OEM hardware user forums might also provide good advice.

1.4  manager in your previous version of Windows because that would be best search term to use.  You can click on the name of each driver in the search results list to identify details about that driver [the search results list itself does not always state, for example, whether an entry is for a 32 bit or a 64 bit driver].  See also this step-by-step guide to using the Microsoft update catalog.

2.1  These searches might identify a driver that is described as being suitable for your hardware on your new OS [including its type - 32 / 64 bit].  In general, if your new OS is 64 bit but all you can find is a 32 bit driver then it would not be reckless to try it [conversely, do not even attempt to install a 64 bit driver in a 32 bit system]. 

2.1.1  If the download is a zip file, cab file or other compressed file then extract its contents into a folder of their own.  Then browse the folder to see what's in it.

2.1.2  If the driver has been made available as an exe file,

  •                right-click, select Properties & "Unblock" the file
  •                right-click on it again & select Run as Admin. 

2.1.3  For other types of driver file, open Device manager as Admin and select Update software then select Browse my computer ... , Have disk.

2.2  If all you can find is a driver for an older OS then you might just have to try it to see if it works or not.  In general, if your new OS is 64 bit but all you can find is a 32 bit driver then it would not be reckless to try it [conversely, do not even attempt to install a 64 bit driver in a 32 bit system]. 

2.2.1  If the download is a zip file, cab file or other compressed file then extract its contents into a folder of their own.  Then browse the folder to see what's in it.

2.2.2  If the driver has been made available as an exe file,

  •                right-click, select Properties & "Unblock" the file
  •                right-click, select Properties, select the Compatibility tab and choose the OS that the driver was designed for, OK, OK
  •                right-click on it again & select Run as Admin

2.2.3  For other types of driver file, open Device manager as Admin and select Update software then select Browse my computer ... , Have disk.  For these types of driver, you'd have to rely on Windows detecting during the installation process that something might be wrong and offering to complete / repeat installation in a compatibility mode.  There is also some advice at Make older programs run in this version of Windows

3  You might have to take some precautions to prevent Windows update from overwriting the device driver with a later one that does not work properly.  There is no method for finding out in advance whether or not a driver that Windows update wants to install is going to improve matters or undermine all your efforts so you might have to search this forum, and relevant computer maker or OEM forums, for guidance beforehand.  Personally, I keep copies of drivers for my problem devices on my computers ready to reinstall after Windows updates if necessary - I keep these drivers ready despite having taken the precautions in paras 3.1, 3,2.

3.1  If your new OS is Windows 7 then you can change Windows update settings to improve your degree of control - Control panel, [View by - Small icons, if necessary], Windows update, Change settings, Important updates - Check for updates but let me choose ...  You can then review all updates and, if there is a device driver update that you do not want, right-click on it and select Hide update.  If you ever want to undo the hidden property then look two lines below Change settings and you'll see Restore hidden updates - this does not commit you to installing all the hidden updates as you'll be able to review them and re-hide any that you decide you still do not want.

3.2  If your new OS is Windows 10 then refer to paras 5.1 - 5.3 of my 13th August 2015 post in Automatic update [do not scroll or move your cursor as the page loads or you'll move off the post that this link points to].  That thread and the rest of my post in it are about controlling internet data usage not about controlling device driver updates.

4  If you still have problem devices then search this forum for the device name to find out if anybody has posted guidance on using it with your new OS.  If not then post a question seeking help.  In general, don't post onto an old thread unless your device, your OS & your problem are exactly the same as that identified in the original question.  Even then I would think twice about posting onto an existing thread if it had more than a single page of posts as things can get confused very easily if multiple users report problems within any given thread - if in doubt, opt for a new question.

4.1  Use a question title such as "Device name - Upgrading {from Windows XP/Vista/7/8/8.1} to Windows 7/10 [32/64 bit] - suitable device driver cannot be found" [since people look through lists that only show question titles, your question title needs to attract the attention of people who know about your specific device for your new OS]

4.2  Personally, I  number my paragraphs & put blank lines between them as the easier they are to read, understand & refer to then the better the responses are.

4.3  In the problem description, explain

 - that the computer make-model is ....,

 - that the device is ...,

 - that the problem is .. [what errors have occurred, what error messages have appeared, what symptoms are apparent],

 - what other circumstances & recent hardware / software changes might be relevant,

 - that you have tried {name, version, date} drivers that you found in the ... support website,

 - that you also tried the ... websites but could not find any useful drivers there [or you'll just get other people suggesting that you try the same websites I have suggested above].

5  Personally, I would not bother with any third party driver providers as the drivers must come from the original hardware maker source in the first place or they would not exist so I don't understand just what these third party driver providers are doing.  Some might be trying to make money honestly whilst others might be distributing hidden malware.

6  If it is any encouragement, I upgraded both my eleven year old XP  & my six year old Windows 7 to Windows 10 and the sources above succeeded for me [I ended up with a mixture of drivers - many Windows 10, some Windows 7, some Windows XP & some Vista].
