davede11 say:

Reading between the lines, it seems maybe you were using a pop3 account in Windows Live Mail, and now an IMAP account in Outlook. IMAP may be the default when you create the account.

With pop3, emails are downloaded and stored in file(s) on your PC. In the case of Windows Live Mail these would be a separate *.eml file relating to each email I think.

In Outlook, pop3 emails would be stored in a single Outlook.pst database file, which would also contain your contacts, calendar entries and tasks. Archives would be stored in one or more archive.pst files, depending on your archive settings. With pop3, you can change a setting to control whether mail you have already downloaded remains on the server, or is deleted from the server as it is transferred.

In Outlook, IMAP emails would be stored on the server and on your pc in a single Outlook.ost file, along with calendar etc. as above, and the server and PC versions always updated in step with each other. If you were working offline, you'd be looking at the data in the Outlook.ost file which is the local copy of the server data.

If you wanted to work as before, you can create a new Outlook.pst file (using_any_name_you like.pst. You set that as your default mail file, and create a linked pop3 type mail account with the required settings to send and receive mail.

You should be able to import existing mail from an Outlook.ost file, or from a folder of .eml files from Live Mail. Likewise contacts and calendars from non-Outlook apps.

Exactly how you do that will vary according to Outlook version, but the help will tell you how to do it.

davede11 say:

Sound like your objective is the same as mine. All emails saved on my PC, so I have them offline on my laptop and can also back them up.

Most ISPs support both pop3 and IMAP. You specify which you want to use when you set up the account in Outlook. If you let Outlook do that automatically by just choosing your ISP, it will default to IMAP. These are primarily the protocols used to transmit the data, but also determine where the data is stored, on the server or on your PC. In the case of Outlook, IMAP emails are stored in both places, so you can access them offline, but if you delete them, they will be deleted from both places.

There's no particular reason to delete emails manually from the server. If you don't, you can access them on a phone or tablet as well as in Outlook on the PC.

Archiving is something else entirely. Just a means of moving old stuff from your current Outlook storage to an archive file, purely to reduce the size of the active Outlook file and hide stuff you may no longer need all the time, but may want to keep anyway. I keep 20 years worth of emails and attachments in my active file as I want to be able to search them seamlessly for various business reasons. My file is under 1Gb and well within Outlook performance guidelines. Obviously, I delete any I don't need to keep.

Outlook always uses a database, in which all emails, calendar entries and tasks are saved. .pst for pop3 email and .ost for IMAP. The default for Office 365 is IMAP and .ost. You can change it, but may need to plan how to migrate any existing stuff across, and import your Live Mail emails. Not especially difficult, but steps for another post if you need help. I can recommend an online support service to do it for you (£60 ish for an hour), they also produce a number of Outlook related utilities for Google to Outlook calendar sync, backing up, etc. and some free ones too. I use them to keep my phone, tablet and PC calendars in sync, and to backup my Outlook files several times a day.

bob jones aka: cybertaz say:
A leading single quote (apostrophe) is a "code" character used to designate the cell content as Text. It isn't recognized as a literal character.

In order to have the leading apostrophe appear in the cell & be treated as a literal, you have to type 2 of them -- i.e., ''Name'

Be sure to type the ' twice rather than using the double quote mark (").
