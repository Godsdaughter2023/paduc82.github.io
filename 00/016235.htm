I'm not guaranteeing that this will solve your problem, but it's worth a try:

  1. Back up the registry using the steps on this page: How to back up and restore the registry in Windows
  2. In Regedit, open Computer\HKEY_CURRENT_USER\Software\Microsoft

  4. Right-click on Office and choose New>Key. Call it 16.0.

  6. Right-click on PowerPoint and choose New>Key. Call it Options.

toddallcock say:

Another possibility is how the files are delivered.  I use Orb, a media server installed on my PC which allows me to access my media over the internet.  Orb is setup to deliver it in .wmv/.wma format, and I used it with my old Windows Mobile phones and PDA devices for years.

 

When I switched to Windows Phone, I got the "sorry, we can't play this file" message when accessing my media on Orb.  The problem isn't the format, but the delivery.  Orb (and, I'm guessing, ADrive) streams Windows Media files as ".asx" files- these are Windows Media "Playlist" files.  (They are little .xml text files that contain the local or network locations of the files you are streaming.)  While WP7 supports .wma and .wmv, it doesn't understand .asx, and you get the "can't play" message.

 

 

 

   

entegy say:
