Welcome to Microsoft Answers.

This issue may occur if the user profile was manually deleted by using the command prompt or by using Windows Explorer. A profile that is manually deleted does not remove the security identifier (SID) from the profile list in the registry.

If the SID is present, Windows will try to load the profile by using the ProfileImagePath that points to a nonexistent path. Therefore, the profile cannot be loaded.
This can also be an issue with the user profile entering into a backup state.

Follow either Method 1 or Method 2 whichever is applicable to resolve this issue.

Important : This section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it.

Methods for backing up your files
http://windows.microsoft.com/en-US/windows-vista/Methods-for-backing-up-your-files

Method 1:

1.    Logon to another administrator account.
       If you don’t have one, go to Method 2.
2.    Open the Start menu.
3.    In the start search, type regedit and press Enter.
 If prompted, either click on Continue or enter the password for the Administrator account.
4.    In regedit, navigate to:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList

5.    In the left pane, look for the S-1-5..... folder (SID key) that has .bak at the end of the numbers.
6.    For Two S-1-5..... folders (SID key) with the same Number :
This is if you have two S-1-5..... folders (SID key) with the exact same numbers.

            A)    In the left pane, right click on the first S-1-5..... folder (SID key) that does not have .bak at the end of the numbers and click Rename.
            B)     Add .bk to the end of the numbers.
            C)    In the left pane, right click on the second S-1-5..... folder (SID key) with .bak at the end of the numbers and click Rename.
            D)    Remove only .bak from the end of the numbers and press Enter.
            E)    Now go back and Rename the first one with .bk to .bak now at the end of the numbers and press Enter.
            F)      Go to step 8.

7.    For Only One S-1-5..... Folder (SID key) with .bak:

            A) In the left pane, right click on the S-1-5..... folder (SID key) with .bak at the end of the numbers and click Rename.

            B) Remove only .bak at the end of the numbers and press Enter

8.    In the right pane of the one without .bak now, right click on RefCount and click on Modify.

NOTE: If you do not have RefCount, then right click on a empty space in the right pane and click New and DWORD (32 bit) Value, then type RefCount and press Enter.

Type 0 (number) and click on OK.

9.    In the right pane of the one without .bak now, right click on State and click on Modify.

Type 0 (number) and click on OK.

10.   Close regedit.
11.   Restart the computer.

12.   See if you can logon now.

This should work.

lemp say:

Once you "empty" the Recycle Bin, the space occupied by those files is available for use. 

That is, the 1s and 0s of the files are still there (which is why data recovery software can work), but the operating system (Windows) will re-allocate those locations -- i.e., put a new pattern of 1s and 0s in those locations -- when there is a need to save something (which is why data recovery software is less and less likely to work the longer you wait to use it).

If you choose to backup your iPhone to your own computer rather than to iCloud, generally, only the most recent backup is kept.  Also, if you have (or have had) more than one iDevice connected to this computer via iTunes, there may be a backup for each device.

There are some circumstances when you may end up with more than one backup per device.  These circumstances have changed over the past several versions of iTunes, and I'm not sure of what causes multiple backups these days.  I've been unable to find any documentation that explains how to prevent the current version iTunes for Windows from overwriting an iPhone backup in order to create multiple backups.

In any event, open iTunes and go to Edit > Preferences.  Select "Devices."

You'll see all of the backups saved by iTunes.  Select any that you want to delete and click the "Delete" button.

You certainly can delete backups for iDevices that you no longer use.  Whether you want to keep more than the most recent backup for any iDevice that is in current use is entirely up to you.  In general, there's no need to do that.

bill smithers say:

Why Deleted Files Can Be Recovered, and How You Can Prevent It

When you delete a file, Windows removes the pointer and marks the sectors containing the file’s data as available. From the file system’s point of view, the file is no longer present on your hard drive and the sectors containing its data are considered free space.

However, until Windows actually writes new data over the sectors containing the contents of the file, the file is still recoverable. A file recovery program can scan a hard drive for these deleted files and restore them. If the file has been partially overwritten, the file recovery program can only recover part of the data.
