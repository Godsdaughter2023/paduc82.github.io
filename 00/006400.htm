Okay, you didn't mention captions, and I do advise using frames to keep captions with figures if you must wrap them. With a frame, you actually don't get the Advanced Layout options. If you used a text box instead, you would get them. Many of the reasons for using a frame instead of a text box are no longer applicable in Word 2010. For a text box or wrapped graphic, you can get to them this way:

 

Text box: Even if the wrapping is In Line With Text, if you right-click on the border, you'll get More Layout Options... on the context menu. Graphic: Right-clicking gives you Wrap Text > More Layout Options... on the context menu, and the contextual Picture Tools | Format tab has the same Wrap Text | More Layout Options... in the Arrange group.

 

The Text Wrapping tab of the Layout dialog allows you to change the wrapping from In Line With Text to Top and Bottom (or some other style). Once you have done that, the Position tab offers more options. "Lock Anchor" just keeps the anchor of the frame, text box, or graphic from being moved to a different paragraph. This just not prevent it from being moved anywhere on the page--or to another page if the paragraph it's anchored to moves to another page. To "lock" the position of the object, you can set its position relative to the page, the margins, or the paragraph it's anchored to. You can experiment with different placement to see what works best for you. All of this, of course, applies only to text boxes and "pictures," not frames.

jllatham say:

Veera,  Here is a macro that will do the basic work of saving the individual sheets as .txt files.  They are actually "tab delimited" text files, which means that the data in each column on a row is written to the text file with a tab character separating the individual column's data for each row.  An option is to save them as comma separated data files, depending on what is needed at the other end that is opening and reading the text files.

You can either put the macro into your workbook so it's always available, or put it into a separate workbook that you would need to have open when you want to perform the operaiton: have both workbooks open, select (activate) the one with the data in it and then use Tools --> Macro --> Macros to identify the macro and [Run] it.  It works with whatever workbook is currently active.

My question is: did you want this to be an automatic process that would take place once per hour without any operator action at all?  That could be done.  Right now the code DOES NOT do the upload to FTP site, but it does create the files and make them ready to be uploaded.  The code goes into a regular code module and instructions for putting it into one in a workbook can be found here: http://www.contextures.com/xlvba01.html#Regular

The code:

Sub SaveSheetsAsTextFiles()
  Dim originalFileName As String
  Dim originalSheetName As String
  Dim anyWS As Worksheet
  Dim rootPath As String
  Dim newName As String
  Dim theYear As String
  Dim theMonth As String
  Dim theDay As String
  Dim whereAmI As String
 
  'you can assign a path to this like
  ' rootPath = "C:\folder1\folder2\folder3\"
  'but this uses path to folder where the
  'Excel file is stored
  rootPath = ActiveWorkbook.Path
  If Right(rootPath, 1) <> Application.PathSeparator Then
    rootPath = rootPath & Application.PathSeparator
  End If
  originalFileName = ActiveWorkbook.FullName ' path and name!
  theYear = CStr(Format(Year(Now()), "0000")) ' 4 digit year
  theMonth = CStr(Format(Month(Now()), "00")) ' 2 digit month
  theDay = CStr(Format(Day(Now()), "00")) ' 2 digit day
  'remember where we are in the workbook
  whereAmI = ActiveSheet.Name
  'turn off alerts to the user
  Application.DisplayAlerts = False
  'prevent screen flickering and improve speed
  Application.ScreenUpdating = False
  For Each anyWS In ActiveWorkbook.Worksheets
    'must actually activate the sheet
    anyWS.Activate
    originalSheetName = ActiveSheet.Name
    'set up new name for the text file

    'change the sheet name back to its original name
    anyWS.Name = originalSheetName

Hi Marte,

OneNote supports Optical Character Recognition (OCR), a process that recognizes text shown in pictures and PDF printouts. Depending on the complexity, legibility, and the amount of text shown in the printout you’ve inserted, you may not be able to search for a word immediately. If OneNote is still reading and converting the text in the printout, wait a few moments and then try again.

  1. Create a new notebook and insert a small PDF printout.
  2. Please make sure the printout image is sharp and clear.
  3. Try editing the page that contains the PDF printout you're having trouble with, type something somewhere on the page, it will cause the indexing service to detect that there was a change and look to see if there are any images needing OCR.
