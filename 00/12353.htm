PROCV and #N/D are terms used for a non-English version of Excel, perhaps Brazilian.  See http://office.microsoft.com/pt-br/excel-help/procv-HP005209335.aspx.

The English equivalents might be VLOOKUP and #N/A.

You can avoid the #N/D error by using the Brazilian equivalent of IFERROR.  In English, I would write:

=IFERROR(VLOOKUP(...), "")

where "..." represents your parameters to VLOOKUP (PROCV).

bob jones aka: cybertaz say:

Are you sure about that?

What I see is that using the Highlight command applies the last used Highlight color. IOW, a keystroke assigned to the Highlight command only removes highlighting if the highlight color that's currently applied is what was last used. If the text is currently Highlighted in Yellow but the last color applied was Turquoise, using a keystroke when selecting the yellow text changes the Highlighting to Turquoise & deselects the text. You then have to select the text a 2nd time & use the keystroke again to toggle the Highlighting Off.

IMO, it's more efficient to just select the passage then choose None from the Highlight tool's palette... Unless you can come up with a macro such as:

Sub NoHighlight()
'
' NoHighlight Macro
'
'
    Options.DefaultHighlightColorIndex = wdNoHighlight
    Selection.Range.HighlightColorIndex = wdNoHighlight
End Sub

NB: This macro is just an example that was quickly recorded. It has at least 2 shortcomings;

  1. It leaves the text selected, &
  2. It leaves the Highlight color set to None, so a keystroke assigned to the command will not apply a Highlight color until you next select one manually.

James: Highlight color cannot be applied/removed by a Style because it isn't a formatting attribute. A comparable effect can be controlled by Styles using the Format> Borders & Shading - Shading formatting property.

rich michaels - mvp (greatcirclelearning.com) say:

The highlighting function is a toggle (on/off).

So if you assign a custom keyboard command such as Cmd+Opt+H to the Highlight function then whatever the setting is currently for the selected text (Red, Green, Yellow, etc) when you select the text again and press the custom keystroke, the highlight function toggles to the opposite... which is Off.

In other words quoting the great philosopher Mr. Miyagi... waxy on/waxy off :-)
