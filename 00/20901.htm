If I wanted to flash the EMMC and the OS (which really should be done every time.. rather than just doing one) I would:

 

power off

press + hold U and power on

wait for USB icon, then let go of U

flasher -F os.bin -f

wait for that to finish, check the phone is still sat at the dark logo screen with the USB icon showing

flasher -F emmc.bin -f -R

Wait for that to finish and the phone should restart / start booting.. If it doesn't start booting (but flasher finished without reporting any errors) I wouldn't panic.. I'd turn off (may be pull the battery out too) then power on and the phone should be wiped clean!

 

the -R option means flasher will reboot the phone after flashing has finished.. if you leave it off, the phone should just sit there, waiting for you to run flasher again

If this shared mailbox is in a hybrid environment (Office 365 & Exchange 2010 on-prem), what is the recommended approach to convert it in user mailbox.

Can we do it in cloud and does that sync attributes on -Prem ?

or we need to move back shared mailbox on-prem and convert it to user mailbox ; then migrate it to cloud ?

* Moved from Community Participation Center

nicu simion say:

Hello,

In a hybrid scenario it is not recommended to perform the mailbox conversions directly from cloud side, as there are few specific attributes which are not written back from cloud, such as RecipientDisplayType, RecipientTypeDetails or RemoteRecipientType. As you can imagine, they differ from a user mailbox to a shared mailbox.

To avoid any potential issues, I advise to move first these mailboxes to on-premises, here to perform the mentioned conversions and after that you can move them back to cloud, so AAD connect can sync properly new Exchange attributes from local AD to the cloud.

teylyn say:

I think Bernard identified the problem.

Adding rounded numbers, as you do in the second screenshot of the file produces a different result than adding numbers that are just formatted to show two decimals.

Column F has the real value but is formatted to show only two decimals. Excel will apply rounding for the display, but the underlying value may not be exactly the same.

Column G shows the value of column F in the General format, which displays as many decimals as needed for the precise value.

Column H uses a formula to round the value of column F to two decimals. Those numbers actually have different values than column F.  In your comparison screenshot with individual digits in each cell you used basically the same numbers as in column H below. 

That explains the difference in the sums.

Excel calculates correctly, but you need to be aware of formatting. It may mask the true value of a cell.
