scottgem say:
Class and Criteria may be reserved words so yes using them as object names could cause a problem. 

But the criteria should be:

Forms![Class]![ClassType]

Since you are referring to a control on the main form, you don't need to refer to the subform. On the flip side, if you were modifying the RowSource of the combo on the subform you would so you might need to do something like this in the After Update event fo ClassType:

Forms![Class]![Result].Form.[Criteria].Rowsource = "SELECT ... WHERE field = " & Forms![Class]![ClassType] & ";"

How to use handwriting recognition in Microsoft equation editor. - Microsoft Community

How to use handwriting recognition in Microsoft equation editor.

I have a writing pad and want to write an equation in the microsoft word 2007. Can I use Microsoft equation editor?
justicestats
afwings say:

The short answer is "yes". Since you're using Windows 7, you can use theWindows 7 Math Input Panel (MIP) to handwrite your equations. Tapping the Insert button puts the equation into whatever is beneath the MIP, as long as it can understand the format the MIP uses. Word 2007 does understand this format, so it should be an easy go.

To find the MIP, click Start and type math input in the search window. Math Input Panel will appear in the results in a couple of seconds.

If you already have MathType, it works with the MIP as well , and you can insert handwritten equations* into MathType. This is handy, because if you also use PowerPoint or other software that doesn't support the MIP, you can insert the MIP equation into MathType, then insert the MathType equation into PPT (or whatever). When inserting handwritten MIP equations into Word 2007 or 2010, you have the choice of inserting it as a Word 2007 equation or a MathType equation.

*(I'm using the phrase "insert handwritten equations", but when you tapInsert , it goes into Word as a nicely-formatted math equation.)
