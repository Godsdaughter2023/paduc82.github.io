jeeped say:
It seems that the macro recorder is reverting to the old ExecuteExcel4Macro command in order to set the number format for the CF rule. This is neither necessary or desirable. Taking your method over to a full VBA sub would look like this,

Sub format()
    With ActiveSheet
        .Columns("L:L").NumberFormat = "General"
        With Range("L2:L500")
            .FormatConditions.Delete   ' remove all CF rules from L2:L500
            .FormatConditions.Add Type:=xlExpression, Formula1:="=$A2=""Trunk"""
            .FormatConditions(.FormatConditions.Count).SetFirstPriority
            .FormatConditions(1).NumberFormat = "[h]:mm:ss"
            .FormatConditions(1).StopIfTrue = False
        End With
    End With
End Sub

Now that deletes any existing CF rules in the L2:L500 range before adding the new rule. If this is not there, running the macro multiple times will create duplicate rules.

Some fonts in the PPT dropdown menu display overlapping/stacking characters on a slide. Some of these are fonts from imported presentations, with no corresponding TTF/OTF in the Windows system font folder. Is this just a way for PPT to verify the missing font? Word 2019 signifies this missing link by maintaining the previously selected font, but Powerpoint seems to display the characters on top of one another.

I'm wondering how to refresh the list of available fonts in the dropdown menu to represent fonts that are installed locally.

john korchok say:

I believe these are 2 separate issues:

  • A video driver issue that is displaying fonts without their normal metrics, so they stack. Please open File>Options>Advanced>Display and check Disable hardware graphics acceleration and/or check with your computer manufacturer's web site to see if they have a recent Windows 10 video driver for your model.
  • Font substitution. PowerPoint normally displays the font name for the expected font of selected text. It's possible that the font whose name you see is embedded in the presentation. If the font is not embedded, then PowerPoint is substituting a font with a similar appearance in it's place, but doesn't tell you what font that is. You should be able to see the list of locally installed fonts by closing all presentations except for a new blank one
