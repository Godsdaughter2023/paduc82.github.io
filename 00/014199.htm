How filter only odd or even number? - Microsoft Community

bruno :o) say:

How filter only odd or even number?

How to display only odd or even number?
j.e. mcgimpsey say: You don't give a lot to go on - is this only one number? or many?  What type of filter?

If you have a list of numbers, you can do an Advanced Filter. Assume that the values are in A2:A100 and you only want to see even numbers. In a non-adjacent column (say, F) enter this in row 2 (leaving row 1 blank)

    =EVEN(A2)

Select a cell in your data range in column A, choose Data/Filter/Advanced Filter, and enter A2:A100 in the List Range box, and F1:F2 (BOTH cells) in the Criteria Range. Click OK.

If you want to do an AutoFilter, then you can use a helper column. With the same scenario as above except wanting to see odd numbers, in B2 enter 

   =ISODD(A2)

and copy down to B100. Then with a cell in the range selected, choose Filter/Autofilter, click the dropdown for column B, and choose TRUE as the criterion.

Managing activeX combobox listfill range - Microsoft Community

dan dndtrading say:

Managing activeX combobox listfill range

Hi guys,

I was wondering what is the easiest method to manage a comboboxes listfillrange.  Basically I have a defined name range as the list fill range.  I want the user to be able to not only select from that list but also add new items to the list and delete items from the list.  After each add or delete the modified list with be resorted alphabetically.

Thanks,

Dan

kergga say:

Hi Dan-

The ability to add/delete items to the listrange from the combobox itself is not (to my knowledge) built-in functionality for end-users. Here are some recommendations:

Adding items:

1. First and foremost, unless you have run into a problem with no other workaround, I recommend using the forms control combobox and stay away from all activeX controls.

2. You don't mention whether you are using the combobox on a worksheet or userform. I would suggest a userform, where you could use a separate textbox for adding new items, then add it after your named range, and use the worksheet functionality to sort the list (all triggered by a command button after a new item is entered)

3. If you aren't already, I suggest making your named range dynamic, so you don't have to change the range every time a new item is added

If you are using the combobox directly on a worksheet, then the approach would be similar, but you might use a specific cell to enter new items (somewhere adjacent to the combobox location). Then use worksheet_change or a commandbutton to trigger code to add the new item to your list.

Deleting items:

When an item is selected, you can return the ID (listID?) of the selected item, and use that to delete it from your source list. You will need to trigger this with a commandbutton or similar trigger

HTH

davem121 say:
Hi Rajesh, here is a sub you could probably adjust to suit your needs, with it you can open the PDF file and extract its contents using the Adobe library. The code relies on having Adobe already installed ion the PC and uses a dll that is installed as part of that

Function getTextFromPDF(ByVal strFilename As String) As String
Dim objAVDoc As New AcroAVDoc
Dim objPDDoc As New AcroPDDoc
Dim objPage As AcroPDPage
Dim objSelection As AcroPDTextSelect
Dim objHighlight As AcroHiliteList
Dim pageNum As Long
Dim strText As String

strText = ""
If (objAvDoc.Open(strFilename, "") Then
Set objPDDoc = objAVDoc.GetPDDoc
For pageNum = 0 To objPDDoc.GetNumPages() - 1
Set objPage = objPDDoc.AcquirePage(pageNum)
Set objHighlight = New AcroHiliteList
objHighlight.Add 0, 10000 ' Adjust this up if it's not getting all the text on the page
Set objSelection = objPage.CreatePageHilite(objHighlight)

If Not objSelection Is Nothing Then
For tCount = 0 To objSelection.GetNumText - 1
strText = strText & objSelection.GetText(tCount)
Next tCount
End If
Next pageNum
objAVDoc.Close 1
End If

getTextFromPDF = strText

End Function

What does this do.. #This Row , #All, #Header - Microsoft Community

ft77777 say:

What does this do.. #This Row , #All, #Header

Im trying to work on a report and I have a formula =IFERROR(VLOOKUP([[#This Row],[Column B]],Project A[[#All],[Column E]],Project C[[#Header],[Column F]])False)),"") What does.. #This Row, #All, #Header.. this mean and How is this created?  Are these macro reference or are these special MS Excel commands?

g north say:

Hi

These are references to a range formated as a table.

[# This Row].[Column B] = is the equivalent to a cell reference or $B2, assuming you ar on row 2.

ProjectA[[#All],[Column E]] = is the evivalent of $E2:$E100, assumning there are 100 rows in the table. However writing this way means the formaul will automatically include any new rows added to the ProjectA table.

Project C[[#Header],[Column F]] is equivalent to $F$1, the heading at the top of the column, in a table named Project C.

The above assumes that A1 is the first cell in the table, however it could be anywhere in your workbook, there's no need to prefix these refernces with a sheet name like conventional cell references.

This link explains them in more detail.

http://office.microsoft.com/en-us/excel-help/using-structured-references-with-excel-tables-HA010155686.aspx?CTT=5&origin=HA010147120
