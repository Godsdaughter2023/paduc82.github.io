Getting Started with co-authoring

Start by uploading a workbook to a supported, shared cloud location such as SharePoint Online, OneDrive or OneDrive for Business.

Share your workbook with others- If you clicked the Share button, people will receive an email message inviting them to open the file. cortig say:
OK, now that the thread has been split, let me try to elaborate.
This error is a sign of a serious corruption that the database utility typically cannot fix (at least on my Mac it failed miserably).
On top of this, it affects the database daemon running in the background and makes Outlook crash immediately no matter what you attempt (e.g.: create a new identity: Outlook will still crash).

One thing you need to do is to quit the daemon to that it can unload the corrupted code. Use /Applications/Activity Monitor to find and quit the process.

After that, the best option is to restore a backup of a previous version of the database if you have one.
In my case, the crashes were coming from a corrupted contact on my exchange server, so the corruption was eventually coming back after a sync with the server. I figured it out because I tried to export all my data, and a drag and drop of the contacts to the desktop created 800 vcf and only 799 could load into the System address book.

Identifying the source of the corruption is really hard. I got lucky in my case, but I have no systematic procedure that would help you identify it.
If you have an Exchange account, I would recommend deleting it from Outlook and re-adding the setting to let Outlook delete any local store of the corrupted data and hopefully redownload cleaner versions from the server.

It is also possible to delete (or set aside) portions of the Outlook database from the Finder. Obviously the data will be “lost” to Outlook but it should force Outlook to recreate its global index and potentially allow you to get back on your feet.
I know it’s a very tedious trial and error process. You also have to remember to quit the database daemon after any crash (see above).

Good luck with that!
