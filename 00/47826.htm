lead3 say:
Go to Disk Management and see if the D: drive is shown to the right of the C: drive. If so the right click on the D: drive area and select delete volume.

This will create unallocated space.  Then right click on the c: drive area and select extend volume. (this only works if D: is shown to the right of C:)

This also assumes that you have no data on the D: drive that you want.  As always, backup any data that you can't affort to lose.

Application hangs & BSOD's - Microsoft Community

palcouk say:

Application hangs & BSOD's

Win7 Pro 64bit, clean install March on all compatible HW, run 24/7 since then without probs until 10 days ago.

Various apps (Office and others started hanging) Office diagnostics showed no prob. Not able to end process in Task Manager, only by reboot. Intensive HD & Memory tests showed no probs. No new HW or software installed, only makers drivers installed. Virus / malware free.

Started getting BSOD's, WinDbg identified a afcdp.sys driver, an Acronis TI sys file. Disabled all Acronis in Services & Startups, that cured BSOD's but apps still hung from time to time. Problems intermittant and not able to replicate. Sys performance was iffy.

Decided to undertake a repair install of win, Repair failed, Compatibility report produced stated "Storage controllers: AMD AHCI Compatible Raid Controller" need updating. I had however installed driver from usb during the repair process, and the existing driver was the latest makers driver. In control Panel, update drivers, no later driver was found.

I'm now thinking the motherboard has a problem, ie controller failing?

palcouk say:

Finally figured it out. I was running mirror raid on the motherboard controler, seems this was the problem. Broke the raid, rebooted with raid disabled in the bios, was then able to run a win repair, which took several hours. System now running fine.

I have a suspicion that the mirror was out of sync, although no problem reported, something I've seen on another onboard raid sys some years ago, when the maker Asus admitted there had been problems.

Gigabyte GA-MA790XT-UD4P was using onboard AMD SB750 South Bridge Controller. Bios and drivers updated

trung n nguyen say: Hello Bob, I'm assuming you have multiple devices, one is Windows 8 and another is Windows 8.1. If so, for each of the accounts on the Windows 8.1 that you are having issues, please try the following:
  • Log in to https://familysafety.microsoft.com/ using your Microsoft Email account.
  • Find the user that is correspond to the computer account on the Windows 8.1 that you are having issue with.
  • Click on "Game Restrictions".
  • Under Game Restrictions, click on "Game list".
  • Here, you can Allow or Block the Apps/Games that you want. These settings will take affect on the Windows 8.1 account(s) only. If you chose to select "Use game rating" as the setting, this setting will be based on the game rating level you have selected for your child under the "Rating" section.

In other words, you will have to manage these apps in two places because you have Windows 8 and Windows 8.1. For Windows 8, you would use the "App restrictions" settings. And for Windows 8.1, you would use the "Game restrictions -> Game list" settings. Alternatively, you can manage these Windows Store Apps in one place under "Game restrictions -> Game list" by upgrading your Windows 8 device(s) to Windows 8.1. It is free and you can do it through the Windows Store on your Windows 8 device(s).
