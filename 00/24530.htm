How do you sort a column in a table if you have a first and last name - Microsoft Community

madeline l say:

How do you sort a column in a table if you have a first and last name separated by a comma?

I'm trying to sort a column in a table on word and it contains first names and last names of people separated by a comma. ( i.e. Last Name, First Name). I have been instructed to sort this column by last name, then by first name. Here are the exact directions:  

1.     Select all the employee names in the first column, but do not select the column headings. Sort the table data based on the last and then the first names of the employees. (Hint: Open the Sort dialog box and then open the Sort Options dialog box. Under Separate fields at, select Commas and click OK. Then in the Sort dialog box, select a Sort by of Column 1 with Field 1 in the Using box, and a Then by of Column 1 with Field 2 in the Using box.)


My issue is that when I highlight the names and go to the sort options, it won't let me select "separate by commas." And for macs, there is no field 1 or 2 option. 

Any suggestions?
john mcghie say:
Hi Madelaine:

Ooohhh, that's nasty :-)  You're right, Mac Word doesn't enable the field sort mechanism in a table.  You need to adjust the method.

You can do this two ways:

1)  Convert the table to text, and sort as described in the question.

2)  Convert the table to text, split the column, convert back to a table and sort as a table.

Which one you do would depend on how many columns you want in the result.

Let's go with method 1 first.

a)  Click in the table
b) Go to Table>Convert>Table to Text
c)  In the dialog that appears, choose "Separate with Tabs"

The table is now expressed as ordinary text, with the columns separated by tab characters.

You can then select the text you converted, and use their method.  The "Field 1", "Field 2" prompts will then appear on the first dialog, and with any luck, it will spot the commas and fill those in automatically for you in the Options.

When you have done the sort, simply select all the text again and use Table>Convert>Text to Table.

You may have to reformat the table to match.

Method 2 is a little more complex.

First, do a, b, and c from Method 1.

d) Now, select the converted text and use Find>Advanced Find

e) In the Find box, place a single comma (and make sure there are no spaces or anything else in the field)

f) In the Replace box, type ^t (caret then lower-case t).  That's the symbol for a tab.  Again, make sure there are no spaces or other junk in the field, and that the correct area is selected.

g)  Replace all.  When it asks if you want to do the remainder of the document, for goodness sake say NO!

Now select all the converted text again, and Table>Convert>Text to table.

Your table will appear, with an added column, first names on one, last names in the other.

If the instructor demands that the result contains "Last name, First Name" in a single column, the first method is better, because that's the result it gives you.

If the instructor is happy with "Last Name" and "First Name" in two separate columns, then do the second method because it is then easy to add or remove names and re-sort without doing any more conversions.

The object of this lesson is to learn why professionals are so rigid about not mixing different kinds of data in "fields".  It may look compact, but when you need to do anything with it other than read it, it's a real pain!

Hope this helps
