The most likely cause is that what you believe to be a contiguous table really isn't one. Make sure you have the non-printing characters [¶] displayed. If there is a ¶ symbol following the 'set of rows' where you invoked the feature or preceding the next set of rows it indicates that you're dealing with more than one table. Click at the ¶ then press the del [fn+delete on a laptop) key to join the following table to the one that precedes it. I didn't suggest that you delete anything :-)

Command+A selects everything from beginning to end, including the final ¶ in the document. However, that marker carries a great deal of data & may well be carrying the corruption, so you don't want to copy it.

To deselect [i.e., remove selection highlighting from] that final ¶ [while leaving the remainder of the document selected], just hold the Shift key while you tap the Left Arrow key on your keyboard. That enables you to copy everything except that ending ¶.

ashish mathur say:

Hi,

Try this

1. Click on cell B2 and go to Home > Conditional formatting > New Rule > Use a formula to determine which cells to format

2. In the formula bar there, enter this formula

=AND(B2<=-5,C2=0)

3. Click on Format > Fill > Yellow > OK

4. While in the same cell, enter this formula in the Conditional formatting formula bar

=AND($B2>-5,$C2=0)

5. Click on Format > Fill > Red > OK

6. Copy cell B2, select cell C2 > right click > Paste special > Formats > OK

7. While in cell B2, enter this formula in the Conditional formatting formula bar

=AND(B2=0,C2>0)

8. Click on Format > Fill > Green > OK

The easiest method of inserting cell references in the header is as follows:

  • Insert a row at the top of the worksheet

End Sub

If you really want to use VBA code then I suggest some modification to Gord Dibben's code.

Private Sub Workbook_BeforePrint(Cancel As Boolean)
    Dim WS As Worksheet
    For Each WS In ActiveWorkbook.Sheets
        Select Case WS.Name

                WS.PageSetup.CenterHeader = Sheets("Sheet5").Range("E3").Value
        End Select
    Next
End Sub

joeu2004 say:

Excel cell values are saved in the CSV file exactly as they appear in the worksheet.  So if the data is formatted as Currency with 2 decimal places, for example, and the value is $123.4567, the text $123.46 will be written to the CSV file because that is how it appears in the cell.

If you open the CSV file using Notepad, you will see that the format of the cells as they are displayed is preserved in the file.

However, if you open the CSV file in Excel, Excel interprets the text exactly as it would interpret the same data that you enter manually.  Usually, Excel assigns the General cell format. But in some special cases, Excel formats the cell according to appearances in the CSV flle.

But that will depend on the regional settings and language options of the Excel application that opens the CSV file, not the Excel application that saved the CSV file.

The CSV file is simply a text file.  And by definition, a text file contains only data, no formatting information.

joeu2004 say:

Shawn wrote:

It was working at one point however. I migrated one entity from that. It just won't keep the formatting. Why did it work then but not now?

Nothing has changed in the way that Excel uses CSV files.  If it seemed to "work" (preserve formatting?) in the past, it was only a coincidence.

For example, if you format your cells as Currency with 2 decimal places, save as CSV, then open the CSV file, the cells will indeed be formatted as Currency with 2 decimal places.

That is not because Excel preserves the formatting.  It is simply be because that data looked like $123.46 when it was written to the CSV file, and when Excel opens the CSV file directly, it interprets $123.46 as Currency, and it formats Currency cells with 2 decimal places by default.

However, if you format your cells as Currency with 4 decimal places, save as CSV, then open the CSV file, the cells will in the cell.  But again, it only formats Currency cells with 2 decimal places.

Since you offer no concrete examples, I cannot explain what you are seeing any better.  But the key to your understanding how CSV files behave is to open the CSV file in Notepad or some similar text-only nonformatting application.
