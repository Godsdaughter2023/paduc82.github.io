jetta48 say:

You do not install Windows XP by installing it like you would a program in Windows 7. Instead, you need to set your PC's BIOS to boot your PC 1st from your CD/DVD drive instead of your current hard disk. You then put your Windows XP CD/DVD in the DVD drive. When you then start your PC, your PC uses the CD/DVD to install Windows XP on your drive. It is more involved than installing a standrard program, but it it is doable for a reasonably IT competent home user.

There are a few things to remember:

- The process will wipe the hard drive/partition it installs to, so you will lose everything on it - personal data (photos, documents, etc.) and programs. Therefore make sure you have separate backups and program installation disks, etc.

- You will need compatible Windows XP drivers for all your hardware. Make sure they are available for all you hardware before you start.

For more detailed information, please see:

http://windows.microsoft.com/en-US/windows-xp/help/setup/install-windows-xp .

 

Finally, I advise you to reconsider switching back to XP from Windows 7. Although you may feel more comfortable with XP, it is at/near the end of its support life as far as Microsoft is concerned so updates will eventually stop. Also, in a year or two's time when you may need to buy a new printer or a new program, you are more & more likely to find XP isn't supported/compatible. You may be wiser to perservere now with the newer Windows 7. The decision is, of course yours though.

    Microsoft have tried to make it as impossible as they can to kill the operating system. In general, users don't want to kill it off either, So, safeguards are added to the list of jobs given to the 'automatic administrator' and "The system administrator has set policies to prevent this installation" is one of them.


    Back in the heady days of DOS and Windows 9x, users really would have been the BOSS and could make decisions that would take out the OS at a key stroke. These were known as 'bad choices' and frequently took the BOSS totally by surprise.


    Some choices were made on a weekly basis and others had daily status. The one good thing was that the ensuing Windows re-installation ensured at least a few minutes of smooth running.


    Windows 2000 was not a consumer OS. Entering it was like walking into a self-locking jail cell, which suited System admins to a tee. It stopped members of the typing pool from screwing things up, not only at their end but also the server end of the business ops.


    XP was like a 'double door' store. One side was locked and the other side wasn't. System admins hated it at first.


    Then along came Vista and UAC. A couple of confrontations with it saw users turning it off. So, instead of getting a gentle message asking 'are you are sure you want to do that', it could be over-ridden. The worst case scenario was having to re-install and having no data backups.


    Windows 7 has UAC but most would never know it. Instead of issuing a warning at every mouse click, it lets you feel like you are in charge, but will not let you deal a terminal blow. That is reserved for less than perfect Microsoft Updates..


    .. and while we are on that subject..


    Windows updates work well 99% of the time, and will work 100% of the time assuming that there has been 0% user intervention at any time since the initial installation. Service packs too will do the same.. BUT..


    Some users have just GOT to play Command and Conquer Red Alert (an example), and if they can't, they will scream until they are sick. So, some bright spark finds a way of manipulating the system (making unauthorized changes), and at the next round of Windows Updates .......... wah, the system has been corrupted and the updates will NOT.


    The same kind of effect is seen by using registry cleaners which take out anything that is not being used. It is like removing the spare wheel from the family car. It may never be used, but if you get a 'flat'? Uh-oh..


    Now, I am not suggesting that it is entirely the fault of the user, because some seemingly 'bona fide' programs are installed in all good faith. Unfortunately, not all programs are created equally. What you don't and may never know as the user is that the program's code writer maybe be using unauthorized memory addresses in order for the program to run as fast as the user hopes it will, or for it to work at all.


    The greatest quality of a PC is that one can do almost anything with it. The worst failing is that users and code writers try to do anything and everything, often blissfully unaware of the global consequences..

    1. Start > type %systemroot% which should show the Windows folder, click on it. Once inside that folder, ensure there is a Minidump folder created. If not, CTRL-SHIFT-N to make a New Folder and name it Minidump.

    2. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Performance > Settings > Advanced > Ensure there's a check-mark for 'Automatically manage paging file size for all drives'.

    3. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Startup and Recovery > Settings > System Failure > ensure there is a check mark next to 'Write an event to the system log'.

    Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.

    4. Double check that the WERS is ENABLED:

    Start > Search > type services.msc > Under the name tab, find Windows Error Reporting Service > If the status of the service is not Started then right click it and select Start. Also ensure that under Startup Type it is set to Automatic rather than Manual. You can do this by right clicking it, selecting properties, and under General selecting startup type to 'Automatic', and then click Apply.
