Thank you for your suggestion.  Since submitting my question I have dicovered a simple solution on more than one website that requires no additional software provided that the PC already has Adobe Acrobat installed.  I have compiled a short note below, together with instructions that I hope others as inexpert as I am will find helpful. 

 

 

 

How to read an Apple .pages file on a PC

 

A .pages file comprises a group of files one of which is a preview.pdf that retains all the text and formatting of the original document and (provided Adobe Acrobat is installed on the PC) can be extracted and read but, because it is a pdf, cannot be edited.   People who use an Apple or iPad/iPhone/iPod at home sometimes find, when in the office on their Windows computer, that they want to consult something they hadn't converted to Windows friendly form.  This is how it's done:

 

1.  Suppose your file is called appletoolbox.pages. Save it on the PC in My Documents

 

2.  Change the name of the file by right clicking on it, clicking on Rename, deleting the .pages extension and replacing it with .zip

 

3.  The file name is now applebox.zip 

 

4.  Double click on the zip file to open it, then on QuickLook, double click on PDF Preview, then click on Open.

OK. Please elaborate on the details of what you are doing when you 'upload' the image.

 

Are you creating a web page using Publisher? How are you creating your web page?

 

300 ppi will not work correctly on the web in most cases. Even some on-line web tools will automaticallly resample a large resolution image like that to 72 ppi when it is rendered which can leave the text blurred or with the jaggies. You can try an experiment. View the page where you have inserted the logo. Right click the image and save it to your desktop. Open that image in your favorite image editing program and see what the dpi or ppi is.

 

I have never been that impressed in how Publisher converts text to an image. It almost always results in 'fuzzy' results. I can sometimes minimize the problems by using a different font, never use italics, save as a png, size the image full size...full scale in the final application, etc.

 

Another option that I have not tried that often is to use word art instead of text boxes for adding text to your composite images. Word art text is already an image and may convert to a composite image more cleanly than a text box.

 

For optimum results I add text to my logos etc using a third party image editor. That text is almost always better and then I resize/resample/optimize that image for the intended use. I may start with a 300 dpi image but I will certainly save it to either 96 dpi or 72 dpi at perhaps now less than 60% compression for the web. I prefer png to jpg but for simple, few color images gif files can be a best choice for web format.

 

I use Photoshop Elements for a lot of my image editing, but also find that irfanview can do most everything and it is free. I also think paint.net or gimp are also good free choices. If you want to optimize your logos try building them in an image editor and you will probably get better results than you will with Publisher.

 

Perhaps Volunteer J or someone else can jump in here on how to optimize image composites that you produce in Publisher. Sorry I don't have better suggestions.

 

DavidF
