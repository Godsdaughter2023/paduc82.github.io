hansv mvp say:

Press Alt+F11 to activate the Visual Basic Editor.

Select Insert > Module.

Copy the following code into the module:

Function CleanComma(ByVal s As Variant) As Variant
    If IsNull(s) Then
        CleanComma = Null
    Else
        Do While Left(s, 1) = ","
            s = Mid(s, 2)
        Loop
        Do While Right(s, 1) = ","
            s = Left(s, Len(s) - 1)
        Loop
        If s = "" Then
            CleanComma = Null
        Else
            Do While InStr(s, ",,")
                s = Replace(s, ",,", ",")
            Loop
            CleanComma = Replace(s, ",", ", ") & "."
        End If
    End If
End Function

Press Alt+F4 to close the Visual Basic Editor and to return to Access.

Activate the Queries tab of the Database window.

Click 'New query in design view'.

Add the relevant table, then close the Show Table dialog.

Add the text field to the query grid.

Select Query > Update Query.

In the 'Update to' line of the text field, enter

CleanComma([NameOfTextField])

substituting the actual name of the text field.

Select Query > Run and confirm.

scott mcdaniel say:

Is this a one-time sort of thing? If so, you could create an Update query and run it a few times. Make a backup of the table or database before you try this, of course.

UPDATE YourTable SET YourField=Replace(YourField, ",,",",")

To run this, create a new query but do NOT add any tables, and then switch to SQL view. Copy the above into the SQL editor, and be sure to change the Table and Field names to match your own. You'll have to run it a few times, but it'll eventually get all the double commas out of the field.

You can change the ending comma to a period like this:

UPDATE YourTable SET YourField = Left(YourField, Len(YourField)-1) & "."  WHERE InStrRev(YourField,",")=Len(YourField)

If this is something that's ongoing - that is, you'll need to run this periodically to fix up data you're moving over - you might be better off creating a Function that will handle this for you
