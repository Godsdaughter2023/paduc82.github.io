Sub elapsedTime()
Dim osld As Slide
Dim sngTime As Single
For Each osld In ActivePresentation.Slides
If Not osld.SlideShowTransition.Hidden Then

End If
Next osld
MsgBox "Total time = " & sngTime \ 60 & " mins. " & sngTime Mod 60 & " seconds"

SUMIF/COUNTIF functions with criteria comparing dates - Microsoft Community

steveu2017 say:

SUMIF/COUNTIF functions with criteria comparing dates

I have a list of transactions sorted by date.  Actual transactions have a date of today or in the past.  Future (forecasted) transactions have a date later than today.  How do I count (or sum) values associated with past transactions only?

hansv mvp say:

Let's say the transactions are in B2:B100 and the corresponding dates in D2:D100.

Count:

=COUNTIF(D2:D100,"<"&TODAY(),B2:B100)

Sum:

=SUMIF(D2:D100,"<"&TODAY(),B2:B100)

steveu2017 say: Thanks Hans.  That is exactly what I was looking for.  It is not intuitive that only the < goes inside the quotes and that the TODAY function must be preceded by an &

hansv mvp say:

"<" is a fixed, literal string

TODAY() is a function whose value is recalculated on the fly.

The concatenation operator & combines the fixed string and the current date into a condition that is evaluated dynamically.

Here is one option- based on your scenario where it appears you only need to do this with one page, and it sounds like a static report (you aren't setting this up in a newly created workbook every day)

Select the entire page that you want to have printed twice

Copy

Paste Special as linked picture, onto a different (blank) worksheet

Reformat that worksheet to landscape, if desired

Select the pasted (linked) image, and paste again next to the original

Set the page printing options to one page wide and one page tall (on the page layout ribbon)

If desired, adjust the page margins

Now when you make changes to the original sheet, those changes will automatically be reflected in both linked images. Print the linked image worksheet, and you will have two identical copies on the same page.

Of course, if you wanted to automate the above steps with a macro, you could do that too.

Hide textbox on a record - Continuous form - Microsoft Community

hugosena say:

Hide textbox on a record - Continuous form

Hi,

I need to hide a textbox in a continuous form, but the problem is that when I use a txtClient.Visible = False, I got all the textboxes from all the rows hide, and I just want to hide the current row...

I have Access 2007 and Conditional Format do not solve the question.

Anyone has a solution?

Thanks
john p spencer say:

Ok, this is a bit complex and may not work for you.

Add a control in the header or footer area of the form
Name: txtCurrentRecordID
Control Source: ID
Visible: No

Now, set the conditional formatting of txtClient to the following

Expression is : [ID] = [txtCurrentRecordID]

Choose the settings you want.

I would choose to set the font to White (my usual background color) and click the Enabled/Disabled button so the control was disabled to prevent the user from clicking in the control and being able at that point to see the value.

The only time this might fail is on a new record.  So you might reverse things a bit and set the normal properties of txtClient to White and Disabled.  And use the expression to set txtClient to Black and enabled using NOT( [ID] = [txtCurrentRecordID])
