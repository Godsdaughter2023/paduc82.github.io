daniel orman say:

I figured it out. Here's what I did:

Set a variable at the top of the module:

Using 2 text boxes, 1 with a deactivated look and the other with a normal look. The following macro updated the saved notes (deactivated look) with the new notes (normal look):

    Me.D1_PB_Saved_Notes = ("# [" & Date & " @ " & Time & " by " & Environ("Username") & "]" & vbCrLf & _
                            Me.D1_PB_Notes_TextBox.Value & vbCrLf & vbCrLf & _
                            Me.D1_PB_Saved_Notes)
   
    With Me.D1_PB_Notes_TextBox
        .Value = ""
        .SetFocus
        .ScrollBars = fmScrollBarsVertical
    End With
   
    D1_PB_Notes_DoTask_RememberNotes = Me.D1_PB_Saved_Notes
End Sub

Finally, if someone tried to edit the notes in the saved notes textbox it would run the following macro:

    MsgBox "Saved notes cannot be altered. Please add a new note."
   
    Me.D1_PB_Saved_Notes = D1_PB_Notes_DoTask_RememberNotes
End Sub

How this works:

When adding a new note to the saved noted section (Sub D1_PB_N_Update_Click) it would set the variable D1_PB_Notes_DoTask_RememberNotes to the value of the saved notes textbox. Each time a new note is added the original note appears below the new note due to the first section of the above macro.

If someone edited the saved notes it would bring up an error message and then revert that text box back to the original notes. This allows for the user to still use the scrollbar but also secures the data and makes it "non-editable".

I hope this helps anyone else who is in need of it.
