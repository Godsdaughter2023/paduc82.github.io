If you fancy having a go at vba, here (in the spirit of JohnK's answer) is some code adapted from Steve's code by John SR!

Make a folder called Files on your desktop

Put into it COPIES of all the PowerPoint files to change AND the potx template.


(Change the name to match the name of your template)

Sub NEW_Template()

' Performs some operation on every presentation file in a folder

    Dim sFolder As String   ' Full path to folder we'll examine

    Dim sFileSpec As String    ' Filespec, e.g. *.PPT

    Dim sFileName As String    ' Name of a file in the folder

    Dim oPres As Presentation

    Const myTemplateName As String = "MyTemp.potx" 'CHANGE THIS NAME

    sFolder = Environ("USERPROFILE") & "\Desktop\Files\"   ' must end with a \ character

    On Error Resume Next

    MkDir sFolder & "\NewTemplates\"

    sFileSpec = "*.PPT*"

    ' Get the first filename that matches the spec:

    sFileName = Dir$(sFolder & sFileSpec)

    While sFileName <> ""

        Set oPres = Presentations.Open(sFolder & sFileName, msoFalse)

        oPres.ApplyTemplate (sFolder & myTemplateName) 'change the name to match

        oPres.SaveAs (sFolder & "\NewTemplates\" & oPres.Name)

        oPres.Close

        ' release the reference

        Set oPres = Nothing

        sFileName = Dir()

    Wend

End Sub

Hi Marc,

Thanks for your explanation.

Scanning is a feature which is added in OneDrive for some simple scanning scenario. So actually OneDrive is still used as a storage tool to store various files which are produced by other applications like Office Lens. 

Office Lens is a simple tool for scanning . For example, the difference between Lens and OneDrive application contains the following points:

1.The scanned picture from Lens can be saved to your choice of OneNote, OneDrive, Word (OCR), PowerPoint, PDF or locally on the device. But OneDrive only store them in OneDrive folders. The scanned pictures' storage range of Lens is larger than that in OneDrive.

2.Office Lens can be imported some pictures from local resource to identify or just as a middle tool to store them in different applications. 

3.Office Lens has the resolution which can be adjusted to adapt to the requirement of different file sizes. The scanned file sizes are fixed in OneDrive.

4.Office Lens has the feature Recent History which is convenient for us to find the recent files. But it is not available for scanning feature in OneDrive.

In a word, Office Lens is a more professional application from scanning to storage than that in OneDrive. But in some simple scenarios to store scanned files without the requirement of size in OneDrive, it is convenient for us to use the scanning feature in OneDrive directly.

john korchok say:

You may be able to fix this by copying styles from the template or main document that has the correct styles. You can use the Organizer to do this:

  1. Click on Tools>Templates and Addins, then on the Organizer button.
  2. The Styles tab should open by default, with your source document or template on the left and Normal.dotm on the right.
  3. Below Normal.dotm, click on the Close button.
  4. The Close button turns to an Open File... button, click on it. 
  5. Navigate to a Word subfile that is in your document, select it and click on Open.
  6. Select all styles on the left-hand list, then click on Copy. Word will ask for confirmation. Choose Yes to All.
  7. Click on the Close button. Word will ask whether you want to save your changes, choose Yes.
  8. Repeat steps 4 to 7 until all subfiles have been updated.
  9. If you inserted the Word file as linked objects, closing and re-opening the main file will force Word to ask if you want to update the links. Choose Yes. If you embedded the objects, you'll have to re-insert them in the main document.
  10. Test the TOC to see if it's working correctly. If it's not, the problem may not lie with the styles. but with how you have applied them. One example would be where you have text that appears in the TOC, that's a misapplied style.

An alternative to your construction method is to create a Master Document with subdocuments. It's best to maintain a Master Document only to create a TOC and print the file. Updating a Master Document over time often creates document corruption. There are no Mac tutorials on the web that I can find, but if you're interested I can post the steps to creating a master document.
