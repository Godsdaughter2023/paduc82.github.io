Hi Eckerlin,

Normal/full backups: All files that have been selected are backed up, regardless of the setting of the archive attribute. When a file is backed up, the archive attribute is cleared. If the file is later modified, this attribute is set, which indicates that the file needs to be backed up.

Incremental backups: Designed to create backups of files that have changed since the most recent normal or incremental backup. The presence of the archive attribute indicates that the file has been modified and only files with this attribute are backed up. When a file is backed up, the archive attribute is cleared. If the file is later modified, this attribute is set, which indicates that the file needs to be backed up.

For the second question:

As mentioned above in Incremental Backup, the presence of the archive attribute indicates that the file has been modified and only files with this attribute are backed up. When a file is backed up, the archive attribute is cleared. If the file is later modified, this attribute is set, which indicates that the file needs to be backed up.

So the next Incremental Backup will be applied to the file which has archive attribute.

For:  If there is a limit for the number of incremental backups associated with a Full Backup: can I change/increase this limit?

After the original full backup is complete, updates to the backup include only files that have changed since the last backup.

The backup would also depend on the scheduled setting: Daily, Weekly or Monthly.

For more control of exactly which files and folders are backed up, you can also set up a new backup and click Let Me Choose from the backup and restore dialog box.
