a. user say:

How should I keep backups of my system then? I know there are things called system images but I don't know how to use them.

A restore point is not a backup although it's frequently confused with a backup.

There are two types of backup: a file-and-folder backup and an image backup. A file-and-folder backup backs up individual files and folders all at once. It's ideal if you want to backup your personal data (documents, pictures, music, videos, downloads, etc.) There are very many great file-and-folder backup programs, like Backup4All.

An image backup is an exact copy of a hard drive, for example, your C: drive. If you store your personal files on the C: drive, as most people do, then an image backup also includes your personal files. Here, too, there are many great image backup programs, like Macrium Reflect (which also comes in a free version,) Acronis True Image and EaseUS ToDo Backup among many others. An image backup is the only way to backup Windows and your software. Imagine never needing to reinstall Windows, ever again - that's the advantage of making image backups.

Once you've learned how image backups work, and how to make and restore an image, you'll see how quick and easy it is to make image backups every day - automatically! And it only takes minutes to make (and to restore) an image. It's the ultimate protection against computer problems. Why waste time and become frustrated with trying to fix a computer when you can easily send it back in time to when everything was working well?

To find terrific backup software, and to learn more about backing up in general, Google is your friend. You can find reviews of backup programs written by professional testers to help you make a choice.

Multi-level numbering in appendix headings - Microsoft Community

hmm1 say:

Multi-level numbering in appendix headings

I have appendices in my document, A through H.  Some have subsections, whose headings I want numbered.  So, for example, in Appendix B there could be subsections with headings B1 and B2.

Does anyone know how I would do this; so far in Word, I have only seen multi-level headings with numbers, not letters.

Thanks.

cindy meister say:

Hi hmmm

It's there :-) The numbering just needs to be customized.

Have you been using Heading numbering otherwise in the document? So Heading 1 is probably already linked with numbering?

In that case, it's usual to choose a much lower heading level (Heading 4 or Heading 5, for example) for Appendix headings.

What you'll want to do is to "Define a new list style" (bottom of the menu when you click the Multilevel list button). Click in a Heading, then choose this menu command. 

In the dialog box, enter a name for your numbering "scheme", such as List Headings.

Go to the bottom left, click Format and choose Numbering to bring up the Numbering dialog box. Click the "More" button at the bottom left.

Starting with level 1 (the list down the left side), check the setting for "Link level to style". Select the appropriate style (Heading 1, etc.) for each level you already have in your document.

When you get to the level you want to use for the Appendix main headings, select the corresponding Heading style. Then go to "Number style for this level" and select "A,B,C". You'll want to remove everything in "Enter formatting for the number" except the "A". You'll probably want to change the settings under "Position", as well.

When you get finished, you'll always want to change the style definition for the Heading styles you use in the appendix so that they look like the corresponding Heading style in the main part of the document.

steve rindsberg say:

You really have to go out of your way to get an image to link rather than embed, and the error msg you reported:

"The picture element with the link-ID rld4 was not found."

almost certainly doesn't apply to a user-created image link.  It's more likely a reference to the way PPT stores images internally.  For example, if you insert an image normally (and NOT linked), it's embedded.  If you copy/paste that image or in some cases, insert the same image again into the same presentation, PPT doesn't store another copy of the image, it references the original copy of the image as an internal link (the kind it's complaining about here). 

It can get quite complex when you, say, insert an image, copy it to some other slides, select one of the images on another slide, copy it elsewhere and so on.  Now the images link back to links that link to other links and finally back to the original stored image. 

Normally PPT sorts all this out reliably, even when you delete a slide out of the middle of this "chain of links".  But sometimes it loses track, which could lead to the error message you're seeing. 

Suggestions:

If your images are huge (any larger than necessary to fill the sceen space they'll be using) you might want to create a downsampled set of them and use those in PPT.

If you have access to a Windows version of PPT, try round-tripping the file to HTML and back; that can repair mild corruption in some cases. 

HTML "Round-tripping" to repair corruption
http://www.pptfaq.com/FAQ00526_HTML_-Round-tripping-_to_repair_corruption.htm

CMYK images or images with embedded color profiles can make PowerPoint unhappy as well.  Convert all images to RGB and make sure your image editing software, if any, hasn't embedded a profile.
