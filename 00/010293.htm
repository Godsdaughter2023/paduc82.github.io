For the benefit of anyone else with this problem, I have found a workaround.

This probably applies for most versions of Office (I have gone back to Office 2007) and works with Windows 10. Instead of Hyperlinking to files in your PPT presentation, insert an Action with the path to the software that is going to open the file and then the file name, both in "quotes" and separated by a single space, e.g.:

"C:\Program Files (x86)\Windows Media Player\wmplayer.exe" "C:\Users\xxx\Documents\film.mp4"

Use whichever program works best for your linked movie or file (e.g. WMP or VLC). This seems to work without fail and doesn't put up any nasty warnings. OK and you're done!

dirk goldgar say:

Please pardon me for jumping in.

I know a way, using VBA code, to keep the MsgBox from appearing *twice* when you close the form under these conditions, but I haven't yet figured out a way to keep it from appearing once.

However, I think the best way to avoid this problem is to not try to prevent the user from exiting the combo box when it's Null.  Instead, focus on the action that the user will subsequently take that requires there to be a value in the combo box.  If some code or macro is going to run that needs the value in the combo box, then check in that code (or macro) for a Null value, and if it is, display a message and send the focus back to the combo box for the user to select a tail number.

Even better is to disable those controls on the form that require the tail number, until the tail number is selected.  When the form opens or in the Current event, whichever is appropriate, disable these controls; then, in the combo's AfterUpdate event, enable the controls if the combo is not Null.  That way, the user can't click the button, or whatever, that kicks off the process requiring the tail number unless the combo has been filled in.  But the user can still move around the form and do anything else -- including closing the form -- that doesn't need the tail number.
