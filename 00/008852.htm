Does every table need a primary key? - Microsoft Community

ctdak say:

Does every table need a primary key?

I have several tables in an Access mdb that do not have primary keys and don't seem to need them.  The application has been running this way fine for many years.  Some are tables for temporary use, for instance during a process the table is programmatically populated with records for report generation purposes only, then all records are deleted in that table ready for the next time the report is generated.  And one table just has one record with application "system data" in it, global default values.

I have primary keys in tables where they're needed for relationships, but otherwise not.  One source I have says that all tables should have a primary key, presumably whether or not they are involved in a relationship.  Is this really the case, and if so why, seeing as the application runs just fine without them?

scott mcdaniel say:

It's generally considered to be a good idea to have a primary key defined on a table, but for you temporary tables used solely for report generation it's certainly not required. I'd think it's a good idea to define one on your "system data" table, but as you're aware it will work just fine without it.

However many database engines will let you get by without defining a Primary Key on a table.

More important is a unique index - that is a field or group of fields that uniquely define a single record in your table.

Note too that if you move tables up to SQL Server or other server-type platforms and link them back in Access, you'll find that in many cases Access won't be able to update them unless a primary key is defined on the table.

How do I center an object in the Header in Word for Mac 2011 - Microsoft Community

marvinz say:

How do I center an object in the Header in Word for Mac 2011

I insert a PDF (logo) in the Header and select it, but the Align choices are all grayed out. Help! Thanks.

daniel g. grau say:

That's because you are dealing with a floating object, whereas those buttons refer to paragraphs.

You can do a visual alignment, using the Ruler as a guide. Moving the object will show a useful shading in the Ruler, while pressing Option will add the left and top distances. It could help if you enable the background grid through the Grid Options button in the Ribbon's Layout tab. (Unfortunately—or uncomprehendingly—, the grid is restricted to the area within the document's margins.)

An alternative is to select the object, right-click on it to select Wrap Text > In Line with Text, and then  you'll be able to use those paragraph-alignment buttons to align the object. To control its vertical placement, use the paragraph's Space Before attribute and/or the document's margins in Format > Document > Margins.

rich michaels - mvp (greatcirclelearning.com) say:

Adding to Daniel's reply...

I recommend that you open Word > Preferences > Edit and in the preference setting for Insert/Paste Images you set the default as "Inline with text".

IMHO that should always be the "default" and wrapping text used as an option when needed.

bob jones aka: cybertaz say:

I'm guessing in the opposite direction from Daniel... My assumption is that the image is set to In Line & that you are referring to the Align options on the Format Picture tab of the Ribbon or those in the contextual menu. Those options do not pertain to In Line objects. With no more detail than you've provided it's difficult to be sure because there are other possibilities as well.

For what it sounds like you want to do, the best option is to leave the image formatted as In Line With Text, then use the same Center Alignment button you'd use for centering a paragraph -- similar to Daniel's 2nd suggestion, but don't have the image selected once you've done so. The paragraph alignment choices will be dimmed any time an object is selected regardless of In Line or style of Text Wrapping.

If there is other content in the Header you may need to use a Center aligned Tab instead, but again, you've not given us nearly enough descriptive information to go on.
