A little VBA code (a macro) can do it.

First, one to simply unhide all worksheets in a workbook:

Sub UnhideAll()
  Dim anyWS As Worksheet

  For Each anyWS In ThisWorkbook.Worksheets
    anyWS.Visible = xlSheetVisible
  Next
End Sub

Sub UnhideSelectively()
  'just change the name of the worksheets referenced and
  'repeat for the sheets you want to unhide
  'just use the names that appear on the name tabs.
  Worksheets("Sheet1").Visible = xlSheetVisible
  Worksheets("Sheet2").Visible = xlSheetVisible

End Sub

 

To reverse the process, just use = xlSheetHidden : but you cannot hide ALL sheets in a workbook, at least one must remain visible.

sockatume---01 say:

I have the same handset with the same software, but don't have this issue. I think you're running up against a combination of a limitation in the way the phone handles messages, and a bug in Ovi Suite. Fasten your seat belt.

Your phone contains a message Index. Each Index item consists of a name, a snippet of the message, an email address if it was an email, and a link to the message file itself, and perhaps some meta-data. This is what you see when you open your Inbox. The rest is stored in the message file.

When you get a message, it looks up the sender in Contacts, grabs the sender's name, then uses that name when it writes the Index entry.

Critically, this information is static. When you change the name format in Contacts, the phone doesn't rebuild the message index. The entries and files remain stamped with the old name, in the old format. Only new messages will be saved with the new format when you change the setting in Contacts.

With regard to Ovi Suite, when it saves the messages back to the phone, it must be rebuilding the index without respecting the phone's current setting for name format. So it rebuilds the index in Last Name First Name format.

Out of curiosity, how are the messages displayed in Ovi Suite? First Name Last Name, or Last Name First Name?
