bob jones aka: cybertaz say:

The built-in PDF function (File> Save As> PDF) in Mac Word 2016 does not offer any options to export only specific pages. That is a design limitation, not a bug. It would require outputting the complete file, then editing the PDF with Preview, Adobe Reader, etc. to delete the unwanted pages form the PDF.

In order to include only specific pages & page ranges you'd have to use the macOS method (File> Print - PDF). You can make the specification just as for any other print job. I don't know what you mean by "I lose the internal structure of the document", though.

As for the issue with using the built-in method: The only reports I've seen of a similar nature involved Office installations which weren't up-to-date. Make sure your installation is current (Version 15.40 as of this writing).

 The problem is with so-called user-agent string [a kind of code (a text string, computer programming sequence of symbols)] that the browser sends to the server of a particular web site. Its purpose is for the browser to identify itself. For some reason, in my case (and supposedly in yours too), that string is not set properly so that server can identify the browser version correctly.
 Here, you can go and see how some server "sees" your browser:

 http://www.useragentstring.com
 It is the test site for user-agent string setting, and in the gray field, it will tell you how your browser is being identified by some server. In my case, it read Internet Explorer 6, although I certainly use IE 8.
 The simplest solution is to download and install IE add-on called "User Agent Picker", then restart the IE, go to Tools menu, then to newly set option "Set UA String" and pick the appropriate UA string setting for your system. From your question, I can see that it's same as mine: IE8.XP . Don't forget to check the 'Reuse...' box if you don't want the setting to return to the previous one.
