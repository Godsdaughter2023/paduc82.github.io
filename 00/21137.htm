jj_951 say:

Prevent Word 2007 from using normal.dotm AND prevent the warning message?

Thanks in advance!

graham mayor say:
You are not then going to like the answer. The normal template for Word 2007 (and 2010) is normal.dotm. If that file is not present, Word will create it. If normal.dot is in the user templates folder then Word will use the configuration of that file to create normal.dotm. You cannot make Word use a different default template or not use the template.

The normal template is an essential individual user system file. It may not be shared between users, and frankly you should leave it to the user to determine how his/her system is configured with respect to this essential file, or you are simply building in more problems.

Set the user templates folder to be a trusted location (which it should be by default) and you will not see any macro warnings from it.

Setting an application designed to use XML file formats by default (and around which the whole of the application revolves) not to use such a format is frankly ridiculous. If you didn't want to use XML files then you should have stuck with Office 2003.

tony jollans say:

I would restate everything Graham says. Word 2007 (and later) need a macro-enabled 2007-format (.dotm) normal template, and there is nothing you can do to change that. Using the GlobalDotName setting names (and potentially locates) your normal template, that's all - it does not, and you can not, change the format of it.

I _think_ you could suppress any message at start up by not having a normal template, so that Word would simply create a factory-fresh one in memory each time. One way of not having one would be to write-protect the location of the normal template but I think that would just trigger messages and/or SaveAs dialogs appearing at shutdown. You may be able to suppress those by having Application Close Event code to flag Normal as Saved so Word didn't try to save it - but, without a Normal template you would have to save the code somewhere else that was always available.

It might be an interesting exercise to try to do what you want but the end result would be superficial and there would be ways round it, no matter what you did. Frankly, the policy is ridiculous and, if that's what passes for security in your organisation, I hope I'm not a customer. If you insist on the policy you will have to live with the consequences.

jj_951 say:

bernie deitrick, excel mvp 2000-2010 say:

There really is no need to select anything, but to do what you want:

Sub FindDate()
    Dim myR As Range
    Set myR = Worksheets("Sheet 2").Range("A:A").Find( _
              What:=Worksheets("Sheet 1").Range("B4").Value, LookIn:=xlFormulas)
    If Not myR Is Nothing Then
        myR.Parent.Activate
        myR.EntireRow.Select
    Else
        MsgBox "That date was not found."
    End If
End Sub

 

Note that I used Sheet 1 instead of the default Sheet1 as the sheet name (as described by you), so check what your actual sheet names are.

jllatham say:

Try this code, should work just fine for you as long as formats of B4 and entries in column A are the same (all as dates or all as text - dates preferred).

Instructions to insert the code into a regular code module can be found here:

http://www.contextures.com/xlvba01.html#Regular

You'll need to edit the code after pasting into the module - the two Const that define the sheet names involved need to have the proper sheet names entered.

The code:

Sub FindFirstDateMatch()
  'change these constant values as required
  'these 1st two may be the only ones you have to change
  Const oneDateSheetName = "Sheet1" ' sheet with B4 on it
  'cell on above sheet with date to find on 2nd sheet
  Const searchDateCell = "B4"
  Const searchSheetName = "Sheet2"
  Const searchCol = "A" ' column to search
  'working variables
  Dim seekDate As Range
  Dim seekRange As Range
  Dim foundDate As Range
  Dim startAtCell As Range
 
  Set seekDate = ThisWorkbook.Worksheets(oneDateSheetName).Range(searchDateCell)
  Set seekRange = ThisWorkbook.Worksheets(searchSheetName). _
   Columns(searchCol & ":" & searchCol)
  Set startAtCell = ThisWorkbook.Worksheets(searchSheetName). _
   Range(searchCol & 1)
 
  Set foundDate = seekRange.Find(What:=seekDate.Value, After:=startAtCell, _
   LookIn:=xlFormulas, LookAt:=xlPart, SearchOrder:=xlByRows, _
   SearchDirection:=xlNext, MatchCase:=False, SearchFormat:=False)
  If Not foundDate Is Nothing Then
    'found it
    Worksheets(searchSheetName).Activate
    Application.Goto foundDate, True
  Else
    MsgBox "Could not match this date.", vbOKOnly, "No Match Found"
  End If
  'housekeeping/cleanup
  Set seekRange = Nothing
  Set foundDate = Nothing
  Set seekDate = Nothing
  Set startAtCell = Nothing
End Sub
