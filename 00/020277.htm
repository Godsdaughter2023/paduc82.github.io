excel import truncates data - Microsoft Community

robschroder say:

excel import truncates data

I am trying to import an excel spreadsheet into an existing access table.  There are a number of fields in the access table that are formatted as memo fields.  The corresponding excel fields contain more than 255 characters.  The data is being cut off at 255 when I run the import.   How can I import into an existing table and NOT have the data in the memo fields get cut off?
jeanette cunningham say: There are some situations where access will truncate data to 255 characters.
1. Access makes a guess about the type of data being imported.
2. Access bases its guess on analyzing the 1st several rows (don't know the exact no.) of data.

If access finds that the 1st several rows are less than 255 characters, it will assume a text field - not memo and truncate the data by putting it into a text field

If you do the export manually, you can select the data type for the field (A2010 version, not earlier versions) - if you set that field to memo, the data will not be truncated on import, access will create a memo field for the data - even when the 1st several rows have less than 255 characters.

Excel - Syncing Two Tables - Microsoft Community

matthewginn say:

Excel - Syncing Two Tables

I have two sheets. Sheet A has data entered for dates, concrete quantities, concrete costs, etc. Sheet B autopopulates some of the information from Sheet A but also requires additional manually entered data. My problem is when I try to sort the information (by date for instance) in Sheet A, the manually entered data in Sheet B does not get sorted since it is not tied to Sheet A, but the information that IS tied to Sheet A does. Is there a way to synchronize the two sheets so that when I sort or filter in Sheet A, all the information in Sheet B (including manually entered data) will also sort/filter and vise versa.

teylyn say:

Hello,

I suggest that the data in Sheet A has a unique ID for each row. 

In Sheet B, use this unique ID as a manually entered value. Then pull the data for that ID with a lookup formula, like Vlookup or an Index/Match combination. Do NOT use direct references to =SheetA!F4 or some such to populate the values. The lookup formula will find the row with the ID, no matter where it is in the sort, and will return the columns for that ID. 

Now Sheet B will not be affected by sorting Sheet A. You can put additional data into sheet B and you can even sort Sheet B on another column if you desire.
