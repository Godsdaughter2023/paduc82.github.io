For example: A user enters January 5, 2018 in the txt_DateOpen field and February 21, 2018 in the txt_Dateclose field. This, in turn, populates the txt_Quarter field with "Q12018. dhookom say: Hi Kimberly,
What do you expect if the dates aren't in the same quarter?

You don't need to store the quarter since it can be calculated any time. If you want to display the quarter, you can set the Control Source of the text box to:

="Q"& DatePart("q",[DateOpen]) & Year([DateOpen])

If you think you need to store the value (not a good idea) you can use the after update event of the text box like:
Me.txt_Quarter = "Q"& DatePart("q",[DateOpen]) & Year([DateOpen])

ken sheridan say:

If the open and close dates span more than one quarter you could return the range of quarters with:

="Q" & DatePart("q",txt_DateOpen) & Year(txt_DateOpen) & IIF("Q" & DatePart("q",txt_DateOpen) & Year(txt_DateOpen)<> "Q" & DatePart("q",txt_DateClose) & Year(txt_DateClose),"-Q" & DatePart("q",txt_DateClose) & Year(txt_DateClose),"")

You can see how it would work in the immediate window:

Firstly if both dates are in the same quarter of the same year:

txt_DateOpen = #2018-02-03#
txt_DateClose = #2018-03-04#
? "Q" & DatePart("q",txt_DateOpen) & Year(txt_DateOpen) & IIF("Q" & DatePart("q",txt_DateOpen) & Year(txt_DateOpen)<> "Q" & DatePart("q",txt_DateClose) & Year(txt_DateClose),"-Q" & DatePart("q",txt_DateClose) & Year(txt_DateClose),"")
Q12018

The single quarter is returned by the expression above, but if we change the close date to another quarter:

txt_DateClose = #2018-04-05#
? "Q" & DatePart("q",txt_DateOpen) & Year(txt_DateOpen) & IIF("Q" & DatePart("q",txt_DateOpen) & Year(txt_DateOpen)<> "Q" & DatePart("q",txt_DateClose) & Year(txt_DateClose),"-Q" & DatePart("q",txt_DateClose) & Year(txt_DateClose),"")
Q12018-Q22018

Now the range has been returned.  The range can be as long as necessary, e.g.

txt_DateClose = #2019-04-05#

? "Q" & DatePart("q",txt_DateOpen) & Year(txt_DateOpen) & IIF("Q" & DatePart("q",txt_DateOpen) & Year(txt_DateOpen)<> "Q" & DatePart("q",txt_DateClose) & Year(txt_DateClose),"-Q" & DatePart("q",txt_DateClose) & Year(txt_DateClose),"")
Q12018-Q22019

The range is now over 5 quarters.

Report months chronologically and not alphabetically in Access 2010 - Microsoft Community

saar3369 say:

Report months chronologically and not alphabetically in Access 2010

Hi!
I have a query that summarises information by month.
How can I get the Date By Month field to sort chronologically and not alphabetically?
Thanks!
ken sheridan say: You need to group the query by year, then the numerical month value, then the month name, e.g. using a Transactions table:

SELECT YEAR(TransactionDate) AS TransactionYear,
FORMAT(TransactionDate,"mmmm") AS TransactionMonth,
SUM(TransactionAmount) AS TotalAmount
FROM Transactions
GROUP BY YEAR(TransactionDate), MONTH(TransactionDate), FORMAT(TransactionDate,"mmmm");

Hello Samuelawe:

 

Thank you for posting to the forum regarding the update error you are receiving.  Update errors can be caused by many things, your phone's hardware, your phone's firmware, the computer you are using, the network connection and more.  We will try some things to see if we can narrow down where this error is coming from.

 

The first thing I will present is this article that discusses the error you are receiving, http://windows.microsoft.com/en-us/windows7/Windows-Update-error-80070424.  This indicates the issue is with your connection to Microsoft updates.  Please try the BITS fix it to see if any progress can be made on this computer.  If you do not make any progress, try connecting to and updating from another computer on a different network.  If you can, try a computer that runs a different OS then the one you are currently using.

 

To run a Windows Update troubleshooter

  1. Click this button:

    Fix this problem

  2. In the File Download dialog box, click Run, and then follow the steps in the wizard.

  3. Open Windows Update, and try to install the update again.

If that didn't work, try the following steps.

 

To start required Windows Update services

  1. Open Administrative Tools by clicking the Start button , and then clicking Control Panel. In the search box, type administrative tools, and then click Administrative Tools.

  2. Double-click Services. If you're prompted for an administrator password or confirmation, type the password or provide confirmation.

  3. In the Services dialog box, click Background Intelligent Transfer Service (you might need to scroll to find it; the entries are in alphabetical order).

  4. Note the status of the selection (the status appears in the third column). If it shows Stopped, right-click Background Intelligent Transfer Service, and then click Start.

  5. In the Services dialog box, click Windows Update (you might need to scroll to find it; the entries are in alphabetical order).

  6. Note the status of the selection (the status appears in the third column). If it shows Stopped, right-click Windows Update, and then click Start.

  7. In the Services dialog box, click Workstation (you might need to scroll to find it; the entries are in alphabetical order).

  8. Note the status of the selection (the status appears in the third column). If it shows Stopped, right-click Workstation, and then click Start.

  9. Close the Services dialog box, and then run Windows updates again.

     

If the Background Intelligent Transfer Service doesn't appear in the list of Services, see the resolution section of this article about problems with the Background Intelligent Transfer Service on the Microsoft website.

 

If the Windows Update Service doesn't appear in the list of Services, see this article about installing the Windows Update agent on the Microsoft website.

 

Before we settle on this being an issue with the phone itself, I would suggest you also try connecting your phone with another data cable.  If you still get the same error under all of the above circumstances, the issue is likely with the phone's firmware.  If this is the case, you may be able to get the firmware updated or reinstalled by your carrier.

 

Please let us know if you have any other questions or concerns.
