greginmich say:

Windows Defender Antivirus is looking much better on paper these days; but we're still not sure how well all of the new features will actually translate into improved real-world protection scores. If we look at the most recent (but dated) Real-World Protection Test, we see that an exceptionally large component of the overall protection is user dependent – and that makes it virtually impossible to say how well Defender would actually stack up against any particular third-party AV app.

https://www.av-comparatives.org/wp-content/uploads/2017/11/avc_factsheet2017_10.pdf

So I’m replying to these questions about whether Windows Defender is “good enough” with some tips on configuring Windows Defender Antivirus for optimal protection. Truth be told, it just might not be quite “good enough” if you haven’t updated to Version 1709, and if you haven’t reconfigured it to provide enhanced Block at First Sight protection; Ransomware protection (Controlled Folder Access), Attack Surface Reduction protection, and PUA protection. None of these features are enabled in the default configuration:

 

Block at First Sight (Seen):

 

Windows Defender can now immediately block suspicious or unknown files; and then automatically upload and analyze a sample, and generate a signature; all within a matter of seconds

 

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-antivirus/configure-block-at-first-sight-windows-defender-antivirus

 

https://blogs.technet.microsoft.com/mmpc/2017/07/18/windows-defender-antivirus-cloud-protection-service-advanced-real-time-defense-against-never-before-seen-malware/  

 

We can also use PowerShell to upgrade the default protection level for the Block at First Sight feature, although the settings for this aren’t documented, and I had to trick PowerShell into telling me the named values for this feature by specifying a value that I knew was out of range:

 

To configure an increased level of cloud protection (Cloud Block Level), run one of these commands at the Administrator PowerShell Prompt:

 

Set-MpPreference -CloudBlockLevel High

 

Set-MpPreference -CloudBlockLevel HighPlus

 

Set-MpPreference -CloudBlockLevel ZeroTolerance

 

And you can also increase the allotted analysis time by running this command at the Administrator PowerShell Prompt:

 

Set-MpPreference -CloudExtendedTimeout 50

Windows Defender Exploit Guard:

 

The exploit protection features that were previously provided by EMET are now integrated into Windows 10; and this is a feature that most users should probably just leave alone:

 

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-exploit-guard/exploit-protection-exploit-guard

 

https://blogs.technet.microsoft.com/srd/2017/08/09/moving-beyond-emet-ii-windows-defender-exploit-guard/

Attack Surface Reduction:

 

We also have the ability to add Attack Surface Reduction rules in Version 1709, but the only practical way to add these rules in Windows 10 Home is with the PowerShell Set-MpPreference cmdlet:

 

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-exploit-guard/enable-attack-surface-reduction

 

https://blogs.technet.microsoft.com/mmpc/2017/10/23/windows-defender-exploit-guard-reduce-the-attack-surface-against-next-generation-malware/

 

For example, here’s the first rule that I set up by running a command line at the Administrator PowerShell prompt:

 

Rule: Block JavaScript or VBScript from launching downloaded executable content:

 

Set-MpPreference -AttackSurfaceReductionRules_Ids D3E037E1-3EB8-44C8-A917-57927947596D -AttackSurfaceReductionRules_Actions Enabled

 

Then to add additional rules, we use the Add-MpPreference command:

 

Rule: Block executable content from email client and webmail:

 

Add-MpPreference -AttackSurfaceReductionRules_Ids BE9BA2D9-53EA-4CDC-84E5-9B1EEEE46550 -AttackSurfaceReductionRules_Actions Enabled

Controlled Folder Access:

 

Controlled Folder Access is turned off by default; so you’ll need to turn it on in the Windows Defender Security Center app > Virus & threat protection > Virus & threat protection settings. Once Controlled Folder Access is turned on, standard document folders will be protected by default, and you’ll also have the option to add ransomware protection to additional folders, as well as whitelist trusted applications in order to allow them access to your protected folders. If you have trouble whitelisting a friendly app, then you can set this feature to run in Audit Mode, where it will identify intrusion attempts, but won’t block them. Run this command at the Administrator PowerShell Prompt to set Audit Mode:

 

Set-MpPreference -EnableControlledFolderAccess AuditMode

 

Then to reset Controlled Folder Access to its blocking mode, run this command:

 

Set-MpPreference -EnableControlledFolderAccess Enabled

 

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-exploit-guard/enable-controlled-folders-exploit-guard

 

https://blogs.technet.microsoft.com/mmpc/2017/10/23/stopping-ransomware-where-it-counts-protecting-your-data-with-controlled-folder-access/?ocid=cx-blog-mmpc

PUA Protection:

 

Windows Defender has actually been able to detect and block Potentially Unwanted Applications for some time now, but many people still don’t know that this feature is disabled by default and needs to be enabled by running this command line at the Administrator PowerShell Prompt:

 

Set-MpPreference -PUAProtection 1

 

Then, optionally, confirm that PUA Protection was enabled by returning the current state for PUAProtection:

 

$Preferences = Get-MpPreference

 

$Preferences.PUAProtection

 

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-antivirus/detect-block-potentially-unwanted-apps-windows-defender-antivirus

[Outlook] Is it possible to have the time zone within the "Sent" attribute when replying or forwarding a message?

Dear All,

When you reply (to) or forward a message in Outlook, the original message is copied (if configured) with the following additional attributes:

-----Original Message-----
From: ...
Sent: 08 September 2011 13:54
To: ...
Subject: ...


The Sent attribute contains the time when the message has been sent expressed in your time zone!


If you have a long e-mail conversation including people from different time zones, you need to be careful when calculating duration between e-mails, and you need to know the time zone of each senders!


In my opinion, this problem can be easily fixed just by putting your time zone within the "Sent", e.g.


-----Original Message-----
From: ...
Sent: 08 September 2011 13:54 (GMT+01:00) Brussels, Copenhagen, Madrid, Paris
To: ...
Subject: ...


If everyone do that, anyone could easily calculate duration between e-mails ;-)

in Outlook 2010,  "File > Options > Mail > Replies and forwards" does not seem to allow this kind of configuration :-/
