berleyg say:

Thanks, Doug.  Unfortunately, that didn't really help my particular issue because the text I need to extract is inconsistent from one email to the next.


However, for future reference, I did develop a workaround. This would apply to multiple emails that are identical other than the information sought (in my case, name, email, address, and phone number). The following procedure will allow us to create an Excel document that lists the names and contact info for all submitters and can be sorted by the submitter's last name.


  1. Go to email folder that contains the form letters.

  2. Sort email folder by subject and ensure that all copies of each form letter are together.

  3. Highlight all copies of the same form letter (shift-click works fine).

  4. Click File > Save As.

  5. Save as plain text (*.txt) file in an appropriate location with an identifiable name.

  6. Open the saved txt file.

  7. Turn on reveal paragraph marks (the ¶ button).

  8. Count (and remember) how many ¶ marks are in the text of a single email.

  9. Highlight the entire text of the document.

  10. On the Home tab, click in the font size box and enter 2. Click enter when done.

  11. With all document text still highlighted, click Insert > Table and select “Convert text to table.”

  12. In the number of columns box, enter the number of ¶ marks you counted before.

  13. Ensure the “Number of rows” box reflects the number of emails you exported from Outlook.

  14. Ensure “Separate text at” is set to Paragraphs.

  15. Click OK.

  16. Go to top of document.

  17. Delete columns that don't contain contact information (hopefully, all except the last 5 columns).

  18. Select all remaining columns.

  19. Change font size back up to normal.

  20. Save file under an appropriate new name with *.docx extension.

  21. Select all columns.

  22. Click Copy.

  23. Open new blank Excel worksheet.

  24. Click Paste.

  25. Adjust column sizes as appropriate.

  26. Save file under an appropriate new name with *.xlsx extension.

  27. Insert a new column A.

  28. In cell A1, enter (there’s a space between the “ “): =MID(B1&" "&B1,FIND(" ",B1)+1,LEN(B1))

  29. Extend formula (use the plus sign in the lower right corner) to rest of Column A.

  30. Proofread and fix mistakes – look for multiple names and middle names out of place

    1. “Marie and Pierre Curie” will end up as “and Pierre Curie Marie” – FIX

    2. “Mary Todd Lincoln” will end up as “Todd Lincoln Mary” – FIX

    3. “Nikolai Rimsky-Korsakov” will end up as “Rimsky-Korsakov Nikolai” – OK AS IS

  31. Save final version.

If you are creating your outline using heading styles, at least one of the options that Charles mentioned will be selected by default: "Keep with next." Word tries to keep any heading with the next paragraph, and the result will be that paragraphs keep "chasing" each other, which ultimately fails (because Word has to start a new page at some point), unless there are some "intervening" paragraphs using other (non-heading) styles. ringer114 say:

Excel INDIRECT function formatting

On 'Sheet1' I am trying to use one reference cell, B2, to refer to 'Sheet2' and a second reference cell, A2,  to specify the range of values on a row in Sheet2 that I want to copy to Sheet1, say D20:Z20 on Sheet2 is to be copied to A33:W33 on Sheet1

I then need to use a 3rd reference cell on Sheet1, say C2, to specify the number of rows below the first row that was copied, say 3, and copy that row of data, in this case D23:Z23, to a new location on Sheet1, say A43:W43

I am trying to do this with a copy/paste of an INDIRECT function specified for the starting location.

max, singapore say:

In Sheet1,
Select A33:W33
Put this in the formula bar: =INDIRECT("'"&B2&"'!"&A2)
then press CTRL+SHIFT+ENTER to confirm the formula (multi-cell array enter)

Likewise, select A43:W43,
put this in the formula bar: =OFFSET(INDIRECT("'"&B2&"'!"&A2),C2,)
then press CTRL+SHIFT+ENTER to confirm the formula (multi-cell array enter)

Hello,

Based on the information that you have provided, you are having issues with the Search functionality in Outlook. If you search for items in Outlook and don't receive the results that you expect, the indexing of your Microsoft Windows and Microsoft Office system data files (the search catalog) may not be complete. We would like to verify if you are having issues with Outlook.com or Windows 10 Outlook mail app?

Meanwhile, since you are done with Indexing and the same issue occurs, let us rebuild your search catalog. Here's how:

  1. Close Outlook and open Control Panel.

  2. Choose Indexing Options, or in the Search box, type Indexing, and then choose Indexing Options.

  3. In the Indexed Locations dialog box, under Change selected locations, select the check box for the location you want to modify, and then click OK.

  4. In the Advanced Options dialog box, on the Index Settings tab, under Troubleshooting, click Rebuild.

Please get back to us with the updated result.

The way that a document appears on the screen is, in part, controlled by the driver for the ActivePrinter.  Try selecting a different Printer, such as the Microsoft Print to PDF and see if that overcomes the problem.  If it does, it suggests that the driver for the printer that was the active printer is damaged.  Try re-installing the printer of replacing the driver by going to the Printer Manufacturer's website.
