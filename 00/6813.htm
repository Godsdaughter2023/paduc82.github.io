    tushar mehta say:

    Herbert's post reminded me I had solved this but not posted the solution. ;-)

    Suppose you have the following in A2:B12:

    #Widgets (Q) 65
    #parts/widget (W) 10
    #widgets-parts (T=Q*W) 650
    #machines (M) 8
    #widget-parts/machine (L=T/M) 81.25
    #widget-parts in parallel (P) 4
    #machine batches (B=L/P) 20.3125
    Speed - #batches/hr - (S) 60
    #hrs (H=B/S) 0.338542
    #full batches N=INT(W/M) 1
    #groups G=W-_N*M 2

    The cells in column B are named using the single letter names as shown in column A.  The exception is the cell that should be N but is _N (that's because N is an Excel function).

    Now, in A15:A22 enter machine ids.  To do so, in A15 enter =IF(ROW()-ROW($A$14)<=M,ROW()-ROW($A$14),"").  Copy to A16:A26.  In B14:K14 enter the widget part numbers.  To do so, in B14 enter the formula =IF(COLUMN()-COLUMN($A$14)<=W,COLUMN()-COLUMN($A$14),"") and copy to C14:K14.

    In B15 enter the formula =IF(OR(B$14="",$A15=""),"",IF(B$14>_N*M,IF(MOD(B$14,M)=INT(($A15-1)/(M/G))+1,(W-M*_N)/M*Q,""),IF(MOD(B$14-1,M)+1=$A15,Q,"")))

    Copy B15 to C15:K15.  Copy B15:K15 to rows 16:22.

    To read the result, read left to right to see what widget-parts will be done on a particular machine.  Read top to bottom to see which machines will be used for a particular widget-part.

    The one thing I have not solved is when a widget part is spread across multiple machines, you may get a non-integer answer.  I know how to solve it but for the time being you (or someone else) will have to adjust the non-integers into integers.

    jennifer bri say:

    Hi Denise,

    Since embedded graphs by group won't update, we suggest that you link objects instead.

    The main differences between linked objects and embedded objects are where the data is stored and how you update the data after you place it in the Word file.

    When an object is linked, information can be updated if the source file is modified. Linked data is stored in the source file. The Word file, or destination file, stores only the location of the source file, and it displays a representation of the linked data. Use linked objects if file size is a consideration.

    In your case, linking can be more useful then embedding when you want to include information that is maintained independently, such as data collected by a different department, and when you need to keep that information up-to-date in a Word document.


    Let us know if you any further inquiries.
