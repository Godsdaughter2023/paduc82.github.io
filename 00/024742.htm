jeeped say:
If Frequency is a named range and not a named table, your syntax is correct and I was able to reproduce your situation with a couple of sample workbooks but not your zero results.

If Frequency is a named table then you should use the following modification,

=IF('[Address Table.xlsx]OmniData from QLPROP'!$I$2<9,VLOOKUP(AN4,'Address Table.xlsx'!Frequency[#All],2,FALSE),"S")

... or,

=IF('[Address Table.xlsx]OmniData from QLPROP'!$I$2<9,VLOOKUP(AN4,'Address Table.xlsx'!Frequency[#Data],2,FALSE),"S")

VLOOKUP return a zero (e.g. 0) when it finds a match but the returning cell is blank. I would imagine that the name of the range or table is not properly defined. Check the definition of the range with Formulas, Name Manager.

Difference between Export File and Copying .pst file on Outlook2013 - Microsoft Community

acejakk say:

Difference between Export File and Copying .pst file on Outlook2013

The title says it all. I need to backup some IMAP email accounts mainly on outlook2013. Is it faster/better to use the export function or just going to the .pst folder and copying all of them?

karl timmermans [outlook mvp 2012-2018] say:

The title says it all. I need to backup some IMAP email accounts mainly on outlook2013. Is it faster/better to use the export function or just going to the .pst folder and copying all of them?

Well - first thing is that OL'2013 doesn't use PST files for IMAP Accounts purely for the sake of accuracy.

In terms of "backing up" some OL'2013 IMAP Accounts - some background items

#1 - If you wanted to make a backup of the local offline Imap datastore (.OST) you would HAVE to export to a PST file since copy the .OST file is of absolute no use once it's disconnected from the Outlook profile/email account that created it in the first place

#2 - Anytime you configure the IMAP Account, a new OST file will be created and the email items will be sync'd

#3 - The MAJOR item that really needs attention for an OL'2013 based IMAP email account are the folders marked with "This Computer Only". Those really need to be backed up regardless of anything else since you'll lost all the data in those folders if the OST file is ever disconnected (or deleted if you delete the IMAP email account itself).

In regards to item #3 - it's far safer to create a NEW PST file in a profile and make it the default data file for the profile where the OL'2013 Imap data file is designated as the default. That way, new contacts will be added to the contact folder in the PST file instead of the OST file.

  9. export your mail which now includes the deleted folder in a .olm file

  15. Fire up identity
