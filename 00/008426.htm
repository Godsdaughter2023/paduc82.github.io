mike h.. say:

Hi,

 

Put this in a cell and ARRAY enter it; see below on how to do that, and then drag down and it will return all the matches. When it runs out of matches it returns blanks.

 

=IFERROR(INDEX($N$1:$N$1000,SMALL(IF($P$1:$P$1000=1,ROW($A$1:$A$1000)-ROW($A$1)+1),ROWS(A$1:A1))),"")

 

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

kergga say:

Just a syntax issue with the reference; you have:

 

=INDEX('RMS Rates'!N:N,MATCH('RMS Rates'!1,P:P,0),0)

 

I think you want:

=INDEX('RMS Rates'!N:N,MATCH(1,'RMS Rates'!P:P,0))

 

Returning more than one value (an array of values) adds some complexity to your request. Do you need each individual value, or the count of how many matches, or something like the sum or average of those values?

 

If you need to return each one individually, then I would use a "helper column"; let's say you are trying to pull these values into a new sheet ('Sheet2'). I would delimit the range you are searching, so that each value excludes anything that was already found, e.g.:

 

Column A:

=Match(1,'RMS Rates'!P:P,0)  'this gives you the first match

=Match(1,Indirect("'RMS Rates'!P" & (A1+1) & ":P5000",0) 'this gives you the first match starting after the row where the first match was found. If your data set is significantly larger or smaller than 5000 rows, change that number accordingly

When you drag A2 down, the formula will autofill properly and give you the row number of each match until it finds no more matches, and the remaining cells on this sheet will just return an #N/A error. Drag down far enough that you will catch your expected number of values.

 

In column B, add your index statement and refer to the value in column A:

=INDEX('RMS Rates'!N:N,A1)

Drag down and it should autofill properly.

graham mayor say:
Every character in a Word document may individually have a proofing language applied. In order to use only one proofing language you must apply that language to all the text. The only times this normally becomes an issue is when either you have the Word language option set to automatically detect the language (turn it off) or when you copy text from other people's documents, such as from a web site, when you should apply the appropriate language to the imported text.

bob jones aka: cybertaz say:

As Graham explained, Language is a property of the text, itself, within any given document. It is not affected by whatever you set as your Default Language. Nor can you 'disable' Word's Proofing Tools without risking failure of the software.

The Default Language you choose pertains only to new documents you create from that point on. It does not change anything in existing documents regardless of who created them. The Languages applied in those documents is determined by the users who edited them, whether intentionally or accidentally.

In order to change the Language formatting in any given body of text - single character or entire document - you need to first select that text, then go to Tools> Language to choose the appropriate Language. Language can also be specified by including it in the definition of a Style [which is the preferred means of applying virtually all formatting].

When using the Spelling & Grammar check Word uses the appropriate Proofing Tools based on which Languages it finds in the document. IOW, if the entire document is formatted as English (US) it has no reason to use the Proofing Tools of any other Language.

See this article for a more detailed explanation:  Spelling, Language, Dictionary

IRR function
Show All
Hide All

This article describes the formula syntax and usage of the IRR function (function: A prewritten formula that takes a value or values, performs an operation, and returns a value or values. Use functions to simplify and shorten formulas on a worksheet, especially those that perform lengthy or complex calculations.) in Microsoft Excel.

Description

Returns the internal rate of return for a series of cash flows represented by the numbers in values. These cash flows do not have to be even, as they would be for an annuity. However, the cash flows must occur at regular intervals, such as monthly or annually. The internal rate of return is the interest rate received for an investment consisting of payments (negative values) and income (positive values) that occur at regular periods.

Syntax 
IRR(values, [guess])

The IRR function syntax has the following arguments (argument: A value that provides information to an action, an event, a method, a property, a function, or a procedure.):

  • Values  Required. An array or a reference to cells that contain numbers for which you want to calculate the internal rate of return.
    • Values must contain at least one positive value and one negative value to calculate the internal rate of return.
    • IRR uses the order of values to interpret the order of cash flows. Be sure to enter your payment and income values in the sequence you want.
    • If an array or reference argument contains text, logical values, or empty cells, those values are ignored.
  • Guess  Optional. A number that you guess is close to the result of IRR.
    • Microsoft Excel uses an iterative technique for calculating IRR. Starting with guess, IRR cycles through the calculation until the result is accurate within 0.00001 percent. If IRR can't find a result that works after 20 tries, the #NUM! error value is returned.
    • In most cases you do not need to provide guess for the IRR calculation. If guess is omitted, it is assumed to be 0.1 (10 percent).
    • If IRR gives the #NUM! error value, or if the result is not close to what you expected, try again with a different value for guess.
Remarks

IRR is closely related to NPV, the net present value function. The rate of return calculated by IRR is the interest rate corresponding to a 0 (zero) net present value. The following formula demonstrates how NPV and IRR are related:

NPV(IRR(B1:B6),B1:B6)
equals 3.60E-08 [Within the accuracy of the IRR calculation, the value 3.60E-08 is effectively 0 (zero).]

You don't need to re-declare z, x, and y, as you are passing them to the function already. Also, you shouldn't declare result as an integer, as an integer can't hold decimal values, and you could get decimal value return when you divide.

Try just declaring it, which will make it into a variant.

Function CALC(x As Integer, y As Integer, z As String)

Dim result

Select Case z
     Case Is = "add"
         result = x + y
     Case Is = "mult"
         result = x * y 
     Case Is = "div"
         result = x / y
 End Select

CALC = result

End Function
