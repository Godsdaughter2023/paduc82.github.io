quick scan only took less than 2 minutes and did only 12900 items. It - Microsoft Community

katajoe say:

quick scan only took less than 2 minutes and did only 12900 items. It has never done a scan that fast. Is something wrong?

I noticed that MSE caught 3 trojans earlier this month and removed them.  Today when it ran a quick scan, the time was only 1 min 58 seconds and scaned 12900 items.  I was surprised at both the short time and the limited items.  Is anything wrong?

 

rob koch say:

Assuming this is a recently modern PC within a few yers old these times and numbers of files seem reasonable.

The reason the number of files may seem to be less is that MSE will only scan about a third of the files it does if the MSE Real-time protection has been disabled recently.  Since restarting the system shuts down the Real-time protection, it will always need to run a larger scan if this has occured since the previous scan.

You can prove this yourself simply by restarting the PC or just going into the MSE Settings and disabling and then immediately re-enabling Real-time protection, as this will trigger the requirement to scan more files.  Then immeditaely scan again and the file count will be reduced to the smaller number again.

This is a symptom of a caching feature of the scanner which recognizes these differences and acts accordingly.  If MSE has been continuously running it has more confidence about the protected status of your system and so doesn't need to scan everything that it does when the recent status of the operating system is unknown.

Rob

john p spencer say:

Records in tables do not have a specific order.  The order is imposed by using sort specifications in a query.  You can think of the table of records as a big bag of marbles.  The only way to impose an order on the marbles is to have a number on each marble and after you pull it out of the bag you can use the numbers to line them up in an order.

If you predefine the table, you can assign a sequential autonumber field and that will normally keep the records in order (you can sort by the autonumber field).
