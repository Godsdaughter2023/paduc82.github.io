That's a bit complicated because stepping into the subform saves the parent form. And for good reason.

So your only recourse is when the user arrives at the next record, to "look back" at what just happened and if no sobform records to delete the parent record. This would be the Form_Current event. ken sheridan say: The only way you could ensure that the new row in the referenced table is not saved without a row being inserted into the referencing table would be to use an unbound form and insert the rows into both tables as a transition.  The following is a somewhat crude and unrealistic little procedure which demonstrates how this would work:

Public Sub TranactionDemo()

    ' this demo procedure uses two empty tables, Families and FamilyMembers
    ' related on the FamilyName columns.
    
    ' the second SQL statement attempts to insert a row for family name 'Black'
    ' into FamilyMembers for which there is no row in Families, violating
    ' referential integrity and causing the transaction to be rolled back, so
    ' no row is inserted into either table

    On Error GoTo Err_Handler
    
    Dim strSQL As String
    Dim strCriteria As String
    
    DBEngine.BeginTrans
        
   strSQL = "INSERT INTO Families(FamilyName) " & _
        "VALUES(""White"")"
    CurrentDb.Execute strSQL, dbFailOnError
   
    strSQL = "INSERT INTO FamilyMembers(FamilyName, Firstname)  " & _
        "VALUES(""Black"",""John"")"
    CurrentDb.Execute strSQL, dbFailOnError
    
    DBEngine.CommitTrans
    
    ' after the transaction has been rolled back change the second SQL statement to insert
    ' a row into FamilyMembers for John White, and run the procedure again.
    ' the transaction should now be committed and rows inserted into both tables
    
Exit_Here:
    Exit Sub
    
Err_Handler:
    MsgBox Err.Description
    DBEngine.Rollback
    Resume Exit_Here
    
End Sub

Otherwise you can do as Tom describes and delete the row in the referenced table after navigating off it (including closing the form) if no matching row exists in the referencing table.  rather than doing it on a row by row basis you could do it when the form closes by deleting all rows from the referenced table which have no matches in the refining table, taking my Families and FamilyMembers tables as an example:

Const conSQL = _
    "DELETE * " & _
    "FROM Families " & _
    "WHERE NOT EXISTS(" & _
        "SELECT * & _
         "FROM FamilyMembers " & _
         "WHERE FamilyMembers.FamilyName = Families.FamilyName)"

CurrentDb.Execute conSQL, dbFailOnError

The problem with deleting rows of course is that they are then lost and would have to be re-entered with matching rows in the referencing table, whereas what I assume you are aiming at is to force the user to enter at least one row.  For this you could look for unmatched rows in the referenced table, and take the user back to that record in the form, e.g.

Const conCRITERIA = _
    "NOT EXISTS(" & _
        "SELECT * " & _
         "FROM FamilyMembers " & _
         "WHERE FamilyMembers.FamilyName = Families.FamilyName)"

Dim varFamilyName As Variant
Dim strMessage As String

varFamilyName = DLookup("FamilyName", "Families", conCRITERIA)

If Not IsNull(varFamilyName) And varFamilyName <> Nz(Me.FamilyName, "") Then
    strMessage = "A row for family " & varFamilyName & _
        " must be entered in the Family Members table."
    MsgBox strMessage, vbExclamation, "Incomplete Data"

    With Me.RecordsetClone
        .FindFirst "FamilyName = """ & varFamilyName & """"
        If Not .NoMatch Then
             Me.Bookmark = .Bookmark
        End If
    End With
End If

You can put this code in the parent form's Current event procedure for when the user navigates to another record.  To cater for the user trying to close the record you can put it in the form's Unload event procedure, but in this context you'd also have to set the return value of the event procedure's Cancel argument to True within the If....End If Construct, bit would not have to test for the key (FamilyName) value not equalling that of the current record (this is necessary in the Current event to prevent the code executing again when the form moves to the miscreant record.  So the code fore the Unload event procedure would be;

Const conCRITERIA = _
    "NOT EXISTS(" & _
        "SELECT * " & _
         "FROM FamilyMembers " & _
         "WHERE FamilyMembers.FamilyName = Families.FamilyName)"

Dim varFamilyName As Variant
Dim strMessage As String

varFamilyName = DLookup("FamilyName", "Families", conCRITERIA)

Me.Dirty = False

If Not IsNull(varFamilyName) Then
    Cancel = True
    strMessage = "A row for family " & varFamilyName & _
        " must be entered in the Family Members table."
    MsgBox strMessage, vbExclamation, "Incomplete Data"

    With Me.RecordsetClone
        .FindFirst "FamilyName = """ & varFamilyName & """"
        If Not .NoMatch Then
             Me.Bookmark = .Bookmark
        End If
    End With
End If

Try this code from Bob Phillips to get a sheet menu.


Sub BrowseSheets()
Const nPerColumn  As Long = 38          'number of items per column
Const nWidth As Long = 13                'width of each letter
Const nHeight As Long = 18              'height of each row
Const sID As String = "___SheetGoto"    'name of dialog sheet
Const kCaption As String = " Select sheet to goto"
                                        'dialog caption


Dim i As Long
Dim TopPos As Long
Dim iBooks As Long
Dim cCols As Long
Dim cLetters As Long
Dim cMaxLetters As Long
Dim cLeft As Long
Dim thisDlg As DialogSheet
Dim CurrentSheet As Worksheet
Dim cb As OptionButton
    Application.ScreenUpdating = False
    If ActiveWorkbook.ProtectStructure Then
        MsgBox "Workbook is protected.", vbCritical
        Exit Sub
    End If
    On Error Resume Next
        Application.DisplayAlerts = False
        ActiveWorkbook.DialogSheets(sID).Delete
        Application.DisplayAlerts = True
    On Error GoTo 0
    Set CurrentSheet = ActiveSheet
    Set thisDlg = ActiveWorkbook.DialogSheets.Add
    With thisDlg
        .Name = sID
        .Visible = xlSheetHidden
        'sets variables for positioning on dialog
        iBooks = 0
        cCols = 0
        cMaxLetters = 0
        cLeft = 78
        TopPos = 40
        For i = 1 To ActiveWorkbook.Worksheets.Count
            If i Mod nPerColumn = 1 Then
                cCols = cCols + 1
                TopPos = 40
                cLeft = cLeft + (cMaxLetters * nWidth)
                cMaxLetters = 0
            End If
            Set CurrentSheet = ActiveWorkbook.Worksheets(i)
            cLetters = Len(CurrentSheet.Name)
            If cLetters > cMaxLetters Then
                cMaxLetters = cLetters
            End If
            iBooks = iBooks + 1
            .OptionButtons.Add cLeft, TopPos, cLetters * nWidth, 16.5
            .OptionButtons(iBooks).Text = _
                ActiveWorkbook.Worksheets(iBooks).Name
            TopPos = TopPos + 13
        Next i
        .Buttons.Left = cLeft + (cMaxLetters * nWidth) + 24
        CurrentSheet.Activate
        With .DialogFrame
            .Height = Application.Max(68, _
                Application.Min(iBooks, nPerColumn) * nHeight + 10)
            .Width = cLeft + (cMaxLetters * nWidth) + 24
            .Caption = kCaption
        End With
        .Buttons("Button 2").BringToFront
        .Buttons("Button 3").BringToFront
        Application.ScreenUpdating = True
        If .Show Then
            For Each cb In thisDlg.OptionButtons
                If cb.Value = xlOn Then
                    ActiveWorkbook.Worksheets(cb.Caption).Select
                    Exit For
                End If
            Next cb
        Else
            MsgBox "Nothing selected"
        End If
        Application.DisplayAlerts = False
        .Delete
    End With
End Sub

teylyn say:

Hello,

Please, you DO NOT need to re-install anything. The behaviour you describe is by design.

When you select a cell and copy it, you can only paste that as a whole cell.

If you want to append the contents of a cell to another cell, you need to double-click the first cell, select the cell contents and copy it. After that, you can double-click another cell and paste what you copied to the end of the second cell. 

In the animated screenshot, Text One first gets copied as a whole cell and when pasting onto the cell with Text Four, it overwrites the whole cell. Next, I double-click and select Text One inside the cell. Now I can double-click the cell with Text Three, go to the end and paste Text One.
