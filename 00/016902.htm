macropod say:

is it possible to add a inputbox instead of a fixed comment under " Do While.Find.Found?

To be able to have a comment other than the fixed 'This is wrong, please fix your mistake', you could use code like:

Sub Demo()
Application.ScreenUpdating = False
Dim i As Long, StrCmnt As String
With ActiveDocument.Range
  With .Find
    .ClearFormatting
    .Replacement.ClearFormatting
    .Text = InputBox("What is the Text to Find?", , Selection.Text)
    StrCmnt = InputBox("What is the Comment to add?")
    If Trim(StrCmnt) = "" Then
      If MsgBox("There is no comment text. Exit Sub?", vbYesNo) = vbYes Then Exit Sub
    End If
    .Replacement.Text = ""
    .Forward = True
    .Wrap = wdFindStop
    .Format = False
    .MatchCase = False
    .MatchWholeWord = False
    .MatchWildcards = False
    .MatchSoundsLike = False
    .MatchAllWordForms = False
    .Execute
  End With
  Do While .Find.Found
    i = i + 1
    .Comments.Add Range:=.Duplicate, Text:=StrCmnt
    .Collapse wdCollapseEnd
    .Find.Execute
  Loop
End With
Application.ScreenUpdating = True
MsgBox i & " instances found."
End Sub

Note that the code checks to see if the comment text is empty and, if so, gives you the option of exiting the sub. If you choose to proceed under such circumstances, you'll end up with a bunch of empty comments.

Can Excel generate emails based off of conditional formating. - Microsoft Community

gotaquestion1 say:

Can Excel generate emails based off of conditional formating.

I have a spread sheet that has conditional formating set up to change a color when a certian condition happens (i.e percentage occurs) .  Is it possible to set it up so that when that color is changed excel generates an email with a copy of that row to notify us that this condition has occured.
jllatham say:

Not directly.  There's no 'event' triggered when conditional formatting kicks in and changes some format feature.  Additionally, you cannot simply test the cell to see what color it is because the answer you get is its default shading.

Now, what you could do is set up a process that examines the value of the cell in question using the same testing that the conditional formatting does and if the results are such that a conditional formatting change is presumed to have taken place, then kick off the email.

Simple example - Cell A1 on Sheet1 has conditional formatting set based on its value, and when its value is above 100, it turns red.

In code you could set up to test it this way:
If Worksheets("Sheet1").Range("A1") > 100 Then

 'code to kick off email, and somehow remember that you've alerted on this situation
 'might even check to see if you've already alerted on it previously and inhibit email send.

End If

As you might infer, there are several ways of getting this done and which way to do it depends on how cell A1 gets its result in this case.  If it is somehow related to user keyboard entry somewhere then you could test after the user has made the entry (test would actually be part of a Worksheet_Change() event for the sheet the user types into).

If you have some type of timed event getting information automatically from say a web page or database, then make the check part of that process.

You can see there are other considerations also: like preventing the routine from kicking off an email everytime the cell is checked and found to meet the criteria.  One way to deal with it would be to have another cell, possibly on a hidden sheet, that is a 'flag' cell.  And when the criteria is met and an email sent, then set the value of that cell to non-zero, and when the value falls outside of the criteria range (no longer conditionally formatted) then set that flag cell to zero so that the next time the value goes out of threshold a new email is sent. 

There are lots of considerations to deal with, but in the end, while the answer is a qualified YES, it is still possible if you want to do it bad enough.
