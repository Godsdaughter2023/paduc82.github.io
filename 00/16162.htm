john582n say:

You could try this registry edit to fix your screen resolution.

  1. Press the Windows key + R simultaneously to launch Run.
  2.  Type in regedit and press Enter to launch the Registry Editor.
  3. Press Ctrl + F simultaneously to open the Find dialog box.
  4. In the Find dialog box, type in this: Display1_DownScalingSupported then click on Find Next button and Windows will automatically search for you the registry key and high light it for you.
  5. Click on the high lighted key (Display1_DownScalingSupported) and change the Value from 0 to 1.
  6. Click on Find Next and do the same change to the second one (if it is present)
  7. Once finished, exit the Registry Edit, restart your PC.
  8. After you've restarted, go to your Desktop, right mouse click, choose Screen Resolution and you should be able to choose 1024x768. Please choose this option, save and close.

Perform these checks, especially the bolded ones:

antoniosantos927 say:

This is a big problem for us as we use both features extensively.  We've set up links to Access queries and these are brought in to Excel 2010 as tables automatically.  Custom Views work great if your program has multiple users with difrent data view requirements; as you can set up views  that are designed with each user's needs in mind. 

 

We're faced with picking the tables feature because it's too cumbersome to do links to txt files.  In our case we have to run crystal reports out of our accounting system and save the report to a specified location that an excel data connection can then link to.  If you use that method,  your custom views will still work.  For us, the convenience of an access link that can refresh at a specified interval trumps the convenience of the custom views feature.  One of our team members suggested creating worksheets that just mirror what's in your main worksheet,  and then you can use auto filters and hidden columns in each sheet to accomplish the same function as custom views.

 

In your new sheet, in cell A1 hit = then point to cell A1 in your main sheet. In the new sheet copy paste the formula to the rest of the worsheet.  Of course, formattinhg changes made in the main sheet won't be reflected in the new sheet(s), but that may not be a big issue depending on the purpose of the view.

 

I've tried to contact the Excel team a few weeks ago,  but no luck.
