If the data source is a Word Document, here is a macro which, if run when the Mail Merge Main Document is open, will insert the necessary number of empty rows before the first row containing data and then execute the merge to a new document which you can then print to your label sheets.

Macro to set the first label on a part sheet of labels for a label type mailmerge.

Dim MMMDoc As Document

Dim dsource As Document

Dim dtable As Table

Dim i As Long, j As Long

Set MMMDoc = ActiveDocument

With MMMDoc.MailMerge

    If .MainDocumentType = wdMailingLabels Then

        If .State = wdMainAndDataSource Then

            Set dsource = Documents.Open(.DataSource.Name)

            Set dtable = dsource.Tables(1)

            i = InputBox("Enter the number of labels that have already been used on the sheet.", "Set Starting Label")

            If IsNumeric(i) Then

                With dtable

                    For j = 1 To i

                        .Rows.Add BeforeRow:=.Rows(2)

                    Next j

                End With

            End If

            .Destination = wdSendToNewDocument

            .Execute

        End If

    End If

End With

dsource.Close wdDoNotSaveChanges

Sub setFooter()
Dim strText As String
Dim osld As Slide
Dim ocust As CustomLayout

On Error Resume Next

strText = ActivePresentation.Name
If InStr(strText, ".") > 0 Then
strText = Left(strText, InStr(strText, ".") - 1)
End If

getFooter(ActivePresentation.Designs(1).SlideMaster).TextFrame.TextRange = strText
For Each ocust In ActivePresentation.Designs(1).SlideMaster.CustomLayouts
getFooter(ocust).TextFrame.TextRange = strText
Next
For Each osld In ActivePresentation.Slides
osld.HeadersFooters.Footer.Visible = True
getFooter(osld).TextFrame.TextRange = strText

Next
End Sub

Function getFooter(osld As Object) As Shape
Dim oshp As Shape
For Each oshp In osld.Shapes
If oshp.Type = msoPlaceholder Then
If oshp.PlaceholderFormat.Type = ppPlaceholderFooter Then
Set getFooter = oshp
Exit Function
End If
End If
Next oshp
End Function

doug robbins - mvp office apps & services (word) say:

To use the Send To option in Office applications with Windows Live Mail, make the following changes to the Window Registry.
Note: Just in case something goes wrong, you should first back up the registry by following the steps in the following article:
http://windows.microsoft.com/en-US/windows7/Back-up-the-registry
Now, back to the original issue:
1. Click on Start > All Programs > Accessories > Run > Type regedit
2. Browse to HKEY_LOCAL_MACHINE > Software > Client > Click on Mail
3. Check what appears in the right pane. It should read as below:
   (Default)         REG_SZ         Windows Live Mail
4. If Windows Live Mail does not appear, right click on (Default) and click on Modify and in the Value Data box, type
Windows Live Mail
Note: If you don’t find the Mail item, right click on Client and Click New > Key and rename the key to:
Mail
Once the Mail key is created, right click on Mail > New > String Value. Name the value as:
(Default)
Right click on (Default) and Click Modify. In the Value Data box type
Windows Live Mail

While doing that should allow you to use the Send facility in Word, and may also allow you to execute a merge to email, I am not sure about the latter.
