It is not complicated, but you do need to realise that there is the ability to have a Header\Footer on the first page of a document or Section of a document that is different from that on the second and subsequent pages of the document or Section of it.  And, also, to have different Headers\Footers on Odd and Even pages.

In your case, you need to use the Different First Page option, and remove any page numbering that you have set up in the Header or Footer of the first page and then use the Next in the Navigation section of the Header & Footer Tools>Design tab of the ribbon to move to the Header\Footer of the second and subsequent pages (referred to as the Primary Header or Footer) and then with the selection in the required position, press ALT+SHIFT+p to insert a page number field. 

If you want the number on that page to start at 1, you will need to Page Number Format dialog and set the numbering to Start at 1.

To get to the Page Number Format dialog, expand the Page Number dropdown in the Header & Footer Tools>Design tab of the ribbon and then click on "Format Page Numbers..."

hansv mvp say:

In your second example, strRGB is the literal string value "RGB(230, 190, 255)". So the line

Me.txt1.BackColor = strRGB

tries to assign a string value to BackColor. It does not execute the RGB function since RGB is just part of the string.

You would have to parse the MyColourRGB string to extract the red, green and blue values, and feed those to the RGB function, much as you do in the first example.

Alternatively, replace MyColourRGB with 3 number fields MyColourR, MyColourG and MyColourB. You can then use

Dim lngRGB As Long

lngRGB = RGB(Me.MyColourR, Me.MyColourG, Me.MyColourB)

Me.txt1.BackColor = lngRGB

(I used a number variable instead of a string variable since BackColor is a numeric property)

There is a really easy way to fix this, especially if the following applies ot you:

 

*  You are using Microsoft Outlook

*  Outlook is communicating with an Exchange Server (business environment)

 

First, go into your Sync profile and edit the sync content settings.  You'll probably note that ActiveSync has automatically selected certain items to sync from the Exchange Server (mail, etc) and other items from your Desktop PC or Workstation (Notes, etc).

 

Pay attention here:  If you are using Outlook in conjunction with Exchange Server, Outlook is already doing all the work for you and having your PDA try to communicate with the Exchhange Server is redundant.  Because of recent NTLM authentication changes to Exchange Server via automatic Windows Updates that you are almsot completely unaware of, it is also often problematic, as you've already been finding out.

1.  Go into your ActiveSync profile for your device and uncheck all the items from the Exchange Server that AS is syncing with your PDA.  Fret not, this is harmless, and we're not done yet. 

 

2.  Next, do you see all those unchecked boxes below for your local computer?  Those are for Microsoft Outlook.  Check them all.  Your PDA will now sync with everything in Outlook, which already contains the same data it would have pulled down from the Exchange Server - only this will work and not cause additional network traffic or authentication delays/problems.
