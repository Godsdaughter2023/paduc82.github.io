a. user say:

I have a feeling there may be more to the message than just that.

In computer terminology, a 'library' usually means a file that has code for functions that a program performs frequently. Rather than write the code for those functions over and over, the programmer tells the program to go look for the code in the library.

Obviously, if there is some problem with the library, the program can't run and will instead show a message like the one you saw.

Typically, the error message tells you which library has the problem, with a file name that often ends in .dll  . If that's the case, just reinstall the program and the library will be restored. If the library is part of Windows, you may have re-install Windows.

jeff liebermann say:

Bingo.  That was the fix for me on two machines.  I'm writing this from memory so please forgive my lack of detail.

On one machine, I was dealing with a customer who performed a last minutes free Win 7 Pro 64 bit to Win 10 upgrade.  Win 10 worked normally, except that the print spooler kept dying, as described in other articles in this thread.  Adding a new printer resulted in a mess.  I tried the usual fixes, such as the MS Printer Troubleshooter, without any luck.  I did NOT try "refresh" as that would have wiped far more than just the printer configurations.

After reading the above article from SMBTech Solutions LLC, I ran regedit and dived into the registry at:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Print\Environments\Windows x64\Print Processors

I found the remains of various printers and scanners that were present in Win 7, but which the upgrade had tried and failed to remove.  Also bits and pieces of the drivers and their printer definition files.  What I guess was happening was the print spooler was trying to load one of these trashed drivers in order to determine if the printer is present during startup.  When it tripped across the trashed drivers, the spooler crashed, issueed a useless error message to the logger, rolled over, and died..

Per the advice in the article, I made a backup of the registry key.  Using regedit, I manually removed all the drivers except for winprint.  Much to my amazement, upon rebooting, all the remaining printers appeared in the control panel app.  Also making an appearance was the MS PDF printer, which I failed to notice was missing.  After undoing the damage that  I did in Windoze Services trying to keep the spooler service from dying, I was able to reinstall the various printers and scanner without difficulty.

The 2nd machine was easier.  Again, the various entries under the aforementioned registry key were a mess.  However, I had the foresight to save a copy of the registry key from a clean working system with no printers installed.  I simply exported the existing mess to a file as a backup, deleted the existing key, and replaced it with the saved key from the working machine.  It's kinda a "heavy hammer" solution, but it was much less work than playing with regedit.  After a reboot, it worked just fine.

In checking with friends in the same business, I found 4 more incidences of machines with the same problem.  The common factor is that they all happened as a result of a Win 7 to Win 10 upgrade. 

I hope this helps.

Hi Rohan, 

It is possible that your TCP/IP is corrupted causing the issue to occur. What we can recommend is that you perform these troubleshooting and check if there are any changes: 

Reset TCP/IP  

To do this, follow these steps:  

  1. Run Command Prompt as Administrator.  

  2. Run this command:  

    • netsh int ip reset and hit Enter.  

Exit Command Prompt and monitor your network connection.  

Reconfigure TCP/IP 

To do this, follow these steps:  

  1. Run Command Prompt as Administrator.  

  2. Run these following commands:  

    • netsh int tcp set heuristics disabled  

    • netsh int tcp set global autotuninglevel=disabled  

    • netsh int tcp set global rss=enabled  

Exit Command Prompt and monitor your network connection. 

Let us know how it goes.
