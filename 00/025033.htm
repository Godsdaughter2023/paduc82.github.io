Yes, it is possible to change the email address of a plan without recreating a new one.

When we add a new plan in Office 365 Planner, a corresponding public group will be created. We can change the plan's email address (actually the PrimarySmtpAddress of the corresponding public group) via PowerShell commands:

1. Launch Windows PowerShell and run the following commands one by one to connect to Exchange Online:

$UserCredential = Get-Credential

$Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri https://outlook.office365.com/powershell-liveid/ -Credential $UserCredential -Authentication Basic -AllowRedirection

Import-PSSession $Session

2. Run
Get-UnifiedGroup | select primarysmtpaddress,displayname,identity

3. Copy the Identity of the specific plan (group) (select and right click) and paste it to a notebook.

4. Run
Set-UnifiedGroup -Identity testplan002_a5eb70f93e -PrimarySmtpAddress testplan005@domain.onmicrosoft.com

hallie bourne say:

I discovered the solution on a discussion thread, which I linked to in my post above (click the word "thread").

Here is more detail from what I learned:

When you create a caption, Word detects the type of layout on the selected image.

  • If the image is "inline" with text, Word just creates a caption that is also inline with text.
  • If the image is laid out any other way--in other words, if the image is floating--Word creates the caption inside a text box so that it, too, will float.

You don't need to delete and recreate the captions. You can just cut/copy the caption and paste it outside of the text box. Then, delete the text box.

  • If the image is centered on the page (or left justified with no text wrapping), you can probably set it to be "inline" and then keep the caption text inline directly below the image.
  • If the image is right justified or has text wrapped around it, you need to create a table and put the image and the caption inside the table. Then position the table as you would have positioned the image (best to keep the image property "inline" relative to the table).  

To summarize, when captions are in text boxes, the external link that Word creates in the ToC field, besides creating problems upon conversion to PDF, is also a security flaw because it reveals the path to the file on the computer where the document was last modified (or where the ToC was updated). These are supposed to be internal/relative links. And as I said, I am flabbergasted that this has not been fixed since at least 2014 when the aforementioned discussion thread was created.
