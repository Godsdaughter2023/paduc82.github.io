Quotes in a formula - Microsoft Community

kelli.w say:

Quotes in a formula

I have created a formula in Excel 2007 in the form, =COUNTIF(F2:F9176,"<5/1/2009").

It does not work, obviously, without the quotes around the date criteria.  Can someone please explain why the quotes are required?  I thought that dates were treated as numerical values, as per the date serial number system.

Thank you  -  Kelli

t. valko say:

I have created a formula in Excel 2007 in the form, =COUNTIF(F2:F9176,"<5/1/2009").

It does not work, obviously, without the quotes around the date criteria.  Can someone please explain why the quotes are required?  I thought that dates were treated as numerical values, as per the date serial number system.

Thank you  -  Kelli


The quotes are only needed when the criteria is a hardcoded text value:

COUNTIF(A:A,"Yes")

Or, when using a comparison operator like these:

  • < (less than)
  • <= (less than or equal to)
  • > (greater than)
  • >= (greater than or equal to)
  • <> (not equal to)
  • = (equal to, although, this operator is not needed because the implied comparison is equal to. For example: COUNTIF(A:A,"Yes"). Count if column A is equal to Yes.)

If the criteria needs a comparison operator and the criteria is a cell reference then you'd use this syntax:

B1 = 5/1/2009

=COUNTIF(F2:F9176,"<"&B1)

Or, if you were to use the DATE function:

=COUNTIF(F2:F9176,"<"&DATE(2009,5,1))

--
Biff
Microsoft Excel MVP

 

luke moraga say:
Yes, dates are treated as numbers, but not in that format. Additionally, in this COUNTIF arguement, you're specifying a "less than" arguement, which must be enclosed in quotes. Alternative formulas (assuming you have a date in A2):

=COUNTIF(F2:F9176,"<"&39934)
=COUNTIF(F2:F9176,"<"&A2)

slance310 say:

Hi,

You can not switch from Legacy to UEFI after the installation is complete...

The OS drive is created differently...

Legacy Bios requires the OS drive to be Initialized as "MBR" type and formatted NTFS...

UEFI Bios requires the OS drive to be Initialized as "GPT" type and formatted NTFS...

UEFI Bios also requires a small FAT32 "EFI" partition where it keeps the boot files for "WindowsBootManager"... It will create this during the installation..

You need to choose which Bios you want to use, then install accordingly...

It would be best to choose a "Custom Install" so you have a better idea, and more control over what is happening...

Another important issue... Installing to UEFI from a DVD is tricky... A DVD normally installs into Legacy Bios...

A workaround is to select your "Boot Options" at the POST screen by hitting F9, F11, F12, ESC, or whatever your machine uses...

You should then see both a CD\DVD drive and a UEFI CD\DVD drive...

Select the UEFI CD\DVD drive...

How can I crop EXACTLY (EG 1.25CM) from left AND/OR EXACTLY (eg 0.67cm) from right of an inserted image
steve rindsberg say:

Thanks for the extra explanation. 

Now I see very clearly what you mean.  The new dialog complicates things, no question.  But to its credit, it adds a bit more flexibility at the same time.

Instead of directly cropping as before, you now adjust the size and position of a kind of cropping window.  Instead of lopping some of the right and bottom, you now move the window to the left or top using the Width and Height controls under Crop Position.

Using the Left and Top controls in the same Crop Position area move the WINDOW to the left/right or top/bottom, meaning that you're moving the other side of the crop as well.

Best bet, I think:  use the Left and Top controls first to get the left and top sides the way you want them, then use Width and Height to get the right and bottom sides.

The useful addition is that you can now move the picture around behind the crop window if you like, using the Picture Position controls (which I'd have put below Crop Position but they didn't ask me, o well.)
