Mail merge files are very portable, the links between a mail merge main document and its data source are more fragile, though. In regular work, it is rare to have a data source and a main merge document be in the same folder.

If you teach your students to break that link before trying to transport a mail merge and to re-establish it when they are trying to use the merge, you will be teaching them a valuable Word skill. I create merges on a daily basis where one main merge document is attached to different data or the same data stored in a different place. My routine is that when I finish with the merge, if I am saving the main merge document for re-use, I save it as a normal Word document, as suggested by Doug Robbins.

If the main merge document and the data source will always be in the same relative locations to one another on a computer, a macro can be written that will attach the data source.

The following macro attaches an excel data list with heading columns named "Student_merge_data.xlsx" stored in the same folder as the saved document.

Sub AttachStudentData()
'   Written by Charles Kenyon
'   19 April 2005 revised 12 March 2014
''
'   Makes activedocument a mailmerge (letter) document and
'   attaches Student_merge_data.xlsx from same folder.
'
'
    On Error Resume Next
'
'   Name of file
    Dim strFileName As String
    Dim strProvider As String
'
    strFileName = ActiveDocument.Path & "\" & "Student_merge_data.xlsx"
'
'   Attach Merge list
        
    With ActiveDocument.MailMerge
        .MainDocumentType = wdFormLetters
        .OpenDataSource Name:=strFileName, _
            SQLStatement:="SELECT * FROM `Sheet1$`" ', _
'
'       Show merge data
        .ViewMailMergeFieldCodes = False
    End With
'
End Sub

For more on using macros from forums see Installing Macros.
