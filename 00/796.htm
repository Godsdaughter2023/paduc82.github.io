I am afraid my friend Stefan is not correct.   As in any style that is based on another style, you can make changes to the child style, however the attribute that you change will not longer be updated to match the parent style if you modify that attribute in the parent style.  Thus, you can modify the color of the font, and hence the numbering in the child style independently of the colour of the font used in the parent heading style.  However, if you subsequently change the colour of the font in the heading style, that will be no change in the colour of the font used for the child style.

vincentwu_msft say:

Hi Jim,

Whether you specify version number or not, Office Deployment Tool(ODT) will always download the whole Office installation package regardless of whether specific version exists. Without specifying version number, the downloaded Office pakcage will default to the latest version.

It is not feasible to prevent ODT from downloading a version it already has, however, you can slightly adjust your scheduled task to make sure that there are no duplicates. As new builds are released on the 2nd Tuesday of every month (ie, “Patch Tuesday”), you’ll need to update your internal source(network share) after a new build is released to the Microsoft Content Delivery Network (CDN). Since the CDN is globally available, typically a build is available at all locations in the CDN within a day of Patch Tuesday. So, updating your source a day or so after Patch Tuesday will make sure that you have the latest build with different version. For detailed information, please refer to:
https://blogs.technet.microsoft.com/office_resource_kit/2014/01/28/managing-updates-for-office-365-proplus-part-2/

Let us know if you need further assistance.

sidewinder---01 say:

Transfering All Contacts from 6230i to 8800 via Bluetooth

Is it possible to transfer all my contacts (over 300 of them) to my new 8800 without having to do them one by one - what is the easiest & quickest way to do it - thanks

petrib say: If you have a SIM card big enough, the easiest way is to copy the contacts to the SIM card.

Another alternative is to use Nokia PC Suite (http://www.nokia.com/pcsuite) and synchronize with a PC app like MS Outlook or Lotus Notes (synchronize first with the old phone to get the contacts on the PC and then with the new phone to get them from the PC).

embedded_system say: You could also use content copier (part of PC Suite).
No need for Outlook/LN.

skipraider say: If you use the PC Suite content copier, choose the third option, "transfer contents from PC to phone" . The "back-up" option will only transfer data back to the phone it came from.

embedded_system say: No need for PC Suite.
You can sync all contacts via BT.

Instructions:
- Put SIM in 6230i
- Remove SIM from 8800
- Switch 6230i on
- Switch 8800 on
- Open slide of 8800
- Press select ('Transfer')
- Highlight Transfer data and press select
- (Eventually you have to enter time/date)
- Options / Add transfer contact /Select
- Sync / Select
- Data to be copied / Select / Contacts / Mark / Done
- Save changes? / Yes
- Done
- Save new transfer contact? / Yes
- Highlight transfer in list (name sth. like 'Contact 1')
- Select
- Overwrite existing data... / Yes
- (Searching for devices)
- Highlight device / Select
- Enter passcode (e.g. 1234)
- Accept on 6230i
- Enter passcode (e.g. 1234) on 6230i
- Accept connection on 6230i
- Phones are syncing

I hope I didn't miss a step :-)
