a. user say:

Assuming that your mail merge main documents are in the standard .doc format, the problem is that there is no way to open the document manually or using Word automation that leaves a connection to the existing data source, unless that data source exists. What that means is that you cannot even discover, manually or via automation, the full details of the data source (name, connection string (if any), SQL query/query options (if any) etc. 

How much that matters depends on how the people in your organisation use Mail Merge. If, for example, you only use a couple of different sources, never use filtering, etc. etc., it may be reasonably easy to tell which document uses which source, then open each one programmatically and connect it to the new source. However, if there is a wide variety of sources the only way you can even discover what the source is would be to ensure that the old drive letter still exists and points to the same place as it used to. That may or may not be possible, depending on what modificaitons have been made.

Other than that, with .doc files, the only other approach would be to write code that understands the Word binary document format and can go and find the MailMerge information in each file and modify it. However, that is not a trivial exercise, and you doubtless have no time, so unless you can get someone to code that for you quickly it is probably not a viable option. Even then, it may be difficult to achieve if, for example, Word is set up to perform Quick Saves (which complicates the binary format). I was working on something along those lines, and was able to extract the mailmerge info, but had not got to the point of trying to modify the source file name, so cannot be certain that it is feasible. But it looks as if it is, particularly since in this case there would be no requirement to increase the length of the data source file name. Unfortunately I have no time to pursue this right now either.

sherwin1 say:

I solved this problem with my x7. 

 

It is because the glass and plastic panel covering the proximity sensor (window next to earpiece, where your logo is) puts too much pressure on the sensor itself and disables its function.   If you insert a thin piece of material (thickness of your fingernail, ex. cardboard) in between the top, back of this panel and the very top to the phone, this will take pressure off of the sensor in behind the glass and will allow it to function properly. 

 

Simply try it with gentle prying with your finger or fingernail during a call and you will immediately see the screen reappear (or disappear if you apply pressure on the sensor area).

    Hi Dinesh,

    Search-mailbox doesn't work on an Inactive mailbox. But we can search the recurring meeting and delete it using search-mailbox from all active mailboxes.
