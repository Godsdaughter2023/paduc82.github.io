teylyn say:

Hello,

you need to write a worksheet change event. To do that, right-click the Sheet tab and select "View Code". In the code window, you can enter this code:

Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Target, Range("A:A")) Is Nothing Then
        Range("D" & Target.Row) = Environ("UserName")
    End If
End Sub

The code monitors all of column A. When a cell in column A is changed manually, the code will put the user name into the corresponding cell in column D.

  • Win10 upgraded itself to a new build.
  • The build update (or some other feature check/change) appears to have created a new, small (500 MB) primary partition on the drive for "recovery".

OneNote for Mac is comparable to the OneNote Universal App for Windows 10 and with it being free now, don't expect it to mature at a decent clip.  It will just sit there as some random reason to "choose Office" even though you can get it on its own from the Mac App Store, and it will probably forever lag behind the Windows Client.  OneNote is actually worse off than Outlook when it comes to feature disparity vs. the Windows version but they are both equally bad.  Microsoft didn't really put much effort (or thought) into these ports.  You have to make some SERIOUS compromises when using OneNote across Windows and Mac.  The feature disparity forces you to plan your use cases around the "lowest common denominator capabilities" of the Mac Client, otherwise you're constantly confuse yourself when moving across machines.

OneNote for Windows blows Evernote for Windows away.  Evernote for Mac blows OneNote for Mac out of the water.
