Jon, this is caused by a DEFECT in the windows 10 upgrade process which causes 8.3 references to Program Files (x86)  to be changed in UninstallString Reg_SZ used by InstallShield uninstallers.

Rundll expects no spaces to be in the path as there is no quotation marks, but the windows 10 upgrader changes PROGRA~2 in the uninstallstring to Program Files (x86)

For instance,

WaveStudio uses

RunDll32 C:\PROGRA~2\COMMON~1\INSTAL~1\PROFES~1\RunTime\09\01\Intel32\Ctor.dll,LaunchSetup "C:\Program Files (x86)\InstallShield Installation Information\{2670895A-4E6C-4450-B868-7B7DB80A3357}\setup.exe" -l0x9  /remove

And during the windows 10 upgrade process, this gets changed to

RunDll32 C:\PROGRAM FILES (X86)\COMMON~1\INSTAL~1\PROFES~1\RunTime\09\01\Intel32\Ctor.dll,LaunchSetup "C:\Program Files (x86)\InstallShield Installation Information\{2670895A-4E6C-4450-B868-7B7DB80A3357}\setup.exe" -l0x9  /remove

Which breaks the uninstaller.

Reinstalling will correct the path back to its 8.3 format and it will work again.
