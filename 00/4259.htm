brian tillman [outlook mvp 2007-2019] say:

587 would never be the port for an IMAP server. That's an SMTP port.  Some IMAP servers use 995, but not TalkTalk.

TalkTalk settings are here: http://help.talktalk.co.uk/app/answers/detail/a_id/1668/~/what-are-talktalk's-email-settings.  Specific settings for Outlook 2007 (and any other version of Outlook) are here: http://help.talktalk.co.uk/app/answers/detail/a_id/1465/~/what-are-talktalk's-email-settings.  It will differ only if the method of finding the specific fields.

This means that for POP you use:

Incoming: mail.talktalk.net, port 110.  No encryption
Outgoing: smtp.talktalk.net, port 25.  No encryption.  No outgoing server authentication.

Username: yourtalktalkname@talktalk.net
Password: yourtalktalkpassword

IMAP incoming would use mail.talktalk.net and port 143, no encryption.

The fact that there's no encryption would make me hesitate to ever use this service on a public network.  Your mail credentials will be sent out over the wire in the clear for anyone to grab.

tom ogilvy say:

I assume that if my lookup value is cell A1, vlookup A2, somehow I can add A3, A4, A5 (columns) respectively and have some sort of vlookup IF scenario? 

Say you have data in M1:Q100 with a header in row1.   Assume the values to be looked up are on the left side of the data in column M 

A1:  value to lookup

A2:  =Vlookup($A$1,$M$1:$Q$100,2,False)     The 2 as the third argument says to return the value from the 2nd column of the lookup range (column N) for the row where the lookup Value is found

You can repeat this formula in A3, A4, A5 with the two changed to a 3, then 4, then 5 

There are various other lookup functions or a combination of Index and match if the lookup- column is not on the left side. 

This portion of your question seems inconsistent with the first part.  If you have multiple instances of the lookup value in your data table, then none of the built in lookup functions support pulling anything but the data associated with the first match.  You would have to combine several functions in an array formula to work that type of information.
