hairyfool say:

So it looks like when you are using the wired connection DNS is failing.

 

To clarify web site names means absolutely nothing to a computer, by itself www.google.com is completely useless. All comms on a network is based around IP addresses and the DNS service is a process whereby dedicated servers who's addresses are known can tell your PC what the address of a website is. For some reason on the wired connection your PC is not getting the address for soundcloud.com.

 

In most domestic situations a PC gets it IP settings from the router, obviously I don't know if that is true in your case. If you open a command prompt and run ipconfig/all, paste the result here and we can check. This would include what DNS server to use which as I have said is normally the router. There are actually millions of addresses on the internet and your router could not store them so instead it passes a request for an address to DNS servers listed in its own internet side IP configuration.

 

If you can log in to the router and look at its WAN or ADSL configuration you should find at least one, possibly two, DNS servers. Make a note of them. If you can't log in to the router then look at your ISPs website for their settings.

 

On your computer go to Control Panel\Network and Internet\Network Connections, right click on your wired card and select Properties. Double click on Internet Protocol version 4. In normal use the 2 Obtain.. buttons should be selected, are they?

 

If they are change only the bottom one to Use the following DNS Server addresses and enter in the boxes the addresses you obtained from the router. click on OK twice but leave the network connections window open as you may need it.

 

Now try and get to your problem website. If you can then the problem is with your router. Instead of asking the router for the address you have bypassed that stage and gone to the server your router is configured to use.

 

If you still can't get to the site then the problem is further down and your router may be OK. So go back to where you set the DNS addresses and replace the ones you got from the router and key in 8.8.8.8 in the Primary DNS Server address and 8.8.4.4 in the alternate address, OK again twice and then try the site again. These 2 addresses are Google's own public DNS servers and should work.

 

If the ISP provided the router they should replace it. Explain what the problem is and these test results although I have no doubt they will insist on their own checks before agreeing.
