Sub AddColumns_ClosedWB()

Application.ScreenUpdating = False

.Cells(1, x + 1).EntireColumn.Insert
.Cells(1, x + 1).Value = v(x)

If .Cells(1, x + 1) = "" Then
.Cells(1, x + 1).Value = v(x)
End If
nnext:
Next
.Range("A1").CurrentRegion.SpecialCells(xlCellTypeBlanks).Value = 0
End With
wb.Save
wb.Close False

ActiveWorkbook.Save

Application.ScreenUpdating = True
MsgBox "done"
End Sub

The company allows John and the rest of the employees to stop work at 3pm every Wednesday, for sports activities. But he as all employees of the company are still paid the full 8 hours for wednesday as part of wages.

Okay now if I schedule John for Task A with a duration=5 days task starting on a Monday. MS Project will end the task on a Friday. If you look at the cost for this Task A from MS project it will give  as 4*8*100 + 1*6*100 = $3800 ( because wednesday is taken as 6 hour day) 

But my actual cost for Task A is 5*8*100=$4000 (as John is still paid for the 2 hours on wednesday even though he is not working on Task A)

dale howard [mvp] say:

KishRanai --

You cannot have it both ways.  Either the 5-day task shows 40 hours of work and cost, or it shows 38 hours of work and cost.  If it were me, I would not bother with changing the calendar to show 6 hours of work on Wednesday.  By doing this, the cost would show correctly.

If you insist on setting Wednesday as a 6-hour work day, then you would need to adjust the Standard Rate for each resource to calculate the cost of 40 hours of work when the resource only works 38 hours.  In your scenario, you would need to set the Standard Rate for the resource to $105.26, which would calculate $4,000 for the task that required only 38 hours of work.

=INDEX('1000 - Check Register'!$E:$E,SMALL(IF('1000 - Check Register'!$B$6:$B$50="O",ROW('1000 - Check Register'!$6:$50),9999),ROW('1000 - Check Register'!17:17)))&""

And there you have it:  by concatenating the null string (&""), you are forcing the result to be text, not numeric.

You should have been told to confirm that by entering a formula of the form =ISTEXT(A1) in the first place, where A1 is a reference to a cell that does not format as intended.

Numeric formats work only for numeric values.  Any cell can have a text value, even if it has a numeric format.  Even the General format is not doing anything for you.  It might seem to "work fine" only by coincidence.

Be sure the cell has a numeric format, be it General or otherwise.  Remove &"" from the formula.  Then confirm that =ISTEXT(A1) returns FALSE.  Do things in that order to ensure proper results.
