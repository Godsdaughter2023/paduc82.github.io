Please try the following to take ownership of the iTunes folder (which will then allow you to make changes):
- Browse to the following: C:\Users\<Your username here>\Music
- Right-click the iTunes folder and select Properties
- Select the Security tab
- Select Advanced
- Select Change next to where it says Owner at the top of this window
- Enter your username in the box and press OK
- Tick the box next to 'Replace owner on subcontainers and objects'
- Under the permission entries, check if you user account has 'Full control'.
- If it doesn't, select Add > Select a principle > Enter your username in the box and press OK > Tick the box next to 'Full control' and press OK
- Tick the box at the bottom next to 'Replace all child object permission entries with inheritable permission entries from this object'
- Select OK
- Note: can sometimes take awhile to take ownership of all sub-folders and sub-files.

Once this has completed, try opening iTunes. If you receive the same error, try doing the rename as mentioned in my first response.

That is not going to happen unless you share the C:\ OS drive. Not recommended

If it an external HDD on another PC you want to 'share\access, you also have to share the drive first.

You never need to map any drive or use remote desktops. I would never use map network drive to begin with. I share 7 drives between 6-8 PC's plus everyone default public folders for sharing are already accessible.

I version 1803, settings are changed.

do on all pcs and restart them all.

dfontana1959 say:

I seem to have fixed the issue, and as an FYI all machines were using local accounts. Found this in another thread in another site. Flushing the network stack seems to have fixed it. The commands issued in a command prompt were as follows:

ipconfig /flushdns

nbtstat -R

nbtstat -RR

netsh int reset all

netsh int ipv4 reset

netsh int ipv6 reset

netsh winsock reset

And then Reboot

I tried those commands on the PC having the issue connecting to the C drive of one of my other PCs and it connected after those commands were entered and a reboot.
