I develop VBA add-ins using Windows Excel and then test on Mac Excel.

I avoid Active-X controls and always use Windows-Mac-compatible Forms controls.

Disclaimer: I have never had to remove Active-X controls, and I cannot speak from experience. 

If I ever did have to remove an Active-X control, I would use the Visual Basic Editor in Windows Excel.

This happens because the custom format 000000 happens to coincide with the built-in format for postal code in China. If you specify a custom format, Excel checks whether it matches one of the built-in formats and if so, displays it as such in the Number tab of Format Cells.

It may look strange, but your data will still be formatted as 000000, so you don't have to worry about it.
