thewinchester say:

You could be forgiven for thinking that paulheu, but I've dug into this a little further.

 

First and foremost, the DNG specification is explicitly designed to permit the writing of metadata, such as EXIF, as part of a DNG file.

 

From Page 14, Digital Negative (DNG) Specification, Version 1.4.0.0, published June 2012:

 

Metadata
Additional metadata may be embedded in DNG in the following ways:
• Using TIFF-EP or EXIF metadata tags
• Using the IPTC metadata tag (33723)
• Using the XMP metadata tag (700)
Note that TIFF-EP and EXIF use nearly the same metadata tag set, but TIFF-EP stores the tags in IFD 0, while EXIF store the tags in a separate IFD. Either location is allowed by DNG, but the EXIF location is preferred.

 

Within EXIF, Tag hex 0x8825 (Key Exif.Image.GPSTag) acts as a pointer to the GPS data in the EXIF structure. Tag hexes 0x000* and 0x001* are used to store the actual data.

 

My thinking currently, is that someone in the team programming Nokia Camera failed to ensure the GPS data is being written to DNG files, just as it is for JPEGs.

That is easy if
a) the sheets are in the order S1,...,S50 and a Summary sheet (behind or in front of all "S"-sheets)
b) the sheets have the same layout

For my example I assume that the top left cell of all tables in all sheets is A1:

- Go into your Summary sheet
- Select cell A2
- Type =SUM(
- Select sheet S1
- Press the SHIFT key and hold it down
- Select sheet S50
- Select cell A2
- Release the SHIFT key
- Press ENTER

The result is this formula: =SUM('S1:S50'!A2)
Drag the formula down till you have data.
