Sub RunMailMerge() 'Will merge data on INPUTS sheet into customized document with fields for all data that needs to be updated
Dim wdOutputName As String, wdInputName As String, strWorkbookName As String

strWorkbookName = ThisWorkbook.FullName 'use the current spreadsheet as the datasource

Set wdApp = CreateObject("Word.Application") 'set object as Word document used as base doc
With wdApp
  .DisplayAlerts = False
  Set wdDoc = .Documents.Open(wdInputName, False, True, False, , , , , , , , True)
  With wdDoc
    With .MailMerge 'run through mailmerge commands for input word document
      .OpenDataSource Name:=strWorkbookName, AddToRecentFiles:=False, Revert:=False, Format:=wdOpenFormatAuto = 0, _
        Connection:="Data Source=" & strWorkbookName & ";Mode=Read", _

      .Destination = wdSendToNewDocument 'merges results to new document

Tricking macOS and OneDrive app into syncing into an external drive or - Microsoft Community

josé ruy say:

Tricking macOS and OneDrive app into syncing into an external drive or sd card

I figured this out joining information about symlinks and Microsoft Support. 

While you can't officially have OneDrive app syncing to an external drive or sd card you can trick the system to do this for you using symlinks. If you don't know what they are just google "Symlinks macOS". 

Basically while you can't symlink the main OneDrive folder you can symlink every folder inside. While this could be cumbersome for some I got it to work easily.

Just quit OneDrive App, copy the folders in question to your External Drive or Sd Card, open terminal and type: ln -s /Path/To/Folder /Path/To/Symlink 

Instead of the actual path to the folder you can just drag the folder you just put on your external drive or sd card and drop it in terminal and it will make the path for you. As for the path of the symlink it should be something along the lines of /Users/"username"/OneDrive/"folder name"

username and folder name are your username and the folder you want to symlink.

Hope this helps.
