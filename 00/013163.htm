trevor rabey say:

In Bar Styles, each style goes From and To as shown in the columns with those names.

If you want to create a new style for a particular type of bar, you need a From date and a To date.

You could think that you could use two custom date fields.

However, as Dale points out, the custom date fields are not available in the From and To dropdown lists.

But the custom finish fields are available.

The From date will always be the start date of the task, and the To date is determined by your % of the task and the duration of the task. In your example, the To date which you will calculate in a custom finish field is:

start + (your task % x duration).

..and you could overlay this as a thin bar on top of the task duration.

..perhaps you could make your % a thin bar upper and the regular progress bar a thin bar lower.

.. or you could stack them. Bars can be stacked 4 high.

So you could have a 10 day task with 2 days of actual duration and % Complete = 20%, but you have done 15 out of 30 reports so your bar runs half way along the duration bar.

This is a fascinating exercise and a good way to study how the bar styles work, but from a practical point of view it is a lot of work for no useful purpose, and I think it is actually wrong.

I have done it myself, not only with the "reports" but also work and cost, so I have had a stack of 4 bars, each one representing a different flavour of "progress"... and then I filed it and forgot about it.

Having described how it could be done, my recommendation is similar to Dale's, don't do it.

Reason is, it will be inappropriate and misunderstood, and it will serve no purpose that is not otherwise better served by another way. The Gantt bars are on a timescale, and every horizontal bar on it should only represent a duration, whereas you are trying to put a bar on it that represents reports as a fraction of duration. Similarly, as I would not overlay a bar representing reports, I would not overlay a bar representing work or cost.

The usual progress bar represents actual duration so that is valid on the timescale.

dale howard [mvp] say:

Pamela --

OK, thanks for the additional information.  The simple answer is that you CANNOT display the custom field value as a progress bar in the Gantt bar.  Let me show you why.  Try the following:

  1. Apply the Gantt Chart view.
  2. Double-click anywhere in the white part of the Gantt Chart pane.  Microsoft Project 2010 displays the Bar Styles dialog.
  3. Scroll down to near the bottom of the list of bar styles and select the Progress row.
  4. In the Progress row, select the cell in the From column (it currently displays the Actual Start value) and then click the pick list.

If you look at the list of acceptable values for the From field, you will see that there is no way to select any custom field value.  Instead, you are restricted to using only a selected set of default fields.  So, the bad news is that you cannot display a progress bar in the Gantt bars for your custom field.

On the other hand, you CAN display the value to the left, right, top, bottom, or inside the Gantt bar.  To do this, complete the following steps:

  1. Scroll back to the top of the Bar Styles dialog and select the row labeled Task.
  2. In the lower left corner of the dialog, click the Text tab.
  3. Select the white cell for the Left row and then click the pick list.
  4. Select your custom Text field and then click the OK button.

The above set of steps tells you how to display the value to the left of the Gantt bar, but you can place the value above it, below it, to the right of it, or even inside the Gantt bar.  Hope this helps.

dianeoforegon say: The Identity has to be in the Microsoft User Data folder.
Move from Desktop to the Office 2008 Identities folder.

~/Documents/Microsoft User Data/Office 2008 Identities/your identity

Hold down Option key when opening Entourage. This will launch the Database Utility. Select the Identity you want to rebuild.

This can take hours depending on the size of your database, so don't despair. While the progress window is often deceiving and you wonder if it's quit or just slow you can also monitor using the Activity Monitor.

Open the Activity Monitor in Applications/Utilities. Enter "Microsoft" in the filter. Sort on the CPU column. You can watch the Microsoft Database daemon. In the bottom of the Activity Monitor window, select the Disk Activity tab. You can see the data as it reads/writes.

When the CPU is 0.0 and the Disk Activity has flat-lined, the process has quit.

After a successful rebuild, if you can open Entourage, you need to move to a new Identity. I never trust a rebuilt Identity to be clear of problems. I've seen it say successfully rebuilt and the Identity wouldn't even open. <sigh>

First step is to make sure no folder has more than 2GB of data. That's approx 10k messages. It's usually the Inbox and Sent folders that are huge. If you have an archive folder check it too. I like to subdivide by year. e.g. Inbox 2011, Inbox 2010, Sent 2011, Sent 2010 etc.

Once you have checked folder, download and run this script.

You can download and run this script on your Entourage Identity. (Yes, you can still have corrupted messages after rebuilding.)

Export folders as MBOX files (includes subfolders)

http://www.office.mvps.org/downloads/scripts/jolly/Make_MBOX.zip

The script will fail on a folder with corruption. If there is no corruption the script will export all folders. Once you find corrupted folder, rename it with z in front of the name so it sorts at the bottom then run the script again. Keep repeating until you have identified all corrupt folders. Once you have identified corruption, you can subdivide folders and keep running script until you get down to 25 messages. Then you drag individual messages to the Desktop. A corrupt message will crash Entourage. After you clean all folders and reduce the size of the Inbox and Sent folders, do the File > Export as .rge file and you should get "Done". You can use this file to import into Outlook.

Tip: Folders saved as .mbox by year will never change. You can save those folders in case you ever need to import again after a corrupted Identity that will not rebuild.

Create a new Identity and import the .rge file. You can drag over Rules and Signature file to the new Identity but you'll have to create your accounts and set some preferences in the new Identity. All rules will have to have folder destinations reset.

This article was written for Outlook but the general concepts are the same for Entourage.

How to move your data to a new Identity

http://www.officeformachelp.com/2012/05/how-to-move-your-data-to-a-new-identity/

Hope this helps!

gord dibben say:

I just use worksheet event code to take care of that.

Enter codes as v9g2d4 and will change to V9G 2D4

Private Sub Worksheet_Change(ByVal Target As Excel.Range)
'Canadian postal codes
    If Target.Column <> 1 Then Exit Sub  'change column number to suit
    On Error GoTo ErrHandler
    Application.EnableEvents = False
    With Target
        .Formula = UCase(Target.Formula)
        .Value = (Left(Target.Value, 3) & " " & Right(Target.Value, 3))
    End With
ErrHandler:
    Application.EnableEvents = True
End Sub
