Sub Set_German()

' Sets the Current Language to German (Germany)

jllatham say:

Very, very hard to do with just a formula.  Best to use a special macro known as a User Defined Function (UDF).  I have written one that is shown below that will do the job for you.  You need to put it into a Regular Code Module as shown here:

http://www.contextures.com/xlvba01.html#videoreg

Then save your workbook as a macro enabled workbook, type .xlsm or .xlsb

for the results you put this formula into a cell

=RemoveDuplicateNames(A1)

Where A1 is the cell with the names that may have duplicates.

As it is written, it is 'case' sensitive, so MIKE will not be the same as mike or Mike.  If you need it to work so that it sees MIKE and mike as the same name, we can make it work that way.

Here is the code for the UDF:

Function RemoveDuplicateNames(sourceCell As Range) As String
'use in worksheet cell like this:
' =removeduplicatenames(A1)
' where A1 is the cell with the list of names
' with possible duplicates in it
'
' this is case sensitive, so "Sami" <> "SAMI"
'
  Const nameSeparator = ","
  Dim namesList As Variant ' for use with SPLIT()
  Dim LC As Integer ' loop counter
  Dim uniqueNames As Object
  Dim uniqueKey As Variant
  Dim tempText As String
 
  namesList = Split(sourceCell, nameSeparator)
  Set uniqueNames = CreateObject("Scripting.Dictionary")
  For LC = LBound(namesList) To UBound(namesList)
    If Not uniqueNames.exists(Trim(namesList(LC))) Then
      uniqueNames.Add Trim(namesList(LC)), LC
    End If
  Next
  For Each uniqueKey In uniqueNames
    tempText = tempText & uniqueKey & ", "
  Next
  RemoveDuplicateNames = Left(tempText, Len(tempText) - 2)
  Set uniqueNames = Nothing
End Function

Public Function wUniqueStr(sinput As String, delimiter As String, Optional Compare As Integer = 0) As String
    Dim objDict As Object
    Dim arrInput As Variant
    Dim uniqStr As String
    arrInput = Split(sinput, delimiter)
    Set objDict = CreateObject("Scripting.Dictionary")
    If Compare = 0 Then 'case insensitive
        For i = 0 To UBound(arrInput)
    
                If objDict.exists(UCase(Trim(arrInput(i)))) Then
            Else
                objDict.Add UCase(Trim(arrInput(i))), i
                uniqStr = uniqStr & arrInput(i) & delimiter
            End If
        Next i
        wUniqueStr = Left(uniqStr, Len(uniqStr) - Len(delimiter))
    Else  'case sensitive
        For i = 0 To UBound(arrInput)
                If objDict.exists(Trim(arrInput(i))) Then
            Else
                objDict.Add Trim(arrInput(i)), i
                uniqStr = uniqStr & arrInput(i) & delimiter
            End If
        Next i
        wUniqueStr = Left(uniqStr, Len(uniqStr) - Len(delimiter))

Another 'For what it is worth" - Excel 64 bit is not just Excel 32 Bit translated and with extended addressing for larger files tossed in.  There are internal processing differences.  I put together one process some time back that worked flawlessly in Excel 2003 and in Excel 2010 32-bit, but would not work with Excel 64-bit giving me a "formula is too complex" error in some VBA code.  i had to break two different lines of code into two separate lines and then combine the results of the two back together in a third line of code to get it to work in Excel 2010, 64-bit.
