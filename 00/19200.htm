Comments table - Microsoft Community

xodiak say:

Comments table

I remember seeing this before and now I can't find it on the site.  I need to create a comments table.  I have a comments field in a form and when I update it I want it to write the comments to another table, but each time I type in the comment field I want it to time stamp infant of my typing.  And how do I link the comments table to the main table, with an ID number I presume or something, I have a ticket number that is entered on the form that is unique.

john w. vinson say:

The simplest way to do this is to have a Comments table in a one to many relationship with the main table. In your case you'd use the ticket number as the primary key (or a uniquely indexed field, in any case) of the main table, and as a non-unique foreign key in the Comments table. One design that should work for the Comments table would be:

CommentID (autonumber, primary key)

TicketNumber (matching the size and datatype of your main table ticket number)

CommentDate (Date/Time, default value =Now() to automatically timestamp new records)

Comment (Memo, or Text if comments never exceed 255 bytes)

(possibly other fields such as the ID of the person entering the comment)

The user interface would be a Form based on your main table, with a Subform based on the comments table; use the TicketNumber as the master/child link field of the subform. This will let you enter new comments and see old ones, without having to use any code or to copy comments from one control to another.

antónio faria say:

Ok. For the next guys that catch the bug.

Take the following steps:

Through Exchange Online web interface

- Create the disclaimer rule

- Set first condition, usually the disclaimer is set for outgoing emails;

Through PowerShell

- Open a PowerShell window as administrator

- Connect to Exchange Online PowerShell using the commands described in the below link: 

https://docs.microsoft.com/en-us/powershell/exchange/exchange-online/connect-to-exchange-online-powershell/connect-to-exchange-online-powershell?view=exchange-ps#what-do-you-need-to-know-before-you-begin

- Check the disclaimer details using the following command:

   get-transportrule "Your_Disclaimer" | format-list

- Finally add the second condition using the following command:

  Set-TransportRule "Your_Disclaimer" -SenderADAttributeContainsWords "Country:DE"

This rule will check if the emails are going outside of the organization and the country' sender (in this case Germany "DE") and adapt the disclaimer according the country set in the user contact info.

julie fka profjulie say: Applications you install to a device are installed to storage memory.  Program memory is used to execute applications.  You can perform a soft reset of the device to flush out any memory that is taken up by opened applications and then check to see what program memory shows.
Prof Julie, Microsoft MVP Mobile Devices | http://pocketprimer.com

julie fka profjulie say: Installing applications to a storage card will not have much impact on program memory.  You are thinking about how memory used to be managed before WM5.0, when memory was allocated between program and storage.  Since WM5.0, memory is fixed between program and storage and there is no advantage to installing applications to a storage card unless, of course, you have filled up the device's storage memory.
Prof Julie, Microsoft MVP Mobile Devices | http://pocketprimer.com
