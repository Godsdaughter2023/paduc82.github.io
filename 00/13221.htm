scottgem say:
You could do this with Option Groups. An Option Group presents multiple choices with radio buttons or checkboxes, of which only one can be selected. You could set the captions and value on the fly if need be.

ken sheridan say: An option group is really only suitable for a relatively small number of possible selections which are guaranteed to be static.  Each option has an integer number as its value, so the option group control can be bound to a numeric foreign key column, StateID in your example, where the set of values of the primary key in the referenced table mirror the values of the set of options in the option group.

Where the number is large or where the available selections might change a combo box bound to the relevant foreign key column in the table would be a better option.  In the event of a new row being inserted into the relevant referenced table, the combo box's list will automatically include the new item.

Where you need to select multiple values to be associated with the current record, e.g. 'which states have you lived in?' a subform is the usual interface.  It is possible to use a multi-select list box, however, and you'll find an example in StudentCourses.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

This little demo file illustrates a number of possible interfaces for a basic many-to-many relationship type, but I'd recommend the conventional form/subform.  A subform requires the parent form to be in single form view, but it is possible to use two correlated subforms so that both are in  continuous forms view.  You'll find an example in CorrelatedSubs.zip in my same OneDrive folder.

You'll also find an example of a list box being used to requery a subform in a questionnaire or survey type of database in Questionnaire.zip in the same folder.  In this the questions per questionnaire are presented in a list box, in which the selection of a question requeries a subform in which a single or multiple answers can be selected on the basis of how the questionnaire has been set up initially.
