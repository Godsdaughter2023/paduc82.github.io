bobgreenblatt say:

First of all, a formula can not change the value of another cell. It can only return a value to the cell that it is in. So, in your illustration, the formula must be in cell E9, not D10; since E9 is the cell whose value you want to change.

Forgetting about buttons for a minute, if you had another cell (for argument's sake say H22) whose value changed the way you want, (say 0, 1 2) then the formula in E9 would be:

=if(h22=1,D8,if(H22=0,"",D10))

Now, all you have to do is to figure out how to set H22 to 1, -1, or 0. A button can trigger a macro, but does not set a value. A check box or option button can set a value to true or false. You might try a spin button with a minimum value of 0, maximum of 2 and increment of 1. Assign the linked cell to H22 whose values will then be 0, 1 or 2.

Post back if you have any further questions.

yang zheng msft support say: Hi Agustin,

Generally, the retention tag about ‘Deleted items’ in your screenshot is not added into the default retention policy. However, the retention tag for ‘Junk mail’ is included in the default retention policy. So the ‘Junk mail’ retention tag will take effect and the ‘Deleted items’ tag will not.

As you want the ‘Deleted items’ retention tag to take effect, you can add the ‘Deleted items’ tag into the default retention policy. Or you can create a new retention policy with this ‘Deleted items’ tag and other required tag and then assign this new retention policy to this user.
