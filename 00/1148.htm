I need to create code to convert all alphanumeric words in a Word document to caps. Example: 'tic123' would become 'TIC123'. It may or may not contain a hyphen and the letters/numbers can be any combination. Thanks in advance for your help.
greg maxey say:

Not the most elegant piece of code but it seems to work:

Sub ScratchMacro()
'A basic Word macro coded by Greg Maxey
Dim oRng As Word.Range
Dim oRngEval As Word.Range
Dim oRngCap As Word.Range
Set oRng = ActiveDocument.Range
  With oRng.Find
    'Find any number or series of numbers.
    .Text = "[0-9]{1,}"
    .MatchWildcards = True
    While .Execute
      'Get the word it is in.
      Set oRngEval = oRng.Duplicate.Words(1)
      If oRngEval.Characters.First.Previous = "-" Then
        oRngEval.MoveStart wdWord, -2
      End If
      If oRngEval.Characters.Last.Next = "-" Then
        oRngEval.MoveEnd wdWord, 2
      End If
      oRng.Collapse wdCollapseEnd
      With oRngEval.Find
        'In that word, see if there are any letters.
        .Text = "[A-Za-z]@"
        .MatchWildcards = True
        If .Execute Then
          oRngEval.Words(1).Font.AllCaps = True
        End If
      End With
    Wend
  End With
     

End Sub

Hi,

 

This FindNext macro will look through row 1 and build a range of every column with telephonenumber in row and delete it

 

Sub Find_Next()
      Dim rFound As Range
      Dim rSearch As Range
      Dim sFirstAddress As String

      Dim Delrange as range
      'Where to look
      Set rSearch = ActiveSheet.Rows(1)
      'do a search
      Set rFound = rSearch.Find(what:="TelephoneNumber", _
          after:=rSearch.Cells(rSearch.Cells.Count), _
          lookat:=xlWhole)
    
    'if we find it this executes
      If Not rFound Is Nothing Then
     'where did we find it
      sFirstAddress = rFound.Address
      Set DelRange = rFound.EntireColumn
      'start a loop to look for the next instance
      Do
 
         'This is where we do things at rfound
        Set DelRange = Union(DelRange, rFound.EntireColumn)
         'Find the next one
          Set rFound = rSearch.FindNext(rFound)
     ' get out when we find the first address again
      Loop Until rFound.Address = sFirstAddress
      End If
      If Not DelRange Is Nothing Then
        DelRange.Delete
    End If
     
  End Sub

Hi,

 

Your correct, what you've written isn't optimal. If you delete columns or rows 'on the fly' then you run the risk of missing rows or columns. Consider this:-

 

let's say we ran a macro to delete columns and we star by looking in column 1 (column A) and it meets our criteria and we delete it. Columns now shift left and what was column 2 is now column 1. If what was column 2 also meets out criteria then it gets missed because we've alreay done column 1!!. I'd do this which deletes the column that meets our criteria and the column to the right but because we delete in 1 go at the end no columns are missed.

 

EDIT.... I should have added that my code is OK if there aren't too many columns to delete but if there are (say) hundreds then it can be slow because the UNION method is always the best way to do it. There are several other methods.

 

Sub Find_Next()
       Dim rFound As Range
       Dim rSearch As Range
       Dim sFirstAddress As String
 
      Dim Delrange As Range
       'Where to look
       Set rSearch = ActiveSheet.Rows(1)
       'do a search
       Set rFound = rSearch.Find(what:="TelephoneNumber", _
           after:=rSearch.Cells(rSearch.Cells.Count), _
           lookat:=xlWhole)
     
    'if we find it this executes
       If Not rFound Is Nothing Then
      'where did we find it
       sFirstAddress = rFound.Address
       Set Delrange = rFound.Resize(, 1).EntireColumn
       'start a loop to look for the next instance
       Do
 
          'This is where we do things at rfound
         Set Delrange = Union(Delrange, rFound.Resize(, 2).EntireColumn)
          'Find the next one
           Set rFound = rSearch.FindNext(rFound)
      ' get out when we find the first address again
       Loop Until rFound.Address = sFirstAddress
       End If
       If Not Delrange Is Nothing Then
         Delrange.delete
     End If
      
  End Sub

toddallcock say:

You don't really need an app, GSM Windows Mobile phones supported dialup right out of the box, but there are several potential issues that will probably prevent you from doing this.

 

First, no cellphone really ever had a dial-up modem in it, at least on the traditional sense. The way cell phones did dialup back in the "1G" days, was that your mobile service provider actually had a bank of dialup modems at their headquarters, and when you nade a dialup connection from the phone, thr carrier relayed data between your phone and one of their modems. This data connection was known as CSD, or circuit-switched data. Virtually no one ever uses it anymore, since 2G was faster, more reliable and cheaper for the carrier to furnish.  Consequently, many carriers stopped supporting it years ago. T-Mobile USA's CSD still worked last time checked (about two years ago). I don't know about AT&T.

 

Secondly, even if your carrier still has a rack of dusty modems lying around and connected, your account has to be provisioned to access them. CSD was a "feature" like caller ID or voicemail, that had to be activated on your account. Years ago, carriers even charged for it (on top of the minutes you used). I used to pay Cingular (now AT&T) $3.99/month to have CSD access, then the actual connections would use up my minutes. "Free" CSD was actually one of the things that convinced me to switch to T-Mobile a decade ago.

 

Last, is speed. Sure, you can SAY you don't care about speed, but you will. GSM cellular phones' can only support 9.6kbps dial up. Yep, that's "9" and "k". One-sixth of a 56k landline dial up connection, and one three-hundredth of a fairly slow 3mbps DSL connection.  That gives a new meaning to "slow" that your generation has never experienced! ;)

 

But, if I haven't talked you out of trying, hop over to http://www.freedialup.org/site/ and get a free dialup account, then go into the settings on the Jack and create a new connection. In the settings you'll see two types of cellular connections- "Cellular Line" and "Cellular Line (2G/3G)". You want the plain ol' "Cellular Line"- that's dialup. Enter the phone number from Free Dialup, and, if prompted for username and password, use your cellphone number and the user, and leave password blank.

After you create the connection, try to select it and see if it establishes a connection. If it does (very unlikely) great, if not, if you can call your provider and ask for the rep who has worked there the longest and ask if they still have CSD and if they can add it to your account. (I'm only half-kidding about the rep. When I last tried CSD on T-Mo two years ago it didn't work at first. Apparently it was turned off when I changed rate plans, and I went through five or six reps before I found someone who even knew what CSD was, and they had to get two different supervisors to find someone who could figure out how to re-add it to my account!)

tisky zheng msft say:

Hi Debbie,

 

Based on your description, you want to move an entire row based on a cell value to another sheet.

 

Here is a VBA code for your reference.

 

Sub Cheezy()

    Dim xRg As Range

    Dim xCell As Range

    Dim I As Long

    Dim J As Long

    I = Worksheets("Sheet1").UsedRange.Rows.Count

    J = Worksheets("Sheet2").UsedRange.Rows.Count

    If J = 1 Then

       If Application.WorksheetFunction.CountA(Worksheets("Sheet2").UsedRange) = 0 Then J = 0

    End If

    Set xRg = Worksheets("Sheet1").Range("C1:C" & I)

    On Error Resume Next

    Application.ScreenUpdating = False

    For Each xCell In xRg

        If CStr(xCell.Value) = "Done" Then

            xCell.EntireRow.Copy Destination:=Worksheets("Sheet2").Range("A" & J + 1)

            xCell.EntireRow.Delete

            J = J + 1

        End If

    Next

    Application.ScreenUpdating = True

End Sub
