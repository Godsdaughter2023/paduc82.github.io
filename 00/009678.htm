jay freedman say:
I'll suggest creating two separate styles for the text under the heading. Their formatting should be exactly the same except that the one you use for most of the text (call it "MainPara" style) has Keep With Next turned on, while the other (call it something like "LastPara" style) has that option turned off.

Then modify the "Style for following paragraph" of each style:

  • In the heading style that you use for the date, the following style should be MainPara. Turn on Keep With Next in this style.
  • In the MainPara style, the following style should be MainPara.
  • In the LastPara style, the following style should be the heading style.


When you type a heading (date) and press Enter, the next paragraph will automatically be MainPara. As long as you're still within the same day, just keep pressing Enter normally and you'll keep getting MainPara style. When you start typing the last paragraph for that day, change its style to LastPara (use the Styles group on the ribbon, or assign a shortcut key for the LastPara style). When you press Enter at the end of that paragraph, you'll get a heading style for the next day.


One other point: when you modify the styles in the journal document, leave the "Only in this document" option selected at the bottom of the Modify Style dialog. This behavior should not be stored in your Normal.dotm template, which I assume is the base for the journal.

steve rindsberg say:

Not directly in PowerPoint, unfortunately.

I think Shyam's Handout Wizard add-in might do the job for you though.  http://skp.mvps.org

Another trick:  use notes pages and PDFs.

Go into your notes master and enlarge the slide image placeholder until it fills the page the way you'd like it to, and make sure there's a slide number placeholder on the page too.  You'll probably want to use Page Setup to make the orientation match that of your slides, by the way.

Then use PPT 2007's built-in Save as PDF feature to create a PDF of your notes pages.  That'll give you a PDF with one slide per page.

Now open and print the PDF.  Acrobat/Reader both allow you to print multiple pages per sheet of paper.  That'll let you print any reasonable number of slide per page of paper and each will have the slide number that you added in the notes master.

karl donaubauer say:

Weather Girl wrote:

I have a report that i created and one of the fields is a combo box
from the table it was based on. I need it to read column 1, not
column 0, as column 0 in a number field and column 1 has the text
that i want displayed,. how do i get it to display column 1 in my
report? as in a report there is no "column to bound" under the
properties.    

As there's no interaction with combo boxes in reports you should
not use combo boxes in reports at all. Instead use a query as
record source of the report and add the underlying table of your
combo box and the text field you want to display to the query.
Join the tables by a field that makes sense (foreign key etc.).

Then you can use a simple text box in the report to show the
value from the lookup table.
