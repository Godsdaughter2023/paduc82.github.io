scottgem say:

The Properities or the Build Event of the buttons go to the Expression Builder and just has, for example, in it:
=HandleButtonClick(2) or some other number. I see no [Event Procedure] anywhere that I am used to seeing to get to the VBA code. (I did create a menu application years ago in straight Visual Basic but not much with VBA).

The developer used the Switchboard Manager to create the Switchboard. Go to Tools>Database Options and open the Switchboard Manager. This is a wizard provided by Microsoft to create menus (switchboards). This form is database driven (the Switchboard items table). The HandleButtonClick() function determines, using the values in the table, what action to take when a switchboard button is pressed. I would not change that code unless to completely understand what its doing. Instead use the Switchboard Manager. Using the manager you can add "pages" to the switchboard and tasks to each page. To view what each task does, select it and press Edit. Each task is assigned a caption, action and argument. The action and argument can be chosen from lists. For example one action is to open a form in Edit mode. You then select the form you want to open.

Many developers look down on the Switchboard Manager and prefer to code their own. Personally I like it, mainly because it allows for using a single form to present and group actions and tasks for a user. With a little coding, you can even secure certain menus from unauthorized users.
