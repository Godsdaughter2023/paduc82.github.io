• All non-owners: Search for access by administrators and delegated users inside your organization. Also includes access user outside of your organization.
• External users: Search for access by users outside of your organization.
• Administrators and delegated users: Search for access by administrators and delegated users inside your organization.
• Administrators: Search for access by administrators in your organization.

For more detailed information, please refer to the article: Run a non-owner mailbox access report

Note: You need to be assigned permissions before you can perform this procedure. To see what permissions you need, see the "Mailbox audit logging" entry in the Messaging policy and compliance permissions topic.

533 email error is usually related to STMP authentication not being used . Most mail server need that you authenticate with legal user on that system before you are allowable to relay remote message out. However some mail administrators will also using the 553 error if a user doesn't exist or for various other reasons as well.

First of all, given the specs on your machine there is no excuse for it being slow. No excuse for you being able to type ahead of it, at all let alone that much.  On to your questions ...

#1 when the little blue wheel  is on it is showing you that the application is busy, but it is still working, it is not "locked up".

If "it" is taking 'too long' (what ever that means for you), there are a a couple of things you can try.  First use the <CTL><SysReq> key combo.  That is a very old command at the "System level" that tells the computer to "break" into / stop what ever it is doing.  It doesn't always work, but it is worth a try.

If that doesn't work you should go into task manager.  You don't have to "know" what all of the running processes are at this time.  We can help you worry about that later.

Start by going to the "Performance" tab.  Under the graph there is some text. What you want to pay attention to here is "Logical Processors". That tells you how many parts ("logical partitions") your computer hardware has been "split" into.  The idea is that even if one part of the computer is "maxing out" the rest is available to do other work so the computer is not completely "locked up".

Click on the "Processes" tab. Then click on the "CPU" column heading. This forces it to sort. You want it set to "largest to smallest". That may take 2 clicks.  I expect Word (or Excel) will be maxing out its "partition", or bouncing off the top of the limit.  As well as CPU maxing, are any of the other three: Memory / Disk / Network also maxing out.

Now would be the time to shut down Word and restart it in "Safe Mode". If Word is still maxed out, in Task manager you can right click on the "MS Word" entry and select "End task" and confirm. This will kill Word, losing any unsaved changes.  Now you can start Word again, this time in Safe Mode.

Running Word/Excel in safe mode disables all addin tools. Sometimes they are not "well behaved" and can cause all sorts of unexpected effects.  Use the application for a while. If the problem does not come back, that implies the problem is in one of the addons.  Restart Word normally and turn off all of the addins. Then slowly turn them back on until the problem comes back.

scott mcdaniel say:

You can move in a Subform from your Mainform like this:

If Not Me.NameOfYourSubformCONTROL.Form.Recordset.EOF Then

  Me.NameOfYourSubformCONTROL.Form.Recordset.MoveNext

End If

The IF line checks to see if you're at the End Of File (EOF), and does not execute if you are.

Note that NameOfYourSubformCONTROL is the name of the Subform Control on the parent form, and may or may not be the same name as the form you're using as a Subform. This can be tricky, so be sure you get it right. See the image below for an illustration:
