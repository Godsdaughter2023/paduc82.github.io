brian tillman [outlook mvp 2007-2019] say:
The POP folder in your Outlook.com mailbox is caused by a POP mail client requesting the download of your messages from the mailbox, then issuing a delete command to the server.  The Outlook.com server ignores that command and instead of deleting the messages, it moves them to the POP folder so you never lose your messages.

If you want to fix this, then configure whatever mail client downloads the messages to leave copies on the server.  Then the messages will stay in the Inbox.

hansv mvp say:

You need a few lines of extra code to insert the field names. Add the following variable declarations:

    Dim fldCount As Long
    Dim iCol As Long

Change the With ... End With part of your code as follows:

With rsPubs
    ' Assign the Connection object.
    .ActiveConnection = cnPubs
    ' Extract the required records.
    .Open SQLselect & SQLfrom & SQLwhere
    ' Copy field names to the first row of the worksheet
    fldCount = rsPubs.Fields.Count
    For iCol = 1 To fldCount
        Sheet1.Cells(1, iCol).Value = rsPubs.Fields(iCol - 1).Name
    Next iCol
    ' Copy records starting at 2nd row
    Sheet1.Range("A2").CopyFromRecordset rsPubs
    .Close
End With
