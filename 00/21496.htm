dale howard [mvp] say:

Anon369 --

Thanks for including the screenshot, as that makes it easier to answer your question.  From the screenshot, I can see that there are two major deficiencies in how you are managing this project.

First, the Baseline Start and Baseline Finish fields for many of the tasks contain NA values, which indicates that you did not baseline these tasks.  Big mistake.  Before you go live with a project, you should save a baseline for the entire project.  From that point forward, the Baseline Start and Baseline Finish fields will show you the original planned start and finish date for every task.

Second, if you have Finish dates in the past for unstarted or in-progress tasks, this means that you are not rescheduling uncompleted work in the past.  This is something Microsoft Project will NOT do for you; you must do this yourself.  To do this, first set the Status Date value in the Project Information dialog to Friday of last week, which should be the last day of last week's reporting period.  Then click Project > Update Project.  In the Update Project dialog, select the "Reschedule uncompleted work..." option, and then leave the "Entire project" option selected as well.  When you click the OK button, Microsoft Project will reschedule all uncompleted task work in the past.  If a task has not started, the entire task will be moved to start in the current week.  If the task has started but is not yet complete, Microsoft Project will split the task and move the uncompleted portion to start in the current week.

Hope this helps.

john korchok say:

This is a different problem than your original post.

A new blank presentation contains 1 slide master and 11 layouts. If someone creates new slide layouts or changes their names, when slides based on those layouts are pasted, PowerPoint also pastes those layouts into the master. There's nothing else it could do: switching a slide's layout to one of the existing layouts would almost inevitably change the slide appearance.

If the layout for the pasted slides is missing a footer placeholder, PowerPoint will not automatically add one, because the original layout didn't have one. Instead, you need to fix the pasted layouts by adding footer placeholders back in.

In general, it's best to leave all existing placeholders and layouts as is during template design. Keep the existing layout names, keep all the placeholders. Often people delete elements that later turn out to be crucial.

chirag. say:

You were almost there. Try the following:

  1. Open two PowerPoint presentations.

  2. On one of the presentations, go to the Slide Show ribbon and change the Monitor to Monitor 2.

  3. Click "From Beginning" button in "Start Slide Show" group.

  4. On the second presentation, change the Monitor to Monitor 1/Primary Monitor.

  5. Click "From Beginning" button in "Start Slide Show" group.

Your slide shows will start full screen - one on each monitor. If you want to run multiple slide shows simultaneously, look at OfficeOne PowerShow from http://www.officeoneonline.com/powershow/powershow.html.

Consider this as a first pass at the code. I already know of several situations that it doesn't handle properly and will require more code if such things occur in your documents.

  • The macro doesn't try to match the capitalization of the replacement to the capitalization of the found word.
  • If the word you search for occurs at the beginning of a sentence in the middle of a paragraph, the "preceding word" will be the last word, punctuation, and space of the preceding sentence, and the replacement will be incorrect (and probably nonsense).
  • If the word you search for occurs at the end of a sentence, the "next word" will be the first word of the next sentence, plus the intervening punctuation and space. Again, probably nonsense will result.
  • If the word you search for occurs at the end of the document, the "next word" will be the punctuation at the end of that sentence, which will disappear.
