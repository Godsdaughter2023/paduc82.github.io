Copy the following code into the module:


Function GetStart(Name As String, Instance As Long) As Variant
    Dim r As Long
    Dim c As Long
    Dim f As Boolean
    Dim n As Long
    Dim wsh As Worksheet
    Dim cel As Range
    GetStart = ""
    Set wsh = Worksheets("Sceduale")
    Set cel = wsh.Range("C4:C" & wsh.Rows.Count).Find(What:=Name, LookAt:=xlWhole, _
        MatchCase:=False)
    If cel Is Nothing Then Exit Function
    r = cel.Row
    For c = 6 To 65 ' F to BM
        If wsh.Cells(r, c).Interior.Color = vbRed Then
            If f = False Then
                n = n + 1
                If n = Instance Then
                    GetStart = wsh.Cells(3, c).Value
                    Exit Function
                End If
                f = True
            End If
        ElseIf f = True Then
            f = False
        End If
    Next c
End Function

How do I tally the number of occurrences of different values in a - Microsoft Community

mclouthier say:

How do I tally the number of occurrences of different values in a column?

I have a column of roughly 58,000 different numerical values (ZIP codes to be specific), many of which are duplicates.  I need to automatically tally the number of times each value appears and generate a percentage of the ones that are statistically significant.  What would be the easiest way to do so?

Also, is there any way I can isolate the first five digits in order to removethe (-xxxx) +4 number that some of these values have without having to go through manually?  As right now there are about 43,000 unique values including them but the +4 numbers are irrelevant to my needs.

Thank you for any help, this project has been a real thorn in my side. and it should be a lot easier after this step.  I'm just not too experienced with this sort of thing.

jllatham say:

OK - assuming that you've gotten the entries down to simply 5-digit ZIP codes (see your other post for a couple of ways to do that), then we can use a couple of functions and formulas to deal with this counting part of it.

 

Select all of the cells in the column of Zip codes (you should have a 'header/label' at the top of it and you should select that as part of the group) then go to:

 

Data tab - Sort & Filter group - Advanced

 

In the Advanced filter dialog, choose
Copy to another location
Check the Unique Records Only box
and click in the "Copy to:" area and pick a cell in an empty column somewhere (with another empty column to its right)
Click [OK]


You now have a list of the unique Zips in your 53000+ source list

 

Next to the first one, put a formula like this (which assumes your original list goes from A2 through A53241).  It also assumes that your unique list was placed into column Z starting at Z1 (with the label), and that the formula is going into AA2:

=COUNTIF($A$1:$A$53241,Z2)

and fill that down to the end of the list of unique entries. 

 

You could then use the same Edit --> Copy and Edit --> Paste Special w/Values to turn the formulas in column AA into the actual counts, and then sort the two columns by the count.  Then you can determine which are "statistically significant" from those results.

 

For this Edit/Copy --> Edit/Paste Special (Values) you'd copy the cells with the formulas in them in AA and simply turn right around and use the Paste Special on that very same group of cells.

 

Hope this helps.

Deleting migration batch

What I have is a migration batch, that has been stopped after it did the transfer of user mailboxes to office 365 from our on premises exchange. This was a cutover migration, users are using the office 365 now for two weeks. The batch was running for 5 days and it did a few incremental syncs, and then it was stopped. 

Now the question which I have is if I can delete this migration batch  (which is currently in a stopped state) without it affecting current mailboxes users are using now for two weeks already. I am afraid it will delete the batch and the mailboxes with it as if I click on the delete (trash can) button, is telling me, that  it will take a very long time ... 

henry huang- [o365] say:

Hi SamuraiS,

It's OK to delete the migration batch if you have confirmed the following key points:
1. You have already pointed the MX record to Office 365.
2. Users are using the Office 365 mailboxes and can receive emails directly in Office 365.
3. All emails that were sent to the on-premises mailboxes before the MX record is changed have been synced to the Office 365 mailboxes.

Deleting the migration batch won't remove the current mailboxes and emails in Office 365. Depending on the number of the users, it may take some time for the batch to be deleted. It's an expected behavior and won't affect the Office 365 services.

Thanks,
Henry
