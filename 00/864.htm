    jon - vertex42 say:

    Sounds like an interesting idea for a template. I'm assuming you are wanting to use Excel for this. To keep things simple for all the staff, you'd probably want a form for data entry, perhaps some macros to make data entry easier, and possibly even a way to select from a common set of standard options. Here are some steps you might want to follow to proceed (not necessarily to answer via this forum, but to answer for yourself and whatever consultant you might find to help you):

    1) Define the purpose and need for the template. Answer these questions:

    Why must we keep these records?

    What data must be reported using this data? How often? What format? Who sees it?

    Why are you wanting to use Excel for this (instead of Access or some other app)?

    2) Create a simple table in Excel to contain each record, with all the columns labeled as needed (worksheet named "Data" and labels on row 1)

    3) Populate the data table manually with all the different types of records you can think of. Use this sample set of data to figure out proper formatting and to experiment with different reports you might want to generate. This will also help if you bring a consultant on board.

    4) Figure out what you want to do with your data (refer back to answers from (1)). Are you going to use Excel as a data source for a Power PI dashboard? Are you going to create your own dashboard in Excel? Is your data table designed such that you will be able to create the needed reports easily?

    5) Create another worksheet to be used as the data-entry form. This will depend heavily on steps (2) and (3). Make a list of all the features you would like the form to do (along with noting whether they are critical features or not).

    6) You can create lists to use in your data-entry form either within the data-entry worksheet, or in other worksheets. Making these lists editable would probably be useful, so consider using dynamic named ranges to name each list. If you have multiple common options for a date field, you could create a list that contains today's date (=TODAY()), a date 14 days from now (=TODAY()+14), etc. With a data validation list, you can also allow the list to contain whatever date is entered.

    7) Use a Shape object to create a SUBMIT button and use a macro to add a row to the data table when you submit the form. You should include some data validation to avoid submitting a junk record, and avoid double-submitting by clearing the form after a successful submission.

    Hope that helps. I probably left some things out, but if you do steps (1)-(4) and make a list of some features for (5), that will help you do what you need before bringing a consultant on board to help with the rest.

    Morning Star Profitable Grade is based on return on shareholders' equity (ROE) using data from the past five years. Companies with less than four years of consecutive ROE figures, including the ROE figure for the latest fiscal year, are excluded from the calculations. For the remaining universe of stocks the profitability grade is based on the following three components:

    (1) The historical growth rate of ROE.

    (2) The average level of historical ROE.

    (3) The level of ROE in the latest fiscal year.

    The Morningstar Growth Grade is based on the trend in revenue per share using data from the past five years. For the purpose of calculating revenue per share we use the past five years' revenue figures and corresponding year-end fully diluted shares outstanding; if year-end fully diluted shares outstanding is not available, we calculate this figure by dividing the company's reported net income applicable to common shareholders by the reported fully diluted earnings per share. A company must have a minimum of four consecutive years of positive and non-zero revenue, including the latest fiscal year, to qualify for a grade.

    Hi, it's me again!

    On Technet (Link) I was given the advice to use ProcMon to see what IE does when it's started. And, I think I might be on to something here:

    I found out that on startup, IE looks for the following registry key:

    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Internet Explorer\Main\FeatureControl\FEATURE_EUPP_GLOBAL_FORCE_DISABLE

    Okay, in ProcMon, I saw that it got a "not found" error. So I created that key.

    Then, I saw that it was looking for the following registry entry:

    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Internet Explorer\Main\FeatureControl\FEATURE_EUPP_GLOBAL_FORCE_DISABLE\iexplore.exe

    So, basically, I deployed the following .reg file to all of my machines:

    Windows Registry Editor Version 5.00
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Internet Explorer\Main\FeatureControl\FEATURE_EUPP_GLOBAL_FORCE_DISABLE]
"iexplore.exe"=dword:00000001
    Could other users in this forum please test to confirm?

    By the way, I also found with ProcMon that if IE cannot find this entry in the HKLM hive, it then also looks for it in the same place only in the HKCU hive, so if someone (for whatever reason) wants to disable EUPP only for certain users, this should work again.
