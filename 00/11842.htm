trevor rabey say:

I am afraid you're going around in circles. If you already know the dates, you don't need the predecessors.  If you already know the dates, you don't even need the software.

You have already found that you can use the bar styles to display any bar between two dates, whether they are the calculated start and finish, or some other pair of dates such as your custom start and finish.

Starting with the dates to generate the durations is somewhat futile, and so much work to maintain. The alternative is so much better and easier.

Starting with the durations, however, the purpose of the predecessors (and successors. Every task should have one of each) is to calculate the earliest start, earliest finish, latest start and latest finish dates, and then to calculate the slack(s) and then to identify the critical tasks, or rather to partition the set of tasks into those which are critical (where critical = 0 total slack) and those which are not.

What you are trying to do seems to be working in direct opposition to the critical path method and how MSP was designed to be used. It will be a lot easier to work with the software rather than against it, or trying to bend it to your will, especially MSP. You can't separate MSP from the critical path method because the CPM is embedded in the scheduling engine.

What you are trying to do could be more directly obtained by just typing in all of the start dates and durations directly into those fields in the entry table, or, "better" (?!) making all of the tasks have a must start on date constraint. Then there is no need to use any predecessors. If you do, then they will be constantly raising scheduling conflicts.

Any help?

john - project say:

MIchael,

I think Trevor covered the overall problem with your approach but let me answer your question a little more directly.

Extra fields (e.g Start1, Finish1, etc.) in Project are static, they have no dynamic relationship with Project's scheduling fields (e.g. Start, Finish, Duration, etc.). You could in fact create custom field formulas that would take a custom field start date along with a custom field duration and create a custom field finish, but none of those values can influence data for any other task (i.e those fields cannot be linked).

My question is very basic. What exactly are you trying to accomplish? As Trevor noted, if you know all the dates, why use Project?

From your first few sentences it sounds like you are trying to create a schedule template that can be used for various types of projects. That's fine, but understand that you will need to edit key parts of the template (i.e. project start, task durations, task linkages), to best describe the logical flow of events for each and every project that is created from the template.

If you want to do all projects in one file, then I suggest you separate each "project" as a summary line and plan out the details as required under each summary.

Hope this helps.

John

It's not clear what you mean by "export", but I think what you're looking for is Word's Table of Contents. You'll find that button at the left end of the References ribbon. Here are some recommended articles about it:

https://shaunakelly.com/word/numbering/tableofcontents.html

https://wordmvp.com/FAQs/Numbering/NumberingExplained/NumberingInAction/TOC.htm

In a document where each heading style has a corresponding Outline Level in its paragraph-related properties, the Table of Contents can be made to include each paragraph that has one of those styles. a. user say:

Hi,
 

Thank you for contacting Microsoft Community.
 
I understand that you would like to disable Numlock at the startup. You may enable it by using the registry fix. Follow the below steps.
 
Note: You must be logged into Windows 10 with a local account and have administrator privileges to update the Windows Registry. If you log in with a Microsoft account, your updates to the registry will not be saved.
1.Press the Windows key + R on your keyboard to open the Run window.

2.Type regedit in the Open text box and press  Enter to open the Windows Registry editor. If a User Account Control dialogue windows opens, click the Yes button to continue.

3.In the Registry, open the below folders:
HKEY_CURRENT_USER\
 Control Panel\
 Keyboard\
 
4.Within the keyboard folder, there should be a string value named "InitialKeyboardIndicators" with a value of 0, 1, or 2. Below is the explanation of each of these values.

 0 = Num Lock is turned OFF after the logon.
 1 = Disable Num Lock.
 2 = Numlock is turned ON after the logon.
 
5. You may change the value accordingly and close the registry editor.
 
6. Reboot the computer and check if it helps.
