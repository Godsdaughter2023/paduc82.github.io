dhookom say: The DateDiff() function always counts the number of "thresholds" that have been crossed. There are 33 month threshold changes between your dates.

dhookom say: To add to my reply it's odd but true that these two expressions both return 1 month even if the difference in days is quite significant.

? DateDiff("m",#1/1/2017#,#2/28/2017#)
1

? DateDiff("m",#1/31/2017#,#2/1/2017#)
1

? DateDiff("d",#1/1/2017#,#2/28/2017#)
58

? DateDiff("d",#1/31/2017#,#2/1/2017#)
1

vijay a. verma say:
34 months answer would be for 8/1/2020 not for 7/31/2020. This behaviour is consistent even in Excel where DATEDIF results the same answer as Access is returning.

The reason is that default date alongwith time is 8/1/2020 00:00 and 7/31/2020 00:00. For 34 months to complete, the clock has to reach 7/31/2020 24:00 which is equivalent to 8/1/2020 00:00

scottgem say:
One more point to add here is that a date/time value may include a time component. Since a date/time value is a double precision number where the decimal portion represents the time, its possible that the decimal portion is not 0 but may be .000000000000011. While this will show up as 00:00:00, Acesss will still count it so that the threshold of Oct 2017 has not been crossed. So you are getting correct results.

If you want to include a data where the day is the first of the month. You can try this:

DateDiff("m",[firstdate]-1,[seconddate])

jay freedman say:

Start a new blank document and save it as a macro-enabled template (*.dotm). 

In a regular module in the template, insert this code:

Dim oAppClass As New ThisApplication

Public Sub AutoExec()
    Set oAppClass.oApp = Word.Application
End Sub

Insert a class module in the template, and use the Properties pane to rename it as ThisApplication. Add this code to it:

Public WithEvents oApp As Word.Application

Private Sub oApp_DocumentOpen(ByVal Doc As Document)
    On Error GoTo errhdl

    With ActiveWindow.View.RevisionsFilter
      .Markup = wdRevisionsMarkupAll
      .View = wdRevisionsViewFinal
    End With
    Exit Sub
errhdl:
    MsgBox "Error " & Err.Number & vbCr & "Couldn't set markup type", , "Error"
End Sub

Save and close the template. Copy or move it to the Word\Startup folder. Restart Word.
