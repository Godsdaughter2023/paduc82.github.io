Current Canary is 16223 (and a definition of "canary") - Microsoft Community

cbarnhorst say:

Current Canary is 16223 (and a definition of "canary")

There should be a good flock to choose from for next week.

For you trivia folks, here is the definition of Canary testing and deployment from WhatIs.com.

In software testing, a canary is a push of programming code changes to a small group of end users who are unaware that they are receiving new code. Because the canary is only distributed to a small number of users, its impact is relatively small and changes can be reversed quickly should the new code prove to be buggy. Canary tests, which are often automated, are run after testing in a sandbox environment has been completed.

For incremental code changes, a canary approach to delivering functionality allows the development team to quickly evaluate whether or not the code release provides the desired outcome. The word canary was selected to describe the code push to a subset of users because canaries were once used in coal mining to alert miners when toxic gases reached dangerous levels. Like the canary in a coal mind, the end user who is selected to receive new code in a canary test is unaware he is being used to provide an early warning.

Of course we do know we are receiving the code, but you get the procedure.  So there you have it.  That is why MS uses "canary."

    cbarnhorst say:

    Here is a link to help answer that:

    https://msdn.microsoft.com/en-us/library/mt238502.aspx?f=255&MSPPError=-2147217396

    "Flighting' is all about streamlining the development process and getting away from "big bang" delivery of feature sets; i.e., replacing the three-year cycle MS has used in the past.  It also contains the now familiar concept of staging feature releases starting with users less likely to experience problems to larger and larger user pools.  Telemetry from early stages are used to head off potential issues for later users.

    "Flighting" is a pretty general description for all of these changes in delivering new features and feature sets, all the way up to new entire versions.

    This weeks flighting of the new OneDrive feature set is a really good example.  It is limited to a subset of users (fast ring, build 16215, and staged based on discovery of issues as the flighting progresses).

    All of this is why MS is reluctant to use "releasing and instead uses "flighing" in the Insider program.  They reserve the word "release" for "software available to the general public."  So they do not release insider builds.  They flight them.  The difference is more than just words as you can see in the article linked above.  It is a defined philosophy of software development.  It is much broader than philosophies of software testing. 

    I think this is what distinguishes the Insider program from past programs like TechBetas.  Because it is much broader than beta testing, the old methods of participant selection are not needed.  User self-selection is fine (if the participant base is sufficiently large).  Insiders are apparently not perceived so much as testers and early adopters and the boundary between beta testing and the general user base is blurred.  Hence the "ring" concept.  In that conceptualization, the final ring is the "release ring."

    In the example, "Banana"is highlighted as a duplicate between columns A and B, but "Mango" is not highlighted since it doesn't exist in column A.

    a. user say:

    Hi Brandon,

    You may use the conditional formatting to highlight duplicates between columns, you may refer to the below steps for the same.

    In the table that you have provided, consider the fruit A is in cell A1 and fruit B is in cell B1, banana(of fruit A) is in A2 etc.,

    1. Now select the first column, A2:A6.
    2. On the Home tab, click Conditional Formatting in the Styles group.
    3. Select New Rule.
    4. In the resulting dialog, select the last option i.e., “Use a formula to determine which cells to format”.
    5. Enter the formula =COUNTIF($B$2:$B$6,A2) in the “format values where this formula is true” option and then click Format.
    6. Select the background color of your choice, and click OK.

    7. Click OK to return to the sheet.

    Repeat the above steps by selecting the data in column B, B2:B6, and entering the formula =COUNTIF ($A$2:$A$6,B2).
