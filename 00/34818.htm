Hi Jim,

The "component 'comdlg32.ocx or one of its dependencies not correctly registered: a file is missing or invalid" error is one of the runtime errors and will appear while the program is running.

The error "The module comctl32.ocx failed to load" is usually cause by un-registered COMCTL32.OCX component.

First, you need to verify if comctl32.ocx is in C:\Windows\System32 folder. If found, run again the regsvr32 comctl32.ocx command.

Note: Run Command Prompt as Administrator.

From the Network and Sharing Center, click the link to "Manage wireless networks" to get this screen:

You can proceed in several different ways. 

1.  Delete all of the configured wireless networks except any that you know relate to networks OTHER than your own that you need to keep (for example, a wireless network at work).  Now connect to your wireless network in the usual manner and carefully enter the encryption password.

2.  Modify the wireless network shown in the list to have the correct encryption password.

3.  Delete all of the configured wireless networks except any that you know relate to networks OTHER than your own that you need to keep (for example, a wireless network at work).  Connect to your router with an Ethernet cable and log in to the router's configuration utility.  Disable ALL security measures configured for the router:  encryption, MAC filter, time of day restrictions, etc.  Now connect to the wireless network in the usual manner.  If this works, you can re-configure the security measures using the router's configuration utility one by one (although you really don't need to use anything other than WPA2 encryption and a strong password).

If you are unable to wirelessly connect to the router when you have disabled all security measures on the router, check any third-party security software installed on your computer to see if it is blocking the wireless connection.

Your private desktop folder is here - C:\Users\<username>\Desktop and the public desktop folder is here - C:\Users\Public\Desktop.  If i'm not mistaken, the public desktop is a hidden folder so you'll seen to turn on the ability to show hidden files and folders in your view preferences (control panel).

steve winograd say: The simplest way is to run Windows Update and install all the updates it offers.  Repeat until there are no more updates, which will probably take several runs and restarts.

If you know what service pack level the restored system has (look in Control Panel > System ), you can download and install a later service pack (there are three).  For example, if the computer has Service Pack 1a or Service Pack 2, you can install Service Pack 3 .  After installing SP3, run Windows Update repeatedly as above.
