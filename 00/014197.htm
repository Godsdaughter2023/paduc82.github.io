jay freedman say:

The following revised macro will always insert smart quotes. It's slightly more complex because it has to know where to insert an opening quote vs. a closing quote; the AutoFormat logic does that without help.

Sub DoubleQuoteSelection()
    Dim smrtOpenDblQt As String
    Dim smrtCloseDblQt As String
    Dim rg As Range

    smrtOpenDblQt = Chr(147)
    smrtCloseDblQt = Chr(148)

    If Selection.Type = wdSelectionNormal Then
        ' if selection ends with spaces and/or nonbreaking spaces,
        ' deselect them
        Selection.MoveEndWhile cset:=" " & Chr(160), Count:=wdBackward

        ' if the characters immediately before and/or after
        ' the selection are single (straight or smart) quotes, include them
        Set rg = Selection.Range
        If Selection.Range.Characters.First.Previous = "'" _
            Or Selection.Range.Characters.First.Previous = Chr(145) _
            Or Selection.Range.Characters.First.Previous = Chr(147) Then
            rg.MoveStart wdCharacter, -1
        End If
        If Selection.Range.Characters.Last.Next = "'" _
            Or Selection.Range.Characters.Last.Next = Chr(146) _
            Or Selection.Range.Characters.Last.Next = Chr(148) Then
            rg.MoveEnd wdCharacter, 1
        End If


        ' if there are single quotes in the range, replace them
        ' with doubles; otherwise nothing happens
        rg.Text = Replace(rg.Text, "'", smrtOpenDblQt)
        rg.Text = Replace(rg.Text, Chr(145), smrtOpenDblQt)
        rg.Text = Replace(rg.Text, Chr(146), smrtCloseDblQt)

        ' if the first and last characters are not double
        ' (straight or smart) quotes, add them
        If rg.Characters.First <> smrtOpenDblQt Then
            rg.Text = smrtOpenDblQt & rg.Text
        End If

        If rg.Characters.Last = smrtOpenDblQt Then
            rg.Characters.Last = smrtCloseDblQt
        ElseIf rg.Characters.Last <> smrtCloseDblQt Then
            rg.Text = rg.Text & smrtCloseDblQt
        End If
    End If
End Sub

I left the processing of straight single quotes in the macro, just in case they're present. If there are none, it doesn't add any noticeable delay.

No worries. Glad you found them.

 

The pictures have the same names because Windows Phone saved them with those names. Zune does not let you change the file name. Because you had a Quantum with a Camera Roll of imported photos, Zune displays these in its Collection view. Now you have a Lumia 900 with its Camera Roll, Zune will import these and, unless the old photos with the same name are in a different directory, Zune will add a (2) or (3) or (4) depending on howmany photos there are witht the same name in that directory.

Hi Nilesh123,

The field that you are trying to update needs to be editable in SharePoint. In order to check if the field is editable, go to SharePoint Library Settings and under Columns section, the column need to be displayed as a link (in blue color), and not a static text.

If it's not a link, then you might have added the promoted property in the publishing wizard but didn't check the option to allow editing. To fix that, republish the form template. When you reach the promoted properties window in InfoPath's publishing wizard, select the field you want to update and click the Modify button. In the property's window, check the box at the bottom that says "Allow users to edit data in this field by using a datasheet or properties page". Finish publishing the form.

Refresh your SharePoint Library's settings, you should now be able to see the column as a link, which means it's now editable. Now when you go to SharePoint Designer, that field will show up in the list.

robbiddle- [o365] say:

I've resolved this issue.  After looking through the Fiddler packet capture I saw the HTTP 407 responses from our Barracuda web filter.  Even though the URLs were white listed it was blocking traffic due to proxy authentication failing.

I added in the following to the Barracuda web filter proxy authentication exemptions:

^(.*\.)*live\.com:443

^(.*\.)*office\.com:443

^(.*\.)*office\.net:443

^(.*\.)*microsoftonline\.com:443

^(.*\.)*microsoftonline-p\.com:443

^(.*\.)*windows\.net:443

Shared Computer Activation is now working properly.
