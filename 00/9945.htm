How can I take a filtered set of data rows and shift the filtered data over 1 column

I have created a filtered subset of data.  The cells in the filtered data contain information in 1000's of rows spanning Columns C through O.  I now need to shift that filtered data over one column to the right.  The end result should be that the data be in Columns D through P.  I tried selecting the filtered data, Find & Select, Special, Visible Cells, Cut (Ctrl X), Paste in the new starting column (Ctrl V).  I won't allow me to do this with "multiple sections".  How can I accomplish what I am trying to do?

 

Thank you!

ronrosenfeld say:

Be sure to do this on a copy of your data, and not the original.

First, filter your data as you desire.

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.

============================================
Option Explicit
Sub ShiftFilteredRows()
    Dim rg As Range, rw As Range
Set rg = Range("C2", Cells(Cells.Rows.Count, "C").End(xlUp))
Set rg = rg.Resize(columnsize:=13)

Application.ScreenUpdating = False

For Each rw In rg.SpecialCells(xlCellTypeVisible).Rows
    rw.Cut Destination:=rw.Offset(columnoffset:=1)
Next rw

kergga say:

I suggest the first step would be for you to identify specifically what about the built-in anchors does and doesn't work for your needs. If you are connecting two shapes (that have anchors, like a circle and textbox) with a line, then when you move either shape the line will move automatically as well. Create both your shapes first (circle and textbox) and then apply the line so that you see the red anchor before dropping each end of the line. The only downside is that the line doesn't redirect to the *nearest* anchor, so if you move the textbox or circle too far around the other object, the line would fall behind or across the shapes. I think if you only have one line, the math is easy- just calculate the distance from every anchor on shape A to every anchor on shape B, and re-draw only the shortest line. You just have to be able to (a) get the name of the existing line to delete it, (b) the name of the connected shape on the other end, so you know who to connect to, and (c) an event trigger [although I'm not sure what you could use for the event, someone else will have to chime in here].
