techfreak_ say:

Skype only stores the last ~30 days of history in the cloud.  All your history beyond ~30 days is stored locally in your %appdata%\Skype folder.  Skype at no time deletes your history through any upgrade or downgrade process.  You can make backups of this folder and even export this data, if the data exists locally.  If you do not have access to this local data due to inaccessibility, corruption, deletion or being overwritten, then the only backups that might exist are any system backups you might have made during this time (that included that folder).  Also if you accessed other PCs in that time frame, some part of your history may be there.  Beyond that is contacting the person you were having conversations with and ask them to export what they have and provide you with a copy.

In reference to other devices with older history, a main.db file may be present in other folders. Search your devices for main.db files that might contain your history.  If you locate any on those devices, you can copy that data to your computer and try to export that data using the Skype application.

a. user say:

I don't think KarthiK fully understood your question - you wouldn't be using XP's backup function at all (it woulld just be a hard drive with XP installed as the OS but mostly just the space which is just NTFS free space).  Your question is can you use a network as a backup target instead of an External HDD, CD/DVD, or FlashDrive.  The answer is absolutely yes.  When asked to select the backup target, one of the options is a network and it allows you to browse for that network connection and target location.  Now you have to make sure that the target location has the appropriate permissions to allow the backup to copy to it (and to allow it to restore back to the PC if that becomes necessary - so the permissions must be in both directions).  Here are some procedures (though they don't apply specifically to a network backup, most of the procedures are the same - it's just that one step that's different): http://www.vista4beginners.com/How-to-backup-your-data.  Automatic File Backup isn't available in Vista Basic but the others have it.  Complete Image Backup isn't available in Vista Basic or Vista Premium, but the others have it.  All can  be done to a network share and the procedure guides you through the process.

The only hard part about this is setting up the permissions so the share is recognized as a valid location by the backup program.  Here's a link to a set of links regarding network sharing and how to set it up http://www.howtonetworking.com/vista/networksharing.htm (but if you're asking about this, I'll bet you already have a shared network set up or already know how to do it).  I have one on my system and I use the network to backup both my laptop and my wife's PC to an external drive attached to my laptop - and it's set to do a complete image backup (which I do manually) and automatically do an incremental backup every day at noon for mine and at 3PM for hers (we put our computers in sleep mode at night so an evening backup wouldn't trigger until we awakened the PCs anyway - and it doesn't take up too many resources running in the background).  And her PC runs XP Professional SP3 and mine runs Vista Business SP2 - so linking the two operating systems isn't a problem. It's not exactly like you plan, but it's quite similar and it does use a network to accomplish the task (at least for hers).

If you need any further assistance setting it up, don't hesitate to post - but I think you'll do just fine.  Now remember, to setup the network there must be a router (perhaps a wireless router) or the computers need to be connected with a crossover cable (NOT a standard USB cable) if you have the proper connections available.  I use a wireless router so the systems don't have to be brought next to each other every time I want to do some networking - they're on different floors of the house.  The PC is connected directly by USB cable to the router and my laptop connects wirelessly. Our internet connection is tied directly into the router so we share that as well and I share the printer attached to her PC.  It all works out quite nicely.  I say this to show that networking can work just as easily in your situation and without a big expense (a wireless router can be obtained for around $50 - maybe even less since I haven't checked recently).

Good luck and I hope this helps.
Lorien - One - MCSE/MCSA/Network+/A+

andre da costa say:

Once in a while, your computer might nor close a program, resulting in the window of the program freezing (or not responding to any mouse action). In some instances, you can wait a short time and the program will begin responding again and you can end that session appropriately. Other times, program remains in a non responding state for a long period of time which can keep you from getting other things done.

The cause of non responding programs can be a number of things, the way the program was engineered, not enough memory resources allocated to the program (sometimes you can’t close out some programs if you have too many open), an add on for the program is causing instability or its just taking some time to leave memory and stop being resident. Regardless of these factors, you just want to end that program and move on. Please understand though, certain programs that are not ended properly might cause you to lose unsaved information. So if its something important try giving it some time to start responding again.

How to close non responding programs using Task Manager

Click Start, type: Task Manager.

Hit Enter on your keyboard.

or you can right click an empty spot on the the Taskbar and click ‘Start Task Manager’.

The above screenshot displays a list of all open applications. One in particular is Not Responding. The first thing you can do is select it then click End Task sometimes this works and sometimes it doesn’t. To work around this, we will will go to the programs process.

Right click the non-responding program and click Go To Process.

Once the process has been selected, just click the End Process button and the program will be closed.

This should be used as a last resort in some instances. As noted earlier, some programs will take time to be released from memory, background dependencies and other factors such as the amount of open applications can also trigger stop responding.
