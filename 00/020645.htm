alexjx say: Hello everybody.

I think that this could be the solution. I had the same problem and solved it.

Well, let me explain this first: when you install symbian software (sis extension) on a Memory Card (MC) on your S60 3rd Edition Phones, the phone would also make the uninstall application file in a folder in [MC]:\private\something. In my phone (Nokia E71), the uninstall files are written in [MC]:\private\10202dce (to check where your phone writes this file, see later). Then, when you open the application manager, the phone will check this directory (in phone and memory card) to find the uninstall list of all application in the phone. The phone also checks this folder when you switch it on, and that's the reason why the annoying message "Unable to complete installation from memory card..." appears during start up.
Actually, the phone built in "file manager" application and 3rd party application don't grant us access to that folder. So, how can you delete an application uninstall program on application manager? Well, you can access that folder via PC. This is possible if you connect the phone with cable data and make phone as mass storage (the option will pop out in the phone when you connect it to PC via USB).
Now that you have your phone connected you can use Windows Explorer and search in the MC's "private" folder the "*.sis" files (without quotation mark). The folder that contains all the sis files in the MC's private folder, is the folder that contains the uninstall application files that the phone makes during the installations of the applications (Like I said before, the uninstall folder on my E71 is [MC]:\Private\10202dce).
You probably will encounter several sis files in this directory, as it happened to me. While on S60 1st and 2nd Edition you can delete uninstall list phone through rename the appropriate files, in S60 3rd Edition phone, you can't, because windows will prevent it. So, the best thing to do is to move all these files into another folder on the PC (you can also delete them, but I don't recommend it, becasue, if you delete the appropriate uninstall file of a program, after, if you want to uninstall it, you have to do it manually, deleting one by one all the files that the application istalled, and this is very difficult), otherwise if you just want to remove some of these, you must find the appropriate file that uninstall the application that causes the problem. To find the appropriate file sis to remove, you'll need to use an application on PC, like "SIS Editor", with which you can extract or view the sis files and find which uninstall file it is. After that you find the appropriate file, you can move it to another folder to delete it from \Private\10202dce directory.
Now, when you want to uninstall an application from application manager list, first you have to move back the appropiate sis file to the uninstall folder \Private\10202dce through PC.

I hope this can be useful, and sorry for my English
