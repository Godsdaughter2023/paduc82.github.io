rob koch say:

Windows (actually Microsoft) Update has always been used by MSE in the background, the change made a few years back was only to add the ability to download these updates directly via the Windows Update interface, allowing for an alternative method to bypass the MSE update mechanism if it was failing for any reason.

Though it does sounds as if the download file you have received is for the x86 (32-bit) version.  However, since the update package installer itself checks for the correct version of Windows/MSE, this will simply display an error message and abort if the file is the wrong one for your system.

The size of the file, though relatively consistent or growing over time, can vary widely due to both changes in the antimalware engines as well as the number and corresponding changes to the signatures based on the updated engines.  The reality is that the full definitions installation package only contains a small portion of the millions of signature for known malware.  Since many of these definitions are for either older versions of Windows or malware that is more common in other parts of the world, your PC only receives a small portion of the definitions that cover malware which is common in your region.

The operation of a cloud-based antimalware system like MSE today typically only contains a small "cache" of the most common definitions, while the remainder exist in an online database which the client PC can access via the Internet.  If something detected by the behavioral and other local monitoring isn't recognized as an exact match, MSE uses the MAPS system to request a [better] match.  If found this will cause a download of the new definition and if not, the malware signature will be automatically submitted (along with those from others) for either automated or manual analysis as possible new malware.

I noticed by accessing the MSE download myself that the name displayed for both the 32-bit and 64-bit versions is exactly the same.  However, the size is slightly larger for the 64-bit which makes sense, so these are likely correct.  Though my Windows 7 x64 based system already had the current definitions installed, I received no error when attempting to run it, so I suspect your download will work fine.

Rob

danida_u say:

Hi

Skype conversation histories are saved in the cloud for a period of time (last 30 days). Any history older than 30 days are saved in the device's local hard drive depending on your history settings.

Your chat conversations will load up after re-installing the program if you set your history settings to keep your history for forever, as long as the Skype folder was not deleted or remove in the process.

Now, if you completely uninstalled the app and deleted all your Skype files without backing them up, then I am afraid that we do not have an option to restore your old conversations anymore.
