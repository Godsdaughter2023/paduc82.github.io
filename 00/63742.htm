xircal say:

Hi Mary,

There are two ways of doing this. The easiest of the two is to make a Registry change. But it depends on how confident you are about changing Windows Registry keys.

The second option is to uninstall KB3035583 which is the Windows Update responsible for nagging you to upgrade.

But I'll deal with both and you can decide which method you want to use.

To block the upgrade using the Windows Registry, do the following.

  1. Right click the Start button and go to System.
  2. In the left hand menu, click the link to System Protection. This will open System Restore.
  3. Click the Create button at the bottom and then type a description which you'll recognise at a later date. Something like "Before editing the Registry" will do.
  4. After Windows creates your manual _Restore point, close all the open menus and proceed with the next step.
  5. Right click the Start button and go to Run (or hit Windows key + R)
  6. Type: REGEDIT and click OK.
  7. Open the tree in the left hand pane to: HKey_Local_Machine\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate
  8. In the right hand pane, look for DisableOSUpgrade and change its value from "0" to "1" (without quotes). To change a value, double click it and choose: Modify. In the dialog box which opens change the zero (0) into a figure 1.
  9. Close the dialog box and exit the Registry editor.
  10. Reboot the computer for the change to take effect.

Steps 1 to 4 in this sequence are a precaution. Changing or deleting registry keys affects the way the operating system behaves and can result in system instability if you edit or delete the wrong key. Creating a System Restore point enables you to wind back the clock to the time prior to making the change.

If you have to run System Restore at any time you can access it via steps 1 and 2 and then click the System Restore button at the top and select the _Restore point which you created.

The second method is to uninstall KB3035583 by doing the following.

  1. Right click the Start button and go to Control Panel. If you have the Category view, change it to "Small icons" via the link top right.
  2. Next, click Windows Update.
  3. In the Windows Update menu, click the link on the left near the top to "Change settings".
  4. In the Change Settings menu, change "Automatic Updates" to "Download updates but let me choose whether to install them" via the down arrow to the right of the setting.
  5. Click Save to exit.
  6. Next, click the link to "Control Panel Home" top left and then click "Programs and Features".
  7. In the next menu, click the link to "View installed updates" on the left.
  8. In the Search box top right in the next menu, copy/paste KB3035583.
  9. When it appears in the list below, right click it and choose Uninstall.

KB3035583 creates a folder called GWX at the following location: C:\Windows\System32\GWX. It contains six executables which are also installed along with a few other files which handle settings. Consequently, it will take about 10-15 minutes to uninstall, so be patient.

After the uninstall process completes, reboot the machine.

Because you changed Windows Update settings in step 4, Windows is going to display a message on your lock screen that "Important updates are available and to go to Windows Control Panel to install them" (or words to that effect).

So go to Control Panel again and click Windows Update to open the menu. This will be update  KB3035583 again. Right click that and choose "Hide update". It'll turn a pale grey colour and you won't see it again the next time you go to Windows Update.

Both methods have their merits and the Registry change is the simplest solution which allows you to maintain Automatic Updates, but you do have to be a bit tech savvy so to speak. When you delete a file in Windows, you always get a warning and can change your mind. In the Registry you can't. If you hit Delete while a value is highlighted, it simply disappears without any warning so you have to know what you're doing.

Uninstalling and then hiding KB3035583 is easier to perform, but does mean you lose the convenience of having Windows install automatic updates for you. If you leave Automatic Updates enabled, then of course Windows is going to reinstall KB3035583 for you and you'll be back where you started.

If there's anything you don't understand, please don't hesitate to say.

Hi Mary,

I'm afraid what's you done there is incorrect and you should delete the key you've created.

I see from your screenshot that you're at this location: HKLM\SOFTWARE\Policies\Microsoft\Windows when in fact, you need to be at HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate

I've created a graphic for you with a 1, 2, 3 step process which I hope you can understand.

In step 1, you have to right click the Windows subkey (which looks like a folder) and create a new one as shown in the menu. This subkey should be called WindowsUpdate and must be written as you see it here.

When you've created that, click it to open it so that you get a blank pane on the right hand side of the Registry editor.

In step 2, right click the blank right hand pane and choose New --> DWORD. Change the DWORD by right clicking it and go to Modify. This will open the menu you see in the middle of my graphic.

Change the value data from a "0" to a "1" as I've indicated with a red arrow. When you click OK, the 0x00000000 (0) setting will change to 0x00000001 (1) as shown by the green arrow.

balaji kundalam say:
Hi
Thanks for contacting Microsoft Community.

You need to replace the built-in CMOS battery. It usually happens if the power source (main battery and adapter) is removed for a long time (meaning, laptop not in use for long time) or if the laptop is in use for than 2 years (motherboard not replaced). You can contact the manufacturer support for replacing the battery - you can either call a technician to replace the battery or you can replace it by yourself by referring your laptop service manual.

Error code 2 is DOS speak for "file not found".

Windows IP Configuration

An internal error occurred: The request is not supported.

tcpip.sys

If the file is missing, you must replace it.

We know that XP likes to keep backup copies of critical system files in this folder:

c:\windows\system32\dllcache

Look there for a backup copy of the tcpip.sys file and copy/paste it into the c:\windows\system32\drivers folder where it belongs.

If you don't see a copy in the dllcache folder, look for a copy here:

c:\windows\ServicePackFiles\i386
