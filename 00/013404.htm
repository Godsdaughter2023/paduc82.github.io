greginmich say:

The MSE scheduled scan actually works quite well for scheduling Quick Scans if you simply reschedule it for a time when the computer is in use, and also uncheck the “on but not in use” option. But since the MSE scheduled scan can’t wake up the computer, or keep it awake when no one’s working on it, or shut it down at the end of the scan, an independent Task Scheduler task comes in handy for Full Scans.

 

Another advantage you have with an independent scan task is that you can limit the “Full Scan” to the C:\ drive by using this command line for the “arguments”:

 

-Scan -ScanType 3 -File C:\

 

I have 3 internal hard drives, and I’ve found that using this command line really cuts down the length of the scan. Since my hours are so irregular, I’ve actually stopped scheduling this “Full Scan” at this point, and now just run it with a desktop batch file that changes the sleep-timeout setting in order to keep the scan going if I leave the computer unattended:

 

@ echo off

 

powercfg -change -standby-timeout-ac 200

 

cd C:\Program Files\Microsoft Security Client

 

MpCmdRun.exe -SignatureUpdate

 

MpCmdRun.exe -Scan -ScanType 3 -File C:\

 

powercfg -change -standby-timeout-ac 20

 

 

If I don’t plan to get back to work, I use a second version of this batch file that shuts the computer down at the end of the scan with a shutdown.exe command.

Hi Mr. Jumbles,

 

The correct application in general is MSE - but some choices can't be done with MSE and so other procedures might be needed.  I do them while working as background tasks - so I've no need for the power saver-related "tricks" - but I do use Task scheduler to be able to add a  scheduled full scan to the daily quick scan scheduled using MSE.

 

Here's a couple of things about scheduled scans. To run, the PC must be on and connected to the internet (it cannot be shutdown, in sleep mode, in hybrid sleep mode, or in hibernation). The power saving settings must be such that the PC does not shut down or go asleep during the scan (or it will be as if it never happened). Scans do not begin precisely when scheduled - some can start an hour or so earlier or later than the officially set time (depending on Microsoft's servers and the loads at that time).  

 

There is a way to use Task Scheduler rather than MSE to awaken sleeping computers, run the scans, and then shut them down when done, but it is somewhat complex.  You can also use it to create different scheduled scans of different types since MSE only allows one choice.

 

Please see http://answers.microsoft.com/en-us/protect/forum/protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02 - especially the later corrections from GreginMich.

 

The post from GreginMich here http://answers.microsoft.com/en-us/protect/forum/protect_start/scheduled-scans-not-working/fd769043-f67e-4990-8805-a02427f22dff will also be useful.

 

Now back to scheduled scans in general in MSE as opposed to the above special cases.  Also, go to settings and check scheduled scans and uncheck the box that prevents them from starting unless the system is on and idle (if you're using the system when it's scheduled or it otherwise isn't idle - perhaps downloading Windows Updates or doing something else - then the scan will not start and will be missed). If a scan is missed it does not start at the next available time, but waits for the next scheduled scan.

 

Go to Start / Control Panel / Administrative Tools / Services / scroll to Task Scheduler and make sure it is started and automatic. If not set it to automatic and restart it. If it won't start, click on the dependencies tab and do the same thing for each of the ones listed. If any of those won't start, continue with it's dependencies and do the same thing. When you get to the end, reverse course and Start the ones you couldn't because of the dependencies as you go until you get to Task Scheduler and hopefully it will now go to automatic and start. If you changed nothing read on. If you changed anything, reboot and see if that helps.
