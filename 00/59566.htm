I had a similar problem - my Vista laptop was usually often very slow to connect to wireless networks, both at home (WAP2) and at free-wifi locations. At home, it could be 6 feet away from the wireless router and would not connect easily; even though my phone connected up fine

I also found this elsewhere in the forums and followed it

1.  ClickStart and type services. Choose the correct result in the list.

2.  Find the Network Location Awareness service, right click on it and chooseProperties

3.  Click theDependencies tab and look under "The service depends on the following system components:". Make a note of all the services listed (which should be Network Store Interface Service, Remote Procedure Call (RPC), and TCI/IP Protocol Driver).

4.  Hit Cancel and locate each service listed, right click it and chooseProperties.

5.  Set each service'sStartup type to Automatic.

6.  Click theStart button for each service.

You may have to start the dependent services of Network Location Awareness's dependent services, so it could take a bit of leg work.

I worked through both of these (there were two RPC services - I had to start one - the local one)

Since then, connection has been much quicker (although it is early days)

    SOLUTION:

    Thank you to everyone that responded quickly to help me. I've now fixed the issue.

    I've been extremely frustrated by this but I'm glad you all took the time to try to find the problem.

    The issue was that my desktop wouldn't communicate with the activation servers first and foremost.

    The watermark left by windows when the desktop was considered invalid caused problems when running games, so in an attempt to fix it I changed the value mentioned below to a '4' (recommended by another internet post) instead of a '2':

    HKEY_LOCAL_MACHINE > SYSTEM > CurrentControlSet > Services > sppsvc. Then in that folder near the bottom there is one called "Start" right click on that and click "modify" make sure it is in "Hexadecimal" and change the "value data:" to "2". After you have changed that restart your computer and go to your settings to re-type your product key and it should work. 

    (solution URL https://answers.microsoft.com/en-us/windows/forum/all/problem-activation-error-0xc0020036-windows-10/85126b3c-8476-4310-83bf-4d9c7801d39b)

    That prevented my desktop from communicating with the servers, but since changing it back to '2' I've seen all my problems go away.  Why microsoft tech support wasn't able to identify this within several phone calls is beyond me, but I realize that it was my own change that caused the problem.  Nobody listened to the "can't communicate with activation servers" part, but I digress.

    What is the Status shown at Settings > Update & Security > Activation? If it says it's Volume Licensed for an organization's server, that means someone has activated Windows with a Volume License Key meant only for business or organizations that activate a large number of PC's, which need to check in with the organization's server regularly to reactivate.

    Sometimes this is done by shops unnecessarily because the PC either came with a Windows Product Key embedded in the BIOS/UEFI firmware, or it was already activated for free Upgrade and would have reactivated after a reinstall had they given it the chance and not interrupted this by installing the illegal key.

    If you know Windows was originally activated legitimately by one of these methods then to fix the problem what I would do is go offline to remove the Volume License key and then restore the Generic Product Key that is used to activate with the Digital License issued by Upgrade or Factory install. Then we'll see if it activates:

    1) Go offline by opening Action center at the right end of Task Bar, then clicking on Airplane mode to turn off internet.

    2) Next type CMD in Start Search, right click to Run as Administrator, then right click to copy and paste this command in and press Enter: slmgr -upk

    3) Restart PC and then insert Generic Product Key for WIndows 10 Pro from here: http://winaero.com/blog/generic-key-to-install-...
