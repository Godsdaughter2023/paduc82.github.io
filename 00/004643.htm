graham mayor say:
For your field change in section 2 the following will limit the code to TOC fields in section 2. If there are more than 1 TOC field in Section 2 it will need changing to reflect which TOC field you wish to address (though in practice it will only affect a TOC field that has 2-2 or 2-3). Your original macro incidentally inserts 1-2 so the following will not affect that field.

Private Sub chkTOC3_Change() 'see note at end of macro
Dim oFld As Field
  'changes TOC with 2 levels to 3 levels and vice versa
If chkTOC3 = True Then
    For Each oFld In ActiveDocument.Sections(2).Range.Fields
        If oFld.Type = wdFieldTOC Then
            oFld.Code.Text = Replace(oFld.Code.Text, "2-2", "2-3")
            oFld.Update
        End If
    Next oFld
Else
    For Each oFld In ActiveDocument.Sections(2).Range.Fields
        If oFld.Type = wdFieldTOC Then
            oFld.Code.Text = Replace(oFld.Code.Text, "2-3", "2-2")
            oFld.Update
        End If
    Next oFld
End If
End Sub

will toggle 2-2 and 2-3 with the setting of the check box - however I would apply the change when you click the command button to leave the form, rather than when you click the check box.

To initialise the check box

Private Sub UserForm_Initialize()
Dim oFld As Field
For Each oFld In ActiveDocument.Sections(2).Range.Fields
    If oFld.Type = wdFieldTOC Then
        If InStr(1, oFld.Code.Text, "2-3") > 0 Then
            chkTOC3.Value = True
        Else
            chkTOC3.Value = False
            Exit For
        End If
    End If
Next oFld
End Sub

The macro recorder is not a perfect tool. It is incapable of doing many things that you can do in the vba editor, and it also adds a lot of stuff that is unnecessary. You can delete the stuff you don't need using the VBA editor and end up with something like

With ActiveDocument.Styles("Heading 1").ParagraphFormat
        .SpaceAfter = 6
    End With

Notwithstanding that this was not the question asked, the better way to do this would be simply to modify the paragraph style and you don't need a macro for that.

saberman say:

The fileDialog is available even if only the runtime package is installed.  What is not available is the typelib which is what you need the MS Office Object 12 library for.

You can use the following construct to get around this problem:

#If False Then ' Set to True during development
    Private fdlg As FileDialog
#Else
    Private fdlg As Object
    Private Const msoFileDialogFilePicker = 3
    Private Const msoFileDialogFolderPicker = 4
#End If

During development with a full version of Office installed include the reference to the MS Office Object 12 library and change the #if to #if True.  That will allow you to use intellisense and the defined constants.  When you are ready to build for release change the #if to #if False and remove the reference to the MS Office Object 12 Libray.  Make sure you have all of the constants you are using defined in the #Else section.  The code will compile clean and the compiled code will run with just the Runtime package installed.

davejb say:

Hello Robert

I think I have found a solution.  I know my computer is configured with the correct MIME Types for all the needed video format because I did this as one of my first attempts to crack this problem.

Anyhow, this being so I knew it must be my server that was not working with correct MIME Types.  I therefore searched for a configuration process to acheive this and it is quite simple

 

Create a .htaccess file with the needed information of MIME Types, and place it in the folder on your server containing the files in question (my video files).

 

The contents of the .htaccess file (in my case) needed to be as follows

 

# Video

AddType video/mp4 .mp4

AddType video/webm .webm

AddType video/ogv .ogv

AddType video/ogg .ogg

 

Of course there are miriads of other file types, which can be included in similar ways, some not as simply written as these.

 

I saved the .htaccess file as an All Files format file in NOTEPAD, named of course .htaccess, and uploaded it the VIDEO folder of my trial Website.

I opened IE9 with my trial page showing, and there were the videos! It Worked!

Thank you so much, cheerio my friend, DaveJB.
