(https://vvvxxxs.com.au:443/autodiscover/autodiscover.xml) succeeds, it will not go for other queries such as https autodiscover domain query or http redirect method for autodiscover. generally, for pure cloud office 365 accounts, the https root domain query and https autodiscover domain query will be failed and continue to perform the http redirect method in which the autodiscover will be successfully and connect to mapi. so we can try manually editing the registry settings under the \autodiscover key to make the outlook clients use the http redirect method. please try the steps below and to see if you can connect to outlook client.
1. press the windows key + r to open a run dialog box. type regedit.exe and then press ok
2. locate the following path
hkey_current_user\software\microsoft\office\x.0\outlook\autodiscover
note: x.0 in the above registry path corresponds to the outlook version (16.0 = outlook 2016, 15.0 = outlook 2013, 14.0 = outlook 2010, 12.0 = outlook 2007).
3. create the following regkeys and it value. 
excludehttpredirect   0
excludehttpsautodiscoverdomain   1
excludehttpsrootdomain     1
excludescplookup   1
excludesrvrecord    1
 

and please note the root cause of the issue is that https autodiscover domain query (https://vvvxxxx.com.au:443/autodiscover/autodiscover.xml) gives you a response which should not happen for pure cloud office 365 accounts. to thoroughly resolve the issue, we recommend you fix your website so that it will not give a response to your office 365 accounts.
