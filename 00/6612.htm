lightseeker999 say:

In case it helps anybody else, this is how I managed to restore my missing Notebooks, (although it has not been possible to establish why they disappeared in the first place):

  1. Take the Mac offline to prevent any OneNote synchronisation until it is required.
  2. Use Time Machine to restore /Users/xxx/Library/Containers/com.microsoft.onenote.mac to the day before the Office 365 installation.
  3. Use Time Machine to restore the OneNote application to the one before the Office 365 installation
  4. Check that the offline notebooks are now present and working, (they were)
  5. Go back online (necessary to create new NoteBooks) and create new empty notebooks (into which you will copy the content of the offline notebooks)
  6. *Manually create (unfortunately*) in the new notebooks, the same section group and section structure that was present in each of the original notebooks.
  7. Copy the content from the offline notebooks into their new notebooks
  8. Wait for all the syncing to complete and then check OneDrive and OneNote online to ensure that the new notebooks exist and are fully populated
  9. Delete the original offline copies and close the new sy

    In conclusion, the situation proved to be recoverable, (although it took several hours to complete), but it should not be possible for the files to be lost in the first place!

    *I was being cautious and copying content rather than moving it - because I was working this process out from scratch.  I learnt that if you move the content then you can move the original section groups without first creating them in the notebooks - section groups do not have an option to copy, but they do have one to move.

It's not an error or even a warning, it's only an information level event exactly as the portion I underlined above indicates.

If you'd actually complete reading the description, the item that's involved is a change of the ScansSinceLastRecap value from 15 to 16.  All this means is that the count of antimalware scans was incremented by one, thus tracking the fact that an additional scan has occurred since the last Recap, whatever that actually means.

So nothing's wrong, only the fact that as with all consumer users, you shouldn't be digging around in the Event viewer without appropriate training, it can only get you into trouble.
