Multi Level List won't continue when I press enter - Microsoft Community

s eham say:

Multi Level List won't continue when I press enter

This is actually for Microsoft Word 2011. On a numbered list, like I, A, 1, a, i, (a), ... , I'll type something in, and when i press enter the list won't continue. Like it won't go from I to II, it will just go all the way to the left with nothing before it. How do I make it so it continues when I press enter?
suzanne s. barnhill say:

Since you appear to have a Mac version of Word, I will move your post to the Word for Mac forum. But if this is the same in MacWord as in WinWord, then the problem is this:

If you have selected one of the list styles that is linked to headings (the gallery preview shows I. Heading 1, A. Heading 2, 1. Heading 3), then your numbered items are all in Word's built-in heading styles. The "Style for following paragraph" for all heading styles, by default, is Normal, so when you press Enter, you get a paragraph in Normal style instead of a new paragraph in the same heading style.

There are two workarounds for this, and which you use will depend on the purpose of your outline:

  1. If you are creating an outline for planning purposes and will later be adding text below the numbered headings, then you can work in Outline view. In that view, when you press Enter, you get a new paragraph in the same style, and you can use the Demote and Promote buttons on the Outlining tab to change outline levels.
  2. If the outline will be your entire document, with no intervening body text, then you could change the following style for each heading style to the same style instead of Normal, but this is not a good idea because all the built-in heading styles are also formatted as "Keep with next," which means that they will all try to stick together in a clump, and your document will be unstable (and if any Body Text-level paragraphs do creep in, they will cause page breaks where you least expect them).

    Instead, with the insertion point in the Heading 1 paragraph numbered I, use the Define New Multilevel List dialog to change the styles to which your numbering is linked. I would advise linking Level 1 to List Number, Level 2 to List Number 2, Level 3 to List Number 3, and so on. The built-in List Number styles go only to List Number 5, so if you will need more levels, you'll need to create additional styles. The indents and tabs you have set in the Define New Multilevel List dialog will remain the same regardless of what paragraph style each level is linked to, but you can modify the paragraph styles, if desired, to change the line spacing or other characteristics.

ken sheridan say: There is no need to insert a new row into tblEnquiry at all before opening the frmEnquiry form, and it is not good practice to do so as it requires Null to be permissible in all non-key columns in tblEnquiry, which undermines the integrity of the database.

You simply need to open frmEnquiry  at a new record and pass the current CustomerID value to it via the OpenArgs mechanism.  In frmEnquiry's Open event procedure set the DefaultValue property of a control (which can be hidden if desired) bound to the foreign key CustomerID column to the value passed into the form as its OpenArgs property.   So the code to open the frmEnquiry form would be:

    DoCmd.OpenForm "frmEnquiry", DataMode:=acFormAdd, OpenArgs:=(Me.CustomerID)

and the code for frmEnquiry's Open event procedure would be:

    If Not IsNull(Me.OpenArgs) Then
        Me.Event.SetFocus
        Me.CustomerID.DefaultValue = """" & Me.OpenArgs & """"
   End If

Note that the DefaultValue is always a string expression, regardless of the data type of the column in question, so must be wrapped in literal quotes characters as above.

You'll find examples of this sort of thing in FormsDemo.zip in my public databases folder at:

It should not be possible to insert a new row into tblEnquiry with a Null CustomerID, for two reasons:
