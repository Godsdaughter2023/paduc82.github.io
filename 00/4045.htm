There is not the slightest need to use this code when you run a macro. Refer the cells in the different workbooks as you like and copy them as you like. Here is an example:

Sub Test()
  Dim Source As Range, Dest As Range
  Set Source = ThisWorkbook.Sheets("Sheet1").Range("A1:A2")
  Set Dest = Workbooks("File.xls").Sheets("qwe").Range("B2")
 
  'Usual copy
  Source.Copy Dest
 
  'PasteSpecial
  Source.Copy
  Dest.PasteSpecial xlPasteValues

  'This shows how to get the selection from an other workbook
  'Assume that ThisWorkbook is the active workbook
  Set Source = Workbooks("File1.xls").Windows(1).Selection
  Set Dest = ThisWorkbook.Sheets(1).Range("A1")
  Source.Copy Dest
End Sub

davidf2 say:

It is possible that the unacceptable font will be a default font from the original template and could be hard or impossible to find on the page. It may be the default font for new text boxes, or....

One way to possibly track it down is to first make Special Characters visible, which would include the paragraph symbol for blank lines. Either Ctrl+shift+y to toggle or go to Home tab > Paragraph group > click the Paragraph symbol. Now you can select the hidden symbols, blank lines, etc. on the page to see what the font is for that symbol.

There is no built-in way to see a list of the fonts used on the publication, but there is a workaround if you are really interested. Go to File > Save As > change the Save in location to your desktop or some other place where you can easily find the file(s) you are going to create. And change the Save As type to Web Page filtered. This will save your Publisher publication as a web page. You will get a *.htm file along with a *_files folder which contains all the supporting graphics for the web page. You can delete all these web files later.

Right click the *.htm file and open with Notepad. This shows the source code of the web page that you just produced. Scroll down until you see the font definitions and the list of font-faces. There is your list of fonts used.

Vnow wrote:

I have alot of times, all ranging from 4:00pm - 3:00am.  A few examples are 5:25pm, 7:00pm, 11:25pm, 12:00am, 1:30am, 2:25am.  They are all listed in a column in excel, however when I run the formula: =AVERAGE(d2:d50) the final average is 12:40 PM (or something around lunchtime). -----

Generally, the solution is nondeterministic.

You probably think the average should be some time between 5:25 PM and 2:25 AM in your example because you think of the AM start times as

But we could just as easily think of the AM start times as before the normal PM times.

More to the point, if the normal start times were any time between 10:00 AM and 10:00 PM, for example, we would not know how to interpret 4:00 AM.  Is that before 10:00 AM; or is that after 10:00 PM?

If you wish to consider all AM times to be after the PM times, the average can be calculated with the following

=MOD(AVERAGE(D2:D50 + (D2:D50 < TIME(12,0,0))), 1)

The formula should appear surrounded by curly braces { ... } in the Formula Bar.

The "+(...)" expression causes AM times to be treated as times in the next day.

The MOD function ensures that the result is a time of day, in case it is an AM time (which would otherwise be 1 + time of day).
