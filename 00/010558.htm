Raw Query for Message Contents - Microsoft Community

brendan leahy say:

Raw Query for Message Contents

I've been playing around with search folders in Outlook 2011 on mac. Often my emails contain one of two things I want to search for, the job number or job title. I'd like to know if there is a way to search the subject or message contents for things like "#1234" OR "Widgets Project" and have them show up in a search folder.
talking moose say:
Use this syntax:

 kMDItemTitle == *Credit* || kMDItemTitle == *Bill*

To search in the subject lines of messages you want to use kMDItemTitle. You cannot use Raw Query to search message contents.

In the example above this will find any subject containing "Credit" or "Bill". You can remove the asterisk wild card characters to define an exact match. The two pipe characters ( || ) denote "or". Likewise, you could also use two ampersand characters ( && ) to denote "and".

Raw queries are case sensitive so "credit" is not the same as "Credit".

Be alert that when you search using a Raw Query in Outlook for Mac that you're only searching the current folder. If you want to search all folders then you must click the "All Mail" or "All Items" buttons in the ribbon to expand your search.

Hope this helps!

Unfortunately, once you've used the Document Inspector to remove the other reviewer names, the document won't keep your name on your changes -- it will be changed to "Author" when you save, just like the others.

  • Test formula ......... =IF($A3="","",IFERROR(SUMPRODUCT(--('2014'!$I$2:$I$400000=$B3)*('2014'!$M$2:$M$400000)),"--"))

SUMIF formula takes 25+ minutes to calculate "1" WS ... WB takes 90+ minutes to calculate (140mg)

joeu2004 say:

Whenever you can use SUMIF or SUMIFS, that is better than an equivalent SUMPRODUCT formula.

First, your SUMPRODUCT formula can be written more efficiently as follows:

SUMPRODUCT(('2014'!$I$2:$I$400000=$B3)*'2014'!$M$2:$M$400000)

or

SUMPRODUCT(--('2014'!$I$2:$I$400000=$B3), '2014'!$M$2:$M$400000)

The latter is best.  The first alternative demonstrates that the double negation ("--") is unnecessary if you perform some other arithmetic operation.

Second and more importantly, the SUMPRODUCT formula requires the creation of at least 1 temporary array of about 400,000 elements each -- at least 3.2 MB of memory; probably much more.  And it requires Excel to perform the multiplication.

The SUMIF (or SUMIFS) formula should not require any temporary arrays; Excel should access the ranges directly.  And of course, there is no arithmetic operations to perform; just the comparison with B3.

PS....  Another benefit of SUMIF instead of SUMPRODUCT:  If you use the range M2:M400000 even when you have only 20K of data in contiguous rows, SUMIF might (probably does) recognize that there are only 20K rows to process; but SUMPRODUCT will process all 400K rows.
