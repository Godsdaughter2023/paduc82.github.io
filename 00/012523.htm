How to remove paragraph marks at the end of ma line? - Microsoft Community

roroco say:

How to remove paragraph marks at the end of ma line?

When importing a non-Word document into Word (2011), there is a paragraph mark/symbol at the end of EACH line. How can I remove the paragraph break?
daniel g. grau say:
You need to do a global replacement using Advanced Find and Replace with wildcards enabled.

If "real" paragraph breaks consist of two ¶ in a row:

    Text text text text¶
    text text text.¶
    ¶
    Text text text …

Find: ^13([!^13])
Replace: SPACE\1

The above replaces all Carriage Returns by a space (you have to type a space, of course), as long as the next character is not a Carriage Return.

If paragraphs are separated by a single ¶:

    Text text text text¶
    text text text.¶
    
    Text text text text¶
    text text text.¶

Find: ^13([a-z])
Replace: SPACE\1

The above replaces all Carriage Returns by a space, but only if it is followed by a lower case character. This replacement will catch most instances of spurious paragraph breaks, but there will be exceptions (for instance, numbers) too numerous to consider here.

Undo in Excel - Microsoft Community

davehome say:

Undo in Excel

Hi,

A friend has told me that the reason the undo stack clears is because I run macros in one workbook is that correct, if yes then is there any way around it?

mike h.. say:

Hi,

Contrary to common belief running a macro doesn't clear the undo stack.

It's the action of the macro writing to a worksheet that clears the stack so if your macro doesn't write to a sheet then the stack is not touched.

So really it depends what your doing. in the past I have gotten around this by outputting my macro result to another workbok thus preserving the undo stack in the workbook with the macro in something like this

'lots of code
myresult = Range("A1") ^ 3 / Range("A2").Value
Workbooks("Results book.xlsx").Sheets("Sheet1").Range("A1").Value = myresult

EDIT. I should have added that back in the original workbook you can use a formula to refer to the cell you just wrote to to pull in the computed value. something like

='[results book.xlsx]Sheet1'!$A$1

If this post answers your question, please mark it as the Answer.

Saving Auto Complete Data

My organization is slowly upgrading our PCs to Windows 7 from Windows XP. I have a particular user that would like to keep her stored auto complete data in Outlook 2007 as a matter of convenience. I.E. when she starts typing an email address, it automatically fills in the entire thing. We will be keeping the same version of Office. Is there any way to save this information and import it into the new computer? Thank you in advance for the help.

deepa raj say:

Hi,

Thanks for posting in Microsoft Community. I shall certainly help you with the query you have.

 

Yes there is a way to back up the .nk2 file and then access the same once the Computer is upgraded to newer version.

 

You need to go to the following location and copy the .nk2 folder to a flash drive or a external drive.

 

Location: drive:\Documents and Settings\<user>\Application Data\Microsoft\Outlook

 

Note: Depending on your operating system (Windows 7, Windows Vista and Windows XP the .nk2 folder might be hidden. To view the files in this folder, do one of the following:

  • Click Start, and then click My Computer.
  • On the Tools menu, click Folder Options.
  • Click the View tab, and then, under Advanced settings, under Hidden files and folders, click Show hidden files and folders.

 

Once the Computer is upgraded, you may go to the same location and replace the .nk2 file.

 

It is little different in case if you have Outlook 2010 installed after the upgrade, you may follow the steps from the article:

http://support.microsoft.com/kb/980542

 

 

I hope the above steps helps. If you need any further assistance reply and we will be glad to assist you.

Thanks.
