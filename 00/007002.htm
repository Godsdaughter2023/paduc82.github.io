ken sheridan say: Are you saying that your table has multiple Boolean (Yes/No) columns, one for each item?  If so your table design is wrong as it is 'encoding data as column headings'.  A fundamental principle of the database relational model is the Information Principle.  This requires that all data be stored as values at column positions in rows in tables, and in no other way.

The table should be broken down into three related tables.  Let's assume for this example that you are a fruit retailer with several stores and wish to record which stores stock which fruits.  The tables would be like this:

Stores
....StoreID (PK)
....StoreLocation
....etc

Fruits
....FruitID (PK)
....Fruit

There is a many-to-many relationship type between these two tables which is modelled by a third table:

FruitStocked
....StoreID (FK)
....FruitID (FK)

The primary key of this column is a composite one made up of both columns - this is important!

For data entry you'd have a Stores form in single form view, and within it a FruitStocked subform, in continuous forms view.  The subform is linked to the parent form on StoreID, and contains just one control, a combo box set up as follows:

ControlSource:    FruitID

RowSource:     SELECT FruitID, Fruit FROM Fruits ORDER BY FruitID;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column.

To enter multiple fruits per store is simply a case of inserting multiple rows into the subform, selecting a different fruit from the combo box in each case.

You'll find an example of this basis form/subform set-up as ParentActivities.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

As an additional feature, note how a new activity not currently in the list can be inserted into the Activities table by typing its name into the combo box in the activities subform on the parents form - try entering Hockey to see how it works.  The new record is inserted by means of code in the combo box's NotInList event procedure.

It would be possible, with this model, to set up a form in the way you describe, with a set of check boxes from which the user can select the relevant items, but (a) the form would need to be unbound and you would need to write code for its module to read/write the data from/to the tables, and (b)  if you should add or delete items from the fruits table the form would need to be redesigned. Using the conventional method of a form/subform is completely data-driven, however, so whatever changes to the data are made are automatically reflected in the form.  If you really want to explore the use of an unbound form you'll find an example as VolunteerRota.zip in my SkyDrive folder to which I referred you above.  This illustrates both approaches, but the use of an unbound form in this case relies in the fact that a fixed number of items (time slots in a day) are involved, so are unlikely to change.

bill mosca say:

Roger

Set up the import to be fixed width instead of delimited. Then set your break lines. be sure to treat each delimiter as a field and mark it to be skipped.

Let's say you lines look like this using commas as delimiters.

123,    Bill,   Mosca

 

I'll use pipes to indicate break positions.

123|,|    Bill|,|   Mosca

When you get to the spot in the import wizard where you can set the data type and such check the box to skip importing all the "fields" with your delimiter.

Let's spend a moment on security in general.

Here's what I do (and it isn't 100% but nothing is, and in some cases only catches things after they've already gotten into the system, but it's a good process while remaining reasonable and they can all work together - as proven by the fact that I use them):

 

I use MSE for real-time protection and run scheduled daily quick scans with a scheduled bi-weekly full scan.

 

I update and periodically (every couple of weeks or so) run full scans with Malwarebytes:http://www.malwarebytes.org/and SuperAntiSpyware: http://superantispyware.com/ and ESET online scanner:http://www.eset.com/us/online-scanner Get only the free versions. Any that also have paid versions include features like real-time protection that cannot be used along with MSE anyway (in these cases), so there's no point in spending the money anyway.

I use Secunia PSI http://secunia.com/vulnerability_scanning/personal/ to make sure all my programs are updated and current (since updates often plug security holes in the software and thus can be important as many people never check and/or update their software unless prompted - and this way I don't need to check everything since notifications and sometimes updates are automatic).

 

I always follow the following advice from Stephen Boots - MVP:

 

-Make sure that the Windows Firewall is enabled. No other firewall truly offers better protection or is needed. This one is perfectly fine.

 

-Make sure that all important/critical updates, including service packs for the operating system and programs are installed from Microsoft Update (Windows Update).

 

-Make sure Internet Explorer is at version 8 or 9 and updated with all patches.

 

-In Internet Explorer 8 or 9, use the SmartScreen Filter.

 

-Make sure that IE Internet Security settings are at least set to medium-high (default).

-Make sure that installed applications, especially Adobe Acrobat, Adobe Flash, and Java are at their latest versions. Many vendors are regularly updating and patching for security holes.Secunia PSI mentioned above can help automate this a lot.

 

-Never click through links from unknown sources and use caution even if they are from a "trusted" source.

 

-Never open unsolicited email attachments.

 

-Practice safe web browsing. -Always use a standard user account unless you have a specific need for an administrator account. Then switch, do the administrative task, and return to the standard account. This greatly limits the access of any type of infection of intrusion into your system since administrative access isn’t in place.

silverwind---01 say:

The specs sheet is correct, actually. .

 

The Camera Sensor resolution (of 8.7 MPs) refers to the maximum width and height of imagery that it senses. . This comes from the Width of 16:9 (3552 x 2000) multiplied by the Height of 4:3 (3264 x 2448). .

 

If yuh run the numbers, it works out something like:

 

3552 x 2448 = 8,695,296

which is roughly 8.7 Mega Pixels. .

 

Anyway, the bragging rights are in being able to capture in 'true' 16:9 ratios, instead of capturing in 4:3 and cropping to 16:9, like other smartphones.

 

Hope it helps to know =)
