Function GetMatch(cel As Range, rng As Range) As String
    Dim f As Boolean
    Dim r As Variant
    Dim cll As Range
    
    f = False
    For Each cll In rng
      If InStr(1, cel, cll, vbTextCompare) Then
         f = True
         r = WorksheetFunction.VLookup(cll, rng, 2, False)
         Exit For
      End If
    Next cll
    If Not f Then r = "<KEYWORD NOT MATCHED>"
    GetMatch = r

dale brandt say:

I have solved the IE8 problem !!!

Check this out...

I had tried doing the fixes to IE8 (Windows 7, 64-bit) but was not having any luck.

The patch previously discussed using ICACLS did not work.

See the following articles: the second one is more detailed.

http://support.microsoft.com/kb/2000843

http://support.microsoft.com/kb/973479


I continued to search the web, and found the following extremely good article.

"Printing Problems with Internet Explorer 8: Known Issues and Troubleshooting."

Here's the web-site:

http://www.brighthub.com/computing/windows-platform/articles/43499.aspx

It discusses two main topics:

1)  "Incompatible HP Smart Web Printing Plug-In"
2)  "Nothing Happens When You Try Printing".


Both of these are worth reading.

The ICALCS fix is the subject of the second article, but it did not solve my problem.
It was the first article that helped lead me to the fix.

One key issue is that IE8 was not working, but that Firefox was. Hmmm ?

I upgraded Firefox to the latest version, and during the install process noticed
the subdirectory to which it was being saved:

"C:\Program Files (x86)\Mozilla Firefox\firefox.exe"

This laptop runs Windows 7, 64-bit Operating System, but it turns out that Microsoft
provides both, a 32-bit and a 64-bit version of IE8, and that the 32-bit version of IE8
is stored in the same subdirectory as Firefox:  "C:\Program Files (x86)\"

"C:\Program Files (x86)\Internet Explorer\iexplore.exe"

Firefox is not a 64-bit application !
If you check its web page, Firefox does not come in a 64-bit version (yet).

So, I created a Desktop shortcut to the 32-bit version of IE8, and tried it.
I could print emails and web pages using the 32-bit version of IE8 !!! ;
but I could not print them using the 64-bit version of IE8.  Why ?

This computer system had been bought new and was working fine until after about 4-5 months.
The computer was an HP G60-533CL Notebook, and the printer was an  HP Photosmart Premium C309g-m
All-In-One (AIO) printer/scanner/copier...both, running wirelessly.

Topic 1) mentioned in the article said that IE8 had problems working with HP Smart Web Printing,
and suggested that you uninstall the plug-in.  Doing this did not help my problem. 
The only other printer software installed on the laptop was version 13 of the HP printer driver Suite.
I uninstalled the printer driver suite (ver 13), then downloaded and installed version 14 of the
printer driver suite (which includes the HP Smart Web Printing Plug-in).  This solved the problem.
I am now able to print email and web pages using IE8 (64-bit).

What triggered this IE8 printing problem ?
I do not really know.  IE8 (64-bit) was working fine, printing email and web pages for 4-5 months.
Then it crapped-out.  HP published its upgraded printer driver suite in late Mar 2010, but at that
time, my laptop was working fine with the previous version (ver 13) of the HP printer driver suite.
I think that the problem may have been triggered when Windows 7 does automatic upgrades to its software,
(but I can't be sure of it) since no other HP printer software updates had been made.
