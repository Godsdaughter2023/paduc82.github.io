Found a solution to part of my problem.  Thought I'd post it in case others are struggling with the same issue.

 

 

Issue 2: Getlink function couldn't pull a named range within the same wkbk but could pull external docs via URLs.

 

Formula in cell stays the same but changed getlink to add a "#" and & before the range if it was a range and not a URL.

 

Function getlink(HyperlinkCell As Range)    
 '  pulls named ranges "P_" in wkbk & external docs  
             
If HyperlinkCell.Hyperlinks.Count > 0 Then    
    If Left(HyperlinkCell.Hyperlinks(1).Name, 2) = "P_" Then  
      getlink = "#" & HyperlinkCell.Hyperlinks(1).Range   
             
    ElseIf Left(HyperlinkCell.Hyperlinks(1).Name, 2) <> "P_" Then
      getlink = HyperlinkCell.Hyperlinks(1).Address     
    Else            
      getlink = ""          
    End If            
End If            
End Function

dale howard [mvp] say:

Andreas_747 --

Have you considered using Expense Cost resources for this purpose?  If you have more than one category of costs incurred by tasks in the project, you can create one Expense Cost resource for each of those types of costs.  You can then assign an Expense Cost resource to each task in the project.  After assigning the Cost resource to the task, you can then enter the expected cost in the Cost field for the Cost resource's assignment.  As you status the project, you can then enter the actual costs in the Actual Cost column for the Cost resource's assignment as well.  Do not forget to save a Baseline in the project before you begin entering progress, which will allow you to analyze cost variance along the way.  Just a thought.  Hope this helps.

diane poremsky mvp (slipstick.com) say:

Was your account migrated to the new server? If you log into Outlook.com and see "Outlook Mail (Preview)", it is on the new server.


When you use an outside address as a Microsoft account and log into outlook.com, mail sent from that account will use the outlook_....@ address unless it is configured to send and receive mail - then messages you send are forwarded to the account's actual smtp server.


If still on the old server: Go to Gear icon, Options, Your email accounts.  does it says Send and Receive or Send only? Click the Details link - what does it say?  Send only accounts will say this:


alias @ gmail.com
Recipients of email from this account may see "From outlook_...@outlook.com on behalf of alias @ gmail.com"
To send and receive email from this address in Outlook.com, add your account as send-and-receive.

If the account was moved to the new server, the non-Microsoft address needs to be set up as a connected account so mail can be forwarded to the correct SMTP server.
