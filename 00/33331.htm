Hi,

Thanks for the update on the issue.

Did you try to give permission to GWX files?

This could be because the Windows Module Installer service is on and running.

Method 1: Disable Automatic Update

  1. Press Windows + X on the keyboard.
  2. Select Control Panel.
  3. Go to System and Maintenance.
  4. Click Turn automatic update on or off option.
  5. Next, depending on your preference choose one of the following options:
    • Never check for updates (not recommended)
    • Check for updates but let me choose whether to download and install them
    • Download updates but let me choose whether to install them
  6. Click OK to save your changes and exit the dialog box.

Method 2: Stop the Windows Module Installer That Runs Trustedinstaller.exe

  1. Click on Start, in the Start Search box, type msconfig and then press Enter.
  2. Click Continue when User Account Control prompt is displayed.
  3. In the System Configuration window that is displayed, open the Services tab.
  4. Locate and clear the Windows Module Installer check box.
  5. Click OK save your changes and exit the dialog box.
  6. Next, open Services window again and change the Startup type of Windows Module Installer to Manual.

rajesh govind say:

Hi Robert,

I apologize for the delay in responding to this thread.

Now, regarding your queries, let me explain you why you need many different versions of Microsoft Visual C++ on the computer.

The Microsoft Visual C++ redistributables are components used by the programs (games, Microsoft software etc) developed using Visual C++. Most programs bind to the most recent update of the same major version, but some programs bind to one particular version of Visual C++ redistributable and don't accept any other versions. When you install an application on the computer, it may also install a particular version Microsoft Visual C++ redistributable package with which it is bind to. Thus, when you uninstall that particular version of Visual C++ then the application that you have installed may not work properly.

So, please do not uninstall any versions of Visual C++ from the computer, as they may be bind to different applications that you have installed on the computer.

Since you have the 64 bit Operating System, you need the Microsoft Visual C++ 64 bit version. To use the SkyDrive, you may install the latest version of Microsoft Visual C++ 2012 on the computer and check if that resolves the issue.

Hope this is helpful. Please get back to us with the updates.
