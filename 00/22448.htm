andreas killer say:

Harlan Groove's pull function did not work????

Well, I don't which version you found, but we can reduce it to a very small one to get just one value, as you asked for.

The basic of the function is the call to ExecuteExcel4Macro with the cell reference as argument, this opens a DDE channel to the file and reads the value.

But ExecuteExcel4Macro needs the reference in R1C1 absolut style and can only read one value at a time. (That's why Harlan's function has much more code.)

If we assume that you pass the reference always in A1 style to the function we can use ConvertFormula to build the R1C1 style.

This works fine inside any macro, but not in a UDF, because many parts of the application are locked if the worksheet is calculated, means ExecuteExcel4Macro did not work inside a UDF.

The trick to get it to work anyway is to open Excel twice and call the ExecuteExcel4Macro of the other (invisible) instance.

Copy the UDF below into a normal module and the formula

=DDE("'\Daily Sales Sheets\Year Ending 2012\[Daily Sales  11-10-3.xlsx]Table'!E$15")

into a cell to get the value from the closed file. After you see that this works, build the string for DDE in the same way as for INDIRECT.

Andreas.


Function DDE(ByVal Reference As String) As Variant
  Static XL As Application
  On Error Resume Next
  If XL Is Nothing Then Set XL = CreateObject("Excel.Application")
  DDE = XL.ExecuteExcel4Macro(Application.ConvertFormula(Reference, xlA1, xlR1C1, True))
End Function

ken sheridan say:

The code will do what you want, but not what it says.  A Boolean column cannot be NULL, only TRUE or FALSE.  NULL is not a value, it's the absence of a value.  So the SQL statement

    UPDATE tblChecklist SET Verified = NULL;

doesn't make any sense.  However, it works because Access is sufficiently indulgent to correct the error and  update the Verified column to FALSE.  The statement should really be:

    UPDATE tblChecklist SET Verified = FALSE;

Other lines in the code are unnecessary:

1.  As the statement is a fixed string expression, why declare a variable, and then assign a value to the variable,  when you can simply assign the string expression to a constant?

2.  Why move focus to the Verified control? That has no effect on the UPDATE operation.

3.  As regards calling the Requery method, as I said earlier, that reloads the form's recordset, which has the effect of moving the record pointer to the first row in the sort order.  If you particularly want to do that, then you can leave it in, but again it has no effect on the UPDATE operation.

In short, you've accidentally achieved the main object, but would it not be better to understand how to do it correctly?

Finally, your statement directed to Scott, "This checklist is ONLY visual.  It will not collect and store data…." isn't true.  I know what you mean, but any value at any column position in any table in the database is a datum.  The fact that the values are not what we might term 'operational data' does not mean that no data is being stored.

You might feel I'm nit-picking, but to me it's axiomatic that an understanding of what is being done is as essential to relational database development, as it is to the fundamental physics to which Richard Feynman was referring in his words to his students quoted below my signature.

Is there a way to convert a 4:3 formatted PowerPoint presentation to a 16:9 format without distorting the photos?

steve rindsberg say:

One trick:

Open the 4:3 presentation

Save it as a new file.

Change the slide setup of the new file to 16:9.  Everything will get distorted as you've noted.  Dont' worry about it.

Now open the two presentations side by side.

Go page by page through each and do this:

Click a slide in the 16:9 pressie, press Ctrl+A then Del to remove all the content from the slide

Go to the same page on the 4:3 pressie, Ctrl+A then Ctrl+C to copy everything.

Now back to the 16:9 and Ctrl+V to paste.

steve rindsberg say:
Girish's suggestions are very good.  In addition, if it's a PPTX file rather than a PPT, you might want to try extracting the audio to play it in some other program.

Make a copy of the PPTX
Rename the copy to give it a .ZIP extension
Doubleclick the renamed file; it should open as though it were a file folder, containing other files/folders.
Navigate to the Media folder.

Drag the sound file out of the Media folder and onto your desktop.
Doubleclick it.
