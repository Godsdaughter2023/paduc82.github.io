How Can I generate a serial number field in a SELECT QUERY? - Microsoft Community

faradaynwonu say:

How Can I generate a serial number field in a SELECT QUERY?

Can anybody help me?

I created a SELECT QUERY from a UNION QUERY which does not not have a serial number field. How can I generate a serial number field in the SELECT QUERY so that I can use the serial number values in FORMS ans REPORTS? I have tried to use the DCOUNT function but it is unimaginably slow.

 

Please I need your help for this problems as soon as possible

ken sheridan say: I don't think you'll be able to improve performance in a query; the overheads of the UNION operation, which presumably returns a significantly large result table, and the JOIN of two instances of the result table are too great.  Hans's solution of using the RunningSum property of an unbound control in a report is the best approach, but there is no means of doing that in a form.

ken sheridan say: You can join two instances of the UNION query. The demo file Balances.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

does this to compute balances from a UNION of separate Credits and Debits tables.  If you amend the SQL statement for the qryRunningSum_Credit_Debit_Union query as follows:

SELECT Q1.CustomerID, Q1.TransactionDate, Q1.Credit, Q1.Debit, COUNT(*) AS SerialNumber
FROM qryCustomerTransactions AS Q1 INNER JOIN qryCustomerTransactions AS Q2
ON (Q2.TransactionDate<=Q1.TransactionDate) AND (Q2.TransactionID<=Q1.TransactionID
OR Q2.TransactionDate<>Q1.TransactionDate)
GROUP BY Q1.CustomerID, Q1.TransactionDate, Q1.TransactionID, Q1.Credit, Q1.Debit
ORDER BY Q1.TransactionDate , Q1.TransactionID;

it will number each row in the result table sequentially in date order.  You may still find that performance is slow with large tables, however,  While this might provide a basis for a form, Hans's solution for a report is a far better one.
