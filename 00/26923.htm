vanderghast say:

The LastModified property still points to the latest SAVED (or updated) record. Since that record still points to the first one that you mentionned in your scenario, the second one has never been saved, after all, any macro/line of code which uses that property, LastModified, will get the bookmark of the first record in the scenario. See LastModified property in the help file.

 

Maybe the end user shoud try to undo the actual record with Esc key  (probably twice, once to undo any typing, if any, in the actual control, once more to undo the actual try to add a new record ), or trough  a command making use of a Me.Undo ?

 

I don't recommand to use DoCmd.Run acCmdSaveRecord since it tries to save the record on the form which has the focus and on relatively slow system, with many forms visible at a time, you may end up with the wrong one having the focus. Me.Dirty = False, in VBA, under a button click,  is safer. A little bit the same for Undoing on a form, I would rather use Me.Undo, under a button click.

brian tillman [outlook mvp 2007-2019] say:

Lets say you want to break a Contact Group in half.  First, open the Contacts folder, right-click the group, and drag it just a little way.  Release the mouse button and choose Copy.  You'll now have two identical groups both with the same name.  Let's say their names are "Finance".

Open the first Finance group and remove the second half of the member list by clicking the first member to remove, sliding down to the bottom of the member list, holding Shift, and clicking the last member in the list.  This will select all members between the first on you clicked and the last one.  Now click "Remove Member" to delete them.  Change the name of the group to, say, "Finance A-M".  Click Save & Close.

Now open the second Finance group and delete all the members you did not delete in the first one, leaving the members you did remove from the first one.  Name the group, say, "Finance N-Z" and click Save & Close.  You've now spit your group into two.  You can extend this method to three or more if you wish.

If you still want a group that will reach all the members of the original group, simply create a new group with the original name and add "Finance A-Z" and "Finance N-Z" as the two members.  You can include groups inside other groups.

Even though you have mentioned that you would not like to use manual filters, you may want to try using advanced filters.  Please note that if this solution suits you, we can, via VBA, automate advanced filters as well
 
1. Suppose your data is in B3:C12 (headings are in row 3);
2. In cell F3, type criteria;
3. In cell F4, enter =OR(LARGE($C$4:$C$12,{1,2,3})=C4)
4. Click on any blank cell and go to Data > Filter > Advance Filter > Copy to another location
5. In the list range, select B3:C12;
6. In the criteria range, select F3:F4;
7. In the copy to cell, select any blank cell and click on OK

Here's a possible formula solution.

 

Using your data and assuming 'Produce' is in cell A1 try this (I've developed this on the same sheet):

[IT IS IMPORTANT THAT YOU ENTER THE COLUMN HEADINGS]

 

Cell G1 enter 'Produce' and enter this formula in G2

=IF(COUNTIF($A$2:A2,A2)=1,A2,"")

Cell H1 enter ‘Weight’ and enter this formula in H2

=SUMIF(A$2:A$10,G2,D$2:D$10)

Cell I1 enter 'Produce' and enter this formula in I2

=INDEX($G$2:$G$10,MATCH(J2,$H$2:$H$10,0))

Cell J1 enter ‘Total Weight’ and enter this formula in J2

=LARGE(H$2:H$10,COUNTA(H$1:H2)-COUNT(H2))

Copy G2 thro’ J2 down

Hide columns G & H

Now you have a list of fruit in descending order.
