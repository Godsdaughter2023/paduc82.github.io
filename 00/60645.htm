azeez n say:
Hi Norton Victim,

a) What's your Operating System?
b) Which program file are you not able to copy?

As the file(s) are being copied, Windows checks that the data that has been copied is the same as the source files. The Cyclic Redundancy Check (CRC) error occurs when the copied data doesn't match the original.
 
This could be down to a read-error from the dvd, either the drive is going bad and has failed to 'read' the file properly, or the file has been placed on a bad sector on your hard disk. (Could be just down to a scratch or some marks on the dvd, maybe clean it if necessary and try again).
 
I'd run a check disk (CHKDSK) against the hard drive to rule that out, failing that though - have you got another dvd drive to try the disk from again? If so, try the dvd in another drive, or even a different machine altogether to try to pin-point the problem.
 
In the command prompt, type the following command and hit Enter:
 
chkdsk /r
 
Note: If any bad sectors are found while running the check disk utility and if attempt recovery of bad sectors is checked while checking the drive for errors, the data in the bad sector may be lost while attempting to recover bad sectors.

john_barnett say:

Hello FamilyGoodman,


As well as disabling the microphone by saying 'stop listening' you can also prevent the speech recognition engine from launching when Windows starts. To do this proceed as follows:


1. Right click on a vacant area of the taskbar at the bottom of your screen.

2. From the drop down menu, click on Task manager.

3. When task manager opens, click on the Start up tab.

4. Locate the Speech recognition and highlight it.

5. At the bottom of the task manager window you will see a button marked Disable. Press this button to prevent speech recognition from launching when Windows starts.

6. Close the Task manager window and then re-start your PC.

7. You should find that you no longer have speech recognition starting when you start windows.
