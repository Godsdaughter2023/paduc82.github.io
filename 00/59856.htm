bossdweeb say:

"What cause this problem..."

So the game was running well, until ? ...

"...I do some OC to my Ram"

The RAM OC would be my first guess. To successfully OC RAM you generally also need to OC the cpu.

If you OC the cpu you should have an after-market cooler as well (Not a bad idea to have a good after-market cpu cooler

when gaming anyway), or the cpu is likely to overheat under the load of the game.

- Using the XMP RAM option in BIOS is generally safe, but for guaranteed stability use the motherboard default settings, at least

to verify the RAM setting is not causing the game to crash.

- Next guess would be the gpu OC.

- Or it may be - "modding a game system file and adding some script to my game" ?

- Basically, anything you have done prior to the game no longer running properly should be undone before

any other possibility can be discovered.

.

bossdweeb say:

"maybe installation of Net Framework "

- Yes, but it's still best to start with a stable base, so we still need to eliminate the OC and other stuff from the equation, at least

until another culprit is discovered.

The RAM OC would still be a good contender for causing instability that could lead to a crash, and Windows errors are often misleading

because Windows tends to pop out an error in relation to what went down last in a chain of events, so the real source of the crash is not necessarily what is shown in the error.

- The modded files or script could be conflicting with .NET, so they also have potential.

- If you have a phone or tablet connected to your PC, I see this can also be involved in .Net-BroadcastEventWindow errors.
