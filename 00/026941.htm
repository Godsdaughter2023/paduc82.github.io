Base the Form ON THE QUERY.

You can improve it a bit by using parameter queries rather than literal strings. You can use a little unbound form  with textboxes FromDate and ToDate and a combo box cboCode displaying the valid options for tsbCode, and use criteria like

hansv mvp say:

yes i do

As others have remarked, you should not have a full name field in the table - it's the cause of your problem.

But if you absolutely want/need it, set the Control Source of the text box to the name of the full name field instead of to the formula =[EmployeeFirstName] & " " & [EmployeeLastName]

Create After Update event procedures for the text boxes bound to EmployeeFirstName and EmployeeLastName:

Private Sub EmployeeFirstName_AfterUpdate()
    Me.EmployeeFullName = Me.EmployeeFirstName & " " & EmployeeLastName
End Sub
 
Private Sub EmployeeLastName_AfterUpdate()
    Me.EmployeeFullName = Me.EmployeeFirstName & " " & EmployeeLastName
End Sub

where EmployeeFullName is the name of the full name field/text box.
