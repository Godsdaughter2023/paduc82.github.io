    steve rindsberg say:

    The monitor setting is stored in the registry.

    PowerPoint reads the registry settings at startup/as needed and writes anything you change via the UI back to the registry, then writes its current settings back to the registry at shutdown.

    This means that it will a) ignore anything that you change in the registry using VBA from within PPT and b) overwrite anything you've changed with its current settings.

    You could write a program in VB or some other language that makes the needed registry changes and then launches PowerPoint, but if you want different settings for different PowerPoint files, it might be a bit more complicated to use than you'd like.

    PPT stores the monitor setting in:

    HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\PowerPoint\Options

    The specific key under that is DisplayMonitor

    You should be able to run Regedit, watch that setting to get the correct values as you change settings in PPT and then refresh the view in RegEdit with F5.

    Another (probably mildly lunatic) approach would be to create several user profiles, one for each default monitor setting you want to use, then log in to the appropriate one before running the show.

    =IFERROR(IFERROR(INDEX($A$2:$A$999,MATCH(0,IF(LEN($A$2:$A$999),COUNTIF(F$1:F1,$A$2:$A$999),1),0)),INDEX($B$2:$B$999,MATCH(0,IF(LEN($B$2:$B$999),COUNTIF(F$1:F1,$B$2:$B$999),1),0))),"")
