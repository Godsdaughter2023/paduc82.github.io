wywyit42 say:

Hi,

This procedure appears to solve the issue.

It MUST be done Exactly by the Steps.

1. DISABLE the Microsoft Antimalware Service

    (To disable: Go to start, then run, and enter services.msc.  Find the Microsoft Antimalware Service in the list.  Change the startup type to disabled.)

2. REBOOT the machine

3. DELETE the EppOobe.etl File
 

Path to file: (Use start> run):

C:\ProgramData\Microsoft\Microsoft Security Client\Support\EppOobe.etl

4. REBOOT the machine

5. RE-ENABLE the Microsoft Antimalware Service (use step 1 above, set startup type to automatic.)

6. REBOOT the machine

7. Frequently CHECK the Administrative Events Log under Custom Views to ensure the error does not reappear.

NOTE: Just Deleting the EppOobe.etl file and Rebooting will NOT work.

From http://www.howtogeek.com/forum/topic/microsoft-security-client-oobe-stopped-due-to-the-following-error-0xc000000d
