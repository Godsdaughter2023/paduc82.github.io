I implement IMS SIP over IPSec according 3GPP standards.
They require manual keying and three stage configuration :
1. Allocate inbound SPI-s and send them to remote peer with other IPSec related stuff like algorithms, keys, ports etc.
   I do that by:
    1. FwpmFilterAdd0 (for FWPM_CALLOUT_IPSEC_INBOUND_TRANSPORT_V4)
    2. FwpmFilterAdd0 (for FWPM_CALLOUT_IPSEC_OUTBOUND_TRANSPORT_V4)
    3. IPsecSaContextCreate0
    4. IPsecSaContextGetSpi0
   At this point the remote port for protected traffic is not known yet,
   so it is not added to filter conditions.

2. Receive SPI-s from the remote peer and other IPSec agreed stuff and create SA-s:
1. IPsecSaContextAddInbound0
2. IPsecSaContextAddOutbound0

3. Update filtering with remote port for protected traffic:
1. IPsecSaContextGetById(..., &pSaContext)
2. pSaContext->inboundSa->traffic.remotePort  = remotePort
3. pSaContext->outboundSa->traffic.remotePort = remotePort
4. IPsecSaContextUpdate(engine, IPSEC_SA_DETAILS_UPDATE_TRAFFIC, pSaContext)

If I don't call the IPsecSaContextUpdate, leaving thus ANY port effectively, the packet is received successfully.
Wireshark shows that the packer was sent from port 'remotePort' indeed.

Solution for the IPMI Interface and PCI Serial Port Missing Drivers issue in Windows XP and Up

Get the ZIP File LAN_Intel_14.3_W7x86x64 from http://ulozto.net/
(i.e. http://ulozto.net/xNJp6hcy/lan-intel-14-3-w7x86x64-zip)

Since the IPMI Interface and the PCI Serial Port detected in the device manager under other devices are components of the Intel Network Adapter (namely PRO 10, Pro 100 or Pro 1000) the solution doesn’t consist of attempting to find a specific driver for each device in order to fix each separately; the solution to this issue consists of enabling the required services and installing the components that these two devices require to work as capabilities of the network adapter, which in fact consists of completing the installation of all the components of the Network Adapter; this is accomplished by first enabling SNMP Services and then Installing the SNMP Agent and later, installing the set of drivers and apps from Autorun.exe contained in the root folder of the File: "LAN_Intel_14.3_W7x86x64" that you can find at: http://ulozto.net/xNJp6hcy/lan-intel-14-3-w7x86x64-zip (i.e. the correct setup files for this task in Windows XP, Vista, Windows 7 and 8).

    http://ulozto.net/xNJp6hcy/lan-intel-14-3-w7x86x64-zip
  2. Install the Windows Component SNMP Services (from the Windows Programs Manager and later installing the SNMP Agent at the moment when the installation wizard prompts you to do so.
  3. Then extract the folder contained inside the download and Instal the drivers and apps by launching the intel’s network driver installation wizard from the file Autorun.exe that is found in the root folder of that download.
  4. Confirm that both devices, IPMI Interface and PCI Serial Port, in the device manager are gone; they actually will be re-assigned as part of the Network Adapter’s drivers.
