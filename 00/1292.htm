How to enable the Find feature in a Locked MS Word Form

jay freedman say:

While a form is locked, it isn't possible to position the cursor in any text that isn't inside a form field. Although a macro could search the protected text, it would not be possible to indicate where the found text is (except indirectly, something like a message box that says "page 8, line 15").

A macro like the following could search the text of just the form fields. (This is just sample code. I haven't taken the time to make it a full-featured search; for example, it always ignores letter case, it doesn't let you specify finding whole words only, and it highlights the entire form field in which the search text is found. A professional-grade code would also need much more error-trapping.)

Sub FindInForm()

    Dim fld As FormField
    Dim rg As Range
    Dim srch As String
    Dim resp As Integer

    If ActiveDocument.FormFields.Count = 0 Then
        MsgBox "There are no form fields in this document."
        Exit Sub
    End If

    srch = InputBox("Enter search word or phrase:", "Find in Form")

Restart:
    Set rg = ActiveDocument.Range
    rg.Start = Selection.Start
    For Each fld In rg.FormFields
        If InStr(LCase(fld.Result), LCase(srch)) Then
            fld.Select
            Exit Sub
        End If
    Next

    resp = MsgBox("'" & srch & "' not found. Seach again from start of document?", vbYesNo)
    If resp = vbYes Then
        ActiveDocument.FormFields(1).Select
        GoTo Restart
    End If
End Sub

A better solution, assuming your users all have Word 2007 or later, is to replace all the form fields with content controls, and select the whole document and use the Group > Group command on the Developer tab to "protect" the text outside the controls. Then Word's built-in Find feature will work on all the text, whether inside or outside the content controls.

Unless the dropdown has the focus it will only show ONE column.

You can solve this problem in a couple of ways.

If you are using a single form (not a continuous form with multiple records showing),

Add a control to the form and set its control source to

   =[NameOfDropdown].Column(3)

SELECT OwnerName, SomeField, SomeOtherField, DNR, OwnerName & " / " & DNR
FROM YourTable

Now hide all the columns except the last column.

john schmacker say:

RETURN THE ADDRESS FOR THE ACTIVE CELL

I'm looking for a function that will cause a cell to contain the address of the selected cell. So that, if the function is in cell A1 and cell J43 is selected, cell A1 will contain "J43". I need to do this without resorting to a macro. Any ideas?
jeeped say:
... I need to do this without resorting to a macro. Any ideas?

I'm afraid that Excel doesn't work that way. However, you can provide a custom Worksheet_SelectionChange routine that will easily do that for you. Right-click the name tab of the worksheet and select View Code. When the VBE opens, paste the following into the pane titled something like Book1 - Sheet1 (Code),

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Application.EnableEvents = False
    If Target.Count > 1 Then
        Range("A1").Value = Selection.Address(0, 0) & " - " & ActiveCell.Address(0, 0)
    Else
        Range("A1").Value = ActiveCell.Address(0, 0)
    End If
    Application.EnableEvents = True
End Sub

Alter the A1 cell reference if you want the active cell position elsewhere. Tap Alt+Q to return to your worksheet. Note that in the case of selecting a range of cells, I've notes the range and the active cell. The workbook will have to be saved as macro-enabled.

a. user say:

All right, I have determined that only the last bar changing was incidental to the data sets used. 

The real reason for the pattern changing is the data point value surpassing some maximal limit that the chart can handle. In one case it was exactly 198.8876, but with another data set (with less data points) it lies somewhere between 200 and 500, so it depends on the input. If there were multiple such points, all of them would have the wrong pattern. 

As I am limiting the value axis to a much lower number anyway, a good workaround for me is to manually adjust the problematic data point's value until it stops acting out.
