tasos_k say:

Hi,

assuming that source data is in Workbook1.xlsx, in Sheet1, in range A1:D100

and workbook1 path is in workbook2, in Sheet3, in cell A1

step1

Save As Workbook2 with extension .xlsm (macros enabled)

step2

in a regular module (in workbook2.xlsm) paste in the following macro:

Sub PT_ChangeSourceData()
For i = 1 To 2
Sheets(i).Activate
ActiveSheet.PivotTables(1).ChangePivotCache _
ActiveWorkbook.PivotCaches.Create(SourceType:=xlDatabase, SourceData:= _
Sheets("Sheet3").Range("A1").Value & "[Workbook1.xlsx]Sheet1!$A$1:$D$100", Version:=xlPivotTableVersion14)
Next
End Sub

if your data source table is dynamic

try this code...

Sub Dynamic_ChangeSourceData()
'Mar 09, 2015

Application.ScreenUpdating = False
Set wb = Workbooks.Open(Sheets("Sheet3").Range("A1").Value & "Workbook1.xlsx")
Dim rng
rng = wb.Sheets(1).Range("A1").CurrentRegion.Address
wb.Close False
For i = 1 To 2

ActiveWorkbook.PivotCaches.Create(SourceType:=xlDatabase, SourceData:= _
Sheets("Sheet3").Range("A1").Value & "[Workbook1.xlsx]Sheet1!" & rng, Version:=xlPivotTableVersion14)

Searching does not work in Outlook - Microsoft Community

diane poremsky mvp (slipstick.com) say:

Searching does not work in Outlook

Because it seems like Outlook 2016/2019/365 often has problems with search not working after an update is installed, fixed in another update only to be broken again in a future update, first check Fixes or workarounds for recent issues in Outlook for PC  to see if search issues were introduced by a recent update. 


A workaround for many search issues is using a different search scope.  

 
A temporary solution when changing the search scope doesn't help, is disabling Windows Desktop Search. When you disable Windows Desktop Search in Outlook, Outlook will fallback to its built-in search.  The built-in search is usually slow, but works. 

To disable Windows Desktop Search Service in Outlook:

  1. Right-click on Start, select Run.
  2. In the Open: box type regedit, and then click OK to open the registry editor.
  3. Go to this registry key: 
  4. HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows
  5. Click Edit > New > Key, enter Windows Search for the name.
  6. With the Windows Search key selected, click Edit > New > DWORD Value.
  7. Type or paste PreventIndexingOutlook as the name of the DWORD. Press Enter.
  8. Right-click on PreventIndexingOutlook and click Modify.
  9. In the Value data box, type 1 to enable the registry entry, and then click OK.
  10. Close the Registry Editor and restart Outlook.

To go back to using the Windows Desktop Search, either delete the Windows Search key you created or  disable the PreventIndexingOutlook setting by changing 1 to 0 (zero).

The way in which recent versions of Word (mis)handle the updating of links (willy-nilly) is really annoying.

A work around for your situation would be to have the following macro in the template

Sub FileClose()
'
' FileClose Macro
' Closes all of the windows of the active document
'
    WordBasic.FileClose
    ActiveDocument.AttachedTemplate.Saved = True
End Sub

As far as my research goes, there are two potential fixes that could work, however my version of Office doesn't have these on offer. bob jones aka: cybertaz say:

Word is a text oriented program... It doesn't have any concept of a 'page'. A page cannot exist unless there is at least one paragraph to justify its existence.

The most likely cause is that empty paragraphs are being used (pressing return twice) to create additional space between paragraphs. Avoid that by using the Paragraph Spacing Before/After attributes in addition to using Page Break Before rather than manual page breaks... Manual page breaks are the equivalent of taking an axe to the text flow.

By default Mac Word does not invoke Space Before when the paragraph carrying the attribute starts at the top of a page.

The Compatibility Preferences you're referring to do not exist in Word 2016.
