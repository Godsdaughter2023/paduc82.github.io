graham mayor say:

Use the replace function (CTRL+H) with the wildcard option set to replace

([0-9]{2}:[0-9]{2}:[0-9]{2}),([0-9]{3})

with

\1.\2

to convert the comma to a full stop (period)

and

([0-9]{2}:[0-9]{2}:[0-9]{2}).([0-9]{3})

with

\1,\2

to convert the full stop (period) to a comma.

See http://www.gmayor.com/replace_using_wildcards.htm

jay freedman say:
The way to do this (compatible across all versions of Word since Word 2000) is to create a userform (a custom dialog box) that is displayed by an AutoOpen and/or AutoNew macro.

First, some cautions about the limitations of this approach. Because malicious macros can behave like viruses, Microsoft has had to make it hard to distribute and use templates and documents that contain macro code. To complicate matters, the descriptions in the security section of the Options dialog are incomplete and confusing, especially for users with little experience. The default, and the usual outcome, is that opening a macro-containing document just gets you a terse message the "macros have been disabled". It can be helpful, but not a guarantee of success, to sign the macros with a security certificate.

If you can deal with this issue, here's what you need:

Go to http://word.mvps.org/FAQs/Userforms/CreateAUserform.htm. Follow the numbered steps, but with the following changes:
  • In Step 1, because you'll be distributing the form to Word 2003 users, you must save the template as a Word 97-2003 Template (*.dot) file rather than as a Word Macro-Enabled Template (*.dotm) file. The recipients will have to save the template into their User Templates folder (as defined in the Options dialog) and use the File > New command to create a document based on the template. (It's possible to store macros in documents, but that just makes the security hoo-hah that much worse.)
  • Skip Step 2.
  • In Step 4, in the Properties panel, click the Caption box and change the text in the title bar of the userform from UserForm1 to something informative.
  • Skip Steps 5 and 6.
  • In Step 7, after inserting a Label control, drag it as large as you need for the text you want to display. You can also drag the sides of the userform itself to enlarge it.
  • Between Steps 7 and 8, insert a Check Box control (by default, named CheckBox1). In the Properties panel, click in the Caption box and enter Do not show this again. Click in the Accelerator box and type D. That will underline the D in the caption and make Alt+D the accelerator key for the check box.
  • In Step 8, change the caption of the CommandButton1 control to Close. Then click in the Default box and change the value from False to True. This causes the Enter key to be the shortcut for clicking this button.
  • In Step 10, the only lines of code you should add between Private Sub CommandButton1_Click() and End Sub are
             System.ProfileString("MyForm", "ShowMyForm") = Me.CheckBox1.Value
             UserForm1.Hide
  • Instead of Step 13, add all of this between the lines Sub AutoNew() and End Sub:

             Dim checked As String
             checked = "False"
             On Error Resume Next ' in case the value isn't in the registry yet
             checked = System.ProfileString("MyForm", "ShowMyForm")
             If LCase(checked) = "false" Then
                 UserForm1.Show
             End If

  • If you want to be nice and let users change their minds about hiding the instructions permanently, you can put this macro into the module, following the AutoNew macro:

            Sub ResetInstructions()
                System.ProfileString("MyForm", "ShowMyForm") = "False"
            End Sub


          This macro can be run from the Macros dialog (Alt+F8).

It's how Microsoft processes inbound email.

If an inbound email is determined to be spam by Microsoft's servers then it is put into your Junk folder and all processing is stopped. Your Safe Senders, Blocks, Rules, etc. are not applied to that email.

Only when an inbound email is determined to not be spam and is put into your InBox do your Safe Senders, Blocks, Rules, etc. get applied.
