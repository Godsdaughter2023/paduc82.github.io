doug robbins - mvp office apps & services (word) say:

The issue is that when CTRL+F3 is used, you are actually cutting the selected text to the clipboard, and in the case of a document with Track Changes active, that is seen as deleting the text.  If you then subsequently reject the deletion, the text is re-instated, but there is no distinction between text that may previously have been deleted (and hence marked as a deletion) and that “deleted” by the use of CTRL+F3 with the result that there will no longer be any text in the document that is marked for deletion.

If, instead of using CTRL+F3, you use CTRL+c, the selected text is copied to the clipboard and can then be pasted into another document without any change and hence without a need to reject deletions.

For more information on using the Spike, see: https://helpdeskgeek.com/office-tips/copy-multiple-text-items-in-word-using-spike/
