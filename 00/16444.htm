If you add the line

    Application.Volatile

at the beginning of the function code, Excel will recalculate the function whenever it performs any calculation. Simply changing a color won't trigger a recalculation - Excel doesn't work that way.

If you want to count conditionally formatted cells, it's better to count based on the condition behind the rule, e.g.

=COUNT($B$9:$DB$23)

counts the number of cells with a number value in the range.

lordalukard say:
Dear domainnameerror,

I am an independent advisor, and I am trying to help other users in Community with my experience in Microsoft products.

In order to close Microsoft Account please do the following:
1. Go to Close your account http://go.microsoft.com/fwlink/?LinkId=523898
2. When you're prompted to sign in to your account, double-check that it's the account you want to delete. If not, select Sign in with a different Microsoft account.
3. Check that the page shows the correct Microsoft account, and then click Next.
4. Read the list, and select the checkboxes to acknowledge you’ve read each item.
5. In the Select a reason drop-down list, choose the reason you're closing the account.
6. Select Mark account for closure.

When you tap or click the Mark account for closure button, Microsoft will wait 60 days before permanently deleting your Microsoft account in case you change your mind or need to access something on the account before it’s gone forever. After 60 days there is no way to retrieve any information.

For more information and other tips please check this support article https://support.microsoft.com/en-gb/help/12412/... Please make sure that you read it carefully, especially part about Before closing the account.

If you are not receiving securoty code, please use tips in this support article https://support.microsoft.com/en-gb/help/12428/... the part about Troubleshooting security code issues.

If you still will not receive code, you can contact Microsoft Account support. At the bottom of the article you will find chat and email option.

Files from OneDrive are attached in emails as cloud attachments by - Microsoft Community

anahita [msft] say:

Files from OneDrive are attached in emails as cloud attachments by default

Hi everyone,

When you attach a file to an email from OneDrive for Business, you may notice that the file is attached as a cloud attachment by default and not a local copy.  

 

The setting for OneDrive files to be shared as cloud attachments by default has now been rolled back. In order for the change to take effect, please restart Outlook 2016, and you should go back to the original experience of seeing both a local copy of your OneDrive file in the Attachments menu, in addition to the cloud link, allowing you to select how you prefer sending your OneDrive file.

 

OneDrive cloud attachments are there to promote collaboration by allowing you to share a single version of the document, automatically permissioned to your recipients without going to OneDrive. In general, if you want to send a copy of the cloud file instead, you can select Attach as copy from the attachment menu. This option has always been available to you to choose.

We do appreciate the feedback and are invested in improving the overall experience of sharing OneDrive files. Expect to see further flexibility around setting default settings for OneDrive sharing in the upcoming year.

Regards,

Anahita
