plong.cityofboise.org say:

Migration to Office 365 failing

I don't know if this is the right forum to post this, but I have two users who are failing migration (on prem to Office 365) with the exact "Error: MigrationPermanentException: Cannot find a recipient that has mailbox GUID...." error.  I assume the GUID belongs to a resource that they both had full access to, but that mailbox no longer exists. Searching for the GUID, either with the Get-MailBoxStatistic command, or using LDP, both return a "doesn't exist" answer to the queries.  So, how can I find out what is actually causing these failures, or how can I have it ignore the error and continue with the migration?

Thanks.

plong.cityofboise.org say:

Thank you for your response, but I finally figured out the issue. During the migration, Office 365 seemed to assign an incorrect GUID to the user's mailbox (it assigned the exact same GUID to two people) as the Get-MigrationUser command returned the same GUID for both users.  That GUID did not match either one of the actual GUIDs on the on-prem Exchange system. So, I removed the users from syncing to the cloud (I moved them into an OU which we do not sync), forced a sync (rather than waiting for the system to do it automatically) so they'd show up in the deleted user list in Office 365. I then removed them from that list, moved them back into their correct OU and forced another O365 sync. I was then able to migrate both users with no issues.  The Get-MigrationUser command also returned the correct GUID for both users.
