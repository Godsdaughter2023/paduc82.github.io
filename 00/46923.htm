cmdrkeene say:
Hi Wayne,
Here's the scoop The tracing folder (showing in your Documents folder) is used by Windows Live programs -- usually Messenger -- for storing debugging and contact sync information.  By default, the Windows .NET system stores this debug information in the %userprofile% location, which is a variable that points to your Documents folder.  Due to an oversight, this default was left for Messenger and causes the folder to be stored there (a more appropriate location for system debugging information would probably not be mixed in with your document files!)

In short, it's an annoyance, but not harmful.  I believe the new versions of all the Windows Live applications have been updated to store their debug information away from your personal file folders.

Source: http://msdn.microsoft.com/en-us/library/bb878897.aspx


If you exit all Windows Live programs, you should be able to delete the folder.  Although it will come back.

what's a dialog box? - Microsoft Community

biscotti say:

what's a dialog box?

what's a dialog box? how is it opened and how is it closed?

a. user say:

It is generated when you activate some option in a program that opens a window with further options or controls (a dialog box).  For example, if you click here on Tools and then Internet Options it opens a Dialog Box for those functions.  It is opened whenever you click on an option that has further information or controls or settings - sometimes you can predict when one will occur but sometimes it occurs when you didn't expect it.  These dialog boxes are built into the programs (both Vista and other programs on your system).

You can close a dialog box either by clicking the red x in the upper right hand corner or clicking on the close or cancel button at the bottom of the dialog box if it has such buttons.

They are generally not controllable by the user except to open them and close them (or use the functions contained within them).

I hope this helps.

Good luck!

¡firedog say:

Setting up an account from scratch is different from changing settings for an existing one. The new att.net website seems to have lost all its support articles, so I'm not sure that these settings are the latest recommended ones, but they should work nevertheless.

Press Ctrl-Shift-T to start the Add your email accounts wizard. Enter your email address and password and your display name for outgoing messages, then select Manually configure server settings and click Next.

On the configuration page, make these entries:

 
  1. Select Incoming server type: POP
  2. Enter incoming server address: inbound.att.net
  3. Select Requires a secure connection (SSL).
  4. Enter incoming server port: 995
  5. Select Authenticate using> Clear text.
  6. In the Log-on username field, enter your full AT&T email address.
  7. Enter outgoing server address: outbound.att.net
  8. Select Requires a secure connection (SSL).
  9. Enter outgoing server port: 465
  10. Select Requires authentication.

Click Next and Finish. Remove any unsent messages from the outbox (below the account folders in the folder pane), then compose a new test message with a unique subject and send it to yourself. Post any error message in your reply. You can click on an error message to select it, then use right-click to copy it and paste it into the text entry box.
