Or, if you want a "crazy" but maybe good idea: How about "Virtualize Session"?

What should that be?

One of the main reasons why home-users use virtual PCs is, that you can try stuff out after making e.g. a "Snapshot". (That's how they call it in VirtualBox.) If all breaks down and a bluescreen appears, you can go back to that snapshot - and every last byte of the failed session is gone.

Simplified: Technically, that's done by reading from the virtual HD, writing all new stuff into something like an "incremental backup" and keeping a list of the blocks which have been re-written. At some point, you will then have to decide, whether you want to make this permanent (write the re-written blocks from the file to the HD) - or discard it. (Don't write anything, discard it, meaning nothing has happened at all.)

Now, that could not only be done with a VPC, but also with a real Windows. You would need a "Hypervisor", which temporarily virtualizes it, starts it and does roughly as written above. VirtualBox already has a "write-through" mode. So it can be done, although at the price of doubling at least the core of the OS.

Once that's in place, all you install, all you try out, all you surf, etc... it all goes to an "incremental backup" of your HD for now. (A file that is on that HD, or elsewhere, as you configure it.) Even if you reboot - all is redirected there. Only if you decide to "Finalize Session" or "Leave Virtual Mode", you will be asked to discard or finalize the changes.

The benefits of that?

(1) You would be able to thoroughly test new software in your REAL environment. A VPC is unlikely to have all your stuff installed - right? And it has "virtual hardware", which is well-known and unproblematic. So bad surprises might still follow, when you install something that worked flawlessly in the VPC on your real machine.

(2) Privacy. Start a "Virtual Session", copy whatever should remain of it to a USB-stick, then revoke the session and zero out the unused space of your SSD. And done you are. Nobody would find anything of that, ever.

(3) Likewise, a "Guest Session" could really be a traceless session, etc.

Though it's not my part to serve new ideas here, THERE you have one. But MS won't be able to keep their timelines for Win-10, if they started to work on that one - at least in this Windows version. So I don't expect to see that anytime soon. But it would have it's benefits.
