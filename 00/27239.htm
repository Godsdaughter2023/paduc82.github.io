doug robbins - mvp office apps & services (word) say:

While I am not sure that the text of the email will be exactly what you want, the following should otherwise work:

Option Explicit
Sub SendMails()
Dim i As Long
Dim strSubject As String
Dim strAddressee As String
Dim strBody As String
Dim bStarted As Boolean
Dim oOutlookApp As Object
Dim oItem As Object
strSubject = InputBox("Enter the subject for the e-mail messages.")
On Error Resume Next
Set oOutlookApp = GetObject(, "Outlook.Application")
If Err <> 0 Then
    Set oOutlookApp = CreateObject("Outlook.Application")
    bStarted = True
End If
On Error GoTo ErrMsg
With Sheets(1).Range("A1")
    For i = 1 To .CurrentRegion.Rows.Count - 1
        If .Offset(i, 10) = "Open" Then
            strAddressee = .Offset(i, 7)
            strBody = .Offset(i, 1) & " " & .Offset(i, 2) & " " & .Offset(i, 3) & " " & .Offset(i, 4) & " " & .Offset(i, 5)
            Set oItem = oOutlookApp.CreateItem(0)
            With oItem
                .To = strAddressee
                .Subject = strSubject
                .BodyFormat = 1
                .Body = strBody
                .Send
            End With
        End If
    Next i
End With
If bStarted Then
    oOutlookApp.Quit
End If
Set oOutlookApp = Nothing
    Exit Sub
ErrMsg:
    MsgBox Err.Number & vbCr & Err.Description
End Sub

scott mcdaniel say:

You should not have to move focus away from Access to fire a menu event of any sort.

Does this happen only on one database, or does it happen on ALL of your databases?

If it's just the one database, then you may have some corruption. To fix that, first make a backup then do this:

1. Compact the database

2. Compile your VBA code - from the VBA Editor click Debug - Compile. Continue doing this until the Compile menuitem is disabled.

3. Compact again.

You may also need to decompile the database. To do this, make a shortcut with this as the Target:

"full path to msaccess.exe" /decompile

Run that shortcut, then open your database. Access will decompile it when opening. After doing this, perform the 3 steps above again.

You may also need to create a new, blank database and import everything from the old db into the new one.

If this happens on ALL databases, then you may have a bad or corrupt installation of Office. Try a reinstall of your Office program. If that does not work, then an uninstall-reinstall might be in order.

Finally, be sure that your Office and Windows installations are fully up to date, with all relevant hot fixes applied.
