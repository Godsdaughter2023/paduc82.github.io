You would need to use a macro.  If you looking for exact matches, use this code as is; if you want partial matches, change xlWhole to xlPart.  The new sheet must be the first sheet in the workbook - that can change but for now that is how I have written the code.

1) Copy this code.
2) Right-Click the sheet tab of the first sheet
3) Select "View Code"
4) Paste the code into the window that appears.
5) Save the file as a macro-enabled .xlsm file.
6) Make changes as needed

Private Sub Worksheet_Change(ByVal Target As Range)

    Dim c As Range

    Dim iSh As Integer

    

    If Target.Cells.Count > 1 Then Exit Sub

    If Target.Value = "" Then Exit Sub

    If Intersect(Target, Range("A:A")) Is Nothing Then Exit Sub

    'Turn off events to keep out of loops

    Application.EnableEvents = False

    Target.Offset(0, 1).ClearContents

    For i = 2 To Worksheets.Count

        With Worksheets(i)

            Set c = .Cells.Find(Target.Value, lookat:=xlWhole)

            If Not c Is Nothing Then

                Target.Offset(0, 1).Value = Target.Offset(0, 1).Value & _

                    IIf(Target.Offset(0, 1).Value = "", "", ", ") & .Name

            End If

        End With

    Next i

    'Turn events back on to get ready for the next change

    Application.EnableEvents = True

End Sub

Hi HaidongJi,

From the error message "Invalid value for parameter.  Parameter Name: licenseOptions", we can see that the issue is related to parameter "licenseOptions".

Actually, if we want to assign license when creating user accounts with Office 365 PowerShell, we need to use "LicenseAssignment" parameter, so the correct command should be like this:

New-MsolUser -DisplayName "haha" -FirstName "f" -LastName "l" -UserPrincipalName "*** Email address is removed for privacy ***" -UsageLocation "US" -LicenseAssignment "myTenent:STANDARDWOFFPACK" -LicenseOptions $LO
