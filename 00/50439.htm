The latest version of Movie Maker can open .mswmm project files from the Windows Vista version of Movie Maker (i.e. WMM 6.0). So if that's where the .mswmm projects came from, just install Movie Maker 2012.

But if you have project files from an older version than that (e.g. WMM 2.1 from XP, or WMM 2.6 from Vista), they can't be directly opened in the latest Movie Maker. You would instead need to install Windows Movie Maker 6.0.

Hello there,

That issue here is that Windows 10 needs  CMPXCHG16b support (in both the CPU and chipset) in the 64-bit version. CMPXCHG16b allows for atomic memory exchanges. While modern 64-bit processors support CMPXCHG16b, some older hardware does not—but the requirement wasn’t a must-have for the 64-bit version of Windows 8.

Trying to upgrade to Windows 8.1 and to Windows 10 on newly non-compatible hardware results in failure.

Solution: Acquire a new computer that can run Windows 8.1 as it will also run Windows 10.

For more details on x86-64 and the CMPXCHG16b instruction, refer to the entry here: https://en.wikipedia.org/wiki/X86-64

More of the same has been detailed here: http://windows.microsoft.com/en-us/windows-8/system-requirements
