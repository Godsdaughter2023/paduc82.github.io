Try the following:

Dim word1 As String, word2 As String
Dim numwords As Long
Dim lrng As Long
Dim rng As Range
word1 = InputBox("Insert the first word.")
word2 = InputBox("Insert the second word.")
numwords = InputBox("Insert the number of words separting.")
Selection.HomeKey wdStory
Selection.Find.ClearFormatting
With Selection.Find
    Do While .Execute(FindText:=word1, MatchWildcards:=False, Forward:=True, Wrap:=wdFindStop) = True
        With Selection
            Set rng = .Range
            rng.End = ActiveDocument.Range.End
            lrng = InStr(rng, word2)
            If lrng > 0 Then
                rng.End = rng.Start + lrng
                If rng.Words.count < numwords Then
                    rng.Select
                    Selection.MoveEnd wdWord, 1
                    Exit Sub
                End If
            End If
            .Collapse wdCollapseEnd
        End With
    Loop
End With

If the second word is found within the specified number of words, the range starting with the first word and ending with the second word will be selected and the execution of the macro will stop.

Note, the count of words within the range treats punctuation as a separate word.

macropod say:

If you know the exact number of words, you could use a wildcard Find expression like:

Find = quick [! ]@ [! ]@ [! ]@ [! ]@ [! ]@ [! ]@ dog

but there's no wildcard Find expression you can use to define an intervening word count range. The other expressions you refer to appear to be RegEx expressions, which require VBA.

Word's wildcard Find expressions, though called regular expressions by MS, are different from those provided by the VBA RegEx library. Each has its strengths and weaknesses: VBA Regex is far more powerful in terms of the expressions it can formulate; Word's wildcard Find expressions can be used with other document characteristics, such as font attributes, etc.

scottgem say:

As noted, Access does not have  Concat function. The ampersand functions and a concatenation operator. If you are concatenated a first and last name you probably want to include a space;

SELECT Firstname & " " & Lastname AS Customer

hansv mvp say:

Access does not have a Concat function. You can use the & operator instead:

…
[Table].[A] & [Table].[B] AS AB,
