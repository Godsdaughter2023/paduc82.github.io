ken sheridan say: To check out multiple items to a student in one step you could use an unbound multi-select list box and, either in its AfterUpdate event procedure, or in the Click event procedure of a separate button step through the list box's items and assign or unassign each item to/from the student, depending on whether the item is selected or not, by executing an appropriate SQL statement.  The following is code which assign/unassigns students to/from courses in this way:

    Dim n As Integer
    Dim strCriteria As String
    Dim strSQL As String
    
    With Me.lstCourses
        For n = .ListCount - 1 To 0 Step -1
            strCriteria = "StudentID = " & Nz(Me.StudentID, 0) & " And CourseID = " & .ItemData(n)
            If .Selected(n) = False Then
                ' if item has been deselected then delete row from table
                If Not IsNull(DLookup("StudentID", "StudentCourses", strCriteria)) Then
                   strSQL = "DELETE * FROM StudentCourses WHERE " & strCriteria
                   CurrentDb.Execute strSQL, dbFailOnError
                End If
            Else
                ' if item has been selected then insert row into table
                 If IsNull(DLookup("StudentID", "StudentCourses", strCriteria)) Then
                   strSQL = "INSERT INTO StudentCourses (StudentID, CourseID, StatusID) " & _
                        "VALUES(" & Me.StudentID & "," & .ItemData(n) & ",1)"
                   CurrentDb.Execute strSQL, dbFailOnError
                End If
            End If
        Next n
    End With

In the form's Current event procedure you would read the data from the relevant table and select the assigned items in the list box.  The following is code form the same form's module which does this:

    Dim varItem As Variant
    Dim strCriteria As String
    Dim n As Integer
    
    ' synchronize go to student combo box
    ' with current record
    Me.cboGoToStudent = Me.StudentID
    
    With Me.lstCourses
        ' deselect all items in list box
        For n = .ListCount - 1 To 0 Step -1
            .Selected(n) = False
        Next n
        .Requery
        ' loop through list box items and select items where student assigned to course
        For n = .ListCount - 1 To 0 Step -1
            strCriteria = "StudentID = " & Nz(Me.StudentID, 0) & " And CourseID = " & .ItemData(n)
            .Selected(n) = Not IsNull(DLookup("StudentID", "StudentCourses", strCriteria))
        Next n
    End With

This code is taken from my StudentCourses.zip demo file which you'll find in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

As regards marking students as active/inactive see AciveEmployees.zip in the same OneDrive folder.  In this the Employees table includes a Boolean (Yes/No) column named Active.  When selecting an employee by means of a combo box only active employees are listed, except where the current value of the combo box is an inactive employee, in which case that employee is also listed.
