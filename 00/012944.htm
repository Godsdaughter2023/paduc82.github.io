mike middleton say:

For each value in the input column of the data table (or, in your case, each formula), the data table function copies that value to the "column input cell," recalculates the sheet, and puts the value of the data table formula (the formula in the top right cell of the data table area) in the adjacent column of the data table.

I think you may have a kind of circular reference during the data table calculations in that =H6+2% is copied to cell H6 before the recalculation and the data table may be using =H6+2%+1% or some other undesired value when it evaluates row 13. But I could be mistaken.

> are there any workarounds? <

Don't use a data table. You have a list of values in cells G11:G15 that depend on H6. In cell H11, enter =NPV(G11,$E$2:$E$42), and copy the formula to cells H12:15. When the value in H6 is changed, the values in G11:G15 change, and the NPV results in H11:H15 change accordingly.

patrick_beal say:

I have finally managed to solve my DESIGN tab scrolling problem on MS Word 2013.

 

I had tried deleting, renaming or moving many of the folders containing user or group templates, quickstyles, building blocks etc... including those in the Microsoft Office\Office15\1046 subfolder.

 

The file responsible for the scrolling however was a .DOTX template which I had saved to the (automatically suggested) following path:

 

D:\Users\####\AppData\Roaming\Microsoft\QuickStyles\$$$$.dotx

 

where #### is your username and $$$$ your given template name.


Adjust your drive letter accordingly as well.

 

If your problem has the same cause, it is easy to solve.

 

With MS Word running (not needed but makes it easier to identify the bad file), rename* the .dotx files on that folder one by one. After each file is renamed, alternate to MS Word and click on any other tab, then back on the DESIGN tab. When the scrolling stops you have found the culprit template. Leave it renamed and the problem goes away.

 

*For quick renaming just select the file, press F2 and delete the dot so the file becomes "filedotx" instead of "file.dotx". Make sure the folder view options are set to show all file extensions.

 

You may want to copy or export the macros, styles etc. you created on that template to another one.

 

My best guess for the cause is some bugged macro, quick part, building block or such -- one that worked fine in a previous version such as MS Word 2007 and was either incompatible or improperly imported to Word 2013.

 

Hope this helps.
