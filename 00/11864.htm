Sub Macro1()
With Selection.Range.Style
.NoSpaceBetweenParagraphsOfSameStyle = True
End With
End Sub

or you could change the paragraph format to remove space from after the current paragraph

Sub Macro2()
'unset with CTRL+Q
Selection.Range.Paragraphs(1).Range.ParagraphFormat.SpaceAfter = 0
End Sub

Just to clarify some definitions

#1 - The data in an OST (Offline Storage) file is a mirror of what exists on the IMAP server as of the last time you connected to the account\

#2 - You do not lose access to the data in the OST file <unless> you delete the email account from the profile at which point the OST file is no longer accessible which has nothing to do with whether or not you still have access to the IMAP account. A new OST file (or PST file in earlier Outlook versions) will get automatically created the next time you configure the account

#3 - If you want a complete backup of the OST file for a given point in time - just export the OST file to a PST file using the Outlook export wizard

#4 - The data that you will lose with an OL'2013 IMAP account if you delete and reconfigure it will be in the folders marked as "This Computer Only" (i.e. Contacts / Calendar etc). Those folders should definitely be backed up or better yet - a separate PST file should be designated as the "default data file" for the profile which will then contain the default contact/calendar folders

suzanne s. barnhill say:
This is addressed in http://wordfaqs.mvps.org/TOCTips.htm.
 
Leaving the right-aligned tab stop for the page number where it is, add a right indent to the TOC style (0.5"-0.75" should be sufficient). If you do this on the ruler, you'll probably find you have to drag the tab marker aside temporarily in order to grab the indent marker, then move the tab marker back.
 
To be even more picky, you'll find that this results in a slight change in the tab stop position. For reasons unknown, if your line length is, say, 6.5", Word (when you generate the TOC) sets the tab stop not at 6.5" but at 6.49". When you drag it back, it will be at 6.5". I've never known this to cause any problem, however.
