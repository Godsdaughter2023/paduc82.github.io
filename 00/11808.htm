Is it okay to use the same text for both modified Broad match and - Microsoft Community

ian y. say:

Is it okay to use the same text for both modified Broad match and Exact match?

If I enter air conditioner as both modified Broad match keyword and Exact match keyword (see below) for the same ad group, how will that impact my campaign?

+air +conditioner

[air conditioner]

And what if I enter them into two different ad groups, respectively?

microsoft advertising - shelly say:

Hi Ian Y.

Thank you for your questions. I would first like to address what a broad match modifier and exact match keyword entail:

Broad Match Modifier: This match type is a relatively new entrant and lies in between broad and phrase match. It gives more control than a broad match and more freedom than phrase match. For the ad to pull, the words that have the ‘+’ sign in front of them should be present in the search query. The order of the keywords doesn’t matter and close variants are accepted. Unlike broad match, the modified broad match won’t show your ad for synonyms or related searches which gives a higher level of control. Also, unlike phrase match, the order of the words in the search query is not important which gives more flexibility.

 

        Example: if your keyword is +air +conditioner it’ll match air conditioner, air fresh conditioner and buy an air conditioner. However, it will not match hair conditioner because the word air is missing from the search query.

Exact Match: A keyword setting that allows your ad to show only when someone searches for your keyword or close variants of your keyword.

Close variants include: misspellings (air conditionr), Singular or plural form (air conditioners), stemmings (air conditioning), abbreviations, accents, and reordered words with the same meaning.

Due to Board Match Modifiers flexibility and range, you will receive more impressions than an exact match keyword. If you add the same keywords in two different ad groups, with the same bid and ad setting, then you have two of the same keywords competing against each another. This will not be a strategy that will produce optimal results for your campaign.

I hope you find this information helpful! Please let me know if you have any additional questions as we are always happy to assist.

Thank you for choosing Bing!

If you mean windows server, Windows Server 2008 Service Pack 2 (SP2) or later or Windows Server 2008 R2 operating system prerequisites that are required for all Exchange 2010 server roles.

Question 2: Is Exchange 2010 supported?  If so, what is the minimum service pack?

The answer is yes. we recommend you use the Exchange 2010 SP3.

About the recommended hardware requirements for Exchange 2010 servers is vary depending on several factors, including the server role(s) that are installed and the anticipated load that will be placed on the servers.
1. Processor: x64 architecture-based computer with processors that supports 64-bit architecture
2. Memory:
a. Client Access/Hub Transport combined role (Client Access and Hub Transport server roles running on the same physical server): Minimum 4GB with a recommended maximum of 2GB per core (8GB minimum).
b. Multiple roles (combinations of Hub Transport, Client Access, and Mailbox server roles): Minimum 8GB with a recommended maximum of 4 GB plus 3-30 MB additional memory per mailbox.
3. File format: Disk partitions formatted as NTFS file systems.

Hybrid deployment prerequisites

Understanding Prerequisites for Exchange 2010 Hybrid Deployments

If you deploy a hybrid, you don’t need to set up ADFS.

Generally, if you want to use Exchange online mailboxes and on-premise mailboxes with the same domain at the same time, we recommend you use the Hybrid environment.

What Version of Office do you have, Recognize Office Menu vs 2007 vs 2010

Office 2003 applications are the last of the menu and toolbar editions.

When you click on a menu item, a drop down list of options appears below it.

For example, the Word 2003 menu has the following headings: File, Edit, View, Insert, Format, Tools, Table, Window Help

The exact version can be found by following this path: Help menu / About MS Office Word to display a dialog box eg. 11.8328.8333 SP3

 

Office 2007 is the first of the Ribbon “Gooey” (GUI)  editions.

For example, the Word Ribbon looks something like this. In the upper left corner of the application window is a (clickable) 4 color “Office” button. Clicking on the Office button drops down a list roughly equivalent to the 2003 File menu. As well the ribbon apps have a row of “Tabs” that on the surface appear much the same as the high level menu headings. Below the Tabs, there is a ribbon that extends the width of the app window displaying 1-3 rows of command icons. The commands displayed change as you click on the tabs (or scroll the mouse wheel). The layout of the commands changes as you horizontally extend or shrink the application window (great, “dancing buttons”). The Ribbon also depends on “Contextual” tabs that appear (an disappear) only when the relevant feature  is active or current.

For example the Word 2007 tabs are: Home, Insert, Page Layout, Reference, Mailings, Review, Developer.

You can get the exact version number information by following this path: Office Button / Word Options button / Resources option. At the bottom of the pane under About: eg. 12.0.6545.5000 SP2 MSO 12.0.6554.5001.

 

Office 2010 has only minor surface changes to the Ribbon.

The Office button has been replaced with a colored “File” tab. The color is coded by application: Blue for Word, Green for Excel, Red for PowerPoint etc. As well the User Interface color scheme has been made more wishy-washy. In 2007, the  individual buttons are clearly delineated, and the colors in general more contrasty than in 2010. In 2010, it is difficult to distinguish the separation between individual buttons and the group name bar.

For example, the Word 2010 tabs are: File, Home, Insert, Page Layout, Reference, Mailings, Review, View Developer.

You can get the exact version number information following this path: File tab / Help command. Look on the right side of the “Backstage Pane” to find the detailed version number, eg 14.0.6024.1000 (32-bit).

 

“Office 7” and “Office 10”

Some users find saying “Office 2007” or “Office 2010”, or the apps in the Office bundle, too long, so they shorten the names to “Office 7”, “Word 7”, “Office 10”, “Excel 10”.   Sorry Charlie, but that is confusing for us poor technicians. The MS marketing department has done a royal job of sowing confusion by “playing games” with version names.  Programmers and technicians use version numbers to identify the level of development of software. Version “2” is newer and “better” than Version “1”. When doing installations it is always advisable to install version 1 before version 2 etc. 

Knowing the correct version number is important to us problems and fixes are often version specific. Just think of the difference in the instructions for opening a file in Menu based Office 2003 vs. Ribbon based Office 2007.

·         Office Version 7 had the marketing name “Office 95”.

·         Office version 10 had the marketing names “Office XP” and “Office 2002”. 

·         Office Version 12 had the marketing name “Office 2007 “

·         Office Version 14 had the marketing name “Office 2010“

Say, 2010 came immediately after 2007, so what happened to Version 13? It was “lost in marketing” due to superstition.

Please use the right version numbers, we poor techies get confused easily.
