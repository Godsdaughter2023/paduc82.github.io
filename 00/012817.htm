rich007a say:

I would use either a "For i = 3 to lastRow... Next i" loop, or a "For Each c In rng... Next c" loop.

For a simple one column range, you can do something like:

    Dim c As Range, rng As Range
    Dim lastRow As Long
   
    lastRow = Cells(Cells.Rows.Count, 1).End(xlUp).Row
    Set rng = Range("A3:A" & lastRow)
    For Each c In rng
        If InStr(c.Value, "fpps", vbTextCompare) > 0 Then c.Offset(0, 1).FormulaR1C1 = "=YourFormula"
    Next c

 

To loop through multiple columns, you can do something like:

    Dim c As Range, rng As Range
    Dim lastRow As Long, i As Long
   
    lastRow = Cells(Cells.Rows.Count, 1).End(xlUp).Row
   
    For i = 1 To 10 '(will do cols A to J)
        Set rng = Range(Cells(3, i), Cells(lastRow, i))
        For Each c In rng
            If InStr(c.Value, "fpps", vbTextCompare) > 0 Then c.Offset(0, 1).FormulaR1C1 = "=YourFormula"
        Next c
    Next i

And to loop through a specific list of columns, you could do:

    Dim c As Range, rng As Range
    Dim lastRow As Long
    Dim varArray As Variant, varX As Variant

        For Each c In rng
            If InStr(c.Value, "fpps", vbTextCompare) > 0 Then c.Offset(0, 1).FormulaR1C1 = "=YourFormula"
        Next c
    Next varX

PS. Cells(row, col) can accept either a number or a letter ("A") for the column.

PPS. Alternatively, you could just use Find in VBA, to take the example from the Excel help file (Range.Find), you could use this:

 

    Dim c As Range, strFirstAddress As String
   
    With Range("A1:D10")
        Set c = .Find(What:="fpps", LookIn:=xlValues)
        If Not c Is Nothing Then
            firstAddress = c.Address
            Do
                c.Offset(0, 1).FormulaR1C1 = "=YourFormula"
                Set c = .FindNext(After:=c)

        End If
    End With

Its an awfull long time since I used OL 2003, so some of this relies on memory, which at my time in life is not allways good.

Even if you create a new Outlook profile (broadly default settings) the outlook data files, Personal Folders, the pst is not lost.

By default the first data file is named outlook.pst, a second data file would be named outlook1.pst, and so on.

The default location of the pst's is shown below

  • Outlook 2007 and previous on Windows XP
    C:\Documents and Settings\%username%\Local Settings\Application Data\Microsoft\Outlook\
  • Outlook 2007 and previous on Windows Vista, Windows 7 or Windows8
    C:\Users\%username%\AppData\Local\Microsoft\Outlook\
  • Outlook 2010 on Windows XP
    C:\Documents and Settings\%username%\My Documents\Outlook Files
  • Outlook 2010 and Outlook 2013 on Windows Vista, Windows 7 or Windows 8
    C:\Users\%username%\Documents\Outlook Files

To create a backup of the data file, you dont export, with Outlook closed you simply copy the pst file to another location.

You dont generally Import a pst file either, within OL you open the pst (that pst has to be located on the local drive, usually a Documents folder)

If configuring OL for the first time, as part of the process, you open that backup data file, and set that data file as the default msg store.
