rajivfonia say:

ken sheridan say: You'll find an example in ComboDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes an option for 'drilling down' through a hierarchy by means of three unbound correlated combo boxes in a form's header.  Each combo box is correlated with that above it in the hierarchy by referencing that above it as a parameter in its RowSource query, and returns an additional 'All.....' row by virtue of the RowSource query being a UNION query, retuning a zero in the bound column for the 'All.....' row.  The district combo box for example has a RowSource property of:

SELECT DistrictID, District, 1 As SortColumn
FROM Districts
WHERE CountyID=[Form]![cboGotoCounty]
OR [Form]![cboGotoCounty] = 0
UNION
SELECT 0,"All Districts", 0  
FROM Districts
ORDER BY SortColumn, District;

The form's RecordSource is a query in the WHERE clause of which each parameter is tested in this way:

(SomeColumn = <some parameter>
OR <some parameter>  = 0)

The full query is:

SELECT Counties.County, Districts.District, Parishes.Parish, Locations.Location
FROM ((Counties INNER JOIN Districts ON Counties.CountyID=Districts.CountyID)
INNER JOIN Parishes ON Districts.DistrictID=Parishes.DistrictID)
INNER JOIN Locations ON Parishes.ParishID=Locations.ParishID
WHERE (Counties.CountyID=Forms!frmDrillDown!cboGotoCounty
OR Forms!frmDrillDown!cboGotoCounty = 0)
AND (Districts.DistrictID=Forms!frmDrillDown!cboGotoDistrict
OR Forms!frmDrillDown!cboGotoDistrict = 0)
AND (Parishes.ParishID=Forms!frmDrillDown!cboGotoParish
OR Forms!frmDrillDown!cboGotoParish = 0)
ORDER BY Counties.County, Districts.District, Parishes.Parish;

djsteele, mvp say:

Sorry, without knowing the names of the fields and the table, I can only be generic.

 

Let's assume that the first column in the combo box contains the primary key for the table. Using DLookup, you'd use something like

Private Sub Combo6_AfterUpdate()
 
  Me.Text8 = DLookup("SomeField", "MyTable", "ID = " & Me.Combo6.Column(0))
  Me.Text10 = DLookup("SomeOtherField", "MyTable", "ID = " & Me.Combo6.Column(0))
 
End Sub

 

Using a recordset, it would be something like:

 

Private Sub Combo6_AfterUpdate()
Dim rs As DAO.Recordset
Dim strSQL As String

  strSQL = "SELECT SomeField, SomeOtherField " & _
    "FROM MyTable " & _
    "WHERE ID = " & Me.Combo6.Column(0)
 
  Set rs = CurrentDb.OpenRecordset(strSQL)
 
  If rs.BOF = False And rs.EOF = False Then
    Me.Text8 = rs!SomeField
    Me.Text10 = rs!SomeOtherField
  Else
' no matching record found
  End If
 
  rs.Close
  Set rs = Nothing
 
End Sub

 

Incidentally, it would be a very good idea to rename your controls so that they have meaningful names. You may know now what Combo6 looks up, and what Text8 and Text10 contain, but will you remember when you go back to do some additional work on the application a month from now?
