Yes, this is Word, as usual, trying too hard to be "helpful". The assumption is that if you opened a document from some other location, the next file you want to open will be there, too.

To override this behavior, install this macro in Normal.dotm (see  http://www.gmayor.com/installing_macro.htm if needed):

Sub FileOpen()
    ' force Open dialog to start in the default document location

    Dim dlg As Dialog
    Dim mypath As String
    ' work around bug described in http://support.microsoft.com/kb/220193
    ' the backslash at the end makes it a folder location, not a file
    mypath = Dialogs(wdDialogToolsOptionsFileLocations).Setting & "\"

    Set dlg = Dialogs(wdDialogFileOpen)
    dlg.Name = mypath   ' force location
    dlg.Show
    Set dlg = Nothing
End Sub

rainald say:

Question: What can I do to improve the result of clicking "Send to
OneNote" button in Internet Explorer 8?

There is no way to influence ON's behaviour of formatting stuff copied
from a web-page.
The results of copying web-stuff are almost perfect if the construction
of the respective page is "simple". If sophisticated formatting
(scripts) is used on a web-page the parser used by ON is having
problems.

For example, when I open the Web site of CNN (http://www.cnn.com/)
and click "Send to OneNote" button, I get in OneNote 2010 the page,
but it loses almost all original Web page formatting

Yes.
This site uses advanced techniques (incl. embedded flash animation) and
therefore ON just can not handle the formatting used there.

The results are hardly better when using
"Select all"
 and
"Copy & Paste" into Word.
Just check that.

If one wants a 1:1 copy if a web-page there's no way around printing
to ON (producing an image).
In many cases marking parts of a web-page and copying that into ON can
be a workaround.
