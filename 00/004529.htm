Another way is to use a custom function.
Place the code below in a standard module.
Use it in any cell (except H7) on any worksheet as:  =CountTcells()
Save the workbook as a macro enable workbook:  .xlsm
'---
Public Function CountTcells() As Variant
  Dim WS As Excel.Worksheet
  Dim lngCount As Long
 
  For Each WS In ThisWorkbook.Worksheets
   If WS.Range("H7").Value = "T" Then
     lngCount = lngCount + 1
   End If
  Next
  CountTcells = lngCount
End Function

You would have encountered no problem if you had written the B1-A1 formula correctly, to wit:

=IF(B1="","",--TEXT(B1-A1+(A1>B1),"[h]:mm"))

formatted as Time hh:mm.

(Note:  Do use "+(A1>B1)" if your "times" include dates that are not formatted.)

The double negative (--) converts text to numeric.

Nonetheless, it would be prudent to write the SUM formula as follows:

=--TEXT(SUM(D1:D11),"[h]:mm")

formatted as Custom [h]:mm.

The purpose of the TEXT expression is to ensure that time is rounded to the minute.  Otherwise, small arithmetic anomalies arise in the numeric calculations of B1-A1+(A1>B1) and SUM(D1:D11).

Sometimes, the small anomalies are innocuous:  you probably will not observe them when formatted as hh:mm, and you might not care unless and until you do some comparison or match/lookup

But that's the point:  as soon as you do a comparison or match/lookup, the small arithmetic anomalies might create false negatives (mismatches) that will seem mysterious because you might not see the differences when the format is h:mm.  (Sometimes not even with the format h:mm:ss.000.)

The arithmetic anomalies arise because of the way that Excel represents time and numbers and performs arithmetic (64-bit binary floating-point).  It is prudent to explicitly round such calculations whenever you expect a specific precision (e.g. to the minute).  For Excel time, --TEXT(...,"[h]:mm") is the simplest reliable way to "round" time.
