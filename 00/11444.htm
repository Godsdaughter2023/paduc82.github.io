bernie deitrick say:

Special cells can find errors, but it is the second value passed to the specialcells object. Sort first (this speeds row deletion up tremendously) and delete rows where the formulas return errors:

 

Cells.Sort Key1:=Range("D2"), Order1:=xlAscending, Header:=xlYes
Range("D:D").SpecialCells(xlCellTypeFormulas, 16).EntireRow.Delete

 

If sort order matters, do the above and then re-sort based on whatever sorting you require. If order doesn't matter, those two lines will be all you need.

rick rothstein, mvp - excel say:

Put this UDF (user defined function)...

Function AfromBlankEs() As String
  Dim X As Long, LastRow As Long
  Const StartRow As Long = 1
  Application.Volatile
  LastRow = Cells(Rows.Count, "A").End(xlUp).Row
  For X = StartRow To LastRow
    If Len(Cells(X, "E").Value) = 0 Then
      AfromBlankEs = AfromBlankEs & ", " & Cells(X, "A").Value
    End If
  Next
  AfromBlankEs = Mid(AfromBlankEs, 3)
End Function

into a module and then use this formula in whatever cell you want to see the result displayed in...

=AfromBlankEs()

If UDF's are new to you, they are easy to implement. Press ALT+F11 from any worksheet to get into the VB editor, click Insert/Module from its menu bar, and then copy/paste the above code into the code window that opened up. That's it. Go back to your worksheet and use the above formula in whatever cell you want to display the results in.

rick rothstein, mvp - excel say:

Here is a UDF that you can use...

Function SumValues(Rng As Range, AfterText As String) As Double
  Dim Cell As Range
  For Each Cell In Rng
    SumValues = SumValues + Val(Split(Cell & AfterText, AfterText)(1))
  Next
End Function

Just pass it the range of cells whose values you want to sum in the first argument and the text after which to find those values. So, if you wanted the sum of all the numbers after FH in the range D7:AJ7, your formula would be...

=SumValues(D7:AJ7,"FH")

and....

=SumValues(D7:AJ7,"V")

By the way, the UDF does not limit you to only "FH" and "V" as the second argument... you can specify any delimiter. Note, if your values are floating point numbers and your regional setting does not use the dot for its decimal point, then I do not know if the UDF will work for you or not (I do not think the Val function is regionally aware and I have no experience with international programming to know). Oh, and as written, the UDF retrieves only the first value after the specified text even if the specified text is repeated within its cell.
