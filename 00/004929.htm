ken sheridan say: An important thing to understand about this method of inserting a new row into a referenced table (Students in your case) is that when the code opens the Add A Student form it passes the names you've entered to it via the OpenArgs mechanism.  The code in Add A Student form's Open event procedure then sets the DefaultValue property for the first and last name controls respectively.  In my demo it's:

    Dim strFirstName As String, strLastName As String
    
     If Not IsNull(Me.OpenArgs) Then
        strFirstName = Left(Me.OpenArgs, InStr(Me.OpenArgs, " ") - 1)
        strLastName = Mid(Me.OpenArgs, InStr(Me.OpenArgs, " ") + 1)
        Me.FirstName.DefaultValue = """" & strFirstName & """"
        Me.LastName.DefaultValue = """" & strLastName & """"
     End If

Setting the DefaultValue property of a control does not initiate a new record, it merely inserts values into the controls ready for when a record is inserted.  A new record is only initiated if further data is added to the form.  If the form is closed no record is inserted into the table.  This may explain why you are getting the error, but it's hard to see why as you should get a message telling you the student wasn't added to the table if you simply close the Add A Student form without adding further data, and the new student won't show in the combo box.  So unless you are getting the message, and the new student isn't showing in the combo box, the error may be due to something else altogether.  Is the ControlSource property of the combo box the relevant foreign key, StudentID?

BTW if you wanted a new record to be inserted without having to enter any of the other data you should amend the above code so that it assigns the names to the Value property of each of the two controls.  This is the default property, so doesn't have to be specified:

    Dim strFirstName As String, strLastName As String
    
     If Not IsNull(Me.OpenArgs) Then
        strFirstName = Left(Me.OpenArgs, InStr(Me.OpenArgs, " ") - 1)
        strLastName = Mid(Me.OpenArgs, InStr(Me.OpenArgs, " ") + 1)
        Me.FirstName = """" & strFirstName & """"
        Me.LastName = """" & strLastName & """"
     End If

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column.

The code in the Add A Student form's Open event procedure should be as in my demo, but as the key is not an autonumber the form must also include a control bound to the StudentID column in which the 9 digit number for the new student can be inserted manually.

ColumnCount:   2

Experiment with the column widths to get the best fit.  The ListWidth should be the sum of the column widths.

=cboStudent.Column(1)

where cboStudent is the name of the combo box.  The Column property is zero-based so Column(1) is the second column, the concatenated name.

If Not IsNull(DLookup("StudentID", "Students", "StudentID = " & NewData)) Then

If Not IsNull(DLookup("StudentID", "Students", "StudentID = """ & NewData & """")) Then

End If

Note that when setting the DefaultValue property of a control you always wrap the value in quotes characters regardless of the data type of the column to which the control is bound, so this code would be the same whether StudentID is a number or text data type.

dale howard [mvp] say:

Andy --

The Task Mode column only shows up by default in only a few task tables, the most notable of which is the Entry table.  I suspect you are not seeing the Task Mode column by default because you upgraded from Microsoft Project 2003 or 2007 to either Microsoft Project 2010 or 2013.  So, here is how to fix the problem:

  1. Create a new blank project.
  2. Display the Gantt Chart view with the Entry table applied.
  3. Right-click on the Task Name column header and select the Insert Column item on the shortcut menu.
  4. In the list of available columns, select the Task Mode column.
  5. Right-click on the Task Mode column header and select the Field Settings item on the shortcut menu.
  6. In the Field Settings dialog, set the Width value to 9 and then click the OK button.
  7. Click File > Info > Organizer.
  8. In the Organizer dialog, click the Tables tab.
  9. In the list of tables on the right (in the project), select the Entry table.
  10. In the middle of the dialog, click the Copy button.
  11. When prompted to overwrite the default Entry table, click the Yes button.
  12. Click the Close button to close the Organizer dialog.

Repeat steps #2-12 for every table in which you want to include the Task Mode column by default.  Hope this helps.

joe men say:

Thank you for providing additional information about your query.

It's a limitation of the current version for the Maps app. For now, we recommend that you drop a pin on the map to save it as a Favorite. Simply tap and hold on the map and select Drop a pin from the contextual menu, you should then be able to save it from the details of the location.

You may also install a third party map application from the Windows Store.

Help us make Windows Mobile better by submitting your concern and suggestions about Windows and Apps, using the Feedback Hub app. To access this app, tap on the icon located on the lower right corner of the Maps app and then tap Feedback.

Thank you.
