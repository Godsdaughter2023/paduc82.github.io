Do you want to return all records for which there is no match on the combination of the fields Phone, Email and Company? If so, create a triple outer join as you have done.

But if you want to return all records for which there is no match on Phone, and also no match on Email and no match on Company, you need to add the second table three times to the query.

Create an outer join to the first instance on Phone (with Is Null in the criteria), an outer join to the second instance on Email (also with Is Null in the criteria) and an outer join to the third instance on Company (ditto).

suzanne s. barnhill say:

Here's how to do it without a macro.

  1. Select the column containing email addresses and copy it.
  2. Paste it outside the table.
  3. Select the new (one-column) table.
  4. On the Table Tools | Layout tab, in the Data group, click Convert to Text.
  5. In the Convert Table to Text dialog, the selection for "Separate text with" is irrelevant because each row will be a paragraph. Click OK.
  6. To remove the empty paragraphs, select the converted text and replace ^p^p with ^p.
  7. Then replace ^p with a comma and space.

doug robbins - mvp office apps & services (word) say:

With the selection in the column containing the email addresses, running a macro containing the following code will create a new document that contains the email addresses in the form that you want.

Dim strEmails As String
Dim crange As Range
Dim newdoc As Document
strEmails = ""
With Selection.Columns(1)
    For i = 1 To .Cells.count
        Set crange = .Cells(i).Range
        crange.End = crange.End - 1
        If Len(crange.Text) > 1 Then
            strEmails = strEmails & "," & crange.Text
        End If
    Next i
End With
strEmails = Mid(strEmails, 2)
Set newdoc = Documents.Add
newdoc.Range.Text = strEmails

If the email addresses in the table do not start on the first row of the table, replace the 1 in the line For i = 1 to .Cels.Count with the appropriate number.
