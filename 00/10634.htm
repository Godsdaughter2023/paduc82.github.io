aaronwinkler say:

Avoiding circular reference error

Hello. I am trying to get a formula to work within the same cell the formula is laid into. I am trying to place 40% on the total the user enters within the cell.  example: in cell c24. I will enter the price of $10.00. within the cell "c24" I want it to automatically multiply by 1.40. example of what I am writing as the formula within cell C24. =SUM(C24*1.40) then entering my price afterwards of $10.00 which it should auto calculate than the info entered as $10.00 should be multiplied by 1.40 and giving me the total within cell C24 as $14.00. But after I enter the formula within the cell, I get the error message of Circular References. I understand what this means and would like to know of a way around it. I have looked at many posts with similar questions but they have not helped. If anyone has a solution to this issue, it would be greatly appreciated if you could share it with me.

PRICE "C24" AMOUNT "D24" "B24 being Quantity"
$10.00 should go through =sum(c24*1.40) becoming $14.00 running into =sum(B24*C24) totaling $24.00

Thank you. 

[Title was "Excel microsoft office 365"]

 
hansv mvp say:

It is not a good idea to do this. $1000 would become 1.4*$1000 = $1400 and this would automatically become 1.4*$1400 = $1960 and that would become 1.4*$1960 = $2744, etc. etc.

One solution would be to use one column for entering data and another column to multiply the values from the first column with 1.4.

Another solution would be to use VBA to multiply the value:

  • Right-click the sheet tab.
  • Select 'View Code' from the context menu.
  • Copy the code shown below into the worksheet module.
  • Switch back to Excel.
  • Save the workbook in a format that supports macros: .xlsm, .xlsb or .xls, but not .xlsx.
  • Make sure that you enable macros when you open the workbook.

In the following, I'll assume that you want to multiply values entered in C2:C100 with 1.4.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim cel As Range
    On Error GoTo ErrHandler
    If Not Intersect(Range("C2:C100"), Target) Is Nothing Then
        Application.ScreenUpdating = False
        Application.EnableEvents = False
        For Each cel In Intersect(Range("C2:C100"), Target)
            If Not IsError(cel.Value) Then
                If cel.Value <> "" And IsNumeric(cel.Value) And cel.HasFormula = False Then
                    cel.Value = 1.4 * cel.Value
                End If
            End If
        Next cel
    End If
ExitHandler:
    Application.EnableEvents = True
    Application.ScreenUpdating = False
    Exit Sub
ErrHandler:
    MsgBox Err.Description, vbExclamation
    Resume ExitHandler
End Sub

This is incorrect design. You should have a child table like the following:

tblMeasurement
MeasurementID (PK Autonumber)
ParentID (FK)
MeasurementTypeID (FK)
Measurement

ParentID would link back to the master record. I'm not clear is this is a house or what, MeasurementTypeID would link to a lookup table of room types.

Now whether you want to store the measure in 2 fields (feet and inches) or one is something for you to decide. You have been given good advice on how to do that.
