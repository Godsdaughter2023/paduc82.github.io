    brian beckers say:

    We had a user experience the exact same error, [0x8004010f-0x8004010f-0x000501].

    It only happened when the user was sending an e-mail with an attachment to two specific outside e-mail recipients.

    After trying several different things, including clearing the "nickname" cache, toggling offline cache, etc., I found a solution that solved the issue:

    Create a new Outlook profile.

    Since you are using Office 365, this will be an easy fix.

    1. Navigate to Control Panel -> Mail
    2. Click "Show Profiles..."
    3. Click "Add..."
    4. Fill in the required information.
    5. Once Outlook has contacted O365 and completed creating the profile, select "Always use this profile" and choose the new profile you just created.
    6. HINT: Don't remove the old profile until you know everything works. Once you have verified that all is good, feel free to remove the old profile.
    7. Open Outlook. After Outlook has authenticated with the user's mailbox, Outlook will start populating the profile with the user's mailbox contents. (Note: Depending upon how much "stuff" the user has in their mailbox, this could take some time).
