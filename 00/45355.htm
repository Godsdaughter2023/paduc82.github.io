Changing a user account from Administrator to Standard User still - Microsoft Community

ehgoodrich say:

Changing a user account from Administrator to Standard User still allows access

In providing support for friends & family, whenever I set up a new Windows box for the first time, the first account I create is an Administrator account (named "Admin" for example) with a strong password.  I then create the actual user account that the person will use for everyday access to the PC as a "Standard User" without Administrator rights.  I have been using this setup since XP and still implementing it on Windows 10 and I believe it is the #1 security measure I've used to avoid malware, especially with inexperienced users.

A common problem I've run into over the years is helping a user who has setup their system with their normal user account as the only one (which must be a full Administrator account).  Implementing my setup on such an existing system requires one of the following options:

  • Backup all data, reinstall Windows from scratch, create the accounts as described above, reinstall all programs & copy back the data from the backup.  This is the cleanest option, but also the most time consuming.  However, I end up using it a LOT since, if someone has been surfing the Internet with a full Administrator account (and most likely a weak or non-existent password) for any length of time, they've picked up enough malware that Option 1 is the only way to clean things up.

  • Rename the existing "User" account to "Admin", create a new "User" account and copy personal data from "Admin" to "User".  Anyone who's tried to rename accounts under Windows knows this is VERY hard to do completely, since the registry is littered with references to the old username, in many cases obfuscated so that a simple find&replace isn't feasible.

  • Create a NEW "Admin" account with full administrator rights and a strong password, and change the existing "User" account type from Administrator to Standard User.  At first glance, this would seem to be the easiest solution for a relatively clean system that doesn't require a rebuild.  However, when I've tried this, I'm usually left with the "modified" User account having MORE access than a Standard User account would normally have.  I believe this is due to the existing User account retaining Ownership of many files, folders & registry entries in areas where a Standard User normally is allowed NO access.

I have given up on trying to get option #2 to work, and would like to use option #3 when #1 is not needed to remove malware.

Has anyone dealt with this issue and, specifically, has anyone come up with a way to make option #3 work correctly??

Thanx for any replies...
