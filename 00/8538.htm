steve rindsberg say:

Yoga's correct in that it's not a feature that's built into PowerPoint itself.

However, with a bit of VBA, you can fairly easily make this happen almost instantly.

Assuming you've put the text/shape you want on the first slide, set it's text fill to transparent and left it selected, this will copy it to every other slide in the presentation, same position etc.

Sub CopyWatermark()
    Dim osh As Shape
    Dim osl As Slide
    Dim x As Long
    
    Set osh = ActiveWindow.Selection.ShapeRange(1)
    
    osh.Tags.Add "WaterMark", "YES"
    osh.Copy
    
    For Each osl In ActivePresentation.Slides
        ' Don't paste the watermark atop itself
        If osl.SlideIndex > 1 Then
            osl.Shapes.Paste
        End If
    Next
    
End Sub

'' And this will delete all of the watermarks for you:

Sub DeleteWatermarks()

    Dim osl As Slide
    Dim osh As Shape
    
    For Each osl In ActivePresentation.Slides
        For Each osh In osl.Shapes
            If osh.Tags("Watermark") = "YES" Then
                osh.Delete
            End If
        Next
    Next

End Sub

    hansv mvp say:

    You could use a generic procedure like this:

     

    Sub HandleKey(ctl As MSForms.TextBox, KeyAscii As MSForms.ReturnInteger)
        Select Case KeyAscii
            Case Asc("0") To Asc("9")
            Case Asc("-")
                If InStr(1, ctl.Text, "-") > 0 Or ctl.SelStart > 0 Then
                    KeyAscii = 0
                End If
            Case Asc(".")
                If InStr(1, ctl.Text, ".") > 0 Then
                    KeyAscii = 0
                End If
            Case Else
                KeyAscii = 0
        End Select
    End Sub

     

    Call it from each of the text boxes like this:

     

    Private Sub TextBox1_KeyPress(ByVal KeyAscii As MSForms.ReturnInteger)
        Call HandleKey(Me.TextBox1, KeyAscii)
    End Sub
     
    Private Sub TextBox2_KeyPress(ByVal KeyAscii As MSForms.ReturnInteger)
        Call HandleKey(Me.TextBox2, KeyAscii)
    End Sub

     

    Make sure that you use the correct name of the text box in each call.

     

    Another option would be to create a class with a custom event handler but that'd be more complicated.

    How do you calculate numbers in a table you have created in Word? - Microsoft Community

    nytrip say:

    How do you calculate numbers in a table you have created in Word?

    I need to calculate numbers in a table I created in Word 7 
    pesach shelnitz say:

    You can use the =(Formula) field to perform calculations in tables in Word. For instructions on inserting this field and other fields, seehttp://office.microsoft.com/en-us/word-help/insert-and-format-field-codes-HA010338798.aspx .

    Here are a couple of examples for using the =(Formula) field in a table.

    1. Place your cursor in the an empty cell in the last row your table in a column containing numerical data and follow the procedure in the link above to insert a field. Select =(Formula) and click the Formula button. In the Formula dialog box, type =SUM(ABOVE) as the formula and click OK. The sum of the numbers in the column should appear.

    2. Add numerical data in the third row of your table from the second column to the fourth column. These cells are numbered B3, C3, and D3. Place the cursor in the any empty cell in your table and follow the procedure in the link above to insert a field. Select =(Formula) and click the Formula button. In the formula dialog box, type =SUM(B3:D3) as the formula. The sum of the cells from B3 to D3 should appear.

    Other variations of the formula include =SUM(LEFT) for calculating the sum of the numbers in the same row to the left of the cell containing the formula.

     

     

    Hope this helps, Pesach Shelnitz, http://makeofficework.com

    macropod say:

    VBA management in Word is the same as in Excel. If you want to restrict who can run your macros, you have a number of options, including:

    1. Add an input box to each of them, to solicit a password before allowing the macro to run;

    2. Add code to each macro to check the username at run time;

    In addition, prefixing each sub with 'Private' or adding a dummy optional parameter to each sub's name (e.g. (Sub MySub(Optional X as Boolean)) will mean they can't be seen in or run from the macros dialogue. That's useful if you want to force the macros to be run from the VBE or via another macro.

    No, in my last reply, I just explain how to sync iCal with iCloud but not referring to Outlook for Mac.

    For syncing between iCal and Outlook for Mac, the only thing for you to do is to set up your account (for example, Office 365 account) via “Exchange”. The detailed steps should be:
    1. Open Calendar or iCal.
    2. From the Calendar menu, select Preferences.
    3. In the General dialog box, click the Accounts tab.

    In the Email Address field, enter your USC email address.
    In the Password field, enter the password you use with your USC email address.
    Click Create. Calendar should find and automatically configure your account.
