hula_baloo say:

 That does seem slow.  If you're used to using FAT32 formatted drives, the more robust NTFS does take significantly longer because it does some things synchronously that with FAT32 things are started and left under the assumption that they will always complete successfuly; and unfortunately that can sometimes result in problems.

  In addition to Ken Blake's suggestion to check for malware, I'd suggest you check the utilities installed.  If you have one of the real time backup programs installed, some of them can do a real job on performance.  Try booting up in Safe Mode and try some of your slow deletes or copies.  Regarding copies, your antivirus program can cause a big performance hit because they can check each file as it's being copied.  Temporarily turning off your antivirus program can greatly improve copy speed in a lot of cases.  One thing you might want to try is DEFRAG if you haven't done so recently.  Modern versions of Windows usually do that automatically, but sometimes doing a manual DEFRAG can help.
