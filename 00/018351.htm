neo yu msft say:

Hi David,

If you’re using hybrid moves to migrate mailboxes to Office 365, the on-premises mailboxes will be changed to mail-enabled users automatically, no matter which Exchange Servers are installed in the on-premises.

However, if we don’t have hybrid deployed, and we want to deploy DirSync/SSO after a cutover migration, then we need to convert the on-premises mailboxes to mail-enabled users manually first, otherwise, the DirSync will fail to link the on-premises users with Office 365 users.

Given the information above, no matter which version of Exchange Server are installed in the on-premises, if we just want to migrate the on-premises mailboxes without deploying DirSync/SSO, then we don’t need to convert the on-premises mailboxes. However, if want to deploy DirSync/SSO after the cutover migration, then we will need to convert these on-premises mailboxes to mail-enabled users.
