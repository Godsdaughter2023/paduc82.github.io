    bob jones aka: cybertaz say:

    Those you refer to as being 'removable' are categorized as "personal information" rather than as Document Properties. So yes, they both are metadata but are handled differently.

    BTW: I'm afraid you may be laboring under a misconception if you mean "...send it back to him" as an email attachment. Contrary to popular belief, emailing attachments does not send the document file... it sends an encoded version. When the recipient downloads the attachment a new copy of the file is constructed which bears the date of its creation, not the Created Date of the original from which it was derived.

    In order to retain the same Properties, each user must work in exactly the same document file stored in a shared location or the original or a duplicate [not a Save As version] has to be physically transported from one location to another.

    bob jones aka: cybertaz say:

    A program offers no straightforward way to remove file properties. If it can be done it would require a programmatic method (I'm not sure whether VBA would do it), possibly a Terminal hack, or [more likely] some sort of file modification utility.

    The simplest method is to send a PDF rather than an actual Word document. That can be helpful in other regards as well, such as making certain that the document displays as intended - You have little control over how a Word document will display when opened on a different computer.

    If you must forward an actual Word document file, as a very last step use Save As to create a new version for the intended purpose. It will still have properties but will be minimal based on the generation of that version of the file -- IOW, the original file properties do not carry over as they would if you just make a duplicate/copy in Finder.

    hansv mvp say:

    You have to join the queries on Employee Number:

     

    SELECT PerformanceTotalEndHC.[FORD Geo], PerformanceTotalEndHC.[FORD Project], PerformanceTotalEndHC.TotalHC AS TotalHC, PerformanceRollofHC.RollofHC AS RollofHC, PerformanceResignHC.ResignHC AS ResignHC, PerformanceNewHC.NewHC AS NewHC, TotalHC+RollofHC+ResignHC-NewHC AS MonthStartHC FROM PerformanceTotalEndHC Inner Join (PerformanceRollofHC Inner Join (PerformanceResignHC INNER Join PerformanceNewHC ON PerformanceResignHC.[Employee Number] = PerformanceNewHC.[Employee Number]) On PerformanceRollofHC.[Employee Number] = PerformanceResignHC.[Employee Number]) On PerformanceTotalEndHC.[Employee Number] = PerformanceRollofHC.[Employee Number])

     

    I have assumed that PerformaceNewHC was a typo and should have been PerformanceNewHC.
