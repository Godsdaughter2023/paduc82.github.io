omer atay say: This is a security feature. Only the client that inserts the image can run OCR on it. This is to prevent someone to insert a malicious image on a shared notebook that would cause the OCR engine to crash or do something worse. This would cause every user of the shared notebook to hit the issue. In the case of a non-PC client like iPhone, Windows Phone or OneNote web application (skydrive or sharepoint), the inserted image is not OCR-ed by the client and shows as OCR disabled for that image. Picking Make Text in Image Searchable > English, should stick though, as that is a user consent that the user is trusting the image and wants OCR to run. Right click > Copy Text from this Picture should also trigger OCR on the image.
