'------ start of example code ------
Private Sub cmdSelectAll_Click()

Wow--that's strange behavior, but I think I understand why. When you check a box in a multiselect list, a new repeating node is added to the DOM, containing the selection. This addition of a new node causes the rules to fire. However, when you clear a checkbox, the associated node is DELETED from the DOM, which as the node is gone rather than "changed" (I know, a technicality), no rules are fired.

In my opinion, this is a bug. However, it has apparently been this way for quite a while, and you shouldn't expect the behavior to change.

I found that if I added a helper node that calculated the current count of the repeating node (category in your case) that it failed to fire rules on it when boxes were checked, but did when boxes were unchecked.
