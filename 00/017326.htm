Excel 2010 pivot table manual filter - selections changed - Microsoft Community

lisawl say:

Excel 2010 pivot table manual filter - selections changed

We build spreadsheets with pivot tables in them referencing an OLAP cube as the data source. We use manual filters to choose several GL acount numbers. When we are done we save the spreadsheet.

We have reopened several spreadsheets from a couple months ago to over a year ago and when we look at the pivot table the filter selections (check boxes) are pointing to different accounts than they were when we created them. How can we retain the original selections so that we can easily pull up the spreadsheet and refresh to just pull in the latest data for the selected GL accounts?

Thanks!

lisawl say:

We finally found the issue. Excel correctly uses the "key" dimension column value when generating the MDX for a cube query. Most dimension columns have the same column defined for "key" value and "name" value in the cube. For example, in the GL Account Dimension, Account Num column uses the same value for "key" and "name", so the MDX will always correctly use the Account Number value in the query.

For one column in each dimension, we used different values for "key" and "name". This was done intentionally to combine the key column and another popularly used column in order to limit the number of dimension attributes (saves processing time). For example, the "Account Num - Name" was defined with GlAccountKey as the key value, and "Account Num - Name" as the "name" value. The "key" is a generic ID which is refreshed every weekend when we do a full refresh of our data warehouse. If a new account was added, or an account was deleted that week, the key value would change. Thus, when you brought up your spreadsheet with a key value selected, a different "Account Num - Name" would be selected.

To fix, we will no longer combine the real dimension key column with another popular column in the dimension. We will change each dimension so that the "key" and "name" columns always match, or at the very least not use the generic key column. This will ensure that the Excel selections do not change since the MDX will refer to the correct values and not a generic key.
