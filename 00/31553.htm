hairyfool say:

With the server connected to the network open a command prompt on the new computer at Admin level

In the command prompt type these two commands

ipconfig /release

ipconfig /renew

The second command may take a few seconds and then produce a few lines including the new address.

To prevent it happening again check the documentation of the router, each are different so I cannot be specific. However in the router's DHCP server configuration you should be able to reserve the specific address for the server so it will not ever be allocated any other address and no other device would get given that address.

If not then look at the range of addresses the DHCP server uses. Change the server to a fixed address instead of using DHCP and limit the DHCP range to not include the fixed address. Static addressing is always preferable for devices that provide a service on the network.

For example my router has been modified to issue in the range x.x.x.16 to x.x.x.127. My server is at .4 the printer at .5, the security camera system uses .128 and above.
