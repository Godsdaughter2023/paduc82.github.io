jay freedman say:

As best I can tell, there is no way to distinguish between an inserted inline shape and a drag/dropped inline shape. Using the Watch Window in the VBA editor, I see that all of the properties are identical; the only difference seems to be the way they're treated in the user interface (the ribbon, the selection border and handles, and the entries on the right-click menus).

However, you have a logical error in your macro, and I cannot understand how it works correctly even for an inserted picture. The problem is this: If the Selection contains an InlineShape and does not contain any Shape objects (floating pictures), then the Selection.ShapeRange.Count value is zero and the WrapFormat and its Type property are not defined. (I can't tell how you've set the Selection; maybe it does contain the anchor of a Shape object?)

The correct way to do the job is to first convert the InlineShape object to a Shape object, and then set the WrapFormat.Type of the Shape object:

Sub x()
    Dim oILShp As InlineShape
    Dim oShp As Shape
    For Each oILShp In ActiveDocument.InlineShapes
        Set oShp = oILShp.ConvertToShape
        oShp.WrapFormat.Type = wdWrapTopBottom
    Next
End Sub

The result in both the inserted and the drag/dropped cases is a picture with Top and Bottom wrapping, but they still exhibit the same differences in the user interface as before.

Why "on error resume next" and "on error goto 0" have - Microsoft Community

poliasks say:

Why "on error resume next" and "on error goto 0" have to be placed where they are in this sub?

Hi,

 

Below is a simple sub to get user selected range - this is an example from John Walkenbach's book.

 

What I can't understand here is that why in this program, "On error resume next" and "On error goto 0" have to be placed exactly where they are. I tried to place these two statements in different places within the procedure, then I got errors. Please explain your thoughts - think about those early days when you just started to learn VBA did this kind of question bother you as well?

 

Apologies if this question sounds stupid to you. And, thanks!

_____________________________________________________________________

Sub GetUserRange()
    Dim UserRange As Range
   
    Prompt = "Select a range for the random numbers."
    Title = "Select a range"

'   Display the Input Box
    On Error Resume Next
 
    Set UserRange = Application.InputBox( _
        Prompt:=Prompt, _
        Title:=Title, _
        Default:=ActiveCell.Address, _
        Type:=8) 'Range selection
   
    On Error GoTo 0

'   Was the Input Box canceled?
    If UserRange Is Nothing Then
        MsgBox "Canceled."
    Else
        UserRange.Formula = "=RAND()"
    End If
End Sub

___________________________________________________________________________

jeeped say:
The On Error GoTo 0 is the default behaviour when a VBA macro encounters an error. The On Error Resume Next signals the VBA engine skip over an error and continue processing subsequent commands. This is necessary as exiting the InputBox with Cancel will result in the Set command throwing an error and halt the macro without it. After that potential error has been handled, On Error GoTo 0 is specified to restore default error handling.

So the Workseet_Calculate event for a sheet often fires when its not the active sheet.
