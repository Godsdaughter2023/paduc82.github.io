Mailboxes - Procedure: Account Soft-Matching - Microsoft Community

alex rizeanu msft say:

Mailboxes - Procedure: Account Soft-Matching

Technical Level : Intermediate

Summary

The below content details the procedure of performing a soft match between an on-premises Active Directory user with AzureAD (Office365) user.

For more details on when this procedure should be used please see: Mailboxes - AzureAD Account Matching - Source of Authority Transfer to On-Premises


Details

Step One: Pause directory synchronization

We do not want a delta sync firing off in the middle of our work. You can either disable the scheduled task used to trigger the sync in 30 minutes intervals or you can disable the service itself.

Step Two: Backup EXO account attributes

Obtain all attribute information including a full list of email addresses from the cloud account.
If the recipient type is a mailbox, run the following commands

$FormatEnumerationLimit = -1 # so we don't truncate the email addresses output
Get-mailbox <mailboxID> | fl

Step Three: Prepare the On-Prem Account

Proper account provisioning in an Exchange Hybrid scenario using dirsync requires that you have at least the on-premises Exchange management tools installed in your environment. This article will not deal with methods outside of using the Exchange management tools. If the you don't have them installed, you need to install them.
If you are linking an on-prem account to an EXO mailbox, then the proper procedure is to create an on-premises remote mailbox account.
This can be done directly when you create the account or an existing AD account that is not yet mail-enabled.

Scenario One: New Account

Use the output you collected in step 2 to fill in the attribute values below 

new-remotemailbox -Name -UserPrincipalName -Alias -DisplayName -FirstName -LastName -Initials -PrimarySmtpAddress -RemoteRoutingAddress 
# For room or equipment mailboxes, add the -room or -equipment switch
# Once the account is created, then you can add all of the additional email addresses that you need to preserve.
set-remotemailbox <accountYouJustcreated> -emailaddresses

Scenario Two: Existing Account 

enable-remotemailbox IDofExistingUser -alias -primarySMTPAddress -remoteRoutingAddress
# For room or equipment mailboxes, add the -room or -equipment switch 
# Once the account is mail-enabled, then you can add all of the additional email addresses that you need to preserve.
#If you want to change attributes like UPN and displayname, do that here on this command.
set-remotemailbox accountYouJustcreated -emailaddresses

Step Four: Enable Synchronization

Verify the account properties are as you intend them to be. Once you are satisfied, enable directory sync and kick off a delta. Watch the changes occur.
