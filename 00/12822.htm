Split from this thread.

Re windows 10 defender updates KB2267602 (Definition 1.223.1723.0) - Error Ox80070643

I have McAfee  antivirus software and can't update my windows defender. How can I fix it.

The problem here is that Windows Defender didn't get turned off completely when McAfee was installed – and that’s why the definition updates are failing. The registry switches that are supposed to disable Defender aren’t getting toggled properly on lots of third-party installations. So one obvious solution would be to just redo the installation by uninstalling the third-party app; running its cleanup utility; and then reinstalling. And of course this does, in fact, usually succeed in throwing the relevant registry switches and fixing the problem. But Norton users, in particular, are reporting that this sometimes fails to resolve the issue – and maybe some other folks just might not want to take the time to recycle the installation.

So we have a shortcut that fixes the definition update issue by using Group Policy to completely turn off Windows Defender. In the Professional edition of Windows 10, you can do this with the Group Policy Editor; and in the Home edition, you can use the Turn_Off_Windows_Defender.reg file. Everything you need is right here:

http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

And there’s also an even quicker method for Norton and Avast users that essentially mimics a reinstall by disabling the third-party AV app; starting Windows Defender; and then enabling the third-party app while Defender is still turned on:

http://answers.microsoft.com/en-us/windows/forum/windows_10-update/definition-update-for-windows-defender-kb2267602/d4edd314-a438-4ac2-b7a9-dc1c45a1ef62

This easy-as-1-2-3 method might also work with McAfee and other AV products, but we don’t have any confirmation for that yet.

Excel chart Floor shading - Microsoft Community

ads365 say:

Excel chart Floor shading

I have a 3d column chart which has both negative and positive values.  I am trying to shade the horizontal axis (at the vertical axis value 0) with a partially transparent colour so that there is a very clear visual distinction between positive and negative values.  I can colour the chart floor but it just shades the lowest value, even if I move the chart floor to 1 or -1 it doesn't change where the shading is, it just shades the lowest value.

teylyn say:
Hello,

maybe it's a  blessing, and not a problem. The Excel 2013 recommended charts no longer offer any 3D charts. And that is for a good reason. We've been lobbying with Microsoft for years to remove the charts that do not follow good data visualization practice, and, finally, with 2013, there has been some insight.

There is not a single 3D chart that I have seen so far that has benefited from a 3D effect.  Quite contrary. 3D representation of charts distorts the data values and makes the chart harder to interpret.

I know that is not what you want to hear, but experts in data visualization are all strongly opposed to 3D charts. 

If you insist on using 3D charts, you may need to learn to live with their short-comings. 

The chart floor is not connected in any way with the Y axis 0 value, as you have found out. 

You may want to try the "Invert if negative" formatting option to use a different color on the negative values of a chart. In a 3D chart that does not necessarily add clarity. See this:


Negative values are the same hue, but paler than the positive values. In a cluster like this 3D it is impossible to establish the zero line for each chart element, though. Note that the darker blue value at "a" of the X series starts at zero and goes up to one. But just behind it you see the pale yellow "a" value of the Y series, which is a negative, starting at 0 and going down to 1.  In the 3D chart, these two values almost overlap and it is impossible to tell from the chart alone what the starting and ending values for each value are.

Let's look at the chart with some helper lines drawn in manually:


The blue line is the Zero line for the blue series. The yellow line is the zero line for the yellow series. The black line is the zero for the gray series. Note that the top of the blue "a" in the X series is aligned with the zero of the yellow series. The -1 mark for the negative Y value in the "a" position aligns perfectly with the zero baseline for the blue series.  Now, how confusing is that? 

So, do yourself a favor and avoid 3D charts. They really don't add anything useful.

If you are interested in learning about data visualization, check out Stephen Few and his books "Show me the numbers", "Now you see it", "Information Dashboard Design" and anything else he has written. His blog is at www.perceptualedge.com.
