I am typing out a poem. The first letter of the first word on each - Microsoft Community

brucegatten say:

I am typing out a poem. The first letter of the first word on each line automatically capitalizes. How to have the program not capitalize the first letter of each line?

Some poems have capitalized first letters of first word in each line. We do not want that to be done automatically, for each line is not always begun with capitals.
suzanne s. barnhill say:

While Andrea's answer is technically correct, it is overkill. Instead of pressing Enter at the end of each verse, press Shift+Enter. Enter inserts a paragraph break, and Word assumes that the beginning of a paragraph is also the beginning of a sentence, requiring capitalization. Shift+Enter inserts a line break, about which Word makes no such assumption.

A secondary advantage of this approach is that you can define a paragraph style for your poetry that has some Spacing After. This will allow you to press Enter at the end of a stanza and get additional space between stanzas without having to press Enter twice.

If you do want to (at least temporarily) disable the capitalization, you will find the setting at File | Options | Proofing | AutoCorrect Options: "Capitalize first letter of sentences." I have to do this when I'm typing dictionary entries.

ken sheridan say: As your code to assign the value to the RESPONSIBLE_ENGINEER_SIGN_DATE control follows immediately after the code to open the login form the cboUser control in the latter will at this stage have no value.  What you can do is open the login form and pass the name of the calling form to it  with:

DoCmd.OpenForm FormName:="frmlogin", OpenArgs:=Me.Name

Then in the frmlogin form's cboUser control's AfterUpdate event procedure you can then assign the value to the control in the calling form with code along these lines:

Dim frm As Form
Dim ctrl As Control

If Not IsNull Me.OpenArgs Then
    On Error Resume Next
    Set frm = Forms(Me.OpenArgs)
    If Err.Number = 0 Then
        On Error Goto 0
        Set ctrl = frm.Controls("RESPONSIBLE_ENGINEER_SIGN_DATE")
        ctrl = cboUser.Column(2)
    End If
End If

If you need to use the login form in other contexts and pass other values to it, then you can  pass named or numbered arguments to it via the OpenArgs mechanism and use appropriate code in the login form's module to do whatever is required on the basis of the arguments passed in.  You'll find a module which enables you to pass multiple arguments in the file Args.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

Note that in the above code I've simply turned off error handling with On Error Goto 0.  It would of course be better to go to an error handler in the procedure.
