I'm not sure I follow exactly what has happened, but

 a. when you create a recipient list in Word 2003's MailMerge function, Word creates an Access/Jet .mdb file that has a table called Office_Address_List with 13 standard columns in it (unless you changed the list of items in Word using the Customize... button), and a query called "Office Address List" that executes the SQL code

SELECT * FROM Office_Address_List;

 b. Word uses the Query as its data source.

 c. each column is defined as an optional 255-character Text field

 d. As long as you added your fields in the Office_Address_List table in Access and set tem to be the same data type, I would expect them to show up in Word's Recipient List when you next opened your Mail Merge Main Document. However, if you did anything in Access while that address list database was open (such as create a Report, or anything that adds tables or queries, then it is likely that Word will no longer treat the data in the same way, because it no longer thinks it is an "Office Address List" (OAL).
