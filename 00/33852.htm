¡firedog say:

It's fairly clear that reverting to Windows 7 from 10 corrupts the message database. Many users have found that a brute-force reconstruction of the store folder cleared up the problems, but there may be gentler ways of dealing with them. However, the status of each message (e.g. whether it's been read, replied to or forwarded) will be lost whatever you do. This status information is stored in the database index, the file that will have to be replaced.

It is this corruption that causes the 'message cannot be found' error; the message list is extracted from the index file but when a message is selected, the program consults the index entry for the message concerned to find its location. It then retrieves the message body for display. If the message isn't in the folder the index points to - because it's in fact somewhere else - then you'll see the error message.

A simple remedy may be to force the program to create a new index. To do this,

1.   Close down Windows Live Mail and wait a few minutes.

2.   Copy the text in the box

FOR /F "TOKENS=4* SKIP=1" %A IN ('REG QUERY "HKCU\Software\Microsoft\Windows Live Mail" /V "Store Root"') DO @CALL explorer %A %B & EXIT
3.   Press Windows key  + R to open a Run box, type cmd and press Enter to open a command window,

4.   Right-click in the window, select Paste and press Enter. This should open Explorer at your store folder.

5.   Right-click on the file called Mail.MSMessageStore and rename it to Mail.old.

6.   Close Explorer and disconnect from the Internet (e.g. disable Wi-Fi or unplug the Ethernet cable).

7.   Launch Windows Live Mail and leave it alone to construct a new index. With thousands of messages, this will take a long time. It's best not to interrupt the process, so do something else meanwhile.

8.   When the program has settled down, close it down and again wait a few minutes before re-launching it. This will enable the program to complete the indexing process after the user interface has disappeared.

Please report back to say what happened.
