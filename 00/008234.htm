Locating all impacted tasks when changing a date - Microsoft Community

tonycaronmy say:

Locating all impacted tasks when changing a date

I have a 500 line project with most tasks linked. When I make a date change to one entry, is there a way to see all related dates that have been affected other than scrolling up/down the screens each time?

dale howard [mvp] say:

Tony --

Before you make the schedule change, drag the split bar to the right edge of the Finish column.  When you make the schedule change, Microsoft Project will apply the light blue cell background color to every task impacted by the schedule change.  You will see this in the Duration, Start, and/or Finish columns.  Unfortunately, there is no way to apply a filter to see only the changed tasks.  Thus, you must scroll up and down the project to see which tasks are impacted.  Hope this helps.

trevor rabey say:

Have you tried setting a baseline just before you make the change? Then make the change. Then in the tracking Gantt view you will see the current bars and the baseline bars together and the difference between them is the effect of the change.

Or before the change, copy the start column and paste it into a spare start field (Start1). Then after the change compare the Start1 values with the Start values. If you use a spare flag field (Flag1) with an IIF(Start1<>Start,1,0) formula it will flag the cells that have changed. You can make a filter or group to extract the tasks where Flag1 = yes.

rod gill say:

Of course in Project 2010+ any change to a field and project highlights in light blue all changed cells. So as long as you don't make another edit, all tasks with one or more cells filled with a light blue color were affected by the last edit.

If you are only interested in changing dates, then you can save to Start1/Finish1 to 10 as well as a baseline before making significant edits.

I always save to a particular baseline every week before doing the weekly update. That lets me see changes every week.

Unfortunately, I don't have that trick up my sleeve.

When you enter data in the Source Manager dialog, Word inserts it into an XML file named Sources.xml. As with all XML, the pieces of data are surrounded by tags, and which tags are used depend on what kind of source you choose in the dropdown. Here are two examples, for a book and for a web page:

</b:Author></b:Author>

<b:Year>2013</b:Year>
<b:City>Redmond</b:City>
<b:Publisher>Microsoft Press</b:Publisher>
</b:Source>

<b:Source>
<b:Tag>Sco16</b:Tag>
<b:SourceType>DocumentFromInternetSite</b:SourceType>
<b:Guid>{824831FC-36D6-46E1-8A2B-3B83D4D1E6EB}</b:Guid>
<b:Title>Scott Adams' Blog</b:Title>
<b:Year>2016</b:Year>
<b:Author><b:Author>

</b:Author></b:Author>

I don't know of any macro or program that can convert an ordinary list of citations into this format. That's not to say it couldn't be done, but I didn't find one by searching the Internet.

You could perhaps speed up the use of the New Citation dialog by opening the list of citations in another window and copying/pasting the data from it to the boxes in the dialog. You can't do this with two Word windows, because the dialog is "modal" -- it won't let you select anything in any Word document as long as the dialog is open. You can open the list in WordPad while the New Citation dialog is open in Word, and work between the two programs.

The simplest way to do formatting in Word is through templates and styles.

wrdApp.Document.Add Template:= "MyTemplate.dotx"

ambious say:

I found a nice little guide that helped me make it work.  What you need to do is this:

  1. Download the Intel driver from their website, but download the ZIP version
  2. Unzip to any folder, and find a file called "igdlh64.inf"
  3. Edit the file, and look for the line that says "Windows 8.1 Install" (enclosed by two lines of "===").
  4. Right under that line, change “NTamd64.6.3″ to “NTamd64.6.4“.
  5. Install the driver manually through control panel, directing it to the inf file directly (you might need to disable driver verification for it to work).

For more detailed information on how to do any of those things, check out the guide I got this method from, here: http://sertacozercan.com/2015/02/how-to-install-latest-intel-graphics-drivers-on-windows-10-technical-preview/

dajhorn say:

Intel only ever released Windows 7 32-bit drivers for the GMA 3600 family before discontinuing the hardware.   These drivers are incompatible with Windows 8, Windows 10, and all 64-bit operating systems.

The only solution is to run the GMA 3600 in generic VESA mode.  The GMA 3600 has a non-Intel 3rd-party GPU core, so driver kludges that might work for other GMA branded parts are ineffective.
