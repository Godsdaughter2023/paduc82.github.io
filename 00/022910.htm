Sub Splitbook_New()
  Dim Sht As Worksheet

  'Be sure all formulas are up to date
  Application.Calculate
  'Screen off, runs faster
  Application.ScreenUpdating = False

  Application.DisplayAlerts = True
  Application.ScreenUpdating = True
End Sub

Private Function ValidFileName(ByVal FName As String, _
    Optional ByVal ReplaceChar As String = "") As String
  'Return a filename without invalid chars
  'Avoid CON, PRN, AUX, NUL, COM1 to COM9, LPT1 to LPT9 as filename
  Const InvalidChars = "\/:*?""<>|"
  Dim i As Integer, p As Long
  Dim Digit As String
  For i = 1 To Len(InvalidChars)
    Digit = Mid$(InvalidChars, i, 1)
    p = InStr(FName, Digit)
    Do While p > 0
      Mid$(FName, p, 1) = vbNullChar
      p = InStr(FName, Digit)
    Loop
  Next
  For i = 1 To 31
    Digit = Chr$(i)
    p = InStr(FName, Digit)
    Do While p > 0
      Mid$(FName, p, 1) = vbNullChar
      p = InStr(FName, Digit)
    Loop
  Next
  ValidFileName = Replace(FName, vbNullChar, ReplaceChar)
End Function

rob koch say:

Though I could dig around to provide you with links to the various information posted by Microsoft around the time that the original questions about the meaning of "Baseline" protection hit the net, I feel this is pointless since those unable to find them often actually do need better protection.

The key reason that Microsoft started using this term to describe their protection is that they had never intended to completely replace or try to put the 3rd-party security vendors out of business.  Their entire intent was to provide a baseline from which all other security products could be judged, since the complexity and lack of true protection provided by many products at the time was simply pathetic.  By creating a simple, but effective protection product themselves they hoped to force the other products to improve and provide real protection, rather than the more common method of the time of selling "features" instead.

However, during this time Microsoft also learned that a large part of the true issue was the 3rd-party testing, which since much of it was tied to the security vendors who either paid directly for their testing or advertised in the publications associated with the testing sites, made the testing itself questionable due to conflict of interest.  At first Microsoft attempted to pass these tests itself by following the testing groups rules, but eventually they realized that much of the testing was done using invalid batches of samples (old or corrupted) and in truth often didn't represent the real distribution of malware which Microsoft knew were most prevalent from their own malware collection telemetry.

The end result was that Microsoft finally decided to stop playing the "testing game" and instead focus their money on the real prevalent malware.  They knew this shift in focus would cause exactly your type of question, but since they didn't wish to "win" an anti-malware sales war and so put their better 3rd-party security partners out of business, they made the change in direction anyway.

In terns of true protection, none of the people I have using either MSE or Windows Defender on later versions of Windows have ever had significant issues with malware,  However, most of the people using these systems are either relatively intelligent and/or don't have children or other less mentally capable people using their PC.  This is important because the Microsoft security doesn't provide protection from certain adware and other software products that typically prompt a user during the installation of other often "free" products.  Avoiding these is usually easy if you actually read rather then just press OK, but quite a few people a few do this.

I disagree with Canadian Tech's preference for antivirus only, since this issue with adware and other less malicious junkware has grown to be a much larger problem for most everyday users.  However, if he encounters mostly more intelligent users like I have, this may not be a common issue for him either.  It's also clear from a quick review of the Bitdefender Antivirus Plus 2016 product that it's far more than basic antivirus protection, so he probably just doesn't recognize that practically none of those simple AV products have existed for several years now.

In over a decade of using both MSE and Windows Defender on various systems I've only experienced the occasional piece of drive-by malware on some less reputable websites, with browser hijacks being the only type that got past MSE since it leaves those to the browser security and they can typically be closed with an AltF4 or similar method.  Between MSE and the Windows built-in security I've never found a reason to pay for any protection since using Norton up until 2005, when Windows Live OneCare first became available.

The valid reason to use another product as I already inferred, is when children or other less capable people may be using the computer.  In the case where there may be less care or recognition that even lesser malware such as adware might be attempting to install, it may be valuable to find a more aggressive product than MSE with a broader range of protection.

Only you can judge whether this level of protection is required, either through your own past experience or your understanding of the others who use the PC in question.  I've personally never needed this, but have considered it in cases where older children have been involved.  To this point though it's never been required and MSE along with limited user accounts for children have been sufficient.

graham mayor say:
Word will attempt to grab any text that looks like it may be an address, so it will pick up short consecutive paragraphs. It will also apply any manual formatting included in those paragraphs. If you want to be sure it always grabs the right address, select the address first. To cut out this step format the address block with a unique style (I always use 'Inside Address' which was an old Microsoft standard that has stayed with me)  and use a macro to grab paragraphs in that style e.g.

Dim oPara As Paragraph
Dim oRng As Range
Dim strAddress As String
Dim i As Long, j As Long
strAddress = ""
If ActiveDocument.Paragraphs.Count >= 8 Then
    j = 8
Else
    j = ActiveDocument.Paragraphs.Count
End If
For i = 1 To j
    Set oPara = ActiveDocument.Paragraphs(i)
    If oPara.Style = "Inside Address" Then
        strAddress = strAddress & oPara.Range
    End If
Next i
MsgBox strAddress

Whether or not the insert address option inserts the Country seems dependent on the format of the address in Outlook contacts. If the country is displayed in Contacts it will be when the address book is used. This seems somewhat haphazard in practice.

It is probably simpler to use a macro to insert the address either at the cursor (as below) or to a bookmark or docvariable in the envelope template. This is more or less what I do in my add-ins, but in http://www.gmayor.com/InsertAddress.htm I give the user the flexibility to insert the address in a variety of formats.

Public Sub InsertAddressFromOutlook()
Dim strCode As String
Dim strAddress As String
Dim iDoubleCR As Integer
Dim oRng As Range

'Set up the formatting codes in strCode
strCode = "{<PR_DISPLAY_NAME_PREFIX> }<PR_GIVEN_NAME> <PR_SURNAME>" & vbCr
strCode = strCode & "{<PR_TITLE>" & vbCr & "}"
strCode = strCode & "<PR_COMPANY_NAME>" & vbCr
strCode = strCode & "<PR_POSTAL_ADDRESS>"

'Let the user choose the name in Outlook
strAddress = Application.GetAddress("", _
strCode, False, 1, , , True, True)
If strAddress = "" Then
    MsgBox "User cancelled or no address listed", , "Cancel"
    Exit Sub
End If
   
'remove United States of America if present
strAddress = Replace(strAddress, Chr(13) & "United States of America", "")

'There shouldn't be any, but eliminate blank lines by looking for two carriage returns in a row
iDoubleCR = InStr(strAddress, vbCr & vbCr)
While iDoubleCR <> 0
    strAddress = Left(strAddress, iDoubleCR - 1) & _
    Mid(strAddress, iDoubleCR + 1)
    iDoubleCR = InStr(strAddress, vbCr & vbCr)
Wend
Set oRng = Selection.Range
oRng.Text = strAddress
oRng.Style = "Normal" 'the style to apply to the inserted address
End Sub

Use a Createdate field for the date in your template.

A "bell curve" usually means a chart of the probability density function for the theoretical normal (Gaussian) distribution.

To get a bell curve in Excel, use the Average and StDev worksheet functions,  enter X values in a column and Y values in an adjacent column on the right where the Y values are NORMDIST(X,mean,stdev,FALSE), select the X and Y numerical values, and create an XY (Scatter) chart.

Instead, you actually may want a "histogram," which is a column chart showing frequency of data values in intervals of the X data. Excel has a Histogram tool in its Analysis ToolPak. Or, you could use my free Better Histogram add-in, available at
