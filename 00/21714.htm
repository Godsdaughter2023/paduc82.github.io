ken sheridan say: The HasData property of a subreport will return False if the subreport returns no data.  You can use this to hide any controls in the parent report with code in the Format event procedure of the section in which the controls are located:

    Dim blnShowCtrls As Boolean
    
    blnShowCtrls = Me.YourSubreportControl.Report.HasData
    
    Me.SomeControl.Visible = blnShowCtrls
    Me.SomeOtherControl.Visible = blnShowCtrls

Note that YourSubreportControl is the name of the control in the parent report which houses the subreport, not the name of its underlying report object, unless both have the same name of course, which is permissible, but not altogether advisable as it introduces a degree of ambiguity.

ken sheridan say:
However, the "on format" event apparently is only executed once per opening of a report.

Absolutely not.  The Format event procedure of each section executes for every instance of that section.  In fact it can, and often does, execute multiple times per instance of a section.

The fact that you mention the report's Cycle property makes me wonder, despite the fact that you refer to printing the report, whether you are opening the report in Report View.  The Cycle property only applies to Report View, which is a strange amalgam of a form and report, and determines what happens when the Tab key is pressed.  Report View uses a different set of event procedures to Print Preview or Normal view, these being akin to form events rather than report events.  In most situations there is no reason for using Report View that I can see.  I always change a report's DefaultView property to Print Preview.

I can see no reason why doing as I described should not achieve what you want when printing or previewing a report, regardless of how many invoices are being printed.
