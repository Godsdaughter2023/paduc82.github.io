What probably may have happened here is:

1. You have installed and using OWA App on your mobile device

2. You have enabled 'Contact sync' in its settings (see screenshots)

3. When 'Contact sync' is enabled, your contacts in your Exchange mailbox (in 'Contacts' folder) are synched locally to the device in its local Contacts folder

4. Now if a contact (Chunna Munna in my screenshots) is already present in the local Contacts folder (identical first & last names) then, the 2 contacts are linked and shown as one single contact by iOS Contacts app with the text in the Notes field that you have quoted above, i.e. 'This contact information was ....' (see screenshots)

5. In the end, what I wanted to convey here is that this may seems like a data loss issue but in reality it is not and your Notes data is still intact with the original contact of the same name and you can always access it thru linked contacts at the bottom of that contact. Hope this helps!

doug robbins - mvp office apps & services (word) say:

Try

Sub MergeCSV()
    Dim strSourcePath As String
    Dim strFile As String
    Dim Cnt As Long
    Dim r As Long
    Dim i As Long
    Dim j As Long
    Dim source As Workbook
    Dim target As Workbook
    Application.ScreenUpdating = False
    Set target = ActiveWorkbook
    'source folder
    strSourcePath = "C:\Documents\Invoices"
    If Right(strSourcePath, 1) <> "\" Then strSourcePath = strSourcePath & "\"
    strFile = Dir(strSourcePath & "*.csv")
    Do While Len(strFile) > 0
        Cnt = Cnt + 1
        Set source = Workbooks.Open(strSourcePath & strFile)
        r = target.Sheets(1).Range("A1").CurrentRegion.Rows.Count
        For i = 1 To source.Sheets(1).Range("A1").CurrentRegion.Rows.Count
            For j = 1 To source.Sheets(1).Range("A1").CurrentRegion.Columns.Count
                target.Range(A1).Offset(r, j - 1) = source.Range("A1").Offset(i - 1, j - 1)
            Next j
            r = r + 1
        Next i
        source.Close SaveChanges:=False
        strFile = Dir
    Loop
    Application.ScreenUpdating = True
    If Cnt = 0 Then _
        MsgBox "No CSV files were found...", vbExclamation
End Sub

bernie deitrick say:

Because your lookup value is in a column to the right of your desired value, and you require an exact match, then you need to use this instead of VLOOKUP or LOOKUP

=INDEX($A:$A,MATCH("Contract",B:B,False))

This assumes, of course, that your dates are in column A, and Customer 1 is column B.

Better yet would be to use a database where each row is a data point:

Customer  Action   Date

Cust 1      Contract   1/5/18

etc.
