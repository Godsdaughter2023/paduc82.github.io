Q. If my customer asks me to upgrade a PC with new hardware components, when is a new operating system needed? When would the PC be considered "new"?

A. Generally, an end user can upgrade or replace all of the hardware components on a computer—except the motherboard—and still retain the licence for the original Microsoft OEM operating system software. If the motherboard is upgraded or replaced for reasons other than a defect, then a new computer has been created. Microsoft OEM operating system software cannot be transferred to the new computer, and the licence of new operating system software is required.

If the motherboard is replaced because it is defective, you do not need to acquire a new operating system license for the PC as long as the replacement motherboard is the same make/model or the same manufacturer's replacement/equivalent, as defined by the manufacturer's warranty.

abrahambaker say:

I figured it out!

It turns out that the installer was failing because I had two different copies of windows 7 on different drives.  To get it to work, I had to go to the device manager and disable the drives that the OS to be upgraded was not on, AND I had to turn off the computer and unplug those drives.

Once I removed the drives, I was able to upgrade to win 10 successfully using the media creation tool cd.  I was also able to plug in the other drives again after the upgrade and everything worked as before!

I don't think this issue has anything to do with multi-booting with linux since the other computer I mentioned earlier upgraded without needing any drives removed (since it was win7 + ubuntu on only one drive).  The problem only happens when there are multiple windows 7 partitions among the connected drives.

I imagine I'll have to unplug the other drives again when I bother to upgrade the other windows 7 :)

Hope this helps people!

One more thing I found out (the hard way): To be able to use any ntfs drives in another OS after rebooting from Win10, you need to disable hibernate by opening a terminal with admin privileges and type

powercfg -h off 

, which unlock the other drives, letting them use them in other operating systems.  Here's a source:

http://www.tenforums.com/tutorials/2859-hibernate-enable-disable-windows-10-a.html
