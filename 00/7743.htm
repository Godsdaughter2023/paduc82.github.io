kristine arp say:

Hi Nancy,

 

 

If you can still access your old email, you can export your emails, contacts, and calendars from the old email address and import it to your new email address. To export your data files from the old email, follow the steps below:

 

1. Log in to Outlook using your old email address.

2. At the top of your Outlook ribbon, click File.

3. Choose Options.

4. Select Advanced from the left menu.

5. Under the Export section, click the Export button.

6. Click Export to a file, and then hit Next.

7. Select Outlook Data File (.pst), and then click Next.

8. Select the name of the email account that you want to export.

9. Check the box beside Include subfolders, and then click Next.

10. Click Browse and select where you want to save the Outlook Data File (.pst).

11. Type your preferred file name, and then click OK.

Please take note that if you have already used the export feature, the previous folder location and file name will appear. If you want to create a new file, change the file name of the existing file. If you are exporting to an existing file, under Options, specify what to do when exporting items that already exist in the file.

12. Click Finish.

 

To import your data files to the new email address, follow these steps:

 

1. Log in to Outlook using the new email address.

2. Click File.

3. Choose Open.

4. Click Import.

5. In the Import and Export Wizard, click Import from another program or file.

6. Click Next.

7. Select Outlook Data File (.pst), and then click Next.

8. Click Browse and choose the .pst file you want to import.

9. Click Next.

Note: If you want the imported information to replace the duplicate items in Outlook, Choose Replace duplicate items imported under Options.

10. Enter your password if prompted.

11. Set the options for importing items.

  • Choose Filter if you want to only import certain emails. Choose the More Choices tab if you want to only import emails that are read or unread.
  • The default settings usually don’t need to be changed. The top folder — usually Personal Folders, Outlook Data File, or your email address — is selected automatically.

12. Select one of the destination options:

  • Import items into the current folder - this imports the data into the folder currently selected.
  • Import items into the same folder in - this imports the data into the destination folder of the same name as the source folder, e.g., from Inbox to Inbox. If a folder doesn’t exist in Outlook, it will be created.

13. Click Finish.

 

Let us know if you need additional assistance.

jllatham say:

With your RAM pretty much maxed out on that computer there's not much you can do about that.

"the process of calculating cells that are dependent on previously calculated cells continues even if the value of a cell does not change when it is calculated"

I interpret that to mean that if you have a cell with this formula in it:

=A5+B5

and let's say that A5 has this formula in it = A1/A2  and at one point A1=10 and A2=5 (with result of 2 in A5), then you make a couple of changes, changing A1 to 8 and A2 to 4, giving the same result of 2 in A5, then A5 and our target cell are all going to get recalculated 2 or 3 times -- once for the change from 10 to 8 and again for the change from 5 to 4,

I don't see a lot you could do about that - even if you made your target formula go back to the original as:  =(A1/A2)+B5 you still end up with it recalculating with the changes to A1 and/or A2.

You simply have a boatload, no, make that a container ship full of formulas to calculate.  As I noted earlier, on that one sheet you've got a million plus.  It simply takes a finite time to calculate them and while that finite time is extremely small at the individual formula level, the sheer volumn of them adds up to a lot of time taken.

You could throw hardware at the problem: moving to a faster computer.  But in my estimation that's usually a poor return unless you're just looking for an excuse to buy a new one.  I will say that with Excel 2003 you have the fastest version of Excel of the most recent 3 versions (i.e., moving to Excel 2007 or 2010 will NOT improve performance for you).

You might want to consider picking up a copy of FastExcel from the DecisionModels page and digging into your workbook using it.  I won't say that FastExcel is that easy to interpret results from, you just have to use it, collect data and then figure out how to improve bottlenecks it may point out to you.
