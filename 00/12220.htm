ronrosenfeld say: I use a User Defined Function for this calculation.  It works by counting the number of "end of month" dates between the starting and ending dates; 1/12 of that is the number of years; and the last "end of month" date is used to compute the remaining # of days.

We can quibble a bit as to how to count where the starting date is Feb 29.

To enter this User Defined Function (UDF), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this User Defined Function (UDF), enter a formula like

=DateIntvl(A1,A2)

 in some cell.

============================================
Option Explicit
Function DateIntvl(d1 As Date, d2 As Date) As String
'Note that if d1 = 29 Feb, the definition of a year
'may not be the same as the legal definition in a
'particular locale
'Some US states, for some purposes, declare a
'leapling's birthday on 1 Mar in common years; England
'and Taiwan declare it on Feb 28
Dim Temp As Date
Dim i As Double
Dim yr As Long, mnth As Long, dy As Long
Dim sOutput() As String

Do Until Temp > d2
    i = i + 1
    Temp = DateAdd("m", i, d1)
Loop

i = i - 1
Temp = DateAdd("m", i, d1)

yr = Int(i / 12)
mnth = i Mod 12
dy = d2 - Temp
   
ReDim sOutput(0 To -(yr > 0) - (mnth > 0) - (dy > 0) - 1)
i = 0
If yr > 0 Then
    sOutput(i) = yr & IIf(yr = 1, " Year", " Years")
    i = i + 1
End If
If mnth > 0 Then
    sOutput(i) = mnth & IIf(mnth = 1, " Month", " Months")
    i = i + 1
End If
If dy > 0 Then sOutput(i) = dy & IIf(dy = 1, " Day", " Days")
   
DateIntvl = Join(sOutput, ", ")

End Function
===================

by default web browsers ignore scripting errors.... to debug web pages you need to display the dev tool (f12), select the debug tab and turn on error handling, eg. Break on all exceptions. The same is true for other browsers.... just because an error appears when you are using IE and not when you are using another browser does not mean that the same error or other errors are also occurring in the other browser...

MSIE browsers support legacy browser emulation modes... the scripting error...for matchMedia, indicates that on MSIE browsers bing uses a lower emulation mode that does not support 'Media queries'.

you can find out what IE Emulation mode is being used for a web site, and how it was established from the f12>Emulation tab.

I expect that on your computer it will show... IE7 - user compatibility view list.

To correct your IE settings for compatibility view for bing.com (or bing.com/maps) go

Tools>Compatibility view settings....

Remove any websites you have listed in your list.

Check the option - Include updated website lists from Microsoft.

Save changes....

Close and re-open IE for the changes to take affect (you must also be connected to the internet)

If you have not already done so.... run windows update to download an install the latest updates and security patches for your windows software... these include updates for your version of IE.... In the latest versions, the user compatibility view list has been disabled, which prevents novice users from accidentally adding websites to their private lists.
