Formula result shows decimal value at 13th+ place - Microsoft Community

bryanthompson say:

Formula result shows decimal value at 13th+ place

I have spreadsheets that give the the answer from a formula that should be 0.00 as (0.00).  When I expand the decimal point out to 30 places, it will show a decimal value starting at around the 13th place.  The formula does not rely on values derived from division, percentage, NPV, or anything that could create a number with an extended decimal.  I am using Excel 2000 but it gave me the same problem when I opened the file in Excel 2010.

Is there anyway to fix this without using ROUND or checking the box for "Precision as Displayed"?

 

Thanks,

Bryan

joeu2004 say: Bryan wrote:

I have spreadsheets that give the the answer from a formula that should be 0.00 as (0.00).  When I expand the decimal point out to 30 places, it will show a decimal value starting at around the 13th place.
[....]
Is there anyway to fix this without using ROUND or checking the box for "Precision as Displayed"?


In a word, no.  Consider this example:  IF(10.1-10=0.1,TRUE).  It displays FALSE(!).

The root cause of the problem is that most non-integers cannot be represented exactly in 64-bit binary floating-point, which is what Excel uses to represent numbers and perform arithmetic.  This leads to various calculation anomalies.

The only remedy is to explicitly round arithmetic expressions that you expect to be accurate to a specific number of decimal places, e.g. dollar-and-cents calculations.

An alternative is to set "Precision as displayed" (PAD).  However, that is not a panacea.  It won't help correct the formula above, for example [1].

-----
[1] The formula above is corrected as follows:  IF(ROUND(10.1-10,1)=0.1,TRUE).
