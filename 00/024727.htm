VBA macro to hide/unhide rows based on values in a range

Hi!

What I'd like to do, is for excel to automatically (without calling the menu to run the macro) apply the following to a certain worksheet:

 if a cell D1 = "PL" (it's a dropdown menu)

--> I want to check in Col. G for #N/A error and hide all respective rows

      I also want to unhide rows when data in column G changes

       and when selection in D1 is other than "PL"

Please, help! :)

hansv mvp say:

Right-click the sheet tab.

Select View Code from the context menu.

Copy the following code into the worksheet module:

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim m As Long
    Dim rng As Range
    Dim cel As Range
    If Not Intersect(Me.Range("D1"), Target) Is Nothing Then
        Application.EnableEvents = False
        If Me.Range("D1").Value = "PL" Then
            Set rng = Range("G:G").Find(What:="*", SearchOrder:=xlByRows, SearchDirection:=xlPrevious)
            If Not rng Is Nothing Then
                m = rng.Row
                Set rng = Nothing
                For Each cel In Me.Range("G2:G" & m)
                    If Application.IsNA(cel) Then
                        If rng Is Nothing Then
                            Set rng = cel
                        Else
                            Set rng = Union(rng, cel)
                        End If
                    End If
                Next cel
                If Not rng Is Nothing Then
                    rng.EntireRow.Hidden = True
                End If
            End If
        Else
            Me.Cells.EntireRow.Hidden = False
        End If
        Application.EnableEvents = True
    End If
End Sub

Remark: you must save the workbook in a macro-enabled format (.xlsm, .xlsb or lxls, but not .xlsx), and you must enable macros when you open the workbook.

Prevent pages from printing? - Microsoft Community

skateboarder873 say:

Prevent pages from printing?

Hello,

I have a document that is ~200 pages.  Pages 1-50 are the Body, and the remaining 51-200 are an appendix.  Is there a way that, if the user goes to File--> Print (or any other method of printing), to not include pages 51-200?

There could be a few ways - maybe have a message box notifying the user that there are 150 pages of Appendices, and to select only 1-50 for the main document.  Or, have the default print settings have print 1-50 only....etc.

Basically, I just want to prevent the user from going to File->Print (or CTRL+P, or some other way) and then print 200 pages when they really just want the 1-50 pages.  I have read about maybe using "hidden text" but I would like the user to be able to see the text when reading the digital document - I just want to prevent them from printing all of the document accidentally.

Even if I broke it into sections, that could still print the whole thing.  I am familiar with Macros/VBA (via Excel) if that helps.  Thanks for any ideas!

jay freedman say:

Any of the kinds of intervention you proposed will require macro programming.

The simplest (but probably not acceptable) method is to mark the text in pages 51-200 as Hidden, and have an AutoOpen macro that sets values for two options: ActiveWindow.View.ShowHiddenText = True and Options.PrintHiddenText = False. There are a couple of "gotchas" here -- all of the hidden text will be displayed with a dotted underline, and the user's original value of PrintHiddenText should be stored and re-set in an AutoClose macro. Another detail is that you probably can't refer to ActiveWindow directly in the AutoOpen macro because the document isn't open yet; instead, use an OnTime function call in AutoOpen to run a separate macro to set the option.

For anything else, you need a DocumentBeforePrint event handler in a class module, as described in this article. Code examples are given in this article and this article. (Note that starting in Word 2010, a macro named FilePrint no longer intercepts the Print command.) When the user clicks File > Print or presses Ctrl+P, etc., the DocumentBeforePrint code runs first.

In the handler, you can work in one of these ways:

  • If the active document is your document, just fire an ActiveDocument.PrintOut command with the arguments set up to print pages 1-50. Set the Cancel argument of the handler to True so the Print dialog (or Backstage page) doesn't appear.
  • Have the handler declare a Dialog object and set it to Dialogs(wdDialogFilePrint). Set that object's properties as .Range = wdPrintRangeOfPages and .Pages = "1-50", call the object's .Show or .Execute method, and set the handler's Cancel argument to True.

Putting code into a document rather than a template has a couple of implications. First, if the document is in the newer file format, it must be saved as a .docm (macro-enabled) document. Second, the recipient's macro security options must be set to enable macros to run, or the document when received must be unblocked (in the file's Properties dialog) and stored in a trusted location.
