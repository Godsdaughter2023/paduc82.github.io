I appreciate your response. I asked my Q because I read that access db can store only 2 gigs of data. I heard from a programmer that it would be better if my current db is in ADO recordset if I want later on to convert it to SQL server!

That statement doesn't make sense.  Data is not stored in an ADO Recordset.  It is stored in the tables, regardless of which database you use - Access, SQL Server, Oracle, MySQL, etc.  Recordsets are used to manipulate and transverse data that reside in the tables.  But you don't store data in an ADO Recordset.  So either you misunderstood what the programmer told you or he doesn't understand databases.

hansv mvp say:

I asked my Q because I read that access db can store only 2 gigs of data. I heard from a programmer that it would be better if my current db is in ADO recordset if I want later on to convert it to SQL server!

ADO is not a database system but a way to work with data in code (for example VBA code). An ADO recordset is only temporary - it "lives" in the memory of your computer, and it vanishes at the end of the session. You can't store data in ADO permanently - you need a database such as an Access database or SQL Server database for that.

 

So ADO is not an alternative for Microsoft Access, but an addition to Microsoft Access.
