If the goal is to maximize the protection for your PC, then it makes sense to pick an AV app with a higher-than-average malware detection rate for your primary (real-time) protection; and then take advantage of Defender’s Limited Periodic Scanning feature in order to supplement your primary coverage. If you take security seriously, then it’s hard to argue with the basic principle that more is better – or with the corollary that adding coverage is better than removing it:

https://www.tenforums.com/tutorials/51514-limited-periodic-scanning-turn-off-windows-10-a.html

https://answers.microsoft.com/en-us/protect/wiki/protect_defender-protect_scanning/limited-periodic-scanning-in-windows-10-to-provide/3dcfc8dd-36b4-43f2-9a0a-732e43191cf3

That’s why my personal recommendation is to extend the principle of limited periodic scanning to include periodic scans with additional on-demand malware-removal tools in order to cover a broader area of the malware spectrum. For enhanced protection from spyware, PUPs, and emergent threats – scan periodically with the free version of Malwarebytes. To maximize your AV coverage – scan periodically with the Kaspersky Virus Removal Tool:

https://www.malwarebytes.org/antimalware/

http://support.kaspersky.com/viruses/kvrt2015

You should also expand Windows Defender’s built-in coverage by enabling PUA protection: Copy, paste, and run (enter) this command at the elevated (administrator) PowerShell prompt:

Set-MpPreference -PUAProtection 1

And of course, for high-risk PCs, consider further supplementing your coverage by scanning periodically with some of these other trusted malware-removal tools:

https://www.bleepingcomputer.com/forums/t/407147/answers-to-common-security-questions-best-practices/?p=2413189

Note that all of these malware-removal tools are on-demand scanners – which means that they won’t interfere with the operation of your primary AV app. And the only time that they’ll ever use any of your PC’s resources is when you’re actually running a scan – so they won’t put a load on the operating system like adding a second real-time (on-access) AV/Antimalware app would.

Hi TarrGR,

Currently, it is not feasible to stop users from dropping files. SharePoint Will only show a yellow warning like this after they dropping the file and the info pane will open automatically to let users fill in metadata:

A workaround I can think of is to enable Check in and Check out. Then users will need to check the files in when they upload files and if the required info is blank, they will get a warning like this:

First, users will need to check out when they need to edit the file, and they cannot edit the same file at the same time since each file can only be checked out by one user at a time. Check out, check in, or discard changes to files in a library

Top questions about check out, check in, and versions

benduan say:

This is my understanding after playing with this for a bit. You can only use the primary alias to log onto Windows 10 machine or associate with the existing local login.

Adding other alias into the Other user list is possible, but logging in Windows will say the password is wrong, despite the fact that you can use the alias and the same password to log into Microsoft account or Outlook.com email account online.

Creating a local account and then trying to link to the alias address is also not possible as the system will detect the primary alias and try to use it instead. If the system already has the primary alias login setup, it is simply not possible to setup another login with the same address.

Indeed Microsoft should look into providing more flexibility for the Microsoft account, especially for family purpose. It may be easier and more cost-effective for a family to share one Microsoft account with multiple alias setup.
