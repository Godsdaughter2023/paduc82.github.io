GhostFairy wrote:

I must be wrong about the * stands, I thought it's like:

You are correct about the equivalency (in some contexts), and we do sometimes use multiply ("*") to act like "and" in some contexts (notably, array expressions).

That does not mean that "*" means "and".  And IMHO, we should use AND() whenever possible, if only for clarity.

But in this case, it would be more efficient to write the following, especially if A2:A20 becomes much larger [1]:

=IF($A2<>"", COUNTIF($A$2:$A$20,$A2)>1)

That avoids the COUNTIF when A2="" is TRUE.

Nevertheless, to explain....

Your original problem is:  you do not have (formulaA)*(formulaB), where formulaA is =COUNTIF($A$2:$A$20,$A2)>1 and formulaB is $A2<>"".  For that purpose, you should write:

=(COUNTIF($A$2:$A$20,$A2)>1) * ($A2<>"")

The problem with your original formula is:  it is evaluated as if it were written as follows:

=(COUNTIF($A$2:$A$20,$A2) > (1*$A2)) <> ""

The expression COUNTIF(...)>A2 is TRUE or FALSE, depending on the values of COUNTIF and A2.  And TRUE<>"" and FALSE<>"" are both TRUE.

In contrast, the correct use of parentheses (my first formula) is evaluated as follows:  COUNTIF(...)>1 is TRUE or FALSE, and A2<>"" is TRUE or FALSE.

The result is TRUE*TRUE, TRUE*FALSE, FALSE*FALSE or FALSE*TRUE.  When TRUE and FALSE are used in an arithmetic expression (multiplication), they are treated as 1 and 0 respectively.  Thus, the result is 1, 0, 0 or 0 respectively.  It is indeed 1 only when both logical expressions return TRUE, the same as AND().

Note that your corrected formula returns a numeric value, not TRUE or FALSE, like the AND() function.  However, non-zero and zero values are treated as TRUE and FALSE respectively when they are used in a logical expression like:

=IF((COUNTIF($A$2:$A$20,$A2)>1) * ($A2<>""), TRUE)

How to migrate mailboxes from one Office 365 tenant to another
