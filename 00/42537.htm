¡firedog say:

I've found the Storage Folder in C:\Users\User\AppData\Local\Microsoft\Windows Live Mail, but there are several mail folders missing from this location, ...

1) if I have these folders active, accessible and working in the WLM program, why are they not in the WLM storage folder?

This is because the folders you see in the program interface are virtual folders, bearing only a superficial resemblance to the ones you see in the file system. The store folder (the one called Windows Live Mail) is one big message database, with an index file (Mail.MSMessageStore) that keeps track of which messages are where. Other files in the folder with names like edbhhhh.log contain details of recent transactions (e.g. messages received, new folders created, messages moved). These log files are replayed in quiet moments, at program shut-down and start-up until all the transactions they contain have been written to the index file. Until this process is complete, you will see the sort of discrepancies you describe. It's not normally necessary for the user to concern himself with the store folder, which is why it's hidden by default.

2) How do I refresh the folder to reflect the actual emails?

Leave the program alone to get on with indexing messages in their new locations, then close down the program and wait a few minutes before re-launching it. You should see a change in the number and names of the edbhhhh.log files after each program launch/close cycle as they give up their secrets and die.

4) If all else fails, how can I back them up?

Use the Export messages function to back them up, then import at the other end. You'll find some helpful tips in this article: Export and import messages

I understand that you are not able to do backup and creating a system image, getting an error “A shadow copy could not be created. Please check "VSS" and "SPP" application event logs for more information. The specified object was not found. Error code: 0x81000019”.

However I would require some more information to assist you better.

1. Which security software are you using?

Usually this error come when 100 MB hidden System Reserved partition created by Windows 7 does not have enough space to store a shadow copy. Please check the free space on this volume.

Follow the following steps:

1. Click on Start, right click on Computer and then click on Manage.

2. Computer Management Window will be open.

3. Click on Disk Management (under Storage),


3. Verify that you have a System Reserved partition of 100 MB size.


4. If yes, right click on this volume and select Properties.

5. View used and free space.

Windows backup requires around 40 MB of free space on this partition to backup successfully. So if the free space is less than 40 GB, backup does not get enough free space. As a workaround try the following solution:

Since the default System reserved partition cannot be extended, creating a new system volume is the workaround. Here are the steps to move the system volume to any other volume.

Create a new volume (say F:) on the same disk that contains the boot partition of size of about 490 MB (be careful to keep it less than 500 MB).

To do this you must be in the manage properties as per the steps above.

1. Click on Start, right click on Computer and then click on Manage.

2. Computer Management Window will be open.

3. Click on Disk Management (under Storage), then right click on the C:\ drive and click on Shrink Volume.

4. Type 490 (against "Enter the amount of space to shrink in MB" option) and click on Shrink.

5. Right click the new drive and click on New Simple Volume.

Note: Assuming Windows 7 is installed on C: on your machine. From an elevated command-prompt run: bcdboot.exe C:\Windows /s F:.

To do so follow the steps given below:

a. Click Start, type cmd in the start search box.

b. Right click on cmd from the search list and select run as administrator.

c. In the Command Prompt Window type bcdboot.exe C:\Windows /s F: and press Enter.

d. Now try to run the DISKPART command. Type DISKPART and press Enter.

e. Type select volume F and press Enter.

f. Type active and press Enter.

g. Now you can reboot your machine and F: will become the system volume. You can undo this by repeating the same steps using the original system volume (you have to assign drive-letter to it) to revert to your previous configuration.

h. Now try to do the backup.

For information, refer the link:
