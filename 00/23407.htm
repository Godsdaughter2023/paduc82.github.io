And if neither condition is true?

Perhaps the following in B1:

=IF(A1="","",IF(ISNUMBER(A1),A1+3,""))

probably formatted as Date.

Note:  If you truly want B1 to appear blank when A1 is not a date, the formula can be simplified, namely:

=IF(ISNUMBER(A1),A1+3,"")

Also note that in general, we cannot distinguish between an Excel date and any other number.  An Excel date (only; no time component) is simply an integer starting at 1.  Formatting makes it appear like 1/17/2012 or whatever.

However, if you want at least to ensure that it is an integer starting at 1, we could write:

=IF(ISNUMBER(A1),IF(AND(A1>0,A1=INT(A1)),A1+3,""),"")

PS:  There is also a maximum date of 12/31/9999, which Excel represents internally as 2,958,465.  So we could write:

=IF(ISNUMBER(A1),IF(AND(A1>0,A1<=2958465,A1=INT(A1)),A1+3,""),"")

But that might be overkill.

You can allow any operation that you desire, to pass through. Without disabling

is set in the Allow an App list, and it should now be functional.

dhookom say:  

Only looking at your first SQL query:

  • You should not create an alias or object name that is a reserved word. "NAME" is a reserved word that could cause issues. Consider using something more descriptive like "EmployeeName".
  • "Last" in an SQL statement rarely returns a consistent value unless you only have one value to choose from or don't care which value is returned.

Your IIf() statements (if you keep them) should return a consistent data type. "" is not a number and should be replaced by either 0 or Null.

 Your "buckets" of date ranges should be managed in data not in an expression.
 Create a table of day ranges:

 tblDaysBuckets
 FromDays  ToDays   DaysName
 0                  69           00-69
 70                79           70-79
 80                89           80-89
 90             9999          90+

SELECT dbo_employees.employee_no AS [ID#],
[last_name] & ', ' & [first_name] & ' ' & [middle_initial] AS EmployeeName,
dbo_employees.udf_Last_Drug_Screen_Date AS [Drug Screen Date],
DaysName
FROM dbo_employees, tblDaysBuckets
WHERE Date()-[udf_Last_Drug_Screen_Date] BETWEEN FromDays AND ToDays
GROUP BY dbo_employees.employee_no, [last_name] & ', ' & [first_name] & ' ' & [middle_initial],
dbo_employees.udf_Last_Drug_Screen_Date, DaysName
