ken sheridan say:
If the user want to set all the persons as selected then clicking the header part should select all records.  But it should not write in table until the user clicks Apply button.

That's not possible, at least if you want to show the proposed values in the rows, with a simple bound form as multiple rows in the form cannot simultaneously be in an uncommitted state.

A.  To update all rows you'd need to do the following:

1.  Include a bound Status control in the form's detail section, with its Visible property True by default.

2.  Include an unbound control in the form, superimposed exactly on the bound control, with its Visible property False by default.

3.  In the option group's AfterUpdate event procedure set the Visible property of the bound control to False; set the Visible property of the unbound control to True; set the value of the unbound control to Null.

4.  In the apply button's Click event procedure execute an SQL statement to update the Status column in the table to the value of the option group; call the form's Refresh method; set the value of the unbound control in the form to Null; set the Visible property of the unbound control in the form to False; set the Visible property of the bound control in the form to True.


B.  To update rows selectively on the basis of  a selection in the option group you'd firstly need to include a Boolean column, named UpdateMe say, in the table and a check box bound to the control in the form's detail section, with its Visible property False by default.  Then:

1.  In the option group's AfterUpdate event procedure set the Visible property of the UpdateMe  check box to True.

2.  Once the user has indicated via the check box which rows are to be assigned the value selected in the option group, in the Click event procedure  execute an SQL statement to update the value of the Status column in the table to the value of the option group WHERE UpdateMe  = TRUE; execute an SQL statement to update the value of the UpdateMe column in the table to FALSE; set the Visible property of the UpdateMe  check box to False

3. Call the form's Refresh method.

You will of course need a means by which the user can firstly elect to update all rows en bloc, or selected rows via the UpdateMe control.  This could be another unbound option group, an unbound check box or an unbound combo box.  Depending on the value of this control you would execute one or other of the two routines described above.

hansv mvp say:

Select the used range in column A.

Press F5 or Ctrl+G to activate the Go To dialog.

Click Special...

Click Blanks, then OK.

Note the address of the active cell within the selection, it's displayed on the left hand side of the formula bar. Let's say it is A3.

In the formula bar, enter the formula =A2 (referring to the cell above the active cell), then press Ctrl+Enter.

This will populate the blank cells in column A with the value of the identifier above them.

bob jones aka: cybertaz say:

When Table is applied to a range it is given the equivalent of a Range Name starting with "Table1". That name, however can be changed on the Table tab of the Ribbon.

Since they're treated as Named Ranges the name of Tables appear in the Name Box @ the left end of the Formula Bar.

As for the delete part of your question: It depends on exactly what you want to delete...

If you simply want to remove the Table effects/features use the Convert to Range tool on the Table tab. It also may be necessary to remove the formatting separately.

If you do want to delete the actual data you can do so just as with any other range of cells by selecting the rage & using Edit> Clear> All.
