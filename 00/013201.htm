lizeidsness say:

I figured it out!  I working on a different problem, with a web page field in a sharepoint list, and noticed interesting syntax when I converted the field to a string...

if you have to build a url field dynamically, the basic syntax is

displaytext#url#

so, in my dynamic field....

=Concat("Click here#/relative/path/on/my/sharepointsite/allitems.aspx&ID=",[ID],"#")

Your tip helped me to construct a conditional hyperlink.

=IIf(Len([VINTextbox])=17,Concat([VINTextBox],"#http://www.decodethis.com/VIN-Decoded/vin/",[VINTextBox],"#"),"")

jim gordon mvp say:

There are three distinct features you are asking about. Fixy Foxy incompletely addressed one of them. I'll try to at least touch on all three.

1. Track Changes.

This is a feature in Microsoft Word desktop. It is also a feature of the web application, but in the web application, track changes is a function of Word's Co-Authoring feature. In the desktop version you can use track changes with co-authoring or without.

For instructions on using Track Changes in the desktop version, go to the Help menu and search on Track Changes. You can see Help highlight the feature as show in this screen shot. Follow the link for Get help on Track Changes for detailed help on how to accept and reject changes.

If you are sharing documents via email, THIS is the feature set you should focus on.

2. Co-Author

You can share and edit a document live in (almost) real-time using Word's co-author feature. This is the feature Fixy Foxy referred you to. In this case the file resides on OneDrive. You will get tracked changes that work differently from the desktop track changes. In particular, to save changes the user must save the document. Then the owner can accept or reject the changes.
