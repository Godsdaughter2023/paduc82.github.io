The following was enough here:

odoc.Variables("LastName").Value = FirstNameTextBox.Text

A few comments:
 1. Although some people here operate in both groups, I think you are much more likely to get assistance with VB.NET + Word in the MSDN groups - in this case, since you're using automation,
http://social.msdn.microsoft.com/Forums/en-GB/worddev/
If you use VSTO (not sure that is possible from the Express version of VS) there's another group at 
http://social.msdn.microsoft.com/Forums/en-GB/vsto

2. Even in VBA, you're probably better off using the document object you're just created ("odoc") rather than "ActiveDocument", wherever possible.

3. In VB.NET, you generally need to spell stuff out more. Assuming you're writing a standard Windows Forms application, it's more like automating Word from Excel, i.e. the VB.NET compiler doesn't understand "ActiveDocument" unless you qualify it. I you really want "ActiveDocument", then you need
oWord.ActiveDocument
Another example is that in this case, although it's using exactly the same COM engine as VBA would, I wasn't completely sure that you would not have to use Variables.Add etc. to cope with the situation where the Variable does not already exist. But that bit seems OK. 

4. I guess you'll want 
odoc.Variables("LastName").Value = LastNameTextBox.Text
but you are doubtless still in "experimental mode" :-)

mcyin say:

Finding words from one document in another document

Hello,

I have a list of terms in document A. I would like to know what words on my list in document A are used in a paper I'm reading, document B. Is there a way to have Word highlight my document A words in my document B text?

Thanks for any help!

daniel g. grau say:

Try this:

  1. Create a file with the list of words, one word (or phrase) per paragraph, and save it.
  2. Go to the other file (using a copy would be safer).
  3. In the main menu bar, select Insert > Index and Tables.
  4. Make sure to click on the Index tab at the top of the dialog.
  5. Click on Automark and select the file containing the list of words. Word will add some cross-entry fields.
  6. Go to Word > Preferences > View and place a mark con Field codes (or press Option F9 or Option Fn F9, depending on your System Preferences settings). This will show the { XE } fields added in step 5.
  7. In the Ribbon's Home section, select a Highlight color, like the customary yellow.
  8. In the main menu bar, select Edit > Find > Advanced Find and Replace.
  9. In the "Find what" field type ^d (caret d), which stands for "field." This will find the cross-entry fields.
  10. Click on the "Replace with" field, clear it (use a blank field), but in the bottom-center dropdown menu (Format) select "Highlight." If you don't see such dropdown menu, click on the triangle so it points up.
  11. Click on Replace All. This will highlight all instances of fields. (It will highlight other fields as well, like TOCs, if they exist.)

You could, as a next step, insert an Index (Insert > Index and Tables, select the Index tab, press OK), and that will gather all words (on a per-page basis) at the end of the document. If that's not enough for your purposes, let me know what else you need and we'll figure out some additional steps.

macropod say:

Try the following macro. Open both documents and make the document containing the comparison words the active one:

Sub CommonalityHighlighter()
Application.ScreenUpdating = False
Dim StrFnd As String, StrTmp As String, StrExcl As String
Dim i As Long, j As Long, k As Long, l As Long, m As Long, Rng As Range
'Define the exlusions list
StrExcl = "a,am,an,and,are,as,at,b,be,but,by,c,can,cm,d,did," & _
          "do,does,e,eg,en,eq,etc,f,for,g,get,go,got,h,has,have," & _
          "he,her,him,how,i,ie,if,in,into,is,it,its,j,k,l,m,me," & _
          "mi,mm,my,n,na,nb,no,not,o,of,off,ok,on,one,or,our,out," & _
          "p,q,r,re,s,she,so,t,the,their,them,they,this,t,to,u,v," & _
          "via,vs,w,was,we,were,who,will,with,would,x,y,yd,you,your,z"
With ActiveDocument
  'Get the document's text
  StrFnd = .Content.Text
  'Strip out unwanted characters. Amongst others, hyphens and formatted single quotes are retained at this stage
  For i = 1 To 255
    Select Case i
      'To strip out numbers, delete ', 58'
      Case 1 To 35, 37 To 38, 40 To 43, 45, 47, 58 To 64, 91 To 96, 123 To 127, 129 To 144, 147 To 149, 152 To 162, 164, 166 To 171, 174 To 191, 247
      StrFnd = Replace(StrFnd, Chr(i), " ")
    End Select
  Next
  'Delete any periods or commas at the end of a word. Formatted numbers are thus retained.
  StrFnd = Replace(Replace(Replace(Replace(StrFnd, Chr(44) & Chr(32), " "), Chr(44) & vbCr, " "), Chr(46) & Chr(32), " "), Chr(46) & vbCr, " ")
  'Convert smart single quotes to plain single quotes & delete any at the start/end of a word
  StrFnd = Replace(Replace(Replace(Replace(StrFnd, Chr(145), "'"), Chr(146), "'"), "' ", " "), " '", " ")
  'Convert to lowercase - omit this line if you need to do a '.MatchCase = True' Find
  StrFnd = " " & LCase(Trim(StrFnd)) & " "
  'Process the exclusions list
  For i = 0 To UBound(Split(StrExcl, ","))
    While InStr(StrFnd, " " & Split(StrExcl, ",")(i) & " ") > 0
      StrFnd = Replace(StrFnd, " " & Split(StrExcl, ",")(i) & " ", " ")
    Wend
  Next
  'Clean up any duplicate spaces
  While InStr(StrFnd, "  ") > 0
    StrFnd = Replace(StrFnd, "  ", " ")
  Wend
  StrFnd = Trim(StrFnd)
End With
With Documents(2).Range.Find
  .ClearFormatting
  .Replacement.Text = "^&"
  .Replacement.Highlight = True
  .Forward = True
  .Wrap = wdFindStop
  .Format = True
  .MatchCase = False
  .MatchWholeWord = True
  .MatchWildcards = False
  .MatchSoundsLike = False
  .MatchAllWordForms = False
  For i = 0 To UBound(Split(StrFnd, " "))
    .Text = Split(StrFnd, " ")(i)
    .Execute Replace:=wdReplaceAll
  Next
End With
Application.ScreenUpdating = True
End Sub

The macro also has an exclusion list, so that various words and phrases can be excluded. That list (defined by the words & phrases in the StrExcl string variable) holds the words and phrases to be omitted from the comparison. As you'll see, I've 'excluded' numerous common words and single letters. Any phrases should be inserted into the exclusions list ahead of any of the single-word exclusions (so that conflicts don’t occur). As coded, the comparison is done without regard to case. That can be changed, as indicated in the code comments.
