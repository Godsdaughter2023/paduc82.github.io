timliu_msft1261 say:

Hi Armands,

Adding custom DNS records options are available for the custom domain which DNS records are hosted at Office 365. For example, if we add a domain to Office 365 and let Office 365 manage its DNS records, we will get the options as the first screenshot shows in your post. For some organizations, they let Office 365 manage their custom domain DNS records. At the same time, the custom domains are used for public web sites outside of Office 365. In this situation, they need to add A records and CNAME records to Office 365. To learn more, see this article.

On the other hand, if the DNS records of custom domain are hosted outside of Office 365, we need to add Office 365 services related DNS records to the same DNS host. Other DNS records are also added to the same DNS host. For this scenario, we don't need to add custom DNS records to Office 365. So we get the setting page as the second screenshot in you post: No options for adding custom DNS records.

    Dim r As Long
    Dim c As Long
    Dim f As Boolean
    Application.ScreenUpdating = False
    For r = 5 To 3804

Skype only stores 30 days of history in the cloud. Before the cloud it was primarily less than a day.  Any "forever" history is stored in a history file located on your machine (main.db). If that file is deleted or corrupted then you will lose part or all of your history beyond 30 days. You can search your computers/devices for any main.db files to see if there is any older history you can retrieve.  You can view the contents of any main.db file you find using third party software like SkypeLogView [nirsoft.net] or export the history to a CSV (available in some Skype clients).
