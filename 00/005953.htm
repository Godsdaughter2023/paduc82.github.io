jay freedman say:
Sorry, marysully, the paragraph properties don't have anything to do with this.

Any document that is not in Compatibility Mode does not offer most of the layout options that were available in previous versions. This is by design, caused by the switch to a new layout engine that has many of the old rules built in.

The "Suppress extra space at top of page" option is one of those built in. It automatically suppresses the Space Before setting of the paragraph format or style when the first line of a paragraph is at the top of a page. It does not remove empty paragraph marks, and it also does not suppress the Space Above if the previous page ends with a Next Page, Even Page, or Odd Page section break.

The only way to gain access to the old layout options, if you still need to do that, is to put the document into Compatibility Mode so the old layout engine will be used.

The quick way to do that is to save the document in Word 2003 (*.doc) format. However, that also makes all of the features added in Word 2007, 2010, and 2013 unavailable. For reasons known only to Microsoft developers <sigh> it's a bit difficult to put a document into Word 2010 compatibility, so that you lose only the newest features of Word 2013. If you want to do that:

Press Alt+F11 to open the macro editor, and press Ctrl+G to open the Immediate window at the bottom of the editor. In that window, paste this line and then press Enter:

ActiveDocument.SetCompatibilityMode wdWord2010

Close the macro editor. Then open the Options dialog, go to the Advanced options, and scroll to the bottom. Change the dropdown "Lay out this document as if created in" to "Microsoft Word 2010". At that point, the list of 2010 layout options will appear below the dropdown, and "Suppress extra line spacing at top of page" will be in that list.

    hansv mvp say:
    A page is not an intrinsic entity in Word - as the document is edited, text may move to a different page. Create a bookmark for the text that you want to refer to, and insert a cross reference to the page number of the bookmark.

    jay freedman say:
    You misunderstood the "numbered items" option -- page numbers aren't "numbered items" for
    that purpose.
     
    Place a bookmark on the page you want to refer to, and then create a cross-reference to
    that bookmark. Choose "Reference type" as "Bookmark", and choose "Insert reference to" as
    "Page number", and the resulting cross-reference will show the page number of the
    bookmark.
