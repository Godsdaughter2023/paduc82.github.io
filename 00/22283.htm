jay freedman say:
A macro is usually overkill for this kind of task. The best way to do it is to create the whole footer in a custom template. Then every document you base on that template will start out with the footer already in place (although the filename field won't automatically update when you save the document, so you'll have to update it manually, most easily by going to the Print screen and back to the document).

Hint: for the date, use a CREATEDATE field instead of a DATE field. In the template it will show the date of the template's creation, but in documents based on the template it will show the date when the document was created -- and it won't change if the document is later reopened, the way a DATE field will.

The next best way is to create the whole footer in any document, select the entire line, and press Alt+F3 to open a dialog for saving it as an AutoText entry. Enter a name, set the Gallery dropdown to "Footers" and click OK. The entry will now appear in the Insert > Footer gallery, and you can insert it with a click.

Another hint: if you give the entry a name that starts with an underscore character, that will put it at the top of the gallery for easy selection.

jay freedman say:
If you still want a macro for some reason, you'll find it surprisingly difficult to deal with multiple fields in footer ranges. The following macro is an elaboration of one posted by Greg Maxey in this thread.

Sub AddFooters()
    With ActiveDocument.Sections(1).Footers
        AddOneFooter .Item(wdHeaderFooterPrimary)
        AddOneFooter .Item(wdHeaderFooterEvenPages)
        AddOneFooter .Item(wdHeaderFooterFirstPage)
    End With
End Sub

Private Sub AddOneFooter(ft As HeaderFooter)
    Dim rg As Range, rg2 As Range
    Set rg = ft.Range
    With rg
        .Text = "" ' remove anything already there
        .Fields.Add Range:=rg, Type:=wdFieldCreateDate, Text:="\@ ""M/d/yyyy""", _
            PreserveFormatting:=False
        .InsertAfter vbTab
        .Collapse wdCollapseEnd
        .Fields.Add Range:=rg, Type:=wdFieldFileName, Text:="\p", _
            PreserveFormatting:=False
    End With
   
    Set rg = ft.Range
    With rg
        .MoveStartUntil cset:=vbCr ' move rg to just before the paragraph mark
        .Text = vbTab & "Page PAGE of NUMPAGES"
    End With
       
    Set rg2 = rg.Words(5) ' the tab is rg.Words(1), count from there
    With rg2
        .Fields.Add Range:=rg2, Type:=wdFieldEmpty, _
            PreserveFormatting:=False
    End With
       
    Set rg2 = rg.Words(3)
    With rg2
        .MoveEnd wdCharacter, -1 ' leave a space between the PAGE field and 'of'
        .Fields.Add Range:=rg2, Type:=wdFieldEmpty, _
            PreserveFormatting:=False
    End With
   
    ft.Range.Fields.Update
End Sub

You can tweak this in various ways. For example, you might want the items in the even page footer to be in the reverse order of the others; that should be done in a separate subroutine to keep the logic from getting out of hand.
