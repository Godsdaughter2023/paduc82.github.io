joseph mcdaniel say: Well,

First I would warn you that converting the processor architecture of a windows installation requires a full re-installation of windows.

Then, I will assume you have a legitimate way of obtaining the installation files for an x64 edition of this software (in *.iso format as you have stated.)

In my experience, a product key will work regardless of architecture (x86 vs. x64) so long as the EDITION (Ultimate) is the same, so long as the Service Pack is the same (SP0 or SP1 on the original disk?), and so long as the disk uses the same activation mechanism.

The real question is, which activation mechanism is this Upgrade disk using? I would be very likely to think that it is using either a) a retail activation key or b) a Volume License MAK key.

I doubt it's oem, and I doubt it's Volume License with KMS, unless your home computer is setup with a Virtual Private Network into your school computer network.

I would recommend downloading the  Volume Activation Management Tool 2.0 (VAMT) from Microsoft at http://www.microsoft.com/en-us/download/details.aspx?id=11936. This download requires you to log into your Microsoft account.

Once you install this program, open it up and go to the last item on the left column which is called Product Keys. Then on the right side, enter the product key you have been using into the product key field and hit verify. If it goes through alright, it will tell you that it is valid, and you will know that it is a Volume Activation key. The remaining number of activations can be found by adding it to the list below it, and refreshing the information.

If it fails and tells you that it cannot use that product key, it is not a volume key, and must be a retail key.

From my experience, you'll need to make sure that whichever activation channel your key is for (VLK MAK or Retail) -- you'll need your x64 iso to match that activation channel. If it does not match, you will have activation issues.

One more warning though: If it is retail, and you have already activated it twice, you may have used up the available activations. If this occurs you would have to call the Microsoft Product Activation Hotline to activate your windows, which is never fun.

Good Luck!

Re: "If the duplicate files exist, it may be due to the corrupted files present in your system."

You completely missed the issue that this user (and more) have. After upgrade from Windows 8.1 to Windows 10, systems can have a OneDrive folder that shows in File Explorer but whose actual path is "C:\Users\name\SkyDrive."

After the Win10 upgrade, File History shows as backing up both

C:\Users\name\OneDrive 

and

C:\Users\name\SkyDrive

This is unnecessarily confusing and leads the user to think that the "SkyDrive" backup is not necessary as long as the "OneDrive" one shows in File History.

But if you check the actual system path to "OneDrive" using the command prompt, it has none of the files that are at the user's true OneDrive. Those are at the C:\Users\name\SkyDrive. It appears that this C:\Users\name\OneDrive folder holds some sort of placeholder of synch data.

As I said, it's unnecessarily confusing. Your reply to the user did nothing to answer the question or ease the concern.
