hansv mvp say:

SQL uses US date format mm/dd/yyyy regardless of your system date settings, so 09/06/2011 is interpreted as the 6th of September. For the 9th of June, you should use 06/09/2011. If you get the date from a variable or from a worksheet cell, you must use the Format function to format the date as mm/dd/yyyy:

 

strSQL = strSQL & " WHERE Date <= #" & Format(DateVariable, "mm\/dd\/yyyy") & "#"

 

or

 

strSQL = strSQL & " WHERE Date <= #" & Format(Range("K3").Value, "mm\/dd\/yyyy") & "#"

guessed say: Hi Jacquelyn

I had a look at your two demo docs and made a few changes and loaded a third into your skydrive. 

Since the recommendation is to do your spreads on a single page, I set up the landscape doc as a 2 column layout so you can put the image on the left and then add text on the right. I removed the page border you added since that eats into your available space and isn't necessary. If you waste page space on things like borders then your images will be smaller (potentially too small to see the finer detail) than they could be.

The graphic size you have used is near enough to that required for the purpose but you could optimise them to get exactly the right resolution if you didn't trust the Amazon conversion to preserve them correctly. At this stage lets just assume that we don't need to optimise the images but if you wanted to do this then you would need to set them up based on the proportion of the page that they are taking up. If the full page is 1024 x 600 then we are only using a bit less than half of that on the two column layout I applied (0.5cm off each edge). This means your target image size should be 560 pixels tall x 472 pixels wide. If you exceed these sizes then the image will be reduced to fit the space. The two images in your skydrive sample docs had different sizes but both were close-ish. One was 505x662 and the other was 488x656. If the ratio of height to width is not exactly the same then the graphic will leave space unused (either on sides or top/bottom) on your final product.

The CSS info that you were sent is no use in Word - it is only relevant if you are authoring in html. Don't worry about that until you are ready to test a doc by uploading to the Amazon docx converter.
