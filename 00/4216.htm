jannakumi say:

The Path Specified for the file Outlook.pst is not Valid

I can't open my Outlook without seeing the above message.  Not understanding the proposed solutions online as they are quite dated.  Anyone who can assist me in easy to understand directions to solve this problem?  Thank you!

karl timmermans (outlook mvp 2012-2018) say:
>>
#1 - I can't open my Outlook without seeing the above message
......
#2 - Not understanding the proposed solutions online as they are quite dated. Anyone who can assist me in easy to understand directions to solve this problem?
<<

Re: #1 - It means that a PST file has been configured in Outlook that can no longer be found at the specified location because it has either been moved or deleted (or become completely corrupted).

To view the PST files configured in Outlook, open the Control Panel <Mail> app --> click on <Data Files> button. Take note of the folder path/file name(s) of the PST file(s) on the list

Hard to comment on the "proposed solutions" you have viewed since no reference was provided. In terms of instructions being dated, might appear to be the case because nothing has changed in terms of how to deal with this.

Very simply, you need to

a) determine which PST file the error message applies to
b) verify whether or not that PST file physically exists at the location configured in Outlook
c) if it doesn't exist, you'll need to determine if the file has been moved to another location. If the file does not exist, for whatever reason, you'll need to restore it from a backup (if one exists) otherwise everything in the PST file has been lost.

d) determine if the PST file is associated with a POP3 account. Switch to the email tab in Account Settings, highlight each POP3 email account and view the delivery location (look down to the <Change Folder> button and take note of the Folder Path/File name of the PST file)

e) if this involves a PST file associated with a POP3 account and that PST file no longer exists and can't be restored from a backup then the simplest solution would be to re-configure the email account again. You'll need to remove the email account from the profile first and then add it back by "manually" configuring the account. If you re-configure the account just by entering the email address/password, Outlook will create an IMAP account by default if the email service provider (ESP) supports IMAP (which virtually all major ESP's do).
