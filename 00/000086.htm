jay freedman say:

There are two kinds of text content controls, the Plain Text control and the Rich Text control. You can tab out of a Plain Text control to the next content control (if there is one).

However, the purpose of a Rich Text control is to enable entry of anything you could put into a document, including tabs. So there is no tab-to-exit ability in a Rich Text control. (It would have been helpful to make the control behave like a table cell, where Tab exits the cell and Ctrl+Tab enters a tab character, but Microsoft hasn't provided that ability.)

When the content controls are in the last cell of a table, there is again a difference in the behaviors of Plain Text and Rich Text controls. In a Plain Text control, the tab key will jump to the next content control outside the table if there is one, and otherwise will do nothing.

In a Rich Text control in that last cell, apparently the table gets to act on the tab press before the content control gets it -- you always get a new row instead of a tab character inside the control. That could probably be considered to be a bug, but the 'fix' would most likely be to give priority to the control instead of the table, and the tab still wouldn't jump to the next control.

I'll note that none of this has anything to do with editing a template as opposed to a document. Ordinarily, Word makes no such distinction; editing is the same no matter where you do it. You could force a difference by writing some fairly complex macros, but I don't believe it would be worth the effort.
