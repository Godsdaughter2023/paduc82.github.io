    "Everything", http://www.voidtools.com/ , is a file name search tool (I have used it for about 10 years). It ties directly into the Window File System. It provides effectively instant file search and filtering as you type. So, if you know all or part of the file name, you can search for partial or full name matches, then figure out which specific file (by folder location) is the one you want to edit.

    You can use the built in windows Indexing and Search (upper right corner) in Explorer to search for text in some file types. Personally, I find it comes up with to many "false positives".

    Taking a step back, lets do a little computing philosophy:

    For your personal data files YOU control where they are saved.

    You define the folder structure. You don't have to use the same folder names.

    Personally, when naming FILES AND FOLDERS, I take full advantage of LONG FILE NAMES. (I started computing limited to 8.3 file names, so long names are a huge luxury).

    Make the names descriptive.

    Include one or more "tags" you can use group files when you do file name searches.

    I often include full creation date in the file name (file date attribute can change) for files that will be static.


    Putting tags at the start allow me to collect articles on the same category of topic just by doing a file name sort

    For example
    win10 feat 2018 05 01- How to use file explorer.docx

    The structure above allows me to group all of the files on windows 10 features by date. The more current the date, the more likely the file is still relevant.

    ie
    WTM Menu.docx
    WTR Tables.docx

    WTM = Word Tips Menu (group tag) for tips specific to menu versions of Word
    Menu.DOCX = This file of "random" tips organized in menu command order

    WTR = Word Tips Ribbon (tag) for tips specific for Ribbon versions of Word
    Table.docx = I had enough Ribbon version tips about the Table feature to break them out into a separate file


    Using prefixes I can force a simple desired sort order, regardless of descriptive names
    ie
    ET1 zjsa;fdjas.docx
    ET2 abc.docx

    Then again, after creating a descriptive file name I also use date suffixes for version control
    ie
    @Answers.docx
    @Answers 2016 01 01.docx
    @Answers 2017 01 01.docx
    @Answers 2018 01 01.docx

    @ - I use the shifted number special char to force files to sort to top of filename alpha sort

    "Answers.docx" is my collection of tips for the "answers" forum
    the date suffixes are backup copies of the file by the date I saved them. This way if the current file corrupts, or I realize that I deleted something "by accident".

    When I'm working on a file that is important to me (ie a Thesis), I will make daily or even hourly backups, more frequent the closer it is to the deadline! On this forum we all to frequently see pleas for help from students in December and March/April when they have a significant file become corrupt or deleted.

    Hint, I still use Explorer "Favorites" because they are individual shortcut files. I have them in a folder structure that is paralllel to my document / article folder structure. So when I use "Everything" to search for files, I get both files and favorites, sorted by tags and dates. I have a collection of over 20,000 favorites.

    STOP: 0xC000021A  

    Can be a difficult problem to solve and you may indeed need technical help from a real computer shop
    (not leeks and peeks at a BestBuy or other BigBox stores) or the System maker's Support.


    Cause

    This error occurs when a user-mode subsystem, such as WinLogon or the Client Server Run-Time Subsystem (CSRSS), has been fatally compromised and security can no longer be guaranteed. In response, the operating system switches to kernel mode. Microsoft Windows cannot run without WinLogon or CSRSS. Therefore, this is one of the few cases where the failure of a user-mode service can shut down the system.

    Mismatched system files can also cause this error. This can occur if you have restored your hard disk from a backup. Some backup programs might skip restoring system files that they determine are in use.

    Resolving the Problem

    Running the kernel debugger is not useful in this situation because the actual error occurred in a user-mode process.

    Resolving an error in a user-mode device driver, system service, or third-party application:Because bug check 0xC000021A occurs in a user-mode process, the most common culprits are third-party applications. If the error occurred after the installation of a new or updated device driver, system service, or third-party application, the new software should be removed or disabled. Contact the manufacturer of the software about a possible update.

    If the error occurs during system startup, restart your computer, and press F8 at the character-based menu that displays the operating system choices. At the resulting WindowsAdvanced Options menu, choose theLast Known Good Configuration option. This option is most effective when only one driver or service is added at a time. If this does not resolve the error, try manually removing the offending software. If the system partition is formatted with file allocation table (FAT), use an MS-DOS startup disk to gain access to the computer's hard disk. If the system partition is formatted with NTFS file system, you might be able to use Safe Mode to rename or delete the faulty software. If the faulty software is used as part of the system startup process in Safe Mode, you need to start the computer using the Recovery Console in order to access the file. If a newly installed piece if hardware is suspected, remove it to see if this resolves the issue.

    Try running the Emergency Recovery Disk (ERD) and allow the system to repair any errors that it detects.

    Resolving a mismatched system file problem: If you have recently restored your hard disk from a backup, check if there is an updated version of the Backup/Restore program available from the manufacturer. Make sure the latest Windows Service Pack is installed.
