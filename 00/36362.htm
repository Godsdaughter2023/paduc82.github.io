My only point was if you have an audio source that is 16 bit, and most will be and your system is set to 24bit, the quality can not be 24bit in reality. Same goes for the frequency. A source of 48000Hz may play on 96000Hz, but the quality is only really 48000Hz and the quality will be distorted at 96000Hz.

This is also what an equalizer is for to adjust the different bandwidths frequencies from low to high to obtain maximum quality per bandwidth. Bass(Low)-Midrange-Treble(High).

chriso747 say:

Hi David,

What error dialogue comes up?   It could be Scenery or Area missing/not found, or similar.   Have you installed any add on scenery prior to this error?   Or, maybe installed a payware/freeware scenery enhancement/modification?

Depending on the above, you would do well to uninstall any added scenery.   Usually, the Base Scenery is not touched, so won't require deleting.   You could do a preliminary check of your Scenery.cfg to ensure that each Area and Layer Number match.  So go here:   Your Drive Letter:\Users\Your Name\AppData\Roaming\Microsoft\FSX\scenery.cfg  or go into your FSX Directory and find scenery.cfg (the same list exists in both).

Then slowly scroll down your list (by default, you should have 115 Areas) and as you go down the list, ensure that each Area has a matching Layer number.  E.g.  Area 004 should have Layer=4;  Area 115 should have Layer=115, and so on).

If all the above checks okay, and you still have a Missing/Not Found type error, you could try re-building your Scenery Library, by launching FSX and then go into Settings.   Click on Scenery Library and have a check down the list that there isn't anything that should or shouldn't be there.   Then click on  Ok.   This action should re-build the Library.

If you still have the error coming up and you know that everything is in sequence and any add-on scenery has been removed, then you may well have to do a Repair of FSX (i.e. shut down FSX, and insert Disk 1 into your drive.  Then select the Repair option).   This will reinstall any missing files (including Scenery) back into your installation.   However, you may lose any add ons, special mapping of controls/configuration files, etc.   You will need to go through the whole process again to reinstate everything.

Hope you have success with this.

usasma say:
memory corruption errors refer to a corruption of the memory at the virtual address space listed in the dump.  This can be due to a number of different things (hardware or software), so it's not a definitive diagnostic result.

cng.sys is a Windows driver named Kernel Cryptography, Next Generation
Another article (here:  http://csrc.nist.gov/groups/STM/cmvp/documents/140-1/140sp/140sp1328.pdf  ) refers to it as the Microsoft Kernel Mode Cryptographic Primitives Library.  Not a whole bunch to go on there either - but it does make one wonder about security problems (such as malware,etc).  Also, Windows drivers are protected by the System File Checker and it's associated functions - so the likelihood of a problem with a Windows driver is much, much less than the other problem areas that we consider.

In general, if we get errors like this, we revert to troubleshooting by the odds:
- first I rule out 3rd party drivers (over 90% of BSOD's are due to this)
- then I rule out hardware (less than 10% of BSOD's are due to this)
- then I look at Windows problems (less than 1% of BSOD's are due to this - as long as Windows Updates are fully up to date).

The latest memory dump points to your Intel graphics drivers (igdkmd64.sys).  Now we're faced with 2 possibilities:
- that this is a different, unrelated problem from the previous dumps
- that this is a hardware problem and the differing errors are symptomatic of that

I'd suggest a 2 pronged approach:
- download a fresh set of the Intel graphics drivers, then uninstall the current drivers and install the freshly downloaded one's
- start with these free hardware diagnostics:  http://www.carrona.org/hwdiag.html
