steve rindsberg say:
>> No idea why it could zap links though. Maybe you could post a copy somewhere.

Because we're replacing one text range's text with a new text string.

Instead of the existing Replace command, use this:

With oshp.TextFrame.TextRange
   .Characters.Replace "$$userName$$, userName
End With

If there's a link on the $$userName$$ portion of the text, it will still get whacked, but other links in the same text range will be preserved.

This will also preserve any formatting on the text (again, except for the text being replaced - the new text will take on the formatting of the first character from the text that's being replaced).

With the usual Replace command, all of the text in the textframe takes on the formatting of the first character in the original text.

Think of it this way:

The Replace command says "Store a copy of the text from this shape in memory.  Do the replacements on the copy in memory.  Finally, replace the text in the shape with whatever you've got stored in memory."  Since the copy in memory is just a text string (no formatting), you end up losing all of the original formatting and links when you use Replace.

Using .Characters.Replace finds the characters to be replaced, stores them in memory, does the replacements, then puts them back in the correct position w/o touching the surrounding text.

Repeat Action Marco in MS Access - Microsoft Community

darlenesarbiewski say:

Repeat Action Marco in MS Access

I would like to have a macro repeat the same number of times as number value entered on a form.  

ken sheridan say: From your description I think that, rather than inserting multiple rows in the table one by one, you should be able to use the technique John described, in which you use a separate table of sequential numbers to insert the multiple rows as a set.  If we take a simple example of a form, frmContactCounters with unbound text box controls, txtFirstName, txtLastName and txtCounter in which the values entered are respectively Tristram, Shandy and 5, then the following query:

SELECT Forms!frmContactsCounter!txtFirstName AS Firstname,
Forms!frmContactsCounter!txtLastName AS LastName
FROM Counters
WHERE Counters.Counter<=Forms!frmContactsCounter!txtCounter;

will return the following result set:

Tristram    Shandy
Tristram    Shandy
Tristram    Shandy
Tristram    Shandy
Tristram    Shandy

If the query is then converted into an 'append query:

INSERT INTO ContactsTemp(FirstName, LastName)
SELECT Forms!frmContactsCounter!txtFirstName AS Firstname,
Forms!frmContactsCounter!txtLastName AS LastName
FROM Counters
WHERE Counters.Counter<=Forms!frmContactsCounter!txtCounter;

Executing the query would insert five identical rows into the Contacts table.  

On the other hand, as you say the data is being returned from existing tables, you don't need to append the data to another table at all, but merely base the report on a query e.g.

SELECT FirstName, LastName, AddressLine1, AddressLine2, PostCode
FROM Contacts, Counters
WHERE Contacts.FirstName = Forms!frmContactsCounter!txtFirstName
AND Contacts.Lastname = Forms!frmContactsCounter!txtLastName
AND Counters.Counter<=Forms!frmContactsCounter!txtCounter;

The above queries are very much simplified of course.
