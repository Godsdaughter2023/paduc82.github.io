Office 2010 is the first release of Office that contains 64 bit edition. It’s recommended to install the 32 bit version Office 2010 due to compatibility issues however if you still want to try 64 bit version keep these in mind.

·         32 bit version works on both 32 bit and 64 bit windows operation systems. But 64 bit version Office 2010 must be installed on 64 bit windows operation systems.

Short answer - no impact other than potentially saving yourself problems when you have 32 bit applications that use outlook or want to use any Outlook addins that have not been converted to 64bit Outlook. Have the relevant tasks been split into parts?  You can most easily see this in the standard Gantt Chart view.  What are the Stop and Resume dates for the affected tasks?  (These columns are not included in the standard tables, so you'll need to insert them.)  The Stop date represents the end of the Actual Duration part of the task, and the Resume date represents the start of the Remaining Duration part.  If there is any working time between these two dates, then a split has been introduced.

Using the Update Project dialog can sometimes lead to unexpected results.  For example, "Reschedule uncompleted work to start after:" typically constrains the Resume date of an in-progress task to be on-or-after the selected date.  Subsequent uses of the same technique may be used to delay - but not to accelerate - the Resume date, and the only way to remove this hidden constraint is to reset the progress on the task.  Maybe your example is the result of this....

jay freedman say:
I would be wary of doing any further editing of documents saved in the Dropbox folder (or the SkyDrive folder of SkyDrive Pro, or any other "automatically copy to the cloud" folder).

When a document is opened in Word for editing, several temporary files are created in the same folder as the document's file. (For the technical details, see http://support.microsoft.com/kb/211632.) The Dropbox software will then copy the temp files to the server every time they change, which can cause an unwanted load on the network and CPU. In extreme cases, especially with large files, that will cause noticeable slowdowns.
