doug robbins - mvp office apps & services (word) say:

Use:

' splitter Macro
' Macro created 16-08-98 by Doug Robbins to save each page of a document
' as a separate file with the name Page#.DOC
' modified 28-09-2010 to use filename of the last 9 digits of the first paragraph
Dim Counter As Long, Source As Document, Target As Document
Dim Docname As Range
Set Source = ActiveDocument
Selection.HomeKey Unit:=wdStory
Pages = Source.BuiltInDocumentProperties(wdPropertyPages)
Counter = 0
While Counter < Pages
    Counter = Counter + 1
    Docname = "Page" & Format(Counter)
    Source.Bookmarks("\Page").Range.Cut
    Set Target = Documents.Add
    Target.Range.Paste
    Set Docname = Target.Range.Paragraphs(1).Range
    Docname.Start = Docname.End - 10
    Docname.End = Docname.End - 1
    Target.SaveAs filename:=Docname.Text
    Target.Close
Wend


-- Hope this helps.

Dim Counter As Long, Source As Document, Target As Document
Dim Docname As Range
Dim i As Long
Set Source = ActiveDocument
Selection.HomeKey Unit:=wdStory
Pages = Source.BuiltInDocumentProperties(wdPropertyPages)
Counter = 0
While Counter < Pages
    Counter = Counter + 1
    Source.Bookmarks("\Page").Range.Cut
    Set Target = Documents.Add
    With Target
        With .Range
            .Paste
            For i = 1 To .Paragraphs.Count
                .Paragraphs(i).LineSpacingRule = wdLineSpaceSingle
            Next i
            Set Docname = .Paragraphs(1).Range
            Docname.Start = Docname.End - 10
            Docname.End = Docname.End - 1
        End With
        .SaveAs filename:=Docname.Text
        .Close
    End With
Wend

kim overstreet say:

Quick parts disappearing

I had the problem below and search several threads on here where people had the same problem. None of those threads helped me fix the problem, and they were all locked, so I'm putting the solution i finally found  here for others who might need it.

I often use quick parts in Outlook 2016 to insert text that I have to type over and over for my teachers. It worked fine until the end of last spring. Now whenever I close Outlook it will clear out the Quick parts that I have saved. I’ve done my research and:

  1. Quick parts are supposed to be saved in normalemail.dotm and I tried renaming that and letting it create a new one in case the old file is corrupt.
  2. I tried saving the quick parts to the normalemail.dotm through Word and when I close word they disappear as well.

  1. In outlook click on the file menu, then Options, click on Trust Center, then on Trust Center Settings, and then on Macro Settings – enable all macros.
  2. Do this in both Outlook and Word.
  3. Then open Word, create the quick parts and save in the normalemail.dotm and save the file on your desktop.
  4. Close Word and Outlook.
  5. Drag the .dotm file into the current location of the normalemail.dotm and replace the current file.
  6. The remember to reset your Macro settings in both Outlook and Word.

Hope this works for you.

To start at the end of your question, the suggestion to do a clean installation or a manufacturer's recovery is a cop-out, indicating that your "local engineer" is either not very knowledgeable or (more likely) just doesn't want to spend any more time on your problem.

Does the complete error message that appears read something like this:

Installation Progress.
Current Action:
Waiting for cabinets to be cached to local Installation source.
File: M9561403.CAB.
Installation error: File not found.
A required installation file M9561403.CAB could not be found.

Original Installation source required:
If you installed Microsoft Office Professional Edition 2003 from a CD, please insert your CD. If you installed Microsoft Office Professional edition 2003 over your computer network, please browse to the installation source on your network. Once you have located your installation source, click OK

This situation is likely to arise if Office was pre-installed on your computer by the manufacturer.  The manufacturer probably used a network installation source and then included the CD for you to use.

Open the Registry Editor (Start > Run > regedit > OK) and navigate to
   HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Office\Delivery\SourceEngine\Downloads\{OfficeProductGUID}\Sources\{OfficeProductGUID}

The value for "OfficeProductGUID" varies depending on the particular version of Office 2003 you have, but it should look like this:  WXYYZZZZ-6000-11D3-8CFE-0150048383C9  (for the different values of W, X, YY, and ZZZZ, see http://support.microsoft.com/kb/832672).

In that subkey, you should see 3 entries in the right-hand pane:  (default), Path, and Priority.  The "data" for the Path item should be D:\office2003\  where D: is the name of your CD drive.  If the data value for Path is something other than that write it down (so you can restore it if necessary) and then right click Path, select "Modify" and enter the correct path information.  Reboot and the install should work properly.

lemp say:
You didn't go far enough ... but be careful, making changes in the Windows registry can be dangerous and there is no "undo" function if you make a mistake.

In fact, I was remiss in not telling you to back up your registry before you start to make changes.  You can make a System Restore point (see http://bertk.mvps.org/html/createrp.html and http://support.microsoft.com/kb/306084).  Alternatively, and it's good alternative, use the free application ERUNT.

  1. Click the + to the left of HKEY_LOCAL_MACHINE and 5 sub-entries appear.
  2. Click the + next to SOFTWARE under HKEY_LOCAL MACHINE and a long list of software names appears
  3. Scroll down the list until you get to Microsoft and click the + to its left and another long list appears
  4. Scroll down the list under Microsoft until you get to Office and click the + to its left
  5. Scroll down this list, which isn't so long, until you get to Delivery and click the + to its left
  6. Click the + to the left of Source Engine (which is under Delivery)
  7. Click the + to the left of Downloads
  8. Now you should see an entry like WXYYZZZZ-6000-11D3-8CFE-0150048383C9.  Click the + to its left.  If there is more than one such entry, you may have to repeat from this step
  9. Click the + to the left of Sources
  10. Now you'll finally see the last WXYYZZZZ-6000-11D3-8CFE-0150048383C9.  Click it and in the right-hand pane you should see the three items I described before.
