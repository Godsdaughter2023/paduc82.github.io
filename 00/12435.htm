john w. vinson say: One to one relationships are quite rare: normally you can simply include the fields in one table and use it. What is the difference in data between these tables? What real-life "Entity" does the HR table model?

If you do have need for a one to one relationship (say if HR is a "Subclass" of the attributes of an Employee, or you're using table-based field-level security - and if you're not, maybe you don't need two tables!) then bear in mind that one to one relationships still have directionality: one table must be the Parent and the other the Child. You CANNOT join two tables autonumber to autonumber!! If Employee is the parent, then it could have the autonumber field; the EmployeeID in the HR table would need to be a Long Integer. To update the tables you would use a Form based on Employees with a Subform based on HR, using the EmployeeID as the master/child link. If it's more appropriate you can reverse the roles of these two tables; but in any case, only one can be the master.


(That is, it's not like Ambrose Bierce's definition of Marriage: A relationship consisting of two masters and two slaves making in all two).

jampe say:

Hey MacAodh,

 

Sounds like you are not doing anything wrong. Try to reset the pairing as per instructed in the user guide on the page 11. Remove the pairing also from the phone: Menu > Settings > Connectivity > Bluetooth > Paired devices > Select the device > Options > Delete pairing. Then create the connection again.

 

If it still doesn't work, try to the phone with another headset, and the headset with another phone, if possible. This way you can find which device is causing the issue.

 

If it seems to be the phone, do this troubleshooting. Make a back up before this just in case with Nokia Suite. Try also a hard reset by typing in *#7370# followed by lock code which is 13254 by default if not changed by the user. This will wipe the phone memory, so be 100% sure you have a back up.

 

If it's the headset, take it to Nokia Care point.

 

If you are unable to find out where the issue is, and resetting the devices does not help, take both devices to Nokia Care point and ask them to test.

 

Hope you get the connection to work!
