rob koch say:

The situation where two AV products are installed is called "conflict" and this is why all 3rd-party products containing AV components are designed to tell Windows Defender to shut down, using operating system API calls provided by Microsoft for that purpose.

The difference with Malwarebytes Anti-Malware is that this product uses a different set of filing system calls to the operating system specifically to try and avoid conflict, though this doesn't work all of the time and occasionally requires that Malwarebytes make changes to adjust its operation.

MBAM also includes more granular control settings over many of its individual protections, so in the case where a conflict or simply too much overhead exists, it's often possible to scale back some overhead by disabling certain protections.

Problem with restricted Word document and watermarks - Microsoft Community

a. user say:

Problem with restricted Word document and watermarks

Our Word templates are restricted (Developer > Restrict Editing) so that people can only use the allowed styles etc. and all our documents keep the same look and feel. We are having some issues with our watermarks as a result.

The possible watermarks are set as building blocks and so can be selected as required from Design > Watermark.

The problem is as soon as a landscape page is used (and so there are multiple sections), the watermarks go mad. If you change the watermark in section 2 it is removed from section 1 and section 3 is left as it was. If you then select the new watermark for section 3, you get two watermarks visible.

Is there a way to apply a watermark across multiple sections without having to turn off restricted editing or involving selecting the watermark text and manually deleting it?

charles kenyon say:

This behavior is -- believe it or not -- "works as designed". It's independent of whether the document has restricted editing, but it's true in any multi-section document.

The "problem" is that the Design > Watermark feature is programmed to remove any other watermarks anywhere in the document. (I'm not sure how you're getting two watermarks in section 3; maybe the programming isn't correct in other ways.)

Yes, there's a way to avoid this issue, even in a restricted document. Open the header pane, and click Layout > Selection Pane. Click on the existing watermark. The selection pane will show the name that was automatically assigned to the watermark, and it will contain the word "WaterMark" along with other words and a random number. Click on the name and edit it to remove the "WaterMark" (or completely replace it with a name of your choice); press Enter to finalize the change. The Design > Watermark feature will no longer recognize the object as a watermark and will leave it alone.

My observation is that even if a Watermark is renamed, the Remove Watermark command may change the watermark for the current section. It usually will not. It may remove a Watermark in a different section,or not. In my opinion, the best way to deal with multiple watermarks in a document is to treat them as images in headers and footers and not use the Watermark tools within the document.

Inserting a watermark using the command in a document already containing a watermark may replace the current watermark. The new one may appear on all pages even if the current one does not.

If you want multiple watermarks in one document, use Insert Watermark to insert them in a different document. Use the method Jay notes to rename them if you want. Then copy and paste into a the destination document's appropriate header (or footer). In that way, it will appear or not along with any other header/footer content. My preference is to always put them in the header unless there is a reason to use the footer.
