brookzhang_msft8047 say:

hi rhaulet,

for normal mailboxes in office 365, this is only one kind of report called non-owner mailbox access report, which lists the mailboxes that have been accessed by someone other than the person who owns the mailbox. by default, the non-owner mailbox access report is disabled for mailboxes. so, admins of office 365 need to use powershell to enable the mailbox audit report. admin needs permissions to run these cmds. to get the permission:

  1. login to exchange admin center with admin credentials.
  2. navigate to permission > admin roles, find organization management and records management, then add the admin as a member of these two admin roles.

then use admin credentials to connect to (eac) exchange online using remote powershell.

and type in set-mailbox <identity> -auditenabled $true.

after that, the non-owner mailbox access report is enabled for the mailbox.

about running a non-owner mailbox access report and exporting the mailbox audit report, you can refer to the following two articles:
https://technet.microsoft.com/en-us/library/jj150575%28v=exchg.150%29.aspx

https://technet.microsoft.com/en-us/library/jj150552%28v=exchg.150%29.aspx

note: the owner activities are not included in the report mentioned above.

for last logon time, you can find that by navigating to eac > recipients > mailboxes > choose the mailbox > edit > mailbox usage.

or you can get it in powershell by typing get-mailbox identity | get-mailboxstatistics.

jay freedman say:
I think you're confusing the terms header and heading.

A header is an area at the top of the page, separate from the text in the rest of the page. Only one header can appear on any given page at one time. You can't put more headers further down on the same page.

A heading is a paragraph of ordinary text, to which you apply a style that makes it look different. There are 9 built-in styles named Heading 1, Heading 2, etc. and you can define other styles if you want them. Read http://shaunakelly.com/word/styles/tipsonstyles.html and the pages it links to.

You need to put a heading over each table, not a header.

charles kenyon say:
As someone else mentioned, you really want to learn how to uses Styles. You would have a TableHeading style. You could apply it to the first row of each of your tables if you want. You would set up the style so it has the formatting you want. Then, in any row where you would want that style you would apply that style. Understanding Styles in Microsoft Word

You can make each table, itself, have one or more Header Rows. These rows would repeat at the top of the table (not the page) if your table has to extend over one or more pages. This is like page headers, but page headers (which are the Headers you see on the Insert Tab) go at the top of each page and sometimes have page numbers in them. Page Headers are not inside of a page but provide a frame for that page. Tables are inside one or more pages. Their header rows are contained inside the page as well as inside the table. Each table can have its own different header rows.

As far as days per month, I would leave it at the default.  What those settings do is to 'translate' when you enter durations.  You enter a task wiht 5 days duration.  Behind the scenes Project converts 5 days to 50 working hours based upon the day = 10 hours.  If you do not enter durations in months, the Days per month setting is not needed.

If you are not using resources the only option is to assign task calendars for the night shift work.  You'll need to create a custom calendar for night shift and then assign that task.  The difficulty will be if the task durations exceed 10 hours.  If you assign the night shift calendar, it will only be worked on during the night shift hours.  So a task that could be started on the night shift and then finished by the day shift will not be scheduled past 3:30 am, the end of the night shift.  Any remaining duration will begin again at 5:00 pm that night.

jeffphillips say: I had this same issues as well and found a solution but its somewhat technical so the reinstall option may be best for most.

Many OSX users have been reporting problems with corrupt symbolic links. There is a long forum here:https://discussions.apple.com/thread/3891215?start=0&tstart=0

Apple has opened BugID #12133587 for the issue.

To manually correct this problem you need to go into the Application/Microsoft Office 2011 folder and search for broken links and manually delete and recreate the links.  

This command will identify broken links:
/usr/bin/find -x . -type l ! -execdir test -e '{}' \; -ls | nl 

I found several corrupted links in the Office/OfficeArt.framework folder, often it was the Version/Current link which should point to Versions/14 but instead pointed to something obviously corrupted like '?!AA?!a!!A?AAA?!?'.

Delete and recreate the broken links and rerun.

The majority of users suffering from broken links seems to be like me and have upgraded to a 3TB drive.
