Excel has special formats for zip codes and phone numbers, but unfortunately they don't appear for each language/regional setting. You need to open the format dialog, select "Special" and then select English (US) to see some. Most other languages don't show any special formats. 

The Zip format retains a leading zero. The phone formats adhere to the US standard (no leading zero in the area code, followed by xxx-xxxx). There are no pre-defined phone or zip formats for Oz, but you can create your own.

For a phone number, you can create a custom format with this setting

000-0000-0000

For a zip code use a custom format with

0000

Once defined, stick them into a style and they are only a ribbon click away.

cheers, teylyn

ossiemac say:

You can also use Conditional Formats to set the Number format for phone numbers where you have a mixture of fixed line and mobile numbers which is quite common now with people only having a mobile number that gets entered into databases as their main number; usually because it is a compulsory field.

 

All numbers are 10 digits (except for special services which I am going to ignore). Mobile numbers can be identified by the first 2 digits of 04 and other standard fixed line with 0 and another digit other than 4. See the following link.

 

http://en.wikipedia.org/wiki/Telephone_numbers_in_Australia

 

To Conditionally Format:
1. Select the column with the phone numbers.
2. Select Conditional format (Towards the middle of the Home ribbon)
3. Select New rule
4. Select “Use a formula to determine which cells to format”
5. Enter the following formula in the Field below “Format values where this formula is true” (Edit B1 in the formula to the first cell in the selected column)
6. =OR(AND(LEN(B1)=9,LEFT(B1,1)<>"4"),AND(LEN(B1)=10,LEFT(B1,2)<>"04"))
7. Select format button.
8. Select Number tab
9. Select Custom
10. Enter the following in the field under “Type:”
11. (00) 0000 0000

Repeat the above but at step 3 select Manage Rules and then select Add rule and use the following formula. (Edit B1 in the formula to the first cell in the selected column)
=OR(AND(LEN(B1)=9,LEFT(B1,1)="4"),AND(LEN(B1)=10,LEFT(B1,2)="04"))
Repeat from 7 above but at step 11 use the format 0000 000 000

natasha gatz say:

Access Text Date From Linked Database

I have a linked database that I have read-only rights to, I am importing data from the database into access and one of the fields is a text date field. I am needing to format this field into a true date field to be able to use various formulas on it. I have tried everything I can think of but the date remains as a text date, I was wondering if it is possible with the linked database.
ken sheridan say: If you create a link to the external table you should then be able to create an 'append' query which inserts the rows from the linked table into a predefined local table.  In the append query call the CDate function to convert the values from the text column to DateTime values.  The text values must be capable of interpretation as valid dates in the regional date format used by the system.  If not you will need to write an expression to parse the string expression into a valid format.

Appending data to a predefined table is preferable to using a 'make table' query as it enables you to define the data types of the columns in the table, and to create the appropriate indexes.  Assuming that the imported table has a primary key column or columns (or at least a reliable candidate key), and the same key is designated as the primary key or otherwise indexed uniquely in the local table, you can execute the same append query again to import further data.  Any rows in the linked table which already exist in the local table will be rejected by virtue of the key or index violations, and only new rows will be appended.
