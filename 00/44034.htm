When Windows Explorer (not Internet Explorer) is misbehaving (especially when right clicking), start to suspect third party Explorer extension add-ons.  You may also see Data Execution Prevention (DEP) errors.  DEP errors are reported when XP feels threatened by a program and Windows will shut down the threatening program.  Windows should never feel threatened by Windows Explorer (or Internet Explorer) unless some add-on is the cause.

Those would be Explorer extensions that do not belong to Microsoft.  That means extensions that you added.  Explorer extensions are usually okay and installing some applications will install Explorer extensions for you, give you a choice and sometimes they can be added without your knowledge when you install new software.

Explorer extensions are sometimes added as a new right click option you see on folders and files (like scan this file, open this file, play this song).

If there is a particular thing you do when Exploring that you know will cause the problem, that will help zero in on the problem and help you know for sure when you have found and fixed it.

http://www.nirsoft.net/utils/shexview.html

ShellExView doesn't install anything on your computer, it just runs and displays.

Next, click the column header called Microsoft (it may be way out on the right side of the screen) to sort the display (by clicking the Microsoft column header) so all the non Microsoft extensions are at the top and easy to see.  They will say "No" and be marked in pink since they are not Microsoft extensions, something like this:

The non Microsoft extensions would be things you have added (non Microsoft) and are what you need to be suspecting.

If you see extensions that do not have anything listed under Description, Product Name, Version, Company or have peculiar names that looks like they might be just random numbers and  letters, you might want to look at those first.

You can also Google the name of a suspicious add-on and see if there are any hits regarding Explorer crashes or DEP errors and what other people have done about it.

<my-suspicious-add-on>  windows explorer crashing

You have to fill in the name of your suspicious add on.

See what kind of search results hits you get and look for solutions or situations that sound like yours.

Right click and disable the non Microsoft extensions one at a time (or perhaps in little groups of 3-5) keeping a list so you can enable them again later if desired. Now, as far as the issue at hand.... if DISM fails due to corruption within WinSXS, there are two courses of action that will fix the issue WITHOUT doing a repair install.

  • Solution 1
  1. run dism /online /cleanup-image /startcomponentcleanup
  2. Restart computer (utilize charms bar only) once cleanup has finished
  3. run dism /online /cleanup-image /restorehealth
    • This should now complete successfully and repair the corruption within WinSXS.
      • If it doesn't, restart computer, then run sfc /scannow , restart once completed, then repeat 1 - 3
    • It would also be a good idea to run sfc /scannow once done
  • Solution 2
  1. Follow the instructions at: The Tech Cookbook using SFCfix to replace the x64 or x86 version of Microsoft-Windows-Unified-Telemetry-Client. 
  2. Restart computer (utilize charms bar only) once cleanup has finished
  3. run dism /online /cleanup-image /restorehealth
    • If it fails again, perform steps 1 - 3 of Solution 1 (after restarting computer)

***Due to how Windows shuts down and boots, you must restart the computer after each and every SFC or DISM repair operation, regardless if it completes successfully or not.  Without going too much into it, this is due to a pending flag that is placed on the system***

A few key points:

  • There is almost always a way to avoid doing a repair install, and all other options should be exhausted prior to doing a repair install for a multitude of reasons (it counts as a separate Windows activation, and while you'll never be unlicensed, it will eventually require you to go through the hassle of a phone activation; some software companies may license their products in such a way a repair install counts as a separate activation [as it does with 4Team software], among many other things)  
  • The unfortunate achilles heel of Windows 8 and beyond is the DISM utility, as it's the only Windows utility capable of repairing WinSXS corruption.  There are times when a repair install is necessary, but it should always be a last resort.  
  • If you spend a decent amount of time on your PC, regardless if you're an amateur or power user, I'd encourage you to look into the MDT (Microsoft Deployment Toolkit), which is free.  It was developed for business use, where multiple PCs must be deployed, however it's great for any user as it allows you to create a Windows install that auto installs all your desktop programs, drivers, and apps with little to no interaction from a user.  It's something you can tinker on and come back to over several months to tweak it just right for your uses.  It definitely saves a lot of time when you eventually do have to reinstall an OS (upgrading a deployment to a different version of Windows is as simple as deleting the old source files and copying the new ones over).  
  • Whenever you do a new install of an OS, after you get all drivers and desktop applications installed, and customize the OS to your liking, boot into WinRE (or WinPE) and take a WIM of your system drive.... it will save you an enormous amount of time if you ever have a corrupted or non-recoverable system.  Set the compression flag to max, start it before bed, and by the time you wake in the morning, the imaging will be completed. Restoring a WIM requires no re-activation of any software, as it's a complete backup of your system partition. (Creating a WIM in WinRE or WinPE requires setting the /ScratchDir flag, otherwise your imaging will fail [this is unique to Windows 8 and beyond due to ImageX no longer being compatible with Windows 8+].)

The Misadventures of Reinstalling Windows In-place: Putting it in - Microsoft Community

anthonym1229 say:

The Misadventures of Reinstalling Windows In-place: Putting it in its place after it repeatedly tried to install itself in the WRONG partition

I know this is a very long read, but I just had to share my awful experience with reinstalling Windows, and how I finally solved it (with some questions still remaining). Maybe it'll help someone, or maybe it'll at least be entertaining. Thanks for reading, if you do.

This past weekend, I discovered a severe issue with ReadyBoot in my Windows 7 Pro x64 install, as I explain in more detail elsewhere. So, after trying to troubleshoot that for most of Saturday, I finally decided I would do a repair install (inplace upgrade) on Sunday morning. This was not the first time I reinstalled Windows--I actually did it just six months prior, and it was a smooth, straight-forward process. Actually, I can't ever recall a time when I've had problems (re-)installing Windows on any of my computers over the years. But, this time was anything but problem-free...

Before I go on, let me detail my PC specs really quick, in case they matter: Dell XPS 15 L502X, Core i7-2630QM, 8GB RAM, 640 GB 7200RPM HDD. I was running Windows Setup from a USB thumb drive, as I've been doing ever since I transitioned to Windows 7 in 2009.

First attempt:

The upgrade process failed during the stage where drivers and Windows features get installed. The error message said something about missing a required driver. And so, I suspected the issue: (pseudo-)drivers left behind by an incomplete SuperAntispyware uninstall. Generating a system health report sometime a day or two before the attempt, uncovered two drivers not able to be found, and they sure enough were leftover traces of SuperAntispyware. I had to download a utility from the program's website to clean them out. I had actually taken care of these before, but then shortly before the upgrade attempt, I restored my system partition after a time-expensive mistake on my part, clean installing Windows over my existing install (yikes). This backup was from before I cleaned out those traces of SuperAntispyware.

Second attempt:

After the botched first attempt, I had to restore my system backup AGAIN, because after the error of "a required driver could not be installed" or something like that, I rebooted and selected "Windows Setup Rollback" or something like that, only to be greeted with a CMD prompt warning me that a winpeshl.ini file was found, but that no commands could be launched successfully. What? So, not only did Windows fail to reinstall, it failed to rollback, prompting my need to restore from my backup. I did so, cleaned out those rogue SuperAntispyware traces, and even uninstalled nonessential programs that came with drivers, like the Virtual CloneDrive ISO-mounting program, and Easeus Todo-Backup (just to be clear, I've been using the ISO image of Todo-Backup on another jump drive to restore my backup). Once I generated a clean system health report, I proceeded with a second upgrade attempt. And...wait for it...it failed again! With the same exact error, too, about not being able to install a required driver. WHAT? I then rebooted, only to be greeted with the same CMD prompt about winpeshl.ini...gee, doesn't this sound familiar? By now, I was fuming. Here I was, on a late Sunday afternoon, having already sunken my entire day up to that point, into trying to repair my computer, and I was getting NOWHERE.

Fortunately, I still had tricks up my sleeve. I wasn't going to cave and do a clean reinstall, because I knew to get back up and running with all my programs and customizations afterward would cost me up to two days (the fact that I had already lost my whole weekend to computer issues notwithstanding). So, I had to figure out, what else could I do? Then I thought, aha! I have Windows PE on one of my jump drives, that I could use to investigate Windows' botch ups. I booted up from that, and peeked into my system partition where the failed setup was. I navigated to the winpeshl.ini file, and saw that it referenced rollback.exe, which was nowhere in the given path, or even anywhere on my system partition. Interesting...

My sleuthing also uncovered the expected $INPLACE.~TR and related temporary directories on my system partition, but then I examined my data partitions (a primary 20GB one for Documents, a primary 525GB one for my media, and a couple tiny logical partitions for usage by particular programs), and grew even more annoyed... Why does Windows Setup bother to enumerate files in data partitions? What a waste of time! I figured that's the culprit for Windows' hanging at least a few times during the upgrade process, gathering user settings, files, and programs on partitions that it shouldn't even bother with...This is why I keep my OS/programs separate from my data! To try to simplify OS installs/reinstalls...and yet, for Windows to make an apparent mockery of my forethought, irked me to no end.

Both my primary data partitions had $INPLACE.~TR folders and other evidence of Windows' being where it wasn't supposed to be. However, my logical partitions maybe only had one file added to them, and surely no $INPLACE.~TR...Hmm...that gave me an idea that I would explore for my next upgrade attempt. Before that, however, I became simply astonished upon discovering a stash of Windows' install files in $WINDOWS.~BT, on my media partition. "Idiot!" I exclaimed, barely restraining the urge to mouth off a litany of curse words. "So here's your problem, eh? Trying to install yourself on my music drive, fool! Stupid retard, freaktard, dumb-butt returd [and on and on...]" I simply couldn't believe this...But wait, there's more! Rollback.exe was still nowhere to be seen. So, after all this searching around, I still had a failed upgrade install, a failed rollback thanks to an MIA rollback.exe, and a whole bunch of blood boiling in me. I now had no apparent choice but to restore my broken Windows from backup, for a THIRD time.

Third Attempt:

By now, night was falling on my wasted Sunday on a wasted weekend, and I could only wish my blood pressure would fall. After restoring my backup again, I then decided that if Windows is gonna be stupid and try to install itself where it doesn't belong, then I'll just have to deny it access to my data partitions, in attempt to corral the wildly wily Windows into my system partition. I also thought of converting my primary data partitions to logical ones, to see if that would help...I knew the latter would be a trivial task using Easeus Partition Master, and after a little research, I knew "hiding" partitions appeared trivial as well--just a matter of removing the drive letter via Disk Management. I grew smugly slightly self-satisfied, as I figured I was soon finally gonna get the upper hand in my battle to reinstall Windows. I figured, since the Upgrade process starts from within an active Windows install, and if the drives are inaccessible with no letter, then Windows can't possibly get its grubby fingers on them, right? Right? *Cue suspenseful silence*

Once I done did my deed of denying my data drives their letter (and converting my media partition to logical), I finally proceeded with my third upgrade install attempt, thinking logically that I would finally win. I waited in eager anticipation, as I saw the progress bar slowly inch forward. After what seemed an eternity, Windows finally finished the third step of expanding installation files...and then, tada!--

FAIL!

Once again, the process failed at the beginning of the installation step, with the SAME error of missing some "required driver". The blasted message couldn't even be slightly less useless, by including the name of said required driver. Ughhhh! And once again, rebooting to attempt a rollback, gave me the same error about winpeshl.ini. I simply couldn't believe this!

So, once again, I fired up Windows PE, and went straight to my media partition...Huh? It was instantly visible in Windows Explorer, along with the other drives I had attempted to hide...Guess that trick backfired, and, converting my media partition to a logical one made no difference either, for Windows, in its infinite lack of wisdom, dumped its installation files there, and once again, there was much grumbling and gnashing of teeth from me. Faced with the prospect of having to restore my Windows backup for a FOURTH time, I did some more snooping around in Windows PE. I hadn't visited the $INPLACE.~TR folder on my media partition till now, for I figured it'd just be a stash of all my personal files that shouldn't have been touched in the first place, as was the case with my documents partition. Well, lo and behold, what do I find lurking in the $INPLACE.~TR folder? Another $WINDOWS.~BT folder! Stupid failed-up Windows install! Not only does it stick its junk on the wrong drive, it can't even keep its junk in one place on said drive! How moronic can this be??? And why would setup files be mixed in with MY files, anyway? More grumbling and gnashing of teeth followed...

But then, I caught a little lucky break...before I could break a window or something. Stashed away in the second set of setup files was rollback.exe, that stupid MIA file...I copied the contents of the whole second $WINDOWS.~BT folder into the first one at the root of my music partition. Surprisingly, I got no prompts for duplicate files--just warnings about merging directories with the same name...meaning, this appeared to be one master set of setup files, split up in two locations...how in the world does this happen?

Not really expecting much, I rebooted after my copy operation, and surprisingly, Windows initiated a rollback...well well, at least Windows finally got something half-right. Sure, it had failed miserably multiple times already to upgrade ITSELF (I mean, come on, I could understand upgrade woes in an XP-to-7 or Vista-to-7 install, but on a self-upgrade? Especially one that I had successfully and smoothly performed previously on the same machine? Never.) but at least it succeeded at doing a quickie rollback...well..."quickie" being relative, say around 15 minutes. Whatever. I celebrated a brief and minor victory, though knowing it could not make up for my wasted day. 15 hours of macrohard hassle would not be easily undone by a micro victory of merely returning my Windows to its former broken state (with a broken Readyboot) without having to resort to my backup. 15 hours of this mess, and that's excluding my Saturday wasted on trying to repair ReadyBoot! Sometime after midnight, I finally gave up for the night, took a break by watching some cartoons online (a good laugh helps), and then went to bed...down and discouraged, but not quite defeated yet. I vowed to be victorious the following day.

Fourth Attempt

Monday morning, I got up, and promptly got back to trying to fix my computer. I don't remember what I did differently for my next attempt, however, if I did anything different. Maybe I just initiated the process so that if it failed again, I could look more closely again via Windows PE, for any log files detailing where this fail-trainwreck-of-a-process went off the fails--er...I mean, rails. Sure enough, the process failed again with the same "required driver" error, and I went back to Windows PE to investigate. However, in that process, I got a bright idea that I wish I would've thought of far sooner: I may not be able to hide the partitions that I didn't want Windows to touch, but maybe instead, I could fill them up with dummy data, so that Windows Setup would not have enough room to take a dump on them? A dummy file creator turned to be one trivial Google search away. "Finally, success is within reach!" I thought.... "Hahaha, fill all the spaces!" I then said, as I started making the first of three 100 GB dummy text files on my music partition. It ended up taking over 20 minutes to write the first file to disk, during which I had breakfast and still had time to spare. Ugh...That meant the whole process could take an hour or longer... But then I got an even better idea: What if I just shrink the size of my partition? The principle was the same...if Windows could not have enough space to store its setup files elsewhere, then that would force it to stay on the system partition, right?

Fifth Attempt

Using Easeus Partition Master, I shrank my media drive to a point that it only had about 10 GB free. I also wrote a 7 GB dummy text file to my Documents drive, leaving it with less than 5 GB free. My final two data partitions were either too small to be of concern, or already nearly full, and so, I didn't touch them. So now, my system partition had the most free space of all my partitions, at about 26 GB--more than enough for installation. I began what I hoped and prayed would be my final upgrade attempt...

The minutes ticked by slowly--too slowly. It felt like the stupid thing was taking even longer than usual. I especially dread the part in Step 3, expanding of installation files, where it reboots the computer, and then stays stuck at 18% for at least 10 minutes, as it "gathers additional information before expanding installation files"..."How much additional information do you need, fool?" I'd keep on thinking... Oh, with my trust level at over -9000, I actually intervened right after that reboot--I went into Windows PE, and checked the location of the $WINDOWS.~BT folder. Surprisingly, it was finally where it belonged, on my system drive, and all in one piece, too! But, I kept in mind that in the next 10-20 minutes to follow (being stuck at Step 3 at 18%, Windows could very well try to dump its junk elsewhere...I prayed that wouldn't happen...

After another eternity, the progress meter finally advanced beyond 18%, and Windows expanded the rest of its files. This time, I actually had some feedback for it--X-amount of MB out of 2918MB or however much it was. I don't recall ever seeing until now, that figure at this step, after the preceding reboot. Seemed to be a good sign...

The progress figure finally crawled to 100%, and I grew increasingly nervous...if it fails, it always fails seconds after transitioning to Step 4...well, Step 4 began... and then... HALLELUJAH! the stupid thing finally advanced without croaking! It began successfully installing all the required drivers and such, and then eventually began transferring my files, settings, and programs, all 800,000+ of them. That was especially nerve-wracking, as so many times, it'd appear to get stuck, and the closer it got to finishing, the more I tensed up at apparent standstills. I wasn't sure if it counted the files on my data partitions...knowing its stupidity, it probably did... but hey, at least I FINALLY GOT WINDOWS REINSTALLED. The whole upgrade took about 2 hours...The whole hassle to get to this point, took me 20 hours* Crazy!

*Excluding my wasted Saturday. In all, I lost Saturday and Sunday to troubleshooting and trying to upgrade, and then Monday to finishing upgrading and getting my computer back to normal for regular use, and now I've spent most of today documenting my tale here and here (same link as given at the beginning).

Conclusion

So now, I finally wrap this up with a few questions...how does Windows Setup go so wrong the way it did for me? As I noted somewhere above, this is not the first time I've reinstalled Windows on my current laptop. Well...I only had three primary partitions back then, before carving some space out of my media partition for two tiny logical partitions, but still, surely that can't be enough to trip up Windows Setup, right? I'm at a loss for any explanation.

melchizedek qui say:

Hello,

Let's troubleshoot the issue by following the steps below:

Method 1: Run the Windows built-in troubleshooter

1. Press Windows key + X, then select Control Panel.

2. Search for Troubleshooting and click on Troubleshooting.

3. On the left pane, click View all.

4. In the list of options, click on Recording Audio.

5. Click Next and follow the on-screen instructions.

Method 2: Try to use different audio formats

1. Press Windows key + R, then type in mmsys.cpl.

2. On the Playback tab, right-click (or tap and hold) the Default Device, and then select Properties.

On the Advanced tab, under Default Format, change the setting, and then retest your audio device. If that doesn't work, try changing the setting again.

Let us know how it goes.

Regards.
