Fixed it! Here's what I did. Some of these steps may not be necessary. Note that I have 32-bit Office 2013 on my 64-bit machine with Visual Studio 2012.

 

  • Started Office as Administrator (Program Files (x86)\Microsoft Office\Office15\EXCEL.EXE)
  • Opened COM Add-Ins and deleted three entries for Team Foundation Add-in
  • Re-added the add-in (C:\Program Files\Common Files\Microsoft Shared\Team Foundation Server\11.0\x86\TFSOfficeAdd-in.dll)

Once I re-added the add-in, its Load Behavior was correctly showing as Load at Startup. I closed Excel and re-launched from the usual, non-administrator start menu shortcut, and the add-in was correctly loaded.

alexsun_2012 say:

hi paul,

i think a shared mailbox is a better solution than distribution group for you.

you can convert a user mailbox into a shared mailbox. unlike distribution group, the shared mailbox has a valid exchange online mailbox and can keep the emails, contacts and calendar, but doesn’t require any exchange licenses. meanwhile, you can set up a forwarding rule to forward the incoming emails to other recipients in exchange admin center-> recipients-> shared-> select the shared mailbox-> mailbox feature-> mail flow-> enable forwarding.

to convert the user mailbox to a shared mailbox, you can run the following cmdlet after connecting powershell to exchange online: set-mailbox user -type shared.


then, remove the license from the user in office 365 admin center-> user list-> edit license.

if anything is unclear or you run into any problems, please let us know.

thanks,
alex
