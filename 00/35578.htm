I'm afraid the latest version of WLMail (2011) has some inexplicable idiosyncrasies with regard to handling attachments to HTML messages. I think you'll find it depends on the content header for the attachment whether Save > Attachments is enabled. If there is a Content-Disposition: attachment header to the MIME part, the option is enabled. If there isn't, it isn't.

You can access the attachments for saving by reading the message in plain text. To do this, press Ctrl-Shift-O for Options. On the Read tab, select Read all messages in plain text.

If you're viewing an HTML message in plain text, you can view the rendered version by pressing Alt-Shift-H.
