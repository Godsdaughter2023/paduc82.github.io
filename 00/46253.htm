hairyfool say:

In order for the computer to be able to use the two network interfaces they need to be on different networks.

The router will most likely be issuing IP addresses to devices that connect but only on the network interface making the connection. It will not issue an address to the Wired LAN side of the Laptop or to the printer.

Most printers have a USB port and as you want to connect the printer direct to the laptop that would be the most logical solution. However if  you particularly want to use the LAN port then you need to configure an address on both devices and that must be on a different subnet to the Wi-Fi.

Not only that both devices are client network interfaces and nominally transmit signals on the same pair of pins in the LAN cable i.e. the Laptop is trying to send data to the output of the printer not the input. More devices can auto sense the LAN interface but not all. To be sure you would have to either connect the two using a LAN crossover cable or fit a network switch between the computer and the printer.

hairyfool say:

If for example the Wi-Fi network is on 192.168.1.X then you would need to set the LAN port on 192.168.2.1 and the printer on 192.168.2.2 with a subnet mask of 255.255.255.0. The gateway is how traffic is supposed to leave the network which would never happen so shouldn't matter (there is no way out of the network anyway). In otherwords the Printer/LAN ports must be on a different subnet as the Wi-Fi but both should be on the same subnet.

Then from the PC open a command prompt and  ping 192.168.2.2, this should generate 4 replies. Then if the printer is already set up go to its settings and add a Tcp/Ip port for the printer address.

My are addresses are just examples as I don't know what you have.
