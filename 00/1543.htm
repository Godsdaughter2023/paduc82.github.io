To compute the monthly average, I am using the following formula:

SUMIF(B2,"Y",C2:E2)/DATEDIF(Checking!$A$2284,Checking!$A$2,"m") where "Y" indicates the item is an expense.

This formula is failing to add the values in columns C through E. It only gives whatever value happens to be in column C. I can see this by putting the following formula in column H and comparing the results with column F:

SUMIF(B2,"Y",C2:E2)

The following formula seems to work:

IF(B2="Y",(F2/DATEDIF(Checking!$A$2343,Checking!$A$2,"m")),0)

In researching this issue, other users have reported issues with sumif. What's up with this function? Should I just presume it doesn't work and not use it?
hansv mvp say:

SUMIF expects that the range in the first argument has the same shape and size as the range in the third argument. Since B2 is 1 cell and C2:E2 is 3 cells, the function doesn't work the way you want.

If you want to sum C2:Y2 if cell B2 equals "Y", don't use SUMIF, but instead use

IF(B2="Y",SUM(C2:E2),0)
