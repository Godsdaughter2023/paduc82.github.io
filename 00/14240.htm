john w. vinson say:

Splitting the database has nothing to do with the size of the database. I split all databases, even little 10Mbyte single-user ones.

Access, from version 2007 on, handles shared databases differently (and less effectively, IMO). Opening a single unitary unsplit database by several people can and will cause poor performance, bloating, the exact kind of conflict messages you're getting, and irreversible, catastrophic corruption.

Splitting is easy; splitting is standard practice; splitting is recommended by Microsoft and by all the MVPs I know. Any database with two or more users should - I'd say must! - be split, for safety and efficiency.

Character style changes when heading style applies to paragraph

Is there a way to avoid this problem?

jay freedman say:

It's a characteristic of Word that whenever you apply a font formatting (such as bold, italic, or underline) twice to the same text, they cancel each other. Because you haven't removed the character style before applying the paragraph (heading) style, each of them is trying to make the font bold, and the result is Not Bold. You can see this if you open the Reveal Formatting pane (Shift+F1) and check the box at the bottom for "Distinguish style source". Here's an example of a Heading 1 (with a bold attribute) and the Strong character style (also bold):

Instead of using direct formatting (a third application of Bold), you can open the Style Inspector dialog (this isn't on the ribbon and doesn't have a default keyboard shortcut, but you can add it to your Quick Access Toolbar) and use the eraser button next to the character style:

That will let the Bold attribute of the paragraph style take effect.

Word - Move Cell into next cell, (snaking) - Microsoft Community

vdmin say:

Word - Move Cell into next cell, (snaking)

I've been trying to figure this out. It's simple what I need to figure out how to do and I basically need to 'snake' when I insert a new cell. 

I have this table: https://i.imgur.com/UmbyfSX.png

I need to insert 1 new cell. Without inserting a new row or column. And I need the other cells to move to the next one.

Example picture: https://i.imgur.com/8HlKTj7.png

1. Create a new cell in place of the left image/cell.

2. Move the original contents of the left image to the right cell

3. Moves the image on the right image to the bottom left cell...

4. Moves the bottom left image to the right

etc etc

My usual issue is that I can insert a new cell, but It moves the old content creating a new column instead of just shifting everything to the next cell.

Or what would happen is that I insert a new cell but the content is moved DOWN 1 instead of snaking (move the next cell to the next one, next one to the next one, next one to the next one, etc).

jay freedman say:

On their own, Word's tables simply don't work the way you want them to. The only way to force them to behave that way is to write a macro (and note that the macro recorder can't handle this job) that does the work.

It would take some time to design and test such a macro, but the overall logic of it goes something like this:

  • Locate the last cell in the table that currently has content.
  • If the cursor is in a cell that is "after" the last occupied cell, stop the macro and let the user insert a picture there (or the macro may have commands to help choose the picture and add its caption). Otherwise, continue...
  • If that last occupied cell is at the end of its row, add a new row to the bottom of the table.
  • Move the content of the last occupied cell to the "next" cell, either one column to the right or to the first cell of the next row. As your picture shows, the content consists of both an inline graphic and a caption.
  • Move the next-to-last occupied cell's content to the newly vacated cell. Repeat that until all the current contents of cells "after" the cell containing the cursor have been shifted.
  • Now the cell containing the cursor is empty, and a picture and caption can be inserted there, either manually or by commands in the macro.
