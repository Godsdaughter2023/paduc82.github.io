a. user say:

Your current formula:

=IF(AND(H3-$D$2<=7,H3-$D$2>=-7),"due soon",IF(AND(H3-I3<=3,H3-I3>=-3),"","non punctual"))

An IF statement has three arguments - criteria, if true, and if false. Your first criteria is:   AND(H3-$D$2<=7,H3-$D$2>=-7)

If that is true, Excel takes the second argument of the function:  "due soon"

If the criteria is false, Excel takes the third argument of the function, which in your case is another IF statement. (Excel will never look at this IF statement if your original criteria is true, because this is in the [value-if-false] section of the formula.)

Your criteria for the inner IF statement is:  AND(H3-I3<=3,H3-I3>=-3)

If that criteria is true (H3 is within 3 days of I3), Excel outputs ""; and if it is false, Excel outputs "non punctual".


My advice would be to start your nested IF statement with an ISBLANK() statement. This statement takes one cell as an argument and returns TRUE if it is blank and FALSE if it contains anything.
Your formula in words would be something like this (I don't know if this makes sense, but I need to think about it this way to compose the actual formula):

If I3 is blank, we want to evaluate if H3 is within 7 days of D2. 

  If H3 is within 7 days of D2 (and I3 is blank, which has already been established), we want to return "due soon".

  If H3 is not within 7 days of D2 (and I3 is blank), ummm you haven't made it clear what you want to do in this situation. My guess is that you skipped it because this situation doesn't occur in your spreadsheet, but I'm gonna return "??" in my formula just to differentiate it from a blank.

If I3 is not blank, we want to evaluate if H3 is within 3 days of I3.

  If H3 is within 3 days, we return "".

  If H3 is not within 3 days, we return "non punctual".

So the actual formula would look like this:

=IF(ISBLANK(I3),IF(ABS(H3-$D$2)<=7,"due soon","??"),IF(ABS(H3-I3)<=3,"","non punctual"))


Note that I used the ABS() function to find the absolute value of the difference in dates. The AND() function you have set up works just as well, but mine involved less typing, and I'm lazy.


Hope this helps!

Issue description
=============
After you change the Office 365 passwords, the legacy OneDrive for Business stops syncing with the following error message:
"OneDrive for Business - We need your credentials to sync some libraries"

Causes
=============
When you sync the first library by using the legacy OneDrive for Business sync app, it will let you input your O365 account and password. After the credentials (account, password) are verified succesfully, they will be encrypted and cached in the local register key.

Everytime the sync app syncs doucments from SharePoint Online sites, it need to provide the cached credentials for authenticating. But if you change your account’s password, while the local cached credentials haven’t been changed, once the sync app tries syncing again, the SharePoint sites will tell the app that the credentials are incorrect. So, the legacy OneDrive for Business sync app prompts the error message.

It points out that you need to sign in again by new password for updating the cached credential.

Action Plan
=============
For updating the cached credentials, we suggest you click the “Enter credentials” button, then sign in by a new password. Please refer to the Sky's post above or the steps below to enter credentials:
Click on OneDrive for Business icon on the taskbar  > on the the following pop-up window, click Enter credentials.

If you are still experiencing an issue after entering the new credentials, please let us know by posting a new thread as this may indicate either a scoping issue or an independent problem.

accessjoe123 say:

Custom validation formula - can't figure out

HI all - excel 2010.  i'm trying to come up with a way to prevent my users from entering the wrong data into a cell.  Basically, here's what I need to see:

 

"A 6/25/2013"

 

-OR-

 

"R 6/25/2013"

 

-OR-

 

"NA 6/25/2013"

 

Now the TEXT (A, R, NA) will always be the same.  It's one of those three that someone will enter.  But the DATE will always change (e.g. tomorrow they would enter 6/27/2013).  So I'd need that flexibility,  However, I need them to enter it in the format shown above.  Instead what's happening is they type dates like "June 2013" or "06/13", etc.  And finally, sometimes they don't put a space between the text and date.  So you can imagine trying to pivot this data or break it up into columns later becomes a pain for us.

 

Does anyone know of a custom validation formula that will ensure people: 1) always type one of those three text options (A, R, or NA), 2) always put a space after it, and 3) always enter any date in the MM/DD/YYYY format? THANKS!

nbvc say: Try this Custom Data Validation formula:

=AND(OR(EXACT(LEFT(A1,2),"A "),EXACT(LEFT(A1,2),"R "),EXACT(LEFT(A1,3),"NA ")),TRIM(MID(A1,FIND(" ",A1)+1,255))=TEXT(MID(A1,FIND(" ",A1)+1,255),"m/d/yy"))

where A1 is cell you are applying validation to.  This also ensures that uppercase is used for the text options...

accessjoe123 say: Thanks.  I applied this validation to cell A1, but it does not seem to be working.  I'm trying various entry combinations (including the one I want), and it won't accept anything I enter.   (Also I noticed the date format you suggested isn't the "MM/DD/YYYY" one I was looking for.  Would your suggestion of "m/dd/yy" make a difference?)

nbvc say: It worked for me... although, we need to add another condition to make sure the date is a date... and I guess the year is 4 digits...

=AND(OR(EXACT(LEFT(A1,2),"A "),EXACT(LEFT(A1,2),"R "),EXACT(LEFT(A1,3),"NA ")),ISNUMBER(DATEVALUE(TRIM(MID(A1,FIND(" ",A1)+1,255)))),TRIM(MID(A1,FIND(" ",A1)+1,255))=TEXT(MID(A1,FIND(" ",A1)+1,255),"m/d/yyyy"))

the m/d/yy allows for single digit or double digit month and day entries... whereas mm/dd/yy means you need to enter two digits, eg. 01 for January or 01 for 1st of the month... your initial sample showed 6/27/2013..

if you want replace it:

=AND(OR(EXACT(LEFT(A1,2),"A "),EXACT(LEFT(A1,2),"R "),EXACT(LEFT(A1,3),"NA ")),ISNUMBER(DATEVALUE(TRIM(MID(A1,FIND(" ",A1)+1,255)))),TRIM(MID(A1,FIND(" ",A1)+1,255))=TEXT(MID(A1,FIND(" ",A1)+1,255),"MM/DD/YYYY"))

nbvc say: Also, make sure that in the Data Validation Wizard, the checkbox in the Error Alert tab is checked (to show error after invalid data entered)...

"Currently, I used the insert|action command from the ribbon to create an internal hyperlink to the correct slide, and also to add the macro."

AFAIK it isn't possible to have a hyperlink and run macro command on one shape. (Having the hyperlink on text in the shape and the run macro ON the shape isn't going to work AFAIK)

You probably need to make the code jump to the correct slide and NOT have a normal hyperlink.

HtmlHelp/CHM files display with warning in Preview 10074 when - Microsoft Community

simdoc say:

HtmlHelp/CHM files display with warning in Preview 10074 when called from Office 2013 Applications

If you invoke HtmlHelp files from Office 2013 applications (Excel and Visio both have been tested here), you get the following warning when the CHM file is displayed:

All that has to be done is to invoke the HtmlHelp API using the following C++ call (assuming Office x86 is installed because of the file folder path):

#include <HtmlHelp.h>

#pragma comment (lib, "HtmlHelp.lib")

HtmlHelp(::GetActiveWindow(), _T("C:\\Program Files (x86)\\Common Files\\microsoft shared\\OFFICE15\\1033\\ADO210.chm"), HH_DISPLAY_TOPIC, NULL);

This can be done through a simple COM call to a method that calls this API.  Other ways of invoking it are available as well.  The message will occur for ANY CHM file that I have tried.  So you can use another one if needed.

If the HtmlHelp API is called from other applications (including Windows Explorer), the CHM file will display without this message.  Only Office applications seem to have this warning.
