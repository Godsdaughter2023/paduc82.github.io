eng_amit say:

Want to change the font size of shrunken fractions in an inline equation in MSWord?

It is easy. Let's say that you have an equation in the inline format where fractions appear to have shrunk.  Copy the fraction and paste it in a new line. It would automatically become a new equation in the display format. It would show up in the center of the page. Change the font size and copy again and paste it in the original equation. Check the font size of the fraction in your equation now by dragging the mouse over it. It would be different from the rest of the equation. So, finally, you have an equation with two font sizes!

jim gordon mvp say:

Can’t Activate Office 2011
Your Office 2011 product key can not be re-used in any of the following scenarios:

 

  • Erased your startup volume and reinstalled using the 2011 installer
    Replaced your hard drive or SSD drive
    Attempt to install on a different Mac

 

In these situations, you will have to buy a new product key. They’re cheap! They sell for about $20 on eBay because Office 2011 is discontinued and because 2011 will not run in future versions of Mac OS.

 

If you attempted to install with an older installer, you’ll have to remove what you have already installed. Here's how to completely remove Office 2011
https://github.com/pbowden-msft/Remove2011

 

You can download the latest full and final Office 2011 install (which is build 14.7.7) from https://go.microsoft.com/fwlink/?linkid=273962. You will need to enter your brand new never been used before product key (25 characters) to activate it.

nathan o. - microsoft support say:

Hi Mike,

 

My first recommendation would be to add your server name to the Local Intranet zone within Internet Explorer.  To do this, you can do the following:

 

+>Close Access on each machine and then do the following steps on each machine (you could test with a couple machines first).

+>Open Internet Explorer.

+>Click on Tools -> Options.

+>In the Internet Options dialog, click on the Security tab.

+>On the Security tab, click on the Local Intranet zone and then click the Sites button.

+>Click the Advanced button on the Local Intranet dialog.

+>On the next screen, type in your server name in the “add this website to the zone” text box.  Only type your server name.  Don’t precede it with any backslashes.  Make sure the “Require server verification” box is NOT checked and then click the Add button.

+>Close out of all the open IE dialogs by clicking Close / OK.

 

After going through these steps, try opening Access again.

 

Best Regards,

Nathan O.

nathan o. - microsoft support say:

If you are getting a message that states “not a valid Win32 application”, then it sounds to me like your file associations for .mdb files could be wrong on the machine.  To ensure your file associations are correct, you can do the following:

 

+>Go to one of the client machines and navigate to the actual .mdb file on the T drive.

+>Right click on the .mdb file and choose properties.

+>In the Properties dialog on the Opens With link, click the Change button.

+>In the Change dialog, click on the Browse button and navigate to msaccess.exe on the machine.  It should be in one of the following locations:

 

C:\Program Files (x86)\Microsoft Office\Office14\MSACCESS.EXE

C:\Program Files\Microsoft Office\Office14\MSACCESS.EXE

 

+>Once you find it, double click on it.

+>In the “Opens With” dialog, make sure you select Microsoft Access and also check the “Always use the selected program” check box.

+>Click Ok and then Ok again.

 

After going through these steps try opening the file again.

 

Best Regards,
