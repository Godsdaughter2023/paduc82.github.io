bernard pauwels say: I have one more in my list of checks .... because we use a proxy server to access the Internet.

GPRS and (internet) wifi always works fine, because they come from the internet.
USB is a problem sometimes if used on the intranet. (PC connected on the intranet network)

So far I checked and had most success by making sure the connection was not over the edge proxy server towards the internet. (The exchange server is on the intranet, and is published by an ISA server to the internet. There are 2 DNS servers to split the IP addresses internet/intranet resolution) The internet access is always OK.

The edge proxy server (another ISA) for 'surfing' defines the Exchange server as internal (client settings on the ISA, filling in the values in WPAD.DAT). So the proxy is NOT used by IE to access the exchange server, automatically.

However I learned that activesync copies that IE (WPAD.DAT) information, but has a different interpretation. It is OK for IE if the proxy server is filled in, and automatic is active. (The IE will decide correctly). However in that case Activesync ALWAYS uses that proxy, independent of the automatic configuration entries in WPAD.DAT)

So I need at least 3 settings to avoid the proxy server.
1. The proxy may not be defined in IE for LAN. Only the automatic configuration is check-marked.
2. Activesync on the PC must be configured as connected to "the Internet"
3. On the PDA "use advanced network functionality" must be unchecked.

If one of the conditions is different, then the traffic goes to the proxy server, and fails (because the proxy does not service intranet servers, and also needs user authentication (respons 407))

It works in 99% of the cases.
