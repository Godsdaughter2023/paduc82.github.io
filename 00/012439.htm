This should do it:

Sub doiteverywhere()
Dim myStoryRange As Range
'First search the main document using the Selection
Selection.HomeKey wdStory
Selection.Find.ClearFormatting
Selection.Find.Font.Italic = True
With Selection.Find
    Do While .Execute(FindText:="", MatchWildcards:=False, Forward:=True, Wrap:=wdFindStop) = True
        With Selection
            .InsertBefore "<"
            .InsertAfter ">"
            .Collapse wdCollapseEnd
        End With
    Loop

For Each myStoryRange In ActiveDocument.StoryRanges
    If myStoryRange.StoryType <> wdMainTextStory Then
        myStoryRange.Select
        Selection.Collapse wdCollapseStart
        With Selection.Find
            Do While .Execute(FindText:="", MatchWildcards:=False, Forward:=True, Wrap:=wdFindStop) = True
                With Selection '.Range
                    .InsertBefore "<"
                    .InsertAfter ">"
                    .Collapse wdCollapseEnd
                End With
            Loop
        End With
        Do While Not (myStoryRange.NextStoryRange Is Nothing)
            Set myStoryRange = myStoryRange.NextStoryRange
            myStoryRange.Select
            Selection.Collapse wdCollapseStart
            With Selection.Find
                Do While .Execute(FindText:="", MatchWildcards:=False, Forward:=True, Wrap:=wdFindStop) = True
                    With Selection '.Range
                        .InsertBefore "<"
                        .InsertAfter ">"
                        .Collapse wdCollapseEnd
                    End With
                Loop
            End With
         Loop
     End If
Next myStoryRange
ActiveWindow.View = wdPrintView
Selection.HomeKey wdStory
End Sub

ronrosenfeld say:

Assuming the weekday abbreviations are in column 7, and are text strings, you can use:

   =(COUNTIFS(E9:AI9,"P",$E$7:$AI$7,"S*"))

Note that only Sa and Su start with an S, so the * wildcard character picks up both.   I made the range reference absolute, which may or may not be appropriate for your worksheet.

andreas killer say:

but I'd like to keep everything under "Toolbox." (in the same way that ActiveSheet has multiple objects within it - rows, cells, columns, etc).

--------------------------------------------------------------------------------------

You can do that, but you have to go the same way as the VBA object model in Excel, your main object must provide a collection (like Sheets, Worksheets, Shapes, ChartObjects, etc.) and maybe some global object variables like (ActiveSheet, ActiveWorkbook).

The technical details are more complicated, let us assume that we want to have a collection named Configs and a global variable named ActiveConfig.

So if you create a class Config (from above) you have to add it to the collection Configs and assign it with a key (i would use the code name property). We can put all that into another class called ConfigTools:

Option Explicit

Private MyConfigs As New Collection

Sub Add(ByVal ThisSheet As Worksheet)
  Dim C As New Config
  Set C.Ws = ThisSheet
  MyConfigs.Add C, ThisSheet.CodeName
End Sub

Property Get Configs(ByVal Item As Variant) As Config
  If IsObject(Item) Then
    If TypeOf Item Is Worksheet Then
      Set Configs = MyConfigs(Item.CodeName)
    Else
      Err.Raise 5, "ConfigTools.Configs", "Invalid object, Worksheet required"
    End If
  Else
    Set Configs = MyConfigs(Worksheets(Item).CodeName)
  End If
End Property

And your main routine can look like this

Sub Test()
  Dim ToolBox As New ConfigTools
 
  ToolBox.Add Sheets(1)
  ToolBox.Add Sheets(2)
  Debug.Print ToolBox.Configs(Sheets(1)).ShowPRIMEToolboxTab.Address
  Debug.Print ToolBox.Configs("Sheet2").ShowPRIMEToolboxTab.Address
End Sub

To make the "ActiveConfig" work you have to install an Application Event handler and catch the SheetChange event. Inside that you can check if a Config object exists inside your collection and if so set ActiveConfig to that object, otherwise to Nothing.
