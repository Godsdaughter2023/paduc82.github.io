For the most part, questions are responded to by other users who volunteer their time and expertise.  It may take some time -- particularly around the holidays -- for someone to respond.

There are a few things to try when a file can't be deleted because it's "in use."

  1. Determine the full path to the file in question by right-clicking on the file, selecting Properties, and looking at the file's "Location."  For example, if you're trying to delete BADFILE.TXT from your Desktop and you are logged in as KENNY P, the full path would be C:\Users\KENNY P\Desktop\BADFILE.TXT
  2. Open a Command Prompt window (click the Start Orb, type cmd.exe and press Enter)
  3. Leaving the Command Prompt open, open Task Manager (press CTRL+SHIFT+ESC) and select the "Processes" tab
  4. Click on "explorer.exe" and then click the End Process button; the icons on your Desktop will disappear, as will the Task Bar -- this is OK
  5. In the Command Prompt type DEL followed by the full path to the file and press Enter.  If there are any spaces in the path, surround it with double quote marks (").  Using the example from step 1, you should type
        del "C:\Users\KENNY P\Desktop\BADFILE.TXT"
  6. To restore the Desktop, go back to the Task Manager window, click File, then click New Task (Run ...), then type explorer.exe in the Open box and click OK
  7. Close the Command Prompt and Task Manager windows

If the method above didn't work, you can unlock the file using a third-party tool.  The easiest tool to use used to be Unlocker, but that tool is now bundled with a "Potentially Unwanted Program" -- a browser toolbar -- that will result in the download being blocked by many antivirus programs.  You can still download it by temporarily disabling your a/v and then being very careful in the installation process to select the "Custom" installation option and UNcheck the boxes to install the toolbar.
