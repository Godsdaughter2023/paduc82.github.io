john sr wilson say:
You should use Paragraphs I guess. Paragraphs can have more than one line if the text wraps. Your third paragraph has two lines for example.

When you say current selected paragraph do you mean the cursor could be in it (ie no text selected) The way to locate the paragraph index is to compare the paragraph end with the character start position. A little clumsy but it's the only way I know.

Sample Code (based on an idea from Shyam Pillai) Note the code assumes a textrange selection you will need to add some error checking.

Sub checkCursor()
Dim lngCharStart As Long
Dim lngParaEnd As Long
Dim L As Long
Dim oShp As Shape

lngCharStart = ActiveWindow.Selection.TextRange.Start
Set oShp = ActiveWindow.Selection.TextRange.Parent.Parent
With oShp.TextFrame2.TextRange
Debug.Print lngCharStart & .Length
If lngCharStart > .Length Then ' selection is at end!
MsgBox "Cursor is in Paragraph " & .Paragraphs.Count
Else
For L = 1 To .Paragraphs.Count
lngParaEnd = .Paragraphs(L).Start + .Paragraphs(L).Length

If lngCharStart < lngParaEnd Then
MsgBox "Cursor is in Paragraph " & L
Exit For
End If
Next L
End If
End With
End Sub

omer atay say:

Pedro, I checked with some people here and it looks like we don't necessarily always persist collapsing information. This was also the case with OneNote 2010, but you may not have hit it much.

 

This may not make much sense at first (why would we not persist a change you make?). The collapse state of a paragraph is sort of per window. If you have two OneNote windows displaying the same page, you can see that you are allowed to collapse a level in one window, while the other window still has it expanded. This is sometimes useful and is achieved by keeping this state in memory for a while not immediately saving it to the page (otherwise the other window would display the change immediately).

 

We will persist the information though if you make other changes to the page. For instance if you type on the page, the collapse state will also be persisted, so that you will see them as you left next time.

 

This is similar to selection and scroll. Your selection on the page and the scroll position is also persisted but not as aggressively as the content. So you may end up finding the selection change after leaving a page and coming back. However it allows multiple windows to have different selection and scroll position on the same page.

 

Hope this clears a bit. I know it doesn't really help you the way you want.
