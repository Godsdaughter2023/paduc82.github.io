tobytu_msft say:

Hi HelpStar,

There are three types of retention tag. End user can only view and apply the “Personal tags”. And the Personal tags will not automatically apply to the end user’s mailbox. One folder can only apply one Personal tag, so there is no conflict.

Default policy tags (DPTs)   These are default retention tags for the entire mailbox. Automatically to entire mailbox

Retention policy tags (RPTs)   These are for default folders. The only valid action is to delete or delete permanently. Automatically to a default folder

Personal tags   These tags become available in Outlook and Outlook Web App. Users can use them to apply to a mailbox folder or an individual item. Manually to items and folders

For more details, please refer to this article.

So if you don’t add personal tags to this user’s retention policy, he/she is not able to customize their retention policy.

To check the end user use what retention policy, please sign in to EAC > Recipient > Mailbox > Double click the mailbox > Mailbox feature > Retention Policy.

Of course, we can also use the PowerShell to check it.  For your reference:

https://technet.microsoft.com/en-us/library/dd298086(v=exchg.160).aspx

A simple example of code creating code follows:

 

Sub CreateProcedure()
    Dim VBComp As VBIDE.VBComponent
    Dim CodeMod As VBIDE.CodeModule
    Dim S As String
    Dim LineNum As Long
   
    Set VBComp = ThisWorkbook.VBProject.VBComponents.Add(vbext_ct_StdModule)
    VBComp.Name = "NewModule"
    Set CodeMod = VBComp.CodeModule
    LineNum = CodeMod.CountOfLines + 1
    S = "Sub ABC()" & vbCrLf & _
            "Range(""A1"").Value = 1234" & vbCrLf & _
        "End Sub"
    CodeMod.InsertLines LineNum, S
End Sub
