jay freedman say:

Chris,

The "official" Microsoft way to write Office add-ins in C# is to use Visual Studio and Visual Studio Tools for Office (VSTO) together. The sticky part is that Visual Studio Express, which is a free download, doesn't work with VSTO, and the versions of Visual Studio that do work with it are quite expensive. So forget that.

Bearing in mind that I haven't used either of these, there are two free-download development tools that I've seen recommended: NetOffice and SharpDevelop. NetOffice is a replacement for the VSTO code libraries, and according to this thread it will work with either SharpDevelop or Visual Studio Express. I hope you don't find SharpDevelop as flaky as the VBA editor.

There may be some features in .Net 4.5 that you'd eventually want to use, but for most purposes .Net 3.5 or later should be sufficient.

The Office Primary Interop Assemblies (PIA) come with Office, but might not have been installed depending on what installation options you chose. If they're there, you'll see an entry in the Windows Control Panel > Programs & Features list. If necessary, you can install them later; see How to: Install Office Primary Interop Assemblies. Per another MS page, "By default, PIA’s are embedded in your solution when you build it so you don’t have to distribute PIAs to users as a prerequisite to using your add-in or customization."

Once you get into the coding, you'll find that a lot of what you want to do is fairly similar to coding in VBA. The objects and their methods and properties mostly have the same names. The biggest difference is that C# doesn't let you just omit optional parameters; if you want to accept the default value of an optional parameter, you need to pass ref missing in that position.
