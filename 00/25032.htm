As for the keyboard shortcut: In Office 2011 & prior Shift+Command+Z was internally assigned to the ResetChar command long before Apple started commandeering key combinations. I don't know which programs you're referring to, but the odds are that they either are Apple programs or are relatively new products from 3rd party developers who elected to adopt the "package" supplied by Apple. e.j. gun say:

The good news is that I was able to duplicate your result exactly in Excel 2007!

 

The bad news is - I get the same BAD result.  A value of 0.775 (77.5%) should show green, but it shows yellow instead.

 

I did an experiment.  Created a data set from 1% to 100%, incrementing by 1%.  Applied the exact same conditional format.  I found that:

1.  The black range is applied up to and including 25%.  That's ALMOST correct, but it should not include 25% because it's supposed to be < 25%

2.  The red range goes from 26% to 50%.  Again it should not include 50%.

3.  The yellow range is between 51% and 75%

4.  The green range is between 76% and 100%

So everything appears almost normal for this data.  I suspect decimal math is causing the boundaries at 25%, 50% and 75% to be in the wrong range (i.e. what looks like 25.0% could really be 24.99999%).

 

So why does your data not work correctly?  I did another experiment.  I edited the conditional formatting rule, and changed the "Type" from Percent to Number, and changed the ranges to <0.25, (0.25 to 0.5), (0.5 to 0.75) and (0.75 to 1.0).  All of the sudden, I get the correct stoplight for each value!  Same data, just changed the formatting rule a little.

 

That still doesn't explain why the conditional formatting rule using Percent doesn't work right, but at least it provides you a workaround.

 

Bug?  Probably!

 

Eric
