john sr wilson say:

John is actually telling you that the ONLY way to do what you need is to replace the placeholders. There is no way to reformat them. 

You could write a macro to automate the process but unless you need to do this often it will probably take longer than just doing it (assuming you know how to use macros)

As it happens I have a saved macro you can use which will do it in seconds:

Sub toText()
   Dim sngL As Single
   Dim sngT As Single
   Dim sngW As Single
   Dim sngH As Single
   Dim oshp As Shape
   Dim L As Long
   Dim odes As Design
   Dim ocust As CustomLayout
   For Each odes In ActivePresentation.Designs
      For Each ocust In odes.SlideMaster.CustomLayouts
         For L = ocust.Shapes.Count To 1 Step -1
            Set oshp = ocust.Shapes(L)
            If oshp.Type = msoPlaceholder Then
               If oshp.PlaceholderFormat.Type = 7 Then
                  sngL = oshp.Left
                  sngT = oshp.Top
                  sngW = oshp.Width
                  sngH = oshp.Height
                  oshp.Delete
                  ocust.Shapes.AddPlaceholder 2, sngL, sngT, sngW, sngH
               End If
            End If
         Next L
      Next ocust
   Next odes
End Sub

If you don't know what to do with it I suggest you just do it manually which will be quicker than coming back to get details of what to do! Whatever use it on a COPY just in case.

scott mcdaniel say:

If no references are marked as MISSING, then you most likely are not missing any files required by your application.

I'm not sure what your code is doing, but you've got quite a bit of redundancy in there. For example:

If DLookup("[productid]", "products", "[productid]= forms![products]![productid]") = Me![productid] Then

That line seems to be looking up the ProductID from a table named Products, based on the value found in the Products!ProductID. If the "Products" form is based on the table named Products (and I'm betting it is), then you don't need to lookup anything - just compare the two:

If Forms("Products")!ProductID = Me!ProductID Then

Next, SendKeys is notoriously unreliable, and in unnecessary in almost every case. I don't know what you're doing with that call to SendKeys, but if you could explain a bit more about what you're after, we can help you to clean up the code. 

Finally, Access 2007 introduced the builtin DatePicker. You should modify your database to make use of that, and remove the reference to mscommctl. While many use that reference with success, that library is not compliant with Access and as you move forward in Access versions you'll find more and more trouble with it. Best to "cut the cord" now.

SELECT ContactID, FirstName & " " & LastName, 1 As SortColumn, LastName,Firstname

FROM Contacts
ORDER BY SortColumn, LastName, FirstName;

Its AfterUpdate event procedure is:


    Const MESSAGETEXT = "No matching record"
    Dim ctrl As Control
    
    Set ctrl = Me.ActiveControl
    
    If Not IsNull(ctrl) Then
        If ctrl = 0 Then
            ' go to new record and move focus to FirstName control
            DoCmd.GoToRecord acForm, Me.Name, acNewRec
            Me.FirstName.SetFocus
        Else
            With Me.RecordsetClone
                .FindFirst "ContactID = " & ctrl
                If Not .NoMatch Then
                    ' go to record by synchronizing bookmarks
                    Me.Bookmark = .Bookmark
                Else
                    MsgBox MESSAGETEXT, vbInformation, "Warning"
                End If
            End With
        End If
    End If

    Dim strMessage As String

    strFirstName = Left(NewData, InStr(NewData, " ") - 1)

    'if user confirms insert move form to new record

It would appear that someone recorded three quick macros and allowed the default name rather than give them a more permanent name. You can,
