rrslider say:

If you have multiple shapes that need a longer duration this is the code I used for four shapes. there's probably an easier way but i'm not an expert.

Sub FixTiming()
With ActivePresentation.Slides(1).TimeLine
With .MainSequence.Item(1)
.Timing.Duration = 500
End With
With ActivePresentation.Slides(1).TimeLine
With .MainSequence.Item(2)
.Timing.Duration = 500
End With
With ActivePresentation.Slides(1).TimeLine
With .MainSequence.Item(3)
.Timing.Duration = 500
End With
With ActivePresentation.Slides(1).TimeLine
With .MainSequence.Item(4)
.Timing.Duration = 500
End With
End With
End With
End With
End With

End Sub

Note: I was making a sand timer that lasted for five minutes.

scott okeif say:

 

First, if you don't already get yourself a MicroSD or SDHC card for additional storage space.

 

Second, note that Storage space and program space are not the same, however on some pre-WM6 devices users could allocate storage space for program memory or vice versa. I'm unsure if your device will allow such a thing.

 

Third, best place to look and be for HTC device is the XDA-devlopers site (there are others, but none as specific for HTC's). I'll list the links below:

 

http://forum.xda-developers.com/index.php -- Any and everything related to HTC's

 

http://ppcgeeks.com/  -- a moderate amount of the above information, again, not as device specific

 

http://pdaphonehome.com/ -- genral device info, beginner-friendly

 

Foruth, Storage location, as with size, is application specific. Some developers choose to store data on storage cards, while others use the main device storage. But, there are many applications out there that don't specify during install. So,  1) Check the latest FAQ's, developer blog, ReadMe file, or other on the specific application and 2) Use this rule of thumb:

 

  • Any application that needs to be started at device boot-up, should,  more often than not,  be run/stored in the device memory/storage;
  • Any application that directly integrates with the device ROM/Shell should also be run from the device storage/memory--most of the time;
  •  Any application which is purely cosmetic or data-oriented in nature (such as a Contacts Program, Google Maps, or OneNote) can be stored on memory cards (or off-of-device);
  •  All media related files (Pictures, Movies, Music, Document-centric) should definitely be stored on seperate memory (off-of-device memory).

 

Hope that helps!

linley meslier say:

By files I assume you mean documents, pictures, music etc. For those, it is recommended you store them on a storage card. Most devices have a slot for SD, mini SD, Micro SD and SDHC card. Store your files on your card. As you said, the phone main memory is limited. most devices have only 64MB of space available.

 

What's the difference between Program and storage memory?

 

Program memory is temp memory used by programs. Applications you installed on your device need to have access to RAM to execute.

 

Use Storage memory to stoore files and programs. Some programs support being installed to the storage card. Starting with WM5, Microsoft introduced flash ROM. Therefore, data stored in ROM are not lost if your battery dies. So almost all devices have at least 64MB of RAM and 128MB of ROM.
