scottgem say:

First, Is the SubUnit defined by the other selections. In other words once you know the subUnit will you also know the subgroup, dept and div? That appears to be the case and if it is, then ONLY the SubUnit needs to be bound to a field. Having fields in your table for SubGroup, Dept and Div is redundant. 

Second, Each Rowsource should have the ID field as the first and bound column, that's why the cascading isn't working. The RowSources should be:

cboDivision:

ControlSource = Unbound

RowSource = SELECT [tblDivisions].[DivID], [tblDivisions].[DivisionAcronym], [tblDivisions].[DivisionName] FROM tblDivisions ORDER BY [DivID]; 

RowSourceType = table/query

BoundColumn = 1

cboDepartment:

ControlSource = Unbound

RowSource = SELECT tblDepartmentsColleges.DeptID, tblDepartmentsColleges.NameinSE FROM tblDepartmentsColleges WHERE (((tblDepartmentsColleges.DivID)=[Forms]![frmDeposits]![cboDivision])) ORDER BY tblDepartmentsColleges.[NameInSE]; 

RowSourceType = table/query

BoundColumn = 1

cboSubGroup:

ControlSource = Unbound

RowSource = SELECT tblSubGroups.SubGroupID, tblSubGroups.NameInSE FROM tblSubGroups WHERE (((tblSubGroups.DeptID)=[Forms]![frmDeposits]![cboDepartment])) ORDER BY tblSubGroups.[NameInSE]; 

RowSourceType = table/query

BoundColumn = 1

cboSubUnit: 

ControlSource = SubUnitID

RowSource = SELECT tblSubUnits.SubUnitID, tblSubUnits.NameInSE FROM tblSubUnits WHERE (((tblSubUnits.SubGroupsID)=[Forms]![frmDeposits]![cboSubGroup])) ORDER BY tblSubUnits.[NameInSE]; 

RowSourceType = table/query

BoundColumn = 1

Finally, Each combo should have a column count of 2 (except cboDivision which has 3) with the first column having a zero width.

ken sheridan say: When you move to a new row each combo box which references another combo box must be requeried in order that the rows returned by its RowSource  query will reflect the value of the referenced control.  If this is not done and the value of the referencing control is a hidden surrogate key, the control will appear empty.  Consequently you should ensure that in the form's Current event procedure each of the relevant controls is requeried.
