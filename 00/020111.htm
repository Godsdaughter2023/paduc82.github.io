hansv mvp say:

You don't need to add a field to the imported table.

Instead, create another table with two fields: Zip (primary key) and Num

Populate this table with all zip codes and the associated numbers:

You can now create a query based on the imported table and the new table, joined on the Zip Code field vs Zip.

If there are records without a zip code in the imported table, double-click the join line and select the option to return all records from the imported table (it's the 2nd or 3rd option, depending on how you set up things).

Add all fields or * from the imported table, plus the Num field from the new table.

You can use this query wherever you need the number associated with the zip code.

hansv mvp say:

I was aware of that, but I don't expect to be alive by then... ;-)

 

Here is a formula that should work for starting dates on or after March 1, 1900:

 

=IF(A1="",OFFSET(A2,-2,0)+1,IF(AND(DAY(A1)=28,MONTH(A1)=2,MONTH(A1+1)=3),"",A1+1))

 

(As you may know, Excel contains a deliberate error - it counts 1900 as a leap year)
