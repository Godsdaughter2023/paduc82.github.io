greginmich say:

There have always been some inconsistencies in MSE’s main-interface event logging for quarantined items, but sometimes these inconsistencies can be resolved by examining the details (detection paths and error descriptions) in the corresponding System Event Log entries, found in the “Microsoft Antimalware” event source:

 

1116 – Malware detection

1117 – Malware remediation

1118 – Malware remediation failure (non-critical)  

1119 – Malware remediation failure (critical)

 

We discovered long ago that items can be listed as “Quarantined” in the ‘All detected items” tab without actually having been quarantined:

 

http://answers.microsoft.com/en-us/protect/forum/protect_scanning/mse-how-to-remove-quarantined-items/638ba93f-187d-44e3-b4b5-434b179fb7a2

 

In English, the option to automatically delete quarantined files reads: “Quarantined files remain disabled until you allow them or remove them.” Once in a while we might want to restore an item from the Quarantine when it appears that the malware detection was a false alarm (a “false positive”). If we want the ability to retrieve wrongly-branded files, then we have to accept the risk of actually releasing malware from the Quarantine; and that’s why the decision to restore a file should always be based on plenty of diligent research. Restoring a file will automatically register a temporary path exclusion (in a "TemporaryPaths" registry key that’s invisible to the MSE interface). The restored file will thus no longer be monitored by MSE’s real-time protection, and will consequently be “allowed” to run. So by restoring a quarantined file we are in effect “allowing” that file.

 

Since MSE’s automatic exclusion for the restored file doesn’t show up in the “Excluded files and locations” settings, there’s no way to immediately re-enable real-time detection for the file; but when the automatic exclusion expires (after a couple of days), the ability to detect the “malware” in the file will be re-enabled, and scanning the file will return it to the Quarantine unless we intervene by manually setting up a path exclusion in the “Excluded files and locations” settings:

 

http://answers.microsoft.com/en-us/protect/forum/protect_scanning/mse-beta-false-positives/601b6136-9433-4482-881c-68fbea546a29

 

It might be a little less confusing if restoring a file from the Quarantine just automatically created a path exclusion for it in the main interface.

Step 1: Clean Boot (MSCONFIG).

Click the Programs Startup tab, Open Task Manager, and then disable any programs that are starting up with Windows.
Close the Task Manager window.
In the System Settings window, click OK, you will be prompted to restart the computer and confirm the request by clicking Restart.

Step 2: Reset the network settings.

Press the Windows key.
Search for Command Prompt, then right-click the Command Prompt icon, click Run as Administrator.

Run the commands below one at a time:
ipconfig / release

ipconfig / renew

ipconfig / flushdns

ipconfig / registerdns

nbtstat -rr

netsh int ip reset all

netsh winsock reset

netsh winsock reset all

netsh int 6to4 reset all

netsh int ipv4 reset all

netsh int ipv6 reset all

netsh int httpstunnel reset all

netsh int isatap reset all

netsh int portproxy reset all

netsh int tcp reset all

netsh int teredo reset all

Restart the computer.

Step 3: Repair the Office package.

Press the Windows + X keys.
Click Programs and Features.
Select the Microsoft Office package and click the Change button;
Click Repair, then click Quick Repair.
Verify that the problem

Method 2: Open Outlook in Safe Mode.

Press the Windows key + R to open the Run window.
Enter OUTLOOK / SAFE and press Enter.
Select the Outlook profile that exhibits the problem and verify that the behavior persists.
Disable one add-in at a time in Safe Mode.

If the Office application starts in Safe Mode, disable the add-ins one at a time.

With the Outlook application open in Safe Mode, click File> Options> Add-ins.
Next to the Manage list, click Go ....
Uncheck one of the add-ins in the list, and then click OK.
Restart the application (this time, not in Safe Mode), verify that the problem persists.
Method 3: Create a new Outlook profile.

Set up the email account and test with the Outlook application.
