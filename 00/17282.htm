ken sheridan say: I think this should do it:

SELECT MemberID, MIN(MD1.EffectiveDate) AS EffectiveDate
FROM MemberDates As MD1
WHERE EXISTS
    (SELECT *
     FROM MemberDates AS MD2
     WHERE MD2.MemberID = MD1.MemberID
     AND MD2.EffectiveDate - MD1.TermDate = 1)
GROUP BY MemberID;

where MemberDates is the name of the table.

miker_123 say:

Hi Marcos,

The example provided by Ken works for the sample data set you offered, but in my testing, only handles gaps of one month.  I approached the problem by using two queries

The job of the first query (qryNonContinuousPeriods) is to isolate all records for non-continuous periods that are to be excluded from consideration;

SELECT MD2.MemberId, MD2.EffectiveDate, MD2.TermDate
FROM MemberDates AS MD1 INNER JOIN MemberDates AS MD2 ON MD1.MemberId = MD2.MemberId
WHERE (((DateDiff("d",[MD2].[TermDate],[MD1].[EffectiveDate]))>1))
GROUP BY MD2.MemberId, MD2.EffectiveDate, MD2.TermDate;'

The second query (qryContinuousEffectiveDates) outter joins your table (MemberDates) onto the first query to return the earliest effective date in a continuous period grouped by MemberId.

SELECT MemberDates.MemberId, Min(MemberDates.EffectiveDate) AS MinOfEffectiveDate
FROM MemberDates LEFT JOIN qryNonContinuousPeriods ON (MemberDates.MemberId = qryNonContinuousPeriods.MemberId) AND (MemberDates.EffectiveDate = qryNonContinuousPeriods.EffectiveDate) AND (MemberDates.TermDate = qryNonContinuousPeriods.TermDate)
WHERE (((qryNonContinuousPeriods.MemberId) Is Null) AND ((qryNonContinuousPeriods.EffectiveDate) Is Null) AND ((qryNonContinuousPeriods.TermDate) Is Null))
GROUP BY MemberDates.MemberId;

The query would be much simpler if you had a single field primary key in your MemberDates table, but I've used the 3 fields provided as a multi-column primary key which may fail if you have two rows with the same MemberId, EffectiveDate, and TermDate.  I hope this helps, good luck!

andreas killer say:

Maybe you set a reference to the AddIn within VBA everywhere?

Try as follows:

- Close Excel

- Zip the AddIn (including all files)

- Delete the AddIn (not the zip)

- Open Excel

- Go to the AddIns section and remove the AddIn

- Close Excel

- Open Excel

until you get not error messages anymore and the Excel\AddIns section doesn't show your AddIn.

If you get the error message as before, check the references within your PERSONAL.XLSB and check also the Startfolder of Excel for any files or links.

- Close Excel

- Unzip and restore the AddIn files

- Open Excel
- Add the AddIn as usual (and you did before)

- Close Excel

- Open Excel

Any error messages? The AddIn is not loaded?

If so, you don't have permissions to install AddIns, contact your admin.
