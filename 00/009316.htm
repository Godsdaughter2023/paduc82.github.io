In the SQL View (or Design View) I feel that there must be a way to add a command under each "Select" section.

Example:
     SELECT [Title]
     FROM [Books]
     WHERE [Author Last]="Asimov"

I know what I want to do and almost know how to do it.  I simply don't know how to write the command if it is possible.  
hansv mvp say:

You can do it like this:

 

SELECT Title, "Book" AS Genre FROM Books WHERE [Author Last]="Asimov"
UNION SELECT Title, "Short Story" FROM [Short Stories] WHERE [Author Last]="Asimov"
UNION SELECT Title, "Article" FROM Articles WHERE [Author Last]="Asimov"

 

Note that you have to specify Genre as field name (column header) in the first SELECT statement only.

    joeu2004 say:

    StevenAlcantara wrote:

    =SUMPRODUCT((IF($C$9:$C$44="ü",($E$9:$E$44),0)),F9:F47)

    I have looked on google for hours trying to figure out what is wrong with it but I cannot for the life of me figure it out.

    A formula of that form must be array-entered; that is, press ctrl+shift+Enter instead of just Enter.  Then it will be surrounded by curly braces ( "{ ... }" ) in the Formula Bar.  That is just for appearances.  We cannot type the curly braces directly.

    However, a potentially equivalent formula can be normally-entered (press just Enter as usual), to wit:

    =SUMPRODUCT(--($C$9:$C$44="ü"), $E$9:$E$44, F9:F47)

    The double negate ("--") converts TRUE and FALSE into 1 and 0, which SUMPRODUCT requires in order for the formula to do what you expect.

    However, there is significant difference between the two formulas:  the original formula is necessary if any cell in E9:E44 might contain an Excel error, namely #VALUE, #NUM, #DIV/0, etc.

    But in that case, you can simplify your original formula (remember to array-enter by pressing

    =SUMPRODUCT(IF($C$9:$C$44="ü", $E$9:$E$44, 0), F9:F47)

    Using If statement will require to enter as an array formula (CTRL+SHIFT+ENTER).

    =SUMPRODUCT(($C$9:$C$44="ü")*($E$9:$E$44)*($F$9:$F$44))
