ics76 say:

I have the same annoyance with the same laptop, a Spectre 13 x360. I have found a workaround that I can live with, although it is far from ideal.

When in normal laptop mode (device orientation should be as a normal laptop), I go into the Device Manager and I disable the orientation sensors. On my unit this is under System devices the "Intel(R) Integrated Sensor Solution". In practice this now locks the unit in laptop mode. When I want its normal behavior back, I enable the device again in Device Manager.

A word of caution: I'm sure HP did not mean for this device to be disabled and enabled all the time, so I'm not sure this is a good idea, it just works for me.

For whomever is interested in the cause:
This is not a Windows problem, the problem is unique to this HP. The sensors in the HP can sense orientation of the laptop as well as the angle on the hinges of the screen. These sensors together drive 2 things: the switch to tablet mode, and the disabling of keyboard and touchpad. These 2 things are driven separately and independently, which is the problem: if you overrule tablet mode because you want to stay in laptop mode, you are still left with a rotated screen and a disabled keyboard/touchpad... It would have been much more useful to me if the sensors would drive the switch to tablet mode (which I can as a user agree to or not) and then the switch to tablet mode triggers the disabling of keyboard and touchpad.

This was almost a deal breaker for me, I had almost returned the laptop before I found this workaround.
Hope this is useful for someone else.

Try the steps listed below:

b. Right click on Local Area Connection and click on Properties. c. Double click Internet Protocol Version 4 (TCP/IPv4) and assign the IP address manually.

d. Let the Subnet mask and default gateway be the same.

Default gateway, usually would be 192.168.0.1 or 192.168.1.1

a. Click on Start
b. In the start search box type 'devmgmt.msc' and press enter
c. In the Device Manager, expand Network Adapters

While uninstalling the device drivers make sure you place a check mark against "Delete the driver software for this device".

http://windows.microsoft.com/en-US/windows7/Turn-Windows-Firewall-on-or-off

netsh winsock reset catalog that resets WINSOCK entries to installation defaults.
netsh int ipv4 reset reset.log that resets IPv4 TCP/IP stack to installation defaults.
netsh int ipv6 reset reset.log resets IPv6 TCP/IP stack to installation defaults.
