One problem with the solution is that if the user does not make any change to the data, the before update event will not be triggered. 

You could use the form's current event to force a change.  Just set any bound control to its own value in the current event.  That will dirty the form and the form's before update event will be triggered.

That said, the posted solution is probably the best you can do.

hansv mvp say:

You can use the Before Update event of the form. This occurs when the record is about to be saved, whether because the user is moving to a different record or because the user is closing the form. You can inspect values, and set Cancel = True if the conditions aren't met; this will cancel the update. For example:

Private Sub Form_BeforeUpdate(Cancel As Integer)
    If Me.chkSomething = False Or Me.chkSomethingElse = False Then
        MsgBox "You can't exit this record before you have ticked the check boxes!", vbExclamation
        Cancel = True
    End If
End sub

microsoft advertising - shelly say:

Hi Kellen W,

First I want to thank you for choosing Bing and taking the time to submit this inquiry.

The best practice for more than ten websites, you should consider creating multiple UET tags.

In Bing Ads, UET tags are stored at the customer level so if you have multiple accounts under the same customer the UET tag is shared across all accounts. We recommend that if each account has a different website that you create a separate UET tag because:

·         Incorrect status: In Bing Ads, we show the tracking status for each tag that tells you whether or not Bing Ads can see the UET tag on your website. If the same tag is used for multiple websites, it is possible that only one of the websites has been properly installed with a UET tag, but you would not be able to tell since Bing Ads will mark the status of the UET tag as active.

·         Incorrect data: Websites pages can have the same file name, for example, thankyou.html. So when you define a conversion goal, there is a higher chance that a goal set with "destination URL contains thankyou" will match both websites and then lead to incorrect conversion counting.

If you have a separate UET tag for each website, you avoid these issues.
