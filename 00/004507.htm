    SMullaney,

    Agreeing with John….

    1. You can’t display hourly scale on a Calendar view.  Calendar views seem pretty rarely used, and there’s not much customization available.
    2. Even if you could show an hourly scale, task bars in both Calendar and Gantt views extend continuously from the start to the finish of each task.  They do not typically neck down, split, or otherwise display non-working time, so any tasks taking more than one shift to finish are going to muddle the view.
    3. Assuming you are using task and/or resource calendars to distribute the work, then the only workarounds I could suggest would involve displaying a customized Task Usage view.

    Here’s a simple example project showing concurrent 3-shift work over 5 days (i.e. there’s no logic and each shift has 40 crew-hours of work to do).  The work is scheduled 4 different approaches – differing with regard to resource and calendar assignments.  Each approach leads to exactly the same schedule outcome.  As shown, none of the bars differentiate the three different shifts except on the starting or finishing day. 

     

    I’ve selected all eight bars and displayed the corresponding Task Usage view in the lower pane with an hourly timescale applied.  If you schedule with Task Calendars, then the custom cell background on the “Work” field in the Task Usage view might give you what you need – these are the top four rows in the table.  On the other hand, if you schedule using Resource Calendars, then the custom cell backgrounds are incorrect, and the work values (i.e. numbers) should be used instead.

    Overall, using the Task Usage view in the lower pane can be a bit cludgy.  I’d suggest just using Task Usage in a single/upper pane view.  I’ve shown that here, along with an illustration of the method for shading the cell backgrounds.  You might get what you’re looking for by combining this with a weekly filter.

    brook zhang msft say:

    Hi Dusan,

     

    Sorry for the delay.

     

    For modern authentication, once the Outlook client is authenticated with MFA, two tokens named access token and refresh token will be stored. The access token is used for accessing Office 365, which is valid for 1 hour. The refresh token is used to obtain a new access token if the initial authentication is still valid.

    The Refresh token is valid for 14 days but if you are continuously using your mailbox during this period it can last up to 90 days. This is the mechanism of modern authentication.

     

    To be specific, when you close Outlook, the refresh token is still here. There will be an access token next time you open the Outlook client. Therefore, no authentication is needed. And if you delete the credentials in the Credentials Management, the refresh will be deleted too. That’s the reason why you need to re-authenticate with MFA.

    Formula to prevent duplicates in Excel spreadsheet - Microsoft Community

    ken krug say:

    Formula to prevent duplicates in Excel spreadsheet

    How do I prevent users from entering duplicate values in an Excel spreadsheet.  Numbers are formatted as follows;

    5-R-1, 5-R-2, 5-R-3, etc.  Thanks for the help!!!

    hansv mvp say:

    Let's say you enter these values in B2:B100.

    Select this range; I will assume that B2 is the active cell within the selection.

    On the Data tab of the ribbon, click Data Validation.

    Select Custom from the Allow dropdown.

    Enter the formula =COUNTIF($B$2:$B$100,$B2)<=1 in the Formula box.

    Activate the Error Alert tab.

    Enter an appropriate message to be displayed if the user enters a duplicate value.

    Click OK.
