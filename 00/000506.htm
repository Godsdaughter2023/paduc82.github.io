Can I add automatic line numbers to records in a sub form - Microsoft Community

robfm say:

Can I add automatic line numbers to records in a sub form

I have created an Access 2003 application that my department uses for writing purchase requisitions.  The design is similar to many of the templates and demos that Microsoft provides.  When a user opens the application they see a main form that asks for their name, account number date, etc and then there is a sub-form where they enter the specifics for materials they are ordering.  Information from the main part of the form goes into one table (RequisitionsTbl) and the individual line items that are entered in the subform end up being stored in a separate table (OrderDetailsTbl).  The subform has a default view of datasheet and the 2 tables are linked through a field called ReqID.

One of the frequent  users has asked me to add line numbers to the sub form.  To do this, I created a field in the OrderDetailsTbl table and a corresponding Text-Box (LineNumbers) on the sub form, so that the person filling out the form can now enter a line number for each item.  However, it would be much nicer if the line numbers filled in automatically.   None of the MS templates have this feature and I am stumped on how to do this.

I tried putting a formula "=Max([LineNumber]) + 1" in the default value property of the LineNumbers Text-Box, but it did not work as I expect.  The LineNumber field in the first record/line goes to "1" as I expected, but then the second line defaults to "0", the third and forth lines both line go to "2" then the fifth and sixth lines are default to "3" and so on.

I suspect this is something people have figured out many times before, but I could not find any references that address it.

Thanks in advance for any suggestions.

Rob M 
ken sheridan say: I assume you want each subset of OrderDetailsTbl rows per requisition to be numbered from 1 rather than all the rows in OrderDetailsTbl to be numbered distinctly.  In which case I don't think any conflict can arise as another concurrent user would presumably always be inserting rows for a different requisition.  You can therefore user the subform's BeforeInsert event procedure, which will assign the number as soon as the user begins to insert a row in the subform.  The code would be:

    Me.LineNumbers = Nz(DMax("LineNumbers", "OrderDetailsTbl", ReqID = " & [ReqID]), 0) + 1

You should base your subform on a query which sorts the rows as follows:

SELECT *
FROM OrderDetailsTbl
ORDER BY ReqID, LineNumbers;

This will ensure that the rows show in the correct order if you navigate back to an existing record in the parent form. The inclusion of the ReqID column in the ORDER BY clause isn't really necessary in the context of the subform as it's recordset is restricted to the one ReqID value by virtue of the linking mechanism with the parent form, but should you use the query in another context it will keep each requisition's rows together as a group in the query's result table.

If an item other than the last for a requisition is deleted there will be a gap in the sequence of course.  If you want dynamic row numbering you'd have to do it in a computed column in the query rather than assigning the number to a column in the base table.

You should include ReqID and LineNumbers in a unique index of course.  The simplest way is to make them the OrderDetailsTbl  table's composite primary key, but if it already has a surrogate key then a separate index is imperative.  In Access 3003 this is done via Indexes on the View menu in table design view.

marshall barton say:

Max is an SQL function.  Because of some internal trickery, it can also be used in control source expressions.

VBA never heard of the aggregate functions, which makes sense because VBA does not operate on records.  You can use VBA/DAO to operate on recordsets that are based on a query with aggregation, but that is not part of the VBA language.

jeeped say:
In order to construct a usable cell address from concatenated text strings (or cell values), you need to use INDIRECT() so that Excel will treat the concatenated text string as a cell address. With theworksheet name in A1,

=INDEX(INDIRECT(A1&"!B2:B200"),MATCH(B1,INDIRECT(A1&"!A2:A200"),0))

Your INDEX(MATCH()) reads left-to-right so you could use VLOOKUP() if you prefer,

=VLOOKUP(B1, INDIRECT(A1&"!A2:B200"), 2, FALSE)

Note that you will need to wrap the worksheet name in single quotes if it contains a space.

=VLOOKUP(B1, INDIRECT("'"&A1&"'!A2:B200"), 2, FALSE)

It does no harm to add the quotes if they are not needed so it is probably better to include them in all cases.

dancinganders say:

Numbers merged from Excel do not show up correctly

macropod say:

I have created a mail merge with an excel file as my data source.  the numbers are showing up as xx.xxxxxxx when my data is only xx.xx.

Hi dancinganders,

To control number & currency formatting in Word, add a numeric picture switch to the mergefield. To do this:
. select the field;
. press Shift-F9 to reveal the field coding. It should look something like {MERGEFIELD MyData};
. edit the field so that you get {MERGEFIELD MyData \# $,0.00} (or whatever other numeric format you prefer - see below);
. position the cursor anywhere in this field and press F9 to update it;
. run your mailmerge.
 
Note: The '\# $,0.00' in the field is referred to as a numeric picture switch. Other possibilities include:
. \# 0 for rounded whole numbers
. \# ,0 for rounded whole numbers with a thousands separator
. \# ,0.00 for numbers accurate to two decimal places, with a thousands separator
. \# $,0 for rounded whole dollars with a thousands separator
. \# "$,0.00;($,0.00);'-'" for currency, with brackets around negative numbers and a hyphen for 0 values
 

The precision of the displayed value is controlled by the '0.00'. You can use anything from '0' to '0.000000000000000'.
 
If you use a final ';' in the formatting switch with nothing following, (eg \# "$,0.00;($,0.00);") zero values will be suppressed. Note that this suppresses 0s resulting from empty fields and from fields containing 0s
