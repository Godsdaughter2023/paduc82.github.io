¡firedog say:
i have looked at the change keyboard option and i havent changed it because i want to retain the spanish keyboard

Aha!

It's a common misconception that changing the input language requires a change of keyboard layout, caused not least by that Change keyboards tab and the options on it. Of course, some input languages will require a different keyboard layout to be of any use - Hebrew or Japanese, say. But there's nothing to stop you using a Spanish keyboard even though the input language is English.

When you install a different language on the Change keyboards tab, you have the option to specify the keyboard layout to be used with that language. There is of course a default, but you don't have to accept it. In your case, you should Remove English (United Kingdom), then Add it back again, but this time select + Keyboard > Show more > Spanish (or Spanish Variation, depending on whether you like to have ç next to ñ or north-east of it. Use the Preview button to see the difference).

You can then select English (UK) as your input language on the Language Bar, but your keyboard will still have the Spanish layout. With any luck, your spelling checkers will now realize that you're typing in English and act accordingly.

Hi Jrhooper,

The recipient will receive multiple emails if he belongs to more than one group because every group will deliver emails sent to it to all members. That’s how Office 365 group works.

Regarding “Can a new contact group be created from existing contact groups to avoid having to add individual contacts”, the answer is yes. And the steps below are the best practice:

1. Create a new Office 365 group.

3. Click it when there is a search result matches it and then click Save.

Mailbox Audit in O365 Query - Microsoft Community

miteshjain_931 say:

Mailbox Audit in O365 Query

Hi Team,

If we enable mailbox audit on a mailbox and if any of the global administrator deletes that mailbox itself will that event be captured in the audit log?

If yes then how do we track that log? 

garyzhu_msft3613 say:

hi mitesh,

after the mailbox audit log feature is enabled, please let your administrator follow the steps below to export the log:
1. connect to exchange online using remote powershell.
2. grant specific permissions: add the administrator to the following admin roles groups (exchange admin center (eac) > permissions > admin roles).
recipient management, records management and organization management
 
3. configure outlook web app to allow xml attachments by running the command below:
set-owamailboxpolicy -identity owamailboxpolicy-default -allowedfiletypes @{add='.xml'}
4. in eac, go to compliance management > auditing.
5. click export mailbox audit logs.
6. configure the following search criteria for exporting the entries from the mailbox audit log:
1) start and end dates   select the date range for the entries to include the exported file.
2) mailboxes to search audit log for   select the mailboxes to retrieve audit log entries for.
3) type of non-owner access
4) recipients   select the users to send the mailbox audit log to.
7. click export.
 
note: microsoft exchange retrieves entries in the mailbox audit log that meet your search criteria, saves them to a file named searchresult.xml, and then attaches the xml file to an email message sent to the recipients that you specified.

if you’ve successfully exported the audit log, you’ll receive a message sent from exchange. in exchange online, it may be a few days before receiving this message. the mailbox audit log (named searchresult.xml) will be attached to this message. if you’ve correctly configured outlook web app to allow xml attachments, you can download the attached xml file. more detailed information can be found in export mailbox audit logs.

please feel free to post back if you have any problems when performing the steps.

regards,
gary
