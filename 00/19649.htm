tom van stiphout say:
Type is a reserved word. Don't use it for field names.

Payment should not be a number; we have the Currency data type for that.

Make sure you have a unique index over the two FKs in OrderDetails. Frankly

OrderDetails.ID is not needed, and that unique index can be the PK. Otherwise your relations are correct, and thank you for enforcing them :-)

OrderDetails.Type does not belong; you already know if a detail is a supplier or customer detail by inspecting Partners.PartnerType.

I have never quite seen a database where vendor purchases and sales to customers are in the same set of tables, but I guess if you're strictly a middleman that makes some sense. It still twists my brain a bit too much though.

To separate customers and suppliers you can start with two queries:
qryCustomers:
select * from tblPartners where PartnerType=2;
qrySuppliers:
select * from tblPartners where PartnerType=1;
Then with any subsequent queries (e.g. CustomerOrders or PartnerOrders) you don't include tblPartners, but one of the two queries.

mccbleue say:

If you connect in USB Mass Storage mode, you are only accessing the phone's memory card which is only used for data storage, you are not accessing the phone's main memory where these memory shortage problems occur.

 

As for the memory usage, when you go to the File Manager and view the Memory Details, check the following:

 

(1) Free memory should be registering several megabytes (Mb) at least.

(2) Calendar and Contacts should be registering only kilobyte (Kb) figures, these are unlikely to be causing a problem.

(3) Messages should be registering a figure in Kb, if the figure goes into Mb then you probably have either MMS ot Bluetooth messages still in the inbox, in which case you should save the attachments to memory card and delete the original messages.

(4) Images should be very little, you shouldn't be storing photos and things here apart from perhaps an image that you use as a wallpaper, images should be stored on the memory card in general - if there are a lot in phone memory, simply move them to card; and make sure that the camera is set to save to memory card and not to phone memory.

(5) Sound files should be low too, if you have a small number of files that you use as ringtones, message alerts etc. then they are OK but put all other music/sound files on the memory card and not in phone memory.

(6) Video clips should be very low, preferably zero - there is no logical reason for the average user to store videos in the phone memory, so move anything that is here to the card.

(7) Documents should be a low figure, if you manage any kind of documents (such as with Quickoffice or PDF-readers), keep the documents on card.

(8) Java Apps shouldn't be too high, wherever possible install Java apps to memory card to save phone memory space.

(9) SIS applications - to some degree this is outside your control because some or all of this figure is represented by apps which are preinstalled on the phone, but you can keep this figure down by installing new Symbian apps to memory card wherever possible.

(10) Other files too is somewhat out of your control, there are any files that don't have extensions identifying them as one of the above types, and are usually system files or files related to apps that you use.

 

If you can't free up sufficient space by moving some items from phone memory to memory card and clearing the browser cache, you may need to reformat the phone and start again. Firstly, back up your data (contacts, messages etc.) using Ovi Suite - don't worry about data on the memory card because this isn't going to be affected by what we are about to do. Then type *#7370# into the dialler, the phone will ask you to confirm the reset then enter the security code (which is 12345 if you have not set a personal code), then the phone will shut down and restart after which you will need to restore your backed-up data and reinstall any third party apps that you use, but your memory counters will be back to a sensible level and all you need to do from then on is maintain them as per the ten-point list above.

Every object in a sheet (comments, charts, buttons, ...boxes, lines, whatever) can be retrieved as shape object.

Sub Test()
  Dim S As Shape
  For Each S In ActiveSheet.Shapes
    Debug.Print S.Name, S.Left, S.Top, S.Width, S.Height
  Next
End Sub

And any shape object can be moved by modifying the position and size properties. So if you want to copy the shape from one sheet to another use this:

Sub TestCopy()
  Dim Source As Shape, Dest As Shape
  For Each Source In Sheets(1).Shapes
    If Source.Type <> msoComment Then
      'Copy this shape
      Source.Copy
      'Paste into the second sheet anywhere
      Sheets(2).Paste
      'Get the shape object
      With Sheets(2)
        Set Dest = .Shapes(.Shapes.Count)
      End With
      'Move it to the same position as in first sheet
      Dest.Left = Source.Left
      Dest.Top = Source.Top
    End If
  Next
End Sub

Any cell in a sheet has also a Left, top, Width and Height property. If you want to copy shapes to a specific location (some cells), the Function below might be helpful.

Andreas.

Function CopyShape(ByVal S As Shape, ByVal R As Range, _
    Optional ByVal StartSnap As fmSnapPoint = -1, Optional ByVal EndSnap As fmSnapPoint = -1, _
    Optional ByVal Move As Boolean = True, _
    Optional ByVal Resize As Variant) As Shape
  'Copy or move and resize the shape S to range R
  '  If R has areas then
  '     the shape is build from first area to last area in R
  '     if IsMissing(Resize) then Resize = True
  '  else
  '     the shape is build to fit R
  '     if IsMissing(Resize) then Resize = R.Cells.Count > 1
  Dim C As Comment
  Dim Left As Single, Top As Single, Width As Single, Height As Single, X As Single, Y As Single
  Dim SnapPoint As fmSnapPoint
  Dim Area As Range
  Select Case S.Type
    Case msoComment
      'Search the comment object
      For Each C In S.Parent.Comments
        If C.Shape.Name = S.Name Then
          'Copy the cell
          C.Parent.Copy
          'Paste the comment only to the destination cell(s)
          R.PasteSpecial xlPasteComments, xlPasteSpecialOperationNone
          Application.CutCopyMode = False
          'Delete old comment
          If Move Then C.Delete
          Exit Function
        End If
      Next
    Case Else
      If IsMissing(Resize) Then _
        If R.Areas.Count > 1 Then Resize = True Else Resize = R.Count > 1
      'To a different sheet?
      If S.Parent.Name <> R.Parent.Name Or Not Move Then
        If Move Then S.Cut Else S.Copy
        R.Parent.Paste
        Application.CutCopyMode = False
        Set S = R.Parent.Shapes(R.Parent.Shapes.Count)
      End If
      'Calculate positions
      If R.Areas.Count = 1 Then
        If StartSnap < 0 Then
          Set Area = R(1, 1)
        Else
          Set Area = R
        End If
      Else
        Set Area = R.Areas(1)
      End If
      If StartSnap < 0 Then _
        If R.Areas.Count = 1 Then StartSnap = fmSnapPointTopLeft Else StartSnap = fmSnapPointCenter
      SnapPoint = StartSnap
      GoSub CalculatePos
      Left = X
      Top = Y
      If Resize Then
        If R.Areas.Count = 1 Then
          If EndSnap < 0 Then
            Set Area = R.Cells(R.Cells.Count)
          Else
            Set Area = R
          End If
        Else
          Set Area = R.Areas(R.Areas.Count)
        End If
        If EndSnap < 0 Then _
          If R.Areas.Count = 1 Then EndSnap = fmSnapPointBottomRight Else EndSnap = fmSnapPointCenter
        SnapPoint = EndSnap
        GoSub CalculatePos
        Width = X - Left
        Height = Y - Top
      End If
      'Move it
      S.Left = Left
      S.Top = Top
      If Resize Then
        S.Width = Width
        S.Height = Height
        If S.LockAspectRatio Then
          'We may exceeds the range now
          If S.Width > Width Then
            S.Width = Width
            'Place in vertical center
            S.Top = S.Top + (Height - S.Height) / 2
          Else
            'Place in horizontal center
            S.Left = S.Left + (Width - S.Width) / 2
          End If
        End If
      End If
      Set CopyShape = S
  End Select
  Exit Function

CalculatePos:
  Select Case SnapPoint
    Case fmSnapPointBottomCenter
      X = Area.Left + Area.Width / 2
      Y = Area.Top + Area.Height
    Case fmSnapPointBottomLeft
      X = Area.Left
      Y = Area.Top + Area.Height
    Case fmSnapPointBottomRight
      X = Area.Left + Area.Width
      Y = Area.Top + Area.Height
    Case fmSnapPointCenter
      X = Area.Left + Area.Width / 2
      Y = Area.Top + Area.Height / 2
    Case fmSnapPointCenterLeft
      X = Area.Left
      Y = Area.Top + Area.Height / 2
    Case fmSnapPointCenterRight
      X = Area.Left + Area.Width
      Y = Area.Top + Area.Height / 2
    Case fmSnapPointTopCenter
      X = Area.Left + Area.Width / 2
      Y = Area.Top
    Case fmSnapPointTopLeft
      X = Area.Left
      Y = Area.Top
    Case fmSnapPointTopRight
      X = Area.Left + Area.Width
      Y = Area.Top
  End Select
  Return
End Function

Creating a subform that shows all records from a table - Microsoft Community

rob ingles say:

Creating a subform that shows all records from a table

Hey all!

I need to create a subform that shows all the records from a table called QuantitativeAssessmentCatogries.  The table has a list of categories that require a yes/no and a comments.  I need the subform to be related to the main form.  I have created a subform related to a main form with combo boxes to select the category but would prefer just to have the 10 categories all listed so the user does not have to keep selecting the next item from the combo box.  How would I go about doing this?

Thanks!

Rob

ken sheridan say: What you appear to be attempting is to insert rows into a table which models a many-to-many relationship type between categories and whatever entity type the parent form's table is modelling, the Yes/No and Comments columns being attributes of the relationship type.

As you say, this is easily done by means of a subform, in continuous forms view, in which rows are inserted, selecting a category in a combo box in each row.  Normally only those categories which relate to the parent form's current record would be entered, but you appear to want all possible categories to be represented in the table which models the relationship type, with a value of True in the Yes/No column for those which relate to the parent form's current record.  You might wish to reconsider this approach in favour of the standard approach, which would remove the need for the Yes/No column.  The only justification I can see for your non-standard approach would be if the Comments column were to contain values in those rows where the value of the Yes/No column is False.

If you do decide to proceed with your non-standard approach then you will need to insert rows into the table which models the relationship type by executing an INSERT INTO statement in code in the parent form's module followed by a Requery of the subform control to show the appended rows.  Where in the module this code would be would depend on when you wish the subform to be populated.  If, for instance, you wish do so immediately after inserting a new record in the parent form, the code would be in the parent form's AfterInsert event procedure.

You have not said what the parent form's table is modelling so let's assume the table is named Foos, with a primary key FooID.  From your description the QuantitativeAssessmentCategories table apparently models the relationship type, though you don't mention any column in it to reference the parent form's table's primary key, which makes me suspect the model might be missing a table and the QuantitativeAssessmentCategories table is in fact the other referenced table, as its name would suggest, in which case it incorrectly contains columns which are attributes of the relationship type.

I would strongly recommend that you change the name of the column Yes/No, if that is its real name, to something else as Yes/No is the term Access uses for the Boolean data type.  I'll call it IsRelevant for this example.  I'll call the table which models the relationship type FooQACats.  The model would thus comprise the following tables:

Foos
….FooID  (PK)
….Foo

QuantitativeAssessmentCategories
….QACatID  (PK)
….QACategory

and to model the relationship type:

FooQACats
….FooID  (FK)
….QACatID  (FK)
….IsRelevant
….Comments


The primary key of this table is a composite one of the two foreign keys, FooID and QACatID.  Consequently the code to populate the subform would be:

Dim strSQL As String

strSQL = "INSERT INTO FooQACats(FooID, QACatID) " & _
    "SELECT " & FooID & ", QACatID " & _
    "FROM QuantitativeAssessmentCategories"

CurrentDb.Execute strSQL, dbFailOnError

Me.[NameOfSubformControl].Requery

Note that NameOfSubformControl is the name of the control in the parent form which houses the subform.  It might or might not be the same as its source form object.  The RecordSource of the subform would be the FooQACats table, or better still a query on the table which orders the rows logically.  The subform would be linked to the parent form on FooID.
