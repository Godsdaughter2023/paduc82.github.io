If you are not equipped to debug this problem, you should use some basic troubleshooting techniques:

If exception code 0x80000003 occurs, a hard-coded breakpoint or assertion was hit, but the computer was started with the /NODEBUG switch. This problem should rarely occur. If exception code 0x80000002 occurs, the trap frame supplies additional information.

If you do not know the specific cause of the exception, consider the following items:

First of all, one should never accept any hardware driver updates from Microsoft (video, audio, network).  If they are offered to you, decline the driver updates and tell Automatic Update to never offer them to you again.  MS doesn't know the best drivers for your system - you should only get hardware driver updates from the manufacturers WWW site.

If, after a driver update your system does not boot, you need to restart your system and start poking the F8 key (poke-poke-poke-poke) before you see the Windows splash screen and get into the Advanced Boot Options menu.   If you wait to long and miss the window of opportunity for F8, start over until you get there.  Just keep poking the F8 key after you restart until you see the Advanced Boot Option menu.  If you see the Windows splash screen, you missed it - try again.

If you see XP starting to load, your system has gone w-a-y past the BIOS loading.  The BIOS (Basic Input Output System) is code that is stored in a ROM chip on your motherboard.  It tests your hardware and gets your system ready to load whatever operating system you are running.  If the BIOS detects a hardware problem, it will deliver a message to you and will not continue.

You can read more about the BIOS here.

Your system has already loaded the BIOS and if there was a hardware problem, it would report it.  That is what it does - then XP starts to load.  What Bruce should has said is "Dang, your XP isn't loading" then if you had mentioned that you downloaded some drivers from MS, Bruce should have suggested Last Known Good Configuration, which I hope you will do.

If you see the Windows loading bar, XP has gotten too far along in the boot process and you are going to have to restart the system and start poking the F8 key until you see the Advanced Boot Options menu and choose LKGC.  If you don't see it or miss it, try again.
