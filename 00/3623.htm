Find and replace with Autotext entry - Microsoft Community

sue sweet say:

Find and replace with Autotext entry

I want to have a macro to find all the text  "abg" and replace it with an autotext entry "tabg" which is actually a picture of a signature.  I tried to record the macro, but it didn't work correctly.   Thanks.
jay freedman say:
You don't have to copy/paste anything. Just change the line

      .Text = "abg"

to put in the abbreviation you're looking for, and change the line

         ActiveDocument.AttachedTemplate.AutoTextEntries("tabg").Insert _

to put in the name of the autotext entry that will replace it.

If you have lots of these things to look for, you could use the following version of the macro instead. On each run, it will ask you for the abbreviation and the name of the corresponding autotext entry.

Sub ReplaceWithSignature()
   Dim rg As Range
   Dim abbr As String
   Dim ATentry As String
   abbr = InputBox("Enter abbreviation:")
   If Len(abbr) = 0 Then Exit Sub
   ATentry = InputBox("Enter AutoText name:")
   If Len(ATentry) = 0 Then Exit Sub
  
   Set rg = ActiveDocument.Range
   With rg.Find
      .Text = abbr
      .Format = False
      While .Execute
         ActiveDocument.AttachedTemplate.AutoTextEntries(ATentry).Insert _
            Where:=rg, RichText:=True
         rg.Collapse wdCollapseEnd
      Wend
   End With
End Sub

Bootmgr loaded to HDD on Update - Microsoft Community

jamesmcclain1962 say:

Bootmgr loaded to HDD on Update

I run two SSDs in hot swap bays, one with Win7, and the other with Win 10. I also have two large capacity HDDs for storage. My problem is that on every build update, the bootmgr gets loaded on one of my HDDs instead of the Win10 SSD. When I swap back to Win 7 SSD I have to go back to the BIOS and tell it to boot from the SSD and not the HDD, then I have to run dskchk for errors. Is there any way to force the bootmgr to SSD on automatic updates. Three times now I've had to transfer all the data from my HDDs and wipe them clean, and re-install data, then wipe the SSD and do a fresh install to Win 10 with the HDDs unplugged. Thank you for any help.

a. user say:

1. Shutdown your machine and unplug the power cable just to be safe.

2. Locate your SATA Connector to the Windows 10 hard drive.

3. Make sure this SATA connector is set to SATA 0 (first SATA port on your motherboard).

4. Disconnect all the other SATA cables to the other hard drives including your CDROM.

5. You can leave the power cables to the other hard drives & CD ROM (just disconnect the SATA Cables)

6. Close the side of your computer and reconnect the power cable

7. Power up your computer, press the bios key and check the drive & SATA number (both should be 0)

8. If not shutdown your pc and move the SATA cable to SATA 0

(check your motherboard PDF for location or sometimes it is written on the motherboard)

9. Repeat the bios check if both SATA 0

10. IF OK then shutdown and power off, connect the SATA 1 to your next hard drive

11. Repeat the bios check if both SATA 0, SATA 1 are OK

12. Repeat for all your hard drives and CDROM

13. Change the boot order to have CDROM first, then window 10, then next hard drives.

14. Start your machine and login to windows

15. Right-click start button, Disk Management

16. Make sure DISK 0 is window 10, DISK 1 - is your 2nd hard drive, DISK 2 - last drive, CD ROM 0 is last

This should make windows update use the first drive during updates.

OR you can ignore all this and just disconnect all other hard drives except windows 10 when upgrading,

then reconnect the other drives after windows 10 has finished updating.

Team managers can delete members of a team from StaffHub web. To do so, navigate to the Team Members page, hover over the team member you'd like to delete and a delete icon will appear on the right. Click on the delete icon, and the member will be deleted.

Members deleted from a StaffHub team are only removed from that particular team and not removed from the tenant. To delete a team member from a tenant, go to the Office 365 admin center.
