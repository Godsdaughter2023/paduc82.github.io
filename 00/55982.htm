As Microsoft does not appear to be willing to assist with this issue, I poked around inside the Seagate GoFlex Home myself using SSH and found out how to make this work.  While the SAMBA shares work perfectly from Windows 10, printing seems to be completely baffed, so the solution (as also noted by others above) is:  don't use SAMBA printing on this device.

You will have to manually select which printer driver to use, but the printer is then functional.  You can verify that your GoFlex Home is running CUPS simply by visiting the http link; if you see a poorly formatted page come up, it's there and should work.
