    eugenedfourie say: I had the same problem.  I eventually found a fix.

    In all the locations where there are media items (meaning internal and external memory), the first time you updated the Windows Library, a folder called MSMETADATA (I found it in the File Explorer, with the option to 'Show all files') was created.  This I think causes the conflict that causes Windows Media Player to crash. 

    I simply deleted the MSMETADATA folder  in my internal and external storage, and updated my library - and Bob's your uncle!
