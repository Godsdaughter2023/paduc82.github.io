wunders say:

Apologies for being not replying sooner but I'm usually only around during my lunch hour at work...

 

JCW>  I can request any particular shared file by UNC (\\server\share\file_path)

           **if and only if** "server" is replaced by the specific IP address of the desired

          computer on the LAN!

 

Yes, that is what you need to do because SMB pretty much tosses out the protocols that automatically discover other devices on the subnet (i.e. broadcasts).  "If and only if" may be a bit too restrictive in your statement above.  More specifically, the "server" must either be an IP address, or a name that can be resolved to an IP address using standard TCP/IP resolving methods.

 

Any "name" can be resolved to an IP address if it exists in either the "hosts" file (always checked first) or if the DNS server configured on your computer returns an IP address when sent the "name".  For a small home or office setup, it is more practical to add a line to your "hosts" file should you prefer to use names rather than IP addresses.  The "hosts" file in Win XP usually exists as the file:

    C:\Windows\System32\Drivers\etc\hosts

The file does not have an extension (which makes it harder to edit with Notepad which seems to insist on a ".txt" extension).  But you can feel free to edit the file with any text editor to associate a name to an IP address.  Simply add a line like:

   192.168.0.5    JasonPC

and from then on you should be able to use \\JasonPC\share instead of  \\192.168.0.5\share.  As you may have figured out, this file is subject to misuse as one could enter a line like:

   198.168.0.100  www.google.com

into the hosts file and from then on anytime someone connected to www.google.com, they would connect to the specified IP address instead of the real site.  The only line required to be in this file (in XP) is:

   127.0.0.1   localhost

so if you see any other lines that don't begin with a "#" character, you had better know what they are.

 

With Printer Sharing, it depends on how your printer is connected to the network.  If it is connected to another computer (e.g. via USB cable) and then shared from that computer, then it works just like file sharing on that computer that you mention above.  On the other hand, if your printer is connected directly to an IP switch or router or is wirelessly connected, then the printer itself must either support SMB or you need to use a print driver that doesn't use Microsoft Networking at all (many printers come with drivers that use port 9100 for printing).  Best to consult your printer manufacturer here.

  JW
