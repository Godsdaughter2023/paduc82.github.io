Mail Merge Labels: sort - Microsoft Community

falcios say:

Mail Merge Labels: sort

I saved my mail merge labels output to a Word 2010 file. 

Before printing out the labels, I needed to remove some addresses.  When I removed cell contents and shift cells up or left it left an empty label.

After deleting cell contents how can I sort the labels in alphabetical order so there are no empty spaces.

I know that I can run the mail merge and filter out the addresses but instead of doing that I saved my output to a Word file.

Thanks in advance.

 

macropod say:

After deleting cell contents how can I sort the labels in alphabetical order so there are no empty spaces.               

Word's table sort tools can only sort within a column, it can't perform the kind of content wrapping your requirements would entail. Not only that, depending on your choice of labels, a label merge doesn't necessarily populate every cell in the output table and every page contains a separate table.

A workaround would be to use a directory/catalogue merge, using a single-cell table sized to match the label size & positioning of your label stationery, with a multi-column page layout using as many columns as your label stationery and an inter-column spacing that matches the space between your labels. The output would be down then across, rather than the label merge's across-then down format, but it would all also be a single table. As such you could delete any cell and all the others would immediately relocate to replace it. No re-sorting required.

suzanne s. barnhill say:
In such situations, it is generally easier to just rerun the merge, removing the unwanted addresses from the data source either manually (by clearing check boxes in the Select Recipients dialog) or by filtering.
