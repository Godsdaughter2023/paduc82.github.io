karl timmermans [outlook mvp 2012-2018] say:
If your Gmail account was configured using IMAP and the your contacts were stored in a folder which included the words "This Computer Only" - then those contacts are lost when you deleted the Gmail account unless you explicitly exported (backed up) those contacts to a PST file.

How things got configured in your Outlook profile depends entirely on the type (POP3, Imap etc) of email accounts that got configured and the sequence in which those accounts got configured.

PST files are only used in Outlook '2013/'2016 in two cases
1) when a POP3 account is configured
2) if you manually added one to the Outlook profile

In terms of contact/calendar info, those would be stored in the default data file for the profile (which be default is set by the first email account configured). The default data file is the location for the default contact/calendar folders for the profile.

jeanette cunningham say: One thing I noticed - the syntax for passing a variable needs to concatenate the value into the query string.

Here is one line from your query where you need to do that.
In the where clause you have:  ((Year([Temperature_ID_Rare_Export9-16-2011].[Datetime]))=GetTempSetYr())

You need: ((Year([Temperature_ID_Rare_Export9-16-2011].[Datetime]))=" & GetTempSetYr())

Note: the above assumes that GetTempSetYr is a number data type.

Another thing, you created 2 variables,
Dim TempSet As Long
Dim TempSetYr As Integer

I  don't see these variables used in the sql string.
Perhaps you intended to use TempSetYr  instead of GetTempSetYr()  ?

Something like this:
((Year([Temperature_ID_Rare_Export9-16-2011].[Datetime]))=" & TempSetYr)

Also note that usually a field called Datetime is usually not a number data type - usually a date. Can you confirm in your table it is a number?

TempSetYr looks to be a number data type, not a date.
