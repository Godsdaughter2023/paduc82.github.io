Add the following code to a VBA module:

Dim I As Integer

Sub Capture()
    If I = 0 Then I = 1
    Sheets("Sheet2").Cells(1, I) = Sheets("Sheet1").Range("A1")
    I = I + 1
    Application.OnTime Now + TimeValue("00:05:00"), "Capture"

t. valko say:

I am trying to nest the indirect function with the sum function calculating across tabs. 

The target formula is : =SUM(START:END!:B4).  I am trying to use the indirect function to change the column and cell reference, so I expected the following to work:

=SUM(INDIRECT("START:END!:"&INDIRECT(A1)&INDIRECT(A2))), where cell A1 would contain the target column and cell A2 would contain the target cell. 

I always get a "#REF" error whenever I try this combination.  I have tried many many permutations of the formula above, but cannot get this to work.  Any ideas?


INDIRECT won't accept a 3D sheet reference like Start:End.

However, if your sheet names follow some sort of naming pattern then you can get it to work by wrapping it all inside a SUMPRODUCT(SUMIF construct.

For example, if your sheet names were the default sheet names like Sheet1, Sheet2, Sheet3, etc., then this would work:

  • A1 = B (column)
  • A2 = 4 (row)

=SUMPRODUCT(SUMIF(INDIRECT("Sheet"&{1,2,3}&"!"&A1&A2),"<1E100"))

That will sum cell B4 on sheets Sheet1, Sheet2  and Sheet3.

Or, you can list the sheet names in a range of cells then refer to that range:

  • A3 = Sheet1
  • A4 = Sheet2
  • A5 = Sheet3

=SUMPRODUCT(SUMIF(INDIRECT(A3:A5&"!"&A1&A2),"<1E100"))

Again, that will sum cell B4 on sheets Sheet1, Sheet2  and Sheet3.

You can even list the sheet names dynamically with a formula:

macropod say:

With so many workbooks, worksheets, letters and forms, you have a fairly complex undertaking. The following macro should get you started. It assumes you have:

• two mailmerge main documents per year - a letter and a form;

• one workbook for each five years; and

• one worksheet per year (five per workbook),

all in the same folder as the document you add the macro to. The assumed naming conventions appearing in ArrMMSrc, ArrMMDoc, & ArrXlWs, are self-explanatory - you can substitute your own names as appropriate. The code also assumes the existence of an 'ID' field for each record. Each record is saved to a separate file using that ID for the filename. If the same ID appears in multiple worksheets, only the last will survive, but that's easily prevented. Some commented-out code allows the output to be saved as PDFs as well or instead.

Sub RunMerge()
Application.ScreenUpdating = False
Application.DisplayAlerts = wdAlertsNone
Dim ArrMMSrc(), StrMMSrc As String, ArrMMDoc(), StrMMDoc As String, ArrXlWs(), StrXlWs As String
Dim wdDoc As Document, StrPath As String, StrName As String: StrPath = ThisDocument.Path & "\"
Dim i As Long, j As Long, r As Long: Const StrNoChr As String = """*./\:?|"

'The source workbook names - 1 per 1/2 decade
ArrMMSrc = Array( _
  "1950-54Source.xlsm"**** "2015-19Source.xlsm")
 
'The source worksheet names - 1 per year
ArrXlWs = Array( _
  "1950"**** "1997"**** "2019")
 
'The mailmerge main document names - 2 per year
ArrMMDoc = Array( _
  "1950Letter.docx"**** "1959Form.docx"**** "1969Form.docx"**** "1979Form.docx"**** "19**** "1999Form.docx"**** "2009Form.docx"**** "2015Form.docx")

For i = 0 To UBound(ArrMMDoc)
  StrMMDoc = StrPath & ArrMMDoc(i)
  StrMMSrc = StrPath & ArrMMSrc(Int(i / 10))
  StrXlWs = ArrXlWs(Int(i / 2))
  Set wdDoc = Documents.Open(FileName:=StrMMDoc, AddToRecentFiles:=False, ReadOnly:=True, Visible:=False)
  With wdDoc
    With .MailMerge
      .MainDocumentType = wdFormLetters
      .OpenDataSource Name:=StrMMSrc, ReadOnly:=True, AddToRecentFiles:=False, _
        LinkToSource:=False, Connection:="Provider=Microsoft.ACE.OLEDB.12.0;User ID=Admin;" & _
        "Data Source=StrMMSrc;Mode=Read;Extended Properties=""HDR=YES;IMEX=1"";", _
        SQLStatement:="SELECT * FROM `" & StrXlWs & "$`"
      For r = 1 To .DataSource.RecordCount
        .Destination = wdSendToNewDocument
        .SuppressBlankLines = True
        With .DataSource
          .FirstRecord = r
          .LastRecord = r
          .ActiveRecord = r
          If Trim(.DataFields("ID")) = "" Then Exit For
          StrName = .DataFields("ID")
        End With
        .Execute Pause:=False
        For j = 1 To Len(StrNoChr)
          StrName = Replace(StrName, Mid(StrNoChr, j, 1), "_")
        Next j
        StrName = Trim(StrName)
        With ActiveDocument
          .SaveAs2 FileName:=StrPath & StrName & ".docx", FileFormat:=wdFormatXMLDocument, AddToRecentFiles:=False
          ' and/or:
          '.SaveAs2 Filename:=StrPath & StrName & ".pdf", FileFormat:=wdFormatPDF, AddToRecentFiles:=False
          .Close SaveChanges:=False
        End With
      Next r
    End With
    .Close SaveChanges:=False
  End With
'Next i
Set wdDoc = Nothing
Application.DisplayAlerts = wdAlertsAll
Application.ScreenUpdating = False
End Sub

stefan blom say:

Headers and footers are section properties, and they usually span several pages, so they don't really have page numbers.

If you want to add something that is in the body of the document into the header, use a STYLEREF field, as others have suggested.

If you do not want headings in the main body, only in the TOC, you will have to make use of TC fields. See http://wordfaqs.mvps.org/TOCTips.htm#TCFields.
