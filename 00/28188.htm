augustenz say: I have a Nokia 6233 and a Mobile Action BT730 bluetooth dongle which always worked fine with it's own software. But it never worked with Nokia PC Suite.

After so much time i finally got the idea to use a microsoft driver for my bluetooth dongle. And it worked! I don't know if this method works for every Bluetooth dongle but try it out, maybe it will work for you too.

Before you start make sure you have uninstalled your Bluetooth dongle along with it's drivers and software.

1. Plugin your bluetooth dongle
2. A dialog opens, "Windows needs to install driver software for your Device_Name_Here". In my case the device name was "Unknown Device". Press Cancel.
3. Go to the Device Manager (Start->Control Panel->Device Manager).
4. If you get another dialog, "Windows needs your permission to continue", just press Continue.
5. Find your unrecognized device in the device list (it will have a yellow exclamation mark on it's icon), in my case it was listed under "Other devices" with the name "Unknown device". Make a right click on it and choose "Update Driver Software".
6. In the dialog "How do you want to search for driver software" choose "Browse my computer for driver software".
7. In the dialog "Browse for driver software on your computer" choose "Let me pick from a list of device drivers on my computer".
8. Select "Bluetooth Radios" from the common hardware types list, and press Next.
9. Now, in the column "Manufacturer" select "GenericAdapter", in the column "Model" select "Generic Bluetooth Adapter" and press next.
10. You should get a dialog "Update Driver Warning", where windows states that it can't verify that it is compatible with your hardware. Press Yes.
11. You hopefully get the "Windows has successfully updated your driver software" in this step. Press Close.

That's it! Happy bluetoothing with your Mobile Action bluetooth dongle and Nokia PC Suite!

janpechacek say:

I am using Windows 7 Professional and IE 11 (freshly updated). I started running into this bug, too, and I stumbled upon a solution that works for me.

tl,dr: Try opening Internet Explorer, clicking on the Options gear (or use Alt-X), go to Internet Options and make the change there. It should stick. At least it does for me.

Longer story: When I used "Internet Options" from Control Panel (or typed inetcpl.cpl into the Start Menu search bar and hit Enter), I could not change the LAN proxy settings, such as whether to use proxy, the proxy address and port. I mean, I could change it, but the change was not saved. I started to dig a little deeper and found out, that even changing the corresponding registry value, such as

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\ProxyServer

does not help, as this value gets restored to the previous value when opening the Internet Options panel.

First of all, I naturally checked for viruses and other malware with Avira Antivir and Malwarebytes AntiMalware, because malware often likes to fiddle with settings like this. The programs found nothing, so I moved on.

An analysis with SysInternals Process Monitor revealed that upon the opening of the Internet Options panel, it is explorer.EXE who rewrites the registry values controlling the proxy, such as

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\ProxyServer .

So it had to be either a system bug or a virus masquerading as explorer.EXE. The latter is likely not the case, as no viruses were found on the system. A careful review of the SysInternals Process Monitor log revealed that immediately before the rewriting of the registry value occured, explorer.EXE loaded the value stored in

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\Connections\DefaultConnectionSettings .

Modifying of this file and opening the Internet Options panel again revealed that the content of this registry entry is indeed used as the template for the modification of the important values controlling the proxy, as the change was propagated to the values that are displayed in the Internet Options panel and to the registry key

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\ProxyServer

as well. Later, I found that when opening Internet Explorer, clicking on the Options gear (Alt-X) and Internet Options, I am able to change the proxy settings with no problem and the change propagates to both

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\ProxyServer

and

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\Connections\DefaultConnectionSettings .

Conclusion: I am not exactly sure when this behaviour started to occur, as I do not change the system proxy setting very often. But somebody at Microsoft should clearly look into this, because the Internet Options Control Panel (inetcpl.cpl) is unable to modify the system's proxy setting, thus requiring the user to open Internet Explorer and do the change there. That is counter-intuitive, as the user wants to change the system setting for proxy, not the setting of an Internet browser.
