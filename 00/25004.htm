jay freedman say:
To reinforce what Doug has said, it should very rarely be necessary to have macros in a document. The macros should be only in the document's base template. The most common time this is not true is when the template won't be accessible from the computer that's opening the document (e.g., a laptop that may be disconnected from the company network) and the local Normal.dotm template is attached instead. Even in that case, it would be better to have a copy of the template downloaded to the computer's local drive so that it is always available.

The main reason for this is Word's (IMHO clumsy) handling of macro security. Unless the document is stored in a "trusted location" and the macro setting in the Trust Center is properly set, upon opening the document the macros will be disabled. Depending on the settings, there may or may not be a notification that the macros are disabled.

One other point: Setting the Save As type to .docm DOES NOT copy the macros from the template to the document. You would need to separately intercept the Save As function and force it to copy the macros over. For that to work, you have to go into the Trust Center dialog ON EACH COMPUTER and turn on the option to "Trust access to the VBA project object model", which is a serious security hole even worse than saving macros in a document.

Updating cross reference CTRL C & F9 and entire text disappears - Microsoft Community

ril_klp say:

Updating cross reference CTRL C & F9 and entire text disappears

when look at file size text all txt seems to be there? when look at file size text all txt seems to be there?
jay freedman say:

Everything between the curly brackets is inside a field (the gray shading is the result of the "Field shading" option in File > Options > Advanced > Show document content). The reason the page appears empty in the second screen shot is probably that the field code (which appears when you press Alt+F9 or Shift+F9) is very short and now appears at the end of page 2.

The solution to this problem is to select everything inside the field result (NOT the field code), cut it to the clipboard, and paste it outside the curly brackets. Then you can delete the empty brackets. (You cannot just delete the brackets and leave the text.)

If this occurs in many documents that are all based on the same template, then the fault is in the template, which you may be able to fix in the same way -- depending on what the field code is. If it's an INCLUDETEXT field referring to a different document, that may not work.

suzanne s. barnhill say:

I suspect what you actually pressed was Ctrl+F9 (possibly preceded by Ctrl+A). If you had the entire document selected, then you converted the whole thing into a field. Since it's not a meaningful field, it disappears when you toggle field display with Shift+F9 or Alt+F9.

If you haven't meanwhile closed the document, you could use Undo to undo this action. Otherwise, you can try unlinking the field with Ctrl+Shift+F9, which might restore the contents. Alternatively, add a couple of spaces at the beginning and end of the field text, then carefully copy the content excluding the spaces (so as to avoid including the field braces) and then paste into a new document (or another part of the current one).
