lemp say:

Congratulations!  Glad it worked for you.

One final thing:  open Devices and Printers, right-click on the icon for your printer, select "Printer properties," and then select the "Ports" tab.  If the line that is checked and highlighted is 192.168.0.13 (or similar), there is the possibility that one of these days you will find that when you try to print the print job disappears from the print queue but nothing prints.  If that happens, it is because while the printer was turned off (generally for more than a day), your router reassigned the printer's IP address to another device on your network.

There are various solutions to this problem.  One way is to add a new "Standard TCP/IP Port" using the "Add Port" button on the same "Ports" tab discussed above.  First, print a Network Status Sheet using the printer's front panel controls to determine the new IP address that the router has assigned to it.  Then click "Add Port," select "Standard TCP/IP Port" and enter the new IP address in the "Printer Name or IP Address" box and complete the wizard.  Be sure to check the new port and uncheck the old port on the "Ports" tab of "Printer properties."  Then click OK.

To print a Network Status Sheet, see page 43 of your User Guide:

  1. Press the up or down arrow button, select Setup, and press the OK button
  2. Press the down arrow button, select Network Settings, and press the OK button
  3. Press the down arrow button, select Network Status, and press the OK button
  4. Press the down arrow button, select Print Status Sheet, and press the OK button
  5. Press one of the Start buttons to print the network status sheet

The proper way to avoid the problem of changing IP addresses is to configure the printer to use a "static" IP address outside the range used by the router's DHCP server.  Because the User Guide for your printer doesn't explain the procedure for doing this (although I'm pretty confident that it's possible), I'll leave this issue for another day if and when you have the problem.

How to manually add Wi-Fi on WIndows 10? - Microsoft Community

michaeldubov say:

How to manually add Wi-Fi on WIndows 10?

*Original Title: how to add wifi ap manually in w10

how to add wifi ap manually in w10, the ssid is not broadcasted. I can make it broadcasted, then I can connect.

a. user say:

Hi Michael,

Thank you for posting your query on Microsoft Community. Let me assist you.

I understand you are trying to create a Wi-Fi access point on your Windows 10 computer. You may create the access point following some easy steps, if your system is connected to Internet.

Steps:-

1. First, create a virtual Wi-Fi Network on the computer.

a. Right click on the start menu and select Command prompt (Administrator)

b. Type the following command to create a virtual network: netsh wlan set hostednetwork

c. Further, you may initiate the hosted network by running the following command on the command prompt: netsh wlan start hostednetwork mode=allow ssid=name key=password

d. Ensure, the name of the network is set using the SSID parameter and the password is set using the KEY parameter.

e. Once you set SSID and KEY, press Enter.

f.  As the virtual adapter is now created, it can be used as an access point for devices. To make use of the adapter, turn it on by running the following command on the Command prompt: netsh wlan start hostednetwork

g. Hit Enter. The virtual wireless route will be initiated to work as a Wireless Access point.

2. Finally you have to allow the internet sharing with the access point.

a. Type and select Network and Sharing Center on the search bar.

b. Click on Change Adapter settings on the left pane.

c.  Right click on the Ethernet adapter and select Properties.

d. Under Sharing tab, check the box near “Allow other network users to connect through this computer’s Internet connection”.

e. Under, Home Networking Connection, select the virtual adapter created.

f. Click on OK and Close.

Hope this information is helpful. Please do let us know if you need further assistance, we’ll be glad to assist you.
