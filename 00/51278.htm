If the problem occurs in clean mode then just restore the system to normal status and reboot - this solution is not going to work.

Let's see if any of those procedures fix or find the problem.  If not, post back with ressults of what you  found and answers to my questions.  We'll see what else to try if those options don't work.

a. user say:

Not sure what DLC files are or what a "bunch of errors" means or why you think you need to be replacing files.

Just FYI - a 64-bit installation will always have the SysWOW64 folder and there is no SysWOW64 on a 32-bit installation.

The SysWOW64 folder (WOW stands for Windows on Windows) is where Windows keeps 32-bit copies of files (like DLLs) and applications that might be needed from time to time on your 64-bit system.

Most of the time things are going to operate from the system32 folder but if Windows needs a 32-bit version of a file it will get it from the SysWOW64 folder. 

You don't have to do anything or worry about it - Windows should know what it needs and where to get it.

There could be files in system32 and SysWOW64 that are exactly the same so if you think you need to be replacing some files for some reason I would be real careful about that and have a plan for what you are going to do if your system malfunctions severely when you get done - how are you going to undo the changes if something goes wrong?
