If you are using text boxes (not placeholders) the user does not have any obvious control over the format of the next level (it is specified in the template XML) but usually it would be the default and hardly ever the style applied by the user. You might want to use TAB instead which should retain the style.

4. Often this is because the user has put large images as backgrounds to the layouts??

5. I don't think that is true. In most cases you can just click the shape or text box. If you double click format painter it will stick on.

If you have very large number of slides you might want to use a macro to e.g. set all text to Uppercase. 

This would change all text not in placeholders to Upper Case pretty much instantly:

Sub UPPER()
Dim osld As Slide
Dim oshp As Shape
For Each osld In ActivePresentation.Slides
For Each oshp In osld.Shapes
If Not oshp.Type = msoPlaceholder Then
If oshp.HasTextFrame Then
If oshp.TextFrame.HasText Then
oshp.TextFrame.TextRange = UCase(oshp.TextFrame.TextRange)
End If
End If
End If
Next oshp
Next osld
End Sub

a. user say:

Hi KEmailA,

Welcome to Microsoft Community. First off, we apologize for the delayed response.

I understand that you want to know to manage email accounts in your custom domain as you are now unable to login to your domain. Let me assist you with this.

In addition to ZigZag3143's reply, we would like to inform you that the Domain Administrator has no longer capability to reset the password of his/her members accounts, since that feature has been removed as part of the decommission process of Windows Live Admin Center started last July 31st of 2014.

The Domain Administrator will also no longer be able to add or remove accounts in the domain. Instead, we recommend that Administrators who wish to continue managing email addresses in their domain should consider Office 365 for business - Microsoft's premium online email service. By migrating to Office 365, Domain Administrator will be able to continue managing the custom domains, as well as benefit from modern enterprise-class mail, collaboration and communication tools.

Please be advised that as part of the decommission process, eventually, listed domains in the admin panel will be removed and all users (domain administrator's account) who will log in to https://domains.live.com will be redirected back to its Windows Live Admin Center home page.  However, email accounts in the domain can still log in and access it at www.outlook.com with their emails intact. As long as there will be no changes in the MX Record set by domain administrator, these users will continue to  receive emails.

Should you have any other concerns, please let us know.
