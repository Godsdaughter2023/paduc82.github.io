lemp say:

Assuming that you have Windows 7, you're getting that error because the version of Powershell that comes with Windows 7 is 2.0 and you need Powershell 5.x in order to use the Microsoft script to check for the Meltdown/Spectre fixes.

You can update by installing Windows Management Framework 5.0.  Although the "System Requirements" info for that download only says that "WMF 5.0 requires Microsoft .NET Framework 4.5 or above," the Release Notes say that the prerequisite for WMF 5.0 is that "WMF 4.0 and .NET Framework 4.5 or above are installed."

Just in case, I installed WMF 4.0 first (https://www.microsoft.com/en-us/download/details.aspx?id=40855) and then WMF 5.0 (https://www.microsoft.com/en-us/download/details.aspx?id=50395)

I already had .NET 4.7 installed.

Windows Management Framework 5.0 shows in the list of Installed Windows Updates as KB3134760; WMF 4.0 shows as KB2819745.  You can check the .NET version in Programs and Features -- look for Microsoft .NET Framework 4.x

After doing all of that, I could run the Powershell commands shown in the article you cited.

Remember, you almost certainly will have to run the following Powershell command first:  

    Set-ExecutionPolicy Bypass

And after you're all done, run

    

     Set-ExecutionPolicy Restricted

   

Details here:  How to Check and Update Windows Systems for the Meltdown and Spectre CPU Flaws

P.S.:

1.  Installation of the WMF updates takes a long time.  Start the installer and go have lunch.

2.  WMF 4.0 (KB3134760) is removed by the installation of WMF 5.0 (KB3134760).

andre da costa say:

When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate. You will prompted to enter a product key a couple times through the installation, click I don't have a key and Do this later.

There is no need to know or get a new key, but if you have issues with Digital Licensing or the Activation Troubleshooter, you can utilize an existing Windows 7 or Windows 8/8.1 product key you already ownto reactivate if necessary. or use the reset function in Windows 10.
