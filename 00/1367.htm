hansv mvp say:

Before running the following code, you must allow programmatic access to the Visual Basic project of workbooks:

  • Select File > Options.
  • Click Trust Center > Trust Center Settings...
  • Click Macro Settings.
  • Make sure that the check box "Trust access to the VBA project object model", then click OK.

Here is the macro. It will prompt you to point to a folder, then it will process ALL workbooks in that folder.

 

 

Sub RemoveOptionExplicit()
    Const strFind = "Option Explicit"
    Dim strPath As String
    Dim strFile As String
    Dim wbk As Workbook
    Dim vbc As Object
    Dim mdl As Object
    Dim startLine As Long
    Dim startCol As Long
    Dim endLine As Long
    Dim endCol As Long
    Dim blnFound As Boolean
    With Application.FileDialog(4) ' msoFileDialogFolderPicker
        .Title = "Please select the folder to process"
        If .Show Then
            strPath = .SelectedItems(1)
        Else
            MsgBox "No folder specified!", vbExclamation
            Exit Sub
        End If
    End With
    If Right(strPath, 1) <> "\" Then
        strPath = strPath & "\"
    End If
    Application.ScreenUpdating = False
    Application.EnableEvents = False
    On Error GoTo ErrHandler
    strFile = Dir(strPath & "*.xls")
    Do While strFile <> ""
        Set wbk = Workbooks.Open(Filename:=strPath & strFile)
        blnFound = False
        For Each vbc In wbk.VBProject.VBComponents
            Set mdl = vbc.CodeModule
            startLine = 1
            startCol = 1
            endLine = -1
            endCol = -1
            If mdl.Find(strFind, startLine, startCol, endLine, endCol) Then
                mdl.DeleteLines startLine, endLine - startLine + 1
                blnFound = True
            End If
        Next vbc
        wbk.Close SaveChanges:=blnFound
        strFile = Dir
    Loop
ExitHandler:
    Application.EnableEvents = True
    Application.ScreenUpdating = True
    Exit Sub
ErrHandler:
    MsgBox Err.Description, vbExclamation
    Resume ExitHandler
End Sub

Just a few additional points:

Now that we have our Start menu back, you can open the Windows Defender interface in the Start menu (All apps > Windows System > Windows Defender). And if you just right-click on the Windows Defender menu entry and “Pin to Start” or “Pin to taskbar” you can make Windows Defender even more accessible. By default, the Windows Defender icon will be missing at startup; but you’ll find a couple of ways to get your icon at startup here:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/is-windows-defender-icon-supposed-to-be-in-system/6cfa9d1d-3db8-49a9-add7-280a284ccfdc

A daily Quick Scan is done by Automatic Maintenance, with no need for changing any settings at all – but you actually do have an option to change the time-of-day setting for Automatic Maintenance. Right-click on the Start button and choose Control Panel > Security and Maintenance > Maintenance > Change maintenance settings:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-10-how-do-i-set-windows-defenders-scan/f399bb87-37c9-4a74-bb35-1d7e3d3c2436

This thread explains why the option for a scheduled Full Scan is missing with Automatic Maintenance:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender/588b2978-dc88-432b-bc8e-066acf6b9258

Winhelp.us deserves a lot of credit, because their tutorial took the right approach to scheduling a scan when hacking the system task was the popular thing to do; but there’s no need to duplicate the Automatic Maintenance Quick Scan – so you’d normally be scheduling a Full Scan. And when you’re scheduling a Full Scan it’s always a good idea to also check the “Wake the computer to run this task” box in the Conditions tab. That not only allows the scan to start when the PC’s sleeping; it also keeps the PC awake for as long as the scan runs. That’s very important with an unattended Full Scan.

If Windows Defender’s status is “Protected”, then it’s a pretty safe bet that all is well. But you can always check Windows Defender’s functionality with the eicar.com test file just to make sure:

http://www.eicar.org/85-0-Download.html

=SUMIFS('SPORTS BUDGET 2015 PRIM'!E$1:E$110, 'SPORTS BUDGET 2015 PRIM'!$A$2:$A$111, $A$4, 'SPORTS BUDGET 2015 PRIM'!$B$2:$B$111, "COACHING*")

=SUMIFS('SPORTS BUDGET 2015 PRIM'!E$1:E$110, 'SPORTS BUDGET 2015 PRIM'!$A$2:$A$111, $A$4, 'SPORTS BUDGET 2015 PRIM'!$B$2:$B$111, "<>COACHING*")
