suzanne s. barnhill say:

I guess at this point I would wonder why you are using "Wrap line as Word 6.0 does." What problem do you expect this to solve?

To begin with, lines should not have trailing spaces: there should be just one space between words and sentences, and this will always fall at the end of a line, outside the margin. Additional spaces will just pile up outside the margin. If you want an indent at the beginning of the following line, you should either start a new paragraph, use a hanging indent, or, for a single instance, end the previous line with a line break and begin the indented line with a tab character. If the text is justified, you can prevent the previous line from being justified by ending it with a tab character.

Beyond that, I think I'd need to see a screen shot to see what kinds of problems you are encountering that you feel you need these Layout Options to deal with.

suzanne s. barnhill say:

This is a bizarre approach to dealing with justified text (especially since it would appear to result in lines that are not justified). When there are large gaps in a line, they should be addressed with hyphenation, either automatic or manual (I prefer manual, using Ctrl+Hyphen to insert optional hyphens). Any other approach is wrongheaded, and I'm wondering what suggestions you received from the community that led you to use these methods.

To return to your original problem, however, I cannot imagine any situation in which a punctuation mark would be wrapped to the next line unless it was separated from the text by a space. Alternatively, if the font you are using is identified internally as "decorative," it will not behave like a text font but instead like a symbol or "dingbat" font and will not obey normal text rules.

The following is adapted from code in my DecomposerDemo file, cut down to eliminate the unnecessary lines for your purposes, and with the lines shown in boldface added or amended:

Private Sub cmdImport_Click()

    On Error GoTo Err_Handler
   
    Dim OpenDlg As New BrowseForFileClass
    Dim strPath As String
    Dim strTable As String
  
    OpenDlg.DialogTitle = "Select File"
    strPath = OpenDlg.GetFileSpec
    Set OpenDlg = Nothing
   
    ' exit procedure if user cancelled
    If strPath = "" Then Exit Sub
   
    strTable = Mid(Left(strPath, InStr(strPath, ".") - 1), InStrRev(strPath, "\") + 1)
   
    ' import Excel data into table of same name as source Excel file
    DoCmd.TransferSpreadsheet _
        TransferType:=acImport, _
        TableName:=strTable, _
        FileName:=strPath, _
        HasFieldNames:=True
   
Exit_here:
    Exit Sub
   
Err_Handler:
    MsgBox Err.Description, vbExclamation, "Error"
    Resume Exit_here

End Sub

Note that this is quite old code and uses Bill Wilson's BrowseFiorFileClass module to open the file dialogue.  These days we'd use the FileDialog property of the Application object.

Function OpenFileDialog() As String
