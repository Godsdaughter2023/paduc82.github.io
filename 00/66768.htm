john rubdy say:

Hi,

 

The difference between Let Windows Choose and Custom:

 

You can let Windows choose what to back up, in which case the OS will make a backup that includes files saved in the “Pictures”, “Music” and other such folders (which are called Libraries in Windows 7), as well as other default Windows folders. In the case that your system crashes, you can restore your computer from this backup file.

 

The other option is to choose your own files in which case you’ll be asked to browse a directory listing and check off which items/folders should be included in the backup. You can also choose to include a system image, which is a backup of the drives, system files and settings required for Windows to run. If you choose to make a system image, you can restore your computer in addition to retrieving your files in the event of data loss or a crash.

 

When you restore your computer from a system image, it is a complete restoration; you can't choose individual items to restore, and all of your current programs, system settings, and files are replaced. Although this type of backup includes your personal files, we recommend that you back up your files regularly using Windows Backup so that you can restore individual files and folders as needed.

szymonp say:

Generic USB Audio drivers bug and 'Listen to this device'

Hi,
I've encountered quite a nasty bug in 'Generic USB Audio' drivers that is used for example by Prodigy Cube usb sound card.

Steps to reproduce:
1. Disable all sound cards in your PC (remove any PCIe sound cards + disable realtek card in BIOS)
2. Plug in any sound card that use 'Generic USB Audio' drivers so that this will be the only one sound card available in the system
3. Go to Sound properties and go to Recording tab.
4. On your microphone tick "Listen to this device" option and choose your input device (the only sound card)

Expected result:
- you should be able to hear yourself as you speak to your microphone

Actual result:
- There is no sound in output device

Workaround:
- Enable realtek card in BIOS, you don't even have to install realtek drivers ('High Definition Audio Device' generic driver is enough).
- It's also enough to have any other soundcard available in the system
- Now "Listen to this device" works fine

It looks like "Listen to this device" doesn't work when only 1 USB sound card is available in the system.

a. user say:

That is because Windows is showing a drive that is suppose to be hidden from users.

Follow these steps now to remove it:

1. Select the Start button, type cmd and then right-click Command Prompt in the search results and select Run as administrator.

2. In the Administrator: Command Prompt window, type diskpart and press Enter.

3. Type list volume and press Enter.

4. Make note of the volume number for the drive letter you want to remove. For example, if the drive letter you want to remove is Drive “D”, note the volume number for Drive D.

5. Type the following command, and press Enter.

select volume <volume number>

When typing this command, in place of <volume number>, substitute the volume number you made note of in Step 4. In the example above, if the volume number for drive G is 5, you would type: select volume 5

6. Type the following command and press Enter.

remove letter=<drive letter>

When typing this command, in place of <drive letter>, substitute the drive letter you want to remove. In this example, you would type: remove letter=G

7. Close the Command Prompt window.

I hope this helps. Please let me know if that resolves the issue or you have any other questions.

a. user say:

Hi Denis,

It is possible that the recent update done on your operating system affected the settings of Cortana/search process that is why you're not getting any results when trying to use it. To resolve this issue, you will need to restart Cortana/search process. Kindly follow the instructions below:

  1. Press Ctrl + Shift + Esc to open Task Manager.
  2. Under Background processes, right-click Cortana.
  3. Select End Task.
  4. Go to the Details tab and locate SearchIndexer.exe and SearchUI.exe.
  5. Right-click the processes and select End task as well.

If that will not work, you may proceed and follow the steps provided by Andre Da Costa here.
