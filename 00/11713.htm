The Primary Key in a table is what Access uses to identify a record and must be unique. Unique indices on other fields force each record to have a unique value for those fields, too. If you enter a new record it must have a value for the primary key. If you have other unique indices on a table those must also be filled in.

If you are just now getting that message when things worked before I bet the primary key had been an AutoNumber and you changed it to a Long. Or perhaps you changed the primary key to some other field.

Open the table in design view. You should see a little key to the left of the table's primary key.

Null is one of those Alice-in-wonderland things. Null is usualy decribed as unknown or not yet determined. Nothing can equal Null because Null is unknown therfore nothing can be mathmatically equal to Null.

cbarnhorst say:

This is a feature of Windows.  A storage pool is a "bunch of disks" from which a Storage Space can be created from two or more of the disks and then formatted either NTFS or ReFS.  It becomes a single resilient data volume with characteristics like RAID.  How many disks are put in the pool determines the types of Storage Spaces available.  You find Storage Spaces in Settings/System/Storage.

cbarnhorst say: The system disk cannot be included in a Storage Pool.  You can read about all of this by searching on "Storage Spaces" in Search/Cortana and selecting any of the web entries (any magnifying glass).
