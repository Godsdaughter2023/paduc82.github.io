For testing, I have this in my Word document (just text fields as results):

{IF {MERGEFIELD Type} = "Spoon" "Free" "$1.20"}

This whole line just appears unchanged in the output document. It seems that it is not being recognised as a field code. Do I have to do something to it to make it work?


a. user say:

First, the field code braces {} are special characters that cannot just be typed on the keyboard. That applies to both sets of them. If you inserted the IF field via the Mail Merge Manager the field may be Ok, but it is often easier
 A. To use cmd-F9 ( or on some keyboards, fn-cmd-F9) to insert these yourself. Everything else can be plain text, except that in this case you should format each result as italic (e.g. Free)!as italic. }
B. unfortunately, because Mac OS reserves some keys for other functions, you may have to reassign cmd-F9 in System Preferences to make it work. An alternative is to use Insert->Field to insert empty field codes. 
So you end up with
{ IF "{ MERGEFIELD Type }" = "Spoon" "Free" "$1.20"  }

And you can italicise the $1.20 as well if necessary.

Finally, the comparison with Spoon is exact. If the MERGEFIELD might contain spoon or SPOON you will need to compare

{ MERGEFIELD Type \*Upper } 
With
SPOON

Next, to get the whole line to be italic, you have to use the IF field to output the whole line. But that means that you have to know in advance what text will be on the line. I.e., you can't tell word to "italicise all the text that end up on the line that contains these fields after you have wrapped the text" .

To do the whole line, let's imagine for now that the value of a single field (type) determines whether or not the line is italicised, but that other fields may be in the line. Then you might have:
{ IF "{ MERGEFIELD Type }" = "Spoon" "The text of the line incuding { MERGEFIELD x } for Free" "The text of the line including {  MERGEFIELD x } for $1.20"  }

You can put a lot of text+fields in the result 'areas' of the IF field but you do have to be careful about double-quote characters. If you need those, you can use a { QUOTE 34 } field, or, e.g.
{ SET q { QUOTE 34 } } to set up a bookmark called "q", then use { q }.

You may find in some circumstances that you need to be specify the formatting of a field result another way. For many field types, you can specify the character formatting by
 a. formatting the first character of the field code the way you want
 b. adding the \*Charformat option

e.g.

{ MERGEFIELD x \*Charformat }
 
to format the mergefield result as italic. For historical reasons, I tend to format everything after the { up to and including the first letter of the code ("M" in this case) with the format I want.
