A workaround wouldn't be a workaround if it didn't work ...

There is no guaranteed way of transferring the default calendar and contacts list from one installation and another. Other calendars and contacts lists don't have to be transferred, because they're available online and accessible simply by signing in in Windows Live Mail. However, here's a procedure to try:

 

The default calendar is in the hidden system folder called #storefolder#\Calendars\DBStore, where #storefolder# is the store folder (!). Because its location is user-configurable, I can't be more specific, but the default store folder is
C:\Users\{user}\AppData\Local\Microsoft\Windows Live Mail,
so you may find it there.

With all programs closed, copy the DBStore folder from the old installation and replace the same one in the new installation. Check that the current user has Full control permissions for the folder and all files in it (especially WLCalendarStore.edb) after relocating it.

The default contacts list is at
C:\Users\{user}\AppData\Local\Microsoft\Windows Live\Contacts\Default\{version}\DBStore
Again, copy the DBStore folder from the old installation to the new one to replace the one that's there. Again, check permissions on the folder and files when they're in place.

When you're done, run wlarp.exe and choose the Repair... option, then reboot, say a little prayer and launch Windows Live Mail.

All of this can be avoided by always signing in in Windows Live Mail.

All of the attached DMP files are of the DRIVER_IRQL_NOT_LESS_OR_EQUAL (d1) bug check.

This indicates that a kernel-mode driver attempted to access pageable memory at a process IRQL that was too high.

A driver tried to access an address that is pageable (or that is completely invalid) while the IRQL was too high. This bug check is usually caused by drivers that have used improper addresses.

tryx3 say:

It sounds like the gain on your built-in microphine is set very high & it's amplifying the noises made when you touch or move anything.  Check your PC manual or the PC maker's support site for guidance but, in the meantime, you can temporarily disable the microphone at

Control panel, Sound, Recording tab

1 right-click in a blank area & tick the options to show disabled devices [or you'll not find it again later]

2 right-click the internal mic & select Disable

- - - -

You might also find useful settings in the Control panel entry for your sound card.  They have different names but mine, just for example, is at Control panel, IDT audio control panel. 

I also have settings for audio in the All programs list under Creative as they provided the software that drives the IDT audio card.  This may not apply to other setups.
