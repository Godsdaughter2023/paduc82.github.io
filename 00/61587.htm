jim_b say: It really depends on how the application was written.  For example if I write an ecoding application the read the video file in at the same speed it would normally play then my CPU utilization will not be very high at all.  I would not reccomend changing the priority to anything above High.  Realtime could actually cause the app to interfere with other apps including the system.  Setting the affinity would not help over all cpu utilization but might speed things up if the application were doing alot of context switching (which you would have noticed as a decrease in time required rather than an increase in utilization).  You could also see if other parts of the system are a bottleneck (is the disk usage pegged, is the ram all in use) which might also make you system unable to use all the cpu time.
