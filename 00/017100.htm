billmerry say:

How can I change sort order of a mail merge from Apple address book?

I have a mail merge set up to create mailing labels in Word for Mac 2011 from the Apple address book. The records don't seem to be in any logical order in the merge and I'd like to be able to order them alphabetically by name. Is there any way to do this?

a. user say:

Word probably is working as it should be, but the "Filter Recipients" Options... button does different things depending on the data source.
 so 
 a. the dialog box Jim's article describes is the one you see for, say, data sources that are Word documents and Excel sheets
 b. if your data source is the Office Address Book, you get to select records by the Categories assigned int he Office address book (AFAICR there used to be problems with the sort there, too, but they might have been fixed since I last looked
 c. if your data Source is the Apple Address Book, you get to select a single Address Book group (I think)
 d. and there may be different things for FileMaker sources. I don't know.

Personally, I would do something similar to Daniel's suggestion, but I would probably use a two-stage merge, along the lines of the following...
 a. Use a Catalog merge to create a new Word Document with a tab-delimited list of all the contact data you need
 b. Use that Word document as the data source for your Label merge.

You can do sorting after step (a) or in the filtering step of the Merge in step (b). 

As long as you need the same fields in your merges you can save the Mail Merge Main Document for the Catalog merge and re-use it by re-opening and merging to a new file.

A problem is inserting a header row in such a way that you only end up with one header row at the end of the merge. To do that, let's imagine you are using three fields - Last_name, First_name and Street. In the following description, <tab> means "the character that you enter bypressing the tab Key, and <cr> means "press Enter", i.e. "start a new paragraph".

When you create your catalog merge document - let's call it cmmmd.docx - ensure that there are three lines. 

1. The first contains a MailMerge IF field. It's probably easiest to enter that by pressing cmd-F9 to create a pair of the "special field code braces" { } 
(See 
http://word.mvps.org/mac/fieldsinmacword.html
if cmd-F9 doesn't do what you expect).

So you have

{  }

Type the following between the braces:
IF  = 1 "Last_name<tab>First_Name<tab>Street"
so you have
{ IF = 1 "Last_name<tab>First_Name<tab>Street" }

Then click after the IF and use cmd-F9 to enter another pair of field braces, so you have
{ IF {  } = 1 "Last_name<tab>First_Name<tab>Street" }
(Make sure there is space after the "F" and before the "=").

Then type the following between those inner braces:
MERGEREC
so you have
{ IF { MERGEREC } = 1 "Last_name<tab>First_Name<tab>Street" }

Click at the end of the paragraph and press Enter.

2. Drag the field codes you need to the next paragraph in your document and put a <tab> between each pair so you have

«Last_name»<tab>«First_name »<tab>«Street»

Click at the end of that and press Enter.

3. Your document should look like this, although you may not be able to see all the fields at this point:

{ IF { MERGEREC } = 1 "Last_name<tab>First_Name<tab>Street" }<cr>
«Last_name»<tab>«First_name »<tab>«Street»<cr>
<cr>

Merge once as an experiment. You only get to merge to a new document. It should have a header row and one row for each data record. If you see a long paragraph filled with text, you need one more paragraph mark at the end of your second line in addressmd.docx. If you have gaps between the rows, you have too many paragraph marks in addressmd.docx.

When you're ready, save the output as as labelds.docx.

Create a new blank document and save it as label1md.docx. Set it up as a Label Merge in the usual way. Use labelds.docx as the Data Source. You should get the standard filtering dialog box for Word document data sources, as mentioned earlier. Re-save and close label1md.docx.

The process of merging these labels then becomes:
 a. close all Word documents.
 b. Open addressmd.docx
 c. Merge to a new document
 d. save it as labelds.docx
 e. close all documents
 f. open label1md.docx 
 g. merge/save/print as necessary
 h. close all documents

If you need more than one type of label merge from the same data, create label2md.docx, and so on. Then repeat (f) to (h), but using label1md.docx, then label2md.docx etc.
