rogeroriginals say:

It is most probably a software issue, so it can be solved fairly easily:

Visit the Nvidia Drivers website, choose your OS (Windows 8.1 in your case) and graphics card version (GeForce GT 720M in your case), download the driver and install it. Reboot your PC after installation, and your PC should detect your graphics card.

If it doesn't, try downloading a bit older version of the same graphics driver, reboot PC and see if it works. If yes, there is an issue with the newer version of the driver, and you'll have to use this old driver until Nvidia fixes the problem.

If the PC still can't detect your card, there might be a hardware issue. In that case you should take your PC to your local repair centre and get it checked.

Windows Device Manager doesn't always download the correct drivers for your graphics card, hence it is highly recommended that you should use the Nvidia Geforce Experience Panel (it gets installed in your PC automatically along with the Nvidia drivers) to update your graphics drivers.

The dllhost.exe process goes by the name COM Surrogate and the only time you're likely even to notice its existence is when it crashes and you get this message.

Method 1:
Run system file checker (SFC) and see if it helps. Please follow these steps:

sfc /scannow

System file checker is a utility in Windows that allows users to scan for corruptions in Windows system files and restore corrupted files.

DISM.exe /Online /Cleanup-image /Scanhealth.
