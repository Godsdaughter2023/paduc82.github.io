bob jones aka: cybertaz say:

I'm glad you got it sorted :-) so just for background information...

Even with outright purchases users do not "own" the software [Microsoft or any other]. The only time we own the software is if there is a contractual agreement with the developer to transfer ownership lock,stock & barrel, which typically occurs only in the case of custom applications costing thousands of dollars. With consumer software the developer retains ownership.

What we purchase is a license to use the software, & those licenses typically allow for a certain number of installations. The number of installations is stipulated in the End User License Agreement [EULA] which varies depending on the release version as well as the specific edition of that release. Based on your description of the history, my guess is that you exceeded the number of installs allowed for your edition.

Part of the problem for Office 2011 & later is caused by using Migration Assistant, because the licensing of the software is associated with the HD on which it is installed. The problem can be avoided if Office 2011 is first Removed from one Mac before installing on another. That frees up the previously used install allowance.

I understand your frustration, but anti-piracy is a very important factor for Microsoft because it is - first & foremost - a software developer. The same is true for Adobe, FileMaker, VMware, Intuit, Symantec, ad infinitum. I'm not offering any sort of defense for the perceived attitude of MS, but Apple, OTOH, is primarily a hardware developer... their software is relatively inexpensive - for the most part supplied free - and runs only on their Operating System which works only with their computers. The threat of pirating Apple software is virtually non-existent, whereas for others it is a devastating concern :-)

Sheet ref in formula for cell in same sheet as formula - Microsoft Community

rich007a say:

Sheet ref in formula for cell in same sheet as formula

Hi,

First up, if this isn't already a widely reported issue, I'd be amazed, but it's one of those things where you just can't find the right thing to put into google...

In answering another post, I disovered that if you have a formula in Sheet1 that includes Sheet1! in a reference to a cell on Sheet1, it all goes wrong when you do a sort.

Example:

On Sheet1

In cells A1:A6 enter numbers 1, 2, 3, 4, 5, 6.

In cell B1 enter  =Sheet1!A1

In cell C1 enter =SUM(B1)

Drag B1:C1 down to row 6.

We should now have this in A1:C6:

1     1     1
2     2     2
3     3     3
4     4     4
5     5     5
6     6     6

Select cells A1:C6, Data tab > Sort > Sort by: Column C, Sort on: Values, Order: Largest to Smallest.  In Excel 2007, I get this:

6     1     1
5     2     2
4     3     3
3     4     4
2     5     5
1     6     6

and the cells now contain the following formulas:

6     =Sheet1!A6     =SUM(B1)
5     =Sheet1!A5     =SUM(B2)
4     =Sheet1!A4     =SUM(B3)
3     =Sheet1!A3     =SUM(B4)
2     =Sheet1!A2     =SUM(B5)
1     =Sheet1!A1     =SUM(B6)

The formulae in column C are as expected, but in column B, the cells are acting like they contained absolute references, i.e. like this before sorting:

1     =$A$1     =SUM(B1)
2     =$A$2     =SUM(B2)
3     =$A$3     =SUM(B3)
4     =$A$4     =SUM(B4)
5     =$A$5     =SUM(B5)
6     =$A$6     =SUM(B6)

Sorting the above by col C produces exactly the same results, i.e. the formulas are now:

6     =$A$6     =SUM(B1)
5     =$A$5     =SUM(B2)
4     =$A$4     =SUM(B3)
3     =$A$3     =SUM(B4)
2     =$A$2     =SUM(B5)
1     =$A$1     =SUM(B6)


If you use the normal relative references (before sorting):

1     =A1     =SUM(B1)
2     =A2     =SUM(B2)
3     =A3     =SUM(B3)
4     =A4     =SUM(B4)
5     =A5     =SUM(B5)
6     =A6     =SUM(B6)

then when we sort col C ascending we get (after sorting):

6     =A1     =SUM(B1)
5     =A2     =SUM(B2)
4     =A3     =SUM(B3)
3     =A4     =SUM(B4)
2     =A5     =SUM(B5)
1     =A6     =SUM(B6)

as expected.

 

[Question 1]: So, is this something new to Excel 2007?  Or has Excel always done this?

(A question you wont be able to answer: I this something I once knew and have since forgotten!?!?) [bonus points available if you can answer that!]

 

But, why would you choose to use a Sheet reference like this in your formula?, I hear you ask.  I wouldn't choose it, Excel chooses it for you!!!

Example, in Sheet1 cell B1, we need the formula     =Sheet2!A1+A1

BUT, instead of typing it, we'll do this:

1) In Sheet1 B1, type   =

2) Activate Sheet2, the formula bar now reads    =Sheet2!

3) Click in cell A1 (on the already selected Sheet2), the formula bar now reads    =Sheet2!A1

4) Press the + key, the formula bar now reads    =Sheet2!A1+

5) Activate Sheet1, the formula bar now reads    =Sheet2!A1+Sheet1!

6) Click in cell A1 (on the already selected Sheet1), the formula bar now reads    =Sheet2!A1+Sheet1!A1

Why doesn't it just say  =Sheet2!A1+A1   ????

[Question 2]: I always thought that Excel was clever enough to remove such sheet references to cells in the same sheet as the formula.  Is that something that has changed from 2007?!?

Rich
