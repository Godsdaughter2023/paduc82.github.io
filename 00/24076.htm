sheenali_msft say:

Hi Rome,

Q: Often times there are countless discussions and revisions being made to a document/on-going project files that are not centralized in one area - what would be a recommended set up for such a task?

A: We recommend using a SharePoint library to centralize the documents. Using SharePoint library gives you benefits such as version history, information policy management, recycle bin, etc.

Q: Is it possible to embed an instant messaging into a team/sub-site interface or something similar?

A: There is no built-in instant messaging service in SharePoint, but there is a site mailbox your team can use to collaborate on composing an important email message or gather relevant team-level email conversations. If you also have a Skype for Business plan, you can send an IM to project members using the full client.

Additionally, you can embed a third-party instant messaging service using a Content Editor Web-Part into your site if you so desire. Do note that this will depend on the IM service you wish to add.

Q: Would I need to create ‘projects’ sub-site?

A: As a best practice, we suggest creating a sub-site for each individual project. This way, documents, files and tasks for a specific project will be centralized. You can also control who has access to the project (and as a result, control sensitive information) this way. You can also use the project task list to manage project tasks.

For your reference:
Organize and configure a SharePoint library.
Get started with a project site.

coldwell shao msft say:

Hi FatalEnema,

Based on my research, This problem occurs if the on-premises server is not sending the required certificate chain during the Transport Layer Security (TLS) handshake to Exchange Online. Would you please try the following steps to see if it works for you?

  1. Get the list of intermediate certificates from the issuer of the third-party certificate.
  2. Export the intermediate certificates from the affected server.

    1. Open the Certificates snap-in on the affected server.
    2. Select Computer account, and then click Local Computer.
    3. In the console tree, expand Personal, and then click Certificates.
    4. Double-click the third-party certificate that's associated with the SMTP service, and then click the Certification Path tab.
    5. Select the intermediate certificate in the certificate path that's listed, click View Certificate, and then click Copy to File on the Details tab.
    6. On the Export File Format page, select DER encoded binary X.509 (.CER), and then click Next.
    7. On the File to Export page, select the file name and path, click Next, and then click Finish.
    8. If there's more than one intermediate certificate i
    9. Import the intermediate certificates that you exported in step 2 to the intermediate certification authorities (CAs) on the sending server. To do this, run the following command in an elevated Windows PowerShell session:

    Get-ChildItem –Path c:\import\intermediateca.cer | Import-Certificate –CertStoreLocation cert:\LocalMachine\CA 

    More details, please refer to the following link:

    https://support.microsoft.com/en-sg/help/3212877/550-5-7-64-tenantattribution-relay-access-denied-smtp-error-when-sendi

    steve rindsberg say:

    This is a recently noticed (but now known) bug.  While we wait for it to get fixed in an update, here's a workaround:

    Crop the picture as you're doing now.

    Right click the picture and choose COPY

    On the Home tab, click the downward pointing triangle under PASTE and choose PASTE SPECIAL.  Choose to paste as PNG.

    Save the newly pasted version of the image as a picture.  It won't be distorted.
