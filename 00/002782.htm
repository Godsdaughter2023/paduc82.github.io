If that doesn't work, try:

 

Re-register the application

There are several different ways you can reset the proper file associations. Word itself provides a way to do it. Open a command prompt window and, at the command line, enter the following:

winword /r

Nothing seems to happen when you press Enter, but Word will reregister itself back in Windows. In the process, it resets the file associations for the files it uses, such as DOC and DOT files. You can close the command prompt window and double-click on a document; Word should start right up.

 

If that doesn't work, try:

 

Alternate Re-register the application

Close the app. Open the Run Window: <WIN> R, use the following command (Modify the Path to match your specific installation):

"C:\Program Files\Microsoft Office\Office\winword.exe" /unregserver

The quotes are necessary, but you may need to change the path so it reflects the location of the winword.exe file on your system. (You can use Windows’ Search tool to locate the path to the winword.exe file.) The /unregserver switch “unhooks” all the references to Word in the Windows Registry.

You should then, immediately, use the following command:

"C:\Program Files\Microsoft Office\Office\winword.exe" /regserver

This command causes Word to rewrite all its Registry keys and to reassociate itself with document and template files. Now you can double-click a document file and see if Windows starts Word and opens the document.

 

If that doesn't work, try:

 

Need to associate the file type with program:

In Windows 7:

  • Right-click the file,
  • Select "Open With" select [Your Program Here]
  • Check “always use this program”.

Alternate method of associating file type with program:

  • Click Start
  • type cmd in the run dialog box then press ok.
  • On the command prompt line type "control panel" without the quotation marks.
  • In control panel > folder options > file types > new (button) in the new window click advanced.
  • In the file extension dialog box type the file extension to fix, ie: exe.
  • In the drop down menu locate and select application > click ok.

macropod say:

A mailmerge can only use one data source. However, that does not of itself rule out conditionally referencing the three tables in the other document, which you can do via an INCLUDETEXT field, coded along the lines of:

{QUOTE{INCLUDETEXT "C:\\Table Document Path\\TableDoc.doc" "Tbl{IF{MERGEFIELD CustAcct}= "*10" 10 {IF{MERGEFIELD CustAcct}= "*20" 20 30}}"}}}

Where 'C:\\Table Document Path\\TableDoc.doc' represents the full path and name of the source document, 'CustAcct' is the field name and the tables in the source document are given the bookmark names Tbl10, Tbl20 and Tbl30, respectively.

Note: The field brace pairs (ie '{ }') for the above example are created in the body of the document via Ctrl-F9 (Cmd-F9 on a Mac) - you can't simply type them or copy & paste them from this message. Similarly, you cannot use single back slashes (ie '\') as the path separators in the INCLUDETEXT field - you must use either double backslashes ('ie \\) or single foward slashes (ie '/').

From this description and the preceding post saying that double-clicking the "Same As Previous" closes the header, I think the problem is a misunderstanding.

In Word 2007 (and 2010), there is a blue tag that says "Same as Previous" attached to the dashed line at the bottom of the header area. That isnot the same as the button on the Header & Footer Tools ribbon that says "Link to Previous", and which is orange when the headers are linked.

If you're double-clicking the blue tag, that's equivalent to double-clicking anywhere else in the body of the document, which closes the header pane. DON'T DO THAT! The blue tag isnot a button. Instead, single-click the orange button in the ribbon.

This is an unfortunate carry-over from Word 2003. There, though, the only place that said "Link to Previous" was the tooltip of the button on the Header/Footer toolbar, and the "Same as Previous" text in the header didn't have any background and didn't look like anything clickable.

Dialling Directly On Your SIM Memory on S60 Phones - Microsoft Community

friedbrains say:

Dialling Directly On Your SIM Memory on S60 Phones

Technical Level : Intermediate

Summary

Details
As S60 users, you are very much aware that you can not set your S60 devices to use the SIM contacts as default not like S40 phones, where you can choose either to have phone memory or SIM memory as default for the contacts.

However, there is a bit of a work around to fast-dial from your SIM card memory or SIM Contacts.

If you press any number say 10 and then [#] key, it will show whatever you have put in the Number 10 slot in your SIM memory, and then you can press the Green Call key to dial that number:

Example:

SIM Slot # 10 = Name: Jenny; Phone Number: +0123456789

Press 10 + # key will bring out -> 0123456789

You then press the Green Call key to call Jenny

Note: This will only work if you have put any number on your SIM memory and of course you have to somehow remember who is that person in that particular slot, in our example above "Jenny". The trick is then to put perhaps your 10 most called numbers in the 1st 10 slots of your SIM Memory and just try to remember who is in slot 1 and so forth.

Its not much but its a work around.

As of S60 3rd edition with Feature Pack 2, these gymnastics are no longer necessary since you can once again elect to have SIM contacts appear in your address book directly. In the contacts list, do this:

Options > Settings > Contacts to display

...and you can choose whether or not to display contacts from the phone memory, the SIM card and service numbers.
