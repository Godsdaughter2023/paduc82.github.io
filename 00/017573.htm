The Control Panel Mail Icon problem is a newly discovered issue, here's how to deal with it:

Symptoms

You have Microsoft Outlook 2013 Click-to-Run installed and Mail does not open when you double-click it in Control Panel.

Cause

This problem occurs after you install the June 2016 update for Office 2013 Click-to-Run. This is version number 15.0.4833.1001, or a later version. To determine whether you are using a Click-to-Run installation of Office 2013 and to locate the version number of the installation, see the 'More Information' section.

Workaround

To work around this issue, use one of the following methods, depending on your needs.


Method 1: Start Outlook with the 'Choose Profile' dialog box by using a command-line switch

This method is useful if you must create a new Outlook profile, select which Outlook profile to start, or set a different default Outlook profile. If you must access Outlook Account Settings, use Method 2.

 

  1. Exit Outlook if it is running.
  2. Open a Run dialog box. To do this, press the Windows Key + R.
  3. In the Run dialog box, enter the following command-line switch, and then press Enter.

    Outlook.exe /profiles

    Note For more information about command-line switches, see the following article: Command-line switches for Outlook for Windows


Method 2: Revert Office 2013 Click-to-Run installation to an earlier version


To work around this issue, revert the Office 2013 Click-to-Run installation to the 15.0.4823.1004 version. To do this, follow these steps:

 

  1. Exit all Office applications.
  2. Open an elevated command prompt. To do this, click Start, type cmd in the Start Search box, right-click Command Prompt or cmd.exe, and then click Run as administrator.
  3. At the command prompt, type the following command, depending on your bitness of Windows, and then press Enter:

    For an Office 2013 installation and a 32-bit version of Windows:
cd %programfiles%\Microsoft Office 15\ClientX86

For an Office 2013 installation and a 64-bit version of Windows:

cd %programfiles%\Microsoft Office 15\ClientX64

 

  1. Type the following command, and then press Enter:
officec2rclient.exe /update user updatetoversion=15.0.4823.1004

 

  1. When the repair dialog box appears, click Online Repair.
  2. Click Repair, and then click Repair again.
  3. After the repair is complete, start Outlook.
  4. Click File, and then click Office Account.
  5. In the Product Information column, click Update Options, and then click Disable Updates.

    Important note This step is very important. The repair process re-enables automatic updates. To prevent the newest version of Office Click-to-Run from being automatically reinstalled, make sure that you follow this step.

bob jones aka: cybertaz say:

The document won't print correctly for several reasons, but the primary problem is that every attribute in Format> Paragraph - Line & Page Breaks is applied to every Paragraph in both tables in both text boxes. Several of those attributes contradict one another, not to mention the conflict created by "Page Break Before" being applied to every paragraph.... especially since Page Breaks can't occur in a Text Box to begin with. Quite honestly, I can see no reason for text boxes to have been used in the first place, but that's another can of worms :-) There's also a Page Break at the very beginning of the 1st page which serves no purpose.

In order to at least get the document printable:

  1. Click in the first text box then use Command+A to Select All
  2. Go to Format> Paragraph - Line & Page Breaks
  3. Clear all of the check boxes
  4. Repeat for the 2nd text box

talking moose say:

The behavior you’re seeing is expected: Outlook for Mac will download the body of the message first and display it while it downloads the attachment in the background.

What I haven’t seen mentioned yet is the sizes of the attachments. If they’re a few KB then they should download fairly quickly, but if they’re multiple MB then I would expect some time for them to download.

Outlook for Mac has no governor for speed. It downloads as quickly as the network allows.

Many things influence speed of download:

  • Wired vs. Wi-Fi
  • Internet connection speed (cable modem vs. DSL vs. corporate network)
  • Your Mac’s wired Ethernet duplex setting: Apple menu > System Preferences > Network > Network connection > Advanced > Hardware tab.
  • Attachment size
  • Corporate proxy server

If you’re dealing with large attachments consider using a free cloud service such as OneDrive or DropBox to share files. You can allow folks to edit online and avoid multiple versions of documents. Also, printing to PDF usually results in file sizes similar to the original, but using an application’s Save As feature (if available) to save as PDF often gives choices for file sizes that are much smaller.

Hope this helps!
