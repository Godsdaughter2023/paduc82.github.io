Install-WindowsFeature RSAT-ADDS

 

3. Run the command as below from Windows Powershell to install the required windows roles/components:

 

Install-WindowsFeature AS-HTTP-Activation, Desktop-Experience, NET-Framework-45-Features, RPC-over-HTTP-proxy, RSAT-Clustering, RSAT-Clustering-CmdInterface, RSAT-Clustering-Mgmt, RSAT-Clustering-PowerShell, Web-Mgmt-Console, WAS-Process-Model, Web-Asp-Net45, Web-Basic-Auth, Web-Client-Auth, Web-Digest-Auth, Web-Dir-Browsing, Web-Dyn-Compression, Web-Http-Errors, Web-Http-Logging, Web-Http-Redirect, Web-Http-Tracing, Web-ISAPI-Ext, Web-ISAPI-Filter, Web-Lgcy-Mgmt-Console, Web-Metabase, Web-Mgmt-Console, Web-Mgmt-Service, Web-Net-Ext45, Web-Request-Monitor, Web-Server, Web-Stat-Compression, Web-Static-Content, Web-Windows-Auth, Web-WMI, Windows-Identity-Foundation

 

4. Once above component gets installed, reboot the server

 

5. Install .net framework 4.5.2

 

6. Download and install Microsoft Unified Communications Managed API 4.0

 

Perform Active Directory schema update by running the following command from command prompt and a path where you’ve downloaded the Exchange Server 2013 installation files.

 

setup.exe /PrepareSchema /IAcceptExchangeServerLicenseTerms

 

7. Next prepare Active Directory by running the following command.

 

setup.exe /PrepareAD /IAcceptExchangeServerLicenseTerms

 

This will prepare the Configuration Container of Active Directory forest, upgrading the AD objects that support the Exchange Organization.

 

Note: The account to run above commands as mentioned in Steps 6 and 7 must be a member of the Schema Admins and Enterprise Admins group.

8. Test AD replication

9. Run Exchange Server 2016 installation file as administrator. Ensure that you’ve latest cumulative update file for Exchange server 2016.

 

10. Once Exchange Server 2016 installed, start Exchange Admin Console and ensure that you’re seeing all the exchange servers in your environment.

 

11. License the Exchange Server 2016 using free Exchange Hybrid Product key.

12. Manage Exchange 2016 Certificate -  not sure if this is required

 

13. Create new Mailbox Database Availbility Group (DAG)

14. Re-point Virtual Directories for autodiscover, ECP, AOB, ActiveSync and WebServices to new Exchange 2016 servers

15. Run Hybrid Server Configuration Wizard on Exchange Server 2016

If you can't get Autorecover to work for some reason you have 2 alternatives:

- save it regularly yourself, manually

- run a macro to save it at regular intervals (something like this:-

Dim dtNext As Date
Const dtInterval As Date = #00:02# ' 2 minutes

Sub Auto_Open()
  dtNext = Now + dtInterval
  Application.OnTime dtNext, "SaveCopy"
End Sub

Sub Auto_Close()
  Application.OnTime dtNext, "SaveCopy", schedule:=False
End Sub

Sub SaveCopy()
  If ActiveWorkbook Is Nothing Then Exit Sub
  Application.DisplayAlerts = False ' don't ask about overwriting etc
  ActiveWorkbook.SaveCopyAs ActiveWorkbook.Path & "\Safe " & ActiveWorkbook.Name
  Application.DisplayAlerts = True
  dtNext = Now + dtInterval
  Application.OnTime dtNext, "SaveCopy"
End Sub

Note - you can put that macro in a separate add-in file which you load on startup - it will make a copy of whatever workbook is active at the time.
