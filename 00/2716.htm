__jthomas say:

The find tags functionality already is, and has been for a long time, available in One Note for Windows 10. Merely "finding" tags is not the issue raised by the OP. Instead, the OP is asking about tag summary, which takes tag "finding" to a completely different level.

A tag summary creates a new page in a notebook. It's called, creatively enough, a tag summary page. On that page, it groups & sorts tags contained within the geography of the notebook that you specify -- the entire notebook, or a section within the notebook, etc. The tags can be affixed to content at any level of granularity -- a tag applied to an entire page, and tags applied to one or more specific pieces of content contained on a single page, including images.

Suppose you have the following tags: Apples, Oranges, Bananas. Within one of your notebook sections, you have applied the Apple tag 3 times, the Oranges tag 5 times, and the Banana tag 2 times.

When you create a tag summary page in that notebook, One Note 2016 will show you, in one place, the 3 items collected together under the tag apples, and in the same way the 5 collected under the tag Oranges, and the 2 collected under the tag bananas. The page will include all the other tags you use, too, but the summary will make it easy to find all the stuff that falls under the tags you want to examine. You don't have to run separate searches for each tag. The summary shows you a nice, grouped and sorted list of all the tags you have in the notebook, and the content to which they are attached. That's not merely "search." That's "find all the tagged content, put them in their own piles, and organize the piles all on a single page."

Even cooler, when you click on any of the items in the tag summary, One Note will take you directly to that page, or to the specific piece of content, in that notebook.

This kind of functionality is crazy useful, and it's a Big Gaping Hole in One Note for Windows 10.

 I understand your post correctly, you want to dynamically take a value from the current row, and run a sumif against a target data set. You want to do this 36,000 times, against a data set of 500,000 rows. ...and you want to do it quickly.

When working with similarly large data sets, the most effective thing I've done to speed it up is to vastly reduce the number of rows I need to reference in advance, like so. Sometimes I do these setup pieces manually, but if it is going to happen more than once a quarter, I automate all of this in code.

1. Sort the source data (the 500K row files) on whatever field will be used for your match.

2. On a fresh sheet, copy just that column and remove duplicates. In the next column, use MATCH(x,y,false) to find out the start row of each value in your 500K raw data

3. In code or in your formula, start with a match against the short list to find the value, then use INDEX to bring back the first row you need to use. Also grab the row number from the next line and subtract 1, and that will be your end row

4. Use just this smaller range in your formula (again, I prefer VBA to an overly complex cell formula). You can do this using INDIRECT to provide the sumif range, but at this point if your data set is sorted you no longer need a sumif, you should be able to just sum the range referenced in your INDIRECT

I think you will be pleasantly surprised at the difference.

jay freedman say:

Between the ActiveDocument.Bookmarks statement and the ActiveDocument.SaveAs statement, insert this statement:

ActiveDocument.Protect Type:=wdAllowOnlyFormFields, Password:=""

If you want to have a password that's required to turn off the protection, include it between the quote marks.

Note that the template should not be protected for forms, because that would prevent the insertion of the order number at the bookmark. The protection must be turned on after that insertion, but before the document is saved.
