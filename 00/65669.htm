rudolf brunner (telematix) say:

I had the same problem. I found a solution for our company but it took time and is in my eyes not the final solution... there is probably a better one.

1. Uninstall/Deactivate the anti virus tool (in our case the symantec endpoint security)

2. Remov the computer from the domain

3. Stopp the Windows Update Service in command shell executed as an administrator with
   net stop wuauserv

   Maybe this command has to be executed twice till service is stopped

4. Delete the folder C:\Windows\SoftwareDistribution

5. Restart the computer and log in with a local admin account

6. Start the Windows Update which will install the 1709 Update.

7. Add the computer again to the domain

8. If a previous user tries to log in again at this computer he will be logged in with a temporary profile

   So that he can use his previous profile a change in the Registry must be done.

   Search for HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList and select the subfolder that belongs to this user (you will see the username in the variable (ProfileImagePath)

This ProfileImagePath contained in my version C:\Users\nbrunr.TELEMATIX or C:\Users\nbrunr.000 whereas the previous profile was simply under C:\Users\nbrunr. So simply change this value to your previous folder. After a re-login you should have your old profile running on this machine.
