Opening a Form through a Combo Box Selection - Microsoft Community

maddyruth say:

Opening a Form through a Combo Box Selection

Hi!

I want the user of my database to be able to open pre-made reports from the homescreen (without having to search through the navigation panel). How do I do this using a combo box?

That is - what is the code needed behind the combo box so that selection X opens report X etc.?

ken sheridan say: First make sure that each of the reports you wish to include in your list has a plain English (or any language of your choice) name for the report as its Description property.  You can do this by right-clicking on the report in the navigation pane and selecting 'View Properties' from the shortcut menu.  Enter the description on the dialogue and clock OK.

I'd suggest you consider using a multi-select list box as it allows more than one report to be selected simultaneously.   For this add a list box named lstReports to your form.  In the form's Open event procedure put the following code:

    On Error GoTo Err_Handler

    Const NODESCRIPTION = 3270
    Dim ctrl As Control
    Dim dbs As DAO.Database
    Dim ctr As DAO.Container
    Dim doc As DAO.Document
    Dim strReportname As String
    Dim strDescription As String
     
    Set dbs = CurrentDb
    Set ctr = dbs.Containers("Reports")
    Set ctrl = Me.lstReports
    
    ctrl.RowSourceType = "Value List"
    ctrl.ColumnWidths = "0"
    ctrl.BoundColumn = 1
    ctrl.ColumnCount = 2
     
    For Each doc In ctr.Documents
        strReportname = doc.Name
        On Error Resume Next
        strDescription = doc.Properties("Description")
        Select Case Err.Number
            Case 0
            ' no error
            Case NODESCRIPTION
            ' report has no description
            ' so ignore and go to next report
            GoTo EndLoop
            Case Else
            ' unknown error
            GoTo Err_Handler
        End Select
        Err.Clear
        On Error GoTo Err_Handler
        ctrl.AddItem strReportname & ";" & strDescription
EndLoop:
    Next doc
    
Exit_Here:
    Exit Sub
    
Err_Handler:
    MsgBox Err.Description, vbExclamation, "Error"
    Resume Exit_Here

If you do use a combo box to select one report the above code to fill its list will be the same, and you can simply put:

   With Me.ActiveControl
       If Not IsNull(.Value)
           DoCmd.OpenReport .Value, View:=acViewPreview
       End If
    End With

in the combo box's AfterUpdate event procedure.

If you do use a multi-select list box add a button to the form with the following code in its Click event procedure;

     Dim strReport As String
     Dim varItem As Variant
     
     With Me.lstReports
        If .ItemsSelected.Count > 0 Then
            For Each varItem In .ItemsSelected
                strReport = .ItemData(varItem)
                DoCmd.OpenReport strReport, View:=acViewPreview
            Next varItem
        End If
    End With

The user can select one or more reports in the list box and by clicking the button each of the selected reports will be opened in print preview.  If you wish you can add another button to the form to clear the selections in the list box by putting the following code in the button's Click event procedure;

    Dim n As Integer
    
    With Me.lstReports
        For n = .ListCount - 1 To 0 Step -1
            .Selected(n) = False
        Next n
    End With

scottgem say:

This can be done pretty simply. The simple answer is use this as the RowSource of your combo:

SELECT MSysObjects.Name
FROM MSysObjects
WHERE (((MSysObjects.Type)=-32764));

This will list all your reports. To launch the report use the After Update event of the combo and run the code:

DoCmd.OpenReport me.lstReports

However, this is very simplistic and requires the user to select from a list of the actual report names which may be cryptic. Another shortcoming to this is that frequently, you enter criteria to filter a report that requires that a form be opened first. This method does not account for that.

So what I do, is create a table of reports.

tblReport

Reportname

Reportdescription

Formname

I then create a combo with a RowSource of

SELECT Reportname, ReportDescription, Formname

FROM tblReport

ORDER BY ReportDescription;

I set the Bound column to 1, the ColumnCount to 3 and the column widths to 0";3";0"

So the user opens the combo and sees a list of descriptions. 

In the After Update event of the combo I use code like:

If IsNull(Me.cboReport.column(2)) Then

     DoCmd.OpenReport Me.cboReport

Else

    DoCmd.OpenForm Me.cboReport.Column(2)

End If

So the user selects a report. If there is a form associated with the report, the form opens, otherwise the report opens.
