john korchok say:

There was an old animation technique from the early days of computing called palette rotation. It works by keying fixed graphics to an indexed color palette, then switching palettes so the graphic appears to change.

This is still a handy technique in PowerPoint. Each master can actually contain several alternate color themes, though you have to add them by editing the XML. Then each graphic element is linked to a position on the main theme. Then when you switch color themes, different elements appear to change color or turn on and off, because the colors actually trade places in the alternate themes.

Of course, this could play havoc with other presentation elements that are also keyed to the theme, like charts. You also probably still need more than one master, since changing the color palette affects all slides based on that master. It requires planning to work well and might be a little more complicated than your current skill level.

I have a series of articles on XML editing that will help you get started. First read the basics: XML Hacking: An Introduction. Then read the article about alternate color themes: Color Themes

Feel free to post back with any issues, this helps me develop the articles and improve the information.
