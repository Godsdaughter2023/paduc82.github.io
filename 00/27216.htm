Take a look at this macro, "EmailClean". It does a bunch of finds and replaces. You can use it as a template to start from to get to what you want

EmailClean Macro

I posted the following macro a long time ago.  It will reformat selected text by changing a paragraph mark into a space, changing 2 or more paragraphs to 1, changing 2 or more line breaks to 1, & putting 2 spaces between all sentences.  It also removes white space at the beginning & end (& middle) of each sentence. You can modify it to put only one space betweeen sentences.  Anything that starts with a ' is a comment & you can delete it, if you want.

Note: Do not use the macro on other macros (as it gets rid of the paragraph marks).  Just copy & paste the macro into a global template.  I've placed it on a menu so that I can call it just by typing Alt, M, E.

Sub EmailClean()

'

' EmailClean Macro

' operates on selected text

' Macro created 6/12/01 by Phil Rabichow

'

    Application.ScreenUpdating = False

    Selection.Range.Style = ActiveDocument.Styles(wdStyleNormal)

    Selection.Find.ClearFormatting

    Selection.Find.Replacement.ClearFormatting

 

' Replace line break with paragraph mark

    With Selection.Find

        .Text = "^l"

        .Replacement.Text = "^p"

        .Forward = True

        .Wrap = wdFindStop

        .Format = False

        .MatchCase = False

        .MatchWholeWord = False

        .MatchAllWordForms = False

        .MatchSoundsLike = False

        .MatchWildcards = False

    End With

Selection.Find.Execute Replace:=wdReplaceAll

 

' Remove white space at the beginning of lines

    With Selection.Find

        .Text = "^p^w"

        .Replacement.Text = "^p"

    End With

Selection.Find.Execute Replace:=wdReplaceAll

 

' Remove spaces before paragraph marks

    With Selection.Find

        .Text = " {1,}^013"

        .Replacement.Text = "^p"

        .MatchWildcards = True

    End With

Selection.Find.Execute Replace:=wdReplaceAll

 

' This prevents extra paragraph marks at the end of the selection

    Selection.MoveLeft Unit:=wdCharacter, Count:=1, Extend:=wdExtend

 

' Removes spaces in first line

    With Selection.Find

        .Text = " {3,}"

        .Replacement.Text = ""

    End With

Selection.Find.Execute Replace:=wdReplaceAll

        .Replacement.Text = "%$%"

        .MatchWildcards = False

    End With

Selection.Find.Execute Replace:=wdReplaceAll

jay freedman say:

The problems you had with the modified macro are caused by mostly by misunderstanding the way the wildcard search works. First, you cannot simply stuff all the things you're looking for into one .Text string and search for all of them simultaneously. You can store the list in a string like that, but the macro has to break them out into separate strings and search for one at a time. The Split function creates an array of strings from a single string that has separators (in this case, the pipe character, "|").

Additional problems come from the way the items are arranged in your .Text string. The \1 code in the Replacement string refers to whatever is in parentheses in the .Text string, so only the person's title (Mr. etc.) should be in the parentheses; the spaces after it must be outside the parentheses so it doesn't wind up in the Replacement.

Finally, the square brackets don't belong in the string that goes into the Split function.

This version will work:

Sub x()

    Dim strTargets As String

    Dim srchTxt As Variant

    Dim idx As Integer

    Dim rg As Range

    

    strTargets = "(Mr.)  |(Mrs.)  |(Lt.)  |(Col.)  |(Ret.)  |(Hon.)  |(Rep.)  |(R.S.V.P.)  |(Dr.)  |(U.S.)  "

    srchTxt = Split(strTargets, "|")

    

    For idx = 0 To UBound(srchTxt)

        Set rg = ActiveDocument.Range

        With rg.Find

            .Text = srchTxt(idx)

            .Replacement.Text = "\1 "

            .Forward = True

            .MatchWildcards = True

            .Execute Replace:=wdReplaceAll

        End With

    Next idx

End Sub

The advice you will get here has been given many times:

1. it is a really bad idea to share a monolithic database on the server

ken sheridan say: What is your objection to splitting the database into front and back ends?  If it is that a user might open the database from different machines on the network, rather than from the same machine each time, then the solution is  a simple one, which was adopted in my organization:

1.  Give each user a personal folder on the system, to which only they and the system administrator has access.

2.  Split the database into front and back ends, putting the back end in a shared location on the system, to which all authorised users have full permissions.

3.  Install a separate copy of the front end in each user's personal folder.  To open the database, they then do so in their personal folder, from any machine on the system.  Each user is consequently using a separate front end to access the database, as should be the case, and the high risk of corruption which results from simultaneous use of a single unsplit database is avoided.

rick xu msft support say:

Hi Jnespo,

 

An email is marked as a spam for multiply reasons. How to identify if an email is a spam or not depends on the value of SCL and BCL (you  can find it in mail header).  When the Microsoft Exchange Online Protection scans an inbound email message it inserts the X-Forefront-Antispam-Report header into each message. These headers can help the admins with information about the message & how it was processed. Also the X-Microsoft-Antispam header provides additional information about bulk mail and phishing.

SFV – means Spam Filtering Verdict

SFV:SFE – Originated from a Safe Sender (EOP learns Outlook safe senders due to Windows Azure Directory Sync)

SFV:BLK – Originated from a Blocked Sender

SFV:SPM – Spam

SFV:SKS – (SKIP) The message was marked as spam prior to being processed by the spam filter. This includes messages where the message matched a Transport rule to automatically mark it as spam and bypass all additional filtering

SFV:NSPM – Not Spam

 

“ I know SCLs are based on spam scores given, but what goes into a spam score and what are some factors that could make it change so much so quickly?”

  1. false positives
  2. message including links or attachments, etc.  For more information, please refer to this article: Advanced Spam Filtering Options
  3. some third party anti-virus

As an Office 365 admin, you can create a transport rule that sets the spam confidence level (SCL) of an email message. For more information, please see this article: Create rule to identify mail as spam or not spam by setting the spam confidence level (SCL).

 

“We're trying to figure out if a change occurred on Microsoft's side in Exchange 365 or if there are targeting or content changes that need to be employed.”

 

As far as I know, there is no related article or channel to get it.
