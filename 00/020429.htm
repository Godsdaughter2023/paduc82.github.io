ken sheridan say: To do this as part of the table definition, in the Lookup tab of Table Properties in table design view, set the DisplayControl property of the Gender column to Combo Box, its RowSourceType property to Value List, and its RowSource property to Male;Female.

However, as users should never interface with a table in raw datasheet view, but only via forms, doing so in the table definition serves little purpose.  You can do exactly the same with a bound combo box in a form.

A value list should only be used where the values are a set immutably fixed in the external world.  Otherwise a referenced table should always be used, with referential integrity enforced in the relationship between the referencing and referenced table.  To do otherwise would be encoding data in the object definition.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.

anula d'souza say:

Hi Wendy,

As I understand you do not want the emails to be present on web-mail once the emails gets downloaded into Outlook.

 

The option to leave a copy of Messages is present only in POP account. If you have any other email account (like EAS (Exchange Account Sync.), IMAP or Exchange) all the emails from web-mail will be synchronized with Outlook and if you delete any email from web-mail the emails gets deleted from Outlook too.

 

What is the type of email account configured in Outlook?

If it’s a POP account follow the steps mentioned:

·         Click on File< Account Settings< Email tab.

·         Highlight the email account and click Change.

·         Click on More Settings.

·         In Advanced tab under Delivery group uncheck the option Leave a copy of messages on the server and click Ok and Finish.

 

I hope the above suggestions help. Let us know if you need any further assistance.

Thank you.
