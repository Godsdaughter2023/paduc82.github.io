    Excel in depth and excel F&F which is better - Microsoft Community

    jackamin say:

    Excel in depth and excel F&F which is better

    Hi there, I need some help before purchasing, I want to get a copy of MrExcel's 2013 excel book. however whether the In depth or the formula and functions I cant really decide. advice please

    chip pearson say:

    If you want to do Excel work commercially, you'll need to be very fluent is (at least) four areas: (1) functions and formulas, (2) the tools built in to Excel, such as Pivot Tables and Query Tables, (3) the VBA programming language, and (4) the Excel Object Model, which is how Excel communicates with VBA and other automation clients.

     

    Strictly speaking, a formula is built up from one or more functions. SUM is a function. =SUM(A1:A10) is a formula with one function, SUM. =IF(A1=1,COS(B1),SIN(B1)) is a formula with three functions: IF, COS, and SIN. You'll also need to know how to effectively use array formulas, described at http://www.cpearson.com/Excel/ArrayFormulas.aspx .

     

    You'll need to learn the VBA programming language. VBA is a fairly simple language. If you've programmed in another language, the transition to VBA will be easy. If you've never programmed before, VBA is a good place to start, much simpler to get started with than C# or C++.

     

    The Excel Object Model is an abstraction of the entire Excel environment. It defines entities like worksheets and ranges of cells, the properties of those entities, and the methods you can use to manipulate them. You'll likely spend much more time learning the Excel Object Model than learning the base VBA language.

     

    You'll need to know VBA and the Excel Object Model very well to do commercial level work in Excel. At some point (probably sooner than later), you'll need to learn some of the technologies that are related to but not part of Excel. These include the NET Framework, a NET-based programming language (e.g., VBNET or C# -- if you're going into NET, go with C# over VBNET), and XML and XML Schemas. You'll also need to know database technologies like ADO to connect Excel to a database like SQL Server or Oracle.

     

    Remember that your clients will expect you to know all this when they hire you. They are not going to pay for the time it takes you to learn something.

     

    I'm not familiar with Mr. Excel's books (though I do have one of his Power Pivot books). John Walkenbach's "Excel 2007 Formulas" comes highly recommended for functions and formulas, and his "Power Programming For Excel" books are very good for learning VBA and the Excel Object Module.

    Create a standard module by selecting Insert > Module in the Visual Basic Editor.

    Copy the following code into the module:

    Function NeedPoints(AssessedNeedAmt As Variant) As Variant
        If IsNull(AssessedNeedAmt) Then
            NeedPoints = Null
        ElseIf Not IsNumeric(AssessedNeedAmt) Then
            NeedPoints = Null
        ElseIf AssessedNeedAmt > 10000 Then
            NeedPoints = 25
        Else
            NeedPoints = (AssessedNeedAmt - 1) \ 400
        End If
    End Function

    In the query based on the table, create a calculated column:

    Points: NeedPoints([AssessedNeedAmt])
