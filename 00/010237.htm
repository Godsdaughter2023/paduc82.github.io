groov85,

First of all, since you are the only resource, you do not need/want to apply your custom calendar at task level. Instead, you should simply make your custom calendar the Project calendar via Project > Properties group > Project Information > Calendar. Each Task Calendar should be "none" meaning tasks will follow the Project calendar. You should also make sure your Resource Base Calendar is set to your custom calendar.

Second, it sounds like you may be assigning yourself to summary tasks. Do not do that. Assign yourself only to individual performance tasks (i,e. subtasks under summary lines).

Third, what is your leveling basis (i.e. minute, hour, day, etc.)? What is your leveling order?

Fourth, do any of your tasks have constraints? Constraint Type for all tasks should be "as soon as possible".

A screen shot or snippet of your plan would also help.

Import Access query result into Excel spreadsheet

I have tried to "connect" connection from Excel to Access via VBA and have run into a wall. I have posted a question about this but have received no response.

I am not a VBA expert .

hansv mvp say:

Select Tools > References in the Visual Basic Editor.

Scroll down until you see Microsoft Office Access 12.0 database engine Object Library.

Tick its check box, then click OK.

Copy the following macro into a module:

Sub GetQuery()
    Dim dbs As DAO.Database
    Dim rst As DAO.Recordset
    Dim i As Long
    Dim wsh As Worksheet
    Set dbs = DBEngine.OpenDatabase("path+filename here")
    Set rst = dbs.OpenRecordset("query name here")
    Set wsh = Worksheets("Sheet1")
    For i = 0 To rst.Fields.Count - 1
        wsh.Cells(1, i + 1).Value = rst.Fields(i).Name
    Next
    wsh.Range("A1").Resize(ColumnSize:=rst.Fields.Count).Font.Bold = True
    wsh.Range("A2").CopyFromRecordset rst
    rst.Close
    Set rst = Nothing
    dbs.Close
    Set dbs = Nothing
End Sub

Substitute the path and filename of the database, the name of the query and the name of the worksheet for the text indicated in bold.
