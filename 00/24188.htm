    to preserve multiple users in security & compliance center, we suggest you use the powershell command “set-retentioncompliancepolicy”. i’d like to share with you the general steps below. however, please understand that to add more than 100 "one drive for business" users’ personal site urls, you will need to export their personal site urls and import them to powershell by using scripts. therefore, we suggest you post the question in our msdn forum which is a specific support channel for the related issues and questions. we appreciate your understanding.

    1. go to permission in security & compliance center, and make sure you have the compliance administrator permission.

    2. add a policy manually in retention. for example: policy name: 123
    3. set the policy by adding the personal site url (onedrive for business site) using powershell (open windows powershell by selecting run as administrator).
    a. connect the tenant running the following command:
    $usercredential = get-credential
    $session = new-pssession -configurationname microsoft.exchange -connectionuri ps.compliance.protection.outlook.com/.../ -credential $usercredential -authentication basic -allowredirection
    import-pssession $session
    for more detailed information, you can refer to: https://technet.microsoft.com/en-us/library/mt587092
    b. add personal sites to the policy you already created:
    set-retentioncompliancepolicy -identity "123" -addsharepointlocation "https://***-my.sharepoint.com/personal/***" (personal site urls)
    note: to enter multiple values of personal sites, use the following syntax: <value1>,<value2>....
