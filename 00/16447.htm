    john - project say:

    Gunner the Intern,

    If you have a dynamic master you are already rolling the dice. Dynamic structures in Project are unfortunately prone to corruption. None of the files in the structure must ever be moved, renamed, overwritten or "saved off" to another location. And ideally, all files should be in the same folder on a local drive and not be operated on a server.

    Now, with that said, if you want to take one of the linked subprojects and  purge it of a bunch of tasks (i.e. everything below the subtasks of the ID 5 summary line), then I suggest you simply select and delete all those tasks and then save. That will be the most efficient and safe way.

    However, you also mentioned that you want to know if Project has something that can help you. Maybe yes. Is there a reason you need a dynamic master? With a static master, (non-linked structure), you can avoid the potential for corruption, although since a static master is a snapshot in time, you would need to create a new one each time you need/want an update of all the files but that is easily automated with a recorded macro.

    Hope this helps.

    John
