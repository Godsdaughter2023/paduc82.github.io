ryan g. steele say: Our users were having issues with certain web sites (including Twitter) on Windows 8 with IE 10. They were seeing the exact same symptoms you are experiencing. I determined that the issue was occurring for any web site which makes use of a feature in HTML5 called Local Storage, which allows web sites to store files on your computer for later use.

After working with Microsoft support, we identified the cause. It turned out to be a problem with the settings on the C:\Users\username\Appdata\LocalLow folder in their user profile.

Each folder on your computer has an integrity setting. More information about the purpose of this setting is here. 

The integrity setting on the AppData\LocalLow folder in each user's profile is supposed to be set to "Low" (hence the name). In our case, the integrity level was not set correctly on this folder. To rectify the problem, run the following command in a command prompt window:

icacls %userprofile%\Appdata\LocalLow /t /setintegritylevel (OI)(CI)L

(If there is more than one user account on the computer and the other users are having the same issue, the command needs to be run under each affected user's account.)

As for how this setting got changed in the first place? In our case, it was caused by a problem in the customized Windows 8 image we deployed to our workstations. More details about that are here. 

For others that are having the issue, a "system cleaner" utility may be to blame.

Hope this is helpful!

Adding elapsed time? - Microsoft Community

make it so say:

Adding elapsed time?

I have an Excel workbook I use to track bicycle rides. I have the amount of time the ride lasted entered in the form of "mmm.ss". What I would like to do is figure out a way to add this up to get the amount of time I ride throughout the year up to the current day as an accumulated amount of time.

I do not have the starting and ending times of the rides, only the amount of time the ride lasted. I have tried to figure this out but have had no success.

Thanks in advance for any help.

joeu2004 say: Make It So wrote:
The cells are formated as number with 2 decimals.  I can change that if needed.  I cannot format the cells as time because Excel wants to turn it into a date/time which is not what I want.

In the long run, it might be best to correct this misunderstanding.  Then you can enter data in the normal time form, and the summation is straight-forward.

Format the cells or column as Custom [m]:ss.  Enter time in the form 0:m:ss.  For example, if your ride is 123m 59s, you can enter 0:123:59 [1].  Note that you must enter something for hours ("0:").  Otherwise, Excel will interpret 123:59 as h:mm.

Having done so, you can sum the range of ride times in A1:A20 simply with =SUM(A1:A20) [2].  Of course, the result should be formatted as Custom [m]:ss.

Yes, you might see time of day in the Formula Bar.  Simply ignore that.

Yes, you might see time of day or date/time in some cells after performing arithmetic on the range of ride times or after using some other functions.  Sometimes Excel goes overboard in trying to intuit what we want.  You might need to simply override Excel by (re)formatting the result as Custom [m]:ss.

Make It So wrote:
The cells are formated as number with 2 decimals.  I can change that if needed.

One way to convert your data already entered in the form m.ss to normal time is as follows.

Assume A1:A20 contains the data in the form m.ss.  Enter the following formula into B1 and copy down:

=time(0,int(A1),mod(A1,1)*100)

Format B1:B20 as Custom [m]:ss.  Then copy B1:B20 and paste-special value-and-number-formats into A1:A20.  You can now delete B1:B20.

-----
Endnotes:

[1] You can enter a maximum of 0:9999:59 in that form.  Since that is more than 22h 39m, I suspect it is good enough.  But if you must enter larger ride times, you will need to enter the time in the form h:m:ss; that is, where "h" is non-zero.

[2] For complicated reasons, it would be prudent to use the following formula to sum the ride times:

=--TEXT(SUM(A1:A20),"[h]:mm:ss")

Since time is stored as integer days plus a fraction of a day, the summation might not be exact due to the way that Excel stores numbers and performs arithmetic.  The --TEXT(...) expression ensures that the stored result is the same as if you had entered the constant, eliminating the artifacts arising during the summation.  It has the effect of rounding to the second.
