You are not required to use the Libraries.

The Libraries are simply a way to view other folders and the files in those folders.

This action does not affect the actual folders that are added to that library.

This problem occurs because the Recycle Bin does not recognize mounted volumes. For example, assume that the following drive layout is in use:

Drive C is a 1-gigabyte (1-GB) drive that uses the NTFS file system.
C:\Host-dir is the mount point for a separate 9-GB drive.

If you try to delete a folder with the path C:\Host-dir\Mydir, Windows Explorer interprets the folder as being on drive C and tries unsuccessfully to place it in the Recycle Bin on drive C.

Use the SHIFT+DELETE key combination. This bypasses the Recycle Bin.

I would suggest to try the following methods and check if it helps.

Method 1: I would suggest you to run the below command in elevated command prompt and see if it helps.

  1. Go to Start, click All Programs and open Accessories. 
  2. Right click on Command Prompt in the Program list and then select Run as administrator. If you are prompted for an administrator password or for confirmation, type your password, or click OK. 
  3. In the Command Prompt, type the following command, and then press ENTER: 
  • TAKEOWN /A /F C:

     4. Then try to run the next command.

  • TAKEOWN /F C:

This will give both admin and your account full permissions to ALL files in the C: drive

Method 2: Active hidden admin account

1.    Press Windows + R key on the keyboard.

2.    Type cmd in the run bar.

3.    In Command Prompt type this command: net user administrator /active:yes

4.    Press Enter and Restart the computer.

Try to delete the Window.old folder.

If you are able to do so deactivate the hidden admin account.

For that follow the step :

1.    Press Windows + R key on the keyboard.

2.    Type cmd in the run bar.

3.    In Command Prompt type this command: net user administrator /active:no

4.    Press Enter and Restart the computer.

r3d p01nt say:

Agreed with Lactos, it'll depend on your stats.  The battle axe scales C and D with STR and DEX, respectively.  If you change it to a Divine weapon, the scaling becomes D, E, and C for STR, DEX, and Fth.  It makes sense to go Divine if you're going to be leveling up your faith.  If you aren't leveling up your faith, the only reason to have a divine weapon is for the Catacombs (or the achievements).

.

Also, split damage is a complicated issue that isn't easily explained.  Generally, split damage has to outdo damage from one type by a bit before you'll break even with the damage you see on the enemy.  For example:

.

Let's say you've got a weapon that has an AR of 200 that is split 100 to physical and 100 to magic.  Both of those 100 need to get through the enemy's defense.  So, if the enemy has defense that stops 50 physical and 50 magic, so you end up doing 100 - 50 = 50 pts of physical damage and 100 - 50 = 50 pts. of magic damage and you'll see 100 total points of damage.

.

Now, let's say you switch to a weapon that has an AR of 150 that is just physical.  The enemy still has 50 physical damage protection, so you'll do 150 - 50 = 100 pts of physical damage.  The AR of 150 physical ends up doing just as well as a AR of 200 that is split evenly between physical and magic damage.

.

That's an overly simplistic example, but the general idea is that split damage needs to be higher in order to see the same output on enemies.

    To the best of my knowledge no printer has been sold that has dedicated support application for RT, if nothing else because RT will only run applications distributed through the store so cannot be bundled with the printer.

    RT will print to LAN based printers with no problem, the main feature that the dedicated driver can add is its ability to maintain connection with the printer. Most home networks use DHCP to assign addresses to devices which therefore can change. The software can reconnect when this happens.
