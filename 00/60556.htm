Windows 7 is not a fix for Windows Vista, its a brand new release with numerous features and functionality. Its an upgrade for Windows Vista, Windows XP and even Windows 2000.

You have to look back at the history of Windows Vista's development. Windows 'Longhorn' as it was originally known started out a very ambitious project as far back as June 2001. Microsoft had scoped the release for sometime in 2002 as a minor interim release that would build on Windows XP. Certain priorities at Microsoft changed and the Company decided to make it a major release incorporating major architectual changes: user experience, developer wise and under the hood. The release was expected to be released into beta summer 2003 with a final release for Summer 2004. Along the way, the Company discovered that building Windows Longhorn was more than they expected. Along the way, stuff happened such as malicious attacks against products such as XP on the market and priorities moved from Windows Longhorn to Windows XP and developing Service Pack 2. After that was completed in August 2004, the scope of the Longhorn project changed again and certain promised features such as WinFS was dropped to focus on delivering a product that would be released in a reasonable time frame. So, in August of 2004, Microsoft reset the Longhorn project.

Regardless of all these changes, Microsoft still managed to deliver a product that is feature rich and secure. What happened along the third party developer path was reluctance by third party ISVs (Independent Software Vendors) and IHVs (Independent Hardware Vendors). Instead of focusing on getting software and hardware drivers ready with the build of Windows Longhorn 5048 released at WinHEC 2005, the beta 1 release of Windows Vista released in August 2005, and the beta 2 release in May 2006 and the RC 1 released in September 2006 and the RC 2 released in October 2006. These ISVs and IHVs actually waited until January 31st 2007 to start getting their products ready. On top of what I previously said, OEMs released substandard products with the OS.

Think about it, some OEMs were releasing Vista on 512 MBs of RAM for a modern operating system thats ridiculous. 512 MBs would have been great for 2004, but come on, 2007? You have not listed your specs, I have been running Vista on a 3.2 GHz Intel Pentium 4, 2.6 GBs of RAM (upgraded), 128 MB AGP video card released in March of 2004 and the system runs beautifully. You have to understand that Vista utilized memory efficiently, instead of having it idle doing nothing, Vista actually uses all the memory by allocating it to desperate resources, so when you launch a program or do something in the system it just happens without you waiting. So memory management in the OS is different, but its not a flaw or something is wrong. As for application compatibility and UAC, Microsoft designed the system so that developers write better, more secure applications. UAC prevents malicious access to parts of the system that third party developers should not be writing to in the first place. So, Vista basically is helping to break bad habits by third party developers.

Vista enforced the policy of not using a Administrator account for day to day activities and help to keep the user informed about changes to their system, whether authorized or unauthorized.
