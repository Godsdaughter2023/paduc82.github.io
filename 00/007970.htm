rich007a say:

OK, I see what you mean now.  I don’t think you can change the chart data range without over-writing the series names in Excel 2007/10 (i.e. unless you have a simple chart data range that includes references to series names (column headings)) .  You have a few options (other than redefinining each series via the bottom half of the Select Data Source window), including:

a)     Move the chart into the same worksheet as the data, click on the chart or individual series and drag the coloured ranges down to the required cells.

 

or

 

b)    Make your chart dynamic by pointing it at dynamic named ranges.  To do this, set up the following three named ranges by going Formulas tab > Define Name:

Name:      Scope:           Refers to:
Cats        Workbook       =INDIRECT("Sheet1!A" & Sheet1!$F$1 & ":B" & Sheet1!$G$1)
Series1    Workbook       =INDIRECT("Sheet1!N" & Sheet1!$F$1 & ":N" & Sheet1!$G$1)
Series2    Workbook       =INDIRECT("Sheet1!P" & Sheet1!$F$1 & ":P" & Sheet1!$G$1)


replacing Sheet1!$F$1 and Sheet1!$G$1 with references to any two single cells where you can enter start and end row numbers for your plot - they could be on another sheet (and you can call the named ranges anything you like, as long as they don’t look like a cell reference, e.g. Ser1 could be confused with cell SER1, so is not permitted).

 

Now change the chart so the Axis label range points to =Sheet1!Cats

and the first series “Series values” points to =Sheet1!Series1

and the second series “Series values” points to =Sheet1!Series2

 

Now the plot should show data from between the rows specified in Sheet1!$F$1 and Sheet1!$G$1 (or whatever you changed them to in the named ranges).

 

You said your categories (x) axis contained dates.  If you were to put a date in cell E1 and put the following formula in F1, you could just enter a date in E1 to indicate where the plotted range should begin:

=MATCH(E1,A:A,0)

 

If the number of rows to be plotted is constant, you could put   =F1+48   in G1.

 

All of the above assumes your data is on a sheet called “Sheet1”.  Adjust the above accordingly if it isn’t.

 

Hope that helps.

anewdesign say:

how to auto-populate data based on previous dependent selections created via indirect

hello all,

i need help please! i'm trying to create a form in which a SKU will automatically populate in the appropriate cell based on the previous two selections made. these selections are made using dropdown lists, and the indirect function allowing multiple lists in one cell

Product Name Product Colour Product SKU

the Product Name is chosen from a dropdown list, and the Product Colour is selected from a list of available options for that particular product. this data is in a list format with named columns on a separate sheet, such as below:

Products Product1 Product2 Product3
Product1 Black Black Blue
Product2 Purple Green Yellow
Product3 Blue Yellow Red

where do i enter the SKU information for it to populate automatically based on my previous two selections? what formula do i use? should i change my entire approach?

any help is appreciated!

-tori

russcm say:

Hi Tori.

There are a few ways to do this, but here is one option...

Create a table with just the products in it and name the range lstProducts:

For the Product Colours and SKUs, list them in one table:

The Product column is named lstSKU_Product and the last two columns (C10:D18) are named tblSKU.

C10:C18 is a formula equalling A & B (=A10 & B10).

Now for the input options...

The validation list formula for C3 (the Product) is =lstProducts

The validation list formula for the Colour cell is more complicated.  It does the following:

  • Looks down the Products table (the one with the SKUs in it) and finds the first cell with the selected Product in it
  • Counts how many of that Product is in that list and selects that many rows from the first one
  • Shifts over one column so it's referring to the Colour options for that Product; this is the list you get

=if($C$3="","",OFFSET(lstSKU_Product,MATCH($C$3,lstSKU_Product,0)-1,1,COUNTIF(lstSKU_Product,$C$3),1))

Finally, the SKU cell is a simple VLOOKUP, looking at the last two columns in the SKU table.  It checks that there is something selected in both the Product and Colour cells first:

=IF(OR($C$3="",$D$3=""),"",VLOOKUP($C$3&$D$3,tblSKU,2,FALSE))

If you recreate this in a single sheet using the same cells and formulae as above, you can then test it and move the cells to where you want them after you know it works.  Alternatively, if everything above makes sense, you could recreate it from scratch.

This option means you can simply add to the Products and SKUs tables and don't need to worry about new named ranges every time you add more Products or Colours.

I hope this helps.

Cheers,

Russ.

making lists of documents by using categories - Microsoft Community

suemcp001 say:

making lists of documents by using categories

In earlier WORD programs, one use to be able to list a document by category, but not just so that one can search the category but so one can see many items all in one list, but within that list, by category. So, for instance, take a library shelf. You can see in front of you, in your own list, all the itmes in category M1003. as well as all the items in the category M1002 and in M1004.

jay freedman say:

As far as I'm aware, you can categorize documents in the same ways as you always could. (Since you haven't said what version of Word you're using now, it's hard to say what "earlier" versions you recall.)

In any version from 2010 onward, the file properties of the current document are shown in the File > Info page. (Word 2007 has an Office button > Prepare > Properties item instead, which displays a properties panel.) Two of those properties, Categories and Tags, are typically used for the purpose you're asking about, and you can also use the Comments property. You can type anything you want into those boxes.

If you want the property to be displayed (and editable) in the body of a document, you can use Insert > Quick Parts > Document Property and choose the property you want. That will insert a content control for the chosen property at the cursor position. Oddly, the property that's called Tags everywhere else is called Keywords here.

In Windows File Explorer, you can display the property you're using as an additional column in Details view. On the View tab, click the Add Columns button and choose which column to show. Then you can click the new column's heading to sort on that column, and all files with the same value will be grouped together.

Unfortunately, although the Open dialog in Word is provided by the same code as File Explorer, the dialog doesn't support additional columns.
