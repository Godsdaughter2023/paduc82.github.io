bradinminnesota say:

I know this is a pretty old thread, but was just working this one out for myself.

My final solution was to watch the Form_Current event.

During the Pasting process, Form_Current will get called for each insert, causing Me.NewRecord to be TRUE.

Any <field>_Afterupdate events will also get called.  I was trying to process some other records during the pasting process during an _Afterupdate, and because Access had a transaction going of pasting records into this table, I couldn't insert anything else into the table during the PASTE process.  However I could see the current ID of the newly inserted record during the AfterUpdate of the field.

When I realized that RecordsAffected was returning 0 for MY inserts, I assumed a PASTE was happening and set a boolean flag and started "remembering"  items to be inserted after the paste completed. (saving ID entries of the new records that were being inserted in an array)

my Form_Current Process (it looks something like this...)

    If Not Me.NewRecord Then
        If CurrentRecord >= 0 Then
            If PasteInProgress Then
                Me.TimerInterval = 1000     ' wait one second - allow paste to complete transaction.
            End If
        End If
    End If

When Me.NewRecord is no longer (True) and I see the PasteInProgress  flag is true, I wait 1 second for the pasting/transaction to complete.  In the Form_Timer sub, I call my post-paste-processing.

Below are the steps, 

Step 1: Run Apple mail and click on File> Import Mailboxes
Step 2: Select MBOX file where you want to export data, select File in MBOX format, and click on Continue
Step 3:  Browse the location where you have saved the MBOX file and click on choose and then click on Continue
Step 4: Click ‘Done’ when process is completed
In order to complete the import process, you need Eudora, as there is no direct feature available to import apple mail into MS Outlook.
Step 1: Download and Run Eudora
Step 2: Firstly, rename file extension of Inbox file form “xyz” to “xyz.mbx”
Step 3: Now move xyz.mbx file name to a directory in which Eudora stores its files 
(C:Documents and Settings<user>Application Data\Qualcomm\Eudora).
Step 4: Open Eudora and double click on xyz.mbx to create the file and close Eudora
Step 5: Now Import mailbox using Outlook Express
Step 6: Import mailbox from Outlook Express to MS Outlook 

Cautions:
