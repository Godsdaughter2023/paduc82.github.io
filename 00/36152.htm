patrick barker say: Hi,

There is really no such thing as 'normal' CPU temperature, as all systems differ, especially if they are a laptop or a desktop. 50C is a bit high, but it's not absolutely horrible. Idle temperatures generally aren't important, and load temperatures (3D application, full screen game, etc) are what determines whether or not your system is close to reaching unsafe temperatures for operation.

Regardless of whether it's a laptop or desktop, clean all of the vents and fans of any dust, etc. If it's a laptop, consider purchasing a cooling pad. Also, remember, as hardware ages, the thermal paste on the components such as the CPU and the GPU get worn out. If you have a desktop past warranty and you'd like to change the thermal paste yourself, it's very simple as long as the manufacturer didn't make it impossible to remove the CPU or video card heatsinks manually.

BSOD are notoriously difficult to troubleshoot, even if the crash dumps get analysed.

- Does the machine BSOD with all games ?

Do the BSOD happen shortly after starting the games, or always after a similar amount of time, say 20min or so ?

- What are the cpu and gpu temps under load, or at the time of the BSOD ?

- Are you using a High Performance Windows power plan ?

- Are you using the current driver from Nvidia, or have you let Windows Updates take charge ?

- By re-setting, do you mean you reinstalled Windows, or did a roll back to a prior version, eg- 1709>1703 ?

- BSOD can be caused by software conflicts. Anti-Virus. Programs such as  Skype, and system 'optimising' programs frequently

conflict with games. The 'Gaming' settings in Win10 1709 can also cause problems.

- You can troubleshoot Services and Startup programs that may be conflicting by running in Clean Boot

(this does not cover programs you may launch manually after booting into Windows though).

How to troubleshoot a problem by performing a clean boot 

- No overclocking. So the RAM settings in BIOS are at default too ?Did the problem start when you added more RAM ?

Or has your machine always had an odd configuration ?

If you have two 8GB sticks try pulling the extra 4GB to verify this is not leading to instability.

If using three sticks, it will force the RAM to run in single channel mode vs the more efficient dual channel mode.

Using a combo of 4 sticks of unequal capacity can also cause issues, such as not using the full capacity of the

higher capacity sticks, and it also may not run in dual channel mode.

- No RAM mfg will guarantee stability if you use unmatched RAM.

the RAM is not causing the BSOD.

If you must multi-task, a matched kit of 4X8GB or 2X16GB would be a safer, better performing configuration.

imran chand say:

Hi squibby25,
Windows Media Player cannot play the file. The Player might not support the file type or might not support the codec that was used to compress the file. You might encounter this error message for one of the following reasons:

1. The file type is supported by the Player, but the file was compressed by using a codec that is not supported by the Player.

2. The file might have been encoded by using a codec that is not supported by Windows Media Player. If this is the case, Windows Media Player cannot play the file.

Step1:
Uninstall third party codec’s installed on the computer.
Uninstall or change a program:
http://windows.microsoft.com/en-us/windows-vista/Uninstall-or-change-a-program

Step 2:
Try playing the file on a different computer and check if the file is not corrupt.

Step 3:
Try disabling the audio enhancements and check if that helps.

a. Open Sound by clicking the Start button, and then clicking Control Panel. In the search box, type sound, and then click Sound.

b. On the Playback tab, click Speakers, and then click Properties.

c. Click the Enhancements tab, and then select the Disable all enhancements check box.

If you don't see an Enhancements tab in the Speakers Properties dialog box, those features are not available for your speakers and this procedure is not applicable to your computer.

If disabling audio enhancements solves the audio problem you're experiencing, this typically indicates that your audio driver needs to be updated.

Step 4:
In some cases people have run into this error when the wrong audio device is selected in Windows Media Player. 
a. Open Windows Media Player.
b. Click on Tools, select Options.
c. Click on Devices, and then select Speakers.
d. Now, select Properties, it will allow you to change it.

Here is some copy/paste about Windows Update and hardware driver from ElderN:

It is generally not a good idea to accept hardware driver updates (network, audio, video) offered by Windows Update and if you ever see them it is best to hide them so you never see them again.  Only get hardware driver updates from your hardware manufacturers WWW site.

Just because Microsoft offers you a driver update does not mean you have an out of date or afflicted driver and it certainly doesn't mean you have to install the driver updates MS offers - it is generally best NOT to install hardware driver updates offered by Microsoft.

Accepting hardware driver updates from Microsoft is one reason there are so many topics in the community that start with "Windows update broke my computer...".

One rule of thumb is "if it ain't broke don't fix it" so if you are not having a problem with video, audio or network don't accept or install those driver updates offered from Microsoft.  If you are having a problem like that you should only get driver updates from the hardware manufacturer.
