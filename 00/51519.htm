swordpal say:

svchost.exe is a legit system file; however, you must be sure that your "svchost" is in the correct folder (x:\windows\system32) and digitally signed by Microsoft. Many impostors have names similar to "svchost".

Find the process ID of the svchost which is reading your files, then go into cmd and type tasklist /svc. Find the PID. If it has no "Services", then this "svchost" is a impostor.

Normally, if you have turned indexing service off, it shouldn't search through your disk.
