suzanne s. barnhill say:
No, once the text is part of the document, it is not possible to determine its origin. If you are in the UK, however, and have the default language set to "English (U.K.)," then any text formatted as "English (U.S.)" has very likely been pasted in.

akashpatel89 say: Hi Suzanne,

suzanne s. barnhill say:

You don't even have to "launder" it through Notepad; you can just use Paste Unformatted. So the bottom line is that no, there's no way to tell.

 

It's risky to base any judgments on metadata because they can be stripped, altered by using Save As, etc. But if you're looking at a document created by a relatively unsophisticated user who has not made any attempt at concealment, the "Total editing time," if suspiciously small for the length and complexity of the document, might be a clue.

fumei2 say:

And even Total editing time is wishy-washy for anything significant as it just returns the time a document is open. If you open a document and do nothing for an hour, an hour is added to editing time. The bottom line is if the question is about whether someone has sneaked in text, Suzanne has answered this: you can not reliably tell.

Hi All,

Cause:

We've determined that incorrect Message Application Programming Interface (MAPI) values within users' Outlook on the web configurations caused impact to signature functionality. We provided a workaround to manually reset the affected MAPI configurations, and received confirmation of resolution.

Workaround:

You can try the following steps to reset OWA User configuration:

1. Download MFCMapi tool: http://mfcmapi.codeplex.com/

2. Before performing any step, please make sure that Outlook client is not running on that machine and user is not connected to OWA.

3. Open MFCMapi on the machine used by the user â we will need the Outlook profile to open the store with this tool.

4. Go to Tools -> Options -> and tick the boxes with Use the MDB_ONLINE flag and Use the MAPI_NO_CACHE flag to make sure that the task is done on the store, and not on the local OST file.

5. Then go to the main menu Session -> Logonâ¦ -> and chose the Outlook profile used by user

6. Select the store for user's mailbox from the list and double-click to open the store. On the new page that is open, right click on the Root Container and chose Open associated contents table

7. On the Hidden Contents table tab, arrange the columns header and scroll to the right until you can see the Message Class details for each entry.

8. Right Click on the Message Class -> Delete message -> Permanent deletion (delete to deleted items retention if supported) and hit OK (For entries bellow)

9. Please repeat the above deletion steps for the below 3 entries:

  • IPM.Configuration.Suite.Storage,
  • IPM.Configuration.OWA.UserOptions,
  • IPM.Configuration.Agregated.OWAUserConfiguration

10. Once the above steps were performed, close the MFCMapi using the X button on the right top corner â no need for specific logoff.

11. Ask user to login to OWA and test if OWA options can be changed.
