Re:  count stuff when cells are merged

Here is some code that works on my dummy data.
You must select the data area first
(only the first column is actually req'd to be selected)
Use Option Compare Text at the top of the module if you want to include lower case "PM"

---
Sub HowManyAreThere()
'Jim Cone - Portland, Oregon
 Dim Rng      As Range
 Dim rCell     As Range
 Dim subCell As Range
 Dim lngQty  As Long
 Dim N         As Long
 Const Alphas As String = "PM*"

 Set Rng = Selection

 For Each rCell In Rng.Columns(1).Cells
  If Len(rCell) And Not rCell.MergeCells Then
    If rCell.Offset(0, 5).Value Like Alphas Then rCell.Offset(0, 11).Value2 = 1
  Else
    If Len(rCell) And rCell.MergeCells Then
      N = rCell.MergeArea.Rows.Count
      For Each subCell In rCell.Offset(0, 5).Resize(N, 1).Cells
         If subCell.Value Like Alphas Then lngQty = lngQty + 1
      Next
      rCell.Offset(0, 11).Value2 = lngQty
      lngQty = 0
    End If
  End If
 Next
End Sub

hansv mvp say:

If you set the Enter Key Beahvior property of a text box to Default, the user has to press Ctrl+Enter to start a new line; just pressing Enter will take the user to the next control.

To suppress Ctrl+Enter, you can use the On KeyDown event of the text box:

Private Sub LastName_KeyDown(KeyCode As Integer, Shift As Integer)
    ' Did the user press Enter and hold down the Ctrl key?
    If KeyCode = vbKeyReturn And (Shift And acCtrlMask) Then
        ' Setting KeyCode to 0 cancels it
        KeyCode = 0
    End If
End Sub

In this example, LastName is the name of the text box.

Make sure that the property "Enter Key Behavior" (on the Other tab of each
text box's property sheet in design view) is set to "Default", not "New Line
in Field".  Normally, when creating a bound text box, Access will use the
"Default" value for text boxes bound to a Text field, and the :New Line in
Field" value for text boxes that are bound to a Memo field.  But you can
change it after the text box is created.  For convenience, you can select
several text boxes all at once by shift-clicking them, open their join
property sheet, and change the Enter Key Behavior for them all at once.
 
That will still allow people to enter a new line in the text box by pressing
Ctrl+Enter.  If you have to, you can suppress that entry using the KeyDown
event (of the form or of a specific text box), or by removing any new-line
combinations (CR+LF)  in the control's AfterUpdate event.
