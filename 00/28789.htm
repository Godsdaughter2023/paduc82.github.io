Hello Dave Hopgood,

Regarding your description about “Teams file expiration “, By design, deletion policies for Teams files are configured through SharePoint Online and OneDrive for Business locations. As a result, it's possible that a policy could delete a file referenced in a Teams chat or channel message before those messages get deleted. In this case, the file will still show up in the Teams message, but if you click the file, you'll get a "File not found" error (this could also happen in the absence of a policy, if someone manually deletes a file from SharePoint Online or OneDrive for Business).

 

So we would like to suggest you to check if any retention policy is configure for Teams, SharePoint and OneDrive and check the action settings.

  • To do so:  Login with Global Admin account in O365 : Admin Center -> Security and compliance-> Data governance-> Retention-> check Retention policy and actions and if your found any related policy with the delete option is enable , suggest you to modify the settings.

I am not sure that you can do this with Word Online.  However with Word on a desktop, you would use an If...then...Else field such as

{ IF { MERGEFIELD Title } <> "" "{ MERGEFIELD Title } " "" }

Note, that after the second { MERGEFIELD Title } there is a space before the closing quote.  Also note that you would NOT have a space between the above field construction and the next Mergefield.
