ken sheridan say: Did you try what I suggested in my last reply?  A check box which is bound to a Boolean (Yes/No) column, which I assume these are, can only have one of two values TRUE or FALSE.  The only time it doesn't is when you first navigate to an empty record in a form, when it is Null, but as soon as data is inserted in any bound control in the form the check boxes will have a value of FALSE.  When you reference the control by name you are really referencing its Value property, which is the default property, so a TRUE or FALSE value will be returned.  Consequently you don't need to examine the equality of the control's value to anything, simply reference it by name in the way I described.

The line of code as a whole, by virtue of the multiple Boolean AND operations, will only evaluate to TRUE if all of the expressions evaluate to TRUE.  If the check boxes are both checked, but the line as a whole evaluates to FALSE, then this would point to one of the other expressions evaluating to FALSE, not the check boxes' values.

Have you debugged the code?  If not, set a breakpoint on the line and when code execution breaks on the line examine the values of the various controls by hovering over them with the mouse pointer.

One thing I would query about your code is the use of the Nz function.  This would only be necessary if the columns in question all zero-length strings.  Is this the case, and if so, why?  It is more usual to disallow zero-length strings and Nulls (the latter by setting the column's Required property to True) and test for a bound control being Null.  In the event of a control then being Null this would also raise a data error of course when the use attempts to save the record, which can be handled in the form's Error event procedure.

Renumbering lines in Microsoft Project

There are line numbers to the left of each task on my microsoft project plan.  As I cut and paste tasks, I end up with the numbers out of sequence.  I would like to renumber the tasks so the numbering starts with 1 and increments sequentially by 1.  How can I accomplish this without resequencing the order of the tasks?  For example, if the line numbers are 1, 2, 3, 635, 45, 4, 5; I would like to renumber it to 1, 2, 3, 4, 5, 6, 7, etc.
julie sheets say:
Dan,

David is using Project 2007, but you note you are using Project 2010. 
So, let's start over again.

You have Project 2010 -- do you have SP-1 installed?

You note problems with inserting new work packages (tasks) and sorting.  When you sorted the schedule, did you permanently renumber the tasks? 

If you simply sorted the project file, but did not renumber the tasks, the IDs will not automatically renumber -- they will appear out of order in both the view as well as print preview.

If you want to sort the schedule and have the task IDs change to reflect the new schedule, choose Sort By and select to "permanently renumber tasks."

Please also note -- I do not work for Microsoft.  Like most all posters here, I am a volunteer.
