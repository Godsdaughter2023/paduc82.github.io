If you’d like to manage your Outlook account using an email app, you’ll need to set up your POP3 or SMTP server and port settings in your email app. You’ll need the following info:

    POP3 server: pop3.live.com (port 995)
    SMTP server: smtp.live.com (port 25)


Choose the Secure Sockets Layer (SSL) option for both POP3 and SMTP connections.
Choose the SMTP authentication for SMTP connections.
Choose the outgoing server requires authentication option. (In most email apps, this option isn't checked by default.)

    User name: Enter your full email address.
    Password: Enter your Microsoft account password.
    If you don’t want email messages to be permanently deleted from your inbox after you download them to your mail program, select the Leave a copy of messages on the server option.

Prevent losing email in Outlook.com when you've set up POP3

Outlook.com has a feature that prevents losing email messages if you accidentally set up POP3 to delete them after reading them on the server. Here’s how it works:

    The first time that Outlook.com receives a POP3 instruction to delete email messages after reading from an email app, Outlook.com won't delete the messages. Instead, Outlook.com moves the email messages that were successfully downloaded by the email app into a POP folder and out of the inbox. The POP folder is available in the Folders list.

    Outlook.com keeps the email messages in the POP folder until you sign in to your Outlook.com account and verify whether Outlook.com should respect or ignore the POP3 delete command.

To verify your POP3 settings:

    In the upper-right of the Outlook.com window, click Options, and then click More options.
    Under Managing Your Account, click Pop and deleting downloaded messages, and then select one of the following:
   
Don’t let another program delete messages from Outlook.com. (If your other program is set to delete messages from the server, we’ll simply move them to a special POP folder. They won’t be deleted.)

Do what my other program says—if it says to delete messages, then delete them.

    Click Save.

Alternative to a Fix -- Restoring Previous Version of Excel 2016 for Mac

After much fiddling, along with waiting for Microsoft Insider Slow Builds that might fix the problem, I looked into restoring the previous version of Excel, and was able to do that.  The problem is now "fixed" by running Excel 15.41.0.

If your interested in restoring a previous version here are the steps --

1. Go to https://support.office.com/en-us/article/update-history-for-office-2016-for-mac-700cab62-0d67-4f23-947b-3686cb1a8eb7#bkmk_history

2. Click on Release History for Mac.

3. Click on the Word "Excel", not Download (unless you wish to also restore the rest of the Office Suite to the same older version).


4. Before installing the Download Excel app, to to the Mac Applications Folder, find the Excel App, and drag it to the trash.  Provide an authentication password to delete the app.

5. Run the Excel installer in your Downloads folder.

6. Open Excel.  You will get the error message "We ran into a problem while accessing the Office Document Cache. Excel needs to be restarted."  Click OK.

7. Start Excel again.  When it says it needs to clear the document cache let it do so.  The process seems to get stuck, so use Force Quit (CMD/Option/Esc) to force quite Excel.

8. Start Excel again.  It works fine this time and you're back in 15.41.0 land.

There may be a more elegant way to deal with Step 7 above, but I couldn't find it in two tries.

OneDrive for Business NGSC – Sync Stuck in updating changes - Microsoft Community

manuela rusu-hack say:

OneDrive for Business NGSC – Sync Stuck in updating changes

   

 Technical Level : Intermediate

1. General scenario

Updates are not being synced

2. New local files are not uploaded by the sync engine

In this case, the sync from the site down is working fine, but when the user is creating a new file, it will never get uploaded to the online library.
User sees the OneDrive sync client get into a Process Changes cycle. Generally all of the users of a tenant will be seeing this behavior with their sync clients.

Resolution

1. For general scenario

  • Check for any sync errors.
    • SaRa Tool for OneDrive for Business could help you identify possible sync errors. Watch how-to use SaRa video HERE

  • Compare local sync folder with website to determine if there are differences in file counts
  • Copy the document that is stuck, to the Desktop
  • Delete it from the sync folder
  • Once the sync is up-to-date, copy that file from the Desktop back to the sync folder

    2. For the scenario where the files are not uploaded (Admins)

    Run the command (in SPO Management Shell):

    Set-SPOTenant -LegacyAuthProtocolsEnabled $true

    This will allow that other types of authentication (not only ADAL) will be accepted and could help you fix the issue.

    A value of False-Prevents Office clients using non-modern authentication protocols from accessing SharePoint Online resources

        *See Set-SPOTenant article for more details on the command

    NOTE: If the above steps are not solving the issue another possible cause could be the number of items that are synchronized.

    If the above steps are not solving the issue another possible cause could be the number of items that are synchronized.

    “Although SharePoint Online can store 30 million documents per library, the sync performance of OneDrive may start to decline when you store more than 100,000 files in a single OneDrive for Business site or team site library. To resolve this limitation, ensure that files are stored in multiple folders/libraries. If you have more than 100,000 files in a OneDrive for Business site, you may need to wait an extended period of time with OneDrive continuously syncing before sync can complete.”

       *See Restrictions and limitations when you sync files and folders for more details.
