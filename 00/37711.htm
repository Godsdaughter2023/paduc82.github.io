Hi Nanc...
I got your problem and i study the link you given let me little exlain what is transparency in PNG file.
Each pixel in the true-color image with alpha is described by four bytes: RGBA. The last one is Alpha, which controls pixel transparency: the value of 0 means fully transparent pixel and 255 means fully opaque. And this means that every pixel (with any RGB value) can be hidden with just Alpha byte set to 0. But this RGB data still exists and, moreover, it prevents PNG encoder from effectively packing and encoding the data stream. Thus, we have to remove this hidden data (fill it with solid black, for example) before saving the image for good compression.finally if transparent pixel,s RGB data is set to 0 or Black than you can save data or memory.
As i studied on the link given by you this problem comes with chrome only and in Internet Explorer its fine Right?So while file compression in Chrome where the Alpha value of PNG image is zero (fully transparent ) and RGB value set to zero for compression . When you download from chrome if you not select the right resource the file is saved as Image.PNG format without having alpha value (RGB only) and when it comes to windows explorer all the alpha value set to 255 (fully opaque). And when you see these file on windows explorer the image background seems black due to hidden black portion is now Opaque.
To Solve this problem you need Any PNG Editor to edit the alpha value set to Zero(Transparent ),and problem solve. you can edit the image online too by typing PNG image editor on Google.
