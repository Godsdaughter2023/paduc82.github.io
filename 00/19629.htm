This means you installed Office 2010 using Click-to-Run.  Click-to-Run is based on Application Virtualization technology. The Q:\ drive is the virtual file system drive where virtualized apps "live" in the file system namespace. The Q:\ drive is not a real drive; it has no space and is not accessible from Windows Explorer or “My Computer”.

 

Any access to the Q: drive, including hiding or deleting it, has been removed.

Today, after opening Menu » Store, I faced the same error message on my Nokia 207. My workaround was to go for

Nokia » Menu » Settings » Date » 01-01-2016.

That is required because the code-signing certificate of the app Store 3.34.5.52 expired on 2nd January 2016. Nobody renewed that certificate. Why ever. After changing the phone’s date, you still get two certificate warnings, because Opera replaced the certificate for the web server on 11th October 2016. However, because those warnings are not about code-signing certificates but a TLS certificate warning, you are able to continue. After the Store installed, I changed the date back to today. Now, when I open Menu » Store, the platform browser is started and the Opera Mobile Store is shown.

If that does not help and you still get the very same error message, I recommend to apply the Recovery Tool (Windows computer required). You are using a quite old firmware. Furthermore, the file system of your phone might be broken and the required certificate authority ‘Nokia Online CA’ might be missing therefore. That cannot be fixed by resetting  via the phone menu. To repair that, you have to use a computer.

karl timmermans (outlook mvp 2012-2018) say:
>>
#1 - What is the best way to synchronize Outlook between the two computers?

#2 - Put my PST file (about 2.3 GB) in the cloud and making sure the custom forms work?
<<

To take the two primary questions out of sequence....

Re: #2 - To make sure that the custom forms work on both computers, the forms should be published to the appropriate folders on each computer, You don't want to keep an actively PST file in the cloud where "active" means that it is associated with an Outlook email account that is open in Outlook. It's not supported and cause data corruption.

Note: backing up a PST file to a cloud drive is different then having Outlook configured to use the PST file that's located in a cloud location. Also, when backing up the PST file to any location (local, NAS, or cloud), Outlook should always be closed otherwise you may end up with a corrupted PST file.

Re: #1 - Sync'ing Outlook between 2 computers. Since you mention a PST file, the assumption is that this involves a POP3 account - is that correct? (Not sure - go to File --> Account Settings --. Email tab).

Assuming that this does in fact involve a POP3 account then it comes down to how often you travel or to put it another way, frequency, you work on the desktop as compared to being on the road. Depending on the answer to that, the simplest solution would be to just copy the PST file from one computer to another as the need arises (i.e. from desktop to laptop when you are set to travel and vice versa when you return.

If on the other hand you want to constantly have these computers in sync, then your only option would be to use a 3rd party "sync" tool such as "Sync2" (from the 4team folks) or "Companionlink" since POP3 accounts don't sync data with the server.

An alternative approach would be to configure the email account(s) being used using IMAP which syncs email folders with the server but this doesn't include contact/calendar data. For that data (contact/calendar), you could create a free Outlook.com account, store your contact/calendar data in your Outlook.com account have it configured on each computer.
