greg carmack - windows mvp 2010-2020 say:
Hey UR. I'm Greg, an installation specialist and 8 year Windows MVP, here to help you.

It should have combined the Unallocated. That it didn't indicates some type of corruption.

Open an elevated Command Prompt, type each of these followed by Enter key:

Diskpart
List disk
sel disk # (replace the # with the disk number for the hard drive in question)
Clean
Exit

If it works it will remove the partition table and any boot level code so all that's left is Unallocated Space. You can then create a new NTFS partition in Disk Mgmt.

I hope this helps. Feel free to ask back any questions and let us know how it goes. I will keep working with you until it's resolved.
