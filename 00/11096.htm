Option Explicit
Sub BoldUnicodeUcase()
    Dim rg As Range, c As Range
Set rg = Range("a1", Cells(Rows.Count, "A").End(xlUp))

For Each c In rg
    With c
        If LeftB(.Text, 2) = UCase(LeftB(.Text, 2)) And _
          Not IsNumeric(.Text) And _
          Len(LeftB(.Text, 2)) > 0 Then
            .Interior.Color = vbYellow

    End With

I tested TasosK's macro using the Wingdings2 font set to simulate a non-english font set (not absolutely certain this is a valid test, but I think it is) and it worked well so long as the first character of the text in column A is a letter.  It also shades cells yellow if you enter numbers and non-alpha characters.

But I'd recommend modifying his code a bit to just work with the current selection in column A after using his current code one time to 'seed' the shadings in existing entries.  Set up his original code, make an entry or change to an entry in column A and then edit the macro so it looks like this (simply copy this and paste over the old)

Keep in mind this is WORKSHEET event processing code and needs to go into the worksheet's code module.  Easy to get to that: right-click on the worksheets' name tab and choose View Code from the list that popsup and you'll be taken to the point that you need to insert both is original code or replace his code with this later.

Private Sub Worksheet_Change(ByVal Target As Range)
'based on TasosK code presented previously
Dim r As Long
Dim i As Integer

If Target.Column <> Range("A1").Column Then
  Exit Sub ' no change in column A, just quit.
End If
If Target.Cells.Count > 1 Then
  'they may have deleted several entries so rework them all
  r = Cells(Rows.Count, "A").End(xlUp).Row

  Target.Interior.ColorIndex = 0
  If Not IsEmpty(Target) Then

    End If
  End If
End If
End Sub

Sub ExtractReps()
  Dim ws1 As Worksheet
  Dim wsNew As Worksheet
  Dim rng As Range
  Dim c As Range
 
  Set ws1 = Sheets("Sheet1")
  Set rng = Range("Database")
 
  'Screen off (enable after testing)
  'Application.ScreenUpdating = False

  'extract a list of Sales Reps
  ws1.Columns("C:C").Copy _
    Destination:=Range("L1")
  ws1.Columns("L:L").AdvancedFilter _
    Action:=xlFilterCopy, _
    CopyToRange:=Range("J1"), Unique:=True

  'set up Criteria Area
  Range("L1").Value = Range("C1").Value

  For Each c In Range("J2", Range("J" & Rows.Count).End(xlUp))
    'add the rep name to the criteria area
    ws1.Range("L2").Value = c.Value
    'add new sheet (if required)
    If Not WksExists(c.Value) Then
      Set wsNew = Sheets.Add(After:=Sheets(Sheets.Count))
      wsNew.Name = c.Value
      'Paste formats from source sheet
      ws1.Cells.Copy
      wsNew.Cells.PasteSpecial xlPasteFormats
      Application.CutCopyMode = False
    Else
      Set wsNew = Sheets(c.Value)
      wsNew.Cells.ClearContents
    End If
    'run advanced filter
    rng.AdvancedFilter Action:=xlFilterCopy, _
      CriteriaRange:=ws1.Range("L1:L2"), _
      CopyToRange:=wsNew.Range("A1"), _
      Unique:=False
  Next
  ws1.Select
  ws1.Columns("J:L").Delete
 
  Application.ScreenUpdating = True
End Sub

marshall barton say:

2013-2-29 is not a valid date and should generate some kind of error. That means you can not use that as a To (or From) date.  Without taking the whole leap year issue into account in the criteria, I can't think of a way to do what you asked.

 

If you always want to get whole months, enter both From and To as 2/2013 (without the day of month) to get just Febuary and change the criteria to:

WHERE IOT.idate >= DateSerial(Year([Date From]) - 1, Month([Date From], 1)

    AND IOT.idate < DateSerial(Year([Date To]) - 1, Month([Date To] + 1, 0)
