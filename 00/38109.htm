Thanks Geoffrey, this worked for me.


Maybe this will help some others out:


1) Backup - I went to my desktop pc (which was syncing fine) and copied all of the files from the SkyDrive sync folder to my backup drive. This is good practice occasionally anyway. I didn't end up needing the backup as I didn't lose anything through this process.


2) Close SkyDrive - I went to the task manager and just ended the task. Closing SkyDrive didn't shut it all the way down.


3) Delete the SkyDrive folder - I have this in the default location on my tablet: C: - Users - Rich(my username) - SkyDrive.      Deleted this after backing up the data above. Or alternatively just rename this folder to "SkyDrive - old" and a new "SkyDrive" folder will be created automatically. In essence this is the same as backing up the file as files from the web version of SkyDrive will populate the new file.


4) I found the local app data under C: - Users - Rich(my username) - Microsoft - SkyDrive - settings


When I started back up things synced fine. I suppose that deleting the settings makes the computer look new to SkyDrive so it pulls from the internet as opposed to clearing the internet to match the PC.


I waited a few days before I posted this to make sure that I didn't revert to having issues, but none so far.


My guess is that during the transition from Windows 8 to 8.1 some "extra" settings got stuck in the app data file. I looked again just now and there are far fewer files in that folder than before I did this process.


I hope this helps others out as well. I'm up and running now and the process took me about 5 minutes. I then did the process on my desktop as well just in case.


Rich


Hey Rich, thanks for translating "geek-speak" to real English so other users (like myself) can understand. I definitely agree with you that the problem originates with upgrading from Win 8 to 8.1.   1. Making a backup of your files is important, but difficult since the files on File Explorer cannot be read properly and give the 0x80040A41 error message. I was lucky enough to have another Win 7 PC with SkyDrive and made a backup copy that way.
  2. Make sure to close both SkyDrive and the SkyDrive syncing host from Task Manager.
  3. This one was the most frustrating steps for me, and I tried it several times...but every time I renamed "SkyDrive" to "SkyDrive Old", Windows would rename it back again to "SkyDrive". It was only when I deleted the SkyDrive folder that I was successful.
  4. This was straightforward, no problems.

The files "synced" in just a few minutes and I now have both File Explorer and the SkyDrive 8.1 app working properly. I will post an update if I have any problems in the future.

andre da costa say:
If you receive the following error when trying to setup BitLocker, it likely means your computer does not support the Trusted Platform Module (1.2) chip.

You can work around the error by launching Group Policy then make an exception. Press Windows key+R then type: gpedit.msc then hit Enter on your keyboard. Expand Administrative Templates then Windows Components then Bitlocker Drive Encryption then click Operating System Drives.

Right-click Require additional authentication at startup then click Edit.

Choose Enabled and then check the box to allow BitLocker without compatible TPM in the Options section.

For more on this workaround, read : How to Enable BitLocker On PCs Without TPM.
