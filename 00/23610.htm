christinakimm say:

Word 2013 Cumulative Word Counts on each line

How do I add the column of the cumulative word counts on each line in the Word document? See the inserted file.
jay freedman say:

It can be done, but the first step is the hardest. The key is to turn the text into a table where each line is in a separate row. Unfortunately, Word doesn't give us a tool to do that without a lot of  manual work. Here's the quickest way I know:

  • Select all the text, go to the Paragraph dialog, and set the right indent to the width you want for the column of numbers plus a little bit, say 1.2".
  • Go down the ends of the lines, pressing Enter at the end of each line that doesn't already have a paragraph mark (turn on the Â¶ button so you can see the marks). This is the long and boring step!
  • Select all the text again.
  • Click Insert > Table > Convert Text to Table. In the dialog, set the Fixed column width box to a little more than the width of the text, say 5.5". Click OK.
  • Right-click the table and click Table Properties. Click the Options button in the dialog; in the Options dialog, clear the check box for "Automatically resize to fit contents". Click OK, OK,
  • Right-click the table and click Insert > Columns to the right. A new second column appears, but it's the same width as the first column. Now go to the Table Tools } Layout tab, and in the Cell Size group set the column width to the right size, say

    Now you're ready to fill in the cumulative word counts. Install and run this macro (if the table isn't the first one in the document, adjust the number in ActiveDocument.Tables(...) to the correct one).

    Sub RowWordCounts()

        Dim tbl As Table

        Dim rw As Row

        Dim count As Long

        Dim rg As Range

        

        Set tbl = ActiveDocument.Tables(1)

        count = 0

        

        For Each rw In tbl.Rows

            Set rg = rw.Cells(1).Range

            rg.MoveEnd wdCharacter, -1

            count = count + rg.ComputeStatistics(wdStatisticWords)

            rw.Cells(2).Range.Text = count

        Next rw

    End Sub

    dianeoforegon say: Sounds like database corruption. Power outages are known to corrupt all sorts of data and hardware.

    Rebuild Basics:

    Do Not Rebuild Identities with Exchange and IMAP accounts. Delete the accounts (this deletes the cache files in Outlook). Do the rebuild then add back accounts. Better yet start over in a new Identity. If the rebuild does not work then you could sync unwanted changes or deletions to your server and lose data.
    Make sure you have plenty of free space on your drive. Make sure that you have at least three times as much hard disk space available as the current size of your Outlook identity PLUS 10% free for virtual memory.
    Rebuilding does not fix all issues. It can make some problems worse. Even though the Database Utility will make a backup, I advise making a copy in the Finder first.
    You can delete the rebuild version after you are satisfied with the results. This copy will be named like this: Main Identity [Backed up 1-21-2012 13.35]  

    Tip: If your Identity will open, before you rebuild, check "On My Computer"Inbox and Sent folders to be sure they are under the 2GB limit. Generally this is less than 10,000 messages but with large attachments could be less. Make subfolders e.g. Inbox 2011, Inbox 2010, Sent 2011, Sent 2010 etc to help reduce the number of messages in a folder. (Check for any large archive folders you might have in addition to Inbox and Sent).

    If rebuilding fails.... I suggest you move to a new Identity. Follow directions here:

    How to move your data to a new Identity
