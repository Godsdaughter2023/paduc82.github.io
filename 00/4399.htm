Shifting the waterfall timeline ?? - Updating the project

I have a waterfall timeline in MS project that I planned a while ago. Now I have all the details of which of theses tasks have been completed and which havent yet (100%, 50%, 0%). Some are completed who are still scheduled far into the future yet we were able to finish them early. Now I want to shift this waterfall in a way that the task that has just been finished is right before todays date and the task that is to come next is right after todays date. So shift the entire group of tasks but keep the structure. Then, once I have done that, I would like to recalculate (taking into account resource limits) from today into the future, and give me a new end date.

john - project say:

CH_MAPLAN,

The best way to show completion of any task, current or future, is to make an entry into the Actual Start and Actual Finish fields. Since Project is interactive, entries in those fields will tell Project what really happened and Project will adjust the current schedule (i.e. Start and Finish fields), to reflect reality. Therefore, future tasks will be moved back to the actual dates.

As far as current tasks that have not started but should have, there are various ways to slip the schedule. One method is to adjust the start date, but that should be taken care of by any predecessor tasks and manually adjusting the start date will set a constraint, so that may not be the best option. Another method is to set the calculation option under File/Options/Advanced group to have Project adjust tasks based on the Status Date.

By the way, I assume you set a baseline. If you did not, then updating tasks as outlined above will change the schedule and you will have no way to compare the current schedule with the original schedule.

Hope this helps.

John

rod gill say:

As John says, best way to update is to enter Actual Start, Actual Duration and Remaining duration. However if you only have % complete then:

  1. File, Options, Advanced
  2. Scroll to bottom of form
  3. Select the first 3 Move calculation options
  4. Now when you set % complete tasks move automatically so all actual work is in the past.
  5. Select Project, Update Project
  6. Select reschedule uncompleted work
  7. For entire project
  8. OK

That should do it. Good luck.

trevor rabey say:

You can do the whole thing in one go as described by Rod. If I did this, afterwards I would reset those three calculate options back to the default (off).

Although updating all at once is a quick way to improve everything, you may prefer to check and update the status task by task. This is because the tasks you know about because you have some progress data may re-schedule their successors automatically when updated.

If you have a task which was scheduled way in the future by predecessors, but the task has started and finished even though the predecessors have not, it may be a good idea to decide that the predecessor link should be removed.

Once a task has an actual start date, its predecessors, right or wrong, are irrelevant.
