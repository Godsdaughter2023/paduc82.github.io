Follow the below steps

  2.  Search for ""Sequence Flow" shape and drop this shape on the page
  3.  Open Document Stencil by clicking  on Developer --> Document stencil checkbox
  4. In the Document stencil you can see "Dynamic Connector" and "Sequence Flow" masters

From now on if you use auto connect then by default it will use "Sequence Flow" shape for connecting

tom ogilvy say:

that is what his formula is doing

=IF($A1="","",SUM($A$1:$A1))

he assumed the daily numeric data would be entered in column A.  So you can put this formula in row 1 of any column and drag fill it down as far as you need.  Each cell will appear empty until you put a value in column A.

You could make it more discriminating with

=IF(isnumber($A1),"",SUM($A$1:$A1))

so it would treat any cell in column A that did not contain a number as if it were blank.

Adjust the formula to reflect the column that will have your numeric values.

john mcghie say:

Sorry, but this is NOT an expected behaviour at all :-)

If you have set the styles up correctly, the document will look almost exactly the same between PC Word 2013 and Mac Word 2011.

There are some tricks to this trade, however:

  1. The first is to ensure that the Format of the Table of Contents is "From Template".  If you use any other format, Word will overwrite the formatting with its hard-coded variations whenever she regenerates the TOC.  "From Template" is confusing: it means "The TOC template stored in the document" and is the only one that enables you to actually control the formatting.
  2. Use the Microsoft Fonts.  The fonts with names beginning with C -- Calibri and Cambria basically -- have been specially built to display exactly the same on each platform.  The letter grid is a different shape on the Mac, the c-series fonts have been coded accordingly.
  3. The random spacing is usually lack of an update to cure that bug.  Check that both she and you have applied all the latest updates.  It can also be "justification".  Justification looks quite ugly in Word for the Mac, my strong suggestion is "don't use it".
  4. If your styles are correctly defined, there should be very little difference when you change printer drivers.

If you can't find the problem, email me a copy to john "at" mcghie.name and I will tell you what's wrong.  Don't forget to tell me what the issue is in your email, I get a LOT of email!  And it's best to send the email in unformatted plain text with  the document as an attachment, otherwise you'll fall in the spam bucket and I won't see it.

Hope this helps
