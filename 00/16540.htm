tom van stiphout say:

Here is my standard answer. It's a multi-step approach.

1. Go back to where this data came from. Maybe they can provide it to you the way you want it.

Assuming no:

2. Analyze the data. See what kind of variability occurs.

3. Write down all the rules you can discern.

In your case, some of those would include:

* Name is on the first line

* Street is on the second line

* City, State, Zip is on the third line

* Lines are separated by vbCrLf

* Line3 is in standard US format, meaning City, a comma, one of 50 state abbreviations, a space, and a 5-digit zipcode.

4. Get buy-in from your boss for this approach.

5. Implement a parser for these rules. This is best done in VBA, although really contorted query expressions could probably do it as well.

The parser may take the form of a procedure that opens a recordset on the source table, reads the data, parses it in its 5 fields, and writes that to a new table (or additional fields in the source table).

The code would be written such that the rules from step 3 are all included, and that records that do not conform are spit out for further analysis or manual parsing by a human.

The parser would probably use a combination of vba functions like Split, InStr, Left, Mid, Right to do the work.

 

Happy coding!

rob koch say:

None of those files look suspicious to me, they all appear to be normal Windows caching, system or temporary files.

The first few are clearly part of the Internet Explorer browser recovery files.  These are used when the browser crashes and after the browser is restarted, when it offers to "Restore Last Session".  These can be cleared by using the Delete browsing history selections in either the Safety sub-menu or the Internet Options, General tab, Browsing History, Delete button.

I haven't found a description of the purpose for the UPPS.bin file, but it's known to be a Windows system file.

There are always at least a couple files in the Temp folder which are kept open while Windows is running and so can't be deleted, and I've seen both of those names before, so again I wouldn't be concerned.

The ElevatedDiagnostics folder contains the history and logs for the Windows 7 Troubleshooting Platform.

All of these are basically system files or folders, though most of them are also temporary files, which is why they are either in use and/or locked and thus unable to be deleted.

Microsoft has also changed many such files to higher security in order to keep malware from modifying them, so the reasons you can't open or delete them are by design.

Moral of the story, just leave them alone, they aren't related to malware and shouldn't be messed with except by using the methods Microsoft provides such as the Delete browsing history selections I mentioned above.

Hi,

 

Use 3 randbetween to miss out O & I and randomely choose which one to use which a randbetween(1,3)

 

 

=CHAR(IF(RANDBETWEEN(2,33)<10,RANDBETWEEN(CODE("2"),CODE("9")),CHOOSE(RANDBETWEEN(1,3),RANDBETWEEN(CODE("A"),CODE("H")),RANDBETWEEN(CODE("J"),CODE("N")),RANDBETWEEN(CODE("P"),CODE("Z")))))


This approach ruins the equal probability for each possible outcome, if that is at all important.
The eight values from A to H will have the same probability as the five values from J to N and the eleven values from P to Z, meaning that e.g. J will be more than twice as likely as P.

Here is another proposal that gives the same probability for each possible outcome.

=MID("23456789ABCDEFGHJKLMNPQRSTUVWXYZ",RANDBETWEEN(1,32),1)
