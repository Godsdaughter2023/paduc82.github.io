Auto fill name and address - Microsoft Community

mary620 say:

Auto fill name and address

I use spreadsheets that contain only three  name and address fields which I currently type in manually.     I would like to find a way to select the name and have the address and other information populate the cells....The catch is that I would like to be able to populate any spreadsheet I open up.    Is this even possible?   Thank you for any help you can give me.

jllatham say:

Yes, it is possible.  Look into the VLOOKUP() formula.

 

Let us say that your table uses column A for the names, B for street address, C for city, state & zip.

The table is on a sheet named [My List].  The list currently goes to row 1000, but you want room to grow to at least twice that size, down to 2000, without having to mess with your formulas.

 

On any sheet in the workbook you can put a formula like this:

=VLOOKUP(A2,'My List'!$A$1:$C$2000,2,False)

Then if you type a matching name into cell A2 on that sheet, that formula would return the street address for the name.  If you change it to:

=VLOOKUP(A2,'My List'!$A$1:$C$2000,3,False)

Then it would return the 'city, state & zip' entry matching the name.

 

By the way, the A2 could be a literal entry, although you probably don't really want that, but it could be like

=VLOOKUP("Mary620",'My List'!$A$1:$C$2000,2,False)

 

If you want the ultimate in convenience, Excel 2007, 2010 and 2013 allow you to reference entire columns without row numbers - making your formulas maintenance free at the expense of a loss of some speed.  So those could be written as:

=VLOOKUP(A2,'My List'!$A:$C,2,False)

and

=VLOOKUP(A2,'My List'!$A:$C,3,False)

 

This will work with workbooks also.  Excel will even do the heavy lifting for you.  Open the workbook with the list and select the sheet in it with the list on it.

 

Now open another workbook that you want to set up a similar formula in.

This is easier to do if you go to the [View] tab and choose to view them Side-by-Side so that you can see both sheets in Excel at the same time.

Pick a sheet and a cell where you want the formula and start typing:

=VLOOKUP(A2,

(assuming the name you want is in A2 on this sheet also)

Now click in the other workbook (the one with the list, and select columns A:C) ... Excel will put something like this into your formula:

=VLOOKUP(A2,'[LinkedTo.xlsx]My List'!$A:$C

now return to your workbook with the formula and finish it out with ,2,False) and you should have something like

=VLOOKUP(A2,'[LinkedTo.xlsx]My List'!$A:$C,2,FALSE)

 

When you close the workbook with the list in it, excel will update that formula to provide the complete path to the file.  You can now 'fill' the formula or copy it as required.  Repeat for other workbooks.

 

Note: if VLOOKUP() does not find a match to what you are looking for in the first column of the table, it will return a #N/A! error.  Looks rather ugly.  You can keep the ugliness out of your life by "wrapping" the formula within an error test like this:

 

 

=IFERROR( VLOOKUP("Mary620",'My List'!$A$1:$C$2000,2,False) ,"")

So if "Mary620" (or the contents of our fictitous cell A2) is not in the list, it will return a blank cell rather than the ugly #N/A! error notification.

jllatham say:

Ok, we might have gotten a bit more efficient if I knew which column the 55x60x19QLF type entries are in on the [...2015] sheet also, but I think this will still work just fine.

Because this is going to alter the information on the [...2015] sheet, you should first test this code in a COPY of your workbook. 

The code goes into a regular code module:  just open the (test) workbook and press [Alt]+[F11] to open the VB Editor (VBE).  Then choose Insert --> Module from its menu and paste the code below into that module.  Make any edits you need to to the Const values I've defined.  I think they are all set up for your sheets, but if I messed up something you can 'fix it'.  I have set it up to copy from columns L, M, N and O of [Page 3] to columns E, F, G and H of the [...2015] sheet.  Values are copied, not formulas.

While the macro is running you won't see anything happening no matter what sheet you have chosen (and any sheet can be selected when you start the macro).  The changes won't appear until you click the [OK] on the message that appears when it is finished.

Sub UpdatePrices()
'we can change these Const values to match your worksheet
'names and layouts
'the sheet with update information
  Const updateSheetName = "Page 3"
  'column that item identification is in
  Const updateItemIDCol = "A"
  'first row that an item identification is in
  Const updateItemFirstDataRow = 2
  'first column to copy with new price information
  Const updateFirstCopyCol = "L"
  'last column to copy with new price information
  Const updateColumnCount = 4 ' # of columns to copy
'
'these define the 2015 price sheet that is to be updated
  Const newPriceSheetName = "Current price sheet 2015"
  Const newPriceFirstPasteCol = "E"
  Const newPriceDateCol = "I"
'
' working variables
  Dim updateWS As Worksheet
  Dim updateItemList As Range
  Dim updateOneItem As Range
 
  Dim newPriceWS As Worksheet
  Dim newPriceSearchRange As Range
  Dim foundCell As Range
  Dim foundRow As Long
  Dim lastRow As Long
  Dim lastCol As Long
  Dim sourceRow As Long
  Dim copyRange As Range
  Dim colOffset As Integer
 
  Set updateWS = ThisWorkbook.Worksheets(updateSheetName) ' Page 3
  'set reference to all item description entries
  Set updateItemList = updateWS.Range(updateItemIDCol & updateItemFirstDataRow & ":" _
   & updateWS.Range(updateItemIDCol & Rows.Count).End(xlUp).Address)
 
  Set newPriceWS = ThisWorkbook.Worksheets(newPriceSheetName) ' Current price sheet 2015
  'find actual last row and last column used on this sheet
  On Error Resume Next
  lastRow = newPriceWS.Cells.Find(what:="*", _
   after:=newPriceWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByRows, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Row
  On Error GoTo 0
  On Error Resume Next
  lastCol = newPriceWS.Cells.Find(what:="*", _
   after:=newPriceWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByColumns, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Column
  On Error GoTo 0
 
  Set newPriceSearchRange = _
   newPriceWS.Range(newPriceWS.Cells(1, 1), newPriceWS.Cells(lastRow, lastCol))
 
  'we turn off screenupdating for speed,
  'we turn off EnableEvents to keep any macros from running
  'we turn off calculation again for speed
  'in case of error, we will turn them back on!
  On Error GoTo CleanupAndExit
  Application.ScreenUpdating = False
  Application.EnableEvents = False
  Application.Calculation = xlCalculationManual
  'now we loop through each item id cell on Page 3 and
  'find match on the 2015 sheet and do the updates
  For Each updateOneItem In updateItemList
    DoEvents
    Set foundCell = newPriceSearchRange.Find(what:=updateOneItem.Text, _
     after:=newPriceSearchRange.Cells(1, 1), _
     LookIn:=xlValues, _
     MatchCase:=False)
    'did we find a match?
    If Not foundCell Is Nothing Then
      'yes!
      sourceRow = updateOneItem.Row
      foundRow = foundCell.Row
      For colOffset = 0 To (updateColumnCount - 1)
        newPriceWS.Range(newPriceFirstPasteCol & foundRow).Offset(0, colOffset) = _
         updateWS.Range("L" & sourceRow).Offset(0, colOffset)
      Next ' end colOffset loop
      'set the date
      newPriceWS.Range(newPriceDateCol & foundRow).FormulaR1C1 = "12/14/2014" ' change as desired
    End If
  Next ' end updateOneItem loop
 
CleanupAndExit:
  If Err <> 0 Then
    MsgBox "Error Encountered! Processing Incomplete!!" & vbCrLf _
     & Err.Number & " " & Err.Description, vbOKOnly + vbCritical, _
     "Error Interrupted Processing!"
    Err.Clear
  Else
    MsgBox "Task Completed Without Error", vbOKOnly + vbInformation, "Job Done!"
  End If
  On Error GoTo 0
 
  Application.EnableEvents = True
  Application.Calculation = xlCalculationAutomatic
'some housekeeping
  Set updateItemList = Nothing
  Set newPriceSearchRange = Nothing
  Set updateWS = Nothing
  Set newPriceWS = Nothing
End Sub
