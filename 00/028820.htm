karl timmermans [outlook mvp 2012-2018] say:

 does anyone know how to configure Outlook 2016 so that it automatically deletes messages from the server when they are downloaded?

Account needs to be configured using the POP protocol with the "Leave a copy of Messages on server" option <unchecked>. Option not available to any other account type since the data store for those accounts (Imap, Exchange/Exchange ActiveSync) are a mirror of what is on the server.

For Imap/Exchange account type, you would need to move the items to a folder in a local data store (.pst file). The <move> function would copy the item(s) to the local folder and delete it/them from the primary folder.
