Access SQL complex join - Microsoft Community

greg hinks say:

Access SQL complex join

I have a dataset with three tables.  The first contains a field of names with corresponding fields of descriptive details.  The second table contains a field of weights that corresponds to the first table, but with multiple weights per name.  The third table is similar, but contains lengths corresponding to the names, again with multiple lengths per name but unrelated in count to the weights.  All three tables are related by a unique primary key, "ID".  I'm having trouble constructing a summarizing table containing the names from the first table, a count of the weights per name from the second table, and a count of the lengths per name from the third table.  This is what I have so far:

 

SELECT CATCH.ID AS ID, CATCH.COMMON AS NAME, COUNT(WEIGHT.VALUE) AS WEIGHT_COUNT

FROM (CATCH INNER JOIN WEIGHT ON CATCH.ID=WEIGHT.CATCH_ID)
GROUP BY CATCH.COMMON, CATCH.ID

ORDER BY CATCH.ID

 

This works fine, until the table of lengths is added and the count results in a cartesian product.  How do I avoid this and get a count of only the lengths per name on the same table?

qa guy commelec say:
Try this --

SELECT CATCH.ID AS ID, CATCH.COMMON AS NAME, COUNT(WEIGHT.VALUE) AS WEIGHT_COUNT, COUNT(LENGTH.VALUE) AS LENGTH_COUNT

FROM (CATCH LEFT JOIN WEIGHT ON CATCH.ID=WEIGHT.CATCH_ID) LEFT JOIN LENGTH ON CATCH.ID=LENGTH.CATCH_ID
GROUP BY CATCH.COMMON, CATCH.ID

ORDER BY CATCH.ID

scottgem say:
First, Names are not a good key fields as they may not be unique and text fields don't perform as well in joins.

Second, you should have a unique Primary key for each table. but you have a one to many relationship between names and both weights and lengths. So tose two tables, need to use the PK from the names table as FKs and those should be your joins. 

greg hinks say: I apologize for the confusion.  Each table does utilize its own incrementing integer for a key as well as the catch table's primary key.  Strings are only data fields.

ken sheridan say: I think you'll need to use a subquery to do it as a single query.  Try this:

SELECT Catch.ID, Catch.Common AS Name, COUNT(*) AS Weight_Count,
    (SELECT COUNT(*)
     FROM Length
     WHERE Length.Catch_ID = Catch.ID) AS Length_Count
FROM Catch INNER JOIN Weight ON Catch.ID=Weight.Catch_ID
GROUP BY Catch.ID, Catch.Common;

If that doesn't do the trick create two queries, one which counts the weights per Catch.ID, the other the lengths per Catch.ID and join the two queries on Catch.ID to return the final result table.
