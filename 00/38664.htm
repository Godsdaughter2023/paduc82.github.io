Bug? Windows Live Mail messes up file extension of attachment - Microsoft Community

win7 was my idea say:

Bug? Windows Live Mail messes up file extension of attachment

I just found out that if a file has an extra dot before the extension, e.g. book1..xls (or book1. if you have "hide known file extensions" enabled in Folder Options), then the following will occur.  If you right click the file, click Send to, click Mail Recipient, a new mail window opens up with the attachment of the file RENAMED TO book1._xls.  Apparently, the extra dot has been replaced with an underscore.  

Also, if there are additional dots before the extension, only the 2nd, 4th, 6th, etc. dots will be replaced with underscores.

That means if the dot JUST BEFORE the extension is in an even-number position, it will become a "_", making the file extension "_xls" and the file unopenable by your recipient.

E.g.

book1...xls will become book1._.xls; file name changed but still openable.

But book1....xls will become book1._._xls; file becomes unopenable.

This does not happen in Outlook Express.  It seems to only happen with Windows Live Mail.  I'm using XP and don't know if 7 is the same.
