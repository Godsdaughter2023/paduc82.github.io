ken sheridan say:

Private Sub Form_load()

Me.ToolID = Forms("ToolSelect").Controls("ToolID")
DoCmd.GoToRecord acDataForm, "SendCal", acNewRec


You appear to be assigning a value to the ToolID control before moving the form's record pointer to an empty new record.  If the form is returning existing records this would overwrite the value in the first record in the form's recordset's sort order.  If, on the other hand, the form's DataEntry property is True or the form has been opened at a new record by setting the DataMode argument of the OpenForm method to the constant acFormAdd, then there is no need to move the record pointer to an empty new record.

When opening a second form to add related data it is in most cases better to open the second form in dialogue mode and pass the value of the key to it via the OpenArgs mechanism.  You'll find an example in FormsDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file if you select the 'Linked Forms - Dialogue' option the code in the Click event procedure of the Events button for instance is:

Private Sub cmdEvents_Click()

    Const FORMNAME = "frmEvents_2"
    Const MESSAGETEXT = "No current client record."
    Dim strCriteria As String
    
    strCriteria = "ClientID = " & Me.ClientID
    
    ' be sure a record has been entered
    If Not IsNull(Me.ClientID) Then
        ' save current client record
        Me.Dirty = False
    
        ' open linked form in dialogue mode
        DoCmd.OpenForm FORMNAME, _
            WhereCondition:=strCriteria, _
            WindowMode:=acDialog, _
            OpenArgs:=(Me.ClientID)
    Else
         MsgBox MESSAGETEXT, vbExclamation, "Invalid Operation"
    End If

End Sub

In the Open event procedure of the frmEvents_2 form the value passed via the OpenArgs mechanism is assigned to the DefaultValue property of the cboClient control, which is bound to the foreign key ClientID column, with:

Private Sub Form_Open(Cancel As Integer)

    If Not IsNull(Me.OpenArgs) Then
        Me.Event.SetFocus
        Me.cboClient.DefaultValue = """" & Me.OpenArgs & """"
   End If

End Sub

Note that the DefaultValue property is always a string expression regardless of the data type of the column in question, so is wrapped in literal quotes characters as above.  Assigning a value to the DefaultValue property of a control is better than assigning it to the control's Value property, which is what your code does as the Value property is a control's default property.  Unlike the latter, the former does not initiate a new record, so if the user decides to abort the insertion of a new record they can simply close the second form, without having to call the form's Undo method.

Hello LauraFidone1

From your description, sounds like the account was originally configured as an IMAP account and the contacts/notes were contained in folders with the words "This Computer Only" in the folder name.

If the above is the case, when the email account was deleted, it also removed the OST data file associated with the account which contained those folders at which point all data in those folders has been lost. When you re-added the account again, a new .ost file would have been created but the "This Computer Only" folders will be empty. There is no recovery process for the data that was lost - certainly not what you were looking for but unfortunately it's the way things work with IMAP accounts which only sync email folders with the server.

The following is not going to help you with recovering the lost data but it will mitigate the same thing from happening again in the future.

#1 - From Outlook <Home> ribbon, click on <New Items> --> <More Items> --> Outlook Data file. This adds a PST file to your Outlook profile. Then

#2 - Go to File --> Account Settings --> Data Files tab
#3 - Select the newly added PST file and click on <Set as Default>. This will add all the standard Outlook folders to the PST file and will be where the default contact/calendar folders are stored. This is also where you'll want to save your Notes. PST files do not get automatically deleted from the system under any circumstance.

Word Batch Macro that includes subfolders? - Microsoft Community

sam bits say:

Word Batch Macro that includes subfolders?

Hi, i am using the below macro i found online to remove protections from word documents in one folder. I would like to extend this capability to include subfolders and folders within the subfolders? Any help would be appreciated as i dont have much experience in VBA :)

Sub RemoveProtection()
    Dim strPath As String
    Dim strFile As String
    Dim Doc As Document
    On Error GoTo ErrHandler
    With Application.FileDialog(msoFileDialogFolderPicker)
        .Title = "Select a folder with Word documents"
        If .Show = False Then
            MsgBox "You didn't select a folder.", vbInformation
            Exit Sub
        End If
        strPath = .SelectedItems(1)
    End With
    If Right(strPath, 1) <> "\" Then
        strPath = strPath & "\"
    End If
    Application.ScreenUpdating = False
    strFile = Dir(strPath & "*.doc?")
    Do While strFile <> ""
        Set Doc = Documents.Open(strPath & strFile)
        If Doc.ProtectionType = wdAllowOnlyFormFields Then
            Doc.Unprotect strPassWord
            Doc.SaveAs FileName:=strPath & "NoPass\" & strFile
        End If
        Doc.Close wdDoNotSaveChanges
        strFile = Dir
    Loop
    Exit Sub
ErrHandler:
    MsgBox Err.Description, vbExclamation
End Sub

vijay a. verma say:
If you had installed it on a computer and activated this, then you don't require a key again. As Microsoft activation servers store a digital license after the first activation and this digital license will be used for activations on other installations. It will not ask for product key again.

You will have to first deactivate Office license which was installed on old computer otherwise it will be counted as a license. Hence, if you were allowed installation on one computer, you will not be able to activate on new computer. If your license was for say 5 computer as in case of Office 365 Home, now your validity will be for 4 computers.

Hence, deactivate first and then install and activate.

2. Install Office again on a new computer - https://support.office.com/en-us/article/Downlo...
