A Table or a Query must be "homogenous" - all records must have the same number of fields of matching datatypes. You can't interleave disparate types of records in the way you describe.

filoudog say:

Recover a lost autosaved document that is not in the File Management section of previous saved versions.

I have tried a number of suggested methods to find a document that has no autosave copy and you are faced with what you began with when you started working on it. This is the only way I have ever been able to reinstate an autosave that appears not to exist when you click on the document Management option to see what saved versions are available, only to find there are none. So:

Open up any Word document, it doesn’t matter which one.

Click on far left File tab at the top of the ribbon

Click Options from the list on the left

Click Save from the list on the left of the opened box

Copy the location of where saved files are kept (AutoRecover file location) to your cursor - likely C:\Users\your name\AppData\Roaming\Microsoft\Word\

Close the whole box

Go back to the Word document, go to File again

Click Open

Click Browse

Make sure ‘*all types’ is selected to the right of the blank box that appears

Right click on the empty box and then left click Paste that will appear in the dropdown menu. However, when you click paste, all saved files will appear in a new dropdown menu under the blank box (you will not get the option to paste the coped location, which is right). You may need to expand the dropdown to see the full file names (left click and drag on bottom right corner of dropdown). The auto saved documents usually have an .asd suffux but you might be lucky and find a rogue Word.doc suffix.

Highlight the file you want to recover by right clicking on it and it will go into the box.

Click on Open and it will open the document in Word format

The document will have an AutoRecover save prefix but will be in Word format.

When you try to save the document, a box will appear. From the left side options choose where you want to save the file to – likely click on ‘Documents’ and then on the specific document folder you want to open it. Save the document to there.

Job done.

tom ogilvy say:

Every cell in a column has the same width.  Every cell in a row has the same height.  If you see a cell that seems to violate this rule, it is sized to encompass multiple cells, then it is probably merged.  If you have gridlines turned off there would probably be not visible indication. 

If you created the workbook and you did not merge any cells or copy and paste into the workbook from an external source, then I would expect there would be no cells that are merged.
