Preventing saving a template - Microsoft Community

kecarlso say:

Preventing saving a template

In older versions of Word, you were never allowed to save a template (.dot file) There were also pop-up boxes that could help others populate just what they needed.

Are these options available?

I have tried restricted editing with fill-in forms. But the document can still be saved and the comments overwritten by what the user has inputed.

jay freedman say:

There was never any built-in mechanism that prevented saving any template. It was always possible to write a macro or add-in that would prevent saving changes to the specific template containing the code, or to all templates (although that makes no sense in most cases). It's also possible to mark the template's file as Read Only, which would force a changed template to be saved to a different file name and/or path. Both of these methods would require someone (user or administrator) to do some special work to set them up.

I suspect that what's causing your problem is that templates are being misused. The user should use File > New and select the desired template, which creates a new document based on the template. Changes in the document are not saved to the template (with the possible exception of changes in the definitions of styles), and nothing gets overwritten. However, if the user opens the template itself and makes changes, there's generally nothing to prevent saving those changes in the template's files.

There are many different methods for giving users ways to input data into a preformatted document.

  • Form fields (since Word 2007, labeled as "legacy form fields") are used together with restricting editing for filling in forms.
  • ActiveX controls serve similar purposes to form fields, but don't require restricted editing. They're also considered "legacy" controls.
  • ASK and FILL-IN fields pop up boxes when the fields are updated. They're intended for use in mail merges, but can be used elsewhere.
  • Starting in Word 2007, content controls allow entry of information. There are a couple of ways to prevent editing of parts of the document outside the controls.
  • A userform -- a custom dialog box programmed in a macro -- is a superb way to collect information to populate a document, but requires a bit of facility with Visual Basic for Applications (VBA).

None of these methods inherently prevent saving changes to templates -- that still has to be provided either by a macro or by setting the Read Only flag.

Private Sub Worksheet_BeforeDoubleClick(ByVal Target As Range, Cancel As Boolean)
  If Target.NumberFormat <> "#" Then Exit Sub
  Application.EnableEvents = False
  If IsEmpty(Target) Then
    'Add a checkmark if cell is empty
    Target = ChrW(&H2713)
    Cancel = True
  ElseIf Target.Value = ChrW(&H2713) Then
    'Clear cell if contains a checkmark
    Target.ClearContents
    Cancel = True
  End If
  Application.EnableEvents = True
End Sub
