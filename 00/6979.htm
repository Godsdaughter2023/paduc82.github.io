I need to use categories and labels within Outlook Calendar--but when I upgraded from 2003 to 2010, this functionality appears to be missing. Is this true? Any other options on how to create similar organization for calendar entires?
brian tillman [outlook mvp 2007-2019] say:

You can certainly continue to use categories.  Labels are gone, but categories have been extended so you can associate colors with them (and more of those colors) rather than using labels to color them.  The first thing you should do is to right-click the root of your folder set, choose Properties, and then click Upgrade to Color Categories.  If you migrated your old Outlook data correctly, the category information you had in the old data file will still be there and upgrading to color categories should add your categories to the Master Category list and assign them random colors.  You can adjust that assignment afterward, if you wish.

Some good informaiton is here: http://www.slipstick.com/outlook/outlook-categories-and-color-categories/

Outlook Room Scheduling - Blue Box/No Details for Meeting Room - Microsoft Community

chupacabraz say:

Outlook Room Scheduling - Blue Box/No Details for Meeting Room

Wanted to spread the knowledge, as this issue frustrated me and I could not find a solution online.

After discovering that you can manage rooms and resources via O365, I added 7 conference rooms. When a meeting was scheduled for a particular conference room (we'll call it Room A), the details did not appear in Outlook's Schedule View. Instead, a solid blue box occupied the space where the meeting was scheduled.

I gathered that it is an Exchange issue, as Room A is the only room that does not appear as a room. Instead of a door icon next to the room name, it is a user icon (little gray bust of a man).

I tried deleting the room via the O365 Admin portal and recreating it, but the issue persisted.

To resolve, I added myself as a delegate with Full Control to the room via the Exchange admin center.

This allowed me to see the details of the scheduled meetings in Outlook, but no one else could.

In Outlook, I right-clicked on the room and went to Properties.

In the properties, I changed the default permission level to "Free/Busy time, subject, location".

Now everyone can see the details of the meeting.

I'm going to mess around with this until I figure out how to FIX IT, but this is a great workaround for now.

alan wu msft say:

Hi Chupacabraz,

Greetings. This is Alan Wu from Office 365 forum support. Very honored to reply to your case here. 

Via your description, my understanding is that you want to view the details (ie, the free/busy, meeting events' details) of a conference room. Please be noted that the users are able to see the conference room's free/busy information only by default. And, there are 2 ways to view the room's free/busy information. 

1. Add the room calendar under the user's default calendar. This needs to be done by an admin via the Exchange Online PowerShell cmdlet Add-MailboxFolderPermission

2. Check the room's free/busy information via Outlook scheduling assistant. https://support.office.com/en-us/article/schedule-a-meeting-with-other-people-5c9877bc-ab91-4a7c-99fb-b0b68d7ea94f

For the above second method, the user can be able to see whether the room is free/busy during the time when the user wants to schedule a meeting. For the above first method, the admin can set the permission level so that he can decide what information (free/busy, meeting details like subject, attendees, etc) the user is able to view. It is like the workaround you are now using. 

Actually, this is not an issue but it is related to the fact that the user is only able to see the room's free/busy information by default. But we can change the permission level. Via the workaround/Powershell cmdlet.
