It is not a change to IMAP at all, it is a change to Office 365 Exchange online.  It will allow you to stop using the Hotmail connector and start using the Exchange Server service which is far more robust and reliable; your contacts and calendar are still on the server and will continue to sync, unlike IMAP.  What you will have to do is delete the Hotmail account in Outlook 2010 and re-create it using the Auto Account Setup feature in Outlook which will notice the change and set the account up as an Exchange Server account.  The only version of Outlook that will require an IMAP setup after the change is Outlook 2003.  Please see:

jeanette cunningham say:

There are 2 places where an error could appear. One is the form error event, the other is in the code above when it is running.

To remove an error in the form's error event, use code something like this:

Private Sub Form_Error(DataErr As Integer, Response As Integer)

If DataErr = 1234 Then
   Response = acDataErrContinue
End If

End Sub

To trap errors in the sub for combo 53, use error handling something like this:

Private Sub Combo53_AfterUpdate()
On Error GoTo Err_Handler

    ' Find the record that matches the control.
    Dim rs As Object
    Set rs = Me.Recordset.Clone
    rs.FindFirst "[ID] = " & Str(Me![Combo53])
    If Not rs.EOF Then Me.Bookmark = rs.Bookmark

Exit_Handler:
    Exit Sub
Err_Handler:
    Select case Err.Number
    Case 1234, 5678
       Resume next
    Case else
        Msgbox Err.Number & " " & Err.Description
        Resume Exit_Handler
   End Select    

End Sub

Replace the error numbers 1234 and 5678 with the numbers for the errors you get when your code above is running.

jeanette cunningham say:

You are right about the form's error event - in your case that error may not be coming from the form's error event. You can comment out the code on the form error event and test to be sure.

The form's error event is a different thing from the other type of error trapping and they require different ways of writing the code.

The form's error event has DataErr and Response.

DataErr gives the number for the error and Response is your way to tell Access what to do with the error.

acDataErrContinue tells access to ignore the error.

acDataErrDisplay tells access to show the err number and message to the user.

Here is what the help says:
acDataErrContinue Ignre the error and continue without displaying the default Microsoft Access error message. You can supply a custom error message in place of the default error message.

acDataErrDisplay (Default) Display the default Microsoft Access error message.

To learn more, type Form error event into the offline access help.

Jeanette Cunningham MVP(Access) Pakenham, Victoria Australia
