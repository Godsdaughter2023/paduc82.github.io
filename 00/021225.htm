Hi, no, that checkbox doesn't solve the problem. It is used to switch between the "traditional" IMAP behavior and the "Exchange-like" behavior.

If you check it, the messages are not deleted, but marked for deletion. Depending on the view you are using they are either hidden (so it looks like they have been deleted) or shown with a strike-through line. Then you can either purge them manually using the command provided under the Clean up section of the Folder menu in the ribbon, or use the other checkbox just under "Mark items for deletion..." to automatically purge the items when you switch folder. However, in this case the items are permanently deleted, not moved to the Deleted Items folder. This is the standard IMAP behavior AFAIK.

If you uncheck the box (and this is the default in Outlook when you create a new IMAP account), deletion should work like in Exchange, that is messages are copied to the Deleted Items folder (instead of being marked for deletion) and then immediately removed. This is what is broken in Outlook 2016. The messages are copied in Deleted Items, but not removed from the original folder.

Checking the box is a workaround at best, not a solution. I prefer having the deleted messages in the Deleted Items folder until I decide to empty that folder.

scottgem say:
OK, I see your problem. The problem is your delete button is creating orphans. Those orphans are being duplicated by your code.

First clean up your data by removing any orphan records in StateListT. Orphans are records with a Ln value not in ClientsT. Then open the Relationships window and set up a relation between ClientsT and StateList on ClientsID to Ln. Then check off the Referential Integrity and and set it for Cascading Deletes. 

When you delete a client all their state records should be deleted also.

I still think that location is an attribute of the course, not a separate item. 

Also, since you are using autonumber PKs, there is no need to display them. Your comboboxes should display the names, but store the code. If you use the wizard to create a combo and accept the defaults you will see what I mean.
