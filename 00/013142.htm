What steps should I follow to delete and then redo headers, section - Microsoft Community

pccostello say:

What steps should I follow to delete and then redo headers, section breaks, and page numbering in a very long document (400 pages) without disturbing paragraph formatting and styles?

What steps should I follow to delete and then redo headers, section breaks, and page numbering in a very long document  (400 pages) without disturbing paragraph formatting, headings, styles, font, etc?

I have finished writing a very long document that I would like to submit online today. However, the headers and section  breaks are chaotic, as is the page numbering, with some adjacent chapters being numbered together and other chapters numbered independently.

I would like the page numbering to be continuous for the overall document and the headers to change with the beginning of each new chapter. There are 11 chapters.

What I think I need help with isa set of steps to strip out and then re-establish section breaks and to re-do the headers that would go with each chapter.

Thank you very much for you help.

Peter.
suzanne s. barnhill say:

Nothing you do with headers, page numbering, or section breaks will affect paragraph formatting. You can easily delete all section breaks at one go by searching for ^b and replacing with nothing. This will automatically get rid of all your headers except those in the last section, along with any page number field that is there, and page numbering will be continuous throughout the document. If you restarted it at any point, you may need to go to the Page Number Format dialog and change it to "Continue from previous section" (which is the default for a single-section document).

If what you want to change in the header at the beginning of each chapter is just the chapter number or title, you can accomplish this with a StyleRef field (see http://WordFAQs.mvps.org/StyleRef.htm), but if you're submitting the document for publication and it's going to be printed just the way you submit it, then you need to do more because you won't want a header on the first page of the chapter.

  1. You can still use the StyleRef field (this will save you from having to unlink headers and run into possible page numbering issues), but you will need some section breaks.
  2. Start by enabling Different First Page in your one remaining section. This will become the default setting for any new sections you create.
  3. Then, at the beginning of each chapter, insert a Next Page (or Odd Page) break, so that the chapter will begin on a new page.
  4. Leave the First Page Header empty.
  5. If the remaining Header is what you want, then you're done, but if you want different running heads on recto and verso (odd and even) pages, or if you want the page number on the outside of the page, then you'll need to also enable Different Odd and Even Pages and create content for the Even Page Header (the primary Header will become the Odd Page Header). Unlike Different First Page, which can be turned on or off for a given section, Different Odd and Even affects the entire document.

When I'm formatting books, I generally find it easier to put the page number in the footer (centered on every page), leave the First Page Header empty, and have different content (author and book title, author and chapter title, book title and chapter title) in the odd and even headers, but you can also place the page number at the outside of the odd and even headers along with whatever other content you include.

If this is a book, you might also want to look at http://www.word.mvps.org/FAQs/Formatting/front_matter_2007.htm.

There are two issues here.  First, I don't believe a delete query in Access can delete from more than one query at a time, so you would have to run two separate queries:  one to delete from GuardiansAndContacts, and one to delete from Students_GuardiansAndContacts. 

Second, when you use CurrentDb.Execute to run a query, you're using pure DAO, rather than Access, to do it, and DAO doesn't know about forms and controls on forms.  So, since you are building the SQL for the delete query in code anyway, what you can do is build the literal value of the StudentID directly into the SQL, in place of the reference to the control on the form.  Your code might look like this (assuming that StudentID is a numeric field:

'------ start of code ------

Dim varStudentID As Variant

' Capture Student ID from the form
varStudentID = Forms!frmStudentInformation!StudentID

With CurrentDb

    ' Delete from GuardiansAndContacts
    .Execute _
        "DELETE GuardiansAndContacts.* FROM GuardiansAndContacts INNER JOIN Students_GuardiansAndContacts " & _
            "ON GuardiansAndContacts.ID = Students_GuardiansAndContacts.GuardianID " & _
           "WHERE Students_GuardiansAndContacts.StudentID = " & varStudentID, _
        dbFailOnError

    ' Delete from Students_GuardiansAndContacts
    .Execute _
        "DELETE FROM Students_GuardiansAndContacts WHERE StudentID = " & varStudentID, _
        dbFailOnError

End With

'------ end of code ------

Note: when using DAO to run the delete queries, you don't need to SetWarnings off and on.
