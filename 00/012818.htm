Need help for query for stock in hand - Microsoft Community

khurramnak say:

Need help for query for stock in hand

Hi there,

I need help for making a query of stock in hand. Please see the scenario.

  1. I buy books and mobile accessories for my small shop (Stock In).
  2. I examine the supply and return substandard goods to the supplier (Stock out).
  3. I sell the goods from my store (stock out)
  4. What i need is the status of my stock in hand. 

The tables I made are shown below.

Kindly help me. I am unable to take report without it.

Regards,

Khurram

scott mcdaniel say:

This might be easiest to visual with three separate queries, after which you could then create a "master" stock query to determine actual quantities.

So a query for Sales would be something like:

SELECT Item, SUM(SaleQty) AS SaleQty FROM tblSale GROUP BY Item.

For Returns:

SELECT Item, SUM(QtyRet) AT ReturnQty FROM tblStockRet GROUP BY Item

For Stock Intake:

SELECT Item, SUM(StockIn) AS StockInQty FROM tblNewStock GROUP BY Item

You could then create a master query and include all those:

SELECT qrySales.Item, StockInQty - ReturnQty - SaleQty AS CurrentStock FROM qrySales INNER JOIN qryReturns ON qrySales.Item=qryReturn.Item INNER JOIN qryStock ON qryStock.Item=qrySales.Item

With that said, the ONLY way this sort of stock count method works is if you have 100% accurate data records, and you have no shrinkage, loss, etc (and that would be rare in the extreme). In most robust stock management systems, you have "stock take" markers, which is essentially a physical stock count that is made, and the stock management system then works against that "stock take date" and provided you with data from that date forward ONLY. 

scottgem say:

I generally design my Inventory databases around a central Transactions table with a Transaction Type field. Each transaction is then identified as incoming, outgoing or neutral. (For example Received would be Incoming, Sale and Return would be outgoing. 

From there I use two queries:

SELECT TransactionID, ItemID, Quantity, TransactionTypeID, IIF(TransactionTypeID<50,Quantity*-1,Quantity) As AdjQty

FROM tblItems;

I set all Outgoing Transaction Types as <50 and incoming as 50 or >.

The result of this query will be to turn the Quantity negative for outgoing transactions.

From there:

SELECT ItemID, SUM(AdjQty)

FROM qryAdjQty

GROUP BY itemID;

Will return the quantity on hand for each item.

This is similar to Scott Mcd's solution which would be more applicable if you keep your transactions in separate tables. 

scott mcdaniel say:
I prefer the Transaction table method outlined by ScottG as well. All data is centrally located, and you can query that table much more easily than you can separate tables.

jay freedman say:
You can use the following macro. See http://www.gmayor.com/installing_macro.htm if you need to know how to install a macro.

Sub MultByOnePointFive()
    Dim tbl As Table
    Dim cel As Cell
    Dim rg As Range
   
    For Each tbl In ActiveDocument.Tables
        For Each cel In tbl.Rows(1).Cells
            Set rg = cel.Range
            rg.MoveEnd wdCharacter, -1
            If IsNumeric(rg.Text) Then
                rg.Text = Format(CLng(rg.Text) * 1.5, "0.0")
            End If
        Next
    Next
End Sub

jay freedman say:
When the PDF was converted to a Word document, it may have created what looks to you like "tables" but that aren't what Word considers to be "tables" (these are a special type of object). To find out, put the cursor inside one of the tables. If the Table Tools tab appears on the ribbon, then it really is a Word table, but if that tab doesn't appear then what you have is just text. The macro only works on Word tables.

If it's text, then the next question is what invisible characters are separating the numbers -- are they tabs, or space characters, or something else? Click the ¶ button on the Home tab. If you see little arrows between the numbers, those are tab characters. If you see tiny dots, those are spaces. You can click ¶ again to turn off that display.

It is possible to change selected text to a Word table, by clicking the Table button on the Insert tab and clicking Convert Text to Table -- but you need to know what the separator characters are, and you need to have only one separator character between each pair of numbers. Also, you'd probably have to do the conversion manually -- I wouldn't trust a macro to be able to figure out which lines should become tables and which ones to leave alone.

Sorry about the formula idea -- Word doesn't work the way Excel does, so that's a non-starter.
