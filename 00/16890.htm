ossiemac say:

If I am interpreting your question correctly, the code below creates combo boxes in the cells and aligns the size and position to the cells.

Note that if you run it more than once for different columns then you will need to initialize i to the next number after the previous set of combo boxes created because you will run into problems later if you duplicate combobox names. (The code does not error; it creates the duplicate names). You can get the last number used by right clicking the last combobox while in Design mode and read it at the left of the formula bar or in the combo properties.

Sub CreateComboBoxes()
    Dim oleObj As OLEObject
    Dim cel As Range
    Dim dblLeft As Double
    Dim dblTop As Double
    Dim dblWdth As Double
    Dim dblHt As Double
    Dim i As Long
   
    i = 1       'Initialize for first combobox name
   
    With Worksheets("Sheet1")   'Edit "Sheet1" to your worksheet name
        For Each cel In .Range(.Cells(2, "A"), .Cells(100, "A"))    'Edit range to required range
            dblLeft = cel.Left
            dblTop = cel.Top
            dblWdth = cel.Width
            dblHt = cel.Height
            Set oleObj = .OLEObjects.Add(ClassType:="Forms.ComboBox.1", _
                    Link:=False, _
                    DisplayAsIcon:=False, _
                    Left:=dblLeft, _
                    Top:=dblTop, _
                    Width:=dblWdth, _
                    Height:=dblHt)
                   
            With oleObj
                .Name = "ComboBox" & i  'Edit name as required
                .LinkedCell = cel.Offset(0, 1).Address(External:=True)

               

                'Edit as required. Single quotes must be used around worksheet names containing spaces.

                .ListFillRange = "'Sheet 2'!A1:A10"   
               

                 i = i + 1
               
            End With
           
        Next cel
    End With
End Sub

I should have added a little more information. If you want to set more properties then you do so in the With oleObj section. However, when you type the leading dot, if you can't see the property that you require in the intellisense dropdown then type .Object. and you will probably find it.

Example to add Font size insert the following line in the With oleObj section. (Note the leading dot)

ossiemac say:

Hello again David,

You can make a table containing 2 columns; the first column containing the last 20 or so significant characters of the longer string and the second column containing the full string.

Assign the first column (with 20 or so characters) to the Combo ListFillRange.

Assign the cell under the combo box as the LinkedCell. (You will not see this value because it is hidden by the combo box.)

In the cell to the right of the combo (the cell that you were previously using as the linked cell) insert the following Vlookup formula to return the full string.

=VLOOKUP(A2,'Data Entry'!$A$1:$B$26,2,FALSE)

Explanation of the above formula:

A2 is the linked cell under the combo

'Data Entry'!$A$1:$B$26 is the table with column A containing the 20 or so characters and column B containing the full string. (Note the table array must be in absolute format with the $ signs)

2 is the second column of the table array to return (Full string)

False says only return if match found. (If no match then returns error #N/A.)

Vlookup always looks in the first column of the table array for the value being searched and that is why the last 20 or so characters must be in the first column of the table.

If you do not want to see the #N/A error returned against combo boxes that have not yet got a value selected then you can use the following formula that will insert a zero length string if not found.

=IF(ISERROR(VLOOKUP(A2,'Data Entry'!$A$1:$B$26,2,FALSE)),"",VLOOKUP(A2,'Data Entry'!$A$1:$B$26,2,FALSE))

Note: When line breaks get placed in long formulas like this in this forum, I have sometimes found (but not always) that when copying them and pasting into the formula bar in Excel that they error. This is due to unseen characters that are added. If this occurs then paste them into Notepad and save the notepad file (You will get a message re content that cannot be saved but just accept it). Close the file and re-open and check for any spurious characters like question marks and delete and then copy and paste into Excel. Alternatively just carefully type in your formula in Excel.

doug robbins - mvp office apps & services (word) say:

It stops there because you are using a "named range" as the data source and the "named range" was defined as A1:E181   181 - 1 = 180, the 1 being for the Header row.

In the case of the thread to which you posted the link, the OP (ArtieParty)stated:

"When I execute the mail merge, the merge stops at 180 pages, which is only 45 rows merged."

In his case, the 180 = 45 x 4  which while it gives the same result of 180, is a completely different situation.
