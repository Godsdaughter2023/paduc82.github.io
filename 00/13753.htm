Update your Microsoft Office products before installing KB890830.  Go to Windows Update, click the bottom bit where it says get updates for other Microsoft products, click accept and install in the browser popup.  Exclude KB890830 and run all updates.

There's some C++ library update from a couple years ago, and maybe one of the recent Windows 7 updates is involved.  If you install the April 2014 KB890830 malicious software removal update BEFORE patching your other Microsoft software it will break Office 2013.  Maybe not immediately - today I found a workstation that had taken KB890830 nearly 20 days ago and today Office 2013 stopped working.  It seems to be some corruption in UAC, it spawns invisible consent.exe prompts in the background when Office apps are launched.

But if you restore to prior KB890830 and install the other Microsoft product updates first you're golden.
