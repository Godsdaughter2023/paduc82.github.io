noyesae say:

Conditional Formatting a Range Dates in Months.

Hello

When January rolls around and I update the spreadsheet with new data and January 1 I want this to happen automatically as this will be my Master file.

 

Any help I can get would be greatly appriecated and I hope I explained that clearly enough.

 

 

jeeped say:
You will have to set up a series of conditional formatting rules. Each will be based upon a formula that uses the following to get the current first-day-of-the-month, 

=DATE(YEAR(TODAY()), MONTH(TODAY()), 1)

That will give you Dec 1, 2013 'cause that is the first day of today's month. Constructing a TRUE/FALSE formula to show TRUE for April 2013 would be something like,

=AND(A1>=EDATE(DATE(YEAR(TODAY()), MONTH(TODAY()), 1),-8),A1<EDATE(DATE(YEAR(TODAY()), MONTH(TODAY()), 1),-7))
... or,
=AND(A1>DATE(YEAR(TODAY()), MONTH(TODAY())-8, 0),A1<DATE(YEAR(TODAY()), MONTH(TODAY())-7, 1))

Both of those perform essentially the same action. They show TRUE for any date that is between 01-Apr-2013 and 30-Apr-2013. Next month the TODAY() portion will shift the TRUE to any date between 01-May-2013 and 31-May-2013. You will need a new CF rule for each month that you want to highlight but these will all adjust automatically next month.

DATE function Returns the serial number of a particular date
YEAR function Converts a serial number to a year
MONTH function Converts a serial number to a month
DAY function Converts a serial number to a day of the month
EDATE function Returns the serial number of the date that is the indicated number of months before or after the start date
AND function Returns TRUE if all of its arguments are TRUE

scottlaing say:

Access Formatting: Need format to display either Date or “0”. 0 keeps displaying as 12:00:00AM, how to fix?

I need to make it so that the one field either displays the Date or the value of 0. They're two separate formats, and so I can't get Access to mix the two up. Logic functions keep displaying errors for me.

scottgem say:

Essentially you can't. You have to understanding how Access stores Date/Time values.  They are stored as a double precision number where the integer portion is the number of days since 12/30/1899 and the decimal portion is a fraction of a day (i.e. .25 = 6 AM). So if you assign a value of 0 to a Date/Time field, it will display as 12/30/1899 12:00:00 AM. If you only display as time then it will be midnight. 

You can however get around this by displaying the time as a Text string:

=IIF([datefile]=0,"0",Format([datefield],"mm/dd/yy hh:mm")

as an example. 

ken sheridan say: Actually, you can do this with the Format property.  To format a date in UK format of dd/mm/yyyy or as 0 if the underlying value is zero (31/12/1899 00:00:00) the Format property would be:

dd/mm/yyyy;;\0

Domain Name change - Microsoft Community

markcaspers say:

Domain Name change

Our company recently changed domain names (email addresses to be specific) how do we change our Yammer access to reflect the new email address and how do we change the "Invite" option page to list our current email address. 

robbie e d'souza say:

Hello Mark,

I would like to share the below information with you.

Example:
You have a free yammer network with Abc.com domain. When you add the domain in Abc.onmicrosoft.com office365 tenant it will get synced in yammer, however it will upgrade the network from a Free to Enterprise yammer network. To make sure that all users have Abc.com in their email address in yammer please follow the below steps.

1) Set the default domain in Office365 as Abc.com (Only domain managed by Office365)

Please find the support article for Default domain in Office365.

https://docs.microsoft.com/en-us/office365/admin/setup/domains-faq?view=o365-worldwide

2) This will change the Yammer URL to Abc.com
3) Under Invite user option in Yammer the default domain will be Abc.com
4) If you want to change the email address of the current user from allansmith@xyz.com to allansmith@abc.com

Steps

1) Do an Export User to get the email aliases of the user, by Clicking on the gear icon on top left > Network Admin > Export Users > All Users then open the export file and copy the email address of the users with Xyz.com domain in your Yammer network.
2) Now, You'll need to create a CSV (comma separated value) file with the user account information. 
Your CSV file should be formatted as a table and must include a header, or first line, that defines the fields in your table.
    Note: If you use Office 365 or Yammer's Single Sign On Integration, your users' passwords are no longer managed by Yammer. Please do not include passwords in your CSV.
    Example:
     
    Example:
    Action    Email Address           New Email     
    Update    tgraves@xyz.com      tgraves@abc.com 
    In Microsoft Excel, you can save a spreadsheet as a CSV file by going to File > Save As and choosing Comma Separated Values (.csv) from the Format dropdown.
     
3) Now Login as a verified admin in Yammer     
4) Click on gear icon on bottom left
5) Go to Network Admin and go to Bulk Update Users.
6) Upload the CSV file and click on Bulk Update Button.

This will change the email address from allansmith@xyz.com to allansmith@abc.com in yammer.

If you have any questions or concerns, please let me know.
