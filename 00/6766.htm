rob koch say:

Nick,

Assuning you are talking about an executable application, the proper way to identify yourself to all current antimalware products is by using a certificate to digitally sign both any executable portions of the program and also the installation package if one exists that contains your entire program.  This digital signature identifies the application as coming from your organization and also allows it to be checked for modification.

 

Along with the above, the signature allows the more recent Internet Explorer browser versions to identify your executable during a download, which the newer SmartScreen Filter uses to provide reputation information to those who download the file.  So this system is effectively the whitelist you seek, though it is more automatic and not a 'submission' system as you expected, since there are literally millions of such files floating around at any given time.

 

Rob

Hello,

Thank you for keeping us updated.

I suggest you to try clearing the OneNote notebook Recycle Bin and check if the Notebook sync as expected. To do so, Open OneNote, click on History> Notebook Recycle Bin> Empty Recycle Bin> Yes. Try syncing the notebook and check if issue persist.

Note: Clearing the OneNote Recycle bin will permanently delete the data stored in Recycle bin. It will not be possible to recover deleted data.

Hope this helps. Post back with updated issue details for further assistance.
