Constrain tasks to happen x days or x weeks before key event?

I'm trying to create a project plan template. One key event date will be fixed (but editable) and the tasks will need to happen x days or x weeks before the event. I am unable to find a way to constrain tasks in this way. Any ideas? THANK YOU!
john - project say:
TZou,
There are a couple of approaches you can try. Let's say the key event is a milestone (e.g. contract award) and that you need to submit a proposal 60 days prior to that milestone. One method uses a lag on a finish-to-finish from the milestone. For example, say the milestone is task 3 and the proposal is task 2. The predecessor for task 2 would be:
3FF-60d

The second method uses a lag on a finish-to-start from the milestone. In this case the predecessor to task 2 would be:
3SF-60d

Keep in mind that if you need calendar days instead of working days, you can use the entry "60ed" (or ~8ew for weeks).

Hope this helps.

John
