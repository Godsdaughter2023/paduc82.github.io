Macro to hide columns in a pivot table

I would like to write a macro to hide a range of columns in a pivot table. The first and last columns should be visible and the interior columns should be hidden. However, there can be a varying number of interior columns when the pivot table is refreshed.

bernie deitrick, excel mvp 2000-2010 say:

Depends on what you mean by "interior columns", but code like

Sub test()
With Sheets("Pivot Table Sheet")
.Range(.Range("B5"), .Cells(5, Columns.Count).End(xlToLeft)(1, 0)).EntireColumn.Hidden = True
End With
End Sub

will hide all but the first and last column of a pivot table that starts in column A and has data fields in row 5.

suzanne s. barnhill say:

As Stefan says, it's not the heading itself that needs to be reformatted (or the heading style) but the style used for that TOC level. It isn't always easy to access TOC styles, but you can find them in the Styles pane if you set the options to display All Styles in alphabetical order. Moreover, since TOC styles are generally set to update automatically, most changes in a single TOC entry will usually be propagated to the style itself (this is more often true of paragraph formatting than character formatting, however).

For (a lot) more on TOCs specifically, see http://wordfaqs.ssbarnhill.com/TOCTips.htm

Another idea you might consider, instead of using a numbered style with a line break, is to use two styles, one for the appendix number and one for the appendix title (which allows you to format them differently). You include the title style in the TOC and apply the numbering to the TOC style for that title level. See http://wordfaqs.ssbarnhill.com/TOCTips.htm#UnnumberedHeadings

2- On the desktop, drag the Microsoft OneNote icon from the Dock to the Trash. (Skip this step if the icon isn’t docked.)

3- From the Library > Containers folder on your Mac’s hard drive, drag all files and subfolders that begin with “com.microsoft.onenote” to the Trash.

bob jones aka: cybertaz say:
I'm not sure what you mean by "the method above", but the way to go about what you want is from Insert> Slides From> Other Presentation. Be sure to use the option for Select slides to insert even if you intend to insert them all -- That provides the choice of whether the original design is retained.

Keep in mind though -- regardless of whether you insert slides from another presentation or simply switch Themes -- the only elements that will reflect the new choice are those which have not been directly reformatted in a way that deviates from the Theme previously applied. Direct formatting severs the relationship between the element & the Theme.

IOW. if the currently applied Theme calls for Arial font for Theme Headings but you reformat a placeholder to use some other font, if you then select a different Theme that calls for Times New Roman as the font for Theme Headings, the placeholder will not be reformatted. It keeps whatever font it was directly formatted to use.
