Tryt this:

 

Sub Delete2WeeksAgo()
    Dim lnLastRow As Long
    Dim rng As Range
   
    lnLastRow = Range("A" & Rows.Count).End(xlUp).Row
    With Sheets("Prints").Range("A1").CurrentRegion
        .AutoFilter
        .AutoFilter Field:=2, Criteria1:="<=" & CLng(Date - 14)
    End With
   
    On Error Resume Next 'in case SpecialCells returns nothing!
    Set rng = Range("B2:B" & lnLastRow).SpecialCells(xlCellTypeVisible)
    On Error GoTo 0
    If Not rng Is Nothing Then rng.EntireRow.Delete
   
    Sheets("Prints").AutoFilterMode = False
End Sub

Hi Frank,

After going through all the above, I think this could be an expected behavior.
The reason could be that files were deleted either in the local or in the cloud side by accident. Files stored in team site library will not be deleted automatically.

Similarly, file deletion in the cloud will result in the files disappearance in the local folder.

Hi Carter，

I’d like to answer your 3 questions one by one:

1. If A is the default global admin, A is able to have full access to Exchange Online Admin.

3. A Compliance Admin has the access to Office 365 Admin Center and Exchange Online Admin Center.
