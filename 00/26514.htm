Saving emails to an Address Book - Microsoft Community

errol99 say:

Saving emails to an Address Book

I am using Outlook 2013.  In previous versions I could save an email directly to a specified Address Book, and if the contact was already saved, Outlook would ask if I wanted to upgrade or save a copy.  This was an extremely useful option and allowed me to very quickly save contacts to a group according to their status (e.g. potential user of Product A, current user of Product B, etc).

create a 2 column table that automatically creates numbers for each - Microsoft Community

gemaschaff say:

create a 2 column table that automatically creates numbers for each line of each one of the 2 columns

I am creating a bilingual document that needs to have one language per column. The lines in each column need to be numbered so that the words in the text can be found easily. Does anybody know how to do this?

daniel g. grau say:

I'm guessing that you want to add line numbers for reference purposes.

Unfortunately, the regular line number feature (Format > Document > Layout > Line Number) does not work with tables.

Line Numbers do work with columns, but lines are numbered sequentially, first on the left and then on the right. Line numbers can only reinitiate with new pages. So that's no good either.

There are two solutions I can think of. Add two thin columns, one on each side, then:

  1. Optionally, merge the thin cells. This has the advantage of facilitating the paste operation that we'll do. Then…
  2. Insert auto-numbered empty paragraphs (left column) or…
  3. SEQ fields (right column).

This is how both look (both appear left-aligned):

I used two tables (1-24 & 25-27) just to illustrate that tables can be discontinuous and that you don't need to merge the thin columns. However, since both of these numbering methods require pasting repetitively, it's easier if they are merged.

For auto-numbering (Format > Bullets and Numbering > Numbered), when pasting the blank paragraphs, you may need to right-click on them and tell Word to "Continue numbering," particularly if you use more than one table.

The SEQ fields are inserted via Insert > Fields > SEQ. What you type in the dialog box when inserting the SEQ field is just a name for the sequence. Then, if you right-click on a field to "Toggle Field Codes," it will look like this: { SEQ Target }  (where "Target" is the name of the numbering sequence). If you use more than one sequence, the other one could be called, say, { SEQ Source }.

After you have a SEQ field, just select it plus its trailing ¶ and paste, paste, paste... or select several fields and repeat the paste until you reach the end of the document.

With either method—particularly if you use varied line spacing and/or Space Before/Space After, or various fonts and point sizes—you may need to compensate for misalignments between the line number and the actual lines, at least once per page.

I would use just one reference in the middle. After all, I can always say "line 10 of the Source Language" or "line 12 of the Target Language" and it will be clear what I'm referring to. Something like this:

By the way, the gray field shading is just a Word preference to flag fields. It is non-printing and can be turned off.

john mcghie say:

Similar to Daniel's suggestion, I would use a two-column table.

In the first column, apply the style List Number.  In the second column, apply the style List Number 2.

Provided you have a different style in each column, you will get a different series of numbers:

You can use one row per line, as I have here.  Or you can use one number per cell.  Either way, the users will be able to find things easily.

The trick is that you MUST use a style to apply the numbering: it is the style that keeps the two numbering series separate.  If you attempt to use the numbering button, you will get the numbers "continuing" in the right column instead of restarting, because Word can't differentiate between the two sets of numbers unless they have a different style.
