    guruaid say:

    Error code 0x80070005 is also known as "ACCESS DENIED." It usually occurs when you lack file or registry permissions (Because file or registry permissions may be changed by malware, we recommend that you scan for malware after you repair the permissions.)
    This error usually shows up while trying to do a Windows Update, but since you're getting the same error while doing a system restore, you can try the same troubleshooting and see if it helps.
    To resolve the issue, make sure that you are logged on as an administrator, and then try system restore.
    If the problem continues, use the SubInACL tool to repair file and registry permissions. To do this, follow these steps:
    1. Download the SubInACL tool and install it in the %windir%\system32 folder. The %windir% placeholder represents the Windows operating system folder. For example, C:\Windows.
    2. Start Notepad, and then copy and paste the following commands:

    @echo off
    subinacl /subkeyreg HKEY_LOCAL_MACHINE /grant=administrators=f  /grant=system=f
    subinacl /subkeyreg HKEY_CURRENT_USER /grant=administrators=f  /grant=system=f
    subinacl /subkeyreg HKEY_CLASSES_ROOT /grant=administrators=f  /grant=system=f
    subinacl /subdirectories %windir% /grant=administrators=f /grant=system=f

    @Echo =========================
    @Echo Finished.
    @Echo =========================
    @pause
    3. Save the file as Reset.cmd.
    4. Right-click the Reset.cmd file in File Explorer (in Windows 8.1 and Windows 8) or Windows Explorer (in Windows 7 and Windows Vista), and then click Run as administrator. In Windows XP, double-click the file in Windows Explorer.
    5. Wait until the execution finishes, and then try to restore the system again.
    Refer to this KB Article for more information:
    http://support.microsoft.com/kb/968003
