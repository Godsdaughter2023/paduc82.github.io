How do I enter dates in Excel? - Microsoft Community

davidg12345 say:

How do I enter dates in Excel?

Today is April 24, 2013.  If I enter "4/24", it returns "Apr-24" and reads in the editing bar "01-04-2024".  If I enter "4/24/13" it returns the same but it has no date value.  Life shouldn't be this difficult!  My Windows 7 Regional Settings are English(Canada) which give the same date format as English(USA)
robert long [msft] say: Hi David - Thanks again for your feedback and help in resolving this. We've investigated this and the behavior you are seeing is actually correct.  Excel and Web Excel are doing the right thing.  The reason there is a discrepancy between the formats you are seeing in Excel and in Excel Web App is that the regional settings you have for the two apps are set differently. Let me see if I can explain. 

There is a concept of a 'regional setting' in Excel and Web Excel, and this regional setting is what defines your data locale.  Your data locale is what controls how things like dates and decimals are formatted. 

In Skydrive/WebExcel, your data locale/regional setting comes from your Microsoft Account.  You told us your Microsoft Account is set to Canada.  So in Web Excel your data locale is "Canada" (en-ca).  Your dates are being formatted for 'en-ca' and the default is 'dd/MM/yyyy' for en-ca.  Thats why you see '01-04-2024' when you enter '4/24' in the browser.  That is the correct behavior.  

In Excel, your data locale/regional setting comes from the Operating System.  In the screenshot you emailed me of your regional settings, I can see that your data locale/regional setting is set to English(Canada) but that you modified the date format to be 'M/dd/yy'.  That is not the default for en-ca.  For en-ca the default is 'dd/MM/yyyy' which is what you see in Web Excel. You customized the date format to be different from the default for en-ca. That is why you see Excel and Excel Web App giving you different formats for the same entry.  They are both behaving correctly. 

You can reset the date format in Excel client to be the default for Canada of 'dd/MM/yyyy' but I suspect what you want is 'M/dd/yy' in both Excel and Web Excel, based on your comments. This means you need to change the data locale/regional setting in Web Excel. Unfortunately there is no great way to do this.  We get a fair amount of feedback about this and we are aware of the issue.  I personally, as well as a few other of my colleagues, am working to make this experience better. 

Until we make the experience better the way to change the data locale/regional setting in Web Excel is to change your Account settings for your Microsoft Account  (http://account.live.com), under Personal Info, from your current Country/Region of 'CANADA' to a different region that has the date format you want. I believe if you change it from 'Canada' to 'United States' you will get the 'M/dd/yy' behavior you want. 

We realize this is not a great solution and not very discoverable or intuitive, and we are working to make it better.  (Also, more than a few of us are Canadian here on the team :) )

Thanks for using Excel Web App and please keep sending us your comments and questions!

Rob

joeu2004 say: Josh wrote:
It appears that in 2013 Excel the XOr functionality is broken? For some reason two positive values are returning a third positive value.  Could someone verify that this functionality is or is not correct? My understanding of exclusive or is that it should return false unless one of the values is exclusive from the other.

I don't know where you are finding the description "exclusive from [each] other", but you are interpreting that too literally.

The Excel (per se) XOR function in XL2013 returns a boolean value, TRUE or FALSE.  As the online help page explains (click here) [1]:  "The result of XOR is TRUE when the number of TRUE inputs is odd and FALSE when the number of TRUE inputs is even" (including zero).

Excel treats any non-zero numeric value as TRUE.  The value of TRUE is 1.

So XOR with an even number of (non-zero) positive values returns FALSE (0).  XOR with an odd number of positive values returns TRUE (1).

The VBA (per se) Xor operator performs a bitwise operation.  Considering corresponding bits of the two operands, Xor is a 1 bit when one bit is 1 and the other bit is zero; otherwise, Xor is a 0 bit.

In VBA, the value of True is -1 (all ones); but any non-zero arithmetic value is treated as True in a boolean expression.

So the Xor of two (non-zero) positive values is zero (False) only when both values are the same.  Otherwise, the result will be a (non-zero) positive value.

For example, 4 Xor 7 is 3.  2 Xor 4 Xor 6 is 0.  True Xor True is False.  But 4 Xor 7 Xor True is -3 (treated as True in a boolean expression).

  1. If the error message was displayed reboot or go to task manager and kill the hanging Excel process
  2. Open Excel and disable all Excel and COM addins. Close Excel. 
  3. Open the word document
  4. Do a "save as" back to itself and let word convert the document to 2013 format
  5. Close the document 
  6. Open the document
  7. For every embedded spreadsheet
    Right Click
    Select WORKSHEET OBJECT
    Select CONVERT
    Make sure "Convert to" is selected
    Select MICROSOFT EXCEL WORKSHEET
    Click "OK"
  8. Click on spreadsheet and verify that status bar says:
    Double-click to Edit Microsoft Excel Worksheet
  9. Save the Word Document
  10. Close Word
  11. Re-enable all add-ins
