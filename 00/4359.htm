Someone will come up with a good way to do something really useful with those BIT.... functions, much more than the apparently obvious.  One or two of them will end up with some seemingly strange, but obviously useful function(s) just as SUMPRODUCT() ended up with.

Now, I would add to the question, why is the upper limit for the shift 47 and not 31 or 63?  What other numbers have a 6-byte value?  I could understand it not being 7 if it was limited to word size rather than byte size, so it seems curious to me.  Something to do with 15-digits of precision?

The online help topic for the function doesn't really suggest a 'practical' use for it.  Also interesting to note that using a negative number for the shift count is same as BITRSHIFT() with a positive number for the shift count.  Now all of that could get pretty confusing - =BITRSHIFT(A1,-3)+BITRSHIFT(A1,-1) = 10!  (where A1 holds a 1).

Actually, the help topic itself raises questions I'd rather not think about at this time of the evening:

First thing to remember/note is that both parameters must be integers, and the first parameter must be a positive integer.

BITLSHIFT(number, shift_amount)

then they hit you with these two limitations:

  • Number    Required. Number must be an integer greater than or equal to 0.
  • Shift_amount    Required. Shift_amount must be an integer.

then they hit you with these two limitations (among others):

  • If Number is greater than (2^48)-1, BITLSHIFT returns the #NUM! error value.
  • If the absolute value of Shift_amount is greater than 53, BITLSHIFT returns the #NUM! error value.

Right this minute my mind says that if you =BITLSHIFT(1,48) it's going to give you the #NUM! error, and since they don't allow decimal values for the 'number', how would you ever get to ,53?  Maybe when shifting a negative number -- oops, no because 'number' must be a positive integer.

jeeped say:

Using the cell references from my sample below, change A12 from Nov to 11/1/2013 and use a custom number format (e.g. Ctrl+1, Number) of mmm. In A13 use =EDATE(A12, 1) and fill or copy down as far as you want to. Use the same custom number formatting on these cells.

This makes the formula in B12,

=IFERROR(VLOOKUP(MAX(INDEX((MONTH($A$2:$A$9)=MONTH($A12))*($A$2:$A$9),,)), $A$2:$B$9,2,FALSE),0)

  ... or more correctly,

=IFERROR(VLOOKUP(MAX(INDEX((MONTH($A$2:$A$9)=MONTH($A12))*(YEAR($A$2:$A$9)=YEAR($A12))*($A$2:$A$9),,)), $A$2:$B$9,2,FALSE),0)

The latter one makes sure that the years match as well as the months.

More information on MAXIF functionality is available through either of my local Wiki articles.
