Petition to get rid of the Office SDI, or allow users to choose - Microsoft Community

michael.hutchens say:

Petition to get rid of the Office SDI, or allow users to choose between SDI and MDI

From Office 2013 onwards, Microsoft moved from a multiple document interface (MDI) to a single document interface (SDI), meaning that each document or workbook now has its own window.

This change to SDI has resulted in negative performance impacts to Office applications, primarily because multiple documents/workbooks must be rendered every time the screen is rendered, compared with prior versions using the MDI, which only needed ever to render Word or Excel once per instance, irrespective of the number of documents/workbooks open at the time.

The SDI also requires a lot more work to manage when developing software for Office applications, so it's a double-whammy.

Does anyone else agree with me when I say that SDI should be an option, rather than forced in Excel 2013 and Excel 2016, so that users can elect to switch it on or off?

If it was an option, power users who like to open many files at once, and/or work with Office add-ins that are negatively impacted by the SDI, can simply switch to MDI mode and avoid all the pain.

Very interested in everyone's position on this, including Microsoft's position.

---

Footnote: A simple example of the significant impacts of the SDI change is running the below TestOpenClose macro on both Excel 2010 (MDI) and then Excel 2016 (SDI). When run on Excel 2016, it takes 4 times as long without custom ribbon items, and 15 times as long with custom ribbon items:

Option Explicit

Private Declare Function GetTickCount Lib "kernel32" () As Long

Sub Test_Open_Close()

    Dim lngStart As Long
    Dim lngDuration As Long

    lngStart = GetTickCount
    Test_Open
    Test_Close
    lngDuration = GetTickCount - lngStart

    Call MsgBox("Duration: " & lngDuration & "ms", vbInformation)

End Sub

Sub Test_Open()

    Dim nWorkbookNumber As Long

    For nWorkbookNumber = 1 To 25
        Application.Workbooks.Add
    Next nWorkbookNumber

End Sub

Sub Test_Close()

    Dim nWorkbookNumber As Long

    For nWorkbookNumber = 1 To 25
        Call Application.ActiveWorkbook.Close
    Next nWorkbookNumber

End Sub

UseOnlineContent value

Description

3. HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Common\Identity\Identities .. is where it stores saved Office profile and credentials.
