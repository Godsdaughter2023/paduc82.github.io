Step 1. Third-party antivirus

Do you use antivirus? There are some antivirus, such as Avast, Bitdefender, Eset Nod32, and perhaps others, which generate this problem in the Internet connection. If so, uninstall it and restart the system. If the Internet connection works well when restarting, and there are no errors, reinstall your antivirus and check again. If the connection error appears again, look for another antivirus.



Step 2. Run the network connection troubleshooter

If the antivirus was not the cause of the problem, follow these steps:

1. In the search bar write solution and click on the search result Troubleshooting (Control Panel)

2. In the column on the left click on See all

3. Click on Internet Connections

4. In the Internet Connections box, click Advanced Options and Run as administrator

5. Click Next and follow the instructions on the screen to complete the problem resolution process.



Step 3. Reset the network configuration

If the network connection solver did not solve the problem, follow these steps:

1. Press the Windows + X keys and select Command Prompt (Administrator)

2. In the console type the following commands and press Enter after each of them:

netsh winsock reset
netsh int ip reset
ipconfig / release
ipconfig / renew
ipconfig / flushdns
ipconfig / registerdns

3. Close the CMD console and restart the system.

4. When restarting, check the Internet connection and if the problem has been solved



Step 4. Check network services.

Check the following network services from the service console. All these services must be enabled and running:

1. Press the Windows + R keys

2. In the Open box type services.msc and press Enter:

3. In the local services console, look for the following services:

- Windows Connection Manager (Wcmsvc)

- NetBIOS auxiliary application over TCP / IP (Imhosts)

- DHCP Client (Dhcp)

- DNS Client (Dscache)

- Automatic configuration of wired networks (dot3svc)

- Automatic WLAN configuration (WlanSvc)

- SSDP detection (SSDPSRV)

- UPnP host device (upnphost)

- Publication of function detection resource (FDResPub)

Although in some network installations it is not necessary for all of them to be configured in automatic startup, to ensure that no service fails, enable them in Automatic startup and start the service. To do this, right click on each one of them and choose Properties. In Startup type, choose Automatic and in Service status, click Start if it is stopped.

4. To ensure correct operation, reboot the system and see if the services are kept automatic and initiated.

5. Check the Internet connection and if the problem has been solved

elohmn say:

A little confusing but both. The Console the Content (any Content, Game, Game Add-on and such) will have a License tied to that Console for any Profile. But the Content must still be there to use and the License must be up to date. The Profile that purchase the Content will always have Access to that Content by DL'ing it again at anytime via the Download History.

You can simply DL everything again (at no cost provided it is with the same Profile that purchased it) on any Console. To remove the License from any prior Console (Console ID) you can complete a License Transfer. This removes the "old" License from the prior Console and adds it to the "new" Console after the Content is DL'd again (and the License Transfer). With the License on the "new" Console (and Content as well), any Profile on that Console will be able to play that Content.

rajesh govind say:

Hi Kaylie,

 

The error that you have mentioned in the issue description need not be just due to the disk errors on the computer. It can also be due to multiple other reasons by the corrupted system files and conflicts from the third party security programs on the computer.

 

I would like to know if you have any third party security program installed on the computer.

 

When you perform a System Restore, it brings changes to many files on the computer. Thus, some security programs prevent the changes to their files and creates conflicts with the System Restore process.  So, if you have any third party security program installed on the computer, then you may first disable it temporarily and check if the issue persists. I request you to contact the technical support for the security program to do this and then check if the issue still persists.

 

Important: Antivirus software can help protect your computer against viruses and other security threats. In most cases, you shouldn't disable your antivirus software. If you have to temporarily disable it to install other software, you should re-enable it as soon as you're done. If you're connected to the Internet or a network while your antivirus software is disabled, your computer is vulnerable to attacks.

 

If that doesn’t fix the issue, then you may run a Check Disk on the computer to see if this is caused by any corrupted/damaged sectors on the disk or due to some other disk errors.

 

Run Check Disk in Command Prompt

 

To run “ChkDsk” utility using Command Prompt, you can follow these steps:

 

a)       Press “Windows Logo” + “X” keys on the keyboard and select “Command Prompt (Admin)” from the list.

b)       In the command prompt window, type “chkdsk /r” and press “ENTER”.

c)       Answer “Y” when “chkdsk” asks you if you want to check the drive next time computer boots (restarts).

 

Important: While performing Error-checking or ChkDsk on the hard drive, if any bad sectors are found and when attempting to repair that sector, the data available on that sector might be lost.

 

Please get back to us with the results. If the issue still persists, we’ll be glad to assist you further.
