    KB4052623 (v. 4.16.17656.18052) Install Fail - 0x80070643: - Microsoft Community

    ausgumbie say:

    KB4052623 (v. 4.16.17656.18052) Install Fail - 0x80070643: Possible workaround?

    I've seen a number of threads about "Update for Windows Defender antimalware platform - KB4052623 (Version 4.16.17656.18052)" failing to install (Error 0x80070643).

    About a week after the 2018-05 Cumulative Update for Windows 10 Version 1803 for x64-based Systems (KB4100403), I noted the above message. Online, I got some ideas I turned into this workaround. It worked in my circumstances, and I post it in the hope it may work for others.

    Here's the dreaded message as first seen in Windows 10 Settings > Update and Security > Windows Update > View Update History:

    And here's what I did ...

    Step 1: Open Windows 10 Settings:

    Step 2: Go Windows 10 Settings > Update and Security:

    Step 3: Go Update and Security > Windows Security:

    Step 4: Go Windows Security > Virus & threat protection: Left click Virus & threat protection.

    Step 5: For me, a popup said the page wasn't available (I imagine thanks to my AV, Norton Security Premium v. 22.14.2.13). Left click on "OK" button.

    Step 6: On my clicking "OK", this view appeared. Left click Virus & threat protection where indicated, not in the left column:

    Step 7: This view appeared. Left click the down-arrow to the right of Windows Defender Antivirus options:

    Step 8: This view appears. Left click Periodic scanning to toggle it on:

    Step 9: Periodic scanning stayed toggled on. However, Norton immediately threw up a security alert with an invite to "Disable Limited Periodic Scanning". Since I had no intention of performing a Windows Defender scan, I proceeded to the next step.

    Step 10: I closed the alert box by left-clicking on the little "x" in the top-right corner.

    Step 11: I left-clicked on Virus & threat protection updates where shown:

    Step 12: I left-clicked on Check for updates. NOTE*: For some reason (I can't remember) I checked for updates twice. This, I think, was to my advantage. See later.

    Step 13: Going back to Windows 10 Settings > Update and Security > Windows Update > View Update History, I now saw this.

    Notes:

    (a) KB2267602 (with later definitions I guess - 1.269.1145.0) at first had failed to update, then - presumably on my second check - successfully updated.

    (b) KB4052623 (Version 4.16.17656.18052 - what this had all been about in the first place - had now successfully updated.

     

    Step 14: I navigated my way back to the Virus & threat protection view to toggle off Windows Defender's Periodic scanning:

    Step 15: In the same view as 14, I left-clicked on View antivirus providers further up the page. Note, that - depending how you navigate to this view, that line might appear as "Manage providers". But, left click on that anyhow, it gets you to the same place - namely, Control Panel > System and Security > Security and Maintenance. Then, left click on the down-arrow to the right of Security. Scroll down and left-click on View installed antivirus apps. (Sorry, I didn't make a screenshot for the last couple of instructions). This step is just to let you confirm Windows Defender is off.

    Well, that's it such as it is. I hope it helps someone.

    Cheers

    HOW TO DECODE EDB FILES - Microsoft Community

    fredswthrt say:

    HOW TO DECODE EDB FILES

    I have purchased a new computer that has Wiindows 7. My older computer had Windows XP and went on the fritz so I am having difficulties getting my e-mail addresses off of my old backup. Any additional help will be appreciated.
    ¡firedog say:
    It looks as if the OP is referring to ESENT database files such as those used to store Windows Live Contacts (and the Windows Search index, and other Windows bits and pieces). The problem with backing these up is that the database consists of a main file, in this case contacts.edb, along with several helper files that are essential to being able to 'decode' the data the main file contains. Some of the helper files may be in a sub-directory. Some of them have the extension .log, because they log transactions as they happen. Those transactions will in normal circumstances eventually be written to the main file. Many backup programs ignore .log files by default, similarly .tmp and.chk, both of which are sometimes used to designate essential helper files for an ESENT
    Woanware has produced a free ESE database viewer which should be able to extract the data from a complete contacts database if all its helper files are present. Another geekier one is under development atCodeplex.
