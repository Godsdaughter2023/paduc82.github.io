scott mcdaniel say:

The LACCDB file is a lockfile, and is created each time you open the .accdb file. If you've closed out of the database, that file should be deleted by Access. If it's not deleted, go ahead and try to delete it. If you cannot delete it, then you may need to restart your machine to do so.

As Sandro said, it sounds as if you may have some corruption in your database. Importing to a new db may work, but if you have a corrupt form/report then you may not be able to do so. I recommend the following to deal with possible corruption AFTER you make a backup of your database:

1. Compact the database

2. Compile your VBA code - from the VBA Editor, click Debug - Compile. Fix any errors you find, and continue doing this until the Compile menuitem is disabled.

3. Compact again

You may also need to decompile your code. To do that, create a shortcut with this as the Target:

"full path to msaccess.exe" /decompile

Run that shortcut, then open your database in that session of Access. Access will decompile the database. After doing that, run the 3 steps above again.

Finally, as Sandro suggested, create a new, blank database and import everything into that new database. As noted previously, if a Form or Report is corrupt you will not be able to import it, and would have to restore it from a known good backup or recreate it entirely.

Also be sure that your Windows and Office installations are fully up to date. Do NOT trust the automatic Windows Updater to do this for you - actually check to be sure all Updates have succeeded.
