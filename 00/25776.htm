SEQ Numbering- skip numbers/letters? - Microsoft Community

laurel :) say:

SEQ Numbering- skip numbers/letters?

I was wondering if there was a way to skip a specific number of letters in the SEQ field.

For example, in my document I have a set of four appendices ("Appendices B through E") that I want to keep together.

Is there a way I can type it into the field to skip three numbers without having to type \r "5" for "Appendix E"?


Thanks!

jay freedman say:
No, the \r (Restart) switch is the one and only way to skip numbers/letters in a series of SEQ fields.

I don't understand, though, why you're using SEQ fields in an expression like "Appendices B through E". I would expect that you have SEQ fields in the heading paragraphs in each of the appendices, and that you should use cross-references to those paragraphs in the expression. If one cross-reference points to Appendix A and the other points to Appendix E, then nothing needs to be skipped or restarted.

ryan t evans say:

This is actually a great question -- especially if you want to introduce the appendices (which may not be part of the .docx) together as a group, and later want to cross-reference to one or more of them individually.  This is very common in the legal context.

The best/only solution that I've come up with is to place each SEQ field in your expression, and hide the intervening fields.  When drafting, it would look something like this: "Appendices {SEQ} {SEQ} {SEQ} {SEQ} through {SEQ}", with the underline/italicized font being hidden.

a. user say:

FWIW another way to hide a SEQ field result is to use the \h switch, e.g.

Appendices { SEQ app }{ SEQ app \h }{ SEQ app \h }{ SEQ app \h } through { SEQ app }

[I generally find it easier to work with fields when the field codes themselves are always visible, which unfortunately isn't the case with some field types (e.g. TC, XE)]

stefan blom say:
For assistance with heading and caption numbering in appendixes, see http://www.shaunakelly.com/word/numbering/numberingappendixes.html.

ryan t evans say:

The /h switch is okay if you are always going to refer to the group of appendices together. 

However, in my work, we often introduce a group of appendices together and later on in the document want to refer to one or more of them individually.  This is where the /h switch fails me.  I use the bookmark and cross-reference functions to refer back to the intermediate references.  But if you use the /h switch, the bookmark "text" is always hidden, and the cross-reference does not display anything.

Even with hidden text, you can still see the field codes (or their result) using the show/hide button, or adjusting your display options - so that shouldn't be a drawback.  I even set up a unique style (with green font) to help distinguish what is and isn't hidden in my working document.

There may be another way to accomplish this - just none that I am aware of.
