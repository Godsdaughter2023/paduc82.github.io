How to avoid prompting "enable macro" twice? - Microsoft Community

joeu2004 say:

How to avoid prompting "enable macro" twice?

I got Excel into a strange state.  I don't know how; ergo, I don't know how to reverse it.

All of a sudden, when I click on a file icon to open an Excel file with macros, I am prompted twice to enable macros (medium security).

More specifically, if I launch Excel (new workbook), add a dummy macro (Sub doit() / End Sub), and save Book1.xls, when I subsequently click on Book1.xls, I get two enable-macro prompts.

I suspect this started after I was experimenting with Advanced File Types under Tools > File Options in a folder.  I did not notice the problem then because I might not have clicked on a file icon for Excel file with macros.

Note:  I suspect the root cause, opening the Excel file twice, might exist even for Excel files without macros.  It might simply not be apparent without the enable-macro prompt.

Also note:  I do not get the double enable-macro prompts when I open the file within Excel, e.g. clicking on the file name in the list of recently-accessed Excel files.

Sadly, I am not sure what the Advanced File Types were before my experiments.  Currently, for XLS, they are:

new:
   action:  "C:\...\EXCEL.EXE" /e /n %1
   DDE message:  [new("%1")]

open:
   action:  "C\...\EXCEL.EXE" /e "%1"
   DDE message: [open("%1")]

It would not surprise me if the %1 reference in the action line is redundant and the root cause of the problem.  It does not appear in the action line on another PC.  (Older Excel 2003 updates and  maybe older MSWin XP SP3 updates.)

But if I try to remove the %1 reference, XP(?) restores them automagically, albeit without the double-quotes in the "open" action line.

That is, if I edit out the %1 and "%1" references, click OK, then click Edit again, %1 (without quotes) appears at the end of both the "new" and "open" action lines again.  The same is true if I close the File Options window, then reopen it.

If I remove the double-quotes in the "open" action line, I do not get the double enable-macro prompts.  However, I get several errors when I click on the Book1.xls file icon in a Temp folder, to wit:

C:\Documents.xls could not be found
and.xls could not be found
Settings\Joe\My.xls could not be found
Documents\Temp\Book1.xls could not be found.

Despite those errors, the Excel file is opened correctly.

Note that the full path is "C:\Documents and Settings\Joe\My Documents\Temp\Book1.xls".

That leads me to two tentative conclusions:

1. The double-quotes around %1 in the action line are needed in order for a path with interstitial spaces to be interpreted correctly.

2. The file is indeed being opened twice, once for %1 in the action line and again for %1 in the DDE message.

Based on that, the double enable-macro prompt is not surprising when %1 is replaced with "%1" in the action line.

I can work around the double-open by editing out the DDE message for "open" action, leaving "%1" at the end of the action line.

But that changes the behavior of opening two Excel files by clicking on their file icons, to wit:  each file is opened in a separate instance of Excel instead of the same instance.  Ironically, that is the alternative behavior that I was experimenting with some days ago; and it might even be the behavior that I prefer.  I am still mulling over the pros and cons.  (Which I am fully aware of and need no assistance understanding.)

But the point is:  that was not always the case.

So the question is:  how can I (least intrusively) restore Excel to its previous behavior, only one enable-macro prompt and probably opening the file only once, and opening multiple files in the same Excel instance when I click on the file icon in a folder?

I certainly want to avoid uninstalling and reinstalling the Excel product.  I also want to avoid going back to some recent Restore Point, since I'm not sure what desirable changes I might have made in the interim.  I am not averse to editing the Registry, if someone can tell me exactly what change to make.

(Changing my macro security to avoid any prompting is also not a viable alternative.)

Arguably, this might be an XP question, not an Excel question.  I will post a similar question in an appropriate XP Answers Forum.  I hate to multi-post; but I don't know how to crosspost, if that is even supported by the Answers Forum.  Nonetheless, I wonder if an Excel-savvy person might recognize and address the problem more readily than an XP generalist.

I am using Office Excel 2003 SP3, Excel version 11.8169.8172, and MSWin XP Pro SP3 last updated 3/29/2009.

I know I am behind in updates.  But all I am asking is to restore the system (XP or Excel) to a state that worked fine before.  "Before" what I cannot say for sure; but certainly some time within the last week or two, during which I have made no major changes to the system other than experimenting with the Advanced File Types.

I ____-u-me that updating XP or Excel is not necessary, unless someone can point to a specific fix since 3/29/2009 that addresses my specific problem.
