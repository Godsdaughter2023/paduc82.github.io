scottgem say:

A subform is embedded on a parent form and linked to a field on that parent form. So it is impossible to link a separate subform to another existing subform. However, its not impossible to synchronize tow different subforms.

Another limitation of a subform is embedded a subform into a continuous form subform. There are tricks but they are cumbersome. 

What I'm guessing here is that you want to select an Employee and show all the clients assigned to that employee. You then want to select one of those clients and display all the contracts associated with that client. 

So you have some choices here. You can use a Single form subform with an embedded subform for Contracts. That would be the easiest way. But if you want separate subforms another option is to is to replace the Recordsource of the Contracts subform when a Client is selected in the first subform. 

You could run some code like:

Dim strSQL AS String

strSQL = "SELECT * FROM Contracts WHERE ClientID = " & Me.ClientID & " ORDER BY ContractNumber;"

Forms!mainformname!subformname.Form.Recordsource = strSQL

tom van stiphout say:

Before I can answer that, I have to ask a clarifying question. Is this correct?

1. Each employee has many clients

2. Each client has many contracts

Assuming yes, you have a one-to-many-to-many relation. Not uncommon.

As you said, mainform is bound to employees.

Drop the clients folder on it and it creates a 1:M relation to model (1) above.

Now I'm thinking what you want is to show the many contracts for the selected client.

To do that, you "publish" the current ClientID on the mainform. Create a hidden textbox and set its controlsource to:

myClientSubformControl.Form.ClientID

(of course you'll use yourObjectNames rather than mine)

Then you drop the Contracts table on the mainform. You set LinkMasterFields to the hidden control, and LinkChildFields to ClientID.

Note that I do not actually advocate dropping tables on forms to create subforms - this is for demonstration purposes only. For Production systems I would create a (sub)form in datasheet or continuous forms view, then drop that form object on the mainform.
