ken sheridan say: A relational database works on entirely different premises to a spreadsheet.  Unlike the latter, where data are addressable by position, in a relational database data are addressable only on the basis of values at column positions in rows in tables.

You should, provided your table designs are correct, be able to do what you want easily enough by joining your tables in a query, but without knowing your table structures I can't say more than that.  Post back with details of your tables and their columns so the dog can see the rabbit.

scottgem say:

Actually yes there is a fairly simple way to do this in Access, but you have to set it up properly. Its not the way you are thinking or have your table designed. 

So first your table should look like this:

tblBonus

BonusID (PK Autonumber)

EmployeeID (FK)

BonusPeriodID (FK)

BonusAmount

And you should have a table like:

tluBonusPeriod

BonusPeriodID (PK autonumber)

BonusQtr

BonusDate

Then you have a form in continuous form mode. In the Header of that form you have an unbound combobox to select the Bonus period (more on that in a moment). So you select the period you are entering bonuses for from that unbound combo. In the Control bound to the BonusPeriodID you set the Default value to the unbound combo. So as you add each new record, the Period automatically populates.. You just need to select the person and enter the amount. Before you start on the next period, just change the period in that unbound combo.

The combo should have the following relevant properties:

RowSource: SELECT BonusperiodID, BonusQtr, BonusDate FROM tluBonusPeriod

Bound Column: 1

Column count: 3

Column Widths: 0";1";1"

Note that you don't need both qtr and date since the date is dependent on the quarter, so you can pull the date from tluBonusPeriod when needed.

Access is not a spreadsheet, its a relational database so tables need to be designed relationally. And you need to put aside the spreadsheet mentality.
