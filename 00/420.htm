pete_uk say:

Suppose they are currently in columns C and D, with other data in columns on either side. Insert a new column E, and put this formula in E1:

=C1 & " " & D1

and copy this down by double-clicking the fill handle (the small black square in the bottom right corner of the cursor). With those formulae still selected, click on <copy>, then right-click on the selected cells and choose Paste Special from the drop-down. Check against Values and then click OK followed by <Esc>. This has now fixed the values of those formulae, so you can now delete the original columns C and D.

Hope this helps.

If you’re having this issue with a third-party AV app installed, then this might be essentially the same issue with Defender's definition updates that’s commonly associated with Error 0x80070643. Be advised that the only person in this forum who’s fully qualified to interpret mpsigstub.log files is CJ. Eichholz [MSFT], and he’s already analyzing submissions for the similar issue with Error 0x80070643 in this discussion thread:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_updating/definition-update-for-windows-defender-kb2267602/55510bcc-61b9-49cf-ba1a-e4e2bd083975

I’m taking my cue from another thread where CJ. Eichholz [MSFT] suggested that the root cause of the Error 0x80070643 issue might be an incorrect value for the DisableAntiSpyware DWORD in the Windows Defender registry key. That struck me as very likely, because I had just been able to reproduce this issue while looking for the cause of Error 0x8007139f when trying to start the Windows Defender Service in the UI. I was frankly pretty amazed to see that Error 0x8007139f when starting the service in the UI; Error 577 when starting the service in services.msc; and Error 0x80070643 for failed definition updates could all be reproduced by just putting a 0 in place of 1 for the value of the DisableAntiSpyware and DisableAntiVirus DWORDs in the Windows Defender registry key:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_updating/windows-10-desktop-pc-kb2267602-definition/d5b551bc-bf40-46d8-a05e-924d33882d6d

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/how-do-i-reinstall-windows-defender-for-windows-10/76c37289-9896-4592-a302-1e819627ad9c?page=3#LastReply

So it seems plausible that we’ll eventually be able to subsume some other error codes under this relatively simple issue, which now appears to have an extremely high incidence.
