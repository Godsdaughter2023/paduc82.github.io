First, I would use the Before Update event instead of After Update. Then I would do it this way:

If Not IsNull(DLookup("[BOP__]", "Employee Database", "[BOP__] = '" & Me.[BOP__] & "'" ) Then
Cancel = True
If MsgBox("This is a duplication. Do you want to bring up this record?, vbYesNo) = vbYes Then

DoCmd.RunCommand acCmdApplyFilterSort
Else
Me.Undo
End If
End If

This will check if the record exists. If it doesn't it will accept the entry and allow you to continue entering the new employee data. If it does exist, it will prompt you to go to that record or clear out the ID.

However, if I filter the list to a specific date the countif formula is still showing me the results for the entire spreadsheet, not what I filtered to. amit tandon say:

You may use below formula which will NOT count hidden values:

=SUMPRODUCT(--($D$2:$D$202="MT"),--($C$2:$C$202=B2),SUBTOTAL(103,OFFSET(D2,ROW($D$2:$D$202)-ROW($D$2),0)))

Range $D$2:$D$202 validates "MT"

Range $C$2:$C$202 validates Date per Cell B2

The formula returns the COUNT of entries (ignores hidden values) in the range $D$2:$D$202 which are "MT" & where the Range $C$2:$C$202 corresponds to the date in cell B2. Use "SUBTOTAL(3" in lieu of "SUBTOTAL(103" to count hidden values also.

Regards,

Amit Tandon

ashish mathur say:

Hi,

Suppose your data is in range L2:M21 - headings are in L2:M2.  Dates are in column L and text entries are in column M.  Apply an date filter and type MT in cell A1.  Enter this array formula (Ctrl+Shift+Enter)

=SUMPRODUCT(IF(SUBTOTAL(3,OFFSET(L3,ROW(L3:L21)-ROW(L3),))*($M$3:$M$21=A1),1,0))

bob jones aka: cybertaz say:
The only thing I see that will widen the columns a little is if you right-click one, select Format Data Series> Options, then set the Gap Width to 0%.

There are at least 2 contributing factors restricting the width of the columns;

  • Because of how your data is arranged, the empty cells in each column are actually being plotted as zero value columns -- you just can't see them because they have no vertical dimension [zero value] but they still occupy the same amount of horizontal space as the columns you can see, &
  • I'm not sure why, but you're including the totals as well as the values that are being totaled in the plot. That makes for more columns being plotted causing each column to be narrower.
I don't mean to be critical -- especially since I have no idea what you need to to do -- but I'd be tempted to rethink how the data is structured as well as what's actually being included in the plot... the file you posted, for example, doesn't even plot the values in column C.
