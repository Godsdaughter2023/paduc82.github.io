Release Preview Ring does not work. MSFT negligent? Who's in charge of it?

I have commented on this problem before.  The gist is this:  The RP ring does not receive builds (it isn't supposed to).  The RP ring is supposed to receive ordinary quality and security roll ups just like the current public release of Windows does (build 15063).  The only difference is that it should receive these roll ups a week or so in advance of Patch Tuesday so that RP insiders can spot problems in the updates and report them.  That is supposed to enable MSFT to catch and fix problems before they reach the general public.  That's all the ring is for.  No builds, just regular updates a few days early.

The problem is that this is not happening.  The Release Preview ring receives quality and security updates ON Patch Tuesdays with the general public.  That is useless for catching problems early enough for MSFT to fix them before they reach the general public.  Its too late.

Since quality and security roll ups come from the release service branch and not the development branch (like Slow Ring and Fast Ring builds) it is possible that MSFT is not making the service branch fulfill its responsibility to provide these updates to the RP ring as originally intended.

john korchok say:

In the first screen shot, OS X has assembled all the font variants into a common list. These might be family fonts (that would change faces by changing bold or italic attributes) or they might be single fonts, where you have to choose a different font to get bold or italic. OS X is smart enough to group them.

In the second screen shot, you've shown the 4 members of a standard (as seen by Microsoft) font family. If the font set includes a Light/Light Italic and Extra Bold/Extra Bold Italic, often those will be combined in a second family, again with 4 members. These 4 members are keyed to the bold and italic buttons in the user interface. Families have to be constructed by the foundry, the operating system can't create them.

As I mentioned earlier, if you choose a single (non-family) font, then press the bold or italic buttons, Word will fake the result, which almost always looks ghastly.

I'm not sure what you mean by the font color in the ribbon. Tracking (as opposed to kerning) is set on the Advanced tab of the Font dialog, where you can select condensed or expanded spacing in increments of 1/10 of a point.
