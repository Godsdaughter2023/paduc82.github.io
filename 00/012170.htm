So, it was probably a DNS error which probably would be resolved eventually.   What host were you trying to access?

 

A problem with the "diagnostic tool"  is that it only does a first-level lookup.   So, if aliases are involved or if the host redirects, it may not be able to detect the same symptom that you see.

 

To see an example of this, open a cmd window and do:

ipconfig  /flushdns

Then using IE try to open:

http://footprint.net/

Then switch back to your cmd window and do a

ipconfig  /displaydns

Then switch back to the IE window and use the

Diagnose Connection Problems button

See message:   Troubleshooting couldn't identify the problem

Then switch back the cmd window and do another

ipconfig  /displaydns

FWIW I see then:

<example>
footprint.net
----------------------------------------
No records of type A


footprint.net
----------------------------------------
No records of type AAAA

</example>

 

So it is blatantly obvious what the problem is:  there is no IP address in either IP4 or IP6 for that domain name.   Yet the troubleshooter thinks that all is fine because it happened to find a record for it.   ; }
