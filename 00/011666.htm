jay freedman say:
Although it would be possible to write a macro to create the entire Word document from code, it's terribly inefficient and will have you tearing your hair out before you're done.

Instead, create a Word template (*.dotx) that contains everything that will be the same in all such documents. (Just use Word to create a document, insert the necessary bits, and use the "Save as type" dropdown in the Save dialog to say you want it to be a Word Template file.) You can use either bookmarks or content controls as placeholders for most of the items you'll insert from Excel, such as the customer's name, company, and address. Your company's name and address can just be ordinary text. Insert a CreateDate field for the date; it will automatically show the date the invoice is generated. Insert an empty table for the invoice data.

Now your Excel macro will create a new invoice by telling Word to create a new document based on this template. The first thing you need to do is, in the VBA editor, click Tools > References and put a check mark next to Microsoft Word Object Library. Then you can set up some code along these lines:

Sub WordDoc()
    Dim wd As Word.Application
    Dim invoice As Word.Document
    Set wd = New Word.Application
    With wd
        .Visible = True
        Set invoice = .Documents.Add(Template:="<path>\<name of Word template>.dotx")
        '
        ' Here, transfer information from Excel to bookmarks and table cells in the Word document
        '
        invoice.Save  ' displays Save As dialog
        invoice.Close
        .Quit
    End With
    Set wd = Nothing
End Sub

I'll note that your description of the table and data is somewhat confusing: You have a table of 7 rows, but the Excel worksheet has 20 rows; the Word table has 2 columns but you're only putting data in column 2, so what goes in column 1?; etc. I can't recommend a method for transferring the data from Excel to Word until those things are resolved.

You really shouldn't use formfields and content controls in the same document. Use one or the other.

albert d. kallal say:

You code does not set the value of the text box on the form.

 You also don’t want nor need to use a attachment type column, but just use a regular “text” column. There are many reasons for this, but the simple story is that you want to save the path to the file in your database, not the actual file (so you don't need nor want to use attachment column from Access - just a plane jane regular text column)

The code behind your button to select a file should look like this:

    Private Sub Button1_Click(sender As Object, e As EventArgs) Handles Button1.Click

        Dim f As New OpenFileDialog

        With f

            .Title = "Please Select a File"

            .InitialDirectory = "F:\AMR"

            f.RestoreDirectory = True

            If f.ShowDialog() <> Windows.Forms.DialogResult.OK Then

                ' user canceled - so exit code

                Exit Sub

            End If

        End With

        ' user selected a file - put file name into text box

        Me.TextBox1.Text = f.FileName

    End Sub

And you can also place a button on the form to “launch” and open that file with this code behind another button (or adopt your double click code idea).

Eg:

    Private Sub Button2_Click(sender As Object, e As EventArgs) Handles Button2.Click

        Process.Start(Me.TextBox1.Text)

    End Sub

So for above you don’t need to drop into the form any filedialog control or some such - it not required and is just messay anyway.

So a simple text box on the form, and then a button with the above filedilog code will achieve the simple result of the file name the user selected being placed into a text box on the form.

Now of course the above is a “un-bound” form without the code to save the values of the text box to the database, but how you do that works much the same for all vb.net applications. So I am sure you have "many" text boxes on your form, and how you now are saving such values to the database will "continue" to work and now save the above text box info along with all your other text boxes that you have on the form.

So part#1 – select a file, put selected file into a text box will work above just fine.

As to how you plan to “save” the various text boxes you have on that form to a database is a VERY different question and process compared to above question.
