Sub PDF_Me()

Dim fd As FileDialog

Dim pr As PrintRange
Set osld = SlideShowWindows(1).View.Slide

End Sub

The command cannot be performed because a dialog box is open. Click - Microsoft Community

mr.mtlz say:

The command cannot be performed because a dialog box is open. Click OK, and then close open dialog boxes to continue.

Around the first of the year (2019) this would keep popping up when trying to open a Word 2007 document on my Windows 10 machine:

    "The command cannot be performed because a dialog box is open.  Click OK, and then close open dialog boxes to continue." 

I have read a lot of threads on fixing this issue and had not had any success on resolving it.  So gathering what information I read about and decided to dig in and try to come up with an answer.    This is how I resolved my problem and have things working like it used to. 

  1. I went to my local disk (C:).
  2. In the View tab, check the Hidden items.
  3. Scroll down to the Users folder. Open it, then open your user folder.
  4. Look for the hidden folder AppData, then open it.
  5. Now open the Roaming folder.
  6. Open the Microsoft folder.
  7. Now open the Templates folder.
  8. On my machine I deleted the Normal.dotm file (it had an exclamation point on the icon file it); it is a Microsoft Office Word Macro - Enabled Template. 
  9. After deleting that, close all your windows.  Everything now has been working as it used to.

Hopefully this can help someone that has the same issue.

Because you're using a third party Anti-Virus program - Norton -, the in Windows 10 in-built Anti-Virus program (Windows Defender) is automatically disabled. That's normal and how it should be. Don't try to enable Windows Defender.

If both of them would run at the same time, they would conflict with each other and none of them could do its job correctly.

If you would want to use Windows Defender, you'd need to uninstall Norton and use the

Norton Removal Tool to get rid of all remnants  (https://support.norton.com/sp/en/us/home/current/solutions/kb20080710133834EN_EndUserProfile_en_us)

graham mayor say:

I think Michael meant to post an example? However the following will do the job. The optional line eliminates attachments that are the graphics in the message, but this will eliminate wanted graphics of a similar name.

Function AttachmentNames(olItem As MailItem) As String
Dim olAttach As Attachment
Dim strAttach As String
Dim i As Long
    strAttach = vbNullString
    If olItem.Attachments.Count > 0 Then
        For i = 1 To olItem.Attachments.Count
            Set olAttach = olItem.Attachments(i)
            If Not olAttach.Filename Like "image*.jpg" Then 'optional
                strAttach = strAttach & olAttach.Filename
                If i < olItem.Attachments.Count Then
                    strAttach = strAttach & Chr(32)
                End If
            End If 'Optional with previous optional line
        Next i
    End If
    AttachmentNames = strAttach
End Function

Sub TestFunction()
Dim olMsg As MailItem
    On Error Resume Next
    Set olMsg = ActiveExplorer.Selection.Item(1)
    MsgBox AttachmentNames(olMsg)
End Sub
