ken sheridan say:

You don't really need a hidden control in the detail section.  The ControlSource of the text box in the report footer can be:

=Sum(IIf([YourField] < -0.3 Or [YourField] > 0.3,1,0))

to count both the reds and greens.

=Sum(IIf([YourField] < -0.3,1,0))

to count the greens.

=Sum(IIf([YourField] > 0.3,1,0))

to count the reds.

substituting the actual name of the field of course.

riceq say:

Thanks, but I get an error when I do that. 

I read somewhere that you can't use the Sum function in a report footer.  It works in the Details section but I would like it in the footer.

I know you need a code to do this, but I don't know how...any suggestions?

Thanks.

ken sheridan say: You can use a Sum operation in a group or report footer, but not in a page footer.  Are you trying to show the sum of the highlighted values per page?  If so it can either be done in code in the Print event procedure of the relevant sections. Do not use the Format event procedure as this can be unreliable unless you are very careful with the code, which gets quite tricky, because you have to account for the code executing multiple times while the FormatCount property remains at 1, which is not unusual with this event procedure.

I'm not clear whether you want a single count of both the red and green values, or separate counts of each.  I'll assume the former, but if not it should be easy for you to translate what I say below so as to return separate counts in two controls.

1.  Add a text box, txtCounter say, to the page footer section.

2.  In the page header section's print event procedure initialize the txtCounter control to zero:

[txtCounter]  = 0

3.  In the detail section's Print event procedure increment the value of the txtCounter control, examining the PrintCount property to avoid any inadvertent double counting:

If PrintCount = 1 Then
   If [YourField] < -0.3 Or [YourField] > 0.3 Then
        [txtCounter] = [txtCounter] + 1
   End If
End If

A variation on the above would be if you wanted a cumulative count in the page footer.  In this case, instead of initializing the txtCounter to zero in the page header's Print event procedure you would do so in the report header's print event procedure.

Note that this will only work in print preview or when sending the report to a printer, not in report view.
