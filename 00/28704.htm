jllatham say:

Try this code - use it with an empty sheet selected because at the end of it, it will put the numbers it generated into column A just as part of the demonstration.

Sub NonDuplicateRandomNumbers()
  'this will generate 50 random numbers
  'betweeen 1 and 100 (inclusive) without
  'duplicating any of them
  'change the values to change the results

  Const howManyToMake = 50

  Const lowestNumber = 1
  Const highestNumber = 100

  Dim theNumbers(1 To howManyToMake)
  Dim LC As Integer ' loop counter
  Dim newNumber As Integer
  Dim newCount As Integer

  Do While newCount < howManyToMake

    DoEvents

    newNumber = Int((highestNumber - lowestNumber + 1) * Rnd + lowestNumber)

'   the more numbers you need to generate, the longer this test will take

    For LC = LBound(theNumbers) To UBound(theNumbers)

      DoEvents

      If theNumbers(LC) = newNumber Then
        'we have this number already, get another
        Exit For
      End If
    Next
    If LC > UBound(theNumbers) Then
      'a new number, add to the array
      newCount = newCount + 1
      theNumbers(newCount) = newNumber
    End If
  Loop
  'this is just to show you the results
  For LC = LBound(theNumbers) To UBound(theNumbers)
    Range("A" & LC) = theNumbers(LC)
  Next
  MsgBox "Done"
End Sub

Hi Mike,

I ran your code several times and had up to >900,000 passes taking > 100 sec.

Following does it in microseconds.

Sub Somesub2()
   Dim LowNo As Long, c As Range
   Dim HighNo As Long
   Dim Amount As Long
   Dim pass As Long
   Dim t As Single
   Dim dic As New Dictionary
   Dim FillRange As Range
   Dim rnd As Long
   Dim var As Variant
   Dim intPos As Long
   
   t = Timer
   LowNo = 1
   HighNo = 1000
   Amount = 1000
   
   Application.ScreenUpdating = False
   intPos = 1
   ReDim var(1 To Amount, 1 To 1)
   Set FillRange = Range("A1").Resize(Amount, 1)
   Do While dic.Count < 1000
      rnd = WorksheetFunction.RandBetween(LowNo, HighNo)
      If Not dic.Exists(CStr(rnd)) Then
         dic.Add Key:=CStr(rnd), Item:=0
         var(intPos, 1) = rnd
         intPos = intPos + 1
      End If
   Loop
   FillRange = var
   Debug.Print pass, Timer - t
   Application.ScreenUpdating = True
End Sub

jllatham say:

The basic problem is that as numbers are chosen, the pool of numbers to choose from grows smaller so the time in/iterations of the loop grow larger looking for an unused number in the pool.  For that last time through you are looking for one specific number out of (in the example) 1000 to come up randomly - and that may not happen for several thousands of iterations.

And it's difficult to even guess what the final number might be, since the low/high boundaries may have already been generated.  Interesting problem in creating a time-efficient method of doing this.
