john mcghie say:
Guruprasad:  Of course it happens in all documents, and in a new user account, and if you reset the preferences!  You should know this: it's the product's designed behaviour!

Sydney:  When you make changes to "Define a new bullet" the changes are saved only in the document you made the changes in, unless you make them in the Document Template.

To do that, you should define the bullet settings into a style, then save the style to your Template.

Or:  You can open Normal.dotm as a document and make your changes there.

Either way, the those changes will then be copied into all documents you create.

To change the style, first apply the "List Bullet" style to one paragraph (it's a built-in style pre-formatted with the settings you need).  You may have to change the styles list to show "All Styles" to make the list show the "List..." series of styles.

Once you have applied List Bullet, you can then go into "Modify Style" click "Format" then choose "Numbering".  Bullets and numbering are the same thing to Word.

Make your customisations in there.  Then on the way out, click the checkbox "Add to Template".

Hold down the Shift key and from the File menu choose "Save All".  This will save the changes in not only your document, but also in the Normal.dotm template that all future documents will be based on.

The above is the quickest way to do this.  You can do it a simpler way, but it takes a bit longer:

1)  Navigate to /Users/john/Library/Application Support/Microsoft/Office/User Templates and use File>Open to open the Normal.dotm template.  You must use File>Open, if you try to double-click it you will get a new document instead.

2)  Make your changes to the Bullets in that file.

3)  Save and Close.

Now each time you create a new document, your new bullets scheme will appear in the Bullets gallery.

You will find that the second method, while logically simple, gets old after a few times, so I suggest you use the first method.  Generally, using styles in Word gives you massive savings in consistency and time.  A style is simply "A collection of formatting to which you give a name so you can find it again." All formatting in Word is a style internally, so you may as well give it a name you can use.

Hope this helps

ossiemac say:

It is not practical to attempt to test code without the associated data.

Firstly don't use "Exit Sub" in events where you have Application.EnableEvents = False. Your code has a label "ExitPoint so In lieu of "Exit Sub" use "GoTo  ExitPoint". Also see my code where I have handled Errors to send the code to an Exit Point that I call ReEnableEvents. This prevents the Events being left turned off after the code has run.


However, you have a very busy Worksheet Change event and I wonder if the Spell Check would be better done with another event where that is all that it is doing. Do you have a "Dashboard" sheet where you select the various operations required in the workbook and then return to it when finished? If so, you could call the Spell checker code from the worksheet Deactivate Event.

I suggest inserting the code below in a standard module and call it. When calling it pass the required worksheet to it.

You said that my previous code example works if inserted at start of your change event sub so why not do that.  Insert the call line of code as first or last line of the event sub. Line of code to call from a worksheet Change Event (or any other event directly associated with the worksheet like worksheet deactivate event) is as follows. Note that you have to get rid of the Exit Subs if you want to call the SpellChecker from the last line of the Event sub.

Call SpellChecker(Me)   'Me is the generic worksheet object to which this module belongs.

If calling from the Workbook_SheetChange Event in ThisWorkbook module then the calling code will be as follows because Sh is the worksheet object.

Call SpellChecker(Sh)

Following code to be copied into a standard Module.

Sub SpellChecker(ws As Worksheet)

    Dim rngToCheck As Range
   
    On Error GoTo ReEnableEvents    'This ensures that Events get Re-Enabled if code error occurs
   
    Application.EnableEvents = False
   
    Set rngToCheck = Nothing    'Prevent retention of previous set range if no new range of xlCellTypeConstants
    On Error Resume Next        'Because Next line will error if no xlCellTypeConstants
    Set rngToCheck = ws.Cells.SpecialCells(xlCellTypeConstants)
    On Error GoTo ReEnableEvents    'Reset Error handling because On Erro Resume Next cancels this command
    Application.DisplayAlerts = False   'Suppresses Alert "Do you want to continue checking from the beginning of sheet"
    rngToCheck.CheckSpelling
    Application.DisplayAlerts = True
   
ReEnableEvents:
    If Err.Number <> 0 Then
        'User needs to be advised if an error has occurred
        MsgBox "Error occurred in Sub SpellChecker"
    End If
   
    Application.EnableEvents = True
           
End Sub

Note that if you are getting an error message then to identify the error line you will need to comment out the On Error GoTo and On Error Resume Next lines so the code will stop on the error. Uncomment the lines after development of the project is completed.
