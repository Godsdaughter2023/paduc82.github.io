brian tillman [outlook mvp 2007-2019] say:

While you cannot set the holidays you added with Outlook's "Add holidays" feature to "Out of Office" when you add them, it's fairly easy to change them all to Out of Office.  Open your calendar, click View>Change View>List.  In the Arrangement section of the ribbon, select "Categories".  This should display your calendar in a table view grouped by category.  If you 've assigned your color categories names, like Holiday, and you've made your holidays part of that category, all of your holidays should be displayed together.  Using the Field Chooser (right-click the header line above the entry list to find it), add the "Show TIme As" field to the header line (click and drag the field to the heder line).  Now, right-click the "Show Time As" header and choose "Group by this column".  Now under each category, you'll have subdivisions for "Free", "Tentative", "Out of Office", and "Busy".


If you don't have any holidays that are yet configured as "Out of Office", click on the "Show Time As" field for one of the holidays.  You should see a drop-down showing the list of available selections.  Choose "Out of Office".  This will creating a grouping for "Out of Office" underneath the "Holiday" category.  Now, select each holiday you with to change to "Out of Office" by click/shift+click to select a contiguous range or click/ctrl+click to select a disjoint connection.  Now click on a memeber of the selection and drag the entire collection to the "Out of Office" group under the Holiday category.  That will change the entire selection at once.

Well, what you were told by support is partially true... Mac Excel does not support the use of multiple language content in a single workbook file. What was described is for changing the User Interface language which determines the language used for display purposes in menus, dialog windows, etc. as well as the default language on the Mac.

What ls described in the reply from Neha Singh is completely untrue. Those steps determine the key mapping [Input Source] of the keyboard being used but have no effect on the actual language attribute of the text. IOW, if you set the Input Source to Spanish but the Language selected in Excel is English, anything you type will be in English, not Spanish. However, certai characters [e.g., ¿] not ordinarily used in English can be input directly from the keyboard &certain keys will insert something other than what is designated on the key cap.
