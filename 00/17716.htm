Moving PowerPoint slides from 2003 to 2013 - Microsoft Community

mylescooley say:

Moving PowerPoint slides from 2003 to 2013

I understand it's not a simple matter to move presentations formatted in PowerPoint 2003 to 2013. Any references on how to do this?
john korchok say:
PowerPoint 2013's Convert command helps smooth the road: Convert Earlier Presentations. Often the most work is with updating charts and tables, since they use a new way of applying colors from the theme that was not present in 2003.

john korchok say:

I got other symptoms the first time I used this feature: First a yellow bar across the PowerPoint window. Clicking on Enable Editing made that go away. Then I got Access Denied errors until I copied all files to my local drive instead of the network, then closed and reopened PowerPoint. After that I was able to use the convert tool.

It's possible you are tripping over an access denied error, but the dialog is not displaying, so make sure all files you're converting are copied to your disk and not on a network server. Or your PowerPoint installation needs some repair, or there is some feature in your presentations that PowerPoint can't handle. If worst comes to worst, you'll just have to open the file and resave as 2013 format. Here are the main issues you may run into:

Multiple Masters: before 2007, PowerPoint implemented Multiple Masters to give more flexibility to master formatting. Slide Layouts were still closed to user formatting, Starting with 2007, Microsoft opened up Slide Layout formatting, so multiple masters were rarely needed anymore. If you import a presentation with Multiple Masters, each master will be given Microsoft's default 11 slide layouts for each master. You can easily end up with a gigantic number of layouts. It's too bad, because usually each master only needs one layout to make it useable. This area can be time-consuming to set up in a genuinely 2013-friendly presentation.

Themes: PowerPoint attempts to transform the Color Schemes from 2003 into a 2013 Color Theme. I won't go into agonizing details, but I certainly would have designed this transformation differently. Colors are moved to different places in the Theme than their corresponding position in the Scheme, and then the theme is mapped to the objects in the presentation. While this results in a presentation that looks the same as the original, it doesn't quite work the same anymore.

Tables: Tables are constructed in quite a different way between the 2 versions. I've seen issue of table elements coming out with the wrong colors after updating the file format.

Charts: 2003 used MSGraph for charts, 2013 uses Excel. All existing charts must be converted before editing. This is the least compatible area between 2003 and later versions.

hansv mvp say:

COUNTIFS was introduced in Excel 2007. It is not available in Excel 2003 and before.

Let's say you want to count how many rows have "Test" in column A and a number greater than 60 in column B, in rows 2 through 51. The COUNTIFS formula would be

=COUNTIFS(A2:A51, "Test", B2:B51, ">60")

You can use SUMPRODUCT to get the same result:

=SUMPRODUCT(--(A2:A51="Test"), --(B2:B51>60))

or

=SUMPRODUCT((A2:A51="Test") * (B2:B51>60))
