considerate_guy say:

Okay, I misunderstood your question.

There are two things you can save from Movie Maker.

  1. The project file -- this has a .wlmp extension and is essentially the save state of your project while you are editing. It contains all the individual elements in your project (photos, video clips, music, titles, etc.). To save your project, you use File > Save project. You always want to edit using the project file. You can't use your project file outside of Movie Maker. It actually contains no video, music or photos, just the instructions for assembling your project into a movie.
  2. The movie file -- once you are done with your edits, you can use File > Save movie. This creates a single file that can be played in Windows Media Player, uploaded to YouTube, shared with friends, etc. The movie applies all your edits into a single video/audio stream. It doesn't have any knowledge of the original source files and how they were trimmed, etc.

If you want to make more edits, you should always return to your *project* file. To do this, go to the File menu and look to the right under Recent projects. Find the project that you were working on and open it back up. That will show you all the individual elements.

If you instead just open the *movie* file in Movie Maker, it treats it like a new project with a single video clip. As you have noted, there are no individual elements shown anymore.

If it helps to have an analogy, the *project* is like a recipe. If you wanted to modify a recipe, you could change the amount of eggs, add some more sugar, switch to using a chocolate frosting, etc.

But the *movie* is like the final cake once you are done baking. The cake can be shared with others, but if you wanted to make changes, it wouldn't be easy to do with the final cake. You can't change the amount of eggs or add sugar (except maybe sprinkled on top). And if you wanted to change the frosting, you'd have to first take some things apart, scrape out the old frosting and put in some different frosting. That's a lot more involved, if it is possible at all.

So for editing, always go back to the project file (the "recipe") rather than the completed movie (the "cake").

Chris,

 

There is absolutely no reason to expect SFC to affect your installed applications.  SFC only checks non-configurable protected system files.  But "confirm"? - I have never had a problem with SFC affecting anything else or heard of such a problem in this forum.

 

I did not want to mislead you by referring to the System repair disk.  It would have just saved a lot of explanation about dialogs that you will see.

 

The syntax for running SFC from the installation disk is simple but the means of getting there is new to you so that's why I am being careful.  It's not difficult but it's awkward without being able to stand at your shoulder & point as dialogs appear.

 

1  Insert the installation disk.  Remove all other connected drives just to keep things simple.  Restart the system. 

 

2  When the PC maker's logo appears [i.e. well before the Starting Windows screen appears] start pressing repeatedly on the Boot options key [which will either be displayed on the screen by the PC maker or else given in the PC manual].  On my Dell, it's F12 but it varies between PC makers.  Select boot from USB.

 

3  After several minutes [that can include periods with a blank screen & no apparent activity], wait during the “Starting Windows” screen then, at the first Install Windows dialog, select the language the time-currency format & the keyboard.  Then click Next. 

 

 

4  At the next Install Windows dialog, select Repair your computer, wait while it searches for windows installations. 

 

 

5  At the System Recovery Options dialog, write down which drive letter it has temporarily assigned to your OS drive [this will not affect your drive when it is rebooted normally - it's just a temporary label but you'll want it during this procedure]..  Then set the radio button for Use recovery tools... then click Next. 

 

 

6  At the System Recovery Options menu select Command prompt.  [ignore where my cursor was during this photo]

 

 

7  Enter all this -  I'll explain it below.

 

sfc /scannow /offbootdir=c:\ /offwindir=c:\windows >”C:\Users\YourUserName\Desktop\SFCDetails1.txt”

 

7.1  Be very careful to use the correct drive letter [the one you just wrote down], to keep the spaces between elements of that command, to keep the command all in one line & to enclose the output file path in quotation marks in case your actual user name has any spaces in it -

                    sfc[Space]/scannow[Space]/offbootdir=c:\[Space]/offwindir=c:\windows[Space]>”C:\Users\YourUserName\Desktop\SFCDetails1.txt”

 

7.1  You are saving a report of the SFC scan to your normal Desktop.  You can save it anywhere you like expect the X:\ drive you see on your screen as that is just a temporary virtual drive.

 

7.2  Repeat the SFC scan but use SFCDetails2.txt for the next results.  Then do it again using SFCDetails3.txt

 

7.3  You will see a summary of results at the end of each run.  Now enter ”C:\Users\YourUserName\Desktop\SFCDetails1.txt” to see your first results open in Notepad then look at the second set then the third set.  You can use the Command prompt to have all three open as they will each appear in their own window - and you can shuffle the windows about using Win key & left/right arrows as normal.  By doing SFC three times from the installation disk, you are increasing the chances of it being able to fix the errors it finds.  The results files are messy but they are the only way of getting an SFC report in these conditions.  The files will actually look similar to that you posted a link to earlier but with a load of symbols making the actual scanning runs awkward to read.  The error reports will be clear though and unfixable errors will have "[SR] Cannot repair member file" in them and will enclose the file concerned in quotation marks.  Each line starts with the date-time and that can help you distinguish between different entries.

 

7.4  Now you can decide whether to run it a few more times or to reboot.  If there are additional errors detected in the 2nd or 3rd runs or there are any errors reported in earlier runs that seem to have been fixed then do try again to let it go further.  If it has three results [good or bad] that look the same then call it a day & reboot normally.

 

8  When you have finished & have rebooted you can post any remaining unfixable errors back here.  It's possible that there will be none.
