mccbleue say:
matt_14 wrote:

I cannot believe Nokia have put 200 to 240hrs standby time on the specifications!!

 

In addition to the comments made already, you also need to consider what the stated times mean - "standby time" is actually a pretty pointless figure as it actually refers to the amount of time the battery could last if the phone is switched on but not *ever* doing anything - each time you make/receive a call you use more power. Each time you light up the screen you use more power. Each time you use an application in the phone you use more power. The list goes on and on. If you are in an area where your network gets a weak signal, your phone will spend more time searching for and connecting to the network each time the connection drops, so unless your phone is in "offline" profile, it is never really doing absolutely nothing.

 

Equally, call times quoted are the amount of time the phone could spend doing that one activity on a full charge. That does not take into account fluctuations caused by other uses of the phone.

 

The truth is there is no real accurate way to determine the functional life on one power cycle of any multi-function product out there in the real world. The mobile industry could come up with "simulated use circumstances" just like car makers did with fuel consumption figures, but even that would still be only a rather inaccurate guide. I drive a car that never achieves published figures because the people who did the testing had no way to know I'd spend half of my life in traffic on the Brussels Ring - just as Nokia didn't know that my operator would have terrible reception in the village where my uncle lives and that my phone would spend every minute there desperately switching between GSM and 3G in the hunt for a half decent signal...

 

If your N96 is really as bad as you say, perhaps try switching the network mode to GSM only in case you are in a weak 3G signal area - you could be blaming the phone for something that it can't control. None of us get the really optomistic battery life of the published figures, but most of us do last much longer than you descirbe on light use,   there is very likely a circumstance peculiar to you that makes your situation worse.

ken sheridan say: You will need to use a UNION ALL operation.  The two sets on which the operation acts will be:

1.  All rows for the current month which do not have a match in the set of all rows for the previous month.  This is achieved by joining two instances of the table on the OrganisationID column and on those columns whose values can change.  The join should be a LEFT OUTER JOIN and return rows where a NULL is returned in the OrganisationID column on the right side of the join.

2.  The second set is a limited converse of the first: all rows for the previous month which do not have a match in the set on the key OrganisationID column in the set of rows for the current month.  This is achieved by joining two instances of the table on the OrganisationID column.  The join should be a LEFT OUTER JOIN and return rows where a NULL is returned in OrganisationID column on the right side of the join.

The first set will return rows where any of the changeable columns' values have changed from the previous month, or where the organisation was not represented in the previous month.  The second set will return rows where an organisation represented in the previous month is not represented in the current month.  Both sets are then returned as a single result table by means of the UNION ALL operation

karl timmermans [outlook mvp 2012-2018] say:
In addition to the suggestions which have already been provided which in summary are

#1 - Remove IMAP account from the Outlook profile
#2 - Re-add the account configuring it manually using POP3 (if Gmail, you would need to set the account to POP3 at the server level)

The major thing you need to do BEFORE removing the imap account is

#3 - If you have contact/calendar items in contact/calendar folders in the IMAP folder tree labeled with "This Computer Only" - be sure to export these folders to a PST file otherwise you will lose that data when the account is removed from Outlook

#4 - Once you have the POP3 account configured, you should import the data from the exported PST file or alternatively, add the exported PST file to the Outlook profile and copy the data items from the exported PST file to the corresponding folders in the new POP3 folder tree.
