bob jones aka: cybertaz say:

Web Layout view is for creating web pages [to be viewed in a browser] which are basically 'unstructured'. Notice that as you change the width of the document window while in Web Layout view the text reflows accordingly. By default, the text flow of what users see is determined [among other factors] by the width of the window of the program being used to view the document... typically a browser. Accordingly, what any given user sees may not be the same as what you see in Web Layout view.

As Daniels suggested, the primary method of imposing structure on the text flow is to use tables. The only other option is to use forced line breaks or paragraph breaks at the preferred points in the text -- which usually provides an undesirable result & can be interpreted differently depending on the actual program being used to view the document.

It isn't clear what the nature of this 'document' is, but it sounds like a letter or message of some kind intended for distribution rather than coauthoring. IMO, it would be best to send it as an attachment rather than as email body content. Also, it would seem to me that it isn't intended to be revised by the recipients. If that's the case & you further want to be certain that all recipients see it as intended, you might want to consider distributing a PDF rather than a Word document in order to avoid other issues (e.g., font substitution) as well.
