djsteele, mvp say:
Just to add onto what Hans has said, the current folder always refers to current folder of the client (i.e. - the user of the front-end), not the back-end. What that folder is depends on a number of factors. If you open the front-end by clicking on a shortcut, it'll be whatever's specified as the Startin folder of the shortcut. If you opened Access and then selected the front-end from the MRU (most recently used) list, it'll be whatever's specified as the Default Folder for Access. If you opened Access and then navigated to the database via File | Open, it'll be the folder where the database exists.

j.e. mcgimpsey say: First - it's very easy to submit a bug report with any edition of Office 2011. Choose Help/Send Feedback... from any application.

Second - what you posted isn't remotely a bug report. A bug report, to be at all useful, needs to have sufficient information to replicate the bug, including detailed step-by-step instructions. What you've posted is at best an assertion that there might be a problem.

You are correct that you can't use VBA to "create a table from a sheet" - tables are child objects of worksheets and can be created from a Range.  However,  it is perfectly possible to create a Table from a range, then convert it back to a range using VBA. You need to know that Tables are implemented in VBA via ListObjects. For example:

    Public Sub CreateTable()
        With Worksheets("Sheet1")
            .ListObjects.Add(SourceType:=xlSrcRange, _
                                          Source:=.Range("A1:D100"), _
                                          HasHeaders:=xlYes).Name = "My New Table"
        End With
    End Sub

    Public Sub ConvertTableToRange()
        Worksheets("Sheet1").ListObjects("My New Table").ConvertToRange
    End Sub

Not sure what you mean by "If you are working with the New Table option" - which option? You can insert a table without having a cell selected on a worksheet (e.g., if a Shape or some other object is selected), but the ActiveCell is then used to anchor the Table.

Open Links using Internet Explorer app and not as desktop version

Original title: Explorer 11, running as app, not in desktop

I have a surface pro2 running 8.1. Is it possible to launch explorer 11 as an app instead of desktop. Its a much better touch experience
kevin d souza say: Hi Roger,

If you wish to use Internet Explorer app instead of the desktop version, then do these changes in Internet Options and check.

- In the Search Charms, type Internet Options and tap on it.
- Tap on the Programs tab.
- In the Option, "Choose how you open links", select "Always in Internet Explorer"
- Uncheck the option "open Internet Explorer tiles on the desktop".
- Tap apply and ok. Close all the Internet Explorer windows and then try to open any links and see if they open in Internet Explorer app version.

Hope this helps. If you have any concerns, let us know.
