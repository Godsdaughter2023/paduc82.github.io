Increment a field using update query

I need to increment a text field (all numbers) on all records where the field is empty.  Field must have format yymmdd999 (date + number starting at 0 for one set of records, and at 500 for another set).  What would the update query look like that would store consecutive numbers in each record where the field is empty?

I found an example using DCount("ID", "LabelArchiveTableTS","ID<" & [ID])+1 that works, but it is dependent on no records being deleted which is not a safe assumption in this case.

I have tried using DMax, but get the same value (correctly the next one for the first record) in every record.  This is my "Update To":

Val(DMax("[LPN]","LabelArchiveTableTS","left([LPN],6)='" & Format(Date(),"yymmdd") & "'")+1)

And and all help would be greatly appreciated!

Thank you -

Michelle

ken sheridan say: Instead of executing a single UPDATE query update each row in code by looping through a recordset:

    Const conSQL = "SELECT * FROM LabelArchiveTableTS WHERE LPN IS NULL"
    Dim strSQL As String
    Dim strUpdateTo As String
    Dim rst As DAO.Recordset
    
    Set rst = CurrentDb.OpenRecordset(conSQL)
    
    With rst
        .MoveLast
        .MoveFirst
        Do While Not .EOF
            strUpdateTo = Val(Nz(DMax("LPN", "LabelArchiveTableTS", "left(LPN,6)='" & Format(Date, "yymmdd") & "'"), 0) + 1)
            .Edit
            .Fields("LPN") = strUpdateTo
            .Update
            .MoveNext
        Loop
    End With

This will update all rows where the LPN column position is Null with sequential values based on the current date.  I don't quite follow what you say about the two subsets of rows, however.  If these are all in the LabelArchiveTableTS table, then your expression does not distinguish between the two subsets.  To do so you would need to add further criteria to both the conSQL constant and the expression for the strUpdateTo variable.  In the latter, the value returned by the Nz function in the event of an there being no rows which match the criteria would be changed from 0 to 499 in the case of the subset starting at 500.

jay freedman say:

Here's a more complete macro, which will catch hyperlinks in headers, footers, text boxes, and footnotes/endnotes as well as getting pointers to bookmarks or HTML anchors if they exist. It puts the list into a separate new document, but you can rework it to put the list at the end of the existing document if you prefer.

Sub ListHyperlinks()
    Dim srcDoc As Document
    Dim destDoc As Document
    Dim srcRg As Range
    Dim HL As Hyperlink
    
    Set srcDoc = ActiveDocument
    Set destDoc = Documents.Add
    
    For Each srcRg In srcDoc.StoryRanges
        Do
            For Each HL In srcRg.Hyperlinks
                destDoc.Range.InsertAfter HL.TextToDisplay & ": " & HL.Address
                If HL.SubAddress <> "" Then
                    destDoc.Range.InsertAfter "#" & HL.SubAddress
                End If
                destDoc.Range.InsertAfter vbCr
            Next HL
            Set srcRg = srcRg.NextStoryRange
        Loop Until srcRg Is Nothing
    Next srcRg
    
    On Error Resume Next
    destDoc.Save
End Sub
