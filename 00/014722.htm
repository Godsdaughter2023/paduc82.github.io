daniel g. grau say:
Yes. It's the paintbrush icon. In Word, it appears in the Standard toolbar, at the top of every document's window.

If you don't see that toolbar, select View > Toolbars > Standard.

In addition, in Word it has a shortcut. Select the source text, press Command Shift C to copy the format, select the target text and press Command Shift V to paste the format. These shortcuts are easy to remember, as they are like regular Copy and Paste, plus the Shift key.

Format painting is quite smart. If your source selection includes the ¶ mark (end of paragraph), or just the ¶ mark, then the copy operation will include character formatting and/or paragraph formatting, according to what you've selected. Likewise, if the target selection includes (or not) the ¶ mark, or just the ¶ mark, Word will paste the appropriate portion of the copied format.

hannoverfist say:

Hey Pam -

Here's something to get you started that I use in one of my crazy file imports. The code will open your file (strImportFileName) and read it line by line.

My code looks at the first character of the line for the type then goes to that sub to import the data. After it returns from the sub, it will loop until the file is complete.

You'll need to create code to parse and import each Record type based on the record type's specs into whatever table that type of data needs to go in.

 


    Dim txtStream As Object
...


Set txtStream = CreateObject("Scripting.FileSystemObject").OpenTextFile(strImportFileName)


Do While Not (txtStream.atEndOfStream)

    strImportRecord = txtStream.Readline


GetRecord:

    intRecordCount = intRecordCount + 1    'Count import records

    strRecordType = Mid(strImportRecord, 1, 1) 'Determine the type of record


CheckRecordType:

    Select Case strRecordType

        Case "0"
            GoSub Header

        Case "9"
            GoSub Footer

        Case "1"
            GoSub DataRecord

    End Select


NextRecord:

Loop

...

txtStream.close

Exit Sub

...

 

DataRecord:

'Update and run query to add data

    gSQL = "INSERT INTO tblTempImportData ( Name, Phone, WorkDate ) " & _
                "Values ( '" & Mid(strImportData, 10, 40) & "', '" & Mid(strImportData, 51, 10) & "', '" & Mid(strImportData, 61, 10)  & "' )"
    CurrentDb.QueryDefs(strQuery).SQL = gSQL
    DoCmd.OpenQuery strQuery

 

Return

 

Hope that gets you started. 

You'll need to add the name of your file (strImportFileName = ?) in the code before opening the text stream.

Identify the type of record based on where the type is identified - my code used the first character (strRecordType = Mid(strImportRecord, 1, 1) ).

You'll also need a separate subroutine for each record type and a line to call it in the CASE statement. Use the MID function to parse the record using your specifications for that particular type.

Yet another possible approach is to link to the text file, using a specification that defines only the defining record-type field and defines the remainder of the record as an undifferentiated text field.  Then build a set of queries based on this linked table, one for each type of record.  Each query will select only the records of one particular record type, and will use calculated fields to parse out the rest of the record into the specific fields appropriate to that record type, using the Mid() function to specify each individual field according to its position and length.
