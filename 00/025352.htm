doug robbins - mvp office apps & services (word) say:

The print button runs a command FilePrintDefault.

If you create a template and in it you create the following macro

Sub FilePrintDefault()
'
' FilePrintDefault Macro
' Prints the active document using the current defaults
' Not any more it doesn't
    MsgBox "Disabled.  Use File>Print instead."

End Sub

and then you save that template in the Word Startup folder on each machine, it will be loaded as an add-in and whenever that button is used, the message "Disabled. Use File>Print instead." will appear.

For a method of distributing the template to all of the machines, see the article "Distributing macros to other users” at:

http://www.word.mvps.org/FAQs/MacrosVBA/DistributeMacros.htm
 -- Hope this helps.
