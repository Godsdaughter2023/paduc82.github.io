lemp say:

When you ran chkdsk with the R switch, it probably found file errors and may have concluded with a message similar to this:

20 lost allocation units found in 5 chains.
Convert lost chains to files?

Assuming that you responded by pressing Y, chkdsk will have saved the "lost chains" as files in a hidden folder in the root directory named FOUNDnnn

In Windows Explorer (file manager), click Tools > Folder Options > View tab and select the radio button to "Show hidden files, folders, and drives" and click OK

Look for C:\FOUNDnnn (assuming you ran chkdsk on the C: drive).  The numbers start with 000 and increase each time you run chkdsk and it finds file errors.

If you find C:\FOUND000 (for example), inside will be a series of *.CHK files. Because you don't know what is in any of these files, it can be difficult to examine them.  For some suggestions, see https://www.raymond.cc/blog/how-to-recover-chk-files-created-by-chkdsk-and-scandisk/

If you don't have a FOUNDnnn folder, it may be because you responded to the prompt after chkdsk ran by pressing N.  You can try using a data recovery utility, but odds are low and get lower the more you use your hard drive.  Recuva is a decent free utility:  https://www.piriform.com/recuva

levelkro say:

FIXED!!!

Ok, After some search about this problem, I have tested others solutions, but it's not working;

- Download ISO from Microsoft with other not-Windows OS for select specific ISO (include WIM instead of ESD).

With this ISO' I got a different error code, after a quick search on Google I find a solution;

The problem is caused with a packge including "TestRoot", this package have a issue with signature/certificate and make the DISM fail. This package also make error with Windows Store. After removing the Registry entries and packages files, you can use DISM without error and Windows Update and WIndows Store work fine.

  1. Open Command Line or Power Shell with level up user and type: regedit
  2. In RegEdit, find the folder; HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\PackageIndex
  3. Find Folder entries with "TestRoom", like;
    Microsoft-Windows-TestRoot-and-FlightSigning-Package~31bf3856ad364e35~amd64~~10.0.15063.0
    Microsoft-Windows-TestRoot-and-FlightSigning-WOW64-Package~31bf3856ad364e35~amd64~~10.0.15063.0
  4. Right click on each and export
  5. Right click on each and change Autorisations for take Full Control
  6. Delete each entry from registry
  7. Open File Explorer search for files named "TestRoot"
  8. When is done, You have 4 files, cut and past in safe place (like with registries saved before)
  9. Reboot computer, it's make more time (15-20 minutes for me).
  10. Done! Enjoy Windows Update and Windows Store!
