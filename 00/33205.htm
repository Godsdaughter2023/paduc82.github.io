frederik long say:

Your approach has two fundamental problems:

  • Telnet is extremely primitive. It has no provision for scripting, e.g. in the manner that FTP has. This forces you to use SendKeys.
  • SendKeys is inherently unreliable in a multi-tasking system. You must set the focus to the Telnet session (which makes it visible) but Windows can by itself change the focus to something else, which will cause SendKeys to do unpredictable things.

I recommend you use psexec.exe instead. It generates the output you want, it is extremely versatile and you do not need to shell out from a VB Script to invok it. A single command line will suffice, e.g. like so:

psexec \\192.168.1.1 -u JSmith -p 12345678 SystemInfo > d:\Info.txt
