Indeed, as Suzanne wrote, the key to getting paragraph formatting to work for a table style is to make sure that the Normal style has the same settings as the document defaults.

Of course, the connection between the document defaults and the Normal style is far from obvious, but you can easily test it and convince yourself.

You'll find the document defaults on the Set Defaults tab of the Manage Styles dialog box (press Ctrl+Alt+Shift+S to display the Styles pane and click the Manage Styles icon, and Word displays the Manage Styles dialog box).

To display the Modify Style dialog box, press Ctrl+Shift+S; in the Apply Styles pane, type in Normal and click Modify.

Study the page and look for overlapping text boxes or pictures that might be pushing text down and out of the text box. Overall just study the page, move around some of the problem text boxes, etc and see if there is some common problem with how you have them arranged on the page.

If that exercise didn't lead to any solutions or insight as to why you are having these problems, the next thing I would do is look at the Paragraph settings and perhaps tweak them. First select a problem text box and go to Home tab > Paragraph group and first click the Paragraph symbol which is a toggle to turn Special Characters on and off. Now you can see Hard and Soft (shift + enter) Returns at the end of your lines, lines between paragraphs etc. You might experiment with using Soft returns instead of Hard to start new lines rather than create a new paragraph and see if that makes any difference.

Then click the down arrow under the Paragraph group > and study the options available under each tab. In particular under the Indents and Spacing tab > Line Spacing > look at the settings. Old versions of Publisher were set at a default of 0 pt before and after paragraphs, and 1sp between lines. As all your text is not fitting in your text boxes in some cases you might want to experiment with these settings and see if this is a way to reformat the problem text box such that it behaves as you want.

You might have to ultimately just increase the height of text boxes beyond what appears to be necessary in order to workaround the problem. Or change the interior margins...

If you can convert to PDF successfully, you might use that as an interim step, before you send the PDF file to the printer.

I know this is an older thread but we experienced this on Publisher2016 today and am posting a workaround for anyone having this issue.  The formatting problem is the result of it not preserving line spacing options when it creates the new file to place the merge into.  Our original document used 1.19 spacing but it defaulted to 1.0.  Additionally the original doc had blank lines of 4pt to force some vertical spacing between paragraphs and thjs exacerbated the problem.  I changed the paragraph formatting to include the 4pt line AFTER each paragraph but it still reverted to an overall line spacing of 1 when merging to new document.

WORKAROUND is to create a blank Publisher document, save and close it, then merge into existing document, selecting the blank doc you just created.  There will be one blank page at the beginning that you need to delete but you can edit specific merge results and print the whole merge from there.

Microsoft, please determine why merging to printer other than pdf or merging to new document loses these line spacing settings.
