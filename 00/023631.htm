Hi Ammar,

From your description, I understand that you would like to run the command "New-ComplianceSearch" to copy the search result to another mailbox. However, with "New-ComplianceSearch" command, it is not available to copy the search results to another mailbox. 

Given this situation, as a workaround, it is recommended to run following command to export the search result to .PST file and then you can import the .PST file to the target mailbox: 

New-ComplianceSearchAction -SearchName "124" -Export -SharePointArchiveFormat SingleZip

How can I format it so that these changes don't happen?

Try substituting either an en-dash (e.g. –) or em-dash (e.g. —) as the delimiter in your formatting mask. Ctrl+H will take care of this handily. A macro could be written if that is too much trouble. It could also be incorporated directly into the formatting mask of =TEXT(<cell>, "00–00–00") or a custom number format of 00–00–00.

 -   hyphen, minus     Alt+0045    U+2010 (8208)
 ‒   dash, figure dash     U+2012 (8210)
 –  en-dash, n-dash, n-rule, nut     Alt+0150    U+2013 (8211)
 –  em-dash, m-dash, m-rule, mutton     Alt+0151    U+2014 (8212) 
 ― horizontal bar    U+2015 (8213) 

I've included the Unicode dash in the above samples, but that is not appropriate for a non-Unicode CSV. I've tested this on the bank codes you provided above with an en-dash and it worked as you intend.

These characters are all available through the Character Map utility program in Accessories. The en-dash and em-dash are available by holding down the Alt key and typing 0·1·5·0 or 0·1·5·1 on the number pad. You know where the hyphen is.
