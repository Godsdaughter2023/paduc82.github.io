doug robbins - mvp office apps & services (word) say:

If the form is in the document, you would need to use an ActiveX textbox (from the Legacy tools collection) and it will be necessary to set the Multiline attribute of the Textbox to True and also set the Scroll bars attribute to 2 - fmScrollBarsVertical.  Note, the scroll bar will not appear until the amount of text entered into the textbox exceeds the available space.  Once it does appear, it is there all of the time.

You also need to be aware that it is not possible to print the document and have all of the text entered into the textbox appear if there is more than can be seen in the textbox without making use of the scroll bar.

As an alternative, you may want to consider using a UserForm as on such forms, TextBoxes can also have scroll bars.

See the article "How to create a Userform" at:

http://word.mvps.org/FAQs/Userforms/CreateAUserForm.htm

and the following pages of Greg Maxey's website :

http://gregmaxey.mvps.org/Create_and_employ_a_UserForm.htm

http://gregmaxey.mvps.org/Populate_UserForm_ListBox.htm

Word 2016 for Mac footnote outside of printable area message - Microsoft Community

katherinefbell say:

Word 2016 for Mac footnote outside of printable area message

Before I print, I get the message from every section of my manuscript, "footnote outside of the printable area." I have very, very few footnotes. However, what do I do to address this message. Fix things so I can print with footnotes.

Thanks!

john korchok say:

Word polls the printer driver to find the maximum print area. Most times it's right, but not always. If you don't actually see any footnote clipping in the printed output, don't worry about it. But if you do:

  1. Choose File>Page Setup.
  2. Change the Settings dropdown to Microsoft Word.
  3. Click on the Margins button.
  4. Increase the Bottom margin slightly.
  5. OK out and try a test print.
  6. Repeat until you don't see any clipping off the bottom, or the error message stops.

You could do something like this. This tests the username of whoever is clicking the button. In the code below if the button is clicked by Mike or Joe then the code executes. If anyone else clicks they get a message and the code terminates.

Private Sub CommandButton1_Click()
Dim user As String
user = Environ("Username")
Select Case UCase(user)
