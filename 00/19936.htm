The crash you have is due to a segmentation fault, which is a memory error. So what causes memory errors? It could be Outlook has a bug. It could be Mac OS has a bug. It could be 3rd party software improperly using memory incorrectly.

Thinking about Outlook, you said you already reinstalled Outlook, and you tried it with a blank profile. That pretty much rules out Outlook and profiles as sources of the crash.

Let's do a test to see if 3rd party software or maybe a bad font is to blame. Try booting your Mac to Safe Mode by holding the Shift key down as your Mac starts from a restart or cold boot. Booting to Safe Mode empties the system font cache, fixes directory errors and fixes volume errors. It also stops other applications from opening that normally start up when you start your Mac. If Outlook opens in Safe Mode, do a regular restart and try Outlook again. If it opens OK. everything is fixed! Yay!

If Outlook will not open in Safe Mode, validate ALL your fonts. Mac OS comes with a free font utility called Font Book, which you can use for this purpose. Get rid of any font that doesn't pass as clean and green. Then, restart your Mac and immediately empty the trash. If dumping duplicate and bad fonts fixes the problem, then Yay! If not, use Font Book app to reinstall system fonts.

Lastly, if Outlook still won't launch, try reinstalling Mac OS
