¡firedog say:

Server Response: -ERR [IN-USE] account is locked by another session or for maintenance, try again.

This is a server error, as the message says. The usual cause is not that the mailbox is undergoing maintenance (which might happen very occasionally, but only for a few minutes or an hour or two), but that some other device has an active connection to the mailbox. There can only be one POP3 connection at once. If you're accessing this same mailbox from a mobile device that's configured for push mail, it will monopolize the mailbox and other clients - like Windows Live Mail - won't get a look in. Because the server response is interpreted as an authentication error, it is presented to the user as a fault with the username or password, but this isn't always the case. That's why it's important to read the whole error message.

When you want to access the same messages from more than one device (e.g. your computer and your phone), it's best to set up the account to use IMAP which doesn't have this limitation*. You cannot change an account in Windows Live Mail from POP to IMAP. You will have to set up a new account with the correct IMAP settings; then, when it's working properly, transfer any messages in the POP account's folders that you want to keep to folders under the new account. When you're sure there are no important messages left in the POP account folders, you can right-click on the account name in the folder pane and select Remove account.
 

To set up the IMAP account, 
 

  1. When viewing your message list, press Ctrl-Shift-T to start the  
  2. Enter your email address, password and display name for outgoing messages.
     
  3. Select Manually configure server settings and click Next.
     
  4. For the incoming server, select IMAP from the dropdown.
    For the incoming server address, enter imap.windstream.net and change the port number to 993.
    Select Requires a secure connection (SSL).
    For Authenticate using, select Clear text from the dropdown.
    For Log-on username, enter the full email address.
     
  5. For the outgoing mail server address, enter smtp.windstream.net and change the port to 465.
    Select Requires a secure connection (SSL) and Requires authentication.
     
  6. Click Next and Finish.
     

The program will try to connect and download the account folders and their contents. This may take some time if there are a lot of folders and/or messages, and it may stall. Be patient.

* In fact, Windstream states specifically that using IMAP on one device will lock the mailbox for POP access from any other client ("If you are going to set up email on your iPhone, iPad, or iPod Touch using IMAP, it is critical that you set up your email clients on all your PCs to use IMAP as well. Failure to do so will cause "POP lock" on your email account due to the nature of how IMAP maintains a session with the mail server." - from

Jennifer, these settings vary from provider to provider, so you cannot give them as a generic solution. Some SMTP servers insist on authentication while others will not work if My server requires authentication is selected. Some mail servers will only work with SSL, others won't work at all with SSL. If the correct settings aren't readily available on the provider's website, it may be necessary to test the various possibilities using Telnet. As it happens, Windstream appear to require SMTP authentication and to support SSL for POP3, IMAP and SMTP. This affects the ports to be used.

You missed a step in your list of instructions. My server requires authentication is on the Servers tab, not the General tab which is displayed when the properties dialogue opens. And Windstream state specifically that the Email username should be the full email address, not the email account username as you say (from What are standard configuration settings for Windstream Internet service: E-mail Address Format: USERNAME@windstream.net).

Are you overlooking click on Apply after changing the setting or forgetting to restart the computer to apply the new setting?

You should not have system restore monitoring your D partition. System restore should only monitor the partition containing your Windows files.

You can create more free space in C by carrying any of the measures suggested below.

The default allocation to System Restore is 12% on your C partition which is over generous. I would reduce it to 700 mb. Right click your My Computer icon on the Desktop and select System Restore. Place the cursor on your C drive select Settings but this time find the slider and drag it to the left until it reads 700 mb and exit. When you get to the Settings screen click on Apply and OK and exit.

A default setting which could be wasteful is that for temporary internet files, especially if you do not store offline copies on disk. The default allocation is 3% of drive. Depending on your attitude to offline copies you could reduce this to 1% or 2%. In Internet Explorer select Tools, Internet Options, General, Temporary Internet Files, Settings to make the change. At the same time look at the number of days history is held.

The default allocation for the Recycle Bin is 10 % of drive. Change to 5%, which should be sufficient. In Windows Explorer place the cursor on your Recycle Bin, right click and select Properties, Global and move the slider from 10% to 5%. However, try to avoid letting it get too full as if it is full and you delete a file by mistake it will bypass the Recycle Bin and be gone for ever.
