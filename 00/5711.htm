A Top Level Domain that seems to be Spammer's Heaven

I discovered something fascinating but useful about the .gdn top level domain.

I have been using the Outlook Rules to try to control the spam that gets through my regular Spam FIlter Program -- SpamFighter.

Some of these spammers are very clever at getting around most of the blocks of names by randomly changing the name part of the address. I block one address but the next one gets through.  So I just decided to block the domains and this worked much better. 

In the process, I was entering dozens of domains--almost all of the spammers who were randomly generating names to go with their domain were using multiple dot domains like "@xxxxxx.xxxxxx.xxx".  So I listed a set of these in Excel separating the portions separated by the dots.  Surprisingly I found that MOST OF THEM HAD THE SAME TOP LEVEL DOMAIN -- .gdn

Apparently this is one of the many new top level domains that have been opened up recently in an attempt to "Internationalize" and "democratize" the Domain naming system.  I then checked my regular (non Spam) e-mail for a recent period.  Not a single one come from the .gdn top level domain.

I know little about them other than the fact that a Russian Company Controls this domain and they have a lot of spammers that reach me.  So I decided to block all .gdn mail, and my spams are way down.

Has anyone else run into this?

    karl timmermans (outlook mvp 2012-2018) say:
    >>
    Outlook still crashes upon opening.
    <<

    Three things that should be taken out of the equation as possible causes

    #1 - You mention that you went through the uninstall/reinstall process. Depending on what you've actually installed which may or not include latest updates, have you checked to see that those have been installed? Go to File --> Office Account --> Update Options

    #2 - What type of email account(s) are involved? (i.e. POP3, Imap, Exchange). Could be that the underlying data file associated with the email account has become corrupted.

    - If POP3 - run ScanPST on the PST file associated with the account
    - if Exchange (including Outlook.com) - with Outlook closed, rename the .ost file associated with the account. Outlook will recreate the file and resync everything with the server next time Outlook opens

    - if IMAP - save as for Exchange with the following caveat. If there are folders in the IMAP folder tree with the added description of "This Computer Only" (i.e. contacts/calendar) and contain data that you need to retain, export those folders to a PST file before renaming the ost file. After you restart Outlook, the email folders will be sync'd with the server. Add the PST file from the export process (if applicable) via File --> Open & Export --> Open Outlook Data File --> browse to the PST file. When done, go to File --> Account Settings --> Data FIles tab --> highlight the PST file just added and set it as the default data file for the profile. This is where your default contact/calendar folders are stored.
