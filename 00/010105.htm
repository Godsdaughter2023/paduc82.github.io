How to transfer files from Word 2007 on my old computer to word 2016 - Microsoft Community

pete e46 say:

How to transfer files from Word 2007 on my old computer to word 2016 on my new computer

How do I transfer my files on Word 2007 on my old computer to Word 2016 on my new computer

jay freedman say:

There are two different things that may be implied by your question:

(1) How do you get the files from the old computer to the new computer?

If both computers are attached to the same network, you can use File Explorer to simply copy the files from a folder on the old computer to a folder on the new computer. If you don't have a network, you can use an online file storage service such as OneDrive or Dropbox or Google Drive; upload the files from one computer and download them to the other computer. You can use a USB memory stick or a writable CD to physically carry the files from one place to another.

(2) How do you get Word 2007 documents to work in Word 2016?

They just work! The file formats are identical. You'll notice, however, that documents that you created in Word 2007 will, when opened in Word 2016, show "[Compatibility Mode]" in the title bar after the file name. That means that Word 2016 recognizes that the document came from an older version, and it will disable those functions that didn't exist in Word 2007. If you don't care about that, you can just leave the document in that mode; you can still open, edit, print, etc. If you want to use the newer features, open the document, click File > Info > Convert. That will enable all of Word 2016's features and remove the Compatibility Mode notation.

john w. vinson say:

A table HAS NO ORDER.

 

It should be viewed as an unsorted "heap" of records. Autonumber values are arbitrary and should not be assumed to have any meaning.

 

If you want to see the records in alphabetical order, it's easy - create a Query and sort by the name field. You can base a Form (for viewieng or editing) or a Report (for printing) on this Query; in the report, use the Sorting and Grouping feature to define the sort order.

 

Table datasheets should be kept "under the hood" - they're not designed for interacting with the data; that's what forms are for.

john korchok say:

A1: It depends on what format is used when the table is pasted  from the other program. The default paste for Excel 2013 in Word 2013 is HTML format, and styles can be applied to this in Word to update the appearance. However, if some other paste format was used via the Paste Special command (i.e. bitmap or metafile formats), you may not be able to change the appearance. You can tell by trying to click on the pasted table and select text. If you can't select, you can't reformat.

A2: Again, your results will vary with the Paste format. I recommend spending a little time with Paste Special to see what format works best in your situation. Pasting as Unformatted Text or Unformatted Unicode Text maintains the appearance of the Word table.

A3: If you want to update data and have the appearance maintained, then you should create an Excel template with formatting that matches your Word document, format spreadsheets with that Excel template and then Paste Special using the Link option.

A4: If you don't need to update the data from an Excel link, then you can paste from Excel into a styled table in Word (see A2). If you need to update the linked Excel file, then the technique in A3 is better, but it is not a Word table.
