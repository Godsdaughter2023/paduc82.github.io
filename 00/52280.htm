lee fuller say: Actually.. this appears to be a simple issue of some of the "user" files being "read only".

I had this same issue - and rather than setup security access for "Everyone" to be able to read/write/delete, etc.. I simply took the "read only" attribute off of the areas I wanted access to that were safe.

Not having problems with "Window+Key" combos.  They seem pretty straight-forward to me.  But that's another discussion.

Summary:
  1. Navigate to the directory above the one you're having trouble with.  (For example, to remove the limitations to your Desktop - navigate to c:\users\<usernamehere>.  Then you'll see "Desktop" listed.)
  2. Right click on the directory you want to "unlock".  (In the example I used in #1, "Desktop" is what I want to unlock.)
  3. Select "Properties", then un-check the "Read Only" attribute.
  4. Click "Ok" - and if appropriate, tell it to apply to all files, folders and sub-folders.  (This is assumed to be the case if you're doing this here.  Otherwise, you would navigate to specific files and folders and unlock them.)

Voila!  I have access to the files and folders again.


Now.. why this happened in the first place, I'm not sure.  That's for someone smarter than I to report to all of us.  But this did the trick without any changes to security settings (well.. TRUE security settings.. obviously "Read Only" IS a security "attribute").
