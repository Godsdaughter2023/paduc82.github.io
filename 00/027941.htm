Word 2008 doesn't do VBA so you will need some applescript. I expect there's something better than this out there. Ask again if you need help installing and using Applescripts

-- script to save the active document in .pdf format

tell application "Microsoft Word"

  activate

  tell the active document

    try

      set theFullName to the full name as alias

      set theFullNameText to theFullName as text

      -- if we got here, the file has been saved to disk at some point

      -- but save it again before continuing

      if not saved then save

      tell application "System Events"

        set theFileInfo to properties of theFullName

        set thePath to the path of theFileInfo

        set theExtension to the name extension of theFileInfo

      end tell

      if theExtension = "" then

        set thePDFFileName to thePath & ".pdf"

      else

        set theFullNameLength to the length of theFullNameText

        set theExtensionLength to the length of theExtension

        set thePDFFileName to (text 1 thru (theFullNameLength - theExtensionLength) of theFullNameText) & "pdf"

      end if

      save as file name thePDFFileName file format format PDF

    on error theErrorMessage number theErrorNumber

      if theErrorNumber = -43 then

        display dialog ¬

        "You haven't saved the document yet - please save it first in the usual way"

      else

        display dialog ¬

        "The following error has occurred:" & return ¬

        & "Number: " & theErrorNumber & return ¬

        & theErrorMessage

      end if

    end try

  end tell

end tell

An on-click embedded audio file vs Animation in Powerpoint?

Hello All,

I have a question about embedded audio-file vs animation in Powerpoint:

If an embeded audio file needs to be clicked on to be played on a slide in powerpoint, would it be considered a Powerpoint animation because of the click it needs?

Here is why the answer to this question is important to me:

I took part in a competition at our University. In the competition's regulation document, it is mentioned that "Slides may contain embedded audio and video, but may not include animation." I had an embedded audio file on a static powerpoint slide. The committee decided that since my audio file needed to be clicked on to be played and not automatically played, it was an animation in Powerpoint, and as a result a violation to the competition regulations. But according to the following links, I consider it as an embedded audio file. 

https://support.office.com/en-us/article/Embedded-and-linked-sound-files-in-a-presentation-0e82ab1c-c655-44fa-868a-80b0d4f39414

I had used the same procedure that is mentioned in the following link to insert an audio file to my powerpoint 2011 static slide, it looks the same as in the photo in the following link, which has a click button to play the sound. I have not used animation tab or anything in the animation tab. 

https://support.office.com/en-US/article/Add-audio-to-your-slide-show-180F7035-95AA-4204-B66E-389125CEABFE

bob jones aka: cybertaz say:

FWIW, I'm in total agreement with Steve. By the interpretation of the "committee", advancing to the next slide would be animation if you have to click to do so rather than using slide timings, as would be hyperlinks on a slide. Quite frankly, IMO that construal is ludicrous.

In PPT, Animation Effects are specific sets [Entrance, Emphasis, Exit] of features of the program which impose transformations on the content to which they are applied. Whether Animation Effects are activated on click or by some other method is a property of the Effect and is irrelevant in regard to defining anything as an animation.
