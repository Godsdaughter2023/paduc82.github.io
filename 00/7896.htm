You're very welcome. I hope that kludge will help you for a temporary basis. I'd like to make a couple of recommendations for a long term solution, so I hope you will accept these in the good faith that they are offered.

Method 1:
Change the input to 8, 8.25, 8.5 or 8.75 for 8:00, 8:15, 8:30 and 8:45. This will allow you to directly add or subtract the times and the result can be directly multiplied against an hourly wage in dollars and cents. There are some functional limitations with this method vs dealing with actual time values and the data input operator needs to remember that a half hour is .5 and not .30. To show an actual time value, divide the individual values or a formula containing the values by 24. e.g.:
=(16.25-8.5-0.5)/24      (<end time>-<start time>-<lunch break>)/<hours in a day> 
... yields 7:15 and,
=(16.25-8.5-0.5)*10.00      (<end time>-<start time>-<lunch break>)×<wage> 
... yields $72.50,

Method 2:
Change the method of input to true time values like 8:00, 8:15, 8:30 and 8:45. This is generally a little more time consuming for the data input operator as a colon is required for each entry. The bonus is that true time values are used and there are no limitations as to any manipulation that can be used upon time values. With this method, you must multiply by 24 to get a mixed number with hours as the integer portion and minutes as the decimal portion to multiply against a wage.
=16:15-8:30-0:30      (<end time>-<start time>-<lunch break>) 
... yields 7:15 and,
=(16:15-8:30-0:30 )*24*10.00      (<end time>-<start time>-<lunch break>)×<hours in a day>×<wage> 
... yields $72.50.

To sum up, I'd recommend that you pick one of the two methods described above and stick with it. Your current protocol tries to use half of each and the further you go down that road, the more problems you will have. A worksheet's formulas should not have to compensate for each value entered.

jeeped say:
I have tried using the time formats in excel but they display odd things either 00:00 or 8.30 entered shows up as 7:12...

The only clue you've left us is "or 8.30 entered shows up as 7:12". This tells me that you are entering 8:30 as 8.3 and formatting it with two decimal places. In another post, you mentioned "need to use base 60" so I'm sure you are aware of the pitfalls with your flawed method of time input. Dates and times in Excel are numbers with a full day being 1 starting with Jan 1, 1900. Today happens to be 41003. Hours are a decimal portion of a day so today at noon will be 41003.5. 8.3 is values as 8 days, 7 hours and 12 minutes when formatted as a datetime. Formatting it as a time simply removed the 8 days from display, not from the underlying value.

In order to get a true time that you can add or subtract against, convert the base 10 mixed number to a true time value. With 8.3 in A1 use this,
=TIME(INT(A1),MOD(A1,1)*100,0)
to get 8:30 or 8 hours and 30 minutes. With Start Time in A1 and End Time in B1 use,
=TIME(INT(B1),MOD(B1,1)*100,0)-TIME(INT(A1),MOD(A1,1)*100,0)

Format as desired. As mentioned above [h]:mm is probably best.

Onenote 2010 check marks with strike-through effect - Microsoft Community

keanivey say:

Onenote 2010 check marks with strike-through effect

Is there anyway to set up the To Do tag (Check Box) so that when it has a check mark in it,  the adjacent text has a strike-through effect? Highlighting the text would work as well.

Does anyone know if this is possible or how to do it? 
omer atay say:

You can do this using a macro through the Onetastic add-in for OneNote. For this you need to create a macro that sets the tagCompleted and strikethrough properties of the selected paragraph to "true".

 

You can find more info about macros and download the add-in at: http://omeratay.com/onetastic/?i=macros

 

Once you install the addin, click on New Macro on the ribbon then click Import. Then paste the following XML and hit OK. Then Save. The macro will be added to the ribbon under the Content dropdown:

 

<?xml version="1.0" encoding="utf-16"?>

<Macro name="Tag Complete &amp; Strikethrough" category="Content" description="Completes the Todo tag and strikes through the selected paragraph">

  <For each="Paragraph">

    <That hasProp="selection" op="not" value="none" />

    <ModifyProp name="tagCompleted" op="set" value="true" />

    <ModifyProp name="strikethrough" op="set" value="true" />

  </For>

</Macro>

 

If you understand what it does, you can modify it to do highlight or other things as you like. The macro reads: For each paragraph that is selected (selection is not 'none', means partially or fully selected), set the tagCompleted and strikethrough to 'true'. So you can even select multiple paragraphs with tags and run the macro and all of them will be strikethrough and completed.

 

For more tutorials on how to write macros, see: http://omeratay.com/onetastic/?i=macro-tutorial and http://www.omeratay.com/onetastic/?i=macro-tutorial-insert-text

 

Let me know if you have any questions.
