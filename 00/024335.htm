dirk goldgar say:

Your conclusion about what's going on seems plausible to me.  If you want to convert possible ZLSes in fields to Null on import, what you can  do is link to the Excel worksheet rather than import it directly, then run an append query to copy the records from the linked worksheet to the table.  In that query, for each field that might contain a ZLS, which you want to convert to Null, use a calculated expression that converts the original field value to Null if it equals "".  So, for example, for a field that in the spreadsheet is named "MyTextField", the cell in the query grid might look like this:

    MyTextFieldX: IIf([MyTextField] = "", Null, [MyTextField])

You would set up each of the problematic text fields in the append query that way.

You can use

Declare Function LockWindowUpdate Lib "user32" Alias "LockWindowUpdate" (ByVal hwndLock As Long) As Long

to lock any update of a given hWmd. So, basiclly, add the declare in the declaration section of the form, and at the start of the OnCurrent procedure, have something like

LockWindowUpdate Application.hWndAccesApp

 

and, on exiting this procedure, have:

LockWindowUpdate 0  

to unlock any update the system may generate to your locked window. Note that if an error occurs, the window will ***appear*** to be frozen. So, add the appropriate error handling to unfreeze it, if an error occur; ie,DON'T FORGET to add also, in your error handling:

LockWindowUpdate 0
