and more...

What it amounts to is an Anatel Wi-Fi card causes major problems with the Win10 updates since 1607. The easiest workaround is to physically remove the Anatel Wi-Fi card from the computer and then allow the update to run.  Just disabling the Wi-Fi card in device manager generally does not allow the update to complete. Nor does disabling or removing the driver. A lot of people have solved their update problem by removing the Wi-Fi card physically from the computer.

A lot of people have reported this problem with Acer / Gateway computers but it has been reported on HPs, Asus, and many others. Please read through those threads to see if it might apply to your circumstance. 

I know this is odd and it seems like none of the normal answers that are constantly given say that a Wi-Fi card will cause this problem. But lots of people have solved there update issues by removing their Wi-Fi card and then letting the update run.

wunders say:

The folder that you see in the C:\Users folder for a user is that user's "profile".  Simply creating a user does not create a profile.  After you create that user, you must logon to the computer as that user in order for the folder to be created in the C;\Users folder.

 

The duplicate "PC-Jim.00x" variant folders would indicate that you have had corrupted profiles in the past and these are "temporary" profiles that yet remain.  Corrupted profiles are almost always caused by File System Corruption, so you should perform a "Disk Check" operation at your earliest convenience, specifying the "Repair" option, to make sure that your file system is in good shape.

 

"Check your Hard Disk for Errors in Windows 7"

  < http://support.microsoft.com/kb/2641432/en-us >

 

A few things you should know:

  • The actual disk check will be performed at your next reboot and executes before Windows fully loads and can't be interrupted
  • The disk check can take upwards of a couple of hours depending on size of disk and amount and type of corruption -- sometimes best performed overnight. 
  • Occasionally, with certain types of damage, it can delete files or cause your system to become unbootable -- so make sure your "important" files are backed up before performing this operation. (By definition, though, "Important" files are always backed up)

HTH,

  JW
