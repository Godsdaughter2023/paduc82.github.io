guruprasad ra say:
How is the email account configured in Outlook (POP, IMAP or an Exchange)?
 In case you are using an IMAP or an Exchange account in Outlook, you will not find ‘Leave a copy of email message on the server’ option in Outlook. It is a feature available only for the POP accounts.
 
If you use IMAP or an Exchange account, emails will get synchronized with the webmail / mailbox. If you delete a message it will also get deleted from the server.
 
In POP accounts if you delete a message in Outlook and have the option ‘leave a copy of message on the server’ enabled, a copy of the email will be still saved on the server.
 
In case you are using a POP account, try the following steps use ‘leave a copy of message on server’ and check if it helps:
 
1.       Click on ‘File > Info > Account Settings > Account Settings
2.       Click on your email account and then on ‘Change’
3.       Now click on ‘More Settings > ‘Advanced’ > Delivery > and you’ll find the option to leave a copy of messages on the server.

  4. A dialog box appears. Click on the Backup button. Word creates a new document and copies all your AutoCorrect entries to it.

  13. A dialog appears Restore Complete. Click on OK.

Switching between forms - Microsoft Community

mark bristoll say:

Switching between forms

I have created an application form that collects a persons personal details (e.g. birthdate, social insurance no., etc.).

In many situations there is also a co-applicant. The personal details collected on the co-applicant are identical to those collected from the applicant.

I would like to create a button on the application form that when selected opens a new form that contains fields that collect the co-applicants personal details. The co-applicant table is a sub-table of the applicant table (i.e. one-to-one or one-to-many relationship established between the two tables)

Can this be done without using vb code? if so, How would one go about doing this?

ken sheridan say: The easiest way would be to place the co-applicant form as a subform within the main applicant form, e.g. on a separate page of a tab control, linked on the ApplicantID/CoApplicantID keys, and for each control whose value you want to mirror that of the parent form set its DefaultValue property along the following lines, e.g. for a DoB control

=Form.Parent.DoB

You can do the same with an independently linked form, which you'd open with code such as:

DoCmd.OpenForm "frmCoApplicants", WhereCondition:="CoApplicantID = " & Me.ApplicantID

In this case the DefaultValue property would be like this:

=Forms("frmApplicants").DoB

Note that setting the DefaultValue property of a control does not per se initiate a new record, it merely inserts values ready for when a new record is inserted, which only takes place when the user inserts further data in the form, changes one of the default values manually or the value of a control is inserted by code.  If the user exists the form without adding further data no new record will be inserted into the table.

BTW you don't really need two separate tables for this.  You can have a single Applicants table with a foreign key CoApplicantID column which references the primary key ApplicantID column of the same table.

ken sheridan say: PS:  If you do use separate linked forms it would be advisable either to open the c-applicants form in dialogue mode so that it has to be closed before focus is moved back to the applicants form, or to synchronize the forms.  You'll find examples of both approaches, along with using subforms, in the FormsDemo.zip in my public databases folder at:
