Put footnote entries into index - Microsoft Community

ruudibear say:

Put footnote entries into index

When I create an index and mark entries, it is not picking up the entries in the footnotes.

I have also tried marking an entry in the footnote itself and it says no entries marked.

Is there a way round it without going through 2500 footnotes manually

 

Many thanks

 

Ruudi

john mcghie say:
Do not place an index entry field in a footnote. Instead place the index XE field before the word that
precedes the location of the footnote reference.  There is no alternative but to place index tags manually.  For more information see https://wordmvp.com/FAQs/Formatting/CreateIndex.htm

The "Footnotes" are fields of generated text.  Word does not search within the footnotes, because they are not stored on a page, so it could not generate page numbers for text in a footnote.  Word holds all of the footnotes in a separate text story, and "places" then on the pages during printing.  "Printing" in this case, includes "displaying on screen".

That's because a Word file does not contain any "pages".  The text of a Word document is "made up" into pages during output to screen or printer.  That's the first and only time that the pages actually exist, and hence, the first time you can actually determine which page an item will land upon.

Before the pages can be generated, the footnotes must be expanded, and the Index must be compiled, so Word can compute how large they are.  This is a long way of explaining why Word is not looking in the footnotes for Index tags: it couldn't do anything with them if it found any.

By adding your index tag one word before the footnote reference character, you can ensure that the page number in the index will be the page that displays the footnote.

Hope this helps

hansv mvp say:

1) You must set the FilterIndex to 2, because Macro-enabled documents are the second file type in the list.

2) Execute doesn't actually save the document, you have to do that yourself. FileDialog only lets the user specify a path and filename.

 

    Set fd = Application.FileDialog(msoFileDialogSaveAs)
    With fd
        .FilterIndex = 2
        .InitialFileName = ProposedFileName
        If .Show Then
            ActiveDocument.SaveAs2 FileName:=.SelectedItems(1), _
                FileFormat:=wdFormatXMLDocumentMacroEnabled
        End If
    End With
    Set fd = Nothing

   xl.Visible = True
   With xl
      .WindowState = xlNormal
      .Left = Application.Width + 100
      .WindowState = xlMaximized
   End With
 
End Sub

   xlwb.ActiveSheet.Range("A1") = "hello"
End Sub

Note: as I said in my first post that you would have 2 instances of Excel running, one known as the application object (that is the one were you ran the macro to create the 2nd instance) and one known as xl.

Each instance of Excel knows their own open workbooks but they do know nothing about the other instance. The first instance knows an Excel object xlwb as child of xl, therefore we have to explicitely address it.
