a. user say:

I just tested installing the SanDisk U3 Launchpad back on this USB flash drive. This system was present on it from the beginning when I first purchased it because it's a U3 Smart drive, but I uninstalled it.

The U3 installer (LPInstaller.exe) creates two partitions on the flash drive, and it formatted the data partition as FAT32 with 4096 byte cluster, and the U3 system partition as FAT16 with 2048 byte cluster or "allocation unit".

The Windows recommended clusters are 4096 byte for FAT32 and 512 byte for FAT16.

Then I uninstalled the U3 Launchpad again and the uninstall wizard formatted the flash drive. This time I had only one partition for data storage, just like any other regular flash drive. It was formatted as FAT32 with 4096 byte cluster.

I tried transfering two files to it and it all worked well and I could open them and delete them.

Then I tried to right-click the USB flash drive icon again in Computer and choose Format to try to format it. It was already pre-set to 4096 byte cluster and FAT32. Because this is what was already used. So I didn't change it and I just clicked Start button.

About 80% into the formatting process it abruptly stopped. "Windows was unable to complete the format".

I then tried using the Disk Management console from Computer Management. Same thing here, I set it to FAT32 and 4096 byte. Only this time it completed the format sucessfully!

Why doesn't it work when I go to the format tool on the Computer view? What's the difference between the Format tool and the Disk Management? Don't they both use the same dependencies/libraries?

There was actually one time that the Format tool sucessfully completed the format process. But only that one time! I could not repeat that success. It's rather strange that what works once doesn't work the second time.

The 4K cluster size is little bit too large for a 2 GB USB flash drive. A cluster size of 512 byte would be more appropriate. But I find that the FAT32 usually performs better than FAT for most situations. The NTFS is not so good for USB flash drives, it's better for real disk drives. Also FAT is required by default for live Linux systems on stick, or BartPE.

Lesson learned: Don't ever use the Format tool from Computer in Windows. Use the Disk Management console instead!
