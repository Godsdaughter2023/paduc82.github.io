Sub ConvertTxtToNum()

Dim r As Range
For Each r In Selection
x = Len(r.Value)
s = ""
For i = 1 To x
If IsNumeric(Mid(r.Value, i, 1)) Or Mid(r.Value, i, 1) = "." Or Mid(r.Value, i, 1) = "," Then s = s & Mid(r.Value, i, 1)
Next
r.Value = s
r.NumberFormat = "#,##0.00"
r.Value = r.Value * 1
Next
End Sub

When I enter, manually type a number in one of the cell, in one of the rows pasted across, the content causes an error message saying Number Stored as Text.

I am not familiar with any such "error message" per se.  Are you referring to the color corner indicator and the explanation that appears when you click on the "!" indicator?

For Excel to call those "errors" is misleading.  They are merely "helpful" suggestions -- pointers to differences that Excel considers suspicious.  Often, Excel's suspicions are misleading.

In this case, they might actually be helpful, since you might not have intended the difference.

But generally, if such indicators annoy and distract you (I do), you can avoid them by disabling Background Error Checking.  Click on File, then Options, then Formulas.  Uncheckmark Background Error Checking.  Also click on Reset Ignored Errors to clear any pre-existing indicators.

First, I think the real question is:  why is the original data type Text in the original Excel file?

Perhaps the real solution is to fix the source of the data.  If you need help with that, we would need more information about the origin of the data in the source Excel file.

Second, assuming you cannot or do not want to fix the problem at its source, as I explained in a previous response, the problem is not with Format Painter per se.

Instead, the problem is with the fact that you are pasting text data.

So your question should be:  how can you paste the text data as numeric values?

First, let me say that I sympathize with your frustrations.  It would be nice if Excel made it easy to get into a Text Import wizard when pasting from the clipboard.  AFAIK, we cannot.

However, if you open the source Excel file in a separate instance of Excel, there might be a way to convert the pasted text into numeric values.

  1. By "separate instance", I mean that you start Excel, not by clicking on the Excel file icon, but usually by clicking on an Excel program icon.  (Alternatively, from the All Programs list accessible by clicking Start in the lower-left corner of the desktop.)
     
  2. Then click on File, Open to open the file in that instance of Excel.
     
  3. Copy the text data from the source Excel file.
     
  4. In the destination Excel file, click on Paste, Paste Special, Csv.
     
  5. You probably still need to use Format Painter to ensure the format is what you want.
     

The point is:  Paste Special behaves differently when copying from a "different" application -- in this case, a different instance of excel.  We are presented with a different set of options.  One of them -- Csv -- causes Excel to interpret text "numbers" as numeric values.

Is Insider build 10240 the same as the Global Release? *Answer - Microsoft Community

jason[msft] say:

Is Insider build 10240 the same as the Global Release? *Answer Inside*

This question has been asked many times and I wanted to share some insight into the answer.

Short answer:

Yes!  They're the same.

Longer answer:

Insiders received 10240.  After receiving it, Microsoft was still fixing bugs, making small changes, etc.  Insiders received all of these fixes in the small daily updates over the week prior to our official release on 7/29.   So if you have installed each of the small updates from Windows Update (WU), then yes, you are on the same build and version as we officially released.

Restated ->  (Insider build 10240 + few small updates) = Same version of the OS released globally.

You don't need any different license keys, revised builds, and you don't need to clean-flash your PC to be on the same OS that was delivered.  You're already on it!

Hope this helps,
Jason

macropod say:
As I said, image pixels have no inherent dimension, so you can't make them larger or smaller. On my laptop, a display pixel is about 0.01in. On my TV, which has many more pixels, a display pixel is about 0.014in. So, if I view an image at 1:1 scaling (i.e. one display pixel per image pixel) on my TV, it will be about twice as large (40% wider and 40% higher) as on my laptop. In some applications, reading the nominal dpi figure in the image's metadata will affect the default size at which the image is displayed; that's all. Depending on the application, that sizing may be for display or for print.
