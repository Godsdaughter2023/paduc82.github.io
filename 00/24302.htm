marshall barton say:

Your question implies that you are running down the wrong road.  First, there is rarely a real need to use a make table query.  At worst, you should precreate a table and when you want to populate it with a selection of records, run a delete query to empty the table and follow that with an append query to add the desired records.  OTOH, the best is to just use a SELECT query as if it were the table you wanted to make.

 

Second, the only thing about an AutoNumber primary key field that you can rely on is that it is unique (as all primary keys must be).  AutoNumber fields can never be guaranteed to be sequential and will normally have gaps.  There are cases where they can go negative or even become random numbers.  Even if they appear to work as you were using them before, things can and may very well change at any time.

 

You should never assume anything about AutoNumber fields, because their only purpose is as a surrogate primary key  that can be used as Long foreign keys is related tables.  In other words, your idea of a range of AutoNumbers relating to a set of email addresses does not hold water.

https://www.microsoftstore.com/store/msusa/en_US/cat/Compare-Office-suites/categoryID.68155000?icid=All_Office_subnav_09122015_Compare

Look at the features for your specific bundle. If you are lucky, you will find that you don't actually need all of those licenses. Some of the business licenses allow you to install on up to 5 machines, so you may only need 1 instead of 3 licenses, depending on the specifics of how you use Office on those machines. 

This page: https://technet.microsoft.com/en-us/library/mt455210.aspx discusses the 3 possible rollout modes for new features, Current (default), Deferred (which is what you are asking about). 

While in Office 365 there tend to be more updates/new features they have not reached a daily or even monthly pace.  Every month or 2 MS may release some new. For the most part you can ignore these new features, they simply won't apply to your needs.  They will be automagically installed by updates, but you simply won't see them or need to worry about them.

Your bundle shouldn't be on this page:

https://products.office.com/en-us/business/compare-more-office-365-for-business-plans

As you noted, the only way to completely block these new features is to block all updates.  VERY BAD IDEA because MS has included the updates for these new features in the same update stream/method that is used for bug fixes.   You DO NOT want to block the bug fixes! So you can't block the new features.

Bottom line, if you really want to, and your 365 subscription allows for it, you can delay the new features to 4 times per year.

First thing to try is to use Excel in Safe mode - hold down Ctrl while launching Excel.  That prevents add-ins and startup programs from being loaded.

Now try the right-clicking operation that was misbehaving.

a) If it doesn't happen then the problem is caused by some add-in or a workbook that is loaded on startup.  Disable all add-ins ( File > Options > Add-Ins ). 

Move any files from your XLSTART folders.
- They vary according to username, Excel version and Windows version.
- From the immediate window of the VB editor you can find those folders by:
 - ?Application.StartupPath
 - ?Application.Path     ' and then add \XLSTART to the result
 - ?Application.AltStartupPath  ' corresponds to File > Options > Advanced > General > At startup, open all files in

c) If it still happens, try
Alt+F11 to VB editor
Ctrl+G to immediate window, and type
Application.CommandBars("Row").Reset
and hit the Enter key

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
    Dim varInput As Variant
TryAgain:
    If Target.Column <> 1 Then End
    If Target <> "" Then
        If Cells(Target.Row, "H") = "" Then
            varInput = InputBox(Prompt:="Cells H" & Target.Row & " should not be blank", Title:="Warning", Default:="Completed")
            If varInput = "" Then
                GoTo TryAgain
            Else
                Cells(Target.Row, "H") = varInput
                Select Case Cells(Target.Row, "H")
                    Case "Completed"
                        Cells(Target.Row, "H").Interior.ColorIndex = 1
                    Case "Over time"
                        Cells(Target.Row, "H").Interior.ColorIndex = 2
                    Case "Else"
                        Cells(Target.Row, "H").Interior.ColorIndex = 3
                    Case "ElseElse"
                        Cells(Target.Row, "H").Interior.ColorIndex = 3
                End Select
            End If
        End If
       
        If Cells(Target.Row, "K") = "" Then
            varInput = InputBox(Prompt:="Cells K" & Target.Row & " should not be blank", Title:="Warning", Default:="Completed")
            If varInput = "" Then
                GoTo TryAgain
            Else
                Cells(Target.Row, "K") = varInput
            End If
        End If
    End If
End Sub
