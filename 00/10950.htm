Sub Init()
  'Setup a scenario
  Dim R As Range
  Dim i As Integer
  Dim L As ListObject
 
  Cells.Clear
 
  'Create some data
  Set R = Range("D5")
  For i = 0 To 10
    If i = 0 Then
      R.Offset(i, 0) = "Nr"
      R.Offset(i, 1) = "Value"
    Else
      R.Offset(i, 0) = i
      R.Offset(i, 1) = Chr(i + 64)
    End If
  Next
  'Convert as table
  Set L = ActiveSheet.ListObjects.Add(xlSrcRange, R.CurrentRegion, , xlYes)
  L.Name = "MyTable"
End Sub

Sub Test()
  'Sort the table by value descending
  SortByHeader ActiveSheet.ListObjects("MyTable").Range, "Value", xlDescending
End Sub

Private Sub SortByHeader(ByVal Table As Range, ByVal Header As String, _
    Optional ByVal Order As XlSortOrder = xlAscending)
  Dim R As Range
  'Search the header in the first row
  Set R = Table.Rows(1).Find(Header, LookIn:=xlValues, LookAt:=xlWhole)
  'Found?
  If R Is Nothing Then Exit Sub
  'Sort this column
  Table.Sort R, Order, Header:=xlYes
End Sub

nsilmike say: Playlists are stored locally on the PC... they're not exactly 'specific' to a PC, but they don't anticipate being accessed from multiple networked PCs.  It probably made sense to do it this way because the software is essentially a sync hub for the zune device or wp7.  In that scenario, there is a home or master PC controlling sync, including playlists.  If you connect to another PC, you would either link there, or guest sync only.

Anyway, you can try copying the playlist file (playlistname.zpl) from the PC where it's created to the other PCs, and it may work.  the playlist file that resides on the PC is just a list with pointers to the file path.  (Now a second limitation may come up if you have a zune pass, which is limited to 3 PCs.  If your network is larger than 3, you may run into problems on the 4th PC...)
