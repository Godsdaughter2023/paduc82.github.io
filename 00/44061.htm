Windows 7 Live Movie Maker Aspect Ratio Problem - Microsoft Community

long in the tooth say:

Windows 7 Live Movie Maker Aspect Ratio Problem


The Requirement

1. To use Windows 7 Media Center (WMC) to record standard-definition wide-screen 16:9 Freeview DVB-T TV programmes from an AverMedia H752 PCIe TV card and store them on the C: drive in Windows Recorded TV format (.wtv).

2. To use Windows 7 Live Movie Maker (WLMM) to edit the .wtv files to top and tail them and remove any adverts in between.

3. To use Windows 7 DVD Maker (WDM) to create a DVD for viewing later on a TV DVD Player.

The Problem

The WLMM editing process distorts the 16:9 video clip frame content by compressing it laterally to 4:3 and then padding it out with vertical black bands to refill the 16:9 frame. This distortion problem persists throughout the editing and rendering process and results in a distorted DVD video with black bands at both sides and 'tall thin people' on the images.

Program Settings.

All the settings in the Windows programs used were set to Widescreen 16:9 and not Standard 4:3.....

1. In WMC: Tasks / Settings / General / Windows Media Center Setup / Configure Your TV or Monitor / Widescreen (16:9).

2. In WLMM: Project / Widescreen (16:9)

3. In WDM: Add Pictures And Videos To The DVD / Options / 16:9 (under DVD aspect ratio)

Investigation

When the recorded .wtv TV programme is played back in WMC it displays correctly as widescreen 16:9, without black bands or distortion.

When the .wtv file is read in to WLMM it displays correctly as16:9 in the Preview Pane, without black bands or distortion. Thus at this stage WLMM is performing as would be expected.

However, when WLMM processes the 16:9 video clip and places it on the Storyboard Panel (with film sprocket holes) it is reduced to 4:3 aspect ratio and black bands are placed at the sides to pad it out to refill the 16:9 frame. (I have a screen capture image that shows thsi, but as I don't know how to paste it here, I have used it as my photo at the top of the page!)

Conclusion

There must be a bug in the software. There is no reason for WLMM change the aspect ratio, or insert black bands. It should retain the same aspect ratio of 16:9 that it had when it was initially read into WLMM. This should be reported to Microsoft.
