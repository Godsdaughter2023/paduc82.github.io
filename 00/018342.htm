Many of us are complaining about file explorer thumbnails not being right in build 9860.  I think I found a way to fix it.

Apparently the system files called 'thumbs.db' aren't compatible between 9841 and 9860.  To fix it on my 9860 system I changed file explorer to uncheck 'Hide protected operating system files'.  Then in file explorer went to the high level folder with all my videos and did a search for 'thumbs.db'.  When the search result populated, I deleted all the thumbs.db files.  The thumbs immediately started to rebuild and I now have all the thumbnails back.

Creating more than one Index in Word 2010 - Microsoft Community

michael harwood say:

Creating more than one Index in Word 2010

Hi

I have just completed a book which requires 2 indexes: one for Surnames and another for Place Names. I have created the Surname Index, but can't find a way of adding a second index. Can you help.

suzanne s. barnhill say:

In order to have more than one index, you must tell Word which index a given index entry belongs to with a \f switch in each XE field and in the corresponding INDEX field (this is called a "Type" switch in the XE field and an "Identifier" switch in the INDEX field). So, for example, you could have XE and INDEX fields with switches \f "s" for surnames and \f "p" for place names.

The default switch is "I"; I think you're probably okay to leave your existing index as is (without a switch) because your new index for place names will include only the entries that contain the switch you assign to those entries, and those entries should not appear in the existing surname index (which has an implied switch of "I" (or "i"). You might want to test this on a small selection before going too far with marking place name entries.

For more information on this and other field switches, see the Help articles on theINDEX field and XE field.
