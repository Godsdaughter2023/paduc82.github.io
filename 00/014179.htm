selection combo boxes in a form returns wrong record in table. - Microsoft Community

markhogan say:

selection combo boxes in a form returns wrong record in table.

selection combo boxes in a form returns wrong value in record in table. Eg lookup table has IDemployee & employeename fields, combo in form displays employeename. When viewing record in tblemployeeTask the IDemployee field will show 6 instead of 7, indicating another employee altogether. Bound column is set to 0 , column count set to 1. Referential Integrity is inforced! What might be causing the problem? Note:Cannot change field value in form, but can change IDemployee field in tblEmployeeTask which then changes the value in combo box on form to the wrong employee.
john w. vinson say:

Your combo box should - must! - contain the EmployeeID field as well as the name:

SELECT tblEmployees.EmployeeID, tblEmployees.[Name] FROM tblEmployees
ORDER BY [Name];

Set the column count to 2 to include both columns; the bound column to 1 to store the ID rather than the name; and the ColumnWidths to

0";1.5"

to hide the ID field from view and display the name.

Note that Name is a reserved word ( a combo box has a Name property, as does a field, a table, a form, everything) and should not be used as the Name of a field; and that you should split the name field into FirstName and LastName fields (at least) for more convenient display and searching.

Bound column should be 1, not 0. It's confusing - the Column() property is 0 based, but the bound column property is 1 based. This is what's causing your wrong results; the 0 column of a combo box is the row position.

Putting combo boxes in your tables can work but it can also be massively confusing: see http://www.mvps.org/access/lookupfields.htm for a critique of this feature. In any case it is certainly NOT necessary.

How to change Default setting in word 2010

suzanne s. barnhill say:
  1. Press Ctrl+D to open the Font dialog.
  2. Select your desired font.
  3. Click Set as Default.
  4. In the ensuing dialog, choose the radio button for "All documents based on the Normal.dotm template" and click OK.

If you find, upon restarting Word, that your change has not been saved, then something is preventing Normal.dotm from being saved. This is usually the result of a poorly written add-in, and that is something you should investigate later. For now:

In Word, press Alt+F11; the Visual Basic Editor opens. Press Ctrl+G to activate the Immediate window. In that window, type

     NormalTemplate.OpenAsDocument

and press Enter. Normal opens, but note that it may not be on top of all windows. You can close the Visual Basic Editor. Make the desired changes to the Normal file (the steps above), and then save and close it. If the changes you have made have not been recognized as changes (the Undo button is not activated), press Spacebar, then Backspace to "dirty" the file, thus ensuring that the template will actually be saved.

rohn007 say:
The "1 user 1 PC" is a "Product Key Card" or "PKC" license.

It is trivially price discounted because it is "disposable". You can only install it on one computer. Period. The license has to always stay with that computer (say you sell/give the hardware to someone) and cannot EVER be transferred to a new computer. Worst case, if your computer dies the same day you install a PKC office, so does the Office. You would have to buy a whole new copy of office!  Good for MS, not so good for you.

There is one more distinction to be made. You can buy a retail or "FPP" license as an online download or in the "retail Boxed CD/DVD".  The tricky part is that typically they are both priced the same. In other words, you get the CD/DVD for "free" in the Boxed version, making it a much better deal.
