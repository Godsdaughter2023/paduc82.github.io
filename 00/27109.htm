Here are the basics:
How to Use the Macros in This Book

You may want to use the macros in this book in your own computer system. This is a great way to add features to your system and to use the techniques shown in this book. You may be curious how you can copy the macros in this book and paste them directly into the VBA Editor.

The answer to this question depends on the format in which you are reading this book. WordTips: The Macros is available in two formats: PDF and Word document.

If you have the Microsoft Word version of this book, then you can use Word’s capabilities to simply select and copy the macro code lines, and then paste them into the desired place in the VBA Editor. (If you are working with macros, you undoubtedly already know how to copy and paste text.)

The defacto format for most electronic books is PDF format. This type of file is opened using Adobe’s Acrobat or Acrobat Reader programs. In this format you can easily read, search, and print information. If you want to copy macro code lines from the PDF file and place them in the VBA Editor, follow these steps:

1.      Display the page containing the code lines that you want to copy.

2.      On the Acrobat toolbar, click the Text Select Tool.

3.      Position the mouse pointer (which should now look like an I-beam) at the beginning of the first line of code you want to copy.

4.      Click and hold down the mouse button.

5.      Drag the mouse pointer to the end of the last line you want to copy. (The text behind the mouse pointer, back to the starting point, is selected.

6.      Release the mouse button.

7.      Press Ctrl+C to copy the text to the Clipboard.

8.      In the VBA Editor, position the insertion point at the location where you want to paste the code.

suzanne s. barnhill say:
By default, the Table of Figures is compiled from paragraphs that include the { SEQ Figure } field, which you have overwritten with a manual number. So you can't use the out-of-the-box Table of Figures. What you can do, however, is base the TOF on the Caption style, provided you haven't used it for anything other than figure captions. To do this, in the Table of Figures dialog, click Options... In the Table of Figures Options dialog, check the box for Style. The Caption style will be selected automatically. If you do have other captions, you can apply a different style to the figure captions and base the TOF on that.

charles kenyon say:

The table of figures depends on the label "Figure" is the Caption's SEQ field.

The field code for the Table of Figures field is { TOC \h \z \c "Figure" }.

That is, this is really a special Table of Contents field.

The Caption field should look like this:

{ SEQ Figure \* ARABIC }

To see field codes you can toggle the view using Opt+F9 or by changing the Preferences.

Here is the documentation on TOC field switches:

https://support.office.com/en-us/article/field-codes-toc-table-of-contents-field-1f538bc4-60e6-4854-9f64-67754d78d05c?ui=en-US&rs=en-US&ad=US

\c "SEQIdentifier"

Lists figures, tables, charts, or other items that are numbered by a SEQ (Sequence) field. Word uses SEQ fields to number items captioned with the Caption command (References > Insert Caption). SEQIdentifier, which corresponds to the caption label, must match the identifier in the SEQ field. For example, { TOC \c "Figure" } lists all numbered figures.

The \h means that it acts as a hyperlink in the Table of Figures.

The \z hides the tab leader ...... and page number in web view (since pages don't make sense in web view).

How to include only part of a Caption in the Table of Figures
- Use a Style Separator
 To do that:

1. Insert a paragraph break after the text you want to keep (in the middle of your Caption). Press Enter to do this.

2. Change the Style of the newly formed paragraph containing the text you do not want in your Table of Figures to something other than the Caption Style. If you want that text to look the same as the Caption text, create a new paragraph Style based on the Caption Style and use that.
3. Return to the end of the paragraph containing the text you wart to keep and press Ctrl-Alt-Enter. That will convert the paragraph break after it into a Style Separator, allowing both 'paragraphs' to flow together on the same line and, if you refresh the Table of Figures, the text after 'paragraph break after the text you want to keep will disappear from it.

More on the Style Separator: http://www.addbalance.com/usersguide/styles.htm#Style_Separator

Mac versions may not yet have the Style Separator. If not, select the paragraph mark at the end of the part of the caption you want to keep and use Font formatting to mark it as hidden. That should give the same effect.

It's generally best to leave autonumbers "under the hood", not exposed to user view and not assigned any meaning. They will ALWAYS have gaps - if you start a new record and hit <Esc> after a single keystroke to cancel it, an autonumber value will be "used up" and never used again.
