tiago.figueiredo say:

Is your mailserver with Godaddy on a shared server, by any chance? About a year ago I had the same problem.

What I found out was that, despite Godaddy's shared servers not being on the Spamhaus' PBL, my domain's mailserver was relaying my broadband provider's IP in the e-mail headers as the source IP, instead of GD mailserver's re-enveloping with its own IP (as it should really). Since, according to a Spamhaus technician, all broadband providers are listed on the PBL, Microsoft's servers block them. Note: the PBL is not a blacklist, but a list of IPs that shouldn't be sending unauthenticated emails. So, in my case at least, it was a problem with Godaddy's mailserver, for which they offered no solution other than giving up using the already paid for hosted email accounts and buy an additional Microsoft 365's mail service, for which you pay per mailbox. Fishy.

My suggestion, if you haven't done this already, check your e-mail headers and trace the source IP that is being relayed:

  1. Open a Microsoft blocked message that you Cc to another address of yours that is able to receive e-mails sent from your mailserver
  2. Click on the File tab and then Properties
  3. Copy the e-mail headers
  4. Go to http://whatismyipaddress.com/trace-email paste the headers and trace the e-mail's Source IP
  5. Now just go to https://whatismyipaddress.com and identify your internet access IP
  6. If they match, you're in trouble. You may go to https://www.spamhaus.org/lookup/ and confirm if your internet access IP is on the PBL, bu that IP from the list (only your broadband provider could, but since all of them are on the list, I doubt it will happen)

You may want to run a number of tests with different e-mails, since - at least in my case - GD's mailserver behaviour was also erratic, sometimes forwarding my internet access IP, othertimes forwarding an unknown IP (127.0.0.1).

If the scenario above matches yours, I suggest you start looking for an alternative hosting company. In my case, despite their admission of the problem, GD offered no solution, other than the one mentioned above (365).

Good luck.

teylyn say:
Hello,

you won't be able to create these axes settings with Excel. The picture you reference has arbitrarily placed labels. The four green columns are exact quarters of the chart width, but the placing of the axis labels has no numeric regularity. 

You can create such a chart with Excel, and you can place axis labels at these positions and let them show whatever text you want, but Excel will plot a value according to the real axis scale and not the label on the axis, therefore, you may want to massage the values that are plotted to show in the correct place.

First, you need to create a stacked column chart with the green and red columns. The data for this is a table like this:


The chart will look like this:

Remove the legend, format the series, set the gap width to zero, format the blue series red, the red series dark green, and the individual data points of the green series in different shades of green. Set the Y axis to a maximum value of 100. Hide the

Now create a data table for another data series. This will be an XY scatter chart overlayed on the column chart on the secondary axis. For example


Temporarily change cell B5 to a lower value, so the first green bar is a bit shorter. Now select just one of the values in the XY points data and copy it. Select the chart and use Paste Special (Alt - E - S) to open the paste dialog for chart series. Paste as new series. This single value will be stacked onto the first column. 

Select the new data point, right-click and change the series chart type to XY Scatter chart. Now you can restore cell B5 to its original value.

Select the chart, and use the "Select Data" dialog. Edit the new series and select the X and Y range from the table. Format the series markers to a solid fill. Show the Secondary X axis. 
Format both primary and secondary Y axes to have the same max setting. Format the secondary X axes to have a max setting of 12 or so. Set all the axes lines and labels to "none".

Now for the labels. Add another data table:
The first three rows are for the X axis labels, the next three rows are for the Y axis labels. Add two more data series to the chart and select these values. You should now see three data markers on the Y axis and three on the X axis. Show the data labels for these new data points. Edit each label, enter a = sign and click the text value that you want to display in that position. Drag the labels in place, so they are to the left of the Y axis and below the X axis. This should now look something like 


Now, be aware that a data point with an X value of 2.5 will be plotted at the place where the axis shows the text label "3%". So, you may want to come up with a formula that takes the original value and performs a calculation that adjusts the data point value accordingly. Exactly which algorithm you require for this I'll leave to you.

doug robbins - mvp office apps & services (word) say:

Yes, it is normal.  Word remembers the printer last used.

You could use an Autoexec macro in your Normal.dotm template, or another template stored  the Word Startup folder, with the following commands to force Word to start with a specific printer

  

With Dialogs(wdDialogFilePrintSetup)
    .Printer = "HP DeskJet 670C Series v11.0 on LPT2:"
  .Execute

To get the name of the printer, use:

Debug.Print ActivePrinter

You can then copy/paste the name of the printer from the “Immediate Window” (Ctrl-G) into your code.

If you go to Page Setup and choose the Different First Page option, you will find that you now have  separate Header and Footer for the first page. So insert the Text Box into the Header or Footer and move it into position.
