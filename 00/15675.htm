kergga say:

Start with 'Data Validation' on the Data Ribbon to create your in-cell dropdown list. Select validation type 'list', then you can hand type (if you only have a few options) or point the list source to a range of cells.

 

Then to color an entire row red, use conditional formatting (on the 'home' ribbon). For example, if your dropdown is in column A starting in row 2, go to that cell and click conditional formatting/New rule/use a formula/ then enter the rule description just like a formula,

=$A2="Systematic Review"

then click the format button and click the fill tab and select your preferred color (red)

 

Then click ok and ok again to get out of conditional formatting, and finally,

 

with that cell selected, click 'format painter' on the home ribbon, and then select all the cells that you want to have this condition applied to.

 

Note that the use of a column anchor ($) in the formula was intentional in the $A2

 

If you are using a different column, or starting on a different row, adjust that cell reference accordingly to match the first drop-down cell location that you select to initially create the conditional formatting.

Here is the logic:

1. Use the Find functionality programmatically to get search results

2. Iterate through each cell in search results

3. Programmatically apply formatting to the desired text.
    To do so you use the syntax: Activecell.Characters(Start:=<startchar>,Length:=<numchars>).Font (or any other formatting property)

Here is a sample code. This adds text "this is test" in a cell.
The word "test" is has Red font color and is in Bold.

Sub test()

' ddd Macro
'

    ActiveCell.FormulaR1C1 = "this is test"
    With ActiveCell.Characters(Start:=1, Length:=8).Font
        .Name = "Calibri"

        .Size = 11
        .Strikethrough = False
        .Superscript = False
        .Subscript = False
        .OutlineFont = False
        .Shadow = False
        .Underline = xlUnderlineStyleNone
        .ThemeColor = xlThemeColorLight1
        .TintAndShade = 0
        .ThemeFont = xlThemeFontMinor

greginmich say:

The functionality that's used in the System File Checker is also silently protecting critical system files in real-time in Windows 2000 and Windows XP (Windows File Protection). In Vista and Windows 7, Windows Resource Protection is used to protect all critical system files and other critical system resources, like registry entries. WRP is based on proactively restricting access to system files and resources rather than simply replacing them when they’re deleted or changed:

http://support.microsoft.com/kb/222193

 

http://msdn.microsoft.com/en-us/library/aa382503(VS.85).aspx

 

The most impressive mechanism for system file protection in Windows Vista and Windows 7 is the automated check done by Code Integrity (CI). Code Integrity starts with a self-test, and then verifies first the boot file images (including ntoskrnl.exe), and then the kernel-mode system file images. It also checks kernel-mode drivers, which share the same elevated code-privilege level. This sequence of integrity checks is repeated every time the operating system is loaded:

http://technet.microsoft.com/en-us/library/cc733982(WS.10).aspx

 

 Additionally, 64-bit versions of the operating systems are protected by Kernel Patch Protection (PatchGuard), which prevents the kernel tables and the ntoskrnl.exe image (according to this source:  http://uninformed.org/index.cgi?v=3&a=3&p=7 ) from being altered in kernel memory space. There are also other mechanisms that protect kernel-mode memory space from specific types of attacks. So, while the battle is never over, the Microsoft engineers have certainly shown some initiative in protecting the integrity of their operating systems.
