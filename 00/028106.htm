richie weldon - msft say: Hi Johan,

I'm sorry to hear about your frustration with Active Sync, WMDC, and Microsoft My Phone.  As a design decision, Microsoft My Phone will not sync contacts, calendar, tasks, or other data if the software detects that a WMDC or Active Sync connection is set up on the device. 

In order to get your contacts back, open the My Phone application and uncheck those data types (Contacts, Calendar, Tasks, Text Messages) from the Sync Options screen.  You will then need to resync your phone with WMDC or Active Sync.

There is a big difference between Active Sync and WMDC.  The former (Active Sync, or sometimes referred to as EAS) allows your phone to synchronize certain data types (Email, Contacts, Calendar, and Tasks) over the air to an Exchanged based email server.  The later (WMDC) will sync those data types through a tethered connection between your phone and your PC.  For more information on EAS and WMDC, I suggest using one of those forums or searching online using your search engine of choice.

Richie Weldon

julie fka profjulie say: Actually there are two types of ActiveSync syncing that can be performed on a windows mobile device.

 
If you sync between a personal computer and a windows mobile device you'll connect the windows mobile device to the computer via USB  cable (or via Bluetooth connection).  If the computer runs:
  • Vista or Windows 7: you'll install Windows Mobile Device Center to the computer, set up a partnership and sync your data;
  • XP: you'll install ActiveSync to the computer, set up a partnership and sync your data
If you sync Contacts, Appointments and Tasks between a personal computer and a Windows phone, you can also sync this data to My Phone.
:
Another type of sync occurs between an Exchange Server and a windows mobile device.  In this instance, you sync via wireless connection to an Exchange Server to sync Contacts, Appointments and Tasks.  If you sync this data from an Exchange Server, the assumption is that the Exchange Server and the data on the server does not "belong" to you (since most of the time this is a work-related arrangement).  When you end that partnership, the data is removed from your device because it never belonged to you. 

If you sync to an Exchange Server, you can still sync to your computer (as described in number 1 above), but you will not be allowed to sync any Contacts, Appointments and Tasks that are stored locally on your computer.  If you sync to an Exchange Server, you will not be able to sync Contacts, Appointments and Tasks to My Phone.

It sounds like you sync to an Exchange Server.  You can back up the Contacts, Appointments and Tasks that are on your windows phone to your computer by copying the PIM.vol file from the root directory of the windows phone to your computer, but you will not be able to do anything with the data (it is not importable to Excel or viewable by any other application).  If you terminate the exchange server partnership the data will be removed from your windows phone, but then you could copy the PIM.Vol file back to your phone to restore the data back to the phone.
