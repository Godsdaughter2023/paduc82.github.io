computer clark say:

WORK-AROUND SOLUTION: who knows where the defect in WLM is, I tried many, many things and I am pretty computer savvy but nothing worked.  I did come up with a work-around solution however: replace the account with an identical one and move your email files there, sort of what LeviJK above is referring to. 

Create a new account with the same settings and servers and preferences, etc. as those of the problem account but with the new password.  When you are doing this you have the option to make it the default account, so take this option if you are replacing the default account.  The new account will have to have a slightly different name but that won't effect anything.

You can re-set the problem account so that it does not check for mail if you wish.

Then move all the existing mail from the problem account to the new (almost identical) account, to their corresponding new folders of the same names.  You can even move custom folders.

When the emails from the folders of the problem account are all moved and empty, delete the problem account.

Voila, email goes on with the new password as if nothing had changed.
