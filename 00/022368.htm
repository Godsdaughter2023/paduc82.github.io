    Here is such a macro: 

    Sub AutoFormatTypeToggleSmartQuotesSetting()
' AutoFormatTypeToggleSmartQuotesSetting Macro
' Charles Kenyon
' Toggle the Smart Quotes setting in AutoFormat as you Type
'
    Dim blQuotes As Boolean, strQuotes As String
    blQuotes = Options.AutoFormatAsYouTypeReplaceQuotes
    If blQuotes = False Then
        strQuotes = "on."
        Else
        strQuotes = "off."
    End If
    Options.AutoFormatAsYouTypeReplaceQuotes = Not blQuotes
    MsgBox prompt:="The Smart Quotes option is turned " & strQuotes, _
        Title:="AutoFormat as you type Smart Quotes Status", buttons:=vbInformation
'
End Sub

    Setting your domain email address as reply-to-address will make you reply to the emails you received using your domain email. To set your domain email as a reply-to address in Outlook.com, please follow these steps:

     

    1. Go to www.outlook.com and sign in with your Microsoft account

    2. Click the Gear icon and select More mail settings

    3. Under Writing email category, click Reply-to address

    4. Select Other address

    5. Enter your domain email address

    6. Click Save to confirm changes

     

    Putting your domain email address as send-only account will give you an option to send emails using your domain email address. To set your domain email as a send-only account in Outlook.com, please follow these steps:

     

    1. Go to www.outlook.com and sign in with your Microsoft account

    2. Click the Gear icon and select More mail settings

    3. Under Managing your account category, click Your email accounts

    4. Click the Add a send-only account button

    5. Type your domain email address and click Send verification email

    6. Go to your domain email account and look for the verification email that was sent to you by Outlook.com

    7. Click on the link provided to verify your domain email account

     

    Follow these steps to send emails using your domain email address:

     

    1. Go to www.outlook.com and sign in with your Microsoft account

    2. Click New to compose a new message

    3. On the upper left corner of your screen, you will see your Outlook.com email address and v button. Click the v button and select your domain email address.

    jeeped say:
    If I am interpreting this correctly, those formulas as used in Conditional Formatting Rules to create conditions for multiple column rules.

    First off, a formula in a conditional formatting rule only needs to return a TRUE or FALSE to adjudicate the rule. Since a FALSE can be defined as 0 (i.e. zero) in most cases and a TRUE as anything that is not 0, I would recommend using LEN(...) or maybe ISNUMBER(...) to determine whether there is a date in column D. I suppose you are achieving this with the "" returned value from the IF(), but it is not a method I would use.

    Secondly, if these formulas are used on Conditional Formatting Rules that govern more than a single column, you need to lock the column reference in G1,not just the row. If $G$1 is not used, then column C sees G$1, column D sees H$1 and column E sees I$1, etc. when a single rule governs all three columns.

    And finally, I really do not understand the purpose of the AND() logical join in the formulas like this,

    =IF($D3="","",AND($D3=G$1,$D3=G$1))

    The logical conditions within the AND() appear to be simply repeated.

    I'm not 100% confident in this proposed formula as there are outstanding questions as to the specific nature of the CF implementation but you can try something like,

    =AND(LEN($D3), $D3<=$G$1,$E3>$G$1)

    ISNUMBER() and LEN() are interchangeable in that formula from my uninformed standpoint. You should be able to use either to determine is there is a date in D3. e.g. 

    =AND(LISNUMBER$D3), $D3<=$G$1,$E3>$G$1)

    If you cannot get these suggestions working for you, try to post back with a little more detail on the exact implementation of your CF rules.

    When an anti-virus or security program quarantines a file and moves it into a virus vault (virus chest) or a dedicated Quarantine folder, that file is safely held there and no longer a threat. The file is essentially disabled and prevented from causing any harm to your system through proprietary security routines which may copy, rename, encrypt and password protect the file as part of the moving process.

    Quarantine is just an added safety measure which allows you to view and investigate the files while keeping them from harming your computer. One reason for doing this is to prevent the permanent deletion of a legitimate file that may have been incorrectly flagged (a "false positive") and placed in quarantine. This can occur if the scanner uses heuristic analysis technology which is not as reliable as signature-based detection (blacklisting) and can potentially increase the chances that a non-malicious program is flagged as suspicious or infected. After confirming the file is legitimate, it can be safely restored from quarantine and added to the exclusion or ignore list.

    When the quarantined file is known to be malicious, you can permanently delete it at any time by launching the program which removed it, going to the Quarantine tab, and choosing the option to delete.

    You can view items in MSE Quarantine as follows:
    1. Launch Microsoft Security Essentials.
    2. Select the "History" tab at the top.
    3. Check the "Quarantined items only" box to view items placed in quarantine.

    Keep in mind, however, that if these files are left in quarantine, other scanning programs and security tools may flag them as a threat while in the quarantined area so don't be alarmed if you see such an alert. Just delete the quarantined items after confirming they are malware and subsequent scans should no longer detect them.

    See if this does what you want. Please test it on a copy of your document first.

    Sub Test()
        Dim tbl As Table
        Dim cel As Cell
        Dim wrd As Range
        Dim c As String
        Application.ScreenUpdating = False
        Set tbl = ActiveDocument.Tables(1)
        tbl.Columns(1).Select
        For Each cel In Selection.Cells
            For Each wrd In cel.Range.Words
                c = Left(wrd, 1)
                If c Like "[A-Z]" Then
                    With wrd.Characters(1)

                        .InsertBefore "["
                    End With
                End If
            Next wrd
        Next cel
        Application.ScreenUpdating = True
    End Sub
