boblarson say:

First off, I would just use the form's Before Update event  as the gatekeeper for saving records or not.  No need to use another table.  Just have a boolean flag which the code checks to see if the boolean flag has been set by your save button and then let it pass through if it does.  If not cancel the update.  You can have a cancel button to also cancel the update and then clear the form by using Me.Undo.  This avoids having to use a completely extra table.

 

Otherwise, if you insist on using the method you seem to have your heart set on, just create an append query and a delete query and then call them one after the other from your button's click event.

Hi Sam,

After you delete the user's personal site by following the steps in your article, the personal site will be created automatically once the user access his OneDrive for Business library via app launcher.


If you don’t want the user create it again, after you delete the site, you need to remove the Create Personal Site permission. If you want the users to create it again, you can add it back.
To remove permision: Go to the SharePoint admin center(e.g. https://contoso-admin.sharepoint.com) >user profiles>Under People, open Manage User Permissions>Uncheck Create Personal Site and click OK to save the change.
 

Note: After you delete the personal site, all the files/folder on the site will be deleted. If you recreate it, it will be a new site.
