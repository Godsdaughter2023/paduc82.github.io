jgreg7 say:

I agree with Graham's initial post, however I would like to add to it.

The issue with card stock usually occurs with the feed rollers (usually rubber rollers or roller wheels).  These rollers get a glazing on them that does not grip the paper properly.  They will appear to be clean, but the glazing is still there.  If you clean them with a soft cloth and alcohol, they should work properly.  If the feed rollers are shiny, they will not grip properly.

Another issue if you are using a printer that feeds the paper in a semicircular path (feed is into the machine then comes back out the same way it went in), is friction on the paper itself due to the stiffness of the card stock.  Check the paper path to make sure it is clean and free of anything stickey that would slow down the paper.

Watch the paper when the printing starts, if it makes the feed sounds, but the paper lags, the start of the printing, it is slipping.  As a alst resort, if you have a limited amount of printing to do, you can always apply a slight "push" to the paer to get it started.  This is not the best solution, buts it does work in some cases.

Good Luck.

rob koch say:

The most basic reason for this might be simply the fact that Explorer will open every file within a newly dispayed folder to find and display an icon if one exists.  Since the file is being accessed, MSE real-time protection would automatically scan the accessed files as a result, causing a perceptable lag as a result.  This lag might be even more pronounced when the file system (hard drive and/or PC 'controller') are relatively low performing, which is common with many consumer PC systems today, processor performance usually has less overall effect in this case.

Since MSE is likely to 'remember' that it has already scanned a particular folder, this would result in less delay the next time a folder is accessed, though this status would be lost after a reboot, since it has no way to know if it might have been disabled for a period of time in between and so must assume that all files require scanning again.

Another common reason that MSE might be slow to scan in general is additional real-time anti-malware still remains on the PC, whether intentionally or because it didn't uninstall completely.  Running the correct clean-up tools for any anti-malware product of any sort that was ever installed, especially anti-virus, is highly recommended for anyone with slow scanning of any sort, real-time or not.

Another thing that MSE includes is called a 'pre-execution detection', which allows it to 'run' an executable program in a virtual environment before actually handing it to the operating system itself.  However, this generally only occurs when an executable program is actually launched, so it's not likely to be involved in the simple 'browsing' case discussed here.
