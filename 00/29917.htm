mem-tek say:

I haven't tried the following, but this is what I would do if I had a similar situation:

1. Make a backup copy of the messed up VM's VMCX file which is associated with your current XP virtual machine. Then rename the original messed up VM's VMCX file. It is probably named Windows XP Mode.vmcx, so rename it to Old Windows XP Mode.vmcx. The point of the backup copy is so that you will have a backup copy should the following not work.

2. Create a new XP VM, install all updates, and then configure XP the way you like it (i.e. taskbar, windows explorer, desktop, et cetera). Don't install any applications yet since the point here is to have XP set up the way you like it and with all updates.

3. Close the new XP VM and then change the Virtual PC Settings for the new VM by enabling undo disks.

4. Copy this new XP VM's VCMX file to another location on your computer so that you have a backup of this bare bones XP installation which includes all updates. This will prevent you from having to go through steps 1 through 3 should you ever want to create another XP VM.

5. Install a good antirvirus product (AVG free, PCAV free, or whatever, but NOT the same AV product which you were using) on the new XP VM and then install your applications. I would suggest using the free version of PCAV since it is lightweight and is extremely good at finding malware infected files. You need an AV product with robust detection of infected files before you attach your old VMCX to this new VM and start copying files.

6. (Optional, but recommended): Close the new VM, merge the undo disk for the VMCX, and then copy this merged VMCX file to another location. Keep the Enable Undo Disk setting enabled. Why? Because we are going to assume that your original VCMX file and possibly its data in step 1 are infected with malware. Thus it would be a good idea to have a second backup of your new VM before you start copying files from the old VM.

7. Configure the new VM by adding the old VM's VMCX file as a second hard drive and then start up the new VM.

8. We don't want to directly copy data from the old VM to the new VM because, as you mentioned, the security descriptors are corrupt. Likewise, NTFS streams might also get copied as well, but you don't know if malware has modified those streams. All you want is to copy the data files and let the new VM assign security descriptors to the copied files depending on where you copy those files to on the new VM. Rather than using windows explorer to copy files from the old VM to the new VM, I would suggest using the free version of Vice Versa since you can use it to copy entire directories or individual files from the old VM drive to the new VM drive. By default, Vice Versa will not copy any security descriptors or related NTFS streams. That is what you want. You want just the files. Before starting, you need to configure Vice Versa's Profile Settings as follows:

Under File Filters, add an exclusion for desktop.ini

Under Execution, set the Method to Replication (Augment Target).

When using Vice Versa, you will want to use windows explorer to first create any new top level directories (folders) on the new VM's hard drive to which you will be copying files and subdirectories from the old VM's hard drive. Let's say that all of your of your data files from the old VM hard drive to the new VM hard drive. If you installed PCAV, then by default PCAV will automatically either disinfect or neutralize any infected files as they are copied to the new VM's hard drive. In other words, you should get back everything that is recoverable from the old VM's hard drive.

9. Double-check that you copied all of your old data.

10. Close the new XP VM, and then remove the old VM's hard drive from the new VM's settings.

11. Start the new XP VM and make sure that you can access your files and that everything appears to be satisfactory. If yes, then close the VM and make a backup copy of its VMCX file. Then and optionally, merge the undo disk of the new VM, but I would wait a few days before doing so, just to be sure that the new XP VM is not infected.

a. user say:

Hi Suhail,

We suggest you uninstall and reinstall the network adapter to fix this issue. Kindly follow the troubleshooting steps below:

  1. Click Windows key + R.
  2. Type devmgmt.msc, and hit Enter.
  3. Navigate to Network adapters, and find your network adapter.
  4. Right-click your network driver, and click Uninstall.

Alternatively, you can reset the TCP/IP protocol by following the steps below:

  1. Press Windows key + R.
  2. Type netsh int ip reset resetlog.txt, and hit Enter.
  3. Note: When you run this command, it'll rewrite your computer's registry keys used for TCP/IP settings. This achieves the same result as removing and reinstalling the protocol.
  4. Reboot your PC.

Let us know how it goes.

Consider this an absolute must - manually updating your drivers.

Vista and Windows 7 love updated drivers so here is how to update the major ones.

This is my generic how to for proper driver updates :

This utility makes it easy to see which versions are loaded :

DriverView - Free - utility displays the list of all device drivers currently loaded on your system.
For each driver in the list, additional useful information is displayed: load address of the driver,
description, version, product name, company that created the driver, and more.
http://www.nirsoft.net/utils/driverview.html

For Drivers check System Maker as fallbacks and Device Maker's which are the most current.
Control Panel - Device Manager - Display Adapter - write down the make and complete model
of your video adapter - double click - Driver's tab - write down the version info. Now click UPdate
Driver (this may not do anything as MS is far behind certifying drivers) - then Right Click -
Uninstall - REBOOT this will refresh the driver stack.

Repeat that for Network - Network Card (NIC), Wifi, Sound, Mouse and Keyboard if 3rd party
with their own software and drivers and any other major device drivers you have.

Now go to System Maker's site (Dell, HP, Toshiba as examples) (as rollback) and then Device
Maker's site (Realtek, Intel, Nvidia, ATI as examples) and get their latest versions. (Look for
BIOS, Chipset and software updates at System Maker's site while there.)

Download - SAVE - go to where you put them - Right Click - RUN AD ADMIN - REBOOT after
each installation.

Always check in Device Manager - Drivers tab to be sure the version you are installing actually
shows up. This is because some drivers rollback before the latest is installed (sound drivers
particularly do this) so install a driver - reboot - check to be sure it is installed and repeat as
needed.

Repeat at Device Makers - BTW at Device Makers DO NOT RUN THEIR SCANNER - check
manually by model.

Manually look at manufacturer's sites for drivers - and Device Maker's sites.
http://pcsupport.about.com/od/driverssupport/ht/driverdlmfgr.htm

andre da costa say:

I forgot to mention upgrade paths:

When making significant changes to your computer such as an upgrade, it is always recommended you backup your system prior to installing a new version of Windows. In your case, you can upgrade to Windows 7 depending on edition and architecture of Windows Vista you have installed. If you are running a 32 bit version of Vista and want to upgrade to a 64 bit version (edition of Windows 7), you will have to a do clean install, this applies vice-versa.

If you are running consumer editions of Windows Vista such as Home Basic or Premium, you can only upgrade to either Windows 7 Home Premium or Ultimate. Also, you cannot do downgrades, which means, if you are running Windows Vista Ultimate now, you cannot downgrade to Windows 7 Home Premium. You must do logical upgrades:

Windows Vista Home Basic > Windows 7 Home Basic, Premium or Ultimate
Windows Vista Home Premium > Windows 7 Home Premium or Ultimate
Windows Vista Business > Windows 7 Professional, Enterprise or Ultimate
Windows Vista Enterprise > Windows 7 Enterprise
Windows Vista Ultimate > Windows 7 Ultimate.
