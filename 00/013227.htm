ken sheridan say:

The query shows 3 master records (all with ID=1)  because of 3 child records?


Not really; it returns three instances of the values from the row in the referenced table.  This is how a relational database works.  If you join a referenced table to a referencing table, then in the result table one instance of the values in each row from the former will be returned for each matching row in the latter.

The real point here is that you should not be interfacing with data by means of a query's result table in datasheet view.  Either return the results in a report, which can be grouped on the key of the referenced table, enabling the values from each row of that table to be shown once only in a group header, while the values from the column sin the referenced table are shown in the detail section.  Or, in a form, return the values from the referenced table in a parent form, and the values from the referencing table in a subform within the parent form, linking the subform to the parent form on the relevant keys.
