ronrosenfeld say:

This is a little bit different approach, but it processes your large file in just a few seconds.

I use the FileSystemObject and read the entire file at once.

I then use the Replace method to get rid of all the CR and LF characters.  I do this separately as there seemed to be some "glitches" doing it as a pair, but you can certainly modify that depending on the results you want.

The results are then Split into an array; the array is transposed (but NOT by using the Transpose function as your data is over the limit for that); and then written onto a worksheet.

===========================================

'Using Tools/References, set Reference to
'  Microsoft Scripting Runtime

Sub ImportFile()
    Dim FSO As FileSystemObject
    Dim TS As TextStream
    Dim sFN, FN As File
    Dim V As Variant, vRES() As Variant
    Dim I As Long
    
sFN = Application.GetOpenFilename("Text Files (*.txt), *.txt")
If sFN <> False Then
    Set FSO = New FileSystemObject
    Set TS = FSO.OpenTextFile(sFN, ForReading, False)
    V = Split(Replace(Replace(TS.ReadAll, vbLf, ""), vbCr, ""), Chr(21))
    
    ReDim vRES(1 To UBound(V) + 1, 1 To 1)
    For I = 0 To UBound(V)
        vRES(I + 1, 1) = V(I)
    Next I

Cells.Clear
With Range("a1").Resize(UBound(vRES, 1))
    .Value = vRES
End With   
End If
End Sub

============================

bill manville say:

My code produced the same as JP Ronse's method.

I think we have both misunderstood what you wanted.

My guess is that you didn't want

FST~289~Z~Z~201501
19~~~~WO~PLANNED WORK ORDER

to be split across 2 rows.

In other words you want to lose all 0d0a sequences and then split into rows using Chr(21) as a delimiter

The following, simpler, code achieves that:

Sub test()
  Dim Row As Long
  Dim vPath
  Dim d1 As String
  Dim vParts
  Dim stLine As String
  Dim LRow As Long
  Dim Seg As Long
  vPath = Application.GetOpenFilename()
  Row = 1
  Open vPath For Input As #1
  Do   ' concatenate all lines, losing their 0d0a terminators
    Line Input #1, stLine
    d1 = d1 & stLine
  Loop Until EOF(1)
  Close #1
  vParts = Split(d1, Chr(21))
  LRow = UBound(vParts)
  For Seg = 0 To LRow
    Cells(Row, 1) = vParts(Seg)
    Row = Row + 1
  Next

End Sub

hans lu msft say:

Hi Liam,

 

Generally, the ost/pst files should be stored under drive:\Users\<username>\AppData\Local\Microsoft\Outlook

You may also search the ost/pst file under:

drive:\Documents and Settings\<username>\Local Settings\Application Data\Microsoft\Outlook

 

If the file of this ISP account still cannot be found, I suggest you configure the account in a new profile of Outlook check whether the email could be downloaded. If yes, which means the emails are stored on server. In this situation, you could go back to the old profile of Outlook to re-configure the ISP account.
