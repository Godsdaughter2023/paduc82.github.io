graham mayor say:

Adding protection for forms severely inhibits what you can do with a document. Page layout settings are amongst the things that cannot be accessed from the user interface in a protected document - even in unprotected sections.

For documents that require free editing, such protection is not the way forward. As you have Word 2007, you could use content controls which require the document to be unprotected, or you could create and employ a userform to populate the areas of the document currently the form fields are used for.

rick rothstein, mvp - excel say:

Can you make use of a macro?

Sub DuplicateData()
  Dim X As Long, LastRow As Long, DestRow As Long
  Const Source As String = "Sheet2"
  Const Destination As String = "Sheet3"
  Const StartRow As Long = 2
  DestRow = Worksheets(Destination).Cells(Rows.Count, "A").End(xlUp).Row + 1
  With Worksheets(Source)
    LastRow = .Cells(Rows.Count, "A").End(xlUp).Row
    For X = StartRow To LastRow
      Worksheets(Destination).Cells(DestRow, "A").Resize(.Cells(X, "B"), 7).Value = .Range("A" & X & ":G" & X).Value
      DestRow = DestRow + .Cells(X, "B").Value
    Next
  End With
End Sub

To install this macro, press ALT+F11 to go into the Visual Basic editor, click Insert/Module from its menu bar and then copy/paste the above code into the code window that opened up. Next, change the worksheet names being assigned  in the first two Const statements to the names of your actual sheets. Now, go back to your worksheet and press ALT+F8, select DuplicateData and click the Run button.
