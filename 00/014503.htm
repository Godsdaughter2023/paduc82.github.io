kerriblack say:

I am using Microsoft Office Profession Plus 2010  --  Two of us add to and update a large document on a daily basis.  Several others need to view only some of the information that we have entered.  Most of the information that we enter and update does not and should not been seen by the others.  My question is this… Is there way to have the document open with only certain columns showing for the others and still open up in normal view for us?

jllatham say:

Well, basically there isn't a way to make absolutely certain that others do not see those columns other than to physically remove them or their contents.  Do not despair, I'll show you how this could be done later.

But to continue ... ordinarily we would do this with some VBA (macro) coding to ask for a password to view those columns when opening the workbook and then unhide the columns when proper password(s) were provided.  But the simple action of not permitting macros to run would defeat that part of it and there are many free tools on the internet that would then make it possible for anyone to unprotect the sheet and make the hidden columns visible and see that information.

I said not to despair - I will put together a sample workbook shortly that will include code to take the one sheet and copy it to a separate workbook without the data in the columns you wish to be 'hidden'.  I will tell you how to put the code into your workbook for future use.

In the meantime, if you can tell me the name of the worksheet itself and list which columns need to be hidden on that sheet, it will help me create a macro that will work "right out of the box" for you.

jllatham say:

Here is the promised workbook - you can download it from this link:

http://www.mediafire.com/download/3d3hm9wyloxqiof/for_KerriBlack_ExportData.xlsm

And here is the functional code for it, this goes into a regular code module in your workbook and instructions for placing it in one of those can be found here:

http://www.contextures.com/xlvba01.html#videoreg

The code.  Note that you need to change the name of the source worksheet and will need to modify some lines of code in the body of it all.  These have been made bold to point them out.

Sub CreateDistributionCopy()
  'change this to the name of the worksheet
  'with the data to be exported, but with 'hidden' columns
  Const sourceSheetName = "TrueDataSheet"
'you'll need to edit code below where it removes the
'contents of some columns on the created sheet
'the section is marked with ********* above and below it
'
  Const exportSheetName = "ExportedData"
  Dim srcWS As Worksheet
  Dim destWS As Worksheet
  Dim currentWS As Worksheet
  Dim dataToCopy As Range
  Dim lastRow As Long
  Dim lastCol As Long
  Dim newWB As Workbook
  Dim newWBName As String
 
  'start by getting a name and place for the
  'exported data sheet's workbook
  'if user cancels, then we do not do anything else
  newWBName = Application.GetSaveAsFilename(exportSheetName & ".xlsx", _
   "Excel Files (*.xlsx),*.xlsx", , _
   "Choose File Location and Name")
  If newWBName = "False" Then
    'user cancelled the operation, just tell them and quit
    MsgBox "You cancelled the operation. Processing halted.", _
     vbOKOnly + vbExclamation, "File NOT Saved"
    Exit Sub
  End If
  'continue processing
  Set srcWS = ThisWorkbook.Worksheets(sourceSheetName)

  'get last row data is in on this sheet
  On Error Resume Next
  lastRow = srcWS.Cells.Find(What:="*", _
   after:=srcWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByRows, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Row
  On Error GoTo 0

  'get last column data is in on this sheet
  On Error Resume Next
  lastCol = srcWS.Cells.Find(What:="*", _
   after:=srcWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByColumns, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Column
  On Error GoTo 0
  'do we have a sheet to copy the data to?
  On Error Resume Next
  Set destWS = ThisWorkbook.Worksheets(exportSheetName)
  If Err <> 0 Then
    'no, doesn't exist, create it
    Err.Clear
    Set currentWS = ActiveSheet ' just to find our way back
    Application.ScreenUpdating = False
    ThisWorkbook.Worksheets.Add _
     after:=ThisWorkbook.Worksheets(ThisWorkbook.Worksheets.Count)
    ActiveSheet.Name = exportSheetName
    Set destWS = ActiveSheet
    currentWS.Activate
    Set currentWS = Nothing ' no longer needed
  End If
  'delete any old data/formatting on the export sheet
  destWS.Cells.Clear
  'set up to copy the data
  With srcWS
    Set dataToCopy = .Range(.Cells(1, 1), .Cells(lastRow, lastCol))
  End With
  dataToCopy.Copy
  destWS.Range("A1").PasteSpecial xlPasteValues
  destWS.Range("A1").PasteSpecial xlPasteFormats
  Application.CutCopyMode = False
  '***********************************
  'Here is where the data in 'hidden' columns is removed
  'add more similar lines as required, and edit the
  'three samples if they refer to incorrect columns
  With destWS
    .Columns("B:B").ClearContents
    'if you want to hide it then
    .Columns("B:B").ColumnWidth = 0
    'now for another column
    .Columns("F:F").ClearContents
    .Columns("F:F").ColumnWidth = 0
    'and a 3rd
    .Columns("G:G").ClearContents
    .Columns("G:G").ColumnWidth = 0
  End With
  'next the special sheet is copied to
  'a new workbook and it becomes Active
  destWS.Copy
  Set newWB = ActiveWorkbook
  'back to this workbook
  ThisWorkbook.Activate
  'save the 1-sheet export workbook
  newWB.SaveAs Filename:=newWBName, FileFormat:=xlOpenXMLWorkbook
  'close that workbook, announce job done and do cleanup
  newWB.Close False ' no need to save changes, it was just saved.
  Set destWS = Nothing
  Set dataToCopy = Nothing
  Set srcWS = Nothing
  MsgBox "Data exported as: " & vbCrLf _
   & newWBName & vbCrLf & "and has been closed.", _
   vbOKOnly + vbInformation, "Job Completed"
End Sub

ashish mathur say:

Hi,

I have done this with the help of Data > Get & Transform.  Here is the code which you have to paste in the Query Editor

let
    Source = Excel.CurrentWorkbook(){[Name="Data"]}[Content],
    #"Changed Type" = Table.TransformColumnTypes(Source,{{"item 1", type datetime}, {"item 2", type datetime}, {"item 3", type datetime}}),
    #"Added Index" = Table.AddIndexColumn(#"Changed Type", "Index", 1, 1),
    #"Reordered Columns" = Table.ReorderColumns(#"Added Index",{"Index", "item 1", "item 2", "item 3"}),
    #"Unpivoted Other Columns" = Table.UnpivotOtherColumns(#"Reordered Columns", {"Index"}, "Attribute", "Value"),
    #"Removed Columns" = Table.RemoveColumns(#"Unpivoted Other Columns",{"Index"}),
    #"Inserted Month Name" = Table.AddColumn(#"Removed Columns", "Month Name", each Date.MonthName([Value]), type text),
    #"Pivoted Column" = Table.Pivot(#"Inserted Month Name", List.Distinct(#"Inserted Month Name"[Attribute]), "Attribute", "Value", List.Count),
    #"Sorted Rows" = Table.Sort(#"Pivoted Column",{{"Month Name", Order.Ascending}})
in
    #"Sorted Rows"
