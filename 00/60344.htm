For previous versions of Windows 10, select the Start button, select Settings > Update & security > Window Update > Advanced options, then select the check box next to Use my sign in info to automatically finish setting up my device after an update.

Windows 10 Digital License
When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate. You will prompted to enter a product key a couple times through the installation, click I don't have a key and Do this later.

There is no need to know or get a new key, but if you have issues with Digital Licensing or the Activation Troubleshooter, you can utilize an existing Windows 7 or Windows 8/8.1 product key you already own to reactivate if necessary. or use the reset function in Windows 10.

a_user_here say:

Well from what I can see File History backup has a bit different purpose than Windows 7 Backup.

File History has no option for image backups, and it is only focusing on backing up user files.

But File History can be set to backup the files as fast as every 10 minutes.  With Windows 7 Backup, the shortest time is 1 day.

Personally I don't use either, I use CrashPlan (which would be closer to file history than Windows 7 Backup), and I have long ago given up on the idea of an image backup.

Over many years and many versions of Windows and different backup programs, I have found that when I needed to restore image backups they failed for me for one reason or another.  On top of that I'm an software expert and basically I can do a clean install and get pretty much everything close the way I like it with about a 1/2 day's work.  I would also say that for the most part the restoring of my data files isn't really connected to my restoring of my machine, since my data files reside on a server.  Of course if the server completely crash and some how wiped both a system disk and a data disk.  That would be reinstall of my system and then restore of my data files.
