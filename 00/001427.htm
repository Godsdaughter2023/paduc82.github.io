Disable blinking i-bar / flashing cursor - Microsoft Community

anglais.jacob say:

Disable blinking i-bar / flashing cursor

Hello, I'm working on an art piece which requires the screen recording of a text being written. Is there a way to disable the blinking i-bar/flashing cursor in Word for Mac?

Thank you very, very much for your help.

Sincerely,
Jacob
daniel g. grau say:

Actually there is a way. You can change the cursor blink rate, so that it will blink on and off much slower. If you manage to record the whole segment without pausing for longer than the blink rate, you're golden.


Changing the cursor blink rate involves adding an extra parameter to Word's preference file, com.microsoft.Word.plist.


1. Quit Word and make a backup copy of com.microsoft.Word.plist by Option-dragging it to the Desktop. You'll find it in your Library > Preferences. This is just a safety measure.


2. Open the Terminal application, in Applications > Utilities. You'll see an old-style terminal interface with a prompt like this:


     123-456-789-000:~ YourUserName$


3. Type the following Unix command (or copy from here and paste) and press Return:


defaults write ~/Library/Preferences/com.microsoft.Word NSTextInsertionPointBlinkPeriod -int 5000


The numerical parameter above is the number of milliseconds between cursor blinks. The normal value is "500" (i.e., 500 ms, for half a second ON, then half a second OFF). Thus, using 5000 ms will give you 5 seconds between blinks. This should be adequate, because the blinking cursor disappears in Word when you start typing, and when you stop, it starts counting the blink interval in the OFF mode. This means you can pause for up to 5 seconds until it blinks back ON. As long as you don't stop for more than 5 seconds, you'll never see it.


4. After you hit Return in Terminal, reopen Word and check the blink rate (don't quit Terminal until you are satisfied with the value). If you don't like what you see, re-issue the command with another number. You need to quit and reopen Word each time, so it will read its preferences file (com.microsoft.Word.plist).


5. To restore the original blink rate, use "500."


6. If you have any problem, quit Word and restore the original preferences file you copied to the Desktop.

jaredcheng_msft1034 say:

Hi Kramer,

As I tested on my side, it's not feasible to add your required columns or embed the Excel document in the Calendar app. We appreciate your understanding.

Besides, as a workaround, you could add a datasheet view in a list, then create and add the Customer and Suburb columns in it. Below are the steps:

In the Calendar view, click CALENDAR on the upper left corner> Create Columns > Create the Customer and Suburb columns > OK.
Then click CALENDAR > List Settings > click Create view on the bottom of the page > Datasheet View > navigate to Columns and only tick the Customer and Suburb columns > OK. In Datasheet View, you can create multiple columns and copy multiple Excel lines in it. Please refer to my answer in this thread: https://community.office365.com/en-us/f/154/t/446129

To let the list be viewed/edited by all members, you can go to List Settings > Permissions for this list > Team Site Members (which has Edit permission by default) > add the members in it if they are not in the list. Besides, you could also click Stop Inheriting Permissions and create a new group and grant permission for it. Please refer to this article: Manage permissions for a list, library, folder, document, or list item.

Regards,
Jared
