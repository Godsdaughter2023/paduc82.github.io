tanyasuarez say:

All of my word files (.doc) have changed to wordpad. How do I change them back?

After a computer cleanup, all of my .doc files changed to wordpad files.  Any .doc files sent to me via email are also converted to wordpad.  I need my .doc files back.  I tried converting one wordpad file, but conversion to .doc was not an option.  How do I make the program convert all of the wordpad files to .doc files?
jay freedman say:
All of your .doc files are still "Word" files -- there is no difference in the file contents, and no conversion is needed. The only thing that's happened is a change to the registry entries that tell Windows which program to use when you double-click a .doc file.

To fix it, click "Run..." on the Windows Start menu. Type or copy/paste this line
winword /r

and press Enter. Note that there is a space before the forward slash.


By the way, if you ran a "registry cleaner" program and this was the result -- don't do that any more! There is really no need to "clean" the registry, and there's always a danger of cleaning things that shouldn't be touched.

kristynsurp say:

Word Mail Merge Question: Conditional Carriage Return

I am using a mail merge to create "certification cards" for our staff. Sometimes during the mail merge, the data for each certification contains more than one word. (ie. CPR AND FIRST AID TRAINING). Many times they are near the end of a line, and so the record gets cut in half. Half of the text for the certification is on one line, the rest on the other line. Is there a way I can tell word to perform a carriage return ONLY if the text is going to get cut in half? I would like each "certification" to be fully contained on one line (with other records following, only using a carriage return if it will not fit all on one line).

Example:

For example, a certification card for employee A. Smith has 4 different courses completed:
COURSE #1, COURSE #2, CPR AND FIRST AID, OTHER TRAINING

On the certification card, after the mail merge i get this:
COURSE #1, COURSE #2, CPR AND
FIRST AID, OTHER TRAINING

But want THIS:
COURSE #1, COURSE #2,
CPR AND FIRST AID, OTHER TRAINING

I would like a carriage return in FRONT of CPR AND FIRST AID only if its going to split. Does anyone know how to go about this?

doug robbins - mvp office apps & services (word) say:

If you are using a Word table as the data source, you should be able to prevent the data from word-wrapping by using non-breaking spaces (CTRL+SHIFT+SpaceBar) between the words.

 

If the data source is in some other format, you could used a Directory type mail merge in the main document of which you have  a one row table into the cells of which you insert the merge fields and then when you execute that merge to a new document, that document will contain a table with a row of data for each record in the data source.  You can then add a row at the top of the table into which you insert the field names and by selecting the columns of data, you could then use the replace facility in Word to replace the spaces with non-breaking spaces.
