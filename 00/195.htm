Access 2007: how to create a running balance on filtered records - Microsoft Community

vassiliscondos say:

Access 2007: how to create a running balance on filtered records

I have a table with expense records.

One field is Actual Amount and another is Account.

(Actual Amount is a calculated field, calculated by another query but that's irrelevant.)

I also have a query which creates a running balance field that is displayed in a form:

SELECT [Transactions Extended].*, (select sum([Actual Amount]) from [Transactions Extended] X where
X.[Transaction Date]<=[Transactions Extended].[Transaction Date]) AS Balance
FROM [Transactions Extended];

Problem is that the running balance takes into account all the records of the table even when a filter is applied in the form...

More specifically, say a filter is applied in the form to show transactions from only one particular Account. What I want in this case is the running balance to be recalculated to take into account only the Account records (transactions), thus producing a running balance of the particular account. Is this possible?

I understand a few complications that may arise, e.g the following...

For instance what happens if a filter is applied for any other field, say category of expense? The answer is, running balance should not be recalculated in this case. In fact let's say running balance should only be recalculated based on filtered records only when the filter is on the Account field.

ken sheridan say:

1.  On a theoretical level I would still like to know how the query could be written in such a way as to be automatically restricted to just the filtered set of records no matter whether the filter makes sense or not.

2.  Next step, is from form A the user should be able to double-click (or click - of little concern for now) on one of the Accounts and then another form (B) should open with the following setup:

- records restricted to only a particular account - that selected by the user - and with a field showing running balance for this account.

1.  This is where the restriction with WHERE (ColumnName = <parameter> OR <parameter> IS NULL) is used.  If the value in the column equals the value of the parameter only those rows will be returned as the first part of the OR operation evaluates to TRUE, but the second part evaluates to FALSE for each row.  If the parameter is left NULL, however, all rows will be returned regardless of the value in the column in question, as the second part of the OR operation evaluates to TRUE for all rows.  This needs to be applied both to the query as aWHJERE clause, and, as you are calling the DSum function, as a criterion for the DSum function call, though in the latter case it would be a VBA expression.  Where a JOIN is used the WHERE clause would restrict both instances of the table in the same way .

2.  You would build a sting expression as the WhereCondition argument of the OpenForm method, which would filter Form B to the selected account, e.g.  if Form B's recordset includes a column name AccountID of a numeric data type, and this also the name of a column in Form A's recordset:

Dim strCriteria As String

strCriteria = "AccountID = " & Me.AccountID

DoCmd.OpenForm "FormB", WhereCondition:=strCriteria

russ valentine say: I understand. This is one of the new "features" that have been implemented by default in Outlook 2010 even though most users probably don't want it. Each time you create a new email account in Outlook, you are given the chance to designate to which data file that account should deliver messages. Just select the data file your other account(s) are using. For accounts you have already created, just go to File > Account Settings > Email Tab > and select the account for which you want to change the delivery location. The click on "Change Folder" and direct it to the file you want to use.

I had " Print Spooler service not started even set on automatic" and found solution that may help some of you to fix your case.

 

I don't know some how WTP 10 turn off LDP port after installing new printer driver and leaves only Internet client port only. That's why the printer spooler service failed to start. Solution could be:

 

1. Go to Programs and Features in Control panel > click Turn Windows features on or off

2. click + sign of Print and Documents services

3. Select LPD and LPR. You should remove Internet Printing Client.

> restart your PC to ensure there is no error of spooler service.

 

4. Add your printer driver again.
