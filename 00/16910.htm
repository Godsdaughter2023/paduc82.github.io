graham mandeno say:

Thanks for the explanation.

 

First some advice about names and primary keys :-)

 

When naming objects (tables, queries, fields, forms, etc) I recommend you avoid any spaces and non-alphanumeric characters (except for "_").  Therefore I suggest you rename your Order# field to OrderNumber, or something similar.  As you gain more experience, you will find it is a great hassle if you have object names that are not valid variable names in your code.  You should also avoid names that are common meaningful words in VBA and SQL code, such as "Name", "Top", "Group", "Date", etc.

 

Also, avoid meaningless names such as "ID" and "Lookup".  Instead, use a descriptive name such as "OrderID".  My own personal preference for field named is to assign a unique lowercase prefix of 2-4 characters to each table, and start every field name with that prefix.  For example, you could have ordID, ordNumber, ordDate, ordCustomer, etc in your Order table, and orlID, orlOrder, orlProduct, ordQuantity, etc in your OrderLine table.  This has two advantages - you can always tell from a field name which table the field comes from, and you avoid ambiguity in queries containing two tables with the same field name.

 

You appear to have two candidates for a primary key in your Order table - ordID and ordNumber.  If ordNumber is numeric or short text, is unique, never changes, and is always present, then it would be a good candidate for a "natural" primary key.  Otherwise use ordID (the autonumber) as a "surrogate" primary key.  If you have a natural primary key then you should delete the autonumber field - there is no purpose (that I can think of!) for an autonumber field EXCEPT as a primary key. 

 

Now to your forms. It appears that you have a form frmContractLog (note that I have renamed it :-) with which you want to select an order for viewing and possibly editing. I'm still a bit hazy about where the subform comes in... you cannot have a subform on a continuous form, so I assume frmContractLog is the subform - but then what is the parent form?

 

Here is one quite elegant solution which will involve very little coding:


- Create a large "unbound" main form (no RecordSource) named "frmOrderSelector"


- On the left-hand side of this form, place frmContractLog as a subform, with no link fields.  Rename the container control "sbfContractLog". 


- On the right-hand side, place frmOrderHeader as another subform, and rename its container "sbfOrderHeader". 


- Somewhere else on the main form, add a textbox named "txtCurrentOrder".  Set its ControlSource to:

=sbfContractLog.Form.ordID (or .ordNumber if you use a natural key)


- Finally, return to sbfOrderHeader and set the link fields:

Set LinkMasterFields to txtCurrentOrder (the name of the textbox)

Set LinkChildFields to ordID (or ordNumber if you use a natural key)

 

Now, if everything is correct, you will see a list of orders on the left-hand side and when you select one, the order will show on the right-hand side.  Note that txtCurrentOrder can be made invisible, since its only purpose is to provide a link between the two subforms.

 

You can further enhance this by adding controls to the main form which allow you to select customer, salesperson, date range, etc, and use these to filter your left-hand subform and narrow down the selection list.

 

Good luck!  :-)

Graham

Internet-connected PCs require continuous real-time AV/Antimalware protection in order to keep you safe from online threats, and that’s why Windows Defender is installed and enabled by default in Windows 10. If you’re not happy with Windows Defender for any reason whatsoever, then all you have to do is install another free AV app – and Windows Defender will automatically be disabled by the OS as soon as the new AV app registers itself with the system as an antivirus provider. Then if you uninstall the third-party AV app, Windows Defender will automatically be enabled so that no lapse in your AV/Antimalware protection ever occurs.

 

If you’re not happy with Defender for any reason at all, then follow the steps in this thread to replace it with another free AV app. Note that the answer in this thread specifically addresses the issue of excessive CPU utilization by the Windows Defender Antimalware Engine:

 

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-to-stop-antimalware-service-executable-from/a3a57d31-4687-43c0-b274-261da7d89245

 

Once you’ve installed another AV app, you’ll have the option to turn on Limited Periodic Scanning – and that will allow Windows Defender to run right alongside your new AV app with its real-time protection turned off – so that it doesn’t interfere with the new app. Limited Periodic Scanning allows you to run any kind of on-demand scan, including Windows Defender Offline and Command Line Interface scans. However, this mode won't allow you to turn on real-time protection or Controlled Folder Access. 

 

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-antivirus/limited-periodic-scanning-windows-defender-antivirus

 

Here’s another great compatibility feature that’s built into Windows 10:

Automatic disabled mode is enabled so that if the protection offered by a third-party antivirus product goes out of date, is not updated, or stops providing real-time protection from viruses, malware, and other threats, Windows Defender AV will automatically enable itself to ensure antivirus protection is maintained on the endpoint.

 

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-antivirus/windows-defender-antivirus-compatibility

 

When this feature is working correctly, Windows Defender will automatically be activated as soon as you disable the real-time protection of your third-party AV app; and then it will be automatically disabled again as soon as you turn the third-party app’s real-time protection back on. So the lapse in protection will be on the order of seconds.

 

For more information, see this thread:

 

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/how-do-i-turn-windows-defender-off/944322a3-9c1e-470b-875a-844677f0af96
