In Word, you can format numbers in a field as Roman.

For example:

  1. Press Ctrl+F9 to insert field brackets { }
  2. Type   =2014 \* ROMAN
  3. Press F9 to hide the field code and update the result.

Or use a macro, as described in http://msgroups.net/microsoft.public.word.newusers/how-do-i-insert-roman-numerals/70297

Excel has a worksheet function ROMAN.

For example:

  1. Enter a number in a cell, say in A1.
  2. In another cell, enter the formula    =ROMAN(A1)

suzanne s. barnhill say:
As a general rule, roman numerals in Word are entered just by typing the component parts: I, V, X, L, C, D, M, etc. (caps or lowercase). But some fonts do have a range of "built-up" roman numerals. Find these at U2160-U2182 in the Number Forms character subset. Even if the font you are currently using does not contain these characters, you can type one of the Unicode numbers and press Alt+X. The character will be inserted in MS Mincho (serif) or MS Gothic (sans serif). If you have Arial Unicode MS installed, you can select it in Word's Symbol dialog and select those characters manually.

jllatham say:

Yes I think you did.  Let's see if I can rewrite it to get it to work for you.

Try this as a new routine in a copy of your workbook.  It does compile without any indicated errors. Both workbooks (.xlsx files) have to be open when you run the macro.

Sub Transpose()
 Dim v As Variant
 Dim N As Long
 Dim X As Long
 Dim rw As Long
 Dim InspectionWB As Workbook
 Dim InspectionWS As Worksheet
 Dim SummaryWB As Workbook
 Dim SummaryWS As Worksheet
 
 Set InspectionWB = Workbooks("Inspection Form Template.xlsx")
 Set InspectionWS = InspectionWB.Worksheets("Form")
 Set SummaryWB = Workbooks("Copy of Summary Template.xlsx")
 Set SummaryWS = SummaryWB.Worksheets("Record")
 N = 11
 'get last row in SummaryWS based on column K entries
 rw = SummaryWS.Cells(Rows.Count, "K").End(xlUp).Row + 1
 v = Array("A8:A15", "C8:C15", "E8:E15", "G8")
 Application.ScreenUpdating = False
 For X = 0 To UBound(v)
  'copy a range from the "Form" worksheet
  InspectionWS.Range(v(X)).Copy
  'paste it into the "Record" worksheet
  SummaryWS.Cells(rw, N + 8 * X).PasteSpecial _
   Paste:=xlPasteValues, Transpose:=True
 Next
 Application.CutCopyMode = False
 Application.ScreenUpdating = True
 'this is just cleanup to prevent potential "memory leaks"
 Set SummaryWS = Nothing
 Set SummaryWB = Nothing
 Set InspectionWS = Nothing
 Set InspectionWB = Nothing
End Sub

 

jllatham say:

I'm glad we got it working.  Any new language is just that to each individual: something new.  And none of us start out knowing everything about something new to us.  Kudos to you for wanting to dig in and learn a bit more.

I actually tossed in a couple of things I could have done without, specifically the As Workbook objects, but it makes for shorter typing later on if you have to refer to them or worksheets in them often in the code later.

Technically it is not required to:

1) include the Application.ScreenUpdating = True statement.  Excel will automatically set it back to true when it exits the Sub.

2) including all of those Set .... = Nothing statements I tossed in.  But I like to take a belt-and-suspenders approach on some things, and that's one of them.  If Excel doesn't release those resources back to the Operating System, then repeated use of that routine could, in theory, result in a smaller and smaller pool of available system resources: otherwise known as a "memory leak".

doug robbins - mvp office apps & services (word) say:

The following macro will allow you to navigate to the folder containing the files and then open each one in turn and activate the track changes feature

 

Dim myFile As String
Dim PathToUse As String
Dim myDoc As Document
Dim strTitle As String
'Select the folder that contains the files
With Dialogs(wdDialogCopyFile)
    If .Display <> 0 Then
        PathToUse = .Directory
    Else
        MsgBox "Cancelled by User"
        Exit Sub
    End If
End With
MsgBox PathToUse
'Close all open documents before beginning
Documents.Close SaveChanges:=wdPromptToSaveChanges
myFile = Dir(PathToUse & "*.doc")
While myFile <> ""
    'Open document
    Set myDoc = Documents.Open(PathToUse & myFile)
    myDoc.TrackRevisions = True
    'Close the modified document after saving changes
    myDoc.Close SaveChanges:=wdSaveChanges
    'Next file in folder
    myFile = Dir()
Wend

Please refer to Use Word Layout Options to move pictures, Word Online cannot move a picture that was inserted while using the desktop version of Word if the picture has text wrapping or a fixed position on the page. greginmich say:

The first observation that we need to make here is that there are as many different answers to this question as there are free antivirus apps, and that by the accepted laws of logic, they can’t all be the best. So most people naturally assume that all they have to do is just decide who to believe based on the experience, level of technical expertise, and impartiality of the person making the recommendation – and that’s simply not true. It really doesn’t matter who says what on this topic – because anybody who’s offering you advice based on their own personal experience just doesn’t have enough information to make a valid comparison. That’s because personal experiences with malware tend to be very limited for careful users, and because personal experiences with AV apps are always sequential in nature and consequentially only allow for comparisons between old and new versions of AV apps.

So in order to be relevant, comparative evaluations of AV apps need to be based on the same scientific methodology that applies generally in the field of product testing and evaluation, and that means that they have to acquire non-sequential data by simultaneously exposing all of the current versions of the AV apps to a large set of malware samples that’s highly representative of the current threat landscape – and that this has to be done in a carefully controlled environment. So if you’re not using this industry standard evaluation methodology on a regular basis, then you’re simply not qualified to answer this question:

… testing labs follow the best industry practices, so users have some valuable sources for security software tests and reviews without having to rely on “recommendations” of sales staff, marketing departments or opinions of unknown “advisors” on forums.

https://www.av-comparatives.org/list-of-av-testing-labs/

https://www.amtso.org/documents/

And on the other hand, we have – which sure sounds like an evasion of the question rather than an answer for it. Now logic dictates that AV apps can’t all be the best, but it certainly doesn’t rule out the possibility that there’s a best AV app – and we really can’t throw out the question based on the fact that different criteria can be applied as the basis for the evaluation, so as far as I’m concerned, this is a non sequitur.

Of course the way out of this self-imposed impasse is simply to evaluate AV apps by ranking them in terms of their performance on specific criteria, which is exactly what all of the independent AV testing labs actually do. But it’s also important to define the essential criterion for AV apps, which is their ability to prevent a PC from being compromised, and this is the starting point of the AV-Comparatives Real-World Protection test. This test operationalizes this fundamental definition with a testing procedure, and that procedure provides a set of protection scores that can be used to rank AV apps – which in turn provides us with an evidence-based answer to our question:

http://chart.av-comparatives.org/chart1.php?chart=chart2&year=2017&month=11&sort=0&zoom=2

The downside is that in order to answer the question honestly, we have to eliminate all of the extraneous variables that might prejudice the result in the testing environment – and once you install an AV app on a machine that’s been in use for a while, all of those extraneous variables will come right back into play. And that means that an app that performs well on a pristine test machine might not do nearly as well on your PC. That’s why it’s a good idea to do a preinstall cleanup of any previously installed AV apps before you install a new one.

So this is not a perfect answer, to be sure, but certainly not something that should be totally neglected. And now I have to correct my interpretation of Defender's status in this test, because I previously speculated that the “user dependent” segment of Defender’s protection score might reflect the user configuration of the real-time protection options – and that’s apparently not the case according to the information provided here:

https://www.av-comparatives.org/wp-content/uploads/2017/12/avc_prot_2017b_en.pdf

Our Whole-Product Dynamic Protection Test aims to simulate real-world conditions as experienced every day by users. If user interactions are shown, we choose “Allow” or equivalent. If the product protects the system anyway, we count the malware as blocked, even though we allow the program to run when the user is asked to make a decision. If the system is compromised, we count it as user-dependent.

Now this is really a bit confusing for me, since I’ve never been presented with the option to allow any threat in all the years that I’ve used Defender – and in fact the option to allow a threat is normally so well concealed that most users don’t even know that it’s there, as I’ve just shown in this thread:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/once-windows-defender-locates-malware-what-does-it/0c7b5592-7121-46e4-8759-b8e0547cf214

I think the possible exception to this might be for low-level threats, but I’ve never seen one of those. In any case, if this is the correct meaning of “user dependent”, then this is actually very good news – because then all we’ll need to do in order to achieve the higher level of protection is to stop allowing threats. And it also means that adding on all of the new optional real-time protection features should put Defender right up there with the best of them:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start-windows_10/if-i-buy-a-new-computer-running-windows-10-do-i/49a25783-647e-48ec-9e06-dbc0e56b8798
