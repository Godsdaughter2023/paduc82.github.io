alika_ say:

ken sheridan say: A combo box which lists multiple columns allows a value from the first column to be selected.  You'll find an example in ComboDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo principally illustrates various means of inserting data into a hierarchical structure in the context of a set of normalized tables, by means of correlated combo boxes.  It does include an alternative, however, which is the use of a multi-column bound combo box and unbound boxes.  In this a parish name can be typed and it will go to the first match in the list, but this doesn't really serve any purpose in this case as the list is sorted hierarchically by County, then District, then Parish.  So a user could enter G and get Gawsworth in Cheshire, but they really want Gnosall in Staffordshire, which is way down the list.  It would only work if the list were ordered first by Parish, in which case they'd then just need to scroll down through the 'G' parishes which would be listed together.  But this would undermine the whole basis of the list, which is that it is hierarchically ordered.

So in your case the autoexpand mechanism would only be of use if the list is ordered by the first visible column.  If not it serves no useful purpose.

As regards your 'other' option, I see no reason in principle why this shouldn't work.  The control's AfterUpdate event procedure would simply need to make visible and move focus to a text box bound to a column in which the other data entered would be stored.  How you incorporate the 'other' option in the list is another matter.  One way would simply to be to have it as a 'dummy' value in the referenced table.  My ComboDemo file for instance includes 'dummy' parishes named N/A for the Districts of Tamworth and Stoke-on-Trent, neither of which have civil parishes.  An alternative is to return an 'other' row in the list without having a value in the referenced table by means of a UNION operation in the combo box's RowSource property.  The FindRecord demo in my same OneDrive folder does this to include an option for entering a new record in a combo box to find a contact. The SQL statement for its RowSource property is:

SELECT Contacts.ContactID,
FirstName & " " & LastName, 1 As SortColumn,
LastName,Firstname  
FROM Contacts  
UNION
SELECT 0, "<New Contact>", 0,"",""  
FROM Contacts
ORDER BY SortColumn, LastName, FirstName;

As this is an unbound combo box it will happily accept the zero value, but with a bound combo box, where there is no row in the referenced table with a key value of zero, it would be necessary to also set the value of the control to NULL in its AfterUpdate event procedure to avoid violating referential integrity.
