Hello Joe,

I would suggest you to completely remove the entries of the Brother HL2170W printer from the Windows registry.

 

In order to do so, refer the steps given bellow:

 

Step 1: Uninstall the Brother HL2170W  printer software.

 

Refer the link: Uninstall or change a program

http://windows.microsoft.com/en-us/windows/uninstall-change-program#uninstall-change-program=windows-7

 

Step 2:

 

a. Click on Start.

b. Type services.msc in the start search box and press Enter.

c. Select Print Spooler service.

d. Right click on Print Spooler service and select properties.

e. Click on the Stop button.

 

Step 3:

 

1. Go to “Program & Features” & uninstall any drivers related to the printer.

2. Create a restore point.

3. Browse  to C:\WINDOWS\SYSTEM32\SPOOL\DRIVERS\w32x86 & delete its contents

4. Browse  to C:\WINDOWS\SYSTEM32\SPOOL\PRINTERS & delete its contents.

 

Step 4: Next, edit the registry

 

Important this section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs. For more information about how to back up and restore the registry, click the following article number to view the article in the Microsoft Knowledge Base:

How to back up and restore the registry in Windows

http://support.microsoft.com/kb/322756/en-us

 

1. Click on Start.

2. Type regedt.exe in the start search box and press Enter.

3. Registry editor windows will open.

4. Navigate to HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Print\Environments\Windows NT x86

5. View the list of subkeys (subfolders). There should only be the following subkeys:

 

1. Drivers

2. Print Processors

 

If, any other subkey is present then take registry backup & delete it.

 

6.  Navigate to HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Print\Environments\Windows NT x86\Drivers & then look for Version 2 or Version 3, subkey. Delete the entries except Microsoft Shared Fax Driver & Microsoft XPS Document Viewer * under the –Version-x. (after taking registry backup).

 

7. Navigate to HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Print\ \Monitors

 

View the list of subkeys. There should only be the following subkeys for the default print monitors:

  

1. Local Port

2. Microsoft Shared Fax Monitor

3. Standard TCP/IP Port

4. USB Monitor

5. WSD Port

 

If, any other subkey is present then take registry backup & delete it.

 

8.  Exit Registry Editor.

 

If the printer is still connected then unplug it & restart the computer.

 

9. Restart the the Print Spooler service. Status should be Started & Automatic.

 

10. Try to install the printer drivers again from the manufacturer’s website.

Question regarding Event Viewer - Microsoft Community

monica naranjos say:

Question regarding Event Viewer

As you know even viewer likes to report errors, however I noticed two kinds of errors: the errors that are marked with an exclamation points, and "Windows error reporting"

A friend told me the later ones are errors that Windows solved(Windows error reporting), and the other ones are errors Windows couldn't solve (Errors with red symbol and the exclamation mark)

So is my friend right?

a. user say:

Far be it from me to stick my nose in between you and your friend. Instead, I'll explain about Event Viewer.

Event Viewer is a gigantic log of what goes on in your computer. Some of what it records are errors at the time, but that doesn't mean they're problems or you have to solve them.

I'll give an example: You flush your toilet and immediately receive a message saying "Error - tank has insufficient water." Well yes, of course, what did you expect? And it is a problem at that exact moment, but you know that error message is nothing to worry about.

Event Viewer is the same way. It records factual information at the moment, but that doesn't mean you should immediately act on it. If you read through the errors and warnings, and invest some time to figure out what they mean, you'll discover that yes, they are errors, technically, but they don't really mean anything.

For this reason I stopped reading Event Viewer long ago. Which isn't to say that Event Viewer has no value. On the contrary, if your computer has a problem, and the problem occurs as a discrete event, like a blue screen, Event Viewer can often provide you with important pieces of information that can help you diagnose what happened (if you pick the correct event to analyze.)

Event Viewer and other error reporting tools were not really meant for mortals like us. They were written primarily for software developers and debuggers that use the information to see technical details about the applications execution.

The error points to the ntdll.dll a component of Windows:

Faulting application name: Explorer.EXE, version: 10.0.15063.447, time stamp: 0x951324bb
Faulting module name: ntdll.dll, version: 10.0.15063.447, time stamp: 0xa329d3a8

However another, possibly third party, module may actually be causing the crash.

I might suggest to obtain a user mode dump which has the potential to pinpoint the cause.

You can configure Windows to create user-mode dumps. Create a System Restore Point first.

If you copy and paste the following in Notepad and save as a .reg file (save with the .reg extension and give it any name such as dump.reg), then right-click the .reg file and select "Merge" to add to the registry, a .dmp file at the time explorer.exe crashes should be created in the C:\CrashDumps folder:

00,6d,00,70,00,73,00,00,00


After Explorer (explorer.exe) crashes, can you then make the .dmp file available (provide link) via a public folder on OneDrive (SkyDrive) or similar site?

Here's a couple of links on using OneDrive (SkyDrive):

http://windows.microsoft.com/en-ca/skydrive/add-photos-files-faq

http://windows.microsoft.com/en-ca/onedrive/share-file-folder

You can delete the C:\CrashDumps folder anytime.
