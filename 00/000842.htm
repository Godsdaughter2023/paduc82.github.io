john w. vinson say:

Step back a bit. You should clearly have an inventory table of the ingredients on hand, but you would emphatically NOT have a separate table for each recipe! Recipe databases involve three tables: a table of Ingredients (perhaps your inventory table, though you would need to have a record for spring onion with a 0 amount rather than deleting the ingredient), a table for Dishes (the name of the dish, "Curried Chicken" or "Ginger Beef" or "Monk's Delight Tofu" for example), and a third table for Recipes, with fields for the DishID, the IngredientID, and the Quantity. The Recipes table would have thousands of records eventually, for all sorts of dishes.

You will really want to read up about "Relational Database Design", "Normalization", and so on, and learn the recipes and techniques to assemble a tasty Database. It's a lot of work (much more work than learning how to create a spreadsheet or Word doc) but can be very rewarding!

That error means it is already installed.
