You may be able to boot into a command prompt window without the Windows Disc:

1. Turn on your computer and start pressing the F8 key continuously. If Windows starts to load and you're still pressing F8, try again.

2. If you've done it correctly, you'll see a screen that should look like this:

3. Select Repair Your Computer from the options.

4. Once Windows finishes loading files, it will ask you for the keyboard layout and then it will ask you to logon to a local user account. Make sure it's an Administrator's account or this won't work.

5. After typing your password, you should see this:

6. Select Command Prompt

7. At the command prompt window, type the following command and press Enter:

del %windir%\system32\fntcache.dat

8. After you delete the file, type exit in the command prompt to close it, and then restart your computer. It should boot up normally.

But we're not done yet, because the file that we just deleted will recreate itself on reboot, causing a loop if it's not fixed. Here is how to fix that problem:

1. When the computer boots up, click Start, type regedit and hit Enter

2. In the Registry Editor, locate the following subkey:

HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\Fonts\

3. Right-click the Fonts subkey and click Export. Save the export to your desktop or somewhere similar and give it a name. 

4. Go back to the registry and, making sure that the Fonts subkey is highlighted, look the right side and see if you find an entry that:

  • Is a full file path
  • The full file path ends in the .otf extension

5. Delete any files that meet the criteria above (there may be only 1 such file, make certain it meets the criteria.)

6. When you've deleted it, close the registry editor and open an elevated command prompt window (Click Start, type cmd, right-click the result, and select Run as Administrator)

7. Type the following command to delete the FNTCACHE.DAT file (remember, it recreates itself when you restart your computer)

del %windir%\system32\fntcache.dat

8. Close the command prompt window. Click Start, type installed updates, and click the View Installed Updates from the search results.

9. In the new window that opens, utilize the search box in the top right of the window to find the following updates:

  • KB2982791
  • KB2970228
  • KB2975719
  • KB2975331

10. Right-click each of them and click Uninstall Update. These are the offending updates that cause this problem. If you're prompted to restart after you uninstall one of them, click Restart Later.

11. Restart your computer when you've uninstalled all of them.

12. After the computer restarts, find the exported .reg file you saved earlier, and double-click it to merge it to the registry.

See the following article:

Microsoft Pulls KB2982791 After 4 Days of Blue Screens

Yes there really was an update like this and it did cause this much pain. Microsoft did re-release the update later to fix this problem, so if you see KB2982791 in the Windows Update queue, don't install it

You can follow these steps or you can be stuck with a non-operational computer.
