This is a known issue reported and tracked by Feedback Hub in a Collection.  What is happening is that the Recovery partition becomes a replicated OEM volume (2 volumes representing one partition).  Run Disk Management and you can remove the drive letter that has been assigned.  This volume contains critical system files (Windows Recovery Environment) so it should not be deleted.  Just remove the drive letter so it does not appear in File Explorer.

Feedback Hub link if you want to upvote or add any details.  There are several other reports that are not in this collection also.

don cuthbert say:

Sorry, you are correct.  I guess you need to use Diskpart from an Administrator Command Prompt to remove the drive letter.

diskpart

select volume F
remove
exit

That should be all it takes.  The Recovery partition is meant to be hidden to protect it from being accessed in File Explorer and Disk Management.  This bug has happened before, over the years, from time to time, with various versions of Windows.  Eventually they change something to make the Recovery partition look normal again.  If you look in the top list in Disk Management, you will see the same volume appears to be replicated.  In diskpart, if you list volume, you may see the same drive letter twice.  It is really one partition with one volume.

assign sms sounds

 

Hello,

melsam [msft] say:

Hello,

If I remember correctly, with WM5, you can only use WAV files for notifications like text messages and reminders. So you'll need to convert your MP3 to WAV first. Here are some quick steps.

  1. Get the mp3 file you want to use. You'll probably want to chop it up to a couple seconds. I recommend using a program like Audacity to do it. Audacity will also let you export the portion you like into a WAV file (look under File > Export as WAV in Audacity's menu).
  2. Once you have your WAV file ready, connect your device to the PC.
  3. Browse to the Windows folder (My Device > Windows). Make sure you're under Windows and not My Documents. In your earlier message it sounded like you were copying the file to "My Documents".
  4. After copying the file successfully, go into Start > Settings > Sounds & Notifications > Notifications. Select the appropriate item in the dropdown list and the WAV file you created earlier should show up in the sounds list.

Let me know if this works for you. These instructions apply only to WM5 Pocket PC devices such as the Cingular 8125.

 

Some of this is easier in newer WM6 devices, but there's no reason you can't get a nice little WAV file going with a little help from Audacity. Good luck!

kiwiant say:

Outlook hangs syncing the address book when using a Microsoft keyboard

When I first figured this out I thought I was insane but testing proves it.

Outlook will hang when doing a complete send/receive of the Global Address List with the message "Copying offline address book template file". Steps to reproduce:

1. From the Send/Receive ribbon click Send/Receive Groups then click Download Address Book

2. Untick the checkbox Download changes since last Send/Receive leaving the Address book set to \Default Global Address List.

3. Click OK (the send/receive will hang with the message "Copying offline address book template file".

4. Unplug the keyboard/mouse dongle

5. Repeat the steps above and the send/receive doesn't hang

I have managed to reproduce this issue in different environments and different keywords. Recently I have been able to confirm with the Microsoft Sculpt Ergonomic Desktop set (and Mouse & Keyboard Centre) with an HP EliteBook 9470m w/ port replicator running with:

  • Windows 8.1 Enterprise x64 and Office 2013 x64
  • Windows 10 Enterprise x64 and Office 2016 x64
