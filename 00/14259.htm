Sub TestMacro2()
    Dim rngF As Range
    Dim strFA As String
    strFA = ""
    
    Set rngF = Cells.Find(What:="LAB001", After:=ActiveCell, LookIn:=xlFormulas, LookAt _
    :=xlPart, SearchOrder:=xlByRows, SearchDirection:=xlNext, MatchCase:= _
    False, SearchFormat:=False)
    If Not rngF Is Nothing Then
        While rngF.Address <> strFA
            If strFA = "" Then strFA = rngF.Address
            rngF.Offset(1, 0).Resize(2).EntireRow.Insert
            Set rngF = Cells.FindNext(rngF)
        Wend
    End If
      
End Sub

EDIT:  I can't point to the exact thing that fixed this problem, but I just did two big copies across the network to my NAS (one 186 GB and the other 46 GB) without problem.  Possible things that fixed it:

  • I powered down my NAS, turned off its power supply, pulled the plug on it, let it sit for about 10 minutes, and then powered it back up.
  • I installed the beta version of Intel's MEI software from my motherboard's site
  • Maybe some update came down from Microsoft that fixed something.

ORIGINAL POST:  There's something very wrong with wired Ethernet transmission speeds with Windows 10 RTM.  After doing a clean install of it, I left the system provided Ethernet drivers in place.  I've got a 1 Gbps connection between my computer to my router to my NAS.  Copying a large file from my computer to my NAS ran at 11 Mbps.  Having seen that problem before with one of the pre-release builds, I "solved" the problem by installing Intel's drivers from their site.  The same copy then popped up to my normal 112 Mbps.  However, about half way through the copy (at the 10 minute point, or so), the transfer rate again dropped to 11 Mbps for about 5 minutes.  Then, it returned to 112 Mbps.
