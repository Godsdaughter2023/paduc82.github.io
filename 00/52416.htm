Restore Point with External Drives - Microsoft Community

laseryt say:

Restore Point with External Drives

Hello. I have a question about restore points with external drives. If I create a restore point into an external drive. And if I reset my Windows PC, will the restore point not be deleted from my external drive? Please leave your answers down below. I will appreciate them.
pcr just pcr say:

Restore points are unable to undo a reset which is a lot like a clean install. Every thing is wiped, except you were given a chance to keep user files.

Before a reset (or anything really big), you should make a Windows system image backup. That can undo anything. It makes a backup of all the Windows partitions required to boot & run Windows: EFI, Recovery, & Windows (C:). If you've moved user folders (Documents, Downloads, etc.) to a different partition, that partition is also offered for backup.

Make it at "START, Settings, Update & Security, Backup, Go to backup & Restore (Win 7)", & make the Repair Disc or ask Cortana for Recovery Drive to put it on USB. Then see...

https://www.winhelp.us/restore-a-system-image-in-windows-7.html

Restore a System Image Backup in Windows 7, 8, 8.1 and 10

Note: That is an informative 3rd-party site, but I know nothing (good or bad) about any downloads or links that are there (or anywhere else).

If your external hard drive (or the internal one I guess) is a Seagate or Maxtor, there will be a free version of Acronis True Image at their site. Seagate calls it DiscWizard. Macrium Reflect, AOMEI Backupper & others also offer free versions of their image makers. These 3rd-party apps are quicker, make more compact images, & are more flexible in use. But the Windows version works in under 15 minutes on my small & uncomplicated system. I must add, DiscWizard's Bootable Media (to restore an image when Windows won't boot) requires a Mouse to operate. Plug one in, & it is instantly recognized.

pcr just pcr say:

You are welcome. Yes, a system image restore wipes out everything that happened to Windows after the image was made. If there was no virus before the image was made, there will be none after the image is restored. It also will undo a reset.

Windows is more than just C:. There are hidden partitions too that are restored. But partitions (if any) that have nothing to with running Windows will not be affected - unless you've moved them in the way. If you did repartitioning subsequent to making the image, anything that was moved to where the Windows partitions originally were located will be wiped out. Windows system image restore seeks to put everything back precisely where it was. (The 3rd-party imagers are more flexible & can put partitions where you want them - but you go figure how to do that.)

A Windows system image has a few surprises...

(1) The Windows partition (C:) that is in the image can be mounted & individual files copied off it. I did that once to recover just my Downloads folder which I had inadvertently wiped. To do so, double-click the largest .vhdx in the backup folder in File Explorer...

That will be the image of the C: partition. It will get a letter (say, E:) & open in a 2nd instance of File Explorer. You may put it side by side with the 1st instance. Then browse to your files & copy them from E: to C:. (But I wouldn't try the reverse.) When done, right-click E:, & select "Eject". [But I must add, the first time you double-click a .vhdx, it may give an error message. So, just OK the error, & open Disk Management. The mounted image will have the same name C: has, but no letter. Put a letter on it, & it will show up in File Explorer.]

(2) You actually don't even have to mount the image to recover files off it. And the external HDD that holds the image doesn't even need to be plugged in. A shadow copy is made to some secret area on the internal HDD each time an image is made. They can be accessed in File Explorer. Just right-click C:, & select "Restore previous versions". Then select the image you want by date, & click "Open". A 2nd instance of File Explorer (the 1st instance is behind it) opens with C: the way it was when the image was made. You may now copy individual files & folders. When done, just close the 2nd instance. (Again, I wouldn't try copying files onto the image. Just copy them off.)

While 3rd-party imaging apps do allow the retrieval of individual files & folders (as well as to restore the entire image) & they do allow imaging non-Windows partitions too - I doubt any of them will create the shadow copies that the Windows version makes available in File Explorer.
