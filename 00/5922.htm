rob koch say:

How can I delete numerous Windows Defender Definition updates? I now have 62 old updates using up disk space. Why doesn't the new update remove the old/obsolete one? My friends all have the same problem/question.

Thanks for your help,    Fred     

Where are you looking to see these updates, in most cases these should be found in system folders with either limited access or shouldn't be manually browsed?

Do not mess with anything in Programs and Features or any other installation page, since new updates simply overwrite the older ones, though the logs don't indicate this, so it appears to the unknowledgeable (i.e. everyone here except Andre) that these older copies are being left on the system.

In any case, the amount of disk space used by these files should typically be relatively small, on the order of 150 to 250 MB.  Since they are managed by the Windows Updates system, they are normally removed automatically after a few weeks and so are typically only a concern on an old PC with a tiny HDD.  In my case these total about 211MB, with the oldest (~3 MB) file dated May 19th.

Personally I'd simply ignore such a small amount of disk use, since even the temporary cache folders used for browsing are typically larger and my system has many GB of available space.  However, if you truly need this tiny amount due to an outdated system with inadequate storage, then you should use the Disk Cleanup tool that Microsoft made available for this purpose.

To access this, open Windows File Explorer from the task bar and then right-click your system drive (usually C:) and select Properties.

In the Local Disk (C:) Properties dialog, click the Disk Cleanup button on the right

In the Disk Cleanup for (C:) dialog, click the Clean up system files button near the bottom

After a few minutes the Disk Cleanup for OS (C:) dialog will display, including additional entries to check like Windows Defender Antivirus.  This will clean up only those outdated definitions and other Defender files that are safe to remove, leaving the most recent copies that might be needed due to possible corruption or for other reasons.

I would not recommend selecting the Windows Update Cleanup checkbox, since some of these files may be needed again and so will simply re-download once they are deleted.

In most cases, the entries found in the initial Disk Cleanup for (C:) dialog are sufficient to free up some space and these are the safest and least likely to be needed again and so re-downloaded by the operating system.

rich michaels - mvp (greatcirclelearning.com) say:
Hi Lukas,

You've got two question in this one post and I'll try to help with both.

Your first question is about the Path property related to ThisDocument. In Office 2011 the Path property returns not only the directory path but also the file name. So you must first strip the filename from the path string. Pseudo code:

source = replace(thisdocument.path, thisdocument.name, "name of excel file")

Your second question is about the Document_New event.

To get it to fire in Office 2011 Word, you have to put a delay into the ThisDocument Class. I do it with the following code:

Option Explicit

Dim weThereYet As Integer, StopCount As Integer
Sub AutoNew()
    HavePatience
End Sub
Sub AutoOpen()
    HavePatience
End Sub
Sub HavePatience()
    
    If StopCount > 9 Then
        MsgBox "Sorry, your document failed to open."
            StopCount = 0
            End
    End If

    StopCount = StopCount + 1

    weThereYet = Application.Documents.Count

    If weThereYet < 1 Then Application.OnTime _
        Now + TimeValue("00:00:01"), "HavePatience"
    
End Sub

Private Sub Document_New()
'now this will execute
End Sub

Private Sub Document_Open()
'now this will execute
End Sub

I have found that the HavePatience delay is enough to allow the Normal template of Word to load and then return control to the macros in your document or template.

Hope this helps

Yes, we can get it via Windows PowerShell. Here are the steps：
