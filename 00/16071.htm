rohn007 say:
the "CVH" is part of the "Virtual" machine that starter actually runs in. Starter is not physically installed on you computer in the traditional way. Starter is running inside a "virtual machine" that connects to servers at microsoft that actually "run" starter for you.

I have only had a brief opportunity to play with Starter and normal installations. In that time I've found that even though I had my installed Office selected as default, simply running either of the Starter apps automatically switched defaults to point to both Starter apps.

Repairing your installed Office will work, but I found it took too long and it required a reboot for some reason. The simplest fix is to "reregister" Word and Excel from you installed copy.  The command is simple enough, you just have to modify the path to match where you have Office installed on your computer.  The first command "un-registers" it, then the second one registers it again, refreshing all of the required registry entries

"C:\Program Files\Microsoft Office\Office\winword.exe" /unregserver

The quotes are necessary, but you may need to change the path so it reflects the location of the winword.exe file on your system. (You can use Windows’ Search tool to locate the path to the winword.exe file.) The /unregserver switch “unhooks” all the references to Word in the Windows Registry.

You should then, immediately, use the following command:

"C:\Program Files\Microsoft Office\Office\winword.exe" /regserver



I actually put the 4 commands, 2 for Word 2 for Excel, in a batch file that I can run any time I need to fix it again.

In Windows 7, the default location for the Startup folder is

C:\Users\[User Name]\AppData\Roaming\Microsoft\Word\STARTUP

If you do not see the AppData folder, i Windows Explorer, click on the Organize drop down and then on Folder and search options and in the Folder Options dialog, go to the View tab and select the item "Show hidden files, folders, and drives".  While there, it is a good idea to uncheck the box for "Hide extensions for known file types".

You can determine if the error is however being caused by an add-in by starting Word in Safe Mode, which can be done by holding down the CTRL key while starting Word until asked to confirm that you want to start it that way.  After starting it that way, see if you can then use File Save As.
