Hi.

1. Be sure that Windows is up-to-date.

2. Remove all accounts from Family in Settings and on MSA website as well and set it up again – based on: https://support.microsoft.com/en-gb/help/10551/...

3. Create a new test child MSA, then create a new user account in windows with the new test child MSA, then turn on family safety for the child MSA and check if it work. If it works then its an issue with the windows user account.

If the problem still persists then follow the below mentioned to delete any corrupted files for family safety.
a. Sign out all the monitored account from the computer.
b. Sign in with the administrator account.
c. Navigate to C:\ProgramData\Microsoft\Windows\Parental Controls and delete all the contents in the Parental Controls folder.
d. Restart the computer and sign into the administrator account again.
This will delete any corrupt files. Your settings will be downloaded again from familysafety.microsoft.com the next time you sign in.

4. Disable/uninstall security software and test it again.

We know this is the presently booted Windows because it says Boot on the C partition, and we know it's Legacy/MBR because it is flagged System Active on the System Reserved boot partition and not EFI System Active flags don't exist on UEFI partitions.

So the question becomes why is there an EFI System on DISK1? Is there still an existing WIndows installation on that hard drive? Why is it installed properly in UEFI mode while the one on Disk2 is not? This is not correct.

As to the SSD Extra G partition which you apparently created after giving up on Extending C on DISK2: You can delete that and the Recovery partition in between C and G to extend C across if you want, but the install willl not have the automatic Repair features that are housed on Recovery on that drive.
