I presume you mean....  You had formulas of the form IF(condition,1,"").  When you copy those cells and use paste-special-value (into the same cells?), the cells that appeared blank originally fail a test of the form ISBLANK(A1) in another cell.  Yet when you select the apparently-blank cells and press Delete or Backspace, ISBLANK(A1) now returns TRUE.

I confirmed that behavior using Excel 2003.  We can use the formula =IF(FALSE,1,"")

The simple solution is:  replace the test ISBLANK(A1) with A1="".  For example, IF(A1="",TRUE,FALSE).

To gain some insight, put ="" into A1.  Note that LEN(A1) returns zero, yet ISBLANK(A1) returns FALSE.  The reason is:  ISBLANK tests whether the cell is truly empty:  no constant and no formula.  The function should have been called ISEMPTY.  A cell is not "empty" if it contains a null string constant (apostrophe preceding nothing) or a formula that returns a null string, even though the cell __appears__ to be blank.

In that experiment, the failure of ISBLANK might be obvious since the Formula Bar clearly shows the formula ="".

When we copy that formula and paste-special-value, we no longer see any formula in the Formula Bar.  The Formula Bar also does not show a single apostrophe, indicating the presence of a null string constant.

But in fact, the cell does contain a null string constant.  That is evident by the failure of ISBLANK.

As noted, the work-around (besides manually deleting the constant) is to use A1="" instead of ISBLANK(A1).

A1="" is a better solution in general since it also works when you have formulas that return the null string -- the original situation before you did the copy-and-paste-special-value.
