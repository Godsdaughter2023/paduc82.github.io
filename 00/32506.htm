1. Go to (ALT to reveal pull down menus) Tools>Folder options...

2. Then select the View tab.

3. Check "Show hidden files, folders, and drives"

4. Uncheck "Hide extentions for known file types"

AND

5. Uncheck "Hide protected operating system files (Recommended)"

7. Next perform a search on the two folders for "*db".  You will probably see a bunch of Thumbs.db files.  You can delete them all, then perform your Properties comparison on the folders again.

File "Size" (not Size on disk) and number of files/folders should now match ensuring your folders are indeed in sync!

P.S. Also, be sure you are viewing file by "Details" to avoid the thumbnail files being regenerated.

    ¡firedog say:

    I can't help with your problem, but I can make a few comments:

    1. Kaspersky produce a reputable anti-virus program. Like all anti-virus software, it has to be invasive to ensure that malicious code doesn't insinuate itself into the deeper reaches of Windows. However, I'm not sure what other functions your Kaspersky software is carrying out, but they might well be interfering with the proper functioning of Windows Live Mail.
    2. Programs like SpeedyPC Pro are in the opinion of many Windows security experts to be avoided at all cost. They are just as likely to cause harm as do good.
    3. You didn't say what the SearchProtocolHost dialogue said. This program is an essential part of Windows Search, passing files to be indexed to the right protocol handler for the particular file type. This makes it possible, for example, to use the search box to find mail messages from a particular sender. Because it's part of Windows, you can't remove it.
    4. An attachment to a mail message is simply a bunch of code at the end of the message itself. It doesn't become a file until you save it. No program will alter mail messages in the way you describe, so it's not at all clear what behaviour you're seeing. When you save an attachment, you have the opportunity to give it a file name. For bank statements, this could be something like Statement-2012-12.pdf for the December one.
      If the sender has included a filename with the attachment (it's not required, so there might not be one), this will be shown in the message header bar in Windows Live Mail. Is it always the same?
    5. It sounds as if Kaspersky is worried about these password-protected PDFs and is salting them away somewhere rather than allowing them to be stored in Temporary Internet Files, as they would be normally when you open them direct from within Windows Live Mail. It is then delivering the wrong one when Windows calls up your PDF reader to open the file.
