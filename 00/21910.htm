It seems that if you have a subform in datasheet view, any VBA code that sets any of the fields default values, column width and field order is lost no matter what you do.

Once you've got your column widths and field orders the way you like it, you can re-enable the VBA code to function as usual and your column widths and field orders continue to be retained.

Later, if you need to make adjustments again, you just disable the VBA code that sets default values, adjust, then re-enable the VBA code.

Just another Access 2010 quirk to work around.
