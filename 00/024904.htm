The database Utility lets you set the default identity, but beside using the Database Utility, there is no other option… As I was saying, Outlook is not intended to be used with more than one identity. It follows the recommended MacOS X scheme: multiple users should each use their own MacOS X account.

Reindexing should affect performances, but it sure affects search results and smart folders.

The Multiple identity option you see in the database utility is related to how the utility rebuilds identities. It creates a fresh new one, but keeps the old one (renamed) so that you could revert back in case something goes awfully wrong. The idea here is to be able to use the database Utility to revert back to a previous version of your identity, not to manage separate identities meant to be used in parallel.

john korchok say:

It's not a bug, it's a fact of Word document structure. It's the same phenomenon as if you create a new Word document, then try to delete the single carriage return in the otherwise blank document. You can't.

Every text box you insert, or table cell you create has an undeletable carriage return as it's initial content (though they look different in a table). The extra space at the end of the comment is the carriage return. Just select the characters you want to delete and it will work.

If you choose Word>Preferences>View and check All in the Show Non-printing Characters section, you'll be able to see the carriage returns.

jpecc say:

The thing is, it won't delete anything in the comment if the selection contains the last carriage return. It's not just the last invisible character. So you have some text selected, you hit the delete key, and nothing happens. And it only affects comments. 

If that's not a bug, I don't know what that is.

Using Word 2013, I opened a new Word file on a network share that had images in the same folder. I am assuming you mentioned that because the images are linked. I used INSERT>Pictures and changed Insert to Link to File, then linked an image, saved and closed the file.

I then opened the file in Word 2016 for Mac. The linked image did not display, but the bounding box contained an icon with a red X and a message The linked image cannot be displayed. The file may have been moved, renamed, or deleted. Verify that the link points to the correct location. This happens because Word 2013 includes the complete path to the linked image using DOS syntax that Word 2016 for Mac cannot understand.

If this sounds like your circumstances, you can click on Edit>Links in Word, choose Change Source... and update the link. Now it will have a macOS syntax and the links will not display in Word 2013.
