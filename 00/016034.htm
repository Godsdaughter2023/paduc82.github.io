jllatham say:

Here's one possible way of doing it.  Set up a table, kind of like the ones you used to see in the paper Rand-McNally road atlas books at the end to get distances between cities vs using MapQuest :)

It might look something like this:

                             Nebraska          Mich State            Michigan

Michigan                 W                    L

Mich State

Nebraska                                       W                         L

In this case it would interpret as Michigan played Nebraska and Won over them.  They played and lost to Michigan State.

You can find the W/L status with a formula like this:

INDEX(I4:L7,MATCH(A3,I4:I7),MATCH(A2,I4:L4))

the table is in I4:L7 on the same sheet, and A3 holds Michigan and A2 holds Nebraska.  Then you can use that result to add a half a point to the PF like this:

=IF(D3=D2,IF(INDEX(I4:L7,MATCH(A3,I4:I7),MATCH(A2,I4:L4))="W",0.5,0),0)

What that would do is come up with .5 if there was a tie but Michigan had Won over Nebraska. It returns 0 if no tie or if they lost to Nebraska.  You could add that into the formula for coming up with the PF value and base the ranking based on those results - and at this point the thinking gets a bit fuzzy:

if you move Michigan above Nebraska, then they will both show 287 (assuming they started as both being 287), but they will be properly ranked: Nebraska would check for a Tie and find that there is a tie, but then go on to do the lookup and find that they'd lost that game, so no extra half point, and the ranking is correct.

If the issue persists, I would suggest you to set the browser mode to Internet Explorer 10 and then check. Refer to these steps:

b)      In the new window that appears at the bottom of the screen select ‘Emulation’ in the left pane.

c)       Click on ‘user agent string’ and select ‘Internet Explorer 10’.

d)      Press the ‘F12’ key to exit the window and the refresh the page and check.

 

Try these steps and let us know the results for further assistance.
