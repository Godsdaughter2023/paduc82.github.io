Public Function GetNextID() As Long
    Dim lCurrID As Long
    Dim File As String

    File = "C:\Sample.txt"
    lCurrID = Val(ReadFile(File))
    lCurrID = lCurrID + 1
    WriteFile File, CStr(lCurrID), False
    GetNextID = lCurrID
End Function


http://www.vboffice.net/sample.html?mnu=1&smp=14&cmd=showitem

Public Function ReadFile(sPath As String) As String
    On Error GoTo AUSGANG
    Dim lFileNr As Long
    Dim sText As String

    lFileNr = FreeFile

    Open sPath For Binary As #lFileNr
    sText = Space$(LOF(lFileNr))
    Get #lFileNr, , sText

AUSGANG:
    If lFileNr Then Close #lFileNr
    If Err.Number Then Err.Raise &H800A0000 Or Err.Number, _
      Err.Source, Err.Description, Err.HelpFile, Err.HelpContext

    ReadFile = sText
End Function

Public Sub WriteFile(sPath As String, _
  sText As String, _
  Optional ByVal bAppend As Boolean)

    On Error GoTo AUSGANG
    Dim lFileNr As Long

    lFileNr = FreeFile

    Select Case bAppend
    Case False
        Open sPath For Output As #lFileNr
    Case Else
        Open sPath For Append As #lFileNr
    End Select
    Print #lFileNr, sText;

AUSGANG:
    If lFileNr Then Close #lFileNr
    If Err.Number Then Err.Raise &H800A0000 Or Err.Number, _
      Err.Source, Err.Description, Err.HelpFile, Err.HelpContext
   End Sub

saint gertrude say:

Return blank cells if no data is entered

Hi there,

I have a spreadsheet that calculates prices for a product. Each product has a column for a set-up or FIXED PRICE another for a VARIABLE/UNIT PRICE and one where I can enter a quantity to work out the total price. For example - a fixed price of $100 + a unit price of $1 x quantity. If I enter a quantity of 100 in the quantity field, I return a total price of $200. 

Right now, the formula is =FIXED PRICE+(UNIT PRICE*QUANTITY). The problem is, that by default Excel returns a price of $100 even when nothing is entered in the quantity column. What do I need to add to this formula so that the price field is completely blank unless a quantity is specified?

Cheers!
tasos_k say:

Hi,

try this..

let's say that quantity field is column B e.g cell B1

and your formula is in C1

in C1 write

=IF(B1="","",(FIXED PRICE)+(UNIT PRICE)*B1)

 

jllatham say:

If you're working with a table then it might look like this:

=IF(Table1[QUANTITY]=0,"",Table1[FIXED PRICE]+(Table1[UNIT PRICE]*Table1[QUANTITY]))

 

NOTE: in either case, when quantity is 0 or empty, the calculated value cell will contan an empty (or zero length) string.  You can't do a lot of math with strings/text.  For example, if these formulas were in cell C1 and you tried to do this when C1 had returned the "" result:

=C1+4

You would get an #VALUE! error because Excel would not be able to deal with the "".

But if you use the SUM() function, it would work, as:

=SUM(C1,4) 

With the result being 4 in this special case since SUM() treats text and empty cells as a zero.

Get-AppxPackage -allusers Microsoft.SkypeApp | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}
