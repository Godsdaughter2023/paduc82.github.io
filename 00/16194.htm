dirk goldgar say:

Unfortunately, you can't do it that way.  The SQL interpreter will not understand the content of a filter provided via a control reference or parameter.  You pretty much have to build the SQL statement as a string, incorporating the filter value, and then pass that off to be processed.  For example,

    Dim strSQL As String

    strSQL = "SELECT Field1, Field1 FROM MyTable WHERE " & Forms!SelectforReport3Dialog!lFilter

    ' And then ...

    ' you might open a recordset:

    ' or you might set the recordsource of a form:

    ' or you might modify the SQL of a saved query, so that something else can use it:

If you need this filtered query as the recordsource of a report, you may need to take that last approach, unless you can have the report itself modify its RecordSource property in its Open event.

teylyn say:

Hello,

you can use a formula along these lines:

=AND(A2>=TODAY()+30,A2<=TODAY()+45)

Applied to cell A2. Because you want to format days in a specific time frame, the date must be greater than today plus 30 days AND at the same time less than today plus 45 days. Therefore, you wrap the two comparisons into the AND statement. The screenshot has this formula in the conditional formatting and also in column C, starting in C2 and copied down.

diane poremsky mvp (slipstick.com) say:

Yes, you can open it. It's been a while since I've had to do it but AFAIK, this still works.

 

Open the registry editor, select the HKEY_LOCAL_MACHINE branch.
Go to the FILE menu and choose LOAD HIVE
Browse to the location of the old registry hive. - it's in the user account - in win7 this is at x:\users\{username}\NTUSER.DAT
You want to open the upper case “NTUSER.DAT” and not “ntuser.dat”.

BTW, if that doesn't have the data you seek,  try opening the reg files C:\Windows\System32\Config\.

 

And it helps to know where they are - go to

[HKEY_CURRENT_USER\Software\Microsoft\Windows NT\CurrentVersion\Windows
Messaging Subsystem\Profiles\profilename\ - usually Outlook - and search for 001f0418 - that is one of the 4 keys. The others will be right under it - one each for the addresses visible in the Junk mail options and one for addresses in your address book (if trusted). ETA: you only need the 3 keys, don't import the trusted address book.

In 2007, there should be one set of keys for all accounts.

 

In the old registry, the exported keys will be fore local machine, change that to current user before importing. Also make sure the subkey is the same as used on the new computer.
