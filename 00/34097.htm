You must either use group policies with the GPMC...

In Windows 7, this can be accessed via GPEDIT.MSC (Pro, Enterprise, or Ultimate only) and the policies to be updated are at

You can use either of Computer or User configuration. Either way, this can only be run by elevation and unfortunately the local user GPO applies to all. You can not designate this to a specific user. It would be a nice feature in the future but... this would be good for a single restricted terminal.

Windows Components\Internet Explorer\Internet Control Panel\Security Page\Internet Zone

 

The other methods are to block via registry or IEAK8.

http://support.microsoft.com/kb/182569/en-us - Advanced Registry editing for Internet Zones

http://technet.microsoft.com/en-us/ie/cc889351.aspx - Internet Explorer Administration Kit IE8 Documentation

http://technet.microsoft.com/en-us/ie/bb219517.aspx - IEAK6,7,8 Downloads Multiple Language selections.

 

Your Alternatives are programs like "Zone Alarm" which may detect downloads, and block it for a particular user but then also have a bypass option for an authorized user (one whom has the password to turn off the block temporarily.).

http://www.zonealarm.com - I recommend the Extreme Security Suite as it's 1, cheap, 2, includes just about all of there other products and you can use the license for 3 systems.

 Another alternative, is check the configurations on your router's firewall or purchase a Cisco home router or something with stateful packet inspections to detect datatypes that you want to restrict. If you need to google for one, search "SPI router." (Stateful packet inspection router)

Another cheap alternative, is you can purchase an old used computer from ebay in a range of $20-50 and throw "Untangle" linux on it with 2 NICs, Once you install it, you can configure it to block filetypes. You'll have a little bit more networking to do but it works out in the end with it. It's a pretty easy, out of the box unified threat management operating system.
