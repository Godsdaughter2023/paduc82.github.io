When choosing Properties on your Display adapter in Device Manger or when running dxdiag you see a message like this:

Device installer error
Windows could not load the installer for the Display.  Contact your hardware vendor for assistance.

The usual cause of this is that this file has somehow come up missing on your system (assuming XP is installed in the Windows folder on your C drive):

c:\windows\system32\desk.cpl

Is the file missing from your system?  If so, you must replace it.

Another symptom of the missing file is when you right click you desktop, Properties, nothing seems to happen.

Another symptom of the missing file is Display option is missing from Control Panel.

Since the desk.cpl file is one of the 3498 files that Windows File Protection looks after, it should never be missing and it takes extra effort to delete the file on purpose.   If yours is missing, it may have been quarantined (effectively deleted) by your unspecified antivirus/antimalware application.

Let me assure you with 100% certainty that running sfc /scannow will not replace the missing file.  That is not what sfc /scannow does, that is not how it works and that is not what it is for.  You are going to have to replace the missing file yourself.

Look for a backup copy of the desk.cpl file on your system in one of these two folders:

c:\windows\system32\dllcache
c:\windows\ServicePackFiles\i386

If you are able to find a copy of the file in one of these folders, copy and paste it here where it belongs (assuming XP is installed in the Windows folder of your C drive):

c:\windows\system32

If you can't find a copy of the file on your system and you do not have a genuine bootable XP installation CD that is the same Service Pack as your installed Service Pack to extract a fresh copy of the file (most people don't), there is a copy of the file on my SkyDrive you can download and replace your missing file.

After replacing the missing desk.cpl file, look in Control Panel for the Display icon to be sure it is there and it may be near the bottom since it was just added.  Right click an empty area in Control Panel and choose to 'Sort by Name' to reorder the items in alphabetical order.

Reboot your system and be sure the changes 'stick'.

The message you are seeing trying to register the sfcfiles.dll file has nothing to do with this problem and the reason you are getting this error:

RegSvr32
sfcfiles.dll was loaded but the DllRegisterServer entry point was not found.

That is because the sfcfiles.dll file is not a registrable file.  Just because a file has a .dll extension does not mean you can manually register it.

    You can run Disk Cleanup - built into Windows XP - to erase all but your latest restore point and cleanup even more "loose files"..

     You can turn off hibernation if it is on and you don't use it..

     When you hibernate your computer, Windows saves the contents of the system's memory to the hiberfil.sys file. As a result, the size of the hiberfil.sys file will always equal the amount of physical memory in your system. If you don't use the hibernate feature and want to recapture the space that Windows uses for the hiberfil.sys file, perform the following steps:

     - Start the Control Panel Power Options applet (go to Start, Settings, Control Panel, and click Power Options).

    - Select the Hibernate tab, clear the "Enable hibernation" check box, then click OK; although you might think otherwise, selecting Never under the "System hibernates" option on the Power Schemes tab doesn't delete the hiberfil.sys file.

    - Windows will remove the "System hibernates" option from the Power Schemes tab and delete the hiberfil.sys file.

     You can control how much space your System Restore can use...

     1. Click Start, right-click My Computer, and then click Properties.

    2. Click the System Restore tab.

    3. Highlight one of your drives (or C: if you only have one) and click on the "Settings" button.

    4. Change the percentage of disk space you wish to allow.. I suggest moving the slider until you have just about 1GB (1024MB or close to that...)

    5. Click OK.. Then Click OK again.

     

    You can control how much space your Temporary Internet Files can utilize...

     Empty your Temporary Internet Files and shrink the size it stores to a size between 64MB and 128MB..

     - Open ONE copy of Internet Explorer.

    - Select TOOLS -> Internet Options.

    - Under the General tab in the "Temporary Internet Files" section, do the following:

    - Click on "Delete Cookies" (click OK)

    - Click on "Settings" and change the "Amount of disk space to use:" to something between 64MB and 128MB. (It may be MUCH larger right now.)

    - Click OK.

    - Click on "Delete Files" and select to "Delete all offline contents" (the checkbox) and click OK. (If you had a LOT, this could take 2-10 minutes or more.)

    - Once it is done, click OK, close Internet Explorer, re-open Internet Explorer.
