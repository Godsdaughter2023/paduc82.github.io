aakkam22 say:

A little background first...

With the release of Windows 8 and Windows 10, many new computers are shipping with UEFI firmware in place of the legacy, BIOS firmware. This changes the way operating systems load and interact with the hardware on your PC.

So what does this mean for me?

If your PC is set to boot in EFI mode, Microsoft has blocked the loading of legacy Windows operating systems in the boot menu. This includes Windows 9x, Windows XP, and Server 2003. You also can't add Linux or DOS entries.

Then how was I able to dual-boot Windows XP before I installed Windows 10?

Some newer PCs continue to include the older, legacy boot in addition to EFI-boot. It is likely that Windows 7 and XP were installed using legacy boot, and the Windows 10 install used EFI boot during its installation.

What is the solution?

You have two options: you can revert to using legacy boot, or you can use a GRUB boot loader instead of the Windows Boot Manager. The first option requires a reinstall of Windows.
