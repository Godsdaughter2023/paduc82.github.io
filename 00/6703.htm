Re-register the application

There are several different ways you can reset the proper file associations. Word itself provides a way to do it. Open a command prompt window and, at the command line, enter the following:

winword /r

Nothing seems to happen when you pressEnter, but Word will reregister itself back in Windows. In the process, it resets the file associations for the files it uses, such as DOC and DOT files. You can close the command prompt window and double-click on a document; Word should start right up.

Alternate Re-register the application

Close the app. Open the Run Window: <WIN> R, use the following command (Modify the Path to match your specific installation):

"C:\Program Files\Microsoft Office\Office\winword.exe" /unregserver

The quotes are necessary, but you may need to change the path so it reflects the location of the winword.exe file on your system. (You can use Windows’ Search tool to locate the path to the winword.exe file.) The /unregserver switch “unhooks” all the references to Word in the Windows Registry.

You should then, immediately, use the following command:

"C:\Program Files\Microsoft Office\Office\winword.exe" /regserver

This command causes Word to rewrite all its Registry keys and to reassociate itself with document and template files. Now you can double-click a document file and see if Windows starts Word and opens the document.

ossiemac say:

Try the code below. It resaves back to the original name after creating the backup.

Note that if the code is in the workbook to be backed up then the following line can be altered edited to ThisWorkbook

Set MyBook = Application.Workbooks("benacrest demo 3.xlsm")

Can be edited to the following

Set MyBook = ThisWorkbook

Example code: (Edited since initial post. Commented out Set MyBook = ThisWorkbook )

Sub SaveBackUp()
    Dim MyBook As Workbook
    Dim strMyBookName As String
    Dim strBackupPath As String
    Dim strBackUpName As String
    Dim Valid As Range
    Dim Numm As Integer
   
    Set MyBook = Application.Workbooks("benacrest demo 3.xlsm")
    'Set MyBook = ThisWorkbook    'Alternative if code in workbook to be backed up
    strMyBookName = MyBook.FullName       'Save full path and name of workbook
    Set Valid = MyBook.Sheets("validationlists").Range("U1")
    'Set Valid1 = MyBook.Sheets("validationlists").Range("U2")
   
    Numm = Valid.Value
    'No need to change directory
    strBackupPath = "C:\Users\New\Dropbox\"  'Note backslash on end
   
    'Include backup number with name; not after the extension
    strBackUpName = "Benacrest back" & Numm & ".xlsm"
   
    MyBook.SaveAs Filename:=strBackupPath & strBackUpName, _
         FileFormat:=xlOpenXMLWorkbookMacroEnabled, CreateBackup:=False
    
    Valid.Value = Valid.Value + 1   'Increment before Save As so it is in original workbook
    'The origfinal workbook name will already exist so suppress the alert message
    Application.DisplayAlerts = False
   
    'SaveAs again back to original workbook name.
    'Note: strMyBookName is full name that includes the path.
     MyBook.SaveAs Filename:=strMyBookName, _
         FileFormat:=xlOpenXMLWorkbookMacroEnabled, CreateBackup:=False
    Application.DisplayAlerts = False
 End Sub
