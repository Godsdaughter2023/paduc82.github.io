bossdweeb say: So true, but it's not always caused by lag from the gpu working hard.
It's often a case of the graphics driver being the 'last man standing' when a crash occurs
which leads to a TDR error, but when a TDR is caused by an overloaded gpu  lowering the resolution one notch can help.
Even with frame rates between 35-35fps  I had occasional lock-ups and one TDR with Crysis 2, but lowering my res to 1680X1050 bumped the fps up to 45-50+ and stopped the lock-ups.
While I was getting a decent fps at full res the gpu usage monitor showed just under 100% usage when playing the game. Lowering the res droped it into the lower 90+% use.
Probably better for the longevity of the gpu than a higher time setting for TDR too.

Personally, I would not use the option to disable TDR altogether as if may cause more
damage if there is an actual component problem.
The default time for TDR is 2 sec. I have been using 5 sec with good results (the one exception with Crysis 2)  for over a year.

TDR can be caused by a number of things including software conflicts, or an inadequate psu,
so if the registry fix  does not solve the problem troubleshooting from Clean Boot may help.

How to troubleshoot a problem by performing a clean boot in Windows Vista or in Windows 7

What brand and watts is your  psu ?

If your machine was built by a local PC shop/builder, and not from a big company such as
DELL or HP, the drivers from Nvidia will be the best option.
I have never uninstalled/reinstalled graphics drivers via Device Manager as these are only the default Windows graphics drivers, and will normally reinstall themselves when re-booting
after uninstalling the drivers via Programs and Features anyway.

Nvidia also has a separate download called  System Tools. It has several tools, one of which
is a hardware monitor which will measure the temps on the gpu and cpu.  It will also monitor
the motherboard chipset temps if  the mobo uses  an Nvidia chipset.

bossdweeb say: It is always best practice to buy RAM in matched kits. Buying an extra kit of two sticks to
add to the original will usually work as long as the RAM is the exact same brand and model,
but having the same revision of that model is better.
Using mismatched RAM is very hit and miss and prone to instability, or loss of full performance.
It is my understanding that using 2X2GB with 2X1GB will cause the RAM to run in single channel
mode rather than the faster dual channel mode.

If the mobo's specs specify it can run four sticks of 2GB RAM it will  certainly do
so, if there is not other problem, and should support at least 8-16GB,
(assuming 64bit Windows Home Premium).
RAM is so cheap these days, I reckon you should buy a kit of two 4GB RAM.

Another potential for RAM to cause a BSOD or TDR  is changing it's settings in BIOS.
For most recent processors (except some of the latest) the max speed of the memory controller
without overclocking is 1066 or 1333MHz, and trying to run 1600MHz at that speed by
setting the RAM to 1600 in BIOS without OC'ing the cpu will cause a BSOD or TDR.
Also, using only two slots tends to be more stable at higher RAM speeds.

Type Printer in Windows Search box > Printers & Scanners > Select your printer > Remove device
