jllatham say:

I almost suggested the same thing OssieMac did - and since he has offered that, I'll offer something a little different.

You'll need some code in two places on your UserForm - one piece to 'remember' the name selected when it is, and one to look and see if there's a remembered name and stick that into the ComboBox.

First part would be the _Change() event for the ComboBox itself:

Private Sub ComboBox1_Change()
  If ComboBox1.Text <> "" Then
    'this could be on a hidden sheet

    'save the selected/entered name in a cell on a worksheet in the workbook.

    Worksheets("Sheet1").Range("B1") = ComboBox1.Text
  End If
End Sub

Next I chose to use the _Activate() event for the UserForm - this will be triggered anytime the userform is opened for the first time with a .Open command or when it is still in memory but not displayed and is redisplayed after it has been hidden from view.

Private Sub UserForm_Activate()

  'if we have a saved name from earlier use, put it into the ComboBox

  If Worksheets("Sheet1").Range("B1") <> "" Then

    Me.ComboBox1.Text = Worksheets("Sheet1").Range("B1")
  End If
End Sub

If you want to go with OssieMac's method, then you don't need the ComboBox1_Change() event at all, and you could modify the UserForm_Activate() process like this

Private Sub UserForm_Activate()
  Me.ComboBox1.Text = Environ$("Username")
End Sub

The catch here might be if this was being used on a system where some individual logs into it at some point in the day, but one or more other people actually use the Excel file without having logged out/in with their own account.  In a situation like that, if Smith logged in, and then Jones came along to enter 1 or 1000 records using the UserForm, then poor Jones is going to have to choose/enter "Jones" into the combobox over and over - just what you're trying to get away from.

The fact that you are having the same problem almost immediately after a hard reset tells us a few things:

1.) The trouble is not with your PDA; therefore the trouble must be with your desktop computer.

2.) If the problem is due to a corrupt BIOS, you won't fix it with a Hard Reset. It may take loading updated BIOS.

3.) The trouble could be with your PDA hardware, in which case the problem will continue forever until repaired/replaced.

 

However, I am still convinced that there is something loaded on your PC that is interferring with (or causing an incompatibility with) ActiveSync, instead of any of the possibilities listed above.

 

Plaxo, Franklin Covey, and others all make "plugins" that can be added to Outlook to make it work better with their service or product offering.

 

If I go to Tools > Trust Center > Add-ins, I can see all the plug-ins that might be causing a problem.  If I want to remove one of them from starting next time I run Outlook, I need only go to the bottom of the Add-In dialog where it says "Manage: COM Add-ins" and pick the "Go" button.  Then I just uncheck the offending Add-in and it won't start next time I run Outlook.

 

In earlier versions of Outlook, you will need to go to View > Toolbars > Customized and find the plugins that are current in your installation of Outlook.   If you find any installed, I would seriously consider unchecking them all, shut-down Outlook, and then restart to see if this solves your ActiveSync problem...
