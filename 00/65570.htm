DEP is a feature that prevents memory that is allocated to data from executing code.

 

In the original design of the Intel 8086 processor strings (text) were handled efficiently by repeating the last instruction until you came to a null (or zero value) when the repeating instructions stopped.

 

Years later, after the internet started, malicious people started sending computers text without a terminating null character. This caused the processor to copy the entire amount of data sent over adjoining memory to what was allocated. There are many protections against this now.

 

Windows doesn't use the CPU instructions that do this at all (except for compatability). Likewise programs that use Windows features don't either.
