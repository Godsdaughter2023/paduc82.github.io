There are 2 types of license that won't allow transfer. An "OEM" factory activated license and some of the volume licenses which your company may have bought.  OEM happens to be the only license type that it is possible to check for.  Go into Help > About. ken sheridan say: In the AfterUpdate event procedure of the combo box in the parent form:

1. Requery a list box whose RowSource is a query on the class types table which references the combo box as a parameter.

2. Use a Select Case…..End Select construct to change the SourceObject property of the subform control to the name of the relevant form object for the class type selected.

In the AfterUpdate event procedure of the list box requery the subform control.

Each source form object's RecordSource should be a query which references the list box as a parameter.  If, by defeault, you want to return all rows in the subform prior to restricting it by making a selection in the list box, then the parameter should be referenced using the following syntax:

WHERE ([ClassType] = Forms![YourParentForm]![YourListBox]
OR Forms![YourParentForm]![YourListBox] IS NULL)

Otherwise simply as:

WHERE [ClassType] = Forms![YourParentForm]![YourListBox]

The reason you have had a hard time finding guides is because your workflow is not common, at least not in Access development.
