mike h.. say:

Hi,

ALT+F11 to open VB editor. Double click 'ThisWorkbook' and paste the code below in on the right. Change the sheet and cell to the one where you want the time stamp

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
Sheets("Sheet1").Range("A1") = Now()
End Sub

If you want you can add who saved the workbook

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
Sheets("Sheet1").Range("A1") = Environ("Username") & " " & Now()
End Sub

Let me throw out my thoughts, others may chime it with theirs.

First, "effective resource management" is a subjective term. If you are looking for an automated built-in feature that will manage all your resources, then no, Project can't do that. What it can do is to help you manage your project and its resources by keeping track of your resource availability (i.e. work schedule, time off, etc.) and insure that no resource is overallocated (i.e. assigned to work more hours in a given period or during resource time off).

Project does have a resource leveling feature that will move overallocated resource assignments based on a priority that you set. It may or may not be able to resolve all overallocations.

If your resources have particular skill sets (e.g. plumbers, software programmers, hedge fund managers, etc.) you can group those resources into skill groups and then assign that skill group as needed.

I'm not aware of any Project add-ins that will provide the level of automated resource management that I think you want. It might be possible to use some VBA to automate the process providing you can create the necessary algorithms to drive the process.

tom_bpc say:

Blair,

This elapsed percentage lagtype is a weird one that Microsoft seems to have included just to fill out the matrix.  It has no rational use in my world.

As implemented in your example, a "50e%" lag with a "10d" predecessor is 40 hours (i.e. 10d x 8h/d x 50%) - applied through a 24-hour calendar.  The negative lag will be satisfied at 1:00 AM on 11Jan'18 (40 hours before the predecessor's completion).  The successor task is then delayed by its own calendar to start at 08:00 on that day.

An arithmetic curiosity, nothing more.

hansv mvp say:

I think Stop and Pause can be the same button. Here is code for the buttons; it is intended for a "standard" module:

Dim dtmTime As Date
 

Sub StartTimer()
    If Range("B2").Value = "" Or Range("B2").Value > Range("B1").Value Then
        ResetTimer
    Else
        StopTimer
    End If
    dtmTime = Now + TimeSerial(0, 0, 1)
    Application.OnTime dtmTime, "DoTimer"
End Sub
 

Sub StopTimer()
    On Error Resume Next
    Application.OnTime dtmTime, "DoTimer", , False
    On Error GoTo 0
End Sub
 

Sub ResetTimer()
    StopTimer
    Range("B2").Value = Range("B1").Value
End Sub
 

Sub DoTimer()
    If Range("B2").Value > 0 Then
        Range("B2").Value = Range("B2").Value - TimeSerial(0, 0, 1)
        dtmTime = Now + TimeSerial(0, 0, 1)
        Application.OnTime dtmTime, "DoTimer"
    End If
End Sub

Dim dtNext As Date

  Application.OnTime dtNext, "StepTimer"

  If Range("B2") > 0 Then Range("B2") = Range("B2") - TimeValue("0:0:1")
  Application.OnTime dtNext, "StepTimer"
