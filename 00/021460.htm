suzanne s. barnhill say:

In another case similar to this, it turned out that the TOC was a hyperlinked one (TOC field contained the \h switch), and the Hyperlink character style, which should be based on Default Paragraph Font, had acquired specific font formatting. The solution was as follows:

  1. Display the Styles pane.

  2. In the Styles pane, right-click on Hyperlink and choose Modify… (If you don’t see Hyperlink, click Options… in the bottom right corner and select All styles and Alphabetical).

  3. In the Modify Style dialog, don’t try to change the font directly (I tried it; it doesn’t work); instead, click Format | Font…

  4. In the Font dialog, don’t change the font; instead, clear the Font, Font style, and Size boxes by selecting the contents and pressing Delete.

  5. Click OK twice to close the Font and Modify Style dialogs.

  6. You should now see the desired font in all your TOC entries.

marilynkcohen say:

Adding a numbered outline in Word 2010 Template/Form

I have created a "Process Template" in Office 2010 that requests the user to enter specific information in several sections. All those sections are working exactly as I want them to.

The last section is where I'm having trouble. This is where users need to describe each step of the process they are working on. I want the template to reflect the outline formatting:
 
1.
    a.
         i.
              i.i. (and so on)

The user should be able to start typing at the top level (1.) and continue as appropriate. This outline section needs to be flexible to allow the user to enter whatever content makes sense. In other words, there may be a step 1., and a step a., but no lower steps. He/she would then go to step 2. and repeat the outline format as needed.

I've requested help several times, but only receive feedback for creating the outline itself. I can do that, but need to know how to adapt the outline to my template.

HELP PLEASE!!!

Thank you,
Marilyn
jay freedman say:
I'm guessing that your template includes legacy form fields in the first part, requiring the document to be protected for filling in forms. Here are the remaining steps:
  • With the cursor in the first paragraph of the last section, create the outline numbering scheme you described. Only the "1." numbering will be visible at that time.
  • Insert a section break (Next Page or Continuous, whichever you need) between the part containing the form fields and the part containing the outline numbered paragraph.
  • On the Review tab or the Developer tab, click the Restrict Editing button.
  • In the Restrict Formatting pane, check the box in the Editing Restrictions section and select Filling In Forms in the dropdown. Then click the Select Sections link below the dropdown.
  • In the little dialog that opens, uncheck the box for the last section. The last section will not be protected.
  • Start enforcing protection, and save the document.

The form fields will operate correctly in the protected part of the document, while freeform editing will be allowed in the last section. The outline scheme you created will provide automatic numbering. Use the Tab and Shift+Tab keys, or the Increase Indent and Decrease Indent buttons on the Home tab, to change the outline level of a selected paragraph.
