OK.  That one looks fine for that version of IE.  Notice that we are apparently supposed to ignore the number 4 that comes after the 9.  But the problem is probably that you are not getting any security patches for related modules.  wininet.dll was just an arbitrary choice for checking.  I imagine that you will have others which are not in that OK state.

Usually I suggest people find the highest version of an IE module which corresponds to their level of IE (because typically they don't even have the level of IE that they thought they did and which currently should be wininet.dll).  Instead for you I will suggest you try looking at the lowest version.  The procedure is to use msinfo32.exe Loaded Modules, sort by Version and then scroll to the first or last one in the group that you are interested in.   The sort is text-based not arithmetic so in your case 9.0 sorted descending would show your wininet.dll with its 16609 at the top of the group.  (The text versus arithmetic factor really only comes up for IE10 and IE11 versions.)  So, using a descending sort, scroll down to the last IE-related module which starts with 9.0 and then see what it looks like using the same fsutil syntax for that name.

Supposedly CheckSUR is supposed to help you untangle all these details and keep you unaware of them but both it and DISM, which is used by W8 users for the same problem, seem to be not as complete or robust as needed for such an aspiration.

Private Sub Workbook_BeforePrint(Cancel As Boolean)
    Application.Calculate
End Sub

 

or if you want to recalculate only a specific cell:

 

Private Sub Workbook_BeforePrint(Cancel As Boolean)
    Worksheets("Sheet1").Range("C3").Calculate
End Sub

 

where C3 is the cell containing the formula =NOW().

I just want to trim the extra spaces and leave the field name the same. I can't just rename the field because the query joins with another table, so the field name needs to remain the same.

scottgem say:
You can't get around this behavior. When you use an expression in a query Access create an alias for it. The way you can get around it is to create a query that does your Trim, and give it an alias of TrimmedFieldname and then use that query as the source for your main query where you give the Trimmedfieldname column an alias back to the original fieldname.
