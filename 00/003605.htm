Slow Word 2013

I experienced a similar issue with slow typing speed in Office 365 Word. I edit a substantial number of Word and Excel documents for work and found the SkyDrive storage incredibly beneficial for working remotely.

I realized though that documents which are being edited from Skydrive, not a local copy, experience the slow typing speed issue.

The fix:   From word:  File, Options. Word Options has an advanced tab on the left labeled 'advanced'. About 3/4 of the way down, a category for 'Save' options exists. The default setting has 'Allow Background Saves' checked. I uncheck this option and instead check 'Always create backup copy' and 'Copy remotely stored files onto your computer, and update the remote file when saving'.

I assume that if you're storing the files on Skydrive, or Dropbox, or web, or maybe even some external drive or flash memory this background save could be lagging and result in typing slowness. Either way, I changed the advanced options, and no more problem. Maybe it'll work for you.

Excel didn't have the option, and I usually save a local copy anyways due to the working file size.

john w. vinson say:

You're still storing JUST ONE FACT in TWO DIFFERENT FIELDS.

 

That's redundant and can be a real problem! Relational databases should store each piece of information once, and only once; if you need to see that it's FINISHED you can just store that fact in the yes/no field, and display it on a different form or a different query.

 

As it is, there's nothing to prevent the checkbox from displaying one value (Finished) while the combo box displays In Process. They can't both be right but there's no solid way in code to ensure that they are reliably kept in synch.

 

I'd say that the Status field in the second table should simply not exist, but should instead be calculated on the fly using a query or a DLookUp.

 

scottgem say:
Since the Status combo only has 2 states, then I agree with John, it shouldn't be in the second table. Just use the check box to indicate On File. 

However, if the status had more than the 2 states, then I would have the Status field in the second table anf remove the checkbox from the first table. Instead I would have an UNBOUND checkbox on the form bound to the first table and populate it in the On Current event with code like:

If DLookup("[Status]","secondtable","[keyfield] = " & Me.Keyfield) = "On File" Then
     Me.checkbox = True
Else
     Me.Checkbox = False
End If
