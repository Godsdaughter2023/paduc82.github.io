cbarnhorst say: No.  A virtual machine copy of Windows 10 requires its own license, unlike a new partition on one of your hard drives.  With a new copy of Windows 10 installed in a multiboot scenario the unique hardware IDs of the computer provide activation for the new
copy.  However, the host machine's unique hardware IDs are not passed through to a virtual machine and cannot be used to activate a copy there.  Instead the virtual machine's emulated hardware has its own unique hardware IDs that are used for activation so
the virtual machine has to be registered as a Windows 10 device on its own using a product key.  The reason is that virtual machines are portable across servers (in order to facilitate load balancing) and dependence on a specific host server for activation
would be chaotic in an enterprise.  You may not be using the vm in an enterprise network environment but the same activation principles apply.  If you want to test on your machine try creating a new partition and installing the new copy of Windows 10 there.  Or obtain a Windows 10 license for the vm.  Or buy a used device on eBay that has Windows 10 pre-installed and use that for testing.  Such used devices are inexpensive.

The answer to your question can be as simple as a Word field (no macro involved) that calculates a series of numbers above or adjacent to the field

{ =Sum(Left) \# "$,#.00;($,#.00);" }

{=Sum(Above) \# "$,#.00;($,#.00);0.00"}

 

Note the first example displays nothing if the sum is zero.

 

up to a complex Word invoice template like the one shown here: http://www.gmayor.com/Invoicer.htm

 

A basic macro for calculating the sum of a column of table cells is shown below.  Note the example focuses on column 3,  ignores the first row (assumed to be a label) and place the result in the last row:

  

 

Sub ScratchMacro()

'A quick macro scratch pad created by Greg Maxey

Dim i As Long

Dim dblSum As Double

For i = 2 To ActiveDocument.Tables(1).Columns(3).Cells.Count - 1

 dblSum = dblSum + CDbl(CellText(ActiveDocument.Tables(1).Columns(3).Cells(i)))

Next i

ActiveDocument.Tables(1).Columns(3).Cells(ActiveDocument.Tables(1).Columns(5).Cells.Count).Range.Text = FormatCurrency(Expression:=dblSum, _

                   NumDigitsAfterDecimal:=2, IncludeLeadingDigit:=vbTrue, _

                   UseParensForNegativeNumbers:=vbTrue)

 End Sub

Function CellText(ByRef pCell As Cell) As String

CellText = Left(pCell.Range.Text, Len(pCell.Range.Text) - 2)

End Function

andreas killer say:
You can copy the string into a byte array, this is very fast. Or use the Mid$ function to store and get the chars from the string, not so fast but much more easier.

Andreas.

Option Explicit

Sub Test()
  MsgBox RandomString("Andreas") & "  " & RandomString2("Andreas")
End Sub

Function RandomString(S As String) As String
  Dim i As Long, j As Long, k As Integer, Length As Long
  Dim b() As Byte, Temp(0 To 1) As Byte
  b = S
  Length = Len(S)
  For i = 0 To UBound(b) Step 2
    j = Int(Rnd * Length) * 2
    For k = 0 To 1
      Temp(k) = b(i + k)
      b(i + k) = b(j + k)
      b(j + k) = Temp(k)
    Next
  Next
  RandomString = b
End Function

Function RandomString2(S As String) As String
  Dim i As Long, j As Long, Length As Long
  Dim Temp As String
  Length = Len(S)
  For i = 1 To Length
    j = Int(Rnd * Length) + 1
    Temp = Mid$(S, i, 1)
    Mid$(S, i, 1) = Mid$(S, j, 1)
    Mid$(S, j, 1) = Temp
  Next
  RandomString2 = S
End Function
