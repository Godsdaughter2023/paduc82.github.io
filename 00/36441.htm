a blue illusion say:

It can get confusing in a hurry when you want to set up a certain button, or set of buttons, to do a certain thing.

First thing to know is... no, the paddles are not inherently tied to the OG buttons. You can map/remap them any way you see fit. Also, it's hard to think of it this way, but you technically have 4 'extra' buttons now if you use the paddles. It may not seem that way, since no game will use more buttons than the OG controller has available. But if you think about it, you could use the paddles as your A, B, X, Y buttons and never have to touch those buttons if you like. Something to think about that can be helpful.

Maybe it will help... hope it won't make the water murkier lol, if I share my "default" button mapping setup for FPS games.

If memory serves, the standard setup (Config 1) I think, already has A B X Y mapped to the paddles. I typically leave them the way they are as long as the game has a controller setting that allows you to assign B to melee. I hate clicking the thumb sticks, so I map the A button to the L thumb stick click and the B button to the R thumb stick click. I rarely have a problem with this... unless you get a game like Fallout 4... that uses the same thumb stick click for 2 purposes. It uses L thumb stick to sprint, but it also uses it to 'hold breath' when you're zoomed in with a sniping weapon.

What makes it so confusing is the fact that the games give you multiple controller settings... which is nothing more than a button map preset. Then you have take that... and start thinking about how to remap.

The idea that seems to work the best for me, is when I first start playing a game, I go with the default... A B X Y mapped to the paddles. When I choose a controller configuration in the game though, I don't think about the paddles at all. I choose something that I feel would work for me if I had to use the standard A B X Y buttons. That 'ensures' that I'll probably feel comfortable playing the game before I start remapping.

I think the best thing to do is come up with a basic plan, mine is just an example. Also, understanding that any button can be mapped to any other helps. You can even have buttons "unmapped" if memory serves.

OH, one thing you definitely need to think about. Be careful about remapping A and B. Even if you don't use them, make sure they're mapped somewhere... it can really get confusing if you have a really specific setup for a game and forget about needing A and B to navigate the dashboard.

b)      In the command prompt type the following command: “sfc/scannow” and press enter

c)       Now, restart the computer.

I would also suggest you to disable the secure boot and check. When your computer starts, wait for the manufacturer logo to check the option for boot menu, it will usually be any of the function key (eg: F12).
