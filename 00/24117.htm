rich michaels - mvp (greatcirclelearning.com) say:
Well you can add this macro to your Normal template and assign it a custom keyboard shortcut. It will insert a custom text watermark.

Sub AddWatermark()

    Dim scn As Word.Section, hdft As Word.HeaderFooter, shp As Word.Shape
    Dim Doc As Word.Document: Set Doc = ActiveDocument
    Dim rng As Word.Range
    Dim sMark As String
    
    sMark = InputBox("Enter Your Watermark Text", "Watermark Maker")
    If sMark = "" Then Exit Sub
    
    On Error Resume Next
    With Doc
        For Each scn In .Sections
            For Each hdft In scn.Headers
                For Each shp In hdft.Shapes
                    If shp.TextEffect.Text = sMark Then shp.Delete
                Next
            Next
        Next
    End With

    With Doc
        Set rng = Doc.Content
        For Each scn In rng.Sections
            For Each hdft In scn.Headers
                hdft.Range.Select
                Set shp = hdft.Shapes.AddTextEffect(msoTextEffect2, sMark, "Tahoma", 10, False, False, 0, 0)
                With shp
                    .Line.Visible = False
                    With .TextEffect
                        .NormalizedHeight = False
                        .FontItalic = False
                        .FontBold = True
                    End With
                    With .Fill
                        .Visible = True
                        .Solid
                        .ForeColor.RGB = 12632256
                        .Transparency = 0.5
                    End With
                    .Rotation = 315
                    .LockAspectRatio = True
                    .Height = Word.InchesToPoints(1.96)
                    .Width = Word.InchesToPoints(7.2)
                    With .WrapFormat
                        .AllowOverlap = True
                        .Side = Word.WdWrapSideType.wdWrapBoth
                        .Type = 3
                    End With
                    .RelativeHorizontalPosition = Word.WdRelativeHorizontalPosition.wdRelativeHorizontalPositionMargin
                    .RelativeVerticalPosition = Word.WdRelativeVerticalPosition.wdRelativeVerticalPositionMargin
                    .left = Word.WdShapePosition.wdShapeCenter
                    .top = Word.WdShapePosition.wdShapeCenter
                End With
                hdft.Range.Collapse (Word.WdCollapseDirection.wdCollapseEnd)
            Next hdft
        Next scn
    End With
    rng.Select
    Selection.Collapse Word.WdCollapseDirection.wdCollapseStart
End Sub

Your formula

=INDEX(Rates!B:B,MATCH($V3,Rates!$A:$A,0))

basically says Return the value from B on the same row where the value in A exactly matches the value in V3.

So, if V3 is a percentage, it cannot match the strings in column A.  Given the values that you have, it also would not match anything in column B exactly.

scottgem say:

I just did something similar for a client but the VBA is a bit complex. . First, don't use the default name of ID for your PK fields. I always name my PKs tablenameID. Second, Forms are not designed for printing, which is why you get all those recommendations for reports. Reports are designed for printing. Third, if you want the contents of the Form to appear in the body of the e-mail that also requires complex coding.  But I think we can simplify this for you. But there are a few steps you need to take

Step 1: Create a query with the data you want to send. In the criteria for the ID field put:

=Forms![Form Shipping Label]!LabelID

where LabelID is the name of the control on the form bound to your primary key field.

Step 2: Using that query, design a report with the data and layout you want to send

Step 3: Add a button to your form. Use the Code Builder for the On Click event to add this line of code:

Dim strTo As String, strSubject As String, strBody As String

strTo = email address of recipient

strSubject = "the text you want to appear in the subject of the e-mail"

strBody = "the text you want to appear in the body of the e-mail"

DoCmd.SendObject acReport, "reportname", acFormatPDF, strTo, ,,strSubject,strBody,False

When you click the button it will send an e-mail with a PDF of the report attached.

Explanations:

The query created in Step 1 will return a record filtered for the PK on the form. Therefore, the report will only display that one record. 

You need to supply the e-mail address of the recipient. You haven't detailed where that comes from. Whether its a value on the form, hard coded or what I don't know so I can't tell you what should go there. Similarly for the Subject and Body. I don't know if you want to make those dynamic to include data from the record or what. 

The SendObject method will generate an e-mail attaching a PDF file of the report and sending it whatever e-mail address you supply with the subject and body you want. 

One last point. The False value at the end sends the e-mail automatically without the user seeing it. if you want to view the e-mail first so you can maybe add something you can change that to True.

Sorry, but no one can provide a "simple cut and paste" without more specific info from you. But you shouldn't expect us to write your code for you.
