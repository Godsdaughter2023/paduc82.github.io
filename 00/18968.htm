ken sheridan say: My usual approach to this sort of thing would be to put the bound controls in a multi-column subreport and the labels in the parent report immediately to the left of the subreport.  Then you don't need any code to hide or show the controls.

ken sheridan say: Are you saying that the subreport returns 6 records in total, i.e. there is just one set of 6 columns of data?  If so that differs from my demo, which returns 12 records, i.e. 3 sets of 4 columns of data with the labels on the left repeating 3 times.  With only one set of 6 records in 6 columns then it makes it very much easier to do as the parent report does not need to be bound to anything.  From your description it sounds like both the parent report and subreport are bound to a table or query which returns the 6 records, so the parent report is printing 6 details, with a separate identical instance of the subreport in each.
