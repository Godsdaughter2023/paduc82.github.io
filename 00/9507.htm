tom van stiphout say:

The query for a combobox can have multiple columns, in your case:
select [primary key], business, calc_value from business

Here we see a problem: it's a Really Bad Idea to have a field name be the same as a table name.

 

You use this query for the RowSource of your combobox, and set the Column Count to 3, and perhaps the Column Widths to "0;1;0"

 

Then the ControlSource for your textbox is:

=YourComboboxName.Column(2)

xaviercriel say:

How can I check what fonts are used in a Word 2011 document?

Hi,
My Word document should only have Arial fonts. However my client is claiming it is sometimes using another font. How can I check if any other fonts are being used in the document (and then change them to Arial). I am using Word 2011, my client is using Word 2003 on windows.

thanks for a quick reply.
kind regards,

daniel g. grau say:
I know of no way to detect such instances.

Start by checking your own computer, as you may have inadvertently typed a non-existing font name somewhere. Go to Word > Preferences > Compatibility and press the Font Substitution button to see if that's the source of the problem. See below how to find it.

If that fails, the problems is with a font that your Mac has and your client does not. This could be a solution:

  • Do Edit > Select All (or Command A)
  • Apply Arial to the whole document
  • Do this with headers and footers as well

Despite the above, there are places that text in a non-Arial font can hide, such as an out-of-margins Text Box, with or without text, that would not be visible yet trigger a missing font message. In such a case, in order to pinpoint it, you'll need to know what's the name of the non-Arial font is. In Word 2003, they can do this through Tools > Options > Compatibility and pressing the Font Substitution button to see a list of non-available fonts.

Once you have this list, look for those font(s) on the Mac with Edit > Find > Advanced Find and Replace. (You should see a Format button; if you don't, click on the disclose triangle at the bottom left.) Choose Format > Font in the dropdown menu, type the name of the font and look for it. Even if Word doesn't show it, as long as it doesn't tell you the font was not found, it's in there somewhere.

As mentioned, a likely place is an out-of-margin text box. > Is there a difference between the "Custom XML" features and basic XML operability?

Word has (or had) a number of XML-related features, and distinguishing one from the other can be confusing. The main one is that you can save any Word document in an XML-based format. This is WordprocessingML, or later (I think) WordML. When you save in .xml (Word 2003), .xml (Word 2007 and later), .docx, docm formats, you are saving in XML formats, except that .docx/.docm are ZIP files containing XML format parts and potentially other parts such as image files.

The Custom XML features are the ones that allow you to attach an XML schema to a document, then insert XML elements from the schema (as in your document). When you save such a document, you can either save it as Word format file with all the Word formatting and XML data, or you can save as an XML data file, in which case all the Word formatting and layout is lost, and what you are left with is the data you typed between the XML element tags, and the tags. When saving as a Word format file, you can even save in the old non-XML .doc format. When saving as "data only" you would typically save as a .xml file.

The other main XML-related feature in Word 2007 and 2010 are content controls, which can be connected to chunks of XML data stored in a "Custom XML data store" that is stored as part of a .docx file. SO there are even two different things called Custom XML data.

I assume that the type of Custom XML you have put in your test file is the one you need to use. But if you do not really have to use it, it's advisable to find some other way to do what you need because the feature has disappeared altogether in later versions of Word (i.e. 2010). (At one point I think Microsoft was trying to keep its options open in case it managed to overturn the court ruling, but that didn't happen. I'm not in a position to know Microsoft's plans but have never heard any suggestion that this feature will be re-introduced_.
