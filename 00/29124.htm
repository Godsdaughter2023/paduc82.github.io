I went into Windows update history, and rolled back the updates installed on 5/20--this did NOT fix the problem.

I uninstalled, then reinstalled the ASIO driver provided for Windows 10 from the TASCAM website. This did have an impact, but, did not fix the problem. The difference after the re-install of the driver was that the playback head in REAPER DOES move, and, in the mixing console, the lighted bars that measure the peaks of the music being played behaved like it was reading the music file. By all appearances, Reaper seems to be playing an audio file. 

However, there is NO output signal from the TASCAM-1200--and, if i insert headphones into the computer, I DO NOT hear any output from Reaper.  Point here is that it appears the Reaper Software recognizes the ASIO driver and is playing the music file,  but the software isn't sending a signal to the TASCAM 1200 interface--or to the internal computer sound system via the headphone jack.

However, audio from the computer (e.g., a Youtube video, or using Media player to play an .mp3 file) IS available thru the headphone jack on the computer; it IS NOT not available on the TASCAM 1200, however. In other words, I can hear a YouTube video fine, via the headphones--but if I set Reaper to record the signal--it DOES NOT have any signal to record..

Moved from: Windows / Windows 10 / Windows update, recovery, & backup

In terms of setup: when I go to Sounds, the Tascam 1200 is shown as the Speakers device (as it always has been). Also, Reaper clearly sees the Tascam 1200 when I look in its Preferences settings. 


Does anyone have any clue what could be wrong, or how a Windows Update could screw things up so bad--let alone what to do now?  This has cost me hours and hours of troubleshooting, and it is just unacceptable to have such a disastrous result from and automatic updated.  And yes, I am 100% certain NOTHING changed between my last successful use of the Tascam 1200 and Reaper, and the failure--the ONLY change was the windows update.

derwoodkirwood say:

Thanks for the reply.

I did resolve this--so here is what happened, in case anyone else has a similar issue.

The Windows update seems to have corrupted the Tascam driver for the USB connector.

This issue manifested in the behavior described in my original post about the playback head in Reaper not moving, or giving any audio output. 

Re-installing the Tascam driver fixed that behavior--as per my note, Reaper appeared to be reading audio, but there was no sound output.  I did not understand it at first, but once I had re-installed the driver, I had fixed the problem associated with the Windows Update. 

The lack of output had nothing to do with Windows update. The new driver install reset a bunch of config stuff on the Tascam-supplied software that manages the audio interface. That was why Reaper seemed to be playing audio, but I was unable to hear anything--I had to fix some of the config settings. 

So, if you have a driver-related issue, remember that if you re-install a driver, you should carefully check any config settings for the device in question. 

Thanks for the help
