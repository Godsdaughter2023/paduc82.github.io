scottgem say:

I'll bet if you look at the RowSource of the Combobox it looks like this:

SELECT ID, LastName

FROM Table

And I'll also bet the Bound Column property for the combo is 1. 

You probably used the combobox wizard to create the combo. By default that wizard, includes the key column as a hidden column and makes it the bound column. So when you select Larson, the combo returns 3. And 3 is not in the Lastname column so no records are returned.

So, my question is are you trying to search for all employees with the same last name or select a specific employee. What you need to do is different depending on how you answer that question. 

If you are searching for all employees with the same last name, then change the criteria in your query to:

[Forms]![nameForm]![cboLastName].Column(1)

This will match against what is in the second column of the RowSource (The column count for this purpose starts at 0). Or you can change the Rowsource of the combo to eliminate the ID column. This will also mean changing the Column Count and Column Widths properties.

If you want to search for a specific employee then I would change the RowSource to:

SELECT ID, Lastname & ", " & Firstname As Employee

FROM tablename

ORDER BY Lastname, Firstname;

Then move the criteria from the Lastname column to the ID column.

By the way, I would not accept the default name for the ID column. I always mane mine as tablenameID.
