suzanne s. barnhill say:
To add to what Hans has said, if you are actually creating a template (that is, you have the template open for editing), then the setting really makes no difference: when you change the styles in the template, you have effectively changed them for new documents based on the template.

hansv mvp say:

Selecting "New documents based on this template" is always a one-time action. It lets you save the current changes to a style to the template.

Having changes to a style in a document always automatically affect the style in the template could have undesired or at least unexpected consequences, so you have to set this option explicitly each time you want to update the style in the template.

 

If you tick "Automatically update", any direct formatting changes that you apply to a paragraph with that style will be applied to the style itself, so that other paragraphs with that style are affected too.

For example, if you tick "Automatically update" for the Heading 1 style, then select a Heading 1 paragraph and center it, center alignment will become part of the style definition, and all Heading 1 paragraphs will be centered.

In most situations this is not desirable.

ossiemac say:

Following suggestion is to use VBA code. It adds a worksheet for the output and selects the output worksheet.

You will see comments with optional methods of displaying the address. Only use one of the lines and the other to be commented out with the leading single quote.

I have included an extra column to the output for the worksheet name for the named range.

If you have any defined names that are not referring to ranges then there will be minimal output for them.

Sub ListDefinedNames()
    Dim wsOutput As Worksheet
    Dim nme As Name
    Dim rngDestin As Range
   
    Set wsOutput = Sheets.Add(After:=Worksheets(Sheets.Count))
   
    With wsOutput
        .Cells(1, "A") = "Name"
        .Cells(1, "B") = "WorksSheet"
        .Cells(1, "C") = "Address"
        .Cells(1, "D") = "Value"
        .Range(.Cells(1, "A"), .Cells(1, "D")).Font.Bold = True
    End With
   
    With ThisWorkbook
        For Each nme In .Names
            With wsOutput
                Set rngDestin = .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0)
            End With
            On Error Resume Next
            rngDestin = nme.Name
            rngDestin.Offset(0, 1) = Range(nme.Name).Worksheet.Name
           
            'Option 1 for output as Absolute address (Includes $ signs)
            rngDestin.Offset(0, 2) = Range(nme.Name).Address
           
            'Option 2 for output as Relative address (Excludes $ signs) 'Optional
            'rngDestin.Offset(0, 2) = Range(nme.Name).Address(0, 0)
           
            rngDestin.Offset(0, 3) = nme.Value
            On Error GoTo 0
        Next nme
    End With
     wsOutput.Columns.AutoFit
 End Sub

Guidelines to install the VBA code if required.

  1. Alt and F11 to open the VBA editor window
  2. In the VBA editor select menu item Insert -> Module
  3. Copy the VBA code above and paste into the VBA editor.
  4. Close the VBA editor (Cross very top right of VBA editor window)
  5. Save the workbook as Macro enabled.
  6. Ensure macros are enabled. See help for how to do this. (Option to "Disable all macros with notification" should be OK.)
  7. If Developer ribbon not displaying then right click anywhere in one of the ribbons and select "Customize the ribbon". On the right side of the dialog check the box against "Developer" and click OK. (Don't forget to click OK; Don't just close the dialog.)
  8. To run the macro, on the Developer ribbon select Macro icon (towards left in Code block) and then select the macro name (InsertLines) and click Run.
