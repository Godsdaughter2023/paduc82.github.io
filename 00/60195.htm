a. user say:

86 and still active in the field - bravo! I hope to be so when I turn 86.

The best way to make a backup image of your hard drive, by far, is to use third-party backup image software.

Windows' own System Image Builder was been deprecated with the Fall Creators Update (v.1709) - which is Microsoft's way of saying 'abandoned.' It's no longer being developed and might be removed by a future update. However, even at its best, SIB is no match for third-party image backup software. Rather, it's like a lot of other utilities that Microsoft adds in to Windows for no extra cost - like Fax and Scan, Voice Recorder, Snipping Tool, etc. - which provide a basic level of functionality for people who don't want to get full-out third-party programs. Like them, System Image Builder is not nearly as flexible or convenient as a real image backup program.

Backup and Restore (Windows 7) was included in Windows 10 for people who used that to make backups in Windows 7, so that they can restore those backups in Windows 10. It's not supported for making new backups, although people try that anyway with varying degrees of success.

Microsoft's other backup program - File History - doesn't make images.

If you want the real deal in image backup software - the bees knees - you want a third-party product. You can find them on the internet; just search for

windows image backup software 2018

and your search results will include reviews of popular image backup software programs. Trust the reviews by the professionals. Take ad vantage of the free trials and pick one that appeals to you. My own choice is Macrium Reflect (which also has a free version) but there are plenty of other good choices.

An image backup is an exact copy of a drive letter, including the metadata which describes the drive (and isn't viewable from File Explorer because it's not contained in a file.) It's the ideal way to backup your entire C: drive in one shot. It's so easy to do, once you learn how, you'll find it convenient to make backups every day.

gerry c j cornell say:

Daavee is going to tell you the software route. There is another way.

What you can do is swap the memory sticks in and out. Thus if your memory consists of two memory sticks you remove one from the slot in the motherboard and test whether the errors you were experiencing occur. After a period of testing you remove the memory stick in use and restore the one you removed and run the computer. It is vital that the computer is switched off whilst installing and removing memory sticks. This method is commonly used by commercial repairers, except that they install new memory and remove all existing memory. If the system still fails they restore the original memory and look elsewhere for a solution

You should check each memorystick with the requirements of the motherboard manufacturer.

Using memory testing software can prove memory is defective. It does not prove memory is not defective. The latter is frequently overlooked.

a. user say:

Another option:

Run a test of your RAM with memtest86+ (I know it is boring and will cost you a CD).

Memtest86+ is a more up to date version of the old memtest86 program and they are not the same.

The memtest86+ will not run under Windows, so you will need to download the ISO file and create a bootable CD, boot on that and then run the memtest86+ program.  

If even a single error is reported that is a failure and should make you suspicious of your RAM.

If you have multiple sticks of RAM you may need to run the test on them one at a time and change them out to isolate the failure to a particular single stick.

Always keep at least the first bank of RAM occupied so the test will find something to do and there is enough to boot your system.

Sometimes, reseating the RAM in the slots will relieve the error but a failure is still cause for suspicion.

The file and instructions are here:

http://www.memtest.org/

Here is a link that shows you have to create and use the memtest86+ CD:

http://www.geekstogo.com/forum/Guide-to-using-Memtest86-t246994.html

If someone says to run memtest86, you can tell them to go pound sand and that you know memtest86+ supercedes memtest86 and here's why:  

http://en.wikipedia.org/wiki/Memtest86
