I had the same problem, and found the solution:

When you open an Excel 2013 file, then two windows appear (your file, and a blank file)

The trick is to drag-resize the blank file (not the opened file).

So do this:

1) Open an Excel file,

2) Close it

3) Click on Excel that is still open (should be a small tab)

4) Resize it to a viewable size

5) Close it

After you done that, you can repeat the steps, but instead of resizing it, you can maximise the empty tab, then close it.

Now, if you open an Excel file, it will open maximised, and if you "Restore Down", then it will be in a viewable size (not a mini tab)

jim thomlinson say: If you have any volatile functions in your spreadsheet then they will update when you open the workbook. A good example would be =Now() or =Today() which return the date or date and time. An updated formula is a change and you will be prompted to save. There is no reason to save the book in this case as the function is constantly being updated whenever you open the book or a calculation runs.

joeu2004 say: Jim wrote:
If you have any volatile functions in your spreadsheet then they will update when you open the workbook. A good example would be =Now() or =Today() which return the date or date and time.

Of course, Kim asserted at the outset that the files "have no field that would automatically change such as today's date".

But there are other functions that are volatile, yet their result does not change unless other data changes.  Examples:  OFFSET and INDIRECT.  These will cause a "save or don't save" prompt even if we merely open, then immediately close the file.

Moreover, I have found that some INDEX expressions cause the same behavior.  INDEX is not a volatile function.  But, for example, a reference of the form INDEX(B:B,C1):INDEX(B:B,C1) causes "save or don't save" prompt if we merely open, then close the file.

And I believe that some other objects cause the same behavior, even though they are not changed.  IIRC, I have encounted such problems with charts -- or at least some charts.  And I believe I read that tables (notably pivot tables) cause this problem.  (Unverified.)

doug robbins - mvp office apps & services (word) say:

In the Table of Contents dialog, click on the Modify button and then in the Style dialog, click on the style for the TOC level that you want to modify and then click on Modify and in the Modify Style dialog, make the change to the font that you want.

Note, the change will apply to all of the entries of that level in the Table of Contents.  The only way you could have the change apply to individual entries is to select the Table of Contents and convert it to ordinary text by pressing CTRL+SHIFT+F9 and then apply the formatting directly to the individual entries.
