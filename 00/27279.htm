Access 2007- Print duplex - Microsoft Community

accountingdepartmentwwc say:

Access 2007- Print duplex

Using a report to generate year end receipts by account. Report currently prints each account on a new page. Line items in each account can vary (as few as one page or as many as several pages). I would like to print duplex but don't know how to be sure each account starts on a new sheet/page. (Fred's account can't begin to print on the back of Mary's.) Boils down to multiply pages possible in the "group by" and duplex printing. 

***Post moved by the moderator to the appropriate forum category.***

ken sheridan say: For this you need to include an account group header in the report, with its RepeatSection property set to False (No).  At the very top of the group header add a page break control,  In the group header's Format event procedure out code like this:

    [YourPageBreakControl].Visible = (Page Mod 2 = 0)

If the page on which the group starts is at an even number a page break will be inserted causing a blank page to print and the group to start on the next page, thus ensuring that each group starts on an odd numbered page.

Note that the 'blank' pages inserted to achieve this will still show any controls in the page header and/or page footer.  To avoid the page header controls showing on the blank pages, two account groups, each with a group header, should be created.  Put only the page break control in the first, and put the code in this section's Format event procedure.  Move the page header controls to the second group header, and in the same section put any group header controls you wish to repeat on subsequent pages of the group below these.  Set this section's RepeatSection property to True (Yes).

If you have group header controls wish you wish to show only on the first page of a group, you can put these in a third account group header and set its RepeatSection property to False (No).  These will be printed below the repeating header controls on the first page of the group only.

Any page footer controls will still print on the blank page, but as these are usually things like page numbers, you might well want this to be the case.

jllatham say:

You should have a new workbook in hand by now.  If you don't let me know.  The code for the Workbook_Open() event is shown below.  I added a few features to it that I hope you find helpful.  I would post a screen shot here, but that might expose information you don't want seen.  Basically the changes are geared to make the output more readable for you.

The problem we had with dates earlier wasn't a problem with the dates - it was a problem with me not having knowledge of how the data sheet was laid out - because column A didn't have entries for each row, the data on the 'report' sheet was often getting overwritten making it look like we weren't dealing with dates correctly.  Once I made allowances for that, everything went quite well.

  'note that these have been moved to a position
  'before the first Sub() declaration
  'so that all code in this module can reference them
  '
  'set this to the name of the sheet with dates to check
  Private Const listWSName = "expire" ' change for your workbook
  Private Const contractorColumn = "A"
  Private Const workersColumn = "B"
  'set this to the column with the dates to test/examine on that sheet
  Private Const firstDateColumn = "C" ' change to match your worksheet
  Private Const lastDateColumn = "T"  ' change to match your worksheet
  'set this to the first row on the sheet that has an entry to test
  'this assumes row 1 has labels in it
  Private Const firstDateRow = 2
 
  'set this to the name of the sheet that will have expired
  'license information copied to - it does not have to exist,
  'if it does not, it will be created when needed
  Private Const xpiredWSName = "Expired"
  Private xpiredWS As Worksheet
  Private expirationDate As Date
  Private listWS As Worksheet
 
Sub Workbook_Open()
  'working variables
  Dim mainRowPtr As Long
  Dim mainColPtr As Long
  Dim nextXpiredRow As Long
  Dim lastMainRow As Long
  Dim lastMainColumn As Long ' last column used on main sheet
 
  Set listWS = ThisWorkbook.Worksheets(listWSName)
  'find last used row on the main sheet for use later
  On Error Resume Next
  lastMainRow = listWS.Cells.Find(What:="*", _
   After:=listWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByRows, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Row
  On Error GoTo 0
  'find last used column on the main sheet
  'to keep copy actions to minimum
  On Error Resume Next
  lastMainColumn = listWS.Cells.Find(What:="*", _
   After:=listWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByColumns, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Column
  On Error GoTo 0
 
  If lastMainRow < firstDateRow Then
    'no work to do, just quit
    Set listWS = Nothing
    Exit Sub
  End If
  'does the needed Expired sheet exist?
  'if not then create it
  On Error Resume Next
 
  Application.ScreenUpdating = False ' may improve performance
 
  Set xpiredWS = ThisWorkbook.Worksheets(xpiredWSName)
  If Err <> 0 Then
    'sheet didn't exist, create it
    ThisWorkbook.Worksheets.Add After:=ThisWorkbook.Worksheets(ThisWorkbook.Worksheets.Count)
    Set xpiredWS = ActiveSheet
    xpiredWS.Name = xpiredWSName
    'copy the labels from listWSName onto the new sheet
    'and initialize their widths
    listWS.Rows("1:1").Copy
    With xpiredWS
      .Range("A1").PasteSpecial xlPasteAll
      .Range(.Cells(1, 1), .Cells(1, lastMainColumn)).EntireColumn.AutoFit
    End With
    Application.CutCopyMode = False
    Err.Clear
  End If
  On Error GoTo 0
  'set expiration date to 7 days prior to today
  expirationDate = DateSerial(Year(Now()), Month(Now()), Day(Now())) + 7
  'put a date time marker on the xpiredWS sheet
  'to help with cleanup use nextXpiredRow for this
  On Error Resume Next
  nextXpiredRow = xpiredWS.Cells.Find(What:="*", _
   After:=xpiredWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByRows, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Row + 1
  On Error GoTo 0
  If nextXpiredRow < 2 Then
    nextXpiredRow = 2
  End If
  With xpiredWS.Cells(nextXpiredRow, 1)
    .NumberFormat = "mmm dd, yyyy hh:mm:ss"
    .Value = Now()
    .EntireColumn.AutoFit
  End With
 
  With listWS
    'test all rows on main sheet from 2 through last used
    For mainRowPtr = firstDateRow To lastMainRow
      'test all columns on main sheet from B thru T
      For mainColPtr = .Range(firstDateColumn & 1).Column _
       To .Range(lastDateColumn & 1).Column
        'first test, verify it could be a date
        If IsDate(.Cells(mainRowPtr, mainColPtr)) Then
          'now, is it beyond the expiration date?
          If .Cells(mainRowPtr, mainColPtr) < expirationDate Then
            'yes it is, copy it to the other sheet
            'find next available row on the Expired sheet
            On Error Resume Next
            nextXpiredRow = xpiredWS.Cells.Find(What:="*", _
             After:=xpiredWS.Cells(1), _
             Lookat:=xlPart, _
             LookIn:=xlFormulas, _
             SearchOrder:=xlByRows, _
             SearchDirection:=xlPrevious, _
             MatchCase:=False).Row + 1
            On Error GoTo 0
            If nextXpiredRow < 2 Then
              nextXpiredRow = 2
            End If
            .Range(.Cells(mainRowPtr, 1), .Cells(mainRowPtr, lastMainColumn)).Copy
            xpiredWS.Range("A" & nextXpiredRow).PasteSpecial xlPasteValues
            xpiredWS.Range("A" & nextXpiredRow).PasteSpecial xlPasteFormats
            Exit For ' we can leave off testing any more cells on this row
          End If
        End If
      Next ' end mainColPtr loop
    Next ' end mainRowPtr loop
  End With
  Application.CutCopyMode = False
  'mark the pending expiration dates on the Expired sheet
  MarkXpiredSheet ' will use xpiredWS in its processing
  'all done, do some housekeeping cleanup
  'to return assigned resources to the system
  '
  'ensure that a worker's name has proper company associated with it
  '
  AlignNamesAndCompanies
  'get rid of left over conditional formatting
  'so that our +7 day alerts are visible
  xpiredWS.Cells.FormatConditions.Delete

  Set listWS = Nothing
End Sub
Private Sub MarkXpiredSheet()
'
'this will shade 'trigger' dates yellow
'it initially sets them to no shading at all
'and shades them yellow with appropriate
'
'dates that have actually expired
'are flagged as red
'
  Dim lastRow As Long
  Dim lastCol As Long
  Dim firstDateCol As Integer
  Dim lastDateCol As Integer
  Dim rowPtr As Long
  Dim colPtr As Long
  Dim currentDate As Date

  On Error Resume Next
  lastRow = xpiredWS.Cells.Find(What:="*", _
   After:=xpiredWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByRows, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Row
  On Error GoTo 0
  If lastRow < 2 Then
    Exit Sub ' no work to do, just quit
  End If
  'find last used column on the expired sheet
  On Error Resume Next
  lastCol = xpiredWS.Cells.Find(What:="*", _
   After:=xpiredWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByColumns, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Column
  On Error GoTo 0
  firstDateCol = Range(firstDateColumn & 1).Column
  lastDateCol = Range(lastDateColumn & 1).Column
  Application.ScreenUpdating = False
  currentDate = DateSerial(Year(Now()), Month(Now()), Day(Now()))
  With xpiredWS
    For rowPtr = 2 To lastRow
      For colPtr = firstDateCol To lastDateCol
        .Cells(rowPtr, colPtr).EntireColumn.AutoFit
        If Not IsEmpty(.Cells(rowPtr, colPtr)) Then
          If IsDate(.Cells(rowPtr, colPtr)) Then
            'default to not a 'trigger' date
            .Cells(rowPtr, colPtr).Interior.Color = xlNone
            If .Cells(rowPtr, colPtr) < expirationDate Then
              .Cells(rowPtr, colPtr).Interior.Color = vbYellow
            End If
            'if a license actually expired
            'flag it as red
            If .Cells(rowPtr, colPtr) < currentDate Then
              .Cells(rowPtr, colPtr).Interior.Color = vbRed
            End If
          End If ' end test for a date in the cell
        End If ' end of test for an empty cell
      Next ' end colPtr loop
    Next ' end rowPtr loop
  End With
End Sub

  ' listWS and xpiredWS will still point to proper sheets
  ' when this process is called
  Dim xpiredNamesList As Range
  Dim anyXpiredName As Range
  Dim lastMainRow As Long
  Dim rowPtr1 As Long
  Dim rowPtr2 As Long
  Dim doneFlag As Boolean
 
  'we have to look at each name on the xpiredWS (column B)
  'find that name on the listWS (column B) and then
  'find the associated company name on the listWS in column A
  'and hook them up here.
  '
  'find last possible row in use on listWS
  On Error Resume Next
  lastMainRow = listWS.Cells.Find(What:="*", _
   After:=listWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByRows, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Row
  On Error GoTo 0
  'find last possible row in use on xpiredWS
  On Error Resume Next
  rowPtr1 = xpiredWS.Cells.Find(What:="*", _
   After:=xpiredWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByRows, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Row
  On Error GoTo 0
  Set xpiredNamesList = xpiredWS.Range(workersColumn & firstDateRow & ":" & _
   workersColumn & rowPtr1)
  'now work through each name and get company names
  For Each anyXpiredName In xpiredNamesList
    'the list can have empty cells in it
    doneFlag = False
    If Not IsEmpty(anyXpiredName) Then
      'find matching name on listWS
      For rowPtr1 = lastMainRow To firstDateRow Step -1
        If listWS.Cells(rowPtr1, workersColumn) = anyXpiredName Then
          For rowPtr2 = rowPtr1 To firstDateRow Step -1
            If Not IsEmpty(listWS.Cells(rowPtr2, contractorColumn)) Then
              'copy the company name!
              xpiredWS.Cells(anyXpiredName.Row, contractorColumn) = _
               listWS.Cells(rowPtr2, contractorColumn)
              doneFlag = True
              Exit For
            End If
          Next ' end rowPtr2 loop
          If doneFlag Then
            Exit For ' move on to next name
          End If
        End If
      Next ' end rowPtr1 loop
    End If
  Next
End Sub

I do want to return to your first post and more directly address a couple of questions you asked about:

"The file may leave the organisation and then come back later - will opening / closing with macros strip out any coding? I think it simply gets "switched off" but cannot remember"

If a workbook contains macros, simply disabling them when the file is opened does not remove them - they just won't be available for use.  But if you want to "strip" the VBA code out of a .xlsm workbook, you can save it as a .xlsx file and the code will actually physically be removed from that copy of the workbook.

"Would something as simple as a For... ...Next loop work? I say 'simple' but I am not sure I could code it in VB - would the count default to zero every time the file is opened?"

I'm not sure how you'd planned to use the For ... Next loop, but.  In VB when you run a macro, all variables are set to a "never been here before" state:  numeric variables become zero; string variables become = "" (empty string); boolean values become False; etc.  VB does not have a way to store a value within the VB code itself from one use of the workbook to the next -- that is, once you close a workbook and reopen it later, everything in VB starts from scratch; only Const values (which by definition cannot be changed in the code) retain their defined values.  This is why if you want to "remember" anything between workbook usage you have to use some other way of remembering it such as storing the information on a worksheet, or (as Rick Rothstein suggested) creating a custom property, or defining a Name in the workbook and assigning it the value you wish to remember, or some other similar method that results in a persistent record of the data.

BTW: Rick mentioned this:

' Elsewhere in your code, you can retrieve this directory this way
Debug.Print ThisWorkbook.CustomDocumentProperties("PrimaryDirectory")

I'm sure that was a minor slip of the pen on his part.  Debug.Print simply prints the value into the Immediate Window of the VB Editor and those values are not available for use by the code.  It's a way of showing you what the value is.  In the VBA code you would do something like this:

Dim previousPath As String

previousPath = ThisWorkbook.CustomDocumentProperties("PrimaryDirectory")
