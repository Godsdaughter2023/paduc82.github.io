How to Change Wireless Channel - Microsoft Community

xb1r hades say:

How to Change Wireless Channel

Login Information

  • Default Gateway - 192.168.0.1
  • Username - user
  • Password - user

Older Firmware

How to Change Wireless Channel

  1. Open a web browser.
  2. Enter http://192.168.0.1 in the address bar.
  3. Select login from the side menu.
  4. Enter the username and password. (user/user)
  5. Select Wireless from the menu at the top.
  6. Select Radio from the menu on the left.
  7. From the Control Channel drop down select the desired wireless channel. (Channels 1,3,9,11)
  8. Select Apply.

Newer Firmware

How to Change Wireless Channel

  1. Open a web browser.
  2. Enter http://192.168.100.1 in the address bar.
  3. When prompted enter user for the username and user for the password.
  4. Select Wireless from the menu at the top.
  5. Select Radio from the menu on the left.
  6. From the Control Channel drop down select the desired wireless channel. (Channels 1,3,9,11)  The device should be able to tell you if there is interference on the particular channel you select so try to select a channel where the level is acceptable as listed by the device.
  7. Select Apply.

Boot (BSOD) errors 0xC000000F and 0xc0000225 on encrypted - Microsoft Community

tonka nl say:

Boot (BSOD) errors 0xC000000F and 0xc0000225 on encrypted drives (solved)

Hi fellow readers,

I recently encountered a BSOD boot error on my (Veracrypt) encrypted laptop.

The initial error code I got was 0xc000000f.

Some options were also shown:

Press Enter to try again, (Which does nothing)

Press f8 for Startup Settings (Which does nothing)

and an option to go into bootmenu/bios.

Sometimes after hitting f8 I got the error message/code:

File: \windows\system32\boot\winload.exe
Error code: 0xc0000225

Error code: 0xc0000225

The solution: change the boot sequence/manager in the bios, or through the boot manager (f12 on my Acer)

I don’t know how, but after an Windows update the sequence was changed to boot the hdd first, instead of the Veracrypt bootloader. So the winload file (and other associated files or processes) were unreadable because of the encryption. 

When the Veracrypt bootloader is chosen as the first boot option in the bootmanager, or the win bootmanager is set as the first boot option in the bios, the computer will boot again normally.

cheers.
