jay freedman say:

The proper way to handle forms is to make a template (*.dotx) file, and require users to make a new document based on the template by using the File > New command. The completed document will be saved as a separate *.docx file, and the template will remain unchanged.

You (the author of the form) can open the current (document) form and use Save As, changing the "Save As Type" setting in the dialog to save a .dotx file instead of a .docx. That doesn't involve any revision of the content.

The solution to the issue of typing over dropdowns depends on what kind of dropdowns you used. In Word 2010 you have (in the Developer tab) the choice of a dropdown list content control, a dropdown legacy form field, or an ActiveX combo box.

If you use a content control, you can open its Properties dialog and check the box for "Content control cannot be deleted" to prevent overwriting it.

If you use a legacy form field, the template (and thus the documents based on it) must be protected for "filling in forms" in the Restrict Editing pane in order to make any form fields work. When that protection is in force, it's impossible to overwrite form fields.

I would recommend against using ActiveX controls, which require macro programming to operate. Also, documents that contain many of these controls become slow and hard to work with.

graham mayor say:

It is perhaps worth bearing in mind that forms with content controls may also be protected for forms to limit text entry to the fields themselves.

http://www.gmayor.com/ExtractDataFromForms.htm includes a function to convert legacy form fields to content controls, a main benefit of which is that they can be processed by VBA much faster than legacy fields, which may or may not be important when you are collating the data from the forms.

There's another method, that takes more setting up, but is perhaps better suited to lay users, prone to screwing things up, and that is to create a VBA userform to collect the data, write it to the fields of the document and then save with whatever name and location you wish. This can work with a template or a macro enabled document, though as Jay indicated a template is the preferred option. The users need have no access to the document.
