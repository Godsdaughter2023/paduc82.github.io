Sub ScratchMacro()
'A basic Word macro coded by Greg Maxey
Dim oTbl As Table, lngIndex As Long, lngTick As Long, lngCross As Long
Dim lngMark As Long
Dim varTMs, varCMs
Dim arrMarks() As String
  Set oTbl = Selection.Tables(1)
  lngMark = 0
  With oTbl
    For lngIndex = 2 To .Rows.Count - 1
     Select Case Left(.Cell(lngIndex, 7).Range.Text, 1)
       Case "p"
         ReDim Preserve arrMarks(lngMark)
         arrMarks(lngMark) = "p"
         lngMark = lngMark + 1
         'Or
         lngTick = lngTick + 1
       Case "o"
         ReDim Preserve arrMarks(lngMark)
         arrMarks(lngMark) = "o"
         lngMark = lngMark + 1
         'Or
         lngCross = lngCross + 1
     End Select
   Next lngIndex
  End With
  varTMs = Filter(arrMarks, "p")
  varCMs = Filter(arrMarks, "o")
  MsgBox "There are " & UBound(varTMs) + 1 & " tick marks."
  MsgBox "There are " & UBound(varCMs) + 1 & " cross marks."
  'Or
  MsgBox "There are " & lngTick & " tick marks."
  MsgBox "There are " & lngCross & " cross marks."
lbl_Exit:
  Exit Sub
End Sub

How to capture web form data in Access 2016 - Microsoft Community

steven muller say:

How to capture web form data in Access 2016

I want to capture contact forms (name, email, address, question etc.) from potential clients on our website in Access 2016. What is the easiest way to set this up?

So a potential clients fills out a form on our website, clicks "SEND" and we would have this data immediately in our Access 2016 database. 

albert d. kallal say:

If your web site allows an external connection to the database on the web site, then you could link tables from Access to that database that resides on the website. Many a web providers don’t allow external connections to the database on the web site (because the internet is a “nasty” place). Some do however allow external connections.

However, if your web site is hosted by a server on your local network, then again you could link your Access database directly to the database that drives the web site. This would mean you have to run and setup your own web server - something smaller companies often are not willing to do.

So much will depend on what features your web hosting provides, or if you are willing to host your own web site.

Another approach is to use your existing web hosting, but have the web hosting connect to a local instance of say SQL server running on your local network. And again in this setup Access can directly link to the tables on that SQL server. I actually like this setup the most!

It also not clear if “anyone” can enter contact information, or if you plan to setup some type of “sign up” process in which those web users have to first sign up, and THEN enter data. (this would prevent hordes of people just entering junk and hitting submit).

Another really simple approach is to have the web site “generate” some txt file that you download once a day and import into Access.

So “much” of your choices will depend on your existing web provider. It also not clear what tools and web development tools your web hosting supports. And likely more important what web development tools you plan to use (or adopt). Remember web hosting is just a computer. So if your web hosting is based on Linux, then you can’t for example install + use windows systems such as vb.net code. Nor can you use SQL server – you likely wind up using MySQL and say PHP.

So what choices and approaches you use will very much depend on your existing web hosting, or what kind of web hosting you are willing to adopt. You have to adopt something of which you THEN build and develop the software that runs on that web server. As noted, this can be a windows based web site, or something else like Linux. Both choices will work just fine. So you are free to choose what web development tools suits your skill set (or the skills that you hire for this choice).

mike h.. say:

Hi,

The value to increment is in A1. The first thing to do is copy the value of the cell to increment by 1% to another; out of the way, location and then ensure it is never altered. I used D1.

The macro then looks like this

Sub add_1_Percent()
Set sht = Sheets("Sheet1") ' change to suit
If sht.Range("D1") = "" Then
sht.Range("d1") = sht.Range("a1").Value
End If
sht.Range("A1") = sht.Range("A1") + sht.Range("d1") * 0.1
End Sub
