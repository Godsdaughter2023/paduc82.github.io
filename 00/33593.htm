avarti say:

Hope this response is not too late.

The message is not an error, but rather a warning. In my case it was the result of using a drive that was once used as part of a RAID array that was motherboard based (rather than dedicated array)...

Your installer is saying 'hang on, this drive is part of a raid array, so I cannot actually tell you whats may be on it already so I will ignore it'... Windows doesn't look at the meta data so it will install over the drive regardless if it was used in a raid array or not and doesn't care.

To cut a long story short, you can tell your linux installation to remove the meta data by starting a shell and under root typingdmraid -x and then following the prompt and confirming its what you want to do then trying the installer again. Run the command from a live CD

It will hopefully work for you (and others) as it worked for me.

 

Regards
