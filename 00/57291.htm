a. user say:

An Event 41 without a BSOD isn't going to tell you much.

When Windows starts it checks to see if it was shutdown properly the last time it was running and if it wasn't you will see Event 41.  It does not tell you

If the EventData looks like yours:

<EventData>
    <Data Name="BugcheckCode">0</Data>
    <Data Name="BugcheckParameter1">0x0</Data>
    <Data Name="BugcheckParameter2">0x0</Data>
    <Data Name="BugcheckParameter3">0x0</Data>
    <Data Name="BugcheckParameter4">0x0</Data>
    <Data Name="SleepInProgress">true</Data>
    <Data Name="PowerButtonTimestamp">0</Data>
  </EventData>

There was no BSOD so something else happened but you won't be able to tell what...  power outage, reset button, thermal event, plug pulling, who knows.

If there was a BSOD the Event still might not tell you much.  That is when you need to have your crash dump file analyzed.

According to the Internet, where everything you read is true, what works for some folks with that error is manually removing a suspicious hiberfil.sys file, shutdown (not restart), power up, create a new hiberfil.sys file like this:

Open an elevated command prompt:

Click the Start orb/button and in the box enter the following:

cmd.exe

Right click cmd.exe and choose to Run as Administrator:


Enter the following command:

powercfg  -h  off


Enter 'exit' to close the Command Prompt window.

Shut down the machine (not a restart)

Boot the machine

Open an elevated command prompt and enter the following command:

powercfg   -h   on

Enter 'exit' to close the Command Prompt window.

Test hibernate through the windows > shutdown > hibernate
