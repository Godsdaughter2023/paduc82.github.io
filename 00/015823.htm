talking moose say:
Internal addresses (those you use while on your institution's network) and external addresses (those you use while at home) aren't always the same address.

Your Exchange administrator should have configured the Autodiscover service so that Outlook can tell which server to use. If he has not done that then you are probably stuck manually switching addresses when you move.

What is the address you would use while at home to connect to your Outlook Web Access (OWA) server in a web browser? Enter that into Outlook and it should work. While on your institution's network see if this address still works. If so then use it all the time.

If your Exchange administrator has partially configured Autodiscover and Outlook switches back to your internal address then you may need to disable it altogether to stop it from switching. You can copy/paste this three line AppleScript snippet into the AppleScript Editor found in /Applications/Utilities and click the Run button:

tell application "Microsoft Outlook"
set background autodiscover of Exchange account 1 to false
end tell

Hope this helps!

You're making your job harder than it needs to be!

It SOUNDS like - and I may be mistaken here - that you're trying to copy data from one table (the combo's rowsource) and store that data redundantly in a second table (the form's recordsource).
