Bigger integrals, sums, and the like - Microsoft Community

neatnit say:

Bigger integrals, sums, and the like

I use OneNote (the Windows 10 version) to take notes in class. I type equations and formulas by pressing alt+= and typing the latex-like syntax.

Today I discovered by accident that there is a way to make integrals and other symbols appear larger, and I want to understand the exact meaning of the syntax.

Normally I'd type an integral like this:

\int_1^2[space](a/b)/(c/d)[space]dx

Result:

The integral symbol stays the same size, even though its contents are tall.

Today I discovered that I could add "130" after the \int:

\int130_1^2[space](a/b)/(c/d)[space]dx

Result:

The integral symbol resizes automatically to be just bigger than its contents.

What is the meaning of the number 130? I discovered through experimentation that using 140 instead of 130 will have the limits (1 and 2 in the examples above) appear above and below the symbol, rather than as subscript and superscript. Is there a list of numbers and what they do?

I cannot find any documentation for the commands, parameters, etc. available in OneNote (and other Office) equation input. Does Microsoft have such documentation?

P.S. this would answer an older question, that I found in a web search: https://answers.microsoft.com/en-us/msoffice/forum/msoffice_onenote-mso_other-mso_2016/bigger-integral-sign-with-onenote-mathematics/2c8c5d15-66ec-41e1-bdc6-0e709863e57c

However, I can't reply to it. It's a shame, since it comes up in searches and it would be better if the answer was in there.

neatnit say:

I've discovered enough info via experimentation to answer my own question.

The number immediately following the integral (before the limits) - I'll call it the attribute - seems to be treated as a bit field.

The 3 least significant bits (values 0 to 7) seem to control whether the limits go above/below the integral or as subscript/superscript:

Values 1 make the limits go above and below the integral symbol.

Values 0, 2, 3 and 7 make the limits appear as subscript and superscript. This is also the default behavior.

Values 4 and 6 make above/below in OneNote for Windows 10 and OneNote 2016, but on the web version of OneNote they make sub/supertext.

Value 5 makes above/below on the web, but not in the apps.

The bit with significant of 128 enables the vertical auto-sizing that I was talking about. It only affects rendering in the apps, not in the web version.

All other bits up to 2048 don't seem to affect anything that I could see.

Conclusions:

If you want the limits to go above and below the integral symbol, use \int1_a^b.

If you want the integral to resize automatically to match the contents, use \int128_a^b.

If you want both, use \int129_a^b.

I still need to check how this affects other big operators like \sum, \prod and the like, but I'm not sure if I will. I'm pretty satisfied already.

Continuing on from Greg's post, the following in the Class module clsApp will modify the field created by selecting Link Here after selecting, right click and dragging to a Ref field

Public WithEvents oApp As Word.Application
Private Sub oApp_WindowSelectionChange(ByVal Sel As Selection)
Dim strCode As String
With Selection.Words(1)
    If .Fields.Count = 1 Then
        With .Fields(1)
            If .Type = wdFieldLink Then
                strCode = .Code.Text
                .Code.Text = "Ref" & Replace(Mid(strCode, InStrRev(strCode, "_") - 5), "a \r", "h")
            End If
            .Update
        End With
    End If
End With
End Sub

I would have thought that the OLE bookmarks would suffice, but the following will ask for a name for the bookmark and add it at the location of the OLE bookmark and then create a cross reference to the text of the bookmark with the name that the user supplies:

Private Sub oApp_WindowSelectionChange(ByVal Sel As Selection)
Dim strCode As String
Dim strBMName As String
Dim strNewBMName As String
With Selection.Words(1)
    If .Fields.Count > 0 Then
         With .Fields(1)
             If .Type = wdFieldLink Then
                 strCode = .Code.Text
                 strBMName = RTrim(Replace(Mid(strCode, InStrRev(strCode, "_") - 3), Chr(34) & " \a \r", ""))

                 With ActiveDocument
                    .Bookmarks.Add NewBMName, .Bookmarks(strBMName).Range
                 End With
                 .Code.Text = "Ref " & NewBMName & " \h"
             End If
             .Update
         End With
     End If
End With
End Sub

From the sounds of it, a simple index/match should achieve it for you

Assuming PR.xls (where the sheet Tracker resides) is open at the same time,
then in sheet: Tracker 2010 (in FDT.xls),
in O397: =INDEX([PR.xls]Tracker!AJ:AJ,MATCH(I397,[PR.xls]Tracker!D:D,0))
Format O397 as date

you could use this IF(ISNA expression, indicatively:
=IF(ISNA(MATCH(...)),"",INDEX(...MATCH(...)))
