What I would need to see is the SQL for your query. 

To calculate a balance as of any point in time, you would have a query that adds the credits and subtracts the debits. I don't recall how you have things setup, but that is what you need. If you have a query with a running total, it's similar to that. 

SELECT Account, Sum(Credits) As Credits, Sum(Debits) as Debits, Sum(Credits) - Sum(Debits) AS Balance

GROUP BY Account

WHERE TransactionDate < #10/1/2016#;

That should give you the balance as of 9/30/2016 for each account.

SUM(T2.Credit-T2.Debit) AS Balance,
    (SELECT SUM(T3.Credit-T3.Debit)
     FROM TransactionsCD AS T3

GROUP BY T1.CustomerID, T1.TransactionDate, T1.TransactionID, T1.Credit, T1.Debit

Saving a Backup Copy when saving an XLSX file - Office 2016 - Microsoft Community

harro53 say:

Saving a Backup Copy when saving an XLSX file - Office 2016

I wish to save a backup copy of XL files (to the same folder on C drive as the base file) when I do an update thereto and save the file.

I am not referring to Autosave. 

ossiemac say:

When you say not referring to Autosave" I am not sure if you mean the Auto Recover or AutoBackup. There is a description of AutoBackup at the following link. (Go down towards the end of the page under heading "AutoBackup")


https://www.online-tech-tips.com/ms-office-tips/excel-autorecover-and-automatic-backup/

However, the above method overwrites the previous backups and if this is not what you require and you want to keep several versions of backups then you will need to change the document to a Macro Enabled Workbook and use the Save Event VBA code below.

To install the VBA Event code:

  1. Manually use "Save As" to create a Backup with a name that will not be over written in case you have a problem with the code. (Should not be any problem but better to be safe than sorry).
  2. Firstly use Save As to save the workbook as a Macro Enabled workbook.
  3. Ensure macros are enabled. See Help for how to do this. (Option to "Disable all macros with notification" should be OK.)
  4. Alt and F11 to open the VBA editor.
  5. Double click "ThisWorkbook" in the Project Explorer (Left column).
  6. Copy the VBA code below and paste into the VBA editor.
  7. Do not rename the sub.
  8. Close the VBA editor (Cross very top right of VBA editor window)

Now when the workbook is saved, if changes have been made since opening,  then a backup file will be created in the same folder as the master. If no changes then the file will save as normal but no backup will be created.

The backup File Name is in the format  Filename yyyy-mm-dd hh mm ss.xlsm

The reverse date with year first is used so the file names can be sorted. All 2 digit months, days, hours, minutes and seconds are used because sorting is based on text and this ensures that sorting occurs correctly.

If the User selects "Save As" then the backup code is not implemented and allows the user to create a file name of choice. ie the user might select a backup version and the use Save As to replace the master copy.

Feel free to get back to me if any problems.

Copy all of the code below into the VBA editor.

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
   
    Dim strBackupName As String
   
    'If user selects "Save As" then the code for backup does not run so that user
    'can select a backup and then save to whatever name is required
    If SaveAsUI = False Then
        'Only run backup code if change made. Otherwise Save simply saves without backup
        If ActiveWorkbook.Saved = False Then
            On Error GoTo ReEnableEvents
           
            Application.EnableEvents = False    'Suppress recursive calls to Save event
           
            ThisWorkbook.Save       'Save normally
           
            'Create backup filename using date and time
            'Colons invalid in file name and hense colons replaced with spaces in time portion
            strBackupName = Left(ThisWorkbook.FullName, InStrRev(ThisWorkbook.FullName, ".xlsm") - 1) _
                            & " " & Format(Now, "yyyy-mm-dd hh mm ss") _
                            & ".xlsm"
           
            ThisWorkbook.SaveCopyAs Filename:=strBackupName
        End If
    End If
   
ReEnableEvents:
    If Err.Number <> 0 Then
        MsgBox "Error occurred in ThisWorkbook module, Private Sub Workbook_BeforeSave"
    End If
   
    Application.EnableEvents = True
End Sub

david marcovitz say:

The easiest thing to do is probably to do some kind of a loop that keeps selecting the second number until it doesn't match the first:

Randomize

num1 = 1 + Int(6 * Rnd)

num2 = 1 + Int(6 * Rnd)

While num1=num2

    num2 = 1 + Int(6 * Rnd)

Wend

The loop just keeps checking if the two numbers are the same. If they are, it gets a new random number for the second number. If they aren't, the loop stops and num1 and num2 contain two different numbers.

There are lots of other ways to do this, but this should work just fine. Here's another method without the loop

Randomize

num1 = 1 + Int(6 * Rnd)

num2 = 1 + Int(5 * Rnd)

If num2 >= num1 Then

    num2 = num2 + 1

End If

In this case, you pick the first number, leaving 5 numbers left. So, what you want is to randomly choose one of the five remaining numbers. If your second number is less than the first number, it uses the number it got. If it is greater than or equal to the number, it adds one. For example, if the second number is 4, it is picking the 4th available slot, rather than the number 4. The fourth available slot is going to be 4 (if the first number is 5 or greater) or 5 if the first number is 4 or less.

The beauty of PivotTables is that you can change how they present the data on the fly. You don't need to add VLOOKUPS or other functions to replicate what the PivotTable can already do.

You can increase or decrease the granularity the PivotTable shows by adding/removing fields from the PivotTable field list to/from the report.

jasles say:

By adding columns to the raw data and using the VLOOKUPs etc, you are in effect creating filters. You are just setting the parameters for each one before the report. That is one way of doing it. However, by using the built-in PivotTable calculations and filters, you won't need to add the columns and functions to the raw data.

If you add calculated items to the PivotTable (see here for how to) you can call one "Mechanical" and tell what six different income types are to be summed together for that; create another called "Performance" and have it add the 5 different income types; and repeat for how many items you want grouped together. Then, you can use the built-in filters in the PivotTable to include/exclude what details or what groups are on the report, or the granularity of how the total are broken out.

Doing it this way reduces the amount of work getting the data ready and reduces the chance for errors as opposed to adding the functions to the raw data. Also, next time you get the data, you can just replace the old data with the new and the groups remain since they are part of the PivotTable itself and not the raw data.

Either method will work - either adding columns to the raw data with functions to group the data, or using the PivotTables built-in features. It's just what you're most comfortable with.

The power of PivotTables is foreign to the majority of Excel users - I didn't want you to have to go through a lot of extra work to replicate something that is already built in.

john mcghie say:

Hi Bree:

With this one, you can't.  While you continue to allow Word to switch fonts with the change of keyboard, it will continue to switch to the Chinese font and refuse to come back: it's a bug they won't fix.  You can't control the font it switches to: that's hard-coded.

Go to Word>Preferences>Edit and turn OFF "Match font with keyboard".  Now, it won't switch the font at all when you type Chinese (so you won't ever get the font you don't like).

Create a set of styles the same as the ones you use for English, except specifying your chosen Chinese font as the font.  Apply those styles to your Chinese text.

However, you may find that you don't need to: Word will automatically call in a Chinese font if the font you are working in does not contain the characters you need.  The larger and more modern the font you use, the less likely this is to happen. You might experiment with Cambria, it looks similar to Times New Roman, but contains a wider range of characters, meaning Word has to switch less often.

Normally, if you simply allow Word to call in a Chinese font when it needs it (rather than let it switch fonts when you change keyboard) the result is very acceptable.  So normally if you turn of "Match font" you can pretty much leave everything else to sort itself out.

Hope this helps
