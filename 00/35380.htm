john john __ say:

0x7b is a drive controller error, the boot device is inaccessible, Windows cannot communicate with the hard disk.  When this suddenly happens on a computer that was previously running without problems disk corruption is the usual cause, running the chkdsk  and fixboot commands from the Recovery Console will often fix the problem.  You can also go in the BIOS and set the laptop to boot in IDE mode rather than in SATA and see if it will boot.

We have no way of knowing what state the computer was in when you received it and what "They wiped the hard drive..." means, usually wiping a hard drive means that there is nothing left on the drive, so had the drive been properly wiped you would be getting any Windows errors because there would be no trace of Windows left on the drive, whatever they did before they gave you the laptop it sounds like a botched job.  In any case when getting a second hand computer it is always best to cleanly reinstall the operating system and set the machine to your liking otherwise you more often than not just end up dealing with unknown problems caused by the previous owners.
