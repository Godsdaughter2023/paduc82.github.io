Fiddler is fine to have on hand as a diagnostic for future connectivity issues.  However, the question has just reminded me of a related problem cause.  Sometimes users have security packages which try to do their scans via pseudo proxies and that causes performance problems of various kinds.  So, if you have such security packages in place, you could try uninstalling them, replacing them with minimal equivalent security from Windows and see if that changes your symptom.  Simply using a security package's Disable feature is typically not enough to change such symptoms because to them all that means is that you want them to stop their scan, when what you really want is to stop the message flow to them.  That can only happen when their hooks are removed from the OS.
