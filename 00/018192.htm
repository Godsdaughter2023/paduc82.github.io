Try this...

 

Having values in N2:N38, dates in G2:G38 and the month (number) in C62.

 

=SUM((N2:N38)*(N2:N38>0)*(MONTH(G2:G38)=C62))/SUM((N2:N38>0)*(MONTH(G2:G38)=C62))

We may get this error when one of the custom columns contains Column Validation and the validation failed. some of the users have a lot of shared calendars and they experienced slow response in outlook, mostly with the calendar. before the upgrade they didn't have any problems. So they had the same mailbox size, same network, same everything but the version of windows and office.
I found something called cached folder handling  NOT cached exchange mode, and using a reg entry, you change the behavior of 2010 to act like 2007 by adding at
hkcu\software\microsoft\office\14\outlook\Cached Mode
then in the Cached Mode key, add a dword valued called
CacheOthersMail, and set the binary to 0
This fixed the problem, and what seems to be happening is that as the users open other people's calendars, outlook is cacheing the calendars and next time they open that calendar, it opens faster.
I know the key is named Cached Mode, but none of these computers is in cached exchange mode.
so I'm trying to find more information on this - is it actually Cached Mode, or a modified version of it?
I have been searching "cached folder handling" and all that seems to come up is stuff about cached exchange mode.
also, i feel like I am missing something in the difference between outlook 2007 and 2010, because definitely they handle something different in regard to the calendar, because like I stated, in 2007, everything worked fine.
has anybody run into something like this, or know how outlook handles cached folders?
