Thanks in advance for your help.

john korchok say:

Here are a couple of macros that let you hide individual pictures and then show all hidden pictures. First you'll have to make the Developer tab visible:

  1. In Word, choose File, then Options.
  2. Click on Customize Ribbon.
  3. In the right-hand list, check Developer.

The Developer tab appears.

  1. Click on Developer, then Macros.
  2. Type in a Macro name like HidePicture, then click on the Create button.
  3. The VBA editor window opens. In the space between Sub HidePicture() and End Sub, add text until it appears as follows:
    ActiveWindow.Selection.ShapeRange(1).Visible = msoFalse
End Sub

Then add a second macro like this:

  Dim oShape As Shape
  For Each oShape In ActiveWindow.Selection.SlideRange(1).Shapes
    oShape.Visible = msoTrue
  Next oShape
End Sub

Close the VBA editor. Now, to make the macros easily accessible, add them to the Quick Access Toolbar:

  1. In PowerPoint, choose File, then Options.
  2. Select Quick Access Toolbar.
  3. Under Choose commands from, change the dropdown to Macros.
  4. Select each macro and click on the Add button. They are copied to the list on the right.
  5. You can select a macro in the right-hand list and click on Modify to assign a new icon and display name. The display name appears as a tooltip when you hover the mouse over the QAT.
  6. Finally, resave the presentation as a PowerPoint Macro-Enabled Presentation (*.pptm) file.

Now you can select photos, hide them for printing and show them again for presentations.

Windows 10 Build 10162: Audio issue detailed - Microsoft Community

daniel_k say:

Windows 10 Build 10162: Audio issue detailed

After some in-depth analysis, I've found that the registry data actually does not get corrupt, but it is not updated and synchronized as it should.

When you change an audio configuration in Windows, it is applied to both the driver and the audio engine.

The bug affects this process: the driver configuration is updated but the audio engine is not.

However, if you change the bit depth keeping the same sample rate, the configuration is properly updated.

I wrote a small utility dubbed "Audio Format Utility" as a proof of concept.

It reads the stream format data from all audio render (output) devices present in your system and displays both Driver and (Audio) Engine values.

You can clearly see the configuration mismatches when audio playback is not working.

Changed from 24-bit, 44100 Hz to 24-bit, 192000 Hz, no áudio:

Changed from 24-bit, 192000 Hz (above) to 16-bit, 192000 Hz. Audio output now works.
