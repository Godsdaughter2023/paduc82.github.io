Hello,

you may want to break this question down, because it covers quite a broad range of topics. 

Re data slicer with OR: Not that I'm aware of.  The selection of the first slicer will determine with options are showing in the other slicers. This is always a subset of the original data, filtered by the first slicer. So, by their very nature, multiple slicers are always connected with AND logic.

Re filter on conditional formatting colors. You can't use the color that is created with conditional formatting to filter or sort a table. It will not be offered in the dialog. BUT, you CAN set up helper columns in the table that use formulas with the same logic as the conditional formatting rules. Let these formulas return the color name as text (or a business term that puts into words what the color stands for, like "overdue"), and then filter by these helper columns. 

You can use Word to create a mail merge with data from Excel. For this scenario you would not filter the data in Excel, but just point Word to the location of the Excel workbook. Word then offers a dialog where you can select which rows of the Excel table to use in the mail merge. Again, this can be very easily achieved with helper columns in your Excel table that identify the rows you need in the merge.

scott mcdaniel say:

You can't trap for a write conflict, so you can't use any sort of VBA code to manage that conflict.

You may be able to get around it by forcing a save on your various forms, as you did previously, but that doesn't change the fact that you should handle data in a single process - either your bound form, or code.

Access works best when you work within the confines of Access, and don't try to do something Access doesn't like. The more you get outside the boundaries of Access, the more difficulties you'll find.
