Thank you.

 

JR

greg maxey say:

You've not missed anything.  There is no change event associated with legacy formfields, so what you are after is not possible using formfields.

 

You could evoke a userform OnEntry to the first legacy form field in the series that you could use for filling out a series of numbers and use the change event of the userform text fields to advance to the next one.   Then use a command button to close the userform and populate the document formfields.

 

A related method for using a userform to overcome the 25 item limit of legacy formfields is demonstrated here:  http://gregmaxey.mvps.org/word_tip_pages/formfield_userform_listbox.html

a. user say:

No, unfortunately, when you mail merge, the only thing you can choose is the Outlook /profile/ that will be used.

You can have multiple Outlook Profiles (or "MAPI Profiles") and each profile can have multiple accounts.

I cannot guarantee that this will work, but your best bet may be to

 a. create a new Profile (e.g. try Control Panel->Mail) that only contains the account that you need.

 b. close anything (such as Outlook, and any other program that uses email)

 c. start Outlook, choosing the profile you need.

 d. start Word

 e. do the merge.
