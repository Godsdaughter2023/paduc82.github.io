As far as I can tell, you should not include the file URI scheme in your argument to "F". I just tried it myself and it does not work. Also, the documentation states the following for the "F" argument: it is of type "file specification" where an example of a such could be: 

relative: "subfolder/spec.pdf ", e.g., F(subfolder/spec.pdf)
absolute: "/c/pdfdocs/spec.pdf", e.g., F(/c/pdfdocs/spec.pdf)

- see page 181 and page 659 of the Adobe PDF spec for version 1.7 - sixth edition - link: http://wwwimages.adobe.com/www.adobe.com/content/dam/Adobe/en/devnet/pdf/pdfs/pdf_reference_1-7.pdf )

- so to borrow your own example I would do something like this:

<</Subtype/Link/Rect[ 77.651 600.07 149.5 613.86] /BS<</W 0>>/F 4/A<</Type/Action/S/Launch/F(Royfs04/database/MJ%20Quality%20&%20Technical/Quality/SANDBOX%20FOLDER/Royston%20Procedures/Series%203%20Procedures/QMS-3-0-5B.pdf) >>/StructParent 4>>

Note however, that the path given to "F" ( in this example ) is a relative path ( relative to the PDF file ) - but I guess that is what you want :)

bill manville say:

Is the event procedure in the module of the worksheet concerned?
If it is in a standard module it will not work on the calculate event.
To get to the worksheet's module, right-click its sheet tab > View code.

The calculate event could be triggered when a different worksheet is active, so the reference to Rows(..) should be preceded by Me.

The procedure doesn't say what should happen if the value is between 2.4 and 2.5 inclusive.
Maybe this is intentional; maybe it can't happen, but if you meant the rows to be hidden only if the value is greater than 2.5 then a better coding would be:

Private Sub Worksheet_Calculate()
    Me.Rows("61:72").EntireRow.Hidden = (Me.Range("H57").Value > 2.5)
End Sub

I didn't quite understand what you meant by problem 2.
But I guess you mean that if H57 contains the text you don't want the rows to be hidden, so

Private Sub Worksheet_Calculate()
    Me.Rows("61:72").EntireRow.Hidden = (Val(Me.Range("H57").Value) > 2.5)
End Sub

[Val returns 0 if the range contains text, otherwise the numeric value]

[Later ... on testing the above]

There is a problem in that the action of hiding the rows triggers another calculation which causes the procedure to be re-entered and eventually a Stack Space Full error occurs.  So

Private Sub Worksheet_Calculate()
    Application.EnableEvents = False
    Me.Rows("61:72").EntireRow.Hidden = (Val(Me.Range("H57").Value) > 2.5)
    Application.EnableEvents = True
End Sub

Michelle,

Rod has given you some guidance that may have gotten lost. 

If you wish to spread 30 hours of work over a 24 day duration, you need to make sure the tasks are Fixed Duration.  Show the Task Form (right click in the Gantt and select "show split).  Change the type from Fixed Units (default) to Fixed Duration and click OK in the Task form.  You may now assign the resource entering the 30 hours of work and click OK in the Task Form.  Behind the scenes Project calculates the Peak units necessary to spread the 30 hours of work out evenly over the 24 day duration.  If you wish to split the work if you assign additional resource, in the Task Form click the checkbox for Effort Driven and click OK before you assign the additional resource.  Project will split the 30 hours of work based upon the max. units for each resource.
