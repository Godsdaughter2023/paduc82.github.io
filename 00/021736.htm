todtown say:
Put this code in Sheet1. Now when you type anything in any cell in Column B of sheet1 (except Cell B1), it will list all of the related addresses below where you typed.

Potential downside - this will delete everything in the column below where you typed.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim ReportBook As Workbook
    Dim MainSheet As Worksheet
    Dim LookupSheet As Worksheet
    Dim LookupValue As String
    Dim Cell As Range
    Dim StartHere As Range
    
    Set ReportBook = ThisWorkbook
    Set MainSheet = ReportBook.Worksheets("Sheet1")
    Set LookupSheet = ReportBook.Worksheets("Sheet2")
        
    'Assuming you have a heading in Column C
    If Target.Column = 2 And Target.Row > 1 Then
    Application.EnableEvents = False
     LookupValue = Target.Value
     Set StartHere = Target.Offset(1, 0)
     MainSheet.Range(Target.Offset(1, 0).Address & ":" & MainSheet.Cells(Rows.Count, Target.Column).Address).ClearContents
    
     'Assuming the column has a heading
     For Each Cell In LookupSheet.Range("C2:C" & LookupSheet.Range("C" & Rows.Count).End(xlUp).Row)
         If Len(Application.WorksheetFunction.Substitute(Cell.Value, LookupValue, "")) < Len(Cell.Value) Then
             StartHere.Value = Cell.Value
             Set StartHere = StartHere.Offset(1, 0)
         End If
     Next Cell
    End If
    Application.EnableEvents = True
End Sub

cbarnhorst say:

You will need to ask X-Rite.  Ordinary software should not have anything to do with the Intel CPU flaw currently being discussed.

In general, here is what we know:

The CPU flaw concerns security.  The flaw is a design flaw involving how the CPU does look-aheads in its instruction queue, not how a program works.

Microsoft released a cumulative update, KB4056892, for regular Windows 10 last week.  MSFT did NOT release the KB for Insider preview build 17063 because Insider previews are replaced by later preview builds, not updated.  Preview builds from 17040 are believed to have the fix integrated into their code.

Some anti-virus solutions do need to be updated to support the Microsoft fix released as part of KB4056892 last week.  AV programs dig deep into the device's hardware and software internals.

It is up to your device's manufacturer to provide micro-code updates (firmware) for your hardware that also addresses the CPU issue.  If your device is no longer supported by the manufacturer then that is problematical, of course, and in such a case it is unknown whether KB4056892 is sufficient for your system under Windows 10. 

If you haven't already, it is important that you review the information about the flaw by doing a search on the internet.  The problem is far too complex and difficult to explain by detailing it in a forum.  Besides an internet search, I suggest you review what Windows Central has published and follow the links to other sources contained in their articles so far.  

Unfortunately, there is not a one-shot solution for all.  You will need to know what your device's manufacturer is doing about it as well as Microsoft since a full solution involves both hardware and software.

If Word is capitalizing the first word in each line, then each line must be a separate paragraph (see Charles's reply for the explanation). sebthirlway say:

Suppressing the "loading image" dialog in MS Access

Hi

I'm programming an MS Access (2003 SP3) database to manage images.  OS is XP Pro.  Images are not embedded as BLOBs in the tables as I've found that doesn't work.

So my form has Picture controls, which are set to load an image from a particular path (stored in the table), using VBA: Me.APictureControl.Picture="C:\bla bla bla"

Problem is Access pops up this annoying "loading image" dialog.  I managed to solve this problem a few years back, but I've lost that application where I solved it, and none of the solutions I remember seem to work.  The solutions I'm trying come from http://www.mvps.org/access/api/api0038.htm

a) The registry tweak: change HKEY_LOCAL_MACHINE\Software\Microsoft\ Shared Tools\Graphics Filters\Import\JPEG\Options\ShowProgressDialog to No

I've tried this (and yes, all the images are JPEGs) - even restarted the machine, but no effect whatsoever.

b) The API kludge:

On the form's Timer event, call the code provided on that page (see URL above) - this detects any child popup window of the current Access window, and calls the API ShowWindow with a HIDE setting.

This works a bit ... The annoying dialog flickers, or only shows momentarily.  Setting the form's TimerInterval property (thus changing how often ShowWindow gets called) varies this - but there's no setting (I've tried from 1ms up to 200ms) that gets rid of it completely.

 

Any ideas?

thanks!

sebthirlway say:

Ahh.... now I've found the solution myself, here: http://bytes.com/topic/access/answers/463821-suppress-loading-image-dialog

The problem is that on XP, the registry value has to be changed in both HKEY_LOCAL_MACHINE... and HKEY_CURRENT_USER...

Once I've done that, i.e. changed

HKEY_LOCAL_MACHINE\Software\Microsoft\ Shared Tools\Graphics Filters\Import\JPEG\Options\ShowProgressDialog

and

HKEY_CURRENT_USER\Software\Microsoft\ Shared Tools\Graphics Filters\Import\JPEG\Options\ShowProgressDialog

both to "No"

it works!
