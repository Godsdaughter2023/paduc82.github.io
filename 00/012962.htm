Hi Daniel

(didn't see your update before - sorry about that). Your problem start here: some values in column [Leave til] are not stored as NUMbers. Excel stores Date, Time and Date/Time values as NUMbers:

Not sure how these 2 offending values were enterred and didn't spend time on this

Solution is presented in a Table (not mandatory, you can work with ranges if you prefer) named Table1. Big benefits of Tables are: dynamic ranges (auto expand as you add rows/columns), formulas & formats auto-copied to the added rows….

Re. Date & DateTime values

When you enter i.e. 28/11/2018 in a cell you write a NUMeric value (except if enterred in a cell formatted as Text). If you change the cell format to dd/mm/yyyy hh:mm your cell will display 28/11/2018 00:00. And if you format that cell as Number with 2 decimals ==> 43432.00

If in another cell enter i.e. 28/11/2018 10:00 and format that cell as Number with 2 decimals ==> 43432.42

43432 is the INTeger part that represents the DATE portion of the DateTime value, while the Decimal part represents the TIME portion of that DateTime value

Re. MOD

Above blabla to explain the use of the MOD function with the [Leave til] values. MOD(G19,1) returns 0 as the NUMeric value "behind" 28/11/2018 is 43432.00. MOD(G23,1) returns a value > 0 as the NUMeric value "behind" 28/11/2018 12:30PM is 43432.52.

So formula in O19:

--(MOD(Table1[[Leave til]:[Leave til]],1)=0)) and copy right & down

or (with the corresponding range)

--(MOD($G$19:$G$33,1)=0))

Re. Table references

In theory I could have written Table1[Job group]=$N$18. Here I used syntaxt Table1[[Job group]:[Job group]]=$N$18 to lock the columns (same as $ on a Range), allowing to copy the formula right

Re. Date and DateTime display

columns [Leave from] & [Leave til] are both are formatted to display values as dd/mm/yyyy + a  Conditional Formatting rule that again use the MOD function: =MOD(F19,1)>0. When it returns true the dd/mm/yyyy hh:mm AM/PM format is applied

Your updated file & suggestion solution available here.

One recommendation: to avoid that people enter something else than Date or DateTime values in columns [Leave from] & [Leave til] implement a Data Validation rule (see in above file)
