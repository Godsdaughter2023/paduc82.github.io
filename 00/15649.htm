afzal taher say:

Hi,

a. Does this happen with only one particular website?
b. Are you able to open that website using other browsers?

Error 403 Forbidden means that you are not authorized to view information on that particular web server. Internet Explorer is able to connect
to the website, but Internet Explorer does not have permission to display the webpage.

First try to change the proxy server settings.

Refer this link for instructions:
http://windows.microsoft.com/en-US/windows7/Change-proxy-server-settings-in-Internet-Explorer

If that doesn’t fix the problem, then you may try these steps and check if it fixes the problem:

1. Go to start and type in cmd in the search box.
2. Right click on the command prompt and click run as administrator.

Type the following commands in the command prompt, each followed by pressing enter.

ipconfig /flushdns

nbtstat -R

nbtstat -RR

netsh int reset all

netsh int ip reset

netsh winsock reset

Exit.

Restart the computer.

Now open Internet Explorer and check if you can access the website.

scottgem say:

This is one of the reasons I don't like composite keys. They make related tables much more complex. What you should be doing is using an Autonumber as the PK in Table A then use THAT value as the FK in Table B. This would eliminate the need for duplicating those 4 pieces of info and make your life a lot easier. 

You can then ensure no duplication in Table A by creating a multi-field unique index in Table A. So the Autonumber PK will therefore define the composite key and eliminate the need for copying multiple pieces of data between tables.

scottgem say:

Jerry,

It doesn't get populated as the Autonumber in Table B. Table B should have its OWN Autonumber PK.  The FK will contain the value of the PK in the related record. So using your example, Table A has an autonumber PK of 1, with the other three fields containing 123, 456, 789. So Table B now has three records. Each with its own autonumber PK so you have 1, 2 & 3. But each record also has a FK which would contain a value of 1 linking that record back to the record in Table A.

The only way the FK gets automatically populated in Table B is if you use a subform. Since you haven't told us very much about what your database does and what these tables represent you are making it harder for us to help you. But if this is a "classic one to many" then you should be able to use an embedded subform. By linking the subform to the main form PK to FK, then as you add records in the subform, the FK will be automatically populated with the PK value from the current record shown in the mainform. If you do not use an embedded subform, then you have to use code to automate populating the FK. This makes it even more important to use the autonumber as a PK rather than a composite PK.
