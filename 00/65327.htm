Hi

Here is more information on the Bug Check 0x8E: http://support.microsoft.com/kb/945658/

The 0x8E KERNEL_MODE_EXCEPTION_NOT_HANDLED can be caused by a faulty service/driver, damaged hard drive or not enough space, Faulty Display Card or memory, Hardware/software incompatibility and third party remote control software. Therefore to start:

1. Test your memory. Run Memtest which you can download from http://www.memtest.org/#downiso. Download the ZIP file, extract the ISO and write it to CD. Once written to CD, boot your PC with the CD and run Memtest. Allow the test to run through at least 3 passes or until it gives errors. If you have more than 1 memory module run the test on each module separately. Try to run Memtest from a cold boot - unplug your PC for an hour for it to cool down then immediately run memtest when you switch it back on. This way memtest will run when the memory is cool and after the 3rd pass when the memory is hotter. 

2. Update your display driver. Dell seems to have brought out 3 different Dimension 3000 models and thus there are 3 different Intel display drivers. See if you can find an updated driver on Intel's website. If you are using an add-on graphics card try to find an updated driver from the manufacturer's website. If you are having trouble finding a driver let us know what make and model graphics card you have.

3. Update the Rhapsody software. (I think this is the software you are using http://www.rhapsody.com/-software ?). If this is the Rhapsody software I am thinking of it seems to integrate into your Web Browser and that might be why Opera and now IE are giving errors? You might want to uninstall Rhapsody for a while just to see if that is the reason for your BSODs.

4. When the 0x8E BSOD is caused by win32k.sys, as yours it, then it is possible it is caused by a third party remote control software. These can include programs like PCAnywhere, VNC, RAdmin, Teamviewer etc.  If you are running one of these programs uninstall it and see if the PC stops giving errors.

5. Run a scan on your hard drive. You can check your hard drive for any errors by going toStart ->Run -> type chkdsk c: /r -> clickOk  . When asked if you want to force a dismount sayNo. When asked to schedule the check at the next startup sayYes. Restart your PC to allow chkdsk to run.

You can also run a diagnostic on your hard drive. You do this by downloading the appropriate diagnotic program from the manufacturer of your hard drive. A list of manufacturers and their diagnostic tools can be found here: http://www.tacktech.com/display.cfm?ttid=287. Once again download the ZIP file, extract the ISO and write it to CD. Once written to CD, boot your PC with the CD and run the advanced/extended test.

Also make sure you have enough free hard drive space. It is always recommended, if possible, to have at least 15% of your hard drive capacity free.

6. Lastly have a look in your Event Viewer at the time of the BSOD for any errors. You will especially want to look in the System and Application logs (and maybe Internet Explorer logs). You can run Event Viewer by going toStart ->Run -> type eventvwr.msc -> pressOk -> the Event Viewer screen will open.
