dirk goldgar say:

How would I make Partner First Name required if Partner Last Name is not ""


The simplest way would be with a table-level validation rule like this:

 

    ([PartnerLastName] Is Not Null) Imp ([PartnerFirstName] Is Not Null)

 

Adjust field names to suit, and set the Validation Text to some message such as:

 

    If there is a partner, both the first and last name are required.

 

This rule uses the little-known "Imp" operator.  "Imp" is short for "implies", and means that the first operand logically implies the second operand.  If you're not familiar with it, you can look it up in the help file in the VB Editor.

 

Note that the validation rule above will allow the PartnerLastName to be Null even if the PartnerFirstName is not.  That was what you asked for.  If you wanted to require both fields or neither, you could write the rule like this:

 

    ([PartnerFirstName] Is Null And [PartnerLastName] Is Null) Or ([PartnerFirstName] Is Not Null And [PartnerLastName] Is Not Null)
