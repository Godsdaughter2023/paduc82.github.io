¡firedog say:

Uninstalled Windows Live Mail and re-installed... didn't work.  

No, it wouldn't. Messaging problems are very rarely solved by re-installing the program. It might be the last resort after all else has failed.

Tried to figure out if Avast is the issue and stopped having it check email.

...

Server Response: 421 Cannot connect to SMTP server (with ip addresses following) connect error 10060

You can see that that error message is not coming from the server - how could it, if the program wasn't even able to connect to it? It's coming from Avast, whose Mail Shield insinuates itself between client and server. The Mail Shield doesn't add any extra security if you're otherwise careful, and I can't think of any good reason to scan outgoing mail. At best, it only adds a little advertisement to your messages, and at worst, it prevents you from sending altogether.

Close down Windows Live Mail and disable the Mail Shield. Wait a few minutes while Windows Live Mail completes its housekeeping, then reboot the computer and check that the Mail Shield is still inactive. Then launch Windows Live Mail, compose a new test message with a unique subject and send it to yourself. Post any new error message in your reply.

PS The IP address you removed will have been that of the mail server you were trying to connect to, followed by the port that Avast was trying to use.

¡firedog say:

So HOW do I get my folders back?

Please make sure your messages haven't just been moved. Make sure that Storage folders is selected (highlighted) on the View tab.

  • When viewing your message list, press Ctrl-Y to reveal the entire folder tree. Look for your missing messages, possibly several layers deep under Recovered items or inside another folder.

  • Use the Search for a message box to look for one of the missing messages. Use a search term in the Subject or To fields for quickest results.

If you can't find the missing messages in Storage folders, on Windows 7 you may be able to use the
 

  1. In Windows Live Mail, press Ctrl-Shift-O for Options.
  2. On the Advanced tab, click Maintenance and then Store folder.
  3. Copy the path shown into the start search box and press Enter.
  4. Right-click on the folder and select Restore previous versions. 
  5. If you can locate a previous version of your store folder that predates the folders' going missing, Open it and then locate the ones that are missing.
  6. Copy the folders into a temporary folder on your desktop.
  7. Use the File > Import messages function (Alt-F I) in Windows Live Mail to import the temporary folder.

Previous versions are ephemeral. If you found something in a previous version, you should recover it sooner rather than later. Upon signing in, the "Skype name does not exist" error message indicates incorrect log in credentials. Please be guided that Skype display name and system-generated Skype name with a prefix of "live:" cannot be used to sign in to Skype. You need to use the credential (email or phone number) when you initially created the account.

You can check all the Skype accounts associated with your email address with the steps below.

  • Go to this link.
  • Enter your email address and select Continue.
  • We'll show you all the Skype accounts associated with that email address.
