mike h.. say:

Hi,

A cell can contain #N/A for a couple of reasons such as a failed LOOKUP or even someone entering =NA() in a cell so let's take a particular example. This formula would return #N/A if it couldn't find the value in A1 in column I

=VLOOKUP(A1,I1:J22,2,FALSE)

There are 3 ways to fix this.

1. Insert the value from A1 into Col I but this of course means the data are no longer valid. i.e. we made it up.

2. Change the formula which is what should have been done in the first place to something like this which makes the error return a null string

=iferror(VLOOKUP(A1,I1:J22,2,FALSE),"")

3. Run code on the worksheet to delete all the errors.

How to proceed?

jllatham say:

One other frequent candidate for #N/A is MATCH().  So be on the lookout for those.

Errors "propagate" ... meaning that if a cell comes up with an error result, then all cells that use a formula that references the one with the error will display the same error.  This chain continues on to other cells that reference the cell(s) that referenced the error cell.  So you've got to track back to the source of the problem and fix it there.

Example:  in cell A1 you have the formula =C1  but it's showing #N/A!.  So you go look at C1 and find it is a VLOOKUP() [or HLOOKUP() or LOOKUP() or MATCH()] and that is returning a #N/A.  So you go verify that it is an honest report - easy way to do that is as Mike H. said, temporarily fudge the data in the first column of the lookup table so that a match should be found and look again.

Once you get to the root of all your evils, then you decide how to proceed.

You can take the easy way out provided by Excel 2007 and later (but that may fail on earlier versions of Excel) ... you use that handy IFERROR(my_formula_here, error_result_here) function.  Potential pitfall with that is that it 'ignores' all errors. 

You can fall back to a more specific, but a little more complex error trap like this:

=IF(ISNA(your_formula), error_response_here, your_formula))

as

=IF(ISNA(VLOOKUP(A1,$I$2:$L:243,3,False), "", VLOOKUP(A1,$I$2:$L:243,3,False))

If the .xlsx file is coming to you with only #N/A in the cell, and no formula, then it is likely being sent out from Einstein with that content.  Perhaps there was originally a formula in that cell, and someone did a copy/paste special values.

If you think your Excel installation is somehow converting the contents of the cell to #N/A and somehow losing the formula/results that produced that, you could inspect the .xlsx file before opening it in your Excel installation.

.xlsx files are ZIP files, so one simple method is to make a copy of the file, and then change the extension to .zip; and open it with Winzip or similar.  If you examine the sheet.xml file, you will see the cell, and, if there was a formula in it when it was sent out, you will see the formula.

If all you see in that cell is #N/A, then that is how it is being sent out, and you'll need to investigate that process more closely.
