The macros are not the problem.

As for whether to save as xlsx (xlsm) or xls:  it depends on whether or not you still want the file to be opened in Excel 2003 eventually.

If you do, you should stick with Excel 2003 compatibility mode ("97-2003" file with xls extension).

If you do not need Excel 2003 compatibility, it would not hurt to save as xlsx (xlsm).  But keep in mind that that will create a new file.  In other words, the old (unmodified) xls file will still exist.  That might cause some confusion.  It is simple to remedy:  remember to delete the xls file manually afterwards.

However, the warning of lost functionality might be overkill on Excel's part.  I see that warning when I know that I have been careful not to use Excel 2010 extensions.

You could disable the compatibility check.  But I prefer to leave it in place, just in case I make a mistake.

jay freedman say:

1) When a document contains more than one section, you must include the section number in the expression in the Pages box. For example, if the pages 10 and 11 that you want are in Section 1, type p11s1-p12s1 in the box. See https://wordmvp.com/FAQs/Formatting/PrintMultipleSections.htm.

2) Go to File > Options > Trust Center > Trust Center Settings > Privacy Options. Clear the check box next to "Make hidden markup visible when opening or saving". This option is intended to make sure you know when a document you're opening contains markup that you can't currently see, which could lead to sending such a document to someone who shouldn't see the markup. Turn it off at your own risk.

Even when the markup is visible in Print Layout view, you can prevent it from printing. On the File > Print page, open the dropdown that contains "Print All Pages" and click to clear the check box from "Print Markup".
