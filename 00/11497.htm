ivanhan_msft say:

To update this thread, the correct sequence to move a class notebook from a SharePoint site to your OneDrive is as follows:

1) Launch the Class Notebooks app from your waffle and click the "Get notebook links" button. This is important if you've never used the Class Notebooks app before as this will create the "Class Notebooks" folder in your OneDrive

2) Open your class notebook that resides in the SharePoint site in your OneNote desktop client. Let the notebook sync completely.

3) Right-click the notebook and select "Properties". Then click "Change Location". You will now need to specify the URL of the "Class Notebooks" folder from step 1. This URL is obtained by going to your OneDrive to obtain its URL, which should look something like:

   yourschool.sharepoint.com/.../onedrive.aspx

4) Replace "_layouts/15/onedrive.aspx" with "Documents/Class Notebooks". This is the URL of your "Class Notebooks"

folder in your OneDrive. Paste this URL into the "Change Location" dialog in step 3. Once the notebook's location has been changed, let the notebook sync completely.

5) Once the notebook has completely sync'ed to its new location under your OneDrive's "Class Notebooks" folder, launch the Class Notebooks app from your waffle.

6) Click the "Create a new notebook" button. Name your notebook the same as the notebook you just moved.

7) Add the same students as the notebook you just moved.

8) Click "Next" and finally "Create". This will repurpose the notebook you just transferred to a class notebook with the proper permissions set

ivanhan_msft say:

That said, it is generally not advised to move class notebooks from your SharePoint site to your OneDrive as it doesn't buy you much.

Each class notebook is tied to a class. For every new class, it is recommended you create a new class notebook using the Class Notebooks app.

If you need to copy content from an old class notebook to your newly created class notebook, simply open both the old and new class notebooks in your OneNote desktop client. When copying content, do not copy the "_Collaboration Space" or "_Content Library" section groups. Instead copy the sections under these section groups. The reason for not copying the "_Collaboration Space" or "_Content Library" section groups is because these section groups have special permissions tied to them which you do not want copied over to your new class notebook.

Copying multiple sections from one "_Content Library" to another is time consuming. The OneNote team will be releasing a tool very soon that will greatly make this process very simple, so stay tuned!

Like so, assuming you've first selected the two shapes (an arrow and another shape) that you want to connect:

Sub ArrowToShape()

    Dim oArrow As Shape
    Dim oTextBox As Shape
    Dim x As Long
    
    ' assuming both are selected
    With ActiveWindow.Selection.ShapeRange
        If .Item(1).Type = msoLine Then
            Set oArrow = .Item(1)
            Set oTextBox = .Item(2)
        Else
            Set oArrow = .Item(2)
            Set oTextBox = .Item(1)
        End If
    End With
    
    With oArrow.ConnectorFormat
        ' The number after oTextBox is the connection site
        ' you want to attach the end of the arrow to
        ' connection sites of text box are
        ' 1 = Top middle
        ' 2 = Left middle
        ' 3 = Bottom middle
        ' 4 = Right middle
        .EndConnect oTextBox, 1
        
        ' if you want to connect the beginning point,
        ' use .BeginConnect instead
        
    End With
    
End Sub

Sub connectTwo()

Dim oArrow, firstRect, secondRect As Shape
Dim myDocument As Object
Dim s As Shapes

Set myDocument = ActivePresentation.Slides(1)
Set s = myDocument.Shapes

'Make Rectangles
Set firstRect = s.AddShape(msoShapeRectangle, 100, 50, 200, 100)
Set secondRect = s.AddShape(msoShapeRectangle, 300, 300, 200, 100)

'Make Arrow
'Arrow location is not important because it will be adjusted when connections are made.
Set oArrow = s.addConnector(msoConnectorStraight, 0, 0, 0, 0)

'Define Arrow Properties
With oArrow.Line
     .BeginArrowheadLength = msoArrowheadLengthMedium
     .BeginArrowheadStyle = msoArrowheadOpen
     .BeginArrowheadWidth = msoArrowheadWidthMedium
     .Weight = 1.5
     .ForeColor.RGB = RGB(0, 0, 0) 'Black

End With
'Define Connection Properties
With oArrow.ConnectorFormat
    .BeginConnect firstRect, 1 'Do not put in ()  (site may change when rerouted).
    .EndConnect secondRect, 1 'Do not put in () (site may change when rerouted).
    .Parent.RerouteConnections 'Sets connection site(s) to be shortest distance.
End With


End Sub

steve rindsberg say:

Methods behave more or less like Subroutines in VBA.

You can invoke a subroutine that requires parameters in two ways:

SubName param1, param2, ... , paramX

or 

Call SubName(param1, param2, ... , paramX)

Well .. three ways 

SubName paramname1:=value1, etc

Functions (as opposed to Subroutines) return a value and are always invoked with:

VariableName = FunctionName (param1, param2, etc) 

or some variation of that, depending on what the function returns.

If you invoke a subroutine with parens w/o using the Call syntax, VBA thinks you're invoking a function, notices that you've not doing anythign with the returned value, or that there IS no returned value.  And scolds you.  ;-)
