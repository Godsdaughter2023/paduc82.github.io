don cuthbert say:

Multi-Edition ISO for Windows Insiders Fast Ring

Note: Creation of unofficial Windows 10 ISO files is neither recommended nor supported by Microsoft.

Background:

Many Insiders would like to have an ISO file for each Windows 10 Insider build that is released. It enables the ability to quickly perform a clean install of that build, if that is desired for any reason.

Prior to the implementation of UUP, each build delivered via Windows Update downloaded a full, unencrypted, install.esd file which could be converted to an ISO file using PowerShell or DISM commands. Various scripts were developed by users to expedite this process.

The UUP process now used for Windows Update delivery of Windows builds no longer delivers the single, full ESD file, but industrious users have developed scripts to convert the UUP downloaded and processed files into a usable ISO. The resulting ISO does not have the same integrity as an ISO produced from a Microsoft-provided ESD file, but because the source files originate from Windows Update, it still produces a reliable and useful result.

Commonly used scripts and utilities to produce an ISO for the latest Insider build are uup-converter-wimlib-13, available through MDL Forums and UUPtoISO, from TenForums.com (which is based on the script found at MDL Forums). Everything that these utilities do could be done using PowerShell or DISM commands, but the scripts very much automate the process. They use a popular library called wimlib, which provides tools similar to DISM, but with some additional features and improved simplicity in the command syntax.

__________________________________________________________________________________

Multi-Edition ISO

Going one step further in producing an ISO is to create one that contains multiple Editions of Windows 10, much like the official ISO that is made available by Microsoft with the public release of each new Windows version. If you check the contents of the install.esd file from the official Windows 10 ISO, you will see that it contains Home/Core, Pro, Home Single Language and Education editions:

Another useful feature of Windows images is the built-in ability to upgrade to a higher Edition. If you mount a Windows 10 Home/Core Edition image from a WIM file, it can be converted to several higher Editions. The resulting image can then be either captured to a new WIM file, or saved as an added image to a new Index in the original WIM file. WIM files can contain multiple images, stored as separate Indexes. They maintain a reduced file size by never replicating any file that is already present in an existing Index when a new image is added. An ESD file is a more compressed version of a WIM file.

Producing a standard ISO after a Windows Update UUP upgrade to a new build

Here is the process I have used to create a Windows Insider Fast Ring standard ISO. I begin by upgrading a Home Edition PC or Virtual Machine, in order to be able to produce multiple editions after only one download and upgrade has been processed.

Let Windows Update upgrade the device (PC or Virtual Machine) to the new build. After the upgrade has completed, the files that were delivered via UUP and processed for the upgrade can usually be found in a subfolder under C:\Windows.old\Windows\SoftwareDistribution\Download\. I usually use the search feature in File Explorer to find the subfolder containing one or more ESD files. Right-click and copy the path to the correct folder for use with the utility for producing the ISO. An example is:  C:\Windows.old\WINDOWS\SoftwareDistribution\Download\f67639de26e692d0341857afb1bfbaf3.

Running convert-UUP.cmd from uup-converter-wimlib-13:

The ISO utility has the option to create an ISO file, or only a standard install.wim file (which would also be included in the ISO). I use the install.wim option when I only want to add an additional image, such as Windows 10 S, which can only be obtained via Windows Update.

Creating a Multiple Edition WIM file

  • Copy the Windows 10 Home install.wim file from the ISO that you have made and copy it to a folder.
  • Mount the install.wim file to an NTFS folder or VHD (cannot mount to ReFS) such as C:\MOUNT.
    • Mount-WindowsImage -Path C:\MOUNT -ImagePath install.wim -Index 1​
  • Check the image for Edition upgrade options
    • Get-WindowsEdition -Path C:\MOUNT\ -Target​
  • Upgrade image to higher Edition
    • Set-WindowsEdition -Path C:\MOUNT\ -Edition Professional
  • Dismount and save the converted image to a new Index in the original install.wim file
    • Dismount-WindowsImage -Path C:\MOUNT -Save -Append -CheckIntegrity
  • You now have an install.wim containing both Home and Pro Editions
    • repeat as desired to convert and add other editions to the install.wim
    • wimlib-imagex.exe can be used to rename the new Indexes (syntax can be shown with .\wimlibimagex.exe --help)
      • .\wimlib-imagex.exe info install.wim 2 "Windows 10 Pro" "Windows 10 Pro"
  • Capturing to a separate install.wim file gives opportunity to rename the new image and description, but adds a new 3-4GB file.
    • Add-WindowsImage -ImagePath "Pro install.wim" -CapturePath C:\MOUNT -Name "Windows 10 Pro" -Description "Windows 10 Pro" -CheckIntegrity -Verify

Exporting images to a more compressed ESD file

Exporting the various install.wim Indexes to an ESD file reduces the size greatly, potentially enabling the ISO to be burned to DVD. DISM can use "recovery" compression, while PowerShell is limited to "max" compression.

  • DISM /Export-Image /SourceImageFile:install.wim /DestinationImageFile:install.esd /Compress:recovery /CheckIntegrity /SourceIndex:1
    • repeat command for each /SourceIndex number 1, 2, 3, 4, etc. that you have saved to the install.wim file.
      • Get-WindowsImage -ImagePath install.wim <-- will display the Indexes in the install.wim file.
  • For any additional Windows Edition install.wim files that you have produced (that may not have been available in the Home image) such as Windows 10 S or Enterprise, you can Export that additional install.wim file to the same ESD file
    • DISM /Export-Image /SourceImageFile:"Windows 10 S install.wim" /DestinationImageFile:install.esd /Compress:recovery /CheckIntegrity /SourceIndex:1

Creating the Multi-Edition ISO

To take your Multi-Edition ESD file and create a bootable ISO file, begin with the Home Edition ISO that you first made (you could also use an Official Microsoft ISO). Mount the ISO file in Windows and copy all the contents to an NTFS folder, such as C:\ISO. Now delete the install.wim file from the C:\ISO folder and replace it with your multi-edition install.esd.

The next step is to produce a bootable ISO file from a folder on your hard drive. I prefer to use as few non-Microsoft utilities as is possible, whenever practical, so I was happy to find a PowerShell function, written by Chris Wu, that will do this without requiring any additional software.

I have incorporated this Function into a PowerShell PS1 file named "Windows 10 ALL ISO.ps1". Here is a link to the same file in TXT format, if you wish to download or edit it for your own purposes. You can rename the downloaded TXT file to a PS1 file.

  • This PowerShell script takes the folder at C:\ISO and produces a bootable ISO file in the current folder.
  • You have to use an unusual syntax to run a PowerShell script: " dot space dot\'Windows 10 ALL ISO.ps1' ":

. .\'Windows 10 ALL ISO.ps1'

  • Also, you need to enable the running of local PowerShell scripts in your Windows Security settings, at least temporarily.

I hope that others in the Community will add suggestions or corrections to the Discussion as needed, or will suggest better or more expedient methods that could help other interested Windows Insiders.

Redistributing Microsoft software is not allowed according to the Microsoft Terms of Use so keep any ISO files you may make for your own use only:

don cuthbert say:

@max green

Yes, I don't know how to change the original author's Function to enable the multi-boot (Legacy and UEFI BIOS modes) like oscdimg.exe can do. However, if you use Rufus to create a Legacy BIOS-bootable USB stick from the resulting ISO file (originally created with NewIsoFile function) it actually does boot on a Legacy BIOS PC properly. Rufus must supply what is needed, since it is not in the ISO file in this case.

So if a user needs to boot a Legacy Virtual Machine using the ISO file, the command line (last line of the PS1 script) to produce that would be:

dir C:\ISO | New-IsoFile -Path "Windows 10 ALL.iso" -BootFile C:\ISO\boot\etfsboot.com -Media DVDPLUSR -Title "Windows 10 ALL Editions"

Oscdimg.exe is definitely the standard Microsoft utility for creating an ISO. But the average user does not have oscdimg.exe and it can be a large download to get it officially from Microsoft (as part of a much larger Microsoft Kit). I should note though that Microsoft's cdimage.exe is included with the uup-converter-wimlib download, so your command line would work with that, without requiring any more downloading.

So, the PowerShell Function, written by Chris Wu, still appeals to me for this purpose. I like the simplicity of the command line, compared to the required oscdimg.exe syntax. It also seems to run very quickly to produce the ISO. I was originally working on doing the entire process via PowerShell, or PowerShell and DISM. I believe that a PowerShell-only method would be possible, but the uup-converter script is very well written and actively maintained by the main author. Plus I have close to zero experience with scripting.

Note:  In some cases (if install.wim is used instead of install.esd and too large to fit on a DVD) you have to switch the DVDPLUSR with another option, such as DVDDASHR_DUALLAYER.

  Application.PrintCommunication = False
    With ActiveSheet.PageSetup
        .PrintTitleRows = "$1:$18"
        .PrintTitleColumns = "$A:$D"
    End With
    Application.PrintCommunication = True
    ActiveSheet.PageSetup.PrintArea = ""
    Application.PrintCommunication = False
      With ActiveSheet.PageSetup
        .LeftHeader = ""
        .CenterHeader = ""
        .RightHeaderPicture.Filename = pLogoFileName
        .RightHeader = "&G"
        .CenterHeader = ""
        .LeftFooter = strCentreFooter
        .CenterFooter = ""
        .RightFooter = pRightFooter
        .LeftMargin = Application.CentimetersToPoints(1.5)
        .RightMargin = Application.CentimetersToPoints(0.75)
        .TopMargin = Application.CentimetersToPoints(1)
        .BottomMargin = Application.CentimetersToPoints(1)
        .HeaderMargin = Application.CentimetersToPoints(0.5)
        .FooterMargin = Application.CentimetersToPoints(0.5)

        .PrintQuality = 300
