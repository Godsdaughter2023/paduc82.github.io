daniel g. grau say:
The error message is misleading. Word should be saying "I could not save the file." Factors that contribute to Word's message:

  • Having the file open and working on it for hours on end. If such is the case, you may find that simply closing and reopening the file, or just doing a Save As under another name, will restore functionality. This appears to be because Word keeps track of Undos in hidden files. If there are too many Undos, Word reaches its limit on the number of open files.
  • Using track changes. This increases the complexity of the document and, consequently, that of the Undo files. If you are only tracking changes by one author, sometimes you can instead work without tracking changes (all changes accepted) and do a file comparison with the original version after you are done with the changes.

Excel for Mac does not have the interface for number formatting in the Conditional Format dialog. As you observed, the Mac version does respond correctly to Conditional Formatting generated in Excel for Windows.

Excel for Mac also honors Visual Basic for Applications (VBA) code regardless of whether it is generated on Excel for Mac or Windows. I recorded the actions required in Excel 2016 for Windows and got this code:

Sub Macro1()
    Selection.NumberFormat = "0.00%"
    Selection.FormatConditions.Add Type:=xlExpression, Formula1:= _
        "=IF($A$1=5,TRUE,FALSE)"
    Selection.FormatConditions(Selection.FormatConditions.Count).SetFirstPriority
    Selection.FormatConditions(1).StopIfTrue = False
End Sub

Unfortunately, it doesn't work exactly right in either Windows or Mac Excel. It requires some tweaking, but proves the point that you can do it.

microsft office not on guest account - Microsoft Community

clos8 say:

microsft office not on guest account

i have microsft office on my personal account but not on my guest account how do i put it on my guest account?
frederik long say: Do you mean that there is no shortcut for MS Office while you're logged on as Guest? If so then this should help:
  1. Log on under an account with Admin privileges.
  2. Use Windows Explorer to open this folder: C:\Users\Public\Desktop
  3. Open another Windows Explorer frame to open this folder:
    c:\Program Files\Micorosoft Office\Office11
    (the "11" might vary depending on your version of Office)
  4. Locate these files:
    excel.exe
    winword.exe
    outlook.exe
    etc.
  5. Use the right mouse button to drag them into the C:\Users\Public\Desktop folder and select "Create Shortcut" in each case.
The MS Office shortcuts will now be visible to every account, including the Guest account. Note also that using the Guest account is not recommended. It is a security risk. You should disable it.

Why does Outlook 2010 download embedded pictures every time the - Microsoft Community

rmilling say:

Why does Outlook 2010 download embedded pictures every time the message is viewed?

I am looking for a way to install or configure Outlook 2010 to match behavior that I used with Outlook 2007.

Outlook 2010:  The first time that I view a message with embedded images, I get a prompt asking if I agree to allow that action.  If I approve, the pictures are downloaded then and the view is updated to show them.  I then navigate away or exit Outlook.  When I return to view that message again there is no prompt.  If connected the pictures will be automatically downloaded without my permisiion.  If not connected, the message will be displayed with just picture place-holders.  This is a significant change from Outlook 2007.

In Outlook 2007, I got a prompt to grant permission to download embedded pictures.  The message display was updated accordingly.  When I navigated away from that message display I got a prompt that the message had changed and was asked about saving those changes.  If I approved the saved message in the .pst file was updated to include the pictures.  I could then exit Outlook and return to that message and display it with the embedded pictures included, even if not connected to any network -- clearly the pictures were not downloaded repeatedly.  The behavior was good for security and for performance.

The Trust Center option settings for Outlook 2010 seem to be broken and lack sufficient controls needed to match the prior Outlook 2007 behavior that was better for security and performance.
