a. user say:

I still don't see how it is booting at all, but on systems that will NOT boot with that error, the problem is with the c:\windows\system32\winlogon.exe file.

It would not hurt to do this first:

Download, install, update and do a quick scan with these free malware detection programs (not at the same time):

Malwarebytes (MBAM):  http://www.malwarebytes.org/products/malwarebytes_free
SUPERAntiSpyware: (SAS):  http://www.superantispyware.com/

SAS will probably report a bunch of tracking cookies and you can just let it delete them.

They can be uninstalled later if desired.


Using Explorer, navigate to this folder:

c:\windows\system32

Locate the winlogon.exe file and rename it to something you can remember:

winlogon.xxx

The winlogon.exe file is one of the 3498 files that Windows File Protection looks after and now it is "missing".

If your Windows File Protection is not afflicted, it will quickly and silently replace the missing file with a fresh copy from the dllcache folder and log an event like this in the Event Viewer System log:

Event Type:    Information
Event Source:    Windows File Protection
Event Category:    None
Event ID:    64002
Description:
File replacement was attempted on the protected system file c:\windows\system32\winlogon.exe. This file was restored to the original version to maintain system stability. The file version of the system file is 5.1.2600.5512.

If, in a couple seconds, if you don't see the winlogon.exe file reappear, press F5  in Explorer to refresh the display and look for winlogon.exe to be there.

If it still does not show up (give WFP a few seconds), rename the winlogon.xxx back to winlogon.exe so you have one.  If WFP does not replace the missing file, you have some other problems.

Reboot and see how things look.
