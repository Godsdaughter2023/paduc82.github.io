Hi

Some thing is wrong with the TCP/IP settings.

Autoconfiguration IPv4 Address:169.254.1 52.88
IPv4 Subnet Mask: 255.255.0.0
Autoconfiguration IPv4 Address: 192.168.0.1
IPv4 Subnet Mask: 255.255.255.0
IPv4 Default Gateway:

This 169.254.1 52.88 is the default IP that Windows assigns when it does Not find a DHCP server, this is not a functional IP.

The second IP is the IP of the of the Wired card. That is why the Router reports conflict.

Configure both cards to full Auto Obtain, The IP and DNS.

Then do a WinSock and TCP/IP Refresh.

Type Cmd in the Start Search text box.

Press Ctrl-Shift-Enter keyboard shortcut to run a  Command Prompt as an Administrator.  Allow elevation.

Type netsh winsock reset in the Command Prompt , and then press the Enter key.

Do the same process for refreshing the TCP/IP just replace the typed command with.

netsh interface ipv4 reset   press Enter

netsh interface ipv6 reset   press Enter

ipconfig /flushdns   press Enter

Restart your Computer.

a. user say:

This could be a couple of things. First, keep it simple. Try power cycling your modem and router (or modem/router). It could also depend on the type of internet connection you have. The main issue is whether you have a dynamic IP address or static. If you have a dynamic IP (which most people do) try the following:

 

1. Hit windows key+R.

2. Type in ncpa.cpl and hit enter. This will bring you to Network connections.

3. If you are connecting via Ethernet cable right click Ethernet and select properties. If you are connecting wirelessly right click the wireless adapter and select properties.

4. Here you should see, "This connection uses the following items" . In that list scroll down to Internet Protocol Version 4. Select it (highlight it blue, but make sure the checkmark stays in the box), and select properties.

5. In the properties menu, make sure the IP address and DNS servers are set to obtain automatically. If they are set to use the following, change them to obtain automatically.

6. Click OK.

7. Check and see if the internet works now.

 

If you have a static IP try contacting your ISP to obtain what the IP address, Subnet mask, default gateway and DNS server are. You will the follow the instructions listed above only instead of selecting obtain automatically, select use the following and enter the info they provide to you.

 

If none of that works, you may have setup a dial up connection or are trying to connect via proxy. To make sure there is no dial up connections (unless you have a dial up connection):

1. Hit Windows key + R, type in ncpa.cpl and press enter.

2. If there are any other connections listed besides wireless LAN and Ethernet (IE ones that say dial up. Right click and delete them.

 

To make sure you have no proxy server setup:

1. Windows Key + R and type in control panel.

2. In control panel select internet options. (you may have to select large or small icons on the top right if it's set to category view).

3. Go to the connection tab.

4. There should be a box that says dial-up and virtual private network settings. If there is anything listed in this box and you are not using a VPN or dial up connection, remove anything listed in this box.

5. Under the heading Choose settings if you need to configure a proxy server. Select never dial a connection.

6. Click the LAN settings button and make sure nothing is selected in the box that pops up except automatically detect settings.

 

Hope this helps.

a. user say:
