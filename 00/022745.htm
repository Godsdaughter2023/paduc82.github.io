julie fka profjulie say:

A lot of WMA music that was purchased years ago, had DRM encryption on it that limits your ability to play the music on only recognized devices and players.  This wasn't usually a problem since the music lived in and was synced from Windows Media Player to other devices. When a device was connected to a computer and WMP was opened up, the device was activated and music could be synced to the device and played on the device - the DRM rights were passed on to the device.

So for years this music could be enjoyed and played without even knowing it was DRM encrypted.  But getting a new computer could often result in problems.  This sounds to me like what you experienced when you got your Windows 8 computer and used laplink to copy the music from your XP computer to your Windows 8 computer.

You can go review the properties of the music within Windows Media Player to see if this is the case with your music (right click on a song, then select Properties > Media Usage Rights tab).

I've taken the screenshot below to show you the Media Usage Rights tab of a WMA formatted song that is stored on my computer that is DRM encrypted (and that I can no longer play on my computer).

and here's a screenshot of the Media Usage Rights tab of a MP3 formatted song that is stored on my computer that has no DRM encryption and that I can play:

DRM encryption on WMA files is NOT just limited to purchased music.  You could have also ripped music from a CD in the WMA format that was copy protected.  Many users were unaware of this, but there's an Rip Music Option within Windows Media Player (click Tools > Options > Rip Music) that for many computers was set to copy protect music by default - see this screen print:

If you ripped music from CDs that wound up being copy protected, you'd need to rip the music again as an MP3 file or as WMA formated files, but with this option disabled.  Then you won't run into the problem. 

Obviously, you don't have this option for any music that you purchased several years ago that is DRM protected.

However, I tried the same tactic for the other two drop-down list content controls in my document and I receive the error message of compile error, Ambiguous name detected: Document_ContentControlOnExit' Can someone help me with this problem?

macropod say:

A simpler approach, if these are the only dropdown content controls having the conditions to be tested, would be to use code like:

Private Sub Document_ContentControlOnExit(ByVal CCtrl As ContentControl, Cancel As Boolean)
With CCtrl
  If .Type = wdContentControlDropdownList Then
    Select Case .Range.Text
      Case "Red": .Range.Font.ColorIndex = wdRed
      Case "Yellow": .Range.Font.ColorIndex = wdYellow
      Case "Green": .Range.Font.ColorIndex = wdGreen
      Case Else: .Range.Font.ColorIndex = wdAuto
    End Select

  End if

End With
End Sub

Unlike Doug's code (which is easily amended), the above also clears the colour if the placeholder text is re-selected.

Additional tests could be added to check whether the content control is in a table has a particular title or tag (the affected content controls could all have the same title or tag), and so on, without significantly complicating the code.

cody c. say: Just adding steps that explain how to reset the Excel Preferences:

Step 1: Quit all programs and close all windows 


 1. On the Apple menu, click Force Quit. 

 2. Select an application in the "Force Quit Applications" window.
Note You cannot quit Finder.

 3. Click Force Quit.

 4. Repeat the previous steps until you quit all active applications.


Step 2: Remove Excel Preferences
  1. Quit all Microsoft Office for Mac programs. 
  2. On the Go menu, click Home . 
  3. Open Library.
    Note The Library folder is hidden in MAC OS X Lion. To display this folder, hold down the OPTION key while you click the Go menu. 
  4. Open the Preferences folder. Click View , click Arrange by then Name. 
  5. Look for a file that is named com.microsoft.Excel.plist. 
  6. If you locate the file, click and then drag and drop the file to the Desktop. If you do not locate the file, the program is using the default preferences.  
  7. If you locate the file and move it to the desktop, start Excel, and check whether the problem still occurs. If the problem still occurs, quit Microsoft Excel, and restore the file to its original location. Then, go to the next step. If the problem seems to be resolved, you can move the com.microsoft.Excel.plist file to the trash. 
  8. Quit all Microsoft Office for Mac programs. 
  9. On the Go menu, click Home . 
  10. Open Library.
    Note The Library folder is hidden in MAC OS X Lion. To display this folder, hold down the OPTION key while you click the Go menu. 
  11. Open the Preferences folder. 
  12. Open the Microsoft Folder. 
  13. Look for a file that is named com.microsoft.Excel.prefs.plist.
  14. If you locate the file, move it to the Desktop. If you do not locate the file, the program is using the default preferences. 
  15. If you locate the file and move it to the desktop, start Excel, and check whether the problem still occurs. If the problem still occurs, quit Microsoft Excel, and restore the file to its original location. Then, go to the next step. If the problem seems to be resolved, you can move the com.microsoft.Excel.prefs.plist file to the trash. 
  16. Close all Microsoft Office applications. 
  17. On the Go menu, click Home . 
  18. Open Library.
    Note The Library folder is hidden in MAC OS X Lion. To display this folder, hold down the OPTION key while you click the Go menu. 
  19. Open the Preferences folder. 
  20. Open the Microsoft Folder. 
  21. Open the Office 2008 or Office 2011 folder. 
  22. Look for a file that is named Excel Toolbars (12) or Microsoft Excel Toolbars. 
  23. If you locate the file, move it to the desktop. If you do not locate the file, the program is using the default preferences. 
  24. If you locate the file and move it to the desktop, start Microsoft Excel, and check whether the problem still occurs.
