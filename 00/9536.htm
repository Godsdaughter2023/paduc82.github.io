jay freedman say:

As you've probably read in some of the forum posts and articles about AutoCorrect, Word stores only "formatted" AutoCorrect entries in a template, specifically Normal.dotm for Word and NormalEmail.dotm for Outlook. (The other templates you mentioned aren't used for anything -- they're leftovers from the upgrade process.)

All other entries, the "plain text" ones, are stored in the .acl files. Ignore the files you'll find in the Program Files or Program Files (x86) areas; those are just the factory-supplied ones. The working copy is in C:\Users\<your name>\AppData\Roaming\Microsoft\Office. (The AppData folder is hidden, so you have to go into the file manager's Folder Options dialog and select the option to show hidden folders in order to see it or search for files in it.) The file name is "MSO" followed by a number that indicates what language it's for; English (US) is 1033.

No matter how many versions of Word you have installed on the same computer (I have four), they all use the same .acl file and thus should have the same plain-text AutoCorrect entries. I don't know how you've managed not to have the same entries in both 2010 and 2013.

If it helps, you can download the AutoCorrect2007.zip file from my site, http://jay-freedman.info, extract the file AutoCorrect.dotm from it, and store that in a trusted location such as your Templates folder. Open the template in Word 2010 and click the Backup button in the box that appears. That creates a document containing all the AutoCorrect entries that Word 2010 can see. Now close Word 2010, and open the template in Word 2013. This time, use the Restore button in the box, and select the document that the backup created. Any entries that were in Word 2010 but not in Word 2013 will be added to 2013, regardless of where the .acl files are.

Default setting for Word document Mac - Microsoft Community

kingbash say:

Default setting for Word document Mac

How can I change the default settings - font, font size, line spacing - for future Word documents? I am using a Mac. 
bob jones aka: cybertaz say:

Although Phillip's suggestion is an option I'd strongly recommend against it. If it were simply a matter of Font and/or Font Size it would be fine, but you mentioned paragraph formatting attributes as well. Most of the Built-in styles in Word are based on the Normal paragraph style. If you make wholesale changes to it you can expect significant repercussions somewhere down the road. There are other reasons as well.

Two other approaches to consider:

  • Create a new Paragraph Style with the preferred attributes. Make certain to check the box for Add to template in the New Style dialog. Unfortunately, it cannot be designated as the default style, so you'll have to select it when you create new documents.
  • Create a custom template that includes your new style. Other customization (such as margin settings, paper size, etc.) can be included as well. Apply that style in the template, then save using File> Save as Template & close the template. [The location of user templates for Office 2016 is

~:User:Library:Group Containers:UBF8T346G9.Office:User Content:Templates.]

To create a new document from the template use File> New from Template.
