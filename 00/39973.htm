Regardless of the cause of error 0xC004F012 when activating Windows 10, the solution is to reconstruct the Tokens.dat file. For this, the first thing we are going to do is open the Windows 10 file browser and go to the path C: \ Windows \ System32 \ SPP \ Store \ 2.0. Once there, we rename the file tokens.dat by tokens.old
The next thing is to open a command prompt window with administrator permissions and launch the net start sppsvc command. Once it has been executed correctly, we write the command cscript.exe slmgr.vbs / rilc and launch it as well. After it finishes its execution, we restart the system twice consecutively and later, we go to the configuration page of Windows 10 and we go into Update and security> Activation and launch the problem solver of system update. This should solve our error 0xC004F012 activation of Windows 10, but if we still have problems, we can try to change the product key from the command prompt.
To do this, we close all windows and open programs that we have on the desktop and open a command prompt window with administrator permissions. jack mvp say: Hi

Virtual computers use the Host hardware but for their own inner purpose, they emulate their own Hardware.

XP-Mode is always "wired" even if it is co-tailing on a Wireless card. I.e., you cannot operate the Wireless from an XP-Mode. A Valid connection is chosen by the

Host and the Virtual computer is using a Wired Intel 100 Mb/sec. PCI card, getting its feed from the active Host connection.

When you configure the VPC set it to use the Adapter of the Host computer.

http://www.ezlan.net/Win7/XP-Mode-NIC.jpg (this is done in the VPC General Setting).

If you have a Router with DHCP it will then Auto obtain an IP like an independent computer.

Otherwise (in the XP VPC ), look at the Network adapter's properties and configured its TCP/IP like you would do with any other XP Pro Networked computer.

You can use static IP or set it to Auto Obtain.

Example, http://www.ezlan.net/example/tcp2.jpg   (this is done inside the XP-Mode whe it is running active on the screen).

Configure Static IP in Win XP, http://portforward.com/networking/static-xp.htm

-----------------

When all the above is configured correctly, the XP-Mode acts like an independent computer on the Network. It follows all the regular rules of connection to Local Networks, Sharing Files, and printers, as well as connecting to the Internet on its own.
