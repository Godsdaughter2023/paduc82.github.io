anthony.b say: One idea is to use MS Word to help to partially-automate this.  For example, if you had MS Access put an unused symbol, such as "%" (percent) for example, at the proper place in the output CSV file.  Then use Word to do a replace all, changing all the "%" to whatever format you wanted (in your case, 48 records each).  I assume that if you have Access, you probably also have Word (not always the case).

tom van stiphout say:

Another idea is:

Using a MakeTable query, copy the data to a new table.

Using a "SELECT TOP 48 * FROM myNewTable" query export 48 rows.

Using a DELETE query delete those 48 rows

Repeat until no more records.

 

The Delete step is really just a convenience. If you know a bit more SQL and/or VBA, you can eliminate this step, adding a WHERE clause to the SELECT query to ignore the records already exported.

scottgem say:
There is no need to use Word to do this. As Tom suggests, using the TOP qualifier will allow you top restrict the query to 48 records. There are several ways to eliminate previously exported records depending on your table structure and data.

zorvek say:
Public Sub DeleteRows()

    Dim Row As Long
    Dim TargetWorksheet As Worksheet
    
    Set TargetWorksheet = ThisWorkbook.Sheets("Sheet1")
    
    With TargetWorksheet
        For Row = .UsedRange.Row + .UsedRange.Rows.Count - 1 To 2 Step -1
            If .Rows(Row).Columns("E").Value = "BSDTUS33AITD" And .Rows(Row).Columns("K").Value = "CA" Then
                .Rows(Row).Delete
            End If
        Next Row
    End With

End Sub

talking moose say:
Others more knowledgeable than I will chime in here, but since you mention fonts and layout I thought I’d go with this.

Your layout issue is more than likely directly related to the fonts you used in your slides. Not all computers have the same fonts. Fonts will vary when moving your slides Mac < — > Mac, Windows < — > Windows and especially Mac < — > Windows.

Office for Mac installs a group of fonts and includes a Font Book group called “Windows Office Compatible”. You’ll find the list is pretty lengthy. Stick with these fonts and your slides should transfer much more smoothly.

In the ribbon choose the font menu > Font Collections > Windows Office Compatible.


No PowerPoint slide deck with any modicum of complexity will transfer perfectly without a lot of tweaking and testing back and forth. If you're composing on any computer (Mac or Windows) to later present on a another computer you've never used then assume you will have problems with fonts.

You can save your slides as a PDF and present using Adobe Reader or Adobe Acrobat instead of PowerPoint. This will preserve the appearance of your fonts and layout practically 100%. Keep in mind you will lose animations and other features of PowerPoint and will not be able to easily edit the PDF.

The first step would be

 

Range("C1").FormulaR1C1 = "=COUNT(R[2]C:R[19999]C)/14"
etc.

 

But since your formulas are all the same, you can simplify this to

 

Range("C1:Z1").FormulaR1C1 = "=COUNT(R[2]C:R[19999]C)/14"

 

where C1:Z1 is the range you want to add the formula to.

 

By the way, this looks like a question about Excel, not Word, and about the full version, not the Starter Edition (there's no VBA in the Starter Edition)

charlie_wright say:

To avoid Selecting each cell, you could write the formulas directly:

    Range("C1").FormulaR1C1 = "=COUNT(R[2]C:R[19999]C)/14"
    Range("D1").FormulaR1C1 = "=COUNT(R[2]C:R[19999]C)/14"

 

But I suspect it's the numerous calculations that are slowing you down - since calculating the results of all of those Count formulas will eventually eat up a lot of CPU.  Try turning off Calculation while you put the formulas in place, then turn it on again to do a single re-calc after you're done:

 

    Application.Calculation = xlCalculationManual
    Range("D1").FormulaR1C1 = "=COUNT(R[2]C:R[19999]C)/14"

   Application.Calculation = xlNormal

 

Also, you might consider turning off Screen Updates, which also slow things down considerably.  Try this:


Application.ScreenUpdating = False
(all your code here)

Application.ScreenUpdating = True

stephane zwel say:

Theme solstice was part of Office 2007 and Office 2010 but no longer part of Office 2013.

Thing is, anyone can export an existing theme as a .thmx file and then import it.

Here is ThemeSolstice.thmx : http://www.arstdesign.com/papers/ThemeSolstice.thmx

Download this file somewhere on your hard drive.

You can then start Excel 2013, click on Page Layout, then on Themes, then Browse for Themes and pick the .thmx file on your hard drive.

The theme gets applied to the current spreadsheet, but what you want is the Solstice icon so you can use it whenever you want. For this, click on Save Current Theme, enter Solstice.thmx and click OK. This saves the theme in a special folder only known by Office applications.

When this is done once, the Solstice icon appears in your existing Themes.

As for why Microsoft chose to dismiss Solstice in Office 2013, it is a good question for someone from the Office group.

palmamaiorano say:

Follow these instructions, whether you are missing themes from Excel, Word, PowerPoint etc.  Too bad Microsoft EXPERTS failed to give you this information!

Step 1:

Open a new Document, Excel, Word, PowerPoint, (Whichever you are missing themes.)

Step 2:

From the "File" Tab, select "Options" towards the bottom of the selection bar.

Step 3:

Select "Trust Center" then select "Trust Center Settings" button.

Step 4:

On the left column, select "Privacy Settings."

Step 5:

Uncheck The box that states, "Allow Office to connect to the Internet: and then click on the "OK" button.

Step 6:

Open the new document and under the Design, or Themes dropdown menu, select and put a check in the box that states, "Enable Content Updates."  A window will open asking if you want to allow Office to connect to the Internet, select "OK."

Step 7:

Exit the application and then reopen it.  All required themes should be available under the Themes.  NOTE: It may take a few minutes, but for me, it was more like seconds.
