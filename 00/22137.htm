Inserting a Calculated Field in a Query

I have a query that has 3 of the following fields (among others): Net value, Month & Year.

djsteele, mvp say:

If I'm understanding you correctly, try:

SELECT Sum([Net value]) AS [Q1 2010]
FROM MyTable
WHERE MyYear = 2010
AND MyMonth IN (1, 2, 3)

If you need to include other fields than simply the total, you'll need to use a GROUP BY clause:

SELECT Field1, Field2, Field3, Sum([Net value]) AS [Q1 2010]
FROM MyTable
WHERE MyYear = 2010
AND MyMonth IN (1, 2, 3)
GROUP BY Field1, Field2, Field3

Note: Change your field names from Year and Month. Those are reserved words, and you should never use reserved words for your own purposes. For a list of names to avoid (as well as a link to a free utility to check your application for compliance), check what Allen Browne has at http://www.allenbrowne.com/AppIssueBadWord.html

john w. vinson say:

Well, you really don't want to store data (a time period) in a fieldname. Perhaps you want a quarter and an amountprinted on a report? If so you could use a Crosstab Query to dynamically display the quarter based on the date, showing the total. You could use an expression such as

DateSerial([Year], [Month], 1)

to generate a date/time field, and use the Format() function to cast it as year and quarter.

Note that Month and Year are reserved words in Access and may cause problems as fieldnames.

jllatham say:

Mike H.. has explained it pretty well.  Another option for dealing with restructuring your worksheets is to "map" the structure out using Const values in a module that can be altered when you restructure the worksheet.  I actually use this method often to make updating the code easy in these situations.  Actually, I quite often have a separate code module that I usually call "Declarations" that contains nothing but a bunch of Public Const values.  Example:

Public Const nameCol = "D" ' column on Sheet1 that holds customer names

then anywhere in the code, when I need to reference that column, you might see a formula like:

Sheet1.Range(nameCol & 1).Select ' go to Name label row

or similar other actions.

If a new column is inserted on Sheet1 anywhere to the left of column E, which would push the original D over to become E, then all that's needed to keep the code functioning properly is to redefine nameCol as:

Public Const nameCol = "E" ' column on Sheet1 that holds customer names - revised in v2.1.0

If a piece of code refers to a particular column then after a row insert it will still refer to the same column and not increment.

 

What you could do is on the worksheet create a named range (say) MyRange which refers to (say) D1:D20. Now if we inser a column the named range moves right but any code referring to it will pick up the named range in it's new position. For example this

 

Sheets("Sheet1").Range("Myrange").Select
