lemp say:

If neither "Allow" nor "Deny" is checked (e.g., Full Control, Modify, and Write in the above figure), the selected user or group does not have permission to do the specified action.  For example, user AJones does not have permission to modify the object.

An explicit "Deny" will override an "Allow" permission.  A "Not-Allow" (i.e., neither box is checked) does not.  See http://www.ntfs.com/ntfs-permissions-allow.htm

For example, if the "Administrators" group has "Full Control" permission of a file and BSmith is a member of the Administrators group, BSmith will have "Full Control" of the file.  If you then set "Deny" "Full Permission" for BSmith for that file, BSmith will no longer have permission, even though she is still a member of the Administrators group.  Windows will warn you about this if you try to set a "Deny" permission:

You might want to read the entire topic, starting with http://www.ntfs.com/ntfs-permissions.htm
