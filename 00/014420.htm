Character encoding misteriously vanished - Microsoft Community

james kurose say:

Character encoding misteriously vanished

Hi again, guys.

Ok, so the thing goes like this. I have Windows Vista SP2 and Internet Explorer 8. Recently, I have noticed that japanese websites that use japanese character encodings, such as Japanese (EUC), Japanese (JIS) and Japanese (Shift_JIS), show garbled text instead of japanese characters. I assumed that was caused by the encodings being set to Unicode, so I went to the encoding menu to change them to any of the character encodings mentioned before and found that no japanese character encodings were present, for they have vanished.

After my initial surprise, I noticed other eastern asian character encodings were still present (korean, chinese, taiwanese, etc.) and, as puzzled as I was, decided to uninstall Internet Explorer 8 to see if the problem was browser-independent. Found the same result after a succesfull uninstall (using Internet Explorer 7) and even after clean installing again Internet Explorer 8.

Knowing that Windows Vista has eastern-asian language support already built-in, I have found no way to repair or reinstall them to see if the problem could be solved that way. My father has a laptop with Windows Vista SP2 and Internet Explorer 8, and he has no problems at all. My mother has a laptop with Windows XP SP3 and Internet Explorer 7, again without problem. I have a laptop with Windows 7 and Internet Explorer 8, still without problem. This means that whatever caused the problem was from my own desktop computer.

Oh, almost forgot, list of stuff that is present on this computer:

  • Windows Vista with SP2
  • Internet Explorer 8 (also tried with Internet Explorer 7)
  • Panda Antivirus Pro 2010 (it has been the culprit of everything a lot of times so, why not add it?)

I need those japanese encodings, as I have the habit of browsing japanese websites to keep my japanese reading skills still working. Any help would be appreciated.

james kurose say:

Ok, after a time browsing and researching, discovered that files responsible for character sets (a.k.a. codepages) are placed at C:\Windows\System32\ and bear the extension '.nls'. After that, I began to search for places where info of those files or the files themselves got corrupted, and found that info about them is stored at HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Nls\CodePage, at Windows registry.

Knowing that the japanese codepages file is called 'c_932.nls', I searched for it and found it at the directory mentioned previously. It wasn't modified since it's creation so I theorized that the problem had to be the registry. Searching the registry at the location previously mentioned, found that there was a value for every codepage... except the japanese codepage. So I added a value named "932", with "REG_SZ" as the Date Type and "c_932.nls" as the string. After that, everything seems to work just fine and I am able to read japanese sites again without problems.

To be honest, this solution wasn't discovered by me, but was provided by a Microsoft's Knowledge Base article (who would have thought that a solution made for old Windows NT would work for Vista?).
