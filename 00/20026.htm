girish m say:

Hi David,

 

Thank you for asking Microsoft Community.

 

Configuring the same email account as both MAPI and POP 3 is indeed redundant.

 

Since you want the data to be same in Windows Live mail, Outlook 2010 and Hotmail, it is better that you configure the email account as a MAPI account in Outlook.

 

Before you remove the instance of the email account which has been configured as a POP 3 account, set the MAPI data file as default. To do that, do the following:

Click on File, Account setting, Account settings.

In the Account settings window, click on the Data files tab.

Click to select the MAPI Data file and then click on ‘Set as Default’.

 

Once done, set the MAPI account as default. To do that, do the following:

Click on File, Account setting, Account settings.

In the ‘Account settings’ window, click on the ‘Email’ tab.

Click to select the MAPI email account and then click on ‘Set as Default’.

 

 

Once done, remove the POP 3 configuration. To do that, do the following:

Click on File, Account setting, Account settings.

In the ‘Account settings’ window, click on the ‘Email’ tab.

Click to select the POP3 email account and then click on ‘Remove’.

 

Once you have done that, you can manually move / archive the data from the MAPI account data file to your existing .pst file, or you can export the data from time to time as a different .pst file to keep as a backup.

 

Since your existing data would be synchronized with the Windows Live server, keeping a local backup also becomes redundant in most circumstances.

http://answers.microsoft.com/en-us/office/forum/office_2010-outlook/sync-my-pop3-and-mapi-calendar-and-contacts-in/7bbc235f-d815-41fa-baea-8e25eb599f2f

daniel g. grau say:
You are right: sending the picture to the back will make it almost impossible to select it. Think along the lines of the software designers: "If we let you format pictures placed behind text, then we'll have to let e-e-e-everybody format pictures behind text." That's why they omitted a very intuitive tool that you can add to a toolbar: the Select Drawing Objects tool.

Actually, you don't need it, but it's a nice tool to have around. The reason is, they don't tell you that when objects become framed with the selection color, you can click on them to select them. Select your table and try it. However, I don't know how to select several object at once, because shift-clicking does not work.

If you want to add the tool, read on. Else, skip to step 3, but bear in mind that my instructions refer to the new tool.

I don't recommend modifying either of the two built-in toolbars (Formatting and Standard). The reason is, it can become confusing if you later need to explain to someone else how to do something by using those two toolbars, because theirs will differ from yours.

1) Create a new toolbar
  • In the main menu bar, select Tools > View > Customize Toolbars and Menus.
  • At the top of the dialog box, select the Toolbars and menus tab.
  • Click on the New button at the top right. Give the toolbar a name and press OK. Pay attention, as the new toolbar will be hard to see it because it's small. You can move it around by grabbing it from the "colon" below the Close circle.

2) Add the missing selection tool
While still on the Customize... dialog box:
  • Click on the Commands tab at the top.
  • In the "Categories" list to the left, click on "All Commands" to select it. This will list just about everything Word has to offer.
  • In the list to the right, scroll to "SelectDrawingObjects."
  • Click on it and drag it to the new toolbar.

This new tool also let's you select several objects—click anywhere but on a picture and drag the selection rectangle.

3) Place the picture for the first cell
Let's concentrate on the first label for now.
  • Click on the new Select Drawing Objects tool (if the tool is enabled, it becomes darker).
  • Click on a picture and Cut it to the clipboard.
  • Deselect the tool, click on a paragraph in the top-left cell and paste the picture. This will anchor it to that paragraph.
When you paste a picture as a floating object, it becomes attached to a paragraph (the "anchoring paragraph"). When you copy the complete cell, you will also be copying the attached picture.

4) Position the picture
Select the picture and change its position by using the cursor arrows. (If you use the mouse, the anchoring paragraph will change!) If the picture moves in large steps, press the Option key together with the arrows.

5) Copy the first label
Once you have your first label ready, copy it onto the rest of the cells. Make sure the new tool is deselected and then click to place the cursor on the first cell:
  • In the menu bar, choose Table > Select > Cell to select the contents of the first cell.
  • Copy to the clipboard.
  • Now do Table > Select > Column (to select the first column) and Paste.
  • Repeat for the second column of labels.

6) Cell display bug
Depending on how your picture is positioned and on the vertical alignment of the cell, you may experience a bug. If the cell's contents  are bottom- or middle-aligned in the vertical direction (as in the default label), the cells in the last row fail to display their complete contents. If this happens, you'll have to slightly redesign the first cell (and then repeat the copy operation). To do this, we need to cancel the vertically centered alignment:

  • Place your cursor on the first paragraph of the first cell. In the Ribbon, click on the purple Table layout  tab. You should see an "Align" tool to the right. If you don't see it, resize your document's window (grab it from the bottom right corner).
  • Click on "Align"  and select either of the first three choices to align the cell's contents against the cell's top border.
  • To adjust the vertical position of the text in your cell, click on the Home tab in the Ribbon. Use the tool with the upward-pointing pink arrow to increase the Space Before parameter for the first paragraph. You can fine-tune it by typing your own number.
  • If the picture has moved out of position, correct it by using the arrows.
  • Repeat the cell-copy operation (step 5).

scottgem say:
The problem is you are formatting the control as a date value, not a time value. The DateDiff does not return a time value. It returns the number of minutes as an integer value. To get it to be a time value you would have to divide by 1440 (the number of minutes in a day).


walt_k say: Thanks - This is the final command that I used:

(DateDiff('n',[Forms]![Retail SLA Form]![SLA Time],[Forms]![Retail SLA Form]![SLA End Time])/1440)

It now returns the correct time value.
