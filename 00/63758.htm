kirkvette say:

Split from this thread.

I will be happy to assist you in resolving your issue with regard to being unable to open email messages in Windows Live Mail.

It is possible that Mail.MsMessageStore file was corrupted. Please follow the steps below on how to repair the file:

1. Restart the computer (this is to ensure that Live Mail is fully shut down, and that none of it's components are in use when the computer starts)

2. Click Start, type CMD but do not press enter

3. You will see cmd.exe in the search results, right click on it and select Run as Administrator - type your admin password if asked, and if UAC pops up asking permission, allow it

4. You should now see the command prompt, with the following displayed:

Microsoft Windows [Version 6.1.7600]

Copyright (c) 2009 Microsoft Corporation. All rights reserved.

C:\Windows\system32>

5. Copy the following (press Ctrl+C):

esentutl.exe -d "%userprofile%\AppData\Local\Microsoft\Windows Live Mail\Mail.MSMessageStore"

Note: the "" are necessary

6. Right click in the command prompt Window, and select Paste, so you now see the following:

Microsoft Windows [Version 6.1.7600]

Copyright (c) 2009 Microsoft Corporation. All rights reserved.

C:\Windows\system32>esentutl.exe -d "%userprofile%\AppData\Local\Microsoft\Windows Live Mail\Mail.MSMessageStore"

7. Press enter. esentutl will now defragment the Live Mail database file

8. Once done, you can close the command prompt, and restart the PC again

¡firedog say:

I haven't been able to document a single instance where this defragmentation routine has solved the problem of not being able to open some messages. It may well be that it has helped in some cases, but if it has, sadly the user concerned hasn't come back to say so.

These are the instructions for recovering a dirty database on Windows 7:

Close down all running programs.

  1. In Explorer (Windows key + E), navigate to
    C:\Users\{user}\AppData\Local\Microsoft\Windows Live Mail
    where {user} is your Windows user name. This is your store folder where all your messages and account details are stored. Shift-right-click on the folder name and select Copy as path.
  2. Type cmd into the start search box and press Ctrl-Shift-Enter (all three keys simultaneously). You'll have to give UAC permission, and then this should open an administrator's command window with a prompt like C:\Windows\system32>
  3. Type cd and a space at the prompt, then right-click and select Paste. Press Enter. This should change the prompt to
    C:\Users\{user}\AppData\Local\Microsoft\Windows Live Mail>
  4. Back to Explorer. Copy the whole folder to somewhere safe - e.g. your Documents folder -  so we can just put it back if everything goes pearshaped. This might take some time if you have a lot of messages.
  5. Back to the command window. Type very carefully into the window at the prompt
    esentutl /r edb
    Note the spaces on either side of /r.
    This should     recover your database, i.e. restore it to the state it would have been in had the last close-down process not been interrupted. With any luck, you'll see a message about success, something like Now let's try to repair the database.
    In the command window, type at the prompt
    esentutl /p mail.msmessagestore
    Again, there are spaces each side of the switch /p.
    You may have to dismiss a warning about potential loss of data. This might happen; you may lose the last few messages and transactions (moves, deletes, replies etc.) before the database went south. It's presumably a small price to pay for getting the thing working again.
    If you see a message saying     Operation completed ..., you're probably in luck. You will probably also see a message about backing up the database. Let's leave that to Windows Live Mail.
  6. In the command window, type Reboot the computer, and then try launching Windows Live Mail.
