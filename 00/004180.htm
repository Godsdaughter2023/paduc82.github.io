I have 2 email addresses and they are both IMAP. There was more than 1 contact account, but only 1 had any contact info in it, but it wouldn't sync to the email so I deleted them. Even when I reload the contacts from my back up, they don't sync with email.

#1 - Contact data does not sync with the server for Imap or POP3 accounts. Nothing to do with Outlook, that is how the "industry standard" protocols work regardless of which email client is used (Outlook is just one of many)

#2 - The reason that you ony had a contact folder in one of the IMAP folder trees is that the first Imap account configured would have its .ost data file set as the default data file for the profile where the default contact/calendar folders are stored. Subsequent Imap accounts would all use the default data file for contact/calendar info and as such would not have any contact/calendar folders in the applicable folder tree.

Things to keep in mind to mitigate losing contact/calendar data when configured using IMAP accounts.

#3 - Contact/calendar folders in an IMAP folder tree would have the added words "This Computer Only" in the folder description. If Outlook cannot find/opened the .ost file for a given IMAP account, it would recreate it, re-sync the email folders but the "This Computer Only" folders would be empty. The .ost file is also automatically deleted from the system if the IMAP account is removed from the Outlook profile

#4 - To avoid the above issue, suggest the following
a) create a new PST file for the profile (go to <Home Ribbon> --> <New Items> --> <More Items> --> Outlook data file
b) when above is done then go to File --> Account Settings --> Data FIles tab --> select the newly created PST file and set it as the default for the profile
c) Copy the contacts from the "This Computer Only" folder to the contact folder in the newly created PST file. PST files are never automatically deleted from the system by Outlook.

Even when you choose to delete your out-of-date security info, MS don't fully remove it for 30 days.
