ajgl say: The problem occurred on each of 7 machines that loaded the MS Office 2010 service pack update. It did not occur on any machines not updated by Office. Investigation showed that the problem only manifested in the account that was active at the time of the update, other accounts on the same machine were unaffected. System restore did not solve the problem.

Investigation showed the settings on all accounts to be the same, there was no obvious reason for the language display issue on the damaged account.

The ultimate resolution turned out to be this:

Create a new account for the affected user. 
Delete all temporary files in C:\users\{affected user}\appdata\local\temp
Delete all temporary files in C:\Windows\temp
Create new user account
Copy all files from C:\users\{affected user}\appdata\local\temp to the new account
Copy all files from the new account back to C:\users\{affected user}\appdata\local\temp

This fixes the problem.
Copying files without deleting temp files does not work. Likewise, simply deleting temp files alone does not work. 

Makes sure to empty or delete browser cache files, my docs and my pictures etc, or else the copy process will take for ever.

Why go through this? It saves having to rebuild Outlook and other programmes registered to that specific user account.
