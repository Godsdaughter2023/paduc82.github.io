malkeleah say: Some reasons for computer slowness (with comments):

1. Computer hasn't been maintained - http://www.elephantboycomputers.com/page2.html#Maintenance
This is probably not your issue since the computer is new.

2. Computer is infected with malware - http://www.elephantboycomputers.com/page2.html#Removing_Malware
You can get infected the first day, so this is worth checking out as I infer from your post that the computer wasn't always slow.

3. Hard drive is too full. Remove unnecessary stuff, uninstall unused programs. Don't forget to back up!
http://www.elephantboycomputers.com/page2.html#Backing_Up
Unlikely on a new computer, but you never know.

4. Flaky third-party software. Toolbars and add-ons are particular culprits in this area. Uninstall toolbars (Google, Yahoo, ISP-branded, etc.) and look carefully at what add-ons are in use in your browser(s).
A definite possibility depending on what you've installed.

5. Computer has too many unnecessary programs/processes running in the background. Manage your Startup.
This could be your case, particularly if you didn't optimize your computer and get rid of unnecessary junk preinstalled by the computer mftr.

For Vista/Win7 - Start Orb>Search box>type: msconfig and when it appears in the Results box above, right-click and choose "Run as Administrator". Autoruns is also very useful in Vista.

6. User is running a bloated/invasive antivirus program such as ones from Norton and McAfee. Replace with a better program. I recommend either NOD32 (commercial) or Avast (free). Avira is also good but the free version has an unpleasant nag screen (Google for instructions as to how to disable this). User may also be running more than one real-time antivirus/firewall/security program.
Could be applicable.

7. User has installed new programs that are processor and/or memory-intensive (Photoshop, AutoCAD, Mathmatica or the like) and doesn't have the necessary hardware power.
Could be applicable.

8. Hard drive is failing and is in PIO Mode. This is very dramatically slow (like being back in 1985). See Hans-Georg Michna's information here - http://winhlp.com/?q=node/10
Unlikely, but if hardware is going to fail it will usually do so quite quickly or go for years. So while troubleshooting the software end first makes the most sense in your case, you can't rule out hardware either.

In addition to the above, also see http://miekiemoes.blogspot.com/2008/02/help-my-computer-is-slow.html

a. user say:

'Setup' is the colloquial name for a software installer.

Are you running Setup from either a USB flash drive or a DVD? That's really how you should be doing a clean install. Doing that completely bypasses the hard drive, so if you see any error messages, they're coming from Setup itself.

It's possible that something went wrong when you created that USB flash drive or DVD, so you may have to redo it. That's something I learned the hard way from reading about it.

Also, when you do a clean install there should be nothing attached to your computer except for a keyboard and a mouse. If you have an old-school wired keyboard and mouse, so much the better. You don't need to be connected to the internet to do a clean install, so if you're using WiFi and your WiFi adapter is removable, remove it.

rohn007 say:
It could be that system restore points have been turned off:
in an administrator ID (you can't make this change in a normal id) go to Control Panel > System > Advanced System Settings > System Protection tab.

Here you will be able to see if it is turned on, and for which drives.

If you need to turn it on, click on the desired drive in the "Protection Settings" box, then click on "Configure" button.

Adjust disk space you will allow check points to use, then make sure that the "Turn on System Protection" radio button is selected. Click on the apply button, then OK button. Back in the "System Properties" dialog, click on the "Create" button to make your first restore point.
