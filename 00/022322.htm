yaqub k say:

Hi Derek,

 

Script error messages are displayed by Internet Explorer when there is a problem with the JavaScript or VBScript code on the website you are viewing. Occasionally a script error can be caused by an error in downloading a webpage, but more often it is an error in the webpage itself.

1.    When you say “when I exit a program “, which program are you referring to?

2.    Which web browser are you using to access Internet?

 

If you are using Internet Explorer, I would suggest you to follow the methods mentioned below.

Method 1: Disable script debugging

To turn off notification about every script error on Internet Explorer 11, follow these steps:

a.       Open Internet Explorer.

b.      On the Tools menu, click Internet Options. If you can't see the Tools menu, press Alt to display the menus.

c.       On the Advanced tab, click to clear the Display a notification about every script error box, and then click OK.

d.      Also turn off the Scripting Debugger, On the Advanced tab, click to select the Disable script debugging box for both Internet Explorer and other and then click OK.

 

Method 2: Test a webpage from another user account, browser, or computer


To try to narrow down the source of the problem, use another user account, another browser, or another computer to view the webpages that triggered the script error.

If the script error persists on multiple webpages, try this method. If the script error persists on a particular webpage, there may be a problem in the way that the webpage is written. You can probably ignore the error and disable script debugging as described in Method 1.

If the script error does not occur when you view the webpage through another user account, in another browser, or on another computer, the problem may be caused by files or settings on your computer. In this case, go to Method 3 to continue troubleshooting.

 

Method 3: Verify that Active Scripting, ActiveX, and Java are not being blocked by Internet Explorer

Active scripting, ActiveX, and Java are all involved in shaping the way that information is displayed on a webpage. If these features are blocked on your computer, this may disrupt the display of the webpage. You can reset your Internet Explorer security settings to make sure these features are not blocked. To do this, follow these steps:
a. Open Internet Explorer.

b. On the Tools menu, click Internet Options. If you can't see the Tools menu, press Alt to display the menus.

c.  In the Internet Options dialog box, click the Security tab.

d. Click Default Level.

e. Click OK.

Note ActiveX controls and Java programs are turned off in the High security level in Internet Explorer.

 

Method 4: Remove all temporary Internet files


Every time that you open a browser to view a webpage, your computer stores a local copy of that webpage in a temporary file. If the size of the temporary Internet files folder becomes too large, some display problems may occur when you open webpages. Periodically clearing the folder might help to resolve the problem.
To remove all the temporary Internet-related files from your computer when you use Internet Explorer 11, follow these steps:

a.  Open Internet Explorer.

b.  On the Tools menu, click Internet Options. If you can't see the Tools menu, press Alt to display the menus.

c.  Click the General tab.

d.  Under Browsing History, click Delete.(http://support.microsoft.com/Library/Images/2677797.jpg)

e.   In the Delete Browsing History dialog box, click to select the following check boxes, and then click Delete:

Temporary Internet Files

Cookies

History

f.  Click Close, and then click OK to close the Internet Options dialog box.

 

 

Please let us know if the issue persists. We will be happy to assist you further.

ken sheridan say: You say that the comments are entered into a column of short text data type, and also logged in a separate table.  This introduces redundancy and the consequent risk of update anomalies.  There should be no column for comments in the referenced table to which the form is bound, only in the referencing Commits table.

One way to do this would be to include an unbound text box control, txtComment say, in the form and set it to NULL in the form's Current event procedure.  In the form's BeforeUpdate event procedure set the return value of the Cancel argument to TRUE if the unbound text box is NULL, otherwise append a row to the referencing Comments table when the current changes to the record are committed to the table, e.g.

    Const MESSAGE_TEXT  = "A comment must be entered."

    If IsNull(Me.txtComment)  Then
        MsgBox MESSAGE_TEXT, vbExclamation, "Invalid Operation"
        Cancel = True
    End If

Then in the form's AfterUpdate event procedure:

    Dim strSQL As String

    strSQL = "INSERT INTO Comments(MyID,Comment) " & _
        "VALUES(" & Me.MyID & ",""" & Me.txtComment  & """)"
    CurrentDb.Execute strSQL, dbFailOnError

Where MyID is the name of the primary key of the form's table, and the name of the corresponding foreign key in the Comments table.

If you wish to show the comments related to the current record in the form, this can be done in a subform, linked to the parent form on the keys, MyID in the above example.  Requery the subform in the form's AfterUpdate event procedure, after the above code, to get it to include the newly added comment.
