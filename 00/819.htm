ronrosenfeld say:

A few questions:

  • I understand that you want to copy the entire row if the value in column B is found anywhere in column A.   What if the value in column B is NOT found in column A, but is found elsewhere in column B?  (i.e. duplicated but only in column B?)  Should that be copied also?
    • In other words, in your example workbook, B10:B11  and B12:B13 are duplicated within column B, but these values are NOT found in Column A.  So should these lines be copied also?
  • Is it only the cell values that need to be copied?  Or are there formulas within the rows that also need to be copied?  Copying only values may be much faster.

If you only need to copy values, and if the rows to be copied exclude duplicates found only within column B, then, with your Source data on "Sheet1", the following macro will copy the 20,000 rows where the number in column B is found in column A, over to sheet2 in about ten seconds.

If you need to also copy formulas, a minor re-writing of the code below enables that to be accomplished in about 75 seconds.  Let me know.  (I tested by adding a tenth column of formulas to the data you provided, re-writing the code to copy the entire cell, rather than just the Value, and timing it).

You may have to change the sheet names in the code to match your workbook.  Also, I only copied over nine columns as you had in your example -- you may need to change that.

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.

========================================

'If a column B number appears anywhere in Column A
'  then copy that row to new sheet
' this version copies only the values, no formulas

Sub CopyDupRows()
    Dim RW As Range
    Dim rSrc As Range, wsSrc As Worksheet
    Dim vSrc As Variant, vRes() As Variant
    Dim colDupRows As Collection
    Dim colDups As Collection
    Dim I As Long, J As Long
    Dim K As String
    
    Dim wsRes As Worksheet
    Dim rRes As Range
    
Set wsRes = Worksheets("Sheet2") 'Results go here
Set rRes = wsRes.Range("A2")
    
Set wsSrc = Worksheets("sheet1")
With wsSrc
    Set rSrc = .Range("a1", .Cells(.Rows.Count, "B").End(xlUp)).Resize(COLUMNSIZE:=9)
End With
vSrc = rSrc

'Collect column A values
Set colDups = New Collection
On Error Resume Next
For I = 1 To UBound(vSrc)
    colDups.Add vSrc(I, 1), CStr(vSrc(I, 1))
Next I

'collect rows where number in column B is found in column A
Err.Clear
Set colDupRows = New Collection
For I = 1 To UBound(vSrc)
    K = CStr(vSrc(I, 2))
    colDups.Add vSrc(I, 2), K
    If Err.Number = 457 Then
        Err.Clear
        Set RW = rSrc.Rows(I)
        colDupRows.Add RW
    Else
        colDups.Remove (K)
    End If
Next I
On Error GoTo 0

'results into an array
ReDim vRes(1 To colDupRows.Count, 1 To colDupRows(1).Columns.Count)
For I = 1 To UBound(vRes, 1)
    For J = 1 To UBound(vRes, 2)
        vRes(I, J) = colDupRows(I).Cells(1, J)
    Next J
Next I

Application.ScreenUpdating = False
Set rRes = rRes.Resize(UBound(vRes, 1), UBound(vRes, 2))
With rRes
    .EntireColumn.Clear
    .Value = vRes
    .EntireColumn.AutoFit
End With
Application.ScreenUpdating = True
    
End Sub

scottgem say:

I would do this with multiple queries. I would create one query that returned the value for the last day of the previous year something like this

 

Select DateAdd("yyyy",1,Dates) AS Matchdate, PXlast AS Prev

FROM StockIssues

WHERE Month(Dates) = 12 AND Day(Dates) = 31;

 

Then another query joining the table to this query.

 

SELECT Dates, PXLast, Prev

INNER JOIN on StockIssues.Dates = qryPrevious.MatchDate

FROM StockIssues, qryPrevious

ORDER BY Dates;

 

This will show the current and previous end of year balances so you can do your percent changes
