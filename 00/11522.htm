You say "brackets", but show parentheses. According to the RFC for email addresses, parentheses are allowed, but designate comments. But I can help you with a workaround for this. If the event fires 2 or more times the Target.Value and Target.Address is allways the same. So we can store the old values and compare them if the event fires the next time. If they are equal we can exit the event. Simple but effective. :-)

Andreas.

Option Explicit

Dim LastAddress As String, LastValue As Variant

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim I As Long, J As Long
  If LastAddress = Target.Address Then
    If IsArray(LastValue) Then
      For I = 0 To UBound(LastValue)
        For J = 0 To UBound(LastValue, 2)
          If LastValue(I, J) <> Target(I, J) Then _
            GoTo RunChange
        Next
      Next
    Else
      If LastValue <> Target Then _
        GoTo RunChange
    End If
    Exit Sub
  End If

RunChange:
  MsgBox "Sheet RunChange macro fired"
  Application.ScreenUpdating = False
  MsgBox "Sheet updating inhibited"
  Application.ScreenUpdating = True
   LastAddress = Target.Address
  LastValue = Target
End Sub

As a workaround you could uncheck "Show error alert after invalid data is entered" in the Error alert tab of Data Validation then use code in the Change event to test for invalid data as follows.

Private Sub Worksheet_Change(ByVal Target As Range)
On Error GoTo ReEnableEvents
Application.EnableEvents = False
Dim rngToFind As Range

If Target = Range("A1") And Target.Value <> "" Then
  Set rngToFind = Range("Fruit") _
    .Find(What:=Target.Value, _
    LookIn:=xlFormulas, _
    LookAt:=xlWhole, _
    SearchOrder:=xlByRows, _
    SearchDirection:=xlNext, _
    MatchCase:=False)

    If rngToFind Is Nothing Then
      MsgBox "Invalid entry. Select from dropdown list only"
      Target.ClearContents
      Target.Select
    End If
End If

ReEnableEvents:
Application.EnableEvents = True
End Sub

Any time you change the program you are publishing with, or the computer you are publishing from, the first thing you need to do is get the local and remote versions of your site in sync with each other.

Step 1 (which you have already done), is to open www.tipsandiego.org in Expression Web and do a full site publish down to your PC.
Step 2 is to open the new version of your site on your PC and do a full site publish back up to www.tipsandiego.org
Step 3 is to publish from your PC to www.tipsandiego.org, using the "changed pages only" option - there shouldn't be any changes, and this means that the 2 versions of your site are now in sync.
Step 4 now you are ready to start making changes to the site on your PC and publish.

It's also recommend that you keep a backup copy of your site in case something goes wrong.

Use the Find and Replace facility.

Click on the More button on the Replace tab of the Find and Replace dialog and then with the selection in the Find what control click on the Format dropdown at the bottom of the dialog and then on Font and then in the Find Font dialog, select the font that has been applied to the text for which you want to change the size.  In your case, it maybe from the Complex scripts dropdown.  Then click on OK to dismiss the Find Font dialog.

Then back in the Find and Replace dialog, click on Replace All
