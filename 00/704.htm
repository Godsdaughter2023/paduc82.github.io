Substitute rangeofcells with your actual range , A1:a1000, for example and then array enter the formula by pressing control-shift-enter. If Excel accepts it as an array formula it will surround the formula with braces {  } in the formula bar.

So then I turned off all of Avast’s shields, expecting the protection status to be updated for both Security Centers – but this is what I got:

Now, the documentation seems to suggest that there should be a connection, and a commonality, between the two Security Centers with regard to monitoring the protection status of third-party AV apps:

The Windows Defender Security Center app uses the Windows Defender Security Center Service (SecurityHealthService or Windows Security Health Servce), which in turn utilizes the Security Center service (wscsvc) to ensure the app provides the most up-to-date information about the protection status on the endpoint, including protection offered by third-party antivirus products, Windows Firewall, and other security protection.

https://docs.microsoft.com/en-us/windows/threat-protection/windows-defender-security-center/windows-defender-security-center

So I was wondering whether this issue with the protection status not updating might be something that was specific to Avast, or if it was something more general. To answer that question, I installed AVG Free on another PC that's running the Creators Edition. But once again, I got the same result – turning off both AVG’s file shield and its behavior shield resulted in a notification from the old Security Center (Security and Maintenance), but the Windows Defender Security Center never noticed that the primary (real-time) AV app had been turned off. So it’s beginning to look like this might be a general issue with the Windows Defender Security Center. Of course you could argue that the Windows Defender Security Center is designed exclusively for Windows Defender users, and that there’s no good reason for it to extend the courtesy of monitoring the protection status of third-party AV apps. But that surely goes contrary to the spirit of building an all-inclusive health monitoring app for Windows 10. At the very least, people who use third-party AV apps need to be aware of the situation.
