john mcghie say:

I would say that's a "Style Separator", a deprecated function from Word 2000-ish.

It's the "paragraph mark you have when you are not having a paragraph mark."  It's purpose was to enable the user to create "run-in" headings by applying a different (paragraph) style to text between the left edge of the paragraph and the Style Separator.  These days, the function is fulfilled by the Linked Style, which enables the user to apply just the Character Style component of a paragraph style to a selection of text.

When you add a "page break", Word adds a paragraph mark after it for internal reasons I do not understand.  Consequently, you will get an extra paragraph mark in the text, immediately after the page break.

Sadly, I have no idea why YOU are getting a Style Separator there :-)  I suspect that's a document that was created in an earlier version of Word that had one in it.  I can tell that it is a style separator because it has text appearing to the right of it: that's not possible with a paragraph mark.

However, if you carefully select everything Except the character immediately to the right of ther style separator and copy, you should be able to paste without it.

There is no issue having multiple people using the database simultaneously!  That said, you must first split your db (Back-End: tables, Front-End: queries, forms, reports, macros, vba, ....), and then distribute a copy of the Front-End (FE) to each user.  Never share a common copy of the FE as that can cause corruption.  Below are a few links on the subject should you not already be familiar with it:

http://www.devhut.net/2010/08/24/ms-access-splitting-and-deploying-your-database/

Tbl_Db_Usage

UsageID

NetworkUserName

LogIn

LogOut

Create a form based on this table (Data Entry set to True).  Set the LogIn default value to =Now(), Set the NetworkUserName to default =fOSUserName() which function can be found at http://www.access.mvps.org/access/api/api0008.htm

Launch the form at the db startup, in hidden mode.

Then use the form's Close Event to set the LogOut value =Now().

That's it.

hansv mvp say:

You could create the following event procedure in the ThisWorkbook module of the workbook.

This version only protects cells in a sheet named Sheet1 but it can easily be modified to protect several or all worksheets.

 

Private Sub Workbook_Open()
    Dim wsh As Worksheet
    Dim rng As Range
    Set wsh = Me.Worksheets("Sheet1")
    On Error Resume Next
    wsh.Unprotect Password:="Secret"
    wsh.Cells.Locked = False
    Set rng = wsh.UsedRange.SpecialCells(xlCellTypeConstants)
    If Not rng Is Nothing Then
        rng.Locked = True
    End If
    Set rng = wsh.UsedRange.SpecialCells(xlCellTypeFormulas)
    If Not rng Is Nothing Then
        rng.Locked = True
    End If
    wsh.Protect Password:="Secret"
End Sub

 

Save the workbook as a macro-enabled workbook (.xlsm, .xlsb or .xls), not as a .xlsx workbook.

Users will have to enable macros when they open the workbook, of course.

Here is what I understand (although I'm not an audio-tech wizard)

AAC, as you know, is an audio codec. when Apple used it in itunes, they renamed the container m4a (for audio), m4b (for books) etc. And since everyone has an iPod, this m4a extension has become a kind of standard. 3GP, AAC, M4A and MP4 are all different implementations of the same thing,intended for different usages.

AAC+ is an enhanced version of AAC, that does clever magic to the AAC file so that it sounds better at very low bitrates. So.. it is recommended that you use AAC for anything over 128, and AAC+ for anything under. Nokia's music manager tells you this (in preferences).
