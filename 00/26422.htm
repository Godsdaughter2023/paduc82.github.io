jeeped say:
Bob;

I had this problem a while ago. This was my solution.

Create a table with a header row and one data row. Specify the header row to Repeat as header row at the top of each page within Table ► Properties. I actually made a table style for this as there were numerous tables with the same basic layout and formatting requirements. Bookmark the (almost) empty table. To keep things simple, I used identically named Word bookmarks and Excel named ranges each with rpt_ as an identifying prefix.

From Excel, my macro opened the Word document and fed the data in cell by cell. This was more time intensive than copying and pasting the table as a whole but it isn't too bad if the Word document is hidden. If it isn't hidden, you can watch the Word table calls populate. Something like,

''Tables transfer
With ThisWorkbook
    For Each nm In .Names
        If Left(nm.Name, 4) = "rpt_" Then
            'Debug.Print nm.Name
            With .Names(nm.Name).RefersToRange
                For x = 1 To .Rows.Count  'change to x=2 if the Word headers are OK
                    If x > wdDOCX.Bookmarks(nm.Name).Range.Tables(1).Rows.Count Then
                        wdDOCX.Bookmarks(nm.Name).Range.Tables(1).Rows.Add
                    End If
                    For y = 1 To .Columns.Count
                        wdDOCX.Bookmarks(nm.Name).Range.Tables(1).Cell(x, y).Range.Text = .Cells(x, y).Text
                    Next y
                Next x
            End With
        End If
    Next nm
End With

You could probably just modify the Word table's header row after the wholesale table paste for to Repeat as header row at the top of each page but this is how I brought in a few dozen data tables to a new Word document based upon a custom template. As this was a small part of a report generation routine that took a couple of hours, time was not mission critical.

I chopped up and redacted the code. I hope it's mostly usable.

brian tillman [outlook mvp 2007-2019] say:

This indicates that he wrote his message in the signature portion of the message and does not have a signature chosen for that account.  When he changed the account, Outlook erased the signature area and since that's where he was composing the message, the message disappeared.

To guard against this, he should have a default signature defined for each account.  For those accounts where he doesn't want a signature, he should use a signature of just "--" so that he can tell where he should type the message (above the signature).  Then when the account is changed, only the signature area will be changed.

Hi,

 

OK because you're going to have to modify this to match your table then I'll go through how it works.

 

The original formula from your first thread modified for this table was this and we need to change the B5:E5 bit

 

=INDEX(B5:E5,COUNTIF(B4:E4,"<"&A2)+1)

 

We change it to this

 

=INDEX(INDIRECT("B" & MATCH(A1,A4:A6,0)+3 &":D" & MATCH(A1,A4:A6,0)+3),COUNTIF(B4:E4,"<"&A2)+1)

 

 

What this does is use MATCH to find the value in A1 to A4:A6 and that returns a number so for (say) 316L it would return 3 because 316L is in the 3rd cell of A4:A6. We must now add 3 to this number to get the correct row and then build a range from all that using INDIRECT which turns text values in a range.

 

So the bit of the formula you will need to modify is this

 

INDIRECT("B" & MATCH(A1,A4:A6,0)+3 &":D" & MATCH(A1,A4:A6,0)+3)

 

 

You will need to change A4:A6 to the correct size for your table. Currently your table starts on row 4 and that why we add 3. so if the table started in row 6 we would add 5
