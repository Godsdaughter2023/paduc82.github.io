Ay + Bn + Cy = Vy

Now at first I construed a long nested IF-statement with exactly those 8 possibilities after each other. This did not seem to work, as the output always was Vz. Moreover, I constantly got the message that my IF-statement was nested too deeply (because it had 20+ layers).

Then I simplified it the nested IF-statement, resulting in the the statement below.

{IF {=AND({IF A = "y"},{IF B = "y"}) = "1"

{SET V = "w"}
{IF {=OR(({IF A = "y"},{IF B = "n"}),({IF C = "n"},{IF A = "n"},{IF B = "n"}) = "1"

{SET V = "x"}
{IF {=AND({IF C = "y"},{IF A = "n"}) = "1"

{SET V = "y"}
{SET V = "z"}

}}}}}}

However, quite frustratingly, I keep having the same problem. All possible combinations of values for A, B and C result in Vz.

As for the layout of my template, I started the file with a short questionnaire of SET-fields so that a user can decide on the values of the variables. After these SET-fields, I have the nested IF-statement mentioned above, and below that is the body of the document, in which the variables are sometimes called upon. 

Could anybody steer my in the right direction?

macropod say:

Since all the questions have a binary response (Y/N), all you need do before saving the template is decide which response should be the default for each question and answer it accordingly, then update the output.

As for any disconnect between the responses and the output, that's because the two field constructions are independent of each other. As coded, the output fields will only update when an event such as printing the document causes them to do so. If you want the output to update immediately the questions are answered, you'll need to combine the two field constructions, thus:

{QUOTE{ASK ValA "Value (Y/N) for A" \* Upper}{ASK ValB "Value (Y/N) for B" \* Upper }{ASK ValC "Value (Y/N) for C" \* Upper }{SET Rslt {={IF{ValA}= "Y*" 1 0}+{IF{ValB}= "Y*" 2 0}+{IF{ValC}= "Y*" 4 0} \# 0}}{IF{Rslt}> 5 "Vw = True" {IF{Rslt}> 3 "Vy = True" {IF{Rslt}= 1 "Vy = True" {IF{Rslt}= 0 "Vz = True" "Vx = True"}}}}}

All that's been done here is that the second field construction in my previous reply has been inserted before the final } in the first field construction.

There is no document-specific switch to control field update behaviour.

PS: For your edited outcomes table, you could use:

{IF{Rslt}> 5 "Vw = True" {IF{Rslt}= 1 "Vy = True" {IF{Rslt}< 4 "Vx = True" "Vy = True"}}}

To save you a lot of frustration:  "No", there is no RELIABLE way to use relative addressing within Word documents on Word for Mac.

If you use File>Save As>Web Options and change "Update links on Save" to switch it OFF, you can then search/replace your links into relative links, and if you do, they will stay the way you set them.  Until someone makes a change and turns that setting back on...

To search replace, you have to set Word>Preferences>View>Field Codes to ON.  That will expose the underlying code of the hyperlinks so you can then remove the Path.

You will find it a lot easier to get this working if the Index page and all the sub pages are all in the same folder (not subfolders of it).

But it's a lot of work, and it's not very reliable.

To be kind to you, I should say "No, you can't do this in Word, use a web editor" :-)

Hope this helps
