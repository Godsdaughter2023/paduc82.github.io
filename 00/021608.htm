jay freedman say:

First, I would switch to using a wildcard search, which can find the whole comment including the matched parentheses with one statement. While storing the comment text, strip off the parentheses. Then delete the found text and insert the new comment in its place. All of this is done inside a While loop that uses the result of the .Execute method (True if it finds something, False otherwise).

Sub FindTextForComment()
    Dim rgSrch As Range
    Dim strCmt As String
    Dim bFound As Boolean
    
    Set rgSrch = ActiveDocument.Range
    With rgSrch.Find
        .MatchWildcards = True
        .Wrap = wdFindStop
        .Text = "\([!\)]{1,}\)"  ' any text between parens
        While .Execute
            rgSrch.Select
            ' store found text without parens
            strCmt = Replace(Replace(rgSrch, "(", ""), ")", "")
            rgSrch.Text = ""  ' deletes entire found text
            ActiveDocument.Comments.Add Range:=rgSrch, Text:=strCmt
            ' prepare for next Execute
            rgSrch.Collapse wdCollapseEnd
        Wend
    End With
    
    ' check for unmatched parens
    Set rgSrch = ActiveDocument.Range
    bFound = False
    With rgSrch.Find
        .MatchWildcards = True
        .Wrap = wdFindStop
        .Text = "[\(\)]"
        While .Execute
            bFound = True
            With rgSrch
                .Select
                .Font.ColorIndex = wdRed
            End With
        Wend
        If bFound Then
            MsgBox "Found at least one unmatched paren, now RED"
        End If
    End With
End Sub

The second half of this macro checks for unmatched parentheses remaining in the document and marks them in red for manual attention.

There is a situation this macro doesn't handle yet: If an unmatched opening parenthesis is followed by a matched opening parenthesis, the first one will be taken as the start of the comment; the second one will be somewhere in the middle of the strCmt string, and the Replace function will delete it. That will leave part of the text inside the new comment, making nonsense of the remaining text. This can be handled with more complicated code if necessary.
