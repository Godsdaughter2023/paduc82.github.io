don cuthbert say:

It can be confusing if you end up with 2 bootable Boot Managers on the same UEFI system. I find that it is much simpler to limit it to one SYSTEM partition that will boot Windows on any number of other partitions or physical disks.

What I would do is boot with Windows 10 install media and delete the SYSTEM partition from your SATA SSD or test system. You may then need to ensure that your UEFI BIOS is pointing to the one you want to use. Often multiple Boot Managers will simply show up as "Windows Boot Manager" and that makes it difficult to choose the correct one.

Once you are successfully booting from the desired Windows image, then you can add in the menu choice to boot the other Windows/Boot partition from the SATA SSD. You can do this by finding the second Windows install in File Explorer or Disk Management and running bcdboot to point to the second Windows install.

For example, if i want to boot a second install of Windows on drive F:, I use this command to add it to my boot menu:

bcdboot F:\Windows

You can then change the Menu item name by this command:

bcdedit /set {default} description "Test System"

You will end up with Windows 10 and Test System as choices during boot. Test System will be the initial default, but you can switch the default OS.
