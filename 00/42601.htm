rob koch say:

The Windows Defender updates for Windows 10 typically occur along with the normal Windows Automatic Update process starting around 10 minutes after the PC is switched on, from what I've observed with my own system.

This update process isn't just checking for the relatively tiny Defender definitions and engine updates, but also the entire update catalog for Windows 10 itself, including security patches and feature updates to the operating system, as well as any apps that may be installed including Office and those included with Windows like Notepad.  For Windows 10 Home versions this update process is not able to be easily controlled by the user, so it will run approximately once each day depending upon the last time it executed.

With older systems which have been upgraded from Windows 7/8.x to 10 and some less capable hardware on newer systems, this catalog checking process can cause significant overhead, since it must read and write large amounts of catalog and file information in order to determine which Windows components and apps may need to be updated.  As there's no way to know when an urgent security or major feature update might become available, Windows Update must perform these deep checks regularly in order to insure they'll be installed.

To prove this, try running just the Windows Defender definitions check manually yourself from within the Windows Defender Security Center, by clicking the Check for updates button within Protection Updates.  If the full Windows Updates check had already occurred earlier that day, then this definitions only process should happen much more quickly and may provide a different definitions package, since several of these can be released in a single day.

For my own 10 year old desktop system, the daily update and other initialization processes that occur when I first turn it on can make the system relatively unusable for certain disk dependent processes for up to the first 15 minutes.  Since this is a quad-core system, it isn't the processor that's creating these issues, it's the relatively ancient hard disk drive that limits the performance of this filing system intensive process.

On the other hand, the similar Windows Update process for my several year old laptop performs much faster and is less noticeable, since though it's dual-core hyper-threaded processor is quite similar to the desktop, the hard drive was replaced with a solid state disk (SSD) when it failed a few years ago.  This means that the filing system processes operate at electronic speeds with much less latency, so disk intensive processes like Windows Update run much faster and more silently.

So what you're likely seeing is just the aging of less modern and capable hardware, which though Microsoft originally promised to provide updates for Windows 10 upgraded systems for at least a few additional years, is the cost for retaining these older systems.

I personally accept this tradeoff for the improved security and abilities this has provided my 10 year old desktop and just try to remember to turn it on and log it in at least 15 minutes before I truly plan to use it for anything that requires heavy disk use.  This way by the time I sit down to actually use it, the update check has typically completed, though a major monthly "Black Tuesday" update (2nd Tuesday) or semi-annual feature update can take an hour or more to complete on that old hardware.
