What happened to "normal View" in Word 2010 - Microsoft Community

tom ripley say:

What happened to "normal View" in Word 2010

I used to use Word 2003.  In addition tothe print layout view, there was a view called "normal". What happened to this in Word 2010? Has this been replaced by draft?
suzanne s. barnhill say:

To add to Stefan's cautions, it's not just graphics that are affected. I routinely use a button that condenses my text by 0.1 point (very helpful in copy fitting). In Draft view, this results in absolutely no change in the text as displayed; the change can be seen in Print Layout view.

The original point of Normal view was to optimize performance by suppressing display of graphics and also any element that Word had to generate programmatically, such as repeated headers, table headings, page numbers, etc. You could make it even more efficient by turning off background repagination and/or using a draft font. Unless you're using a really old computer (which of course you wouldn't be with Windows 7 and Word 2010), this is no longer an issue. Your computer is quite capable of displaying a fully WYSIWYG view without breaking a sweat.

Many users (including me) continued to use Normal view because it allows you to concentrate on text (especially when you're composing) without being distracted or interrupted by page breaks (the space between pages in Word is not nearly as distracting to one's train of thought as having to put a new sheet of paper in the typewriter, but it's still a break). If that's your reason for wanting to use Normal/Draft view, you may find, as I do, that using Print Layout view with the "white space between pages" hidden is the best of both worlds: a fully WYSIWYG view with minimal distraction from page breaks. See http://sbarnhill.mvps.org/WordFAQs/WhiteSpace.htm for more.

jay freedman say:
There is a limit to the length of the command line that can be passed from Windows Explorer (which I assume is where the 307 documents were 'selected') to any other program. For most purposes that limit is 32,767 characters, although it can be less than that under some circumstances -- see http://blogs.msdn.com/b/oldnewthing/archive/2003/12/10/56028.aspx.

The command telling Word to print a batch of files includes the full path and file name of each selected document. Word will only "see" up to the maximum number of characters, and the rest will be chopped off.  If the paths are very short (say, if the folder containing the documents is directly under C:\ and has a short name), you can include many more file names than if the folder is buried many levels deep. Very long file names also contribute to the problem.

Also, if you are exceeding the maximum character count, the last path/file name in the command will probably be incomplete, which will result in a 'file not found' error.

There is, literally, no way to predict how many documents can be printed in one pass by this method. The best method is a macro or other program that prints one document at a time, such as the one in http://support.microsoft.com/?kbid=306346. (Although that article says it applies to Word 2003, it's valid for all later versions as well.)

jay freedman say:
Roberta,

Going back to your post about templates vs. documents:

First point: All of the forms should work (using editable regions as Greg suggests) whether you use documents as you have been doing or if you make templates.

Second point: Using editable regions or any of the other editing restrictions would not prevent a user from filling out a form and accidentally doing a Save rather than a Save As, resulting in everybody getting that person's data instead of blank forms. Preventing that scenario is one of the major reasons to use templates: everybody always gets a blank form because you can't accidentally save a document as its template. (You can do it, but it takes some deliberate actions.)

Third point: There are only reasons to put templates into a specific Templates folder. The main one is that Word's File > New command only shows templates that are in that folder. The other is that if a template contains macros, it needs to be in a "trusted location" for the macros to be enabled.

But... if the users go to whichever folder (not the Templates folder) contains the templates, and double-click the one for the right municipality, they'll get a blank form based on that template -- not the template itself. (The default action for a document is Open, but the default action for a template is New.) I assume your forms don't contain any macros, so that's not a concern.
