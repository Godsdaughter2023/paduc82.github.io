suzanne s. barnhill say:

It would probably be instructive for you to investigate the various fields you can insert into a document. Go to Insert | Text | Quick Parts | Field to see a list of the fields that are available. A summary of the types of date fields can be found at http://WordFAQs.mvps.org/DateFields.htm. For the name of the document, you need a FILENAME field, but it won't show the actual filename until you have saved and named the file (which it is recommended that you do within the first 5-15 minutes of typing).

 

In your question you say you're "writing a book" and "keep on adding to the document." This implies a single document file; if that's the case, then none of the date fields will be especially helpful to you unless you just want to know the date when it was last modified (the SAVEDATE), which you can also see in Windows Explorer. A DATE field will be updated to the current date every time fields are updated, so all DATE fields will have the same date. You can, however, insert a DATE field with Alt+Shift+D and then immediately unlink it with Ctrl+Shift+F9, so that the date becomes ordinary text.

 

When your computer crashes, you should not lose any data. Provided you are saving regularly (every 15-30 minutes is recommended), you should always have the latest version. In addition, if you have AutoRecover enabled, Word should be saving AutoRecovery backups periodically and will present them when you open Word after a crash. If your computer crashes frequently, you should investigate and deal with the cause of the crashes.

Thanks for the input, but unfortunately that wouldn't work in our situation...

  • Make an "inactive milestone" inside a summary task. (2 at the bottom in my case)

steve(boma) say:

I had the same issue and thought I was done for.  As it turns out it is an easy fix.  I am no expert but this worked for me.  I upgraded from Window 7 Pro to Windows 10 about a month ago and everything was fine.  Then one morning I woke up and had the endless loop of going through the repair process on a system that would not boot.  Here is what I discovered in my quest.

PROBLEM:  The problem file (symELAM.sys) is a Norton Security file.  It is part of the early malware checking that is done during the bootup process.  I did not have this file with my Windows 7 OS.  Why it took so long to show up I can't tell you but Norton in Windows 10 needs this file.

SOLUTION:  Since my system is now fix I cannot describe in great detail I will try.  

1.  From the main repair screen select the Advanced option.  This takes you to a screen with about 5 or 6 choices and among them a command prompt option.  The option right below that is the one you want which says something about config settings or something.  Select that one.

2.  It should now take you to a page with about 10 options like boot in Safe More or boot in safe mode without network.  Item #8 on the list is boot without the pre-checking of malware.  Select that one.

3.  Your system should now boot up.  Open a browser and navigate to Nortons web site and download the uninstall program for Norton which is free.  Run the Norton uninstall program.  When finished it may want to reboot your computer.  Go ahead and reboot which should work fine since Norton is not installed now.

4.  Once it boots back up re-install Norton Security on your Windows 10 image.  Your done in 20 minutes without loosing anything.  By the way, your system will now have that file after the re-install.

I hope this helps someone because I was really stuck and would have lost a lot of data.  Good luck.

keratl say:

This is one solution- a little clunky to set up, but saves you from the password.

Add the code below to a module.

On a sheet that you will hide, add:

A1: the start time you want to prevent entries as =time(16,0,0)

A2: the end time for preventing entries as =time(18,30,0)

A3: =GetUser()

B1 = time(hour(now()),minute(now(),0)

Then add data validation for the range you want to prevent entries, something like:

=AND(    OR(Sheet1!B1<Sheet1!A1,Sheet1!B1>Sheet1!A2),   Sheet1!A3="ABCDEFG")

where ABCDEFG is the username that shows up in A3 when you are the one logged in

Note: the one area where this might not work consistently is if you have a user who always keeps calculation turned off- if you were the last user, A3 may retain your userID. If you wanted you could add a line of code in Workbook_exit to delete A3, and put the formula back in on workbook_open (if you were really worried about it)

Note2: if you need to have multiple 'authorized' users to enter data during the blackout period, then instead of A3 maybe list all valid users in column C, and compare A3 to that list using Match, say in cell A4. Then just reference A4 instead of A3 in the DV, checking to see if it returned an error:

=AND(    OR(Sheet1!B1<Sheet1!A1,Sheet1!B1>Sheet1!A2),   not(iserror(Sheet1!A4)))

(or something close to that, this is aircode)

Add this code in a module to get the current user ID:


Function GetUser()

    Dim ObjWshNw As Object
    Set ObjWshNw = CreateObject("WScript.Network")

    GetUser = ObjWshNw.UserName

End Function
