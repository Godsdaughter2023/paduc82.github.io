User Form to Create Employment Contract - Microsoft Community

a. user say:

User Form to Create Employment Contract

Hi

At present we complete an Excel sheet with details for an Employment Contract where there are IF statements to select the correct paragraphs for the contract type.

I can create a simple User Form but was wondering how to include IF statements; for example: if "1" is selected in the User Form, the Paragraph A is selected, but if "2" is selected, then Paragraph B is used.

We are using Word 2002

Thanks

jay freedman say:
In the design view of the userform, double-click the OK button (or whatever button you
designed to indicate that the entries have been completed and the userform should
disappear). That will open a code window and create the first and last lines of the
procedure that will run when the user clicks that button -- something like
 
Private Sub CommandButton1_Click()
 
End Sub
 
Between those two lines, you need to write code that does whatever you want to happen at
that time. You haven't said where the text for your paragraphs is stored, or where in the
Word document you want it to be placed, so you'll need to make those decisions first.
 
Another problem with your request: When you say "if "1" is selected in the User Form",
what does that mean? Is "1" an option button, or an entry in a list box, or something
else? What's the actual name of the control that you're referring to? For the example
below, I'll assume it's an entry in a listbox named "ListBox1".
 
At the simplest, your Word template could have a bookmark named something like
"Description" at the proper place in the body text. The texts of the two paragraphs could
be written directly in the macro code, or stored as AutoText entries in the template, or
even pulled from the existing Excel worksheet. Let's say they're stored in the template as
AutoText entries named, respectively, "ParagraphA" and "ParagraphB". Then the code you
write could look like this:
 
Private Sub CommandButton1_Click()
    If ListBox1.Value = "1" Then
        ActiveDocument.AttachedTemplate.AutoTextEntries("ParagraphA").Insert _
            Where:=ActiveDocument.Bookmarks("Description").Range
    ElseIf ListBox1.Value = "2" Then
        ActiveDocument.AttachedTemplate.AutoTextEntries("ParagraphB").Insert _
            Where:=ActiveDocument.Bookmarks("Description").Range
    End If
    Unload Me
End Sub
 
The If statement gets the value that's selected in the list box, and does one of the two
Inserts (or maybe neither, if there are other things in the box and one of those is
selected). The quoted name in the AutoTextEntries item tells it which of the two
paragraphs to use, and the quoted name in the Bookmarks item tells it which location to
insert the text at.
 
There are many other ways to do this, some of them much better from a viewpoint of
user-friendliness, ability to correct mistakes, and resistance to deleting things that are
required to make the form work. Once you understand how this simplest example works, come
back and learn about the others.
 
Jay Freedman
MS Word MVP  FAQ: http://word.mvps.org

john korchok say:

Neither one is easily implemented in PowerPoint for Mac. 

It's possible to insert an ActiveX dropdown in a Windows version of PowerPoint, but these cannot be created or used in PowerPoint for Mac.

For a custom list of icons, there is no hidden storage space for them as seen in Word's AutoText/Building Blocks. It would be possible to create a macro that could hold and insert them, but that's not a simple task. Windows versions can use add-ins like ShapeChef to create icon libraries, but they don't work on a Mac.
