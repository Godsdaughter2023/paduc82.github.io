jay freedman say:
You can install this macro in your Normal.dotm template (see http://www.gmayor.com/installing_macro.htm) and add a button for it to the Quick Access Toolbar or to a custom group in the ribbon:

Sub NewSectionUnlinked()
    Dim i As Long
    Selection.InsertBreak Type:=wdSectionBreakNextPage
    With Selection.Sections(1)
        For i = 1 To 3
            .Headers(i).LinkToPrevious = False
            .Footers(i).LinkToPrevious = False
           
            ' optional - uncomment if wanted
            ' .Headers(i).Range.Text = ""
            ' .Footers(i).Range.Text = ""
        Next i
    End With
End Sub

You could make similar macros for the other section break types, but that's probably overkill.

scottgem say:
As John mentioned, there is a process to designing a database, that process  is referred to as normalization. One of the principles of normalization is that data exists in one place only. So when data repeats, you need to establish another table. 

So I imagine the evaluation would contain information like this:

The course being evaluated
The time when the course was held
The instructor who led the course
The student doing the evaluation
The elements being evaluated

These are FIVE different entities, that need to be kept as separate tables linked together. In addition, your application is basically a questionnaire. And questionnaires have their own specific structure. So this is what I would recommend

tblElement  (these are the elements being evaluated like instructor skill, course curriculum, etc.)
ElementID (Primary Key Autonumber)
Element

tblCourse
CourseID (PK Autonumber)
CourseTitle

tblSession
SessionID (PK Autonumber)
SessionYear
SessionQtr
CourseID (Foreign Key)
InstrucotrID (FK)

tblStudent
StudentID (PK Autonumber)
Firstname
LastName
Other info about student

tblInstructor
InstructorID (PK Autonumber)
Firstname
LastName
other info about instructor

tblEvaluation
EvaluationID (PK Autonumber)
SessionID (FK)
StudentID (FK)

tblRating
EvaluationDetailID (PK Autonumber)
EvaluationID (FK)
ElementID (FK)
Rating

To enter the data you use a main form with a subform. The mainform is bound tblEvaluation and the subform to tblRating. The student select a session, probably from a list box, that displays Course and when it was held and who the instructor is. Then in the subform they rate each element.

To create reports, you use a query to combine the tables using the key fields and pulling the descriptions as needed.
