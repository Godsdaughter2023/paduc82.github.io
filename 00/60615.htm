sanmartin_02 say:

You must tell us the *name* of the game. Some games of this era were using Windows (98) only as starter for a pure DOS process, others were using built-in graphical features of Windows (Win32). As Palcouk said, the bitness (16- or 32-bit) is also to consider, 16-bit games will only run on a 64-bit system in a sandbox where all hardware/software functions are emulated (e.g., www.dosbox.com). There are even community patches or game-specific emulators for games which still enjoy interest among their fans (e.g., Exult for Ultima VII), allowing to play the old games on a modern computer.

As a last thing - as long as we don't go into details depending on the game in mention -, don't forget that the OS and its restrictions/features aren't the only thing to be thought of. The "replay" of some old games ("productivity" software is much less picky) is confronted with massive hardware changes, too. Too much RAM or too powerful CPUs may cause an old game to go nuts, if the game publisher - or you - didn't adjust some parameters to today's standards.

That said, don't be pessimistic. For nearly every problem in this area, there is a known and proven workaround. If you keep in mind that things possibly won't be so easy like inserting a CD/DVD, installing and playing, your chances to get even the most exotic game to run on Windows 7 aren't bad. It may need knowledge and preparation, that's it.
