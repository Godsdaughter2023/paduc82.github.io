steve rindsberg say:

Andreas,

I ran into something like this a while back in PowerPoint.  Two different add-ins, two different tabs.  Each tab had a button with an OnAction setting with the same name, and each add-in had that Sub.  Let's say onAction="Startup" in both and in both add-ins, there was Sub Startup.

It would always run the Startup sub from one or the other of the add-ins, but always the same one, no matter which button was clicked.

I changed the xml to use onAction="add-in_name_Startup" and changed the sub to Add-in_Name_Startup and haven't had the problem since.

Actually, what I usually do for each of the buttons is add a sub like 

Sub xxx_yyyButtonHandler()

where xxx is the name or shorthand name of the add-in and yyy is the name of the function, so 

Sub MyAddin_StartButtonHandler()

   Call Start

End Sub

Sub Start()

   ' Now this can have the same name in every add-in if I like, because the button handler

   ' code/name makes sure the code gets called from the right addin

End Sub

    5. Move the slider until the security level is Medium.



    6. Click Apply.
    7. Click Custom Level.
    8. Find the section "Scripting."
    9. Set Active scripting to Enable.



    10. Click OK.
    11. Select the Internet zone and repeat steps d to j.
    12. Click OK. Close Internet Explorer and then restart it.

    1. In Internet Explorer, select Tools > Safety. If ActiveX Filtering is selected, deselect it.
