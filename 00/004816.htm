rohn007 say:
Short answer.

Creating a DOCM does not embed the macros into the DOCM file. It creates links to the original template. Therefore, in a corporate environment you would have the macro templates in a shared network resource. And that location would be identified to Word in the File locations dialog as the "Workgroup location".  That is fine for documents that stay inside the corporate network where the drive mappings have been standardized.

But, if you are moving the document to locations that do not include the standard template mappings you are hosed.

This is where Words original roots are showing. Remember, Word was originally designed to run on a standalone machine, NO NETWORK at all. The document was created and saved on the machine it was created on. The document was distributed by printing.  End of story.  Some people used "sneakernet" to move the file to other computers, but only rarely. This lasted through the 80's

Then came corporate networks, one building at a time, through the late 80's to early 90's.

Around the mid 90's inter building networking became common and the first email and internet connections started showing up.


So, there still isn't a good mechanism for distributing files with macros outside of the company.

Bottom line, the only way I can see of distributing files with macros outside of the local network is to abuse the design principles used to create Word. 

OPEN the template for editing, then do a SAVE AS to DOCM format, effectively creating a copy of the template, but using it as a document.

Good luck teaching users to do it.

Another brute force approach is to give the access to the template file in Windows explorer and teach them to copy and paste the template into their working directory and follow that up by renaming it to their desired working file name.

graham mayor say:
When you create a document based on a template that contains macros, the macros remain in the template and the template is attached to the document. As long as the document can 'see' the template the macros should be available to it. If you then move the document to (say) another PC which does not have the template in the same location, then the document will not 'see' the template and thus cannot use the macros it contains.

You need either to supply the template to the users, or create the macros in an add-in template and supply that. Macros in an add-in will always be available.

A document created from a template and saved as docm will also not contain the macros, unless you copy them there before saving. You may however have problems persuading users to run macros in documents.

daniel g. grau say:

Word also has the facility to ignore spelling mistakes, and that's because of two reasons: wrong language or right language that's being ignored. This may provide a solution:

  • Select the text in question (in general, this would be Edit > Select All or Command A)
  • Go to Tools > Language and apply the proper language
  • Make sure there is no checkmark at the bottom ("Do not check...")
  • If you need that change applied to all your new documents, click on the Default button

Note that I stressed "apply." This is because language is applied to the text, just like you do with Arial, boldface, a color or a point size. Being an attribute of the text (instead of a setting or preference) makes it possible to combine several languages in a single document.

In addition, let me refer you to the official page for that problem: Spell check does not work in Word for Mac 2011.

If you still can't make it work, post back and we'll try something else.
