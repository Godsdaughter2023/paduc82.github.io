sheen valentine ali msft say:

Hi Yann,

 

Based on the confirmation of the related team, we would like to post a summary here to let other members see the information.

 

Issue Definition: 

==============

When checking the Office 365 Audit Log, you notice that the Activity is always Uploaded files to document library (FileSyncUploadedFull), even when you are editing a file.

 

According to this article:

Office 365 Management Activity API schema

 

FileSyncUploadedFull only happens when a user successfully uploads file for the first time from their computer to a SP/ODfB library.

 

What should happen when a user is editing a file is FileSyncUploadedPartial, which indicates that changes to a file in local is uploaded to SP/ODfB.

 

Resolution:

==============

After confirming with the related team, when the option in OneDrive client settings “Use Office 2016 to sync Office files that I open” is off (like the screenshot below), it is an expected behavior that the changes to the file are showing as FileSyncUploadedFull. 

 

You can refer to the following article for the requirement of differential sync:

Restrictions and limitations for OneDrive

 

 

Otherwise, when you edit a file in OneDrive on desktop Office client, the event in the audit log will shows full upload.

 

We would like to thank you for all the help and information you have provided to us during the investigation. If you have any feedback about this, we welcome and encourage you to provide them to our OneDrive UserVoice.

Regards,

Sheen

ranmo_msft say:

Hi Lakshmi,

Regarding “How do you decommission the on premise server ?”, yes, you need to point the MX record to Office 365 to keep the mail flow working. Regarding “We are using federation servers for our single sign on, what needs to be done if the server is down ?”, do you mean the assumption that the Exchange server or the ADFS server for your SSO are down? In the first case, it will not affect federation servers such ADFS for your SSO but as I mentioned above, we suggest you keep at least on Exchange server. In the latter case, if the federation are down, it will not affect the office 365 mail flow.

Regarding “office 365 mail will not have any affect even if the domain controllers are down.”, no, it will not affect the mail flow but it may have adverse effects on your SSO.

If anything is unclear, please feel free to let us know.

Ran
