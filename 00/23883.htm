Win 8/8.1 Total Backup System: File History + RecImg.EXE + Refresh + SlimImage

 

Win 8 has refined your backup and restore options. Before Win 8 you could create an Image Copy backup of the full drive (or selected directories), but it was hard to separate backups of the OS, Applications and personal Data unless you created a separate

 

Out of the box, MS has been advertising the “Refresh” and “Reset” options. Personally, I think they missed a valuable selling point by not mentioning RecImg.EXE in conjunction with Refresh. It removes my main concern/objection to Refresh.

 

Reset is equivalent to using the “Recovery Partition” on previous versions of Windows. The factory would create a “Recovery Partition” that you could use to “reset the computer to factory fresh”. It would typically include all of the factory “crapware” and it would wipe out all of your data files and installed applications.  A less than desirable outcome but an appropriate reaction to a bad malware infection.  The “recovery partition” type of restore typically had to be accessed at boot time while Reset can be accessed from within Windows 8.

 

Refresh is roughly equivalent to an image copy backup.  A Reset will restore a backup of the OS and Metro applications. It will wipe out “legacy” / x86 desktop type applications BUT it will not affect your personal data files, which is much better than a Reset.  But there is still that issue of wiping out your installed desktop applications. I do not have documented confirmation, but I’m guessing that Metro apps are treated like personal data and those directories/folders are simply not reset back to the old condition.

 

RecImg.EXE is a backup tool that ships with Windows 8. It creates the backup “image” that is used by the Refresh tool. This image that was created at the factory is why using Refresh will wipe out installed desktop programs in the “C:\Program files” and “C:\Program Files (x86)” folders because they haven’t been backed up. However, using RecImg.EXE you can create new images that will include recently installed desktop applications.

 

RecImg Manager is a third party tool the provides a GUI to access the RecImg.EXE command line tool and backup files.

 

File History is a new Win 8 feature. It is an extension to the “Previous Versions” and “Volume Shadow Copy” features. File History continuously monitors files stored in Libraries, Desktop, Contacts, and Favorites folders. It runs on a predetermined schedule, and when it detects changes in any file, it makes a backup of that file to an External drive. It isn’t enabled by default.

amgalloway say:

Transferring Software ownership, can I donate my copy of Office 2003 to the church

I am now using Office 2007.  I still have my discs for Office 2003.  My church is still using Office 2000.  How can I donate my copy of Office 2003 to the church?
dan at it associates say:

First verify that it is a transferable license.

 

If the product key is printed on a yellow-orange Proof of License (POL) label and the product came in retail packaging, then it is a retail license and is transferable to whomever you wish. 

 

If the product key is on a Certificate of Authenticity (CoA) sticker, it is an OEM license and is NOT transferable.

 

Assuming it's transferable, give the product packaging (because the packaging has the CoA and the POL label) and the product discs to the church.  If you made any backup copies of the discs, send them along to the Church as well.  Ensure that the product has been uninstalled from all of your computers, even old ones you no longer use.
