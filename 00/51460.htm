Why is there a discrepancy with disk space usage? - Microsoft Community

m_buzz say:

Why is there a discrepancy with disk space usage?

I have a old hard drive from an older XP machine. I connected it up externally to take a snap of the data for backup purposes. The hard drive is 80GB and Windows is reporting that there is 21GB free (or 69GB used). If I take the properties of all the files (including hidden and system files), it is showing 60GB worth of data. I did a clone of the disk and that container also was 60GB. Why does the 'pie chart' say 69GB? Where is the 9GB? I've also run chkdsk as well.


Just makes me nervous that I'm missing some data...

lemp say:
You have at least three different things that may be confusing you. 

The most likely reason for a "discrepancy" as large as 9 GB is the System Volume Information folder.  This "super hidden" folder is primarily used for System Restore points, but also can include content indexing databases (when you permit the default behavior of "Allow Indexing Service to index this disk for fast file searching").  Even though you can see the System Volume Information folder in Windows Explorer when you uncheck the option to "hide protected operating system files," it reports a size of 0 bytes -- regardless of what is actually in the folder.

Although there is a way to access the SVI folder -- which is strongly discouraged -- start by opening System Restore and see how many restore points you have.  By default, Windows XP allocates 12% of the partition size for System Restore, so if you have a nominal 80 GB partition, you would have 9.6 GB allocated for SR.  Assuming that you've been running for some time, that allocation is probably filled up (old restore points are deleted on a FIFO basis as new ones are created).

If you're reasonably confident in the stability of your system, run Disk Cleanup (Start > Run > cleanmgr.exe > OK) and select the "More options" tab.  Then click the "Cleanup" button in the "System Restore" section to delete all but the most recent restore point.  If you do this, my guess is that your 9 GB discrepancy will disappear.

In addition to the SVI folder, there are a couple of relatively minor points:

Not all "gigabytes" are the same.  When a disk manufacturer says that you have an 80 GB drive, it means that your drive has approximately 80,000,000,000 bytes.  Windows would report a drive of that size as having 80,000,000,000 / 1,073,741,824 = 74.51 GB (where 1,073,741,824 is 1024*1024*1024).

In addition, there is a difference between the number of bytes in a file (the "size" reported by Windows when you look at a file's properties) and the space occupied by the file on the hard drive.  Windows reports this as the file's "size on disk."

For most recent Windows operating systems, including XP, the default "cluster size" for NTFS-formatted drives is 4 KB.  See http://support.microsoft.com/kb/140365.  When a file is saves to the hard drive, it occupies an integral number of clusters.  On average, half the cluster size is "wasted" for every file stored.  For a drive with a normal mix of file sizes and about 150,000 files, only about 300 MB of space will be wasted.

P.S.:  You should reduce the allocation of SR space to about 1 GB.  See http://bertk.mvps.org/html/diskspace.html

Depending on your TV, the Audio Return Channel might not support surround sound from other inputs, like the other HDMI input from your Xbox. Most TVs only send multi-channel audio via ARC if the sound is produced in the TV's tuner, like Live TV, or attached media drives.
