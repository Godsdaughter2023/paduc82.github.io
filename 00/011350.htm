greginmich say:

This threat was detected in the Windows Imaging Format (WIM) file used for system recovery. So nothing new was added to this archived recovery container that caused it to be flagged as Trojan:Win32/Bitrep.A. And it’s most likely just another case of a false positive detection resulting from a recent definition update – like the one we just had with TrojanDownloader:JS/Jesdow.B!url:

 

https://answers.microsoft.com/en-us/protect/forum/protect_scanner-protect_scanning-windows_10/trojandownloaderjsjesdowburl/7f1240f2-9db4-488b-a37d-583c7ef94df9

 

But since this false positive is located in the recovery drive of an old laptop, as opposed to the system drive of Windows 10 machines, I’m not sure that anybody is going to be in a big hurry to fix it. You can try excluding the file/folder/drive, depending on the available options, but you won’t be able to remove the file that’s being flagged as Trojan:Win32/Bitrep.A, and you shouldn’t waste your time trying.

 

However, since no one else seems to be reporting this issue, it’s also possible that this detection could be an artifact of some other problem with malware or with system corruption – so in addition to scanning for malware (as recommended above) you might also want to run the appropriate system repair tool:

 

https://support.microsoft.com/en-us/help/929833/use-the-system-file-checker-tool-to-repair-missing-or-corrupted-system
