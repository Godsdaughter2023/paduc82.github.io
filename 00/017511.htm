Users that are on Office 365 already are having trouble accessing each others mailbox. The permissions for delegation are correct. steve drury say:

Similar questions have been asked on the TechNet forum and redirected to this forum. Everyone is more than happy to pass their problems off to other people. 

We did end up finding the fix for this yesterday. 

Because we're still in the migration period we have an incomplete domain setup, and the autodiscover DNS record hasn't been forwarded to O365. Outlook seems to be the only service that uses autodiscover which explains why the symptoms were not present in OWA or Mail/Calendar. 

Adding the autodiscover.domain.com record manually immediately rectified this problem.

scottgem say:

Can you tell us what you are doing? You have not provided us with enough information to answer you. What text box? What combo? What are you filling the textbox with? Why are you tryign to save this data?

Generally, if you are displaying data that was selected from another record using a combo, you do NOT want to store it in another table. 

gpi27201 say:

I have three tables

Invoice Table 

Location Table

Vendor Table

I have a Invoice Form that stores all the invoice information. I have two unbound combo boxes within the Form Invoice one combo box is used to auto populate the vendor number and vendor name.  The second combo box is used to auto populate the location number and location name.  

Combo Box #1 has the following logic in the row source:

SELECT [QRY-Vendor Info].[Vendor ID], [QRY-Vendor Info].[Vendor Name] FROM [QRY-Vendor Info]; 

The vendor ID text has =[cboVendor].[Column](0) (this helpful logic came from your blog)  is located in the control source.

Once the form is activated, the information does not change when I create a new record.  If I change the information in one record, it changes the information in all the other records.  I would like this to save each record and open to a blank field when I began to enter information for the next record.

scottgem say:

If you have an unbound combobox then data doesn't change until you change it. 

I'm not sure why you are working the way you are. Your Invoice table should have a field for Vendor ID (and Location ID). The comboboxes should be bound to those fields. It sounds like you have the first column width set to 0 so the ID isn't displayed. If you really need to display the ID as well, then you are doing it correctly, but there is no need to save that data. The bound column of the combos should be 1 and the controlsource should be the FK fields in your Invoice table. 

gpi27201 say: Ok, we are getting close.  The form is working.  The next problem I have is when I close the form and open the invoice table, the vendor name is not populated on the table like it populates on the form. scottgem say:

You don't. One of the principles of relational databases is that data exists in ONE place only. Since the Vendor and Location names already exists in the Vendor and Location tables, there is NO reason to duplicate those fields in your Invoice table. 

If you need to produce an Invoice that includes the Vendor and Location Names, you create a query that joins the three tables and pull those fields into your query. Then use the query as the source for you Invoice.
