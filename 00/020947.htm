ken sheridan say: As Karl pointed out, many of us would advise against the use of a multi-select field except in the very limited context where the database interfaces with SharePoint in such a way that SharePoint requires this feature to be used (see Help for more details).  Otherwise I'd strongly recommend using the conventional means of modelling a many-to-many relationship type by means of a third table.  The model would be along these lines:

Contacts----<ContactsProductServices>----ProductServices

The ContactsProductServices table resolves the many-to-many relationship type between Contacts and ProductServices into two one-to-many relationship types and would have two columns such as ContactID and ProductServiceID, each of which is a foreign key  referencing the primary keys of the referenced tables respectively.  The primary key of ContactsProductServices is a composite one made up of both columns.

You'll find an example of this basic many-to-many relationship type and how to represent it in a form/subform as PatrentActivities.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

In the same SkyDrive folder you'll also find a file DecomposerDemo.zip which illustrates how data imported form a single Excel worksheet can be decomposed into a set of correctly normalized related tables in Access.

ken sheridan say: PS:  You didn't answer Karl's question as to how the multiple values are stored in your Excel worksheet.  If they are stored in a single cell as a delimited string you would have to parse the string after importing into a single column in an Access table as part of the process of decomposing the table into a set of normalized tables.  This is probably doable, but will depend on how values are delimited.

bernd p. say:

#1:

Using printout you see the "printed" content of a file and can annotate it.

But you can't edit the file anymore.

Using attachment embeds the file into a Onenote section. You only see an icon and a name.

You can edit this file later by starting with a double click; opens the file outside Onenote in the associated application.

#2:

It depends ;-)

A printout is not editable, but can be annotated , as said before.

A printout can be looked at on all platforms/devices.

An attachment is not portable; to look at you need an associated application. This makes it platform/device dependend. In reality you will not see the file especially on mobile clients.

#3:

Normally printouts are bigger.

#4:

Look at my answers and decide yourself ....
