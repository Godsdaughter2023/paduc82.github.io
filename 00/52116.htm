rpwooste say:

How do you disable GPU acceleration in Media Center / Media Player?

I am trying to smoothly play 1080p60fps MTS video files from a Panasonic HDC-TM700K camcorder. Note these are 60fps progressive video files, not the typical 60i interlaced (so double the data rate).

They play smoothly in applications that use the CPU for decoding, e.g. Panasonic's HD Writer AE, which runs my overclocked 3.6Ghz Core2Quad at ~60% CPU utilization.

However, in Windows 7-64bit, Media Center, or Media Player the CPU runs at less than 3% and the frame rate drops to around 20fps (Media Player Statistics data). This is because it's clearly offloading the decode task to the GPU, a nVidia GTX275, which obviously isn't up to the task.

So the question is: how do I disable the GPU acceleration in WMP or WMC so that it uses the CPU for decode and probably would then work correctly?

 

 

 

samuel tai say:

This is kind of brute-force, but I believe nVidia uses application-specific CoolBits in the registry to determine which applications to offload to the GPU.

Look in HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Video. One of the GUIDs will be associated with your GTX275. You'll know which one it is because there will be a Display sub-key a couple of levels down with application CoolBits.

Media Center will be listed as _ehshell.exe:D3DOGL_..., and Media Player as _WmPlayer.exe:D3DOGL_.... If you set the entries to 00 00 00 00, this should disable all GPU acceleration for both these applications. (If you delete these entries, I believe the nVidia driver will fall back to default GPU settings for both applications.) I recommend you write down the values for both entries before mucking around with them, in case either application becomes unhappy afterwards, and you'll have to revert. Remember to reboot after modifying the CoolBits.

There is something strange going on. The registry gives these locations for your downloads folder:

%UserProfile%\Downloads

These might be the same location, depending on the value of %UserProfile%.

Your problem file resides here:

\\?D:\Download (no trailing s!)
