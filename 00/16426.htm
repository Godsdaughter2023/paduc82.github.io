nicole [msft] say:

Hi Tim

You can customize the Grammar Checker in Word. For Mac: Go to Preferences, then click on "Spelling & Grammar", then click on "Settings" next to the "Writing Style" dropdown at the bottom of the window. The Grammar Settings window will open up. To avoid seeing suggestions for simpler words, uncheck the box next to "Complex Words". Once this option is disabled, the Grammar Checker will stop highlighting the strings you pointed out.

If you only want Grammar rules enabled and not get any stylistic suggestions at all, you can also change the Writing Style to "Grammar". Just change it in the dropdown. This will disable all stylistic rules.

We also regularly fix bugs in our Grammar Checker and we can look into the Hyphen Use problem for "Common law". However, we need to see the whole sentence to know in which context the rule misfires.

tushar mehta say:

The behavior is unchanged in 2010.  So, after getting the 1st error and clicking Retry, entering 19 results in 1900%

Here's what I found and it is only tangentially related to Data Validation.

If you format a cell as %, then when one uses "in-cell editing" Excel adds a % to the right of the data entry point.  So, when data entry completes one sees 19% just before hitting the ENTER (or TAB) key.  Also, if one presses the DELETE key just before completing data entry the value is treated as without the % (as in the para below).

On the other hand, if one uses the formula bar to enter a value, Excel does *not* add a %.  So, any value in there will be treated as that value * 100%

Consequently, what is happening with Data Validation in a cell formatted as % is that the first time around there is a % to the right of the data entry point and whatever value, n, one enters is interpreted as n%.  But after the validation failure, Excel leaves the entire existing entry -- including the % -- selected.  So, this time around, entering n results the entire older entry including the % being replaced by just n (and no % at the end) and Excel treats the value as 100n%

Is it a bug?  Maybe.  But, the justification would be fairly marginal and rely on the fact that Excel selects the entire existing content after d.v. failure rather than everything except the % sign.
