Try this:

Sub CreateCF()
  Dim wshC As Worksheet
  Dim wshD As Worksheet
  Dim rng As Range
  Dim r As Long
  Dim m As Long
  Set wshC = Worksheets("AlertTable")
  Set wshD = Worksheets("PatientLabs")
  wshD.Cells.FormatConditions.Delete
  Set rng = wshD.UsedRange.Offset(1, 1)
  Set rng = rng.Resize(RowSize:=rng.Rows.Count - 1, _
    ColumnSize:=rng.Columns.Count - 1)
  m = wshC.Cells(wshC.Rows.Count, 1).End(xlUp).Row
  For r = 2 To m
    rng.FormatConditions.Add(Type:=xlExpression, _
      Formula1:="=AND($A2=""" & wshC.Cells(r, 1) & """,B2" & _
      wshC.Cells(r, 2) & wshC.Cells(r, 3) & ")").Interior.Color = vbRed
  Next r
End Sub

Bill shared that with me. Track Changes keeps track of edits until you accept them. Normally, deleted/edited text is marked as such by TT, but not just a single, isolated Page Break. The Page Break was probably deleted and if you had clicked Accept Change, it would have disappeared.

a. user say:

There is no 10535 build (yet) , you probably mean build 10532.

Yes build 10532 will expire in one year (July 16, 2016)

To find out the expiry (evaluation end date) for preview builds

Right-click the start button, command prompt (admin)

Copy and paste this command and press enter

slmgr /dlv

No the watermark remains on this 10532 preview build and every preview build.

When the preview build becomes the next official build (after 10240) the water mark will go away.

If you stay in the Insider program you will keep getting new preview builds and each will expire

in another one year and you will stay activated.

But if you leave the Insider program you need to restore back to the Official build 10240

or your build / system will expire after the one year and then not boot up at all.
