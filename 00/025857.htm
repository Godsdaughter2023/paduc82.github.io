marshall barton say:

Because of the concatenation, you can not pass what the user typed to the contacts form and expect it to be pre entered in the form.

To use the NotInList event to open the contacts form for users to enter the contact's information, use code like:

Private Sub Patient_NotInList(NewData As String, Response As Integer)

Dim ans As Variant

'Contact

   ans = MsgBox("Do you want to add this Patient?", vbYesNo, "Add New Patient?")

   If ans = vbNo Then

      Me.Patient.Undo

      Response = acDataErrContinue

      Exit Sub

   End If

'Patient

   DoCmd.OpenForm "QContactsTbl4". WindowMode:= acDialog, DateMode:= acFormAdd

   Response = acDataErrAdded

End Sub

You do NOT need to add any code to the contacts form.

Setting Response to  acDataErrAdded  will automatically requery the combo box.

Using WinowMode acDialog opens the form in dialog mode, which will pause the code until the form is closed or made invisible.  That means the response argument will not be set until after the new contact was added.

The DataMode argument value acFormAdd restricts user to only adding new records so they can not inadvertently modify existing records.

ken sheridan say: Marshall's not quite right about passing a concatenated name to a form via the NotInList event procedure.  It can be done via the OpenArgs mechanism and then parsed into its component elements in the Open event procedure.  You'll find an example in NotInList.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

The opening form in this little demo file does this, but the important point is that while it's possible, it should not be done in this context because personal names can legitimately be duplicated, so the NotInList event procedure is an inappropriate means of inserting a new row into the referenced table.  My form makes this point and includes a button to do it, which is the correct method in a case like this.  The contacts form in my demo is opened by the button in dialogue mode, and when the new row is inserted the key of the new record is passed back to the original form, whose combo box has first been requeried, and assigned to the combo box as its value.

BTW, in case you think that the duplication of personal names is unlikely, I was once present at a clinic where two patients arrived within minutes of each other, both female, both with exactly the same first and last names and both with the same data of birth.
