stephen boots say:

Quarantine is a location where suspected malware is moved to prevent it from harming the system. If you are sure that the files are not needed and are infections, you can remove them from quarantine -- which means that they will be deleted. You might need to restore a quarantined item that is critical and may have been flagged falsely -- a false positive. Without quarantine, you would be unable to do this.

In Settings, default actions can be selected for the different classifications of threats. In v4 the default is always quarantined and can't be changed (that is the way it is in the current beta, at least.)

MSE automatically cleans up History and Quarantine. Why you see these old detections in History, I can't tell you, but if they don't appear in the Quarantined items list, they are gone.

-steve

rob koch say:

The problem isn't the documentation, it's knowing how and where to find it, as with all Microsoft products there's simply too much of it.

http://www.microsoft.com/en-gulf/security_essentials/Support/bc8e74bf-93e1-4a42-9aa8-d49f849ae79e.aspx

 

Your issue is you're looking for a single answer to this question and there isn't one.  The Recommended Action is based on the Alert Level, which itself is an elusive term, since it varies based on how bad the particular malware is.

To understand this it's better to simply think of this as "Do what Microsoft Recommends" rather than try to define it for every individual case.  If you trust Microsoft implicitly, just leave them as is.  If you would rather decide, then pick one of the other selections.

This confusion for what is really a very simple concept is probably why these settings have been totally removed from MSE 4.0, since explaining is really far more difficult than just always quarantining, which gives a user time to decide if they want to keep the item before they or the automatic cleanup in MSE actually deletes it.
