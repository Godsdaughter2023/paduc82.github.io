Access Many to Many Relationship Query - Microsoft Community

jackmu say:

Access Many to Many Relationship Query

Hello,

 

I have a form which is based on 2 tables with a Many to Many Relationship. I have a Vendors Table, Occupation Table, and a Junction table. If I have, for example, 1 Vendor with a relationship to 2 Occupations, the query correctly returns a recordset with 2 lines like I want.

My question is if I have 1 vendor. but that 1 vendor is not related to any occupations in the other table, can I still show one line in the results of the query with only the vendor table info. and blanks for the Occupation table fields? In other words what an outer join would return in a one to one relationship. Is that possible? I tried to change the join properties but I get a message that that type of Join is not supported. But maybe I am just doing something wrong.

 

I hope I am explaining this correctly. Any help would be greatly appreciated!!!!!

 

armen stein say:

To expand on Gina's correct answer, and for future readers - in the query designer, make sure the query type is set so that the arrowhead is pointing from the Vendor to the Junction table, and then the arrowhead is pointing from Junction to Occupation.  This is a "Left Outer" join, though that is confusing because in the designer the tables can be anywhere, not necessarily left or right.

That query will return each Vendor/Occupation combination, and each Vendor that has no records in the Junction table will also be listed with null values from the Junction table.

gina whipp say:
Sounds like you need a LEFT Join...  That should give you all the Vendors whether they have an Occupation or not.
