Hi Tammy:

I agree with Daniel.

1)  In Word go File>Save As...

2)  Give the file a new file name

3)  Choose "Plain Text" (.txt) as the Format.

That will instantly strip all of the formatting and other rubbish (as well as the tracked changes).

Now, reformat the whole thing.  Try to use styles for all of your formatting: that way it's easy to change the document very quickly if you do not like the way it looks.

Now change the Heading 1 and Body Text styles to have the format you want.

Hope this helps

daniel g. grau say:
I would advise you to start over. In my experience, converting OCR'd text with Acrobat Pro does not yield satisfactory results.

The problem with many OCR/PDF-to-Word converters is they will produce extra Returns, too many floating text boxes and even fail to OCR tables, inserting graphics instead. Sometimes, instead of tables they use columns; instead of paragraph indents they use section breaks and change the document's margins; instead of centering a title, they indent it; repeating footers and headers are replaced by floating text boxes on every page. They use character scaling and condensed spacing to fit the text. The result is a fragile document, barely better than a simple copy and paste, where deleting or adding just one word will propagate down with a domino effect and cause everything that follows to become misaligned.

electrosonics say:

Grey listing works great when the sending SMTP server farm is in the same IP block. When more than one IP block is used, Grey listing creates an impediment to rapid delivery of email. Large ISPs with many SMTP servers can choose to ignore that issue or they can choose to be Grey Listing friendly.

Grey Listing friendly ISPs will resend outgoing SMTP email from any SMTP server in the same IP block allowing rapid delivery of email. Microsoft has chosen not to do this but instead will use any SMTP server from a large number of IP blocks thus breaking grey listing and causing delivery issues.

As for the recipient experiencing the delays, your options may be limited. You might be able to white list the sender's email address to bypass grey listing. You can check and see if you can add a rule to your grey list filter to allow email from *.outbound.protection.outlook.com. You could also white list all IP blocks that outlook.com uses to send email but I have no means to determine what those IP blocks are.

The best solution would be for large ISPs to engineer their outgoing SMTP server farms to be Grey List friendly.

Run-time Error 6 Overflow in Excel VBA - Microsoft Community

yj_959 say:

Run-time Error 6 Overflow in Excel VBA

Very weird:

Sub test()
Dim i As Double
i = 35000 'works
i = 30000 + 5000 'bugs (Run-time Error 6 Overflow)
End Sub

andreas killer say:

The reason is that the numbers 30000 and 5000 within the VBA editor are text and must be converted into a data type first. And as both values fit into an Integer, the compiled VBA code becomes

Sub test()
  Dim i As Double
  Dim a As Integer, b As Integer
  a = 30000
  b = 5000
  i = a + b
End Sub

Which also generate a RTE 6 and the reason is that the limit of an Integer is 32767.

The compiled code sums a + b, resp. Integer + Integer and put the result into an Integer, then converts the Integer into a Double and write into i.

It is enough to change one of the Integer into a larger data type:

Sub test()
  Dim i As Double
  Dim a As Integer, b As Long
  a = 30000
  b = 5000
  i = a + b
End Sub

Now, the compiled code converts a into a Long, calculate Long + Long and put the result into a Long, then converts the Long into a Double and write into i.

Therefore a faster way is to use Double for all

Sub test()
  Dim i As Double
  i = 30000# + 5000#
End Sub

The # after the number is a type converter in VB, it tells the compiler to convert the static number into a Double, and the RTE is gone.
