andreas killer say:

In Excel you have a lot of collections, e.g. all OLEObject are stored in OLEObjects, Worksheet in Worksheets, etc.

A good naming practice is to do the same. And the benefit is that the code becomes short and easy.

Furthermore if you place the init code in Thisworkbook, you can refer to all objects directly.

BTW, it's bad to compare the name of an object to find the type, because it can easily fail. (Create a shape and name it "CheckBoxesFrame").

BTW2, there is no need in VBA to set an object to Nothing at the end of a routine, because the memory manager removes objects automatically if they are no longer used.

Andreas.

Code within class module ThisWorkbook:

Option Explicit
Option Compare Text

Public ClsChks As Collection

Private Sub Workbook_Open()
  Dim ThisOLEObject As OLEObject
  Dim xChk As ClsChk
  'Create a new collection
  Set ClsChks = New Collection
  'Check all objects
  For Each ThisOLEObject In ActiveSheet.OLEObjects
    'Whick ID?
    Select Case ThisOLEObject.progID
      Case "Forms.CheckBox.1"
        'Create a new instance of our class
        Set xChk = New ClsChk
        With xChk
          'Enable the events
          Set .MyCheckBox = ThisOLEObject.Object
          'Refer to the embedded object (easier to identify)
          Set .Parent = ThisOLEObject
        End With
        'Keep it alive in our collection
        ClsChks.Add xChk
    End Select
  Next
End Sub

Code within class module ClsChk:

Option Explicit

Public WithEvents MyCheckBox As MSForms.CheckBox
Public Parent As Excel.OLEObject

Private Sub MyCheckBox_Click()
  Dim Item As ClsChk
  'Visit each item in our collection
  For Each Item In ThisWorkbook.ClsChks
    'En-/Disable depending on our value, but skip ourself
    Item.MyCheckBox.Enabled = Not MyCheckBox.Value Or (Item.Parent.Name = Parent.Name)
  Next
End Sub

Word 2010 version: How do I fix the prompt to save changes to - Microsoft Community

gg73 say:

Word 2010 version: How do I fix the prompt to save changes to Norml.docm file all the time?

Word 2010 version: How do I fix the prompt to save changes to Norml.docm file all the time?
graham mayor say:
The prompt is intended to notify you of potential changes to the normal template and should be left active. The normal template is essentially a scratch pad that stores a variety of settings. If you switch off the prompt then Word automatically saves any changed settings into the normal template. That, while being the default behaviour, can be extremely undesirable. By having the prompt you can choose when to save changes made to the template and thereby not screw up your personal settings.

The prompt should be used in conjunction with the option immediately after it to always create a backup copy, so you still have the option to revert any changes you make to the template by restoring the backup.

charles kenyon say:
Word 2010 version: How do I fix the prompt to save changes to Norml.docm file all the time?

Turning this prompt off is the equivalent of taking the battery out of your smoke detector because it keeps beeping. The question should be: Why is it beeping?

This is an alert that something is changing your normal.dotm template. If you didn't do this deliberately, you want to know if some other program is trying to change your default Word template.
