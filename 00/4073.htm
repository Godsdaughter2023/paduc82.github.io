Alternate Re-register the application

(Note: You may have to adjust the path to the EXE to match your installation. Verify the path before running the command.)

Close the app. Open the Run Window: <WIN > R, use the following command (Modify the Path to match your specific installation):

“C:\Program Files\Microsoft Office\Office\winword.exe” /unregserver

The quotes are necessary, but you may need to change the path so it reflects the location of the winword.exe file on your system. (You can use Windows’ Search tool to locate the path to the winword.exe file.) The /unregserver switch “unhooks” all the references to Word in the Windows Registry.

“C:\Program Files\Microsoft Office\Office\winword.exe” /regserver

“C:\Program Files (x86)\Microsoft Office 14\Office14\winword.exe” /regserver

This command causes Word to rewrite all its Registry keys and to reassociate itself with document and template files. Now you can double-click a document file and see if Windows starts Word and opens the document.

Sub Mail_workbook_Outlook_2()
'Working in 2000-2010
    Dim wb1 As Workbook
    Dim wb2 As Workbook
    Dim TempFilePath As String
    Dim TempFileName As String
    Dim FileExtStr As String
    Dim OutApp As Object
    Dim OutMail As Object

    Set wb1 = ActiveWorkbook

    If Val(Application.Version) >= 12 Then
        If wb1.FileFormat = 51 And wb1.HasVBProject = True Then
            MsgBox "There is VBA code in this xlsx file, there will be no VBA code in the file you send." & vbNewLine & _
                   "Save the file first as xlsm and then try the macro again.", vbInformation
            Exit Sub
        End If
    End If

    With Application
        .ScreenUpdating = False
        .EnableEvents = False
    End With

    'Make a copy of the file/Open it/Mail it/Delete it
    'If you want to change the file name then change only TempFileName
    TempFilePath = Environ$("temp") & "\"
    TempFileName = "Copy of " & wb1.Name
   
    wb1.SaveCopyAs TempFilePath & TempFileName & FileExtStr
    Set wb2 = Workbooks.Open(TempFilePath & TempFileName & FileExtStr)

    Set OutApp = CreateObject("Outlook.Application")
    Set OutMail = OutApp.CreateItem(0)

    On Error Resume Next
    With OutMail
        .to = "*** Email address is removed for privacy ***"
        .CC = ""
        .BCC = ""
        .Subject = "Galashiels Holidays Block Booking"
        .Body = "Dear East WFP," & vbNewLine & "" & vbNewLine & "Please find attached Galashiels Holidays Block Booking " & vbNewLine & "" & vbNewLine & "Regards," & vbNewLine & "" & vbNewLine & "Gala AETL"
        .Attachments.Add wb2.FullName
        'You can add other files also like this
        '.Attachments.Add ("C:\test.txt")
        .Display   'or use .Send
    End With
    On Error GoTo 0
    wb2.Close savechanges:=False

    'Delete the file
    Kill TempFilePath & TempFileName & FileExtStr

    Set OutMail = Nothing
    Set OutApp = Nothing

    With Application
        .ScreenUpdating = True
        .EnableEvents = True
    End With
End Sub

Excel/VBA Question -- Decrementing Amounts From Different Starting Numbers

I am knitting a blanket in sections and want to keep count of how many stitches I knit in each section. For each section, I start with 31 stitches and knit 2 fewer stitches every other row. So, two rows of 31 stitches, 2 rows of 29, I am knitting a blanket in sections and want to keep count of how many stitches I knit in each section. For each section, I start with 31 stitches and knit 2 fewer stitches every other row. So, two rows of 31 stitches, 2 rows of 29, 2 of 27, etc., until I get to just 1 stitch, which I only knit once. I want to figure out how many stitches I knit in each square. I can do (31*2)+(29*2)...and I have Excel set up for that right now, but I'd like to figure out total stitches if I started with 47 or 23 or something instead without having to redo all the arithmetic each time. The best I can figure out is that I need something like this but I'm not familiar enough with using VBA in Excel to know how to get it to work so that I can plug in a starting number in a cell and have another cell display the total stitches. Can anyone help me out, or at least let me know if I'm on the right track?
bobgreenblatt say:

I don't think you need VBA to do this. An Excel formula should work just fine.

Since you are starting with an odd number, just add 1 and divide by 4 to get the number of steps. Then multiply this by the sum of the first and last number. So, assuming the starting number of stitches in in A1, and the number of repetitive rows in in A2 the formula would look like:

=A2*((a1+1)/4)*(a1+1)

Giving you an answer of 512. Since you are multiplying each factor by 2, multiply the result by 2 for 512.  1,152 for 47, and 288 for 23

scottgem say:
Application.FollowHyperlink will use the Windows association feature to open a file. it is the same technology that allows you to find a file in Windows Explorer and launch the application that created it along with the file. There is no need for a hyperlink field at all. 

The rest of your scenario is correct. You store, just the filename, then build the path by concatenating the default folder with the filename. Then use that path as the argument for the FollowHyperlink method/

I would take a different tack here. Assuming there is only one scanned image associated with a record, you would store the filename of that image file in a field. If there is no image, then the field will be null. 

So you can use the On Current event to disable the view image button by testing is the field is Null:

Me.cmdView.Enable = Not IsNull(Me.imagepath)

Or you could do this behind the View button

If IsNull(Me.imagepath) Then
     MsgBox"No images associaed with record!"
Else
    Application.FollowHyperlink  lookup ("AttachmentPath", tblParameters)  & Me.imagepath
End If


tom van stiphout say:

One option:

1. In a SystemSettings table, store the path to the folder with documents. Handy when the folder location changes over time. Rather than using a mapped drive letter (which could be different for different users) use a UNC path.

2. Ideally you scan the document from inside Access (should be easy if the scanner supports the TWAIN interface). If not, you use a File/Open dialog with proper initial directory for the user to identify the doc.

3. You copy the doc to the server location. You store the filename in the database, e.g. the Payments table.

4. On the Payments form you provide a "View Check" button. In it you use the ShellExecute Windows API call to open the PDF file (concatenating root folder plus filename) in the default viewer.
