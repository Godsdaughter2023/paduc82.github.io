ken sheridan say:
still wondering how to make the form blank until I select the desired info from the drop down combo box...?

The simplest way is to base the form on a query which references an unbound combo box in the form as a parameter.  Say you have a form named frmFindContact and a table Contacts, then the form's RecordSource property would be like this:

PARAMETERS [Forms]![frmFindContact]![cboFindContact] LONG;
SELECT *
FROM Contacts
WHERE ContactID=[Forms]![frmFindContact]![cboFindContact]
ORDER BY Lastname, FirstName;

In the form there is an unbound (i.e. with no ControlSource) combo box set up as follows:

Name:   cboFindContact

RowSource:    SELECT ContactID, [FirstName] & " " & [LastName]
                      FROM Contacts
                      ORDER BY Lastname, FirstName;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column.

In the combo box's AfterUpdate event procedure requery the form with:

    Me.Requery

And in the form's Current event procedure put the following to synchronize the combo box with the current record:

    Me.cboFindContact = Me.ContactID

As the combo box will be Null when the form is opened the form's RecordSource returns no records, so the form is empty.  It is in fact at a new record, but if you want to prevent the user inserting a new record put the following in the form's BeforeInsert event procedure:

    Const MESSAGETEXT = "A new record cannot be inserted in this form."
   
    MsgBox MESSAGETEXT, vbExclamation, "Invalid Operation"
    Cancel = True

Once you select an item from its list the form is requeried and, by virtue of the reference to the combo box as a parameter, the RecordSource will return the row where the ContactID matches the parameter value, and will therefore show the matching record.  The combo box will be exactly the same for any form which includes the ContactID column in its underlying recordset; only the form's RecordSource property need be changed to reference the combo box as a parameter.

PARAMETERS [Forms]![frmFindContact]![cboFindContact] TEXT(50);

assuming its never more than 50 characters long.
