geoffc1 say:

In both the Windows 10 and Windows 10 Mobile versions of Groove, it is possible to filter your music collection by Genre. Of course, this being Microsoft, the implementation of handling Genre is both broken and different between the two versions of Groove. Really, the Groove apps are still appallingly bug-ridden and limited.

However, support for Genre is sort of there...

In both versions of Groove (for Windows 10, and Windows 10 Mobile) you can filter your albums and songs by Genre, but displaying your collection in the Artist view does not give you the means to filter by genre. This is actually logical, because if Groove is using the ID3v2 Genre tag, this is assigned at a track level.

Now, the Windows 10 version of Groove does seem to be using the ID3v2 Genre tag. However, it is a broken implementation. The ID3v2 Genre tag can have multiple values (e.g. Classical; Concerto; Piano Concerto), but this version of Groove only seems to read the first tag in a multi-value tag; so in this instance, it would list the track (and the album containing the track) as "Classical", but would not display the track (or the album) under the "Concerto" or "Piano Concerto" Genres.

The Windows 10 Mobile version of Groove appears to do things differently. I'm not certain whether it reads the ID3v2 Genre tags at all. Instead, it seems to rely on genre metadata held in Microsoft's online music metadata service. As a result, I see a completely different Genre list on my phone from that given in Groove on my PC. It also appears to assign an Artist to a Genre.

In my opinion, Groove has a long way to go before I consider it usable.
