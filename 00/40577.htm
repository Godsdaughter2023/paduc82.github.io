The problem is the outdated on board USB drivers. If you can update your drivers through Device Manager before upgrading Windows 10, then you might avoid this problem. But the fact that you're reading this tells me not.

Your Windows 10 Creators Update will crash at 75%, then on reboot make it to about 78% then crash again and roll back to previous known version. Been there done that.

Try the update again and after your first 75% completed freeze/crash...

Boot your Gateway DX4860 into your system BIOS (should be P02-A2 dated 10/31/2011).
Disable on board USB controllers and let the system boot back into the update.

It will finish. Only problem now is you have no keyboard or mouse to answer the setup questions.

Power off PC and then boot back into BIOS and enable on board USB controllers. Let system boot through to setup screen that's prompting you with questions. Answer questions and let boot into Windows 10.

Wait to see if Windows 10 will update the USB drivers for you now that it's able to boot. If not, go to Device Manager and selected each USB type device and look for driver updates on the internet. You will be able to pull down a number of them for all of the USB devices reported by Device Manager.

This should fix the problem with upgrading to Windows 10 Creators Update Edition.

P.S. Don't try and upgrade the system BIOS. P02-A2 is the only version Gateway has for this PC. The newer versions on their website are for later version desktops. If you have BIOS P02-A2, there is no newer version per Gateway.
