Windows 10 upgrade error 80004004-40019 - Microsoft Community

michaelstonebridge say:

Windows 10 upgrade error 80004004-40019

On attempting to download Windows 10 upgrade I get error 80004004-40019 Any solutions?
asmodee say:

This is a strange problem and I wasn't able to find much information on it.  However, I was able to discover a simple fix.  While looking into changing the registry settings to not allow Windows 10 updates I came across an entry I hadn't seen before.  Looking into that I found 2 registry keys, each containing exactly half of the error code.  One key contained 80004004, the other 40019.  So I just deleted the entry and the error went away.

ASSUMING YOU ARE STICKING WITH WINDOWS 7 INSTEAD OF UPGRADING, the solution is simple.  First, open an elevated Command Prompt.  To do this click the Start button and in the "Run" box type "cmd" without the quotes, then hold down CTRL and SHIFT and press Enter.  Depending on your security settings you may get a security warning because this command prompt does have administrative rights.

Next, highlight the following line.  To do this, place the mouse cursor just to the left of the line, click and hold down the left mouse button and drag the cursor to the right of the line, then let up.  When the entire line is highlighted in blue hold down the CTRL key and press C to copy it to the clipboard.

Finally, go to the Command Line window you have open, right-click on it somewhere in the black area and left-click on "Paste" from the menu that comes up and then press "Enter" on the keyboard.  Answer "Yes" to any questions, then Press "OK" when it says it was successful.

That's it!  Error gone!

Now for that line.  Here it is:

reg delete HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\OSUpgrade\Rollback /f

BE SURE to get the entire thing.  It ends with "/f" and it is all a single line.

Windows 7 will let you change resources like IRQs in the Device Manager. But you only have legitimate sharing, not conflicts.
 
Your only hope is to uninstall the drivers for all the onboard devices that you can, shut down, reboot and then disable the same onboard devices that use IRQs in the BIOS. Hi Rob,

Thank you for posting in Microsoft Community and being part of Windows 10.

I understand that you are unable to access the printer under desktop after upgrading to Windows 10 with error: Print queue is full, we will help you with this issue.

Try the below troubleshooting steps on desktop device and check if it helps.

Step 1: I suggest you to check the status of Print Spooler service and check.

Print Spooler service, spools print service and handles interactions with the printer. If you turn off this service you won’t be able to print or see your printer. 

Steps:

  1. Press Windows + R to open Run command, type Services.msc and press Enter.
  2. Search for Printer Spooler service, right click on it and Start the service.
  3. Also, set the Startup type to Automatic.
  4. Now check if you are able to install the driver.

Step 2: I suggest you to run Printer troubleshooter and check if it helps.
  1. Press Windows + X, click on Control Panel, select Category and then select Large or Small icons.
  2. Click on Troubleshooting and click on View all in the left pane.
  3. Select Printer and follow on-screen instructions.
