dirk goldgar say:
When I create action query in Access, there is always an warning message. I also know how to turn it off by using Set warning off in Macro if neccessary.
However, warning message is missing recently even I did not turn it off by using macro. 
So, I want to know how to turn warning message on by default, so that  It is only turned off in case I want to turn it off by using Macro
 


There are two possible reasons your warnings may have disappeared.  The first is that somewhere in VBA code you may have executed the line

 

    DoCmd.SetWarnings False

 

without a subsequent line to SetWarnings True again.  Unlike macros, where the SetWarnings action only lasts for the duration of the macro, in VBA code the SetWarnings method lasts as long as the Access instance is open.

 

The second possibility is that you turned off the application-level option "Confirm Action Queries", which is in the Client Settings page of the Options dialog.  If you uncheck the box for that option, Access won't display the action-query warnings you are expecting.

gord dibben say:

Assign this code to each picture.

 

Click once on picture to enlarge. . . .click again to reduce to original size.

 

Sub PictureBigSmall()
    Dim shp As Shape
    Dim big As Single, small As Single
    Dim shpH As Double, shpOriH As Double

    big = 2.5   'Set large scale, 2.5 = 250%
    small = 0.5 'Set small scale, 0.5 = 50%

    Set shp = ActiveSheet.Shapes(Application.Caller)

    shpH = shp.Height
    shp.ScaleHeight 1, msoTrue, msoScaleFromTopLeft
    shpOriH = shp.Height

    If Round(shpH / shpOriH, 2) = big Then
        shp.ScaleHeight small, msoTrue, msoScaleFromTopLeft
        shp.ScaleWidth small, msoTrue, msoScaleFromTopLeft
        shp.ZOrder msoSendToBack
    Else
        shp.ScaleHeight big, msoTrue, msoScaleFromTopLeft
        shp.ScaleWidth big, msoTrue, msoScaleFromTopLeft
        shp.ZOrder msoBringToFront
    End If
End Sub

suzanne s. barnhill say:

What Doug has showed you is not programming code but simple fields. You would put these fields in the header (or footer). For example, if you use Insert | Page Number (or, more simply, Alt+Shift+P) to insert a page number in the header and then press Alt+F9 to display the field code, you will see { PAGE }. The PAGE field inserts a page number based on your current Page Number Format settings.

To insert a field, you press Ctrl+F9 to insert the field braces, then type between them, repeating Ctrl+F9 as needed to insert fields within the field.

If you use a StyleRef field in the header, you don't need to start a new section each time the subhead changes; see http://office.microsoft.com/en-us/word-help/field-codes-styleref-field-HP005186193.aspx for instructions and http://WordFAQs.mvps.org/StyleRef.htm for more things you can do with StyleRef fields.

Also, even if the first page of the chapter doesn't have a folio (printed page number), the second page should still be page 2.

suzanne s. barnhill say:
You can include a title in the document body and make it invisible. Unfortunately, the Hidden format is the only one that is respected by the StyleRef field (if you make the title Hidden, then the StyleRef result will also be Hidden), so that is not helpful. But what you can do is insert the title and format the font as 1 point in size and white in color, with 1 point line spacing and no Spacing Before or After. This will make it virtually disappear; the downside, of course, is that it is easy to forget what you have done, and the "invisible" title can be almost impossible to find and deal with later.
