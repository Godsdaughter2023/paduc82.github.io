Open WD and look in Quarantine or All Detected Items and look carefully at the "complete path."  If you are using an HP computer and the detection is in the D drive of the computer then you are most likely seeing a false detection in the recovery drive.
 
If other scanners such as Malwarebytes, Kaspersky and Eset do not identify malware in the “D” Drive then that’s probably the false detection we’ve intermittently heard about (for several years) involving the D drive on HP computers. 
 
This is a known intermittent issue (being reported for years) involving probable false malware detections by MS antimalware programs only. Other scanners such as the Eset Online Scanner, Kaspersky Virus Removal Tool, Malwarebytes typically detect no issues in the D drive of HP laptops (installed by HP when the computer was manufactured) and thus far neither HP or MS has taken steps to prevent it from occasionally occurring.
 
There are two methods of dealing with this problem discussed on the HP Forum.  One solution is to exclude the D Drive from being scanned by Windows Defender or MSE depending on which program you are using.  The other solution is to remove/delete the recovery partition (D Drive).

You've never stated, nor has anyone asked, exactly what steps you're taking to create the AutoCorrect Entries, but...

In order for those options to be active you must insert the With: item into the document & select it before going into the AutoCorrect dialog. Be careful to not select the ¶ following the character (if there is one)... Select only the character but include a trailing space if you wish.

  1. Use the OS X Character Viewer to insert the symbols into a document in the preferred font [rather than Word's symbol tools],
  2. Select an OpenType TrueType font [such as Times New Roman] rather than Times, &
  3. Make sure your installation of Office 2011 is fully updated [14.3.9 is current].
