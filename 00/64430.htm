csapgo say:

Wrapped my head around it a little more and found the offending file in regedit after the following:

If you look in the Properties of the font, you'll notice its location. Apparently, newly installed fonts can end up in your Windows USER folder:

C:\Users\"USERNAME"\AppData\Local\Microsoft\Windows\Fonts

This led me to wonder if I would find the problem font in the USER registry, so I looked there:

HKEY_CURRENT_USER\Software\Microsoft\Windows NT\CurrentVersion\Fonts

The file, as well as the others listed in the Windows USER folder were visible. So I deleted the offending file, restarted my system, and re-installed the proper fonts I'd previously removed (they could otherwise have been left in the fonts folder to begin with, but I was trying to figure out which ones were locked out due to being "In Use". My documents look normal again.

If this helps anyone else to prevent the time wasted, yee haw.
