macropod say:

You would need to copy all the relevant content from the document body, then delete everything except the footnote references from the destination document. Assuming your footnote references are in the 'Footnote Reference' Style, that process is actually quite straightforward:

1. Format all the document body content in the destination document as hidden text.

2. Do a Find/Replace for the 'Footnote Reference' Style as the Find expression, with '^&' and 'not hidden' as the Replace expression.

3. Do a Find/Replace for hidden text as the Find expression, with nothing for the Replace expression.

speechappsdesigner say:

I experience this problem occasionally on Windows 7.  Through lots of forum searches plus trial and error, I have found there are two principal causes:

  1. Latency during auto-save operation because the default printer is on a network.
  2. Corrupted Normal.dotm template, which causes an error in auto-saves and results in the read-only transition.

First, try setting your default printer to a local printer, or (even better) to a virtual printer like OneNote.  If that does not resolve the probem, follow these steps to build a new Normal.dotm.

  1. If you have macros, edit one of them to bring up all the macro definitions and code.  Save all to a text file.
  2. Exit Word.
  3. Find the location of Normal.dotm on your operating system and rename it (don't delete it).
  4. Restart Word.  Word will create a fresh Normal.dotm.
  5. If you have macros to restore, create a new one to bring up the macro code.  Copy all from the saved text document.
  6. If any other customized settings need to be restored, you can manually open the old, renamed template to stare and compare.
