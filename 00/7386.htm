Alphabetizing Index in Word 2016 - Microsoft Community

jason ritter say:

Alphabetizing Index in Word 2016

I have an 80 page program book with 250 student names and information for each student.   I want to make an alphabetical index by student last name at the back of the book so students can locate their entry...

How do I do this?

macropod say:

Assuming you have a list containing just the student names in given name, surname, order, create a new document with that list. Then:

1. Insert an empty paragraph before that list.

2. Use a wildcard Find/Replace, where:

Find = [^13^l]([!^13^l]{1,})

Replace = ^p\1^t\1

This will duplicate all the entries, separating them with tabs.

3. Delete the empty paragraph you inserted.

4. Select the entire document and choose Insert|Table>Convert Text to Table>OK

5. Save the file with a name like 'Index Concordance', then close it.

6. In your program book, go to References|Index|Insert Index>Automark

7. Select your 'Index Concordance' document, then click 'Open'.

All your student names will now be marked for indexing.

8. Select wherever you want the index to appear, then go to References|Index|Insert Index>OK.

You will now have all the students indexed in alphabetical order by given name.

If you want the students to be listed by surname, then given names, you will need swap the surnames & given names around, in the table's second column, between steps 4 & 5, perhaps also adding a comma after the surname.

rich michaels - mvp (greatcirclelearning.com) say:

After some additional experimentation I found the answer to the problem... or at least a workaround for you.

The Mac Office Development Team has not implemented the Word VBA Object Model properly/fully (you choose the adjective) and the result is that you have to execute each method separately. In other words, when the methods are broken down individually, then you can programmatically fill the shapes.

    ActiveDocument.Shapes(1).Select
    ActiveDocument.Shapes(1).Duplicate
    ActiveDocument.Shapes(2).Fill.PresetGradient msoGradientVertical, 1, msoGradientGold

The above code assumes there is only 1 shape in the document collection and when it is duplicated, then the selected shape is now the 2nd shape in the document. If you are working with documents that already have more than 1 existing shape then a different method of identifying which shape to duplicate and then fill, will have to be programmed.

suzanne s. barnhill say:

As Paul and Charles have said, this setting is part of the relevant TOC style. By default, Heading 1 entries in the TOC are in TOC 1 style, Heading 2 in TOC 2, and so forth. Because these styles are set to update automatically, many changes that you make in a given entry will be made to the style and propagated to the other entries at that level.  Unfortunately, this process is entirely unreliable for tab settings (I found that changes I made in TOC 1 were propagated to TOC 2 instead and other anomalies).

So, assuming your chapter titles are Heading 1, you need to make the change in TOC 1. With the insertion point in a TOC 1 entry, press Ctrl+Shift+S to open the Apply Styles dialog. With any luck, TOC 1 will already be selected. If not, scroll to it. Click Modify... (You may also see TOC 1 in the Styles pane; if so, you can right-click and choose Modify... there.) In the Modify Style dialog, click Format, then Tabs... Change the leader style to None and click OK twice to close the Tabs and Modify Style dialog.

That's how it's supposed to work. Unfortunately, it doesn't for TOC styles, and I can only assume that that is because the tab stops are set dynamically, based on margin width, when the TOC is inserted (sometimes the Tabs dialog will not show any tab setting at all). To make this work, therefore, you have to insert an extra step. With the insertion point in a TOC 1 entry, drag the tab marker on the horizontal ruler slightly to the left. You'll see all the TOC 1 entries follow suit. Then double-click the tab marker to open the Tabs dialog, change the leader to None, and click OK. Then drag the tab marker back to the margin. If you have the default 1" margins, the original tab position will have been 6.49", and you'll end up dragging it to 6.5", but only the sharpest eye could see the difference.

For more on formatting your TOC, see http://wordfaqs.ssbarnhill.com/TOCTips.htm#Appearance

Update: I have discovered further oddities about modifying tab leaders in TOC fields and have added a section to the above-referenced article under the heading "A frustrating glitch." I haven't entirely sorted out the baffling logic of Word's behavior, but I think I've presented a (tedious) workaround.
