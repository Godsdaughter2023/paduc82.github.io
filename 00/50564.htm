daavee say:
The actual message is as I quoted above "The CD Key you entered is not valid".

 

These are the typical reasons one would get such a message:

 

1. User enters the key not associated with the Retail disk but another key, usually the one on the PC's COA sticker.

 

2. At least one of the characters was entered incorrectly. This could be because of user error or a faulty keyboard.

 

Just to make sure: Are you using a real Retail CD? Or is this a generic OEM CD that was purchased at a retail store?

 

One way around this issue is to use an XP installation CD that has SP3 integrated into it. If your CD is SP2 or earlier, you can create one at the SP3 level, using a process called slipstreaming. Then simply when it is time to enter your Product Key, DON'T! Then you will see this window:

 

http://ethertubes.com/wp-content/uploads/winxpsp3-2.png

 

You will have thirty days to activate (and enter the Product Key). This page should help:

 

http://windows.microsoft.com/en-US/windows/help/genuine/product-key

noel d paton say:

Slipstreaming is a complex process - and it may have gone awry.

While your original disk may be perfect, the slipstreamed disk may be missing parts of the code which are relevant for Key-parsing, witht the result that pre-SP3 codes are not properly recognised (MS released a new tranche of Keys for SP3 and netbooks, so the coding must have changed).

I'd suggest creating a fresh slipstreamed disk.

(burn the disk at the slowest possible speed, as well)

Bug Report Task Manager - Microsoft Community

ff-2018 say:

Bug Report Task Manager

This bug was introduced in Windows 1809 (Redstone 5 Oct 2018 Update). The Task Manager reports total CPU use correctly, as is confirmed by the Resource Monitor. The Resource Monitor reports CPU usage by individual apps correctly but the Task Manager does not. Instead the Task Manager reports a fictitiously low figure, less than 10% of the actual usage for individual apps, even though total CPU use is reported correctly. This bug can be reproduced by doing a CPU-intensive task such as video encoding (H264 or H265), and in multiple instances if need be.
