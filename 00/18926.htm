daniel g. grau say:

The Web Page format attempts to encode all of the file's info as HTML code. The idea is that this format, while being HTML code, will yield a file that is very close to the original .docx from which it was prepared. If you open such files in a text editor, you'll see lots of Word-only code (like zoom factor, working time and unused styles), all in the form of comments.

Web Page Filtered strips out all of the Word-only stuff , leaving only essential HTML code.

It would seem that Word has trouble encoding it's own parameters as HTML code and crashes. Try saving as .RTF or .doc, close the file, reopen it and create the HTML file from there.

For more details on both Web Page formats, take a look here.
