jllatham say:

I didn't quite understand exactly what you wanted initially, but I believe I do now.  The routine below could be called from your existing processing when you need to sort the data.  It will sort it "in place" on the sheet it is in.  It is set up to sort the range A2:D295 in the custom order you need, with a secondary order of "by CUSIP".  Any of that can be changed by editing the Const values in the code - I've highlighted those so you know which ones you can change as required (like the name of the worksheet itself).  Test it with a copy of your actual workbook to make sure it works the way you need it to.

Sub CustomListSort()
  'change these to match your worksheet name
  'and column layout
  Const dataSheetName = "Sheet1"
  Const firstDataRow = 2 ' assumes 1 has labels
  Const lastDataRow = 295 ' make =0 to include all
  Const firstColToSort = "A"
  Const lastColToSort = "D"
  'this is column w/Investment Types in it
  Const keyCol_1 = "D"
  'for a second sort by col, such as CUSIP Number
  Const keyCol_2 = "A"
  'end of user editable values
  'working variables
  'this defines the custom sort list order
  Const CSOList = _
   "BANK LOANS,ASSET BACKED SECURITIES,COMMON STOCKS," _
    & "CORPORATE BONDS,PREFERRED STOCKS,SHORT-TERM INVESTMENTS,CASH"
  Dim dataWS As Worksheet
  Dim sortRange As Range
  Dim sKey1 As Range
  Dim sKey2 As Range
  Dim lastRow As Long
 
  Set dataWS = ThisWorkbook.Worksheets(dataSheetName)
  If lastDataRow = 0 Then
    'use all entries based on entries in keyCol_1
    lastRow = dataWS.Range(keyCol_1 & Rows.Count).End(xlUp).Row
  Else
    'use value assigned by user
    lastRow = lastDataRow
  End If
  Set sortRange = dataWS.Range(firstColToSort & firstDataRow & ":" _
   & lastColToSort & lastRow)
  Set sKey1 = dataWS.Range(keyCol_1 & firstDataRow & ":" _
   & keyCol_1 & lastRow)
  Set sKey2 = dataWS.Range(keyCol_2 & firstDataRow & ":" _
   & keyCol_2 & lastRow)
  With dataWS.Sort.SortFields
    .Clear
    .Add Key:=sKey1, SortOn:=xlSortOnValues, Order:=xlAscending, _
     CustomOrder:=CSOList, DataOption:=xlSortNormal
    .Add Key:=sKey2, SortOn:=xlSortOnValues, Order:=xlAscending, _
     DataOption:=xlSortNormal
  End With
  With dataWS.Sort
      .SetRange sortRange
      .Header = xlNo
      .MatchCase = False
      .Orientation = xlTopToBottom
      .SortMethod = xlPinYin
      .Apply
  End With
'sort is done, release assigned resources back to system
  Set sortRange = Nothing
  Set sKey1 = Nothing
  Set sKey2 = Nothing
  Set dataWS = Nothing
End Sub

cbarnhorst say:

Windows 10 activation is based on the device and not on the product key (unlike all previous versions of Windows).  All the product key is for is the initial registration of the device as a "registered Windows 10 device" with MSFT.  From that point the PK is not needed unless the unique indentifiers of the device change.

You will not get anywhere focusing on Windows.  It's all about the device (your VM in this case).

Transferring a Windows 10 license from one VM to another as a repair replacement requires use of the activation wizard on the new VM or help from Microsoft Support.  Your strategy is to concentrate on this issue as a repair replacement of the old VM.

Be aware that it is perfectly possible to copy a VM without having to activate the copy afterwards.  This is my strategy for protecting my VMs.  Your virtualizaion solution should include an export/import feature.  I use that to export all my Hyper-V VMs to a folder on a backup drive.  When one of my VMs gets messed up I delete it and import the archived version.  I import as a copy and choose to have the Hyper-V manager give it a whole new identity but retain the name.  This all works because MSFT views VMs as primarily a buisiness product and not consumer.  Most VMs are used in domain-based networks where it is necessary to move VMs freely between servers (for load-balancing and such).  Requiring re-activation would be insane in that environment.  If you have even one Windows 10 VM exported you should be home free for replacing your problematical VM without having to purchase a new license.
