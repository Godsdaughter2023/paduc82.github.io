burrwalnut say:

I think the ‘mystery’ as you call it arises from the myriad of different ways of doing the same thing. Extensions, associations and defaults can be added/changed/initiated via typed commands, manual registry changes, control panel choices/defaults, double-clicking, right-clicking and choosing default or open with. Then there are installation choices, ‘make this program the default’, ‘allow all users access’ or ‘current user only’. I’m sure there are others that I’ve missed, but you can see what I mean.

 

1. Anyway, to recap, HKEY_CLASSES_ROOT is the merged view of HKEY_LOCAL_MACHINE\Software\Classes and HKEY_CURRENT_USER\Software\Classes. If there is an entry in HKCU\Software\Classes it takes precedence over the HKLM. If you delete a key in HKCR it is removed from HKLM and/or HKCU and vice versa.

 

Here is a scenario that makes it a little easier for me to explain the HKCU. Let's say we have two users on the computer, Jack and Jill, who want to use different programs to manipulate their photographs. A well-written setup program will ask the user if all users are to have access or just the current user (if this installation option is missing, it has to be set in the logged-on user’s Control Panel). So Jack installs program A which creates entries in his HKCU. Next, Jill logs on and installs program B and creates HKCU entries. When each of their HKEY_CLASSES_ROOT is examined, Jack’s will look different to Jill’s. I hope this makes sense.

 

2. KEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\extension\OpenWithList contains the entries made by right-clicking and making an Open With selection. To delete a program from here, look in the right pane, right-click the unwanted program entry and choose Delete.

 

3. In my summary (6), I was referring to your deletion of actual extensions, not my example, which I said delete the file type in (4). I also said delete the HKLM extension keys because you said there was nothing in HKCU. In reality it’s best to delete from the merged view of HKCR.

 

4. I don’t know anything about Ramesh’s program, so I cannot comment other than he’s a highly-respected person who knows what he’s talking about.
