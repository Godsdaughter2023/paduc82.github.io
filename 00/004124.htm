hansv mvp say:

Here is the code for the function.

Function ConcatenateIf(CriteriaRange As Range, Condition As Variant, _
        ConcatenateRange As Range, Optional Separator As String = ",") As Variant
    Dim i As Long
    Dim strResult As String
    On Error GoTo ErrHandler
    If CriteriaRange.Count <> ConcatenateRange.Count Then
        ConcatenateIf = CVErr(xlErrRef)
        Exit Function
    End If
    For i = 1 To CriteriaRange.Count
        If CriteriaRange.Cells(i).Value = Condition Then
            strResult = strResult & Separator & ConcatenateRange.Cells(i).Value
        End If
    Next i
    If strResult <> "" Then
        strResult = Mid(strResult, Len(Separator) + 1)
    End If
    ConcatenateIf = strResult
    Exit Function
ErrHandler:
    ConcatenateIf = CVErr(xlErrValue)
End Function

You should activate the Visual Basic Editor, select Insert > Module, then copy/paste the code into the module.

Then switch back to Excel.

Make sure you save the workbook as a macro-enabled workbook (.xlsm).

Private Sub MultivalueField_AfterUpdate()

 

Dim rs As DAO.Recordset

Dim rsMV As DAO.Recordset

Dim strOut As String

Dim lngLen As Long

Dim strResult As String

Dim strSQL As String

 

strSQL = "SELECT [MultivalueField] FROM [Tablename]"

Set rs = DBEngine(0)(0).OpenRecordset(strSQL, dbOpenDynaset)

 

Do While Not rs.EOF

        Set rsMV = rs(0).Value

        Do While Not rsMV.EOF

            If Not IsNull(rsMV(0)) Then

                strOut = strOut & rsMV(0) & ", "

            End If

            rsMV.MoveNext

        Loop

        Set rsMV = Nothing

    rs.MoveNext

Loop

 

lngLen = Len(strOut) - 2

If lngLen > 0 Then

    strResult = Left(strOut, lngLen)

    Me.[stringFieldName] = strResult

End If

 

Set rs = Nothing

Set rsMV = Nothing

 

End Sub

With Workbooks("SERVICES CKG 2011.xlsx")
    For Each myS In .Worksheets

Updating two fields with a combo box - Microsoft Community

rob ingles say:

Updating two fields with a combo box

Been struggling with this for a couple days now - is it possible to store two values using a combo box?

I have a combo box that I want to update two fields on my form rather than one.

Combo box cboPositionServiceType has three columns:

Column 0: PositionI_Service_TypeID - Control Source

Column 1: Position_Service_Type -

Column 2: CostCenterID - I would like this to update the CostCenterID text box on my form

It appears that I cannot have two Control Sources so how do I go about updating PositionI_Service_TypeID  and

CostCenterID?

hansv mvp say:

Since CostCenterID depends on PositionI_Service_TypeID, there is no need to store it in the record source of the form: it would be redundant. But you can make a text box display it by setting the Control Source of the text box to

=[cboPositionServiceType].[Column](2)

The text box will be updated automatically when the user selects an item from the combo box.

dirk goldgar say:

From what you've written, PositionI_Service_TypeID completely determines CostCenterID -- that is, every PositionI_Service_TypeID has only one CostCenterID.  If that is so, then there is no reason to *store* both PositionI_Service_TypeID and CostCenterID in another table, and doing so would create a redundancy and open the door to the possibility of conflicting data.  By contrast, if you store only the selected PositionI_Service_TypeID in your form's underlying table, then a query can always determine the associated CostCenterID by looking it up in the related table.

On the other hand, your form can readily *display* the data in column 2 of the combo box, using a text box with its ControlSource property set to this expression:

   =[cboPositionServiceType].[Column](2)

Now, it could be that you need to store the CostCenterID as it was at the time the selection was made, and preserve that information even if later the CostCenterID associated with that PositionI_Service_TypeID should change.  In that case, you do need to store that CostCenterID separately, and can use code in the combo box's AfterUpdate event to copy the combo column's value into a separate bound text box.

scottgem say:

To state it another way the answer to your question is yes. You can have multi columns in the RowSource of a combobox and reference those values using the Column(x) property.

However, as noted, generally the bound column of a combobox is a unique identifier for the record being selected. If that is the case, then there is little need to store any values related to that identifier since it would be redundant.

My Zune's battery indicator always reads 3/4 full when the - Microsoft Community

skiracerdude say:

My Zune's battery indicator always reads 3/4 full when the battery dies. Whats the problemo?

My Zune's battery indicator always reads 3/4 full when the battery dies. Whats the problemo? I have a 16 GB.
nsilmike say:

common issue with lithium battery gauges...they lose calibration.  to recalibrate:

1) set your zune to play and let it go til it dies.  speed this up if you like by turning on wifi, setting the display to bright, etc.

2) when it dies, start it again. repeat until it stays on only maybe a few minutes at most.

3) connect it to charge and let it charge without any interruptions.  Done.

4) don't forget to change your settings back (brightness, etc.)

 

What you are doing here is retraining the gauge to correctly recognize empty and full.  It's 'normal' and does not indicate any problem.
