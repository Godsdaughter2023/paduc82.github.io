Any function to solve 'n' within the quadratic formula? - Microsoft Community

oemsoft say:

Any function to solve 'n' within the quadratic formula?

n^2-n(12X+13.6125)+36X^2=0

Does anyone have any suggestions on any Excel's function to solve 'n' within the quadratic formula?

 

Thank everyone very much for any suggestions
joeu2004 say:

oem7110 wrote:

n^2-n(12X+13.6125)+36X^2=0
Does anyone have any suggestions on any Excel's function to solve 'n' within the quadratic formula?

Sure, assuming that X is a constant, or at least  not  a function of n.

Suppose that B1 is =12*X1+13.6125, and C1 is =36*X1^2.  Then your formula is in the classic form that can be solved with the quadratic formula (see http://en.wikipedia.org/wiki/Quadratic_equation), namely:

n^2 - B1*n + C1 = 0

n = (B1 +/- SQRT(B1^2 - 4*C1)) / 2

So in N1:

=IF(B1^2-4*C1<0,NA(),(B1+SQRT(B1^2-4*C1))/2)

And in N2:

=IF(B1^2-4*C1<0,NA(),(B1-SQRT(B1^2-4*C1))/2)

Note there are theoretically two solutions to such formulas.  However, the pragmatics of your problem might rule out one solution.  For example, if you are counting something, a negative number is not valid for your purposes, even though it might be mathematically correct.

ken sheridan say: You can put code in the form's BeforeUpdate event procedure which sets the return value of the Cancel argument to True if the validation criteria are violated.  As the combo box appears to be bound to a hidden numeric foreign key, you need to reference its second column not the value of the control, e.g.

Const MESSAGE_TEXT = "Invalid case status."
Dim blnCriteria As Boolean

'the following will evaluate to TRUE or FALSE
strCriteria = Not IsNull(Me.[Case Closed Date])  & Nz(Me.[Case Status].Column(1),"") <> "Completed"

If blnCriteria Then
    MsgBox MESSAGE_TEXT,vbExclamation,"Invalid Operation"
    Cancel = True
End If

This will disallow the saving of the form.  If, on the other hand, you wish the user to be able to override the warning:

Const MESSAGE_TEXT = "Invalid case status.  Do you wish to continue nevertheless?"
Dim blnCriteria As Boolean

'the following will evaluate to TRUE or FALSE
strCriteria = Not IsNull(Me.[Case Closed Date])  & Nz(Me.[Case Status].Column(1),"") <> "Completed"

If blnCriteria Then
    If(MsgBox MESSAGE_TEXT,vbYesNo + vbQuestion,"Invalid Operation") = vbNo Then
        Cancel = True
    End If
End If

Note that the Column property of a combo box is zero-based, so Column(1) is the second column, i.e. the visible column.

This should work fine if the user tries to save the record without closing the form, but if they attempt to do so by closing the form it won't be entirely satisfactory.  In situations like this my preference is to force the user to save the record via a command button before they can close the form.  You'll find an example as SaveDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.
