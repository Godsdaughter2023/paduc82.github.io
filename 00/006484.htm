Dsum Calculation Results in "#Error"

Hi

In a very simplistic Example I have a table called "Testdata", which contains three fields:
Text Field Called "Name" which contains the customer name
A Date Field called "TestDate" which contains a Date (In UK format (dd/mm/yyyy) as that is where I am based)
A Numeric Field call "Value" which contains the value of the transaction

What I am trying to do is for each customer I would like a running sum based on the previous balance plus or minus the current transaction value. So, for Example, I might have Customer A with the following transactions:

Customer A  -  01/01/2012 = £100
Customer A  -  28/02/2012 = (£50)
Customer A  -  31/03/2012 = £75

I would like the RunningSum to be shown as follows:

Name              Date               Value     Running Sum
Customer A  -  01/01/2012 = £100   -  £100
Customer A  -  28/02/2012 = (£50)   -   £50
Customer A  -  31/03/2012 = £75    -   £125

I then want to repeat the process for Customer B, & Customer C etc etc

I have found the following SQL code on-line which I have adapted to suit my field and table names, but all I get in the RunningSum Column is "#Error"

SELECT NAME, TESTDATE, VALUE, 

DSum("[VALUE]","TestData","[NAME]=" & [NAME] & 

" And [TESTDATE]<=#" & [TESTDATE] & "#") AS RunningSum

FROM TestData

ORDER BY Name, TestDate;


Any help that anyone can provide would be very much appreciated.


Note that I have tried this on Access 2007 running on an XP machine and also Access 2010 running on a Windows 7 Machine with the same results

 

karl donaubauer say:

Hi,

 

the syntax for both criteria is wrong. In your SQL string try to replace the DSum field by this:

 

DSum("[VALUE]","TestData","[NAME]=" & Chr$(34) & [NAME] & Chr$(34) & 

" And [TESTDATE]<=" & clng([TESTDATE])) AS RunningSum

 

The Chr$(34) build quotes arround the string parameter, clng() converts the date to a long that is processible for the query engine.

 

BTW I hope these field names really only exist in your example. "Value" and "Name" should be avoided as they are keywords in Access and VBA.
