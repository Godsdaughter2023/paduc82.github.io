Slowdown of PC During MSE Updates? - Microsoft Community

stephen boots say:

Slowdown of PC During MSE Updates?

Recently, there have been a large number of posts in the forums regarding PC performance problems occurring during MSE's automatic check for updates, which should typically happen once per 24 hours or so (Update FAQ -http://social.answers.microsoft.com/Forums/en-US/mseupdate/thread/74e507b8-f6da-4eca-8ce7-d1aca7d3f1ba).

Note: You cannot change the schedule or choose a time for MSE definition updates, nor should you attempt to disable updating as the latter will impact your computer's security.

Obviously, MSE updates should *not* impact performance in any way.

If your PC becomes virtually unusable or is particularly slow and unresponsive during the update process (you'll notice the MSE tray icon animate with a blue down arrow overlaying the normally static green "house" image), please confirm that you are not running any other security software that provides real time protection.
(See: Check list for installing Microsoft Security Essentials)

If you have confirmed that there is no conflict, please post the following in a reply to this post:

1. What is your operating system?

2. How much RAM is installed (Control Panel/System will tell you this if you are unsure)

3. What is your Internet connection method - broadband or dial-up?

4. Does the performance issue only happen when an automatic update occurs? Open MSE by double clicking the tray icon and click on the update tab and the update button.

5. What version of MSE are you using? (Open MSE, click the down arrow next to Help in the upper right, then click About...)

Additional questions suggested by PA Bear (once again, see: Check list for installing Microsoft Security Essentials):

  • When (approx. date) did you install Microsoft Security Essentials and was the computer fully-patched at Windows Update at the time?
  • What anti-virus application was installed before you installed Microsoft Security Essentials, was your subscription still current, and did you uninstall it before you installed Microsoft Security Essentials?
  • Has a(nother) Norton or McAfee application ever been installed on the computer?
  • Did a Norton or McAfee free-trial come preinstalled on the computer when you bought it? (Doesn't matter if you never used or Activated it.)

Additionally, please open a support case to report the issue, too. (To open an email support case, start here:https://support.microsoftsecurityessentials.com/)

Some people using Windows XP have reported that temporarily switching from Microsoft Update to Windows Update has resolved the issue for them. You will still receive important Windows security updates and MSE definition updates using Windows Update, but some other Microsoft programs will not receive updates through this path. If you do switch to Windows Update, be sure to check back regularly to see if the problem has been identified and corrected by Microsoft so that you can switch back to Microsoft Update. (see this KB article for how to switch this on XP - http://support.microsoft.com/kb/901037)

Try this

=IFERROR(INDEX($A$1:$A$10,SMALL(IF(1*(INDEX(($B$1:$B$10+$C$1:$C$10)>0,,))=1,ROW($A$1:$A$10)),ROW(1:1)),1),"")

mark_n13 say:

Hi

If your message is stuck on "Sending" status, it doesn't mean that your contact request is pending.

Messages status should say "Sent" unless there was an issue with sending the message.

If you are using the Skype app for Windows 10 version I suggest reset the app.

Steps are as follows:

1. Sign out from the Skype app and then end the application from the task manager
2. Restart your computer
3. Open the Start menu and click Settings
4. Click Apps
5. Locate and click on the Skype icon from the Applications list
6. Click Advanced Option
7. Select Reset
8. Re-launch Skype and login using your Skype account

Let me know what the outcome is.
