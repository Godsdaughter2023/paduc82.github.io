mccbleue say:

The phone memory is dynamic, meaning that it is shared between various functions including processing data related to features being used and storage. The amount of free space on your memory card is not relevant here, as it is used only for storage. The phone still needs sufficient phone memory free to process the incoming image. 

 

With S40 phones like this, an incoming Bluetooth transmission (you don't say in your post, but I assume this is what you mean) could happen in one of three ways:

 

(1) If there is enough space in phone memory, then the file will be stored in the "Received files" folder. If it is a large file (such as music, video or high-resolution image) it is good practice to move it to memory card afterwards.

(2) If there is not enough space to store the file in phone memory but there are sufficient system resources to process it and a memory card available, then the file will be stored straight to the memroy card.

(3) In the absence of sufficient system resources, the receipt of the transmission will be abandoned.

 

It is possible that a software update may inadvertantly contain a bug, but this is not likely the case as other users are not reporting this issue en masse. In case something went wrong with your update, you could try reinstalling the current software; however first check your phone memory because it is far more likely that there is not enough free space.

The following requires a reference to the Microsoft DAO 3.6 Object Library

Dim myDataBase As Database
Dim myActiveRecord As Recordset
Dim i As Long, j As Long
'Open a database
Set myDataBase = OpenDatabase("C:\Users\Doug\Documents\Cost DataBase\Procurement Plan.mdb")
'Access the first record from a particular table
Set myActiveRecord = myDataBase.OpenRecordset("Currencies", dbOpenForwardOnly)
'Populate the cells in the first row with the field names
For i = 1 To myActiveRecord.Fields.Count
    ActiveWorkbook.Sheets(1).Range("A1").Offset(0, i - 1).Value = myActiveRecord.Fields(i - 1).Name
Next i
'Loop through all the records in the table until the end-of-file marker is reached
j = 1
Do While Not myActiveRecord.EOF
    'Populate the cells in the the data from the current record
    For i = 1 To myActiveRecord.Fields.Count
        ActiveWorkbook.Sheets(1).Range("A1").Offset(j, i - 1).Value = myActiveRecord.Fields(i - 1).Value
    Next i
    j = j + 1
    myActiveRecord.MoveNext
Loop
'Then close the database
myActiveRecord.Close
myDataBase.Close

Build 14986 is failing to get along with Windows 7 drivers, especially graphics drivers, that worked well up through build 14971. You didn't say, but I presume you're using the onboard graphics adapter.

In the case that I'm personally familiar with, the graphics driver is Nvidia, written for Windows Vista/7, with no newer Nvidia drivers available ... and the unit crashes a minute or two after login, yielding a variety of different error messages.

If that driver is deleted and replaced with the Windows basic graphics driver, the unit will keep running, but not with the desired resolution.

Since your machine crashes in a few seconds, one of the other drivers might be causing it, instead, so, if you can have time to delete the graphics driver before the crash, and then restart, you will be able to see.

If no newer graphics driver is available for the onboard adapter, you could change to an add on graphics card that uses a newer driver.

"This Identity Cannot Be Opened With This Version of Outlook" - Microsoft Community

syedkaleem pasha say:

"This Identity Cannot Be Opened With This Version of Outlook"

Is there a solution for this issue?


dianeoforegon say: It's unclear exactly what situation led you to this topic.

This Identity Cannot Be Opened With This Version of Outlook"

1)  This message usually means any Identity saved with a newer version will not open in an earlier version. We see this a lot when users do a re-install of Office then fail to upgrade.

There was a database upgrade in 14.1.0, 14.1.3 and again in 14.2.0. Until you update the Identity will not open.

2)  If you experienced this error after upgrading your Identity from an older version of Outlook e.g. 14.1.x to 14.2.x then you will have to revert to Office 14.1.x in order to recover your data.

Once we know which situation is causing the problem, we can offer help.
