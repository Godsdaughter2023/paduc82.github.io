pfg_juli say:

I finally, with Ken Sheridan's help of course!, nailed this down. Just need to go back and make super sure that everything is named properly in order to get them working without strange errors....

 

Before the code will work, you will need to:

 

a) save your Word Docs as Templates (a simple save as feature where you can change the file type is all you need). make sure the field names in the document match exactly the names on the database, or you will run into a headache. [my current problem... the names don't seem to want to save properly]

 

b) create a query where you pull all of the data you will need for filling your word merge document, use multiple tables if necessary.

 - name it: "qry[name of your query]"

 

c) enter the following codes: one is for a general module, the other is for the on-click.

 

Code for a general module in your database so you can use the CurrentFolder() option throughout:

 


Public Function CurrentFolder() As String

    Dim strFullPath As String
    Dim n As Integer
    
    strFullPath = CurrentDb.Name
    
    n = InStrRev(strFullPath, "\")
    
    CurrentFolder = Left(strFullPath, n)
    
End Function

Code for on click of button:

 

Dim pathMergeTemplate As String
Dim appWord As Object
Dim docWord As Object

On Error GoTo errorhandler

'Get the word template from the Letters folder

   pathMergeTemplate = CurrentFolder()


' Export the data using TransferText

                DoCmd.TransferText _
                    acExportDelim, , _
                    "qry[name of your query]", _
                    pathMergeTemplate & "qry[change to something quick & unique to this particular merge. ie Acct]MailMerge.txt", _
                    True
                Set appWord = CreateObject("Word.Application")

                appWord.Application.Visible = True

' Open the template in the Resources\Letters folder:

                Set docWord = appWord.Documents.Add(Template:=pathMergeTemplate & "[full name of your word doc template, include the '.dotx']")

                docWord.MailMerge.OpenDataSource Name:=pathMergeTemplate & "qry[same as the quick & unique to this particular merge as above. ie Acct]MailMerge.txt", LinkToSource:=False

                docWord.MailMerge.Execute
                docWord.Close False
sub_exit:
            Set docWord = Nothing
            Set appWord = Nothing
            Exit Sub

errorhandler:
            MsgBox Err.Description & " " & Err.Number
            Resume sub_exit

 

Good Luck!

teylyn say:

First, you are stacking the End date on the bars, instead of the duration. Assuming that you want to see the duration in the Gantt chart, you need to calculate the difference between the End date and the Start date, so use a calculation like

=C6-B6

and copy down. Now you have the duration in days for each task. Select the orange series in the chart and move (drag) the selection in the worksheet to the duration column (or use your favourite way to set the series data source).

For the minimum and maximum values of the horizontal axis, decide which date you want to see at the  crossing of the vertical and the horizontal axis.  I assume it should be 1/1/17, since that is the start of the first task.  Write that date into a cell and format the cell as "General". The number you see needs to go into the "minimum" setting for the horizontal axis, i.e. 42736.

Use the same approach to determine the maximum date you want to see on the chart.

The horizontal axis of a bar chart is not a time axis, so you cannot set the interval for the units to months. You'll either have to compromise using fixed time intervals like 30 days, or hide the axis labels and use a more involved charting technique with a helper series on the secondary axis. Explaining that is outside of the scope of this question, though. 

Finally, select the blue series and format it to have no fill and no line to make it invisible. 

Let me know if that gets you any further.

If ms anti-virus means Microsoft Security Essentials, then I'm somewhat at a loss. There are basically only three ways items can vanish from an Outlook folder:

1. The user deletes it or moves it.
2. It meets Auto Archive criteria and is archived.
3. An Outlook add-in removes or moves it.

So, we're pretty sure you didn't delete or move the messages and, from what you say, it is unlikely Auto Archive had anything to do with it, so we're left with an add-in. The overwhelming majority of the time, this add-in takes the form of an Anti Virus/malware email scanner.supplied by a third party A/V application. These scanners do far more damage than the malware they're supposed to keep out. In these cases, the solution is to remove the email scanning components. MSE, however, is the only A/V product I know that "gets it" and does not have an email scanner. If you're absolutely sure that no other A/V application is installed on your system (and they're sneaky, often hitching a ride with Java and Flash player updates), then I'm at a loss. I have:

  • tried deleting the paragraph mark
  • changing the size of the paragraph mark to a smaller font
  • changing the paragraph spacing on the last TOC entry to single spaced
  • verifying that the font is correct (Times New Roman)
  • verifying the paragraph spacing for the entire table and specific line have "Before or After" Spacing set to zero.

I've searched and not found any other posts that seem to be helpful.  I'm hoping that someone here will have new ideas on how to remove this paragraph mark before I make the radical decision to try to shorten or lengthen my TOC headings.

Thank you, in advance, for your help.

stefan blom say:

If you are saying that there is a blank paragraph after the TOC field, you can change its formatting so that it doesn't affect the layout of the page.

  

Note that I don't recommend deleting the paragraph mark. After a table of contents field is the only place where you should actually keep a blank paragraph.

   

Do the following: Press Alt+F9 to display the TOC field code. Select the paragraph mark immediatey below the field and change its formatting so that it has 1 point of font size, line spacing "Exactly" 1 point, and spacing before/after set to zero.
