john mcghie say:
Hi Don:

I have never had much faith in AutoRecover: now you can see why.

The first thing you must do to get AutoRecover to work is to SAVE the document.

AutoRecover is actually a "Temporary Logging File" mechanism.  It saves only the CHANGES to a file, and only after the Original file has been saved at least once so that it has a file on disk with a file name.

So the first problem is that if the file has NEVER been saved by the user, AutoRecover has nothing to record the changes to, and so it doesn't.

The next problem is that if whatever damaged the main file damaged it to the point that Word is unable to read the main file, then it cannot apply the AutoRecovery changes to it, so again you get nothing back.  AutoRecovery can recover your changes only if it can read the main file and then apply the stored changes to what it reads.

The third problem with AutoRecover is that if Word does not know that it has crashed, it will discard the AutoRecover files when it opens.  This can be a problem if you quit Word and re-open it, then quit and re-open again after a failure.  On the second re-open, it will clean up all the AutoRecover files.

So:  Your last line says it all: "what do I do to make the autosave feature autosave?"  Sadly, the answer is "Word does not have an autosave feature, there is nothing you can do to make it autosave because it cannot do that." (You can write a macro to automatically save, but that means you cannot use any other macros, and the operation of it is so annoying you would most likely switch it off after a few minutes.  It locks up your cursor and typing every four minutes until the save completes.)

That said, there are things you can do that will make data loss like this much less likely.

The first is to save the file as a new name each morning when you start work.  That way you can never lose more than a day's work.  I pre-pend the date in ISO format to the front of the file name so the versions sort nicely in the folder and I do not get confused: Files I was working on today are 20131202 File Name Here.docx

Now, go to Word>Preferences>Save and turn on "Always create backup...".  That causes Word to write a new file each time you save.  The previous file is renamed to "Backup of..." and sits in the folder just above the current version.  A new file, each time you save: makes document corruption less likely, and if Word blows up, the document as it existed one paragraph ago when you paused to think and hit Command + s,  is right there in the same folder.

Of course, Time Machine will save a copy every hour.

There is a new version of Word for Mac on the way.  We can expect that this will have been designed around Mavericks: in which case, we can hope that it will support the automatic versions saving that is new in Mavericks.

In the meantime, turn on "Always create backup..." -- next time it blows up you will be so glad you did.

Hope this helps
