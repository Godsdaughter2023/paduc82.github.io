It sounds like one ore more of the P21, O21 and/or W21 cells may contain empty strings returned by formulas. Empty strings are not the same as blanks, but you can check for a zero length which is the same as a blank. =IF(LEN(P21),SUM(P21,-O21,-W21),"")

=IF(LEN(P21),IF(LEN(W21),SUM(P21,-O21,-W21),SUM(P21,-O21)),"")

andreas killer say:
You said "Text format" that means the content of the cell(s) is not a number, it is a text.

A text (that looks like a number) is not converted to a number if you copy and paste (even if you paste as values, because a text value stays as a text value). And it doesn't matter how you've formatted the destination cell.

You have to convert the text into a number first. Please read this article:
https://support.office.com/en-us/article/conver...

The opposite, copy a formatted number to a text (e.g. if you have a number in a cell and the cell if formatted as currency), is not possible using copy & paste. You need a workaround:

a) Convert the number to a text using a formula, e.g.
=TEXT(A1,"$#,##0.00")
b) Copy that cell
c) Paste into an other cell as value

Now you really have a text in the cell which can't be formatted.

graham mayor say:

The macro you have listed does not require that the user be connected to your company network. It requires only access to an e-mail account when the user is working remotely, and of course Microsoft Excel and Outlook installed on the remote PC.

Whatever type of account you are using to provide the company e-mail accounts, it should be possible to connect to the mail server wherever there is internet access available.

The question then should not be about the macro, which looks OK, but how your users connect to their e-mail accounts when working out of the office.

You might find it better to use the following alternative version of the code:

Sub Mail_with_outlookD()
Dim OutApp As Object
Dim OutMail As Object
Dim olInsp As Object
Dim wdDoc As Object
Dim oRng As Object
Dim strto As String, strcc As String, strbcc As String
Dim strsub As String, strbody As String
Dim signature As String

    'Save the workbook to ensure any changes are saved before attaching it
    ActiveWorkbook.Save
    
    On Error Resume Next
    'Get Outlook if it's running
    Set OutApp = GetObject(, "Outlook.Application")

    'Outlook wasn't running, start it from code
    If Err <> 0 Then
        Set OutApp = CreateObject("Outlook.Application")
    End If
    
    On Error GoTo 0

    Set OutMail = OutApp.CreateItem(0)

    strto = "*** Email address is removed for privacy ***"
    strcc = "*** Email address is removed for privacy ***; *** Email address is removed for privacy ***"
    strbcc = ""
    strsub = "Budget 1st Draft approved by EM"
    strbody = "Hi," & vbNewLine & vbNewLine & _
              "Please note that I have approved the 1st draft of the attached budget."

    With OutMail
        .BodyFormat = 2
        Set olInsp = .GetInspector
        Set wdDoc = olInsp.WordEditor 'Edit the message body directly
        'Set a range to the start of the body thus retaining the signature
        Set oRng = wdDoc.Range(0, 0)
        .Display 'Required
        oRng.Text = strbody
        .to = strto
        .CC = strcc
        .BCC = strbcc
        .Subject = strsub
        .Attachments.Add (ActiveWorkbook.FullName)
    End With

    Set OutMail = Nothing
    Set OutApp = Nothing
    Set olInsp = Nothing
    Set wdDoc = Nothing
    Set oRng = Nothing
End Sub
