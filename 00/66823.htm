aashishkapadia say: This is a classic case of Microsoft emulating Apple. OS X lacks detailed user power management controls because Apple believes the customer experience should be as simple as possible and the system should be as smart as possible to meet conflicting goals of battery life and performance. The disadvantage is that advanced users are never satisfied. The advantage is that average users don't need to deal with the complications that result from a plethora of options. For example, I rarely have seen coherent explanations of how the Windows CPU Power Management Min and Max % options relate to P-states, C-states, S-states, performance, and battery life and how this behavior is different in the Power Saver profile.

Today, the Apple model extends to mobile operating systems where users have even less control over power management. Windows 8 represents a hybrid model for Microsoft where it can transition users from the traditional desktop environment into a more integrated app environment where the computer simply works (i.e. the traditional desire for users switching from Windows to OS X). I expect Microsoft to continue in this direction for each successive release as more features are changed. For the dimmed display time change, here are the pros and cons:

Pros: simplifies power management, reduces likelihood of customer confusion with a given power management setting, encourages users to reduce power consumption by setting the computer to sleep more quickly (i.e. computer in sleep consumes less power than computer awake with a dimmed screen), 15 second delay is sufficient notice to user that the screen is about to power down, aligns usage model with mobile devices

Cons: removes a previously useful feature, encouraging computers to sleep more quickly causes headaches for business users who have connected software that may be interrupted by sleep mode (e.g., VPN, remote desktop, VMs), 15 second delay is insufficient for users who commonly need to see a status on their screen without active attention (e.g., waiting for a communication response via email or chat), encouraging computers to sleep more quickly also discourages laptop manufactures from investing in panel self-refresh displays (i.e. computer in sleep consumes less power than computer awake and in panel self-refresh mode)

The trend I see in the pros and cons above is that the importance of the feature is heavily dependent on the type of user. Consumers are likely to benefit from it much more than business users. What I can't seem to understand is this: if consumers will likely benefit from a change and business users could see a detriment, why not provide the flexibility to satisfy both types of customers. The majority of consumers are unlikely to mess with the detailed power management settings (i.e. will keep their laptop set to a balanced profile without any modifications). The 15 second setting should be the default; however, more advanced users should be able to modify the default behavior as needed. The modification could be made either through the advanced power settings menu or some more advanced approach (i.e. registry).

In the process of evolving Windows, I am concerned that Microsoft will forget how it reached such a large audience in creating it. The world market for computing devices drives a diverse array of needs. Some users use computers in their education, others use it for entertainment, while even more use it for communicating. Imagine a venn diagram. Circles would represent each usage model. Windows traditionally has appealed to customers in each circle and within each intersecting region in the diagram. Apple has NOT. For example, how many gamers use OS X exclusively?

While it may be overblown to consider such a minor change in such a major light, I think the message from Microsoft supports such a conclusion. The removal of the dimmed display time option is just a step in Microsoft's journey toward Apple. If Microsoft ever reaches such a destination, users would be told what is best for them. You don't need to see the estimated remaining time on your battery (estimations are unreliable and change frequently), you don't need an option to select more than three power plans (most people don't use more than that), you don't need a easy way to shutdown your computer (just put it to sleep), you don't need a way to side-load apps (can't screen them for problems).

Having sufficient wisdom to make decisions for your customers is a difficult place to reach. I have yet to see a technology company accomplish this. Apple has come close over the years, but only for brief periods. Every time Apple has creating something admirable, consumers eventually find the company non-responsive to their needs and move on. Instead of emulating Apple, I hope Microsoft learns from both Apple's successes and failures. I welcome a Windows OS that simplifies the experience for the user, but also encourage an OS where the flexibility and power of the past remains. In this case, Microsoft has clearly missed the mark.

To enable us to assist you with your computer's BSOD symptoms, upload the contents of your "\Windows\Minidump" folder.

The procedure:
* Copy the contents of \Windows\Minidump to another (temporary) location somewhere on your machine.
* Zip up the copy.
* Attach the ZIP archive to your post using the "paperclip" (file attachments) button.
 *Please upload them to a file sharing service like Skydrive or"Rapidshare" and put a link to them in your reply.

To ensure minidumps are enabled:
* Go to Start, in the Search Box type: sysdm.cpl, press Enter.
* Under the Advanced tab, click on the Startup and Recovery Settings... button.
* Ensure that Automatically restart is unchecked.
* Under the Write Debugging Information header select Small memory dump (256 kB) in the dropdown box (the 256kb varies).
* Ensure that the Small Dump Directory is listed as %systemroot%\Minidump.
* OK your way out.
* Reboot if changes have been made.

Is it possible to use and older version of printmaster on windows 7 - Microsoft Community

jacklyncannon say:

Is it possible to use and older version of printmaster on windows 7

I had Windows XP but just bought Windows 7 , when I tried to use A program Printmaster which I had used on Windows XP, I got a message saying this program is not compatable with Windows 7. Is there a way I can get Windows 7 to let me use this program?
marilyn o say:

If the program is not compatible, then you may try to install and run the program in compatibility mode.
Use the following steps:
1) Right click on the Program
2) Click on Properties
3) Click on Compatibility tab
4) Select Run this program in compatibility mode and select Windows Vista or whatever operating system the program was running successfully.


Refer to the link here: Make older programs run in this version of Windows
http://windows.microsoft.com/en-us/windows7/Make-older-programs-run-in-this-version-of-Windows


If you run Windows 7 Professional or Ultimate version, install Windows Virtual PC and Windows XP mode (http://www.microsoft.com/windows/virtual-pc/default.aspx) and execute your software in Windows XP.


If you run Windows 7 Home Premium, you cannot run Windows XP mode but you can try VirtualBox (www.virtualbox.org) or VMware Player (www.vmware.com) or any other 3rd party virtualization software and install a separate copy of Windows XP (either Home Edition or Professional) for that purpose. You can find virtualization software free of charge that will run on Windows 7 Home Premium.

 

If that doesn't help, then contact the manufacturer of your version of PrintMaster and see if they have any patches that might help with compatibility.

 

Thanks,

Marilyn
