hansv mvp say:

A "standard module" is the kind of module that you create by selecting Insert | Module in the Visual Basic Editor.

The code

 

Public shtPrev As Worksheet
Public shtCurr As Worksheet

Public Sub GoBack()
    If Not shtPrev Is Nothing Then
        shtPrev.Activate
  End If
End Sub

 

should go into such a standard module. The ThisWorkbook module doesn't have to be created - each workbook has it. It is the equivalent of the ThisDocument module in Word.

You should see ThisWorkbook in the Project Explorer on the left hand side of the Visual Basic Editor. You can open ThisWorkbook by double-clicking on it. The following code should go into it:

 

Private Sub Workbook_Open()
    Set shtCurr = ActiveSheet
End Sub

Private Sub Workbook_SheetActivate(ByVal Sh As Object)
    Set shtPrev = shtCurr
    Set shtCurr = ActiveSheet
End Sub

bchapman255 say: Fax Outlook Address Book (OAB) integration works in two ways:
1) Outlook Address Book integration with the native Fax Client or Windows Fax and Scan
2) Fax Mail Transport..

The first is 32-bit only (yes, I've even tried using the forgotten stepchild that is Windows XP x64 and Outlook 2003, and even that combination has no Fax OAB support). 

The second will work with 64-bit operating systems (but has serious problems with Vista x64 for some reason) with all versions of Outlook EXCEPT for the native 64-bit Outlook 2010.  If you're using Outlook 2010, use the 32-bit version on 64-bit Windows and Fax OAB will still work with Fax Mail Transport. 

It's a shame that Microsoft will never fix this (at least according to MVP Russ Valentine).  Unfortunately, Microsoft has had very half-baked support for faxing in general over the past few years.  It's as if one team wants to wash their hands of it, while another team wants to keep it.  Although the Server and SBS teams want to keep it, the Office/Outlook teams wants it to die die die...

The only workaround is to export the OAB to a CSV file and then import it to WAB.  It's a major pain to have to constantly do this as your OAB gets updated, and its a whole lot of inconsistency when your 32-bit Windows 7 PCs' Windows Fax and Scan can access the Outlook Address Book just fine while the 64-bit Windows 7 PCs have to use this half-baked workaround because some teams with Microsoft support faxing, while other teams will not.

...sad...and pathetic.  Either commit to the feature or drop it altogether.  None of this "well, it was easier to leave the code there than to take it out" mess.
