In Word 2007 the Quick Styles Gallery was added. If you want to see a style there, you need to configure it to show up there. Otherwise, it is still in the styles list with all the other styles unless you marked it as hidden.

Blank lines in my Access report - Microsoft Community

jflo65 say:

Blank lines in my Access report

I created a small, simple Access database for someone.  The intent was to create a report in the format of a directory listing, with each piece of information on it's own line (see below).  The only issue I still have is that occasionally a record will not have a physical address or a phone number.  How can I keep the report from leaving a blank line when those fields have no data?

 

Formatted like this:

 

John Doe

ABC Company

1111 Main St.

Somewhere, State

888-888-8888

email address

djsteele, mvp say:
Am I correct in assuming that your report has 6 separate text boxes on it? Instead, have a single text box. Make that text box a single row high. Set its CanGrow property to True, and set its ControlSource property to something like ([FullName] + Chr$(13) + Chr$(10)) & ([CompanyName] + Chr$(13) & Chr$(13)) & ([Address] + Chr$(13) + Chr$(10)) & ([City] + ", " + [State] + Chr$(13) + Chr$(10)) & ([TelephoneNumber] + Chr$(13) + Chr$(10)) & ([EmailAddress] + Chr$(13) + Chr$(10))

Chr$(13) + Chr$(10) is Carriage Return/Line Feed, so that each row will be a separate line in the text box. The equation takes advantage of the fact that using + for text concatenation propagates Null values. In other words, Null + something results in Null, whereas Null & something results in something. That means if, say, you don't have a telephone number, the part of the equation ([TelephoneNumber] + Chr$(13) + Chr$(10)) will result in Null, so nothing will appear. However, since you're concatenating the various parts of the equation using &, the rest will be okay.

ken sheridan say: To save having to enter a convoluted expression every time Microsoft publishes a CanShrinkLines function for this, given below in a lightly modified form:

Public Function CanShrinkLines(ParamArray arrLines())

    ' Pass this function the lines to be combined
    ' For example: strAddress =
    ' CanShrinkLines(Name, Address1, Address2, City, State, Zip)
    Dim X As Integer, strLine As String
    
    For X = 0 To UBound(arrLines)
        If Not IsNull(arrLines(X)) And Trim(arrLines(X)) <> "" Then
          strLine = strLine & vbCrLf & arrLines(X)
        End If
    Next
      
    ' remove leading carriage return/line feed
    CanShrinkLines = Mid(strLine, 3)
 
End Function
