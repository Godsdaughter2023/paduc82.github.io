cindy meister say:

Hi Neil

The best way to do this would be to format the text in the "master list" with STYLES with a different name than the styles used in the other documents.

By design, Word will try to make everything "look the same" - to prevent a mish-mash of different formatting. So if text coming in is formatted with the same style as present in the target document, the style formatting of the target document will be applied to the text coming in. Text formatted with a unique style name, however, will retain that formatting (and the style will be imported into the target document to support it).

scottgem say:

You need to get your terms right before we can provide a solution. 

A Recordsource is the source of all values that a form can display. This value is either a tablename, queryname or a SQL statement. If it is a SQL statement (SELECT ...) then it has to be a valid statement. You can only set the Controlsource for controls on a form to fields that are listed in the Recordsource. A Controlsource is the field that a specific control is bound to.

Not sure what you mean by listings. There are controls known as List controls. Among these are combobxes and listboxes. These 2 types of controls have Rowsources. A Rowsource specifies what values will be listed in the list control.

So it does appear you are trying to set a Recordsource ("set the record source property to an existing table or query"). However, it also appears that you have controlsources, that are not in the Recordsource selected. That would generate the error message you are getting.

If that is the case, then you probably need to create a query for your report, that joins multiple tables so that all the fields needed for the report are included in the Recordsource. On a form, however, you probably don't want the Recordsource to be multiple tables, but rather to use a mainform/subform to display data from multiple tables.

daniel g. grau say:
Note there were two extra quotation marks around "Application.PathSeparator" that should not have been there. I deleted them in the previous post.

The statements are not that complex:

  • If the active document is named HD:Folder:FileName.doc, then
  • Make string variable DocPath$ equal to the left part of HD:Folder:FileName.doc taking as many characters as it has minus the length of Filename.doc minus 1 (this to exclude the separator)
  • The result is the string variable DocPath$ containing HD:Folder
  • Then you concatenate the separator (whatever it is)
  • Finally, add the file name proper

Now, have you spotted the redundancy? We're taking out the separator and then adding it. We only need this one single-line statement, which is universal:

  • Open Left(ActiveDocument.FullName, Len(ActiveDocument.FullName) - Len(ActiveDocument.Name)) & "Lab4BasicIn.txt" For Input As 4
