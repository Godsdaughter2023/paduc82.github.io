geoffrey chaucer say: One way to do this is:

1- Find out the size of the cell in the source code in preview either on your local browser or from a preliminary upload on a host server.

2- Modify the background image with a graphic editor to match the size of cell.

3- When your website is complete and ready to upload, do another preview of the page carrying the table in Internet Explorer. Click >Page >Edit with Notepad and in the source code locate the tag of the cell in question i.e.:
<TD WIDTH="xxx" HEIGHT="xxx"> text</TD>

Modify the code to read:

<TD WIDTH="xxx" background="name-of-folder-where-the-image-is-stored/name-of-image.jpg (or gif, or whatever) HEIGHT="xxx"> text</TD>

then

4- >Save the modified Notepad document and upload on the server.

That certainly works and is the way I'd usually do it. An alternative approach is to open the Form in the Report's own Open event:

 

Private Sub Report_Open(Cancel as Integer)
DoCmd.OpenForm "frmCriteria", WindowMode:=acDialog

End Sub

Private Sub Report_Close()

DoCmd.Close acForm, "frmCriteria"

End Sub

 

Opening the form in Dialog mode means that code execution will stop until the form is closed, or is made invisible! So you could have a textbox (or more than one control) on frmCriteria to enter the search criteria; the form would also have a command button labeled "Show Report" with code like

 

Private Sub cmdShowReport_Click()

Me.Visible = False
End Sub

 

to resume running the report. When the report is Closed, its Close event will also close frmCriteria.
