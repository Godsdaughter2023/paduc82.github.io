robinmk say:

I figured it out. In short,

$plansToDisableString = 'SHAREPOINTWAC_EDU,SHAREPOINTSTANDARD_EDU'

needs to be

$plansToDisable = 'SHAREPOINTWAC_EDU','SHAREPOINTSTANDARD_EDU'

Whilst -LicenseOptions will accept a comma separated list directly, if it's passed an object, it should be an array object. Makes sense, as passing a comma separated list directly will be treated like an array too.

You cannot share a pst data file, Outlook requires full read/write access to the data file.

Nor can you share, or use, a pst over a network, or use OneDrive for the same purpose.

rohn007 say:

As Palcouk has already said, you cannot open a single PST file in 2 computers at the same time.

This is a legacy of Outlook's origins back in the 1980's and 90's as a "single computer" DOS program. The underlying design assumptions, so deeply underlying that they weren't even thought about at that time, is that files, like PST, existed on a single computer and would only be opened by one computer at the same time.  That underlying assumption in Outlook has never been changed or updated, even though many programs like Word, Excel and Sharepoint now do allow multiple people/computers to open and edit the same file at the same time.

This issue is the reason that when you are installing more than version of Office, ie trials of new versions prior to Office 365 you could not install more than one version of Outlook at the same time.  Office 365 changes the paradyme a little by isolating the 365 PST file inside of a "virtual computer".

joe's answer say: how are you trying to convert the document to PDF, are you using the Microsoft XPS/PDF converter, Try using the Print to PDF by downloading the tool called PDF 995 or primo PDF which should give you better results

brucekrymow say: Was personally having this issue in Word 2007 where the Print Preview displayed the background image correctly, centering and filling the entire background, yet printing to a PDF or printer results in the background image being tiled repeatedly on the entire page ==>  finally figured it out! 

Since the Header formatting option presented by others typically is not an appropriate option for most encountering this issue, and the background printing/spooling options don't work, the resolution to prevent the background image from tiling when printing to a PDF or a printer is as follows:

• Open the document in Word 2007
• Click on the 'Page Layout' tab
• 'Page Background' section > click 'Page Color' > click 'Fill Effects...'
• 'Fill Effects' window opens > click 'Texture' tab > click 'Other Texture...' button 
• 'Select Texture' window opens > navigate to and click on the desired background image > click 'Insert' button
• 'Fill Effects' window opens > click 'OK' button

Since there is a known issue between MS Word and some print drivers resulting in pictures in the background being tiled instead of centered and filled, the trick is to insert the background image as a 'Texture' instead of a 'Picture'. Save your Word document and then edit as needed for any content to be displayed on top of the background image.
