scottgem say:
First, The better way to do this is to use an embedded subform. If you want to display multiple tables on a single form, do it with subforms rather than a multi-table query. 

The Calls subform could be in Continuous form mode and list all the calls. When you use the subform wizard to embed the subform it will prompt you to link the subform on Client_ID.

Note: this is the only way to "Link" forms. If you use separate forms, you can SYNCHRONIZE the forms, in a few different ways. Easiest is to filter the child form for key fields:

DoCmd.OpenForm "formname",,,"[Client_ID] = " & Me.Client_ID

This will open the calls form filtered for the currently displayed client.

 

ken sheridan say: You'll find examples of both the approaches described by Scott in FormsDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file, where subforms are used these are placed on separate pages of a tab control.  The identifying data from the parent form's table appears above the tab control, so is always visible.  The first page of the tab control shows the other
Where separate filtered forms are opened note how a combo box is bound to the ClientID column, but shows the full name.  To do this with a form bound to your Distress_Calls table the combo box would be set up as follows:

ControlSource:    Client_ID

RowSource:     SELECT Client_ID, First_Name & " " & Last_Name FROM Client_List ORDER BY Last_Name, First_Name;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

The demo also illustrates two different ways in which separate forms can be opened; one where the form is opened in dialogue mode, which forces the user to close it before returning to the main form; the other where the form is opened as normal, allowing the user to move focus back to the calling form while the second form (or multiple forms for that matter) remain open.  With this second method the secondary form(s) is(are) re-filtered to show the data relating to the main form's current record if the user moves to a different record in the main form while the secondary form(s) is(are) still open.

karl timmermans (outlook mvp 2012-2018) say:
No, you can't "find" the messages on the server - what you need to do is copy those messages back. Process would be as follows

#1 - Go to File --> Account Settings --> Data Files tab --> take note of the folder path/file name of the PST file involved.

#2 - Close Outlook
#3 - Open the Control Panel <Mail> App to create a new Outlook profile (no need to delete your existing profile)
#4 - Configure the email account as an IMAP account by entering your email address/password
#5 - Go to the Data Files tab --> Add the PST file from item #1
#6 - Set the newly created profile as the default for the next time Outlook is opened

#7 - Open Outlook
#8 - Set Outlook to "Work Offline"
#9 - Open the <Inbox> of the PST file added to the profile --> press CTRL+A to select all the items --> right-click on the selected items --> select <Move> --> select <Copy to Folder> --> Select the <Inbox> in the Imap folder tree
*** reason for selecting <Copy to folder> instead of <Other Folder> is purely as a precaution in case something goes wrong. <Other Folder> is a Move operation where the source items will be deleted automatically from the source folder

#10 - When the copy is done, set Outlook to "Work Online" b toggling "Work Offline" and do a manual Send/Receive on the account to sync it with the server

*** the caveat to this is that all the items just uploaded will be downloaded again when you connect to the account again using POP3 so you will either have to
a) first delete all the items in the existing PST file configured in the original profile OR
b) create a new profile, configure the account using POP3 and let Outlook create a new PST file (making sure to set the "Leave Messages on server" option

The idle question I have is, if you want to keep your items on the server - why aren't you configuring the account using IMAP under normal circumstances.

Quit Excel

  • Re-launch Excel
