bob jones aka: cybertaz say:

The most likely explanation is that the workbooks were not saved from the same original file.

In order to use the feature you must create an original Shared workbook. The additional files must be cloned from it using Save As. IOW, the feature cannot be used on separately created files no matter no 'identical' they may appear to be.

Further, the original must be designated as a Shared workbook before creating the clones. if the Save As is done before the original is Shared the copies cannot be used for merging.

Accordingly, the tool will not be active, nor will the Merge Workbooks command in the Tools menu unless those conditions are met.

My impression, though, is that this is not the feature you want to use in the first place. It's intended for updating an original based on changes made in the clones. Typically, that means that the incoming data will overwrite the content in the original.

It sounds like what you may be intending to do is to combine the sheets from various workbooks into a single workbook file but maintaining the integrity of each sheet. If that's the case it would be more appropriate to use the Move or Copy feature to move or copy sheets from one workbook to another. See:

Move or copy a sheet in Excel 2016 for Mac - Excel for Mac

jay freedman say:
They're right where they always were: In the Paragraph dialog, click the Line and Page Breaks tab.

This is available as direct per-paragraph formatting (by right-clicking a paragraph of text and clicking Paragraph), or as part of a style definition (by right-clicking the style in the Home tab, clicking Modify, then clicking Format in the dialog and clicking Paragraph).

If you're referring to the list of "compatibility options" that were in the Options dialog in previous versions, most of that is gone from Word 2013, but few or none of them affected the behavior of the Widow/Orphan Control option.

You might find a better solution in http://wordfaqs.mvps.org/BottomLine.htm.

Yes absolutely.  However, only one version of Outlook can be installed.  Microsoft has a KB article relating to the installation of more than one Office suitehttp://support.micr osoft.com/kb/928091/en-us
 
Simply Install Office 2003 first, update it fully.
 
Install 2007 using the custom install option and do not let it overwrite or upgrade the previous install.  Once installed update fully.
 
Click Start, Run, type Regedit in the run field and click OK.
 
In regedit follow the instructions on the KB page about 2/3 the way down about running more than one version of Word.  Office/12.0 in the instructions is Office 2007.  Once you create the NoReReg in that location go to the Office/11.0 location and do the same edit.  That will prevent the registration of Word each time a different Word program starts.
