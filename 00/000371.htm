zhang_yi say:

Hi Shashi,

To answer your questions:

1) In general what happens if we update the master page and a new version is rolled out by MS. How hard is it to replicate the changes over to the new master page. Also, what is the typical release cycle for this?

Generally, SharePoint Online won’t update master page frequently. If there is a major upgrade, we will post it on our Office 365 Roadmap. You can monitor it for the latest information. If you encounter an issue during the process of upgradation, feel free to post a thread in our forum. We are always here to help

2) How hard is it to replicate the changes over to the new master page?

To replicate the changes over to the new master page, you can follow the steps below.

1. Go to the Site settings > under Web Designer Galleries, click Master pages and page layouts to access Master Page Gallery.

2. Find and tick the master page you created, it usually named page name.html > FILES ribbon, Download a Copy.

3. Save the page name.html as a different file name.

4. Go back to Master Page Gallery > FILES ribbon, Upload Document > Browser to select the new page name.html > OK.

5. At the next page, select the Content Type as Html Master Page to save it.

6. Find and tick the uploaded .html > FILES ribbon, Publish to publish the file.

7. Go to back to Site settings > under Look and Feel, Master page > chooes the master page for the both options

3) What is the alternative really to build a common navigation across site collections?

If you want a common navigation across site collections, you can use the same managed metadata drop down menus across the top navigation on the site collections.

Here are some articles describing the detailed steps of enabling managed navigation in site collections:

Enable managed navigation for a site in SharePoint

Set up metadata navigation for a list or library

Configure metadata navigation for a list or library
