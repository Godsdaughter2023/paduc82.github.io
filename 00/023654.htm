How to define dynamic text for a hyperlink?

to refer to the title of other files (each file has a couple of bookmarks defined, like "Doc_Title", that provide access to some of the otherwise inaccessible document properties). Thus the only condition is, that the two files are in the same directory to be able to cross-reference them.

 

Now, to make this even more convenient, I would like to underly the above name-reference with a hyperlink, such that one can click on the name in one file to open the other file.

Such a hyperlink is represented (again in the internal field-code representation) as:

{ HYPERLINK file:///{FILENAME \p}\\..\\filename.docx }

 

But how do I represent the combination of both? I.e. where in the above hyperlink specification do I have to place the text, that is being displayed displayed? I tried

{ HYPERLINK @1 file:///{FILENAME \p}\\..\\filename.docx @2 }

i.e. I placed the first expression above into the two possible locations (here marked @1 and @2) but neither did the trick.

What I would needed the possibility to paste such field codes into the fields of the "Insert hyperlink" dialog (but that dialog of course only accepts plain text, no field codes).

 

Any ideas?

 

Hope I could make myself clear...

M.

 

a. user say:

You can't specify a replacement display text in the HYPERLINK field itself, but you can do this:
 a. insert your hyperlink field
 b. ensure the display text is showing, not the field code
 c. click after the display text, and back-arrow one character so that the insertion point is inside the display text
 d. use ctrl-F9 to insert a fresh pair of field braces
 e. insert FILENAME \p inside those braces
 f. carefully delete the original display text characters
 g. select the field and update it. You should see the pathname in the result, but the hyperlink still points to your original location.
 h. save the file in another folder, and updat the field codes to verify that the filename field is still there, even though you cannot see it.

I'm not that keen on this, precisely because (from a maintenance point of view) you cannot actually see the field that is generating the result. But it does seem to work, AFAICS on every version of Word that supports hyperlink fields.

steve rindsberg say:
When you look at your text document, do you see a .TXT or other extension?

If not, it's because Windows' default setting is to hide extensions from you.  In that case, if you start with a file that's shown as SOME_FILE (but is really SOME_FILE.TXT) and you give it an HTML extension, you're really getting:

SOME_FILE.HTML.TXT

Still a Text file as far as Windows is concerned, though it shows you SOME_FILE.HTML

To fix:

Control Panel | Folder Options | View tab and remove the check next to "Hide extensions for known file types"
