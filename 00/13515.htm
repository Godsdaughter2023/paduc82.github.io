julie fka profjulie say:
Your hp Compaq h3850 is a very, very old pocket pc that runs the windows mobile 2002 OS.  The only thing you could do with this device is sync it to a computer, in which case the only software you'd use would be a full version of Outlook and Activesync 4.5.  However, Activesync 4.5 can only be used on an XP computer. 

Since you have a Windows 7 (64 bit) computer, you'd need to install the 64 bit version of Windows Mobile Device Center (which is the version of ActiveSync that is compatible with Vista or Windows 7 computers).  Unfortunately, Windows Mobile Device Center is not compatible with windows mobile 2002 devices and can only be used to sync windows mobile 2003 and later editions of the windows mobile OS. 

The bottom line to all this is that you are stuck with a device that can not be synced to your Windows 7 computer. 

Your only course of action would be to find an old XP computer that has some version of Outlook on it (if you want to sync contacts, appointments & tasks to the device) and install ActiveSync 4.5 to it (the computer).  ActiveSync will also allow you to sync files to the device.

You can go ahead and uninstall the windows server solution phone connector add-in and the Windows Phone SDK 7.1 software since these pieces of software have never been used to sync any windows mobile devices and so is not needed and will never be used.

charleshacker say:

After much trial and error... mostly error, I managed to fix the issue.

As it turns out, Outlook 2016 automagically creates the new account as iMap UNLESS you check the box to manually setup the account.  What I got was only partial old emails and folders.

So I deleted the iMap account and created a new one as POP3.  Then I imported my old .pst file to the new account and whahla!!! All my old emails and folders appeared.  One thing I also had to do during setup was run the repair tool on the imported .pst file.

What took the longest to figure out was just exactly which folder to put the old .pst file.  Of course the folder is hidden and I had to do a search in file explorer for the current iMap file, then I knew where to put the imported .pst file.  Took a VERY long time, but everything is now working properly.

Seems to me that this should not be as difficult as it was for someone just moving an Outlook 2007 POP3 account to a new computer with Outlook 2016.

Steps I performed:

  1. Export old email account (creates a .pst file) to external device.
    •   You will need to know if this old account is POP3 or iMap
  2. Find the new "Outlook" data folder
  3. Copy the exported old email .pst file to the new Outlook data folder
  4. Create the new Outlook account
    • Remember to create the same as the old account either POP3 or iMap
    • NOTE: Outlook 2016 defaults to iMap.
      • If you want POP3 you have to create the account manually by checking the box
  5. Import the old .pst file to the new account
    • May need to run the repair tool during this process (I did)
      • It should come up automatically if needed
  6. Be patient as Outlook will take a bit of time to load everything properly.

jessen p say:

Hi Peter,

Thank you for the update.

What you’re experiencing is called “Form spoofing,” and it’s a favorite tool in the spam artillery.

Form-spoofing is annoying, but it’s not dangerous. It’s the sender entering your name and address into the From field in their mail program instead of their own. Outlook, for one, makes this a snap. During the setup process, when asked to enter one’s own email address, spammers choose this option.

Since the name of the game for spammers is hiding their tracks and making sure you receive their mail, from-spoofing is natural. By putting your address in the From field, they obscure the true sender and nearly guarantee that your spam filter won’t block their email.

To prevent such emails from reaching you, take these steps:

1. Set up a filter. You can create a special rule to automatically route anything sent from your email address to the Trash folder. You’ll still see emails sent by you in your Sent folder, and if you have intentionally added yourself as a recipient, you can always dig that piece of mail out of the trash.

2. Verify. You can also set up a sender verification tool that asks every new sender to verify that they are a real person before the mail reaches you. (From then on, everything from that address is processed as legitimate and sent directly to you.)

3. Take caution. Simply receiving these emails does not put you at risk, but just like any email from an unknown source, you should think twice before clicking on any links.
