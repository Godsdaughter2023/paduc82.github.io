2/29 issue: You're right, the formula version is not correct. The problem is that the DATE funciton in Excel "calculates" the date, regardless if the input is valid or not.

The best example is the answer to the question "What is the previous day before the 1st of a month?"
=DATE(2019/3/0)
As you see the day is 0, so Excel calculates the logical day before.
The same happens for =DATE(2019/2/29) it's just the other direction => 3/1/2019
Therefore the MATCH function may find this date i column A.

The simplest way around this is to remove the formula in the columns of the non-leap years. Please download the sample file again, I've added also a sheet with a Pivot Table and a Power Query solution.

Power Query is a bit complicated for this task, I've written some M-Code to accomplish the result your want...

The Pivot Table is really simple and has a benefit that you can't get so easy with formulas. If the dates in column A has a duplicate, then you'll see a 2 inside the Pivot Table, because the Pivot Table "sums the days up".

Note: If you refresh the Pivot Table, the formulas in column A are also calculated. Means the Pivot Table did not show the result of the random generated dates in my sample file.

Andreas.
