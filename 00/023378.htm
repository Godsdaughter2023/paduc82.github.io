joepehoushek say:

What does the small square format symbol mean?

In WORD 2013 I have occasionally, small black squares which appear at the beginning of some paragraph headers and occasionally at the beginning of some regular text lines.  They are visible only when displaying all format symbols (Shift-Ctl-8).   What do they mean?  How do I get rid of them?
jay freedman say:

Those squares indicate that at least one of the options in the Line and Page Breaks tab of the Paragraph dialog is checked, except for Widow/Orphan Control (that is, Keep With Next, Keep Lines Together, and so on).

Because the heading styles almost always include the Keep With Next option, you'll see the square next to the heading paragraphs. However, they'll also appear with any paragraph in which the style or directly applied formatting includes one or more of those options.

jay freedman say:

The "small square" you showed is a table cell marker. You'll find it described at http://wordfaqs.ssbarnhill.com/NonprintChars.htm under the heading "Cell markers".

The presence of that symbol implies that the lines around the text are table cell borders. When the cursor is inside the table cell, the Table Tools tab and its two children will appear in the ribbon. On the Table Tools | Design tab, the Borders section contains tools for changing or deleting the borders of the selected cell(s).

Your second screen shot indicates that you don't want a table cell at all. If that cell stands alone rather than being part of a larger table, you can remove the table formatting and just leave the contents as plain text. To do that, click in the cell, go to the Table Tools | Layout tab, and click the Convert to Text button:

So the first step is to give it execute permission:
chmod u+x kindlegen

You can drag and drop the kindlegen program into the window to make sure that you have the path and spelling correct.

Once you do that, you need to specify the full path to the program when you're running it from the command line. If in the terminal, you are in the same directory as the program, the command would be this:

./kindlegen name_of_word_file

You would have to have the dot-slash at the head for it to work correctly from the same directory. If you're not in the same directory, you need the full pathname. Again, you can drag and drop the program to get the full path.

Hopefully that helps.

Ctrl + Click is not an event that at button recognizes so you are out of luck on that one. That being said the stanard practice for this type of thing is to use a message box or something similar to control the execution.

Private Sub CommandButton1_Click()
    If MsgBox("This can not be undone. Are you sure...", vbYesNo, "Careful") = vbNo Then
        MsgBox "Operation Canceled"
    Else
        MsgBox "Do stuff"
    End If
End Sub

jay freedman say:

If you don't want to show users a message every time as described by the other answers, your alternative is to let the macro run, but have it check for the Ctrl key and immediately quit if the key isn't down.

Start by adding Chip Pearson's module from http://www.cpearson.com/excel/keytest.aspx (note that it works the same in every Office app that supports VBA, not just Excel). Then use the IsControlKeyDown function like this:

Sub MacroButtonProc()
    ' this is the procedure called by the
    ' macrobutton in the document

    ' if neither control key is down,
    ' don't do anything
    If Not IsControlKeyDown() Then
        Exit Sub
    End If

    ' if you get here, a control key was down
    MsgBox "This is the rest of your macro"
End Sub
