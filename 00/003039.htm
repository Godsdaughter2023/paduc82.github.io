Cant retrieve BinarySecurityToken from SAML Response of Office365 operated by 21Vianet in China

In our REST client, we send the SAML request to the IdP and read BinarySecurityToken from SAML response, retrieve the cookie (FedAuth & rtFa) and complete the authentication process.
The SAML response is different for Microsoft operated Office365 and 21Vianet operated Office365.

---
With Microsoft operated Office365 our REST client is able to completed authentication process as it could read the BinarySecurityToken.
Microsoft's IdP returns BinarySecurityToken and the token is used to retrieve the cookie.

Login page: login.microsoftonline.com
IdP URL: login.microsoftonline.com/extSTS.srf

---
With 21Vianet operated Office365 our REST client is unable to completed authentication process as it could not read the BinarySecurityToken.
IdP for Chinese didn't return a BinarySecurityToken. It returns encrypted data <CipherData> and it is not possible to check the contents.

Login page: portal.partner.microsoftonline.cn
IdP URL: login.partner.microsoftonline.cn/extSTS.srf
