scottgem say:
You might find my blog on What is Access (https://scottgem.wordpress.com/2015/12/30/what-...) to be helpful.

Modules are collections of VBA code containing functions and procedures that can automate processes in your application. There are different types of modules, Global modules, object modules and class modules. Global modules are independent, they can contain one or more functions/procedures. These functions/procedures can be used throughout the application. Object modules are tied to objects like forms or reports. The functions/procedures are tied to events connected to controls on the form/report or the form/report itself. Class modules are more complicated, and a steeper learning curve, but allow you to create even greater customization.

Code is entered and managed using the VBE (Visual Basic Environment).

Windows Update takes care of updating Windows Defender along with other Microsoft services and applications automatically. Updating Windows Defender does not require system restart.

Windows Defender includes Real-Time Protection (RTP). Real-Time Protection (RTP) continuously monitors your computer and scans everything you download or run on your PC. You don't feel it when it is workings in the back round. It pops up only to inform you when it finds malware or when some action is required from you.

However, you can use Windows Defender anytime to scan for malware if your computer is not working properly or if you clicked a suspicious link online or in an email message.

When you access any file on your computer or download it form Internet, Windows Defender RTP checks that file before it opens or executes. Therefore, you don't need to scan the container folder or driver before you open it. However, you can use custom scan to scan specific folders and/or drivers.

1- Open Windows Defender.

2- In the main page (Home) select "Custom" option from "Scan options" pane (appears in the right side)

3- Click on "Scan now" button.

4- From a popup window appears, select the drivers and folders you want to scan.

greg edwards mvp alumnus (2010-2014) say:

I'll agree with Jason's earlier comment. Previous implementations of Media Player and Windows Media Center filled up my music library with wonky image files that stored various sizes of album art thumbnails. Most of the time the files were hidden, but once in a while (especially when moving sets of files between PCs), they'd become visible, and whoa what a mess! I was also surprised at how much disk space they all consumed.

Groove uses the album art stored within the IDv3 tag instead. Now if you have a large music collection separated into folders with folder.jpg files, then I'd recommend using a tool like the excellent (and free) Mp3tag to batch write the album art into each file's tag. Here's how:

  1. Download, install, and open Mp3tag.
  2. Click File > Change Directory, select the top-level folder to search for music, and click Select Folder. The mp3 files will be imported into Mp3tag.
  3. Click Actions > Actions. The Action Group window is displayed.
  4. Click New, type a name for the action (such as "Fix Album Art", without the quotes), and click OK. The Actions window is displayed.
  5. Click New, select Import Cover from File, and click OK.
  6. Type a format string for the image filename (such as "folder.jpg", without the quotes), select a metadata tag for the imported image (I typically choose Front Cover), and click OK.
    Note: If you already have album art embedded into some mp3 files, you can check the Delete Album Art option to prevent Mp3tag from storing multiple cover art images in the metadata.
  7. Click OK to close the Actions window and click OK to close the Action Group window.
  8. Mp3tag runs the action and displays the result.

If you no longer need the assorted folder images, you can also use Mp3tag to rename/restructure you library into a fresh set of folders and then delete the old ones. Voila!

Hope this helps. Let me know if you have any questions.
