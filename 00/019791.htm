john w. vinson say:

Well... this isn't very easy. A scanned image, whether in a graphics format such as .jpg or in a .pdf file, is just that; it's not letters and numbers, it's an image - a picture of data, not data itself. You will need to use Optical Character Recognition to convert the pixels of the image to text, and then (nontrivial) VBA code to parse that text into fields in your table. This is doubly complex if (as seems likely) the users handwrite the answers to their assignments; OCR has gotten really good lately but if you're trying to read particularly bad handwriting (mine for example) there will be lots and lots of errors.

If all you want to do is store and categorize images of the scanned assignments, that's a bit easier but correspondingly less useful. You can't automate grading of a picture.

ken sheridan say: You can provide a link to a file of any type quite easily, simply by storing the path to the file in a text column in a table.  The file can then be opened in its default application, or printed to the default printer, from within Access by calling the Windows API ShellExecute function.

I'm afraid I have no idea of how you could add the path automatically when you scan the document, but  adding a file 's path to the database manually once it's been scanned would be quite simple.  Let's assume you include a table StudentFiles in the database, with columns StudentID, FilePath, and FileTitle of long integer number, text, and text data types respectively.  In a form based on a Students table, when at the relevant student's record you can insert a row into the table by browsing to the file, then entering a meaningful plain English title for it by means of an input box.  the code, e.g. in a button's Click event procedure would be like this:

    On Error GoTo Err_Handler
    
    Dim dbs As DAO.Database, qdf As DAO.QueryDef
    Dim strPath As String, strTitle As String, strSQL As String
    
    ' open 'file open' dialogue and get path to selected file
    With Application.FileDialog(1)
        .Title = "Select File"
        ' following line sets default folder at which dialogue opens
        .InitialFileName = "F:\Students\Assigments\"
        .Filters.Add "PDF Files", "*.pdf", 1
        If .Show Then
            strPath = .SelectedItems(1)
            strTitle = InputBox("Enter a title for the file:","New file")
            strSQL = "INSERT INTO StudentFiles (StudentID, FilePath,FileTitle) VALUES (" & _
                Me.Student ID & ",""" & strPath & """,""" & strTitle & """)"
            CurrentDb.Execute strSQL, dbFailOnError
       Else
            MsgBox "No file selected", vbInformation
            Exit Sub
        End If
    End With
        
Exit_here:
    Exit Sub
    
Err_Handler:
    MsgBox Err.Description, vbExclamation, "Error"
    Resume Exit_here

Rather than setting a single default folder at which the file dialogue opens, depending on how the folders to which you scan the assignments, you might be able to build a string expression to a path specific to the current student.

To open or print a file by means of the Windows API ShellExecute function you'd add the following module to the database:

' basShellExecute Module
Option Compare Database
Option Explicit

Public Const SW_HIDE = 0
Public Const SW_SHOWNORMAL = 1
Public Const SW_SHOWMINIMIZED = 2
Public Const SW_SHOWMAXIMIZED = 3
Public Const OP_OPEN = "Open"
Public Const OP_PRINT = "Print"


Declare Function ShellExecute& Lib "shell32.dll" Alias "ShellExecuteA" (ByVal _
hwnd As Long, ByVal lpOperation As String, ByVal lpFile As String, ByVal _
lpParameters As String, ByVal lpDirectory As String, ByVal nshowcm As Long)

Sub ShellToFile(strPath As String, _
            Optional strOperation As String = OP_OPEN, _
            Optional lngShow As Long = SW_SHOWNORMAL)

    Dim lngRetVal As Long
    Dim lngHwnd As Long
    
    lngHwnd = Application.hWndAccessApp
    
    lngRetVal = ShellExecute(lngHwnd, strOperation, strPath, _
        vbNullString, CurDir, lngShow)
        
    If lngRetVal <= 32 Then
        MsgBox "Unable to open file " & strPath, vbInformation, "Warning"
    End If
    
End Sub

You can then select a file, e.g. via a list box or combo box whose list is restricted to a currently selected student, and whose value is the path in a hidden column but which shows the plain English title in a visible column, and call the ShellToFile function, passing the path into it, plus "Print" as the optional strOperation argument f you wish to print it rather than open it.
