A cleanup operation from a previous installation attempt is still pending and a system reboot is required in order to continue the upgrade.

gerry c j cornell say:

System Protection only needs to be monitoring OS C System. If you think about it monitoring a Recovery partition is pointless and just wastes disk space. Using a recovery partition does not rely on current settings held in the Registry. Everything it needs is in the recovery partition.

Monitoring external drives can cause problems because they get connected and then disconnected. If system restore is monitoring a drive and it is removed the restore point won't work because it references something that is no longer present so you cannot use it to restore settings relating to your C drive.

You mentioned using cCleaner. cCleaner is an excellent tool but cCleaner also comes with a Registry tool. Do not use this tool as like any registry cleaner it can lead to the creation of very difficult to resolve system errors.

The system allocates a certain amount of disk space to system restore (and to backup ). If this space is fully utilised the older restore points will be removed to accomodate new restore points. From reading your messages I do not get the feeling this has happened on your computer.

How large is your C partition amd how much free disk space does it have?

gerry c j cornell say:

I have not said you should not use cCleaner. What I said was do not use the Registry Cleaner component which is accessed through the Registry button in the main window.

 A disk allocation should be more than enough.36 gb should be more than enough.
