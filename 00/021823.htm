hansv mvp say:

You cannot make the empty new record appear as the first record, but you can make the form go to the empty new record automatically when it is opened:

 

  • Open the form in design view.
  • Activate the Event tab of the Property sheet (press F4 if you don't see the Property Sheet).
  • Click in the On Load event.
  • Select [Event Procedure] from the dropdown list.
  • Click the Builder button to the right of the dropdown arrow (the button with three dots ...)
  • This will take you to the Visual Basic Editor.
  • Make the code look like this:

Private Sub Form_Load()
    On Error Resume Next
    RunCommand acCmdRecordsGoToNew
End Sub

  • Switch back to Access.
  • Save the form's design.

Updating unbound comboboxes based on another combo box - Microsoft Community

accesshelpwanted say:

Updating unbound comboboxes based on another combo box

Attached are pictures of my relationships and a rough view of what I would like for my form. This form is for the purpose of saving information to the Product Tag table, however it is also necessary that the PC Product Code and Manufacturer name are also displayed on this form. Currently, PC Product Code and Manufacturer name are unbound comboboxes, while the Model Number is a bound combo box with a control source of Model ID. Right now, I can choose values for the unbound boxes to be anything, and once I chose the model number, even if it has different data (as in this case where the combo box says PC Product code of 2 but the model number of 111 has been saved with a PC Product Code of 1), it doesn't update to reflect already saved record's data. How can I program this to update the two comboboxes if the values that were entered are different from those saved in the  underlying model record?

ken sheridan say: You need to look up the values from the referenced tables and assign them to the unbound controls in the form's Current event procedure.  The following is an example from my DatabaseBasics demo:

Private Sub Form_Current()

    ' if new record set unbound combo boxes to Null,
    ' for existing record call functions to return
    ' region and country for current value of CityID
    If Me.NewRecord Then
        Me!cboCountry = Null
        Me!cboRegion = Null
    Else
        Me!cboRegion = GetRegion(Me!cboCity)
        Me!cboCountry = GetCountry(Me!cboRegion)
    End If
    
    ' requery unbound combo boxes to list
    ' only regions and cities for current city, if any
    Me!cboRegion.Requery
    Me!cboCity.Requery
    
End Sub

This looks up the region and country for the selected city in a bound combo box by calling the following functions in the form's class module:

Private Function GetRegion(varCityID)

    ' get RegionID for current value of CityID field
    If Not IsNull(varCityID) Then
        GetRegion = DLookup("RegionID", "Cities", "CityID = " & varCityID)
    End If
    
End Function

Private Function GetCountry(varRegionID)

    ' get CountryID for current value of Region combo box
    If Not IsNull(varRegionID) Then
        GetCountry = DLookup("CountryID", "Regions", "RegionID = " & varRegionID)
    End If

End Function

Note that the GetRegion function is called first to assign a value to the cboRegion control, whose value is then passed into the GetCountry function as its argument, to assign a value to the cboCountry control.

In the AfterUpdate event procedure of each of the unbound combo boxes, the cboCity bound combo box is set to Null to remove any CityID value which does not map to the changed CountryID or RegionID value of the updated unbound combo box.  Otherwise, while the cboCity control would appear empty, its value could still be an invalid CityID for the country or region selected.

The cboRegion combo box is correlated with the cboCountry control by referencing the latter as a parameter in its RowSource property's SQL statement.  The cboCity combo box is similarly correlated with the cboRegion control.

In your case model is analogous to city, manufacturer to region and product code to country.
