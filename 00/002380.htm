Hi Elena,

This duplication is caused when your account is sending through Gmail's SMTP server and Outlook is configured to save a copy of each sent item.

You may choose to disable this setting by following these steps if you are using Outlook 2013 or 2016:

  1. Click on File.
  2. Click on the Account Settings drop down and choose Account Settings.
  3. Select your Gmail account then click on Change.
  4. Click on More Settings.
  5. On the Advanced tab, make sure that the check box beside the Do not save copies of sent items is not ticked.
  6. Click on Apply then OK.

After setting this up, you shouldn't have duplicate sent emails in your Sent Items folder.

felix tao msft say:

Hi Skip,

 

I need to explain that move to and copy to only work in the New UI of the SharePoint Online library. The screenshot you provided is about the Classic UI of the library. To switch the library from Classic UI to New UI, you may try the steps below and check the outcome.

1.Enable First Release:

Let the global admin log into the Office 365 Portal>Go to Office 365 admin center>Expand SERVICE SETTINGS and select Updates>Select Entire organization.(Note: The First Release may take some time to take effect)

 

2.Change the default experience for all document libraries:

In the Office 365 admin center, expand ADMIN and select SharePoint to get into SharePoint admin center>Choose Settings.>Next to SharePoint Lists and Libraries experience, select New experience (auto-detect).

 

3.Go to the SharePoint Online libraries in the browser, select the files or folders you want to move.

 If you select one file, you need to click ellipses(…) to find Move to and Copy to.

 

 If you select multiple files, Move to and Copy to will display as below.

 

 For the details, please refer to Switch the default for document libraries from new or classic

 

Thanks,
Felix
