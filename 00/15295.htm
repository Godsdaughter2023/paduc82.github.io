The Exchange Online license is required in the sync process. Thus, you need to firstly make sure that the user photo is synced to OWA. I would also like to share with you the replication process for the synced photos:

1. Users’attributes from the on premise Active Directory syncs with Office 365 Azure Active Directory(AAD).

2. Office 365 syncs the data from AAD to the Exchange Online Directory Services after the user was assigned an Exchange license.

3. Skype for Business pulls the photo from Exchange Online (the user must have a Skype license assigned).

bob jones aka: cybertaz say:

I'm not certain what it is that you're asking for help with. Unless you explicitly select a passage to be checked or already have marked certain content to not be checked, the spelling checker checks the entire document from start to finish every time it is run. Where the automatic spell checking markup may have run into a wall is completely irrelevant.

To address your other points, the document doesn't outgrow the Spelling & Grammar Check, itself unless it exceeds the 32MB text size limit or the 512MB overall file size parameter. The number of "pages" is irrelevant... page is not a measurement unit or physical element in Word documents.

The main influence on performance is the available RAM & disk space as well as other performance hogs [such as having Track Changes in use, excessive use of objects containing text, heavy reliance on direct formatting rather than applying Styles, etc.].

Automatic checking as you type, however, does have a limitation & will not function beyond that point. I've never seen documentation on what that limit is, but I'm sure that it has been increased now that the program has been enhanced to 64-bit.

Text Wrapping Issue - Microsoft Community

toddalbertson say:

Text Wrapping Issue

I have written an 80,000 word document.  The last draft was assembled cleanly, ensuring all formatting was applied with styles.  The time has come to add the illustrations, which are simple line drawings with captions placed upon drawing canvases.  For the first 30 pages, the illustrations have worked fine.  The reason for this is because so far, the end of a paragraph has never occurred at the bottom of a page.  When a paragraph does occur at the bottom of a page and there is an illustration following it, Word does the following:  1) If the illustration is "In Line With Text," Word will indent the illustration and the beginning of the following paragraph will be unindented.  If you change the style so the paragraph is unindented, it unindents the illustration and the paragraph will also be unindented.  There is no way to have the paragraph following the illustration indented unless you switch to another text wrapping method.  2) If you switch to another text wrapping method, Word will force a line from the following paragraph onto the illustration's page.  Even if you extend the illustration's canvas all the way to the bottom margin, Word still forces the line with the anchor point onto the page, even if it must place the line into the footer.

The effect of this is that there's no way to put a full-page illustration into a Word file when the illustration directly follows the end of a paragraph.  I consider this to be a major error.

Now, I have been able to reproduce this problem in a blank file.  So, I will illustrate this problem for you.

Above, we see an illustration that is set "In Line With Text."  As expected, the text is broken by the end of the page and it continues again below the illustration.

Above, we have a full-page illustration, also set to "In Line With Text."  The important thing to note is that the text breaks in the middle of the paragraph and, as before, the text ends mid-paragraph and then continues beyond the illustration, which is two pages beyond where the text breaks.  Everything is still behaving properly.

Above, note how the end of a paragraph occurs at the bottom of a page immediately before an illustration.  I have left the outline for the drawing canvas on.  Note how the canvas is indented and the paragraph below the illustration is unindented.  This is to be expected because, and I quote: "Pictures inserted in line are tied to the place within the text where they were inserted and are treated like any text character—'in line with text'" (source).  In other words, the illustration, which is as wide as a full line, is treated like a line of text and is thus indented, and the following lines, which are the new paragraph, are not indented since they are not the first line.

Okay, okay, I get this.  So, what do I do about it?  If I take away the first-line indent, it fixes the illustration, but the following paragraph remains unindented.  It seems I need to change the illustration's layout option.

Here's where any other option for text wrapping fails.  The "With Text Wrapping" options all require the illustration to be anchored to text.  Thus, a line of text is forced upon the page, even if it must go into the bottom margin to fit:

Notice how the illustration is no longer indented, but since an anchor is required, the first line of the next paragraph now appears in the bottom margin.  No, toggling widows & orphans doesn't fix this.  No, trying each of the other layout options doesn't fix this.  No, pulling the text wrapping points as far out as they can go doesn't fix this.  No, inserting text-wrapping breaks into the document doesn't fix this.  Nothing, as far as I can tell, fixes this.  There is simply no way to have a full-page illustration in a document following the end of a paragraph.

So, what can Microsoft do, design-wise, to fix this?  It seems to me that a user should be able to anchor an illustration to the page.  You can anchor to text, you can anchor to the header or footer.  Why can't you anchor to the page?

suzanne s. barnhill say:

See http://wordfaqs.ssbarnhill.com/FullPageGraphics.htm

BTW, if I were you, I would not create the graphics in a drawing canvas with the caption. Instead, I would insert the graphic directly (in an unindented paragraph), followed by the caption in a text paragraph. This would put both of them in the text flow, with no wrapping required. In addition, I'm wondering if a caption in a drawing canvas can be accessed for the purpose of creating a Table of Figures.

Incidentally, the reason your illustration is indented is that you're inserting it as the single "character" in a paragraph that has a first-line indent. Use an unindented paragraph to get around this.

Hi, Suzanne.  I appreciate your article because it explains my problem exactly.  My frustration is that full-page illustrations are commonly found in books, but Word has no direct way to incorporate them into a document.  See, while this is my final draft, it will go off to a publisher and require several edits after this.  So, it is not the final draft.  And what I'll have to do through each edit is slog through moving and adjusting illustrations instead of doing my actual work, which is polishing the language.  The process of "fiddling" with Word so it behaves as desired doesn't appeal to me because I'll have to "re-fiddle" with it every time a line is added or the pagination changes.

I also appreciate your suggestion regarding the captions.  I'll give that a shot to see if that improves my process any, but I don't think it'll make a difference with the text wrapping issue.

Lastly, I realize that the first-line indent is why the illustration is being thrust into the right margin.  My issue is this:  If I use an unindented paragraph, the new paragraph on the following page will also be unindented.  In the "With Text Wrapping" options, there is no way to unindent the illustration while at the same time indenting the following paragraph.

What I need to do is fix each illustration to their positions on their page so that, as I edit, everything stays put.  This can be accomplished by using the "With Text Wrapping" option "Top and Bottom" and then selecting "Fix Position on Page."  But, as you noted, Word forces at least one line of text onto the page when a "With Text Wrapping" option is selected.  This is fine with a small illustration, but unacceptable with a full-page one.  In my opinion, Word's software folks need to consider this issue and consider their customers and deliver a solution to this problem--one that doesn't require "fiddling."

keith a. miller say:

It looks like Word thinks your full-page illustration is the first character of the following paragraph.

So what happens if you add a paragraph mark between the illustration & the following paragraph ---  essentially making your illustration a one-character paragraph? I think this will allow you to turn off indent for the picture while retaining it for the paragraph, as well as doing away with forcing the single line.

Good luck,

Keith
