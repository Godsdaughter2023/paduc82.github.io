ken sheridan say: I'm sorry to say that the Wizards are not immune to bad design practices.  While useful, they should be used only as a means of saving time by cutting out some of the drudgery, and you need to have an understanding of what they are doing and the possible pitfalls.

As Doug says, your problem stems from the fact that you've used a combo box in the report.  While this appropriate in a form it is not the best approach in a report.  The solution is to base the report on a query which includes the referencing table, i.e. the table from which the combo box's RowSource draws its values, joining it to the relevant referencing table in the report's current RecordSource.  You can then return the relevant non-key columns from the referenced table in the query and bind text box controls to these in the report.  It is then a simple task to sum the value of the relevant column in the footer.

brian tillman [outlook mvp 2007-2019] say:

Outlook.com cannot read the data store Outlook Express used, nor can it read individual message files generated by OE.  On Windows XP, the last version of Windows to support Outlook Express, Outlook Express kept its data in %UserProfile%\Local Settings\Application Data\Identities\{GUID}\Microsoft\Outlook Express.  In that folder were a number of files whose extension was .DBX.  These were the folder files and all of the messages were in these files, not stored as individual messages.  The DBX files are also not human-readable.  They contained binary metadata that allowed OE to keep track of various information about the folder and the individual messages.

 

If the Windows XP system that ran Outlook Express is still running, you can install Windows Live Mail on it, transfer the OE messages and contacts to WLM, set up your Outlook.com account in WLM, then copy the messages and contacts to the Outlook.com folders and contacts.

 

If you saved the entire data store from the Windows XP system, the process will be similar.  You can install Windows Live Mail, import the data from Outlook Express into it and then copy it to the Outlook.com folders you'll see in WLM after connect it to your Outlook.com mailbox.  You'll also need the Windows Address Book (.WAB) file from the old Windows XP system if you want to move your contacts.

tammy.m say:

Hi Truth0000Seeker,

Outlook uses Personal Folders (.pst) to store email on the local machine as long as you are using POP email this will be the case.

If you are connected to an Exchange Server then you will need to talk to your administator about restoring your mailbox.

From the sounds of things you may have accidentally closed the .pst file that you were using.

I would suggest that you use the Search function included with Windows and look for a file ending in a .pst extension.

Once you find it or them then you can use Outlook to open the files and see if your folders are there.

To open a .pst file in Outlook here are the steps:

  1. Open Outlook.
  2. Go to File - Data File Management.
  3. In the Data files tab, click on Add to add a new file group to the list.
  4. In the New Outlook Data File window, select Office Outlook Personal Folders File (.pst) as the type of storage, and then click on OK.
  5. Select the the .pst file you wish to add and then click on OK.
  6. Rename the generic file name to something more descriptive, enter additional options, and then click on OK.
  7. The added file and its associated folders are displayed in Outlook’s Folder List view.

When you rename a Microsoft account, the old account name will be the alias of the new one.

 

Example:

*** Email address is removed for privacy *** was renamed to *** Email address is removed for privacy ***

*** Email address is removed for privacy *** will become the alias of *** Email address is removed for privacy ***

 

In this case, you won't be able to sign in using your old account (*** Email address is removed for privacy ***). You have to use the new account name to sign in.

 

An alias is a secondary email address added to an already existing Microsoft account address. To put it another way, you have a mailbox with an existing email address created when you signed up for your Microsoft account and sometime later you added another email address going to the same mailbox. If you can’t sign in with it, but you can send and receive email with it, it’s an alias.

 

For more information about alias and renaming accounts and how to rename back to your old email address, please refer to these articles:

  • Switching back to old email account
