Hi,

The "Help" file for Items.Find is surprisingly helpful in this case: http://msdn.microsoft.com/en-us/library/office/ff869662(v=office.15).aspx

When filtering text fields, you can use either a pair of single quotes (') or  a pair of double quotes (") to delimit the values that are part of the filter. For example, all of the following lines function correctly when the field is of type String: 

sFilter = "[CompanyName] = 'Microsoft'"  
sFilter = "[CompanyName] = ""Microsoft"""  
sFilter = "[CompanyName] = " & Chr(34) & "Microsoft" & Chr(34)

In VBA, strings are enclosed in quotation marks (").  e.g.:
   strMyString = "Some text here"

If you need the quotation mark character to appear in the string, you can double it up, e.g.:
   strMyString = "Bob said ""Boo!"" and I was surprised."
will return:           Bob said "Boo!" and I was surprised.

So the use of Chr(34) in the example you gave is another way of including a quotation mark in the string.

So to search for "Test" in the Subject field, you need to build a string that looks like this:

    [Subject] = "Test"
and that string must include those quotation marks.  So you can do:

    Dim strFilter As String, strSubj As String
    strSubj = "Test"
    strFilter = "[Subject] = """ & strSubj & """"
    Set olMail = Fldr.Items.Find(strFilter)


Make sense?
Cheers
Rich

Usually this occurs if the document has all of the following conditions:
  1. Nested tables are in the header or footer and the nested tables are formatted to wrap text.
  2. The document contains a {NUMPAGES} field in the header or the footer.
  3. Change tracking is turned on and the document contains more than 10 pages.
  4. Print Layout view is turned on.
To avoid this problem, use any of the following methods:
  1. View document in Normal View
  2. Turn off text wrapping for the nested table.
  3. Remove the nested tables in the header or footer.
  4. Remove the {NUMPAGES} field in the header or the footer.
  5. Turn change tracking off.
