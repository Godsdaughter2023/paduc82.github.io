and that returns all the records for that day.  Now I want to narrow it down by two specific times each day.

How can I query for dates and specific times?  Something like [Pick Date] 6:00 AM and 6:00 PM.  To make it more complicated, the times are never exactly 6:00 AM and 6:00 PM  the times vary by a few minutes, but there will never be more than one record in any given 15 min. period so there will always be one time around 6 AM and one around 6 PM give or take a few minutes.

Thanks in advance!

Edited: fixed original query code

ken sheridan say: For maximum flexibility the following query:

PARAMETERS [Enter start date/time:] DATETIME,
[Enter end date/time:] DATETIME;
SELECT *
FROM [YourTable} AS T1
WHERE T1.[DateTIME] =
    (SELECT MIN([DateTIME])
      FROM [YourTable} AS T2
      WHERE T2.[DateTIME] >=[Enter start date/time:])
OR T1.[DateTIME] =
    (SELECT MIN([DateTIME])
      FROM [YourTable] AS T3
      WHERE T3.[DateTIME] >=[Enter end date/time:]);

should return the row with the earliest date/time value on or after the value entered as the start date, and the row with the earliest date/time value on or after the value entered as the end date/time.

If the start and end times are fixed as 6:00 AM and 6:00PM, and only the date is the variable parameter, then the query would be:

PARAMETERS [Enter date:] DATETIME;
SELECT *
FROM [YourTable} AS T1
WHERE T1.[DateTIME] =
    (SELECT MIN([DateTIME])
      FROM [YourTable} AS T2
      WHERE T2.[DateTIME] >=[Enter date:]+#06:00:00#)
OR T1.[DateTIME] =
    (SELECT MIN([DateTIME])
      FROM [YourTable] AS T3
      WHERE T3.[DateTIME] >=[Enter date:]+#018:00:00#);

To automate the query completely to return the two rows with nearest date/time values on or later than 6:00 AM and 6:00PM on the current date:

SELECT *
FROM [YourTable} AS T1
WHERE T1.[DateTIME] =
    (SELECT MIN([DateTIME])
      FROM [YourTable} AS T2
      WHERE T2.[DateTIME] >=DATE()+#06:00:00#)
OR T1.[DateTIME] =
    (SELECT MIN([DateTIME])
      FROM [YourTable] AS T3
      WHERE T3.[DateTIME] >=DATE()+#018:00:00#);

steve rindsberg say:

You say that this part is not working.  Can you define "not working"?  What happens that shouldn't?  What doesn't happen that should?

'With myTable.Table

My guess: it deletes the first row correctly but after that seems to delete rows at random rather than the ones you want it to delete.

Use

For iRow = .Rows.Count to 1 Step -1 

instead of

For iRow = 1 to .Rows.Count

Why?  Because when you delete, for example, Row 1, Row 2 becomes the new Row 1 but your code moves on to look at Row 2, which is now Row 3 and it all goes pearshaped.

Sub Test()
  Dim mItem As MailItem
  Dim ToValue
  Set mItem = ActiveInspector.CurrentItem
 
  ToValue = Trim(mItem.To)
  'Replace all possible delimiters with a comma
  ToValue = Replace(ToValue, ";", ",")
  ToValue = Replace(ToValue, " ", ",")
  'Remove double delimiters
  Do While InStr(ToValue, ",,") > 0
    ToValue = Replace(ToValue, ",,", ",")
  Loop
  'Split into words
  ToValue = Split(ToValue, ",")
  'Use the first name only
  'Note:
  '  For the 2nd name use ToValue(1) instead (enable this line):
  'If UBound(ToValue) > 0 Then ToValue(0) = ToValue(1)
  mItem.HTMLBody = Replace(mItem.HTMLBody, "@to", ToValue(0), Compare:=vbTextCompare)
End Sub
