After so many hours in Internet I find this information...
In Windows 7, there may be occasional error where user cannot drag and drop an application program to Windows Taskbar (Superbar) and Start Menu anymore, with the icon shows a red cross when user attempts to place it on Taskbar or Start Menu. When right click on a program icon or shortcut, the “Pin to Taskbar” and “Pin to Start Menu” context menu is missing and not showing too.
The cause of the problem is because users have made the shortcut in Windows 7 to be ‘not a shortcut’ by removing the shortcut arrow on icons, including desktop icons, by removing IsShortcut registry key.
Thus, to fix the issue that application programs unable to pin to Taskbar and Start Menu, just re-add the IsShortcut registry key back to .lnk (and other extensions if applicable) shortcut to reset and restore back original registry values.
The error happens because of missing registry value data for lnkfile registry key, which defines the behavior of all shortcut or .lnk files on the system. The registry value data in question, IsShortcut, can be accidentally deleted when user tries to hide and remove the arrow overlay icon on shortcuts.
=============================================================================================
To resolve the missing or lost “Open file location” entry, the IsShortcut registry value and the file associations for shortcuts has to be fixed.
  1. Run Registry Editor (regedit.exe).
  2. Navigate to the following registry key:

    HKEY_CLASSES_ROOT\lnkfile

  3. Right click on the right pane, and select New -> String Value (REG_SZ) to create a new registry value.
  4. Name the new registry value as IsShortcut . No value data required.
  5. Restart the computer.
If you already delete and other registry keys to remove shortcut arrows you must to do the same for all of them...
=============================================================================================

a. user say:

Thank you for posting the updated status of the issue.

The error code 0x80071A91 translates to " Transaction support within the specified file system resource manager is not started or was shutdown due to an error."

This issue occurs if the Windows file system transaction log is corrupted. The Windows file system uses the transaction log to recover system transactions when a file error occurs.

If the error occurs while installing Windows Update, I recommend you run the following command. This will in essence reset the File System Resource Manager (FSRM), and should resolve the issue with the failed updates.

Follow below steps recommended by our forum member Bsod:

1. Right click Start and select Command Prompt (Admin),
2. In the Command Prompt window, type the following command, and press Enter:
fsutil resource setautoreset true C:\

Note: These steps assume that Windows is installed in the default location, on drive C. If this is not the case, adjust the drive letter of the folder path to match your configuration.
3. Restart your computer, and then attempt to install the failed updates.

  • how many memory sockets are in your computer
  • how much memory is in each
  • the maximum memory that can be installed in your computer and how

Corel 12 compatibility with Windows 10

I am unable to install Corel 12 on Windows 10? Is this a universal problem? or is there something I should know about Corel 12?

bill.p. say: Hi Fiora, thanks for your question. I'm Graeme, a Microsoft Community expert and fellow user.

As Corel 12 is very old, it may not be compatible with Windows 10.

You could try running the program in compatibility mode. Here are the steps for that:

- Right-click on the program, and then select Properties.
- Go to the Compatibility tab.
- Check the box for Run this program in compatibility mode for.
- Click the drop-down menu, and then select older versions of Windows. Whichever works.
- Click Apply, and then OK.

Otherwise, the only other option you'd have would be running Corel 12 on an older version of Windows through Virtual Machine. Here is an article explaining the steps, if you'd like to attempt this:

fatou8 say:

Hello Stephan Bolliger,

Thank you for the feedback and clarification regarding the issue you are experiencing. We understand that holding down the comma on the Swiss version of the onscreen keyboard doesn’t allow the choice of a decimal point whereas, this is available on the German (Germany) version of the keyboard. We’re sharing this information with our colleagues to have this addressed.

While we are working on this, please either use the Germany version of the onscreen keyboard, or if you prefer, you can:

  • Click on the onscreen keyboard symbol.
  • Click on “DEU” in the lower right of the onscreen keyboard.
  • Choose either the 2nd or 4th keyboard icon displayed under the available languages. The 2nd has a split keyboard with a number pad in the middle, allowing you to type a comma and use the number pad. The 4th option is a full keyboard without a number pad. You may have to enable this option in your Surface’s settings first, but we would be glad to help you with that.

To enable the 4th layout for the keyboard:

  • Swipe from the right.
  • Tap “Settings” (Einstellungen).
  • Select “Change PC Settings” (PC-Einstellungen ändern).
  • Choose “PC and devices” (PC und Geräte).
  • Open “Typing” (Eingabe).
  • Set “Add the standard keyboard layout as a touch keyboard option” to on (Standardtastaturlayout als Bildschirmtastaturoption hinzufügen)

Thanks again for the patience and your description of this issue. We value customer feedback and will pass this along.
