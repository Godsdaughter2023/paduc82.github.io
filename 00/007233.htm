ken sheridan say: So, it's division, not multiplication.  The best way to do this is to model it relationally by having a table, PaymentTypes say, with columns PaymentFrequency and PaymentType.  Don't use Type as an object name BTW as it's a built in 'reserved' word in Access.  So it would have rows:

4      Quarterly
12    Monthly
52    Weekly

Your combo box would then be set up as follows:

Name:    cboPaymentType

RowSource:  SELECT PaymentFrequency, PaymentType FROM PaymentTypes ORDER BY PaymentFrequency;

BoundColumn:     2
ColumnCount:     2
ColumnWidths:    0cm

Access will change the last to inches if you are not using metric units.

Then in the form add an unbound text box with a ControlSource property of:

=[AnnualRent]/[cboPaymentType].Column(0)

The Column property is zero-based, so Column(0) is the first column, the payment frequency, which is hidden by virtue of the ColumnWidths property being zero.
