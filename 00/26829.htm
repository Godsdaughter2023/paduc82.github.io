hansv mvp say:

Thanks, that's helpful.

You execute the code in the After Update event of TextBox5. You can't set focus to a control in its own After Update event.

 

Use the Before Update event instead and set Cancel to True; this prevents the focus from leaving the control:

 

 

 

Private Sub TextBox5_AfterUpdate()
    Dim CheckArea As Range, TgtCell As Range, lstReconCell As Range
    Dim ExitLoop As Boolean
    Dim SearchName As String
    Dim i As Integer
    On Error GoTo Err

    Worksheets("RawData").Range("C1:N1").EntireColumn.ClearContents
    Worksheets("RawData").Range("D1").Value = "Reference"
    Worksheets("RawData").Range("E1").Value = "Cell"
    Worksheets("RawData").Range("F1").Value = "Name Entered"
    Worksheets("RawData").Range("G1").Value = "Date No.1"
    Worksheets("RawData").Range("B6").Value = 1
    Set CheckArea = Range(Range("A2"), Range("A1048576").End(xlUp))
    SearchName = TextBox5.Value
    Set TgtCell = CheckArea.Find(What:=SearchName, LookIn:=xlValues, _
        LookAt:=xlWhole, SearchOrder:=xlByRows, SearchDirection:=xlNext, _
        MatchCase:=False, SearchFormat:=False)
    Set lstReconCell = TgtCell
    'MsgBox TgtCell.Address
    Sheets("RawData").Select
    Range("E1048576").End(xlUp).Offset(2, 0).Select
    ActiveCell.Value = TgtCell.Address
    ActiveCell.Offset(0, 1) = TgtCell.Offset(0, 0)
    ActiveCell.Offset(0, 2) = TgtCell.Offset(0, 1)
    ActiveCell.Offset(0, 2).NumberFormat = "DD-MMM-YY"
    ActiveCell.Offset(0, -1).Value = (ActiveCell.Offset(0, 1) & _
        Format(ActiveCell.Offset(0, 2).Value, "DD-MMM-YY"))
    Sheets("Database").Select
    Do While ExitLoop = False
        Set TgtCell = CheckArea.FindNext(After:=TgtCell)
        If Not TgtCell Is Nothing Then
            If TgtCell.Address = lstReconCell.Address Then Exit Do
            'MsgBox TgtCell.Address
            Sheets("RawData").Select
            Range("E1048576").End(xlUp).Offset(1, 0).Select
            ActiveCell.Value = TgtCell.Address
            ActiveCell.Offset(0, 1) = TgtCell.Offset(0, 0)
            ActiveCell.Offset(0, 2) = TgtCell.Offset(0, 1)
            ActiveCell.Offset(0, 2).NumberFormat = "DD-MMM-YY"
            ActiveCell.Offset(0, -1).Value = (ActiveCell.Offset(0, 1) & _
                Format(ActiveCell.Offset(0, 2).Value, "DD-MMM-YY"))
            Sheets("Database").Select
        Else
            ExitLoop = True
        End If
    Loop
    'Align Cell References in order
    Sheets("RawData").Select
    Range("D1048576").End(xlUp).Select
    Range(ActiveCell, ActiveCell.Offset(0, 3)).Select
    Selection.Cut
    Range("D2").Select
    ActiveSheet.Paste
    ActiveCell.Select
    'Set-up ListBox1
    Dim DisplayRng As Range
    'Dim lngIndex As Long
    Set DisplayRng = Sheets("RawData").Range(Range("F2"), Range("G1048576").End(xlUp))
    'Fill the ListBox1
    ListBox1.RowSource = DisplayRng.Address
    With Me.ListBox1
        .ColumnWidths = "70;40"
    End With
    Exit Sub
Err:
    MsgBox Err.Description
End Sub
 

Private Sub TextBox5_BeforeUpdate(ByVal Cancel As MSForms.ReturnBoolean)
    Dim CheckArea As Range, TgtCell As Range
    Dim SearchName As String
    Set CheckArea = Range(Range("A2"), Range("A1048576").End(xlUp))
    SearchName = TextBox5.Value
    Set TgtCell = CheckArea.Find(What:=SearchName, LookIn:=xlValues, _
        LookAt:=xlWhole, SearchOrder:=xlByRows, SearchDirection:=xlNext, _
        MatchCase:=False, SearchFormat:=False)
    If TgtCell Is Nothing Then
        MsgBox "'" & SearchName & "'" & " not found in database." & _
            " Please type a new search name."
        Me.TextBox5 = ""
        Cancel = True
    End If
End Sub

sheldrick say:

Unfortunately there is a known issue with Ovi Maps (version 3.04) on the N8.

 

Ovi Maps will use the device's C: drive as location to store data (map data) if the memory card is not there or full.

 

The biggest problem is that in case of Maps there is no way to delete any of the map data or voice files on C: Drive because they are stored under private folders which can’t be accessed by end users.

 

Create a full backup of your device using Nokia suite (this is an insurance as if you restore from this back up after a hard reset the problem would persist).

 

You will have to go onto your phone, and using file manager extract the files that you wish to keep after the reset,  place them in a backup folder that you have created.

 

Perform a Hard reset on your phone by pressing *#7370# and using the security code of 123435 (if prompted and if you have not set your own personal security code).

 

Put back all your data from the back up folder that you have manually created, and hopefully this should fix your issue.

 

If your issue still persists, you need to go to a Nokia Care Point to get the device Factory resetted. This will of course cause a loss of personal data, too, so a back up is heavily recommended before flashing the device in Care Points.

 

Of course, the issue has been fixed in all coming Maps builds. Users who have a Maps version later than 3.04 10wk34_b03 are not affected by this problem.

 

I hope you manage to fix this problem, please get back to me as to whether the troubleshoot has worked or not!

 

 

Sheldrick

First copy the Name column (column B) and paste in column E. Select Column E and go to: Data>Remove Duplicates>check 'My data has headers' > OK. Now you have a unique list for the names.

=TEXTJOIN(", ",TRUE,IF($A$2:$A$100=E2,$B$2:$B$100,""))
