a. user say:

> Must be that Microsoft does not have technical capability to add bar code to an envelope.

 
There are other possibilities. Before going ahead, it's perhaps worth noticing that the USPS Postnet barcode system is the only one, worldwide, that Microsoft Word ever supported natively. The { BARCODE } field never worked with any other system (I don't know what others there have been, but there are plenty of countries out there). OK, so Microsoft is a US company and all that, but it is also a global company and like other such entities has to pay attention to legislation and perceptions across the world.

It is also the case that Word allows the insertion of graphics (in a number of different possible ways) and AFAICS it would not be beyond the bounds of possibility for any postal service, including the USPS to provide a mechanism that would allow a user to provide an address, and have the appropriate barcode graphic inserted. The question is who provides those features, how easy they are to use, whether or not they are free, and if not, how they are paid for.

In fact, for Word 2002, Microsoft created a mechanism specifically to support the insertion of ePostage and barcoding. This enables a provider such as USPS or anyone else to create addins that allow address verification, insertion of e-stamps and barcoding. As far as I know, the necessary infrastructure for that is in every version of Windows Word from 2002 (XP) to 2013, presumably with the exception of the Windows RT version. So I assume that the general idea was that any postal provider could support its customers using that and Microsoft was no longer going to do that itself.

In general, the Postal Service recommends that, unless you are preparing a mailing for automation prices, you should NOT put barcodes on the mailpieces. I wouldn't personally make the assumption that just because Microsoft is headquarted in the USA, it is any easier for Microsoft to discover which USPS requirements it really needs to deal with.

Restart NGSC

=================

1. Right click on the blue NGSC cloud icon in the System Tray, and select "Exit"
NGSC is the one that says OneDrive - (sitename)
2. Verify using task manager that NGSC is not running anymore
If the Exit button on the notification area cloud icon doesn't work
3. Open Task manager (ctrl+shift+esc), select the Details tab, right click on OneDrive.exe entry and select End Task
4. Turn NGSC back on
Search for OneDrive and click it

 

Unlink / Relink your account

=========================

1. Right click on the blue NGSC cloud icon in the System Tray, and select "Settings"

2. On the Account tab, click "Unlink OneDrive"

>This will immediately stop your sync connection, and bring you to the Sign in page

3. Click "Sign in"
4. Select the appropriate account
5. Provide the password

6. Click "Sign in"

7. Click "Next"

> This brings up an informational screen if there are already files in that location

8. Select "Use this location"

If these steps did not resolve the issue, repeat the unlink/sign-in process and select a different location

9. Select the folders you'd like to sync and click "Next"
