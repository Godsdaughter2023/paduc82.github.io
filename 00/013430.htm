Sub Fill_Down()
Dim LastRow As Long
Dim r As Range, x As Long
Dim v1 As Variant, v2 As Variant, v3 As Variant
Dim s1 As String, s2 As String, s3 As String
LastRow = Cells(Cells.Rows.Count, "A").End(xlUp).Row
s1 = "132105320,201105320,642015320,611035320,612025320"
s2 = "453115110,321015110,132105320,201105320,611035320,612025320,61301.5320,642015320"
s3 = "422115130,321015130,192105690,53105610,92505610,132105610,151125610,132105620,612025330,691305320,651105910,663105910,669205910,741105910,674105690,311015130,611015330,668205521,761105910,713105910,791105980,791205980,791305980"
v1 = Split(s1, ",")
v2 = Split(s2, ",")
v3 = Split(s3, ",")
For Each r In Range("A6:A" & LastRow)
If r.Offset(, 2) <> "" Then
    If Not IsError(Application.Match(CStr(r.Value), v1, 0)) Then
        x = r.Row
        r.Offset(, 23).Formula = "=MAX(I" & x & ",L" & x & ")"
    End If

    If Not IsError(Application.Match(CStr(r.Value), v2, 0)) Then
        x = r.Row
        r.Offset(, 24).Formula = "=if(z" & x & "<x" & x & ",ab" & x & "*ah" & x & ",max(i" & x & ",l" & x & ",u" & x & "*ah" & x & "))"
    End If

    If Not IsError(Application.Match(CStr(r.Value), v3, 0)) Then
        x = r.Row
        r.Offset(, 24).Formula = "=MAX(J" & x & ",M" & x & ")"
    End If
End If
Next r
End Sub

hansv mvp say:

By default, Excel creates an absolute cell reference when you link to a cell in another workbook:

 

=[OtherWorkbook.xlsx]Sheet1!$B$1

 

The $ characters make the cell reference absolute, i.e. the row and column will not change if you copy othe formula or fill it down or to the right. To make the cell reference relative, remove the $ characters:

 

=[OtherWorkbook.xlsx]Sheet1!B1

ActiveSync Error Code 85010004 - Microsoft Community

schmoov say:

ActiveSync Error Code 85010004

Why do I get this error during synchronization?

schmoov say:

Possible Cause:

A forbidden HTTP communication or protocol was used.

- OR -

Your account does not have permission to sync with your current settings. Contact your Microsoft Exchange administrator.


Possible Resolution:

Remove the item from the synchronization list.

1. The Microsoft-Server-ActiveSync virtual directory on your server is configured to require SSL and you are using a device without SSL.

2. The Exchange virtual directory on your mailbox server is configured to require SSL.

3. This error could occur if host headers are being used and the request goes to the wrong Web site.


User Action:

1. a. On your Pocket PC 2003-based device, click Start, ActiveSync, Tools, Options, Server and check the box “This server uses an SSL connection”.

2. On your Smartphone 2003-based device, click Start, ActiveSync, Menu, Options, Server Settings, Connection and check the box “This server uses an SSL connection”.

3. Verify that host headers are configured correctly.

4. Also, see Microsoft KB Article 817379 for information on configuring the Exchange virtual directory. See this link: http://support.microsoft.com/default.aspx?scid=kb;en-us;817379

5. See Error message when you try to synchronize a mobile device with Exchange Server 2003: "HTTP_403". Link: http://support.microsoft.com/kb/916958/en-us

6. XCSI/PRE/E2K3/Unable to synchronize with Exchange server using Active Sync. See this link: http://support.microsoft.com/kb/924216/en-us 


Thanks to Chris De Herrera for the source information in this answer: http://www.pocketpcfaq.com/faqs/activesync/exchange_errors.php

There is no mechanism in Word for editing or reformatting saved building blocks. It would be possible, however, to automate the procedure you mentioned by writing a macro. (And no, that kind of macro cannot be recorded; it would have to be written.)
