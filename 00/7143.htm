then this code I think solve the question:

            

Function TestCsv_A(yourFileName)
    Dim myRecord As String
    Dim myField As Variant

    TestCsv_A = False
    Open yourFileName For Input As #1
    Line Input #1, myRecord
    myField = Split(myRecord, vbTab)

    If Replace(myField(0), """", "") <> "Model" Then GoTo CloseFile
    If Replace(myField(1), """", "") <> "Serial No." Then GoTo CloseFile
    If Replace(myField(2), """", "") <> "Classification 1" Then GoTo CloseFile
    If Replace(myField(3), """", "") <> "Customer Name" Then GoTo CloseFile
    If Replace(myField(4), """", "") <> "SMR(H)" Then GoTo CloseFile
    If Replace(myField(5), """", "") <> "Last Communication Date" Then GoTo CloseFile
    If Replace(myField(6), """", "") <> "LAT." Then GoTo CloseFile
    If Replace(myField(7), """", "") <> "LONG." Then GoTo CloseFile
    If Replace(myField(8), """", "") <> "Night Lock" Then GoTo CloseFile

    TestCsv_A = True
CloseFile:
    Close #1
End Function

Hi Irshad,

try this code for generalize the check on any input file and table:

Function TestCsv_Table(yourFileName, yourTableName)
     Dim myDb As Database
     Dim myRecord As String
     Dim myField As Variant
     Dim i As Integer
    
     TestCsv_Table = False
         
     Open yourFileName For Input As #1
     Line Input #1, myRecord
'     myRecord = Mid(myRecord, 3)    'If needed
     myField = Split(myRecord, vbTab)
    
     Set myDb = CurrentDb()
    
     With myDb
          If .TableDefs(yourTableName).Fields.Count <> UBound(myField) + 1 Then GoTo CloseFile
          For i = 0 To .TableDefs(yourTableName).Fields.Count - 1
              If Replace(myField(i), """", "") <> .TableDefs(yourTableName).Fields(i).Name Then GoTo CloseFile
          Next i
     End With
    
     TestCsv_Table = True
CloseFile:
     Close #1
 End Function

Outlook Contact Groups - Microsoft Community

old blind dog say:

Outlook Contact Groups

If you frequently or even occasionally send Outlook emails to a large group of recipients, it can be a time saver and a way to avoid missing a recipient (or including an unintended recipient) if you create and save the Contact Group in advance.  You just give the group a name and add the recipients by selecting them from a list.

That way, you can simply put the Contact Groupâ€™s name in the To: field and Outlook will do the rest, sending a copy of the email to each recipient in the Contact Group.

Unfortunately, if you delete a Contact Group (or if one suddenly disappears, as they have sometimes been known to do), you cannot retrieve it from the Deleted Items folder. Deleting the Contact Group doesnâ€™t delete the contacts, just how they are grouped together with a name. For some reason, a Contact Group isnâ€™t treated the same way as other deleted items are by Outlook.

Even if you were to export your email and contacts to an Outlook PST file (personal storage), a typical way to backup or archive emails, you would find that the Contact Groups are not saved in those files either.

Rather than risk losing one or more Contact Groups, you can export them to a different file format so that they can be easily added back to Outlook should they be needed, but this is not an obvious procedure in Outlook.

OFT stands for Outlook File Template. Most people that use OFT files will use them to create a re-usable template for a type of email that they regularly send out, such as a welcome message to new members or employees, or an announcement or reminder of a regularly scheduled meeting. After you compose the body of such an email, you just select Save As and select the File Type as OFT.

But the OFT file can be used for saving and reloading Contact groups also.


With the Contact group displayed, select File, then Save As. Pick the OFT (Outlook Template File) file type, then check the folder and click Save.

To restore the group should it be deleted, just find the file and double-click on it.

shane devenshire say:

Tell us what you see when you do as Bill say, with the cell selected choose Format, Cells, Number and note the format.  There are many formats that would cause this. 

The number part of the format code is made of 3 components - the 1st one is for positive numbers, the 2nd one for negative number and the third for 0.  These components are separated by semi colons ;

For example:

0%;-0%;0%

defines the format for all thee components, however

0%;-0.0%;

indicates the first two components and forces the third one not to display

0%;;

would only display positive numbers

;0%;

would only display negative numbers

0%

would display all three in this format, the negative would show a - if you type the minus into the cell.
