Hi,

So Windows does get its time from the system BIOS. You need to check to make sure that the system time is working correctly.

Reboot your system and press the F1 key...I think...could be the Delete or F2...to enter the BIOS as your computer first turns on. If Windows starts...you missed it.
There will be a system setting for Date and Time. Make sure it is counting correctly as you view the time and that it is set correctly. If it isn't, Windows will be off as well.

It is possible that the motherboard battery is bad...if you specifically did not check it. Your Laptop battery is not the same one that the BIOS uses. If your main laptop battery goes completely dead, the BIOS uses MB battery, which is a little pancake battery inside. These usually will not go bad unless you don't use your computer much or you have an older system. These batteries can be replaced.

The Print Screen key is near the upper-right corner of your keyboard. (Depending on the type of keyboard you have, the exact key names on your keyboard may vary slightly.)

Note: You can take a screen shot of your entire desktop rather than just a single window by pressing the Print Screen key without holding down the Alt key.        

  1. Click Start, click Accessories, and then click Paint.

  2. In the Paint window, click Edit, and then click Paste.

  3. When the image appears in the Paint window, click File, and then click Save As.

  4. In the Save As dialog box, in the File name box, type a name for the screen shot, and then click Save.

Above mentioned issue DO NOT OCCUR on this environment. 

Analysis using network logs:
On investigating through network logs we found that, DPWS events from Scanner/MFP are blocked by Client PC Firewall. DPWS will send intermediate scan events which are not allowed by Firewall as 5357 port is blocked by default for in bound network traffic in Public network.
MFP retries sending scan events multiple times but failed for the same reason. Due to which Client side WIA service time out and hence sending RESET packet to MFP and close the connection.

Note that, in 'Private' network scanner events are reaching Client PC through 5357 port and scanning happened without any error.
