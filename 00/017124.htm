gary's student say:

Here is an example for function key F1 and clubs in Excel.  First enter this macro:

 

Sub club()
With ActiveCell
    .Value = .Value & "§"
    n = Len(.Value)
    .Characters(Start:=n, Length:=1).Font.Name = "Symbol"
End With
End Sub

 

this macro puts a club in the activecell.  Then enter and run this macro:

 

Sub setF1()
Application.OnKey "{F1}", "club"
End Sub

 

this macro assigns the F1 key to sub club.

 

After the assignment, touching F1 will put a club in the activecell.

 

Use similar code for the other suits.

john w. vinson say: Please post the SQL view of the Form and the Subform's Recordsource, the Master and Child Link Field properties, and any code or macros which might be affecting the data (by default no code should be needed just to add data).

a. user say:

A Form (“frm_Issue_Food_to_a_Client”) used to enter data into a table (“tbl_Food_Issued”).  The Form has a combo box used to choose a ClientID, and a text box used to enter a VisitDate and a subForm (“subfrm_ Issue_Food_to_a_Client”).  The subForm contains a combo box used to choose a FoodIDCode and a text box used to enter Quantity. 

The Problem:   When a Client Visit has occurred and the Form used, the table has correct records from the subForm, but it also has one record from the Form itself, with blank fields for Food IDCode and Quantity.  How can the entry from the Form (containing empty fields FoodIDCode and Quantity) be avoided?

For both the Form and SubForm, the Recordsource is tbl_Food_Issued.

After many trial-and-error attempts I solved the problem. 

The ‘problem’ involved a data entry Master/Child form/subform with two Master controls and two Child controls.  The form was recording in the data table a record with only the master control fields, followed by records with both the Master fields and also the Child fields; I wanted only the rows with records with both the Master and Child fields. 

In this situation, the Master fields in the data entry form were ClientID (a combo box control) and VisitDate (a text box control).  The Child fields in the subform were FoodID (a combo box control) and QtyIssued (a text box control).   For both the data entry form and the subform, the RecordSource was a table, tbl_Food_Issued. 

The solution: Open the enter the name of both Master fields (ChildID;VisitDate) – this may seem a little puzzling, since neither of the Master fields is a Child field on the subform.  However, that is the answer!

dhookom say: The main form should not have a record source.

in VBA
forms(“subfrm_ Issue_Food_to_a_Client”).AllowAdditions =False
