7.75 hours (pronounced 7 and 3/4 hours) is the same thing as 7:45 (45 minutes is 3/4 of an hour), just presented differently. I think you would want to leave the number in decimal form (7.75) so you can do numerical calculations with it (for example, multiply it by an hourly rate). If you convert the number to time format, you will have to remember to convert it back to decimal form when you want to do numerical calculations with it. If you absolute insist on seeing the value in Column G in time format, then divide the formula in Column G by 24 and then format the cell using h:mm as the pattern.

Hi Carl

First and most important thing to realize that in order to share contact/calendar data across multiple devices requires the use of an Exchange (including Outlook.com) account (this ignores the use of any 3rd party sync'ing programs)

Next, everything described below applies to any version of Outlook whether '2010 or from the Office 365 suite. It's not the Outlook version that makes a difference but the type(s) of email accounts that are configured so at the risk of being the bearer of bad news, if the only reason for purchasing Office 365 was for a central source of contact/calendar data, you made an unnecessary purchase.

POP and IMAP accounts do not sync contact/calendar data with the server.

re: your statement of
<<
I have 3 email addresses- outlook, Pop and Google IMAP.
<<

Not sure what exactly that refers to specifically in terms of whether you have an Outlook.com account configured as an Exchange account PLUS some POP account OR you have an Outlook.com account configured as a POP account.

In any case, even if you have an Outlook.com account configured as an Exchange account, the next important item is which data file is designated as the "default data file" for the profile. The "default data file" is where the default contact/calendar folders are stored. To determine which data file is set as the "default data file" - from Outlook menu, go to File --> Account Settings --> Data FIles tab.

......... next up - importing et al

If you installed Office 365 on a machine where '2010 was installed and Outlook, configured (otherwise known as an "in-place" upgrade) - there is absolutely nothing to import.

From what I can surmise from your description, what you want to do is get all your contacts into the contact folder in the Outlook.com (Exchange based) folder tree. Once that's done, as long as you Outlook.com configured as an Exchange account on any other device, you now have a single location shareable by other devices.

Am sure you're going to have some follow-up questions so feel free to ask away but be sure to take note of type of your Outlook.com account (EX or POP) and which data file is set as the default for the profile. For what you want to do, the .ost file for the Outlook.com account should be configured as the default data file. By the same token, there is no requirement to make the Outlook.com account the "default email account" (under the Account Settings --> Email tab) - those are two entirely separate and distinct settings which are mutually exclusive.
