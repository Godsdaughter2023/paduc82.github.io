jllatham say:

Quite frankly, probably not with any reliability.  Both the format (sequence of information) and content (words used) would have to be consistent throughout the entries.

It appears relatively simple to split off the mailing codes (DL3 0LF and BR1 4PP), but then almost immediately you get into questionable areas:

Should it be "Darlington Co" and "Durham"  or should it be "Darlington" and "CO Durham" - an almost human must make the call situation.

Going the other way you've got a problem with the question of 'where does the street address end?'.  In your examples both end with the word ROAD, but how many other possibilities for that are there?  RD, Lane, Ln, Drive, DR, Way, Wy, Avenue, AVE, etc.

If you had a list of all possible endings for the street address (all those RD, ROAD etc words to use as a marker) then it would be possible to divide them into at least 3 groups:

83 WEST AUCKLAND ROAD

DARLINGTON CO DURHAM

DL3 0LF

for that first example.  That could be done with a macro, or 3 separate User Defined Functions (UDF).  A UDF is a macro that can be called in a worksheet formula just like a built-in Excel function.  So we might create 3 UDFs, one to 'find' each of those 3 sections.  So we might end up with

GetStreetAddress(whole address cell)

GetGeoArea(whole address cell)

GetPostalCode(whole address cell)

So in 3 cells you might have formulas like this where the entire address was in cell B2:

=GetStreetAddress(B2)

=GetGeoArea(B2)

=GetPostalCode(B2)

But this doesn't deal with 2-line street addresses and still leaves the City and County grouped together.

each record of the employees  should hold the date of certification and the calculated expiration date based on it:

Expiration date = DATE(YEAR(DateOfCertification)+2,MONTH(DateOfCertification),DAY(DateOfCertification)-1)

Use the conditional formatting to tell you that the certification expires:

If the actual date is greater or equal to the expiration date Excel formats the record of the employee in the chosen format.

crtate say:

How to stop showing a negative week number in a new row

I'm using the following formula to calculate week numbers from a given start date:

=TRUNC((($C8-$D$3)+6)/7)+(WEEKDAY($C8)=WEEKDAY($D$3))

When I start a new row however, I get a horrible looking large negative number which I feel is likely to deter users.

Does anyone know how I'd amend this to show a 0 when starting a new row, until a date is entered?

Thanks.

davem121 say:
Hi CRTate, you can do that by wrapping your existing formula in a Max Function:

Generic form:

=MAX(value, 0)

So your formula becomes:

=MAX(TRUNC((($C8-$D$3)+6)/7)+(WEEKDAY($C8)=WEEKDAY($D$3)),0)
