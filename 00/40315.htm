    a. user say:

    Here is what is happening (The mediocre teacher tells, the good teacher explains...):

    If you install Windows 7 on a clean disk with no existing partitions, it creates a System Reserved partition at the beginning of the disk and uses the remainder of the unallocated space to create your system drive. That small partition isn’t assigned a drive letter, so you won’t even know it exists unless you look in the Disk Management console or use a low-level utility, such as Diskpart, to inspect the disk structure.

    The System Reserved partition consumes 100 MB of space on Windows 7, 350 MB of space on Windows 8, and 500 MB of space on Windows 10.

    Example for Windows 7:

    The reason it seems to only take a moment or a flash of seconds to defrag System Reserved it because the size is so tiny compared to the rest of the drive space making up your system drive.

    TechNet says:

    This “stub” of a partition, which is new in Windows 7, serves two functions. First, it holds the Boot Manager code and the Boot Configuration Database. Second, it reserves space for the startup files required by the BitLocker Drive Encryption feature.

    You can't use the Command Prompt to defrag it because it has no drive letter associated with it (sorry Microsoft engaged alleged "experts") and you would not even know it was there unless you use something like Disk Management.  That means the advice from the Microsoft engaged alleged "experts" to defrag System Reserved makes no sense at all.

    If you defrag System Reserved from Disk Management (which won't take long) you will only see a message like this in the Event Viewer:

    The disk defragmenter successfully completed defragmentation on System Reserved

    After Windows startup is complete the System Reserved partition is no longer needed and everything after that involves only your main Windows partition (usually the C drive) so System Reserved will have nothing to do with the performance of your system.

    If you don't like it, don't want to see it, find it annoying and want to get rid of it you will have to reinstall your Windows from scratch and the TechNet article tells you one way to do that.  If you do that, you will never be able to use BitLocker on your system in the future.

    Since System Reserved contains Boot Manager and Boot Configuration data if you decide to mess with it, try to move it and make a mistake you might find yourself with an unbootable system and messages like this:

    More information (which I just copied and pasted here) can be found with some simple Google searches and here is one such article:

    https://www.howtogeek.com/192772/what-is-the-system-reserved-partition-and-can-you-delete-it/
