rob koch say:

The vulnerability is a problem that exists in the original code (operating system or application) that can be exploited in order to compromise the operation of the application or the entire system.

Exploit code is capable of taking advantage of the vulnerability it relates to in order to perform some action, typically to install other malware code on the system via the temporary remote execution or raised privilege access that the vulnerability allows.

The malware installed usually performs the sometimes more visible actions such as taking control of the system and/or stealing or siphoning off information.

An update or "patch" as they are often called actually fixes the problem in the original code, removing the specific vulnerability permanently and thus completely blocking the operation of any current and future exploit code using that specific vulnerability and so any attempts to install malware using it as well.

Antivirus and most other security applications typically only detect the final malware installation at best and then attempt to block and remove them before they can become fully active.  In some cases they can detect an exploit, but since the attack has typically already taken place before the detection occurs, it may be too late at that point to actually stop the malware infection.

So in truth, the only real way to insure protection is to install the updated patch, with antivirus at best performing as a "last resort" attempt to stop or at least detect the malware infection and inform the user.

So no, antivirus can't fully replace patching and is always nothing more than a last resort.  And as time wears on and more unpatched vulnerabilities are found in Windows XP, the risk of successful exploit and infection will climb rapidly, no matter what the antivirus programs attempt to do to stop them.

Rob
