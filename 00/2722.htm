ron sommer mvp 2009-2017 say:

When you setup pop3 access to other email accounts, you are having the outlook.com server download emails from other servers using pop3 protocol.

Hotmail/outlook.com uses several servers.  Some of your accounts could be on the same server, so the outlook.com server would be using pop3 to download emails from itself.

The outlook.com server will send emails for other accounts.

 

You can use Windows Live Mail to move emails between accounts.

Add you outlook.com account.

All emails and folders will sync to Windows Live Mail.

 

Now add your Hotmail accounts using pop3. 

Turn on POP for your Hotmail/Outlook.com account.

  1. Sign in to your Hotmail/Outlook.com account.

  2. Click the Options icon , and then click More mail settings.

  3. Under Managing your account, click Connect devices and apps with POP.

  4. Under POP, select Enable.

  5. Click Save.

 

By default, Windows Live Mail will setup a Hotmail account as a http account.  Manually enter the settings to add the account as pop3.

  • Incoming (POP3) Server

    • Server address: pop-mail.outlook.com

    • Port: 995

    • Encrypted Connection: SSL

  • Outgoing (SMTP) Server

    • Server address: smtp-mail.outlook.com

    • Port: 25 (or 587 if 25 is blocked)

    • Authentication: Yes

    • Encrypted Connection: TLS

  • User name: Your email address

  • Password: Your password

  • If you don’t want email to be deleted from your Outlook.com inbox after it’s downloaded to your email app, select the Leave a copy of messages on the server option.

After the account emails have downloaded, copy emails to an outlook.com account folder.

After doing a send /receive, the emails will sync back to the outlook.com server.

tom van stiphout say:

There is no such thing as code that works all the time. There are hundreds of millions of Windows computers, and most of them have a unique installation.

For example, unbeknownst to the user Outlook may already be running in the background.

Or it may already be running minimized.

There are just too many permutations, and it is not cost-efficient to deal with them all. Unless your customers are paying for your time (which I'm guessing they are not).

There are a few things you can do with Windows API calls, e.g. normalize a minimized application.

The only real improvement in your code that I would recommend is around the CreateObject line. You should first GetObject, so you can re-use an existing instance (if any), and only if that fails, use CreateObject.
