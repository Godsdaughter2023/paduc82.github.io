doug robbins - mvp office apps & services (word) say:

That is not correct.  You can add items to the existing tabs/groups, without having to re-create them completely

See the following page of Greg Maxey's website:

http://gregmaxey.mvps.org/Customize_Ribbon.htm

Here is some code that adds some additional items to the Finish Group of the Mailings tab of the ribbon in Word

<customUI xmlns="http://schemas.microsoft.com/office/2006/01/customui";>
  <ribbon>
    <tabs>
      <tab idMso="TabMailings">
        <group id="CustomGroup1780555409" insertBeforeMso="GroupMailMergeFinish" label="Merge Tools">
          <button id="Custombutton163655576" imageMso="FileMenuSendHeader" size="large" label="Many To One" onAction="ManyToOneBtn"/>
          <button id="Custombutton163655577" imageMso="ChartPrimaryHorizontalGridlines" size="large" label="Chart Merge" onAction="MergeChartBtn"/>
<button id="Custombutton163655578" imageMso="CacheListData" size="large" label="Duplex Merge" onAction="MergeDuplexBtn"/>
<button id="Custombutton163655579" imageMso="ReviewProtectWorkbook" size="large" label="Merge Form" onAction="MergeFormBtn"/>
<button id="Custombutton163655583" imageMso="ProtectDocument" size="large" label="Protect Merge Form" onAction="ProtectFormBtn"/>
<button id="Custombutton163655580" imageMso="ContactSendMenu" size="large" label="Merge with Attachments" onAction="MergeAttachmentsBtn"/>
<button id="Custombutton163655581" imageMso="RecordsSaveAsOutlookContact" size="large" label="Merge to Individual Docs" onAction="MergeIndividualBtn"/>
<button id="Custombutton163655582" imageMso="NameManager" size="large" label="Merge Print and Staple" onAction="MergePrintStapleBtn"/>
        </group>
      </tab>
    </tabs>
  </ribbon>
</customUI>

You can Use the Built In Ribbon Customizer to Build transferrable Custom Ribbons

Actually you can export your current ribbon customizations and thus you CAN use the built in ribbon customizer.

Because you customized the main application ribbon as you note they are mumbled jump up all together. I not sure if it worth your trouble to EXPORT the customization and THEN try to pull out each part (but it certainly worth a try).

However, assuming you wanted to use the built in ribbon customizer?

When you are done you export the ribbon customization to a text file.

You get something like this:

<mso:customUIxmlns:mso="http://schemas.microsoft.com/office/2009/07/customui">
<mso:ribbon>
<mso:tabs>
<mso:tab idQ="mso:TabHomeAccess">
<mso:group id="mso_c1.53A65CF" label="EmailTest" autoScale="true">
<mso:control idQ="mso:FileSendAsAttachment" visible="true"/>
</mso:group>
</mso:tab>
</mso:tabs>
</mso:ribbon>
</mso:customUI>

The above can THEN be pasted into the USYSRibbons table as your custom ribbon.

I built tons and tons of ribbons and I never learned to read XML and I don't really have to know anything about xml.

The only addition you need to above is this one line:

<mso:customUI xmlns:mso="http://schemas.microsoft.com/office/2009/07/customui">
  <mso:ribbon startFromScratch="false">   <-- this one you need to add
  <mso:tabs>
… rest of custom ribbon here

You can use "true" or false in the above start fromscratch if you want to hide the rest of Access built in ribbon.

So you can MOST certainly use the built in ribbon customer. You can MOST certainly export that custom ribbon you created. And you can MOST certainly use that ribbon as the basis for a custom ribbon in your application.

As noted I built tons of ribbons before I knew anything about xml and even to this day I NEVER bothered to read or worry about the XML. It just a simple cut + paste process.

I would consider resetting your ribbon. Then create the one custom ribbon and THEN export that ribbon, save it.

You then re-set and then do the process again for the second ribbon.

Thus you can most certainly use the built in ribbon customizer to create ribbons for your application using this approach.

So ribbon customizations you create using the built in ribbon customizer can be exported for use in a single custom ribbon that is not part of the Access overall customized ribbon.
