Nokia 1520 - how to forward calls? - Microsoft Community

kumarchhugani say:

Nokia 1520 - how to forward calls?

Hi. In setting>phone.......there is no option for call forward on Lumia 1520

 

Moderator's note: We amended the subject as the post was moved to the appropriate board.

paul van der heu say:

Call forward features are a function of the network and as such need to be supported by the network for your contract. In most cases call divert can be setup as follows:

 

Feature Mode Sequence
All calls Activate & register number ** 21 * Number # SND
Deactivate & deregister number ## 21 # SND
Activate to registered number * 21 # SND
Deactivate # 21 # SND
Verify *# 21 # SND
No answer Activate & register number ** 61 * Number # SND
Deactivate & deregister number ## 61 # SND
Activate * 61 * Number # SND
Deactivate # 61 #
Verify *# 61 # SND
Unreachable Activate ** 62 * Number # SND
Deactivate ## 62 # SND
Verify *# 62 # SND
Busy Activate ** 67 * Number # SND
Deactivate ## 67 # SND
Verify *# 67 # SND
All types Deactivate ## 002 # SND
All conditional types Deactivate ## 004 # SND

 

If you want to extend the time before a divert activates you can add **[time in seconds] after the number. So to activate divert for all calls to number 12345 after 30 seconds you would enter:

 

** 21 * 12345 ** 30 # SND

 

 

The codes mentioned are what most operators use, but they can be different so check with your operator. Also if these codes are not supported by your network you will get a 'Unknown request' or ' request denied' from the network as a response.

allisonpc say:

Calculating "TEXT" data fields in Access

I have a table that I am storing all "Text" data.  I need to perform Average calculations and IF statements on these values.  The reason I am storing the data as "text" and not "Number" is because some of the values are "<" values.  In the end, what I am wanting to do, is display the average of the values in the "Result" textbox,BUT, if the values in the two fields are "<1", the the result field would show "<1".  Is there a way to do this without getting "#Error" like I am now?

Thanks!!

john w. vinson say:

Rats. I guess that Access goes ahead and calculates both branches of the IIF. Try

SELECT STAnalysis.Date, STAnalysis.SampleID, STAnalysis.WaterA, STAnalysis.WaterB,
(IIF(Left([WaterA], 1) = "<" OR Left([WaterB], 1) = "<", [WaterA], (Val([WaterA]) + Val([WaterB])) / 2) AS AvgWater FROM STAnalysis;

The Val() function will return 0 if the field value doesn't start with a digit; you'll get a zero sum, but what you'll see is the value in [WaterA], whatever that might be.
