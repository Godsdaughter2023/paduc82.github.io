john mcghie say:

Most people who use Word have this issue at one time or another: your problem is that you have not seen it before and don't recognise it :-)

What has happened is that Endnote has made a change to your template, and in doing so, has changed the Outline Numbered list template.  You can simply put it back the way it was.

  1. On the Ribbon, in the Paragraph chunk click the Outline List button.
  2. In the dialog that appears, look in the List Library section.
  3. Choose the sample that shows 1. Heading 1, 1.1 Heading 2, 1.1.1. Heading 3...
  4. Click it.
  5. That will fix the current document.
  6. To fix documents you create in future, on the Styles chunk choose Heading 1, right-click it, choose Modify...
  7. At the bottom left of the dialog, check the box that says "Add to template". OK it.
  8. Save the document and quit Word.  If you are prompted to save the template, say "Yes".

That's all you need.

Hope this helps

ken sheridan say: I would strongly recommend that wherever possible when passing a date/time value as a string expression the ISO Standard format for date/time notation of YYYY-MM-DD hh:mm:ss be used.  The standard differentiates months and minutes by case.  In Access the format is yyyy-mm-dd hh:nn:ss, with 'm' representing months and 'n' minutes.

If you cannot control the format in which the date/time value is passed as a string expression, but are forced to use the regional date/time format, then you can either format the value to ISO standard when inserting it as a date literal in Access, or you can call the CDate function, which respects the regional format.  You can see this in the debug window:

strDate = "#" & Format("09/12/2014 13:00:01","yyyy-mm-dd hh:nn:ss") & "#"
dtmDate = Eval(strDate)
? dtmDate
09/12/2014 13:00:01

strDate = "09/12/2014 13:00:01"
dtmdate = CDate(strDate)
? dtmdate
09/12/2014 13:00:01

In each case the underlying value of dtmDate is a 64 bit floating point number which represents the date/time regardless of the regional date/time format in use, so will be correctly interpreted on any system.  In the above the values are returned in my local UK format of dd/mm/yyyy. You can see the actual value with:

? CDec(dtmdate)
 41982.5416782407

i.e. the date/time is that number of days from 30 December 1899 00:00:00, which is 'time-zero' in Access's implementation of the date/time data type.

bob jones aka: cybertaz say:

Regardless of the accuracy of his history lesson, Phillip is correct that the option has been eliminated in both the PC & the Mac versions of Word.

The Page Color feature you're attempting to use is not a viable alternative. For one reason, the old blue background/white text preference technically affected the document display in the User Interface. It had no bearing on the actual properties of the document nor did the document print in those colors. Page Color, OTOH, is a document property which is unrelated to font color. Also, both Page Color & Font Color changes are printable.

Note the separator line between the Document Formatting group & the Page Background group on the Design tab. Page Color is not a component of a Document Theme, which is why your choices are not being set as Default... Only the items in the Document Formatting group are included when you click the Set as Default button.

My guess is that you'd find the Accessibility setting for White on Black to be just as problematic if not more so, unless you have a true vision impairment. What may work better for you is to adjust the Brightness settings for your display using the designated F-Keys. I've also found it helpful to be selective about positioning whenever possible in order to avoid lighting conditions that necessitate max brightness.

For full disclosure I would point out that you can open your Normal.dotm template, then save the preferred Page Color & Font Color settings in the template. That would cause any new documents you create to open using those settings. However, as I mentioned before, every time you Print or PDF a document, changes would have to be made in order to not print using those colors as well for any documents you forward to others... Trust me, it's more of a PITA than it's worth for most users :-)
