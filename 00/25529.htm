Wi-Fi ad-hoc connection between two Nokia N8 phones - Microsoft Community

funking---01 say:

Wi-Fi ad-hoc connection between two Nokia N8 phones

Dear all,

 

Short and to the point: how can I start an ad-hoc connection between two N8s?

 

Long version:

 

I wrote an app that allows two Nokia N8 phones to exchange data wirelessly between each other, via Wi-fi. It works fine no matter what kind of access point is used (infrastructure or ad-hoc), but the ad-hoc connection is preferred since you won't need a router and can use the app anywhere.

 

Unfortunately it seems to be really complicated to start the ad-hoc connection because it is never listed in the WLAN wizard (unless one phone is already using the ad-hoc connection).

 

First, here's how I create the ad-hoc connection in each phone (this operation is done only once, of course):

 

  • Open the menu by pressing the menu key.
  • Navigate through: Settings → Connectivity → Settings → Destinations → Access Point (Add new acc. pt.) → No (when asked "Automatically check for available access points?") → WLAN → (give it a name: adhoc) → Ad-hoc (public) → WEP
  • When asked for a "WEP key for Wi-Fi", enter an easy to remember code with 10 or 26 hex. characters (only numbers 0 to 9 and letters A to F)
  • Select destination: Internet → Back → Back
  • Now I set a fixed IP for each phone: Settings → Destinations → Internet → adhoc → Options → Advanced settings → IPv4 settings → Phone IP address →
  • 10.0.0.1 for one phone, 10.0.0.2 for the other
  • OK → Subnet mask → 255.255.255.0 → OK → Default gateway → 10.0.0.0 → OK

Before I open the app I start the "adhoc" connection following these steps:

  • Using phone "A", open the Wi-Fi wizard (here's one way: Menu key → Settings → Connectivity → WLAN wizard)
  • Other (hidden netw.) → Connect → WLAN name: adhoc →
  • It will search for an existing network with that mode. It should not find one, though... but we will be able to continue, anyway.
  • Select Ad-hoc (public) → Security mode: WEP → WEP key for WLAN: use the same created during the first-time configuration
  • The phone will appear as connected to "adhoc(01)" or something similar. If I use this access point my application will not work!
  • Now open the WLAN wizard on the "B" phone and the "adhoc" is listed. Just select "adhoc" → Connect (if it doesn't appear in the list, click Options → Refresh).
  • Disconnect "A" from "adhoc(01)" and connect it to "adhoc".
  • Now one can start the app in both phones.

I think this is too complicated for most users, not to mention time consuming. I could write an app to automatically start the access point, but the reasons that makes this complicated are many and not related to this post.

 

So my question is: could someone tell me how can I start an ad-hoc connection between two N8 phones in a very simple, easy and fast way?

 

Thank you!

Yes, it's not only possible, but the whole point of an ad-hoc connection is that you don't need a router - it is a direct, peer-to-peer network. You can visit this Wikipedia article for more information.
