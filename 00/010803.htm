barb way say:

There is not a central 'Off' setting to disable the Auto Sizing function.  Some templates in Visio enable if by default and some do not (such as Floor Plan and other scaled drawing types), based on common usage expectations.  Once you disable it for a page, it will retain that setting (stored in the shapesheet), and any pages you create based on that page will inherit the setting, but all existing pages that you wish to change will have to be updated through manual methods or VBA as you have discovered.

doug robbins - mvp office apps & services (word) say:

By default, the page numbering of a new Section will be set to Continue from previous section.  However, if the numbering in the previous Section is set to Start at 1, then that setting will also apply to the page numbering in the new Section.

Make sure that in the template from which you are creating the documents (assumed to only contain a single Section, that the page numbering in that Section is set to the default of Continue from previous section.

hoverdog say:

These entries on the Start Menu represent broken MS apps (Bing Travel, Reader, Windows Scan, etc. in your case) In my case the entries appeared when I tried to uninstall their apps (all except that last one, winstore, which was already there. Haven't tracked it down yet. The Store app appears as "windowsstore"). The uninstall process had evidently failed, leaving these non-functioning stubs. I tried to re-uninstall those apps having stubs left on the StartMenu, using PowerShell, failing again, but generating useful error info. I found that the failures happened due to xml files specific to each of those apps that did not exist when the uninstaller looked for them in the AppRepository folder (C:\ProgramData\Microsoft\Windows\AppRepository).

Feeling like I had nothing to lose, I copied xml files belonging to other apps and renamed them to the names of files not found and re-ran the uninstalls. They worked.

I then ran general reinstall of all MS apps from Power Shell, and the apps I had repaired came back just fine.

A couple others I had not previously tried to uninstall generated errors and broke in the general reinstall, so I have to fix those now.

Evidently the missing xml file will break an app coming or going.
