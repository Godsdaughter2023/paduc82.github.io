rohn007 say:

404 means that page is not found on the internet.  Either the file name is different, or it has to be reloaded up to the web.

As an external user, when I get that message the first place I check is the "Waybackmachine":  https://web.archive.org/web/

It is an organization that scans the web and archives "backup" copies for exactly this sort of problem.  But it appears that this site has not been scanned and archived there. There is a technique that webmasters can use to block this sort of scan by making an no scan entry in the ROBOTS.TXT file

As well, the archive has setup a process where you can automatically redirect 404 messages to the archive, in the hope of picking up an archived copy of the missing page

https://blog.archive.org/2013/10/24/web-archive-404-handler-for-webmasters/

diane poremsky mvp (slipstick.com) say:

Past appointments shouldn't show on the To-do bar.

If you are going to make a lot of exceptions, set the recurring appointment to end after fewer occurrences or create the apointmenta in excel (so you can use excel features to set the dates) and import. You can also make one event in outlook and copy and paste to the orher dates to create individual events.

to fix the existing ones-  open the series and reset the date to start now and end after 10 occurences. That will rebuild the event and remove all the exceptions. You could repeat this every few occurances.  Note that if there are any past appointments in the series that you wanted to keep, they will be lost if you do this.
