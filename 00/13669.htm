The autonumber will be generated as soon as you begin editing any bound control.  So of course it will be generated when you click a check box and change its value.  I would not expect it to be generated if you just click in a text box, unless you have code or a macro that in some way dirties the record when one of the fields receives the focus.  If that is happening, you should look to see what code or macro is making it happen.

If you want to discard the "dirty" record, press the Escape key once or twice until the "editing pencil" disappears from the record selector and the autonumber field resets to "(autonumber)".

scottgem say:

This is the way Access works. If you "dirty" the record as soon as you move to another record or close the form the record will be saved.

You can do as Dirk suggested and hit Escape until the record clears. Or you can code a Cancel button that will do, essentially, the same thing. But once the record is dirtied, the autonumber will be used and you can't get it back.

If you need to sequentially number the records, see my blog on Sequential Numbering. However, you should make sure you need sequential numbering. I don't really see a need for it based on what you have told us.
