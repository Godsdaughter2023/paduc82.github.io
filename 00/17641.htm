The common issue with referencing something on a subform is the name of the form used as a subform is irrelevant.  The name that must be used is the name of the subform Control that contains the form display as a subform.  Access usually defaults the control name to the form name, but that depends on how you created the subform control so it's always the first thing to check when the reference doesn't work.


ANother situation that can break a reference is when the main form is a Navigation Form where there is only one subform control for all subforms and the current subform is not the one you want to reference.

how to delete the last white page? - Microsoft Community

glorimolin say:

how to delete the last white page?

I created a document with one table with l8 rows and 5 columns, includes page numbers, at the end of the document
shows up a white extra page with continuous  page number. I cannot delete the number and the white page.


bob jones aka: cybertaz say:
Turn on the non-printing characters (¶) if they aren't already displayed. (Either use the button marked with that symbol or use the
In order to resolve the blank page you'll have to make one of several adjustments which will allow that last ¶ to move to the bottom of the previous page. When it does, the blank page will automatically disappear along with the page number. These are several ways to make the adjustment;
  • If there are any blank ¶ symbols in the document, delete at least one of those empty paragraphs. Empty paragraphs for vertical spacing purposes is not a good practice in the first place :-), or
  • Modify the Row Height of the table, or
  • If Space Before/After paragraph spacing has been used you can modify the appropriate styles to reduce the amount of space or directly change the paragraph spacing (not Line Spacing) of specific paragraphs, or
  • Select the last ¶ & change the font size to 1 pt, or
  • Set the top and/or bottom Margin to a slightly lesser value.
Exactly what to change & how much to change it depends on a number of variables, especially when dealing with a table where the rows may be differing heights.

Bob J.

My question is: what are the maximum settings for resolution and bitrate for H.264 encoded video's for the N95 (8gb)?

el_loco---01 say: Hi,

Check my blog:
http://nokiavideo.blogspot.com/

I haven't done extensive testing but from my experience;

Maximum resolution for H264 is QVGA. That's it. It's confirmed. you could try CIF but then it doesn't match the screen so it doesn't look as good. VGA doesn't work. PERIOD. It's unsupported.

Bitrate, well, I've encoded videos at H264 1 Mbps and even over this, and they play BUT, and please keep this in mind, any bitrate over 500 kbps (IMHO) yields no visible improvements to the quality of the video when played on the phone. The only thing you get with higer bitrate videos is larger file sizes. IMO.

So I see no point in using bitrates over 500-600kbps.

If you want better quality videos I recommend using MPEG-4 at VGA resolution and 1200 kbps. These videos will look really good on the phone screen and better yet when used with TV-out in an external TV or monitor.

el_loco---01 say: Well in theory it shouldn't make a difference, but in practice I found that MPEG-4 seems to be more stable as a codec.

There are not many advantages to have VGA resolution in a phone with QVGA screen. The only one is if you use TV-out cable.
