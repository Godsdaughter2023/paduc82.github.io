jay freedman say:

I don't see any way that macro could ever have worked, for two or three reasons.

Most immediately, there should not be quote marks around the variable name strPath in the Documents.Open statement. With those marks being there, you're passing the literal string "strPath" rather than the value of the variable, so it's no wonder that the file can't be found.

Second, if this macro code is stored in a template (such as Normal.dotm or some other global template/add-in), then the statement strPath = ThisDocument.Name will place the name of that template in the strPath variable. Instead, it should say strPath = ActiveDocument.Name, or possibly strPath = ActiveDocument.FullName to include the folder path.

Third, if the macro code is not stored in a template but in the active document, the macro will stop executing as soon as the ActiveDocument.Close statement runs.

jay freedman say:

Is there a way to make this happen very quickly?  I'm getting a blank screen for a very short period of time and I would love (in a perfect Word world) to make this save, close, reopen transparent to the user.

Cindy

I doubt you can make it happen any more quickly than it already does -- that depends on the size of the document and the speed of the drive where the document is saved -- but you may be able to stop the screen from blanking. Try this: Put the statement

   Application.ScreenUpdating = False

before the Close statement, and put the statement

   Application.ScreenUpdating = True

after the Open statement. That should "freeze" the screen during the close-and-reopen operation. I can't guarantee that it will work, as some things can ignore the "freeze", but it's worth a try.

How can I see all keyboard shortcuts that are actually operative?

Is there a way to display a full list of all keyboard shortcuts that are actually operative in a particular installation of Word 2010?

That gets quite laborious when you have to try one combination after another before you find one that is available because it's unassigned or can be switched from a less important command.

The particular need arises because I use various Word add-ins that seem to have reassigned a number of  familiar key combinations -  but, again, the only way to check this is to test each key combination, one at a time, using the customization dialogue.

I know you can get Word to print a list of assigned shortcuts from the Print dialogue, but that only shows the few shortcuts that I have myself assigned, not all the shortcuts that actually function in the installation.

hansv mvp say:

Try the following:

Select File > Options.

Click 'Customize Ribbon'.

Click Customize... next to 'Keyboard shortcuts'.

Click in the 'Press new shortcut key' box.

Press Shift+F3.

Select 'All commands' in the list of categories.

ChangeCase will be in the list of commands.

Select it and click Assign.
