aiscer say:

I was very intrigued by this issue. Hence I tried to scour the net looking for solutions. I am assuming that you may be fearing worms or any forms of attack when port 445 is exposed. I also have read the specifications of your router and it seems it offers a wide range of protection.

Here's a link that I got from the internet that may also help answering your question : Enable or disable the File and Printer Sharing exception.

If I understood what you are trying to achieve correctly, here is an excerpt from that resource : Enabling the File and Printer Sharing exception allows incoming Internet Control Message Protocol (ICMP) echo traffic, which enables computers to ping your computer. To block incoming ICMP echo traffic, you must edit the File and Printer Sharing exception so that TCP port 445 is disabled.

In Windows XP blocking or disabling port 445 is quite messy. Although a thread I found <here> had considerable success in disabling Port 445 and was still able to use File and Printer sharing. But your mileage may change depending on how the computers are actually accessing the shared resources.

Here is how you can disable the TCP port 445 with the following steps:

•Open regedit using the RUN option. Make sure you back up the key by using export! If at anytime there is either an impact to network sharing go back to the key then add back \Device\ or merge the exported key back into the registry.
•Go to the following registry key :

HKLM\System\CurrentControlSet\ Services\NetBT\Parameters

•In the right-hand side of the window Double click TransportBindName then delete the default Value Data. This means give it a blank value and not the key itself.
•Close the registry editor window.
•Restart your computer. After Restarting open a Command Prompt & run the following command:

netstat -an

See if there are any replies to port 445. What the steps performed will actually force communications through NetBIOS and use port 139 instead to connect to shared resources. Make sure that in the Advanced TCP/IPv4 settings under WINS is set to Default for your NIC.

Under Vista or Windows 7 you may need to create a New Rule for this, since the File and Printer Sharing Inbound Rules will not allow you to customize or configure its ICMP Settings under the Protocols and Ports selection within a Private Profile.

Be wary also of the impact or functionality this may cause to your shared resources.

Good luck!

Beware of Product Keys Sold Separately


With the exception of Product Key Cards (PKC’s) distributed with COA’s, Microsoft does not distribute products keys as standalone products. If you see a listing on an auction site, online classified ad, or other online page advertising product keys for sale, it’s a good indication that the keys are likely stolen or counterfeit. If you were to purchase and use a stolen or counterfeit product key to activate Windows installed on your PC, the key may not work for activation, may already be in use on another PC, or it might be blocked from use later by Microsoft when the key is reported stolen.
