jay freedman say:
(Please don't shoot the messenger!)

There was a security patch in December 2012 that disabled PFM/PFB fonts by mistake, and a revised patch was released shortly afterward (http://www.thomasphinney.com/2012/12/windows-security-patch/). You can install the revised patch if you don't have it. If you already have the revised patch, or if installing it doesn't help, then read on...

In the Word section of Changes in Office 2013, the row titled "Unsupported Font Types" says:

"This change is due to a transition from the GDI rendering engine to the new D2D/DWrite rendering engine provided in Windows 7 and Windows 8.

Word will no longer support the following font types:

BitmapPrinter Adobe Type 1 CFF/Type 2 TTO .FON


We have switched our rendering engine to take advantage of the new engine that Windows is providing. This new engine doesn't support these font types (a choice made by Windows)."

I don't claim to understand all the variations of PostScript terminology, but it seems that "Adobe Type 1 CFF/Type 2" describes the way the fonts are defined, while PFM and PFB refer to the file formats. So I'll guess that not all PFM and PFB files are unsupported, but some -- and possibly all the ones you have -- are not supported.

I don't have any information about whether that support might be added later; but given that a change of that kind would affect all of Windows 8, it seems unlikely.

hansv mvp say:

Hiding a file doesn't mean much - anyone can set their Windows Explorer to show hidden files and folders.

You can set a password that will be required to open a document:

  • Open the document in Word.
  • Click the Office button, then Prepare > Encrypt Document.
  • Specify the password, then click OK.
  • Enter the same password again, then click OK.
  • Save the document.
  • Next time you open the document, you will have to provide the password. Do not forget it!
