rob koch say:

The Microsoft active security applications (MSE, Defender, Safety Scanner, etc.) have had 3 different package sizes which they may download for many years.

Which package is downloaded depends upon the currently installed definitions and engine as compared to the new package being downloaded and installed.  The designations for these were "delta" and "full" as I recall from the WL OneCare days, with the delta being the smallest at around half a megabyte or less and the full being the largest at an amount that begins growing from the first delta to around 50 MB (or larger) by the typical release of the new antimalware engines that usually occurs around the third Thursday of the month.

Back in the early days, the maximum definitions release was 3 within a 24 hour period, since there were 3 global centers where malware analysts developed, tested and released new packages daily.  Over time this increased to several locations, but then with the creation of "Cloud" protections which are automated, this has likely been reduced, since this requires fewer human analysts worldwide in order to perform the manual investigation of completely new classes or families of malware.

I explained all of this so you can see that the determination of which definitions package(s) you will receive is at least partially time dependent, since if your system hasn't been switched on recently enough that the smallest delta package will suffice, this may require that the larger full package be downloaded and installed instead.  And in the case where either new engines have released or the existing definitions package installation fails for some reason, the most recent full engine and definitions package may be required.

From your description and the fact that I just turned on a Windows 7 laptop that I rarely use, I can tell you that this downloaded a 44.5 MB definitions package, while I can also still find a few older definitions packages which range in size from a 469KB (small "daily" delta pack), to just under 2MB (medium sized full pack from a few days after last engine release) to the current 44.5MB, which may get reset if a new engine is released within a few days.

I suspect that whatever changes have occurred to this system since the OneCare days have been relatively minor, since the package sizes still seem similar, though I'm sure both the cloud and other changes to analysis and package delivery this has caused may have changed the package designations as well.

However, at least the older 32-bit Windows 7 installation I have still allows the ability to browse the C:/Windows/SoftwareDistribution/Download folder that contains the package files which Windows Update most recently downloaded and hasn't yet automatically removed.  These typically contain a clear Description in the file's (right-click) Properties, General tab such as Antimalware Definition Update to aid in their identification.

So this should help you determine whether all of this svchost activity is truly related to MSE definitions and what the true package size for these is, since there's always communications overhead and the daily Windows Update traffic related to downloading the Windows Update Catalog information that it uses to determine which updates are required for that particular system as well.
