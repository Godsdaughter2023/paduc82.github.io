petrula say:

I was finally able to install a clean UEFI Windows 7 Pro 64-bit to a GPT partition with my UEFI based motherboard. However, there are many "Got You" issues to overcome.

 

Before starting the Windows installation process from the Windows Installation DVD, your boot disk must already have a GPT format. Why? - because the Windows installation process cannot format a disk to GPT. You can use Windows Disk Management to format the disk to GPT. Next, while still in Windows Disk Management, do NOT create a Local partition for the Windows installation process. We will discuss why below.  

 

You need to use the Asus boot menu (F8) and select your boot menu with a UEFI prefix before the name. If you attempt to boot your media in Legacy mode, you cannot install to GPT disk.

 

In my case, I selected

UEFI: (FAT) HL-DT-ST BD-RE BH10LS30 (3035 MB)

 

During the installation process, the Windows screen for selecting which partition to install the OS to will show the following because I had already created a Local partition.

 

Disk 0, Partition 1, 100.0 MB System i.e. EFI System Partition

Disk 1, Partition 1, 931.3 MB Primary

 

I selected the Primary partition. The following error occurred:

 

"Windows cannot be installed to this disk. The selected disk is of the GPT partition style."

 

This seemed strange so I deleted both the Disk 0 and Disk 1 partitions.

Now there are no partitions so everything is unallocated space.

 

I used the "New" icon resulting in

Disk 0, Partition 1, 100.0 MB System i.e. EFI System Partition

Disk 1, Partition 1, 128.0 MB MSR (Reserved)

Disk 2, Partition 1, 931.3 MB Primary

 

The installation now proceeded as expected.

 

Hence, one should use a GPT disk with no partitions and unallocated space before starting the installation process.
