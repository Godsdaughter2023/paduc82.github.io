a. user say:

This behaviour is reported from time to time and I dont think anyone has ever found "the answer". I've just tried another small batch of experiments with Office 2007 in an attempt to discover /any/ circumstances in which messages will not reach the Outbox. I doubt if any of this is applicable to your situation and it's a lot more "techie-oriented" than I would like, but perhaps it will spark further thoughts.

Before detailing that, much as I dislike messing around with mail profiles on an otherwise stable system, what I would probably do next is create a new profile containing the account you want to send by /right now/, set that to be the default, close all software that uses email (Outlook, Word, etc.), or even restart your system, and see if that works. If so, then I'd brace myself and go through the business of deleting and recreating my profile.

But here is the info. from the experiments...

1. I do not actually think the hotfix described in the following link is relevant to this problem, but it is worth mentioning:

http://support.microsoft.com/kb/980681

2. Word appears to use a different method when sending as HTML than it does when sending as plain text or attachments. Usually, HTML works more easily because this method appears to bypass the security checks on the address book. However, it seems that Word does not respond to or report errors with the HTML method as well as with plain text/attachment. i.e., at least some such errors are not reported at all. For example, if you have an outlook profile that has no "email" defined (i.e. no sender's address etc.) then when you merge to HTML, the messages just disappear wiithout trace. I haven't been able to find them anywhere else in the MAPI messages stores using the MFCMAPI tool (see http://mfcmapi.codeplex.com/ ). OK, so you have a profile with "emails" defined, but what I'm driving at is that

  a. it doesn't take much to make send to HTML fail. It could be a probem with your profile (but I do not know exactly what)

  b. therefore, when experimenting to find out what might be going wrong, it's probably better to send as plain text, because Word is more likely to report an error that can give people more information about what is going on.

3. Other than the disappearing HTML messages, the only scenario I have been able to create where plain text messages "disappear" is as follows:

  a. I have 2 email profiles - call them "profile1" and "profile2." Let's say profile1 is set to be the default.

  b. I start Outlook and it logs onto the profile1 profile. Although you don't see this happening, this creates a "shared session". Here, we'll refer to it as session A 

  c. I start something else that uses the maii subsystem, and log on to mail in such a way that it does not use an existing shared session, but that the new session is also a shared session, which we'll call session B.

  d. I start Word, and open/create my merge to email main document, then proceed to start the mailing. First, I see the dialog box that sets up the subject line, the mailing type etc. I set the mailing type to Plain text. Now if there was no default email profile, and Word was the only program running, the next thing I would see is the email Profile selection dialog box. However, we now actually have 2 shared sessions, and Word picks one of them. At this point I do not know what chooses which one to use. Word asks the usual security questions about the address book, and then sends the emails. But they aren't there in Outlook, which suggests that Word might have connected to Session B, i.e. using profile 2. Sure enough, if I close Word, close Outlook, then re-open Outlook so it logs into profile2, I see the messages in the Outbox.

Obviously, that's a pretty unlikely scenario, and AFAICS it could not happen unless you have two profiles. Although you say you have "I have both an Exchange and PoP account set up in a single Outlook profile," and I would normally assume that means you only have one, it's possible that you meant "I have more than one profile, but my default profile has two accounts"

The other thing I haven't got around to yet is whether, in a multi-account profile, it might make a difference which account is set up to be the default account for sending. COuld be worth setting the other one to be the default if you haven't already done that.

 

Peter Jamieson

john p spencer say:

No it is not possible with a message box. 

You would need to create your own form to be able to have that degree of control.  To make it behave similar to the Msgbox you would have to open the form in dialog mode

Docmd.OpenForm "MyMessageForm",,,,,acDialog, SetTimeFormat(DateDiff("s", PreviousTime, Me.StartTime))

The last argument to OpenForm is OpenArgs and is a string.  You can use the string to pass in the text you want and based on the string you could control the color of the text.
