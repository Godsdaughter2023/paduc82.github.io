j_hall_in say:

Based on my experience adding our business Exchange account to my surface, my guess is it is because of a certificate issue.  If the mail client doesn't "trust" the certificate, it will not add the account.

 

If it is an SBS server, usually the Server Self-Signed certificate is located in C:\UserShares\Public\Public Downloads.  Just place a copy in the "Trusted Root Certificates" and you should be good to go.

 

Unless you manually copy and add a self-signed certificate to your surface, it will not allow you to access mail from it.

 

Hope this helps. It resolved my issue

hairyfool say:

How big is the partition and does your system have an UEFI BIOS, how big is the whole drive itself?

This BIOS type will by default create essentially two small partitions one of which is the EFI System Partition (ESP)

Reinstall sometimes does not clean up these partitions or re-use them. Ideally during the reinstall if possible make sure all old partitions are removed and let the reinstall start with a completely clean drive.

If the space is small then it probably is a leftover from the original installation, not worth trying to reclaim the space and can be ignored. The easiest way to do that is to use Disk Manager to remove the drive letter, if it does not have one it won't be listed in Explorer.

Information on UEFI disk structure is here:

http://technet.microsoft.com/en-gb/library/dd744301(v=ws.10).aspx
