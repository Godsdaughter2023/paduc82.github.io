davidf2 say:

Grid or guide lines are designed to help layout your page. They are not intended to print. If you want a grid to print then draw lines along the guide lines.

Go to Insert > Shapes > or Home > Shapes and choose the line. Click where you want to start drawing the line and if you hold down the shift key you will find it easier to get a perfectly horizontal or vertical line. Then right click the line > format autoshape and you can choose the color and weight of the line and a dashed version if you prefer. Then once you get your line styled the way you want it go to Page design > Layout group and check Align to Guides and Objects. Now you can right click, drag the line to the next guidline and it will 'snap to' that guidline > Copy Here and create your grid quickly.

If you continue to have problems with the Adobe PDF then you might want to try the built-in PDF tool in Publisher. Go to File > Export or Save and Send > Create PDF/XPS doc and when the click the options button to the right of Optimize for to choose the appropriate pdf.

Outgoing server: smtp.gmail.com

Check the 'Use SSL to connect' for both the incoming server and the outgoing server.

Finish the procedure.

     Access 2007 - Output Format: Excel 97 - Excel 2003 Workbook (*.xls)

    1, Summary, MicrosoftExcelBiff8(*.xls), C:\Summary.xls, Yes, , 0

kathy s - microsoft support say:
I suspect that your macros that show Biff8 were either originally created in Access 2003, or were modified/saved in 2003 after being created in 2007. You *could* use Access 2003 to open any macros that were created in Access 2007 and resave them so that they would have the Biff8 format.

Also, it's not very clear in Access Help, but you can use either a Table or Query name for the "Table Name" argument when using TransferSpreadsheet. I just tested this to confirm that it works with a query.

From Access help:
You can export the data in Access select queries to spreadsheets. Access exports the result set of the query, treating it just like a table.

HTH!
Kathy

kathy s - microsoft support say:
I tested this scenario and I think I was able to repro the behaviour you are experiencing. I would first encounter the error:

"The format in which you are attempting to output the current object is not available.
Either you are attempting to output the current object to a format that is not valid for its object type, or the formats that enable you to output data as a Microsoft Excel, rich-text format, MS-DOS text, or HTML file are missing from the Windows Registry. Run Setup to reinstall Microsoft Office Access or, if you’re familiar with the settings in the Registry, try to correct them yourself. For more information on the Registry, click Help."

When I clicked OK, I would receive an Action Failed Dialog:

1, Query1, Excel97-Excel2003Workbook(*.xls), c:\test2.xls, Yes, , 0


I have found a couple of ways that you could work around the issue:

1. Instead of using the output format Excel 97 - Excel 2003 Workbook (*.xls) for Access 2007, open your macro in Access 2007 and change it to MicrosoftExcelBiff8(*.xls). Once I made this change, I was then able to go back and forth exporting in both Access 2003 and 2007 without any errors.

2. Use the TransferSpreadsheet action instead of the OutputTo action. This worked successfully switching between programs.


I did some additional digging around and found that this issue has already been reported to the Access developers.

HTH,
Kathy
