Mark ~ Welcome to the Communities. I've seen this bug as well, but currently can't reproduce it. Whilst awaiting a possible response from the OneNote for Mac team, if you're up for some experimentation (at your own risk!), the following may re-index notebooks for searching...

1. Close a notebook on your Mac via OneNote's File menu. This removes the synced disk cache of that notebook from the Mac, whilst the master copy on OneDrive remains unaffected. Note: Choose a small notebook to start with, because when you re-open the notebook it will be re-synced down from OneDrive to the Mac.

2. Re-open that closed notebook on your Mac via OneNote's File menu. ...Since OneNote for Mac doesn't know whether anything in that notebook was changed via OneNote Online, I would expect it to re-index the notebook for searching.

There are some basic instructions regarding closing notebooks in the Help menu > OneNote Help > enter "Close" in the search bar. You'll see in one of the search results there that closing and opening notebooks is suggested as a way to reset notebooks that can’t sync.

rob rolnick say:

Hi Kelsey,

The Office Web Apps do very little file management.  In fact, I believe that the only time they create or delete files is in the following cases:

  • The Office Web Apps will convert legacy Office files (doc, ppt, etc.) to Office Open XML files (docx, pptx, etc) when going from the viewer to the editor.  The original file will be left untouched, but a new file will be created (with the same name, but different extension.)
  • Microsoft OneNote notebooks put every section are essentially folders on disk, with every seciton being a .one file.  If you delete a section in the OneNote Web App, I think that the corresponding .one file might get removed from disk.  (Although I'd have to verify that.)
As such, you might want to ask on the  SkyDrive forum if they have any ideas, as they do the bulk of the file management and hosting.  That said, here is one thing you can try:
  • Go to  http://office.live.com
  • Sign in.
  • At the top center of the window hover over "Office" to bring up a flyout menu.
  • Click on "Your documents" in this flyout menu.
  • Find the folder containing the file(s) you are looking for.
  • Hover over the name of the file to see the full name with extension.  Are they all the same?  Are any different (e.g. name.doc instead of name.docx.)
  • When you find the file(s) your curious about hover over them and click on "More" this should bring upa nother flyout menu.
  • In this flyout menu click "properties".
  • You should not be on the file properties page.  From here you can see the filesize of your file, and access its version history.  You can also quickly compare it to other files in the same directory through the file picker widget on the right (or use your keyboard arrow keys.)
