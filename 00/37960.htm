Activating Windows 7 manually is very simple. First of all, you need to make sure you have a working Internet connection. Once that is confirmed, go on to the next step.

Click Start, type Activate Windows

Hit Enter on your Keyboard

Activation in progress

Other work around – Extend your Activation Grace Period

Another work around for Activation is to extend Activation Grace Period for another 30 days. You can do this up to 4 times (3 times for volume licenses) for a total of 180 days extension. After this grace period, you will need to make sure Windows is activated.

If you are implementing KB307545 in an effort to fix  problems like these:

Windows XP could not start because the following file is missing or corrupt: \WINDOWS\SYSTEM32\CONFIG\SYSTEM

Windows XP could not start because the following file is missing or corrupt: \WINDOWS\SYSTEM32\CONFIG\SOFTWARE

KB307545 is never, never, never the fist thing to do when you see messages like that.

The first thing to do is always, always, always run a chkdsk with error correction as the problem is usually that XP just can't find the files due to some corruption in the NT File System (NTFS). 

The file system could get corrupted from a power interruption, plug pulling, using the power button, aborted restart or ungrateful shutdown.  Have any of those events occurred?

Until you know that your file system is intact, running any other commands to make directories, copy files, etc. doesn't make sense and is very likely to fail and complain.  If things in KB307545 do not go perfectly, you may find yourself in bigger trouble and it sounds like you are off to a bad start already :-).

If the CD you booted on is some kind of recovery CD from the manufacturer and not a genuine bootable XP installation CD, that could also explain why what you are trying to do might not work (the two are not the same).  But, as usual, we know next to nothing about your system.

What is your system make and model?

Does your system have IDE drives or SATA drives?

Describe your current antivirus and anti malware situation:  Microsoft Security Essentials (MSE), McAfee, Symantec, Norton, Spybot, AVG, Avira!,  Panda, Trend Micro, CA, Defender, ZoneAlarm, PC Tools, Comodo, etc.

If your system has SATA drives, none of those Recovery Console commands are going to work for you unless you do some fiddling with your system BIOS and that is too much trouble.

A good sign that the system has SATA drives will be when you boot into the RC and you end up in the C:\> folder instead of the  C:\WINDOWS>  folder where you should be.  And, you will see things like 'enumeration' errors for even simple commands like 'dir' or error messages like "Access denied".  You will not get much done like that.

KB307545 has it's place in the world - like if you have modified your registry without making a backup and messed things up so bad that your system will not boot (if you really corrupted it) - then KB307545 might be a good idea (but there are better ideas), but KB307545 is not the first thing to do for the aforementioned errors.  It is also good practice but you better have a plan for what you are going to do if it doesn't go smoothly.

If you did not accidentally corrupt your registry what happened prior to this incident?

People often start with KB307545 since that is the first thing to come up in a Google or Bing search of the error or when the Microsoft engaged Support Engineer "experts" will always start with that (and that is too bad), but I wold never start with that and have never used it for this problem and will never suggest it.

Here is where you should start.

If you are not sure what you have for bootable media, from a working system, make yourself a Hiren's Boot CD, boot on that into the Mini XP mode and run the chkdsk from there.

Make yourself a Hiren's Boot CD which you can download from here:

http://www.hirensbootcd.org/download/

The ZIP file is large, so the download will probably take a little while to complete bit it is worth it.  Then unzip the download to extract the Hirens.BootCD.ISO file that will be used to create your new bootable CD.

Creating a bootable CD from a .ISO file is not the same as just copying the .ISO file to a blank CD.  You have to use software that understands how to burn a .ISO file to a CD to create a bootable CD.

In the Hiren's ZIP file are the BurnToCD.cmd file that you can double click to launch it, but I have never used it and prefer to use ImgBurn.

If you need a free and easy CD burning software package, here is a popular free program:

http://forum.imgburn.com/index.php?showtopic=61

These adjustments are  made before Windows tries to load.  If you miss it, you will have to reboot the system again.

When booting on the Hiren's CD you will see a menu of options.  Choose the Mini XP option.  Then it will appear that Windows is being loaded and you will be presented with a desktop that has the look and feel of the Windows Explorer interface you are already used to using.

cmd

chkdsk  c:  /r
