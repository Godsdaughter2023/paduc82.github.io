andreas killer say:
That is not possible in one step, but in two:

You must have at min. 2 values, e.g. in your sample write CN1002 into A3.
Then select A2:A3 and use the fill handle and drag down till e.g. A10. (As you see the format is copied)
After releasing the left mouse button you should see an icon appearing, on the bottom right of the last cell.
Hover the mouse over the "Auto Fill Options" icon, left click and choose "Fill Without Formatting".

About MSDN or TechNet Product Keys

They are genuine Microsoft product keys for retail licenses, but not for Retail sale. They are intended for a particular product channel either the Microsoft Software Developer Network (MSDN) or TechNet for IT Professionals who pay a subscription fee. The main purpose is for evaluation purposes. It is for one person only to use and no one else. Unfortunately, regardless of the licensing terms, persons still abuse the program, either giveaway product keys or resell it on auction sites.

Expr2: IIf(IsNull([Facsimile])," ",[AreaCode] &  [Facsimile])

Facsimile is blank or null

Result: is Area Code 203- 

why doesn't this work

If the result of the expression is "203-", then [Facsimile] is not Null.  You said, "Facsimile is blank or null".  "Null" is ruled out be the results.  Therefore [Facsimile] is either a zero-length string, or else it is made up only of one or more spaces (or possibly other characters that display as blanks).

The following expression will return a space if [Facsimile] is Null or spaces:

   IIf(Len(Trim([Facsimile] & "")) = 0, " ", [AreaCode] &  [Facsimile])
