Confirmed resolution to the MAPI protocolhost not responding issue - Microsoft Community

squall_leonhart69r say:

Confirmed resolution to the MAPI protocolhost not responding issue (10023) after recent windows updates.

Users Experiencing this issue will notice under control panel \ indexing options that where it should say Microsoft Outlook, they see mapi16://<UserUIDString> where the UID String reflects the users Profile as listed in HKEY_USERS\S-1-5-21-<additional numbers>

Other solutions that recommend deleting specific keys and making sure others exist may not resolve this problem (Probably won't) and the solution involving the system path's variable is not required.

A full reinstall would probably do the job, but why do something that takes so much longer when there is a quicker fix.

Solution:
Log out any other users and start Outlook on an Admin enabled account,

Navigate through the File menu to the options screen and select the search tab in the side panel.

Click Indexing Options.

Now your experience can differ here,  On one machine, this gui listed the item as Microsoft Outlook (Windows 7) but on another it listed the mapi16://<UserUIDString> (Windows 10). Regardless, the fix is to deselect this item and hit ok, then restart Outlook and then wait.

Search will soon try and interact with the mapi protocol triggering the MSI installer to pop up and service the office and office components.

The broken MAPI item will now be listed as Microsoft Outlook with the correct icon in both the Outlook and Control panel Indexer Options and this error will no longer occur.

Reconnect to .ost file after deleting exchange email account from outlook 2003 so that I can recover offline calendar/email items

Hi Team,

brian tillman [outlook mvp 2007-2019] say:
Use a System Restore Point from just prior to when you removed the Exchange account.  That will restore your mail profile as it was before you deleted the account.  Next, add a PST to Outlook with File>New>Outlook Data File and copy each of the folders in the Exchange folder tree to the root of that new PST.  That will save all the data in the OST.  You can then remove the Exchange account safely and add the type of account you want.
Brian Tillman [MVP-Outlook]
