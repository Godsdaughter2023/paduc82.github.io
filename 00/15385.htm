scottgem say:
There are no records in subforms, there are only records in tables. It sounds like you have Referential Integrity set on the joins between tables without having Cascade delete turned on.

This will prevent you from deleting a record in the parent where there are related records in child. So what you need to do is turn Cascade Delete on for those tables. Then deleting a record in the parent will remove the related records in the children. 

If that is not the situation, then you will need to code your own delete button which will generate and run DELETE queries on all child tables filtered for the FK that links it to the parent.
