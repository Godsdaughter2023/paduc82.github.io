suzanne s. barnhill say:
I'm not sure what has changed. The Print button in earlier versions of Word represents the FilePrintDefault command, meaning that it will print using the default Print settings, which include using the Windows default (or currently active, i.e., MRU) printer, a single copy of the entire document (i.e., not specific pages or sections or a selection). If tracked changes are visible onscreen (and in Print Preview), they will print; if the view is set to Final (rather than Final Showing Markup), then the document will print without markup (this can be verified by opening the Print dialog and seeing that either "Document" or "Document with markup" is selected in these given instances). In no case does Word retain any previous settings of the Print dialog (except the active printer), though Print Options (such as "Reverse print order") are honored.
 
If the QuickPrint button does something other than run FilePrintDefault, then this is the first I've heard of it, and I do find it dismaying.
