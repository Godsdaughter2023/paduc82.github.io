Sub FillRandomNumbers()

jasonusa say:

OK, I think I have found all the information I need.  I eventually found the following blog:

https://blogs.office.com/2012/08/21/updating-layout-in-word-2013-while-maintaining-compatibility/

This was written by the "program manager responsible for layout in Word".  It explains that with Word 2013 a new way of positioning tables on a page was introduced (and I couldn't agree more that this new method of positioning the border rather than the content is more logical) and that Compatibility Mode ensures backwards compatibility.

If you open a 2010 .docx file into 2013/2016 and click Save, it's resaved in the 2010 .docx format and no formatting is altered.

If you click Save As however then you're presented with the following notice:

Click OK and it's converted to the newer 2013/2016 file format and formatting may be altered; click Cancel and you can elect to save the file in the existing file format with no formatting being altered.

Now that I know this I can see that what I've been facing is the effect of users clicking the "Do not ask me again" checkbox in the above message box.  Upon our users' demand, our software retrieves a pre-existing Word document, saves it to a default location and opens it for viewing; some users are then saving it to their own location using the Save As function and in the blink of an eye the formatting is altered.

I suppose now that the user has placed a checkmark in this checkbox our only course of action is to reach out and provide instruction on how to effectively uncheck this checkbox so they're once again prompted when using Save As such that they can save the file in the original format with original formatting preserved.

a. user say:

Hi Barbara,

Thank you for posting your query in Microsoft Office Community. 

 

We understand how you feel about this issue, let me assist you further.

 

Let’s start Word in Safe Mode by referring to below steps and verify the results.

* Hold Windows key + R.
* Copy and paste, or type the following command in the Open box, and then press Enter

Winword /safe

Note: There is a space between Winword and '/'.

If Winword works as expected in safe mode, disable the add-ins by following steps mentioned below and determine the problem causing add-in:

a) Click on File menu, click Options, Add-ins, Go button besides Manage: Com-in Add.
b) Check if there are add-ins listed, clear the checkbox.
c) Close the Office program and restart it.
d) Disable each add-in one at a time, restart the Word, and repeat the above procedure. If the issue reappears again, it will help you to determine which add-in is causing this issue.

 

You may also refer to the below Microsoft article and check if it helps.

https://support.microsoft.com/en-us/kb/921541

 

Hope it helps. Try the suggestions and let us know the result at your earliest convenience.
