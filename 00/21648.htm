john korchok say:

This is possible, but there are limitations. The requirements are:

  • You have to have a document that will be protected for forms.
  • This document will have at least 2 sections separated by a Continuous Section Break. One section will be protected and one will not.
  • You need to add a frame and anchor that frame to the protected section.
  • You need to put a Text Form Field in the frame.

Once the document has protection applied, you will be able to type text in the frame, but you will not be able to format the text. The frame will not move.

  1. In Word, choose Word>Preference>View and check All. You will also want to check Object Anchors in the same dialog, and you need to make the Developer tab visible (though you've probably already done so, if you found frames). Now you can see the hidden formatting characters. 
  2. Insert a Continuous Section Break in your document using Page Layout>Breaks. When I'm creating these kind of forms, I usually put it at the top of the document before any other text. The section at the top only needs to contain one paragraph mark.
  3. Choose Review>Protect Document.
  4. Check Protect document for: and choose Forms.
  5. Click on the Sections button. Keep Section 1 checked, but uncheck Section 2. Click on OK, do not enter a password.
  6. The above action sets which section will be protected. Click on Review>Protect Document again to unprotect the document so we can finish formatting.
  7. Insert a frame from the Developer tab.
  8. Drag the anchor for the frame to the very top of the page, so it sits to the left of the section that will be protected. If you prefer, you can lock the anchor, so that it doesn't move when you position the frame.
  9. Click inside the frame and insert a Text Form Field from the Developer tab. Format this text field as you want it to appear.
  10. Protect the document once again for forms, then test it. You should be able to type in the frame, but not be able to move it. The text area below the section break will be mostly editable with some restrictions.

bob jones aka: cybertaz say:

One of the differences I referred to is that Text Wrapping is the default for a Frame. They are throwbacks to the days before Text Boxes, graphic objects & Text Wrapping evolved to their current state.

The In Line with Text property cannot be applied to a Frame. However, there are only 2 settings to control how text flows relative to a Frame: None, which means that text will flow above/below it comparable to Top/Bottom Text Wrapping, & Around, which means that text flows around it comparable to Square Text Wrapping. IOW, you can think of a Frame as being similar to a single Table cell. In neither case will the Frame move with the text, though.

Another point to re-emphasize is that no object - Frames, Text Boxes, Pictures, Shapes, etc. - can be prevented from being dragged. The only control is whether they will move with the text.

For objects that can have Text Wrapping applied you can either;

  • Select the object & go to Arrange on the Shape Format contextual tab of the Ribbon, or
  • Right-Click the shape & go to Wrap Text in the contextual menu
