frederik long say: It is rare that one sees a batch file of almost 300 lines code. If is even more rare that one sees a batch file of 300 lines of unstructured code that almost works, in spite of 49 "goto" statements that cause the program to jump all over the place. I hope you won't take offense at me suggesting that you're making life unnecessarily tough for yourself by not modularizing the code. If this was my own code then I would rewrite it completely as per the sample below, if only in order to make it maintainable. In its current form you can modify it with confidence because you know it inside out. If you look at it again one year from now you will be lost. I guarantee it.

About your actual problem: Without knowing the contents of your various .set files, it is impossible to say why your code fails. Here are a couple of traps I notice:
  • You do not enclose your file names in double quotes. If one of them contains a space then your batch file will fail.
  • You do not fully specify the various batch files and executables with drive letter and path name but you do change directory on some occasions. It is likely that you sometimes find yourself in the wrong directory. Adding debugging code of the following form would reveal the problem:
    echo Current directory=%cd%
    pause
Note also that this forum addresses Windows 7 security issues. Your post is about batch file programming, not about security. The following forum would be a far more appropriate place to ask such questions: http://social.technet.microsoft.com/Forums/en-us/ITCG/threads.

- Each module is invoked with a Call statement at the head of the batch file.
- Each module performs a well-defined task.
- At the head of each module there is a description of its purpose.
- No module contains more lines than will fit onto one screen.
- With the exception of "goto :eof", no goto statements are used.
- Conditional code execution is handled by if-then-else statements.
@echo off
call :SetVars
call :CheckParms
if  /i  Exit=true goto :eof
call :Compare
etc.
goto :eof

------------------------------------
Set the operating parameters
------------------------------------
:SetVars
set dir=c:\nm72
set f=gfortran
set op=-O3 -ffast-math
set lfile=%dir%\license\nonmem.lic
set ndir=%dir%\nm
set mpilib=fmpich2
if "%f%"=="df" set mpilib=fmpich2s
set compile_type=ifort
if "%f%"=="gfortran"  set compile_type=gf
if "%f%"=="g95" set compile_type=gf
if  "%compile_type%"=="gf" (
    set obj=o
    set comp=-c
    set n=%ndir%\nonmem_recoded.%obj%
) else (
   set obj=obj
   set comp=/c
   set n=%ndir%\nonmem_recoded.%obj% %ndir%\nonmem.lib
)
rem added 4/2008
set u=
goto :eof

--------------------------------
Check the command line parameters
rem added 4/2008
--------------------------------
:CheckParms
set Exit=false
if "%2" == ""  (
   echo Usage:
   echo nmfe72 infile outfile [-background] [-PARAFILE=file.pnm] [-licfile=filename]
   echo [-prsame] [-prdefault] [-prcompile] [-trskip]
   echo [-rundir=dirname]
   echo [-runpdir=dirname] [-nmexec=filename] [-xmloff]
   echo "Default:"
   echo nmfe72 infile outfile
   echo   -licfile=%lfile%
   echo   -prsame \(if possible\; else -prcompile\)
   echo   -rundir=%cd%
   echo   -runpdir=temp_dir -nmexec=nonmem
   set Exit=true
   goto :eof
)
