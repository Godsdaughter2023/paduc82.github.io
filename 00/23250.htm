Undo undoes one letter at a time, instead of the whole word - Microsoft Community

joachimbo say:

Undo undoes one letter at a time, instead of the whole word

For some reason, Word (2016, by subscription) undoes one letter at a time, rather than the whole word. This happens on my Surface Pro 3, which has Word 2016. I've never had this problem with my other computer (which still uses Word 2010).
jane s... say:

This weird Undo issue goes back many years and occurs across many different Windows and Office versions. Microsoft still hasn't fixed it, though they've certainly been aware of it for a long time.

On my system, the issue was triggered by the presence of the page number field in the footer. As long as that was there, the Undo button (or ctrl-z) would only remove one letter at a time.  If I deleted the page number text from the footer, Undo suddenly worked correctly and would delete a whole word/phrase at a time. If I put the page number text back, Undo started playing up again. I tried many different solutions with no success. Finally by my own trial and error, I found a reliable workaround. 

(Note: My preferred page number format for my reports is usually "Page X of Y", though the issue occurs with any kind of page numbering text.) 

I eventually worked around the issue like this - seems to be successful so far:

First I removed the original page number text (which was generated via Insert toolbar -> Page Number).

I then re-created the page number fields manually, like this:

1) In the footer, type "Page " in normal text (without quotes)

2) On the Insert toolbar, click the Quick Parts icon -> Field

3) Select the Page field from the list, and click OK. The page number will now appear, e.g. "Page 4".

If you want the "of Y" bit as well, continue like this:

4) Type " of " in normal text (without quotes)

5) On the Insert toolbar, click the Quick Parts icon -> Field

6) Select the SectionPages field and click OK

You should now see a new "Page X of Y" in your footer, e.g. "Page 4 of 16". This one should auto-update just like the original one, but should not interfere with the Undo functionality.

This worked for me, but might not work for you. Hopefully this does help some of the others out there who are going through the same issue. And hopefully Microsoft eventually fixes this very simple bug!

quietman7 say:
In Windows 10,

When Windows Defender identifies malware which it cannot remove, it will recommend that Windows Defender Offline be run and on the next reboot, Windows Defender Offline will automatically scan and remove the malware, then reboot into Windows 10. This removes the need to interact with the Windows Defender Offline product manually but that option is available when you want to run an offline scan at any time. When Windows Defender Offline runs automatically it performs a Quick Scan by default which typically takes about 15 minutes to run and then your computer will restart normally.

a. user say:

Hi Imtiaz,

I understand that you are receiving error code :0x80070102 while updating Windows Defender and I will be happy to answer your query. Let me ask you;

  • Are you using any 3rd party security software?
  • Are you able to install any other Windows update?
  • Are you able to install any other file online using web browser?

There could be several reasons for this cause and to diagnose the issue, let's try the following methods and check.

Method 1: Stop, modify and start the Windows Update service

Step 1: Stop Windows Update service.

You must be logged on as an administrator to perform these steps.

  1. Open Administrative Tools from Control Panel. Open Control Panel, type administrative tools in the search box, and then click Administrative Tools.
  2. Double-click Services.
    Note If you're prompted for an administrator password or confirmation, type the password or provide confirmation.
  3. Click the Name column heading to sort the names in reverse order.
  4. Find the Windows Update service, right-click it, and then click Stop. If you are running Windows XP, find the Automatic Updates service, right-click it, and then click Stop.
Step 2: Delete temporary update files in the SoftwareDistribution folder
  1. Open File Explorer by pressing Windows key + E.
  2. Double-click the local hard disk where Windows is installed, typically drive C.
  3. Double-click the Windows folder, and then double-click the SoftwareDistribution folder.
  4. Double-click the DataStore folder to open it, and then delete all files and folders in the folder.
    Note If you're prompted for an administrator password or confirmation, type the password or provide confirmation.
  5. Click the Back button. In the SoftwareDistribution folder, double-click the Download folder to open it, and then delete all files in the folder.
    Note: If you're prompted for an administrator password or confirmation, type the password or provide confirmation.
Step 3: Restart the Windows Update service
You must be logged on as an administrator to perform these steps.
  1. Open Administrative Tools from Control Panel. Open Control Panel, type administrative tools in the search box, and then click Administrative Tools.
  2. Double-click Services.
    Note If you're prompted for an administrator password or confirmation, type the password or provide confirmation.
  3. Click the Name column heading to sort the names in reverse order.
  4. Find the Windows Update service, right-click it, and then click Start. If you are running Windows XP, find the Automatic Updates service, right-click it, and then click Start.
  5. Close the Services window and the Administrative Tools window.

Method 2: If you are using a 3rd party antivirus application on your computer, it disables the Windows defender. Disable third-party Security Software and check.

 

a. On your Computer screen Task Bar next to System Clock.  

b. You will see the third-party Security Software Icon.  

c. Click on the third-party Security Software Icon, you will see a pop up.  

d. In that pop and select the option to disable, click on it.  

e. Now the third-party Security Software will be disabled.  

Important Note: Antivirus software can help protect your computer against viruses and other security threats. In most cases, you shouldn't disable your antivirus software. If you have to temporarily disable it to install other software, you should re-enable it as soon as you're done. If you're connected to the Internet or a network while your antivirus software is disabled, your computer is vulnerable to attacks.
