how to point Linked Table Manager to tables located on OneDrive - Microsoft Community

dmkkg say:

how to point Linked Table Manager to tables located on OneDrive

I would like to connect front end of a database on my computer to tables located on a friend's OneDrive.  I have full access to the One Drive but dont know how to point to it in the Linked Table Manager.

Is it possible to do this?

albert d. kallal say:

Unfortunately, Access does not support linking to a back end OVER the internet. So the “short” version is this is not possible.

Some workarounds are to use hosted SQL server or an office 365 plan that lets you “migrate” the table data to some sort of “server” based system that supports individual tables.

As to why? (long version)?
The reason is that Access “requires” the windows networking system to read/write the data INSIDE of the file. When you use Excel or Word, the WHOLE file is loaded from the OneDrive on to your computer. You now edit, and then when done re-save the WHOLE FILE back up to the OneDrive.

With Access, this would mean you

   Download the whole file local,
   update the file,
   and then re-upload the file.

Of course just like word or Excel, you are thus overwriting the WHOLE FILE and if someone in the meantime made changes, you would overwrite the whole file and thus their “individual” changes to a single row of data.

You never can read/write “parts” of a file on OneDrive. You thus need windows networking to do this.

So keep in mind that Access is different then Excel or word that says “give me the whole file and let me edit it local”. While Access is file based, the front end “requires” the ability to not download and open the WHOLE file. This “ability” to dig inside the file and grab ONE ROW of data means Access can open a HUGE file, and pull one row of data “instantly”.

So Access for reasons of high performance needs a system that lets you grab “tiny” parts of the data file. A database system is NOT whole file based like Word or Excel etc.

So systems like DropBox or OneDrive don’t work with linked tables.

You could download the whole back end to your local drive, and have your front end linked to the copy now residing on your local drive. When done editing, you have to re-save (re-upload) the back end to the cloud drive. However just like with word or Excel, you would overwrite changes if someone in the meantime did this download/edit/upload sequence.

So you can use a low cost office 365 plan or SQL Azure,  but linked tables to these cloud “drive” systems are not compatible with Access and linked tables.

This solution worked for me.  Thank you.  Here are some (slightly) more detailed instructions:

1. Open a Finder window and go to the Applications folder

2. Hold down the option button on your keyboard and click on the Microsoft Outlook icon and pick "Show Package Contents" from the pop-up menu  

3. Go to the Contents/SharedSupport folder and open the "Outlook Profile Manager" app

4. Delete the Main Profile: Select it once it shows up and then click the minus sign

5. Start the Outlook app.  Your mail will be gone, but the app won't crash.

6. Click Preferences in the Outlook menu

7. Click the Exchange or Other icon to add back your account

8. Enter your email address and password to add your account

shyamal debnath - msft say:
Hi Matt, 

There is indeed no direct way to set up a rule for a shared mailbox when it only has been linked to your own Exchange mailbox account using Outlook desktop client.

However, you can set up Inbox rules by using Outlook Web App. When you have been granted Full Access permissions, then you can log on with your own username and password and click on your own name in the top-right corner to get to the option of opening another mailbox.

Here is steps for your reference:
• Sign in to your account in Outlook Web App.
• On the Outlook Web App nav bar, click on your name. A list appears.
• Click Open another mailbox.
• Type the email address of the other mailbox that you want to open. Another Outlook Web App session opens in a separate window, allowing access to the other mailbox.

For your reference: Open and use a shared mailbox in Outlook on the web for business

Once logged in, you can create a message rule in the following way:
• Select Settings gear icon > Options.
• In Options, select Organize email > Inbox rules. Or, in Options, under MAIL > Automatic processing, select Inbox rules.

For your reference: Inbox rules in Outlook

Your time and understanding will be highly appreciated.

Best regards,
Shyamal
