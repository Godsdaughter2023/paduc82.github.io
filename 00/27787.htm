steven _fyiguy_ hughes say:

Jeroen,

 

What version of Windows Mobile are you using and what are you using to authenticate the WPA/TKIP certificate?

 

I know there is an issue in authenticating against Radius Servers using EAP-TLS with the same issue of getting a username / domain prompt  you presented others have also had problems in connecting WM devices to WPA-PSK APs due to non-support of the standard. The WM device 802.1x client doesn't associate a certificate to a SSID connection until it connects for the first time.This means that even though you selected a certificate in the client EAP-TLS setup before connecting, the client doesn't use this certificate for authentication until after it establishes a connection. Hence the username/domain prompt you are getting is the mechanism for creating this association. If you have multiple personal certificates on your device, this is when the right cert is used for the right SSID for 802.1x authentication.

 

 A work around I had found was using a registry editor on the WM device and manually changing the reg key so that is set to the same value as when the association is created:

 

HKCU\Comm\EAPOL\Config\<SSID>\Identity (REG_SZ) - <Domain\Username>

 

Also what version of WPA/TKIP are you using WPA1 or WPA2? From what I know WPA2-TKIP isn't supported. in versions previous to WM5 MSFP.

 

Another thing to check is if you have the latest firmware for your AP and make sure you entered all the information correctly to match your wireless router’s settings. I think you did, but this may help others:

For 64-bit or 128-bit WEP:
1. In ‘Authentication’, select either ‘Open’ or ‘Shared’ to match your router’s setting.
2. In ‘Data Encrytion’, select WEP.
3. Uncheck ‘The key is automatically provided’.
4. In ‘Network Key’, type in your WEP key. It must be HEX key (ASCII keys will not work). For 64-bit WEP, it is 10-characters long. For 128-bit WEP, it is 26-characters long.
5. Match key index to your router’s setting.

For using WPA Pre Shared Key:
1. In ‘Authentication’, select WPA PSK.
2. In ‘Data Encryption, select TKIP.
3. In ‘Network Key’, type your WPA key or pass-phrase of your router.

 

If you can give some more information it can help me to help you.

 

Hope this helps.

hao zhong say:


After a few hours of struggling, I finally figured it out:

WM6 apperantly checks the certificate's Subject field and takes the content of the first subfield as username. In my case, the first subfield is E (e-mail address), and for some reason WM6 only took the first half (the part before @) of the E subfield as username. This behavior unfortunately is unchangeable (I tried with regstry editor and failed).

So in my case, the problem was my CN (common name) subfield was different from my E subfield. So when RADIUS server received the authentication request, it compared the username from WM6 to the CN subfeild of the certificate. And since they were different, the RADIUS server rejected the authentication request.

Long story short, in order to make EAP-TLS work on WM6, you will need to keep E and CN same for your client certificate. Feel free to shoot me an e-mail if you have any questions.

Hao Zhong

daniel g. grau say:

The picture has got to be inserted in line with text. Word should not let you insert a floating image, anyway.

What's probably happening is your footnote's paragraph has a fixed line height, thus truncating the picture except for its lower portion. If that is the case, just select the footnote paragraph and change its line spacing. Easiest is to select "Single."

It would be better to work with the paragraph style applied to the footnotes, usually called "Footnote Text." Just use Format > Style, click on Modify, select "Paragraph" in the bottom-left dropdown menu and choose a larger line spacing, or use "Single."
