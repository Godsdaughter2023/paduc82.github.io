lemp say:
Consider yourself nagged about not having installed sp3.

First of all, 6 GB is too much space to allocate for System Restore,  The 12% default was created back when hard drives were much smaller than they are today.  About 1 GB is sufficient.

Second, why do you think that System Restore is using 30 GB?  Which particular folder are you looking at?  Assuming that you are looking at the System Volume Information folder, you should know that although System Restore certainly does store its restore points in the SVI folder, a few other system processes also use that folder.

When you open System Restore and select Restore my computer to an earlier time, approximately how many restore points are shown in the calendar (each date in bold has at least one restore point; there may be more than one restore point in any given day)?

If your system is otherwise working OK and you don't anticipate a need to use System Restore at this time, I suggest that you do the following:

Turn off System Restore on all drives and reboot.  This should purge all restore points.  When you reboot, take a look at the SVI folder.  Did it shrink or is it still very large?

Turn System Restore back on (if you have more than one drive/partition, make sure that System Restore is only monitoring the system partition -- the one where Windows is located).  Configure System Restore to use just under 1 GB.  Test System Restore as described here --> http://bertk.mvps.org/html/tips.html#3

One of the other systems that use the SVI folder is the Indexing Service.  Unless you frequently use Windows Search, you might want to turn this off -- which will both make the SVI folder smaller and put less load on your CPU.  See http://lifehacker.com/031440/turn-off-indexing-and-speed-up-windows-xp

wunders say: If I read your posts correctly, I pick up that, among other times, this always happens at startup.  This being the case, you can first verify that the time is set correctly in BIOS before starting into Windows.  If it is, and, as you suspect, a program is causing this problem then the task is to find out which one.  A pile of programs start each time you start Windows.  The startup configuration with the bare minimum amount of processes running is called "Safe Mode".  So start your computer up in Safe Mode (repeatedly press F8 as your computer boots).  When it comes up, check the date/time.  If it is OK and hasn't changed, then one of the other optional components must be causing this.  To narrow down which one, you will need to use "msconfig" to enable/disable groups of startup items until you find the one that causes your problem.  The procedure for doing this is outlined in this article:

"How to troubleshoot configuration errors by using the System Configuration utility in Windows XP"
  < http://support.microsoft.com/kb/310560 >

I start by disabling everything in the "Startup" tab.  If this fixes the problem, then I start enabling things in that tab a few at a time until I find the culprit. If Startup Tab doesn't fix the problem, then I try disabling items in the "Services" tab.  I usually start with the non-Microsoft services.

Of course on the machine that has only recently had this problem, you could try a shotgun approach and do a simple "System Restore" to a point in time before this problem.  It will probably fix the problem but you won't know what caused it.
