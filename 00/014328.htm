v_agarwal say:

Lets do in this way.

Assumed you have 48 sheets, create 2 more sheets. One with the Tab Name "Master" and other with Tab name "Report"

Now in your Master sheet in A1, put the number of rows which you want to consolidate in Report sheet.  Say in your case 100. So put 100 in A1 cell of Master sheet.

In B2 to B50 give the name of different sheets which you want to merge.

In A2, put this formula =$A$1*(ROW()-2)+1 and drag down till your data is in B column.

 

And now finally give this formula in Report sheet at cell A1

=IF(OFFSET(INDIRECT(VLOOKUP(ROW(),Master!$A$2:$B$500,2,1)&"!A1"),IF(MOD(ROW(),Master!$A$1)=0,Master!$A$1,MOD(ROW(),Master!$A$1))-1,COLUMN()-1)="","",OFFSET(INDIRECT(VLOOKUP(ROW(),Master!$A$2:$B$500,2,1)&"!A1"),IF(MOD(ROW(),Master!$A$1)=0,Master!$A$1,MOD(ROW(),Master!$A$1))-1,COLUMN()-1))

 

and drag down right and down.

Your data is consolidated. 

Cell A1 of Master sheet is dynamic which you can change as per your requirement. Like presently it is 100, which you may change to 1000 or 10 and accordingly report will be merged for all those number of rows for all the sheets given in "B" column.

I have imported our order schedule lists from our online Sharepoint as linked tables, and have other tables for their contact information (company name, address, contact details) and competency (insurances, third-party accreditations). I want to be able to store the returned value (i.e. most recent order date) for each specific company in a field on their file, and then use this value to see if they need to be marked as Active or Inactive. 


ken sheridan say:

The query would be along these lines:

SELECT Suppliers.*, OrderDate AS LatestOrder
FROM Suppliers INNER JOIN Orders
ON Suppliers.SupplierID = Orders.SupplierID
WHERE OrderDate =
    (SELECT MAX(OrderDate)
     FROM Orders
     WHERE Orders.SupplierID = Suppliers.SupplierID);

The latest date should not be stored in a column in a table, however, but computed in a query like the above when needed.  To store the value would introduce redundancy and the consequent risk of update anomalies.

ken sheridan say: 1.  Date should not be used as a column name.  As the name of a built in function it is a 'reserved keyword' and should not be used as an object name.  Enclosing it in brackets will probably avoid any ambiguity in its interpretation, but ideally the column should be named more explicitly, e.g. OrderDate.

2.  As you have spaces in column names these must be enclosed in brackets:

WHERE [Date] =
    (SELECT MAX ([Date])
     FROM [Master Order Schedule 2016]
     WHERE [Master Order Schedule 2016].[Contractor/Supplier] = [JLES Approved Suppliers].ID );

Spaces or other special characters should avoided in object names.  Instead use CamelCase or represent a space by an underscore character Like_This.  Similarly with special characters, e.g. ContractorSupplier or Contractor_Supplier.

Svchost is a system process, not a virus. It won't affect your system.

Here's more info about it:

On Windows 10, Service Host or svchost.exe is the host under which multiple services (with their PID) can run. If there are multiple services which require use of same resources, they’re usually coupled to share same host, in order to reduce overload on machine.

According to Microsoft, ‘Service Host is a shared-service process that serves as a shell for loading services from DLL files‘. This host file can be located at %systemroot%\System32\Svchost.exe.

j.e. mcgimpsey say:
The arguments After and LookAt are mostly self-explanatory - 

   After:=<range>

starts the search in the cell after the specified range - either one row down or one column to the right, depending on the SearchOrder argument's value.

    LookAt:=xlPart    or   LookAt:=xlWhole

determine whether to do a partial match, or to require the whole cell to match.


However, there's no need to activate sheets or cells. Working with the objects directly is faster, more efficient, and generally easier to maintain.

Here's how I might do it:

    Dim rFound As Range
    Dim rDest As Range
    Dim strFirstFoundAddr As String
    Dim strMyString As String

    strMyString = InputBox("Enter the client")
    With Worksheets("dump")
        Set rDest = .Cells(.Rows.Count, 1).End(xlUp).Offset(1, 0)
    End With
    With Worksheets("sales ledger").Cells
        Set rFound = .Find( _
            What:=strMyString, _
            After:=.SpecialCells(xlCellTypeLastCell), _
            LookIn:=xlValues, LookAt:=xlPart, _
            SearchOrder:=xlByRows, SearchDirection:=xlNext)
        If rFound Is Nothing Then
            MsgBox "Client """ & strMyString & """ was not found"
        Else
            strFirstFoundAddr = rFound.Address
            Do
                rFound.EntireRow.Copy Destination:=rDest
                Set rDest = rDest.Offset(1, 0)
                Set rFound = .FindNext(After:=rFound)
            Loop Until rFound.Address = strFirstFoundAddr
        End If
    End With

Note that to ensure that Find finds the first match, I start the search after the last used cell in the workbook, so it loops back around to the first row.

     http://msdn.microsoft.com/en-us/library/aa220733(v=office.11).aspx

It's not Mac-specific, but 98% of the language is identical.
