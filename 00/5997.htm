Introduction to the Office Macro Editor, Part 1 - Microsoft Community

jay freedman say:

Introduction to the Office Macro Editor, Part 1

Technical Level : Basic

Summary
Know the parts of the Visual Basic for Applications (VBA) macro editor in Microsoft Office programs: Access, Excel, Outlook, PowerPoint, Publisher, and Word. Applies to Office 2003, Office 2007, Office 2010, and Office 2013/Office 365 (not Office Online or Office RT).
Details

Use the macro editor to install macros that you find in online forums; to modify macros that you recorded so they’ll work better; or to create your own macros for solutions that can’t be recorded.

The macro editor is the same in all six Microsoft Office programs that offer it, throughout the 2003, 2007, 2010, and 2013 versions of Office.

Open the macro editor

In Office 2003, click Tools -> Macro -> Visual Basic Editor.

In Office 2007, 2010, or 2013, click Developer -> Visual Basic.

Note: The Developer tab is not displayed by default. To enable it, click File -> Options -> Customize Ribbon. In the list on the right, check the box next to Developer and then click the OK button.

The default keyboard shortcut to open the macro editor is Alt+F11.

Project Explorer pane

The top pane on the left side of the macro editor is the Project Explorer. It’s similar to the folder view in the Windows Explorer file manager. In the Project Explorer, each open template and document appears as an icon. The icon can be expanded to display folders and their contents.

     

Every template and document in Excel, Outlook, Publisher, and Word automatically has a Microsoft <program> Objects folder. In programs that base documents on templates, a document may also have a References folder containing a reference to the document’s template.

You can use the commands on the Insert menu to add userforms, modules, or class modules to the selected template or document. When you add the first item of each of these types, the Project pane automatically adds a folder to contain it.

Properties pane

The Properties pane appears below the Project Explorer. In this pane you can change the names of projects, modules, and other objects, and set the values of properties in userforms when they’re available.

Note: If the Properties pane isn’t visible, click View -> Properties Window (shortcut F4).

To change the name of a project or module, select the item in the dropdown of the Properties pane or click that item’s icon in the Project Explorer. Then edit the text in the (Name) box, and press Enter.

Code window

The large area on the right side of the macro editor is the code window, where you type or paste the VBA code for macros.

The two dropdown lists at the top of the code window are navigation aids. Select an entry in each of the lists to move the cursor to the start of the code of that macro.

  • When the cursor is in the code of a module, the left list contains only the (General) entry, and the right list shows (Declarations) and the names of all the macros in the module.
  • When the cursor is in the code of an item under the Microsoft <program name> Objects folder, the left list contains (General) plus the container for the program (Document, Workbook, and so forth). If you select the container in the left list, the right list shows the events (Open, Close, etc.) for which you can write macros that run automatically when the event happens. The article http://msdn2.microsoft.com/en-us/library/aa140279(office.10).aspx explains how to write event macros.
  • When the cursor is in the code of a userform, the left list contains (General) plus a list of the controls (command buttons, list boxes, etc.) on the form. If you select a control in the left list, the right list shows the events (Click, Change, etc.) for which you can write macros that run automatically when the event happens.
  • When the cursor is in the code of a class module, the left list contains the (General) entry, a Class entry, and the names of any application objects that you define in the class. If you select Class or an application object in the left list, the right list shows the events for that object.

More Information

The second part of this article, describing additional parts of the macro editor, is at Introduction to the Office Macro Editor, Part 2.

karl timmermans (outlook mvp 2012-2018) say:

I have uninstalled and reinstalled ,lost all addresses, have 2 data files but cannot open the 2nd one that has my addresses re-entered.
<<

#1 - What are the file extensions of the 2 data files that you're referring to? (i.e. .pst or .ost)

#2 - The most common reasons for people losing contact/calendar data when using Outlook desktop (part of the Office suite) are that
a) the account was configured as an IMAP account and then removed from the Outlook profile. (see general comments about .ost files below)
b) Outlook was unable to open/find the .ost file for the IMAP account and automatically created a new one in which case the contact/calendar folders (which have the added words of "This Computer Only" in the folder name) will be empty
c) A new Outlook profile was created and the account configured again as IMAP - a new OST file will be created but same thing applies to the contact/calendar folders as for item (b)

#3 - <IF> you have an .ost file that contains your contact/calendar data, then your only option to recover that data is to use a 3rd party OST2PST conversion utility.

General comments re: ost files/Imap accounts

#4 - An OST file can only be opened/read by the original Outlook profile/email account used to create the file
#5 - Contact/calendar folders will have the added words of "This Computer Only" which means just that, the data is local to the computer on which the account is configured.
#6 - The only way to recover data from an "orphaned" .ost file is to use a 3rd party OST2PST conversion tool (none are free)

To mitigate the loss of data in "This Computer Only" folders in the future would be to do the following (for the purposes of this reply, the assumption that this is the first and only account configured)

#7 - Add a PST file to the Outlook profile - do this by
a) going to Outlook Home ribbon --> <New Items> --> <More Items> --> Outlook Data file
b) when done, go to File --> Account Settings --> Data Files tab --> select the newly added PST file and set it as the default. This will now be where the default contact/calendar folders will be stored
c) if you have any items in the "This Computer Only" folder, then you will need to copy those to the corresponding folders in the PST folder tree.
