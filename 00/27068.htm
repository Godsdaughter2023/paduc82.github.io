I was able to find the control in Pub 2003 that allows you to set the color on individual pages to automatic, Grayscale, black & white and washout. This control is on the picture toolbar and when you mouse over it is is called 'color'. Unfortunately I cannot find exactly the same control in Pub 2010.

 

I did find something similar that might work. I added it to my quick access toolbar. Go to your quick access tool bar in the upper left corner and click the right side drop down option to 'customize quick access toolbar' > more commands > commands not in the ribbon and find 'color printing' and add it to the quick access toolbar. (this for convenience...you could choose a different toolbar if you prefer).

 

Now if you select a page in your newsletter and click that 'color printing' icon, you get a dialog box that lets you choose to define all colors as: single color etc. I chose single color black and it seemed to result in a grayscale view of the page. Unfortuanely in my tests it recolored the entire publication. I hope you are able to keep it to the interior pages. If not I found a workaround.

 

This is a bit of a pain but since you save so much money it may be worth it.

 

Download and install www.primopdf.com a free pdf making tool. When you install it watch for the optional toolbars and junk and it should install easily and without problems otherwise. Now the reason I suggest this is that primopdf allows you to append a pdf file to an existing file. You are going to create a pdf file that has color and b&w pages that hopefully will work as intended in the printer.

 

First you will choose to print the color cover page to pdf. Go to Print > select Primopdf as your printer, go to printer properties and select color. Now print only the first page and save the pdf file to your desktop.

 

Now select the second page and click the color printing icon in the quick access toolbar > choose single color and ok out. Your publication is now completely grayscale (or hopefully just the interior pages). Go to Print > primpdf and again choose printer properties and this time select black and white instead of color and print pages 2 - 19, or all the interior pages of the publication that you want in b&w. And this time when you print to primopdf select the same file name as the first pdf file and primopdf will ask whether you want to overwrite or append to that file, choose append. This will add the black and white pages to the color page.

 

The last step is probably obvious. Go back to your publication and the click the undo icon in the quick access toolbar to undo the color change to black and white. Print to primopdf > properties > color > print last page of publication and append it to your original pdf file.

 

The result will be a pdf file that has color front and back pages and b&w interior pages that hopefully will trigger the correct response on your color printer.

 

Hopefully you can use the 'color printing' icon to recolor just the interior pages, but if not maybe this will work for you.

 

You could also create two publications from the one. One publication would be the color cover pages and the other would be the black and white interior pages and printing them seperately. However that would probably preclude the option to have your machine do the stapling for you.

 

You could also install Publisher 2003 on your Win7 machine if you still have the installation discs and keycodes. You can even keep Pub 2010 if you want. Just do a custom install and install Pub 2003 to its own custom folder. I run most versions of Publisher on this computer without problems.

 

Hope something I suggested helps. I hate it when MS strips away a feature or function on new versions. No matter how obscure it seems someone like you has a use for it.

a. user say:

The problem is, when a new build comes in, it upgrades everything, but then creates a Windows.old folder, with enough breadcrumbs to revert to the previous build if need be--and those Windows.old folders eat up a lot of drive space, which the installers have to account for.  I just upgraded to 14332, and the Windows.old folder left from the previous build is 14.7GB in size.

I wouldn't have suggested installing an active OS on a 64GB drive, especially if that's the only drive you've got.  Personal files and programs/apps eat up that space quicker than you might think--and this coming from someone running Windows 10 on a system with a 256GB SSD.

I suggest getting an external drive of no less than 3TB, and storing all your files on that.  Let your system drive serve as the place for your programs and apps, but store your files (photos, videos, etc.) on the external.

It might also help to try running Disk Cleanup (cleanmgr.exe) to clear out any debris that might be sitting on your system.  If space is an issue, where it hasn't been before, one explanation is that temporary files and such have piled up and are now crowding everything else out.
