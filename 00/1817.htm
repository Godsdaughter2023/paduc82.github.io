Stormy Girl,
The reason the outline numbering sequence is unique at both levels (i.e. master and subproject) is that the subproject is not actually part of the master. The master only contains a pointer to the actual subproject. What you see at the master level is a visual representation of the subproject.

In order to get sequential outline numbering at the master level, you will need to use an extra number field and some VBA code to populate the field. I don't have time to write the code for you right now but give me a couple of days, unless someone else steps in with it.

Update: After declaring a "Duh", I realize I missed the obvious simple solution. By converting the dynamic master to a static master, the outline numbering WILL be sequential. To do that, select the summary line insertion point for the subproject in your master. Go to Task/Properties group/Information. Select the Advanced tab and uncheck the "link" option. Hit "OK" and answer "yes" to the next message about saving changes to the subproject. You now have a static master wherein the subproject IS part of it because it has been copied to the master. Outline numbering will be sequential. Note: you might want to get rid of the old summary line that was the insertion point of the subproject by outdenting everything under it and then deleting the redundant summary line.

However, be advised that the static master will no longer be dynamically linked to the subproject. If you don't care, then you're all set, but if you do still want the master and subproject to be dynamically linked (i.e. changes in one are automatically reflected in the other), then you will need the macro I mentioned above.

jasperp say: Say the table is in A1:B24 and the values of Dmin in E1 and Dmax in E2.

Next closest value below Dmin:   =MIN(IF(A2:A24<=E1,B2:B24,""))

Next closest value above Dmax:   =MAX(IF(A2:A24>=E2,B2:B24,""))

These are array formulas and should be entered with Ctrl+Shift+Enter.
