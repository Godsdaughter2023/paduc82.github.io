If fields are required at the table level, then you will not be able to set them to Null, and if they don't allow zero-length strings you won't be able to set them to "", either. 

What exactly are you trying to accomplish by clearing out all the fields, just to have the user re-enter them?  Why doesn't the user just enter new values in place of the old ones?

If you really have to present the user with a blank record (except for the EntityCode field, you could preserve the new entity code, undo the change to the current record, delete that record, go to a new, blank record on the form, and enter the new entity code there.  Bear in mind, though, that the user wouldn't be able to undo the delete and so avoid wiping out the old record.  Or I suppose you could get the user to enter the new record, and only then go back and delete the old one.

Another alternative might be to pop up an unbound form for the user to enter new values for all the other fields, and then when they click OK, enter those values into the currrent record.  That would allow you to update the current record without ever setting required field to Null/"".

However, I don't really understand why you would want to do any of this.  Why do you need to present the user with blank fields?
