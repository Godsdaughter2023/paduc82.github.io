jim gordon mvp say:
Then let’s start some more serious troubleshooting.

First, make sure the documents you are working on are not in one of the old .doc formats. When working on these old file types Word goes into Compatibility Mode and shuts down certain features intentionally to maintain compatibility with the old formats. This link explains compatibility among the various file formats Word uses. To convert an old format file to current Word format, on Word’s File menu choose Convert.

Now we can test whether a Word preference file has gone bad. Make a new user in Mac OS X System Preferences. Switch to the new user account, and try Word in the new user account. When you make a new user account, you start Word off with a fresh set of default preferences and a pristine Normal.dotm template, which is the template Word uses to make a new, blank document among other things.

If Word is OK in the new user we can get rid of bad preferences and/or fix the Normal.dotm template in your regular user account. If the problem remains, then we can rule out bad preferences and the Normal.dotm template. So please try the new user approach and report the results so we can know which fork in the road to take to troubleshoot this problem for you.

OK. Finally, after trying several methods and times. I can fix this issue now.

I found that there were still some IE9 registry key

under HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\Packages\

So, I deleted all IE9 key there. And reinstalled IE11.

After restart, everything normal now. IE11 can run, Windows feature on/off went back. :)

If you cannot uninstall IE9 by the usual process or cannot install IE11. Please try the following:

1. Delete IE 9 by the command:  FORFILES /P %WINDIR%\servicing\Packages /M Microsoft-Windows-InternetExplorer-*9.*.mum /c "cmd /c echo Uninstalling package @fname && start /w pkgmgr /up:@fname /norestart". 

And restart and check if the IE 9 uninstalled successfully or not. If not, continue to below steps.

2. Delete IE *.mum and *.cat files in the directory: “C:\Windows\servicing\Packages”

3. Download the Run as system tool. Open Registry by this tool and go to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Component Based Servicing\Packages\. Find the IE9 key and grant permission manually (full control) then delete all IE9 registry keys.

4. Restart and DONE...
