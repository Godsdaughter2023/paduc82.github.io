hansv mvp say:

Even though Excel has been set to open the files, tab-delimited text files are NOT a native Excel file type. Unfortunately, Microsoft has made it extremely difficult to edit the action a program performs for a file type in Windows 7, so it's not easy to "tell" Excel how to handle .tsv files correctly.

Your best bet might be a macro that displays the Open dialog, then opens the selected .tsv file(s) as tab-delimited:

Sub OpenTSV()
  Dim i As Integer
  With Application.FileDialog(msoFileDialogOpen)
    .Filters.Clear
    .Filters.Add "Tab-delimited files", "*.tsv"
    .AllowMultiSelect = True
    If .Show Then
      For i = 1 To .SelectedItems.Count
        Workbooks.OpenText Filename:=.SelectedItems(i), _
          DataType:=xlDelimited, Tab:=True, Semicolon:=False, _
          Comma:=False, Space:=False, Other:=False
      Next i
    End If
  End With
End Sub

You could add a button to your Quick Access Toolbar that runs this macro.

I have experienced this problem in the past, and fortunately, there are multiple solutions to the problem.

  • Select your account name in the upper-right-(ish) corner of your screen and select Sign Out.

  • You should see a list of cards with ways to earn points, like this.

rohn007 say:
No easy way to do that. Up until Word 2007 there was a "blue screen" emulation mode that recreated the WordPerfect blue screen mode. That was removed in 2007.

Have you considered making the change to Windows itself?

(the details will vary slightly depending on Windows version, this is for Vista) . Right click on the desktop, select Personalize, then "Windows Color and Appearances"  Try the various Color scheme options. One of the "High Contrast ... " options may do the trick for you.

If that doesn't work, click on the "Advanced..." button, then select "Window" from the Item drop down. You can then select a more "eye friendly" color to replace the white background in all applications.
