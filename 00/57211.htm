a. user say:

Hi Matthew,

Windows Management Instrumentation is used to gather system statistics, monitor system health, and manage system components. To work properly, WMI relies on the WMI service. This service must be running and properly configured for your environment. This might not be a case as this service maintains the Windows components.

This issue could occur if Fast startup mode is enabled on the computer.

What is the Model of the Asus Zenbook?

Let me explain what exactly Fast Startup does. Fast Startup (aka: hybrid boot or hybrid Shutdown) is a new feature in Windows 8 to help your computers start up faster after shutting down. When turned on, Windows 8 does this by using a hybrid shutdown (a partial hibernate) method that saves only the kernal session and device drivers (system information) to the “hibernate (hiberfil.sys)” file on disk instead of closing it when you shut down your computer. When you start your computer again, Windows 8 uses that saved system information to resume your system instead of having to do a cold boot to fully restart it.

I would suggest you to disable Fast startup and check if it fixes the issue. Follow the methods:

Method 1:

Follow the steps to turn off Fast startup under Power settings:

  • Press Windows+W keys and type "Shutdown Settings"
  • Click the "Settings" Icon on the right of the screen.
  • Click "Change what the power buttons do" on the left of the screen.
  • Click "Change settings that are not currently available"
  • Uncheck the box titled "Turn on fast startup (recommended)."
  • Click "Save Changes" button.

Restart the laptop and check if the touch screen works. If not, proceed to the next method.

Method 2:

Run the hardware and device troubleshooter and check if it finds and fixes issues with the device:

Refer to the section “Run the Hardware and Devices troubleshooter”

http://windows.microsoft.com/en-US/windows-8/what-device-isnt-installed-properly

For further troubleshooting, download and install the latest chipset driver from Asus’s website for the specific mode for Windows 8.1.

http://windows.microsoft.com/en-us/windows-8/all-drivers

Please keep us posted. We will be glad to assist you further.

As a first step of troubleshooting, I would suggest you to boot the computer in Safe mode.

c.    Once you login to Safe mode, check if you receive this error message.

 

Step 2: Perform a clean boot

While in safe mode, put the computer into clean boot and restart the computer. A clean boot helps eliminate software conflicts. Visit the link below to know how to put the computer in a clean boot state.

 

How to troubleshoot a problem by performing a clean boot in Windows 7

http://support.microsoft.com/kb/929135

Note: Once you’re done with troubleshooting, reset the computer to start as usual.

 

Step 3: Perform a check disk from Safe mode

You can also try performing check disk.

a.    Click on Start; in the start search type cmd.

b.    Right click on cmd.exe and choose to “Run as administrator”.

c.    Type the following command and press enter.

Chkdsk /r

The operation starts once the computer is restarted.

Disclaimer: Check Disk will try to fix the bad sectors on the hard disk. So data present on the bad sectors will be lost.

 

Step 4: Check for errors in Event viewer

Event Viewer can be helpful when troubleshooting problems and errors with Windows and other programs. Visit the link below to know what event viewer does.

 

What information appears in event logs (Event Viewer)?
