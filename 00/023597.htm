the following screen shot shows the Silverlight downloads page with IE's ActiveX filtering turned ON.... you will see a 'blue circle with a line through it' icon in the IE Address bar on web sites when any content is being blocked by IE's ActiveX Filtering or Tracking Protection settings....

On any web site where you see the blocked content icon in the IE Address bar, double-click the icon to adjust your IE ActiveX filtering and Tracking Protection settings FOR THAT SITE....

You can also switch your ActiveX filtering and Tracking protection settings FOR ALL WEBSITES from the IE Tools menu....Tools>ActiveX filtering, and Tools>Tracking Protection... (not recommended)....

Now, normally Company websites ( aka. Intranet websites) are not affected by your browser settings for ActiveX filtering or Tracking protection......

If you are seeing the 'Blocked content icon' (blue circle with a line through it(above)) in the IE Address bar when you visit your companys' intranet site... it means that you IE security zone settings are incorrect....

the exceptions are online corporate portals like Outlook.com and Office365....(contact your Help Desk for assistance with configuring your browser for these types of websites)

If this occurs, then in the first instance you should contact your company's Help desk or System Administrators....

To determine which IE security zone a public website or your company's website(s) are mapped to, use the File>Properties menu in IE (for public websites, the Internet zone is expected, for company websites, the Intranet zone is expected)....

To correct this you should seek the help of your Company's Help Desk or System Administrators...

or if you are a small business and do not have a help desk or specialist IT staff

1. Find out which IE security zone the website you are using is mapped to...

2. If its a company web site and step 1 does not show that it has been mapped to the Intranet zone then, go Tools>Internet Options>Security tab, click the Icon of the IE Security zone that you found in step 1.... (probably you have mapped it to your Trusted sites zone).

3. Remove the website from your Trusted sites list and save your changes.....

Return to your website in the browser, and use the File>Properties menu to confirm that your company website now is mapped to the IE Intranet zone....

If it is not mapped to the intranet zone

1. Tools>Internet Options>Security tab, click the Local Intranet icon...then the 'Sites' button, check "Automatically detect Intranet network"....

Generally, you should not change any of IE's Security zone settings.... 'In the hope of getting things to work'....

To reset your IE Security zone settings back to their factory or your company's defaults, go Tools>Internet Options>Security tab, click "Reset all zones to default".... save your changes....

"Windows Feedback" feedback - Microsoft Community

ahaleiii say:

"Windows Feedback" feedback

Currently, the Windows 10 Technical Preview seems stable, usable, and for the most part the same at the core (user experience-wise).  The main issues I am currently having with productivity involve the Windows Feedback app.  There are many individual issues I've seen, and I see no central location to discuss them, so I'm making one.

Things I could be improved upon:

- Screenshot taking (generally all things screenshots)

- View original screenshot(s) (if any)

- Ability to browse "Me too!"s

- View the screenshot(s) for a "Me too!"

- Ability to "Me too!" a "Me too!"

- Ability to say "Not me" (if someone knowingly is not have the same issue, or if they have a different opinion on an issue)

- Upon providing a "Me too!" feedback, have ability to go back to previous search context instead of going to Feedback home (so far I have been doing Feedback "sessions" and would like to continue browsing similar, yet distinct, issues)

- Better search features. Currently, there is no global search in the home, but it seems like searching in a particular category will also search globally.  This can be confusing.  More robust search features would allow for more efficient feedback (sometimes I want to spend 20 seconds or less providing feedback; any more and it's not worth it) as well as reduce the number of redundant issues.

- Ability to sort results by the number of "Me too!"s or by a created date

- Ability to only return issue results that have been created within a certain date range

Additional features:

- More robust discussion features (should it have forum elements..?)

- Ability to link to issues!

- Ability to search go from Windows Feedback app directly to Windows Technical Preview forums with the current context

- A dashboard that allows you to view the issues you've created and the "Me too!"s you've given

- Ability to "star" or "favorite" an issue, and ability to view that in your Feedback dashboard

*edit:  Basically, I want to give Feedback the functionality of answers.microsoft.com discussions.

Yes, it's only the first day of feedback, so most likely some of these are already being addressed.  That said, Microsoft has (wisely) given themselves a full year to get feedback, and a fully featured Feedback app will allow them to get the most out of it.  It will also allow users to give better, more accurate feedback.

Regards,

Alan
