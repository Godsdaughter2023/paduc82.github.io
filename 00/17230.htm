Hi,

ALT+F11 to open VB editor, right click 'ThisWorkbook' and insert module and paste the code in

change Src & Dst sheets to the correct ones

Sub Look_4_P()
Dim MyRange As Range, CopyRange As Range
Set Src = Sheets("Sheet1") 'Source data
Set dst = Sheets("Sheet2") 'Destination sheet
dst.UsedRange.ClearContents
LastRow = Src.Cells(Cells.Rows.Count, "A").End(xlUp).Row
Set MyRange = Src.Range("A2:A" & LastRow)
For Each c In MyRange
If UCase(c.Value) = "P" Then
        If CopyRange Is Nothing Then
                Set CopyRange = c.EntireRow
        Else
                Set CopyRange = Union(CopyRange, c.EntireRow)
        End If
    End If
Next
If Not CopyRange Is Nothing Then
CopyRange.Copy Destination:=dst.Range("a1")
End If
End Sub

maor bracha say:

Hi Robert,

It is possible to migrate mailboxes from a different forest.

Assuming that you already migrated AD user accounts, your steps would (roughly) be:

_____________________

Domain preperation:

_____________________

1. Add the domain to Office 365.

2. Maintain mailflow during migration:

   2.A. Set the domain external relay and create a send connector to route messages using DNS.

   2.B. Create a new remote domain using "New-RemoteDomain" command, and set it to trust outbound and inbound messages : "Set-RemoteDomain <Domain> -TrustedMailinbound:$True -TrustedMailOutbound:$True"

Note: If using Hybrid configuration, you would also need to consider internal mailflow needs (send\receive connectors, etc.)

_____________________

User preperation:

_____________________

3. Mail enable users, set primary SMTP and external routing address to match current primary SMTP address.

Make sure to maintain the email addresses (secondary), including X500's to prevent NDRs.

4. Match the MailboxGuid attribute - make sure to "migrate" the MailboxGuid attribute from old mailboxes to your newly created MEUs.

5. Run DirSync.

_____________________

Migration preperation:

_____________________

5. Create a migration endpoint as you normally would ( technet.microsoft.com/.../jj874458(v=exchg.150).aspx )

6. Create and start a migration batch.

_____________________

Completing the batch would be as usual.

User mailboxes on source Exchange server would be converted to Remote-Mailboxes, and would be enabled on Office 365.

Don't forget licensing your newly migrated mailboxes.

If using a Hybrid environment, you would also have to convert your MEU to Remote-Mailbox objects - "Enable-RemoteMailbox <Alias>"

1 - It's possible to use this method to migrate from a source Exchange 2010 forest to target forests using various Exchange solutions: Hybrid, DirSync Only, or Office365 standalone.

2. This has to be done on your target environment servers.

If using a Hybrid, then you would have to configure both on your local Exchange servers and Office 365 Exchange online.

Of course that once completed the migration you would have to modify your connectors to route according to your Hybrid.

3. Same domain as the accepted domain from 2.A. , should be performed on target environment much like 2.A.

4. No, this has to be done on your target environment in order to "map" the mailboxes to be migrated and allow Office 365 Exchange online to match those mailboxes with the source mailboxes hosted on the remote forest.

If your target environment is using a Hybrid solution, then this step should be performed on-prem and synced using DirSync\AADconnect.

The migration process itself is seamless, much like a database move or hybrid migration - Users are able to access their data on the source Exchange during the data move.

The only interference is completing the migration.
