the quick scan is very useful. It looks at all of the actively running processes on the machine, as well as files that are pointed to by the various registry entries and other locations used by spyware to start automatically or otherwise interfere with your system.

Sub SplitData_01() 'start vba ###

'Apr. 13, 2016

Const sStr$ = "|" '<< pipe delimited

Dim ws As Worksheet
Set ws = Sheets("Sheet1") '<< source sheet name, change as needed
Dim r As Long, c As Long, t As Long, x As Long
r = ws.Range("A1").CurrentRegion.Rows.Count
c = ws.Range("A1").CurrentRegion.Columns.Count
Dim v As Variant, vv As Variant
Sheets.Add
ws.Range("A1").Resize(, c).Copy Cells(1, 1)
t = 2
For x = 2 To r
If InStr(ws.Cells(x, c), sStr) = 0 Then
ws.Cells(x, 1).Resize(, c).Copy Cells(t, 1)
t = t + 1
Else
v = Split(ws.Cells(x, c), sStr)
For Each vv In v
ws.Cells(x, 1).Resize(, c).Copy Cells(t, 1)
Cells(t, c).Value = WorksheetFunction.Trim(vv)
t = t + 1
Next
End If
Next
ActiveSheet.UsedRange.EntireColumn.AutoFit
End Sub 'end vba ###

evapvd say:

It is ridiculous to have to download all the non-Latin fonts! I did some research, and it turns out that these fonts are from the older Mac OS versions and no longer come with the newer versions (I have 10.13.5).

The workaround is:

  • Upon startup, one of the first things I do is as you had recommended and launch Font Book (even though I normally use, Suitcase Fusion, seems this is a necessary step)
  • Select the fonts that are greyed out, which are the ones Outlook is asking for
  • Right click and select Download
  • Then you can launch Outlook and Word and shouldn't have any more notifications

TIP: To make it easier every time you reopen Font Book:

  • Hold down the Command + Shift keys and select all the grey-out fonts
  • Then don't click anywhere else! This should assure that when you restart Font Book the next time, the fonts will still be selected and you only have to right click on one of the highlighted ones to Download all of the fonts.
