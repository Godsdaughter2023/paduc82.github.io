steve winograd say:

You can use Windows Anytime Upgrade to upgrade from Windows 7 Home Premium to Windows 7 Professional and keep your existing information, programs, etc.

However, you can't then transfer Windows 7 Home Premium to another computer, for these reasons:

  • When you install an upgrade version of Windows, the previous version (which qualifies you for an upgrade version instead of a full version) must stay on the computer.
  • A version of Windows that's installed at the factory (OEM version) can't be transferred to another computer.  It's licensed to the original computer only.

  • The PC boots into Windows RE (Recovery Environment).

jholmes900 say:

Unable to Fix a Corrupted User Profile

Please read my resolution posted on October 14, 2011. My problem was never a corrupted user profile. It was a Anti-Virus program blocking my access.

I tried to follow the instructions for fixing a corrupted user profile, according to http://windows.microsoft.com/en-US/Windows7/Fix-a-corrupted-user-profile

I created a third User Account (UA3); Logged on to UA2 and attempted to copy and paste the files from the User file for the old account (UA1) to the third account (UA3).

First, I am confused by the instructions that told me to copy all files except for ntuser.dat, ntuser.dat.log and ntuser.ini. Did that mean all files that start with those names or all files with exactly those names? There are other files that start with those names but are not just those names. For example: ntuser.dat.LOG1, ntuser.dat.LOG2, ntuser.dat{3f2f6031-3445-11e0-a837-8bc..., etc.

I attempted to copy all of the files except those that start with ntuser.dat etc., but I must have done something wrong because it did not work. I got a message that said I needed administrator privilages and was asked if I wanted to continue. I responded continue, but the files did not paste.

If I am able to successfully copy the files, after Profile UA1 is copied to UA3, when I log on to UA3, should it look the same as UA1 or do I need to reset the background, etc.? It is not clear in the article what I am really copying and what remains to be done to make them the same ( except for the corruption of course).

This all surfaced from a problem that I had trying to add email accounts to Outlook 2010. After working on the problem for several days, the error has been identified as stemming from a corrupt user profile. If you need background information, please see http://answers.microsoft.com/en-us/office/forum/office_2010-outlook/access-denied-when-adding-a-email-account/7bb6576e-53ab-4c94-830c-0e58be115dcf?page=1&tm=1318178723530

I have seen this kind of corruption problem reported several times in answers.microsoft.com, but nothing I have found so far has addressed what causes the problem in the first place.

jholmes900 say: RESOLVED. I found the problem. It was a setting in the Anti-virus program. Somehow it had been set to block access to the registers for Office 2010. I removed the Registry Control setting and I am now able to add/change email accounts. There was no corruption of the User Account. I am sorry it took so long to find it. I did however learn a lot during the process.

There are a couple of things that need to be addressed in your instructions for "Fixing a Corrupted User Profile". It would help others in the future.

1. They should mention that, after creating the new User Account, you need to logon to that account, otherwise the files are not created.
2. Instead of saying "copy all files except for ntuser.dat, ntuser.dat.log and ntuser.ini." it should say " copy all files except those that start with.....(those names)".
3. Something should be mentioned about the shortcut files, with the upward arrow in their icon. They do not copy at all in Windows 7.

I was never able to fix a User Profile with the instructions the way they are written. (I think someone should try them with a Windows 7 system and deal with the permissions that are set in Windows 7.)

In your reply about what programs might cause something like this, you might want to mention blocked registry access by a virus protection program.
