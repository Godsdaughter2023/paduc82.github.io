Greetings! I am not sure if this is still valid request but you can try this solution I have found:

ReadyBoost Issue With 32GB SD/SDHC Class 10

Initially, been using ReadyBoost with Patriot LX 8GB Class 10 SD 3.0 on my Microsoft Windows 7 without any problem. However, been thinking of putting my portable application into the sd card. Therefore i bought Transcend TS32GSDHC10 32GB Class 10 SD 3.0. However, the ReadyBoost feature is turn off for this device as it's not fast enough according to the system test. C'mon, it's class 10 one, man. Microsoft claimns that ReadyBoost minimum requirement is with sd class 4 and above.

I suspect that the size of the sdhc play part in here. By using EASEUS Partition Master Home Edition, i partition the 32GB into 8GB and the ReadyBoost feature is working. So i tried several combination of the size and found out that the maximum size that a ReadyBoost can work is about 26,400MB for FAT32 or NTFS format. Nicrosoft claims that the ReadyBoost can work on 32GB or higher capacity of the sdhc. But seem like it's not the case in here.

So now i have to find a workable solution for this issue. Turn out that we can force to enable the ReadyBoost by changing the registry as per following:

  1. Plug in the device
  2. Right click on the device and select property follows by ReadyBoost tab
  3. Select "Do not retest this device" and click OK to apply
  4. Unplug the device
  5. Run regedit (start->run->regedit)
  6. Browse the registry to HKLM (Local Machine)\SOFTWARE\Microsoft\Windows NT\CurrentVersion\EMDMgmt
  7. Expand the sub folder and find your sd folder
  8. Change the Device Status to 2 (decimal, not hexadecimal)
  9. Optional: change ReadSpeedKBs to 2000
  10. Optional: change WriteSpeedKBs to 2000
  11. Plug in the device and now the ReadyBoost should be enable for this device
