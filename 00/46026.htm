Your System registry file became corrupted.  You will need to boot your machine from a bootable CD to recover your machine.  There are several around but my preference is to use a Live Linux CD to boot your machine such asKnoppix or Ubuntu .  (If you use Ubuntu, take care not to install but instead choose the "Try me" option when you boot from the CD.)  These CDs can be created on another machine by downloading a .iso file and burning it by using you CD burner's "Burn Image" option.  After booting from this CD, you will have access to your hard drive and by plugging in a USB drive, you can backup any files that you wish to back up.

After you back up any files that you need to back up, you can try to recover your system.  A backup of your registry files are kept in the "\System Volume Information" folder as part of System Restore.  Navigate to this folder and open it.  Inside you will find a folder that begins with _restore{xxx}.  Open that folder.  Inside, you will find many folders named RPxxx.  Pick the latest folder dated before your problem and open it.  Inside, open the folder "snapshot".  One by one, copy the following files (R-click, Copy) and paste them (R-click, Paste) into the \Windows\System32\config directory:  _REGISTRY_MACHINE_SAM, _REGISTRY_MACHINE_SECURITY, _REGISTRY_MACHINE_SOFTWARE, _REGISTRY_MACHINE_SYSTEM, _REGISTRY_USER_.DEFAULT. 

After copying the files  to the Config directory, then rename (R-click, Rename) the following files in this directory by adding ".xxx" to the name:
     sam, security, software, system, default
(In otherwords, rename "sam" to "sam.xxx", "security" to "security.xxx", etc).  After that, Rename the files that you pasted there to the corresponding files that you just renamed (In other words, Rename "_REGISTRY_MACHINE_SAM" to "sam", "_REGISTRY_MACHINE_SECURITY" to "security", etc).  When done, shutdown the system, remove CD, and reboot your machine.  It should start up into Windows as normal.

After it boots, perform a "System Restore" (Start -> Programs -> Accessories -> System Tools -> System Restore) to insure integrity of your system.  If it doesn't work, then reboot to Safe Mode (press F8 repeatedly at boot) and try again.

This procedure is obviously more involved so don't feel bad if you want to stop after recovering your files.

The corrupted registry idea is often the first approach taken since it is the first thing to come up in a Google search for the error message.  That is too bad and all one needs to do is reproduce the problem a few times and actually fix it to get a little practice.

After a power interruption (or using the power button) the file system can become corrupted and can usually be fixed with a chkdsk /r from the Recovery Console.

When you open Process Explorer you need to customise settings. You do this by checking options on the View menu drop down. Click on Select Columns and then the Process Image tab. With the CPU column you will see System Idle Process frequently showing a high figure. It is important to always remember at all times that this figure represents CPU capacity not being used. The figure displayed is 100.00 less the CPU being used by all running processes.

When you are experiencing high CPU usage you can see which process is using the CPU. What you should then do is post the command line of the process you have identified as using the CPU. We can then suggest how you should investigate further in order to discover the cause of high CPU activity.
