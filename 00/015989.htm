faradaynwonu say:

How to compact and repair a Back-End database from the Front-End one

dirk goldgar say:

 

right now, I'm just preparing the database for deployment.

please help me with the code to compact and repair the database when every user is logged out. And if possible, I need a code to forcefully log every user out too when I want to compact the BE.

Here's a slightly modified copy of code from one of my applications.  You can use it as a model if you want, but naturally you'll have to adapt it.

'------ start of code -----

' Copyright: Except for function fncFileExists, written by Allen Browne,

'       distribute it freely, provided that the copyright notice
'       remains unchanged.

Public Const conBACKEND_NAME As String = "MyApplication_Data.accdb" 

Public Const conPROFILE_TABLE As String = "USysProfile"

Function fncCompactDatabase(Optional NoClose As Boolean = False) As Boolean
    ' Compact the backend in place
   
    Dim colObjects As Collection
    Dim strBackendPath As String        ' name and path of current backend
    Dim strBackupFile As String         ' path to backup copy
    Dim strActiveForm As String
    Dim strRenameDB As String
    Dim strBackupPath As String         ' current path & \backup
    Dim strError As String
    Dim i As Integer
    Dim blnFailedToReopen As Boolean
    Dim blnFail As Boolean
   
    ' Get the name of the active form, if any.
    On Error Resume Next
    strActiveForm = Screen.ActiveForm.Name
   
    ' Establish normal error-handling.
    On Error GoTo Err_Handler
   
    ' Proceed.
 ' get current backend path & name
    strBackendPath = fncDataDBName()

    strBackupFile = strBackupPath & "\" & conBACKEND_NAME
    strRenameDB = CurrentProject.Path & "\old_" & conBACKEND_NAME
   
    If Not Len(Dir(strBackupPath, vbDirectory)) > 0 Then
        MkDir strBackupPath
    End If

    If fncFileExists(strBackupFile) Then Kill strBackupFile
   
   
    If NoClose = False Then
        ' Close all connections to the back-end database, by closing
        ' all data-bound objects that are currently open.  Keep a record
        ' (in a collection) of the state of all these objects.
        Set colObjects = fncCloseAllExcept(acForm, "fm_Backup")
        DoEvents
    End If
  
    With DBEngine
        .Idle dbRefreshCache
        .Idle dbFreeLocks
        .CompactDatabase strBackendPath, strBackupFile
    End With
   
    If fncFileExists(strRenameDB) Then Kill strRenameDB
    Name strBackendPath As strRenameDB
   
   FileCopy strBackupFile, strBackendPath
      
    If Not fncFileExists(strBackendPath) Then
        Name strRenameDB As strBackendPath
        fncCompactDatabase = False
        MsgBox "Compaction did not work correctly. Please call customer suppport.", vbCritical, "Compaction failed"
        blnFail = True
    Else
        fncCompactDatabase = True
    End If
       
    If NoClose = False Then
        ' Attempt to reopen all the objects that we closed before
        ' the backup.
        For i = colObjects.Count To 1 Step -1
            If fncReopenObject(colObjects(i)) = False Then
                blnFailedToReopen = True
            End If
        Next i
       
        ' Bring the formerly active form to the front, if possible.
        If Len(strActiveForm) > 0 Then
            If CurrentProject.AllForms(strActiveForm).IsLoaded Then
                DoCmd.SelectObject acForm, strActiveForm
            End If
        End If
       
    End If
   
    If blnFailedToReopen Then
        MsgBox _
            "Note: The Compaction was successful, but the application couldn't " & _
                "reopen one or more of the objects that were previously open. Please contact customer support.", _
            vbExclamation, "Unable to Reopen"
    ElseIf blnFail = False Then
        MsgBox "Compaction Successful", vbInformation, "Compaction Successful"
    End If
   
Exit_Point:
    Set colObjects = Nothing
    Exit Function
   
Err_Handler:
    subDisplayAndLogError "CompactDatabase", Err.Number, Err.Description
    Resume Exit_Point

Function fncDataDBName() As String

    ' this application is working with.  If the data tables are local, not linked, return
    ' the name of the current database.

        If Len(strDBFileName) = 0 Then
            strDBFileName = .TableDefs(conPROFILE_TABLE).Connect
            If Left$(strDBFileName, 10) = ";DATABASE=" Then
                strDBFileName = Mid$(strDBFileName, 11)
            Else
                strDBFileName = .Name
            End If
        End If
    End With

Function fncFileExists(ByVal strFile As String, Optional bFindFolders As Boolean) As Boolean

    'Arguments: strFile: File name to look for. Current directory searched if no path included.
    '           bFindFolders. If strFile is a folder, FileExists() returns False unless this argument is True.
    'Note:      Does not look inside subdirectories for the file.
    'Author:    Allen Browne. http://allenbrowne.com June, 2006.
   
    Dim lngAttributes As Long

    lngAttributes = (vbReadOnly Or vbHidden Or vbSystem)

        lngAttributes = (lngAttributes Or vbDirectory) 'Include folders as well.
    Else
        'Strip any trailing slash, so Dir does not look inside the folder.
        Do While Right$(strFile, 1) = "\"
            strFile = Left$(strFile, Len(strFile) - 1)
        Loop
    End If

    On Error Resume Next
    fncFileExists = (Len(Dir(strFile, lngAttributes)) > 0)
   
End Function

Function fncCloseAllExcept( _
                Optional ObjectType As AcObjectType = acDefault, _
                Optional ObjectName As String) _
            As Collection

    ' type were passed as arguments, and return a collection containing the
    ' names and states of the objects that were closed.

    Dim colObjects As Collection
    Dim strObjectState As String
    Dim lngX As Long
   
    Set colObjects = New Collection
   
    ' Close all open tables
    For Each ao In CurrentData.AllTables
        With ao
            If .IsLoaded Then
                If ObjectType = acTable And ObjectName = ao.Name Then
                    ' Spare this one
                Else
                    strObjectState = fncObjectState(ao, acTable)
                    DoCmd.Close acTable, ao.Name, acSaveNo
                    colObjects.Add strObjectState
                End If
            End If
        End With
    Next ao
   
    ' Close all open queries
    For Each ao In CurrentData.AllQueries
        With ao
            If .IsLoaded Then
                If ObjectType = acQuery And ObjectName = ao.Name Then
                    ' Spare this one
                Else
                    strObjectState = fncObjectState(ao, acQuery)
                    DoCmd.Close acQuery, ao.Name, acSaveNo
                    colObjects.Add strObjectState
                End If
            End If
        End With
    Next ao
   
    ' Close all open reports
    For lngX = Reports.Count - 1 To 0 Step -1
        With Reports(lngX)
            If ObjectType = acReport And ObjectName = .Name Then
                ' Spare this one
            Else
                strObjectState = fncObjectState(.Report, acReport)
                DoCmd.Close acReport, .Name, acSaveNo
                colObjects.Add strObjectState
            End If
        End With
    Next lngX
   
    ' Close all open forms
    For lngX = Forms.Count - 1 To 0 Step -1
        With Forms(lngX)
            If ObjectType = acForm And ObjectName = .Name Then
                ' Spare this one
            Else
                ' If the form is dirty, find out if the user wants to save the record.
                If .CurrentView <> acCurViewDesign Then
                    If Len(.RecordSource) > 0 Then
                        If .Dirty Then
                            Select Case _
                                MsgBox( _
                                    "You have unsaved data in form '" & .Caption & "'. " & _
                                        "Do you want to save it?", _
                                    vbQuestion + vbYesNoCancel, _
                                    "Save Data?")
                                Case vbYes
                                    .Dirty = False
                                Case vbNo
                                    .Undo
                                Case vbCancel
                                    Err.Raise conERR_ACTION_CANCELLED, , "Operation cancelled by user."
                            End Select
                        End If
                    End If
                End If
                ' Now close the form.
                strObjectState = fncObjectState(.Form, acForm)
                DoCmd.Close acForm, .Name, acSaveNo
                colObjects.Add strObjectState
            End If
        End With
    Next lngX
       
    Set fncCloseAllExcept = colObjects
   
    Set colObjects = Nothing
       
End Function

Public Function fncObjectState(pObject As Object, pObjType As AcObjectType) As String
   
    Dim ObjName As String
    Dim ObjType As AcObjectType
    Dim ObjView As Integer
    Dim ObjActiveControl As String
    Dim ObjFilter As String
    Dim ObjOpenArgs As String

   
    ObjName = pObject.Name
    ObjType = pObjType
    ObjView = pObject.CurrentView
    ObjActiveControl = pObject.ActiveControl.Name

    If pObject.FilterOn = True Then
        ObjFilter = pObject.Filter
    End If

    ObjOpenArgs = pObject.OpenArgs

    fncObjectState = _
        ObjName & Chr$(0) & ObjType & Chr$(0) & ObjView & Chr$(0) & _
        ObjActiveControl & Chr$(0) & ObjFilter & Chr$(0) & ObjOpenArgs

End Function

Public Function fncReopenObject(ByVal pObjectState As String) As Boolean

    ' attempt to reopen that object to that state.
   
    Dim os As DBObjectState
    Dim astrProperties() As String
    Dim intView As Integer
   
    astrProperties = Split(pObjectState, Chr$(0))
   
    On Error Resume Next
    os.DBObj_Name = astrProperties(0)
    os.DBObj_Type = Val(astrProperties(1))
    os.DBObj_View = Val(astrProperties(2))
    os.DBObj_ActiveControl = astrProperties(3)
    os.DBObj_Filter = astrProperties(4)
    os.DBObj_OpenArgs = astrProperties(5)

   
        Case acForm
            Select Case os.DBObj_View
                Case 1: intView = acNormal
                Case 2: intView = acFormDS
                Case 7: intView = acLayout
                Case 0: intView = acDesign
                Case Else: intView = acNormal
            End Select
            DoCmd.OpenForm os.DBObj_Name, intView, , os.DBObj_Filter, , , os.DBObj_OpenArgs
            If Len(os.DBObj_ActiveControl) > 0 Then
                Forms(os.DBObj_Name).Controls(os.DBObj_ActiveControl).SetFocus
            End If
           
        Case acReport
            Select Case os.DBObj_View
                Case 5: intView = acViewPreview
                Case 6: intView = acViewReport
                Case 7: intView = acViewLayout
                Case 0: intView = acViewDesign
                Case Else: intView = acViewPreview
            End Select
            DoCmd.OpenReport os.DBObj_Name, intView, , os.DBObj_Filter, , os.DBObj_OpenArgs
           
        Case acQuery
            DoCmd.OpenQuery os.DBObj_Name, acViewNormal
           
        Case acTable
            DoCmd.OpenTable os.DBObj_Name, acViewNormal
           
    End Select

   
End Function

Now, for knowledge sake:

1.  If I want to only compact the backend without running the fcnCompactDatabase  function and without making any backup copy, how will I write the code?

You're welcome.  To answer your questions:

1. You still need to do most of what fncCompactDatabase does, regardless of how you do it.  You must close all objects with connections to the database, which means all bound forms, reports and datasheets, and all unbound objects that use combo or list boxes with table/query rowsources.  Plus, since all methods that compact a database -- DBEngine.Compact or Application.CompactRepair -- create an output database rather than compact "in place", you must provide an output file name and then, after verifying that the compact was successful, replace the original database with the compacted one.  You can do that by deleting the original database and renaming the compacted one to the original name, which I believe is what Access does when you tell it to compact the current

2. When Access itself compacts a database, it uses a slightly different process than the DAO DBEngine.CompactDatabase method. The process used by Access is said to be slightly superior, but I don't recall in what respect.  I believe that the CompactRepair method of the Access Application object (not CurrentProject) exposes this process, so in cases where you have an existing Access Application object to work with, I expect that would be somewhat superior to using DBEngine.CompactDatabase.  So why didn't I use that method in my fncCompactDatabase() function?  I don't know.  Maybe that method wasn't exposed when I wrote that function, or maybe I didn't know about it.  If I were rewriting that function now, I would probably seek to use Application.CompactRepair, particularly since it provides a LogFile option to report corruption detected in the source database.

Make a new file and add this code into the code module ThisWorkbook:

Dim WithEvents App As Application
