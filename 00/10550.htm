OData support in Excel - Microsoft Community

vincent croquette (vincent_croquette) say:

OData support in Excel

I've been using OData for years, it's a great way to expose data that can be consumed easily by many client types (Excel, Tableau, PowerBI...).

But to me there are 3 issues in the Excel OData connector creating data and performance problems regularly:

1 For some reason, when Excel retrieves data from an OData feed, it is requested 3 times. 

You can easily spot with tools like Fiddler. It's OK when the data can be retrieved quickly (although it creates unnecessary processing and network traffic) but as soon as the data needs >20s to be retrieved, the user sees >60s for no reason. 

I'm sure something could be done in Excel to get the data just once and work with it without requesting it multiple times from the source.

2 Excel seems to completely ignore metadata in the OData payload and tries to 'be smart' by detecting data types automatically. 

You have to deal with 'Schema Sample Size' for instance to force it to read more records for its (unnecessary) sampling otherwise numeric data can be considered as text in cases where there are null values. This is very frustrating since the OData payload contains all the type information, Excel should not have to 'guess' anything.

The other side effect of this 'smart' behavior is that Excel takes the user's regional settings in consideration when reading the data although OData has been designed to avoid any formatting issue. I regularly get complains from users who have different windows regional settings than US unable to get data from OData sources because Excel thinks the number separtor, date separator... is invalid.

3 The new OData connector available in Power BI and Excel 2016 reads the service metadata and caches it regardless of the HTTP header specifying it should not be cached. I have a case where new entity sets are added dynamically and metadata does change. The user has to clear the cache after each data retrieval otherwise only the entity sets available during the first call are available. 

The workaround is to use the legacy data importer which I hope MS will continue to support in future Excel releases.

I don't expect MS to fix these issues anytime soon since they have been in Excel for years but does anyone have the same issues?

Two Columns in Word - one for headings one for associated text - Microsoft Community

zed mac say:

Two Columns in Word - one for headings one for associated text

What breaks do I need to have to have two columns. On the left side are the headings and the right the associated text?
jay freedman say:
There are two ways to handle this. It's unintuitive, but neither of them involves the Columns button on the Page Layout tab or any kind of breaks!

One: Insert a two-column table. Type the heading in the left cell of a row and the text in the right cell. When it's time to insert the next heading, start a new row by pressing the Tab key at the end of the text in the right cell. (If one row extends for more than a few pages, it's a good idea to start a new row just to keep Word from getting "unhappy".)

Two: Create a page setup with a wide left margin. Then use the technique described in http://wordfaqs.mvps.org/MarginalText.htm to put the headings in that margin.

The trouble with using the Columns feature is that you'd have to put a break (in this case, a Column break, Ctrl+Shift+Enter) in the right-hand column at the end of each page to prevent that text from continuing into the left column on the next page. That would mean that any little edit, or a change to a different printer driver, could cause the breaks to occur in the wrong places resulting in partially blank pages.

stefan blom say:

You could use a two-column table (hide the borders). For more options, see http://wordfaqs.mvps.org/MarginalText.htm.

First of all: saving an ics-File and importing it is NEVER any solution when you're trying to share calendars that you want to sync.

The ics file is just one file that contains all your calendar entries from the moment you created that file. The file only updates when you manually create a new file and import it again.

So the way to go is definitely the "link" option. This way your program or interface connects automatically to the file on the Outlook servers were the file is kept up to date.

As I've written before: out of 3 attempts, 2 worked when I pasted the link from the "Get a link" page on Outlook.com and substituted the webcals-prefix with https. The second time I faced the same problem as you: for no reason it just wouldn't work until my Thunderbird erased all account information and I tried again.

And still, there's always the issue of not being able to write data to the calendar if it's shared. It's always read-only.

But when you're using Outlook 2010 or 2013 there shouldn't be any problem with viewing and modifying your own Outlook.com calendar. In 2010 you must download the Hotmail Connector from

http://office.microsoft.com/en-us/outlook-help/microsoft-office-outlook-hotmail-connector-overview-HA010222518.aspx

From what you're writing I assume you use a not synchronized, offline, profile-based Outlook 2010 calendar. You should then definitely switch to using the Outlook.com one in Outlook 2010 with the above mentioned tool since from what you write MS seems to have canceled the feature that would allow you to share internal calendars (publish option).

The connector tool adds full Hotmail/Outlook.com support to Outlook 2010. I just upgraded my Outlook to 2013 so I'm not sure about specifics but in my opinion full support is included in 2013 without the need for the addon.
