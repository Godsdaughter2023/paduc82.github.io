markb384 say:

If the emails are random in nature or order though, unfortunately as Raju said, there isn't a way to just count up emails and get an alert when it reaches a number. For instance, if you are in the head office of a company and get an end of day report from 28 different branches in what ever order they send them, I can't see a way around this.

However, if they emails are always of the same nature and always come in the same order, which is likely to be the case if they are automated, you could look for some uniqueness in the last email you expect to receive (i.e. it might contain some sort of phrase or subject line the other emails do not have) and use this to trigger a sound, or even a mobile alert. This doesn't guarantee that you have received 28 emails, as there is no counting etc., but it does alert you to receiving the last email that you are expecting to receive, letting you know that there are no more emails to be sent.

don sorcinelli say:

The simple answer is "yes", it is usually possible. The configuration is always the hard part, however

 

If you set up your device for ActiveSync via Bluetooth, you would then have the ability to surf in the same way as you do with your USB ActiveSync.

 

For a "Bluetooth Router", there are two possibilities:

1) If your device supports PAN and your PC supports being a PAN service (in essence acting like a Bluetooth equivalent of a router), then you could pair that way.

2) There used to be some Bluetooth Acees Points on the market (I haven't checked in a long time). I don't know of any access point/router combinations, but I can't say for sure that they do not exist.

 

After all of this, I would recommend the WiFi approach over Bluetooth whenever possible as Bluetooth is crippled in most commerical versions by serial baudrates (115K) vs. the typically higher speeds that 802.11b and g provide.

john - project say:

te5s3rakt,

It sounds like your organization has a lot of document information beyond just the Project planning and scheduling and I'm wondering if Project Server wouldn't be a better match, since that is what Server is designed to address - the whole enterprise environment. However, I don't do Project Server so I only mention this as a point of reference.

That said, if I were tasked to do what you want I would use VBA. You could possibly use Visual Reports but there have been some issues with Visual Reports and it just doesn't have the depth of flexibility the VBA offers, at least in my opinion. You mentioned that you found some stuff on VBA but it was for earlier versions of Project. When it comes to Project VBA nothing major has changed since Project 98 except newer versions tend to add more objects, methods, properties, etc to the overall Project object library. Did you actually try to run the code you found? If so, what did or did not work for you?

The type of macro you need to export the desired data is pretty straightforward. You would use either the resource or assignment TimescaledData Method to extract the data. The syntax is:

expression.TimeScaledDate(StartDate, EndDate, Type, TImeScaleUnit, Count)

where "expression" is either a resource object or an assignment object. The other variables allow you to pick the date range (i.e. start and end), type (e.g. work), time unit (e.g. days, weeks, etc.)

Once the data is in Excel it is relatively easy to format any type of report you need. Personally I like to gather the Project data into a set of arrays, then open Excel, pre-format the report workbook, dump the arrays, and then tidy up. I like this method because you do everything you need to in Project, then you open Excel and finish everything there. There are many different methods for exporting Project data to Excel, including a direct read from Project, write into Excel, which bounces back and forth between the two applications. Yet another method is to dump Project data into a text file (e.g. CSV), and then open that file from Excel. This method essentially creates a copy of the report data in a separate file that can be accessed even if Project is not open or even present on a given user's PC.

If you want to learn Project VBA and want some text based help, I recommend Rod Gill's book on the subject. You can find out more at: http://www.project-systems.co.nz. If you already know some VBA but have specific questions, post those to this forum or it might even be better to post to the TechNet forum at: http://social.technet.microsoft.com/Forums/projectserver/en-US/home?forum=project2010custprog. Just don't cross-post to both forums.
