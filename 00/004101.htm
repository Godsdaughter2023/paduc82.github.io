    Next

a. user say:

re:  why is that?

Text aligns left in a cell and numbers align right.
The DIV/0! errors, even when replaced by values, are centered.
I get "Error 2007" as the value.

Your loop can check for both errors and non-errors and proceed accordingly...
'---
Dim rCell As Range

For Each rCell In Selection.Cells
   If IsError(rCell.Value) Then
     rCell.Offset(0, 1).Value = 11
   Else
     rCell.Offset(0, 1).Value = 12345
   End If
Next

'--- OR---

For Each rCell In Selection.Cells
   If rCell.Text = "#DIV/0!" Then
     rCell.Offset(0, 1).Value = 11
   Else
     rCell.Offset(0, 1).Value = 12345
   End If
Next

'---

apo__1 say: hoping you are not looking at the spreadsheet counting rows.... the ticks (on the graph!) are these little lines or dots that extend from the axis to give you a sense of the scale..

for example, a horizontal axis with 5 tickmarks could look like this:

 ------|--------|--------|--------|--------|------  

        0        50      100     150     200

you could get something like that with a "major unit" = 50

..but you can set the "minor unit" to 50 and the major to 100, for example to get :

 ------|--------|--------|--------|--------|------    

        0                   100                 200

...which looks cleaner.

 

A log/broken scale is out of the question then (since 50k increments should be visible)..

marysully say:

An easy way to create lines that you can type on is to create a one-column Table. Insert number of Rows in the one-column table depending on the number of lines needed. The lines can be adjusted by selecting the Table and dragging the column to the left on the ruler. You can then remove the side borders by selecting the Table, right click, click on Table Properties and then at Table tab click on “Borders and Shading” and remove side borders.  If you need to delete lines you have to select the rows, right click, and delete rows.
