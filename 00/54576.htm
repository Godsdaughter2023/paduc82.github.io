Assuming all is well, close Device Manager.  If the box at the bottom said, "Do not use this device (disable)", change it to enable and OK your way out.

Now, open a Command Prompt window (Start > Run > cmd > OK).

In the black Command Prompt window that opens, type the following line and press Enter.

     ipconfig /all > "%UserProfile%\Desktop\ip.txt"

Go to your desktop, open the file ip.txt using Notepad, copy and paste the contents into your answer.

a_user_here say:

From the lack of postings on this issue I'm assuming very few people are using virtual desktops in Windows 10, and even less so in Tablet mode.

I think you summed it up pretty much in your own words, but I would also add that if you look at what happens when you go into tablet mode, I get the feeling that they both wanting a tablet user to feel "at home" (working the way them might expect on a tablet) and also taking into account devices that might have small screens.

When you are in tablet mode mostly you work with one application for the whole screen.  So there would be very little difference between switching "desktops", and applications.  Now you can split the screen and get two applications on it, so there is some merit in having virtual desktops for that.  You will also notice that tablet mode restricts the splitting of the screen to two, not like the 4 for snapping in desktop mode.  Clearly they seem to be targeting touch friendly arrangements, and smaller screens.

So yes you are correct, there are no virtual desktops in tablet mode, and there is nothing to "fix".

If all you want is the full screen Start, you can change that under:

Settings -> Personalization -> Start -> Use Start full screen

You can also use the Windows Feedback app to suggest virtual desktops in tablet mode.

Just search for Feedback.
