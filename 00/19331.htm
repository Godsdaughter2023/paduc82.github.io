Hi techtiger,

Thanks for your feedback. Now we can be sure that the mailbox itself does not have any problem.

Based on the current situation, I have done some tests from my side. By default, the showInAddressBook attribute should contain three entries as it shows below. The second entry indicates that the user is included in the All Users directory, and the third one indicates the default GAL.


In your original post, you had mentioned that this attribute of the affected user is <not set>. That should not be a normal behavior. Have you or any other administrator changed anything about this user?

Moreover, I also tried to run Set-Mailbox -Identity *** Email address is removed for privacy *** -HiddenFromAddressListsEnabled $true. After that, the user was hidden from GAL and I could see that the showInAddressBook attribute had lost its third entry, which represented the default GAL. Therefore, I suggest you check whether the affected user has been hidden from GAL. You can run Get-RemoteMailbox -Identity *** Email address is removed for privacy *** | FL *Hid* to see whether the HiddenFromAddressListsEnabled attribute is True or False. And if it’s True, please use the Set-RemoteMailbox command to set it as False. (You need to use Set-RemoteMailbox instead of Set-Mailbox since the affected mailbox has been moved to Office 365. And you do not need to use Exchange Online PowerShell to do it. Just use the Exchange on-premises PowerShell.)
