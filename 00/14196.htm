jay freedman say:

Most of the sample code you'll find on the 'Net concerns either legacy dropdown form fields or dropdown content controls. Unfortunately, neither of those types of lists allows multiple selections. The only reasonable solution is to create a userform (essentially a custom dialog box), place the list boxes on that userform, and transfer the user's selections to the document body when they click the OK button. The kind of list box that's available in a userform lets you set the MultiSelect property to "fmMultiSelectMulti".

I've posted an example macro-enabled document, UFmultiselects, on my public OneDrive folder, which you can access with the link https://1drv.ms/w/s!Ak3EW6OhDndBgdca4x-oAYmRek8WWw. Download it to a folder on your PC; open the file's Properties dialog in File Explorer and check the box for Unblock. In the document you can click the boxed "Click to select keys" (a macrobutton field) to run the userform (assuming macros are enabled). You can start the macro editor (Alt+F11) to examine the userform's code and modify it as needed.

macropod say:

A number of possibilities come to mind as to why you might get a warning even if the document contains no Active Content:

1. A faulty Office installation

2. A corrupt document; or

3. A corrupt table in the document.

You could try repairing:
1. the Office installation, via Start > Windows Control Panel > Programs > Programs & Features > Microsoft Office (version) > Change > Repair;
2. the document by inserting a new, empty, paragraph at the very end, copying everything except that new paragraph to a new document based on the same template (headers & footers may need to be copied separately), closing the old document and saving the new one over it.
3. all tables in the document, by converting them to text and back again or by saving the document in RTF format, closing it, then re-opening it and re-saving in the doc(x) format ('repairing' the document, per se, doesn't fix this).
