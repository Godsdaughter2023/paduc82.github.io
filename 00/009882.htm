echo s say:

I would use File | Save As and save each slide as an image. Then I'd use Insert | Picture | From file and insert the images, then rearrange them on a blank slide.

You can also switch to slide sorter view and copy a slide thumbnail.

Then switch back to normal view, to a blank slide, and click Home | Paste Special | MS Office PPT Slide Object.

This pastes the slide onto the blank slide so it's editable and can be resized. Do this for all the slides in the poster.

ossiemac say:

Hello Andy,

There are 2 code examples below of resizing the table to accept another record plus how to reference the added row to add the data. The code works irrespective of whether the table has a totals row.

In the first example I have separated Resizing the table range and then Resizing the Table to the new range because I thought that it will be easier to follow.

In the second example I have condensed the Resizing of the Table so that the Range Resize and Table resize is all in the one line of code.

Notes on the code:

When a table is created, if only the column headers are initially included in the range, then the table appears to have at least one row in the DataBodyRange.

If the table is selected and Resize is selected then 2 rows are actually shown as the table range.

Unfortunately, as far as I can determine, this blank row is not able to be referenced in VBA code to test if it is empty. (VBA returns an error). If two rows are selected when initially creating the table then it works fine but one cannot always be sure of how the table will be created.

After another row is added to the table then the first row can then be referenced. Therefore the workaround is to add a row for the new data and after adding the data to the new row, test the first row for data and if empty then delete the row. This workaround works fine all of the time.

Private Sub CommandButton1_Click()
    Dim ws As Worksheet
    Dim lstObj As ListObject
    Dim rngTable As Range
    Dim rngCelNew As Range
   
    Set ws = Worksheets("Sheet1")
   
    Set lstObj = ws.ListObjects("Table1")
   
    '********************************************************************************
    'Code between the asterisk lines can be condensed. See the second example
    Set rngTable = lstObj.Range 'Assign the table range to a range variable

    With rngTable
        'Resize rngTable to the new size required for the table (eg. Rows count + 1)
        Set rngTable = .Resize(.Rows.Count + 1, .Columns.Count)
    End With

    'Resize the table to the new range size
    lstObj.Resize   rngTable
    '********************************************************************************
   
    'Assign first cell of the last row of the DataBodyRange to a range variable
    With lstObj.DataBodyRange
        Set rngCelNew = .Cells(.Rows.Count, 1)  'First cell of added row
    End With
   
    'Assign first value to the rngCelNew
    rngCelNew.Value = CDbl(Me.TextBox1.Value)   '1st cell of added row
    'Assign additional values to adjacent cells using Offset
    rngCelNew.Offset(0, 1).Value = CDbl(Me.TextBox2.Value)  '2nd cell of inserted row
    rngCelNew.Offset(0, 2).Value = CDbl(Me.TextBox3.Value)  '3rd cell of inserted row
   
    'Delete the first row if it is left blank.
    With lstObj
         If WorksheetFunction.CountA(.ListRows(1).Range) = 0 Then
            .ListRows(1).Delete
        End If
    End With
End Sub

digit grouping..

Sir, how to change the digit grouping in excel 2010 despite it was set to <hundred> in <number> group..
rick rothstein, mvp - excel say:

Here are two methods to format numbers in "Indian Style"...

Macro
===============
Sub IndianNumberFormat()
  For Each C In Selection
    C.NumberFormat = Trim(Replace(Format(String(Len(Int(C.Value)) - 1, _
           "#"), " @@\\,@@\\,@@\\,@@\\,@@\\,@@\\,@@0"), " \,", "")) & ".00"
  Next
End Sub

To install the Macro (which you would have to run manually), press ALT+F11 to go into the VB editor, click Insert/Module on its menu bar and then copy/paste the Macro code above into the code window that opened up. To run the macro, first select the cells containing values you want to change to Indian Style, then press ALT+F8, select IndianNumberFormat from the list and click the Run button.

Event Code
===============
Private Sub Worksheet_Change(ByVal Target As Range)
  Dim R As Range
  Set R = Range("A:A,C3:E10")
  If Not Intersect(Target, R) Is Nothing Then
    With Target
      .NumberFormat = Trim(Replace(Format(String(Len(Int(.Value)) - 1, "#"), _
                           " @@\\,@@\\,@@\\,@@\\,@@\\,@@\\,@@0"), " \,", "")) & ".00"
    End With
  End If
End Sub

To install the Event Code above (which will run automatically when numbers are entered into the designated cells), right click the name tab at the bottom of the worksheet where your Indian Style numbers will be, select View Code from the popup menu that appears, and copy/paste the Event Code above into the code window that opened up. Next, change addresses between the quote marks in this line of code...

Set R = Range("A:A,C3:E10")

to a valid range whose entries you want to format with the Indian Style. The above line of code, as written, will automatically change entries anywhere in Column A plus within any cell in the range C3:E10 to the Indian Style.
