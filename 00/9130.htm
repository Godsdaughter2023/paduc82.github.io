doug robbins - mvp office apps & services (word) say:
You will need to have the Title Page in its own Section with the header\footer in the second Section of the document (the one that then starts on Page 2) unlinked from those in the First Section.  You can then have the page numbering in the Second section start at i as required.

suzanne s. barnhill say:

See http://wordfaqs.mvps.org/FrontMatterRibbon.htm for general principles.

The only way to get the second page of a section to start with 1 is to start numbering at 0, which you can't do with roman numerals. To restart numbering at 1, you must insert a section break, but that is not necessary here because, although the folio (printed page number) starts on page 2, the numbering starts on page 1 (the title page), so the number ii on your second page is correct. There should be no unnumbered pages even though some of them do not display the page number.

BTW, "Link to Previous" applies only to headers/footers in different sections. There is no connection or link between the First Page Header/Footer and the primary Header/Footer. The First Page Header/Footer in Section 1 can be linked to (or unlinked from) the First Page Header/Footer in Section 2, and so forth. Linkage must be set for each of the three possible headers and three possible footers in each individual section.

doug robbins - mvp office apps & services (word) say:
The OP wants the second page to be numbered as i.   The only way to achieve that is to have it be the first page in a Section.

suzanne s. barnhill say:
Yes, if that is the actual requirement. But publishing convention is that the first page is i even if it doesn't bear a folio; my suspicion was that he had misunderstood the numbering requirements (you can have page numbers--i.e., folios--starting on the second page without having numbering starting on the second page).

Changing the default language when the insertion point is in a document based on the Normal template only affects future documents based on that template.

For existing documents, you have to change languages manually. Sometimes you can accomplish that by modifying your styles; in particular, you can modify the Normal style which is the "parent" style for many other styles in a document. Often, though, you will have to select text and manually apply the language you want to text by picking the appropriate language in the Language dialog box (Review | Language | Set Proofing Language).

Uninstalling a language removes the corresponding proofing tools but it has no effect on the language formatting applied to text.

The mention of columns suggests that the problem may be a longstanding bug that is occasionally seen in all versions of Word where the "Equal column width" box does not get checked and the width of the first column gets set to the full margin width of the page (or the second column gets set to 0). This makes the column width "floating," so that it adjusts to the width of the page (for example, if you change from landscape to portrait) instead of being fixed.
