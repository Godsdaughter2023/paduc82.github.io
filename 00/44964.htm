tryx3 say:

Ricky,

1  You do not always need to boot from anything else to restore a system image.  If Windows is working well enough to cope, you can restart into the Windows recovery environment [WRE] by using

  • Start menu,
  • Power item,
  • Restart - keep the Shift key pressed as you click on this.

2  If Windows is not working well enough or if you are restoring the image onto a new [blank] HDD, boot from a Windows 10 installation disk, a System repair disk or a Recovery drive. 

2.1  I always use an installation disk made on USB using http://windows.microsoft.com/en-us/windows-10/media-creation-tool-install  After the regional settings are done, you land on a screen with a big Install now icon and a smaller Repair your computer link, choose the latter.

2.2  You then see a WRE that is almost the same as you would see using the procedure in para 1.

3  In both cases, when you see the WRE & complete the regional settings that you have seen before, click on Troubleshoot, Advanced options, System image recovery.

4  I would hazard the guess from your description that the problem booting up was using a Recovery drive. 

  • If your mouse & keyboard are connected by WiFi or Bluetooth then that might be the cause of problems. 
  • If you have a built-in touchpad as well or if you have an old USB mouse knocking about then that will work and you can click on the Ease of access logo in the bottom-right to bring up the onscreen keyboard for any typing that is needed.

5  Starting with the imminent Fall creator's update [Ver 1709], the Windows system imaging facility is no longer going to be updated and will, at some currently uncertain future time, become incompatible with the latest Windows updates.    At that stage, any system images you have already made can be restored using an existing installation disk, System repair disk or Recovery drive but you won't be able to create new ones or [necessarily] restore them using any new installation disk, System repair disk or Recovery drive.

6  There are lots of alternative system imaging utilities. 

6.1  Many people recommend Macrium Reflect [free edition or paid-for edition] or the paid-for Acronis True Image.  Both are good tools.  You need to make their "rescue disk" on USB & boot from it to restore an image.

6.2  I use free editions of Acronis TI that are available for those with WD drives [Acronis TI WD edition] or Seagate / Samsung / Maxtor drives [Seagate DiscWizard]. 

  • These free TI editions cannot use scheduling to make images - you have to do it manually when you feel like it which I have no problem with. 
  • Both free TI editions have the potentially serious limitation that you also need to have their brand of drive itself. 
  • So if, for example, my WD drive failed I would have to replace it with another WD drive to be able to boot up for image restoration.  
  • If, for example, a Seagate-Samsung drive failed on my other computers, I would have less of a constraint because I could just connect one of my external hard drives during bootup as they are all Seagate-Samsung drives {even the cheap unbranded xHDDs turn out to be Samsung ones}.
  • I once did a free 30 day trial of the full version of Acronis TI and I made its rescue disk.  I can still use that to boot up with to restore system images on all my computers without any branding limitations even though the image I am restoring will have been made by the fee editions I have on each computer.

7  I think that's everything.  Let me know if I have left anything out or not explained anything well enough.

Best of luck,

Denis

Hi Jim,

What all troubleshooting steps have you performed?

The COM Surrogate is a fancy name for Sacrificial process for a COM object that is run outside of the process that requested it. Explorer uses the COM Surrogate when extracting thumbnails, for example. If you go to a folder with thumbnails enabled, Explorer will fire off a COM Surrogate and use it to compute the thumbnails for the documents in the folder. It does this because Explorer has learned not to trust thumbnail extractors; they have a poor track record for stability.

 

Explorer has decided to absorb the performance penalty in exchange for the improved reliability resulting in moving these dodgy bits of code out of the main Explorer process. When the thumbnail extractor crashes, the crash destroys the COM Surrogate process instead of Explorer.

Please try the steps mentioned below, if you have not tried it.

    Windows 10 came preinstalled on my computer, how do I find the product key?

    Press Windows key + X

    Click Command Prompt (admin)

    Enter the following command:

    wmic path SoftwareLicensingService get OA3xOriginalProductKey

    Hit Enter

    The product key will be revealed, copy the product key then enter it

    If you are unable to activate Windows 10 after all the above and you are certain you have the correct edition installed, then please attempt phone activation:
