andreas killer say:
Once the tab is moved from the original worksheet, chart number 21 (and only number 21) is changed to number 20.  There is already a chart named "Chart 20".  So in turn, I have two charts named "Chart 20".  This is problematic because part of my code (LATER ON) is to delete chart 20 and chart 21 at different intervals.

You are right, this is a bad behavior and unfortunatelly we can't change it.

But if your charts are stored in worksheets, we can use the ChartObject.ShapeRange.AlternativeText property to store an unique name. But be warned, if you place a chart as standalone sheet (and back to a worksheet later), this property is lost!

 

Click on a chart, run sub NameChart from below and name the chart as "MyChart". Move the chart manually to an other sheet if you like, then run sub Test.

 

Andreas.

 

Option Explicit

Sub NameChart()
  Dim CO As ChartObject
  Dim S As Variant
  If ActiveChart Is Nothing Then
    MsgBox "Select a chart before"
    Exit Sub
  End If
  If TypeOf ActiveChart.Parent Is Workbook Then
    MsgBox "This did not work for chart sheets"
    Exit Sub
  End If
  Set CO = ActiveChart.Parent
  S = Application.InputBox("Name for this chart:", "", CO.ShapeRange.AlternativeText, Type:=2)
  'Canceled?
  If VarType(S) = vbBoolean Then Exit Sub
  CO.ShapeRange.AlternativeText = S
End Sub

Function FindChart(ByVal AlternativeText As String, _
    Optional ByVal WB As Workbook) As Chart
  Dim CO As ChartObject
  Dim WS As Worksheet
  If WB Is Nothing Then Set WB = ActiveWorkbook
  For Each WS In WB.Worksheets
    For Each CO In WS.ChartObjects
      If StrComp(AlternativeText, CO.ShapeRange.AlternativeText, vbTextCompare) = 0 Then
        Set FindChart = CO.Chart
        Exit Function
      End If
    Next
  Next
End Function

Sub Test()
  Dim C As Chart
  Set C = FindChart("MyChart")
  If C Is Nothing Then
    MsgBox "Chart did not exists"
  Else
    MsgBox "Found in " & C.Parent.Parent.Name
  End If
End Sub

No problem Jerome,

Here are the following steps you should take to implement your element how you desire:

  1. Open the name manager (Ribbon>Formulas>Name manager) and change the formula behind the "Picture" name to =INDIRECT(Sheet1!$K$17)

aristokles say:

I am now able to answer my own question, thanks to a series of email replies from Nokia customer service, one of which was completely unhelpful and did not even respond to my query, but the third and last of which provided a solution that resolved the issue for me.  This is what worked for me:

 

1.)  Using Nokia Suite 3.4.49 and a USB cable, I backed up all my phone data and settings to a file on my computer.  In order to maximize speed of transfer and ensure a stable connection, this should be done via USB cable and not Bluetooth.  Fortunately, as my phone is fairly new, I did not have a lot of data to back up, but this stage could take a while if you do have a lot of data, so ensure the phone is fully charged at the start.  As the phone does not restart during the backup process, it should also be possible to ensure continuous power during backup by plugging in the charger, but I have not tested that and have no authority for it.

 

2.)  After a successful backup, I formatted the phone by dialling *#7370#.  NB and note of caution:  This is a fairly drastic step and doing it will clear data, messages, settings, and applications, and will return your phone to a similar state to when it was new, so should only be done after a successful backup and if everything else has failed.  It will not undo updates to the device software, but it might be necessary to redo some application updates.  After doing it and restoring the backup successfully, I also had to repeat the registration process for QuickOffice, and install as of new my Opera Mini 7 browser (meaning that browser settings and bookmarks all had to be entered anew - again not much trouble for me as my phone is newish, but those with a lot of bookmarks would want to ensure they had a record or were using Opera Link).  This also suggests that one might have to reinstall and / or re-register other third-party apps, and that could of course cause problems for some people.

 

3.)  After the format was complete and the phone had restarted, I used Nokia Suite 3.4.49 and a USB cable to restore the latest backup.  During this process, it is essential that the phone stay on, so again one would need to ensure the battery was fully charged.  It is also essential to maintain a stable connection (so use a USB cable and make sure it is securely connected at the start).  It is also essential not to turn off or use the phone while restoring.  After the restore had completed successfully, my messages, display settings, tone settings, and email account settings were all restored;  however, as mentioned in (2) above, I did have to re-register QuickOffice and reinstall Opera Mini 7.

 

4.)  I was now able successfully to restore the missing voice for English (UK) by transferring the downloadable file TTS_Martin-English_UK.SIS (available on the page linked by aspergerguy in this thread) and installing it.  This was the step that would not work before, so the format clearly resolved whatever was blocking installation.  The voice is now listed as installed the same way that other voices were when I tested the install process by trying a new language package.  I did not have to re-install the pre-installed English (UK) language package or the other pre-installed voice ("Ellen");  only the missing voice.
