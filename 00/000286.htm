ken sheridan say: One point which hasn't been addressed is just what the prefix signifies.  As the example you've cited uses your initials my guess is that it's the student's initials.  If so you should not store these either in a separate column (field) or as part of a structured key StudentID column.  To do so would introduce redundancy and leave the table open to the risk of update anomalies.

The correct way to handle this would be to have columns FirstName, MiddleName (or MiddleInitial), and LastName.  You can then use a conventional autonumber column as the StudentID primary key column, and show the structured value in a computed control in a form or report, or a computed column in a query using the following expression:

    Left(FirstName,1) & Left(MiddleName,1) & Left(LastName,1) & StudentID

There is now no risk of inconsistent data being entered inadvertently.

If there is a valid need for the numbers to be sequential (and this is rarely the case) then you would need to compute the number when a row is inserted into the table.  No doubt Scott's link has a solution for this, or you can find examples in the demo file CustomNumber.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

In my file it would be the 'Sequential Numbering' option would be the second you'd use.  This is derived from Roger Carlson's published method and includes the means to protect against possible invalid duplication in a multi-user environment where two or more users are inserting a new row simultaneously.  Note what one way in which my example differs from Roger's original is that the user is forced to save a record via a command button on the form.  The reason for this is that otherwise, if a user attempts to save a record by closing the form, the record would be lost in the event of a conflict with another user having saved a new record while the current user is entering their data.

dude112 say:

Upon further checking, it looks like I have found two ways to get the attachments to stay in chronological order.  First, click and hold your mouse in the folder to drag select the files (it can highlight the files in any order) and the selected files will be attached in chrono order.  This seems to be the fastest way to do it.  Alternatively, one can hold ctrl and select the files, but you must select them in order all except the file you want listed first - which be selected last.
