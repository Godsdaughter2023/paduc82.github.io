yusufat say:

Integration Features enables WVPC7 to share drives and resources. Open the virtual machine's settings and these options are available:

  1. Audio. Without integration, sounds played inside virtual machine are emulated using SoundBlaster 16 sound card. Audio output is quite scratchy and is unpleasant to play MP3s using that emulated sound card. With audio integration, sounds inside VM are routed through remote desktop protocol to your physical sound card. Audio output will be much better.
  2. Clipboard. Host OS and supported guest OS can share clipboard's contents.
  3. Printer. Applications inside virtual machine can print using your physical printer. This is useful if you have a parallel printer (LPT1). For USB printers, click menu "USB" on virtual machine display's toolbar then attach your USB printer to your virtual machine.
  4. Smart cards. (I don't have any smart card device, so experts please reply)
  5. Drives. Here guest OS can access and write files directly to your physical hard disks (or DVD drive or USB flash disks); depends on which physical drive(s) you have shared.

For dragging and dropping, until today (2 July here) is still not possible. I used to drag-drop files from my windows vista desktop to virtual machine's desktop (windows XP inside) like you do, but now we can't. Another approach is to share your drive(s) in integration features options described above.

After I shared my C: drive, a new "shared" drive appeared in my virtual Windows XP's [My Computer], that is "C on RYAN-PC" (name can be different on your PC). Whenever I need to copy some files from inside my virtual machine to my windows 7's desktop, I just open that shared drive, navigate to \\tsclient\C\Users\Ryan\Desktop\ , copy the files, and they will arrive at my windows 7 desktop.

Personally I agree that even this method is easier than copying files to shared network, drag-and-drop method is much easier.
