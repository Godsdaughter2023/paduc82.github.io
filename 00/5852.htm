graham mayor say:
Creating footnotes for the first occurrence of each of the words in the list is fairly straightforward, but doing the same for each page may not be possible because as you add the footnotes, the page lengths change and thus what is the first reference on the page will change. I can't think of a simple way to do this.

If you want to create a footnote for each first occurrence then  you can use the following function to turn your worksheet data to an array. Add the function (and the macro which follows) to a new module. Change the bold text to reflect your worksheet. http://www.gmayor.com/installing_macro.htm   

Option Explicit
Private RS As Object
Private CN As Object
Private i As Long, iRows As Long
Private iFN As Long
Private Arr() As Variant
Private sFindText As String
Private sReplaceText As String
Private oRng As Range
Private oFN As Footnote
Private iPage As Long
 Const strWorkbook As String = "C:\Path\WordList.xlsx"        'The path of the workbook
Const strSheet As String = "Sheet1"        'The name of the worksheet

Public Function xlFillArray(strWorkbook As String, _
                            strWorksheetName As String) As Variant
    strWorksheetName = strWorksheetName & "$]"
    Set CN = CreateObject("ADODB.Connection")
    CN.Open ConnectionString:="Provider=Microsoft.ACE.OLEDB.12.0;" & _
                              "Data Source=" & strWorkbook & ";" & _
                              "Extended Properties=""Excel 12.0 Xml;HDR=YES"";"

    Set RS = CreateObject("ADODB.Recordset")
    RS.Open "SELECT * FROM [" & strWorksheetName, CN, 2, 1

    With RS
        .MoveLast
        iRows = .RecordCount
        .MoveFirst
    End With
    Arr = RS.GetRows(iRows)
    If RS.State = 1 Then RS.Close
    Set RS = Nothing
    If CN.State = 1 Then CN.Close
    Set CN = Nothing
End Function

Then the following macro will use that array to replace the words and create the footnotes.


Sub ReplaceFromArray()
    xlFillArray strWorkbook, strSheet
    For i = 0 To UBound(Arr, 2)
        iFN = 0
        sFindText = Arr(0, i)
        sReplaceText = Arr(1, i)
        Set oRng = ActiveDocument.Range
        With oRng.Find
            .ClearFormatting
            .Replacement.ClearFormatting
            Do While .Execute(FindText:=sFindText, _
                              MatchWholeWord:=True, _
                              Forward:=True, _
                              Wrap:=wdFindStop) = True
                oRng.Text = sReplaceText
                iFN = iFN + 1
                If iFN = 1 Then
                    With oRng
                        With .FootnoteOptions
                            .Location = wdBottomOfPage
                            .NumberingRule = wdRestartContinuous
                            .StartingNumber = 1
                            .NumberStyle = wdNoteNumberStyleArabic
                        End With
                        .Footnotes.Add Range:=oRng, Reference:="", Text:=sFindText
                        oRng.Font.Underline = wdUnderlineSingle
                    End With
                End If
                oRng.Collapse wdCollapseEnd
                DoEvents
            Loop
        End With
    Next
    MakeGlossary
End Sub

The following macro called by the previous macro will list all the footnotes on a new page at the end of the document with the pages they occur on. You can format that last page as you wish.

Sub MakeGlossary()
   Set oRng = ActiveDocument.Range
    oRng.Start = oRng.End
    oRng.InsertBreak wdPageBreak
   
    For Each oFN In ActiveDocument.Footnotes
        iPage = oFN.Range.Information(wdActiveEndPageNumber)
        ActiveDocument.Range.InsertAfter vbCr & oFN.Index & vbTab & _
        oFN.Range.Text & vbTab & "Page " & iPage
    Next oFN
End Sub

If a field name isn't recognized by Word, it is usually treated as a cross-reference (REF) field, meaning that Word looks for a bookmark whose name matches the specified field name. If no bookmark name is found, you get the reported error.

andreas killer say:

How to use a 3rd excel addin in own recorded macro ?

When you record a macro, the actions inside the AddIn are only recorded if the AddIn supports recoding.

Technical insight into the details for developers:

This macro does nothing when you execute it:

Sub Test()
  Application.RecordMacro "MsgBox ""Hello World"""
End Sub

But you can start the macro recorder, execute the macro, stop the macro recorder and the recorded macro is:

Sub Macro1()
'
' Macro1 Macro
'

'
    MsgBox "Hello World"
End Sub

That means for you if the recorded macro does not include the steps that the macro really executes in Excel, your AddIn doesn't support recording.

Maybe the AddIn exports some public routines like the Solver AddIn...?

Open the VBA editor, click Tools\References and check the AddIn in that list. Click Ok and then press F2, select the AddIn in the Project/Library combobox and the public routines are now shown below.

But how, if and in which order the sub/functions can be called depends on the AddIn. To be sure you have to ask the developer of that AddIn.

Hi Darren,

This might be caused by a graphics driver issue. We appreciate the information that you've provided. Since the issue happened after the update, what we can suggest is that you update the graphic drivers as well and see if it helps. If the issue still persists, then try rolling the driver back before the issue occurred. Follow the steps below:

  1. On the search bar, type Device Manager and hit Enter.
  2. Enter admin credentials if needed.
  3. Expand one of the categories to find the driver name you want to roll back.
  4. Double-click on it to open Properties.
  5. Under Driver tab, click Roll Back Driver to roll the driver back to its previous version.

If it still doesn't work, try performing a system restore point to go back in time before the issue happened. Refer to this article and go to the Restore from a system restore point section to perform a system restore.

stephen boots say:

If Security Essentials is still installed and running on the machine post upgrade to Windows 10 - please execute the following command from an elevated command prompt (Search > cmd.exe > (Right-Click) > 'Run as Administrator')

MsiExec.exe /X{75812722-F85F-4E5B-BEAF-3B7DA97A40D5}

A dialog box will prompt you to continue.

This will start the uninstall and automatically enable Windows Defender.

 

From <http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-defender-only-partially-enabled/6b03889f-a14f-476e-8df2-337e6e94be06?page=2>

stephen boots say:

This is unfortunate. The steps I posted were provided by Microsoft to deal with the issue.

I'll provide the reply I had been using prior to this latest solution offered:

There have been several reports of the Windows 10 upgrade from Windows 7 with Microsoft Security Essentials installed has left MSE on the machine and disabled Defender. The uninstall procedures for MSE do not appear to work.

Security Essentials

In another thread,  truespace  reported that removing all registry keys related to Microsoft Security Essentials manually and disabling the startup service from services. While this worked, I suggest that you back up your data files and do a reset of Windows 10 which will turn your upgrade into a clean install.
