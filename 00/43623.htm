sdmike1974 say:

Why does my laptop run "scanning and repairing drive (\\?\Volume...)" for a few seconds during every boot.

The only fix I've found so far is to eliminate the 'boot check' by removing

BootExecute at:

\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager

Lots of hits with Google including this one:

https://www.tenforums.com/performance-maintenance/86791-windows-10-check-disk-every-startup-since-creators-update.html

But none worked, including this suggestion:

"Press the Windows key+X choose Command Prompt (Admin)
Here type chkntfs /X C: This tells Windows not to check the C: drive on bootup."

sdmike1974 say:

I had forgotten that you can use CHKDSK even if you do not have a drive letter.

So I used the volume GUID from my boot error message (see below). This seems to have fixed the boot scan error.

After entering the elevated command prompt (as an Admin) I typed the following:

chkdsk "\\?\Volume{6cc2e875-9be1-4080-a4d9-c76a47e5d59f}" /f

The very detailed information that was displayed on my command prompt screen was not totally clear. But I did notice, "Windows has made corrections to the file system."

BTW, contrary to what I (falsely) assumed earlier, the volume GUID did not appear to be for the C drive after all.

Problem solved. Voila!!
