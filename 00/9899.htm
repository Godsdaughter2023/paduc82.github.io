rohn007 say:

Since you say you have not done any customization, there is no need for you to follow the OfficeWatch instructions. You can use the newly generated templates.

A template is just a file that holds a bunch of setting for Word.  Word always needs a template to work with.  By default that is "NORMAL.DOTM".  Now that Outlook uses Word as it's editor it also has a default template, NORMALEmail.DOTM.  You can work with other "default templates" and you can also "attach" a different template to replace NORMAL.DOTM if you wish.

If that template file is not found when Word starts, it will generate one. 

In the list of names you provided you do not see the .DOTM because you have file name extensions hidden.  That is the (unfortunate) default Windows Explorer setting.

What happened with this update problem is that for some unfathomable reason the update decided to be nice to us and "save" our NORMAL.DOTM file from deletion by renaming it, adding the .OLD to the name.  So if you look at your list, you'll see that the existing templates were both created Nov 25.  Then you ran the Windows Update, and on Dec 18 you started Word and Outlook at which point they both regenerated their "missing" templates.

This business of deleting, or simply renaming the default templates is not an issue if you do not customize your installation. But if you add commonly used macros, or custom shortcuts or customized styles, then you would have noticed the missing template and would have said many rude words directed at MS for messing up your installation.
