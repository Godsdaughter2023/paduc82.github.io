julie sheets say:

You can have Project calculate assignment units by setting up the tasks as fixed duration and then entering the work estimate when you assign the resource.

For example:  a 6 month duration task with 200 hours of work - Project calculates 1.67 hours per day and a peak units of 21%. 

You may also assign the resource at a specific percentage. For example: if I know I need the resource to work on 3 tasks simultaneously, I cannot assign above 33.3% on each task.  You can either enter work and assignment units, and let Project calculate duration or fix the duration, enter work, and have Project calculate units.

Project will not automatically change the work on tasks based upon what other tasks the resource is doing.  It will also not automatically ensure the resource is working exactly 8 hours per day.

jay freedman say:
That's just a little more complicated. Let's say you assign the tag "first" to the first
content control. Then you can do this:
 
Sub AutoNew()
    Dim oCCs As ContentControls
    Set oCCs = ActiveDocument.SelectContentControlsByTag("first")
    If oCCs.Count > 0 Then
        oCCs(1).Range.Select
        Selection.Collapse wdCollapseEnd
    End If
End Sub
 
and similarly for AutoOpen().
 
Some notes on this:
 
The SelectContentControlsByTag function is case-sensitive, so the statement above won't
find the control if the tag is "First".
 
The tag values of content controls don't have to be unique within a document, so the
function returns a ContentControls collection that may have zero, one, or more members;
the If statement checks that there is at least one control in the collection and then
chooses the first one. It's up to you when designing the form to ensure that there's only
one control with the specified tag value.
 
In a content-control-based form, I wouldn't expect any legacy form fields (they don't play
well together). You could insert a bookmark surrounding the content control and do
something like this:
 
    If ActiveDocument.Bookmarks.Exists("first") Then
        ActiveDocument.Bookmarks("first").Select
        Selection.Collapse wdCollapseStart
        Selection.MoveRight wdCharacter, 1
    End If

jay freedman say:
In a legacy protected form, the fact that the cursor starts in the first form field is
just a byproduct of the fact that the protection prevents the cursor from being placed
anywhere outside the form fields. Try clicking in in the text of a protected form, and the
cursor simply jumps to the next form field. This also happens when the document first
opens, except you can't see the fraction of a second that the cursor is at the first
character of the document before it jumps to the form field.
 
In a content-control-based form protected with grouping, the cursor can be placed anywhere
in the document; you just can't  /edit/ anything except inside the content controls.
There's no automatic forcing of the cursor to the first or next content control. Tabbing
through the controls will work only if the cursor starts in a control (and not even then,
if the control is a Rich Text one).
 
To start the cursor in the first control, add a macro named AutoNew and one named AutoOpen
to your template, both with the same contents:
 
Sub AutoNew()
    ActiveDocument.ContentControls(2).Range.Select
    Selection.Collapse wdCollapseEnd
End Sub
 
Sub AutoOpen()
    ActiveDocument.ContentControls(2).Range.Select
    Selection.Collapse wdCollapseEnd
End Sub
 
Note that the Group command creates a content control around the entire document (or
whatever part of it you protected) that becomes ActiveDocument.ContentControls(1). That's
why the macros refer to ActiveDocument.ContentControls(2). Be sure to test your template
while both creating new forms and reopening ones already created from the same template,
and adjust the number if necessary.

macropod say:

You can use the following process to create a dropdown selection list without the need for formfields or content controls:
1. Insert an AutoTextList field into the document or its template, by pressing Ctrl-F9 to create a pair of field braces (i.e. { }), then filling in between them, thus -
‘{AUTOTEXTLIST "Name List: Right-click to select a name" \s ListEntries \t "Right-click to select a name"}’
This field code: names the field 'Name List: Right-click to select a name'; uses the \s switch to specify a new Style named 'ListEntries'; and uses the \t switch to provide some hover-tip text.
Note: If you insert the AutoTextList field into a document, Word will save it to that document’s template by default.
2. When you’re done, press F9 to update the field, which will look like:
Name List: Right-click to select a name.
3. Create a list of names, for example:
St Matthew, St Mark, St Luke, St John, St Jude, St Peter, St Paul
Each Office Addresses can have multiple lines.
Note: The maximum length of an AutoText entry is 32 characters.
4. Select the paragraph(s) of list entries and assign a Style name ‘ListEntries’ to it.
5. Select each name from the list in turn and press Alt F3>OK or choose Insert|AutoText|New>OK. You can then delete the list.
6. Now, right-clicking on the field will allow you to select one of the names (which appear in alpha-order) from the list.

You can, of course, use different strings/names for the field variables. If you need to, you can add more names to the list by repeating steps 3-5. Similarly, you can use Insert|AutoText|AutoText to delete unwanted entries.
