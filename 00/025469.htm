Outlook 2003 uses a single PST for all of its POP accounts within a single mail profile and unless you specifically created folders to contain the sent items for the non-default accounts and rules to move those outgoing message to those folders, there is exactly one Sent Items folder.

brian tillman [outlook mvp 2007-2019] say:
Ah.  I understand.  Notice that your rule says to "move a copy" of the item, not "move" alone.  If you want the message to move ONLY to the extra folder you created, then you must disable the option in Outlook to save outgoing mail in Sent Items, provide the rule you've provided, make sure "stop processing more rules" is an action on that rule, then provide one more rule after that rule that moves all outgoing messages to your main Sent Items folder.  In other words, you must save all outgoing messages explicitly with rules and not rely on Outlook's option to save outgoing items, because that save will happen in addition to whatever save your rule performs, thus giving you a copy in your own folder and in the default Sent Items folder.

ronrosenfeld say:

The Spaces:

To add a space before and after each word within the array constant, you can either do that for each word individually:

   {" cat "," dog "}

OR you can add a space before and after the array constant itself (handy if you have a bunch of words within the array constant, probably not if you just have two:

    " " & {"cat","dog"} & " "

Both forms resolve to the same.

The reason for the spaces there, and also at the beginning and end of within_text, is so that if find_text exists at the beginning or end of within_text, the spaces will be there so it can be detected.
