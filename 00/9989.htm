It sounds like you are modifying the style so that the change is appearing where ever the style is used.

One way that this could happen is the "offending" style has the "Automatically Update" attribute turned on.  Turn the attribute off.

1.1  MACRO to disable “Auto Update” ALL Styles

NOTE:          The Automatically update document styles under Tools/Templates & Add-ins means that no matter what changes you make to style definitions (NOT direct formatting changes) will NOT stick.  Every time you close & reopen the document (& possibly when you save, I can’t recall), the styles from the attached template are copied to the document & all your style changes are lost.

The automatically update under the Format/Style/Modify means that when you add any direct formatting to a charcter or paragraph, the style definition is changed to reflect your addition.  Therefore, all styles with that name in the document change.

By default in NORMAL.DOT MS creates some styles with “AUTO UPDATE STYLE” option turned on.  So whenever a change is made to format of text using that style, all uses of that style are changed also.  This macro will scan thru all styles and turn of the option.  Most useful when you have to recreate NORMAL.DOT.

Public Sub RemoveAutomaticallyUpdate()

' RemoveAutomaticallyUpdate Macro

' Created by Gary Frieder

'Dim aSty As Style

For Each aSty In ActiveDocument.Styles

    If aSty.Type = wdStyleTypeParagraph Then

    aSty.AutomaticallyUpdate = False

    End If

Next aSty

End Sub

Leaving cells blank when using formulas - Microsoft Community

cass5181 say:

Leaving cells blank when using formulas

I am trying to create a database and have a date formula  =DATE(YEAR(N5),MONTH(N5)+3,DAY(N5)) to add 3 months to a free text date. I want to apply the formula to the specified cells and columns but I don't want it to display a defalt result if the source cell is blank. At the moment it displays "Mar 1900" which is the format I want it to be in but I dont want a column full of "Mar 1900". can it be done?

Cheers

t. valko say:

I am trying to create a database and have a date formula  =DATE(YEAR(N5),MONTH(N5)+3,DAY(N5)) to add 3 months to a free text date. I want to apply the formula to the specified cells and columns but I don't want it to display a defalt result if the source cell is blank. At the moment it displays "Mar 1900" which is the format I want it to be in but I dont want a column full of "Mar 1900". can it be done?

Cheers


You can test the cell to see if it contains a date (a number) and if not, then return a blank.

=IF(COUNT(N5),DATE(YEAR(N5),MONTH(N5)+3,DAY(N5)),"")

Also, you might find the use of the EDATE function more accurate.

=IF(COUNT(N5),EDATE(N5,3),"")

For example, consider this date: 3/31/2010

You want to return the date 3 months from that date. Your DATE formula will return 7/1/2010 and the EDATE formula will return 6/30/2010.

Essentially, the DATE formula is simply adding 3 months to the date. However, there is no 6/31/2010 so Excel "compensates" by returning the next valid date which is 7/1/2010. However, the true date that is 3 months from the start date of 3/31/2010 is 6/30/2010.

Note that the EDATE function requires the Analysis ToolPak add-in be installed for Excel versions prior to Excel 2007.

If you have Excel 2003 and earlier and you try the EDATE function but get a #NAME? error see Excel Help on the EDATE function and it'll tell you how to correct the problem.
