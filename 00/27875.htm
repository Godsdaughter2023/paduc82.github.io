Sub UnlinkFields()
    Dim doc As Word.Document

For the position of the closest value array enter:

=MATCH(SMALL(ABS(L2-N2:S2),1),ABS(L2-N2:S2),0)

For the closest value array enter:

=INDEX(N2:S2,MATCH(SMALL(ABS(L2-N2:S2),1),ABS(L2-N2:S2),0))

Array enter means that you enter the formula with CTRL + SHIFT + ENTER, not just with ENTER.

wayne-i-m say:

Change the name of the control to

txtDate

and change the reference to the control in the macro to show the new name

 

like this after update - this is VBA so use the AfterUpdate event (code)

Me.Parent!NavigationSubform.Form.txtDate.requery

So you are pointing to
Me.  (the subform)
Parent. (the main form where the subform Me is)
NavigationSubform.   (the name of the control on the parent form)
Form (you are saying that the control is a form)
txtDate.  (the control on the subform refered to)

 

Oh - the reason for not using Date as a control name is that it is used by Access to mean "this" date (ie.  today's date)

So if you have a code that says

txtMyControl = Date

then you will get 19/09/211 in the control

If you want to add the content of a control called txtDate into txtMyControl you would use

txtMyControl = txtDate (this would simply copy the contents of one control into another)

 

Hope this helps

As I said in my earlier post, you need to join on both the SctID field and the ChID field.  Try these queries:

qry_PaymentsForCharges

SELECT Charges.SctID.Value As SctID, Charges.ChID, Charges.ChDate, Charges.ChDescript, Charges.ChAmt, Payment.PayID, Payment.PayDate, Payment.PayAmt, Payment.PayNote
FROM Charges INNER JOIN Payment ON (Charges.SctID.Value = Payment.SctID) AND (Charges.ChID = Payment.ChID);

qry_ChargesForEachScout

SELECT Scouts.SctID, Scouts.LastName, Charges.ChID, Charges.ChDate, Charges.ChDescript, Charges.ChAmt
FROM Scouts INNER JOIN Charges ON Scouts.SctID = Charges.SctID.Value;

qry_PaymentsByScoutCharges

SELECT qry_ChargesForEachScout.*, qry_PaymentsForCharges.PayID, qry_PaymentsForCharges.PayDate, qry_PaymentsForCharges.PayAmt, qry_PaymentsForCharges.PayNote
FROM qry_ChargesForEachScout LEFT JOIN qry_PaymentsForCharges ON (qry_ChargesForEachScout.ChID = qry_PaymentsForCharges.ChID) AND (qry_ChargesForEachScout.SctID = qry_PaymentsForCharges.SctID);

Please note the changes in the first two queries.  (Note: I didn't actually set up a multivalue field for this, so I may not have the syntax exactly right to explode it into the joining records.  Let me know if those queries don't work as written.)

With those changed queries in place, see if the third one, qry_PaymentsByScoutCharge, doesn't give you what you want.  Be aware that I didn't set this up to return anything for scouts that have no charges at all.  That can easily be changed by using a left join instead of an inner join in qry_ChargesByScout.
