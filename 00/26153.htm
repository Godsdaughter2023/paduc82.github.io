If it is happening with just one file, chances are almost certain that there is something wrong with that file.

And chances are, it's tracked changes.

I don't believe it's the plist (which will be there, in Library/Preferences, but if other files are OK, that's not the issue).

What I recommend first is to remove all the tracked changes if there are any. Just "Accept all changes in document".  You can't continue collecting changes forever in a Word document: ultimately the presence of the changes will eventually break the document.

Then save it to the "other" file format.  (If it's a .doc, save it out to .docx.  If it's .docx, downgrade it to .doc). That forces Word to re-express all of the code in the file, while doing so it should omit the bad bits.  Then if you downgraded to .doc, re-save that as .docx again so you can continue working.

Your crash dump doesn't tell us anything because you cut off the details of what "Thread 0" was doing, but even if they were there, chances are all it tells us is "Word crashed".  Not helpful.

Try re-saving to a different version and see if that cleans it up.

Post back with your results: there are other things we can try if that doesn't fix it.

amit tandon say:

Refer below image:

Formula in cell B2:

=INDEX($B$3:$B$12,MATCH(NOW()-TODAY(),$A$3:$A$12))

Alternately, formula in cell C2:

=LOOKUP(2,1/(NOW()-TODAY()>=$A$3:$A$12),$B$3:$B$12)

Both the above formulas have the same effect. They match the current time "NOW()-TODAY()" to the latest (or last) time in column A (if latest time is same as current time or the latest time less than current time) and return the corresponding value from column B. The above has been captured at current time after 1:30 PM. If the current time was 1:15 PM, then the value of 22 would have been returned.

doug robbins - mvp office apps & services (word) say:

As I said, to set the position at which the Horizontal Axis crosses the Vertical Axis, you need to access the options for the Vertical Axis and vice-versa.

In Excel 2016 with chart selected, access the Chart Area dropdown in the Current Selection section of the Chart Tools>Format tab of the ribbon.

From that dropdown, select Vertical (Value) Axis

The Vertical Axis will then be selected and then by clicking on the Format Selection item in the Current Selection section of the ribbon

you will see the task pane for formatting of the Vertical Axis, in which you can set the position at which the Horizontal Axis crosses the Vertical Axis
