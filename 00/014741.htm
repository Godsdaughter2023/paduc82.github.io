Decimals in Access - Microsoft Community

robert_v say:

Decimals in Access

I have a Currency field that is a calculated one. The number in there is for example 1.2368. these fields have been filled in over time and I want to convert them to a rounded fixed 2 decimal number so that my future equations will be better. How can I do that?

dirk goldgar say:

This is a calculated field, so it's defined by an expression in the table design?  If so, you could use the Round() function in the expression to round the result to 2 decimal places.

Note that this will naturally introduce a greater amount of rounding error than when the result was being implicitly rounded to 4 decimals, the maximum precision of the Currency data type.

robert_v say: Thank you!

ken sheridan say: As Dirk says, you need to call the expression which computes the value within the Round function.  Taking a simple example of computing a gross price of a product using the old UK standard rate of VAT of 17.5% you can see how it works in the debug (aka 'immediate' window):

NetPrice = 1.25
VATRate = 0.175
? NetPrice*(1+VATRate)
 1.46875

As you can see this returns a value to 5 decimal places.  If the expression is then called within the Round function:

? Round(NetPrice*(1+VATRate),2)
 1.47  

It returns a value to rounded to two decimal places.  Access uses the 'round half even' algorithm (banker's rounding) by default BTW.

Think carefully about whether you really want to do this, however.  The precision of the Currency data type to 4 decimal places is designed to suppress cumulative rounding errors.  In the above example, if it’s the retail price of a product which is being computed, rounding per item would be correct.  If, on the other hand, an invoice is being generated in a normal commercial environment, then the VAT is on the invoice total, so you would not wish to round each gross price, but only the aggregated sum, as to do the former could lead to an incorrect VAT amount for the invoice being computed due to cumulative rounding errors.
