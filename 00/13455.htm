david dickinson say:

SOLVED: Cannot activate Microsoft Office 365

SUMMARY: A fully updated installation of Microsoft Internet Explorer 11 on Microsoft Windows 8.1 failed to process the web site at www.office.com/setup correctly, preventing successful activation of Office 365. Installing Mozilla Firefox solved the problem.

PROBLEM: A client purchased Office 365 Home Premium from Best Buy. I needed to install it on a brand new and fully up-to-date Windows 8.1 machine with Internet Explorer 11. The instructions said to go to www.office.com/setup. There, I was instructed to enter the 25-character product key, which I did, noting that the cursor did no automatically advance to the next field when I had filled a field with five characters. After clicking the "Continue" button, I was instructed to log in with the relevant Microsoft account, which I did. After clicking "Continue" again, I was instructed to select the country (United States). The language selector was disabled. I read the FAQ, and it said that for the version of Office 365 that I was installing, I would not be able to select the language at that time. However, when I tried to click the "Continue" button to activate the subscription, nothing happened. The "Continue" button was not disabled, but clicking it did nothing at all.

SOLUTION: I called Microsoft Office 365 support (I wasted several hours trying to get into the chat, but there were never any techs available), and got a call back within 10 minutes. The tech connected to the machine via LogMeIn and saw what I saw. He then said there was a problem either with Windows or with Internet Explorer, and passed me off to Windows support. The tech that I spoke with there was as mystified as I was about why the first tech had sent me there, and reconnected me with a different technician at Office 365 support. That tech suggested trying another web browser. I installed Mozilla Firefox 29, and started the activation process again at www.office.com/setup. This time, when I entered the product key, the cursor automatically advanced to the next field, and the language selector was available, so I selected "English". After clicking "Continue", the product was activated successfully.

I offered to work with Microsoft's Internet Explorer and Windows tech support team to diagnose the root problem and find a solution that would allow using Internet Explorer, but I was told that since the computer was more than two weeks old, I would be charged a support fee. Since my problem was solved, I decided that helping Microsoft was not in my interest if they were going to charge me to do so.

Convert formulas that use Named Ranges to use cell references

Having just looked at another thread (http://answers.microsoft.com/en-us/office/forum/office_xp-excel/make-a-list-of-the-named-cells-defined-names-or/ec43beca-9704-44d5-9d2f-ae2b8d771116) I started playing with Named Ranges a bit more and it got me thinking about a potential issue I think I'm going to hit in the near future.

In Excel 2007, I found that we can use the Apply Names... button (hidden in the drop-down of the Define Name split-button) to replace references to cells with the Named Range that references that same cell.

So for example, say the Named Range "Fred" refers to Sheet1!$A$1:$A$5, and cells A1:A5 contain the numbers 1 to 5, then put in cell B1 the formula =A1*2 and drag it down to cell B5.

If we select B1:B5 (or the whole sheet) and click Formula tab Defined Names group dropdown next to Define Name Apply Names... with the default settings applied, if we select Fred from the list, the formulas in B1:B5 all get changed to =Fred*2

Now this, I thought might be quite useful, one day....

Why would I want to delete all my Named Ranges? Well it's a pain if I have a large sheet of data, where each column is named, if I want to copy a column to another sheet to do some calculations with it, or plot it against data from another test, and if a column with the same named range already exists, I get the following warning:

A formula or sheet you want to move or copy contains the name "Fred", which already exists on the destination worksheet. Do you want to use this version of the name?

So, I'm thinking, once I've finished with my Named Ranges (they are used in some VBA code for setting-up the data initially), we could delete them all and not have to go through the above pain.

jeeped say:
You should be able to do this with a macro that involves a couple of For Next statements and a couple of If Then statements.

With Excel open to the workbook that you want to remove defined Name references from, press Alt+F11. Immediately use the pull-down menus to Insert, Module. In this new pane titled something like Book1 - Module1 (Code), paste in the following,

Sub mcrRemoveNamesFromFormulas()
    For Each n In Names
        For Each c In Range("A1", ActiveCell.SpecialCells(xlLastCell))
            If c.HasFormula Then _
                If InStr(1, c.Formula, n.Name, vbTextCompare) > 0 Then _
                    c.Formula = Replace(c.Formula, n.Name, Mid(n.RefersTo, 2, 999))
        Next c
    Next n
End Sub

Press Alt+Q to close the VBE and return to Excel. Run the macro through the Developer tab's Macro commands.
There could be a heap of error control added to make this safer, e.g. it doesn't discern the difference between Current_Month and "Current_Month", but depending upon your formulas and the naming conventions you've applied to your defined names, this should provide a starting point. It does check to see it it is looking at the value of a cell or the underlying formula that calculates a cell's value.
