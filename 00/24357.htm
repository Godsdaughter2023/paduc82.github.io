PowerPoint O365 Copy/Paste of Grouped Objects Adds Unwanted Space - Microsoft Community

kkkwj say:

PowerPoint O365 Copy/Paste of Grouped Objects Adds Unwanted Space Outside of 8.5x11 Slide Size

This is the third ticket (for three different situations) for PPT creating unwanted extra space around objects.

I am trying to use PPT as a poor man's Illustrator to create graphic images for page backgrounds, headers, and footers in Word. I could create the same shapes in Word, but it's way easier to work with the shapes and text in PPT.

In this case, I created some simple shapes in PPT on a canvas 8.5x11inches in size. 

I carefully checked the position of all shapes to make sure they did not hang off the 8.5x11 canvas.

I selected all shapes with Ctrl-A, and then grouped them so I could move them as one object.

I copied the group, moved to Word, and pasted the group.

Presto! The group was wider than 8.5in. It was impossible to align the grouped object with both the left and right edges of the 8.5x11 Word page, because the grouped object was now bigger than 8.5x11. A narrow 1 or 2 pixel blank space was visible down the left side of the page.

So the problem is that PPT grouping creates extra space around objects. Not usually obvious or a problem when the objects are smaller than the page and not aligned with the page edge. But a big problem if you want to align a grouped object to the edge of a page.

WORKAROUND

The workaround was to ungroup the objects. This removed the extra blank space on the left side. I was able to successfully align the objects in Word, to achieve my goal. 

But things are really touchy when you ungroup half a dozen objects like that. They are all still selected after ungrouping, but any false move will unselect the group. That forces you to try to reselect them all, which can be difficult if you don't have the wrapping, positioning, and size values all set exactly correctly.

Good luck. Hopefully the devs can go in and fix all of these PPT extra space issues.

MasterTab - H24 has Inv#3662189

Hile

jeeped say:

... Is there an easy way to use Index/Match to get a Yes or No in my results column on the MasterTab ...


Assuming that the InvNo column is column B on each of the 4 lookup worksheets, you could nest a series of IFERROR(...) functions, passing into the next MATCH(...) if #N/A is returned. The final result is evaluated as either Yes or No. Something like,

=IF(ISNUMBER(IFERROR(MATCH(H24,Sheet1!B:B,0),IFERROR(MATCH(H24,Sheet2!B:B,0),IFERROR(MATCH(H24,Sheet3!B:B,0),MATCH(H24,Sheet4!B:B,0))))),"Yes","No")

You only require MATCH(...) instead of an INDEX(MATCH(...) pair as you only want to test for the existence of the InvNo, not to return a value.
