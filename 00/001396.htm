You'll need a field coded like:

{QUOTE{SET Den {={=txt2>0}+{=txt3>0}+{=txt4>0}+{=txt5>0}+{=txt6>0} \# 0}} {IF{Den}= 0 0 {=(txt2+txt3+txt4+txt5+txt6)/Den}}}

doug robbins - mvp office apps & services (word) say:

It is better to eliminate the cause of the error.  In many cases that can simply be done by appending 

& " "

to the value that is being assigned to the variable.  Then if the value is null, the variable will contain a single space.

Depending upon the location of the docvariable field in the document, in some cases, it may be more appropriate to add the space before the value.  If you cannot tolerate a space being included in the result of the docvariable field, then in the code you would need to use and if then else field to check if the value is null and set the variable to a space if it is.

    Excel templates all in Russian!

    socaldavid say:

    The solution to this problem lies in the registry. Run regedit (Start button, then type in the RUN box "RegEdit" (without the quotes) and hit ENTER). When RegEdit starts, click on the EDIT Tab and click on FIND (Or you can hit CTRL-F). For the search string, type in "ru-ru" without the quotes and then click the FIND NEXT button. Make sure that you click on COMPUTER in the tree structure in the left pane before you do your search so that your search begins from the beginning of the registry file. As mentioned in earlier posts, when you are doing anything in the registry, make a backup before you proceed.

    If you know how to navigate around in RegEdit, you can simply go to the registry key where these Russian files can be found and delete at:

    HKEY_CURRENT_USER/Software/Microsoft/Office/15.0/Common/Internet/WebServiceCache/AllUsers/clienttemplates.content.office.net

    Just delete all the registry file name entries which contain "ru-ru" in the as opposed to "en-us".

    I would also delete all of the "ru-ru" files in C:\Users\USERNAME\AppData\Local\Microsoft\Office\15.0\WebServiceCache\AllUsers\clienttemplates.content.office.net

    Exit Regedit and then run your program again. The Russian templates should now all be gone and only the blank template will show.

    I hope this helps. Good Luck!

    One way to wipe the drive is to boot with Windows 10 install media. From the first screen of Setup that loads, use SHIFT-F10 to get a Command Prompt.  Then run diskpart:

    select disk 0

    detail disk

    clean

    exit

    exit

    That will totally wipe all partitions from disk 0, leaving no OS and no partitions at all. This assumes that disk 0 has no info that you need.  Then you should be able to continue with Setup normally.

    The detail disk command should give enough info that you can tell if it is the intended disk to wipe.

    However, deleting every disk partition from the GUI interface of Custom Windows setup should accomplish the same results.

    Linest Function in latest Excel 2016 update on Mac is not working - Microsoft Community

    anneamos1 say:

    Linest Function in latest Excel 2016 update on Mac is not working

    I can no longer enter Linest function in 2016 Excel on a Mac. Entry with control+shift+enter does not produce an array result of slope and intercept. It does not produce an array bracket around the formula so only slope is shown.  What has happened ?


    anneamos1 say:

    I found the answer after swapping several keyboards among 2 Macs running Excel 2016.

    If you are using an extended Mac keyboard (the wired version) you will fail to get an array function to work if you use Microsoft's instructions to enter the formula with Shift+Control+Enter. In all earlier versions you could successfully use the "Enter"  key at the bottom of the numeric keypad.

    That is no longer the case. You must use Shift+Control+Return (the one on the "typewriter" part of the keyboard). Apparently, the "Enter" key and the "Return" keys are being interpreted differently in the latest update.     If the instructions say "+Enter" just use "+Return"
