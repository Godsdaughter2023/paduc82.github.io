MS Word Breaks OneDrive Business for Mac - Microsoft Community

ianscott-fleming say:

MS Word Breaks OneDrive Business for Mac

Have just switched all my teaching materials from DropBox to the university's OneDrive for Business cloud system.

OneDrive Business for Mac stops sync'ing if one has a file for folder whose name ends with either "_file" or "_files".  Logs an error "A file problem is blocking all uploads.  The name or type isn't allowed..."

(Curiously, this is not a problem for vanilla OneDrive...just for "OneDrive for Business".)

I suppose this can't be called a bug because Microsoft does say "..._file" and "..._files" are not allowed:  https://support.microsoft.com/en-us/kb/3034685 .  Problem is that MS Word creates a folder called "XXX_files" when one saves a Word document called XXX as html.  It places all graphics, along with some metadata, in this subfolder, and places references in the html to load the graphics.  Since the main .htm file contains URL references to files in the "XXX_files" folder, it can't be renamed (unless one wants to go in and manually edit the .htm file and change all the references.)

Do the MS Office folks and the OneDrive folks at Microsoft not talk to one another?  Are they not aware that one of their products breaks OneDrive for Business?

I write all my assignments for my students in MS Word 2011 for Mac, and save them as html so students can access them via the web.  

Anybody have a workaround for this?

ianscott-fleming say:

Thanks for the info.

Have opted for the following bash-script to rename the folders/undo the renaming as needed, and run the script from my crontab.  This won't work on a Windows machine, but if I understand your reply, this is not an issue on Windows machines, so no workaround is needed there.  This allows all my files to be backed up, including those with unsupported filenames.

My solution only fixes the problem filename endings that are a problem for me, but the entire list of unsupported could be added to line 23 if needed.

ian

#!/bin/bash

#

#   script to rename files and folders which break MS Office OneDrive for Business on Macs

#

#   See  support.microsoft.com/.../3034685

#

#   Renames files and folders with wrong extensions by adding extension ".OneDrive_unsupported"

#   or removes that extension from existing folders to undo the changes

#

#   logs all renaming to logfile

#

#  

#............defaults and run params

basedir="/Users/iscottfl/OneDrive_ttu"      # for me, symbolic link to OneDrive without spaces in the name

logfile="docx_dir_rename.log"

       #a complete list of blocked extensions is at support.microsoft.com/.../3034685

       #I'm just fixing the ones that are a problem for me.

bad_exts=( "_file" "_files" )

ext_flag=".OneDrive_unsupported"         # extension to append

#....................................

usage()

{

       cat << EOF

usage: $0 [-r] [-d basedir] [-h]

       -h          help (this output)

       -r          remove extension

       -d basedir  process folder 'basedir' instead of $basedir

EOF

       exit 1;

}

#..............parse options on command line

while getopts "hrd:" OPTION

do

       case $OPTION in

               h)

                       usage

                       exit 1

                       ;;

               r)

                       do_remove=1

                       ;;

               d)

                       basedir=$OPTARG

                       ;;

               ?)

                       usage

                       exit

                       ;;

       esac

done

#...........................................

cd $basedir

       # renaming unsupported files and dirs:

if [[ ! $do_remove ]]

then

   echo "Renaming unsupported filenames and folders";

       # for each unsupported name:

   for ext in "${bad_exts[@]}"

   do

       bads=( $(find . -name "*${ext}" -print) )       # create bash array of bad filenames

       printf "%s : %d bad names found\n" $ext ${#bads[@]}     # report to user

       printf "\t%s\n" "${bads[@]}"

       printf "\n"

       if [[ ! $bads ]]                    # nothing found.  put timestamp in log file

       then

           printf "%s %s\n" "$(date +'%Y-%m-%d %H:%M')" $ext >> $logfile

       else

           for dir in "${bads[@]}"         # for each bad filename

           do

               newname="${dir}${ext_flag}" # append extension flag    

                                           # log command to be run

               echo "$(date +'%Y-%m-%d %H:%M ' ) mv \"${dir}\" \"${newname}\" " >> $logfile

               mv "${dir}" "${newname}"    # rename the file

           done

       fi

   done

else

       # Undoing previous renames

   echo "Resetting renamed OneDrive-unsupported filenames and folders"

                                           # find all renamed files & folders

   bads=( $(find . -name "*${ext_flag}" -print) )

   printf "%s : %d bad names found to reset\n" $ext ${#bads[@]}

   printf "\t%s\n" "${bads[@]}"

   printf "\n"

   if [[ ! $bads ]]                        # nothing to do!  timestamp the logfile

   then

       printf "%s %s\n" "$(date +'%Y-%m-%d %H:%M')" $ext >> $logfile

   else

       for fn in "${bads[@]}"              # for each renamed file

       do

           base_fname=$(basename -s $ext_flag $fn )

           dir_name=$(dirname $fn)

           newname="${dir_name}/${base_fname}"     #remove the extension flag

                                           # log the rename, and do it.

           echo "$(date +'%Y-%m-%d %H:%M ' ) mv \"${fn}\" \"${newname}\" " >> $logfile

           mv "${fn}" "${newname}"

       done

   fi

fi

In MS Word 2003, how do you slant a cells borders in the top row to - Microsoft Community

sheana say:

In MS Word 2003, how do you slant a cells borders in the top row to match the diagonally place text?

Sometimes you see this particular type of formatting in a ledger sheet. Your help is appreciated. Thanks in advance,

 

SHeana

jay freedman say:
Slanted borders are easy, it's the slanted text that's hard to do.

Select the top row of the table. Click Format > Borders & Shading. In the right-hand (Preview) section of the dialog, click one of the two diagonal-line buttons. If you want to remove the top horizontal border or any of the other borders, click those buttons (or click the border itself in the preview picture).

If you insert ordinary text in the cells of the top row, that can be made either horizontal or vertical (right-click the selected cells, click Text Direction, and choose one of the three options) but not diagonal. It will run right across the diagonal border as if it wasn't there. In Word 2003, the only way to make diagonal text is to use WordArt, which makes it unreasonably difficult to create "just plain text".

jay freedman say:
In Word 2007, select the top row of the table. The Table Tools tab will appear on the ribbon, with two "sub-tabs" under it -- Design and Layout. Click the Design tab if it isn't already selected. At the right end of the Table Styles group there is a Borders button, with a down arrow to its right. Click the down arrow and choose either of the diagonal borders. To remove other borders, click the down arrow again and click the border you want removed.

If you'd rather deal with the same dialog you see in Word 2003, you can get that by clicking the "Borders and Shading" item at the bottom of the Borders menu.

Allow sharing in one of three document libraries in a Site?

Hello!

I have a site containing three document libraries.

I want two of these libraries to send me requests when users want to share files. I want the other section to allow users to share files. (All within the company, no external sharing)

Currently, I can only set sharing permission for the entire site, so I get notifications any time someone wants to share a file.

Please advise. 

jerryduan_msft8021 say:

hi allision,

understand that you’ve turned on the “access request settings” for the site in site settings > site permissions. this setting is only available for the site level instead of library/list level.

thus, if you want users to send request for specific libraries, you need to firstly turn this feature on for a site. if you want a user to share items freely without request in a library/list, you shall share the user a “full control” permission in the library/list. to do so, access the library/list settings > permissions for this library/list >grant a “full control” permission to the user. for those users who have “full control” permission in this situation, they do not need to send request for sharing items.

regards,
jerry

If I turn that setting on, won't it let people share the other libraries without sending me a request?

Also, if I give someone "Full Control" - will they be able to see all documents in a library? Currently, the library I am referring to has a lot of files which each have individual permissions, no one should be able to see them all.

jerryduan_msft8021 say:

hi allison,

1. “if i turn that setting on, won't it let people share the other libraries without sending me a request?”. 

no. after turning this setting on for a site, the request will be sent to the email address that is filled in the blank for approval.

when a user’s permission is lower than “full control” for an item/library, the user will not be able to directly share the item/library without approval.

2. “if i give someone "full control" - will they be able to see all documents in a library?”

no. you don’t need to assign users “full control” permission for the whole library if you don’t want. just assign users “full control” permission for an item is sufficient to let the users share this item without request.
