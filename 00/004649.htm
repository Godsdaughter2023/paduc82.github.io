tehm say:
I have had this sort of problem so many times and it's been my experience (yours may vary) that wherever you search for a solution you just find "What changed?" or "So delete some items!" type responses.

OK, so "What changed?" is at least a sensible question, but personally I have never had this sort of issue occur as a result of any unusual change I have made, and never have I had it as the result of my device genuinely being "out of memory" (well, except as in 1) below ...).

Your post is over a year old, and at the time you fixed it with a hard reset, so I assume you haven't had the problem again or found some other sort of solution in the meantime.  However in case you have and you haven't (if you see what I mean ...), and in case anyone else with the same problem finds this page, here is what has been my experience with this problem - I hope it helps someone!

Firstly, yes, of course, try the obvious - see whether you have free memory (this varies from 'phone to 'phone but usually there is some "Program" or "Settings" utility which will report this.  If in doubt, close everything, try a soft-reset (usually by pressing and holding the power button), try moving all the files from your "My Documents" on the device to somewhere safe on your PC, etc. etc. etc. etc.  In my experience, that will make not the slightest difference, but it has to be tried just in case you genuinely are out of memory.

Still reading?  OK then ...

I've always had this as a result of one of the following issues (usually 2) or 3); sometimes 1)):

1) There is something trying to synchronise that is just way too big.
This is the closest I have come to genuinely being out of memory, but it's not always immediately obvious why.
1a) If you're using "ActiveSync", go to the "Tools" menu, "Options", click on wherever your "E-mail" is synchronised to and click the "Settings..." button.  Check whether you have a "Download size limit" set; whether you have "Include file attachments" ticked (and if so try un-ticking it, or pick a small "Only if smaller than" size such as, say, 1Kb).  Try synchronising.  If you're using the synchronisation features of something other than "ActiveSync" (E.g., you're running "Vista" or "Windows 7") then I'm afraid I can't tell you how to change this as I'm still sticking with "XP" and "ActiveSync" for the moment ...
1b) Try un-ticking each of the synchronisation item types in turn (E.g., "E-mail", "Contacts", ...) to see precisely which of them is causing the problem.  Eventually, you'll probably un-tick one and find you can now synchronise.  Concentrate your trouble-shooting on that item.  If the problem is with something other than e-mail, skip to the next section.
1c) If the problem is with e-mail, you need to be aware that there are folders which are apparently automatically synchronised no matter what you tell "ActiveSync" (in fact, there's no way to tell "ActiveSync" not to synchronise these folders).  In particular, the "Drafts" folder seems sometimes to be a problem; I've wound up with enormous items in there when "Outlook" has had a problem, and trying to synchronise these has caused this issue (E.g., items where the message, and any attachments or embedded objects such as pictures, are bigger than the entire 'phone memory!).  So check your "Drafts" folder and, if there's anything in there, finish and send it or move it to another folder (not "Inbox" or it'll still just try to synchronise it as that's another folder for which you can't turn synchronisation off - except by not synchronising e-mail at all, that is ...).  Or delete it, if you no longer need the draft ...
1d) Likewise, it seems that sometimes items try to synchronise even in folders which are not set to synchronise.  I guess this is just one of the joyous little quirks of "Outlook"/"Exchange"/"ActiveSync" ...  You may be able to get around this by going into the folder list menu (look at the e-mail "Settings" as described in 1a) and click on "Select Folders..."), then ticking all the items, letting "ActiveSync" try to synchronise (it won't, so you should probably click the "Stop" button to stop it when you get bored with waiting), then going back into the folder list menu and un-ticking everything (that you don't want to synchronise) again; this time, let synchronisation complete.  This may or may not work - sometimes I have had it work perfectly, other times it's made no difference ...  In the latter case, again, proceed as below.

2) A synchronised item is corrupted.
3) The synchronisation database is corrupted.
This is the most common scenario I've encountered.  Fortunately, the resolution is relatively straight-forward, and the same for both.
Having identified which type of item is causing the problem, you need to find which specific item (or items) are causing the problem.
If you're using "Outlook", create yourself a new data file.  How you do that varies wildly from version to version, but if nothing else, you can get at this function by exiting "Outlook" and running the "Mail" control panel item - this should have a "Data Files" button which brings up something which will allow you to "Add..." a new data file.  Do so, and call it something like "Temp"; accept all the default answers and (unless you are terminally paranoid ... what ... wait ... who said I was paranoid?! ...) don't bother setting a password.
If you exited from "Outlook", re-start it now.
Click on the "Folder list" icon.  Again, where this is varies with version, so you might have to search the help to find it.  When you do, it will bring up everything in your mail profile, including "Contacts", "Calendar", "Journal", "Suggested Contacts", and generally a whole load of folders which you may or may not otherwise be able to see.
At this point, while you have this up, look at your "Sync Issues" folder, and any sub-folders in there as well - you may, if you're really lucky, find some useful explanation in there.  Don't hold out too much hope though ...  It's possibly also worth examining any items under here and deleting anything irrelevant, as otherwise this can significantly add to the size of your mailbox.
For whatever was causing your problems, whether it was e-mail items, contacts, etc., move that folder into your new "Temp" data file.  For e-mail folders you have created, you can just drag and drop them; for any system folders ("Inbox", "Sent Items", "Drafts", "Contacts", "Tasks", etc.), you'll need to create a correspondingly named folder in "Temp" first; if it is for something other than an e-mail item ("Contacts", "Calendar", ...), pick the appropriate type from the "Folder contains:" drop down list when you right-click in "Temp" and say "Create New Folder".  Then go into your original folder, select all items ("<Ctrl>A" will usually do this for you) and drag them into the new folder.

When you have done this, you should have all items in the problematic category moved into your new data file.  There should be no items in the problematic category left in your normal mail profile.

Now press "F9", or click "Send/Receive All Folders", to make sure "Outlook" has synchronised with your "Exchange Server" (if you are using one).  When that has completed (and you've moved anything which may have arrived in the problematic category as a result), try synchronising your device again.
If it was a corrupt synchronisation database, this will usually resolve the issue, as there are now no items of that category to synchronise, and "Windows Mobile" will usually recognise this and effectively clear out the database, restoring it to a known state.

Now for the tedious, and finger-crossing, part ...  Start moving items back.  If e-mail was identified as the problem, start moving your e-mail folders back, pressing "F9" (or "Send/Receive ..."), and re-synchronising; if e-mail was the problem, leave the contents of your "Inbox" until last and if you get that far without problems, move items a few at a time.  Likewise, if  "Contacts", "Calendar", "Tasks", etc. were your problem, move items in one or a few at a time until the problem recurs.

If you get to the stage where all your items have been moved back, and you can still synchronise correctly, well done - you had a corrupted synchronisation database and it's now all fixed.  Otherwise, by now, you should have found which item was causing the problem and you can delete, edit, re-create, or whatever, the problematic item.
If you moved everything of the problematic category into your "Temp" data file and you still can't synchronise correctly, try moving everything else as well to see whether that helps, then move things back category by category (E.g., move your e-mail back first, then "Contacts", then ...) until either everything has been moved back, or synchronisation breaks again.  Then fix what broke it.

By now, you should have a 'phone which will correctly synchronise without this darn' error!  You will doubtless have experienced frustration along the way - for example, it's not uncommon to have a problematic "Contact"; you delete the original item, the problem goes away; you create a brand new item with the same information in, the problem comes back.  The item displays correctly in "Outlook" (and however else you access it, E.g., through "OWA") but simply will not synchronise correctly.  If you experience this sort of problem, you have my sympathies - it's really annoying ...

And finally ...

Once you've completed all this, you'll either have a working synchronisation set-up again, or not.  Either way, you may well have pulled out a lot of hair in frustration ...  Whichever, do remember to double-check you've moved everything back out of your "Temp" data file, and then delete it - it can be deleted through the same method used to create it (there should be a "Remove" button near the "Add..." one).
If you've done all that and still can't synchronise without a "Not enough free memory ..." problem then I'm afraid all I can suggest is to back up your device, hard-reset, and re-set the whole darn' thing back up from scratch (re-install any software, reset any configuration changes you might have made, restore your data, ...) as it's probably a corrupted synchronisation database which simply will not reset itself.  Please do be aware that a hard-reset will lose absolutely everything on the 'phone - I don't want people complaining at me because they followed this guide, lost everything, and want to blame me for it ...

a. user say:

Hi Melissa,

 

Welcome to Microsoft Community Forums and thank you for choosing Windows as your preferred operating systems.

 

I see that you want to know more about Temporary Files in Internet Explorer and its uses.

 

WebPages are stored in a temporary Internet files folder for the first time you view them in your web browser. This speeds up the display of pages you frequently visit or have already seen, because Internet Explorer can open them from your hard disk instead of from the Internet.

 

Deleting the temporary files and the browsing history increases the performance of Internet Explorer. However, you can store the temporary files till the space allocated to it is completely filled up. So it is wise to delete the files and the browsing history, every couple of months.

 

However, not deleting the files and history won’t really affect Windows operating system in any way. However, as the space allocated to the temporary files gets filled up, Internet Explorer may become quite slow. To improve its performance, just delete the temporary files and browsing history.

 

Refer to the following link to learn more about temporary files and browsing history in Internet Explorer.

Temporary Internet files: recommended links
http://windows.microsoft.com/en-us/windows7/temporary-internet-files-recommended-links
