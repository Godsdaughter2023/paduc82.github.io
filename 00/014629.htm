gary's student say:

Consider using an event macro.  The following is an example that sets the color of a cell equal to its typed value after you type an entry.  Clearly this is only valid for valid values of Colorindex:

 

Private Sub Worksheet_Change(ByVal Target As Range)
Target.Interior.ColorIndex = Target.Value
End Sub

 

Because it is worksheet code, it is very easy to install and use:   1. right-click the tab name near the bottom of the window 2. select View Code - this brings up a VBE window 3. paste the stuff in and close the VBE window   If you save the workbook, the macro will be saved with it.   To remove the macro:   1. bring up the VBE windows as above 2. clear the code out 3. close the VBE window   To learn more about macros in general, see:   http://www.mvps.org/dmcritchie/excel/getstarted.htm   To learn more about Event Macros (worksheet code), see:   http://www.mvps.org/dmcritchie/excel/event.htm

No, it won't help. A recurrent detection of a threat does not mean a malfunction of MSE. It might mean the subject threat might've been only 'partially' removed, so you would need to complete the clean-up; or maybe it was removed, but a new instance is being detected (again). This may happen sometimes.

IObit Malware Fighter may also play a role here. If it was my PC, I would remove 'everything' IObit, i.e., ASC, Malware Fighter, Driver Booster, etc..

Often, this detection might be just one piece of a much larger attack. If it is being detected in your %TEMP% folder or Java cache, it’s likely that an attempt to compromise your PC has been made. This means that a website that contains a malicious exploit code has tried to infect your PC, whether it was successful or not.

Were you recently offered to install an update (probably a fake) to Flash Player?

Would suggest you first delete the contents of your Temporary Internet Files folder before running the scanners recommended above per the excellent advice provided by Le Boule.

Also, Clear the Java cache and uninstall older versions of Java from your system. Better yet, remove Java all together; and unless you 'definitely' need it, DO NOT reinstall it back!

Then, update MSE and run a full scan. Report your results.

Since subject threat belongs to the Win32/Zlob family of Trojans that modify your browser(s) settings (browser hijacking), redirect the default internet search and home pages, and attempt to download and execute additional malware from the Internet; a good-dedicated adware removal tool, such as AdwCleaner, may also help cleaning the PC off any Potentially Unwanted Programs (PUP) or adware that may have been installed without your knowledge or consent.

It's also recommended you reset Internet Explorer settings afterwards, as well as any alternate browser (Chrome, Firefox, etc..) that is installed there.

Finally, please also make sure you have Windows, IE, and all third-party software (Your alternate browser(s), Adobe Reader, Flash Player, Java (if 'absolutely' needed), etc..) fully patched.

Please report back with your results. Good Luck!

Formatting multiple cells, using data from one cell? - Microsoft Community

martyquiram say:

Formatting multiple cells, using data from one cell?

For example:

I have data in cells A1:D1.  A1, B1, and D1 contain number values that change daily.  The data in C1 can change daily, to one of four different terms, "FR", "98", "99", or, "PS". 

I want all 4 cells in this example to turn a certain (same) color, based on which of the 4 terms is entered into C1 for that day.

To sum it up,  if C1=fr, then make A1, B1, and D1 green.  If C1=98, then make A1, B1, and D1 orange, etc.

I assume that this will be done with conditional formatting, but maybe there is a formula that I can put on the page that will do this?
bernard liengme say:

Select A1:D1; open Conditional Formatting dialog

Use New Rule, use Use a Formula ....

Enter this formula =$C1 = "fr"  (the important thing here is the $ symbol) then set the required format

Do likewise for other conditions.

Don't want C1 to change colour? Click the Cf icon on Ribbon, use Clear Rule, use Clear rule in Selected cell
