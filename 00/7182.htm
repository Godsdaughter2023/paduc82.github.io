Appreciate any help anyone can give me..

Brgds,

Faiz

john w. vinson say:

I don't often disagree with my colleague Daniel, but I do have to quibble with him here. An Access Date/Time datatype is appropriate for recording an exact point in time; it's internally stored as a Double number, a count of days and fractions of a day (times) since midnight, December 30, 1899. It's possible to use it for durations, but as you have found it's awkward; there is no Format for 319:30 hours as there is in Excel.

What I'd recommend instead is that you internally store the flying time duration in a Long Integer field counting the minutes flown (e.g. for 100 hours the table would contain 6000). You can display this in a report with an expression like

ShowTime: [FlightTime] \ 60 & ":" & Format([FlightTime] MOD 60, "00")

and you can enter it using a Form with two unbound textboxes for hours and minutes, with some simple VBA code to load the fields in the form's Current event and to translate the values entered by the user into this duration field.

jay freedman say:

If you're referring to the Set As Default button on the Design tab, that does not save the page background color. The message in the prompt is explicit about what it does save:

To save the page color, first open the Normal.dotm template as a document using these steps:

  1. Press Alt+F11 to open the macro editor.
  2. Press Ctrl+G to open the Immediate window at the bottom of the editor.
  3. Type or copy/paste the following line into the Immediate window and press Enter:

NormalTemplate.OpenAsDocument

   4. Close the macro editor.

Now change the page color, and save and close the Normal.dotm template. From then on, each new blank document will start with that page color. This will not affect any existing documents.
