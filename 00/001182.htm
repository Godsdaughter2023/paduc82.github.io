chemist18 say:

Data structures to use for A-star pathfinding algorithm in VBA

Hello All,

I'm disabled and can't work so in order to keep my brain active I'm trying to learn a little about AI programming.  Initially, I was trying to bring myself up to date with VB.NET but the available documentation is insufficient for anything other than very basic exercises and long time VB.NET programmers are very intolerant of new people asking questions, they are too attached to their Wizards.  So I'm stepping back and just using VBA in Excel 2010.

I've set up a grid on an worksheet:

The A-star algorithm is used to find the shortest path from one spot to another.  Rather than explain the entire process I'll just describe the part of looking for some advice with.

As part of this algorithm you maintain what is called an "Open" list which are tiles that are possibly part of the path and have not been eliminated yet.  Associated with each open tile are that tiles X, Y coordinates, pointers to track to the previous tile (again an X, Y), the number of steps to have reached this tile, and this tiles "score" which is an estimate of which tile to check first in the open list.  This list will contain an unknown number of entries, the number of entries will change continuously as tiles are checked for suitability.  The process runs roughly like this:

You are in a tile and want to move to another

1 - You check each of the adjacent 8 tiles,

2 - if the tile contains an obstacle or it's off the edge of the grid, you ignore it

3 - if the tile is open then you assign it a score (number of steps to reach it plus an estimated distance to the destination ignoring obstacles)

4 - one you've done this for all the tiles, you sort the open list with the lowest scores first

5 - when you check the first tile in the sorted list, you look at all of it's neighboring tiles

6 - if a tile passes the test in step #2 then you add it to the open list

7 - once you've looked at all the neighboring cells, you remove that cell from the open list and add it to the closed list (there is still data that needs to be maintained in case it is ultimately part of the chosen path

8 - you continue this process until you reach the destination tile

So the Open list length can vary significantly depending on the path, the Closed list will grow steadily until the destination is found

I've thought of several techniques to track the necessary data

1 - put two tables (Open list and Closed list) on a hidden sheet

 - this would allow the length of the two lists to be changed easily, any gaps in the open list from moving a tile to the closed list would be filled by resorting the open list

2 - use arrays with very large bounds, or arrays that I adjust using ReDim Preserve statements to increase the array size when needed

3 - use a collection with a user defined variable Type.  I find collections confusing because you can't depend on the indexes remaining the same, e.g. if you Remove item number 1 from a collection, then item number 2 is now number 1, etc.  Sorting a collection seems to be problematic as well since they are read only

Typing this up I've decided the best place to start since I'm using Excel is a hidden worksheet which will allow me to use Excel's built-in Sort, Copy, Paste methods.
