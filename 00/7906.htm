andreas killer say:

I have a notion this could be achieved with a combination of INDEX and MATCH functions, but I'm completely unable to understand how these work! :-)

Let us assume the list of cities is in K1:K5 in the same sheet.

We can use MATCH to search for a value (from cell A1) in the list in K1:K5 and get the row number. But if the value is not found we get an error, that's bad. Therefore we "cover" the error with IFERROR and return 0 instead.

E1:  =IFERROR(MATCH(A1,$K$1:$K$5,0),0)

Drag the formula down till E3, then E1:E3 to the right till column G. In the next step we can sum up the results for the final row number:

H1:  =SUM(E1:G1)
Drag the formula down till H3

And finally get the City in column D

D1:  =INDEX($K$1:$K$5,H1)
Drag the formula down till D3

That's it.

But there are 2 possible issues.

a) It is possible that you have 2 cities in a row
b) It is possible that the City in a row is not in your list

You can see a) if a row in E:G has less then 2 zeros and b) if the result of a row in E:G is 0

In that case you should throw an error:

D1:  =IF(COUNTIF(E1:G1,0)<2,"#Double City!",IF(H1=0,"#No City!",INDEX($K$1:$K$5,H1)))

jay freedman say:

The representative you spoke to was wrong. Unfortunately, this is not uncommon when asking about features that aren't the most commonly used.

I have confirmed on my own computer that Word 2016 behaves exactly as you described as your standard operating procedure. With the cursor in any footnote, Ctrl+A followed by F9 does update any cross-references (and any other fields in all footnotes). If the insertion of a footnote earlier than the cited note changes the numbering, the cross-references do update to show the new numbers.

If your laptop has a keyboard button labeled "Fn" or similar, you may have to press that together with the F9 key to make it work as a function key instead of some hardware-dependent command.

If you don't see the correct behavior, first verify that the cross-reference is still a field with a code like { NOTEREF _Ref482199094 \h }. (The number identifies a hidden bookmark that surrounds the referenced footnote number.) Even if it looks correct, it may help to remove and re-insert the cross-reference.

If you've mistakenly pressed Ctrl+F11 while a cross-reference field (or all of the footnotes) was selected, the field will be locked to prevent updating. That can be reversed by pressing Ctrl+A and then Ctrl+Shift+F11.
