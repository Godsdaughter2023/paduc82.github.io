yankosheiretov say:

Did some debugging and found this to be a bug in Windows 10. I was also able to find a workaround.

I also found a way to fix the problem of computers not showing up under "Network" even when they are accessible directly as \\computer\share.

Workaround for problem of Win 10 not being able to see Win 7 files even if it works in reverse:

Using Wireshark I dug deeper into the IP packets exchanged between the client and server. Basically, Windows 10 clients stop the process after receiving the smb and smb2 protocol negotiation response from the Windows 7 server. I compared the packets to the ones sent in the reverse direction. Although Win7 sends the right response, Win 10 gets stuck.

My happy workaround is to force the computers to use Smb1.0 only. To do that you must enable smb1.0 on Windows 10, which in my case was already done to enable them to see XP and Samba. To do this, go to Enable/Disable Windows features under Control Panel/Programs and Select the smb1.0 boxes to enable them. You must also force the Windows 7 server to only offer Smb1.0. To do that add and "Smb2" registry value of type REG_DWORD and value 0 under HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\LanmanServer\Parameters. Reboot everyone.

I also had the problem of browsing not working, such that none of the computers showed up under "Network" on any of the systems. On a small network with no windows server, this functionality is provided by one of the computers becoming "Master Browser", keeping a list of active computers and providing it on request. Which computer becomes the master server is decided by a process that favors systems with newer operating systems, Windows 10 in this case. Well, the browsing service on Windows 10 is broken, such that they insist on becoming the master, then not doing the job. So, you must prevent them from becoming master. To do this, go to the following registry location on each Win10 machine: HKey_Local_Machine\System\CurrentControlSet\Services\Browser\Parameters and edit the key IsDomainMaster to read FALSE and the key MaintainServerList to read NO. This should do the trick. On my network I was lucky to also have a Linux computer, so to make doubly sure I also set it up to claim the highest level OS. You can find instructions on how to do this online.

So now finally after many hours, everyone is happy. I must say it is more than annoying to have adding a single Windows 10 computer foul up my entire network at home, so I hope this post can reach others and help save them time and exasperation. And no, I don't have the choice of upgrading everyone to Windows 10, because I have expensive audio gear that is no longer supported.

Yanko
