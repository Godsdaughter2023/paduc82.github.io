tryx3 say:

As Forest brook wrote, you just select the drive letter for the Windows backup imaging tool.


If you select drive J:\, it creates the image in a folder J:\WindowsImageBackup.


You cannot change this.


Storing multiple images

You can move the images around after creation to keep them all separate and therefore you can use the same external hard drive for both.  Admin permission is needed for each move & it is possible that the move must be done whilst connected to the computer that made the image [but I cannot test that at the moment].  Whatever moves you do, you must move the whole of J:\WindowsImageBackup not just its subfolders. 


So you might move todays image for one computer to J:\ImageLibrary\Computer1\20131221 and then later on you might make an image for the second computer then move it to J:\ImageLibrary\Computer2\20131221. 


If you accidentally leave an image in J:\WindowsImageBackup and then make another image, Windows will seek to combine the two into one, still within J:\WindowsImageBackup.  If both images are of the same computer then it should be able to cope with this but it might well create a complete mess if the images are from two different computers.  So you'll have to get into the habit of moving images into the library folder as soon as they are made if you want to avoid problems.  {Additional note Friday 3 Jan 2014}  There is a good explanation of Windows' management of multiple images in the TechNet blog Learn more about system image backup.  This blog explains, in particular, the method used in updating images & the circumstances that can lead to older image versions being removed by the system itself.


If you ever want to restore from an image you will need to move the one you want back into J:\WindowsImageBackup as Windows can only restore from that location. [Don't be misled by the fact that during the restoration process you see that Windows can read the date of the latest image on J:\ - it can only work with the one in J:\WindowsImageBackup].


Do consider using two external drives instead of one.  You are trusting the integrity of two computers to their images.  If an external drive fails then you lose all your images.  You might wish to do as I do.  This week, I saved images in J:\ and next week I will save images in H:\.  If my computer & an external disk both break at the same time then I'll still be able to restore the computer to its state two weeks ago or less.  In your case, I'd suggest saving both the computer 1 & computer 2 images on J:\ and both images next week on H:\ but that's just so it's a bit easier to keep track of what you are doing. 


Finally, whilst Windows can do restorations using onboard tools, they can & do fail sometimes so make sure you have a  System repair disk or Windows 7 installation disk with which to boot the system so you can run an image restoration from there. 


Whichever type of disk you have or make, test it:-


1  Boot from it, then

2  Select keyboard & language,

3  [for installation disk you then also select - Repair your computer],

4  Select Use recovery tools...* [and note the drive letter being used for the Windows drive** before clicking on Next] then

5  Select Command prompt or just restart the system from that menu to get back into Windows itself. 


The disk has now proved that it can boot correctly - there are some errors with, in particular, System repair disks that only become apparent when you boot from them [nothing seems to be wrong with them when you make them].


*  This is not part of the test but you'll see in this dialog another option at the bottom of the Windows - Restore from an image.  This is what you will select if you are ever using the disk for real.


** This is not part of the test but you would need to know this if you then tried looking around while you were in the Command prompt.  These disks apply rules that are different to those of Windows itself so your C:\ drive might be D:\ [or anything except X:\] but will revert to C:\ when you later reboot into Windows normally].


Best of luck,

When I upgrade a preinstalled (OEM) or retail version of Windows 7 or Windows 8/8.1 license to Windows 10, does that license remain OEM or become a retail license?

If you upgrade from a OEM or retail version of Windows 7 or Windows 8/8.1 to the free Windows 10 upgrade this summer, the license is consumed into it. Because the free upgrade is derived from the base qualifying license, Windows 10 will carry that licensing too.

If you upgrade from a retail version, it carries the rights of a retail version.

If you upgrade from a OEM version, it carries the rights of a OEM version.

Full version (Retail):

- Includes transfer rights to another computer.

- Doesn't require a previous qualifying version of Windows.

- Expensive

Upgrade version (Retail):

- Includes transfer rights to another computer.

- require a previous qualifying version of Windows.

- Expensive, but cheaper than full version

OEM :

OEM versions of Windows are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system

What happens if I change my motherboard?

As it pertains to the OEM license this will invalidate the Windows 10 upgrade license because it will no longer have a previous base qualifying license which is required for the free upgrade. You will then have to purchase a full retail Windows 10 license. If the base qualifying license (Windows 7 or Windows 8.1) was a full retail version, then yes, you can transfer it.

From the Windows 10 end user license agreement:

b. Stand-alone software. If you acquired the software as stand-alone software (and also if you upgraded from software you acquired as stand-alone software), you may transfer the software to another device that belongs to you. You may also transfer the software to a device owned by someone else if (i) you are the first licensed user of the software and (ii) the new user agrees to the terms of this agreement. You may use the backup copy we allow you to make or the media that the software came on to transfer the software. Every time you transfer the software to a new device, you must remove the software from the prior device. You may not transfer the software to share licenses between devices.
