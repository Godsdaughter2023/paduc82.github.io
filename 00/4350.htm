If not, you may need to copy the code from the mac template, (or, if it can be done on the Mac, export the Module containing the code) and then create a template on a Windows PC, into which you insert a Module and paste the code, or, in the case where you exported the Module, import the Module into the template.

macropod say:

Recently, we developed a new macro embedded template,  created by a Mac User in the dotx format.  This template does not load on Windows machines.

When I suggested to the creator of the macro that if it were saved to the *.dotm format for macro embedded templates,  he responded that only *.dotx is available for Mac based tempates.

That is wrong on two counts:

1. Files saved in the dotx format cannot contain macros. Period.

2. Macs do use dotm templates - they must do so for the template to be a macro-enabled one. Indeed, Word's own 'Normal' template for Macs is a dotm file...

tonythomasindexing say:

I should have replied  when I resolved this problem.

The issue here was really that the template maker was using Word 2011 for Mac. 

That form is a hybrid between XML and pre XML versions of Word, similar to Word 2007 for Windows.

2011 for Mac is coded in XML but does not have the special document embedded template format developed with more recent versions of Word.  It only offers template makers the alternative of making the old dot type templates for  document embedded templates, even though it has the new form of dotx templates for templates with only text and formatting.

Once the template maker was informed that he should make the template in dot. we had no problems since we have become very experienced transforming older pre-XML *.dot embedded templates into *.dotm templates in the project.

To sum up, this was not a question of migrating templates from PC to Mac or vice versa, but a problem of our not recognizing the hybrid nature of template formats available in this older version of Word for Mac

Thanks to all who contributed.
