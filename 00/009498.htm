mizzouece say: I just stumbled across the answer to my question.

You must use the "Custom Sort..." option if you want lexicographic sort order. Selecting "Custom Sort..." opens the "Sort" dialog. On that dialog, select the column(s) you want to sort by, and then click on the "OK" button. Excel will then present the "Sort Warning" (?) dialog box that lets you choose the following sort options:

( ) Sort anything that looks like a number, as a number [selected by default, derp]
( ) Sort numbers and numbers stored as text separately

Select the second option (Sort numbers and numbers stored as text separately) if you want the selected cells to be sorted using lexicographic sort order. then click on the dialog's "OK" button.

FWIW, this option appears to be "sticky"; once it's selected it persists--i.e., you can now use the "Sort Ascending" or "Sort Descending" options without having to use the "Custom Sort..." option each time.

jay freedman say:

The two lines just before the End Sub should be combined into one line with a comma between them, and the equal sign between Password and "password" must replaced with a colon-and-equal-sign like the one after NoReset.

ActiveDocument.Protect Type:=wdAllowOnlyFormFields, NoReset:=True, Password:="password"

It's a bit surprising to me that the mistaken line Password = "password" sets the password to open the document (it doesn't do that here); to do that, it should have to be written as ActiveDocument.Password = "password".

To remove the password from a document that already has it, open the document (using the password). Press F12 to open the Save As dialog. Click the Tools link at the bottom of the dialog and choose General Options in the menu. In the General Options dialog, delete the contents of the "Password to open" box and click OK, then finish saving the document in the same folder and with the same name.

maratae1 say:

NoReset:=True

jay freedman say:
If you don't include the Password:= clause in the ActiveDocument.Protect statement, the protection will no longer have a password. That won't harm anything in the document, but users will then be able to turn off the protection and edit the parts of the document that are outside the form fields.

UNEXPECTED_KERNEL_MODE_TRAP (error code 0x0000007F) is one of the Windows 10 errors that often shows Blue Screen of Death (BSoD) which causes the computer to crash and freeze. It is a bug check indicating that the CPU has generated a trap and the kernel failed to catch the trap.

If SFC and clean boot did not help resolving the issue, try performing a driver verifier to track down misbehaving drivers that might be causing the issue. Refer to this Forum Article created by ZigZag3143 (MS -MVP) on how to use the driver verifier.
