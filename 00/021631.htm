fran tremblay say:

I'm by no means an expert (and this is my first post on this site), but I use INDEX / MATCH all the time, between sheets and between files.  If your example is having Sheet1 with Employee# in Column A, and Sheet2 with CostCentre in Column A and Employee# in Column B, then your calculation should look something like this:

=INDEX(SHEET2!A:A,MATCH(SHEET1!A2,SHEET1!B:B,0))

(You don't need to enter "SHEET1!" in your calculation since it's a function that's already referring to Sheet1.)

There are a couple of additions to this calculation that I often use:

  • Not all of the values that I'm using for the first part of the Match function will lead anywhere in the Index table, so I will often pair it up with an IFERROR to either blank the field if it doesn't find anything, or to highlight that there isn't a match.

=IFERROR(INDEX(SHEET2!A:A,MATCH(SHEET1!A2,SHEET1!B:B,0)),"")

=IFERROR(INDEX(SHEET2!A:A,MATCH(SHEET1!A2,SHEET1!B:B,0)),"<missing>")

  • If you're bringing across multiple columns worth of data, especially consecutive, use the "$" addition to the references in the Match function (but not the Index function).  Note that setting your references by clicking the columns in a different file will automatically add the "$" to the Index reference, so you'll need to remove those to allow those to adjust over when you copy the formula across columns.

=INDEX(SHEET2!B:B,MATCH(SHEET1!$A2,SHEET1!$A:$A,0))

=INDEX(SHEET2!C:C,MATCH(SHEET1!$A2,SHEET1!$A:$A,0))

=INDEX(SHEET2!D:D,MATCH(SHEET1!$A2,SHEET1!$A:$A,0))

These are all set up as full-column searches - you can limit the columns as usual by entering column & row references instead of just columns, but make sure to use "$" to limit the movement of the range when you copy/paste vertically.  You can also set the search to work across a row instead of down a column by using whole row references instead.

joeu2004 say: Denise wrote:
I want to delete multiple alternate rows in excel without having to delete each row indivulally.

It is not clear if you mean every alternate row starting with the first row (every odd row) or with the second row (every even row) of a selection [*].  Use whichever of the following macros is appropriate.

To enter the macro(s)....    In Excel, press alt+F11 to open the VBA window.  In VBA, click on Insert > Module, and copy-and-paste the text of the macro(s) below into the VBE window pane.  You can now close the VBA window.

To run a macro....    In Excel, select a column of cells in the rows to be deleted.  Then press alt+F8, select the macro, and press Run.


Sub deleteEveryEvenRow()
' delete rows 2 to 2k in Selection
Dim r As Range, i As Long, lastrow As Long
Dim oldcalc
oldcalc = Application.Calculation
Application.ScreenUpdating = False
Application.Calculation = xlCalculationManual

Set r = Selection
lastrow = 2 * Int(r.Rows.Count / 2)
For i = lastrow To 2 Step -2
     r.Cells(i).EntireRow.Delete
Next

Application.Calculation = oldcalc
Application.ScreenUpdating = True
End Sub


Sub deleteEveryOddRow()
' delete rows 1 to 2k+1 in Selection
Dim r As Range, i As Long, lastrow As Long
Dim oldcalc
oldcalc = Application.Calculation
Application.ScreenUpdating = False
Application.Calculation = xlCalculationManual

Set r = Selection
lastrow = 2 * Int((r.Rows.Count - 1) / 2) + 1
For i = lastrow To 1 Step -2
     r.Cells(i).EntireRow.Delete
Next

Application.Calculation = oldcalc
Application.ScreenUpdating = True
End Sub


-----
Aside....    I wrestled with whether or not to set Application.EnableEvents=False.  On the one hand, the macros might run faster.  On the other hand, correctness of some Excel designs might depend on an event macro being called when rows are deleted.  I normally do the former for myself; but I finally settled on the latter for greater reliability in general.  Comments?

rich007a say:

An alternative method that does all the delting in one go, and so is much quicker.

 

Sub DeleteAlternateRows()
    Dim rngRow As Range, rng2Del As Range
    For Each rngRow In Selection.Rows
         'In the next row, use "= 1" to delete odd numbered rows
         'or "= 0" to delete even numbered rows
        If rngRow.row Mod 2 = 1 Then
            If rng2Del Is Nothing Then
                Set rng2Del = rngRow.EntireRow
            Else
                Set rng2Del = Union(rng2Del, rngRow.EntireRow)
            End If
        End If
    Next rngRow
    rng2Del.Delete Shift:=xlShiftUp
End Sub
