kchad99 say:

Subtracting Military Time

I am trying to subtract two military times.  For example 4 hours and 45 minutes from 3 hours and 45 to result in 2300. 

Every formula I use, I find two conflicts:

All formulas I seem to use do not like my time format. I am looking to keep as simple as possible so when 2345 is typed, Excel sees it as a time, removing the need to use a semicolon.  I.e., when typing 1945 it sees the number the same as 19:45 (hh:mm)

Also, all formulas found come out with a negative number.  So in the examples case above I get -100, not 2300.

This would likely be easier if I could use VBA, but I need this document to be shared.

tom ogilvy say:

> All formulas I seem to use do not like my time format. I am looking to keep as simple as possible so when 2345 is typed, Excel sees it as a time, removing the need to use a semicolon.  I.e., when typing 1945 it sees the number the same as 19:45 (hh:mm)

Excel does not have that behavior as an option without specifically programming code to perform that interpretation. 

You would need to work with integers and write formulas to calculate the results you want to mimic time.  

for example

A1:  345

A2:  445

=TEXT(MOD(TIMEVALUE(LEFT(RIGHT("0000"&A1,4),2)&":"&RIGHT(RIGHT("0000"&A1,4)))-TIMEVALUE(LEFT(RIGHT("0000"&A2,4),2)&":"&RIGHT(RIGHT("0000"&A2,4))),1),"hhmm")

Produces 2300   (as a text string)

=TEXT(MOD(TIMEVALUE(LEFT(RIGHT("0000"&A1,4),2)&":"&RIGHT(RIGHT("0000"&A1,4)))-TIMEVALUE(LEFT(RIGHT("0000"&A2,4),2)&":"&RIGHT(RIGHT("0000"&A2,4))),1),"hhmm")*1

would make it the number 2300

A1:  1945

A2:  2345

produces    2000

scottgem say:
As noted, some e-mail systems restrict the size and/or type of files. So you may need to ZIP the files together as a compressed file. Another solution to this is to use a cloud storage like OneDrive or Dropbox. You can create a folder and share it with someone at the client site. daniel pineault say:

Database files on there own are often automatically blocked. 

Most of the time simply zipping them and sending the zip file is sufficient to get the database through.  In some rare cases, I have even had to change the extension of the zip, to say txt, and then change it back at the other end.

This all works beautifully when the database is relatively small, but once it grows a bit you may hit the size limit of your e-mail service.  Then you need to look into FTP, dropbox, ...
