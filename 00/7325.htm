macropod say:

To compare a mergefield date against the system date, you could use a field coded as:

{IF{DATE \@ YYYYMMDD}= {MERGEFIELD MergeDate \@ YYYYMMDD} "True Output" "False Output"}

You can change the '=' comparison to '>', '<', or even '<>', depending on what you want to do.

Likewise, if you only want to compare the:

• year, you could omit MMDD from the field switches;

• month, you could omit YYYY and DD from the field switches; or

• day, you could omit YYYYMM from the field switches; or

• month and day, you could omit YYYY from the field switches.

Note: The field brace pairs (i.e. '{ }') for the above example are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practicable to add them via any of the standard Word dialogues.

As you can see, it is quite unnecessary to manipulate the data in the data source, which is often not possible and, IMHO, is generally not advisable.

the code is very simple

Private Function IsPaper(R As Range) As String

 IsPaper = Empty
 If R.Interior.Color = vbRed Then IsPaper = "Paper"
 If R.Interior.Color = vbGreen Then IsPaper = "Plastic"
 
End Function
  
The Contextures sites show how to add a code module 
http://www.contextures.com/xlvba01.html#Regular

To use your new formula you just type in into a cell like any other

A1                   B1
(red fill)          =IsPaper(A1)

The function will return Paper for red fill, Plastic for green fill and nothing for any other fill

One wrinkle could be the fill colors chosen vbRed and vbGreen are the bright red and bright green fills.  

You may need to change this to match your exact fill color
