Excel (actually Windows not Excel) allows you to set only the first day of the week. You can't set the first month of the year. That capability is neither there nor has been demanded by enough numbers of persons for Microsoft to consider this.

The only way is to create a helper column and populate the month numbers there.

If you want Jul to be 1 and Jun to 12, use below formula

=IF(MONTH(A2)>6,MONTH(A2)-6,MONTH(A2)+6)

You can perform sorting, filtering on this column now.

a. user say:

Thanks for your help.

I managed to get this working yesterday.

My findings are below:

If you create a connector using the GUI, it does not work.

I created an outbound connector using powershell to forward all email to our smarthost. The other trick is make this an outbound connector from Office 365 to a partner organisation (even though I was just sending to Messagelabs).

Once the connector had been created, I then added a Rule ... there is a trick here too - you need to click on More Options... before you are able to set recipient domain matching, and before you are able to redirect email to a connector - I set the rule to act on all emails, and excluded the one domain that has been migrated so far... as I migrate the other domains I will add them to the exclusion list - this will ensure that Office 365 accounts will use Exchange routing, and all non-migrated accounts will use Rule-based routing.

Seems to have worked a treat!

The powershell script is below, should anyone else need to do this:

Set-ExecutionPolicy RemoteSigned

$Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri https://outlook.office365.com/powershell-liveid/ -Credential $UserCredential -Authentication Basic -AllowRedirection

Import-PSSession $Session

New-OutboundConnector -Name "Outbound Connector" -ConnectorType "Partner" -SmartHosts cluster4.eu.messagelabs.com,cluster4a.eu.messagelabs.com -Enabled $True -UseMXRecord $false -IsTransportRuleScoped $True

Remove-PSSession $Session

P.S. Don't forget to change the Smarthost from messagelabs to wherever you want to send to... this can also be a single ip address or a comma-separated list of ip addresses.

P.P.S Thank you to Elisha - the domain does need to be set as an internal relay - this can be done in the Office 365 portal Exchange Admin Center, then select Accepted Domains, then double-clicking on the RECIPIENT domain and selecting Internal Relay under the section headed by This domain is: ... in my scenario, I have one authoritative domain (the one that has already been migrated), and 7 Internal relay domains (the ones yet to be migrated). Just in case, I made the *.onmicrosoft.* domains Internal relays too.

Thanks again,

Martin
