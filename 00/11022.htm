ken sheridan say: I read Paul's question rather differently, as not to search for a specific word containing an accented character, but to find any words containing any accented character.  Searching for an accented word, 'étude' for instance is no different from searching for any word.  It will be found in just the same way.  

Another possible interpretation of his question is that searching for an unaccented word will find the same word with one or more of the characters accented, e.g. searching for 'etude' will find 'étude'.  For this a second column could be added to my suggested AccentedCharacters table, with the unaccented equivalent character in the second column.  A function into which the unaccented word is passed could then loop through a recordset of the rows from the table and call the Replace function to replace any unaccented characters in the word with each of their accented equivalents.

Perhaps Paul would care to clarify his requirements.

BTW the two dots over a letter is called a diaeresis.  It denotes that a vowel is pronounced separately, e.g, as in naïve rather than in laid.  It's derived from the Greek verb diairein, meaning to divide.

ken sheridan say: I'm afraid that doesn't tell us anything we didn't know already.  What we don't know is what you want to do:

1.  Do you want to want to return any rows in which the LastName column contains a specific name containing an accented character, e.g. de los Ángeles?  If so you do so in the normal way, e.g. in a query:

SELECT *
FROM Customers
WHERE LastName = "de los Ángeles";

2.  Do you want to return any rows in which the column contains any word or words containing any accented character?  If so you need to create an Accented Characters table and write a little function as described in my original reply.

3.  Do you want to want to return any rows in which the LastName column contains a specific name, with or without any accented characters, e.g. de los Ángeles, by entering the name without any accented characters, i.e. de los Angeles?  If so you'd again need to create an Accented Characters table, but with a second column containing the equivalent unaccented character, and write a little function as described in my last reply to Tom.

poppyyang_msft6299 say:

Hi Oli,

Regarding the error “A recipient wasn‎'t found for "*** Email address is removed for privacy ***" on the target. Create a recipient of the appropriate type for this migration on the target and try again”, I would like to share a workaround first:
1. Run Get-Mailbox -SoftDeletedMailbox to list the deleted mailbox users. If the affected users are included, run Remove-Mailbox -PermanentlyDelete to completely remove them.
2. Run Get-MailUser -Identity <user> | FL GUID to get the GUID of the user.
3. Run New-MoveRequest -Identity <GUID> -Remote -RemoteHostName 'mail.domain.com' -RemoteCredential $cred -TargetDeliveryDomain 'domain.mail.onmicrosoft.com' -BadItemLimit 10 -LargeItemLimit 20 to try a new batch to move the mailbox.


Note: please replace all the domain names included in the commands above.


If this cannot work either, please choose an affected user and run Get-Recipient -Identity "*** Email address is removed for privacy ***" | FL from both Exchange Online and Exchange on-premises.
