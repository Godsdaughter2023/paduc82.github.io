ken sheridan say: Data should never be entered or edited in a table's datasheet, but only in forms.  The normal interface for this would be a quote form in single form view, within which is a subform, in continuous forms view based on the LineItem table and linked to the parent form on ID/QuoteID.  In the subform bind a combo box, cboProduct, to the PartID column, giving it a RowSpource property along these lines:

SELECT ID,Cost,Desc FROM Products ORDER BY Desc;

Its ColumnCount property should be 3; its ColumnWidths property 0cm;0cm;8cm (or equivalentv in inches; the important thing is that the first two dimensions are zero); and its BoundColumn property 1.

In the AfterUpdate event procedure of the combo box assign the value to the Cost column in LineItem with:

    Me.Cost = Me.cboProduct.Column(1)

The Column property is zero-based, so Column(1) is the second column.

You'll find an example of a subform which behaves in a similar way in InvoicePDF.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

The invoice details subform in the main invoice form includes a combo box bound to the ProductID column.  In its AfterUpdate event procedure values are assigned to the UnitCost and TaxRate columns.

Mac PPT never has had the same capability as what you're accustomed to in the PC version. In fact, [IMO] it was implemented so disparately that MS removed it from the main menu altogether in Office 2011.

What you can do is use File> Save As & select Outline / Rich Text Format (.rtf) from the Format: list to generate a file which can then be opened in Word. All:

The shortest and simplest answer is "Re-make your document in Office 2016".

  1. You will need to re-create the document. It must be in .docx format.
  2. Visit the File menu in Word.
  3. If the Convert document item appears, click it.
  4. Then insert your spreadsheet.
  5. Make sure the columns in the spreadsheet are wide enough to allow things such as dates and numbers to grow (that's what's producing the # signs).
  6. Now save and close the document.

It should work after that.

The problem is that Office 2016 has a new "rendering engine" in it, which draws everything on the screen.  While it gets old documents "close" it can not display old documents "exactly" because they do not contain the formatting information required by the new format.
