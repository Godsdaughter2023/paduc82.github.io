lady anastil say:

Had this issue myself.  Found a fix.

To fix this: Hit the File tab, go down to Options, and in the General section right at the top, there's a box that reads: "Show mini toolbar on selection".  Un-check this box, then hit okay, and the problem should be solved.  Right click should work properly again with no more text deselect.  Worked for me.

What people are saying is accurate in that it has to do with tiny mouse movements, BUT this is caused by a feature that was added recently.  It's that mini formatting toolbar that pops up automatically whenever you highlight something that's causing the problem.  This added feature creates an overlap of systems where Word thinks you're trying to select new text and interact with the new toolbar instead of just right clicking on the text that's already been highlighted.  Best way I can explain why I think the problem happens.  Turning off the mini formatting toolbar avoids this confusion, and the right click option is freed up again.  Not sure why this is even a feature since the mini formatting toolbar shows up just above or below the right click pop up window anyway.  I guess it was added to erase the need to right click at all?  This idea sounds like it would add greater efficiency, but in reality it seems to have just made things less efficient.  Poor design, in other words, but not necessarily a bug.  Oh well.  Anyway, that's the fix everybody.

bob jones aka: cybertaz say:

In Excel, Dates are stored as serial values offset from Jan. 1, 1900, but [usually] are displayed based on Date Formatting. IOW, your "6-14" is interpreted as June 1, 2014 which is 41,791 days from the starting date of January 1, 1900.

In order to display "6-14" in that way you need to either do as you're doing or precede the entry with an apostrophe ['6-14]. Which you use makes no difference with regard to calculations because "6-14" is not a numerical value, so neither are usable for calculation purposes.

If you intend to use the start/end times as precedents for calculations each needs to be stored in a separate cell as a time or number of hours. These references will provide details for working with Dates & Times:

The button for link to previous is telling Word that you want the header in this section to be the same as in the previous section (or not). The property to restart page numbering is only tangentially related to the header/footer. The link to previous setting is for each header and footer in a section. In practice this need checks to eliminate random errors. 

IMO we should do many more tests to be sure that the values are stored at the correct position and be sure that we did not overwrite any other data.

Option Explicit

Sub Main()
  Dim ID As Range, Patient As Range, R As Range
  Dim SourceWBC As Range, DestWBC As Range
  Dim DestMonth As Range
  Dim SourceMonth, WBCValue
  Dim Answer As VbMsgBoxResult

  With Sheets("Report")
    'Get the destination month
    SourceMonth = .Range("B1")
    'Find the ID
    Set ID = .Cells.Find("PatientID", LookIn:=xlValues, LookAt:=xlWhole)
    If ID Is Nothing Then
      MsgBox "PatientID not found. Abort."
      Exit Sub
    End If

    'Find the SourceWBC in this row
    Set SourceWBC = ID.EntireRow.Find("WBC", LookIn:=xlValues, LookAt:=xlWhole)
    If SourceWBC Is Nothing Then
      MsgBox "SourceWBC not found at right side of PatientID. Abort."
      Exit Sub
    End If

    'The first patient ID is below the ID, visit it and each cell below
    For Each Patient In Range(ID.Offset(1, 0), .Cells(Rows.Count, ID.Column).End(xlUp))
      'Patient sheet found? (Note: The .Text property converts numerical entrys to text!)
      If Not SheetExists(Patient.Text) Then
        Answer = MsgBox("No sheet for Patient '" & Patient.Text & "' found. Abort?", vbYesNo + vbExclamation)
        If Answer = vbYes Then
          Exit Sub
        Else
          GoTo Continue
        End If
      End If
      'Get the WBC value
      WBCValue = Intersect(Patient.EntireRow, SourceWBC.EntireColumn)

      With Sheets(Patient.Text)
        'Search the destination month in the patient sheet
        Set DestMonth = .Rows(1).Find(SourceMonth, LookIn:=xlValues, LookAt:=xlWhole)
        If DestMonth Is Nothing Then
          Answer = MsgBox("Month '" & SourceMonth & "' not found in sheet '" & Patient.Text & "'. Abort?", vbYesNo + vbExclamation)
          If Answer = vbYes Then
            Exit Sub
          Else
            GoTo Continue
          End If
        End If

        'Search for the WBC id in column A in the patient sheet
        Set DestWBC = .Columns(1).Find("WBC", LookIn:=xlValues, LookAt:=xlWhole)
        If DestWBC Is Nothing Then
          Answer = MsgBox("WBC id not found in column A in sheet '" & Patient.Text & "'. Abort?", vbYesNo + vbExclamation)
          If Answer = vbYes Then
            Exit Sub
          Else
            GoTo Continue
          End If
        End If

        'Store the value
        Set R = Intersect(DestMonth.EntireColumn, DestWBC.EntireRow)
        If Not IsEmpty(R) Then
          Answer = MsgBox("WBC in sheet '" & Patient.Text & "' is not empty!. Overwrite?", vbYesNoCancel + vbExclamation)
        Else
          Answer = vbYes
        End If
        Select Case Answer
          Case vbYes
            R = WBCValue
          Case vbNo
            GoTo Continue
          Case vbCancel
            Exit Sub
        End Select
      End With
Continue:
    Next
  End With
End Sub

Private Function SheetExists(ByVal SheetName As String, _
    Optional WB As Workbook = Nothing) As Boolean
  'Prüft ob das Tabellenblatt SheetName existiert
  Dim s As Worksheet
  If WB Is Nothing Then Set WB = ActiveWorkbook
  On Error GoTo ExitPoint
  Set s = WB.Sheets(SheetName)
  SheetExists = True
ExitPoint:
End Function
