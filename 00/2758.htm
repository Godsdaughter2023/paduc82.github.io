stefan blom say:
If you have set a decimal tab stop, any character except the thousands separator will "respond" to the decimal tab. However, you don't get to choose which character. See if simply applying center paragraph alignment helps.

suzanne s. barnhill say:

To clarify what Stefan has said, a decimal tab stop will cause text to align on the first non-numeric character other than the thousands separator. This could be a decimal point, a letter, an asterisk, a closing parenthesis, or other symbol. So this should work for the dash. If your +/- symbol precedes the numbers, then it probably won't work; it will work if it follows them; OTOH, if it precedes them and you want to align on it, then it seems like simple left alignment would work.

 

There are other, more subtle workarounds that I sometimes use when simple centering will work for all but one or two of the numbers. In this case I pad the short numbers out with spaces (two spaces per digit) or zeroes colored white. The main reason for doing this is a situation where the column width might later change, and a decimal tab stop would have to be moved to visually center the figures.
