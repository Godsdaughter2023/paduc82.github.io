jllatham say:

As Jan Karel Pieterse has suggested, perhaps a more detailed description of what you need to do would enable someone here to assist you in some fashion.

 

If it's a matter of finding certain patterned text and replacing it, it is very possible that a macro (VBA code) could be developed to do the job for you.

 

Just as a partial example, here is code that would allow you to select a range of cells and have enough information written to a text file to work with it and then import the altered results back into presumably the same workbook.

 

Sub ExportFormulas()
'will export formulas along with the worksheet name and cell address
'from a selected range
  Dim cellsToExport As Range ' will be cells you select
  Dim anyCell As Range ' individual cells within cellsToExport
  Dim fileBuff As Integer ' buffer for output text file
  Const testOutputFilename = "myTest.txt"
 
  On Error Resume Next
  'Type:=8 = select a range of 1 or more cells
  Set cellsToExport = _
   Application.InputBox(prompt:="Select cells:", _
   Title:="Choose Cells", Type:=8)
  If Err <> 0 Then
    'user cancelled operation
    'but if they click [OK] with empty box will still get warning
    Err.Clear
    On Error GoTo 0
    Exit Sub
  End If
  On Error GoTo 0 ' reset error trapping
  'prepare to examine and write formula information to a text file
  fileBuff = FreeFile() 'get next available buffer number
  Open ThisWorkbook.Path & Application.PathSeparator & testOutputFilename _
   For Output As #fileBuff
  For Each anyCell In cellsToExport
    'we look for cells that actually have = type formula in them
    If Not IsEmpty(anyCell) Then
      If Left(anyCell.Formula, 1) = "=" Then
        'text file with:
        'worksheet name   a cell address    the cell's formula
        Print #fileBuff, ActiveSheet.Name, anyCell.Address, anyCell.Formula
      End If
    End If
  Next
  'all done, close text file and announce completion
  Close #fileBuff
  Set cellsToExport = Nothing ' release resource back to system
  MsgBox "Data has been written to file:" & vbCrLf & _
   ThisWorkbook.Path & Application.PathSeparator & testOutputFilename, _
   vbOKOnly + vbInformation, "Task Completed"
End Sub

jllatham say:

By the way - first I erred in saying the previous routine wrote a tab delimited text file - I forgot that the Print # statement turns them into a bunch of spaces.  But no matter ... here is the inverse VBA operation that would take the file written by the other routine and read it and replace the original formulas with any changes you'd made to them within that text file.

 

Maybe between these two routines and your skill with Unix tools and regular expressions you could come up with the results you need?

 

Sub ImportRevisedFormulas()
  Dim anyWS As Worksheet
  Dim anyCell As Range ' individual cells within cellsToExport
  Dim fileBuff As Integer ' buffer for output text file
  Const testInputFilename = "myTest.txt"
  Dim rawData As String
  Dim sheetName As String
  Dim cellAddress As String
  Dim newFormula As String
 
  fileBuff = FreeFile()
  Open ThisWorkbook.Path & Application.PathSeparator & testInputFilename _
   For Input As #fileBuff
  'halt screen updating and recalculation while all this
  'takes place to improve performance.
  Application.ScreenUpdating = False
  Application.Calculation = xlCalculationManual
  Do While Not EOF(fileBuff)
    Line Input #fileBuff, rawData
    sheetName = Left(rawData, InStr(rawData, " ") - 1)
    rawData = Trim(Right(rawData, Len(rawData) - Len(sheetName)))
    cellAddress = Left(rawData, InStr(rawData, " ") - 1)
    newFormula = Trim(Right(rawData, Len(rawData) - Len(cellAddress)))
    'write the revised formula to the workbook
    With ThisWorkbook.Worksheets(sheetName)
      'if the worksheets are Protected
      '.Unprotect
      'and if proteced with password, must provide it
      '.Unprotect Password:="Your Passw0rd H3r3"
      .Range(cellAddress).Formula = newFormula
      'don't forget to re Protect if needed
      '.Protect
      'or
      '.Protect Password:="Your Passw0rd H3r3"
    End With
  Loop
  Close #fileBuff
  're-enable disabled features
  Application.Calculation = xlCalculationAutomatic
  'Application.ScreenUpdating is automatically reset upon End Sub
End Sub

john korchok say:

Yes, this is possible. Here's how:

  1. In PowerPoint 2013, choose FILE>New and choose a template.
  2. As soon as the file opens, choose File>Save.
  3. Change the Save as type dropdown to PowerPoint Template (*.potx). By default the template is saved to the Custom Office Templates folder in your user folder. Click on OK.
  4. Repeat for all other templates you like.

There are a couple of gotchas to watch out for:

  • Many of the new templates are "super templates" that have color variations. Saving as a template in PowerPoint 2013 will save an ordinary template with only one color theme. So you might have to save a template several times (with varying file names) to capture every variant in the super template.
  • PPT13 saves to the Custom Office Templates folder, but 2010 doesn't know about that. So you might want to copy the saved templates to your workgroup templates folder (if your company uses that feature) or the regular Office Templates folder.
