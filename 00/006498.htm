' This imports a text file into Excel.

' Since ImportTextFile takes input parameters, it should be called from other VBA code

 

Public Sub ImportTextFile(FName As String, Sep As String)

 

Dim RowNdx As Long

Dim ColNdx As Integer

Dim TempVal As Variant

Dim WholeLine As String

Dim Pos As Integer

Dim NextPos As Integer

Dim SaveColNdx As Integer

 

Application.ScreenUpdating = False

'On Error GoTo EndMacro:

 

SaveColNdx = ActiveCell.Column

RowNdx = ActiveCell.Row

 

Open FName For Input Access Read As #1

 

While Not EOF(1)

   Line Input #1, WholeLine

   If Right(WholeLine, 1) <> Sep Then

       WholeLine = WholeLine & Sep

   End If

   ColNdx = SaveColNdx

   Pos = 1

   NextPos = InStr(Pos, WholeLine, Sep)

   While NextPos >= 1

       TempVal = Mid(WholeLine, Pos, NextPos - Pos)

       Cells(RowNdx, ColNdx).Value = TempVal

       Pos = NextPos + 1

       ColNdx = ColNdx + 1

       NextPos = InStr(Pos, WholeLine, Sep)

   Wend

   RowNdx = RowNdx + 1

Wend

 

EndMacro:

On Error GoTo 0

Application.ScreenUpdating = True

Close #1

''''''''''''''''''''''''''''''''''''''''''''''''''''''''''''''''

' END ImportTextFile

''''''''''''''''''''''''''''''''''''''''''''''''''''''''''''''''

End Sub

jllatham say:

Here is a macro that will do it.  It is set up to work through all worksheets in a workbook, so the 50 sheets isn't a problem.  If they are in 50 different workbooks, that's something else completely.

It is set up to copy from row 2 in each of the B, C and D columns, you can change that in the code if you actually want to either include row 1 also, or start farther down the sheet.  It will copy empty/blank cells in each of the 3 columns also.  If you need to skip those, a minor mod will be needed.

If you want to cleanup the 3 columns after the data has been copied, yet another minor mod.

Sub COPYColumns()
  Const firstRowWithData = 2 ' assumes labels in row 1
  Dim anyWS As Worksheet
  Dim copyRange As Range
  Dim CP As Integer
  For Each anyWS In ThisWorkbook.Worksheets
    For CP = Range("B1").Column To Range("D1").Column
      Set copyRange = anyWS.Range(anyWS.Cells(2, CP).Address & ":" & _
       anyWS.Cells(Rows.Count, CP).End(xlUp).Address)
      copyRange.Copy anyWS.Range("A" & Rows.Count).End(xlUp).Offset(1, 0)
    Next
  Next
End Sub

daniel g. grau say:

Starting Word with the Shift key pressed (called "Safe Mode") suppresses the effects of the Normal.dotm template and the .plist preference file, which are the most likely culprits when Word's problems are solved in this way.

Safe Mode also prevents add-ins from loading, so start in Safe Mode once again, then use Tools > Templates and Add-Ins in the main menu bar to sequentially deselect any add-ins you find there.

Finally, Safe Mode prevents loading of customizations (like toolbars) and the auto-correct list. These are less likely candidates for trouble.

Tend to these for now.
