How can I copy a hyperlink from one Excel sheet to another and hav the destination cell still be a hyperlink?

chip pearson say:

First off, I'm assuming you are inserting a hyperlink from the Insert Hyperlink panel item, not using the HYPERLINK worsheet formula. Given this, you cannot do what you want with a formula alone. You need a VBA procedure that will scan the input Source range and copy hyperlinks to the output Destination range. The following code does this:

 

Sub CopyHyperlinks()
    Dim Source As Range
    Dim Destination As Range
    Dim SearchRange As Range
   
    Set SearchRange = Sheets(1).Range("A1:A10") '<<< CHANGE
    Set Source = SearchRange(1, 1)
    Set Destination = Sheets(2).Range("B1") '<<< CHANGE
   
    For Each Source In SearchRange
        If Source.Hyperlinks.Count > 0 Then
            Destination.Hyperlinks.Add Destination, _
                Source.Hyperlinks(1).Address, , , Source.Text
        End If
        Set Destination = Destination(2, 1)
    Next Source
End Sub

Change the SearchRange range to the range containing the hyperlinks you want to copy and change Destination to the range where the new hyperlinks are to be placed.

ken sheridan say: The most efficient way of ranking rows is by means of a JOIN of two instances of the table or query.  The following is an example of ranking on the basis of two columns a Transactions table, ranking the rows by TransactionDate and TransactionAmount:

SELECT T1.TransactionDate, T1.TransactionAmount,
COUNT(T2.TransactionID)+1 AS Rank
FROM Transactions AS T1 LEFT JOIN Transactions AS T2
ON (T2.TransactionDate>T1.TransactionDate
OR (T2.TransactionDate=T1.TransactionDate
AND T2.TransactionAmount>T1.TransactionAmount))
GROUP BY T1.TransactionDate, T1.TransactionAmount, T1.TransactionID
ORDER BY T1.TransactionDate DESC , T1.TransactionAmount DESC;

Note that it is necessary to include a candidate key from the first instance of the table or query in the GROUP BY clause to ensure that all rows are returned distinctly.  Otherwise only one instance of each subset of rows with the same TransactionDate and TransactionAmount values would be returned.

The same result table can, less efficiently, be returned by means of a subquery:

SELECT T1.TransactionDate, T1.TransactionAmount,
    (SELECT COUNT(*)+1
     FROM Transactions AS T2
     WHERE T2.TransactionDate>T1.TransactionDate
     OR (T2.TransactionDate=T1.TransactionDate
     AND T2.TransactionAmount>T1.TransactionAmount)) AS Rank
FROM Transactions AS T1
ORDER BY T1.TransactionDate DESC,T1.TransactionAmount DESC;

You might find that the latter is more intuitive to write, and it does not need to reference a candidate key which might make things easier where you are dealing with a query rather than a base table.  The underlying logic in each case is the same, that the TransactionDate in one instance is greater than that in the other, or that TransactionDate values are equal and the TransactionAmount value in one instance is greater than that in the other.

I can't say whether the use of a 'calculated field' will make any difference to the correlation, as it is not something I would ever use, but if it is not possible, then you'd need to repeat the expression on which the 'calculation' is based rather than referencing the column by name.

As always, take a backup of any registry location (i.e. do an export) before making any modifications, in case things go wrong.

STEP 1

Close Edge.

Fire up regedt32 and navigate to ...

HKEY_CURRENT_USER\SOFTWARE\Classes\Local Settings\Software\Microsoft\Windows\CurrentVersion\AppContainer\Storage\microsoft.microsoftedge_8wekyb3d8bbwe\MicrosoftEdge\Protected - It is a violation of Windows Policy to modify. See aka.ms/browserpolicy

There are two entries under this key, shown below. Delete both 'ProtectedHomepages' and 'ProtectedSearchScopes' entries ...

Deleting these appears to delete current Edge "Search in the address bar with" settings and reset you back to searching with bing.

STEP 2

Load up Edge, and you should find that searching from address bar works, although it's Bing's search engine.

Next, as deleting the above entries wipes your search engine list, go to the search engine homepage of choice (i.e. www.google.co.uk) and then open up the settings in Edge (you have to have visited the search engine home page before it appears in Edge's list) and go to "View Advanced Settings", "Search address bar with", drop down the list, Add New and choose your search engine from list (i.e. Google Search).

I'd be tempted to click the 'add' button rather than 'add as default' as I had lots of fun repeating the above steps before it worked. Then make sure that you have your search engine selected in the drop down list for "Search address bar with", and try searching from the address bar.

STEP 3 (you might need to do)

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Internet Explorer\Main

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Internet Explorer\SearchScopes

Sure, they belong to IE, but Edge still has a lot of interaction with IE settings in Windows 10 from what I can see. You may have to then repeat Step 1 and Step 2 again, before you get everything working.

You may have to play about with the various steps as I was doing these steps in all sorts of order before I got my Edge to work with google.co.uk so I'm not sure on the exact combo of the steps, but I believe these steps are the key to fixing the issue.
