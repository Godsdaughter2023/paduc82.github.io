a. user say:

Doug's initial comment "You can set the format of the Publish Date Content Control in the Content Control Properties dialog" is probably still the thing you need to do, but it does depend on which problem you are seeing that you need to solve.


When you insert a "property" from Insert->Text->General->Document Property, (or on some versions, Insert->Quick Parts->Document Property) Word actually inserts a content control that is linked to a data item in the document's "XML Data Store".**

Each time you insert a control from that Document Property list, Word will create a new control with some default properties, but the control will always point to the same data item in the XML Data Store.

So if you want to display the value in the XML Data Store in two different ways (August 2, 2017 and August), you will need to have two different versions of the control.

So let's suppose you have a copy of the control set up with the "full date", i.e. so it displays August 2, 2017. Then you can copy/paste the control, then change the copy so that its date format is MMMM (you don't have to pick one of the items in the list - you can type it in under "Display the date like this"). When you change the date in the "full" version to, say, September 1, 2018, the date in the copy should change to "September". 

If you do not want the user to be able to change the month in the copy, you can also check the "Contents cannot be edited" box, and so on. You will also need to ensure that the "Store XML Contents in the following format when mapped" option is set to the Date or Date and Time option, not the "Text (same as display)" option.

So what you then need is to put a copy of the modified control everywhere where you just want the month to be displayed. If you want the *user* to be able to do that, you will need to provide another mechanism (because they cannot insert the modified version of the control using Insert->Text-General->Document Property). You could consider selecting the entire control and saving it as a Building Block.


Another problem you may have is that if the user types a 'date' manually (i.e. without using the date picker) they may type something that Word does not regard as a valid date. In that case, Word will just store the typed text in the item in the XML Data Store and replicate it everywhere. If that is the problem that you are trying to deal with, I do not think you will be able to solve it just using content controls or date fields, and your suspicion that you will need a macro is correct.

** In the case of the Publish Date 'property', Word also creates the necessary data item in the XML Data Store if it does not already exist.

petrib say: A password on a memory card is used to prevent the use of the card in another phone (in case you loose it, or it gets stolen). If you set it, you should also remove it on the original phone (with the same password that was used you set it), before using the card in a new (another) phone.

If you've lost or forgotten the password, then there is no easy way to recover any data from the card (if it was easy, there wouldn't be much point in a password feature like that in the first place, would there?).

It might be possible to reformat the card in some other device (most devices won't do/allow it, if the password has been set and is not supplied), but reformatting the card will also cause all data to be lost.

A data recovery service (such as Ibas might get the data for you, but the cost for using such a service will probably far exceed the price of the card and the phone (you can probably get many new cards and phones with that money).

doug robbins - mvp office apps & services (word) say:

The reason that error was occurring was because there were content controls in the document that were not of the Checkbox type.  To prevent it, the code needs to first check that the content control is of the wdContentControlCheckBox type.

That can be done by using the following modification of the code:

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)

Dim cc As ContentControl

With ActiveDocument

    For Each cc In .ContentControls

        If cc.Type = wdContentControlCheckBox Then

            If cc.Tag = "PM_No" And cc.Checked = True Then

                .Bookmarks("PM_Options").Range.Font.Hidden = True

            Else

                .Bookmarks("PM_Options").Range.Font.Hidden = False

            End If

            If cc.Tag = "PM_Yes" And cc.Checked = True Then

                .Bookmarks("PM_Options").Range.Font.Hidden = False

            Else

                .Bookmarks("PM_Options").Range.Font.Hidden = True

            End If

        End If

    Next cc

End With

End Sub

Note however that as the Content Controls are not mutually exclusive, and two of them are not really required for your application, it is better just to have one of them.

If you have another situation where you only want one of multiple items to be selected, see the following page of Greg Maxey's website:

http://gregmaxey.mvps.org/word_tip_pages/ADCAP_mutually_exclusive_content_control_option_buttons.html
