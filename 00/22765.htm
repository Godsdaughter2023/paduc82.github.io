But forget about using a label control and use an unbound text box control with the above as its ControlSource property.  If you use a label you'd just make extra work for yourself as it would require you to update the value of the label control's Caption property with code in an event procedure whenever the first or last name value changes, whereas the above will do so dynamically.

To make it look and behave like a label set its Enabled property to False (No ), its Locked property to True (Yes), its BackStyle property to Transparent, and its BorderStyle property to Transaparent if you don't want it with a border.

SELECT PersonID, Lastname & ", " & Firstname AS Person FROM tblPeople;

meioticoverload say:

Deselecting Snap to Grid works great (as always).

However, the other option (Control+Command+Arrow) was not working for me.

As I was typing up this reply, I actually figured out my problem. This option does not work if you are also running the program SizeUp (Irradiated Software) on your Mac. This keyboard shortcut is used by SizeUp and overrides the ability to use it in PowerPoint 2016 for Mac. Disabling Shortcuts or changing the Spaces Actions default keyboard shortcuts under Preferences in SizeUp fixes the problem. Of course, closing SizeUp also works.

Figured I'd post this anyway, just in case someone stumbled upon this same problem.
