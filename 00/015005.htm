hansv mvp say:

My guess is that you selected A3:D7 starting at a cell in the bottom row 7, so that the active cell within the selection is in row 7, e.g. D7. So the rule in row 7 refers to row 3, that in row 6 refers to row 2, that in row 5 refers to row 1, that in row 4 refers to row 0 but that doesn't exist so it rolls over to the last row 1048576, and finally that in row 3 refers to row 1048575.

To avoid this, select the range starting at the top row 3, so that the active cell within the selection is in row 3. That way, the formula correctly corresponds to the active cell.

daniel pineault say:

Add the following function into a standard VBA module

Public Function StripAllChars(strString As String) As String
'Return only numeric values from a string
'Source:
Dim lngCtr As Long
Dim intChar As Integer

    For lngCtr = 1 To Len(strString)
        intChar = Asc(Mid(strString, lngCtr, 1))
        If intChar >= 48 And intChar <= 57 Then
            StripAllChars = StripAllChars & Chr(intChar)
        End If
    Next lngCtr
End Function

Then, instead of querying your fields directly, add a new field

HomePhoneNo: StripAllChars([Home Phone])

and Search it.

Do the same for all your other phone number fields.

This is why it can be beneficial to only store the number, and not the formatting for such fields, and then use the Format control property in forms and reports to display them as you please for visualization.

revjenny say: I had virtually the same problem running an Access 97 database under windows 7.   A workaround, which may work for other combinations was as follows:

Edit the form and set the graphs 'Enabled' property to Yes and and 'Locked' to 'No', and the autoactivate to 'GetFocus'.   After these changes, when the form is opened normally, the graph can be editted simply by clicking it.   I found that after it had been editted, the graph displayed normally, until after the form was closed and subsequently opened.

The graph can automatically put in and out of the edit mode when the form is opened as follows:

In the form open event procedure add the line

 Me.TimerInterval = 200

The timer code is then set as follows.   This briefly edits the graph, then disables the timer and truns graph editing off.  

Private Sub Form_Timer()
 Me.<graph>.SetFocus
 <Anotherobject>.SetFocus
 Me.<graph>.AutoActivate = acOLEActivateManual
 Me.TimerInterval = 0
End Sub

where <graph> is the Name of the graph object
