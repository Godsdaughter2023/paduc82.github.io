brian tillman [outlook mvp 2007-2019] say:

Mail services usually make it fairly easy to export or import contacts and calendar, but not so with messages.

Outlook can certainly access Hotmail mailboxes.  See this: http://windowslivehelp.com/solution.aspx?solutionid=618dcd05-8494-4827-bba6-8072406b3fb7.  Outlook cannot easily pull mail from Yahoo.com mailboxes unless they're Yahoo Plus mailboxes.  Yahoo expects you to pay for POP access.  However, you can access free yahoo.com accounts using IMAP.  See this: http://help.yahoo.com/l/us/yahoo/mobile/comms/mail/imap-01.html.  It's a little buggy, but it does work.

Many mail providers will also allow you to use POP to pull messages from other providers' mailboxes.  For instance, if you log into Yahoo mail and click Options>Mail Options>Accounts, you can add references to other mailboxes and Yahoo will pull the mail in those mailboxes to your Yahoo mailbox.  This should work for both Yahoo addresses and non-Yahoo addresses.  You should be able to use this to pull your other Yahoo mailbox contents to a specific Yahoo mailbox.  This is possible for free yahoo.com addresses as well as Yahoo Plus mailboxes.  There will usually be an option to leave the messages on the other server as well, so one mailbox won't drain the other.

Unfortunately, Outlook is not a web browser.  There wa a time when it almost was.  Outlook used to use the Internet Explorer HTML rendering engine for rendering HTML and when it did, you could define the Home Page value of a folder (right-click a folder and choose Properties.  Select the Home Page tab) to be the webmail's login page (http://mail.yahoo.com/, for instance; see this: https://skydrive.live.com/?cid=4bf63ce8f68e1985&sc=documents&uc=1&id=4BF63CE8F68E1985%21132#cid=4BF63CE8F68E1985&id=4BF63CE8F68E1985%211162&sc=documents) and when you selected that folder, that web page would appear there.  You can still do that, but now Outlook uses Word as its rendering engine and it's not as good.  SOme things don't want to work correctly with some webmail providers.  Even when it does work, you can't move things between the webmail mailbox and other folders in Outlook.

a. user say:

> Regarding if it becomes a Window/Word API issue, what are the chances of getting a hotfix or the like to address that? I'm thinking close to nil.

I suspect you will need to go through a better support channel (this is basically "peer-to-peer", i.e. mostly volunteers) for a reliable answer to that. 

Further thoughts:
 a. (Almost certainly irrelevant, but...) in more recent combinations of Win/Word, the default location for data source files (i.e. the one that would open if you were trying to connect to a data source manually) is actually a "search folder", not a traditional folder. I don't know exactly what query is used to populate this folder with file names, but I have noticed that a newly created or even newly saved data source can take some time to appear. I expect this all relies on the Windows search and/or indexing engine. But in any case I think you would be using explicit pathnames rather than relying on Word to locate the file.
 b. I can imagine a couple of scenarios in which Word might encounter a sync problem.  I can't point you to specific KB articles, but it may be worth looking for Win32 APIs that ensure that various views of the file system are in sync. e.g. you might have problems if
     - your files are being delivered to a folder in a roaming profile (i.e. not directly to a local folder) 
     - your files are being delivered to a folder that Windows regards as being "on the Internet" (and, e.g., files in Windows network shares may be regarded as such these days)
