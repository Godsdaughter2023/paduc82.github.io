a_user_here say:

You don't really "transfer" your files to get them in to show in Picasa.

Your pictures should be under your Pictures folder (but they can be wherever you like).  You can see your Pictures folder by running File Explorer and expanding This PC -> Pictures.  You might also have pictures under OneDrive -> Pictures.

When you install/run Picasa by default it is going to look for pictures in your Pictures folder, but you can also control where it looks.  For instance you might want to add your OneDrive\Pictures folder.  When it sees a picture in a folder that it is monitoring it puts it in a database as a "pointer/link" to the actual file, and shows you a thumbnail for it.  Note the scanning can take little bit after you start Picasa to complete.

You can control the folders Picasa looks in: Tools -> Folder Manager

You can also use: File -> Add Folder to Picasa...

For downloading your pictures from a camera of USB drive you can in fact used any method you want to get them to your Pictures folder or other folder monitored by Picasa.

Also from inside of Picasa:

File -> Import From...

To get Windows to allow you to choose Picasa when you connect the camera/USB drive do this:

Right click on the Start button -> Control Panel -> View By -> Large icons -> Default Programs

Change AutoPlay settings -> under Removable drive select, Choose what to do with each type of media

On the Pictures: select, Copy pictures to your computer (Picasa3)

Under Memory card: select, Copy pictures to your computer (Picasa3)

2016-01-02- This an update to my request for answers to the community.

This came as a surprise but it seems that Windows Live Mail does have a naming convention for the alternate storage folder location. The constraint is that none of the component of the full path name of the designated folder should carry an accented letter such as "è" or "â". Such a constraint is not applied when assigning an alternate TEMP Windows folder for example so I did not give that aspect much attention. And the absolute lack of any feedback on this error from the WLM application did not help ... So here is the issue in details and its resolution:

1) Initially, the storage folder location I wanted to define in WLM was something like: D:/Système/WLM (note the accented "è" here). This folder is a brother of the very functional D:/Système/TEMP, also parented by the D:/Système folder. The TEMP folder was assigned without an issue while the WLM folder could not be assigned.

2) The solution is simply to create a new WLM compliant structure such as: D:/Systeme (note that the accent on "e" is removed) then assign/move TEMP and WLM components to that new structure. Et voilà ! Now works like a charm.

It is very possible that if I had exposed my accented pathname issue at start, some of the generous helpers of this community would have been able to point this constraint right away so sorry if I ommited this aspect as I initialy discounted it as a factor.

The sad aspect to this is that if for example, someone as I did created a folder structure for WLM, TEMP, etc. that has an accented letter in any of the parent folders involved in the full pathname of the WLM folder, that structure cannnot be used as an alternate location for WLM storage folders. I don't believe one can simply change such name to make it compliant if it is being used for the TEMP files for example. So, the only solution to keep the WLM folder together with other user assigned system related folders is to redo/rename/reassign everything in a WLM compliant fashion.

This is something to keep in mind for spanish, french, etc users who tend to define folder names and file names in their own language. Should be noted as well that if one sets-up an IMAP relationship with the Mail Server from the WLM client, this application will not accept any accented folder names (synchronized/duplicated from the server structure) and will actually rename locally any such folders replacing accented letters with numbers within the storage folder area. This can be very confusing when navigating the folders later on from Windows Explorer. Very messy looking in the end.

Thanks to all that have given time seeking to help me resolve that issue. Hope I can return the favor at some point on this community forum.

Happy New Year 2016 to all.

DISM.exe /Online /Cleanup-Image /RestoreHealth /Source:C:\RepairSource\Windows /LimitAccess
