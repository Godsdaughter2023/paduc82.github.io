Extracting a text from a string delimited by - (hypen) - Microsoft Community

sreekanth pillai say:

Extracting a text from a string delimited by - (hypen)

Extracting a text from a string delimited by - (hypen)

I got this formula from the net and it seems to be working fine but I just wish to know what the formula does. Can anybody explain.

=MID(LEFT(A2,FIND({"-","/"," "},A2&"-/ ",FIND({"-","/"," "},A2&"-/")+1)-1),FIND({"-","/"," "},A2&"-/ ")+1,255)

Thanks in advance.

rick rothstein, mvp - excel say:

First off, in case it confused you any, I had one section where I concatenated two hyphens on to the text when only one was needed (it didn't hurt to use two hyphens, it is just the second one wasn't needed). Here is the formula modified as I should have posted it originally...

=MID(LEFT(A2,FIND("-",A2&"--",FIND("-",A2&"-")+1)-1),FIND("-",A2&"-")+1,255)

Your question asked about two different parts of the formula, so maybe it would help to separate the arguments so you can see the individual parts. Here is the formula spaced to show the argument for the MID function call...

=MID(    LEFT(A2,FIND("-",A2&"--",FIND("-",A2&"-")+1)-1),      FIND("-",A2&"-")+1,     255)

The 255 belongs to this function call and it says to look at up to 255 characters in the first argument to the MID function. The text for the first argument is probably less than 255, but that doesn't matter for the length argument of the MID function... it won't look beyond the end of the text, so 255 acts as a maximum value... the assumption is the text in the first argument won't be more than 255 characters long. Now, your next question deals with this first argument to the MID function...

LEFT(A2,FIND("-",A2&"--",FIND("-",A2&"-")+1)-1)

The part of the argument we are really interested in is this...

FIND("-",A2&"--",FIND("-",A2&"-")+1)-1

which tells the LEFT function call how many characters to retrieve... everything in A2 in front of the second hyphen. The reason we need this is so that the MID function can then retrieve everything after the first hyphen from this returned text. The -1 says to use the character position just in front of the second hyphen, so the part we care about is this...

FIND("-",    A2&"--",    FIND("-",A2&"-")+1)-1

Again, I have separated the arguments with spaces for clarity. That second FIND function locates the position one past the first hyphen and set the starting point the first FIND will start seaching from; namely, the position after the first hyphen. Let's say the text in A2 is this...

ONE-TWO

then the first FIND will start looking at character position 5. So, if we just concatenated one hyphen onto the text making it this...

ONE-TWO-

it would have no trouble finding that second hyphen. However, what if the text were this...

ONE

Concatenating only one hyphen onto it would make the text this...

ONE-

and the second FIND function would still return 5 for the position from which to look for the second hyphen... but if you concatenated only one hyphen onto the text in A2, there would be not hyphen for it to find after the 5 character position, so it would error out. To prevent that, we just give it a second hyphen to find by concatenating two hyphens onto the text in A2; thus, the text being searched is this....

ONE--

Also note that doing this protect against the error when A2 contains nothing as well.
