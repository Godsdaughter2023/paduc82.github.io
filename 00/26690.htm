julie sheets say:
Hi Derrick,

The Resource Leveling command will NOT change the assignment unit.  After attempting to Level, you likely saw a message telling you that Project could not resolve the overallocation because you have the resource assigned at above his/her maximum units.

You will need to replace the resource at 100% with the same resource at 50%.

Turn off the Project summary task (if visible).
Select all tasks in the project by clicking the Task Name column heading.
On the Resource ribbon, click Assign Resources.
Select the resource you have assigned at 100%.
Click Replace.
Select the same resource and manually enter the new max units (50%).
Click OK.
Repeat as needed to replace each resource with his/herself at the lower assignment unit.

Project will recalculate the duration of the tasks assuming the tasks are not Fixed Duration.

We will start with the issue adding your Microsoft account to the phone.  There are two likely things you are running into.  One, you have the wrong password or your password is to long.  Two, you have two factor authentication (2FA) enabled.  I would start by verifying if you have 2FA enabled.  You can do this by going to http://accounts.live.com and signing into the account on your phone.  Once you are signed in, click on security info.  You should see a section regarding this security feature and an option to disable/enable.  If you have the option to disable then you will need to create an application password and use that as your account password.  You should find the app password tool on this same page. 

 

If you are given the option to enable, then it is time to change your password.  You can do this by clicking on the password option in overview.  When you set up your new password, be sure you set the password to something 13 characters or less.  After making this change, go to the phone and try adding your account one more time.

 

In regards to getting the contacts from your phone.  There is no internal memory that is accessible for contacts.  Contacts are to be associated with a cloud based account and backed up in this fashion.  To see if your contacts are attached to an account on your phone, go to the people's hub and access one contact at a time.  At the top of the contacts page you may see a listing of Windows Live, Yahoo, Facebook, etc....  This would indicate what account that contact is associated with.  If you find that these contacts are not currently associated with an account, you would just see phone along the top of the contact page, the contacts may have to be manually entered.  If you are able to get the Microsoft account added to the phone the contact will be associated with that account and synced.

 

When it comes to getting your media content from the phone you will need it to be recognized by the computer and software so you can sync.  From what you have described there are two things we should check out.  The first is that you have a proper cord.  There are charge cables and data cables.  A charge cable would not lead to the phone be recognized.  We suggest you get your hands on another cable, a sync cable, and try connecting the phone.  We would also suggest you try connecting to another computer.  This issue could be with the PC and connecting to another computer is the easiest way to determine this.

TRUST THE ZUNE CERTIFICATE

  1. Close the Zune software and disconnect your phone from your computer.
  2. Open Certificate Manager. To do this, click Start, type certmgr.msc in the Start search bar, and then press yourEnter key.

    Windows 8 users:     On the Start screen, type Certificate, tap or click Settings in the right column, and then tap or click Manage computer certificates.
  3. In the pop-up window, tap or click the Personal folder in the left column, and then double tap or double click theCertificates folder in the right column.
  4. Click the heading that says Issued By to sort the column, and then look for one or more certificates that read "zune-tuner://windowsphone/...".
  5. Right-click the certificate file and click Copy. If there is more than one file, hold the Ctrl key, click each file, then right-click a file and clickCopy.
  6. Tap or click the Trusted Root Certification Authorities folder in the left column, right-click theCertificates folder, and then click Paste.
  7. Click Yes to install this certificate.

Create a new certificate

  1. Close the Zune software and disconnect your phone from your computer.
  2. Open Certificate Manager, if it is not already open. To do this, clickStart, type certmgr.msc in the Start search bar, and then press yourEnter key.

    Windows 8 users:     On the Start screen, type Certificates, tap or click Settings in the right column, and then tap or click Manage computer certificates.
  3. In the window, tap or click the Personal folder in the left column, and then double tap or double click theCertificates folder in the right column.
  4. Click the heading that says Issued By to sort the column, and then look for one or more certificates that read "zune-tuner://windowsphone/...".
  5. Right-click each certificate file, click Delete, and then click Yes.
  6. Tap or click File in the menu, then Exit.
  7. Click Start and in the search box enter %userprofile%\Application Data\Microsoft\Crypto\RSA, and then pressEnter.

    Windows 8 users:     Right-click in the lower left-hand corner of the screen and tap or clickRun, or type Windows Key + R, and then type the string above.

    Note For some enterprise customers, the string will be %userprofile%\AppData\Roaming\Microsoft\Cryptoinstead.
  8. Find the folder that begins with "S-1-5-21-".
  9. Right-click the folder and then click Rename. Rename the folder so that it ends in .bak. This means you will likely be adding ".bak" to the end of the number.

    Note If there is more than one folder that begins with "S-1-5-21-", try this procedure with one folder at a time. If it does not work, rename the folder to the original name and try the next folder.
  10. As soon as you're done, open your Zune software and connect your phone to your computer. This creates a new certificate and folder.

If you continue to have connection issues, please go to device manager and check for error codes.
