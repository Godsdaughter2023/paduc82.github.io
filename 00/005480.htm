norman david jones say:

Hi JW,

Headers are on row 11.

Data (Names, dates, etc) is in A:I.  

Based on last row used in range A:I up to row 11, search all rows and delete entire row if J:P is the range is blank.

On a copy of your data, try:

'=========>>
Option Explicit

'--------->>
Public Sub Tester)
    Dim WB As Workbook
    Dim SH As Worksheet
    Dim Rng As Range
    Dim rCell As Range
    Dim delRng As Range
    Dim LRow As Long
    Dim CalcMode As Long

    Set WB = ThisWorkbook
    Set SH = WB.Sheets("Sheet1")                                           '<<==== Change

    With SH
        LRow = .Cells(Rows.Count, "A").End(xlUp).Row
        If LRow < 11 Then Exit Sub
        Set Rng = .Range("A11:A" & LRow)
    End With
    On Error GoTo XIT
    With Application
        CalcMode = .Calculation
        .Calculation = xlCalculationManual
        .ScreenUpdating = False
    End With

    For Each rCell In Rng.Cells
        With rCell
            If Application.CountA(.Offset(0, 9).Resize(1, 7)) = 0 Then
                If delRng Is Nothing Then
                    Set delRng = rCell
                Else
                    Set delRng = Union(rCell, delRng)
                End If
            End If
        End With
    Next rCell

    If Not delRng Is Nothing Then
        delRng.EntireRow.Delete
    Else
        'do nothing
    End If
XIT:
    With Application
        .Calculation = CalcMode
        .ScreenUpdating = True
    End With
End Sub
'<<=========

===

judith_c. say: Hi RikDatta,

Welcome to Microsoft Community. I understand that you want to get rid of the notification that informs you that your emails are in the POP folder. 

The notification you received means that you are accessing your account via POP connection. Generally, if you connect your account via POP connection using an email client or device, it will delete the messages from the server unless you've told the email client not to.

You need to set the option from your email client to leave a copy to the server. Also, set your Microsoft account to Don't let devices or apps delete messages from Outlook. If the app or device is set to delete, Outlook will move the messages to a special POP folder instead. Here's how to enable this:


1. In your Outlook.com Inbox page, click the gear icon .

2. Click More mail settings.

3. Under Managing your account, click Connect devices and apps with POP.

4. Under POP, click Enable.

5. Under Deleting downloaded messages, select Don't let devices or apps delete messages from Outlook. If the app or device is set to delete, Outlook will move the messages to a special POP folder instead.

6. Click Save.


You can also read this thread for more information regarding POP connection:


Why do messages from my inbox keep repeatedly getting moved to my "POP" folder?


Let us know if you have further questions.

karl timmermans [outlook mvp 2012-2018] say:
Why, in the world, would anyone ever really want the 'last modified' date to change for any reason whatsoever, other than "you modified the data associated with the contact".

Explanation is actually a very simple one. <Last Modification Date> indicates the last time <any> data in an any Outlook item (contact or otherwise) has changed for whatever reason however done This can result for any number of reasons starting with changes via the UI, running code a user wrote themselves, to any of  an endless array of 3rd party programs (addin or external). Same rules apply as would be the case in any database application where information has been modified.

To put this another way, if one ended up in a court of law and information status was being reviewed between two dates, there are only two states - changed and unchanged. Can't be a scenario where the explanation is "well yes, all kinds of things may have been physically changed by all kinds of different programs and the data may or may not be different <but> "I didn't change anything personally via the user interface (honest) so it doesn't count"

If on the other hand, whatever is changing your data is "updating" a contact item when no data has changed (i.e. app opens the contact item and and <updates> the item whether it needs to or not) - that is something that needs to be taken up with whomever provides the application.
