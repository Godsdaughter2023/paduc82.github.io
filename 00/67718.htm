If it is, then you need to boot from the disc.

If not, you need to use Windows 7 32 bit.

32 vs 64 bit

http://windows.microsoft.com/en-US/windows7/32-bit-and-64-bit-Windows-frequently-asked-questions

Is my PC running the 32-bit or 64-bit version of Windows?

http://windows.microsoft.com/en-us/windows7/find-out-32-or-64-bit

http://windows.microsoft.com/en-US/windows7/Installing-and-reinstalling-Windows-7

Windows 7 64 bit comes in handy when you need to address at least 4 GBs or more of RAM. Windows 7 32 bit can utilize up to 3.2 GBs of RAM. Because the memory address space is much larger for 64 bit Windows, that means, you need twice as much memory than 32 bit Windows to accomplish some of the same task, but you are able to do so much more, you can have more applications open, do things like run an Antivirus scan in the background without it affecting your system performance. Windows 7 64 bit is more secure too, malicious code cannot easily infiltrate it, drivers are more reliable since they must be signed before they can work with 64 bit Windows 7.

As for compatibility, you will need 64 bit device drivers for any hardware devices you might have. Also, there is no 16 bit subsystem in Windows 7 64 bit, which means, your applications must be 32 bit only, not 16 bit installer or uninstallers.

marsf say: I had a similar problem that I was just able to fix. I had a file type that would open the right program, but then deleted the program (just like you moving the program) and tried to re-associate the file with a newer version of the program (in your case where you moved the program to). It wouldn't ever do it even when I used the "open with" and "browse" to find exactly where the program was located.

I was able to fix the program by manually editing the system registry, which didn't know I had deleted (moved) the program. After updating the file location for the program to where the program was now located it fixed the problem. "Open with" and "browse" then worked (in fact I didn't have to browse because it could find the program again-> you might not even have to do "open with" if you haven't accidentally associated it with another program (like I did).

You will need administrator access/permission to use it. Then press "ctl-F" or menu edit and find and type in the name of the program that you changed the location of. You can do menu, find next or F3 to repeat the search to find more entries. You should see somewhere that points to the old location of the program, which would be why it doesn't work anymore. Change it to the correct location. I change an entry by double clicking on the Name in the same row, which in my case said "(Default)". I only had to change it in one place, but I double checked that it changed it everywhere by continually pressing F3 to find everywhere that referenced it.

Hopefully that works for you, too. good luck.
