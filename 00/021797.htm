ken sheridan say: One additional point:  it's always a good idea to declare parameters in a query, and especially so with date/time parameters as this (a) avoids the parameter values being misinterpreted (we've had two cases posted here of this happening with a BETWEEN....AND operation in the last few weeks where parameters have been misinterpreted as text), and (b) it takes account or regional variations in the date format used on the system.  So a query would be along these lines:

PARAMETERS Forms.MyForm.Date_Start DATETIME,
Forms.MyForm.Date_End DATETIME;
SELECT *,
Forms.MyForm.Date_Start AS StartDateTime,
Forms.MyForm.Date_End AS EndDateTime
FROM MyTable
WHERE MyDateTimeField BETWEEN Forms.MyForm.Date_Start
AND Forms.MyForm.Date_End;

Note that a BETWEEN....AND operation is inclusive, so with your criterion of 01/11/2011 07:00 AM till 01/12/2011 07:00 AM this would include rows with a date/time value of 07:00 AM on all days in the range.  If you want the rows on the final day to be those
AND DATEADD("s",-1,Forms.MyForm.Date_End);

to subtract 1 second from the end of the range.

Base the report on a query like the above, and open the report in print preview from a button on the form in which the date/time values are entered.

An alternative would be to base the report on a query without parameters and open it filtered to the date range.  If doing this then be sure to format the date literals in an internationally unambiguous format, such as the ISO format for dates of YYYY-MM-DD hh:mm:ss.  So the code to open the report would be:

Dim strCriteria As String

strCriteria = "MyDateTimeField Between #" & _
    Format(Me.Date_Start,"yyyy-mm-dd hh:nn:ss") & "# And #" & _
    Format(DateAdd("s",-1,Me.Date_End),"yyyy-mm-dd hh:nn:ss") & "#"

DoCmd.OpenReport "YourReportName", _
    View:=acViewPreview, _
    WhereCondition:=strCriteria

If doing this include two unbound text boxes in the report, e.g in its header, which reference the two controls on the form to show the range in question in the report's output.

How do you want to publish the report's output to your colleagues?  You could for instance generate a PDF file from the report in print preview, using the 'PDF or XPS' item on the External Data ribbon, and circulate this to your colleagues as an attachment to an email, or make it available in a shared location on your network.

ken sheridan say: NB:  I've edited the above to include the hours, minutes and seconds in the format strings, which I'd omitted.  Not that the ISO standard uses case to distinguish between months and minutes, whereas Access, not being case sensitive, uses m for month and n for minute.

There are some causes of pictures not displaying in Microsoft Outlook email messages:

• You are blocking external content.
• You have Word's Picture placeholder setting enabled.

You may try to follow the steps below to resolve the issue:

Method 1: Block or unblock automatic picture downloads in email messages

• You may follow the steps listed under Unblock picture downloads for all messages

Method 2: Checking the Picture Placeholder setting

If you do not see a red x but have a blank box shaped/sized like the graphic or photo, you have Show picture placeholders enabled. (A great feature if your Internet connection is slow.)

Take note that this dialog is accessible from a COMPOSE MESSAGE window, not the main Outlook window.

Outlook 2010, 2013, 2016

These steps work for Outlook 2010, Outlook 2013, and Outlook 2016 (including Outlook which is included in Office 365 subscriptions).

1. Click the New Email button or press Ctrl+N while viewing the Inbox.
2. Go to the new message's File, Options dialog.
3. Click Mail, then Editor Options.
4. Select Advanced and look near the bottom of the dialog for the option to Show Picture Placeholders.
5. Uncheck the box to Show picture placeholders
6. Close the dialog then close the message.
After changing the settings, close the message. Do not send it. This, like many other message settings (zoom, etc), will apply only to the current message if you send it. You need to close the message to change the global setting.
