Accessibility issues in Build 10041 - Microsoft Community

sharief k say:

Accessibility issues in Build 10041

What are the known issues regarding Ease of Access and accessibility in the newest build of Windows 10 Technical Preview (Build 10041)?
rohit siddegowda say:

Hi Sharief,

Build 10041 currently has several known issues regarding screen readers, and other

  • Magnifier is currently not working correctly.  When you start Magnifier, the Magnifier lens may appear black and artifacts may appear around the screen, requiring a reboot of the PC.  Customers that rely on Magnifier are not encouraged to install Build 10041 at this time.  Microsoft is aware of the issue and is investigating a fix. 
  • Narrator and some other screen readers do not correctly read user names at the lock screen.  They instead read user names as something like “1 of 2”.  When you type in the Feedback app, Narrator does not read the text back as expected.
  • You may be unable to unlock Windows if the JAWS screen reader is running.  There is a known issue that prevents keyboard focus from moving to the password box when unlocking or signing into the PC if the JAWS screen reader is running.  The only workarounds are to either use Narrator instead of JAWS, or to press the ALT + SPACE keys and choose Restore; then press ALT + SPACE and choose Minimize in order to minimize the JAWS window.  This will then allow keyboard focus to go to the password box.
  • You may be unable to select items within the Settings app by using only keyboard input.  This also applies to any app that uses XAML listview controls, due to a known issue that prevents the control from responding to certain keyboard events.  As a result, anyone who is using only a keyboard to navigate the Settings app will be unable to adjust some settings.  There is no current workaround for this issue, but Microsoft is aware and is investigating a fix.

andreas killer say:

But my colleague insists we need to rebuild the file to make it a "64-bit spreadsheet".

That's nonsense!

An Excel file is an Excel file, there is no difference between 32-bit and 64-bit.

You only need to make a distinction if you use special VBA code that calls external DLL's directly.
You can check that very easy, search for "Declare" in your VBA code.
If that keyword doesn't exists, forget it, your file is 64-bit compatible.
Otherwise you have to rewrite your VBA code, but you don't need to make any changes in the sheets anywhere!

The only difference between Excel 32-bit application and Excel 64-bit application is that the 64-bit version can address more memory, the 32-bit version is limited to 2GB of memory (means RAM, not file size!). That's all.

Andreas.

harry stevens say:

Hi Terranitem,

I can suggest two things that you can do for your situation :

1. Rename your account

- By renaming your account to another address (ending in @outlook.com), that 'someone' who is using your address will no longer be able to do so, since your original address won't exist anymore. So, you will still be able to use your Microsoft services without 'migration'.

2. "Partial" migration

- Technically, you can't 'move' everything on your old account to a new one. You can, however, move specific services instead. For example, you can move an XBOX Live Account to another Microsoft account. Transferring an Office 365 subscription to another account, according to what others had mentioned on this forum, is very difficult. You can contact Accounts and Billing Department regarding the transfer of the subscription and apps, but I think they'll just refund you those and let you rebuy them once more.

Moving OneDrive files, I don't think that's possible automatically. You may have to download everything that you have, then upload it to the new account.

I think the renaming will be the easiest option that you can do. Here's how you can do it.

1. Login to "https://account.live.com".

2. Click on "Manage your sign-in address"

*If you are asked for a code, please go ahead and provide it on the process.

3. Click "Add email alias".

4. Type in a new username for your account. Take note that you can only add in an alias that ends in @outlook.com.

5. After adding the alias, click the text next to it "Make Primary".

6. Click on "Remove" next to the old address.

So after renaming, everything under your Microsoft account will stay in tact, only the login address has changed. But if you have third party services that was using your old address, you will need to contact their respective support regarding the rename, as they will still try to use the old address of your Microsoft Account.

daniel g. grau say:

If you access the repeating footer of any section but the first, you'll see the light blue tab says "Same as Previous." That means Word is taking the immediately preceding footer as a template for the current one (except for the value of the page numbers).

You need to unlink that footer from the previous one. To do that, while you are in the footer, click on the Ribbon's "Header and Footer" purple tab and take the checkmark off from "Link to Previous." Once you do that, adding a letter to the footer will no longer affect the preceding footer.

Repeat in every subsequent section. (Note: The purple tab includes convenient Navigation buttons.)
