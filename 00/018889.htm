Delete duplicates but leave one - Microsoft Community

spark0227 say:

Delete duplicates but leave one

Hello

I am combining two tables into one and I used created two append queries in order to do that.

I am not an sql master nor an expert in vba... I just consider myself as a power-user...

Anyway,, I combined two tables and these are fields: invoicen, startdate, enddate, description, rbtdescription, totalvalue, vendor, dpmt, buyer, and pmtdate.

there are some duplicates with same invoicen and I want to delete the duplicates but leave one record alive.

How can I obtain this result? 

scottgem say:

use the query wizard and create a Find duplicates query. Use that query to create a temp table (using SELECT DISTINCT) of the duplicates. The table should have one record for each duplicate.

Run a Delete query on your main table (make a backup first), that deletes all records that match the records in your temp table. Finally Append the records from the Temp table to the main table.

This could have been avoided had you set the Invoice # field to no dups BEFORE you combined the tables. 

spark0227 say:

I can't do that because the duplicates are coming from two different tables. there is no duplicate record in each table.. when I combine them to create this one combined table, duplicates appear there.

how do I create a temp table? I am not sure how to follow your instruction 

scottgem say:

You can't do what? You said you used Append queries. I assumed that you created an empty table, then ran two Append queries to append the data first from one table then the other. So I don't see why you could not set a unique index (No Dups) on the Invoice Number in the destination table. Then any duplicates would have been rejected when ran the second Append query. It might be easier to empty that table, set the value to unique then run your Append queries.

If you want to follow my original instructions use a Make Table query using the Find Duplicates query as the source.

ken sheridan say:
is there a different way to do this.. using delete query? 

Yes, provided that the table has an autonumber primary key column, e.g. InvoiceID:

DELETE *
FROM YourTable AS T1
WHERE InvoiceID <>
     (SELECT MAX(InvoiceID)
      FROM YourTable AS T2
      WHERE T2.invoicen = T1.invoicen);

jeeped say:
You can imitate more than 3 conditional formatting rules in Excel 2003 by mimicking the CF behavior with a Worksheet_Change macro event. Right-click the name tab of the worksheet and choose View Code. When the VBE opens, paste the following into the pane titles something like  Book1 - Sheet1 (Code),

Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Target, Me.Columns(2)) Is Nothing Then
        On Error GoTo FallThrough
        Application.EnableEvents = False
        Dim b As Range
        For Each b In Intersect(Target, Me.Columns(2))
            Select Case b.Value
                Case (-1)
                    b.Offset(0, -1).Interior.ColorIndex = 9
                    b.Offset(0, -1).Font.ColorIndex = 2
                Case 3
                    b.Offset(0, -1).Interior.ColorIndex = 5
                    b.Offset(0, -1).Font.ColorIndex = 2
                Case 4
                    b.Offset(0, -1).Interior.ColorIndex = 10
                    b.Offset(0, -1).Font.ColorIndex = 2
                Case 5
                    b.Offset(0, -1).Interior.ColorIndex = 45
                    b.Offset(0, -1).Font.ColorIndex = 1
                Case 7
                    b.Offset(0, -1).Interior.ColorIndex = 1
                    b.Offset(0, -1).Font.ColorIndex = 2
                Case Else
                    'do nothing
                    b.Offset(0, -1).Interior.Pattern = xlNone
