The quick answer is - don't use Value List for combos. They are almost never appropriate for use in this manner, and IMO should have never been included in Access.

If you need to restrict values that go into that field, then create a table that stores all the values and use that as the Rowsource of a combo if you need to restrict values that go into that field. Your macro would function properly at that point, since it'd be nothing more than a Text field.

scottgem say:

2nd. I took on board the overarching suggestion you both made to reformat the field to reference a linked table.

I was not looking forward to doing that because as you are aware, you start with an project and soon it becomes larger as you plan for more events that the client asks for. Dependent tables and fields require a systematic repair.


I'm not sure I follow what you are saying. One of my mentors taught me that everything should be in a table. Data, settings, parameters etc. Your database program can then read these from a table to do what is necessary. As you have noted, its a LOT easier to just add items to a table then to adjust a Value list.

But I'm curious here about how that solved your problem. As I said, an MVF is a special type of control. To pull the list of selected items from an MVF, you have to use the Value property. For example. If you were to create a query from this table, you would see, in the field list:

Reason

     Reason.Value

To show the selected values as a comma separated list, you would need select Reason.Value.

So changing the Rowsource of the Combo to a table would not change that. 

The foundation of an efficient database is the table structure. You need to get that right before you start designing the rest of the application. While its is possible that you may need to modify the structure later, you will have less headaches if you get the table design right in the first place. As I said, your table design is off. if you are repeating data from record to record, then there should be a parent and child tables.

jllatham say:

Two possible solutions:

 

Sub JustClearEntries()

  Dim anyWS As Worksheet

  For Each anyWS In ThisWorkbook.Worksheets

    'this only clears entries/formulas, not formatting

    anyWS.Cells.ClearContents

  Next

End Sub

 

If you really want to make them "clean" the .Clear does away with everything, values, entries, formulas, formatting and all:

Sub ClearTheSheets()

   Dim anyWS As Worksheet

    For Each anyWS In ThisWorkbook.Worksheets

      'this only clears EVERYTHING
      anyWS.Cells.Clear

  Next

End Sub
