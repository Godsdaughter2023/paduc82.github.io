brian tillman [outlook mvp 2007-2019] say:

Those non-delivery reports in no way indicate your computer or mail account has been compromised.  All it means is that your address is being "spoofed"; that is, some spammer found your address somewhere and is using it as the sending address of the spam.  Some of the recipient addresses don't exist and so, because your address appears as the sender, the receiving mail system sends a non-delivery report to you.

There's nothing you can do to prevent this spoofing, but it's harmless.

If statement checking for zero not working - Microsoft Community

beckjr say:

If statement checking for zero not working

I typed the following formula =If(f7="","",if(f7-g7-h7)=0,0,"not in balance")) in cell i7.  It works sometimes and doesn't work sometimes.  For example, if f7=1561.54, G7=1405.39, and H7=156.15, it should return a value of zero, but it returns the "not in balance".  It appears the calculation returning the value of zero, is intermittent.  Is this an issue others have seen, and is it resolvable?
joeu2004 say: Beck wrote:
I typed the following formula =If(f7="","",if(f7-g7-h7)=0,0,"not in balance")) in cell i7.  It works sometimes and doesn't work sometimes.  For example, if f7=1561.54, G7=1405.39, and H7=156.15, it should return a value of zero, but it returns the "not in balance".  It appears the calculation returning the value of zero, is intermittent.  Is this an issue others have seen, and is it resolvable?

First, your formula syntax is incorrect.  I don't know if that is just a posting error, or if you copy-and-pasted the formula from the Formula Bar, as you should.  Well, it must be just a posting typo; otherwise, there is a syntax error.  I assume the formula is:

=If(f7="","",if(f7-g7-h7=0,0,"not in balance"))

Second, it is not uncommon for F7-G7-H7 to not be exactly zero.  (See explanation below.)

If you expect the expression to be zero within 2 decimal places, you should write:

=If(f7="","",if(ROUND(f7-g7-h7,2)=0,0,"not in balance"))

Explanation....

Excel uses 64-bit binary floating-point to represent numbers.  Consequently, most non-integers cannot be represented exactly.  This problem is compounded when the non-integers (and division of integers) are used in arithmetic.

Moverover, simply formatting to 2 decimal places rounds the displayed value, but not the underlying actual value.

So you need to use the ROUND function explicitly when you want the arithmetic to be "accurate" to a specific number of decimal places.

My internet printer requires CMYK colors. How do I accomodate - Microsoft Community

martinsburg pa say:

My internet printer requires CMYK colors. How do I accomodate

My internet printer requires CMYK colors.  How do I accomodate?  The problem is that Yellow uploads/prints as a Yellow/Green.  Help?
steve rindsberg say:

PowerPoint doesn't work in CMYK; it does RGB only, and some RGB colors simply don't reproduce well in CMYK.

A couple of things you might try as workarounds:

1) Companies like TrueMatch and Pantone sell CMYK color sample charts and may also provide recommended RGB equivalents.  You might be able to use something like that to work backward from choosing the sample you want to the RGB value (theoretically) required to produce it.

2) Export the PPT slide to e.g. a PNG image and then modify that in Photoshop or a similar program that understands CMYK.

3) Create a "ringaround" of the color(s) you want to work with, have the printer print that, and use that to choose the color you actually want to use.

A ringaround is a slide with a rectangle or other shape in the middle set to the color you think you want (in RGB, of course).  Then you make a series of dupes of the shape, move each to the right and bump each new shape's Red value up by, say, 10.  Do that again only move 'em to the left and subtract 10 from the Red value.  Then do it up and to the right +green and down to the left -green, and repeat for blue.
