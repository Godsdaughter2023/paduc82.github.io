sudeshkolte say:

LUMIA 2520 tab has this common problem of 'plugged in but not charging' issue.

Over a lot many trials and errors I could settle with a permanent solution this problem.

The battery shows 0% charged , charging not happening while it is connected to the mains. Actually this problem is because of battery driver function. When the charger is left connected to the tab for a longer duration after full charge, the battery gets seemingly overcharged. That's where the problem starts.

Solution: 1. Leave the tab without charging for two to three days. (The battery would eventually drain a bit) Connect the charger while powering the tan 'on'. Disconnect the charger immediately after you get to the desktop screen. Whrooooom...... your charger indicator is showing 99% and tab runs well on batteries.

On a safer side do 'power on - sound up' battery reset & 'power on - sound down' tab reset before doing the above gamut!

Solution 2: While the charger is disconnected, let the tab go on recovery (blue screen) mode. The full screen is lit and consumes battery. Leave this state for about 45 mins. Do the same procedure as in 1 above......   Bingo!...........

    First the bad news: Dropdown content controls don't support multiple selection. If you must have that feature, you must use either an ActiveX list box or a list box control in a userform.

    To clear the confusion...

    To start working with the ContentControlOnExit event handler that Paul mentioned, first double-click the ThisDocument icon in the project you're coding. At the top of the code pane, open the left-hand dropdown and select Document. Then open the right-hand dropdown and select ContentControlOnExit. Word will automatically insert the beginning code shown here:

    This procedure is automatically executed by Word whenever the cursor is about to exit any content control. The first parameter will contain the content control object from which the cursor is exiting. If the second parameter (Cancel) is set to True when the procedure ends, then the cursor will stay in the content control instead of leaving it.

    The other thing Paul mentioned, the SelectContentControlsByTitle function, is a member of the Document object. So if you type ActiveDocument and a period in the code, the popup list of members will include SelectContentControlsByTitle.

    This function returns a collection of all content controls in the document whose Title is specified in its parameter. Word allows more than one control to have the same title, so it's a ContentControls collection rather than a single ContentControl object. In Paul's reply, he assumes there is one and only one control named "ClientDetails", so he uses the index (1) to isolate the first control in the collection. In practice, you would need to verify first that the collection's .Count value isn't zero, or else trap the possible error.

    Note 1: The Title parameter of the function is case-sensitive. For example, a parameter value of "clientdetails" would not find a content control whose title is "ClientDetails" even if it exists.

    Note 2: There is a similar function, SelectContentControlsByTag, that returns a collection of content controls whose Tag value is specified.

    Insert a { DOCVARIABLE ClientDetails } field where you want the client details to be displayed and in the ThisDocument object for the document\template in the Visual Basic Editor, insert the following code

    Private Sub Document_ContentControlOnExit(ByVal CC As ContentControl, Cancel As Boolean)

    With ActiveDocument

        For Each CC In .ContentControls

            If CC.Title = "Client" Then

                Select Case CC.Range.Text

                    Case "Round House Nurseries"

                        .Variables("ClientDetails").Value = "Phone: 08 1111 1111" & vbCr & "Fax: 08 2222 2222" & vbCr & "Email Someone@somewhere"

                        .Range.Fields.Update

                    Case "Another nursery"

                    .Variables("ClientDetails").Value = "insert details"

                End Select

            End If

        Next CC

    End With

    End Sub

      

    Then, when you exit from the Client Dropdown, the DOCVARIABLE field will be populated with the details for the selected client.

    Note however, it is not possible to select multiple clients in such a control.  For that, you would need to use a UserForm containing a ListBox with its MultiSelect attribute set to appropriately.

    For information on UserForms, see the following pages of Greg Maxey's website :
