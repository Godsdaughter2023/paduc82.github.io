The reason the installation is failing is the installer cannot find the Microsoft Office 2011 folder or its various applications (Word, Excel, etc.). Three reasons come to mind:

  1. You have moved the "Microsoft Office 2011" folder from it's required location in the Applications folder
  2. You have renamed Microsoft Word, Microsoft Excel, etc.
  3. Your hard disk's name contains one or more characters that is confusing the installer (typically slashes, we love slashes). For instance, renaming "Macintosh HD" to "Macintosh/PC HD." Microsoft installers don't handle non-standard characters well.
  4. Your Office software is not at the version the installer is expecting. Check it in "Word > About Word." Then check the installer's requirements in the webpage where you downloaded the installer.

Look for one of those problems!

brian tillman [outlook mvp 2007-2019] say:
You can't.  IMAP accounts always have their own folders.  IMAP accounts are server-based and leave messages on the server.  Any IMAP mail client must show the exact same set of folders because IMAP accounts are two-way accounts, unlike POP accounts which are one-way (i.e., incoming messages flow from the server to the client and cannot move back).

Windows 10130 OneDrive not syncing down due to Permissions problem - Microsoft Community

fortytribes say:

Windows 10130 OneDrive not syncing down due to Permissions problem [resolved]

I first noticed this problem when using the scanner app, which worked a couple of times and then stopped working because it reported that it didn't have permission to save files to the Pictures folder (in OneDrive).  I worked around that by setting the save destination to the Desktop.

Then something REALLY scary happened. After loading 146gb of my personal photos to the Pictures folder in OneDrive, they upsynced fine, and so I decided to leave most of my older files online by using Selective Sync to turn off syncing of the those folders. Again... worked fine. They files disappeared from my computer but were nicely available online, and I could share them with others.

But when I needed to actively work on one of the folders, and turned the sync back on for it, it didn't come back to my computer and was stuck in the cloud. I soon realized that NONE of my files would resync to my computer and the only way to get them back was to manually download the files in each folder via the online interface. This is OK if you just have a few files, but when you have over a hundred gigabytes in a myriad of folders... uggh!

(of course I had a back up of all my photo archive on an external disc here, so no big deal - nobody should be that dumb to risk data on a technical preview).

SOLUTION:

After a week or so of putting around and searching I remembered the problem with the scanner app.  I also noticed that I was being asked for "Permission" whenever I wanted to create a new file or folder in Pictures. But this didn't happen with the Documents or Music folders in my OneDrive. So I compared the Permissions for the Documents and Pictures folder and saw the problem. The Pictures folder was not set to "Inherit". The fix was:

Right click on OneDrive\Pictures and open Properties

Click the Security Tab

Open the Advanced dialogue

Click the Enable Inheritance button (it complained with a few errors, but I just clicked OK)

After rebooting OneDrive started happily downloading all my missing photos.

So, there are two bugs here to report regarding Windows 10130/OneDrive. The first is that somehow the permissions on this folder got goofed up, which then interferes with OneDrive's ability to resync files back into the messed up folder.

The second bug is that when this occurs, OneDrive just keeps trying and trying and trying to sync, but is not able to tell the user the reason for the failure.  It should be smart enough to detect that permissions are needed, and either override them and somehow alert the user to the need to modify the permissions.

I hope this helps others who are experiencing similar problems.

Just to clarify, I subscribe to Office 365, and I don't know if that makes a difference in this cabungo.
