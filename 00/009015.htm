greginmich say:

The scheduled time for Automatic Maintenance, which includes a daily Defender Quick Scan, can be set here:

 

Control Panel > Security and Maintenance > Maintenance > Automatic Maintenance > Change maintenance settings

 

However, Automatic Maintenance is designed so that it only runs during system idle states in order to prevent the maintenance chores from degrading the system’s performance when the PC is in use – so it will only start at the scheduled time if you stop using your PC prior to the scheduled time.

 

But in Version1607, we have a new issue where the Windows Defender Scheduled Scan task (the Automatic Maintenance task) is only running sporadically at best. This issue is completely unrelated to the inherent issue with Automatic Maintenance not starting/completing the daily scan when the PC is in constant use. The failure that we're seeing in Version 1607 is immediate; and is specific to the scheduled scan task; and unfortunately also occurs when Windows Defender is running in its new Limited Periodic Scanning mode:

 

------------------------------------------------------------------------------------- 

MpCmdRun: Command Line: "C:\Program Files\Windows Defender\MpCmdRun.exe" Scan -ScheduleJob

 Start Time: ‎Mon ‎Oct ‎17 ‎2016 10:51:04

Starting RunStartStart: IdRunCommandScan is using default scan type: 1.

Scanning path as file: (null).

Start: MpScan(MP_FEATURE_SUPPORTED, dwOptions=16385, Timeout in days = 1)

MpScan() started

MpScan() was completed

ERROR: MpScan(dwOptions=16385) Completion Failed 80508018

MpCmdRun: End Time: ‎Mon ‎Oct ‎17 ‎2016 10:51:04

-------------------------------------------------------------------------------------

 

But we’ve discovered that this new issue is only present when the Windows Defender Scheduled Scan task is running under the default System user account – and once the user account for this task is changed over to the local administrator, the scan will run on a daily schedule right along with the other Windows Defender maintenance tasks (time permitting).

 

There are several viable options for dealing with the issue that we’re seeing in Version 1607:

 

1. Just ignore it and wait for a fix (there’s very little added risk involved in relying solely on Defender’s real-time protection).

 

2. Run manual scans occasionally.

 

3. Schedule a scan with the Create Basic Task Wizard that runs under the (default) local administrator account:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/automatic-update-broke-windows-defender/fad7be45-cf9e-4e5c-a6cd-cf2c2d83c345

 

4. Change the user account for the Windows Defender Scheduled Scan task to the local administrator – exactly as described here:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender-daily-quick-scan-windows-10-os/af9b6bf8-7880-48c1-afde-4043534b55d3

 

If your PC isn’t password protected, then first change the account to the local administrator, and then check the Do not store password. The task will only have access to local resources option.

 

Note that I would strongly advise against scheduling a batch file that immediately shuts down the PC after running a Defender scan. Simply scheduling a daily Quick Scan with the Create Basic Task Wizard is a sufficient workaround for the new issue in Version 1607 (as well as for the inherent issue with only-on-idle scheduling), and this scheduled scan can also be run in the familiar Windows Defender UI by using one of these command lines:

“C:\Program Files\Windows Defender\MSASCui.exe” -QuickScan

“C:\Program Files\Windows Defender\MSASCui.exe” -UpdateandQuickScan

teylyn say:
Hello,

There is no need to change the pivot data source. The month name alone can lead to wrong results if your data spans more than 12 months. 

Instead, just group the pivot table by date: right-click any of the date labels at the top of the pivot table and select Group. Highlight "Years" and "Months" and click OK

If your data spans more than one year, for example, you have data for January 2011 and January 2012, then you will see this ordered nicely in the pivot table in a multi-level heading, starting with Year and then showing the months for that year. 

Grouping also allows for different units like quarters, days and even hours, minutes and seconds. This is quite a powerful tool that does not need any changes to the source data.
