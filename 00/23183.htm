jllatham say:

It's possible there's only 1 worksheet in the workbook?  Thus the attempt to delete Sheets(2) would fail.

The other thought I have is that the workbook is protected from changes to its structure.

Third possibility: Sheets(1) is hidden.  So your process would attempt to delete all visible sheets (and maybe some hidden ones also).  But a workbook MUST have at least 1 visible sheet.

Perhaps modify the code to something like this

Public Sub DeleteSheets()
Dim Counter

If Sheets.Count > 1 Then

Sheets(1).Visible = xlSheetVisible ' make sure 1 sheet will be visible

Application.DisplayAlerts = False ' do outside of the loop

For Counter = 2 To Sheets.Count
    Sheets(Counter).Delete
Next Counter

Application.DisplayAlerts = True ' let system alert now

End If

End Sub

zorvek say:

If there is only one sheet the VB code will not fail as the For loop will never do an iteration since the end value is less than the start value.

Here is what I use to delete all but the first sheet:

Public Sub DeleteSheets()
    Application.DisplayAlerts = False
    Application.ScreenUpdating = False
    ThisWorkbook.Sheets(1).Visible = xlSheetVisible
    While ThisWorkbook.Sheets.Count > 1
        ThisWorkbook.Sheets(2).Visible = xlSheetVisible
        ThisWorkbook.Sheets(2).Delete
    Wend
    Application.DisplayAlerts = True
    Application.ScreenUpdating = True
End Sub

There is probably something else causing the error. If it isn't a hidden sheet problem as discussed by JLLatham, please post the error and a description of your workbook.

neha singh msft say:

Hi Tiffany,

When you have an Excel data source that contains formatted numbers like percentages and currency values, some of the numeric data may not retain its formatting during a mail merge. You’ll get the mail merge results you want and can preserve the format of your numeric data in Excel by using DDE.

  1. Open Word, choose File > Options > Advanced.
  2. Under General and select the Confirm file format conversion on open check box.
  3. Choose OK.
