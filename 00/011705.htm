You can do that, but you have to have a date field that tells when the item was added.  If that is the date/time when the record was created, all you need to do is set the field (in table design) DefaultValue property to:  Date() or Now(), depending on if you need to know the time of day when the record was added.

Word 2013 List layers and font formating

Is there someway where I can match the list content (i.e. what I type) with the font formatting of the numbering of the list? In other words I want to say: all type on level one should be Futura Light 16 pt, all type on level two Helvetica Neue Thin 12 pt.. and etc.

suzanne s. barnhill say:

To clarify what Pam and Doug have said, you must have a separate style for each level in your multilevel list. You can format that style any way you like (except that any indent, hanging indent, or first-line indent must be set in the Define New Multilevel List dialog); then, in the Define New Multilevel List dialog, link the appropriate style to each level of the list. Shauna's article makes it clear how to do this.

You do not need to (and should not) define the number font for any level of the list; each level will, by default, use the font and font size specified in the style linked to that level.

FWIW, I would not advise combining Futura and Helvetica Neue in the same document. By and large, you should choose a single serif and a single sans serif font for most of the text in any document; using two sans serif fonts just makes readers uneasy; they know something is wrong even if they can't put their finger on it.
