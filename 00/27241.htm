ronrosenfeld say:

To try to answer some of your questions:

With dates, it's usually better (more robust), to refer back to the initial date as I did, changing the number to be added, rather than make each date dependent on the immediately preceding entry

If the date formulas are separated by a known and fixed number of rows, you can adjust the formulas I provided to account for that fixed difference.  I chose ten rows as you alluded to entries in B11 and B21 in one of your posts, but the formulas are easily adapted.

And no, there is no "easy" way of having B11 see A12 as a date.  A12 is not a date, it is a text string.  The text string would have to be deconstructed, reassembled as a date, and then turned into a different text string to match the type of formatting you want.  Much simpler to refer back to the initial date for that information.

However, if you really wanted to do it, this would be the formula:

B11:  =TEXT(DATE(RIGHT(A12,4),MID(A12,4,2),MID(A12,7,2)),"yymmdd.") &CHOOSE(WEEKDAY(DATE(RIGHT(A12,4),MID(A12,4,2),MID(A12,7,2))),"Sn","Mn","Tu","Wd","Th","Fr","Sa")

This part obtains the date from A12; the remainder of the formula is to produce your formatting

   =DATE(RIGHT(A12,4),MID(A12,4,2),MID(A12,7,2))

If the dates are separated by varying numbers of rows, then you will probably need a VBA solution, but I don't have enough information about your layout to be able to advise.

rrdodge say:

In addition to the other Ron's response to your question, there's a few things to keep in mind, or you may also want to look at.

For starters, I like to keep the date values as they are for the cell values, so as formulas don't have to be so complex, but rather use the format tool (Ctrl-1, and Number tab).  Not only that, but that also makes it much easier for other formulas and VBA codes that are looking at either the visible value (The Value Property of the Cell Object within VBA, such as showing the actual date, not the Excel date value), or the raw value (The Value2 Property of the Cell Object within VBA, which Excel has its own date numbering format depending on if you are using the normal version or if you using the Mac version of Excel.  For the VBA codes I write, I use the Value2 property quite a bit, so as to minimize the confusion that can otherwise take place by using the Value property).  For your specific situation, you may want to look at using the Custom Number format within the Format dialog box to use on the cells, unless you really have to have your cell values formatted like that for other formulas or VBA code that are dependent on those specific formats you have laid out.  I just like to keep my stuff as clean as possible.  The one issue with the custom number format route, it probably won't allow for the exact way you are wanting the 2 letter abbreviations of the week days.

Also, Excel can convert dates in text format to date values, but the text format must match to what your regional settings are set to in regards to your short date format and long date format.  Be careful with the short date format, if the year only contains 2 digits.  Does anyone remember the Y2K issues?  I sure do as that also aided in the excessive high demand in the economy in 1999 due to so many people acting out of fear, which then led to the downfall in 2000 - 2002.

As to reading the cell values, you can use the INDIRECT function, but then you may also need to use the VALUE function outside of the INDIRECT function.  However, the VALUE function will attempt to convert the text of the date to the value of the date as how it is in Excel, which again is dependent on how your regional settings are setup for the short date and long date formats.  The INDIRECT function pulls the value of the cell (text or numeric depending on the format and result of the cell).  Most often times, I'm using this INDIRECT function as a wrapper to the ADDRESS function.  Given this complicated issue, that's just another reason why you would rather have the cell contain the value of the actual date (unless other factors overrides this argument), so as you don't have this added layer of complication.

As to using the formula from one column to the next, if you need to have the formula to refer only to one column to the left regardless which cell it's in, then you can convert that Absolute cell reference to a Mixed cell reference.

Example:

The $A$2 would become A$2

This way, when it's in column B, it refers to Cell A2, but then when it's in column C, it refers to Cell B2.

Ron Dodge
