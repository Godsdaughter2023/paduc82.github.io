pesach shelnitz say:

Hi,

There are two more little differences (at least in 2007).

  1. Quick Parts is the default gallery in the dialog box that opens when you press Alt+F3 after selecting the building block content. If you want to store a building block in the AutoText gallery, you have to do something to change the default.
  2. The contents of the building blocks in the Quick Parts gallery are displayed in a drop-down list when you click Quick Parts on the Ribbon to help you select the building block that you want to insert. Those in the AutoText gallery are not displayed.

Hope this helps,
Pesach

stefan blom say:

The only difference is that they are different "galleries." Both Quick Parts and AutoText are Building Blocks (you can see the different kind of "galleries" if you look in the Create New Building Block dialog box).

The Building Block feature is an extension of AutoText (which was the only type in Word 97-2003).

How to delete cells without messing up border formatting in other - Microsoft Community

hojo2010 say:

How to delete cells without messing up border formatting in other cells?

The border formatting is messed up when deleting a line of cells above the formatted cells. Somebody authored some code to correct this problem. Do you know how to obtain it?

jim rech say:

I don't know about a macro rectifying this situation but understanding why it occurs may help.

Whether a border disappears when an adjacent row is deleted depends on which row the border belongs to.

Try this to see what I mean:

Select A9:D9 and add a bottom border. Then delete row 9.  The border disappears.

Now select A10:D10 and add a top border. Then delete row 9.  The border remains, moved up one row.

In both cases the border appeared on the line between rows 9 and 10 but in the second case the border did not disappear because the row it belonged to (10) was not the deleted row.

There is no way to see which row a border belongs to after it is created.  You can't see it on screen and the Format Cells dialog doesn't help either since the border appears in it when a cell on row 9 or row 10 (in my example) is active.  So when you create borders consider which row (or column) you want them linked to and make sure that row (or column) is selected, not the one next to it.

If you have a problem already I'd suggest removing and then re-creating the borders with the row not to be deleted selected.
