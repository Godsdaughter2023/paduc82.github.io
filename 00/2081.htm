hansv mvp say:

OK, but only if you use complete, absolute cell references such as R1C2. If you use R1 or C2 by itself, it will be interpreted as A1 notation, and references such as R1C or R[1]C2, which are valid in Excel, are not valid in Word.

 

Unlike in Excel, there is no global setting to set A1 or R1C1 notation.

a. user say:

You can use R1C1 notation, but 
 a. not in all the same places you can use A1 notation. For example, {=R2C4*R2C6} should work, but { =SUM(A1:A2) } works but {=SUM(R1C1:R2C1)} causes a syntax error
 b. if you are using a non-English language version of Word, you may find that the "R" and "C" have been localised. I can't easily check this on the Windows version of Word, but for example on the Dutch language version of Mac Word 2011, {=R2C4*R2C6} would cause the error you describe, but {=R2K4*R2K6} should be OK. You may also find that you need the localised words for things such as "ABOVE", "BELOW" and so on. Some localisations may accept either native language or English language - I cannot remember.

There are many other quirks in the {=} field in Word - a few are described in my post in the following conversation:

                    Debug.Print "'" & ffld.Name & "': " & ffldID & " " & ffld.Result

                    Sheet1.Range(ColLetter(PasteRow) & PasteRow_Comments).Value = ffld.Result

Private Function GetPathToUse() As Variant
Dim fDialog As FileDialog
  Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)
  With fDialog
    .Title = "Select Folder containing the completed form documents to and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
      GetPathToUse = ""
      Set fDialog = Nothing
      Exit Function
    End If
    GetPathToUse = fDialog.SelectedItems.Item(1)
    If Right(GetPathToUse, 1) <> "\" Then GetPathToUse = GetPathToUse + "\"
  End With
lbl_Exit:
  Exit Function
End Function

Cross-referencing a SEQ field

I frequently use a SEQ field to identify exhibits that will be attached to or otherwise accompany the document I am writing.  The advantage, of course, of using the field instead of typing "A," "B," etc., is to automatically manage insertions or other changes in exhibits.  Keeping that flexibility, i.e., the ability to use Word's automation to keep the correct sequence, I want to be able to refer by letter to an exhibit I entered earlier as a SEQ field.  In other words, in one paragraph, I'll refer to Exhibit A by having entered a SEQ field for the "A," and then in another paragraph I want to be able to refer back to Exhibit A, where the "A" will change as necessary if another SEQ field has been introduced.

Someone asked a similar question a couple of months ago, and the answer was to use Word's built-in headings.  That won't work at all.  For one thing, they're already being used--as headings.

Is it possible to cross-reference a SEQ field?  If not, is there another way to go?

 

mr andrew say: The simplest answer without changing the way you work is to bookmark the SEQ numbers and insert a reference to the bookmark.  This is done with a REF field, or you can use the dialog found under insert->reference->cross-reference or insert->cross-reference (depending on which version of Word you are using).

So, for example you refer to Exhibit A which is say a photograph.  Bookmark the SEQ field (insert->bookmark) and choose a sensible name for the bookmark like "Photo".  Later on you can refer back to it by inserting a cross-reference to the bookmark.  Using a descriptive name for the exhibit is a good idea as it makes inserting the references very easy (which names like "Exhibit23" don't), and it makes verifying your referencing easy too by looking at the field codes (Alt-F9 to switch on/off).

There are other ways (e.g. LISTNUM fields) but I think bookmarks are the best way.

Andrew - www.stoptyping.co.uk

graham mayor say:
One lash-up that will work is to create a new label for a caption. Make the label a single space. This will of course add a space before your number, but if that is acceptable the rest is simple.
 
Either change the Caption style to match the underlying paragraph, or re-apply the underlying paragraph style after applying the caption. You can then use a cross reference to a numbered item - picking the blank space from the list of numbered item types.
 
The following macro will insert such a caption add a full stop (period) and a space after it and format it with whatever the underlying style is.
 
As these numbers are for exhibits, you could, if you prefer add the Word 'Exhibit: ' as a label instead of the space. In which case you will need to add the label in place of the space between the quotes in the macro.
 
Sub MyCaption()
Dim sStyle As String
With Selection
    sStyle = .Style
    .InsertCaption Label:=" ", _
        TitleAutoText:="InsertCaption1", _
        Title:="", _
        Position:=wdCaptionPositionBelow, _
        ExcludeLabel:=0
    .TypeText ". "
    .Style = ActiveDocument.Styles(sStyle)
End With
End Sub

In VBA, you can add a new caption label using:

CaptionLabels.Add Name:=[your label name]

So, to add Exhibit, you would use:

CaptionLabels.Add Name:="Exhibit"

The other option, if you want to avoid VBA, is to insert a caption (References -> Insert Caption). In the Caption box, click "New Label..." and add your label type (eg "Exhibit").

Either way, the label type should now show up as its own category in the cross-reference box.
