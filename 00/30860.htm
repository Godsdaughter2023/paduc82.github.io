rohn007 say:
Since you are a windows newbie, any question is valid.

There are 2 (actually 3, but I'm ignoring "Store" apps) types of Windows application when it comes to installing them.

The first is a "portable" application. This is a design left over from the 1980's. Everything you need to run the program is in a zip, or RAR, file. You unpack them and the application is ready to run. Typically you would unzip it some place other than the desktop.

The other, "new" style application use a "setup" program to do the install. The setup program unzips files, creates required folders, creates required registry entries etc.

From the sound of it you have one of the first type. After unpacking the rar, in theory you should be able to double click on the program EXE file to run the program.

So the list of items that you pasted in your previous post outlines the problem registry keys.  It appears that the proper permissions for these keys have been stripped.  Here are the steps you can try to resolve the problem.  Please be sure to mark the thread as "Answer" if it resolves the problem.

 

1. Download the Windows PowerShell:

First you'll need to download a tool called PowerShell V2 (CTP3) from the Microsoft Download Center: http://www.microsoft.com/downloads/details.aspx?FamilyID=c913aeab-d7b4-4bb1-a958-ee6d7fe307bc&displaylang=en 

 

2. Install the Windows PowerShell:

Next, double click on the file that you downloaded to install the tool.  Please note that you may get a message stating:

"Windows PowerShell(TM) V2 (CTP3) remoting features may not work correctly in this version of the Operating System".  This is expected depending on the version of Windows Vista (e.g. Home Premium).  On that same screen, click Install to continue.

 

3. Launch the PowerShell:

Once you have downloaded the PowerShell, click on the Start menu, and type "PowerShell" in the search field.  Click on Windows PowerShell from the search result to loach the tool.  If the tool is launched correctly, you will have a window with a blue background, with a command prompt "PS C:\>"

 

4.  At the command prompt (PS C:\>), type each of  following commands (a to f) followed by the Enter key.
Tip: You should be able to paste each of the commands one by one.  Simply right-click in the Window, and select Paste.

 

a. $acl= get-acl -path "hklm:\software\Microsoft\Windows\CurrentVersion\Component Based Servicing"

b. $inherit = [system.security.accesscontrol.InheritanceFlags] "ContainerInherit","ObjectInherit"

c. $propagation = [system.security.accesscontrol.PropagationFlags]"InheritOnly"

d.  $rule=new-object system.security.accesscontrol.registryaccessrule "NT SERVICE\TRUSTEDINSTALLER","FullControl",$inherit,$propagation,"Allow"

e. $acl.addaccessrule($rule)

f. $acl|set-acl

(If you get an error message, take a brief description of the error, and let me know.)  

 6. Close the PowerShell Window and test Windows Update

Récapitulons. Tu as un profil principal et quand tu t'es inscrit au Xbox Live, tu t'es créer un profil secondaire ... Ce qui veut dire que ton profil principale est en hors ligne. Donc pour que ton profil principal puisse bénéficier du Xbox Live, il faut l'inscrire.

 

Tu vas sur ton profil hors ligne et tu choisis "rejoindre le Xbox Live". Tu n'as plus qu'à suivre les instructions qui te seront proposées.

 

Par contre, si tu as acheter des dlc, thèmes, images de joueurs ou autres avec ton profil secondaire, tu devras les repayer et les retélécharger avec ton profil principal. De même que tes sauvegardes du profil secondaire ne fonctionneront pas avec ton profil principal.

Car comme l'a écrit Boombaa, aucun transfert n'est possible entre profil.
