ken sheridan say: The important thing to understand is that updating a control and changing a value in the control are not the same thing.  Changing a value automatically updates the control, but updating a control does not necessarily change its value.

If an existing record is edited, and the user updates the control in question without changing its value, then the code will execute and will detect the same record as a specious duplicate of itself.  You would not want the code to fully execute in this situation however, as there is no duplication.  By adding an autonumber column to the table, the code can detect a specious duplicate of the current record by virtue of the fact that the value in the autonumber column, which will not change, is the same as the current value.  By making the code conditional on the autonumber value not being the same as that of the current record, only true duplicates are identified as such.

You may well say that that situation will not arise in practice.  However, it is possible for it to arise, and good programming caters for what is possible, not just what is expected.  As Murphy's Law tells us, 'If something can go wrong, sooner or later it will go wrong'.
