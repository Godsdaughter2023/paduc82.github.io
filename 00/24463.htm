jllatham say:

With 13 in A2 and 14 in A3  and the squares of those in B2 and B3 respectively, then this formula would do the check:

=IF(AND(RIGHT(B2,1)=MID(B3,LEN(B3)-1,1),MID(B2,LEN(B2)-1,1)=RIGHT(B3,1)),"AHA!","")

When you have last two digits transposed, it returns AHA! - stays blank otherwise.

If you put the cubes of the 2 numbers in A into column C, then just change B to C in the formula to do the same checks for cubes.  But this only tests the results of the two values entered into A2 and A3.  But I don't see another such match for 2 sequential numbers, either squared or cubed from 1 through 500.

If you need to check each result in a long list of entries against the others, then I suggest a macro solution.

If you are missing just a short portion at the bottom of the Save As window, open a blank document, click on the document's title area just above the horizonal line and move the document's window all the way to the top of the screen. Hopefully, this will be enough to bring back into view the Save As dialog's resize handle (its bottom-right corner), so next do a Save As and if you see the resize corner, shrink the dialog. If moving the document's window does not provide enough distance, try the following.

Now I don't have Word 2008, but this works in Word 2011 to externally resize the Save As dialog, so try it:
  • Quit Word
  • Open Applications > Utilities > Terminal
  • At the $ prompt, paste the following command (it's a single line), then press Return
          defaults write com.microsoft.word NSNavPanelExpandedSizeForSaveMode:PutFile:0 -string "{621, 458}"
  • If Terminal reports an error, then the parameter does not exist in Word 2008
  • Otherwise, you'll get a $ prompt; quit Terminal and test Word
