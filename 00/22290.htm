marshall barton say: Make sure that the combo boxes are either unbound (if you do not need to save their selection) or are bound to different fields in the form's record source table/query.

ken sheridan say: You can't 'unlink' them.  Two bound control's with the same ControlSource property map to the same field in the underlying table, so any changes you make in one are reflected in the other; it's just two instances of the same data you are seeing.

If you want to record multiple Fee Description values for one record, then you will need to create a related table with a foreign key column referencing the primary key of the Fee Schedule table, and a non-key Fee Description column.  You can then include a continuous forms view linked subform bound to this table in your current form.  To enter more than one description you'd then insert a separate record in the subform for each.

This process of splitting a table into two related tables like this is known as 'decomposition' and is one of the fundamental mechanisms of relational database design.  It ensures that there is no 'redundancy' in the tables, which would be the case if you had to repeat the values in all the other columns every time you entered another Fee Description value. The process of eliminating redundancy is known as 'normalization' and there are formal rules governing this.  You may well find that you need to hive off other columns into the new table as well of course to achieve a set of normalized tables, but without knowing more of your business model I can't comment on that.
Ken Sheridan, Stafford, England

diego rios_06 say: HI,

I'm Diego Rios, an Independent Advisor willing to help other Microsoft customers.

I suggest you to follow the troubleshooting methods listed below:

1, Try to run the Apps troubleshooter to fix issues that may prevent app from working properly in the link below:

https://support.microsoft.com/en-us/help/402749...

2. Clear browsing history.

3. Run the SFC scan
Run the Microsoft Windows Resource Checker tool, also know as "System File Checker" or "sfc". This tool scans and verifies the integrity of all protected system files and replaces incorrect versions with correct versions. Here's how:
1. Press Windows key + X, click Command Prompt (Admin).
2. In the Command Prompt, type the following command, sfc /scannow and then press Enter.
3. Wait until the process completes. This usually doesn’t take longer than 20 minutes with a regular hard disk
and takes much shorter when you have a fast SSD drive.
4. Check to see if the issue is resolved. If not, restart the system.

4. Repair and reset the app:
Right-click on the Start menu button, and select Settings.
Select App, click on IE from the list, and select Advanced options.
Select Repair.
If the issue persists, repeat the steps and select Reset.

In case the procedures above didn't work you can try the procedures in the links below:

https://answers.microsoft.com/en-us/ie/forum/ie...

https://support.microsoft.com/en-us/help/318378...

I hope it works,
