bob jones aka: cybertaz say:

Well, for better or worse, that isn't how it's designed to work :-)

The Based On option primarily is for more expediently creating new Styles which include [most of] the same attributes as the parent Style but differ in one or more ways. If any of the attributes in the new Style are changed to something different, it breaks the link between that attribute & what was specified in the parent. If that attribute later is modified in the parent Style, the Styles that are Based On that Style will not update in regard to those attributes.

That's the case with the Heading Styles... They're Based On Normal, but the font specified in Normal is Calibri while the font specified in the Heading Styles is Calibri Light. Accordingly, changing the font of Normal to something other than Calibri has no effect on the Heading styles because they don't use the same font. If the Heading Styles did use Calibri the change to Normal would be passed on to them.

Themes come into play here as well with the default Body font being Calibri & the default Headings font being Calibri Light in the default Office Theme. Unfortunately, the only set of Theme Fonts that uses TNR for Headings uses Arial for Body... More unfortunately, the Mac Office UI has no provision for creating customized Theme Fonts.

Alternative approachess include:

  • If you have access to a Windows version of Word there may be a built-in set of Theme Fonts that provides what you want, depending on version. If not...
  • Custom Theme Fonts can be created in Word for Windows. What's specified in the document will travel with it when it's brought back into Mac Word.
  • Custom Theme Fonts can be created on a Mac if you have the necessary knowledge of XML.
  • Modify the built-in Heading Styles to suit your preference.
  • Create new Heading Styles of your own.

arvind_m say:

I'm afraid this is a limitation that you cannot work around.

The explanation is that the two OS instances are, as far as the Bluetooth mouse is concerned, equivalent to two different PCs.  The mouse can only 'remember', and therefore work with, the most recent OS instance that it was bonded or paired to.  When you switch to the other OS instance, the mouse will not connect to it.
