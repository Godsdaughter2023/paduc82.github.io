steven. s say:

Hello Kahunals,

 

Welcome to the Microsoft Answers Community!

 

This is usually caused by a third party program that causes a conflict with the auto-refresh that should occur after a file is deleted/moved/renamed. You can use msconfig to troubleshoot/identify the program causing the conflict:

·         Click Start

·         Click Run…

·         In the Open: field, type msconfig

·         Click OK

·         Click the Services tab

·         Check the Hide all Microsoft services box

·         Click Disable all

·         Click OK

·         Click Restart Now

 

Once your computer has restarted, test to see if Windows Explorer now correctly auto-refreshes. If everything now works correctly, you just have to find which service was causing the conflict.

·         Click Start

·         Click Run…

·         In the Open: field, type msconfig

·         Click OK

·         Click the Services tab

·         Check the Hide all Microsoft services box

·         Check (Enable) the top half of the service boxes

·         Click OK

·         Click Restart Now

 

Again, once your computer has restarted test again to make sure everything is working correctly. If everything is still working as it should, the services you enabled are good to stay. Repeat the enabling process for half of the remaining disabled services and continue this process until the problem returns. Then the last batch of enabled services will need to be tested one by one to identify which service needs to be disabled/uninstalled/updated to avoid further problems.

 

Hope that helps, look forward to hearing back from you.
