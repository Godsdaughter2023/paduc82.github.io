Transferring large file to external disk drive - Microsoft Community

patricknbl say:

Transferring large file to external disk drive

I am trying to transfer a large file ( 4.5Gb movie .mpeg) from my computer to an external disk drive. The external disk drive has 320Gb space with 82Gb utilised. I get the following message when I try to move the file " The file ___ is too large for the destination file system." Please tell me how to change the settings to allow the transfer.

a. user say:

Be careful. While Michael Murphy is correct in identifying the reason for your problem (in fact, if it is in FAT16, it would have only a 2GB limit (you didn't mention the version of FAT you are using)), he is not correct in the solution (or how to implement the solution - though it would work but maybe with consequences you do not want).  To convert a partition from FAT32 to NTFS you would NOT format the drive as indicated above in the prior post (or you would lose everything on the external hard drive and have to restore your data from backup, re-install all your programs loaded on that drive, reset all your preferences on programs loaded on that drive,...).  Instead, you would use the convert command. http://techblissonline.com/fat32-to-ntfs-conversion-in-windows-xp-or-vista/.  That would preserve all your data, programs, and settings while still doing the job.

It is probably better formatted in NTFS anyway because that gives you access to features and options which are otherwise not available (like file/folder security) and is overall a better system type.  There ARE reasons to keep FAT32 as the system type (though they may be less important than being able to copy large file sizes and some of the other features of NTFS) but you mentioned none of them in your post and they are uncommon (mostly compatibility with older systems or non-computer systems).  Here's an article on making that choice: http://www.theeldergeek.com/ntfs_or_fat32_file_system.htm.  You can read that to see if any apply to your situation.  If you decide to keep FAT32 (for some reason assuming that's the version you have), you will have to live with the transfer size limitations of that system type - there's no other way around it.

I hope this helps.

Good luck!
