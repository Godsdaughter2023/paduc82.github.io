ken sheridan say: It looks like somebody has dug you into deep hole here.  As regards the query, it is drawing on four tables, so is almost certainly not updatable, hence your problem.  For entering data into multiple tables via a form the form should only be for entering table via a subform linked to the parent form on OrderID or similar.

Data can be entered into a table referenced by the parent form's table, e.g. in the case of Orders, into a Customers table, but this would usually be done by means of the NotInList event procedure of a combo box bound to a CustomerID column or similar, which either transparently inserts a value into a new row in a single column, or opens a form in dialogue mode, passing the new customer name typed into the combo box to the form ready for other data such as the customer's address to be entered.  You'll find an example as InvoicePDF.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file typing a new customer name into the combo box in the invoice form opens a customers form in dialogue mode.

Tables which are referenced by the parent form's main table can be included in the form's RecordSource query, and my invoice form does this, but the customer address data from these tables is shown in read-only controls.  These automatically show the customer address data when a customer is selected by name in the combo box.

A more fundamental problem, however, is that your Business_Service_Tracker table appears to have a huge number of columns, many of which are doing what is known as 'encoding data as column headings'.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1).  This requires that all data be stored as values at column positions in rows in tables, and in no other way.  The table is clearly in need of decomposition into a set of correctly normalized related tables in which the values currently encoded as multiple column headings are represented by values in multiple rows in the related tables.

It would be possible to apply a band aid solution to your current problem by breaking down the form into a parent form and subforms, but the underlying structural problems would still apply.  Correcting these would require a substantial redesign both of the logical model and of the interface, so could be a significant task.  I strongly believe that the first thing to do when in a hole is to stop digging, but the feasibility of that is your judgement call.

jay freedman say:
Select a footnote number and click Format > Styles and Formatting to open the style list. At the top, the first box should say "Footnote Reference", which is the name of the style that Word applies to footnote numbers. When you hover the pointer over that box, a tooltip should show "Footnote Reference: Default Paragraph Font + Superscript".

If the Superscript part is missing from the description, right-click the box and choose Modify. In the dialog, check the box for "Add to template", then click the Format button and choose Font. When you exit Word, if you're asked whether to save changes to Normal.dot, say yes.

From your description, I understand that you have 150 terminated users’ data. All of these date can only be exported to PST files. And you would like these PST files to be imported to Office 365 without normal licenses assigned. Please correct me if I misunderstood.

I’d like to clarify that archive mailboxes belong to normal mailboxes. So if the mailbox doesn’t exist, its archive mailbox doesn’t exist. So in your scenario, it’s not feasible to import these archive mailboxes to Office 365 just with Exchange Online Archive licenses.

However, these is a workaround may fulfill your wishes. You can create shared mailboxes for each terminated users respectively and then grant one user the Full Access permission of the shared mailbox so that when you set up this user in Outlook client (like Outlook 2013), this shared mailbox show up in the Outlook. Note: for shared mailbox, 10 GB is free. If lager than 10 GB, you need to assign a license for the shared mailbox.
