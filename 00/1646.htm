Try this instead:

 

Sub TestIt2()
    Dim lnLastRow As Long
    Dim rng As Range, c As Range
   
     'Clear highlighting from entire sheet:
    With Cells.Interior
        .Pattern = xlNone
        .TintAndShade = 0
        .PatternTintAndShade = 0
    End With
   
     'Get last used row in col E:
    lnLastRow = Cells(Rows.Count, "E").End(xlUp).Row
   
    Set rng = Range("E2:E" & lnLastRow)
    For Each c In rng
        If Application.WorksheetFunction.CountIf(rng, c) > 1 And c.Value <> "" Then
            Cells(c.Row, "A").Value = "abc; match"
            Cells(c.Row, "B").Value = "111aaa000"
            Cells(c.Row, "C").Value = "222bbb333"
            c.EntireRow.Interior.Color = RGB(0, 200, 0)
        End If
    Next c
   
    On Error Resume Next 'SpecialCells will error if no cells are found!
    Columns("S:S").SpecialCells(xlCellTypeConstants, 16).Value = "None"
    Columns("S:S").SpecialCells(xlCellTypeFormulas, 16).Value = "None"
    On Error GoTo 0
End Sub

julie sheets say:

A resource may be overallocated but on less than a day by day or week by week setting.  Check your leveling options. 

If, for example, your leveling options of "look for Overallocations on a" is set to Day by Day but your resource is assigned to four one-hour tasks all starting at 08:00 at 100%, the resource will not be overallocated.  For that one hour from 08:00 to 09:00, the resource's peak units is 400% but if the 4 hours of work were spread out over a full day the resource would not have more work assigned than s/he can do in a single day.

If you change the "Look for Overallocations on a" to Hour by Hour - the resource is now overallocated.

steve rindsberg say:

Hi Russell

The first thing to try is to press Alt + F11 to go to the Visual Basic editor.

Look for a folder in your presentation's VBA project called "Microsoft PowerPoint Objects"

Under that, each slide with an ActiveX control should be listed.

Or try this wee bit of VBA:

Sub LocateActiveX()

    Dim oSld As Slide
    Dim oShp As Shape
    
    For Each oSld In ActivePresentation.Slides
        For Each oShp In oSld.Shapes
            If oShp.Type = msoOLEControlObject Then
                MsgBox "ActiveX object found on slide " & oSld.SlideIndex
                
            End If
        Next
    Next
End Sub

dale howard [mvp] say:

AA --

You can "lock" completed tasks in Project Online to prevent a resource from entering additional time against the tasks.  Is that what you mean?  If so, here is how you lock the tasks:

  1. Log into Project Web App with project manager permissions.
  2. Navigate to the Project Center page.
  3. Click the name of the project in which you want to lock tasks.
  4. At the top of the Quick Launch menu, click the Schedule link.
  5. Click the Task tab to expand the Task ribbon.
  6. In the Project section of the Task ribbon, click the Edit pick list button and select the In Browser item on the menu.
  7. In the Data section of the Task ribbon, click the View pick list and select the Close Tasks to Update view.
  8. Specify a Yes value in the Locked column for each task you wish to close to updates by your team members.
  9. In the Task ribbon, click the Save button and then click the Publish button.
  10. Close and check in the project.

If you are using Microsoft Project 2016 or Project Pro 2016 for Office 365, Microsoft has just added a new feature to the desktop tool:  the Locked column.  If you are using the latest version of the desktop client, you can open and check out the project, and them temporarily insert the Locked column in any task view.  Set the Locked column value to Yes for any task you want to close to further updates, then save and publish the project.  Hope this helps.
