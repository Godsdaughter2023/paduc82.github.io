hansv mvp say:

Let's say your data are in A1:A8.

 

In B1 as an array formula, confirmed with Ctrl+Shift+Enter:

 

=SUM(IF($A1<$A$1:$A$8,1/COUNTIF($A$1:$A$8,$A$1:$A$8)))+1

 

In C1, as an 'ordinary' formula:

 

=(COUNTIF($A$1:$A$8,"<"&$A1)+COUNTIF($A$1:$A$8,$A1)/2)/COUNT($A$1:$A$8)

macropod say:

Although you can retrieve a document's headers & footers easily enough, adding those to an email body is fraught with difficulty, as an email message doesn't support anything like a Word page layout. Accordingly, you'd have to either have the header appear just once (presumably at the start of the inserted document) or manually insert it into the email text wherever you want it to appear - which could also result in it appearing in odd places when printed by the recipient. Then there's the matter of a Word document having as many as three headers & footers per Section (of which a document can have many) that you might need to copy.

 

That said, the code to copy both the body content and the document's 1st Section primary header would look something like:

 

'Copy the open document
ActiveDocument.Range.Copy
'Insert into email body
...

'Place the current document under the intro and signature
wdEditor.Characters(i).PasteAndFormat (wdFormatOriginalFormatting)
'Copy the document's 1st Section primary header
ActiveDocument.Sections.First.Headers(wdHeaderFooterPrimary).Range.Copy
'Insert into email body

 

The first part of the above code would replace your:

'Copy the open document
Selection.WholeStory
Selection.Copy
Selection.End = True

 

I'll leave it to you to sort out where you'll insert the header. If you're inserting it only once, you might want to copy & paste it first, then do the body content.

suzanne s. barnhill say:

There are two cases I know of in which this option is checked (aside, of course, from cases where the user turns it on intentionally):

  1. In very old documents. When "Check spelling as you type" was introduced, documents created in previous versions would open with the option enabled by default.
  2. When Word tells you that you have "too many errors" in the document and will no longer display them, it turns this on (in the same way that previous versions might enable "picture placeholders" when the document became too complex to display). Word's ability to check spelling and grammar as you type, and especially to use the contextual spelling checker, depends on available memory, so if you are low on RAM or HD space, these features may run slowly or be disabled.

jay freedman say:

You can add this line after the Options line:

ActiveDocument.ShowSpellingErrors = True

to uncheck the "Hide spelling errors" option. If you go back and check it manually, it will stay that way until you manually uncheck it or run the macro again.

How does it get checked? One way is if the template on which you based the document has it checked. Other than that, I don't think there's any "normal" way. Possibly an add-in causes it, although I wouldn't expect that to seem random.

stefan blom say:

You can add a line of code which ensures that spelling errors are not suppressed:

Sub Macro1()
Options.CheckSpellingAsYouType = True
ActiveDocument.ShowSpellingErrors = True
Application.ScreenRefresh
End Sub
