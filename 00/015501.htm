jeeped say:
... if i put anything over 1 in the order column it will show the item name and amount to order on another sheet. there ... 

You can create a unique list with one or more conditions (e.g. greater than 1) to list from one worksheet to another with an array formula.

                       

The formula in Sheet2!B2 is,
=IFERROR(INDEX(Sheet1!$B$2:$B$999,MATCH(0,IF(Sheet1!$A$2:$A$8>1,COUNTIF($B$1:$B1,Sheet1!$B$2:$B$999),1),0)),"")
... which is an array formula and must be finalized with Ctrl+Shift+Enter↵ not just Enter↵.
The formula in Sheet2!A1 is,
=IFERROR(INDEX(Sheet1!$A$2:$A$999,MATCH($B2,Sheet1!$B$2:$B$999,0)),"")
... and the formula in Sheet2!C2 is,
=IFERROR(INDEX(Sheet1!$C$2:$C$999,MATCH($B2,Sheet1!$B$2:$B$999,0)),"")
... and both of those are regular formulas.
When you have those three formulas in place, you can fill down as necessary to cover all items to be ordered.

I'm pretty sure you should be able to adapt that for your restaurant supply order list. Post back if you have any questions about translating it for your own purposes.

Here's a quote echoed by everyone I know...

 

***QUOTE

 

...autonumbers are not truely incremental. If you need something at is incremental with no possibility of a break in the numbering, do NOT use autonumbers. For example if you start a new record that has an autonumber field, then change your mind, that autonumber is 'burned' and you will have a gap. Sometimes Access will pull out an out of sequence autonumber, including even negative numbers, for no apparent reason even when set to incremental.

 

Your best bet is to do a DMax of the primary key field in the table and add 1 to it just before saving the record. For this you need to use a form and hope that more than one person is not inputting a new record at the same time.

ken sheridan say:

If you have a real need for a consecutive sequence of numbers, i.e. the numbers have some ordinal or cardinal significance rather than just being distinct identifiers, then don't use an autonumber.  The number should be computed when a row is inserted via a form (and data should always be entered via forms, never in raw datasheet view of a table).  You'll find examples of how to do this in the file CustomNumber.zip in my public databases folder at:

 

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

 

As regards the insertion of rows into referencing table the actual values are immaterial provided they match, and with a proper set-up you need never know what the numbers are.  Say you have a table Employees with an autonumber primary key column EmployeeID and non-key columns FirstName and LastName.  And say you also have a table EmployeeProjects which includes a foreign key column EmployeeID, not an autonumber in this case but a straightforward long integer number data type.  To select an employee when inserting a row into EmployeeProjects you'd have a combo box on the form set up like this:

 

ControlSource:    EmployeeID

 

RowSource:     SELECT EmployeeID, FirstName & " " & LastName FROM Employees ORDER BY LastName, FirstName;

 

BoundColumn:   1

ColumnCount:   2

ColumnWidths:  0cm

 

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the dimension is zero to hide the first column.

 

In the combo box's drop down list  you'll see the employee's names listed in the format Ken Sheridan and you can select the employee by name from the list.  By virtue of the combo box's BoundColumn property being 1 the first column of its RowSource query, EmployeeID will be the value of the control, and therefore of the underlying column to which it is bound.  But because the control's ColumnWidths property is zero the first column will be hidden and you'll see the second column, which is the concatenation of the first and last name columns from the table.  So at no stage during the data entry process will you see the EmployeeID numbers, but behind the scenes the correct number will be assigned to the foreign key EmployeeID column in EmployeeProjects.

The way this is supposed to work is to follow these steps:

  1. In PowerPoint, Click on the FILE tab, then on Options.
  2. Click on Trust Center, then on the Trust Center Settings button.
  3. Click on Privacy Options.

Word Count - Include Textboxes, footnotes, endnotes - Microsoft Community

a. user say:

Word Count - Include Textboxes, footnotes, endnotes

We need to enable the Word Count Dialog Box to always "Include textboxes, footnotes, and endnotes" default set to "Enable" allowing a user to turn off if they wish.  Is there an easy way to keep this checkbox on?

Thak you. 

[Moved from: Office / Word / Windows other]

jay freedman say:

Install the following macro:

Sub ToolsWordCount()
    With Dialogs(wdDialogToolsWordCount)
        .CountFootnotes = msoCTrue
        .Show
    End With
End Sub

This macro will automatically run when you click the word count in the status bar, or if you add a Word Count button to the Quick Access Toolbar and use that; or if you click Tools > Word Count in Word 2003. The macro has been tested in Word 2000, 2003, 2007, and 2016, so I expect it will also work in Word 2010 and 2013.

jay freedman say:

As far as I know, there is no such option anywhere else.

What sort of rollout are you planning? Is it a full disk image, or just an installation of Office, or an update to existing Office installations?

The most likely way of using the macro is to put it into a template that you store in the user's profile, in the folder %appdata%\Microsoft\Word\Startup. That will make the macro available in every document the user creates or opens. The different kinds of rollouts generally require different methods of distribution of the template.

One method that should work for everyone on the company network is to include a batch file or PowerShell script in the network logon script. This would look for the template in the user's profile. If it's not there, or if the file date is earlier than the date of the template on the server, then the server's template gets copied to the user's profile.
