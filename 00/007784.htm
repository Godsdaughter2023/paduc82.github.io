    Okay, I've created a document based on that template. It's interesting looking at it in Draft view, as it contains only a page break and a following paragraph--not even a paragraph before the page break. Everything on the first page is a graphic anchored to the page break, and everything on the second page is anchored to the single paragraph.

     

    The bad news is that the colored boxes are formed by a single background art piece anchored to the page break. I hoped they might be grouped shapes, but instead everything except the photo box is a single picture. The text boxes are added in front of the background art.

     

    The only way you could change the colors of the panels would be to create a new background. For this purpose, I would suggest you create AutoShapes anchored to the header paragraph. If you want to tackle this, here's how I suggest that you proceed:

     

    1. Switch to Draft view and display nonprinting characters (Ctrl+* or click the Show/Hide ¶ button in the Paragraph group on the Home tab).
    2. You'll see a Page Break followed by a paragraph mark (¶).
    3. Place the insertion point to the left of the page break and press Enter three times. This will create three paragraphs before the page break.
    4. Insert a page break (Ctrl+Enter) between each pair of paragraph marks. You'll now have ¶, Page Break, ¶, Page Break, ¶, Page Break, ¶.
    5. When you switch back to Print Layout view, you will have two blank pages before the two pages with color boxes.
    6. On either of the blank pages, you can double-click in the header area to open the header for editing. The contextual Header & Footer Tools | Design tab will be displayed.
    7. In the Options group, check the box for "Different First Page." This will give you a separate First Page Header so that you have different color blocks on the first and second pages.
    8. With the insertion point in the First Page Header, draw blocks of your desired size and color.
    9. Repeat for the Header.
    10. When you have the color blocks to your satisfaction, double-click in the body of the document to exit the header.
    11. Select one of the text boxes on the first page of the brochure (now page 3) and Cut it (Ctrl+X)
    12. Move to page 1 and paste (Ctrl+V).
    13. Repeat for each of the text boxes on the first page, then move to the second page

      The reason I've described this rather roundabout procedure is to preserve the existing color blocks for reference while creating the new one. A much more efficient solution that would not require creating blank pages (or moving the text boxes) would be this:

      1. Print a reference copy of the brochure so that you can measure the sizes and positions of the color blocks.
      2. Open the header as described above and enable "Different First Page."
      3. While still in the header, draw the desired color blocks.

      A third approach that is more efficient still but requires a leap of faith is this:

      1. Without deleting the background graphic, open the header as described and enable "Different First Page."
      2. Draw color blocks directly on top of the background, matching the size and position. Because the header is always the farthest-back layer, you will not see the colors you select until after you have deleted the background.
      3. After you've created all the required shapes, click Close Header and Footer, then select and delete the background graphic. Your colored shapes will be revealed.

      Note that no matter how you handle this, the colors of shapes anchored to the header will be dimmed when you're working in the document body, but they will print at full strength.

      joeu2004 say:

      Ranges like 60-65 and 65-70 are abiguous. Where do you want 65 to be counted?!

      It would be great if you say 60-65.

      Suppose your data is in D1:D23. Select F2:F7 and array-enter the following formula (press ctrl+shift+Enter instead of just Enter):

      =FREQUENCY(D1:D23, {65, 70, 75, 80, 85})

      All values less than or equal to 65 are counted F2. That includes any values less than 60. All values greater than 85 are counted in F7. That includes any values greater than 90. I purposely omitted 90 from the array constant {65,...,85} for that reason.

      You can enter the text "60-65" etc (without quotes) into E2:E7, if you wish.

      shinmila h say:

      Hi Richard D Harris,

      As mentioned by Thahaseena, Master Cook application is not yet compatible with Windows 7.

      However there are few workarounds to make it work in Windows 7.

      Workaround 1: To use web import tool, you will need to run IE as administrator.

      Run IE as administrator.

      1.    Go to start and type internet explorer in the search field.

      2.    Right click on Internet explorer from the programs list and select “Run as Administrator”.

      Test and see if you can use the web import tool after this.

      If you still face the same problem, follow the next workaround method.

      Workaround 2:

      After installing MasterCook and you launch the Internet Explorer web browser there should be a MasterCook icon in your toolbar. If there isn't, it could be caused by several things. See below.

      1.     It won't appear until after you restart your computer after installing the program. So, restart your computer and see if that helps.

      2.     If you have customized your Internet Explorer toolbar at some point in the past, you may need to do that again to get the icon to appear in the bar.

      a)        Right click on a blank area of your IE toolbar and choose customize from the menu that pops up.

      b)      From the left side of the screen find the icon for Mastercook Web Import Bar.

      c)       Select/highlight it and press the ADD button.

      d)      Close out of the customize window and look for the button on your toolbar.

      e)        Otherwise, you can toggle the Import Bar off or on by selecting Explorer Bar > Mastercook Bar from the View menu at the top of the IE web browser window.

      Refer:

      Customize the Internet Explorer toolbars

      http://windows.microsoft.com/en-US/windows-vista/Customize-the-Internet-Explorer-toolbars

      3.    Web tool bar doesn’t work properly if your Internet security settings are set too high. You may reduce the security level of IE to make it work.

      To do so:

      a)    Go to Tools in Internet Explorer.

      b)    Select Internet Options and navigate to Security Tab.

      c)    Press the Custom Level button at the bottom of that tab.

      d)    In the drop-down list in the Security Settings, make sure that the security level is set to Medium.

      4.    Configure the following settings in the security tab.

      a)       Follow the steps from 3 to go to security tab.

      b)      Under" Active X Controls and Plugins," these two options must be enabled:
           --Run Active X Controls and Plug-ins
           --Script Active X controls marked safe for scripting.

      c)       Under" Scripting," all three options must be enabled, that is:
           --Active Scripting
           --Allow paste operations via script
           --Scripting of Java applets

      5.    Check the manage add-ons window. From the Tools menu at the top of Internet Explorer select manage add-ons. Double check to see that the MasterCook Web Import Bar item is ENABLED.

      6.    Make sure that you don’t reset your IE unless extremely required. If you reset the browser settings that can disable the web import toolbar.

      Hopefully you should be good to go from here.
