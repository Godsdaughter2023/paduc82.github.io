Populating unique IDs in multiple tables - Microsoft Community

a. user say:

Populating unique IDs in multiple tables

Hello,

I'm relatively new to Access and I'm sure this is a common question.

I am trying to make a database of patients where they will be given unique IDs. I would like these unique IDs to be used on multiple tables. I was wondering how I might get these unique IDs to automatically populate on all needed tables when I enter them into one table (or form, if that is the better route to go). 

I currently have a relationship made between my two mock tables where the unique IDs are linked one-to-one with enforced integrity, and cascade update and delete.

Thank you!

scottgem say:
Access has a datatype called Autonumber. This will generate a unique ID for every record in the patient table.

There is no way to automatically populate your child tables when the parent patient record is created. Nor should you. Creating placeholder records is not best practices.

the way access generally works, is data is entered through forms. Generally the parent records i bound to a main form. The main form, then has subforms embedded in it and linked to the mainform on the parent ID field (I.E. patientID). When you enter a record into the subform, the PatientID Foreign key is automatically populated.

scottgem say:
Also tables in a one to one relation are a rarity. it indicates your database design may not be accurate. If you want to describe the tables more I can advise on that.

scottgem say:
So the Caregiver is a separate person from the patient? Then the table should look like this:

tblCaregiver
CaregiverID (PK Autonumber)
PatientID (FK)
CaregiverFirst
CaregiverLast
CaregiverPhone
CaregiverStreetAddress
CaregiverCity
CaregiverState
CaregiverZip
EffectiveDate
EndDate

This way you can keep a history of caregivers. Also fields should contain each small piece of data. so you want to separate the address components and name components.

Then yes you would have a main form bound the patient table and a subform bound to Caregivers and linked on PatientID.

Two other points. Age is a calculation and should not be stored since it changes constantly. And its not a good idea to use spaces in object names.
