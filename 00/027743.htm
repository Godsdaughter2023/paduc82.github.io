philipradler say:

While we wait for a real expert to reply, here's how I'd do it:

  1. From the Home tab, open the Paragraph dialog box by clicking the drop-down arrow in the bottom right corner of the group.
  2. In the "Indentation" cluster, set a hanging indent, and set a distance (the "By" value) of about 1" (leave your left margin at 0" for now).
  3. Click on the Tabs button (bottom left) and set a left margin at about 0.15" or 0.20"; click OK.

To use:

  1. Type your first number (e.g., "1.")
  2. Press the Tab key.
  3. Type your item name (e.g., "Item 1")
  4. Press the Tab key.
  5. Type your description. It will wrap to that handing indent of 1".

You can accomplish the same thing by setting a left tab at 0.15" on the ruler, then dragging the ruler's "Hanging Indent" up-arrow to 1" (the rectangular "Left Indent" block will come along automatically; don't drag the block itself, or you'll move your "First Line Indent" down-arrow too, with unhappy results).

Hope this works for you.

Best regards,

Phil

 

stefan blom say:
For an auto-numbered list, you can't set indentation in the Paragraph dialog box; it must be set in the numbering options. Do the following: Click the actual list number so that Word selects all numbers belonging to the same list. Right-click and choose Adjust List Indents. In the dialog box, use "Number position" to determine the location of the list number. Use "Text indent" to specify where the text of the first line starts. Select "Follow number with" and choose "Tab character" in the drop down; then specify a value in the "Add tab stop at" box. All measurements are from the left margin of the text container (page, text box, table cell, etc.). For a hanging indent, choose the same value for "Text indent" and "Add tab stop at."

suzanne s. barnhill say:
For multilevel (bulleted, numbered) lists, all indents on the left side (Left Indent, First-Line Indent, Hanging Indent) must be set in the Define New Multilevel List dialog. Indents set in the Paragraph dialog will be superseded by those set in the Define New Multilevel List dialog. In this case, you need to define different settings for "Text indent at" (which will be the hanging indent, where the description starts and wraps to) and "Add tab stop at," which will define where "Item 1," "Item 2," and so on will appear. An "automatic tab stop" will be set at the hanging indent location, but you'll need to add a tab character after "Item x" to go to it.

rohn007 say:
Raju gave you the official party line.


Office 2013 * is the generic the name of the "traditional" local install "Retail" license version. The 2013 Retail license was "improved" to allow install on only one machine at a time. The original 2013 Retail license did not allow transfers, but that was retroactively changed to allow transfers.  The local install license is a pay one time fee.

Office 365 * is the name for the MS Preferred "new and improved" Office bundle. The key difference is that it is delivered in a different installation method. It is no longer installed directly on your computer. Office 365 uses the "Click to Run" virtual computer environment to deliver Office 2013 to you. The CTR is installed on your computer, and Office 2013 runs inside of it.  One of the associated features is that you no longer have to apply Windows Updates to your Office programs. MS handles that for you, "automagically".  As you noted, Office 365 is a subscription service, monthly or yearly. 

The Office 2013 that runs inside of the CTR has identical functions to the local install.  Office 365 can be linked to a local install if you wish.

There is also a loosely related "Office 2013 Web Apps". They are the free for everyone crippleware replacements for Office 2010 Starter and earlier Office Works.  The Web apps run inside of a browser. They are feature limited compared to the full versions, but adequate for basic document editing.  The Web Apps include Word, Excel, PowerPoint, OneNote and Excel Surveys.

To make the pay for-ever subscription more desirable MS has bundled a number of additional features:
  • most 365 licenses allow you to install on up to 5 devices at the same time
  • the user had control to activate and deactivate those installations without contacting MS
  • you get "free" Skype phone time
  • you get 20GB of Skydrive storage
  • "Office on Demand" allows you to temporarily run Office on any computer
  • when Office V-Next is released in 2 or 3 years it will be automagically rolled out to you (when ever MS decides to) at no "extra" charge


Yes, you can run Office 2013 local install along side Office 2007.  You CANNOT run 2 versions of Outlook. You have to decide which one you want to use.  I actually have 2003, 2007, 2010 and 2013 local installs ALL running in the same Windows 8 session.  You have to be careful when you do the installs, and you have to do a simple registry hack. 


Both the local install and 365 do have one outstanding issue when running along side earlier versions. Windows can only allow 1 program to be defined as the default for any given file type, ie DOCX.  That is actually a Windows, not Office limitation.


Yes you can run Office 365 (2013) along side Office 2007. It is actually "better" than running local 2013 because the CTR almost completely isolates 2013 from 2007.  MS actually advertises that you can install 365 trials along side earlier versions until you decide to upgrade. 


The Office 365 install is relatively quick when compared to the local install. MS has a bunch of "dumb install" videos showing 365 being installed in stupid environment, ie Skydiving. After the CTR is set up, only small "stubs" of each program is installed inside of the CTR. Just enough of a shell to show the program starting. At the same time more of the program code is being downloaded from the internet. Each time you use a new discrete function for the first time, it is downloaded from the internet.


CTR is nice when it works. But there is a subset of people for whom it does not work. Sometimes it just won't install, sometime it carps out "at random". Unfortunately, when that happens MS does not provide any support.  Just read through this forum and you'll see lots of people with these problems. I have no idea what proportion of all 365 users are affected. Apparently not enough because no media reporters have picked up on it, YET.

We have learned from this incident, and have made two key changes to harden our systems against future failure – one that involved increasing network bandwidth in the affected part of the system, and one that involved changing the way error handling is done for devices using Exchange ActiveSync. We will continue to monitor the system and make additional changes as needed to keep the service healthy. We are now fully through the backlog and have restored service so all customers should have normal access from all of their devices.

c.r.cavazos say:

I found the culprit. It was a family of damaged fonts.

The suggested actions, after deleting the old suite and installing a new downloaded pack, are:

1.-Open Font Book

2.-Select all your fonts.

3.-Right click to resolve duplicates

4.-Then, run a "Validate font" process.

5.-Remove any font that appears to be damaged.

6.-Restart your Mac in safe mode. To do this hold shift when restarting.

7.-Once in safe mode, open the apps one by one. Word, Excel and Powerpoint.

8.-Create a blank document, type something. Change the font.

9.-Save and quit.

10.-Restart in regular mode.

11.-Open the apps, one by one, try the newly created documents. They shall work fine.

This seemed to work for me.
