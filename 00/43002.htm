Since you are unable to access the administrator account, you can start the computer in WinRe mode and then try to access the administrator account.

 

Please follow the steps mentioned below to boot into WinRe mode:

1.     Boot yourcomputer from a Windows DVD. To do so, insert a Windows Vista, restart your computer and press any key when prompted.

2.     Click "Next," then click "Repair your computer" to bring up the "System Recovery Options" dialog box.

3.      Select the Windowsoperating system you want to repair and click "Next."

4.      Select the option that reads “Command Prompt”

5.      At the command prompt, type “notepad” and hit enter.

6.      Once you have the opened the notepad, in the “Menu Bar”, click on “File” and select “Open”

7.      On the right pane select “Computer”, Select “C:\” drive, “Users” and then select the administrator account

8.      Similarly open the External HDD, which is connected to the computer.

9.      Select the user, whose data needs to be backedup and copy it into the External HDD.

Note: The process should be followed with utmost care.

 

Alternatively, you can reinstall the operating system, without formatting the HDD, the data will get saved in a folder called “Windows.old”, you can then copy and paste the data, wherever you want it to.

Sorry I forgot that you mentioned having all updates etc.

 

I think this problem is quite common throughout the community, I may have found answers from a different page below, hopefully this helps better.

 

Scenario: The user attempts to open/load the CCC through various methods, but it fails to load. The issue may happen with no changes to the system or a Catalyst driver update or reinstall. This is issue mainly affects Windows 7 machines.

 

Cause: Possible corruption of the Windows Global Assembly Cache (GAC). This usually requires the user to format and reinstall Windows.

 

Resolution: This solution has been tested - reproduced and resolved.

 

Remove all ATI through programs and features in the Control Panel using the Catalyst Install Manager (CIM). Reboot the system in safemode and do the following:

 

-          Delete all ATI folders in C:\programs files and C:\programs files (x86)

-          Go to C:\Windows\assembly and sort by "Public Key Token". Locate the following Public Key Token "90ba9c70f846762e" that is

-          associated with ATI. Usually only 2 entries. Delete the 2 entries. Note: the entries should be deleted (right click and choose uninstall) in safemode since regular

-          mode may result in a permissions error. The 2 entries only show up in systems affected by this issue. This means that searching for the 2 entries on systems where CCC loads, the entries are not there.

-      Shut down and re-start normally.

-          Reinstall the ATI Catalyst package with driver/ccc bundle from here: http://support.amd.com/us/gpudownload/Pages/index.aspx

 

Hopefully this will get you back in working order!

Hi,

 

Use the Windows Backup Files wizard to restore all of the files that you backed up using the wizard. Follow these steps to restore your backup files:

 

  1. If the backup files are on a CD or DVD, insert the CD or DVD into the disc drive.
  2. If the files are on an external hard disk drive, connect the hard disk drive to the computer.
  3. Click Start , and then click Control Panel .
  4. Under System and Security , click Back up your computer .
  5. The Backup and Restore Center opens.
  6. Click Restore my files or Restore all users' files .
  7. Add files and folders to be restored using Search , Browse for files , or Browse for folders .
  8. When done adding files and folders, click Next .
  9. Choose to save the restored files In the original location or In the following location .
  10. If you choose to save the restored files in a different location, click Browse to save to another location.
  11. Click Restore .
  12. The Windows Back Up Files wizard now restores the files.
  13. When the wizard has finished restoring the files, click Finish on the verification screen.

 

Hope this helps
