rohn007 say:
The Fall Creators Update added a new "feature" that automagically opens any applciations you had open when you shut down Windows. The intent is to make you life easier. Some people don't like it because they are used to using shutdown as a "close all" process, and don't want to see those apps open on restart.

Others have reported the same problem you have. Each time they start windows, more and more duplicate application windows are opening. They too fear reaching a point where all of the open applications will crash windows.

In theory, if you close down all applications before triggering the shutdown should prevent the problem. In practice, as you've found, that doesn't always work as expected.

A workaround, rather than shutting down Windows, us Sleep or Hibernate to avoid triggering the 'improvement".

a_tempest say:

Right excellent! Thank you for that bit of information, knowing that I've found a solution to the issue. If you launch command prompt (search cmd.exe) and shutdown using the command "shutdown -t 0 -s" it breaks the cycle of ever-increasing ms word opening. and I no longer have Word launching on startup.

However, if I use Word again in the next session and shutdown or restart normally the issue returns at square one, one Word document launching on startup. 

To get around this you can create a shutdown shortcut on your desktop (or anywhere) by right-clicking, new, shortcut, in the location or item box enter the command "shutdown -t 0 -s" name it "shutdown" or whatever tickles your fancy and use this to shutdown instead of the start menu item and the issue will cease.

If you want to restart you can use the command "shutdown -t 0 -r"

I hope this can help anyone suffering the same bug.

Thank you again for being generous with your time and for the help.

p.s. you can make the shortcut look the part by right-clicking the shortcut, properties, change icon, select an appropriate icon.
