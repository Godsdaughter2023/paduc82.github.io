However, the dashed-border picture cannot be placed outside of a text box, while the blue-border pic can be placed anywhere. What is the difference between these pictures? bob jones aka: cybertaz say:
One point to be aware of is that the document is saved in the old .doc format -- that's why the one boundary appears as a dotted black line but the other is solid blue.

The main distinction, though is that the dotted black line indicates that the image is set as In Line with Text. It is treated as though it were a text character so it can be positioned only in a paragraph. The other image has Text Wrapping applied to it which makes it a floating [rather than In Line] image. Floating images can be dragged to virtually any location but are still anchored to a paragraph.

To change the status of either image, right-click it & select Format Picture - Layout. You can also make the change on the Format Picture contextual tab of the Ribbon by clicking the Wrap Text button in the Arrange group.

See Word Help on Align a picture with text for more details. You'll also find a more in-depth explanation with additional links at the bottom of the page here:

Word Doesn’t Know What a Page Is

Try this worksheet event code.

No need to hide any rows, sorting takes place from row 7 and below if a value is changed in any cell below B6

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim iSect As Range
    Dim Lrow As Long
    Lrow = Cells(Rows.Count, "B").End(xlUp).Row

End Sub

john - project say:

gztag,

To be honest I don't know why it doesn't work. A little investigation revealed that the "Add" method seems to somehow shift focus from the current task assignment loop such that the loop never terminates. It indeed copies over all the resource assignments from the first task to the second task, but then instead of exiting after the last assignment, it repeats the loop indefinitely. That's why you get the error message.

One small comment about your function. Even though you don't use the aDest variable, it is NOT declared as an assignment as you may think. It is in fact declared (by default) as a variant type. Why? Because in VBA all variables must be explicitly declared. So it you were using the aDest variable and wanted it to be of type assignment, the declaration should be:

dim aDest as Assignment, aSrc as Assignment.

Okay, as I said, I don't know why your code doesn't work, but this variation of it will work:

Public Function CopyResources(src As Task, dest As Task) As String
    Dim aDest As Assignment, aSrc As Assignment

    On Error GoTo ErrHandler
    For i = 1 To src.Assignments.Count
        Set aSrc = src.Assignments(i)
        dest.Assignments.Add ResourceID:=aSrc.ResourceID, Units:=aSrc.Units
    Next
    On Error GoTo 0

    CopyResources = "Success"
    Exit Function
ErrHandler:
    CopyResources = Err.Description
    On Error GoTo 0

End Function
