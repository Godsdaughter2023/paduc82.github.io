There's really not much difference between the two. 7.0 basically added compatibility for the 64 bit environment, which included the addition of a couple of new keywords and one or two functions. 7.0 is backwards compatible with earlier versions, so if you're having troubles with your application in different environments the cause is almost certainly NOT the version of VBA in those environments. There is very little difference in the language itself. But in my experience, each version of Access has become less forgiving of syntax errors then the previous versions. So code that may have worked in an older version may cause a problem in newer versions.

vanderghast say:

As John's mentioned it, you can't use a Validation rule to implement a rule across records, which is required to find a SUM, but you can use a CHECK constraint, in Jet. Unfortunately, the CHECK constraint, in Jet, has no user interface, so you have to add it 'by code', or in the Immediate Window. As example, to a table table2, if I want that the sum of values of f2 over all records to be less or equal to 1, I can use:

 

CurrentProject.Connection.Execute "ALTER TABLE table2 ADD CONSTRAINT SumF2LessThan1 CHECK( 1>= (SELECT SUM(f2) FROM table2))"

 

Note: the contraint name, here SumLessThan1, should be unique IN THE DATABASE  (not just for table itelf), so you may append the table name to it:

CurrentProject.Connection.Execute "ALTER TABLE table2 ADD CONSTRAINT SumF2LessThan1_Table2 CHECK( 1>= (SELECT SUM(f2) FROM table2))"

 

Again, since that CHECK constraint does not have any User Interface, it is to be use only if required. 

The CHECK constraint is ...checked... at INSERT and at UPDATE (but not at DELETE), and cannot involve VBA functions.

The CHECK constraint used like this is only a feature of Jet, I mean, it does not work with MS SQL Server (across the records of the same table, or of another table) which uses a trigger to acheive the same effect.

I just migrated 2010 to O365, and had some Public Folders to migrate. I decided during pre-migration that I was going to replace the public folders with shared mailboxes.
