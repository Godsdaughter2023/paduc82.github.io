Creating a Microsoft account using an existing email address should come with a warning.

 

When you create a Microsoft account, the account will have an Inbox, SkyDrive, People, ....

The Inbox will not receive emails to the Gmail address.  All emails sent to the Gmail address will still go to the Gmail server.

 

To get the Hotmail emails to your new Microsoft account, you will have to add an alias and forward the emails to the alias.  Add the Hotmail account as a Send only account.  Note: you must sign in to your Hotmail account using a browser at least once every 365 days to keep the account active.

 

To get the Gmail emails to your new Microsoft account, forward the emails to the added alias.  Emails will send with the Gmail address.  Any replies will go to the Gmail servers and will have to forwarded to the alias address.

 

Emails sent from the Microsoft account will be saved in the Sent folder.  Emails sent from the Gmail account will be saved on the Gmail server.

 

You now have two passwords to maintain.  The Gmail password and the Microsoft account password.

daniel g. grau say:

No, it's not possible to do that without a set of macros. Here they are:


Sub RememberPosition()
    ActiveDocument.Variables("SaveTop") = Application.ActiveWindow.Top
    ActiveDocument.Variables("SaveLeft") = Application.ActiveWindow.Left
    ActiveDocument.Variables("SaveHeight") = Application.ActiveWindow.Height
    ActiveDocument.Variables("SaveWidth") = Application.ActiveWindow.Width
    ActiveDocument.save
End Sub
Sub ForgetPosition()
On Error GoTo NoParametersDefined
    ActiveDocument.Variables("SaveTop").Delete
    ActiveDocument.Variables("SaveLeft").Delete
    ActiveDocument.Variables("SaveHeight").Delete
    ActiveDocument.Variables("SaveWidth").Delete
    ActiveDocument.save
NoParametersDefined:
On Error GoTo 0
End Sub
Sub AutoOpen()
On Error GoTo NoParametersDefined
    Application.ActiveWindow.Top = ActiveDocument.Variables("SaveTop")
    Application.ActiveWindow.Left = ActiveDocument.Variables("SaveLeft")
    Application.ActiveWindow.Height = ActiveDocument.Variables("SaveHeight")
    Application.ActiveWindow.Width = ActiveDocument.Variables("SaveWidth")
NoParametersDefined:
On Error GoTo 0
End Sub

To add the macros to your Normal.dotm template

  1. In your main menu bar, select Tools > Macro > Visual Basic Editor
  2. You should see a Project box
  3. In there, select Normal (Normal.dotm)
  4. In the main menu bar of the Editor, select Insert > Module
  5. A new window will open
  6. Paste the above code (from the first "Sub" to the third "End Sub")
  7. You can save your Normal.dotm template now (File > Save Normal.dotm) or Word will prompt you when quitting
  8. In the Editor's main menu, select Close and return to Microsoft Word (or do Command Q)

To create a new toolbar, add the macros to it and define shortcuts

  1. In Word's main menu bar, select Tools > View > Customize Toolbars and Menus.
  2. At the top of the dialog box, select the Toolbars and menus tab.
  3. Click on the New button at the top right. Give the toolbar a name and press OK. Pay attention, as the new toolbar will be hard to see it because it's small. You can move it around by grabbing it from the "colon" below the Close circle.
  4. While still in the Customize... dialog box, click on the Commands tab at the top.
  5. In the "Categories" list to the left, click on "Macros" to select it.
  6. In the list to the right, locate "RememberPosition."
  7. Click on that name and drag it to the new toolbar.
  8. If you click on the "Keyboard" button at the bottom, you'll be able to add a shortcut (for instance, Control R).
  9. Repeat steps 6-8 for the "ForgetPosition" macro
  10. As before, you'll need to save Normal.dotm (if Word prompts you, do so).

Comments

Although it would be possible to make these macros run automatically, this is risky because the document needs to be saved in order to store its window parameters. Many a time one opens a document  with the intention to close it without saving, and an automatic save operation would interfere with this approach. Therefore, I decided to simply tell each document that you want to permanently remember its window position (the "RememberPosition" macro). As a complement, I added a forget option (the "ForgetPosition" macro) to restore the usual behavior. Both of these macros work on the active document (the frontmost window) and do a save operation. Thus, if the active document has unwanted changes, don't use the macros at that point in time. 

The third macro (AutoOpen) will execute automatically each time you open a document, so you don't need it in the toolbar and neither do you need a shortcut for it. If the document contains window position parameters, AutoOpen restores the window according to those values. Otherwise, it does nothing.

I did not add restore View or Zoom options, although they could be implemented as well. I just didn't think they were as useful.

Finally, window positions cannot be remembered in .TXT documents because they don't support the storage of variables.
