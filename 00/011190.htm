macropod say:

The only practical way of doing this in Word would be to use formfields or content controls in columns C through G, coupled to 'on-entry' and 'on-exit' macros to do the calculations. For a formfield solution, where the formfields are defined as the "Date" type with a "00:00" format, the macro encoding might look like:

Option Explicit
Dim iRow As Long

Private Sub Document_Open()
Call GetRow
End Sub

Sub GetRow()
iRow = Selection.Rows(1).Index
End Sub

Sub Calc()
Dim sTm1 As Single, sTm2 As Single, sTm3 As Single, sTm4 As Single
Dim sTmEarly As Single, sTmLate As Single, sTmSum As Single, i As Long
With Selection.Tables(1).Rows(iRow).Range
  sTm1 = CSng(CDate(.FormFields(1).Result))
  sTm2 = CSng(CDate(.FormFields(2).Result))
  sTm3 = CSng(CDate(.FormFields(3).Result))
  sTm4 = CSng(CDate(.FormFields(4).Result))
  If sTm1 <> 0 And sTm2 <> 0 Then
    sTmEarly = sTm2 - sTm1
  End If
  If sTm3 <> 0 And sTm4 <> 0 Then
    sTmLate = sTm4 - sTm3
  End If
  .FormFields(5).Result = CDate(sTmEarly + sTmLate)
End With
With Selection.Tables(1)
  For i = 2 To .Rows.Count - 1
    sTmSum = sTmSum + CSng(CDate(.Rows(i).Range.FormFields(5).Result))
  Next
  sTmSum = Round(sTmSum * 24, 0)
  .Rows.Last.Range.FormFields(1).Result = Format(Int(sTmSum), "00") & ":" & Format((sTmSum - Int(sTmSum)) * 60, "00")
End With
End Sub

where 'GetRow' is the on-entry macro for every formfield in columns C through F and  'Calc' is the corresponding on-exit macro. The formfields for holding the calculated hours in column G all have the 'fill in enabled' option unchecked. A table with a single heading row and a tally on the last row (in the only formfield on that row) is assumed and that cell is configured for plain text input (again, with the 'fill in enabled' option unchecked).

Frankly, it's a whole lot easier in Excel, which has all the required functionality built in. Even with all the bother of creating the form in Word, it can be crippled by something as simple as the user's macro security settings.

vanderghast say:

It depends on your version of Visual Studio. With VS2010, it is rather simple.

- Create a new project (say, a vb class library)

- From the menu, under Data, Add new data source ....

- Database  [next]

- Entity data model [next]

- Dataset [next]

- [New connection] browse and spot your accdb file; credentials ( user Admin, no password by default),

- [next]

- Dialog:"The connection you selected uses a local data file that is not in the current project. Would you ... Press F1 fo rmore info." Press F1 if it is new to you. I chose [no], to keep the file out of the project, but with the connection accessible from the config file that come with the dotNet application. You can change the NAME of the connectionString, since it is a string that can be read in the configuration file, or assigned programatically.[Next]

- Select the tables and queries to have as context [finish]

- Click on the generated xsd file, you have the equivalent of the Relations Views under Access, but in addition, you can modify the tables, the relations, and define methods (like local stored procedures) under each table (queries). You can add more tables/ relations with the right clickcontext menu.

 

You can now reach your data in a very convenient way, with strong typed names, quite easily, such as:

Public Sub Sample()
   Dim t As New database1DataSetTableAdapters.Query1TableAdapter
   t.GetData()
  ... do something with your disconnected data
End Sub

 

What you do next is up to you.

  Dim R As Range
  Dim i As Integer
  Dim FormField, DataField
  FormField = Array("D5", "C5", "B5", "B7", "B9")
  DataField = Array("A1", "B1", "C1", "D1", "E1")
 
  With Sheets("Database")
    'Search the ID in column A
    Set R = .Columns("A").Find(Range(FormField(0)), LookIn:=xlValues, LookAt:=xlWhole)
    'Found?
    If R Is Nothing Then
      'No get the next empty row
      Set R = .Range("A" & Rows.Count).End(xlUp).Offset(1)
    End If
    If LoadData Then
      For i = 0 To UBound(FormField)
        Range(FormField(i)) = Intersect(R.EntireRow, Range(DataField(i)).EntireColumn)
      Next
    Else
      'Store the data
      For i = 0 To UBound(FormField)
        Intersect(R.EntireRow, Range(DataField(i)).EntireColumn) = Range(FormField(i))
      Next
    End If
  End With
End Sub

You can also build dynamic input forms, e.g. if the headings in the database sheet in row 1 are the same in the cells on the left side of the input cell in the input form.

And if you create a named range for all input cells, you can visit each cell in that range, get the value of the cell on the left side (the heading), search it in row 1 in the database sheet, and store it in the same way as in the routine above. Here is an example:

jay freedman say:
A simple way is this: Select the line of text and copy it to the clipboard. On the Home tab, click the down arrow on the Paste button and choose Paste Special. In the dialog, choose Picture (Enhanced Metafile). On the inserted picture, use the circular-arrow handle at the top center to rotate it 180 degrees.

Unfortunately, the conversion to a picture tends to make the picture of the text a little fuzzy compared to the original text. A different approach can avoid this at the cost of being a little more complicated, and not suitable for complicated layouts: Change the page orientation to Landscape. Insert a 1-row x 2-column table. Right-click in the left column and choose Text Direction, and select the button that makes the text read top-to-bottom. Click OK. Now right-click in the right column, choose Text Direction, and select the button that makes the text read bottom-to-top. The text you write into each cell will be upside down with respect to the text in the other cell. When you print the page and hold it in portrait orientation, one of the cells will be inverted.
