maor bracha say:

The easiest and supported way of doing this would be to use the "New-MailboxRestoreRequest" command.

1. Disable JohnOld mailbox.

2. Enable JohnNew mailbox.

3. Use "New-MailboxRestoreRequest" to restore soft deleted mailbox content to the new mailbox.

New-MailboxRestoreRequest -SourceMailbox "JohnOld" -TargerMailbox "JohnNew" -AllowLegacyDnMismatch:$True;

In case you would like to restore all data to a new folder, such as "Old Data", you could add the switch:

-Target RootFolder "Old Data"

Of course that using ADSI edit is always possible for any MS related issue, and fiddling with ADSI+DirSync to "trick" Office 365 in particular, but sometimes it might be a better idea to take the highroad- especially if not familiar with the process.

I'd recommend using this method in order to safely "migrate" your data between accounts, in the supported way.

Maor

How can I stop Microsoft Security Essentials from scanning my WD - Microsoft Community

elsie1937 say:

How can I stop Microsoft Security Essentials from scanning my WD External Hard Drive?

How can I stop Microsoft Security Essentials from scanning my WD External Hard Drive?  It takes hours to perform a full scan because of WD.
jimr1 say:

First make sure the drive is attached and can be seen in Windows Explorer. Then check that "Scan removable drives" in the advanced settings of Microsoft Security Essentials is unchecked. Go to "Excluded files and locations" and use the "Browse" button to locate the drive letter of the removable drive. Select it and the "Add" button should activate and enable you to exclude the drive.

Jim
