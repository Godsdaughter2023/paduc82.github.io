donna men say:

Hi Larry,

Although there is a setting for enabling Fn keys in the BIOS, there's a chance that this might be just from the device setting and it may differ depending on what device you are using since it is programmed by the manufacturer.

To disable or enable the function key (fn) in the BIOS, simply follow these steps:

  1. Press the power button to turn on the computer.
  2. Press the F10 key to open the BIOS setup window. (Key may also depend, some devices needs to hold down the Esc key to be opted to the BIOS settings.)
  3. Navigate to the System Configuration option but using the right or left arrow keys.
  4. Use the up or down arrow keys to navigate to the Action Keys Mode option, and then press the Enter to display the Enable / Disable menu.
  5. Select the desired mode.
  6. Press the F10 key to save the selection and restart the computer.

If issue persist, we recommend you to check with your device Original Equipment Manufacturer (OEM) for more appropriate solution.

Like Spirit MS MVP indicates...  when you see a BSOD that implicates a core Windows system file like tcpip.sys those are extremely unlikely to be the cause of the BSOD - it is always something else.

It may take more than one crash dump file to figure out the cause of the crash.

It could be that if the systems are set up to receive and install Windows Update automatically they have downloaded some hardware driver updates from Microsoft and that is never a good idea.  You can configure Windows to not accept hardware drivers from MS and that is a good adjustment to make that you can read about here:

Hot to stop Windows 7 from automatically installing drivers

Since Microsoft puts out updates the second Tuesday of every month (aka Patch Tuesday) that would be yesterday so if these systems download/installed updates from MS I would suspect they got a bad driver from MS (surprise, surprise).

What is Patch Tuesday?

You can look in Device Manager at the currently installed network device driver version and make a note of it as being the 'suspicious' driver version and if the Roll Back Driver button is available roll back the driver to the previous version, restart, check the driver version again to be sure it worked and see how things look.
