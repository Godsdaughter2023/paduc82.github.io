You could use the worksheet change event - put this code into the Thisworkbook object's codemodule - when any C7 is changed, all C7 cells are changed.  Change "Source List" to the name of the sheet with the source list, and Sheet2 and Sheet3 to the data sheet names, or if your described three sheets are always to the left, then you can use the counter...

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
    Dim sht As Worksheet
    Dim DVCell As String
    Dim i As Integer
    
    DVCell = "$C$7"
    
    If Target.Address <> DVCell Then Exit Sub
    
    Application.EnableEvents = False
    
    'If the sheets can move, check their names
    For Each sht In Worksheets
        If sht.Name <> "Source List" And sht.Name <> "Sheet2" And sht.Name <> "Sheet3" Then
            sht.Range(DVCell).Value = Target.Value
        End If
    Next sht
    
    'If the sheet with DV are always 4th and on, use this
'    For i = 4 To Worksheets.Count
'        Worksheets(i).Range(DVCell).Value = Target.Value
'    Nexti
        
    Application.EnableEvents = True
    
End Sub

a. user say:

I suspect what is happening here is that when you paste into the dialog box, Word "helpfully" picks an address from the "Most Recently Used" (MRU) address history that happens to start with the address you actually pasted. The MRU is the thing you see when you click the dropdown next to address.

In that case, Word appears to select the part of the link text that it has added, so you can clear it using backspace.

I do not know of any option you can set to change this behaviour. You *can* modify the list in the MRU by modifying the Windows Registry, but that's usually a bit of a last resort and the problem will only reappear after you have visited a few more pages.

Could be worth posting an item in https://word.uservoice.com/ - I don't see an existing one on this subject.

The option is available in IE 8 unless it’s grayed out for some reasons. In this case, post back and let us know so that we can take call on this.
3)    Once you select Import and Export, you should get another window “How do you want to import and export your browser settings?”
4)    If you select “Import from another browser”, you would be provided with a list of other browsers installed on the computer. Click on “Import” to start importing.
5)    Likewise you can select “Import from a file” to import “Favorites”, “Feeds”, “Cookies”.

You can find Chrome’s Bookmarks Location and Others by following these steps.

1)    Right-Click the Google Chrome Icon and Properties
2)    Get the Target value (up to Chrome Folder only) and paste the following to windows explorer:

C:\Users\[USERNAME]\AppData\Local\Google\Chrome\
The AppData is a hidden folder. Enable the explorer to show hidden files and folders.

Read:

Show hidden files
http://windows.microsoft.com/en-US/windows7/Show-hidden-files.

3)    Now you will see the User Data Folder.
4)     Continue Browsing and open the Default Folder
C:\Users\[USERNAME]\AppData\Local\Google\Chrome\User Data\Default

This directory has two files: "Bookmarks", and "Bookmarks.bak". Bookmarks is for your current bookmarks, and the other one (with the .bak extension) signifies that it is a bookmarks backup, if anything goes wrong. Bookmarks.bak gets overwritten every time you close Chrome.
