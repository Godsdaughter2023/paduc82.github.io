ossiemac say:

It is quite a long explanation but I want to provide a little more than just edit your code with the hope you will understand why. Explanation is based on your example formula in cell G3 of Example wkbk.xlsm that you uploaded.

I renamed the worksheet to "Data" for this purpose.


Your array formula in E3 is as follows (without the curly brackets for array formula). This formula is on the same worksheet as the data table.


=INDEX(C2:C11,MATCH(1,(E3=A2:A11)*(F3=B2:B11),0))


To use that formula in VBA and return the calculated value to a VBA variable is the following code.


    Dim result
    result = Evaluate("=INDEX(C2:C11,MATCH(1,(E3=A2:A11)*(F3=B2:B11),0))")


Note that the formula is identical to the formula on the worksheet but it is enclosed in double quotes so essentially the formula is a string containing a worksheet formula and the Evaluate function calculates the value for what is basically a worksheet formula and the syntax within the string is the same as used on a worksheet.


If the formula is placed in a cell on a different  worksheet then it needs to reference back to Data worksheet like the following.


=INDEX(Data!C2:C11,MATCH(1,(Data!E3=Data!A2:A11)*(Data!F3=Data!B2:B11),0))


To use that formula in VBA and return the calculated value to a VBA variable is the following code.


result = Evaluate("=INDEX(Data!C2:C11,MATCH(1,(Data!E3=Data!A2:A11)*(Data!F3=Data!B2:B11),0))")

Again note that the formula within the double quotes is identical to the formula on the worksheet but is enclosed in double quotes.

If the actual string values of E3 and F3 are  used in the formula on the worksheet then the formula would be like the following with those values enclosed in double quotes

=INDEX(C2:C11,MATCH(1,("Rocker"=A2:A11)*("8A"=B2:B11),0))

The VBA would then be as follows. Note that because the strings are enclosed in double quotes in the formula on the worksheet and entire formula is also in double quotes, we need to add the additional double quotes in the string to evaluate.

result = Evaluate("=INDEX(C2:C11,MATCH(1,(""Rocker""=A2:A11)*(""8A""=B2:B11),0))") 

Now if you want to use values from outside of the values on the worksheet (eg. lbltype.Caption), you still need to finish up with a string which will emulate a worksheet formula to evaluate. This involves concatenation whereby you close off the double quotes and insert an ampersand and the VBA control reference then another ampersand and then open the double quotes again.

Plus we need add the double quotes around the lblType.caption because it becomes a string without double quotes when concatenated in the overall string. Chr(34) is a double quote.


See following example. (I used ranges matching ranges in the example workbook. Edit the ranges where I have used Data!C2:C11, Data!A2:A11 and Data!B2:B11 to match your project ranges.)

    Dim Bkr
    Dim result
   
    Bkr = "8A"      'Variable populated for testing.
    
result = Evaluate("=INDEX(Data!C2:C11,MATCH(1,(" & Chr(34) & lblType.Caption & Chr(34) & "=Data!A2:A11)*(" & Chr(34) & Bkr & Chr(34) & "=Data!B2:B11),0))")

Feel free to get back to me if you are still having problems with it.

    Dim Bkr     'Default is variant (Must be variant NOT String so value can be either numeric or string)

    Bkr = 5      'Variable populated with a numeric value for testing
   
    If Not (IsNumeric(Bkr)) Then

        Bkr = Chr(34) & Bkr & Chr(34) 'Concatenate double quotes if NOT numeric

    result = Worksheets("Data").Evaluate("=INDEX(C2:C11,MATCH(1,(" & Chr(34) & lblType.Caption & Chr(34) & "=A2:A11)*(" & Bkr & "=B2:B11),0))")

marshall barton say:

If there are more than one user, splitting the database is mandatory.  It is strongly advised even for one user.  Then the "front end" db with all the forms, report, queries, etc. need to be copied to each user's hard drive or private share area.  The "back end" with only the tables and relationships then can be place on the server.

 

There are several ways to make data read only, but it depends on exactly what you want to do on each form (reports are always read only).  using the Allow... properties pretty much prevents users from doing anything in a form.  If you want to allow users to do a few things, such are filter the form's records, then you need to Lock (and probably disable) the controls that display the protected data, but leave the other controls unlocked and enabled.
