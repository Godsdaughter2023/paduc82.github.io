Well, you need them to be changable because at some point you'll want to select something from the list.  If you don't want to be able to enter something that is not in the lookup list, just set the combo box's LimitToList property to Yes.

ken sheridan say: You may be able to anticipate, and avoid, the error in the combo box's BeforeUpdate event procedure by checking whether the selected value if the already exists in the underlying table, and if so setting the return value of the procedure's Cancel argument to True, e.g.

Const MESSAGETEXT = "Selected product already exists in another record."
Dim strCriteria As String

strCriteria = "ProductID = " & Me.cboProduct

If DLookup("ProductID", "OrderDetails", strCriteria) Then
    MsgBox MESSAGETEXT, vbExclamation, "Invalid operation"
    Cancel = True
End If

The above is an unrealistic example of course, implying each product can be ordered in one order only, but serves as an adequate illustration.

However, if the database is operating in a multi-user environment, one or more other users could make the same selection before the first user saves their record, in which case all would be accepted at this stage, but all but the first to save the record would cause a key violation error, so even with the above you should also handle the error in the subform's Error event procedure as Marshall suggests.

If the key is a composite one, e.g. of OrderID and ProductID (a more realistic, and indeed very common scenario) where OrderID is the subform controls LinkChildFields property the this should have been assigned a value automatically by the linking mechanism so you should be able to extend the criterion:

strCriteria = "ProductID = " & Me.cboProduct & " And OrderID = " & Me.OrderID

In other circumstances involving a composite key you may not be able to guarantee that both components of the key have been assigned values at this stage, so the form's Error event procedure is the only safe bet, but you should do that in any case except in a guaranteed single user environment.

Ken Sheridan, Stafford, England

yacinthus say:

How to solve driver support problems especially when you run Windows 10 on older hardware architecture – on an example with ATI Mobility Radeon x1600 video card driver

In the above mentioned scenario you might be faced with various driver incompatibility problems due to producers’ lack of support which does not mean that your hardware is not Windows 10 capable (if you are aware of what you have on board), but mainly to unsupported drivers for the new OS. Windows 10 upon installation will install its own, usually basic drivers and guard them like a dog not allowing any modifications. In my case it was the unsupported driver for my ATI Mobility Radeon X1600 video card (not a new one, for sure, but under Omega drivers it has allowed me to play Mass Effect 3 with all settings maxed out; the original ATI drivers did not allow me even to play Mass Effect 1). Anyway, the basic Microsoft Display Adapter driver did could not recognize my hardware properly and I was stuck with lower screen resolution (1024 x 768) instead of the native one, which is 1280 x 800.

yacinthus say:

Hi Sam_R,

I am happy to hear it worked for you. In the Device Manager, under display adapters (= your currently installed video cards or graphics cards, whatever you call them) I had Microsoft Basic Display, which was the Win 10 bundled (I guess) driver that had overridden my original Radeon X1600 driver upon Windows 10 installation. The name "Microsoft Basic Driver" in case of other machines might differ as some other driver might have got installed, though I believe that whenever Windows cannot identify correctly your video card it will install some sort of MS basic driver to handle your display. Anyway, the idea is that whenever you experience a downgrade in your display experience after installation of the MS Windows 10, due to the improper recognition of your original driver, just follow the steps to replace it with the proper one, in the Device Manager under Display adapters (whatever the name it has, not corresponding to your original one, in my case that name was Microsoft Basic Display).
