Access 2016: calculate Age from DateOfBirth - Microsoft Community

andrewlouwrens say:

Access 2016: calculate Age from DateOfBirth

In Microsoft Access i have a Table, PlayersT. In that table I have , amoungst others, the following Field Names: ID, DOB and Age. I need the Age to be calculated from ID.

All our ID's are 13 digits, the first 6 being yymmdd. A typical ID would be 3812275007081. This person would be 80, i.e. born on the 27th December 1938.

In the PlayersT in design view the field Name Age would be a calculation but I can't get the expression right. Also i could pull the Date of Birth (DOB) field name from the ID, so I would need an expression for DOB field name.

Possible?

scottgem say:
Hi Andrew, I'm an independent adviser and will try to help.

First,, good database design says that fields should be atomic. This means that each field should be the smallest piece of information. So including the DOB within the ID is not good design. I'm going to assume, however, that you are getting these IDs from another source that you have no control over. So if the ID contains the DOB, then you should NOT have fields for DOB and Age. A DOB field is redundant and Age is calculation.

You can pull the DOB from the ID with this expression:
DateSerial(Left(ID,2), Mid(ID,3,2), Mid(ID,5,2)

This can then be used to display the DOB on a form or report and use it in a calculation of the Age. You can use this expression to calculate age:

DateDiff("yyyy",[DOB],Now())+Int(Format(Now(),"mmdd")<Format([DOB],"mmdd"))

DOB would be the variable you assigned to the DateSerial expression.

If you need further help with this please feel free to ask.

a. user say:

Hello my name is Michael.

This is by design. Everything is converted in the Junk folder to plain text by default. This is to protect you. Some junk can contain links or images that can contain malware and viruses, so it is converted to plain text. I would suggest creating a new folder and a rule for each sender you want to keep and read and have them automatically moved to a different folder.

You can read this article by Microsoft on how to manage emails using rules:

https://support.office.com/en-us/article/manage...

I hope that helps
