When you paste in code that uses line feeds as endings (LF only, the default in Unix/Linux), it doesn't display the line feeds as new lines. They are still there, embedded in the text.

Notepad++ is a full-fledged text editor, please reference the link in my previous post, I think you'll find it useful. It can automatically display any variant of LF, CR, CR+LF or LF+CR as a line ending.

Your illustration shows what normally happens when you copy and paste code between operating systems. Different OSs use different line endings. This isn't a bug, it's just something you have to learn to deal with.
