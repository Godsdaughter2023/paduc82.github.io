ryanlv_msft1519 say:

Hi TechID,

Yes, forest B users will be impacted.
It is expected that when forest B users have their own tenant B, we need to remove domain B in tenant A and add it to tenant B. During this transition process, forest B users will be removed from tenant A and lose the ability to access the shared files. After adding to tenant B, we will need to share files with forest B users again. And then they will able to access using their tenant B Office 365 account as an external user.
The advantage of this plan is that when forest B users are added in tenant A, they are treated as internal users and we can better manage them.

On the other hand, if we plan to have forest B users access the files without syncing them to tenant A, they will either use Microsoft accounts or no-login required method(which might be unsafe) to access the shared files.
Using Microsoft account is regarded as an external user, however, the advantage is that they won’t be affected in the process of setting up tenant B.

You may consider one of the plans above which serves your situations better.

Ryan
