scottgem say:
Yes, First your application needs to be split. You have a back end (just the tables) that resides on a network share. Then you have a front end (with everything else) that is deployed to each user's workstation.

Next, you also have 2 other copies. You have a master which is used to update the local copies whenever changes are made and you have a development copy where you do all your enhancements, bug fixes, etc. 

When a new version has been tested and is ready to deploy, the master copy is replaced with the new version which is then downloaded to the local drive using one of several front end updater methods (I use AutoFEUpdater).

ken sheridan say: You seem to have everything you need.

1.  The OBFFDatabase_be.accdb file goes in a shared location on the network to which all users have full permissions.

2.  A separate copy of the OBFFDatabase.accde file goes on each user's local machine.

3.  Keep the OBFFDatabase.accdb file in a safe place; this is your development file on which you will make any design changes.

4.  The OBFFDatabase.Iaccdb file is a temporary locking file created when the database is in use.  You don't need to do anything with this.

When you install a copy of the OBFFDatabase.accde file on a  user's local machine you can if necessary use the built in Linked Table Manager to refresh the links to the back end OBFFDatabase_be.accdb file.  This might not be necessary if the back end has not been moved to a new location, but if it is then it's a once-only task unless the back end is subsequently moved to another location on the system.

As regards backing up you must of course back up both .accdb files to start with, preferably in at least two separate and secure locations.  The OBFFDatabase.accdb file should be backed up whenever you make any changes to it.  The most important thing is that the OBFFDatabase_be.accdb file is backed up on a regular basis, preferably daily.  A specialised back-up application can automate the backing up of your system, and has much to recommend it, but you'll find a cheap and cheerful Access solution demonstrated in BackUpDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file also includes a means of checking the validity of the links to the back end at start-up, and refreshing them if necessary.

charles kenyon say:

There is nothing built into Word for this.

Try:

  1. Paste as a whole into Word.
  2. Apply Heading 1 style to it all.
  3. Change the style to have paragraph formatting of page break before.
  4. Press Ctrl+H to Replace.
  5. Replace ". " without the quotation marks with ".^p" also without the quotation marks.

You'll have to do some cleanup, I expect.

doug robbins - mvp office apps & services (word) say:
Copy the list into an Excel Workbook and use it with a label type mailmerge, the main document of which is based on an Avery 3483 Shipping Label template in which you merge the two cells in each column so that you have a 1 row x 2 column table, each cell of which is half an A4 landscape sheet.  Set the text alignment in each cell so that it is centered vertically and horizontally and insert a mergefield into each cell, with a «Next Record» field before the merge field in the second column.  Then execute the merge.
