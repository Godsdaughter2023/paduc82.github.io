hansv mvp say:

If you enter the formula =COUNTIF($B$3:$B$10,B3) in a cell, and then fill this formula down, or copy it to another location, $B$3:$B$10 will remain the same because the reference is absolute, while B3 will change because that reference is relative.

But if you leave the formula in the same cell, and drag B3:B10 (the range that it refers to) to another location, both $B$3:$B$10 and B3 in the formula will change to reflect the new location, since the formula refers to those cells and you have moved them.

If you would like the formula to refer to B3:B10 even if you move that range to another location, you can use INDIRECT("B3:B10"). Since "B3:B10" is a text string, Excel will leave it unchanged. So your formula would become

=COUNTIF(INDIRECT("B3:B10"),B3) <= 1

kareninsf say:

Redlining

How do I turn off redlining beyond accepting all changes so that if I reopen the document the redlining does not appear?
herb tyson say:

If you're using Word 2007 or 2010, choose File (Office button if Word 2007) - Options - Trust Center - Trust Center Settings - Privacy Options, and remove the check next to "Make hidden markup visible when opening or saving".

Note, however, that this affects only what YOU see--not what anybody else sees when opening your red-lined documents. So, the redlining might be invisible to you... but it won't be invisible to others unless they've also chosen this dubious option. Personally, I absolutely want to know when documents contain tracked changes. So, I would never turn this setting off. The reason it's turned on by default is to prevent users from accidentally sending tracked changes to clients. Before this became the default, there were countless embarrassing episodes when clients received comments and tracked changes the document author had forgotten about or thought to have been removed.

It was not working great, it was crippling Word! They just did not realize it.

However, it is easy to have a document based on the template created whenever anyone opens Word.

Put the template in a standard location on each computer (see Workgroup Templates). I am assuming the Letters & Faxes folder in the Workgroup Templates folder. So long as there is such a template in the designated location these macros will work. They will work even if the designated workgroup templates folder is in a different location on each computer and even named differently.

Workgroup templates can be pushed to users, as can global templates, as part of the login.

Assume the name is Letterhead.dotm.

Create a macro that goes in a Global Template like the following:

Sub AutoExec()
    On Error GoTo NoOpenDoc
    ActiveDocument.Close SaveChanges:=False
NoOpenDoc:
    Documents.Add Template:=Application.Options.DefaultFilePath(wdWorkgroupTemplatesPath) & _
        "\Letters & Faxes\CompanyLetterhead.dotm", DocumentType:=wdDocument, Visible:=True  

End Sub

Whenever Word is started, a document based on the letterhead will be the new document.

The FileNew macro to match would be:

Sub FileNew()
    On Error Resume Next

        "\Letters & Faxes\CompanyLetterhead.dotm", DocumentType:=wdDocument, Visible:=True  

That would intercept Ctrl+N.

People would still have access to other templates using File > New.

In addition, you could give them access to other custom templates by adding the classic fileNew dialog to the QAT in your global template. File New Variations in the Versions of Word

For information on using macros, see Installing Macros.
