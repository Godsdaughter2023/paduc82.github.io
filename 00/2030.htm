erickchu_msft say:

Hi Alex,

You can sync the password from AD to Office 365 without on-premise Exchange servers. Generally, users have the on-premise Exchange servers to manage more Exchange related attributes. If you don’t have it, you need to use Powershell or ADSIEdit to manage them as Vasil mentioned. Meantime, if you haven’t the on-premise servers, you may not manage some Exchange attributes. Therefore, you can sync passwords without Exchange servers. The on-premise servers are related to attributes, not the passwords.

However, if you want to sync passwords, you need to use DirSync. Normally, we can use three tools to do it, such as DirSync tool, AADSync tool and AAD connect tool. Briefly, they are similar tools for the same purpose. Since AAD connect tool is a new one and you can download it from Portal, we recommend users to use it.

About matching email address, based on your description, I think you create mailboxes in Office 365 directly. Now, you want to connect AD users to Office 365 users and sync the AD password to Office 365. If so, you need to use SMTP matching (primary SMTP address) to connect AD users and Office 365 users. After that, when you use AAD connect tools to sync accounts, the password will be synced as well.
