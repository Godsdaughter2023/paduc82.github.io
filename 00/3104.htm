ossiemac say:

Can be done with Event driven VBA code (Macro) as per the code example below.

The columns/cells where the date is entered as numbers without delimiters must have Number Format set as General.

The code will convert it to the required Date Number Format after the conversion to a date. Reason for this is if Number format is set to a Date then Excel will convert any serial number to the equivalent date and the code will not run. This is the reason that your serial number was converted to August 25, 2125.

The number can be entered as 082418 and it will be converted to a date.

Caveat: If a date needs to be edited at a later time then it must be entered properly as a date. Entering as numeric will not work because the cell is formatted as a date and not General.

To install the Event VBA code:

  1. Right click the worksheet tab name
  2. Select "View code" (will open the VBA editor window at worksheet's module)
  3. Copy the VBA code below and paste into the VBA editor.
  4. Edit the code where I have indicated that you need to set the column where the numbers are entered for the dates.
  5. See the comment where I have indicated to edit the required Number Format for the date,
  6. Do not rename the sub.
  7. Close the VBA editor (Cross very top right of VBA editor window)
  8. Save the workbook as Macro enabled.
  9. Ensure macros are enabled. See help for how to do this. (Option to "Disable all macros with notification" should be OK.)

The code should automatically run each time you make an entry in the required column.

Feel free to get back to me if any problems encountered with the code.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim varTemp As Variant
    Dim dte As Date
   
    If Target.Cells.Count <> 1 Then Exit Sub
    If Target.Value = "" Then Exit Sub
   
    On Error GoTo ReEnableEvents
    Application.EnableEvents = False
   
    If Not Intersect(Target, Columns("C:C")) Is Nothing Then   'Edit "C:C" to column where the dates are entered
        If IsNumeric(Target.Value) And Len(Target.Value) <= 6 Then
            If IsDate(varTemp) Then GoTo ReEnableEvents         'Already a date so no change required
            varTemp = Format(Target.Value, "000000")            'Converts to 6 text digits (eg. if 82418 is entered)
            dte = CDate(Left(varTemp, 2) & "/" & Mid(varTemp, 3, 2) & "/" & Mid(varTemp, 5))   'Convert to a date
            Target.Value = dte    'Copy date back to the cell
            Target.NumberFormat = "m/dd/yyyy"   'Edit "m/dd/yyyy" to your preferred date format
        End If
    End If
   
ReEnableEvents:
    If Err.Number <> 0 Then
        MsgBox "An error occurred in module " & Me.Name & " Private Sub Worksheet_Change" _
                & vbCrLf & "Report this error to this project Administrator."
    End If
    Application.EnableEvents = True
       
End Sub

rohn007 say:
OK, now that you've confirmed there are no other office installations we can rule that issue out. That leaves Access Rights. 

Windows has been tightening up granting of "access rights", starting in Vista and they get tighter in Win7 and Win8.

Windows apps run from a "generic" registry key called "Current User" to make programming easiers. When you log in to a specific ID, your registry settings are copied to the "Current User" key.  When you shut down, if there have been changes, like installing Office, the current user settings are copied back to your specific ID in the registry.  What appears to be happening is that at shut down the settings are not being copied from current user to your specific ID. Therefore they have to be recreated each time, and you see the "configuring" message.  There are also specific folders and files setup under your specific user profile but they are less of a problem.
