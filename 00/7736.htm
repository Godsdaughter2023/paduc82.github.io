69l46 say:

You are right to be wary.

 

I strongly suggest that you do not attempt to use visual reports for earned value measures. Visual reports only exports EV measures from the resource view, which can be different fom the EV measures in the task view. Yes, I know that doesn't make sense, but it is how it "works". At least one problem is that MSP forces BCWP to zero if BCWS is zero. For a task, that makes sense, but if you add a resource to a task after baselining, the task BCWP can be non-zero, but because the BCWS for the new resource is zero, it forces the BCWP for the resource to zero. Thus, when you roll up the BCWP for all the resources on a task, the BCWP for the task in the resource view will be less than the BCWP for the task in the task view. Unfortunately, visual reports ignores the task-view numbers and only exports the resource-view numbers. There is no way to export time-phased EV measures from the task view with visual reports. The EV measures, and all the information for manually calculating EV measures on a time-phased basis are drawn from the resource view.

 

This means that if you change resource or task assignments that you have to rebaseline to get EV measures exported via visual reports. This, in turn, means that all schedule variances before the date the new baseline was entered are lost. However, even then I have had visual reports give me nonsensical numbers (-1E+122, 1E-122) for EV measures.

 

So, in my opinion, visual reports suffers from structural errors and implementation errors. Avoid at all costs.

 

The way I manage to do it is by setting up the details in the time-phased task view to include the EV measures. Then I select the entire time-phased table and paste it into an Excel spreadsheet. You have to make sure that you have the same details selected and have the task view listing the same levels each time.

 

Also, be careful in that sometimes the value displayed in the time-phased portion of a view will differ from the actual value of that cell that you can see in the entry line when you click on the cell. When this happens, you have to restart MSP and possibly reboot.

macropod say:

Try the following macro:

Sub CitationsReformat()
Application.ScreenUpdating = False
With ActiveDocument.Range
  With .Find
    .ClearFormatting
    .Replacement.ClearFormatting
    .Text = ""
    .Replacement.Text = ""
    .Font.Italic = True
    .Format = True
    .Forward = True
    .Wrap = wdFindStop
    .MatchWildcards = False
    .Execute
  End With
  Do While .Find.Found
    If InStr(.Text, ":") > 0 Then
      .Start = .Start + InStr(.Text, ":") + 1
      .Font.Italic = False
    End If
    .Collapse wdCollapseEnd
    .Find.Execute
  Loop
End With
Application.ScreenUpdating = True
End Sub

For PC macro installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm

Question about EnableCalculation? - Microsoft Community

oemsoft say:

Question about EnableCalculation?

I would like to know the property for EnableCalculation:

  • Once worksheet has been calculated, would EnableCalculation be set to be FALSE?
  • If I set manually EnableCalculation to be TRUE, would excel calculate again?

Does anyone have any suggestions?

Thanks in advance for any suggestions :>

joeu2004 say:

oemSoft wrote:

Once worksheet has been calculated, would EnableCalculation be set to be FALSE?

Not automatically.  See below.

oemSoft wrote:

If I set manually EnableCalculation to be TRUE, would excel calculate again?

The worksheet calculation occurs only when EnableCalculation transitions from False to True.

So I create the following doCalc macro:

Sub docalc()
Dim oldCalc As Boolean
oldCalc = ActiveSheet.EnableCalculation
ActiveSheet.EnableCalculation = False
ActiveSheet.EnableCalculation = True
ActiveSheet.EnableCalculation = oldCalc
showCalc
End Sub

Sub showCalc()
MsgBox ActiveSheet.Name & ": EnableCalc = " & ActiveSheet.EnableCalculation
End Sub

Of course, showCalc is optional.

The point is:  first I set EnableCalculation=False.  If it was already False, this is a do-nothing.  Then I set EnableCalculation=True to cause the worksheet calculation.  If the workbook is in Automatic calculation mode, dependent and volatile formulas in other worksheets are also calculated.  Finally, I set EnableCalculation back to its original state.

This last step does not cause any recalculation, regardless of whether oldCalc is False or True.

Also note that whenever the workbook is reopened, EnableCalculation is True for all worksheets, regardless of the state of EnableCalculation when the workbook was last saved.

So I put the following into the Workbook_Open macro:

Private Sub Workbook_Open()
Sheet2.EnableCalculation = False
End Sub

Note that I use the worksheet object name, not the worksheeet name or number, which might be Sheets("Sheeet2") or Sheets(1).  This allows me to change the worksheet name or position without having to make coordinated changes in Workbook_Open.

For completeness, I also create the following macros, which I can select and run by pressing alt+F8 in Excel:

Sub disableCalc()
ActiveSheet.EnableCalculation = False
showCalc
End Sub

Sub enableCalc()
ActiveSheet.EnableCalculation = True
showCalc
End Sub

------

To experiment with the subtleties of EnableCalculation, you might create the following VBA function:

Function cntCalc(x As Double) As Long
Static cnt As Long
cnt = cnt + 1
cntCalc = cnt
End Function

Then put the following into Sheet1:

A1: =cntCalc(RAND())

A2: =Sheet2!A1+0.1

A3: =cntCalc(0)

And put the following into Sheet2:

A1: =cntCalc(RAND())

A2: =Sheet1!A1+0.1

A3: =cntCalc(0)

You might also create the following Worksheet_Calculate macro for both Sheet1 and Sheet2:

Private Sub Worksheet_Calculate()
MsgBox Me.Name & ": Calc"
End Sub

Activate Sheet2 and run disableCalc.  Then notice what is and is not recalculated when you press F9 and when you run doCalc while Sheet2 is activated.

amit tandon say:

To recalculate all dependent formulas every time you make a change to a value, formula, or name, in the Calculation options section, under Workbook Calculation, click Automatic. This is the default calculation setting.

To recalculate all dependent formulas — except data tables — every time you make a change to a value, formula, or name, in the Calculation options section, under Workbook Calculation, click Automatic except for data tables.

To turn off automatic recalculation and recalculate open workbooks only when you explicitly do so (by pressing F9), in the Calculation options section, under Workbook Calculation, click Manual.

When you click Manual, Excel automatically selects the Recalculate workbook before saving check box.

To manually recalculate the active worksheet and any charts and chart sheets linked to this worksheet, on the Formulas tab, in the Calculation group, click the Calculate Sheet button.

When you manually set EnableCalculation to True again, the worksheet will automatically be recalculated.

The last 2 statements might be most relevant for your query.
