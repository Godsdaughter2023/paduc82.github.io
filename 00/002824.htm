Hi GalinKalamazoo,

 

The behavior you are seeing is by design when creating web forms. If you want to adjust the size of the form, you need to adjust the size of the controls on the form or you can add or adjust the size of the empty cell controls on the form. To add these empty cell controls, you would open your form in layout view and then on the Arrange tab of the ribbon, you will see a place to insert Rows and Columns.

 

There most likely isn’t an option to just resize the web form itself because the web form is created in a state to be web friendly. When you open your web application, these forms are going to resize themselves automatically based upon the size of the IE window and screen resolution.

 

If you want to see the form in a maximized state, you could either use tabbed windows within your client application or you could create client forms in addition to your web forms. If you create client forms, this will give you the ability to size them however you want and will also give you the ability to make your client side application look a bit different then your Web app. In addition to this, within your Access Options for the database, you can choose which form should be your startup form on the web and which form should be your startup form when running the database from a client machine. The article below should provide some additional information on how to help you design your application for the web.

i_nic say: OK.. WiFi... I recommended against this as for me personally, I feel it would be easier to connect on a USB cable. If you find it easier to use WiFi, then I guess there's no reason why you shouldn't. It was really just my preference, as I prefer not to worry about switching the WLAN on and off for this purpose. The other disadvantage of using WiFi is that if you're going to buy an SMB tool, then there's a cost, and a potential issue by using a 3rd party application. Assuming you're happy to buy the tool, and it works well, then great! It sounds like it's the solution for you. :)

However... (just thought)... Some people have reported issues with WiFi connectivity. I don't think this will cause you a problem, but if it does - there are plenty of posts on here which give some excellent advice for how to solve the issues. The only real issue I've seen reported is that some people have had to adjust their "advanced settings" to remove power saving options due to persistent drop-outs of connectivity. If you do experience this kind of issue, then just search for WLAN/WiFi connectivity issues, and I'm sure you'll find some answers - or let me know and I'll see what I can do to help! :)

With regards to battery life - I am not aware of any stats about the impact of the various connectivity features on the battery. What I can tell you is that the standard battery life figures (standby and talktime) are the poorest I've seen for a mobile phone. These are the Nokia published figures, which you can find in this site.

Any radio device will use a vast amount more power when transmitting than receiving, (hence the standby and talktime figures being so different for every phone). Therefore, it stands to reason that the battery will be drained more quickly (don't know how much more quickly), when using any radio frequency transmission feature - i.e. bluetooth, WLAN, 3G, GPRS, GSM etc.

When the phone is "idle", it will still transmit to the cellular network occasionally, but for the most part it is simply in receive mode and therefore using less power. Bluetooth is more-or-less the same (or it should be) when not actually running an active connection, and WLAN/WiFi will also be similar (although I believe it will "pole" more frequently - not 100% certain of this).

Don't get me wrong - I love the phone... but for the very first time, I am planning to buy a spare battery for emergencies. I rarely run out of power on my phones, but I have done it twice already on this one, and I've only had it since April 2nd!

As for time-outs on any or all of the radio functions - I don't believe there is such a feature - but I think it's a great idea, certainly for bluetooth (infrared does this), and I know that some Motorola models have this option. I'm surprised I've never seen it on a Nokia.

saif yaseen say:

I have found the solution, basically the files are being stored in a TEMP folder and you access it doing the following :- 

Step 1: Locate the folder

The folder location is stored in the registry in the following key;

Outlook 97 HKEY_CURRENT_USER\Software\Microsoft\Office\8.0\Outlook\Security
Outlook 98 HKEY_CURRENT_USER\Software\Microsoft\Office\8.5\Outlook\Security
Outlook 2000 HKEY_CURRENT_USER\Software\Microsoft\Office\9.0\Outlook\Security
Outlook 2002/XP HKEY_CURRENT_USER\Software\Microsoft\Office\10.0\Outlook\Security
Outlook 2003 HKEY_CURRENT_USER\Software\Microsoft\Office\11.0\Outlook\Security
Outlook 2007 HKEY_CURRENT_USER\Software\Microsoft\Office\12.0\Outlook\Security
Outlook 2010 HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Outlook\Security
Outlook 2013 HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Outlook\Security
Outlook 2016 HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Outlook\Security

Step 2: Get to the folder

  1. Open the OutlookSecureTempFolder registry key from the location provided in Step 1.
  2. Copy the path from the key.
  3. Open Explorer.
  4. Paste the address in the Address Bar and press Enter.
