brahm o.dorst say:

What does "delete" mean in advanced special permissions?

The definitions I have found so far are circular, for example:

"Delete: Allows or denies deleting the file or folder. If you do not have Delete permission on a file or folder, you can still delete it if you have been granted Delete Subfolders and Files on the parent folder."

I could guess that "delete" allows or denies deleting files or folders.  What does "delete" mean?

Does over-writing a file count as deleting?

Does renaming a file count as deleting?

Does moving a file count as deleting?

Does changing the contents of a file count as deleting?

Does saving a file after making changes count as deleting the old file?

a. user say:

Hi, Brahm,

I am an Independent Advisor.

1. Deleting a file means it is removed completely from the folder to Recycle Bin. This ensures the file can be recovered if necessary until the Recycle Bin is emptied.

NOTE: If you delete a file or folder from a resource that is not local (such as a network computer), the file or folder does not go to the Recycle Bin, the item is instantly deleted.

2. If you press Shift when deleting the file, it will bypass Recycle Bin and be removed completely from the computer.

3. Does over-writing a file count as deleting? No
Does renaming a file count as deleting? No
Does moving a file count as deleting? No
Does changing the contents of a file count as deleting? No
Does saving a file after making changes count as deleting the old file? No
