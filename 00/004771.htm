As you have no doubt realized, in order to get both the paragraph number and the paragraph text, you need two StyleRef fields. The one to get the paragraph number (only) should include both \n and \t switches:

{ STYLEREF  "Heading 2" \n \t }

If you are using Insert | Text | Quick Parts | Field to insert the field, you need to check the boxes for "Insert paragraph number" and "Suppress all non-delimiter chars."

Since you want the number to follow the chapter title, you will insert that StyleRef field following the one with no switches.

rob koch say:

In addition to what Jose has already mentioned, the most common reason that a computer shuts down during a stressful activity such as an antivirus full scan is simply overheating.

If the fan(s) are running at high speed during these latter parts of the scan or worse yet, not running at all, it's quite possible this is the true problem.

In that case, I'd recommend using the crevice attachment of a vacuum to suck air out through the (typically side) vents of the laptop, trying to draw air through this area as quickly as possible in order to remove any dust from the internal fan mounted on the processor.

There are programs available which can allow you to verify this processor overheating as well, but since these do nothing to actually solve the problem itself, these should be reserved as a last resort to confirm that this is the problem if all else has failed.

In many cases simply cleaning out an older laptop or PC vents or fans is enough to resolve the problem, insuring no shutdown issues not only for antivirus scans, but any other high utilization applications as well.

These overheating issues are actually far more common with modern antivirus than is commonly recognized, since unlike in the past, they often use far more processor than disk due to both application efficiency and the more common use of SSD drives which cause very little delay.

If the problem continues even after cleaning and the use of a processor heat monitoring application confirms it's actually overheating, it may be necessary to have the laptop serviced, including potentially reconfiguring any processor control settings within the system BIOS.  Or in certain cases, using one of the more capable processor monitoring programs to actually configure processor throttling in order to workaround the problem.

Note that a properly designed and operating modern laptop should automatically throttle its mobile processors in order to avoid the need to shutdown, even if the fan(s) aren't operating optimally.  Only when these are either highly inefficient or not working at all should a shutdown need to occur, so this is never the fault of the application triggering the failure, these are simply creating the high utilization that exposes the weakness in either design or operation of the cooling system.
