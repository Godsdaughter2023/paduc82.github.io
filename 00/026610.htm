rob koch say:

Malware scanning and most especially emulation processes are typically linear dependent functions, so multi-core processing for these is rarely, if ever done.

However, the real problem here isn't with the antimalware, it's an unsigned installer file and based on this, most likely additional unsigned files contained within as well as other non-standard methods of packing or installing.  These slacker methods were once as common in the Windows world as in Linux/Unix, but have evolved much more quickly in the last few years to be unacceptable for major Windows applications.

A relatively small application using few, mostly common signed DLLs from Microsoft or other major vendors will typically have many signed components and so be recognized or relatively quickly scanned or emulated by current antimalware products.  Any of the commercial vendors who wish to sell their products widely have been signing them for at least several years now.

So the problem is with the unsigned Git for Windows installation, which since Microsoft doesn't provide whitelisted bypass for anyone any longer, will need to "git with the times" if they want to avoid this and other potential issues related to not following current best practices.

Rob
