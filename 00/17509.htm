ken sheridan say: You can call the CDate function to return a value of date/time data type from the string expression, e.g. in the debug window:

strDate = Format(VBA.Date,"Long Date")
? strDate
15 December 2017
? CDate(strDate)
15/12/2017
? Month(CDate(strDate))
 12  

So when querying the result table of the first query you can call the function in the second query's WHERE clause to restrict it by date.

dhookom say: You don't want a string.

SELECT DISTINCTROW DateValue([Date]) AS [Date By Day], 
Avg([Stage Ft NAVD]) AS [Avg Of Stage Ft NAVD], 
Min([Stage Ft NAVD]) AS [Min Of Stage Ft NAVD], 
Max([Stage Ft NAVD]) AS [Max Of Stage Ft NAVD],

FROM [OLD 15 Min Data]
GROUP BY DateValue([Date])
ORDER BY DateValue([Date]);

ken sheridan say: What I posted was an example of the use of the CDate function in the debug (aka 'immediate') window, not something to be inserted into the query.  To call the function it in your query, the SQL statement would be:

SELECT DISTINCTROW CDate(Format$([MRS Hourly].[Date],'Long Date'))AS [Date By Day], AVG([MRS Hourly].[Flow (cfs)]) AS [Avg Of Flow (cfs)],
COUNT(*) AS [Count Of MRS Hourly]
FROM [MRS Hourly]
GROUP BY CDate(Format$([MRS Hourly].[Date],'Long Date'));

But:

1.  The DISTINCTROW predicate serves no purpose.  It was originally introduced in early versions of Access to cater for the lack of support of subqueries, and is now pretty much redundant.

2.  By calling the CDate function the query would return the date in the default short date format, as my example demonstrated, so calling the Format function would be pointless.

3.  Why format the values in the queries at all?  If you want to present the dates in a specific format, do so in the output medium for the query, which should be a form or report.  There is no place for presenting the result table of a query in raw datasheet view in any developed application worth its salt.  If the date/time values can include non-zero times of day and you want to return the date values only, then call the DateValue function as GustavBrock suggests.
