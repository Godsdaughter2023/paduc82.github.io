herb tyson say:

What format is the file in, and how are you trying to edit it?

If you have already associated it with a merge, then you need to use the Edit Recipient List tool in the Mailings ribbon tab.

It's "acting" as if the file is already open... which is what would happen if you try to edit a data source already associated with an open merge operation. This presumes that you're trying to open/edit the file using something other than the Edit Recipient List.

Another possibility is that you do not have write access to the folder where the data file is stored. Unfortunately, I've seen this happen fairly often. If this is the issue, then navigate to the folder using Windows Explorer, and give yourself full access to the folder.

If one of these isn't the problem, then additional description of what you're doing might help get to the problem more directly.
Herb Tyson Microsoft Word MVP Herb Tyson Word Bible Blog

doug robbins - mvp office apps & services (word) say:
As a work around, use the data source for a Directory type mail merge in the main document of which you insert the mergefields into the cells of a one row table and have nothing else in the document.  When you execute that merge to a new document, that document will contain a table with a row of data for each record in the data source.  Insert a new row into the top of that table into the cells of which you insert the field names (not the mergefields themselves, but their names) and then add new rows with the new records to that table.  Then save that table and use it as your data source.
