j$b say:

I had overwritten XP on my older Dell laptop with a full install of Linux Ubuntu/Mint occupying the entire hard drive.  I made a DVD for the W10tp and it would not install.  The problem the installer identified is the Linux hard drive format is incompatible with Windows.

I knew from previous experience that the XP installer disc that came with the PC would overwrite and reformat the drive, but the "format" option on the W10 installer was greyed out.  My solution was to use the XP disk to reformat the hard drive and then the W10 "format" option was lit up and usable.  From there it was a very normal install of W10.  I didn't try the "Repair" link on the W10 installer because my hard drive had no Windows at all to repair - but it might work for you since you "should have" a working Windows partition, if the installer can find it.

(Note that when you installed Linux for dual boot the Windows boot loader MBR was replaced by GRUB - a Linux loader - and sometimes GRUB doesn't find Windows.  There is simple Linux software called "Boot Repair" that will fix GRUB and include a choice for Windows in the boot list.  That said, Linux distros really should caution Windows users about this, since GRUB is a pain to get rid of once it's there.)
