I think you are near the dead end with so much extensive formulas. IMHO if your file grows it becomes unusable.

In the current file, the best way for you to get the raw data into the Level sheets is to use an Array and a Dictionary to collect the need data only. That is much faster, read the comments into the code below, I've done some timings to give you a feeling what is fast and what is slow.

The code below fills only the sheets "Level 1" "Level 2" "Level 3", to keep the things as easy as possible for you.

IMHO it makes no sense to develop the code further, the basic issue in your file are all those extensive formulas.

much faster to collect the needed informations directly with VBA and flush them as values into the sheets.

consideration.

Option Explicit

Sub LevelOne()
  LevelPrim 1
End Sub
Sub LevelTwo()
  LevelPrim 2
End Sub
Sub LevelThree()
  LevelPrim 3
End Sub

Sub LevelPrim(ByVal Level As Integer)
  Dim Dict As Object 'Scripting.Dictionary
  Dim Key As String, Digit As String
  Dim Data
  Dim i As Long, k As Long, CL As Long
  Dim R As Range
  Dim SaveCalculation

'!!AK-20.12.15 begin
  Dim Organization As Object 'Scripting.Dictionary
  Set Organization = CreateObject("Scripting.Dictionary")
  Organization.CompareMode = vbTextCompare

  'Refer to the data cells of the table contaning the short names
  Set R = Worksheets("VLOOKUP").ListObjects("Table7").DataBodyRange
  'Read in all values
  Data = R.Value
  'In each row
  For i = 1 To UBound(Data)
    'Get the long name
    Key = Data(i, 1)
    'Already added?
    If Not Organization.Exists(Key) Then
      'No, associate the short name
      Organization.Add Key, Data(i, 2)
    End If
  Next
'!!AK-20.12.15 end

  'Read in all values into a two dimensional array
  Data = Worksheets("Raw Data").UsedRange.Value

  'Create a dictionary to compare the rows
  Set Dict = CreateObject("Scripting.Dictionary")
  Dict.CompareMode = vbTextCompare

  'This variable is the row marker for the last data row
  k = 0
  'In each row
  For i = 1 To UBound(Data)
    'Parse the Certification Level number from column 9
    Digit = Right$(Data(i, 9), 1)
    If Not IsNumeric(Digit) Then GoTo Skip
    'Match?
    If Digit <> Level Then GoTo Skip

    'Build the key from 'Organization Description', 'Last Name' and 'First Name'
    Key = Data(i, 4) & vbNullChar & Data(i, 5) & vbNullChar & Data(i, 6)
    'Already found?
    If Dict.Exists(Key) Then GoTo Skip
    Dict.Add Key, i

    'Copy the values to the top left to match the destination column order:
    k = k + 1
    Data(k, 1) = UCase(Data(i, 5)) 'Last Name'
    Data(k, 2) = UCase(Data(i, 6)) 'First Name'
'!!AK-20.12.15 begin
    'Do we have a short name?
    If Organization.Exists(Data(i, 4)) Then
      'Store the short name
      Data(k, 3) = Organization.Item(Data(i, 4))
    Else
      'Store the name as is
      Data(k, 3) = Data(i, 4) 'Organization Description'
    End If
'!!AK-20.12.15 end
Skip:
  Next

  'At this point we have all unique items in data in the left 3 columns till 'k' rows

  With Worksheets("Level " & Level)
    'Access the data cells
    Set R = .UsedRange.Offset(1).Resize(, 3)

    SaveCalculation = Application.Calculation
    Application.Calculation = xlCalculationManual
    Application.ScreenUpdating = False

    'The runtime till this point is around 0.008 seconds
    'The next step is slower, because of the use of a table:

    'Clear previous results
    R.ClearContents
    'Are there any new results?
    If k = 0 Then GoTo ExitPoint
    'Yes, flush into the sheet
    R.Resize(k, 3).Value = Data
    'Sort
    Set R = .Range("A1").CurrentRegion
    With R.ListObject.Sort
      With .SortFields
        .Clear
        .Add R.Range("C1")
        .Add R.Range("A1")
        .Add R.Range("B1")
      End With
      .Header = xlYes
      .Apply
    End With
ExitPoint:
    'The runtime till this point is around 0.04 seconds
    'But anything that follows needs a lot of time, because of the many slow formulas:
    Application.Calculation = SaveCalculation
  End With
End Sub

Processing time is my topic. The bottle neck in your code (and in most of your formulas) is that you evaluate the same data again and again and again.
I made a test before and tried to use an Autofilter etc. same as your code, but without using SELECT. That is possible, also faster, but not really a step forward.

The VLOOKUP is a good example for that, of course we can do that. When you search the web for such things you'll find codes that use e.g. WorksheetFunction.VLookup and access the table to get the names.

But IMHO that makes no sense, because your raw data has 2106 rows * 21 rows in the VLOOKUP tab = 42.120 operations! Possible but time consuming. VLOOKUP itself is fast, but when you need it often other techniques becomes faster.

That is what a dictionary can do for you, I don't know of a faster way to have a "vlookup-table" in memory. You can store 2 things in a dictionary:

The a) key is a unique string that is used to identify the b) data that is associated with the key.

So we have to build a dictionary from Table7 (VLOOKUP!A2:B21) and we have to call the dictionary 2106 times and get the short name, means 21 + 2106 = 2127 operations.

An idea for the future: Why not replace the 'Organization Description' in the raw data with the user friendly name once? No operations anymore!

  5. Download and install the latest version of iCloud from the Apple website, then choose the Repair option.
  6. Reboot when being prompted to do so by the iCloud installer.
  7. From the iCloud Control Panel, select the Mail, Contacts, Calendars, and Task with Outlook option. This will reconfigure iCloud with Outlook and immediately start the sync process.
  8. Start Outlook and verify that your iCloud data are available again in Outlook.

hansv mvp say:

The advantage of a macro is that - once you've written it - it's fast, and still works if your table has more rows or columns.

As I mentioned, it's possible to use formulas:

On Sheet2, enter the column headings (field names in A1:C1.

In B2: =INDEX(Sheet1!$A$2:$A$6,INT((ROW()-2)/3)+1)

In C2: =INDEX(Sheet1!$B$1:$D$1,MOD(ROW()-2,3)+1)

In D2: =INDEX(Sheet1!$B$2:$D$6,INT((ROW()-2)/3)+1,MOD(ROW()-2,3)+1)

dansantee say:

I was able to correct this by removing Groove using PowerShell and then reinstalling it from the store.

For anyone who's not familiar with this process, run PowerShell as an administrator. For me, the easiest way is to type "powershell" (without the quotes) into Cortana. You may need to close and re-open the Store in order to get it to show that Groove is available to install.
