marshall barton say:

When you open a dialog form, the code following the OpenForm line is paused until the opened form closes or is made invisible.  That means your line to make the combo box invisible didn't run until the employee detail form was closed.

To make it invisible when the form is opened, add the OpenArgs argument to the OpenForm line:

DoCmd.OpenForm "frmEmployeeDetail", , WhereCondition:="lngEmployeeID =" & Me.lngEmployeeID, WindowMode:=acDialog, OpenArgs:= "Hide"

Then use code in the employee details form's Open event procedure to make the combo box invisible:

If Me.OpenArgs = "Hide" Then

   Me.cboSelectName.Visible = False

End If

On a separate issue, you should NOT use the Form_frmEmployeeDetail syntax to refer to an open form.  That syntax refers to the Class associated with the form.  Since there can be more than one instance of the form, you may be operating on the wrong instance of the form and never figure out why it didn't work.  Another reason is it will mess with the form's class even when the form is closed.  Your line of code should have used the Forms!frmEmployeeDetail syntax so you'd get an error telling you the form was not open.

patc---01 say: For some incomprehensible reason, NMM transfers music files to Phone Memory not Card Memory by default. To change this:

- connect the phone via PC Suite
- select Transfer Music
- in Nokia Music Manager click on Options
- click on the right hand tab, named after your phone model
- in the top box, click on the Browse button
- select a directory on the card, usually:

[card]:\Sounds\Digital

(sometimes these names are expanded).

Your transfers will now be directed to card memory.

Alternatively, don't use NMM at all (it's a pain in the proverbial), instead, in ascending order of transfer speed:

- connect in PC Suite mode and use File manager to drag and drop files to the above directory

- connect in Data Transfer mode and drag and drop files to the card which will appear as an external drive in Windows Explorer

- take the card out of the phone and connect it to the PC using a card reader then use Windows Explorer to drag and drop files.

Note that if you use any of these options, you will need to refresh your music library afterwards. In Music player:

Options .. Music library .. Options .. Update Music library

bbraune say:

After over a month of the live tiles not updating on my 920 I finally got them working again.  Here is what I did:

 

I had 72 apps listed under "advanced" Background tasks.  I deleted 7 apps that I have never used.  Five of those apps do have live tiles on my start menu that I wanted to continue to be updated.  For the remaining 60 apps, I opened each app one at a time to clear out any initial alert messages.  I then went into the settings of each app to disable push notifications, live tile updates, and/or lock screen control.  (I left the location service enabled if available) None of these 60 apps were pinned to the start menu, but one or more of those settings were turned on by default.  Once I hit the magic sweet spot, my five live tiles starting updating again.  No phone restart/reboot was necessary.  My Weather Channel live tile that wouldn't update unless I opened the app is now updating again and I haven't opened the app since.

 

It's like I have a brand new phone.

 

I hope this information helps someone else with the same issue.
