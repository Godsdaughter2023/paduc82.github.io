stefan blom say:

You can certainly use Normal.dotm as the basis for a custom template, if that's what you're asking. What I'm advising against is overwriting Normal.

stefan blom say:

1. Normal.dot(m) is a template where users save their own customizations. It's best to let them do that and use a specific template for specific needs. Adding styles to Normal shouldn't do any harm, though.

2. You can basically choose how to create templates. Sometimes, one wants to create a new template based on an existing document, for example. Basing a template on a new file will be safer, because (at least in theory) it reduces the risk of document corruption.

3. If you plan to save in *.doc format, you might as well use *.dot templates. That way, you won't risk getting anything into the template that isn't supported in Word 2003 (and earlier).
