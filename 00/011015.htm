joeu2004 say: Judson wrote:

Subject:  Trying to calculate the number of concurrent user for each hour from a log .csv file.
I am trying to calculate the number of concurrent user for each hour. I have login date/time column A[.]  And logout date/time Column B


Download the example Excel file "log summary.xlsx" (click here) [1].

The "data" worksheet is imported from a CSV file per the specifications above.

The "summary" worksheet contains the required formulas.

I suspect this can be done using a pivot table.  But I am not a pivot table person.  So I created the summary formulas manually as follows.

The following formulas are array-entered; that is, press
D1:  =MAX(--TEXT(INT(data!A1:INDEX(data!B:B,E1))
          +TIME(HOUR(data!A1:INDEX(data!B:B,E1)),0,0)
          +TIME(1,0,0),"m/dd/yyyy hh:mm"))
E1:  =MATCH(2,1/ISNUMBER(data!A1:A100000))

E1 is the row number of the last login/logout pair in the "data" worksheet.  (See footnote [2] regarding data!A1:A100000 v. data!A:A.)

D1 is the last+1 hour represented in data!A:B.  (See footnote [3] regarding INDEX v. OFFSET.).  We cannot assume that the last time in column B is the latest logout time.

The expression --TEXT(...) is necessary to convert computed time to the exact internal representation of the equivalent constant in order to ensure proper comparisons below, in deference to arithmetic anomalies that cause infinitesimal differences.  The double-negative (--) converts text to numeric time.

Column A is a list of the hours represented by data!A:B.

A1:  =MIN(INT(data!A1:INDEX(data!A:A,E1))+TIME(HOUR(data!A1:INDEX(data!A:A,E1)),0,0))
A2:  =IF(OR(A1="",A1=$D$1),"",--TEXT(A1+TIME(1,0,0),"m/dd/yyyy hh:mm"))

Copy A2 down until you see a blank cell.  Alternatively, simply copy A2 down through at least as many rows as there are in worksheet "data".

The formula in A1 is array-entered (press
Alternatively, if data!A1 is always the earliest login time, use the following normally-entered formula in A1 (just press Enter as usual):

A1:  =INT(data!A1)+TIME(HOUR(data!A1),0,0)

Column B is the count of concurrent logins during the corresponding hour in column A.

B1:  =IF(A2="","",COUNTIFS(data!$A$1:INDEX(data!A:A,$E$1),"<"&A2,
          data!$B$1:INDEX(data!B:B,$E$1),">="&A1))

Copy B1 down until you see a blank cell or at least as many rows as you have formulas in column A.

A login is counted during the hour in A1, for example, if login time in data!A:A is before the next hour in A2 and the logout time in data!B:B is on or after the hour in A1.

[2] In summary!E1, it is better to use data!A1:A100000 instead of data!A:A.  The latter causes Excel to create an array of 1+ million entries.  We rarely have that much data.  If you might have more than 100,000 login/logout pairs, change A1:A100000 accordingly.  Exception:  It is okay to write INDEX(data!A:A,...) because Excel does not build an array for the first parameter.


[3] I use range references of the form data!A1:INDEX(data!A:A,E1) instead OFFSET(data!A1,E1-1,0).  Although the OFFSET expression is easier to write and read, it is a "volatile" expression.  Consequently, any formula with OFFSET and any dependent cells (directly or indirectly) are recalculated every time any cell in any worksheet in the workbook is modified.  In this example, that might slow editing significantly due to the number of COUNTIFS formulas, each of which is time-consuming.  INDEX is not a "volatile" function.

Why and how to setup My Family on Windows Phone (8 and 8.1) - Microsoft Community

denislepoivre say:

Why and how to setup My Family on Windows Phone (8 and 8.1)

Technical Level : Basic

Summary

When you first setup your Windows Phone, it's possible that you get a "My Family Setup needed" upon trying to download apps from the Store.

This will happen if you're not an adult, since the phone will use the date of birth you provided when you setup your Microsoft Account.

If however you are an adult and you still get this message, that means there was a mistake in the date of birth you entered upon creating your account.

In most cases, this cannot be corrected. If it can, you will have the opportunity to change that by following the steps below.

Either way, you will need to setup My Family in order to unlock app downloads on your Windows Phone. Hopefully, this article will help you get around these steps in an easy and straightforward way


Details

How to setup My Family in a few easy steps

1) Identify the Child Account

The account detected as a "child" is the Microsoft Account of your phone. You can check with email account is the primary Microsoft Account on your phone by navigating to:

Settings > Email+Accounts > Microsoft Account. The date of birth on this account is that of a child.

2)a Change the date of birth (if it's wrong)

If you're an adult and the date of birth is wrong, you can try to change it here:

https://account.live.com/SummaryPage.aspx

Once you've saved the changes, try downloading an app again.

This won't work in all cases. If it doesn't, don't panick and refer to part 2)b

2)b Create/Use a Parent Account

You need to designate an account to be the parent in My Family (it has to be a different account). Based on the same logic as for the child account, you need to setup the date of birth accordingly.

Connect to the designated parent account on https://account.live.com/SummaryPage.aspx to check the date of birth.

If you need to create an account to be used as the Parent, go to: https://signup.live.com/

3) Add the Child in My Family with the Parent Account

Last step in the My Family Setup, connect or stay connected to the Parent account on this page:

https://www.windowsphone.com/en-us/family?signin=true

Click on"Get Started" then on "OK". A login page will prompt you to enter the child account and password. Do that to add the Child to My Family

Once that's done, simply accept the terms and conditions on behalf on the Child. You can do that on the same page you're already on.

Congratulations! You've succeeded in setting up "My Family" for Windows Phone

I clicked Yes to continue and then recreated the email account with the same settings associating the original .pst file that was with the first email. I tested it and it worked. Basically the cache needed to be cleared out from the old Outlook configuration. This problem will probably only come up with email servers that need to be manually setup. Ours is Bluehost.
