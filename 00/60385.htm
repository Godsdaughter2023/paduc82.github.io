Indexing has nothing to do with it.  And if you want your system to reward you with increased performance, you should consider disabling the Indexing Service on your system if it is not already and uninstalling the troublesome Windows Desktop Search if you are using it (unless somebody can tell us a good reason why not to do so).

When you type in a command, XP first looks in the Registry here under the 'App Paths' key to see if it can find it there:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\App Paths

If it is not listed there in the Registry, then XP uses your PATH variable to try to find it (it does not look in the PATH variable first).

The msconfig.exe is usually located here assuming XP is installed in the Windows folder on your C drive:

C:\WINDOWS\pchealth\helpctr\binaries

The msinfo32.exe (not misinfo) is usually located here assuming XP is installed in the Windows folder on your C drive:

C:\Program Files\Common Files\Microsoft Shared\MSInfo

If you were to look in your Registry under 'App Paths' for those two programs, that is where those two should be pointing.  If things are set up properly, neither of those folders should be or need to be in your PATH statement.

Why don't you take a look and see if those files are where they are supposed to be and if not, search your system for the files and copy/paste them in the right place and then see how things work.  Both should work all of the time and not just some of the time.

c:\windows\system32\dllcache

If that doesn't work and you don't feel like figuring it out, a workaround would be when you find the two files, copy/paste them into this folder which should certainly be in your PATH:

c:\window\system32
