bob jones aka: cybertaz say:
You're correct that Mac Word doesn't support embedding fonts. The document file contains metadata that indicates what fonts are used. When the file is opened on any system the program being used [presumably, but not necessarily, Word] attempts to use the specified fonts. If they are not available on that system Word selects a substitute that matches as closely as possible in order to display the document. The selection is based on analysis of the font metrics, not simply the name of the fonts, but I don't claim to fully understand the process :-)

Knowing nothing about the design or type of the "custom fonts" you're using my best guess is that it is an issue on the recipient's end. However, it could be that one version of the fonts were created to be compatible with the Windows OS, a separate version for Mac, and the two do not cross-match. I've seen situations where standard fonts of the same name were available but substitution occurred because the version of the font differed on the systems involved.

My suggestion would be to contact the designer/foundry for suggestions on resolving the issue.
