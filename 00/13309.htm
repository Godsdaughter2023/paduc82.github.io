myka---01 say: Hi,

The thing is that Lifeblog creates a diary of your life by means of snapshots. Snapshots consist of personal Multimedia objects that come into existence on your N70, such as images, videos, text messages, and Multimedia messages and linked context information. This linking between objects and context is only done at exactly the point in time when the object is created. Any subsequent changes of the linked information are not tracked anymore.

Thus I am afraid, uninstallation and reinstallation would not help.

-Andreas

cameronpaterson say: Hi

Thanks for the reply, but are you sure? Surely Nokia Lifeblog (of which I am a big fan) works by creating a linked database of the messaging and multimedia content stored on the phone, so uninstalling and reinstalling will force it recreate that database...

myka---01 say: I still would recommend not to mess with the database but if you want to have a more complete story, here it comes. ;-)

Uninstallation itself does not affect the Lifeblog database - neither on the N70 nor on the PC.

If you manually delete the Lifeblog database from the N70, you will remove all your metadata from the N70. If you reinstall later, you can get the multimedia data and messages themselves back. But only the contact-related metadata for messages will be recreated (based on the current contact db) - nothing else. That is, all other metadata -whether created manually or automatically- will be lost unless you have already transferred it to PC.

Thus, a potential strategy -just for getting the info from the contact db- could be:
1) Sync Lifeblog 1.7 on N70 with your PC Lifeblog 1.7
2) Uninstall Lifeblog 1.7 from N70
3) Remove Lifeblog db from N70
4) Install Lifeblog 1.7 on N70
5) Reimport data

Important: all items coming from the reimport will be treated as new items. Hence, if you sync again with the PC, you will get these as duplicates.

I would like to repeat that I do not recommend this approach, because the side effects might be more annoying than the symptoms that you wanted to cure.

Is there a way of making the 'blank' cells actually blank?

hansv mvp say:

The "empty" cells contain an empty string, which Excel doesn't see as the same as a real blank.

 

You could run the following macro:

 

Sub ClearBlanks()
  Dim oCell As Range
  On Error GoTo ExitHandler
  Application.ScreenUpdating = False
  For Each oCell In Selection.SpecialCells(xlCellTypeConstants)
    If Trim(Replace(oCell.Value, Chr(160), " ")) = "" Then
      oCell.ClearContents
    End If
  Next oCell
ExitHandler:
  Application.ScreenUpdating = True
End Sub

 

Select the range you want to apply it to before running it.
