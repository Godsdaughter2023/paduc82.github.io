    jta49410 say:

    Although Groove and SharePoint Workspace need the groovebinaryfilestore and the other files you refer to, in order to function, a backup of these files will not allow you to restore your workspaces.

     

    You should create archives of your critical workspaces, either manually (save workspace as / archive), or by using a backup solution for Groove.

    Microsoft has a server based solution for Groove 2007 workspaces only (= Groove Server 2007 Data Bridge including "backup services").

    Another existing solution for Groove 2007 and SharePoint Workspace 2010 is GrooveIT! Backup http://solutions.hommesetprocess.com/grooveit/grooveit-backup-enterprise

     

    Beside the use of a backup solution for Groove workspaces, another best practice is to restrict permissions for each role.    You could remove the ability to delete files/entries and to delete tools.  This prevents you agains "accidental deletions".
