runningmanhd say:

Greg,

Referring back to your last question, I provided you with a database example demonstrating how to model your data.  I don't guess that you ever looked at it because you never posted any type of response. 

As far as your Excel spreadsheet goes, you most certainly need to normalize the data.  Normalization means breaking the data up into respective entity groupings (tables) with associated attributes about each entity (fields). 

Think of a table as a subject and you are going to provide the details about that one subject.  You will then break down all of your data into subjects.  For instance, you have a client.  The client is the first subject.  You are going to store only information specifically about the client in the Client table such as name, address, phone, and age, etc..  Looking to your form, it asks for such things as References.  You will want to create a separate subject/table for the Client References.  You will relate the References with the Client by the client's record id.  So what you will have a is one-to-many relationship where you will have one unique client with many (2 or more) references.

As Scott explains, you will want to link your spreadsheet and parse out the data to the respective tables once you have designed a properly normalized system to house the data. 

However, I disagree with Scott's description of how to break your data into questions and answers.  In my opinion you need only define the appropriate subjects that your questions attempt to acquire information for and parse your data into the corresponding tables.

Hi

Depending on your Skype client, you can try deleting the conversation from the Chats/Recent tab. Read through the list below:

Skype for Windows 10 version 12.13:

  • From the Recent tab, right-click on the number you recently called.
  • Select Delete conversation.

New Skype on Desktop, Mac and Linux version 8.13:

  • From the Chats screen, right-click on the number you recently called.
  • Select Delete chat.
  • Click Delete from the confirmation window,

Note: The conversation will be deleted from the Chats screen, however it will still appear on your Calls tab. You cannot delete it from there.

Classic Skype for Windows desktop version 7.40: This version of the application currently does not have the option to delete a conversation, however as a workaround you can opt to hide it instead.

  • From the Recent tab, right-click on the number you recently called.
  • Select Hide conversation.

Hi SPO student,

The better way may be to insert the code to create a table of content and customize the classic page layout for your requirement in SharePoint Online.

Here are some codes for your reference. You can paste them in the Script Editor web part to see the outcome. If you need more requirements or need to change the page layout, I suggest you post a new thread on ourMSDN forum which is a specific channel to cope with SharePoint development related issues to see if there is a feasible way.

<script type="text/javascript" src="//code.jquery.com/jquery-1.11.0.min.js"></script>

<script type="text/javascript">

 function hideTOC() {

  document.getElementById("theTOC").style.display = 'none';

  document.getElementById("show").style.display = 'block';

  document.getElementById("hide").style.display = 'none';

 }

 function showTOC() {

   document.getElementById("theTOC").style.display = 'block';

   document.getElementById("show").style.display = 'none';

   document.getElementById("hide").style.display = 'block';

 }

</script>

<div id="wikiTOC">

 <table style="padding: 5px;">

  <tr>

   <td><div class="TOCheader">Table of Content</div></td>

   <td>

    <div class="closehide" id='hide'>[<a href='#' title='Click to hide' onClick="hideTOC()">Hide</a>]</div>

    <div class="closehide" id='show'>[<a href='#' title='Click to show' onClick="showTOC()">Show</a>]</div>

   </td>

  </tr>

 </table>

 <div id='theTOC'>

  <script type="text/javascript">

   $(document).ready(function(){

    var L1=0, L2=0, L3=0, L4=0;

    $(".ms-rtestate-field > h1, .ms-wikicontent h1, .ms-rtestate-field > h2, .ms-wikicontent h2, .ms-rtestate-field > h3, .ms-wikicontent h3, .ms-rtestate-field > h4, .ms-wikicontent h4").each(function(i){

     theLevel=$(this).get(0).tagName;

     if (theLevel=="H1") {

        L1=L1+1;

        L2=0;

        L3=0;

        L4=0;

        theLevelString=""+L1;

     }

     else if (theLevel=="H2") {

      L2=L2+1;

      L3=0;

      L4=0;

      theLevelString=""+L1+"."+L2;

     }

     else if (theLevel=="H3") {

      L3=L3+1;

      L4=0;

      theLevelString=""+L1+"."+L2+"."+L3;

     }

     else {

      L4=L4+1;

      theLevelString=""+L1+"."+L2+"."+L3+"."+L4;

     }

     $encAnchor = $(this).text().trim();

     $encAnchor = $encAnchor.replace(/\s/g, "_");

     $encAnchor = encodeURIComponent($encAnchor);

     $encAnchor = $encAnchor.replace(/%/g, "");

     $(this).attr("id", $encAnchor );

     $("#theTOC").append("<a href='#" + $encAnchor + "' class='wikiTOC-" + theLevel + "'>" + theLevelString + " " + $(this).text() + "</a><br />");

    });

   });

   showTOC();

  </script>

 </div>

</div>
