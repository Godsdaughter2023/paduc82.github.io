jeeped say:
KEY. A='s how many numbers appear only one time in multiple cells.
          B='s how many numbers appear only two times in multiple cells.
          C='s how many numbers appear only three times in multiple cells.
          D='s how many numbers that do not appear in any of the specified cells.
          E='s how many numbers appear four times or more in multiple cells. These numbers are indicated with their power. (very important)

I've written a quick routine to get you these results,

                      

Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module. Paste the followign into the new pane titled something like Book1 - Module1 (Code),

Function fcnCustomCounts(rThese As Range, sFreq As String) As Variant
    Dim c As Range, s As String, r As String, f As Integer, n As Integer
    Select Case UCase(sFreq)
    Case "A", "B", "C"
        f = Asc(UCase(sFreq)) - 64
    Case "D"
        f = 0
    Case "E"
        f = 4
    Case Else
        fcnCustomCounts = "Frequency out of range"
        Exit Function
    End Select
    For Each c In rThese
        s = s & c.Text
    Next c
    For i = 1 To 10
        If f < 4 Then
            If Len(s) - Len(Replace(s, i - 1, "")) = f Then _
                r = r & " - " & i - 1
        Else
            If Len(s) - Len(Replace(s, i - 1, "")) >= f Then _
                r = r & " - " & i - 1 & Chr(94) & Len(s) - Len(Replace(s, i - 1, ""))
        End If
    Next i
    fcnCustomCounts = Mid(r, 4, 99)
End Function

Tap Alt+Q to return to your worksheet. The syntax for your new User Defined Function (i.e. UDF) is,
=fcnCustomCounts(<range to search>,<key letter>)
... which is used (as above) in B7 as,
=fcnCustomCounts($A$1:$C$3,A7)
... or,
=fcnCustomCounts($A$1:$C$3,"A")
... then filled down.

That should be enough to get you started. Have a look at the questions posed in my earlier response.

bob jones aka: cybertaz say:

The answer is rather simple: Object Linking & Embedding [O:E] is a Windows protocol which never has been supported by the Mac operating system. The only programs which support it at all are Word & Excel and that support is limited to the object types created by those programs. In short, PDF is not a supported OLE object type on a Mac.

The simplest solution is to have the sender provide you with a separate copy of the PDF.

Also, if you have access to a Windows PC with Office you can open the PDF there & save a copy.

VBA & Internet explorer - Microsoft Community

ajeesh_nair say:

VBA & Internet explorer

Can we use VBA to 'Uncheck' IE->internet Options->Advanced & 'Enable Automatic Crash recovery*' option? I am using IE8.

 

graham mayor say:
When the option is disabled, the value is stored in a registry key

HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Recovery\AutoRecover

The value for disabled is 2. The value for enabled is 0

The stored value is read the next time IE is opened.

You can set the value of that key using VBA, but if you are making use of IE in your application you will need to open IE for that use AFTER you have set the value. It may also be a plan to read the stored value so that you can reset it to that value after you have run your process.


The following will toggle the value

'Set a reference to Microsoft Shell Controls and Automation
Dim WSHShell, RegKey, rKeyWord, wVer
Set WSHShell = CreateObject("WScript.Shell")

Start:
RegKey = "HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Recovery\"
On Error Resume Next
'The registry key does not exist
rKeyWord = WSHShell.RegRead(RegKey & "AutoRecover")
If rKeyWord = "" Then
    WSHShell.regwrite RegKey & "AutoRecover", 0, "REG_DWORD"        'set it at zero
    GoTo Start:        'and read it again
End If
If rKeyWord = 2 Then
    WSHShell.regwrite RegKey & "AutoRecover", 0, "REG_DWORD"
    MsgBox "Recovery is enabled", vbInformation, "IE Recovery"
Else
    WSHShell.regwrite RegKey & "AutoRecover", 2, "REG_DWORD"
    MsgBox "Recovery is disabled", vbInformation, "IE Recovery"
End If
