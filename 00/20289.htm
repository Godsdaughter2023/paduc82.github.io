ken sheridan say: You can probably get the data into correctly normalized tables more easily than you might think, but it would require a knowledge of how to write the update and append queries to do it, and these would need to be executed in a specific order.  I can see that might be a big ask, though I think they'd be impressed if you brought it off.

Coming back to the original question, you can base a query on each of the eight tables, and you'll get the count per value.  Let's say the first table deals with a home named Longridge ( a real one just down the road from me), you can include the home's name as a constant in the query, group by the lookup field and count the rows per group, like so:

SELECT  "Longridge " AS Home, [NameOfLookupField],
COUNT(*) AS [Number of instances]
FROM [NameOfFirstTable]
GROUP BY[NameOfLookupField];

You can do similarly for the other seven tables.  To return all the counts in one result table you can tack the eight queries together in a another query with a UNION ALL operation:

SELECT *
FROM [NameOfFirstQuery]
UNION ALL
SELECT *
FROM [NameOfSecondQuery]
UNION ALL
< and so on to>
SELECT *
FROM [NameOfEighthQuery];

However, while the individual eight queries will show the 'looked up' value in the lookup field, when you tack them all together in a UNION ALL operation they will show the numeric value which is the real value of the field.  To avoid this you should do as John described earlier and join each of the eight tables to the lookup table in the individual queries, so the first would be like this:

SELECT  "Longridge " AS Home, [NameOfTextFieldInLookupTable],
COUNT(*) AS [Number of instances]
FROM [NameOfFirstTable] INNER JOIN [NameOfLookupTable]
ON [NameOfFirstTable].[NameOfLookupField] = [NameOfLookupTable].[NameOfPrimaryKeyField]
GROUP BY[NameOfTextFieldInLookupTable];

The results of each query will look the same as before, but if you tack them all together in a UNION ALL operation you'll now see the text values rather than the numeric values.

I figured it out. Had to add value data decimal 15 (or more?) to these reg keys:

[HKEY_CURRENT_USER\SOFTWARE\Microsoft\Exchange]
"maxnumexchange"=dword:00000015

[HKEY_CURRENT_USER\SOFTWARE\Policies\Microsoft\Exchange]
"MaxNumExchange"=dword:00000015
