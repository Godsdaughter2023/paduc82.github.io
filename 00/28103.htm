Stop Name AutoCorrect crashes by deleting Log table - Microsoft Community

ricknta say:

Stop Name AutoCorrect crashes by deleting Log table

I guess I'll post this as an answer - hopefully save some other folks the hours it took me to fix this! 
We were getting weird crashes and found we had to not just turn off Name AutoCorrect, but also delete the Name AutoCorrect Log table.  Summary:

Access 2010 crashes with "Microsoft Access has stopped working" when:

 

You enter a word beginning with "n" in query grid (query Design View) in:

  1. Criteria
  2. Update To

Doesn't matter if "n" is initial letter or later - like "is null"

Examples:

  1.  "Now()"
  2.  "Is Null"
  1. "Is Not Null"
  1. "<>Null"

Happens only in query Design View (not in SQL View)

Doesn't crash if you paste in "Now()",  "Is Null" etc.

Doesn't crash if you enter text value with preceding quote - like "n

 

Fix:

  1. turn off Name AutoCorrect completely (File-Options- Current Database)
  1. delete Name AutoCorrect Log table

 

Good luck!

Do not worry, it can be fixed.

I suggest you to try the steps provided below and check if it helps.

1.     Run the Windows Store Apps troubleshooter.

2.     If Microsoft Edge only crashes after some time has passed, try deleting cached data and files. In Microsoft Edge, select the “…” icon in the top right corner and chooseSettings. Under Clear browsing data, select Choose what to clear and thencached data and files.
