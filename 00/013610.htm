jay freedman say:

In a Word document, every object (graphics and text boxes) with a wrapping style other than "In line with text" is anchored to a particular paragraph in the main text. If you go to File > Options > Display and check the box for "Object anchors", an anchor icon will appear in the left margin when an object is selected, showing the paragraph that contains the anchor.

I suspect that in your document, there is a paragraph mark with no text, at the top of the page you showed. (To see the paragraph mark, press Ctrl+Shift+8 or click the ¶ button on the Home tab.) Further, both text boxes are probably anchored in that single paragraph.

This is the cause of the problem: When two anchors are placed in the same paragraph, their order (as far as Word's programming is concerned) is not related to the objects' positions on the page, and may or may not be related to the order in which the objects were inserted.

The solution is to insert a second empty paragraph on the same page, and drag the anchor icon of Table II into that new paragraph. Then select both text boxes (or the entire document with Ctrl+A) and press F9 to update the fields.

stefan blom say:
If you must use tables in text boxes, ensure that their anchors are in different paragraphs, as Jay describes in more detail in his reply. SEQ fields will be numbered based on the relative positions on the anchors.

GIGO!  Nothing wrong with GEOMEAN.  You are just using it incorrectly in this case.

Apparently, you wrote effectively GEOMEAN(12%,17%,35%).  So GEOMEAN dutifully calculates what you told it to do, namely:  (12%*17%*35%)^(1/3) = 19.26% (approximately).

But that's not the way to calculate the geometic average of percentages.

The correct usage is:  GEOMEAN(1+12%,1+17%,1+35%)-1.  That correctly yields 20.94% (approximately).

(That can also be written GEOMEAN(1.12,1.17,1.35)-1.)

If 12%, 17% and 35% are in A1:A3, we might write the following array-entered formula (press ctrl+shift+Enter instead of just Enter):

=GEOMEAN(1+A1:A3)-1

"Why 1+12% and GEOMEAN(...)-1?", you might ask.

To understand that, look at what "inflation rate" means.  If we start with $100,000 and an inflation rate of 12%, the amount at the end of year 1 (start of year 2) is 100000*(1+12%).  At the end of year 2:  100000*(1+12%)*(1+17%).  And at the end of year 3:  100000*(1+12%)*(1+17%)*(1+35%), which is $176,904.

So we want the average rate "r" such that 100000*(1+r)^3 = 176904.  So (1+r)^3 = (1+12%)*(1+17%)*(1+35%).  And r = ((1+12%)*(1+17%)*(1+35%))^(1/3) - 1.

If you look at the GEOMEAN help page for the mathematical definition, you will see that the formula for r looks like the formula for GEOMEAN(...)-1 with the parameters 1+12%, 1+17% and 1+35%.

There are many criteria for creating DMP files.  For them to be created you need

For dumps to be produced. . .

▪ Page file must be on OS drive
*If manually set Page file base allocation size must be > than installed physical RAM
