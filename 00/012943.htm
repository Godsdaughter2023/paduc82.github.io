jeeped say:
I would first recommend that you do not attempt to open the PST file directly from any external drive. Read/write operations to/from an external drives are buffered and removing the drive without using the Safely remove hardware command will leave the buffers unflushed and render the PST file useless. This happens to DOCX files all the time. It does not matter whether you close the program (e.g. Outlook). The read/write buffers need to be flushed and terminated properly in order to preserve the integrity of the PST file. Beyond the dangers of corrupting your PST backup, the security permissions for routinely opening alternative PST files from a removable drive are prohibitive.

If you simply want to verify the validity of the PST file, run scanpst.exe (click through link below for more information) on the file. This will check integrity and recommend any repair maintenance operations that may improve it.

If you want to check old emails, use an archive.pst file for old emails. Click through the link below for more information.

suzanne s. barnhill say:

You don't actually need any section breaks for this type of layout. Here's the procedure:

  1. On the contextual Header & Footer Tools | Design tab, check the box for Different First Page.
  2. Put your desired image in the First Page Header.
  3. If the document will have any footer throughout, insert it in the First Page Footer.
  4. For the last-page footer, you will need a field in the Footer:
    { IF { PAGE } = { NUMPAGES } "Content you want on the last page" "Content (if any) that you want on the other pages" }

To insert a field, you must press Ctrl+F9 to insert the field braces and then type the content between them. A shortcut for inserting the PAGE field is Alt+Shift+P. When you construct the field, you can use placeholders for the TrueText and FalseText, then paste in the desired content (the image for the last page). When the field is complete, press F9 to update it and display the desired content. You can press Alt+F9 as needed to redisplay the field code for editing.

If only IMAP accounts are configured in the profile, then one of the IMAP .ost files would be set as the default data file for the profile.
