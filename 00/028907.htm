Firstly I see you have Option Explicit set which is good. If you select menu item Debug -> Compile VBaProject, does the code compile without error? (If the Compile VBaProject is disabled (as in greyed out) then it means it has been compiled since last code change)

We need to commence with accurate information. In your text you show the following line as a problem.

SPath=Environ("USERPROFILE") & "\Desktop" & '\OCDSRecords"

The above line has a single quote at the start of the following  '\OCDSRecords and this will make the last part of the line a comment and therefore the & sign before the single quote with no following string will cause an error. If you were attempting to save to the Desktop without the last folder then place the single quote in front of the & sign.

In the code example you provided the line of code is as follows which does not match the code in your text.

 sPath = Environ("USERPROFILE") & "\Desktop" & "\testing\01 OCDS Documents"

You can open the Immediate window (Ctrl and G) and enter the following line and see if the path is actually valid. If valid then you will see nothing but if invalid then it will return an error "Path not found".

ChDir Environ("USERPROFILE") & "\Desktop" & "\testing\01 OCDS Documents"

For the above you can type in the ChDir plus a space and then copy the path after the equals sign from your code and paste after the ChDir.

To locate an error, insert some MsgBox's in the code so you can see how far the code runs without error. See the example below. You can use more or less MsgBox's until you determine where the error is. Alternatively if you are familiar with the Immediate window you can use Debug.Print in lieu of the MsgBox's
