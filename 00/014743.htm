nigc say:

Ah, right, I think I have it now.

 

My solution (hopefully) requires a small macro and a rather long formula (similar to the one posted above by G North). Here goes.

 

I've assumed your data starts at cell D1 - you'll need to adjust the formula below to compensate if it's elsewhere.

 

Start by hitting Alt+F11 and adding a Module (right click on the left where it says VBA Project(your filename) and click Insert>Module). Then paste in this:

 

 

 

Option Explicit

Public Function LinkMe(AddressWithLink As Range) As String
    If AddressWithLink.Hyperlinks.Count > 0 Then
        LinkMe = AddressWithLink.Hyperlinks(1).Address
    Else
        LinkMe = ""
    End If
End Function

 

 

 

Then in the cell where you want your link and text to appear, put in this formula:

 

=HYPERLINK(linkme(INDIRECT(ADDRESS(MATCH($B$2,D2:D5,0)+1,MATCH($B$1,E1:G1,0)+4))),INDIRECT(ADDRESS(MATCH($B$2,D2:D5,0)+1,MATCH($B$1,E1:G1,0)+4)))

 

Here's where you need to adjust for wherever your data is. The "+1" refers to the start row (including the header) and the "+4" refers to the start column number (including the header). I placed my data starting at D1, which is why I have used +4 and +1. Each number appears twice in the formula - you must change both.

 

This will only work if you have inserted a hyperlink, not used the "=HYPERLINK(...)" formula in your data cells.

 

Hope this helps
Nigel

ken sheridan say:

 'This throws an error if more than one item is selected in the list

DLookup("IncidentID", "tblIncidentBehavior", "BehaviorID = " & strBehaviorIDList)


It will, because the value of the variable is a value list, not an integer number.  More to the point, calling the DLookup function does nothing here, so you can drop the line.

To bring the lstModifications control into play you do the same for that control and tack the two expressions together with the AND operator:

Private Sub cmdApplyFilter_Click()

    Dim varItem As Variant
    Dim strBehaviorIDList As String
    Dim strMotivationIDList As String
    Dim strCriteria As String
    Dim ctrl As Control

    Set ctrl = Me.lstBehaviors
    'loop through behavior list of all selected behaviors
    For Each varItem In ctrl.ItemsSelected
       strBehaviorIDList = strBehaviorIDList & "," & ctrl.ItemData(varItem)
    Next varItem

    strBehaviorIDList = ",0" & strBehaviorIDList
    'remove leading comma
    strBehaviorIDList = Mid(strBehaviorIDList, 2)
    strCriteria = "BehaviorID IN(SELECT BehaviorID " & _
    "FROM tblIncidentBehavior WHERE IncidentID IN(" & strBehaviorIDList & ")" & _
        "OR " & (Me.lstBehaviors.ItemsSelected.Count = 0) & ")"

    strCriteria = strCriteria & " AND "

    Set ctrl = Me.lstMotivations
    'loop through Motivation list of all selected Motivations
    For Each varItem In ctrl.ItemsSelected
        strMotivationIDList = strMotivationIDList & "," & ctrl.ItemData(varItem)
    Next varItem

    strMotivationIDList = ",0" & strMotivationIDList
    'remove leading comma
    strMotivationIDList = Mid(strMotivationIDList, 2)
    'Not sure if this part is correct
    strCriteria = strCriteria & "MotivationID IN(SELECT MotivationID " & _
        "FROM tblIncidentMotivation WHERE IncidentID IN(" & strMotivationIDList & ")" & _
        "OR " & (Me.lstMotivations.ItemsSelected.Count = 0) & ")"

     ' open frmIncidentHistory form filtered to relevant incidents
     DoCmd.OpenForm "frmIncidentHistory", WhereCondition:=strCriteria

End Sub

I've assumed in the above that the list boxes are in an unbound dialogue form from which the frmIncidentHistory form is opened.  If they are in the bound form itself, then the code would be:

    Me.Filter = strCriteria
    Me.FilterOn = True

You'll appreciate that I'm unable to debug the code.  It looks OK, but I make no guarantees!

Edited: provision made for zero length value lists.
