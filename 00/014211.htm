suzanne s. barnhill say:

In Word 2003, the box for "Equal column width" is not checked by default when you insert columns from the toolbar, and that has often caused problems. Obviously an attempt was made to solve these problems in Word 2007 and 2010, in which that box is checked by default; in those versions the distance between columns can be changed (equally) by dragging the ruler markers, but not the actual column widths (except by unchecking the box). Word 2013 further locks down this behavior so that, even after clearing the check box, nothing can be changed. I believe this is a bug, and I will report it.

 

One helpful tip: Instead of a tedious trip through Page Layout | Page Setup | Columns | More Columns, you can quickly open the Columns dialog by double-clicking on the hashed area between the column border markers.

suzanne s. barnhill say:
This has been reported to the Word team, and they have agreed that it is a bug. Meanwhile, they provide a workaround: If you uncheck “Update document content while dragging” on the General tab of Word Options, it will work as expected. You lose the live layout functionality, so it’s not a very satisfactory alternative, but, until this is fixed, perhaps it will help.

bernd p. say:

 I'm advocating for fewer notebooks, with smarter use of the sections, etc.  But, I'm getting pushback from my supervisor, as she thinks a notebook shouldn't be "too big."  But, it's all stored on the same network drive, so I don't see what the issue is. Advocating "for fewer notebooks, with smarter use of the sections" is always a good idea.

AFAIK there are only two problems with "large" notebooks:

  1. really large embedded files/documents which are modified (Onenote cannot make a differential update as for its notebooks)
  2. size limit for sections using Skydrive (storing in the cloud)

In general Onenote isn't so size dependent because it uses a differential update/synch mechanism, avoiding copying complete files.
