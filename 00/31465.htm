bill smithers say:

Hi,

HP's Recovery partition and Microsoft's System Image Backup are 2 completely different process.

As you correctly mentioned, HP's Recovery process is to make an exact copy of how your System was when made., in case you have Hard drive failure.

But the HP Recovery partition that you make the Recovery disks from is still there in case you have to do a reinstall back to Factory Settings in case of a complete System crash.

"Performing an HP system recovery (Windows 8)"

http://support.hp.com/us-en/document/c03489643

Now to System Image backup.

It is a quicker way to reinstall, as you have the Updates, your Programs are already installed and you do not have to copy your Data back.

But remember, ALWAYS keep a backup of your Data anyway to external Media, not matter what you do about System Image Backup.

It is up to you what you do about the number of times you do a System image Backup.

"What happened to Backup and Restore?"

http://windows.microsoft.com/en-us/windows-8/what-happened-to-backup-restore

Use System Image Backup

You can use System Image Backup like you did in previous verison of Windows.

To open System Image Backup

  1. Swipe in from the right edge of the screen, and then tap Search.
    (If you're using a mouse, point to the lower-right corner of the screen, move the mouse pointer up, and then click Search.)

  2. Enter File History in the search box, tap or click File History, and then click System Image Backup.

The rundll32.exe file should never be missing from the dllcache folder, and it should never be missing from the system32 folder.  It takes extra effort for those critical XP system files to be missing.

When it is missing and you try to use things like the Control Panel, you will see a message like this:

Control Panel
Windows cannot find 'C:\WINDOWS\system32\rundll32.exe'.  Make sure you typed the name correctly, and then try again. To search for a file, click the Start button, and then click Search.

On an unafllicted system, rename your system32\rundll32.exe file to something you can remember (now it is "missing") and almost before you can blink a few times, Windows File Protection will quickly and silently replace it with the backup copy it finds in the dllcache folder and put a message like this in the Event Viewer System log:

Event Type:    Information
Event Source:    Windows File Protection
Event Category:    None
Event ID:    64002
Description:
File replacement was attempted on the protected system file c:\windows\system32\rundll32.exe. This file was restored to the original version to maintain system stability. The file version of the system file is 5.1.2600.5512.

If the file is also missing from the dllcache folder, that is a good time to run sfc /scannow to replace it (if you have a genuine bootable XP installation CD that matches your installed Service Pack), or you can download the file from my SkyDrive and put a copy in the dllcache folder (the backup copy) and put also put on in the system32 folder and you will be all set.


I uploaded a copy of the XP Pro SP3 file you need to my SkyDrive (everybody has a SkyDrive for sharing files).

You can download it and when you do, put a copy of the file in these two folders (assuming Windows is installed on your C drive):

c:\windows\system32
c:\windows\system32\dllcache    (that is where XP keeps backup copies of important files).

Here is the link to my SkyDrive and you can look for the file you need there:

http://cid-6a7e789cab1d6f39.skydrive.live.com/redir.aspx?resid=6A7E789CAB1D6F39!311

When you see the files available for download, you may not see the file extension (.exe, .dll, .cpl, .sys, etc.), but when you download them they will have the correct extension.

When you download the file (especially if you use Internet Explorer), when you get a chance to save the file, it may not save the file with an extension (like .exe, .dll, .cpl, .sys) so you will have to manually add the proper extension to the file when you download the file before you save it.

You can download the file with no extension and then rename the file to add the proper extension.  You don't want to try to use a file called rundll32 when the file name really needs to be called rundll32.exe (if the downloaded file has no extension you will have to change the name of the file to add the proper extension to get it to work.

You then need to put the downloaded files in the correct folders on your system.
