rich007a say:

How about something like this:

 

Sub LoopThroughCustomers()
    Dim rng As Range, c As Range
    Dim lnLastRow As Long
    Dim strAns As String
   
    lnLastRow = Cells(Rows.Count, "B").End(xlUp).Row
    Set rng = Range("B2:B" & lnLastRow)
    For Each c In rng
        strAns = c.Offset(0, 1).Value & c.Offset(0, 2).Value
        Select Case strAns
            Case "YesYes"
                Call MyMacroA
               
            Case "YesNo"
                Call MyMacroB
               
            Case "NoYes"
                Call MyMacroC
               
            Case "NoNo"
                Call MyMacroD
               
        End Select
    Next c
End Sub

 

This assumes you have your table of Cust#, Cust Name, Q1 and Q2 in columns A, B, C and D.

It loops through each cell in your Cust Name column, concatentates the contents of Q1 and Q2 and calls the required macro based on the combination of YesYes, YesNo, NoYes or NoNo.

Note that if you need to refer to the customer name (e.g. to parse the cust name to one of your macros, you can use c.Value, e.g.:

    Call MyMacroA(c.Value)

would parse the customer name to MyMacroA so it could be used in the file name when the "values only" file is saved.

Hope that helps.

Like I said, the current customer name is available in c.Value

The loop:

    For Each c in rng
        ...
    Next c

is looping through each cell in the range Range("B2:B" & lnLastRow)

Both c and rng are Range objects.  A range is just like any other collection, so you can for example loop through each worksheet in a workbook by doing:

    Dim ws as Worksheet
    For Each ws in ActiveWorkbook.Worksheets
        Debug.Print ws.Name
    Next ws

Since a range is a collection of single cells, and a cell is also a range, a range is a collection of one or more single cell ranges (make sense?).

When in the loop, we can access the current object like this:

c.Value

So, either you could place the current customer name in a cell using:

    For Each c in rng
        Range("F1").Value = c.Value
        ...
    Next c

But you could just keep the current customer in VBA memory and parse it to each of your macros as required.

    Sub MyNewMacro()

?

    I am combining chapters into one Word doc to submit to my publisher. All endnotes should appear at the end of the document, but should retain the numbering that appears in the individual chapters (endnotes 1-90 for Chapter 1; 1-88 for Chapter Two, etc.). Any tips on how to do so? Thanks!!!

    Moved from Office / Word / Windows 10 / Office 2010

    doug robbins - mvp office apps & services (word) say:

    IF each chapter contains only one Section, and you insert each chapter into a separate Section (insert a Section Break), then after combining the documents, if you go to the access the Footnote and Endnote dialog and set the options as shown below, I think that the Endnotes will be relocated to the End of the document with their original numbering retained.

    If each chapter contains multiple Sections, then, you could run the following macro on each of the chapter documents before combining them and the manually copy and replace the "textnotes" from the end of each chapter to the end of the document (in the appropriate order)

    ' Macro created 29/09/99 by Doug Robbins to replace endnotes with textnotes at end of document

    ' to replace the endnote reference in the body of the document with a superscript number.

    '

        Dim aendnote As Endnote

        For Each aendnote In ActiveDocument.Endnotes

            ActiveDocument.Range.InsertAfter vbCr & aendnote.Index & vbTab & aendnote.Range

            aendnote.Reference.InsertBefore "a" & aendnote.Index & "a"

        Next aendnote

        For Each aendnote In ActiveDocument.Endnotes

            aendnote.Reference.Delete

        Next aendnote

        Selection.Find.ClearFormatting

        Selection.Find.Replacement.ClearFormatting

        With Selection.Find.Replacement.Font

            .Superscript = True

        End With

        With Selection.Find

            .Text = "(a)([0-9]{1,})(a)"

            .Replacement.Text = "\2"

            .Forward = True

            .Wrap = wdFindContinue

            .Format = True

            .MatchWildcards = True

        End With

        Selection.Find.Execute Replace:=wdReplaceAll
