The filename in your first screen shot suggests that it is a so-called lock file (sometimes called an owner file). The second screen shot suggests that this file is loaded into memory when Word starts, presumably because it is located in Word's Startup folder.

    3. Under Customizing Outlook, click on Rules for sorting new messages.
    4. Click New.
    5. Under Step 1; on the first box, choose To or Cc line; on the second box, choose contains; then on the third box, type in your alias email address.
    6. Under Step 2, choose the radio button Move to a new folder (then type inside the box provided the folder name for your alias).
    7. Click Save.

    In addition to Doug's explanation (which should say "they were either created using Word 2007 or 2010"), the step after 'Go to "file"' should be 'Go to "Info"'. That's where you'll find the "Check for Issues" button.

    That test isn't always useful for another reason. Here's the Compatibility Checker of Word 2013 for a document created in Word 2016 that contains an inserted online video, which is supported only in Word 2013 and 2016:

    This tells you nothing about what version created or opened the document!

    You may get some useful information by opening the macro editor, pressing Ctrl+G to open the Immediate window at the bottom, typing this line and pressing Enter:

    Print ActiveDocument.CompatibilityMode

    A version number will appear on the next line, where

    11 = Word 2003

    12 = Word 2007

    14 = Word 2010

    15 = Word 2013

    (Note that Microsoft skipped version 13, and Word 2016 shows the value 15 instead of its version number 16.)

    This is usually (but not always) the version that created the document, or that last did a Save As without choosing to preserve backward compatibility; but it's also possible for a macro to set a document's CompatibilityMode to a completely different value.

    But in general, the tool you want to use is called "Get & Transform" in 2016, "Power Query" in 2013. Rather than creating a macro, you do your "transformation" while Excel records what you do. Later you can import more data and excel will re-apply the transforms (like a macro).

    Here is the scenario. We export data out of some system, and save it in a CSV file. We then need to prepare it for use. Perhaps to import into another system, or perhaps to use in a PivotTable or formula-based report. We basically need to clean up the data, remove some columns, change the headers, and so on.
    Back in the old days, we could automate such a task with a VBA macro that reformatted the data. Yes, it was difficult to write initially, but, it felt great when we got it working. And life was good, until something in the data changed….like a new column. Some type of change like this could break the macro. Then, we would need to crack open the Visual Basic Editor and troubleshoot. It took some time to figure out how to resolve the issue, but, it felt great once we got it working again. Until something else changed. Argh.
    The good news is that a Get & Transform Query is an easy alternative to building such a VBA macro. Best of all, modifications are easy to make when something changes.
