gerry c j cornell say:

What you see when you go to shutdown is indicating that you have a programme that is slow to respond to the shutdown instruction. If you proceed with ( or force ) the shutdown you risk losing data and damaging files. The message is a warning by the system giving you the chance to wait until the situation resolves itself.

If you click on Cancel and then select Ctrl+Shift+Esc you will see on the Applications tab a list of running applications. Place the cursor on one application and click on End Task. When the application is removed from the list repeat the procedure with next application. When all are removed you can shutdown.

A slow shutdown can be mirrored by a slow boot, although a slow shutdown is more obvious because of the warning screen. You do not see a comparable warning screen with a slow boot. It just tales longer than it should take. There is no warning screen because there is no risk of damage by premature action by the user.

You can troubleshoot to discover the cause of the problem using the Clean Boot method suggested by Linney.This method has  been used for years but with Vista and Windows 7 Microsoft has added Performance Reporting to Event Viewer to help identify the underlying causes of slow booting, slow shutdown and slow resume from standby. To access these reports select Start, Control Panel, Performance Information and Tolos, Advanced Tools, View performance details in Event Log. These reports are complicated so you will most likely need help with their interpretation.

Event Viewer Reports

1. Normally when an error occurs on your computer looking in Event Viewer should be your starting point for finding a solution. Most system related errors are logged and getting an exact copy of the relevant report is important. Unfortunately understanding the reports is not easy and most computer users need help with their interpretation. I have more to say later on interpretation.

2. Event Viewer comprises three main Windows logs. These are Application, Security and System. For troubleshooting purposes System is by far the most important.

3. To access the System log select Start, Control Panel, Administrative Tools, Event Viewer, from the list in the left side of the window select Windows Logs and System. Place the cursor on System, right click and select Filter Current Log. Check the box before Error and click on OK and you see only Error reports. Click on the Date and Time Column Header to sort. You may need to click a second time to see the latest Report at the top.

4. A tip for posting copies of Error Reports! Run Event Viewer and double click on the error you want to copy. Click on the Copy button to place a copy on your Clipboard and close Event Viewer. Now start your message and paste into the body of the message. Make sure this is the first paste after exiting from Event Viewer.

5. There are three types of Report, being Information, Warning and Error reports. In most situations it is Error Reports that offer the best information but occasionally Warning Reports provide useful clues.

6. All reports have date and time stamps and when troubleshooting it is important to concentrate on more recent reports. Study reports since the point when the computer was last booted and then check whether a similar report appeared in the previous session. If errors do not repeat investigation as to why they occurred is wasted effort.

7. Within individual reports the more important information is Event ID and Source as these help when looking for help on the internet. The description is equally important and copying the exact text for use as the search criteria greatly helps getting better results when using Google. Do not paraphrase descriptions when asking others for help.

Hi I'm Peter an independent advisor, when you install Windows 10 or Windows 8/7 on a cleanly formatted disk, it first creates a partition on the disk at the beginning of the hard disk. This partition is called the System Reserved Partition. After that, it uses the balance unallocated disk space to create your system drive and install the operating system. It would recreate itself when performing a clean install of windows if you were to delete it however I would recommend just to leave it as it contains the boot loader and bit locker information if enabled.

In regards to not being able to select the version the installation media retrieves the information from the UEFI and auto selects the edition of Windows X to install. ken blake say:

All 32-bit client versions of Windows (not just XP/Vista/7/8/10) have a
4GB address space (64-bit versions can use much more). That's the
theoretical upper limit beyond which you can not go.

But you can't use the entire  address space. Even though you have a  4GB address space, most people can only use *around* 3.1GB of RAM. That's because some of that space is used by hardware and is not available to the operating system and applications. The amount you can use varies, depending on what hardware you have installed, but can range from as little as 2GB to as much as 3.5GB. It's usually around 3.1GB. Your 2.94GB is a little lower than average, but it's nothing to worry about.

Note that the hardware is using the address *space*, not the actual RAM itself. If you have a greater amount of RAM, the rest of the RAM goes unused because there is no address space to map it to.

Your say your system is capable of running 64-bit, but how do you know? Are you sure? You say it's an older computer, but note that the older it is, the less likely it is.

Having done a backup is good, but let me point out that it's only backup of data files that are useful. You will have to do a clean installation, and that means that any backed-up programs are useless. Programs will have to be reinstalled from their original media.

One other point. Are you sure that changing to 64GB is worth doing? It largely depends  on what applications you run. It may be valuable for some people, but not necessarily for you.

lemp say:

The IP address 255.255.255.255 is not a valid address for a website.  Rather, it is the "broadcast address" for your local network, intended to send a message to all devices connected to your local network.

Svchost.exe is a "host process" for running programs defined as *.dll files rather than as *.exe files.  It's common for multiple instances of svchost.exe to be running simultaneously.

You'll need to do some investigation to determine what actual process is causing this issue.  The method discussed in this article may work, but because I haven't encountered a problem such as you describe, I don't know for sure.  http://support.sasktel.com/app/answers/detail/a_id/11365/~/using-netstat-to-find-programs-using-up-internet-bandwidth-in-windows-xp%2C

In step 4, you'll be looking for 255.255.255.255 in the "Foreign Address" column or perhaps 68 after the colon in the "Local Address" column.

In step 5, use CTRL+SHIFT+ESC instead.

After step 7, check the box (or click the button) to "Show processes from all users" and respond to the UAC prompt as needed.

After step 9, if you click on "PID" in the header row, you'll sort the entries by PID, which should make it easier to find a PID that you're looking for.

Assuming that you find the PID corresponding to the instance of svchost.exe that's attempting to use the IP address, right-click on it and select "Go to Service(s)" -- this will let you know what services are being hosted by the "bad" svchost.exe. 

If it isn't obvious (or easily determined through a Google search) what is actually causing the issue, post back here with the list of services corresponding to the instance of svchost.exe that was causing the issue.
