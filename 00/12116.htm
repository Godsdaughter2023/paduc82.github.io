jay freedman say:
Probably the easiest way to avoid the message is to set Options.CheckSpellingAsYouType=False before assembling the final document. That's equivalent to going into the Options dialog, selecting the Proofing page, and unchecking the box for "Check spelling as you type". It means that the spell checker won't execute unless you manually request it by pressing F7 or clicking the Spelling & Grammar button on the Review tab.

The best permanent solution is to define a "No Proofing" character style in the underlying template for the individual files. In the New Style dialog, click Format > Language and check the box for "Do not check spelling or grammar". That should be the only non-default attribute in the style. In the individual files, instead of "ignoring" text, apply the No Proofing style to it. When the file is pulled into the main document, the style will still apply, unlike the "ignore" marking, so the spell checker won't see those words as errors.

The VBA property Range.NoProofing corresponds to the setting of the "Do not check spelling or grammar" check box for the specific range (character, word, paragraph, etc.) that you're accessing -- it's true if the box is checked, false if it's unchecked, or 9999999 if it's checked for only part of the range. In a macro, you can set NoProofing = True for certain words that would otherwise be marked as errors, which would be equivalent to selecting those words and going to the Set Proofing Language dialog and checking the box.

ActiveDocument.ShowSpellingErrors only controls whether the red wavy underlines appear in the document. It does not stop the spell checker from noticing them, so it wouldn't do anything about the error message.

You can set ActiveDocument.SpellingChecked = True to make Word think that the Spelling dialog (F7) has been run, but that also won't stop the spell checker from going nuts if Options.CheckSpellingAsYouType is still True.

jay freedman say:
I'm a little dubious about that macro. As I wrote before, the SpellingChecked and
ShowSpellingErrors properties affect whether spelling errors are displayed in the current document, but they don't prevent the spelling checker from continuing to look through the document. I suspect that the message can still occur if this macro is all you've done. Setting Options.CheckSpellingAsYouType=False does stop the checker and should prevent the message.

A minor point: you don't have to include the statement
Selection.EscapeKey, because the
Selection.Collapse statement will take care of deselecting the text.

Regarding how the information is stored: The fact that a document has been spell checked is not stored in the document's file, but only in memory for the current session. If you close the document and reopen it, Word does not know that it was previously checked. Similarly, the list of words and rules that have been ignored is only in memory for the current session and is not stored in the file (which is something we've complained about, but isn't likely to be changed).

You're correct that you can't define a character style that has the "do not check" option unless you also specify a language. As a replacement for the NoProofing style suggestion, you could include the following macro to be used on the individual documents after they have been given their final spell check, to mark the words you want ignored.

Sub MarkAllNoProofing()
    Dim rgToMark As Range
    ActiveDocument.SpellingChecked = False   ' un-ignore all ignored words
    For Each rgToMark In ActiveDocument.SpellingErrors
        rgToMark.NoProofing = True
    Next
End Sub

jay freedman say:
> It seems strange that when you select ignore it does not perform "NoProofing"

Yes, it is strange, but true. Many people have commented that once they mark something as ignored, they want that to be permanently saved with the document so they never have to check it again, but Microsoft has never done anything about it. Marking such words with a No Proofing character style is much more cumbersome and, as you've remarked, requires a specific language to be chosen for the style.

If you know of words that you want the spelling checker to ignore in all documents, you can add them to an exclusion dictionary (http://www.word.mvps.org/FAQs/General/ExcludeWordFromDic.htm).

If you're going to set options for the whole Word program, as opposed to those for the ActiveDocument, then it's good practice to save the existing value and restore it at the end of the macro. For example,

Sub x()
    Dim CkSpellAsType As Boolean
   
    CkSpellAsType = Options.CheckSpellingAsYouType
    Options.CheckSpellingAsYouType = False
   
    ' do the work
   
    Options.CheckSpellingAsYouType = CkSpellAsType
End Sub

You already have the equivalent of hide spelling errors in this document only, which is  ActiveDocument.ShowSpellingErrors = False.

To avoid warnings about missing proofing tools, set the proofing language for the whole document like this (choose the proper constant for your installation):

    ActiveDocument.Range.LanguageID = wdEnglishUS

That may or may not also cure the 4608 error, but I have no idea what that is and no way to find out.

Hi Birb,

Rohn007 has the correct answer that you could use ODT to install the Office if you want to exclude Skype for business when you install Office: Overview of the Office Deployment Tool.

To simplify the process, there are only thress steps. 

1. Download the tool and open it. It will let you choose a location (for example C:\ODT2019) in your machine to extract it. 

2. Open the location you chosed in step 1, open one of the configure file with notepad and change the content to the following script and save it(for example, the name is Office365Configuration):

<Configuration>

  <Add OfficeClientEdition="64" Channel="Broad">

    <Product ID="O365ProPlusRetail"> 

      <Language ID="en-us" />

      <ExcludeApp ID="Lync" />

    </Product>

  </Add>

<Updates Enabled="True" Channel="Broad" />

</Configuration>

The ProcuctID is based on your subscription. The ExcludeApp ID="Lync" means don't install Skype for business.

2. Open Command Prompt from start menu

3. input the following command:

cd C:\ODT2019

setup.exe /download Office365Configuration.xml (This is the file name in step 2)

After this command, Office start to download, you could wait until the C:\ODT2019 show up in next line and input the next command:

setup.exe /configure Office365Configuration.xml

Then Office will start to install.

paul herber say:

The off-page reference shape can be used in conjunction with any template - all you have to do is open the Basic Flowchart shapes stencil and drop the shape on the page.

To make the shape into a triangle is also very simple. Right-click on the shape once it is on the page and open the shape's shapesheet. Scroll down to the Geometry1 section. First delete line4 then delete line2. Job done. Save this shape to a new stencil and always use this version in the future.

paul herber say:

Also, you can make any shape you like into an off-page connector with all its functionality.

Open the shapesheet of the original off-page connector and scroll down to the Events section. Copy the contents of the cells EventDblClick and EventDrop into the new shape. Save this shape into a new stencil. Job done.
