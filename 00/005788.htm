This can occur when one or both of the following conditions is true:

Now, to solve it, do the following:

Depending on whether you are using Outlook anywhere, follow the steps in the applicable section.

If you are using Outlook anywhere, If you are using Outlook anywhere, install the appropriate Microsoft Outlook patch. Outlook 2010 This problem was corrected in the Outlook 2010 hotfix package that is dated February 11, 2014.

For more information, click the following article number to view it in the Microsoft Knowledge Base: 2863918 descriptions of the Outlook 2010 hotfix package (Outlook-x-none.msp): February 11, 2014, Outlook 2013.

This problem was corrected in the Outlook 2013 hotfix package dated April 8, 2014. For more information, click the following article number to view it in the Microsoft Knowledge Base: 2878323 Description of the Outlook 2013 hotfix package (Outlook-x-none.msp): April 8, 2014, After installing the patch, Outlook considers Outlook anywhere to have a secure connection for the purposes of processing unwanted email messages.

Note: Outlook anywhere is also known as RPC over HTTPS (secure HTTP). As the term HTTPS indicates, Outlook Anywhere connects to Exchange through a Secure Sockets Layer (SSL) connection.

If you are not using Outlook Anywhere.

If you can not install the latest version of Outlook 2010 or Outlook 2013 and you are not using Outlook Anywhere, you can avoid this problem if the mailbox is found in Microsoft Exchange Server 2010 or an earlier version.

To work around this problem, try to enable RPC encryption between Outlook and Exchange. For this, follow these steps:

    1. In the Control Panel, open the mail item.

    2. Click on Show Profiles.

    3. Select your profile, and then click Properties.

    4. Click on Email Accounts.

    5. Select the Microsoft Exchange Server account, and then click change.

    6. In the dialog box that contains the user name and mailbox server, click More settings.

    7. In the Microsoft Exchange Server dialog box, click the security tab.

    8. Click to select the Encrypt data check box between Microsoft Outlook and Microsoft Exchange, and then click OK. Important if the checkbox is available (grayed out), you are probably connected to Microsoft Exchange Server 2013, or you are using a mailbox in Office 365. In this case, RPC encryption cannot be enabled. Note that Outlook Anywhere connections (RPC over HTTPS) are secure as they are encrypted using secure HTTP (HTTPS).

    9. Click on Next, and then click Finish.

    10. Click Close and OK.
