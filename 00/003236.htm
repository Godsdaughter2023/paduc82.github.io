spike zhang msft say: Hi Colstoh,

Thanks for your reply in PM. From your description, the issue could be caused by the external user profile in your tenant is corrupted. When we share a SharePoint site with an external user, it actually follow the process below:

1. The user received the invitation mail using his own account (E.g.: *** Email address is removed for privacy ***) and using another Office 365 account (E.g.: *** Email address is removed for privacy ***) to log in to the site.

2. In this situation, it will create a new external user profile in Office 365 Active Directory with the Display name user and Email address: *** Email address is removed for privacy ***. And in SharePoint Active Directory, the external user profile will be like *** Email address is removed for privacy ****** Email address is removed for privacy ***.

3. When you input the user name in SharePoint people picker, it will display the username using the user information in Office 365 Active Directory like user:*** Email address is removed for privacy ***.

For this kind of issue, we need to remove the external user from your tenant completely and re-invite him. Here are the steps:

Note: The following steps require an Office 365 global admin account. If you are not an Office 365 global admin, please contact them to help.

1. As an Office 365 global admin, go to Office 365 admin center > Users > Guest users. Find and delete the affected external user account.

2. Go to the Office 365 SharePoint Admin center > user profiles >Manage User Profiles > find and delete the affected external user.

3. Delete the external user from your site:

a. As a SharePoint administrator, go to the site collection in which the external user is invited.

b. Open a new tab in the browser, and enter

<site collection URL>/_layouts/15/people.aspx?MembershipGroupId=0.

c. In the "People and Groups: All People" page, delete the affected external user from these site collections.

4. Use SharePoint PowerShell to check whether you can get that external user. If so, delete that user. Here are the steps to use SharePoint PowerShell:

a. Download the PowerShell and connect it according to this article:

http://technet.microsoft.com/en-us/library/fp161372(v=office.15).aspx

b. Get all external users by running Get-SPOExternalUser -Pagesize 20

c. Delete that unique external user by running Remove-SPOExternalUser -UniqueIDs <UniqueIds>

d. Wait for a while, and then run Get-SPOExternalUser -Pagesize 20 to see if the external user has been removed completely.

5. If you use the same browser to share with the external user that you’ve shared with before, please remove the browser cache since the People picker cache will remain even though the external user is fully removed from SharePoint Online.

6.Reinvite the external use.

If the issue persists after performing the steps above, could you involve your Office 365 admin in this thread for further troubleshooting? Thanks for your effort.

wayne-i-m say:

Hi Map point is a screen based product and so will always be slightly less than you need for print/PowerPoint/enlarge/etc.

The standard method is just to save each map that you want use elsewhere as a .ptm file and use the PowerPoint insert object. Another method (still using insert object) would be to save the map as a webpage.

The 2 methods shown will produce OK results, but to get a better (higher res) map you will need to open the map you have saved in something like Photoshop or PhotoPaint and then resample. Although this may be against the map point terms and conditions so please check this 1st. If you can open the map in a design programme (as a vector graphic so it can be enlarged without loss of detail) then you can choose which layer you need for your PP presentation.

After you have resampled the map you should save it as .PGN - the reason for that is that power point will convert most bitmap file types to PGN anyway (except jpeg) so I would thing you would be better doing this your self.
