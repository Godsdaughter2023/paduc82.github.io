ivy guo msft say:

Hi Ragna，

  

I'm glad to help you explain the permission and permission level of SharePoint Online. During my  test and research, I want to do a summary about it.

First, user can see the file at least have read permission. If he don't have permission for this file, this file won't appear in the library list of this site for this user. So he won't have access denied page when he click on some specific files/folders.

Second, the SharePoint site permission is inherited.  By design, all the sites and site content in a collection inherit the permissions settings of the root or top-level site. When you assign unique permissions to sites, libraries, and items, those items no longer inherit permissions from their parent site.

Third, user can access this site that means he have permission to access this site also its library(That are what you mentioned driving, outlay, vacation. They are all library/ list.) And if you remove library's permission to a user, he can't see the whole library.

graham mayor say:
The following should work for you

Sub Black()
Dim oRng As Range
Set oRng = Selection.Range
oRng.MoveStart wdWord, -1
oRng.InsertBefore "Black - "
oRng.Collapse wdCollapseEnd
oRng.Select
End Sub

Sub Pink()
Dim oRng As Range
Set oRng = Selection.Range
oRng.MoveStart wdWord, -1
oRng.InsertBefore "Pink - "
oRng.Collapse wdCollapseEnd
oRng.Select
End Sub


Assign the macros to buttons on the QAT (Quick Access Toolbar) of to keyboard shortcuts - see http://www.gmayor.com/installing_macro.htm
