    find links in view source - Microsoft Community

    questioning1 say:

    find links in view source

    Our company has purchased third-party software to find broken hyperlinks within files on Sharepoint.  However, I was not able to find a way to find broken hyperlinks within links lists on Sharepoint pages. 

     

    I created the VBA code below as a work-around.  What I can do is go to the Sharepoint page, right-click and select view source, select all, copy to clipboard, and paste within a new Excel worksheet.  I run the macro and have a worksheet created with all hyperlinks beginning with "http" and "mail". Keep in mind, I am not a programmer and only dabble with VBA.

     

    Is there a better way for me to do accomplish the task of finding broken hyperlinks within Sharepoint links lists?  If not, is there better VBA code I can use?  I tried writing VBA to also include the hyperlink name, but there were too many variations within the source code (e.g. <nobr>, <font", etc. all in different locations within HREF). So if I encounter a broken hyperlink, I can simply find the hyperlink within view source and determine the hyperlink name on the Sharepoint site that needs to be updated.

     

    Sub AllLinksInSourceCode()
    '
    ' This macro will return hyperlinks found within a source code page
    '
    Application.EnableCancelKey = xlInterrupt 'allows user to interrupt processing
    Application.ScreenUpdating = False 'screen will not show the work being done

    Dim fromSheet As Worksheet 'Excel worksheet with source code
    Dim toSheet As Worksheet 'Excel worksheet with only hyperlinks from source code
    Dim rowCount As Integer 'Total rows used in fromSheet
    Dim cellCounter As Integer 'used to iterate through a cells value to look for hyperlinks
    Dim counter As Integer 'used to iterate through all rows in fromSheet
    Dim strCellVal As String 'stores the cells value
    Dim strCellEnd As Boolean 'determines if the cell iteration has reached the end
    Dim beginHREF As Integer 'finds the location of beginning hyperlink string
    Dim endHREF As Integer 'finds the location of the ending hyperlink string
    Dim lenHREF As Integer 'determines the length of the hyperlink string
    Dim pasteRow As Integer 'used to iterate the rows in toSheet
    Dim strHyper As String 'stores the hyperlink value

    Set fromSheet = Application.ActiveSheet 'sets active sheet as fromSheet
    AddAsLastWorksheet 'creates a worksheet called "Hyperlinks"
    Set toSheet = ActiveWorkbook.Worksheets("Hyperlinks") 'sets Hyperlinks sheet as toSheet
    toSheet.cells(1, 1).Value = "Hyperlink" 'Makes a header of Hyperlinks in toSheet
    toSheet.cells(1, 1).Select 'select the header cell
    toSheet.cells(1, 1).Font.Bold = True 'bolds the header cell
    fromSheet.Activate 'ensures fromSheet is active

    rowCount = fromSheet.UsedRange.Rows.Count 'counts all rows in fromSheet
       
    pasteRow = 2 'initiates variable
           
    For counter = 1 To rowCount 'will iterate through all rows in fromSheet
        If Len(fromSheet.cells(counter, 1).Value) > 0 Then 'checks if cell has a value
            strCellVal = fromSheet.cells(counter, 1).Value 'stores cell value
            cellCounter = 1 'initiates variable
            strCellEnd = False 'initiates variable
            Do Until strCellEnd = True 'will iterate through the cell with value
                If InStr(cellCounter, strCellVal, """http") Then 'checks if cell has "http
                    beginHREF = InStr(cellCounter, strCellVal, """http") 'finds where link begins
                    endHREF = InStr(beginHREF + 1, strCellVal, """") 'finds where link ends
                    lenHREF = endHREF - beginHREF 'stores length of link
                    strHyper = Mid(strCellVal, beginHREF + 1, lenHREF - 1) 'stores link value
                ElseIf InStr(cellCounter, strCellVal, """mail") Then 'checks if cell has "mail
                    beginHREF = InStr(cellCounter, strCellVal, """mail") 'finds where link begins
                    endHREF = InStr(beginHREF + 1, strCellVal, """") 'finds where link ends
                    lenHREF = endHREF - beginHREF 'stores length of link
                    strHyper = Mid(strCellVal, beginHREF + 1, lenHREF - 1) 'stores link value
                Else
                    strCellEnd = True 'has reached end of value within cell
                End If
                If strCellEnd = False Then 'checks if end of value within cell reached
                    toSheet.cells(pasteRow, 1).Value = strHyper 'stores link value in toSheet
                    toSheet.Activate 'activates toSheet
                    toSheet.cells(pasteRow, 1).Select 'selects row link pasted in on toSheet
                    toSheet.Hyperlinks.Add Anchor:=Selection, Address:= _
                        Selection.Value, TextToDisplay:=Selection.Value 'changes value to a hyperlink
                    pasteRow = pasteRow + 1 'provides next cell available on toSheet
                    fromSheet.Activate 'activates fromSheet
                    cellCounter = endHREF 'stores where link ends within cell value to start iteration from that point
                End If
            Loop
        End If
    Next counter
    End Sub

    Sub AddAsLastWorksheet()

        Dim ws As Worksheet
        For Each ws In Application.ActiveWorkbook.Worksheets
            If ws.Name = "Hyperlinks" Then
                With ws
                    Application.DisplayAlerts = False
                    .Delete
                End With
            End If
        Next ws
    Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = "Hyperlinks"

    End Sub

    greginmich say:

    This question doesn’t have anything to do with scheduling a scan, and having to explain the sorry situation with Microsoft's new method for scheduling a scan is getting pretty old. Any reference to that method is confusing and counterproductive, and the page really needs to be taken down. The basic problem is that there's no need at all for adding a trigger to the Windows Defender Scheduled Scan task, because that task is already being triggered daily by Windows Automatic Maintenance. And you can also schedule Automatic Maintenance  for a specific hour of the day in the Security Center:

    Right-click on the Start button > Control Panel > Change maintenance settings.

    But the Automatic Maintenance Quick Scan isn’t really designed to run on a schedule; it’s designed to run when you’re not using your computer – in order to prevent the scan from interfering with your work. So if you schedule a scan by adding a trigger to the Windows Defender Scheduled Scan task, as per the method in the link above, what you’ll actually get is just a scheduled Quick Scan that doesn’t run on schedule, just like the Automatic Maintenance scan. But once again, Automatic Maintenance is already running a daily Quick Scan that doesn’t run on schedule – every day of the week – so what’s the point of replicating that scan? Frankly, I think the people that came up with this dandy little ditty for scheduling a scan in Windows 10 either didn’t know how Automatic Maintenance works – or were maybe just looking for a way out of actually having to explain how it works.

    If you want a scheduled scan that actually runs at the scheduled time; and that runs without interruptions; and that also includes options for things like an unattended Full Scan – then this is the method that you'll have to use:

    http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-i-set-windows-defender-to-a-custom-auto/ec19aab4-10ce-4f41-9990-343056fd104f

    You can also use this method to schedule a Custom Scan, but that option is only available for the Command Line Interface. There's no option at all for scheduling a Custom Scan or a Full Scan with the Microsoft method.

    As for the original question; you would use a Custom Scan when you want to scan specific drives or folders. Just click on the “plus” to expand the drive/folder, and then check the box for each of the folders that you want to scan. There's no option for scanning specific files here, but it wasn’t long ago that we witnessed the long-awaited return of the right-click context menu scan for Windows Defender; so now we once again have an easy way to scan a specific file or folder, or group of selected files or folders:

    http://windows.microsoft.com/en-us/windows-10/scan-an-item-with-windows-defender
