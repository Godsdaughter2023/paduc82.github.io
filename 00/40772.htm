hairyfool say:

A system image is NOT incremental, it contains a complete snapshot of the drives at the time of the backup and on restore is not dependent on previous images. No matter how may times you have done a system image on different devices the recovery only requires access to one image for restore to work therefore it has to be complete. The only reason why a second image might be different is if the file set is a different size. For example if you do an image soon after the free upgrade Windows.old is included and could be quite large, after 28 days the Windows.old is purged leaving only user files and the image would be smaller.

Only dedicated backup programs have the ability to perform incremental backups because it requires the maintenance of a database of the files and usually include relatively complex user interfaces to set what is to be included on backup and to select individual files, folders or the entire system to restore. The confusion may arise because when using the Restore tool system images will appear as additional restore points however what you can't do using the system image tool is restore individual files. The complexity of doing full, incremental or differential backups over what schedule tended to put the home user off proper backup procedures so were seldom done. File History is a continual incremental backup of only the selected files and by default does not contain any system level archive.

The System Image tool has been retrograded with Windows 10 and only appears as a secondary function of File History. The philosophy behind this is instead of restoring a complete system you can reinstall Windows then by logging in using your Microsoft Account along with OneDrive and File History you can recover all your files. It is a matter of opinion as to whether that represents a suitable alternative to a traditional backup program

hairyfool say:

One limitation of system image is it will only restore to a disk the same size or larger than the source, not normally a problem as most people upgrade when replacing disks. However as SSD becomes more prevalent it can be an issue as replacing a multi terabyte disk with an SSD is prohibitively expensive if the same size.

Most 3rd party tools can cope with shrinking the drive because it only needs one big enough for the data being restored.

Let me assure you with 100% certainly that Method 1 and the Volume Shadow Copy Service has nothing to do with System Restore.  And, there is no need for the VSS Service to be running all day with nothing to do.  If you do anything, you should set it to Manual and it will start and stop when it needs to instead of burning CPU cycles and memory all day and night that you would rather have going to other things.

Let me assure you with 100% certainty that you do not describe symptoms consistent with a need to run the System File Checker and even suggesting it is akin to saying "I have no idea what the problem is or how to fix it".

Those are just Microsoft engaged Support Engineer "expert"  wild goose chases.

System Restore (and the Restore Points) are not magnets for viruses.  You may have heard or read about situations where a system was afflicted with malicious software and somehow perhaps some of the afflictions made it into a Restore Point so after the malware was removed, the affliction might still be in some older Restore Points.  Allegedly, if you restore your clean system using an infected RP, your system will become infected again.  I am not sure about that, but you can mitigate the risk.

If that is the case (or you suspect it), once your system is clean, you can create a new "clean" Restore Point and wipe out all you old RPs prior to that to get a fresh start.

System Restore should certainly work but there are many reasons it will not work either because your system is afflicted with malicious software that prevents it from working, perhaps your System Restore is just broken and needs to be fixed, and some antivirus antimalware programs are known to interfere with it - but is should work.

The day you need System Restore to work is not the day to find out it is broken.
