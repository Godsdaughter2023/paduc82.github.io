d6a6n6 say:

I believe historical data is stored on a Mac in the following path:

[Home]\Library\Application Support\Skype\[your skype name]\main.db

If you don't have any backups with this path and file then you are out of luck.  I believe skype only stores 30 days in the cloud.

If you do have a backup you might be able to copy the file to a new machine to be able to view the history in skype but I haven't tested this (I have only done it on PC and with all files within the %appdata%\Skype\[skype username]\ folder).

If you have deleted this path on your Mac then the only way you get can get it back is from one of your own backups.  So if you have a backup you would have to:

 

-  create a new backup,

-  restore from the old backup that has the historical data

-  save everything in the [your skype name] path to a thumb drive

-  restore from your latest backup

-  log into Skype (to ensure the path above is created)

-  close Skype

-  copy the folder over the top of your existing [your skype name] folder in the path above.

Note: you can download applications to view the history in log form from the main.db file if you want to double check that the history is there before you do all the backups and restores.  On PC there is SkypeLogView by NirSoft which is easy to use...not sure for Mac.

bill manville say:

I see what you see in 2013, but I see the same result (25) in 2010 and indeed in 2002.

The reason would appear to be that Excel is recognising that the strings in A1 and A2 could be representations of dates - in each case 1-July-2016 (assuming your regional settings are DMY). 

This seems a little over-helpful, and certainly not what I would have expected.

=SUMPRODUCT(1*(A1:A2=A1),B1:B2)

will give the result you wanted.

I will ensure the product team are aware of this, but they may decide that changing a behaviour that has been around for at least 14 years (and probably more) would be undesirable.

bill manville say:

The use of semi-colons in 2010 and commas in 2013 is further evidence of a difference in regional settings between the two computers. When the regional date setting is MDY the SUMIF will deliver 10 and when it is DMY it will deliver 25.
