shan mcarthur say:

In my case, I was using Windows Server on my laptop, and it was a domain controller.  When the Active Directory Domain Controller role is added to Windows Server, write caching is disabled on the physical drives that house the NTDS folders.  By default that is your system drive, and disabling write caching means a server can go from 67MB/s write performance to 1.6MB/s write performance.  I now use Boot2Vhd which doesn't allow for the disabling of the physical drive, and as a result, I no longer notice the REGBACK performance issue.  I don't know if this is the same configuration as you have, but that is what worked for me.
