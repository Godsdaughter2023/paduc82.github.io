a. user say:

Just about the only way to get the data from Excel in a form close to what you had in Excel is to connect to the data source using DDE. However, 
 a. that only works if you have Excel on the system performing the merge
 b. it only works for the first sheet in an Excel workbook
 c. there are potentially other problems, e.g. if the sheet contains Unicode character data, e.g. some accented characters etc.
 d. no method other than export to a text file gets you beyond 255/256 columns of data, and with DDE it may be less than that.

<<
Is there a way of manually telling Word 2010 that I know what form I want my data in better than it does?
>>

Not really. 

If the data is in a format that you can guarantee will be read by one of Word's internal text converters (e.g., a Word .doc, .docx, .rtf) the chances are that Word will leave the data unchanged. 

If the data is in a plain text format that will be read using the OLE DB text provider or ODBC text driver, it is possible to indicate data types using a schema.ini file in the same folder as the data source.

(But to add some complication that you might not even have considered, even then, there can be problems, e.g. if you have data created in a locale where they use "," as the decimal point, and you use ".", it would be difficult to get Word to recognise the data as numeric without changing your locale, and so on.)

<<
If not, why not?
>>

You'd really have to ask Microsoft that (and I doubt you would receive a satisfying answer even if you could work out where to ask). Here, the people are mostly volunteers who cannot know the reasoning behind Microsoft design decisions or decisions not to fix errors. I have my own views but there is really little point in posting them here.

For some of the gory details about Excel data sources, please see:

http://tips.pjmsn.me.uk/t0003.htm

That page does not reflect everything that is now known about Excel sources or workarounds.

To connect using DDE (if you have not already tried)
 a. go to File->Options->Advanced->General (near the bottom) 
 b. check "Confirm file format conversion on open"
 c. go through the process of opening your  Excel workbook again
 d. when you see Confirm data source, if you see a DDE option, choose it. If it's a .xlsx/.xlsm data source, you will need to check Show All first, then look for and select the MS Excel Worksheets via DDE (*.xls) option

reverting all slide masters to default - Microsoft Community

cvhorie say:

reverting all slide masters to default

I have been using Powerpoint for about 20 years.  I cut, paste and create slides from many presentations to add to the next.  As a result, the slides have acquired a random collection of footers and other styles, many of which cannot be removed by any means.

How do I clear all my style settings from all the slides in a presentation? 

steve rindsberg say:

With Office 2007, the way PPT handles footers/slide numbers/dates changed. Previously, they were added on the masters and could be covered up by content that you added on the individual slides.

People complained about this, so now, footers (and the rest, and I'll just call 'em footers from now on) are duplicated from the master onto individual slides; in other words, they're shapes that you can select and move/edit/delete on each individual slide.

This means that you can select a footer and bring it to front so it sits atop any other content on an individual slide.  That part's useful, but it also means that when you bring in older files, it can create "orphaned" footers that are shapes on a slide but that have no connection to the slide master.

Try turning all of the footers (etc) off in slide view and choosing Apply to ALL, then turn them off on all of the masters/layouts. Any remaining footers you see on the slides at that point are orphans and should be deleted.  Then you can turn footers back on as desired.
