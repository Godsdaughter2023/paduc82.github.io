"Specific table" 1:

** Moved from: Office/Word / Windows 10 / Office 2019 **

doug robbins - mvp office apps & services (word) say:

If each of the specific tables is identified by a bookmark (Table1, Table2, Table 3, etc) and the Overall table is identified by the bookmark Overall, and that table contains only the header row, the following macro will add rows to it as required and populate each row with the contents of the cells in the second column of each of the specific tables:

Dim i As Long, n As Long, t As Long
Dim rng As Range
Dim newrow As Row
TryAgain:
n = InputBox("Insert the number of specific tables to be processed")
If Not IsNumeric(n) Then
    If MsgBox("You must insert a number. Do you want to try again?", vbYesNo + vbCritical) = vbYes Then
        GoTo TryAgain
    Else
        Exit Sub
    End If
End If
With ActiveDocument
    For t = 1 To n
    If Exists(.Bookmarks("Table" & t)) Then
        With .Bookmarks("Table" & t).Range.Tables(1)
            For i = 2 To .Rows.Count
                Set rng = .Cell(2, 2).Range
                rng.End = rng.End - 1
                Set newrow = ActiveDocument.Bookmarks("Overall").Range.Tables(1).Rows.Add
                newrow.Cells(1).Range.FormattedText rng.FormattedText
            Next i
        End With
    Else
        MsgBox "Table" & t & "Does not exist."
        Exit Sub
    End With
End With

daniel pineault say:
You need to fix the Combo Box so it lists the employees full name, not just the last name.  What is the Combo Box's Row Source?

gina whipp say:

Hmm, IMHO...

 

You really should base your Combo Box Row Source on the Primary Key or some other unique value that each Employee has.  That field can be hidden but then you would have no issues returning the correct record.  AND, as Daniel says, you need to pur LastName, FirstName or how would anyone know the correct *Smith*?

daniel pineault say:
Gina is quite correct.  You should be using the PK, not the [Operator Name Last] to drive your 'search'.

What is your Combo Box's after update event code?

Now if you were to switch your Row Source to something like:

SELECT [Operator Name Last], [Operator Name First] FROM [Personnel Info] ORDER BY [Operator Name Last], [Operator Name First];

Then set the:

Column Count: 2
Column Widths: 1.5";1.5"

Obviously you can adjust the column width (I just gave you a starting point)


Now, as Gina mentioned, you should be changing everything to us the PK, so you'd use a row source like:
SELECT [PrimaryKeyFieldName], [Operator Name Last], [Operator Name First] FROM [Personnel Info] ORDER BY [Operator Name Last], [Operator Name First];

Then set the:

Column Count: 3
Column Widths: 0";1.5";1.5"

As you can see, we end up hiding the PK field in the Combo Box, but it is the best field to use to code your VBA to ensure you get the record you are after.
