jay freedman say:
For each session, Word reads the building block templates only at the time when you first use any building block or display the Organizer or any gallery (e.g., cover pages) that shows building blocks. After that, the building blocks stay in memory until you shut down Word, and changing the template doesn't change what you see in the user interface. Deleting or creating individual building blocks does modify the in-memory set at the same time that it changes the template.

To see the old building blocks go away when you replace a template, you'll have to shut down Word, restart it, and open the Organizer or do one of the other actions.

john p spencer say:

You should not do this using a field.  The status "complete" can always be calculated when needed in a query.  You would have an expression like the following (I'm guessing about your table structure).

IIF([Qty-Ordered] - (Nz([Shipment-1])+NZ([shipment-2])+NZ([Shipment-3])+NZ([Shipment-4])) <= 0, "Complete", Null)

If you absolutely can't live without that calculated value in your table then you will need to use that expression in the before update event of your form to assign "Complete" or Null to the field in the record.  You cannot enter information directly into a table or a query and get this functionality.  If you do so, the status will not be current.
