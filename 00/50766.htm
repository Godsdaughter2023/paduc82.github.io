they just let me to mail all the faulty tablets to them, and told me over and over again that I had to pay for the mailing,
and I was told too that all the systems, apps, files and other data on my tablets all would be deleted during the repair processes, so I had to back up them before mailing...
Sigh...


Fault symptom
Touch screen of the tablet no any responses to any touches when using desktop mode in Windows 10.
The tablet has installed dual OS of Windows 10 and Android, with the first fresh booting:
touches in Android work properly,
touches of tablet mode in Windows 10 always work properly,
touches of desktop mode in Windows 10 don't work, but... only during the first fresh booting round, touches don't work, if reboot, touches will work properly.
Namely, Windows 10 is already set to use desktop mode, at shutdown state, I turn on the tablet, freshly boot system, go to sign-in screen, now if there was not attached any mouse or keyboard to my tablet, because any touches don't work, so I can do nothing, certainly after attached a mouse or keyboard, I can sign in, but after signed in, touches still don't work, then, I reboot Windows 10 system, go to sign-in screen, now touches work properly.
But then I shutdown, wait a moment, then turn on the tablet, touches don't work, that is, the fault symptom comes back again.  

How could lead to this symptom?
To this tablet,
if freshly install a new Windows 10 system in it,
or,
upgrade the current Windows 10 system with:
in "Ready to install" window, click "Change what to keep" option,
then in "Choose what to keep" window, select "Nothing [Everything will be deleted, including files, apps and settings]" option, then click [Next] button...
After finished installation, then it occurs.


Speculation for probable cause
The driver of touch IC can be correctly identified and loaded by OS,
anyhow, after OS upgraded, the driver can be loaded properly.
But, work together with the driver, another driver's configuration file provided by the manufacturer cannot always be kept after the OS is newly installed or upgraded with the keep nothing option, it will be lost.
The touch IC driver's loading (specifically to the first fresh booting round of the desktop mode) depends on this driver's configuration file, if it not be found, the driver cannot be loaded, so the fault happens.

Now this problem already has been solved by myself.


My sister's tablet still has the configuration file,
so I copy it into a USB flash drive,
then copy it from USB flash drive to my system subfolder,
then reboot (note: must be "reboot", but not "shutdown then newly boot"),
then it works.


Certainly, it was not easy to find it... that's a hard work, after a careful comparison of the related files on both computers, finally found it, maybe a little similar to Hercule Poirot finally found the suspect.


Solution in detail
Save the configuration file which named TouchSetting.gt into a USB flash drive,
then attach the USB flash drive to my tablet,
the USB flash drive's letter is H,
then right-click Start button, then click Windows PowerShell (Admin), then run the below command:
robocopy "H:\" "$env:WinDir\INF" "TouchSetting.gt" /b
then reboot, then can see that everything is OK.


Hope this thread, more or less, could help or inspire some person who has met similar problem.
