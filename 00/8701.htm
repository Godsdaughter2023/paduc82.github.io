mike h.. say:

Hi,

I'm not sure if you want this as you change Col L to closed or not. This code hides the row automatically if you change a cell in Col L to closed.

Right click the worksheet tab, view code and paste this code in. Close VB editor.

Private Sub Worksheet_Change(ByVal Target As Range)
If Target.Cells.Count > 1 Or IsEmpty(Target) Then Exit Sub
If Not Intersect(Target, Range("L:L")) Is Nothing Then
    Application.EnableEvents = False
    If UCase(Target.Value) = "CLOSED" Then
        Target.EntireRow.Hidden = True
    End If
    Application.EnableEvents = True
End If
End Sub

If you have instances of Closed in Col L that you want to hide then put this code in a general module like this. ALT+F11 to open VB editor, right click 'ThisWorkbook' and insert module and paste this code in and run it.

Sub somesub()
Dim LastRow As Long
Dim MyRange As Range, c As Range
LastRow = Cells(Rows.Count, "L").End(xlUp).Row
Set MyRange = Range("L1:L" & LastRow)
For Each c In MyRange
    If UCase(c.Value) = "CLOSED" Then
        c.EntireRow.Hidden = True
    End If
Next
End Sub

jeeped say:

While a drop-down list typically means you are dealing with changes in value one at a time, importing, copy & pasting or otherwise providing bulk changes is not out of the question. This would be my suggestion,


Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Target, Me.Columns("L")) Is Nothing Then
        On Error GoTo FallThrough
        Application.EnableEvents = False
        Dim l As Range
        For Each l In Intersect(Target, Me.Columns("L"))
            Me.Rows(l.Row).Hidden = (UCase(l.Value) = "CLOSED")
        Next l
    End If
FallThrough:
    Application.EnableEvents = True
End Sub

Edit: with that in place, you can copy column L and paste it right back into column L for an immediate update.

echo s say:

Hi, OfficePerfectionist,

 

Let me echo Steve and say thank you for your thorough and well explained post. Then let me begin by saying you are not crazy. Yes, PowerPoint and Excel change the colors of data in charts.

 

It happens when you add the sixth data series. Until then, the chart colors are your theme accent colors. After you add the sixth data series, the chart colors adjust slightly. You are correct that this is hard-coded -- you're not doing anything wrong, it's just Microsoft attempting to be helpful. If you apply one of the monochromatic versions of the chart that's in the chart quick styles gallery on the Chart Tools Design tab, you can see how this would be helpful. On a default-ish "colorful" chart that relies on accuracy of theme accent colors, though (which are often based on corporate brand colors), it's maddening.

 

Unfortunately I do not have a perfect solution for you.

 

The best thing I can suggest is that you create chart templates. Make a chart that has as many data series as you're likely to need. I usually suggest 8 to 10. Format the chart the way you want it to be -- apply the correct colors, font sizes, legend location, axis scale, etc., then click Save As Chart Template on the Chart Tools Design tab.

 

Once you've created a chart template, you can apply it to any chart. To do so, add all your data, then select the chart, click Change Chart Type, and then click the Templates folder in the dialog box. Select and apply the template to your chart.

 

Note that, while the chart template you created may be based on your theme colors, the chart template -- and any charts you apply the template to -- are no longer attached to the theme. (This is called absolute formatting.) If you apply a different PPT template, Office theme, or color scheme, the colors on those charts / chart templates will not change.

 

Note also that it's easier to apply a chart template to an existing chart. You can start with the chart template if you want (Insert | Chart | Templates | choose chart template), but as you add more than 3 data series, the chart will revert to using the theme colors for the extra data series and the colors will eventually be off so you will have to apply the chart template again anyway. Best just to make the chart and then apply the template so it will apply to all the data series.

 

I hope that helps a bit. Holler back if you have questions.

suzanne s. barnhill say:

If you have Widow/orphan control enabled, you don't need "Keep lines together" to keep two lines together. If you were likely to have headings that ran to three lines, then this would be needed. Believe it or not, I have worked for authors who tried to sum up the entire content of a section in the subheading, and it could easily run to more than two lines if the (left-aligned) heading was broken at two-thirds or so of the width of the line (headings should never run full-width).

stefan blom say:

Remember that, in Word, a paragraph is created when you press Enter. "Keep with next" keeps a paragraph with the following paragraph. "Keep lines together" is only relevant for multi-line paragraphs.

In theory, a multi-line paragraph may have "Keep with next" enabled but still fall over two pages, say with two lines on page 2 and two more lines on page 3. Enabling both options prevents that from happening.

Sub TestSelect()
Dim oRng As Word.Range
Set oRng = Selection.Range
If oRng.Font.Underline <> wdUnderlineDouble Then
  MsgBox prompt:="This is not double underlined"
  Exit Sub
End If

On Error GoTo Err_Start
Do While oRng.Characters.First.Previous.Font.Underline = wdUnderlineDouble
  oRng.MoveStart wdCharacter, -1
Loop
Err_Reentry1:
On Error GoTo Err_End
Do While oRng.Characters.Last.Next.Font.Underline = wdUnderlineDouble
  oRng.MoveEnd wdCharacter, 1
Loop
Err_Reentry2:
oRng.Select 'or do whatever with it
Exit Sub
Err_Start:
  Resume Err_Reentry1
Err_End:
  Resume Err_Reentry2
