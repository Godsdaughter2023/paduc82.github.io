The licensing model has FUNDAMETALLY changed from Office 2010 to Office 365 (2013).  2010 was licensed to specific machines it was installed on. 2013/365 is now licensed to specific email accounts.  Office 365 now has an online web page that allows users to easily manage the up to 5 concurrent machines their license is installed on (it really is easy to install and uninstall 365 on various machines).

The product key you have is of no use to you if they have already used it to activate the license using their own email account.  The 365 product key allows one-time use only, unlike Office 2010 and earlier product keys.

I know of no process to change the email account, which is what you are asking to do. Many people have asked about that after accidentally using the "wrong" email account . MS has simply not allowed for that in the licensing system

The only way I know that they can give you the 365 license is by giving you the email address and password they initially used with the product key.  Then you can reset the password for the remaining time on the license.

If it is still in the first month of the subscription, you could try calling MS Support, ask for a "supervisor", and if you are lucky (after wasting a lot of time and being disconnected several times <sigh> ) you may find someone who is willing to cancel the subscription and issue you with a new product key.  Other than that they own the license for the rest of the year.

You may use either of below:

=IFERROR(LOOKUP(U3,{0,2.5,4.5,8,10,12},U3-{0,0.25,0.5,0.6,0.7,"NA"}),"")

or

=IF(U3<0,"",IF(U3<2.5,U3,IF(U3<4.5,U3-0.25,IF(U3<8,U3-0.5,IF(U3<10,U3-0.6,IF(U3<12,U3-0.7,""))))))

Pls note the change done to account for values between 9.9 & 10, etc. The above formulas consider <12, <10, <8, <4.5 & <2.5. Any negative value or value of 12 & above will return a blank.

I cant connect my Lumia 710 bluetooth with other mobiles

steven _fyiguy_ hughes say:

Unfortunately the only Bluetooth profiles Windows Phone currently supports are the following:

  • Advanced Audio Distribution Profile (A2DP 1.2)
  • Audio/Video Remote Control Profile (AVRCP 1.3)
  • Hands Free Profile (HFP 1.5)
  • Headset Profile (HSP 1.1)
  • Phone Book Access Profile (PBAP 1.1)

If you are trying to connect to transfer files, etc. via Bluetooth it won't work.

 

Hope this helps.

damiendeery say:

HI ParadronMedic,

Looks like this problem came about when 2013 was launched... I've been using 2010 since recently hence why I have just come across this now. By now you have probably resolved the issue; this is just to help anyone else that comes across it.

I was typing up a document and my MS Word was auto-correcting minimise and utilise to ize. It was very frustrating as I was sure my PC and Word language settings were on Australia. I then opened a new document and did the "color/colour" test from above. In the new doc, color was deemed incorrect however colour got through the spell check fine!

I realised that the reason my MS Word was auto-correcting in the other document is because I downloaded a template which had it's own language setting. So if you are working from a template; I reckon you are having the same problem.

All you need to do is click on the "Review" tab --> Language --> Set Proofing Language --> English (Australia)

So if you create a new document it will be in Australian language.

I believe you can check other documents by doing this but it's fiddly and it will revert back  to US. It will however have Australian English to new documents you create.
