How do I find pictures that have a resolution less than 1920*1200 - Microsoft Community

ethanhenderson say:

How do I find pictures that have a resolution less than 1920*1200 using Windows 8 search?

I'm trying to find pictures in a folder than have a resolution less than 1920*1080, but can only find how to search for specific sizes - which is useful, but not what I need.

Any help is appreciated.

shenan stanley say:

Don't over think it - just try the logical.  :-)

Search for the width and height...

Example - if I had a folder (with subfolders even) I wanted to locate all pictures in that had a width of less than (but not equal to) 1920 (which is what you said, halfway anyway) then I would search for:

width:<1920

If I wanted all pictures with a width less than 1920 and height less than 1080, then I would search for:

width:<1920 height:<1080

(... which is what you said you wanted.)

Of course that does not take into account rotated images where the width is now 1080 and the height is now 1920.  ;-)

If I wanted all images that were less than or equal to a measurement...

width:<=1920 height:<=1080

Etc. and so on...

All this assumes the pictures have been properly encoded with metadata, indexed, etc.

It also can take some time to process...

Worked since Windows XP I think.

What you might see in a result pane:

That picture is Windows 7 - because that happens to be what i was sitting in front of at the time.  Should be very similar/the same for a file search (FILE SEARCH is key there) in Windows 8.1 ...

Note that I added the Width and Height to the columns for easy sorting - heck, that could solve your problem right there.  Go to the root directory (has all the sub-directories included in it you might want pictures out of) and search for *.* and with the Width and Height columns add, once the search results show all the files ion the folders - click on one of the column headings to sort (say by Width) and you will find a list of pictures in all folders below the one you started in that have a width of whatever grouped together, etc.

  

But - you seemed to want to put the width & height together, so...

dimensions:<1920x1080

However, in my testing, it is less reliable than the Width and Height - as I got less than or equal to 1920 and the height was whatever it felt like.

Might look through this:
http://www.eightforums.com/tutorials/4595-search-windows-8-a.html

Examples of ways to search:

http://arstechnica.com/information-technology/2010/03/mastering-windows-search-using-advanced-query-syntax/
(older article, still valid - both pages - and valuable - in my opinion.  Using Boolean operators and properties mixed with the many things you can search for makes searching in Windows/File Explorer a very powerful tool.)

  

Although - when it comes to file searching - I still live an old reliable friend:
Agent Ransack

  

FileLocator Pro is the professional version (and cost version) of the Agent Ransack tool.  Nice product, well worth he money if you do a lot of file searches.

  

If you do a lot of mass renaming of files, I have found one freeware that has been VERY user friendly and reliable/flexible for me for years:  Advanced Renamer
  

Overall - for most people - the built in searching capabilities of Windows and even the renaming capabilities are more than enough.  Here's a decent run-down of the AQS (Advanced Query Syntax):
http://msdn.microsoft.com/en-us/library/aa965711(v=vs.85).aspx
