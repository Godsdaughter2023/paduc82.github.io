olivierasselin say:

Alright so I've solved my issue.

I tried creating a new user and transferring the files, but this created a whole new problem, where I wasn't able to run explorer.exe for some reason, which led me to the following article :

https://social.technet.microsoft.com/Forums/lync/en-US/6bbdd2df-5bca-453c-9dc6-4579610fcc1d/explorerexe-windows-can-not-access-the-specified-device-path-of-file-windows-7?forum=w7itproperf

About 3/4 down the thread, user pedk proposes the following :

"I found that there were a whole string of UsrClass.dat files in the AppData\Local\Microsoft\Windows directory.  When I deleted these, suddenly Explorer started working again!"

I tried that, which solved my explorer issue, and then thought I'd just try that on my original problematic user account, and bingo!

Essentially there was just one UsrClass.dat file for the original problem user account, so I moved that to a backup folder and restarted Windows.

All good now but still no idea what might have caused the problem in the first place.
