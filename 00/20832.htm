ossiemac say:

Is this Find method broken?

IMHO it has always been broken.

So what I do with the VBA code if I have formulas creating the dates is to set the NumberFormat of the column to Numeric, Find the date with xlValues and then convert the NumberFormat back to the previous format.

I usually save the format of the first data cell after the column header for the Existing NumberFormat and then re-apply that format to the entire column at the end of the code unless there is good reason not to do it this way.

However, use a Long type variable if only dates without times included.

Private Sub Find_Today()
    Dim strSaveFormat As String
    Dim firstAddr As String
    Dim c As Range
    Dim testDate As Long
   
    With Worksheets("Sheet1")
        strSaveFormat = .Cells(2, "I").NumberFormat
        .Columns("I:I").NumberFormat = "0"
       
        With .Range("I2:I365")
           testDate = Date - 1
       
           Set c = .Find(What:=testDate, LookIn:=xlValues, _
                           LookAt:=xlPart, _
                           SearchOrder:=xlByRows, _
                           SearchDirection:=xlNext, _
                           MatchCase:=False)
       
           If Not c Is Nothing Then
               firstAddr = c.Address
               Do
                   Set c = .FindNext(c)
               Loop While Not c Is Nothing And c.Address <> firstAddr
           End If
        End With
       .Columns("I:I").NumberFormat = strSaveFormat
    End With
 End Sub

A hyperlinked page number is a PageRef field, and this field doesn't represent an absolute number but rather the page number of a bookmark, heading, numbered paragraph, caption, etc. So no, there is no way that this could be achieved using Replace All because bookmarks have to be created (by you or by Word) for the pages being cross-referenced.

jon - vertex42 say:

To assemble the name of a file using a number in cell A1 and concatenation, you could do something like this:

A1 = 13, A2 = 14, A3 = 15, etc.

B1 = "'\\filepath\[INV16-"&TEXT(A1,"0000")&".xlsx]Sheet1'!$B$19"

Then, you could use B1 within the INDIRECT() function to reference the value IF the workbook was open:

C1 = INDIRECT(B1)

The problem is that INDIRECT() will only reference values in other files if the files are OPEN (a major drawback to using INDIRECT).

So, instead of using INDIRECT(), you could include the "=" sign in the formula for cell B1 like this:

B1 = "='\\filepath\[INV16-"&TEXT(A1,"0000")&".xlsx]Sheet1'!$B$19"

Then, copy that formula down. This will create the text for the reference formulas, but won't execute them. Then copy the range of cells containing those formulas and then use Paste Special > Values to change them to just text. Then starting with the first one, press F2, Enter, F2, Enter,  etc. really fast to turn the text values into working formulas.

(Sometimes, a bit of brute force is faster than trying to develop a special VBA solution if what you are doing is a one-time thing.)
