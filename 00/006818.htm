rich007a say:

Firstly, do you understand the difference between a Slide Master and a Layout?  When in “View tab > Slide Master” view, in the thumbnail pane, you will see the Slide Master(s) are larger, and each Slide Master will have child Layouts, slightly smaller and indented in from the left.  Clear so far?

 

When you are working with slides in your Presentation, you apply a Layout (not a slide master).  The Layout you use will include the graphics, etc. that are on its parent Slide Master.  So you need to set up your Slide Master with what you want to appear on EVERY slide (e.g. company logo), and then create layouts for that Slide Master with the required arrangement of Placeholders to contain content on your slides.

 

So, open your existing design, do View tab > Slide Master, then right-click on your Slide Master and Copy.

 

Now create a new presentation from Blank.

 

In Normal view, delete all the slides.

 

Now do View tab > Slide Master.

 

Click just above the top slide master in the thumbnail pane.  Right-click > Paste.  Your Slide Master will appear at the top with all the existing child Layouts below it.  You can now scroll down and delete any unwanted Slide Masters, and their Layouts, and any unwanted Layouts belonging to your new Slide Master.  (You couldn’t delete some before (you said some are “hard wired”) because they were being used by slides in your template, now you’ve deleted all the slides, that should not be a problem).

 

Create as many Layouts as you think you will need, remember that only Placeholders can handle formatting of multi-level bulleted lists (textboxes can’t), so make Layouts with all the options for Placeholders that you might need (Slide Master tab > Insert Placeholder).

 

Tip: In an empty presentation, the Layout in the first position will be the default for the first time you click New Slide (so you could put your title slide Layout there).  Subsequent clicks of New Slide will use the Layout in the second position under the Slide Master, so put your most used Layout in the second position.  That said, when you click New Slide in a Presentation that already contains other slides, the new slide will use the same Layout as the preceding slide.

 

I recommend keeping to only one Slide Master.  Have a read of this thread to understand why: http://answers.microsoft.com/en-us/office/forum/office_2007-powerpoint/copying-slides-between-presentations-and/1f4d46bb-c60e-4907-9f0f-a43c06c766e6?msgId=66584c60-4c9e-4353-a2d2-670a60d4fa58

 

Final tip, be sure to Preserve your slide master.  In Slide Master view, right-click on the Slide Master thumbnail, and ensure Preserve Master is selected.  This stops your Slide Master being automatically removed when you delete all slides/layouts that use it.

 

Save as *.potx and away you go.

 

Hope that helps.

Cheers
Rich

PS. The above is all true in 2007.  AFAIK it hasn’t changed in 2010.

Right now the order is somewhat messed up, and it would probably print in an order like "1, 2, 4, 3, 5, 7, 6, 8", whereas I would like it to print in a chronological order "1, 2, 3, 4, 5, 6, 7, 8".

Is this possible to adjust the macro, so it will print the attachments in order? :)

My macro is as listed below:

Sub BatchPrintAllAttachmentsinMultipleEmails()
    Dim objFileSystem As Object
    Dim strTempFolder As String
    Dim objSelection As Outlook.Selection
    Dim objItem As Object
    Dim objMail As Outlook.MailItem
    Dim objAttachments As Outlook.Attachments
    Dim objAttachment As Outlook.Attachment
    Dim objShell As Object
    Dim objTempFolder As Object
    Dim objTempFolderItem As Object
    Dim strFilePath As String
 
    Set objFileSystem = CreateObject("Scripting.FileSystemObject")
    strTempFolder = objFileSystem.GetSpecialFolder(2).Path & "\Temp for Attachments " & Format(Now, "YYYY-MM-DD_hh-mm-ss")
    'Create a new temp folder
    MkDir (strTempFolder)
 
    Set objSelection = Outlook.Application.ActiveExplorer.Selection
 
    For Each objItem In objSelection
        If TypeOf objItem Is MailItem Then
           Set objMail = objItem
           Set objAttachments = objMail.Attachments
 
           'Save all the attachments in the temp folder
           For Each objAttachment In objAttachments
               strFilePath = strTempFolder & "\" & objAttachment.FileName
               objAttachment.SaveAsFile (strFilePath)
 
               'Print all the files in the temp folder
               Set objShell = CreateObject("Shell.Application")
               Set objTempFolder = objShell.NameSpace(0)
               Set objTempFolderItem = objTempFolder.ParseName(strFilePath)
               objTempFolderItem.InvokeVerbEx ("print")
           Next objAttachment
        End If
    Next
End Sub

Hi,

Unfortunately the SUBTOTAL function won't work on rows, only on columns of data so here's a VB solution.

ALT+F11 to open VB editor, right click 'ThisWorkbook' and insert module and paste the function below in,

Back on the worksheet call with

=Sumvisible(A1:E1)

It will sum visible cells only

Function SumVisible(rng)
Dim c As Range
Application.Volatile
SumVisible = 0
For Each c In rng
If IsNumeric(c) Then
If Not c.EntireRow.Hidden And _
Not c.EntireColumn.Hidden Then _
SumVisible = SumVisible + c
End If
Next
End Function
