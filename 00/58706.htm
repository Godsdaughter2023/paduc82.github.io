luismarsano say:

Found a solution on the Windows Server forums right away, tried it, and it worked.

However, I'm shocked at the solution, and frankly don't like it.

Here it is: turn off Secure Boot.

If you go to the UEFI menu and disable Secure Boot, then the updates will install without incident.

You can then re-enable Secure Boot.

So that system Microsoft instituted to lock down the boot environment in a way that inconsiderately denies PK management by the user (so they can decide what to sign and authorize) is broken by Microsoft's own update.

If you're getting an "update failed due to error code 80246007" message, try disabling Secure Boot and see if that works.

You should always re-enable Secure Boot to prevent malware from exploiting boot time vulnerabilities.
