albert d. kallal say:

There are different limitations if you talking about macros in a web form as opposed to a client application.

 

However, the answer is yes. As a general rule you can. Just don't use the forms qualifier, but JUST use the name of the subform control (this control name is "usually" the same name as the sub form, but it does not have to be, so this issue can often trip you up).

 

So a typical reference to a sub form will look like this:

 

[MySubFormControl![MyControlOnSubForm]


Note how you don't need any [Forms]! part in front if the code is running in the parent form.

Also note that for a macro in a web form, while you can reference and grab values from a sub form, you cannot "set" values from that parent form to the child form like you can with client forms.

 

So what limitations exist are not only due to you use a macro, but if you building Web forms with Access 2010.

 

So the answer is yes, you can grab values, but depending on the type of form (web or client), this does not mean you also set those values.

 

As a general rule you thus don't need to place forms! in front.
