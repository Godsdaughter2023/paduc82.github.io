jay freedman say:

There are a couple of places to look. Note that files in these locations are never removed by ordinary uninstallation of Office (on the theory that you might be reinstalling afterward and want the customizations stored there -- even though the most likely reason for an uninstall/reinstall cycle is precisely to remove corrupted customization!).

With Word shut down, enter the following in the address box of Windows Explorer:

%appdata%\Microsoft\Document Building Blocks

and press Enter. The environment variable %appdata% automatically expands to the Roaming folder in your Windows profile. Look for the file Building Blocks.dotx in that folder or one (or more) of its subfolders. Delete any that you find. The next time you use a header, footer, cover page, or any of the Quick Parts (building blocks), Word will create a fresh copy of the file from the Office installation in the Program Files folder.

Next, enter

%appdata%\Microsoft\Word\Startup

and press Enter. Delete any files you find in that folder, which is where add-in templates are stored. If you had any legitimate add-ins, reinstall them from a known-good source.

Word 2013 not saving default font preferences - Microsoft Community

zereph say:

Word 2013 not saving default font preferences

I have Word 2013 as part of Office 365 through work. When I try to change my default font to Times New Roman it's fine as long as Word is open but if I close Word and open it again it changes back. I've tried changing the style and I am clicking the use as default button but it still doesn't stick. I even tried creating my own style but it disappears once I close and reopen. Help please.
jay freedman say:

Something is preventing Word from saving changes to the Normal.dotm template, which is where settings such as the default font are supposed to be stored. Usually that something is a poorly written add-in. One of the known culprits is PDF Complete.

As a workaround, you can open Normal.dotm itself, make the change there, and save it. The easiest way to get Normal.dotm opened properly is:

  • In Word, press Alt+F11 to open the macro editor, and press Ctrl+G to open the Immediate pane at the bottom right of the editor.
  • In the Immediate window, type or copy/paste NormalTemplate.OpenAsDocument and press Enter.
  • Close the macro editor. The Word window now contains Normal.dotm (you'll see its name in the title bar at the top).

A permanent fix is to disable the add-ins (in the dialog at File > Options > Add-Ins, open the Maange dropdown; select Templates and click the Go button; and uncheck one of the items in the list) one by one to see whether changes can be saved when you click the Default button.*

When you find one for which disabling it allows saving, decide whether to uninstall that add-in or try to find an updated version of it.

*Note: If you uncheck an add-in to disable it and then restart Word, it will be checked again in the new session. The way to permanently disable an add-in is to remove it from the %appdata%\Microsoft\Word\Startup folder, either deleting its file or moving it to a different folder.
