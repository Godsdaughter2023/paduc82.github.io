karl timmermans [outlook mvp 2012-2018] say:

That is catastrophic - there is data I must have in that contacts file!

I guess I'll have to try rebuilding the PC with the old drive and then exporting the contacts I need to a .pst file   

Just to summarize things

#1 - Your "default" contact/calendar folders are located in the "default data file" for the profile which generally is going to be the data file associated with the "first" email account configured. Since you have a contact folder located in a folder labeled with the words "This Computer Only" - means that the first account configured was the IMAP account. Any subsequent POP3 or IMAP accounts would not have any contact/calendar folders in their folder tree (exception to this are any Outlook.com/Exchange accounts which sync all folder types with the server)

#2 - You really do NOT want to store any info in folders labeled as "This Computer Only" for several reasons

a) if you remove the IMAP account - the .ost file associated with the account will also be deleted from the system

b) if Outlook can't find/open the .ost file associated with the account, it will recreate the file and sync the email folders with the server but the "This Computer Only" folders will be empty (ergo - that data will be lost) 

c) even if you backup up an .ost file by making a copy of it, it can only be opened/read by the original Outlook profile/email account that created it. To access the data in the ost file outside of Outlook requires a 3rd party OST2PST conversion tool

You would be far better off maintaining your contacts in your Outlook.com (Hotmail) account which not only prevents the loss of data experience but also lets you share that data across devices in case that's ever of any interest. Would suggest configuring the Hotmail account as the first one when reconfiguring Outlook but if everything is already configured, go to File --> Account Settings --> Data Files tab --> select the .ost file for the Hotmail account --> Set as default

PowerPoint is not designed for print production. It's designed to print slides on a sheet of paper with borders around it. That's how all the preset slide sizes work. But you can get around this by creating a custom page size that is the same size as the paper you will use for the booklet.

Here's a formula based solution

1. Assume there are three worksheets named Jan, Feb and March
