Create macro for "Go to next sentence" in Word - Microsoft Community

bobby_960 say:

Create macro for "Go to next sentence" in Word

Up until recently I used a very old version of MS Word for one single reason: It had the possibility to use shortcuts for "Go to next sentence" and "Go to previous sentence", with the added possibility to select by pressing shift at the same time. 

Now in the current versions "Go to next sentence" et al is no longer an option, so my question is:

- if and how one could create macros for "Go to next sentence", "Go to previous sentence" as well as selecting the text between your original position and the end (or beginning) of the sentence by adding Shift.

I use Mac.

Thanks!

a. user say:

Word already has some commands to help you move/select sentence by sentence but by default, they do not have any keystrokes assigned. When I say "sentence by sentence," these are the things that Word considers to be sentences.

You can assign some keystrokes using

Word->Tools->Customize keyboard...

Under "Categories", select "All Commands"

In the search box under "Commands", type "sent". You should see a number of commands including SentLeft, SentLeftExtend, SentRight and SentRightExtend

For each command you want, select the command, click in the "Press new keyboard shortcut:" box, press the shortcut combination you want (e.g. I used control-option with the 4 arrow keys), check it's not already used for something you want, then click the Assign button.

If you need more than that, you could try the following very simple macros. Again, they work with Word's notion of what a sentence is. They will probably need work to do exactly what you expect. For example, if you have a gaphic selected, one or two of the options will probably have an unexpected effect, such as rotate the graphic.

How to create and use them? Well, there are a lot of steps, but I would do this:

Use Word->Preferences->Ribbon and Toolbar, and ensure the Ribbon button is pressed. Then check the box next to "Developer" in the right-hand column. (You don't have to do that but it can make things easier).

Then use Word->File->New document to create a new document. 

Click File->Save As, then use the File Format dropdown to select "Word Macro-Enabled Template (.dotm)". Word should change the folder to the location where all such templates really need to go in Mac Word 2016. Change the document name (e.g. let's say you call it "mymacros.dotm") and save it.

Click the Developer tab, then the Visual Basic icon at the left. The Visual Basic (VB) Editor should display. Please note that the VB editor in Mac Word 2016 is currently very crude compared to the versions in Windows Word and Mac Word 2011. 

The left hand column should list some "projects" - e.g., you should see "Project (mymacros.dotm)".

Right click on that project, mouse over the "Insert" button that pops up, and click "Module".

Click on the little right-pointing triangle to the left of the Project to open up the project's list of objects. You should see "Modules". Click on the little triangle to the left of that, and you should see "Module1". Click on that. 

Select the macro code I have provided below and click command-C to copy it to the clipboard. In the VB Editor, click in the blank box at the top right (under the windows' title bar) and use Command-V to paste the code. If you have a document open, you can try running the macros by
 - clicking in a macro
 - click the "start" button below the code pane - it's the 2nd one from the left, looks a bit like |>
 
Click in the actual .dotm so you get the normal Word menu back (rather than the VB Editor menu), and save and close the .dotm file.

Click Word->Tools->Templates and Add-Ins...

Click the + button under the Global Templates and Add-Ins box.

Select myaddins.dotm from the Finder list, and click the Open button. The template should appear in the list. Check its box, and click OK.

Finally, you can go into Word->Tools->Customize keyboard..., and select "Macros" under "Categories". You should see the 4 macros in the list on the right. Assign keystrokes to them in the same way you did for those "SentLeft" commands earlier.

Unfortunately, this isn't quite the end of the story, because if you quit and restart Word, your macros won't work any more. A quick look at the Global Templates and Add-Ins box will show that the mymacros.dotm template is still listed, but its box is not checked. You can check it, but to have your macros enabled without that step, you need to move the .dotm from the "ordinary" templates folder to Word's startup folder. For a user called "fred", you would need to move the .dotm from 

/Users/fred/Library/Group Containers/UBF8T346G9.Office/User Content/Templates

to

/Users/fred/Library/Group Containers/UBF8T346G9.Office/User Content/Startup/Word

(you may also see ".localized" in some of these path names, e.g.

/Users/fred/Library/Group Containers/UBF8T346G9.Office/User Content.localized/Startup.localized/Word

As far as I know, you can ignore that for now.)


The macros...

  Selection.MoveRight unit:=wdSentence, Count:=1, Extend:=wdExtend

End Sub

  Selection.MoveLeft unit:=wdSentence, Count:=1, Extend:=wdExtend

End Sub

  Selection.MoveRight unit:=wdSentence, Count:=1, Extend:=wdMove

  Selection.MoveRight unit:=wdSentence, Count:=1, Extend:=wdExtend

End Sub

  Selection.MoveLeft unit:=wdSentence, Count:=1, Extend:=wdMove

  Selection.MoveLeft unit:=wdSentence, Count:=1, Extend:=wdExtend

End Sub
