Doch. Firewalls bei (Heim-)Routern sind eigentlich "nur" Marketing-Blödsinn.

Der Router ist auch mit ausgeschalteter Firewall IMMER NOCH eine Firewall, da ungefragt ankommende Pakete ohne Weiterleitungsregel GENERELL immer geblockt werden.

Eine Firewall bräuchtest Du eigentlich nur, wenn Du beispielsweise in einem Firmenumfeld einen Server hinter dem Router stehen hast, der NUR für eine Zweigstelle Deiner Tochterfirma zugägnlich sein soll, aber für andere Internetteilnehmer gesperrt sein soll (also eine Regel wie: "Öffne Port 4711 und leite das an den Server weiter, aber erlaube NUR der IP 123.45.67.89 den Zugriff auf diesen Server, sonst niemand".

Im Heimbereich möchtest Du aber quasi "mit der ganzen Welt" spielen. Also richtest Du eine Portweiterleitung ein, die es "der ganzen Welt" erlaubt NUR Xbox Live Daten NUR an Deine Konsole zu schicken. Alles Andere wird auch ohne Firewall von dem Router geblockt, weil Du dem Router nicht ausdrücklich gesagt hast, was er mit "fremden" eingehenden Paketen tun soll.

Also: Ruhigen Gewissens die Firewall des Routers abschalten. Solange Du nicht den Datenverkehr auf BEKANNTE statische IP Adressen im Internet einschränken willst, bringt Dir die Firewall am Router keine sinnvolle zusätzliche Sicherheit.

Lies dazu auch den kleinen Absatz "Zu restriktive Firewall oder IDS (Intrusion Detection System)" in meinem NAT Post unten in meiner Signatur.

cmdrkeene say:

There's no real shared "network name".  Any computer that joins the wifi or network will come up with it's own name there, it's more of a local profile that remembers the topology and settings for that network "location".  

The more important bit is that all computers say "private network" there.  If it doesn't, most sharing/homegroup features won't run.

In any case, I'd run the built-in homegroup troubleshooter diagnostic tool from each computer, one at a time.  The tool is actually just a collection of PowerShell commands that resets and re-initializes all the Homegroup components, network location privacy settings, and re-broadcasts the discovery packets to the network.  If you're really nerdy, you can even look inside at what the tool does
