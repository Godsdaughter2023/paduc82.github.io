marcine lohman say:

I have found a solution that seems to work for me but it is only for Windows 7.  Our problem was with the Windows 7 Credential Vault.

 

Windows 7:
 1. Close Outlook.
 2. Click Start.
 3. In the search box enter "Credential". You will then see Credential Manager.
 4. Select Credential Manager.
 5. Under Generic Credentials, you will most likely see an entry that has Outlook in the name.
 6. Delete it. In fact, you can delete all the entries in Generic Credentials if you like.
 7. Close the Credential Manager window and open Microsoft Outlook.
 8. If it asks for a password, enter the password and check for it to remember the password.
 9. A new entry will be created in the credential vault with your latest login and password.
 10. Problem fixed.

 

The problem for us has been that the saved password in the credential vault file doesn't appear to work (especially if we've changed the password recently) so by deleting it, another credential will be created the next time you log into Outlook with your latest password.

 

I hope this helps.

Skype numbers are landlines. Call forwarding in Skype is performed at the account level, not the number level. If an external call is forwarded to your UK number or US number, it will forward to that single Skype account, if purchased under one account. If they were purchased under separate accounts, they will forward to their respective Skype accounts. Any time you forward calls out of Skype across POTS, it will require credit or an outgoing subscription to cover the cost of the calls. That service has never been free using Skype. If the forward being made from Skype's end is to a UK landline (like a UK Skype number), you could use an unlimited UK landline subscription to cover that cost. If you have separate accounts, you can forward to that separate account by Skype name for free. That stays within Skype's  system, not requiring POTS.

The following works for me

Private Declare Function UnhookWindowsHookEx Lib "user32" _
                                             (ByVal hHook As Long) As Long

Private Declare Function GetCurrentThreadId Lib "kernel32" () As Long

Private Declare Function SetWindowsHookEx Lib "user32" _
                                          Alias "SetWindowsHookExA" _
                                          (ByVal idHook As Long, _
                                           ByVal lpfn As Long, _
                                           ByVal hmod As Long, _
                                           ByVal dwThreadId As Long) As Long

Private Declare Function SetWindowPos Lib "user32" _
                                      (ByVal hwnd As Long, _
                                       ByVal hWndInsertAfter As Long, _
                                       ByVal x As Long, _
                                       ByVal y As Long, _
                                       ByVal cx As Long, _
                                       ByVal cy As Long, _
                                       ByVal wFlags As Long) As Long

Public Declare Function GetWindowText Lib "user32" Alias "GetWindowTextA" _
                                      (ByVal hwnd As Long, _
                                       ByVal lpString As String, _
                                       ByVal cch As Long) As Long

'Message box title
Private MsgBoxTitle         As String

' Handle to the Hook procedure
Private hHook                 As Long

' Position
Private msgbox_x              As Long
Private msgbox_y              As Long

' Hook type
Private Const WH_CBT = 5
Private Const HCBT_ACTIVATE = 5

' SetWindowPos Flags
Private Const SWP_NOSIZE = &H1      ' Retains the current size
Private Const SWP_NOZORDER = &H4    ' Retains the current Z order


Public Sub MsgBoxPos(strPromt As String, _
                     vbButtons As VbMsgBoxStyle, _
                     strTitle As String, _
                     xPos As Long, _
                     yPos As Long)

    ' Store position
    MsgBoxTitle = strTitle
    msgbox_x = xPos
    msgbox_y = yPos

    ' Set Hook
    hHook = SetWindowsHookEx(WH_CBT, _
                             AddressOf MsgBoxHookProc, _
                             0, _
                             GetCurrentThreadId)

    ' Run MessageBox
    MsgBox strPromt, vbButtons, strTitle
End Sub

Private Function MsgBoxHookProc(ByVal lMsg As Long, _
                                ByVal wParam As Long, _
                                ByVal lParam As Long) As Long
    If lMsg = HCBT_ACTIVATE Then
        If GetWinCaption(wParam) = MsgBoxTitle Then
            ' Change position
            SetWindowPos wParam, 0, msgbox_x, msgbox_y, _
                         0, 0, SWP_NOSIZE + SWP_NOZORDER

            ' Release the Hook
            UnhookWindowsHookEx hHook
        End If
    End If

    MsgBoxHookProc = False
End Function

Public Function GetWinCaption(WinHdl As Long) As String
    Const BufSize             As Long = 255
    Dim Buffer                As String * BufSize    'Use fixed-length buffer
    Dim CaptionLen            As Long

    CaptionLen = GetWindowText(WinHdl, Buffer, BufSize)    'Load the buffer
    If CaptionLen > 0 Then GetWinCaption = Left(Buffer, CaptionLen)    'Truncate to actual size
End Function

And then you simply do:

MsgBoxPos "MsgBox Text", vbOKOnly, "MsgBox Title", 400, 500

jan karel pieterse say:

Are all the linked files and the file with the links on the thumb drive, whereever you open them from? If so, it should be relatively easy to write a macro that relinks the files when you open the linked file:

Sub RelinkFiles()
    Dim vLinks As Variant
    Dim lLink As Long
    vLinks = ThisWorkbook.LinkSources(xlExcelLinks)
    For lLink = 1 To UBound(vLinks)
        ThisWorkbook.ChangeLink vLinks(lLink), Replace(vLinks(lLink), Left(vLinks(lLink), 1), Left(ThisWorkbook.Path, 1)), xlLinkTypeExcelLinks
    Next
End Sub

E65 software update - memory card locked - Microsoft Community

abusalem say:

E65 software update - memory card locked

i did update my e65 software with new firmware through nokia site and it was perfect unless it comes after restart the phone set to ask about the memory card lock password which i dont create it before.
what shall i do to recover it.

bhumbro say: For S60 1st and 2nd Phones

For S60 1st and some of 2nd phones, such as Nokia 3650, Nokia 6600 and Nokia N-Gage, the password is stored in a plain text file.

* Firstly, you need to install FExplorer, a free file manager that allows us to browser through S60 file system.
* Next, launch FExplorer and then browser to c:\system folder. There is a file, called mmcstore.
* Rename the file to mmcstore.txt and then open it. Your password must be several characters at the end of the file.

For S60 3rd Phones, Nokia N-Series and Nokia E-Series

Solution One
---------------
This solution for Phone that have execution file name : unlockMMC.app on folder z:\system\app\
Note :
Not all S60 Phone have this execution file.
To unlock Memory Card :
- Run execution file unlockMMC.app
- Enter new Password you like (example "12345")
- Run app Memory (Phone app Memory Standard)
- Choose To Unlock MMC with that new password

Solution Two
----------------
Every times you set to lock ur Memory Card with lock password , phone made a file to save your password with file name mmcstore - you can find ths file in folder c:\system\.
Note :
To see the file you must set ur file browser app to show hidden files.

To get your Lock Password :

- Send that file into your PC
- Change file extension with .txt
- Open that file using app Notepad
- You will see your lock password
or
(This way not always success)
- Make sure that your phone have app Notepad
- Copy that file into other folder
- Change its attribute not a system file and not a hidden file
- Change its extension with .txt
- Run app Notepad on your Phone
- Open that file
- You will see your lock password

Solution Three
------------------
Use app for Break Memory Card Password - this app name MMCPWD
You can find this app using this link :
http://imserba.com/index.php?name=PN...&start=360
To get your Lock Password :
- Install app MMCPWD into your phone
- Run app MMCPWD
- Choose Option then choose Restore
- You will see your lock password

otherwise
The only way that I know so far is by reformatting the memory card. It can be done from Memory Card application by selecting Options | Format memory card menu. Of course, you will lose all your data in the memory card… :(
