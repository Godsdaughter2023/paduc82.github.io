OLE links like the ones you're working with require full paths and will not allow you to set fake paths.

It used to be that if the file wasn't found on the path specified, PowerPoint would look for it in the same folder as the PPT itself, so long as you hadn't opened the file from the Most Recently Used Files list.

MS seems to have thought better of making software work and broke this feature a couple versions back.

Workarounds:

Embed rather than link the file.  Yeah.  I know.  Bad idea.

Put the PPT and files to be linked into a folder that another user can easily re-create ... C:\ExcelFiles for example.  Have them put all the needed files there and they'll be good to go.
