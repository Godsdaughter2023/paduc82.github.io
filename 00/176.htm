Excel to Word Mail Merge Format Problems (Office 2010) - Microsoft Community

gabrielin07 say:

Excel to Word Mail Merge Format Problems (Office 2010)

I am mail merging an excel doc to a word doc for personal letters. After executing the merge, everything is fine except one thing; the date format. I typed in the excel formula bar (10/30/17) and changed the format to what I want, (October 30,2017), but when it merges the letter displays: 10/30/17 not October 30,2017. I realize that I can type what I want in the formula bar and that will change it, but that will be too time consuming for the current process. How can I get the formatting to merge? Thank you in a advance for any assistance.  

a. user say:

By default, Word gets its data from Excel using a thing called OLE DB. OLE DB ignores Excel's formatting and passes the date to Word  in US date format (M/D/YYYY).


To get the format you want, you should format the fields that you use to insert the date in the Word document. e.g. if you inserted a date field called "mydate" in your Word Mail Merge Main Document, you should be able to use Alt-F9 to toggle between "field code" view and "field results" view until you see something like { MERGEFIELD mydate } (it may say { MERGEFIELD mydate \*Mergeformat } but you can almost always delete the \*Mergeformat bit).

Change that to { MERGEFIELD mydate \@"MMMM D, YYYY" }, use Alt-F9 to show the results, select the field and press F9 to update it.

NB, that should work correctly even if your regional date settings are not in the US format - Word knows which number is the month number and which is the day number in this scenario.
