ryanlv_msft1519 say:

Hi Jeffrey,

As SharePoint Online is a cloud service and the total resource quota are automatically allocated, we have no documentation listed for those technical specs. So we use Server Resource Quota to represent the computing capability.

For managing SharePoint Online regarding Server Resource Quota aspect, we’d recommend you follow the two suggestions below:
1. Allocate a larger quota to sites where there is heavier usage, planned code deployments, heavy app consumption, or heavy collaboration
2. Set an alert for sites so that when a site exceeds a certain percentage of server resource quota, you will receive a notification email and assign more resource quota.
