david_g---01 say:

Ok... I have answered my own question. A HATE it when I have to do that!

 

The answer to your problem (burglar alarm sending to SIM) is that the sending device (presumably a modem of some sort) has set the wrong message type.

 

The Data Coding Scheme setting should be for a class 1 message (modem command AT+CSMP will set this) for which the GSM 03.38 specification says...

 

When a mobile terminated message is Class 1, the MS shall send an acknowledgement to the SC when the message has successfully reached the MS and can be stored. The MS shall normally store the message in the ME by default, if that is possible, but otherwise the message may be stored elsewhere, e.g. in the SIM. The user may be able to override the default meaning and select their own routing.

 

or in simple language, the text message is sent to the "mobile equipment" (ME) and not to the SIM card.

don varnau say:

I don't know how you "downloaded" IE11 since IE11 came preinstalled on Windows 8.1, but...

On Desktop IE11, not modern/metro.

Try these procedures in order. Check after each. You may not have to do the reset.

1. Internet Options> General tab> Browsing History> Delete> uncheck "Preserve Favorites website data" then delete Temporary Internet Files. No reason, yet, to delete cookies, history,  passwords, etc.

2. Internet Options> Advanced> use the Reset IE Settings button. Don't yet check "Delete personal settings."

3. Reset IE Settings again. This time delete personal settings.
