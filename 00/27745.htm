Replace text as you type incorrectly matches all caps - Microsoft Community

bartb_42 say:

Replace text as you type incorrectly matches all caps

"Replace text as you type" incorrectly matches all caps. This happens even when "Ignore words in UPPERCASE" is selected.

Note: this same issue has been brought up for Office 2016, but the proposed workaround only solves for some users.

http://answers.microsoft.com/en-us/office/forum/office_2016-officeapps/2016-proofing-tool-not-ignoring-all-caps/85eab532-9cb4-4eb0-988e-b5e1976589b5

I describe this issue as seen in Office Professional Plus 2013.

I am often using "replace text as you type" to add needed accents into various foreign names.  For example, Francois -> François

If I type all lowercase, it (correctly) does not match, so that:
   francois -> francois

However, if I type all uppercase, it (very incorrectly) replaces the name and removes the uppercase:
   FRANCOIS -> François

This is quite clearly a bug in the AutoCorrect, and because sometimes users need to type names in all uppercase, this bug in Office autocorrect can be very problematic. 

The only workaround that I've found is to:
* go into Autocorrect options, 
* uncheck the "Replace text as you type", 
* close the options dialogs, 
* type the word in uppercase where it is needed, 
* then go back into Autocorrect options and re-enable the feature,
* then to add insult to injury, a complete shutdown of all Office applications (and re-opening) is necessary to actually get the feature enabled again (thus, a second bug regarding that).
As I'm sure we can all understand, this is very inconvenient.

Does anyone know of a better workaround or solution?

In addition to fixing these bugs, I would suggest Microsoft should add another checkbox:
* Apply replace text on all capitalizations (with capitalization preserved)
Note: the additional checkbox would require a table of lowercase to uppercase (and vice-versa) mappings for characters from all languages, which could admittedly be a challenge to implement, especially if there are some conflicts between different languages.

And by the way, it would be nice if the Autocorrect options box were resizeable, so that the users could more easily review the "replace text as you type" list. With the current fixed sizing, it is extremely tedious to view this list because only 5 lines can be shown at a time.

Also, please don't answer that update to Office 2016 is required. Huge numbers of users do not have a choice about this in their corporate environment. Please include a fix for this also in Office 2013.
