bjrch say:

After some investigations it turned out to be an issue on the Exchange server side. In our environment the exchange server is not capable to handle TLSv1.2 request properly. With the previous version the outlook client negotiated down to TLSv1.1, etc.

However after the update when the Outlook client does not receive a proper answer after sending a TLSv1.2 client Hello, it does not negotiate down it retries sending a TLSv1.2 client Hello. With the result it will never connect to the exchange server.

Workaround is to use a simple proxy (e.g. stunnel) to enable the client to connect TLSv1.2 and the proxy relaying the message to the Exchange in TLSv1.1.  Not the best solution, but till the moment Microsoft resolves this issue.......

When you send an SMS text message from Skype, the recipient will see the first 11 characters of your Skype Name. For example, if your Skype Name is thisismyskypename, the recipient will see thisismysky. They won’t be able to reply to your SMS text message.

However, you can change your sender ID so it displays your mobile number instead. This means the recipient can reply to your SMS text message and the reply will go to your mobile phone (not your Skype).

The option to change your Sender ID is only available in the classic version of Skype. You can download it using this link. Once you successfully downloaded the classic version of Skype, follow the steps below to change your Sender ID:

  1. Click Tools, then select Options...
  2. Under IM & SMS, select SMS Settings.
  3. Click the link Verify your mobile phone number and follow the instructions.

For more information, you may refer to this FAQ. If you have further questions, please do not hesitate to reply to this post.

Using the output of the following Access Queries, the Many to One facility on my Merge Tools Add-in can create the required letters

SELECT Sheet1.[Account Number], Left([Name],InStr([Name]," ")) AS FirstName, IIf(DateDiff("d",[Date],Date())<30,[Invoice Amount]," ") AS CurrentArrears, IIf(DateDiff("d",[Date],Date())>30,[Invoice Amount],"") AS AgedArrears, IIf(DateDiff("d",[Date],Date())>30,[Date],"") AS AgedArrearsDate
FROM Sheet1
ORDER BY Sheet1.[Account Number] DESC , IIf(DateDiff("d",[Date],Date())<30,[Invoice Amount],"");

and

SELECT Query1.[Account Number], Query1.FirstName, Query1.CurrentArrears, Query1.AgedArrears, Query1.AgedArrearsDate

FROM Query1

WHERE (((Query1.AgedArrears)=" "));
