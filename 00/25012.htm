When the saved document is opened OR the template Letter.dotx is opened via File New if the Body of Letter CC has text in the CC I want to put a macro to

Clear Body of Letter CC content only so the user can reuse a letter (not best practice but usual) or clear text from the precedent on File New because they want to type new text in body or paste or use a building block etc.

  1. I am not sure that I even need to encase the CC in the CC. Is it necessary?
  2. Macro to clear [Body of Letter] CC only and place cursor ready for typing.

Many thanks.

Libby

jay freedman say:

Instead of looping through the whole .ContentControls collection and hitting the locked control, select just the Body control like this:

       

Sub CCClearBodyOfLetter()
    Dim CC As ContentControl

    On Error Resume Next
    Set CC = ActiveDocument.SelectContentControlsByTitle("Body")(1)
    If Err.Number > 0 Then Exit Sub

    If Not CC.ShowingPlaceholderText Then
        CC.Range.Text = ""
        CC.Range.Select
    End If
End Sub

The .SelectContentControlsByTitle method returns a vector of zero or more controls whose titles match the argument (matching is case-sensitive, so be careful), and the (1) selects the first element of the vector. The error trap is there in case the returned vector is empty.

Enclosing the whole document in a locked Rich Text control is one way to do what you want, but there are two other ways.

  • The designed-into-Word way is to select the entire document (which must include one or more content controls and at least one paragraph mark that isn't in a control) and click Developer > Group > Group. That inserts an invisible content control (of type wdContentControlGroup) that prevents editing of any text that isn't in one of the embedded controls.
  • If there is nothing in the document except the Body content control, you don't need the outer Rich Text CC; just turn on the "Cannot be deleted" property of the Body control.

On the issue of template vs. reusing a document, there are really good reasons to teach users -- whack them upside the head if necessary -- to use a template. The big one is that it's far too easy to hit Save instead of Save As and overwrite an existing document that you intended to keep. Once that's done, you can't retrieve the old version unless you have a backup (and people sloppy enough in their work habits to reuse and lose documents almost never have backups...).

The phone comes initially with a non-carrier-specific firmware ROM, and will not support T-Mobile WiFi Calling.

zigzag3143 (ms -mvp) say:

We have recently implemented a security update called Familiar location protection to keep our customers’ accounts secure. The message "You haven't signed in from this location recently." will occur if you are signing in from a location more than 300 miles from your normal location. The Microsoft account service will prompt you for an additional challenge to complete the log in. Codes can be the date of birth, or receiving a code sent to your SMS number or alternate email if configured.

 

To prevent this you will need to configure your computer as a trusted device 

Given the new security measures, Account Settings requires an extra security, through the use of a security code.  Also known as 2 step authentication

If you mark your computer, phone or tablet as a Trusted Device, you won't be asked for any codes when you access these settings in the future.  To do this

1. Login your account here.

2.  Under Security & Privacy  Click on "Manage Advanced Security" 

3. You will see the "protect your account" page. Select any option to get a code, and request for it.

4. You will see a checkbox that says : "I sign in frequently on this device. Don't ask me for a code". Tick the checkbox, and enter the code.

Your computer/device is now Trusted. Whenever you use your account on that specific computer/device, you will no longer asked for a code.

Steps for Windows Vista or for Windows Server 2008 To uninstall Internet Explorer 8, follow these steps:
  1. Close all programs.
  2. Click Start, and then click Control Panel.
  3. Click Uninstall a Program under the Programs category
  4. In the Tasks pane, click View installed updates.
  5. In the list of installed updates, double-click Windows Internet Explorer 8.

    Note If Windows Internet Explorer 8 does not appear in the list of installed updates, try the alternative steps for Windows Vista or for Windows Server 2008.
  6. In the Uninstall an update dialog box, click Yes.

    Note If you are prompted for an administrator password or for confirmation, type the password, or click Continue.
  7. Follow the instructions to uninstall Internet Explorer 8.
  8. When the uninstall program is finished, restart your computer.
