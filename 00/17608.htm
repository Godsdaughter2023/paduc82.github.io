Try something like:

'=========>>
Option Explicit

'--------->>
Public Sub HideRows()
    Dim WB As Workbook
    Dim SH As Worksheet
    Dim Rng As Range, rCell As Range
    Dim hideRng As Range
    Dim Res As Variant
    Dim arrHide As Variant
    Dim iLastRow As Long
    Dim CalcMode As Long

    Const sSheet As String = "Sheet1"                           '<<=== Modify
    Const sCheckColumn As String = "M:M"
    Const sHideCriterion As String = "No"

    Set WB = ThisWorkbook
    Set SH = WB.Sheets(sSheet)             '

    With SH
        iLastRow = LastRow(SH, .Columns(sCheckColumn))
        Set Rng = .Range("M2:M" & iLastRow)
    End With

    On Error GoTo XIT
    With Application
        CalcMode = .Calculation
        .Calculation = xlCalculationManual
        .ScreenUpdating = False
    End With

    For Each rCell In Rng.Cells
        With rCell
            If UCase(.Value) = UCase(sHideCriterion) Then
                If hideRng Is Nothing Then
                    Set hideRng = rCell
                Else
                    Set hideRng = Union(rCell, hideRng)
                End If
            End If
        End With
    Next rCell

    If Not hideRng Is Nothing Then
        With hideRng.EntireRow
            .Hidden = Not (.Hidden)
        End With
    Else
        'nothing found, do nothing
    End If
XIT:
    With Application
        .Calculation = CalcMode
        .ScreenUpdating = True
    End With
End Sub

'--------->>
Public Function LastRow(SH As Worksheet, _
                        Optional Rng As Range)
    If Rng Is Nothing Then
        Set Rng = SH.Cells
    End If

    On Error Resume Next
    LastRow = Rng.Find(What:="*", _
                       after:=Rng.Cells(1), _
                       Lookat:=xlPart, _
                       LookIn:=xlFormulas, _
                       SearchOrder:=xlByRows, _
                       SearchDirection:=xlPrevious, _
                       MatchCase:=False).Row
    On Error GoTo 0
End Function
'<<=========

It's classic Microsoft XML with heavy use of attributes, but what you want is the last line that references your file name.  This is the link that needs to be changed.

I didn't figure out a good, user-friendly way to do this, but, if you have a presentation full of charts and you do something crazy like version the source spreadsheet using filenames (because nobody would ever do that), you need to update the links or recreate a bunch of files.  For me, the procedure below is easier than repasting all my charts and changing the color scheme for about 25 charts, but it might not be optimal if you only have one or two.

Note that this isn't pretty, but it does work.  However, if you get it wrong, you could corrupt your file, so make a backup first.  YMMV.

BTW, this will also require command line and text editing, so it depends on how much you're comfortable with using these tools on a Mac as an Office user.  I live there, but most people don't.

To update the links in the powerpoint to a new spreadsheet (if you've changed the name):

  1. BACK UP YOUR ORIGINAL FILE
  2. No, seriously, create a backup of your original file.
  3. Open a Terminal window from Applications > Terminal.app
  4. type (without the quotes) 'cp ' into the terminal window (note the space)
  5. Select the file you want to fix in the Finder and drag/drop it to the Terminal window [see * below]
  6. type (without the quotes) ' /tmp' (note the space) and hit return so the file is copied to the /tmp directory
  7. Change to the /tmp directory using 'cd /tmp'
  8. Make a new directory pptdoc
  9. Change into the new directory 'cd pptdoc'
  10. Unzip your document using 'unzip ../XXXX.pptx'
  11. List all of the files you need to change: 'grep -rn XYZZY.xlsx' where XYZZY.xlsx is the name of your spreadsheet without the path you pasted before
  12. [OPTION 1] open all of them with TextEdit so you can manually edit them in a Mac application: 'grep -rn FILENAME.xlsx | xargs -e'
  13. [OPTION 2] open all of them in vim so you can manually edit
    1. Make sure you're in the right directory:  'cd /tmp/pptdoc'
    2. Zip up the updated source bundle: 'zip -r ../fixed.pptx *'
  14. Open and make sure your charts are updated to the right document.

Hope this helps.  It worked for me.  When you change this, it may ask you to grant permissions or it may ask you this if you try and edit the file.  Just select the right target sheet, and things will track the updates correctly.

In the command below, the old spreadsheet is 'OLDNAME.xlsx' and the new one is 'NEWNAME.xlsx' and it is assumed that both OLDNAME and NEWNAME are in the same directory.  If NEWNAME has spaces in it, they need to be replaced with '%20' for each space.  If you need to do this, you can easily replace the spaces as follows where you replace the text inside the quotes with the name of your target file:

echo "My File" | sed -e 's/ /%20/'

The commands you need to execute to process all of the files in one go are as follows, replacing the OLDNAME with NEWNAME.  The "$" is the shell prompt.


$ grep -rl OLDNAME.xlsx . | xargs sed -i '' -e 's/OLDNAME/NEWNAME/'
