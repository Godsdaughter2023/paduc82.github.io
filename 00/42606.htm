arminatorx say:

The problem is, that the search takes spacing into consideration, while "internally" the gamertags are stripped from spacing when compared for existence.

So for example, if you want "CleverGamertag" (without spacing), and someone already has "Clever Gamertag" (with a space), then this gamertag is taken for the system. However, when you search for "CleverGamertag" you won't find it. 

If you searched for "Clever Gamertag" (with space), then the search will find the Gamertag. 

However, if some people use creative spacing like "Cle Ver Gamer Tag" it might be very difficult to find the gamertag. 

So if the service says the gamertag is taken, then it is taken, but with a different spacing. 

If you really want that name, wait for a bit, as Xbox is changing the way Gamertags can be named. 

They will add Discord-Style tags after your Gamertag, so if you really want to be named "e" for example (the very first Gamertag ever created), you can do so later. 

You will be named "e#1234" however, with the number tag being less visible.

If the Store app does not open, hangs, or not working correctly, then re-registering the Store will usually fix this issue.

Re-register Microsoft Store app

 You must be signed in as an administrator to be able to do this option.

1. Terminate the Microsoft Store app to make sure it's closed and not running.

2. Open an elevated PowerShell

3. Copy and paste the command below into the elevated PowerShell, and press Enter.

Get-AppXPackage *WindowsStore* -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

4. When finished, close the elevated PowerShell,

5. Restart the computer to apply.

6. Try opening the Microsoft Store app again.
