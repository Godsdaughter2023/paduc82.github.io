don cuthbert say:

When an upgrade to a new build fails, these steps may enable it to succeed:

  1. Completely uninstall any non-Microsoft anti-virus software.
  2. Perform a clean boot before upgrading.
  3. Disconnect any un-needed peripherals, such as devices connected via USB.
  4. Disable all network adapters after the build has downloaded but before it Restarts, or before running setup.exe, if you are upgrading via a mounted ISO or Windows 10 install media.

Sometimes disconnecting the Internet, such as unplugging the Ethernet cable, will enable a stalled upgrade to complete.

When Windows Update downloads a new build, it puts the main ESD file in a hidden folder at C:\$WINDOWS.~BT\Sources\Install.esd.  You can use this file to create a bootable USB or an ISO file for the new build, so that you don't have to keep re-downloading it.  If you are very short of space, this file may get deleted, but usually it remains.

don cuthbert say:

A bootable Windows install USB can be created from the Install.esd file (that was downloaded by Windows Update) and a few diskpart and DISM commands. You don't even need to create an ISO file first.

If you already have the ISO file, a utility called Rufus works nicely to create the bootable USB. https://rufus.akeo.ie/ 

https://www.microsoft.com/en-us/software-download/windowsinsiderpreviewiso 

You do have to know how to get your PC to boot from USB.  Many PC's have the ability to launch a one-time boot menu at power-up. Dell computers use F12 and other brands use the ESC key or F2, etc. at power-up for this option. Some you have to change the boot order in BIOS settings for it to work. (I have also found that not every USB stick will be detected by every PC. I have a new stick that my old laptop does not detect at power-up, but if I put in a different USB stick it will see that one and I can then swap it for the one it doesn't detect.)

The simplest and quickest method to prove if your PC can boot from USB is to run recoverydrive.exe and choose to not include System files. It will format and create a small bootable USB stick with less than 400MB on it, which can boot to Windows Repair Environment.  (If you do include System files, you can clean install that same build of Windows 10 with it, but it takes longer to create and will tell you how large of a USB stick you need).
