this absolutely make zero since, there has to be away to unallocate the resources...


This is hardly a debatable point. A 32bit OS has a 4GB limit to the amount of memory space it can address. Whether those addresses are occupied by RAM or by hardware, the same addresses can't be used for different things at the same time. All memory addresses beyond that 4GB are inaccessible so the computer's hardware addresses are reserved within the 4GB limit not above it. The excess memory above 4GB isn't allocated to anything.
