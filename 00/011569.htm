john p spencer say:

You can open the query in SQL view, copy the SQL statement, and then paste the SQL statement into a new query.  When you do this the datasheet view of the new query will agree with the query design view.

Access has the ability to remember settings for datasheet view. So when you move columns, hide columns, resize columns, etc., reorder, or filter while in DATASHEET view, Access will remember the changes.  If you close the query in datasheet view, you will usually get prompted to save the changes to the datasheet view.  If you do so, the next time you open the query in datasheet view the changes will be used.  The query design view (and SQL) is not changed by the properties of the datasheet view.
