marshall barton say:

There are very few options for how a query's datasheet should look and Nav buttons is not one of those few.  Use a real form as a subform (with its record source set to the query) and set the form's NavigationButtons property to No.

The NavigationButtons property is a property of a form object, it is not a property of a subform control that is used to display a query's datasheet or a form object.

It sounds like you clicked once on the subform control in the main form's design view.  If you click a second time, you should then be in the form.  To get to the form object in the subform control, click on the little box in the subform's upper left corner above and to the left of the design view rulers.

Hi Doug

 

Thanks for that, it worked out well once I figured out some of the detail. To help everyone else, this is how to do it :

 

MailMerge from Outlook 2010/Word 2010

This way it works for me, with labels.

1. Start from Outlook 2010
2. Switch to the Contacts view
3. Select, if needed, the contacts that you need. I have my Xmas list as a Colour Category, so I can just use Shift + Cursordown to select them. Others can be added uing Ctrl + Click.
4. Select MailMerge on the Home toolbar. Either 'All Contacts' or 'Selected Contacts' will be enabled, depending whether contacts were selected.
5. By default, you are set to a new document and do not save the contact list as a file. These can be changed.
6. In the Merge Options, set the type of document. I needed to change to 'Mailing Labels' and it defaulted to merge to a new document.
7. Click on OK. You get Word document loaded and a message 'Outlook created a mail merge document and prepared your contact data for you. To complete the setup for Mailing Labels, press the "Setup" button in section 1 of the MailMerge helper dialog.' is displayed. Click on OK.
8. The Mail Merge Helper dialog appears. Click on Setup and the Label Options dialog appears. Here you can select the settings for the label sheets, in my case Avery J8161, and Click on OK.
9. In the Mail Merge Helper dialog, you do not need to touch the Data Source, as it is already set up. In the Query Options, you can set a filter on the contacts and sort them. The settings show that it will 'Suppress Blank Lines in Addresses' - in practice this does not work, see below. Click on Close.
10. At this point, you have a table to match the label with the first one blank and the rest flagged as <<Next Record>>. You have now run through the options in the Mailings 'Start Mail Merge' block.
11. Click on 'Address Block' on the toolbar and Word will show the 'Insert Address Block' dialog, with the fields from Outlook matched to the those in the MailMerge. You have options to change these as needed, and can review through the contacts. Click on OK and the <<Address Block>> will appear on the first label.
12. Click on 'Update Labels' on the toolbar. The <<Address Block>> will duplicated onto all of the labels.
13. Click on 'Finish and Merge' on the toolbar. You have options to Edit Individual Documents, Print Documents or Send E-maill Messages. I normally use the Edit option to ensure that the label are correct, and add a senders name/address at the end, before I print them. You get an options to select all, one or a range of records. At this point, Word will create a new document with the labels.

At this point you are in normal Word with the labels in a table. I find that I need to change the font and size to a suitable value, switch to a 'No Spacing' style, indent the lines 0.2cm from the left, insert a blank line at the top of each label and a sender's name on the bottom. Also, there is a problem with the address lines. If the Street Address is multiple lines of text, such as a road and village, it is followed by a single blank line before the city name. I know that the value in Outlook is correct, having checked them and re-edited to check, so I have to remove the blank lines as well. Then I can print my labels.
