xircal say:

Hi,

Do the following.

Step One

  1. Right click the speaker icon on the taskbar and click Recording devices.
  2. If the microphone is present in the menu, click it to select and then click the Properties button.
  3. Click the Listen tab and then checkmark the box where it says "Listen to this device".
  4. Say or sing something.
  5. If you can hear yourself, remove the checkmark afterwards.

If the microphone isn't present when you click Recording devices, go to:

Step Two

  1. Right click the Start button and click Power Options.
  2. Click the link to the right of whichever power plan you have configured called "Change plan settings".
  3. In the next menu, click the link to Change advanced power settings.
  4. Scroll down to USB Settings | USB selective suspend and change Plugged In to Disabled.

This will prevent the computer from suspending the driver if it determines it's no longer in use. When the driver is enabled, it can sometimes lead to a device not being recognized when you plug it in.

Repeat Step One to check if the microphone is detected or not now.

If it still doesn't work, make sure both the USB connector and the Audio/mic jack are plugged in properly and then repeat Step One.

If you still don't have any luck:

Step Three

  1. Right click the Start button and click Device Manager.
  2. Expand Universal Serial Bus controllers.
  3. You'll see a list of USB Hubs. Some of these will be named "Generic" and some "USB Root Hub". Right click the first of these and click Properties.
  4. Click the Power Management tab and remove the checkmark from where it says: "Allow the computer to turn off this device to save power" and then click OK.
  5. Repeat for the other USB Hubs and then exit Device Manager.
  6. Repeat Step One again.

If you're still out of luck, it might be due to file corruption. To check go to:

Step Four

  1. Hit Windows key + X and click "Command Prompt (Admin)".
  2. Type (or copy/paste) sfc /scannow and hit Enter.

System File Checker will check Windows system files for corruption and try to fix any found.

On occasions though it fails in which case you'll see the message: "Windows resource protection found corrupt files but was unable to fix some of them".

If you see that message, copy and then right click and paste the following command into the same command prompt and hit Enter:

DISM.exe/Online /Cleanup-image /Restorehealth

When DISM runs, it downloads original system files from Windows Update where necessary so it's important to maintain your Internet connection while it's running.

While DISM runs all you'll see most of the time is a plain black window. But eventually the figure 20% will appear followed by 100% about 15 minutes later. You'll also get a report of what was fixed.

If you see any error messages, make a note of them and post those here please.
