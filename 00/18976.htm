Taskbar color - Microsoft Community

begem0t say:

Taskbar color

In build 9926 it seems that the Windows Taskbar transparency is gone. Which, for me, is not a good thing.

On the positive side, we've gained control over taskbar color through registry keys:

To change the Taskbar color add DWORD:

HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced\TaskbarColorOverride

Format is hex, color foormat is BGR (not RGB, as would be more intuitive)

To change Plate color (the color under active icon) add DWORD:
HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\TaskbarPlateOverrideColorBGR

Format is hex, color format is BGR (again)

To change Plate opacity add DWORD

HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\TaskbarPlateOverrideOpacityA

Format is decimal, range 0-256


To refresh just press Win key (Open and close Start menu).

Example registry file (use at your risk and don't blame me for the global warming, etc, etc)

Windows Registry Editor Version 5.00

[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer]
"TaskbarPlateOverrideColorBGR"=dword:00054de0
"TaskbarPlateOverrideOpacityA"=dword:00000100


[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced]
"TaskbarColorOverride"=dword:00635a56

scottgem say:
First, a form doesn't hold data it displays data that is in your form's recordsource. So what you want to export is the data, not the form.

So using TransferSpreadsheet is the way to go. But you need to do it with Formatting to preserve the content. So go through an export manually, selecting the with Formatting option. At the end you will be prompted to save the settings. give them a name and save it. In the Transfer Spreadsheet method, there is place to enter the name so it will use those settings.

a. user say:

I tried the TransferSpreadsheet method. I get a 3011 error. The error mentioned "FPR SEARCH RESULTS" cannot be found. "FPR SEARCH RESULTS" is the name of my form. 

My form is a filtered form based on a table. 

DoCmd.TransferSpreadsheet acExport, acSpreadsheetTypeExcel12Xml, "FPR SEARCH RESULTS", "C:\ExportedResults.xls", True

scottgem say:
Again, you aren't getting the data from the form, you are getting the data from the form's Recordsource. The TransferSpreadsheet method requires a table or query in that argument. Since no table or query named FPR SEARCH RESULTS exists, it can't find it.

You need to create a query that looks to the form for criteria. For example:

SELECT * FROM table WHERE Company = Forms!formname!Company;

Then export that query.
