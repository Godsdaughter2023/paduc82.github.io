doug robbins - mvp office apps & services (word) say:

IF the control IS IN THE CELL OF A TABLE, the tab key works as it normally does in a table and in that case, if you do want to insert a tab space inside the content control, you need to use CTRL+Tab.

mike h.. say:

Hi,

You can try this but I can forsee some potential issues.

1. How far do we fill down? Tp establish this we need something; anything, in one of the cells that we can look for. The bold bit in the code does this but if all the cells are empty then it will fail.

2. When you manually change the cell to f then the original colour is overwritten by the code and we don't know what to put back when you delete the f

Right click your sheet tab, view code and paste this in and try it.

Private Sub Worksheet_Change(ByVal Target As Range)
Dim LastRow As Long
If Target.Cells.Count > 1 Then Exit Sub
If Not Intersect(Target, Range("K7:AB7")) Is Nothing Then
LastRow = Range("K7").Resize(100000, 18).Find(What:="*", After:=[K7], _
    SearchOrder:=xlByRows, _
    SearchDirection:=xlPrevious).Row
If UCase(Target.Value) = "F" Then
Application.EnableEvents = False
       With Target
        .Resize(LastRow - 6).Value = "f"
        .Resize(LastRow - 6).Interior.ColorIndex = 15
        .Orientation = 90
    End With
    ElseIf UCase(Target.Value) <> "F" Then
        With Target
        .Offset(1).Resize(LastRow - 6).ClearContents
        .Resize(LastRow - 6).Interior.ColorIndex = xlNone
        .Orientation = 0
    End With

End If
End If
Application.EnableEvents = True
End Sub

scottgem say:

arguments, variables, parameters are all passed to User Defined functions the same was as to build in functions. 

For example you have a DLookup() function. The arguments are field, domain, criteria. So you might have a DLookup like: DLookup("[myfield]","mytable","[PKfield] = " & Me.PKcontrol)

Similarly when you create a user Defined function you place parameters in the function definition:

Public Function myfunction(dblValue1 As Double, dblValue2 As Double) As Double

myFunction = dblValue1 + dblValue2

End Function

You would call the function:

=myfunction(1.5,2,75) or =myfunction(Me.value1,Me.value2)

and it would return 4.25

OpenArgs passes a value using the OpenReport (works with OpenForm too) to the object being opened. You can then retrieve that value using Me.OpenArgs.

ken sheridan say:

I'm not sure I understand how to pass variables from one function to another, but I would love to. Would I HAVE to name the Public Function GetReportName?

You can call the function whatever you like provided that the name does not conflict with the name of a built in function or of any object.  You'd pass a reference to the report into it as an argument, and within the function use that argument to identify the report from whose module the function is called, e.g.

 

Public Function SortReport(rpt As Report) As Boolean

 

    Const FORM_NOT_OPEN = 2450

   

    On Error GoTo Err_Handler

 

    Select Case [Forms]![menu_CaseListReports]![SortOrder]

        Case 1

        rpt.GroupLevel(0).ControlSource = "Nickname"

        Case 2

        rpt.GroupLevel(0).ControlSource = "FileNo"

        Case 3

        rpt.GroupLevel(0).ControlSource = "DateOpened"

        rpt.GroupLevel(1).ControlSource = "Nickname"

    End Select

 

    SortReport = True

 

Exit_Here:

    Exit Function

   

Err_Handler:

    Select Case Err.Number

        Case FORM_NOT_OPEN

        ' anticipate error,set return value to True and exit function

        SortReport = True

        Case Else

        ' unknown error, set return value to False and inform User

        SortReport = False

        MsgBox Err.Description, vbExclamation, "Error"

    End Select

    Resume Exit_Here

       

End Function

 

In each report's Open event procedure call the function, and pass a reference to the report into it by means of the Me keyword:

 

Private Sub Report_Open(Cancel As Integer)

 

    If Not SortReport(Me) Then

        Cancel = True

    End If

   

End Sub
