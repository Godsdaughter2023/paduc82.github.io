a. user say:

Hi Corentin,

Yes, the link that Skype saves to the desktop is a .inetloc file with a bit of HTML/plist inside like this:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
<key>URL</key>
<string>skype:thecontactname</string>
</dict>
</plist>

Click on one of those, and Skype opens and does the right thing.

You can get the skype:thecontactname into Word as a Hyperlink like this

Sub insertSkypeLink()
Dim h As Hyperlink
Dim s As String
s = "skype:thecontactname"

' The Add hyperlink comman seems to cause some problems that I haven't 

' explored, so try to persuade VBA not to give up

On Error Resume Next

' this does the right thing except Word/VBA modifies the TextToDisplay...

Selection.Hyperlinks.Add Anchor:=ActiveDocument.Content, Address:=s, ScreenTip:=s, TextToDisplay:=s

' so fix it up

ActiveDocument.Hyperlinks(1).TextToDisplay = s
End Sub

But then it's as you say - Word won't process the URL in the way you might hope.

I couldn't think of any other way to insert something into Word that would "do the right thing" without code (e.g. inserting an object from a file doesn't seem to achieve very much).

But instead of a hyperlink, using a { MACROBUTTON } could be a little more direct for the user. e.g., insert

{ MACROBUTTON skypecall thecontactname }

(use cmd-F9 or if necessary fn-cmd-F9 to enter the special field code braces { }

then have VBA more like this:

Sub skypecall()
'
'MsgBox "we're in"
Dim strAddress As String
Dim strResult As String
Dim strScript As String
Dim v As Variant
v = Split(Selection.Fields(1).Code.Text, " ")
strAddress = Trim(v(3))
strScript = "" & _
  "tell application ""Skype""" & vbCrLf & _
  "    activate" & vbCrLf & _
  "    send command ""CALL " & Trim(v(3)) & """ " & " script name ""Word VBA""" & vbCrLf & _
  "end tell"
strResult = VBA.MacScript(strScript)
End Sub



Edit:
Unfortunately, none of these things will survive into either HTML or PDF. In the former case, Word insists on changing the URLto a file: URL, even if you uncheck the "modify links on save" option. THe PDF link produced by the standard Save As doesn't do anything.

tina chen msft say:

Hi Roger,

 

As described in this article, starting in October 2017, we're changing how the previous OneDrive for Business sync client installs for enterprise customers using Office 365 ProPlus or Office 2016 with Click-to-Run.

The previous OneDrive for Business sync client (Groove.exe) will no longer be installed by default with Office 2016 Click-to-Run. If your organization provides an Office deployment configuration file to Setup.exe, you’ll need to update your file to exclude Groove from the install. Please refer to article.

 

When not in use or running, the previous OneDrive for Business sync client (Groove.exe) will be uninstalled, unless: (a) Groove is already configured to sync one or more SharePoint Online or SharePoint Server libraries or (b) a “PreventUninstall” registry key is present on the computer.

 

On Office upgrade, the installer runs on each computer to detect whether Groove.exe is currently in use or the “PreventUninstall” registry key is set. If either Groove.exe is in use or the registry key is set, Groove.exe is left in place. Otherwise, if Groove.exe isn’t in use and the registry key isn't set, Groove.exe gets uninstalled automatically on that computer.

 

Depend on what version of Office you are using, please check the article I mentioned above.

Tina

jmh3143 say:
If you prefer you can revert to your "old" Hotmail account.




 You can simply right click the tool gear next to you user  name and click the "revert back to Hotmail" and wait a bit.

If that doesnt work try this.

Note: The following steps will currently only work if your previous account was @Hotmail.com or @Live.com.

 

    In your browser, navigate to https://account.live.com/ManageAssocIds
    Sign in with your new @outlook.com account.  (if you are already signed in with your outlook.com account this step will not apply)
    Under “Other email addresses” click the Remove link below your previous address.
    Click the Remove button to confirm the message.
    You should now see a message on the page stating, “You've removed an email address from your Microsoft account.”
    Navigate to https://account.live.com/ChangeId.aspx in your browser.
    Verify that “Create a new email address” is selected.
    In the New Microsoft account box, enter your previous address and select your previous domain (hotmail.com or live.com).  Note:  You must enter your previous address or the process will fail.
    Select the Save button.

 

Now you can use your original @hotmail.com or @live.com account to sign into your Microsoft services as you did before renaming to an Outlook.com account.  The @outlook.com address will now be an alias on your account from which you can still send and receive messages within Outlook or Hotmail.

 

Note: This applies to other domains as well, such as .jp, .kr, .es, .au, .ca etc.
