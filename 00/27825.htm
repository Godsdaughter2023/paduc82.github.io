graham mayor say:
If you insert a table of contents from the References tab - use the 'Insert Table of Contents' option from the bottom of the dropdown. From the dialog box that then appears, click 'Options'. You can set which styles to include in the TOC. By default Heading 1, 2 & 3 , Title and Subtitle styles are included.

stefan blom say:

I've added a Table of Contents to a Word 2007 document, using the standard method on the References tab of the Ribbon.  The field code is "{TOC \o "1-3" \h \z \u}", which seems pretty vanilla to me.  But the Table of Contents contains a reference to the Title of the document (style "Title"), and to every cell of a revision tracking table (style "Table Cell"), and to every heading of style "Heading" (not "Heading n").  Why does it do this, and how do I stop it?  I really only want styles "Heading n" to appear in the TOC.

Thanks.

 


Clear the \u switch from the field code (it picks up paragraphs to which an "Outline level" other than "Body Text" has been applied as direct formatting).

Also, modify the Title style so that it has an outline level of "Body Text" (Modify Style dialog box, Format, Paragraph).

Finally, update the TOC field.

 

 

Stefan Blom, Microsoft Word MVP

ronrosenfeld say:

On Sat, 6 Nov 2010 04:09:38 +0000, drof_06 wrote:

I'm hoping someone can give me some help to split a source name column into separate columns for first names and last names. Some sample data could look like the following;

O'CONNOR Sean

MC MANUS James

DEAN ANDERSON Alex

SANDERS Jaimie Lea

 

Please note that in the source column last names will always be in upper case font if this is of any help.

Assuming the First Names will always start with a single Upper Case
followed by a Lower Case, you can use this VBA macro.

The Source Column and First Row of data is specified in the code, then
every line below with data is processed.  For the example, I assumed
column A and row 2.

The split data is written to the two adjacent columns.  But once you
are satisfied this works as required, it would be easy to overwrite
the source data.

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select
the macro by name, and <RUN>.

====================================
Option Explicit
Sub SplitCaps()
 Dim re As Object, mc As Object, sm As Object
 Dim rg As Range, c As Range

 'Specify column and first row of Source Data
 Const rSrcCol As String = "A"
 Const rSrcRw As Long = 2

Set rg = Range(Cells(rSrcRw, rSrcCol), _
    Cells(Cells.Rows.Count, rSrcCol).End(xlUp))

Range(rg.Offset(0, 1), rg.Offset(0, 2)).ClearContents

Set re = CreateObject("vbscript.regexp")
With re
    .Global = True
    .ignorecase = False
    .MultiLine = True
    .Pattern = "(^.*?)\s(?=[A-Z][a-z])(.*$)"
End With

For Each c In rg
    If re.Test(c.Text) <> True Then
        c(1, 2).Value = c.Text
    Else
        Set mc = re.Execute(c.Text)
        Set sm = mc(0).SubMatches
            c(1, 2).Value = sm(0)
            c(1, 3).Value = sm(1)
    End If
Next c

End Sub

jay freedman say:

In the line of code that shows the error, is it exactly like txtocip Value = "a"  -- that is, with a space before the V? If so, type a period instead of the space.

In VBA and similar languages, the period or dot is an operator that means "the item after the dot is a property or method of the object before the dot". The space character doesn't have that meaning.

By default, "IntelliSense" is turned on in the VBA editor. That's Microsoft's name for the feature that when you type the name of an object variable and then the dot, a list of all the object's properties and methods appears so you can just click one. Not only does this help to avoid misspellings, but (usually) it tells you whether the property you want to use really belongs to that object.
