ynthia say:

This is actually a very common issue for Gmail accounts lately.

Google, Yahoo!, Apple, and many other major email providers have been very strict with their email set up lately. Especially to a third-party email client like Outlook. These email providers have increased their email servers, and when doing so, initiated the necessity to activate an additional protection for accounts that belong to their domain.

The fix is actually very easy: generate an app password

App Passwords create another layer of protection for your email account, since you can only create one when you access your account from the webmail. 

You may refer to this link for a more visual guide on how to create an app password (under Set up two-factor authentication for Gmail) - Add an email account to Outlook

Once you've enabled the two-step verification, and created an app password, copy the app password and paste it on the password field of that Gmail prompt you see in Outlook. It'll prompt you twice, for IMAP and SMTP, so you'll need to paste the password twice as well.

Re-launching Outlook then should not give you the password prompt issue again.
