You will only see that when it will actually do something.   Otherwise, you will see the Refresh icon.

 

That was actually how the button was designed originally, as a tri-state button with a different icon for each state:  

  • typed input waiting for Enter,
  • input Entered but not responded to so Stop can be used,
  • or Refresh (the most common state).  

Web hosts are getting so good with their performance that you may hardly notice the button being in its Stop state now.

 

What we need instead is better and easier control of scripting.   E.g. currently that is possible only either very coarsely using the Security tab's Custom Settings dialog for its Active Scripting options, or very arcanely by using the IE Developer Tools, Script Debugger.

jon peltier (excel mvp) say:

The stacked chart shows the cumulative sums of the series. Say you have series A, B, C, and D. What gets plotted then are Y values for A, A+B, A+B+C, and A+B+C+D.

So off in your worksheet, make a table that has these sums. Then use SLOPE() and INTERCEPT() to calculate the pseudo trendline for these sums.

Note that in a line chart, Excel uses the counting numbers 1, 2, 3, etc. for X values, regardless of what the data looks like to you and me.

To plot the lines, figure out the Y values at each X value using Y = INTERCEPT + SLOPE * X, then add these series to the chart. To prevent them from being stacked onto the existing stacks, convert these added series to XY type, and display them with lines and no markers. Also if Excel puts them on the secondary axes, reassign them to the primary.
