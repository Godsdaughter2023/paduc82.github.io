Word 2010 Bulk add macro - Microsoft Community

darcykellogg say:

Word 2010 Bulk add macro

In Word 2010 is there any way to add a macro to multiple documents at once? I have several hundred documents which need a macro inserted into them for the addition of an automated header and was hoping there was a quick way to do this instead of having to open every document and paste in the macro.
Thanks

jay freedman say:
It's generally a bad idea to add macros to Word documents at all. Also, if the documents are saved in docx format, you can't save macros in them; they would have to be opened and resaved in docm (macro-enabled) format.

If the documents are all based on the same template (often that's Normal.dotm), you can put a macro into that template, and it will be available to every document based on it.

Before you start on that project, though, realize that you may not need a macro at all. Depending on what you want the headers to show, you might just need some fields such as StyleRef fields instead. Explain the purpose of the "automated header" so we can guide you.

darcykellogg say: the files are .doc files and the macro is used to build a header table based on information pulled from the file name. The macro needs to be embedded in each document and cannot run from a shared template.

jay freedman say:
You are mistaken in asserting that it cannot run from a shared template. Macros in a template can read filenames, open documents, construct headers and tables in the headers, and save and close the documents. All of this is far easier and safer than trying to add macros to documents.

If you insist on going the hard and dangerous way, first you need to go into the Options > Trust Center > Trust Center Settings dialog and check the box for "Trust access to the VBA project object model", which is unchecked by default -- by design, macros should not create macros because that's a classic macro-virus procedure. In fact, some antivirus software will remove such macros or quarantine files that contain them.

Next, in the macro editor, you need to select the project where you're going to build the macro that creates macros, click Tools > References, and check the box for "Microsoft Visual Basic for Applications Extensibility". That library contains the commands for creating and modifying macros.

Then you need to have already written and tested the macro that you want to insert into the document files, and have that available. Make absolutely sure that you've considered all the ways that macro can fail -- for example, if some of the documents have passwords that are needed to allow editing, or if they're protected for forms. With hundreds of documents, you don't want to have to go back later to fix up a misbehaving macro.
