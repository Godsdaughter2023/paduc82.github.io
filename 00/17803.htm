faysal farooqui say:

There are 2 ways to do that:

(1) Delete Conditional Formatting Rules from Sheet B: While you press Ctrl+V (including formatting) in Sheet B to paste that, then all formats are also copied to Sheet B. Lookup on multiple fields - Microsoft Community

eric 98 say:

Lookup on multiple fields

I often want to do a VLOOKUP based on multiple fields.

Example: I have a table with 1000 rows.
Col A is first name
Col B is last name
Col C is city

I want to look for someone with a certain last name and also a certain city.

What I typically do in a case like this is create an auxiliary column which concatenates columns B and C. Then I do the lookup based on a concatenated search value. Is this the right way, or is there a better way?

Thanks.

jeeped say:
... Is this the right way, or is there a better way?

A lot of answering this question is in knowing how many cells are referenced in the lookup, how many lookups are performed. Array formulas increase calculation load exponentially, so the size of the ranges and the number of array formulas is of highest concern.

                 

In the single two-column VLOOKUP(...) with limited range lookups, the following array formula can be used to lookup column C using both columns A and B references.

=VLOOKUP(E3,IF($B$1:$B$16=F3,$A$1:$C$16),3,FALSE)

... which requires Ctrl+Shift+Enter↵ rather than just Enter↵.

eric 98 say: Can you explain a bit further exactly how this works? For example, I don't understand what's the effect of an IF() with only two parameters.

Thanks.

jeeped say:
Can you explain a bit further exactly how this works?

<*sigh*> That's a pretty open-ended question. Perhaps you could you post back with a description of the parts that you do understand?

A standard formula VLOOKUP(...) has a primary condition in its lookup_value parameter. The above array formula adds a condition to the table_array parameter by saying "only consider rows that have 18 in column B". If a match is found to ted in a row that has 18 in column B, then the value from the 3rd column is returned from the table_array (e.g. A1:C16). The formula is an array formula and as such requires Ctrl+Shift+Enter↵ rather than just Enter↵.

garym123 say:

If you are having problems with a memory caard:

1. Switch off the phone and then remove the memory card. For information on how to remove the memory card, see your Nokia phone's user guide.

2. Connect the memory card to a PC with a USB or SD card adapter; your memory card should appear as a separate drive on the computer:
If the memory card problem message box appears, follow the on-screen instruction to fix the problem.
If the memory card is not recognized by the PC, it may be broken or password protected by the previous device where it was used. Neither Windows nor Windows Phone support password protection. Remove the password from the memory card with the previously used device.

3. Format the memory card on a PC. Before formatting the memory card, make sure you have a backup of the data as it is lost during formatting:

Use Windows Explorer to navigate to the removable storage on the computer, right-click the memory card and select Format.
For 4-32GB cards, select the file system FAT32 and allocation unit size 32 kilobytes and press Start. For 64GB cards, select the file system exFAT and allocation unit size 128kB.
Confirm the data loss warning message. When the formatting process is complete, press OK.
Right-click the memory card, select Eject, and wait until it is safe to remove the card.
Insert the memory card into the slot in the phone until it locks into place.
The previous actions usually solve your problem. If they do not help, test with another compatible memory card to see if the issue is with the card. Some memory cards are currently incompatible with your phone. For optimal performance, we recommend a fast 4-32GB card by a well-known manufacturer, for example, SanDisk Extreme Pro, Toshiba Exceria, or Samsung Pro.
