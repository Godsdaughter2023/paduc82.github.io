Since moving to SP Online, the documents will not stay grouped by manager after an edit, apparently because the manager name is erased from the document properties after an edit/save.  I thought I might be able to get around this by using the Document Information Panel and requiring users to re-specify the manager name each time they edit the document, but I cannot even get the Document Information Panel to appear when opening any of the Excel files.

cliff gu msft say:

Hi Kathryngrapes,

As SharePoint Server 2010 and SharePoint Online may be different, you may need to add a custom column to the default document content type in the site library in SharePoint Online. Please first create a custom Choice column called Manager in Library Settings ( the column will be added to all content types if you leave the default settings)  and then go to Library Settings>Advanced settings>Allow management of content types>Yes>go to Content Type>Documents>Document Information Panel settings>check Always show document properties on open and initial save for this content type.

 

Now when you upload the Excel files to SharePoint Online and open them in Excel 2016, you should be able to change the property value.

 

However, based on my test, when you use a password-enabled file, changing the property value in Excel 2016 may not be synced to SharePoint Online. You may need to use a non-password-enabled file instead. For security, you can set permissions for specific users to open the files as well.

Macro to Copy and Paste range to the END of another range on another - Microsoft Community

hometraderinc say:

Macro to Copy and Paste range to the END of another range on another sheet

I have the following macro setup that works fine, but I want to paste (VALUES ONLY) the "CurrentRegion" to the end of the range on the "Database" sheet instead of cell "A1" everytime.

 

Sub CopyCurrentRegion()
    Range("A1").CurrentRegion.Copy Sheets("Database").Range("A1")
End Sub

jeeped say:

... but I want to paste (VALUES ONLY) the "CurrentRegion" to the end of the range on the "Database" sheet instead of cell "A1" everytime.


A Copy, Paste Special, Values can be accomplished within VBA with a simple = .Value from one range to another. In the following, I've added ActiveSheet to clarify the origin.

Sub CopyCurrentRegion()
    With ActiveSheet.Range("A1").CurrentRegion
        Sheets("Database").Range("A1").Offset(Sheets("Database") _
          .Cells(Rows.Count, "A").End(xlUp).Row) _
          .Resize(.Rows.Count, .Columns.Count) = .Value
    End With
End Sub

The above assumes that there is at least one value in the Database sheet's column A. If the target column A is entirely blank, it will start the paste in A2.

Hi,

When you try to open a Word document on your mobile device, you might receive a message stating that the file is locked for editing by another user. Typically, you will be able to open the file as read-only, but you won’t be able to edit it.

The file might be locked because:

The file is shared and another user is currently editing it.

An instance of the Office app is running in the background with the file already opened.

The file has been marked as Final and can no longer be updated.

In some situations, you can use your mobile device to unlock the file and continue editing it.

Note: If the Office file is password-protected, you won’t be able to open it without supplying the password. There is no way to circumvent password protection.

If the file is being used or editing the Word document, we suggest that you create a copy of the file:

Open the file.

Tap the file icon Image.

On your iPad, tap Name or Duplicate. The Save As window opens. If you have any pending changes, you will be asked whether you want to apply them to the both the original and duplicate copy or the duplicate copy only. Make your selection.

Type a new name for the file and choose a storage location for the new file. On your iPhone, use the backspace key to delete the current name, and then enter the new one.

Tap Save or tap Duplicate.

If an Office app is running in the background or the file is not shared on a network or if you are certain that no one is currently editing it, then an instance of the Office app with the file open might be running in the background. We suggest that you close the app or background process on an iPad or iPhone.

If the file is marked as Final:

Office 2016 includes a file protection feature that allows authors to mark a file as Final. This locks the file, preventing any further editing. You can easily turn off this feature in the Office 2016 applications; however, you can’t turn it off from your mobile device. If an Office file is marked Final, you will need to make a copy to enter your changes from your mobile device.

Keep us updated if anything comes up so we can further assist you.
