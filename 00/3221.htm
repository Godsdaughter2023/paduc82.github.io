Project 2010 9/80 calendar - Microsoft Community

bobk827 say:

Project 2010 9/80 calendar

For a 2 year project I wish to create a calendar where some of the resources work a 9/80 week, working 9 hours all Mondays - Thursdays, with either an 8 hour Friday or a 0 hour Friday, alternating week by week.

Thank you!

john - project say:
BobK827,
Understand, but what's the question? If the question is, how do I do that, then it's simply a matter of creating a custom calendar for the 9/80 schedule. The process isn't that difficult but it will take a little time. You could use VBA to automate it but unless you know how to create macros in Project, it may be easier just to do it out manually.

As for the process itself. I'd start out by creating a copy of the standard calendar and call it, oh I don't know, maybe 9/80. Decide when you want the cycle to begin. Then set your 9 hour work hours for Monday through Thursday. Create an exception for Friday and hit the "details" button to set up a recurring every other Friday off (i.e. non-working). Set the pattern for "weekly" to recur every 2 weeks. Set the range of recurrence for however long you want to run it (e.g. 52 weeks = 1 year). You may also have to do additional editing for those holiday week when you likely need to revert back to a normal 8 hour day for weeks with holidays.

Something else you need to consider. The new normal day will be 9 hours except for every other Friday. Unfortunately Project will only take one setting for the definition of a day. And that definition is used by Project to calculate durations. So take your pick, do you want your durations to be right most of the time using a 9 hour day definition or do you want to stick with the standard default 8 hour day and get really confused when it comes to duration?

How to get TOC in Word 365 to display lowercase roman numerals when headers are uppercase?

I know it can be done manually using shift+F3 or the "change case" drop-down feature under the Font panel, but can it be done automatically and can it be done so when I update the TOC it remains lower case?

Any help is appreciated.


stefan blom say:

The page numbers in the TOC reflect the selected format in the Page Number Format dialog box. However, you can (re-)format the page numbers themselves (in the first section) with a numeric picture switch. In other words, choose uppercase Roman numerals for page numbers in the first section and then add \* Roman \* lower to the PAGE field in the footer (or header):

To show/hide field codes, press Alt+F9. When editing is done, press F9 to update and press Alt+F9 again to hide field codes.

Note that you will have to unlink the footer (or header) in the first section from those of the following sections; otherwise the switches will be added there as well.

charles kenyon say:

You could create two separate Tables of Contents: one for front matter and a second for the body.
