karl timmermans (outlook mvp 2012-2018) say:
>>
How does one go about solving this problem?
<<

First to cover off some items from your post..........

#1 - Nothing has changed between 2016 and 2019 in terms of how Outlook sets up an account. Yes it will use IMAP by default "if the server supports it" which in your scenario, it obviously does.

#2 - In reference to this being an "old problem" - for the sake accuracy, it's not a problem, it's the way the POP3 and IMAP protocols work regardless of what <email client> is used (of which Outlook is only one). Any <email client> will function the same way in terms of what gets downloaded/sync'd etc

#3 - The "unknown" in all of this in terms of your description "since I've started deleting emails off the server that I have already pulled down to my laptop" - there is one thing that is very important to understand which is -

Outlook uses an .ost file to store data locally for an IMAP account. The .ost file is a <mirror> of what is contained on the server. Hence, if you've deleted items from the server, those items would be deleted in the .ost file or to put this another way - you've lost those unless you saved a copy somewhere

#4 - An IMAP account stores contact/calendar data in folders with the added words "This Computer Only" in the folder name. If you have any data in "This Computer Only" folders then you absolutely want to back up (export) those folders to a PST file otherwise all that data will be lost if you delete the IMAP account from the profile.

#5 - If you've created any new email folders in the IMAP account, that data will be stored on the server. However, when you configure the same email account using POP3, it will only perform a one-way one-time download of "new" <Inbox> items. When the download is completed, the items will deleted from the server unless you have checked the "Leave Messages on server" option which is only available for POP3 accounts.

The safest approach to resolving this without losing any data would be to perform the following

#6 - Create a new PST file in this profile
(go to Home ribbon --> <new items> --> <More items> --> Outlook data file
#7 - When done, go to File --> Account Settings --> Data FIles tab --> highlight the PST file that was just created --> set this PST file as the default data file for the profile. This will create all standard folders in the PST file (i.e. Inbox, Contacts etc) and will be where the default contact/calendar folders are stored
#8 - Move any email folders that may have been created in the IMAP folder tree to the PST data file
#9 - If you have data in contact/calendar "This Computer Only" folders, copy that data to the corresponding folders in the PST file - to do this
a) select the contact folder --> press CTRL+A to highlight all the items --> right-click on the selected items --> <Move> --> <Other folder> --> select the folder in the PST file
b) for the calendar "This Computer Folder", change the view for this folder to a "List" and then rinse and repeat step (a) for calendar items

*** DO NOT copy items from the IMAP <Inbox> to the <Inbox> folder in the PST file
Once you're sure you have all the desired data copied to the PST file, you can safely configure the account using POP3 by

#10 - remove the IMAP account from the profile (this will also automatically delete the OST file that was associated with the account
#11 - <manually> configure the account as a POP3 account - do that by clicking on <advanced options> beneath where you entered your email address followed by checking the box for "Let me set up my account manually"

The default uninstall will remove everything. You can do a custom change to the installation to remove only selected applications.
