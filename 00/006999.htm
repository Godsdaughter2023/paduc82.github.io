While this isn't a permanent solution, its a decent workaround until the issue is solved.  I've found that closing the Explorer process and restarting it will "refresh" the taskbar icon and show all of the open programs again. 

There are a couple ways to do this, but I found that the easiest was to put together a little batch program to kill the explorer task and restart it.  Here's what I'm currently using:

taskKill /im explorer.exe /f
explorer.exe

Paste that into Notepad, go to File and hit Save As.  Name the file reload.bat, and then in the dropdown below the filename make sure to select All Files.  (You can change the name if you want, just make sure the .bat part at the end stays!)  Save it somewhere you can get to easily.  I have mine pinned to the start menu, but it doesn't really matter where it's saved.

Now, any time that your Excel windows vanish, you can run this file and it should get your taskbar showing everything again.  The one downside is that this will close any folders you have open at the time you run it.  Also, running it repeatedly might cause stability issues in Windows, since Explorer is built so heavily into the OS.  I haven't run into any yet myself, but just throwing the warning out there!

what does "disable linked styles mean" - Microsoft Community

jo chop say:

what does "disable linked styles mean"

Under Styles in Word 2010, what does "disable linked styles mean"?
stefan blom say:

The concept of linked styles was added as a means to easily apply heading formatting to the beginning of a paragraph and still have Word recognize it for the purpose of TOC creation (I don't remember when the term "linked" was introduced, though). When you make use of this feature (what I refer to as "painting" the formatting of a paragraph style), a special character style, linked to the paragraph style, is created by Word.

The side-effect of linked styles turned out to be that the additional character styles quickly accumulated to create a formatting mess with documents, and the first fix was the option to hide these "char" styles from view. More recently, in Word 2007, the option to prevent the use of linked styles was added.

To answer your direct question, yes, I would recommend selecting the "Disable Linked Styles" option.

stefan blom say:
It prevents you from applying paragraph styles to parts of a paragraph (meaning that it prevents you from "painting" the character formatting of the paragraph style on the selection).

kosh vorlon say:

Hi Webmaster,

 

Aside from Steve's good suggestions, you may also want to check with support for the program or programs you used to develop/design the site as the redirect is most likely "hidden" somewhere in the code.  This of course will be difficult {perhaps impossible} if you used basic free tools (HTML, CSS) and did it "by hand" but may work if you used something like Microsoft Web Essentials or a similar product.  Sometimes a Bing/Google search for "<insert coding language/program used> development forums" (or try design forums or programming forums) may lead you to someplace useful.

 

I'd also try a search of the code for the redirect location which may get you to the section that was added and from there it may be possible to search forward and backward (and revision date depending on what you're using) to find all the culprit software/code.

 

Finally, (though I suggest you use these just for testing if they are free and don't be sucked into buying anything), you could try the options in the following article: http://www.ehow.com/how_8685078_website-tested-antivirus.html.

 

Hope this helps.

 

Good luck!

 

Kosh
