Batch file gurus: how to copy only missing files between parallel - Microsoft Community

roy feague say:

Batch file gurus: how to copy only missing files between parallel folder trees using UNC paths

The problem: you have two identical directory structures, a source and a target.  (Let's say \\srchost\srcfolder and c:\destfolder.)  The goal: using a batch file on Windows XP, traverse the source folder tree and add any files to the destination tree that don't already exist there.  We're not concerned with dates, only whether the file exists.

The Replace command has a /a parameter that will do this for a single folder, but it won't traverse the tree.

Using the ForFiles command to iterate over the directories and pass them to Replace doesn't work because it doesn't support UNC paths. 

Using the For command winds up being ... hellish.  Here's the best I have so far, in the form of a batch file called with src and dest paths as parameters:

set srcpath=%~f1
set destpath=%~f2
for /r %srcpath% %%i IN (.) DO (
  set tmpsrc=%%~fi
  set tmpdest=%destpath%
  if not !tmpsrc!==%srcpath% call set tmpdest=!tmpdest!!tmpsrc:%srcpath%=!
  replace /a !tmpsrc!\*.* "!tmpdest!"
)

I think we can all agree that is really heinous, especially since it only works if you start the command interpreter with the /V:ON switch.  

This is turning into an unhealthy obsession for me.  I've wanted a simple batch file solution to this since, oh, around 1990.  Surely there must be a better way!  Can anyone enlighten me?

2.    Remember it has to work with UNC paths, so ForFiles is out.

Thanks for your ideas!  I'll really sleep better if there's a sane way to do this seemlingly simple task.

Hi Syed,

If the manufacturer set up your PC to run from a compressed Windows image file and included an option to restore factory settings, the “Restore factory settings” option will be removed after you upgrade to Windows 10, but other recovery options are still available.  The “Restore factory settings” option is not supported in Windows 10 on compressed operating systems.

To check if your device has a compressed operating system, right-click the Start menu icon, and click Disk Management.  If your device has a compressed operating system, you can see “Wim Boot” listed on the system drive:

 

To restore your system to factory settings, you can undo the upgrade and restore Windows 8.1 using the “Go back to Windows 8.1” option.  You can find this option in Settings -> Update & security -> Recovery. This functionality is available for 31 days after upgrading to Windows 10, provided that you do not delete the Windows.old file.  Once you’ve used the “Go back to Windows 8.1” option, you can then perform a “Restore factory settings” from the recovery options in Windows 8.1.

Note:

  • If you used a USB drive or other external storage to upgrade to Windows 10, the WIMBoot image will be removed at the end of the upgrade process.  You will need the USB drive you used to upgrade to Windows 10 plugged into the system for “Go back” to work during the 31-day window of opportunity.

  • If external storage such as a USB drive is will be removed.

For reference, you can find this listed in the Compatibility Report FAQ:

Why did I see a notification that I won't be able to restore factory settings? You’ll see this message if the manufacturer set up your PC to run from a compressed Windows image file and included an option to restore factory settings. After the upgrade, you won't be able to restore factory settings, but other recovery options will be available.
