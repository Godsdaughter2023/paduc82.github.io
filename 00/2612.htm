    gustav brock say:

    An autonumber is for identifying a record only.

    It should not be used for anything else, indeed not formatted in any way.

    If you need some kind of custom ID, create a separate field to hold this.

    ken sheridan say: An autonumber is merely a column of long integer data type whose distinct values are usually, though not necessarily, assigned automatically.  The column's Format property has no effect whatsoever on the value, merely on how it is displayed.

    Regardless of the format, you can easily reference the table in another table by setting the data type of the foreign key column in that table to a long integer number, which is the default when you select number as the data type in table design view.  It follows from this of course that you can then join the tables on the keys in a query.

    The value of an autonumber column or its corresponding foreign key should never be exposed to users, so there is no need to format them.  The assigned values are essentially arbitrary, with no semantic significance.  While they will usually be given incrementing sequential values, this cannot be guaranteed, so if there is a cogent business reason for an unbroken sequence the values should be computed in the application.  You'll find an example as CustomNumber.zip in my public databases folder at:

    https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

    Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

    If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

    This little demo file illustrates how to compute sequential numbers, either overall or by group, catering for possible conflicts in a multi-user environment.

    BTW, if you have used the 'lookup field' wizard when creating a column in table design view, note that this feature is regarded as both unnecessary and dangerous by most experienced MS Access developers.  For reasons what see: http://theaccessweb.com/lookupfields.htm

    scottgem say:

    I disagree with Gustav and agree with Ken. Gustav is not wrong, rather this is a philosophical difference of opinion. An AutoNumber has one purpose only and that is to uniquely identify a record. However, I see nothing wrong in using one as a public facing identifier. 

    As Ken pointed out, a format does not affect what is actually stored. So your foreign keys or search variables, need to be Long Integer. 

    You also refer to a lookup, are you using a lookup field on the table? That is not recommended. If you want to lookup the ID, then use a Combobox on your form with a RowSource like:

    SELECT ID, formattedID FROM table;

    Then set the ID column as hidden.
