http://www.mediafire.com/view/3kc5ayvl8zv7g4v/ForMano_001.xls

The code for the UDF is here (and of course it is in the workbook)

Function SummaryTotal(customerName As String, _
 Optional minimumDays As Variant, _
 Optional maximumDays As Variant) As Currency
'depending on what parameters are passed, it will act like
'any of the 3 SUMPRODUCT() formulas you are now using.
'  See each section below for use of the function and
'  how to 'call' it from a worksheet cell
'  It can work like any of these formulas:
'   =SUMIF($C$2:$C$15,$G3,$D$2:$D$15)
'   =SUMPRODUCT($D$2:$D$15,($C$2:$C$15=$G3)*($A$2:$A$15>TODAY()-I$1))
'   =SUMPRODUCT($D$2:$D$15,($C$2:$C$15=$G3)*($A$2:$A$15>TODAY()-J$1)*($A$2:$A$15<=TODAY()-I$1))

'Known information:
' the details table will be on 'Sheet1'
'   The details table uses these columns for types of information
'   data starts on row 2 and continues to
'   1 row above last used row in column A
'   A = Date of event
'   B = Invoice Number
'   C = customer name
'   D = amount

  Const detailFirstDataRow = 2
  Const offset2Invoice = 1 ' from column A to column B
  Const offset2Customer = 2 ' from column A to column C
  Const offset2Amount = 3 ' from column A to column D
 
  Dim detailsWS As Worksheet
  Dim detailsDatesList As Range
  Dim anyDetailDate As Range
  Dim cutoffDate1 As Date
  Dim cutoffDate2 As Date
 
  Dim theTotal As Currency ' temporary value to be returned.
 
  'this next statement is "expensive": anytime anything on the
  'sheet changes the cells using this function will all be recalculated
  Application.Volatile ' so it will update when changes take place
 
  Set detailsWS = Worksheets("Sheet1")
  'set reference to all dates in the details table
  Set detailsDatesList = detailsWS.Range("A2:" _
   & detailsWS.Range("A" & Rows.Count).End(xlUp).Offset(-1, 0).Address)
  'do they want just a grand total?  if yes, then both 'Days' values will be missing
  'call with formula similar to this on the worksheet
  '   =summarytotal(G15)
  '   G15 = customer name to get total for
  If IsMissing(minimumDays) And IsMissing(maximumDays) Then
    'yes, just get grand total for customerName
    'basically emulating this formula:
    '   =SUMIF($C$2:$C$15,$G3,$D$2:$D$15)
    For Each anyDetailDate In detailsDatesList
      If UCase(anyDetailDate.Offset(0, offset2Customer)) = UCase(customerName) Then
        theTotal = theTotal + anyDetailDate.Offset(0, offset2Amount)
      End If
    Next ' end Each anyDetailDate loop
    SummaryTotal = theTotal
    Exit Function
  End If
  'do they want the first day limit total (Below 30 days)?
  'if yes, then only 1 date will be provided
  'call with formula similar to this on the worksheet
  '   =summarytotal(G15,I$13)
  '   G15 = customer name,  I$13 is 30 days in this case.
  If IsMissing(maximumDays) Then
    'yes, basically emulating this formula
    '   =SUMPRODUCT($D$2:$D$15,($C$2:$C$15=$G3)*($A$2:$A$15>TODAY()-I$1))
    For Each anyDetailDate In detailsDatesList
      If UCase(anyDetailDate.Offset(0, offset2Customer)) = UCase(customerName) Then
        cutoffDate1 = DateSerial(Year(Now()), Month(Now()), Day(Now())) - minimumDays
        If anyDetailDate > cutoffDate1 Then
          theTotal = theTotal + anyDetailDate.Offset(0, offset2Amount)
        End If
      End If
    Next ' end Each anyDetailDate loop
    SummaryTotal = theTotal
    Exit Function
  End If
  'all parameters are present, so we want totals within a date range
  'basically emulating this formula:
  ' =SUMPRODUCT($D$2:$D$15,($C$2:$C$15=$G3)*($A$2:$A$15>TODAY()-J$1)*($A$2:$A$15<=TODAY()-I$1))
  'call with formula similar to this on the worksheet
  '   =summarytotal($G15,I$13,J$13)
  '   G15 = customer name to get total for
  '   I$13 = lower date limit (as 30)
  '   J$13 = upper date limit (as 45)
  For Each anyDetailDate In detailsDatesList
    If UCase(anyDetailDate.Offset(0, offset2Customer)) = UCase(customerName) Then
      cutoffDate1 = DateSerial(Year(Now()), Month(Now()), Day(Now())) - maximumDays
      cutoffDate2 = DateSerial(Year(Now()), Month(Now()), Day(Now())) - minimumDays
      If anyDetailDate > cutoffDate1 And anyDetailDate <= cutoffDate2 Then
        theTotal = theTotal + anyDetailDate.Offset(0, offset2Amount)
      End If
    End If
  Next ' end Each anyDetailDate loop
  SummaryTotal = theTotal
End Function

Adding to what Le Boule already mentioned, the list found in "View detailed scan results" is simply a partial listing of the malware "families" that the MSRT is able to detect and clean.

Malware Families Cleaned by the Malicious Software Removal Tool

Though this is still only a small portion of the malicious software which can be detected by the more complete Microsoft antivirus products such as the on demand Microsoft Safety Scanner, that's the purpose since these are the most common malware Microsoft's security products have seen in the wild.

"The Microsoft Windows Malicious Software Removal Tool removes specific, prevalent malicious software families from computers running compatible versions of Windows. Microsoft releases a new version of the tool on the second Tuesday of every month, and as needed to respond to security incidents.                                    

Additions are made each month to address the latest threats."

The MSRT has the benefit of making the download package much smaller than others like the Safety Scanner, but the negative of only checking for a small number of different malware, so it's most useful when performing a "Quick scan".

If you intend to perform an exhaustive "Full scan" to check for any potential malware, then either the Microsoft Safety Scanner or a 3rd-party on demand tool might be a better choice.  That's due to the fact that a Full scan will always take more time to examine every single file and registry entry on a PC, but will be less effective if it's only searching these for a small number of all known malware.

The percentage of PCs actually being cleaned by the MSRT is very small (less than .2% - .4%, according to the article below), and although Microsoft reports that the MSRT is actually finding things on PCs that are running antivirus software, I think we would probably need to qualify that statement. It‚Äôs quite likely that the vast majority of these detections were made on PCs with older operating systems that either weren‚Äôt running any AV protection at all, or that were running an AV app with an expired subscription. When I was servicing PCs, I‚Äôd frequently get a machine that was still running its preinstalled AV app years later, without ever having renewed the subscription. So yeah, in a scenario like that, the MSRT might be making some legitimate detections.

https://blogs.technet.microsoft.com/mmpc/2016/05/26/limited-periodic-scanning-in-windows-10-to-provide-additional-malware-protection/

But it's unlikely that the MSRT is making legitimate detections on PCs that are running a supported OS and an up-to-date AV app ‚Äì unless maybe the user hasn‚Äôt run a manual (or scheduled) scan for quite a while. In that case, most AV solutions can still miss archived malware, since only a few of them actually provide the option for scanning archived files with real-time protection. But if you want to scan for malware that's hidden in archive files then you should run occasional Full Scans with Windows Defender Antivirus. This will unpack and scan the contents of archive files just like the MSRT does ‚Äì but it will also scan your files with the full Microsoft definition set instead of just the small subset of definitions that are included in the MSRT. The MSRT really just isn‚Äôt up to the task of finding hidden malware, because it's blind to more threats than it can see. And when this archaic tool starts running forever and detecting threats that aren‚Äôt really there ‚Äì well then maybe it‚Äôs time to move on.
