Public Function ConcatValues(strSeparator As String, ParamArray arrVals())
 
    ' Pass this function a character or characters
    ' to be used as the separator followed by the values to be combined
    ' For example: strFullName =
    ' ConcatValues(" ",FirstName, MiddleName,LastName)
    Dim X As Integer, strLine As String
    
    For X = 0 To UBound(arrVals)
        If Not IsNull(arrVals(X)) And Trim(arrVals(X)) <> "" Then
          strLine = strLine & strSeparator & arrVals(X)
        End If
    Next

End Function

john - project say:

fordmgw,

The short answer is "no", but I'm curious, what kind of corporate policy would forbid users from setting the date format to something applicable to the region?

Now the longer answer. You could set up a series of custom fields wherein the date format is structured as you want it. For example, let's say your regional settings are mm/dd/yy and you want to display dd/mm/yy. For each date field you want displayed in the latter format you will need a matching custom field with the following formula:

Text1=format([Start],"dd/mm/yy")

Then you might also want to rename the Text1 field as Start-F (i.e. Start reformatted)

Hope this helps.
