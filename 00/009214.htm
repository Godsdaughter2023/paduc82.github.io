zixuan yuan msft say:

Hi Mike_Park_88,

 

Thanks for your updating.

 

Based on our investigation, this problem may be caused by one of the following:

1. The line spacing of your text may be set to an exact height. When you set the line spacing to Exactly and select a measurement that is equal to or less than the height of the font, underscore characters may not be displayed and characters that have an ascender, descender, or an umlaut may be cut off. For example, if you set line spacing to Exactly 9-point for a 10-point font, underscore characters may be cut off.

NOTE: This problem may occur differently with different fonts.

2. The printer driver may incorrectly substitute screen fonts at certain view percentages. If you change the printer driver, you may notice an immediate difference, depending on the view you are using.

3. The zoom percentage that the document is set to may cause characters to have the top or bottom portion of the screen font cut off. You can adjust the zoom percentage from 75 percent to 100 percent in most cases to correct this visually.

 

Considering this problem is a display-only problem. It won’t affect when we print the file.

As you mentioned, you can use one or more of the following methods appropriate for your situation:

1. Change the Document Zoom Setting. (View>Zoom)

2. Change the Document View. (View>Outline)

3. Change the Line Spacing.

kosh vorlon say:

Hi Muahammad,

 

It is not likely that you are infected if this is the only symptom you have.  It may be that some of the Windows files are corrupt (perhaps because of a bad sector on the hard drive or due to simple file damage or possibly the use of a registry cleaner (if you've ever done that) or just simple corruption that sometimes happens).  Malware is not the only cause by any means (and while it's possible, it could also be the result of a cleared up malware infection that corrupted files before being removed - but like Steve said, I've never heard of any infection of any type doing what you're experiencing - it's always been some type of hardware, driver, or system file problem - including the registry). 

 

And even if the mouse works fine on another computer and the drivers are updated and properly installed, the Windows OS can be the culprit.  I personally would be trying chkdsk /f /r as well as SFC /scannow to start (after triple-checking the drivers) and then move from there to other diagnostic procedures to try to find the cause - but you'll get more help in the Windows Forum (and frankly I'm not sure if the best place to start would be Hardware and Drivers, Desktop and Personalization, or System Recovery and Repair).  I suspect any would do, but I personally would start with Hardware and Drivers even if you're certain it isn't that - they may still be able to find the Windows source of the problem or move you to a different forum is a more intensive troubleshooting effort is needed.

 

I also suggest you go to the technical support page of your computer manufacturer's website and see if they have any suggestions for such a situation (especially if everything is original including especially the mouse).  If the mouse was separately purchased, try that manufacturer's web site and look for similar information and help with this type of problem.
