Hi Lunar,

Sorry to hear that the issue is not fixed by the troubleshooting steps provided earlier.

If you receive Windows Update error 80070002 while checking for updates, you need to remove the temporary update files that Windows uses to identify updates for your computer. To remove the temporary files, stop the Windows Update service, delete the temporary update files, restart the Windows Update service, and then try to check for Windows updates again

Follow the steps below to stop Windows update service:

  1. Press Windows + R and type services.msc
  2. Find the Windows Update service, right-click it, and then click Stop

To Delete the temporary update files follow the steps below:

  1. Double-click the local hard disk where Windows is installed, typically drive C
  2. Open Windows folder, and then double-click the SoftwareDistribution folder.
  3. Double-click the DataStore folder to open it, and then delete all files and folders in the folder.
  4. Click the Back button. In the SoftwareDistribution folder, double-click the Download folder to open it, and then delete all files in the folder.

Restart the Windows update service:

  1. Press Windows + R and type services.msc
  2. Find the Windows Update service, right-click it, and then click Start

The Recovery Disk (System Repair Disk) is more like a ISO that is burned to a CD all in one process...

You can find this Option on the same screen where you would make a System Image Backup...

In File History click on System Image Backup on the lower left, and then you will see the options on the upper left of the window that opens...

This uses something called WinRE which is a set of Windows files that enable you to boot your computer and do repairs... It is the same process as booting from Windows Installation media, but without the actual windows installation...

It will do the same type of repairs as the installation media will...

If the files needed to create this CD are missing, it would explain why your Recovery Drive won't create.

There is a remote chance that if you put a windows installation DVD in the drive before you start to make a Recovery Drive, the process may be able to find the needed files there...

In the process of checking this stuff out, I found that a Recovery Drive is of no more benefit than installation media... All it did was reinstall a "Clean Install" of Windows...

I was expecting that it would retain the installed programs as well, but it did not...

If you're machine has a Partition to Restore To Factory the Recovery Drive may include that, and be of more benefit, but if you are backing up a Clean Install, it isn't worth the trouble...

Creating a System Image backup would be of much more benefit... It backs up virtually every program, and setting that exists at the time it is made...

That doesn't answer the question as to why the Recovery Drive isn't being created though...
