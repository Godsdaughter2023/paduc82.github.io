Multi-value combo box not populating on form

Thank you for any help that you may be able to provide.

scottgem say:
What is the Controlsource of the combobox control on the form?

carlainjax say: The Controlsource is a field in my master table with as a value list with a lookup.

ken sheridan say: If the form was created with the form wizard, or if you added the combo box to the form manually by dragging it from the field list, it should look and behave exactly as it does in the table's datasheet.  Try deleting the control from the form and adding a new one in form design view by dragging the field onto the form from the field list, which is opened by means of the 'Add Existing Fields' icon in the Tools area of the Design ribbon.

I notice you've used a value list as the RowSource property.  A value list should only be used where the values are a set immutably fixed in the external world.  Days of the week or months of the year would be examples.  Otherwise you should use a query as the RowSource property, drawing on a table in which each value is represented by a row.  This can be built in form design view in the same way as a normal query, but does not need to be saved as a separate querydef object; the query's SQL statement will be saved as the RowSource property of the control.

carlainjax say: I added a new combo box with a table/query as my RowSource, but it will not let me make multiple selections.

ken sheridan say:

First make a copy of the table under a different name as a back-up.

You then need to create and fill the single column table which supplies the values if you haven't done so already, making the single column the primary key. 

Then you can amend the definition of the multi-valued field in your current table in table design view, by changing the RowSourceType property in the field's properties sheet to Table/Query.  Then change its RowSource property to an SQL statement, so if the table of values were named Foo and its single column (field) were named Bar for example, the SQL statement would be:

SELECT [Bar] FROM [Foo] ORDER By [Bar];

Save the table, and you should be able to open it and the combo box will work exactly as before.

In the form bound to the table, in design view delete the existing combo box.  Then replace it with a new one by dragging the field from the field list and drop it onto the form.  Save the form, and it should behave exactly as it does in the table's datasheet.

Because you are now getting the values from the new single column table, if you want to augment the values you can simply insert more records in the new table.
