Instead of using "Filling in forms" protection, select the "No changes (Read only)" protection type. Yes this also disables the watermark so you can not add or remove the water mark

That's not surprising, because a watermark is just a floating picture anchored in the header.

If you require users to remove or replace a watermark in a read-only protected document, you'll have to provide a macro that temporarily turns off the protection, does the necessary action, and turns the protection on again. This can be problematic if users don't allow macros to run, or if the document is stored in an untrusted location so that macros are disabled.

The delivery address on an envelope uses the Envelope Address style that includes a Frame into which the address is inserted.

To modify it, in a blank document, press CTRL+SHIFT+s to display the Apply Styles dialog and the in the Style Name control type Envelope Address and then click on the Modify button.  Then in the Modify Style dialog, expand the Format dropdown and select Frame and reduce the height of the Frame.

Before dismissing the Modify Style dialog, select the radio button for New documents based on this template so that the changes will be retained.

steve rindsberg say:
In addition to John's merge tool and mine, you could start with a presentation that has something unique like @NAME@ wherever you later need the actual name of the person.

Then use Replace ( Home tab, Editing group ) to search for @NAME@ and replace it with the person's name.

[added later]

Before doing the replacements, save the presentation with a new name so you don't accidentally mess up the original with all the @NAME@ thingies.
