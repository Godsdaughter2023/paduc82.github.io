Need to handle match not found when MATCH function does not find a - Microsoft Community

tom wikle say:

Need to handle match not found when MATCH function does not find a match

I have tow worksheets I need to run a match on for values in Column A. I need a way to handle a match not found that keeps the macro running and not stop the macro when a match is not found. The code is:

 

shRow = Application.WorksheetFunction.Match(ssfReport, Worksheets("Score_History").Range("A:A"), 0)

 

If a match is not found, shRow is 0 and an the macro stops and errors out. If there is no match in a row, I want the macro to continue on to the next row. Not all rows in one worksheet will match the other rows in the other worksheet.

jeeped say:
VBA'a IsError(...) function can be used to catch an #N/A before it halts macro execution but you need to call it as a test before calling it to assign a value.

If IsError(Application.Match(ssfReport, Sheets("Score_History").Range("A:A"), 0)) Then
'do something on #N/A
Else
    shRow = Application.Match(ssfReport, Sheets("Score_History").Range("A:A"), 0)
End if

I've played around with the On Error Resume Next and then test for shRow Is Not Nothing but I find the above more to my liking.

BTW, generally speaking you need Application. or WorksheetFunction. but not both. At least not for Match(..). Worksheets can likewise be truncated to Sheets to keep everything important in view.

paulcordts say: This should work, but check to make sure that shRow is getting the correct value.

shRow=0 'Sets shRow to 0 in case of error
On Error Resume Next 'Forces macro to go to the next line 
shRow=Application.WorksheetFunction.Match(ssfReport,Worksheets("Score_History").Range("A:A"), 0)
On Error Goto 0 'Sets error handling back to normal

todtown say:

You can do a countif statement before your Match statement to test if the value is there. Something like this:

 

    If Evaluate("=COUNTIF(Score_History!$A:$A,""" & ssfReport & """)") <> 0 Then _
    shRow = Application.WorksheetFunction.Match(ssfReport, Worksheets("Score_History").Range("A:A"), 0)

 

chip pearson say:

You're calling Application.Match twice under normal circumstances. That isn't very efficient. Instead,  catch the error in  a Variant and convert to shRow as Long if successful.

 

 

Dim V As Variant
Dim shRow As Long
V = Application.Match(ssfReport, Sheets("Score_History").Range("A:A"), 0)
If IsError(V) Then
    '  ssfReport not found in A:A.

Else
    shRow = CLng(V)
End If

 

See http://www.cpearson.com/excel/CallingWorksheetFunctionsInVBA.aspx for a discussion of whether to use WorksheetFunction to access the built-in worksheet functions (e..g, Application.Match versus Application.WorksheetFunction.Match). There are differences in error trapping.

In the applicable worksheet code module (not a regular module)  paste the following code...

'---
Private Sub Worksheet_Change(ByVal Target As Range)
 'Jim Cone - Portland, Oregon USA - June 2015
  On Error GoTo ReturnToNormal
  Application.EnableEvents = False

  Dim objList As Excel.ListObject
  Dim rngRow As Excel.Range

  On Error Resume Next
  Set objList = Me.ListObjects(1)
  On Error GoTo ReturnToNormal

  If objList Is Nothing Then
    GoTo ReturnToNormal
  ElseIf Application.Intersect(Target.Cells(1), objList.Range) Is Nothing Then
    GoTo ReturnToNormal
  Else
    With objList
       Set rngRow = .Range.Rows(.Range.Rows.Count).Cells

       '[EDIT]
       If Application.WorksheetFunction.CountA(rngRow) = _
         .Range.Columns.Count Then .ListRows.Add
    End With
  End If

ReturnToNormal:
  Set rngRow = Nothing
  Set objList = Nothing
  Application.EnableEvents = True
End Sub

Create Plan Failed when creating Planner in Microsoft Teams - Microsoft Community

ed hansberry (work acct) say:

Create Plan Failed when creating Planner in Microsoft Teams

There was a thread on this a few months ago, but no resolution. I found a workaround. Mainly so next time it happens to me, Google will bring me back to this thread.

I am the owner of a team and when I tried to create a plan as a new tab, it simply said "Create Plan Failed" after pressing save. It also didn't show the existing plans in the team to create a new tab from an existing plan.

I went into the Admin panel and I was the Owner, but I was not a Member. I could do everything in that team except create plans. I could create planners in other teams.

So while the ultimately solution might be shorter than the below steps, these steps work.

  1. Make someone else an owner through the Office 365 portal. Save.
  2. Remove myself as an owner. Now I am not in the team.
  3. Make myself a Member. 
  4. Make myself an Owner again.

Now I can create and see other plans. Something in my permissions was jacked up and this cleaned it up.

You're welcome. :-)
