Project 2010 Changing Default Calendar to a 7 day working week - Microsoft Community

aspiredigital say:

Project 2010 Changing Default Calendar to a 7 day working week

Hi - I'm having trouble changing the working week to a 7 day working week to an already populated schedule.  It is not bringing in the resources in over the weekend automatically.  Is this something i have to change using manually scheduled tasks?

john - project say:
spiredigital,

No, the calendar has nothing to do with manually scheduled tasks.

Assuming you created a custom 7 day workweek calendar and didn't simply modify the Standard calendar, make sure the Project calendar (Project > Properties group > Project Information) is your custom 7 day workweek calendar. Then, as long as you didn't change any of your resource's Base Calendars (View > Resource views > Resource Sheet) and you didn't set any Task Calendars, your plan will follow the 7 day workweek.

However, if you have tasks already completed or in progress before you change the calendar, those tasks will not change.

Hope this helps.

John

tom_bpc say:

Aspiredigital,

First, definitely don't use manually scheduled tasks.

Second, I suspect that you changed the default Project Calendar by creating and applying a new 7dx8h calendar rather than modifying the existing "Standard" calendar.  (That would be the recommended approach in any event.)  Each resource's calendar inherits most information from its "Base calendar," which was set (by default) to the Project Calendar of the active project at the time of resource creation; this is probably the "Standard" calendar (i.e. 5dx8h).

To get what you want, go the resource sheet and ensure that the Base Calendar of each resource corresponds to your new 7dx8h Project Calendar.

Good luck, tom
