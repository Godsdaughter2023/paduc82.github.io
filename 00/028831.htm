teylyn say:

Hello,

you're on the right track with Book.xlsx in the XLSTART folder. 

The next thing you need to do is to disable the Start screen when Excel starts. Click File > Options > General and untick "Show the start screen ...".

Now, when Excel starts, it will use the Book.xlsx form the XLSTART folder to create a blank workbook.

If, however, you click File > New > Blank Workbook, or if you use Ctrl-N, it will not use the file from the XLSTART folder.  For that scenario, you may want to consider using a custom template.

There was a similar discussion recently on Technet (click here), with a suggestion to start a new Excel instance for each file, but I don't think that this is practicable, since interactions between two files differ a lot when the files are open in two different instances of Excel.

Word Merge Switches - Microsoft Community

brenda a reid say:

Word Merge Switches

I have a one page merge document where one side is English and the duplex side is French.  The data file is Excel so I'd like the amounts on the French side to be the French format.  Is there a merge format switch I can use?  i.e.  make $4,123.45 to be 4 123,45$ on the French side?

 

 

a. user say:

I've put this together in a bit of a hurry so some details may need fixing, but...

It's not completely straightforward for a number of reasons. In the case where all your numbers are positive up to around 999,999,999,999.99 and you don't mind how many spaces you have in front of the digits, you can use something like this:

{ ={ MERGEFIELD myamount }*100 \#"### ### ### ##0','00'$'" }

All the {} are the special field braces you can insert using ctrl-F9, and you subsitute your merge field name for "myamount".

If you need more digits, you would need something more like this (there are various ways to do it, and you will need to check how many digits it will work for.

{ QUOTE "{ SET n { MERGEFIELD myamount } }{ =INT(n) \#"-### ### ### ### ##0" },{ =ABS((n-INT(n)))*100 \#00 }$" }

If you don't want all those spaces, I suggest you look at the following document for a different approach:


http://goo.gl/FqikW

I agree with Dana's explanation on use of AND / OR,

In VBA or even Excel worksheet, the general idea of condition is

If  condition is true TRUE  (if more than one conditions, all conditions must return TRUE)

Run seomthing

In your code

If Cells(i, 7).Value <> "02" Or Cells(i, 7).Value <> "06"....

Just imagine if (i,7) = "02"

For the first condition, Cells(i, 7).Value <> "02" returns FALSE

For the 2nd condition, Cells(i, 7).Value <> "06", returns TRUE

OR means as long as one of the conditions is met, overall it will return TRUE

But if you use AND, all conditions must return TRUE in order that the overall returns TRUE, if one of the conditions is not met, it will return an overall FALSE
