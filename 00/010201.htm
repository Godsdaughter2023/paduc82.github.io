Epp.msi - Microsoft Community

gingasue say:

Epp.msi

I am getting a msg from Microsoft Security Client (while trying to run updates) stating, "The feature you are trying to use is on a Network source that is unavailable". It is looking for a folder containing the installation pkg 'epp.msi'. I have run a search and this file cannot be found. I have Windows XP. What do I do?
prashantkumar96 say:
Please confirm that you get this error when you update MSE's virus definitions (thorough Update tab) as I have not seen any case where updating MSE throws this error. It is possible that MSE is upgrading and this error is shown.

This error occurs when a MSI file with necessary logic and data required for removal of MSE is absent from the system. This file is placed in C:\Windows\Installer with name '<random string>.msi'. Originally these logic and data comes in epp.msi package in installation file of MSE and during installation necessary part is extracted from epp.msi, packaged in another MSI file and placed in C:\Windows\Installer. That's why the installer asks for epp.msi when it do not find that MSI file.

OK, enough (actually more than enough, <wink>) explaining part, now the solution:
A backup of epp.msi file is saved in "C:\Program Files\Microsoft Security Client\backup\x86" folder. If the epp.msi file is there in that folder then copy it to another folder like C:\MSE_tmp and point to it in the dialog box you get during the error. Copy dw20shared.msi file to MSE_tmp also, just in case if MSE asks for it after dealing with epp.msi.

If the file is not in x86 folder then you have to download the setup file of MSE and remove MSE through it. For it follow Alternate method #1 given at the bottom of this article:
Uninstalling MSE

If this also fails then run Uninstall Fix it method given in the same article then download and install latest version of MSE from here:
Microsoft Security Essentials

NOTE: You can get epp.msi file from MSE setup file by following thee steps:
  1. Place mseinstall.exe (MSE installation file) in let's say D: drive.
  2. Go to Start --> Run or press Win key + R to open Run dialog.
  3. Type "cmd" and hit enter.
  4. Command Prompt windows will open. Type "D:\mseinstall.exe /x"
    [Notice space between ...exe and /x]
  5. It will then ask for a destination folder, type D:\MSE_setup and hit enter.
  6. After its extraction go to D:\MSE_setup\x86 folder and you will find epp.msi there.

jay freedman say:

The SaveViewMulti add-in (now superseded by SaveViewLocMulti) contains code that runs every time the active document is saved. That currently includes both manual saves and automatic saves started by the AutoRecover feature. Also, if you're opening documents directly from OneDrive or SharePoint in the latest versions of Word, the AutoSave feature will activate, and SaveViewMulti will run during that, too.

It may possible for the code to distinguish between saves that are started manually (File > Save, File > SaveAs, and their keyboard shortcuts) and those that are started programmatically, but I don't yet know how to do that. Apps installed on SD cards do not survive a phone reset or moving the card from phone to phone. Anytime the phone senses the SD card has changed to a different card this happens and the app has to be downloaded again.

This should not happen with just a phone turn off and back on.

I do not move apps that save data, like games I routinely play or apps I don't want to setup again to the SD card, I put them on the phone.
