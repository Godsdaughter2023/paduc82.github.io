Thank you for the suggestion to look at specific events. I have this same issue with NisSrv. It is now consuming a lot more memory than usual and more memory than it had been in the past. I used PowerShell to do a bit of system analysis on my machine. Below is the script that I created and ran in an administrative context to troubleshoot the issue.

$events = Get-WinEvent -FilterHashtable @{ProviderName='Microsoft-Windows-Resource-Exhaustion-Detector'} | Sort-Object TimeCreated

# Collect non-informational exhaustion events (i.e. warning, errors, etc.)
$exhaustions = $events | Where-Object LevelDisplayName -NotLike '*Information*'

# Collect Microsoft Network Realtime Inspection Service exhaustion events
$nise = $exhaustions | Where-Object Message -Like '*NisSrv.exe*'

# Collect Windows Update events
$wuevents = Get-WinEvent -FilterHashtable @{ProviderName ='Microsoft-Windows-WindowsUpdateClient'}

# Collect Device Setup events
$devsups = Get-WinEvent -FilterHashtable @{ProviderName = 'Microsoft-Windows-DeviceSetupManager'}

# Collect Windows Defender Client events
$defen = Get-WinEvent -FilterHashtable @{ProviderName = 'Microsoft-Windows-Windows Defender'}

# Sum up and sort the system change events collected
$sume = $wuevents + $devsups + $defen

# Display culprit exhaustion events
$nise  | Select-Object -Property TimeCreated, Id, LevelDisplayName, TaskDisplayName, Message | Format-List

# Collect system change and software update events up until the earliest (aka first) NisSrv exhaustion event
$LookHere = $sume | Where-Object TimeCreated -le $nise[0].TimeCreated | Sort-Object TimeCreated  -Descending

# Output in a new window to browse adhoc through the events
$LookHere  | Select-Object -Property TimeCreated, Id, LevelDisplayName, TaskDisplayName, Message | Out-GridView

In my case on my computer, the NisSrv exhaustion events started January 8, 2018.  The nearest system or software update event recorded is: "Windows has started installing the following update: Definition Update for Windows Defender Antivirus - KB2267602 (Definition 1.259.1279.0)"

To resolve the NisSrv memory grab I have to perform a "shutdown".  A simple restart does not do it and I am not able to successfully restart *all* the Windows Defender services.

I hope Microsoft releases an update that may resolve this issue. It does force me to shutdown my computer more often than I typically would. And, of course shutdowns shorten the life of a computer

or any machine where temperature deltas are abrupt.
