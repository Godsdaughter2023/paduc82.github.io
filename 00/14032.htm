macropod say:

If you insert the surname2 mergefield via the Insert Merge Field button, it will appear as «surname2». Pressing Alt-F9 toggles the field code display between «surname2» and { MERGEFIELD surname2 }. With it displaying { MERGEFIELD surname2 } you can add the \b switch, thus:

{MERGEFIELD surname2 \b " "}

The \b and \f switches insert unformatted content before and after the b and f, respectively. See: Insert>Quick parts>Field>mergefield. Microsoft did have an explanation at http://office.microsoft.com/en-us/word-help/field-codes-mergefield-field-HP005186172.aspx but that web page seems to have disappeared and there's not even a mention of mergefields in its field reference guide: https://support.office.com/en-us/article/Field-codes-in-Word-b0536e6f-5a23-4408-b90f-e7fc24c75416?CorrelationId=93d015dc-0b26-4ef6-9982-3e04331b178b&ui=en-US&rs=en-US&ad=US. Go figure!

As for the IF field, do be aware that neither the field braces (i.e. '{ }') nor the chevrons (i.e. '« »') can be typed of copied from a site such as this. The field brace pairs (i.e. '{ }') are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac) and the chevrons are how mergefields normally appear when you insert them from the 'Insert Merge Field' dropdown.

macropod say:

It's not clear from your posts what else might be of the same line as the «surname2» field. If that and the preceding spaces are all there is on the line, you could use:
{MERGEFIELD surname2 \b "¶

"}

where ¶ is a real paragraph break or line break and you delete the paragraph break or line break from the previous line. Alternatively, you could use:

{MERGEFIELD surname2 \b " " \f "¶

"}

and delete the paragraph break or line break from the end of the «surname2» line.

For more information, see Mailmerge Empty Space Suppression and Mailmerge Empty Line Suppression in the Mailmerge Tips and Tricks thread at:
http://www.msofficeforums.com/mail-merge/21803-mailmerge-tips-tricks.html
or:
http://windowssecrets.com/forums/showthread.php/163017-Word-Mailmerge-Tips-amp-Tricks

As for the header, you could use a field coded along the lines of:
{IF{MERGESEQ}= 1 "Header Text{SET Index {=INT(«Unit»/100) \# 0}}" {IF{=INT(«Unit»/100) \# 0}<> {Index} "Header Text{SET Index {=INT(«Unit»/100) \# 0}}"}}

where 'Header Text' is whatever you want to display in the header, including additional mergefields.

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. Likewise, you can't type or copy & paste the chevrons (i.e. '« »') - they're part of the actual mergefields, which you can insert from the 'Insert Merge Field' dropdown. The spaces represented in the field construction are all required. A macro to convert the above example to a working field code can be found in the Mailmerge Tips and Tricks threads mentioned above.
