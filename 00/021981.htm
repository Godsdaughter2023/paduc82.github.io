brian tillman [outlook mvp 2007-2019] say:
Your phone accesses your mailbox using a protocol called Exchange ActiveSync.  That's a two-way channel so that whatever you do on the phone is synched to the mailbox and whatever you do in the mailbox using a web browser is synched to the phone.  Your Mac, on the other hand, cannot use Exchange ActiveSync.  Instead, it uses a protocol called POP, which is, with one exception, a one-way protocol and only for new messages.  Messages flow from the mailbox to your Mac, but nothing can go the other way.  Once it has downloaded messsages from the mailbox, your Mac cannot detect changes made to that mailbox from the web of from your phone.

The normal way POP operates is that once the POP client downloads new messages (and here is the one exception I cited earlier), it sends a command to the server to delete the messages it downloaded.  POP is a very old protocol and when it was created, mailboxes were very small and could not hold a lot of messages.  Thus, it was easy to fill up a mailbox and so, after downloading the messages, the client would tell the server to delete the messages from the mailbox so it would be able to accept more.  This behavior is still the default in most POP clients.

The Hotmail/Outlook.com server has much more capacity than servers used to have.  So, you don't need to delete messages after downloading them if you're using POP.  Moreover, many people want to be able to see the same messages when using a web browser as when using a mail client (like Mac Mail).  Since most POP clients still are configured with the default to delete messages after downloading them, the mailbox uses the POP folder, moving messages to that folder when it receives a delete command, instead of actually deleting the messages.  That way, it tries to protect messsages from those people who have forgotten or do not know how to configure their mail clients to leave the messages on the server instead of deleting them.  You can enable an option on the server to obey the delete command and not use the POP folder, but I think most people prefer to keep their messages.
