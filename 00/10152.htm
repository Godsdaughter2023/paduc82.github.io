Whether you can transfer / install your copy of Office to a new machine depends on the type of Office license you purchased.  If you bought it factory installed and activated, you have an "OEM" license that does not allow the transfer.

allanlu_msft say:

Hi Mindy,

Thanks for you effort.

The error indicates that the issue may related to that the lookup column exceeds the thresholds in the library you sync. We have got some other cases about the situation, for your reference:
https://community.office365.com/en-us/f/148/p/349609/949323#949323
https://community.office365.com/en-us/f/154/p/414312/1030825#1030825

Just as LingYu said, it’s not feasible to change the threshold in SharePoint Online because increasing this value from the server side is likely to cause performance problems for multiple tenants. To resolve this issue, we need to reduce the number of lookup columns in the affected SharePoint Online list, and then try to synchronize the library. Do this until the error is no longer triggered. You can follow the steps in this KB to do this.
https://support.microsoft.com/en-us/kb/2927386

Note: If you do not have permission to do this, you can involve the admin into this case for help.
Lookup column types contain “Lookup”, “Person or Group”, “External Data” and “Managed Metadata”.

If the issue persists after trying this, feel free to let us know and we will continue to assist you.

Regards,
Allan
