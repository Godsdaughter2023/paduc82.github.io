jeeped say:

I will assume that the Asda, Tesco and ShoppingList worksheets have all the same columns with the exception that the ShoppingListy worksheet has a column J for Location (e.g. Asda or Tesco).

Tap Alt+F11 and when the VBE opens, look at the upper left for the Project Explorer. If it isn't there, tap Ctrl+R or use View ►Project Explorer.

Right-click ThisWorkbook and choose View Code. Paste the following into the new pane on the right titled something like Book1 - ThisWorkbook (Code),

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
    Select Case LCase(Sh.Name)
        Case "asda", "tesco"
            With Sh
                If Not Intersect(Target, .Columns("I")) Is Nothing Then
                    On Error GoTo mb_Safe_Exit
                    Application.EnableEvents = False
                    Dim t As Range, vITM As Variant
                    For Each t In Intersect(Target, .Columns("I"))
                        If CBool(t.Value) Then
                            vITM = .Cells(t.Row, 1).Resize(1, 10).Value2
                            Debug.Print LBound(vITM, 1) & ":" & UBound(vITM, 1)
                            Debug.Print LBound(vITM, 2) & ":" & UBound(vITM, 2)
                            vITM(1, 10) = .Name
                            Sheets("ShoppingList").Cells(Rows.Count, 1).End(xlUp) _
                                .Offset(1, 0).Resize(1, 10) = vITM
                        End If
                    Next t
                End If
            End With
        Case Else
            'do nothing
    End Select
mb_Safe_Exit:
    Application.EnableEvents = True

End Sub

Tap Alt+Q to return to your workbook. If you change the quantity in column I of either the Asda or Tesco worksheets to any non-zero number, that line will be transferred to ShoppingList along with the name of the worksheet that it came from.

john mcghie say:

The bug is that Word 2016 no longer "saves" a file where it "opened" it from.  This is a design change from Apple.

So it may not have saved it where you thought it did.  Frequently, the subscription versions of Office will attempt to force you to save to the cloud.  If you don't catch this, the changed version will not be where you thought it was.  If the cloud drive is no longer open, it may not be accessible at all.

Try this trick:

  1. Open (or create) a new document
  2. Use File>Save As...
  3. Take a careful note of where Word is proposing to save the document.
  4. Have a look there for your missing file.

This is really quite irritating, but it's the way it currently works.
