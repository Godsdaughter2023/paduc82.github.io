    rst(1).MoveNext
    Debug.Print "=============================="
Loop

End Function

Now the .RecordCount  in rst(1) shows the correct record count and loops as many times. However, the .RecordCount in rst(2) always show that there is only 1 record - which is wrong. I tried to execute a SQL query to create a new table using the same query I used to create the recordset and it showed the correct number of records. 

What am I missing?
john w. vinson say:

When you first open a recordset Access just verifies that it has at least one record. To avoid having to wait an unknown time (recordsets could be huge!) it resumes the rest of the code immediately, rather than waiting to traverse the entire recordset.

 

To get a count you must do a MoveLast after opening and before checking the RecordCount.
