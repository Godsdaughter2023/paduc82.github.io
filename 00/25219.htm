Alternative Simple Dual-Boot Setup for Testing Insider Preview Builds - Microsoft Community

don cuthbert say:

Alternative Simple Dual-Boot Setup for Testing Insider Preview Builds

The advantage of using dual-boot for Windows Insider builds is that if a build causes your PC to quit functioning, you can always boot to the current public build so that you can keep using the PC.

_____________________________________________________

This method allows you to create a partition for Insider Builds, copy Windows 10 to it and easily change the BCD to boot to the new install from within your current running Windows 10 public build.  The usual dual-boot setup requires booting from Windows 10 install media.

First decide how much disk space you want to allocate to Insider Preview Builds.  This document suggests a minimum of 20GB to upgrade to Windows 10 64-bit. https://www.microsoft.com/en-us/windows/windows-10-specifications?OCID=win10_null_vanity_win10specs 

Use Disk Management or Diskpart to shrink drive C: by the amount that you have chosen for Insider Builds.  Create a new Primary partition/Simple volume.  Quick-format the new volume with NTFS and give it a drive letter.

Mount your current public build ISO for Windows 10.  Open a Command Prompt and navigate to the \Sources folder on the mounted ISO.  Now use DISM to apply the files from \Sources\Install.wim to the drive letter of your new volume.

Below uses drive F:\ for the new volume that was created:

DISM /Apply-Image /ImageFile:Install.wim /Index:1 /ApplyDir:F:\ /Verify

After DISM finishes copying Windows to the new volume, run this command to create the dual-boot entry in your BCD:

bcdboot F:\Windows

If you type bcdedit, you will see that F:\Windows is now {default} and C:\Windows is {current}.  It is important to leave the new install as {default} until Windows finishes installing, as it will restart before is complete.  To change the description of the new install, you can enter:

bcdedit /set {default} description "Insider Preview"

Restart your computer and Windows will setup a second install of Windows 10 on the new partition.  You can then sign-in to Windows using your Insider Microsoft Account and join that install to the Fast Ring.

When a new Insider Build is released, set the Insider install to be the default boot using msconfig or Advanced System settings, so that during upgrade of that partition, the reboots work properly.

Here's an approach that uses a variant of a function that is not often useful but is perfect for this scenario.  The code support is minimal and maybe you can even do without it.

The key is to use the CELL function as part of a conditional formatting formula.

My test: Highlight the data in C1:C20 that contained the same value as a selected cell in the range G1:G20.

So, select C1:C20 and create a conditional formatting formula

=IF(CELL("col")=COLUMN(G:G),C1=CELL("contents"),FALSE)

For the c.f. format I picked a background fill color.

Now, click on any cell in G1:G20 and press F9, which calculates the worksheet.  The cells in C1:C20 with the same value as the selected cell in G will have the background fill visible.

 

To skip having to press the F9 key, add the code below in the worksheet's code module:

 

Option Explicit

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Target.Calculate
    End Sub

 

 

jeeped say:
I wasn't originally thinking of a multiple selection in G57:G96, but that should be accounted for. Here is a multiple selection modification to my originally posted snippet.

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Range("B57:G96").Interior.Pattern = xlNone
    If Not Intersect(Target, Range("G57:G96")) Is Nothing Then
        For Each gc In Target
            For Each bc In Range("B57:B96")
                If bc.Value = gc.Value Then bc.Interior.Color = 65535
            Next bc
        Next gc
    End If
End Sub
