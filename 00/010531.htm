ossiemac say:

Not able to test your code fully because have not got the UDF  findcolumn.

Your comment " the highlighted line is giving me an error" What error message do you get? The highlighted line of code is OK.

I see one problem with your code with the following line.

If Target.Count > 1 Then Exit Sub

That line should be before Application.EnableEvents = False otherwise events are going to remain turned off and the code will not run again. maybe this is the real problem and the code is not running.

An alternative method is instead of Exit Sub is to send it to a label just before the line to turn events back on.

Private Sub Worksheet_Change(ByVal Target As Range)

Dim aCol As String

Dim gaWs As Worksheet

'-------------------------------------------------------------------

Application.EnableEvents = False

Set gaWs = Worksheets("Site list")

    If Target.Count > 1 Then GoTo ReEnableEvents

    If Not Intersect(Target, Range("C5:L5")) Is Nothing Then

        'aCol = findcolumn(Target.Value, gaWs, 1, True)

    ElseIf Not Intersect(Target, Range("C4:L4")) = Empty Then

        Target.Offset(1, 0) = ""

        Range(Target.Offset(3, 0), Target.Offset(18, 0)).ClearContents
        'Range(Target.Offset(3, 0), Target.Offset(18, 0)).Select
    End If

    Me.Select

ReEnableEvents:

Application.EnableEvents = True

End Sub

An even better example is to ensure events are turned on again in the event of an error like the following and include a message that an error has occurred.

Private Sub Worksheet_Change(ByVal Target As Range)

Dim aCol As String

Dim gaWs As Worksheet

'-------------------------------------------------------------------
On Error GoTo ReEnableEvents
Application.EnableEvents = False

Set gaWs = Worksheets("Site list")

    If Target.Count > 1 Then GoTo ReEnableEvents

    If Not Intersect(Target, Range("C5:L5")) Is Nothing Then

        'aCol = findcolumn(Target.Value, gaWs, 1, True)

    ElseIf Not Intersect(Target, Range("C4:L4")) = Empty Then

        Target.Offset(1, 0) = ""

        Range(Target.Offset(3, 0), Target.Offset(18, 0)).ClearContents
        'Range(Target.Offset(3, 0), Target.Offset(18, 0)).Select
    End If

    Me.Select

ReEnableEvents:
    If Err.Number <> 0 Then
        MsgBox "An error occurred in sub Worksheet_Change, module " & Me.Name
    End If

Application.EnableEvents = True

End Sub

what causes word document bloat? - Microsoft Community

jamesip say:

what causes word document bloat?

What causes Word docx to grow? I have a 220K word file that measures 1.8 MB. It may be in the range of normal, but just want to know if there's any way to reduce the fat. I know that .docx is slimmer than .doc. Perhaps there's nothing else to do. tx
bob jones aka: cybertaz say:
For the most part, file size is determined by what you put in the file additional to the text. Images, Shapes, Diagrams, Tables & poor construction (including reliance on direct formatting) are some of the major contributors. Extensive & ongoing use of Track Changes will also add to the file size as will corruption often caused by those issues.

Also, saving files back & forth between .doc & .docx will increase the size of the file due to it having to retain multiple personalities.

I have an 86,000 word (207 page) .docx document which contains numerous screen shot illustrations, an extensive TOC, quite a few Fields, many complex Tables, varying Headers/Footers, Footnotes, Hyperlinks... It weighs in at a mere 1.22 MB, but it was created by one of the most capable Word users I've had the pleasure of knowing.
