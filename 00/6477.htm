petrib say:

Separate dictionary apps are possible, but you can't separately add new T9 languages.

 

The T9 language is always one of the languages supported by whatever firmware version (variant) is installed on the phone.

 

At least that's how things used to be with Nokia phones.

 

In general, individual languages cannot be added to the phone firmware, either. Nokia decides which language combinations it supports and in which regions/countries, and creates firmware variants for those combinations. And then those are only use for devices manufactured for the region/country in question, but not available elsewhere, and without any supported means for the phone user to switch to a different firmware variant on their own.

 

That's different from, e.g., the Apple approach, where every iPhone has built-in support for all languages that the iPhone operating system supports. It requires, of course, more memory as you have to store the languages, fonts and dictionaries for all languages, instead of just a few (and all the translations/localizations needs to be ready before you release the operating system, too, more or less). For a phone user, the Apple approach is more convenient, though.

Hi Mario,

A PTR record resolves the IP address to a domain/hostname. Since the error messages says that the recipient side checks the PTR record of your domain and it failed, we suggest you contact the recipient side to add your IP address(es) into his white list/allowed list and check if the issue persists. If the recipient isn’t an admin and he is not familiar with the white list/allowed list, he can ask for his admin for help.
