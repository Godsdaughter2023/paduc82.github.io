I have had the same problem it seems it is pretty global.

Found this solution and it works fine for me, but I did need to open Word twice at stage.  The first time I opened it the error message had gone away but it was still unlicensed.

1. Run Command Prompt as Admin

2. Type:

cd %programfiles%\Microsoft Office 15\Clientx64

3. Press Enter

4. Type:

OfficeC2RClient.exe /Update USER updatetoversion=15.0.4605.1003

5. Press Enter

6. Wait for Updates to complete – if there is an error reboot machine and try steps 1-5 again

7. Once rollback is complete, open Word

8. Enter in O365 Org ID to activate.

9. You will receive the 0x80041015 error message again.

10. Click Repair twice on the error window, this will cause Word to close.

11. Open Word again, Office should now be activated.

rick song msft say:

Hi Elisabeth ,

 

Thanks for your reply . According to my experience , the spam filter may still spam some emails from the domain as junk mail according to the high Spam Confidence Level(SCL) . Those emails may include some sensitive words or phrases , or with some attachments . If you can trust all emails from that domain , creating a new transport rule can preventing them from moving to junk mails .

 

As an admin , you can create a transport rule to set the emails from that domain to be trustworthy . You can refer to this link about setting the SCL in messages : https://technet.microsoft.com/en-us/library/dn798345(v=exchg.150).aspx

 

If you have any questions while setting SCL , please feel free to contact us .
