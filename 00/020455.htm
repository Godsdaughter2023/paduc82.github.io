Word unexpectedly applies a style with deleting carriage return - Microsoft Community

finleymon say:

Word unexpectedly applies a style with deleting carriage return

I have looked everywhere, and I can't seem to find the answer to this one... it's either a bug or a very, very non-intuitive "feature" in MSW. 

I'm using Word 2008 on a Macbook Pro. I have a large document with periodic page breaks, then a paragraph mark, and then a heading style applied. I want to delete the paragraph marks, but when I do so (so that the heading text is at the top of the page subsequent to the page break), the style unexpectedly changes from a heading style to "normal", and the text no longer is formatted as a heading. There don't seem to be ANY automatic formatting controls for this sort of thing, and I can' find a description anywhere of someone with the same issue. 

Thanks in advance for any help!
rich michaels - mvp (greatcirclelearning.com) say:
What you are observing is the designed behavior for how all paragraph styles work. If you delete the paragraph mark from the previous paragraph, the style formatting of the previous paragraph is carried over to the next paragraph. 

A paragraph style begins at first character of the document or first character following a previous paragraph and ends with the last character of the paragraph and includes the paragraph mark itself. Even an empty paragraph carries a style format; an empty paragraph mark represents not only the end of paragraph mark but also the first and last character of the empty paragraph.

The design logic is, if the paragraph marked is deleted then the style formatting of the previous character(s) is used to format all subsequent characters until a next paragraph mark is found; the next paragraph mark's style settings are then updated.

    Just another vote for moving to a relational database in such scenarios.  Excel was designed as a single-user application and even though Microsoft has tried to hammer it into a multi-user capable tool, they really haven't succeeded.  As others have said, it's not a matter of IF the file will become corrupt, but more of a WHEN will the file become corrupt situation.

    • The device starts using too much battery
