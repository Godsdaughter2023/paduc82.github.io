tomgwynn say:

We also encountered this problem. The cause appears to be that when Office tries to send the "sharing" emails to share the calendar with recipients it doesn't check to see whether that username is already in the calendar's permission list. So (for example) if you make a secondary calendar and then try to share it with Jane Doe as "editor", if Jane Doe is already in the calendar permission list with permission level "None" Office will not replace the permissions. Instead it gives the error "Error While Preparing Send Sharing Message."

To fix the issue, bring up the Calendar and click "Calendar Permissions" on the Home menu of the ribbon. If there are any usernames in the permissions that are the same as the username you want to add, then remove them. Once they've been removed you can re-add them manually with the correct permission level, and re-send the sharing email.

bernie deitrick say:

You can use a User-Defined-Function to create a number that will allow sorting based on your heirarchy strings.

Copy the code below into a standard codemodule in your workbook, save the file as a macro-enabled .xlsm, and use the code like this:

=SortVal(A2,".",5)

Then copy down, and sort your entire table based on the column of formulas.

Function SortVal(r As Range, strd As String, i As Integer) As Double
    Dim v As Variant
    Dim j As Integer
    v = Split(r.Value, strd)
    For j = 0 To UBound(v)
        SortVal = SortVal + v(j) * (10 ^ ((i - j - 1) * 2))
    Next j
End Function
