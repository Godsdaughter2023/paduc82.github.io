toddallcock say:

I'm a little confused. Do you mean files synced from the PDA open as read-only when you open them from the PC's sync folder, or that clicking on files on the PDA itself in the WMDC file browser open as read-only?

 

If the latter, that's normal operation. It's also normal that you can't save a file directly to the connected PDA from the PC. The correct procedure is either to save the file in the PC sync folder and let it sync to the PDA, or to save it on the PC and copy it (drag and drop) to the PDA.  WMDC is a sync conduit and device manager; it was not designed to convert the PDA into an external disk drive.

Try pressing Ctrl-F5 instead of doing all that.   However, if you are really getting the cache cleared, then clicking Back after that or clicking Refresh after that should have the same effect.   E.g. there should be nothing in the TIF to allow the browser to avoid doing a normal request of the server.   An implication of that not happening would be that there is an intermediate cache active that you (and your browser) would have no control over.   Then it would just be coincidental that clicking Refresh worked.

BTW with IE9 you have a trace tool (in Developer Tools, Network tab) which could help you see the differences in each step.   Even better would be to use Fiddler2 and capture the whole set.

If you need more help and you have a publicly accessible example, please post the URLs and the sequence that you used them in.

 

HTH
