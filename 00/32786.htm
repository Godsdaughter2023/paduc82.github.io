This event is generated when a logon request fails. It is generated on the computer where access was attempted.


The Subject fields indicate the account on the local system which requested the logon. This is most commonly a service such as the Server service, or a local process such as Winlogon.exe or Services.exe.


The Logon Type field indicates the kind of logon that was requested. The most common types are 2 (interactive) and 3 (network).


The Process Information fields indicate which account and process on the system requested the logon.


The Network Information fields indicate where a remote logon request originated. Workstation name is not always available and may be left blank in some cases.


The authentication information fields provide detailed information about this specific logon request.

\- Transited services indicate which intermediate services have participated in this logon request.

\- Package name indicates which sub-protocol was used among the NTLM protocols.

\- Key length indicates the length of the generated session key. This will be 0 if no session key was requested

How to find SSID (Network Name) - Microsoft Community

xbr dionysus say:

How to find SSID (Network Name)

Login Information

  • Default Gateway - gateway.2wire.net / 192.168.1.254
  • Username - N/A
  • Password – “Device Access Code” on the side of the device

 

How to find SSID (Network Name)

  1. Open Web Browser.
  2. In the address bar of the browser enter in "gateway.2wire.net" (if this does not work, try one of the following IP address: 192.168.1.254).
  3. If asked for a password it ill be located on the side of the device labeled “Device Access Code”
  4. From the menu at the top select Settings.
  5. From the sub-menu select LAN.
  6. From the sub-sub-menu select Wireless.
  7. The SSID is located here in the box labeled Network Name.

  • Restart a few times and try again.
  • Disable General USB Devices (example - Smart Card Reader).
