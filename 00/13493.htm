karl timmermans (outlook mvp 2012-2018) say:

Hello RBStein

Since you mention that calendar and contacts are not an issue then the way to sync you email across folders is to configure the email account as an IMAP account on all devices. There is no sync'ing capability using a POP account which is a one-way one-time download of the <Inbox> to each device connected to the account provided that the <Leave Messages on server> option has been checked. This is not Outlook specific - it applies to any <email client> (of which Outlook is only one) used whether on a PC or mobile device.

IMAP is a bi-directional sync of all email folders which means that anything added/changed/deleted on one side (i.e. server) is reflected on the other side (computer/device) and vice-versa.

Things to take note of if you opt to go the IMAP route

#1 - Make sure that you have sufficient capacity on the server to store the emails that you want to have accessible across devices
#2 - You will need to remove the POP3 account from Outlook before adding it again as an IMAP account

jay freedman say:

It's been a very long time since I looked at a sample of VBA how-to books and sites, so I'm reluctant to make recommendations. Here are just a few that I'm aware of.

http://www.word.mvps.org/FAQs/MacrosVBA/index.htm has articles ranging from gentle introductions to workarounds for obscure bugs. Unfortunately a majority of them haven't been updated recently, but the basics haven't changed.

http://gregmaxey.mvps.org/word_tips.html and http://www.gmayor.com/word_pages.htm are excellent sources of more recent VBA examples, and are worth studying to see how they approach specific problems. You'll probably also find some finished projects that will save you from reinventing wheels.

There's a good but slightly outdated list of resources at http://shaunakelly.com/word/word-development/resources-for-word-vba.html. Shauna passed away in 2011, and she's sorely missed by all who knew her.

Besides this forum (and, better, the Programming topic at http://answers.microsoft.com/en-us/office/forum/customize), good places to ask questions include the VBA forums at http://www.eileenslounge.com, http://www.vbaexpress.com, and http://windowssecrets.com/forums.

----

There are very few differences in what I'd call the infrastructure of VBA from Office 2007 through 2013 -- the If, Select, and other decision statements, declaring variables and constants, managing code through modules and procedures, and so forth. What changes with each version, sometimes drastically, is the object model -- how VBA represents the parts of a document or other Office structure, what methods and properties those objects offer, and how they interact with each other.

In most cases, a macro written for Word 2007 will run perfectly in the later versions, but the reverse is not necessarily true.  If you write a macro in Word 2013 that uses an object that doesn't exist in an earlier version, that macro either won't run or will throw a runtime error if you try to use it in the earlier version.

Even the vast majority of macros written for Word 2003 will run well in Word 2013, but not those that depend on manipulating menus and toolbars (although even some of that will succeed).
