scottgem say:

Use of lookup fields on the table level is not recommended ( see Evils of Lookup fields). Lookups should be done on the form level using list controls.

As you note, redundant data is not good design. Your client should NOT be dictating table design. That's your job. he shouldn't even be seeing the table design. He can dictate interface design and your job becomes to fit his interface desires into a properly normalized table design.

See my blog on Displaying Data from Related Tables to see how to display the information he wants to see on his forms.

Task names may be long, but there should be an autonumber ID field for the tasks and that should be used as the foreign key so you don't repeat the long names in other tables.

It took a few moments to figure out how to setup an alias but the process is fairly simple.  First login to the problem email account via a web browser.  Once logged in click on the “avatar” in the upper right corner – you will get a drop down menu – and select “View account” which will open a new browser tab for the associated Microsoft account.  From the menu bar select “Your info”.  On the next screen select “Manage your sign-in email or phone number”.  At the next screen (at least for me) I get a Microsoft Account confirmation screen from which you can send a code to verify your account to an email address or phone number.  Enter the code on the next screen – you will get another screen asking if you want to setup a code generator app on your phone – you can select “Later” for now.  The next screen will be for managing your account aliases.  Select “Add email”.  At the next screen you have two choices – “Create a new email address and add it as an alias” or “Add an existing email address as a Microsoft account alias”.  I selected the first option which guides you in creating an email address with an @outlook.com domain.  This address will only be used to login to your existing outlook.com account using your existing password.

This is how I add an @outlook com alias. 

First login to the problem email account via a web browser.

Click the Cog icon.

Click Options.

Click Create an Outlook.com alias.

...Microsoft Account confirmation screen from which you can send a code to verify your account to an email address or phone number.

Enter the received code.

Using this route, you will only be able to create an @outlook.com alias address.

Note: after being switched to Outlook Mail (Preview), use the alias address to setup an IMAP account in email programs.  The automatic setup will use the outlook.com servers which no longer work.

Edited note: after being switched to Outlook Mail (Preview), use the alias address instead of the custom domain address to setup the account in a device or app.  Also use the alias address as the login address.

Note 2: it may take 48-72 hours for the alias address to become fully active.

teylyn say:

Hello,

you can use a pivot table to aggregate the data by date. Click a cell in the data, then go Insert > Pivot Table. In the pivot panel on the right, drag the date field name into the row area and drag the values field name into the value area. Now you see a pivot table with each date only one. You may need to adjust the values formula to sum instead of count (click the drop-down arrow of the value field name in the values panel and select "Value Field Settings")

You can now use this table to create an XY Scatter plot.

Or you can use a Pivot Chart to chart the data. Pivot charts don't support XY Scatter plots, but a line chart with markers and no line, using a date axis, will do the same thing.

As an alternative to a pivot table, you can create a helper table that has all the individual dates in the first column. In the second column you can use a Sumif() function.

With your original data in columns A (date) and B (Value) and a helper table in columns D (dates) and E (Sumif() function for values), put the individual, unique dates in column D, and in E2 put this formula and copy down

=SUMIF(A:A,D2,B:B)

Then build an XY Scatter chart from that helper table.

The Pivot table is ultimately less work, since it can be configured to update automatically if new data is added to the source table. With the helper table, new dates need to be added manually and the formula copied down.
