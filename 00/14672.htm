Try this

MS-Publisher: Stepchild of the Office family? Or are we going - Microsoft Community

shutterfly1974 say:

MS-Publisher: Stepchild of the Office family? Or are we going to get a better & more usable build sometime soon?

Is there a MS Team working on better new release of MS-Publisher? Can we expect a more stable and debugged release of MS Publisher sometime soon? Having intensely worked hands-on with MS-Publisher 2007 and 2013 releases for some time with pretty large, complex and colourful design jobs for the last several months, I know a thing or two. I have made a list of must-haves for the next release of MS-Publisher, which will help the MS-Loyal design fraternity who hate to go through the rather unfriendly learning curve of Adobe packages (not to mention their cost). 

First, the good things:

  1. MSPUB is a very easy-to-learn software for anybody having a working knowledge of Word & Powerpoint.
  2. UI is intuitive - almost similar (but yet you can feel the stepchild effect nevertheless) to other Office components like Word, Powerpoint etc. 
  3. File --> Export --> Pack and Go --> Save for Commercial Printer is a brilliant tool to take your file and go to a large-scale printing house for CMYK offset printing. This feature can indeed be compared to a Bose home entertainment system, that offers minimum buttons or switches to the user, but does a great job for most practical purposes with preset controls behind the curtains.

Now, lets see some areas where we really, badly need some improvements to make this program more usable for at least reasonably serious work.

If I were to manage the dev team of MS-Publisher, the things I would really consider as a must-revisit for the immediate future would be as follows:

Need to stabilise and debug the following functionalities:

    • Styles and Page Parts: Need to make this as user-friendly as MS-Word. Right now, there are several bugs, e.g., colour & formatting changes when you copy-paste a specific styled item from one Publication to another Publication, while both PUB files are simultaneously opened in your PC.
    • Custom-coloured Bullet character: In a bulleted list (or for that matter a numbered list too) I need to be able to make the bullet orange (for example) while the text in that list-item remains black (or any mix of other colours). This versatility is a must-have for a Page Layout design package worth its salt.
    • Bug in Bullets and Numbering: Numbered lists in MS Publisher get blotched up at the drop of a hat, and you get horrified by a list with all the items suddenly numbered as “1”. We need the “Continue Numbering”, “Restart Numbering” and “Set Numbering Value” features at right-mouse-click. If you can have the same things working fine in MS-Word, why can’t you make the same features available in MSPUB?
    • Problem with custom-size documents and Pack and Go Wiz: e.g. I am working on a book publishing project with page dimension 8.50” X 11.50”. When I create the final PDF using Pack and Go Wiz, it silently creates a PDF with the last used paper-size while printing. If I just printed a page of this publication to my HP laserjet printer (default setting A4), then the next time I use the Pack-and-Go wiz – it will create the final production PDF in A4 size without asking me anything, INSPITE of the fact that all the pages of my publication is setup to 8.50” X 11.50”.
    • File size: The PUB file gets abnormally large as you keep working with it and say, cross 100 pages while creating an illustrated book.
    • Memory Management: Somehow, by the time I have inserted 50+ images/illustrations in my publication and crossed 100 odd pages, the file size crosses 150MB, and the behaviour of every aspect of the program starts getting s-l-o-w. Even with an Intel i5 3210M 2.5GHz processor on a 64bit motherboard and 8GB DDR3 RAM, every now and then, a mouse-click is followed by 5-7 seconds of very unnerving “not responding” periods where I can be seen praying really hard with tightly clasped fingers. This gets worse with each passing day and each growing page. A file with similar number of pages and illustrations in Adobe Indesign runs on the same PC much faster, smoother and never stalls.
    • Clipboard Memory management: This is a pain. Every now and then the clipboard seems to be full and refuses to take in any more stuff. You select something and press Ctrl+C and go to the destination and press Ctrl+V – and lo! You have just pasted something you had copied two hours back. This point onwards, you realise to your horror that Ctrl+C does not do anything more than the clicking sound of the keyboard. Now if you attempt to exit MSPublisher – it will ask you if you want to discard or save the contents of the clipboard.
    • Output PDF in CMYK colour model: MS-Publisher creates RGB files by default, and offers absolutely no ability for the user to control or change that to CMYK colour mode. All commercial printers expect PDF files where black text is in proper CMYK black (CMYK: 0,0,0,100) and definitions of other colours are also in CMYK colours. In MSPub 2013, You can select a blob of text and force it to have CMYK Colour by doing a complex sequence: Go to Toxt Box Tools --> Text Fill (Drop Down) --> More Colours --> Custom --> Colour Model --> Choose “CMYK”. Instead, we need to have a setting “save this Publication in CMYK realm” so that all colour controls and palettes in the menu/toolbar default to CMYK – e.g. text, shadows, borders & lines, etc everything.
    • For the advanced user, we need more control on the Pack and Go Wizard. File --> Export --> Pack and Go --> Save for Commercial Printer is a brilliant tool, as mentioned already. But some more visibility and control on what is happening behind the iron curtain of the wizard would be a lot more helpful. For example, the following switches (as checkboxes) would be really sweet to have in the Pack-and-Go wiz:
  • Flatten Layers of resulting PDF to optimize file size
  • Convert all RGB images to CMYK while packaging
  • Convert all RGB black text to pure CMYK black
  • Analyse errors and warnings & create an error log (and later save as txt)
  • While creating PDF, a) maintain the original Paper dimensions of publication including bleed space or b)Choose another custom size …
