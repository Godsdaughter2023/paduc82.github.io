zorvek say:

If you are working in Excel 2010 or earlier (you placed this question in the Office 2016 category) then here are some notes about doing what you want in 2010 and earlier versions.

Every version of Excel up to and including Excel 2010 operates using a multiple document interface or MDI. This means that a single instance of Excel can have multiple workbooks open inside of a single application window. Workbooks can also be opened in separate instances of Excel. The advantage of this is each instance has a completely separate application window (menus, toolbars, and status bar.) The disadvantage is that workbooks in one instance cannot be linked to workbooks in another instance nor can cell range formatting be copied from one instance to another.

The default behavior when opening a new workbook from outside Excel such as from File Explorer is that Excel opens the workbook in the same instance. Excel and/or Windows can be configured to open each workbook in a new instance of Excel.

In Excel 2003 and earlier this is accomplished by changing how Windows handles Excel files. In Windows XP and earlier, open Windows File Explorer and chose the menu command Tools->Folder Options. Navigate to the tab File Types. Find the XLS entry and click Advanced. Select Open and click Edit. Uncheck "Use DDE". Add  "%1", with the quotes, to the end of the string in the field "Application used to perform action" - making sure there is a space before the first quote - and click OK, click OK again, and click Close. In Windows Vista and later versions or in Windows NT, use the free third party utility File Types Manager (http://www.nirsoft.net/utils/file_types_manager.html) to make the above changes.

In Excel 2007 and 2010, open the Excel Options window (in Excel 2007 click the Office Button and click "Excel Options", in Excel 2010 navigate to the File tab and select "Options"), select "Advanced" in the left list, scroll down to the "Display" section, and check on "Show all windows in the Taskbar".

To restore the default behavior in Excel 2003 and earlier (open all workbooks in the same instance of Excel), quit all instances of Excel, click the Start menu and select Run. Enter "Excel.exe /UnRegServer" and click OK, wait for the process to finish, then enter "Excel.exe /RegServer" and click OK. The full path to Excel may have to be entered for the command to work.

To get the command to work, the full path to the Excel application may be required. If so, the path usually looks like:

   C:\Program Files\Microsoft Office\OFFICE\Excel.exe

or

   C:\Program Files\Microsoft Office\OFFICExx\Excel.exe

where xx is the Excel version: 10 for 2002 and 11 for 2003. So the commands for Excel 2003 are:

   "C:\Program Files\Microsoft Office\OFFICE11\Excel.exe" /UnRegServer
   "C:\Program Files\Microsoft Office\OFFICE11\Excel.exe" /RegServer

To restore the default behavior in Excel 2007 and later, open the Excel Options window (in Excel 2007 click the Office Button and click "Excel Options", in Excel 2010 navigate to the File tab and select "Options"), select "Advanced" in the left list, scroll down to the "Display" section, and uncheck on "Show all windows in the Taskbar".

If that doesn't work, quit all instances of Excel and open the "Add or Remove Programs" control panel, select the Excel installation, click "Change", and select "Repair".

If that doesn't work, delete the values .xlsx in HKEY_CLASSES_ROOT in the Windows registry. Right-click on any .xlsx file and select Open With->Choose Program. Navigate to the desired Excel application (Excel.exe) and select it. The default location for Excel is C:\Program Files\Microsoft Office\Officexx\excel.exe where xx is the Excel version: 12 for 2007 and 14 for 2010.

zorvek say:

Now your question makes more sense.

I don't recommend changing a user's Windows/Excel configuration to do this because, when copying or cutting between workbooks, if the source and target are in different instances, then the Windows clipboard is used to transfer the information and Excel doesn't provide nearly as many options when pasting (paste formulas, paste formatting, etc.) So setting up a user's box to do what you want will also be taking away expected functionality.

The alternative is to launch the specific workbook in it's own instance. To do this you will need to create a control center that displays a ribbon from which the timesheet is launched, or create a launcher application workbook who's sole purpose is to launch the timesheet in it's own instance--the launcher application appears to the user as the timesheet itself but all it does is launch the real timesheet workbook. All that said, you would still have to deal with the user opening additional workbooks and how to keep them from opening in the same instance that is running the timesheet workbook.

But let's take a step back and look at the problem with different eyes. It turns out my team is building this exact same functionality for our consulting company using Excel and a SQL Server back end with a web service API. We abandoned the notion of a persistent worksheet from the beginning--it made more sense to us to download and upload data on demand and not keep any data locally. In the event that the user was not connected to the Internet or could not accesses the web service, we keep a cache of data in the user's local registry. This model makes it more conducive for multiple people to be working with the same data over time and at the same time.

The client logic is presented as an Add-In that is loaded in every instance of Excel and is accessible via a ribbon tab. We show user forms for most activity (entering hours, maintaining client information, etc.) and generate worksheets for reports.

We only have rough prototypes at the moment as it's a work in progress. Perhaps there is opportunity here to collaborate.

You may be able to transfer your Office license to the new computer. But it entirely depends on the EXACT type of Office license you bought.

Sounds like you bought the Office 2010 Home and Student bundle online. 

If it cost around US$150, it is likely a Retail license that allows install on up to 3 computers at the same time and transfer of those installations when required

If it cost around US$100-120 then it is likely a Product Key Card license. That allows only install on one computer and does not allow transfer to new computers.

In general line art and text need a higher resolution than photos do, because the crisp edges make jaggies more obvious. In addition, jpeg lossy compression introduces square blocks around hard edges, which makes it look worse. Try a higher resolution to a PNG output instead. Your file size will be larger, but with a mix of type and photo, you'll have to live with it to get the quality.
