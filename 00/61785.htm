Back up process makes multiple copies of files.

After having this run all day I checked the backup files on the drive. It makes multible copies of everything. I really don't need 7 copies of every photo and document. Why is it doing this?

Stan

cmdrkeene say:

If you use the "File History" function, it only makes a new copy if that particular file has changed since the last snapshot was taken.  That way you can "go back in time" and get a copy of the file from before it got changed.  It works like the Apple Time Machine backup.

hairyfool say:

On what basis are you seeing multiple copies.

As Shawn says the 1st backup is complete then only file changes are recorded.

However each backup event will list all files available to restore at that particular point which will include all files in the original master backup. So if the there are 10 backup events the original file will be listed 10 times but there is only one copy if it is unchanged.

Use explorer to examine the File History folder. Each file name will be modified with the date and time but only those that have actually altered will have multiple copies.

unclezip say:

This is happening on files and photos that are not modified in any way. there should be no new copy's of my pictures. I can understand a document that is worked on to have new and old copy.

This is not the case here!

cmdrkeene say:

Although you can, I'd recommend against just browsing your File History folder directly.  You'll even get blocked if you try until you modify the security permissions of the folder. 

I would recommend instead, always go to Control Panel and use the File History interface "restore personal files" option as your only method of exploring the files, and here's why:  if you modify the security permissions to allow any user to traverse the backup, then you are at risk of any future malware or ransomware will also have very easy access to destroy your backups right when you need them most.  The default security is set so that only the backup engine can access the files, not other processes or even the user themselves.  This protects the user because even running a virus from an email (which runs using the user's security access) wouldn't be able to hurt the backup.

hairyfool say:

I was merely trying to point out the difference the File History interface has over what is actually stored.

I would agree that no modifications should be made but I get no permissions warnings when I view the folders and I have made no changes to the default configuration.

cmdrkeene say:

You should get this the first time you try to open your file history folder.  It's not because you aren't admin, it's just that only the trusted process is allowed to use it by default.  Clicking continue will permanently change the permissions.  That itself won't break it, but it's risky because of the security hole it opens up.
