zipone say:

Have a continuous form field set by a field from a different table

Hi,

Is this possible?

Thanks

dirk goldgar say:

I want to have a table with a single number in it that the user can set to any value they want and to be able to change this number at any time.  I then want a continuous form based on a different table that will contain the number for each record entered.  When the form is closed I want to increment the single number in the first table by one.  The next time the form is opened I want to restrict the form to single number in the first table so that the previous records will not appear in the form.

 

Is this possible?


So, then, the "single number" you refer to would be something like a "session number", that would be used to stamp each record entered in an edit session, where an edit session is defined as the period between the opening of the edit form and the closing of the edit form.  This session number would be automatically incremented for each edit session, and the edit form would only show the records for the current edit session. Is that what you had in mind?

You could do this with a table called (for example) "tblEditSession", with a Long Integer field named "SessionNo".  This table would only contain one record -- you can enforce this at the table level, if you need to, by creating a primary key field with a validation rule that requires it to have a specific value such as zero.

Now, to use this SessionNo field, suppose your editing form is called "frmEdit" and the table to be edited is called "tblYourTable" -- names to be changed to match your reality, of course.  In addition to its other fields, let tblYourTable contain a Long Integer field named "SessionNo", similar to the matching field in tblEditSession.  This field will hold the value of SessionNo at the time the record is created.

There are a couple of different ways you can get this field stamped with the current value of tblEditSession.SessionNo.  Mybe the simplest one is to change frmEdit so that it now has a main form/subform arrangements.  The main form would have tblEditSession as its recordsource.  It would have a subform in continuous form view with tblYourTable as its recordsource.  Set the Link Master Fields and Link Child Fields of the subform to SessionNo -- on the main form, it's the SessionNo field in tblEditSession, while in the subform it's the SessionNo field in tblYourTable.  Now the subform will show only the records entered in the current session, and all records entered via the subform will be stamped with the current SessionID.

There remains the business of incrementing the SessionNo in tblEditSessions.  You can use the Close event of frmEdit to do that. In that event, execute an update query to increment the SessionNo field.  For example:

 

'------ start of example code ------
Private Sub Form_Close()

    With CurrentDb

        .Execute _
            "UPDATE tblEditSession SET SessionNo = SessionNo + 1", _
            dbFailOnError

        If .RecordsAffected = 0 Then
            .Execute _
                "INSERT INTO tblEditSession (SessionNo) VALUES (1)", _
                dbFailOnError
        End If

    End With   

End Sub
'------ end of example code ------

This code attempts to take into account the possibility that there might not be a record in tblEditSession, and adds one if there is no record.  The specifics might need to be adjusted; for example, to specify a primary key value if there's no default value for it.
