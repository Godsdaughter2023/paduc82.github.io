Let's say that the data never extend beyond column Y.

Assuming that the data begin in row 2, enter the following formula in Z2:

 

=COUNTA($B2:$Y2)

 

This formula can be filled or copied down as far as needed.

Searching with a query. - Microsoft Community

sigyo say:

Searching with a query.

I am using Microsoft Access and I'm making a form that can search a table based on multiple criteria. I currently have 4 searchable criteria, therefore I have 4 different text boxes in my form. The form runs a query and the query then displays the result from an already existing table, at least it should. The parameter that I use for each criteria looks something like this: Like "*" & [Forms]![FormName]![TextBoxName] & "*". What I noticed is that this syntax works fine if the initial table is completely filled out, mine has some blank records but not a single row where all 4 records are blank. When I leave the form blank and run the query, the query ends up blank. Without having to put "N/A" in place of each record (which works), how can I change the criteria so that blank fields are displayed by the query anyway. I'm new to Access and I do not know VBA. Thank you.

john w. vinson say:

The LIKE criterion may not do exactly what you want even if it does work: it will search for any record containing the TextBoxName value even if it's embedded in other text (e.g. a criterion of "Al" will not just find people named Al, but also Crystal, Alfred and Challis. Is that what you want?

The problem you're having is that a NULL value in your table is not "LIKE" anything. Any criterion you apply to it will return - not yes, not no, but NULL. You might try a somewhat more complex criterion:

=[Forms]![FormName]![TextBoxName] OR [Forms]![FormName]![TextBoxName] IS NULL

This will return exact matches, but will ignore any textbox which is left blank.

Of course a criterion of XYZ in the field will not (and, IMO, should not!) match a table field value of NULL. That's not what you intend, is it?
