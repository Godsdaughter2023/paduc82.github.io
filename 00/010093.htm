doug robbins - mvp office apps & services (word) say:

Use the following code:

Dim rng as Range

Dim i as Long, j as Long

With Selection
    If .Information(wdWithInTable) = True Then
        i = .Information(wdEndOfRangeRowNumber)
        j = .Information(wdEndOfRangeColumnNumber)
        Set Rng = .Cells(1).Range
        Rng.End = .Tables(1).Cell(i + 1, j).Range.End
        Rng.Cells.Merge
    End If
End With

yang zheng msft support say:

Hi DeepaV,

It is an expected behavior that the objects that have already synced to Exchange Online still exist. You will need to remove these objects from Exchange Online. Here are the steps:
1. Install the Azure Active Directory Module for Windows PowerShell and connect to Azure AD by using Windows PowerShell. For more info, go to the following Microsoft website:

Manage Azure AD using Windows PowerShell

2. Disable Azure AD connect. To do this, type the following cmdlet, and then press Enter:

Set-MsolDirSyncEnabled –EnableDirSync $false

3. Check that Azure AD connect was fully disabled by using the Windows PowerShell. To do this, run the following cmdlet periodically:

(Get-MSOLCompanyInformation).DirectorySynchronizationEnabled

This cmdlet will return True or False. Continue to run this cmdlet periodically until it returns False, and then go to the next step.

Note: It may take 72 hours for deactivation to be completed. The time depends on the number of objects that are in your cloud service subscription account.

4. Try to remove the objects by using Windows PowerShell or by using the cloud service portal.

Regards,
Yang
