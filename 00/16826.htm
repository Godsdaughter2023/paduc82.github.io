The bottom line answer to your question is, you can't, at least not directly. What you need for your daily productivity is an extra field (e.g. Number1) that is timescaled and Project does not timescale any extra fields.

This cannot be done with a formula in a custom field.

From there it is an easy step to calculate whatever performance metric you need.

andrew simpson uk say:

Hello Husam,

 

Only way I could think of getting this was by a VBA script. Executing the following code on your project will calculate the following:

  • Units  - Using the task.number12 field
  • Actual Units - Using the task.number11 field
  • % Units Complete - Using the task.number10 field

Run this script, drop these columns into your gantt view and you should get the values you need. Values won't calculate automatically though, so you'll need to rerun the script to re-calculate these values when things change.

 

Hope this helps,

Andrew

 

--------------------------

 

 

Option Explicit

 

Sub CalculatePerctUnitsComplete()

 

Dim t As Task
Dim summary_t As Task
Dim a As Assignment
Dim pj As Project
Dim msp As Application
Dim i As Integer

 

Set msp = MSProject.Application
Set pj = ActiveProject

 

msp.CustomFieldSetFormula pjCustomTaskNumber10, Formula:="IIf([Number12]=0,0,[Number11]/[Number12]*100)"
msp.CustomFieldProperties pjCustomTaskNumber10, Attribute:=pjFieldAttributeFormula, summarycalc:=pjCalcFormula
msp.CustomFieldRename pjCustomTaskNumber10, "% Units Complete"

 

msp.CustomFieldProperties pjCustomTaskNumber11, Attribute:=pjFieldAttributeNone, summarycalc:=pjCalcRollupSum
msp.CustomFieldRename pjCustomTaskNumber11, "Actual Units"

 

msp.CustomFieldProperties pjCustomTaskNumber12, Attribute:=pjFieldAttributeNone, summarycalc:=pjCalcRollupSum
msp.CustomFieldRename pjCustomTaskNumber12, "Units"

 

'loop through the tasks collection to calculate the assignment values,
For Each t In pj.Tasks
    If t Is Nothing Then GoTo NextTask
    If t.ExternalTask Then GoTo NextTask
    If t.Summary Then GoTo NextTask

       
    For Each a In t.Assignments
        If a.Resource.Type = pjResourceTypeMaterial _
        And a.Units > 0 Then
            'get actual units
            a.Number11 = a.ActualWork
            t.Number11 = t.Number11 + a.Number11
           
            'get units
            a.Number12 = a.Units
            t.Number12 = t.Number12 + a.Number12
           
            'calculate % units complete for assignment
            a.Number10 = a.Number11 / a.Number12 * 100
           
        End If
    Next a
   
NextTask:
Next t

 

End Sub

afwings say: I can answer your question, but first I want to address an inaccuracy in Daniel's answer. He said "...the format used by Word in its equations [is](called MathType)...". That's incorrect. MathType is a commercial product, available by separate purchase, and is a more powerful equation editor than its little brother "Microsoft Equation Editor 2011 for Mac", which is made by the same company and has been included with Word 2011 and every previous version of Word since v2. If that's the equation editor you're using (let's call it MSEE), you're going through Word's Insert menu to do it. There are actually 2 equation editors in Word 2011 though, and Equation, this is a different equation editor. This editor is called the OMML equation editor (OMML = Office Math Markup Language).

Now what Daniel said was true about your online school system being able to understand the format used by the equation editor. This is true regardless of whether you're using MSEE or OMML. However, in his instructions for Paste as Picture, you'll only need to do the cropping if you're using OMML, because the OMML equation pictures will be page-wide, as he describes. The MSEE equation pictures will be actual size.

Let's go back to MathType. You don't say which "online school" you're using, but MathType has special capability to work with many learning management systems (aka "online schools"), like Blackboard, Moodle, ANGEL, and many others. Go through the list at www.dessci.com/workswith to see if yours is listed there. If it is, there will be additional instructions for using MathType with it.

daniel g. grau say:
Unless your online school system can understand the format used by Word in its equations (called MathType), you cannot transfer them. You can convert equations to pictures, if the system supports images. To do that:

Paste as picture in Word
  1. Select the equation and copy it to the clipboard.
  2. In the main menu bar, select Paste Special. Make sure "Paste" is selected (not "Paste Link").
  3. The equation will paste as a graphic, and it will seem to lose resolution, but a quick Zoom-in will show it is of sufficient quality. Also, it will be page-wide. If you don't want this, you can crop it:
  • Click on the picture-equation.
  • In the Ribbon, click on the Format Picture purple tab.
  • Select the fifth icon, Crop.
  • Move the cursor to the left edge. When it looks like a sideways T (⊣) move the left border to the right.
  • Likewise on the other side (⊢). Zooming will help.

Take a snapshot
  1. Zoom real close on your equation. The larger you zoom in, the larger the quality (more pixels).
  2. Press Command Shift 4. This will turn your cursor into crosshairs.
  3. Click and drag across the equation. (Press ESC to Cancel.)
  4. When you let go, you'll have a picture on your Desktop. Rename it suitably.
What doesn't work
Saving the .docx document into the older .doc format will convert all your equations into graphics, all at once. However, the display quality is miserable.
