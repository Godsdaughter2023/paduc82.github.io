scottgem say:

So the question is, how to get it to duplicate the current record and append and replace the value in Field-1 with the next highest number and make it a function under a button on the form?

The correct answer is; if you are duplicating a large part of an existing record, changing only a couple of fields then your database is not normalized properly. One of the main principles of normalization is to eliminate redundancy. For example: If you have an Order record with customer shipping info And you are repeating everything but the OrderNo and date, that is bad design. Instead, you should have a Customer table, with address info and then only have the CustomerID in your Order record. 

So if you find yourself duplicating records as you described, then you need to rethink your structure.

The incorrect answer, is to generate and execute an Append statement in a VBA module that would concatenate in the incremented number.

You don't have fields on a form, you have controls, that may or may not be bound to fields in a table This is a subtle but important distinction. saataja say: WLAN usage without SIM card (or with SIM card when Offline profile is activated) has been working fine with E51.

However there seems to be some limitations with locating / scanning WLAN networks in Offline mode.

First limitation is that hidden WLAN networks (SSID's) don't seem to be discoverable or accessible when operating in Offline mode.

You should probably also change your WLAN "Network Status" from "Public" to "Hidden" in IAP settings in order to be able to locate and connect to hidden WLAN network when phone has a SIM card in place and some non-Offline profile is activated.

Other WLAN scanning related limitation seems to be that in Offline mode phone scans only WLAN channels between 1 and 11. Thus in case WLAN access point happens to be configured to use WLAN channel 12 or 13 phone will is not able to discover and connect to this WLAN network in Offline mode.

Other thing worth verifying is that your WPA/WPA2 Pre Shared Key (password) matches between the phone and your WLAN access point.
