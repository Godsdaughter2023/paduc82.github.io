jay freedman say:

I haven't been able to reproduce the error 4198, but that may be because the fake document I used for testing doesn't look like your document. If you can post that document on a sharing site such as OneDrive or Google Docs and provide a share-and-edit link to it, I may be able to work that out.

I will say, though, that once you get past the error you'll find that the macro doesn't perform quite as expected. Because the macro retrieves the variable strTheText only once at the beginning, every section will be saved with the same file name; each one will overwrite the one that precedes it. I presume that each section will contain a different Job Title value, and the macro needs to find that value within the Sections loop. You need to set rng1 to the range of the

There are several other minor things, such as unused variables, undeclared variables, unnecessary End statements, and the On Error statement being in the wrong place (it needs to be placed before any statement that may trigger an error). Try this version of the macro:

Sub BreakOnSection()
    Dim rng1 As Range, rng2 As Range
    Dim I As Integer
    Dim strTheText As String
    
    Application.ScreenUpdating = False
    
    For I = 1 To ActiveDocument.Sections.Count
    
        Set rng1 = ActiveDocument.Sections(I).Range
        If rng1.Find.Execute(FindText:="Job Title:") Then
            Set rng2 = ActiveDocument.Range(rng1.End, ActiveDocument.Sections(I).Range.End)
            If rng2.Find.Execute(FindText:="Wage/Hour Status: ") Then
                strTheText = Trim(ActiveDocument.Range(rng1.End, rng2.Start).Text)
            End If
        End If
        
        ActiveDocument.Sections(I).Range.Copy
        
        On Error GoTo CopyFailed
        Documents.Add
        Selection.Paste
        ActiveDocument.SaveAs FileName:="C:\Users\h\Desktop\HR\" & strTheText & ".docx"
        ActiveDocument.Close
    Next I
    
CopyFailed:
    Application.Quit SaveChanges:=wdSaveChanges
End Sub

There are still some things that could go wrong, depending on what's in the document on which you run the macro. For example, if the specific strings "Job Title:" and "Wage/Hour Status: " don't exist in the first section, then strTheText will be an empty string, which will trigger an error. If the first section is OK but one of the later sections doesn't contain those strings, then strTheText will keep the same value from the previous section, and the new section's text will overwrite the previous one. You need some code to prevent that, using Else statements in the two If/End If structures.

Sub MakeItBigger_R1()
  Dim rng As Excel.Range
  Dim NewRow As Long
  Dim N As Long
  Const STR_CHARS As String = "zzzzz"        '<< unique entry in cell
 
  Set rng = ActiveSheet.Range("C12:C500") '<< adjust as necessary
  For N = 1 To rng.Cells.Count
    If rng(N).Value = STR_CHARS Then
      NewRow = rng(N).Row
      ActiveSheet.Rows(NewRow).Insert shift:=xlShiftDown
      Exit For
    End If
  Next
  MsgBox "New blank row is row " & NewRow & "  "
End Sub
