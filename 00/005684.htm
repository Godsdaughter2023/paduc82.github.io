You can create a basic macro to ungroup and assign it to a key.

1) group some objects

2) select "record macro" on Developer tab (you may have to put the tab on the ribbon)

3) assign a name and a key to store the macro on (e.g., Control M, or another key combination you are not using)

4) save it to your normal template or only the document as you wish

5) Click OK

6) Select the grouped objects

7) Click on Run Macro

8) Right click grouped object and ungroup

9) Click Pause Recording

Test the macro using the key combination you chose.

This impacts workflow and introduces errors in documentation.

Turns out the issue is caused by some weird way that the  \*MERGEFORMAT switch is working.

Its annoying because it appears to be inconsistent with where and how it is decides to mess up the formatting.

The only way I could reproduce it (and then not every time) was: If the value being put into the Field is longer than what was previously held AND there is a space after the length of the original value, Then it appears to apply some default formatting to what is beyond the length of the original value. 

In our case, the font was very similar (same font but 1 size up) so not very obvious, but it was also hidden text, so really obvious when it messed up.

When I removed \*MERGEFORMAT from the Fields everything works fine. 

Doesn't appear to be anything to do with SharePoint, its just that SharePoint forces changes to the Fields. (File path especially, but also Title)
