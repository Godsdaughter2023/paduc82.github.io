john w. vinson say:

You can get both of these by using "Correlated Subforms". You still need a logical parent and child relationship - I presume that the meds would depend on the treatment?

If so, you can put two continuous Subforms on the main form, side by side. The first would display the treatments, and would need to include the TreatmentID or whatever field links treatments to meds. On the mainform you would have a textbox, txtTreatmentID, with a control source like

=[sbfrmTreatments].Form![TreatmentID]

This text box can be invisible if you wish.

The Master Link Field of the medications subform would then be

[txtTreatmentID]

referencing the name of the form control, not the fieldname; the Child Link Field would be the TreatmentID.

When you move from record to record on sbfrmTreatments, the second subform should reflect the currently active record.

Retina Signature Images Replaced with low res images by Outlook - Microsoft Community

pauljameson say:

Retina Signature Images Replaced with low res images by Outlook

Integrating large group of Mac users into our corporate Windows environment and working well until dealing with company logo in signature as their retina screens make the logo seem blurry.

Since we can't use SVG in mail clients, we followed the advice from MailChimp to handle this issue. "Keep High-Density Displays from Uglifying Your Emails"

  
We adjusted the logo file on the server to be double the dimension required in width and height which then quadruples the pixel density.  Since the image attribute for the html markup forces it to show at the correct dimension there is no change in the layout.

For example, a 100 x 50 pixel image was displayed with <img src=”logo.gif” width=”100” height=”50”>.  This showed fine in standard displays and poorly in retina displays.  

Now we use a 200 x 100 pixel image with the same code <img src=”logo.gif” width=”100” height=”50”> and it renders fine in both standard and retina displays.

This technique has worked well for our Windows users on Office 2013, but the Office 2016 for Mac users have a bizarre issue.  The moment they paste the HTML into the signature pane, it looks great and the logo is sharp for a moment.  As soon as focus is changed and they come back or if they compose a new email, the logo is blurry.

I asked them to send me a test email and in view source I can see that an embedded PNG at the reduced, layout dimension is instead used.  <img src=cid:*** Email address is removed for privacy ***66290 >

Maybe Outlook for Mac is being parental in correcting someone who might use a 10,000 x 5,000 pixel image to fit into a 100 x 50 pixel space by generating that new embed image.
