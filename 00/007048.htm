suzanne s. barnhill say:

If the figure has any wrapping other than In Line With Text, the caption will be inserted in a text box, and you should be able to group the caption and text box.

If the figure is In Line With Text, the caption will be in an ordinary text paragraph. Formatting the figure (assuming it is on top) as "Keep with next" (on the Line and Page Breaks tab of the Paragraph dialog) will suffice to prevent the figure and caption from being separated at a page break but will not allow you to move one and have the other accompany it (though this is easily enough done just by selecting both).

For that purpose, you can, as Doug suggests, insert both in a single (borderless) table cell, or you can put the figure in one cell and the caption in another, with the first row formatted as "Keep with next." You can drag the table by the table handle, but be aware that this will result in the table wrapping becoming Around instead of None, which rather defeats the purpose of having it In Line With Text to begin with.

try this,

Sub qazx2()
Const N As Long = 5 '<< source sheet, headers in row 5
Const sDate As String = "G" '<< header 'Dates' in column G

 Dim ws1 As Worksheet, ws2 As Worksheet
Set ws1 = Sheets("INPUT_DATA") '<< source sheet name
Set ws2 = Sheets("Schedule") '<< target sheet name

Dim i As Long, r As Long, t As Long
r = ws1.Cells(Rows.Count, sDate).End(xlUp).Row

Application.ScreenUpdating = False
ws2.Cells.ClearContents
ws1.Rows(N).Copy ws2.[A1]
t = 2
For i = N + 1 To r
c = ws1.Cells(i, Columns.Count).End(xlToLeft).Column
c = c - Cells(1, sDate).Column + 1

ws1.Cells(i, "A").Copy ws2.Cells(t, "A").Resize(c) '<< col A

ws1.Cells(i, "B").Copy ws2.Cells(t, "B").Resize(c) '<< col B

ws1.Cells(i, "C").Copy ws2.Cells(t, "C").Resize(c) '<< col C

ws1.Cells(i, "D").Copy '<<< col D
ws2.Cells(t, "D").Resize(c).PasteSpecial xlPasteValues

ws1.Cells(i, "E").Copy '<<< col E
ws2.Cells(t, "E").Resize(c).PasteSpecial xlPasteValues

ws1.Cells(i, "F").Copy ws2.Cells(t, "F").Resize(c) '<< col F

ws1.Cells(i, sDate).Resize(, c).Copy '<< col G
ws2.Cells(t, sDate).PasteSpecial xlPasteValues, Transpose:=True
ws2.Cells(t, sDate).Resize(c).NumberFormat = "m/d/yyyy"
t = ws2.Cells(Rows.Count, sDate).End(xlUp).Row + 1
Next
Application.CutCopyMode = False
ws2.UsedRange.EntireColumn.AutoFit
Application.ScreenUpdating = True
End Sub

jay freedman say:
You can't. MSLineDraw and a bunch of other fonts have not been supported in any version of Word since Word 97 (http://support.microsoft.com/?kbid=163059).

Courier New and the Unicode fonts, Arial Unicode MS and Lucida Sans Unicode, include the full set of line drawing characters in the Box Drawing range between hex 2500 and 256C. Many other fonts include some of those characters You'll have to replace the MSLineDraw characters with their Unicode equivalents in a supported font.

Word 2013: determine border color and weight in a table cell - Microsoft Community

beejie say:

Word 2013: determine border color and weight in a table cell

I have several tables that use slightly different border colors and rule weights. If I point to a cell, how can I see a summary of these attributes? I'm using Word 2013 on a PC.
john korchok say:

About the best you can do is to first select the whole cell, then use TABLE TOOLS>LAYOUT>Properties. Click on Borders and Shading, then change the Apply to: dropdown to Cell.

If the cursor is blinking inside the cell, you're not going to get any useful information. The whole cell must have a gray selection across it.
