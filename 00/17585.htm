if I put a range  cells (A1:A10) into an element of array, is it inserted as an object? Are all the properties that apply to a range gone? Addressing a single cell is therefore impossible, right?

I assume you mean something like this (for future note:  concrete examples are always helpful to avoid ambiguity):

Dim v As Variant    ' if omitted, this is the default interpretation of v below
v = Range("a1:a10")

In that case, only the value of each cell in the range is copied into array v.

The "properties" (format, font, etc) are not copied.

You can "address a single cell" by indexing the array.  Since you assigned a single column, you would use v(1,1), v(2,1) etc.  If you had assigned a single row, you would use v(1,1), v(1,2) etc.  For a 2-dimensional range:  v(1,1), v(1,2), v(2,1), v(2,2) etc.

But you are really addressing the values in the array v, not the cells themselves.  For example:

v(1,1) = 2

changes part of array v in VBA, but not A1 in Excel.

To change Excel, eventually you might write:

Range("a1:a10") = v

That is an efficient way to modify Excel.  Alternatively, you might alter individual cells.  For example:

Range("a1") = v(1,1)
