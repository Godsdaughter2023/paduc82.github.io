Hello Kokokox,

The pagefile.sys is the Windows paging file, also known as the file that Windows uses as Virtual Memory. You do not want to delete this file.

If you delete the hiberfil.sys file you will turn off hibernation. You can free up a decent amount of space by doing this but you computer will no longer hibernate.

The file C:\BOOTSECT.BAK it is a copy of the WinXP-style boot sector for the System Partition. It would most likely be on your system if you had upgraded your computer from Windows XP.

Hi, I am using the Surface Pro with win 8.1 pro, and all I want is for the tablet to wake me up in the morning. The Pro is a full fledge computer (more like an Ultrabook than iPad), and does not support Connected Standby (what you are calling InstantGo).


A work around would be leaving the Pro Plugged in and preventing Sleep or Hibernate (you'll want to setup the Notification so they don't make noise all night). Set the Alarm and continue.


The Surface 2 would support the scenario you're looking for as it supports Connected Standby and can be used as an Alarm without needing to modify the Power Scheme.

jeffrey roach say:

The Pro and Pro 2 uses the Intel Core i5 CPU and Chipset (Pro Gen one uses IvyBridge and Pro 2 uses Haswell) These don't support Connected Standby as they function as Ultrabooks in Tablet Form.


The Surface 2 runs on the Tegra4 ARM CPU and Chipset (like a smartphone or consumer tablet) and supports Connected Standby.


Currently Connected Standby requires a System on Chip (SoC) that runs a power efficient design.


Intel has their ATOM line (Clovertrail and Baytrail) and then any of the ARM SoC's from NVidia. Qualcomm, TI, etc.

    jeffreyheck say:

    I have many Windows 7 Professional PCs that had this problem. Here is my solution:

    1. In Windows Update, right click on the update and hide it.
    2. Check for updates and install new updates it finds.
    3. If any of those updates fail, hide those, check for updates again, and install. Restart if instructed to.
    4. Repeat step 3 until all updates successfully install. Restart if instructed to.
    5. Restore the most recently hidden update.
    6. Check for updates and the previously hidden update will either install or Windows Update will say it is not needed. Restart if instructed to.
    7. Repeat steps 5 and 6 until no updates are left to install.
