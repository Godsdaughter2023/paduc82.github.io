davem121 say:
Hi Justin

If I understand correctly you are booting Windows from the 500 gig NVMe m.2 and you also have a 4TB drive for data, is that correct?

Did you leave the 4TB drive connected to your PC when you were installing Windows?

If so, that is the error you made, there is a little known issue with the Windows 10 installation routine. If another drive is connected at the time of the installation, the boot config and/or page file can end up on the wrong drive

You need to re-install Windows 10 on the 500 gig NVMe m.2, this time, disconnect the 4TB drive and do not connect it until the installation has completed

Then you can use Diskpart to convert that drive to GPT, you cannot do that now, because Windows Boot sector is on that drive . . .

amrita m say:

Hi,

 

 

What changes did you make to the computer that the network started conflicting?

 

Method 1:

As you said it was working fine you may try to perform s system restore and check if it works fine as earlier.

You may want to perform a System Restore on the computer to restore back the previous settings.

 

1. Click Start, type system restore in the Start Search box, and then click System Restore in the Programs list. 

If you are prompted for an administrator password or confirmation, type your password or click Continue.

2. In the System Restore dialog box, click Choose a different restore point, and then click Next.

3. In the list of restore points, click a restore point that was created before you began to experience the issue, and then click Next.

4. Click Finish.

 

 

Note: When you perform System Restore to restore the computer to previous state, programs and updates you installed are removed.

 

Method 2:

If it still fails try the steps given below.

This might be happening if the DHCP server is assigning the same IP address to the Local Area Network Connection as well as the wireless connection. You may try to assign a separate IP address to the Local connection and check if that works fine.

1) Click on Start

2) Go to Control Panel

3) Double Click Network Connections

4) Under LAN or High-Speed Internet Double click the enabled device (the Local Area Connection)

5) On the Status Box Window that opens Click the Properties button

6) Drag the Properties window beside the Status window

7) Click the SUpport tab on the Status window

8) Click the Details button--this window provides the information you will need to manually configure your IP

9) In the Properties window that you have open beside the Status windows scroll down through the devices until you get to Internet Protocol TCP/IP

10} Select Internet Protocol TCP/IP and click the properties button

11) In the Properties window that opens select "Use the following IP address"

12) If you have a number of machines begin by setting the IP address to 192.168.1.100 then go to 192.168.1.101 and so forth giving each machine a unique IP

 

After changing the IP address of the LAN check if you are able to connect both together.

 

 

Hope this information is helpful.

with their own software and drivers and any other major device drivers you have.

Maker's site (Realtek, Intel, Nvidia, ATI as examples) and get their latest versions. (Look for

Download - SAVE - go to where you put them - Right Click - RUN AD ADMIN - REBOOT after
each installation.

shows up. This is because some drivers rollback before the latest is installed (sound drivers
particularly do this) so install a driver - reboot - check to be sure it is installed and repeat as
needed.

Repeat at Device Makers - BTW at Device Makers DO NOT RUN THEIR SCANNER - check manually
by model.
