Starting a week on a Friday using the =weeknum function? - Microsoft Community

zgarrison say:

Starting a week on a Friday using the =weeknum function?

I'm wanting to start my week on a Friday.  How can I use the weeknum function to have the week run Friday-Thursday?

Also, is there a way to have a column next to that to represent the week period?  For example, Week 14 is April 6-April 12.  I want column A to post the week # and column B to post the week range column A represents.
ronrosenfeld say: Easiest way is to use a simple UDF.

To enter this User Defined Function (UDF), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this User Defined Function (UDF), enter a formula like

=VBWeekNum(A1,6)

 in some cell.

(Note the second argument is for the week starting day where 1=Sunday and 7 = Saturday; so 6 = Friday, so you could use this for any week starting date)

================================
Function VBWeekNum(dt As Date, WkStart As Long) As Long
    VBWeekNum = DatePart("ww", dt, WkStart)
End Function
======================================

And to get the week range, you don't need to know the week number since you have the date.  You just need to compute the previous Friday and the following Thursday, making the appropriate adjustment if the date IS a Fri or Thu:

So with your date in A1:

=TEXT(A1+1-WEEKDAY(A1-5),"mmm dd")&"-"
&TEXT(A1+7-WEEKDAY(A1-5),"mmm dd")

And by the way, you wrote you wanted this to work like the Weeknum function.  The Weeknum function assumes week 1 is the week containing Jan 1.  That being the case, Week 14 (of this year, anyway) would be Mar 30-Apr 5.  Or do you have something else in mind?

brian tillman [outlook mvp 2007-2019] say:
Ron's correct.  Mail transports listen for incoming connections all the time.  They don't poll for incoming mail, they sit there waiting for a transport to contact them and make the transaction.  When a mail transport has a message intended for another domain, it makes the connection then and there if it can.  Thus, the sender's mail transport will contact its next hop immediately.  That one will contact the next in line, and so on until finally a transport will connect with the Outlook.com transport, tell it there's a message coming in , and the Outlook.com transport will accept it, sticking it in the recipient's mailbox.  No polling involved.  The only time polling is involved is when someone has created a send/receive account inside their Outlook.com account.  That's what polls every 30 minutes.
