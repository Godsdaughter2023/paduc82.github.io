john w. vinson say:

Query datasheets aren't really designed for data presentation. Instead, create a Form (for onscreen display) or Report (for either onscreen or for printing) based on the query, and use the Form or Report Footer to display the total by including a Textbox with a control source like 

=Sum([NumberOfTimes])

It's then not necessary to open or see the Query at all - just open the Form/Report.

Open the query in datasheet view.

This adds a Total row at the bottom of the query.

Click in the NumberOfTimes column in this row.

Select Sum from the dropdown menu.

scott mcdaniel say:

What happens if you make a new form similar to one of those, and the try the filtering options? If the new form exhibits the same behavior, then you may have a corrupt database (see below). If the new form does not exhibit the same behavior, you may have a corrupt form.

To deal with corruption:

Make a backup of your database and perform a Compact/Repair. If that doesn't resolve the issue, you may need to Decompile it. To do that, create a shortcut with this as the Target:

"full path to msaccess.exe" /decompile

Run that shortcut, then select your database. Access will decompile the database, and it will open. Before doing anything else, however, do these 3 steps:

1. Compact the database

2. Compile it - from the VBA Editor click Debug - Compile. Fix any errors, and continue doing this until the menuitem is disabled

3. Compact it again

You might also consider creating a new, blank database and import everything to the new database. Note that if you have a corrupt form, you may not be able to import that form. If not, you'd have to recreate it from scratch, or import it from a known good backup.

Finally, be sure that your Office and Windows installations are fully up to date.

myall_blues say: Open a new blank workbook and change all the columns to width 24.
Save the workbook as a template file (type xlt) and save it as Book.xlt in the XLSTART directory.
The XLSTART dorectory will normally be at C:\Users\username\AppData\Local\Microsoft\Excel\XLStart.

andreas killer say:
A template in the XLSTART folder affects all new created workbooks.
Therefore IMHO this is not the way to go.

The only way to accomplish that in only this workbook is to use macro.

Right-click on the sheet tab
Choose "View Code"
Within the Project Explorer, search for "ThisWorkbook"
Double click on it
Paste the following code
Close the VBA editor
Save the file as .XLSM
Be sure your macros are enabled 

Private Sub Workbook_NewSheet(ByVal Sh As Object)
  Sh.Columns.ColumnWidth = 24
End Sub
