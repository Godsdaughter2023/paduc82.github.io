How do you want this to work? Do you want to select and reply to individual messages, or do you want to process a whole folder of messages. Are the messages to be HTML or plain text? Do you want to add body text to your pre-written messages?

Maybe the following will get you started. This will create replies to selected messages. While testing the mesages are created but not sent.


Sub CustomReply()
Dim olItem As Outlook.MailItem
Dim olOutMail As Outlook.MailItem
Dim vText As Variant
Dim sText As String
Dim vAddr As Variant
Dim sAddr As String
Dim i As Long
Dim strBody As String

If Application.ActiveExplorer.Selection.Count = 0 Then
    MsgBox "No Items selected!", vbCritical, "Error"
    Exit Sub
End If
sAddr = ""
'Process each of the selected messages
For Each olItem In Application.ActiveExplorer.Selection
    sText = olItem.Body
'Process each line of the message body looking for the @ symbol which hopefully only appears in e-addresses
    vText = Split(sText, Chr(13))
    For i = 1 To UBound(vText)
        If InStr(1, vText(i), "@") Then
            sAddr = vText(i)
            Exit For
        End If
    Next i
    If sAddr = "" Then
        MsgBox "Message titled" & vbCr + vbCr & olItem.Subject & _
               vbCr + vbCr & "skipped."
        GoTo skip:
    End If
    If InStr(1, sAddr, "HYPERLINK") Then
        vText = Split(sAddr, Chr(34))
        For i = 1 To UBound(vText)
            If InStr(1, vText(i), "@") Then
                sAddr = vText(i)
                Exit For
            End If
        Next i
    End If
   
'Define the categories and their messages here

    Select Case olItem.Categories
        Case "Category Name 1"
            strBody = "This is the message for Category Name 1"
        Case "Category Name 2"
            strBody = "This is the message for Category Name 2"
        Case Else
            strBody = "This is the message for messages without a category listed" ' above
    End Select
   
 'Compile the reply  
    sAddr = Replace(sAddr, "mailto:", "")
    Set olOutMail = olItem.Reply 'CreateItem(olMailItem)
    With olOutMail
        .To = sAddr
        .Body = strBody
        .Display        'Change to .Send after testing
    End With
skip:
Next olItem
Set olItem = Nothing
Set olOutMail = Nothing
End Sub

You are better off with using a DMAX and then committing the record immediately afterward. You do not need to store the PP, just the generated number. 

ken sheridan say:

I figure for that it would be smart to limit the form to one user at a time to prevent conflicts

There is no need to do that.  As my demo shows, conflicts can easily be handled.  The sequential numbering option in my demo is a development of a method originally published by Roger Carlson, which automatically increments the  number in the event of a conflict.  This has the advantage of not prejudicing the integrity of the database by unwarranted Nulls, which is an inherent consequence of other approaches.

Thank you for posting to the forum regarding the issues you are having changing the APN settings on your phone.  You can manually add APN settings to a Windows Phone 7.  This platform does only support one at a time so if you have different APN's for data v. calling/texting, you may need to manually change these depending on the action you are trying to take.

You can add the APN settings manually by going to settings > cellular > add apn > enter the requested details.  If you do not have this option, this is related to the firmware on the phone and would be something you need to discuss with HTC.  If they cannot help to get the firmware updated, this may be a limitation of the unlocked phone.

Please let us know if you have any other questions or concerns.
