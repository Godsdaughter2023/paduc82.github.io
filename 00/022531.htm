anthracyte say:

5 Day Weekly Recurring Task split across resources

So there are 3 of us engineers and we all rotate around various projects and support tasks. All 3 of us have one day a week dedicated to personal training, or office administration tasks. The other 4 days of our week are dedicated to project or support work for our clients. Each of our days of the week are different, for instance, mine is a Thursday, whilst another of us have Monday and the other Friday.

So I've created Weekly recurring tasks for each engineer i.e Engineer 1 - Weekly Recurring task - Fixed Duration - 1 day - for the next two years for instance. The same for the other two engineers.

Then we have a regular task for a client which means that at least one of us must be onsite at their premises every working day. So for this I created a weekly recurring 5 day task - starts on Mondays ends on Fridays, fixed duration. From here I want to assign all 3 of us to the particular task for the week and I'd like Project to be able to schedule or split this task between us, but the task itself cannot move or be delayed. So if Engineer 1 is unavailable then Engineer 2 or 3 should go to site.

Of course unfortunately no amount of levelling or auto scheduling is making this work and all Engineers are assigned the full 5 days and are then over allocated. I have been able to manually level this by giving someone 3 days worth of work, and the other two one each, and manually moving them in the team planner, but this is very time consuming and not how it should be I'm sure!

I'd be very grateful for any help you can provide me here!

Thanks
john - project say:
Anthracyte,
Well first of all there isn't a quick and simple way to do what you want.

The first part (training, admin tasks) is easy to implement. The second (on-site at client's facility) requires a little more effort and it depends on how you want to track stuff.

I would not use recurring tasks, at least not to control when each resource works on which task.  Instead control engineering effort via resource calendars. Here's a starting point.

1. Create three resources called "Joe in-house", "Bob in-house" and "Tim in-house". Create resource calendars for each based on the day of the week they will be doing the training, admin, etc. in-house tasks. For example, assume you are Joe and create your calendar as having only one working day per week - Thursday. You do this by selecting the Resource on the Resource Sheet and then going to Project/Change Working Time. On the Change Working Time window "Joe in-house" will appear in the For Calendar selection box at the top of the window. Click on the Work Weeks tab in the lower half of the window, then click Details. Select each weekday and set it as non-working time except for Thursday. Then hit OK. You will see the "Joe in-house" calendar shows Thursday as the only working day for all months and years. Do the same thing for each of the other two engineers based on their in-house work day.

2. Create a task that is fixed duration and lasts two years. You could, at this point create a recurring monthly task lasting two years. This task is the in-house training, admin, etc. effort.

3 Assign each of the three engineers ("Joe in-house", "Bob in-house", and "Tim in-house") at 100% to the task in step two. If you then look at the Resource Usage view you will see each engineer only working on in-house task on their specific day of the week throughout the task's duration

That was the first easy part. The second part, on-site at the client's facility, can be done a couple of ways, (maybe more), and the best way depends on how you want to track effort. I'll outline a couple of approaches.

4. First create three more resource calendars called "Joe at client", "Bob at client", and "Tim at client". Set Joe's calendars such that the working days are Mon, Tue, Wed, and Friday with Thurs being a non-working day. Do a similar thing for each of the other two resources.

5. Create another task that is fixed duration and lasts two years. Again make it recurring if you want to break down effort into more manageable time spans (e.g. monthly). This task is the on-site client effort

6. Assign all three "at client" resource to the task in step 5 at 33%. On the Resource Usage view you will see all three engineers supporting the client on-site effort at 2.63h per day with no work on their in-house effort day.

If you don't want the three engineer's effort equally spread each day, the only option is to manually contour each resource's "at client" effort day by day or week by week. The latter would be a little quicker but nonetheless an arduous effort if the task span is long. There is no automated way to get Project to do it for you - sorry.

However, another approach to the client effort would be to create a resource called "3 engineers". Set the Max unit level to 300%. That effectively creates a 3 man engineering team. Now, instead of assigning all three "at client" resource to the client on-site task, assign the single "3 engineers" resource to the task. The advantage of this approach is it gives 8 hours of effort at the client's site and it can be any one of the 3 engineers, because they are a non-individualized team. The disadvantage of this approach is that you will not be able to see which particular engineer is on-site at the client, but then maybe you don't need that level of detail.

There you have it, simple as pie.

Hope this helps.

John

trip_to_tokyo say:

PowerPoint 2007

Here is a simple example.

This is only one way to do it.

1. Start PowerPoint.

It should start with one slide in it.

2. Home / Slides group / click on the arrow to the right of New Slide / click in Title and Content.

A second slide should now be added.

3. Click on the icon Insert Chart / Insert Chart should launch / click on OK.

4. An EXCEL Workbook should now launch and it will have some trial data in it.

You can change this if you want to but let’s leave it unchanged.

5. You should see a chart for that data in slide 2.

6. Now let’s save the PowerPoint presentation:-

Office Button (top left hand corner) / Save As (on left hand side) / Save As window should launch / in the File name field: type in the file name, for example:-

C:\presentations\embedded_file_testing

(You need to change the above to your own choice).

7. Click on Save (lower right hand corner).

8. Close the file and then reopen it.

You should now have your embedded EXCEL file on the Presentation in front of you.
