a. user say:

<<
Maybe is there some setting for activate old-style behavior non-breaking space in a justified paragraph or I can only to save document as Word 2003 format?
>>

If you open a document in compatibility mode, then go into the File tab, Options, Advanced and scroll to the bottom of the list, you will see a long list of Compatibility Options". But as far as I can tell, none of those options is there to let you specify the behaviour of the non-breaking space. Further, if you save your document in full Word 2013 .docx format, all of the compatibility options disappear. As I understand it, this is because in Word 2013, Microsoft made major changes to the "layout engine" and wanted to leave all those options behind.

So there are no such options (the only one that is "close" is still there under the Layout options "Don't expand character spaces on a line that ends with SHIFT-RETURN", and that option won't affect the layout in the case you describe.)

So as far as I can tell, your only option is to try to ensure that you create and/or keep a document's compatibility mode. The trouble is that once you have saved in Word 2013 "native" format (there are now two, .docx and the Strict version of .docx), nothing in the user interface helps you to go back to compatibility mode. So I think you have to do it in VBA - you can use, e.g.

ActiveDocument.SetCompatibilityMode 11 ' to set Word 2003 compatibility mode
ActiveDocument.SetCompatibilityMode 12 ' to set Word 2007 compatibility mode

So here is my way, maybe that is easier for you. But even this way is not different from all previous ones, only the execution is a different one.

The scenario is:

Here is my macro:

Sub SplitTextAndNumber()
  Dim Source As Range, Dest As Range
  Dim Data, DataN, Result
  Dim i As Long, j As Long, m As Long
  Dim s As String

  On Error Resume Next
  Set Source = Application.InputBox("Select source cells", "SplitTextAndNumber", Selection.Areas(1).Address(0, 0), Type:=8)
  Set Source = Intersect(Source, ActiveSheet.UsedRange)
  If Source Is Nothing Then Exit Sub
  Set Dest = Application.InputBox("Select destination cell", "SplitTextAndNumber", ActiveCell.Offset(, 1).Address(0, 0), Type:=8)
  If Dest Is Nothing Then Exit Sub
  On Error GoTo 0

  Data = Source.Value
  ReDim DataN(1 To UBound(Data), 1 To 1)
  For i = 1 To UBound(Data)
    s = Data(i, 1)
    For j = 1 To Len(s)
      If Mid$(s, j, 1) Like "#" Then Exit For
    Next
    DataN(i, 1) = Mid$(s, j)
    If IsNumeric(DataN(i, 1)) Then DataN(i, 1) = CDbl(DataN(i, 1))
    Data(i, 1) = Left$(s, j - 1)
    If Len(Data(i, 1)) > m Then m = Len(Data(i, 1))
  Next
  ReDim Result(1 To UBound(Data), 1 To m + 1)
  For i = 1 To UBound(Data)
    s = Space$(m - Len(Data(i, 1))) & Data(i, 1)
    For j = 1 To Len(s)
      Result(i, j) = Mid$(s, j, 1)
    Next
    Result(i, j) = DataN(i, 1)
  Next
  Dest.Resize(UBound(Result), UBound(Result, 2)).Value = Result
End Sub

dirk goldgar say:

When the A2013 user opened your database, at least one library reference was updated to a later version of the library, and has not been successfully downgraded.  As a first attempt to fix the problem, open the database while holding down the Shift key (to suppress any startup code), press Alt+F11 to open the VB Editor environment, then click Tools -> References ... to open the References dialog.  All active references will be at the top of the list.  First look for any reference marked MISSING and, if possible, uncheck it; then look down the list for the appropriate version of that reference.  Note that Access may not allow you to un-check it some references, which can force a more complicated fixup process if they are broken. 

Also look for references that are set to a later version of Office or Access than are installed on your machine (they should be MISSING as well, I think) and try to uncheck them and add the correct reference from the list.

If this doesn't work, then I'll have to search my files and probably online for the more complicated fixup procedure, which involves changing the registry.
