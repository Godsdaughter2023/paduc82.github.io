Win 8, 8.1 and 10 have a new feature called Fast Startup. It can sometimes cause problems like you mention as well as others.

The easiest way to tell if this is the problem is to turn it off and see what happens.

This feature only works when you do a SHUTDOWN and then Boot.

It doesn't effect a RESTART.

To turn it off or on...

Go to Control panel...Power Options, and select Choose What the Power Buttons Do on the left.

Then select Change Settings That Are Currently Unavailable near the top center of screen...

Lower down on the window, uncheck Fast Startup.

If this resolves the problem, then you can investigate what isn't loading properly.

j.e. mcgimpsey say: Sounds like the problem you're having is not as much about Full Screen as it is that Mac and Windows screen resolutions are different, so when WinXL users have to set their Zoom setting to 75% in order to get a reasonable amount of their worksheet in view, it's very small when the same setting comes up on the Mac. 

I use a routine in my startup add-in to reset window size and zoom for every workbook I open (except for those I exempt):

I put this in the ThisWorkbook code module of the Add-in (I've deleted all the other code for clarity):

    Private Sub Workbook_Open()
        SetupClasses
    End Sub

    Private Sub Workbook_BeforeClose(Cancel As Boolean)
        KillClasses
    End Sub

I put this in a regular code module:

    Public clsMyWindow As CMyWindow

    Public Sub SetupClasses()
        Set clsMyWindow = New CMyWindow
    End Sub
    
    Public Sub KillClasses()
        Set clsMyWindow = Nothing
    End Sub

I put this in a Class Module (named CMyWindow):

    Public WithEvents oApp As Application
    Private Const mnWindowDefaultTop As Long = 1&
    Private Const mnWindowDefaultLeft As Long = 1&
    Private Const mnWindowDefaultVerticalMargin As Long = 400&
    Private Const mnWindowDefaultHorizontalMargin As Long = 600&
    Private Const mnWindowDefaultZoom As Long = 100&

   ' Class Methods
   ' ****************************
    Private Sub Class_Initialize()
        Set oApp = Application
    End Sub
    
    Private Sub Class_Terminate()
        Set oApp = Nothing
    End Sub
    
   ' Event Methods
   ' ****************************
    Private Sub oApp_SheetActivate(ByVal Sh As Object)
        On Error Resume Next
        bExempt = Sh.Parent.Names("JEM_Do_Not_Resize").Value
        If Not bExempt Then _
            ActiveWindow.Zoom = mnWindowDefaultZoom
        On Error GoTo 0
    End Sub

    Private Sub oApp_WorkbookOpen(ByVal Wb As Excel.Workbook)
        SetWindows Wb
    End Sub

   'Private Methods
   ' ****************************
    Private Sub SetWindows(ByVal wbBook As Excel.Workbook)
        Dim wnWindow As Window
        Dim bExempt As Boolean
        
        On Error Resume Next
        bExempt = wbBook.Names("JEM_Do_Not_Resize").Value
        If Not bExempt Then
            For Each wnWindow In wbBook.Windows
                With wnWindow
                    .Top = mnWindowDefaultTop
                    .Left = mnWindowDefaultLeft
                    .Height = oApp.UsableHeight - mnWindowDefaultVerticalMargin
                    .Width = oApp.UsableWidth - mnWindowDefaultHorizontalMargin
                    .Zoom = mnWindowDefaultZoom
                End With
            Next wnWindow
        End If
        On Error GoTo 0
     End Sub

You can save this workbook as an add-in and add it in using Tools/Add-ins...

From then on, any time you open a workbook, the window size will and zoom will be set to your values rather than what they were saved at. When you switch worksheets, the zoom will be reset to whatever you've got mnWindowDefaultZoom set to.

Change the mnWindowDefaultVerticalMargin and mnWindowDefaultHorizontalMargin values to suit - I set them to give me a strip of desktop below and to the right of my workbook window. Full-screen would require setting both of them to zero.

Everything looks like it's working fine on the form.  When I enter the SSN, the FirstName and LastName fields correctly populate and I can complete the rest of the form and save my work.  However, when I look at the Status table, the FirstName and LastName fields are empty (everything else is saved correctly).

Help, please!

jeanette cunningham say:

The first name and last name fields are in the participants table. In the Status table, use the SSN without the first name and last name fields.

Whenever you want to know the status of a person by name, use a query to get it. The query will join the participants table to the status table on the SSN field.

Drag the ssn and first name and last name fields from the participants table onto the query grid and drag the status field from the status table onto the grid. Switch the query to datasheet view to see the status for each person.

In summary, avoid putting the first name and last name fields in more than one table.
