create a text box that only allows 25 characters per line - Microsoft Community

bregal say:

create a text box that only allows 25 characters per line

In a form i want to create a text box that only allows 25 characters per line. And if possible, number each line with the text.
jay freedman say:

The only way you'll get a fixed line length in a text box is to set the contents to a nonproportional font (such as Courier New or Lucida Sans Typewriter) at a specific size, and set the width of the
box to allow only 25 characters in that font and size. If the user changes the font or the size, all bets are off.

Word can't do automatic line numbering inside a text box. That function is enabled only for the body text of the document (it's in the Page Layout information).

Jay Freedman

wje1987 say:

Angle and Time conversion formula: H°mm'ss" to D.dddd° and vice versa

I have been unable to find a time/angle conversion formula in excel's built in formula bar, so I wrote my own. I'm a surveyor and currently taking GIS classes, so I convert bearings and azimuths back and forth for calculations along with latitude and longitude quite often. This conversion is standard on scientific calculators, so I am a little disappointed that there is no excel function for it (that I can find). 

Anyway, I post these here to hopefully help others who need to convert these numbers quickly without constantly fumbling around with a calculator. I can't find any formulas online either. I put a decent bit of time into writing these and debugging them. The INT and TRUNC functions have some sort of stupid bug that makes the conversions wrong (by exactly 40"). But by multiplying them by 1.000000001, it fixes it. Who knows why that is.

Degrees, Minutes, Seconds to Decimal:  H.mmss/D.mmss to D.dddd (H°mm'ss" to D.dddd°)

Example: 12°34'56" is input 12.3456, and converted to 12.5822 (12.5822°).

Conversion Formula:

This will handle negative angles (aka "left hand angles" or "west" longitudes):

IF(F10<0,(INT(ABS(F10))+(((((ABS(F10)-INT(ABS(F10)))*100)-INT((ABS(F10)*1.000000000001-INT(ABS(F10)))*100))/0.6)+INT((ABS(F10)*1.000000000001-INT(ABS(F10)))*100))/60)*-1,INT(F10)+(((((F10-INT(F10))*100)-INT((F10*1.000000000001-INT(F10))*100))/0.6)+INT((F10*1.000000000001-INT(F10))*100))/60)

Decimal to Degrees, Minutes, Seconds: D.dddd to H.mmss/D.mmss (D.dddd° to H°mm'ss")

Example: 12.5822° is input 12.5822, and converted to 12.3456 (12°34'56").

Conversion Formula:

This will handle negative angles (aka "left hand angles" or "west" longitudes):

IF(F14<0,((((ABS(F14)-INT(ABS(F14)))*60)-INT((ABS(F14)-INT(ABS(F14)))*60))*0.006+(INT((ABS(F14)-INT(ABS(F14)))*60)/100)+INT(ABS(F14)))*-1,(((F14-INT(F14))*60)-INT((F14-INT(F14))*60))*0.006+(INT((F14-INT(F14))*60)/100)+INT(F14))

Like I said, I hope these help others. If anyone from excel is lurking around, PLEASE add this function in the future. Something like "=H.ms" or "=H.hh". It would be just so convenient to not have to change 15 cell addresses every time I use these functions.

daniel pineault say:
Use a date picker/calendar for the date input and set the control's format to display date the way you want.

ken sheridan say:

Example (user enter in dd-mm-yyy) 13-04-2016

After upddate of the field is change by it own to regional - mm-dd-yyyy - 04/13/2016


With the example you've given the date saved to the table is the correct one. Dates are stored as a 64 bit floating point number, so the value stored is the same regardless of the local date format.  It's simply presented in the local format.  However, the only reason the date entered in the incorrect format by your user is interpreted correctly is that in the local date format it is incapable of being interpreted correctly, there being no month 13.  If the user had entered 04-07-2016, intending this to be 4th July 2016 it would be accepted, but as the date in this format is a valid one in the local date format, the date value stored would be one which represents 7th April 2016.

If you anticipate users entering data on machines which use different regional data formats without the user being aware which is in use, then you can  force users to use the data picker by putting code like this in the control's KeyPress event procedure:

    Const MESSAGE_TEXT = "Please use the date picker to select a date."
    
    KeyAscii = 0
    MsgBox MESSAGE_TEXT, vbInformation, "Warning"
    
though you'd probably want to extend the code a little to allow keys like the Esc, Tab etc keys to be pressed.

Note that when using the date picker the date will at first be shown in the local regional format, and will only change to the format set as the control's Format property when the control is updated by moving focus off it.

Alternatively, use three unbound controls for the day, month and year and put code in the form's module to read/write the true date value from/to a column in the table.  This only works easily in single form view, however.  It's possible in continuous forms view by using hybrid controls, but it's tricky to implement.  Or you can store the values in three columns of Integer data type and build a date value by calling the DateSerial function whenever you need to process it as a date data type.  This is easily done by always using a query which returns the date value whenever you would normally use the base table.
