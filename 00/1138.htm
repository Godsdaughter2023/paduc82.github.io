The right-click context menu file scan is entirely unnecessary.  To prove this to yourself, simply go to the Eicar.com website and try to download the ZIP file versions of the Eicar test file while Real-time protection is active.  You'll notice that the copy of the malware test file will be detected and placed in quarantine.

Since the executable version of the file is blocked by the SmartScreen Filter in Internet Explorer, it can't even be downloaded with Defender Real-time protection disabled.

If you wish to try simulating something that got downloaded while Defender was disabled, turn off Real-time protection and then download either or both zip files to a folder.  Then re-enable Real-time protection and simply try to "touch" the files by right-clicking on them.  You'll discover that Defender will immediately scan and detect the malicious test file before you can even choose an item in the context menu.

This is why the right-click to scan menu item is unnecessary, since Defender is inherently called by any attempted access of a file by the Real-time protection component.

Microsoft designed both Microsoft Security Essentials and Windows Defender such that only the Quick scan is required to detect known active malware.  The Full scan is only required if an infection has already occurred or is suspected.  For this reason the only automation required is a Quick scan and for what should be a very rare manual Full scan, manually selecting the High Performance power setting takes literally 2 seconds (I just did it).

I personally have done only 2 Full scans of my Windows 8.x PC in 2 years.  The first immediately after I set up the PC initially and the second shortly after I upgraded to Windows 8.1.

Also note that Defender will continue to scan if the screen goes blank, but it will stop if the computer sleeps, which is much more common and quick with most Windows 8 power settings but won't occur while High Performance plan is selected.  Also note that with most newer PC systems the scan is barely noticeable during operation, so you can perform the scan while working and avoid using extra power after hours.  Of course this may not complete quickly if you store large amounts of data, especially on externally attached USB drives.

Key Office 365 Home license terms to your question

  • you can use Office 365 Home
  • you can share it with up to 5 other people (up from previous limit of 4 other people
  • you and each user can have Office 365 active on up to 5 devices AT THE SAME TIME
  • a device can be a Windows desktop / Windows laptop / Mac / tablet / smartphone
  • there are other "features" not relevant to your question

Note: application feature sets will not be identical on the various device types, due to operating system limits.  On tablets and smartphone there are free "mobile" versions of Office Apps. When you have an Office 365 subscription this activates more features.

So you as the "subscriber" can have Office active on 5 devices at the same time.  If you want to go beyond 5 devices, only then you have to deactivate one of the current 5 devices. BUT, it will not be uninstalled, just deactivated. Office will then go into "limited function mode" on the deactivated device. When you return to the deactivated device you can activate it without having to install again.

Each of the people you share with has the same limits and control.

You can "unshare" any of the other people you shared with at any time. Their installations will be deactivated, going into limited function mode.

Office 365 Personal is effectively the same as 365 Home, except that it limited to just you as the subscriber, no shares.

ken sheridan say:
You'll find an example of this sort of thing in InvoicePDF.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file a button on the invoice form sends the current invoice as a PDF file of a report to the customer's email address.  The code is as follows:

Private Sub cmdEmail_Click()

    Dim strTo As String
    Dim strSubject As String
    Dim strMessageText As String
    
    Me.Dirty = False
    
    strTo = Me.Email
    strSubject = "Invoice Number " & Me.InvoiceNumber
    strMessageText = Me.Customer.Column(1) & ":" & _
        vbNewLine & vbNewLine & _
        "Your latest invoice is attached." & _
        vbNewLine & vbNewLine & _
        "Customer Accounts Department, Widget Supply Company"

    DoCmd.SendObject ObjectType:=acSendReport, _
        ObjectName:="rptInvoice", _
        OutputFormat:=acFormatPDF, _
        To:=strTo, _
        Subject:=strSubject, _
        MessageText:=strMessageText, _
        EditMessage:=True
    
End Sub

The report is based on a query which references the invoice number in the form as a parameter in the query's WHERE clause:

.........WHERE Invoices.InvoiceNumber = [Forms]![frmInvoice]![InvoiceNumber];

This restricts the report to the current invoice.  Note that when sending a report as an attachment like this its recordset must be restricted by a parameter in this way.  There is no equivalent of the WhereCondition argument of the OpenReport method to filter the report.

ken sheridan say: You really do need to go back to basics on this and design a set of related tables, each of which models an entity type, e.g. Suppliers.  At the moment you are encoding data as table names; this is just not how a relational database works.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.

Take a look at the relationships window in my demo to get an idea of the sort of model a database like this needs.  The basic structure is that customers are modelled by one table, to which an invoices table is related.  The many-to-many relationship type between invoices and products is modelled by a table, InvoceDetails, which resolves the relationship type into two one-to-many relationship types.  Other tables modelling other entity type make up the rest of the model.

amit tandon say:

Refer below image:

Enter below formula in cell C2 & copy down - will rank equal sales as same (ie. 3 sales are ranked as 3) and next rank will be 6 (skips 4 & 5):

=RANK(B2,B$2:B$12)

Enter below formula in cell D2 & copy down - will rank equal sales as same (ie. 3 sales are ranked as 3) and next rank will be 4 (continuous):

=SUMPRODUCT((B2<B$2:B$12)/COUNTIF(B$2:B$12,B$2:B$12))+1

Enter below formula in cell E2 & copy down (distinct ranks for each sales even if same):

=RANK(B2,B$2:B$12)+COUNTIF(B$2:B2,B2)-1

Enter below formula in cell F2 & copy down (distinct rank for each sale where same sales become 3.1, 3.2 & 3.3):

=SUMPRODUCT((B2<B$2:B$12)/COUNTIF(B$2:B$12,B$2:B$12))+(COUNTIF(B$2:B2,B2)-(COUNTIF(B$2:B$12,B2)=1)*1)/10+1

The various options have been listed above. You may ideally need the rank in column F.
