jp ronse say:

Hi,

IFERROR(IF(O3=P3,"IAB","IBU"),"")

Quite normal that the formula returns "IAB" when O3 & P3 are empty because nothing is indeed equal to nothing.

Try this:

=IFERROR(IF(AND(LEN(O3),LEN(P3)),IF(O3=P3,"IAB","IBU"),""),"")

The LEN function will return 0 for empty cells what can be translated as FALSE. If both cells contain some input AND will retrun TRUE, next we can compare the contents.

As Sheeloo explained, the surroundig IFERROR will give the result of the formula or the given value in case of an error.

Note, the return value of the IFERROR in case of an error is not limited to a literal value, you can put also an alternative formula to handle the error.

>>And here is something odd, the current outlook pst file is 1,753 KB. When I made the back-up and said to include all the sub-folders, it was 606,873 KB.
>>
I would expect the export to be smaller by at least 20% but 1 GB is an unusually large difference. This difference is due to the way outlook handles deletions. When you delete an item, its removed from the index but not removed from the data file until its compacted. The export doesn't have the deleted data hanging around.

Outlook will compact when there is more than 20% of this wasted space... but only during idle time. If you don't leave outlook open when you aren't using it, it won't compact.

Your mailbox size is in the General section in Settings - the export should be close to this size (but not exact.)

To check the sync settings, go to File, Account Settings, open the account settings dialog. Double click on the account - the age of mail synced to the desktop will be listed there.

jcisla say:

Insert PDF as icon in Word (Office 365 for Mac, 2016)

Is this not possible? After Insert>Object, choose From File, click options and select Show As Icon, then select given pdf, no icon appears. Just the actual pdf file.
bob jones aka: cybertaz say:

Correct, it is not possible using Mac Office software. PDF is not a supported OLE file type despite the misleading behavior of the Insert> Object dialog. If Link to File is checked it will not enable opening the file from within the Word document, either.

Essentially, a PDF is inserted as a picture, and if there is more than one page in the PDF only the first page is displayed. The others are not accessible from the Word document.

How can I mailmerge from excel to word KEEPING THE EXCEL - Microsoft Community

karenmulders say:

How can I mailmerge from excel to word KEEPING THE EXCEL FONTFORMATTING (colors, underlining etc)?

How can I mailmerge from excel to word KEEPING THE EXCEL FONTFORMATTING (colors, underlining etc)? Thusfar I have been able to produce beautiful combined mailmerges but everytime I need to redo all the formatting from the excel sheet by hand in my generated worddocument. Is there a way (a fieldcode perhaps) to copy the data with the formatting from excel into word?

a. user say:

There isn't an "official" way to do it, unless you are willing to write VBA to get the data and format information and put it in the document.

If the same field in Word is always formatted the same way, it is probably simplest to format the field the way you want it (and if necessary, change a field code such as

{ MERGEFIELD myfield } or { MERGEFIELD myfield \* Mergeformat }

to

{ MERGEFIELD myfield \* Charformat }

)

If that is not possible, you may be able to retain some formatting as follows, but be warned: this uses a very old feature in Word that you should not rely on too much. I have listed some "gotchas" at the end.

1. Copy your data and column headings in Excel and paste the material into a Word document. Already you may lose some Excel formatting.

2. Save the document, and use that as your data source.

3. Use alt-F9 to reveal your merge fields. They will look something like

{ MERGEFIELD fieldname }

4. remove the text "MERGEFIELD".

5. Select all the fields and use F9 to update them.

6. Do the merge.

There are some "gotchas".

a. If your data has fewer than 64 columns, it should paste as a table. If it has more columns, you may need to deal with cells that contain multiline data, and cells that contain "delimiter" characters such as double quotation marks ", tabs and/or commas. Frankly, if you have that kind of data I think this method is probably more trouble than it is worth.

 b. when you update the fields, you may see an Error! Bookmark not defined! message. Clicking "Preview results" should show the formatted data.

 c. this probably won't work with special merge fields such as ADDRESSBLOCK or GREETINGLINE.
