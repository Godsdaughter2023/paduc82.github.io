Two answers already, I'll toss in some more. 

It depends on what you're after.  If you want a hyperlink that you can click to open an Excel file while in slide show view, trip_to_tokyo's first answer is the way to go.

I dd poke around and discovered, at least in my case, that the problem is in the link but in the text to the link.  I am not a programmer, but in the Edit Hyperlink dialog box in Access, there is a "Text to Display" and an "Address".  If you just paste into the hyperlink field, the Text to Display and Address are the same.  If you make the Text to Display different (for example, I just used "OneNote" and the text to display, but put the link in the Address field in the dialog box, it will work as it should.  

No why would the contents in the the "Text to Display" cause the link to malfunction when the link is in the Address field?  I don't know, but I have seen this behavior before and I think it relates to using the character "#" in the Text to Display field.  The OneNote link uses the "#"; so, if the link is used for the Text to Display, the hyperlink will not work.

I have done this twice and it works.  

In the process of tracking this down, I discovered another related item.  As many of you know, when you grab the link from OneNote that you want to use in other programs (Excel, Access, etc.), the memory captures two links -- the first link is to the SkyDrive iteration of the paragraph and the second is to the local copy iteration of OneNote.  I prefer to link in most cases to the local version and not the SkyDrive version.  So, in the past, I have pasted the link into notepad (a txt file) and deleted the first link and than used the onenote local link whenever I want to link to the local version.  That seems like a lot of extra and unnecessary work, but that is how I could solve the need to link to the local version rather than the SkyDrive version.

I found that there is another way to do the same think, but it too is clunky.  If you grab the link to the paragraph (right clicking) which, as noted above, puts both the SkyDrive link and the local version link into the clipboard, and then, where you want to create the link you use <ctrl-K> to create the link, merely doing <ctrl-V. in the Address field will post the SkyDrive link (the first link in the clipboard).  But, if you stay in OneNote and do <ctrl-v> in at an empty point, the program creates a link with (i) the name of the page as the Text to Display and the local version link as the link.  Then do <ctrl-K> and grab the address which is the local version address and use it whereever want to so long as the Text to Display does not contain a #.  

I hope this makes sense.  It makes sense to me.

It seems to me that the fix is for Microsoft to offer a choice that when the link to the paragraph is created, the user can default to either the SkyDrive llink of the local version link, so that however the person inserts the link in the hyperlink dialog, it can be the link the person has chosen as the default.

shirley douglass say:

Task to occur on 1st business day

I'm working in MS Project 2010 Standard and need to create a task that will occur on the first business day of a month after a given predecessor task completes. 

I tried creating a calendar for this, but there is not an option for "business day" only Monday, Tuesday, etc.

I also tried creating a recurring task but that works off of a "start no earlier than" constraint so as soon as I put in a predecessor the "1st of the month" part is thrown out of the equation.

Does anyone know how to do what I'm trying to do?

trevor rabey say:

Just to be sure of what you want, is it like this? The predecessor finishes on the 25th, and the task could then start as early as, but no earlier than, the 26th, but you want to force it to start on, or no earlier than, the first working day of the following month?

I would do it with a milestone (zero duration) in between the predecessor and your special task. Make a task calendar for the milestone which excludes all but the first working day of each month. You need to do this so that your special task can still have nonzero duration.

Any help?

shirley douglass say:

Trevor, yes, this s exactly what I'm trying to do.  My first thought was to create a calendar as you suggested, but I can't figure out how to set it to the first business day of the month.

tom_bpc say:

Shirley,

There are two issues: 1)Creation of the special Task Calendar, and 2)Application of the special Task Calendar.

1.  I think applying a calendar exception to "business day" would require defining one calendar as the function of another, which MSP doesn't support.  There may be other ways, but the only obvious way to create a "First Business Day of the Month" calendar is to start from the established project calendar (e.g. Standard) and custom-define a non-working exception for each month (running from the second working day through the end of the month.)

2.  I would do as Trevor suggests and apply the special calendar to an intermediate dummy task between the predecessor and your SpecialTask successor.   (This is necessary to allow your SpecialTask to continue to work on days that are not the first business day in the month.)  Unfortunately, MSP does not restrict milestones to their calendars, so your dummy task needs a positive duration.  An SS relationship is the cleanest way to avoid downstream complications.  (An FS with negative lag equal to the dummy's duration would be equivalent.)

Good luck, tom
