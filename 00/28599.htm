If so, you're seeing not a bug, but the designed Restore feature of OS X - Apple has decided that you like to reopen all the documents you were using before you quit.

One way:

Close all documents before quitting Word, or hold the OPT key down when quitting.

Alternatively:

To turn off Lion's Restore  feature globally, go to System Preferences/General, and uncheck the "Restore windows when quitting and re-opening apps" checkbox. This disables Restore for all applications.

To turn it off just for Word:

Find your ~/Library/Saved Application States/ folder for Word, empty it, then lock the folder. Note that Library is a hidden folder  - hold down the Option key while selecting Go in the Finder.

I don't really know exactly how Access determines which to load first, but I'd assume it has something to do with the manner and sequence in which those controls were added.

Check the Tab order of those controls to insure they're in "descending" order, with the 1st subform have a "lower" tab index than the second, and the second having a "lower" tab number than the third.

Me.SubformControl1.SourceObject = "subform1"

atifsiddiqui say:

I had the same problem and I solved it by removing the character 'Z' from the time format. The 'Z' character is used to specify the time zone and for some reason it doesn't read the zone correctly. Luckily in my case I had only one time zone so I took the 'Z' character off and it worked. Use the following format: 

DTSTART:{0:yyyyMMddTHHmmss

DTEND:{0:yyyyMMddTHHmmss

instead of 

DTSTART:{0:yyyyMMddTHHmmssZ

DTEND:{0:yyyyMMddTHHmmssZ
