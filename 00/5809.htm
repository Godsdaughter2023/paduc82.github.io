For your final question, yes, we can have all of your one day reports save to the Daily Reports folder on your desktop.  I have something that I just need to re-write that should work.

EDIT--Here is what I came up with.  Let me know if this is what you are looking for for this part.  This just saves to your Daily Reports folder on your desktop, renaming to the current date, to easy search, etc.  Make sure that change the SaveAs path to your desktop reports path.

[CODE]

Sub SaveToDesktop()
Application.ScreenUpdating = False
Application.DisplayAlerts = False
        With ActiveWorkbook
            .SaveAs "C:\Users\my name\Desktop\Daily Reports" & "\" & Format(Now, "mm-dd-yyyy") & " " & ActiveWorkbook.Name
            '.Close
        End With
Application.DisplayAlerts = True
Application.ScreenUpdating = True
MsgBox "Completed" 'To let you know when it is done
End Sub

Slow editing or review panel in Word for Mac when using track changes.

Hi everyone, 

Having searched through numerous community and support pages, it seems many people, myself included, have struggled to resolve a persistent problem with MS Word (from current versions back through to Word 2007 - as far back as I've looked), which is that it can slow down dramatically when you are working on a document that is using track changes and has extensive edits. I haven't seen any workable solution posted on any of these forums (though one may exist somewhere), but I've got a work around that at least solves the problem in terms of allowing the user to keep the features of track changes that are useful, but still have a functional piece of software.

Working from Word for Mac 16.16.4 (but the process appears to be the same for Word for Mac generally). Firstly, under your 'Review' tab, you'll see a "Markup Options" button beside the "Track Changes" On/Off switch. Click this and you'll see a drop down menu with "Comments", "Ink", "Insertions and Deletions", etc, with check marks beside them. Turn-off any options you don't need displayed (for example, I only want "Insertions and Deletions" and "Comments"). There are further options you can explore in the drop down that may work for you (e.g., only showing changes from specific authors if you have multiple people working on the document etc). I found simply turning off "Ink" and "Formatting" was enough for track changes and the review panel to work normally (from being hopelessly laggy), but that may depend on how old the computer is or how large the document is. To be clear, this changes the display of the tracked changes. They are still recorded and can be viewed by you or anyone else accessing the file is they change their mark-up options. 

If this still doesn't resolve the problem, you can also use the option button directly above "Markup Options" from "All Markup" to "Simple Markup", "No Markup" or "Original", which will speed things up as well. Again, this affects display of your changes, but so long as "Track Changes" is set to "ON", the changes are recorded and can be displayed again simply by changing your Markup settings back to how they were before. 

For reference, I am running a 2017 MacBook Pro with a 2.7Ghz i7 processor and 16Gb of RAM (i.e., a machine that should be more than fast enough to handle document editing in word). It seems obvious that Word simply doesn't handle tracked changes very well and hasn't for a number of years. This might not work for everyone and it doesn't fix issues with the functionality of this tool, but it works and at least it allows you the ability to track changes without slowing Word down to an unusable crawl.

Find and remove text in multiple documents in Word 2010 - Microsoft Community

iltzearen say:

Find and remove text in multiple documents in Word 2010

Situation: someone managed to use a kind of manual Track Changes, where he marked the original text that he deleted by highlighting and strikethrough. (I don't know why he did that, but that's just the way it happened.) Now I have to solve this by deleting all of this marked text. But he has done this to several documents, so I was wondering whether there is a way to do this automatically, perhaps with the help of Visual Basic for Applications. I'm using a computer with Office 2010 here.

In short: is it possible to delete text that is marked in a particular way (in this case: strikethrough and text highlight color) from multiple documents?

macropod say:

You could use a macro like:

Sub UpdateDocuments()
Application.ScreenUpdating = False
Dim strFolder As String, strFile As String, strDocNm As String, wdDoc As Document
strDocNm = ActiveDocument.FullName
strFolder = GetFolder
If strFolder = "" Then Exit Sub
strFile = Dir(strFolder & "\*.doc", vbNormal)
While strFile <> ""
  If strFolder & "\" & strFile <> strDocNm Then
    Set wdDoc = Documents.Open(FileName:=strFolder & "\" & strFile, AddToRecentFiles:=False, Visible:=False)
    With wdDoc
      With .Range.Find
        .ClearFormatting
        .Text = ""
        .Replacement.Text = ""
        .Forward = True
        .Wrap = wdFindContinue
        .Format = True
        .Font.StrikeThrough = True
        .MatchCase = False
        .MatchWholeWord = False
        .MatchWildcards = False
        .MatchSoundsLike = False
        .MatchAllWordForms = False
        .Execute Replace:=wdReplaceAll
      End With
      .Close SaveChanges:=True
    End With
  End If
  strFile = Dir()
Wend
Set wdDoc = Nothing
Application.ScreenUpdating = True
End Sub

Function GetFolder() As String
Dim oFolder As Object
GetFolder = ""
Set oFolder = CreateObject("Shell.Application").BrowseForFolder(0, "Choose a folder", 0)
If (Not oFolder Is Nothing) Then GetFolder = oFolder.Items.Item.Path
Set oFolder = Nothing
End Function
