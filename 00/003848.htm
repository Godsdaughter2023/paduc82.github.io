when I Make a accde File , the problem ( don'y save the moving fiels ) appear , again.
Please research it and give me a way for solving it.

 

Although it looks like the same problem, this time it is not a bug.  It is a principle of the MDE/ACCDE format that design changes cannot be made or saved, and changes to the column order in a datasheet are design changes.

 

However, there's a workaround.  You can save the column orders when the form is closed, and set them again from wherever you stored them when the form is opened.  For example, you can have a table named "ColumnOrders", with these fields:

 

    ColumnOrders
    -----------------
    FormName       : Text, PK (compound primary key)
    ControlName   : Text, PK (compound primary key)
    ColumnOrder  : Integer

 

Then you could have these functions defined in a standard module:

 

'------ start of code for standard module ------
Option Compare Database
Option Explicit

 

Sub SaveColumnOrders(frm As Form)

 

    ' Save the current column orders of the controls on the given
    ' datasheet form in table ColumnOrders.

 

    Dim ctl As Access.Control
   
    With CurrentDb
   
        .Execute "DELETE FROM ColumnOrders WHERE FormName = """ & frm.Name & """", dbFailOnError
       
        On Error Resume Next
        For Each ctl In frm.Controls
            .Execute _
                "INSERT INTO ColumnOrders(FormName, ControlName, ColumnOrder)" & _
                    " VALUES(""" & frm.Name & """, """ & ctl.Name & """, " & ctl.ColumnOrder & ")", _
                dbFailOnError
        Next ctl
       
    End With

 

End Sub

Sub LoadColumnOrders(frm As Form)

 

    ' Set the column orders of the controls on the given
    ' datasheet form from records in table ColumnOrders,
    ' if such records exist.

 

    Dim ctl As Access.Control
    Dim rs As DAO.Recordset
   
    Set rs = CurrentDb.OpenRecordset( _
        "SELECT * FROM ColumnOrders WHERE FormName = """ & frm.Name & """ ORDER BY ColumnOrder")
   
    With rs
   
        Do Until .EOF
            frm.Controls(!ControlName).ColumnOrder = !ColumnOrder
            .MoveNext
        Loop
   
    End With

 

End Sub
'------ end of code for standard module ------

 

Finally, you would call the functions from the form's Load and Unload events:

 

'------ start of code for form's module ------

Option Compare Database
Option Explicit

Private Sub Form_Load()


    LoadColumnOrders Me

 

End Sub

 

Private Sub Form_Unload(Cancel As Integer)

 

    SaveColumnOrders Me

 

End Sub
'------ end of code for form's module ------

 

This solution, by the way, should substitute for the workaround we worked out before, for the bug involving the form header/footer.  If you are going to save and restore the colummn positions this way, you don't care whether Access stores them automatically or not.
