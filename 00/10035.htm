andreas killer say:
The problem occurs when you open a document with an ActiveX combo box, click inside the combo box, type something and try to close the application (or the document).

The problem seems to be that a change in the combo box is recognized by application, but not by the Saved property. And also setting the property did not affect the saved state of the application in this context. This obvious a BUG.

The only workaround that I've found is the code below in the code module ThisDocument. The Document.Close method successfully suppresses the dialog.

Andreas.

Option Explicit

Dim WithEvents App As Application

Private Sub App_DocumentBeforeClose(ByVal Doc As Document, Cancel As Boolean)
  'Did the user saved our file?
  If Not ThisDocument.Saved Then
    'Close our file?
    If Doc.Name = ThisDocument.Name Then
      'Cancel the dialog always!
      Cancel = True
      'Set the save state
      ThisDocument.Saved = True
      'Close the document after this event
      Application.OnTime Now + TimeSerial(0, 0, 1), Me.CodeName & ".Document_AfterClose"
    End If
  End If
End Sub

Sub Document_AfterClose()
  'Close the document without saving
  ThisDocument.Close False
End Sub

Private Sub cboEquip_Change()
  'Let the document know that a change is made
  ThisDocument.Saved = False
End Sub

Private Sub Document_Open()
  Set App = Application
End Sub

rob koch say:

Is Edge already open to one or more websites in tab(s) in Edge when these unsafe websites pop-up?  Or are the pop-ups displaying when no browser of any sort appears to be open even in the background?

If even a single tab is already open with Edge, the pop-ups are likely being opened via that page or pages, possibly through an advertising network or exploits delivered from within them.  In this case you'd want to look into an ad blocker extension to the Edge browser, now that these are available with the addition of extensions in the Anniversary Edition.  I personally still use Internet Explorer for browsing and use the Tracking Protection built into the browser which also suppresses ads and so only see such pop-ups if I purposefully browse to less reputable websites.

If, however, no browser appears to be open when the pop-ups are displayed, you've more likely got some sort of malware present on the system already, though these might also be delivered via another application with vulnerabilities which are susceptible to attack via the Internet.  So if any other 3rd-party clients programs are open which include browser like capabilities that can access the Internet, look at whether these are known to contribute to such issues, though this seems less likely.

The previous suggestions were based upon the above, but didn't explain how these worked or why they might help.

The Malwarebytes suggestion might help, but in truth that is simply finding the incursions after they reach the machine and not solving the true problem, just like anti-virus software which is always after the fact.  Finding the core pathway that the pop-ups are using to enter the PC is the key and then either avoiding the website(s), applications or advertising networks aiding in their delivery.  If that includes malware, then that must be removed before looking for this initial pathway or it will continue to be re-introduced regardless of its temporary removal.

For reasons unknown, the default numbering for endnotes in Word 2003 is lowercase roman numerals. When you click Convert, the notes are converted (and display the i, ii, iii numbering), but the Footnote and Endnote dialog actually still shows "Footnotes" selected, with their 1, 2, 3 format. You must click Close to close the dialog, then go back to Insert | Reference | Footnote... (which will now have the Endnotes radio button selected), change the number format to 1, 2, 3, and, making sure that "Whole document" is selected for "Apply changes to," click Apply.
