sofspra say:

this is a letter a deleted a few years ago.

This is a pretty general request.  Without more details as to just what you are dealing with, it is hard to provide other than a general response.

The delete of a file on a hard drive does not typically delete the file from the hard drive.  The first move is to the Re-cycle Bin where it will remain subject to Restore.  When a file is "deleted" from the Re-cycle Bin,  it should put a  little flag on the file that this data space is now available for further use.   This available space could sit there for a long time before a request comes for use of this "space" for another file storage.

Now, many things can hasten this "space" to be re-purposed for another need.  If the PC is very active, there is more chance that new  data will use this space.  A hard drive can be re-formatted which wipes out all data or can be de-fragmented.   Over time as the hard drive fills up, a single file can be split into pieces to fit into available spaces.  The de-frag moves all data on the hard drive to the front of the drive and brings together all the pieces of a file for faster access.  This process will likely remove "deleted" files.  The De-frag process should be by request.

In the good old days, I recall doing this.  As I recall, a "deleted" file simply had a "1" placed at the front of the file name and could still be searched on the amended file name and just be re-stored.

In short, a "deleted" file is subject to being retrieved but under many circumstances where the file is subject to permanent loss.  Your local experience will dictate your level of success.

WMDC should sync in such a way that both Outlook and the phone contain exactly the same set of contacts.  When you add a contact to the phone, WMDC will pull the contact into Outlook and when you add a contact to Outlook, WMDC will push it to your phone.  Likewise, when you delete or change a contact on either the phone or in Outlook, WMDC should make the same change on the other.  I've occasionally seen contact duplicates, but I find it rare.  Calendar events are more of a problem for me.  I don't have a TouchPro2, however.  A search for this problem shows that it or issues similar to it are not uncommon on the TouchPro2.  I have no solution, only commiseration.

Hi,

As per the description you do not want the complete contents of the subscribed IMAP folder data to download in the Outlook program. I will help you with this.

 

You may change the settings to download only headers so that the complete data will not be downloaded from the server. Only when you open the email the contents will be downloaded and can be viewed.

  • Click on the Send/Receive tab, point to Send/Receive groups, and then click Define Send/Receive Groups.
  • Click a group that contains your IMAP account, and then click Edit.
  • Under Accounts, click your IMAP account.
  • In the list, select the check box for each folder that you want to include when checking e-mail, and then select Download headers only.
