Just a bit of an update here ...

I have come up with a way to display these web pages on a "per-record-navigation" basis. (i.e. as you navigate from record-to-record, the appropriate information is presented to the user.)

*IF* you are lucky enough to find a website that uses a common filepath and partnumber (Like eiko.com or cumminsfiltration.com does ...)

EIKO example:

http://www.eiko.com/Products.aspx?CatID=77&ProductIndex=1157

=IIf(Len([INDTextBox])>0,Concat([INDTextBox],"#http://www.eiko.com/Products.aspx?CatID=77&ProductIndex=",[INDTextBox],"#"),"")

Cummins Filtration example:

https://catalog.cumminsfiltration.com/catalog/partsearch.do?reqCmd=PartInformation&partNumber=FF5011

=IIf(Len([FleetguardTextBox])>0,Concat([FleetguardTextBox],"#https://catalog.cumminsfiltration.com/catalog/partsearch.do?reqCmd=PartInformation&partNumber=",[FleetguardTextBox],"#"),"")

Using one of the above strings as the Control Source of a hyperlink control on a web-view results in showing the current number (only if it isn't blank) as a hyperlink. Clicking on the hyperlink opens the web page.

IF you're *really* lucky and they use the same format for their images ... you can use much the same method to display an image in a web-browser control. (As was the case with Cummins Filtration.)

=IIf(Len([FleetguardTextBox])>2,Concat("https://catalog.cumminsfiltration.com/catalogImages/",[FleetguardTextBox],".JPG"),"")

I do realize that this won't fix everyone's issues ... but it did help in a couple of my apps, so I thought I might share it.

try this..
All file names in column A on Sheet1.

Sub FileNamesXML_ColumnA()
Dim WB As Workbook, ws As Worksheet
Dim myPath As String, myFile As String
Dim r As Long
Set ws = ThisWorkbook.Sheets("Sheet1")
myPath = "C:\FilesXML\" '<< change path
myFile = Dir(myPath & "*.xml")
r = 1
Application.ScreenUpdating = False
Do While myFile <> ""
Set WB = Workbooks.Open(myPath & myFile)
ws.Cells(r, 1) = WB.Name

myFile = Dir()
r = r + 1
Loop

End Sub

Difference between IRR and XIRR - Microsoft Community

my21 say:

Difference between IRR and XIRR

Hi can anyone enlighten me why there's a slight difference between IRR and XIRR when I stretch out the decimal places?

I have structured the dates such that there are no leap year effect (365 vs 366 days) that could cause difference between IRR and XIRR. 

Much appreciated! 

joeu2004 say:

Good of you to heed my suggestion to start a new discussion.  But the answer is the same....

We can only speculate about the difference.  IRR and XIRR are physically different implementations; infinitesimal differences can arise simply due to the order of arithmetic operations.  But the algorithms might also be very different.  Frankly, I'm surprised the difference is as small as it is.

Differences due to order of operations.  For example, (a+b)/c is often not the same as a/c + b/c.  Another example:  a-b+c is often not the same as a+c-b.  The anomalies arise due to limitations of 64-bit binary floating-point representation.

Differences in algorithms.  You might not know:  IRR usually cannot be calculated algebraically.  Instead, it is calculated by iterating over converging approximations.  There are several different algorithms; and different ways to implement the same algorithm.

    Hi

    Office 2016 apps do take longer to open than 2011 apps. There's more to them, and they are hobbled by Apple's "sandboxing" scheme. Still, more than a minute to open is not normal, even if you have a very low end Mac (I know, until a week ago that's what I had).

    Your observation is correct regarding "calling home." If you have a slow internet connection, that adds time to the launch process. Having a lot of fonts active adds time to the launch process. File system and cache troubles can add time to the launch process.

    Safe Mode Troubleshooting

    For the next troubleshooting step, I advise testing in Safe Mode. Restart your Mac and hold the Shift key down the moment you hear the startup chime. Once you see the swirling circle you can let go of the Shift key. Your Mac will automatically fix problems with your startup disk and reduce the available font set, and prevent other apps from launching at startup. It takes longer to boot to safe mode, so don't be alarmed if it seems like it's taking forever.

    Once you get to the desktop, open the problem app(s). Does it work OK (yes or no)?

    If yes, then maybe simply starting in Safe Mode fixed the problem. Do a regular restart and test again. Does the app work now (yes or no)?

    There are a lot of possibilities for troubleshooting based on the results of testing is Safe Mode. Please reply with the results of your testing.

    In safe mode, Word opens instantly with no discernible delay. Back in standard startup, it still takes 10 seconds for the application name to appear in the menu bar and 45 seconds to load completely and display a new document page.

Unless Word will launch rapidly when FEX (which is a startup item) is active, then the time necessary to quit Font Explorer, then Word, and then relaunch Font Explorer followed by Word will take just as long as the delayed launch of Word that takes place now when FEX is active at startup.
