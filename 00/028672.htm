Drop shadows use Microsoft transparency, not PostScript transparency. This technology was invented by Microsoft for display only. It does not print well, does not translate to PostScript transparency and looks particularly bad in PDFs. Even Microsoft admits it's bad: The transparent objects cannot be printed correctly in PowerPoint

When I create PowerPoint templates for clients, I never include objects that are made transparent in PowerPoint. You can use placed graphics that have been made using the Save for Web... feature in PhotoShop, or WMF/EMF graphics that are built outside Office with transparent areas, they print well to PDF.

per jessen say:

Hi Pete

As HansV says, you can not refer to a list in another workbook.

The one way to solve this is to build a list in a variable based on the content of the range in the master workbook. The only thing is that you have to update all validation lists if the list in MasterBook is changed!

Dim Master as Workbook
Dim ListRng As Range
Dim MyList As String

Set Master=Workbooks("MasterBook.xlsm")
Set ListRng = Master.Range("f1:f4")
For Each cell In ListRng
    MyList = MyList & cell.Value & ","
Next
MyList = Left(MyList, Len(MyList) - 1)
 With Selection.Validation
        .Delete
        .Add Type:=xlValidateList, AlertStyle:=xlValidAlertStop, Operator:= _
        xlBetween, Formula1:=MyList
        .IgnoreBlank = True
        .InCellDropdown = True
        .InputTitle = ""
        .ErrorTitle = ""
        .InputMessage = ""
        .ErrorMessage = ""
        .ShowInput = True
        .ShowError = True
    End With

john korchok say:

As thg250 has pointed out, the fonts are embedded if you use the export/save as option Best for printing when creating the PDF.

An embedded subset means that only the characters used in the document are embedded in the PDF. This only affects the editability of the PDF, not it's printability.

In a belt-and-suspenders approach (nothing succeeds like redundancy!), send the fonts along with the file, just like we used to do 10 years ago. If the printer is using the same operating system (Windows or Mac, doesn't even have to be the same version) and installs the font on their computer, then opens the PDF, all characters will render correctly, even if embedding did not occur.

You can choose to embed the full font if you buy a full copy of Acrobat and create PDFs with that. But I've only found that necessary for really odd jobs like creating PDF business cards from a web server.

If your print house can't figure this out, you need a smarter printer.
