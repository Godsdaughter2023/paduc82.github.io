The suggested solution does just what you want (look at the example screen shots). You use two different styles. You don't need a style separator because you're not trying to combine them in a single paragraph. You number the style you use for the chapter number. The chapter title style is not numbered. joeu2004 say: FSPH_FW wrote:

I want to enter the following formula into my Excel 2002 (SP3) following the syntax:
IF(logical_test, [value_if_true], [value_if_false])
=IF(((RC[-6]/100)*R84C2)<=RC[-2], [(RC[-6]/100)*R84C2], [RC[-2]])
For some reason (unknown to me, :=)) I get the error message "That name is not valid". The cursor points to the "angled bracket" just in front of value_if_true.   Would you have any idea what this means?


The use of square brackets in the help page is to indicate an optional parameter.  You cannot use the square brackets yourself around expressions.  You can only use square brackets where Excel permits them, for example RC[-2].  So your IF expression should be:

=IF((RC[-6]/100)*R84C2<=RC[-2], (RC[-6]/100)*R84C2, RC[-2])

PS:  Alternatively, you can write:

=MIN((RC[-6]/100)*R84C2, RC[-2])

PPS:  Regarding the error message "That name is not valid".  In general, we cannot take error messages literally.  Excel does its best to specify the nature of the error; but "its best" is often not good enough.  "Name is not valid" should be read as:  "I'm confused.  I don't know what you meant to write".

Documents that are created by Word 2007, 2010 or 2013 as .docx files  are in a completely different format to files created with .doc extensions either by versions of Word prior to 2007 or by saving them in Word 97-2003 format in a later version.


Changing the file extension from .doc to docx does NOT change the format of the file and, as you have discovered, it will render the document unable to be opened in Word.

rupertmenneer say:

RunDll32 Outlook Mail - Windows Host Process Error (SOLVED)

Hello All,

Recently I had a problem with one of my users laptops the laptop model was a Dell M3800 I believe. The problem related to Office 2013.

The issue was - When you create a new mailbox it crashes and gives your an error message or similar to "RunDLL32 WINDOWS HOST PROCESS ERROR".

After hours of trying to solve this issue, rebuilding the laptop, updating, fiddling with windows settings, running SFC /Scannow, etc, etc. I solved the issue! so i thought id share.... 

  1. Open Microsoft Word 2013
  2. Go to FIle, Options, Advanced
  3. Scroll down to "display"
  4. Check the box "Disable Hardware Graphics Acceleration

The problem arises when you have a dedicated graphics card on your computer. This option disables the use of the dedicated card and uses the integrated one instead (I have zero idea why office needs to use a graphics card for its applications since its 2d and runs the same without it).

This issue caused me pain! hopefully I can help others too as I hadn't seen any solutions to the problem like this one.

Rupert
