what ever happened to a simple merge cells? Can this be done simply?

john korchok say:

Each cell ends each with a formatting character that is the equivalent of a paragraph mark. These table cell characters are hidden by default, but they contain the formatting for the cell. When you merge cells, the formatting character of the rightmost merged cell assumes control over the cell. In this illustration, if the 3 cells of the bottom row are merged, the formatting character circled in red will assume formatting control:

Table can have styles applied to them which can be very different from your intention. Most users will handle this by locally formatting the cells to take on the preferred appearance. But when the cell contents are merged, the style can reassert itself over the cell formatting and the appearance can change drastically.

The solution is to select the table or at least the row you're editing and apply a style that has your preferred formatting. Then cell merging and splitting has no effect on the table appearance.

OK, this is how I would do this. 

Change the Rowsource of the combo to be specific. 

SELECT EquipID, EquipItem, EquipDayRate, EquipWeeklyRate, EquipMonthlyRate

FROM tblWBSCode

ORDER BY EquipItem;

Change these properties as well:

Bound Column: 1

Column count: 5

Column Widths: 0";1";0";0";0"

In the Rental Period combo's After Update event I would use Code like:

Dim intPeriod as Integer

If IsNull(Me.cboEquipItem) Then

    MsgBox "You need to select the equipment first!"

    Me.cboEquipItem.SetFocus

    Me.Undo

    Exit Sub

End If

Select Case Me.cboRentalPeriod

    Case "Days"

        intPeriod =2

    Case "Weeks"

        intPeriod = 3

    Case "Months"

        intPeriod = 4

End Select

Me.Cost = Me.cboEquipItem.Column(intPeriod)

This should populate the cost control with the proper cost amount.
