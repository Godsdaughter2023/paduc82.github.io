rob koch say:

Though as Edd mentioned the email might be fake, by your description it might also have been a valid notification from Outlook,com or the Microsoft Account website indicating a possible attempt to hack your account.

To confirm this without using anything within the email message, you can instead go directly to the following Microsoft.com Recent Activity web page which allows you to examine the recent activity for your account, including things like login attempts, IP addresses and other details to help you determine whether these were actually you or someone else.

What is the Recent activity page?

http://www.microsoft.com/en-us/Account/Security/recentactivity.aspx

If this link first takes you to the general Account page with things like recent purchases and your personal identity information, then click the Security & privacy selection on the menu bar near the page top.  You'll find a See my recent activity option listed beneath Account security on the left side of that screen.

You'll be prompted at least once to login along with other possible security verifications if those had been configured for your account, to insure it's actually you accessing this info.  At that point you should finally be viewing a see when and where you've used your account page.

There are options available to tag an IP address as "This wasn't me", but don't do that unless you are certain.  For example, I've had a rare false notification questioning my access from an IP address used by my phone network when my phone was trying to receive email messages from Outlook using POP3 protocol.

Read the full page at the link above before clicking the Recent activity button to fully understand what you can see and do.

There's also a Learn how to make your account more secure selection on the recent activity page which provides tips on security and additional items like two-step verification which you can enable to increase your protection from hacking.

1. Empty the recovery folder linked at the top of the Deleted items folder. (The recovery folder has a different, smaller quota, but emptying it can help speed up the processes.)
2. Sort the deleted items folder by size and delete the largest messages - select the first, scroll down and hold Shift as you select another message. Then hit Delete.
3. Empty the recovery folder again.
4. Click the Empty Folder button.
5. Go do something else for a couple of hours, more if you have a lot of deleted mail.

Some people prefer to repeat step 2 to clear the Deleted items folder. If you do this, do step 3 every few thousand messages.

        ◦ Archive items using AutoArchive.

If the email is stored on an email server such as Exchange, when the mailbox reaches its upper limit, the administrator may begin to limit the functionality.
