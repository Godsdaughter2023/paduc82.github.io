jim gordon mvp say:
Microsoft XML 6.0 is not available on the Mac, so you will have to get your information using a Web Query.

To create and run your own Web query, do the following:

 

1. Start a text editor, such as SimpleText.

 

2. Type the following lines in this document:

 

WEB

 

1.http://search.msn.com/results.asp?MT=seattle&CO=20&RS=CHECKED&UN=doc&Form=SMCB

 

3. Save your document in the Microsoft Office 98:Office: Queries folder.

 

4. Close your document.

 

5. Repeat Steps 1-6 from above, except in Step 5 select the Web query file you just created.

 

The third line of your sample Web query file could have been just about any URL, followed by a list of parameters. In this case, I used the following steps to generate the string used in the sample file:

 

1. In your browser, type http://search.msn.com in the address bar.

 

2. In the search the web for box type "Seattle", without quotes.

 

3. Click Search.

 

4. Copy the resultant string from the address bar in the browser.

 

5. Paste the string into the third line of your Web query file.

 

NOTE: The list of parameters that follow the URL of the Web site is completely of function of the script being used on the site.

 

Using Dynamic parameters in a Web query

 

In Web queries, you can use either Static or Dynamic parameters. Static parameters send query data without prompting you for any values, whereas Dynamic parameters prompt you to type one or more values when the Web query is executed.

 

To change your sample (Static) Web query into a Dynamic Web query, do the following:

 

1. Open your Web query file in a text editor, such as SimpleText.

 

2. Change the third line in this document to:

 

http://search.msn.com/results.asp?MT=["SearchString"]&CO=20&RS=CHECKED&UN=doc&Form=SMCB

 

3. Save and close your document

 

4. Run your Web query from a new worksheet in Excel.

 

5. Click OK in the Returning External Data to Microsoft Excel dialog box.

 

6. Enter any search string on which you want to search.

 

7. Click OK.

 

NOTE: The only change we made to the third line of the Web query file was to replace the hard-coded Static value (Seattle) with square brackets [ ] and a string enclosed in quotes. The string you use inside the brackets can be anything as it will be the text of the alert that appears when you run your query. The text you enter in the dialog that appears will be the parameter value that is used in the actual query.

rohn007 say:

"Compatibilty mode" is displayed when a file has a file format created in an older version of Word that does not support all of the features of the version of Word opening the file. These features unsupported by the file structure will be disabled. For example, on the Home tab / Font group / there is the fuzzy blue A button for the new 2010 Text Effects. This feature is not available in 2007 or 2003. If you open a file in compatibility mode, the blue 'A' is disabled, greyed out. It is really obvious if you open a DOC and look at a feature like SmartArt.

So DOC files opened in 2010 will always be in compatibility mode

DOCX files created in 2007 will always be in compatibility mode. The problem is just by looking at the name you can't tell the difference between a DOCX created in 2007 or 2010.

I'm not sure about the renaming and saving to temp folder ... Take a look in your macros list, use <ALT F8> to display macros. Look for macros with names starting in "auto..." or "open" or "save" or "saveas". The auto... macros will run automatically when certain actions take place. Macros with names identical to commands: ie open, save, saveas, will override the matching function.

The save as defaulting to 97-2003 DOC format is controlled at  File tab / Options command / Save option / Save Documents section / "Save Files in this Format:" yours is set to "Word 97-2003 DOC...". Change it to "Word Documentn DOCX).

bernd p. say:

As I thought: some of your notebooks have been moved the correct way to OneDrive, some not.

All the OneNote "files" without a file type (Anita's notebook, Art etc.) are correct notebooks. You see them all in OneNote Online and Android.

All the OneNote files with the file type .one (e.g. Gardening.one) are only sections (no notebooks) and have been moved the wrong way.

Solution:

  1. Identify what sections (.one files) belong together as a notebook (look for shortened names !)
  2. Create a folder on your computer with a name for the notebook
  3. Move these sections back from OneDrive to this folder
  4. Open this folder as a local notebook in OneNote 2016
  5. Use File --> Share  --> OneDrive to move the notebook the correct way to OneDrive
  6. The result should be one "file" with the notebook's name on OneDrive which now appears in OneNote Online

Save Trusted Templates to My Documents? - Microsoft Community

valley_girl1919 say:

Save Trusted Templates to My Documents?

 Hi,

 

My company uses Office 2007 SP3 and we have a few users that create templates for Word. I have noticed that when a template is created, it is saved in the following location: C:\Documents and Settings\user\Application Data\Microsoft\Templates. Is there a way to set the trusted templates to default to save into a different location? Like the users My Documents folder? We back up all the users My Documents and this would be good incase the users hard drive crashes.

jay freedman say:
In theory, there's no reason you couldn't do this, but there is at least one drawback.

Setting this up is a two-step process:

  • In the Options dialog, go to the Advanced page, scroll to the bottom, and click the File Locations button. In the next dialog, click on the "User templates" item in the list, click the Modify button, and choose the My Documents folder. Click OK to get back to the Options dialog.
  • Go to the Trust Center page, click the Trust Center Settings button, and go to the Trusted Locations page. Click the Add New Location button and select the My Documents folder. It's your choice whether to trust subfolders of it as well. Click OK to get back to the document window.


The drawback is that, when you go to Office button > New > My Templates, the dialog shows all the files in your My Documents folder, not just the templates. If that folder is loaded with tons of documents, finding the templates among them can be a chore. Two things can help:


  • If the templates must stay in the My Documents folder itself, switch to Details display and click the header of the Type column to sort on it. That will at least group all the templates together, although you'll probably have to scroll to see any of them.
  • If your backup of the My Documents folder includes its subfolders, just make a separate subfolder for templates only. Point the "User templates" location to that subfolder, and make that subfolder a trusted location, instead of the whole My Documents folder.
