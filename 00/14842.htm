How can i make a volatile function run "last" at recalculation - Microsoft Community

royiz say:

How can i make a volatile function run "last" at recalculation

Hi,

I have a volatile function which i cannot avoid being volatile for many reason,

I want this function to run (evaluate) last at recalculation, after all other cells have been calculated.

 

can this be done?

 

thanks,

rik_uk say:

You could try this code... Assuming the formula to calculate last is in a single cell, then name the cell MyVolFormula  and place the code in a standard module and assign a shortcut to it...

Sub CalcVolFuncLast()
Dim strVolFormula As String
    Application.ScreenUpdating = False
    Application.Calculation = xlCalculationManual
    strVolFormula = Range("MyVolFormula").Formula
    Range("MyVolFormula").ClearContents
    Application.Calculation = xlCalculationAutomatic
    Application.ScreenUpdating = True
    Range("MyVolFormula").Formula = strVolFormula
End Sub

Rik_UK Please mark the message or messages that answer your question as the "Answer" or vote if a reply has been helpful.

_charles_williams say:

The only safe way to handle this kind of problem is to make sure that all references to cells within your UDF are passed to the UDF as parameters, then your UDF will automagically be calculated after all the cells it refers to. (of course it may be calculated many times during the recalc, so if this is a problem you would need to check the cell references for uncalculated cells).

Rik's code will work if there are no other volatile functions in the workbook, and there are no formulae in the workbook that refer to the cell containing the UDF, and there is only one call to the UDF.

For more information about how to check for uncalculated cells and other calculation sequence UDF problems see

http://www.decisionmodels.com/calcsecretsj.htm

 

andreas killer say:

Am 28.10.2010 12:21, schrieb RoyiZ:

I have a volatile function which i cannot avoid being volatile for many reason,
I want this function to run (evaluate) last at recalculation, after all other cells have been calculated.

AFAIK calculates excel the formulas first, then volatile functions, then UDF's and at last calls the Worksheet_Calculate event.

I'm not sure if there are differences between the excel versions, but in fact the event is called at last in all versions.

If you want to be sure that your calculation is done after all, you have to remove the volatile function and place the code inside the Worksheet_Calculate event.

But be warned, if you write in a cells (or the cell which holds your volatile function) is used by a formula, you force excel to calculate the sheet again. That's why we can set some options inside excel for calculating sheets, i.E.: Application.MaxIterations

In this situation you can not be sure when the calculation stops and which cell is calculated at last.
