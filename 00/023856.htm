jeeped say:

The CAPs LOCK key is exactly underneath the TAB key on most keyboards. The TAB key's designed purpuse is to move the active cell 'forward' as the programmer has designed the form or function. Perhaps the two have become 'stuck' through wear, tear or neglect. However, to properly evaluate the issue:

  1. Does this behaviour repeat itself when using the on-screen keyboard (Start, <type> on-screen keyboard<enter>)?
  2. If not, is this a wireless keyboard? Is it outside of 6-8 feet of the USB receiver's line-of-sight when this occurs? If it is, perhaps this is the problem.
  3. If not, is this a 64bit Windows 7? If it is, get the latest drivers from the keyboard's manufacturer for the 64bit operating system.
  4. If not or it still ocurs, have you tried another keyboard? Even laptops can handle an external keyboard through USB. 
  5. And finally: how long has it been since it been dusted, how many times has it been dropped, how much coke/coffee/sneezes has it absorbed?

BTW, SHIFT+TAB is designed to move the active 'cell' backwards.

scott a. helmers say:

There doesn't appear to be a way to attach files here, so feel free to send an email to me at *** Email address is removed for privacy *** with a subject line like "Reformat org chart".

The file I will send is a macro-enabled Visio template (Visio 2013/2016) that contains code to reformat the org chart after you've created it. You will need to enable macros when you create a new document from this template.

When you're ready to run the macro, you can do so by clicking the Macros button in the Macros section on the View tab, selecting the macro, and then clicking the Run button. You can also assign a keyboard shortcut to the macro by using the Options button in the Macros dialog box.

The current code changes all shapes to the "Shapetacular" style, applies the "Center" layout in the "Horizontal" section, and the hides photos. To use a different style or a different layout, you need put on your geek hat and start by turning on a trace feature in the org chart wizard so it will tell you the name of the style and layout you want. If you aren't familiar with how to do any of the steps below, let me know:

  1. Open the Visual Basic Editor window (Alt+F11)
  2. Open the Immediate window if it's not already open in the VB Editor (Ctrl+G).
  3. Type this in the Immediate window and then press Enter:
     Application.TraceFlags = visTraceAddonInvokes
  4. Return to the Visio drawing window.
  5. Apply the style you want.
  6. Apply the layout you want.
  7. Replace the style name and layout name in the lines marked in the code.
  8. Turn off the org chart trace feature by typing this i the Immediate window and pressing Enter:
    Visio.Application.TraceFlags = False

As a final note: it is possible to automate the creation of the org chart. If you want the additional code that prompts for the name of the Excel file containing your data and then creates the chart, let me know.

Selecting Current Page Range - Microsoft Community

l.a. lawyer say:

Selecting Current Page Range

I want to do some work (primarily replacing text) on the current physical page.  I presume that I will need to extract and then replace the current page's range.

How is this done?

TIA

doug robbins - mvp office apps & services (word) say:

Using VBA,

Dim rngPage as Range

Set rngPage = ActiveDocument.Bookmarks("\page").Range

will set a rngPage to the Range of the page on which the Selection is located.

Then you could use

With rngPage

    .Text = Replace(.Text, "old text to be replaced", "new text to be used")

End With
