I personally I would use a VBA userform for this - http://www.gmayor.com/Userform.htm  The form (which you can download from http://www.gmayor.com/Forum/FormEmail.zip )  could look like

the code it contains could be

Option Explicit

Private Sub CommandButton2_Click()
    If Me.ComboBox1.ListIndex < 1 Then
        Me.ComboBox1.ListIndex = 0
        MsgBox "Select Contact"
        Me.ComboBox1.SetFocus
        GoTo lbl_Exit
    End If
    If Me.TextBox1.Text = "" Then
        MsgBox "Enter Custom #"
        Me.TextBox1.SetFocus
        GoTo lbl_Exit
    End If
    Me.Tag = 1
    Me.Hide
lbl_Exit:
    Exit Sub
End Sub

Private Sub CommandButton1_Click()
    Me.Tag = 0
    Me.Hide
lbl_Exit:
    Exit Sub
End Sub

and it would be driven from a macro called from a button on the ribbon or QAT (Quick Access Toolbar) with the code:

Option Explicit

Sub NewMessage()
Dim olItem As Outlook.MailItem
Dim olInsp As Outlook.Inspector
Dim wdDoc As Object
Dim oRng As Object
Dim oFrm As New frmEmail

    With oFrm
        With .ComboBox1
            .AddItem "[Select Contact]"
            .AddItem "Jim"
            .AddItem "Leticia"
            .AddItem "Mike"
            .AddItem "Steve"
            .ListIndex = 0
        End With
        .Show
        If .Tag = 0 Then GoTo lbl_Exit
        Set olItem = CreateItem(olMailItem)
        With olItem
            .BodyFormat = olFormatHTML
            Set olInsp = .GetInspector
            Set wdDoc = olInsp.WordEditor
            Set oRng = wdDoc.Range(0, 0)
            oRng.Text = "Going forward, " & oFrm.ComboBox1.Text & _
                        " is our contact for this " & _
                        oFrm.TextBox1.Text & _
                        ". The work schedule includes: "
            oRng.collapse 0
            oRng.Select
            .Display
        End With
    End With
lbl_Exit:
    Unload oFrm
    Set oFrm = Nothing
    Set olItem = Nothing
    Set olInsp = Nothing
    Set wdDoc = Nothing
    Set oRng = Nothing
    Exit Sub
End Sub

    irish beast say:

    Update: It works again.

     

     All you need to do is go to your Charms Bar, click the search option and under all apps type "regedit" (this will open in desktop because it isn't a metro app).  you want to search through these subfolders: 


    [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Audio]


    find


    "DisableProtectedAudioDG"=dword:00000001


    Double click the words and change the value that should be highlighted when the pop-up appears and change it to 0. After that, reboot your PC and try playing music again.

    juankaos say: i had this same problem, i did not have the "DisableProtectedAudioDG"=dword:00000000 entry in [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Audio]

    so what i did was just add the entry myself and it worked.

    how to do it:

    go to [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Audio]

    right click -> New -> DWORD (32-bit) Value

    name it DisableProtectedAudioDG

    double click on the name

    it should already have a 0 in the Value Data: field if not change it to 0

    OK

    reboot

    this worked for me, good luck!
