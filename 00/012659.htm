Corrupted Word 2007 file - Microsoft Community

stressedgradstudent say:

Corrupted Word 2007 file

Got a corrupted file in Word 2007 - error details state that "An attribute value must not contain '<'.  Location: Part: /word/document.xml, Line: 2, Column: 1588862"

What can I do to recover this file?  Thanks for any help you can provide!
rohn007 say:
Fixing boils down to "decompressing" the file, then finding the location indicated in the error message  "Line: 2, Column: 1,588,862"  .   The error message says that 1,588,862 characters into "line 2"  Word finally encountered errors it could not get past.   So, what you have to do is look at the XML codes at that location, then work "backward" toward the start of the file to find mismatched XML codes that would affect this one. 

Fixing the problem is basically puzzle work.  You have to puzzle out which matching XML code is missing.

Manual Fix

Notepad ++ is a good, free editor for this type of task

·         Make a copy of the file

·         Rename the copy from DOCX to ZIP

·         Open … .ZIP/word/document.xml in notepad

·         Copy the contents of the file to clipboard

·         Open Word

·         Paste a copy of the copied XML into Word

·         (optional) the XML is one long string too hard to read, you can replace some tags, with that tag plus a para mark to break up the text to make it more people readable.

·         Open an XML validator, ie this site on the internet: http://www.w3schools.com/xml/xml_validator.asp

·         Paste another copy of the XML into the “Syntax Check Your XML” input window

·         Click on “validate” button

·         Copy the missing tag, ie </mc:Fallback>  (yours will be different)

·         Return to word Find: mc:Fallback>  (without the </ so you find both open and closing tags).  Repeat find until you hit 2 open tags in a row.  Then you just have to figure out where to put the closing tag between them.  Look for other tags before and after a proper closing tag so you can match the problem area to a good area.

1. PowerPoint for Mac manages right to left Hebrew correctly. I paste the text into a PowerPoint title box (text box may work?) and format accordingly (e.g., font, size).

2. Paste the title box into Word 2011 normally, but choose from the little drop down menu that appears to "Keep Source Formatting".

It can take more time than you'd ideally want, but this is a self-contained way to work within the constraints of Office 2011.

wangyanyu say:

hi jeff,

to allow other users to view a document without downloading it, you can refer to the following steps to remove the open item permission from the read permission level:

1. sign into your site. click the options icon > site settings > site permissions.
2. click permission levels and click on the read permission level.
3. scroll down to the button of the page and click copy permission level.
4. name it for example “read without download”
5. untick the open items permission and click create.
6. share the document with the user and allow him/her to view.


    
7. click share > shared with > advanced > select the user you shared with > edit user permission > untick the read permission and tick read without download permission > ok. then, the user have read without download permission as below:
 


 
in addition, the download option still exists in the new interface of onedrive for business site. however, the user can’t download it until you approve the request as below:

1. go to sharepoint admin center > user profiles > people > manage user profiles.
2. type the user into the find profiles box > find > manage site collection owners

3. add your account into site collection administrators > ok.
4. then you can help manage his/her onedrive for business site or library.

steve rindsberg say:

Keep in mind that PowerPoint's paper sizes are never the same as real-world paper sizes.  For example, if you choose A4 in PPT's Slide Size dialog box, you get 10.833 x 7.5 inches, which isn't proportional to true A4.

Try creating a test presentation with a custom size of 10.572 x 7.5 inches.  See if that gives you a more accurate result when saved as PDF.

If so, you can either change A COPY of your current presentation to that size or, if that causes problems, start a new pressie at that size, then copy/paste the contents of your original presentation onto new slides in the correctly sized presentation.

The fact that both are 7.5 inches high will eliminate most distortion, though you may need to make a few further adjustments.
