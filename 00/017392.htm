ken sheridan say: As you are using a multi-valued field my guess would be that the query would be:

SELECT *
FROM People
WHERE NOT EXISTS
    (SELECT *
     FROM Classes
     WHERE Classes.PeopleID.Value = People.ID

However as I have never had occasion to use a multi-valued field I'm not entirely sure of the syntax.

Do you really need to uses the multi-valued field PeopleID, i.e. is the database interfacing with SharePoint in the limited context in which this feature is required?   If not I'd recommend modelling the many-to-many relationship type conventionally by means of a table:

People
....PersonID (PK)
....FName
....LName

Classes
....ClassID (PK)
....Class_Name
....Class_Abbrev

and to model the relationship type:

People_Classes
....PersonID (FK)
....ClassID (FK)

The primary key of the last table is a composite one of both columns.  The table can of course also include non-key columns e.g. Enrolment_Date etc representing other attributes of a person's enrolment in a class, something which a multi-valued field does not allow.

The query would then be:

SELECT *
FROM People
WHERE NOT EXISTS
    (SELECT *
      FROM People_Classes INNER JOIN Classes
      ON People_Classes.ClassID = Classes.ClassID
      WHERE People_Classes.PersonID = People.PersonID
      AND Class_Name = "X");

In the context of this model I would recommend against using the NOT IN operator for this as in the event of Class X not having any students enrolled (a theoretical if unlikely possibility) it would fail, as NOT IN NULL evaluates to NULL, neither TRUE nor FALSE.  In any case the EXISTS or NOT EXISTS predicates are usually faster as they allow the optimizer full rein.

van dinh say:

You need to use a Frustrated Outer Join with a subQuery in this case

 

The SQL should be something like:

 

 

SELECT Pepople.Lname, People.Fname
FROM People LEFT JOIN

  (

    SELECT Classes.PeopleID.value

    FROM Classes
    WHERE Classes.Class_Name = "x"

  ) AS SubQ
ON People.ID = SubQ.PeopleID.value

WHERE SubQ.PeopleID.Value Is Null

 

 

The SubQ selects people who attended Class "X".  The main Query (the "Frustrated Outer Join") selects the difference between the set of all people (Table [People]) and the set of people who attended Class "X" (the SubQ) which gives you the set of people who have not attended Class "X" according to your dataset.
