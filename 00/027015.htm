doug robbins - mvp office apps & services (word) say:

The error message "Word could not create the work file. Check the temp environment variable". Can be caused by incorrect registry values in the key:

HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders.

Click Start and in the start search field, type

Regedit

and press enter.

In the registry editor, browse to:

HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders

On the right hand pane, check the value data for Cache.

It has to be %USERPROFILE%\AppData\Local\Microsoft\Windows\Temporary Internet Files.

If it is different, double click on Cache and type:

%USERPROFILE%\AppData\Local\Microsoft\Windows\Temporary Internet Files

under value data and click ok. Close the registry and  restart the computer and check if it works.

You are right, MSN contacts won't sync with Outlook.

 

This is what you can do:

 

Access your MSN account through your web browser on your computer.

 

Log on and open Contacts.

 

Click Options on the right of the screen, under your username.

 

Select Export Contacts.

 

Click Export Contacts button and click Save.

 

On your computer, open Outlook and contacts.

 

Click File > Import and Export.

 

Select Import from another program or file. Click Next.

 

Choose Comma Seperated Values (Windows)

 

Click Next.

 

Browse to the file to Import. Click Next

 

Select the destination folder and click Next.

 

Click Finish.

 

This is it.

robert long [msft] say: Hi Szymon_M - I'm one of the Test Leads on Excel Web App team.  Yes there is a chance to add this feature but sorry it is not currently supported as you point out.  We do listen to all the feedback very closely and its really important to us to help us plan and prioritize our work.  I want this feature too! So thanks for sending us this, it helps.  

if you want to know more, I added some background here...This feature is an example that there are some features (‘unsupported features’) that can block an Excel file from being viewed, or edited, in Excel Web App and in Excel for Mobile phone. These are inherent limitations of these products.  We are working to remove many of these blocking issues..it’s a priority for our teams. But it’s important to know these limitations do exist currently for some features.  You won’t unfortunately be able to view or edit 100% of all features in the web app (or phone).   

·         For help understanding issues with unsupported features in Excel for Mobile phone:
To know what features may cause your workbook to not be editable in Excel for Windows Phone, please refer to this support article.  Look for the link there that says “Why can’t I edit some Microsoft Office documents on my phone.”  The info there helps identify what features are not supported in the phone and which may cause you to see the “Can’t Edit Workbook” message at the top of the view when viewing on the phone.  

·         For help understanding issues with unsupported features in Excel Web App:
To understand the story around unsupported features that may block viewing or editing in the Excel Web App, please refer to this support article.  Some features such as Sheet Protection can block the file from being opened at all in the browser.  Some features only block editing of the file in the browser. 

Once you understand what features are blocking,  you can then consider whether it is an option for you to remove these features from your workbook.
