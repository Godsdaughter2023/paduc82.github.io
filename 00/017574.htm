When Microsoft introduced the new look with Outlook.com, a few changes were made to make the experience a whole lot smoother. I’ve outlined the changes below:

 

Public publishing/sharing of Calendars  

When accessed, you will see a message in the original shared Calendar that the Public sharing function is no longer supported.

 You can still privately share Calendars on a one on one basis by sending a link or giving permissions. Here’s how:

Note: People you share your Calendar with must have a Microsoft account.

Here’s how to share with people you choose:

  1. In Calendar, click Share, and then click the name of the calendar you want to share.
  2. Click Share with people you choose.
  3. Click the To: box and enter the email address for someone you want to share with.
  4. Repeat Step 3 to add any other people with whom you want to share the same level of access.
  5. Select the level of access from the dropdown.
  6. Click Share.

 

Get a link

You can give anyone a link that will give them view-only access to your calendar—and they can pass it on to others. People you give a link to can access your calendar via other calendar apps, like Microsoft Outlook, Google Calendar, Mozilla Lightning, or iCal.

 

You can choose from two access levels: Show event details, and Show free/busy time. You can also have one set of links for each access level.

 

Here’s how to get a link:

 

  1. In Calendar, click Share, and then click the name of the calendar you want to share.
  2. Click Get a link.
  3. Choose Show event details or Show free/busy time. Click Create under the type you want.
  4. Email the link to those that need it.

 

Print

The Calendar service no longer offers a Print icon/function.

You’ll need to use the Print function available in your web browser.

 

Using Mobile devices

If you’re using a mobile device to access Outlook.com, you should either use the mail app already on your device or install the app appropriate for your device. You can find device specific apps at: Outlook tips to try

greginmich say:

We now suspect that Windows Defender is oftentimes not getting properly disabled in the Windows Defender registry key when a third-party AV app is installed. The presence of the issue can easily be determined with this registry query (right-click on the Start button, select Command Prompt, and then copy, paste, and enter this command):

reg query "hklm\software\microsoft\windows defender" /v DisableAntispyware

If you get a return value of 0x0, then Windows Defender is enabled in the Windows Defender key when it should be disabled (0x1). We know that this problem is sometimes correctable simply by uninstalling the third-party AV app, running its removal tool, and then reinstalling with the latest version of the app – which presumably succeeds by setting the proper values for the DisableAntiSpyware and DisableAntiVirus DWORDs in the Windows Defender key.

We also know that the problem can often be ostensibly “fixed” just by disabling Windows Defender in the Policies key with the Ten Forums reg file. This seems to successfully turn off the Windows Defender UI by overriding the bad entries in the Windows Defender key – but it obviously leaves those faulty entries in place, and it also appears to leave Error 577 unresolved (on attempting to start the Windows Defender Service in services.msc).

So When we combine the uninstall, cleanup, and reinstall method with the Ten Forums edit to the Policies key, the issue gets confounded – because we don’t know whether the entries in the Defender key have actually been corrected by the combination fix, or if they’ve just been overridden by adding the DisableAntiSpyware entry to the Policies key and setting its value to 1. So I’m not suggesting that that people refrain from using the combination fix, I’m merely suggesting that people who do use this fix should rerun the registry query afterwards in order to discern whether or not the issue with the entries in the Windows Defender key has actually been corrected – or is just being masked by the entry in the Policies key that was added by the Turn_Off_Windows_Defender.reg file.  

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-defender-error-code-0x8007139f/93096739-16ce-471b-8716-1776f0c69f89?page=2
