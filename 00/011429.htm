Is there a shortcut for advancing to another text box within a Word - Microsoft Community

frankwurtz say:

Is there a shortcut for advancing to another text box within a Word 2007 document

Original Title:  Word 2007 textbox
Is there a shortcut for advancing to another text box within a Word 2007 document
jay freedman say:
In the general case -- a document with multiple unlinked text boxes -- there is nothing, neither shortcut nor button, to go to the "next" text box. At least part of the problem is that there's no reliable information in this case about which box is "next". The closest approximation would be the order in which the boxes were inserted. It might be possible to go through them in the order of location of their anchors, but the positions of the boxes aren't necessarily related to the positions of the anchors. If that doesn't bother you, then read the macro solution at the end of this post.

There is one specific case where it is possible to use built-in shortcuts: If you Previous Text Box buttons on the Quick Access Toolbar or assign keyboard shortcuts to the GoToNextLinkedTextBox and GoToPreviousLinkedTextBox commands (these are in the All Commands category of their respective Customize dialogs). If the text boxes aren't linked, the buttons and shortcuts don't work.

The following macros, installed according to http://www.gmayor.com/installing_macro.htm, can be assigned to Quick Access Toolbar buttons or keyboard shortcuts. They will move forward and back through the text boxes in a header, footer, footnote, or endnote.

Sub GoToNextTextBox()
    Dim testBoxIx As Long
    Dim thisBoxIx As Long
    Dim otherBoxIx As Long
   
    With ActiveDocument.Shapes
        ' determine whether selection is in a text box
        ' and if so, which one
        For testBoxIx = 1 To .Count
            If .Item(testBoxIx).Type = msoTextBox Then
                If Selection.Range.InRange( _
                        .Item(testBoxIx).TextFrame.TextRange) Then
                    thisBoxIx = testBoxIx
                    Exit For
                End If
            End If
        Next testBoxIx
       
        If thisBoxIx = 0 Then
            MsgBox "Put the cursor in a text box and rerun macro"
            Exit Sub
        End If
       
        ' find the next shape that is a text box
        ' and go to it
       
        For testBoxIx = thisBoxIx + 1 To .Count
            If .Item(testBoxIx).Type = msoTextBox Then
                otherBoxIx = testBoxIx
                Exit For
            End If
        Next testBoxIx
       
        If otherBoxIx > thisBoxIx Then
            .Item(otherBoxIx).TextFrame.TextRange.Select
            Selection.Collapse wdCollapseStart
        End If
    End With
End Sub

Sub GoToPreviousTextBox()
    Dim testBoxIx As Long
    Dim thisBoxIx As Long
    Dim otherBoxIx As Long
   
    With ActiveDocument.Shapes
        ' determine whether selection is in a text box
        ' and if so, which one
        For testBoxIx = 1 To .Count
            If .Item(testBoxIx).Type = msoTextBox Then
                If Selection.Range.InRange( _
                        .Item(testBoxIx).TextFrame.TextRange) Then
                    thisBoxIx = testBoxIx
                    Exit For
                End If
            End If
        Next testBoxIx
       
        If thisBoxIx = 0 Then
            MsgBox "Put the cursor in a text box and rerun macro"
            Exit Sub
        End If
       
        ' find the next shape that is a text box
        ' and go to it
       
        For testBoxIx = thisBoxIx - 1 To 1 Step -1
            If .Item(testBoxIx).Type = msoTextBox Then
                otherBoxIx = testBoxIx
                Exit For
            End If
        Next testBoxIx
       
        If (otherBoxIx > 0) And (otherBoxIx < thisBoxIx) Then
            .Item(otherBoxIx).TextFrame.TextRange.Select
            Selection.Collapse wdCollapseStart
        End If
    End With
End Sub

Internet Explorer runs in 32-bit mode unless you check either of these boxes, which then makes it work in Enhanced Protected Mode, or 64-bit.  I suspect these boxes are NOT checked on your computer by default, which means it is running in 32-bit and your application should run fine.  If either are checked though, uncheck them, click APPLY, click OK and restart your computer

bernie deitrick say:

You could try this code in the Thisworkbook's code module - if they save it as an xlsx and then try to close it, the code will resave the file and remove the .xlsx

Private Sub Workbook_BeforeClose(Cancel As Boolean)

    Dim sttemp As String

    If Right(Me.Name, 4) <> "xlsb" Then

        strtemp = Me.FullName

        Me.SaveAs Left(Me.FullName, InStrRev(Me.FullName, ".")) & "xlsb", 50

        Kill strtemp

    End If

End Sub
