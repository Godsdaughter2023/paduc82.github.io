scottgem say:
Peter,

There are 2 ways to update data in tables. One is through SQL statements, the other through DAO/ADO. Both have their purposes. With DAO/ADO, you open a recordset and loop through that recordset one record at a time. You then perform your operation on each record and move to the next record. This is the second example mimmo gave you. 

However, you can also use SQL statements embedded in your VBA code. So you could use:

Dim strSQL As String

strSQL = "UPDATE table SET NewGender = IIf([Gender]="Male",1,2);"

CurrentDB.Execute strSQL, dbFailOnError

Unless there are a variety of conditions to test for, a loop would not be my first choice. 

On a separate point. I'm assuming this was either an example, or you are converting a field that holds text, into a numerical code. So I would suspect this is a one time thing. In your original post you stated you wanted; "to populate a column in an access table based on the values of another column." However, this is a violation of normalization rules. A column should only be dependent on the record's primary key. If you need to DISPLAY another value based on the value of a column, you would do that in a query, not store the data in a table.

Hi,

 

this is an Vba code example :

 

Sub AAA()
    Dim myDB As Database
    Dim myRst As Recordset
   
    Set myDB = CurrentDb
    Set myRst = myDB.OpenRecordset("tableName")
   
    Do While Not myRst.EOF
       myRst.Edit
       myRst("NewGender") = IIf(myRst("[Gender]") = "Males", 1, 2)
       myRst.Update
       myRst.MoveNext
    Loop
   
    myRst.Close
    myDB.Close
    Set myRst = Nothing
    Set myDB = Nothing
End Sub

 

 

If you prefer you can use next statments instead of this statement:

 

myRst("NewGender") = IIf(myRst("[Gender]") = "Males", 1, 2)

 

If myRst("[Gender]") ="Males" Then

   myRst("NewGender")=1

Else

   myRst("NewGender")=2

andreas killer say:

The situation hasn't changed.

I can not quite agree.

The name of the current "code block" (sub/function/property) is not readable with a VBA code, correct.

But the VBA debugger has all informations that we need:

To reproduce that, copy the code below into a regular module, run Sub Main, enter Debug mode and press CTRL-L

Andreas.

Option Explicit

Sub Main()
  On Error GoTo Errorhandler
  LevelA
 
  Exit Sub
Errorhandler:
  If Err.Source = "" Then Err.Source = Application.Name
  Debug.Print "Source     : " & Err.Source
  Debug.Print "Error      : " & Err.Number
  Debug.Print "Description: " & Err.Description
  If MsgBox("Error " & Err.Number & ": " & vbNewLine & vbNewLine & _
      Err.Description & vbNewLine & vbNewLine & _
      "Enter debug mode?", vbOKCancel + vbDefaultButton2, Err.Source) = vbOK Then
    Stop 'Press F8 twice
    Resume
  End If
End Sub

Sub LevelA()
  LevelB
End Sub

Sub LevelB()
  LevelC
End Sub

Sub LevelC()
  LevelD
End Sub

Sub LevelD()
  Main
End Sub
