As a short term solution until you figure out what you want to do I put together some steps that might help workaround getting your account configured in Outlook using Autodiscover.  Give these a try and see if they work for you.

Temporary workaround to get Outlook to connect via Autodiscover for a custom domain

 

1. Open a Command Prompt and type the following and press Enter:

 

ping autodiscover.outlook.com

or

ping autodiscover.outlook.com -4

 

Note:  If you are on an IPv6 network add a -4 at the end of the ping to get the IPv4 address.

 

You will see a result similar to the following depending on your Geo Location:

 

Pinging autodiscover-namnorthwest2.outlook.com [132.245.71.8]

 

2. Open your Windows Hosts file in Notepad e.g. right click on the file and click Open With and pick Notepad.  The Hosts file will be in the system directory path such as:

 

C:\Windows\System32\drivers\etc

 

3. In the Hosts file add an Autodiscover entry for your custom domain pointing to the IP Address you got in Step 1.  It will look like the following.  Save the Hosts file after you add it.

 

 

# localhost name resolution is handled within DNS itself.

#             127.0.0.1       localhost

#             ::1             localhost

 

autodiscover.exampledomain.com   132.245.71.8

 

 

4.  With the entry added above try adding your account in Outlook by adding just your Name, Email, and Password.  For instructions adding your account back in Outlook refer to this topic, https://support.office.com/en-us/article/Reconnect-Outlook-2016-or-Outlook-2013-to-Outlook-com-for-uninterrupted-email-access-cda1751d-9503-40bf-bf76-e79454ac5eb3?ui=en-US&rs=en-US&ad=US.

 

Note:  The Office 365 IP Addresses can change over time.  Save these steps to refer back later if you run into connectivity issues or if you need to reconnect your account again.
