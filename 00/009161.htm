hansv mvp say:

Create the following macros in your personal macro workbook Personal.xlsb:

Sub MoveDown()
    Application.MoveAfterReturnDirection = xlDown
End Sub

Sub MoveRight()
    Application.MoveAfterReturnDirection = xlToRight
End Sub

You can assign these macros to

  • Quick Access Toolbar buttons, and/or
  • Ribbon buttons (you must first create a custom ribbon group), and/or
  • Custom keyboard shortcuts

The requirements for using the system are:

  1. The mail merge main document must be of the Letter type, though that does not mean that the output cannot be sent as an e-mail message.

 

  1. For the Many To One, Merge with Attachments and Merge to Individual Docs utilities, the data source may be either a table or query in an Access database, or in the form of an Excel worksheet and that worksheet must be the first sheet in the Excel workbook. If the data is on some other sheet, you can easily move that sheet so that it is the first sheet in the workbook by clicking on the sheet tab and dragging it to the left.  For the Chart Merge utility, download the Mail Merging with Charts document that is also on that page of my OneDrive for additional requirements of the data source for use with that utility

  2. For a data source in the form of an Excel worksheet, the field names must be in the first row of the worksheet and there must be a field name in all of the cells in that row that are within the range of columns that contain the data.

  3. For both types of data source, the field names must contain only alphanumeric characters (No @,#,$,%,&,(,), etc) and the field names must not start with a numeric character (0-9). The number of characters in the field names, including spaces, must not be more than 40.

  4. For a data source in the form of an Excel worksheet, the data must start in the second row of the worksheet and there should be no empty rows within the range of data that is to be processed.

  5. If there are multiple addressees in either the To or CC fields of the data source, the email addresses need to be separated by a semi-colon.

NOTE: The MergeTools applications cannot handle “Compound” MergeFields such as the «AddressBlock» or «GreetingLine».  Instead of using those fields, you will need to insert the individual merge fields.

how do I make an address label using windows 7 - Microsoft Community

jacquelinebowen say:

how do I make an address label using windows 7

How do I make a whole page, or pages, of labels, each with a different address on it.    
jay freedman say:
The preferable method involves using the Mail Merge feature. That's described at http://www.gmayor.com/merge_labels_with_word_2007.htm, and http://www.gmayor.com/convert_labels_into_mail_merge.htm may also be helpful.

However, a mail merge may be more work than warranted for a one-time job. In that case, follow these steps:

Click "Mailings" at the top of the ribbon. Then click the Labels button. Use the Options button in the Labels dialog to choose the "product number" of the labels you're going to print on.

Leave the Address box empty, and leave the "Full page of the same label" option selection. Click the New Document button. This creates a document containing a table, where each cell matches the location of a label on the paper. If you don't see the edges of the table as dotted lines, click "Layout" at the top of the ribbon and click the button at the left end marked View Gridlines.

Type the addresses into the cells, and print and/or save the document.
