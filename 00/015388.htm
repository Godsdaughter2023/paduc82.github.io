If you have to keep the text boxes, then the thing to bear in mind is that the index entries do not have to be right next to the text they are indexing. Mostly, they just need to be on the same page. So you could consider

gary's student say:

To extend Mike's idea.  If a cell already has 43 in it (or a formula returning 43), then use:

=43+TODAY()-DATEVALUE("2/6/2012")

or

=yourfomula+TODAY()-DATEVALUE("2/6/2012")

 

Use a date in DATEVALUE() that matches TODAY(), when you enter the formula you will see the 43, but tomorrow you will see 44.

mike h.. say:

Hi,

 

Dates in Excel are numbers that are formatted to look like the dates we recognise. Today is the 6 February 2012 and that is the number 40945 or that many days after 1 Jan 1900 and tomorrow will be 40946.

 

Because of that we can start anywhere we want and if we want to start at 43 we simply subtract 40902 by using the formula

 

=TODAY()-40902

 

Now that returns 43 and tomorrow it will return 44

robert aldwinckle on forums say:

Good thing you included a picture.   That Adobe Acrobat thing is nonstandard so I would try to get rid of it first.  It might be something that a safe mode test would not be able to avoid.  E.g. where does the Acrobat thing go?  The Menu bar and the Command bar go in their own row, so without a row to go into there is no reason to show the Menu bar.  For an example of this look at the Fullscreen mode with AutoHide off.  You can tick Menu bar in that mode but it doesn't show.  I does show in its own row if you toggle out of Fullscreen mode.  Similarly, because you are not using Show tabs in a separate row and you don't use the Command bar there would be no reason to have a Menu bar except for having it checked.  Are you aware that you don't need to have a Menu bar in order to use it?  Just press an Alt- key and then you will see it inserted somewhere among your other toolbars when you want to use one of its menus (e.g. assuming you can't remember the letter involved with showing one of its menus). 

Heh.  I just noticed the presentation that the Fullscreen mode Menu bar could get but now I can't reproduce it.  It happened when I was experimenting with turning the Menu bar on for Fullscreen mode.  What happened is that I was shown the Menu bar vertically!  That would be helpful for new users because it would get them used to seeing how each menu from the Menu bar appears when Alt- plus a "menu bar menu accelerator key" is pressed.  For example, in Fullscreen mode press Alt-F.  Notice how the File menu appears without its Menu bar?  This has been a long-standing deficiency that was pointed out when Fullscreen mode was created.   Once you have one menu showing you can cycle through the rest of them by pressing a cursor key but even knowing that this is how it works it is jarring IMO not being able to see the full Menu bar pop up for use simply by pressing an Alt- key exactly the same way that it does for the other IE window modes (normal and Maximized).
