Do you have Abbyy Finereader installed on the PC? A number of people have reported that removing it overcomes problems with 2013

Do you have “Fun Tools” add-in? A number of people have reported that removing it overcomes problems with 2013

Do you have Toshiba Book Reader installed on the PC?  It has been reported that uninstalling that brings Office back to working status.

Do you have the Acrobat PDFMaker add-in.  It has been reported that uninstalling that overcomes an issue with a message appearing that states "The document contains macros. For the error message "There was a problem sending the command to the program", use Windows Explorer to access the C:\Program Files\Microsoft Office\Office15 folder and right click on WINWORD.EXE then on Properties, go to the Compatibility tab and uncheck all the boxes.

graham mandeno say:

Does your underlying query have a WHERE clause that refers to [Forms]![Your Form]![txtDT] and [Forms]![Your Form]![txtDTE]?

I suggest you remove the WHERE clause from your query so that it returns all records, and add the date criteria to FilterStatusForm() like this:

    If Not IsNull(Me.txtDT) Then
        strWhere = strWhere & "[DateField]>=" & Format(Me.txtDT, "\#yyyy-mm-dd\#") & " AND "
    End If

    If Not IsNull(Me.txtDTE) Then
        strWhere = strWhere & "[DateField]<=" & Format(Me.txtDTE, "\#yyyy-mm-dd\#") & " AND "
    End If

Then replace Me.Requery with a call to FilterStatusForm() in your date event procedures.

You should also call FilterStatusForm in Form_Load and cmdReset_Click.

jack cook - aka help_line say:
As for the initial post that gave the error 0x8503001C, I found this comment that Scifiaddict made:

The error 8503001c generally relates to an error state in that the index of a synced conduit was interupted before it completed resolving the information on previous sync so the index of the AS partnership is stuck in an error state.  Though most often caused by email, any conduit in AS can generate this error but most often related to Outlook.. 

In the partnership settings would recommend trying to uncheck all items that go to Outlook which include the Contacts, Calender, Email, Tasks, and Notes.  Also uncheck Favorites, Files, Media and any third party conduits listed after Media as well.  This "everything unchecked" state of partnership is considered a null sync which should release any errors in the index file. 

Once a null sync completes successfully then I would reenable 1 conduit at a time and sync.  If one conduit consistently generates the 8503001c error each time it is enabled then moving all content out of that PIM may be necessary to resolve the error state.  A 10 second soft reset where instead of removing the battery just hold in reset button and count to 10 slowly will make sure that AS on the device starts up properly in case one of the conduits on the device crashed.  A full system shutdown of the PC and restart will make certain that all services used by the Activesync programs  on the PC are up and running properly including DHCP client, Universal Plug and Play, COM+ and DCOMM. 

I hope that this too might offer some insight into your issue

I'm guessing that if you uncheck contacts that it may well sync but more importantly if the original partnership is still on the device it may well be similar to an issue that I have experienced.  Some devices have an issue when a second device is added to the sync process.  Of course it should work fine but I have experienced it with two different devices that have two partnerships.  Unfortunately my only reliable resolution was to sync that particular device with only one PC.
