young yang msft support say:

Hi Harry,

From your subject, you refer to cutover migration.

For “Observed single mailbox move throughput is in the 0.3-1.0 GB/hour range”, this applies to hybrid move migration, but does not apply to cutover migration. For hybrid migration, we can put any number of mailboxes in the migration batch. So that meaning is: if we just migration one mailbox, the throughput is in the 0.3-1.0 GB/hour range.

For cutover migration, all mailboxes are migrated in one single migration batch. So, please refer to this information: 10-14 GB (20 concurrency).

•What is the limitation in reaching higher total rate?

Do you mean achieving higher rate than 14GB? If so, as far as I know, there is no official solution to achieve.

•If we set up the concurrent mailbox uploads to 50 or 60 will we have much better rate? If this is the case, we can arrange with our ISP for a temporary increase in upload speed.

For cutover migration, the maximum concurrent migrations is 20. Reference link: https://technet.microsoft.com/en-us/library/jj874016(v=exchg.150).aspx #Step 3:

“By default, this migration endpoint is configured to support 20 maximum concurrent migrations”

Moreover, as far as I know, even if we can set up the concurrent mailbox uploads to 50 or 60, there is no better rate.

Anyway, as mentioned above, due to various factors, the actual migration velocity might be slower or faster.
