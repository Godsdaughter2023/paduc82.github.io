ken sheridan say: A 'multi-valued field' really just disguises a many-to-many relationship type between two entity types, rather than modelling it in the time-honoured way by a table which resolves the relationship type into two one-to-many relationship types.

To get rid of the 'multi-valued field' therefore, you'd need to create the table to model the relationship type, and then execute an 'append' query which inserts multiple rows for each row in the current table, one for each value in the 'multi-valued field'.  To do this you need to reference the Value property of the field in the 'append' query, which would then, for each row in the current table, insert rows in the new table, each with the value of the primary key of the current table in one column and, in the other column, each of the primary key values in the table which the 'multi-valued field' currently references.

The interface for inserting rows into the new table would then be a subform, linked to the parent form on the primary key of the current table, and the corresponding foreign key in the new table.  The subform, in continuous forms view, would have a conventional combo box bound to the other foreign key column in the new table.

    Continuous page numbering across multiple documents - Microsoft Community

    mpennicott say:

    Continuous page numbering across multiple documents

    Hi,

    I have followed Cindy Meister's great article on continuous numbering:

     

    1.At the end of the first document insert a PAGE field; select it.

    2. Format the field as "hidden".

    3. Place the field in a bookmark (Word 6/95: Edit/Bookmark; Word97 and later: Insert/Bookmark).

    4. Save this file; open the next file.

    5. In the header or footer, where the page number should appear, insert the following set of fields:

    { = { PAGE} + { INCLUDETEXT "C:\\My Book\\Chap1.doc" Chap1Pages \* CharFormat \! } }

    6. Repeat the steps for each document in sequence.

     

    I have become unstuck in relation to step 6.  How exactly do I repeat the steps eg. do I repeat 1-4 in the next document to be numbered and then for step 5 do I change the .doc it is calling up to the previous document numbered, or does it still refer back to the very first page document? How does the latest document know to add the total number ofpages of the first page document + second document to the latest document?

    rohn007 say:
    you have to create a chain of links.

    In Chap1.doc, create bookmark Chap1Pages 

    In Chap2.doc insert the fields from step 5: 
    { = { PAGE} + { INCLUDETEXT "C:\\...My Book\\Chap1.doc" Chap1Pages \* CharFormat \! } }

    In Chap2.doc create bookmark Chap2pages (again with the PAGE field inside it.


    In Chap3.doc insert the fields from step5
    { = { PAGE} + { INCLUDETEXT "C:\\...My Book\\Chap2.doc" Chap2Pages \* CharFormat \! } }

    just keep chaining them that way.

    charles kenyon say:
    See Creating a Table of Contents Spanning Multiple Documents by Jonathan West.
    http://pubs.logicalexpressions.com/Pub0009/LPMArticle.asp?ID=148

    In answer to your specific question, each document refers back to the previous document, not to the first document.
    For documents after the first you insert the field shown in #5 instead of the PAGE field in #1, hide that field and bookmark it for reference in the next document. Note that you'll have to sequentially update fields in your documents before you print or prepare a Table of Contents.
