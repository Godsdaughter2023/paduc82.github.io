Unavoidable autocorrect on PowerPoint - Microsoft Community

inalja say:

Unavoidable autocorrect on PowerPoint

I had been experiencing automatic corrections to a number of words in PowerPoint while creating presentations for a class on dead and artificial languages.

  • Certain spellings were changed to match similarly spelled English words (e.g., "eldar" > "elder")
  • Diacritical marks, specifically umlauts/diaereses over word-final "e" were removed upon hitting the spacebar, when such removal resulted in English words (e.g., "barë" > "bare")
  • certain lower-case words were capitalized, for example "tuluva" (will come) was changed to "Tuluva" (a population living in India)

I thought I had disabled all autocorrect options ( file > options > proofing > autocorrect options > AutoCorrect ) BUT:

  1. I disabled "Replace as you type" before disabling "Automatically use suggestions from the spelling checker".
  2. Upon disabling "Replace as you type" I was no longer able to toggle "Automatically use suggestions from the spelling checker" which had been grayed-out, although it was still enabled.

It was necessary to first disable "Automatically use suggestions from the spelling checker" and then disable "Replace as you type".

My assumption was that disabling "Replace as you type" would prevent any replacement, but that assumption was incorrect. That option must be enabled before the other can be disabled.

This misunderstanding engendered a lot of fury in me, but was easy to fix, if not intuitive. I hope this record will help others avoid similar frustration.
