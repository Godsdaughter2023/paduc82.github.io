By design Remote Desktop [RDC] users must have a password.

Your options would be to create a password for the user account your trying to login in with on the RDC host PC or to create a special user account, with password, that you only use to login in with RDC. Keep in mind the account user ID and password you use to login is for the RDC host and not your RDC client.

Example:

Home PC = PC1, users Dale (administrator, automatically a member of the Remote Desktop Users Group), Sue (standard user) and Jim (standard user and member of Remote Desktop Users Group)

Laptop = PC2, users Stan (administrator) and Sue (standard user)

When connecting to RDC host PC1 from RDC client PC2 login with either the Dale or Jim account and password. Sue is not a member of the Remote Desktop Users Group or an administrator so that account can not be used.

In all cases use a strong password and make sure Network Level Authentication [NLA] is enabled and used. NLA is supported by Windows 7/Vista/XP SP3 clients.

https://www.microsoft.com/protect/fraud/passwords/checker.aspx?WT.mc_id=Site_Link

http://windows.microsoft.com/en-US/windows7/What-types-of-Remote-Desktop-connections-should-I-allow

When calling over your home LAN use either the computer name or its LAN IP. From a true remote location use the public IP of your router or a fully qualified domain name if you have one setup with a service like No-IP.com (free).

Personally I also recommend disabling RDC connections on the laptop RDC client, unless of course you wanting to remotely control the laptop (unlikely). Also remember to use a Public network location on the laptop when away from home and on any type of untrusted network, ie. bar/restaruant/library/etc.

You can rule out the server because the package is verified at step 3.  The installer tool would not progress past that point if your server theory was correct.
