marshall barton say:

Requery will make the first record the current record, but I don't see how the active control (with the focus) is changed to a different control.  Maybe you have a SetFocus somewhere (e.g. Current event)??

 

You can get the subform to return to the record that was current before the Requery (if the record is still in the subform's recordset) by adding code to the area with the Requery:

 

Dim temp

       'save current record's primary key field's value

temp = Me.[subform control name].Form.pkfieldname

 

Me.[subform control name].Form.Requery

 

       'find record with saved primary key value

With  Me.[subform control name].Form.RecordsetClone
   .FindFirst "pkfieldname=" & temp    'if the pk field is a number field
  '   .FindFirst "pkfieldname=""" & temp & """ "    'if the pk field is a Text field

   If Not .NoMatch Then Me.[subform control name].Form.Bookmark = .Bookmark
End With

 

If, for some reason I don't see, you need to reset the focus, then you could use code like:

 

Dim ctltemp As Control

      'save active control
Set ctltemp = Me.[subform control name].Form.ActiveControl

  . . .

      'reset the focus

ctltemp.SetFocus

brian tillman [outlook mvp 2007-2019] say:

If you're using POP accounts, first, copy the Inbox contents of all of the non-default data stores to the Inbox of the default data store, then delete all accounts except for the default account.  Then remove from the mail profile all of the data stores for the non-default accounts.  Re-add each account manually (do not let Outlook add them automatically), and specify that you wish to use an existing PST.  Once you are done, your accounts will all be back and all will deposit incoming messages into the one Inbox.
