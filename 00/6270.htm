ken sheridan say: I'd failed to notice that you'd remarked out the following line:

    'strCriteria = Mid(strCriteria, 6)

Unremark it and the leading ' And ' will be stripped from the expression.

ken sheridan say: So, the InsuranceExpiry column is not of date/time data type, as 0/0/0000 is not a valid date/time value.  I assume it is a text data type.  I would recommend that you change the column to a date/time data type, but you will not be able to give it a default value which is not a valid date.  If you keep it as text you will need to convert its value to a date/time value by means of the CDate function:

   Dim strCriteria As String

   If Not IsNull(Me.txtStartDate2) Then
        strCriteria = " And CDate(InsuranceExpiry) >= #" & _
            Format(Me.txtStartDate2, "yyyy-mm-dd") & "#"
    End If

    If Not IsNull(Me.txtEndDate2) Then
        strCriteria = strCriteria & " And CDate(InsuranceExpiry) < #" & _
            Format(Me.txtEndDate2, "yyyy-mm-dd") & "#+1"
    End If

    ' remove leading ' And '
    'strCriteria = Mid(strCriteria, 6)

    ' open report
    DoCmd.OpenReport "reportInsuranceExpiryDate", _
        View:=acViewPreview, _
        WhereCondition:=strCriteria
