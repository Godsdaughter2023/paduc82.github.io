marshall barton say:

The only way I have found to get rid of phantom objects is to create a new blank database with all of your database settings, especially make sure the Name Auto Correct option is OFF.

 

Then import the objects from the problem database making sure you do not import any of the phantom objects.  That means you can not use the SELECT ALL option and instead you need to check each valid object before importing.  If the new database has a problem with an imported object, make a note of its name and start over skipping that object when selecting the objects to import.

 

Note:  This is just feeling, but I suspect the phantom objects happen when you edit a form/report's VBA module while the object is not in design view and then doing other things like copying it and deleting it.  Regardless of my suspicions, editing a form/report's VBA module when not in design view is a well known cause corruption.
