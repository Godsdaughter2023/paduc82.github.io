xld say:
As an aside, if your concern is that column A might be extended by the user and you want to ensure that column B still reflects that associated value, then you could use worksheet change event code to setup the formula.
 
This would be the code based upon my previous post example
 
Private Sub Worksheet_Change(ByVal Target As Range)
Const WS_RANGE As String = "A:A"     '<== change to suit
 
    On Error GoTo ws_exit
    Application.EnableEvents = False
 
    If Not Intersect(Target, Me.Range(WS_RANGE)) Is Nothing Then
        With Target
       
            If Not Target.Offset(0, 1).HasFormula Then
           
                Target.Offset(0, 1).FormulaR1C1 = "=IF(RC[-1]<>"""",VLOOKUP(RC[-1],Sheet2!R1C1:R20C2,2,1),"""")"
            End If
        End With
    End If
 
ws_exit:
    Application.EnableEvents = True
End Sub
 
This is worksheet event code, which means that it needs to be
placed in the appropriate worksheet code module, not a standard
code module. To do this, right-click on the sheet tab, select
the View Code option from the menu, and paste the code in.

The formula syntax is correct.

Try changing the range_lookup argument from 1 to 0:

 =IF(B7<>"",VLOOKUP(B7,Sheet2!$A$1:$B$14,2,0),"")

0 means that you want an "exact" match. If an "exact" match is not found then the formula will return #N/A. When you use 1 as the range_lookup argument, if an "exact" match isn't found then the formula will return a "closest match".

bill manville say:

I don't know of a built-in way to do this.

But the following function might help

Function Numbers(Values As Range) As Double()
  Dim adNumbers() As Double
  ReDim adNumbers(1 To Application.Count(Values))
  Dim lVal As Long
  Dim R As Range
  Dim lNumber
  For Each R In Values.Cells
    If TypeName(R.Value) = "Double" Then
      lNumber = lNumber + 1
      adNumbers(lNumber) = R.Value
    End If
  Next
  Numbers = adNumbers
End Function

So, if you have X values in A2:A20 and Y values in B2:B20 but some rows have non-numeric data IN BOTH RANGES then =Slope(Numbers(A2:A20), Numbers(B2:B20)) will give you the Slope etc.

If the X values are all numbers but the B values are not then a modified approach would be needed

mike middleton say:

I am using excel 2013, trying to automate a bit of data analysis.  It would be incredibly helpful if there were a way to exclude nonnumeric data from a number of functions in excel.  Is this possible?  I can't seem to figure out a simple way.  Is there one?   

As far as I know, Excel does not have a way to exclude non-numeric data (or handle missing values, etc.) as full-featured statistical software does.

So, if you want to use Excel's functions for data analysis, I suggest performing "data cleansing" before any computations. 

One work-around is to use Data > Sort for each variable by specifying the key in the "Column Sort by" drop-down box. Missing values and non-numeric data will show up at the top or bottom of each sort. For each sort you then can manually exclude the nonconforming rows from the data range before proceeding with another sort for the next variable.

Excel's statistical functions were completely overhauled for the Excel 2010 release (to fix some numerical instability problems), but I don't think those changes can accommodate non-numeric data.

shane devenshire say:

A year late and a dollar short:

Excel automatically excluded non-numeric and blanks from many calculations, such as STEYX.  It does not exclude errors though.  Here is an example which addresses #NA errors text and blanks:  Suppose you have the following data:

X Y
1 7
3 6
5 9
#N/A #N/A
6 5
2 4
4 6
   
t 45
1 b

=STEYX(IFNA(IF(ISTEXT(I5:I11),"",IF(I5:I11="","",I5:I11)),""),IFNA(IF(ISTEXT(H5:H11),"",IF(H5:H11="","",H5:H11)),""))

Note the sequence of IFs is important.
