Word 2003 not auto-formatting fractions - Microsoft Community

pat02050 say:

Word 2003 not auto-formatting fractions

I've recently upgraded from WinXP to Win7, and I've discovered that Word 2003 is not auto-formatting fractions anymore.

In Tools/Autocorrect options..., under the Autoformat tab, I have Fractions checked.  My font is arial.  I've tried editing previous documents that already have the fraction symbols in place, and it doesn't work there either.  I tried switching to Times New Roman, but no change.

Autocorrect IS working in some cases, but not others.  If I type "teh", it corrects it to "the".  But I've also noticed that it doesn't upper-case the first letter of a sentence if I missed that.  I can live without that, but if I type "1 1/2", I'd really prefer the symbol to be substituted.

suzanne s. barnhill say:

A few observations:

  1. Having "Fractions (1/2) with fraction character (½)" enabled on the AutoFormat tab of Tools | AutoCorrect Options will not help. Where you need to have it enabled is the AutoFormat As You Type tab.
  2. AutoFormat should fire if you type "1 1/2," but that's not really helpful since you then need to go back and delete the space in order to have 1½; (typing 11/2 obviously doesn't work). That's why I have custom keyboard shortcuts for ¼, ½, and ¾; if you type recipes a lot, you might also want to add ⅓, ⅔, and ⅛, and ⅜, ⅝, and ⅞ are available from the Symbol dialog.
  3. First-letter capitalization does not work if the previous sentence ends in a numeral. Several versions back, Microsoft responded to complaints about numbered lists being automatically capitalized in cases where this wasn't desirable. The fix unfortunately affected not only numbered lists but also numbers in running text.
  4. Automatic capitalization also fails if a word is misspelled. In your example, if the first word in the sentence is "teh," it will be corrected to "the," but it won't be capitalized.
