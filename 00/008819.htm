ken sheridan say:
I was hoping to avoid SQL, as I am not as familiar with that as I am with programming directly in Access.

You don't need to do it in SQL view, it can be done in query design view.  If we stick with Scott's LOC column in the table, which will be a Required field with possible values of Home or Road, then in the form use an option group rather than three independent check boxes.  You can build this with the wizard and set it up so that the values correspond with the text labels like this:

1    Home
2    Road
3    Either

I'd suggest making Either the default, but the choice is yours.

Base the report on a query and in the criteria row of the LOC column in query design view enter the following as  a single line:

([LOC] = "Home" AND [Forms]![YourFormName]![YourOptionGroup] = 1) OR ([LOC] = "Road" AND [Forms]![YourFormName]![YourOptionGroup] = 2) OR [Forms]![YourFormName]![YourOptionGroup] = 3

substituting the real form and option group names of course.  You can then simply open the report from a button on the form.  You'll find that after saving the query in design and opening it again in design view Access will have moved things around.  The logic will be the same, but not so readily apparent, and the query will work just the same.  This is one reason what it's better to do it in SQL view, as it doesn't obscure the underlying logic, but in this case it is not necessary.

ken sheridan say: However you export the result set of the query the parameter has to be capable of evaluation, which means it has to be a control in a an open form, not a simple prompt.  Why you are getting the error when using a literal string rather than referencing a parameter, I've no idea, but there are one or two unusual things about your query:

1.  Some object names are wrapped in square brackets although none include spaces or other special characters.  While unnecessary, this should not cause any problems however.

2.  You have in a number of places using the exclamation mark (!) rather than the dot operator (.)  The latter should only be used in a reference to a control, e.g. Forms!MyForm!MyControl, not as a column qualifier e.g. dbo_iminvloc_sql.qty_allocated rather than [dbo_iminvloc_sql]![qty_allocated].

3.  The WHERE clause has unnecessary parentheses; harmless but unnecessary and confusing.

There should be no problem calling the TransferText method of the DoCmd object to export a query which references a control in a form as a parameter, so you can easily have a dialogue form with a combo box to select the category and a button to call the TransferText method in its Click event procedure, making this conditional on the combo box not being Null of course.  You can wrap the code in a function like this:

Function ExportToDelim(strQuery As String, strExportTo As String, Optional blnHasFieldNames = True)

    DoCmd.TransferText _
        TransferType:=acExportDelim, _
        TableName:=strQuery, _
        FileName:=strExportTo, _
        HasFieldNames:=blnHasFieldNames

End Function

If you need to specify a new file to which the data are to be exported you can include a text box for the file name and select a folder by calling the following function from a button in the form:

Function OpenFolderDialog() As String

    With Application.FileDialog(msoFileDialogFolderPicker)
        If .Show Then
            OpenFolderDialog = .SelectedItems(1)
        End If
    End With

End Function

If you need to select an existing file the following function can be used:

Function OpenFileDialog() As String

    With Application.FileDialog(msoFileDialogFilePicker)
        If .Show Then
            OpenFileDialog = .SelectedItems(1)
        End If
    End With

End Function

An alternative to calling the TransferText method would be to call the following function to write the data to a file line by line.  This allows you to specify the delimiter character, whether text data is enclosed in quotes or not, and whether to export the field names as the first line (the default) or not.  As you see the code in this function evaluates the query's parameters:

Function ExportToText(strQuery As String, _
                    strExportTo As String, _
                    strDelim As String, _
                    blnQuoteText As Boolean, _
                    Optional blnHasFieldNames As Boolean = True)
                    
'Accepts:
' strQuery - Text - name of query to be exported
' strExportTo - Text- path to file to export to
' strDelim - Text - delimiter character(s) to separate fields
' blnQuoteText - Boolean - True to enclose text or memo data in quotes
' blnHasFieldNames - Boolean - (optional) - True (default) to export field names as first line

   Dim dbs As DAO.Database
   Dim rst As DAO.Recordset
   Dim qdf As DAO.QueryDef
   Dim prm As DAO.Parameter
   Dim Fld As DAO.Field
   Dim n As Integer
   Dim strPrintList As String
   Dim strQuote As String
   
   Set dbs = CurrentDb
   Set qdf = dbs.QueryDefs(strQuery)
   
   ' evaluate query's parameters, if any
   For Each prm In qdf.Parameters
       prm = Eval(prm.Name)
   Next prm
    
   Set rst = qdf.OpenRecordset
   
   If Dir(strExportTo) <> "" Then
       If MsgBox("Overwrite " & strExportTo & "?", vbQuestion + vbYesNo, "Export Query") = vbYes Then
           Kill strExportTo
       ElseIf MsgBox("Append rows to " & strExportTo & "?", vbQuestion + vbYesNo, "Export Query") = vbNo Then
           Exit Function
       End If
   End If
           
   With rst
       If Not (.BOF And .EOF) Then
           Open strExportTo For Append As #1
           
           If blnHasFieldNames Then
                ' include column headings in text file
                For n = 0 To qdf.Fields.count - 1
                     strPrintList = strPrintList & strDelim & qdf.Fields(n).Name
                Next n
                ' remove leading delimiter
                strPrintList = Mid$(strPrintList, Len(strDelim) + 1)
                Print #1, strPrintList
                strPrintList = ""
           End If
           
           Do While Not .EOF
               For n = 0 To qdf.Fields.count - 1
                   Set Fld = .Fields(n)
                   strQuote = IIf(blnQuoteText And (Fld.Type = dbText Or Fld.Type = dbMemo), """", "")
                   strPrintList = strPrintList & strDelim & strQuote & _
                   .Fields(n) & strQuote
               Next n
               ' remove leading delimiter
               strPrintList = Mid$(strPrintList, Len(strDelim) + 1)
               Print #1, strPrintList
               strPrintList = ""
               .MoveNext
           Loop
           Close #1
       End If
   End With

Exit_Here:
   rst.Close
           
End Function
