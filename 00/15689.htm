Does your double notification look something like this in File menu > Account command?  This notice is for 2 2010 installations, but 2013 should look similar.

This situation occurs when you install a purchased copy of Office on a computer that has an unactivated factory installed trial.  In this case, it sounds like you did 2 installs, without a SUCCESSFUL uninstall. Same basic concept.

The simple fix is to uninstall the factory trial and do an Office repair of your purchased copy. If you can't tell which is which, then uninstall both and re-install and activate your purchased copy.

Use method 1 in this tip to get rid of the factory trial:

KB 2739501 Uninstall Office 2013 / 365 - Method 1 Fix It

http://support.microsoft.com/kb/2739501  Office 2013, Office 365 and Office 2013 Preview uninstalls

(sorry, they have re-written the tip since I first looked at it!)

  • Method 1: Uninstall using Fix it
    Note: If you are using a 64-bit operating system, save the fixit tool on the system and then run it from the saved location.
  • Method 2: Uninstall from Control Panel
  • Method 3: Remove manually

Method 1 in this tip does a more thorough job that the “normal” uninstall via Programs and Features (aka Method 2 in the tip). The Normal uninstall is designed with the assumption that you will be re-installing Office. It does NOT delete everything. It leaves configuration files like NORMAL.DOTM and other templates etc. and it also leaves configuration Registry entries. These “carry-overs” may contain corruption that is causing your problem.

Boot the computer. After the restart go to http://office.com/myaccount to re-install Office 365.

I want to hide & unhide columns based on a certain cell value

Hello guys

What I want here is that I want to just display column NN for January (hide other columns for this month). And for February I want it just display column NO (hide columns NN and other columns). one column for one month. is there any way I can do this? 

Note: I want to enter data into these columns for each month manually. 

amit tandon say:

Enter below code in the Sheet Module:

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
On Error GoTo ErrHandler
Application.EnableEvents = False
Application.ScreenUpdating = False
Dim rCell As Range, i As Long
Set rCell = Range("B3")
Range("NN:NY").EntireColumn.Hidden = False
If Not (Target.Column >= 378 And Target.Column <= 389) Then
    rCell.Value = ""
    GoTo ErrHandler
End If
If Target.Column >= 378 And Target.Column <= 389 Then
    rCell.Value = Target.Column - 377
    Columns(rCell.Value).EntireColumn.Hidden = False
    For i = 378 To 389
        If i <> Target.Column Then
            Columns(i).EntireColumn.Hidden = True
        End If
    Next i
End If
ErrHandler:
    Application.ScreenUpdating = True
    Application.EnableEvents = True
End Sub

This will hide all columns "NN:NY" except the active cell column. It will also enter the month number 1 to 12 in cell B3 (1 if a cell in column NN is selected, 2 if a cell in column NO is selected, ... 12 for column NY). If none of the columns "NN:NY" is selected then B3 will be blank.

Regards,

Amit Tandon

Hi, for displaying 2 columns for each month, so as to double the range to 24 columns ... below code will display 2 columns like NN/NO, NP/NQ, and so on for any of these which are selected, and then put the month number in cell B3 (for NN/NO it will be 1, for NP/NQ it will be 2, ....):

Private Sub Worksheet_SelectionChange(ByVal Target As Range)

On Error GoTo ErrHandler
Application.EnableEvents = False
Application.ScreenUpdating = False

Dim rCell As Range, i As Long, lStartCol As Long
Set rCell = Range("B3")
lStartCol = 378    'Range("NN1").Column onwward 24 columns, 378 to 401

Columns(lStartCol).Resize(, 24).EntireColumn.Hidden = False
If Not (Target.Column >= lStartCol And Target.Column <= lStartCol + 23) Then
    rCell.Value = ""
    GoTo ErrHandler
End If

If Target.Column >= lStartCol And Target.Column <= lStartCol + 23 Then
    Columns(lStartCol).Resize(, 24).EntireColumn.Hidden = True
    If Target.Column Mod 2 = 0 Then
        Columns(Target.Column).Resize(, 2).EntireColumn.Hidden = False
    Else
        Columns(Target.Column - 1).Resize(, 2).EntireColumn.Hidden = False
    End If
End If
rCell.Value = Int((Target.Column - lStartCol + 2) / 2)
   
ErrHandler:
    Application.ScreenUpdating = True
    Application.EnableEvents = True

End Sub

jim gordon mvp say:
Phonetic guides are included in Microsoft language packs in Windows versions of Microsoft Office. This article explains the phonetic guides.

Office for Mac has a very different internal structure regarding languages. There are no language packs in Office for Mac. Instead, the entire suite is installed using a single language. Because there are no language packs for the Mac, this feature is not available on the Mac.

If you would like to suggest to Microsoft that they change the way language support works on the Mac, go to Word's Help menu and choose Send Feedback. This displays the suggestion form.

jim gordon mvp say:
You're making an assumption that Word on the Mac and PC are the same software. They are not. The Mac version and the PC version are completely different. The Mac version of Office takes the approach that the entire interface is written for a particular specific language. Spelling and grammar tools allow documents and portions of documents to support different languages (far fewer than the PC edition of Office). 

On the Mac, you have to obtain an installer that was written from scratch in a particular lanaguage. That is the only way you can change the language of the Office user interface.

In Office on the PC a very different approach was taken. On the PC side, Microsoft decided that it would be better to have a common base user interface that could be modified to support various languages via modifications that are made by running a language pack to change the interface language. The Mac version of Office was not written with this in mind and has no support for language packs. It is not possible to run a language installer pack on a Mac.

If you would like to make a suggestion to Microsoft that they adopt the PC approach for the Mac, then in Word, go to the Help menu and choose Send Feedback. That displays the suggestion form, which you can fill in and offer your suggestion in the hope that in some future edition of Office for Mac that the entire approach to language is revised to support language packs.

Suggestions are counted. The more people who send in the same suggestion, the more likely it is to be implemented. Keep in mind that replies here in Microsoft Answers are read only by fellow Office users like you and me, so use the feedback form if you want Microsoft to know what you would like to see happen.

solkeys say:

Set Cells to Fixed Size

I created a Table and inserted the same size picture in each cell. This increased the size of each cell. Now I want the size of the cell to stay same even if I decrease the size of the pictures in the row, and then have the pictured center. How can I do it?
alice_941 say: Go to Table Tools.  Click on the Design tab.  Click on Borders.  A window will open up.  In that window, on the Borders tab, under Setting (at the left), click "None."  Then you will have no borders around the cells of your table.  I don't really know how to save as a Web Page filtered without the cells.  I know that if you go to the View Tab, you can select, under Document Views, "Web Layout," but after that, I don't know. 

alice_941 say:

Graham, I was glad to learn from you two useful things:  1)  (from your October 18, 2012, post), how to set column widths to fixed width:  Layout tab > Autofit > Fixed Column Width.  I had never seen that before; and 2) (from your October 20, 2012, post:  "If you set the cell width to fixed, then it will not grow to accommodate the original image size, but the image will be shrunk to fit the cell width....Once the image is inserted it will not automatically change size to accommodate any further change in cell width."  That last bit is not what I would have expected and is new to me.  I will check it out.  I do think that is the best method if the picture must be a certain size, or if the pictures are more important than text.  However, I have found when using both text and pictures in a table that it is more useful for me to determine column width based mainly on how much text I have for the cells in a particular column.  I usually want to keep row heights to the smallest I can so as to spread the table over as little paper as possible when I print, and that requires adjusting column widths so that the columns that contain the most text are wider than the columns that contain less text.  Therefore, if I choose a column width (or row height) to accommodate a picture, it may end up making a row unnecessarily high or a column unnecessarily wide.  Thus, if I have established row and column heights already to be the most economical of space, I would rather adjust the picture to the cell size than the cell size to the picture (I would, however, enlarge the row height slightly if the picture were too small to look good).   On the other hand, if the picture was paramount, then I might "insert cells of the size you require for the display of the image," as you advised in your October 20, 2012, post.
