#1 - Yes the same account can be configured as both Imap and POP in the same profile. The key operational words to keep in the back of your mind are "same account". In short, if you access the account both via POP3 protocol and Imap - it's the same Inbox which means if the POP3 is configured to delete messages from the server - you will no longer see those messages in your Imap Inbox.

#2 - Rules - would be configured for the email "account" - not the protocol used - as mentioned above - one and the same thing.

#3 - Not sure I would use the word "convert" as the operative word when you re-configure the account. What you are doing when you add the same account to the profile, simply using a different protocol to access the info. Some things to keep in mind when you configure the same account since the "sequence" matters in terms of what PST files you have.

Some example scenarios (specific to Outlook '2010 - not the same for O'2013).......

(S1) New Profile - Imap account

- will result in 2 PST files being created. One called "Outlook Data FIle" which will be set as the default data file where you contacts/calendar items will be stored. The 2nd PST file will act as the offline storage file

(S2) New Profile - POP account

- will create one PST file containing all folder items (contacts/calendar) and will be the default for the profile

(S3) New Profile - First configured - POP account - Second configured - Imap account

- PST file for POP account created as described above (S2)

- Unlike S1 - a secondary PST file for contacts/calendar will not be created. The PST file associated with the POP account will be used.

(S4) New Profile - First configured - Imap account - Second configured - POP Account

- the PST files described in S1 will be created

- the "Outlook Data File" will be used for the default contact/calendar folders when the POP account gets configured

#4 - In terms of using PST files - any PST file can be manually opened/used in any profile. If the PST file is not associated with an email account - nothing be added/deleted automatically.

a. user say:

Unfortunately, Word's number/currency formatting only follows some of the "rules" that are set up in Windows Control Panel, and one of the things it ignores is the grouping pattern - Word always groups in 1000s (i.e. blocks of three digits).

You can work around this using some complicated field coding in Word, but it's not pretty - see, e.g. my suggestions in https://answers.microsoft.com/en-us/office/forum/office_2007-word/number-format-like-20000000-its-two-lacs/2ae3020c-c5e9-417f-9664-879393f00f37 and my sample document at https://goo.gl/vyR8RR

Since your data source is Excel, it might be easier to create a column that uses Excel formulas to generate the correct formatting as a text string rather than a number.

For example, if the amount in your example is in cell B4, a formula in cell C4 such as 

=TEXT(B4,"₹ #,##0.00;[Red]₹ -#,##0.00")

*might* allow you to avoid using a formatting switch in Word.

Personally, I think it would be better if this problem was fixed in Word (and perhaps some of its other formatting deficiencies could also be fixed) - you could post a request at https://word.uservoice.com/ .

hansv mvp say:

You can create a default template with the desired table style:

  • Create a blank new workbook.
  • On the Home tab of the ribbon, in the Styles group, click Format as Table.
  • Right-click the style you want to use, and select 'Set as Default' from the context menu.
  • Press F12 to display the Save As dialog.
  • Select 'Excel Template (*.xltx)' from the 'Save as type' dropdown.
  • Enter   %appdata%\Microsoft\Excel\XLSTART\Book.xltx   in the File name box (the path and filename are obligatory).
  • Close Book.xltx.

To test it:

  • Press Ctrl+N to create a new workbook.
  • When you insert a table in the workbook, it'll use the style you selected.
