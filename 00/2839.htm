A survey database requires a VERY specific structure, not the one you are using, unfortunately. This is the base strcuture I use for all survey apps:

tblQuestion

QuestionID (PK Autonumber)

Question

tblRespondent

RespondentID (PK Autonumber)

names, other info about respondent

tblResponse

ResponseID (PK Autonumber)

RespondentID (FK)

ResponseDate

tblAnswer 

AnswerID (PK Autonumber)

ResponseID (FK)

QuestionID (FK)

Answer

With a structure like this each Answer is a single record in tblAnswer. So you don't need multiple comboboxes. If you need to have a specific choice list per question then you would add a table like :

tblChoices

ChoiceID (PK Autonumber)

QuestionID

Choice

You would use a Rowsource for your combo that would filter this table for the QuestionID.
