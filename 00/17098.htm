The #value error occurs when the wrong type of argument or operand is used.
Click the cell that displays the error, click the button that appears , and then click Trace Error if it appears.

Entering text when the formula requires a number or a logical value, such as TRUE or FALSE

Supplying a range to an operator or a function that requires a single value, not a range

 

joeu2004 say: skat100 wrote:

It can mean many things, depending on context.  The Excel 2003 Help page "Correct a #VALUE Error" hints at some root causes:

* Entering text when the formula requires a number or a logical value, such as TRUE or FALSE

   Note:  But text that looks like a number can be used in arithmetic expressions.

   ... Instead of ctrl+shift+Enter

   Also:  Using ranges of inconsistent sizes; e.g. SUMPRODUCT.

* Running a macro that enters a function that returns #VALUE!

   Also:  A formula that references a VBA user-defined function that contains an error.

* Also:  A reference to a cell that results in a #VALUE error.

bob jones aka: cybertaz say:
I don't mean to contradict anyone, but Word Work Files are not in any way associated with the AutoRecover feature... the files are similar in nature [temporary] but are completely disparate from AutoRecovery files, although they can be interdependent.

One primary distinction is that AutoRecovery files are generated strictly for the benefit of the user, whereas Work files are created based on the needs of the program. Those needs include copy/paste, OLE Objects, links & a variety of other high overhead operations. IOW, Work files are created when the program requires them regardless of whether AutoRecover is active or not. And, unlike AutoRecover, generation of Work files can't be turned off - it's a hard coded facet of how the program is designed to function in order to enhance performance.

Another distinction is that AutoRecovery files are placed in one specific location for the User to which they pertain, but Work files are usually created in the same folder as the document they're created for. As I understand it, there can also be several Work files for the same document & any particular Work file can pertain to more than one document, dependent on what purpose each is created to serve. Normally, though, the existence of Work files is totally transparent to the user. However, unlike AutoRecover files, Work files are not necessarily deleted when a specific document closes because the program may "think" the Work file content is needed for further use.

One of the main reasons Work files can become a problem - nothing personal :-) - is file mismanagement on any combination of several levels: User, Network, OS. Correcting the issue can often be as simple as Quitting & relaunching the program. That's why it's recommended that no app be allowed to run or files be left open 24/7 - launching daily as well as restarting the Mac several times weekly will usually obviate any issues. Since Work files are simply temporaries they safely can be deleted like any others, provided Word is not running at the time.

See if a restart of Word resolves the issue. If not, Quit Word, do a Spotlight search for Word Work File, delete any that you find then empty the Trash. Likewise, once you Quit Word you can safely delete any remaining AutoRecover files.

If not or if there are other problems involving AutoRecover or installation of AutoUpdate they'll have to be addressed separately. It would help to know your exact update level of Office 2008 as well as the complete & exact content of any error messages.

Bob J.

rohn007 say:

Natively, Word 2003 only handles DOC format files. You have to install a separate update, the Office 2007 Compatibility Pack

Office 2000-2003 Compatibility Pack 32/64-bit EXE Offline installer - http://getintopc.com/softwares/office-tools/office-compatibility-pack-free-download/  open 2007-2010 format files in 2000-2003

This will enable 2003 to open the new 'x-file' formats, like DOCX. 

Any new features added after 2003, ie "Text Effects" in 2010, that are not supported by 2003 will be "gracefully degraded".  For example, some animation effects will become static, while the "Text Effects" for the most part will simply be stripped out, leaving just the text.

You also have the option in 2016/365 of saving them in the old 97-2003 formats, ie DOC.

NOTE: 2003 is no longer supported by MS. That means they no longer provide any security patches via Windows Update.  Since change in Office (and Windows) is "evolutionary" rather than "revolutionary", when hackers find defects in newer versions of Office, they can often very easily adapt them for 2003 (and earlier).  While in 2007 and later they will be (eventually) fixed, they will no longer be fixed in 2003.

Bottom line. It is no longer a good idea to keep 2003 on machines connected to the internet without taking extreme precautions.
