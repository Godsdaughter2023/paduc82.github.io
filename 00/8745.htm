Spell checking is being done by the browser and not by outlook.com in this case.

In IE10, click on Tools.

Absolute direct connectivity via WiFi with no intermediate application is essentially a no-go, even when using a Windows PC.

albert d. kallal say:

It not clear here if you are talking about using SQL server here and pass though query?

If you talking about a native Access only or JET/ACE database engine application, then the concept of pass-through has no bearing and will not make any difference at all here.

So you have to clear up if you are talking about a connection to sql server here.

If you are in fact connecting to sql server then the main issue would likely be that of the type of network connection.

Remember, using a VPN or some type of connection over the internet is 100 times slower then you standard typical "local" area network (LAN) in a typical business environment.

So something that takes a simple 1 minute on your local typical office network will now take 100 minutes if your connection is OVER high speed internet. This means over 1 and half hours!

So perhaps the issue is the network connection involved here?

A pass-through query to pull 90k records from SQL server on a typical officer LAN should only take about a second at most.

So either there is a network connection form your desktop client software to where the accDB or mdb Access table resides, or there is some kind of network connection to the SQL server that is bandwidth limited here.
