a. user say:

First know that defragging will not recover any more disk space.

Second know that Windows 7 has a Scheduled Task that runs defrag every Wednesday at 1AM or as soon as possible after that if the system happens to not be running at 1AM:

That does a pretty good job and you can adjust the task to your liking, nothing wrong with defragging manually and some folks like third party defraggers better (like Defraggler).

When defrag runs and if it finds bits and pieces of files here and there that need to be put back together it will try to do that but that changes some other things too - specifically your Volume Shadow Copies which are involved with System Restore and Restore Previous Version.

So what has probably happened is when you ran the defrag it also needed to adjust some of the Volume Shadow Copies on your system and some of those Volume Shadow copies got larger.  If you have lots of Restore Points you have lots of shadow copies and if many of them got adjusted many of them could have gotten larger.

Chances are this "problem" (really a mildly annoying but normal observation) has been happening all along with the Scheduled Task but you never noticed it or has never amounted to much more space being consumed.

It is a very common complaint/observation and has nothing to do with bad sectors - it is normal.

You can check the amount of storage your VSS take up from a Command Prompt like this:

You can adjust the amount of storage your VSS copies consume, you can use Disk Cleanup to whack all but the last Restore Point and all the old VSS copies:

Then Windows will start accumulating them again like it usually does depending on how your system is configured.

Disable automatic restart on system failure
