How do I search unregistered file types in Windows 7 for a phrase - Microsoft Community

ed_derz say:

How do I search unregistered file types in Windows 7 for a phrase

Going back to early versions of Windows, it was easy to search for a phrase in all files in a directory and subdirectories by simply providing the phrase.

Then in Windows XP, things changed.  "Indexing" was added and by default, Windows would only search "Registered" file types..  .TXT, .DOC.,  etc.
However, there was an ability to override that "Default" behavior and search all files even if they were not registered.

Then in Windows 7, the ability to search unregistered types appears to have been eliminated.  If you search for the phrase "Find Me" and a file containing that phrase exists in an unregistered file type "I Have It." , you will not get any results in the search because the file is skipped.

I've seen numerous posts on this topic but the only answers available are
   1) Register all types -impossible.
   2) Use a third party search product.  Yes there are some that work.
   3) Batch file from the command line that can't be integrated effectively into Winwods explorer.

I have also seen numerous posts from people thinking they have a solution for the existing Winwods 7 Search engine that when tested do not work.

I understand technically why searching for a string in all files is inefficient, so no explanations of why this is "a feature" please.  Has anyone found a way to do this in Windows 7 explorer?

tryx3 say: Surely that is just what the Tools menu, Folder options, Search tab, Always search file names & contents setting is for, isn't it?

the larch say: Unfortunately, that setting can't override the limitation that we are only permitted searching the contents of file types that are explicitly listed in the Indexing Options settings, even if we turn off indexing altogether.  We are able to add others manually one at a time, but we can't simply specify that we'd like to try to search contents of all file types (probably because trying to do so for many types is usually not practical due to the way they may be encoded, and would certainly be impractical if used in conjunction with indexing).  Also, there is no way to specify that we'd like to try to search the contents of extensionless files.

ed_derz say: I would like to see a solution "Native" to Windows 7 Explorer.  However, I have confirmed an alternative that Windows Professional users might find acceptable.

You can use Windows7 "XP Mode"'s Virtual machine and use the older Windows Explorer.  After configuring it to "ignore registered extensions" in file searches you can use it to search your Windows 7 "C Drive" (or any other).

So once you launch XP Mode under Windows 7, configure XP Search as follows:

1. Click Start, then click Search.

2. Click Change preferences, and then click With Indexing Service..

3. Click Change Indexing Service Settings.

4. On the toolbar, click Show/Hide Console Tree.

5. In the left pane, right-click Indexing Service on Local Machine, and then click Properties.

6. On the Generation tab, click to select the "Index files with unknown extensions" check box, and then click OK.

7. Close the Indexing Service console.


While I haven't gone this far, I assume you can create a shortcut in the XP VM That opens Explorer in the Windows 7 Root drive.  This shortcut would also be available in Windows 7.

This works for those with Windows 7 Professional who have installed XP Mode or willing to do so to search for phrases in unregistered file types (or files with no extensions).
