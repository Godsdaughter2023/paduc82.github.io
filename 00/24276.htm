a. user say:

Hi,

To import .nk2 files into Outlook 2013 or Outlook 2016, follow these steps:
Make sure that the .nk2 file is in the following folder:
%appdata%\Microsoft\Outlook
Note The .nk2 file must have the same name as your current Outlook 2013 or Outlook 2016 profile. By default, the profile name is "Outlook." To check the profile name, follow these steps:
Click Start, and then click Control Panel.
Double-click Mail.
In the Mail Setup dialog box, click Show Profiles.
Click Start, and then click Run.
In the Open box, type outlook.exe /importnk2, and then click OK. This should import the .nk2 file into the Outlook profile.

Note After you import the .nk2 file, the contents of the file are merged into the existing nickname cache that is currently stored in your mailbox.
Note The .nk2 file is renamed with a .old file name extension on the first start of Outlook 2013 or Outlook 2016. Therefore, if you try to re-import the .nk2 file, remove the .old file name extension.

webbrewers3 say:

The answer is yes of course you can. You "own" your domain and can do what you like with it. But first, disregard any instructions from Msft. - they don't follow industry standards for domain records. 

Make note of the dns records in your O365 account (if you can find them)

Look your domain up at https://who.is

If the nameservers (NS) reference "microsoft" your domain records are hosted at microsoft

You need to change that by going to Melbourne and selecting the default option for nameservers.

Once that's ready, you can begin adding dns records

Get the ip address from weebly and add that as an A record.

Get the MX record noted earlier from Msft and add that as an MX record to keep your email running.

If you don't plan on continuing to use O365 email, you'll need to find another email host of course. That host will provide the new MX record if needed.

That's basically it.

I'd also suggest you take this opportunity to look into transferring your domain away from Melbourne who are very expensive compared to other domain registrars, like GoDaddy. If you transfer, I think you can ask GD to set the dns records up for you.
