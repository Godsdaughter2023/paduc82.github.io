Excel for Mac 2016 workbook with password protection leaves folders - Microsoft Community

yusuf muraben say:

Excel for Mac 2016 workbook with password protection leaves folders and files when saving on a network share

Hi 

This feels like a bug to us.
We have been able to reproduce the error from a client on our machines.
To re-create the error, make a password protected xlsx file with Excel 2016 onto a network share using SMB, CIFS or AFP.
When you open the file with Excel (sometimes it doesn't happen first time but subsequent attempts reveal the error) and then press 'save' a temporary folder is created with the filename and a random suffix - e.g. myfile.xlsx.444d3a23-JGnhgN.
Several folders are created every time 'save' is pressed.
In normal operation the folder is created temporarily and a few seconds later it is deleted.
In the error, the folder is not deleted as normal and there is often in at least one folder a file that begins with .smbdelete or .afpdelete etc depending on the share connection.

This file can be difficult to delete on the share. 
On Windows shares we get errors as follows: cannot delete the .smbdelete file - it gives permission issues
When we check the security and ownership of the file it can give errors such as: 
'the permissions are incorrectly ordered, which may cause some entries to be ineffective' and 'you do not have permission to view this object's security properties. Try taking ownership etc.'
If we take ownership and propagate permissions we can then delete the file.
If we wait a while without modifying ownership or permissions, we can eventually delete the file (at least a few minutes)

This does not happen for xls files or for xlsb files. Only for xlsx files that are password encrypted, and sometimes not on the first try (on the first try it behaves normally and the temporary files and folders are deleted). Sometimes to reproduce the error we have to open a file with the same name in another folder then back to the first password encrypted xlsx and the error then shows.

So far it looks like it is most easily reproducible for password encrypted files.

We have tried many workarounds, different file protocols (smb, affp, cifs), different share locations (OS, permissions etc) and resetting permissions.

It feels like a bug in the way OSX writes to shares:
From https://opensource.apple.com/source/smb/smb-759.40.1/kernel/smbfs/smbfs_smb.c

* We have an open file that they want to delete. Use the NFS silly rename

The Pending DNS issue can happen after signup if the DNS entries are not set properly or if they do not match. The DNS errors include: 

  • Failure to propagate to the internet from the registrar. 
  • Incorrect record
  • Mixed record – Microsoft can be the only domain on the MX record
  • Missing record

To rectify this error, please follow the instructions on the link below.

 

Handling issues with MX Record or Pending DNS Configuration error

 

Note: please ensure that the MX record that you are using is in this format: xxxxxxxxxx.pamx1.hotmail.com. If it’s in xxxxxxxxxx.msv1.invalid format, please replace it with xxxxxxxxxx.pamx1.hotmail.com format.
