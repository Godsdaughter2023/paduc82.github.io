Hi,

This can happen if the installation media is not created properly for the type of Bios in use, or is selected wrong at boot...

UEFI Bios requires that the Flash Drive be GPT\FAT32, and Legacy Bios requires MBR\NTFS...

The Media Creation Tool does not always get it right, especially if created on a different machine than it is being used on...

You might try just saving a ISO and using a free tool called RUFUS to create a flash drive... It provides options for either Bios type..

Even with the correct media, you may need to go into "Boot Options" at the manufacturer's screen to choose "UEFI USB", or simply "USB" as a for instance...

This kind of varies by motherboard manufacturer..

If possible, it would be best to use a ISO of Windows 10 Creators  (1703) 15063 or earlier...

They can be a it hard to find...

Some of the newer versions have issues with some of the command prompt commands...

You might look at heidoc.net for a downloader that redirects to a Microsoft server...

I've reviewed the other similar posts on this issue and seen a number of  proposed fixes, some overly-simplistic and some very convoluted like the above. None of them worked............

......................until I stumbled on this simple solution which immediately restored sound to my pc:

-----------------------------------------------------------------------------------------------------------

1. Select IDT High Definition Audio CODEC on Device Manager.

2. Select Driver tab.

3. Click Update Driver... button.

4. Select  -> Browse my computer for driver software.

5. Select -> Let me pickup a list of device drivers on my computer.

6. Select  High Definition Audio Device (Not  IDT High Def...  this is important.) and click Next.

    Appears Warning message, but reply Yes.

7. ! Windows has successfully update your driver software.
