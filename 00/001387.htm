stefan blom say:

If you want to change the setting for default tab stops, you have to do that in your template. Open the template as a document, make the change, save and close the template file.

If you are not using a specific template, your documents make use of the default Normal.dotm template which you can open quickly by using the following macro "trick": Press Alt+F11 to display the Visual Basic Editor. (Depending on your keyboard, you may have to press Alt+FN+F11 or Alt+F-LOCK+F11 to make the function keys work.) In the Editor, click View | Immediate window. Type in

NormalTemplate.OpenAsDocument

and press Enter. The Normal template opens in a different window (and you should now close the Visual Basic Editor). Make the desired changes, save and close.

The new settings for default tab stops apply to future documents that you create.

phillip m jones say:

No bug.

Recent Mac OS(X) versions support a security feature called "Sandboxing." Office 2016 is the first Microsoft application to use 100% of Apple Code which uses "Sandboxing." Files written to the system are placed within "Containers" (Containers and Group Containers). How Sandboxing works is that the system when components are placed within theses containers, they are scanned, and anything within that container(s), are frozen and prevented from affecting any files and applications remaining. This procedure prevents any Malware from affecting the operation of the computer, by Malware. The very nature of this slows down the process.

Recently Microsoft came out with 64-bit applications, and that is supposed to help. However, the biggest help would be to go to Solid state hard drives (or SSDs).  Also regardless of what people or articles tell you, you need a minimum of 16 Gb of Ram or maybe even more (32 Gb maybe even 64 Gb), for fast operation so that this Sandboxing can be done all inside the Processor and System RAM instead of the scratch disk available on the Hard Drive.

jllatham say:

You can certainly set up buttons on the worksheet to allow the user to easily adjust the zoom.  In VBA the zoom can be from 10% to 400% and those are expressed as integers as you'll see in the code, so 10% zoom =10 and 100% zoom =100.

The code below is from an actual project where we did exactly that - we used 2 shapes: a big PLUS symbol and a proportionately sized rectangle and 'attached' these macros to them.  These change the zoom factor by

'limits for zooming in Excel are from 10% to 400%
Private Sub ZoomIn()
  'increment enlargement in 10% steps to maximum of 400%
  Dim currentZoom As Variant
  ActiveSheet.Unprotect Password:=worksheetPW
  currentZoom = ActiveWindow.Zoom
  If currentZoom + 10 > 400 Then
    currentZoom = 400
  Else
    currentZoom = currentZoom + 10
  End If
  ActiveWindow.Zoom = currentZoom
  sheetZoom = currentZoom
  ActiveSheet.Protect Password:=worksheetPW, DrawingObjects:=False, Contents:=True, Scenarios:=True
End Sub

Private Sub ZoomOut()
  'reduce enlargement in 10% steps to minimum of 10%
  Dim currentZoom As Variant
 
  ActiveSheet.Unprotect Password:=worksheetPW
  currentZoom = ActiveWindow.Zoom
  If currentZoom - 10 < 10 Then
    currentZoom = 10
  Else
    currentZoom = currentZoom - 10
  End If
  ActiveWindow.Zoom = currentZoom
  sheetZoom = currentZoom
  ActiveSheet.Protect Password:=worksheetPW, DrawingObjects:=False, Contents:=True, Scenarios:=True
End Sub

jllatham say:

Now to the question of automatic zooming.  You may be able to add this to your ThisWorkbook event handler to get it to work.  It basically looks at what cells actually have data in them and zooms the view of that sheet so that everything from A1 to that cell is squeezed onto the screen.  See http://www.contextures.com/xlvba01.html#Workbook

Private Sub Workbook_Open()
  Const mainSheetName = "Sheet1" 'change as needed
 
  Dim mainWS As Worksheet
  Dim lastCell As Range
  Dim lrw As Long
  Dim lcol As Long
 
  Set mainWS = ThisWorkbook.Worksheets(mainSheetName)
'find address of last truly used cell on the sheet
  On Error Resume Next
  lrw = mainWS.Cells.Find(What:="*", _
   After:=mainWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByRows, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Row
  On Error GoTo 0
  On Error Resume Next
  lcol = mainWS.Cells.Find(What:="*", _
   After:=mainWS.Cells(1), _
   Lookat:=xlPart, _
   LookIn:=xlFormulas, _
   SearchOrder:=xlByColumns, _
   SearchDirection:=xlPrevious, _
   MatchCase:=False).Column
  On Error GoTo 0
  'make the sheet active so we can zoom in on selection
  mainWS.Activate
  mainWS.Range(Cells(1, 1), Cells(lrw, lcol)).Select
  ActiveWindow.Zoom = True
  mainWS.Cells(1, 1).Select ' go to A1
End Sub

If you want to use a password on the sheet, then just change the password:= portion of those statements to read

With it all formatted up you can continue on to assigning a macro to run when it is clicked (you can do these steps in either order:  format/assign or assign/format).

If the shape gets unselected, you can get back to the set options stage by right clicking on it at the very edge of it, get it selected and move the cursor towards the lower right edge until it becomes a small + symbol then right-click to get the options list up again.

IMPORTANT: in the code examples above, they all start with the words

Private Sub

for this step, the word Private needs to be removed - if you don't, then the macro will not appear in the list of available macros to assign to anything.  Edit them to remove the word Private - editing within the VBA editor is just like working in a simple text editor like Notepad.

So you get the options list up and choose Assign Macro... you'll get a dialog like this:

Notice that it offers you a default macro name, ignore that and instead choose the one you want

and click the [OK] button.  The dialog will close and you're good to go.

Now, at this point you can go back into the VBA Editor and put the word Private back in front of the word Sub at the start of the assigned macros.  What this achieves is to remove the macro from the list of macros available to be assigned to shapes but more importantly, it removes it/them from the list of macros that the user can choose to run from either the [View] or [Developer] tab.  So they can't choose a macro to run when the wrong sheet is in use - it can only be executed by clicking the button which presumably is on the page that needs to be in use when the macro gets used.
