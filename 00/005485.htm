Rich's code will give you a set of PNG images (or JPG, or whatever you specify) at whatever resolution PowerPoint defaults to, usual about 960 pixels wide.

You can also use the Slide.Export method, which lets you specify the resolution you want.

Dim oSl as Slide

For Each oSl in ActivePresentation.Slides

oSl.Export ActivePresentation.Path & "\" & _

    "Slide_" & Cstr(oSl.SlideIndex) & ".PNG", _

    "PNG", 800, 600

Next

 

That should give you an 800x600 PNG of each slide, named Slide_1.PNG, Slide_2.PNG and so on, saved to the same directory as the presentation itself.

Sub Newton()
'   Value guess for C48

    Dim g
    Dim N
    Dim D, t1, t2, t3
    Dim J As Long
   
   
    'Start Guess
    g = 0.1
   
    For J = 1 To 10
    t1 = (50931525 * Sqr(15 / 2)) / (Sqr(g) * (64604536416# * g + 853616325 * Sqr(30) * Sqr(g) + 36250000))
    t2 = (((853616325 * Sqr(15 / 2)) / Sqr(g) + 64604536416#) * (50931525 * Sqr(30) * Sqr(g) + 4250000))
    t3 = (64604536416# * g + 853616325 * Sqr(30) * Sqr(g) + 36250000) ^ 2
   
    N = -g + (50931525 * Sqr(30) * Sqr(g) + 4250000) / (64604536416# * g + 853616325 * Sqr(30) * Sqr(g) + 36250000) - 1 / 10000000
    D = t1 - t2 / t3 - 1
   
    g = g - N / D
    Debug.Print FormatNumber(g, 15)
    Next J
   
    [C48] = g

End Sub

a. user say:

With a localised version, the most likely problem is that the {=} fields will not work because the list separator and/or decial point characters are localised - e.g. with Windows English regional settings they are "," and ".", whereas in the Windows French (France) seetings they are ";" and ","

So e.g. a field like 
{ =MOD(3,2) } would need to be { =MOD(3;2) } in the French version.

Unless you can be sure that the localised characters can only be one or two things, it would be difficult to do anything about this without e.g. using VBA, but in the case where you know it can only be those English/French standards, you may be able to use something like the following to set the list separator

{ IF { =1/2 } = "0,5" "{ SET LS ";" }" "{ SET LS "," }" }

(you can junk some quotes and space if you prefer)

then use e.g.

{ =MOD(8{ LS }3) }

But you can't do a similar thing with the DP directly - I think if you need numeric literals with decimal points, you would either have to do

{ IF { =1/2 } = "0,5" "{ SET LS ";" }{ SET TWO5 "2,5" }" "{ SET LS "," }{ SET TWO5 "2.5" }" }
or

{ IF { =1/2 } = "0,5" "{ SET LS ";" }{ SET DP "," }" "{ SET LS "," }{ SET DP "." }" }{ SET TWO5 "2{ DP }5" }

It would be simpler and more useful if Microsoft gave us syntax that worked whatever the localisations, but I very much doubt that will ever happen.

You can do more testing to discover what the various localised characters actually are, but since in some cases you'd probably have to write fields that would have invalid syntax in a t least one localisation, you
 a. have to hide their results and if you are doing a MailMerge
 b. either have to accept that you will get error popups for each syntax error, or suppress messages during the merge.
