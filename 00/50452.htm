a. user say:

In your message you type "Chkdsk \f" and the correct method would be "chkdsk /f" with the forward slash for chkdsk options.

Here is one of many articles that may help you understand how chkdsk works and how to use it to your advantage.  It is good to know how to do it, but is is also good to understand it.

http://vlaurie.com/computers2/Articles/chkdsk.htm

If you perform a Google search for:

xp chkdsk

You will find several short articles you can read in various levels of detail and scope.

The chkdsk can only fix problems when it runs when XP is not fully loaded yet.  That would be on the next reboot of XP before XP is fully loaded or from the XP Recovery Console.

Chkdsk can only fix problems when the volume is locked and in what Microsoft calls a "static state".  Two ways to be sure you are in a static state is to run chkdsk from the Recovery Console or to schedule a chkdsk to run the next time the system restarts by responding in the affirmative to the question:

Would you like this volume to be checked the next time the system restarts?

If the chkdsk operation you run includes error checking and recovery options (if you choose to let it fix things) and you let chkdsk run automatically on the next reboot, you might be interested in knowing what if found (the results will disappear from the screen when it is finished).  

Here is how to do that.

 

When chkdsk runs automatically on a reboot either because XP thinks it needs to (perhaps your NTFS volume has been marked as dirty) or you chose to do it yourself, the results are shown in the Event Viewer Application log.  You need to look there for the chkdsk log and any problems after chkdsk has run.

To see the Event Viewer logs, click Start, Settings, Control Panel, Administrative Tools, Event Viewer.

A shortcut to Event Viewer is to click Start, Run and in the box enter:

%SystemRoot%\system32\eventvwr.msc

Click OK to launch the Event Viewer.

Look in the Application log for an event sourced by Winlogon, something like:

manage-bde -protectors -disable c:
5/ Then exit and reboot. The computer should now successfully boot Windows. Once there, use the BitLocker control panel to resume BitLocker protection.

samc1 say: I think there is some confusion between Library and folders. If you can find your documents like Excel files indicated, all you need to do is point to a folder of interest, right-click on it and then click on the Include in Library command and select the desired Library (which most likely in this case is Documents library). You can also do it from the other end: Right-click on Documents Library > Properties and then click Include a Folder button. Hope this helps.
