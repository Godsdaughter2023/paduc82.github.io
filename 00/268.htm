Hi Peter,

I believe that relates to the axiom "Word doesn't know what a page is.". If printing directly from the main document the merge is being done one record at a time. Pagination is imposed on the fly, so Word has no idea how many 'pages' will be generated. Along with that, Labels actually are a Table which is being built on the fly as well.

When merging to the printer the range of records has to be controlled prior to printing, either:

  • in the record source,
  • by using Word's Sort/Filter feature, or
  • by using the Merge Range control [But it appears that the job can't be constrained to partial sheets. IOW, if there are 30 labels/sheet & you specify a Range that comprises only 1 record, all 30 records on the sheet containing that 1 will be printed.]

What can we hope for in Win 10 re: VSS?

Volume Snapshot Services for managing shadow copies are clearly still a part of Windows (as well they should be; they solve several problems marvelously well).

However, the level of support that will be available in Windows 10 is unclear.

  • We see, for example, that the Previous Versions tab has reappeared in File Explorer.  Thank you for this, Microsoft! 
     
  • But there is no GUI for configuring/scheduling a recurring VSS-integrated System Image backup (as there was up to Windows 8).  This can still be accomplished in a bit more geeky way by scheduling an appropriate wbadmin command in the Task Scheduler, but that's not really going to be accessible to a lot of users.
     
  • Also, the vssadmin command is hobbled as compared to the one available to users of Windows Server operating systems.  You can't, for example, manually create a shadow copy with the vssadmin.exe that is provided with Windows 10.

I guess my question is this: 

Can we hope for the delivery of more complete VSS functionality in Windows 10, including some of the missing features I mentioned above?

And, I'd like to make a suggestion: 

As long as Previous Versions is being resurrected, it would be really nice if a fully featured browser could be provided (e.g., maybe a kind of "view" in File Explorer showing folder trees and files for a particular snapshot, similar to 3rd party software Z-VSScopy).

This kind of stuff may not interest portable device users much, but folks running desktop computers care how well their backups are integrated with Windows.

-Noel

The settings you should have in the Group dialog are as follows:

under Auto

Starting at unchecked and 27/02/2012

Ending at checked and date will be the largest one in your data.  (as this is for one month only then 30/03/2012 is the last working day)

What this does is force the start day to be a Monday and lets Excel determine where to stop.  If the data sheet is repopulated with the following months data then the PivotTable will still know to start on a Monday and be able to adjust the end date according to the new data.

By checking the Start at under Auto refreshing the pivot will allow Excel to recalculate where to start grouping by seven days, based on the data in the source list.

If you need/want to change the label for the first group 27/02/2012 - 01/03/2012 you can amend it but you may have to re-edit it each time the pivot table is refreshed.

graye say:

What is an SSL?

SSL is the name of the technology that encrypts the conversation between your PC and a web server.   It's what keeps your web traffic private.   Each web site has the option to provide a normal "http://www.xxxxx" web address plus a "https:/www.xxxx" web address.   The extra "s" shows that the connection is considered secure.

There are a lot of technical details that I'm not sure you're interested in.... so I'll just refer to this article:  https://en.wikipedia.org/wiki/Transport_Layer_Security
