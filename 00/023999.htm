This is the cause and how to fix it. I've had the exact same problem, with the same machine (Macbook Pro).

Cause: In a distant past, I created a notebook while signed in under a certain email. One day, somehow, I got into OneNote signed in with a different email. The old notebook was still there and visible, but I had no "permission" to sync it.

Solution:

1. Update OneNote to the newest version

2. Then go:  File - Open Notebook. In the sidebar, click the top icon. It will show the email you're signed in under right now. Sign out of that email, close OneNote

3. Go to Onenote.com (online), and try signing in with another email (the mission here is to sign it with the original email that you were signed in under when you created the notebook that won't sync). If you find it, the notebook will be there (online).

4. In the OneNote App, close the screwy notebook (File - close this Notebook). Only do this after you're sure you can see it and open it online.

5. Open it online, then click "Open in OneNote".  This will actually open the OneNote app, with the notebook now syncable, without that annoying error. Sign into the One app with the same email, if necessary.

Warning! Project is a dangerous application. We normally suggest a new user get at least 2 days of hands on training before attempting to use the application.

Second, when creating a schedule you will not normally enter dates, project will calculate those for you based on task durations and task linkages (i.e. sequential flow of tasks). Issue solved. Bug found in AMD Catalyst 15.200.1023.0 which prevents XAML apps from working and can cause flickering prior to login

Solution:

1. Access CCC > Power > Switchable Graphics Global Settings > Plugged In > set to Optimize Performance

Do not set to Maximum Performance as this will cause a bug which disables XAML apps and can cause a login issue.

2. > Power > Switchable Graphics Application Setting > reset settings via "Default"
