robert aldwinckle on forums say:
Diagnose Connection Problems


You need to use that button and find out what the troubleshooter says.   If it says nothing found then close it and allow IE to do an automatic refresh.


If that workaround doesn't change your symptom try to supplement the help and diagnostics that the troubleshooter is giving using this procedure.

Before starting the troubleshooter (e.g. a second time), switch to a Command Prompt (aka cmd.exe window) and enter:

ipconfig   /flushdns

 

Switch back to the IE window to start the troubleshooter and follow its instructions up to the point you would be closing it.

 

Switch back to the cmd window and enter:

 

ipconfig  /displaydns

 

If you notice that the host name you are trying to access is represented only by a CNAME entry you should try to follow that chain of lookups using ping commands.   That is the most common problem cause I believe.   E.g. I think the troubleshooter just stops when it finds an entry for the target host name not considering that there may be insufficient accessibility to the other names it chains to.   Try to ping the other host name(s) to make sure that their lookups are loaded too.

In fact, recently I have become aware that another possible problem people may be having could be due to IPv6 addresses being looked up before IPv4 addresses so it may be a good idea to force an IPv4 lookup with your ping.

E.g.  facebook is an example of the latest symptom, so if you see that the troubleshooter has left an IPv6 address in the dnscache I would clear the cache again (e.g. with another  flushdns)  and then issue a ping this way:

ping  -n 1  -w 1  -4  www.facebook.com

(Note that that is just an example, replace facebook with the name of the host which you have a problem with.)

Then do another  displaydns  to make sure you know what you got (and how much time you have to use them, e.g. by their Time To Live values, in seconds).

Then let the troubleshooter close and IE's Refresh be tried with the amended cached lookups.

 

If you need more help capture the output from the last displaydns in the cmd window and paste it in a reply.   Hint:  you can use right-click, Select All  and  press Enter to capture the whole cmd window's buffer to ClipBoard.

Tip:  since some of the Time To Live values you may be dealing with may be small enough that the speed of your typing could make a difference about their usability by IE note that you can reuse typed commands by pressing F7 or just Cursor commands.

 

Good luck

Robert Aldwinckle

andreas killer say:

The AVERAGE formula is a bit oversized, so I like to explain how such formulas work and how you can design your own.

Let us start with a new file and a simple scenario, the numbers from 1 to 10 in the cells D3:M3

The 1st step is to understand what MOD calculates, add this forumla

D4:  =MOD(COLUMN(),5)

and drag to M4. As you see we get 4,0,1,2,3,4,01,2,3 as result and if you click Formulas\Evaluate Formula you can see the steps what Excel calculate.

Okay, the 4 at the beginning is not nice, let us shift the 0 to the left, means mathematically  we have to subtract 4 (the number of the column):

D5:  =MOD(COLUMN()-4,5)

or the same as in your AVERAGE formula

D5:  =MOD(COLUMN()-COLUMN($D3),5)

and drag to M4.

Now we use many cells, and if we write an array formula into all cells, we can see the result of the formula in parts.

D6:M6:  {=MOD(COLUMN(D3:M3)-4,5)}

Select D6:M6, write the formula =MOD(COLUMN(D3:M3)-4,5) into the formula bar and press CTRL-SHIFT-ENTER.

If you did everything right, the result is the same as in row 5.

The next step is to pick every 5th value, but before that let us "mark" each of these value using IF:

D7:M7:  {=IF(MOD(COLUMN(D3:M3)-4,5)=0,1,0)}

If you compare row 7 and row 3, you'll see that we have a 1 below the 1 and 6 in row 3, so the next step is to get these values:

D8:M8:  {=IF(MOD(COLUMN(D3:M3)-4,5)=0,D3:M3,0)}

And the final step is to add a function that can combine all returned values from the array formula into one value:

D9:  {=SUM(IF(MOD(COLUMN(D3:M3)-4,5)=0,D3:M3,0))}

D10:  {=AVERAGE(IF(MOD(COLUMN(D3:M3)-4,5)=0,D3:M3,0))}

Uuups? Why did D10 not show 3.5? Because our array formula returns many zeros!

D11:  {=AVERAGE(IF(MOD(COLUMN(D3:M3)-4,5)=0,D3:M3,""))}

AVERAGE ignores "", as GEOMEAN (and must functions) does:

D12:  {=GEOMEAN(IF(MOD(COLUMN(D3:M3)-4,5)=0,D3:M3,""))}

Now you should be able to bring your spreadsheet really in to the future. ;-)

Andreas.

andreas killer say:

{=GEOMEAN(IF(MOD(COLUMN($F3:$NB3)-1,5)=0,$F3:$NB3,""))}

{=GEOMEAN(IF(MOD(COLUMN(F3:NB3)-1,5)=0,IF(F3:NB3>0,F3:NB3,""),""))}

Here is what I found recently and was misled by the result for quite a while before finding out this error. 

In my Excel 2010 spreadsheet, three columns representing Year, Month, day, say Column A, B, and C, respectively.  Column D has the formula =DATE(A1,B1,C1). Thus, convert the three numbers into a Date format internal quantity. 

In Column E,  formula =D2-D1 is entered, the results are shown. The top cell in Column F has the formula =D3-D1, and the result is 60. Both Columns E and F are formatted in "Number" with no places after the decimal point. The difference between to dates should give the number of days between them. 

What is odd is that D2-D1, D3-D2, and D4-D3 all give 30. It's is obvious that D2-D1 and D4-D3 are correct. However, it's not clear why D3-D2 gives 30. Year 1900 is not a leap year. Feb, 1900 has 28 days. Thus the result should be 29, instead of 30.

The general rule for deciding leap year is divisible by 4, and 400, but not 100. Thus, both Year 1900 2100 are not leap years while Year 2000 is.  Interesting enough, Excel gives correct answer for Year 2100, a non-leap year and Year 2000 which is a leap year.

Not a defect, but a conscious design choice in order to be compatible with other spreadsheet that were market leaders at the time that Excel was first introduced.  So, Excel (but not VBA) does indeed treat 1900 as a leap year.

This is explained in http://support.microsoft.com/kb/214019.

PS:  The KB mentions that the problem will not appear when using the 1904 date system (click on File, Options, Advanced, and page down to the option under the title "When calculating this workbook").  But I would not recommend setting that option just to "correct" this problem.

julie fka profjulie say:

Use the Windows Live account that is associated with Zune and Xbox live as your primary account.  This will the live id and password that you enter into the screen the very first time you boot up the device.  Doing so will ensure that you will be able to enjoy the benefits of your Zune Pass and Xbox live account on the phone.

This is how I've handled things on my WP7 and it works great!
