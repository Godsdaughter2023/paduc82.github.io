I haven't used replication in Access 2007/2010, so please understand that I am writing theoretically. 

 

As I understand it, replication in Access 2010 is only supported for database files in the .mdb format, which (in A2010) is the Jet 4.0 format.  Tools such as those from the website linked by Bob Larson that work with the Jet 4.0 format, should work on Access 2010 replicated databases.  So if you don't have a copy of Access 2010 to use in manually unreplicating your database, you might try Michael Kaplan's "TSI Access 2000 Un-Replicator": http://www.trigeminal.com/lang/1033/utility.asp?ItemID=7#7 .  Yes, it says "Access 2000", but the important point is that it is targeting Jet 4.0. 

 

If that doesn't work, you could use the laborious manual procedure described in the KB article, http://support.microsoft.com/kb/290052 , "How to make a replicated database a regular database in Access 2002".  Again, the fact that is says "Access 2002" doesn't mean it won't work for a Access 2010, since the database must still be in the .mdb format used by Access 2002.

 

If you were hoping to actually use replication with Access 2013, I'm sorry but that will not be possible, as support for it was removed completely.  Your only option is to un-replicate the database and design another method to share/synchronize the data.

Having one cell add tehvalue of another cell onto itself - Microsoft Community

peters018 say:

Having one cell add tehvalue of another cell onto itself

I have one cell in my spreadsheet that contains the current value of a loan.  I have another cell that contains the value of the loan payment (which is made once a month).  The expenditures of the spreadsheet are set up to accommodate two pay periods in a month; so one pay period will contain a loan payment, while the other one will not.  When filling in the expenditures for pay period one, for example, and the cell for the loan payment contains a value, I want to deduct that value from the cell containing the current loan value.  If, for pay period two, the cell for the loan payment is zero (no loan payment) I want the value of the cell with the value of the loan to remain unchanged.  When I try to set this up, it gives me a circular reference error.  How do I do this?  
myall_blues say: You can do it using VBA but I wouldn't recommend it, because with such a scenario if you make a data entry error it can be a nightmare to recover from. Better to just use a bit of additional real estate by implementing a third column.
So starting values in row 2, A2 =current loan balance, B2=payment amount, C2=balance as a formula =A2-B2
Then in row 3, in A3 put =C2 to reflect any new balance as the current loan balance, B3 will contain any payment (or not), C3 is again =A3-B3. And so on - copy the formulas downward.
Then it doesn't matter if there is a payment or not as the formula in column C will just treat a non payment as zero.

emerald77 say: I finally found a solution to my problem, thanks to a post by boblarson (http://www.access-programmers.co.uk/forums/showthread.php?t=233924).  Hopefully others can benefit from the same solution.

My revised code is shown below with boblarson's solution in bold.  I also replaced DoCmd.RunSQL with object.Execute (in order to avoid having to call the SetWarnings method).

    Private Sub DisplayPrintFlag_AfterUpdate()
        Dim dbs as DAO.Database
        Dim strSQL As String
        strSQL = "UPDATE WBSStatus SET WBSStatus.DisplayPrintFlag = " & _
                 Me.DisplayPrintFlag.Value & " " & _
                 "WHERE WBSStatus.SPRFNumber = '" & Me.SPRFNumber.Value & "' " & _
                 "AND WBSStatus.PhaseDeliverableSequence Between " & _
                 Me.PhaseDeliverableSequence.Value + 1 & " And " & _
                 Me.PhaseDeliverableSequence.Value + 99
        If Me.Dirty Then Me.Dirty = False   ' removes any locks on the form.
        dbs.Execute strSQL
    End Sub

suzanne s. barnhill say:

I guess you mean you inserted a TOC from the References tab, and it sounds like you must have created a manual TOC. If that's the case, the page numbers won't necessarily update automatically. If you use an automatic TOC, you don't have to create bookmarks or hyperlinks or leader lines; Word does all that for you. See

https://shaunakelly.com/word/numbering/tableofcontents.html For contrast, see http://wordfaqs.ssbarnhill.com/ManualTOC.htm.

When you use an automatic TOC (and I advise starting from Custom Table of Contents... instead of using either of the automatic ones in the gallery, which use content controls), Word updates page numbers when you print or print preview (provided you have the option to update fields at print time enabled), and you can update at any time by clicking in the field and pressing F9.
