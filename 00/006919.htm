How and where is the Zune cover art stored? - Microsoft Community

fred applegate say:

How and where is the Zune cover art stored?

That pretty much says it: How is the cover art stored, and where is it? I want to tag all my files properly, which is really, really hard in Zune, and can't find the art.

Anybody?

Thanks.

cs188 say:

You should be able to copy any image stored on your computer (or from the web), then right-click on an album in your Collection and select "Paste album art".

If you're really nit-picky like me, what I recommend doing is using a free program such as MP3Tag to embed album art into the tags of the files themselves.  Zune can read this image metadata, and this way wherever the file goes, the album art goes with it.  I personally love this method, but it takes a little longer.  You can also adjust all your other tags with this program, as long as you're at it.

Also, if you want to prevent Zune from overwriting / automatically updating metadata and album artwork, you can change that under Settings > Software > Metadata and Ratings, then un-check "Automatically update album art and metadata".  Keep in mind this may prevent some album linking in the Social, however.

(For reference, Zune's art cache is located under C:\Users\username\AppData\Local\Roaming\Zune\Art Cache, but you really can't edit anything there - that's only a reference to all your pasted art used by the Software.)

My responses only come from personal experience.

jay freedman say:

The text from the Convert Table to Text operation will be in a frame if the table was "floating" before the conversion. If you right-click in the table and choose Table Properties, such a table has the Around button highlighted in the Text Wrapping area instead of the None button. This can happen if the table handle (the little square off the top left corner) has been dragged even a little bit.

If you check this before converting the table to text, you can just click the None button and click OK. If you don't notice it until after the conversion, place the cursor on one of the edges of the frame (the cursor changes to a four-way arrow), right-click, and choose Format Frame; in the dialog that opens, click the Remove Frame button. The resulting text will go where the anchor was.

Let's see if your update was successful. Go to the Word menu and choose About Word. Is the version 16.12 or something else? If it is something else, do the following, then test again and reply with the results.

I think the first troubleshooting step to take is: Ensure your Mac and Office have the latest updates. Here’s how to install updates:

 

Make sure your Mac has a working internet connection, then:

 

  1. Update Mac OS X. Open the App store and click on the Updates button. Install the updates.
  2. After, in any Office application go to the Help menu and choose Check for Updates to launch the AutoUpdate app. Let AutoUpdate install updates and then restart your Mac.

 

If you use Antidote, Webex, Silverlight, EndNote, Zotero or other add-ins there is a potential for crashes unless you also update your add-ins to the latest update. Install updates for these programs if you have them.

 

SAFE BOOT

If you already have 16.12 and your add-ins are all up to date, I think the next thing to do is another relatively simple procedure: Perform a Safe Boot. It will take a long time for your Mac to start up in Safe Mode. When you do a Safe Boot Mac OS fixes troubles with your Mac's directory and file system. The font cache is emptied. Only a limited number of fonts are enabled. Applications that run at startup don't run automatically. This allows you to test your app(s) by running them with a minimal set of fonts, an empty font cache, and without potentially conflicting apps running.

 

Once you reach the desktop in Safe Mode try running the app having a problem. Is the problem still there?

 

If the problem is gone: Performing a Safe Boot may have fixed the ailment. Or perhaps one of the fonts that wasn't loaded is bad. Or perhaps some other software utility is interfering. After testing in Safe Mode, do regular restart.

 

If the problem is still evident in Safe mode, you may still have a bad font. Use Font Book to restore system fonts, then test again in Safe Mode.

 

If the problem was gone in Safe Mode but returns during a regular boot: I think the first thing to do would be to use a font utility to validate ALL your fonts. Mac OS comes with a free font utility called Font Book, which you can use for this purpose. Get rid of any font that doesn't pass as clean and green. Then, restart your Mac and immediately empty the trash. If you've gotten this far and still have a problem, then there are more things to explore. But if you get this far you've probably gotten rid of the crash. Either way, please reply with your results.

Ron wrote:

All of this works fine until I attempt to add column C to determine the total number of minutes or hours the employee work for the year.  When it reaches 24, it rolls back to the beginning.  I have tried all the options and suggestions on the Internet to insert different formats such as ; hh:mm; h:mm; [h]:mm, [hh]:mm but nothing seems to work.

Of course, h:mm and hh:mm "roll over" after 24.  By definition, h and hh are 0 through 23 in that context.

The only correct formats are [h]:mm [hh]:mm.  And they work just fine for me.

For example, if A1 is 6:00, B1 is 6:15 and C1 is 48:00,, =A1-B1-C1 displays -48:15 when formatted as [h]:mm and the 1904 date mode is set.

So something is very wrong with what you are doing, or your description is missing something important.
