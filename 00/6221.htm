Down then across won't work for a subreport - pagination in a report is controlled by the main report; the subreport doesn't "know" when the page ends, so it cannot decide when to start a new column. You *must* use across then down.

jllatham say:

The first place to look is in your Recycle Bin to make sure you haven't accidentally deleted it.  If it's there, just restore it and it'll be put back into its original location.

The second place to look would be in the folder where it was stored.  If it's still in your Excel 2003 recent files list, that will show the full path to it.  Go look and see if it actually is in the folder.  If it is, definitely make a copy of it and put that somewhere safe.

If you can't find it where it should be by actually looking in the folder then do a search of My Computer for the complete filename.  If that doesn't turn up anything, do another search of My Computer for *.xls which will show you all Excel files on your computer (be sure and include hidden and system folders in the search).

If that fails, then try to remember when the last time was that you made a copy of it, whether it was to another hard drive, to a thumb drive, to a CD or DVD or anywhere and try to recover and go from there with it.

If you don't already have one, now might be a good time to develop and implement a backup strategy.  You don't want to wake up sometime in the future and find other valuable files, pictures, etc. lost for some inexplicable reason.

ossiemac say:

I am a bit confused. You dimension lngProduct as a string and then assign "oak" to lngInventory that has not been dimensioned. Also it is usual to prefix string variables with str; lng is the prefix for long integers.

The following assigns oak to strProduct and then concatenates the wild cards (asterisks) with the variable.

For your 2nd question, you need to create an array of the  list.

Note that AutoFilter.Range is the generic object for the filtered range so you do not need to specify a specific range.

AutoFilter toggles AutoFilter On and Off so if using it then you need to set AutoFilterMode to False to ensure AutoFilter is off first otherwise the code might turn if off if AutoFilter has been previously set.

To turn on AutoFilter, you only need to specify one cell in the range and AutoFilter is applied to all of the adjacent data. (Called as the CurrentRegion in VBA.)

I am not sure about where you are getting Range("B7"). If from another worksheet then include the worksheet name as I have done in the example.

Sub Filter()

    Dim strProduct As String
    Dim arrFilter() As Variant
   
    strProduct = Worksheets("Sheet2").Range("B7").Value ' in this example "oak"
    arrFilter() = Array("525", "553", "597")
   
    With Sheets("Inventory")
        .AutoFilterMode = False 'Ensure AutoFilter is Off
       
        .Range("A1").AutoFilter 'Turn AutoFilter on
       
        'Set AutoFilter with wild cards
        .AutoFilter.Range.AutoFilter Field:=1, _
                Criteria1:="*" & strProduct & "*"
        
        
         'Set AutoFilter using an array for the list
        .AutoFilter.Range.AutoFilter Field:=2, _
                Criteria1:=arrFilter(), Operator:=xlFilterValues
               
    End With

End Sub
