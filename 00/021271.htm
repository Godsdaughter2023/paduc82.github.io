robert aldwinckle on forums say:

i just tried to install a PC Game via steam and it says i am not connected to the internet.


Doesn't sound like an IE issue.   More of a Networking or Games or Programs one.  ; )

Another program that often shows the same symptom is  Help and Support, e.g. press Win-F1 and use its Support menu.   When you have your problem symptom does H&S detect an Internet connection or not?...

Actually that may be a bad example.   I was thinking of how it looks (uncustomized) under XP and don't know what you would be getting that is equivalent under Vista (even if yours was just an "Out-of-the-box experience", uncustomized by your manufacturer).  I just tried it on my W7 but found it is completely different, much customized by HP, for example.

In any case, using my XP H&S experience as an analogy for a possible explanation for your case, one problem seems to be that some apps think they are connected through a LAN and therefore expect LAN performance criteria.   So, for example, if a DNS lookup does not get satisfied instantly, the unfortunate conclusion by that app is that there is no Internet connection.   In my case, with XP (and I think with OOB W7--before HP changed it) I had a PPPoE bridge connection and the only time that H&S was happy was if it was the very first connection.   I'm guessing that there is a different set of timing criteria in that circumstance which let H&S understand that there really was a usable Internet connection.

 

The wirless connection is on and signal is fine

Can you try a different connection, e.g. direct to modem instead of wireless?  That could let you try the same sort of workaround that worked for me with H&S.

Alternatively, perhaps you could avoid a DNS issue by doing a manual lookup (e.g. using nslookup) and then trying to do your install via the IP address you found?

If these guesses don't help the best tack to take would be to do some diagnosis of the network exchanges which are involved.   To do that you could use either NetMon or WireShark.   Or, if  IE  is really involved you could also try using an HTTP tracer such as Fiddler2.   ; )

 

HTH
