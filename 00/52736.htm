Try Clean Installing the Printer: 

Power up the printer but unplug it's USB cable from PC.

Right click Start button in Windows 10 to open Programs and Features, Uninstall all the printer software. Restart.

 Next copy and paste in Start search box: printui /s /t2  

 Right click result to Run as Administrator.

Open the Print Server properties,  Remove the printer’s drivers for this printer and for any other  printers  you don't use presently.  If you get a message about access denied or the printer being in use and you cannot find the driver or any installed program that would account for this, you may restart the computer in a diagnostic startup and then use the printui command to remove the driver package(s) (see How to perform a diagnostic startup in Windows). You must restart back to normal mode before you will be able to install the printer again.

Now in Start Menu open Devices and Printers, right click the printer to delete the Printer and any scanner or fax of the same name.  Restart PC. 

Go to the printer model’s Support Downloads webpage to download and install the basic driver.  If a wireless printer you will need the full software package to get it set up fully.  If you want to be able to scan from the printer top to the PC then you'll also need the full package. 

Extract Printer software, right click Setup file to Run as Administrator.   Follow the setup screens closely as you install the software.  For USB it should prompt when to plug in the printer.

Revo

It has long been established good practice that the primary user of a Windows computer should set up a second user account using the built in Administrators Account for day to day use. When this is done the Administrators Account should always be available to create a new account if the second user account corrupts. Many users encounter problems because they do not create the second user account.

 

Each user account is accessed by a single password or pin. You can use a blank password, although this practice is not generally recommended. You can use a Local Account i.e. an account not linked to an email address and Microsoft Account or a Microsoft Account which is one linked to an email address and Microsoft Account.

amoraluv4ever say: need more details and the steps u took to find the problem

gavin haley say: I think I found the problem, I was having the same issue, then someone told me what was wrong, and it fixed it for me on Windows 8.

The issue is from files not being installed properly because the game was meant for the Windows XP era, which is quite different from Windows 8. The way I fixed it was to go to the "setup.exe" file and run that in compatibility mode for Windows XP SP3 with admin privileges *before* installing the game.

Just in case you aren't sure how, right click "setup.exe," click properties, go to the compatibility tab, and check the boxes for "run this program in compatibility mode for" (I used Windows XP SP3, but others may work as well) and "run this program as an administrator." This causes the files to be installed to the places that they would be if you were using windows XP, which is how the game is coded to read files. Windows 8 likes to move files around during installation to where it thinks they need to go instead of where programs want to place them.

Note: If you are installing from a disc, you may need to make a digital copy of that disc to modify setup.exe, then reburn that to a blank disc and use that instead of the official disc to play.

Hope this helps!
