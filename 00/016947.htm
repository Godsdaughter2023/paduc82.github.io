Hi Guys,

I fixed the problem and will post it here as it may help someone else.

The problem was that the dotm template had references to my normal template which contains my macros.

To create the dotm template successfully I had to open Word

File/New/My templates

Then in the dialog choose Blank Document (macro enabled) and Choose Template.

The reference to Normal was then missing. 

I then added my macros and did Ctrl+S in the VBA editor and saved as a dotm.

I then added the button to the Quick Access Toolbar, specifying the dotm I'd just saved. If not already saved then only the "For all documents" is available.

I then did File/Save As to and overwrote as a dotm with the same name.

alex sun msft say: Hi Hendri,

In a Hybrid environment, the mailboxes can be located in on-premises and cloud. Dirsync syncs the user objects from local AD to the cloud. An synced object can only be managed on the sync sources (here your on-premises AD and Exchange). When you want to decommission your Hybrid server, please move all the on-premises mailboxes to the cloud. After the migration is completed, they will be converted to mail enabled users automatically. You don't need to convert the users or mailboxes manually.


As mentioned above, you cannot manage the objects in the cloud if Disycn is enabled. If you want to manage them in the cloud, you need to deactivate your Dirsync. If you want to manage the objects still, you should keep Dirsync and at least one Exchange server. Here is an article that explaining why you should have at least one Exchange Server on-premises.

alex sun msft say:

Hi Hendri,

No, manually converting the users into cloud users is not an option in Exchange Hybrid.

When you want to create a cloud mailbox in Exchange Hybrid, the suggested way is to create a remote mailbox for Office 365 in your on-premises Exchange server. This will also create a mail-enabled user in on-premises Exchange with a targetAddress*** Email address is removed for privacy *** that is used
to configure the coexistence with Office 365. If you create the user and mailbox directly in Office 365 admin center, the on-premises cannot recognize him and the targetAddress is missing, which will lead to mail flow and autodiscover issues between on-premises Exchange and Office 365.

In Exchange Hybrid, the on-premises mailboxes and data need to migrated to the cloud. When all the mailboxes are in the cloud, you can decommission your on-premises Exchange server and Dirsync. After Dirsync is disabled, the source authority will be changed to Azure AD and the user status is In cloud, and you can manage the users and mailboxes in Office 365 Online.

If anything is unclear, please let us know.

Alex
