balaji kundalam say:

IP address with 169.254.xxx.xxx means that DHCP (Dynamic Host Configuration Protocol) delivery is affected. DHCP  servers allocate the IP address to the computers, in your case local IP address.

There are three chances for the issue to occur:

1. Wired AutoConfig (for wired connectivity) or WLAN AutoConfig (for WiFi connectivity) service is not running.

2. Bad network configuration of network resources.

3. Bad network cable.

Solutions:

1. The following link will be helpful for starting Wired AutoConfig service (look for WLAN AutoConfig service in the same list and repeat the process involved in starting as per the link):

http://computerstepbystep.com/wired_autoconfig_service.html

2. The following link will be helpful in resetting TCP/IP settings. Make sure the options are set to automatic:

http://windows.microsoft.com/en-us/windows/change-tcp-ip-settings#1TC=windows-7

3. Check with some other ethernet/network cable. Also, check the current cable in some other computer for the cable's working. Try replacing the cable and then check.
