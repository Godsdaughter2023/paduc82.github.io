qa guy commelec say:

It sounds like you are using the same table to store data as is your selection source.

A combo has a Control Source and a Row Source.  The row source is where the information comes from that is displayed as choices.  The control source is where the selection is stored.  The query for your form pulls records from the table that stores the information and the sort will display the stored data in a given order.  That order does not control the sort of the data presented as available choice items.  You can have a second query for the Row Source or a SQL statement that includes ORDER BY as the Row Source.

Build a little, test a little.

dare-devil say:

Dewey's pretty much got the answer you need here.  Your combo is getting data either from a table or a query.  If you are supplying the table in the Row Source, you can't expect the data to in any particular order.

To ensure the order of your combo list, use a query in the Row Source and define the order by statement in the SQL. 

SELECT [FIELD1] FROM [TABLE1] WHERE [CRITERIA] ORDER BY [FIELD1]
