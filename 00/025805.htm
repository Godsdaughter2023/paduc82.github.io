ron coderre say:

The COUNT function only returns the count of numeric cells.  Excel has no native function for counting cells of a certain color.

Perhaps this User Defined Function (UDF) will help:

• ALT+F11...to view the Visual Basic Editor
• Select your workbook name in the VBA-Project window
• Insert.Module
• Copy the below code and paste it into that module:

Function CellColorIndex(rRng As Range) As Variant
    Dim cCell As Range
    Dim arrRng() As Variant
    Dim lngRows As Long
    Dim lngCols As Long
    Dim lngRCtr As Long
    Dim lngCCtr As Long
   
    Application.Volatile
   
    lngRows = rRng.Rows.Count
    lngCols = rRng.Columns.Count
   
    ReDim arrRng(lngRows - 1, lngCols - 1)
   
    For lngRCtr = 0 To lngRows - 1
        For lngCCtr = 0 To lngCols - 1
            arrRng(lngRCtr, lngCCtr) = rRng.Cells(lngRCtr + 1, lngCCtr + 1).Interior.ColorIndex
        Next lngCCtr
    Next lngRCtr
   
    CellColorIndex = arrRng()
End Function


• Select the worksheet with the colored cells
• Assuming cells A1:B20 may have cells colored orange (color index: 44)

This formula returns the count of cells with an interior color index of 44:
=SUMPRODUCT(--(cellcolorindex(A1:B20)=44))

Note: To determine the color index of any particular cell....eg cell B1...try this:
=CELLCOLORINDEX(B1)

a. user say:

This tool is developed by Microsoft and is used to convert files' encoding format between GB18030-2000 and Unicode. The current version of this tool is 1.0 and it has been shipped with different versions of Windows, including Vista. It is copyrighted 2001.

GB stands for Guójiā Biāozhǔn which means national standard. GB standards are the Chinese national standards issued by the Standardization Administration of China (SAC), the Chinese National Committee of the ISO and IEC. Mandatory standards are prefixed "GB". Recommended standards are prefixed "GB/T" (T for tuījiàn, "recommended").

While the GB18030 supersedes the older GB2312 which was released in the 1980's, the GB2312 is still widely used and covers 99.75% of the characters used for Chinese input (it includes over 6000 Chinese characters along with symbols, punctuation, Japanese kana, Greek and Cyrilic alphabet, Zhuyin and double-byte set of Pinyin letters with tone marks).

It is unlikely for this program to be a malware, unless it is infected of course. Whether or not you should remove it would depend on where it is stored and whether or not it is infected.     A Microsoft Account is not required to setup Windows 10:

    Since the release of Windows 8, Microsoft has embraced the Cloud as a first class citizen in Windows. Making it a primary part of authenticating on Windows using a Microsoft Account. Microsoft Accounts can be either a (Outlook.com, Live.com, Hotmail.com or MSN.com) domain. The benefits of a Microsoft Account are numerous when using modern versions of Windows such as Windows 8, 8.1 and Windows 10. They allow you to sync your personal files to Microsoft's online storage service called OneDrive.

    The ambiguous error indicates to me that you placed all the code in one form's module, and this is incorrect as per the comments in the code.  The second Form_BeforeUpdate is for a subform.  If you do not have a subform then you can simply remove the second Form_BeforeUpdate from the code.

    Remember a form can only have one of any event.  So it can only have one Form_BeforeUpdate event, and by having 2, the compiler becomes confused and doesn't know which one to use, what to do, so you get the error you receive.

    ken sheridan say: As an alternative approach the following code can be used in the check box's AfterUpdate event procedure to lock/unlock the controls:

         Dim ctrl As Control
         Dim sfctrl As Control
         
         On Error Resume Next
         For Each ctrl In Me.Controls
            If ctrl.Name <> "chkLockControls" Then
                If ctrl.ControlType = acSubform Then
                    For Each sfctrl In ctrl.Form.Controls
                        sfctrl.Locked = Me.chkLockControls
                    Next sfctrl
                Else
                    ctrl.Locked = Me.chkLockControls
                End If
            End If
        Next ctrl

    However, this code is rather crude as it attempts to lock all controls, and the error is ignored if the control does not have a Locked property.  A better solution is to set the Tag property of those controls to be locked/unlocked to something like LockMe.  The code can then be amended as follows:

         Dim ctrl As Control
         Dim sfctrl As Control
         
         For Each ctrl In Me.Controls
            If ctrl.ControlType = acSubform Then
                For Each sfctrl In ctrl.Form.Controls
                    If sfctrl.Tag = "LockMe" Then
                            sfctrl.Locked = Me.chkLockControls
                    End If
                Next sfctrl
            Else
                If ctrl.Tag = "LockMe" Then
                    ctrl.Locked = Me.chkLockControls
                End If
            End If
        Next ctrl

    To unlock the controls when moving to a new record, and lock them by default when moving to an existing record put the following in the parent form's Current event procedure to set the value of the check box and then call its AfterUpdate event procedure:

        Me.chkLockControls = Not Me.NewRecord
        chkLockControls_AfterUpdate

    ken sheridan say: This error seems to be becoming a frequent subject of posts here.  Quite why I don't know.  The first thing is to be sure that the check box's Name is chkLockControls.  If so try changing the code to one of the following:

    sfctrl.Locked = Me!chkLockControls
    sfctrl.Locked = chkLockControls
    sfctrl.Locked = Me.Controls("chkLockControls")
    sfctrl.Locked = Forms(Me.Name).chkLockControls
    sfctrl.Locked = Forms(Me.Name)!chkLockControls
    sfctrl.Locked = Me.ActiveControl

    There are in fact many permutations which could be used. The last assumes that the code is in the check box's AfterUpdate event procedure.  If none of those work try changing the name of the check box, and the reference in the code to it, to something else.
