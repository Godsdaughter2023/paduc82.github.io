Sub BreakAllLinks()

    astrLinks = ActiveWorkbook.LinkSources(Type:=xlLinkTypeExcelLinks)

If you are looking for a unique identifier within a local network then the computer name would do

Declare Function GetComputerName Lib "kernel32" Alias "GetComputerNameA" (ByVal lpBuffer As String, nSize As Long) As Long

Function ComputerName() As String
  Dim stName As String * 255
  Dim lLeng As Long
  Dim lResult As Long
  lLeng = 255
  lResult = GetComputerName(stName, lLeng)
  ComputerName = Left(stName, lLeng)
End Function

You can then use ComputerName() to get the computer name.

For a truly unique ID you could try the MAC address of one of the computer's network interface cards.

There is some code to get the MAC addresses of all the network interface cards here: 
http://www.ehow.co.uk/how_6920281_use-vba-mac-address.html 

.. I have corrected an error in it and got it to return the MAC address of the first card here:

Function GetMACAddress()

  Dim objVMI As Object
  Dim vAdptr As Variant
  Dim objAdptr As Object
  Dim adptrCnt As Long
  Set objVMI = GetObject("winmgmts:\\" & "." & "\root\cimv2")
  Set vAdptr = objVMI.ExecQuery("SELECT * FROM Win32_NetworkAdapterConfiguration WHERE IPEnabled = True")
  For Each objAdptr In vAdptr
    If Not IsNull(objAdptr.MACAddress) And IsArray(objAdptr.IPAddress) Then
      For adptrCnt = 0 To UBound(objAdptr.IPAddress)
        If Not objAdptr.IPAddress(adptrCnt) = "0.0.0.0" Then
          GetMACAddress = objAdptr.MACAddress
          Exit Function
        End If
      Next adptrCnt
    End If
  Next
End Function

No antivirus can keep up with these since the files used to infect a system change too fast for the distribution and update systems to keep up.  The beginning of these attacks is typically malicious scripts which attack via your browser and then exploit vulnerabilities within software on your PC to install the obfuscated files which are part of the infection itself.


So since the malicious scripts are the first step, it's really within Firefox that the first problem lies, since unlike current Internet Explorer versions Firefox doesn't contain the levels of protection against these itself and also doesn't feed the information necessary to MSE to provide the script protection that might have stopped the attempt at this level.  For that you must find a separate script protection for the Firefox browser, since in reality it is the browser vendor who needs to protect from these types of attack.


The next level of these attacks is through vulnerable software, often including those with browser plug-ins such as Java or various Adobe and other products.  The problem here is most typically outdated or unpatched software, so only by keeping all of this software current as well can attacks via these be stopped.  Unfortunately, any new vulnerabilities which have yet to be patched can also provide entry, so the only way to reduce this risk is to keep the amount of 3rd-party software on a system to the minimum number possible, since each application will have its own vulnerabilities.


The last portions of these attacks can often be social engineering against the PC user themselves, so in this case only being aware of what you are accepting and how to properly answer any warnings or prompts provided by various software including your browser, MSE and any other applications on the system can avoid the possibility that you actually accept the installation of such malware.


So blaming MSE is disingenuous, since in reality there is little to nothing that MSE or any other typical antivirus protection can do about these.  Only applications specifically designed to provide browser protection can even attempt to stop such an attack and in most cases these are relatively complex to use and require significant levels of user knowledge to properly operate.

You aren't understanding the deeper issues with what you are expecting.  Quite simply the fault with the browser hijack is in the browser, not the antivirus.  These hijacks take advantage of scripting vulnerabilities in the browsers themselves and there's nothing an antivirus can do about them, which is why plug-ins exist for Firefox and other 3rd-party browsers that perform script blocking.


Anything that MSE itself might do within this realm in relation to Internet Explorer exists because they both control the development of that browser and establish the standardized hooks into the browser that allow any antivirus including MSE to monitor and manage its operation.  Firefox and other browsers have chosen to provide these interfaces as plug-ins, which isn't how MSE is designed to operate since it instead provides API calls from within the operating system and applications that control these functions.


The simple truth here is that Microsoft provides MSE to protect Windows and their own layered applications like Internet Explorer, Office and others.  They have designed all of these applications to support the antivirus and other API support calls into the filing system and other functions through which any compliant antivirus can monitor, detect and block malware.


Firefox along with many 3rd-party applications choose to implement their protection schemes in other ways and thus it isn't Microsoft's responsibility to provide the protection for their specific design choices.  If they do this for Firefox then they need to do it for every 3rd-party browser and application, which a moment of thought will show as a ridiculous impossibility.  Even if they tried it's obvious that any significant changes made by a 3rd-party to their application would likely break whatever Microsoft had created, so they'd be constantly needing to change their code to support changes in these other applications.


So Microsoft simply provides standardized interfaces to their own products, which 3rd-party antimalware can use to better protect their applications.  They also provide standardized filing system and other API calls to protect portions of the operating system itself.  This leaves the security of the 3rd-party applications up to those who create them, including patching, upgrades and in the case of network based applications like a browser, the external interfaces these products provide as well.

1. Messages were manually deleted either due to accidental deletion or were moved or deleted due to your account filter settings.

You can't prevent text from wrapping when it runs into the right margin, if that is what you are asking. Although you can set a negative right indent, which would make the text go past the margin, you would still get word wrap eventually.
