Failed Exchange 2007 to Office 365 migration. - Microsoft Community

troyferrell- [o365] say:

Failed Exchange 2007 to Office 365 migration.

OK.  Someone decided it would be a grand idea to migrate all the mailboxes from an old Exchange 2007 server out to Office 365.  Great idea right!  I agreed it would be a great idea at the time.  Exchange server 2007 was getting flakey as it was about 8 years old at the time and was starting to run out of disk space.  We got 4 mailboxes moved and BAM! dead hard drive on the Exchange 2007 server.  I was able to recover the database and so I figured the hybrid deployment from exchange 2010/13/16 would be a better move.  The client had a license for Exchange 2013 already so I grabbed a spare server and loaded it up.  Got the database put in, configured it for hybrid, and everything was happy!  That was until I started trying to migrate users.  Apparently, when you do a migration from Exchange 2007 it doesn't delete your on-premise mailbox.  It just sets up forwarders to your office 365 email address.  So now I've got 4 users who are using the office 365 mailbox but show up in the on-premise exchange server as local mailboxes.  I can't migrate them.  I can't edit their Office 365 settings.  What do I do now?

So, I banged my head into the proverbial wall until I felt my head about to pop when it hit me.  I'll create a new user.  Create their mailbox through the on-premise exchange server as an Office 365 user.  Then I'll take a look at the user attributes and copy.  This actually worked!  My users now show up as Office 365 mailboxes and I'm able to move them back and forth between Office 365 and the Exchange 2013 on-premise server.  Does anyone want to know how I did it?

The instructions are as follows!

From the on premise exchange server open the exchange management shell and run:
Get-Mailbox <username> | FL

Open your favorite text editor and type "On Premise" at the top.
Hit enter a couple of times and then copy the output from the command above and paste it into the text file.

Now, open the regular powershell as administrator.
Run the following in order.

Set-ExecutionPolicy RemoteSigned  -->> This only ever has to be run once!
$UserCredential = Get-Credential  -->> A dialog window will pop up asking for your office 365 admin login.
$Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri outlook.office365.com/.../ -Credential $UserCredential -Authentication Basic -AllowRedirection
Import-PSSession $Session

Now we are going to need to get the user info from Office 365 by running the same command as earlier.

Get-Mailbox <username> | FL

Open a your favorite text editor in another window and type "Office 365" at the top.
Hit enter a couple of time and then copy the output from the powershell window connected to Office 365.

Now close the session by typing the following:

Remove-PSSession $Session

Next we need to open the on premise ECP page and log in.
Create a new Office 365 user, if you don't already have one that works, from the local exchange 2010 or up server.

If the mailbox you are wanting to keep resides on the local exchange server, then open outlook or use powershell to export their mail to a .PST file.
Then find their On-Premise mailbox and disable it.  This will remove the local mailbox.

Login to a domain controller.
Open Active Directory Users and Computers.
click on "View" at the top of the window.
Put a check box by Advanced Features.
Find your Office 365 email user and open properties.
Find your trouble user and open properties.
Go to the Attribute Editor tab on both.
Now we need to look at a few attributes.

legacyExchangeDN = /o=First Organization/ou=Exchange Administrative Group (FYDIBOHF23SPDLT)/cn=Recipients/cn=<username>
mail = <User Email Address>
mailNickName = <username>
msExchMailboxGuid = ExchangeGuid from Office 365 text file.
 This one has to be entered funny.  Separate all the digits into 2 digit pairs.  Then for the first 3 sets enter each set backwards.
 For example, if the ExchangeGuid is 0abc21ba-337f-4dcf-9146-6ff1bb2d565a then you will need to enter it as - BA 21 BC 0A 7F 33 CF 4D 91 46 6F F1 BB 2D 56 5A

msExchMDBRulesQuota = Match Office 365 user.
msExchPoliciesInclude = Match Office 365 user.
  There are 2 entries you will need to copy.

msExchRecipientDisplayType = -2147483642
msExchRecipientTypeDetails = 2147483648
msExchRemoteRecipientType = 4
msExchTextMessagingState = 302120705; 16842751 -->> Don't worry if they show up reversed.
msExchUserAccountControl = 0
msExchVersion = Match Office 365 user.
proxyAddresses = Copy the EmailAddresses setting from both text files.  Don't enclude any doubles.
 You also need to add "X500:/o=First Organization/ou=Exchange Administrative Group (FYDIBOHF23SPDLT)/cn=Recipients/cn=<username>"
showInAddressBook = Match Office 365 user.
targetAddress = SMTP:<username>@<Office365domain>.mail.onmicrosoft.com

Apply those settings to the trouble user and open the ECP page for your local exchange server.  The user should now be listed as an Office 365 mailbox type and you
will be able to move them from Office 365 to On premise exchange and back with no issues.

At this point you can use outlook to upload the backed up emails if you need to.

Any questions?

Change Sheet name in a Formula dynamically - Microsoft Community

haries maulana akbar say:

Change Sheet name in a Formula dynamically

Hi, there.

I'd like to ask, is there a way to change sheet name in a formula dynamically based on a cell value?

For example, from the formula below:

=IF('sheet-name'!A1="80", "Good score", "Bad score")

I'd like to change the sheet name to another sheet name based on a cell value, with data validation.

I really appreciate the help.

amit tandon say:

Try below:

=IF(INDIRECT("'sheet-name'!A1")=80, "Good score", "Bad score")

where sheet-name is the name of the sheet. If the sheet name is in a cell (say B2), then use as below:

=IF(INDIRECT("'"&$B$2&"'!A1")=80, "Good score", "Bad score")

Regards,

Amit Tandon
