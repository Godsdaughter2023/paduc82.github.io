suzanne s. barnhill say:

By default, all Word's built-in heading styles are formatted as "Keep with next." If you have a lot of headings in succession without any Normal text, they will stick together in a block. As soon as you give them a chance to break to a new page, they will. This is likely what has happened here.

If you are using headings to create an outline, there are two ways to approach this problem:

  1. If you are entering all the headings to create the outline first, intending to go back later and fill in with Body Text, then you can ignore the problem temporarily (or you can work in Outline view).
  2. If the entire document will be an outline, it would be better to use nonheading styles. If you just need outline (multilevel list) numbering, you can apply that to styles that have an outline level of Body Text rather than Level 1, Level 2, etc. Or you can modify the built-in heading styles (at least the lower levels) to not have the "Keep with next" property.

cliffzhu say:

After a couple of hours work with MS support, I have figured out what's happening here.

This only happen to mailbox who is receiving emails of address *** Email address is removed for privacy ***.

Here is what happening:

1. Spam sender sent email to your organization with attached malware. (at this time, the sender is a invalid address)

2. Office 365 anti-malware filter blocks this email. (however, this email hasn't arrived Exchange system yet. So there is no way to see any log of this email. I believe MS missed this part.)

3. Anti-malware filter response to the sender. (by default setting) However, the sender is an invalid address. So it's not reachable. Anti-malware use *** Email address is removed for privacy *** as sender. (it's not configurable)

4. The bounced back email return to Postmaster mailbox. That's what you are viewing.

The mystery is: 

1. administrator has no way to know where that email from because Anti-malware doesn't log that email in anywhere. So we don't know what to block.

2. in returned Undeliverable mail, most time there is no addess at all other than Postmaster@. You are able to trace this. this email transition happens inside of Office 365 servers. there is no way to know what external address we are dealing with. So we have no way to know what to block. If the sender keeps sending same thing, we will keep receiving those Undeliverable email notice.

My solution is:

1. Do not send notice to external senders who sent malware in Anti Malware setting (under Protection-Malware Filter)

2. Block sender IP. I mentioned there is no sender infor in undeliverable email. But if you do a search by message ID. you will find the sender. and if you trace that sender against all your maillboxes, if you are lucky, you will find some message were logged by Exchange for other reasons (like non-existing address). From those log, you will be able to find out sender's ip and block it in Protection->Connection filter.
