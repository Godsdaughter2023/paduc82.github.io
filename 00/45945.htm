Hi Bill,

 

 

It’s great to know that you are using Windows 8 and thank you for providing us an opportunity to assist you with this issue.

 

You have mentioned that you get an error message: “There's a problem with this drive. Scan the drive and fix it now” when you connect your iPod Classic with the Windows 8 computer. I would like to ask you a few questions first, to get a clear picture of the issue.

 

 

1.       Does this happen with all the USB devices that you connect to the computer? Or is this issue specific to the iPod device?

 

2.       Did you try running the scan for checking the disk errors?

 

3.       Have you made any recent change on the computer prior to this issue?

 

4.       Did you check the issue by connecting the iPod on a different USB port?

 

 

This error message is usually displayed when there is an error within the USB drive that you connect with the computer. These errors could be due to some corrupted files. You will also get this message if you remove the USB drive abruptly without clicking “safely remove this device” from taskbar. I would suggest you to try these steps and check the status.

 

 

Method 1: Run the Hardware Troubleshooter

 

The Hardware Troubleshooter is an automated tool which will check the hardware connected to the computer for any known issues with it and provides the details and on how to fix them. Follow these steps and run the troubleshooter.

 

a)       Press “Windows Logo” + “W” keys from the keyboard.

 

b)       Type “Troubleshooting” in the search bar and press “Enter”.

 

c)       In the “Troubleshooting” window, click on “View All” on the left pane.

 

d)        Click on “Hardware and Devices”.

 

e)       Click on “Advanced” and then click on “Run as Administrator”.

 

f)        Click “Next” and follow the on-screen instructions to complete the troubleshooting process.

 

 

Method 2: Disk Error Checking

 

 I would suggest you to perform a complete scan on iPod drive to check if there is any errors and check. Refer to these steps:

 

a)       Right click on the iPod drive from the Windows Explorer.

 

b)       Click on “Properties” and then click on the “Tools” tab from the properties window.

 

c)       Click the check button under “Error Checking” to scan errors in the drive.

 

 

Note: Data available on bad sectors in the disk might be lost when windows tries to repair your disc.

 

If this did not help, then I would also suggest you to uninstall the USB driver and restart the computer so that Windows 8 will automatically detect and install the suitable driver files once again. Try this and check.

 

Method 3: Uninstalling and reinstalling the USB drivers.

 

I would also suggest you to uninstall the USB driver and restart the computer so that Windows will automatically detect and install the driver files once again. Refer these steps:

 

a)       Disconnect the iPod device from the computer.

 

b)       Press “Windows Logo” + “X” keys on the keyboard and select “Device Manager” from the list.

 

c)       Locate USB controllers and expand it to find the USB ports.

 

d)       Right click and select “Uninstall”, repeat the same for all ports.

 

e)       Now, restart the computer and connect the iPod back, for Windows 8 to detect and re-install the suitable drivers.

 

 

Hope this helps. If the issue remains unresolved, please get back to us and we’d be happy to help you further.

    a. user say:

    You need to re-install Vista on the new drive ( a clean installation) and then activate it and then upgrade it to Windows7 and activate that.  First backup your data - that's pretty much all you're going to be able to save.  You're going to need to reinstall all your programs and reset all your settings - they cannot be copied over to the new drive.  It just doesn't work that way.  One option that may allow you to transfer a bit more than just data (but still not the programs) is to use EasyTransfer: http://support.microsoft.com/kb/928635. Review the process and see if you can make it work in your situation.

    So I did some investigation, and I believe these errors are harmless (but please, MS, get rid of the errors if they're harmless!)  If you're having issues with your apps, these errors may be totally unrelated as you can get them even when nothing's actually wrong.  Keep reading if you want to know if they're actually causing you a problem.

    Anyways.  What I did was go to the Event Viewer, and find Applications and Services -> Microsoft -> Windows -> Appmodel-Runtime -> Admin.  That's the log these errors are coming from.  Reading it will be MUCH easier if you create a custom view based on it, then "Filter Current Custom View", and where it says "<All Event IDs>", type in -39,-40,-41.  That will filter out a ton of useless messages.

    What's left after you do that is interesting.  For every one of these "error" events, there are two accompanying "Information" events that basically contradict the error message.  All 3 will usually happen in the same second.

    Here's an example of a set of these 3 events from my log.  Since all 3 occur in the same second, they aren't properly sorted by time.  There's no reason to think they actually happen in the order they're displayed in the log.  I'd guess the real order is most likely the middle Error happens first, then the bottom "Information" that sets the status successfully, then the top "package successfully updated" event.  In fact, I'll paste them here in that order:

    Event 1 - the error of failing to set the status to it's proper "finished" status:

    Event 2 - the instantly successful retry which sets the status to the desired value:

    Event 3 - and finally the app updated successfully event:

    Looking through my log, every single time I get this error, I also get the successful retry and completion of the update within 1 second (EDIT:  Oops, not quite, see the EDIT at the bottom of this post).  If you're actually having trouble using your apps, you can check your own log as described above to see if these errors are actually related (i.e. the status *doesn't* get fixed a fraction of a second later) to your problem.

    So, if you're only trying to get rid of these errors in the log, and don't actually have any problems with your apps, I'd stop trying all the "AppManifest" solutions that have been posted.  You're more likely to break something than to fix anything.

    In fact, I doubt there's anything at all we can do to get rid of the errors.  I think only Microsoft can fix these false error messages.  If they can't fix what's causing the status to fail to update on the first try, they could easily (and IMHO should as a matter of good programming practice) inject a bit of exception handling code that will always retry the status change a couple of times before actually throwing an error.

    In fact, consider this my official request to Microsoft that they do that very thing.  Unnecessary error messages like this are very distracting to the purpose of the Event log in tracking down REAL problems, as evidenced by *every single previous poster in this thread* who has real problems with their apps that thinks this harmless error is pointing to the root cause, when it quite probably lies elsewhere..
