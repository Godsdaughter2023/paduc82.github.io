When you create a document based on a template, and the template contains macros, those macrosstay in the template. Saving the document does not transfer the macros from the template to the document. The document simply has a reference to the macros in the template, which itself must be available on the user's computer.

If you want the macros to be in the document, you'll have to open the VBA editor and copy the code from the template's project to the document's project. You'll also have to digitally sign the document with a certificate, or advise your users to set their macro security low enough, so that Word doesn't disable the macros. jay freedman say:
Yes, you're missing the fact that no version of Word has ever placed
macros in documents except manually. You double-click the template,
Word creates a document based on the template, and stores a pointer in
the document to the location of the template. The next time the
document is opened, the template is automatically loaded along with it
*if the template is accessible in the same location*. The macros --
still in the template -- are available in the opened document.

The only time you have a problem is if an existing document is opened
when the template is not accessible -- for example, if someone takes a
document on a laptop and disconnects from the network. Then Word
replaces the template pointer with a reference to the user's local
Normal.dotm, which (probably) doesn't have the same macros.

jay freedman say:

Yes, that's correct. The important point is that the template has to be accessible from whatever computer is opening the document. It could be on the same computer, or on a network server, but Word has to be able to "see" the template and load it into memory along with the document.

Another implication of this is that the document does NOT have to be saved as a .docm (macro-enabled document). It will work exactly the same if it's saved as a .docx file. The reason there are two different kinds of file is that you can be sure there is no macro code stored in a .docx file, so it's safe -- but it can still get access to macros in its template.

A .docm file may (but doesn't have to) contain macro code; but if it does contain code, it was put there manually and not just by saving a document based on a macro-containing template.

I'm not sure I can make it any clearer without drawing diagrams. :-)
