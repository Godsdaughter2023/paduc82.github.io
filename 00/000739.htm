The problem is that one of the headings is placed just before the end of the page and gets cut off from its text. Using the "Keep with next" paragraph option doesn't work, presumably because of the section break. The document happens to be in right-to-left but I don't think that's the problem.

suzanne s. barnhill say:
Columns are a section property, so no, there's no way to change the number of columns without inserting section breaks. I suspect there's no way to correct your problem except manually. It will be best to tackle this when editing is complete, so that text flow is not likely to change. At that point, you could format the heading as "Page break before" or change the section start type from Continuous to New page (on the Layout tab of Page Setup)

bonglepot say:

Thank you Suzanne. I think you're probably right, as stupid as this is. For very large documents it is completely impractical to start doing this manually. Perhaps a macro? I'm not much good at those.

I think the problem here is a basic design flaw in word. If you're going to use section breaks for formatting changes like columns, then you can't treat it as a normal section break as though it was something completely new. So you should still be able to use options like "Keep with next".
