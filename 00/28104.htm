Hello Paul,

 

Glad to hear from you.

 

First, if you are using Office 365 Exchange Online and set up the email account in your PC via Exchange method (default), you won’t worry about transferring data between different machines, because all accounts will sync data from the cloud. If you change another new PC, just set up the account via Exchange method and then everything will be synced from the remote server again. All the things you did in that old PC have been synced backed to the cloud, so you will see everything the same as before.

 

Second, if you just want to use OWA. You can add multiple (up to five) accounts as connected accounts to your primary account. But, you cannot add the email address which has the same suffix as yours. Such as *** Email address is removed for privacy *** is your address, another colleague or group in your company is using *** Email address is removed for privacy ***. However, if you use Outlook for Windows client, you can add Office 365 account to one profile and everything will go to the correct folder list. Go File > Account settings > Account settings > New > type the related account.

'''''''''''''''''''''''''''''''''''''''
Function NbrS(xIn, xTh As Integer) As String
Dim yWords
   yWords = Split(xIn, " + ")
   If (xTh - 1) >= LBound(yWords) And (xTh - 1) <= UBound(yWords) Then NbrS = yWords(xTh - 1)
End Function

Function Stack(Rng As Range) As String
Dim i0 As Integer, i1 As Integer, i2 As Integer, i3 As Integer, i4 As Integer, i5 As Integer, i6 As Integer
Dim s, N As Long
s = Split("40000,30000,20000,15000,10000,8000,6000", ",")
For i6 = 0 To Int(Rng / s(6))
For i5 = 0 To Int(Rng / s(5))
For i4 = 0 To Int(Rng / s(4))
For i3 = 0 To Int(Rng / s(3))

For i1 = 0 To Int(Rng / s(1))
For i0 = 0 To Int(Rng / s(0))
N = i0 * s(0) + i1 * s(1) + i2 * s(2) + i3 * s(3) + i4 * s(4) + i5 * s(5) + i6 * s(6)
If N = Rng Then
    For N = 1 To i0: Stack = Stack & " + " & s(0): Next

canadian tech guy say:

Had the same issue when a user moved to office 2013, but we managed to solve it. 

First, close Outlook and then navigate to C:\users\<username>\AppData\Local\Microsoft\Outlook\Offline Address Books and delete the folder holding the offline address book. Reopen Outlook and when you create new meetings you should see the proper username. It also creates a new OAB from Exchange. 

This only works for calendar items from this point forward.  It does not correct existing ones.
