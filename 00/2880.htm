f z z say: There's no way to access environment variables from worksheet formulas. You'd need a user-defined function. Hold down an [Alt] key and press [F11] to display the Visual Basic Editor (VBE). Multi-core process and Word for Mac

I am running Word for Mac 16.18 on an iMac with MacOS 10.14.1. It runs fine, but starts to show spinning beach ball with very large documents. Does anyone know if Word uses multiple cores? Or does it do all its processing on a single core? Thanks.
igor leyko say:
Hi,
Both yes and no are the answers. Since Word 6.0 for Windows NT version Word can use different threads for different operations. E.g. pagination, hyphenation, spell-checking and so on can be executed simultaneously at different cores. But each of this threads can use only single core so if you have document with very complex pagination this pagination will use only one core. Same is correct for a spell-checking - it may be done simultaneously with pagination but cannot use two or more cores itself.
So yes, Word can use multiply cores for different tasks and no, each basic function in Word cannot be parallelized.
Right now I opened very complex Word document on my 4-core 8-threads PC and saw eight peaks on eight CPU graphs. But six peaks were short, one peak slightly longer, and one pick - much longer.
