alexsun_2012 say:

Hi Dhey,

Yes, the user will still receive the quarantine email notifications after he blocks the original sender.

After he add the original sender *** Email address is removed for privacy *** to the block list, Exchange Online Protection (EOP) will detect the setting and mark the coming emails as spams and move them to Quarantine.

When an email is moved to quarantine, the quarantine notification is generated by system automatically and sent to the user from the MS address *** Email address is removed for privacy ***. Since *** Email address is removed for privacy *** is not blocked, the notification emails will be delivered to his mailbox.

To block the notification emails, the user can set up an inbox rule based on the address *** Email address is removed for privacy *** to delete or move them to a specific folder. However, this rule will move or delete all the notification emails that come to the user.

To do this I put Track Changes on, then did a global search & replace on formatting for Italic to Bold, hoping this would make them all show up in the list of revisions... but they don't. Whether I do Replace All, or individual Find Next -> Replace, it makes no difference. Trying different options within Replace With (e.g. different font, font size, colour, highlight, style, etc.) does not work either. If I make the change directly (not from the Find & Replace dialog), it works fine - the format change is tracked and shown in the list. Hence I assume this is a Word bug/feature.

I managed to produce a workaround via a macro as shown below.

------------------------------

Sub TrackFormatChanges()

' Change all Italic to Bold in Track Changes, one by one as Find/Replace dialog doesn't work
'
    ActiveDocument.TrackRevisions = True
    Selection.Find.ClearFormatting
    Selection.Find.Font.Italic = True
    With Selection.Find
        .Text = ""
        .Replacement.Text = ""
        .Forward = True
        .Wrap = wdFindContinue
        .Format = True
        .MatchCase = False
        .MatchWholeWord = False
        .MatchKashida = False
        .MatchDiacritics = False
        .MatchAlefHamza = False
        .MatchControl = False
        .MatchByte = False
        .CorrectHangulEndings = False
        .HanjaPhoneticHangul = False
        .MatchAllWordForms = False
        .MatchSoundsLike = False
        .MatchWildcards = False
        .MatchFuzzy = False
    End With
Do
    Selection.Find.Execute
    If Not Selection.Find.Found Then Exit Do    ' not found = EOF()
    Selection.Font.Bold = True
    Selection.Font.Italic = False
    Selection.MoveRight Unit:=wdCharacter, Count:=1
Loop    ' REPEAT until not found, i.e. EOF()

End Sub

macropod say:

This behaviour has been around for a long time and is one of a number of anomalies with using Find/Replace macros with Track Changes. Some simpler and more efficient code to work with:

Sub TrackFormatChanges()
Application.ScreenUpdating = False
Dim bTrk As Boolean
With ActiveDocument
  bTrk = .TrackRevisions
  .TrackRevisions = True
  With .Range
    With .Find
      .ClearFormatting
      .Replacement.ClearFormatting
      .Text = ""
      .Replacement.Text = ""
      .Format = True
      .Font.Italic = True
      .Forward = True
      .Wrap = wdFindStop
      .Execute
    End With
    Do While .Find.Found = True
      .Font.Bold = True
      .Font.Italic = False
      .Collapse wdCollapseEnd
      .Find.Execute
    Loop
  End With
  .TrackRevisions = bTrk
End With
Application.ScreenUpdating = True
End Sub

Turning off ScreenUpdating allows the code to run faster. The above code also restores the original 'Track Changes' state.
