muralidhar_g say:

Hi Steam,


Thank you for writing to Microsoft Community Forums.

I would like to inform you that the options which is required to disable specific external hard drives during the boot phase is not available. The reason is because during the BIOS Initialization phase, the platform firmware identifies and initializes all hardware devices which are connected to the computer, and then runs a Power-on self-test (POST). The POST process ends when the BIOS detects a valid system disk, reads the master boot record (MBR), and starts Bootmgr.exe.

During the OS Loader phase, the Windows loader binary (Winload.exe) loads essential system drivers that are required to read minimal data from the disk and initializes the system to the point where the Windows kernel can begin execution. When the kernel starts to run, the OS Loader loads the system registry hive and additional drivers that are marked as BOOT_START into memory.

The Operating system and the hardware is configured in such a way that the hardware including drives will be loaded first. So you can set the priority for the drive which needs to be loaded first. You need to disable the other drives from BIOS, if you doesn’t want the other drives to get loaded. However, the drawback of this task is that every time you need to manually enable those disabled drives from the BIOS, and I hope this is a time consuming process.
