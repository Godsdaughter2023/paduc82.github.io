Outlook will reply to an email in the same format as the incoming mail.

If the incoming email is in text format, the reply will be too.  In this case the fancy graphic designed for a HTML email will not show up.

Copying Custom Form Outlook 2010 - Microsoft Community

bestfsbodotcom say:

Copying Custom Form Outlook 2010

I have an exchange account and created a custom form and saved it in my personal forms.

I now have a new exchange account and want to import or copy the custom form into this new Outlook Exchange account so I do not have to recreate it !

Where can I find the form or make it available?

 

karl timmermans [outlook mvp 2012-2018] say:

Forms published to the Personal Forms Library exist in the "Inbox" folder of the "Mailbox" in use (old Exch account in this case) at the time the form was published so in order to get access to it, you will need to run Outlook at least once using your old Exch mailbox.

From there, once you have Outlook running using your old mailbox

- open any "new" (new = as in blank) contact

- from O'2010 menu --> File --> SaveAs --> change the file type to .oft (that file is your custom form template)

That gives you a copy of the Outlook custom form template. Would also suggest you start keeping a backupof any custom forms at all times - one of the biggest mistakes people make and suddenly find themselves in trouble because they don't have a copy of the form to work with if/when required.

Since you've already created a custom form so going on the assumption that you are familiar with the process of publishing etc. Using your new Exch account  - open the custom form via the Forms designer selecting the source to come from "User Templates in File System". Re-publish the form to your Personal Forms library using the exact same name was was used when the contacts were created. Worst case scenario is that you would need to run a message class modification program to reset the custom form to the new location.
