hi dave,

after my test, ai files support the version feature in sharepoint online. but if you delete a file, its version record will be deleted accordingly. if you don’t recover it in 90 days, it will be automatically deleted and you will be unable to restore it. you can also check the second-stage recycle bin page. please make sure if the file has been deleted more than 90 days.

now i try to answer your question.

sharepoint online has a reliable version feature. generally, if a user update the ai file, other people who have the full control permissions will see the current version of the ai file and see the version record via version history. the file which has the old version will not go to the recycle bin. it is just replaced with the new version. but users can go to version history to find the old version and restore it.

if you have only one version, i think the senarios are as following.
1. as i said above, if you delete a file, its version record will be deleted accordingly. then you don’t recover it in 90 days and you upload a new file with the same name.

for example, there are three users named usera, userb and userc in your company. userc has the full control permission, userb has the edit permission and usera has the read permission in the onedrive for business library.

if userb edit a file, only userc and he can see the edited version because you set only users who can edit items  in draft item security and userc has the higher permission than the edit permission. in usera’s ui, he still sees only the previous version because he only has the read version. if userb want s to publish the edited version to everyone, it needs to be approved by userc. only when userc approves it, usera becomes able to see all the version records. so please confirm if you or other people have done some settings and unpublished a major version.

Well it depends. In general, Microsoft's statement is the following:

“Mailbox permissions migration On-premises mailbox permissions such as Send As, Receive As, and Full Access that are explicitly applied on the mailbox are migrated to Exchange Online. Inherited (non-explicit) mailbox permissions and any permissions on non-mailbox objects—such as distribution lists or a mail-enabled user—are not migrated. Therefore, you have to plan for configuring these permissions in Office 365 if applicable for your organization.”

In some configurations it might work, however I'd simply advise you to re-apply the permissions after the migration.

Most of the text in the Word 2016 document I'm working on has - Microsoft Community

beverleyruddock say:

Most of the text in the Word 2016 document I'm working on has disappeared !

Most of the text in the Word 2016 document I'm working on has disappeared !

Any help gratefully received !

jay freedman say:

It's possible that the "missing" text is inside a field. To test this, open the document and press Alt+F9. If the missing text appears, but is enclosed in bold curly brackets { }, select the text but not the brackets, cut it to the clipboard, and paste it outside the brackets. Then delete the brackets.

If that doesn't pan out, it's possible that at some point you had part of the document selected and you pressed the spacebar or typed something, and that replaced the selected text. In that case, the only thing that may allow you to get it back is if one of several backup mechanisms saved the document before the text went missing. Unfortunately, most of those mechanisms aren't turned on by default.

Two of the mechanisms are within Word.

  • Click File. On the left side, click Open. Scroll the right side all the way to the bottom, and click the Recover Unsaved Documents button. If you find a file with the same name as your document plus "Unsaved" and a long number, open it and look for the text. These files are automatically deleted after four days, so don't delay.
  • In the Word Options dialog, click Advanced on the left side. Scroll the right side to see the section headed "Save". If the option "Always create backup copy" is checked, Word will have stored the document as it existed immediately prior to the most recent Save command, with the file name "Backup of <name>.wbk" in the same folder with the document. You can open that to look for the text. This option isn't turned on by default, but it's a good idea to enable it.

Another mechanism is part of Windows, the File History feature. You must set up the feature before it will save anything, but if you have already done it then you can use it to recover earlier versions of the document:

  • Click in the search box in the Windows task bar and type filehistory. When that appears in the "Best match" at the top of the results, click it. With the arrow buttons at the bottom of the window, scroll back to the last date and time when you know the document was undamaged. Double-click a folder to open it. Click on the document file and then click the big green Restore button.
