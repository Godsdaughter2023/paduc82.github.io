However, both tapes pertain to the same holiday and I would like to 'stitch' the end of Video Tape One to the beginning of Video Tape Two. I did have the clips of Video Tape One in the Contents Pane, but when I opened Video Tape Two (thinking its clips would appear after those belonging to Video Tape One they appeared in the Contents Pane all jumbled up with Video Tape One so there was no chronological or thematic order.

What is the best way of combining these two files, please?

<Windows 7>

considerate_guy say:

When you first open Windows Live Movie Maker, you have an empty storyboard area.  You should click on this area, or choose 'Add Videos and Photos' button on the Home tab.  You can do this in one step or two, but for better control, you might do it as two steps.

First choose the video file that is from your first tape.  It should appear in the storyboard and you should be able to preview it now.

Now click that button again and choose the second movie.  It should appear in the storyboard after the first.

Perhaps you actually have multiple clips from each of the tapes? In that case, as you are about to select the clips, make sure you have them sorted correctly.  You can right-click on the File Open dialog (where it is showing your clips to be added) and choose 'Sort By'.  You probably want to sort them in Date order.  Now select them and they will be added in that order.  Repeat for the second tape.

If by chance, this still results in a couple clips in a different order from what you want, simply select that clip and drag it in the storyboard to the desired location.  Do this until the movie is in the order you want.  Add other things (titles, captions, effects, transitions), then save.

You need to save your project (it contains the information about what goes where, etc.) and then in order to have your final video, you must also save your movie.

Both options are under the File menu (Alt+F).  To save your project, use Save.  To save your movie, use Save Movie.

mike hall... always say:
Homegroups are for networks where all computers are running Windows 7, so don't use the feature. Go to 'Network and Sharing' in Control Panel and set up a regular network..

Re printing.. It doesn't matter which computer is running 32 or 64-bit. The connection method is the important part..   

I am assuming that you know how to and have made the necessary share permissions on the computer which is hosting the printer.

 

To share a network printer being hosted by a computer running a 32 bit operating system with another computer running a 32 bit operating system, Vista or XP, you would go to Control Panel > ‘Printers’ > ‘Add Printer’ > ‘Add a network printer’.  Apologies if you consider that I am stating the obvious.

 

Sharing a printer being hosted by a computer running 32 bit Vista or XP with a computer running 64 bit Vista is not obvious, because if you use the above method, it will not work.

 

So how? First, you need to know the name of the computer which hosts the printer, and the name of the printer to be shared.

 

Sitting at the host computer: The procedure is the same for Vista and XP

 

Start > My Computer > Properties > Computer Name tab > Change button. Write down the name that you see here because this is the computer name.

 

Start > Settings > Printers and Faxes > Right click on the printer to be shared > Properties > Sharing tab. Write down the name that you see here because this is the printer name.

 

OK. Now sit in front of the Vista 64 computer:

 

Start > Control Panel > Printers > Add printer > Add a local printer > Click on ‘Create a new port’ > ‘Local Port’ > ‘Next’.

 

The port name will look like this: \\Computer name\printer name

So enter the names already retrieved, ensuring that you use the same format, then click ‘OK’.

 

At this point, you will be presented with the ‘Add New Hardware’ process. You can either select the make and model of your printer from the lists or insert your printer installation disk if you can’t find yours. The printer will be added, and you should be able to print a test page.

frederik long say:

The only advantage I know of for having OS and data partitions it making it slightly easier to do backups.  With Windows 7 that is largely gone because virtually all data is held in the User folders.

Keeping user data on a separate partition delivers one big advantage: The ability to separate Data backups from Image Backups.

An image of a system partition needs to be created or updated only once every few months.

An image containing both system folders and user data has two drawbacks:

  • The file is unnecessarily large. It can easily grow to a few hundred GBytes instead of the basic 50..100 GBytes required for system files only.
  • Restoring such an image will replace current user data with data that is several months old.

Both drawbacks are eliminated when user data is stored on a separate partition. My philosophy is therefore exactly to opposite of yours:

  • Every machine I maintain has two partitions: One for the System, one for user data.
  • I have an image of the System partition of every machine I look after.
  • This image gets updated about once a year.

When a machine fails catastrophically I restore the image, thus reducing repair time from a whole day for a rebuild to less than an hour.

After Windows 7, system drivers and all updates are installed and any stable applications like Microsoft Office are installed and updated, create a system image.  This image does NOT contain any a hard drive, you will have a precise duplicate of your system as it was at that point.  You can restore that image to a hard drive in about 20 minutes.  Creation of System Image is found in your menu under Maintenance, Backup and Restore. 

Another great feature about using the image is that you do not need an install disk or a product key to do the re-install the next time.

I must emphasize the need to use PLUS R DVD's and not the more commonly found MINUS R's.  I have never had a failure and have used this at least 20 times.  Time savings is usually about 6 hours and avoids a plethora of problems.
