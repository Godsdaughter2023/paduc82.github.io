I don’t know about such an inbuilt method and obviously nobody else. Therefore, the following, somehow elaborated macro proved hundreds-time useful for me. It is set up for embedded pictures and selected types of charts.

I would invite any improvements or simplifications. The functionality would deserve it. Consider, for example, even the developed size changes of objects when intersected rows or columns are inserted or size modified.

Regards

PB

 

 

Option Explicit

 

Sub ObjectSize()

'Sets the sizes of the selected object(s) in mm.

'Inputs:

'- Cancel - in both cases null action

'- Height:

'         - number -> new dimension,

'         - "m" -> both actual dimensions are saved and shown,

'         - no input -> actual height is kept,

'- Width:

'         - number -> new dimension,

'         - "s" -> width is set to create a square,

'         - "a" -> actual aspect is kept (ratio of height to width)

'         - no input -> actual width is kept.

'When more graphical objects are selected, the sizes of the first are taken

'as relevant. All of them are converted to the set size.

'Using of a preset shortcut key is recommended.

 

  Dim ErrMsg As String, MB As Long, ChH As Double, ChW As Double, NewSizes As Boolean, _

    S As Variant, Sp As Variant, NewChartHeight As String, NewChartWidth As String, _

    ObjDim As Variant, I As Long

  Static ChartHeight As Double, ChartWidth As Double, NChart As Long

 

  Const PointSize As Single = 2.831   'this gauge value can be even tuned

  Const Title As String = "Object size"

NewSizing:

  On Error Resume Next

  NChart = 1

  NChart = Selection.Count

  'ErrMsg = "object"

 

  Set S = Selection

  If NChart > 1 Then

    Set Sp = S(1)

  Else

    Set Sp = S

  End If

  ErrMsg = "correct chart/picture"

  On Error Resume Next

  I = Sp.Cells.Count

  If I > 0 Then GoTo ErrExit

  On Error GoTo ErrExit

  If Sp.Parent.Type = -4167 Then Set ObjDim = Sp Else Set ObjDim = Sp.Parent.Parent

  Select Case Sp.Parent.Type

  Case -4169, -4167, -4111, 4, 3

  Case Else

     GoTo ErrExit

  End Select

ActualSize:

  If ChartHeight = 0 Or NewSizes Then ChartHeight = ObjDim.Height / PointSize

  If ChartWidth = 0 Or NewSizes Then ChartWidth = ObjDim.Width / PointSize

  If NewSizes Then Exit Sub

  NewChartHeight = InputBox("H e i g h t /mm" & vbCrLf & _

    "Cancel:  leave actual hight" & vbCrLf & _

    "'m':     save actual sizes", Title, Format(ChartHeight, "0"))

  If LCase(NewChartHeight) = "m" Then

    MB = MsgBox("H e i g h t : " & Format(ObjDim.Height / PointSize, "0") & vbCrLf & _

      "W i d t h   : " & Format(ObjDim.Width / PointSize, "0"), vbOKOnly, Title)

    NewSizes = True: GoTo NewSizing

  End If

  If NewChartHeight <> "" Then ChartHeight = CDbl(NewChartHeight)

  If ChartHeight <= 10 Then GoTo ErrExit

  NewChartWidth = InputBox("W i d t h  /mm" & vbCrLf & _

    "Cancel:  keep actual width" & vbCrLf & _

    "'a':     keep aspect" & vbCrLf & _

    "'s':     square dimension", Title, Format(ChartWidth, "0"))

  Select Case LCase(NewChartWidth)

  Case "": Exit Sub

  Case "s": ChartWidth = ChartHeight

  Case "a": ChartWidth = ChartHeight * (Sp.Width / Sp.Height)

  Case Else: ChartWidth = CDbl(NewChartWidth)

  End Select

  ErrMsg = "size great enough"

  If ChartWidth <= 10 Then GoTo ErrExit

  ChH = ChartHeight * PointSize

  ChW = ChartWidth * PointSize

  On Error GoTo ErrExit

  Application.ScreenUpdating = False

  If NChart = 1 Then

    ObjDim.Height = ChH

    ObjDim.Width = ChW

    Sp.Select

  Else

    For I = 1 To NChart

      S(I).Height = ChH

      S(I).Width = ChW

    Next I

    S(NChart).Select

  End If

  Application.ScreenUpdating = True

  On Error GoTo 0

  Exit Sub

ErrExit:

  MB = MsgBox("No " & ErrMsg & vbCrLf & " has been selected", vbCritical, Title)

  On Error GoTo 0

End Sub

brook zhang msft say:

Hi Paul,

The issue with the crash moving email to an IMAP account was caused by a change in the service.  A new property was introduced that Outlook was not handling well.  The Outlook Team has developed a fix for it and it will first release to Current Channel as build 16.0.8229.2073.  Fixes for all other versions including MSI are in process but not yet scheduled. 

Since the issue causing the problem was turned off in the service you should not see issues with new emails that youmove into the IMAP account.  The property that the service was stamping on the emails is no longer being applied.  Existing emails that you moved prior will have the property stamped on it and still cause the issue until you install the client fix.

If you need the Outlook 2016 Client to handle the problem today then you could install Office Insider Slow.  That channel currently has the fixed build that is going out tomorrow to Current Channel.  To install Insider builds see this documentation, https://products.office.com/en-us/office-insider.    

For the future updates, we suggest you monitor the updates Gabriel Bratton shared in this thread: https://answers.microsoft.com/en-us/msoffice/forum/msoffice_outlook-mso_win10/outlook-2016-crahes/d0528755-6495-4a9f-b8fb-cd5c9a1fd53a. 

Thanks,

Brook
