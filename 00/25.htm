You're linking the Main and Sub forms on tblMemberInfo.PropertyID, and that value does not exist when I add a new Property record.  I tried adding a new record, and then dropping to the BoatInfo subform, and received the same error as you did.

Before you can add a Child record, Access must find the related Parent record. Since you've linked the Parent and Child form on tblMemberInfo.PropertyID (and that value does not exist), Access does not know what value to add as the PropertyID in the BoatInfo subform.

If a Boat is related to a property (and NOT a Member), then you'd have to modify your parent form a bit, I'd think. Your ParentForm should be based on tblPropertyInfo only, and should not include tblMemberInfo. You would need to add tblPropertyInfo.PropertyID to your query, and change the Property textbox to show that value (and not the one from tblMemberInfo). Master project in read-only mode - Microsoft Community

ab_c say:

Master project in read-only mode

Hi,

I would like to follow several logical sub-streams of my project (or subproject in MS Project glossary maybe?) in one consolidated view.  However, I would like to only have a read-only view.  Thus at the master level I do not want that any changes that I make are saved to the individual sub-streams.  But the other way round should be true, i.e. changes made in planning of the sub-streams are visible at the master level the next time I re-open the master.

How could I do this?

Only reason for read-only is that I want to prevent accidental changes made at master level getting saved in individual sub-stream plans.

Many thanks in anticipation.

john - project say:

AB_C,

The only way that changes will be saved is if the user answers "yes" to the save dialogue the pops up when you close the master. With a dynamic master you also get a second chance since a second dialogue pops up asking if you want to save changes to the subprojects so just answer "no" and "no to all" and you will be okay.

You could also save the master as "recommended read only" but let's say you want to change formatting of your master. You will need write access to be able to save those formatting changes. You can still say "no to all" when Project asks if you want to save changes to subprojects.

As long as each subproject users saves their changes when the edit their file, you will automatically see those updates when you open your master.

John

julie sheets say:

You may insert a subproject into a master project as read-only - but you still need to be careful.  Read-only will not prevent you from making a change to the subproject file - but will prompt you about saving changes.

If you want a truly read-only then do not link the subproject file when you insert.  However, this will not update the master if changes are made to the subproject.

I hope this helps.

Julie

brian tillman [outlook mvp 2007-2019] say:

> 1) Can I have some rules on Office 365 and some on Outlook and have them all

> run on mails accessed from outlook, whether that be when they arrive or after.

When you say "on Office 365", which server - Outlook.com or Office365.com - are you using? The former is for Office 365 Personal and Home suites and the latter is for the business suites.

You can have rules both on the server and on the client. Unless you're using Exchange, (Office 365 for Business or the new Outlook Mail Preview) the server-side rules won't appear in the client. They'll always run when the message arrives in the Inbox and whatever action they specify will happen before the client Outlook 2016 ever sees the message. Once you open Outlook 2016, whatever rules you have there will then operate on any new message remaining in the Inbox.

When you have an Exchange account, rules can still be server-side or client-side, with the server-side rules running even if Outlook isn't and the client-side running once you start Outlook, but you'll be able to see and control them all from the client.
