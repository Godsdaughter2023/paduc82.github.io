macropod say:

I inserted a thumb drive containing hundreds of WPD files, and copied them to My Documents.  I then converted them to Word...

When I transferred the thousands of the files which were in Word Perfect on the thumb drive to my PC, I somehow converted them to .wpd files which are readable by Word 2016, but are not searchable.

If they still have their wpd extensions, those documents have not been converted to Word documents - they're still WordPerfect documents (which Word can read/open/edit/save).

The following macro will convert a folder of wpd documents to Word's native docx format:

Sub ConvertWordPerfectDocuments()
Application.ScreenUpdating = False
Dim strFolder As String, strFile As String, strDocNm As String, wdDoc As Document
strDocNm = ActiveDocument.FullName
strFolder = GetFolder
If strFolder = "" Then Exit Sub
strFile = Dir(strFolder & "\*.wpd", vbNormal)
While strFile <> ""
  If strFolder & "\" & strFile <> strDocNm Then
    Set wdDoc = Documents.Open(FileName:=strFolder & "\" & strFile, AddToRecentFiles:=False, Visible:=False)
    With wdDoc
      .SaveAs FileName:=strFolder & "\" & Split(strFile, ".wpd")(0) & ".docx", FileFormat:=wdFormatXMLDocument, AddToRecentFiles:=False
      .Close SaveChanges:=False
    End With
    'Kill strFolder & "\" & strFile
  End If
  strFile = Dir()
Wend
Set wdDoc = Nothing
Application.ScreenUpdating = True
End Sub

Function GetFolder() As String
Dim oFolder As Object
GetFolder = ""
Set oFolder = CreateObject("Shell.Application").BrowseForFolder(0, "Choose a folder", 0)
If (Not oFolder Is Nothing) Then GetFolder = oFolder.Items.Item.Path
Set oFolder = Nothing
End Function

Simply select the folder to process. The code includes the line:

'Kill strFolder & "\" & strFile

If you delete the ' at the start of that line, the WordPerfect files will be deleted after conversion.

If you'd prefer to save the converted files in the older Word doc format, change:

.SaveAs FileName:=strFolder & "\" & Split(strFile, ".wpd")(0) & ".docx", FileFormat:=wdFormatXMLDocument, AddToRecentFiles:=False

to:

.SaveAs FileName:=strFolder & "\" & Split(strFile, ".wpd")(0) & ".doc", FileFormat:=wdFormatDocument, AddToRecentFiles:=False

Note: The converted files will now have today's date/time as their date/time stamps. More elaborate code would be required to restore the original date/time stamps, but the files' internal metadata would still have today's date/time as their creation/saved date/time stamps. More elaborate code would also be required to automatically process sub-folders.

For PC macro installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm
