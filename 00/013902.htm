Hi Emanuel,

You may need to refresh your understanding of "Duration" as opposed to "Work".

For a summary task:

- Work is the sum of the work of the subtasks.

- Duration is the difference between the early finish and early start of the summary task, computed according to the applicable calendar.

While your at it, I'd suggest removing the logical relationships on the summary tasks - both predecessors and successors.  Not much good and a whole lot of bad things are associated with them, and they are considered bad practice.

hansv mvp say:

Here is code you could use:

 

Sub SendMessages()
    Dim dbs As DAO.Database
    Dim rst As DAO.Recordset
    Dim strTo As String
    Dim strSubject As String
    Dim strMessage As String
    Set dbs = CurrentDb
    Set rst = dbs.OpenRecordset("tblData", dbOpenForwardOnly)
    Do While Not rst.EOF
        strTo = rst.Fields("Email")
        strSubject = "Report incomplete" ' could be made variable
        strMessage = "Dear " & rst.Fields("FirstName") & vbNewLine & _
            "Please complete your work hour report ASAP"
        DoCmd.SendObject To:=strTo, _
            Subject:=strSubject, _
            MessageText:=strMessage, _
            EditMessage:=False ' False to send, True to view/edit
        rst.MoveNext
    Loop
    rst.Close
    Set rst = Nothing
    Set dbs = Nothing
End Sub

 

You have to substitute the name of the table and of the relevant fields, and you can modify the strings as needed, of course.

You could keep a form open all the time, set the Timer Interval to (for example) 3600000 (the number of milliseconds in a day), and create an On Timer event procedure. In the procedure, check if the time of day is - say - between 9 AM and 10 AM, and if so, run the SendMessages procedure.

Q: Converting Publisher docs to InDesign - Microsoft Community

graeme_bennett say:

Q: Converting Publisher docs to InDesign

We've switched from Publisher to Adobe InDesign. How can I convert my docs?
graeme_bennett say:

Part of your question is best asked over on the Adobe site, where other users may be able to share their experiences. However, one easy way to republish fully-formatted Publisher docs is to print your publisher docs to one or more files as PostScript, using a PostScript printer driver and the print to file option from within Publisher. Then, use Acrobat Professional's Distiller (or a free alternative such as "GhostScript") to process the PS (or .prn) files into PostScript-valid PDF or EPS files. (Don't use "PDFwriter" -- it creates less efficient code that can cause printing quality and font-substitution problems.) Then, place the converted files into InDesign. If you use Acrobat Pro, you can edit the files, as well.
