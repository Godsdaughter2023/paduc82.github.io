why did my computer shut down by itself when I left it alone and on to - Microsoft Community

ogara say:

why did my computer shut down by itself when I left it alone and on to do full scan over night and how can I do a full scan without that happening

Can someone help me. I want to do a full scan with microsoft essentials over night. I tried this last night. I woke to find my computer off and only 1/4 of the scan had been done. Why would this happen, and how can I avoid that happening again? 
greginmich say:

In order to run an unattended MSE scan, we need the ability to wake up a sleeping computer and keep it awake for the duration of the scan. While Task Scheduler has a built-in option that will do this, MSE’s implementation of Task Scheduler disables that option. So the “normal” way to run an unattended MSE scan (or any unattended task) is to set up an independent scan task in Task Scheduler:

http://answers.microsoft.com/en-us/protect/forum/protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02

 

Of course we can also run unattended scans if we simply turn off the idle-to-sleep feature of Windows 7; but that feature is designed to help save energy in accordance with the EPA’s strategy of integrating the benefits of energy-efficient products and energy-efficient practices; and the EPA actually publishes guidelines specifically for the sleep-timeout setting. So let’s take a quick look at some alternatives that might be a little easier than setting up an independent Task Scheduler task, but that don’t involve defeating the main energy-saving feature of Windows 7:

http://www.energystar.gov/index.cfm?c=power_mgt.pr_power_mgt_manual_act_win7

 

1.) Run your MSE Full Scans “attended”

The MSE scheduled scan was really designed to be scheduled for a time when the computer is in use (which doesn’t require the ability to wake the computer) and to be run “attended” – which basically means not taking breaks longer than 30 minutes (the default value of the sleep-timeout setting). The CPU-throttling feature allows the scan to be run in the background while you work; but CPU throttling also increases the length of the scan, so it’s sometimes difficult to fit a Full Scan into a normal work session:

http://answers.microsoft.com/en-us/prmotect/foru/protect_start/auto-scan-not-working/8041f4c9-1088-4a67-bca4-54c9bd5a99bd

 

2.) Don’t run Full Scans

You can reschedule the MSE scan for a time when the computer is in use; and leave the scan-type setting at “Quick scan” – and simply not worry about running Full Scans. That might sound extreme; but you’re actually more likely to find active malware by running Quick Scans with third-party (on-demand) scanners than you are by running Full Scans with MSE; and the use of third-party scanners should really be a matter of routine, rather than just a reaction to obvious signs that something was missed; because not every piece of malware that MSE misses is going to announce its presence by changing your wallpaper or demanding a ransom for its cleanup services:

http://answers.microsoft.com/en-us/protect/forum/protect_scanning/when-we-run-a-full-scan/8ed1c27c-4ce0-44e5-a1a6-a2ed198061f3

 

3.) Toggle the Windows energy-saving feature instead of turning it off

We can run unattended Full Scans (either scheduled or manual) and still stay in compliance with the Energy Star guidelines if we manually change the sleep-timeout interval before and after each Full Scan:

http://answers.microsoft.com/en-us/protect/forum/protect_scanning/does-microsoft-security-essentials-keep-scanning/59c5f16b-4fda-4e25-86bb-9c94a7d866a6

 

I’ve suggested using desktop shortcuts to make changing the sleep-timeout setting a little easier; and we can also automate the reset by dropping the shortcut for it into the Startup folder:

http://answers.microsoft.com/en-us/prmotect/foru/protect_start/auto-scan-not-working/8041f4c9-1088-4a67-bca4-54c9bd5a99bd

 

4.) Run your Full Scans with a batch file that fully automates the power-management settings

You can launch a Full Scan with a batch file that fully automates the sleep-timeout changes needed for an unattended scan, and you can (optionally) include a shutdown.exe command line that will power off the computer at the end of the scan. You simply copy and paste the batch-file “script” into a Notepad file, give the file a name like “MSE Update and Full Scan”, and a “.bat” extension; and then save it to your desktop. This example scans only the system drive:

 

@ echo off

powercfg -change -standby-timeout-ac 200

cd C:\Program Files\Microsoft Security Client

MpCmdRun.exe -SignatureUpdate

MpCmdRun.exe -Scan -ScanType 3 -File C:\

powercfg -change -standby-timeout-ac 20

 

This works well for me because my hours are so jumbled now that scheduling a scan doesn’t make any sense, and scanning only the system drive cuts the scan time drastically, because I have 3 internal hard drives. You can also launch a conventional Full Scan with a similar batch file:

http://answers.microsoft.com/en-us/protect/forum/protect_scanning/switch-off-pc-after-scan/5dabd655-209b-4002-acb3-b77e1ce77ae1

 

 

GreginMich

tom ogilvy say:

select your data and in the home tab, go to Sort & Filter on the right side and select Filter from the dropdown.  This puts dropdowns in your header row and you can select what you want to see.

If you zipcodes are in their own column, then just select the ones you want to see.

If you have zipcodes mixed in with other address information, you can click on the dropdown and under text filters, choose contains, then type in a zip code.

If your zipcodes are mixed with other address information possibly you could extract the zipcodes to a separate column.  You could start with the Right function 

=Right(Trim(f2),4)

assuming F2 contains address information that ends with the zipcode.

If you have a list of zipcodes (which you allude to), then you could use the Advanced filter under the Data menu.  This allows you to set up a criteria range which could contain a list of zip codes.

I have a ComboBox in one Userform, which uses a named dynamic range in a worksheet as rowsource.

This list is simply an all-in-upper-case version of another named dynamic range and placed as another column side-by-side to the proper-case version of the same list.

To make things minimalistic and simple, I would like to eliminate this column (named range) from the worksheet and use the proper-case dynamic range to be used in this ComboBox's rowsource but somehow convert the items into all upper case.

How can I do this?

Maki

jan karel pieterse say:

First of all, you should convert your range to a table and convert the dynamic range name to a normal range name (the table takes care of the dynamics).

You could read the range into an array first and convert it in VBA:

Dim vArr As Variant

Dim lRow As Long

vArr = Worksheets("TheSheet").Range("TheDynamicRangeName").Value

For lRow = LBound(vArr,1) To UBound(vArr, 1)

vArr(lRow,1) = UCase(vArr(lRow,1))

Next

ComboBox1.List = vArr
