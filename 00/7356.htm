scottgem say:
You are raising different issues then you originally did. The original request for a database that displays an image and information about that image. That's very easy to do. As explained, you have a record with the information you want to display about the image. One of the fields in that record is the location of the image. 

You create a form based on the table using the Form wizard. This is SOP. To add the image, you go into Design mode and add an image control. Then use code in the ON Current event to assign the location of the image to Picture property of the control. That's about as step by step as we can get, plus you have Daniel's sample to illustrate how its done.

Now you are referring to a different scenario. Now you describe a page of images that can be clicked on to drill down to get more detailed information about the image. That can be done with an unbound form. The unbound form would contain a bunch of image controls hard coded to the images. You can then use the Click or Double Click events of the image controls to open forms that contain more details about the image.
