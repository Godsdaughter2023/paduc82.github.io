ken sheridan say: Here's are examples, each using two list boxes, the first listing employees, the second listing projects to which employees can be assigned.

1.  In the first example projects are returned where any of the selected employees is assigned to the project.  To do this a value list is built in code and assigned to a hidden text box in the form.  The second list box is then requeried to show those projects to which any of the selected employees are assigned:

The code for the first list box's AfterUpdate event procedure is:

Private Sub lstEmployees_AfterUpdate()

    Dim varItem As Variant
    Dim strEmployeeIDList As String
    Dim ctrl As Control
    
    Set ctrl = Me.lstEmployees
    
        For Each varItem In ctrl.ItemsSelected
            strEmployeeIDList = strEmployeeIDList & "," & ctrl.ItemData(varItem)
        Next varItem
        
        ' remove leading comma
        strEmployeeIDList = Mid(strEmployeeIDList, 2)
        ' assign value list to hidden text box control
        Me.txtEmployeeIDList = strEmployeeIDList
        
        Me.lstProjects.Requery

End Sub

The RowSource of the second list box is a query which uses the InParam and GetToken functions (the former calling the latter) published by Microsoft at:

http://support.microsoft.com/kb/100131/en-us

SELECT DISTINCT Projects.ProjectID, Projects.Project
FROM Projects INNER JOIN ProjectEmployees
ON Projects.ProjectID=ProjectEmployees.ProjectID
WHERE InParam([EmployeeID],Forms!frmFilterFormAnyEmployees!txtEmployeeIDList)
ORDER BY Projects.Project;

2.  In the second example projects are returned where all of the selected employees are assigned to the project.  To do this the RowSource of the second list box is also restricted by a subquery which restricts the rows returned to those where the count of rows returned by the subquery equals the count of items selected in the first list box:

SELECT DISTINCT Projects.ProjectID, Projects.Project
FROM Projects INNER JOIN ProjectEmployees AS PE1
ON Projects.ProjectID=PE1.ProjectID
WHERE InParam(EmployeeID, Forms!frmFilterFormAllEmployees!txtEmployeeIDList)
AND (SELECT COUNT(*)
         FROM ProjectEmployees AS PE2
         WHERE PE2.ProjectID = PE1.ProjectID AND
         InParam(EmployeeID, Forms!frmFilterFormAllEmployees!txtEmployeeIDList)) =
Forms!frmFilterFormAllEmployees!lstEmployees.ItemsSelected.Count;

To extend this to a third list box you'd do the same with the second and third list boxes as done with the first and second in the above examples.

To restrict the form to the final selection the form's RecordSource would be a query which calls the InParam function in the same way as the first query given above, referencing a value list in a hidden control generated in the final list box's AfterUpdate event procedure, but requerying the form this time.

Hello ArulmuruganS:

 

Thank you for posting to the forum regarding you question about call history.  The call history is heavily tied to your mobile operator.  This update issue you are seeing may be related to issues on the providers network.  Before we settle on this there are a few things we should try on the phone.

  • Check the call history carefully to make sure that old and new calls are not being merged.  Often times the call histories of a contact will be put together, be sure that the new and old calls are not showing up in the same string.
  • Go to the phone and tap settings > applications > Internet Explorer > delete history.  After you do this, go to settings > emails + accounts > tap and hold the Windows Live account and select sync.
  • If you continue to have this issue try performing a soft reset of the phone.  The easiest way to do this is to pull the battery for 2 minutes, replace and restart the phone.  If you are unable to remove the battery, consult your owner's manual or contact your phone manufacturer.

If you are unable to make progress with all of the above, contact your mobile operator to inquire why all calls are not being track or added to your call history.
