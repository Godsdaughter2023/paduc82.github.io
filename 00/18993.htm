[Bug Report] auto-tagging's msclkid query string parameter - Microsoft Community

stevemaguire say:

[Bug Report] auto-tagging's msclkid query string parameter not added properly in some situations

Hello,

I've found that the new auto-tagging feature has a slight bug in it when used with links that don't have query strings.

Right now, when auto-tagging is on, Bing adds this query string parameter to the tracking template:

&msclkid=...

Many tracking systems that work fine with Bing don't use query strings but break when auto-tagging is enabled simply because Bing doesn't verify that a query string exists before adding the parameter. This is a common oversight for programs that add parameters to query strings, and fortunately, it's really easy to fix:

If the tracking link does not already have a query string, simply add the msclkid parameter with a "?" mark rather than a "&":

?msclkid=...

By using the "?" mark, that creates the query string, whereas the "&" adds to an existing query string.

If you pass this on to your developers, they'll understand the problem within a microsecond.

It's easy enough to turn off auto-tagging and the conflict goes away, but auto-tagging gets enabled automatically in some cases, such as when creating a conversion goal.

Thanks,

- Steve

reynaldo ugarte say: Please follow the below instructions:

1-In Windows, right-click Start, and then select Run. In the Open: box type regedit, and then click OK. This will open the registry editor.

2-Find this subkey in the registry and then click it:

HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows

3-Click Edit > New > Key, and name the new key Windows Search.

4-Select the new Windows Search key.

5-Click Edit > New > DWORD Value.

6-Type PreventIndexingOutlook for the name of the DWORD, and then press Enter.

7-Right-click PreventIndexingOutlook, and then click Modify.

8-In the Value data box, type 1 to enable the registry entry, and then click OK.

Exit Registry Editor, and then restart Outlook

A clean install is required since it is a different version of Windows 10. Enterprise is not included in the free upgrade. There are diff upgrade paths for volume users.* You will have to have Windows 8.1 installed to qualify for the free upgrade and if you want win 10 Pro, you'll need to have win 8.1 pro. Download the latest iso file for windows 10 pro and do a clean install. Installing on a separate partition or second hard drive is recommended so you can keep your Enterprise installation. Testers are supposed to get the final version automatically, but that is not clear yet. From MS about the free upgrade--

* Some editions are excluded: Windows 7 Enterprise, Windows 8/8.1 Enterprise, and Windows RT/RT 8.1. Active Software Assurance customers in volume licensing have the benefit to upgrade to Windows 10 enterprise offerings outside of this offer.
