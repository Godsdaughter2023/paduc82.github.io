steve rindsberg say:

It sounds like you've got a transition sound, not an animation effect.

With your presentation open, go to Slide Sorter view and select ALL slides.

Click the Transitions tab.

On the right side of the tab, in the Timing group, make sure it says "No Sound" next to "Sound:"

Click "Apply to all"

Try your presentation again.

Assuming that fixes it for all existing slides, let's also make sure that the masters are sound-free.  Choose View | Slide Master.

Click the topmost thumbnail image on the left and again choose Transitions | Timing | Sound: No Sound

Click the indented thumbnail just below the first master, scroll to the bottom, SHIFT+Click the last thumbnail and again do Transitions | Timing | Sound:  No Sound.

If there's more than one master, repeat the steps above for each master.

First Impressions of build 9926: No. - Microsoft Community

dustiiwolf say:

First Impressions of build 9926: No.

Seeing as I've found myself with numerous grievances with build 9926 (enough to put me back to using 8.1, rather than "roughing it" through the preview), I figured I would write about my first impressions and see what discussion comes of it.

Installation

I seriously hope that this is just a preview issue, but as it stands, one cannot upgrade from Windows 8.1 on most WIMBoot machines. Not everyone can afford decent computers with an SSD and a minimum of 64GB memory. This is something that has caused quite a headache for me (trying to find ways around it before backing up my disk and saying "to heck with it!" and doing a fresh install.), and others too.

So, I go ahead and do a fresh install, and lo and behold, the touchpad off the bat is not installed (despite Windows supposedly having drivers built in for these things and the hardware being a simple Synaptic's clickpad). So, I have to drag out my USB mouse and go through the setup (which never asked me about sync settings; It just assumed i wanted the theme, explorer, and accessibility settings from my old tablet I no longer own which had settings optimized for touch). So setup completes and now I'm logged in.

1 Step Forward, 2 Steps Back: The Worst of Both Worlds

At first glance, the latest preview looks pretty nice. However, build 9926 is the epitome of "Don't judge a book by it's cover". In an attempt to appease both touch and mouse users, Microsoft has been working on merging the interfaces, which was working for them up to this point; Then they just had to go a spoil everything. Many issues were present (not all of them bugs) in many areas. To list some of them:

  • The Start menu is slow to respond (even post patch)
  • The live tile layout, while in concept is better, is completely destroyed (live tiles on my 1366x768 screen were small and fuzzy, and the counter text was oversized), especially when compared to previous builds (where the actual implementation was better, despite lacking the needed resizability and scalability that is poorly demonstrated in the current build)
  • Again, as with the live tiles, the All Apps layout is poorly implemented within the latest build, with earlier builds having a much more sensible implementation (this feels like its using the layout from previous builds, but the implementation of the All Apps screen in Windows 8.x, making for a hard to navigate all apps menu)
  • Letterboxed Dialogs. Remember those modern dialogs in Windows 8.x that would dim the screen and display a message in a banner the spanned the entire width of the display? Well, now it's cut off at both ends, only displaying the box from where the text begins to where the buttons end, and the screen never dimmed as well, making for a jarring experience (i didn't know a dialog had displayed till i realized that other elements weren't responding.)
  • Cortana really feels weird in that little box of hers that pops up from the taskbar. I appreciate the inclusion of her, but the interface feels all wrong. I can already see 2 better methods of implementing her ui:
    1. Have her appear in the start menu (in place of the All Apps display) once a user starts typing, presses the mic icon, or says "Hey Cortana". Have the start menu expand to display results, replacing the entire menu display with Cortana
    2. Have her slide out from the side in a "sidebar" that hovers over the desktop, similar to charms or the share dialog, and expand the sidebar for results display.
    3. For smaller tablets, just duplicate the experience from Windows Phone. No need to complicate things there.
  • Windows 10 isn't so charming anymore. No seriously, where are the charms? While, granted, they were better suited for touch, the complete removal of them feels awkward, especially as all the dialogs and such that came with them (exceptions being app settings and share) are either gone or repressed. Clicking the network icon in the taskbar now requires me to wait for the Settings app to open (if it will open) to select a wireless access point, and hinders anything I may of already been doing in the first place with Settings. Again, this makes no sense, as the original charms themselves were quite intuitive and mouse friendly, but rather, the method of accessing them itself was a bit one-sided (that side being touch).
  • Less is definitely not more, if the new settings app is to be believed. Not only is it less friendly for any input (Icons don't always equate to "better for a mouse"), but there are a considerable amount of settings missing (backup, onedrive, and more). The interface, overall, feels messy and with very little thought put into it. It's kind of like someone looked at Control Panel in category view, replaced the icons with weird new outlined versions, and then stripped any kind of setting they figured touch users wouldn't like.
  • While the new titlebars are decent, they still feel weird, and there has yet to be any kind of adaptive font color implemented for darker themes (which is a must have). The text also doesn't line up quite with the window's manipulation buttons, making it feel improperly aligned.
  • It would appear there is no longer a modern file picker, or at least, it didn't pop up when I went to change my account photo (I wanted the one I'm using in the People Hub). The ability to access files within other applications is key.

So as one sees, a lot has gone awry in the latest Windows build where the interface is concerned. Makes me worry about the Windows Phone preview (I wasn;t to fond of the new background system seen in the presentation. Much prefer the current one in 8.1, especially since numerous developers updated their apps with transparent tiles for the current implementation.)

To the... cloud? Maybe not.

Once again, Microsoft has decided to backtrack, as if Windows 8.x was some vile thing they wish to forget, and are killing good things in the process. What I mean is, the integrated OneDrive implementation found in Windows 8.1 and earlier Windows 10 builds is gone. And while this isn't too big off a deal (It still works the same for most of you desktop using folks), there are two glaring issues. The first is, with the newer integrated version, external drives are no longer an option for the location of your onedrive folder (in fact, the "Location" tab is completely missing from the folder's properties page. You have to delete the OneDrive folder and restart OneDrive to change its location). So to those 32GB and 16GB machines with limited space but a [SD] card in place, kiss goodbye to ever using any OneDrive files offline, as if Microsoft doesn't change this in future builds, you're looking at a forecast of sunny skies without a cloud in sight. The second issue I previously mentioned is the omission of a touch app. Now maybe I just got a faulty install, but I looked and didn't find a single onedrive app in the app list. What this means is touch users are back to using windows explorer and changing the icon size for their fingers.

I can has pinned sites?

Now this may have to do with the way Windows 10 syncs, but considering Windows 8.1 never did this with the same exact settings enabled, I'm not holding my breath. We all know Windows 10 has done away with Modern IE and Microsoft is prepping IE's successor, project Spartan (IE12? IEx?). However, remember when you had that windows 8 machine, and pinned all those tiles in modern IE? Well, they've just been synced to your computer (despite apps being turned off in sync, and those pinned sites being previously removed). I had dozens upon dozens of pinned sites I didn't even know I pinned, that synced down, presumably under "browser" settings. (on my previous Windows 8.x machines, they were either under Apps or Tiles (can't remember which at the moment, as I always have both of those options disabled in sync.).) This was easily remedied though, as in 10 I could actually right click them and view the shortcut source (to which I ctrl+a 'ed & del. 'ed them). Oh, and since i'm on the topic of IE, despite my home page being set to nothing but about:tabs, the Internet Explorer Preview page launches every time you open IE. EVERY. TIME. Even in no addons mode. It also seemed to render pages poorly in comparison to IE11 on Windows 8.1 (I had noticeable lag and page issues requiring a refresh or more.)

Drivers? Hahaha. Oh, youre serious.

"Drivers for basic functions like storage, networking, input, and display come with Windows. These drivers allow you to complete the Windows installation and connect to the Internet. You might be able to get more drivers from Windows Update." - Windows Insider ('Is it right for me?')

This is seriously misleading. I have a simple Synaptics Clickpad, which supports generic HID mouse input drivers. Yet, for some reason, It's incompatible with Windows 10 (as well as the Windows 10 setup environment and recovery environment, to which the Windows 8.1 setup and recovery environment DOES support.). I had to use a USB mouse to manually install the drivers. Even then, I couldn't install the generic Synaptics drivers from the Synaptics website, as the installer said I already had a newer version installed (lies), and attempting to install using the inf manually resulted in "file not found" and a once more non-functional touchpad I had to rollback (this issue I've experienced before, and is typically due to an error in the INF file, but unlike in the past, I couldn't find anything noticeably wrong with it). I also had numerous drivers related to the Intel Dynamic Platform that had to be installed. But hey, the screen and speakers worked.

Conclusion

Amongst all the lag, bugs, and downright errors on Microsoft's behalf, along with the issues mentioned above, Microsoft has really let me down. I watched their webcast, got my hopes up for a new and improved preview, and then had them dashed. TBH, I haven't had a preview this bad since the Windows 8.0 Developer Preview. Now, the latest build isn't all bad; The new universal apps that were included look pretty good, and the store is headed in the right direction with it's interface (though it needs everything: apps, games, music, movies, books {Nook powered?}, etc..). They also showed with this build that they have some pretty great ideas. But in the end, It just doesn't work.

No, really. I'm back on 8.1 as of writing this.

Please note that as I am a person on the internet and not a journalist or someone being paid, that this is my personal opinion, and does not necessarily reflect the views of others. If I was a journalist and or being paid, these would still be opinions, none the less, but the point stands.

trevor pott say:

Build 9879 issues. Warning: nit picky!

Below are a list of issues I have with the current build of Windows 10, in no particular order.  Some (okay, a lot) of these are really nit picky. 

I would also like to point out for the record that I am not asking Microsoft to apply my personal UI preferences to all and sundry, or to make them default, so on and so forth.  I've done the dance of "you're just a filthy customer, you're supposed to take whatever UI Microsoft feeds you and like it" during every previous iteration of Windows.  Let's put that discussion to one side and note that when I discuss UI elements here I am asking for the option of choice, not to have my person views or desires foisted upon everyone. 

I would also gladly pay a little extra for a "Windows 10, choice edition", where the cheap-o version was Windows UI design-by-fiat and the "choice edition" contained a bunch of little nerd knobs that we could twiddle until we had a UI that worked for us.  Though if Microsoft chooses to go this route, I respectfully request they give us the ability to "export UI configuration" to an XML that we could then load onto another user on another PC so that we don't have to spend all our time hunting through registry hives to pull that stuff out into a .reg file.

All of these issues have, of course, all been submitted through the official feedback mechanism.  I am posting it to the forum because I am running on the "fast" track, and felt that it would be good to have a talk about the issues with other Windows Insiders before Microsoft releases another "stable" version.

My hopes here are not to chastise Microsoft, but rather to spark ideas in others for their feedback.  Maybe there are items of consensus amongst my feedback, maybe I'm just completely off base on some items, and maybe some items are unique to my setup. 

With that said, thanks to Microsoft for doing a Technical Preview and giving us the opportunity to voice our concerns and thoughts.  Here are my nit picks with the current version. 

Windows General

Some weird crashes are bothersome, but the big issue is the things that "broke" in the latest upgrade.  A great example: closing the lid on the laptop now only "sleeps" the computer about 25% of the time.  When the computer comes out of sleep, about 1/10th of the time it doesn't fully wake up.  The light "breathes", but no combination of button presses will wake it.  Hard off then back on is the only way.

Still have issues with text in the actual window chrome: when a window is not primary focus, you can't read the title text.

Can we please have an option to get away from the steamrollered-flat square look?  Preferably something without the wasted space for text everywhere?  PIXELS!  They're important.  Especially when you are stuck on 1377x768 (still the most common resolution.)

I'm not a fan of the overly intrusive chrome.  Things like Windows Media Player as an overlay that eats 1/8th the screen on a 1377x768 screen when I play a song?  What?  WHY?  Similarly, notifications are a pain.  Some giant box that pops up and eats 1/8th the screen to inform me Dropbox has received a file?  Who thought that was a good plan?  I have sads.  Many, many sads.

WiFi seems to be unable to exceed 100Mbit/sec no matter what I do, or what hardware I use.  Unsure if Windows 10 or an issue with the Intel Drivers for my various WiFi adapters.

A number of items are in Windows Firewall which don't have proper "names".  @icsv.cll, ajrouter.dll, Cortana, etc.

Can we please have spell check integrated at the OS level?  This is built into modern browsers like Chrome and Firefox.  It's built into proper instant messengers, mail apps, you name it.  Why are we still typing into things like "feedback boxes" or "the Windows Preview Hub" or Internet Explorer without a basic spell check?  Why isn't this functionality available for everything from Notepad to the command line to IE?  Spell Check is so fundamental to modern OSes, apps and browsers that using an OS/apps without it feels a little like banging two rocks together.

Task Manager

Task Manager can take up to 1 minute to show up after "right click on task bar, task manager" approximate every 10th time.  

Task Manager doesn't group applications.  (For example, it would be nice to see "Chrome takes up this much RAM" instead of each individual chrome thread.  I'd love the ability to see "Chrome" threads as a group, then "expand" them as a tree to see what's eating RAM on a per-thread basis.)

Task Manager's RAM consumption under "processes" doesn't add up to the amount of RAM consumed under "performance".

Task Manger doesn't have a means by which to set the CPU display to "logical processors" and have it stay that way as a default.

Task Manager's disk option doesn't have the ability to show Disk Queue Length, instead showing Active Time (which is far less useful.)

Using Multiple Source Files in Mail Merge-MS 2010 - Microsoft Community

susancollicott say:

Using Multiple Source Files in Mail Merge-MS 2010

Hi-

Can you use multiple source files for one document in Mail Merge-vs. 2010?  And if so, how do you set it up?

Many thanks!

macropod say:

If your data from the secondary sources are required in a table format, you could use DATABASE fields in the mailmerge main document to import the data from those other sources.

For example, suppose you have a mailmerge with a data source listing students and various data concerning them in one file that you want to use as your main data source and another file containing information about their classes, birth dates & teachers in another file, from which you want to list all students in the same class. You'd set up the mailmerge main document in the normal way, pointing to the main data file and, to extract the data from the secondary file, you might use a DATABASE field coded like:

{DATABASE \d "C:\\Users\\Susan\\DataFiles\\Students.xlsx" \s "SELECT [LAST NAME], [FIRST NAME], FORMAT([BirthDate], \"DDD, D MMMM YYYY\") AS [Birth Date], [Class] FROM [Master List$] WHERE [Class] = \"{ MERGEFIELD Class }\" ORDER BY [LAST NAME], [FIRST NAME]" \l "37" \b "51" \h}

where 'Master List' is the name of the worksheet concerned. Note, too, that the field braces (i.e. { }) are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practicable to add them via any of the standard Word dialogues.

The  \l "37" \b "51" at the end of the DATABASE field are parameters setting the table format. Details of field switches that control the table format are given at: http://office.microsoft.com/en-us/word-help/field-codes-database-field-HP010260104.aspx. That link refers to a 'Table AutoFormat dialog box', which you can access via Insert|Quick Parts|Field>DataBase. You'll need to set up a connection to the datasource before you can access the dialogue. Alternatively, you could create a temporary table in the document then use the following macro:

Sub ShowTableAutoFormatDialog()
Application.Dialogs(wdDialogTableAutoFormat).Show
End Sub

The dialog box options show what formats can be used. To use them, make a note of which table # you choose from the listbox (not its name) and which other formatting options you've checked. You'll need both of those for the \l and \b switches.
