dhookom say:

You can create a function that uses Split() to push the memo value into an array and return a specific section (numbered 0 to whatever):

Public Function SplitString(strText As Variant, intSection As Integer) As Variant
    Dim arText() As String
    Dim intLastSection As Integer
    If IsNull(strText) Then
        SplitString = Null
     Else
        arText = Split(strText, Chr(10))
        intLastSection = UBound(arText)
        If intSection = -1 Then   'get the final
            SplitString = arText(intLastSection)
         Else
            If intSection <= intLastSection Then
                SplitString = arText(intSection)
             Else
                SplitString = ""
            End If
        End If
    End If
End Function

To get the first "row" of the memo field, use:

SplitString([MemoField],0)

To get the last "row" use:

SplitString([MemoField],-1)

graham mandeno say:

I'm glad to hear that your database is properly designed after all!  :-)

What you need is a query which includes fields from both tables, but only selects the record from the "ManyTable" which has the maximum (latest) date of all the records thay match the "MainTable" primary key.

The query's SQL should look like this:

SELECT
    [MainTable].[Field1],
    [MainTable].[Field2],
    [MainTable].[Field3],
    [ManyTable].[DateField],
    [ManyTable].[MemoField]
FROM
    [MainTable] INNER JOIN [ManyTable]
    ON [MainTable].[PrimaryKey] = [ManyTable].[ForeignKey]
WHERE
    [ManyTable].[DateField] =
    (SELECT Max([DateField]) FROM [ManyTable] AS T1
    WHERE T1.[ForeignKey] = [MainTable].[PrimaryKey]);

Of course, you will need to change the table and field names to match your own.

Good luck!  :-)
Graham

T1 is an alias for ManyTable in the subquery.  Because ManyTable is already included in the main query, you need a way to distinguish between the two.
