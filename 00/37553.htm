ethan b. say:
Hey munchiesnOOb,

Unfortunately, what you're experiencing is a new feature on Windows 10 since the Fall Creators Update. It dates back to the old days when Windows 95 came out, and applications that you left while restarting or shutting down reappear when you start up. This isn't the kind of stuff that would pop up in your Startup tab in Task Manager. I've lived it myself, I would restart my computer and it would connect me back to the same Discord session I was in before I would reboot, *right* before I unlock my computer.

You can do the following to hinder any applications from reopening after a reboot: go to Start, type in, without quotes, "Sign-in options", and click the item that pops up with that name. Under the Privacy section, turn off "Use my sign-in info to automatically finish setting up my device after an update or restart."

Now, according to this thread, there are mixed results, but it seems to work fine for me: https://superuser.com/questions/1229963/windows...

Personally, I just came around to this method and it works for me on version 1709, build 16299.402. Given that this solution works well, Windows will stay true to what's set up in the Startup tab in the Task Manager... for the most part.

There's nothing wrong with being too careful, and you can use Sysinternals Autoruns to further change what is enabled on startup, and what isn't: https://docs.microsoft.com/en-us/sysinternals/d...

The Startup tab in Task Manager is not a be all, end all solution, so Autoruns should do the job further, even if you fix the initial issue of applications reopening after a reboot, it's a nice utility to see what starts up after logging in.

Hopefully that helps. Let me know if you need any further assistance.

burrwalnut say:

Here are a few more of the major keys that can be used:

 

1. Check all the \run, \runonce, \runservices and \runservicesonce keys (not all of them will necessarily be present).

 

2.. HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon\UserInit. This points to the program C:\WINDOWS\system32\userinit.exe and the entry ends with a comma. Other programs can be started from this key by appending them and separating them with a comma. Malware sometimes uses this key.

 

3. HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon\Notify

This key is used to add a program that will run when a particular event occurs. When Winlogon.exe generates an event, Windows will look in the Notify key for a DLL to handle this event. Malware sometimes uses this key.
