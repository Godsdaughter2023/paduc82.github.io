rohn007 say:

No you don't have to repair, upgrade or update anything. All a waste of time for this specific problem!

This is a symptom that shows up after the application has abended/crashed, or Windows was powered off. The actual problem is that the "lock file" used by Excel (and Word) is left behind.  When the file (and application) is closed normally, the "lock file" is deleted.  This is a remnant of a 1990's answer to the problem of more than one person trying to edit files on that wonderful new invention, the "corporate Local Area Network (LAN)"!  Before that, the files were only stored on the local HD, so there had been no provision for file sharing. File sharing was done via "sneaker net" and literal floppy diskettes. Back then the applications assumed that they had exclusive access the file, BECAUSE THEY DID. This "lock" file was a way for the application to warn you that someone else already had "locked" the file for editing. And if your Office installations were configured correctly at installation the name of the user would be displayed.

.

The reason it is not showing a name is that the information has not been saved to Office. In Excel, go to File menu > Options command > General option > "Personalize your copy of MS Office" and fill in your name (it is probably blank right now). This info is stored in the registry and shared with all of the installed Office Applications.  If you go to this option in Word, there is also a field for your Initials (used when secretaries provided their "typed by" initials, a leftover of the typewriter era).

.

To fix this problem, what you have to do is find and delete the "orphan" lock file associated with the file.  The lock file replaces the first 2 char of the file name with "~$".  It should be in the same directory as the file itself.

.

PS: While you are in the Options dialog go to the Save option.  In there you can define custom locations for the application to put autorecover files and the default file save location.

Drafts (This Computer Only)

I installed Office 365 and added my Outlook.com account. The drafts folder appear as "Drafts (This computer only)". How to make it use the Drafts folder in Outlook.com? How can I remove the "this computer only" warning?

roady [mvp] say:

You can't. An Outlook.com account in Outlook 2013 is connected as an EAS account and an EAS account doesn't support synching the Drafts folder.

You could configure it as an IMAP account instead, but the IMAP protocol doesn't support a whole lot of other features such as Flag Reminders, Categories or synching Contacts, Calendar and Tasks Folders.

See: http://www.howto-outlook.com/howto/accountsettings.htm

A workaround would be to send the message to yourself and then use Forward or Resend This Message to continue it. You can create a rule in Outlook.com to move messages which are From and To yourself to another folder.

footnote continuation notice - Microsoft Community

chris_655 say:

footnote continuation notice

Can I remove the "footnote continuation notice"? It continues to appear in areas where the footnote does not continue to the next page. All it ends up doing is adding extra space at the bottom of the page.

a. user say:

Hi Chris. I am a fellow community member here to help. You can remove it without an issue.

stefan blom say:

Switch to Draft view (click Draft on the View tab). On the References tab, click Show Notes. Word will display footnotes in a pane at the bottom of the document window. In the drop down, change "All footnotes" to "Footnote continuation notice" and clear its contents.

Also, you may want to inspect your footnote text and be sure to remove any blank paragraphs (¶) which may flow onto the next page, causing the footnote continuation notice to appear.
