Spambot leaks more than 700m email addresses in huge data breach  Technology  The Guardian

This is a great scam for the attackers, since the data being used to create the messages is real, it's just typically very old, so in most cases people receiving the messages tell us it was an old, currently unused password.

Restore a mobile device from an ActiveSync 4.5 backup, when main - Microsoft Community

steve tuck say:

Restore a mobile device from an ActiveSync 4.5 backup, when main computer running Mobile Device Center 6.1

Hi,
I've just had an *interesting* experience restoring my trusty old Dell Axim X50 pocket PC from a backup made around 6 months ago.

Back then I was running Active Sync 4.5 under Windows XP.  It generates an "STG" file which I had put away on a fileserver volume for safekeeping, so I could always get back to a working device image (the X50 has a nasty habit of losing its programs if the backup battery is left to go flat).

Since the backup was made, I'd upgraded my machine to Windows 7.  Mobile Device Center recognized the X50 and all was happy.  Until I needed to restore.  CLANG! note the following gotcha's:
- Windows Mobile Device Center (6.1.6965 according to the About screen) cannot open STG files created from ActiveSync 4.5
- Downloading and installing AS4.5 on Win7, gives a warning saying "this is not compatible"
- WMDC 6.1 also creates / reads files with extension STG but they are not compatible.  WMDC 6.1 just barfs if you try to open older STG files (it doesn't even suggest what you should do about this).

Also note:  Windows itself does not recognize the extension STG, nor does Microsoft's website - pretty darn hilarious!  Look at this:  http://shell.windows.com/fileassoc/0409/xml/redir.asp?EXT=stg

So - beware - your backup images are probably only good with your current version of WMDC 6.1 and desktop OS used for the backup/restore.  Don't assume these files have any longevity or usefulness beyond that!

My workaround is:
- Restore the mobile device on a computer running AS4.5 and Windows XP
- Move the device onto the new computer running WMDC6.1
- Backup immediately to the default folder (which will be something cryptic like C:\Users\Steve\AppData\Roaming\Microsoft\ActiveSync\Profiles\c6afb643)
- Copy this file from there, to somewhere with permanent storage e.g. a network volume that gets backed up

Goes to say, if/when MS bring out any new versions of WMDC, keep the downloaded install files also on the backed up volume, otherwise your backup files may be useless in future.

This total lack of forward / backwards compatibility is pretty lousy, and is a good reason for not relying on built in backup/restore software in any OS.

Anyone know any good 3rd party backup software for these?

Interesting dilemma to run into.  In the past when I did backups, I would back up my device to a storage card so I could run it from the card to the device.  I have changed what I do with a device and find the need for a total device backup unnecessary for my needs.  As long as I have my files on a storage card, I am secure.  That said, if I were going to backup, I would use Sprite Backup or Spb backup.  Both are outstanding solutions for FULL device backups.
