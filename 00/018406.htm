    don cuthbert say:

    I also have some VMs which still have digital license for Windows 10, inherited from the older product-key-activated Windows 10 Insider builds from prior to July 2015. There was one old Insider Product Key that still worked to activate Windows 10 Pro on new devices until August 2016, but only for activating a few old expired Insider Builds, which then had to be upgraded to the current public build to retain activation.

    The Internet Explorer Product Key, shown by Nirsoft's ProduKey App, often is the same as the previous OS Windows Product Key. So if you had upgraded a PC from Windows 8.1, IE would still show the Windows 8.1 Product Key. If you upgraded that VM from an Insider build, it may be showing one of the old Insider Product Keys. There are 2 generic Product Keys in use by most of the free upgrades, one for Home and another for Pro. But these keys are not used for activating Windows. They are more or less a placeholder, although if you changed the key, it would remove the activation status, at least until you changed it back, or clean-installed Windows without using a key.

    I don't know if you can move a VM from one host to another and transfer the digital license, but maybe. It doesn't work between a Desktop and a Laptop. I believe you would have to use the Activation Troubleshooter and a Microsoft Account to do it, if it will work. Another question is whether you could transfer a digital license from a VM to a physical PC, if it does not use a Product Key for activation (such as one of the free upgrades to Windows 10). The Software License Terms do allow for transfer of upgraded licenses for Stand-alone software:

    "Stand-alone software. If you acquired the software as stand-alone software (and also if you upgraded from software you acquired as stand-alone software), you may transfer the software to another device that belongs to you. You may also transfer the software to a device owned by someone else..."

    You can upgrade from any activated non-expiring build, such as a July 2015 build to the August 2016 public build or even to the older November 2015 public build, if you have install media for it. It does not have to be an Insider Build.
