ACHTUNG!!!! Du darfst dann aber NICHT den PC und die Konsole gleichzeitig im Netz betreiben!!! (Es geht zwar nicht die Welt unter, und das Netz stürzt nicht ab, aber Dein PC und die Konsole bekommen dann immer Datenpakete mit denen sie nichts anfangen können, weil eben der andere die Anfrage gestellt hat, und die Verbindung ist dann kaum richtig möglich).

rajesh govind say:

Hi,

 

The reason why the File History does not recognize the new hard drive is because, whenever we setup the file history on an external hard disk, a specific hardware ID is linked to the whole process. If in between we end up removing this hard drive and connect another one, the computer will not recognize it as the hardware ID for the new hard disk will be different from that of the first one.

 

So, in this case if you want to use two hard drives to create file history back up, then you may run the File History and complete the backup on one of the hard drive and then turn OFF the file history. Then, you may connect the other hard disk and start file history all over again.

    Port Forwarding

    1. Open a web browser.
    2. Enter http://192.168.1.1/admin in the address bar and press enter.
    3. Enter “admin” for the username and “admin” for the password when prompted.
    4. Select Advanced Setup.
    5. Select NAT.
    6. Select Virtual Servers.
    7. Click the Add button.
    8. Enter the Server Name.
    9. Set External Port Start and End to 3074.
    10. Set the Protocol to both if it is available, if not create an additional rule so you have one for TCP and one for UDP.
    11. Set Internal Port Start and End to 3074.
    12. Enter the IP Address of your Xbox console for Server IP Address.
    13. Click the Apply/Save button.
    14. Click the Add button.
    15. Enter the Server Name.
    16. Set External Port Start and End to 3544.
    17. Set the Protocol to UDP.
    18. Set Internal Port Start and End to 3544.
    19. Enter the IP Address of your Xbox console for Server IP Address.
    20. Click the Apply/Save button.

     

     

    Disable Port Forwarding

    1. Open a web browser.
    2. Enter http://192.168.1.1/admin in the address bar and press enter.
    3. Enter “admin” for the username and “admin” for the password when prompted.
    4. Select Advanced Setup.
    5. Select NAT.
    6. Select Virtual Servers.
    7. Check the Remove box next to any of the rules you created.
    8. Click the Apply/Save button.
