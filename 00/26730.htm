    Scenario:
    Game scores are kept in a range of cells (A6:B20) with the first row being column labels. The handicapped scores in column A need to be ranked and returned in descending order. There may be two or more matching scores. These need to be used to look up the player's name for a scoreboard tally in columns D and E at the top of the worksheet.

    Solution:
    You can enumerate and rank the scores with the standard LARGE(…) function but you will need an array formula for the multiple placements with tied scores.

        

    D3 is a standard formula,
        =IFERROR(LARGE($A$7:$A$999,ROW(1:1)),"")
    Copy or fill down until all scores are retrieved.

    E3 is an array formula.
        =IF(LEN(D3),INDEX($B$7:$B$999,SMALL(IF($A$7:$A$999=D3,ROW($1:$993)),COUNTIF(D$3:D3,D3))),"")
    This requires Ctrl+Shift+Enter to finalize rather than just Enter. Once entered correctly, it can be copied or filled down until all players have been matched to their scores.

    Formula Breakdown:

    The standard formula in D3 produces a ranked list of the scores in descending order.
  1. a) This is a fairly standard use of the LARGE(…) function with one extra thrown in.
    b) The ROW(1:1) is used as the LARGE(…) function's k parameter (aka position). When filled down to subsequent rows, this will advance to ROW(2:2), ROW(3:3), etc. The result is a series of numbers (1, 2, 3, etc) that are used as k within the LARGE(…) function to return the highest score, then second place, third place and so on.
  2. LARGE(…) will return duplicate values, ordered as they appear in the list. There are six duplicate scores in this data set; 541, 538, 523, 498, 490 and 0.
  3. The IFERROR(…) function is used as a wrapper to return an empty string (e.g. "") when there are no more values to return.
  4. This is a standard formula and is entered conventionally. Once in place it can be copied down to generate a full ranked list of the scores found in column A.
        =IFERROR(LARGE($A$7:$A$999,ROW(1:1)),"")
  5. If the scores were required to be returned in ascending order, simply substitute SMALL(…) for LARGE(…).
    The array formula used in E3 produces cyclic processing to return the associated players.
  1. The outside IF(LEN(D3), …, "") wrapping function avoids errors. It checks the length of the contents of D3 and only processes the primary functions of the formula if a value (score) exists. If there is no score in column D, an empty string is returned.
  2. a) The INDEX(…) function returns a value from B7:B999. The returned value is governed by INDEX(…)'s row_num parameter.
    b) The row_num is returned by a combination of the IF(…) and SMALL(…) functions. It is important to note that when looking to return a row number within the B7:B999 range, you are not trying to return a number between 7 and 999. That would return the row number on the worksheet, not within B7:B999. You want to return a row number between 1 and 993 which is what the ROW($1:$993) formula returns.
    c) The IF(…) statement is used to only consider rows where the score matches the score in column D being processed. The resulting array of row numbers is passed into the SMALL(…) function's array parameter.
    d) The COUNTIF(…) function helps SMALL(…) return the first, second, etc. value by supplying SMALL(…)'s k (aka position) parameter. If the score in column D is the first time that score has occurred from D3 down to the row being processed, the COUNTIF(…) returns 1 so SMALL(…) will return the lowest row within B7:B999 that that score can be found on. If the score appears a second time further down the list, COUNTIF(…) will return 2 and SMALL(…) will return the second lowest row that the score is found on. This is the real basis of how the first, second, third, etc values are returned. The process can be illustrated by putting just the COUNTIF(…) portion of the formula into F3 and filling down.

            

    The above image demonstrates the use of COUNTIF(…) to supply an ordinal based upon the value and occurance of each score.
    e) INDEX(…)'s column_num parameter can be ignored as the B7:B999 range used in the array parameter is a single column.
  3. Stitched all together and it comes up as,
    e) INDEX(…)'s column_num parameter can be ignored as the B7:B999 range used in the array parameter is a single column.
  4. Stitched all together and it comes up as,
            =IF(LEN(D3),INDEX($B$7:$B$999,SMALL(IF($A$7:$A$999=D3,ROW($1:$993)),COUNTIF(D$3:D3,D3))),"")
    This formula requires that it be entered with Ctrl+Shift+Enter tapped together. When entered correctly in this manner, Excel will wrap the formula in mathematical braces; e.g. { and }. You do not type these characters yourself. When entered correctly, the formula can be copied or filled down to catch all valid scores in column D.

    Function Reference (2010): 
    IFERROR function¹ Returns a value you specify if a formula evaluates to an error; otherwise, returns the result of the formula
    LARGE function Returns the k-th largest value in a data set
    ROW function Returns the row number of a reference
    LEN function Returns the number of characters in a text string
    INDEX function Uses an index to choose a value from a reference or array
    SMALL function Returns the k-th smallest value in a data set
    IF function Specifies a logical test to perform
    COUNTIF function Counts the number of cells within a range that meet the given criteria
¹ The only function used in this solution that is not compatible with a Excel 2000-2003 worksheet is IFERROR(…) which was introduced with Excel 2007. This can be substituted with an IF(ISERROR(…), "", …) nested formula.
