For a dynamic / roll over formula, try these options:

In case there will always be minimum 12 months data then below will work:

=SUM(OFFSET(B6,,MATCH(TRUE,INDEX(ISBLANK(B6:W6),),0)-2,,-12))

In case data could be less than 12 months, say only 5 months (and other cells are blank), then below formula will account for this and return SUM of only 5 months:

=SUM(OFFSET(B6,,LOOKUP(2,1/(B6:W6<>""),COLUMN(B6:W6)-COLUMN(B6)+1)-1,,-MIN(12,COUNTIF(B6:W6,"<>"))))

ken sheridan say: 1.  Generally a separate copy of the front end would be installed on each user's local machine, or in a 'personal' location on the system to which only the user has full permissions.  Consequently there is no need to name each copy differently.  Doing so would only cause confusion when rolling out updated front ends to the users.

2.  A form or report is an interface object; it stores no data.  By including links to the tables in the common back end in the front end, a form or report based on a linked table, or more usually a query which references one or more of the tables, will show the data from the tables in the common back end, and in the case of forms, provided that the form's RecordSource returns an updatable recordset, allow the user to update the data in the common back end.
