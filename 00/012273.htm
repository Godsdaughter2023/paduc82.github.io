echo s say:

Please check on the data points in the problem charts, too -- there's a bug with displaying pie charts with data points that are zero in PPT 2007, and I wonder if that's the issue here.

There are differences in color schemes in general between PPT 2007 and 2003, but if you are not editing the file, then just opening it and presenting shouldn't cause any color shifts. What you're reporting is very unusual! That's why I'm wondering if that zero is coming into play somehow....

As for the file size, you got it exactly. When you'd copy a chart from Excel and paste into PPT 2003, the whole workbook came along for the ride. That created very large presentations and also opened up people to unwittingly sharing their data because they didn't realize the whole workbook would be included.

In PPT 2007, when you copy a chart from Excel and paste onto a slide, it creates a link to the Excel workbook. So when you send the file to someone else, they only have access to the chart pieces, but not the data itself (unless they have access to your server or drive where the Excel file lives, of course). So the chart can be edited as far as formatting -- colors, fonts, etc., but the data is not accessible or editable by the recipient.

This wouldn't affect the color of the charts, though -- unless, as I mentioned, you actually opened the charts to edit them. Or if you told PPT to convert all charts. (And if you didn't see a message prompting you to convert all or edit existing, then you probably didn't do that.) Or if you appled a new color scheme to your deck. Or if you edited it some other way. But just opening and presenting shouldn't change your colors (sorry to keep repeating myself!) -- I know the PPT developers worked really hard to ensure that fidelity (display) was the same in both 2003 and 2007 when you'd just open and present. So right now I'm leaning toward that 0 data. :-)

Return to same folder automatically - Microsoft Community

waltermurphy say:

Return to same folder automatically

With a file open, to open another file, how can I return to the same folder automatically to find and open another file?O
doug robbins - mvp office apps & services (word) say:

Run a macro containing the following code:

 

Dim strPath As String
Dim fd As FileDialog
strPath = ActiveDocument.Path
Set fd = Application.FileDialog(FileDialogType:=msoFileDialogOpen)
With fd
    .InitialFileName = strPath & "\"
    .Title = "Select the Document that you want to open."
    If .Show = -1 Then
        Documents.Open .SelectedItems(1)
    Else
        MsgBox "You did not select a file."
        Exit Sub
    End If
End With
