graham mayor say:

The macro takes no account of existing files of the same name, or illegal filename characters in the 'field', but does check that the named path is available and that the bookmark is present. There are functions to check for illegal filename characters and to provide unique filenames at http://www.gmayor.com/useful_vba_functions.htm

Option Explicit

Const strPath As String = "S:\Dubbo\New Pricing Model (Quotes)\2015\Quote Requests\"
Dim strName As String
Dim strBM As String
Dim oBm As Bookmark
Dim bBM As Boolean

    If Not FolderExists(strPath) Then
        MsgBox "The folder '" & strPath & "' is not available!" & vbCr & vbCr & "The document has not been saved."
        GoTo lbl_Exit
    End If

    For Each oBm In ActiveDocument.Bookmarks
        If oBm.name = "Org_Name" Then
            strBM = oBm.Range.Text
            bBM = True
            Exit For
        End If
    Next oBm

    If Not bBM Then
        MsgBox "Bookmark 'Org_Name' is missing!" & vbCr & vbCr & "The document has not been saved."
        GoTo lbl_Exit
    End If

    strName = "Quote Request - " & strBM & " - " & Format(Date, "yyyymmdd") & ".docm"

    ActiveDocument.SaveAs2 Filename:=strPath & strName, _
                           FileFormat:=wdFormatXMLDocumentMacroEnabled, _
                           LockComments:=False, _
                           Password:="", _
                           AddToRecentFiles:=True, _
                           WritePassword:="", _
                           ReadOnlyRecommended:=False, _
                           EmbedTrueTypeFonts:=False, _
                           SaveNativePictureFormat:=False, _
                           SaveFormsData:=False, _
                           SaveAsAOCELetter:=False, _
                           CompatibilityMode:=15
lbl_Exit:
    Exit Sub
End Sub

Private Function FolderExists(strFolderName As String) As Boolean
'Graham Mayor
'strFolderName is the name of folder to check
   Dim fso As Object
   Set fso = CreateObject("Scripting.FileSystemObject")
   If (fso.FolderExists(strFolderName)) Then
      FolderExists = True
   Else
      FolderExists = False
   End If
lbl_Exit:
    Exit Function
End Function

alexsun_2012 say:

Hi Joakimc,

Unlike IMAP and Cutover migration, incremental synchronization is not included in a Staged Migration. In the Initial synchronization, messages, contacts, and calendar items are copied to the corresponding Exchange Online mailboxes. After mailbox items are migrated, new emails sent to the on-premises mailbox are forwarded to the corresponding Exchange Online mailbox so you won't lose the emails. The calendar items will not be synchronized to Exchange Online mailboxes.

In this situation, if you haven't converted the on-premises mailboxes to mail-enabled users and they can still access their on-premises mailboxes from the old Outlook profile, they can export the calendar events and then import to their online mailboxes.

If the mailboxes have been disabled, we are unable to get the items back. If you want to restore them from your on-premises Exchange server, we suggest you contact the dedicated Exchange Support Forum to see if they can provide further assistance.

Thanks for your understanding.

ron.5 say:

Use Index Match to retrieve the first larger number greater than a criteria value

I would like to use Index Match to retrieve the first value (Col b) larger than a criteria value (Col C). The order of the "lookup" values is based upon dates in column A which are in descending date order hence the order of the "lookup" column values is random. The criteria of selection is in column c.


Date   (A) Value   (B) Criteria   (C)

08-14-14

 8
08-13-14 3
08-12-14 7
08-11-14 4
08-10-14 10
08-09-14 4
08-08-14 5
08-07-14 3  6

In this example for criteria value of 6 the correct answer is 10.  The value of 10 is the first value greater than criteria 6 which is closest by date to the criteria date. Thus the search direction is from the oldest date up to the most recent date. 

Thanks in advance for your inputs and help.

Ron

ashish mathur say:

Hi,

Try this

=LOOKUP(2,1/($B$2:$B$9>=C9),$B$2:$B$9)

Adjust the row number 9 to the row where you type the criteria.  So if your criteria is typed in cell C7, then the formula will be:

=LOOKUP(2,1/($B$2:$B$7>=C7),$B$2:$B$7)

Hope this helps.

ashish mathur say:

Hi,

You are welcome.

The 1/($B$2:$B$7>=C7) portion generates an array of either 1 or #DIV/0! error.  Since 2 is greater then the maximum number which can appear in this array I.e. 1, the lookup direction reverses i.e. it searches from right to left and stops at the first number.  Accordingly, the number is picked up from column B.
