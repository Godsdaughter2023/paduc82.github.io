It may help to make a few changes to your code:

Add Option Explicit as the first line of each module.

This will force you to declare any variables before using them and is good general practice.

Next, always Dim variables as the specific data type or object they'll hold, otherwise VBA will make assumptions about what you intended; when it's wrong, it can lead to problems.

How do you lookup a certain value in a row and then look up another - Microsoft Community

blackfalcon7842 say:

How do you lookup a certain value in a row and then look up another value in that row and then count the amount of times that second value appears in the row

In Microsoft excel how do you lookup a certain value in a row and then look up another value in that row and then count the amount of times that second value appears in the row is the question.

So in other words, the first value identifies the row the second value identifies what to look for in that row then instead of returning that exact value it returns the amount of times that value shows up in the specific row. 

Example: So lets say you have a workweek sheet and a workweek summary sheet. On the workweek sheet it highlights the workers and the work done at different times during the day. On the summary Sheet it uses that data to tell you who (Value 1)  did what assignment (Value 2) and how many times they did it (Adding it). Of course that person would be doing different things during the day so it needs to be able to tell the difference (case sensitive). It does not need to tell you what time they did it since its based on a week that would be unnecessary data it just needs to tell you how many times they did a assignment during that week. And since persons may change through out the week once you type in a name to search and work assignment it should auto change its total(automated system i am looking for) . 

Example code that didn't work but this is what I came up with: 

=MATCH(Summary!$G$2,INDEX(Tuesday!$B$2:$E$58,MATCH(Summary!A5,Tuesday!$A$2:$A$58),))

Summary!A5 = Would be the person I look for in a row. this is a vertical lookup of names ( summary sheet )

Summary!$G$2 = Would be the assignment to look for on the workweek sheet but this ( text to look at ) is on the ( summary sheet )

Tuesday!$B$2:$E$58 = Would be the assignments to search through ( workweek sheet )

Tuesday!$A$2:$A$58 = Would be the row of names to search through ( workweek sheet )

Its like a two way lookup but on the second look up it looks at the whole row and identifies all the matching values/text in the row and adds the amount of times they show up.

amit tandon say:

Formula in cell D1:

=SUMPRODUCT(--(EXACT(INDEX($B$2:$O$58,MATCH(A1,$A$2:$A$58,0),),B1)))

This identifies the row in column A which matces the cell A1 name. And then from that row ie. B2:O2 it counts the number of times "As3" appears. And this is a case-sensitive match so that the 2 occurrences of "as3" (marked in red font) are not counted.

In case you do not want case-sensitive then use the below formula entered in cell C1:

=SUMPRODUCT(--(INDEX($B$2:$O$58,MATCH(A1,$A$2:$A$58,0),)=B1))
