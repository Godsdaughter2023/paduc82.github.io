pamcaswell say:

Doug's macro are, I'm sure, wonderful, but if you have to do this by hand, know that the SEQ identifier (or name) is the same as the caption label.   So you can fix things in just a few steps.  First, toggle the field code for the first (to be safe) caption you can't see in the cross reference list.  It should look something like this { SEQ silly \* ARABIC }.  The word "silly"  is the identifier.  Now go into insert caption and create the new caption labelsilly .  It must exactly match the identifier.  When you open the cross reference list, all the captions tied to the label silly should be there.   Of course, if you have several identifiers that should be the same (like silly, sillee, silly[space]), you'll need to standardize them before you create the new label. A careful find and replace of the field codes should handle that. 

Added later:  And if you want to change the user-defined label to Figure or another label, like Map or Annex, a search and replace can handle that as well.

 

HTH,

Pam

doug robbins - mvp office apps & services (word) say:

If you can get the originator of the document to run a macro containing the following code

Documents.Add
Dim i As Long
For i = 1 To CaptionLabels.Count
    ActiveDocument.Range.InsertAfter CaptionLabels(i) & vbCr
Next i

it will create a document that contains a list of the caption labels.  If you then get a copy of that document and delete any empty paragraphs from the end of it and then run a macro containing the following code on your machine

Dim i As Long
Dim CapLabel As Range
With ActiveDocument
    For i = 1 To .Paragraphs.Count
        Set CapLabel = .Paragraphs(i).Range
        CapLabel.End = CapLabel.End - 1
        CaptionLabels.Add CapLabel.Text
    Next i
End With

It will create those captions on your machine.
