gary's student say:

Say we want to insert a value in cell A1.  When a value has been inserted, the value in B1 will be incremented by that amount.  So if B1 contains 11 and we enter 2 in A1, B1 will become 13.

 

First insert the following event macro in the worksheet code area:

 

Private Sub Worksheet_Change(ByVal Target As Range)
If Intersect(Range("A1"), Target) Is Nothing Then Exit Sub
Application.EnableEvents = False
    [b1] = [b1] + [a1]
Application.EnableEvents = True
End Sub

 


Because it is worksheet code, it is very easy to install and automatic to use:

   1. right-click the tab name near the bottom of the Excel window
   2. select View Code - this brings up a VBE window
   3. paste the stuff in and close the VBE window

If you have any concerns, first try it on a trial worksheet.

If you save the workbook, the macro will be saved with it.


To remove the macro:

   1. bring up the VBE windows as above
   2. clear the code out
   3. close the VBE window

To learn more about macros in general, see:

http://www.mvps.org/dmcritchie/excel/getstarted.htm

To learn more about Event Macros (worksheet code), see:

http://www.mvps.org/dmcritchie/excel/event.htm

Getting rid of OLE LINKs in Word Bookmark. - Microsoft Community

ttxyz say:

Getting rid of OLE LINKs in Word Bookmark.

How do you remove OLE LINKs in the bookmark without having to delete them one at a time?  
How do you keep them from accumulating in bookmarks?
rich michaels - mvp (greatcirclelearning.com) say:

I'm not exactly sure what you want to do, but if you want to delete all linked OLE objects from bookmarked ranges, then this macro will do that (also assuming they are inline objects).

Sub DeleteOLElinkedObjects()
    Dim doc As Word.Document
    Dim bmk As Word.Bookmark
    Dim sRng As Word.Range
    Dim iShp As Word.InlineShape
   
    Set doc = Word.ActiveDocument
    For Each bmk In doc.Bookmarks
        Set sRng = bmk.Range
        For Each iShp In sRng.InlineShapes
            If iShp.Type  = wdInlineShapeLinkedOLEObject Then
                iShp.Delete
            End If
        Next iShp
    Next bmk
End Sub

graham mayor say:
The processor and memory have little to do with speed of saving a document. It has more to do with the speed of the hard drive and if that hard drive is in externally based resource, the speed of the interface to that resource. 256 mb is a huge file for Word, and while 30 minutes does seem excessive, without more information about where you are saving it it is impossible to say whether that time is reasonable in the circumstances.

graham mayor say:
Hmmmm. It sounds as though there may be some corruption in this document. I would suggest making sure that this was not your only copy of the document. Copy it to external media such as a CD/DVD or flash drive etc for safety.

Next run chkdsk on your hard drive. (From a command prompt window run as administrator enter chkdsk /f at the prompt. You will get the option to run the repair when you next boot Windows. Then restart Windows.

It is not necessary and probably undesirable to defrag regularly. You will not notice much, if any, difference to disc access speed and you run the risk of unwanted disc corruption. If defragging once didn't help with your access problem, running it again will not help either.

Chkdsk will take a while to run - leave it to do so!

If that doesn't help I would start to rebuild the document by copying and pasting it a section at a time into a new document saving after each addition and see if you can spot where the problem lies.

graham mayor say:
There should be no need to buy Word 2010. I doubt that it would make any difference.

Don't work directly from the removable media. This can lead to document corruption. Always work from the hard drive.

My feeling is that either your anti-virus software or third party add-ins are the problem. I  noted that you turned the AV software off, but that may not have disabled document checking if the dll file it uses was still loaded and unless you rebooted it probably would have been. I am not familiar with Avast, but a quick look around the web suggests that it does have some issues with Office. There are lots of complaints of Office hanging when it is installed. I would certainly explore that avenue.

AVG Free works happily with Office. You might consider uninstalling Avast (I spotted a utility to remove all traces of it) and installing AVG.
