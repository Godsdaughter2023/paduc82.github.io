I’d like to make a clarification regarding the Outlook SSO experience.

 

Previously, Outlook client used basic authentication, in which mode Outlook prompted users for their UPN & password on profile creation or password change. It’s not a SSO experience.

 

Now, modern authentication has been released by Microsoft. With modern authentication in a domain jointed computer, there’s no need to input password on profile creation or password change. 

 

However, modern authentication is turned off for Exchange Online by default. To enable it:

1.Connect to Exchange Online as shown here.

2.Run the following command:

Set-OrganizationConfig -OAuth2ClientProfileEnabled:$true

3.Run the following command to verify that change was successful:

Get-OrganizationConfig | ft name, *OAuth*

 

Furthermore, Office 2013 on Windows devices doesn’t support modern authentication either. Please refer to this article to enable it: https://support.office.com/en-us/article/Enable-Modern-Authentication-for-Office-2013-on-Windows-devices-7dc1c01a-090f-4971-9677-f1b192d6c910.

ralphcellis say:

As it turned out it WAS an Outlook problem.

The fact that Outlook supports Rules, Filters AND Junk Mail settings meant that we took quite a while to find the cause of the issue.

A bit of background: our institution was having problems with spam and several users started looking for their own solutions.

This user had managed to get into the Junk E-Mail folder properties and had turned on High junk mail sensitivity, AND selected Permanently Delete for those suspicious emails.  Turning that off caused the emails to appear in the Junk E-Mail folder, and from there we could right-mouse-click on the non-junk emails and mark them as such - which then caused it (and subsequent emails of that type) to appear in Inbox.
