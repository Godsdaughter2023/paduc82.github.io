“net stop wuauserv” and press Enter.

Click start again, type cmd, right click and run as admin, type:
“net start wuauser” and hit enter.

Reboot, check for updates (let them install and finish if they are out there) and then check the store to see if you download MineCraft.

What Steve said, but I'll elaborate on it a bit.


Tools | Accounts | Mail | Properties | Advanced. Uncheck: Leave a copy of messages on the server. Then delete the Pop3uidl.dbx file. You will get the messages one more time, but that should be it.

Tools | Options | Maintenance | Store Folder will reveal the location of your Outlook Express files. Write the location down and navigate to it in Windows Explorer or, copy and paste it into Start | Run.

In WinXP, Win2K & Win2K3, the OE user files (DBX and WAB) are by default marked as hidden. To view these files in Windows Explorer, you must enable Show Hidden Files and Folders under Start | Control Panel | Folder Options Icon | View, or in Windows Explorer | Tools | Folder Options | View.

With OE closed, find the Pop3uidl.dbx file and delete it.  A new one will be created automatically when you open OE.

General precautions for Outlook Express:

Do not archive mail in the Inbox or Sent Items. Create your own user defined folders and move the messages you wish to save to them. Empty Deleted Items folder daily. Although dbx files have a theoretical capacity of 2GB, I recommend about a 300MB max for less chance of corruption.

Information about the maximum file size of the .dbx files that are used by Outlook Express:
http://support.microsoft.com/?kbid=903095

After you are done, follow up by compacting your folders manually while working *offline* and do it often.

Click on Outlook Express at the top of the folder tree so no folders are open. Then: File | Work Offline (or double click Working Online in the Status Bar). File | Folder | Compact all folders. Don't touch anything until the compacting is completed.

Turn off e-mail scanning in your anti-virus program. It is a redundant layer of protection that eats up CPUs and causes a multitude of problems such as time-outs and account setting changes. Your up-to-date A/V program will continue to protect you sufficiently. For more, see:
http://www.oehelp.com/OETips.aspx#3

Surface Pro 4 Wi-Fi adapter cannot switch access points - Microsoft Community

jaygourley say:

Surface Pro 4 Wi-Fi adapter cannot switch access points

This is an old problem.  From reading other threads, I believe it affects a small but significant number of Surface Pro 3 and 4 systems.

Problem description:

  1. On startup, the Surface Pro 4 typically connects to a recognized wi-fi access point as expected.  It maintains that connection. Throughput is good.  No errors or problems.
  2. Using the systray icon, the user asks Windows 10 to switch to a different access point.
  3. Windows 10 shows that the adapter is checking network requirement.
  4. But instead of connecting to the new Wi-Fi station after a second or two, the "checking network requirements" continues for more than a minute or two (until I assume it times out).
  5. Then Windows reports no Wi-Fi stations available.  The Windows Settings Wi-Fi on-off switch is unresponsive. The adapter in Device Manager shows that the adapter is functioning normally.
  6. Wi-Fi remains unavailable.  The problem usually does not self-correct even after several hours.  Until...
  7. If the user disables and re-enables the adapter (or restarts Windows), the adapter returns to life every time and connects to the new access point

    With one exception, this problem occurs in all locations, on all access points, every time in every environments where I normally work.  My usual environment includes old N and new AC access points, both secured and unsecured, on both 2MHz and 5MHz bands, and regular wi-fi stations and those guest access points that require web log-in.  The one exception is at my university, which has a distributed Wi-Fi system, the Surface Pro 4 will sometimes switch wi-fi access points successfully there.

    I believe the problem is at least partly software related and probably not related directly to the Wi-Fi driver software.  Following are my reasons for this belief:

    1. The problem is intermittent.  Sometimes my Surface Pro 4 will go for several weeks without exhibiting this problem.  During these periods, it switches access points normally and quickly.  Then the problem will return and it will fail consistently for several weeks.
    2. One time, I asked the local Microsoft Store to copy my C:\drive image onto a known good Surface Pro 4.  After doing this, the known-good Surface Pro 4 exhibited the same problem described above.  It would not switch access points without resetting the adapter and worked fine otherwise.
    3. The problem sometimes disappears when Windows is reinstalled from an .iso using the option to preserve apps.  I think this is what some Microsoft people call "refreshing" Windows.  Once or twice this did not work the first time but did work when the process was repeated.
    4. From reading support forums, it appears this problem is not unique to me but affects only a small minority of Surface Pro 4 users.

    Troubleshooting: Uninstalling and reinstalling the Marvell wi-fi adapter has no affect on the problem.  System File Check (SFC) indicates no corruption.  Windows Defender shows no malware.  The problem existed in pre-1511, 1511 and 1607 builds.  Even though I believe this probably has something to do with my set of applications and/or Windows settings, I don't do anything exotic with my Surface Pro.  It mainly browses the Internet, runs Office 365 applications, stores data on OneDrive and runs a few common applications like Quicken and Texas Instruments math software.
