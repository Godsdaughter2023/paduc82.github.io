marshall barton say:

That Order By clause is redundant because it's the same as the Group By, which also sorts the records.

As I said before, you really need to use the report's Sorting and Grouping to sort the report.  Even if the report looks like it's sorted correctly without it, some future, seemingly simple change to the report can throw the query's sorting out the window.

You said you added =CDate(monthyear) to the Sorting and Grouping list, but did you put it at the top of the list?  (As far as I can see, that should be the only entry in the list.)

Assuming you get it sorted properly, the only problem is to display the date the way you want.  Since the desired date formating is not in the query, you can do it in the report's date text box.  Set the text box's expression to  =CDate(monthyear) and set its Format property to  mmm-yyyy

 

Side note:  the use of the format function in the query is just one of several ways to ignore the day of the month part of the date field.  There is no particular significance to using Format instead of any other calculation that generates an equivalent result.

If it would help your understanding of what's going on and maybe make it simpler in the report, you can get the query to use a Date/Time value instead of a formatted text value with any of these:

CDate(Format([DaysDate],"yyyy/mm")) AS MonthYear

DateValue(Format([DaysDate],"yyyy/mm")) AS MonthYear

DateSerial(Year(DaysDate), Month(DaysDate), 1) AS MonthYear

Then the MonthYear field in the report would be a date value and you can format the text box and do Sorting and Grouping on the field without using an expression.

novamuros_msft say:

Hello,

 

Thank you for your response!

Just to clarify, we have 2 procedures in joining a domain in Yammer. First is called domain add. This process performed to add a domain that has never been used in Yammer. Second is network merge, this is performed to add a domain in Yammer that has already a Yammer network. For more information about network merge, see https://support.office.com/en-us/article/Network-migration-Consolidate-multiple-Yammer-networks-a22c1b20-9231-4ce2-a916-392b1056d002?ui=en-US&rs=en-US&ad=US.

 

If you will perform a network merge in the future, we can only retain a single network, that is the primary network. That means that the network contents and groups that can be retained in Yammer are from the primary network only. The contents of the other networks that you will merge to the primary network will be lost (This includes the posts and groups). You need to manually update the users in the network via bulk update function in Yammer Admin portal.

 

After the merge, the only groups that will be retained will be from the primary network only; in your example this is contosoeurope.com. Yes, this network should contain users that has an email domain *** Email address is removed for privacy ***. This is the standard setup. If you perform network merge to add other domains in the network then you can also add users from these added domains. Just make sure that all of the domains that you will add in Yammer are also added and verified in your Office 365 tenant.

 

You are right, Yammer will match the primary SMTP of the user from Office 365 to his email address in Yammer.

 

I hope this helped you. Let me know should you have any questions.
