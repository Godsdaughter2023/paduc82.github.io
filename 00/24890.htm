rob koch say:

The Thunderbird email client has a sketchy history of known problems with many AV products going back years.  This is because like many early email clients such as Microsoft's Outlook Express, it uses a local file storage format that can sometimes be misidentified or more accurately validly identified as malware and removed/damaged.

For this reason, many AV products have chosen to purposefully exclude these Thunderbird files from being scanned.

Since Thunderbird doesn't contain any of the more recent technologies supported by many modern browsers which pass the attachments to the Microsoft Antivirus scanning API before opening, the advanced Real-time scanning protection the Le Boule mentioned isn't available.  Instead, the malware will only be protected once the file attachment is written to disk.

Since support for the Microsoft Antivirus API has existed since the early 2000's and most web browsers have supported it for at least several years, this is a side effect of a rarely used and not well supported product.  Since most people today use either a browser based email client or the built-in Windows email clients, which have supported the Antivirus API since at least Outlook Express 6, this is a non-issue for most other users.

This is a well known set of problems for those using 3rd-party browser and email products, since their support and integration in Windows is often limited and so thy require their own protection schemes for things like Phishing, Social-engineering and other attack vectors.  When using 3rd-party products to browse or access other Internet sourced information such as email, the expertise required to protect fully requires the PC user to become an expert with that 3rd-party providers product security as well.

Microsoft tries to provide standardized interfaces for protection, but unless the 3rd-party developers actually use them there's nothing more they can do than wait until the malware attempts to either write to disk or perform some other recognizable malicious action in order to detect and block it.

Rob

You are definitely seeing a typical phishing attack via a macro (disguised as a DOC - that can be a PDF as well) downloader to serve as the gateway for other nasty malware to get in and deliver its payload. It can be whatever. Probably one of the most prevalent crypto-ransomware families around - most likely CryptoWall (AKA Crowti - Microsoft detection).

A macro is a piece of Visual Basic (VB) programming code that is embedded in the file. This programming code runs a script… which also means that if it contains a virus, you allow the virus program to run on your computer. The macro code is obfuscated to bypass traditional antiviruses.
