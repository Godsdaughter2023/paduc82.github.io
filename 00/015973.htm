jeeped say:
The simplest way to disregard projects with a closing date prior to the first billing date in the worksheet would be a simple IF() wrapper with relative and absolute referencing. The formula in E2 would be,

=IF($C2<$E$1,0,<previous formula>)

... or,

=IF($C2<$E$1,0,IF($C2<DATE(YEAR(E$1),MONTH(E$1)+1,1),$D2*IFERROR(INDEX($C$10:$F$12,SUMPRODUCT(($A$10:$A$12=$A2)*($B$10:$B$12=$B2)*ROW($1:$3)),(YEAR(E$1)*12+MONTH(E$1))-(YEAR($C2)*12+MONTH($C2))+1),0),0))

The row in $C2 will progress but the $E$1 reference will remain the same no matter where it is copied for filled to. Again, that formula presupposes that E1 contains an actual date (e.g. 7/1/2011).

doug robbins - mvp office apps & services (word) say:

For the error message "Word could not create the work file. Check the temp environment variable". See:

https://support.microsoft.com/en-us/kb/2285187

It can be caused by incorrect registry values in the key:

HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders.

Click Start and in the start search field, type

Regedit

and press enter.

In the registry editor, browse to:

HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders

On the right hand pane, check the value data for Cache.

It has to be %USERPROFILE%\AppData\Local\Microsoft\Windows\Temporary Internet Files.

If it is different, double click on Cache and type:

%USERPROFILE%\AppData\Local\Microsoft\Windows\Temporary Internet Files

under value data and click ok. Close the registry and  restart the computer and check if it works.

  • click on Uninstall
  • Messages will look like it is “installing” but in fact it is not - - it does the uninstall.
