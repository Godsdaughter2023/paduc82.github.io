pucknstella say:

Defender Controlled Folder Access: how to allow desktop application w/ multiple exe files access a protected folder?

After enabling Controlled folder access in Windows Defender, one of my desktop applications that uses a cloud location for file storage was blocked. That's fine, and that's what Defender is supposed to do.

However, this program has MANY MANY executable files and functions, so it's enormously time consuming to add them all. So, my question is:

is there any way to allow that program and all of its functions access or must I go through every single exe file and allow it access individually? (That means not only finding the exe files in the Programs Files (x86) directory for that program but also sorting through all the myriad executables it has in the /Common Files directory as well.

greginmich say:

You’ll only need to add the specific files that are blocked by Controlled Folder Access – and that are identified in the Unauthorized Changes Blocked notification. That will normally be just the main executable for the app that you need to allow:

 

The general steps for allowing an app through Controlled Folder Access:

 

1. Click on the notifications icon at the far right in the taskbar.

2. Write down the file path provided in the notification.

3. Click on the notification, and then on the Add button (+).

4. Navigate to the file identified in the notification, then select and “open” it.

 

The file path in the Unauthorized Changes Blocked notification does tend to be truncated – although there’s usually enough of it there to locate the app without any trouble if you just assume that it’s in the Program Files or Program Files (x86) directory. If the file path is truncated to the point where you can’t locate the blocked app in the file-picker dialog; then open Event Viewer; navigate to the Windows Defender Operational log; and locate the blocking event (Event ID 1123):

 

1. Right-click on the Start button and select Event Viewer.

2. Navigate to Applications and Services > Microsoft > Windows > Windows Defender > Operational

akash_nath say:

Its quite possible.

1. You may link your files and it would refresh whenever you open it (based on your excel settings)

2. You may write the below code in a Module and assign it to a button/shape in the workbook and copy your data when clicked:

Sub CopyPasteData()

Dim sWB As Workbook, dWB As Workbook

'Source Workbook
Set sWB = ThisWorkbook
'Destination Workbook, replace the file path/name with yours
Set dWB = Workbooks.Open("C:\Users\xxxxxx\Desktop\sample_file.xlsm")

'I am copying A1 to D500 from sheet1, you may replace it appropriately
sWB.Worksheets("Sheet1").Range("A1:D500").Copy
dWB.Worksheets(1).Range("A1").PasteSpecial xlPasteAll

'Saving and closing your destination workbook
dWB.Close SaveChanges:=True
Application.CutCopyMode = False
Set dWB = Nothing
Set sWB = Nothing

End Sub


3. Event based execution (file open) of the same macro:

Private Sub Workbook_Open()
Dim sWB As Workbook, dWB As Workbook

'Source Workbook

dWB.Worksheets(1).Range("A1").PasteSpecial xlPasteAll
