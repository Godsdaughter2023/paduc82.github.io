There is nothing wrong with your VBA code. It's the way Excel sorts data: text is "higher" than numbers, so if you sort in descending order, the text comes before the numbers.

Hi Joseph,

Actually, what you are experiencing is an expected behavior. This is because GetUserCollectionFromRole call isn't supported in SharePoint Online.

This issue occurs because loopback protection is enabled in the SharePoint Online environment. Loopback protection must be disabled for InfoPath forms to be able to connect to some SharePoint Online web services.

For security reasons, loopback protection is always enabled in SharePoint Online. This is a known limitation of InfoPath forms in the Office 365 SharePoint Online environment.

Note Some requests are supported and functionality exists to convert these requests to object model calls which does not cause a loopback. The following calls are supported in this scenario:

  • lists.asmx
    • CheckOutFile
    • CheckInFile
  • usergroup.asmx
    • GetUserCollectionFromGroup
    • GetUserCollectionFromSite
    • GetGroupCollectionFromWeb
  • UserProfileService.asmx
    • GetUserProfileByName
    • GetUserPropertyByAccountName
    • GetCommonManager
    • GetUserMemberships
    • GetCommonMemberships

This is mentioned here: https://support.microsoft.com/en-us/kb/2674193

Below are general troubleshooting steps for Outlook. Try one by one

1. Create new Outlook profile to rule out corruption in Outlook profile. - http://office.microsoft.com/en-us/outlook-help/...

2. Type command in your Windows Search Box > Right click on Command Prompt > Run As Administrator > Type """"sfc /scannow"""" without quotes > Run it and when it completes, configure Outlook again.

3. Please check if you have Lync or Skype installed on this computer which starts when the computer starts. If yes, then get out of Skype and start Outlook. Basically, Outlook need to start first than Skype.

4. Repair your Outlook data files - https://support.office.com/en-us/article/Outloo...

5. Disable Anti-virus temporarily.

6. There is a possibility that some Add-in is causing the problem. To rule this out, start Outlook in safe mode. Hold CTRL key and click on Outlook icon. Don't leave CTRL key, till it asks for the confirmation about safe mode.

7. Login to Windows with a new profile to rule out problem in Windows profile.

8. Right click on Outlook icon and run as Administrator.

jim gordon mvp say:

Hi Bob,

The behavior you described will happen if there is more than one copy of an Office application on your drive.

AutoUpdate will update one copy, notice another copy and will try to update it, but it will fail, as that is not something it expects to encounter.

Use Spotlight to locate extra copies of Word, Excel, PowerPoint and/or Outlook and delete the extra copies. Then AutoUpdate won't keep pestering you.
