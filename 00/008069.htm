ossiemac say:

I have assumed that the colors can be repeated if the numeric do not all match. Hope that is what you wanted.

Test the code below and see if it does what you want. If the data already exists then I have coded for the existing data to be selected. If you don't want this then comment out the line of code which I have bolded.

BTW I assume that Lentgh is a typo and should be Length.

Sub AddData()
    Dim wsData As Worksheet
    Dim rngColors As Range
    Dim rngToFind As Range
    Dim arrNew As Variant       'For array of new entries
    Dim arrExist As Variant     'For array of Excisting data with matching color
    Dim strFirstAddr As String
    Dim bolMatch As Boolean     'Set to True for match, False for no match
    Dim r As Long               'Each element of arrays
   
    Set wsData = Worksheets("Sheet1")
   
    With wsData
        Set rngColors = .Range(.Cells(2, "D"), .Cells(.Rows.Count, "D").End(xlUp))
        'Assing the new entries to an array
        arrNew = .Range(.Cells(1, "B"), .Cells(5, "B")).Value
    End With
   
   
    With rngColors
        Set rngToFind = .Find(What:=arrNew(1, 1), _
                        LookIn:=xlFormulas, _
                        LookAt:=xlWhole, _
                        SearchOrder:=xlByRows, _
                        SearchDirection:=xlNext, _
                        MatchCase:=False)
                       
        bolMatch = False    'Initialize to match not found
        If Not rngToFind Is Nothing Then
            strFirstAddr = rngToFind.Address
            Do
                arrExist = WorksheetFunction.Transpose(Range(rngToFind, rngToFind.Offset(0, 4)).Value)
                For r = LBound(arrNew, 1) To UBound(arrNew, 1)
                    If arrNew(r, 1) = arrExist(r, 1) Then
                        bolMatch = True
                    Else
                        bolMatch = False
                        Exit For
                    End If
                Next r
                If bolMatch = True Then Exit Do
                Set rngToFind = .FindNext(rngToFind)
                If rngToFind Is Nothing Then Exit Do
            Loop While rngToFind.Address <> strFirstAddr
        End If
    End With
       
    If bolMatch = True Then
        Application.Goto Range(rngToFind, rngToFind.Offset(0, 4))      'Optional to select existing data
        Exit Sub    'Record data all matched
    Else
        With wsData
            .Cells(.Rows.Count, "D").End(xlUp).Offset(1, 0).Resize(1, UBound(arrNew, 1)) _
                = WorksheetFunction.Transpose(arrNew)
        End With
    End If
   
End Sub

yang zheng msft support say: Hi Tek-nerd,

Here are the answers to your questions:
1. As we are not migrating all users, is there anyway migrated users incoming mails can be forwarded from 1and1 to Office 365 till we change MX record for mail flow?
The article you found is the only way to achieve your requirement. This is because you will not migrate all the users at the same to Office 365 and keep the MX record pointed to 1&1 instead of Office 365 until all the users are fully migrated.
As you said it didn’t work, could you let us know which issue you have encountered? Have you verified the custom domain into Office 365 and change it to ‘Internal Relay’ type?

2. Can we migrate users in batches and schedule continuous batch sync in office 365, so that users no need to login to 1and1 and start using office 365? I don't find any source to run migration batches continuously or scheduled intervals like every 10 minutes.
The answer is yes. As 1&1 is a third party email service, you could use IMAP migration as said in the article you found to migrate the email data from 1&1 to Office 365. And IMAP migration is a continuous migration until you disconnect the migration and remove the migration batch. IMAP migration has two stages to sync the emails to Office 365. In initial sync, all the current emails will be migrated to Office 365. And then after initial sync completes, the incremental sync will keep sync new incoming emails into Office 365 every 24 hours. However, the every 24 hours incremental sync can’t be changed. You can refer to the article below for more details.
https://technet.microsoft.com/en-us/library/jj200751%28v=exchg.150%29.aspx?f=255&MSPPError=-2147217396

3. Is there any way we can use 1and1 admin account credentials and migrate user mailboxes without using user password in 1and1?
No, as the IMAP migration needs to access the emails in each user’s mailbox, each user’s password is required.

Yang
