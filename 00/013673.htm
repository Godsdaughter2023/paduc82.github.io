graham mayor say:
Every character in a document can have a different proofing language applied. If the text is formatted as UK English (CTRL+A then Review > Language > Set Language > UK English) then 'color' and 'favor' etc are marked as incorrectly spelled in Word 2013 (and earlier versions). if they are not for you then you have the wrong proofing language applied to the text or have the no proofing option set, or there is an error condition. The basic functionality of the dictionary has not changed.

Words ending in ize or ise where both are acceptable in English UK are not flagged. Because both are valid spellings, the only way to cause Word to flag one or other as incorrect is to use an exclude dictionary. chad a gross say:

OK - I'm happy to report that I was able to get this to work exactly how I wanted it to behave.  In my navigation form's On Open event I now have:

 

If IsCurrentWebUserInGroup("CaseMgr Billing Users") Then
    'User gets to see Billing tab
    Me.navButtonBilling.Width = 1455
    Me.navButtonBilling.Enabled = True
    Me.navButtonBilling.LeftPadding = 0
    Me.navButtonBilling.RightPadding = 30
Else
    'User doesn't get to see Billing tab
    Me.navButtonBilling.Width = 0
    Me.navButtonBilling.Enabled = False
    Me.navButtonBilling.LeftPadding = 0
    Me.navButtonBilling.RightPadding = 0
End If

When I realized I could set the width of the navigation control (in twips), I used  msgbox(Me.navButtonBilling.Width) to get the current value, then added the code to toggle the control width between 0 and the original (1455).  Once the width was set to zero - I noticed that the gap between the navigation controls was slightly (albeit noticiabely) larger where the Billing control would have been.  That ultimately led me to find the Left & Right Padding properties - which control the gap.  MsgBox commands told me the default values of each (0 Left; 30 Right).  Setting the Right Padding to 0 in addition to setting the Width to 0 effectively hides the control with consistent spacing between the visible controls. 

 

Thanks!

 

Chad
