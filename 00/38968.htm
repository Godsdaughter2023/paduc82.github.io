gambush say:

My Experience With Windows 10

I became familiar with Windows 10 by first installing the pre-release OS onto a "Boot Camp" partition on my Mac Mini (late 2014) running OS X Yosemite (10.10.4). I used a boot USB key containing Windows 10 to install developer release 9926 onto a boot camp partition and I later installed every update through 10240. Windows 10 ran quite nicely on the Mac Mini although many features were not finalized. This experience convinced me that it was time to upgrade an older Intel Series 5 Windows 7 machine built from an ASUS motherboard and Intel i7-875K processor as well as a Gigabyte Series 8, i5 desktop system used by my wife.

I started with the older ASUS PC that had the latest BIOS version installed and all the chipset drivers were up-to-date as well. I made sure that all the latest Windows 7 updates were installed for the existing hardware and software. I downloaded the Windows 10 installer and created a bootable USB stick. The USB device booted just fine, but I did not want to do a fresh Windows install on this machine so I booted into Windows 7 and then started up the x64 setup file on the USB device. On the first try Windows 10 installed successfully on this PC.

The only real issue I had with the ASUS machine was an outdated "Intel Rapid Storage Technology" driver that was easily corrected by downloading and installing the latest release from Intel for Windows 10. I might mention that this computer was and is booting from two SSD drives in a RAID 0 configuration.

The home built Gigabyte machine was next. Surprisingly, I ran into a brick wall with this box, The install failed on 4 attempts with automatic roll-back to Windows 8.1. It was clearly time to regroup. I started by taking a hard look at the BIOS on the Gigabyte machine. To my surprise the BIOS version for the Gigabyte Z87M-D3H was listed as F3. The Gigabyte website did not even list F3 as a viable bios version and the latest bios version available was F7.

The Gigabyte support site has available a utility called "App Center 2" which includes a module that analyzes the machine's BIOS and chipset driver versions. The utility detected the out-of-date bios as well as six outdated chipset drivers. After updating the bios and drivers, Windows 10 installed as an upgrade to the existing Windows 8.1 with no problem.

The only other issues or problems I've had so far:

1. The Gigabyte PC required powering on twice before booting. I went into power settings and disabled the fast startup option. This appears to be a known bug. I did not have this power problem on the ASUS box, but disabled it on that PC anyway.

2. A consistently irritating "feature" of Windows is the requirement for a default user login on a home PC. I'm the only user on my PC. I do not want to have to enter a password every time I start my machine. The fact is this is a two part issue. MS would like you to login with a Microsoft Account for whatever their business reasons are. On the other hand you must at least login as a user. If you are the only "User" and don't want to enter a password to start a session, go to your account settings and select sign in with your PC account. When prompted click in but leave blank the password fields.

I think the major lesson learned here is if you want to upgrade your PC; whether home built or not, you should update your bios and chipset drivers first.

So far I like what I see with Windows 10. A vast improvement over Windows 8 and probably what Windows 7 should have been from the start.
