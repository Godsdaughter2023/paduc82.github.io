john w. vinson say:

Use the tools that Access provides.

 

Tables are intended for data storage. Microsoft has (unwisely, in my opinion) added features such as Subdatasheets and Lookup Fields that encourage new users to work directly in Table datasheets. Table datasheets are VERY LIMITING and can cause all sorts of problems!

 

What you can, and should, do instead is reserve your Tables for data storage; you would only look at a Table directly if you were debugging or designing. Think of the table as the engine of your database, like the engine of your car - you would only open the hood and look at the running engine if there were a problem you were trying to fix!

 

Instead, use a Form based on the Patients table, with two Subforms - one based on the patient history table, the other on the visits table. These Subforms would use the PatientID primary key of the patients table as the Master Link Field, and the PatientID long-integer foreiign key field as the Child Link Field. The Subform feature will ensure that you see only the history or visits of the currently selected patient in the subform, and that new history or visit data added to the subform will inherit just that patient's ID.

 

 

Note also that electronic data systems containing information that would allow a patient to be identified, or containing medical information about that patient, are covered under the very stringent HIPAA patient privacy regulations. Unless you have a few hundred thousand bucks handy to cover your fines, you need to review these regulations and be sure that your database is sufficiently secure, or is stored on a suitably secured system. Some would argue that Access security is simply not strong enough to qualify.

hansv mvp say:

Create the following function in a module in the Visual Basic Editor:

Function ExtractNumber(s)
    Dim p As Long
    ' Default return value
    ExtractNumber = Null
    If Not IsNull(s) Then
        ' Loop through the characters
        For p = 1 To Len(s)
            ' Stop when we find a digit
            If IsNumeric(Mid(s, p, 1)) Then
                ExtractNumber = Val(Mid(s, p))
                Exit Function
            End If
        Next p
    End If
End Function

In the query, create a calculated column

Num: ExtractNumber([TextField])

where TextField is the name of your text field.
