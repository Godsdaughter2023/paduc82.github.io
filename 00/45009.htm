Mobile provider flat rate of 5 cents a minute (inbound or outbound)

jrhoburton say:

I did some more searches in the meantime and discovered that this is normal. Windows does not actually run a program via scheduled tasks, if it is set to run whether the user is logged on or not, even though admin user credentials are supplied. What happens is that the program is run in some limited state where it shows up in the process list, but is unusable. (I call this the "coma" state.)

The workaround is to set up your PC for automatic log in, then use Scheduled Tasks to start the program when someone is logged in. I set a short screen timeout to reduce security risks, but this is still riskier than the preferred method.

It's possible that all the people who posted about this are wrong, and you can actually start a program in the sked without logging in, but until someone succeeds, I must conclude that this does not work.
