suzanne s. barnhill say:

What you are describing is a "mail merge." As Hans suggests, the first prerequisite is an address list in a form Word can use. If you start a mail merge from scratch, using Mailings | Start Mail Merge | Step-by-Step Mail Merge Wizard, your first step will be to create such a database, for which Word uses an Access .mdb format. But if you already have the addresses in Excel or Access, you can use either as a data source. You can also use a Word table. The chief requirement is that you have the data in tabular form, with each element in a single "field" (usually a table or spreadsheet cell).

You can find general information about mail merges and mail merge data sources in these articles:

How to create a Mail merge
http://word.mvps.org/FAQs/MailMerge/CreateAMailMerge.htm

Creating a mail merge Data Source
http://word.mvps.org/FAQs/MailMerge/CreateADataSource.htm

These articles were written for previous versions of Word, but the same principles apply in Word 2007. You can also apply the principles in http://WordFAQs.mvps.org/ExpressMailMerges.htm; the Mailings tab in Word 2007 has pretty much the same buttons as the Mail Merge Toolbar in previous versions.

If you don't want to use the Mail Merge Wizard (which can be confusing, not to mention very inefficient if you already have a data source and know what you're doing), you can choose Envelopes under Start Mail Merge. Select your desired envelope size, then click Select Recipients, which gives you a choice of using an existing list, creating a new one, or using Outlook Contacts.
