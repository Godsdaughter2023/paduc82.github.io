roelpaulo say:

I know this is old, but for someone who's still stuck, let me share what I did to fix it.

EDIT: I can't turn the codes into "codes" so I only highlight it in bold. :D

run powershell as admin,

run this commands in powershell:

* view all packages:

Get-AppxPackage -AllUsers | Select Name, PackageFullName

first, we have to remove all the metro apps that can be removed (including windows store).

* remove all packages:

Get-AppxPackage -AllUsers | Remove-AppxPackage

you will get some errors that is colored in red, just ignore that.

now we will put them all back again (including the ones that you uninstalled which is pre-installed before).

* install all packages:

Get-AppxPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}

That is what I did and now I can click or search in the store without crashing and it's working like a charm now :).

some other useful codes:

* remove a certain package:

Get-AppxPackage <PackageFullName> | Remove-AppxPackage

* install a certain package:

Add-AppxPackage -register “C:\Program Files\WindowsApps\<PackageFullName>\AppxManifest.xml” –DisableDevelopmentMode

NOTE: Just don't forget to replace "<PackageFullName>" with it's full name :).
EXAMPLE:

Get-AppxPackage Microsoft.WindowsStore_11610.1001.10.0_x64__8wekyb3d8bbwe | Remove-AppxPackage


Add-AppxPackage -register “C:\Program Files\WindowsApps\Microsoft.WindowsStore_11610.1001.10.0_x64__8wekyb3d8bbwe\AppxManifest.xml” –DisableDevelopmentMode

hope this helps :)

    Solved ‘This file does not have a program associated with - Microsoft Community

    johnny s. say:

    Solved ‘This file does not have a program associated with it’.

    Finally, I found the solution to this particular problem that was happening to me after I successfully installed the normal monthly Windows updates. When I double clicked on any folder I got that long message: ‘This file does not have a program associated with it for performing this action. Please install a program or, if one is already installed, create an association in the Default Programs control panel’.

    I posted my question late in 2018 and I was given all sort of suggestions and none of them worked so I gave up and stopped installing updates, which was a very bad idea! I just restored my computer to a point before I installed my last update. However, after learning that the May 2019 updates included software patches to prevent ransomware being installed on the desktop, I kept surfing and trying new solutions, here is a solution that finally worked for me: https://www.youtube.com/watch?v=psn9IxRvUow

    Please beware that there are several other solutions (some are just text instructions and some solutions are via YouTube) that are being offered on the Internet that are more complex and as such, put your registry at risk for further problems down the road! As well, some of these suggested solutions are useless.

    I found that the solution in the above link worked for me. But I made a restore point first before I downloaded and installed EACH SET OF MONTHLY UPDATES. Because I had not updated my computer at all in 2019, I only downloaded and installed the updates one month at a time. This way, if I had a problem, I would restore my computer before the monthly updates and then I would install them, just 1 update at a time. If I download and install many monthly updates all at once and then have a problem, it would be much harder to narrow down which specific update(s) was the culprit.     lemp say:

    Although you are to be congratulated on finding a solution that worked for you, the solution presented in that YouTube video will NOT fix every occurrence of the "This file does not have a program associated with it" error.

    In general, Windows determines how it should handle a file by looking at the file's "extension," which is the 3 or 4 character (sometimes more than 4) portion of the file name following the last dot in the file name.  For example, the file myletter.doc has the "doc" file extension which in general means that it is a file that was created by a version of Microsoft Word from Word 97 through Word 2003.

    The "association" of a file extension is handled with the Windows Registry.  The wrong change to the Registry can damage your installation of Windows to the point of inoperability, and changes made to the Registry take effect immediately and there is no "undo" function.  For this reason, you should not make changes to the Registry unless you're confident you know what the change will do, and even then, always make a Registry backup or a Restore Point before changing the Registry -- just in case. 

    Opening folders (directories) is a special case of the "file association" problem because there is no file extension involved with a folder.  Nevertheless, the way that Windows "knows" how to open folders is handled in the same part of the Registry as file associations for data files.

    There are two easy ways to deal with the "This file does not have a program associated with it" error.

    For most files, you can right-click on the file and select "Open with > Choose default program".  This will open a dialog box that will present a number of "Recommended Programs," another collection of "Other Programs" (after you click the down arrow at the right), and a "Browse" button.  Select the program that you want to use to open the file, check the box to "Always use the selected program to open this kind of file" and click OK.

    The "Open with" procedure won't work in every case, and in particular, it won't work if the association with directories (folders) is corrupted, as was the case with your system.  In this situation, go to this website:  https://www.sevenforums.com/tutorials/19449-default-file-type-associations-restore.html

    Clicking on any of the blue "file extensions" on that page will download a *.reg file that when "Merged" (see the directions at the top of the web page) with your Registry will restore the entry to its default state. 

    If the problem is that attempting to open directories results in the error, download the "Directory" and "Folder" files (most of the time, "directory" and "folder" are used interchangeably, but Windows considers them to be different things -- https://devblogs.microsoft.com/oldnewthing/?p=11473).  Thus, just to be sure, use both files.

    In your case, and in the case shown in the YouTube video, the problem was in the Registry section for "Directory."  And sure enough, part of the Default_Directory.reg file will change the data in the Registry entry for HKEY_CLASSES_ROOT\Directory\Shell to "none" just as you described.
