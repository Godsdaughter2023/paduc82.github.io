Is locating a footnote "below text" before a continuous - Microsoft Community

galahad72 say:

Is locating a footnote "below text" before a continuous section break no longer a feature?

Sometimes you want to run sections together by placing a "section break (continuous)" before the new section.  Sometimes the last page of the ending section will have a footnote. In Word 2010 it was easy in the Footnote dialogue to relocate a footnote belonging to the ending section below that text, where it belongs. Then the new section would begin. If the new section had a footnote on the same page, it would start its own numbering, and the footnote would be at the page bottom, well separated from the note belonging to the ending section. This was an excellent feature.

 

In Word 2013 the note belonging to the ending section won't relocate to below the text of that section, although it obeys the section break and starts its own numbering. Is this a known departure from Word 2010? Can it be overcome?

 

One workaround is to replace any footnotes in the ending section with endnotes located at the end of this section, and renumber and reformat them. Unfortunately, if you've already relocated the ending footnotes below text in Word 2010 and you now save the file in the Word 2013 format, they will be re-relocated back to the page bottom. Imitating the original feature this way is time-consuming.

 

There is another workaround. That is to open the file in Word 2010, do your relocating, and save it there. Then, back in 2013, reopen the file and keep it in Compatibility mode. (There doesn't seem to be any way to return a 2013 .docx file to Compatibility mode from within 2013.) It seems odd to employ Word 2013 to work in compatibility mode for 2010, and goodness knows what features unique to 2013 would be lost in a backwards conversion.

suzanne s. barnhill say:

Microsoft has confirmed that this is a design change in Word 2013, as yet undocumented. Some trade-offs were necessary in order to implement the new layout engine, and this one provided the ability to match the number of columns in a given section with the number of columns in the footnotes, with proper column balancing. Another change is preventing figures in the document from overlapping the footnote region when pushed down by edits.

 

According to a Microsoft source, the page is now essentially split into two regions--the main body of the document and the footnote area. Once a footnote appears on the page (as when placed "Below text"), additional document text cannot appear until the next page. The suggested workaround is to use "End of section" endnotes instead of footnotes.

hansv mvp say:

I guess 12.10 should be in the 12-15 range.

 

Create a small table:

 

0     0-3
3     3-6
6     6-9
9     9-12
12   12-15
15   15-18
18   >18

 

Let's say this is A1:B7 on Sheet2.

 

With a customer volume in D2 on the active sheet, enter the following formula in the cell next to it (E2):

 

=VLOOKUP(E2,Sheet2!$A$1:$B$7,2)

 

and fill down as far as needed.

john korchok say:

Usually, the major cause of this is that you've used a font that is not installed on the receiving computer. Then Word uses a different font and the formatting changes.

Recently, however, there have been recent reports of of changes in formatting when using Times New Roman in Word. I haven't been able to reproduce this yet to find out whether it's a document or font issue, but the cure seems to be to use Calibri or Arial instead. This would also solve the issue in paragraph 1.

charleskenyon (ia) say:

Here are a couple of links:

  • Why has my page layout changed when I open my document on a different machine? 
  • What happens when I send my document to someone else? Will Word mess up my formatting? 

There are a number of things that you can do to help with this. 

  1. Use common fonts that are available on both computers.
  2. Use wide margins.
  3. Make sure that styles are not set to update frcom the template.
  4. If possible have both computers use the same default printer driver software. This is not possible if they use different operating systems.

These will mitigate the problem. As I said, it is inherent in how Word works, though. There is much more information and ideas in the linked articles.

Consider sending both the pdf and the Word document so that the recipient can see what you are seeing.
