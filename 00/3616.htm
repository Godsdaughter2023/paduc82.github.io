john korchok say:

The visual representation of the Excel object in PowerPoint is not the object itself. It's a separate graphic file, like a thumbnail. So when you adjust the size in PowerPoint, you're changing only the thumbnail, not the original object file.

The sizes of inserted objects are determined automatically by the host program. The automatic choice may be incorrect, as you've discovered. You'll get more control by using Home>Paste>Paste Special>Paste link>As Microsoft Excel Worksheet Object, because you can define the source area that you are copying. PowerPoint automatically resizes the pasted link to fit the slide if the height is larger than 80% of the slide height, so you shouldn't have to make any resizing adjustments afterward. stefan blom say:

You can specify the picture location relative to the page edges, which means that it won't move around on the page.

However, all "floating" objects in Word are anchored to a paragraph, which means that if the anchor paragraph moves to the following page, the object will move to that page too.

doug robbins - mvp office apps & services (word) say:

If you can live with the picture being located in the Header of Footer of page 3, you can use the following field construction

{ IF { PAGE } = 3 { INCLUDEPICTURE "C:/PATH/FILENAME" } }

You must use CTRL+F9 to insert each pair of field delimiters { } and you use ALT+F9 to toggle off their display.
