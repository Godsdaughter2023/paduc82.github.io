To do this you should start a command prompt as administrator and start cleanmgr /sageset:nn

where nn is a arbitrary number.

fi. cleanmgr /sageset:01

In the settings dialog you should select:

  Temporary Setup Files, Setup Log Files, Windows Update Cleanup, Windows Upgrade log files, Windows ESD installation files, Previous Windows Installation(s), Update Package Backup Files, Temporary Windows installation files

   cleanmgr /sagerun:01

This command will run disk cleanup using the sets defined in set 01.

You can define different sets with different selections and refer to them with nn.

You seem to have the free OneNote 2016 which is unwilling to open a .onepkg file. This is a builtin restriction. To copy the .onepkg file to OneDrive is useless.

But you can outwit OneNote.

At first you should know that a .onepkg file is a ZIP file in disguise.

In a first step we have to open it. To change the file type from .onepkg to .zip is neither necessary nor sufficient, because the Windows builtin ZIP software refuses to open the file.

  2. Extract the notebook (folder with notebook's name = notebook)

  4. Start OneNote 2016
  5. Create a new empty notebook on OneDrive
  6. Double click the first .one file of your extracted notebook
  7. It should open in a special area in the left tab of OneNote
  8. Drag&Drop it into the new notebook on OneDrive
  9. Proceed with the other .one files

Song Artist vs. Album Artist? - Microsoft Community

webstuff say:

Song Artist vs. Album Artist?

So what exactly does MS intend with this distinction? Off hand I would have thought that Album Artist would be the primary artist for the song (the name of the artist the album was released under), and that Song Artist might only be for songs where another artist was featured, a duet artist, etc.

However, by default the Zune player seems to display Song Artist as the primary artist field, so I'm confused as to what the actual purpose of the two fields was meant to be. Any insight? Thanks!

nsilmike say:

It's not very well thought out, IMHO. Let's take an example: Shiny Happy People by R.E.M. featuring Kate Pierson.

You have to enter R.E.M. as the Album Artist, because otherwise the song won't even show up when you browse your artists and select R.E.M. So the logical move would be to put Kate Pierson in as the song artist, but when you do that the Zune player shows the song as being BY Kate Pierson, which of course it's not.

So there appears to be no consistency and no real thought put into what these two fields are to be used for and how they function in the player. My preference would be to see the player display album artist, and for that to be the primary artist field used by the player, so song artist could be used to indicate an additional artist in such instances as this (and any other song by an artist that "features" vocals from a guest artist as well). Song artist could even be renamed Featured Artist, Guest Artist or something along those lines.

My work-around is to use album artist AND song artist for the primary artist, but to append any guest vocalists to the song artist field. In the example above the album artist would be R.E.M. and the song artist would be R.E.M. & Kate Pierson.

It's set by an informal standard that most, if not all mainstream players use:  ID3v2.3  And, the tracks are tagged by the content owner.  You can, of course alter the fields to suit you, and you may even be correct about them being not well thought out, but this is the common practice, and it pre-dates the Zune.  And if you set the player to use both artists, you essentially can have your cake and eat it, too, except for the case where you have additional guest artists.  I see no realistic way to handle that except by adding such a field to ID3 (if it's not already there) and getting the content owners to use it (if or when it gets there.)  If you do your own customization of the tags, you run the risk of confusing the issue in the future if you move to a different player that treats the tag priority differently... that may or may not be a big deal, but it's tough to tell now.
