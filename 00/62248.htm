mtlipsc say:

Cursor hovering over task bar icons gives small screen image for MS Word, but only tiny icon on white background for MS Excel

I love the new Windows 7 task bar, but am puzzled by inconsistencies.  If I have a file open in MS Word (2007), and I put the cursor on the MS Word icon in the task bar, I see a small screen image of the open file.  But if I do the same thing to Excel (also 2007), instead of seeing the spreadsheet image I see a tiny Excel file icon in a blank white square.  This isn't helpful-- I want to see the small image of the open spreadsheet.  Excel is the only program that does this on the task bar.

cmdrkeene say:
there's info about this on the net (Paul Thurrott's site, WinSuperSite.com, has covered it).

Excel is what you call an MDI interface.  You can read about it on Wikipedia.  Basically, each Excel window is basically the same incarnation that a tab would be in a browser.  I can't think of any other MDI programs that come with Windows anymore, but old examples of similar programs include the File Manager or several examples on Wikipedia.

To get it's child windows (documents) to appear as separate application instances on the taskbar, Excel does some tricky stuff that isn't really optimal.  It's basically tricking the taskbar into showing them.  Because of this, no preview of child documents, unless you launch an entire new instance of Excel (so that the two documents are cousins (or step children?), rather than siblings of the same 'parent' window).

In Office 2010, they fix this issue by using some documented Windows 7 enhancements to how the taskbar works (more info about that on MSDN if you are feeling nerdy).

- - - 

For your 2nd issue:   Good guess, but this is actually not a compatibility program.  This is actually because the program's real executable code is running from another file than what the original button on the taskbar (the 1st button) points to.  To fix it:  run the program.  While it's running, right-click and remove the 1st button (un-pin).  Then (while the program is still running), right click the other icon (the one that reacts to the cover and such) and pin it.  This should work in most cases, but there are a few special (very very limited) cases where it won't.

tryx3 say:

David,

1  If the account is an MSAccount then log into it online and change its name there.  The links posted by SpiritX might help you with that.

2  If the account is a local one then run a command in an elevated command prompt

  1. First, run net user to see all the user accounts so you can quote the OldName correctly.
  2. Then run this using the OldName you have just found and the NewName you want to change it to

wmic useraccount where name="OldName" rename "NewName"

[Taken from

3  NewName [only] is then shown everywhere that I have been able to find

  • Settings, Accounts, Family & other people
  • Control panel, User accounts, Manage another account
  • Start menu, {click on current user to get a list of all users}
  • Command prompt net user
  • Elevated command prompt wmic useraccount list full

[this method is better than using Control panel to do the job because the 5 locations listed above would then contain a mixture of OldName & NewName references]

3  For both local user accounts & MSAccounts, you can change the folder name at C:\Users\YourUserName using the procedure in   Change Name of User Profile Folder - TenForums.
