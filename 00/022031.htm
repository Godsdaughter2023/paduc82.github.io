john mcghie say:
Hi Steven:

In addition to Phillip's suggestion, make sure that the file you are sending has a file name extension, and that the extension matches the content.

The file name extension .doc tells the receiving software that the file consists of a stream of bits, in binary format.

The file name extension tells the receiving software that the file contains a stream of computer code in XML format.

If either of those is not true, the website will be unable to process the file and will simply throw it on the floor.

suzanne s. barnhill say:

What Clear All Formatting does is apply the default Normal style. This is a dangerous thing to do if you have used other styles in the document (as you should). A different way to get the same effect is to select the text and press Ctrl+Spacebar. This will reset the font formatting to the Default Paragraph Font, and it may also remove some direct font formatting that you do not want to lose (italicized titles, for example), but if will also show the spelling errors.

The fact that you are not seeing the spelling errors could mean one of two things:

  1. "Hide spelling errors in this document only" could be enabled. If that were the case, however, then using Clear All Formatting would not reveal them.
  2. The text could be formatted as "Do not check spelling or grammar." This is undoubtedly the case in this instance. Instead of using the nuclear option (Clear All Formatting), instead select all the text, go to Review | Language | Language | Set Proofing Language and clear the check box for "Do not check spelling or grammar."

The exporting getting disconnected issue may be related the huge size of the in-place archive. To exclude the Outlook profile effect, you can create a new profile and then do the exporting to see if the issue persists.

Currently,  the eDiscovery tool is used to export the whole mailbox data, not applicable to some specific folders in the mailbox.

If you can accept it as a workaround, here is the detailed steps for your reference.

First, please check whether the In-Place hold is a supported in the Office 365 plan you are using right now.

You can refer to this article to check it: Which service plans or licenses do I need for eDiscovery?

Now, follow the steps below to do the eDiscovery:

1. Sign into Exchange admin center with an ADMIN account.

2. Go to permissions and double click Discovery Management.

This and the next step are necessary, because you need to be a member of the Discovery Management role group to be able to use eDiscovery features.

3. In the Discovery Management Role Group window, add your account to the Members list using the plus button and click Save.

4. Make sure that Legal Hold, Mailbox Import Export and Mailbox Search roles are added in the Roles section (if not, correct this using the plus button).

5. Back to the Exchange admin center, go to compliance management, click the plus button to start a new in-place eDiscovery & hold wizard, type in a name and click Next.

6. If you want to export data from all mailboxes, select Search all mailboxes.
Or select specific mailboxes using the Specify mailboxes to search option, and then clicking the plus button. This will open a new window  – highlight the mailboxes you want to export data from, click add -> and OK.
Additionally, you can enable exporting from Public folders by checking the Search all public folders option (in the Public folders section at the bottom of the window)

7. The next step gives you several filtering options. Select your conditions and filters. (If any)

8. After clicking Finish, a dialog box will show the progress of settings being saved. When it finishes click Close.

9. Now that you have set up an eDiscovery search, you can export its results to PST files. To do this, click the downward arrow button (marked red in image below).

10. This will download the eDiscovery PST Export Tool and offer to select a location to download the PST to.

Do not check the Enable deduplication option, unless you want the search results to be exported to a single PST file.

 

For more on unsearchable items see this TechNet article.

In-Private browsing is not the same as anonymous browsing. If you log in to Facebook or Twitter during a private browsing session, for example, your browser will keep no record of these visits or your login credentials, but the site in question will be able to identify you.  Likewise, if you have Google Web History enabled and you're logged in to the service during a private browsing session, your searches are stored on Google's servers, even if they aren't saved in the local browser's records. Any bookmarks you save while browsing in private mode are kept when you return to normal operation, as are any browser settings you make changes to.

 

Best you can do is clear the Domain Name Server (DNS) cache which saves internet files to facilitate rapid connection.  Open Command Prompt (admin)...enter ipconfig /flushdns  and hit Enter on your keyboard.  (one space before the (/)
