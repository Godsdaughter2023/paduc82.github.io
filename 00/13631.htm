charles kenyon say:
The download that Doug gave you is comprehensive, but if you want to get a look at what a field like this looks like, see Calculated Dates in Microsoft Word using Fields or vba

The field looks like:

{ QUOTE "{ SET " Delay" "30" }
{  SET "DaysInMonth" { IF { CreateDate \@ "MM" } <> 2
{  = ROUND(30.575*{ CreateDate \@ "MM" },0)-
ROUND(30.575*{ = { CreateDate \@ "MM" } -1 },0) }
{  IF {  = MOD( { CreateDate \@"yy" } , 4 ) } > 0 "28" "29" } } }
{ SET "NextMonth" {  IF {  CreateDate \@ "MM" } = 12 "1/97" 
"{ = {  CreateDate \@ "MM" } + 1 }/97 } }
{  IF {   = {  REF "Delay" } + { CreateDate \@ "dd" } } <= { REF"DaysInMonth" }
{ CreateDate \@ "MMMM { = {  REF "Delay" } + { CreateDate \@ "dd" } }, yyyy"}{ QUOTE { NextMonth \@ "MMMM" } 
{ = { REF "Delay" } + { CreateDate \@ "dd" } - { REF"DaysInMonth" } }, 
{ IF { CreateDate \@ "MM" } <> 12 { CreateDate \@ "yyyy" }
{ CreateDate \@ "{ = 1 + { CreateDate \@ "yyyy" } \# "xxxx" }" } } } }" }


I do not recommend that you try to create this field on your own. The download from Paul Edstein (macropod) has working fields like this, and more, that you can copy into your document. The above field is for display only. It has spacing that would make it not work if translated into an actual field.

I concur in his recommendation that you download it and play with the fields in it to get a feel for how this works.

Remember that the braces { } cannot be simply typed in from the keyboard, you have to get Word to insert them using Ctrl+F9 or some other method. Alt+F9 toggles display of all field codes. F9 updates field codes.

e.j. gun say:

Here is what I did:

  • Select the cells you want for format.  I assume they already have 0, 1 or 2 in them.
  • Under Conditional Formatting, select "Icon Sets"
  • Under Icon Sets, select "Three Traffic Lights (Unrimmed)"
  • That's it!

The default for that type of conditional formatting is Percentage, splits 100% into 0% to 32% = red, 33% to 66% = yellow, and 66% to 100% = green.  A value = 0 is 0%, so it ends up red.  A value of 1 is in the second range (50% of 2), so it becomes yellow.  A value of 2 is, of course, 100% so it turns green.

If you do that and you still don't get the Red/Yellow/Green lights as in my picture, then there is probably something wrong with your data.
