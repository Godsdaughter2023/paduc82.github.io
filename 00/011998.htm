Footnotes formatting: footnote text aligned to the right of the footnote number

http://imgur.com/WhqVnrp

How to achieve this in Word 2016 for Mac? 

john korchok say:

Your sample would require either a macro to insert a tab character in place of the normal space character that's in between the footnote reference number and the text, or you would have to manually replace the space with a tab for every footnote. That's kind of an elaborate solution.

But if you can settle for somewhat less space between the reference number and the footnote, you can revise the Footnote Text style:

  1. With your document open in Word, choose Format>Style.
  2. Change the List dropdown to All Styles.
  3. Find Footnote Text and select it, then click on the Modify button..
  4. In the Modify Style dialog, if you want to add this style change to your template so that all future papers have this formatting, check Add to template.
  5. Click on the Format button and choose Paragraph.
  6. Select the Special dropdown and set it to Hanging.
  7. In the By: field, change 0.5" to 0.11". This amount will align the left edge with the first line for footnotes number 1 through 9. OK out.

If you regularly have many footnotes where there are often  double digit footnote numbers, you could set the hanging indent to a larger amount.

jay freedman say:
Thanks, Joe, that information makes all the difference.

The dates in the table are just plain text, which means there's no way to format them all at once the way you could in Excel (or in a real "chart" that uses Excel as its data source). I don't think there's any Find/Replace way to do this, either. But a macro will make short work of it...

Sub x()
    Dim cel As Cell
    Dim rg As Range
   
    For Each cel In ActiveDocument.Tables(1).Columns(1).Cells
        Set rg = cel.Range
        rg.MoveEnd wdCharacter, -1
        rg.Text = Format(CDate(rg.Text), "MM-dd-yyyy")
    Next
End Sub

As written, this assumes that the table containing the dates is the first table in the document, and that the column containing the dates is the leftmost column in that table. If those assumptions are wrong, change one or both of the (1)s in the expression ActiveDocument.Tables(1).Columns(1).Cells.

The macro goes into each cell in the column and grabs the text. (The MoveEnd statement is needed because the range of the cell includes the end-of-cell marker, which isn't part of the date text.) That text is converted to a Date object by the CDate function. That date is fed into the Format function, and we tell Format to use two-digit numbers for both the month (MM) and the day (dd).

Because CDate isn't too fussy about the formats it accepts as dates, you wouldn't have to replace the slashes with hyphens in the table before running this macro -- it will produce the same output either way.

macropod say:

More flexible and faster:

Sub FixDates()
Application.ScreenUpdating = False
With ActiveDocument.Tables(1).Range.Find
  .ClearFormatting
  .Replacement.ClearFormatting
  .Forward = True
  .Wrap = wdFindStop
  .Format = False
  .MatchAllWordForms = False
  .MatchSoundsLike = False
  .MatchWildcards = True
  .Text = "(<[0-9]/)"
  .Replacement.Text = "0\1"
  .Execute Replace:=wdReplaceAll
  .Text = "(<[0-9]{2})/"
  .Replacement.Text = "\1-"
  .Execute Replace:=wdReplaceAll
End With
Application.ScreenUpdating = True
End Sub
