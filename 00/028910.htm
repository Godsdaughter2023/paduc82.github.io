You are mixing the vocabulary, referring to tables when you mean views and vice versa.

Tables and Views are different things and the terms are not interchangeable.

rich007a say:

Excel does not have a double click event for charts.

Hi Telylyn,

That's not true, there is a BeforeDoubleClick Event for charts.  To demonstrate:

Insert a Class Module into your vba project.

In the Properties window, name the class module EventClassModule

Paste the following into the class module:

Public WithEvents myChartClass As Chart

    MsgBox "BOO"
End Sub

Insert a normal module, and paste the following:

Dim myClassModule As New EventClassModule

Sub InitializeChart()
    Set myClassModule.myChartClass = ActiveChart
End Sub

With a chart selected in your workbook, run the InitializeChart procedure.

Now double-click on the chart, you should get the "BOO" message box.

Getting this to work robustly is far from trivial though.  And of course, figuring out how to select the chart data is not easy (what if the data is spread over more than one sheet?!?), but I just wanted to point out that it can be done (with a lot of work).

Create a separate form for each of the tables.

Unless the number of fields makes it unwieldy, create a continuous form based on the Tasks table and another continuous form based on the Deliverables table.

Create a standard ("single") form based on the Projects table, and place a tab control on it with three tab pages.

Place controls bound to the fields of the Projects table either outside the tab control (if you want to see them all the time) or on the first page of the tab control (if you only need to see them when you work with them).

Outlook 2013 sent folder not sync with Google mail

Anyone has any clue? Kindly help and hope to see respond soon. Thank you.

a. user say:

Hi Jin,

If you are using Outlook IMAP account on several devices like desktop, laptop and phone, you can keep your Inbox and Sent mail synced across your devices. This works great, however, you must ensure you do not store Sent Items locally on the machine. Otherwise, these will not sync to the other devices via IMAP.

The result is that Outlook will ignore the existing Sent Items and Deleted Items folder on the server and create some local ones of its own. When you set the Root folder path option in your account settings, Outlook will usually recognize the correct folders. Follow the steps below on how you can root your mailbox:

  1. Go to File click on Account Settings.
  2. Double click on your IMAP account.
  3. Select More Options.
  4. Click on Advanced.
  5. Select Option and click on Root folder path.
  6. Usually setting this to Inbox will do the trick.
  7. Click OK out all of the opened dialogs.

Update us with results for further assistance.
