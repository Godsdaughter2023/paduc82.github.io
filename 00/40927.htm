Hi, it's highly unlikely it's the hard drive. There are so many factors to take into account, your connection to the live servers isn't a direct one, your connection passes through so many places during it's journey and if one of those places slows things down you're up the creek.

Time of day can be a factor too, if I test my connection at peak times it's always slower, during the early hours of the morning it's pretty much maxed out.

In case if issue still persists, try to re-register Start Menu using these steps:

1. Open Task Manager, click File > Run new task. 


2. Type powershell in the Create new task box. Check the option 'Create this task with administrative privileges'. Click OK.

Get-appxpackage -all *shellexperience* -packagetype bundle |% {add-appxpackage -register -disabledevelopmentmode ($_.installlocation + “\appxmetadata\appxbundlemanifest.xml”)}

This issue could be caused when password protected sharing is turned on.

You can follow the article below to further narrow down the root cause of the issue.

Method 1: Turn off password protected sharing.

  1. Now open Control Panel once again.
  2. Then click on All Control Panel Items.
  3. Then click on Network and Sharing Center.
  4. Then click on Advanced Sharing Settings.
  5. Under All Networks > Password protected sharing.
  6. Check Turn Off password protected sharing.
