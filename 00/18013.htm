suzanne s. barnhill say:

This can happen when one of the number fields gets out of order. In general, the easiest way to fix this is to start from scratch for that level, clearing the Number format box and then reinserting each level in turn using Previous level number. It helps if you temporarily set the starting number of each level to its level number (that is, start Level 1 at 1, Level 2 at 2, etc.). Then your Level 4, when correctly set up, will show 1.2.3.4. Once you get it straightened out, set the starting number for all levels back to 1.

In this particular instance, however, it looks like you're using bog-standard 1, 1.1, 1.1.1 numbering, so it will be much easier just to choose one of those options from the Multilevel List gallery. If you're using the built-in heading styles (much the best plan), choose the one that shows 1 Heading 1, 1.1 Heading 2, 1.1.1 Heading 3. If you're using custom styles, choose the one that's not linked to the built-in heading styles and manually link the levels to the appropriate styles.

For more, see http://shaunakelly.com/word/numbering/numbering20072010.html

I think you want SQL like:

     [Customer Service Information]![Customer] & ", " & [Customer Service Information]![Location] &

[Production Order Activity].Procedure, [Production Order Master].DateReceived, 
[Customer Filter Information]![Item] & ", " & [Production Order Master]![FilterID] AS ItemFilterID, 
Date()-[Production Order Master]![DateArrived] AS DaysOnTheYard, 
IIf([Production Order Master]![StatusNotes] Is Null,[Production Order Master]![SalesNotes],
IIf([Production Order Master]![SalesNotes] Is Null,[Production Order Master]![StatusNotes],
IIf([Production Order Master]![StatusNotes] Is Null And [Production Order Master]![SalesNotes] Is Null,"",
[Production Order Master]![StatusNotes] & " (Sales: " & [Production Order Master]![SalesNotes] & ")"))) AS SalesStatusNotes, 
[Production Order Master].TotalCount, [Production Order Master].InterimNumber, [Production Order Master].[R&DItem], 
[Customer Service Information].Customer, 
IIf([Production Order Master]![DateReceived] Is Null,"To Be Received",
IIf([Procedure]="Received","To Start In Production",IIf([Procedure]="Awt. P/U-Delivery","Ready To Ship",
IIf([Procedure]="Packed","Ready To Ship",IIf([Procedure]="Hold","On Hold","In Production"))))) AS Status, 
[Production Order Activity]![Procedure] & ", " & [Production Order Activity]![DateTime] AS Expr1, 
[Production Order Master].AdjustedShipDate, 
[Customer Filter Information]![NormalTurnaround]+[Production Order Master]![DateReceived] AS [Target Ship Date]

FROM ([Customer Filter Information] 
     INNER JOIN ([Production Order Activity] 
INNER JOIN ([Production Order Master]

ON ([Production Order Activity].WorkOrder = LastProductionActivity.WorkOrder) AND 
([Production Order Activity].DateTime = LastProductionActivity.MaxOfDateTime)) 
ON [Customer Filter Information].CustomerItemCode = [Production Order Master].CustomerItemCode) 
INNER JOIN [Customer Service Information] ON [Customer Filter Information].[Customer Code] = [Customer Service Information].[Customer Code]

WHERE [Production Order Master].DateArrived) Is Not Null AND

ericqc say:

I've had this problem as well Adrienne, and would love to see a proper solution. Although Shift-Return (new line) will cause a justified line to spread to the right margin, pressing Return ends the paragraph, and the final line will just be set left.

I do have a workaround specifically to break a page with a justified final line:

  • Position the cursor where you want it to end and press Shift-Return and then Return. Highlight both the ¶ symbol and new line symbols and change the font attribute to something very small (like 2pt). This will keep the last apparent line of the now-split paragraph justified. (If it isn't already on, you may need to turn on "Show All" so you can see what you'll need to select.)
  • The remainder of your "split paragraph" will now be a new paragraph on the following page. I've only used this when I've needed to have text flow past a full-page graphic or a landscape section.

This will work for what you want, but there is a better method — and it makes use of Paul's 2nd point about text wrapping even if you need to include a caption.

  • Insert a 1 row x 2 column table after the paragraph you wanted to split. Insert the image in the upper cell, and enter your caption in the lower cell.
  • Open Table Properties and set the preferred width to 100%; Options to 0 margins; and Borders to None.
  • Then click Text wrapping=Around and click Positioning... to have the table located horizontally Left relative to Column, and Vertically Top relative to Margin, with distances to surrounding text all set to 0.
  • The table will now occupy the top of the page with the anchor point, leaving you with a justified last line on the previous page, and the rest of your paragraph continuing after the table.

In the screen shot below, the blue-highlighted paragraph ends with a justified last line on the 1st page using my first method above. The 2nd page is then able to start with an inline image followed by a caption, and then the remainder of the "split paragraph" content. I used the 2nd method to have the image and caption placed at the top of page 3. You can see how the yellow-highlighted paragraph flows from page 2 past the same image and caption being held in a 1x2 table.

The "floating table" method is easier to manage because new content added above it won't cause the workaround break to flow over to a new page, and you aren't left with hard-to-notice small paragraph breaks.

macropod say:

Ok, there are multiple issues here you need to deal with. Regardless, splitting the paragraphs is not required.

When artwork is positioned with text wrapping 'Square' (i.e. as a Shape object) and you then use Word's tools to add a caption, Word inserts the caption into a text box. In such situations, the caption ordering for multiple shapes anchored to a single paragraph is hard to control because it’s based on the order of the anchors in the document stream (which you can’t see). Captions in textboxes create other problems of their own if you create a Table of Figures; in that case, Word inserts a fully-qualified path into the TOF, which creates problems when the file is saved as PDF (the TOF links back to the original document). Captions for artwork inserted inlineshape (i.e. as an InlineShape object), on the other hand, are likewise inserted inline with the text.

A better approach, IMHO, is to insert the artwork as an inlineshape in a floating table, with the caption in the table also. The following macro processes all shapes & inline shapes, but it could be stripped down to work with just the shapes or even just a selected shape. Either way, the disconnect between the shapes and their captions will be resolved. Moreover, by grouping the artwork and its caption in a table, the risk of the artwork appearing on one page and the caption on another is eliminated.

As coded, the macro embeds all Shapes & InlineShapes in a document as InlineShapes in tables with a row for their Captions (comments at the end of this post suggest changes to limit which artwork the macro processes). Whatever positioning applied to the original Shape object will apply to the table also.
As coded, the MakeImageTable macro (which is called by the AddImageCaptionTables macro) limits the inserted InlineShape dimesnsions to 7.5*5.0cm. That code can, of course, be edited/omitted, in which case, so too can the corresponding code that re-sizes the inserted InlineShape to fit. The purpose of that code is to ensure consistency in the size of the artwork.

To have the caption row appear above the artwork, instead of below, change the Rows(1) references to Rows(2), change the Cell(1, 1) references to Cell(2, 1) and change the change the Cell(2, 1) references to Cell(1, 1).
Note that, due to the extra space taken up by the caption row, all except for the first of the document's artwork is liable to shift position. Dragging any existing captions into the caption row and deleting any unwanted caption rows should resolve most, if not all, of that.

Sub AddImageCaptionTables()
Dim iShp As InlineShape, Rng As Range, Tbl As Table
Dim i As Long, PicWdth As Single, PicHght As Single, VPos As Single
Dim HPos As Single, VRel As Long, HRel As Long, BShp As Boolean
With ActiveDocument
  For i = 1 To .InlineShapes.Count
    If .InlineShapes(i).Range.Information(wdWithInTable) = False Then
      PicWdth = .InlineShapes(i).Width
      Set Rng = .InlineShapes(i).Range
      With Rng
        If .Characters.Last.Next.Text = vbCr Then .Characters.Last.Next.Text = vbNullString
        PicWdth = .InlineShapes(1).Width
        PicHght = .InlineShapes(1).Height
        .InlineShapes(1).Range.Cut
      End With
      BShp = False: VRel = 0: HRel = 0: VPos = 0: HPos = 0
      Call MakeImageTable(Rng, PicWdth, PicHght, BShp, VRel, HRel, VPos, HPos)
    End If
  Next
  While .Shapes.Count > 0
    BShp = True
    With .Shapes(1)
      PicWdth = .Width
      PicHght = .Height
      VRel = .RelativeVerticalPosition
      HRel = .RelativeHorizontalPosition
      VPos = .Top
      HPos = .Left
      Set iShp = .ConvertToInlineShape
    End With
    With iShp
      Set Rng = .Range
      .Range.Cut
    End With
    Call MakeImageTable(Rng, PicWdth, PicHght, BShp, VRel, HRel, VPos, HPos)
  Wend
End With
End Sub

Sub MakeImageTable(Rng As Range, PicWdth As Single, PicHght As Single, BShp As Boolean, _
  VRel As Long, HRel As Long, VPos As Single, HPos As Single)
Dim Tbl As Table, sngScl As Single, iShp As InlineShape
'Constrain maximum image size
If PicWdth > CentimetersToPoints(7.5) Then
  sngScl = PicWdth / CentimetersToPoints(7.5)
  PicWdth = PicWdth / sngScl
  PicHght = PicHght / sngScl
End If
If PicHght > CentimetersToPoints(5) Then
  sngScl = PicHght / CentimetersToPoints(5)
  PicWdth = PicWdth / sngScl
  PicHght = PicHght / sngScl
End If
'Create & format the table
Set Tbl = Rng.Tables.Add(Range:=Rng, Numrows:=2, NumColumns:=1)
With Tbl
  .Borders.Enable = True
  .Columns.Width = PicWdth
  .TopPadding = 0
  .BottomPadding = 0
  .LeftPadding = 0
  .RightPadding = 0
  .Spacing = 0
  .Rows(1).HeightRule = wdRowHeightExactly
