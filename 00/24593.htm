doug robbins - mvp office apps & services (word) say:

While you could run a macro containing the code

ActiveDocument.ConvertNumberstoText

on the source document so that when you bring across text from that document, the paragraph numbers will be "preserved", there is no similarly automatic way to grab the page number on which the text was located other than to put the selection in the source document at the location with respect to the text where you want the page number to appear and run a macro containing the code

With Selection

    .InsertBefore .Information(wdActiveEndPageNumber)

End With

That will insert the number of the page on which the selection is located so that it could be included in the text that you bring across.

I think the next thing to do is another relatively simple procedure: Perform a Safe Boot. It will take a long time for your Mac to start up in Safe Mode. When you do a Safe Boot Mac OS fixes troubles with your Mac's directory and file system. The font cache is emptied. Only a limited number of fonts are enabled. Applications that run at startup don't run automatically. This allows you to test your app(s) by running them with a minimal set of fonts, an empty font cache, and without potentially conflicting apps running.

 

Once you reach the desktop in Safe Mode try running the app having a problem. Is the problem still there?

 

If the problem is gone: Performing a Safe Boot may have fixed the ailment. Or perhaps one of the fonts that wasn't loaded is bad. Or perhaps some other software utility is interfering. After testing in Safe Mode, do regular restart.

 

If the problem is still evident in Safe mode, you may still have a bad font. Use Font Book to restore system fonts, then test again in Safe Mode.

 

If the problem was gone in Safe Mode but returns during a regular boot: I think the first thing to do would be to use a font utility to validate ALL your fonts. Mac OS comes with a free font utility called Font Book, which you can use for this purpose. Get rid of any font that doesn't pass as clean and green. Then, restart your Mac and immediately empty the trash. If you've gotten this far and still have a problem, then there are more things to explore. But if you get this far you've probably gotten rid of the crash. Either way, please reply with your results.

How to get OneNote SharePoint to open in the desktop version

 

The problem starts with installing the “Sharepoint Document Viewer 2007” for certain groups. It installs a file named “OWSSUPP.DLL” and its loaded into the IE Add-Ons and points to Office 12 when the Office365 Sharepoint needs it to point to Office15. So follow the directions below and you can get the correct DLL to register.

 

  1. Open IE and go to Tools – Manage Add-ons
  2. Change the “Show” option to “Run without permissions”
  3. Find the “SharePoint OpenDocuments Class”. Click on it to highlight it and look at the “more information” link on the bottom left hand side, if it shows its pointing to Office12 then it needs to be disable (there maybe 2 listed, disabled both)

 

 

  1. Once that is done, go to the Control Panel – Programs and Features and run a “repair” on Office2013. It will want to reboot afterwards.
  2. Then you can go back to IE – Tools – Manage Add-ons and check to see that it’s now pointed to the “Office15”.  It shows “disabled” but when I tested OneNote on SharePoint, it worked fine.
