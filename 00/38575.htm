Again, there are limitations going from still to video that you may still not be able to overcome. In particular fine line, high-detail portions will get blurry. Higher end video programs might do a better job of maintaining the fidelity of images compared to Movie Maker, but even there they can struggle.

The error 0x80042308 is related to Volume Shadow Copy.

Check if the Volume Shadow Copy service is running on your computer or not.

To check for the same,

1. Click ‘Start’, type ‘services.msc’ and hit Enter.
2. Locate “Volume Shadow Copy” and check if the service is started.
3. If not ‘Start’ the service.

If the service was already started, I’d suggest you to stop the service and restart the same. Repeat the above steps for ‘Windows Backup service”, too.

Shadow copy errors are often caused by disk problems (usually minor) which check disk will find and correct.

You can solve some computer problems by checking a drive or drives for errors. For example, you can check your computer's primary hard disk to fix some performance issues, or you can check an external hard drive if it's not working properly.

Follow the below steps:

1. Open Computer by clicking the Start button, and then clicking Computer.

2. Right-click the drive that you want to check, and then click Properties.

3. Click the Tools tab, and then, under Error-checking, click Check now. If you're prompted for an administrator password or confirmation, type the password or provide confirmation.

To automatically repair problems with files and folders that the scan detects, select Automatically fix file system errors. Otherwise, the disk check will report problems but not fix them.

To perform a thorough check, select Scan for and attempt recovery of bad sectors. This scan attempts to find and repair physical errors on the drive itself, and it can take much longer to complete.

To check for both file errors and physical errors, select both Automatically fix file system errors and Scan for and attempt recovery of bad sectors.

4.    Click Start.

hairyfool say:

Then that has nothing to do with VPN but is entirely about port forwarding.

When you create your Web server you will define a listener port, default for WWW is port 80, secure is 443 but whilst in development it is better to define a non standard port so you know but no one else does (8080 is fairly common for admin connections. Also the web server needs a static IP address so you need to assign that and make sure the routers DHCP excludes that address.

Then on the router you need to forward any inbound connections on that specific port to the address of the server. Some routers config utilities will define both the forwarder and firewall rules as the same task others require it as two separate settings. Allow the inbound connection is firewall, send it to the server is forward.

One thing to bear in mind is most domestic ISP use DHCP for your internet facing connection so the address your server is available on the internet will change (it is not the address you assign the server itself). Dynamic DNS gets around this but if you are serious you need to speak to your ISP about getting a fixed IP which might be chargeable

The process for game servers is much the same except they will have their own listener ports and a general guide can be found here

http://portforward.com/english/routers/port_forwarding/routerindex.htm
