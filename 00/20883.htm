What basically Bruce mentioned in the thread above is correct. I would like to complement some points to it to let you better understand the ADFS/SSO with Skype for Business client.

For domain-joint computers, you will get seamless experience of SSO when ADFS is deployed even if the password is changed in AD. Users will not get prompted for credentials.

ranmo_msft say:

Hi Jean,

Regarding “This would be true if we synced passwords using DirSync, orrect?”, for domain-joint PCs, if you only deployed DirSync with password Sync, when the password changed, users will still get prompted for updating credentials. The seamless experience can only be obtained for domain-joint PCs when ADFS 2.0/3.0 is deployed. It doesn’t matter whether password sync is deployed in this case. Users will not get prompted when passwords changed.

So, briefly, regarding “What would we need to implement to remove the reliance on Credential Manager and have true SSO?”, two conditions are needed.
1. Domain joint PCs.
2. ADFS deployed.
