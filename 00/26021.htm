ken sheridan say: Setting the DefaultValue property of a check box control to False will show it unchecked when the form opens if it is an unbound control, or when you move to an empty new record if it's a bound control.

john w. vinson say:

The green "fuzz" in the checkbox simply means that the value in the underlying table is NULL - unknown, undefined, neither yes nor no. You'll see this on the New Record, or if the checkbox is "unbound" (nothing in its Control Source property).

Note that a Yes/No field actually stored in a table must be either True (stored as -1, show as Checked in a checkbox, or as Yes, True, -1 in a textbox, depending on how you have it formatted) or False (0, unchecked, No, False). The NULL value only exists on a new record that hasn't been written to disk yet.

Note also that data is NOT stored in Forms. A form is only a tool, a window that lets you manage data stored in a Table. Data can be stored in tables, and ONLY in tables; you will use queries and forms to manipulate the data, but the Tables are the bedrock of the application.

phillip m jones say:

Here lately, for some reason, the installers for Office 2016 updates, has been messing with Office 2016 Passwords stored in KeyChain Access. The following procedure might help, also, it might not make a difference, or you may find items I suggest be changed, have been unaffected:

  1. Quit all open Applications (including Microsoft Office applications).

  2. Next open KeyChain Access.

  3. Click on: Passwords.

  4. In the search box type in: Office.

  5. The should be 3 to 4 items that appear to have a serial or code number.
    (In the Description area you should see reference to Version 15.0 or 15.2, or 15. with another number related to current 2016 series)

  6. Now click on the icon with an i in the center.


  7. Click on checkbox next to : Show Password.

  8. Check, If the password you used to activate, does not show in each of the files, or is mangled.

  9. If so then enter the correct Password. 

  10. Repeat, above steps starting at step 5, for each file you find (3-4 possibly).

  11. Once finished make sure to save changes.
    Quit KeyChain Access.
    Try word, Excel PowerPoint or OneNote.
    It's still possible you may have to enter password but then only have to do so once, and others should already be signed in.

Note: This may not apply to El Capitan
