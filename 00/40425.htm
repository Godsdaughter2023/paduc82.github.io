carolina calling say:

You have not told us which version of Windows® you are running or your system hardware.  (While this is the Windows XP hardware forum, Vista and others often slip in unannounced.)

The reason I ask is, if you have a Windows® XP installation disk, you can get past the first the phases of the boot process.

If you boot the Windows® XP install CD-ROM and then do nothing, it should load the boot block from the CD, type the message "Press any key to boot from CD" and sit there for a short time.  If nothing is typed, it boots the Windows® system on the primary Hard Disk Drive (HDD).  This skips the reading of the Master Boot Record (MBR) and the loading of the boot sector from the HDD.  If your system boots into Windows® this way, you will know the problem is either the MBR or the boot sector and you will at least have a fighting chance to recover without a full re-install.

The MBR and boot code can be restored using the Microsoft Windows XP Recovery Console if you have an x86 CPU .

First read: KB307654 - How to install and use the Recovery Console in Windows XP   (CC: skip the install, use the CD)
                KB314058 - Description of the Windows XP Recovery Console for advanced users
                KB291980 - A discussion about the Bootcfg command and its uses

You may want to go to http://support.microsoft.com and search for "windows recovery console xp" for further information.

To boot the Microsoft Windows XP Recovery Console using the CD:

  • Be sure the BIOS is set to boot your CD/DVD first,
  • boot,
  • press a key when it asks and
  • type "R" (for Repair) at the first prompt.

The Recovery Console  will present a numbered list of Windows installations available on your disk.  You normally look for the installation with x:\WINDOWS where "x" is the primary HDD's drive letter.  Note: this is the BIOS' list of drive letters which is not necessarily the same as your Windows installation.  Type the number of the installation you want to boot and press <enter>.  The next prompt will ask for the Administrator's password.  Type the password and press <enter> or just press <enter> if there is not one.  The Recovery Console will return a prompt as in DOS.  i.e. {drive letter}:{directory}>.

"Help" will show a list of commands. "Help fixmbr" will explain how to enter the command to fix the Master Boot Record (MBR).  "Help fixboot" will tell how to write a new boot sector.  Use them only if you have an x86-based computer.

malkeleah say: Do you have a floppy in the floppy drive? If yes, take it out and reboot. Is all OK now?

Do you have a CD in the CD drive? If yes, take it out and reboot. Is all OK now?

If the answer to both questions above is "no", then your computer is not finding any operating system files on the boot device, usually a hard drive. Check to see if your hard drive is found in the BIOS. If not, you can turn off the computer, unplug it, and make sure the cable going from the drive to the motherboard is snugly seated. Do the same for the power supply connector. If reconnecting these cables doesn't help and the drive is still not seen in the BIOS, pull it and test in another computer. If it is not seen there, it has died. If it is seen there, test it as described at the link below. If the drive is seen in another computer and tests fine, then the motherboard connection on the original computer is bad. Depending on the motherboard, you can sometimes put the hard drive on another cable but this is a sign that the motherboard is failing. If the computer is elderly, it usually is not cost-effective to replace the motherboard.
