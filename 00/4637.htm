albert d. kallal say:

There are two issues here. “Security” usually refers to a process in which say Tom can run certain reports for sales, and Sally cannot.

 

Then there is the separate issue of locking down the Access interface. You can quite well lock down and hide the Access interface from users – and do this without having to adopt some kind of Access security.

 

Most (if not all) of the options to lock down the application, and keep users “out” of the developer and design parts can be set via the start-up options. However, when you do this, then it is assumed you developed the application with this goal in mind. Given that the user can’t see the navigation pane etc., then “you” the developer are assumed to provide forms and possibly say a custom ribbon to ONLY display the bits and parts you want.

 

Or you may well hide the ribbon altogether.

 

The basic options for this would be to set a start-up form, and then un-check most of the options like [x] show navigation pane, and un-check “special access keys”.

 

And to hide the ribbon, you can in your start-up form (or start-up code you have now), simply add this one line of code:

 

DoCmd.ShowToolbar "Ribbon", acToolbarNo

 

There are a number of additional things you can (and should) also do. For example, you likely want to “compile” the application into an Access executable file (an accDE). This thus assumes you split your application (else you not be able to issue updates and changes to the application).

 

Of course when working on the application, you have to use (hold) down the shift key to by-pass all these start-up settings else you not be able to work. So as a very last step after compiling the application to a accDE, then you likely want to disable the shift by-pass key.

 

The above is a short summary, but the real work comes in now you ensuring that all things like reports etc. have some form or some kind of user interface to launch reports etc. since the user will not have use of the “built in” options such as the navigation pane.

 

Regards,

Albert D. Kallal (Access MVP, 2003-2017)

Pardon me for bumping in -- but budget resources will not do what you are seeking.  Project will calculated total project cost based on hourly rates and work.  (Note: even if you express rates for resources as daily rates, the cost is based upon work -- not passage of time).

ossiemac say:

I have assumed that you require a separator between each selection so my example uses a comma and space.

Because the comma and space is added to each selection, there will be a comma and space on the end of the final concatenated string. Therefore the code removes the last separator. If not using a comma and space then edit to your requirements where it is concatenated and if necessary edit the line that removes the last comma and space with the correct number of characters to remove.

The code could have been written to call another sub to create the string from the selections but for simplicity I have just included it with your other code.

Private Sub cmdSave_Click()
    Dim currentrow As Long
    Dim i As Long
    Dim strTemp As String
    Dim Lastrow As Long
   
    With Worksheets("tblData")
        Lastrow = .Range("A" & Rows.Count).End(xlUp).Row
        .Cells(Lastrow + 1, "A").Value = lstPris
        .Cells(Lastrow + 1, "B").Value = txtdate
        .Cells(Lastrow + 1, "D").Value = lstquart
        .Cells(Lastrow + 1, "F").Value = lstanswer
        .Cells(Lastrow + 1, "H").Value = lstcomp
       
        For i = 0 To lstresp.ListCount - 1
            If lstresp.Selected(i) Then     'If selected is True
                strTemp = strTemp & lstresp.List(i) & ", "      'Edit ", " to required separators
            End If
        Next i
       
        If Len(strTemp) > 2 Then    'The variable will always be greater than 2 if any selections in the listbox
            strTemp = Left(strTemp, Len(strTemp) - 2) 'remove last comma and space (Edit 2 to length of separators)
        End If
       
        .Cells(Lastrow + 1, "I").Value = strTemp
        .Cells(Lastrow + 1, "J").Value = txtfree
   
    End With
 End Sub
