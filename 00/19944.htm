jllatham say:

How to do it without VBA (I'll come up with VBA solution later and post if you still want to go that way).

You need to add 2 columns, and if you will put this first one next to one of your A,B,F,G,H or I columns that has an entry in it for every used row then you can rapidly fill the formulas I'm about to give you to cover all used rows.

First helper column formula, assumed to be on row 2, in column M, so in M2 enter this formula:

=A2 & B2 & F2 & G2 & H2 & I2

second helper column, row 2, assumes that the first helper column in M, so in N2 enter this formula

=COUNTIF($M$2:$M2,M2)>1

That will return FALSE when the first of an entry is encountered and TRUE when 2nd and subsequent duplicates are found.  Then you can filter column N for "TRUE" entries, select all displayed rows and delete them, leaving only the individual, unique entries.  Remove the filter, delete your helper columns and you're done.

 

Remember, you might even consider inserting a couple of new columns to be the helper columns.  Say that column A is the only column guaranteed to have an entry in it on each used row.  You could insert new B and C columns and modify that first formula to become:  =A2 & D2 & H2 & I2 & J2 & K2  then the second formula in C2 would become =COUNTIF($B$2:$B2,B2)>1

You can then rapidly fill both new formulas by selecting both B2 and C2 and moving your cursor to the lower right corner of C2 until it becomes a + (thin cross) instead of the normal fat cross and double-click to fill those formulas to the end of the list of entries on the sheet.

[EDIT] Actually, the above is probably the fastest way to get it done.  I experimented with code and 55,900 rows of data, all but 6 of which needed to be deleted, and you don't want to wait that long!!  Actually, even with the above solution, if a good percentage of your 40,000 rows need to be deleted, when you select the rows and use Delete, you may want to go take a long coffee break - or lunch.

The foundation of a well running database is a proper table structure. And I have some concerns about your structure. You should have at least three tables here:

tblEmployees: EmployeeID (PK), names, etc

tblTraining: TrainingID (PK), TrainingName, CertLength, other info about courses

tblEmpTraining: EmpTrainingID (PK), EmployeeID (FK), TrainingID (FK), TrainingDate

With this structure you can produce a query that shows the certifications expiring. 

Something like:

SELECT EmployeeID, TrainingID, DateAdd("m",CertLength,TrainingDate) As Expire

FROM tblEmpTraining, tbpTrainin, INNER JOIN ON tblTraining.TrainingID=tblEmpTraining.TrainingID;

You can then use a WHERE clause to specify the ones close to expiring. From that query you can add in the name info and send an e-mail to all employees listed.
