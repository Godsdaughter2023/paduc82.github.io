can you automate / import drop down list or combo box details? - Microsoft Community

itjunkie say:

can you automate / import drop down list or combo box details?

I'm using Office 2007.

I have an Access database with about 5700 records.  I created a query to refine the data down to unique site codes - which in turn is about 120.  I want to include these side codes into a drop down or combo box within a Word document. 

Is there a way to auto populate the data for the drop down or must it be entered manually?

The query data resembles this format

site id (unique) / country / city / country code / region

The end result I'm looking for is a drop down list within my Word document that lists all the site id's from my query and display it in the format mentioned above.

jay freedman say:

The legacy dropdown field has a fixed limit of 25 items, and requires protecting the document for forms to operate, so that isn't going to do the job.

If you and all users are running Office 2007 or 2010, you can use a dropdown content control, which you can insert from the Controls group of the Developer ribbon (you have to go into the Word Options dialog and enable that tab).

After inserting a content control, click the Properties button in the same group and enter a tag value such as siteID.

Then write an AutoNew() macro and an AutoOpen macro like this:

Sub AutoNew()
    InitialzeSiteID
End Sub

Sub AutoOpen()
    InitialzeSiteID
End Sub

Sub InitialzeSiteID()
    Dim cc As ContentControl
    Dim i As Long
   
    On Error Resume Next

    Set cc = ActiveDocument.SelectContentControlsByTag("siteID")(1)
    If cc Is Nothing Then Exit Sub
    If cc.Type <> wdContentControlDropdownList Then Exit Sub
   
    cc.DropdownListEntries.Clear
   
    ' EXAMPLE ONLY! Modify this to run through the
    ' result of your query and insert the real site ID values
    For i = 1 To 120
        cc.DropdownListEntries.Add Format(i, "000")
    Next
End Sub

As the comment says, you need to replace the example loop with code to retrieve the results of your query and insert them into the dropdown's list.

Note that the name in quotes in the Set cc line needs to match whatever you enter as the tag of the content control; it is case-sensitive. Also note that the error handling in this example is primitive -- if the content control doesn't exist or if it isn't of the dropdown type, the macro simply exits without doing anything.

Jay Freedman

How can I get the file name to show up in a cell of the excel - Microsoft Community

dak say:

How can I get the file name to show up in a cell of the excel sheet?

For example, If i have a file named 2011-06 Household Budget.xlsx, it should say 2011-06 Household Budget in the selected cell?

 

Is this easily done?

chip pearson say:

You can use the CELL function to get the file and sheet names. CELL returns both the file name and sheet name in the format of "C:\Path[FileName.xlsx]SheetName". You can parse out the desired component from that string. So, to get the filename including the extensdion with

 

=MID(CELL("filename",A1),FIND("[",CELL("filename",A1))+1,FIND("]",CELL("filename",A1))-FIND("[",CELL("filename",A1))-1)

 

Include "filename" as shown. Do not substitute the actual file name. The reference to A1 can be to any cell on the worksheet. It doens't matter which cell.

 

To get the filename without the extension, use

 

=MID(CELL("filename",A1),FIND("[",CELL("filename",A1))+1,FIND(".",CELL("filename",A1),FIND("[",CELL("filename",A1))+1)-FIND("[",CELL("filename",A1))-1)

rob koch say:

A PST file is a data file, not a program, so no it can not truly "become infected", though since email attachments can contain malware, it can of course contain them.  This is very similar to a zip or other container file, since though the programs that access them are different, the storage methods of compression have similarities.

Since the existence of multiple copies of a PST file for the same account on different machines are really just duplicate copies of the same file which are synced by the Outlook program(s) accessing them, any malware contained in an attachment will exist in all copies of the file until removed and synced to the others.

As with any other compressed file format, they can be read by programs that are aware of their format, so PST or OST security are basically the same as any compressed file security would be.  In other words, the security of the computer and drive on which the data file is stored are the important point, not the file itself.

Though PST encryption is mentioned in some documents, I don't believe this is truly a protection so much as a translation, making the file not simple readable text, but still not truly protected from reading by programs aware of the format used.

[MS-PST]: Outlook Personal Folders (.pst) File Format

Loop to update records - Microsoft Community

jwcrosby say:

Loop to update records

Using Access 2007

I have two tables in a one-to-many relationship. On the “one” side is the table of employee names, each with an autonumber ID. On the “many” side is a table with twelve records for each employee record with each of the records representing a different month of the year (Jan-Dec). I put a date field in each record set to the first of the month to indicate which month it's for.

tblEmployees

  • ID (auto number)
  • EmployeeName

tblMonthlyData

  • ID
  • MonthDate (ex. 1/1/2011, 2/1/1011, 3/1/1011, etc.)
  • MonthlyGoal (number)

I have a form where I enter the monthly goal for each employee for each month. Anytime a goal changes during the year I want to update all the subsequent monthly goals to the new number. I want to have a button so I can control if/when the updating occurs, rather than have it happen automatically, like as an “after update” event.

For example, say I enter an employee’s monthly goal for January to be 100. Unless I make a change later in the year, that goal will remain at 100 for each of the remaining months (Feb-Dec). Since I’m lazy, I don’t want to have to open each monthly record to enter “100” for the monthly goal. I’d like the computer to do that!

Similarly, say in June that employee’s monthly goal changes to 125. I’d open the record for June and enter 125. I’d like the computer then to update all the subsequent months (Jul-Dec) to 125 when I click the button.

I know it would involve a loop through a recordset, but I’m struggling with the exact wording.

Thanks in advance for any help.

Jerry

ken sheridan say: You'd select the latest row per ID and, append a new row per ID, using the DateAdd function to insert a date one month later than the latest date:

INSERT INTO tblMonthlyData(ID,MonthlyDate,MonthlyGoal)
SELECT ID,DateAdd("m",1,MonthlyDate),MonthlyGoal
FROM tblMonthlyData As MD1
WHERE MonthlyDate =
    (SELECT MAX(MonthlyDate)
     FROM tblMonthlyData As MD2
     WHERE MD2.ID = MD1.ID);

Make ID and MonthlyDate the composite primary key of the table to prevent duplicate rows per ID/month being inserted.
