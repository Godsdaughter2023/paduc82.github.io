jridge say: In many cases, the fix to this problem is on the web server itself. It is likely the web server is sending "no cache" headers for items a  user will download. 

IE tends to take "no cache" headers seriously - and won't save the bits of a downloading file to the local system - causing the error reported here.

The fix is to remove the "no cache" headers from requests that result in a downloading file. This will allow IE to save the file and work as normal. The only other solution I know if is to use the link DobrinDimov provided ;)

It is possible clearing cache "might" work - it could even be a permissions problem with the cache folder on the end-user machine. However, it is more likely to be caused by sloppy work on the part of the web service engineer (I should know, I've done it before ;)

davidf2 say:

Select all the content on the page and drag it to where you want it. When you change the length of the page as you did the extra length in the page is added to the top and bottom of the page, but the content itself is not changed.

 

If you want all the content moved to the new top of the page, I would Ctrl + A to select it all, then Arrange > Group to keep everything properly aligned and then use the arrow keys to move all the content up to where I wanted it. Or hold down the shift key and drag the group up with your mouse...holding the shift key will keep the content aligned vertically. Of course that then means you would have a blank space at the bottom of the page for more content. Ungroup the content after you move it all in order to move or add individual design elements.
