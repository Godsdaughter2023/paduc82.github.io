You don't say what you mean by "not working correctly".  However, there is no need to insert a new row after the last row of the table.  If the selection is in the last row of the table, the cells from the clipboard will be pasted after that row:

bernie deitrick say:

You really only need one entry for each:  Red Panda, and Cake  This macro will replace, for example, any string that contains the sub-string 'red panda' with Red Panda  - "I love red Pandas holding umbrellas"  "Red Pandas are great" "Who doesn't love red pandas?"   will all be replaced by "Red panda"  and anything with Cake with Cake

So, create a sheet named List, and start your table of words to look for in A2 running down column A. Then run the macro.

If you want to replace and string with a specific word with another word - say, Cake with the word Pastry - put Pastry into column B next to Cake and the code will do that.

Sub ReplaceList()
    Dim sh As Worksheet
    Dim rngC As Range
    Dim rngList As Range
    
    With Worksheets("List")
        Set rngList = .Range(.Range("A2"), .Cells(.Rows.Count, "A").End(xlUp))
    End With
    
    For Each rngC In rngList
        For Each sh In Worksheets
            sh.Range("J:J,O:O,T:T,Y:Y").Replace "*" & rngC.Value & "*", _
                IIf(rngC(1, 2).Value <> "", rngC(1, 2).Value, rngC.Value)
        Next sh
    Next rngC

End Sub

Sub ReplaceList2()
    Dim sh As Worksheet
    Dim rngC As Range
    Dim rngList As Range
    
    With Worksheets("List")
        Set rngList = .Range(.Range("A2"), .Cells(.Rows.Count, "A").End(xlUp))
    End With
    
    For Each rngC In rngList
        For Each sh In Worksheets
        
            If Application.CountA(rngC.Resize(1, 3)) = 3 Then
                sh.Range("J:J,O:O,T:T,Y:Y").Replace "*" & rngC.Value & "*" & rngC(1, 2).Value & "*", _
                rngC(1, 3).Value
                sh.Range("J:J,O:O,T:T,Y:Y").Replace "*" & rngC(1, 2).Value & "*" & rngC.Value & "*", _
                rngC(1, 3).Value
            End If
            
            If Application.CountA(rngC.Resize(1, 3)) = 2 Then
                sh.Range("J:J,O:O,T:T,Y:Y").Replace "*" & rngC.Value & "*", _
                rngC(1, 2).Value
            End If
            
            If Application.CountA(rngC.Resize(1, 3)) = 1 Then
                sh.Range("J:J,O:O,T:T,Y:Y").Replace "*" & rngC.Value & "*", rngC.Value
            End If
            
        Next sh
    Next rngC
    
End Sub

cortig say:

There are a few Mac apps that can extract the content from the .dat attachments.

I use TNEF’s Enough.

So that you understand what’s going on: Outlook for Windows supports different formats for rich text. One of these formats is not supported by anything but Outlook for Windows… The message is sent in two parts: the rich text, sometimes with attachments, in the .dat portion, and a plain text version in the second part. Since Outlook for Mac (or any other client for that matter) doesn’t understand that format, it displays the .dat portion as an attachment and shows you the plain text version instead.

The person sending you emails has to set Outlook to NOT use that format (and I bet you’re far from being the only recipient who can’t see the emails properly). Plain text will work of course, but HTML encoding would be fine too.

The Web is full of websites telling people how to change that setting.
