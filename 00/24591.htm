Outlook does not support CalDAV or CardDAV so sync between Outlook and Apple applications are slowly going away. CalDAV & CardDAV are protocols used by Apple, Google, Yahoo and others to sync contacts and calendars.

Apple has deprecated Sync Services in Mountain Lion and it no longer works reliably.

  • You can no longer sync with an iPhone or iPad via iTunes. You can only sync with an iPhone or iPad via a cloud-based service like iCloud or Google.
  • You can no longer sync local calendars with OS X Calendar (formerly iCal). You can, however, continue to sync calendars hosted on a cloud-based service like iCloud or Google with OS X Calendar.
  • You can no longer sync with any third-party apps running on your Mac that use Sync Services, like Entourage or Outlook.

_ronx say:

The presence (or absence) of FrontPage extensions normally does not affect the appearance of a website - the absence does affect the functionality of forms and other components (hitcounters) that require the extensions to run.

http://support.microsoft.com/kb/281532  lists the components that require the extensions, and I did not see any during a very brief look at your site (I could have missed the page that needs them).

The error message (beginning  " http://www.diabetes-your-blood-sugar-usa.com/ does not appear to have Windows Sharepoint services installed.  Possible causes: "
often indicates the extensions are not running correctly. My own tests show they are installed.
From what I have seen, switching to FTP publishing is feasible, though I would use a dedicated FTP client such as Filezilla rather than FrontPage.
Before you switch (if you do) check that you do not need the extensions for a feature I may have missed.

Using FTP will definitely corrupt the extensions - so the choice is FTP or Extensions, you cannot have both. This has nothing to do with the page formatting, which is governed by stylesheets (external, embedded and inline) and HTML tags.

"1. re: The original remote site  attempts to open, if/when the login credentials appear click cancel.
ron: i never get to see the 'remote site' details because it keeps on presenting me with the 'publish box' to publish by frontpage or ftp etc.
The 'publish box' is where you set FTP etc, and the location for the remote site.
If we are talking about the same thing, the box gives a choice of:
FrontPage or SharePoint services
WebDav
FTP
File System

There is also a text box that shows the remote website location.

If you are publishing to http://www.diabetes-your-blood-sugar-usa.com  and the servers were only switched in the last 48hours, it is possible FP is trying to publish to the old site - DNS can take 2 days to propagate.

"i am starting to think its NOT a matter of needing to change the long in credentials, but an issue with the new singlehop host server accepting upload by by the classical frontpage upload method??"

The most valuable part of the FP extensions (in my opinion) is the ease of publishing. Everything else can be done by other methods, usually with better results. If your present host does not accept frontPage publishing then the extensions are broken, or (in your case, possibly) the DNS change has not found its way to your ISP's DNS server.

  1. Place cursor in the first cell of my column header row (mine was row 13)
  2. INSERT + V
  3. Type in "name"

  4. Define Name Column and Row Titles Override = On for the Current File

  5. Regions Type Indication = Single (Sheet 1)

  6. Title Reading Detection = Off

  7. Titles Read For = Any Data Cell

  8. Column Titles to Row Range Set = 13 (the number of the row my column titles are in)

  9. Go back to first list box Define Name Column and Row Title Override and change to Off

  10. OK

Next step after selecting the file is to select the destination folder. Is there any chance you skipped a step in the above description before encountering the <Next> button being greyed out? Screen after selecting the destintion folder is the most common place where the scenario described occurs. That's the screen titled <Import a file> with a list box on the left side titled <The following actions will be performed>.

In that box - there will be at least one line with a checkbox labeled: Import <FileName> into folder <FolderName>. At least one checkbox in the list that appears (which typically is a list of one item) MUST be checked and then the MAP CUSTOM FIELDS button and NEXT button become enabled.
