graham mayor say:
There doesn't appear to be a built-in method of doing this, other than one at a time, but by using a bit of lateral thinking, it should be possible. Try the following macro on a COPY of your document.

Dim iFld As Field
Dim oRng As Range
For Each iFld In ActiveDocument.Range.Fields
    If iFld.Type = wdFieldCitation Then
        iFld.Select
        Set oRng = Selection.Range
        oRng.Start = oRng.Start - 1
        oRng.End = oRng.End + 1
        oRng.Select
        oRng.Text = iFld.Result
    End If
Next iFld

jllatham say:

Is it possible to limit Excel to 4 CPUs?

 

[File] tab -> Options, {Advanced} item, scroll down to the Formulas section and you will find the option to restrict the number of available CPUs from 1 to as many as are available.

 

I suspect the load is being caused by paging data in/out of Excel.  Not sure what you can really do about it.

 

You could do this programatically for the workbook as shown below.  Note that the _Activate() event triggers when you open the workbook, and when you return to it if you have several workbooks open and have been working in another one.  Similarly, the _Deactivate() event triggers when you use another workbook that is also open and when you close the workbook:

 

Private Sub Workbook_Activate()
  'will fire on _Open() also
  Application.MultiThreadedCalculation.ThreadMode = xlThreadModeManual
  'this can be set from 1 to as many physical/virtual CPUs you have
  Application.MultiThreadedCalculation.ThreadCount = 2
End Sub

Results:

 

Then in the _Deactivate() event:

Private Sub Workbook_Deactivate()
  'will fire when workbook is closed also.
  'return to automatic mode will also set
  'number of CPUs to us to "all available"
  Application.MultiThreadedCalculation.ThreadMode = xlThreadModeAutomatic
End Sub

with this result:

 

 

Finally, if you really want to tone it down, you can turn multithreading off entirely which basically takes you down to 1 CPU - it's a one-liner to turn it off, then another one-liner to reset it which takes you back to the "normal" setup state.

Application.MultiThreadedCalculation.Enabled = False

Results in this setup:

 

To return to normal setup

Application.MultiThreadedCalculation.Enabled = True
