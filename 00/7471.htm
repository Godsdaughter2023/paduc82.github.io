    CORREL of two OFFSETs with ROW-based argument(s) returns - Microsoft Community

    andyspragg say:

    CORREL of two OFFSETs with ROW-based argument(s) returns #N/A or #NUM! !?

    Hi folks,

    This has had me bamboozled for a couple of days. I can see no reason why it should not just work. I created a little illustrative spreadsheet but attaching it doesn't seem to be an option, so here's the bottom line. Two of the three formulae below, placed out of harms way and pointed to legitimate data, do not compute an answer.

    =CORREL(OFFSET(R1C1,1,RC1-1,20,1),OFFSET(R1C1,1,RC2-1,20,1))

    =CORREL(OFFSET(R1C1,1,RC1-1,20,1),OFFSET(R1C1,1,8-ROW(RC),20,1))

    =CORREL(OFFSET(R1C1,1,ROW(RC)-1,20,1),OFFSET(R1C1,1,8-ROW(RC),20,1))

    The first one returns the correct answer, and demonstrates that it is possible to provide integer-type arguments and not just integer constants (as we would surely expect)

    The second one returns #N/A, and demonstrates that it is not possible to use the ROW() function to provide an integer-type argument to OFFSET

    The third one returns #NUM!, and demonstrates that whatever is going on is even more weird than the second one illustrates.

    I have ruled out a couple of red herrings (e.g. that CORREL does not allow the two OFFSETS to be on different worksheets).

    I can achieve the desired effect other ways e.g. with INDIRECT, but since discovering OFFSET I have been gradually phasing out my use of INDIRECT except where absolutely necessary. Looks like this might be one such necessity ... but for goodness sake, why?

    Andy

    hansv mvp say:

    The problem is that ROW(RC) returns an array, even though RC is a single cell. In many situations, Excel converts this array to a single number automatically, but that fails here (I don't know why). You can get around this by using SUM(ROW(RC)). Since the array has only one element, the sum is equal to that element. So:

    =CORREL(OFFSET(R1C1,1,SUM(ROW(RC))-1,20,1),OFFSET(R1C1,1,8-SUM(ROW(RC)),20,1))

    (MIN or MAX would work as well as SUM)

    jay freedman say:

    When you just scan a form or document, the result is a  picture  of the original, but not  something you can edit in Word.

    There are two things you can do.

    The easier one is to insert a text box that sits in front of the picture of the form, and type into the text box. You can right-click the edge of the
    text box, choose Format Text Box, and set it to No Line so the box isn't obvious.

    The harder one is to use Optical Character Recognition (OCR) software to convert the picture into an actual Word document, which you can then edit.
    The problem is that a lot of OCR programs make mistakes, especially if the text in the original form is small, smudged, or otherwise hard to read.
    Then you have to spend time correcting the mistakes, as well as filling in the form.

    Actually there's a third choice: Find somebody who owns a typewriter, that most ancient and reviled of machines, and borrow it for a few minutes.

    Jay Freedman
