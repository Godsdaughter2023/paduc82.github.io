Windows installer (msiexec.exe) is the correct program to handle .msi file types. If you are getting the above said error message then either the .msi file or the Windows installer service is corrupt.

If the other files are also not opening, the probably there is something wrong with the Windows installer. I would suggest you to try these steps in this case:

 

SFC scan:

 

System File checker (SFC) scan is done to check if there are any corrupted system files that could be causing this issue. Refer to these steps to perform a SFC scan:

 

a)      Press Windows key + X, select Command prompt (Admin) to bring up elevated command prompt

b)      In the command prompt type the following command: sfc/scannow and press enter

c)       Now, restart the computer.

 

Re-register Windows installer:

 

I would suggest you to reregister the Installer service and check. Refer to these steps:

 

a)  Press Windows key, type cmd and right click the cmd option select “Command prompt (Admin)”.

b)  Typemsiexec.exe /unregister, and then press ENTER.

c)  Typemsiexec.exe /regserver, and then press ENTER.

d)  Close the command prompt and now try installing the software.

   - SSDP Discovery

   - UPnP Device Host
