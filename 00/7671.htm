Linking Tables Access 2010 Employee Database - Microsoft Community

fortydeuce say:

Linking Tables Access 2010 Employee Database



I have an employee table (SSN, name, address etc.) The SSN is used as the primary key.  I have another table that I called TimeOffT which stores each employees vacation hours, comp time, sick time, etc.  Another table is called InsuranceInfoT which stores all the information about each employees insurance info. 

My question:  I'm am trying to link all of these tables together.  Should the SSN be the primary key in each table?  How do I do this without retyping each SSN into each table?  I want to be able to retrieve what ever info I need about each employee just by using the employee's SSN without storing all information in one table.

Thanks in advance for any help.

john w. vinson say:

First off, a SSN is not really a good primary key. It's illegal for one thing (read your card, if you can find it); it's not guaranteed to be unique; and not everyone has one (nonresident aliens). I'd treat the SSN as data, which you can display on the form if you wish, but not as the PK; you could use a concealed autonumber as the PK. It's up to you but that's my advice!

 

That said - no, you would not use the same field as the PK in each table. A Primary Key is PRIMARY - it must be unique in the table. If you made thye SSN the PK in the Vacation table then you could have one and only one vacation for each employee, since the value must occur once and once only in the table - that's what "unique" means!

 

Each related table must have a "Foreign Key" field - the SSN ifi that's what you use, a Long Integer linked to the autonumber primary key if that's what you choose to do. The foreign key is not unique, so you can have multiple records with that value in the vacation table, comp table, insurance table etc. with the same value.

 

And you never need to type the foreign key into the table; in fact you shouldn't! For one thing, you should never be entering data directly into table datasheets. That's not their function: tables store data, and stay "under the hood". In a production Access app you should never even SEE a table, much less type into it!

 

Instead, you would use a Form based on your employee table, with multiple Subforms for the related tables. The Subform object has properties called the "Master Link Field" and "Child Link Field" - the Master would contain the name of the primary key field of the employee table, and the Child the name of the foreign key field. The subform will automatically synch the child record with the parent.

Enable your GPS with PowerShell! - Microsoft Community

debilo_2 say:

Enable your GPS with PowerShell!

I've made a small PowerShell Script which lets you switch GPS on/off. In contrast to the other solutions you don't need to install any software.

 

If you haven't enabled PowerShell to execute local scripts, open a PowerShell as Administrator and enter "Set-ExecutionPolicy RemoteSigned" (w/o quotes). Then make a new .txt file, name it agps.ps1 and copy/paste the following script to it (set in courier). If the GPS isn't using the standard ports on your machine, you have to change the ports in the script (COM5/COM4). Also you have to change the APN to your own (mine is internet.t-mobile). Then move agps.ps1 to System32 or some other folder that is in your environment path.

 

Param ($modus)


if (($modus -eq "on") -or ($modus -eq "off")) {

   $port = new-Object System.IO.Ports.SerialPort


   $port.PortName = "COM5"

   $port.BaudRate = 9600

   $port.Parity = "None"

   $port.DataBits = 8

   $port.StopBits = 1


   $port.Open()


   if ($modus -eq "on") {

      $port.Write("AT_OGPSP = 7,2 `r `n")

      Start-Sleep -m 500

      $port.Write("AT_OGPSCONT = 1, IP, internet.t-mobile `r `n")

      Start-Sleep -m 500

      $port.Write("AT_OGPSLS = 1, http://supl.nokia.com `r `n")

      Start-Sleep -m 500

      $port.Write("AT_OGPS = 2 `r `n")

   }


   else {

      $port.Write("AT_OGPS = 0 `r `n")

   }


   $port.Close()

}


elseif ($modus -eq "log") {

   $port = new-Object System.IO.Ports.SerialPort


   $port.PortName = "COM4"

   $port.BaudRate = 9600

   $port.Parity = "None"

   $port.DataBits = 8

   $port.StopBits = 1


   $port.Open()

   $port.ReadExisting()

   $port.Close()

}


else {

   $error = "Invalid parameter. Valid parameters are `"on`", `"off`" and `"log`"."

   Write-Error $error

   return

}

 

 

Now you are ready to enable GPS with it. Open PowerShell and enter...

 

"agps on" to start A-GPS tracking

"agps off" to stop it

"agps log" to have a short look at whats going on on COM4, i.e. if it works.

PARAMETERS [Current Year:] LONG, [Number of Years in Period:] LONG,
[Species Id:] LONG;
SELECT
  y.RAOU,
  y.Period,
  ([Current Year:] - (y.period+1) *[Number of Years in Period:] + 1) AS
  startYear,
  ([Current Year:] - (y.period * [Number of Years in Period:])) AS endYear,
  Y.Month,
  y.A,
  y.R
FROM
  (
    SELECT
      INT([Species Id:])                             AS RAOU,
      nz(num.NumBirds, 0)    / denom.numWeeks        AS A,
      (nz(num.numRecords, 0) * 100) / denom.numWeeks AS R,
      denom.period,
      denom.Month
    FROM
      (
        SELECT

              COUNT(*)         AS numRecords,
              d.observation_year,
              INT(([Current Year:] - d.observation_year) /
              [Number of Years in Period:]) AS period,
              d.week_number
            FROM
              tblData AS d
            WHERE
              d.RAOU_Number = [Species Id:]
            GROUP BY
              d.RAOU_Number,
              d.observation_year,
              d.week_number
          )              AS x,
          tblWeekToMonth AS wm
        WHERE
          x.week_number = wm.GBSWeek
        GROUP BY
          x.period,
          wm.Month,
          x.RAOU_Number
      ) AS Num
    RIGHT JOIN
      (
        SELECT
          y.Month,
          y.period,
          COUNT(*) AS NumWeeks
        FROM
          (
            SELECT
              wm.Month,
              x.period,
              x.site_id
            FROM
              (
                SELECT
                  d.observation_year,
                  INT(([Current Year:] - d.observation_year) /
                  [Number of Years in Period:]) AS period,
                  d.week_number,
                  d.site_id

                  tblData AS d

                  d.observation_year,

          ) AS y
        GROUP BY

      ) AS Denom

    AND

      )
  )        AS y
