david a matson say: Greetings Manuylov!  Welcome to the Windows Phone forums!

I would agree with Laura that since the battery level is not mentioned anywhere in the previous conversation (or if it was, I apologize), it would be my first point to consider.

Once a mobile device falls below a 30% charge, the Bluetooth and Wireless LAN radios will turn-off to prevent the device from losing too much of a charge (the idea that the device must be capable of lasting from Friday over a long weekend until Monday or Tuesday before being recharged again due to the owner forgetting to recharge it on Friday when getting home set the stage for the 30% requirement). 

Here are some additional steps I would recommend if a complete recharge of the battery to 100% doesn't fix the problem:
1.) I also didn't see anything in the previous posts about whether a soft-reset was performed.  If you have never done this before on this device, you really should do it now to make sure it's not a simple fix.
2.) If that fails to resolve the problem, then I would remove the battery from the device and wait 10 or 15 minutes to ensure that all radio components in the device have lost any residual electrical charge and are completely neutral.  Then I would place the battery back in the device and restart it.  This is a more definitive way to ensure that all components are really reset properly (barring a "Hard" reset, which returns the device to it's factory state).
3.) If the above two fail, then a Hard Reset is the last suggestion I have to ensure the problem is not a problem with the device itself.   Do not load ANY software onto your device after the Hard Reset until after you have confirmed that the problem is no longer apparent.   If it works fine after a Hard Reset, then some software you are loading is likely causing the problem.
4.) If nothing works to resolve the problem, you could very well have a defect device, in which case you would be best to return it to the original point of sale (and be sure to request a "new" replacement device since you only just got this one) for an exchange.

I hope this helps!

ronrosenfeld say:

I won't say it is impossible without VBA, but it will be a long and tedious procedure.

I am assuming you want the results in Columns A and B, with column labels in row 1, and your data starts in C2.  In the example, I am assuming 12 columns of data -- change the 12 to whatever seems appropriate.

Enter the following formulas:

A2:

ISNUMBER(INDIRECT(ADDRESS(INT((ROWS($A$1:A1)-1)/12)+2,2+MOD(ROWS($A$1:A1)-1,12)+1)))),
INDIRECT(ADDRESS(INT((ROWS($A$1:A1)-1)/12)+2,2+MOD(ROWS($A$1:A1)-1,12)+1)),
"")

B2:

Select A2 and B2, and fill down "far enough".  Far enough = Number of columns x Number of rows of data.

This will give one entry for every cell in your matrix of dates and data

The first formula checks two conditions:  Is the cell formatted as a date?  And Is there a number in the cell  (Excel stores dates as numbers).

The second formula says -- if there is a value in Column A (not a ""), then return the adjacent cell to the one being checked by column A.

After you have done this, select the range in Columns A:B to the last row used; the use the Data Filter and deselect "Blanks".

That should give you the list you are looking for.

bill mosca say: I found what might be a solution for you here.

Here is the code:
Dim ado as New ADODB.Connection
Dim rs as New ADODB.Recordset
Dim sql as String
Dim PicPath as String

ado.ConnectionString = "Provider=Microsoft.Jet.OLEDB.4.0; etc."  ' Use yours
ado.Open

sql = "SELECT ID, MyImage FROM MyTable Where ID=1"
rs.Open sql, ado, adOpenKeyset, adLockPessimistic, adCmdText

PicPath = "C:\MyImage.jpg"

Set mStream = New ADODB.Stream
mStream.Type = adTypeBinary
mStream.Open
mStream.Write rs.Fields("MyImage").Value
mStream.SaveToFile PicPath, adSaveCreateOverWrite

rs.Close
ado.Close

Word 2007 - Corrupted TOC - Microsoft Community

phhew say:

Word 2007 - Corrupted TOC

While working on version 10 of a document the index suddendly got corrupted.  The index has entries and links to lines in the header page, stand alone single lines (e.g. Example 1:...),   single lines in paragraphs, etc.

 I have deleted the TOC and recreated it several times with the same results. How do I fix this problem?

 

suzanne s. barnhill say:
Assuming you're referring to the TOC and not the index, I would guess that you are experiencing something like the usual Document Map problem, in which paragraphs become formatted with a heading style and are thus automatically included in the TOC. If by "header page" you mean a cover page, then if you have used the Title and Subtitle styles there, they are also automatically included because the styles include Level 1 and Level 2 designations. You must use the TOC Options to remove styles you don't want included or edit the TOC field to include only those you want to include.
 
For a better understanding of how the TOC field works (including its switches), see http://word.mvps.org/FAQs/Formatting/TOCSwitches.htm and http://wordfaqs.mvps.org/TOCTips.htm
