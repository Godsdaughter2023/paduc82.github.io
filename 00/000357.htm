bob jones aka: cybertaz say:

With the release of Office 2008 Microsoft introduced a new graphics engine which has undergone further enhancements in Office 2011 & 2016. The new features are not supported by the older .doc file type which is why the SmartArt, etc. are not available.

Likewise, the older graphic objects are not editable using the new tools as long as the document is in the older file format. That doesn't make them uneditable, it just means that the document must be Converted before you will be able to do so.

In each document, go to File> Convert Document to upgrade the file type. You'll then be able to not only use the new features but also will be able to edit the existing objects.

Note that conversion may cause some minor changes to layout & text flow depending on the individual document. Most changes should be relatively simple to adjust.

As for any sort of user manual, there really isn't anything reliable. Unfortunately, Object Linking & Embedding [OLE] is not fully implemented in PowerPoint for Mac. steve rindsberg say:

The root of the problem is that both versions of PPT give us features that are inexpensive to develop because they rely on features provided by either MacOS or Windows.   Embedding and linking objects is built into Windows; the PPT devs didn't need to write the code, they just latch onto what Windows already provides and give us a way to use it.  

There IS a way to extract an embedded PDF from an existing PPTX file though.

Make a copy of the file first; work on the copy so you don't mess up your only copy of the original.

Give the copy an extension of .ZIP and open the ZIP file. 

Open the ppt folder inside the ZIP.

Open the embeddings folder within the ppt folder.

The embeddings folder will contain any embedded objects.

If there's just one embedded object in the presentation, it'll be called oleObject1.bin  

Let's assume that's our object for now.

You'll need to open the .bin file in a binary or text editor (not a word processor).  I haven't tried it, but SimpleText might be sufficient.  Start at the top of the file and delete everything up to but not including this:

%PDF

Leave that and anything following it.  Save the file as, say, oleObject1.PDF

Close the text editor and doubleclick your new PDF.  With any luck, it'll open.
