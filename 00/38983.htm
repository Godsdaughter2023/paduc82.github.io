    Language Bar disappears as you type

    OK this is a long standing issue which is not being fixed by Microsoft.

    It started after free upgrade to Win 8.1.

    Applies to installations with two or more keyboard input languages.

    The language bar suddenly disappears and you cannot switch between languages anymore.

    To reinstate the language bar and language switching you either need to logoff/log back in, or open Language settings in the Control Panel and either add/remove another language or just shift one of languages up and down couple of times.

    The problem does not occur if you have the checkbox "Let me set a different input method for each app window" cleared. Only occurs if you want to control input methods in each window separately.

    How to reproduce:

    1. Add second keyboard input language.

    2. Make sure that in the advanced settings, "Let me set a different input method for each app window" IS checked.

    3. Check using MS Word and/or another application that you can switch between input languages using Alt+Shift combination.

    4. Open Windows Explorer. Set the cursor in the "Search" field, start typing in one language, then switch to the second language, type several more character, switch back to original layout, type again... And, watch the language bar disappearing and keyboard switching disabled.

    5. Open Control Panel - Language. See that both languages are still there. Select the second input language. Press "Move Up" and then "Move Down". Watch your language bar reappear.

    6. Repeat Steps 1-5 with "Let me set a different input method for each app window" UNTICKED. Problem does not occur.

    I've seen many recipes such as fiddling with CTFMON auto-starting, etc. But CTFMON starts automatically, it is then in the process of using that it disappears.

    It was not an issue with Windows 8 before upgrading to Windows 8.1

    As a temporary solution, I unticked the checkbox "Let me set a different input method for each app window" for several months expecting the issue would be gone with updates, but it is still there.

    Detecting the type of disk

    Detecting the Type of Disk

    There is no specific function to programmatically detect the type of disk a particular file or directory is located on. There is an indirect method.

    First, call GetVolumePathName. Then, call CreateFile to open the volume using the path. Next, use IOCTL_VOLUME_GET_VOLUME_DISK_EXTENTS with the volume handle to obtain the disk number and use the disk number to construct the disk path, such as "\\?\PhysicalDriveX". Finally, use IOCTL_DISK_GET_DRIVE_LAYOUT_EX to obtain the partition list, and check the PartitionType for each entry in the partition list.
