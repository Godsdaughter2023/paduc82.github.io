If you're keeping all the data you need except for the material in the first two pages in the Excel sheet, then Mailmerge is certainly a viable option. All you probably need to do is create a Directory merge, and put the material you want in the first two pages inside a field code like this at the beginning of the mail merge main document.

{ IF { MERGESEQ } = 1 "your page 1 image and page 2 text" }

(If you have any special characters such as double quotation characters in your text you would need to deal with those specially).

However, if the idea is to add text in Word after data from each Excel row, I just don't think the Mailmerge route is particularly straightforward. In fact, I don't think having Word LINKs or DATABASE fields is a particularly easy way to go, either, although I certainly think it could be viable for a relatively small number of records in your Excel sheet. It's a .dotx because it contains some AutoText entries that you can insert using e.g. 

At1 followed by F3, At2 followed by F3 etc.

It sounds like you have Office starter and a purchased full copy of Office both installed. Correct? That is OK. You are running into a common problem.  For some reason, Windows seems to favor Starter over full editions of Office, "forgetting" to reassign the file associations to the Full version. Here are several techniques for assigning the default file associations to the "right" applications.

Try repairing your Office installation to fix your file associations:

Repair or remove Office 2010 - http://office.microsoft.com/en-us/excel-help/repair-or-remove-office-2010-HA010357402.aspx?CTT=1

 

If that doesn't work, try:

 

Re-register the application

There are several different ways you can reset the proper file associations. Word itself provides a way to do it. Open a command prompt window and, at the command line, enter the following:

winword /r

Nothing seems to happen when you press Enter, but Word will reregister itself back in Windows. In the process, it resets the file associations for the files it uses, such as DOC and DOT files. You can close the command prompt window and double-click on a document; Word should start right up.

 

If that doesn't work, try:

 

Alternate Re-register the application

Close the app. Open the Run Window: <WIN> R, use the following command (Modify the Path to match your specific installation):

"C:\Program Files\Microsoft Office\Office\winword.exe" /unregserver

The quotes are necessary, but you may need to change the path so it reflects the location of the winword.exe file on your system. (You can use Windows’ Search tool to locate the path to the winword.exe file.) The /unregserver switch “unhooks” all the references to Word in the Windows Registry.

You should then, immediately, use the following command:

"C:\Program Files\Microsoft Office\Office\winword.exe" /regserver

This command causes Word to rewrite all its Registry keys and to reassociate itself with document and template files. Now you can double-click a document file and see if Windows starts Word and opens the document.

deanh say:

The "Error Reference Source not Found" message is standard and indicates that the destination of the link has been deleted from the file or the linked file moved.

The message is appearing because in the Print Options the box for "Update Fields" is checked.

Before PDFing again, I would select the whole document (Ctrl+A) then F9 to refresh all links and references and then use Find (type in Error) to go to all the occurences of this Error message and correct them as required.

You can uncheck the box for Update fields so you don't get the error message but the problem will still be there in the PDF, that is the link will not work in the PDF as it is still broken.

Hope this makes sense :-)

All the best
