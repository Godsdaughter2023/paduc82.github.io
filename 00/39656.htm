petercon [msft] say:

I figured out some things that may be relevant to this case, so even though you have resolved your issue, I'll explain some of what may be going on in case others look here.

When a font is installed, an entry is created for it in the registry. Some font files, known as TrueType Collection files (usually with an extension .ttc), contain multiple fonts, and some can contain a large number of fonts. I just tried installing one that has 12 fonts, and it led to symptoms similar to what you describe. The large number of fonts within the TTC file can result in a registry entry that has an overly-long name. I'm not sure why the long name creates problems, but it does.

If you have these symptoms then here's something you can investigate to see if this is your issue.

First, launch a console window. Even though you can't easily navigate the user interface because of the issue, you can launch a console from the keyboard by pressing Windows key + R, which opens the Run dialog, enter "cmd.exe" (without the quotation marks), and press enter.

When the console window opens, it may display with a small font. If you remember the layout of the console properties dialog, you may be able to figure out how to change the font size. (In the list of sizes, the larger sizes are lower in the list.) Otherwise, hopefully the text is still large enough for you to read.

In the console window, run this command --- this will retrieve some information from the registry but will not change anything:

reg query "hklm\software\Microsoft\windows nt\currentversion\fonts"

You should see a number of lines displayed; these are all registry entries associated with different installed fonts. Look for a really long line. Here's an example from the system on which I was able to reproduce this issue --- in this case, I knew what font I was looking for, and so used a slightly different command that retrieved just the one entry:

Most entries won't be anywhere near this long.

If you find an entry like that, then that could be the cause of your symptoms. A fairly quick and easy way to fix the issue is to delete that registry entry. You probably won't be able to do this in the registry editor (regedit.exe) since it will have problems with the overly-long entry. Instead, it can be done from within an elevated console window using the reg command.

I'm not going to give step-by-step details here because there's a big risk: if you enter the command wrong, you could end up uninstalling all your fonts rather than just the one! That would leave your system in a state that would be harder to recover from than the current state. If you're comfortable doing registry edits from the console, then you can probably figure out how to delete the bad entry.

We'll be investigating why the overly-long entries create problems with GDI.

I hope this information is helpful.

andre da costa say:

If you are referring to an entirely new OEM System Builder license, not previously in use, yes, you could use it as long as it meets the minimum requirements. But keep in mind the restrictions with OEM licenses.  

OEM versions of Windows are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system

What happens if I change my motherboard?

As it pertains to the OEM license this will invalidate the Windows 10 upgrade license because it will no longer have a previous base qualifying license which is required for the free upgrade. You will then have to purchase a full retail Windows 10 license.
