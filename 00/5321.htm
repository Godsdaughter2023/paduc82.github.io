If they are really dates, they should be sortable no matter what the format.

hansv mvp say:

A Date variable doesn't have a format. It is stored as a number, for example today (17-Jan-2013) is stored as 41291.

A format such as dd-mmm-yyy or dd//mm/yyyy is merely a display property, it doesn't change the underlying value.

So the line

 

SearchDate = Format(SearchDate, "DD-MMM-YY")

 

doesn't affect the value of SearchDate in any way.

 

Searching for dates in an Excel sheet using VBA is problematic. I usually resort to looping through the cells in the range I want to search, and comparing each cell's value to the "target" date in the loop.

There is no rule in relational database design that specifies the order of fields.

But certainly, if I was to assist you with a project where you had created the database and (to take this to extremes) randomized the order of the fields in each table, I would decline the privilege. The convention is to put fields in a "natural" order, and the PK would be at the top.
