tigadr2015 say:

creating date ranges for a report

In my report, which is created off a form with a subform, I need to have the information from certain date ranges that come from a field in the subform.  How do I create the date range event procedure?

tom van stiphout say:

> date range event procedure?

I'm assuming what you mean is that you have a button on your form to preview a report, and this report should only show data in a date range given by some fields in the currently selected row in the subform. The code would look like this:

DoCmd.OpenReport "myReport", acViewPreview, , "myOrderDate between #" & Me.mySubformControl.Form.myStartDate & "# and #" & Me.mySubformControl.Form.myEndDate & "#"

(of course you replace myObjectNames with yours)

ken sheridan say: Note that date literals mist be in US short date format, or an otherwise internationally unambiguous format.  For Tom's code to work on my UK system for instance would require the following amendment:

DoCmd.OpenReport "myReport", acViewPreview, , "myOrderDate BETWEEN #" & Format(Me.mySubformControl.Form.myStartDate,"yyyy-mm-dd") & "# AND #" & Format(Me.mySubformControl.Form.myEndDate,"yyyy-mm-dd") & "#"

This uses the ISO standard for date notation of YYYY-MM-DD.  Even so, it's still not completely bulletproof as any rows with date time values on the final day of the range would not be returned if the values have a non-zero time of day element, which you cannot rule out with complete confidence unless you have specifically disallowed such values in the table definition.  The following syntax will cater for this:

DoCmd.OpenReport "myReport", View:=acViewPreview, WhereCondion:="myOrderDate >= #" & Format(Me.mySubformControl.Form.myStartDate,"yyyy-mm-dd") & "# AND myOrderDate < #" & Format(Me.mySubformControl.Form.myEndDate,"yyyy-mm-dd") & "# + 1"

This of course will only restrict the report's output on the basis of the dates, so if the parent form is a customers form for instance, it will not restrict the report to the current customer.  For that the string expression would need to be extended:

DoCmd.OpenReport "myReport", View:=acViewPreview, WhereCondition:="customerID = " & Me.customerID & " AND  myOrderDate >= #" & Format(Me.mySubformControl.Form.myStartDate,"yyyy-mm-dd") & "# AND myOrderDate < #" & Format(Me.mySubformControl.Form.myEndDate,"yyyy-mm-dd") & "# + 1"

However, this assumes that the report is based on a single query which returns both the customer and order data, and is grouped accordingly.  If the report mirrored the form/subform and included an orders subreport, it would not work.  In a situation like this I would recommend redesigning the report to exclude the subform.  If a subreport is used it would be simpler to restrict this to the date range in question by means of parameters in its RecordSource query which reference the subform's controls.  The parent report would then simply be restricted to the customerID:

DoCmd.OpenReport "myReport", View:=acViewPreview, WhereCondtion:="customerID = " & Me.customerID

We are only using customers and orders as an example here.  You'll need to translate this to whatever are the equivalents in your case.

andre for directly say:
By default the NAP service is stopped. To check the service status, follow these steps:

Press Windows key + R.
Type services.msc and hit enter.
Search for the Network Access Protection service from the list.
Right click on the service and select properties.
Check the service status.

For more information, refer to the link.

http://msdn.microsoft.com/en-us/library/windows...(v=vs.85).aspx
