graham mayor say:

It is fairly simple to do (less simple to explain) using a macro and userform.
Add a reference to the Outlook object library in Excel VBA Tools > references
Create a simple user form containing a list box and a command button (I have left the default names for all these items, but you can change them if you wish).
Associate the following macro code with the user form Then create a macro to call the userform containing just the line -     UserForm1.Show
Add that macro to a button on the QAT or when you feel more adventurous on the ribbon.

Option Explicit
Private strAddr As String
Private i As Long
Private bStarted As Boolean
Private oOutlookApp As Object
Private oItem As Object

Private Sub CommandButton1_Click()
Me.Hide
With Me.ListBox1
    For i = 0 To .ListCount - 1
        If .Selected(i) Then
            strAddr = .List(i)
            Exit For
        End If
    Next i
End With
On Error Resume Next
Set oOutlookApp = GetObject(, "Outlook.Application")
If Err <> 0 Then
    'Outlook wasn't running, start it from code
    Set oOutlookApp = CreateObject("Outlook.Application")
    bStarted = True
End If
'Create a new mailitem
Set oItem = oOutlookApp.CreateItem(0)
With oItem
    .BodyFormat = olFormatHTML
    .To = strAddr
    .Subject = "Order"
    .Display
End With
If bStarted Then
    'If we started Outlook from code, then close it
    oOutlookApp.Quit
End If
'Clean up
Unload Me
Set oItem = Nothing
Set oOutlookApp = Nothing
End Sub

Private Sub UserForm_Initialize()
With Me
    .BackColor = &HC0FFFF
    .Caption = "Send order to vendor"
    .CommandButton1.Caption = "Next"
    With .ListBox1 'add your list of email addresses
        .AddItem "*** Email address is removed for privacy ***"
        .AddItem "*** Email address is removed for privacy ***"
        'etc
    End With
End With
End Sub

graham mayor say:

Really, if you have a little vba knowledge, it is much simpler to do what I had suggested than it seems from the explanation.
With the worksheet open Press Alt+F11 to open the vba editor.
With the cursor in the workbook project - Tools > References - ensure that both Microsoft Outlook and Microsoft Forms 2 object libraries are checked.
From the Insert Menu select Userform.
With the form selected, in the properties dialog change the name to frmSendOrder
Drag the bottom handle of the userform to increase its height by about 50% (not critical)
From the controls toolbox drag and drop a list box onto the form and drag its size handles to leave a 1/4" border at the sides and top and a 1/2" border at the bottom.
Drag a command button from the controls toolbox to the form and drop it in the space under the list box.
Right click the form and select View Code.
Select the following code from this message and paste it in place of anything that appears in the code window associated with the userform

Option Explicit
Private strAddr As String
Private i As Long
Private bStarted As Boolean
Private oOutlookApp As Object
Private oItem As Object
Private dfName As MSForms.DataObject
Private sClipBoard As String

Private Sub CommandButton1_Click()
Me.Hide
Set dfName = New MSForms.DataObject
dfName.GetFromClipboard
sClipBoard = dfName.GetText

With Me.ListBox1
    For i = 0 To .ListCount - 1
        If .Selected(i) Then
            strAddr = .List(i)
            Exit For
        End If
    Next i
End With
On Error Resume Next
Set oOutlookApp = GetObject(, "Outlook.Application")
If Err <> 0 Then
    'Outlook wasn't running, start it from code
    Set oOutlookApp = CreateObject("Outlook.Application")
    bStarted = True
End If

'Create a new mailitem
Set oItem = oOutlookApp.CreateItem(0)
With oItem
    .BodyFormat = olFormatHTML
    .To = strAddr
    .Subject = "Purchase Order " & Range("B6").Text
    .Body = sClipBoard
    .Display
End With
If bStarted Then
    'If we started Outlook from code, then close it
    oOutlookApp.Quit
End If
'Clean up
Unload Me
Set oItem = Nothing
Set oOutlookApp = Nothing
End Sub

Private Sub UserForm_Initialize()
With Me
    .BackColor = &HC0FFFF
    .Caption = "Send order to vendor"
    .CommandButton1.Caption = "Next"
    With .ListBox1 'add your list of email addresses
        .AddItem "*** Email address is removed for privacy ***"
        .AddItem "*** Email address is removed for privacy ***"
        'etc
    End With
End With
End Sub

Towards the end of the code you will see the line
         .AddItem "*** Email address is removed for privacy ***"
        .AddItem "*** Email address is removed for privacy ***"
        'etc
Create a similar .AddItem line for each e-mail address that you wish to access.
(obviously you can remove the dummy lines)
From the project window, select ThisWorkbook
In the vba window enter

Option Explicit
Sub eMailOrder()
    frmSendOrder.Show
End Sub

Save the project
Close the vba editor and add a button to the ribbon to call eMailOrder.
Job done!

Ignoring the how's & why's of how a PST file on OneDrive (SkyDrive) came into play, it's really important to understand what the <default> data file is used for which is to store your default contact/calendar folders for the profile and applicable to any and all email accounts that may be configured 

Based on the description, sounds like you would be much better off by creating a new PST file and making that the default for the profile instead of using the one named "outlook_backup.pst" 

Steps to accomplish that from within OL'2010 - on the <Home> ribbon tab for any item type

--> Click <New Items> --> <More Items> --> <Outlook Data File>

--> Pick the location and name for the pst file you want to create

--> go to File --> Account Settings --> Data File tab

--> set the newly created pst file as the default for the profile

--> if you have contact/calendar items that exist in other folders, you may want to copy them to the contact/calendar folders in the new PST file

--> this new PST file will have a full set of folders once designated as the default (i.e. <Inbox>, <Outbox> etc) - ignore them, you can't delete those from within the Outlook UI.

Now - something else that doesn't make any sense in reference to your description of existing data files. You stated that there were only 2 data files listed (1) to Skydrive and (2) the other called <outlook_backup.pst). 

If you are using an IMAP account - then there should have been at least 2 PST files for the account. The first for the default data file and the second for the one associated with the IMAP email folders. You could not have set the latter as the default data file for the profile if the it's an IMAP account which points to this involving a POP3 account. If that's the case, the data file being used as the delivery location for the POP account should be set as the default for the profile. How or why someone would have named that PST file "Outlook_backup" indicates that someone somewhere along the line went through some misguided steps in configuring your Outlook config (as in thinking that having the default data file on Skydrive would enable multiple devices to share contact/calendar data).

If you're not sure about your config

go to File --> Account Settings --> Email tab

- that lists the email accounts configured in your profile along with the email account type (POP, IMAP etc)

- for any POP account --> highlight it --> go down to the <change folder> button and beside it, you'll see which data file is being used as the delivery location for that specific account. This PST file may or may not be designated as the default data file for the profile but from the sounds of things - your POP3 account is set to use the <outlook_backup.pst> file - a name which could only have been manually and explcitly set.
