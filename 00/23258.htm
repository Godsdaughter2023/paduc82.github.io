ronrosenfeld say:

It is pretty simple to do using VBA, much more complicated using worksheet functions.

For example, if your list of names to be processed is in column A, and you want the results in Columns B & C, then this Macro will do the job, depending on the possible variations in your names.

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.

It is ".pattern" that does the work of separating the initial capitalized words from the rest of the string.

========================================
Option Explicit
Sub SplitUpperName()
    Dim re As Object, mc As Object
    Dim r As Range, c As Range
    Dim s As String
Set r = Range("A1", Cells(Rows.Count, "A").End(xlUp))
Set re = CreateObject("vbscript.regexp")
With re
    .Global = True
    .ignorecase = False
    .MultiLine = True
    .Pattern = "^\s*([A-Z\W]+\b)\W+([\w\s]+)"
End With

For Each c In r
    s = c.Text
    If re.test(s) = True Then
        Set mc = re.Execute(s)
            c(1, 2) = mc(0).submatches(0)
            c(1, 3) = mc(0).submatches(1)
    Else
        Range(c(1, 2), c(1, 3)).ClearContents
    End If
Next c
   
Range(r(1, 2), r(1, 3)).EntireColumn.AutoFit
End Sub
========================================

    Custom tabs are missing in Excel 2010 when changing User - Microsoft Community

    lorenzobarone say:

    Custom tabs are missing in Excel 2010 when changing User

    I have a simple excel spread sheet. 

     

    I have setup one custom tab (under Main Tabs), with one group and two items with icons.

    All works well and as expected.

     

    When another user signs on to my computer the custom tab is not there.

    What has happened?

    teylyn say:
    Hello,

    customisations to the ribbon are saved with your user profile. When another user logs in they do not access your profile, but their own profile.

    What you can do:
    Click File > Options > Customize Ribbon
    Click the Import/Export Button > Export all customizations
    Save the file in a location where the other user can access it.
    Let the other user log in, then open the Customize Ribbon dialog and import the customizations from the saved file.

    Now you will both see the same custom tabs. If either of you make changes that the other one should see, you will need to do the Export/Import routine again.

    An alternative to this is to build an Add-in that is applied at the application level instead of the user profile, but this requires some deeper knowledge of VBA.
