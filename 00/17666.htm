If the problem goes away when you start in safe mode, then it is definitely an addin problem.

You will have to turn off the addins to find the one that is causing the problem.  Best way is to turn off  half addins, then test to see if problem is fixed.  If not, turn them back on and turn off the other half and test. Continue until you narrow down to one specific addon.

Trouble Shoot Word problems that occur when you start or use Word 2010, 2007, 2003, or 2002

http://support.microsoft.com/kb/921541  2003-2010

 

http://support.microsoft.com/?kbid=820919 -2002-2003

 

NOTE: part of this procedure is disabling addins. If you want to take a shortcut, check to see if you have a BlueTooth device driver or addin, ie “send to Bluetooth”, installed. Disabling BlueTooth is rapidly moving to the top of my Office trouble shooting process. Blue Tooth drivers have been identified as the cause of problems with many different “random” symptoms. If you have BlueTooth, disable it in control panel, device manager and possibly as an addin in Office apps.  It’s not that I think there is anything inherently bad with Blue Tooth. The problem is with the people writing the drivers.

 

Don’t Assume ANYTHING!

Don’t assume some part of the trouble shooting processes listed below doesn’t apply to you. Often there are things going on in the background that you just aren’t aware of. For example, when installing software it is all too easy to have an unwanted app installed if you don’t carefully read every screen.

 

Uninstall Recent Software

At this point, you have also think about any software or plugins that you may have installed on your computer. Annoyingly, this problem with Word can be caused by a program that is completely unrelated to Word!

Did you recently install some new hardware? A printer, scanner, label machine, web cam, tablet, etc? Have you installed any software? If so, uninstall it and see if the problem goes away.  The following Skype links are an example of this issue

shyam pillai say:

Hi,

You have set the text value on the shape which is on the custom layout and not on the slide itself. The text on the slide is more of a visual cue. In your case, you need to check which layout is being used by the slide and then iterate thru the placeholder collection on the layout for the text match, the same index will map onto the slide placeholder assuming that you haven't deleted any of them on the slide. This example should prove useful.

Sub Usage()
Dim I As Long
Dim oSld As Slide
Dim oShp As Shape
Set oSld = ActivePresentation.Slides(1)
'Get index of the placeholder on the custom layout
I = GetPHIndexOnLayout(oSld, "HymnTitle")
If I > 0 Then

   'Reference the placeholder on the slide using the same index
    Set oShp = oSld.Shapes.Placeholders(I)
    ' with the shape reference obtained set the value on the slide
    oShp.TextFrame2.TextRange.Text = "Database value"
End If
End Sub

Function GetPHIndexOnLayout(oSld As Slide, FindText As String) As Long
Dim I As Integer
For I = 1 To oSld.CustomLayout.Shapes.Placeholders.Count
    With oSld.CustomLayout.Shapes.Placeholders(I)
        If .HasTextFrame Then
            If LCase(.TextFrame2.TextRange.Text) = LCase(FindText) Then
                GetPHIndexOnLayout = I
                Exit Function
            End If
        End If
    End With
Next
End Function

Hi Harishiyer,

Actually you need to convert your security groups to mail enabled distribution groups in order for Office 365 to migrate them. You can do this with the following cmdlet: Enable-DistributionGroup -Identity "name of group goes here"

Important: When you assign mailboxes with (distribution) groups, the automapping functionality doesn't work anymore. This means that a user needs to add mailboxes himself. I recently created a case at Microsoft because one of our customers had everything set up with groups and after migration the mailboxes were gone for every user. If you want your users to be able to see the mailboxes after migration you need to work with individual user permissions on your shared mailboxes.
