Unable to load odbcji32.dll - Conflict with Office 365 and Microsoft - Microsoft Community

j coughlin say:

Unable to load odbcji32.dll - Conflict with Office 365 and Microsoft Access Runtime - Solved

My configuration is:

  • 2016 Office 365 32 bit

  • 2013 Microsoft Access Runtime 32 bit

First install 2016 Office 365 32 bit
Second install Microsoft Access Runtime

After doing this the C:\WINDOWS\SysWOW64\odbcad32.exe will be able to find odbcji32.dll.

The Conflict:

After installing the Microsoft Access Runtime, the default program associated with all Microsoft Access file extensions is the Microsoft Access Runtime. So when you double click a file from the Windows Explorer, the Runtime version of Access will fire up and preclude you from accessing any of the development tools associated with the 2016 Microsoft development version. You can always first fire up Microsoft Access from the Start Menu to get around this.

It appears that sometimes when updates are downloaded and applied to Microsoft Office 365, you might encounter the missing or unable to load odbcji32.dll issue again.

Anytime this should occur, you can reset the default application to the development version of Microsoft Access associated with the file extensions so that you can invoke Access from the Windows Explorer:

  1. Via the Control Panel\Programs\Programs and Features:

    1. Locate Microsoft Access Runtime  in the list

    2. Left-mouse-click it.

    3. Left-mouse-click Change in the column heading

    4. When presented with a dialog, choose the Repair option.

  2. Via Control Panel\Programs\Default Programs\Set Default Programs

  3. In the left hand column, select Access 2016 (this is the development version, whereas Access (desktop) is the Runtime version).

  4. Left-mouse-click “Set this program as default”

After doing this, .MDB extensions will still be invoked by the Microsoft Access Runtime, whereas the .ACCDB extensions will be invoked by Access 2016 (Office 365).
If you run the Control Panel\Programs\Programs and Features and choose Microsoft Office 365 and then choose Change->Repair, all file extensions associated with Microsoft Access will point to the Office 365 development version not the Microsoft Access Runtime. However, C:\WINDOWS\SysWOW64\odbcad32.exe will no longer be able to locate odbcji32.dll.
What a paradox!

Note:

It is also required to have the Microsoft Access Runtime installed with Office 365 when you want to run programs or scripts such as Windows Scripting Host (WSH, files with extension wsf). The reason for this is that the drivers for Office that are installed for Offices 365 are only in a location known by Office 365 applications and none others.
