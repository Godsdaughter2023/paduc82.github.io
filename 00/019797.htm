ken sheridan say: You'll find a simple example of the use of correlated subforms, using data from Northwind, as CorrelatedSubs.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

From your description the example in my demo for nested rather than correlated subforms seems to be similar to what you are doing in that it navigates to a record by means of a combo box in the first subform.  The second subform is nested in the first, and linked to it in the usual way by means of the LinkMasterFields and LinkChildFields properties, so no further code is required.

The other example, for correlated subforms, restricts the rows  returned in the second by means of a parameter in its query which references the key of the first subform's recordset, which seems to be what you are doing.  In this one the first subform is in continuous forms view however, so the user navigates to a row (the order date) simply by selecting a row in the subform rather than via an unbound combo box.  In this case the second subform is requeried in the first subform's Current event procedure.  If you are navigating to a row in the first subform's recordset via the unbound combo box, then this would be the event procedure to use in your case also.
