Here is some advice about using Windows Update:

 

You do not have to install a problem update that will continue to cause problems for you.

There is a way to get around that.

Sometimes Microsoft gives you Updates for 3rd Party Hardware: Graphics, Network Adaptors, etc.

You should only obtain the latest Drivers for them from the actual Hardware Manufacturer's website, as Microsoft normally does nor have those latest Drivers available as Microsoft relies of the 3rd Party Hardware Manufactuer to supply them.

But unfortunately, the only way to find out which is the problem update is to install them one at a time, until you find the "bad" one for your setup.

 

You can do a System Restore to go back to before it was installed (which is NOT the best way to remove an Update) or follow this method:

"Remove an update"

http://windows.microsoft.com/en-us/windows-vista/Remove-an-update

 

When you have found the problem Update from the list of updates that is presented > you then right click on the problem update > select the Hide Update Option > and it will not be offered to you again

"How to Hide or Unhide a Windows Update in Vista"

http://www.vistax64.com/tutorials/72491-windows-update.html

 

Here is how to chanage update settings, if you find it necessary to:

"Understanding Windows Update and Extras in Windows Vista and Windows 7"

http://www.bleepingcomputer.com/tutorials/windows-vista-updates-and-extras/

 

Either of these two update Settings will give you the chance to see what you want to install through Windows Updates;

Download updates but let me choose whether to install them - If you select this option, Windows will download updates to your computer, but not install them automatically. If you want to install the updates then you must install them manually. You should only select this option if you have a reason for not installing updates automatically. Only advanced users should use this option.

Check for updates but let me choose whether to download and install them - If you select this option, you will be alerted when there are new updates available to download and install. You can then choose to download and install the updates that you wish. This option should really only be used by people who know exactly what updates they need or those who have limited access to the Internet.

 

But remember you normally only have problems with 3rd Party Hardware Drivers. The other Updates are to enhance your System's Performance and Security

Manually Clear Jump Lists

All recent items in any Jump List are stored by Windows in a hidden location, which you can get to in Windows 7 or higher by browsing to the following folder location in Explorer:

%APPDATA%\Microsoft\Windows\Recent\AutomaticDestinations

Copy and paste that into Windows Explorer and press enter. Now you will see a list of files with very long and convoluted names. This is because all the Jump Lists are encoded. Each file represents a list of recent items for a particular programs’ Jump List.

It’s impossible to tell which entries go with which Jump Lists unless you open the file in a text editor and browse through it. However, since all of the files are just recent items in a Jump List, you can delete all the files and it will clear all recent items on all Jump Lists.

    cmdrkeene say:

    There's no log.

    As a design principle, Windows will never use a balloon to provide critical information.  If something is important (something that requires your action, consent, or might cause an interruption in your work or loss of data), the balloon will not be used.  Think of balloons like billboards: nice info, but never critical or important.

    If something was important or critical, it will be persistent and the icon near the clock won't go away until you specifically dismiss it yourself.  Example: a failed backup or virus scan alert.

    Further reading

    Note:  according to the Windows Development Guidelines, the balloon tip notifications are designed to be ignored.  Literally.  That is, if you completely miss it and never see it, it should not affect you.  They're more like whispers or hints.  According to Microsoft, "Notifications are for non-critical information that users can freely ignore. If users really must perform an action, don't use notifications balloons."
    The display and repeat of the balloon notifications are covered by these rules:
    • When to Show Balloon
    • How often to Repeat Balloon


    Given this, you should feel more confident that, if you miss a balloon, it wasn't important anyway.  If it was important, you will either be notified again ("How often to repeat"), or will receive progressive action (a dialog box instead).

    Still, you might wish them to show longer.

    peted66616 say:

    My Movie Maker crashes when I click full-screen preview. Otherwise, Movie Maker behaves well for what it can do.  I only have .jpgs and mp3s in the movie file.

    This may not apply, but I thought I'd mention it just in case:

    Ever since I started using Movie Maker way back when (ten years ago?), I have found it to be unreliable on multi-core systems. Even on an old P4 with hyperthreading, that was enough to trigger some threading bug in the program.

    A work-around that I found works very well is to set the CPU affinity to a single core for the program. There's no convenient way to do this as a permanent setting (there are inconvenient ways :) ), but you can do it each time you run Movie Maker:

    1. start Task Manager (e.g. Ctrl-Shift-Esc)
    2. right-click Movie Maker and choose "Go to details"
    3. In the Details tab, right-click Movie Maker again and choose "Set affinity"
    4. Uncheck all but one core (doesn't matter which core you keep)
    5. Click "OK"

    This will force Movie Maker to run on only a single CPU, which helps avoid the worst of the thread synchronization bugs in a program (any program). Obviously this will reduce performance for setting operations (unfortunately, one important one is transcoding/exporting video). But slow Movie Maker is better than crashing Movie Maker.

    I am just revisiting Movie Maker after having not used it for some time, and I have found it still crashes and that the above work-around helps the situation, at least for me. Maybe it will help for you and/or other people too.
