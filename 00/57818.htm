a. user say:

XP will automatically check all volumes (hard disks) on reboot to see if there is a chkdsk scheduled to run (by you) or if the volume has been marked "dirty".

A volume can be marked dirty if there have been such things as a power failure, aborted restart or ungraceful shutdown.  Has anything like that happened?

You could tell XP just to never check the drive for errors, but that is probably not a good idea.

The only way to clear the dirty bit on a volume is to run a chkdsk with error correction.  If the chkdsk that runs automatically on a reboot does not clear the dirty bit, you should run a chkdsk with error correction (chkdsk /r ) from the XP Recovery Console and that will usually straighten it out.

Why don't you query the dirty bit on your external HDD and see if it is dirty or not dirty.

Here's an example of how to do that:

You can query the dirty bit on a volume from the command prompt window.

Click Start, Run and in the box enter:

cmd

Click OK to open the command prompt window.

To query the dirty bit on drive E, type:

fsutil dirty query E:

Sample output:
Volume E: is dirty
Volume E: is not dirty

Is your external HDD dirty or is it not dirty?

Type 'exit' to close the command prompt window.

If you want to run the chkdsk /r from the XP Recovery Console, you either need to already have the RC installed as a boot option (most people don't), have a genuine bootable XP installation CD that matches your unspecified installed Service Pack (if SP3 is installed, most people don't), or you can create a bootable XP Recovery Console CD (no XP media required) and run the chkdsk /r from there.
