hal hostetler - mvp-outlook say:
Don't save them to disk folders, save them to .PST files instead.  Each quarter, create a new .PST file and move that quarter's email into it. Close Outlook and save these .PST files to DVD.  Note that DVDs and CDs present a special circumstance in that both are Read Only media.  Saving a .PST file to either sets the Read Only attribute which must be removed before Outlook can open the file again - if you need something from one of these files, copy it from CD/DVD to your hard drive, reset the Read Only attribute in the file's properties, then open it with Outlook using "File > Open and Export > Open Outlook Data File".

jay freedman say:
If you're inserting both section breaks and page breaks, don't do that. Insert only "Next Page Section Breaks" (from the Breaks button on the Page Layout ribbon), one before the chart and one after it. When you click the ¶ button to turn on nonprinting characters, each section break should appear as a double dotted line with "Section Break (Next Page)" inserted in the middle of it.

With the cursor anywhere between the two section breaks, click the Columns button and choose One. That should change only the single section between the section breaks.

If that still changes the whole document, there's another way to try. Instead of clicking One, click the More Columns item at the bottom of the dropdown to get a dialog. There, choose the number of columns you want and make sure the Apply To box is set to "This section".

Further to Polaris380's original post, this appears to be a feature, however, I cannot find any documentation on the Microsoft website advertising it as such. This appears to happen with Outlook 2007 and 2010. Please try the below steps to replicate this (steps are for Outlook 2007):

1. Create a new message.

2. In the To field, type *** Email address is removed for privacy *** (or any fake email address you know you have not addressed a message to in the past and will issue an undeliverable).

3. Add any text in the Subject field, and in the message, type Dear Spongebob.

4. Click Send. If you have spellcheck enabled when sending messages, cancel the spell check. Message sends (but doesn't really since it's a fake address).

5. Create a new message, click the Office Button>Editor Options>Proofing>Custom Dictionaries>Edit Word List. Notice 'Spongebob' is in the list.

6. Enable the option 'Always check spelling before sending' (Tools>Options>Spelling tab).

7. Repeat steps 1-3, and you will notice that Spongebob does not appear as misspelled because it was automatically added to the custom dictionary.

I can see how this could be a feature because if a user types a name, which is usually part of the e-mail address, then it will automatically accept it as correct spelling (therefore adding it to the Custom.dic). Then, when the user addresses that same name in the main message, it will not come up as misspelled. In the reverse, if the e-mail address was mistyped, this "feature' could cause you to visit the custom.dic to remove the misspelling.

In B2 try

=COUNTIFS(J4:J6,">=0",J4:J6,"<=7")

0 and 7 both inclusive... if you want >0 and < 7 then use

=COUNTIFS(J4:J6,">0",J4:J6,"<7")

Use similar formulae in other cells.

cortig say:

To be clear though, the emails in Outlook for Mac are not stored in a .pst file. They are all in the Outlook identity.

Outlook only exports to .olm archive files which are notoriously difficult to work with.

To export the emails in a different app, you can drag and drop the messages to the desktop then import the message files in another app (tedious to say the least)., or simply use an IMAP or Exchange account configured in both apps: all messages will sync across.
