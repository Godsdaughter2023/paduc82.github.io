RAID to AHCI (use both) - Microsoft Community

afeight say:

RAID to AHCI (use both)

I am using 2 HDDs in RAID 0 to combine storage space and a small SSD which I installed Windows 7 to with this setup. Now I need to switch to AHCI but have had bad experiences in the past switching between the two (switched from AHCI to RAID and back, Windows couldn't find BOOTMGR and had to reinstall). Could someone help me in setting up my system to maintain this RAID 0 array properly while switching to AHCI? Motherboard: GIGABYTE GA-Z77X-UD5H with UEFI BIOS (very configurable)
tricky300 say:

Hi,

Having looked at the specs for your mainboard, I see that it supports RAID 0 (stripe) therefore I assume that this is how you implemented your RAID 0.

In the mainboard's BIOS you will be able to specify RAID or AHCI operation (and probably IDE too but that is irrelevant to your problem). A RAID 0 setup cannot be reverted to AHCI. This is because RAID 0 splits data across your two disk drives at the file level.

This means that any given file will part reside on one disk and also part reside on the other. This allows (in theory) faster disk access because the two disk drives can be read simultaneously to read one file. In practice this saving is rarely, if ever, measurable by human perception. The saving can only really be measured by a benchmark that involves repeating the same disk I/O task a set number of times (probably at least into the 100,000s). Breaking the RAID 0 by reverting to AHCI breaks all files and therefore renders either disk unbootable.

Not that it is pertinent in your case, but just for the sake of completeness, breaking a RAID 1 by reverting to AHCI would not normally cause a boot failure on either drive because in RAID 1, files are replicated on each disk rather than spread across each disk.

In short, you cannot have RAID 0 (or any other RAID for that matter) and AHCI mode at the same time, as far as the motherboard settings are concerned AHCI is a non-RAID mode of disk operation.
