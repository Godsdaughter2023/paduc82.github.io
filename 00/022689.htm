rich michaels - mvp (greatcirclelearning.com) say:

There are 3 major problems with this code, in order for it to run on a Mac, particularly MacExcel 2016 and a few minor ones that may or not cause problems later. But for now these are the major ones...

pathh = "Macintosh HD:Users:GiovanniVitale:Dropbox:ASD Docs:Temps:TEMP ExitADOS.doc"

The path separators in MacExcel 2016 are forward slashes "/" and not colons ":", which are the separators for MacOffice 2011.

Set WA = CreateObject("Word.Application")

This is the ActiveX control that does not work on either MacOffice 2011 or MacOffice 2016. The new command should be: Set WA = New Word.Application

However, with that said, the command when run in Excel does open a Word document but then produces an "Out of Memory" error and fails to set the object variable, in this case "WA" and without the object variable set, nothing else in your code will work.

Try it on your system, and if you get the same error as I have, file a bug report by using the Smiley control on Excel's ribbon.

Script error messages are displayed by Internet Explorer when there is a problem with the JavaScript or VBScript code on the website you are viewing. Occasionally a script error can be caused by an error in downloading a webpage, but more often it is an error in the webpage itself.

 

To avoid script errors in the future, you can turn off script debugging and notifications in Internet Explorer.

 

Refer the following link for more information on turning off script debugging and notifications in Internet Explorer:

 

What should you do about Internet Explorer script errors?

Note: The information provided in the above article applies to Windows Internet Explorer 7 and Windows Internet Explorer 8.
