talking moose say:
A message can only reside in one location not two. You need to copy your message to the first location and move the message to the second location.

Go to Tools menu --> Rules... and create two rules.

The first rule should filter for the first address or filter for messages that contain both addresses and then Copy the message to your first Inbox. Be sure the option at the bottom Do not apply other rules to messages that meet these conditions is UNchecked.

The second rule should filter for the second address or filter for messages that contain both addresses and then Move the message to your second Inbox.

Be sure your second rule is below the first in your rules list. Rules are executed from top to bottom. You can drag rules in the list to rearrange them.

Hope this helps!

ken sheridan say: Rather than trying to compare values in the report I would suggest computing the values in a query which can be used as the subreport's RecordSource property.  Without knowing the details of the base tables from which the data is drawn I can't give you the query of course, but here's an example using Northwind (with the order dates adjusted to cover 2017/2018):

SELECT FORMAT([Order Date] ,"mmmm yyyy") AS OrderMonth,
SUM([Unit Price]*Quantity) AS Total,
     IIF(NZ((SELECT SUM([Unit Price]*Quantity)
                 FROM Orders AS O2 INNER JOIN [Order Details] As OD2
                 ON O2.[Order ID] = OD2.[Order ID]
                 WHERE YEAR(O2.[Order Date]) =  2017
                 AND MONTH(O2.[Order Date]) =  MONTH(O1.[Order Date])),0)
    >SUM([Unit Price]*Quantity),"Down","Up") AS Movement
FROM Orders AS O1 INNER JOIN [Order Details] As OD1
ON O1.[Order ID] = OD1.[Order ID]
WHERE YEAR([Order Date]) = 2018
GROUP BY YEAR([Order Date]), MONTH([Order Date]),
FORMAT([Order Date] ,"mmmm yyyy");

It should not be too difficult to compute the movement of the yearly totals directly in the report, or you can use an amended version of the above, grouping by year only:

SELECT YEAR([Order Date]) AS OrderYear,
SUM([Unit Price]*Quantity) AS Total,
     IIF(NZ((SELECT SUM([Unit Price]*Quantity)
                 FROM Orders AS O2 INNER JOIN [Order Details] As OD2
                 ON O2.[Order ID] = OD2.[Order ID]
                 WHERE YEAR(O2.[Order Date]) =  2017),0)
    >SUM([Unit Price]*Quantity),"Down","Up") AS Movement
FROM Orders AS O1 INNER JOIN [Order Details] As OD1
ON O1.[Order ID] = OD1.[Order ID]
WHERE YEAR([Order Date]) = 2018
GROUP BY YEAR([Order Date]);

Neither of the above allows for exact ties of course, which would be returned as 'Up', but that's a remote possibility.

That said, I finally figured out a solution. **** yeah! I'm so happy, because I've been banging my head against the wall for so long on this problem.

Alright, here it is:

First off, use this gem of a UDF called execShell() by Robert Knight.

Using that function, you feed in a shell script:

rngSomeRange.copy

scriptToRun = "export LC_ALL=en_US.UTF-8 && " & _
        "touch " & sOutFolder & "/" & sOutFile & ".tex && " & _
        "pbpaste > " & sOutFolder & "/" & sOutFile & ".tex"

Then you execute:

    execShell scriptToRun, exitCode

That first command, "export LC_ALL=en_US.UTF-8" is the really important bit. It sets the locale of the shell session and pretty much guarantees that the file will be encoded as UTF-8...or so it seems. At any rate, it ****ing works!!!!!

hansv mvp say:

Open the form in design view.

Select CodeTemp.

Activate the Event tab of the Property Sheet.

Click in the On Change event.

Select [Event Procedure] from the dropdown list.

Click the builder dots ... to the right of the dropdown arrow.

Make the code lool like this:

 

Private Sub CodeTemp_Change()
    Dim strFilter As String
    On Error GoTo ErrHandler
    If Me.CodeTemp.Text <> "" Then
        strFilter = "[Code] Like '*" & Me.CodeTemp.Text & "*'"
        Me.Filter = strFilter
        Me.FilterOn = True
    Else
        Me.Filter = ""
        Me.FilterOn = False
    End If
    With Me.CodeTemp
        .SetFocus
        .SelStart = Len(Me.CodeTemp.Text)
    End With
    Exit Sub
ErrHandler:
    MsgBox Err.Description, vbExclamation
End Sub
