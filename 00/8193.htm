I had this horribly annoying problem for months.  It happened every time I modified the value of any cell.  I spent many hours searching for a solution, and while some of the solutions presented fixed the problem for some people, it did not for me and many others.

Some of the other solutions that didn't work for me:

- Clear the temp folder after a fresh boot.

- Delete items from the Office clipboard.

- Repair Office.

- Close or change options of other programs (such as TeamViewer) that use the clipboard.

The solution that worked for me:

- Delete the offending object.

     - Home Ribbon > File & Select > Selection Pane.

        Clicking on an object shown in the Selection Pane will select the object on the sheet.

        Yay, problem solved and a sigh of relief released.

kosh vorlon say:

Hi Leeeok,

 

Please see the following thread: http://www.sevenforums.com/system-security/76867-stdrt-exe-running-windows-temp.html.  All the parts you need are there, but not in a rational order.

 

Let me try to condense them (credit to authors from the above thread) into something that helps:

 

1.Try a clean boot (Vista and W7) http://support.microsoft.com/kb/929135 (except for MSE). If the problem goes away then it's just a matter of tracking down the culprit causing the problem. Follow the procedures in the article. Once found, delete, remove, deactivate, or uninstall it. Once done be sure to reset your system back to normal status as explained in the procedures. If the problem occurs in clean mode then just restore the system to normal status and reboot - this solution is not going to work. If using XP, seehttp://support.microsoft.com/kb/310353 andhttp://support.microsoft.com/kb/316434.

 

2. Step 1 should remove the processes which might otherwise prevent this step.  Remove/delete the following files:

C:\WINDOWS\SYSTEM\REGSRV.EXE
C:\WINDOWS\TEMP\MRT2.TMP\STDRT.EXE

 

3.  You might also want to use (and keep) TFC by Old Timer TFC - Temp File Cleaner by OldTimer - Geeks to Go Forums and save it to your desktop.
Save any unsaved work. TFC will close ALL open programs including your browser!
Double-click on TFC.exe to run it. If you are using Vista/Windows 7 right-click on the file and choose Run As Administrator.
Click the Start button to begin the cleaning process and let it run uninterrupted to completion.
Important! If TFC prompts you to reboot, please do so immediately. If not prompted, manually reboot the machine anyway to ensure a complete clean.

TFC (Temp File Cleaner) will clear out all temp folders for all user accounts (temp, IE temp, java, FF, Opera, Chrome, Safari), including Administrator, All Users, LocalService, NetworkService, and any other accounts in the user folder. It also cleans out the %systemroot%\temp folder and checks for .tmp files in the %systemdrive% root folder, %systemroot%, and the system32 folder (both 32bit and 64bit on 64bit OSs). It shows the amount removed for each location found (in bytes) and the total removed (in MB). Before running, it will stop Explorer and all other running apps. When finished, if a reboot is required the user must reboot to finish clearing any in-use temp files.

TFC only cleans temp folders. TFC will not clean URL history, prefetch, or cookies. Depending on how often someone cleans their temp folders, their system hardware, and how many accounts are present, it can take anywhere from a few seconds to a minute or more. TFC will completely clear all temp files where other temp file cleaners may fail. TFC requires a reboot immediately after running. Be sure to save any unsaved work before running TFC.

 

4. There's another file which runs a while in the background called "adbcnsl.exe" which is connected to the stdrt.exe , it cant be found from the windows file/folder exploring even with show hidden files option on.

SOLUTION: Using ms dos prompt go to windows\syswow64 folder, example;> c:\windows\syswow64 and type "dir /p" and check if the file is there listed; if it is, then type "del adbcnsl.exe".  If not found try checking c:\windows\system32 folder.  Then reboot.

 

5.  Backup the registry.  Then go to Regedit and search for stdrt and remove JUST the entries relating to that (don't wipe an entire key just to remove one entry - remove only what needs to be removed).  If uncomfortable with this, then seek professional help as noted by Steve.

 

6.  Disable system restore to clear all restore points which probably have the infection and then when done, create a new restore point to start over again and have one just in case.

That should be that.

 

I hope this helps.

 

Good luck!
