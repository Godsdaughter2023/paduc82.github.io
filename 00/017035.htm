' calling code
If fctModuleExists("YourModule") Then
DoCmd.DeleteObject acModule, "YourModule"

' in a standard module
Function fctModuleExists(strModuleName As String) As Boolean

Dim mdl As Object

For Each mdl In CurrentProject.AllModules
If mdl.Name = strModuleName Then
fctModuleExists = True
Exit For
End If
Next

End Function

bill manville say:

No.  I don't know what causes it.  Wish I did because then I could tell you!

If it doesn't happen in Safe Mode then it's not caused by an add-in or startup file

There have certainly been problems with screen updates not happening because of the hardware graphics acceleration option, but that didn't fix it either.

I assume you keep your Office software up to date using Microsoft update?

Office 2013 changed the way that Excel uses Windows - there is now one (or more) window for each open workbook where previously all workbooks were accessed via one window.  The updating of the screen can be controlled by macros using Application.ScreenUpdating = False.  Since there are now multiple windows it would seem conceivable that one workbook setting it False would cause problems for another workbook's window (though I have not seen this myself). 

When the problem is occurring it might be worth doing Alt+F11 to the Visual Basic editor, Ctrl+G to show the immediate window and typing ?Application.Screenupdating in the immediate window and pressing the Enter key.  If it comes back saying False we are on the right track and need to find out how it is being set False.
