ossiemac say:

It is most likely a problem with the number not being exactly zero but something line 0.00000000001. this is a well documented problems and is caused because of the binary method that computers use to calculate values and sometimes there is this small error. There is nothing we can do to completely eliminate such errors and we therefore need to round the number to the required number of decimal places.

Have a look at the Round, Roundup, RoundDown and MRound functions. See Help for more information on these functions.

joeu2004 say:

Neil wrote:

In the faulty worksheet in the balance column I have an opening value of $2368.50 (keyed as a value). In the next row I have a keyed value of ($2368.49)  which leaves a balance of $0.01. In the next row I have a keyed value of  ($0.01)  which leaves a balance of 0 (zero). Unfortunately that 0 shows  in that cell and the ones following.

Adding to OssieMac's explanation, if A1 has 2368.50, A2 has -2368.49, A3 has -0.01 and A4 has the formula =SUM(A1:A3), A4 would display $0.00 despite the format option.

If you temporarily format A4 as General, you would see that the actual value is about 2.18E-13, an infinitesimal number in Scientific form.

Change the formula in A4 to =ROUND(SUM(A1:A3), 2) .

cliffgu_msft5984 say:

Hi Karl,

If you mean you share a file with an Office 365 group in the OneDrive for Business library, you are able to access it via Delve and through the shared email messages. But you can't make files shared with a group in the Shared with me folder because the Shared with me folder is a shortcut for the individual user to see the shared files from other people's OneDrive for Business libraries with a private view, and when a user share files with a group, the files will be stored in the Shared With US folder in a group library with a public view. The OneDrive for Business library is like a private library and a group library is like a team site library. The two libraries are separated, so the files shared with a group will not appear in the Shared with me folder.

If you mean you share a file with a distribution group in the OneDrive for Business library, a distribution group can’t be synced to SharePoint Online. So it is not feasible for you to share a file with a distribution group in the OneDrive for Business library.

Moreover, for a security group, you can share a file with it. But the documents won't show in the "Shared with Me" page. The users can access the documents via the documents' URL. Only when sharing the documents to individual users, they will receive the emails. And the documents will also show on the "Shared With Me folder" page.

For your requirement, I suggest you submit a feedback on our UserVoice because many new features come from customers’ feedbacks.

Thanks,
Cliff

First: to keep from having to .Select anything look into using "objects" to reference worksheets, cells, and other things.  Quick Example:

 

Dim baseCell As Range ' a Range can be one or more cells

Set baseCell = Cells(Rows.Count, "B").End(xlup).Offset(1,0)

'now baseCell is same as ActiveCell in your previous code.

 

Now you know what row you are on (either ActiveCell.Row by old code, or baseCell.Row in the new manner), so 212-baseCell.Row + 1 would give you the number of rows you need, or

 

Try this code, I believe it does what you desire:

 

Sub Test()
'make sure the proper worksheet is selected
'before running this - it is going to work
'with the ActiveSheet since we have not
'specified otherwise.
 
 Dim baseCell As Range ' a Range can be one or more cells
'set baseCell to point to cell in column R
  Set baseCell = Cells(Rows.Count, "B").End(xlUp).Offset(1, 16)

 baseCell.FormulaR1C1 = 1
 baseCell.Offset(1, 0).FormulaR1C1 = 2
'redefine baseCell to refer to all rows from found row to row 212

 Set baseCell = baseCell.Resize(212 - baseCell.Row + 1, 1)
 
  baseCell.DataSeries Rowcol:=xlColumns, Type:=xlLinear, Date:=xlDay, _
   Step:=1, Trend:=False

End Sub

hansv mvp say:

Excel has a special command for that: DataSeries.

 

Sub FillTo212()
    Dim r As Long
    ' Determine last used row
    r = Range("B" & Rows.Count).End(xlUp).Row
    ' If this is below row 212, there is nothing to fill
    If r > 212 Then
        MsgBox "Nothing to do!", vbInformation
        Exit Sub
    End If
    ' Set initial value
    Range("R" & r).Value = 1
    ' Fill series
    Range("R" & r & ":R212").DataSeries RowCol:=xlColumns, _
        Type:=xlDataSeriesLinear, Step:=1, Stop:=1000
End Sub

 

The stop value is an arbitrary large number greater than or equal to 212.
