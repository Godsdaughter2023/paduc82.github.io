DataFile contains the full path to the workbook.

The string used to nominate a workbook from the Workbooks collection is just the filename.

(e.g. "MyData.xls")

 

So you could either do:
  DataFile = Mid(DataFile, InStrRev(DataFile,"\")+1)
before using Workbooks(DataFile)

or

you can use ActiveWorkbook for the workbook you just opened and ThisWorkbook for the workbook containing the code, thus:

 

    ActiveWorkbook.Worksheets(1).Select  ' this line is unnecessary
    ActiveWorkbook.Worksheets(1).Copy Before:=ThisWorkbook.Sheets("1") '
    ActiveWorkbook.Close False

 

I am assuming that you have a worksheet named "1" in your workbook.
Otherwise, if you just mean you want the copied sheet to be the first in the workbook it should be

  ... Before:=ThisWorkbook.Sheets(1)

langware say: It looks like the latest version of Java is Version 7 Update 40. If you are going to keep Java installed, then it is a very good idea (for security purposes) to install the latest updates ... which tend to come out frequently. You can either set Java to automatically install updates (When I had Java I did not select that option because frequently bad updates would be released ... and those who had auto install enabled ended up with problems) ... or you can set Java to periodically check for updates and let you know ... then you can decide if you want the update and manually install it (I used that option ... and usually waited a few days after the update was announced to see if anyone reported problems with the update).

If you remove Java, then any programs/apps that depend upon Java will not work (such as your nieces' games).

Here are a couple of references for removing Java ...

http://www.howtogeek.com/122934/java-is-insecure-and-awful-its-time-to-disable-it-and-heres-how/

http://www.java.com/en/download/help/uninstall_java.xml

Javascript is different from Java. Java is used to create create executable programs (such as your nieces' games). These Java programs are compiled prior to runtime and run on your PC. Java programs do not need a browser to run.

Javascript is a scripting language that usually runs in a browser. Browsers have options that will enable or disable the ability of Javascripts to run (in the browser). Javascripts running in the browser do not need, and do not depend upon, Java being installed in the PC. Javascripts (that get downloaded from a website and run on your PC) can also be a source of malware .... so I use Firefox as my browser, and a free Firefox add-on called NoScript that prevents Javascripts from running unless I authorize scripts for a given web-site .... it does take an extra effort (to enable scripts on a site by site basis), but it does help prevent rogue scripts from running.

Here are some links that provide further explanations of the difference between Java and Javascript  ...

http://www.java.com/en/download/faq/java_javascript.xml

http://wiki.answers.com/Q/What_are_the_differences_between_java_and_JavaScript

http://www.htmlgoodies.com/beyond/javascript/article.php/3470971/Java-vs-JavaScript.htm

I am at a loss as to why you cannot get to the site to update Java .... if it is a problem with Oracle, then hopefully by Tuesday someone will fix it.

For your future reference, there is an Office for Mac forum that will get you better answers: PowerPoint 2011 for Mac

I use a Wacom tablet myself and used to experience spurious page jumps, sometimes from ordinary hand gestures! 

If you have a tablet Prefs panel (in System Preferences), please open it. With a Wacom, I clicked on the Grip Pen (their name for a stylus) and turned the Tip Double Click Distance field all the way down to off. That solved it for me.

Of course, your mileage will vary. Your may have a different make and certainly a different model, so your control panel is likely very different. But by playing with the options you can discover settings that will tame the stylus.
