jason[msft] say:

We've root caused this and know what is happening.  Here's the workaround (while we work on a permanent fix):


  • Right click the start button
  • Select "Command Prompt (Admin)"
  • When command prompt opens, copy/paste the following:

    reg add HKLM\SYSTEM\ControlSet001\SERVICES\i8042prt\Parameters /v AsyncStart /t REG_DWORD /d 1
  • Confirm that you want to add to the registry if asked
  • Close the command prompt window
  • Reboot your PC
  • Check for updates again and you should be able to update successfully this time

What's happening?

There are two .sys files that are trying to write to the same section of registry and only one of them can win.  :)

We'll be implementing a permanent fix, but adding this regkey will bypass the issue and allow you to update as expected.

karl timmermans (outlook mvp 2012-2018) say:
>>
If I uninstall the existing version what do I need to save from outlook to retain my existing emails, contacts, calendar items, etc.
<<

Hello Dan

It is recommended to uninstall the previous version of Office prior to installing a new version.

If you uninstall using the Control Panel <Programs> option, the Outlook profiles will remain in tact and will be used by the new version of Outlook. It's the Outlook profile that contains info on your email accounts and data files so there would be nothing for you to do when opening the new version of Outlook.

<IF> on the other hand, you use the MS <Easy Fix> uninstall tool, the Outlook profiles will also get removed but removal does not include PST files that were in use. In this scenario, you would have to configure all your email accounts and for POP3 accounts you should make sure to use the existing PST file for the account instead of letting Outlook create a new one. Really would only need to use this option if there are some problems being encountered that you;re trying to resolve. Doing an "in-place" upgrade of Office doesn't fall in that category.

zenfrog say:

AVERAGEIF based on day of the week criteria

I'm trying to average sales data based on day of the week. I tried =AVERAGEIF(B13:B101,"Mon",D13:D101), where B13:B101 is Mon, Tue, Wed etc and D13:D101 is the column with sales dollars. It returns a #div/0 error. Any help is greatly appreciated.

 

Thanks

jaeson cardaño say:

Hi,

 

This one will work…

 

=SUMPRODUCT(--(WEEKDAY($B$13:$B$101,1)=2),($D$13:$D$101))/SUMPRODUCT(--(WEEKDAY($B$13:$B$101,1)=2),--($D$13:$D$101>0))

 

Getting the average for those greater than 0 in column D.

“=2” stands for Monday, just change it to other weekday.

1=Sunday

2=Monday

…

7=Saturday

 

Hope it helps,

~jaeson
