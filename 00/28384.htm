Fields are a Word feature that contain variable information within the document. For instance, today's date, page number, document name, a reference to a figure. All these can change and you'd want them to update, but this is done manually.

There are other types, like links to other files, that are controlled through Word > Preferences > General > Update automatic links at open. Start by disabling this option.

The GoBack bookmark is added whether you use the feature or not. Another thing you can try is saving those documents as .docx, where the feature stays invisible.

In the Word MVP site they describe a related problem (GoBack (Shift+F5) doesn't work in some newly-opened documents), so this may end up being an unsolvable Word bug. It could be sidestepped with a macro looping through all open documents, but it would have to be a "Save-this-document-and-close-it" kind of functionality, as leaving the document open might expose it to the unsaved state.

This erroneous indication can be very misleading ... the 100% downloaded indication shown is left over from the downloading of the Windows Defender definitions update that downloaded before the new build ... it should have changed to 0% for the build. Then, the build waits a long time prior to actually beginning to download, while Service Host: Delivery Optimization runs in the background. Once that is completed doing its thing, the build begins its download at 0%.

So, don't think in terms of minutes, but rather hours ... two or so. The objective, of course, is to allow the build to only download the files that are new, compared to the last build, and therefore, a smaller download.

Likely, it will get faster, later, but for now, it will give one plenty of time to out to dinner, etc, while the upgrade proceeds. For ones with anxiety attacks, it's best to not sit and stare, but find something interesting to divert one's attention.
