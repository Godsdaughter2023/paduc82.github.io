ken sheridan say: Rather than concatenating the values I find that a better, and code free, solution is often to return them in a multi-column subreport whose column layout is set to 'across then down'.  This way the values are vertically aligned, which to my mind is generally a better form of presentation than a delimited list.  You'll find an example as ClubMembership.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

The file also includes a single report solution in which the report layout is manipulated in code in the report's module at runtime.  Ignore this; it was only included because this file was originally produced in response to a question by a reader of a magazine column written by a contact of mine.  For some bizarre reason the reader wanted a code solution, but it's a pointless way of doing what can be far more easily done with a subreport, requiring no code whatsoever.

If you do want to pursue a concatenation solution you might like to take a look at the file Concat.zip in the same SkyDrive folder.  this includes a solution using the very efficient GetRows method of the ADO recordset object to concatenate values.  As the example it uses data from Northwind.

Allen Brown also provides a concatenation function at:

http://allenbrowne.com/func-concat.html

You also appear to want to return the lowest model value alphabetically per part.  I don't follow the rationale behind this, but it can be done with the MIN function, so the basic query to do this and return the total demand per part would be:

SELECT Part,
MIN(YourTable.Model) AS Model,
SUM(Demand) AS TotalDemand
FROM YourTable
GROUP BY Part;

This would be the RecordSource of the parent report if you adopt a multi-column subreport solution.  If you concatenate the model names in the query you'd add a computed column to the query calling either mine or Allen Brown's function.

timgleason say:

Avery Label issue

Built a page using an Avery template to create a table for printing labels. The first cell, top left, will not accept certain formatting. Within the cell, a CR will only push the entire table down a line. Cell margins, etc., do not work either.

All other cells in the table are fine. Only the first one. Ideas?

TimG

jay freedman say:
I agree with Charles about using the built-in Labels dialog instead of Avery's "templates" (which in fact are documents, not templates, as far as Word is concerned). I'll just explain what you're seeing.

This is actually a "works as designed" behavior. If it didn't do that, it would be very difficult to add text to any document that already starts with a table. Microsoft programmed special behavior for the Enter key in that particular scenario.

When Word detects that you're pressing Enter as the first character in the first cell of a table that is at the very beginning of the document, it puts the resulting paragraph mark above the table instead of in the cell.

This is one of two possible ways of inserting text before the table (if that's what you wanted to do); the other way is to put the cursor anywhere in the first row and click the Split Table button on the Table Tools > Layout ribbon.

There are three ways to insert a paragraph mark at the beginning of the first cell, instead of outside the table:
  • Type any character in the first cell, then press Enter after it; and delete the extra character. This works because the Enter isn't the first character in the cell at that time.
  • Copy a paragraph mark from anywhere else (such as another cell of the table) and paste it into the first cell. This works because the paste doesn't involve pressing Enter.
  • Press Enter in the first cell; when it jumps out of the table, either select and drag that paragraph mark into the first cell, or go back into the first cell and press Enter again, then delete the unwanted paragraph mark from outside the cell. The first of these methods works because a drag doesn't involve pressing Enter, and the second works because the table isn't at the beginning of the document at that time.
