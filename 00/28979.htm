bossdweeb say:

- The gpu is meant to reach 90-99% usage in a graphics intensive game.

- While an idle of 57C  is a bit hot for idle speeds, the max of 80C is warmer than desirable,

these temps should not be an issue.

If the machine has started running at higher temps than it used to new thermal paste may help, but ...

" I have never cleaned the fans or opened it because I am scared it would break from static electricity."

I'm guessing your temps are a bit higher than desirable because of dust. 

Static electricity is over hyped.  Don't worry about it too much.  Components are shipped in anti-static bags because there is no knowing if a shipment may be subject to electrical charges.

It's not a good idea to use a vacuum cleaner as they are said to create static, but working

on a PC at home is safe enough.

Just unplug the machine and wait until the LED on the motherboard goes out, or 30 sec

if there is no light (If you have speakers on they should 'pop' when it is safe to work).

Touch an unpainted part of the case to create electrical continuity between body and machine

and get into it.
Exercise due care, and remove the graphics card and RAM (I remove the RAM because dust can

get into the slots if you bump them when dusting, which creates a bad connection).

Do not touch the copper terminal strips .

Brush out the case, filters on the fans, (the PCIe slot and  RAM slots last) with a soft brush (or use canned compressed air) , and clean the heat-sink on the graphics card before reseating it.

If the temps don't drop, try applying new thermal paste (after doing a search for a tutorial  ).

- There are many other potential sources of your problem, which sounds like  TDR errors.

If you are using a recent driver, I would be inclined to install an older driver.

Recent drivers would have little or nothing in the way of optimisations for an HD5000 series card, and it may run better on an older version.

See how you go with version 12.1.

http://support.amd.com/en-us/download/desktop/previous?os=Windows%207%20-%2064

TDR can be caused by a corrupt driver, so a 'clean' driver install may help.

After uninstalling the AMD drivers and CCC, reboot and run Driver Fusion.

Driver Fusion will 'clean' any remaining AMD graphics driver files and registry entries.

Reboot again before installing the 'new' drivers.

Treexy external page

That should sort out any possible driver issues, so you can move on to...

Hardware:
- RAM. Run  memtest to verify the RAM is not producing errors
- HDD. Run  chkdsk to test the HDD for errors
- Inadequate or ageing power supply  can cause lock-ups or random re-boots.

- Are your BIOS settings at default, or have you changed any settings recently, especially

the RAM or cpu overclock ?

- Software...

- If TDR occurs when using the internet it is sometimes due to browser add-ons, or the
version of the browser you are using.
- Anti-Virus software, particularly AV suites that run a number of programs in the background
are another possibility. Running more than one AV program in resident (always on) can also

cause conflicts.
- Malware, Spyware or Virus are possible. Run a full boot-time scan with your AV.
A virus called iehighutil.dll  (not sure if it's still a problem) that come as an added bonus from some torrent downloads is one example.
- System 'optimising' software is another possibility .
- Games often cause TDR. Bugs, mods and in some cases using too high graphics settings.

- Did you install anything around the time the crashes started ?
- Conflicts with other drivers can also cause TDR. One such driver comes with Daemon Tools.

You can also check for conflicts from Startup programs or Windows Services by running in

Clean Boot -

http://support.microsoft.com/kb/929135

btw - which processor are you looking at ? A cpu for a motherboard from 2010 may be

difficult to source new. Particularly if it is the first of the i3's with the 1156 cpu socket.
