tapestryguy say:

CR's instructions fixed my "disappeared start menu" problem.  For those of us who are not "power users," I add a few elucidations to his instructions:

- Step 3 actually is two separate steps. First, you run the PowerShell command, and then you run the longer command (which I am abbreviating) Get-AppXPackage ... \AppXManifest.xml"}

- You run both of those commands at a Command Prompt. Specifically, you need to run them at the Command Prompt (Admin). You can find that command by right-clicking on the Start button. A menu will appear, and you will set a few options, among them the Command Prompt (Admin) option. When you click on it, a window will open. Inside the window will the an old-fashioned command-line prompt. If you have worked with DOS or Linux, you will understand it immediately. If you have not, and if the window looks unfamiliar to you, fear not.  Where the cursor is blinking, type PowerShell, then hit the "Enter" key.  Wait a few minutes until the blinking cursor returns, and then do the same thing with the longer command. (You actually can copy-and-paste the longer command; but make sure you copy the full command, from the initial letter G to the ] bracket at the very end, and without any extra spaces before or after.

- That second command also takes a few minutes to run. As it does, you will see a bunch of command lines scrolling up the screen. When I ran it, several of the commands were in red, indicating errors. The problem was that a Store App application was still running and blocking several of the corrective commands. I opened the Task Manager (Ctrl-Alt-Del), found the offending application easily in the Apps list on the Processes tab, and closed the Store App manually. Then I reran the second, longer command. This time, it ran without a hitch.

When all of this was finished, the Start Menu was back.

hairyfool say:

The Get Windows 10 utility only does a fairly basic check of hardware compatibility, primarily specific features required in the CPU, chipset and flags known incompatibilities the commonest one being particular graphics devices.

Manufacturers will only evaluate limited, newer hardware for running newly released operating systems, often because they would be current when issued pre release builds. They get no benefit for testing legacy hardware but to cover themselves anything not tested would be deemed incompatible.

Intel do not list drivers for the H61 chipset on Windows 10, these would be needed by Dell at least in order to incorporate them.

It may be that your system WILL run Windows 10, nobody will confirm that officially.
