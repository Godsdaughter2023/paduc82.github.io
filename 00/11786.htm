As a rule, for most of the change events, (workbook, worksheet, etc) you can only have one instance.  So, only one Private Sub Workbook_SheetChange.  With that said, there is a "cheat".  What I do is create the different events as separate macros, like macro1 and macro 2 and then within the change event, place a call to the separate macros.

An example would look like this

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)

  Call macro1(target)

  Call macro2(target)

End Sub

Sub macro1(ByVal Target As Range)

my code

End Sub

Sub macro2(ByVal Target As Range)

my other code

End Sub

You have to specify if you are watching a target, etc in the different subs.  I am not guaranteeing this to work, but it works for me when I use the multiple ranges to target.

daniel g. grau say:
I don't think the problems are related, because the updater does not look at a valid license number. Only Office applications do, when executed. Make sure that:

  1. You have not renamed or moved the original Office folder or the Office applications.
  2. The folder should be called Microsoft Office 2008 and reside in the top Applications folder (the one you see when you open the disk's icon in your Desktop).
  3. The applications themselves (which should be inside this folder) need to be named "Microsoft Word," "Microsoft Excel," etc.

Under any other conditions, the updater won't find the proper folder and/or files to update. This is why users are able to apply the updates after a full reinstall—all files, folders and applications go back to Applications with their original names, which allows the updater to locate them.

Your MS account is inextricably tied to many functions on the phone, Cortana, main e-mail, Onedrive, and Live services. Setting a PIN on the account amounts to setting a PIN on the phone, i.e. the lock screen. When you unlock the phone using this PIN, you then have access to all MS services that are tied to that account, from that phone.  Without the PIN (or some other option) anyone picking up the device will have full access. There are some instances were you may be required to authenticate in some manner even if you unlocked with your PIN.

This PIN does not transfer off that phone. Another phone set up with the same MS account will not automatically adopt the same PIN, and in fact a different PIN could be set. Neither would affect the MS account password that you would use on a PC, or the web to access services or your account.  

You can turn the PIN back off, and reverts back to what was being used.

Hope that helps.
