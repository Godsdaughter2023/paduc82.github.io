How to code, Excel VBA form button to: - Microsoft Community

computerxllc say:

How would this be coded in 1 button? For some reason my brain just doesn't want me to understand this!
hansv mvp say:

Create text boxes txtRows and txtColumns on the userform, and a command button cmdOK.

Create the following On Click event procedure for the command button:

 

Private Sub cmdOK_Click()
    If Not IsNumeric(Me.txtRows) Then
        Me.txtRows.SetFocus
        MsgBox "Please enter a valid number of rows"
        Exit Sub
    End If
    If Me.txtRows < 1 Or Me.txtRows > Rows.Count Then
        Me.txtRows.SetFocus
        MsgBox "Please enter a valid number of rows"
        Exit Sub
    End If
    If Not IsNumeric(Me.txtColumns) Then
        Me.txtColumns.SetFocus
        MsgBox "Please enter a valid number of columns"
        Exit Sub
    End If
    If Me.txtColumns < 1 Or Me.txtColumns > Rows.Count Then
        Me.txtColumns.SetFocus
        MsgBox "Please enter a valid number of columns"
        Exit Sub
    End If
    Application.ScreenUpdating = False
    Cells.EntireRow.Hidden = False
    Cells.EntireColumn.Hidden = False
    Range(Cells(Me.txtRows + 1, 1), Cells(Rows.Count, 1)).EntireRow.Hidden = True
    Range(Cells(1, Me.txtColumns + 1), Cells(1, Columns.Count)).EntireColumn.Hidden = True
    Application.ScreenUpdating = True
    Unload Me
End Sub

scottgem say:

OK, First, did you just type the line of code in the Click event or did you open Code Builder to enter the code? You have to do the latter.

To use a macro you need to use:

="fieldname = " & Forms!formname!controlname

in the WHERE clause. Note you have to use your Actual object names.

Note also, that I'm assuming that IPO No is a Number datatype.

Finally, if you are going to have a lot of table (which is normal for a relational database), you NEED to understand and use queries. One of the principles of a relational database is that data exists in one place only. So, if you need to pull a customer name, Part description etc from the one table where it resides you need to use queries to do so.
