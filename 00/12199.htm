Following the creation of a new record, hitting enter after the last field (Production Task Volume), I'd like to create an event which duplicates the following fields from the last committed record: Date, Associate ID, & Associate Name. Such that the user may now enter a new Production Task and Production Task Volume for the same Associate without having to reenter the current Associate information.

5. When I have entered the last Production Task for the current Associate, I'd like to have a Toggle button in place. scottgem say:

First, Associate Name should NOT be in this table. You should have a table of Associates (which I assume you have by the fact you have an Associate ID). That's all you need about the Associate in your Production tracking table, You can display the name using a variety of techniques (see my blog on Displaying data from Related tables). I would use a combobox, that displays the name, not the ID. Second, I don't know if those are your actual field names or not, but using spaces in Object names is not a good idea and can come back to haunt you. Also Date is a reserved word in Access and shouldn't be used for an Object name.

What I would do is use the AFTER UPDATE event of the ProductionDate and AssociateID (notice what I have named them) controls to run this one line of code:

Me.ControlName.Default = Me.Controlname.OldValue

where Controlname is the name of the control you are working with. This will change the defualt value of the control so, when you add a new record, those controls will default to the last entered values.

One more point. People names should be broken out into its components, at least first and last.

ken sheridan say: You can call the FollowHyperlink method of the Application object in the control's Click event procedure, e.g. for FedEx use the following as a single line of code:

Application.FollowHyperlink "https://www.fedex.com/apps/fedextrack/?action=track&cntry_code=us&tracknumbers=" & Me.txtFedExTrackingNumber

where txtFedExTrackingNumber is the name of the text box in which the number is inserted.  

If Access 2010 supports these properties, set the DisplayAsHyperlink property of the control to Always, and its IsHyperlink property to True (Yes) to give it the appearance and behaviour of a hyperlink.
