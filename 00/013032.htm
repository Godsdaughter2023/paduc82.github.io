Office 2016 & 2013 Parallel Install without Office 2016 Background Stream Install Error - Something went wrong client update error

Office 2016 & 2013 Pro Plus Parallel Install

The install of office 2016 with out Background Stream Install Error - Something went wrong client update error is simple:

My Install Office flow note: My install is on Windows 10 64 bit

  1. I had office 2013 Pro Plus installed prior to the office 2016,
  2. I uninstalled Office 2013 like the update error & Office help page stated this just seemed logical
  3. I was able to install Office 2013 again after my failed 2016 install even when I did not uninstall Office 2016.
  4. I receive the warning that office was installing another version (Retry or Ignore) to proceed
  5. I selected Ignore and installs completed with out issue I can find

1) get your office 2016 .iso or installer media

  • (I use .iso & Power ISO)
  • (use what ever your fav. app is)

2) Extract your Office 2016 .iso\office media file to a directory location

  • I use directory location + office version folder
  • Power ISO allows for you to create a name in the directory location the specify a folder that it will create for you
  • Example C:\temp\office version
  • (C:\temp\office 2016)
  • (C:\temp\office 2013)
  • (C:\temp\Visio version)

3) Once the Office 2016 files are extracted to the directory location

  1. Go to the Directory Location
  2. Then open Office 2016 folder that your media was extracted to
  3. You will possible notice both a setup32.exe & setup64.exe Office installer's along with a data folder
  4. Validate your version prior to moving forward
  5. Example:
  6. If you have a 32 bit version of Office 2013 or Visio installed you have to stick to 32 bit 2016 same applies to x64 Office products

4) Once you have validated your version that you would like needs to be used

  1. Open the Data folder
  2. you will notice a folder labeled (16.0.4266.1003) possible some other Office 2016 version number & a bunch of .cab files
  3. The .cab files will be labeled
  • (v32\vu32),
  • (v64\vu64)
  • (v64_16.0.4266.1003\vu64_16.0.4266.1003)

     4. If you are installing setup32.exe then extract all  (v32\vu32 & then (v64_16.0.4266.1003\vu64_16.0.4266.1003) .cab files

     5. If you are installing setup64.exe then extract all  (v64\vu64) & then (v64_16.0.4266.1003\vu64_16.0.4266.1003) .cab files

folder labeled (16.0.4266.1003) Did have .cab files within it, I did not have to extract these or mess with any of the streaming data within this folder

6) Once the .cabs are extracted return to the parent directory that the setup32.exe\setup64.exe & Data folder were displayed

  1. Select the setup.exe that correlates to the unzipped .cabs & office versions already installed 
  2. kaboom, Office 2016 & 2013 Parallel Installed without Office 2016 error

a. user say:

Debbie

I've just posted a response in the original conversation. The idea is that you use a set of nested fields like this:

{ IF { INCLUDEPICTURE { IF TRUE “{ MERGEFIELD "picturepathfield" }” } \d } { INCLUDEPICTURE { IF TRUE “{ MERGEFIELD "picturepathfield" }” }  } }

where "picturepathfield" the column name of a column in your data source that contains the picture path name, e.g.

gofl1:mypics:abc1004.jpg

All the {} need to be the special field code braces that you can (usually) insert on Mac using cmd-F9 (or fn-cmd-F9).

As far as I know, this approach only works with .doc format files, not .docx format files. 

If your merge field does not contain the entire path name, e.g. if it just contains the file name "abc1004.jpg", you will need to build the whole path name in the two bits that say
{ IF TRUE “{ MERGEFIELD "picturepathfield" }” }
e.g. all your pictures are in gofl1:mypics
then you might use
{ IF TRUE “gofl1:mypics:{ MERGEFIELD "picturepathfield" }” }

As far as I know, these particular INCLUDEPICTURE constructs were invented by Word MVP macropod.

ronrosenfeld say:

The problem is the Apple keyboard has changed its default double quote to a smart quote.(on mine it is Unicode 8220). So you are not typing "*" but rather something like ʺ*ʺ

One way to get the right double quote is to long press the double quote key on the soft keypad.  You will then see options for the various different types of double quotes.  Slide your finger up to the correct double quote symbol (the "straight one"), and you should be OK.

If you want the straight double quote to be the default, go to Settings-General-Keyboard and turn off "Smart Punctuation"

Making Fillable Forms Accessible for Screen Readers - Microsoft Community

krystaliseminger say:

Making Fillable Forms Accessible for Screen Readers

I am in charge of accessibility training at my institution and am needing to train on making fillable forms accessible. What is the best method for creating forms that work with assistive technology like Jaws and NVDA? Should I use Legacy controls or content controls? Any suggestions would be helpful.

-Krystal

karenmccall1 say:

I thought I should clarify. You can make an accessible form template in Word. You can then use that template to create an alternate format if requested.

To make an accessible PDF form from a Word template, here are some basics:

  1. Use the title style for the name of the form/do not put it in the Header of the document.
  2. Do not use tables for design layout unless you are creating ratings questions, then make sure to have column and row titles/table headers.
  3. To position information without using tables, use Tab Stops or Columns.
  4. Do not use symbols for form control stand-ins. In the PDF document they may show up as Unicode errors or require remediation to make them Artifacts.
  5. Make sure you give enough room in the form for the answer you expect. for example, don't give someone two inches of a line to answer a question where you ask for a lot of detail.
  6. Use Heading styles to identify different sections of a form. For example, contact information, education or other areas.
  7. Use standard fonts like Calibri or Cambria and be consistent in the use of fonts.
  8. Organize the information you are requesting in a form in a logical reading order.
  9. If there are any specific instructions, ma

    Once you have the accessible Word template:

    1. Press F12 and choose PDF as the file type.
    2. Before yu activate the Save button, activate the Options button and make sure that Headings are used for Bookmarks.
    3. Then save the Word document as a tagged, accessible PDF document.
    4. Once the save is complete, it should open in Adobe Acrobat Pro. You will need that application in order to make an accessible PDF form.
    5. Run the Accessibility Full Check and get the form template conforming or passing an Accessibility Full Check in Adobe Acrobat Pro.
    6. Then add the form controls to the PDF document making sure to add them/arrange them in their Tab Order/logical reading order.
    7. Once the form controls are added to the PDF document, use the Find tool in the Tags Panel to add the form controls to the Tags Tree.
    8. Each form control must have its own <Form> Tag and be placed with the corresponding .text for the question it is related to.

    Do a final Accessibility Full Check to make sure the document passes. You may need to make some minor remediations.

    In the Document Properties for the PDF document, under the Advanced tab, make sure that a plain vanilla language like English, French or Spanish is used not a localized one. Under the Intial View tab, make shure that Doucment title not File name is going to be shown.

    This is a VERY brief overview of the process to make an accessible word form template and convert it to an accessible PDF form.

    Cheers, Karen
