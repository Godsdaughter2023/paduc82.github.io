I am able to reproduce your issue on a Nokia Lumia 1520 with Windows Phone 8.1 (OS version 8.10.12397.895). www.aa.com:443 supports a single cipher-suite only (RC4-SHA; 0x0005). Furthermore, Windows Phone does not support RC4 at all. You face a cipher-suite mismatch. Therefore, American Airlines returns a TLS alert (handshake failure) on the very first TLS-Client-Hello. . see below

 

You are able to reproduce this yourself via a UNIX terminal (command line) with

openssl s_client -connect www.aa.com:443 -cipher AES128-SHA

Because RC4 is consider deprecated by Microsoft, you have to convince American Airlines to add further TLS cipher-suites to their server installation.
