It sounds like you're possibly another victim of the failed 16288.1 update and may need to manually reset the Windows Update process...... 

If you've already tried running the Windows 10 Update Troubleshooter and that hasn't fixed the problem (and it quite often doesn't) then you'll need to take more drastic action and attempt to reset the Windows Update process manually.

simple method is to copy the text between the ++++ lines and paste it into Notepad

++++++++++++++++++++++++++++++++++++++++++++++++

net stop wuauserv

net stop cryptSvc

net stop bits

net stop msiserver

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

+++++++++++++++++++++++++++++++++++++++++++++++++

Save as UpdateReset.BAT (note the .BAT and NOT .TXT) to your desktop and close it.

Go to the desktop and right click on the .BAT file and select "Run as Administrator"

That should flush out any corruption in the Windows update files and enable you to move on to download the latest build. If you get a message saying that wuauserv can't be stopped then run the Update Troubleshooter and then try the batch file again.

Once done, go back and check for updates and also check to see if the C:\Windows\SoftwareDistribution SoftwareDistribution.old and C:\Windows\System32\catroot2 Catroot2.old directories were created and, if they exist, delete them in case you need to re-run the batch file again.

Finally, before trying to update again make sure you detach any non-essential peripherals, USB devices and completely remove any 3rd party anti-virus software as they may interfere with the update process,

Hope that helps and good luck.

macropod say:

There seems to be a bug with updating the dialog to show the current selection's shading. You can force it to do so, however, with a macro like:

Sub ShowShading()
With Dialogs(wdDialogFormatBordersAndShading)
  .DefaultTab = wdDialogFormatBordersAndShadingTabShading
  .BackgroundRGB = Selection.Shading.BackgroundPatternColor
  SendKeys "{TAB}{TAB}{TAB}{DOWN}M"
  .Show
End With
End Sub

For macro installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm

jay freedman say:

Since you need a macro anyway, you might as well bypass the dialog and just grab the numbers:

Sub ParaBackgRGB()
    Dim lgBackColor As String
    Dim R As Long, G As Long, B As Long
    ' Get the background color at the cursor
    ' as a 6-digit hex number
    lgBackColor = Hex(Selection.Paragraphs(1).Shading.BackgroundPatternColor)
    ' Convert each pair of hex digits to the
    ' corresponding decimal number
    B = CLng("&H" & Mid(lgBackColor, 1, 2))
    G = CLng("&H" & Mid(lgBackColor, 3, 2))
    R = CLng("&H" & Mid(lgBackColor, 5, 2))
    ' Display the result
    MsgBox "R = " & R & vbCr & "G = " & G & vbCr & "B = " & B
End Sub
