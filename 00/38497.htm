lemp say:

Congratulations and thanks for the feedback.

For the benefit of someone else who might end up in a similar situation, what was the path to this hidden temp folder?

Disk Cleanup doesn't do a very good job on temporary files. 

Each user account on a Windows computer has its own "Temp" folder.  By default, that is located at

                    C:\Users\<username>\AppData\Local\Temp

There is, however, also a "system" temporary folder.  By default, that's located at C:\Windows\Temp.  And, as I mentioned earlier, there's also C:\Windows\Logs.

And, for what it's worth, my Windows 7 Pro system has 40 folders named TEMP.  Fortunately, most of these folders are empty or close to empty.

The "temporary files" that Disk Cleanup looks for are in the folder identified by the %TEMP% environment variable.  That points to the current user's temporary folder.  You can confirm this by typing %TEMP% into the Start > Search box (including the percent symbols). 

As far as I know, Disk Cleanup doesn't even look at the system temporary folder and thus doesn't delete anything from it.  I've seen some websites that claim that Disk Cleanup will go after C:\Windows\Temp and some other system locations (like the Logs folder) if you run Disk Cleanup by right-clicking on its shortcut and selecting "Run as administrator"... but I've never been able to confirm that behavior.

And Disk Cleanup won't look at any other "temp" folders that might be scattered across your hard drive(s).  I doubt that even CCleaner will get most of them. 

In addition, Disk Cleanup will not delete everything in the TEMP folder that it does look at.  It won't delete:

  1. Files with Read-Only, System or Hidden attributes
  2. Files which have been accessed in the last seven days
  3. Directories that have been created in the last seven days.

For all of the above reasons, it's a good idea to periodically use a tool like TreeSize Free or WinDirStat (which is more popular than TreeSize Free) to see if some folder is growing out of control.  It's best to deal with this sort of thing before you run out of space.  See http://lifehacker.com/5915921/the-best-disk-space-analyzer-for-windows

a. user say:

Hi,

Thank you for posting your query in Microsoft Community.

As your system is stuck in initializing screen while performing a System Restore, it is recommended to wait for some more time as sometimes it takes a longer time if the data is more in the computer, and check for status. If the system is still stuck in the same screen, then you will have to force shut down the computer, restart and check for status. If you return to the same screen, then follow the steps provided below:

  1. On a working PC, go to the Microsoft software download website and create Windows 10 installation media.
  2. Insert the installation media you created, and then restart your PC.
  3. On the Install Windows screen, select Repair your computer. (If you’re not seeing this screen, your PC might not be set up to boot from a drive. Check your PC manufacturer's website for info on how to boot from media, and then try again.)
  4. On the Choose an option screen, select Troubleshoot. From there, you can use Safe Mode. Select Advanced Options > Startup Settings > Restart.
  5. After your PC restarts, select 5 or F5 to restart you PC in Safe Mode with Networking.
  6. After the system boots in Safe Mode with Networking, try to perform a System Restore. 

Hope this helps. Keep us posted on the status for further assistance.
