a. user say:

The name "Content Section" is, AFAIK, a term introduced by WIndows Explorer or whatever property inspector it uses to display this information. 

The history of properties in Windows/Word is a long and messy tale.

In this case,
 a. both "Content status and "Content type" were introduced as Builtin Document Properties in Word 2007 (along with "Language" and "Document version")
 b. None of those new properties exists in Mac Word 2011
 c. Of those 4 new properties
    - All 4 can be set via VBA. 
    - All 4 (I think) survive if you then save as .doc format
    - All except "Content type" survive a save to .docx format
    - All 4 can be set in the tab in WIndows Explorer that you are using. All except "Content type" survive opening a .docx.
    - None can be inserted/updated using a Word field
    - only "Content status" appears in the standard Document Information Panel (as "Status")
    - only "Content status" can be inserted using a Document Property Content control (i.e. you can link a Content Control to its value in the data store
    - "Content type" is the only one that does not appear in the XML schema for the "coreProperties" in the ISO 29500 standard for .docx etc.

I would guess that Word might have reserved the "content type" property for one of two things:
 a. a Sharepoint content type
 b. an XML content type (e.g. application/vnd.ms-word.document.12 )

However, neither of those things seems to have happened yet.

If your company has been using the property, it is probably because someone in your organisation wanted a property that 
 a. corresponded roughly to some kind of Content Type
 b. could be set in Windows Explorer, and viewed using a custom column in WIndows Explorer
 c. could be set in Word, and possibly even...
 d. wouldn't work with .docx documents (though that seems rather unlikely).

Another possible explanation is that the property is used by Sharepoint in some way that I don't know of, or by another document management system that your organisation happens to use.

graham mayor say:
I can confirm similar behaviour here.

The issue is undoubtedly related to the fact that XML DOCX formats do not have the same metadata configuration to DOC format.  Content Status is a built-in document property, but Content Type is not. Thus Content Status can be changed and recorded in the document properties, however Content Status is not associated with a docproperty field definition so its value is of little practical use.

You can evaluate all the properties, both built-in and custom, and change those that are editable from Word 2007/2010 using http://www.gmayor.com/BookmarkandVariableEditor.htm
