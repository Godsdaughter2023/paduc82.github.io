This is what templates were invented for. The default action of a template is to create a new document from the original.

Word has a feature called Workgroup Templates that allows templates to be shared on a network. Users create new files using FILE>New>CUSTOM and the resulting documents cannot over-write the template accidentally, since they are saved by default to the user's folder, not the workgroup templates folder. If, for some reason, that solution is not to your liking, I would fix this on a operating system level by making the network share folder read-only for all users except yourself. Then they can't accidentally over-write your file.

If you're using rich text content controls, that sort of code isn't applicable at all -- it works only for

Your post is confusing because you speak of "tag names," which exist only for content controls, but the LostFocus event applies only to ActiveX controls.

If you are using content controls, then the code needs to look very different, starting with the fact that the applicable event is Document_ContentControlOnExit, and you need to use the expression ActiveDocument.SelectContentControlsByTag("something")(1) to get access to a content control with the tag value of "something".

Private Sub TextBox111_LostFocus()

charleskenyon (ia) say:
Hi,

Sometimes Office error messages are cryptic or misleading. I suspect this one is accurate. It is likely seeing one dimension of the pdf as exceeding 22." To see if that is the case, view the pdf in a pdf viewer at 100% zoom. Even if it can open it, you may still have trouble with conversion.

Since Word 2013, Word has had the ability to directly open pdf document. However, some pdf documents (from scanners mostly) are really pictures of text rather than the text itself. Just as you can't look under the hood of a picture of a car, you can't edit a picture of a document.

In that case, you need to convert the picture to text. This is a process known as optical character recognition. This is built into Adobe Acrobat and is also in Office OneNote. Most scanner software comes with an OCR component as well. Once you have text, you can edit it with Word.

If you simply want to write on the document (but not in it) you can add a Text Box floating on top of the document layer, whether or not it has been put through the OCR process.

Word documents that have been saved as PDF will not need the OCR process, they retain their text, although not all of their Word structure and formatting.

Finally, documents converted from pdf (or really any other format) to Word can be tough to edit because the conversion process never has a one-to-one matching of how formatting is done. This means that a converted document will seldom be formatted in Word in a way that makes sense. An example is multiple section breaks to change margins, where in Word you would simply change the paragraph indent.

I hope this information helps. Please let me know if you have any more questions or require further help.
