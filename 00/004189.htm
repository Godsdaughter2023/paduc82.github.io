sonamarg say:

Hello,

First of all >>> New-MailboxImportRequest

This cmdlet is available only in on-premises Exchange Server 2010/2016 NOT for Exchange Online (Office 365).

There are two ways to import PST files or SharePoint data to Office 365:

1. Network upload

2. Drive shipping  

How long does it take to import a PST file to a mailbox?

It depends on the capacity of your network, but it typically takes several hours for each terabyte (TB) of data to be uploaded to the Azure storage area for your organization. After the PST files are copied to the Azure storage area, a PST file is imported to an Office 365 mailbox at a rate of at least 1 GB per hour. If different PST files are imported to different target mailboxes, the import process occurs in parallel; in other words, each PST/mailbox pair is imported simultaneously. However, if multiple PST files are imported to the same mailbox, each PST file is queued and imported to the mailbox one at a time. In a future update, multiple PST files will be simultaneously imported to the same mailbox.

First Terminology; a Database, in Access terms is a collection of objects. Objects are tables, forms, reports, queries, etc. So I think you mean attach itself to a secondary TABLE, not database. Both can be done, but in different ways.

Every form has a Recordsource. A Recordsource is generally a table or query. It does not have to linked to the table in the main form, but generally it is. Without an understanding of your table structure, its hard to explain further.

Second a subform is generally embedded on a main form so you can work with them together. Therefore there is no need to "access some Text fields in the first form from the second form" as they both are displayed on the same form. 

DLookup can be used to DISPLAY data from another table, but then you would not be able to enter/edit data in that field. There are other ways as well to display related data as I detail in my blog On Displaying Data from Related Tables (which I believe I have mentioned before).

Finally, you have made it very hard for us to help you by not explaining the real world situation you are trying to model. As Albert explained a database is a model of how data exists in your real world. The model has to follow some rules if you want to use a relational database.
