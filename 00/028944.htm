andy kroell say:

@Parth P Patel: This won't help at all!

The problem is the result of the "by design" behavour of Windows 10 in tablet mode:

Once you close an app - or ANY window that is (!) - the UI reverts to the start menu.

If anyone uses a multi-windowed desktop app like Outlook, this is quite irritating - Outlook opens every new email in a seperate window that (obviously!) closes when the email is sent.

To my understanding there would only be two solutions to this problem, both of wich must be done by Microsoft, as the user can not change this behaviour:

Option 1:  make Outlook use an in-window-pane for writing new emails (comparable to the viewing pane in inbox or like the Windows Store Mail app), when using table mode.

Option 2:  Have Windows 10 tablet mode revert to the last used window when closing a desktop app window! (This is the one option I would prefer).

Hope this will help change the problem in a future update...

This is probably not what you want to hear but no one with any kind of experience would ever suggest something with a guarantee of "....will NEVER hang" no matter what the specifications of a given system are (some "sales" types working on commission excepted).

Certainly would suggest a system with the max amount of memory installed (at least 16gb at a minimum) an the most powerful processor you can afford (i.e. Intel i8) and using the 64bit version of Office.
