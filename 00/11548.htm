In XP, turn off DEP with thses steps:

1. Click Start
2. Select Control Panel
3. Select System
4. Click the Advanced tab
5. In the Performance region select Settings
6. Click the Data Execute tab in the dialog box that opens
7. Select Turn on DEP for all programs and services except for those I select
8. Click Add.
9. The open dialog box will open. Browse and select your application.
10. Click Open
11. Click Apply
12. Click Ok
13. Reboot


To disable DEP in Vista:

1. Click Start
2. Click All Programs
3. Click Accessories
4. Right Click on Command Prompt
5. Left click on Run as Administrator
6. Click the Allow button if it asks you for permission
7. Type or copy and paste this into the command prompt window:

bcdedit.exe /set {current} nx AlwaysOff

8. Hit enter and you should see the confirmation: Operation Completed Successfully
9. Reboot and DEP should be off.

vincentchoy say:

Not sure if your reseller is also the company that hosts your DNS as your answer isn't clear.

We have encountered cases where the licenses were purchased with the company that also hosts the DNS. Do note that if you also terminate your DNS hosting service with such provider, then you have to plan for DNS migration, and do remember that new DNS settings may take up to 72 hours to propagate and you may experience temporary disruption of service.

We have also encountered cases where a client relies on a the supplier to manage their tenant, and the supplier having exclusive ownership of the Office365 Global Admin account. In such a case it is advisable to regain control of the Global Admin account before the move.

As a precautionary measure, it is also advisable to terminate any delegated admin access from the original supplier, to prevent "Revenge" sabotage to your tenant. 

In general you should not have any problems moving to a different reseller. Unless your circumstances are somewhat unique and unusual (eg the reseller is operating in a different country, or they are handing 3rd party Exchange Online backup or other 3rd party Office365 related software for you, or some of the scenarios mentioned above) that would prevent a clean break.

Like I mention, if your new reseller is any good, they would be able to provide you convincing answers on the best practices for a smooth transition.

vincentchoy say:

It depends on who you subscribed with initially.

If you subscribed with a syndicated provider, usually a Telco or someone like Godaddy, then a migration might be required, as they have a special arrangement with Microsoft.

If you subscribed via any of the regular IT resellers, then you are hosted by Microsoft, the resellers are merely providing you the license. In such a case a migration is not required and when your subscription reaches the anniversary, just get the licenses from the 2nd reseller.

Any competent reseller should be able to look into your subscription in detail and provide you good advice.

ken sheridan say: The problem with your Student Details form is that it is based on a query which joins Students and Preceptors.  It should be based on Students alone.  As there are currently no matching rows in Preceptors the query will return no rows.

There is a more fundamental problem with the database, however.  I'm unfamiliar with the educational model in your case, but I assume that there is a many-to-many relationship type between Students and Preceptors, i.e. each student can have more than one preceptor, and conversely each preceptor can have more than one student.  A many-to-many relationship type is modelled by a third table which resolves the relationship type into two one-to-many relationship types, in your case like this:

StudentPreceptors
….PreceptorID   (foreign key referencing the primary key of Preceptors)
….ID                   (foreign key referencing the primary key of Students)

The primary key of this table is a composite of the two foreign keys.  Note that Preceptors does not contain an ID foreign key referencing the primary key of Students.  The table might contain other columns which represent attributes of the student's relationship with the preceptor.  It is essential that all relationships between the tables have relational integrity enforced; this is extremely important.

A preceptors subform would then be based on a query which joins the StudentPreceptors table to the Preceptors table.  In the subform would be a combo box bound to the PreceptorID, but set up to show the preceptor names.  Other controls in the subform would be bound to columns from the Preceptors table and to any other columns representing attributes of the student's relationship with the preceptor.  The former would be automatically be populated once a preceptor is selected in the combo box; the latter would require user input.  By setting the Locked property to True (Yes) and the Enabled property to False (No) in the case of the former, these controls can be made read-only.

You might like to take a look at DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

This little demo file provides an introduction to the basics of designing a relational database.  It should give you some understanding of how the different relationship types are used to model the real world entity types with which the database is concerned, and how the model is reflected in the user interface.

FIPS-197 defines AES as an standard for high encryption applications, as a substitute to the infamous DES.  I didn't read the FIPS-197, but I'm almost sure that this document defines AES as a encryption based in the Rijndael algorthm, defines the work modes (CBC, XTS, etc).

 

In fact, Nokia doesn't use AES-128, but 256.  I read your link and says:

 

"Encryption key length: 128 bits;initialization vector: 128 bits"

 

That means, although 128 bits are used a as ciphering key, it needs another number, variable for each sector read or written.  That number is the IV (initialization vector), and this number is added with the encryption key. This makes the cipher key extends to 256 bits.

 

So, although the E71 encryption is not FIPS197-validated,  I think Nokia E71 is more secure thanany Blackberry, mainly because all the mail and web traffic passes thru Blackberry servers.  Instead, the E71 process all of this tasks as its own.

jim gordon mvp say:

Run-time error '5' is almost always caused by incorrect syntax within the VBA code. Often, it something hard to spot, such as a missing quotation mark.

The code base of Excel for Mac is the same as the Excel for Windows, although the object model is not identical.

Macro enabled workbooks have file names that end with the letter m

.xlsm - Macro enabled Excel workbook

.xltm - Macro enabled Excel template

.xlsb - Excel binary workbook (also supports VBA even though it doesn't end in m)

Do not manually change file extensions. Doing so corrupts workbooks. Instead, use File > Save As and change the file type.

You should be able to open and edit any workbook made in Excel for Windows on your Mac, assuming your install is up to date. Go to the Excel menu and verify you have version 16.13 or higher. If your version is something else, do the following:

Make sure your Mac has a working internet connection before checking for updates. Install Mac OS X updates followed by Office updates.

Install Mac OS X updates first:

To get Mac OS updates, go to the App store on your dock, and then click the Updates button to reveal available updates.

Then Install Office Updates

To get the update:

1.     Open any Office application

2.     Go to the Help menu and choose Check for Updates

3.     Let AutoUpdate check for and install updates

4.     Restart your Mac

If you have any add-ins such as WebEx, Mendelay, EndNote, Zotero, TypeIt4me, etc it is essential that you install the latest updates for your add-ins.

Once you have everything up to date, you can pretty much rule out any issues being caused by your software and focus on what's wrong with the file.

Since you work with files having macros, you should turn the Developer tab of the Ribbon on. The setting is on the bottom of the View preferences in Excel preferences.

    david j parker say:

    You are correct that the Rounding has been protected, but you can remove this:

    When you drag and drop a shape (Master) from a Visio library (Stencil), a copy of the Master is made in the Visio document that you are editing. This copy is editable, but the original one in the library is not.

    First, you need to be in Developer Mode:

    1. File / Options / Advanced

    2. Scroll down to the General section

    3. Check Run in developer mode

     

    Then you need to edit the Master shape:

    1. Select More Shapes / Show Document Stencil

    2. Locate the "Rounded process" master shape

    3. Right mouse click the shape and select Edit Master / Master Properties ....

    4. Check "Match master by name on drop"

    5. Select "OK"

    6. Right mouse click the shape and select Edit Master / Edit Master Shape

    7. Select the Shape in the Master page that opens

    8. Right mouse menu Show ShapeSheet

    9. Scroll down to the Line Format section

    10. Click on the Rounding cell

    11. Replace

    =GUARD(7.5 mm*User.AntiScale)

    with

    =Theme("Rounding")

    12. Close the ShapeSheet

    13. Close the Rounded process Master window

    14. Select Yes to Update "Rounded process" and all of its instances

     

    You can now edit the Rounding of each shape instance or use the Theme setting
