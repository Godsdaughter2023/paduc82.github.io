roger carlson access mvp say:

In order for a query with a JOIN to be updateable, there must be a Relationship established between the tables.  You  can't create a relationship between an external, linked table and a native table.

You may be able to work around this with a form/subform, however. I have not tested this, but it's something to try: 

Create your mainform based on the read-only table and your subform on the updateable table.  If there is a one-to-one relationship between the tables, link the parent/child forms on the respective primary key fields.  This should allow you to view the data in the main form and edit the values in the subform.

There is not only one "flavor" of PDF, there are a lot of things going on when "compressing" a PDF.

A PDF containing images can be compressed mainly using 2 methods.

1. Resampling the images.

2. Compressing the images using JPG compression using different settings.

It seems Powerpoint has no way of configuring these export settings, You only have the two default options. "Small and very small".

One option is that you send it to print to a virtual printer. This is a printer that instead of sending it to paper, it sends it to a file.

echo s say:

A couple of other things that often cause PPT to crash.

1. Printer drivers. PPT likes a known good printer driver installed locally and set as the default driver. This was more an issue in 2007, but it can still rear its head in 2010. Seehttp://www.pptfaq.com/FAQ00952.htm as an initial troubleshooting test.

2. Add-ins. The Send to Bluetooth add-in especially seems to cause crashing in PPT 2010. Go to File | Options | Add-ins, and check what add-ins are installed. There will be some listed as type Document Inspector. Those are part of PPT's Document Inspector and won't cause issues. I'd say all others could be suspect, though. Disable or unload them and see if the issues go away. You can reinstall/re-enable one at a time to see if you can isolate the problem add-in.
