jay freedman say:

Store the tables as AutoText entries in the template on which the documents will be based. (You are creating a template, correct?) All of them must have the same Category value, preferably a custom value that isn't used for anything else.

Then you can choose one of these approaches:

  • Add an AutoText gallery button to a custom group on a ribbon (maybe the Insert tab). This enables the user to insert their choice of table into the text at the current cursor location.
  • Place either a building block content control or an AutoTextList field in the text where you want the table to appear.
  • An ordinary dropdown (either a content control or a legacy dropdown field) together with macro code to recognize the selected value in the dropdown and insert the corresponding AutoText at the desired place.

All of these options are discussed at https://gregmaxey.com/word_tip_pages/building_blocks_autotext.html.

jay freedman say:

The only way to get that much control over what users can do is to write macro code. You probably want to create a userform (a custom dialog box) containing a dropdown list box. The code that makes the userform work can then insert the corresponding table at a specific location in the document. The tables that are available for insertion should still be stored in the template as AutoText entries or another type of building block. The macro code has access to the building blocks and can insert them at bookmarks or other predetermined locations.

A reference for userforms is https://gregmaxey.com/word_tip_pages/create_employ_userform.html.

05/06/2010     25     8
06/06/2010     11           0
Grand Total      4     92

shane devenshire say:

1.  This result can be obtained by filtering on the Date field in the pivot table. 

2. If you still want to type the dates in in another sheet and get the results you can setup a copy of the pivot table on that sheet, and write a macro that responds to the entry of two dates by automatically filtering the data.

3. You can use SUMIFS or SUMPRODUCT against the original data rather than the pivot table.  Suppose the raw data is:

    A            B                 C     (Sheet1)

1 Date    Country   Passengers

2 1/1/01    France      3

You enter your two dates on another sheet in A1 and B1  and in C1 the country then the formula would be

=SUMPRODUCT(--(Sheet1!A:A>=A1),--(Sheet1!A:A<=B1),--(Sheet1!B:B=C1),Sheet1!C:C)
