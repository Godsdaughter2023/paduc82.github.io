suzanne s. barnhill say:
The methods shown in that article won't help with adding a caption number (manually or automatically) for a left-aligned equation, but you can use the old-fashioned method of putting the equation in a two-cell, single-row borderless table. The equation goes in the left cell, left-aligned. The caption (using References | Captions | Insert Caption as shown in the article) must be inserted outside the table to begin with, then copied into it (right-aligned). Once you have the caption field, you can save it as an AutoText entry, or, to save even more time, you can save the entire table row (with a dummy equation) as an AutoText entry. The caption numbers will increase automatically as you insert the AutoText.

That often happens because the network inadvertently got changed during the update from private to public. Right click on the network icon and select open network and sharing center ... if your network shows as public, that's the problem. Change it to private, and you'll have internet access.

There was a bug in a previous build that caused the false indication of no internet, when, in fact there was; and, as Lawrence said, at least one other person is reporting an issue similar to that old bug.

On the subject of restore points, it seems that many don't realize that the purpose of system restore is to be able to go back to an earlier system state in the currently installed OS build, and therefore, installing a different build is intended to delete the restore points that were set in the previous build and turn system restore off. Part of setting up the newly installed build is to configure system restore for the C partition to on, and set a restore point; therefore, if you don't do that, there will be no system restore point.

The CPU cap can still be set with PowerShell; although as always, this setting only applies to scheduled scans ‚Äì but this function now appears to have been broken by a recent update:

-ScanAvgCPULoadFactor

Specifies the maximum percentage CPU usage for a scan. The acceptable values for this parameter are: integers from 5 through 100, and the value 0, which disables CPU throttling. Windows Defender does not exceed the percentage of CPU usage that you specify. The default value is 50.

https://docs.microsoft.com/en-us/powershell/module/defender/set-mppreference?view=win10-ps

Right-click on the Start button and select Windows PowerShell (Admin), and then copy and paste this command; add the value; and enter.

Set-MpPreference -ScanAvgCPULoadFactor [Maximum Average CPU Load Factor]

The old trick for taming a hyperactive Full Scan was to schedule one in Task Scheduler, and then set the CPU cap lower than the default value ‚Äì if the 50% default wasn‚Äôt already low enough.
