ken sheridan say: You'll find a number of functions for doing working days arithmetic in Workdays.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes the following function for subtracting working days, optionally taking account of public holidays, in this case for a single country with a single set of public holidays:

Public Function WorkDaysSubtract_SingleCountry(varDate As Variant, _
                            intDays As Integer, _
                            Optional blnExcludePubHols As Boolean = False)

    Dim dtmWorkDaysSubtract As Date
    Dim strCriteria As String
    Dim n As Integer
    
    If Not IsNull(varDate) Then
        dtmWorkDaysSubtract = varDate
        
        ' increment date, excluding weekends and holidays
        For n = 1 To intDays
            dtmWorkDaysSubtract = dtmWorkDaysSubtract - 1
            
            ' exclude weekends and public holidays if required
            If blnExcludePubHols Then
                strCriteria = "HolDate = #" & Format(dtmWorkDaysSubtract, "yyyy-mm-dd") & "#"
                Do Until WeekDay(dtmWorkDaysSubtract, vbMonday) < 6 _
                    And IsNull(DLookup("HolDate", "PubHols", strCriteria))
                    dtmWorkDaysSubtract = dtmWorkDaysSubtract - 1
                    strCriteria = "HolDate = #" & Format(dtmWorkDaysSubtract, "yyyy-mm-dd") & "#"
               Loop
            Else
                strCriteria = "HolDate = #" & Format(dtmWorkDaysSubtract, "yyyy-mm-dd") & "#"
                Do Until WeekDay(dtmWorkDaysSubtract, vbMonday) < 6
                    dtmWorkDaysSubtract = dtmWorkDaysSubtract - 1
                    strCriteria = "HolDate = #" & Format(dtmWorkDaysSubtract, "yyyy-mm-dd") & "#"
                Loop
            End If
            
        Next n
        
        WorkDaysSubtract_SingleCountry = dtmWorkDaysSubtract
    End If
    
End Function

In your case the expression to determine if an order date is more than three working days earlier than the current date would be:

[OrderDate] < WorkDaysSubtract_SingleCountry(Date(),3)

jim gordon mvp say:

It depends on what name you are talking about.

PowerPoint names each object that's included within a presentation. You can get this information using VBA.

If you want to know the FILE name of a picture or movie object, you can get that information from Finder provided you save your presentation in a current file format (.pptx not the old .ppt for example).

  1. Quit Powerpoint.
  2. In Finder, change the file extension from .pptx (or .potx) to .zip
  3. Double click the zip file to display the contents (a folder will display)
  4. Navigate within the folder to the Media folder.

That's where you will find copies of the original content, and the file names will be whatever they were when you inserted them.

When you're done, in Finder, rename the file what it was in the first place.

If you're the type that will fiddle around with the contents, make a copy of the presentation before you go mucking it up.
