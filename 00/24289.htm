Word 2007 Cross-Reference giving error message

I have created several cross-references to Exhibits and Headings in a long document (Word 2007 - not tracking changes). I haven't changed the reference source, so I can't determine why this is happening. Even when I do change the reference source (move the heading or the exhibit) the cross-reference sometimes does what it is supposed to do and tracks the new number when I update it and sometimes it doesn't. How can I fix this problem? Thanks.

cindy meister say:

Hi Dakota

When this error occurs, it means the bookmark, which the cross-reference uses to find the source, has been deleted. If you haven't heard of this before, that probably doesn't mean a lot to you, I know :-)

In the Insert tab, look for the Bookmark button and click on it. You may not see anything listed, and probably not the cross-reference bookmarks since they will be hidden by default. At the bottom of the dialog box there's a checkbox to display these. Click it three times (as it usually doesn't "take" on the first activation) and you should get a list that includes things that start with _Ref, followed by a longish number.

These are the bookmarks Word generates when you create a cross-reference.

Now close the dialog box and go to a cross-reference that's giving an error. Click in it and press Shift+F9 to toggle on the underlying field code. This will look something like { Ref _Ref######### }. You can see the resemblance of the last part to the list you saw in the Bookmark dialog box. These _Ref bookmarks are what have gone missing.

You can either delete the cross-reference and create a new one. Or you can copy the _Ref###### in the field code, select the correct text in the document, Insert/Bookmark and Ctrl+V to paste in the _Ref#####. Click on Add to recreate the bookmark.

As to "why" the bookmarks disappear, that's not easy to say without watching you edit the document :-) It could happen if, for example, you select the text in question, then overtype.

Cindy Meister, VSTO/Word MVP
