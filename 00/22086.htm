Activating a Mobile hotspot on the Samsung mobile phone grants shared access for other devices to the Mobile/LTE network's Internet access.  The phone becomes a Wi-Fi router to the mobile network.  Any Internet access through it by other devices consumes mobile data at those rates.

ronrosenfeld say: Perhaps:

=INDEX(INDIRECT(ADDRESS(1,MATCH(Hedge!Z3,IntradayData!$1:$1,0),,,"IntradayData")
&":"&ADDRESS(ROWS($A:$A)-1,5+MATCH(Hedge!Z3,IntradayData!$1:$1,0))),
MATCH(Hedge!Y4,INDIRECT(ADDRESS(1,MATCH(Hedge!Z3,
IntradayData!$1:$1,0),,,"IntradayData")&":"&ADDRESS(ROWS($A:$A)-1,
MATCH(Hedge!Z3,IntradayData!$1:$1,0))),0),6)

This assumes that Z3 and Y4 are both on Hedge.  The formula can be on any sheet in the workbook, though.

Note the final 6 in the formula.  An offset of five results in referencing the sixth column.  If you really mean column 5 and not an offset of 5, merely change that 6 to a 5.  You don't need to change anything else.

You may also need to make the references to Z3 and Y4 absolute instead of relative, depending on how other things are constructed and if you want to fill the formula in some direction.


joeu2004 say: shnickins wrote:
Sorry for confusion... I want to match Y4 (sheet Hedge) which is a date value in the column that is the match of the value in Z3 (sheet hedge).  So if Z3  = pear... and pear is found in sheet IntradayData on the 10th column (always row 1).  Then I want to find Y4 in column 10 on sheet IntradayData and find the offset which is always 5.

It would be best to do the IntradayData lookup only once.  Suppose Z4 contains your formula:

=match($Z$3,IntradayData!$C$1:$ll$1,0)

Then you can use the following VLOOKUP formula:

=vlookup(Y4,index(IntradayData!$C$2:$II$1000,0,Z4):index(IntradayData!$H$2:$IN$1000,0,Z4),6,1)

The use of INDEX avoids volatile functions like INDIRECT and OFFSET.

You can replace Z4 with the MATCH expression.  But doing the same lookup multiple times is efficient, especially an exact-match lookup.

Change the 4th VLOOKUP parameter to 0 if you want an exact match.

Note that columns H and IN are 5 offset from columns C and II respectively.

You can use $C:$II instead of $C$2:$II$1000; similarly for $H:$IN.  I avoid such open-ended ranges, especially when doing a non-exact-match lookup.

***EDIT***    Ignore the comment above.  I wrote it when I thought the range would be C1:II1000.  Later, I realized that since row 1 is your MATCH lookup table, your dates would start in row 2 or higher.

david a matson say: Greetings CardHost!  Welcome to the Windows Mobile Total Access forums!

There is a difference between copying files and converting files.  If the files you are copying do not need to be read by your PDA, and you are just using the PDA to ferry the files from one PC to another, then you may wish to have ActiveSync treat your PDA as a memory stick instead.  This can be accomplished for WM6 (and maybe WM5) by going to Start > Settings > Connection Tab > click on "USB to PC" icon, and change the check from ActiveSync to Disk Drive.

If you need to actually view or interface with these files on your PDA, then you will need to successfully convert the files from regular PC format to PDA format, which does create a situation where data can be lost or not converted.  For example, a complex Excel spreadsheet will not be easily converted from PC to PDA format.  The bulk of the data might be there, but some functions won't work (because the PDA version of Excel can't possibly duplicate the number of functions included on the desktop version of Excel).
