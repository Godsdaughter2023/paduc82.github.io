karl timmermans (outlook mvp 2012-2018) say:
Since this involves an IMAP account and there is only one contact folder, it means that the contact folder includes the words "This Computer Only" so going to start with a suggestion unrelated to the export problem.

You really don't want to maintain your contact/calendar data in folders labeled as "This Computer Only" since if the Imap account is ever removed, the underlying .ost file in which folders reside will also be automatically deleted. Also, if the .ost file cannot be found for some reason, Outlook will re-create the .ost file and re-sync the email folders with the server. The contact/calendar folders will be empty. You can also not simply make a copy of the ost file as a backup. An .ost file can only be opened/read in Outlook by the same Outlook profile/email account that originally created it.

So with all of the above out of the way, the suggestion to protect your contact/calendar data would be to export the "This Computer Only" folders to a PST file and add that PST file to the Outlook profile. Once done, make the PST file the "default data file" for the profile which will be where the default contact/calendar folders will be located.

To return to the export problem, to eliminate the .ost file as having any kind of issue as it relates to contact/calendar data, once the above is completed, confirm that the contacts in the PST file have the expected data. Once confirmed, export the contacts from the contact folder in the newly exported/added PST file.
