bob jones aka: cybertaz say:

Lines are one-dimensional shapes which do not have a Rotation property. They are anchored at their endpoints & pivot at them, rather than rotating around a midpoint.

After-the-fact you should have no problem with what Doug advised... But I suggest dragging one end or the other to get the line in a 'generally' horizontal perspective first. Otherwise, you may have a surprising result. Just use the corresponding value field on the Shape Format contextual tab. (Note that there is no Height value displayed on the Format Shape Task Pane.)

To create a horizontal line, hold the Shift key before you drag with the drawing tool... Keep the key pressed until you release the mouse button. The Shift key constrains the line to the direction you begin dragging.

Likewise, to change the length of an existing line without pivoting it off-axis, hold Shift while dragging.

BTW: For the complete array of relevant tools, right-click the object & select the Format [whatever] command or choose it from the Format menu to display the object's Task Pane.

psychomania---01 say:

The above answer is incorrect. That number would be the same for someone who had updated to v50 or someone who had v50 out of the box.

The custom version number is a more specific number that takes into account regional (and sometimes network) variances.

In the past on some models there has been updates that don't change the software version but do change the custom version number. These are believed to have been extremely minor updates, perhaps to correct a spelling mistake in a certain language.

When talking about your phone on forums it's better to refer to your firmware just using the shorter software version number.

Also the bit that says "not updated" is normal. If you update via FOTA it shows the date you updated. If you update using NSU or reset the phone it will always say not updated.

A formula that will automatically add the current date to Cell A when - Microsoft Community

pgagne say:

A formula that will automatically add the current date to Cell A when I enter a number in Cell B

I an trying to create a Formula that will automatically add the current date to Celll A when I enter a number in Cell B
ron coderre say:

If you use a formula to set the value of Col_A to TODAY()...the value will change to the current date whenever you open the workbook.

Perhaps a better approach might be to engage a VBA macro that sets the Col_A value to the current date, only if the entry is in Col_B AND the Col_A cell is currently blank...Otherwise, it leaves the previous Col_A value.

• Right-click the sheet tab...Select: View Code
• Copy the below code and past it into the Sheet module:

Private Sub Worksheet_Change(ByVal Target As Range)
    With Target
        If .Column = 2 And Target <> "" Then
            'Entry is in Col_B
            With Range("A" & .Row)
                If .Value = "" Then
                    'Col_A cell is blank...enter today's date
                   .Value = Date
                End If  'blank cell test
            End With  'corresponding Col_A cell
        End If   'Col_B test
    End With    'Target
End Sub

Now...whenever you enter a value in a Col_B cell
...if the corresponding Col_A cell is blank, it is set to the current date.

Is that something you can work with?
