Almost certainly somebody has posted this elsewhere, however!

This is a pretty messy and not entirely satisfactory solution but -

After the search results have been returned, select them all and copy them into a new empty separate directory. First snag, duplicate names - hopefully the duplicates will be numbered in the order encountered.

Then search for the CMD program and open a DOS window.

Navigate to your new folder using good old fashioned DOS commands Drive letter:  and CD, then screen list the files, dir *.*, to check. Then pipe the screen output to a text file - dir *.* > filename.txt.

The last command can be used crudely to include only certain file types i.e. dir *.rtf  (and the txt files later concatenated).

Close the DOS window by entering exit.

The txt file(s) will need a bit of manipulating (in Excel?) to tidy up.  Not an ideal solution by any means  but it's an improvement on squinting at screen dumps.  Any corrections/suggested improvements gratefully appreciated.

cmdrkeene say:

Two conditions must be met for a program to appear in that list:

1) The program must have registered itself as being capable of handling at least one file type

2) At least one other program must have registered itself of being capable of handling at least one of those same file type(s).

 

If a program is the only present program capable of handling a specific file type (such as .PPT for PowerPoint files), that program will not appear in the list because there is no alternative.
