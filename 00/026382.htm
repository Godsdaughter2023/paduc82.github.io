langware say:

In the new outlook user interface, you can "pin" messages and you can "flag" messages. Unfortunately, Microsoft was not consistent ... so "flagging" messages in the old interface now correspond to "pinning" messages in the new interface.

Look at the banner above your inbox. If it says Outlook Mail or Outlook Email, or Outlook, then your account has been moved to the new Outlook interface (and there is no way to go back). If it says outlook.com, then your account is still on the old interface. I assume that your account is on the new interface ....

Flagged messages in the new outlook interface behave differently than flagged messages in the old interface ( thanks Microsoft ). 

In the new interface, flagged messages do not appear at the top of the inbox. If you want a message to appear at the top,  you have to use the "pin" feature .... but note that the pin feature is only available when your inbox is sorted by date!

To pin a message, select the message and click the tiny icon that looks like a push-pin (to unpin, just click the pin icon again).

You still can see all  of your flagged messages ... just expand the "All" item (top right of inbox) and click "flagged" ... your flagged messages will be displayed (but only flagged items ... not the rest of your inbox with flagged messages on top).

Microsoft did not consider their customers when they changed the behavior of flagged messages in the new outlook interface.

graham mayor say:

You could paste the Excel data using an Outlook macro to insert the table and apply the formatting e.g. as foillows. I think that would be preferable to Outlook 2013.

Change the bold bit to reflect the name of your table style and add the macro to a QAT (Quick Access Toolbar) or ribbon button.

Sub PasteTable()
Dim oRng As Object
    On Error GoTo ErrHandler
    If TypeName(ActiveWindow) = "Inspector" Then
        If ActiveInspector.IsWordMail And ActiveInspector.EditorType = olEditorWord Then
            Set oRng = ActiveInspector.WordEditor.Application.Selection.Range
            oRng.Paste
            oRng.Tables(1).Style = "Medium Grid 3 - Accent 1"
        End If
    End If
lbl_Exit:
    Exit Sub
ErrHandler:
    Beep
    Resume lbl_Exit
End Sub

Juriasu:

As you've discovered, certain fonts associated with particular languages are in optional packages. In past versions of Windows, these fonts would have been installed on every device, whether you used those languages or not. In Windows 10, we don't install them by default so they won't be consuming disk space or other resources for users that have no need for them. 

Most likely what is happening is that Chrome is relying on fonts from those packages for font fallback.

As you've also discovered, adding to your profile a language associated with a given font package will automatically cause the font package to be installed. But you can also go into Settings > System > "Apps & features" > "Manage optional features add a feature" to install the font packages without having to enable any additional languages.

You can see a list of all of the optional font packages here:

https://msdn.microsoft.com/en-us/goglobal/bb688099.aspx?f=255&MSPPError=-2147217396#W10

Note that the "Pan-European Supplemental Fonts" package has fonts that are entirely new for Windows 10. So, neither Chrome or any other existing apps would rely on those fonts for font fallback.

I hope this clears things up for you.

    bob jones aka: cybertaz say:

    Standard installation of Mac Office 2016 includes German Proofing Tools along with those for a number of other Languages:

    Whether they are invoked is determined by whether the text in the document is marked as being German (or whatever). IOW, Word will use the correct Proofing Tools if it knows which ones to use, but the user has to make that known. Otherwise, the Spelling & Grammar checker uses those for the Language specified as the default.

    Select the text, then choose German from Tools> Language as illustrated by Phillip's screen shots. The Language dialog can be accessed from either:

    1. The Tools menu
    2. The Language tool on the Review tab of the Ribbon
    3. The Language tool on the Status Bar (if it has been added)

    When using multiple Languages in a document it's recommended to use Styles which include the Language specification. Apply the appropriate Style to the respective passages.
