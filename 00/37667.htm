cubelodyte say:

@BMHSONIC

Theoretically, the key your Windows 10 ISO requires is the key for your Windows 7 install. 

I say "theoretically" because there are a few caveats; you can't use a key for a group (corporate) license, student licence, or MSDN license. If your copy of Window's 7 doesn't fall into any of these categories, this should work for you. 

Some things to keep in mind: 

  • If your HP is more than a year or 2 old, there's a strong possibility that Windows 10 (WinX) will attempt to install video drivers that don't work. I've experienced this with ATI/AMD video cards. If you can catch it early enough, it's possible to tell the WinX Update to never install hardware drivers by searching for "change device install..." in the Settings window.
    • Even if you manage to complete the install of WinX, it can easily be broken by installing older versions of software. In my case, it was Adobe CS5. Adobe's official position is that CS5 is not supported on WinX and my experience is that it will break all functionality that uses the "Metro" interface (this includes all native apps like Edge, Maps, etc. as well as the Settings window). This has also been reported for Lenovo notebooks using certain pre-installed feature software.

My Experience:

13 upgrade/install attempts over 10 days (more than 70 hours total)

My first upgrade was on a 1 year old Lenovo Ideapad that had come with Windows 8, but recently had reverted to Windows 7 Ultimate after a hard drive crash and replacement. 

This PC had very little legacy software installed at the time of the upgrade.

This upgrade went without a single problem and is happily running Windows 10.

Upgrades 2-12 were on a 4 year old HP DV7-4177nr running Windows 7 Ultimate.

In some ways, my experience with the older notebook has been the opposite of yours:

  • When I run an upgrade on my existing system, the install gets to about 50% and freezes on a black screen. If I force shutdown and power back up, it will continue to 85% and then freeze again; this time reverting back to previous version after force shutdown.

    Many report that you just have to wait out the freezes, but I've waited stretches of over 6 hours and the PC starts to get pretty hot and shows no sign of activity whatsoever.

  • I made 6 consecutive attempts to perform the upgrade from a clean install of Win7. In each case, the upgrade to WinX completed successfully, but decayed rapidly.

    It took 2 installs to determine that working ATI/AMD video drivers were being overwritten by non-working drivers. It took the other 4 to figure out that I had to disconnect the internet during the upgrade process to prevent this overwrite until I had the chance to block hardware driver installs during the automatic, nearly continuous, and otherwise totally unstoppable Windows Update process.

Conclusions:

  • Microsoft has little or no control over the myriad manufactures of legacy PC hardware and software with regard to support for OS upgrades.
  • Machines old enough to have come with Windows 7 are likely to have significant hardware/driver conflicts.
  • PCs running older versions of professional or enterprise software are likely to have significant conflicts with built-in WinX features.
  • Neither of these last 2 observations are likely to be fixed any time soon.
  • Get used to Windows 7, it's what you'll be using until you buy a new machine.
  • If you do buy a new machine give up any notion of installing your legacy versions of enterprise software.

wilfredo_soto say:
Hi, Wolftalker
I am Wilfredo an Independent advisor


For Administrator permissions you have the following 

1. Press and hold the Shift key (the one with the up arrow) while you restart the computer. The device will restart immediately and a screen with different options will appear.

2. Go by clicking in this order the different options that appear on the screen until you reach Start Configuration:
Solve problems
Advanced Options
Start configuration

3. On the next screen, click on Restart. When restarting, the screen with startup options will be displayed. Press the 4 key on the keyboard (Enable safe mode)
The system will now boot in safe mode. Observe in the login screen how many users appear to be able to start in safe mode. Log in to your account.

4. Once you have logged in to your user account, press the Windows + R keys. In the Open box type

netplwiz

and press Enter. The User Accounts window will be displayed.

5. Select your user (who currently does not have privileges) and click Properties:

6. Click on the Group Membership tab and select Administrator. Apply the changes and restart the system

I hope this can help you
