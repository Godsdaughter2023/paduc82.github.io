Add year to Month and Day date in a cell

The first column has the month and year

ROW 1 4/01
ROW 2 4/03
ROW 3 4/07

How do automate adding the year to multiple rows so that all cells read 04/01/10 ?

Thanks in advance for your help and expertise!
joeu2004 say: Darrell wrote:
The first column has the month and year
ROW 1 4/01
ROW 2 4/03
ROW 3 4/07
How do automate adding the year to multiple rows so that all cells read 04/01/10 ?

That depends on why the cell value looks like 4/01 in the first place.

If 4/01 is in A1, enter =ISNUMBER(A1) into a parallel cell.  Drag the formula down the column to test all such cells.  What does ISNUMBER return?

If ISNUMBER returns TRUE, the cell probably already contains an Excel date.  To see the year as well, select the cell(s), click on Home tab and select the Short Date format from the pull-down menu in the Number section.

If the dates do not show the intended year, you have bigger problems.

If ISNUMBER returns FALSE, enter the following formula into a parallel cell and drag the formula down the column:

=--DATE(2010,LEFT(A1,FIND("/",A1)-1),RIGHT(A1,2))

Then copy the new column and paste-special-value over the original cells in column A.

Note that I assume that all dates are intended to be in the year 2010.  If that is not the case, you need to make appropriate adjustments.

Caveat:  Be sure to make a backup copy of the Excel file before making any such changes.

This has been a common problem, one that was supposed to be fixed by 365 and "click to (not) run".  Sometimes it still occurs.

The situation is you buy a new computer with a factory installed (crapware) "free copy" of Office ,aka a "trial".  Then you buy and install and activate a legal copy of Office.  

The problem is that the when you try to activate your purchased and installed copy, the free trial 'hijacks" the activation attempt. The result is that you have 2 copies of Office installed, neither activated.

Soooo, to confirm this diagnosis:

First, log into the MyAccount website: http://office.com/myaccount

and confirm that your purchased copy has been activated and assoctiated to your email account.

Second, go into Control Panel > Programs and Features.  Look through the list of installed programs to see if you have 2 copies of Office installed.  Run the fix it to uninstall Office 2016.  In theory this should uninstall both.

Third, go back to MyAccount and install a fresh copy.

joeu2004 say:

Convert them to what: mm/dd/yyyy or dd/mm/yyyy?!

You wrote: ``Interestingly, all of the dates with "dd" numbers above 12 have dd/mm/yyyy format, and all of the dates with "dd" numbers below 12 have mm/dd/yyyy format.``

I suspect that they were input on a system where the actual regional Short Date form is mm/dd/yyyy. Thus, the dates in the form dd/mm/yyyy were treated as text values.

We cannot tell the difference between text and numeric data values in a CSV file because all data are written to the CSV file as they appear in the Excel cells.

Be that as it may, the issue now is: what is the regional Short Data form on your system?

If it is dd/mm/yyyy and you opened the CSV file directly in Excel instead of importing it and (perhaps) setting the column format to Text, then all of the data are numeric dates. Confirm by entering =ISNUMBER(A1) in a parallel column.

In that case, what you interpret as mm/dd/yyyy is actually treated as dd/mm/yyyy by Excel.  So you might enter the following in a parallel column:

=IF(DAY(A1)>12, A1, DATE(YEAR(A1),DAY(A1),MONTH(A1))

Copy the resulting values and paste-value in column A.
