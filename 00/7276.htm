AutoRecover Time interval - Microsoft Community

shemomasire say:

AutoRecover Time interval

Hi. I am familiar with how to change the autorecover time interval in MS word 2013 but i notice it goes back to the default 10 minutes after shutdown or restarting the machine. Is there a way i can permanently change it to my desired time interval? Thanks. 

jay freedman say:

Something (maybe an add-in or maybe a permission problem) is preventing Word from writing the value from the Options dialog to the registry entry that should store it permanently.

You can change the registry value directly, and later you can investigate what caused the problem.

First, follow the instructions in https://support.microsoft.com/en-us/kb/322756 to back up at least the key HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Word\Options and possibly the entire Word key. That will create a file that you can use to restore the original data if necessary.

While the key HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Word\Options is selected, the right side of the registry editor shows the list of values that it contains. Double-click the name of the AutosaveInterval value, which opens a small dialog box.

Select the Decimal option button, and type the desired number of minutes into the Value data box. Click OK, verify that the value has changed (the right-hand side shows both the hexadecimal and decimal values), and close the registry editor. Restart Word and verify that the Options dialog shows the new value.

For troubleshooting possible add-in problems, follow http://support.microsoft.com/kb/921541 starting at the heading "Disable the Startup folder add-ins"about halfway down the page.

I have a brilliant macro for MS Word that will copy a term that I have highlighted in Word and search for it in a free online English dictionary (in this case http://oxforddictionaries.com/).

The code is below. Can anyone please tell me what changes I need to make so that it will work for people who use Mac, too?

Thank you ever so much for any tips you can give me!

Tanya, Switzerland

Sub Oxforddictionaries()
'
' Oxforddictionaries Macro
' Oxforddictionaries created in 2015 by Tanya
' Searches the online Oxforddictionaries dictionary

' Macro works with MS Word and Internet Explorer

'
Dim theTerm As String ' Term to look up in dictionary
Dim MyUrl, myURL1 As String ' The URL to open in Internet Explorer
Dim oIE As Object ' Internet Explorer

On Error GoTo MainStop

' If nothing is selected
If Selection.Type = wdSelectionIP Then
  ' Select the term where the cursor is located
  theTerm = Selection.Words(1).Text
Else
  ' Use the current selection
  theTerm = Selection.Text
End If

theTerm = Replace(theTerm, vbCr, "")  ' Remove any soft returns in the phrase that you have highlighted
theTerm = Replace(theTerm, vbLf, "")  ' Remove paragraph breaks in the phrase that you have highlighted
theTerm = Trim(theTerm) ' Remove any spaces

MyUrl = "http://oxforddictionaries.com/search?searchType=dictionary&isWritersAndEditors=true&searchUri=All&q="
MyUrl = MyUrl + theTerm
MyUrl = MyUrl + "&contentVersion=WORLD"

Set oIE = CreateObject("InternetExplorer.Application")
' Activate the Internet Explorer window i.e. bring it to the foreground
    oIE.Visible = True
' Navigate to the URL created above
    oIE.Navigate (MyUrl)
   
MainStop:
If Err.Number <> 0 Then
MsgBox "This macro searches the online Oxforddictionaries dictionary. First you need to open an MS Word document and place your cursor over the word you wish to look up... " & vbCr & "" & vbCr & "For an updated version of these macros, please contact me."
End If
End Sub

john korchok say:
Your macro uses ActiveX to call Internet Explorer, but ActiveX doesn't exist on the Mac. Neither does Internet Explorer anymore, for that matter.

daniel g. grau say:

You already have access to that one and many other dictionaries, courtesy of Apple's Dictionary application. Just highlight a word, press Option Command D and the dictionary will open to that word (by the way, this should work in any application). Once there, check the preferences to select the dictionaries you want to activate.

Word also includes the Encarta dictionary and other goodies. Highlight a word, then:

  • Select Tools > Dictionary or
  • Right-Click (or Control Click), select Lookup > Definition or
  • Press Command Option Shift R (you can change this awkward combination)

There's a web-search feature as well.

Having said that, the macro can be adapted if you change these:

Set oIE = CreateObject("InternetExplorer.Application")

To this:

MacScript "tell application ""Safari""" + vbCr + "open location """ + MyUrl + """" + vbCr + "end tell"

Just to give you an idea of how I tweaked this method in another macro that can be used by translators for terminology searches, here is a search that can be launched from within Word with a single click. It basically searches for a highlighted term on 3 different multilingual websites via Google:

https://www.google.ch/search?hl=en&q=%22insurance%20wrapper%22+site:ubs.com+OR+site:credit-suisse.com+OR+site:snb.ch&ei=9lsLVY_XLaHlywOaiIDQAg

This is the code that works with MS Word and Firefox and which I would like to modify so that it works on a Mac:

Sub Banks()

'

' Banks Macro

' Macro created in 2015 by Tanya.

' This macro works with MS Word and Firefox browser.

' It searches for your highlighted term on various multilingual

' banking websites via Google.

' For the macro to work, you may need to modify the path of the file firefox.exe according

' to its location on your own computer. See "C:\Program Files\Mozilla Firefox\firefox.exe" below.

'

Dim Newsite As Variant

Dim theTerm, MyUrl As String

On Error GoTo MainStop

If Selection.Type = wdSelectionIP Then

  theTerm = Selection.Words(1).Text

Else

  theTerm = Selection.Text

End If

theTerm = Replace(theTerm, vbCr, "")  ' Remove any soft returns in the highlighted phrase

theTerm = Replace(theTerm, vbLf, "")  ' Remove paragraph breaks in the highlighted phrase

theTerm = Trim(theTerm) ' Remove any extra spaces in the highlighted phrase

theTerm = Replace(theTerm, " ", "%20")  ' Remove paragraph breaks in the highlighted phrase

theTerm = Replace(theTerm, "ä", "%E4")  ' Replace any ä characters with code the browser will understand

theTerm = Replace(theTerm, "ö", "%F6")  ' Replace any ö characters with code the browser will understand

theTerm = Replace(theTerm, "ü", "%FC")  ' Replace any ü characters with code the browser will understand

' Construct the URL to search various websites via Google

MyUrl = "https://www.google.ch/#hl=en&q="

MyUrl = MyUrl & "%22" ' add opening quotation marks

MyUrl = MyUrl & theTerm ' add closing quotation marks

MyUrl = MyUrl & "%22"

MyUrl = MyUrl & "+site:ubs.com+OR+site:credit-suisse.com+OR+site:snb.ch"

' Launch the Firefox browser and navigate to the URL created above

Newsite = Shell("C:\Program Files\Mozilla Firefox\firefox.exe " & MyUrl)

MainStop:

If Err.Number <> 0 Then

MsgBox "This macro searches various multilingual Swiss bank websites through Google for the term that you have highlighted in Word. First you need to highlight a term..." & vbCr & "" & vbCr & "For an updated version of these macros, visit http://www.facebook.com/groups/TranslatorsSwitzerland . If the macro does not work, you may need to modify the location of your file firefox.exe in the macro code."

End If

End Sub

Internet Explorer Feedback - Microsoft Community

jordan313 say:

Internet Explorer Feedback

I would like everyone to know that this is a rather long post, so bear with me. I will discuss my ideas and then get to the bugs in IE.


Internet explorer on the desktop looks and feels the same as it did years ago. It is old and its interface looks it.  What IE needs is a complete UI overhaul that will simplify the browser. I expected something very different when switching to IE 11, but it is the same except for a setting or two and some "under-the-hood" features. I really like the setup of the top bar when you switch to full screen mode; everything is on one bar, which looks much simpler and cleaner than when you are not using this mode. However, I don't always want to be in full screen mode when using the desktop IE (of course, metro IE is a different story). Desktop IE could also use some features of Windows 8's modern UI: Dropdown menus could use the modern style found in metro IE (or something like it), which would look better than the old menus in the browser. Also, when you click the Tools cog, maybe a menu like the one you see when you click "Options" under an app's settings will be displayed, with everything in the background darkened (to help compensate for any color/graphics disagreements). Such a  menu (or something like it) can display everything in that dropdown in style. The Feeds, Favorites, and History icon could be made simpler if it only addressed favorites and nothing else; with the star as the icon, this makes sense, and there is a good one-click means of viewing favorites in the metro IE11. The History and Feeds portions of the menu can be moved to the Tools icon, which should be renamed "Settings" for clarification purposes. The star icon could be used just to view all favorites in something like a file explorer rather than the list that exists now; my list of favorites isn't fully organized and scrolling through them now in IE is rather inconvenient for me. The process of adding a site to your "Favorites" can be simplified by creating a feature in the browser that allows you to do so when you click the icon for a webpage on its tab, and you should then be able to choose what Favorites folder to put the page in from there. Internet explorer, like every browser, needs a tab grouping feature, which will be very, very useful when managing multiple tabs in it. I do not have much to say about the metro IE11 except for that the New Tab and Tab Tools icons can be repositioned next to the right-most tab opened in the browser, to help keep things together and make it similar to its desktop counterpart (and to other browsers, in that respect). Maybe desktop IE can adopt this one's navigation arrows on the side of the screen as an option. Lastly, when in full screen mode, the bar at the top should have smoother auto-hide graphics; they much better than those in IE 10, but are slightly choppy still.


When using IE, sometimes page elements such as pictures and text fields would become blacked-out or would become all one color. Also, pages may sometimes refuse to load, do so slowly, or close out randomly. Sometimes, even parts of entire pages won't work right. I cannot purposely duplicate these events. What's more is that when you have more than one tab open and close one of them, the space that was occupied by all of them before closing the tab is now occupied by the remaining tabs for a while. A similar effect happens when you put your tabs on their own row and then set them back to their normal row.


Please comment! And remember these are only my opinions of IE, so try not to be harsh!
