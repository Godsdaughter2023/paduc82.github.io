flu8 say:

There is an ongoing bug in Windows 10 Anniversary Update version 1607 that is known in current builds where copies or moves of folders over different partitions causes the copied folders to lose their attributes (such as "+r" or "+s"), which are needed for customized icons for folders to work.

Affected folders can be fixed using the "attrib +r <folder>" command in a console/command prompt window for a single folder (if the folder contains spaces, enclose the folder name in double quotes, ", eg: attrib "New Folder" +r).

After using this command for the copied folder, assuming that folder contains the hidden, system desktop.ini file, the referenced custom icon for the copied folder appears on refresh.

Hey Vic, the indexing service should return the correct results with the *ID*.xlsx phrase you're using.  Do keep in mind that the search only searches the folder you're in and subfolders.  If you want to search your entire C: drive, be sure to open up 'Computer' then go inside the C drive before you type your search phrase.
