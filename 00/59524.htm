This is my full post on WinSxS

 

WinSxS Folder

 

The WinSxs is the central file repository for Windows. All files installed are placed into the WinSxS folder including updated files. So you'll have lots of different versions of each file allowing you to roll back Windows updates and service packs to any previous state.

 

The current version of a file is hardlinked to the appropiate place in the rest of the file system. A hardlink is a file name. All files have at least one hardlink. Windows system files have two hard links, one in WinSxS and one in (usually) c:\windows\system32. Both names point to the same bytes on disk.

 

As the files have two names they are counted twice by explorer, once in System32 and once in WinSxS.

 

There are other uses for it as well. It allows multiple programs to use different versions of dlls.

 

Ensure Pending Operations are Completed

Try a reboot. Before rebooting set the trustedInstaller service (DisplayName Windows Modules Installer) startup to automatic. Normally it's manual unless an operation such as windows update requires boot time install.

Sizes Are Not What They Seem

Files in WinSxS are usually hardlinked files. A hard link is a filename. Every file has at least one hard link. Files in system32 and files in WinSxS are usually hardlinks to the same file. Therefore you count the same file in system32 and WinSxS twice.
