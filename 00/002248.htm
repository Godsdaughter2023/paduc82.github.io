Can a field retrieve live data from an SQL database? - Microsoft Community

guitar-picker say:

Can a field retrieve live data from an SQL database?

Suppose an SQL database maintains real-time totals such as "total widgets produced since 8AM today" = 278.

Is there a way to embed a field into a word.doc that, when the field is updated, it will go out, retrieve, and display today's up-to-the-moment sales total right in the word.doc?

This could help a lot of people who have spaces for stats like this, embedded in proposal templates, but have to tediously retrieve the info time after time.

a. user say:

It depends on the SQL database, and exactly what you mean by "real-time totals"

As Angela has explained, it is sometimes possible to use a DATABASE field to do this, but 
 a. you have to be able to write the SQL necessary to retrieve the item of data you want, and you'll need the necessary permissions to run that SQL. If the total isn't /stored/ in the database, you would typically need to write a SQL query such as "SELECT sum(widgets_produced) FROM some_table WHERE (some sort of date criteria)"
 b. A DATABASE field normally inserts a table of data into your document, which means that the data would not appear "in-line" with your text. The workaround for that used to be that if your DATABASE field only returned a single datum (i.e., one row, one column), Word inserts "inline" rather than as a table. To do that, you have to ensure that the column headings aren't included in the result, so do not include the \h switch in the field. Unfortunately, after about Word 2000, Microsoft changed the way DATABASE fields worked, and the field result is now either preceded or followed by a paragraph mark. You can get rid of that if the result is a number, by inserting a numeric formatting switch such as \#0 , or if it is a date, by inserting a date switch such as \@"DD MMM YYYY", but not if it is any other kind of text. In that case, you might be able to achieve want you want by using a query such as 

SELECT 'total widgets produced since 8AM today =' + sum(widgets_produced) FROM some_table WHERE (some sort of date criteria)

(but you might need to use a different method to concatenate the 'total widgets...' string and the number, depending on the database you are using).

There are other possibilities, but it depends on what technologies you can turn to and how much effort you are willing to put in.

andrea jones - all about resources say:

Yes, you can do this using a Database field which you can insert using the Field item on the menu under the Quick Parts button which you should find on the Insert ribbon.  However, you may find that it you set the query to only display certain fields or match where a field contains a certain value then the result doesn't display correctly due to a bug with the SQL Jet Engine.  To remedy this, after inserting the database field into your document ignore any dialogues offering the chance for you to match unrecognised fields, etc and then press Alt+F9 to display the field code.  Read through the contents of the DATABASE field to find the SELECT statement and then find the first comma in this statement, e.g. 'SELECT Total, Date ............

Place an extra space before the comma after the first field in the SELECT statement (Total in the example above) and then press Alt+F9 to turn off display of the field code.  Now press F9 to update the field and you should see the result.

Hi CSD Surface Tester,

Thank you for contacting Microsoft Support.  I see you have some questions about the developer preview and future updates.  I'd like to try and clarify a few things and see where we go from there.

So the thing to keep in mind with the developer preview and stability is that as a preview, the firmware still needs carry the updated drivers for the hardware on the phone, which has to come from the OEM.  Think of it like formatting your PC, and Windows installs the basic sound and video drivers so you at least get the basic sound and graphics display, but they won't work at their best and may have performance issues until you install the drivers from the manufacturers.

Now in terms of updates for the developer preview, you can still receive the official updates released for consumers by your mobile carrier, as well as developer preview updates.  When you uncheck the box to participate in the developer preview app, all that does it keep your phone on it's current build.  You can then check for updates that your mobile carrier provides.  Now, if your mobile carrier has release an official build, but your phone can't install it because it's on a developer preview build, you may have to try contacting the manufacturer and see if you can roll back the phone to a previous revision.  Just note that when rolling back, it typically involves a factory reset so you'll want to back up your phone accordingly.

steve rindsberg say:

A lot may depend on the inner workings of the LMS, so you may also want to ask their tech support people about this also.

I assume that the show is self-running via timed automatic transitions.  In that case, how about this:

Add a slide to the end of the presentation and HIDE it. We'll call this the Pause slide.

It can have whatever content you like, but should have no links other than the one described below.

The Pause slide should have NO automatic transition. 

Add a rectangle that covers the entire Pause slide, assign it a "Last slide viewed" action setting, and  a 99% transparent fill/no outline (ie, make it invisible).  That way, when the user clicks on it, the action setting will take them back to the previous slide they were viewing.

On whatever slides you want to give the user a pause opportunity, add an action button or some other shape and assign it a hyperlink to the Pause slide.

This should work as you want it to *in PowerPoint*.  Whether it works in the LMS will depend on how well they support PPT's features.
