LAt's say the list of contractors is in A2:A100 and the open/closed info in B2:B100.

The following formula will count the number of "open" items for a contractor whose name is in K2:

=SUMPRODUCT(($A$2:$A$100=K2)*($B$2:$B$100="open"))

and similar for "closed" items. This formula can be filled down if necessary.

I have updated the firmware to the latest version and I have deleted the old tracks and transfered them again, no joy. I have moved them from the sd card to the ringing tones file and so far it seems to have worked, but is now taking up phone memory!

Any thoughts would be appreciated

mccbleue say: There is an issue with the memory card on the 6500 Slide - there is a metal wedge near the camera hole on the battery cover which must make contact with the phone's circuit board in order for the memory card to be activated.

The problem with the design of the battery cover is two-fold: (1) It is a stainless steel panel which flexes and (2) it only clips nto place with one clip at the top end of the phone which is off-centre. The two things combined mean that the contact with the circuit board is often temporarily lost, which is why your ringtone resets.

You could take the phone to a service point and ask them to adjust it under the warranty to try to make the battery cover fit tighter - don't let them get away with reflashing the phone software and handing it back to you, this is a hardware fault not software.

Until a better resolution is available, it is best to keep files that you have set as ringtones, wallpapers etc. in phone memory rather than on the memory card.

RR130230625DO

The only thing that changes in sequence of 1 is the last 3 digits (625), as it needs to increase by one up to 680 or 690, it depends. Everything else remains the same. If I can get a simple formula so that I can use everyday that would be helpful.

benishiryo say: hi JR, although you are only expecting the last 2 digits to change (from 25 to 80 or 90), you can safely use:
=LEFT(A1,2)&MID(A1,3,9)+ROW(1:1)&RIGHT(A1,2)

this will keep adding 1 to the number & change the whole thing up to:
RR999999999DO

shane devenshire say:

I don't know of any Excel workaround other than upgrading to 2007 or 2010.  You don't need powerpivot with such a small data set and it won't work in versions earlier than 2007.

Access also has a PivotTable, although somewhat different than Excel.  I can't find specifications indicating what its limits are on unique records but since Access can handle far more records it may be prepared to hand far more unique values.

Is there any way you can group the data in the field that's causing the problem.  For example if it was a date & time field you could group it by day.  If a date field you could group it by Month or even week.
