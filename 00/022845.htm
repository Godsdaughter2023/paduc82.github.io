You can use File Explorer. Navigate to the documents folder then you have more than one option:

  • single click on the file name, press the <delete> button on your keyboard
  • single click on the file name, in the Home menu > Organize group, click on the "Delete" drop down button
  • right click on the file name, select "delete" from the context menu

You can perform many of the same actions in the File menu > open command (or <CLT><O> shortcut). The file Open dialog is actually a type of File Explorer window. So you can:

  • single click on the file name, press the <delete> button on your keyboard
  • single click on the file name, in the Organize drop down > click on the "Delete" option
  • right click on the file name, select "delete" from the context menu

stefan blom say:

Use File Explorer, as Rohn suggested. You can't delete a file which is open in some other program, though, which means that deleting a file from Word's Open dialog box only works if the file isn't open in Word at the time. Of course, it also means that you can't delete a file in File Explorer if it is open in Word at that time.

jm_sloan say:

Office has built-in, automagic version control.  Every time you save a file a new "version" is created. To view, compare, and restore previous versions of a document, select File -> Info -> Versions (File -> History in Office Mobile).

What you do not get with built-in version control is an implicit branching capability.

macropod say:

Office has built-in, automagic version control.  Every time you save a file a new "version" is created.

That is not a feature of Office. Microsoft Office 2010, 2013, 2016 - none have native version control. You can use Sharepoint though.
