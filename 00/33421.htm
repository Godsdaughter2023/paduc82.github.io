  1. Because the GTX card is physically in the base, you cannot attempt to use it, or set the machine up to use it if the screen is detached. You will get detach errors. When detached the video uses the Intel Graphics in the screen section.
  2. The GTX card only "kicks into action" when you run an application "that requires it to run", so if you run a 3D game or Autocad's Autoviewer, for example, it will start to run if the screen is attached to the base. If you want to detach after starting one of these applications you will need to shutdown the application and restart it after detaching, it will use the Intel graphics. When you run Dxdiag it reports the GTX card as a "3D rendering device" or something like that, not an alternative video card.
  3. I wanted to drive a 4K TV with the GTX card, because I paid for the card and wanted to use it, but I think the fact is the Intel video drives the 4K TV just fine, so maybe it really doesn't matter from a computer design point of view.

Based on what I have observed, if you are using 3D architectural software, and the screen is attached to the base, the GTX card should start to take over video duties.
