ken sheridan say: You can do both with the code in the button's Click event procedure rather than within the report design.  If the reports are designed so that they return all invoices if opened directly then you can select the appropriate report and filter it when opening it with code like this in the button's Click event procedure;

Dim strReport As String
Dim strCriteria As String

' determine report to open
If Me.CboInvPrefix = "TRS" Then
    strReport = "rptInvoicesTrust"
Else
    strReport = "rptInvoicesFoundation"
End If

strCriteria = "InvIDNum = " & Me.InvIDNum

' ensure current record is saved
Me.Dirty = False

' open report in print preview
DoCmd.Openreport strReport, View:=acViewPreview, WhereCondition:=strCriteria

The problem of the subreport not being restricted to the selected invoice number suggests that in the report design the subreport is not linked to the parent report.  The LinkMasterFields and LinkChildFields properties of the subreport control in the parent form should both be InvIDNum.

The above method of filtering a report is fine if you are opening it using the OpenReport method, but there are occasions when you might want to use the OutputTo method to send the report to some other output format.  the OutputTo method does not include a WhereCondition argument, so in this situation you need to restrict the report's underlying query by means of a parameter which references a control on the form.  You'll find an example as InvoicePDF.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

It sounds like you've done a nice bit of troubleshooting, eliminating both the network and USB ports as the issue and narrowing it down to the Xerox printer only.

However, let's make absolutely sure you have a clean set of drivers installed.

Disconnect the Xerox printer from the USB or Netwrok.  Go to Devices and Printers.  Hit Alt-F and click on Device Manager.  Click View, Hidden Devices.  Click on Printers, right click on the Xerox printer and click uninstall. If you have any other Xerox printers installed, you should also uninstall them and reinstall them later.  Manufacturers frequently share files for their printers and you may think you have tthe most up to date drivers but you don't. Close the Device Manager.

In the Devices and Printers window, select Add a printer, Add a local printer, select USB port and click Next.  Click Have Disk and install the (64 bit?) print driver for the Xerox Phaser 6280DN PS printer.  Do not connect the printer until prompted to do so.

If this doesn't do it and you're running the 64 bit print driver, try using the 32 bit print driver instead repeating the steps above.

If that still doesn't do it then are you running the Adobe PostScript 3 or PCL 6/5e emulations drivers?  Try switching.  Your best bet is probably going to be the PCL drivers.

I know what I'm proposing is laborious, but that's the nature of troubleshooting these types of things.
