daniel g. grau say:
When saving as a PDF document, most links become non-functional (particularly so if the links have a hierarchy). This is a known bug. One way to fix that:

  • Open the PDF file in the Preview application (you'll find it in your dock or in Applications > Preview)
  • Choose Tools > Text tool
  • Highlight the faulty link
  • Select View > Show annotations toolbar (the toolbar it will appear at the bottom of the document's window)
  • Click on the "link" symbol
  • A new dialog box will appear (if you see an "Inspector" window showing "No document," just click on the title bar)
  • You should select the last button at the top, "Annotations"
  • Rebuild your link (change the "Link type" option as appropriate)
June 26, 2015 – Note: When this post was written, Preview was at version 5.0 in Snow Leopard. Newer versions have dropped the hyperlink feature. They call this progress.

marshall barton say:

It's probably as simple as using both the report header and the Project group footer section's Print event to make the page header invisible:

    Me.Section(3).Visible = False

and use the Project group header section's Print event to make the page header visible:

    Me.Section(3).Visible = True

If you don't have header and footer sections for the Project group, create them and make them invisible.  Make sure the Project group header section's ForceNewPage property is set to Yes.
