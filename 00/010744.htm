scottgem say:

Ariane,

First I want to confirm that you are using the built-in Switchboard Manager that comes with Access. This is an interface that allows you to create Menu pages and items on those pages.

Second, Frankly, what you appear to be doing is a not a good idea. If I understand you correctly, you want to segregate your data by year. So when you open a form or print a report you only see data for the specified year. If that is the case, creating new object (forms, reports, etc.) for each year is a waste of time, effort and space.

The way I would do this is to add a control to my Switchboard to select a year. That control can default to the current year. The next step is to add a column to each of your queries:

Yr: Year([datefield])

with Criteria of

=Forms!Switchboard!cboYear

Doing so will mean that any forms, reports, queries etc. Will be filtered by the year selected on the Switchboard. Need to look at a different year, then just change the selected year. You only need ONE set of forms, reports and one Switchboard. You won't have to go through the process of creating new ones each year. You can easily move from year to year.

If you need more help setting up your app this way please describe your table in more detail and we can advise.

ken sheridan say: I have to agree with Karl and Scott that this is bad relational database design.  Data is being encoded as table names.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1).  This requires that all data be stored as values at column positions in rows in tables, and in no other way.

A single table with data for all years, and a single switchboard should be used in the way Scott describes.  If sequential numbering of rows per year is important then you should ether compute the numbers when a row is inserted into the table, or not store the number at all and return it in a computed column in a query.

You'll find an example of the former as CutomNumber.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file there is an option for 'sequential numbering by group'.  In my demo it is on the basis of gender, but in your case would be per year.  For examples of how to number rows sequentially per group see RowNumbering.zip in the same OneDrive folder.

The difference between storing the sequential numbers in a column in a base table and returning the numbers on the fly is of course that, in the former case the numbers will remain constant if rows within the sequence are deleted, leaving gaps, whereas in the latter case the numbering will always be sequential, reflecting the current data.

This should be really easy but it's wasting my entire afternoon! After all these years of word processing it is staggering that things like this don't just work.

bob jones aka: cybertaz say:

Select the text, then go to Format> Paragraph - Indents & Spacing. You'll most likely see that there is a value other than 0 entered for Space Before and/or Space After. Change them to 0.

The Paragraph Spacing tool on the Design tab pertains only to the Document Formatting presets on that same tab. I don't believe it will override Paragraph Spacing that has been applied to the text by a method other than the use of those presets.
