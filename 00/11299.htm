To expand on @Diane P's response

#1 - As was stated, if your GMail account was configured as an IMAP account AND the contacts were located in a folder marked as "This Computer Only" then that data is lost unless you explicitly made a backup to a PST file

OR

#2 - The GMail account was configured as an IMAP account AND there was a PST file configured as the "default data file" for the profile (which would have contained the default contact/calendar data files) AND that PST file wasn't backed up, the contact/calendar data is list

In short, if you have opened all the PST files found and there is no contact/calendar data in those PST files, then the wrong PST file(s) were backed up if item #2 applies. Case #2 would be applicable if you did an in-place upgrade to Outlook '2013/'2016 (Office 355) on the same machine from an earlier version of Outlook which had your GMail account configured as an IMAP account

jeeped say:
Assuming that only one issue (i.e. row) is marked as 1 at a time and the dates in column A are actual dates and not textual representations of dates, then SUMIF() or SUMPRODUTC() would be your best bet. The only remaining problem is correctly identify a 1 regardless of whether is is a number or a textual representation of a number that may have leading or trailing spaces. Try one of these in 'Paragraph B'!C5,

=SUMPRODUCT((TEXT(TRIM('Paragraph A'!$AC$2:$AC$9999),"0")="1")*('Paragraph A'!$A$2:$A$9999))
=SUMIF('Paragraph A'!AC:AC,"1",'Paragraph A'!A:A)
