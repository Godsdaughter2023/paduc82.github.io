rob koch say: mc48911,

Though you have some good reasons for your preferences, they aren't necessarily likely to cause change for several simple reasons.

The amd64 designation exists throughout Microsoft software and apparently has more to do with the original licensing of the 64-bit microcode which belongs to AMD than it does to a specific processor vendor, since the code is the same for both.  In fact, the only ones generally confused by this specific issue have been those that actually know the difference between Intel and AMD, which most users of MSE don't.  In fact, I can count the number of times this question has been asked here on one hand, right now I believe it has reached a total of two (2).

However, the issue of automatic selection of the installation file is real as you've stated, though I'm not completely clear myself why it exists.  Apparently the detection of a 64-bit operating system isn't always working, though the 32-bit seems to be fine, so yes this could use some more effort to see if it could be improved.  The selections themselves are specified as follows though, so AMD only exists in the final file names, not the selection list itself.

Windows XP 32-bit
Windows Vista/Windows 7 32-bit
Windows Vista/Windows 7 64-bit

Since the code base for the different versions is entirely different, the files size for a combined installation would be the sum of the three separate files plus any required operating system detection logic and other required package overhead, resulting in a file roughly 21MB or larger.  Though this may seem small to you, a primary goal in the creation of these three files was keeping the individual download size small for those not having broadband Internet, which is a common issue for many worldwide.

Finally, the reason the download is made available via the main Microsoft Security Essentials download site beofre being released as an upgrade is simple, to allow installation on a smaller number of systems before going to wide release.  Since not all issues are always discovered within beta and other pre-release testing this allows a more limited and controlled release schedule initially until the decision is made to perform a wide release.  If a few zealots decide to be fools and choose to download from unofficial channels to be the 'first' with a new version, that's their problem.  From my experience with beta releases at Microsoft and other developers, that problem won't ever be solved by any sort of release controls, no matter how good they are.

Rob

Sub Insert_Tab_For_Selected_Cell()
    Dim oSh As Shape
    Dim oTbl As Table
    Dim lRow As Long
    Dim n As String
    Dim lCol As Long
    Set oSh = ActiveWindow.Selection.ShapeRange(1)
    Set oTbl = oSh.Table

    With oTbl
    For lRow = 1 To .Rows.Count
    For lCol = 1 To .Columns.Count
        If .Cell(lRow, lCol).Selected Then
        n = .Cell(lRow, lCol).Shape.TextFrame.TextRange
        With .Cell(lRow, lCol).Shape.TextFrame
        .TextRange = vbTab & n
        End With
        End If
    Next
    Next
    End With
End Sub

Sub crop()
Dim n, i, m As Integer
Dim L, T, CL, CT, CR, CB As Double
Set aws = ActiveWindow.Selection
n = aws.ShapeRange.Count
'reset
For i = 2 To n
With ActiveWindow.Selection.ShapeRange(i)

.ScaleHeight 1, msoTrue
.LockAspectRatio = msoTrue
End With
Next
'middle center
ActiveWindow.Selection.ShapeRange.Align msoAlignCenters, msoTrue
ActiveWindow.Selection.ShapeRange.Align msoAlignMiddles, msoTrue

For i = 2 To n
CL = aws.ShapeRange(1).Left - aws.ShapeRange(i).Left
CT = aws.ShapeRange(1).Top - aws.ShapeRange(i).Top
With aws.ShapeRange(i)
    .PictureFormat.CropLeft = CL
    .PictureFormat.CropTop = CT
    .PictureFormat.CropRight = CL
    .PictureFormat.CropBottom = CT
End With
Next
' apply position
For i = 1 To n
m = m + 20
ActiveWindow.Selection.ShapeRange(i).Left = -150
ActiveWindow.Selection.ShapeRange(i).Top = 0 + m
Next
End Sub

Sub Casing()
With ActiveWindow.Selection
Select Case .Type
Case Is = ppSelectionShapes
Set shp = ActiveWindow.Selection.ShapeRange
'if table select
If shp.HasTable Then
    Dim oTbl As Table
    Dim lRow As Long
    Dim lCol As Long
    Set oSh = ActiveWindow.Selection.ShapeRange(1)
    Set oTbl = oSh.Table
    With oTbl
    For lRow = 1 To .Rows.Count
    For lCol = 1 To .Columns.Count
        If .Cell(lRow, lCol).Selected Then
        With .Cell(lRow, lCol).Shape.TextFrame
        .TextRange.ChangeCase ppCaseTitle
        .TextRange.Text = Replace(.TextRange.Text, " The ", " the ")
        .TextRange.Text = Replace(.TextRange.Text, ". the ", ". The ")
        
        
        End With
        End If
    Next
    Next
    End With
'if shape select
Else
ActiveWindow.Selection.ShapeRange.TextFrame.TextRange.ChangeCase ppCaseTitle
        If shp.HasTextFrame Then
            If shp.TextFrame.HasText Then
                shp.TextFrame.TextRange.Text = Replace(shp.TextFrame.TextRange.Text, " The ", " the ")
                shp.TextFrame.TextRange.Text = Replace(shp.TextFrame.TextRange.Text, ". the ", ". The ")
            
            
            
            End If
        End If
End If
Case Is = ppSelectionNone
Case Is = ppSelectionSlides
Case Is = ppSelectionText
On Error Resume Next
ActiveWindow.Selection.TextRange.ChangeCase ppCaseTitle
Set shp = ActiveWindow.Selection
                shp.TextRange.Text = Replace(shp.TextRange.Text, " The ", " the ")
                shp.TextRange.Text = Replace(shp.TextRange.Text, ". the ", ". The ")


End Select
End With
End Sub
