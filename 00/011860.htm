Workaround: It appears there is a problem with the build 9926 version of ndu.sys on certain systems. Whether it is outdated network drivers or the ndu.sys file itself is a moot point when one cannot use their computer effectively on a network. Here's what I came up with and it works perfectly:

1. Go to system32\drivers and locate ndu.sys

2. Rename it to whatever you choose e.g. ndu.sy0. You will have to change the permissions on the file which are set to "trusted installer" to your own administrator account to do the rename. You can find out how to do this with a google search.

3. Copy the previous version of the file from your windows.old folder (system32\drivers) to your 9926 system32\drivers.

4. If you did not upgrade from the previous build, I have found that the windows 8.1 version of the ndu.sys works as well.

This information is specific to a Lenovo Yoga 2 Pro. I cannot vouch for any other system that might exhibit this same behavior. I can now use the Yoga with wireless internet connectivity.

hansv mvp say:

Let's say your text entry is in cell A1.

In B1, enter the formula

 

=RIGHT(A1,LEN(A1)-FIND("@",SUBSTITUTE(A1," ","@",LEN(A1)-LEN(SUBSTITUTE(A1," ","")))))

 

where @ is a character that is guaranteed not to occur in the text entries (you can use another character than @ if necessary).

 

This formula can be filled down.
