adamf---01 say:

FOTA

Technical Level : Intermediate

Summary

Details

FOTA stands for:
Firmware Over The Air

The firmware is another word for the phones operating software, and you can update this over the air, free of any cables and without being tethered to a computer.

Most of the Nokia's newer models support this feature, as well as some older models. 

Here is how you use FOTA:
as demonstrated with my N96

Start on the main screen and type *#0000#.
You will see the following page.


    1. Press centre key.

 


    1. Select "Check for updates"

 


    1. Wait, the phone connects to FOTA Server



The phone connects directly to the FOTA server, and looks for a newer version of software.
If one is available, you will get the option to download and update your handset.

This is a much easier way to update the phones firmware, and saves the use of a PC which means you can do it anywhere, away from your desk.

I hope this helps with learning what FOTA is and how to use it.

Could you clarify these two tables? Does the PartNumber table contain only one field, Part Number, and the other table contain only one field, Description?? If so, how can you identify which of the hundreds of Description records to choose when you select a part number?

 

As Karl says, you may want to step back a bit and think about normalization. For one thing, there is a basic principle - the Grandmother's Pantry Principle: "A place - ONE PLACE! - for everything, everything in its place". The Description of a part should exist once and once only, in the Parts table (with a PartNumber as its Primary Key), and noplace else; you're using a relational database, and the way such databases work is that you store data, efficiently and non-redundantly, and use Queries (and other tools) to display that data where it's needed. So if you have a description once in your Description table (or I'm guessing better in a Parts table) then the Master table - whatever it is - should have a foreign key link to the table containing the description, but it should certainly NOT contain a field for the description itself.

macropod say:

A simple macro for counting words in Word citation fields is:

Sub CitationWordCount()
Dim Fld As Field, l As Long, StrTmp As String
For Each Fld In ActiveDocument.Fields
  With Fld
    If .Type = wdFieldCitation Then
      StrTmp = .Result
      l = l + UBound(Split(StrTmp, " ")) + UBound(Split(StrTmp, "-")) + 1
      StrTmp = .Code.Text
      l = l + Len(StrTmp) - Len(Replace(StrTmp, "\n", "\"))
    End If
  End With
Next
MsgBox "There are " & l & " words in citations in this document."
End Sub

As coded, the macro adds '1' for every citation for which the author's name has been omitted.
