Sub XML_Append_ConvertToRange()
'Dec 21, 2015

Set ws = ActiveSheet

Dim sUrl
sUrl = "http://api.eve-central.com/api/quicklook?typeid=9950&regionlimit=10000002"

Dim x As Long

If [A2] = "" Then
Application.DisplayAlerts = False
ActiveWorkbook.XmlImport URL:=sUrl, ImportMap:=Nothing, Overwrite:=True, Destination:=[A2]
Application.DisplayAlerts = True
ws.ListObjects(1).Unlist
ws.UsedRange.ClearFormats

Else
x = Cells(Rows.Count, 1).End(xlUp).Row + 1
Application.DisplayAlerts = False
ActiveWorkbook.XmlImport URL:=sUrl, ImportMap:=Nothing, Overwrite:=True, Destination:=Cells(x, 1)
Application.DisplayAlerts = True
ws.ListObjects(1).Unlist
ws.UsedRange.ClearFormats
Cells(x, 1).EntireRow.Delete

End If

Range("Q:Q, AB:AB").NumberFormat = "mm/dd/yyyy"

ws.UsedRange.EntireColumn.AutoFit

'Delete XML Map

Dim obj As XmlMap
For Each obj In ActiveWorkbook.XmlMaps
obj.Delete
Next obj
End Sub

There's something missing in this description. If a default data file already exists for the profile - adding an IMAP account would not result in the creation of a secondary data file - it would default to the one that already exists.

The default behaviour is for the <Inbox> in the "default data file" to be the one that gets the focus. Changing that via Brian's suggestion will solve that problem.

File -->Options-->Advanced--> Outlook Start & Exit

Just point it to the <Inbox> of the IMAP PST file. Only thing is that changing the "startup" <Inbox> folder doesn't change where email's are delivered using this setting. Delivery locations for the IMAP <Inbox> can't be changed regardless of which <Inbox> is set to be shown as the startup.

As far as the secondary <Outlook Data File> originally created when you configured the IMAP account - you can delete that given that you've set the original POP3 PST file as the default. As an aside, if you simply added the IMAP account to the existing profile with the POP3 account, the PST file associated with the POP3 account would have remained the default. All you would then have needed to do was delete the POP3 email account which would have not had an impact on the associated data file.
