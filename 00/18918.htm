tom van stiphout say:

You can't do that.

Multivalue fields do not conform to relational concepts but were introduced to get some SharePoint interoperability. The two don't really mesh but in the simplest of situations. You may be better off designing this using relational concepts.

ken sheridan say:

As the user is the primary point of contact for the vendor, i.e. there is only one per vendor, you appear to have the relationship the wrong way round.  There is a one-to-many relationship type from Vendors to Users, so there should be a PrimaryPOC_ID or similar foreign key column in Vendors.  You can then use:

DLookup("[VendorID]","Vendors","[PrimaryPOC_ID]=" & [TempVars]![tmpUserID])

Note that when using a variable in a DLookup function call you should concatenate the value of the vraiable into the string expression as above, not reference it.
