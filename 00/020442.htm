jnetopil say:

To everybody interested in this problem: I believe I have found the reason for this problem. It seems that the problem is not in the phone´s network stuff neither in the VoIP itself. Maybe surprisingly the problem seems to be caused by the low power of the phone´s HW, the CPU in particular. It seems to have not enough power for processing the high frequency ping and highly compressed VoIP voice stream as well.

 

In fact I recognised this problem when I was using VoIP with the iLBC codec in its default settings. When I changed the codec to uncompressed PCMA or PCMU the problem has gone. After that I tried the iLBC again but set on the 20ms frame length instead of the default 30ms. With 20ms the resulting bitrate is slightly higher and therefore I suppose the compression to be less CPU power demanding. I was really surprised how the test with iLBC on 20ms frame length went well throughout the home wifi. There were no noticeable problems. Now I´m fine with the current VoIP usability.
