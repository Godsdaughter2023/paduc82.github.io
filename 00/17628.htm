To protect your PowerPoint presentation.:

  1. Click the File tab to go to Backstage view.
  2. From the Info pane, click the Protect Presentation command.
  3. In the drop-down menu, choose the option that best suits your needs. In our example, we'll select Mark as Final. Marking your presentation as final is a good way to discourage others from editing the file, while the other options give you even more control if needed.
  4. A dialog box will appear prompting you to save. Click OK.

scottgem say:
Hi Earth, I'm an independent adviser and can help,

First, a combobox (what I think you refer to as drop down menu) has two ways to supply items to the list. One way is to use a list of values. But you only use a Value List when the list of items is short and static. For your purposes a Value List is not appropriate. so the other way is to use a table as the Rowsource. You can then add records to the table as needed. So you create a table like this:

tluCategory
CategoryID (PK Autonumber)
Category

If the category isn't already in the table, then Combobox has a NotInList event that will allow you to add the entered item to tluCategory. If you search on NotInList event you will find several examples of how to code this event.


Second, you need to determine where to store the CategoryID. this depends on whether a contact can fall into more than one category. If they will only fall into one category, then you add a Long Integer field to your contacts table for CategoryID and populate it using a combobox (use the wizard to create the combo) on your form. However, if there could be multiple categories, then you need to add another table like this:

tjxContactCategory
ContactCategoryID (PK autonumber)
ContactID (FK)
CategoryID (FK)

You would add a subform to your form to populate that table.
