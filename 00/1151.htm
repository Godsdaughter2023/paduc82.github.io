Thanks in advance!

dale howard [mvp] say:

stacij42 --

When you create a custom Map in Microsoft Project 2010 and save it, the software saves the new Map automatically in your Global.mpt file.  This gives you access to this Map for all current and future projects.  Although it is technically possible to copy the new Map from your Global.mpt file into the project, users WILL NOT be able to access the Map when they try to use an existing Map in the Export Wizard dialog.  Therefore, what I would recommend you do is to create a new blank project and use the Organizer dialog (File > Info > Organizer) to copy the custom Map from your Global.mpt file into the blank project.  Then save the blank project with a descriptive name, like Project with New Map.mpp, and then send it to each of your PMs.  Ask them to use the Organizer to copy the new Map from the blank project into their own Global.mpt files.  This way, everyone will have access to the new custom Map.  Hope this helps.

I ran into this very issue yesterday while exporting a project file to Excel.  There is no way to force Microsoft Project to eliminate the 'h' value for you in the Work column.  You will need to use Find and Replace in Excel to take care of the problem.  Excel works well in this situation, changing the formatting of the Work from Text to Number.  If my response answers your questions, please remember to mark the post as the Answer.  Thanks!
