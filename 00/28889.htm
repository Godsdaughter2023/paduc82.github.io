  2. What is the meaning and purpose of Default Paragraph Font in the Style box?

jay freedman say:
I think it's safer to do it in the opposite order: Insert a Rich Text content control, open its Properties dialog, and check the box for "Use a style to format text...". Then either select a style from the dropdown, or click the New Style button and define a custom style.

The advantage of doing it this way is that Word will show you only the choices that are allowed in a content control. If you're choosing a built-in style, the list contains only character styles and linked styles. If you're defining a custom style, you can choose to make it a character style or a linked style (by default it's a character style), but you won't get the opportunity to make it a paragraph style that you can't use in a content control. Then, when you click the Format button at the bottom of the New Style dialog, you can choose Font, Border, Language, or Text Effects, but the choices that aren't allowed in a content control are grayed out.

One other thing: After you define a custom style this way, that style will appear in the Style dropdown in another content control's Properties dialog so you don't have to repeat the definition.

jay freedman say:
The first, most important fact about styles applied through the content control Properties dialog is that you can apply only character styling, not paragraph styling. If you apply a 'linked' style -- one that appears in the Styles pane with the ¶a icon -- then only the character-related part of the style is applied. You may notice that styles that are only paragraph styles -- with the ¶ icon -- don't appear in the list in the Properties dialog.

That restriction doesn't apply to the text paragraph that contains the content control; you can apply any paragraph or linked or character style to that paragraph.

Now to your questions, slightly out of order:

2. "Default Paragraph Font" -- anywhere in Word, including in the Styles pane and the Manage Styles dialog -- refers to the font defined as part of the underlying paragraph style. In the style definition, if the font is specified as "+Body" or "+Heading", then the font in use is determined by the Theme setting. That's true for most of the styles in Word 2007 and later.

1. I don't see the same behavior you described, as long as the style I apply to the content control is a character or linked style. The Properties dialog opens with the applied style already selected in the Style list.

3. Yes, applying direct formatting (formatting that is not part of the applied style) will override whatever style was applied, and it will affect both the placeholder text and any entered text. That includes color, bold, italic, super/subscript, etc.
