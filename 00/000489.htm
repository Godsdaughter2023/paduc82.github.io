steve rindsberg say:
To expand a bit on what Rich wrote, PPT will try to fill the printed page with your slide but won't distort anything; that means that if your slide's proportions aren't the same as your paper's proportions, there will be extra border L/R or Top/Bottom.

You can use Slide Size on the Design tab to change the page size, but if you do that with an existing presentation, THEN PPT will distort stuff to fit the new page size.

You may instead want to create a new presentation, set its dimensions to match your paper size, then copy/paste everything from your existing slide to the new one and adjust the position/size of everything on the slide to suit your new layout.

graham mayor say:

I think you are mistaken about how the macro works compared with the GUI version. When a message is received, the rules are automatically processed sequentially in the order they appear in the GUI. If you run the rules manually, you can select a range of rules to apply and obviously only those rules are applied, but otherwise the rules are processed sequentially on each message. The macro is effectively the same as setting all the rules to run manually. However in my tests, the macro runs rather faster than manually selecting all the rules and running from the GUI.

Your other post questions whether it would be possible to select the records to process via the GUI from the macro. The answer to that appears to be no. There does not appear to be any access to the dialog from Outlook VBA that I am aware of.

You could emulate it by creating a userform and reading the rules into a list box from which you could select the rules that you wish to run fom that list and that would speed up the processing a small amount, but frankly I don't see what the advantage of the macro would be over the tools provided by Outlook. There seems to be some measure of wheel re-invention going on. :)

If I can quote from my own use of Outlook, I have a about 25 rules configured which caters for most of the messages I receive, all of which are moved to other folders according to a variety of conditions. The rules run automatically upon receipt and the messages filed until read. This leaves relatively few unprocessed messages in the inbox. For the purposes of testing the macro I moved a large selection of messages back to the inbox and processed them. Even with several hundred messages the macro only took a few second to run on a PC that is now getting decidedly long in the tooth compared with current hardware.

However if you insist, create a userform in Outlook VBA and call it frmRunRules. Add a large ListBox and a single CommandButton at the bottom each with their default names i.e. ListBox1 and CommandButton1

Apply the following code to the userform

Option Explicit
Private st As Outlook.Store
Private myRules As Outlook.Rules
Private rl As Outlook.Rule
Private count As Integer
Private ruleList As String
Private i As Long

Private Sub UserForm_Initialize()
Set st = Application.Session.DefaultStore
Set myRules = st.GetRules
Me.BackColor = &HC0FFFF
Me.Caption = "Select the Rules to Run on the Inbox"
Me.CommandButton1.Caption = "Run Selected Rules"
With Me.ListBox1
    .Clear
    For Each rl In myRules
        .AddItem rl.Name
    Next rl
    .MultiSelect = fmMultiSelectExtended
End With
End Sub

Private Sub CommandButton1_Click()
Me.Hide
Set st = Application.Session.DefaultStore
Set myRules = st.GetRules
With Me.ListBox1
    For i = 0 To .ListCount - 1
        If .Selected(i) Then
            Set rl = myRules.Item(.List(i))
            If rl.RuleType = olRuleReceive Then
                ' if so, run it
                rl.Execute ShowProgress:=True
                count = count + 1
                ruleList = ruleList & vbCrLf & rl.Name
            End If
        End If
    Next i
End With
ruleList = "These rules were executed against the Inbox: " & vbCrLf & ruleList
MsgBox ruleList, vbInformation, "Outlook Rules"
Set rl = Nothing
Set st = Nothing
Set myRules = Nothing
Unload Me
End Sub

Create a macro to call the userform e.g.

Sub RunMyRules()
frmRunRules.Show
End Sub

stuart e. say: The problem got fixed with the help of a friend who checked it out and came up with the solution.  It was to create a new Identity in Microsoft Database Utility and Rebuild the database.  (Deleting the existing identities.)  The Address Book is now reconnected to Word.  Many thanks to all who provided help and assistance on this.

stuart e. say: Per my friend these were the exact steps followed:
1. Make sure the Outlook application is closed.
2. Make sure you have the update installed and you are running on version 14.2.0 or higher.  You can check by opening Word and going to "About Word."
3. Locate your identities folder using Finder. Probably in Documents; User Data/Office 2011 Identities.  Make a backup of your entire identity folder, just in case.
4. Create a new identity using the Microsoft Database Utility, located in Applications/Microsoft Office 2011/Office.
5. Go back to your identities folder.  Now you should see two folders, one with your old identity, and one with the new identity.  Each one of those has a "Data Records" folder that stores all messaging data.
6. Manually copy folders from the "Data Records" folder in the old identity to the "Data Records" folder in the new identity, except for the three folders that were already created in the new identity; i.e., "Database Headers," "Preferences," and "Recent Addresses."  
7. Go to the Database Utility, select your new identity and tell it to rebuild the database.  It will go through 5 steps and might take some time.
8. When you get confirmation that your database was successfully rebuilt, make the new identity your default.
