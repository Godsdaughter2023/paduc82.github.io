roady [mvp] say:

Redirecting ost-files to a network share is not supported. Ost-files should reside on the local hard disk.

The ost-file is nothing more than a local cache of the Exchange mailbox. Their default location is the within the AppData\Local tree which doesn't roam. Setting it to a roaming location would defeat half the purpose of having it and would immensely increase the user's logon and logoff time, the network usage and has a high chance of getting corrupted.

Changing the location of the ost-file is intended to move it of the OS disk or to a larger or cheaper disk which still has enough performance.

Also note that changing the setting only applies to newly created ost-files after the change has made. It does not affect existing mail profile configurations.
