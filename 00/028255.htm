However, when the range is copied and pasted, the names become workbook-wide names.  This complicates the name deconfliction in later iterations of the copy and paste.  It doesn't make sense to me that the name scope changes after the copy/paste, but is there a simple way to ensure that copy/paste keeps the names local the the sheet (without tracking every range name in every range that might undergo such a copy/paste)?

In stead of copying the formulas you can set one range equal to another ranges local formulas:

 

Sheets("Sheet1").Range("A1:H100").FormulaLocal=Sheets("Sheet2").Range("A1:H100").FormulaLocal

paul domaskis say:

I was thinking of copying formulas...your example points out the relevant methods.  But the context of the problem is that I also want the copy&pasted range contains named ranges that I also want copied.  So I do want to copy the range and all named cells within rather than just copying formulas.  The addresses referred to by the named ranges can change, so I don't want to keep the old definitions in the destination range.
