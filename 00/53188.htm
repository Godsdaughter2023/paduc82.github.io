internet expert say:

The buttons are labeled after the ABXY scheme that's been carried over from many generations of gaming before. It's easy to differentiate, contrast, and identify AB/XY button groups, possibly due to their designative roots in science and math. As a result, this has since been applied to user interface design in gaming controllers.


The Xbox is named after the fact that it uses the DirectX API for graphics (it was originally suggested to be called "DirectXbox"). It wasn't Microsoft's first choice of possible names for the console, but it ultimately stuck with the focus testing groups.


The controllers are still called "controllers" not because they don't resemble recessed coastal bodies of water that connect to larger oceans, but rather their function is stated directly on the tin. Doesn't really sound right when you hear, "ayy bruh, pass the XBAY" and see your friend waggle his cheeto-coated fingers in front of your face, just demanding you slap him upside the head with your Xbay.

a.       Press Windows key.

b.      Open the camera app.

e.      Move the Webcam and Microphone turn off slider towards turn off.

 

You can disable the video camera using the following steps.

a.       Press Windows key and X key together.

b.      Select “device manager”.

c.       Click on the Imaging devices.

d.      Right click on the Webcam device.

e.      Select Disable.

davidmcandy say:

It has the following not very important line in it.

Files=40

The WinSxs is the central file repository for Windows. All files installed are placed into the WinSxS folder including updated files. So you'll have lots of different versions of each file allowing you to roll back Windows updates and service packs to any previous state.

 

The current version of a file is hardlinked to the appropiate place in the rest of the file system. A hardlink is a file name. All files have at least one hardlink. Windows system files have two hard links, one in WinSxS and one in (usually) c:\windows\system32. Both names point to the same bytes on disk.

 

As the files have two names they are counted twice by explorer, once in System32 and once in WinSxS.

 

There are other uses for it as well. It allows multiple programs to use different versions of dlls.
