gord dibben say:

This Sheet event code will keep track of changes to a Worksheet

The changes are saved and date/time are inserted on "Log Sheet"

Copy the code then select Sheet1 tab and "View Code"

Paste code in there, change the target range if you want.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim strAddress As String
    Dim val
    Dim dtmTime As Date
    Dim Rw As Long
    
    If Intersect(Target, Range("A1:M1000")) Is Nothing Then Exit Sub
       'change range to suit
    dtmTime = Now()
    val = Target.Value
    strAddress = Target.Address
    
    Rw = Sheets("Log Sheet").Range("A" & Rows.Count).End(xlUp).Row + 1
    With Sheets("Log Sheet")
        .Cells(Rw, 1) = strAddress
        .Cells(Rw, 2) = val
        .Cells(Rw, 3) = dtmTime
    End With
End Sub

bernd p. say:

Short answer : NO !

Longer answer:

You don't understand the storage concept of Onenote, but you are not alone ... MS delivers no basic and easily understable documentation about this.

A look behind the curtain:

A Onenote notebook is at first realized as a folder within the file system of Windows (it can come to existence also at first in Skydrive, but I want to keep things simple).

A notebooks contains one or more sections. These are realized as files of type *.one within the notebooks folder. These can be duplicated/synched by using Dropbox (as you did).

BTW: that's deadly if more than one user makes changes, because DB and ON work against each other during concurrent changes. A singular problem only with Onenote files. No one to blame, just incompatible synch solutions.

If you open a notebook it's content is always and inevitablly copied to a local cache file only usable by the Onenote desktop application. The thread you mentioned, about performance problems with that cache has absolutely no relation to your problem.

BTW: this cache contains all open notebooks in only one file.

Back to your problem:

May I call the fresh created notebook within the Windows file system the original .

The location of this notebook can be changed to share it between 2 or more computers. A simple solution is to a server file share for computers on the same LAN.

The more interesting case is sharing over the Internet. Before the introduction of Skydrive one possibility was Dropbox.

If you use Dropbox you have in reality 5 (!) copies of the notebook by sharing it with 2 computers:

  1. The original on computer #1
  2. its local cache copy on #1
  3. its remote copy on the Dropbox server
  4. its duplicated "original" on computer #2
  5. its local cache copy on #2

If you use Skydrive you have in reality 3 copies:

  1. The original on Skydrive (no obscure cache, as you called it)
  2. a local cache copy on #1
  3. a local cache copy on #2

Both local cache copies can be modified without Internet connection.

Now I hope you understand that it's not necessary to use any combination of Dropbox and Skydrive.

The iPhone is only a 3rd computer within this configuration which needs Skydrive.

Bernd
