marshall barton say:

No, queries will not help.  The problem is that they are each editing the same record.  If you are using optimistic record locking, when the second user tries to save the record they will be prompted to overwrite whatever the first user changed and if the second user does not do that then his changes will be discarded.  If you are using pessimistic locking, the first one to get the record will lock the record so other users can not make changes.

If the fields the two users are editing are separate so there really is no conflict, then put the two sets of fields in two separate tables that are linked in a one to one relationship.  This way the two forms can be bound to two different tables and they will be able to edit their fields independently.

Note: two tables are in a one to one relationship if they are linked by their primary key field.

jbarranco75 say:

This is the issue..... the updates are happening on 2 different forms and 2 different tables.....  They both query off the main table so that I can access all the ECO's....

marshall barton say:

Either I have failed to grasp what your tables, queries and forms are doing or you can just lock the fields from the main table so neither user can mess with them (if one user is allowed to mess with the main table fields, lock the fields on the other user's form).

Note: a form will "go blank" if there are no records that satisfy the criteria or filtering AND AllowAdditions is set to No (False)
