To install the latest version of NSU, instead of the single msi file you need to find a subdirectory under the "C:\Documents and Settings\username\Local Settings\Temp" containing
- Nokia Software Updater.msi
- Setup.INI
- Nokia Connectivity Cable Driver.prq
(and some other files as well...)

Copy that directory into a safe place, and click OK to let the installer die as the orginal message suggests. Now delete the _ISMSIDEL.INI file, you don't need it. Build 14931 no start menu for new users when Windows Search is - Microsoft Community

mdedeboer say:

Build 14931 no start menu for new users when Windows Search is disabled

As per Citrix support (see here: http://support.citrix.com/article/CTX125874?_ga=1.221086744.1202815851.1446564037)

it is common practice to disable the Windows Search service on virtual desktops.  This is typically due to either slow disks on the back end storage array or due to non-persistent disks (which would cause the index to constantly rebuild).

As of build 14931 we have noticed that when the Windows Search service is disabled, the start menu and cortana simply do not launch for new users.  This appears to be consistent with Microsoft's direction of a search based experience, so I think it is important as IT professionals to rethink and redesign how Windows Search fits into VDI.  The lay of the land as it appears in Windows 7 won't cut it for Windows 10.
