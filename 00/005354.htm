The operative word is "appears".

The likely explanation is that TotalIncome is infinitesimally larger than 25000.  It only appears to be 25000 because of the way that VBA formats non-integer values.

The same thing happens in Excel.  And the same remedy applies, to wit:  whenever you combine non-integer values arithmetically, it is prudent to round to the precision that you expect, 2 decimal places in this case for dollar-and-cents calculations.

Ostensibly, you should write (also see footnote [1] below):

HalfSocSec = Round(SocSec/2,2)
TotalIncome = Round(FedTaxable + MuniBonds + HalfSocSec,2)
WrkAmt = Round(TotalIncome - 25000 - 9000,2)

All of the above is valid.  However, I hasten to point out that the following fails to duplicate the problem with the values that we see in the watch table.

Sub testit1()
Const fedTaxable As Double = 2385.38
Const muniBonds As Double = 20348.12
Const halfSocSec As Double = 4533 / 2
Dim totalIncome As Double
totalIncome = fedTaxable + muniBonds + halfSocSec
MsgBox totalIncome = 25000
End Sub

Surprisingly, MsgBox displays True.  I say "surprisingly" because 2385.38 and 20348.12 cannot be represented exactly as a type Double.  It is an accident of implementation that totalIncome is indeed exactly 25000.

In any case, that suggests that some of your input parameters -- fedTaxable, socSec and muniBonds -- are not exactly as they appear either.

I presume they are computed in Excel or in some other VBA procedure.  In either case, with those computations as well, you should explicitly round to 2 decimal places.


-----
[1] Caveat:  VBA Round behaves differently than Excel ROUND when the value to be rounded is usually cannot be represented exactly).

Nevertheless, if you are concerned, you could replace Round above with WorksheetFunction.Round.  It will perform more slowly, but I suspect you will not notice unless you are doing many thousands of these calculations.

Here some examples with some results that are different from Excel ROUND:

Sub testit2()
MsgBox Round(0.005, 2)    ' VBA: 0; Excel: 0.01
MsgBox Round(0.015, 2)    ' VBA: 0.02; same in Excel
MsgBox Round(0.025, 2)    ' VBA: 0.02; Excel: 0.03
End Sub

Typically, messages in Outbox have not been sent and one common issue is a failure to connect with the outgoing server.  That's why I suggested changing the outgoing server's name to what AT&T suggests currently.  You never mentioned that the messages, despite remaining in the Outbox, are still arriving in your recipients' mailboxes.
