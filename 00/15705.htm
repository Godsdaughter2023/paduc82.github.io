Jeremy,

First of all what version of Project are you using? Is it updated with the latest service pack?

In the description of your process, there are some items missing. You indicate you enter work, start date and predecessors. What about resources, do you assign any? How about the duration of your tasks? Note that work and duration in Project are two different things. Duration is the working time span from start to finish. Percent complete is a duration based measure. Work on the other hand is the amount of effort one or more resources need to complete a task. Percent work complete is a work based measure. If a single resource is assigned to a task full time, then work and duration will be the same. Normally that is not the case.

When you update your tasks, unless you tell it otherwise, Project assumes you are executing on plan, so why would you expect the finish date to change? If the task is not on plan, then you need to perform your update such that the finish date is pulled in or pushed out to reflect the current plan. Saving a baseline at the onset, allows you to compare how your plan is unfolding compared to how it was initially planned.

It sounds like you jumped into Project without really understanding how it works and how to best use it. [Reposted to correct incomplete formulas.]

Nicole wrote:

The total row at the bottom is also not working properly.  Formulas are simple SUM, like: =SUM(G14:G17)

It would, if you use the format Custom [h]:mm .  The "[h]" displays more than 23 hours, unlike "h".  I would use that format for all time calculations (column G, H and I) as a matter of "good practice", even if "h" might suffice.

But you said you want decimal hours.  The following formulas allow for spanning across midnight.  They also avoid misleading and zero calculations when input is incomplete.

Hours Travel, G14:  =IF(COUNT(B14:E14)<>4, "", (C14-B14+(C14<=B14) + E14-D14+(E14<=D14))*24)

Hours Service, H14: =IF(G14="", "", (D14-C14+(D14<=C14)-F14)*24)

Total Hours, I14: =IF(G14="", "", G14+H14)

Format cells as Number with 0 or more decimal places (your choice).

Notes:

1. Note a correction to my previous formula:  the "midnight factor" should be of the form C14<=B14, not just C14<B14, so that the result is 24 hr when C14=B14, not zero.

2. Excel stores numbers in a binary form.  Binary arithmetic can introduce infinitesimal "errors".  It is prudent to explicitly round calculations to the precision that you want to be accurate.  For example,

MATCH(("5:00"-"4:00")*24,{1},0) returns #N/A (not a binary match), whereas

MATCH(ROUND(("5:00"-"4:00")*24,0),{1},0) returns 1 (exact match).

3. Likewise, if you want fractional decimal hours accurate to the minute, it is prudent to use the following paradigm.

MATCH(("5:23"-"5:08")*24,{0.25},0) returns #N/A (not a binary match), but

MATCH(ROUND(("5:23"-"5:08")*1440,0)/60,{0.25},0) returns 1 (exact match).

4. The explicit rounding in #2 and #3 might not be necessary if you only intend to display the results, not reference them in other formulas, especially lookup tables.
