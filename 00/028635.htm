suzanne s. barnhill say:

In the Customize Outline Numbered List or Define New Multilevel List dialog, select Right for the "Number position" or "Number alignment." When you do this, you'll find that the "Aligned at" setting automatically changes. You'll probably still need to do some fine-tuning to get the alignment just to your satisfaction.

 

I have defined the List Number style to have right-aligned numbers at 0.13" with "Aligned at" and "Text indent at" set to 0.25". This puts a single numeral (followed by a period) in 12-point TNR flush left, with double digits hanging outside the margin. This is ideal if you have few lists with more than 9 items. If you have many double-digit numbers, you may prefer to use larger settings for both "Number position" and the other settings, either to align the second digit flush left or to indent the entire paragraph a little more. Generally, speaking, for text sizes of type, about 1 point between number or punctuation and text is sufficient. So a right-aligned number at 0.15 and text at 0.25 works fine.

brian tillman [outlook mvp 2007-2019] say:

The issue here is that "All Mail" is the real mail store. It's just one big pile where your mail resides. The "folders" aren't actually folders. They're just labels, as you say. So, the All Mail collection has labels associated with some of the messages and that's why they appear together in what looks like "folders".

Now, the way the Google IMAP works is that it presents anything with labels to the IMAP client as being in a folder. So, when Outlook.com polls Google, Google tells it about the messages with labels, but not about the messages without the labels. This, Outlook.com can't see those messages and won't pull them over.

You could get around this by giving everything in the All Mail collection some label so that it will appear as a folder on the Outlook.com side of things. That should allow Outlook.com to pull the messages over, where you can put them in whatever folder, Archive included, you wish.

OP here. Thanks all for the feedback. I now have a better understanding of how the outlook.com importer works. Between the replies in this thread and this stackexchange post, I have devised the following solution.

Note that some people have been reporting that has:nouserlabels seems to work inconsistently and that the very similar -has:userlabels may work better. For maximum accuracy, the recommended workaround is to specifically exclude each custom label, though this may not be feasible if you have a lot of labels. I don't have an insurmountable amount of labels so I opted to exclude each one from the search and then applied a label toarchive to label all the messages that met the search criteria. I used the following search query.

-label:{label1 label2 label3} -in:sent -in:chats -in:Inbox -in:draft -in:spam -in:trash

Where label1 label2 label3 ... are custom labels

Some caveats:

  • If you have multi-word labels and you use the braces notation {} as above you should separate the words with hyphens. For example the label "National Parks" would be "national-parks". Despite some saying that multi-word labels do not work inside braces, I tested it and it works for me.
  • The chats tag is plural not singular as described above. You should reference it as -in:chats
  • In addition to the excluded tags mentioned by others, I also excluded spam and trash. These are typically excluded by default anyway, but I believe in being thorough.
  • Turning off conversation view may help make things less confusion as multiple messages with potentially different labels will not be grouped together.

Good luck!

Sub RemoveNonNumber()
    Dim X As Long, Cell As Range, CellVal As String
    For Each Cell In ActiveSheet.UsedRange
        CellVal = Cell.Value
        For X = 1 To Len(CellVal)
            If Not Mid(CellVal, X, 1) Like "#" Then Mid(CellVal, X, 1) = " "
        Next
        Cell.Value = Replace(CellVal, " ", "")
