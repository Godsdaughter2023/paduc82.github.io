tom ogilvy say:

Phil Hageman,

 

this worked for me.

 

Private Sub Worksheet_Change(ByVal Target As Range)

     'This event will make a group of three cells mutually exclusive

    If Target.Count > 1 Then Exit Sub
    If Target.Row > 200 Then Exit Sub
    On Error GoTo ErrHandler:
    Application.EnableEvents = False
    If Target.Column = 11 Then
         Cells(Target.Row, "L").Resize(1, 2).ClearContents
    ElseIf Target.Column = 12 Then
         Cells(Target.Row, "K").ClearContents
         Cells(Target.Row, "M").ClearContents
    ElseIf Target.Column = 13 Then
         Cells(Target.Row, "K").Resize(1, 2).ClearContents
    End If

ErrHandler:

Application.EnableEvents = True

End Sub

suzanne s. barnhill say:

When you display markup in balloons, in All Markup view, you see deletions in the balloons and additions in the text itself (underlined and in color if you've opted for that formatting). In Simple Markup view, you see only change bars (you can click on them to toggle between Simple and All). In No Markup view, you see the result of your changes without any markup. In Original view, you see the document before you started making changes.

In Word 2010 and earlier, in addition to Final Showing Markup, which was the equivalent of All Markup, there was another option, Original Showing Markup, in which deletions were shown with strikethrough in the text, and additions were shown in balloons. This view appears to be no longer available.

In All Markup view, to see the identity of the reviewer who made additions, mouse over the added text. The reviewer's name (and the date and time) will be shown in a ScreenTip.

ruel mayoya msft say:

Hi Alt-Mika,

I understand you want to filter emails with pdf attachments that have suspicious links. In the standard EOP, to scan the content of a file attachment in email is not feasible. However, if you have an Office 365 E5 subscription or have purchased an ATP subscription , Advanced Threat Protection for safe attachments and safe links feature can be used to detonate these PDFs by setting up a safe attachments policy in ATP.

In ATP, to find out if a link was followed and if it is malicious, use URL Trace report. As an admin, you can find this report in Exchange Admin Center > Mail flow > URL trace.

If you want to block all these emails with PDF attachments, check Exchange Online Protection (EOP) service spam and virus protection. As an admin, you can login to the Exchange Admin Center (EAC), and create a transport rule to block any file with the (.pdf) extension (EAC > mail flow > create a new rule > more options). Select the condition “file extension includes these words” equals to “pdf”.

Feel free to post back if you have further questions.

macropod say:

You could do this with your existing mailmerge main document, but with the mergefields replaced by a DATABASE field that groups the data. Assuming the mailmerge main document is kept in the same folder as the data source, the DATABASE field could be coded as:

{DATABASE \d "{FILENAME \p}/../MM datasource.xlsx" \s " SELECT [Subordinate Forename], [Subordinate Surname], [Subordinate ID], [Subordinate Job Title] FROM [Sheet1$] WHERE [Manager ID] = {MERGEFIELD Manager_ID} ORDER BY [Subordinate Job Title] " \l "15" \b "49" \h}

where 'MM datasource.xlsx' is the data source filename - it could be a database filename, if that's what you use. If the mailmerge main document is NOT kept in the same folder as the data source, you'd need to replace all of "{FILENAME \p}/../MM datasource.xlsx" with the full path & filename (plus the encompassing double-quotes). The \l and \b switches control the table format, while the \h switch insert a table header row - see: https://support.office.com/en-us/article/Field-codes-Database-field-04398159-a2c9-463f-bb59-558a87badcbc?ui=en-US&rs=en-US&ad=US

Note: The field brace pairs (i.e. '{ }') for the above field code are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. The spaces represented in the field construction are all required.

Then, to drive the process, you could use a macro like:

Sub Merge_Groups_To_Individual_Emails()
Application.ScreenUpdating = False
Dim MainDoc As Document, StrMgrID As String, i As Long
Set MainDoc = ActiveDocument
With MainDoc
  For i = 1 To .MailMerge.DataSource.RecordCount
    With .MailMerge
      .Destination = wdSendToEmail
      .SuppressBlankLines = True
      With .DataSource
        .FirstRecord = i
        .LastRecord = i
        .ActiveRecord = i
      End With
      If .DataSource.DataFields("Manager_ID") <> StrMgrID Then
        StrMgrID = .DataSource.DataFields("Manager_ID")
        .Execute Pause:=False
      End If
    End With
  Next i
End With
Application.ScreenUpdating = True
End Sub
