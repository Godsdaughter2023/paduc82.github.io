Task 'Internet Calendar Subscriptions' reported error… - Microsoft Community

ross1b say:

Task 'Internet Calendar Subscriptions' reported error… VEVENT

“Task 'Internet Calendar Subscriptions' reported error…”

I use a Google Calendar, which I access in Outlook as an Internet Calendar. It allows me to share my calendar over multiple platforms and to multiple users. I was frustrated with the VEVENT error message in Outlook for a couple of years, with it occasionally corrupting my local data file, before finding the following advice.

https://answers.microsoft.com/en-us/msoffice/forum/all/task-internet-calendar-subscriptions-reported/5bccc5bd-c8b5-4fac-b544-0c1f66d3e74b?page=1

 

I’ve re-typed the steps below, in the way that I had to use them.

Open Google calendar in a browser.

Click on: Settings “cog” | Settings | Import & Export | Export

Download the file using SaveAs in a place you can find it.

Open the resulting zip file, and then open the ics files contained within it in something like Notepad. Do NOT simply double-click it, or it will just open in Outlook.

Search the file for the error data. It may not be the first event you find using that search string.

It will be contained between the lines BEGIN: VEVENT and END: VEVENT.

Also contained in that section will be a line that starts reads UID: [data we want] @google.com

Copy the UID data without the @google.com at the end.

 

Go to the Google Calendar API and give it permission to access and edit your calendar.

You will find it here:

https://developers.google.com/apis-explorer/#p/calendar/v3/

You can authorise by moving the toggle at the top-right of the page that says “Authorize requests using OAuth 2.0”, and complete and execute the permissions page (tick the top box).

Scroll down the list of commands and click on “calendar.events.get” and complete the following parameters:

calendarId [email address for the calendar]

eventId [copied UID data]

Click “Execute” and it should find your identified event.

 

Go back to the Google Calendar API.

Click on “calendar.events.delete” and complete the following parameters:

calendarId [email address for the calendar]

eventId [copied UID data]

Click “Execute” and it should remove the offending event.

Refresh your calendar and the error message should not appear, unless you have other offending events to remove - in which case, repeat and rinse.

Good luck! I hope it’s of use to someone.

Just found the easiest answer.  Insert the photo into the header, select any text wrapping style for your picture other than "in line".  Then drag your picture wherever you want it in the document and click out of the header.  Done.

Details

  1. Click "View" tab and go down to "Header and Footer"
  2. Once you have the header and footer section open in your document, insert your picture into the header.
  3. Single click to select your picture.  Then go to the "Format" tab and select "Picture".

    jllatham say:

    This code below will actually build up the suggested name and place it on the clipboard ready for you to paste into the filename entry section of the File SaveAs dialog.

    It requires a reference to the Microsoft Forms 2.0 Object Library.  You set that reference from within the VBA Editor (press [Alt]+[F11] with your workbook open to open the VBA Editor).  Then from the Tools menu item, choose References and scroll down the list until you find

    Microsoft Forms 2.0 Object Library

    and check the box next to it.

    Here is the code - it goes into the ThisWorkbook code module just as before (but only one of the two routines can be there, although we could combine them).

    Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
    'Requires Tools --> Reference
    ' to the
    'Microsoft Forms 2.0 Object Library
    '
      Dim objData As New MSForms.DataObject
      'this places the suggested name on the 'clipboard' so you can paste it into the
      'File SaveAs dialog if you wish to use the suggested name
      'using [Ctrl]+[V] would be a quick way to paste it.
      objData.SetText Worksheets("Sheet1").Range("G4") & Worksheets("Sheet1").Range("F7") & ".xlsb"
      objData.PutInClipboard
    End Sub

    [Edit] Changed & ".xlsx" to ".xlsb" so that the macro can be saved in the more compact .xlsb file format, although if you do not want to do that, then use & ".xlsm" so the code gets saved properly with the workbook!!

    And Norman David Jones has a really great addition to it right below this.

    http://www.mediafire.com/download/s1mhffjaktnwyye/Averheim_001.xlsm

    And here is a copy with the additions that Norman David Jones came up with:

    http://www.mediafire.com/download/wrqm1yco5b176cc/Averheim_002.xlsm

    Now, I did change the code a bit - I've added some lines to ensure that the company name does not contain any characters that are invalid in filenames.  It will replace any of those with the underscore character, _  The potential invalid characters are: /|\?*:<>"  While none are too likely in a company name, this makes sure.

    Also, I've noted at times that Norman's autopopulate process seems to get ignored by Excel, or else there's something coming up between the time SendKeys is used and the SaveAs dialog actually gets displayed (as in Excel 2013 wanting to have you pick the location before asking for the filename).  If it doesn't get autopopulated, it is still on the clipboard so you can use [ctrl]+[v] to put in the name.

    Also!! In the code I changed the way the worksheet is referenced.  I have used what is known as the Code Name for the sheet instead of the name shown on the tab.  This means you can change the name on the tab and the code will still work as long as you base all future files on this one.  You can add and delete other sheets without affecting the code, and of course you can name the workbook anything you want to.

    Here is my revised code:

    Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
    'Requires Tools --> Reference
    ' to the
    'Microsoft Forms 2.0 Object Library
    '
      'these are for ensuring valid filename
      Const newChar = "_"
      Dim invalidList As String
      Dim newName As String
      Dim LC As Integer
      Dim objData As New MSForms.DataObject
     
      'this defines the list of invalid filename characters
      invalidList = "/\|:?*<>" & Chr$(34)
      'this pulls the 'raw' name into newName
      'referenced sheet by its Code Name so that you can
      'change the name on the tab without affecting the code.
      newName = Sheet1.Range("F7") _
       & Sheet1.Range("F4") _
       & Sheet1.Range("G4") & ".xlsm"
      'replace any invalid characters with _ character
      For LC = 1 To Len(invalidList)
        newName = Replace(newName, Mid(invalidList, LC, 1), newChar)
      Next
      'this places the suggested name on the 'clipboard' so you can paste it into the
      'File SaveAs dialog if you wish to use the suggested name
      'using [Ctrl]+[V] would be a quick way to paste it.
      objData.SetText newName
      objData.PutInClipboard ' ready to be pasted
    End Sub
