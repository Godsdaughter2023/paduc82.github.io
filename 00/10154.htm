The best answer to your decimal point problem does not lie in Excel. Rather, you should leave the values in Excel alone and tackle the problem within the mail merge field of Word. Select the mail merge field in your document and hit ALT+F9. It should look something like this,

{ MERGEFIELD myfield }

... and you need to add number formatting like this,

{ MERGEFIELD myfield \# "$  #,##0" }

Note that you cannot use a proportioned distance from the currency symbol to the first digit like you can with Excel's Accounting style format, but you can add a couple of static spaces that will increase the distance for all entries. For more information, see these two articles from Microsoft Support,

Format merged data
http://office.microsoft.com/en-us/word-help/format-merged-data-HP005187180.aspx

Numeric Picture (\#) field switch
http://office.microsoft.com/en-us/word-help/numeric-picture-field-switch-HP005186226.aspx?CTT=3

IMHO, merge data text, number or date formatting should always be performed at the destination (e.g. Word) and never relied upon for the formatting at the source (e.g. Excel).

"Col_index_num" in VLOOKUP - Microsoft Community

mmamoun say:

"Col_index_num" in VLOOKUP

The problem with "Col_index_num" is that when you insert a new column in the table the reference number takes info from the new column instead of keeping reference with old one. VLOOKUP should always takes info from selected colmun and must auto update if new column is inserted.
28 people found this helpful

amit tandon say:

You may enter a VLOOKUP as below:

=VLOOKUP("c",A2:F20,5,FALSE)

In this case the column no is 5 from where the value is being returned ie. column E. If you insert a column before column E, then the column no remains constant as 5 in the formula and the formula will return value from column E whereas the old column E has become column F.

If you want to return the lookup value from the updated column ie. in this case column F, then your original formula should be as below:

=VLOOKUP("c",A2:F20,COLUMNS(A2:E2),FALSE)

In this case "COLUMNS(A2:E2" return 5. But after inserting a column before column E the formula gets updated to "=VLOOKUP("c",A2:G20,COLUMNS(A2:F2),FALSE)" so that the value will noe be returned from column F (the earlier column E).
