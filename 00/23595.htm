I'm afraid this functionality has not yet been announced as part of the Mango update.

Each app on WP7 manages its data in a completely different way - a different model to that found on all other Windows operating systems (including Windows Mobile). There is no shared storage area and no unified management mechanism. This means, in practice, that each file type has its own storage mechanism (and its own sync mechanism, also) - Zune for music, podcasts and videos, Office for Word/Excel/Powerpoint documents, Adobe Reader for PDFs, and so on.

Sub Fixit()

Can Shared Mailboxes be removed from the Unlicensed Users view in - Microsoft Community

chocadmin9970 say:

Can Shared Mailboxes be removed from the Unlicensed Users view in Admin?

It's very difficult to use the Admin console to determine what, if any, unlicensed users there are in my tenant because the Unlicensed Users view option is filled with all the Shared Mailboxes from the environment.

Since Shared Mailboxes don't need a license, surely it would make sense not to list them in a view that indicates users that require licenses but don't have any? If it's not possible to accurately list actual users that don't have licenses then the view itself is pretty much useless.

Is there any way I can update the Shared Mailboxes to exclude them from the Unlicensed users view?

shyamal debnath msft support say:

Hi Choccie,

It can be achieve via a custom view in admin portal.

Admin portal > USERS > Active Users > Select a new > New view > Select an account to further restrict the view, choose User with Exchange mailboxes or archives and no license.

This is used to show the users who have been created after migration (like Stage Migration) but without licenses, in which scenario they still have Exchange mailboxes.

However, if a user had a license before and then the license was removed, it will not show up in the custom view as no mailbox is available for it.

Best regards,
Shyamal
