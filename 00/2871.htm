Many experienced Excel users steer well clear of shared workbooks because of problems that can be encountered.

=IF(INDIRECT("Associates!A" & ROW())<>"",INDIRECT("Associates!A" & ROW()),"")

~*kento*~ say:

We have had the same problem with shared workbooks and they definitely got worse when we went to Excel 2010, which is unfortunate, since Excel 2010 runs faster for us. Also, users were sometimes not being removed from the list of users who have the workbook open, so you might check that. Both of these were fixed when we removed all objects (including images) from the workbook. If you unshared the workbook you can run the macro below to remove all objects:

Sub EatButtons()
'Delete all Objects except Comments
    On Error Resume Next
    ActiveSheet.DrawingObjects.Visible = True
    ActiveSheet.DrawingObjects.Delete
    On Error GoTo 0
End Sub

 

We tried this in a desperate response to a tip found somewhere in the forum and have had no problems since. I don't claim to know why this worked for us, but hopefully it will work for you too. We use both the INDEX  and the INDIRECT functions in a large (30M) shared macro-enabled workbook and they do not seem to have been the source of the problem.

ken sheridan say:

Hi,

I need to validate whether all the first 2 letters from column 'Code' should be the same (Either 'GB' or 'OR') for all similar values in column 'AccountNumber'


Try this:

SELECT *
FROM YourTable As T1
WHERE LEFT(T1.Code,2) <> "GB"

OR LEFT(T1.Code,2) <> "OR"

OR EXISTS
     (SELECT *
      FROM YourTable As T2
      WHERE T2.AccountNumber = T1.AccountNumber
      AND LEFT(T1.Code,2) <> LEFT(T2.Code,2));

BTW your sample data does not agree with your description as the Code values for 129610896 start with 'GL' not 'GB'.  The above query would pick this up as well as the invalid combination of different code prefixes with the same account number.
