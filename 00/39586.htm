wunders say:

All the problems you describe are problems that could occur if the user you are logged on as does not have administrator privilege.  The quick check for administrator privilege is to right-click on the "Start" button.  If "Open All Users" is one of the choices that pop up, then you do have administrator privilege.  Otherwise, you must logon to your machine as the built-in "Administrator" user or a user in the "Administrators" group.

 

Assuming you do have Administrator privilege, things get more complicated.  User Rights can be granted or denied to any user -- even administrators -- by Group Policy.  If you are an administrator and you have XP Pro, then you might be able to check or change user rights using the Group Policy Editor (Start -> Run -> "gpedit.msc").  Look in the:

  Computer Configuration -> Windows Settings -> Security Settings -> Local Policies -> User Rights

path.  One of these user rights is "Change the System Time".  If your XP Pro machine was never part of a domain, I would consider also running an anti-malware program on your system.

 

If you have XP Home, then you do not have the group policy editor.  You will need to check / set / change your user rights using the "NTRights" command-line program provided with the Resource Kit.  Having problems like this with XP Home would raise my suspicions that you have a malware infestation.

 

" How to set logon user rights by using the NTRights utility"

  < http://support.microsoft.com/kb/315276 >

"It's not a true crash, in the sense that the Blue Screen was initiated only because the combination of video driver and video hardware was being unresponsive, and not because of any synchronous processing exception".

Since Vista, the "Timeout Detection and Recovery" (TDR) components of the OS video subsystem have been capable of doing some truly impressive things to try to recover from issues which would have caused earlier OS's like XP to crash.

As a last resort, the TDR subsystem sends the video driver a "please restart yourself now!" command and waits a few seconds.
If there's no response, the OS concludes that the video driver/hardware combo has truly collapsed in a heap, and it fires off that stop 0x116 BSOD.

If playing with video driver versions hasn't helped, make sure the box is not overheating.
 Try removing a side panel and aiming a big mains fan straight at the motherboard and GPU.
 Run it like that for a few hours or days - long enough to ascertain whether cooler temperatures make a difference.

If so, it might be as simple as dust buildup and subsequently inadequate cooling.

I would download cpu-z and gpu-z (both free) and keep an eye on the video temps

If File Explorer is not showing thumbnail previews for photos in Windows 10, You may try out the following solutions:

Check if thumbnail preview supported view is turned on

Make sure that you have turned on the thumbnail preview supported view for the folder where you have your photos. Note that every folder can have its own view setting, and when you change the view, it won’t be applied to all the folders on your PC.

To turn on thumbnail preview supported view, click on the View tab, and then click on one of the following views: Extra-large icons, Large icons, Medium icons, Tiles, Content.

Check if thumbnails are disabled

When thumbnails are turned off, the File Explorer displays icons of the default program for the file type instead of the thumbnail preview. You may try these steps:

  1. Open up File Explorer. Click the File menu and then click Change folder and search options to open Folder Options. If a file is selected in the particular folder, you might see just Options instead of Change folder and search options. Clicking Options will also open Folder Options.
  2. Switch to the View tab. Under Advanced settings section, uncheck the option labeled Always show icons, never thumbnails, and then click Apply button to enable thumbnails in Windows Explorer.
