gary's student say:

Here is an example.  Assume the single column is in Sheet1 Column A.  This little macro takes the data and copies it to Sheet2 in multiple columns.  The columns in Sheet2 are adjusted for the fact that the number of items may not be an even multiple of 4:

 

Sub reformat()
Dim s1 As Worksheet, s2 As Worksheet
Dim NumberOfColumns As Long, ColLength As Long
Dim I As Long, J As Long, K As Long
Dim Nitems As Long
NumberOfColumns = 4
Set s1 = Sheets("Sheet1")
Set s2 = Sheets("Sheet2")
s2.Cells.Clear
Nitems = s1.Cells(Rows.Count, "A").End(xlUp).Row
ColLength = Int(Nitems / NumberOfColumns) + 1
J = 1
I = 1
For K = 1 To Nitems
    s2.Cells(I, J) = s1.Cells(K, 1)
    I = I + 1
    If I = ColLength Then
        I = 1
        J = J + 1
    End If
Next
End Sub

1. ALT-F11  brings up the VBE window

    ALT-M opens a fresh module
3. paste the stuff in and close the VBE window

To remove the macro:

   1. bring up the VBE window as above
   2. clear the code out
   3. close the VBE window

To use the macro from Excel:

   1. ALT-F8
   2. Select the macro
   3. Touch RUN

To learn more about macros in general, see:

http://www.mvps.org/dmcritchie/excel/getstarted.htm

 

Once the macro has run, set the print area to the table in Sheet2 and specify the print scaling to fit on a single sheet.  You can change the "4" to another value if you desire.

 

Be warned that the print will be quite small.

What kind of device do you want to pair with your phone?

To delete an original pairing and start over again:

  • Turn off the bluetooth or move out of range of the device you want to unpair from the phone - you do not want to connect or attempt to connect to the phone right now.
  • Go to Settings > Bluetooth > tap the the Bluetooth toggle button to the ON position

  • Press & hold your finger on the device you want to delete & select the "delete" option from the popup menu (I selected the WM_Julie device in the screenshot below)

Now you're ready to pair the device again with your phone, so

  • Turn on the bluetooth radio for the device you want to pair with your phone (make sure you are in close proximity to it)
  • Do whatever you usually do to make the device discoverable to other bluetooth devices, if the device is not already discoverable (this varies, based on the device, so check the device's documentation). 
  • As long as the phone's screen remains at the Bluetooth screen, the phone will search for and be discoverable mode to other Bluetooth devices that are in range.  The phone should find any available bluetooth devices and any available bluetooth devices should find the phone - in the screenshot below, you'll see that my phone found my Roku Player.

  • Tap the "found" device you want to pair with the phone & follow any additional directions you are prompted to do. Tap the OK button to complete the pairing.

Be sure to consult the other the other device's screen or its documentation since every bluetooth enabled device is different from every other bluetooth enabled device.  The exact steps you'd follow may differ than the one's I've provided above.

joeu2004 say:

PS.... Alexander wrote:

=if(F3<=100,”0”,if(F3<=200,”10”,if(F3<=300,”20”,if(F3<=400,”30”,”40”))))

and 

=if(0<F2<=100,”0”,if(100<F2<=200,”10”,if(200<F2<=300,”20”,if(300<F2<=400,”30”,if(400<F2<=500,”40”)))))

The first formula is the best way to implement the intended logic, except for trying to quote the results, as I noted previously.

The second formula is incorrect syntax insofar as Excel will not interpret as you intended.  For example 0<F2<=100 is interpreted as (0<F2)<=0, which is FALSE<=0 or TRUE<=0, depending on the value in F2.  Either way, the result is FALSE because Excel considers all logic values (TRUE or FALSE) to be greater than any number.

The way to implement that logic in the second formula is syntax of the form AND(0<F2,F2<=100).

But that is overkill in this case because Excel evaluates the first formula left-to-right, stopping with the first true condition.

ken sheridan say: I'm unclear as to what you are trying to achieve here.

Your append query inserts the values of the controls into the tblProjectsCDCR table, but also selects rows from the join of three tables.  As no values from the tables are inserted into tblProjectsCDCR this would have the effect of inserting the same values into multiple new rows, however many are returned by the join of the three tables.  This seems an unusual thing to do, and could fail due to a key or index violation if the two columns are the primary key of tblProjectsCDCR  or otherwise uniquely indexed.

The second query similarly updates the two columns to the values of the controls, but without a WHERE clause to restrict the query will update every row in the table.  Again this seems an unusual requirement, and could fail for the same reasons.

If you just want to insert a single row into the table the query would be:

PARAMETERS
[Forms]![frmEnterProjectDetails]![txtproProjectNo] Text ( 10 ),
[Forms]![frmEnterProjectDetails]![frmCDCRNo].[Form]![txtcdcrCDCRNo] Text ( 20 );
INSERT INTO tblProjectsCDCR(procdcrProjectNo, procdcrCDCRNo)
VALUES([Forms]![frmEnterProjectDetails]![txtproProjectNo],
[Forms]![frmEnterProjectDetails]![frmCDCRNo].[Form]![txtcdcrCDCRNo]);

If, on the other hand, you want to insert multiple rows into the tblProjectsCDCR table each with the same procdcrProjectNo value, but with different procdcrCDCRNo values, one for each row in the subform then you cannot do this by inserting the values of the controls.  The append query would need to INSERT the values SELECTed  from a join of the relevant tables, referencing the [Forms]![frmEnterProjectDetails]![txtproProjectNo] control as a parameter in a WHERE clause to restrict the rows SELECTed, assuming of course that the subform and parent form are linked on procdcrProjectNo.

Otherwise, perhaps you could explain just what you do want to do .
