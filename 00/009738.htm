john gill say:

This error is the result of trying to import into an Exchange ActiveSync (EAS) account. The EAS service provider (Outlook.com / Hotmail) does not support importing CSV, or moving and importing items from a .pst into the Outlook.com/Hotmail folders.

You can copy calendar and contacts from a local data file into the Outlook.com/Hotmail folders.

Email messages can’t be imported or moved into the EAS folders.

If you need to import calendar or contacts from a CSV file, you’ll need to import them into a local .pst file then copy the appointments or contacts into the EAS account’s calendar or contacts folders. If they are in a .pst file already, open it using File, Open, Outlook Data File and copy the calendar and contacts to the Outlook.com folders.

To copy the items to the Outlook.com folder, use a list view, especially in a Calendar, then select all and right-click, drag and copy the items to the EAS account.

You can also use the Move > Copy to Folder command to copy the selected items to the Outlook.com folder.

If you have an organised set of data, I suggest you to backup and delete the messed up contacts and open the orgranized data and copy it into Microsoft Account and then sync it

If you chose to create a new folder, the default name of the folder will be the name of your old account and the folder will appear in the left pane under ‘Folders’. And all your old emails will be stored in this folder.

 

If you had Self-recovery of emails turned on in your account before renamed, you can try to recover some of the deleted messages yourself, please check this link. The recovered message will be presented on your Deleted folder.

Sub FormatAllTables()

    Dim t As Table
    For Each t In ActiveDocument.Tables
        t.PreferredWidthType = wdPreferredWidthAuto
        t.Range.Rows.HeightRule = wdRowHeightAuto
        t.TopPadding = 0
        t.BottomPadding = 0
        t.LeftPadding = 0
        t.RightPadding = 0
        t.Spacing = 0
    Next t

End Sub

The above code sets the table width to auto, the row height to auto, and it sets the cell margins to zero, for all tables in the active document.

It sounds like there may be an issue with your definition of "day".  If Task 1 finishes on Monday at 5:00 pm and that is the end of the working time defined in your calendar, task 2 can't start until the next working day - Tuesday at 8:00 am.

Unless task 2 is a milestone (zero duration) it can only be scheduled during working time.

julie sheets say:

Let's start back to at the beginning.

Working time is set in Change Working time on the Project Ribbon.  The default working time is Monday through Friday 8:00 am to 12:00 pm and 1:00 pm to 5:00 pm.  If that is not what you wish to define for your project, you'll need to change it.

https://support.office.microsoft.com/en-us/article/Change-working-days-for-the-project-calendar-323ff403-a676-428c-be11-6942bfb7210c?CorrelationId=5f456f8e-8004-4725-901f-4dcada928484&ui=en-US&rs=en-US&ad=US

If you change the working time to something other than default, you'll need to match those changes in Options.
