Can you upgrade Smartphone from Mobile 5 to Mobile 6? - Microsoft Community

bill burley say:

Can you upgrade Smartphone from Mobile 5 to Mobile 6?

Can smartphones be upgraded from Mobile 5 to Mobile 6?

elrendhel say:

Greetings Bill Burley!  Welcome to the Windows Mobile Owners Circle forums!

 

Yes, if your Smartphone vendor/reseller is providing the option to do so. 

 

Let me explain: Microsoft provides the original OS to the OEM manufacturers (like HTC) for a fixed rate.  Those OEM manufacturers in turn provide the OS image (WM5 to WM6) to the resellers (such as AT&T) as well.  The reseller then tweaks the OS slightly so as to provide their logos and specialized programs.  This means that your device is now specialized to the Resellers own wireless network.

 

As such, you would only want to install an OS upgrade from YOUR reseller/vendor, in order to maintain your warranty and keep compatibility with your vendor's network.

 

Contact your wireless network to see if they have an upgrade available for your device. 

 

Alternately, you may of course install an upgrade from the OEM Manufacturer, but I would expect that this would invalidate your warranty and may ruin your device.  Use this option at your own risk!

Yes. Keep one "master" frontend for all the changes, and create an .accde "compiled" frontend for the users. There are numerous websites (including Tony's, which I posted upthread) describing how to distribute the new frontend without disrupting the user's work.

One way is to have code in the startup form's Open event to check a defined backend folder for a new frontend, and download and run it if there is one.

Again, it's simply not possible to make changes to the VBA project (any module, or any form or report containing a module) in a shared database in 2007 or 2010; all the VBA code is kept in one "chunk" in the system tables, and changing it is an "all or none" operation.

jllatham say:

I'm afraid that without a backup copy of the "A" file, you may have totally lost it.  "B" has now overwritten it on the disk.

As for setting the computer back a day or two, if you are talking about System Restore - that will not help you because System Restore specifically does not 'backup'/'restore' your data files, and files created by your applications fall into that category.  It deals only with system related files and folders.

Think hard: who have you ever sent a copy of that "A" file to as an email attachment, or copied it to a CD or DVD or even a thumb drive.  These are all potential sources of that file, that while they may be old, would at least save you from a total loss if its data.  If this is on a corporate system, does that system get backed up routinely?  If so, then that would be the place to look for a recent backup copy.
