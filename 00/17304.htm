Mary,

While we wait for one of the experts to chime in, a couple of suggestions:

  1. When you create your field (presumably using <Ctl><F9>), don't enclose the basic formula in another set of field braces. In other words, after pressing <Ctl><F9>, just type
                        =b2-b3\# "#,##0"

    within the one set of braces. The final result should look like this:

                      { =b2-b3\# "#,##0" } 
  2. Once the formula is in place, select it and press <F9> to update the result. (Otherwise you'll probably get that dreaded empty cell forever.)

3. Then press <Alt><F9> to toggle from the formula to the result and see if it worked . . .

Hope this helps!

    Why does it run so slowly?  Sometimes the macro doesn't run at all.  It just hangs up.  Usually it just runs slowly. 

    a) It depends on the macro why it is slow. Most of this macros use a lot of SELECT, SELECTION, ACTIVECELL and/or color cells, changes font.

    Every "eye catcher" needs more time in never versions, because they support tons of this things. It's like a library, when you just have 5 books, you'll find the right one quickly. When you have 5.000 it needs some time.

    b) The macro doesn't hang, it just needs a lot of time. In modern windows systems the OS "pings" each window from time to time and when the window doesn't answer, the OS change the "state" of the window to "not responding".

    Try to call DoEvents in the inner loops in your macro, that should help on this.

    The problem occurs because the font characteristics in the list template have become corrupted.  It can be fixed by running a macro containing the following code:

    Sub Fixit()

    Dim templ As ListTemplate

    Dim lev As ListLevel

    For Each templ In ActiveDocument.ListTemplates

        For Each lev In templ.ListLevels

            lev.Font.Reset

        Next lev

    Next templ

    End Sub
