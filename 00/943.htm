micah [msft] say:

Hello everyone,

I have another update with helpful hints that has helped us fix the issue for almost everyone and if these steps don't help you, I'd like to discuss further over email. I have met with a couple more of you and we have found 3 distinct issues resulting in the same behavior:

1) The "DDE" option in File -> Options -> Advanced -> "Ignore other applications that use Dynamic Data Exchange (DDE)" is checked, causing files to fail to open.

     a) The solution is to go into the options and uncheck the option. This is a feature useful for some programmability scenarios, but most people should have this turned off. We are checking to see if there are any places in our code that are turning this option on by default, but so far haven't found any.

2) All of the windows in a workbook are hidden, so the file appears like it didn't open.

     a) Go to View -> Unhide (in the "Window" section) and unhide any hidden windows.

3) The DDE exchange between Windows and Excel is failing, causing files to not open.

     a) The temporary workaround is to try reopening the file while Excel is already open (either from File -> Open or by double clicking on the file again) without closing Excel. We are actively investigating what is causing this and are pursuing a fix as quickly as we can, but to this point we still haven't been able to cause this state on any of our test computers. If you are experiencing this issue, please reply to the thread and let me know, because we would be very interested in seeing the error logs from your computer to determine exactly what is going wrong.

Also, there is a slightly different issue that enough of you have encountered that I felt I should mention it here. Files are failing to open or opening in Protected View due to "corruption" messages.

     a) There are two causes of this. One is xls files (the older Excel file format) that contain hyperlinks that were created in Windows 10. We are actively investigating a fix for that and should have more news soon.

     b) The second is files opened from email attachments or the web. This is expected behavior and was designed that way to protect from accidently opening files that contain malicious content.

Again, if you are seeing 3 (where if you double click on desktop and no file opens, but you try again with Excel already running and the file opens fine), please reply to this thread and we will private message you to follow up in email so that we can gather logs from you to determine the cause of the issue.

Thanks,

Micah
