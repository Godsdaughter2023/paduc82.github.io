davidf2 say:
I'm not a frequent Publisher user but use one critical file to update images as needed on a Sharepoint site.  My monitor was replaced yesterday with a widescreen monitor and the standard font at the only resolution where it wasn't blurry requires a larger DPI for me to be able to function.  Adjusting the DPI seems to be affecting my Publisher document.  Now when I go to save as picture in Publisher at 150dpi it adjusts and saves to 114dpi.  This affects the image quality and size.  After two hours with my IT department, a new video driver and a new cable the issue isn't solved and I was told they don't know why this is happening and they have nothing else to offer me.  If I switch my sytem back to the old DPI setting it will save the graphic appropriately but as this requires my system to restart every time this isn't a practical long term solution.  Any advice is appreciated.


I am not sure I have a solution for you but I may offer a little insight.

 

You did your original design with a standard monitor that was probably set at the traditional 96 dpi setting. Publisher is designed to work with that standard. If you save a text box as an image then you get the options of 96 dpi for the web, 150 dpi for print and 300 dpi for commercial print. In reality dpi is dot per inch and is a print term and your monitor is actually set at ppi or pixels per inch which is different. When you increased the text size for your new higher resolution monitor you threw off those defaults. Then when you save as an image Publisher saves at 114 dpi to compensate and keep everything proportional for your page layout.

 

Publisher uses 'absolute positioning' to keep the text, pictures and other design elements aligned with each other. When you change the text size or resolution on your monitor Publisher has to change the resolution of the publication or any image you save to compensate and maintain the layout. This is part of the reason that MS gave up on the web building functionality in Publisher. The web files are produced for 96 ppi monitors and the pages 'break' when viewed on monitors set at a higher ppi. The text increases in size but the other elements do not and thus the layout is broken.

 

I know that is clear as mud and I am sorry that I cannot explain it better, but the bottom line is you may not be able to do anything about it. But if you are posting these images on the web where they are only going to be viewed on a monitor and not printed then you did not need 150 dpi anyway. Chances are the browser is resampling the images and viewing them at 96 ppi or even 72 ppi.

 

You might try redoing your entire document at the higher resolution and see if you can get the image results you want. You could try an experiment and just create a new publication, create a text box, insert some clip art or an image and/or a shape and group those objects, save as a 150 dpi image and see if you get a differ result.

 

If you really need a higher resolution for print, then perhaps save at 300 dpi which will also result in a smaller dpi and use that image, and if necessary you can edit, resample and reduce that image to 150 dpi with a third party image editior.

 

Sorry I don't have a better answer.

musaddict say:

There is an alternative connection method which should allow you to pick union queries.

Try:

Data>From Other Sources>From Microsoft Query>MS Access Database*

Use the Dialog window to navigate to the database that contains your union query.  After your source database is selected, the resulting Query Wizard Dialog window should display your union query along with the rest of the items you would expect.

This did not fix my issue, but it led me on the right path! The correct answer is below; I would mark my reply as the correct answer, but I'm not allowed to :(

Turns out the nz() function is not picked up by Excel, so it assumed that the query had junk expressions and omitted them. Attempting to use your method flat out told me it can't import the query because it found an Undefined Function "Nz" In Expression.

So, I changed my SQL expressions in Access from:

Val(Nz([My Query].[My Field],0))

to:

Val(IIf(IsNull([My Query].[My Field]),0,[My Query].[My Field]))
