Any thoughts?

a. user say:

You may have found the answer, but if not, 
 a. DDE is easily blocked by other programs using DDE on your system. it is almost impossible to avoid such blocking except by identifying and removing the blocking software.
 b. it can be handy to test the DDE link using another "client" program. For example, if you have Word, you can 
  - create a new document
  - use ctrl-F9 to insert an empty field { }
  - type DDE theServiceName theTopicName theItemName
where you should be able to get theServiceName, theTopicName and theItemName from your Excel sheet
so you see
{ DDE theServiceName theTopicName theItemName }

Then select the field, press F9 to update it, see what messages you get, then press Alt-F9 and see what the results are, if any.

If Word appears to hang, you'll probably need to wait one or two minutes for the DDE connection attempt to "time out"

If that does not work it can be helpful to try the standard topic/item combinations

{ DDE theServiceName SYSTEM TOPICS }
{ DDE theServiceName SYSTEM SYSITEMS }
{ DDE theServiceName SYSTEM FORMATS }

just to see what happens.

Your macro uses the AutoFilter, which works, but manipulating the AutoFilter and get only some specific columns is not so easy, so I suggest to go a different way.

Use RANGE.FIND and search a column for a specific value, combine all found cells. (I've posted my FindAll routine many times here, give me a sign if you cant find it.)

For the next column, build a range that intersect with the found rows. Search that range for the next value as in the previous step.

After you've searched all criterias, build a range that intersect with the desired columns and the rows of the search result and copy it to the destination sheet.

Andreas.

Efficiency is always based on the requirements. You said the headings in the sheets can vary, so we need a code that can handle that scenario. And as more dynamically a code has to work, the longer it takes to run. Don't worry, IMHO the code should runs far below one second.

I suggest that you debug that macro line by line (except FindAll, assume that it works), read the comments and you should be able to understand how it works.

If you don't know how to debug a macro have a look into this tutorial:

Option Explicit

Sub Main()
  Dim AllSheets, AllHeaders, AllCriterias
  Dim ThisSheet, ThisHeader, ThisCriteria, OutputHeaders
  Dim Header As Range, All As Range, Dest As Range, DestHeader As Range
  Dim i As Long

  'Setup section:

  'We loop through this sheets
  AllSheets = Array("Sheet1", "Sheet2", "Sheet3", "Sheet4", "Sheet5")
  'We search for this headings in row 2
  AllHeaders = Array("FullTime", "Grade")
  'And get the cells for this criterias
  AllCriterias = Array("Y", "A")

  With Sheets("OS tracker")
    'Clear previous results, from row 3 downwards
    .UsedRange.Offset(3 - .UsedRange.Row).ClearContents
    'The output goes to this cell:
    Set Dest = .Range("A3")
  End With
  'Only this headings are copied
  OutputHeaders = Array("Serial #", "name", "FullTime", "Age", "Notes")

  'Check the setup section:

  'Be sure that you has set a criteria for each header
  If UBound(AllHeaders) <> UBound(AllCriterias) Then
    Err.Raise 13, , "Number of Headers <> Criterias"
  End If
  'The lower bounds of the arrays must match!
  If LBound(AllHeaders) <> LBound(AllCriterias) Then
    Err.Raise 13, , "LBound Headers <> Criterias"
  End If

  'We are ready to run:

  'For each sheet
  For Each ThisSheet In AllSheets

    'Step 1: Searching

    'No results for this sheet
    Set All = Nothing
    'For each header/criteria
    For i = LBound(AllHeaders) To UBound(AllHeaders)
      'Get the values from the arrays (makes the following easier to read)
      ThisHeader = AllHeaders(i)
      ThisCriteria = AllCriterias(i)
      'Find the header in row 2
      Set Header = Worksheets(ThisSheet).Rows(2).Find(ThisHeader, _
        LookIn:=xlValues, LookAt:=xlWhole)
      'Found?
      If Header Is Nothing Then GoTo SkipSheet
      'Did we have previuos results?
      If All Is Nothing Then
        'No, expand the header to the entire column
        Set Header = Header.EntireColumn
      Else
        'Get the rows in the previous results only
        Set Header = Intersect(Header.EntireColumn, All.EntireRow)
      End If
      'Find the criteria in this column
      Set All = FindAll(Header, ThisCriteria)
      'Found?
      If All Is Nothing Then GoTo SkipSheet
    Next

    'Step 2: Copy the result rows in the headings step by step

    For Each ThisHeader In OutputHeaders
      'Find the header in row 2
      Set Header = Worksheets(ThisSheet).Rows(2).Find(ThisHeader, _
        LookIn:=xlValues, LookAt:=xlWhole)
      'Found?
      If Header Is Nothing Then GoTo SkipHeader
      'Find the header in the output sheet
      Set DestHeader = Dest.Parent.Rows(2).Find(ThisHeader, _
        LookIn:=xlValues, LookAt:=xlWhole)
      'Found?
      If DestHeader Is Nothing Then GoTo SkipHeader
      'Copy the rows
      Intersect(All.EntireRow, Header.EntireColumn).Copy
      'Paste the values into the desired column
      Dest.Offset(, DestHeader.Column - Dest.Column).PasteSpecial xlPasteValues
SkipHeader:
    Next
    'Next empty row
    'Note: I assume that all cells in column A are filled!
    Set Dest = Dest.Offset(Rows.Count - Dest.Row).End(xlUp).Offset(1)
SkipSheet:
  Next
  'Copy mode off
  Application.CutCopyMode = False
End Sub

Private Function FindAll(ByVal Where As Range, ByVal What, _
    Optional ByVal After As Variant, _
    Optional ByVal LookIn As XlFindLookIn = xlValues, _
    Optional ByVal LookAt As XlLookAt = xlWhole, _
    Optional ByVal SearchOrder As XlSearchOrder = xlByRows, _
    Optional ByVal SearchDirection As XlSearchDirection = xlNext, _
    Optional ByVal MatchCase As Boolean = False, _
    Optional ByVal SearchFormat As Boolean = False) As Range
  'Find all occurrences of What in Where
  Dim FirstAddress As String
  Dim C As Range
  'From FastUnion:
  Dim Stack As Object 'Dictionary
  Dim Temp() As Variant
  Dim i As Long, j As Long

  If Where Is Nothing Then Exit Function
  If SearchDirection = xlNext And IsMissing(After) Then
    'Set After to the last cell in Where to return the first cell in Where in front if _
      it match What
    Set C = Where.Areas(Where.Areas.Count)
    'BUG in XL2010: Cells.Count produces a RTE 6 if C is the whole sheet
    'Set After = C.Cells(C.Cells.Count)
    Set After = C.Cells(C.Rows.Count * CDec(C.Columns.Count))
  End If

  Set C = Where.Find(What, After, LookIn, LookAt, SearchOrder, _
    SearchDirection, MatchCase, SearchFormat:=SearchFormat)
  If C Is Nothing Then Exit Function

  'Initialize our internal stack
  Set Stack = CreateObject("Scripting.Dictionary")

  FirstAddress = C.Address
  Do
    Stack.Add Stack.Count, C
    If SearchFormat Then
      'If you call this function from an UDF and _
        you find only the first cell use this instead
      Set C = Where.Find(What, C, LookIn, LookAt, SearchOrder, _
        SearchDirection, MatchCase, SearchFormat:=SearchFormat)
    Else
      If SearchDirection = xlNext Then
        Set C = Where.FindNext(C)
      Else
        Set C = Where.FindPrevious(C)
      End If
    End If
    'Can happen if we have merged cells
    If C Is Nothing Then Exit Do
  Loop Until FirstAddress = C.Address

  'FastUnion algorithm © Andreas Killer, 2011:
  'Get all cells as fragments
  Temp = Stack.Items
  'Combine each fragment with the next one
  j = 1
  Do
    For i = 0 To UBound(Temp) - j Step j * 2
      Set Temp(i) = Union(Temp(i), Temp(i + j))
    Next
    j = j * 2
  Loop Until j > UBound(Temp)
  'At this point we have all cells in the first fragment
  Set FindAll = Temp(0)
End Function
