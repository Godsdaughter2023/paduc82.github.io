•The basic input/output system (BIOS) does not detect the hard disk.

•The hard disk is damaged.

•Incorrect or malformed master boot record (MBR).

Refer to the following methods and check if it helps:

Method 1: Disconnect all the unnecessary external devices connected to the computer (except the mouse, keyboard and the recovery media) now check if the operating system loads.

Method 2: Verify the BIOS settings

Verify the computer's BIOS settings to make sure that BIOS lists and recognizes the hard disk. See the computer documentation or contact the hardware manufacturer for information about how to verify the BIOS settings.

After you verify that the computer's BIOS detects the hard disk, check if the hard disk is selected as boot device, then restart the computer, and then test to determine whether the issue is resolved. If the issue is not resolved, or if the computer's BIOS cannot detect the hard disk, you may have issues with your hardware. a. Boot from the Windows 10 TP reinstallation DVD/USB.

b. Click on Repair your computer.

c. Click on Troubleshoot and then select Advanced Options.

d. Click on Command prompt and type the following commands and press Enter after each command:

Bootrec /fixmbr

e. Restart the computer.

cbarnhorst say:

Do not use a primary or production computer for preview builds.  Period.  That is Microsoft's instruction and the experience of hundreds of users who ignored it.  Many of those folks never even bothered to backup their systems, which made their plights untenable.

Use a spare device or a new partition on your present computer, as Joe suggests, or a virtual machine on your present computer if you do not have a test box.  The big advantage to Hyper-V virtual machines is checkpoints to instantly revert to a known-good state.  The big advantage of a new partition is that Microsoft gets useful telemetry about your hardware configuration.

If you don't want to do one of those then I respectfully suggest that you are not equipped to run previews.  It is pretty common to have to reinstall Windows to repair a device that has been rendered inoperable by a blocking bug.  That is not painful on a test box but is very painful on one's every-day device.
