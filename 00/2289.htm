Excel 2013 PowerPivot wouldn’t come up -- here’s what solved it for me, maybe this will help solve the problem for others

 

http://answers.microsoft.com/en-us/office/forum/office_2013_release-customize/excel-2013-powerpivot-I-come-up-heres-what/4d5993d1-9ad9-452e-b1c6-4a6305d1e3bb

 

I installed Office 2013 (typical install - no customization and no errors during it) earlier this week and could not get PowerPivot to come up in Excel. After Office had been installed, I went to the Add-Ins under the Excel options on the start page and clicked on the PowerPivot COM add-in, then selected OK. As it said it would come up after I restarted Excel I did just that, only to see that the PowerPivot tab wasn’t ackg. Next, I went to the Developer tab and selected it from the COM add-ins drop down. I saved the worksheet (even though I knew it wouldn’t make a difference but I just had to do it), exited Excel and restarted it. Again, no PowerPivot, and when I looked at the COM add-ins dropdown on the Developer tab, it was no longer selected.  I checked an Office 2013 Excel help article, which included the steps I had just tried and it also suggested that I make sure the add-in wasn’t disabled; it wasn’t. I then spent 2 days trying every suggestion that I could find on the internet, I repaired Office 2013, I uninstalled and reinstalled it, and I also messed with every Trust Center setting combination possible - all without success. So this morning I decided to go back to basics, which is when I finally came up with what needed to be done to get this to work - and it’s something that shouldn’t have even been necessary. You’ll understand why as you keep reading.   

I thought the typical, non customized install would take care of what it needed for the COM add-ins to work, but that was incorrect apparently. For some incomprehensible reason the path to the PowerPivot add-in wasn’t considered a “trusted location” and had to be manually added as one in Excel’s Trust Center. 

I added this path on that tab:   C:\Program Files\Microsoft Office\Officel5\ADDINS\PowerPivot Excel Add-in\PowerPivotExcelClientAddIn.dlI 

To do this, go to Excel Options -> Trust Center -> Trust Center Settings and select the Trusted Locations tab, which is where you add it. While on this tab you’ll probably notice a couple of things:

1.    There are already several of the Excel locations for it’s extension in here as a default. (Not sure why this one wasn’t included as a default too.)

2.    You may notice on the bottom left of this tab there is an option to select “Allow Trusted Locations on my Network” but it says “not recommended”. (So does this mean we have to manually add all of the paths to the Trusted Center that weren’t added during the install of the Microsoft product?)

After I added the path, I wanted to make sure all possible “trust issues” would be addressed, so I checked the Trusted Publishers tab to see if there was a row with the  Microsoft Corporation (found in the “Issued to” column) and Microsoft Code Signing PCA (found in the “Issued By” column). It was - luckily during my numerous attempts to make this work it got added when I had selected an option presented in the trust center to make the publisher of this add-in a trusted one.     

 

After I did these things, voila, I now have PowerPivot up and running. But man, this took far too much effort and was very frustrating, which is why I’ve added this post - hopefully this helps reduce the amount of frustration for someone else with this same issue.

There are probably more elegant/less geeky ways to accomplish this but:

Make a copy of your file. DO NOT work on the original, because minor mistakes in what follows could corrupt your file.  Now, working on the COPY:

Change the file's extension to .ZIP

Doubleclick to open it as a ZIP file

Doubleclick the "ppt" folder

Doubleclick the "media" folder

Inside the "media" folder, you'll find any pictures/sounds/videos you've inserted into your presentation.  With luck, your recorded sounds will have names that match the slide sequence.  You can doubleclick them to verify that you've got the right one.

Use an external sound recording program to re-record your narration (Audacity is excellent and free).  Save the recording to the Windows desktop (or any other convenient location) under same name and type as the sound you want to replace, the one you found inside the unzipped PPT file. 

Delete the sound file from inside the ZIP, then drag the newly created recording (same name, same file format) into the ZIP to replace it.

Close the ZIP, rename it to Whatever.PPTX and fire it up.

XPSP3 - Explorer Window at startup and no MSE tray icon - AUTO FIX - Microsoft Community

robinb9 say:

XPSP3 - Explorer Window at startup and no MSE tray icon - AUTO FIX INSIDE!

If the icon doesn't appear on the task bar it could be one of two things.  Either the Microsoft Security Essentials application has failed to start, resulting in the icon being missing, or it could be that the icon is in fact there but is hidden. 

If the icon is hidden, you can right click on the taskbar and choose Properties.  Here you will see an option to "Hide Inactive Icons" which can be unchecked as a test, then click OK.  This will display all the icons that are hidden or inactive; please check to see if you see the Microsoft Security Essentials icon appear. 

If not, we can try to launch the application manually to see if it starts.  Check the desktop or under Start / Programs for the Microsoft Security Essentials icon and try to launch it manually.  If you receive an error message make sure to write it down.  After launching it manually, check again for the icon in the taskbar. 

Lastly, the problem might be that Windows is not configured to launch the Microsoft Security Essentials application when you boot the computer.  Click Start then Run, and type MSCONFIG and click OK.  Here click on the Startup tab and ensure that Microsoft Security Essentials is listed here and it is checked.

If all else fails you can contact support for assistance.

I hope this helps!
