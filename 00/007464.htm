Excel VLookup - Microsoft Community

karennicholson say:

Excel VLookup

I would like to use VLookup to return values from a sheet that will have duplicates.  Where I run into trouble is usually Vlookup returns the first value found.

I have a sheet with 100+ rows.  I want to return all values for a particular part number.  There may be 1 or there may be 30.  I want to see all of them.

Can this be done with a VLookup?

hansv mvp say:

Not with the built-in VLOOKUP function. You'll find a modified version below, based on code by Steve Aprahamian (aka sdckapr):

Function VLookupAll(vValue, rngAll As Range, iCol As Integer, Optional sSep As String = ", ")
    Dim rCell As Range
    Dim rng As Range
   
    On Error GoTo ErrHandler
    Set rng = Intersect(rngAll, rngAll.Columns(1))
    For Each rCell In rng
        If rCell.Value = vValue Then
            VLookupAll = VLookupAll & sSep & rCell.Offset(0, iCol - 1).Value
        End If
NextCell:
    Next rCell
   
    If VLookupAll = "" Then
        VLookupAll = CVErr(xlErrNA)
    Else
        VLookupAll = Mid(VLookupAll, Len(sSep) + 1)
    End If
    Exit Function
   
ErrHandler:
    Select Case Err
        Case 13
            ' Type mismatch - resume
            Resume NextCell
        Case Else
            VLookupAll = CVErr(xlErrValue)
    End Select
End Function

Use like this:

=VLookupAll(A1, $D$1:$D$100, 2)

to look up the value of A1 in D1:D100 and concatenate the corresponding value from column E  (because the iCol argument is 2) for each match, separated by a comma and a space.

  2. number
  3. percent
  4. currency
  5. yes/no
  6. date/time
  7. duration
  8. keywords
  9. combination
  10. formula 
  11. integer
karl timmermans [outlook mvp 2012-2018] say:

UDF's represent how data is stored (a field of information) whereas <controls> represent how data is shown so the answer to your question is no.

To be even more specific if the question is further refined to:

"Is there a way to create a dropdown list for a contact that can be dynamically populated based on the underlying values contained in a given field to which that dropdown list control is attached?"

Answer is also no.

#1 - UDF's represent singular field value which may be comprised of multiple indivdual values separated by a comma (a keyword field). That value is based on the selection(s) made from the given control - the values of which are defined at the "control" level - not the associated field level.

#2 - There is no way to "automatically dynamically" populate a dropdown list control which can only be added to a custom form
