It is always good to make a copy as a backup before running Compact and Repair.  But you should not lose any records as a result.  It is possible sometimes for data to become corrupt (actually at any time), so having backups is a good practice.  Also, if it isn't split with each person using their own copy of the frontend, it would be the thing to do as well.  In a non-split database you have more of a chance of corruption occuring whereas you have much less chance if there is a database for just the tables and then a frontend consisting of everything else and have each user have a copy to run from their machine.  I have a free tool on my website http://www.btabdevelopment.com which will enable auto updating for the frontend where you don't have to worry if they have the latest copy.

a. user say:

In the case where your document is set up as a mail merge main document connected to an OLE DB data source (any OLE DB data source, Word does indeed interpret all other dates, such as the ones you are talking about, that are in aa/bb/cccc format, as US format dates, I.e. the aa will be treated as the month, unless it has other cues, e.g. The month being > 12.

There may be other situations in which the same assumptions are made, but that's the only one I know of.

The supplier would ideally provide the dates in yyyy-mm-dd format, which AFAICR is always interpreted correctly.
