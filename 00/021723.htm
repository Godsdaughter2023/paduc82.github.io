rob koch say:

Though your detection of this JavaScript redirection is occurring on a MAC, the basic issue should be the same as that for any system making such a detection.  So looking at the Microsoft Encyclopedia entry for this detection we find the following in the Technical Information tab.

Trojan: JS/Redirector.BZ

Threat Behavior

Trojan:JS/Redirector.BZ is JavaScript code that is embedded in a malicious or hacked webpage.

When you go to a website that contains the code, your browser can be redirected to another website that can use exploits to download malware onto your PC.

What this means is that the detection is likely occurring as your system is accessing some web page, attempting to redirect it to another possibly containing malicious exploits and malware.

So the point would be that the continued detections mean that something on your system is accessing a web page.  If you are browsing at the time, then the web page(s) being accessed at the time may contain this script.  However, if it's occurring when you're seemingly using only local apps or doing nothing at all, then your system may already contain some form of simple JavaScript malware or browser hijacker that's causing these detections and notifications.

It would still make sense to contact the support for your security application, but understanding the above should help you to determine where the detections might be coming from.

< EDIT > I should also mention that typically such a redirection detection would be a transient condition, with the primary concern the potential for the follow-on exploit attempts to install malware from the redirected website.  If the detections have stopped since blocking and adding the malware into the "chest", actually just a pretty name for what's commonly called a quarantine by everyone else in the world, then the entire situation is likely over.

Unless you receive an additional warning relating to either exploit or other malware detections closely following this redirection notification, I wouldn't be concerned myself.  In fact, the indication here is that your protection both detected and then blocked the initial redirection, which should hopefully mean that the redirection itself had failed, so no additional exploit or malware should thus be encountered.
