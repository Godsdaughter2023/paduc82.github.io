a. user say:

It's not a good idea to have only one account in Vista.  If it becomes corrupt, you're in big trouble.  Vista also prefers to have multiple accounts.  I suggest at least two Administrator accounts (neither of which is the built-in account) and a standard account for every user on the system (you should not use an Administrator account on a day-to-day basis for security reasons - it makes it easier for a hacker to gain access).

It is possible that the other administrator account is the built-in administrator account which cannot be removed - only disabled.  That's probably the case here.  If so, this account should be disabled anyway.

I hope this helps.

Good luck!

Hi Manuela,

No, there isn't.

Please try the following steps to troubleshoot this issue:
1. Remove Skype meeting Add-in. Run Outlook client as administrator (right click Outlook icon and choose “run as administrator”) > File menu in Outlook client > Options > Add-ins > Go.. > Choose Lync meeting scheduling Outlook Add-in or Skype meeting Add-in for Microsoft Office > Remove.
2. Re-install Skype meeting Add-in. Click on Add > File name: C:\Program Files (x86)\Microsoft Office\root\Office16\UCAddin.dll (if you use Outlook 2016); C:\Program Files (x86)\Microsoft Office\Office15\UCAddin.dll (if you use Outlook 2013) > OK.

3. Restart Outlook client > try to schedule a SFB meeting and check to see if it works.
4. Update office client to the latest version.
5. Quick repair Office application in Control Panel.
6. If the issue persists, please go to event viewer to check if there are error logs related to Outlook or Skype for business. If so, please post the detailed log in the forum for further investigation.
