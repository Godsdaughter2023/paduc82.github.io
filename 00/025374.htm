doug robbins - mvp office apps & services (word) say:

Use a macro with the following code

Dim docsource as Document
Dim doctarget as Document
Dim i as Long

Dim FD as FileDialog

Dim strFolder as String

Dim strFile as String
Dim rng as Range

Set doctarget = Documents.Add
Set FD = Application.FileDialog(msoFileDialogFolderPicker)
With FD
    .Title = "Select the folder that contains the documents."
    If .Show = -1 Then
        strFolder = .SelectedItems(1) & "\"
    Else
        MsgBox "You did not select a folder."
        Exit Sub
    End If
End With
strFile = Dir$(strFolder & "*.docx")
Set doctarget = ActiveDocument
With doctarget
    While strFile <> ""
        Set docsource = Documents.Open(strFolder & strFile)
        For i = 1 To docsource.Tables.Count
            docsource.Tables(i).Range.Copy
            .Range.InsertAfter vbCr
            Set rng = .Range
            rng.Collapse wdcollapsend
            rng.Paste
        Next i
        docsource.Close wdDoNotaSaveChanges
        strFile = Dir$()
        DoEvents
    Wend
End With

The general steps for allowing an app through Controlled Folder Access:

1. Click on the notifications icon at the far right in the taskbar.

2. Write down the file path provided in the notification.

3. Click on the notification, and then on the Add button (+).

4. Navigate to the file identified in the notification, then select and “open” it.

The file path in the Unauthorized Changes Blocked notification does tend to be truncated – although there’s usually enough of it there to locate the app without any trouble if you just assume that it’s in the Program Files or Program Files (x86) directory. If the file path is truncated to the point where you can’t locate the blocked app in the file-pickerdialog; then open Event Viewer; navigate to the Windows Defender Operationallog; and locate the blocking event (Event ID 1123):

1. Right-click on the Start button and select Event Viewer.

2. Navigate to Applications and Services > Microsoft > Windows > Windows Defender > Operational

3. Filter for (or just look for): Event ID 1123

The  CD Key is on a yellow or bright orange sticker on the CD package. Always keep your CD key and install CD handy. If you ever lose it, it is very difficult to get a replacement.

The Product ID is given during the Setup Assistant process. The Product ID will look like this: 11111-130-0001111-12345). It is used to register. Write this Product ID down or take a screen snap. It's used for support only.

"Registration" for Office 2008 has nothing to do with the operation of the software - unlike the Activation required for MS Windows software and now for Office for Mac 2011. It's little more than a marketing tool. It will put you on a Microsoft email list.

scottgem say:
DateSerial(CInt(Left(datefield,4)),Cint(Mid(datefield,5,2)),Cint(Right(datefield,2)))

Dates are stored as a serial number counting the dates since 12/30/1899. You format the date field to include leading zeros.
