a. user say:

It is not uncommon to get that error after patching SEO or tracker stuff into the beginning of an html file.

The elements <html> <head> <body> are all optional in HTML.  Their end tags are optional too.  If they are not present in the sourcecode, they will be inferred and automatically inserted where they belong,

If an element like an <iframe> or <img> is mistakenly pasted into the <head> (along with some SEO script for example), it is incorrect there and will trick-up the auto tag generation.  The <head> will get closed at that point with a </head> end tag, and a <body> start tag will also be automatically inserted there.  (because iframe and img tags belong in the body). The original </head> end tag which exists later down in the source will then be reported as a stray end tag.  The unexpected <body> which got auto-inserted can cause an avalanche of errors in all that follows, as you describe.

In the screenshot above, an invisible </head><body> was auto-inserted before the <img>.  Without knowing that, the HTML1512 and HTML1514 warning messages are bewildering.

Here's that simplified sample code.  You can copy+paste it to  https://validator.w3.org/#validate_by_input  and see the same error.

<!doctype html>
<html>

<head>
<title>x</title>
<img src=trans.gif alt=beacon>
</head>

<body>
</body>
</html>

But there are lots of other reasons, including just a harmlessly benign error that has existed all along, that might produce alarming console messages.  If you could post a screenshot of the F12 console errors, maybe it will be possible to figure out what is happening without needing to study the whole sourcecode.
