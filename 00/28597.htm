Create error if field in form is left blank

Sorry if this is a redundant question but I didn't find anything like it when searching.

I have a form and I want to make sure that certain fields are not left blank.  What would I do to make that happen?

djsteele, mvp say:

Put logic in the form's BeforeUpdate event. Something like

Private Sub Form_BeforeUpdate(Cancel As Integer)
Dim strMessage As String

  If Len(Me!Control1 & vbNullString) = 0 Then
    strMessage = strMessage & "You must provide a value for Control1" & vbCrLf
  End If
 
  If Me!Control2 < 0 Then
    strMessage = strMessage & "You must provide a positive value for Control2" & vbCrLf
  End If

  If Len(strMessage) > 0 Then
    Msgbox strMessage
    Cancel = True
  End If

End Sub

john - project say:

Mucko72,

First of all, is your version of Project fully updated? For Project 2010 and Project 2013, that means the August 2015 Public Update. If they are not, run Windows Update.

Assuming everything is updated, since your plan is working 24hr per day, you need to set your Project calendar (Project/Project Information/Calendar) to "24 hrs". That will set the stage for all tasks in your schedule. You do NOT need to set any task calendars unless selected tasks are NOT on the 24 hr schedule, then you should of course set the appropriate task calendar.

With regard to resources, you need to create a separate resource calendar for each shift group. There are probably various ways to do this, but I suggest you create two shift calendars. These will not be resource calendars, they will be task calendars but your resource groups will be assigned those calendars.

1. Go to Project/Change Working Time and create a new dayshift calendar. You probably want to base it on the 24hr calendar and then edit from there.

2. Set the shift hours as necesary

3. Do the same for a nightshift calendar

4. Assuming the resources in each shift group of are equal talent, create a Monday dayshift group, a Monday nightshift group, a Tuesday dayshift group and so on for the full week.

5. Set the Max Units of each group to equal the number of resources in that group

6. For each resource group, set the Base Calendar to the appropriate shift calendar

7. Assign resource group to tasks as you normally would

Caveat, I haven't actually gone through this process to see if there is anything I forgot, but it should give you a starting point.

howieem say:

forcing a save as to always save as a macro enabled workbook

I have a workbook that has a lot of macro's and VBA as part of the workbook.  This workbook is saved many time during the day, and then it will be "Save As" in the evening and renamed for use the next day.

 

Every time that the end user does a "Save As", they have to be sure and save-as a macro enabled workbook.  Is there any way to put some code in the workbook (or in a custom UI, which this file also contains) so that it will always save as a macro enabled workbook.  I'm constantly getting calls from the end user that the buttons on the custom UI ribbon don't work, and it's because when they did the "Save As", they forgot to change the type of file to macro enabled.

rich007a say:

Try adding this to the ThisWorkbook code module in VBA:

Option Explicit

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
    Dim fname As Variant
    On Error GoTo ErrorHandler
    If SaveAsUI Then
        Cancel = True   'Cancel the original SaveAs
         'Get filename (with path) for saving
        fname = Application.GetSaveAsFilename(fileFilter:="Excel Marcro-Enabled Workbook (*.xlsm),*.xlsm")
        If fname = False Then Exit Sub  'Exit if user hit Cancel
        Application.EnableEvents = False  'Prevent this event from firing
        ThisWorkbook.SaveAs Filename:=fname, FileFormat:=52
          '52 = xlOpenXMLWorkbookMacroEnabled = xlsm (with macro's in 2007-2010)
        Application.EnableEvents = True  'Re-enable events
    End If
Exit Sub
ErrorHandler:
    Application.EnableEvents = True   'So events are never left disabled.
    MsgBox "An error occured during save." & Err.Number, vbCritical, "Error"
End Sub

 

If you want to be slick, you can use the InitialFilename argument of GetSaveAsFilename to suggest the name, e.g. the current file name, modified to include todays date and time (yyyy_mm_dd).

Hope that helps.

Cheers
Rich

PS. I edited the code above to include a test for if the user is doing a SaveAs rather than a Save.  I also added an error handler that sets EnableEvents back to true, just in case.

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
    Dim fname As Variant, DateTime As String, myInitialFilename As String
    On Error GoTo ErrorHandler
    If SaveAsUI Then
        Cancel = True   'Cancel the original SaveAs
       
        DateTime = " [" & Format(Now(), "yyyy_mm_dd") & "]" '= " [yyyy_mm_dd]"
        'DateTime = " [" & Format(Now(), "yyyy_mm_dd  hhmm_ss") & "]" '= " [yyyy_mm_dd  hhmm_ss]" (use instead if you want time in the name)
        myInitialFilename = "Your File Name Here" 'EDIT THIS
       
         'Get filename (with path) for saving
        fname = Application.GetSaveAsFilename(InitialFileName:=myInitialFilename & DateTime, fileFilter:="Excel Marcro-Enabled Workbook (*.xlsm),*.xlsm")
        If fname = False Then Exit Sub  'Exit if user hit Cancel
        Application.EnableEvents = False  'Prevent this event from firing

          '52 = xlOpenXMLWorkbookMacroEnabled = xlsm (with macro's in 2007-2010)
        Application.EnableEvents = True  'Re-enable events
    End If
Exit Sub
ErrorHandler:
    Application.EnableEvents = True
    MsgBox "An error occured during save." & Err.Number, vbCritical, "Error"
End Sub
