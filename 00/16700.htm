e.j. gun say:

This is one suggestion for how to create the gantt chart simulation with a chart in an Access form:

In Excel (or Graph), in order to simulate a gantt style chart, you need to create a "floating bar chart".  In a normal bar chart, each bar in a series has one end firmly anchored to zero, and the other end is at the specified value.  In a floating bar chart, both ends of the bar are at a specified start or end value.  Normally, you can't do this in Excel or Graph.  Here is a way to simulate it:

Let's say you have data in your Access table that looks like this:

ID

1

TaskName

Task 1

TaskStart

3/14/2012

TaskFinish

3/28/2012
2 Task 2 3/19/2012 4/17/2012
3 Task 3 4/23/2012 5/23/2012
4 Task 4 4/18/2012 11/22/2012
5 Joe's Coffee Break 6/20/2012 7/18/2012
6 The Big Test 5/23/2012 8/21/2012

 

You need to create a query that converts that data into something that looks like this:

TaskName

The Big Test

TaskStart

5/23/2012

Duration

3/30/1900
Joe's Coffee Break 6/20/2012 1/27/1900
Task 4 4/18/2012 8/5/1900
Task 3 4/23/2012 1/29/1900
Task 2 3/19/2012 1/28/1900
Task 1 3/14/2012 1/13/1900

 

"Duration" is a calculated field that uses the expression "Duration: CDate([tblTasks]![TaskFinish]-[tblTasks]![TaskStart])".  I use CDate to ensure that Access treats "Duration" as a date field, which will be important when creating the graph.

 

Here is the SQL for the above query:

SELECT tblTasks.TaskName, tblTasks.TaskStart, CDate([tblTasks]![TaskFinish]-[tblTasks]![TaskStart]) AS Duration
FROM tblTasks
ORDER BY tblTasks.ID DESC;

Now create a form in Access.  Add a chart object to it.  Make the chart's Row Source "qryBarGraph" or whatever you call your query.  Double-click on the chart, then right-click on it and change the chart type to a stacked bar chart.  You should see two series.  The first is labeled "TaskStart" and the second "Duration".  Remove the fill and border from the "TaskStart" series, and suddenly you have a floating bar chart (i.e. a gantt chart)!  Adjust the x-axis scale to your liking.  Note in the SQL above that the query is sorted by ID in descending order, to give the gantt chart the typical Project order - top down, not bottom up.

Whenever you open the form, this chart will update itself to the latest data in your original table by running the "qryBarGraph".  You can probably also use this technique in a report, but I haven't tried that myself.

Your data seems a little more complicated than my simple example, but you can probably make it work with a slightly more complex query that includes your desired parameters and sorts/groups them the way you want them to appear.

 

HTH,

 

Eric

hansv mvp say:

Press Alt+F9 to show field codes in the document. This should make it easier to select the fields produced by selecting mail merge rules. Fields are always enclosed in curly brackets { }, and if you start selecting at the opening bracket { of a field, you will automatically select the entire field.

When done, press Alt+F9 again to hide the field codes.

Click File > Options.

Click Advanced.

Scroll down to the Show document content section.

Tick or clear the check box "Show field codes instead of their values".

Click OK.
