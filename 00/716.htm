Hi paulrosa,


Welcome to Microsoft Community. I understand that you do not want your email messages from your Inbox folder to be move in your POP folder.


Generally. if you connect your account via POP connection using email client, it will delete the messages from the server unless you've told the email client not to.

1. In your Outlook.com Inbox page, click the gear icon .

3. Under Managing your account, click Connect devices and apps with POP.

4. Under POP, click Enable.

5. Under Deleting downloaded messages, select Don't let devices or apps delete messages from Outlook. If the app or device is set to delete, Outlook will move the messages to a special POP folder instead.

6. Click Save.

chip pearson say:

The simplest way is to bypass Excel's Save functions completely and write to the output file directly using VBA's I/O procedures.  The following code will save the range specified by the line marked with '<<< CHANGE RANGE to a file with the extension named by Extension, in the line of code marked with '<<< CHANGE EXTENSION. Change the "abc" to whatever extension you want (don't include the period in the extension). If you don't want to prompt the user for a file name and specify the file name in the code, get rid of the FName = Applicaiton.GetSaveAsFileName and replace it with FName = "C:\YourFileName.abc".

 

Sub WriteRange()
    Dim R As Range
    Dim FName As Variant
    Dim FNum As Integer
    Dim Extension As String
    Dim InRange As Range
    Set InRange = Range("A1:A5") '<<< CHANGE RANGE
    Extension = "abc" '<<< CHANGE EXTENSION
    FName = Application.GetSaveAsFilename( _
        filefilter:=Extension & " files,*." & Extension)
    If FName = False Then
        Exit Sub ' user canceled
    End If
   
    FNum = FreeFile
    Open FName For Output Access Write As #FNum
    For Each R In InRange
        If Len(R.Text) > 0 Then
            Print #FNum, R.Text
        End If
    Next R
    Close #FNum
End Sub

Setup Outlook Contacts to Merge "Mr. and Mrs. John Smith" when my contact is filed as "Jane Smith"

My questions seems confusing to me when I try to plan it out in Outlook so let me apologize up front for that.

I use my Contact List for personal friends, not business.  And in general, my contacts are filed as the female's names.  When I use mail merge in Word, I generally keep my address labels formal, like "Mr. and Mrs. John Smith."  My problem is that most of my Contacts are "Jane Smith."  So I get stuck with the Mr. and Mrs. John part of it all.

I have tried to use the Spouse Field in Outlook, but that doesn't clearly label the spouse and the "Mr." or the "Mrs."  And in some cases, my contacts are filed as the Mr. Name, and the spouse would of course be the Mrs. in that situation.

So my question is, what fields can I use in Outlook to designate as the Salutation, the Mr., and the Mrs.?  I've tried to make my own in the user-defined list, but that didn't work right for me.

Maybe this will help...  I have all the contact info for Jane, my closest friend, and I have a cell for John, Jane's husband, also a friend.  I want to use John's name, Jane's name, and Jane's address for Christmas labels.  I also might want to address the letter to John, Jane, and children, Jimmy, Janie, and Johnny. How do I set that up in Outlook one time, the first time, so that Word will give me the output I want?

doug robbins - mvp office apps & services (word) say:
You should populate the following fields in Outlook

Last Name
First Name - populate with Jane, etc
Spouse/Partner - populate with John
Children
Title - populate with Mrs., Ms. or Miss

Then, if you want to use mail merge to address Mr and Mrs use the following field constructions

{ IF { MERGEFIELD "Spouse/Partner" <> "" "{ MERGEFIELD "Spouse/Partner" \b "Mr. and Mrs. " }" "{ MERGEFIELD "Title" } { MERGEFIELD "First Name" }" } { MERGEFIELD "Last Name" }

which would result in

Mr. and Mrs. John Smith

if there is an entry in the Spouse/Partner field, but will give just

Mrs. Jane Smith

if there is no entry in the Spouse/Partner field

If you just wanted to send something the all of the Janes, use

{ MERGEFIELD "Title" } { MERGEFIELD "First Name" } { MERGEFIELD "Last Name" }

If you want to send to John, Jane, and children, Jimmy, Janie, and Johnny, use

{ IF { MERGEFIELD "Spouse/Partner" <> "" "{ MERGEFIELD "Spouse/Partner" \b "{ IF { MERGEFIELD "Children" } <> "" ", " " and " }"" }" "{ MERGEFIELD "First Name" }" "{ MERGEFIELD "First Name" } { MERGEFIELD "Children" \b " and children, " }

Which will give you

John, Jane, and children, Jimmy, Janie, and Johnny

if there is a John and children, or

Jane, and children, Jimmy, Janie, and Johnny

if there is no John

and

John and Jane

if there are no children.

I haven't dealt with the case where there is John and children, but no Jane, but it can be handled with additional If...then...Else field constructions.
