scottgem say:

Ok, that is not going to work. You need to modify some things. 

1) Add a field to your table named GradeID as a Number>Integer

2) Create a table named tluGrades with the structure:

    GradeID (PK Integer)

    GradeName

I would assign the IDs like this:

GradeID GradeName

-1           Preschool

0             Kindergarten

1             1st Grade

2             2nd Grade

3             3rd Grade

4             4th Grade

5             5th Grade

6             6th Grade

7             7th Grade

8             8th Grade

9             9th Grade

10           10th Grade

11           11th Grade

12           12th Grade

13           Young Adult

3) Change your combobox's RowSource Type to Table/Query, the Rowsource to:

SELECT GradeID, Gradename FROM tluGrades ORDER BY GradeID;

Change the ControlSource to the new GradeID field. Set Column Count to 2 and Column Widths to 0";2"

4) Create an Update query that joins your table to tluGrade on Gradename, Update GradeID to tluGrades.GradeID. So the new GradeID field contains the GradeID value for each person's grade. Verify that this worked and then you can remove the old grade field.

5) When you want to update everyone's grades you run the following Update query.

UPDATE tablename SET GradeID = GradeID +1;

I've checked with engineering and the only current known issues that resemble your problem as follows:

The only recent related bug is this 2807827, Title: Ctrl+Z has no effect.  One of the devs ran into this and opened that bug. 

1) Start composing a new reply

2) Shift+TAB until you loop all the way around back to the body.

3) Type some text

4) Ctrl+Z 

Result:

Nothing

Expected:

Undo

Another unrelated bug but interesting nonetheless 2708757:

Unable to undo (Ctrl + Z) items moved via Quick Step

Issue:

Customer verbatim:

"When I use quick steps to move an email then undo does not work which is a bit odd. You have to manually go into the folder to find the email and move it back"

Repro:

1. Select an email in the Inbox and use a Quick Step to move it to another folder.

2. Attempt to use Ctrl + Z to undo the action

 Expected results:

1. Email is moved back to the Inbox from the target folder via Ctrl + Z  

Actual results:

1. Ctrl + Z  does not move the item back

2. Note that the undo arrow on the Quick Access Toolbar is not activated 

Dev notes:

Repros since the feature was released in Office 2010.  Users have been requesting this feature since at least 2011:

https://social.technet.microsoft.com/Forums/office/en-US/4675afd9-1a2d-4d78-8c40-3a0f524522d5/outlook-2010-quick-step-action-cannot-undo-by-action-ribbon-or-ctrl-z?forum=officeitproprevious

With a bug this old, it is clearly not DogQual. Removing DogQual tag.

Unclear if this was By Design, since the Quick Steps *can* contain undoable actions, such as Permanently Delete.  It is possible we didn't want to confuse users with some QuickSteps that are undoable, and others which are not (and blow away the undo stack to boot).  I am requesting that our documentation note that QS is undoable.
