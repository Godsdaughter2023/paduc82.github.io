greginmich say:

DisableEmailScanning<Boolean>

Indicates whether Windows Defender parses the mailbox and mail files, according to their specific format, in order to analyze mail bodies and attachments. Windows Defender supports several formats, including .pst, .dbx, .mbx, .mime, and .binhex. If you specify a value of $True, Windows Defender performs email scanning. If you specify a value of $False or do not specify a value, Windows Defender does not perform email scanning.

 

https://technet.microsoft.com/en-us/library/dn433291.aspx?f=255&MSPPError=-2147217396

 

The documentation is wrong for all parameters with the “Disable” prefix. This parameter is set to true by default (Disabled True, email scanning disabled), which can be confirmed by running this command: Get-MpPreference. So you would enable email scanning by setting the parameter to false (Disabled False, email scanning enabled) by running this command at the elevated PowerShell prompt:

 

Set-MpPreference -DisableEmailScanning $False

 

Some of the PowerShell MpPreference commands seem to work fine, and others seem to be totally dead. Since I’m distracted with other issues at present, you’ll have to test this one for yourself.

Defender does not scan incoming or outgoing mail.

If you're accessing your Hotmail account via Win10's Mail app, MS Outlook, Windows Live Mail, Thunderbird or other Mail Client, Defender scans messages on your HDD (including when you open them).

greginmich say:

No mail store on the hard drive will be scanned with the default setting for DisableEmailScanning. This function has to be explicitly enabled with PowerShell or Group Policy. The Group Policy registry location for the DisableEmailScanning DWORD would be HKEY_LOCAL_Machine\SOFTWARE\Policies\Microsoft\Windows Defender\Scan. The DWORD created in this subkey would be set to the value of 0 (Disabled False, email scanning enabled) to enable email scanning.
