My Documents made in Word for Mac 97-2004 are messed up with Word for - Microsoft Community

marcia sue say:

My Documents made in Word for Mac 97-2004 are messed up with Word for Mac 14.3.6...HELP

TWO PROBLEMS....

1. Ordinary (words only, no special formatting) documents that fit on one page in Word for Mac 97-2004 are opening in Word for Mac 14.3.6 requiring 2 pages to print. Each one will require manual changes unless there is a software fix. Is there a patch to fix this?

2. Documents made in Word for Mac 97-2004 with extensive formatting are opening in Word for Mac 14.3.6 with all but the check-off-boxes missing. The document "information"/words are all missing. Why? Is there a fix...a patch?

If these issues are addressed elsewear, please direct me to the correct place. I can't find an answer anywhere. I use these documents every day in my work and it would be a MASSIVE UNDERTAKING to have to recreate them. Thanks in advance.

john mcghie say:
Hi Marcia:

There are no patches (you have the latest) but there are some things you can try.

There are two competing issues at play here:

1)  Word 2011 displays documents differently to Word 2004.  The difference is small, but it is intentional, it's the way it is supposed to work.

2)  If a document is formatted correctly in Word 2004, you can open or print it in Word 2007/2008/2010/2011/2013 and I would defy you to pick the difference (usually, I can't either...)

Unfortunately the corollary of that is that there are ways of building a document that will cause it to collapse like a house of cards.  Essentially, a Word document is not a sheet of paper.  When it starts, it is not "blank", it is "empty".  As you add content, Word stacks that content up from the top left of the document: if you will, each piece of content is "pushed down" by the content above it.

A document that uses styles and paragraph properties to provide flow control will happily open anywhere and print without change.  Sadly, as you say, it's too late to change it easily now.

There are two main problems:

1)  Some of the fonts have changed size
2)  Word has a new rendering engine that is drawing the text a different size.

The first thing we should do is make sure you are actually using the modern versions of the fonts, that are designed to avoid this problem.

Try this:
http://word.mvps.org/Mac/fontweeding.html

With luck, that will make sure that you are using the latest versions of each font and you do not have any faulty fonts in there.

That's likely to solve your second problem.  Those symptoms are characteristic of a corrupted font.

The first problem is a little tougher to solve.  There's a variety of things you could do: two of the simplest are a)  Changing to Calibri (or Cambria) as the font; or b) By tweaking the document margins or compatibility options slightly.

What I suggest is that you try the Font Weeding first.  That may bring those documents right. 

Then report back here and we can suggest the next steps.

Hope this helps

jim gordon mvp say:
John is correct that new versions of fonts ship with new versions of Office. For example, the Arial font hs undergone several revisions over the years. With each revision, the amount of space used by each character varies by tiny amounts, which can add up enough to cause the text to spill into a new page.

What does a double unary operator in a SUMPRODUCT() mean? - Microsoft Community

john 12345 say:

What does a double unary operator in a SUMPRODUCT() mean?

What does the double - - in a sum product mean. below is an example?

 

Thanks

 

=INDEX(B2:B1000,SUMPRODUCT(--ROW(1:999),--(A2:A1000=F5),--(C2:C1000=SUMPRODUCT(MAX((A2:A1000=F2)*C2:C1000)))))

joeu2004 say: John wrote:
What does the double - - in a sum product mean. below is an example?
=INDEX(B2:B1000,SUMPRODUCT(--ROW(1:999),--(A2:A1000=F5),--(C2:C1000=SUMPRODUCT(MAX((A2:A1000=F2)*C2:C1000)))))

Expressions like A2:A1000=F5 return an array of logic values, TRUE and FALSE.  Normally, SUMPRODUCT treats all logic values as zero.  The double negation converts the expression to an array of numbers, 1 for TRUE and 0 for FALSE.

But any idempotent arithmetic operation has the same effect.  So you could avoid the double negation in your formula by writing:

=INDEX(B2:B1000,SUMPRODUCT(ROW(1:999)*(A2:A1000=F5)*(C2:C1000=SUMPRODUCT(MAX((A2:A1000=F2)*C2:C1000)))))

The double negation before ROW(1:999) was unnecessary in any case because ROW(1:999) returns an array of numbers.
