dirk goldgar say: A table does not have fields "that are either combo boxes or list boxes". A combo box or list box is a way to display a field, but is not an essential characteristic of the field itself. I stress this, because thinking about it the wrong way leads to problems like this. Data entry in tables can be constrained in a couple of ways. One way is by a validation rule. For example, if a numeric value must always be 0, then in the table design you can specify a validation rule " 0" for the field. Access won't let any value that is 0 or negative be stored in that field, no matter where the value comes from. Or, for a field that has values restricted to a small, unchanging list -- for example, a field [****} that allows only "M" or "F" -- then you can specify a validation rule that itemizes the acceptable values; for example, "In ('M', 'F')". sithman00 say: I had this same problem on two of my grandparent's computers.  I have tried for hours to correct this, to no avail.  After reading your post, and realizing the inherent intelligence behind the form and writing contained within, I decided to approach this problem from a new angle: try everything that doesn't make any sense to try.  Well, I fixed it!  Problem was solved by doing the following:
Step 1: In IE 8, go to Tools > Internet Options > Advanced > Reset (lower right corner).  On the new window that opens, click Reset (there is no need to delete personal settings).  Once it is done it will instruct you to close and reopen IE.  After doing this you will be able to update to IE 9 with no problem.  I installed it manually, not sure if it will work through Windows Update or not.
Step 2: Grab your favorite beverage and bask in the joy/anguish of knowing that this makes absolutely no sense at all, but that it did work and life is better now.

Neither can I locate it in the recycle bin on both levels. It's like the page has disappeared!!! Can you please help me find where my home.aspx page is?

tina chen msft say:

Hi KK[2019],

 

Thanks for posting back. For the white space on top you want to get rid of, I understand you want to remove the following. Unfortunately, it cannot be modified via SharePoint UI.

You may refer to following and see if it will help remove the space.

Note: these steps need to be done on Windows computer, I'm sorry for the inconvenience, it seems you are using Mac. If it is possible, please use a Windows 10 computer to do the steps.

 

1.Open Windows PowerShell as administrator, run command below to install the module for

Install-Module SharePointPnPPowerShellOnline

(Enter Y when the confirmation message shows up)

Here is an article for your reference: PnP PowerShell overview

2.Install SharePoint Online Management Shell on Windows via this link.

 

3.Open SharePoint Online Management Shell and run commands below:

 

1) Run command below to connect to your SharePoint site, sign in SharePoint administrator or Office 365 administrator account and password in the prompt window:

Connect-PnPOnline https://yourtenant.sharepoint.com/sites/yoursitename

 

2) List all pages, and find the ID of your site page which you want to set as home page.

Get-PnPListItem -List SitePages

 

3) Change the layout to "Home" to remove the top white space. (please change the id with your site page id)

Set-PnPListItem -List SitePages -Identity <id> -Values @{"PageLayoutType"="Home"} 

 

For your reference:

4)Sign out Office 365 and sign in again, go to your SharePoint site, see the result in the site home page.

 

Please feel free to share with us your test result.

 

Best Regards,

Tina
