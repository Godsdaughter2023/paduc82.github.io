If you're moving your domain on your own and configuring the email accounts over again once you've pointed your domain to the new servers - the approach is going to be to save your IMAP data as described (export to a PST file) before you make that transition - otherwise you'll lost everything. For accuracy, this isn't a case of Outlook wiping anything out since Imap email data is "server based" and the OST file reflects what Outlook finds (syncs with) when it connects - the exception being the data stored in folders marked as "This Computer Only".

Actually, I found a way around using the GUI for sorting.  The problem with the GUI is that I generate a whole slew of tables from nonexistence using a SQL script, so I have to re-set the multi-field sorting every time a table is regenerated.  However, I had to use the GUI because a SORT clause cannot refer to an alias, such as wwwwwwww below:


         SELECT HHHH, ooooooo,
            SWITCH(
               ssssssss=eeeeeeeee, 'child',
               ssssssss=PPPPPPPPP, 'parent'
            ) AS wwwwwwww,
            eeeeeeeee, ssssssss, PPPPPPPPP
         FROM sourceTable


The way around this seems to be to treat the query as a subquery.  The outer query can then use the alias from the inner query.


      SELECT * FROM (
         SELECT HHHH, ooooooo,
            SWITCH(
               ssssssss=eeeeeeeee, 'child',
               ssssssss=PPPPPPPPP, 'parent'
            ) AS wwwwwwww,
            eeeeeeeee, ssssssss, PPPPPPPPP
         FROM sourceTable
      )
      ORDER BY
         wwwwwwww DESC, eeeeeeeee, HHHH, ooooooo, ssssssss, PPPPPPPPP

guhanv say:

@Webcolin70: I found this same problem, and recently had to deal with comparing several documents. 

I agree with your description of the behaviour in earlier MS Word versions.  Here is the link with excerpt from the Microsoft Word Version 2002 Step by Step, chapter 8 ("Collaborating with Others") that describes how it worked then.  http://support.microsoft.com/kb/306484

I ended up writing a Macro, that more or less does what used to be the functionality in 2003 and before.  Here is the code. 

' Compare current open document to user selected document.
Sub CompareMe()
  Dim fOpen As FileDialog
  Dim compareDoc As String
 
  On Error GoTo CompareMe_Err
 
  ' Display File Selection Dialog
  Set fOpen = Application.FileDialog(FileDialogType:=msoFileDialogOpen)
  fOpen.AllowMultiSelect = False  ' Don't allow multi-select.
 
  If fOpen.Show = -1 Then  ' Selected a File
    compareDoc = fOpen.SelectedItems.Item(1)  ' Get the name of the Doc
    ActiveDocument.Compare Name:=compareDoc   ' Compare to current Doc
  End If ' If fOpen.Show ...

CompareMe_Err:
  ' Continue to end
End Sub

You can add it to your document or to normal.dot.  I also added it as a Custom Ribbon item, so I can get there in one click.  Or use shortcut keys.

Hope it's what you are looking for.
