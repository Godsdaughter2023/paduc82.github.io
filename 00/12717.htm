But, I want this to run only on the current workbook.

     - it will not run while the workbook is still open but another workbook or some other programme is active and not this workbook.

joeu2004 say:

Harrywindows wrote:

I have used ontime to run a procedure every minute on a workbook.

    Application.OnTime Now() + TimeSerial(0, 1, 0), "procedure"

[....]

What is the easiest way to do it so that

     - it will cancel the procedure already scheduled if I decide to close the workbook

     - it will not run while the workbook is still open but another workbook or some other programme is active and not this workbook.

Some things in life are never easy.

Try the following.

In a normal module:

Public evtime As Double    ' personal preference; some people prefer type Date

Public Sub startEvent()

    ' don't schedule if already scheduled (evtime <> 0)

    If evtime = 0 Then

        evtime = Now() + TimeSerial(0, 1, 0)

        Application.OnTime evtime, "myproc"

    End If

End Sub

Public Sub stopEvent()

    If evtime <> 0 Then

        ' race condition: evtime might arrive between statements

        On Error Resume Next

        Application.OnTime evtime, "myproc", , False

        evtime = 0

    End If

End Sub

Public Sub myproc()

    ' race condition: stopEvent might be called just when evtime arrives

    If evtime = 0 Then Exit Sub

    evtime = 0

    startEvent

    ' .... the rest of myproc ....

End Sub

In the ThisWorkbook module:

Private Sub Workbook_Activate()
    startEvent
End Sub

Private Sub Workbook_BeforeClose(Cancel As Boolean)
    stopEvent
End Sub

Private Sub Workbook_Deactivate()
    stopEvent
End Sub

Caveat:  evtime is set to zero when VBA is reset.  We might reset VBA manually.  But VBA also resets itself sometimes, e.g. after some edits.  AFAIK, there is no way for us to trap VBA resets in order to stop or restart any scheduled OnTime event.  Consequently, it is still possible for an OnTime event to occur after we close or deactivate a workbook.  Sigh.

To avoid that, arguably it might be more reliable to write evtime into a cell (perhaps in a hidden worksheet) instead of using a global Public variable.  But that's not "the easiest way".  :-)

davidf2 say:

Try a different web safe font than Garamond. It is not the best choice for HTML format or webs.

It may be that Publisher is converting the text to a picture which can result in blurry text. If you cannot left click, drag to select the text after the merge is created then it is a picture. You can also try right click and see if you are offered options that relate to pictures to see if this is what is happening.

Go to File > Options > Advanced > General > Web Options and make sure you have not checked the option to send as a single JPG.

Not all print formatting can be converted to HTML and when that happens Publisher will opt to convert it to a picture. You might need to deconstruct that part of your publication that is converted to an image to see what part of your formatting or layout is causing Publisher to choke and convert the text to a picture. Keep it simple.

You could also skip all of this and convert the publication to a PDF file and attach it to the email message. That is the only way to always maintain your layout and design.
