That's correct.  Assuming a reference to the shape in oSh:

In 2003, this would give you -2

Debug.Print oSh.TextFrame.TextRange.Font.Size

In 2007 and up, it gives you the size of the first character in the text range.

 

In either, you can do something like this:

 

With oSh.TextFrame.TextRange

If .Runs.Count = 1 Then

  Debug.Print "All of the text is the same size"

Else

  For x = 1 to .Runs.Count

    Debug.Print .Runs(x).Font.Size

  Next

End If

End With

daniel pineault say:

It is because API calls change depending on the office bitness.  As such, you need to develop your code for your target bitness or use conditional compilation techniques so, at runtime, the program adjusts the call according to the available bitness.

You might want to read:

  • https://msdn.microsoft.com/en-us/library/office/ee691831(v=office.14).aspx
  • http://www.utteraccess.com/wiki/index.php/Conditional_Compilation

So

Private Declare Function apiShowWindow Lib "user32" Alias _

"ShowWindow" (ByVal hwnd As Long, ByVal nCmdShow As Long) As Long

for a 64 bit office becomes

Private Declare PtrSafe Function apiShowWindow Lib "user32" Alias _

"ShowWindow" (ByVal hwnd As Long, ByVal nCmdShow As Long) As Long

Private Declare Function apiGetUserName Lib "advapi32.dll" Alias _
"GetUserNameA" (ByVal lpBuffer As String, nSize As Long) As Long

for a 64 bit office becomes

Private Declare PtrSafe Function GetUserName Lib "advapi32.dll" Alias _

"GetUserNameA" (ByVal lpBuffer As String, nSize As LongPtr) As Long

Then add to that condition compilation, and you'd get something like

#If VBA7 And Win64 Then '64 bit

Private Declare PtrSafe Function apiShowWindow Lib "user32" Alias _

"ShowWindow" (ByVal hwnd As Long, ByVal nCmdShow As Long) As Long

Private Declare PtrSafe Function GetUserName Lib "advapi32.dll" Alias _

"GetUserNameA" (ByVal lpBuffer As String, nSize As LongPtr) As Long

#Else '32 bit

Private Declare Function apiShowWindow Lib "user32" Alias _

"ShowWindow" (ByVal hwnd As Long, ByVal nCmdShow As Long) As Long

Private Declare Function apiGetUserName Lib "advapi32.dll" Alias _
"GetUserNameA" (ByVal lpBuffer As String, nSize As Long) As Long

#End If

The document may be damaged. Have you tried copying the content, minus the final paragraph mark (¶), into a new blank document? To show/hide paragraph marks as well as other nonprinting marks, you can use the ¶ button on the Home tab.

jay freedman say:

Another possibility is that the Track Changes options are set wrong.

If I have the Deletions markup set to "(none)" and display set to All Markup, then backspacing appears to do nothing. However, the characters really are deleted, as seen when the display is switched to Simple Markup.

To verify and possibly fix this, click the dialog launcher arrow in the bottom of the Tracking group on the Review ribbon. Then click the Advanced Options button. If the Deletions dropdown is set to "(none)", change it to one of the other values, such as Strikethrough.
