mikethecat say:

The program that is creating the high-CPU problem is already known.  It is the NGEN program that is identified as Microsoft Common Language Runtime native compiler.

And it is NOT helpful that the program will release CPU to other processes when the computer is not idle, for at least 3 reasons.  

1) when the computer is a laptop that is being used on battery, we do not want optional processes to max the CPU just because it detects no user activity, because this runs the battery down quickly 

2) it does not make sense to subject the computer to the thermal stress of 100% CPU utilization at all times

3) it makes no sense to have a process run hundreds of billions of machine level instructions that apparently accomplish nothing at all even after DAYS of running at 100%

If a clean boot stops NGEN from doing this, that is not helpful unless it leads to the answer to this question:

What is causing ngen to run when it never accomplishes anything?
