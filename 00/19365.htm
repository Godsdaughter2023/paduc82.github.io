My mistake, it turns out diagnosing hangs have a different method than diagnosing crashes.  I appreciate all the effort you've gone through!

The best way to diagnose a hang is as follows:

1. Boot Word

2. Run your scenario to get Word to hang

3. Go to the Task Manager (Ctrl-Shift-Esc)

4. In the Details tab, find WINWORD.EXE, and right click on it to bring up a context menu.

5. Click "Create dump file".

You should instantly see a "Please wait..." dialog, followed some time later by a dialog telling you where the WINWORD.DMP file is saved.  It's probably somewhere like C:\Users\<username>\AppData\Local\Temp.

This dump file is probably huge, on the order of 100MB - 1GB in size.  I'd compress it (use your favorite zip utility, or Windows 10 can do it by right clicking the file and choosing Send To > Compressed folder) then place it on OneDrive, and we can take a look.
