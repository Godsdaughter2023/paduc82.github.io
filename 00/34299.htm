HOW SYMBOLIC LINKS, OR “SYMLINKS” WORK
Junction works by using a symbolic link or “symlink” - this is basically a placeholder with a redirection to another location. When you create a symbolic link to a file or folder, the file / folder will still appear as it normally does, but the system will automatically point to another location you previously specified. For example: you could create a symlink for C:\Documents relocate it to D:\Documents. Clicking on C:\Documents would actually take you to D:\Documents, and any program that referenced the old location would automatically and seamlessly link to the new location.
If you intend to move the entire C:\Program Files folder (for example), you should reboot into Windows Safe Mode first, as using Junction on an open program will either result in an ‘Access Denied’ error, or possible data corruption. Put another way, you cannot move around files and folders that are currently open; by booting into Safe Mode, you will effectively run Windows in ‘bare bones’ mode, which does not automatically launch programs that typically start when Windows boots into the desktop.
HOW TO MOVE INSTALLED PROGRAMS TO ANOTHER DRIVE IN WINDOWS 7, 8, 10
Using Junction means that you will need to run it from an administrative command prompt. To do so, click Start and then type in “cmd” (no quotes), then wait for Command Prompt or CMD.EXE to appear in the list, then right click it and choose Run as Administrator.
As a working example: in the past I had to move the ‘..\appdata\Apple Computer’ folder for a user because it was taking up too much space on the C drive. I made an ‘AppData’ folder on the D drive using the command below; note that the ‘-s’ switch means to include all directories inside of the parent folder (otherwise known as directory recursion). The command was:
junction.exe -s “C:\Users\%USERNAME%\AppData\Roaming\Apple Computer” “D:\AppData\Roaming\Apple Computer”
Note that I used quotes around the directory names - that is required if you have any spaces in folders or files. It’s a safe bet to use quotes around the source and destination folders, regardless. I suggest using the “-s” (recurse directories) for all your symbolic links, otherwise your programs may not work properly. If you wanted to remove the junction, then the command would be:
junction.exe -d “D:\AppData\Roaming\Apple Computer”
JUNCTION VERSUS MKLINK
Please note that Junction is not supported over network shares. For that you can use the ‘mklink’ command, but only if it is available / supported in your version of Windows. just creech say:

Hi Repathic,

I was a PC master race gamer for many years, so I can definitely appreciate the mindset you had coming into the console world. 

There is a bit of a difference between the console and the service for Xbox (and for all gaming console ecosystems; this is not exclusive to Microsoft). Purchasing the hardware, in your case the Xbox One S, provides you all the necessary equipment needed to play games made for the console. Purchasing a subscription to the service, in our case Xbox Live, provides you access to a gated online experience that includes multi-player matchmaking.

Many games are single-player, local only games, such as The Wolf Among Us...these do not require any type of connection to the internet other than perhaps downloading a digital copy.

I think the vast majority of games these days have a combination of local play and online play, such as Forza Horizon 4, Minecraft, and Halo. With these titles you can play locally with no subscription to Xbox Live, or with a valid subscription you can access the multiplayer aspects.

Then there are some titles that are online only. Shadowrun did this back on the Xbox 360 years ago, and more recently Titanfall and a few others. The choice to not include an offline local game experience is made by the individual developers and not mandated by the service owner (such as Microsoft in the case of Xbox Live, Sony for Playstation's service, etc.)  

There is a subset of online only titles, including Fortnite, that are online only AND are free to play. The "free to play" part means you don't have to buy a copy of the software to play it. The developer makes money via in-game advertising or microtransactions such as convincing you to pay real money for better gear or cool attire.

You still need to buy the platform and the means to connect to other players. From a PC perspective, you often need to have a solid gaming rig (and if you can do that for under $1000, I am impressed with your building skills!) plus you have to pay for an internet connection to be able to join a service that will connect you with other players. Maybe that is Steam, maybe the developer hosts servers on their own, or maybe it is all done peer-to-peer. With games played on an Xbox, that service is Xbox Live. The game is still free to play, and they are still going to try to monetize your experience so that they make money. However, if you are playing it on the Xbox, you need an Xbox AND you need the connection to Xbox Live so that you can connect to others. 

I hope this explains things well for you, and that you join Xbox Live for at least long enough to try it out. I think you will agree that it is a great way to connect with other gamers, both existing friends and new ones. I joined Live more than 13 years ago and love the community, far more than the experience I had while I was exclusively a PC gamer.
