doug robbins - mvp office apps & services (word) say:

Hi Sharon,

Formfield shading has always remained in place when text is entered into the formfield when the document is protected for Filling in forms.  There is nothing bizarre about what you are encountering.

The reason that it, as you say: behaves normally, when you turn off editing restrictions is because when you then enter text into a Formfield, you are destroying that field.

If you toggle on the display of field codes, in the location where there was a formfield "into which" text has been entered, you will no longer see { FORMTEXT }.  The text that was entered has replaced the formfield.

The only way to have the shading removed would be to run a macro containing the following code on exit from the last formfield

With ActiveDocument
    .Unprotect
    .FormFields.Shaded = False
    .Protect wdAllowOnlyFormFields, NoReset:=True
End With

Alternatives to using Formfield shading would be to have the display of bookmarks activated in which case, as long as the formfields have bookmarks assigned to them, would result in them appearing as

[        ]

or, turn on the display of non-printing characters by using the pilcrow (¶) button in which case you will see

°°°°°

(non-breaking spaces) at the location of the formfield.
