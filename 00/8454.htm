Custom Fill Pattern made with line patterns - Microsoft Community

alkenn say:

Custom Fill Pattern made with line patterns

I have started to use Visio 2013, used to use 2003.  I need to create simple line drawings for work, which must be black and white (no colour allowed).  I created several custom fill patterns using diagonal lines, for use in showing cross-sections.  I need custom patterns because I need more different patterns than the standard ones and also the standard ones have the wrong scaling behaviour.  I created my patterns by drawing lines with the line tool.  They work fine in Visio 2003, but not in Visio 2013.

At first they didn't show up at all in Visio 2013 - the shape would appear to have no fill.  I found out that the reason is that fill patterns for Visio 2013 have a foreground colour and a background colour.  For some obscure reason, if you have a custom fill pattern made with drawn lines, Visio 2013 treats the fill pattern lines as being fill pattern background.  By default, the fill pattern background colour in "No Themes" is white, and so my lines were invisible because they were white on white.  Obviously, I could change the fill pattern background colour to black, but then other fill patterns would not come out right, so what I really want to do is get my fill patterns lines to be fill foreground not fill background.

So my question is this:  How can I get the line pattern of a custom fill pattern to be fill pattern foreground, not fill pattern background?  I have searched and tried everything I can think of, but nothing works.  Can anyone help please?

alkenn say:

I have now found a partial answer to this, when I was searching for something else.

According to this web article https://grahamwideman.wikispaces.com/Visio+fill,+line+and+end+patterns there is special behaviour depending on the line colour you set for your custom fill pattern.  If you set it at Black, it will render as the shape's background fill colour, and if you set it at White it will render as the shape's foreground fill colour.  This seems back-to-front to me.  Surely most people make drawings with black lines on a white background, not white lines on a black background.

The trouble is that in fact this only seems to be half true.  Black lines in the fill pattern certainly render as the filled shape's background fill colour, but white lines simply render as white, and not the shape's foreground fill colour.  However, it is true that non-black line colours in the fill pattern render as the specified colour, and this gives a workaround.  As suggested in the web article, I tried setting the line colour to nearly black.  To do this I went to line colours, "More Colours", and then the "Custom" tab, where I set the colour to R=1, G=1, B=1 (as opposed to true black which is all 0).  This renders as the set colour, which is indistinguishable from black.

This will probably do me for now, but I still haven't found a way to set a line in a custom fill pattern to be the filled shape's foreground fill colour.

patrick rossiter say:

If you are operating within Excel with the data generated by the automation tool, you can select the data, choose and XY scatter and the chart will plot in time scale.  My guess is that the inclusion of the days will make time of seconds so insignificant that you are unlikely to be able to plot both together. Well lets be honest, would the seconds be significant on a scale with months weeks and days? [If tripping to Mars this summer, maybe:¬) ]

If you need to extract the time data from the date data use a MOD(A1,1) function, where the date data is in A1.  This will releave you of the burden of the days.  You are now dealing with time values only. Hours, mins and seconds.

I am guessing that you are analysing server data and wish to establish times of high frequency usage without the date data getting in the way.  if you are then this should do it.  If you are not then perhaps you could add more to the description of the problem that will elicit further help.
