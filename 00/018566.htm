bob jones aka: cybertaz say:
Everything you describe wanting to do can be accomplished if you use the option for "Full page of the same label" - just don't enter an any information into the address field. When you go to Tools> Labels you need only select the label model, confirm the option, then click OK. A new blank label document will be generated - nothing more than a Table based on the dimensions specified for that model of label. Enter the content into any cells you wish and/or format as you please.

The only thing that's 'new' is that the single label option doesn't generate a document. All the "Single label" option enabled, though, was taking advantage of a used sheet to print a single label in a specific cell without having to edit the label document, itself - and that it still does.

Previewing is now done in the Print dialog in compliance with OS X guidelines. If you want a larger preview than the Quick Preview provides click the Print dialog's Preview button to see the document in Apple's Preview app. I'm not saying that it's a
As for the printing issue, the document dimensions in Word are not what need to be confirmed. The Paper Size of the label document is defined by the specs of the model of label. What you need to check is in System Preferences> Print & Fax. Select your printer there (if you have more than one driver installed) to see that the Default paper size is set to US Letter.

Also, if you do edit a label document in any way you have to be very careful about it. Label layouts are quite precise. If anything you do causes the slightest change in the actual layout it will result in the behavior you described in your first post - A fraction of the last row cannot be on one page with the remainder on the next due to the settings for the table, so the entire row gets moved to another sheet. It may not be apparent in Word when that happens because the pagination of the document is determined when it's printed. IOW, it may look OK on screen but when the print driver imposes pagination as the document is printed you wind up with 'overflow' onto another sheet of paper. That's one reason why previewing is important :-)

Bob J.

Hi Kolitha,

Good to know that you can recreate the root site collection by following the steps below. We'd also like to share the steps to the community members who have the same requirement. But because the root site collection is a special site collection of SharePoint online, we need to avoid to remove and re-create it frequently. 

To recreate the root site collection (e.g. https://contoso.sharepoint.com), please follow the steps below:

1. Go to SharePoint admin center in the browser and select the root site collection.

2. Click Delete, and please understand that nobody can access SharePoint until you create a new site collection at https://contoso.sharepoint.com.

3. Do not remove it from the recycle bin. Directly click New > Private Site Collection to recreate it.

4. It will prompt you to Permanently delete the site collection from the recycle bin and continue. Tick it and create the site collection. 

Alternatively, you can do it in PowerShell with the following steps:

1. Set up the SharePoint Online Management Shell Windows PowerShell environment

2. Open SharePoint Online Management Shell and run the following commands:

Connect-SPOService -Url https://contoso-admin.sharepoint.com #then enter your credentials

Remove-SPOSite -Identity https://contoso.sharepoint.com

Remove-SPODeletedSite -Identity https://contoso.sharepoint.com

3. Wait for 10 minutes and go to the SharePoint admin center to create the root site collection in the UX.

If this answers your question/helps you, please let us know by marking "Yes" the post which was helpful, or answered the question.

Thanks,
Lingyu Sun
