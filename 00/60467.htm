Disabling Windows 10 updates by renaming + moving system files:

Windows 10 basically ignores metered settings, services.msc, ignores task scheduler, ignores the registry keys. As of now, all those ways do not work. The way I'm using that in Windows 7. Warning: this method may damage your system. Here are the 5 files you need to move, located under /Windows/System32:

  • wuapi.dll
  • wuapihost.exe
  • wuauclt.exe
  • wuaueng.dll
  • wuautoappupdate.dll

Also, rename them and add a new extension just for good measure. Ideally, you also want to delete the /Windows/SoftwareDistribution, as well as any UpdateAssistant folders within the Windows folder. As the other answers said, it is also good to disable the windows update service afterwards (although doing so alone will not prevent windows from updating). After doing so, it shouldn't update again. If it does (probably due to residual files), start it in safe mode, then delete folders within Software Distribution, as well as Update assistant folders.

I've tried this method on two computers. Update works again when I move the files back and change them back to their original names.

Changing permissions:

The permissions are tricky. To get possession of the files, go to Properties > Security. Click advanced. Change change at the top of the Advanced Security Settings dialog to change ownership. Type in your username into the blank box that appears and hit enter. Now click OK until you exit. Now go back into the Advanced Security Settings dialog. Now the options to edit won't be grayed out. Click on administrators, and click edit. Make sure it is in allow mode, which it is in by default, and check the box that says full control. Now click OK all the way until you exit. You can now move and rename this file.
