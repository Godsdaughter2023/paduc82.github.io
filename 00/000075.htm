steve rindsberg say:

The other answers have some good suggestions and follow-up questions.  One thing it's important to understand, though, is that Windows and Mac programs, with a few renegade exceptions, do not handle their own fonts.  They depend on the fonts installed in Windows or MacOS. 

If the font's available on a computer, it'll be available to all programs on the computer.  If it's not, it won't be available to any programs.

So unless Gigi is installed on the PC with 2010, it won't be available to PPT or any other programs.

 

BUT

 

Some fonts are embeddable to one degree or another, and Gigi is "edit embeddable".  If you embed it in your presentation when you save from PPT 2007, it will be temporarily available on any Windows PC when you open it in 2007 or 2010.

To embed the font, choose Save As, then in the save dialog box, click Tools and choose Save Options.

Underneath "Preserve fidelity ..." choose "Embed fonts in the file".

If you only want people to be able to view your presentation and editing's not important, choose "Embed only the characters ..."   This will make the file smaller.

If you want them to be able to edit the text, choose "Embed all characters ..."

a. user say:

hi george,

i agree with vasil and sukhdeep. passwords are synchronized more frequently than the standard directory sync window for other attributes. the password sync feature checks every two minutes whether passwords need to be synchronized. if you still want to manually sync the attributes to office 365, please refer to the following steps:
for syncing standard directory attributes except password:
go to aad connect server > c:\program files\microsoft azure ad sync\bin > to perform a manual update, use the directorysyncclientcmd.exe tool.  the delta and initial parameters specify the relevant task.


for password sync:
to use the following scripts, you need to replace the names of the connectors with the values from your environment.

$adconnector  = "o15sso110.tk"

$aadconnector = "o365bpt.onmicrosoft.com - aad"

import-module adsync

$c = get-adsyncconnector -name $adconnector

$p = new-object microsoft.identitymanagement.powershell.objectmodel.configurationparameter "microsoft.synchronize.forcefullpasswordsync",  
string, connectorglobal, $null, $null, $null

$p.value = 1

$c.globalparameters.remove($p.name)

$c.globalparameters.add($p)

$c = add-adsyncconnector -connector $c

set-adsyncaadpasswordsyncconfiguration -sourceconnector $adconnector -targetconnector $aadconnector -enable $false

set-adsyncaadpasswordsyncconfiguration -sourceconnector $adconnector -targetconnector $aadconnector -enable $true

note: this requires powershell version 4 to run, if you are using 2008 r2 download the latest management framework.

What is keeping the led light on is the Fast Startup Option in the Power Options of the Control Panel.
