I found a way to uninstall this update WITHOUT having to reinstall Windows.

  1. Press the WIndows and then the R keys to open up the Run box.
  2. Type in Control Panel.
  3. Click the OK button.
  4. Click on "Uninstall a program" under Programs.
  5. Click on "View installed updates" on the left hand side.
  6. Click on the "Installed On" column heading to sort by that if not already selected.
  7. Scroll down to the Microsoft Windows section.
  8. Security Update for Microsoft Windows (KB4056892) should be close to the top of that section.
  9. Right-click on that update and click on Uninstall.
  10. Reboot after uninstallation.

jsmitty99 say:

The Show or Hide Updates Troubleshooter does work to hide and not install certain updates but I found a better solution to the Start Menu problem.

Although what I did above fixed my problem temporarily, the problem came back even though that particular update was not installed.  I did this instead:

  1. Run MSCONFIG by File | Run new task in Task Manager and check "Create this task with administrative privileges."
  2. Go to the Services tab.
  3. Check the Hide all Microsoft services box
  4. Click on the Disable All.
  5. Then click on all the services you definitely know you need.  Or keep them all disabled and add them back one at a time as needed.
  6. Click on the Apply button.
  7. Reboot.

My start menu started working again.  I then installed the security updates that I uninstalled previously and everything still worked.  Apparently one of the updates conflicted with one or more of the services I had running.
