Nothing in the Event Viewer log should defy explanation.

Is your system some how misbehaving or did you just notice this in the Event Log and wonder why you see it?

What the message is trying to tell you is your system is being told to look for and load at startup what are probably third party DLLs from something you have installed so it is just an informative message/warning. 

It could be that something you have installed/uninstalled in the past may have used this Windows feature that you can read about here if interested:

https://msdn.microsoft.com/en-us/library/windows/desktop/dd744762%28v=vs.85%29.aspx

You can check to see if there are any DLLs specified to load and if there are none, turn that thing off and you will not see the message anymore.

You need to look in the registry to see what is going on and since there is no undo or quit without saving in the registry editor you should first make a manual System Restore Point and consider backing up your registry with the popular free tool:

http://www.larshederer.homepage.t-online.de/erunt/

The click the Start button/orb and in the box enter:

regedit

And choose to run the registry editor as Administrator:

Then navigate to here:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Windows

The value AppInit_DLLs is where any of these DLLs are specified and if yours is empty (meaning there are no DLLs to load) just tell Windows not to try to load something that is not there.

Check the value of LoadAppInit_DLLs and if it is set to 1, double click it and change the value to 0 (zero), close the registry editor, reboot and check your Event Viewer again.

If there IS something in the AppInit_DLLs field (a comma or space separated list of DLLs) then you should leave things alone since they are there for some reason and accept the fact that you will see that informative message every time your system boots.
