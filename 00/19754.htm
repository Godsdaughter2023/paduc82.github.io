scottgem say:
First, you do not have fields on a form, you have controls that may or may not be bound to a field in a table. This is a subtle, but important distinction. 

I would use the On Current event as suggested and code like:

Me.Label112.Visible = NOT IsNull(Me.Field1)
Me.Field1.Visible = NOT IsNull(Me.Field1)

So each time you change a record, if the Field1 is null the controls will remain hidden. If the field has data it will be visible. 

Also, I would not accept the generic object names Access supplies. Field1 and Label112 tell you nothing. You should rename your controls as soon as you create them with meaningful names.

To me it sounds like you want all rows to be checked, when you're saving the workbook, and not when a change occur in the worksheet?

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)

      Dim WS As Excel.Worksheet
      Set WS = ActiveSheet

      Dim Row2Check As Range
      For Each Row2Check In WS.UsedRange.Rows
            If Range("E" & Row2Check.Row).Value = "" Then Range("A" & Row2Check.Row).ClearContents
      Next

End Sub

The macro assumes, that the worksheet is the active sheet, but you can always change that in code line 2.

Disable Word Editor - Microsoft Community

lenzeek say:

Disable Word Editor

Can we disable the new Word Editor?  My team has used it, and does not like it.
jay freedman say:

You can disable the wavy and double underlines that mark suspected spelling and grammar errors by going to File > Options > Proofing and clearing all the check boxes under "When correcting spelling and grammar in Word". When you press F7 or click Review > Spelling & Grammar, though, you'll still get the Editor.

However, the old Spelling and Grammar dialog is still present in the software. To make it available, install the following macro in a global template (one that's stored in the %appdata%\Microsoft\Word\Startup folder) and assign the F7 key to run the macro instead of the ToolsProofing command:

Sub OldSpellingChecker()
    On Error Resume Next

    Dialogs(wdDialogToolsSpellingAndGrammar).Execute
End Sub

If you ever want to run the Editor, it's still available on the Review > Spelling & Grammar button.
