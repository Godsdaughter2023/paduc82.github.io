See the following page of fellow MVP Shauna Kelly's website:

Note, it is not too hard to get the order of the number from the previous heading level and the number from the present level reversed when setting up the numbering.  If that is the case after you get the A.2.1 that should be A.1.2, the next time you apply that level of number it will be A.3.1, then A.4.1, etc.

pamcaswell say:

If you want to fix the list yourself, try this. After entering the define new multilevel list dialog, temporarily change the start number for each level to  its level.  So, start level 1 at A, level 2 at 2, level 3 at 3 , and so on.  The preview and the number format input box should show, for example, A.2.3.4 for level 4.  If the number fields are misplaced, they won't appear in the  right order.   Once you have corrected the order,  set the restarts back to one.

 

Note that the faster, less error-prone way to  create a new list is to first select the built-in Word list icon that is closest to what you want.  All the fields and restarts are already set.  For instance, if you are setting up numbered headings, you might choose 1 Heading, 1.1 Heading 2, 1.1.1 Heading 3. In your case, all you'd have to do is define a new multilevel list and change the  number style for level 1 to A, B, C.

HTH,

Pam

pamcaswell say:

Sorry, I left out the word "level".

The option does insert the numbers, but the cursor must be in a level where  it's possible.

  • Click level 3 in the Define new multilevel list dialog. 
  • In the Enter formatting... box, clear the contents. 
  • In the Number style for this level box, choose "1,2,3...".  ---A 3 will appear in the Enter formatting... box (if you've  chage the start numbers as I suggested).
  • Move the cursor in front of the 3.
  • Click the dropdown arrow to the right of the Include level number from box and choose level 1 and press . (period).  ---You should see  A.3 in the Enter formatting  box.
  • without moving the cursor, click the dropdown arrow again and choose level 2 and press  . (period). --- You should see A.2.3. 

Change the start at  values for all levels to 1.
