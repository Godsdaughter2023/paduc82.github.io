karl timmermans (outlook mvp 2012-2018) say:
>>
I would drag and drop all of the files back to the POP address and then simply delete the 'saved' folder in the IMAP account. I have no need for all of the information to be on the server, only on my computer. Will that work?
<<

Again just for the sake of accuracy - you wouldn't be copying things back to the POP address but rather to the PST data file that used to belong to a POP3 account. It's a nit for the sake of this question/issue but always good to keep in mind that what appears in the Outlook Nav Pane are the "data files" associated with the email accounts configured along with any stand-alone PST files that may have been added or previously associated with a POP3 account.

Once you have moved everything back to the PST file and it's all removed from the IMAP folder tree(s) - your sync issues will be gone

As for the problematic IMAP email account sending everything to Trash - going to suggest that once you've have everything out of the account that you don't any there anymore, delete the account and re-add it and hopefully that should set everything back to normal. When the account gets re-added, a new OST file for the account will get created and the email folders will be re-sync'd which should only involve the <Inbox> given the description to date.

NOTE: Just to cover all the bases, the Imap account that you're about to delete as per the above, should not have any "This Computer Only" folders in the folder tree. If "This Computer Only" do exist AND there is data that you want to retain, be sure to backup (export) those folders to a PST file. However, would further suggest that if this is in fact the case, that you go to File --> Account Settings --> Data files tab --> select the PST file and set it as the default data file for the profile. That's where your default contact/calendar folders will be located.

How to remove anchor from picture in Word 2016

I am track change editing a document of a colleague who is using Word 2010. This is the first issue I have come accross in this project.

Any help is appreciated.

Thank you!

macropod say:
Unless you change the picture's wrap format to in-line with text - which means the text will no longer wrap around it - you cannot delete the anchor. The anchor shows what the picture is anchored to. Delete it and you delete the picture.

rohn007 say:

You can't actually "remove" a graphic anchor. It is there all the time. As Paul explained what you can do is change the "text wrap" attribute

As Paul suggested, I too personally prefer "In Line with Text". That way I know where the image is going to be.  Using text wrapping is "nice", it makes for pretty layouts (WHEN IT WORKS) but too unpredictable for my taste.  It is too easy to accidentally drag an image and move it someplace you don't want it to be, then you have to go through the hassle of getting back in the "right" place.

macropod say:

if the whole document is selected , anchored images aren't selected.

That is not correct - the anchored images are selected, via their anchors. For example, selecting the whole document and pressing the Delete key deletes the anchored images also.

As for the problem of anchored images moving when page breaks are deleted, you will need to either reformat the images as in-line or reposition them (from the start of the document) after you've deleted the page breaks.
