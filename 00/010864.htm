cbarnhorst say:

Don't believe speculation about Microsoft deactivating old licenses.  As with upgrade licenses in the past, an old license is part of the new upgrade license.  The principle involved is called "retained value" by Microsoft.

In the past a user was always permitted to restore an old license on the same computer as long as the new upgrade license was no longer in use.  The same principal applies to Windows 10 upgrades.  

However,

Windows 10 licencing works a bit differently from previous versions in that the computer, not  the software, is what is registered with Microsoft when you first install (or upgrade to) Windows 10 using the Get Windows 10 free upgrade program.  

Since it is the computer that is registered as a Windows 10 device, Windows 10 (same edition) can be reinstalled at any time anywhere on the device without need of additional licensing.  The computer, once registered, is digitally entitled to run the software.

There is no deadline when reinstallation is done because digital entitlement is permanent (i.e., for the life of the computer).  Reinstallation is not required to be by upgrade.

The document string expression is [Symbol] & [Status] & [ID] & [Category1] & [Category2]

If the symbol field is manually entered instead of using a lookup, the string displays the value: ABCA011212. However, to make it easier on the user, I prefer to use a lookup field instead of relying on the user to manually look up the symbol and enter it.

What do I need to do to force the lookup to display the field value of Symbol instead of the ID?

john w. vinson say:

That's just one of many problems with Microsoft's misleading, limited Lookup Field. See http://www.mvps.org/access/lookupfields.htm for a critique. The Lookup Field appears to contain the category, but it doesn't; as you've seen it actually contains the numeric ID.

To concatenate the looked-for values you need to create a Query joining your main table to the Categories lookup table, and pull the category text from the Categories table, rather than the ID from the main table.

Note that users should NEVER be required to work with table datasheets; create a Form instead. It's easy to put combo boxes (lookups) on a Form, even without using the Lookup Field type in the underlying table.
