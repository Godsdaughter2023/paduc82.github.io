So what you are saying as I understand it is that there is no way to determine from the message which project it belongs to, so that there will have to be some user interaction to direct the message?

That being the case, it is easy enough to read an Access table into an Excel userform combobox (or listbox), from which it is but a simple matter to use that information to save the message in a suitable location. With access to some key information about your data, not available in the thread, you should be able to work out how to modify the following Outlook macros to achieve your aims.

You will need the name of the access database, the table name, the column of that table to display and the column of the combobox to use to name the file. This assumes that your data contains valid filename characters.

Obviously I test this as I don't have enough information, but the modules themselves are well tried in other projects and you should be able to adapt them to your needs, if you have VBA knowledge.

You need a simple VBA userform with a combobox (or a list box) and two command buttons. The code for that userform would be something like the following.

Option Explicit

Private Sub CommandButton1_Click()
    With Me
        If .ComboBox1.ListIndex < 1 Then
            MsgBox "Select the project!"
            .ComboBox1.SetFocus
            GoTo lbl_Exit
        End If
        .Tag = 1
        .Hide
    End With
lbl_Exit:
    Exit Sub
End Sub

Private Sub CommandButton2_Click()
    With Me
        .Tag = 0
        .Hide
    End With
lbl_Exit:
    Exit Sub
End Sub

In an ordinary Outlook module, add the following:

Option Explicit

Sub ProcessMsg()
Dim olMsg As MailItem
Dim oFrm As New UserForm1
Dim strPath As String
Const strDatabase As String = "C:\Path\DatabaseName.accdb" 'The name of the access database
Const strTable As String = "TableName" 'The name of the table

    On Error Resume Next
    Set olMsg = ActiveExplorer.Selection.Item(1)
    With oFrm
        AccessFillList .ComboBox1, 1, strDatabase, strTable
        .Show
        If .Tag = 0 Then GoTo lbl_Exit
        strPath = "C:\Path\" & .ComboBox1.Column(1) 'The combobox column to use
    End With
    Unload oFrm
    SaveMessage olMsg, strPath
lbl_Exit:

    Exit Sub
End Sub

Private Sub SaveMessage(olItem As MailItem, fPath As String)
Dim Fname As String
    Fname = Format(olItem.ReceivedTime, "yyyymmdd") & Chr(32) & _
            Format(olItem.ReceivedTime, "HH.MM") & Chr(32) & olItem.SenderName & " - " & olItem.Subject
    Fname = Replace(Fname, Chr(58) & Chr(41), "")
    Fname = Replace(Fname, Chr(58) & Chr(40), "")
    Fname = Replace(Fname, Chr(34), "-")
    Fname = Replace(Fname, Chr(42), "-")
    Fname = Replace(Fname, Chr(47), "-")
    Fname = Replace(Fname, Chr(58), "-")
    Fname = Replace(Fname, Chr(60), "-")
    Fname = Replace(Fname, Chr(62), "-")
    Fname = Replace(Fname, Chr(63), "-")
    Fname = Replace(Fname, Chr(124), "-")
    SaveUnique olItem, fPath, Fname
lbl_Exit:
    Exit Sub
End Sub

Private Function SaveUnique(oItem As Object, _
                            strPath As String, _
                            strFileName As String)
Dim lngF As Long
Dim lngName As Long
    lngF = 1
    lngName = Len(strFileName)
    Do While FileExists(strPath & strFileName & ".msg") = True
        strFileName = Left(strFileName, lngName) & "(" & lngF & ")"
        lngF = lngF + 1
    Loop
    oItem.SaveAs strPath & strFileName & ".msg"
lbl_Exit:
    Exit Function
End Function

Private Function FileExists(filespec) As Boolean
Dim fso As Object
    Set fso = CreateObject("Scripting.FileSystemObject")
    If fso.FileExists(filespec) Then
        FileExists = True
    Else
        FileExists = False
    End If
lbl_Exit:
    Exit Function
End Function

Private Sub AccessFillList(ListOrComboBox As Object, _
                           iColumn As Integer, _
                           strDatabase As String, _
                           strTableName As String, _
                           Optional PromptText As String = "[Select Project]")
Dim RS As Object
Dim CN As Object
Dim dbEng As Object
Dim db As Object
Dim numrecs As Long, q As Long
Dim strWidth As String
    Set dbEng = CreateObject("DAO.DBEngine.120")
    Set db = dbEng.Workspaces(0).OpenDatabase(strDatabase, 2)
    Set RS = db.openrecordset("SELECT * FROM " & strTableName)
    With RS
        If .EOF Then Exit Sub
        .MoveLast
        numrecs = .RecordCount
        .MoveFirst
    End With
    With ListOrComboBox        'load the records into the columns of the named list/combo box
        .Clear
        .ColumnCount = RS.Fields.Count
        If RS.RecordCount > 0 Then
            .Column = RS.GetRows(numrecs)
        End If
        strWidth = vbNullString
        For q = 1 To .ColumnCount
            If q = iColumn Then
                If strWidth = vbNullString Then
                    strWidth = (.Width / 2) - 4 & " pt"
                Else
                    strWidth = strWidth & (.Width / 2) - 4 & " pt"
                End If
            Else
                strWidth = strWidth & "0 pt"
            End If
            If q < .ColumnCount Then
                strWidth = strWidth & ";"
            End If
        Next q
        .ColumnWidths = strWidth
        If TypeName(ListOrComboBox) = "ComboBox" Then
            .AddItem PromptText, 0
            If Not iColumn - 1 = 0 Then .Column(iColumn - 1, 0) = PromptText
            .ListIndex = 0
        End If
    End With
    RS.Close
    db.Close

    Set RS = Nothing
    Set db = Nothing
    Set dbEng = Nothing
lbl_Exit:
    Exit Sub
End Sub

willy denoyette say:

Inspect your power settings using powercfg.exe.

Start a command prompt and issue the following commands:

powercfg /a

This will return the sleep states your system supports.

powercfg /l

Lists the defined power schemes and the active scheme.

powercfg /q <guid-of-the-active-scheme>

Will list all properties of the active scheme, check them and adjust if needed.

Note also that there is a difference between sleep and standby. Older systems do support standby , but not sleep (also called modern standby).

tblDistList

Dim rs As Recordset

Dim strTo as String

diane poremsky mvp (slipstick.com) say:
When you export, the pst is opened in Outlook at least temporarily, and after the export is finished the PST is closed again. If you have the PST open in Outlook all the time, you don't need to close Outlook to sync the PST. None of the sync utilities, including dropbox, sugarsync or SkyDrive will be able to sync the PST when it's opened in Outlook.
