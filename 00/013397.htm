Hi Joseph,

If you only deployed AD sync ( not deployed an Exchange Hybrid) and want to manage all Exchange attributes from the local AD, it is not feasible currently.

Since there is no Exchange server in your organization, all the mailboxes are provisioned in the Exchange Online. The attributes of the mailboxes are managed in the Exchange Online Admin Center. Although you can edit some Exchange attributes via ADSI edit after extending the AD Schema, it is not supported by Microsoft. If you want to manage all the Exchange attributes of the mailboxes from the local, you need to perform an Exchange Hybrid deployment.

Moreover, some attributes can be written back to on-premises server, but it is only in a Hybrid deployment. For more detailed information, please refer to Table 2 in this article.

The short answer is, I just copied the contents of the Apex.xml file that came with Office 2010 -- whatever is there, is what works.

The long answer is that the xmlns attribute declares a namespace (hence the "ns"). There is no requirement that the value be a valid URL, or even that it be in the format of a URL; it just has to be unique. URLs, real or fake, that begin with the address of a real site are often used as namespace IDs because they are unique.

General Windows Phone 8.1 feedback - Microsoft Community

essthree say:

General Windows Phone 8.1 feedback

I know I could have probably put all of these in existing threads on this board, but then I'd be bumping like 10 threads and that would probably end up just as/potentially more annoying.

Anyway, I pretty much prefer almost all of the changes made in Windows Phone 8.1. There are a couple of things I don't like though:
1. Music app lags a lot and one time when locking the handset the music skipped. It's just not as good as the original music hub in performance (though I do like the new features like removing tracks from playlists etc.)
2. Can't change default alert tone any more (you can choose custom tones, yes, but you can't change the default one).
3. Have to go into a menu to access photo uploads rather than having text you can touch (sometimes I take photos I don't want to back up and quickly jump into the photos app to stop them uploading. It takes more button presses to do that now).
4. Picture quality (in MP) options have been removed from both the stock camera app and the Nokia Camera app.
5. You can no longer change specific regional settings (ie. first day of the week, date formats, etc.), only country specific settings.
6. mobile data speed settings are in another settings screen in the cellular settings (ie swapping between 2G/3G/4G) requiring more taps/screen transitions to get to.
7. Battery saver doesn't remember your previous setting if you have it set to "now until next charge". If you switch it on, it does nothing. You have to change the setting back to "when battery is low" or "always" and then BACK to "now until next charge" for it to do anything. The on/off switch should toggle it, but doesn't.

8. Internet explorer doesn't force pages to give you the desktop version even if you have that option enabled (I've been getting mobile versions since installing 8.1).

9. No one-touch control for silencing the phone (have to put both ringer and media volume levels to zero).

That's all I can think of off the top of my head. There are more though. I understand that it's a preview and a lot of these things may be fixed, but better to speak up, I think. I really am happy I updated though. It's worth the annoyances.
