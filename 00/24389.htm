ken sheridan say: Like most experienced Access developers I very rarely have occasion to use macros, but to replace the current subform in a subform control with another form, you would need to change the SourceObject , LinkMasterFields and LinkMasterFields properties of the subform control.  Assuming the form you want to load into the subform control is not to be linked to the parent form, the latter two would be set to zero-length strings.  The following is an example of how to do it in code as the button's Click event procedure, which I used to test this in one of my own forms:

    Dim ctrl As Control
    
    Set ctrl = Me.Parent.sfcContacts
    
    With ctrl
        .SourceObject = "fsubContactEmployers"
        .LinkMasterFields = ""
        .LinkChildFields = ""
    End With

If the new form is to be linked to the parent form the LinkMasterFields and LinkMasterFields properties would be set to the appropriate column names, both as string expressions delimited by quotes characters.

To switch back to the original source object in the subform you would of course need a button in the new form to reverse the above.

If you are unfamiliar with entering code into a form's, report's, report section's  or control's event procedures, this is how it's done in form or report design view:

1.  Select the form, report, section or control as appropriate and open its properties sheet if it's not already open.

2.  Select the relevant event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new line(s) between these.
