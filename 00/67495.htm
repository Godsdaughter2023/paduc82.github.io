lava k say:

Hi puzzled09,  

 

Welcome to Microsoft Answers Community Site.

 

The Code 10 error code is generated when Device Manager has insufficient information or the error that is bubbled up by the device drivers is not recognized by Device Manager. Additionally, this error may occur when one of the drivers that the device needs does not start. If many drivers are being used by a particular device, it can be difficult to determine the one that failed.

 

One of the major cause for this is that network adapter is not currently supporting this feature. You may download the latest driver / update the driver for your network adapter or chipset, and then try to uninstall and reinstall this component.

 

Reference Link:  Description of Code 10-related errors that Device Manager generates in Windows-based computers

 

Update the driver


Open Device Manager, in the device Properties dialog box, click the Driver tab, and then click Update Driver to start the Hardware Update Wizard. Follow the instructions to update the driver.


You may be prompted to provide the path of the driver to the location where you have download the latest driver from the hardware vendor’s Web site.

Reference link:

1: Get help with Device Manager errors

2: Update a driver for hardware that isn't working properly

3: Tips for fixing common driver problems

 

Before installing the updated drivers you can create a system restore point, so that if something gets corrupted, there is a backup in place and where can use a roll back option.

 

Follow the steps below to create a system restore point:

1. Click Start, type systempropertiesprotection in the Start Search box, and then press ENTER. If you are prompted for an administrator password or for a confirmation, type the password, or click Allow.

2. Wait for Windows to search for available disks and most recent restore points. In the System Properties dialog box, on the System Protection tab, click Create.

3. Type a name for the restore point and then click Create.

4. After the restore point has been created successfully, click OK two times.

Note:   If System Restore is turned off, click to select the local disk, click Apply and then click Create.

 

For more information check this link: System Restore: frequently asked questions

 

Let me know if that work. Good luck!

heath herpin say:

Actually, yes.  You CAN set up a Windows backup to include a system image of ONLY the C: drive.  Here's your problem...

The Windows 7 style backup and restore software will automatically select each drive/partition for the image that Windows calls on during a full system startup.  What this means is that if you have ANY programs on the D: that start up automatically either via a Registry Key or a service that starts it (such as iTunes), then Windows will automatically include the D: in the image backup.  You can't do anything about it except scrub your Windows Startup for ANYTHING pointing to the D: and either move it to the C: or remove it completely.  Disabling a service doesn't work.  You'd have to completely remove it.  There are pages and pages on the internet on how to do it and you could damage your system if you do it improperly, so be careful.  But you can use Microsoft's SysInternals to find those items.

Once you've eliminated all those D: calls, you can reset your backup and suddenly the D: is selectable as a backup location (where you can save your backups to).

How do I know this?  I've been working with Windows Backup for about 8 months now since Windows 10 and my new PC.  I have a 256GB m.2 PCI-Express Boot Drive, a 1TB SSD D:, and a 3TB L: which is my backup drive.  I image my 256GB boot drive, and every few months, I'll catch the backup size jump from ~50GB to ~570GB.  And I always invariably find out I installed something to the D: instead of the C: (like MSI motherboard tools) that now start via a service and it lumped the D: into the image backup.  And I have to go and remove the MSI software and bam, back to 50GB.

a. user say:

Go to Start / Control Panel / Administrative Tools / Computer Management / Disk Management and see if you can find the USB Drive.  Right click on it and click on change Drive Letter and see if F is available.  If so, change it to F and see if the process now works.  If F is in use, see if you can change it to something else temporarily and that should free up  F for the USB drive (remember to change things back when you're done).  If that doesn't work, then I have no idea why the password reset process isn't working.

If that doesn't work and you don't have another Administrator account on the system you can use to fix this problem,enable the Hidden Administrator Account (HAA) using http://www.computerworld.com/s/article/9001970/How_to_access_the_true_Administrator_account_in_Windows_Vista?taxonomyId=17&pageNumber=1.  If you can't remember the password (and it requests one – it may not), try nothing as that's probably what you (or the vendor) did during installation. Login to this account and change the passwords for the other accounts (and I'd create another Administrator account while you're at it just in case this or something else happens again). Once done, be sure to deactivate the HAA to save it in case it's needed again and for security purposes (since people often try to hack into systems using this account).  Do NOT use the HAA as your administrator account because if you lose the only administrator account on the system or it becomes corrupt again then you are hosed.

That's as far as I can go assisting you due to Microsoft Policy:http://support.microsoft.com/kb/189126.

I hope this helps.

Good luck!
