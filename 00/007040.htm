Access cannot display this join in Design View, but it should work (hopefully) if you switch to Datasheet View. You'll have to use SQL View to edit the query.

If you'd rather work with Design View, you can remove

(Entitle_Blowout.DataValue_2WC = Left(CASHSWP3_Entitlement_Values.[Data Value],3) AND

scottgem say:


But I get the error message that MS Access can't represent the join.  Can I not join just part of a field to another field between two tables?

No, Access cannot represent such a Join. This is why fields should contain only one piece of data. Its a lot easier to concatenate values together then to parse them apart. So this represents a fundamental design flaw.

However there is another workaround. Create a query with a calculated column that parse out the 3 leftmost characters. Then use that query in your join.

Finding third party applications running under your application - Microsoft Community

erik davidsson (msft) say:

Finding third party applications running under your application

Sometimes, you may find an issue that you suspect is caused by an application from a third party manufacturer that has loaded .DLL files into your application. This article explains how you can find out what .DLLs are loaded into your application.

Alternative 1:

Use Process Explorer, as outlined here: https://support.microsoft.com/en-us/help/970920/using-process-explorer-to-list-dlls-running-under-the-outlook-exe-proc

Alternative 2:
Use ProcessMonitor by following these instructions:

1. Gather a ProcessMonitor trace of your application using these instructions.

2. Open the Filter menu and click "Filter..."

3. Click Reset

4. Choose "Process Name" "is" and write your application (Excel.exe was used in the example below)



5. Click Add, and it will show up like this in green:



6. Click "OK". You will now see a list where only the application you specified is listed. If you see nothing, you may have misspelled the application .exe, or what you wrote was not running.

7. Scroll down a few thousand lines, as it may be that not all .DLLs are loaded immediately, and doubleclick on any entry. You will see this:


8. To the bottom right, there is a column "Company". Click on that header to get the list sorted by company that published the .DLL.

9. Most entries will probably be Microsoft, but at the bottom and top of the list you will find entries that are from other companies, or are from Microsoft or other companies but are not listed as such.


Based on what you find when performing the above steps, you may want to consider disabling or temporarily uninstalling the 3rd party components there, as they may be what's introducing the unexpected behavior you are seeing. This is not guaranteed to find all influence that 3rd party applications can have on your application if the issues is that e.g. 2 running application are competing for the same files, but it usually is sufficient.

One thing you may want to perform is to search online for each 3rd party application that you find in this list together with the problem that you are facing, and you may find if this is a known issue.

You can also use this method to find the versions of the 3rd party applications that you are using. You may find that you are using files that are maybe written several years ago, and don't necessarily work well with the current application you are using.

ossiemac say:

Try the following

Sub CopyIfZero()

    Dim rngColG As Range
    Dim rCel As Range
   
    With Worksheets("Sheet1")   'Edit "Sheet1" to your sheet name
        Set rngColG = .Range(.Cells(2, "G"), .Cells(.Rows.Count, "G").End(xlUp))
    End With
   
    For Each rCel In rngColG
        If rCel.Value = 0 Then
            rCel.Value = rCel.Offset(0, -1).Value
        End If
    Next rCel
   
End Sub

Caveat: Blank cells also equal zero and the above code will copy the value from column F for blank cells in column G as well as cells containing zero. If you want to skip blank cells then replace the If condition with the following.

If rCel.Value = 0 And rCel.Value <> "" Then
