null values in queries

I have a query to retrieve data from a table that has some records with null values in the fields.  When I use an IIF statement to retrieve certain data and also retrieve records that have null values with the record, the final result retrieves only the records with all the fields populated with data and does not retrieve the records with null values in some of the fields.  I need to be able to retrieve both sets of data within a single query request.
john w. vinson say:

The problem is apparently that you're applying LIKE criteria to multiple fields... and a NULL value is not LIKE anything. Any comparison - equality, inequality, LIKE, greater or less than - to a NULL field returns NULL, and the record is not retrieved.

Instead of a criterion such as

((([Customer Contact List].Customer) Like (IIf([Forms]![Customer Contact List Parameter Form]![Customer]="All",([Customer Contact List]![Customer]),[Forms]![Customer Contact List Parameter Form]![Customer])) & "*") ...

consider using

((([Customer Contact List].Customer) = [Forms]![Customer Contact List Parameter Form]![Customer] OR ([Forms]![Customer Contact List Parameter Form]![Customer] = "All")...
