ben m. schorr -bschorr say:

You can't share an Outlook PST file between two instances of Outlook at the same time.  If you close Outlook on the Vista machine you could, in theory, open it on the XP computer.  I say "in theory" because using a PST file, as your default message store, across a network is not a supported configuration - there is considerable risk of database corruption if you do it that way.

The better plan is probably to just copy the PST file from the Vista machine to the XP machine, configure your e-mail account to leave a copy of messages on the server (or use IMAP instead of POP) and just run two separate installs against the same mailbox that way.  It could get out of sync, I guess, but probably won't.

Odd question.  Why are you wrong?  Because you are, as the actual results demonstrate.  Sorry, but it is that simple.

Excel (MS) defines its own behavior.  It has chosen to define that an empty cell is treated as zero.  It's been that way "forever" -- for as long as I can remember.

=A1 returns zero if A1 is empty.  We must write =IF(A1="","",A1) to avoid it.

One fix for your formula above is:

=IF(AND(F10<>"",F10=N10),"match","no match")

We cannot really say that F10=N10 implies that either is zero.  If you had replaced N10 with 0 in the formula, then we could.

[EDIT] Technically, even that fixed formula above is incorrect, since if both F10 and N10 are empty, I presume you would consider that a "match".  I don't know how far to go with rewriting your formula because I don't know how you want to treat all possible situations.
