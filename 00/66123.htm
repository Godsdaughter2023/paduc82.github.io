Disable the IP Helper service:
1. Hold the Windows key and type R, enter "services.msc" (without the quotes) and press Enter
2. Scroll down to the IP Helper service, right click on it and select Properties
3. In the dropdown box that says "Automatic" or "Manual", set it to Disabled and then click on "Apply"
4. Then click on "Stop" to stop the service from running in the current session
5. Click OK to exit the dialog
 

Method 3


Disable IPv6 and remove IPv6 virtual adapters:
 
Try uninstalling IPv6 on all interfaces, removing any IPv6 virtual adapters, and resetting the TCP/IP stack. To remove IPv6, go into the properties for each network adapter and either uncheck the box next to the protocol "Internet Protocol version 6 (TCP/IPv6), that will disable it, or select it and click uninstall, which will remove it off the computer. NOTE: You should do this for each network connection, even if they are disabled.

Important: This section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Windows Vista cannot obtain an IP address from certain routers or from certain non-Microsoft DHCP servers
 
To resolve this issue, disable the DHCP BROADCAST flag in Windows Vista. To do this, follow these steps:
1. Click Start, type regedit in the Start Search box, and then click regedit in the Programs list.
2. If you are prompted for an administrator password or for confirmation, type your password, or click Continue.
3. Locate and then click the following registry subkey:
4. HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\Interfaces\{GUID}
5. In this registry path, click the (GUID) subkey that corresponds to the network adapter that is connected to the network.
6. On the Edit menu, point to New, and then click DWORD (32-bit) Value.
7. In the New Value #1 box, type DhcpConnEnableBcastFlagToggle, and then press ENTER.
8. Right-click DhcpConnEnableBcastFlagToggle, and then click Modify.
9. In the Value data box, type 1, and then click OK.
10. Close Registry Editor.
 
By setting this registry key to 1, Windows Vista will first try to obtain an IP address by using the BROADCAST flag in DHCP Discover packets. If that fails, it will try to obtain an IP address without using the BROADCAST flag in DHCP Discover packets.

john_barnett say:

Hello Danny,


Most newer PCs running Windows 8 have EUFI BIOS's. To ensure fast booting works with the EUFI BIOS legacy booting is disabled by default. You can dual boot by enabling legacy booting in the BIOS 'but' the PC manufacturer doesn't like you doing that and, if it is anything like my Laptop manufacturer, Samsung, you will immediately invalidate your warranty.


First thing I would contact the PC manufacturer to see if dual booting will actually invalidate your warranty. If it does then you and you alone take the risk as the manufacturer will simply not want to know. If the warranty is not affected then the manufacturer can tell you how to enable the legacy BIOS.


Once the legacy Bios is enabled you need to create a new partition that will hold the Windows XP operating system. You also need to know whether the manufacturer has relevant drivers for the XP operating system so that all hardware will function correctly. If no drivers are available then you will have problems with the XP install.


Installing Windows XP is no different to installing any version of Windows. You just have to remember that XP has to be installed on the partition you recently created. If you don't do that it will install on the C: drive and over-write your Windows 8.


The next major hurdle is getting the two to work. Because Windows 8 is already installed you have a problem with the boot menu. Technically XP should be installed first and then Windows 8 - obviously this sis something you cannot do. Because XP relies on a boot.ini file and Windows 7/8 relies of GPedit file you will not be able to immediately run Windows 8. When you reboot you will simply boot directly to Windows XP. To change the boot option I suggest that you take a look at this link: http://www.tomshardware.com/answers/id-1680664/dual-boot-windows.html#. In particular pay attention to the post by WinOutreach2 (May 24 2013).


Personally, I have a feeling that you may have problems with your warranty and also difficulty getting XP drivers for your hardware so you may not be able to readily solve your problem in the way you want.

    dogtamer say:                

         This is a rather old thread that I just found as it's one of the first-page "hits" for a Bing search for "BackgroundConfigSurveyor". I was searching due to the fact that while perusing Task Scheduler I found that this PerfTrack task was disabled and had never run on a Windows 7 SP1 Home Premium build. It would appear that most folks searching for this task have done so due to rcv'g a "Task BackgroundConfigSurveyor:The task image is damaged or been tampered with" error message; interesting info concerning how some users have repaired that may be found @ this 7Forums thread:

    Windows 7: task backgroundconfigsurveyor:The task image is corrupt; there's also useful information about troubleshooting & repairing such an error @ this TechSupportGuy thread: Task Scheduler error msg.

         My purpose in posting this reply is add information more specific to Jimkie's original question of Jan. 2010, in case other folks should come across this particular thread. A detailed, albeit long, explanation of what PerfTrack is may be found in the video @ this link: David Fields and Bill Karagounis: Inside Windows 7 - Reliability, Performance and PerfTrack. 

         The way to change the start time or other triggers, actions, conditions, or settings (the four middle tabs of a task in Task Scheduler) for any task found in the Task Scheduler Library is to highlight (single-click) the task in the upper pane of Task Scheduler, right-click to invoke the context menu, and select "Properties" in the resulting dialog box.  In the Properties sheet that will subsequently open, one may click on any of the tabs and change the information therein.

         The "Triggers" and "Actions" tabs require that the "Edit" button at the bottom of those displayed tabs in order to change any of the information; the conditions & settings tabs allow direct access to the boxes that may be "ticked" (checked) or "un-ticked" - no "edit" invokation is required for those tabs. It would appear to be fairly standard in W7 builds that clicking the "Edit" button for the Triggers & Actions tabs appears to be a little sluggish in responding - perhaps more so with the former than the latter.

         As a last bit of information that may prove helpful, it's always a good idea to back things up before one changes them; tasks may be backed up through use of the "Export" function available in the right-click context menu of the task as mentioned earlier in this post, which will open the Windows Explorer "navigation window" and allow you to browse through files & folders to find a location where the exported task may be saved - one must of course remember where it was saved, which is why so many simply choose the desktop (it's typically at the top of the "folder tree" displayed in the navigation window, and everyone can usually find their desktop {well, maybe not so much for Windows 8 users, @ least perhaps not yet!} and easily access their saved file if need be).

    frederik long say:

    I was unable to remove those files.

    This method should at least get the files out of harm's way:

    1. Press Shift+Restart to force Windows to boot into the Repair Menu.
    2. Select Advanced Options twice.
    3. Select Command Prompt.
    4. Select an admin account to log on.
    5. Enter its password and select the the appropriate keyboard layout.
    6. Identify the correct drive letter for your Windows installation. It could be a drive other than C: but it won't be X:. You do it by typing the commands and pressing Enter after each:
        dir C:\Win*
        dir D:\Win*
        dir E:\Win*
        until you find the Windows folder.
        I shall call this drive "Q:" even though it won't be Q:.
    7. Type these commands and press Enter after each:
        cd  /d  Q:\
        md  Junk
        cd "Program Files (x86)"
        cd "Microsoft Office"
        cd "Root"
        cd "Office16" (If this name is incorrect, type dir{Enter} to see the correct name)
        takeown  /f  one*.*  
        icacls  one*.*  /t  /grant everyone:F
        move one*.*  \Junk

    The above commands will move all OneNote files into the Junk folder, thus making them inactive. You might be able to delete them after a reboot. The process is, of course, fully reversible.
