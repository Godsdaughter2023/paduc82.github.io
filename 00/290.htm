A PDF is essentially a graphical representation of the document. Embedded files are converted to images. You can, with Adobe Acrobat, embed the original document in the PDF, but even that will lose the embedded objects.

There are two possibilities. One is to hyperlink to a web location where the embedded PDF files are stored, that the users of the PDF may have access to. Is it just the number that you want bold, or can the text to which the Heading 3 Table is applied also be bold.  If the latter, you should get what you want by modifying the Heading 3 Table style so that its font is bold.  If doing that also causing the Heading 3 to also be bold, then it would appear that you have Heading 3 based on Heading 3 table, rather than the other way around.

I strongly suggest not using color as a way to convey meaning. Instead, if you use a helper column and put the color name in that column, you can easily use conditional formatting to apply the color from the color name in the helper column and you can easily use the color name for counting values.

If, however, you manually set a text or cell fill color, counting these cells by their color will be a lot harder, impossible with functions and will require VBA. 

Color is not data. If you want to keep things simple, don't use color as data.

rick rothstein, mvp - excel say:

This macro should do what you want; just set the FirstRow constant (in the Const statement) to the row number for the row containing the first index formula (here I have assumed Row 2 leaving Row 1 as a header row)...

Sub ExtendIndexedFormula()
  Dim LastColumn As Long, LastRow As Long
  Const FirstRow As Long = 2
  Application.ScreenUpdating = False
  LastColumn = Cells(FirstRow, Columns.Count).End(xlToLeft).Column
  LastRow = Cells(Rows.Count, LastColumn).End(xlUp).Row
  Cells(FirstRow, LastColumn).Resize(LastRow - FirstRow + 1, 2).FillRight
  Application.ScreenUpdating = True
End Sub
