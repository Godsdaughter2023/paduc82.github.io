balaji kundalam say:

1. Paging is a technique where only the necessary system files and drivers are stored in RAM during boot. When any program is opened, the necessary files for the program is loaded to RAM. When paging is enabled, these files are moved to hard drive in a file (C:\pagefile.sys). This is done to provide space for more programs. This file size is proportional to the amount of RAM. It is set, by default, by Windows as about twice the size of RAM in use. It is recommended not to change the page value.

 

2. Yes, that's true. More and more temporary files are created each time an installaion/uninstallation is made. Some of the files are deleted automatically, some are left. So, these files pile up and eat free space.

The reason it doesn't work is that even if you close eraser and reopen it as administrator there is still a process running (that wasn't shut down when you closed eraser) in the background without administrator rights.

The solution is to ctrl-alt-delete -> windows task manager -> processess.

Kill the process called "eraser.exe", and then you re-start Eraser as administrator. 

Thus the background process will also gain administrator rights.

malkeleah say: First of all, you shouldn't be using Guest. The Guest account is a special system account, not one meant for when you are feeling hospitable. It is disabled by default in Windows XP, Vista, Linux, Unix, and OS X for a reason.

"The Guest account is intended for users who require temporary access to the system. However, if this account is enabled, a security risk may exist because an unauthorized user could gain anonymous access to the system through this account."

http://technet.microsoft.com/en-us/library/bb418978(TechNet.10).aspx

So if you enabled Guest, disable it now and create a Standard/Limited (Vista, XP Pro/XP Home) for your visitors. Call the new user account something like "Visitor".

Any restrictions you set using Group Policy (Vista Business/Ultimate) on a non-domain computer will apply to all user accounts.
