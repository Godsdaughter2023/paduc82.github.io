jay freedman say:

You cannot create or edit equations in Word Online (https://support.office.com/en-us/article/Differences-between-using-a-document-in-the-browser-and-in-Word). To do anything with math areas, you must choose the "Edit in Word" command to download the document into desktop Word.

In desktop Word, you can indeed convert text in LaTeX format to Word equations. The steps are:

  • Select the text representing a single equation or math expression.
  • Press Alt+= or click Insert > Equation to create a math object containing the selected text. This causes the Equation Tools context tab to appear on the ribbon.
    • In the Equation Tools ribbon, if the LaTeX button isn't selected yet, click it. Then it should stay selected for the rest of the current Word session.
  • At this point you have two choices: You can click the down arrow at the right end of the current equation and click Professional (but you'd have to do this individually for each equation), or you can wait until you've marked mos or all of the math areas as equations and then click "All - Professional".

petrib say: There is no "standard version JVM". There are three different Java specifications; J2ME ( Java 2 Micro Edition), J2SE (Java 2 Standard Edition ) and J2EE ( Java 2 Enterprise Edition ).

Then there is a multitude of different Java VM implementations from various vendors/manufacturers for all these Java versions. All with their own quirks, extensions, defects, etc.

Anyway, the J2ME implementation in Symbian/S60 is, as far as I know, based on the J2ME Java VM from Sun, so it is as compatible as they come. ( There is no complete J2SE version that is small enough to run on mobile phones yet; some partial implementations on some [older] models, only. )

petrib say: Those phones already have the Java runtime for J2ME (Java 2 Micro Edition), also known as MIDP/CLDC.

There is no Java runtime for J2SE (Java 2 Standard Edition), sometimes just called JRE, Java Runtime Environment for those phone models (or any phone model I know of).

daiichi---01 say: J2ME is the Java version designed to run on Mobile phones. If you're trying to run an applet that was designed for the J2SE ("Standard Edition")--it will not run on mobile phones.

OK, despite what all the "E90 is not a communicator" folks out there want, all Symbian-based devices (including the E61i) are CELLPHONES FIRST. If someone looking for something that gives them all the power of their laptop, they're looking in the wrong place. Especially for us Symbian people, it's likely the best that we'll ever do is the J2ME version.

Of course J2ME is very powerful in its own right... but Java applications have got to be specifically written for it.

If that's not good enough, you might want to look at the more powerful of the Windows Mobile devices. They too have J2ME, but people have ported J2SE to it such as the Mysaifu JVM (http://www2s.biglobe.ne.jp/~dat/java/project/jvm/index_en.html). Of course, this is "hackware" and you'll probably run into lots of oversights and incompatibilities with a typical J2SE app anyway... but it's about the only way in the near future where you can even hope to get a full J2SE JVM working in a pocket-sized device; Also remember that the power comes at a price: typical Windows Mobile PDAs that have the power to run a full J2SE rarely has the battery life necessary for a true road warrior.

lizette ags say:

Hi,

In determining Sender Reputation, many companies use Bayesian Spam Filtering method. This method uses series of word-use to identify spam e-mail. There is an algorithm involved to categorize those words. Example of those words are "free" or words in all caps. As technology goes, spammers found a way to bypass this method. Anti-spam filters pursued a better way to identify spam from permission-based email marketing messages. By creating a profile of what a spammer looks like based on sending and user behaviors, mailbox providers and anti-spam solution providers were able to do a much better job of keeping spam out of their users’ inboxes.

This also provided a more predictable framework for senders on what to do, and what not to do to get their emails delivered to their users’ inboxes. This is what sender reputation is all about.

So using this method, Outlook detects the "reputation" of the sender and acts accordingly.
