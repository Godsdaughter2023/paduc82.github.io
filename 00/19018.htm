Function Convert_Degree(Decimal_Deg) As Variant
'source: http://support.microsoft.com/kb/213449
'
'converts a decimal degree representation to deg min sec
'as 10.46 returns 10° 27' 36"
'
'Added Note: the ° can be put into a cell directly
'by using the Number Pad and pressing and holding the [Alt] key
'and pressing [0] [1] [7] [6] on the number pad.
' - JLatham
  Dim degrees As Variant
  Dim minutes As Variant
  Dim seconds As Variant
  With Application
     'Set degree to Integer of Argument Passed
     degrees = Int(Decimal_Deg)
     'Set minutes to 60 times the number to the right
     'of the decimal for the variable Decimal_Deg
     minutes = (Decimal_Deg - degrees) * 60
     'Set seconds to 60 times the number to the right of the
     'decimal for the variable Minute
     seconds = Format(((minutes - Int(minutes)) * 60), "0")
     'Returns the Result of degree conversion
    '(for example, 10.46 = 10° 27' 36")
     Convert_Degree = " " & degrees & "° " & Int(minutes) & "' " _
         & seconds + Chr(34)
  End With
End Function

ken sheridan say:

Format(Date, "mm-dd-yy-hh-mm-ss")

Another point to note is that, unlike the ISO standard for date/time notation, which differentiates between months and minutes by case, the Format function in Access does so by using 'm' as the month character and 'n' as the minute character.  Also, the Date function returns a zero time of day.  To return the current time also, call the Now function.

So the above would be:

Format(Now(), "mm-dd-yy-hh-nn-ss")

hi indu,

to be able to create a new site, a user needs to have the appropriate permission on the parent site.

if you would like all your users to be able to create a site, you may create a specific permission level especially for that.

to do this, you can follow the guide below:
1.    first go to the parent site’s ‘site settings’.

2.    under users and permissions, click ‘site permissions’.

3.    on the permissions ribbon, click permission levels.

4.    click ‘add a permission level’.

5.    insert the name of the permission, and select other permissions that you would like your users to have.

6.    to be able to create new sites, you would need to select the ‘create subsites’ permission.

7.    after creating the permission, assign it to those who you would like to have the new ‘create subsites’ permission.

alternatively, you can create a group that contains users that you would like to have this permission; this makes it easier to check who has the permission. if you would like everyone to be able to create a site, you can also enter ‘everyone in my organization’ to a group.

8. after assigning the appropriate users with the permission, they may create a site by going to either site contents > new subsite or going to the sites page and clicking on the 'new' button.
