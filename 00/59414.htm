System Restore doesn't know or care about any of your personal files.

If you accidentally delete a file and it is not recoverable from the Recycle Bin, there are third party tools that you can look into to see if they can recovery your lost file(s).  Generally speaking the data from "deleted" files are still on the hard drive until that space they occupy is needed for something else, then that space where your deleted file is will be overwritten.

Keep in mind that the more you "use" your disk and read/write data to it, the less your chances of recovery the file are going to be since XP may decide to use that area of the disk where your deleted file is.  So if you delete something accidentally, you should try to recover it ASAP.

Recuva

paulo reichert say:

This unfortunately is not up to the device itself but to the software you're connecting to.

Your Surface Book has a very high-DPI screen with a resolution of 3000x2000 pixels and to make the artefacts on screen usable on a 13.5 inches display it needs to employ scaling (Settings -> System -> Display -> Change the size of text, apps, and other items). Scaling is basically telling the application to multiply the horizontal and vertical pixels it would use to draw an element by the scaling factor (2x in the case of my Surface Pro 4).

Unfortunately some applications (mostly older versions) do not support scaling and will draw pixels 1:1, displaying the problem you describe.

All that said, if you give more details of what you are connecting to (is it a remote desktop? are you using a remoting client such as Citrix or Remote Desktop? etc?) we may be able to give you a bit more help.

bossdweeb say:

d3dx9_42 is actually one of the most recent DX files (along with d3dx10 and 11 _42, so this game (title unknown)

would not be all that old.


However these files did not come with Win7, so if you have upgraded to Win10 from Win7 or Vista installation which

never had a DX update you will  need to update DX.

This used to be done with the DirectX Web installer, but that seems to have been abandon by Monopoly$oft.

I believe that Windows updates is supposed to be doing the job of the DirectX Web Installer now, but if not, you can use the DirectX runtime installer (June 2010) . This will overwrite all DX files and add any that are 'missing'.

Download DirectX End-User Runtime Web Installer from Official Microsoft Download Center

Most recent games  install (re-install) DirectX by default, but the error may actually be due to a corrupt d3dx9_42

file. This will be replaced when you reinstall DX with the runtime installer.
