Custom word templates & Ribbon - Microsoft Community

gaz260610 say:

Custom word templates & Ribbon

Hi

Is it possible to create a customised ribbon in word 2010 that can show only customised letter templates?

 

I'm trying to create quick access for staff so all they need do is click on a template from the ribbon and up pops the company logo and footer etc...

 

Is this at all possible because i've spent all morning trying to figure this one out but to no avail.

graham mayor say:
The best way to do this is probably to create an add-in template with a custom ribbon - http://gregmaxey.mvps.org/Customize_Ribbon.htm that calls a macro, that opens a document based on a particular company template depending on whch button is clicked.. The following example adds two buttons to call a letter and a memo template.

The ribbon code could be something like -

<customUI xmlns="http://schemas.microsoft.com/office/2009/07/customui">
  <ribbon>
    <tabs>
      <tab id="CustomTab0001" insertBeforeMso="TabHome" label="Templates">
        <group id="CustomGroup0001" label="Company Templates">
        <button id="Custombutton0001" size="large" imageMso="DocumentTemplate" label="Letter" onAction="MytRibbonBtn" />
          <separator id="PersSeparator0001"/>
        <button id="Custombutton0002" size="large" imageMso="DocumentTemplate" label="Memo"             onAction="MyRibbonBtn" />
        </group>
      </tab>
    </tabs>
  </ribbon>
</customUI>

and the associated macro would be

Sub MyRibbonBtn(control As IRibbonControl)
Select Case control.ID
    Case "Custombutton0001"
        Documents.Add Options.DefaultFilePath(wdUserTemplatesPath) & "\Letter.dotm"
    Case "Custombutton0002"
        Documents.Add Options.DefaultFilePath(wdUserTemplatesPath) & "\Memo.dotm"
End Select
End Sub

You will of course need a Letter.dotm and a Memo.dotm template to call from the macro.
Add uniquely named buttons and separators to the ribbon as required and add case statements to reflect the button names. The OnAction="MyRibbonBtn" statement is used throughout.

See also http://www.gmayor.com/installing_macro.htm
