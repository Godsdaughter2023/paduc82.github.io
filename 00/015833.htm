techfreak_ say:
Skype 7.x can only read the last 30 days from the cloud. Any older history is obtained from your local history file (main.db). If you stopped using Skype 7.x on that machine during the months you are missing, or your local history file became partially corrupt, you would be missing data. Skype 8.x/12,x can pull up to 2 years of old history from the cloud, though that current amount is a  little over a year. Unfortunately there is no easy way to extract cached cloud data from Skype 8.x (other than copy and paste), at this time. If you have Skype for Windows 10 installed, you could browse to that point in history and try to extract the data from its local history file (skype.db). Unfortunately that history file is incompatible with SkypeLogView and would require you to manually extract the data with a SQLite Viewer.

ruwim.b say:

No. The new Microsoft Skype for Desktop (the 8.X version) does not have a local storage file similar to what was used with the classic (the 7.X) version. The chat history is accessed from the Cloud (Microsoft server network).

 

There is a local settings folder where some info on how and where the data can be accessed, but not a readable history file. The path to the local settings folder is:

 

%appdata%\microsoft\skype for desktop
