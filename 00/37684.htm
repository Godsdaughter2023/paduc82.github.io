You need to set windows firewall to block all outbound connections. Then create rules for what you want to allow to make outbound connections. 

Thanks, I figured this out a few days ago. Unfortunately, this is not the solution I was looking for. Although I have since switched to Norton Firewall, I will post what I found out here, for others who might have the same problem.

Here is what I wanted to do: I wanted to select certain programs on my computer, and (1) block all inbound connections to them, plus (2) block all outbound connections made by them,except (3) allow outbound connections made by those programs only to the specific IPs that I permit.

This cannot be done with the default firewall behavior, which is to allow all outbound connections made by all programs. You can indeed make a firewall rule like the one I described above, but Windows will simply ignore it. So while your rule says "block all outbound except to this specific IP", in fact Windows will allow all outbound connections to all IPs. This is because the default behavior of the Windows firewall is to allow all outbound connections freely.

As you said, this default can be changed to block all outbound connections. If you change this default behavior, then your rule will indeed be enforced. However, changing the default behavior of the Windows firewall in this way is impractical for most people, because Windows firewall was not designed for this purpose. Let me explain what I mean.

  • If you change the default behavior of Windows firewall to block all outbound connections, then all outbound connections will be blocked. However, many Windows services need to make outbound connections, and so all these services willstop working. Your Windows installation will be crippled.
  • What is worse, you will have no indication why it's crippled. All you will see is that some things stop working. Windows firewall provides no means to figure out what stopped working and why.
  • How do other firewalls let you do this then? It's simple - if you set default behavior on most firewalls to block outbound connections (hey, you don't even have to set it that way, that's their default behavior from the very start), then this is what happens: when a Windows service needs to start an outbound connection, the firewall detects it, blocks it, but thenpops up a message on your screen telling you "Program X, located on your hard drive at the path Y, attempted to open a connection to IP address Z. Do you want to allow this connection?" Further, this pop-up box gives you the option to answer "yes" either temporarily (allow this once), or permanently (always allow this).
  • As a result, Windows continues to work fine. The firewall just has an initial learning phase of a day or so in which you have to tell it "these Windows programs are fine, let them through", and it is easy to do so by just clicking "allow" on the pop-up warning. You remain safe from all other potentially dangerous programs, because the default continues to be that the firewall will block all outbound connections unless you specifically allow them.
  • However, Windows Firewall is broken in this sense because it does not give you any pop-up warnings when it blocks outbound connections. As a result, there is no easy way to allow certain essential Windows components through the firewall. You are left to your own devices to figure out what stopped working and why. Then you have to manually make an exception for it in the firewall, searching for the program on your hard drive, then making the rule for it. Worse, for many Windows components, it is not immediately obvious which program to allow. For example, if your Windows Update stops working and you think it's because of the firewall, even if you figure out which file on your hard drive does Windows Update, you still won't get it to work just by allowing that program through the firewall. This is because many Windows programs depend upon some other Windows service to do the job. In case of Windows Update, you would need to find out which Windows Service the Windows Update program uses to connect to the update servers, then allow that in your firewall.
  • Needless to say, this is no job for an amateur, and is very tedious even if you are knowledgeable about this sort of thing. You may ask why Windows Firewall has such broken behavior when it comes to outbound rules. The answer is, it was a deliberate design decision. MS figured that giving the user too many pop-up warnings was a bad idea, it irritated the users. Perhaps they were influenced by such complaints re: Vista and its UAC warnings. And they also figured that blocking outbound was not needed, on the premise that if a program is already on your computer, it must be okay. Therefore, they changed the default behavior on Windows firewall to allow all outbound, and while the option exists in the firewall to change this default, actually changing the default turns the firewall into an unusable mess, because it was never intended to be used that way.
  • I can't blame MS for making this design decision. They are probably right that most people don't like pop-up warnings, even though they only occur once per program or service. They are probably also right that if the program is already on your computer, it must be safe. Hey, if it weren't safe, wouldn't your anti-virus and malware detector have already caught it? You do run an anti-virus and anto-malware program, don't you? You know they're infallible, right? So everything's fine! john_barnett say:

Hello Jags,

If the PC is sold on then the copy of Windows can go with it. Having an OEM licence simply means that a) the licence is less expensive than a retail licence and b) the licenced software can only be installed on the original machine. In other words you cannot use the CD of Windows XP that you have to install another copy of Windows on another machine. It must always be installed on the first machine you installed it on.

While 'retail' versions of the software are more expensive, the buyer is not restricted as an OEM would be. For example, with a retail copy, you could install it on one machine then, after a few years, scrap that machine and re-install the copy of Windows onto a new machine. With an OEM copy you can only install it on the very first PC. If that PC is ultimately scrapped the software licence dies with it.
