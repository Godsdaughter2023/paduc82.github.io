michelle v [msft] say:

Hi Michael,

This is the information shared to us by our engineering team. This issue typically occurs when the user has multiple copies of the same app installed (e.g. perhaps they copied the original Outlook/Word apps before applying the update). The best way to see if this is the problem: 

  1. Open Terminal
  2. Type “defaults read com.microsoft.autoupdate2”
  3. Look for duplicate app registrations in the output (e.g. two registrations for MSWD15 (which is Word), or OPIM15 (which is Outlook)
  4. Beside each registration is the path to the app, so you may see something like this:
  • MSWD15     /Applications/Microsoft Word.app
  • MSWD15     /Applications/Backup/Microsoft Word.app
  1. Using the information from above, delete the duplicate/backup copies of each app using Finder. You should be left with a single copy of each app directly in the /Applications folder
  2. Launch one of the apps, such as Word, and then quit it
  3. Go back to Terminal and type “defaults read com.microsoft.autoupdatge2”  …the duplicate entries should now be gone
  4. Launch Microsoft AutoUpdate (e.g. Help -> Check for updates), and accept any updates that it detects
  5. You should no longer be prompted multiple times for the same updates

Let us know if this solves the issue. Hope this helps!

Thanks,

Michelle  V.
