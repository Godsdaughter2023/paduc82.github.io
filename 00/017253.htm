Access 2016 and Access 2010 - Microsoft Community

jennysearcy say:

Access 2016 and Access 2010

If I design a database using Access 2016 can I run it using a computer system that uses Access 2010?

scott mcdaniel say:

Possibly, but in general you'll have troubles with references. The Access/Office references won't "downgrade" from 2016 to 2010. You can generally go the other way (2010 >> 2016), since Access can upgrade references (for the most part).

That said - it could be done, but you'd have to (a) be sure to not use any features introduced after 2010 and (b) re-create the references on the 2010 machine after installing the database. This would mean you couldn't deploy a compiled file, since you can't change references in a compiled file.

As a general rule of thumb, you should develop in an environment that matches the "lowest common denominator" for all the machines you must support. In your case, that would be a machine running Access/Office 2010, and no higher.
