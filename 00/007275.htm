john mcghie say:
Hi Mitja:

Jim's correct, we have no reports of font issues for the past several releases of Word.  Word "cannot" in fact, switch fonts on you: but the system can!

There are two conditions under which you may see formatting variations upon opening documents:

  1. If the required font is not installed, the system will switch to the closest alternative.
  2. If there is an Attached Template and it has been attached with "Automatically update document styles" turned on.  Since people have forgotten or never learned to use templates these days, that is a very rare circumstance.
With Times New Roman vs Arial, I think we can safely assume that the font is installed, on any computer in the world.  You might like to try switching in Calibri or Cambria and round-tripping to see what happens.

There is a chance that the "C"-series fonts Calibri and Cambria are not installed on an open-source system, so it would give you another data point in your research.

You can also (in Word go to Tools>Templates and Add Ins and observe the setting of "Automatically update document styles".  If it is on, that's your problem, switch it off.

What I suspect is happening is that the formatting is being applied as direct formatting, not as a style.  This would be very difficult to detect if the underlying style is Normal Style.  But you can do it:
  1. On the View menu, choose Reveal Formatting.  The cursor will change to a text balloon icon.
  2. Click in the suspect text.
  3. You will see a pop-up with two information boxes, one for Paragraph formatting, the other for Font Formatting
  4. In each box there are two or three information lines: the top one shows you the properties coming from the Paragraph Style, the middle one shows you the properties coming from the Character Style (if there is one) and the bottom line shows the properties coming from any direct formatting that has been applied.
If direct formatting has been applied to the text, the style name will be unchanged; but the formatting will be wrong.  If that's what's happening, a quick whack over the ears of the LibreOffice user along with an exhortation to be a little less "Libre" with his formatting may just solve your problem!

Clutching at straws here: I suppose it is POSSIBLE that text pasted in from a web page in an open-source word processor "might" use the HTML text sizes of "larger", "smaller" etcetera. In that case, the text run actually has no size specified.  Which means that every time you open the document, the font size will become "larger" or "smaller".

I have never seen Word do this, but it's a well-known bug in Entourage/Outlook when working in HTML.  To the best of my knowledge, Word ALWAYS specifies an exact font size.  In .DOCX, I know it does, because you can read the code and see it.

I can only caution you to do your very best to avoid the obsolete .doc format.  Not only is it excessively large and somewhat fragile, but the damage you do saving a modern document back to the obsolete format is getting quite substantial.  There was a rumour out there that .doc and .docx were equivalent.  They are not and never were: .doc won't store the modern graphics and tables and styles that .docx will.  So all of that gets stripped or degraded on the way to .doc.

Hope this helps

john mcghie say:
Hi Mitja:

Well, it seems to me that what you have IS an interoperability issue, and as you say, it does not happen if you use .docx

But yes, the freeware word processors are fairly limited, and we should expect them to have some disconcerting habits when trying to handle full-function Word documents.  We should not expect miracles at that price.  By all means continue to use the freeware if it meets your needs, but you should expect there to be some "issues".  There's a limit to what unpaid volunteers can accomplish.

Have you tried the operation with Apple's Pages and TextEdit?  They seem to do a pretty good job, with both .doc and .docx.

I am not aware of any change in 10.3.6 that would cause this problem, but Microsoft never tells us what's in the updates anyway, so that's not saying much.

Just for completeness, I should mention that there is no such thing as a "Default" font in Microsoft Word.  Internally, Word is a collection of "Lists of things", styles being one, fonts being another.

The style table goes from 0 to "many" depending how many styles are added to the document. It starts off with the built-in and pre-defined styles, of which there are about 140.  They don't appear until you use them.  Normal style is style 0.  Every other style inherits from Normal style until we break the chain.

Often (but not always) Normal style is set to be Times New Roman font.  But usually at 12 points size.  So I do not know where your font is coming from, but it's unlikely to be Word.  Some other styles are set to Arial or Calibri in an unmodified Normal template: but they can be changed by things such as the Table of Contents generator.

The font in Normal style is the closest thing that Word has to a "default" font; but technically, it's not a "default" it's a "user setting".

Hope this helps
