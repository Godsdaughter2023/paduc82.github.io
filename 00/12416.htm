Concurrent events Query - Microsoft Community

colin jarvis say:

Concurrent events Query

Hi,

I have a table that contains telephone call traffic and need to write a query that will calculate the maximum concurrent calls during a day.

 

The table contains the start time and end time of each call and there are simultaneous calls throughout the day.

 

I need to count the maximum number of calls that are connected at the same time.

 

Kind regards

 

Colin 

 

hansv mvp say:

Let's say you have a table tblTimes with (among others) date/time fields StartTime and EndTime.

1) Create a query to return the unique start times:

 

SELECT DISTINCT StartTime
FROM tblTimes;

 

Save this as qryStartTimes.

 

2) Create a query to count the number of concurrent calls at each start time:

 

SELECT qryStartTimes.StartTime, Count(tblTimes.StartTime) AS NumberOfCalls
FROM qryStartTimes, tblTimes
WHERE tblTimes.StartTime<=[qryStartTimes].[StartTime] AND tblTimes.EndTime>[qryStartTimes].[StartTime]
GROUP BY qryStartTimes.StartTime;

 

Save this query as qryNumberOfCalls.

 

3) Create a query to return the maximum numner of concurrent calls per day:

 

SELECT CDate(Int([StartTime])) AS StartDate, Max(NumberOfCalls) AS MaxNumberOfCalls
FROM qryNumberOfCalls
GROUP BY CDate(Int([StartTime]));

j.e. mcgimpsey say: As Bob said, Excel can't read your mind regarding which cell to use, and there's no mechanism within copy/paste to change what's on the clipboard.

Possible alternatives (though Find/Replace is almost certainly the best option):

  • Put the address of the base cell to use (e.g., Q8) in a separate cell (say, Z1) and use INDIRECT():
           =Q9 * (INDIRECT($Z$1) + 1)

      Enter a different address in Z1 when you want/need to change base cell.
  • Use a named range (Insert/Name/Define):
            Name:  BaseCell
            Refers to:  =Sheet1!$Q$8

      then in your cells:

             =Q9 * (BaseCell+ 1)

      choose Insert/Name/Define again to edit the cell reference.

Using Mail merge together with SharePoint Lists is a real pain

Unbelievable but using a normal SharePoint list and no document library is a painful thing. The customer wants to use a central address list an extended contact list in SharePoint 2013 Foundation and Office 2013.
He wants to filter the addresses and then use this partial list for mail merging with Word 2013.

First I tried to export the SharePoint List to Excel or Access. But filtering is useless you will always get the whole list. The only workaround to get a partial list is to use views.  I personal think for the most end-users this is no practicable way.

Next I made a live link to SharePoint with Access, creating the queries and starting Word mail merge from Access. But this isn’t working because Word needs a local database and you always get an error that the database is open exclusive mode.  The only workaround is to make a local database and to reimport the SharePoint list if something is changing.

Using Outlook for mail merging is also ill-conceived because you don’t get easily extended contact-fields. You have to use existing fields but it doesn’t work with lookup fields.  And you get the whole list there is no way to filter it before.

With Excel the same problem can't use live sync together with queries. Filtering on the Excel side is possible and to copy the only visible filtered data with copy and paste for a second source. Unfortunately the customer doesn’t want to work with Excel.

There is no way to make a direct connection from SharePoint to Word mail merge. You have to write a macro. But again you get only the whole list and you can’t use lookup fields. That is way filtering afterwards in word is useless in our case. By the way Office 2013 doesn’t include the right driver you have to download Microsoft.ACE.OLEDB.12.0 driver separately.

All together this simple task was a complete nightmare.

What is missing for a real integration?

  1. A common SharePoint Database Connection provider is missing.
    This includes one provider regardless of using 32 Bit or 64 Bit version of Office.
  2. Support of SharePoint pre-filtered lists in the Office applications without a view.
  3. SharePoint should have a icon to connect directly to Word.
  4. SharePoint should be able to export only selected items to the Office applications.
  5. Support of lookup fields list in the Office applications.

It seems that Microsoft works too much in the cloud world and is forgetting the on premise customers. They didn’t make their homework.
Sorry for the hard words but this is not the integration that I expect from a formerly called Office Server and we are already at version 15 of Office

Linking dropdown boxes in a form to filter a subform. - Microsoft Community

a. user say:

Linking dropdown boxes in a form to filter a subform.

I have a single table with 3 columns and 1000 records. The first column can contain the numbers 0-6. The second column can contain the numbers 1-9. And the third column can contain letters A-Z.

I would like to have a form that has two dropdown boxes. The first one will contain all the unique values for column 1 and the second one will contain all the unique values for column 2 based on the value that was chosen for the first dropdown. After selecting the choices from each dropdown the resulting matches from column 3 will be displayed in datasheet view.

Example:

In the first dropdown I choose "6". In the second dropdown I choose "7". The values from column 3 are displayed for all records which have "6" in column 1 and "7" in column 2.

I am using Access version 16.0.11601.20174 64-bit.

scottgem say:

This is standard process called synchronized or cascading combos. It is well documented if you look up either of those terms. 

Basically, Your first combo has a Rowsource of 

SELECT DISTNICT field1 FROM table Order by field1;

The second come has a Rowsource of

SELECT DISTINCT field2 FROM table WHERE Field1=Forms!formname!field1 ORDER BY field2;

You can then use the After Update event of Combo 2 to Filter the form.

Me.Filter = "Field1 = " & Me.Combo1 & " AND Field2 = " & Me.Combo2

DoCmd.RunCommand acCmdApplyFilterSort.

ken sheridan say: You might like to take a look at DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

In this little demo file the section on 'retrieving data from the database' includes a form in which values can be selected in country, region and city combo boxes in the form's header.  The combo boxes are correlated in their RowSource properties in the way Scott described.  To restrict the form's recordset to those rows which match the selected values the form's RecordSource references the combo boxes as parameters like this:

SELECT [FirstName] & " " & [LastName] AS FullName, Address,
City, Region, Country
FROM (Countries INNER JOIN Regions ON Countries.CountryID = Regions.CountryID)
INNER JOIN (Cities INNER JOIN Contacts ON Cities.CityID = Contacts.CityID)
ON Regions.RegionID = Cities.RegionID
WHERE (Countries.CountryID = Forms!frmCorrelatedCombos!cboFindCountry
    OR  Forms!frmCorrelatedCombos!cboFindCountry IS NULL)
AND (Regions.RegionID = Forms!frmCorrelatedCombos!cboFindRegion
    OR  Forms!frmCorrelatedCombos!cboFindRegion IS NULL)
AND (Cities.CityID = Forms!frmCorrelatedCombos!cboFindCity
    OR  Forms!frmCorrelatedCombos!cboFindCity IS NULL)
ORDER BY Contacts.LastName, Contacts.FirstName;

To effect the restriction as a value is selected in each combo box, the form is requeried in each combo box's AfterUpdate event procedure.  You can see the full code by clicking on the 'Show Code' button in the form.
