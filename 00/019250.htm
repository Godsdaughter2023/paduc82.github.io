If the data is already alvailable on the SubForm, the you should use the data already retrieved for the SubForm rather than another Query that would need to retrieve the data again.  Doing this will improve the response of your Form.

marshall barton say:

Deleting and creating the LDB file might take a longish time, depending on the infrastructure that is needed to do that.  Things like your network speed, network and server workload and even the depth of the path to the file.  Assuming nothing extraordinary going on, the LDB file should disappear within a few seconds, especially if the files are on your local hard disk and your system is not overloaded with other tasks.

marshall barton say:

I don't know what your Delay procedure does, but if it's a sleep kind of thing, then that would probably be the dominate factor in calculating the total time.

DoEvents tells Access to go ahead and handle other waiting events and if there are none of those, tell Windows to go onto other tasks' time slices.  That means DoEvents can take an indeterminate amount of time, but without consulting a Windows MVP, I would guess time slices are probably in the range of a few milliseconds.  (Don't forget that processor speeds can do at least two billion operations per second or a couple of million ops per time slice.)  I think one implication of that is you don't gain much by calling your delay function.  Another point is that Access creates several tasks while running your program (e.g. running queries, calculating control expressions, displaying forms, saving data sectors, etc.) so it's conceivable your program has a large background of activities that have to be completed before it can clean up and close.

The For loop uses iterations and I used 1 To 100, because I figured 100 would be more than enough time slices for Access and Windows to clean up the LDB file.  You can change that to whatever is enough, but I'm pretty sure that more than a few thousand would be excessive.  After however many iterations, the loop bails out and displays a message that things are taking so long that you should assume something is broken.

Hi Fernando,

 

In your case, you may try the steps below to see if it works:

  1. Use Keyboard shortcut Win+R to open run function and input regedit to open Registry Editor.
  2. Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\OfficeSoftwareProtectionPlatform.
