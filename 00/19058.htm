stefan blom say:

Since Word has more than one type of style and different style types can apply the same kind of formatting, a hierarchy of formatting/styles is required. The hierarchy determines which style "wins" when different settings are applied to the same object. For example, if a paragraph style applies the Arial font to text and you apply a character style whose font is set to Times New Roman, the font of the character style "wins." Of course, this principle works for any formatting that two style types have in common (such as borders or font size). Table styles are particularly complex because they apply table-specific formatting but they also apply font and paragraph formatting to text in cells.

In this particular case, you have no other option than creating a paragraph style and use it to apply shading to the content of table cells.

The example below shows a table with a table style whose shading is red color for all rows. One cell is formatted to have yellow shading via a paragraph style (and cell margins are set to zero for the table style).

Of course, in a protected document or template, the paragraph style used to mimic cell shading must be included in the list of of permitted styles.

stefan blom say:

For paragraph styles to display font settings properly inside a table cell, you have to make sure that the font settings specified in the document defaults matches those of the Normal style.

This design doesn't make sense, but it is part of the convoluted styles hierarchy involving paragraph styles, table styles, and the document defaults.

Here's how to proceed: In the Styles pane, which you can display by pressing Ctrl+Alt+Shift+S, click the Manage Styles icon. In the dialog box, click the Set Defaults tab. Make sure that you change the font settings (font name and size in particular) so that they match those of the Normal style. When you are done, click OK to confirm the settings.

After you have done this, you can apply any style, with any settings, to text in table cells, and you will see the correct formatting (= the same formatting that you see when you apply those same styles to text outside tables).

Credit goes to Pam Caswell for discovering the connection between these settings.

Are the images inserted other than in-line with text?  If so, the caption may be tied to the anchor of the image and that can sometimes end up somewhat removed from the image itself and hence the numbering of the captions may not be what is expected.

stefan blom say:

A caption will end up in a text box if you add a caption to a selected object in the drawing layer (such as a picture whose Text Wrapping is set to something other than "In line with text").

As Doug suggested, it is often easier to insert objects "In line with text." That way, captions will be normal text paragraphs in the body of the document. You can always wrap a frame around the object and its caption if you want text to wrap around it. Use the Insert Horizontal Frame command which can be added to the Quick Access Toolbar.

lady16 say: Depending on what is in your document, it may be easier to manually change.  However, here is a work around, if you so choose.  Tab leaders will be the font size you have selected when you put the leader in, so if you want your font to be 12 pt, but the leader to be 5 pt, you change your font size to 5 pt, then insert all your leaders, then go to Find and Replace, Special, Any Letter - then Replace With - select Font and change your font to whatever size you desire, but this does not change numbers, so you would have to repeat those steps to change any numbers as well.  HTH!!!

  2. Select the TOC
  3. Replace ^t with ^t formatted to the smaller font size.
