hansv mvp say:

First, see what happens if you import all database objects from the frontend into a blank new database.

If the size of the new database is not significantly smaller, do the following:

  1. Compact the new database.
  2. Note its file size in Windows Explorer.
  3. Delete a database object (form, report, etc.)
  4. Go back to step 1, and repeat.

Perhaps you'll find that the file size suddenly drops a lot. The database object that you just deleted is the (most likely) culprit.

Word 2011 & Word 2016 do not use the same Normal.dotm template - each has its own respective default template. Nor are they "interchangeable".

When you modify a Style or create a new one, the modification affects only the current document unless you check the box labeled Add to template. However...

The simplest means of choosing a different default font is through the Format> Font dialog. Select the font & attributes you want, then click the Default button & confirm the updating of your Normal.dotm template.

You cannot avoid the use of a Theme in Word documents. However, if you don't want to get involved with them, just ignore them & continue to use the default [Office] Theme. Your default font will be honored in new documents unless you select a different Theme or different set of Theme Fonts.
