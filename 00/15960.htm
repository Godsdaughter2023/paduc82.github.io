jay freedman say:

The folder where Word saves AutoRecover files is not (usually) the same as the Unsaved Files location.

By default, AutoRecover saves to C:\Users\<your name>\AppData\Roaming\Microsoft\Word, and that's where you should find an .asd file after a crash. To verify the location of the folder on your computer, look on the Save page of the Options dialog. On starting Word, if there are files there, Word should open a panel on the left side offering to let you recover them.

Unsaved files -- files that are closed normally, but never saved to a document file -- go to C:\Users\<your name>\AppData\Local\Microsoft\Office\UnsavedFiles. Each time you open Word, any file in that folder that's more than four days old is deleted. In the case of a crash while editing a file that has been previously saved, I wouldn't expect to find anything there.

This information still may not help you if the technician who fixed your computer deleted an .asd file from the AutoRecover location, or saw the panel in Word and clicked the button to discard the file.

Selecting all the text in a document, including in text Boxes - Microsoft Community

dianemkim say:

Selecting all the text in a document, including in text Boxes

Hi, I want to change the font size in a very large document.  When I select all (I'm using ctrl A) and change the font size, the font size in the Text Boxes remains unchanged.  Is there a way to select all the Text Boxes also please (I have over 100, so don't want to have to do it manually!)?
jay freedman say:

No, there is not a way to select all the text boxes along with the main text. Text boxes are in the drawing layer, like floating graphics, and they don't respond to Select All or to the button in the Styles dialog that lets you select all instances of a style.

You can use a macro like this (see http://www.gmayor.com/installing_macro.htm if needed) that resizes the main text and then resizes the text in each individual text box (but not headers/footers, text boxes anchored in headers/footers, footnotes, endnotes, etc.):

Sub FontSizeAll()
    Dim newSize As Integer
    Dim shpTextBox As Shape

    newSize = Val(InputBox("Enter new font size:"))
    If newSize = 0 Then Exit Sub

    ' set font size of main document text
    ActiveDocument.Range.Font.Size = newSize

    ' set font size of text boxes
    For Each shpTextBox In ActiveDocument.Shapes
        If shpTextBox.Type = msoTextBox Then
            shpTextBox.TextFrame.TextRange.Font.Size = newSize
        End If
    Next
End Sub

When you do this, though, you'll find that the text boxes don't move proportionally when the text around them expands; and they don't resize to fit the larger text inside them. This will leave you with still a lot of work to do.

john korchok say:
to John S R Wilson for the original code):

Sub ChangeTextColor()
Dim oSld As Slide
Dim oShp As Shape
Dim x As Long
  For Each oSld In ActivePresentation.Slides
    For Each oShp In oSld.Shapes
      If oShp.HasTextFrame Then
        If oShp.TextFrame.HasText Then
          With oShp.TextFrame.TextRange
            For x = .Runs.Count To 1 Step -1
              If .Runs(x).Font.Color.RGB = RGB(255, 255, 255) Then
                .Runs(x).Font.Color.RGB = RGB(0, 0, 0)
              End If
            Next x
          End With
        End If 'has text
      End If 'has textframe
    Next oShp
 Next oSld
End Sub

Sub ChangeTextColor()
