The changes of Symbian Belle comparated to Anna.Detailed info, with - Microsoft Community

alb3530_ say:

The changes of Symbian Belle comparated to Anna.Detailed info, with screenshots

With Symbian Belle release date becoming closer each day that passes by us, I think it's a good idea to spot the main changes it'll bring to Nokia Phones.

Besides the already-known-by-many features, here you'll find some small details that you probably didn't knew about Belle...

Note: The phones I used to make this post were, representing Symbian Belle, the Nokia 700, whereas Nokia N8 is the one with Symbian Anna.

A important thing to notice here is, the version that will be brought to Anna phones may differ from the one found in Nokia 700 (though I think it won't), so I can't guarantee every detail found in this post will actually apply to the "upcoming" Symbian Belle.

With that said, it's time to start them...


As most people know, the most noticeable changes are related to visual elements.
There're new widgets, and the old ones went through changes as well.
Clock now is bigger, and available as two widgets: analog and digital.
Time, battery, signal and other regular indicators are now located at the top of the screen.
You can also see buttons are now identified by an icon prior to a text label, and so, there's room for more buttons, like the call button available in home screen (between "Menu" and "Options" buttons)



Belle (on the top): Bigger clock, changed widgets, and more home screens.


When editing the home screen, placeholders to widgets/shortcuts are now in the size required for that object. As opposed to Anna, where there were fized sizes regardless of what application would be put to that place, now this place is dynamically allocated, depending on the widget/application.

You will also notice there's no need to use a Shortcuts-like widget in Belle, as now you can natively place applications in the home screen without needing a specific widget for that task.



Belle: Native support to applications in home screen, and dynamic placeholders for widgets/applications.In the third screen, Anna, with its fixed placeholders and two "shortcuts" widgets



A relevant addition was the "pull-down status bar": it contains buttons to toggle Mobile Data, WI-FI, Bluetooth, and silent profile.Besides, you can see notifications there too.To call this bar, just drag the top status bar to the botton, and the pull-down status bar will be revealed.
The cool thing is you can open and close this notification area from any application, and without delays.



The pull-down status bar gives quick access to common tasks, such as toggle WLAN, bluetooth, and view notifications.



Menu was prepared not to rely on folders anymore: all applications are, by default, in root folder.Yet, folders can be created.
A newly installed application appear with a red star at the top left corner of their icon.

Aditionally, a cool effect in menu is the current home screen's wallpaper remains in background when you open Menu, as seen below

 

 


Belle (on top): red star helps identifying new applications, that are placed, by default, in root folder.


A search function was implemented, allowing you to quickly find the application you need.
By pressing and holding the application icon, you have the option to add it to the home screen.



Menu: Background image, search function, and possibility to add application to home screen.


A small change that also affects Menu application is it was removed from open applications view.


Belle (top screenshot): Menu doesn't appear as an open application.




To make it easier to find applications, it's now possible to arrange them by name.
User Interface in general also had changes.
From Symbian 9.3 to Symbian Anna, a "Show open applications" option was in almost (if not all) all context menus, appearing always on the top of the list. That option was taken away from Belle.Now the only way to show open applications is by holding the Home key.
Another change is related to context menus: (like the one in the screenshots just below, called by pressing "Options" in menu)
When the context menu is appearing, you can cancel it by clicking outside it.In Symbian Anna, the only way to cancel such a menu is by pressing cancel.Personally, I like the approach present in Anna better, cause it avoid cancelling the menu should you click outside it unwillingly.Anyway.....


Belle (on top): Itens in menu can be arranged alphabetically.Also, no more "Show open applications", previously present on top of context menus


A cool thing is if you can't identify the function of some button by looking at the icon, you can press and hold it, and its label will appear.


The label is still available by holding the button.Here, you see "Shuffle" label.



The dialogs are now centered when in portrait orientation.In Symbian Anna, dialogs were already centered, but only when phone was in landscape mode.
For me, it wasn't a cool change, since you can't be sure the exact place where an "OK", "Yes", or (put your favorite button name here) will appear.As the dialog is centered, depending on the amount of text, the buttons can appear more or less near the botton of screen.In Symbian Anna, you know exactly where the button will appear, so that you can confirm or cancel dialogs more quickly.


In the first picture, Belle, with its centered dialogs and non-predictable buttons.In the second, the same dialog in Anna.



The typing system was also improved.Now, more screens support in-line editing.It means, in screens like the one where you setup the alarm, you won't see that editing screen that covers the whole application's screen anymore.Instead, the keyboard just appears in the bottom, leaving application's fields visible.

Belle (top screen): in-line editing for more applications




Lock screen is also more stylish.More notifications (like calendar entries), and background image.


On the top, Belle's lock screen, with calendar entries and colorful background.On the botton, Anna's black lock screen.


If you find some mistake, or have something to add to what is posted here, feel free to have your say in this story....



Best Regards

graham mandeno say:

I suggest you write a function to turn the ID number into a vertical string.  Paste the following function into a module:

Public Function PivotNumber(n As Long, Optional ByVal MinDigits As Integer) As String
Dim sTemp As String
Dim sResult As String
Dim i As Integer
  If MinDigits < 1 Then MinDigits = 1
  sTemp = Format(n, String(MinDigits, "0"))
  For i = 1 To Len(sTemp)
    If i > 1 Then sResult = sResult & vbCrLf
    sResult = sResult & Mid(sTemp, i, 1)
  Next i
  PivotNumber = sResult
End Function

Then you can create a long, narrow textbox on your report and as its ControlSource, use:

If you want to include leading zeroes for smaller numbers, then specify the optional MinDigits argument.

It looks like you have put the code in the report's class module, rather than creating a standard module.  This is OK, provided the report to which the class module is attached is the ONLY place you call the PivotNumber function.  I would put it in a "standard" module.  Cut the function out from where it is, then create a new module by clicking Insert > Module, and paste the code in the new window.  Also, ensure that you have Option Explicit after Option Compare Database in EVERY module.  (This stops mis-spelled variables from being implicitly declared and can save hours of hair-pulling!)

Then you should compile all your code to check for errors (Debug > Compile <project name>).

Finally, save the module with a name like mdlGeneralCode (it should be a name you have not used for anything else in your database).

What is the name of the autonumber field?  Is it "ID"?  If not, then you are using the wrong field name in your ControlSource.  It should be:

=PivotNumber([name of autonumber field])

If you want a minimum of, say, 5 digits, then use the second argument like this:

=PivotNumber([name of autonumber field], 5)

Then 123 will be printed as:

0

0

1

2

3
