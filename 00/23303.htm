 "The Outlook data file  C:\Apps\Mail\Outlook.pst has reached maximum size. To reduce the amount of data in this file , permanently delete some items that you no longer need. #1 - First important to identify is whether or not you are using the old format ANSI based PST file or a Unicode format. This can be done by right clicking on the <Root> of the PST file where your contact folder exists -->  Data File Properties --> Advanced -->

If the <Format> box contains the value of "Outlook Data File (97-2002)" - this PST file is the old ANSI format with a max capacity of 2gb regardless of what Outlook version is used. If the value is simply "Outlook Data File", it is a Unicode PST file with a max default size of 50gb. Unicode based PST was introduced in Outlook '2003 with default file sizes increasing with newer versions of Outlook but unfortunately, there is no "automatic" conversion from ANSI to Unicode in any version of Outlook.

A fairly comprehensive article about PST files worth reading can be found at:

Managing PST Files in Microsoft Outlook

http://windowsitpro.com/outlook/managing-pst-files-microsoft-outlook

Rather than go through all the various possibilities - the starting point would be to identify whether you are using an ANSI or Unicode PST file and include the current file size as shown in Windows Explorer. What you're various options are will depend on the answer to those questions.

The OAB is simply a specialized view of all the contact folders that are designated to be members of the OAB. This will not resolve the larger problem identified above.

Yes, it would.  I'm going to bet that the form is actually in continuous forms view, not datasheet view.  If that's the problem, then you have to implement the up- and down-arrow key behavior yourself.

One way to do that is to put this procedure in a standard module:

'------ start of code for standard module ------
Public Sub StandardKeydown(ByRef KeyCode As Integer, Shift As Integer)

    If Shift = 0 Then
        On Error Resume Next
        Select Case KeyCode
            Case vbKeyDown
                RunCommand acCmdRecordsGoToNext
                KeyCode = 0
            Case vbKeyUp
                RunCommand acCmdRecordsGoToPrevious
                KeyCode = 0
            End Select
            If Err.Number <> 0 Then DoCmd.Beep
    End If

End Sub
------ end of code for standard module ------

Then create an event procedure for the KeyDown event of each of the text boxes in the form's detail section to call the above procedure.  Each such event procedure would be modeled on the following"

'------ start of example code for form module ------
Private Sub YourTextboxName_KeyDown(KeyCode As Integer, Shift As Integer)

    Call StandardKeydown(KeyCode, Shift)

End Sub
'------ end of example code for form module ------

As I said, you would have a similar event procedure for each text box or other control that you would want to behave this way.

a. user say:

Our department upgraded from Mac Office 2011 to Mac Office 365 2016. 

I searched for and found at least 4 possible solutions. Only one worked for me (Solution 2 below).

Two of our department Users of Microsoft Office 365 Outlook for Mac found that once an upgrade from Office Outlook 2011 had taken place, they no longer were able to search within Outlook for emails to or from different contacts.

The reason is that in Mac Outlook 2011, users were able to recreate the Main Identity using the "Rebuild the Database" program. When the software recreated the database, it automatically added dates with slash marks within the new identity. 

With Outlook 2016, the software program changed in that the main database was no longer recognized because Spotlight does not recognize any name with special characters in it.

I found I had more than one identity listed, and that the default Main Identity being used as the default was not "Main Identity" and it included a date with slash marks. I deleted the identities not in use (not bolded). Then I renamed the default bolded folder to "Main Identity" and made sure to click on "Default". That solved it. 

Be sure that Outlook is not open when you try this. 


Solution 2: Eliminate Special Characters
If Outlook for Mac 2016 has some special characters such as *, /, >, +, @, etc. then, users need to change the identity name by following the steps mentioned below:

View Profile Name
Select Go >> Applications.
Click on MS Outlook >> Show Package Contents
Increase Contents, SharedSupport >> Outlook Profile Manager

Change Profile Name
Open Identity folder of Outlook 2016 from the mentioned location:
Now, rename the identity name by removing the special characters
Confirm the changes by again viewing the profile name.

Techniques to Fix Email Search Not Working in Outlook for Mac Issue
Hannah Roy | July 2nd, 2016 | Fix Errors

Nowadays Outlook is the most popular email application that is used to send and receive bulk of emails in an organization as well as personal level. It must be maintained and upgraded timely otherwise many issues may arise. Many times, it is difficult for users to execute and understand Outlook search with updated edition of application. As Search option, is the most important feature of the Outlook application. It helps to perform a specific search of a defined criterion. It helps in navigating through mails and other components quite proficiently. In the following section, we will discuss the trouble faced by the users.

Problem Description
In Outlook 2016 for Mac, users receive a message "No Results," when they try to search specific mail message or task items are not displayed in Task folder. Moreover, when users search for mail items by utilizing Mac OS spotlight search, the search is not successful. They face an issue of email search not working in Outlook for Mac:

Note: While performing the search option, user must be aware of the following facts:
Operating system must include Spotlight Search Option
Spotlight Search should be synchronized with Outlook 2016
When new data is introduced, it takes several minutes for indexing

Causes behind the Issue
The problem may arise due to following reasons as mentioned:

Indexing of Spotlight Search is incomplete. 
The name contains the special characters. 
Profile or its parent folders are in privacy tab in Spotlight. 
Index Search of Spotlight is corrupted. 

Way to Resolve the Problem
Until now, we have discussed the issue and causes behind its occurrence. Now, we will be discussing the methods to solve the problem faced by the users. There are some solutions mentioned below which helps to resolve the issue, i.e. email search not working in Outlook for Mac.

Solution 1: Complete Spotlight Indexing
When the user creates new Outlook profile in Outlook 2016 for Mac or they have imported the new data from PST or OLM files. Then the Spotlight indexing is not completed. In this situation, "No Results" message is displayed. To solve this problem, user should wait for a time to finish the indexing and then search again.

Solution 3: Remove the tab of Privacy
If the Outlook 2016, profile folder are added to Privacy tab then, the Spotlight will not index that folder location. Now, remove the location of these folders from Privacy tab in spotlight and wait for the time until these folders to finish indexing.

Solution 4: Fix Spotlight Index Corruption
If the user is still facing problem then, Spotlight index may get corrupted. To resolve this, user needs to re-index the Spotlight Index by following few steps:

Select Apple menu then, go to system preference >> Spotlight 
Click on the privacy tab and now drag the folder, which user wants to index to list of locations that is prevented from Spotlight search 
Choose the folder which is added and remove it from the list by clicking on the remove(-) button 

Now, quit the preferences of system and the Spotlight will re-index folder contents. 

After following the mentioned solutions, if the users still face an issue then, they can uninstall the Outlook 2016 from Mac machine and again reinstall.

Conclusion
In the above discussion, various methods to resolve the occurrence of issue, i.e. email search not working in Outlook for Mac are discussed. According to experts, these solutions help to improve the functioning of Spotlight as it plays a major role in searching process.
