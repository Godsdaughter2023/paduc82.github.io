Why it happens?

Due to a number of reasons the variables under registry key HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders\Cache gets changed to incorrect values and you start getting this weird message whenever you use any microsoft office application.

2. Solution:

WARNING: If you use Registry Editor incorrectly, you may cause serious problems that may require you to reinstall your operating system. Microsoft cannot guarantee that you can solve problems that result from using Registry Editor incorrectly. Use Registry Editor at your own risk



a. Start->Run-> enter "regedit" and hit 'enter' to run the program to edit Registry variables.

b. Go to folder -> "HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders\Cache"

c. Check the variable values- All the variable should have values starting "%USERPROFILE%\". Any variable which is not having this starting point is probably corrupted. In my case, all the corrupted variables had a corresponding "Backup_VariableName". Example: my "cache" variable had a corrupt value while there was a variable called "backup_cache" with correct values

d. Change the value of all the corrupted variables to correct values. Example change value of "cache" variable to "%USERPROFILE%\Local Settings\Temporary Internet Files" and so forth.

e. Close all the <acronym title="Microsoft">Ms</acronym> Office Programs and re-start. The problem should be fixed now.

saltgrass say:

The discussion of a Dynamic disk, to me, really only applies to RAID type operations.

If you are not using a drive as a Boot drive for your OS, you can configure it as GPT or MBR.  GPT is required to see all of a drive larger than 2.2 TB.

But Legacy Windows will only boot from a MBR configured drive and a UEFI install will only boot from a GPT configured drive.

There have been utilities used to set a drive up for use with MBR, but I don't have any experience.

Sub Insert_Lines_At_Cursor()
'
' Insert lines At Cursor Macro
' Macro created mm/dd/yyyy
'
' Keyboard Shortcut: Ctrl+l
'
    'ascertain number of lines to insert, convert to an integer not more than 50
    Answer = InputBox("How many lines to insert? (50 lines maximum)")
    NumLines = Int(Val(Answer))
    If NumLines > 50 Then
       NumLines = 50
    End If
    'End sub exectuion if no lines to insert
    If NumLines = 0 Then
        GoTo EndInsertLines
    End If
    'Insert specified number of lines at the cursor
    Do
        Selection.EntireRow.Insert
        Count = Count + 1
    Loop While Count < NumLines
EndInsertLines:
End Sub

 

rick rothstein, mvp - excel say:

@LJ-G

In case you are interested, your posted macro can be replaced with this much shorter (and more efficient) one...

Sub Insert_Lines_At_Cursor()
  On Error Resume Next
  ActiveCell.EntireRow.Resize(Int(InputBox("How many lines to insert?"))).Insert
End Sub

Note that I removed the 50-line maximum as I saw no reason for such a limit. If, on the other hand, you really want to impose that limit, then this slightly longer (but still shorter and more efficient than your posted code) can be used instead...

Sub Insert_Lines_At_Cursor()
  Dim LinesToInsert As Variant
  LinesToInsert = InputBox("How many lines to insert? (50 lines maximum)?")
  If Val(LinesToInsert) <= 50 Then
    On Error Resume Next
    ActiveCell.EntireRow.Resize(Int(LinesToInsert)).Insert
  End If
End Sub

To Irish628,

The following code will insert the lines. However, on my computer it takes about 50 seconds to run.

Note the comments where you might need to edit the code to suit your worksheet name and also if you do not have a column header.

Sub InsertRows()
Dim lngRow As Long
Dim lngRowsToInsert As Long

lngRowsToInsert = 29  'Edit to number of rows to insert

With Sheets("Sheet1") 'Edit to your worksheet name
  'Start from last row of data to insert rows above
  lngRow = .Cells(.Rows.Count, "A").End(xlUp).Row

  Application.ScreenUpdating = False
 
  Do
    .Rows(lngRow & ":" & lngRow + lngRowsToInsert - 1) _
        .Insert Shift:=xlDown
   
    lngRow = .Rows(lngRow).End(xlUp).Row
   
    'Next line. If no header row exists then change 2 to 1
    If lngRow <= 2 Then Exit Do
  Loop

End With

Application.ScreenUpdating = True

End Sub

If I assume that all rows after the last row are empty, then it should be much faster to move the rows themselves.

Andreas.

Sub MoveRows()
  'The line number with the header
  Const HeaderRow = 1
  'Rows to insert between each line
  Const RowsToInsert = 29
 
  Dim SourceRow As Long
  Dim DestRow As Long
  'Get the last row number
  SourceRow = Cells(Rows.Count, 1).End(xlUp).Row
  'Calculate the row number to move to
  DestRow = (SourceRow - HeaderRow - 1) * (RowsToInsert + 1) + HeaderRow + 1
  'While the source row is below the first data row
  Do While SourceRow > HeaderRow + 1
    'Move the row
    Rows(SourceRow).Cut Rows(DestRow)
    'Calculate next row numbers
    SourceRow = SourceRow - 1
    DestRow = DestRow - RowsToInsert - 1
  Loop
End Sub
