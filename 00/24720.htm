Agree max green.


Maybe another approach is also worth to try it.

First, please open a "Command Prompt (Admin)" window by one of the following ways:
Keyboard: Win+X, A, Alt+Y
Mouse: rightclick Start button, click "Command Prompt (Admin)", click "Yes" button
Touch: press-and-hold Start button, tap "Command Prompt (Admin)", tap "Yes" button


Then in the "Command Prompt (Admin)" window, please execute the following command to start a PowerShell console:
powershell


Then the command prompt window transfer to the PowerShell console window, in it please execute the following command to set its execution policy to "UnRestricted":
Set-ExecutionPolicy UnRestricted


Then in the PowerShell console window, please execute the following script command to disable the Windows Defender's service:
Set-ItemProperty -Path HKLM:\SYSTEM\CurrentControlSet\Services\WinDefend -Name Start -Value 0x00000003


Then please execute the following command to exit the PowerShell console:
exit


Then please execute the following command again to exit "Command Prompt (Admin)" window:
exit


Now please reboot your computer. Then perhaps you would get the result you want at this time.

Attention:
After you have finished your task, please remember to set the startup type of the Windows Defender's service back to "Automatic", please use the following PowerShell script:

Set-ItemProperty -Path HKLM:\SYSTEM\CurrentControlSet\Services\WinDefend -Name Start -Value 0x00000002

Good luck.

When previously I replied this post, the running system is just Windows 10 Insider Preview Build 10130,
at that time Microsoft did not restrict the permission of WinDefend, so it can be directly modified.
Maybe later Microsoft changed its permissions for accessing.
So if you met the "Set-ItemProperty: Attempted to perform an unauthorized operation." error prompts,
perhaps you need run a PowerShell script to get the permissions.


You could use Notepad to make a PowerShell script file that reads as follows and save it as "%UserProfile%\Documents\GetModiDefendPermission.ps1":
$key = [Microsoft.Win32.Registry]::LocalMachine.OpenSubKey("SYSTEM\CurrentControlSet\Services\WinDefend",[Microsoft.Win32.RegistryKeyPermissionCheck]::ReadWriteSubTree,[System.Security.AccessControl.RegistryRights]::ChangePermissions)
$acl = $key.GetAccessControl()
$rule = New-Object System.Security.AccessControl.RegistryAccessRule("Builtin\Administrators","FullControl","ContainerInherit","None","Allow")
$acl.SetAccessRule($rule)
$key.SetAccessControl($acl)


Then you could open a "Command Prompt (Admin)" window by one of the following ways:
Keyboard: Win+X, A, Alt+Y
Mouse: rightclick Start button, click "Command Prompt (Admin)", click "Yes" button
Touch: press-and-hold Start button, tap "Command Prompt (Admin)", tap "Yes" button


Then in the "Command Prompt (Admin)" window, you could execute the following command to start a PowerShell console:
powershell


Then the "Command Prompt (Admin)" window transfer to the PowerShell console window, in it you could execute the following command to set its execution policy to "UnRestricted":
Set-ExecutionPolicy UnRestricted


Then in the PowerShell console window, you could execute the two following commands to run the script:
cd $Home\Documents
.\GetModiDefendPermission.ps1
