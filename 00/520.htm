trevor rabey say:

Evan,

first of all, you should be very careful with the use of "% complete" of anything as an indicator of progress. When you say that the 20 accounting reports are "35% Complete", what exactly does that mean? Does it mean that all of the reports are in progress, or that some have not started, some have started but not yet finished, or that some are finished, or a combination of these?

When the mixed bag of percentages is rolled up to summary level (your Reports section) you get a % complete for the summary, but it has had the meaning washed out of it, so it does not really have any useful meaning.

A percentage has to be a fraction with a numerator and denominator, so what are they in your context?
% complete in MSP is strictly about the duration, being actual/total, where total = actual + remaining.

So if the task is to lay 10000 bricks in 10 days, after an actual start date (when the first brick is laid) and 4 days of continuous actual duration the task is 40% complete and has 6 days of remaining duration, regardless of how many bricks have been laid (some number between 1 and 9999 out of 10000).

That said, putting a summary section at the top of the task list and having the % completes reflect the % completes of the respective summaries lower down in the list, looks like an easy copy, paste special, paste link. Just display/insert the % complete field into the entry table first and then copy, paste special, paste link. Like this. Tasks 2 and 3 have 1 day duration but have been "marked as milestone" in task information, advanced to hide the bars on the chart. They still appear as diamonds and could be made to disappear completely by checking "hide bar" in task information.

IS there a setting to change the number of rows - I only have the 65k rows, don't know how to get all the rows to appear.
bill manville say:

65536 is the maximum number of rows for the Excel 97-2003 file format.

Any .xls files that you open will be limited to that number of rows.

 

If you have specified that you want to save files in that format by default then when you create a new workbook it will be limited to 65536 rows and [compatibility mode] will appear in Excel's title bar.

 

To change your default save format: File > Options > Save > Save fils in this format:
Excel Workbook (*.xlsx)

andy pope say: The VB code in that example is vb.net code.

Here is the VBA equivalent.

'-----------------------
Private myRibbon As IRibbonUI
Private groupWindowHidden As Boolean

Public Sub OnLoad(ByVal ribbon As IRibbonUI)
    groupWindowHidden = True
    Set myRibbon = ribbon
End Sub

Sub flipGroupWindow(control As IRibbonControl, pressed As Boolean)
    groupWindowHidden = Not groupWindowHidden
    myRibbon.Invalidate
End Sub
Sub groupWindowVisible(control As IRibbonControl, ByRef returnedVal)
    returnedVal = groupWindowHidden
End Sub
'-----------------------
