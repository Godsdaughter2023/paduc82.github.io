Another reason for this is that you have a resource issue. That means that you have many items starting when Windows starts and memory and CPU power is being consumed so that MSE can't start. If this is the case, after the PC starts, if you open MSE and turn on Real Time Protection, it should stay on. To resolve this, you should remove items from startup using MSCONFIG that you do not actually need running when the PC and Windows start.
