bob jones aka: cybertaz say:
It's difficult to give a specific answer without knowing what the Error Message states. However, if it involves PDFs as well as Word documents the odds are that it isn't caused by an issue with Word... PDF is a file format which doesn't involve Word once the command is given to generate the PDF.

The problem more likely is either with the email program's encoding of the attachment or the manner in which your email service or that of the recipient is handling it.

The fact that the Mac is new doesn't mean the either the OS installation or the Office software is up to date. Updates are often released while product is in the distribution pipeline -- Don't "suspect", confirm that all is current.

Use Apple Menu []> Software Update to confirm that OS X is current, then use Help> Check for Updates in any of the Office apps to make sure Office 2011 is at the 14.2.5 version. If you're using an email program other than Apple Mail, Entourage, or Outlook, make sure it is fully updated also.

Additional steps you can take on your end include;

  • Checking the settings of your email app. Many email programs have a setting or preference to control encoding of attachments -- Apple Mail, for example provides the Edit> Attachments> Always Send Windows-Friendly Attachments menu selection. Many email clients offer a similar setting.
  • Always use the proper file name extension [e.g., .pdf, .docx] when saving files. Without the extn the email systems have to 'guess' what the file type is... They may guess wrong, add an incorrect identifier, or otherwise damage the attachment -- or the recipient's software may simply refuse to open the file due to security concerns because it cannot be correctly identified. These problems are quite prevalent if a 'free' email service is being used on either end... you get what you pay for :-)
  • Zip the file, then attach the zipped version.
  • You can also politely suggest to the recipients that they save the attachment to disk then open the saved copy with the correct application rather than trying to open the file directly from the message.

john w. vinson say:

The DateDiff function will work, but you need to tweak it a bit. The function itself returns a Long Integer count of the number of time intervals (seconds, minutes, hours, up to years) between the two arguments. To get hours and minutes I'd calculate the number of minutes using DateDiff and then formatting that result as hours and minutes:

ProcessTime: DateDiff("h", [StartTime], [EndTime]) & ":" & Format(DateDiff("n", [StartTime], [EndTime]) MOD 60, "00")

The MOD operator ("modulus") gives the remainder after a division, e.g 64 MOD 60 is 4.

un-check the ISO 1905-1 compliant box.

If Internet Explorer 11 determines that the current video card or video driver doesn't support GPU hardware acceleration, it'll use Software Rendering mode, and that can also cause scroll issue.

Let’s try these methods below to identify the actual cause of the issue and troubleshoot accordingly.  

Method 1:

Uncheck the smooth scrolling option in Internet Explorer settings and check if that helps.

Follow the steps below to do so:

  1. Open Internet Explorer.
  2. Click on Tools and select Internet Options
  3. Select the Advanced tab and uncheck the option which reads "Use Smooth Scrolling".
  4. Restart Internet Explorer and check.

Proceed to the next step if the problem persists.
