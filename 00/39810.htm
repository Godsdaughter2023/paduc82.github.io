HP OfficeJet 4650, Now my wireless does not work. Please help - Microsoft Community

juanitarandleman say:

HP OfficeJet 4650, Now my wireless does not work. Please help

I have a new ip address (hpeprint.com) 
and now my wireless for my laptop does not work. Please help!

todd_s_88 say:

Reading your issue it sounds like the situation is you setup a new printer and after that your wireless is not working on the laptop. We may need a bit more information but going to try and help with some basic things you can check.

The first thing to do is open a command prompt and run the command "ipconfig /all". When it gives you the information you'll want to look for something like "Wireless LAN adapter Wi-Fi"

The fields in red are the main points of interest. In particular whether or not your laptop uses a static or dynamic (DHCP) address could be a big aspect.

If you see that DHCP Enabled - Yes, then an easy thing you can try is:

  1. Make sure you are showing as connected to your wireless network
  2. Run the command "ipconfig /release" and after it finishes that command run "ipconfig /renew"
  3. If you see that DHCP is enabled but you see an IP address of 169.254.X.X this is called an APIPA address, and means your computer tried to get a DHCP address from the router but did not get a response and self-configured itself. APIPA is essentially useless outside of indicating that your computer isn't getting a DHCP address, but that by itself can be a good troubleshooting point. If it isn't getting a DHCP address it may just mean the computer isn't on the wireless network anymore

That will have your computer request a new dynamic address from the DHCP server which is more than likely your home router. If this works it could mean that your printer and laptop had the same IP address

If you see that DHCP is not enabled it means you are on a static address, and depending what changes were required with the printer could be causing the issue. From here just as a general test in the command prompt run the command "ping <ip address of the Default Gateway>", so in my picture I'd try "ping 172.16.1.1". If you get time outs it means your laptop isn't reaching the local router.

From here you would have two general options, and in each case I would write-down your current IP address information in case you need to go back:

  1. Before doing any of these steps make sure you are showing as connected to your Wifi network, because the point of the next steps is to change from a static IP address to a dynamic one from your router
  2. Go to Control Panel->Network and Sharing Center->Change Adapter Settings. Find your Wi-Fi adapter and Right Click->Properties
  3. Under the section that says "This connection uses the following items:" find "Internet Protocol Version 4 (TCP/IPv4)" and double click it
  4. Again the assumption here is you have a static address, and what you can do from here is change "Use the following IP addresses" and "Use the following DNS server addresses" to "Obtain an IP address automatically" and "Obtain DNS server address automatically"
  5. See if you can connect

If going the DHCP route does not work and it needs to be static what you can do is:

  1. Again make sure you are connected to your wifi
  2. If you have another computer on your wifi that is working go to that computer and do an "ipconfig" command to see what IP address is on that computer and jot it down. If you don't have another computer but the printer is working, check the printers documentation on how to find the IP address of the printer
  3. Repeat the same steps as earlier to get to the TCP/IPv4 options, but you are going to set the IP address to something specific based on what you found earlier

For what IP address you'd need to set, lets say for example on a working computer/printer you got that it had:

-An IP address of 192.168.1.25

-A subnet mask of 255.255.255.0 (99% of home routers will be like this)

You could statically set your laptops IP address to any address in the range of 192.168.1.1 - 192.168.1.254 with the caveat that:

  • The IP address does not overlap with any other IP addresses on the network, especially that of the router, which 99% of the time will be the .1 on the subnet. You can also verify the router address by whatever comes up as Default Gateway on your other devices

Those are some basic steps you can try. If that doesn't work or your setup is different we'll need a bit more information to go from.
