aseem nayar say:

Hello Everyone,

I believe the problem you are running into is outlined in this KB article: 

http://support.microsoft.com/kb/2123563

 

The issue is you are accessing a non-ssl site using Basic Auth which is a high security risk due to the password being sent over the wire in clear text.  You should implement a SSL site if you are using Basic Auth.  If you can not create a SSL enabled site then the KB article outlines the workaround of creating the regkey and allowing for Basic Auth over non-ssl.

 

For Windows7 you would create two BasicAuthLevel regkeys one located in HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\WebClient\Parameters and one located in HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Common\Internet.  This key will have a value of 2 which corresponds to Basic Authentication enabled for SSL and for-Non SSL connections.

Hope this helps,

-Aseem Nayar

rob koch say:

Though it's not prudent to use the word "never", it seems highly unlikely for the forseeable future.

The primary purpose behind the creation of Microsoft Security Essentials (Morro) was to provide the Windows consumer market with a complete anti-malware product that was as simple and resource efficient as possible.  This included providing support for both the new netbook and also less resource capable systems being sold internationally.

Since the above was the product focus, neither business nor server OS concerns were planned to be addressed by the product initially.  So though there is an obvious hole between MSE and the ForeFront enterprise products, at least initially there was no plan for MSE to fill it.

Any attempt to turn MSE into a product supporting servers would potentially threaten the ForeFront series of products, since that's really the core of what those products are designed to support.  Since the devlopment work for these has already been done, it would seem foolish to me to create a situation where Microsoft is competing with itself using a free product.  This is all just my own speculation, but the fact that OneCare for Servers has also been killed along with the consumer version seems to bear this out.

Again, there's always change in both the security and software industry in general, but until malware security moves to the level that it's assumed to be included as part of every software based electronic product, I wouldn't expect it.

Rob

chip pearson say:

I think the problem is with your the line of code

 

    If ActiveCell.Offset(0, 10).Select <> "" Then        

 

This will ALWAYS return True. Select always returns True. You then compare than to not equal an empty string. True will never equal an empty string, so the entire conditional will always evaluate to True. Therefore, the Else block containing Exit Do is never executed.

 

In that same line of code, your comment reads "'Column J contains customer...". But ActiveCell.Offset(0,10) is column K, not column J, so either your comment is wrong or your code is wrong.

 

You will find it much easier and efficient not to use Select and Activate, and reference ranges directly. It is (almost) never necessary to Select or Active anything in VBA. Direct variable reference is much simpler to write and to read and debug.

Word VBA - Tools>References

In a Word template, I'm trying to set a VBA reference to another template. When I set the VBA reference (using Tools>References) to a mapped drive, it resolves it to the UNC location, rather than the mapped drive.  Is there a way around this?   I need it to be the drive.  Thanks...

andreas killer say:

In this case you have to set the reference with your macro and your macro must have access to the visual basic project.

Search all references for your template, if found remove it. Then add a references to the file from the mapped drive.

Andreas.

Option Explicit

Sub Test()
  Const TemplatePath = "G:\"
  Const TemplateName = "Test.doc"
  Dim Ref As Object 'Reference
  With Application.VBE.ActiveVBProject
    For Each Ref In .References
      If InStr(1, Ref.FullPath, TemplateName, vbTextCompare) > 0 Then
        'reference found, remove it
        .References.Remove Ref
        Exit For
      End If
    Next
    'Add it from mapped drive
    Set Ref = .References.AddFromFile(TemplatePath & TemplateName)
  End With
End Sub
