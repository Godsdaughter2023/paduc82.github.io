john mcghie say:

That's either a mis-set Outline Level on one of the styles, or it is a cross-reference that has gone wrong.

My first guess is Outline Level.

Go to the Normal style and Format>Style>Modify>Format>Paragraph (we need to reveal the setting).

Now check the Indents and Spacing tab in the Outline Level box.  It should be "Body Text".  If it is not, re-set it to Body Text.

OK your way out and re-generate the TOC.  That should fix it.

If it is a cross-reference that has gone bad, you need to check each cross-reference until you find which one it is.

Use Option + F9 until you reveal the field codes.  If there are cross-references in there you will see codes like { REF _Ref272953068 \r \h }.  If you then hit Option + F9 again, the GOOD cross references will immediately collapse to just the one line of text they are supposed to contain.

The BAD one will suddenly explode to almost as many pages as the whole document.  The explanation is that its end tag has migrated to the end of the document (or close to it) as a result of a deletion or move of some text.

When you find the bad one, simply delete it.  Then regenerate the TOC.  The problem should be solved (although there may be more than one bad field in there...)

jay freedman say:
If the "fillable form" is protected for filling in forms (using the Legacy Tools form fields), the only way to change the watermark is to unprotect the form first. Then you can use the built-in Watermarks gallery on the Page Layout tab and reprotect the form.

If you put watermarks in a gallery such as Custom1 (because the Watermarks gallery isn't one of the ones offered in the properties dialog of a building blocks content control), and you then insert a building blocks content control in the document or its header, you can't click on it while the form is protected.

Sub mcrLastNonError()

mike h.. say:

Hi,

Looking at you snippet of code it looks like a prime candidate for being put into a loop instead of; what I suspect, are a lot of individual copy paste statements and if; as seems, your worksheets are sequentially numbered then it can be done. However if it works for you then it's fine.

I altered this for sheet12 to include a test for B6 & C6

If ActWkbk.Sheets("12").Range("b6") <> ActWkbk.Sheets("12").Range("c6") Then
   Set SourceRange = ActWkbk.Sheets("12").Range("b142:b142")
   Set DestRange = DestWB.Worksheets("12").Range("b142:b142")
  
   SourceRange.Copy Destination:=DestRange
  
   Set SourceRange = ActWkbk.Sheets("12").Range("f142:f142")
   Set DestRange = DestWB.Worksheets("12").Range("f142:f142")
  
   SourceRange.Copy

    DestRange.PasteSpecial xlPasteValues, , False, False
    Application.CutCopyMode = False
End If
