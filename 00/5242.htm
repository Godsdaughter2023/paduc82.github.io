suzanne s. barnhill say:

A StyleRef will always reference the most recent instance of the referenced style, whether it is on the same or a previous page. Even if you had a header on the first page (which would show the first-occurring revision number and date, 03), subsequent pages would still reference the last instance (01).

In order to have the "Final" (03) revision number in your page headers,  you'll need to reverse the order of the table, which would seem more logical to me, anyway. In order to show both the revision number and the revision date, you will need to use different styles for the two items and have two StyleRef fields.

If standards don't permit reversing the order of the table, then apply the referenced styles to only the most recent revision number and date. Since this is a template, however, it seems like the natural action for a user creating/revising a document based on this template would be to add a row to the table for each revision. The table would then be in chronological order, the styles would persist throughout the table, and the StyleRef field would work as expected.

Dear friends!

andreas killer say:

Hi Peter,

that looks like that you access an ActiveX CommandButton through the WorkSheet.OLEObjects collection... at first: when you ask such questions next time, please post the whole macro, not only a part. If the macro is very long, shorten it if possible. That makes it easy for us to understand what you are doing and we can give you more precisely help.

Okay, where should I start... the bad news first: The CommandButton has no "MouseOver" property that can be set to change the caption. What you want can be done, with some tricks, but I guess the solution is far beyond your skills. And maybe you don't like it. So let us start at a lower level and the (as you said) more important part how to navigate through this jungle.

I guess that your macro looks like this and it is placed in a regular module:

Sub Test()
  With Worksheets("WhatEver").OLEObjects
    .Item("CommandButton1").Object.BackColor = 4210943
  End With
End Sub

Almost anything what you see on screen is an object, a cell is an object (Range), a sheet (WorkSheet), the file (WorkBook) and Excel too (Application).

These objects are predefined by Excel and allocated/created automatically, and they are ready to use, please type in this sub into your VBA editor:

Sub Test2()
  Debug.Print Application.Name
End Sub

When you type the dot a drop down should appear that lists all possible properties of that object. If the drop down doesn't appear, click Extras\Options and in the Editor tab check all boxes, except the 2 at the top (that makes your live easier).

Now make a new file, add a CommandButton to a sheet and go into the code module of the sheet. In there type this

Sub Test3()
  Me.OLEObjects.Item
End Sub

Me is the (class) object where the code is placed in, in this case the worksheet. When you type "Me." you see the drop down and you can choose OLEObjects, but when you type the dot after OLEObjects you do not see a drop down. Why?

The reason is that the OLEobjects property is "untyped" (type of Object), but "Me" is type of WorkSheet. Only typed objects shows the drop down. The reason why some objects are untyped is that they are able to return different (other) objects and so there properties are different, depending on the object that are referred.

Similar objects are the collections Sheets and Worksheets, Sheets is untyped, because this collection can contain Charts (Chartsheets) and Worksheets, but the Worksheets collection contain only work sheets.

Alright, I think I should come to an end before you lose patience. But we still have to go a little further, please stay with me. :-)

The CommandButton in the sheet is an object, that means when you place anything in a sheet it becomes a property of Me, try this:

Sub Test4()
  Me.CommandButton1.BackColor = 4210943
End Sub

At this point you can see all properties of the command button in the drop down, but before you search that list, press F2 and the object catalog becomes visible.

Write "CommandButton" into the text box left of the binoculars and click them (or press ENTER).

The first item is the result list should be MSForms.CommandButton and you can see all properties in the right frame below.

When you select the 2nd one VBAProject.Sheet1.CommandButton1 you can see all properties of that sheet. Scroll down and you can find OLEObjects, scroll more down and you can see also Test3 and Test4 and they are bold, because this properties have code in the class module.

When you select e.g. Test3 you can see below how it looks like for the compiler, this sub is "Public" and has no arguments.

Okay, back to the command button properties:

As you see there are items that sounds "Mouse..." and some of them have a lightning icon in front. That means this are not properties, this items are events. Before we start with the events, select the MousePointer property and press F1. Looks promising, right?

But that is all what is possible automatically when you hover the mouse over the button. I'm sure you have seen the MouseMove event, which sounds promising, how can we get it to work?

Press F7 and you are back in the code module of the sheet. Over the editor you see 2 combo boxes, open the left one and select "CommandButton1". As you see this sub is created automatically in the editor:

Private Sub CommandButton1_Click()

End Sub

and the right combo box now shows "Click". Open it and select "MouseMove". The created event routine is called every time when the mouse moves over the button.

Add this lines

  With Me.CommandButton1
    .Enabled = Not .Enabled
  End With

switch back to Excel and hover the mouse over the button. As you see the caption flickers when you move the mouse. But when you do not move the mouse nothing happens.

And that is the reason why I said at the beginning "maybe you don't like the solution".

You can disable the button and change the caption when you move the mouse on the button, but I guess you want to restore the state when the mouse leaves the button, right?

For that we need a Worksheet.MouseMove event, but unfortunately such thing doesn't exist in Excel. Here are 2 ideas at which point we can restore the button:

a) We can use the Worksheet_SelectionChange event, means if the user select a (different) cell in the sheet we can restore the button.

b) We can use a timer and restore the button after a period of time when the CommandButton_MouseMove is fired the last time. To make that safe we have also to save the mouse position and compare if the mouse has really moved, otherwise the user can move the mouse over the button and wait for a while till the button is restored.

IMHO you should now be able to implement solution a) by yourself, the other needs a lot of API code, it is possible, but I would not recommend it.

Andreas.

jeeped say:

I have seen this behaviour before. It happens frequently when importing.CSV's, but can also occur from copying data from a web page or other circumstances as well. The culprit is a leading space  or a leading/trailing non-breaking space character, AKA ALT+0160 or &nbsp; in HTML terms. The latter is essentially a forced space character that cannot be trimmed by conventional means.

For example, if I type these values in A2:A7,

9/16/2010         < just the date with no embellishments >
'9/16/2010        < the date with a leading tick or single quote CHAR(39) >
9/16/2010         < the date with a normal leading space CHAR(32) >
 9/16/2010        < the date with a trailing non-breaking space CHAR(160) >
9/16/2010         < the date with a leading non-breaking space CHAR(160) >
 9/16/2010        < the date with leading and trailing non-breaking spaces CHAR(160) >

... and evaluate the results with =TYPE(), =LEN() and =VALUE(), I get something like this,

... then I applied the Data tab's Text to Columns command and got this,

... and I did a Search and Replace on column A, Find what: <space>, Replace With: <nothing> and I got these results,

... and finally I did another Search and Replace on column A, Find what: ALT+0160, Replace With: <nothing> and I got these results,

... which is what I expect you are looking for. So you may need a combination of techniques that are at your disposal in order to get your information the way you want it. There are very few programs that export CSV data with non-breaking spaces by default, but those are common on web page copied text.

BTW, the non-breaking space or CHAR(160) is produced by holding the left ALT key down and typing 0160 on the numpad. Releasing the ALT will produce the non-breaking space. You cannot just type ALT+0160.
