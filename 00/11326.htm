bdjbdj say:

I have run into such issue multiple times. This is what I did about it.

First, I needed to determine if 1N sync process is alive or not. This is how ...

1) Go to file / info / grab the notebook url and paste it in browser. This will take you to 1NOnline.

2) At top, Click on 'documents' visible in the path 'onedrive/documents'. This will take you to folder where your notebooks are located.

3) Click on the icon on top right corner to change view so that you can see notebook sizes.

4) Start sync on the desktop client and monitor the notebook size. You should see size increase.

5) If you do not see size increasing, insert a new a page in this notebook and try resyncing. Again, you should size go up.

6) If size is up, wait till it completes. This error message should eventually be gone. In some cases, I have seen it to take few days till is disappears on its own.

7) If size is not changing and you do not see your changes in the online version, then sync is broken.

Second, determine why sync is not working. The most common cause I have seen is large page size. In multiple instances, there was a single page that was just too large causing 1N to choke. This is how to go about finding it.

1) Usually the sync dialogue will till you what page it is choking on.

2) If listed, break it down to multiple pages. This worked for me on multiple occasions.

3) If not listed, go through the  new pages you added since last known good sync. Try to find out large ones and break them down.

I have not found a way to tell a page size in 1N.

Hope this helps.

    Sub LoopThroughDirectory()
    Dim x As Long
    x = 1
    Application.DisplayAlerts = False
    'Change this to your directory
    MyPath = "C:\"
    activefile = Dir(MyPath)
    'ActiveFile = Dir(MyPath & "*.xls")
    Do While activefile <> ""
       activefile = Dir()
       Cells(x, 1) = activefile
       x = x + 1
    Loop
    Application.DisplayAlerts = True
    End Sub

    Like [Enter First Letter] & "*"

    hansv mvp say:

    Alternatively, create a calculated column in your query:

    FirstLetter: Left([LastName],1)

    where LastName is the name of the relevant field, and use this in the Criteria line of this column:

    [Enter First Letter]

    karl donaubauer say:

    jtparks wrote:

    I need to create a parameter query based on the first letter of a
    persons last name, sounds easy, but for some reason I cannot get it
    to work. Any advice?

    You could simply use a criteria in the last name column of the query:

    Like [Last name begins with] & "*"

    This considers an index on the last name field.

    Of course you could also create a column:

    Left([LastNameField],1)

    with criteria:

    [Last name begins with]

    But this has to calculate all rows and does not consider indexes.
