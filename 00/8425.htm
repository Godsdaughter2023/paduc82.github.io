LAT Agency wrote:

UNION in combination with PIVOT are not supported in Jet. Reason is
probably because UNION queries require identical output columns over all
queries within a UNION construct. Pivot queries, however, have the nature
of returning flexible numbers of columns, dependent on underlying data.

I think you just changed the cell format for "Mth Date" column as "MMM" from "Format Cells > Number > Custom". That is why while you apply the Slicer on "Mth Date" it shows a different date.

scott mcdaniel say:

That's the way Continuous forms work with the Unbound stuff. You might be able to change to a Bound Object Frame, but that means storing the objects in Access which is a bad idea and can cause significant bloat (although newer versions of Access are much better at managing this than older versions).

If you can use a Report you'd be better off. If the user doesn't need to interact with the data, and just needs to view it, then a Report is the correct object to use. If the user does need to interact with the data, then you may be stuck with a standard Form object showing a single record.

Using a different chart tool won't have any effect - you'll just see that new chart repeated over and over.
