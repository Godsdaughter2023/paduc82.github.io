  1. Go into Windows File Explorer, on the View Tab, make sure 'File Name Extensions' is checked, close Windows File Explorer.
  2. On your Desktop, right click and select 'New' - 'Text Document'
  3. Copy & Paste this text into it: C:\Windows\System32\DisplaySwitch.exe /clone
  4. Save the file, give it a meaningful name, e.g. Projector Settings
  5. Change the file extension from .txt to .bat
  6. Go back into Windows File Explorer, on the View Tab, uncheck 'File Name Extensions', close Windows File Explorer.

Now when you want to project to the digital projector, double click the file . . .

greg_id say:

In April after many trials and errors the process finally stopped but I couldn't identify which of the trials resolved the issue. Few days ago the problem reappeared this time I have the answer what helped:

1. Stop Windows Search service

2. Delete and Rebuild the index while Windows Search is stopped

After indexing is complete the Windows Search service will execute correctly - in my case it now stops after 10 minutes (still annoying as it is doing it quite often but at least it doesn't last a week!).

My interpretation of what was happening is that there was a conflict between indexer process and search host filter - running indexing process triggered search filter host and search filter host blocked indexing from completing causing feedback loop.
