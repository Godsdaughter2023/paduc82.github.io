Joining Tables

ken sheridan say: 'Long Text' is the current term for what used to be called memo fields.  As the error message says, you cannot join tables on columns of this data type.  The nearest you could get would be to create another column of 'short text' data type in each table and populate these with the first 255 characters from the current columns by means of simple UPDATE queries.  You should then be able to join the tables on the new columns.

john w. vinson say:

As Ken says... no can do. A Long Text field could be gigabytes in size; cannot be indexed; and would be hideously inefficient as a joining field.

What is the real life problem you're trying to solve? I'm sure there must be a better way!

albert d. kallal say:

It not 100% clear why or what that query you were running does? Normally you do not need to build a query in code, but simply build up a filter and pass that to the report you are about to open.

 

More important web based solutions does not have a make take query anyway.

 

So, about the best here is to setup a filter, and execute an open report.

 

In other words, it much sounds like you have a form, the user enters some values they want to filter for the report, and then the report is launched based on that filter.

 

The first thing is to remember that web query cannot have forms!SomeCollumName expressions in them (the reason for this is the query running on the server, but the form is running in a web browser on your iPad.

 

The general approach here is thus:

 

Base the report on a web query you build with the query builder

(remember no forms!FormName!Collum name expressions allowed in the web query builder)

 

Then build a form that allows user to select values,

 

Then use code to setup a filter and then open the report.

 

Launch the report based on this filter.

 

The example code for a form that prompts user on City and State and then launches a report would look like:

 

SetTempVar (sCity,[cboCity] & "*")

 

SetTempVar (sState,[cboState] & "*")

 

 

OpenReport

   Report Name   : "Staff Report"

Where Condition: [City] like [TempVars]![sCity] and [State] like [TempVars]![sState]

 

The above is really very much how this is usually done in client based applications also.
