lemp say:

It's quite normal to have multiple svchost.exe instances.  That's because svchost.exe is a "host" to various system services that run in the background.

If you open a Command Prompt window and type

     tasklist /svc  {Enter}

You'll get a display that looks much like the list of processes in Task Manager but opposite each svchost.exe, you'll see the list of services that it's hosting.

The anomalous entry in your list of processes is userint.exe.  The real userinit.exe is part of the Windows boot up sequence, but it should not continue to be present for more than a few seconds after startup.  I don't know if your userinit is still there because it's malware (pretty likely, given the recent history of your computer) or if it's stuck looking for the apparently missing explorer.exe.

You have two options.  The option most certain of a good outcome is to do a complete reinstall of Windows.  This requires that you have some method of reinstalling Windows as well as any needed drivers for your hardware, the original install media for all of your applications, and, of course, a good backup of your data.

If you want to spend some more time attempting to get your system working properly (with no guarantee that you'll succeed), first check userinit.exe.  This file should be located in C:\Windows\system32.  It should show as version 5.1.2600.5512, it should be 26,112 bytes in size, and it should show as being authored by Microsoft.

If you find a userinit.exe somewhere else (other than in some servicepack uninstall file or the prefetch directory), chances are it's bad.  Confirm by using Start > Run > msconfig (or AutoRuns) to see if userinit.exe is in your startup list and if so, what directory it's in.  Normally, userinit is NOT shown in the msconfig Startup tab or in AutoRuns. 

The next thing you should do is go to this site and follow MVP Malke's directions for removing malware.  Don't skip the preliminary work, even if you think you don't need to do it because you've just "cleaned" your machine.  To be candid, your best choice would probably be to skip down that page to "Specialty sites for guided help" -- or just take your computer to a good local professional.  In the latter case, don't go to BigComputerStore or a GeekSquad type of place.

   4. Restart Windows Media Player.
