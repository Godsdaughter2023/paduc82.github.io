You can get the "Upgrade to Outlook.com" option to appear in Hotmail by turning off "Co-branding" in Windows Live Admin Center as follows:

  1. Log in to http://domains.live.com and select the domain you want to manage
  2. In the left navigation menu, select "Co-branding"
  3. Click the "Un-publish from the web" button to turn off the co-branding feature

  1. Log in to http://mail.live.com with your custom domain email address
  2. From your inbox, click the "Options" menu in the top right corner and select "Upgrade to Outlook.com" from the drop-down list
  3. Follow the prompts to enable the new Outlook interface

  1. Log in to http://www.outlook.com with your custom domain email address
  2. From your inbox, click the options "cog" icon in the top right corner, and select "More mail settings" from the drop-down list
  3. Under the "Managing your account" heading on the "Options" page, click on "Create a Outlook alias"
  4. Follow the prompts to setup your new @outlook.com alias

richardpbartlett say:

rdesktop 1.6 is my current recommendation for Win2k RDP access from Windows 10 desktops (and any ServerOS of the same series).

This bundle of executables uses rdesktop version 1.6.0 ported to Win32. This has the additional complication that it needs an X11 server running locally and an environment variable set prior to launching rdesktop.exe.

Courtesy of this site:

http://www.soren.schimkat.dk/Blog/?p=69

downloads:

http://www.soren.schimkat.dk/Downloads/Rdesktop-1.6.0-Win32.zip
http://sourceforge.net/projects/xming/

The Rdesktop ZIP needs extracting to a location that suits you.

The Xming download needs to be run to install the software for general consumption, but it won't start at runtime

XLaunch is a tool for creating an xlaunch suffix file. I chose alt-F4-server.xlaunch as my configuration file and exported it to the same folder that I'd extracted Rdesktop into. If you do the obvious and run Xming, then bewarned that you'll get a root desktop. Where using XLaunch for the config file, I recommend setting the 'Additional Parameters for Xming' to
-rootless

Then you need a suitable batch file to launch (if you hate living in a command line environment):

@*************

@    alt-F4-to-close-<servername>-RDP.bat

@*************

@

@echo off

REM

start .\alt-F4-server.xlaunch

REM

REM DO NOT CONTINUE BEYOND THE LOOP BELOW UNTIL X11-SERVER IS RUNNING

:SERVER

tasklist /FI "IMAGENAME eq Xming.exe" 2>NUL | find /I /N "Xming.exe">NUL

if %ERRORLEVEL% neq 0 goto SERVER

REM

REM DO NOT PASS UNTIL SCRIPT HAS WAITED A SUFFICIENT LENGTH OF TIME = PORT ACTIVE

:PORT

netstat -o -n -a | findstr 0.0.0.0:6000

if %ERRORLEVEL% equ 0 (@echo "port is now available... moving on...") ELSE goto PORT

set DISPLAY=127.0.0.1:0
.\rdesktop -a8 -P -g1024x655 -4 <ip.add.re.ss>

Where you see ip.add.re.ss then you should put the machine-name or IP address (whichever works in your networking environment).

Finally, remember (even though I've quite overtly stated it above) to use alt-F4 to close if you've set the X11 preferences to be borderless.

One major failing with rdesktop as a Win32 based 'X' Windows app is that the clipboard integration is inherently poor due to the unwieldy nature of the different layers of software that integrate the software/API/GUI/haptic elements together. To replace Clipboard, I'd recommend dragging documents and files back and forth using WinSCP(client) and SolarWinds-SCP(server) on either/both ends of the machines in question.

A registry edit that may help:

32-bit browser applications may not work as expected in Internet Explorer 10

https://support.microsoft.com/en-us/help/2716529/32-bit-browser-applications-may-not-work-as-expected-in-internet-explorer-10

The FixIt in that article is no longer available. Here are more detailed instructions to make the registry edit.

Work slowly and carefully and create the registry backup that I mention.

Go to Start> Run (or Windows key + r) type in regedit and hit Enter. 

In the left pane of the regedit window, select Computer, right-click it and choose Export. Save that REG file to the desktop. That's your backup. By double-clicking or right-clicking> Merging that REG file you can restore the registry and undo changes, if necessary.

You navigate the registry just like you would drill down through a folder structure to get to a specific file (key.) Just click the + to open the next level.

Start> Run> type in REGEDIT and hit Enter.

By clicking in the [+} drill down to

HKEY_CURRENT_USER

  Software

     Microsoft

       Internet Explorer

         Main

With Main selected in the left pane, right-click in the right pane and choose New> DWORD (32-bit)

Name the new DWORD Value TabProcGrowth then right-click it and set the value to 1.

If TabProcGrowth already exists (you didn't have to create it) just change the value from 0 to 1.
