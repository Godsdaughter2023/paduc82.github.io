allen z. msft support say:

Hi Zoubeir,

There may not be an actual problem, but I’m afraid your requirement cannot be perfectly met.

If you choose to do a cutover migration, the migration batch will automatically create new online mailboxes. And the migrated data will be imported into those new mailboxes, instead of the old ones you have created from the cloud side. We cannot force it to put the data into the mailboxes you have already created.

And if you choose to do an IMAP migration, the situation I mentioned above will not happen. The “match” will be done by creating a CSV file. However, only mail data can be migrated through an IMAP migration, while calendar items and contacts cannot.

Additionally, I also need to let you know that we may not be able to provide technical support if you choose a cutover migration. This is because you’re using an SBS 2008 Exchange on-premises server, while the cutover migration solution is specifically for standalone Exchange on-premises servers. Thanks for your understanding.

Sumproduct to Act as Vlookup

Scenario:

I’m working in a sheet named ‘Summary’; I have 2 pretty small Pivot Tables on this sheet.  The data actually looks like Pivot Table, after Copy > Paste-Special > Values  -- basically this is hard coded.  In this ‘Summary’ sheet, I’ve got account codes in Column B and account codes in Column F.   -- basically this is hard coded.  In this ‘Summary’ sheet, I’ve got account codes in Column B and account codes in Column F.  I also have a mapping table on a sheet named Matrix; this is how I know the relationships.  Most relationships are 1:1, so a vlookup would give me the matches, except that I have a 1:many mapping setup, and the vlookup returns only the first match. 

 

Question:

How can I look for an account code in column B, and find the matching account code in ColumnF, like this:

Column B = Derivatives

Column F = 123

Column B = Derivatives

Column F = 822

 

Based on the ability to find ‘Derivatives’, I need to sum the values in Column G that are in the same row as 123 & 822.

ryguy72 say:

Ah!  I got it!  I just had to use 2 helper columns.  This is the solution that worked.

=SUMPRODUCT(--($A$3:$A$1000=AB6),--($B$3:$B$1000=AC6),$C$3:$C$1000) - SUMPRODUCT(--($E$3:$E$1000=AD6),--($F$3:$F$1000=AE6),$G$3:$G$1000)-SUMPRODUCT(--($E$3:$E$1000=AF6),--($F$3:$F$1000=AG6),$G$3:$G$1000)

The key was to build over to the right, not down.  I have 6 columns now.  I have Derivatives 1230 in Columns 3 & 4 and I have Derivatives 8229 in Columns 5 & 6.  So, the 1230 and the 8229 are in the SAME ROW.

wolfsden3 say:

I JUST FIGURED IT OUT!!!  I'm pumped!!!

 

I did some more looking on Nokia's site and the settings for EAP-MSCHAPv2 are a bit hidden.  When you're in the edit screen looking for settings at the top right hand corner there is little tab section, you can push the thumb scroller right and I hadn't noticed it until reading the instruction below.  The part I have highlighted in red below is the section you push the thumb scroller right for.

 

 Nokia E71 - Settings - EAP-MSCHAPV2 settings

Select Menu > Tools > Settings > Connection > Access points.

1. Select Options > New access point, or select an access point and Options > Edit.

2. Select WLAN security settings > WPA/WPA2 > EAP > EAP plug-in settings.

3. Select a plug-in, select Options > Edit, and open the EAP-MsChapV2 plug-ins tab.

Select the EAP-MsChapV2 (Microsoft challenge-handshake authentication protocol) plug-in, select Options > Edit, and define the following settings:

User name - Enter a user name to be used for authentication. If you enter your user name here, you do not need to enter it during the authentication session when logging in to a wireless network.

Prompt password - Select Yes to be prompted for a password to authenticate you when you connect to a WLAN. If you select No, you are not asked for your password during login authentication sessions, since it is stored in your device and automatically provided. This may decrease the level of security.

Password - Enter a password to be used for authentication.

 

 

After all that simply using the Windows XP instructions will get you on the PAL2.0 Network at Purdue University.
