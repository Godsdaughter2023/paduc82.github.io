Yes, after spending a couple hours of trial and error, the command line suggestion in this YouTube video was successful in getting my Windows 8.0 Pro with Media Center to Windows 8.1 upgrade to proceed.

I downloaded the zip file named "Sentinel_LDK_Run-time_cmd_line.zip" from http://hasp.com/srm

I then unzipped it and opened its folder in an administrative command prompt and ran the following command:

haspdinst.exe -purge

 

It is interesting that the -help option didn't list "-purge" as an available option, but when I executed the purge command it took just a few seconds and I was able to continue successfully with the Windows 8.1 upgrade.

You may want to run: 

haspdinst.exe -info

before and after the purge just to compare to see if anything changed. 

You will also want to check Services in the services.msc console to make sure the SafeNet Sentinel service has disappeared (make sure you refresh the services list after the purge).

HP printer/fax suddenly has stopped working

lemp say:
You seem to have ignored quite a few suggestions listed in the page linked by John Rubdy.

If I had to guess, I'd guess that your problem caused by having configured your printer to obtain its IP address from the router automatically (i.e. by DHCP) and that the router has assigned an IP address different from the one that was assigned when the printer was first installed.

See "solution 3" at the page linked above.  Even though the HP driver has a box to "always print to this device even if its IP address changes," it is much better practice to assign a static IP address to the printer and then configure the port to point to that address.  Pick an address in the same subnet (first 3 groups of digits are the same) but outside the range assigned by the router.  For example, if the router DHCP server assigns addresses such as 192.168.1.100, 192.168.1.101, 192.168.1.103, etc. assign the printer an address like 192.168.1.50.

Steps to take:

  • Print a network diagnostic page to determine what IP address has been assigned to the printer (see printer User Guide for instructions)
  • Open the Configure Port dialog as described at "solution 3."
  • If the IP address shown in the Configure Port dialog is not the same as the one on the network diagnostic page, that's the problem.
