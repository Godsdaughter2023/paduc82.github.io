john korchok say:

With text, PowerPoint applies hinting that makes font edges look smooth. With bitmap graphics, PowerPoint applies antialiasing to smooth them. The program doesn't handle vector graphics quite as nicely. I didn't see any issues with the circular shape you drew, except that some shapes don't overlap others, so a fine white gap appears. The hero picture is drawn with straight line segments, and that doesn't render as nicely as curves.

You can get a better appearance with the hero picture by converting it to a high-resolution (220 d.p.i. or so) indexed-color (maximum 8 colors) PNG file. Since you only have 2 colors, the file size can still be quite small and it will look smoother in PowerPoint. Please post if you need help on producing an indexed-color bitmap.

If the images were SVG, Scaleable Vector Graphics, chances are you would not see these glitches.

An ACCDR file is most properly used to test how the user experience. It hides all the built-in menus, toolbars, ribbons. The only time I use that extension is when I am testing. There is no benefit to an ACCDR over an ACCDE. As Bill said, using an ACCDR forces Access to load as if it was the runtime. So its most useful in testing. The way to create an ACCDR is simply to change the extension. As such its easy to change it back. But in my environment users don't do that, so its not a concern.
