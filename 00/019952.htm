if you use Shift+Enter to start the new lines, a blank line will appear where the mergefield is empty.

{ IF { MERGEFIELD "fieldthatmightbeempty" } <> "" "{ MERGEFIELD "fieldthatmightbeempty" }¶
" "¶
" }{ MERGEFIELD "NextMergeField "}
 
where you press Enter where the ¶ appear.  You must use Ctrl+F9 to insert the field delimiters { } and Alt+F9 to toggle off their display.

beth melton say:

The option to leave a blank line if the data field is empty is in the old Mail Merge dialog box and it isn’t in the Ribbon by default. To add it to your Quick Access Toolbar:

  1. Right-click your Quick Access Toolbar and then click Customize Quick Access Toolbar.
  2. From the Choose Commands from drop down, select Commands not in the Ribbon.
  3. In the Commands list, select Mail Merge..., and then click Add to add it to your Quick Access Toolbar
  4. In the Merge dialog box, you'll see the option for Print blank lines when data fields are empty. Select the option and perform the merge using the Merge dialog box.

Unfortunately, you'll need to use the newly added Mail Merge command each time you want use the option since it doesn't stick between merges. Or use Doug's suggested method so you can by-pass the dialog box.
