Calendar permissions in an Office 365 Group - Microsoft Community

a. user say:

Calendar permissions in an Office 365 Group

Is it possible to give group members read-only permissions in a 365 group calendar and if so, how?

barry xie msft say:

Hi Jamie,

 

Yes. You can run PowerShell commands to set the read-only permissions. Below are the steps:

1. Connect to Exchange Online PowerShell with an Office 365 admin account.

2. Run this command to set the read-only value:

Set-UnifiedGroup -Identity group@contoso.onmicrosoft.com -CalendarMemberReadOnly

3. Run this command to check the CalendarMemberReadOnly in this group.

Get-UnifiedGroup -Identity group@contoso.onmicrosoft.com -IncludeAllProperties | Format-List

Note: Replace group@contoso.onmicrosoft.com to your Office 365 group’s email address.

 

Thanks,

Barry

steve rindsberg say:
Oddly (for which give thanks) PPT will let a macro launch external files though it won't let the user choose to do so.  I suppose the (half-baked) logic is that by allowing a presentation containing macros to open, you've implicitly cast your fate to the winds, allowed the code to have its way with your computer.  

You can assign this macro as the Action Setting : Run Macro for any shape you like.  It'll launch Windows Media player and play the video you specify full screen (alas, there's a certain amount of bouncing around before it gets to the final "Aha!  THAT full screen!  Is that what you meant, then?" and really does it.  Of course you can leave the sCommandLineSwitch part = "" (blank) and it'll play the video in whatever size it would normally play it.

I can modify this a bit further so that you could, for example, put the name of the file to launch in the text of a rectangle; the macro could pick it up from there ... that way you'd only need one macro per file instead of one per each video.

Edit the bits indicated before using:

Sub LaunchMediaPlayer()

Dim sViewerApp As String
Dim sMediaFile As String
Dim sCommandLineSwitch As String

' Edit the following three lines as needed
' The first is the full path to Windows Media Player
'   (though you could substitute a different player, so long as it can
'    accept the file to play on its command line)
' The second is the full path to the video you want to play
' Finally, there's the command line switch if needed
' See the comment in line if you want to look for other switches
sViewerApp = "C:\Program Files (x86)\Windows Media Player\wmplayer.exe"
sMediaFile = "P:\testfile\media\beepfc.wmv"
sCommandLineSwitch = "/fullscreen"

If Len(Dir$(sViewerApp)) > 0 Then ' player exe exists
    If Len(Dir$(sMediaFile)) > 0 Then   ' media file exists
        ' Media player command line switches:
        ' http://msdn.microsoft.com/en-us/library/windows/desktop/dd562624(v=vs.85).aspx
        Call Shell(sViewerApp & " " & sMediaFile & " " & sCommandLineSwitch, vbMaximizedFocus)
    End If
End If

End Sub
