Private Function Combinations(Arr, ByVal Count As Integer) As Collection
  'Gibt alle Kombinationen von Count Elementen in Arr als Arrays in einer Collection zurück
  'Returns all combinations of Count elements in Arr as arrays in a collection
  Dim Index() As Long
  Dim Result()
  Dim i As Long, j As Integer
  'Create a collection
  Set Combinations = New Collection
  'Return empty collection if Count exceed the bound's of Arr
  If LBound(Arr) + Count - 1 > UBound(Arr) Or Count < 1 Then Exit Function
  'Setup space for index and result array
  ReDim Index(1 To Count) As Long
  ReDim Result(1 To Count)
  'Setup index for first combination
  For i = 1 To Count
    Index(i) = LBound(Arr) + i - 1
  Next
  Do
    'Build combination
    For i = 1 To Count
      Result(i) = Arr(Index(i))
    Next
    'Save it
    Combinations.Add Result
    'Get next index
    i = Count
    j = 0
    Do
      'Max. position for this index reached?
      If Index(i) = UBound(Arr) - Count + i Then
        j = j + 1
        'Move index before up one step in next loop
        i = i - 1
        If i < 1 Then Exit Function
      Else
        'Increment position
        Index(i) = Index(i) + 1
        'Setup next indices
        For j = 1 To j
          i = i + 1
          Index(i) = Index(i - 1) + 1
        Next
        Exit Do
      End If
    Loop
  Loop
End Function

dianeoforegon say: First, rebuilding an Entourage Identity takes lots of free disk space. What you need to do first is identify what you need to keep and what you need to trash.

In each Identity you will find a database file. Check the Date Modified. This will tell you the last time the Identity was opened. I suggest you put the date in front of the name of the Identity like this so they will sort by date.

07-11 4GB Main Identity [Rebuild: xxxxxx]
10-12  5GB Main Identity [Rebuild: xxxx [Rebuild xxxx]

You might also want to add the size of the database to help determine which ones have data. If it's a few MB, it's an empty Identity. Delete the ones that have no database file or have no data.

Delete all Identities that are too old or empty from your drive. If you have an external move everything you can off this drive. See the basics below and read the article about space required.

Once you get your Identity to open, you don't trust this to have no corruption. You need to move all your data to a new Identity.

You can download and run this script on your Entourage Identity.

Export folders as MBOX files (includes subfolders)

http://www.office.mvps.org/downloads/scripts/jolly/Make_MBOX.zip

The script will fail on a folder with corruption. If there is no corruption the script will export all folders. Once you find corrupted folder, rename it with z in front of the name so it sorts at the bottom then run the script again. Keep repeating until you have identified all corrupt folders. Once you have identified corruption, you can subdivide folders and keep running script until you get down to 25 messages. Then you drag individual messages to the Desktop. A corrupt message will crash Entourage. After you clean all folders and reduce the size of the Inbox and Sent folders, do the File > Export as .rge file and you should get "Done". You can use this file to import into a new Entourage Identity.

This is a long process so take your time and do it right. I do this for clients and it can take quite some time, but with enough free disk space you'll be able to recover your data and create a new working Identity.

Rebuild Basics:

  • Do Not Rebuild Identities with Exchange and IMAP accounts. Delete the accounts (this deletes the cache files in Entourage). Do the rebuild then add back accounts. Better yet start over in a new Identity.
  • Make sure you have plenty of free space on your drive. More info for Entourage rebuild IMPORTANT READ THIS!
  • Rebuilding does not fix all issues. It can make some problems worse. Even though the Database Utility will make a backup, I advise making a copy in the Finder first.
  • You can delete the rebuild version after you are satisfied with the results. This copy will be named like this: Main Identity [Backed up 1-21-2012 13.35]  

Tip: If your Identity will open, before you rebuild, check "On My Computer" Inbox and Sent folders to be sure they are under the 2GB limit. Generally this is less than 10,000 messages but with large attachments could be less. Make subfolders e.g. Inbox 2011, Inbox 2010, Sent 2011, Sent 2010 etc to help reduce the number of messages in a folder. (Check for any large archive folders you might have in addition to Inbox and Sent).

shashank gandhi say:

Dear User,

To autolink shapes to new rows in your spreadsheet you need to do the following:

  1. Link your spreadsheet to the Visio diagram using Quick Import (only in Visio 2016) or Link Data option under Data Tab in Ribbon --> Advance Data Linking section
  2. If your spreadsheet has new rows and you hit 'Refresh All' in the Data Tab then Auto-Link will link the new row to the shape on Visio Page where match condition is found as specified in Link Data earlier or it auto-identifies if it was linked using Quick Import
    • For this step to happen you have to create Visio shapes on the page(s) where you want the new rows to be linked
    • Match condition should work (e.g. Some column value matching Shape Text)
    • After clicking 'Refresh All' it will auto-link only on the page which is in selection, to do it across all pages you will have to click refresh page by page
