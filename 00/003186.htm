George

jay freedman say:

Press Ctrl+Alt+Shift+S to open the Styles pane. Right-click the renamed Heading 1 and click the Delete item in the menu. Answer Yes to the prompt.

Because built-in heading styles such as Heading 1 cannot be truly deleted, Word will restore the style to its original name and definition. At the same time, every paragraph that had that style will be reformatted as Normal style, so you'll have to go through the document and reapply the Heading 1 style to them.

Alright thanks to NicoCoetzee's post i found the sollution for this too :)

a) Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

b) Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

net stop wuauserv

net stop cryptSvc

net stop bits

net stop msiserver

c) Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

d) Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

e) Type Exit in the Command Prompt to close it

After this commands i didn't received any errors with the Windows update. So one more thanks to NicoCoetzee's post.

andrea jones - all about resources say: Set up your icon sets so they format the other values correctly (including the zeroes as red crosses) and then click on the New Rule button in the Manage Rules window and create a rule using the 'Format only cells that contain' rule, set the value to 'equal to' and 0 and set the format to No Format.  This rule should be at the top of your rules list (use the up arrow to move it up if not), click the tick in the 'Stop if true' box and this should prevent your zero values from being formatted.

If you set up the first document so that each piece of information can be identified (enter unique names in the Title and/or Tag of each content control in its Properties dialog; insert bookmarks to identify data not in content controls), then a macro can be written to make the transfer with a click on a single Quick Access Toolbar button. The macro would have to be specific to the structures of the two documents, saying in effect "get the data in location X and place it in location Y" for each piece that needs to be transferred. tim kimber say:

I did manage to get this working

Create an excel sheet with entries like the below.

 

Item Date Description Country Start Date End Date UID
1 25/12/2014 Christmas Day USA 20141225 20141226 0333D4CA-04F59CE4-05BBB8DF-0773643C-09407554
2 01/01/2015 New Year Day USA 20150101 20150102 049FBABF-0FF53A97-0162AF70-078908A9-010323E6

Use the below VBA Code

' Workbook:     PublicHolidayImportFileCreate
' Author:       Tim Kimber
' Date:         26th November 2014
' Version:      1.0
' Description:  This sheet will create a file of Public Holidays that can
'               be imported into Outlook. See Instructions tab for Use.
'
' Version History
'
'   Date        Version     Who                 Description
'   =====================================================================================
'   26/09/2014  1.0         Tim Kimber          Initial version
'   01/12/2014  1.1         Tim Kimber          Added URL for Bank holiday instructions.
'   =====================================================================================
'

Sub GenerateICalendar()
   
  Dim fso As New FileSystemObject
  Dim fstream As TextStream
  Dim filePath As String
  Dim fileDir As String
  Dim sHostName As String
  Dim sUserName As String

  ' Get Host Name / Get Computer Name
  sHostName = Environ$("computername")

  ' Get Current User Name
  sUserName = Environ$("username")
   
  fileDir = Sheets("Config").Cells(3, 2).Value
  Filename = Sheets("Config").Cells(4, 2).Value
 
  filePath = fileDir & Filename
  Set fstream = fso.CreateTextFile(filePath)
   
  Application.Calculation = xlManual
  Calculate
 
  ' Write Header record for ics import file.
 
  fstream.WriteLine ("BEGIN:   VCALENDAR")
  fstream.WriteLine ("PRODID:-//Tim Kimber - Mizuho International PLC//Outlook 14.0 MIMEDIR//EN")
  fstream.WriteLine ("VERSION:2.0")
  fstream.WriteLine ("METHOD: Publish")
  fstream.WriteLine ("X-MS-OLK-FORCEINSPECTOROPEN:TRUE")
   
  For Tuple = 8 To 999
   
    Item = UCase(Trim(Sheets("Data").Cells(Tuple, 1).Value))
   
      
    ' Test if at END record.  If so close file and exit sub.
   
    If (Item = "END") Then
      fstream.Close
      Exit Sub
    End If
   
    ' Set fields
   
    Random = Sheets("Data").Cells(Tuple, 7).Value
    UID = Random & sHostName & "-" & sUserName
    TStamp = Format(Now, "yyyymmddThhmmssZ")
    HolName = Sheets("Data").Cells(Tuple, 3).Value
    DateStart = Format(Sheets("Data").Cells(Tuple, 5).Value, "yyyymmdd")
    DateEnd = Format(Sheets("Data").Cells(Tuple, 6).Value, "yyyymmdd")
    Location = Sheets("Data").Cells(Tuple, 4).Value
    Reminder = Sheets("Config").Cells(5, 2).Value
    fstream.WriteLine ("BEGIN:VEVENT") ' **
    fstream.WriteLine ("CLASS:PUBLIC") ' **
    fstream.WriteLine ("CREATED:" & TStamp) ' **
    fstream.WriteLine ("DESCRIPTION:" & Location & " Public Holiday - " & HolName & "\nPlease schedule the necessary jobs.\n\nFor more details see: http://global-risk.uk.mizuho-sc.com/gvar2/programme/Shared%20Documents/5.%20Support/00.%20Runbook/EE07_Bank_Holiday_Processing.docx") ' **
    fstream.WriteLine ("DTEND;VALUE=DATE:" & DateEnd)
    fstream.WriteLine ("DTSTAMP:" & TStamp) ' **
    fstream.WriteLine ("DTSTART;VALUE=DATE:" & DateStart) ' **
    fstream.WriteLine ("LAST-MODIFIED:" & TStamp) ' **
    fstream.WriteLine ("LOCATION:Mizuho - " & Location) ' **
    fstream.WriteLine ("PRIORITY:1") ' **
    fstream.WriteLine ("SEQUENCE:0") ' **
    fstream.WriteLine ("SUMMARY;LANGUAGE=en-gb:" & Location & " Public Holiday - " & HolName) ' **
    fstream.WriteLine ("TRANSP:TRANSPARENT") ' **
    fstream.WriteLine ("UID:" & UID) ' **
    fstream.WriteLine ("X-MICROSOFT-CDO-BUSYSTATUS:FREE") ' **
    fstream.WriteLine ("X-MICROSOFT-CDO-IMPORTANCE:2") ' **
    fstream.WriteLine ("X-MICROSOFT-DISALLOW-COUNTER:FALSE") ' **
    fstream.WriteLine ("X-MS-OLK-AUTOFILLLOCATION:FALSE") ' **
    fstream.WriteLine ("X-MS-OLK-CONFTYPE:0") ' **
    fstream.WriteLine ("BEGIN:VALARM") ' **
    fstream.WriteLine ("TRIGGER:" & Reminder) ' **
    fstream.WriteLine ("ACTION:DISPLAY") ' **
    fstream.WriteLine ("DESCRIPTION:Reminder") ' **
    fstream.WriteLine ("END:VALARM") ' **
    fstream.WriteLine ("END:VEVENT") ' **
  Next
  fstream.WriteLine ("END:VCALENDAR")
  fstream.Close
  Application.Calculation = xlAutomatic
End Sub

Then from within outlook from File Menu Select Open then Import and select the ICalendar file.

I use this to import all holidays from multiple regions I support.
