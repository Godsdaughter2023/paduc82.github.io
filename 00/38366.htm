ronald clement say: When you delete files (from any hard drive) they are placed in your recycle bin (unless you pressed shift when pressing delete). The recycle bin on your operating system drive is a 'virtual' folder where all the deleted files are 'stored' for deletion. What actually happens is that the file deleted is marked for deletion, but is still taking up the same space as it did before. Even when you look at that drive from another operating system, the space is being used but the file is not there (unless maybe you have some program that can look at the file allocation table and gather information on the files that are there, much like undelete programs do). So that should answer your question, but also when the file is deleted for good it is not really deleted, but just taken off the file allocation table (so it can still be recovered). a. user say:

cgrmn,
8007045d = "ERROR_IO_DEVICE - The request could not be performed because of an I/O device error."

possibly your hard disk is going bad.  run a full chkdsk as follows:

open "My Computer".

right click drive C:, select "Properties".

click tab "Tools"

click "Check now"

check *both* check-boxes, and click start.

follow instructions to restart windows.

before Windows reloads, it will run a full chkdsk, which might take an hour or more.  take note of any significant errors reported: if it says it has replaced bad clusters in files, those files might now be corrupt.

if there are very many errors, you shoud consider backing up your data files, reformatting, and installing a fresh version of windows.
