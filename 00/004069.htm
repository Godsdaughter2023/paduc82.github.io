joeu2004 say: zakynthos wrote:
Subject:  Setting a VBA timer to display seconds
I've tried subsitituting "seconds" for minutes below but I get a 0 seconds answer (should be about 15 sec on the macro I'm running it with) [...].  How should I amend the code?

Dim d1 As Double
Dim i As Long
d1 = Timer
For i = 1 To 10
     NOGAPS
     UPDATER
     DELETECOLE
     Range("e2").Offset(0, -4).Select
Next i
d1 = Timer - d1
If d1 < 0 Then d1 = d1 + 86400
MsgBox Format(d1, "0.000") & " seconds"

Notes:

1. The adjustment d1=d1+86400 handles the case when the measurement is across midnight.  We assume that the total time is no more than 24 hours (86400 seconds).

2. Timer is more accurate to use than Now.  On a Windows system with a Pentium-compatible CPU, Timer has a resolution of microseconds, and it is updated every 15.625 milliseconds.  In contrast, Now has a resolution of seconds, and it is accurate to the truncated second.

Because Timer is updated only every 15.625 msec, your measured time may be short by as much as nearly 15.625 msec.  But that sounds like it is good enough for your purposes.

3. Timer returns time in seconds since midnight.  So it should be stored into a variable of type Double, not type Date.  (Although in practice, the difference can be seen only in how the variable is formatted by default.)

4. As an aside, there is no need to use Application.Run, and it is probably less efficient to do so.

5. As an aside, there is no need to select Range("e2") separately.
