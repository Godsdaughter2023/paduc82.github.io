scott mcdaniel say:

First - users don't generally like disappearing textboxes. It tends to confuse them, and make them think they've done something wrong, or something is wrong with your application. If you must make a textbox a "writeonce" sort of thing, then just Disable and Lock it after the user enters the data.

As far as persisting this when you "close and reopen access" - that would essentially mean that textbox is only ever written ONE TIME for the life of your application, which seems to be wrong. You can store the user entered value somewhere (the registry, a table, etc), and read that location when the application opens to enable/disable the textbox, but if you do that, then you would never be able to change that value. I cannot think of a scenario where this would be a desirable feature.

Finally, you can toggle the caption of your button based on the state of your data. So if I stored my Textbox data in a table, I could do this:

If Nz(DLookup("MyField", "MyTable"),"") <> "" Then

  Me.SomeButton.Caption = "Submit"

Else

  Me.SomeButton.Caption = "Enter"

End If
