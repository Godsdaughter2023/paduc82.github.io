For the benefit of others, the following is my response to the email that I had received:

When I first opened the document, while the ribbon customization appeared, the macros were disabled.  That would have been because under File>Options>Trust Center>Trust Center Settings>Macro Settings, the radio button for “Disable all macros without notification” was checked

When I changed that to “Disable all macros with notification” and then re-opened the document, a macro warning message was received

If I clicked on Enable Macros, the macros in the document were then enabled.

However, the document should really be saved as a macro enabled (*.dotm) template in the Word Templates folder, which being a “Trusted location” will allow the macros to be enabled, even with the File>Options>Trust Center>Trust Center Settings>Macro Settings, the radio button for “Disable all macros without notification” being checked, when you use File>New and select that template as the basis of the document that you want to create.

tarothfuss say:

I figured out what the problem was (and I'm a little embarrassed to say what it is, since it's one of the most basic troubleshooting questions).  There was some new third-party software installed on the machine and, while I have to look into it some more, it appears that it may have overwritten/updated some critical files (during the installation, it was copying files to the System32 folder).  Once I uninstalled this version and reinstalled the previous version, the problem went away.  So, if anyone else runs into this problem, you may want to check to see if a seemingly unrelated application installation/upgrade could be the problem.
