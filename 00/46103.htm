Make sure that the system is up to date and you have the latest Windows Defender installed. The svchost.exe manages all the running services however, there may be third-party applications which also installed services or may be using some of Windows services. Performing a clean boot will help us identify which application or services uses the svchost.exe and attempts to modify some of the files from one of the folders where controlled folder access is enabled.
