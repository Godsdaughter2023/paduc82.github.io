How do I get quanities from one worksheet to tally in another? - Microsoft Community

tawaunweber say:

How do I get quanities from one worksheet to tally in another?

I have a spreadsheet that lets me know quanitites of items mailed out daily.  I need to get the total quantites for each item to show up in a different spreadsheet by item. 
jllatham say:

Lets say your primary sheet, with all the items and quantities, is on 'Sheet1' and the item ID is in column A, the mailing date is in column B and the quantities are in column C.

On the second sheet, you decide to use row 2 for your first 'summary'.  In column A you would enter a product ID, and in column B you would enter the date (or formula =TODAY()) and in C2 you would enter this formula:

=SUMIFS(Sheet1!C:C,Sheet2!A:A,A2,Sheet1!B:B,B2)

You can fill that formula down the sheet, and enter more product IDs in column A and more dates or =TODAY() formulas in column B and each row will provide the quantity for the matching item/date on each row.

 

jllatham say:

BTW - an error in my previous formula.. the reference to Sheet2 should have been Sheet1!.

With the same assumed setup on the first sheet (ID in A, shipping date in B, and quantity shipped in C) with the added provision that if the item has not shipped yet that the date in column B remains empty.

On the summary sheet you would have a simplified setup, with the item ID in A and a formula in B:

=SUMIFS(Sheet1!C:C,Sheet1!A:A,A2,Sheet1!B:B,">0")

If you want to put anticipated ship dates into column B, but only want a count of ones that have actually shipped, then:

=SUMIFS(Sheet1!C:C,Sheet1!A:A,A2,Sheet1!B:B,">0",Sheet1!B:B,"<=" & Today())

Sub InsertDocTitle()
' Changes window title to include path with filename
    Dim NameArray As Variant
    Dim NameStringL As String
    Dim NameStringR As String
    Dim Count As Long
    Const maxLen = 120   ' set this value to fit your window width
    ' (avoid error if no active window)
    If Windows.Count > 0 Then
        NameStringL = ActiveDocument.FullName
        If Len(NameStringL) > maxLen Then
            ' separate the folder names
            NameArray = Split(NameStringL, "\")
            ' check the folder depth
            Count = UBound(NameArray)
            If Count > 3 Then
                NameStringL = NameArray(0) & "\...\"
                NameStringR = NameArray(Count)
                Count = Count - 1

                ' continue adding folders to the left of the string
                ' until you run out of folders or one won't fit
                Do While (Count > 0) And _
                   (Len(NameStringL) + Len(NameStringR) + _
                    Len(NameArray(Count)) < maxLen)
                    NameStringR = NameArray(Count) & "\" _
                                & NameStringR
                    Count = Count - 1
                Loop

                NameStringL = NameStringL & NameStringR
            End If
        End If

        ' Change the window's caption
        ActiveWindow.Caption = NameStringL
    End If
End Sub

bob jones aka: cybertaz say:
Whether the check box displays is determined by the OS X Finder Preferences settings. In Finder> Preferences there is a check box which determines whether file name extensions are displayed:

  • If that preference is checked the Hide Extension check box does not display in the Save As dialog
  • If the preference is not checked the Hide Extension check box does show up

However, that really should not matter. Word always defaults to saving an existing document in whatever its format happens to be. IOW, when you open a .doc, make changes & Save, those changes are written to that same .doc file... Likewise, .docx files are saved in .docx format, etc. This is the same regardless of whether the extension is displayed or not.

I'm not sure what you mean by "...I can force Word to save all files as default to either .doc or. docx" but from a literal standpoint this isn't true. There is a Word Save preference to specify the default file format for saving new documents you create from scratch. However, it has no effect on existing documents. It does not cause the file format to b changed even when using the Save As command.

The only time there is any option to use a different file format is if you specifically choose Save As rather than Save. Even then, the default is to use the same format as the original version & is displayed in the Format: dropdown menu of the Save As dialog. When you name the file there is no need to type the extension, itself... It's appended automatically determined by the selected format [even if it isn't displayed in the file name field). In fact, if you attempt to use an extension that differs from the displayed Format a warning dialog is triggered.

The pertinent question is: Why do you believe the file format is being changed?"

If your belief is because of reports of difficulty on the recipient's end I suspect that there is an issue with the means of transmitting the file. Perhaps a more complete & detailed description of the issue would provide a better basis for suggestions on how to resolve/avoid it.

a)      Press Windows key +X, select device manager

b)      Locate the audio device and check if there is any exclamation icon over the device

c)       Right click on the device and click on properties

d)      Under General tab, check the device status.

Windows 8 has a built in Audio troubleshooter that can detect and fix errors with playing audio. I would suggest you to run this troubleshooter and check. Refer these steps:

b)      Change the view by option on the top right to Large icons

c)       Click on troubleshooting and click on the view all option on the left panel

d)      Run the “Playing Audio” troubleshooter.

 

Method 3:  Compatibility mode

 

You have mentioned that you have the updated drivers installed. To check if the issue is caused to incompatibility, I would suggest you to download Windows 7 audio driver from the manufacturer’s website and run in compatibility mode and check. Refer these steps:

 

a)      Press Windows key + X, select Device manager

b)      Locate your audio device, right click and select properties

c)       Under driver tab, click on uninstall driver and restart the computer to check if it works with the default Windows drivers.

d)      If it doesn’t work, then right click the Windows 7 driver installation file and select Properties then tap or click the Compatibility tab.

e)      Place a check in the Run this program in compatibility mode.

f)       Tap or click the drop down box and select Windows 7, then tap or click OK.

g)      Try installing the driver and check.
