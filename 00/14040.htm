rickneilson say:

Outlook.pst file locked - thus can not be copied

The file “OutLook.pst” (and only that file – problem does not occur with any other file) is locked, and thus cannot be copied.  Whenever ANY Office application is opened (no need to open a file or do anything – just open the app) “Outlook.pst” becomes locked – IFF, no Office application has been opened within the last minute.   Whenever ANY Office application is opened (no need to open a file or do anything – just open the app) “Outlook.pst” becomes locked – IFF, no Office application has been opened within the last minute.  If any Office application has been opened within the last minute then the file is not locked and can be continually copied until a pause of more than a minute.  When Microsoft create bugs they are beauts!   It took me about 2 weeks and many hundreds of tests to be able to describe this fault accurately.  The fault first appeared on 23 May 2019.  After trying many possible fixes – like those that suggested the fault related to a “damaged” “.pst” file that required a ($163.00) repair - I recognized that at about the same time I had been “given” an Office “upgrade” – so I restored the original version of Office – and, problem fixed.  Hope this is of use to someone.

hal hostetler - mvp-outlook say:
FWIW, any .PST file connected to Outlook will be locked while Outlook is open and will remain that way until Outlook fully closes.  This is normal .PST behavior and has been for ages.  This file lock should only happen when an Outlook process is running, not the rest of Office - Word, Excel, etc.  If the file is locked when, say, only Word is open, My suggestion would be to check Task Manager to see if an Outlook process is also running.

freya [msft] say:

The Excel team has released a change in HTML/XLS file behavior in today’s security update for Excel 2010, 2013, and 2016. Excel will warn about the mismatch between the file extension and HTML content, but will now open the workbook in Protected View as an additional layer of security. If you trust the workbook, you can then enable editing. XLA and XLAM files are not part of this change, they will continue to not open in Excel.

 

How do I get the update? It depends on which version of Officeyou have.

  • Office 365 subscription (Click-to-Run)—install the latest updates

  • Windows Installer version (MSI)—you could wait about a week for the next security update for MSI, and it will update automatically. Or, to get the fix today, use the Download Center to get the KB for your version of Office:

    • Office 2016: https://support.microsoft.com/kb/3115438

    • Office 2013: https://support.microsoft.com/kb/3115455

    • Office 2010: https://support.microsoft.com/kb/3115476

 

Issue recap:

The Excel team has made a change in the behavior of certain file types to increase security. This change came in the security updates KB3115262, KB3170008, and KB3115322. Previously, when you tried to open an HTML, XLA, or XLAM file with an .XLS file extension from an untrusted location, Excel would warn about the mismatch between the file extension and content, but would still open the workbook without Protected View security. After the security updates Excel no longer will open the workbook because these files are not compatible with Protected View and there is no warning or other indication it was not opened.

 

Workarounds for .xla and .xlam files: https://support.microsoft.com/kb/3181507

 

Thank you for your patience and input—community suggestions drove our decision to change this behavior in a way we hope will be much better for your workflows!

idainc say:

 

Went to google and searched a number of variations of this issue and came up with :

 

 

The problem: Every time you plug your phone into your PC to sync it, a stupid add/remove programs box pops up. Just started one day perhaps after an install and now, it won't go away.

Had this annoying problem and searched the net and found some attempts at fixes, tried em all, and they failed. Then, I played around with it for several hours and finally fixed it. We'll go over all of them here for the benefit of those who found this post through google. Try these in order. Each solution below is a complete solution, not a step. If one fails to work for you, try the next one.

(1) Uninstall and reinstall anything that you recentely installed on your device. The true nature of this problem afterall is that something went haywire during an install.

(2) Delete your partnership with the device on your PC in sync center, restart your machine and phone, and reestablish the partnership setting all the preferences the way you had them.

(3) Delete your partnership with the device on your PC in sync center, uninstall mobile center, restart, reinstall mobile device center, restart, reestablish the partnership with your phone setting all the preferences the way you had them.

(4) With your phone unplugged, go into regedit and go to the key HKEY_LOCAL_MACHINE/SOFTWARE/Microsoft/Windows CE Services/AutoStartOnConnect and look for an entry in the right pane that says appmgr.exe . If it's there, remove it. Do you see anything else there? If you do, it's not the cause of THIS problem but, sometimes other stuff that you don't want gets there like neromobileadd which unless you use it shouldn't be there. If nothing else is there, navigate to the top of your registry (back to root) and search for Windows CE Services . The reason is because there will be other instances of it like in KKEY_CURRENT_USER and in other keys. If you find the Windows CE Services key elsewhere, go into StartOnConnect and repeat the same check you did above removing appmgr.exe if you see it. When finished, try syncing your phone again.

(5) This is the one that I had to do. Unplug your phone from your PC and delete it's partnership from sync center. Now, open regedit. Go to the top and this time, just search for Windows CE Services and repeat the following process every time you find it in a folder on the left. (If it's an entry on the right, just search for the next key.) In all instances of Windows CE Services that has an AppMgr key, click it. In each AppMgr key, if there is an Apps key, click it. In every Apps key you find under Windows CE Services/Appmgr, look to the right pane. Other than the entry at the top named Default, you should see app names that look familiar. Delete them. Remember, don't delete the entry called Default, just all the others. When you are finished, you can exit out of regedit. Now, sync up your phone. If you did this correctly, FINALLY, that annoying add/remove programs box will not come up anymore!

 

 

The regedit fix worked.  I had stray programs listed under both keys that were never deleted so every time I synced it was looking for them.

ashish mathur say:
In reply to deleted message

Hi,

 

Try this

 

1. Ensure that the spelling in range A2:A5 of the Overall % sheet match with the spellings in column F of the Attendance sheet

2. In cell B2 of the Overall % sheet, enter this formula and copy down till cell B5

 

=SUMPRODUCT((Attendance!$F$2:$F$221=A2)*(Attendance!$G$2:$G$221="x"))

 

3. In cell D2 of the Overall % sheet, enter this formula and copy down till cell D5

 

=SUMPRODUCT((Attendance!$F$2:$F$221=C2)*(Attendance!$H$2:$H$221="x"))

 

Now when you sort column F data on Attendance sheet, the data on Overall % sheet will remain intact.

jay freedman say:

Just get rid of the extra .Me in that line. In fact, you can get rid of all the Me's, because they're optional in a simple case like this.

The code for the button (usually labeled OK) that closes the userform can be this:

Private Sub cmdOK_Click()
    Dim strCaption As String
    If optB.Value = True Then
        strCaption = optB.Caption
    ElseIf optC.Value = True Then
        strCaption = optC.Caption
    Else
        strCaption = optD.Caption
    End If
    
    ActiveDocument.Variables("Class").Value = strCaption
    
    Me.Hide
End Sub

Hello stb100,

 

 

I have reposted your queries (in bold letters) below and the corresponding responses.

 

I'm confused. I thought you said customers cannot close a Skype account once it is linked with a Microsoft account. So if we have an old Skype account that is not linked to Microsoft, and if we want to delete that account, why would we link it to a Microsoft account? It doesn't follow.

 

A Skype Account cannot be closed once it has been linked to a Microsoft Account because both will be cancelled out. If this linked account is closed, you won’t be able to sign in to other Microsoft products or services such as Outlook.com, Office Online, OneDrive, Xbox Live, or Windows.

 

When deleting a Skype account that is unlinked, you will be asked to link it first to a new/unused Microsoft account - which will also be closed with it. This is because the process of closing Skype accounts is already integrated to Microsoft. However, since you are linking a newly-made Microsoft account, it can be easily disposed of with the Skype account.

 

So it seems Microsoft automatically, without my permission, created a default-kind of account for me.

 

The “live:xxxxxx” account that you are seeing was created when you signed in using your email address instead of using the Skype name. This account is definitely linked to your Microsoft account. To use the correct account, always log in using the credentials you’ve utilized in creating them. If you used a Skype name, sign in with the same information.

 

"Skype for Business" also now appears in my PC's apps. I don't need two Skypes.

 

You need not use the Skype for Business application, nor create an account with it. You can just use regular Skype. For information on Skype for Business, kindly refer to this FAQ: What's the difference between Skype, Skype Meetings, and Skype for Business?

 

I have a very old, old-school Skype account. It isn't an active nuisance (like #2 and #3 are). But I'd like it gone, as I will never use it again.

 

As long as you are able to sign in to the Skype accounts, and if you do not need the Microsoft accounts linked to each (if any), you can proceed with the account deletion.
