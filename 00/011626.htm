ken sheridan say: In a situation like this the path to the file should not be hard-coded in a macro or VBA code, but stored in a column in a table in the database.  That way it is easily edited by the user in the event of the path changing.  The path can then be looked up from the table when the file is to be opened.  The following is some code from one of my own applications which does this with a hyperlink to a OneDrive folder, but it would be just the same for a file on your local hard drive or elsewhere on the system:

           Const CONFIRMTEXT = "Open OneDrive Finance Folder?"
           Dim strOneDriveLink As String

            If MsgBox(CONFIRMTEXT, vbQuestion + vbYesNo, "Confirm") = vbYes Then
                 strOneDriveLink = DLookup("Link", "Links", "LinkDescription = ""OneDrive Finance""")
                DoCmd.Hourglass True
                Application.FollowHyperlink strOneDriveLink
               DoCmd.Hourglass False
           End If

For a means of browsing to a file and inserting its path into a control in a form take a look at BrowseDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the form is unbound, but it could of course equally well be a form bound to a table.
