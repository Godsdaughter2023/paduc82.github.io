Remain line break when import excel to Access - Microsoft Community

tuan pham 1973 say:

Remain line break when import excel to Access

When I import data from excel file to Access, data in Memo field does not remain line break as they are in excel file.
How can I solve the problem

Thks
hansv mvp say:

This is because Excel uses the line feed character (ASCII 10) for a line break, while Access uses carriage return + line feed (ASCII 13 + ASCII 10).


You can do the following in Access, after importing:


- On the Create tab of the ribbon, click Query Design.

- Select your table, then click Add.

- Close the Show Table dialog.

- Click Update in the Query Type group of the Design tab of the ribbon.

- Add the Memo field to the query grid.

- Enter the following in the 'Update to' line:


Replace([MemoField], Chr(10), Chr(13) & Chr(10))


where MemoField is the name of the field.

- Click the Run button (the big exclamation mark).

- Confirm that you want to update the records.


The memo field should now display line breaks.

Since you have a varying number of leading zeros, I assume the cells are formatted as Text.

You can put the following formula into a parallel column, assuming your data starts in A2:

=IF(LEN(A2)<=3, A2, LEFT(A2,3) & "." & MID(A2,4,99))

If you wish, you can then copy the parallel column and overwrite the original data using paste-special-value.

Alternatively, you can use the following macro, assuming you select the first code to be converted (0010):

Sub doit()
Dim c As Range
For Each c In Range(ActiveCell, ActiveCell.End(xlDown))
    If Len(c) > 3 Then
        c.Value = Left(c, 3) & "." & Mid(c, 4)
    End If
Next
End Sub
