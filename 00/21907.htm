yangzheng_msft6300 say:

Hi Nils,

From your description, you can’t delete an email around 20MB size in the Outbox folder on Outlook for Windows client. So firstly, I’d like to clarify some details about deleting emails in Outbox folder.
When an email enters outbox, it means that this email is in the process of sending out and users can only delete this email when the sending process is stopped or interrupted. Emails still in the sending out process can’t be deleted. How long the sending process will take depends on the network, email size and some other factors.
In your case, since you are sending an about 20MB email which is relatively big, the sending process could be slow. So you can’t delete this email when the process has not ended. When you shut down wifi, the sending process will stop. So the email can be deleted.

Yang

    Can I use a macro to generate the same report multiple times based on - Microsoft Community

    kimb.in.denver say:

    Can I use a macro to generate the same report multiple times based on a variable defined in a table?

    I have a table of [test scripts] each assigned to a different user contained with a [users] table.  I need to generate (and output to pdf) a [test script report] specific to each user.  When I had only a handful of users it was no big deal to create a handful of macros to handle it.  But now I am up to 26 users and I don't want to create/manage 26 individual macros nor do I want to manually generate each report.  Guessing it is possible to use the [users] table and some sort of repeat function (redo?) but need some guidance.  P.S.  I am not that adept at VB and typically use the Macro Tools/Design wizards to create my macros but have been able to use the responses here to migrate some of my other macros to VB.

    john w. vinson say:

    Well, this may be something that requires  VB. There are things that macros just aren't designed to do, and looping through a recordset is one of those things!

    You'll need to have some sort of a Query on the Users table identifying which users need to get a report, and use a loop in a VBA function to step through the query, calling the OpenReport method for each user and then sending the generated pdf. The details of this code will depend greatly on your particular system, and it's hard to think of an example that would be more helpful than it would be misleading!

    How are you generating and sending the report now? Could you post the macro steps?

    kimb.in.denver say:

    This is what I am doing in VB now to generate 1 report at a time...

        DoCmd.OpenReport "Test Scripts by Tester", acViewPreview, "", "[tester]=[Forms]![Checkout]![Print tester]", acHidden
        DoCmd.OutputTo acOutputReport, "Test Scripts by Tester", acFormatPDF, _
             "c:\Test Scripts\" & [tester] & ".pdf"
        DoCmd.Close acReport, "Test Scripts by Tester"

    Note:  [Forms]![Checkout]![Print tester] is set via a combo box. 

    I just created the following query to figure out who needs a report...

    SELECT Name.Testers, Count([Checkout].[Install Date]) AS [CountOfInstall_Date]
    FROM Testers LEFT JOIN [Checkout] ON Name.Testers= [Checkout].[Tester]
    GROUP BY Name.Testers

    HAVING (((Count([Checkout].[Install Date]))>0));

    And the [Testers] table just has 1 column - "Name"

    Checkout has a few but the only that matter are "Tester" and "install_date"

    SELECT Count([Install Date]) AS [CountOfInstall Date], [Checkout].[Tester]
    FROM [Checkout]
    GROUP BY [Checkout].[Tester]
    HAVING (((Count([Checkout].[Install Date]))>0));

    john w. vinson say:

    Thanks... that helps! OK, you can probably save your second query as qryTesters and generate the reports with code like this (untested air code, but try it):

    Dim rs As DAO.Recordset

    Dim qd As DAO.Querydef

    Dim db As DAO.Database

    Set db = CurrentDb ' pointer to the current database

    Set qd = db.OpenQuerydef("qryTesters")

    Set rs = qd.OpenRecordset, dbOpenSnapshot ' open a read only recordset of the testers

    Do Until rs.Eof

        DoCmd.OpenReport "Test Scripts by Tester", acViewPreview, "", "[tester]=""" & rs![Tester] & """", acHidden
        DoCmd.OutputTo acOutputReport, "Test Scripts by Tester", acFormatPDF, _
             "c:\Test Scripts\" & rs![tester] & ".pdf"
        DoCmd.Close acReport, "Test Scripts by Tester"

        rs.MoveNext

    Loop

    rs.Close

    Set rs = Nothing ' clean up after yourself

    Set qd = Nothing

    Edited 12/9: added missing quotemark after [tester]=
