Hi,

“Automatic Repair couldn’t repair your PC” error on Windows 10 Automatic Repair is a built-in troubleshooter which should help you fix some of the most common issues regarding your PC. However, due to some system failures or corrupted components, the troubleshooter sometimes fails to be successful.

Here are some troubleshooting steps that you may perform to get this issue fixed:

Solution 1: Troubleshooting with Command Prompt
Since Automatic Repair is a tool which usually deals with startup issues, here are some useful commands which might help you get rid of this error message and possibly even eliminate the option of fixing your PC at all

  1. Navigate to your Windows login screen and click on the Power button.
  2. Hold the Shift key while clicking on Restart in order to enter Boot Options.
  3. Choose Troubleshoot > Advanced options and click on Command Prompt.
  4. When Command Prompt opens, copy and paste the following lines and make sure you click Enter after each command if you want to run it:
    bootrec.exe /rebuildbcd
    bootrec.exe /fixmbr
    bootrec.exe /fixboot

In most cases, the commands above were able to fix the boot of the system and got rid of the “Automatic Repair couldn’t repair your PC” message once and for all. While you are still in the Command Prompt, run the commands below in order to check your disk for errors:

chkdsk /r c:

chkdsk /r d:

Note that you will have to know just what are the letters corresponding to your hard drive partitions and the letters C: and D: are the usual ones for each computer.

Solution 2: Troubleshooting in Safe Mode

There are still a couple of quite useful commands in Command Prompt which you should try but it’s the best if you use these commands while is Safe Mode because the error message we are dealing with is sometimes caused by a faulty driver or a system file which might not show up if we did the scan in normal boot.

  1. Reboot your computer and wait for the boot screen to appear. It’s usually the screen with your PC’s manufacturer with the options such as “Press _ to Run Setup”, etc.
  2. As soon as that screen appears, start pressing the F8 key on your keyboard. If the F8 key doesn’t work, reboot your computer again and start pressing the F5 key.
  3. The Windows Advanced Options Menu should open, enabling you to choose several options to boot your computer.
  4. Boot into Safe Mode with Command Prompt.
  5. As soon as Command Prompt opens, try copying and pasting the commands below in order to check your Windows image for errors (DISM) and to check your system for any missing or broken files (SFC).
    DISM /Online /Cleanup-Image /RestoreHealthsfc /scannow

Please allow these commands enough time to finish and don’t restart or shut down your computer before they finish.

Solution 3: Disabling early launch Anti-Malware protection

  1. Navigate to your Windows login screen and click on the Power button.
  2. Hold the Shift key while clicking on Restart in order to enter Boot Options.
  3. When the Boot Menu opens, navigate to Troubleshoot > Advanced Options > Startup Settings.
  4. Your PC should restart and boot to a list of options for you to choose. Choose the number next to the Disable early launch anti-malware protection option and restart your computer.

Solution 4: Delete the file which is causing these issues

There's a huge possibility that one of the files on your computer simply malfunctioned due to corruption and now causing problems or conflicts with your device which the Automatic Repair can’t deal with. If the file at hand is not a system file, you can locate it and delete it easily.

  1. Navigate to your Windows login screen and click on the Power button.
  2. Hold the shift key while clicking on Restart in order to enter Boot Options.
  3. When the Boot Menu opens, navigate to Troubleshoot >> Advanced Options >> Command Prompt.
    cd C:WindowsSystem32LogFilesSrt
    SrtTrail.txt
  4. The file should open immediately and try to look for a message looking like this:
    “Boot critical file _________ is corrupt.”
  5. Any file can be displayed in the message and that it most definitely the file that is causing this issue and you will have to delete it if it’s not a system file. In order to delete the file, you need to navigate to its location using Command Prompt. For example, if the file is locating in the “drivers” folder in System32 (these files are usually the cause for the issue), you can locate it via this command:
    cd c:\windows\system32\drivers
  6. Delete the file by using the “del” command and the name of the file next to it, divided by a single space.del errorfile.sys
  7. Restart your computer, uninstall the program to which the file belonged to and check to see if your issue is resolved.

First off, most Android phones have internal (built in) and external (micro SD card) storage. Unfortunatly the 360 will only recognize the internal storage so any media files you want to access will have to be stored there.

There are a couple of things that you will have to do in order for your phone to be recognized by the 360. *These steps will be done on your phone, not the 360.*

 

 Turn off USB debugging.

  1. While on the home screen press the menu button and select settings.
  2. Scroll down towards the bottom of the screen and select Developer options.
  3. Deselect USB debugging if it's enabled.

Connect as a Mass Storage device.

  1. Plug your phone into the 360.
  2. While on the home screen open the notification drawer. (swipe down from the top of the screen where the time and data connection are displayed)
  3. Select the notification titled "Connected as a (connection type)."
  4. Select Mass Storage.


I just did all of this on my Droid 4 and it works. I hope it does the same for you.
