jllatham say:

Ok, a little about how things are segregated behind the scenes in an Excel file:

Most code goes into regular code modules (and this is where you probably currently have your code).  Code in these regular code modules can pretty much be used from anywhere else within the workbook.

Workbooks have their own special code module that is typically used to handle "workbook event" processing - such as having a workbook display a specific worksheet when you open the workbook.

Each worksheet in a workbook also has its own special code module that is typically used to handle "worksheet event" processing such as responding to a changed value in a cell.  You may also put "regular" code into these modules!  And when the worksheet is copied, the code in its code module goes right along with it!

And therein lies the solution to your problem as I now understand it (better than I did before).

So copy the code you want to be "copied" with the specific worksheet into that sheet's code module (deleting it from the regular code module).  AT that point you should find that when you copy the worksheet, the code will end up in the new workbook.

You'll see them listed something like this if you look at the list of macros available to be used in either workbook (assuming sheet name is Sheet1, and macro names are Macro1, Macro2 and Macro3):

Sheet1.Macro1
Sheet1.Macro2
Sheet1.Macro3

Instructions for placing code into a worksheet code module can be found here:

http://www.contextures.com/xlvba01.html#Worksheet

Good luck.
