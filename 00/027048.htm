2010/2013 even if they were temporarily offline from Sharepoint. (ie: laptop out of office building)

In Sharepoint 2013 no such local storage of the MPP file takes place . The expectation appears to be that MS Project will always

that can sync when they get back online. They now appear to be out of luck when it comes to offline editing of project tasks..

To connect to the task list we select the 'Open with Project' button from the Sharepoint list ribbon. This opens the task list in MS Project 2013.

Saving from MS Project syncs the work in MS Project to the Sharepoint task list. This works fine. This also creates an MPP file in 'Site Assets' list in the SharePoint 2013 site.

9.)   OneDrive will ask to create a folder in the 'User' path on the workstation/laptop.

10.) Once the OneDrive folder is created the content from 'Site Assets' appears there

11.) In file manager (Windows Explorer Not SharePoint) navigate to the folder where the OneDrive has the files shared.

15.) Check the project task list. Sometimes it gets updated. Mostly it does not. Create a few similar sites to test the behavior.

The SharePoint task list is disconnected from the MPP file in the Site Assets list more often than not.
