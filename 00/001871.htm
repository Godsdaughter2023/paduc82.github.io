jay sun msft say:
Hi Laura,

The owner of the shared calendar can grant the Owner permission to other users or a group. You can do the related operations in Outlook client by following the steps below:

1. Navigate to Calendar pane > Right-click the shared calendar > Choose Properties….
2. Turn to Permissions tab > Click Add… to choose a user or a group you want > Click OK.
3. Choose the user or group you want > Choose Owner in the drop-down list of Permission Level under Permissions.
4. Click OK on the prompt window > Click Apply on the Calendar Properties window.

It should be noted that as only the owner of the shared calendar can grant the owner permission. In this way, as admin, you can contact this user to grant the owner permission of the shared calendar.

be aware that saving files in DOC format may affect any "new features" that have been added since that file format was current (ie 2003). How these features are affected will vary. Some will be stripped out, some will be "degraded".

<ALT><F9> should work in Word 2016. It is possible that it has been reassigned. To check, go to File menu > Options command > Customize Ribbon option > Keyboard Shortcuts Customize ... button

In the Customize Keyboard dialog, click in the "Press New Shortcut Key" input box. Press <ALT><F9>. Look BELOW the "Current Keys" input box to see "Currently assigned to:" which should be ViewFieldCodes.

I found a work around for this. I never found a resolution to this issue which I think is a bug in Word 2016: When I try to insert a picture, and select "Link to file" in the insert dialogue window, alt-f9 does not work for the image that is inserted, although it does so through the INCLUDEPICTURE field code. This was a problem for me, because I wanted to adjust the file source path from absolute to a relative subfolder of the current folder where the docx currently resides.

My work around was that I used Quickparts > field > IncludePicture. For the file path, I input \\subfolder\image.jpeg which worked. When I insert a linked picture this way, alt-f9 works. macropod say:

To see how to implement relative paths in Word, check out the solution I've posted at:
http://windowssecrets.com/forums/showthread.php/154379-Word-Fields-and-Relative-Paths-to-External-Files

With docx/m documents, there's an added complication in that Word refuses to show INCLUDEPICTURE fields in those formats. As a workaround, you could create an empty temporary document that you save in the older .doc format, then insert your linked picture into that. Pressing Alt-F9 will then allow you to edit the field code, after which you can press Alt-F9 again to hide the field code, then use Ctrl-A, F9 to update the link before copying the picture back to the docx/m document.
