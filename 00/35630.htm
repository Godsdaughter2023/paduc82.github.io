Why are msconfig.exe and winver.exe still in the - Microsoft Community

cptn.lee say:

Why are msconfig.exe and winver.exe still in the %SystemRoot%\system32 folder after upgrade to 64 bit version of Vista?

Original Title:System Configuration properties conondrum ,%SystemRoot%\system32\msconfig.exe

Why is this still showing in administrative tools after conversion to a 64 bit system ?{ C:\Windows\system32\winver.exe.}

a. user say:

Why does the existencer of C:\Windows\System32\winver.exe on a 64-bit system surprise you?  It's a useful program for showing the version of Windows being used and there's no reason to remove it because the system is 64-bit.  There are MANY programs that exist both in the 32-bit and 64-bit systems - and some are exactly the same (not every program is different due to the bit size difference).  What problem are you having that makes this an issue - perhaps we can help you with that.

The same goes for msconfig.exe which is a VERY useful program for Vista (and other Windows systems).  If properly configured, it can make a great difference in how quickly the system and programs operate (and how much RAM is available for normal use rather than being tied up with unnecessary programs that startup with Windows by default and aren't really ever used).   I've configured my startup programs to eliminate most of the entries and keep only those that are necessary (like MSE for anti-malware protection) - and it freed up a great deal of RAM and made my system run a bit faster (except for the programs that had entries I eliminated which now open normally instead of faster as was the purpose of including the entries in msconfig - but I rarely use them so that's OK).  If winver is in the msconfig startup group, then I would uncheck the listing as it is not necessary for it it run with Windows at all times (just in case that's the problem).

I assume you upgraded from some 32-bit version of Vista to some 64-bit version of Vista and are concerned with why some of the same programs exist in both versions.  This is perfectly natural and is not a problem with your upgrade.  If you go to Start / Computer / System Properties it will tell you if you have the 64-bit version installed - if it does, then that's the case and all is well.  If you have more than 4GB RAM it is now using the additional RAM (up to its maximum which I don't know off the top of my head).  You will have needed to update some of your device drivers to 64-bit compatible and if the devices are working then you have a 64-bit system.  There will be some 32-bit programs still on your system but this is to be expected (and done by design).  It's nothing to worry about.

I hope this helps.

Good luck!
