IEECo,

 

What happens in Word 2007 if you press Alt-F9? Does the content re-appear? If so, the cause of the problem is that you've managed to embed the 'missing' content in a field code but you can see it in Word 2010 because it's configured to display field codes instead of their values. In that case, pressing Alt-F9 there would hide the content too. If this is what you have:

• in Word 2007 go to Office Button|Options|Advanced>Show Document Content (in Word 2010, it's File|Options|Advanced>Show Document Content) and set the 'Field shading' option to 'Always';

• copy all of the field content, but not including the field braces (i.e. { }) then cut it from the field, leaving just the braces behind;

• delete the field braces; then

• paste the copied content back to where the field braces were.

john korchok say:

Word calculation fields don't work well when chained:

A + B = C, D + E = F, C + F = Word gives you an incorrect total.

Instead, include all original terms in the grand total:

A + B = C, D + E = F, A + B + C + D = Word will calculate this correctly.

bob jones aka: cybertaz say:

How is the data arranged? Is it in a Table? If you provide an example of what you're trying to calculate perhaps there is a method.

John may be right about the accuracy of string calculations, but Word supports summing in very much the same way as Excel. The main differences being that you have to figure out the column/row references yourself [Column A being the leftmost column, Row 1 being the topmost row but the R1C1 style also can be used] as well as update the fields if changes are made to the precedent values. For example, =sum(a1,c3,f2) will sum the values in those 3 non-contiguous cells & =sum(a1:a50) will sum that range as will =Sum(left/right/above/below).

I've never had a problem with the accuracy when using functions rather than strings for simple calculations.
