greginmich say:

Windows Defender (in Windows 8/8.1/10) actually consists of two major components. One is the Antimalware Engine (MsMpEng.exe), aka the Antimalware Service Executable, the Windows Defender Service, and WinDefend. This is the part that provides the primary protection; and this is the part that gets started automatically (by an entry in a registry run key) every time you start up your PC. This part has to run continuously in order to provide real-time protection – so if it ever stops running, you’ll get a warning notification from the Security Center.

The other major component is the Windows Defender User Interface (MSASCui.exe), which is really only a “control center”, if you will – and although the User Interface is required for configuration and communication, it isn't a critical component in terms of protection. So this component normally only gets loaded when it’s opened by the user – and then it remains running in the background even when it’s “closed”, as you'll see if you check in Task Manager.

The Windows Defender icon is associated with the Windows Defender UI component, rather than with the Antimalware Engine; so it will normally be missing at startup. But since you’re fully protected even when the UI isn’t running, this is no cause for concern. Nevertheless, it’s easy to get the icon at startup just by loading the Windows Defender UI at startup – and that’s something that you really don't need a downloaded reg file for. Loading apps at startup is what the Windows Startup folder is for. And if you get your icon by adding a startup shortcut for the Windows Defender UI to your Startup folder, then you can also get an automatic Quick Scan and/or update every time you start your PC:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/is-windows-defender-icon-supposed-to-be-in-system/6cfa9d1d-3db8-49a9-add7-280a284ccfdc

Note that Brink’s reg file actually just starts the Windows Defender User Interface by adding exactly the same startup line to the run key for the current user. Here’s the entire content of the file:

Windows Registry Editor Version 5.00

; Created by: Shawn Brink

; http://www.tenforums.com

; Tutorial: http://www.tenforums.com/tutorials/11974-windows-defender-notification-area-icon-hide-show-windows-10-a.html

[HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run]

"Windows Defender"="\"C:\\Program Files\\Windows Defender\\MSASCui.exe\" -hide"

=IF(G30<=(F30-12),F30-13,F30-6)

when G30=12/30/2015 and

F30=1/15/2016

logical test=False?Also, no matter how I change it True and False answers always give the same date.

joeu2004 say:

How do G30 and F30 appear in the cell, and how do they appear in the Formula Bar?

And what value does the IF formula return?  Not literally TRUE or FALSE.

I suspect that you write literally =1/15/2016 and =12/30/2015 in the Formula Bar, and they appear in the cell as (if formatted General):

Note the equal signs ("=") in the Formula Bar.  It does not belong there.  As such, for example, the content in G30 is the expression 12 divided by 30 divided 2015, not a date.

Remove the equal signs.

Then enter the formulas =ISNUMBER(F30) and =ISNUMBER(G30) somewhere to be sure that 1/15/2016 and 12/30/2015 are indeed recognized as dates in your system configuration.  They should return TRUE.

(If not, the IF formula should return a #VALUE error.)
