Deep linking of region-specific apps on marketplace sucks - Microsoft Community

james ross say:

Deep linking of region-specific apps on marketplace sucks

You can link directly to apps on the marketplace website, which is great. The marketplace website detects browser language and selects a region automatically, which is great. The handling of region-specific apps, however, is far from great. If you are unlucky (very easy when linking to UK-specific apps, since browsers have a real mixture of using en-GB, en-US and en for their language here) and don't get the region the app is available in, you get this abysmal error:

"Error loading application details

This application is not compatible with your device selection, or has been removed from publication."

Neither possibility is correct! The problem is the region. I suggest two possible ways to make this experience significantly better, in increasing order of user-friendlyness:

ken sheridan say:

The Pages property forces two iterations through the report, so I suspect what is happening is that the value of the Page property is still that of the final page when the second iteration begins.  The way to do it is to cancel the footer in its Format event procedure by conditionally setting the return value of its Cancel argument as Duane describes, though, as you want it visible on the first page only, the code should be:

    Cancel = (Page>1)
