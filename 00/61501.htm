ralf_g say:

If the line only appears when Windows is running the graphics driver may have been corrupted. If it was a graphics card driver update you can try rolling back to the previous driver in Device manager. Otherwise download a copy of the latest driver for your graphics card (from computer OEM if possible, otherwise the card manufacturer). Uninstall the existing driver and install the downloaded driver.

If the line appears as soon as the computer is turned on (ie. before Windows is running) or if the display cable is unplugged from the computer it is likely a hardware failure in the monitor. This could be due to component failure, or impact damage to the screen, or sometimes a glitch in the monitor's firmware. If it is a separate monitor you could try doing a factory reset on the monitor itself (check the owners manual for how-to). If that doesn't work it will need servicing or replacement.

BCCode: d1  0x000000d1

Cause

A driver tried to access an address that is pageable (or that is completely invalid) while the IRQL was too high.

This bug check is usually caused by drivers that have used improper addresses.

If the first parameter has the same value as the fourth parameter, and the third parameter indicates an
execute operation, this bug check was likely caused by a driver that was trying to execute code when the
code itself was paged out. Possible causes for the page fault include the following:

  • The function was marked as pageable and was running at an elevated IRQL (which includes obtaining a lock).
  • The function call was made to a function in another driver, and that driver was unloaded.
  • The function was called by using a function pointer that was an invalid pointer.

 

 

 

Resolving the Problem

To begin debugging, use a kernel debugger to get a stack trace.

If the problem is caused by the driver that you are developing, make sure that the function that was executing
at the time of the bug check is not marked as pageable or does not call any other inline functions that could be paged out.

Change Wireless Channel - Microsoft Community

xb1r apollo say:

Change Wireless Channel

2Wire 2701HG-B(AT&T)

Login Information

  • Default Gateway - gateway.2wire.net / 192.168.0.1 / 192.168.1.254 / 192.168.254.254
  • Username - N/A
  • Password - N/A (Windstream uses admin)
    (AT&T login pw: set by customer *click forgot password*)

 

Change Wireless Channel

 

  1. Plug the computer into the LAN port of the device via an ethernet cable.
  2. Open Web Browser.
  3. In the address bar of the browser enter in "gateway.2wire.net" (if this does not work, try one of the following IP addresses: 192.168.0.1/192.168.1.254/192.168.254.254).
  4. If it asks for a password check the side of the device.
  5. From the menus at the top select Home Network.
  6. From the sub-menus select Wireless Settings.
  7. Scroll to the section labeled Identify Network, and from the drop down menu labeled Wireless Channel, select the desired channel. (Channels 1,3,9,11)
  8. Click Save.
