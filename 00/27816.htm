Especially with cross-tab queries, it's critical that you explicitly declare the parameters:

PARAMETERS [Forms]![MgmtReportMenu]![StartDate] DateTime, [Forms]![MgmtReportMenu]![EndDate] DateTime;
TRANSFORM Count(*) AS [Count]
SELECT (Format([DATE],"MMM 'YY")) AS Expr1
FROM USER_TRAINING_RECORDS
WHERE (((USER_TRAINING_RECORDS.COMPLETED)=-1) AND ((USER_TRAINING_RECORDS.DATE) Between [Forms]![MgmtReportMenu]![StartDate] And [Forms]![MgmtReportMenu]![EndDate]))

PIVOT USER_TRAINING_RECORDS.COMPLETED;

dale howard [mvp] say:

Kelly --

You would not be able to use a Dashboard Report for this purpose.  Instead, you would need to create a Visual report to show the data you want to see.  To do this, try the following steps (knowing that you may need to customize some of the fields used in the Visual Report):

  1. In the Visual Reports - Create Report dialog, click the New Template button.
  2. In the New Template dialog, select the Assignment Usage option in the middle of the dialog.
  3. Click the Field Picker button, select any additional fields you want to add, and then click the OK button.
  4. Click the OK button to create the new Visual Report.  This should create a workbook with a single worksheet containing a blank PivotTable.
  5. Add the Resources and Tasks dimensions to the Row fields drop area.
  6. Add the Time dimension to the Column fields drop area.
  7. Add the Work field to the Values drop area.
  8. Expand the Time column to show the level of time data.
  9. Expand each of the projects and summary tasks to show the tasks for each resource.
  10. If this report meets your reporting needs, save the new Visual Report in your default Templates folder.

The reality is that you cannot duplicate the EXACT Resource Usage report found in Microsoft Project 2010 and earlier versions, but you can come close by using a custom Visual Report for Excel.  Truth be know, the Excel Visual Report gives more data than the paper-based Resource Usage report ever provided in earlier versions!  :)  Hope this helps.
