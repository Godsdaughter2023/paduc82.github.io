Critical Path Not Showing Properly in MS Project 2013 - Microsoft Community

cvbanks say:

Critical Path Not Showing Properly in MS Project 2013

I have a very large project (1200 lines) and I am having trouble showing the Critical Path. I can identify the Critical Tasks, and they show in red in the Gant Chart View, but they do not connect together at all.  All the tasks are separate. Here is a screenshot of my Critical Task Summary and corresponding Gant View.  As you can see, there is no continuity among the tasks - they are simply showing independently, thus the critical path is not clear. Any suggestions? 

dale howard [mvp] say:

Cvbanks --

This problem is caused by using Manually Scheduled tasks for both summary tasks and detailed tasks.  When the detailed tasks in any summary section slip past the Finish date of their associated summary task, the Gantt bar for the summary task will turn red.  In this case, it does not mean that the summary task is on the Critical Path.  Instead, it means that the Finish date of the summary task is incorrect.  You can see this in your Gantt Chart view, as indicated by the red squiggly underline directly below the Finish date of each summary task. 

To fix this problem, at a minimum you need to fix the wrong Finish dates for every summary task.  One way to accomplish this would be as follows:

  1. Apply the Summary Tasks filter to display only summary tasks.
  2. Click the Select All button to select all summary tasks.
  3. In the Task ribbon, click the Auto Schedule button to temporarily convert the summary tasks to Auto Scheduled mode.
  4. Click the Manually Schedule button to reset the summary tasks with their new Start and Finish dates to Manually Scheduled.

I personally would not recommend using Manually Scheduled tasks at all in your project.  You are seeing only some of the unexpected consequences.  Instead, I would recommend you use Auto Scheduled tasks, and then link the detailed tasks and milestones into a dependency network so that you can see the true Critical Path in your project.

Hope this helps.

Hi JosephMilioni and welcome to the Skype Community! 

There are a number of reasons why users experience issues with calling a mobile number. Reasons include: 

  • Insufficient credits or an your active calling subscription does not cover your calls. 
  • You might be dialing the wrong number. Try dialing the number in this format: Enter the "+" and then the country code plus the number you want to call. 
  • The number you are calling might not be supported. Unfortunately, Skype does not support calls to all number ranges. Certain premium-priced or geographic/country specific numbers are forbidden and cannot be reached through Skype. 
  • The connection you are using might not be stable. Make sure to use a strong connection whenever you make calls. You can refer to this link for the recommended upload and download speeds for optimal Skype call experience. 
  • If it is happening to just one specific number then it might be due to the fluctuations in the public switched telephone network (PSTN). These are usually temporary and we recommend to try again later.
