    ossiemac say:

    The following assigns the blank cells only between the selected cell and the first row to a range variable and thenselects the range variable.

    The MsgBox simple confirms the selected address/s of the blank cells.

    Code edited since initial posting because xlCellTypeBlanks will error if no blank cells and therefore it is advisable to use On Error Resume Next and assign to a range variable and then test to see if anything is assigned to the range variable.

    Sub SelectBlankCells()

        Dim rngBlanks As Range
       
        On Error Resume Next
        Set rngBlanks = Range(ActiveCell, Cells(1, ActiveCell.Column)).SpecialCells(xlCellTypeBlanks)
        On Error GoTo 0
       
        If Not rngBlanks Is Nothing Then
            rngBlanks.Select
            MsgBox Selection.Address
        Else
            MsgBox "No blank cells to select"
        End If
       
    End Sub

    brian tillman [outlook mvp 2007-2019] say:
    Naturally if you move a message from an IMAP folder to a folder in a local data file, it will be removed from the server. That's what "move" means. Also naturally, if you delete it, it will be removed from the server. Even if the option to mark messages for deletion instead of put them in Deleted Items is chosen, if you are using a view to hide messages marked for deletion, then you won't see them.  The whole idea of IMAP is to sync between the client and the server so that whatever you do on one is reflected on the other. That's how IMAP is designed to work. If you wan to manipulate messages local without having an effect on the server, then use POP.
