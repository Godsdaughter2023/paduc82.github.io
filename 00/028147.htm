allen z. msft say:
Hi colinm2293,

The major difference is that a remote user mailbox is indeed a mailbox, which is hosted in the Exchange Online environment, but a mail user is not a real mailbox, itâ€™s just a local AD account with a valid SMTP address.

According to your description, I assume the difference is caused by the fact that you had assigned an Exchange Online license for this user previously. Actually, this is not recommended. In an Exchange hybrid deployment, we suggest that every mailbox be created on the Exchange on-premises server instead of Exchange Online. (Assigning an Exchange Online license is technically equal to creating an Exchange Online mailbox.)

Based on the current situation, I suggest you try to run Enable-RemoteMailbox for this mail user to see if this can concert it to a normal remote mailbox. If not, then you need to perform some complicated steps:
1.    Remove the Exchange Online license from the Office 365 user account.
2.    Put the on-premises user account into a non-synced OU in order to let the AAD Connect not synchronize this account, and run a force sync.
3.    Check if the corresponding Office 365 user account is deleted. If yes, move on.
4.    Put the on-premises user account back to a synced OU, and run a force sync to let AAD Connect synchronize this account again.
5.    Move the corresponding on-premises mailbox to Exchange Online by referring to Move mailboxes between on-premises and Exchange Online organizations in hybrid deployments. Do not assign Exchange Online license until the mailbox has indeed been moved to Office 365.
After these steps, you can then check if the user is listed as a normal remote user mailbox. And you can feel free to let me know any updates.
