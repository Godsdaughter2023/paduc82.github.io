if statement in a field of a table in Access - Microsoft Community

kyletpa say:

if statement in a field of a table in Access

Hi,

Is there a way to create an if-statement type thing in Access without having to use VBA?  I'm creating a table and within the table I have a number of fields.  One of these fields is directly linked to another, so I'd like to have an if statement that says if Field A = "affirm", then  Field F = "required".

I know how I could do this in excel, but I'm still trying to learn VBA and so if possible I'd like to avoid using it in this database.  Thanks!

 

scottgem say:

When working with Access you need to get out of the spreadsheet mentality. Access is NOT a spreadsheet its a relational database and should be used that way. 

A field who's value is based on another field, is a calculation. As a general rule you don't store calculated values. 

In this case, you would use a table of values like so:

tluStatus

StatusID (PK Autonumber)

Status

Related

so the values might be:

StatusID: 1

Status: Affirmed

Related: Required

Now you can set a control to display the related value without binding it to a field in a table. This doesn't require VBA. You would use a combobox the select Status with the following relevant properties:

RowSource: SELECT StatusID, Status, Related FROM tluStatus:

Bound Column: 1

Column Count: 3

Column widths: 0";1";0"

So the combo would display Affirmed, but store 1 as a Foreign key. On your form you would set the ControlSource of a control that you want to display the Related value as:

[cboStatus].Column(2)

This assumes cboStatus is the name of the combobox. If you need a report you would create a query that links your main table to tluStatus on StatusID and pulls in the Status and Related fields.
