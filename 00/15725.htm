graham mayor say:

Like Doug, I don't have a clue what you are trying to do with your 'loop', or why you have used 4+1 when you could use 5. However to address the business of writing to a Word bookmark from Excel, a simple sub routine that you can call from your macro will enable that. The following will put the value of cell B5 in a bookmark called 'MyBookmark' then resets the bookmark to encompass the added value.

You can call the sub FillBM as many times as you wish, but bookmark names must be unique. You don't have to activate the document when using Word ranges.

Sub Example()
Dim WordApp As Object
Dim wdDoc As Object
Dim oRng As Object
Dim i As Integer
Dim strValue As String

    On Error Resume Next
    Set WordApp = GetObject(, "Word.Application")
    If Err Then
        Set WordApp = CreateObject("Word.Application")
    End If
    WordApp.Visible = True
    On Error GoTo 0
    Set wdDoc = WordApp.Documents.Open("S:\BOKIS skabeloner\MB.docx")
 

       'read the value from the cell
        strValue = Cells(4 + 1, 2)
        'write the value to the named bookmark
        FillBM wdDoc, "BookmarkName", strValue

End Sub


Private Sub FillBM(oDoc As Object, strBMName As String, strValue As String)
Dim oRng As Object
        On Error GoTo Skip
        Set oRng = oDoc.Bookmarks(strBMName).Range
        oRng.Text = strValue
        oRng.Bookmarks.Add strBMName
Skip:
End Sub

Hi,

That can't be done with a formula.

Right click the worksheet tab | view code and paste the code below in on the right.

Private Sub Worksheet_Change(ByVal Target As Range)
If Target.Address = "$D$21" Then
Select Case Range("D21").Value
    Case Is = "Yes"
    Rows("25:38").Hidden = True
    Rows("41:54").Hidden = False
    Case Is = "No"
    Rows("25:38").Hidden = False
    Rows("41:54").Hidden = True
End Select
End If
End Sub

Potential Bug in SharePoint Integration of Microsoft Office Apps

I have run into a recent problem using Microsoft Office apps with SharePoint on my Mac. I have use Microsoft Document Connection for years without an issue, but when I recently purchased a new system and upgraded I ran into a issue that several of my co-workers have also reported.


The Issue


The Office apps (PowerPoint, Excel, and Word) are having problems connecting to the SharePoint server. The problem is not with the SharePoint server, as will be shown, but is definitely in the main Office apps. It manifests itself with an error dialog from the Office app that states there is no connectivity to the SharePoint server, as shown below:


Office Error Dialog


This dialog appears if one of the two options below are used to attempt to open the file:

  1. The URL of the file is used in the File > Open URL…
  2. If the file is double clicked from within Microsoft Document Connection with the preference set that a double click should open the file in Read Only.

It is fairly simple to determine if there is no connectivity to SharePoint server from within Document Connection, as it shows the follow dialog if you attempt to access a file when not connected to a network.


Proxy Error Dialog


But if you’re not seeing this error, MDC is connected to the server and should be able to access the file.


Workarounds


There are a couple of workarounds from within MDC, although neither restores the true flexibility of MDC. One of the greatest benefits of MDC is the ability to read and or edit files without having to store a local copy and worrying about it being out of date when you want to review or edit it later; it is a very decent “cloud” syncing option for multiple team members.

Option 1, change the preferences to always checkout a file when double clicked from within MDC, as shown below:


Checkout MDC Option


Not a terribly great option if you want to simply read a file that is on SharePoint, as this requires discarding the checkout when done and the Office app does not show the SharePoint integration commands like Checkin… or Discard Checkout… as a result of this bug. Forgot to discard the checkout and you’ve locked a colleague from editing the file later.


Another potential workaround is to use the Save As… option when you right click on the file from within MDC. This will save a local copy, but then requires extra steps if you later want to edit it, or you have to download it again if a colleague has made a change. While MDC is actually doing a download every time you use it to read or edit a file, it is maintained in an internal cache and you don’t have the “is this the right version” worry that you have using theSave As… option.


Neither of these workarounds should work if the server cannot be accessed due to a proxy issue. This is what leads me to believe it is a problem with Office and not with MDC or the SharePoint itself.


Hardware / Software


I have seen this bug occur on 13“ or 15” MacBook Pros with or without Retina Displays. It also occurs on systems running 10.7.x or 10.8.2.

I’ve seen this bug on Office 14.2.3 and the recently released 14.3.0. My older machine, a 15" MBP running 10.8.2 and Office 14.2.3 didn’t have this problem.

I have tried uninstalling Office 2011 and reinstalling, without any luck.


Conclusion


It is not clear where Office is getting it’s proxy settings. I would have thought it came from the system proxies, but given that MDC appears to be working would suggest otherwise. Not clear why it works for some systems and not for others.


I’ve done several different Google searches over the last week and cannot find a reasonable explanation. I am going to post this to both the SharePoint and Office forums, as I have seen a trend in the forums for the Office pros to point to SharePoint as soon they see the word SharePoint and vice versa.
