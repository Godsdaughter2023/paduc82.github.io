Explorer Network Browsing

The Computer Browser service relies on the SMBv1 protocol to populate the Windows Explorer Network node (also known as "Network Neighborhood"). This legacy protocol is long deprecated, doesn't route, and has limited security. Because the service cannot function without SMBv1, it is removed at the same time.

However, if you still have to use the Explorer Network in home and small business workgroup environments to locate Windows-based computers, you can follow these steps on your Windows-based computers that no longer use SMBv1:
Start the "Function Discovery Provider Host" and "Function Discovery Resource Publication" services, and then set them to Automatic (Delayed Start). When you open Explorer Network, enable network discovery when you are prompted.

All Windows devices within that subnet that have these settings will now appear in Network for browsing. This uses the WS-DISCOVERY protocol. Contact your other vendors and manufacturers if their devices still don't appear in this browse list after the Windows devices appear. It is possible they have this protocol disabled or that they support only SMBv1.

Other things you can try:
- shut all computer and network gear down.
- Try browsing to some of the computers manually, press Windows key + R, type: \\computername, hit Enter
- Press Windows key + R
Type: services.msc
Hit Enter

1. Start the "Function Discovery Provider Host" and "Function Discovery Resource Publication" services, and then set them toAutomatic (Delayed Start).
2. When you open Explorer Network, enable network discovery when you are prompted.

Press Windows key + X > computer management-->shared folders--> choose the shared folder, right click it then click Properties > Share Permissions -->protection that everyone can read and execute, see folder contents and read permissions.

p376p say:

I have had the same problem and luckily I accidentally found a fix.

Use 'Ctrl+Alt+Delete' to get to task manager. Under 'Background Processes', you will see 2 'Microsoft Edge'. In my case, one with over 200MB memory and the other with less than 100MB memory. I think that the larger one is for the blue screen and the smaller one is for the grey screen. Select the larger one and use end task to close it and then click the Microsoft Edge icon to reopen it. Repeat the closing and reopening a few times, until a screen appears saying that the browser could not open this website and asking whether you want to return to the start page. Click yes to return to the start page and then everything will return to normal.

This fix worked for me and I hope it will also work for you.

trying to connect jambox via bluetooth - Microsoft Community

davidjenkins_bburg say:

trying to connect jambox via bluetooth

I am trying to connect a mini jambox to windows 10 pc.  Connected usb power for jambox to pc, then pc recognized the jam box in list of devices.  How do I direct sound to jambox?  If I disconnect from the PC's usb port, I can get sound from jam box by using headphone output from PC.  Could not connect via Bluetooth--after blue tooth recognizes jambox, there is a message "cannot connect, try again".  Many tries and restarts did not succeed in getting a blue tooth connection.

a. user say:

The device might be hidden. Since the device is detected and you have successfully paired it. It should somehow appear on the audio devices connected. Here, follow these steps, see if it helps.

  1. Click Start then go to Control Panel. Or type Control Panel on the Cortana search bar beside the start button.
  2. Click Hardware and Sound.
  3. Click on Sound. A new window should open.
  4. Click Playback tab.
  5. Right-click on any space inside the tab and see if there's a check on Show Disabled devices. If not, click on it.
  6. Check if your audio device is in there, if yes, right click on it, then choose Enable.
  7. Highlight/Select that speaker afterwards, then click on Set as Default at the bottom.
  8. Test if issue still persists.

If there's anything else you need, don't hesitate to come back to us.
