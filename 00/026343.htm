karl timmermans [outlook mvp 2012-2018] say:
I now have two birthdays for every contact.   If I delete them they come back.  Does anyone know why Outlook would re-create a birthday event on my calendar? 

This is in response to originator and all other who have posted since then and applies equally to the Anniversary field as it does to the Birthday field.

There is only one way that an Anniversary/Birthday would get re-added by Outlook after is has been deleted from the calendar and that is by an internal contact property getting cleared which is not accessible/visible via the Outlook UI. To do it manually, programs like MFCMapi or OutlookSpy would need to be used AND you know exactly which property to clear (delete/set to null) for each respective field (there are two properties in play, one each for Birthday/Anniversary in addition to the actual Birthday/Anniversay fields).

Some background info on how the Birthday/Anniversary fields work when it comes to the calendar:

Birthday / Anniversary calendar entries via Outlook UI
http://www.contactgenie.info/understanding-outlook-birthday-anniversary-fields-via-UI

Importing Outlook Contact Birthday / Anniversary date values
http://www.contactgenie.info/how-to-import-outlook-contact-birthday-anniversary-dates

Understanding Default calendar folder for new Outlook Birthday / Anniversary dates

http://www.contactgenie.info/understanding-outlook-default-folder-birthday-anniversaary-dates

Unless the specific internal property(ies) gets cleared, Outlook would never re-add a Birthday/Anniversary value back to the calendar. There is just no mechanism within Outlook on its own for that to occur. The properties in question contain the links to the calendar - or more specifically - the EntryID of the Calendar entry. Deleting an Anniversary/Birthday from the calendar does not reset the applicable property in the contact item and as long as that property contains a value - Outlook does not re-add the date since it does not re-verify the existance of the date in the calendar.

The question then becomes, what other function(s) are being run with Outlook (i.e. any kind of add-in, sync process etc)? There are a number of different ways/reasons why this could be occurring - some of which are

#1 - If a contact was deleted and re-added for whatever reason - something that you may not know is occuring but a quick way to determine if it is, is to check the "Create Date" for a contact item where a date is suddenly getting duplicated.

#2 - A process external to Outlook is clearing the applicable internal property field which would "reset" the triggers for the creation of a Birthday/Anniversary in the calendar when the contact is opened in the Outlook UI

#3 - Some other process is adding another entry in the calendar.

This applies to all versions of Outlook and ne thing is certain - Outlook would/does not add duplicates on its own unless something else has occurred and that something cannot be done by a user via the conventional Outlook UI.
