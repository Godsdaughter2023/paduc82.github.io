rohn007 say:

I have a (small) spreadsheet with a similar need. I need to be able to mark cells to make them easy to find again.

I created a couple of macros to mark them, and later clear the marks. They are not exactly what you want, a thicker than thick border, but if you combine the border with background color it  does make the cells visible

Sub Borders_none()
'
' Borders_none Macro
'

'
    Selection.Borders(xlDiagonalDown).LineStyle = xlNone
    Selection.Borders(xlDiagonalUp).LineStyle = xlNone
    Selection.Borders(xlEdgeLeft).LineStyle = xlNone
    Selection.Borders(xlEdgeTop).LineStyle = xlNone
    Selection.Borders(xlEdgeBottom).LineStyle = xlNone
    Selection.Borders(xlEdgeRight).LineStyle = xlNone
    Selection.Borders(xlInsideVertical).LineStyle = xlNone
    Selection.Borders(xlInsideHorizontal).LineStyle = xlNone
    
End Sub
Sub border_thicker()
'
' border_thicker Macro
'

'
    Selection.Borders(xlDiagonalDown).LineStyle = xlNone
    Selection.Borders(xlDiagonalUp).LineStyle = xlNone
    With Selection.Borders(xlEdgeLeft)
        .LineStyle = xlContinuous
        .ColorIndex = xlAutomatic
        .TintAndShade = 0
        .Weight = xlThick
    End With
    With Selection.Borders(xlEdgeTop)
        .LineStyle = xlContinuous
        .ColorIndex = xlAutomatic
        .TintAndShade = 0
        .Weight = xlThick
    End With
    With Selection.Borders(xlEdgeBottom)
        .LineStyle = xlContinuous
        .ColorIndex = xlAutomatic
        .TintAndShade = 0
        .Weight = xlThick
    End With
    With Selection.Borders(xlEdgeRight)
        .LineStyle = xlContinuous
        .ColorIndex = xlAutomatic
        .TintAndShade = 0
        .Weight = xlThick
    End With
    Selection.Borders(xlInsideVertical).LineStyle = xlNone
    Selection.Borders(xlInsideHorizontal).LineStyle = xlNone
End Sub

'
' border_thick_red Macro

        .Color = -16776961
        .TintAndShade = 0
        .Weight = xlThick
    End With
    With Selection.Borders(xlEdgeTop)
        .LineStyle = xlContinuous
        .Color = -16776961
        .TintAndShade = 0
        .Weight = xlThick
    End With
    With Selection.Borders(xlEdgeBottom)

Exact Matches in Query Criteria, Access 2007 - Microsoft Community

laurabean say:

Exact Matches in Query Criteria, Access 2007

I'm having trouble with this simple query.  I want to exclude all columns from my query where the entry for the 'Status' field exactly matches 'burned' and so in my query I set the criteria on the 'Status' column to Not 'burned'.   Instead of excluding all entries where the data field exactly matches burned, the query is excluding all fields that contain the word 'burned'.  How can I avoid this?  Some of the records have a 'Status' of 'unburned' or 'burned/wildfire' and I don't want these records to be excluded in the query.


john w. vinson say:

Mixing AND with OR logic in a query is tricky even in SQL view - you have to be able to do Boolean algebra in your head! - but the query grid makes it even harder; and the other issue is that the criteria apply to the query as a whole. It either updates all the fields in the record retrieved, or none of them. If you only want to update fields if they are NULL it's tricker yet! I'd either do one field at a time, or use a (probably slow and inefficient) update using IIF:

 

UPDATE ALL_Unique_Macroplots INNER JOIN AllMacroplots

ON (ALL_Unique_Macroplots.Macroplot_Name = AllMacroplots.Name2) AND (ALL_Unique_Macroplots.Park_Unit = AllMacroplots.[Park Unit])

SET ALL_Unique_Macroplots.MacroPlot_UID =

IIF(IsNull(ALL_Unique_Macroplots.MacroPlot_UID),

[AllMacroplots]![MacroPlot_UID],

ALL_Unique_Macroplots.MacroPlot_UID), 

ALL_Unique_Macroplots.Burn_Unit =

IIF(IsNull(ALL_Unique_Macroplots.Burn_Unit),

ALL_Unique_Macroplots.Burn_Unit,

[AllMacroplots].[Burn Unit]),

<etc. for all the fields you're updating>

WHERE ALL_Unique_Macroplots.MacroPlot_UID) Is Null OR ALL_Unique_Macroplots.Burn_Unit Is Null OR ALL_Unique_Macroplots.[Burn Date1] Is Null OR ALL_Unique_Macroplots.Project_Unit Is Null OR ALL_Unique_Macroplots.Burn_Status) Is Null;

The trick is to see if the target field is NULL. If it is null, update it to the source field; if it isn't NULL, update it to itself (i.e. leave it alone and don't overwrite it).

Generally, when we upload or add a file to the group site document library, the file will be shared with Group Members, Group Owners and the Group Visitors. To achieve your requirement, we need to remove these three groups and re-invite the users we need to share the file to the users we need.

3. After the prompt, the windows disappears. We need to re-click Share > Shared with > ADVANCED, then we will be able to remove these three groups’ permission via clicking these three groups > Actions > Remove User Permissions.

 

4. After that, we can re-share the file or folder to users we need via clicking Share > Invite People.

There are two sort orders for the Outlook AddressBook (1) First Last & (2) File As

In terms of setting the order for the AB, this is done by by going to
