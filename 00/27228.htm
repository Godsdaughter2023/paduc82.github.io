    It's the sheet protection, do as follows:

    Open the file, resp. edit the XLTM

    go into the module "password"

    run Sub passwordoff

    What program are you looking for?  FWIW I had to resort to using ProcExp (Run as Administrator) to use its Find tool to see that it is supposedly ApplicationFrameHost running under a svchost running under a wininit.   None of those go away when I close the Feedback Hub window.  I imagine we will need to run ProcMon to try to understand better where the window should be coming from.  In fact, I was able to notice it opening and closing.  It's called PilotsHubApp. That appears when I press Win+F and disappears when I close the Feedback Hub window.  There may also be an additional instance of a RuntimeBroker involved.

    More conventionally I found this PowerShell pipeline we can use to achieve the same detection

    tasklist /apps /v | sls "pilot"

    The /V verbose switch also shows me the equivalent of 

    (Get-AppxPackage *Feedback*).PackageFullName

    which is then seen to be 

    (Get-AppxPackage *Feedback*).Name
    Microsoft.WindowsFeedbackHub

    If you have a problem with RuntimeBroker we may be able to list its Status using WMIC.  E.g. enter WMIC and then enter

    process where (Name = "RuntimeBroker.exe") list status

    FWIW all my RuntimeBroker show no Status but if you have one that is not running well you may be able to identify its PID that way and then get some more specific information about it.
