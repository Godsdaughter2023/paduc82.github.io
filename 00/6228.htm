j.e. mcgimpsey say: Bob's right, but just in case, I also remembered a user defined function I wrote a long time ago to parse names into 3 columns (I'd probably write it differently now, but it works). The function is entered as an array function (i.e., Select B1:D1, enter the formula in B1 and hit CMD-Return):

   =ParseName3(A1)

Put this in a regular code module:

    Public Function ParseName3(ByVal sName As String) As Variant
        Const csSpace As String = " "
        Dim i As Long
        Dim sTemp As String
        Dim vResult As Variant
        
        On Error GoTo Err
        vResult = Split(Application.WorksheetFunction.Trim(sName), " ")
        Select Case UBound(vResult)
            Case Is = 0
                ReDim Preserve vResult(0 To 2)
                vResult(1) = vbNullString
                vResult(2) = vbNullString
            Case Is = 1
                ReDim Preserve vResult(0 To 2)
                vResult(2) = vResult(1)
                vResult(1) = vbNullString
            Case Is > 2
                For i = 3 To UBound(vResult)
                    vResult(2) = vResult(2) & csSpace & vResult(i)
                Next i
                ReDim Preserve vResult(0 To 2)
            Case Else
        End Select
    ExitFunction:
        ParseName3 = vResult
        Exit Function
    Err:
        Debug.Print Err.Number, Err.Description
        Resume ExitFunction
    End Function

bob jones aka: cybertaz say:
The straight answer to all 3 questions [main, 1. & 2.] is No :-)

For several OS-related reasons, Office (both 2008 & 2011) simply doesn't use the Microsoft User Data folder in the same fashion as it did for Office 2004 & earlier. Further, Office 2011 did create a new MUD folder, but -- for those same reasons -- put it in a different location. It added content to but did not delete or otherwise modify the Office 2004 Microsoft User Data folder. Preexisting content is preserved so that items could be retrieved from it as needed...

The installation of Office 2011 does check to see if a Normal template exists from a prior version, as well as other personal settings & information. If so it assimilates such items as macros & certain types of customizations for you. You can also use Organizer to import Styles & other customizations from your previous Normal template. Had you done as you asked in your main question your macros & other customizations would have been lost.

The only potential "downgrade" is if you choose to continue saving files in the deprecated [e.g., .doc, .dot, .xls] formats rather than using the newer OOXML [e.g., .docx, .dotx, .xlsx] formats native to Office 2011... the older formats do not support many of the features available in Office 2011.

amit tandon say:

Refer below image:

Formula in cell G1 - this counts the number of cells in B1:E1 which are not blank and have "MI" as the first 2 letters:

=SUMPRODUCT((LEFT($B1:$E1,2)=$A1)*1)

Formula in cell G2 - this counts the number of cells in B1:E1 which are not blank and DO NOT have "MI" as the first 2 letters:

=SUMPRODUCT((LEFT($B2:$E2,2)<>$A2)*($B2:$E2<>""))

Formula in cell I1 - this counts the number of cells in B2:E2 which are not blank and contain "MI" within (not necessarily the first 2 letters):

=SUMPRODUCT(ISNUMBER(SEARCH($A1,$B1:$E1))*1)

Formula in cell I2 - this counts the number of cells in B2:E2 which are not blank and DO NOT contain "MI" within:

=SUMPRODUCT(NOT(ISNUMBER(SEARCH($A2,$B2:$E2)))*($B2:$E2<>""))

Have given both options - (1) where "MI" are the first 2 letters (2) contain "MI" within the cell (not necessarily the first 2 letters).

amit tandon say:

The formula =SEARCH("MI",A1,1) searches for the text "MI" within the cell A1 and returns the number of the starting position of "MI". If the formula returns 8, it means that there are 7 characters before the text "MI" appears in cell A1 and "M" is the 8th character. The 1 in the formula means that the search starts from the first character of cell A1, and omitting this also defaults to 1 so that you can use the formula =SEARCH("MI",A1) alternatively.

If the formula =SEARCH("MI",A1) returns 1 it will mean that the first 2 characters in cell A1 are "MI".

You may also use the formula =LEFT(A1,2)="MI" to determine if the first 2 characters in cell A1 (LEFT(A1,2) are equal to "MI". The formula will return TRUE if this is so or else the formula returns FALSE if the first 2 characters in cell A1 are not "MI".

The formula =SEARCH("MI",A1) returns a number if the text "MI" is present in cell A1 and if "MI" is not present then the formula returns a #VALUE! error. Hence, using the formula =ISNUMBER(SEARCH("MI",A1)) will return TRUE if "MI" is present in cell A1 and the formula returns FALSE if "MI" is not present.

Amit Tandon

There are scripts to deal with duplicates if you can get the Identity back. You can set to "leave on server"  in your Account. I suggest no more than 30 days or you could get duplicates again.

  • Select Accounts under Tools in the Menu bar
  • Click on the Options tab
  • Set how many days to "leave on server" 


When your mail server receives a new message it gives the message a unique ID. This is what mail applications use to determine whether or not to download messages. New IDs get downloaded. Existing IDs don’t. When mail servers are rebooted or reset in some way they often reset the unique IDs. The next time the mail client connects it sees that it has a lot of “new” messages and downloads them again.

You can work through this a couple of different ways:

  • Edit your POP account settings to delete messages that are older than a certain length of time. Start with five days. If the server is reset then you will only receive the last five days worth of messages again.
  • Convert to an IMAP account instead of POP. IMAP is made to synchronize with your mail server unlike POP, which downloads and deletes.
  • Trying to use your POP server as an IMAP server by leaving messages on your server is not a good solution
