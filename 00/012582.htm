steve rindsberg say:
One reason for (apparently) "orphan" POWERPNT.EXE processes is the preview pane in Windows explorer.  When you have it open and click a PPT file, the preview pane launches a hidden instance of PowerPoint to provide the previews.  

If the PPT file is highlighted in Explorer and visible in the preview pane, the hidden instance may be holding the file open, so it can appear as though the file's already in use by another user, and hence open as R/O when the user opens it.

tom ogilvy say:

royboymaps,

Perhaps this will do what you want:

Sub DCopy()
Dim i As Long, iMatches As Long
Dim aTokens() As String: aTokens = Split("2015 Ash", ",")

For Each cell In Sheets("Sheet1").Range("B:B")
    If (Len(cell.Value) = 0) Then Exit For
        For i = 0 To UBound(aTokens)
            If InStr(1, cell.Value, aTokens(i), vbTextCompare) Then
                iMatches = Sheets("Sheet2").Cells(Rows.Count, "B").End(xlUp).Row + 1
                Sheets("Sheet1").Rows(cell.Row).Copy Sheets("Sheet2").Rows(iMatches)
            End If
        Next

Next
Dim bTokens() As String: bTokens = Split("2015 Meg", ",")
For Each cell In Sheets("Sheet1").Range("B:B")
    If (Len(cell.Value) = 0) Then Exit For
        For i = 0 To UBound(bTokens)
            If InStr(1, cell.Value, bTokens(i), vbTextCompare) Then
                iMatches = Sheets("Sheet3").Cells(Rows.Count, "B").End(xlUp).Row + 1
                Sheets("Sheet1").Rows(cell.Row).Copy Sheets("Sheet3").Rows(iMatches)
            End If
        Next

Next

End Sub
