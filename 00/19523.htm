Private Sub Workbook_Open()
    Application.ScreenUpdating = False

    ActiveWorkbook.Close False

Actually I didn't get that out of your initial posting.  But ...

Excel acts kind of strangly with regards to the calculation mode.  A particular session will take on the calculation mode of the first workbook opened, whether it was set to manual or automatic.  This mode is workbook-dependent, not an application setting.

This is the reason a lot of people end up with many workbooks that are mysteriously set to manual instead of automatic calculation.

So if the first workbook you open during an Excel session is set to Automatic calculation, unless you change the setting at that time, all other workbooks opened during the session will be set to automatic when they are opened.

joeu2004 say: Tom wrote:
I understand how Excel changes depending on what is opened first.  For my work, I wish all would open set to manual recalculate.

Then perhaps you understand that you can start Excel -- that is, click or double-click the Excel icon on the taskbar or desktop, not an Excel file icon -- set manual mode manually by clicking on Tools, Options, Calculation, then use File > Open to open each Excel file in the same instance.  (Note:  I use XL2003.  Adapt as needed for XL2007.)

Alternatively, you might save an unused workbook in manual mode.  (Not surprisingly, I call it manual.xls.)  Then when you want to open another file and force manual mode, you can first open manual.xls, then in the same Excel instance, use File > Open to open the intended Excel file.  At that point, you can close manual.xls.

Those approaches obviate the need to change personal.xls[b], which would apply to every Excel file opened.

If you choose to use Workbook_Open in personal.xls[b], I suggest that you use InputBox to offer yourself the option of forcing manual or automatic calculation mode.  Unless, of course, you will never want to use automatic calculation ... ever (!).
