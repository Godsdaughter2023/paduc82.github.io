OneNote has two types of ink: Handwriting and Drawing. Drawings are created on the page surface while handwriting is in a container. Whether the ink you put on the page is considered Handwriting or Drawing is dependent on a few things.

 

First there is a setting that decides whether ink is treated as:

  1. Handwriting only
  2. Drawing only
  3. Handwriting or Drawing based on the shape

In the third mode, OneNote will try to see whether you are drawing something or handwriting based on the shape of your input.

These options are available if you go to Insert > Favorite Pens > Drop the menu and at the bottom there is Pen Mode. Under Pen Mode these three options are available.

 

If you always want your ink inside containers, you can choose Create Handwriting Only. Also there is a way to turn a Drawing into Handwriting. Simply right click on the ink and choose Treat Selected Ink As > Handwriting.

ossiemac say:

If you could offer some further advice on Option Explicit, I'd be most grateful.

Hello Corin,

Basically Option Explicit says that all variables in the code must be declared (ie. Dim statements).

The single biggest advantage that I see with using it is that if Debug -> Compile is used before running any code then it will stop on any variable that is not declared and in my case it is usually a misspelt variable name. (I still can't find a keyboard that types what I meant to type instead of what I did type)

Copy the following simple example into a module in new workbook. Initially run the code with Option Explicit commented out.  (If you already had Option Explicit at the top of the module then delete it). The MsgBox returns "No value assigned to strMyText" when the lines of code immediately above assigned a value to it. This is because the If statement has the variable misspelt and the misspelt variable becomes a new variable and does not show up as an error. Problems like this are very hard to diagnose because the code can go ahead and process the data incorrectly and return the incorrect results and it is hard to determine why it is not working properly.

Now uncomment the Option Explicit and re-run the code and you will get an error. The variable will be highlighted  on the If statement line and a message telling you that the variable is not defined.

Also, before running the code, you can use Debug -> Compile and it will find the error before running the code. This will also find any object like a TextBox or ComboBox control that is misspelt.

'Option Explicit      'Initially leave commented out.

Sub Test()

    Dim strMyText As String
   
    With Worksheets("Sheet1")
        strMyText = .Cells(2, "A").Value
    End With
   
    If strMyTest = "" Then      'Variable is misspelt
        MsgBox "No value assigned to strMyText"
    Else
        MsgBox "strMytext = " & strMyText
    End If

End Sub

boboneill say: hi!

i am an IT professional with 15 years experience. I have worked for (and currently work for) very large organizations and I have participated in 3000+ seat upgrades of all Windows versions since version 3.1 (including upgrades of Microsoft office).

i can tell you from the "IT perspective" that any large-scale software upgrades are a signficant undertaking.

to save you the time of reading a long reply, i'll give you the short answer first: if you are on Office 2007, there is no significant reason to upgrade to Office 2010, and no answer you hear on this website will change your CIO's mind. 

if you use Office 2010 at home, then you already know of the features it has over any previous version of Office...but from an overall corporate perspective, there really is not compelling argument to upgrade.

while i dont know your organization my past experience tells me that you wont see an Office upgrade until the company upgrades all of the computers in the company (sorry)

if your company is on Windows XP now then an upgrade should be in the works soon...but if you are on Windows Vista or Windows 7, then you probably wont see an upgrade for several months.

the most significant upgrade "driver" is cost. computer equipment brings no competitive advantage...every company has computers...so computers bring some efficiencies but upgrading to new stuff doesnt mean your company will show a massive bump in shareholder value...so the cost of upgrading computer software is just cost that is eaten by a company.

in addition to no competitive advantage, the cost of upgrading includes the cost of training how to use the new software, the cost of licensing, the cost of compatability testing, the cost of deployment, the cost of post-deployment support and the on-going cost of troubleshooting errors.

nicdw say:

I had a similar issue and resolved it by deleting my existing PERSONAL.XLSB file from the XLSTART folder and then causing Excel to create a new PERSONAL.XLSB file.

 

Note: if you have VBA code in your current PERSONAL.XLSB file that you would like to keep then you should copy that code before deleting the current PERSONAL.XLSB file and manually add the code back to the new PERSONAL.XLSB file after you have followed the instructions below.  

 

1) Locate file PERSONAL.XLSB

 

Perhaps the easiest way to locate the PERSONAL.XLSB file is to use your computer's search engine and search for "PERSONAL.XLSB". 

 

Alternatively, you can navigate to the XLSTART folder.  Please note: you will need to toggle on view Hidden Items.  The default location is:

Local Disk (C:) > Users > [select your user folder] > AppData > Roaming > Microsoft > Excel > XLSTART 

 

2) Delete file: PERSONAL.XLSB.

 

3) Open Excel.

 

4) Open a blank workbook.

 

5) Record a Macro. 

 

This will cause Excel to recreate a new PERSONAL.XLSB file.

In Excel, select the View tab.

Click the dropdown arrow under Macros, select Record Macro, click OK. 

Return to the View tab, click the dropdown arrow under Macros again, select Stop Recording.

 

6) Close Excel and save the Personal Macro Workbook.

Click to close Excel and you will be prompted to save the changes to the Personal Macro Workbook. Click Save.
