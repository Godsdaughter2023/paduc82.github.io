All you need to do is set the option in the Windows Update to "Download Updates but let me choose whether to Install them."

Open task Manager and click on the CPU tab and then open the "Resource Monitor" and look for the wuauserv in the lower view pane column  is running for the Windows Update.

My CPU runs between 4% and 5% idling and between 4% and 100% when it's installing updates and after the updates it drops back to the 4% idle.

You can use the Resource Monitor to suspend a process.

Just look in the top column to the far wight for the "Average CPU" and it will show the process that's running the CPU at 100% and right click on that process and suspend it and then go to the Windows Update page and select the "Download Updates but let me choose whether to install them."

You will still get the updates as long as you resume the process you suspended "wuausrv." for Windows Updates.

The "BITS" will only show in the Resource Monitor when your WIN 7 PC is actually downloading updates and you'll see your CPU meter running high so don't panic as long as the BITS is listed in the Resource Monitor.

In other words you want to suppend the wuausrv process to stop it from running the CPU at 100%.

Then you go select the "Download Updates but let me choose whether to install them." and then go back and resume the suspended "wuausrv" process and then reboot your system and when your back on the desktop check to make sure you selected the Download Updates but let me choose" option.

You can pin the Resource Monitor to your task bar so you can get to it without having to open the Task manager.

1.Unzip the sdutil.zip file and rename it from sdutil.ex_ to sdutil.exe

8.Try to restore the backup as usual using the advanced restore option and next the option restore an older backup created in this machine
9.If done: it’s over!
10.If it fails: Map a network driver at your LAN
11.Copy the MediaId.bin file to the root of the previous mapped LAN driver
12.Copy the whole folder BackupSet “aaaammdd” “xxxxxx” to this same driver
13.Try to restore the backup as usual using the advanced restore option and next the option restore backup created in another machine
14.At the next menu, point the backup folder at your LAN mapped driver and start the restore process
15.Now you shall have your backup proper restored at the right machine
16.Good Luck!
