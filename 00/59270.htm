I'd suggest you perform the following in the order of appearance and check if the issue persists.


a. Open a command prompt on your computer (Click Start, type "cmd" in the search box)

b. Type: manage-bde -status <drive letter> eg. manage-bde -status e:

This should let you verify the drive is unlocked and encrypted

c. Now type "manage-bde -off <drive letter>" eg  manage-bde -off f:

 

This tells Windows to turn off encryption on that drive and decrypt it.

Be patient, decryption time can take a very long time, 12hours per 1.5Tb

Windows update 1803 (spring feature update) corrupts profile - Microsoft Community

lp3456345673456 say:

Windows update 1803 (spring feature update) corrupts profile folder

the 1803 update consistently (tried on 3 different installations) corrupts files in the User folders including %appdata%, renaming existing files from filename.ext to filename (1).ext and replacing the original filenames with files from some backup or restore-point (in my case those are dated from 2013).

The applications then startup as though never configured or with some old settings that were already made at that date.

This effects most programs that rely on config files inside your profile folder

eg.:

Thunderbird

Spark (jabber client)

Google backup

Virtualbox

...

to fix this:

if linux subsystem is installed make a backup of C:/Users and do

cd /mnt/c/Users/
find . -name '* (1).*' -execdir sh -c 'for arg; do mv "$arg" "${arg% (1).*}${arg#*(1)}"; done' _ {} \+

this will remove the (1) from the filenames overwriting the old files.

If you are a sysadmin I recommend to pause all feature updates until this has been fixed.

How can I file a bugreport?

edit: affects the whole C:/Users not just %appdata%

It can't be done directly with Windows Contacts (well, possibly depending on how iPhones work).  You can create separate folders for each of you to keep the contacts separate (and move them to the correct folder every time a new contact is added), but they will all synchronize if you use the normal method (unless the normal method allows you to input a precise folder - then you're all set).  I don't know if this will help as I'm only vaguely aware of how to synchronize contacts, but if it is possible that the contacts are stored in different folders if created in different folders within Windows Contacts (both in Vista and in the iPhones), then it may be possible to just synchonize those folders instead of Windows Contacts in general and thus keep them separate.  I don't know if this is possible in iPhones but in Vista the contact information is stored in C:\Users\<Username>\Contacts and within Contacts it can be further separated by folders (Organize / New Folder) which will carry over to the main Contacts folder as separate folders within it - in the iPhone it may all just go into the one contacts database and thus be inseperable with no separate folder of information but if you contact them they may know. 

But if it can't be done normally as described above it may still be possible.  I believe the product is SyncToy:http://www.microsoft.com/downloads/details.aspx?familyid=C26EFA36-98E0-4EE9-A7C5-98D0592D8C52&displaylang=en.  I'm not 100% sure SyncToy will work with iPhones (in fact, I'm pretty sure it won't so this is probably not an option) but you can try with the main folder and see if it works (the trick and the key will be if it installs on the iPhone).  If so, you can then go ahead and separate the contacts into the two folders you would create and then setup the sync to use those sub-folders instead of the primary folder.

I hope this helps.  If neither option works, then I'm afraid I don't know of any other options to try except to contact iPhones and see if they have any suggestions.

Good luck!
