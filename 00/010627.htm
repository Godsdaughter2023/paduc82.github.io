You can't easily disable or hide those items, but you can protect the VBA code in the master workbook with a password. This will then also hold for copies of the master workbook.

Open the master workbook.

Activate the Visual Basic Editor.

In the Visual Basic Editor, select Tools > VBAProject Properties...

(If you have changed the name of the VBA project to something else, you'll see the changed name instead of VBAProject)

Activate the Protection tab.

Tick the check box "Lock project for viewing".

Enter the password that will be needed to see the VBA code in the Password and Confirm password boxes. Make sure that you remember this password!

Click OK.

Switch back to Excel, and save the workbook.

(Warning: the above will only be effective if the workbook actually contains VBA code)

Next time you open the workbook, you will only be able to view and edit the VBA code in it if you provide the password:

If you create the NewWb  directly from the Master work book, then it will inherit the VBA protection which you  previously applied to the Master workbook. As an example, if the new workbook (destWB) is created as shown in the following code, then it will inherit the VBA protection status of the Master workbook (srcWB):

'=========>>
Option Explicit

'--------->>
Public Sub Tester()
    Dim srcWB As Workbook, destWB As Workbook
    Dim sFilename As String, sStr As String
    Const sPath As String = "C:\MyFolder\"                                  '<<=== Change
    Const srcWbName As String = "ToBeMaster.xlsm"                  '<<=== Change
    Const destWbName As String = "Distribution.xlsm"               '<<=== Change

    Set srcWB = Workbooks(srcWbName)

    sStr = Application.PathSeparator
    If Right(sPath, 1) = sStr Then
        sFilename = sPath & destWbName
    Else
        sFilename = sPath & sStr & destWbName
    End If

    srcWB.SaveCopyAs (sFilename)
    Set destWB = Workbooks.Open(sFilename)
    With destWB
        '\\ your code
        .Close SaveChanges:=True
    End With

End Sub
'<<=========

If, however, the NewWb already exists and was not created in analogous manner from the Master workbook, then you will have to open it and apply the VBA protection manually as shown by Hans.
