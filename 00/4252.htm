Just wanted to add my experience to this...

 

I faced the same issues while installing and uninstalling the PC Suite. I even tried to use the msi file but it did not go through. So these are some steps I followed:

  1. Open task manager. 
  2. Add a column "i/o writes"
  3. run the msi file
  4. At a point when the installation freezes, I check the task manager for a file called msiexec which has 0 (zero) I/O writes.

  6. I continue to kill the msiexec process which has 0 I/O writes until the installtion is complete.

Finally this worked. I am able to sync my nokia with PC suite.

bob jones aka: cybertaz say:

I can't offer a detailed technical explanation or a fix by way of a Preference or settings change, but the effect appears to be caused by the fact that many basic text editors are based on Rich Text Formatting (RTF). Word is based a more "sophisticated" HTML formatting foundation than many other programs, which is the default when text content is pasted. Apparently something is lost in the translation from the OS X Clipboard .

To retain the source formatting use Edit> Paste Special Formatted Text (RTF) rather than pasting.

[BTW: Just to clarify, Copy/Paste is not the equivalent of Import :-) If you open a TextEdit or other RTF file in Word (rather than in TextEdit, for example) then copy/paste to another Word document the alteration doesn't occur because Word's text import filters are allowed to do their job when the file is opened. When you copy/paste (text or graphics) from another source the import filters are bypassed.]
