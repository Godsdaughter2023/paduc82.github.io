Symptoms

 Access is denied. Risposta di Active Directory: 00000005: SecErr: DSID-031521D0, problem 4003 (INSUFF_ACCESS_RIGHTS), data 0

Cause

In my case the user "Exchange Trusted Subsystem" have the correct permissions, the problem was attributable to owner of the "Distribution Group" that did not have the correct permissions

Resolution

  1. Open Active Directory Users and Computers.
  2. Click View, and then click Advanced Features.
  3. Right-click the OU that contains the distribution list, and then click Properties on the right object.
  4. In the Security tab, click Advanced.
  5. In the Owner tab, click Other users or groups...
  6. type the correct owner in the form and click OK.
  7. Click OK.

Note

I found this Microsoft article on the same mistake, and I proceeded to report the solution relative to my case:

https://support.microsoft.com/en-us/kb/2983209

suzanne s. barnhill say:
Part of your problem is likely the hanging indent (and associated tab stop) in the TOC styles. See http://WordFAQs.mvps.org/NumberAlignment2007.htm, which explains the problem. If a tab character in the entry is occurring to the right of where the hanging indent is, then the tab is going to go to the tab stop for the page number, and the page number is then going to run over to the next line. You must have a hanging indent/tab stop in each TOC style in a position where it can be used by that first tab character.
