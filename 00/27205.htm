a. user say:

The "field language" doesn't really let you validate a response in that way, primarily because it has no looping constructs. You might, for example, be able to use a set of fields /after/ the ASK to pop up another ASK if the original response was not one of the ones you wanted, but there's actually no way back - once the user has responded to the second dialog box, the merge is going to continue.

So for anything even slightly more complicated, you will probably be better off using VBA and a Userform to do all the user interaction and validation that you need, then either proceed with the merge or cancel it.
