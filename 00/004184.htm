agrob80 say:

Hi Sylvie

Thanks for your reply. I found the solution in the meantime.

Some weeks ago, we set the following reg key for outlook for all users:

HKCU\S oftware\Microsoft\Office\16.0\Outlook\Options\General

DelegateWastebasketStyle (DWORD) , Wert 4

With this setting, if a user is deleting an Outlook item in another Users (or shared) mailbox, it moves the item to the deleted items folder of the origin mailbox, not in the delete items folder of the user which deletes the file. Because for Shared Mailboxes, Delegated User had full access rights, so ther was no problem.

But this setting did now result in the following problem. when the user only has editor rights on a calender, but no rights to the deleted folder of the same mailbox, calender items can not be deleted. so what i did for testing is to add editor rights for the user on the deleted items folder with powershell:

Add-MailboxFolderPermission "Mailbox:\Deleted Items" -user "USER" -AccessRights Editor

User had to close and open outlook and was then able to delete calendar items again. I'll check further with other users/mailboxes if this fixes the issue on all those affected calendars.

Delete index - Microsoft Community

gsusser1 say:

Delete index

How do I delete an index and all traces of it in Word 2011?  Thanks.
bob jones aka: cybertaz say:

To delete the Index, itself, just select & delete it as you would any other body of content.

Also, you may not need a macro. If there are no other types of fields in the document you can easily remove the { XE "indexitem" } code entries as well as the Index field by using Advanced Find & Replace.

  1. Make sure you have the non-printing characters displayed [Command+8 or use the ¶ button on the Standard toolbar]
  2. Make sure that the Index Field Code is displayed [click in the Index, press Shift+F9 (depending on keyboard type you may need to include the fn key)]
  3. Type ^d into the Find what box
  4. Leave Replace with box blank
  5. Click Replace All.

If there are other types of fields: Depending on the length of the document, number of index entries, how many non-XE fields etc. you still may be able to do as above but using the Replace & Find Next buttons rather than Replace All.
