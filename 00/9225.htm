andreas killer say:
A simple way is to use the CommandBars object and create a menu "on the fly" as in Xl2003 and before. Such things are shown in Xl2007 and above in the Add-Ins tab automatically.

Make a new file, insert e regular module (I assume that the name of that module is "Module1") and add this sub:

Sub Main()
  MsgBox "Hello World"
End Sub

Then add this code into the code module of "ThisWorkbook":

Sub Workbook_Open()
  Dim Menu As CommandBar
  Dim Ctrl As CommandBarButton
 
  Set Menu = Application.CommandBars.Add("MyAddIn", Temporary:=True)
  With Menu
    .Visible = True
    Set Ctrl = .Controls.Add(Type:=msoControlButton, Temporary:=True)
    With Ctrl
      .Caption = "MyButton"
      .OnAction = "Module1.Main"
      .FaceId = 343
      .Style = msoButtonIconAndWrapCaptionBelow
    End With
  End With
End Sub

That's all, your interface is ready.

Andreas.

graham mayor say:
If you are merging to different documents - see http://www.gmayor.com/individual_merge_letters.htm Then setup the printer tray allocations in the merge document page setup and the individual merge documents should follow that setup.

Word does not create 'PCL calls'. The printer driver is what creates the output and if that is a PCL driver then PCL is what is created. If a Postscript driver then the output is Postscript. The issue of printer tray allocations is printer driver specific and frequently an issue when printer drivers are changed. There is some information at http://www.gmayor.com/fax_from_word.htm and for greater depth fellow MVP Jonathan West has done some great work on controlling printers from VBA. See http://pubs.logicalexpressions.com/Pub0009/LPMArticle.asp?ID=183 (and associated articles).

macropod say:

Hi JLInstructor,

Word relies on the attached printer driver for info about what commands direct the output to different trays. A document configured for one printer's setup won't necessarily work on another printer, even when they're the same make & model if the printer settings differ. This really has nothing to do with PCL vs Postscript.
