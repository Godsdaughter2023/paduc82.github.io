You might like to test the following to create a progress bar on a Userform. It is reasonably generic and you should be able to apply it to any processing that uses loops.

3.    Drag side and bottom of Userform to required size to contain the progress bar.

 

4.    In the Userform Properties, change Caption to ‘Processing – Please wait.’ (Without the single quotes.)

 

5.    Using the Controls Toolbox, insert a Frame in the Userform and drag its size out to suitable height for progress bar and the maximum Width required for the progress bar when 100% processing completed.

 

6.    In the Frame Properties set the following:

a)    BorderStyle to 1 – fmBorderStyleSingle.

b)   BorderColor to Black.

c)    Caption to blank. (Delete the default caption)

d)   Make a note of the Width and Height of the Frame. (Required in steps 10c and 10d)

 

7.    Insert a label inside the frame. At this point just drag its size out to something smaller than the frame. (Size of label at this time does not matter; it is set later.)

 

8.    Note: When a label is placed inside a frame that its Top and Left position is relative to the Frame; not the Userform.

 

9.    Tip: After performing the following steps, you may have difficulty in selecting the correct control with the mouse for setting/adjusting properties. 10. In the Label Properties set the following:

a)    Top:                0 (Zero).

b)   Left:                0 (Zero).

c)    Height:             Same Height as frame you made a note of in step 6d.

d)   Width:              Same Width as frame you made a note of in step 6d.

e)    BackColor:       &H000000FF&  (Red if selected via the palette)

f)    BorderColor:    &H000000FF& (Red if selected via the palette)

g)   ForeColor:       &H80000012& (Black if selected via the palette)

h)    Special Effect: 0 – fmSpecialEffectFlat

i)     TextAlign:        3 - fmTextAlignRight

 

11. You will now have the Frame and Label the same size. When the code runs, the label Width is adjusted down to the percentage of the code that has run and the actual percentage is displayed at the right of the label. The frame remains at the maximum size so the user can see the proportion of code that has run.

 

12. Copy the following code (Between the asterisk lines) into a standard module andnote the comments.

 

'************************************************************************
'Following lines must be at top of module in Declarations area
Public subToRun As String  'Available in all modules
Dim initProgressWidth As Double 'Available in all subs this module only.

 

Sub ShowUserForm()    'Run this sub to start demo
  'Assign the name of the sub which will be run
  'while progress bar is operating to the SubToRun variable.
  'subToRun variable is used in UserForm_Activate event.
  subToRun = "TestLoop"
  UserForm1.Show
End Sub

 

Sub TestLoop()
  Dim rng As Range
  Dim cel As Range
  Dim Counter As Long
  Dim PortionDone As Double
  Dim totIterations As Long
 
  'Initialize variables.
  Set rng = ActiveSheet.Range("A1:Z10000")
  initProgressWidth = UserForm1.LabelProgress.Width
  totIterations = rng.Cells.Count
 
  'Loop through cells.
  For Each cel In rng
      cel.Value = Int(Rnd * 1000)
      Counter = Counter + 1
     
      'Calculate the percentage completed.
      'Use the mod function to limit the number of calls
      'to update the progress bar otherwise may be too slow
      'processing all of the data.
      If Counter Mod 100 = 0 Then
        PortionDone = Counter / totIterations 'This is a decimal fraction
        'Call subroutine that updates the progress bar.
        Call UpdateProgressBar(PortionDone)
       
      End If
  Next cel
 
  'Unload the UserForm. (Optional. Comment out if you want to leave displayed.)
  Unload UserForm1
End Sub

 

Sub UpdateProgressBar(PortionParam As Double)
    With UserForm1
        'Set the width of the Label.
        .LabelProgress.Width = initProgressWidth * PortionParam
       
        'Express decimal fraction as percent.
        .LabelProgress.Caption = Format(PortionParam, "0%")
    End With

    'DoEvents required so that Userform updates.
    DoEvents
End Sub
