Regarding the script, I've never used powershell before and I'm not sure how to use it correctly. 

Just Paste it into the PowerShell Command line and press Enter.  If you prefer you can pipe it to clip and then Paste the result of that into a Notepad window.

cd \

LS -Re -Fi * -Fo -EA:SilentlyContinue | where LastWriteTime -GE $This_Hour | sort LastWriteTime | ft LastWriteTime, Length, FullName  -AutoSize -Wrap

Note that there are 3 lines, the last line wraps in here.   Piping to clip just means to append  | clip  at the end of the last line.  It's the first and last lines which are important.  The second and the where filter stage in the last line are just a way of optimizing so you don't have to make a list of everything on your drive.  In fact, it may be that this technique depends on the Date and Time format I use being sortable.  ISO date format is yyyy-MM-dd   and ISO time format is HH:MM:ss.   In fact, it might even be worthwhile trying that with an American style date and time (unsortable) anyway in case it can be automatically converted.   E.g. if you don't get the desired range try substituting this for the second line:

$This_Hour = Get-Date -Hour ((Get-Date).Hour -1)  -Format "yyyy-MM-dd HH:MM:ss"

and just in case I put the $This_Hour in the where expression in parentheses.   So here is the whole thing again revised as described

cd \

$This_Hour = Get-Date -Hour ((Get-Date).Hour -1)  -Format "yyyy-MM-dd HH:MM:ss"

LS -Re -Fi * -Fo -EA:SilentlyContinue | where LastWriteTime -GE ($This_Hour) | sort LastWriteTime | ft LastWriteTime, Length, FullName  -AutoSize -Wrap

That does speed up the output phase but it is still slow because it has to list every file on the drive and then compare the timestamp in the filter.

Sub SquareActiveChart()
  'Squares the chart object in a sheet
  If ActiveChart Is Nothing Then
    MsgBox "Select a chart and try again."
    Exit Sub
  End If
  If Not TypeOf ActiveChart.Parent Is ChartObject Then
    MsgBox "Select a chart in a sheet and try again."
    Exit Sub
  End If
  With ActiveChart.Parent
    If .Width > .Height Then
      .Width = .Height
    Else
      .Height = .Width
    End If
  End With
End Sub

Sub SquareActiveChartArea()
  'Squares the plot area inside a chart based on the axis sizes
  Dim XSize As Double, YSize As Double
  Dim W As Double, H As Double
  If ActiveChart Is Nothing Then
    MsgBox "Select a chart and try again."
    Exit Sub
  End If
  With ActiveChart
    With .Axes(xlValue)
      YSize = .MaximumScale - .MinimumScale
    End With
    With .Axes(xlCategory)
      On Error Resume Next
      XSize = .MaximumScale - .MinimumScale
      XSize = UBound(.CategoryNames)
      On Error GoTo 0
    End With
    With .PlotArea
      H = .Width / (XSize / YSize)
      W = .Height / (YSize / XSize)
      If W > .Width Then
        .Height = H
        .Width = .Height / (YSize / XSize)
      Else
        .Width = W
        .Height = .Width / (XSize / YSize)
      End If
    End With
  End With
End Sub

Sub SquareActiveChartGrid()
  'Squares the plot area inside a chart based on the major grid
  Dim XSize As Double, YSize As Double
  Dim W As Double, H As Double
  If ActiveChart Is Nothing Then
    MsgBox "Select a chart and try again."
    Exit Sub
  End If
  With ActiveChart
    With .Axes(xlValue)
      YSize = (.MaximumScale - .MinimumScale) / .MajorUnit
    End With
    With .Axes(xlCategory)
      On Error Resume Next
      XSize = (.MaximumScale - .MinimumScale) / .MajorUnit
      XSize = UBound(.CategoryNames)
      On Error GoTo 0
    End With
    With .PlotArea
      H = .Width / (XSize / YSize)
      W = .Height / (YSize / XSize)
      If W > .Width Then
        .Height = H
        .Width = .Height / (YSize / XSize)
      Else
        .Width = W
        .Height = .Width / (XSize / YSize)
      End If
    End With
  End With
End Sub

sharonmary say: Hello "flowers",

OK, I think I now have a better understanding.

When you open a document and edit it and then do a Save As, Word will automatically put the current document name in the Filename: box of the Save As window. So, when you press on Save it is saving the version of the document you have open (with the changes applied) and not a new version.

In that case, you will not get the warning box because you have the document open for editing; in this case Save As has the same effect as Save.

Try the following; it is a more clarified version of the above and will allow you to see the whole process in one exercise:

Create a document called Flowers
Type a few characters and Save it

Make some changes and then do a Save As
Change the name in the Filename: box to Flowers2

Close the document Flowers2

Open the document called Flowers
Type a few characters and do a Save As
Type the name Flowers2 in the Filename: box

Now, because there is already a saved version of Flowers2 you will surely get the message that Flowers2 already exists and you will have the choice to Replace existing file or Save changes with a different name.

- If you click on OK it will overwrite Flowers2
- If you change the name it will save it as a new document with the new name incorporating the edits.

I'm sure that will clarify. If not ... I will eat my hat!  :-)
