a. user say:

Hi Jenny,

I will be glad to help you with the issue you have with the error on the computer while taking the backup.

During backup a VSS process running under NETWORK_SERVICE account calls cryptcatsvc!CSystemWriter::AddLegacyDriverFiles(), which enumerates all the drivers records in Service Control Manager database and tries opening each one of them. , The function fails on MSLLDP record with "Access Denied" error.

The binary security descriptor for the record is located here:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\MsLldp\Security

 

It should be modified.

Follow the below steps to know how to add access rights for NT AUTHORITY\SERVICE to MSLLDP service:

1. Open Command Prompt and type SC sdshow MSLLDP.

You'll get something like below (SDDL language is documented on MSDN):

 

D:(D;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BG)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SY)(A;;CCDCLCSWRPDTLOCRSDRCWDWO;;;BA)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SO)(A;;LCRPWP;;;S-1-5-80-3141615172-2057878085-1754447212-2405740020-3916490453)S:(AU;FA;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;WD)

2. Type SC sdshow MUP

You'll get:

 

D:(A;;CCLCSWRPWPDTLOCRRC;;;SY)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BA)(A;;CCLCSWLOCRRC;;;IU)(A;;CCLCSWLOCRRC;;;SU)S:(AU;FA;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;WD)

. Take NT AUTHORITY\ SERVICE entry, which is (A;;CCLCSWLOCRRC;;;SU) and add it to the original MSLLDP security descriptor properly, right before the last S:(AU... group.

 

4. Apply the new security descriptor to MSLLDP service :

 

sc sdset MSLLDP D:(D;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BG)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SY)(A;;CCDCLCSWRPDTLOCRSDRCWDWO;;;BA)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SO)(A;;LCRPWP;;;S-1-5-80-3141615172-2057878085-1754447212-2405740020-3916490453)(A;;CCLCSWLOCRRC;;;SU)S:(AU;FA;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;WD)

5. Check the result:

>accesschk.exe -c mslldp

mslldp
  RW NT AUTHORITY\SYSTEM
  RW BUILTIN\Administrators
  RW S-1-5-32-549
  R  NT SERVICE\NlaSvc
  R  NT AUTHORITY\SERVICE

 

6. Run the backup and check.

Thank you.

aj don say:

Hello,

We appreciate your post in our Community forum page. Let us explain to you what the prompt is about. The prompt "The system is entering connected standby" doesn't indicate that the system is going to sleep. It is an indication that the system is entering a power saving mode like how your smartphone functions, where-in the screen turns off when not being used but the machine is still active, this is also called the Modern Standby. In this mode, the Surface Book 2 will still get notifications, continue downloads and all active tasks will remain at work while saving power until the 2 hour time frame sets in and it enters the Sleep/Hibernate mode. The Modern Standby requires little to no time to be back on, compared to Sleep and Hibernate. This function was an expansion of the Connected Standby from Windows 8.1.

We hope this would clarify your concern.
