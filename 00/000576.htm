I just ran into this problem, and this was the first result on Google. I figured this out, and felt I should share.

I had a folder full of files I had encrypted on my storage drive using Windows 10. When I copied them to my flash drive. Windows 10 maintained the encryption, and still showed me those attributes when I viewed the flash drive.

When I took that same flash drive out and plugged it into a machine running an older version of Windows (in this case, Windows 8.1 Professional) the files did not list as encrypted. Instead, they showed with the .PFILE extension, and Windows 8.1 had no idea what to do with them. (It offered to search the Store for something to handle .PFILEs, which isn't helpful.)

The solution, once I figured this out, was simply to plug my Flash Drive back into my Windows 10 computer that had created the files, then right-click on the folder, select [Advanced], and un-check the box to "Encrypt contents to secure data." Doing this took a few minutes while the changes were applied, and after that, my Windows 8.1 machine (and presumably all other machines) could read the files without issue.

I'm not sure what the root cause is for the confusion, but I would guess that Windows 10 changed the way that encrypted files are stored on disk somehow compared to prior Windows versions. In either case, I figured I'd leave this here in case anyone else runs into something similar. I'm really curious about what would have happened if I had another Windows 10 machine to test the encrypted files on, but I've just got my own two machines here to play with.
