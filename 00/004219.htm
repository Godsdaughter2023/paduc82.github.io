scottgem say:
OK, the error message is telling you what the problem is. You are using a Multi-value field for Ministry and that can't be used in a WHERE clause. Most developers shy away from MVFs because of problems like these. What an MVF does is create a many to many relationship. It makes for a nice checkbox interface, but makes it harder for using the data. You can try using the .Value property ([Ministry].Value) and see if that helps.

Or you can change the structure to the traditional method of doing many to many. Add a Junction table:
tjxContactMinistry
ContactMinistryID (PK Autonumber)
ContactID (FK)
MinistryID (FK)
You would then add ministries in a subform.

I might also suggest changing your form to a split form. A split form shows a datasheet view of the data along with a single form view. The advantage is that you can use datasheet filtering to drill down to the records you want.

laura rooke say:

This is a time when you need to change the setting on your device to have the PC treat it as a disk drive rather than connect it via ActiveSync.

Go to Start, Settings, Connections, USB to PC - select the Disk Drive option.

Don't forget to change it back to ActiveSync when you are finished with this option.
