To be absolutely sure that the view is the same for all documents, you have make use of a couple of macros:

Sub AutoOpen()
ActiveWindow.View.Type = 3  'Print Layout view
With ActiveWindow.View.Zoom
     .PageColumns = 1
     .Percentage = 100  'you can change the zoom if you wish
End With
End Sub

Sub AutoNew()
ActiveWindow.View.Type = 3  'Print Layout view
With ActiveWindow.View.Zoom
     .PageColumns = 1
     .Percentage = 100  'you can change the zoom if you wish
End With
End Sub

Save the macros to the Normal template. See http://www.gmayor.com/installing_macro.htm.

macropod say:

OK, lets start at the beginning.

 

Suppose you have a mergefield named 'BRONZE', which you insert via 'Insert Mergefield'. You should see:
«Bronze»

If you select the field and press Shift-F9, you should see:
{MERGEFIELD BRONZE}
You can add a switch to that to control the output format, thus:

{MERGEFIELD BRONZE \# "$,0.00;($,0.00)"}

 

For the first calculation, insert another copy of the field. Select the field and press Ctrl-F9 to enclose it in another field, thus:

{ «Bronze» }

Then edit the field so you end up with:

{=«Bronze»*0.13 \# "$ ,0.00;($ ,0.00)"}

Press F9 to update the field. At this stage it may or may not return '"!Syntax Error, «'. Don't worry about that. If you select the field and press Shift-F9, you should see:
{={MERGEFIELD BRONZE }*0.13 \# "$ ,0.00;($ ,0.00)"}

 

For the second calculation, you could simply copy the entire field from the first calculation and change 0.13 to 1.13, so you end up with:
{={MERGEFIELD BRONZE }*1.13 \# "$,0.00;($,0.00)"}

 

Note: You cannot type the field braces or chevrons. They must be created via Ctrl-F9 or by mergefield insertion.

 

When you execute the mailmerge, by clicking on 'Finish & Merge', the fields will all update with the correct calculations.
