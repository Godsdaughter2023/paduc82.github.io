el_loco---01 say: There may be a number of reasons why this is happening:

1) H264 in N81 is only at 15 fps

2) Resolution should be maximum QVGA 320x240

3) You have to use Baseline profile not Main

4) The version of the AVC codec (h264) has to be supported (i.e. Sorenson H264 is not supported)

Post the properties of the video so I can help you further. IF you don't know how to get the full properties of the video check this site to find out how to do it:

http://portablevideo.blogspot.com/2008/03/how-to-check-your-video-properties.html

drewkerwin say: When you say "use baseline profile" are you referring to the encoding?

el_loco---01 say: Yes.

H264 (and many codecs) have different Profiles and Levels. Not all profiles and Levels are supported in all devices.

H264 has at least 2 Profiles with many levels. I use Baseline Profile Level 1.1 for the most part and it works.

Not all encoders make it easy to set the Profile Level.

Check out how to do it in SUPER here:

http://portablevideo.blogspot.com/2008/03/using-super-encoder-with-nseries.html

The settings there are for N95 so they may not work in N81 you would need to lower some of the settings, but if you scroll down you will see how to set h264 properties.

greginmich say:

If you’re using a third-party AV app, and you’re experiencing this failed Windows Defender definition update issue, then the quickest fix (workaround actually) would be to just force Windows Defender to turn off completely with the Group Policy switch:

http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

See the answer of

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_updating/windows-defender-conflict-in-windows-10-with-other/36fc885f-d2b7-4ca4-b2c8-c91cd9533a7b

And also notice this statement in the reply of DisableAntiSpyware DWORD in the Windows Defender registry key:

This should be set to 0x1.  If you don't this may be why you are still trying to get Windows Defender updates. 

…and then run the registry query that he suggests for determining if this is actually the issue:

reg query "hklm\software\microsoft\windows defender" /v DisableAntispyware

If you get a return value of 0x0, then Windows Defender is enabled in the Windows Defender key when it should be disabled. But if you then just disable Windows Defender in the Policies registry key by running the Ten Forums Turn_Off_Windows_Defender.reg file (see above), this should override the faulty setting in Windows Defender’s own registry key. But this solution is only a workaround, since it doesn’t actually correct the problem with the entries in the Windows Defender key.

In order to actually fix this issue, you would have to change the value of the DisableAntiSpyware and DisableAntiVirus DWORDs in the Windows Defender registry key to the correct value of 1. However, this would only be recommended for people who've had some experience with editing the registry, since it requires taking ownership of the Windows Defender key. I mentioned the registry fix in my long reply here:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/error-code-0x8007139f-while-starting-windows/10632a04-9a62-47ab-92d0-026d73bca899
