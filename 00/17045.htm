Jonathan,

No you're not doing anything wrong. Although Project has several text fields, all except for the Notes field are limited to 256 characters.

Project is very weak in handling text. It is after all a scheduling application, not a word/text processor.

The Notes field was designed to accommodate and even do rudimentary formatting on significant textual information for tasks, resources and assignments. If for some reason you can't use that field, then you will either have to limit the verbiage or spread it across multiple spare text fields.

Sorry it's not the answer you wanted, but it's the way the application works.

If you trust the source of this document, click yes'.  Then when I click on yes; it gives the same error; 'the open xml file cannot be opened because there are problems with the contents or file name might contain invalid characters . Details: Microsoft Office cannot open this file because some parts are missing or invalid.' 

How can I recover it? When I check the sizes of the files, the new copy that I cant open is 160 byte, means that my words could be in there. I cant open it. Is there a way???

graham mayor say:
The file beginning ~$ is a lock file - a normally hidden file created in the document folder every time a document is opened. It is not a copy of the document. If the document normally is closed, lock files such as this are deleted as the file no longer needs to be locked. When a crash causes Word to close unnaturally the lock files pertaining to the open documents, to the normal template and to any add-in templates are left where they can cause future problems - see http://www.gmayor.com/what_to_do_when_word_crashes.htm
