andre da costa say:

Normally patch Tuesdays can take longer than usually because the Windows Update servers are being overwhelmed. You can try the following to see if it resolves the problem:

1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Typethe following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

5. Type Exit in the Command Prompt to close it and then restart the computer.

Restart then try updating again.

    This is a specific design limitation in Windows 7.  Basically, Explorer now only remembers the size and position of the very last window closed, and stacks subsequent windows.  People seem ready to lynch Microsoft over this move, but it is what it is.  I suspect maybe Microsoft carved out the window positioning/sizing functionality because it was not reliable long-term in Vista.  Maybe it will be returned in a future version, who knows?

    The good news is that there's a nice workaround for this right now:  Check out the 3rd party freeware tool called ShellFolderFix:

    http://www.sevenforums.com/customization/40916-shellfolderfix-manage-folder-window-positions-size.html

    This does a nice job of managing folder positions.  I have been using it on both my 64 and 32 bit Windows 7 systems and find it to really help with Explorer window positioning.  It also allows you to turn on display of the path in the folder title bar, which I find a great enhancement.

    regedit = how to create a .reg file - Microsoft Community

    daveowitz say:

    regedit = how to create a .reg file

    Recently upgraded to 4Gb by installing new chips. Purchased Encyclopedia Judaica about 14 years ago on CD. Opening EJ brings up a 2004 error - the old program can not deal with >512 RAM. I have the regedit verbiage fix from EJ's successor. But, writing it into Notepad (or Word, or whatever, saving as a .reg file, then attempting to import (or double-clicking the file) brings Regedit error message "this isn't a .reg file, so go away". What software writing program will create an acceptable .reg file which may be imported (saying "don't tell EJ that there's more than 512 available for it to use")? Thanks for any help.

    sc tom say:


    You can use just about any text editor (Notepad works just fine).
    Here's an example for how to delete a key:

    _________________________________
    Windows Registry Editor Version 5.00

    [-HKEY_CLASSES_ROOT\XXX]
    @="XXX Software"
    _________________________________

    If you were to save that with a .reg extension, right-click on it and pick "Merge," it would delete that key ( the minus sign tells it to do that). You don't need the lines- I just put them there to separate it from the rest of the text.

    This example would add these keys:

    __________________________________
    Windows Registry Editor Version 5.00
     [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\SharedDlls]
    "C:\\windows\\system32\\ZoneLabs\\isafeif.dll"=dword:00000001

    [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\SharedDlls]
    "C:\\windows\\system32\\ZoneLabs\\vetredir.dll"=dword:00000001

    [HKEY_CLASSES_ROOT\.m2f]
    @"="M2.Filter"
    ___________________________________

    If the key is already there that you want, you can create a .reg to change the values and merge it. Or if it's not there, just create one, right-click on it and pick "Merge". You can always create it as a .txt file, then change the extension when you're done.

    SC Tom
