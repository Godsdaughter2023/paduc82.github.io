jay freedman say:

You'll need a macro to do this. The macro can be selected as the "exit macro" in the properties dialog of the form field in which the number of rows is specified, so the macro will run when the cursor leaves that form field.

In the code of the macro, you'll have to turn off the forms protection, insert the requested number of rows, and then turn the protection on again.

In order for the macro to know where to insert the rows, you'll have to use some method of locating the correct table. For example, if the table is the first (or only) table in the document, you could have the macro add rows to ActiveDocument.Tables(1). Or you could insert a bookmark named "TestTable" around the table, and the macro would add rows to ActiveDocument.Bookmarks("TestTable").Range.Tables(1).

If you need more help with the macro, post back.

doug robbins - mvp office apps & services (word) say:

Assuming that the formfield into which the user enters the number of tests had the bookmark name of numtests and that you have a one row table in the document into which you insert a bookmark named "Table"

Test Number Type of Test

If you run a macro containing the following code on exit from the numtests formfield, it will add the required number of rows to the table and insert the test numbers into the first cell in each of the rows.

Dim atable As Table
Dim arow As Row
Dim i As Long
With ActiveDocument
    Set atable = .Bookmarks("Table").range.Tables(1)
    If IsNumeric(.FormFields("numtests").Result) Then
        .Unprotect
        For i = 1 To .FormFields("numtests").Result
            Set arow = atable.Rows.Add
            arow.Cells(1).range.text = i
        Next i
        .Protect wdAllowOnlyFormFields, NoReset
    Else
        MsgBox "The entry must be a number."
        .FormFields("numtests").range.Select
    End If
End With

If you do not know what to do with the above code, see the article "What do I do with macros sent to me by other newsgroup readers to help me out?” at:

Ping www.msn.com

Step 1: One of the components of the Internet connection on your computer is a built-in set of instructions called TCP/IP. TCP/IP can sometimes become damaged or corrupted. Let’s reset the winsock entries and TCP/IP stack and to troubleshoot the issue further.

Follow the steps:

a. Press Windows Key + X together and select Command Prompt (admin).

b. Type the following command and hit Enter after each:

netsh winsock reset catalog (reset winsock entries)

netsh int ip reset reset.log hit (reset TCP/IP stack)

c. Reboot the computer and check.

Step2: Flush your computer's DNS cache.

a. From Start screen, type command prompt.

b. Right-click on Command Prompt and choose Run as Administrator.

c. At the command prompt, type the following commands. Press Enter after each command.

ipconfig /flushdns
ipconfig /registerdns
ipconfig /release
ipconfig /renew

Okay....how about a little more information.

1. What version of Project and Office are you using?

2. Are they fully updated with the latest Public Updates?

3. How large is your Excel plan (e.g. number of tasks, resources, etc.)

In general terms, you can try the import Wizard in Project but that can be a little finicky with regard to formatting. Also, know that Project is a scheduling application that uses a working time calendar (e.g. day is by default 8:00 to 5:00) while Excel dates are based on a 24 hour clock (e.g. 12/9/15 is really 12/9/15 12:00:00 AM). Depending on the field, Project may or may not convert the date for you.

A common problem users have with an import of data from Excel to Project is that they tend to import too much information. All Project really needs is a task description, a start date, estimated duration and links between tasks. Project calculates everything from there

If you have timescaled data in Excel (e.g. Bob was assigned to task A and worked 4 hours Monday, 3 hours Tuesday, 4 on Wednesday, etc.), Project will only take the last entry (i.e. you cannot import timescaled data).

That's just for starters, depending on how detailed your Excel plan is, the conversion can get more interesting.

Actually the screen should ask you if you want to create a new map or use an existing map. A "map" simply pairs Excel columns with Project fields. For example, let's say you have a set of columns in Excel representing the task name, start date, finish date, and duration. In order to map those into Project you would map the Excel task name into the Project Name field, the Excel start date into the Project Start field, and so forth. The Global file contains a set of standard field mappings which may or may not be appropriate for your Excel format. When I import, I always create a new map of just those fields I want to import.

Okay, so you have 200 tasks, what data do you want to import?
