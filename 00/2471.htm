ken sheridan say: Because the Workplacename column is in the referencing WorkplacesTable, not the form's underlying ContactsTable you cannot restrict the query on this column.  You need to include a subquery in the WHERE clause and restrict the outer query to those rows where the WorkplaceID is in the rows returned by the subquery:

strsearch = _
    "SELECT * FROM ContactsTable " " & _
    "WHERE ((LastName LIKE ""*" & strText & "*"") " & _
    "OR (FirstName LIKE ""*" & strText & "*"") " & _
     "OR (MobilePhone LIKE ""*" & strText & "*"")) " & _
     "OR (WorkplaceID IN" & _
         "(SELECT WorkplaceID " & _
         "FROM WorkplacesTable " & _
         "WHERE Workplacename LIKE ""*" & strText & "*""))"

The use of the LIKE operator with wild cards characters does assume that there will not be a NULL at the relevant column position in any row in the table.  As NULL can never equate to anything, rows where there is a NULL at the relevant column position would not be returned.  A reliable method of allowing for NULLs is:

WHERE (SomeColumn = <some parameter>
OR <some parameter> IS NULL)
AND (SomeOtherColumn = <some other parameter>
OR <some other parameter> IS NULL)
AND etc

Also, when assigning the value of the txtSearch control to the strSearch variable you might need to use:

    strText = Me.txtSearch & ""

to allow for the control being NULL.  If you do test for a NULL parameter as above, however, you would be better omitting the strSearch variable from the code and referencing the txtSearch control directly.  Alternatively test for the strText variable being a zero-length string.

  2. To Zip a file: In Windows Explorer Right click on the selected file and select Send to -> Compressed (zipped) folder). By holding the Ctrl key and left click once on each file, you can select multiple workbooks before right clicking over one of the selections to send to a compressed file and they will all be included into the one Zip file.
  3. Do not use 3rd party compression applications because I cannot unzip them. I do not clog up my computer with 3rd party apps when there are perfectly good apps supplied with windows.
  4. Go to this link.  https://onedrive.live.com
  5. Use the same login Id and Password that you use for this forum.
  6. Select Upload in the blue bar across the top and browse to the zipped folder to be uploaded.
  7. Select Open (or just double click). (Be patient and give it time to display the file after initially seeing the popup indicating it is done.)
  8. Right click the file on OneDrive and select Share.
  9. Select "Get a Link" from the popup menu.
  10. Click Copy the link and Paste into your reply on this forum.

Currently, there's no Microsoft official solution to migrate mailbox permissions (Full Access or so) or mailbox folder permissions (calendar access or so).
