Here's my take on this:

 

Sub EditVisibleBlankCells()
    Dim rng As Range, rngVis As Range, rngVisAndBlank As Range
    Dim lnRowStart As Long, lnRowEnd As Long, lnLastRow As Long
    Const lnSCLim As Long = 2 * 8912 'Special cells limit
   
    Application.ScreenUpdating = False
    Application.EnableEvents = False
    Application.Calculation = xlCalculationManual
   
     'Find last row containing data in col DB:
    lnLastRow = Cells(Rows.Count, "DB").End(xlUp).Row
   
     'Specifiy first row containing data:
    lnRowStart = 2
   
    While lnRowStart < lnLastRow
        lnRowEnd = Application.WorksheetFunction.Min(lnRowStart + lnSCLim, lnLastRow)
        Set rng = Range("DC" & lnRowStart & ":DC" & lnRowEnd)
       
         'Get Visible Cells in the current block:
        Set rngVis = rng.SpecialCells(xlCellTypeVisible)
       
         'Get Blank Cells in the visible cells in the current block:
        If Not rngVis Is Nothing Then
            Set rngVisAndBlank = rngVis.SpecialCells(xlCellTypeBlanks)
        End If
       
         'Add the required CODE to the blank and visible cells:
        If Not rngVisAndBlank Is Nothing Then rngVisAndBlank.Value = "CODE"
       
         'Move to next block:
        lnRowStart = lnRowEnd + 1
        Set rngVis = Nothing
        Set rngVisAndBlank = Nothing
    Wend
   
    Application.ScreenUpdating = True
    Application.EnableEvents = True
    Application.Calculation = xlCalculationAutomatic
End Sub

Hi Anish,

Sorry for the late response.

If you are using the Outlook 2016 client and your search shows no results, one thing that could be happening is that your Outlook data file is not actually checked off for indexing. It is important to have the Outlook Search Index working correctly, as it can send the entire email client into a spiral of malfunction. It is quite easy in Outlook 2016 to check if your mailbox is being indexed. To check what you’re indexing in Outlook 2016 just follow this specific path:
1.    Navigate to the search options by clicking File>Options> Search.
2.    Under Indexing Options, click Indexing Options> Modify.

This is where you can make sure that Outlook has been selected for indexing. In previous versions of Outlook, you could specify which mailbox or .pst/.ost files within Outlook to index, but in Outlook 2016 you either index Outlook completely or you do not index it at all.
When you check if it is marked for indexing, you should also try disabling and then re-enabling Outlook. To do so, follow these steps:
1.    Deselect Outlook from Indexing
2.    Close Outlook
3.    Wait 5 minutes, check in the task manager that Outlook.exe is not in the processes tab
4.    Now open OL again and go back and select Outlook for indexing.

What’s more, there could be a corrupted pst-file that’s causing the search problems with the Outlook Search Index. To check the file and repair it in Outlook 2016 just search for scanpst.exe file. You can search for this file on your computer by typing in Scanpst.exe into the File Explorer(make sure it is not just searching the folder you are currently in, but choose “This PC” on the left hand side of File Explorer.) If you run this, it will repair any corrupted pst or ost file.
For more information, please refer to:
https://support.office.com/en-us/article/Repair-Outlook-Data-Files-pst-and-ost-64842a0a-658a-4324-acc6-ef0a6e3567cd

Additionally, If nothing else works, you may need to rebuild your index. This is easy to do, it just may take some time. All you need to do is follow these steps to rebuild your Outlook 2016 search index:
1.    Click on File, then Options. Click Search on the left hand side
2.    Now click on the button that says Indexing Options
3.    Click the button that says Advanced
4.    Within the tab that says “Index Settings” click on the button that says “Rebuild”
Note: When the index is being rebuilt, your search will not work. You should wait until the index is done rebuilding. The time it will take for the index to be rebuilt depends on the system you are using, the amount of files that need to be indexed, and file size.
When your index has finished rebuilding, you can go ahead and restart your Outlook searches. This should have fixed your Outlook 2016 search problems.

How to calculate in Excel - Microsoft Community

tomg. say:

How to calculate in Excel

Hi,

Need some help with calculating the following in Excel.  I'm trying to configure a truckload of products so the customer receives an equal number of components as each product is in a set consisting of 2 boxes (Box 1a & 1b) as follows:

Box 1a:  32 boxes per pallet
Box 1b:  9 boxes per pallet

The trucks we use take 22 or 34 pallets in total so the end result should be 22 or 34 pallets with an equal number of Box 1a & b's (part pallets are acceptable but I can't mix Boxes 1a & 1b on the same pallet).

Thanks,
Tom


jasperp say: If x is the number of 1a pallets, then 32x is the number of 1a boxes. Say y is the number of 1b pallets, then 9y is the number of 1b boxes.

You need to solve: 32x=9y and x+y=22 or x+y=34 and you can use Solver for this.

You may need to install the Solver add-in first.
http://office.microsoft.com/en-us/excel-help/load-the-solver-add-in-HP010021570.aspx

Set up a table like this, for example in A1:D4:

TRUCK             Boxes    Pallets    Total boxes
Box 1a                32          1           32
Box 1b                9            1            9
Total pallets                      2

The Total boxes cells (D2 and D3) must contain the formula Boxes * Pallets (e.g. =B2*C2) and Total pallets (C4) sums the number of pallets (=C2+C3).

Now click Data -> Analysis -> Solver

Total pallets (C4) is your target cell, set it equal to 22 or 34.
The changing cells are the pallet cells C2:C3.
Add the constraint D2=D3 (Total boxes 1a = Total boxes 1b).

Click Solve.

jasperp say: An example of finishing the calculation, just in case.

Copy the table from, say A1:D4 to A7:D10. Replace the values in the Pallets cells (C8 and C9) with the formulas:

=FLOOR(C2,1)
=FLOOR(C3,1)

This will give you the number of whole pallets you can deliver. The difference between the Total boxes cells (D8 and D9) will give you the part pallet you need to add.
