young yang msft support say:

Hi Stefan,

Thanks for our efforts and the valuable information. I think we have found the cause.

In hybrid environment, the Autodiscover of the co-existence domain must point to on-premise Exchange. HCW will use Autodiscover service to create Exchange Online organization relationship. Now, the Autodiscover of the new domain brsanalytics.com point to Office 365. So, HCW will create the incorrect organization relationship.

To resolve this issue, I suggest you refer to the steps below:

What is your Exchange 2010 version?

Exchange 2010 SP3 RU1 or later introduced an Autodiscover domain feature. We can specify one of the domains as the “autodiscover” domain. And then use PowerShell to add other domains. Please perform the following steps:

1. Run HCW without the new domain brsanalytics.com. It seems that you have done this step.

2. Open Exchange Management Shell (EMS) from on-premise hybrid server, and run this command:

Set-HybridConfiguration –Domains “domain2.com, domain3.com”, “autod:domain1.com”

Note: For “autod:domain1.com”, please use your old default domain computime.com.mt

For Domains, please include all other domains that you want to use in hybrid.

3. Run Get-HybridConfiguration | fl to check the results. All domains should be included.

References:

Here is a similar thread I handled previously: https://community.office365.com/en-us/f/156/p/287430/878062#878062

Autodiscover domain feature: https://technet.microsoft.com/en-us/magazine/dn249970.aspx  # Hybrid coexistence

So, for your scenario, you just need to run one PowerShell command.

If any error occurs, we will do further analysis.

Thanks,

Young

cristin yan say:

Hi,

 

The acFormatPDF is a  predefined constant member of Access Constants Enumeration. It is defined as below:

    Const acFormatPDF = "PDF Format (*.pdf)"

 

Please type the following code (do not copy and paste) in the VBE environment to see if there is an auto-complete list after typing the dot:

    Debug.Print Access.Constants.acFormatPDF

 

If you cannot see the auto-complete list, please check Options >> Editor >> Auto List Members.

 

Then add the acFormatPDF declaration before DoCmd...

     ' Force to declare a constant.

     Const acFormatPDF = "PDF Format (*.pdf)"

     DoCmd.OutputTo acOutputReport, "rptMonthEndComputerCounts", acFormatPDF, "C:\Temp\ComputerCounts.pdf"

 

OR directly use the constant value

 

     DoCmd.OutputTo acOutputReport, "rptMonthEndComputerCounts", "PDF Format (*.pdf)", "C:\Temp\ComputerCounts.pdf"

 

If no avail, you might try to repair your Office via Control Panel >> Uninstall a program >> Right-click the Office program >> Change >> Repair

 

Best Regards (=^ω^=),

Cristin
