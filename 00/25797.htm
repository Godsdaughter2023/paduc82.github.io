Based on the research, we found a workaround for your reference, please follow the steps below:

  1. Exit the Outlook and make sure this OST file isn't opened in any other program/software.
  2. Open Registry Editor by running regedit in RUN dialog box.
  3. Navigate to the path: HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Outlook, then right click on any blank area of the right-side pane, and then select New -> Expandable String Value.
  4. Name the newly created expandable string value as ForceOSTPath, and then double click on it > On the Edit String dialog, enter your desired path under the Value data box > click

              

         5. Close Registry Editor and restart computer for the changes to take effect.

    brian tillman [outlook mvp 2007-2019] say:
    You usually want to keep the "stop processing more rules" unless you want more than one rule to act on a particular message.  Rules act in the order they are shown in the Rules Manager.  If you have a message that an early rule would delete, you probably don't want a later rule that would move the message to another folder to act.  Likewise, if you have one rule that moves a message to folder A, you probably don't want a later message to move it to folder B as well or you'll have two copies, one in each folder.  Thus, the first rule should have the "stop processing more rules" action so that once processed by one rule, the message won't be considered by later rules.  Outlook applies all rules to every message you receive.  The "stop processing" means that no later rule will apply for that single message, not for all subsequent messages.
