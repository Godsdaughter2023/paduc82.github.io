Workaround For "Unattended Wakeup" Sleep Timeout Bug Now Also - Microsoft Community

timc. say:

Workaround For "Unattended Wakeup" Sleep Timeout Bug Now Also Bugged

There has been a bug in Windows sleep that has existed in both Windows 10 and in previous versions. When Windows wakes from sleep, on some machines it sees the wake event as a remote wakeup event. The result is that the normal power settings are completely ignored, and a specific registry settings overrides the sleep timer.

This link describes the problem:

https://answers.microsoft.com/en-us/windows/forum/windows_8-performance/windows-8-pro-goes-to-sleep-after-60-seconds-of/b1a32492-f9cc-4db9-ad70-987af074fa08?msgId=e37ac74a-6c29-4f1d-82af-ad837dadc371&tab=question&status=AllReplies&page=5

Note that this link is from all the way back in 2012 under Windows 8. However, the same bug still exists in Windows 10. EVERY time Windows 10 has a major update, these registry settings are set back to the default of 120 seconds (2 minutes). The workaround is to use regedit to change the corresponding registry keys to some larger number.

The registry keys are located at:

Computer\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Power\PowerSettings\238C9FA8-0AAD-41ED-83F4-97BE242C8F20\7bc4a2f9-d8fc-4469-b07b-33eb785aaca0\DefaultPowerSchemeValues

As of update 1809 for Windows 10, this workaround no longer works. After waking from sleep, at least one computer that I own goes to sleep after 2 minutes no matter what the power settings say. This registry setting no longer fixes it.

My guess is that another registry setting needs to be edited. Anyone have an idea how Microsoft may have broken this again?

To reinstall preinstalled apps
  1. Download the Reinstall-preinstalledApps.zip PowerShell Script to your PC and copy it to your desktop.
  2. Open an elevated PowerShell window.
    1. Click Start.
    2. Type "Windows PowerShell" in the search bar.
    3. Right-click Windows PowerShell in the results list and click "Run as administrator".
    4. A User Account Control dialog displays. Click "yes" to proceed.
  3. Navigate to the script download folder, which is your desktop if you followed step 1. Your command will look similar to the following example.

    PS C:\Users\Abby>CD Desktop

  4. Temporarily allow unsigned PowerShell scripts to execute. Your command will look similar to the following example.

    PS C:\Users\Abby\Desktop>Set-ExecutionPolicy Unrestricted

  5. Execute the script. You can install a specific app (like the Windows Store) or all preinstalled apps.

    If you try to install an app that is already installed, it will be skipped without an error.

    Reinstall a specific app

    • Add a string argument to the powershell command that represents the string containing the name of the app. In our example of the Windows Store, the string is *Microsoft.WindowsStore* (asterisks included). Your command will look similar to the following example.

      PS C:\Users\Abby\Desktop>.\reinstall-preinstalledApp.ps1 *Microsoft.WindowsStore*

      The system will prompt for approval to execute the script. Typing “y” will allow the script to continue.
