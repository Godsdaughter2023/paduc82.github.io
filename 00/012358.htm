joeu2004 say: JAndrewSmith wrote:

The bug has to do with VBA rounding numbers that end with one decimal digit, xxx.5 -- for example, if you round 0.5 you'll get 0, not 1, but if you round 1.5 you'll get 2!


This is not a defect.  It is called "banker's rounding".  When the decimal fraction is exactly 0.5, integers are rounded to the nearest even value.  Read more at http://en.wikipedia.org/wiki/Rounding.

If you prefer "normal" rounding, use WorksheetFunction.Round -- that is, the Excel functionality.
