Hi,

Not sure if this has been resolved from the previous comments. But I was able to find a solution.

*Go to regedit - 

HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU

Change 'UseWUServer' value data to '0'

1. disconnect from corporate network (ethernet or VPN) and connect to only a public wifi (home/starbucks/etc)

2. Windows+R - type in optionalfeatures and hit enter

3. uncheck Microsoft XPS Document Writer and click ok. Then go back in and check the box and click ok.

4. restart machine

5. go to Apps & Features in Settings from Start Menu

6. click Manage optional features

7. click + Add a feature

8. XPS Viewer should now be listed for installation

*Go to regedit - 

HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU

Change 'UseWUServer' value data back to '0'

Hi,

I would think your work machines are set up to install updates at night when not in use, or something similar... That wouldn't work well for you, but your machine can be set to do that.

In reality, once the first large batch of updates are installed, they only appear once a month...

Usually the 2nd Tuesday of every month...

Most of these monthly updates can be installed at one time... no need to keep rechecking and rebooting.

About the only exception to that are the nearly daily security definition updates. These install quickly and don't normally require a reboot.

There is no doubt that when you first use a new machine that there are tons of updates that may require multiple checks and restarts, but once that is done, it's done.

I suppose you may have other installed programs and\or apps that are constantly looking for updates as well.. Adobe Reader and Flash Player are two frequent ones, but those are separate from Windows Updates.
