Hello,

Thank you for your interest in Windows 10.

I would suggest you to follow the methods given below and check.

Method 1: Perform Clean Boot.

Placing your system in Clean Boot state helps in identifying if any third party applications or startup items are causing the issue. Check this issue in clean boot state.

  1. Press Windows key + X key. And select Run.
  2. Type msconfig in the Run box and hit Enter.
  3. On the Services tab of the System Configuration dialog box, tap or click to select the Hide all Microsoft services check box, and then tap or click Disable all.
  4. On the Startup tab of the System Configuration dialog box, tap or click Open Task Manager.
  5. On the Startup tab in Task Manager, for each startup item, select the item and then click Disable.
  6. Close Task Manager.
  7. On the Startup tab of the System Configuration dialog box, tap or click OK, and then restart the computer.

Note: After clean boot troubleshooting step, follow the section "How to reset the computer to start as usual after troubleshooting with clean boot" in the provided link to return your computer to Normal startup mode.

If issue still persists try method 2.

Method 2:

Try to run a System File Checker (SFC) scan to check for any file corruption.

  1. Press Windows key + X, click Command Prompt (Admin).
  2. In the Command Prompt, type the following command, and then press ENTER:
    sfc /scannow

If the issue still exists, try the next method.

Method 3:

You may check the same issue with the new user account.

Try these steps to create a new user account and check if it fixes the issue. Refer to the steps below to create a new User Account:

  1. Go to Settings.
  2. Choose Accounts and then select Other User Accounts towards the left.
  3. Select Add an Account.
  4. Enter a user name and hit Next.
  5. Click on Finish.
  6. Sign out from the current Account and Log into the new account.
  7. Check if issue persists.

    Yes this is an ongoing problem that is not likely to be ever solved.

    Regardless of what ever software you are using, they all seem to have their own idea of where to save a file.

    So here is a tip to make the best of a bad job......

    Always have a "Windows Explorer" window open set to the folder you want to work/save in.

    (I have 'Display the full path in the title bar' set in Tools>Folder Options>View, because I'm a 'Classic' guy.)

    When it comes to saving your file and you have the dialog menu open that shows you the path it is about to be saved to, Type the file name first* 

    Go across to "Windows Explorer" do a single left hand mouse click in the title bar and windows creates a unique clip independent of the clipboard.

    Go back to the save your file dialog menu and again do a single left hand mouse click in the save path bar, it will 'paste' the path from 'explorer'

    then click save.

    * I have found that it you attempt to type the file name after the "magic paste" then the path reverts back to where your software wants to save it.

    Sorry if this a bit of a long winded description, but when you have used this methode a few times it becomes second nature. 

    Also if you have more than one screen it makes easier to click between the two.

    Hope this helps.

    a. user say:

    If that is the problem with your game those are the right commands.

    The Windows update stops the driver from running and then disables the driver as a security measure to plug some hole I can't remember.

    So from your command prompt:

    The first command enables the driver and sets it to start on demand.

    The seconds command demands the driver to start.

    Then play your game.

    If you are worried about the security hole the third command will stop the driver:

    sc  stop  secdrv

    If you just want to play your games and are not worried about starting and stopping the driver all the time and don't care about the security hole you can set the driver to start automatically with Windows and then when you restart your system it will start automatically:

    sc  config  secdrv  start=  auto

    Restart your system.

    You will have to read the MS articles about the security issue...  I am not a gamer so I don't know.
