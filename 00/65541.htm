  3. If  this does not work, use File, Run new task in Task Manager to start an Administrator CMD window. Use this to create a new administrator user with the following commands (changing “<password>” to something of the customer’s choosing):
             net user test <password> /add
         net localgroup Administrators test /add         
  4. This will give you a test user to try, which will confirm if the issue is user or system-specific.

If you see the blue/black screen or have a flickering screen before sign-in:

1. Try connecting to different video outputs on the PC. Sometimes a change in driver can cause the video to be sent:

  • To a different Adapter: from integrated graphics to a discrete card or vice versa.
  • To a different output: from HDMI to DVI, DisplayPort to VGA, or any other combination.

2. Try to boot into Safe Mode with Networking or Enable low-resolution video.

  • Advanced boot options, Safe Mode with Networking, or Low Resolution Video.

http://windows.microsoft.com/en-us/windows-10/change-startup-settings-in-windows-10#v1h=tab01

  • Check Device Manager and look for Display Adapters.
    1. If you see 2 items under Display Adapters, disable to integrated video card (e.g. Intel HD 4000 or AMD Radeon HD 4200) and restart. (try vice versa if it does not work)
    2. If you see 1 item under Display Adapters, rollback the driver or uninstall the display adapter in Device Manager, and enable the option to delete driver software.

Check the PC or display adapter manufacturer’s site for an updated driver.

Hope this helps in resolving the issue. If the issue persists, do get back to us. We will be happy to assist you.

    I would suggest you to try the below methods one by one and check.

    Method 1:  Uncheck Allow the computer to turn off this device to save power option in device manager and check.

    1: Click on Start and go to Control Panel. In the Control Panel, Click on Small Icons to VIEW.
    2: Find Network and Sharing Center and open it.
    3: On the left side, you will see an option that says "Change adapter settings". Open it
    4: You will two connections there, local and wireless. Right Click on WIRELESS one and select properties.
    5: Now Click on CONFIGURE.
    6: You will see 5 tabs, click on the last one that says "power management"
    7: Now simply UNCHECK the first box that says "Allow the computer to turn off this device to save power"

    Method 2: Perform power cycle modem/router.

    1) Turn off the power to the modem (disconnect from power source). If there is Router turn it off (disconnect from power source).
    2) Disconnect the Ethernet Cable that runs from the modem to the back of the computer or the Router to the back of the computer. Disconnect at the computer's end. If there is a router follow the same steps, disconnect at the computer's end.
    3) Shut the computer down (Do not restart).
    4) Wait for at least 4 minutes.
    5) Turn the power on to the modem. If there is a router turn the router on as well. Wait for the lights on the modem\router to stabilize.
    6) Connect the Ethernet Cable back to the computer.
    7) Turn the computer on. Now try to connect to the internet.

    Method 3: Perform system restore on your computer.

    System restore will take your computer to pervious state when it was not facing such problem, this can be done by selecting the proper restore point.  

    To restore the operating system to an earlier point in time, follow these steps:

    1. Click Start, type system restore in the Start Search box, and then click System Restore in the Programs list.

    If you are prompted for an administrator password or confirmation, type your password or click Continue.

    2. In the System Restore dialog box, click Choose a different restore point, and then click Next.
    3. In the list of restore points, click a restore point that was created before you began to experience the issue, and then click Next.
    4. Click Finish.

    But this time, when I 'applied' the shortcut icon displayed on the desktop was changed to the windows general icon.  Selecting it to execute the program works fine.  the issue is the icon is now incorrect.  And I have been unable to change it back to the correct one which was displayed before the name edit occurred.  I have tried the following:

    - creating a new shortcut from the program folder.  However, the windows general icon is also displayed in explorer alongside the program exe file, so this method does not work:  more specifically, it simply copies the circumstance I am trying to correct..

    - changing the icon (shortcut properties/shortcut tab) - quite a different tab from the name change and not one commanded (hence the 'without authority' part of the topic name), and its not even one that's displayed in the change icon list.  The correct program icon is displayed in the change list:  but selecting and applying it does not remove the general icon on the desktop shortcut.

    - changing the icon but choosing a suitable alternative icon from the change list - applying works on the desktop shortcut.  However, it does not change the situation where the windows general icon is listed in explorer as the program icon.  Repeating this process and choosing the correct icon and applying simply puts the windows general icon back on the desktop shortcut.

    So, for the moment, I am living with the less than optimum alternative desktop icon.

    The questions from this are:

    1.  what causes windows to substitute the general icon for a program specific item on both the desktop and in the explorer program folder?

    2.  How can the correct program icon (the visually correct .ico file is listed in the program folder) be restored to both the shortcut and the explorer program folder exe file listing?

    Thanks for the response.  This is the reply re result requested.

    I read the documents at both links before actually doing anything.  Method 2 seemed - on the face of it - more focussed and less intrusive (I had an icon problem, not file or other issues the 1st method also talks about, and the on-line scan approach made me wary), so I did that first.

    method 2 result - no change:  the old general icon was still displayed as assigned to the program.

    Method 1 seemed to involve an on-line scan, in part because some of the text in the execute box was a link for unconnected machines.  I clicked the link and the result was a download of 2 fixit "starters" - one called maintenance and one called WinFileFolder, each one 340Kb.  I executed both, in succession.

    Both of them offered choices of detecting and auto fix, or detecting and user selecting the fix desired.  The maintenance fixit found nothing to fix.

    The winFileFolder fixit found icon issues, and when the fix was authorised it made changes and then on completion exited.  All the old desktop icons were gone.  When re-started, the correct ones were back Including the one with the problem, now fixed.  The previous desktop arrangement (icon size and position) was destroyed but ot only a minute or so to restore the preferred layout.

    So, thanks a bunch.
