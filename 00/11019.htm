nathan li msft say:

Hi Joe,

 

Per my tests in Access 2016, the “Chart” button to create a scatter plot isn’t in the ribbon by default. Please refer to the following steps to add it:

  1. Go to File > Options > Customize Ribbon > switch “Popular Commands” to “Commands not in the Ribbon”;
  2. Click “New Tab” to create a new tab (then you may rename the new tab and the new group under it);
  3. Click on the new group to select it;
  4. Select “Chart” from the commands list in the left side;
  5. Click Add to add it to the new group.

 

After that you may follow these steps to create a scatter plot:

  1. After you imported the X-Y table in to Access, go to Create > Report Design;
  2. Click the new tab you created > click the Chart button;
  3. Select an area in the report, then the Chart Wizard will appear;
  4. Select the view and field;
  5. Select “XY (Scatter) Chart”;
  6. Select how you want to lay out the data;
  7. Edit the title of the chart and select whether to display a legend;
  8. A demonstration chart will be shown on the form. To check the scatter chart you created, please go to Design > View > Report View.

sven johannsen say:

Lets start with the BT profiles that WP8 supports. From that, things like Mice and Keyboards aren't supported (Human Inteface Device, HID). There are also no networking profiles so you can't pair two devices and browse each others file systems. There is no user accessible file system per-se on Windows Phone so that sort of makes sense.

 

The Object Push Profile (OPP 1.1) does let you send and receive files via BT. The basic process from the Phone side is to have BT on, find an app, like photos, or Office Hub that allows sharing, and select Bluetooth as the method and proceed. In each case a file, photo, document, needs to be selected to get the Share option in the ... menus.  Once you hit Bluetooth, and it is on, it will search for other BT devices that are visible. That is important. Most of the time other devices, like your PC or Surface aren't visible. You need to set them to visible, or in the case of the Surface, you set it to receive a file from the BT controls.  Then if they have been previously paired, the transfer will happen, if not it will prompt pairing the first time.

 

To send to the phone, the phone needs to be visible for the sender to see it, which you do by being in the BT control panel. The phone is not visible normally for security. The file transfered of course needs to be something the phone understands, or it won't know where to put it or what to do with it.

 

IMHO, BT transfer is a pain, and not just with WP. It has always been a pain with the pairing, spotty standards compliance, and significant interaction required to both send and receive. I get that we don't all have unlimited bandwidth, but posting to a publis or shared skydrive, or facebook or twitter account is so much easier.

 

 

For audio stuff, headsets, headphones, things should work. I don't have the devices you guys do, but have an assortment of headsets, speakerphones, headphones BT speakers, that I really haven't had much problem with.

ruel mayoya msft say:

Hi Daveli_lov,

Setting edit permissions to Office 365 group files and folders is not possible. For this scenario, you can use Team Sites, as it can provide file and folder permissions. Office 365 Groups brings collaboration tools together so users can easily connect with the colleagues, information, and applications they need to do more. When you add members to the group, they automatically get shared access to the files. By default, group members have view/edit access to the files, while everyone else has view access.

Also by default users can Create Groups via their Outlook mailboxes in Office 365, so you can Disable Group creation for all users via PowerShell

If you have placed users in a Distribution Group, checking a group member availability on the schedule assistant is possible by expanding the DG to see all the members Free/Busy Status.
