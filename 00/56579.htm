malkeleah say: You forgot to tell us where you are attempting to save files. Perhaps you are trying to save them in protected directories such as the root of %systemdrive%, Program Files, or Windows. Having protected directories is common with lots of modern operating systems, not just Windows Vista and Windows 7. It is a Good Thing whether you think so or not.

As for not being the administrator, even though you are running in an administrative account (not best practice - see below) you still need to elevate privilege when making global changes to the OS.

If you are trying to save files in your own user directory and failing, please let us know. Otherwise the operating system is working as designed. If you find it too objectionable, then you can always switch to another operating system. You should, however, be aware that you will find the same "limitations" in Linux and OS X. And no one who knows what s/he is doing would run as root in Linux.

You do not have your system set up optimally regarding users. You absolutely do not want to have only one user account. Like XP and all other modern operating systems, Vista and Windows 7 are multi-user operating systems with built-in system accounts such as Administrator, Default, All Users, and Guest. These accounts should be left alone as they are part of the operating system structure.

You particularly don't want only one user account with administrative privileges on Vista and Windows 7 because the built-in Administrator account (normally only used in emergencies) is disabled by default. If you're running as Administrator for your daily work and that account gets corrupted, things will be Difficult. It isn't impossible to activate the built-in Administrator to rescue things, but it will require third-party tools and working outside the operating system.

The user account that is for your daily work should be a Standard user, with the extra administrative user (call it something like "CompAdmin" or "Tech" or the like) only there for elevation purposes. Running as a Standard user is best practice for security purposes and will help protect your computer from infection. After you create "CompAdmin", log into it and change your regular user account to Standard. Then log back into your regular account.

If you want to go directly to the Desktop and skip the Welcome Screen with the icons of user accounts, you can do this:

Start Orb>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).

aileen alf say:

Hi Wailana,

One of the possible reasons why your desktop icons might be missing is if you have unknowingly enabled tablet mode. If this is the case, everything should be back to normal once you disable tablet mode. Here's how you can enable or disable tablet mode:


  1. Click on the Action Center icon to open the Action Center window on the far right of the screen.
  2. Near the very bottom of the Action Center window are a number of squares. One of them says Tablet Mode. If you click on Tablet Mode, the system will change into Tablet Mode instantly and icons on the task bar and the desktop will disappear. The taskbar will remain on the screen, including the Action Center, so you can easily get back in to reverse the changes.
  3. To disable Tablet mode (and to re-enable PC mode), click the Action Center icon again and then click on the Tablet Mode square. Your system will then change back to PC mode and the icons on the desktop will reappear.

Another possibility is that the Show Desktop Icons feature is deactivated. To activate it:

  1. Right-click on an empty area of the desktop to bring up a dialogue menu.
  2. Select View, then ensure that the Show Desktop Icons has a check mark beside it.
