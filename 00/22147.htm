greginmich say:

The question that you should really be asking yourself at this point is whether this relic antimalware tool is really adding any protection for your system. It’s pretty unlikely that the MSRT includes any kind of emergent detection technology that hasn’t already been included in Windows Defender – but we do know that it lacks many of Defender's more advanced detection technologies. And we also know that the MSRT operates with just a small subset of the signatures that Defender uses for scanning. As far as I can see, the only thing that sets the MSRT apart is its limitations. The MSRT was a stopgap measure in the days before the Windows Security Center was fully developed, and it’s pretty much of an anachronism at this point in time.

Microsoft generally releases MSRT monthly as part of Windows Update or as the standalone tool. Use this tool to find and remove specific prevalent threats and reverse the changes they have made (see covered threats). For comprehensive malware detection and removal, consider using

https://support.microsoft.com/en-us/help/890830/remove-specific-prevalent-malware-with-windows-malicious-software-remo  

Of course the rationale for using on-demand scans is simply that they take the time to unpack container files and scan the contents. Real-time protection doesn’t do that because it’s too busy monitoring all of the read/write file access operations on your PC, and keeping up with file system activity can be a daunting task at times. Real-time protection (on-access) will only scan the contents of archive files when they’re extracted or opened up later on. So it’s always a good idea to run an on-demand scan every now and then just to make sure that nothing bad is hiding in one of those archive files that could jump out at you when the file gets opened up. But in order to do a thorough job of looking for hidden malware, you’ll need to run a Full Scan with Defender, because that will cover the entire drive, and it will also scan with a much larger definition set. The MSRT just isn’t up to the task.

So run your manual scans with Defender rather than the MSRT, or better yet, with Windows Defender Offline, where you’re running with the larger set of signatures, and you’re also running outside of the Windows environment, where the operating system can’t be manipulated by the malware in order to conceal itself. And if you really want to maximize the coverage for your Windows Defender scans, then enable PUA Protection and you'll be running your scans with the full Microsoft signature set:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/windows-defender-not-detecting-known-adware/9cfe114b-8d1b-42a2-8268-34dc3acf9390

But even then, you’ll still be a long way from covering the entire spectrum of online threats, because no single AV/Antimalware app can possibly detect everything out there. So if you really want to maximize your overall threat coverage, then you’ll need to scan periodically with one or more of the third-party malware-removal apps, like the free version of Malwarebytes. Third-party AV/Antimalware apps often have signatures that aren’t included in the Microsoft signature set, and they consequently have the potential for finding active malware (not just archived malware) that Windows Defender simply can’t detect. On the other hand, if you’re using a third-party AV app for your main antivirus protection, then exactly the same principle applies – you should have Windows Defender’s Limited Periodic Scanning mode enabled so Defender can find the bad stuff that your third-party AV app misses.

And last, but certainly not least; you need to be aware of how critical the Cloud-delivered Protection and Automatic Sample submission features are in dealing with unknown and zero-day threats. Defender can’t protect you against emergent threats if you disable its emergent-malware fighting features. These features work hand in hand with Defender's new Controlled Folder Access and Exploit Protection features to provide a quantum-leap in the level of protection against ransomware.

https://blogs.technet.microsoft.com/mmpc/2017/07/18/windows-defender-antivirus-cloud-protection-service-advanced-real-time-defense-against-never-before-seen-malware/

Microsoft generally releases MSRT monthly as part of Windows Update or as the standalone tool. Use this tool to find and remove specific prevalent threats and reverse the changes they have made (see covered threats). For comprehensive malware detection and removal, consider using

https://support.microsoft.com/en-us/help/890830/remove-specific-prevalent-malware-with-windows-malicious-software-remo  

Of course the rationale for using on-demand scans is simply that they take the time to unpack container files and scan the contents. Real-time protection doesn’t do that because it’s too busy monitoring all of the read/write file access operations on your PC, and keeping up with file system activity can be a daunting task at times. Real-time protection (on-access) will only scan the contents of archive files when they’re extracted or opened up later on. So it’s always a good idea to run an on-demand scan every now and then just to make sure that nothing bad is hiding in one of those archive files that could jump out at you when the file gets opened up. But in order to do a thorough job of looking for hidden malware, you’ll need to run a Full Scan with Defender, because that will cover the entire drive, and it will also scan with a much larger definition set. The MSRT just isn’t up to the task.

So run your manual scans with Defender rather than the MSRT, or better yet, with Windows Defender Offline, where you’re running with the larger set of signatures, and you’re also running outside of the Windows environment, where the operating system can’t be manipulated by the malware in order to conceal itself. And if you really want to maximize the coverage for your Windows Defender scans, then enable PUA Protection and you'll be running your scans with the full Microsoft signature set:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/windows-defender-not-detecting-known-adware/9cfe114b-8d1b-42a2-8268-34dc3acf9390

But even then, you’ll still be a long way from covering the entire spectrum of online threats, because no single AV/Antimalware app can possibly detect everything out there. So if you really want to maximize your overall threat coverage, then you’ll need to scan periodically with one or more of the third-party malware-removal apps, like the free version of Malwarebytes. Third-party AV/Antimalware apps often have signatures that aren’t included in the Microsoft signature set, and they consequently have the potential for finding active malware (not just archived malware) that Windows Defender simply can’t detect. On the other hand, if you’re using a third-party AV app for your main antivirus protection, then exactly the same principle applies – you should have Windows Defender’s Limited Periodic Scanning mode enabled so Defender can find the bad stuff that your third-party AV app misses.

ken sheridan say: This is probably not as simple an issue as you might think at first sight.  Spreadsheets and relational databases are very different in both purpose and concept, so a straight import of a worksheet from Excel is in most situations only going to be the first stage of a process.

You say that your worksheet has 65 columns.  While it's not impossible for a correctly normalized relational database table to have that many columns, in my experience it would be unusual.  So, if such a worksheet is imported into Access as a single table, the table is almost certainly not correctly normalized and will either contain significant redundancy leaving it open to the risk of update anomalies, or will be doing what is known as 'encoding data as column headings'.  The latter is common in a worksheet, e.g. you might have a sheet recording stocks held by stores, with each store as a row and columns headed Apples, Oranges, Pears, Bananas etc.  The intersections of the rows and columns might record either the current stock levels per store, or simply whether or not a store holds that item.

In both cases the imported data would need to be decomposed into correctly normalized related tables.  You'll find examples in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

For an example of how to import and decompose a table which includes redundancy into a set of correctly normalized related tables see the DecomposerDemo file.  This takes you through the stages of decomposition by executing append queries step by step.

For an example of how to decompose a table which encodes data as column headings see the UnencodeColumns file.

In neither case is the process trivial, though not as complex as it might seem at first sight to an inexperienced Access developer.  However, the process of updating the set of normalized tables with changes made in Excel would not be a simple one.  It gets even more complex if the data can be amended in both Excel and Access as you then have to reconcile conflicts.

Things become much easier if you only update the data in either Excel or Access.  In the former case you then simply need to link to the Excl file and no updating is necessary.  In the latter case you can periodically export the result table of a query which joins the tables to an Excel file, replacing the current file.  The easiest solution of all, of course, is to do everything in either Excel or Access.
