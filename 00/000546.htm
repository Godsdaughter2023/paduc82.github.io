Where you want the dependent text to appear, insert a field coded along the lines of:

{IF{MERGEFIELD MyField}= "Yes" "Place the item in the {MERGEFIELD boxcolor} box"}

or:

{IF«MyField»= "Yes" "Place the item in the «boxcolor» box"}

where 'MyField' is the name of the controlling field.

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Likewise, the chevrons (i.e. '« »') are part of the actual mergefields - which you can insert from the 'Insert Merge Field' dropdown (i.e. you can't type or copy & paste them from this message, either). You need to change,
=SUMPRODUCT((ROW($1:$5)={1,2,5})*($A1<$A$1:$A$5))+1

How would (could?) you use the =RANK formula IF, instead of an array reference,


You can imitate a RANKIF() function with SUMPRODUCT().

                        

The formula in D1 is,
=SUMPRODUCT((ROW($1:$5)={1,2,5})*($A1<$A$1:$A$5))+1
... and is copied to D2 and D5. D3 and D4 do not contain formulas. There is probably a way to use an IF() formula to fill down the formula while not displaying values for the rows you want discarded, but I haven't thought of one.

i figured it out  - not the most elegant code but works

 

in the sheet i used function

Private Sub Worksheet_FollowHyperlink(ByVal Target As Hyperlink)

 

didnt know how to name my filter data values before the hyperlink linked to the new sheet so my code simply goes back to the main table, references the data for the filter off the active cell (which i selected to activate the hyperlink) and then goes back to the hyperlinked sheet - actually I just set the sheet name as a column within my main table so it doesnt matter where my original hyperlink goes as this function executes when any hyperlink is executed within the main sheet but ultimately will go to a sheet that I name in the table
