tom ogilvy say:

1) Files are passive entities that sit on hard drives or other storage location.  Executing code will use and change files.  For excel files, this code is in the Excel application.   It is the Excel application that has the code that will update links and the Excel application only has knowledge of the information in open files.

2)  if you open a source file (file without the link) and update it, it has no knowledge in that file that another file is linked to it.   Neither does the excel application if the linking file is not open.   So updating the source file does not cause any updating activity on any other unopened file that may be linked to the source file. 

3)  when the linking file is opened, Excel reads that the link exists and updates the data by going to the source file and getting the information.    The source file is opened by excel to read the data - open in this case is not in the same sense as when you manually open a file and it is transparent to you.  From an operating system perspective, when the data in a file is manipulated, it is considered opened.   Excel opens the file, reads the information and I would expect it would close the file immediately.  All that would be hidden from you.  You would only see the current values.

4)  in the case where both the source and linking files are open in the same instance of excel, the links are live and update simultaneously or nearly so.

I rarely speak in absolutes because someone is likely to race in with some arcane special exception but in my opinion, it is "Must be open".
