Try to re-register Start Menu:

1. Open Task Manager, click File > Run new task.

2. Type powershell in the Create new task box. Check the option 'Create this task with administrative privileges'. Click OK.

3. In the administrative Windows PowerShell window, type/paste following and press Enter key:

Get-appxpackage -all *shellexperience* -packagetype bundle |% {add-appxpackage -register -disabledevelopmentmode ($_.installlocation + “\appxmetadata\appxbundlemanifest.xml”)}

a. user say:

Non federated migration from Lync 2013 to Skype for Business online

We have a simple deployment of Lync 2013 where we never federated our deployment so it was only used internally. We have now started migrating our services over to Office 365 using an E3 license which includes Skype for Business. To migrate our users, is our migration going to be as simple as uninstalling Lync 2013 Server and then updating our internal and external DNS for SFB? Or is there more steps required? We don't even use an Edge server.

young yang msft support say:

Hi Branden,

“To migrate our users, is our migration going to be as simple as uninstalling Lync 2013 Server and then updating our internal and external DNS for SFB?”

Yes. If you just want to decommission on-premise Lync server, and start to use SFB, you can verify domain in Office 365, directly create Office 365 users, assign license, and update internal and external DNS for SFB.

Another way is to deploy Lync hybrid, and then move Lync users to Office365: https://technet.microsoft.com/en-us/library/jj205386(v=ocs.15).aspx. This deployment is difficult. If you do not need hybrid environment, do not need to deploy hybrid.
