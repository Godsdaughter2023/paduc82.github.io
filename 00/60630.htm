Let's walk though that...

You cannot run defrag on a volume where the dirty bit is set - you must clear the dirty bit (somehow) first before chkdsk will run.

Checking the dirty but with fsutil is a good idea and you said your drive was dirty. 

It probably got dirty from a power interruption (usually), aborted restart or ungraceful shutdown.  Did any of that happen or sound familiar?

When XP starts, it checks to see if any volume is dirty and if any volume is dirty, XP will automatically run chkdsk on it because it would like to fix it for you.

The only thing that can clear the dirty bit is a successful run of chkdsk /r.

The next idea:

chkntfs /x c:

That puts a bit in the registry that tell XP to not even check the dirty bit on the volume anymore... to be missing from those instructions (how to undo it).

If I were you, I would undo that now because, it it really in your best interest to let XP check the volume and try to fix it if the dirty bit if it set, right?  Why would you tell XP not to check your volumes when they really might need checking?

To undo that command, enter this command:

chkntfs /d    (you don't have so specify a volume).  That sets everything back to "normal" and also removes any pending chkdsk command you may have said YES to previously.

Now your volume is still dirty (you can check it again).

syed n say:

Hi Tiffany,

 

·         What is the operating system installed on the computer?

 

Method 1: Disconnect any non-essential external devices connected to the computer and restart. Check to see if it boots fine.

 

Method 2: You may boot into WinRE (Windows Recovery Environment) to perform Startup Repair. To get to the WinRE, follow these steps. (For Windows 7)

 

a.        Restart your computer and start pressing the F8 key on your keyboard. On a computer that is configured for booting to multiple operating systems, you can press the F8 key when the Boot Menu appears.

b.       Use the arrow keys to choose Repair your Computer in the Windows Advanced Boot Menu Options and then press ENTER.

c.        Select System Restore from the System Recovery Options. Select the date and time when you installed the updates.

 

Refer:

What are the system recovery options in Windows 7?

 

For Windows Vista: Startup Repair: frequently asked questions

 

Method 3: Perform system restore to check for the issue. You may follow the same steps above and select System Restore to run the tool manually.
