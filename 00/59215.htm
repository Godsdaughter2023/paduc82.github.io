jrchicago say: Found a solution.  After dozens of hours and reading hundreds of posts, and trying almost everything, I've concluded there's simply NO one solution that will work for everyone.

So, here goes:

The Intel ICH9ME/M SATA AHI Controller driver was the problem.  It became incompatible or out-of-date at some point and neither Windows or Lenovo updates ever picked up on it.  It had an 2009 date on it, which seemed suspiciously old to me.  I started looking at it because of the Disk Error 11 in Event Viewer.  I downloaded the Intel Rapid Storage Technology software, which seemed to automatically update it with a driver from late 2012.  Errors went away and image created fine.  Many of the posts I read said driver problems were rare and unlikely to be the problem but I went that direction based on Disk Error 11 and it turned out that was it.

Always got 0x8078012D in combo with 0x8007045D.  It's important to look at Event Viewer to get more detail on what's going on.  Disk error 11 is "driver detected a controller error on Harddisk0/DR0," which is the main volume.  A year ago, I replaced the HDD in this machine with an SSD when I did a clean install W7 Pro.  It's been very speedy, partly because I decided not to load all the Intel and Lenovo software that was on the original machine.  Backups and images worked fine until a about 3 months ago.  My theory is that it was a W7 update or a Lenovo update in combo with the Crucial firmware update that caused the driver controller error (I ended up adding back most of the Intel and Lenovo software and performance actually has not suffered).

Side note:  Almost all the posts say run CHKDSK and they are correct - do it.  Tricky, though.  It seems you might need to run it 3 or even 4 times for Windows to clear bad clusters.  I had another error code (not these) that pointed to bad cluster but chkdsk and chkdsk /r kept coming up clean as well as SMART data.  If finally ran it from Windows instead of the cmd prompt, checked "recover bad sectors" and it did.  That was quick once I figured it out.  Does NOT hurt your SSD.
