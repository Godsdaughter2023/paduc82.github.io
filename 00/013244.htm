Do I understand correctly you have a form in Form View (showing a single record) with Physical address fields and Contact address fields, and a checkbox "Same", just like we see in many ecommerce sites?

In the checkbox' AfterUpdate event write:

if me.myCheckboxControl = True then

endif

(of course you change myObjectNames to yours)

    shanezeppelin say:

    That link isn't quite what I was looking to do, but it was helpful to learn that Publisher doesn't make it too easy. I did not find info online about this, so hopefully this helps others. I thought I'd use the Master Pages and create a different page for each section (for example "SECTION A, SECTION B, SECTION C" and so forth). Then in the footer I placed the automatic page numbering code (#) within a text block preceded by the section designation (A-#, B-#, C-#).

    As a bonus, I created "edge tabs" with the section letters in them on each of the master pages.

    As I created a new section in my publication, I used the section master page I created for that section, and have the numbering start fresh in each section (A-1, A-2, A-3, B-1, B-2, B-3, etc).

    It's not automatic, but it works fine. Now as I build my publication (which happens to be an owner's manual) and I need to insert a page in a certain section, I can do so without disrupting all of the following section page numbers. Makes changing the Table of Contents easier, too.
