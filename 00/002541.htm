bill manville say:

If the problem goes away in Safe mode then it is very likely caused by an add-in or a startup program.

Disable all add-ins (File > Options > AddIns) and restart Excel normally.
If Ctrl+N no longer causes a problem then it was one of the add-ins you disabled; enable them one-by-one until the problem recurs and you then know the culprit.

But if Ctrl+N still causes a problem after disabling add-ins then it could be a startup program causing the problem.  To locate any startup programs you can use this macro [paste it into a standard module in the VB editor and run ShowXLSTARTFiles]:

Sub ShowXLSTARTFiles()
  ShowFiles Application.StartupPath
  If Application.AltStartupPath <> "" Then ShowFiles Application.AltStartupPath
  ShowFiles Application.Path & "\XLSTART"
  Debug.Print "Done"
End Sub

Sub ShowFiles(Path As String)
  Dim stFile As String
  stFile = Dir(Path & "\*.*")
  Do Until stFile = ""
    Debug.Print Path & "\" & stFile
    stFile = Dir()
  Loop
End Sub

In the immediate window (Ctrl+G if it isn't visible) you will see listed any files in any of the 3 folders.If there are none it will just show "Done"

If there are such files, move them elsewhere, restart Excel normally and check the problem has gone away.

Files you need in XLSTART can be replaced one-by-one until you find the culprit.

steve rindsberg say:

A bit of VBA code will fix this. This'll replace soft returns with a space; you might want to change it to replace them with something visible but unlikely to occur otherwise, maybe "@@@" so you'll be able to distinguish between the original spaces the places where there were soft returns.  And you can later replace the "@@@" with " " to turn them back into spaces (just using PPT's own search/replace feature)

Sub SofterSoftReturns()
    Dim oSl As Slide
    Dim oSh As Shape
    Dim sReplaceCharacter As String
    
    ' edit this if you like:
    sReplaceCharacter = " "
    
    For Each oSl In ActivePresentation.Slides
        For Each oSh In oSl.Shapes
            If oSh.HasTextFrame Then
