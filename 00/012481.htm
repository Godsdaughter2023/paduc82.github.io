suzanne s. barnhill say:

There are two possibilities here:

  1. If you are using the default settings for Word 2010, Spacing Before and Spacing After are not added together but rather merged, the larger amount being used. This behavior was introduced in Word 2000, which added a lot of features designed for Web pages. To restore the functionality of earlier versions, go to File | Options | Advanced: Layout Options and check the box for "Don't use HTML paragraph auto spacing."
  2. Given your follow-up post noting that this is affecting List Bullet styles, the likelihood is a different setting: "Don't add space between paragraphs of the same style" in the Paragraph dialog. You can modify the style not to include this setting.

It should be noted that the setting referenced in (2) had been requested by users who wanted space before and after a list but not between list items, a common layout choice.

I tried updating the fields with F9, removing the page numbers and re-inserting, I even tried a round-about way to just start the numbering from "3" but then it skipped over to starting at "4". I should add that a few pages later, the document does decide to cooperate and picks up on the correct numbering. HELP!!
suzanne s. barnhill say:

Either you have restarted the page number in one of the sections, and every subsequent section is restarting at the same number (and you have a LOT of sections), or (more likely) the page number has just been typed.

Try this:

  1. Delete the current page number.
  2. Press Alt+Shift+P to insert a PAGE field.
  3. In the Header & Footer group of the contextual Header & Footer Tools | Design tab, click Page Number and choose Format Page Numbers...
  4. In the Page Number Format dialog, choose the radio button for "Continue from previous section."
  5. Repeat for each successive section.

stefan blom say:
After you have removed page numbers from the affected footers, carefully reinsert them by pressing Alt+Shift+P (in the appropriate locations). You can make sure that page number fields get inserted by using the Alt+F9 toggle.

If numbers are still restarting in unexpected places, you have more section breaks than you think. Run the following macro to ensure that page numbering continues across sections:

Sub SetContPageNum()
    Dim s As Section
    For Each s In ActiveDocument.Sections
        s.Footers(wdHeaderFooterPrimary) _
        .PageNumbers.RestartNumberingAtSection = False
    Next s
End Sub

For assistance with the macro, see http://www.gmayor.com/installing_macro.htm.

cbarnhorst say:

The key you posted is the generic key distributed with all copies of the free Windows 10 upgrade in 2015 and 2016.  Generic keys cannot activate a copy of Windows because they are not unique keys and are blocked.  They are placeholders used to tell MSFT the channel through which you received Windows.  There are two generic keys, one for Home and one for Pro and everyone who upgraded from Windows 7 and 8.1 has one or the other same exact key.  

The free Windows 10 upgrade is good for the life of the device and is not transferable to another device.  By device, MSFT means the motherboard.  Windows 10 uses a new system for activating Windows.  At the time of installation, the unique hardware IDs are registered with MSFT permanently.  The product key is only used for the initial registration.  If the hardware IDs change then the registration is not found on the MSFT servers and Windows does not activate.  In such a case a new license is required.  New motherboard, new device.
