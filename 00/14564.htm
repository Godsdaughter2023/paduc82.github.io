Since you're using content controls (Rich Text, and maybe some of the other controls) and not legacy form fields (from the Legacy Tools button in the bottom right corner of the Controls group), do not use the Filling In Forms protection.

Yep. Just add another <sip:email_address> bit to the end for each person you want to include.

im:<sip:email_address_1><sip:email_address_2><sip:email_address_3><sip:email_address_4>

nepcior say:

I found a workaround for this problem:

  1. Locate your template/presentation
  2. Do a backup copy of it
  3. Change the extension from pptx to zip
  4. Create an empty folder
  5. Unpack archive into this folder
  6. Go to the ppt folder there
  7. Edit file “presentation.xml”
  8. Locate all texts representing unwanted font name and replace it with this one you wish. In my case I replaced all “Corbel” with “Calibri” (without quotation marks of course)
  9. Save
  10. Go back to your folder you created
  11. Compress all data back using ZIP
  12. Change the extension from zip to pptx

After this each pasted text has Corbel font no more

 

Please be aware that I am not sure if this method won’t damage your presentation. This workaround is not basing on knowledge. I just tried and succeeded.
