macropod say:

You could work around that via anASK field embedded in an IF field, coded along the lines of:

{IF { REF BkMk }= "" {ASK BkMk "Enter current commission reporting month" \d "########"}}{ REF BkMk }

or:

{IF { REF BkMk }= "" {ASK BkMk "Enter current commission reporting month" \d {QUOTE "{=MOD({DATE \@ MM }+10,12)+1 \# 0}-00" \@ MMMM}}} { REF BkMk }

and with an empty bookmark named 'BkMk'. Do note you will need to save the mailmerge main document with the empty bookmark; otherwise you won't get the ASK prompt next time you run it. I suggest doing a preview beforehand just to ensure there is no month displayed. If you accidentally save it with the bookmark populated, you can correct that by simply re-inserting the empty bookmark before finishing the mailmerge.

Of course, if all you ever want is last month's month, you could use just:

{QUOTE "{=MOD({DATE \@ MM }+10,12)+1 \# 0}-00" \@ MMMM}

and forego the ASK field.

Note: The field brace pairs (i.e. '{ }') for the above examples are created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practicable to add them via any of the standard Word dialogues. The spaces represented in the field construction are all required.

jllatham say:

Try this code with a copy of your workbook.  You need to set up the values of the Const entries in bold to align with your workbook setup (sheet names, etc).  To put the code into your workbook in a regular code module, follow these instructions:

http://www.contextures.com/xlvba01.html#videoreg

Run the code from Tools --> Macros --> Macro

 

Sub CopyTopSet()
  'name of sheet with data to be copied
  Const srcSheetName = "Sheet1" ' change as required
  'this (keyDataCol) MUST be a column that will always have
  'data in all used rows in your first set of data, it is
  'the column we will use to determine which rows to copy
  Const keyDataCol = "A" ' change as required
  Const firstColToCopy = "A"
  Const lastColToCopy = "Z"
  'next assumes row 1 has labels and
  'you don't want to copy labels
  Const firstDataRow = 2
  'name of sheet to copy the data onto
  'assumes column use is same as on source sheet.
  Const destSheetName = "Sheet2" ' change as required
 
  'working variables
  Dim srcWS As Worksheet
  Dim srcLastGroupRow As Long
  Dim destWS As Worksheet
  Dim destNextRow As Long
 
  Set srcWS = Worksheets(srcSheetName)
  srcLastGroupRow = srcWS.Range(keyDataCol & firstDataRow).End(xlDown).Row
  Set destWS = Worksheets(destSheetName)
  destNextRow = destWS.Range(keyDataCol & Rows.Count).End(xlUp).Row + 1
  srcWS.Range(firstColToCopy & firstDataRow & ":" & lastColToCopy & srcLastGroupRow).Copy _
   Destination = destWS.Range("A" & destNextRow)
  Application.CutCopyMode = False
  'cleanup your room! (best practices)
  Set destWS = Nothing
  Set srcWS = Nothing
End Sub

Or you could convert all of the PPM values to text, ie <100PPM, '=100PPM, >100PPM.  This will sort or compare appropriately

Or you could pull the =,>,< symbols out of the number column and put them into a helper column.  Then you would have to compare both pairs of columns, numbers first, if they are equal, compare the symbols (if symbol is blank, treat it as "=").  It will be a big nasty nested IF().
