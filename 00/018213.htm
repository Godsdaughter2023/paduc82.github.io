Referencing document property to a cell - Microsoft Community

vikki-1951 say:

Referencing document property to a cell

Not being a programmer, how do I reference a document propert to a cell?  It's very easy in WORD but not so for Excel.  Any help would be appreciated/
ossiemac say:

The following UDF (User Defined Function) will display the document property.

Public Function DocProp(strProp As String)
   Application.Volatile
   DocProp = ThisWorkbook.BuiltinDocumentProperties(strProp)
End Function

 

However, you need to know the exact property name. The following code will list all of the document properties by name.

Sub AllProperties()
  'Will display the list of available workbook properties
  Dim DocProp As DocumentProperty
  Dim lngRow As Long
  lngRow = 1
  With Worksheets("Sheet1") 'Edit to required sheet name
    .Cells(lngRow, 1).Value = "Built in Properties"
    .Cells(lngRow, 1).Font.Bold = True
    lngRow = lngRow + 1
    For Each DocProp In ActiveWorkbook.BuiltinDocumentProperties
        .Cells(lngRow, 1).Value = DocProp.Name
        lngRow = lngRow + 1
    Next
    lngRow = lngRow + 1
    .Cells(lngRow, 1).Value = "Custom Properties"
    .Cells(lngRow, 1).Font.Bold = True
    lngRow = lngRow + 1
    For Each DocProp In ThisWorkbook.CustomDocumentProperties
        On Error Resume Next
        .Cells(lngRow, 1).Value = DocProp.Name
        lngRow = lngRow + 1
    Next DocProp
    .Columns("A:A").Columns.AutoFit
  End With
End Sub

 

Enable macros. (See Help for how).

To use the code, Alt/F11 will open the VBA editor.

Click menu item Insert then Module.

Copy the code and paste it into the white area.

Edit the code in the Sub AllProperties() where the comment in green says to "Edit to required sheet name" This must be a blank worksheet because it will overwrite anything in the worksheet so add a worksheet if necessary.

Close the VBA editor. (Cross with red background top right of editor)

Save the workbook as a Macro Enabled workbook.

Run the code AllProperties to get the list of available properties.

Now in any cell in the worksheet you can use a formula like the following to get the property value. Replace "Title" with whatever property is required.

=docprop("Title")

dale howard [mvp] say:

Mike --

That's a pretty broad question.  In Microsoft Project, there are 5 fields that reveal the overall health of a project.  They are the Start Variance, Finish Variance, Duration Variance, Work Variance, and Cost Variance fields.  You can create custom fields that reference each of these fields and then create stoplight indicators that show the severity of variance.  For example, you could create a custom field named something like Cost Over Budget, include a formula that calculates the percentage of cost variance, and then create stoplights that might show a red stoplight if it is more than 20% over budget, a yellow stoplight if it is more than 10% over budget, and a green stoplight if it is less than 10% over budget.

The graphical indicator criteria you use really depends on your organization's requirements for variance reporting.  Just a thought.  Hope this helps.
