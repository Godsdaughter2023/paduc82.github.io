There's a possibility that the Photo Gallery proprietary data-
base is corrupted.

Uninstalling and reinstalling PG does not update the database...
it must be manually renamed.

If you rename the file...it will force Photo Gallery to create a
new database file the next time it's launched and the gallery
should be as it was the first time it was launched and the
gallery will have to be re-populated.

I'm not positive that this will solve the problem you are dealing
with but if you would like to give it a try the filename is...
Pictures.PD6. If you have the most recent version of Photo
Gallery...the file might be named Pictures.PD7...I am not
certain...but the steps would be the same.

Just rename it to OLDPictures.pd6. This may cause you to
lose some tags and metadata that were not written back to
the photo files.

The path to the file (in Windows 7) is :

Windows\Users\%Username%\AppData\Local\Microsoft\
Windows Live Photo Gallery\Pictures.pd6

(Sorry but I am unsure of the path in Windows 8 but if you
do a search for PD6, PD7 you should be able to find it.

Pictures.pd6 is a hidden file...the following article explains
how to show hidden files:

Windows 8 - How to see hidden files in Windows 8
http://www.bleepingcomputer.com/tutorials/show-hidden-files-in-windows-8/

If the result is unsatisfactory...you can delete the new PD6
file and rename the old one back to Pictures.PD6 and the
gallery should be as it was before the change.


Also, there is a possibility that your Video Driver requires
an update.

The following link may be worth a look:

Windows 8 - Troubleshoot video card problems
http://windows.microsoft.com/en-us/windows-8/troubleshoot-video-card-problems

wunders say:

Each log is its own file, so it is quite possible one is corrupt without including the others.

You likely will never recover the contents of the corrupted log, but to clear things out and start recording new log entries, first bring up the event viewer:

    Start -> Run -> "eventvwr.msc"

Then Right-click on "System" in the left-hand column and then select "Clear All Events".

This should clear the log and with it should go the corruption.

  JW
