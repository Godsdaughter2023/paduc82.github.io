The XBOX Networking adaptor seems to like AES encryption more. So trying WPA or WPA2 with AES only should work. Sometimes WPA/WPA2 mixed settings in the Router can be difficult to connect to so choose one or the other. And also The Router encryption TKIP+AES mixed settings can cause connection problems so try just AES or TKIP. I have found AES to work more successfully. Consult your Wireless Modem Router/WiFi Router User manual to learn how to access the setup page/screen.

I suspect that currently the XBOX 360 wireless Network adaptor's driver has encryption set to auto-negotiation but this type of setting can cause errors with some Modem Router/WiFi Routers. Hi PuchsucKr:

For some malwares and viruses User Account Control (UAC) is just a
security layer (nothing more), more advanced malwares and viruses can
silently bypass UAC without prompting the user for permission.

UAC is a pretty easy concept to understand it’s a bit adapted from old
Windows® operating systems (and now I count Windows® XP and
Windows Vista™), the only difference it that programs hold a tag that
allows the system to know whether the application will execute through
UAC at start or NOT.   

Explaining the User Account Control (UAC) technology
User Account Control (UAC) was a new security-feature and technology
introduced first in Windows Vista™ and later migrated and added to Windows® 7.

The way UAC works: When we execute a normal program, UAC first checks if that program has keywords like "setup", "install", "update" etc, using the UAC Installer Detection Technology (IDT) (If is a setup.exe then elevate).  UAC also checks the programs manifest file for the "requireAdministrator" tag added in its manifest configuration, if that is found, UAC will then go to the next step. When UAC has confirmed the first step, UAC will check if that program is blocked by policy or if the programs publisher is blocked, when UAC successfully confirms that it goes to the next step. 

When UAC has confirmed the first two steps, it then checks if the publisher of that program is "Windows®" it's self, but in our case it isn't so UAC goes to the next step and checks for a regular publisher. If the publisher is verified (this means also that the program has a valid digital signature) then UAC will prompt with a gray background and gold shield icon: "The application is Authenticode signed and trusted by the local computer".

UAC in Windows® 7: UAC under Windows® 7 is basically the same, except that they done some "small" modifications. But, still the same rules apply in Windows® 7 as in Windows Vista™.  
 
The modern programs under UAC restrictions
Modern applications/programs that run under UAC restrictions have their power to become “trusted”-apps. Programs like Google Chrome or other (e.g. Avast Antivirus, Mozilla FireFox or any other), are put into trusted directories (e.g. C:\Program Files\ and C:\Program Files (x86)\), and they are of course signed with a digital signature [For example: A signature from the VeriSign Time Stamping CA].
