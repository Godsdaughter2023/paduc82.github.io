ken sheridan say: A problem with using the CanShrink property of a control, as mentioned by Scott, is that if there are any non-shrinkable objects in the report alongside the control, e.g. graphics, the control will not shrink.  Also any 'white space' above or below a control won't shrink, so the controls have to be vertically contiguous to avoid uneven line spacing.  For addresses I normally use the following function:

Public Function CanShrinkLines(ParamArray arrLines())

    ' Pass this function the lines to be combined
    ' For example: strAddress =
    ' CanShrinkLines(Name, Address1, Address2, City, State, Zip)
    Dim X As Integer, strLine As String

    For X = 0 To UBound(arrLines)
        If Not IsNull(arrLines(X)) And Trim(arrLines(X)) <> "" Then
          strLine = strLine & vbCrLf & arrLines(X)
        End If
    Next
      
    ' remove leading carriage return/line feed
    CanShrinkLines = Mid(strLine, 3)
 
End Function

This is published by Microsoft, but I've amended it slightly.  To use it you would simply add a single deep text box control to the label report's detail section, so that the control pretty much fills the section from top to bottom.  The text box's ControlSource property is an expression which calls the function.  Any Null lines will be suppressed regardless of any other controls which might be alongside the control.

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

charles kenyon say:

Sorry. This is not a bug, it is Word operating as designed.

It has been this way for more than 20 years and  is unlikely to be changed.

Cross-reference:

When you placed your insertion point at the beginning of the heading you put it inside what is being cross-referenced.

When you hit the Enter key, you remain inside that. It is a hidden bookmark created when you made the cross-reference.

Table of Contents:

When you placed your insertion point at the beginning of the heading and pressed Enter you created a new paragraph, still in the heading style. That style will be picked up in the Table of Contents field. To change this behavior, you can apply a new paragraph style (not set to be picked up it the TOC) to the entire new paragraph.

For both, if you move to the end of the (body text level) paragraph preceding the heading and press Enter, you will not have this problem.

  • Generating a Table of Contents - Complex Documents
  • Complex Documents - Cross-References
  • TOC Tips and Tricks by Suzanne Barnhill, MVP
