Sounds like you have set the computer to sleep and also to turn off the display.

Sleep is a power-saving state. Sleep saves all open documents and programs, and allows the computer to quickly resume full-power operation (typically within several seconds) when you want to start working again. Putting your computer to sleep is like pausing a DVD player: the computer immediately stops what its doing and is ready to start again when you want to resume working.

Basically, when the computer goes to sleep, programs (SpyBot) will ideally not run the computer.

It means that you can, if you wish, open the CBS.log file in Notepad. This file shows the results of running the System File Checker.

To find the CBS.log file, go to C:\Windows\Logs\CBS\CBS.log

Two things you should know about reading scanner log files:

First: You need to be an uber-geek to understand what these log files mean.

Second: Scanning tools like SFC or Event Viewer or Reliability Monitor or similar third-party tools will almost always find all kinds of errors, warnings, corruptions, etc., that almost always have no real meaning. They are typically just Windows internal plumbing.

If you have a specific problem with Windows, the best thing to do is to concentrate on solving that problem. Running scanners just to see what they output is like investigating your wife's old boyfriends: You're not really accomplishing anything and just making yourself upset.

a. user say:

I can only offer limited advice here because, not meaning to boast, my last BSOD was when I had Windows 7 and it was just once. (More on the BSOD later.)

A blue screen almost always has an error message (which LOOKS_LIKE_THIS) and an error code (which starts with 0xC000...) The first step is to search for that message and/or code on the internet to find out what it means. That tells you what type of error you're dealing with. Frequently, that's all the information you'll need.

If the explanation is too generic or doesn't sufficiently explain what went wrong, you can usually find more details in Event Viewer. Look for an error that occurred around the time you saw the blue screen. That will provide additional details. Understand that you're wading into geek waters here. If you're a computer person you'll run another search with the additional details and usually find your answer.

An important part of interpreting a BSOD is being aware of what's going on with your computer. The experts also get blue screen errors but they can usually identify the source because they are attuned to changes that are being made to their computers.

If you think that researching blue screen errors is complicated, figuring out what the **** a DISM log means is much worse. On top of everything else, they contain red herrings that don't really amount to anything but just take up your time.

I don't know what you've tried already, so here's my advice:

1- Post the error message and/or code here and describe exactly what you were doing when you saw the error. Someone here might be able to interpret it for you.

2- If you can find the error is Event Viewer, post the details here.

3- If no help emerges, there are some people here who occasionally volunteer to read through your DISM log (must be insomnia) and pinpoint the cause of the error.

About BSODs: They really should be called the Blue Screen of Life, because they stop your computer from committing suicide by executing incompatible or incorrect instructions and they provide an error code for diagnosis. Before BSOLs, computers kept on executing bad instructions until they couldn't go on any longer, and all you got was a cryptic message pointing to an address in memory.
