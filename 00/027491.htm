Predecessor/Task Relationship Question - Microsoft Community

m141 say:

Predecessor/Task Relationship Question

I would like Task A (ID 1) to start 5 days prior to Task B (ID 2) and end 10 days prior to Task C (ID 3).  Not sure how to write that in Task A's predecessor column--can someone please assist?

1.  Task A 

2.  Task B

3.  Task C

Thank you! :)

trevor rabey say:

Mark this one as the answer.

When you set predecessor links, you are not saying when you want the task to start or finish. Rather, you are setting limits on how early the successor can start or finish relative to the predecessors start or finish.

So if A is the successor of B, and assuming "start 5 days prior to Task B (ID 2)" means start no earlier than 5 days before the start of B, then A's predecessor is SS2-5, meaning A cannot start earlier than 5 days before B starts. Note that something else, such as other predecessors, may still cause it to be scheduled later than that, but nothing will allow it to be sooner than that (except an actual start, because actuals trump predecessors).

Assuming "end 10 days prior to Task C (ID 3)" means "finish no earlier than 10 days before Task C starts, then A's predecessor is 3FS-10.

Or, assuming "end 10 days prior to Task C (ID 3)" means "finish no earlier than 10 days before Task C finishes, then A's predecessor is 3FF-10.

Since A has predecessors which are relative to the start or finish of both B and C, then the later one will determine the earliest start or finish of A, but predecessors at its start and finish will not reduce or increase the duration of A.

daniel pineault say:

MS Office has great backwards compatibility, ie: new versions continue to support older version without issue (2016 can open a 2010 db without any problem), but it is impossible for any program to have forward compatibility (2007 cannot work with a 2016 db, as you have found out).

MS Access 2007 know nothing of Access 2010, 2013, 2016, and whatever is going to come.  They didn't exist when it was created and thus it can not handle new files.

As a developer, you always develop using the lowest common denominator between all of your users.  So in your case, if your db will be run in Access 2007, then you need to develop your db using Access 2007.  The db should then run fine in Access 2010, 2013, 2016 as well (assuming it was created properly).

You could create a Access 2007 database and then try to export your objects into it and hope it all goes well.  If you didn't use any new features, you may be able to salvage your work.

Hi

How do you access the Skype account that you want to close? Is it with a Skype Name, email or a phone number?

If you are signing in with a Skype Name, you need to use a new email to link your Skype account with. Follow the below steps to delete the account:

  1. Sign in to the Microsoft website using your Skype account.  
  2. When you're prompted to add an email address to it, select "Get a new Outlook email".
  3. Type your preferred email and follow the process to update your Skype account to a Microsoft account.
  4. Go to our sign in to cancel the closure. 

If you're signing in with a phone number or email to access your old account, you simply need to follow the steps on this support article: How do I close my Skype account?
