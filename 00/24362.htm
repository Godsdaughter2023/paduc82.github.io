If you try the /StartComponentCleanup repair before you are able to boot Windows, you would need to run it on your offline Windows with a modified Command:

When you setup the VirtualBox VM, you would have had to choose between a Legacy BIOS mode and a UEFI virtual machine. In Hyper-V, they describe these as Generation 1 (Legacy BIOS) or Generation 2 (UEFI) VM's.

With an MBR disk (Legacy BIOS) the System partition is by default also the Recovery partition (contains a file Winre.wim, used to launch Windows RE). With an MBR disk you have to repair that partition, or if you want to replace it, you need to preserve a copy of the winre.wim to put back after you have replaced the partition.

jay freedman say:
Yes, you do run that risk with a long TOC. One workaround I'd suggest is to put a Next Page section break between the TOC and the start of the document proper, and restart the page numbering of the second section at 1. Then the page numbers in the document body will be independent of the number of pages in the TOC (and any other frontmatter). See http://www.word.mvps.org/FAQs/Formatting/NumberingFrontMatter.htm.

If your customer objects to that workaround, the alternative is to modify the definition of the TOC1 style in that customer's document after the document is created from a standard template. If you leave the "Only in this document" option selected at the bottom of the Modify Style dialog, and uncheck the Automatically update box, the style in the template won't be affected and the style won't be "refreshed" from the template.

You said "I tried modifying the line spacing on the underlying TOC1 style, but it was grayed out." Exactly what was grayed out? And were you trying this in the document or in the template? (It should be done in the document.)

charles kenyon say:
Your TOC1 style should be particular to that document - not global. You should be able to change it without making any changes to other documents or to your other non-TOC styles in your document.

This is unless you for some reason tell Word to change your template.
