john - project say:

JenLG15,

I don't think you are "overthinking it", but as a newbie to Project you simply don't yet have a grasp on how Project works. Ideally when we who respond on these forums encounter a new Project user, we recommend they get training on what Project is and how to use it. Unlike other Office applications, Project is not intuitive and can have a steep learning curve.

That said, here's a quick primer applicable to your question. When a plan is developed in Project, it should consist of a set of tasks that are linked in a logical sequence to achieve an end result. Each task is assigned one or more resources to complete the task. Duration, work and resource "units" are related by the Project work equation:

Duration = Work/Units

Once the basic plan is laid out with resources assigned, the first question is, does it meet the end goal which usually is a date dictated by business plans or a customer. If the plan does not meet the end goal, the plan must be revised as necessary to meet the requirements. Once the plan is ready, a baseline is set. The baseline captures all the important plan details (i.e. duration, stat, finish, cost, etc.). As the plan is executed, many thing may change (i.e. durations, starts, finishes, resources, etc.) as a result of normal realities of life (e.g. this task took longer/shorter than expected, this resource isn't available, etc.). Those realities are incorporated into the plan and the plan changes accordingly. For example, let's say task A was planned as a 5 day task that was to start on Monday. Due to various reasons it didn't start until Tuesday, so the plan is updated by entering Tuesday's date into the Actual Start field. Project will automatically shift the plan over by one working day. Then, it turns out that the 5 day task, which was assigned to a single full-time resource for a total work content of 40 hours, will likely take 60 hours instead of the original 40. Again the plan is updated to reflect this change and Project automatically adjusts the duration and finish date accordingly. This is the type of dynamic scheduling you seemed concerned about. Not to fear because the baseline data still has the original plan and Project provides multiple fields that compare the original plan (baseline) to the current plan (schedule plan). Examples are, the Work Variance, Start Variance, Finish variance, etc. If you are familiar with earned value, Project also has several earned value fields (BCWS, BCWP, ACWP, SPI, CPI).

Hopefully this gives you a little feel for how Project preserves your "numbers". Again, I suggest you consider getting some formal training in order to get the most out of Project.

John

jeeped say:

=HYPERLINK(("C:\Data\Cardall Collection\AG Scans Backup\photos\"&=OFFSET(B14,0,0)&".tif","photos\"&=OFFSET(B14,0,0)&".tif"))


There is no need for the OFFSET() function for the purposes you've described. Similarly, the equals only starts a formula, it is not necessary for the nested functions within a formula and the double bracket sets are redundant. Try,

=HYPERLINK("C:\Data\Cardall Collection\AG Scans Backup\photos\"&B14&".tif","photos\"&B14&".tif")
  ... or,
=HYPERLINK("FILE:///C:/Data/Cardall Collection/AG Scans Backup/photos/"&B14&".tif","photos\"&B14&".tif")

Edit: you may want to consider a full shell path to describe both the file and the application to open it.

bob jones aka: cybertaz say:

I can only guess, but this appears to be another casualty inflicted by the attempt to make Mac Excel more like the Windows version... but only to the comfort of those accustomed to using Windows. Even so, I cannot even guess why an alternative shortcut wasn't provided. Despite its importance to some, perhaps it simply was deemed to be in little overall demand.

At any rate, here's an oddity you can, perhaps, live with:

Record a macro to apply Align Right. You can assign a kybd shortcut to it but it must use Option+Command plus an alpha character... use the letter r. The oddity is that since Option+Command+r is hardwired to Hide/Display the Ribbon it apparently is automatically converted internally, so the Control+r keystroke will apply Align Right... If you look at the code in the VBA window you can see that it has happened but unless you actually look you never would have known. Go figure :-/

Another option is to create a Cell Style, but AFAIK you can't assign a kybd shortcut to a Style. You'd still have to use a macro, so that's pointless unless other formatting needs to be included, as well.
