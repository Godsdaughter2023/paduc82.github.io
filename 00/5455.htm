river monsterhunter say:

Does windows defender on windows 8 have boot time scanning

Windows 8 pro 64bit
rob koch say:

It depends what you mean by "boot time scanning".

 

If you mean does it allow you to initiate a manual boot time scan of the files in general on the PC, similar to the other old and now archaic methods used by some third-party antivirus products of the past then no, it doesn't contain this now nearly useless 'feature', since boot time malware had  rendered this method ineffective years ago.

 

If you mean rather does it support the new 'Secure Boot' ability made possible by the UEFI firmware standard, which is a replacement for the traditional PC BIOS, then yes.  Windows 8 utilizes secure boot to ensure that the pre-OS environment is secure, assuming that UEFI firmware is available and enabled (in place of BIOS) on the PC itself.

 

http://blogs.msdn.com/b/b8/archive/2011/09/22/protecting-the-pre-os-environment-with-uefi.aspx

 

There are huge differences in the two items mentioned above, so you need to understand these differences to determine if what you are seeking is really something useful or simply an anachronism from another time in the distant past.

restorewetlands say:

Making a .ACCDB database read-only for colleagues in Access 2010

I have a .ACCDB (2007/2010) database that I need to be able to edit and update, but set user-level security for my colleagues so that they are only able to read and look at the database and not make any changes.  How do I go about doing this?  I am using Microsoft Access 2010.
daniel pineault say:

User-Level Security (uls) was done away with with Access new formats (accdb).  That said, if your db is still in mdb format, you can still support it.

 

I'm assuming since you are using 2010, you have an accdb (oh, wait.. just reread your post and you clearly state this, I'll wake up eventually).  As such, to implement such security, you need to do it yourself.  You'd need to create a security system to suit your needs or search around for examples.  I know there are a number of examples of such setups on utteraccess.com in their archives.  everything depends on the level of security you need.  It can very easy or very complexe to create.

 

In its simplest form, you could create you user interface, save a 2nd version where you change everything over to snapshots, allow edit = false, etc..., so no editing can occur and then distribute this version to your colleagues.  But then further development requires continuously updating 2 dbs.
