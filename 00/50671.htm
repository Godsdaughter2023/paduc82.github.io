zram say:

Revisited: NTFS RAM DRIVE page file compression - NEW Scheme

After reading the thread referenced in my subject, original thread here, and the unfortunate dismissiveness of many comments before it was closed, I felt compelled to register and clarify the topic a bit in case others readers come along. It would seem that the one poster, "ScrewMSN" was indeed correct, and that the original poster, "DrCurtisScott" not only had a valid idea, but one that already exists and has already been put to use on Linux desktop/laptop/netbook systems since at least 2008, and even Android mobile phones, with widely accepted and measured benefits in practice.

The original name for this technology was "CompCache", but it now is referred to as "zRam Swap".

http://en.wikipedia.org/wiki/ZRam

The zRam compression, which uses the LZO algorithm and is far more aggressive than NTFS compression, shows little to no strain on modern CPU's while yielding very high compression ratio's in RAM. Implementing the same concept in Windows, using NTFS's lighter compression, which uses the LZNT1 algorithm (a variant of the LZ77), would still provide significant benefits, though slightly more conservative, but with even less concern on CPU impact.

Hey there Mikey,  Manual settings for your IP settings is never a good idea on a router that is capable of DHCP reservation and can actually cause NAT issues within routers.  In your case you're lucky that you have a router capable of DHCP reservation so there's an easy workaround for your issue.  All you need to do is leave the xbox on automatic and follow these steps and I think you'll be pleased with the end results.  

1.Plug the computer into a LAN port via an ethernet cable.

2.Open a web browser.

3.Log into the router by entering the default gateway (192.168.1.1) into the address bar of the internet browser.

4.When prompted, enter in "admin" for the user name and "admin" for the password. (Provided you're still using the defaults.)

5.From the setup tab (default tab after logging in) click the DHCP reservation button.

6.From the resulting pop up menu choose the select option to the right of the xbox.

7.In the client name box type “Xbox” so if you ever have to log back in you will remember which device the reservation is for.

8.Select Save Settings..

This way all your port forwarding rules will always be opened for the xbox and the linksys will automatically keep the xbox within the IP you setup for the rules.

If you have any issues please feel free to post here or PM me.

cmdrkeene say:

Thanks for the clarification. If I understand you correctly, you have a alternate option when you right-click PDF files, other than the standardOpen with Adobe Reader option, and you would like to invoke this option without the need of right-clicking the file.

I know that this is possible, and in fact would have been easy to determine the command in Windows XP using the File Types dialog.  Unfortunately this is no longer available in Windows Vista and 7, so we have to look in the registry.  If you look at this path in the registry, you can see what is registered to show on the right-click menu:

HKEY_CLASSES_ROOT\AcroExch.Document

You can see the (Default) name is Adobe Acrobat Document, which is what is registered to handle PDF files.  Under the Shell key of that same group, there is listings for everything registered.  For example, the default "Open" command runs this:

"C:\Program Files (x86)\Adobe\Reader 9.0\Reader\AcroRd32.exe" "%1"

Where "%1" is the name of the .pdf file you clicked on.  To see what runs with your other menu item, look for it, and you'll have the command that is run on the file when that menu item is selected.  Then you'll be equipped to run this command in a batch file, or from the Run dialog, etc.
