PROBLEM SOLVED...

Upon booting Windows 10 it decided to allocate the drive letter F:\ to the boot partition/Windows 10 OS, so the OS went into meltdown as it always expects to be C:

SO the solution came from Windows XP days, when the OS could be assigned any drive letter.

FIXED by

1 Recloning the working OS to the new partition

2 Before booting the OS on the new partition, edit its registry and change HKLM\SYSTEM\MountedDevices.

SO looks like this problem can be added to the fixes due out with Windows version 20.
