Windows System Backups

            There are a number of things that cause a computer to become unusable so it‚Äôs a very good idea to make full system backups.  Some of the things that can cause this to happen are:

  • Your hard drive can fail.

  • Your system becomes infected by a virus.

  • You accidently install a faulty program.

  • Your computer gets powered off without going through a normal shutdown.

There are numerous others, but you get the idea.

            Depending on how much of your C: drive is being used and what kind of device your C: drive resides on, the time this takes can vary greatly.  To give you an idea, here are some actual timings:

  • 50 GB SSD desktop to USB 3 hard drive took 8 minutes.

  • 88 GB laptop (5400 rpm) to USB 3 hard drive took 21 min., 11 seconds.

  • 205 GB desktop (7200 rpm) to USB 3 took 77 minutes.

As you can see, there‚Äôs a wide degree of variation depending on what kind of C: drive you have and how many GB of data you‚Äôre backing up. 

            So what do you need to have for accomplishing this?  First you need a good backup/restore program.  There are many free ones, but the two recommended here are:

  1. Macrium Reflect (download from http://www.macrium.com/reflectfree.aspx)

  2. Easeus Todo Backup (download from http://www.easeus.com/download/tbf-download.html).

Both of the above are free for non-commercial use and can be easily downloaded and installed.  They both can create a recovery CD or flash drive you can boot from to restore your C: drive in case your computer becomes inoperative.  Both of these programs can be set up to do everything automatically at a time when you‚Äôre not using the computer. 

 The other thing you need is an external hard drive to put the backups on.  You can buy 1 TB USB 3 external drives for around $50 to $60.  These should be fine for most people.  If you have more than 250 GB of used space on your C: drive, you should consider a larger capacity drive, but the great majority of people can get by with the 1 TB models.

Doing this can save you a lot of grief and expense when the inevitable problems occur.

scottgem say:

My first question is WHY are you using an unbound form. With bound forms, Access does all the I/O between the form and the table for you. All of your code is unnecessary if using a bound form. With an unbound form you have to write all the code yourself. While unbound forms have their uses, I see nothing in your code that justifies using an unbound form. If you are having  "a hard trouble to figure it out", maybe you shouldn't be using unbound forms until you improve your knowledge of VBA.

You also don't show us the code that retrieves the record for the form. 

Another question is what is the datatype for the ID field? If it is an autonumber, then you cannot UPDATE it, not should you be doing so. That could be what's causing the error. 

Also the correct syntax for an UPDATE statement is:

UPDATE table SET fieldname = value WHERE comparison;

Your SQL statement doesn't include the tablename.  And your WHERE clause is incorrect. Also I wouldn't use sql as a variable name. Try strSQL. Finally, Description is a reserved word and shouldn't be used as an object name. 

Try:

strSQL = "UPDATE Master_Data SET Category = " & Me.Update_category & _

               ", SubCategory = " & Me.update_subcategory & _

               ", Description = '" Me.update_Description & _

               "', Status = " & Me.update_Status & _

               " WHERE ID = " & Me.Update_ID & ";"

dbs.Execute strSQL, dbFailOnError

You don't need to specify the .Value property as that is the default. Also I'm assuming the Description field is a Text datatype, so I've included single quotes. I'm also assuming the other three fields are number datatypes so there is no need for delimiters.

SOLVED: (For very small values of "Solved")

Well, there is one fix for this issue that I discovered while futzing around:

1.  Download the latest .iso for your language.  Burn a DVD and stick it in the drive.

2.  Update the WIN 7 (or WIN 8) you're going to upgrade to WIN 10 (I had some trouble trying with Build 10133, so I went all the way back to WIN 7 Ultimate -- YMMV).

3.  Start your WIN 7 / 8 and open the DVD with Windows Explorer (you may want to try this with your present non-working WIN 10 and let us know if that option worked for you).

4.  Click "Startup" and follow the instructions.

5.  When you get to the page that says "This is what I am going to do" (or something similar -- I am paraphrasing from Japanese), click the "Edit" choice underneath the list (usually two items).  In the Japanese DVD, it is highlighted in blue, in the English DVD, it is highlighted in white.

6.  You should be presented with three options:

- Full upgrade.

- Upgrade preserving files and selected Windows programs / apps.

- Full clean install.

In my experience, only the last two options leave you with an installation that will allow you to create a local and MS Account and the ability to switch between them.

The "Upgrade + Preserve Files and Apps" wiped Firefox, Chrome, Avast, LibreOffice, Adobe Flash, and H-P Printer Control programs from my completed WIN 10 Build10162 upgrade as well as requiring a reinstallation of the English Language Pack, printer drivers, and VirtualBox drivers (yeah, I'm not about to install directly on an HDD just yet).  But, I am able to open my MS Account and my local account and I also was automatically upgraded to Build 10166.

So, still not quite ready for Prime Time, but this gets you closer when Microsoft can't or won't.
