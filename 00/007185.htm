teylyn say:

Forget that. I completely misread your intention. I think I have it now.  Like this:

- manually enter numbers on Sheet1

- on Sheet2 reference numbers on Sheet1

- on Sheet3 use a reference to Sheet1 that is two rows below the reference used on Sheet2

- on Sheet4 use a reference to Sheet1 that is two rows below the reference used on Sheet3

Is that it?

That construct is a bit more complex than simple cell referencing. In short, you can't get the formula in another cell and then increase its row number. You need a different approach. 

You can do this with the Index function and two helper cells in the sheets.

In Sheet2 you want the value from Sheet1, row 1. The row number 1 needs to go into a cell where Excel can read it as a parameter.

 

In Sheet3 you want the value from the row specified in Sheet2, increased by 2. You need to look up the row number used in Sheet2 and put that into a cell. Then you need another cell that has the number by which you want to increase the row. 

Consider this screenshot:

The formula in Sheet2, cell C1 is =INDEX(Sheet1!A:A,B2)

In words, Look at column A in Sheet1 and return the value of the row specified in B2 of the current sheet.

In Sheet3, there is a formula in cell B2 that fetches Sheet2!B2 and adds the value from A2.  =Sheet2!B2+A2

Then the Index formula in Sheet3 is the same: =INDEX(Sheet1!A:A,B2)

In Sheet4, there is a formula in cell B2 that fetches Sheet3!B2 and adds the value from A2.  =Sheet3!B2+A2

Then the Index formula in Sheet4 is the same: =INDEX(Sheet1!A:A,B2)

So, in each consecutive sheet, you look at the cell with the row number applied in the previous sheet and add the desired number of rows, then feed that to the Index function.

It's not a pretty construct. If you explain the bigger picture of what your project wants to achieve, there may be better ways to organise your data and simpler ways to arrive at the result.

If you're willing to use a User Defined Function stored in a General Module in your workbook.

Function PrevSheet(Rg As Range)
'accounts for more than one workbook open
'and has hidden sheets
'Bob Phillips  October 4, 2009
Dim n As Variant
    With Application.Caller.Parent
        n = .Index
        Do
            If n = 1 Then
                PrevSheet = CVErr(xlErrRef)
                Exit Do
            ElseIf TypeName(.Parent.Sheets(n - 1)) <> "Chart" And _
                .Parent.Sheets(n - 1).Visible = xlSheetVisible Then
                PrevSheet = .Parent.Sheets(n - 1).Range(Rg.Address).Value
                Exit Do
            End If
            n = n - 1
        Loop
    End With
End Function

Example of usage is. . . . .

Say you have 13 sheets,  sheet1 through sheet13. . . . sheet names don't matter.

In sheet1 you have a formula in A10  =SUM(A1:A9)

Select second sheet and SHIFT + Click last sheet

In active sheet A10 enter =SUM(PrevSheet(A10),A1:A9)

Ungroup the sheets.

Each A10 will have the sum of the previous sheet's A10 plus the sum of the
current sheet's A1:A9

Perhaps you can adjust the example above to suit your case.

jay freedman say:

That behavior indicates that the Batang font was applied to the character at the time the AutoCorrect entry was created (which was done by you or someone who created a template or document for you, since \alpha is not a built-in entry) and it was saved as a "formatted" entry.

To fix this, first insert an alpha character in any document, by typing the hexadecimal number 03B1 and pressing Alt+X. The character will be in the same font as the surrounding text (assuming that font includes the Greek alphabet, which many do).

Select the alpha character. Click File > Options > Proofing > AutoCorrect Options. Note that the alpha will appear in the With box in the dialog, and the Plain Text option button will be highlighted. Type \alpha in the Replace box, and click the Replace button.

Click Yes in the prompt:

 Then click OK in the AutoCorrect Options dialog. When you close Word, you may be asked whether to save changes in the Normal.dotm template; if so, click Yes. (The formatted Batang alpha was in the Normal template, and it has now been deleted; the new plain-text alpha is stored in a different file.)
