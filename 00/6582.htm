Unfortunately, Access uses a slightly different syntax than usual.

SELECT *
FROM Lots
LEFT JOIN (Bids LEFT JOIN Products ON Products.ID = Lots.Product_ID)

john korchok say:

This is where a comparison with a Microsoft theme can tell you what is wrong. You have set all the backgrounds as picture fills, which is effectively the furthest-back layer you can create. These are not capable of hiding or replacing normal graphics.

By contrast, if you examine a Microsoft theme, background are graphic files that can be selected. These graphics cover the background and are capable of hiding or replacing graphics. So just remove all the picture fills and replace them with regular graphics that float over the background.

I don't know if this is intentional, but you've used alpha-channel transparency in your background graphics. PowerPoint renders these transparent areas as a light grey. For use in PowerPoint, it's better to create transparent areas by using PhotoShop's Save for Web tool, which sets a single color as transparent. Depending on your version of PhotoShop, Save for Web is found under the File menu, or under File>Export.

rich007a say:

Here's an improvement:

Private Sub Worksheet_Calculate()
    Dim c As Range, rng As Range, denom As Long, slashPos As Long
    Set rng = ActiveSheet.Range("C2:C100")
    For Each c In rng
        If c.HasFormula Then
            slashPos = InStr(1, c.Formula, "/", vbTextCompare)
            If slashPos > 0 Then
                denom = Mid(c.Formula, InStr(1, c.Formula, "/", vbTextCompare) + 1)
                c.NumberFormat = "?/" & denom
            End If
        Else
            c.NumberFormat = "General"
        End If
    Next c
End Sub
