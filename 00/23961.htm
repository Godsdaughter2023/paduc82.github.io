Hello Richt_T,

 

We would be more that happy to assist you with your concern. There are possible reasons why some of our customers are getting an error message saying that 'Account does not exist' when logging in to your account. Here are some:

 

Renamed account

In some cases, some people thought they were simply adding a new address on the account only to find out that they've renamed it. In this case, all the account information like emails, contacts, calendars, etc. are now accessible using the new email address. The only change that happened was, the sign address is new and the old address became an alias and cannot be used in logging in to the account but can be used for sending and receiving emails, hence the error "Account does not exist". In some instances, account rename happens when the account gets hacked.

 

Deleted account due to inactivity

There are people who barely access the account often wonder why they receive the error "Account does not exist". In this scenario, the account gets deleted because the Microsoft account was never accessed during the 90 day reservation period after the inactivity period of 270 days has expired. The error appears because the account has been recycled after the user failed to log in in the allotted reservation period. Please note that in this case, all messages, contacts, and folders will be deleted and there's no way we can recover them anymore.

 

Now, here's my question, when was the last time you were able to successfully log in to the account? I want to exclude the possibility that the account has been recycled already because you did not access it for the period I've mentioned above. Do you remember making any change on the account (like renaming it)?

 

Giving us these details will help us determine if the emails are still recoverable.

dhookom say: I have generic list box functions in every application I create. This function accepts the listbox control, a field name, and the delimiter and returns something like:

" AND [Section] In ('PHM','DM') "

You can then use this as a form filter or to modify the SQL of the Record Source.

Function BuildIn(lboListBox As ListBox, _
strFieldName As String, strDelim As String) As String
'send in
' List box control object
' Name of the field
' Delimiter which is typically a single quote for text
Dim strIn As String
Dim varItem As Variant

If lboListBox.ItemsSelected.Count > 0 Then
strIn = " AND " & strFieldName & " In ("
For Each varItem In lboListBox.ItemsSelected
strIn = strIn & strDelim & lboListBox.ItemData(varItem) & strDelim & ", "
Next
'remove the last ", " and add the ")"
strIn = Left(strIn, Len(strIn) - 2) & ") "
End If
BuildIn = strIn

End Function
