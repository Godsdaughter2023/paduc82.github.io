I am not sure what you are asking but I have this from a Microsoft Knowledge Base Article - 210249 on how to work with a fiscal year.

 

Option Compare Database   'Use database order for string comparisons
Option Explicit   'must declare all variables

 


'HOW TO: Get the Fiscal Year or Month of a Particular Date in Access 2000

 

Const FMonthStart = 1   ' Numeric value representing the first month
                        ' of the fiscal year.
Const FDayStart = 1    ' Numeric value representing the first day of
                        ' the fiscal year.
Const FYearOffset = -1  ' 0 means the fiscal year starts in the
                        ' current calendar year.
                        ' -1 means the fiscal year starts in the
                        ' previous calendar year.


Function GetFiscalYear(ByVal x As Variant)
'    Print GetFiscalYear(#7/1/1999#)
'    Note that this line returns the year 2000.
   If x < DateSerial(year(x), FMonthStart, FDayStart) Then
      GetFiscalYear = year(x) - FYearOffset - 1
   Else
      GetFiscalYear = year(x) - FYearOffset
   End If
End Function
                   
Function GetFiscalMonth(ByVal x As Variant)
'    Print GetFiscalMonth(#8/1/1999#)
'    Note that this line returns the number 2. The 2 represents the second month in the fiscal year.

   Dim m
   m = month(x) - FMonthStart + 1
   If day(x) < FDayStart Then m = m - 1
   If m < 1 Then m = m + 12
   GetFiscalMonth = m
End Function


Function GetFiscalQuarter(ByVal x As Variant)
'    Print GetFiscalQuarter(#8/1/1999#)
'    Note that this line returns the number 2. The 2 represents the second month in the fiscal year.

   Dim m
   m = month(x) - FMonthStart + 1
   If day(x) < FDayStart Then m = m - 1
   If m < 1 Then m = m + 12
   Select Case m
   Case 1, 2, 3
        GetFiscalQuarter = 1
   Case 4, 5, 6
        GetFiscalQuarter = 2
   Case 7, 8, 9
        GetFiscalQuarter = 3
   Case Else
        GetFiscalQuarter = 4
   End Select
End Function

ken sheridan say: The following function returns the accounting year for any date in the commonly used format yyyy-yy:

Public Function AcctYear(DateVal As Date, MonthStart As Integer, DayStart As Integer) As String

    Dim dtmYearStart As Date
    
    If MonthStart = 1 And DayStart = 1 Then
        ' accounting year is calendar year, so return single year value
        AcctYear = Year(DateVal)
    Else
        ' get start of accounting year in year of date value
        dtmYearStart = DateSerial(Year(DateVal), MonthStart, DayStart)
        
        ' if date value is before start of accounting year
        ' accounting year starts year previous to date's year,
        ' otherwise it starts with date's year
        If DateVal < dtmYearStart Then
            AcctYear = Year(DateVal) - 1 & Format(Year(DateVal) Mod 100, "-00")
        Else
            AcctYear = Year(DateVal) & Format((Year(DateVal) + 1) Mod 100, "-00")
        End If
    End If

End Function

So for an accounting year starting 1 September, the accounting year for today is returned like this:

? AcctYear(#2013-02-18#,9,1)
2012-13

For an accounting year starting 1 February:

? AcctYear(#2013-02-18#,2,1)
2013-14

And for an accounting year which coincides with the calendar year:

? AcctYear(#2013-02-18#,1,1)
2013

ken sheridan say: As John has pointed out, we don't have a lot to go on, but the following query might point you in the right direction. It returns the transactions per transaction type per quarter for an accounting year beginning 1 October where the Transactions table includes a TransactionDate column:

SELECT YEAR(TransactionDate)+YearAdjust & "-" &
FORMAT((YEAR(TransactionDate)+YearAdjust+1) MOD 100,"00") AS AccountingYear,
TransactionQuarterText AS AccountingQuarter, TransactionType,
SUM(TransactionAmount) AS TotalTransactions
FROM Transactions INNER JOIN AccountingQuarters
ON MONTH(Transactions.TransactionDate)=AccountingQuarters.TransactionMonth
GROUP BY YEAR(TransactionDate)+YearAdjust & "-"
& FORMAT((YEAR(TransactionDate)+YearAdjust+1) MOD 100,"00"), TransactionQuarterText,TransactionType;

teylyn say:

Hello,

This support article has some info about the possible reasons for AutoSave to be disabled. 

"

AutoSave is enabled when working on a file that is saved to OneDrive, OneDrive for Business, or SharePoint Online. If the file is saved to another location, then AutoSave is disabled. This will happen if your file is on a SharePoint On Premises site, a file server, or saved to a local path like C:\.

There are other reasons AutoSave could be disabled as well. For example, if your file is in an older format like .xls, .ppt, or .doc., it will be disabled. It will also be disabled if your presentation is in slide show mode, or if your workbook contains password protection. If AutoSave is disabled, you can always hover over the AutoSave switch and a tooltip will tell you why it's off.

"

Unfortunately, the tooltip will not tell you exactly WHY it is off in each situation. I have already reported this to the Excel development team. Please check if your workbook meets the requirements for AutoSave.
