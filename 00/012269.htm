bob jones aka: cybertaz say:

One key point to emphasize is that setting the Default Language pertains only to new  files created from that point on and new slides/text boxes inserted into existing files. It does nothing to change the Language attribute of content in an existing presentation file.

Pursuant to that, Language in Office documents is a character level attribute similar to bold, italic, etc. except that it isn't visible. The Spelling & Grammar checking features in Office software detect the Language attribute of the text and invoke the corresponding Proofing Tools on the fly. The choice of which Proofing Tools are used is determined by the program based on the Language attribute of the text being checked, it's not a decision or choice made by the user.

This existing Answer may prove useful:  Wrong Spell Check Language

Further, it doesn't matter which idiom is used when the content is entered. Initially, the Language attribute of the text is determined by the Default Language setting in the program they're using, which typically is based on the Regional settings of the computer. If the user's Default Language is Dutch the content will be invisibly marked as Dutch even if typed in the English idiom. IOW, even what you translated to French continues to carry the Dutch attribute... For example: The sentence Parlez-vous Français ? can carry the Dutch, English, Spanish or almost any other Language attribute. In fact, each character in that sentence can carry a different Language attribute.

macropod say:

Reformatting your computer will erase Word's sources.xml file, which is where Word stores the master list. As a result, the master list on your new installation would start off empty.

Simply opening a document with reference sources doesn't add them to your master list. Imagine the chaos that could cause if you opened someone else's document just to read it...

To copy a reference from your document back to the master list:

• Click on References|Manage Sources to start the Source Manager

• Click on an entry in the current list, then click on copy.

• Repeat previous step for each reference you want to transfer.

For a large number of references you could use a macro such as:

Sub SourcesImport()
Dim oSrc As Source
'In case the source is already present
On Error Resume Next
With ActiveDocument
  For Each oSrc In .Bibliography.Sources
    Application.Bibliography.Sources.Add oSrc.XML
  Next
End With
End Sub

For PC macro installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm

For Mac macro installation & usage instructions, see: http://word.mvps.org/Mac/InstallMacro.html

You could use the above macro for importing references into your master list from any Word document, including documents others have created. Likewise, the following macro can be used to export your master list to another document:

Sub SourcesExport()
Dim oSrc As Source
'In case the source is already present
On Error Resume Next
With ActiveDocument
  For Each oSrc In Application.Bibliography.Sources
    .Bibliography.Sources.Add oSrc.XML
  Next
End With
End Sub

As a safeguard against future reference losses, you could create a new macro-enabled document named 'Sources Backup' and add the above two macros to it and use them for backup/restore purposes. Alternatively, you could use the following two macros:

Sub SourcesBackup()
Dim oSrc As Source, StrSrc As String
With ActiveDocument
  .Range.Text = vbNullString
  For Each oSrc In Application.Bibliography.Sources
    StrSrc = StrSrc & vbCr & oSrc.XML
  Next
  .Range.InsertAfter StrSrc
  .Paragraphs.First.Range.Delete
End With
End Sub

Sub SourcesRestore()
Dim i As Long
'In case the source is already present
On Error Resume Next
With ActiveDocument
  For i = 0 To UBound(Split(.Range.Text, vbCr))
    Application.Bibliography.Sources.Add Split(.Range.Text, vbCr)(i)
  Next
End With
End Sub

The difference between the two pairs of macros is that, with the the second pair, the SourcesBackup macro writes the data to the body of the backup document - and the SourcesRestore macro restores them from there.

Since the introduction of Windows Update as part of Windows, Microsoft has delivered updates as way to keep Windows updated and secure. This has always been the way the company keeps commercial versions of Windows on the market current.

With Windows 10, Microsoft is making significant changes to this model by delivering new methods of servicing Windows. Taking into account we now live in a mobile, constantly connected world, where threats happen all the time. Part of the solution to this is to keep users current by providing the latest updates as soon available in addition to delivering features as soon as they ready too.

Microsoft will do this using three types of service branches for Windows 10:

  • Security Updates, Features and Fixes are automatically applied.
  • There is no option to delay or customize these updates.
