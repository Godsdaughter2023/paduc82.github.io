tom van stiphout say:

VBA is not VB.NET, just like it is not C# or Java or Python or any other language. You have to abide by its rules and syntax. Handles is not part of that (unless you can quote me a section of the vba help file I am not familiar with :-))

 

vba is a much simpler language and it does not have powerful constructs like Handles. Rather, in each event procedure, you could call a central procedure to do some common work. As an example in one app I had 21 flexgrid controls pretending to be a calendar with 21 days. For each control I wanted to do something upon Click. So I named my controls FlexGrid1 through FlexGrid21 and in each of those 21 click events I wrote:
HandleFlexgridClickEvent(i)

with i being the number of the flexgrid control.

Then in that central procedure I might write:

Private sub HandleFlexgridClickEvent(byval intGrid as integer)

  msgbox "You clicked on grid #" & intGrid

  'TODO: something more useful.

end sub

 

That's as good as it gets with VBA.

 

ken sheridan say: With your example, as the command is related to the active control you can simply add a function to a standard module:

Public Function ZoomMemo()

    RunCommand acCmdZoomBox
    
End Function

and call it for any text box control as the On Dbl Click event property in the control's property sheet with:

=ZoomMemo()

marshall barton say:

The VBA capability that Ken described is very versatile.  This is especially convenient when you can set the OnClick property for all the selected controls in a form in one shot by selecting multiple controls and setting their common properties all at once.

 

You only need to create a standard module function when you want to use the function for more than one form.  If the function is used in more than one form, the function can use an argument as in:

   Public Function MyHandler(frm As Form)

and then refer to the active form and its control with the focus by using:

   frm.ActiveControl

and the event's On... property would be set to:

   =MyHandler(Form)

 

OTOH, when the function is specific to a single form, then the function can be created in that form's module and use:

   Me.ActiveControl

to refer to the form's control with the focus and the On... property set to:

   =MyHandler()

Ok, you asked for it.  With this setup, even if you manuall choose File --> Save As, the process is going to override you and save it the way it wants to!

A kind of BAD SIDE EFFECT of this process is that you cannot just close the file and have the changes saved - you'll get into a loop asking if you want to save changes, but the process screws it up and it becomes a perpetual loop until you respond NO (don't save changes).  So to actually close it, you have to save it first (automatically giving it a new name and clearing the "book is dirty" flag so that it will simply close for you).  If you do choose NO - then the book will close but you'd lose your changes.  Clicking [Cancel] at that point will allow you to use the Save and then go on to close the book without problems and with your changes saved.

This code is workbook event level processing.  We intercept the File-->Save event and reroute things so that a new filename is created using a basic file name string that you define and a date-time string we put together along with the file type (.xlsx, .xlsm, .xlsb or whatever) that you designate and then go ahead and 'save' the file.

To put this code into the proper place in your workbook:

Open the workbook

Press [Alt]+[F11] to enter the Visual Basic Editor (VBE)

Press [Ctrl]+[R] to make sure that the VBA Project window is visible in the VBE.  It normall appears at the left side of the whole VBE window.

You'll an entry in it like VBAProject (name of your workbook) and below that will be an entry "Microsoft Excel Objects".  If either of those entries has a + in the box next to them, click it to expand the list.  Now you are looking for an entry under "Microsoft Excel Objects" named "ThisWorkbook".

Either double-click the ThisWorkbook object or right-click on it and choose [View Code] from the popup list that appears.

Copy the code below and paste it into the code module presented to you.

Make changes to the two Const values in the code.  The first one is the basic name for your workbook.  Suppose you want all the workbooks to be named "BusyBook_2011-05-28_080523.xlsx",  then
Const basicFileName = "mySpecial workbook"
would be changed to
Const basicFileName = "BusyBook"

and
Const fileType = ".xlsm"
would be changed to
Const fileType = ".xlsx"

close the VBE and save it (it's going to get a new name right off the bat!)

If you want to set the name of this baseline copy, you'll have to rename it in the Windows Explorer window.

Here's the code:

  Const basicFileName = "mySpecial workbook" ' change as desired
  Const fileType = ".xlsm" ' change as appropriate to .xlsx or .xlsb or whatever

  Dim okToSaveFlag As Boolean
  Dim newFileName As String
  Dim dateTimeString As String
 
Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)
  If Not okToSaveFlag Then
    Cancel = True
    GiveNewName
    Exit Sub
  End If
End Sub
Private Sub GiveNewName()
  Dim rightNow As Date
  'this format of the time will help keep your files list in order
  ' by YEAR - MONTH - DAY
  rightNow = Now()
  dateTimeString = Format(Year(rightNow), "0000") & "-" & _
   Format(Month(rightNow), "00") & "_" & _
   Format(Day(rightNow), "00") & "-" & _
   Format(Hour(rightNow), "00") & _
   Format(Minute(rightNow), "00") & _
   Format(Second(rightNow), "00")
  'create new filename with date and time stamp
  newFileName = ThisWorkbook.Path & Application.PathSeparator _
   & basicFileName & "_" & dateTimeString & fileType
  okToSaveFlag = True
  ThisWorkbook.SaveAs newFileName
  okToSaveFlag = False
End Sub

rblack40 say:

conditional mail merge

Can you explain how Microsoft Word will process a Conditional Mail Merge of a letter or email? This can also be applied to labels and other elements.
doug robbins - mvp office apps & services (word) say:

An Outlook contacts file will typically have the following fields:

Last

First

Title

Company

Address

City

State

Zip/Postal Code

Country/Region

As an example, assume that in some cases, some of the contacts that you wanted to merge do not have any company association and for those contacts, you do not want to have an empty line in the address.  To achieve that you would use the following field construction where you use Ctrl+F9 to insert each pair of field delimiters { }

{ IF { MERGEFIELD Company } <> "" "{ MERGEFIELD Company }¶
{ MERGFEFIELD Address }" "{ MERGEFIELD Address }" }

Where the ¶ appears in the above construction, you would press Enter of Shift + Enter to create a new line.

You use Alt+F9 to toggle of the display of the field codes.
