INSERT INTO [target table] (...)

marshall barton say: I don't think so, but why would you want to do that?  The target table name is only used once so an alias would not serve any purpose.

ken sheridan say: Not as far as I'm aware, but why would you want to?  Aliases are used to distinguish two or more instances of the same table in a query.  As this can't arise in the case of the 'target table' there would be no advantage in doing so.
