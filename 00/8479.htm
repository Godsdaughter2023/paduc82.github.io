lori-nc say:

Fill-in the space using Word

suzanne s. barnhill say:

Any document you save as a template (.dotx or .dotm) and store in the folder for User Templates will be available from Word's File | New tab (under Personal or Custom). A document (.docx) based on the template will start out with the blanks you have created and will have to be saved under a new name, leaving the template untouched.

There are numerous ways to create fillable forms, but to me the simplest method for this type of document (and one that will work in any version of Word) is the use of MacroButton NoMacro fields. The construction of these is described at http://word.mvps.org/FAQs/TblsFldsFms/UsingMacroButton.htm, but the bottom line is that a MacroButton field is intended to run a macro when double-clicked, but if you name a macro that doesn't exist (NoMacro), then it doesn't do anything except sit there. The beauty of it is that, as with any other field, when you click in it, the entire field is selected and will be overwritten by whatever is typed.

So the way this works is that you put in this MacroButton field:

{ MacroButton NoMacro [Click here and type name] }

and what you see in the document is:

[Click here and type name]

You can click anywhere in that text, and it will all be selected so that (provided you have "Typing replaces selection" enabled, which it is by default), what you type will replace the field, becoming ordinary text.

Unlike a document with legacy form fields, this template/document doesn't have to be protected for forms, and unlike a template/document with content controls, you don't have a lot of visual clutter. You can't use Tab to advance from one field to another, but, if the document doesn't contain other fields, you can use F11.

Of course, the Cadillac of forms is the UserForm, but creating one of those does require some familiarity with VBA, whereas anyone can create MacroButton fields (just press Ctrl+F9 and type in the content).

jay freedman say:

This can get to be a very long discussion, but here are the simplest parts:

  • First, use an actual template (.dotx file), not a 'template' document (.docx) file. The main reason is that most people tend to forget to use Save As, and when they use just Save, that overwrites the 'template'. Then they'll come to you whining that they can't get back to the original 'template'. As it is, you'll have to keep reminding them to use File > New to create a new document based on the template. (Hint: Have them add the "New Document or Template" button to their Quick Access Toolbar -- it's in the Customize dialog, in the category of Commands Not in the Ribbon -- and use that to select your template.)
  • To create a new template file, start with a blank document and immediately save it; in the Save As dialog, click the "Save as type" dropdown and choose "Word Template (*.dotx)".
  • In the body of the template, insert the parts of the text of the contract that won't change from one document to the next.

Now you have a decision to make. If all the sales reps have Office 365, you can use content controls, but if any of them have Word 2003 (a very old version, but still popular) you'll have to use legacy form fields instead. The buttons for inserting both of these things are on the Developer tab of the ribbon, which is disabled by default. To enable it, go to File > Options > Customize Ribbon and check the box next to "Developer" in the right-hand list.

  • If you're using content controls, click at a place in the text where the rep must enter some information, and then click the appropriate button in the Controls group of the Developer tab. Most often you'll need a plain-text control, but there are other kinds, as explained in the article I linked above.
    • When you've inserted all the content controls you need, select the whole template (press Ctrl+A) and click the Group button on the Developer tab, and choose Group from the menu. That will prevent any changes in the parts of the contract that aren't inside one of the controls.
  • If you're using legacy form fields, click at a place in the text where the rep must enter some information, and then click the Legacy Tools button in the Controls group of the Developer tab. In the menu that drops down, choose the appropriate kind of form field from the top row. Most often you'll need a text form field, but there are several other kinds.
    • When you've inserted all the form fields you need, click the Restrict Editing button on the Developer tab, which opens a pane on the right. Check the box for "Editing restrictions". In the dropdown below that, select "Filling in forms". Click the Start Enforcing button, and enter a password if you want to use one (you can leave it blank).
  • For either content controls or legacy form fields, you may want to use one or more tables to align and constrain the entry locations. You can turn off a table's border lines so the contract doesn't show or print them.

There's lots more to talk about if your contract is more complicated than simply filling in a few entries.
