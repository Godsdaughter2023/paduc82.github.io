graham mayor say:

There is no 'autosave' in Word. What you are saving is auto-recovery information, which is used by Word to attempt to restore the document in the event of a system crash. See http://www.gmayor.com/automatically_backup.htm

Copying all of the document isn't going to help. Display the hidden formatting characters by pressing CTRL+SHIFT+8 (or the ¶ button) then copy all but the last paragraph mark ¶ to the clipboard and paste to a new document.

See also http://word.mvps.org/FAQs/AppErrors/CorruptDoc.htm

If you have been regularly saving the document and have the option set to automatically create a backup, then the backup will be the previously saved version. Thatmay be more amenable to recovery.

Frankly I don't see what Norton would have to do with this issue. In any case it is probably better to switch off Norton's active document checking as it only slows down Word to no good purpose, but you can do that later.

The default template is 'normal.dot' or 'normal.dotm' depending on Word version. It is a normally hidden file, but if other documents are OK, I don't see how deleting or renaming it would affect this document. If you want to check before doing anything so drastic, start Word in its safe mode - by holding the CTRL key whilst starting and acknowledge the safe mode prompt that ensues.

Orphan temporary files can cause problems, http://www.gmayor.com/what_to_do_when_word_crashes.htm but they won't fix a corrupt document.

Configuring Skype SIP with Cisco CP-8851 - Microsoft Community

nici789 say:

Configuring Skype SIP with Cisco CP-8851

Because I haven't found much on the internet, I'm posting instructions for how to configure a Cisco business class phone (CP-XXXX, not SPA-XXXX) with Skype Connect. No PBX is needed. 

1) Setup a Skype Connect account, add a SIP profile, and assign a phone number to the SIP profile. In the SIP profile, look for the "Authentication details" section. On the "Registration (username/password)" tab, note the SIP User, Password (click the link to have one sent to you if needed), Skype Connect address (should be sip.skype.com) and the UDP Port (should be 5060).

2) Install a TFTP server on your computer. This is how you will configure the phone. The CP-XXXX series of phones cannot be configured via a web browser like the "small business" SPA-XXXX phones. "Open TFTP Server" is available on SourceForge and works well enough. Once installed, I had to manually start the server via the Windows Services panel. I also had to turn off the Windows Firewall so that my phone could find the TFTP server. By default, the server offers files in this directory "C:\OpenTFTPServer", so this is where you'll place files that need to be sent to the phone.

3) Connect the Cisco phone to your network.

4) Configure your router so that your PC (where the TFTP server is located) and Phone have static private IP addresses. For example, my phone has 192.168.1.17, and my PC has 192.168.1.15

5) On the phone, open settings (push the "gear" hard button) and open "Phone Information".  Note the hostname, which should be a long hexadecimal name starting with SEP

6) On the phone, open settings and open "Admin Settings", "Ethernet Setup", "IPv4 setup". Scroll down to "Alternate TFTP" and set the option to "On". Enter the IP address of your PC in the field "TFTP server 1" (i.e., for me this is 192.168.1.15)

7) On your PC, create a text file with the name SEPXXXXXXXXXXXX.cnf.xml (replaces the X's with your phones hostname noted in step 5) and paste the contents at the end of this message into the file. Update a few of the lines to match your configuration:

<timeZone>Mountain Standard/Daylight Time</timeZone>

<phoneLabel>YOUR NAME</phoneLabel> 

<featureLabel>SKYPE PHONE NUMBER</featureLabel> 
<name>SKYPE CONNECT USERNAME</name> 
<displayName>SKYPE PHONE NUMBER</displayName> 
<authName>SKYPE CONNECT USERNAME</authName> 
<authPassword>SKYPE CONNECT PASSWORD</authPassword> 
<contact>SKYPE PHONE NUMBER</contact> 

These websites may be helpful for descriptions of the options:

https://www.voip-info.org/wiki/view/Standalone+Cisco+7941/7961+without+a+local+PBX

https://www.voip-info.org/wiki/view/Asterisk+phone+cisco+79x1+xml+configuration+files+for+SIP

8) Place the cnf.xml file in the TFTP server directory as noted earlier.

9) Configure your router to forward specific UDP ports to the phone's IP address.

     a) Service Name SIP, UDP 5060

     b) Service Name RTP, UDP 16384-20000 (note that these should match  the <startMediaPort> and <stopMediaPort> in the cnf.xml file. And really, you only need one port, unless you have multiple lines.

10) Reset your phone. It should register with skype and then the settings you edited in the cnf.xml should be reflected on the phone's main screen. The Skype Connect authentication tab shown in step 1 should show that the phone has registered. If it doesn't appear to work, check that the phone is reaching the TFTP server. You may have to disable the windows firewall on your PC to get it to work, then reset the phone (it only looks for the cnf.xml file during a reboot)

11) Once your phone is working, don't forget to turn on your windows firewall if you turned it off to get the TFTP server working. You only need to use the TFTP server if you need to edit the cnf.xml file. The phone only looks for the server when you reset (reboot) the phone. So, otherwise, you can leave the firewall on and the TFTP server off.
