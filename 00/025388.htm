brian tillman [outlook mvp 2007-2019] say:

Outlook 2013 can connect to Yahoo's calendar as an internet calendar.  Go to http://calendar.yahoo.com/, log in with the Yahoo credentials of the calendar owner, and click Actions>Share>Continue.  Check"Generate links" and copy the URL that appears in the box labeled "To import into a Calendar app (ICS)".  Click Save.

Now, in Outlook, click File>Account Settings>Account Settings.  Select the "Internet Calendars" tab and click New.  Paste into the dialog the URL you copied from the previous step.  Click Add.  You will be prompted to enter the credentials for the Yahoo account.  Do so, check the box to save the credentials, and click OK.  You'll get one more dialog to give the calendar a name and set a few other options (which I just left defaulted).  OK that one as well.  The Yahoo calendar should appear in the Calendar and Folders views of the Navigation Pane.  Note that the calendar will be read-only from the Outlook side, but if you update the calendar online, the next time Outlook performs a send/receive, the events will show up.

This all worked for me, so I'm pretty sure it will work for you.  Note that if you have two-step verification enabled on the Yahoo account, you'll need to generate an app password to use when prompted for the Yahoo credentials.

a. user say:

On-Prem Mailboxes Synced to Exchange Online in a Hybrid - Is there a limit?

In a hybrid environment is there a limit, or maximum recommended number, of on-prem mailboxes that can be synchronized with Exchange Online? I know the default number of concurrent incremental syncs is 10 and that number can be bumped up to 40. What my organization is planning to do is to get all of our mailboxes synced over a period of weeks (~1500 mailboxes in total) and then finalize them all over a short period of time. 

I haven't been able to find any documentation that states there is a limit to the number of mailboxes that can be in a synchronized state. But given that the default number of concurrent syncs is only 10 and we are going to have over 1000 incrementals happening daily, I wanted to verify that what we are doing is a recommended scenario.

neo yu msft say:

Hi JoeLaughlin,

We don’t have any limitations unless the migration exceeds the Resource health-based throttling. Resource health-based throttling is the least aggressive throttling method and occurs only when there is a service availability issue that affects end users and critical service operations. For example, if a service incident occurs during a hybrid migration, and the service degrades to the point where end-user performance is degraded, the hybrid migration will be queued until performance is recovered and the service returns to a level below the throttling threshold.

Another part is the Migration-service throttling, like the default number of concurrent incremental syncs is 10. This part can be modified in the migration batch.

The observed throughput per hour and per client migration speed is 10-14 GB per on-premises Exchange 2013 or 2010 CAS (MRS Proxy) with 20 concurrent moves in a hybrid deployment. Observed single mailbox move throughput is in the 0.3–1.0 GB/hour range. Greater than 1000 MB/h per mailbox throughput rate can be achieved with a network that can sustain less than a 2% transient failure stall time and less than 100ms network latency. More concurrent mailbox migrations can be used to achieve higher data migration rates. Single mailbox move throughput will slow down when the on-premises CAS (MRSProxy) server is at hardware capacity, if the network bandwidth isn’t sufficient, or the network latency is too high. Consider adding more servers or temporarily improving network connectivity to increase migration velocity.

For more information, please refer to: Exchange Online migration performance and best practices.
