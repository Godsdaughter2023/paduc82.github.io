steve rindsberg say:

I realize that pasting as picture isn't always a solution, but the fact that it allows you to print does give us a bit more information.

Sometimes vector graphics can get overwhelmingly complex and cause printing problems or, if you're really having a bad hair day, can totally crash PowerPoint.  Or it might just be a problem with the printer driver.

Some other things to look at, then:

In some cases, you don't so much need to edit the underlying map as the stuff placed ON it.  Company facility locations and that kind of thing, for example.  You could start with a map that's been converted to a picture but add the other stuff atop it and leave that as vector graphics.

How did the graphics get into PPT in the first place?  For example, if they started as Illustrator drawings with lots of curves, then got copy/pasted into PPT, they may have come in as WMFs, which can't represent curves and turns them into skillions of straight line segments.  Paste Special, As EMF would give better results usually.  To check this, ungroup one of the maps and try using Edit Points on a few of the shapes that include curves.  If you hit one that's been WMFed, you'll know it right away.  Points everywhere.

Try printing to the Windows XPS driver to see if that allows you to print.  If so, you might want to check for printer driver updates or test different printer modes.

And also go to File | Options | Advanced and scroll down to the Print area.  Make sure there's no checkmark next to "Print in background".  That silly thing's been a source of problems since PowerPoint 97 if not before.  MS really needs to turn it off, leave it off, and remove the option for turning it on.   Since they won't, we have to see to it.
