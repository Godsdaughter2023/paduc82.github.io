Method 1:  Hardware and devices troubleshooter

The built-in hardware and devices troubleshooter can automatically detect and fix most of the incorrect settings and errors that could cause issues with hardware. I would suggest you to run this troubleshooter to check if there is any issue with the display adapter. Refer top these steps:

a) Open Control Panel

b) Click on Troubleshooting and select the view all option on  the left pane.

c) Click on Hardware and devices troubleshooter and follow the prompts.

Try method 2, if the issue persist

Method 2:

If the issue reoccurs after running Hardware troubleshooter, then the issue could be caused due to a conflict between the display driver and other program. In this case, I would suggest you to boot the computer in a clean boot state and check if the issue persists. If you don’t face the issue in clean boot state, then one of the program disabled is causing the issue. Refer to the article for more information:

http://support.microsoft.com/kb/929135

Note: After troubleshooting, reset the computer to start as usual. Refer to the step How to set the computer to start as usual after clean boot troubleshooting under more information.

Try method 3, if the issue persist

Method 3:

a)  Press Windows key + X, select Device manager.

b)  Expand the “Display adapters” section and locate the display adapter.

c)   Right click the device and select “Properties”.

d)  Click on the driver tab to check the current driver installed for that device.

e)  Click on the uninstall button to uninstall that driver.

f)   Restart the computer, now windows will load the generic display driver for this device. Check if the issue persists with the generic driver.

If you haven't found the solution then it's pretty simple although I did struggle to find out how.

On a Surface Pro 4 with Windows 10 (x64), in Cortana, search for 'Windows Features'.  When the Windows Features app starts, uncheck 'Hyper-V' and restart your Surface.  When it restarts, you should see 64-bit options in VirtualBox.  If not then you may need to uninstall and re-install VirtualBox (or Repair) to make sure it's installed with it's own 64-bit virtualisation components (mine worked without having to re-install or repairing VB).
