How to keep table height fixed/static in PPT 2010 - Microsoft Community

leo_sun say:

How to keep table height fixed/static in PPT 2010

In PPT 2010, the table in our presentation needs to be kept the fixed/static height when user manually input any content in any row(rows) or add row or delete row, so is there any properties can be configured to implement it?
john korchok say:

PowerPoint doesn't have an option for fixed or exact height rows. As a workaround, you can insert a Word document, create a table in it and set the row height to be exact:

  1. In PowerPoint, choose Insert>Object.
  2. Leave the Create new option selected.
  3. Scroll down the list of Object types to find Microsoft Word Document. Select it and click on OK. A new blank Word document is created on the slide.
  4. Choose Insert>Table and select the number of rows and columns. Click inside the first row.
  5. On the Table Tools>Layout tab, choose Properties and select the Row tab.
  6. Check Specify height, set the row height and change Row height is: to Exactly.
  7. Click on the Next Row button.
  8. Repeat step 6 and 7 until you have set the row heights for the entire table.

Unlike a PowerPoint table, users must double-click on this table to activate it before entering text.

Hi,

The result of 7/5/17 is correct.  July 1, 2017 is a Saturday.  3 business days from Saturday will be a Wednesday which is July 5. 2017.  I think what you want is not the first day of the month but the first working day of the month (Sat/Sun being weekend).  If that be the case, then try this

=WORKDAY(DATE(2017,7,1)+CHOOSE(WEEKDAY(DATE(2017,7,1),2),0,0,0,0,0,2,1),4-1)

The result of this formula will be July 6, 2017.  I have basically adjusted the first input of the WORKDAY() function to return the first Monday of the month and from there go 3 working days forward.
