ken sheridan say:
Is there a way to force Access to force a save as soon as the user initiates a new record. 

Yes, but it's unnecessary and inadvisable as it undermines your ability to protect the integrity of the database by disallowing Nulls in the non-key columns.

For a solution which directly address this issue see CustomNumber.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

The 'Sequential Numbering' option in this demo file addresses the possibility of conflicts in a multi-user environment, using an amended version of the method originally published by Roger Carlson (the file includes a link to Roger's original).  The error which occurs in the event of a conflict is trapped and resolved by incrementing the number.

Another, albeit more complex, option in the demo is for a 'Pseudo Autonumber'.  This differs from the above method in two ways.  Firstly it has the advantage that the number once obtained cannot change, whereas with the above method, if another conflicting user saves their record first the number originally seen by the current user will not be that saved as this will be incremented.  Secondly, it behaves like a true autonumber in that, in the event of the insertion of a new row being aborted the number is discarded and not re-used, i.e. the method is designed to generate distinct incrementing numbers, but not sequential numbers.
