john korchok say:

I'm seeing the same bug under Windows 7. Here are 2 workarounds you can try. The first is to save without the preview, then add it back in again:

  1. With your presentation open, choose File>Info.
  2. Click on the Properties heading and choose Advanced Properties.
  3. Uncheck Save preview picture and OK out.
  4. File>Save As to a new name.
  5. Repeat steps 1 - 3, but check Save preview picture, OK out and save again.

If that doesn't force a refresh of the icon, try the second method, deleting the thumbnail:

  1. Add .zip to the file ending and click OK when Windows warns you.
  2. Expand the file.
  3. Open the docProps folder and delete thumbnail.jpeg.
  4. Rezip the folders and the [Content_Types].xml file.
  5. Rename the file ending to .pptx.
  6. Open the presentation in PowerPoint and click OK when it asks if you want to repair.
  7. Resave the file and it should have the correct icon.

jay freedman say:
The Bibliography style simply uses the same line spacing as the Normal style that's applied to the whole document.

According to the General Guidelines at http://owl.english.purdue.edu/owl/resource/747/01, the entire paper should be double-spaced. Therefore, you should modify the Normal style to be double-spaced, and the bibliography will then be double-spaced as well.

On the Home tab of the ribbon, right-click the Normal style and click Modify. In the center of the Modify Style dialog, there are three buttons showing horizontal lines; they represent single-spaced, 1.5-line-spaced, and double-spaced. Click the double-spaced button and click OK. (At the bottom of the dialog, you should leave the "Only in this document" option selected, to avoid double-spacing all new documents.) Click OK, and save the document.

john mcghie say:

Hi Toby:

It seems to me that you have always been crashing in the "QD" module.  If that is "QuickDraw" then it indicates you have a bad font in there somewhere.  The fact that it is blowing up in a new user suggests that the font is in the System library, not the User library.

I suggest a quick test:  Go to Font Book, select ALL fonts on the system and disable them ALL.

You can’t disable fonts that are used by Mac OS X. If a font is needed by Mac OS X, when you select the font in the Font column a lock icon appears in front of the name of the font at the bottom of the Font Book window.

Now re-start the system (must re-start, all the way to power-off, to clean out the font caches).

Now test: Things may look very ugly, but do they crash?  If they don't crash, the problem is a bad font. See below.

If it's still crashing, you need to do a complete Remove and Replace (without skipping any steps).  Even if you have done a step before, once the system runs, it needs to be done again.  Here are the instructions to Remove.  You must follow them exactly, and in the correct sequence, or they won't work:  http://support.microsoft.com/kb/2398768

If the issue is a bad font, then we need to find which one.

Re-enable the fonts in batches.  Re-start the system each time to force the caches to rebuild.  Keep a record of what you have re-enabled.  When the problem returns, the bad font was in the batch of fonts most recently enabled.

You can be selective: first, re-enable only the fonts supplied by Apple.  Then the fonts supplied by Microsoft (and check for duplicates, there will be some).  Remove the duplicates: remove the oldest copy in each case.

Fonts to be suspicious of are non-TrueType fonts.  Microsoft Office will crash if offered non-standard TrueType fonts.  It cannot use Type 1 fonts at all.  And it can use Open Type Fonts ONLY if they contain TrueType characters (which most of them do, get back to us if you do not know how to check for this). PostScript fonts, and raster fonts (Type 1) cannot be used by Office, and won't be displayed.  But they shouldn't crash it. You should be extremely suspicious of freeware TrueType fonts: those that don't follow all the rules can crash Word.

Hope this helps

vluzacn say:

I solved this for my friend's computer. The problem is exactly the same. Whenever I clicked Excel.exe or any other program of Office, nothing happens for about 20-40 seconds, and then a message is displayed saying there's some problem. Trying to run a quick repair will instantly gives me the error 30175-13. The online repair does work, but only temporarily, and the problem is back after the next reboot.

I tried to investigate into the problem but with no success. For Office 365, there are two processes that are central to the problem. One is excel.exe and one is officeclicktorun.exe. The former is launched every time you clicked excel.exe and the latter is always running in the background as a Windows service. I suspected that when Windows restarted, the officeclicktorun service failed to start for some reason. In that case, it will explain all the mysteries I have. However, all the two processes seemed to be running properly. I checked the Procmon, and there were no errors at all, and there were no internet traffic, so the problem is definitely not caused by firewall. I checked the log files of the two processes, one in C:\Windows\Temp and another in C:\Users\your name\AppData\Local\Temp. The log files didn't tell me what the error was. They only mentions an errorcode, which may have something to do with COM or RPC. So probably something went wrong when the two processes tried to communicate with each other. Anyway, I concluded that the problem is caused by Office itself and the only solution is to reinstall Office.

Finally I just followed the instruction of "Uninstall Office manually" inside the third link in Aravinda's answer, and then restarted the computer and reinstalled Office 365, and my problem was solved. The instruction is not very clear and there are some spelling errors in it, but generally you have to remove all the things that you are sure is a part of Office15. And there is one dll file in "Program Files" that you can only remove it after you have terminated explorer.exe. Also don't forget any corresponding registry keys in Wow6432Node and any corresponding files in VirtualStore. I had also run the Chkdsk and the Sfc before that.

I didn't try the Fix It tool, but you can try it first. From my experience, it did once solve a problem in installing Office on my computer.
