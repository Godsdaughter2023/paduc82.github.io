I’m very pleased to see that Microsoft people are talking about the automatic Windows Defender scan these days – but the Windows Defender Scheduled Scan task is still strictly a Quick Scan. And, as always, you don’t want to convert this system task into a Full Scan (contrary to what some people might suggest), because Automatic Maintenance just wasn’t designed to handle a Full Scan (and you also don’t want to add a trigger to activate the Windows Defender Scheduled Scan task, because it’s already being triggered by a hidden Automatic Maintenance task). So if you manually run the Windows Defender Scheduled Scan task, you'll always be running a Quick Scan, and a Quick Scan is already being run by Automatic Maintenance on a daily basis.

http://answers.microsoft.com/en-us/windows/forum/windows_8-performance/windows-defender-windows-8-schedule-option-missing/b6603c6a-cba9-4ba7-a894-37f895fc6c92

So more likely than not, it'll be a Full Scan that you want to run manually – and in order to run a manual Full Scan, you'll still need to use the Windows Defender user interface. The good news is that the user interface is way more accessible now that we have our Start menu back, and since we also now have an icon for Windows Defender in the notification area. You can start Windows Defender with our happy-to-be-back Start menu (All apps > Windows System > Windows Defender). Then, to make the Windows Defender icon visible – just drag it out of the “Show hidden icons” box (the “up arrow” icon) onto your taskbar. Then, if you want the icon present at startup, you’ll need to start the Windows Defender user interface at startup – either with a registry startup entry (which Brink’s reg file adds); or with a DIY solution like the one I described here:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/is-windows-defender-icon-supposed-to-be-in-system/6cfa9d1d-3db8-49a9-add7-280a284ccfdc

If you want an automated Full Scan, you’ll still have to set up your own scheduled task with the Create Basic Task Wizard. You'll find some resources and guidelines for doing that in this recent thread:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-10-how-do-i-set-windows-defenders-scan/f399bb87-37c9-4a74-bb35-1d7e3d3c2436

shane devenshire say:

1. You can use something like

=IF(ISERR(myformula),"",myformula)

but if there are a lot of formulas you may not want to do that.

2. You can suppress the errors at print time but leave them on the screen by choosing File, Page SetUp, Sheet tab and changing the Cell errors as option.

3. You can apply conditional formatting as follow:

a. Highlight the cells and choose Format, Conditional Formatting, and pick Formula is from the first drop down

b. Enter a formulas such as

=ISERR(E9)

c. Click Format, and on the Font tab choose White as the font Color.  The key to getting this to work properly is that the formula must reference the active cell of a selected range.  So in the above example, I highlighted E9:F100 with E9 the active cell.

Public Sub DeleteUnusedStyles()

 

'Simon Jones, MillStream Designs Ltd, 05/03/2007

'Email: *** Email address is removed for privacy ***

 

Dim objStyle As Word.Style

Dim intBuiltInStyles As Integer

Dim intStyle As Integer

 

'Count the number of built-in styles in this document

 

For Each objStyle In ActiveDocument.Styles

   

    If objStyle.BuiltIn Then

        intBuiltInStyles = intBuiltInStyles + 1

    End If

   

Next objStyle

 

'Show summary and ask permission to continue

 

If MsgBox("There are " & intBuiltInStyles & " built-in styles " & vbCrLf & _

          "and " & ActiveDocument.Styles.Count - intBuiltInStyles & _

          " custom style(s) in this document." & vbCrLf & vbCrLf & _

          "Delete unused custom styles?", vbOKCancel + vbExclamation, _

          "Delete Unused Styles") = vbOK Then

         

    'OK to continue

    'Run through all the styles in the document BACKWARDS -

    'so we only delete styles from the end of the collection

   

    For intStyle = ActiveDocument.Styles.Count To 1 Step -1

       

        If Not ActiveDocument.Styles(intStyle).BuiltIn Then

            'This is a custom style

            'See if it is in use

           

            With ActiveDocument.Content.Find

                .ClearFormatting

                .Style = intStyle

                .Execute Format:=True

               

                If Not .Found Then

                    'Style is not in use

                    'We can delete it

                    

                    With ActiveDocument.Styles(intStyle)

                        'If this style is linked to another we have to unlink them -

                        'otherwise both styles will be deleted which causes problems -

                        'this style may be linked to a built-in style which will cause -

                        'an error if we try to delete it.

                        

                        If .Type <> wdStyleTypeCharacter Then

                            'Character styles can't be linked from, only linked to

                           

                            'The Linked property only exists in Word 2007 and above -

                            'for previous versions, comment out the "If" and "End If" lines below

                           

                            If .Linked Then                 '### Word 2007+ only

                                'The LinkStyle property only exists in Word 2002 and above -

                                'for previous versions, comment out the lines below

                               

                                If Not .LinkStyle Is ActiveDocument.Styles(wdStyleNormal) Then '### Word 2002+ only

                                    .LinkStyle = wdStyleNormal                  '### Word 2002+ only

                                End If                                          '### Word 2002+ only

                               

                                'LinkStyle is a very badly behaved property and this is the -

                                'closest we can get to unlinking styles short of copying all -

                                'text except the last paragraph mark to a new document.

                            End If                          '### Word 2007+ only

                            

                        End If

                       

                        'Ignore any error caused by trying to delete

                        'a style whose name begins with a space

                        On Error Resume Next

                        .Delete

                        On Error GoTo 0

                    End With

               

                End If

               

            End With

           

        End If

   

    Next intStyle

   

    'Display summary

    MsgBox "There are " & ActiveDocument.Styles.Count - intBuiltInStyles & _

           " custom style(s) left in this document.", vbInformation, "Unused Styles Deleted"

End If

 

End Sub

I have this macro that deletes unused user styles from a document. Here it is:

This is how to clean out a document of all user styles that are not actively in use—the Style.InUse property is of no help here since InUse is TRUE if the style was EVER used.

So, here's a function that checks if a style is actually in the document.

The sample routine nukes user styles that are not in the document, and hides built-in styles from the drop down list if they are not actively in use.

Sub NukeStyleNotInDocument()

 

‘ Macro: NukeStylesNotInDocument

‘ by Guy Gallo

‘

' Purpose: Remove all user styles that are not ACTIVELY

' in use in the current document

' Remove built-in styles that are not ACTIVELY in use

' from the style list

 

'Adapted from a macro by Paul Cornell

'http://office.microsoft.com/assistance/2002/articles/pwWordStyles.aspx

 

Dim strTitle As String

Dim astrStyles() As String

Dim objStyle As Style

Dim objDocument As Word.Document

Dim intCount As Integer

 

' Get the title of the active Word document.

 

strTitle = _

 

ActiveDocument.BuiltInDocumentProperties(wdPropertyTitle)

 

' Store, in memory, each style in the active document,

' including whether the style is in use.

 

 

For Each objStyle In ActiveDocument.Styles

ReDim Preserve astrStyles(intCount)

 

'GG Mod

 

If objStyle.BuiltIn = False Then

   If Not StyleInDoc(objStyle) Then

      Debug.Print "Removing " & objStyle

      objStyle.Delete

   Else

      Debug.Print "In use: " & objStyle

      astrStyles(intCount) = objStyle & " (Style in Use)"

      intCount = intCount + 1

   End If

Else

   If objStyle.InUse And Not StyleInDoc(objStyle) Then

      objStyle.Hidden = True

      Debug.Print "Hiding: " & objStyle

   Else

      Debug.Print "In Use: " & objStyle

      astrStyles(intCount) = objStyle & " (Style in Use)"

      intCount = intCount + 1

   End If

 

   '**********

End If

 

Next objStyle

 

' Create a new Word document to report the results.

 

Set objDocument = Word.Documents.Add

With objDocument.Range

 

   ' Report the results.

 

   .InsertAfter "Styles in " & strTitle & ":"

   .InsertParagraphAfter

   .Collapse Direction:=wdCollapseEnd

 

   For intCount = 1 To UBound(astrStyles)

      .InsertAfter astrStyles(intCount)

      .InsertParagraphAfter

      .Collapse Direction:=wdCollapseEnd

   Next intCount

 

End With

 

MsgBox Prompt:="Done!"

 

out:

End Sub

 

Function StyleInDoc(sty)

 

'gg

'Check if a style is in active use.

 

Dim myRange As Range

Set myRange = ActiveDocument.Content

 

With myRange.Find

   .ClearFormatting

   .Style = sty

   .Text = ""

   .Replacement.Text = ""

   .Forward = True

   .Wrap = wdFindContinue

   .Format = True

   .MatchCase = False

   .MatchWholeWord = False

   .MatchWildcards = False

   .MatchSoundsLike = False

   .MatchAllWordForms = False

   StyleInDoc = .Execute(FindText:="", Forward:=True, _

   Format:=True) = True

   .ClearFormatting

End With

 

End Function

Sub NukeStyleNotInDocument()

 

' Macro: NukeStylesNotInDocument

' by Guy Gallo

' modified by Ben Raynes

 

' Purpose: Remove all user styles that are not ACTIVELY

' in use in the current document

 

' Remove built-in styles that are not ACTIVELY in use

' from the style list

 

'Adapted from a macro by Paul Cornell

'http://office.microsoft.com/assistance/2002/articles/pwWordStyles.aspx

 

    Dim strTitle As String

    Dim astrStyles() As String

    Dim objStyle As Style

    Dim objDocument As Word.Document

    Dim intCount As Integer

    Dim intTotal As Integer

 

    ' turn off screen updating and automatic repagination to speed things up a bit

    Application.ScreenUpdating = False

    Options.Pagination = False

 

    Application.ActiveDocument.UndoClear        ' clear the Undo queue

 

    ' Get the title of the active Word document.

    strTitle = ActiveDocument.BuiltInDocumentProperties(wdPropertyTitle)

 

    ' Store, in memory, each style in the active document, including whether the style is in use.

    intTotal = ActiveDocument.Styles.Count

 

    For Each objStyle In ActiveDocument.Styles

 

        ReDim Preserve astrStyles(intCount)

 

        If objStyle = "Normal" Then

 

            'MsgBox "Reached the normal style!  Ignoring... "

            Debug.Print intCount & " of " & intTotal & ": " & objStyle & " - IGNORING!"

            intCount = intCount + 1

 

        'check to see if an object style is built in and if it isn't, delete it.

        ElseIf objStyle.BuiltIn = False Then

 

            'Debug.Print intCount & " of " & intTotal & ": " & objStyle & " is a built-in style."

            'intCount = intCount + 1

 

            If Not StyleInDoc(objStyle) Then

 

                Debug.Print "Style " & intCount & " of "; intTotal & " - Removing: " & objStyle

                objStyle.Delete

                intCount = intCount + 1

 

            Else

 

                Debug.Print "Style " & intCount & " of "; intTotal & " - In use: " & objStyle

                astrStyles(intCount) = objStyle

                intCount = intCount + 1

 

            End If

 

        'check to see if an object style is built in and if it is, hide it.

        Else

 

            'Debug.Print intCount & " of " & intTotal & ": " & objStyle & " is a user-defined style."

            'intCount = intCount + 1

 

            If objStyle.InUse And Not StyleInDoc(objStyle) Then

 

                objStyle.Hidden = True

                Debug.Print "Style " & intCount & " of "; intTotal & " - Hiding: " & objStyle

                intCount = intCount + 1

 

            Else

 

                Debug.Print "Style " & intCount & " of "; intTotal & " - In Use: " & objStyle

                astrStyles(intCount) = objStyle

                intCount = intCount + 1

 

            End If

 

        End If

 

    Next objStyle

 

    ' Create a new Word document to report the results.

    Set objDocument = Word.Documents.Add

    With objDocument.Range

 

        ' Report the results.

        .InsertAfter "Styles in " & strTitle & ":"

        .InsertParagraphAfter

        .Collapse Direction:=wdCollapseEnd

 

        For intCount = 1 To UBound(astrStyles)

 

            .InsertAfter astrStyles(intCount)

            .InsertParagraphAfter

            .Collapse Direction:=wdCollapseEnd

 

        Next intCount

 

    End With

 

    MsgBox Prompt:="Done!"

 

    ' turn screen updating and automatic repagination back on

    Application.ScreenUpdating = True

    Options.Pagination = True

 

out:

 

End Sub
