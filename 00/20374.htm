Wait........wait......!    Unless you just went in an consciously deleted the iexplore.exe file, it is NOT missing.  It only appears that way because, when the Internet Explorer 10 box ( in "Turn Windows Features on and Off")  is UNCHECKED, it means that you want that feature turned OFF.  The system responds by removing the executable file (iexplore.exe) during its reconfiguration so that you can't start the feature.  CHECKING the box replaces the .exe program file and the feature will start..

 

Try this process one more time..........and restart your computer afterwards so that Windows can reconfigure the operating system.

kambanka say: Yep, .exe files are unknown about Symbian.
The way to execute S60 application is to install it. So, no matter how the application got to the phone, you need to explicitly install it. From that point on it can access most of the phone, but still cannot access vital locations (unless the Nokia signature is spoofed and your device thinks the application has come from Nokia itself). Furthermore, S60 won't allow execution of applications outside some specific folders. So, executable file in the 'Music' folder is just a file which cannot be run.
As for S40 - you can install only Java applications and they are in no way harmful. Meaning that they can send 100 SMSes, but still need permissions - this affect S60, too.
The bottom line is that it is vital that you install applications only from trusted sites. Because once installed, applications (especially Symbian ones) can do quite a lot of damage - including deleting all your Contacts; though, you will be warned upon installation what access the application requires - 'Reading user data', for example.
The SMSes crashing 3310 was due to wrong implementation of the SMS format decoding. It was not a 'virus', because it could not replicate itself and did no harm but to make deletion a bit of a hurdle.
There was a 'virus' about Symbian, though. It spread over Bluetooth, but still it needed consent to install itself. So, receiving some file out of nowhere shall say a lot to the casual user ;)

Note, that by 'unleashing virus potential' actually means 'installing'.

So, there are no 'viruses' for Symbian so far - these are all 'malicious applications'.

Got it!

I was actually doing things the wrong way round.

So the answer is on record, what I did was change the value of the cell that teh option buttion groups are linked to, which forceds the buttons to "select themselves" as it were:

 

Private Sub OptionButton1_Click()
Application.ScreenUpdating = False
Sheets("Option").Unprotect
Sheets("Option").Range("k4").Value = 1
Sheets("Option").Range("k6").Value = 1
Sheets("Option").Range("k8").Value = 1
Sheets("Option").Range("k10").Value = 1
Sheets("Option").Range("k12").Value = 1
Sheets("Option").Select
Application.ScreenUpdating = True
End Sub
