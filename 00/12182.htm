jay freedman say:

You can't do this with the legacy dropdown control. You can do it with an ActiveX combo box, and if your choices are 256 characters or less you can do it with a dropdown content control.

If you can stay within the 256-character limit, and if users all have Word 2007 or later (currently Windows only, not Word for Mac), then the content control is the preferable solution because it will display the entire choice in place.

If that's not possible, the ActiveX combo box *may* work for you. First, I'm not sure ActiveX is supported in Word for Mac; someone else will have to weigh in on that. Second, the combo box requires macro code to operate, so it will run into security issues on some computers. Third, the combo box itself will display only one line for each choice (it has to scroll sideways to read the whole thing), and the code will have to transfer the complete text of the choice to some location in the document outside the combo box.

The code will look something like this:

Private Sub ComboBox1_Change()
    ActiveDocument.Range.InsertAfter ComboBox1.Value & vbCr
End Sub

Private Sub Document_New()
    Combo_Initialize
End Sub

Private Sub Document_Open()
    Combo_Initialize
End Sub

Private Sub Combo_Initialize()
    With ComboBox1
        .AddItem "Four score and seven years ago our fathers brought forth " & _

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
Dim Content
  Select Case ContentControl.Title
    Case "Test"
      With ContentControl
       .Type = 1
        Select Case .Range.Text
          Case Is = "X": Content = String(500, "X")
          Case Is = "Y": Content = String(500, "Y")
          Case Is = "Z": Content = String(500, "Z")
        End Select
        .Range.Text = Content
        .Type = 4
      End With
  End Select

  Exit Sub
End Sub

aaron zhu msft say:

Hi Jeff,

 

From your description, I understand that you want to move your Office 365 subscription bought from GoDaddy to Microsoft.

 

Please note that it is not feasible to directly moving the data from a subscription from one tenant to another. And also directly moving an Office 365 tenant in GoDaddy to Microsoft is not feasible. It is required that manually remove the custom domain from the existing tenant in GoDaddy and cancel the Office 365 subscription in GoDaddy. Then purchase an Office 365 subscription in Microsoft and add the domain into this Office 365 subscription. Here are the steps:

 

1. Remove your custom domain from the existing tenant bought from GoDaddy.

2. Cancel your Office 365 plan in GoDaddy.

Note: As you purchase the plan from GoDaddy which is a Syndicate Partner of Microsoft, the interface or steps might be different from the ones bought from Microsoft directly, you may need to consult GoDaddy support for the detailed steps to do the above actions.

3. Purchase an Office 365 plan from Microsoft directly.

4. Add and verify your domain in Office 365.

 

Note: To back up data before performing the steps above, you can check out this article.

 

Given the information above, here are the answers to your questions:

 

Q: What is not clear is whether or not I can just move my MS Office / Office 365 subscription away from GoDaddy, yet keep my domain services with GoDaddy?

 

The answer is yes, you can keep your domain service with GoDaddy. However, as what mentioned above, you need to Remove your custom domain from the existing tenant bought from GoDaddy and then add and verify it in the new tenant purchased from Microsoft directly.

 

Q: Are the services GoDaddy provides for Office separable from their domain services?

 

Please contact the GoDaddy support to identify it.

 

Q: If I move my MS Office subscription do I also need to move my domain support?

 

You don’t need to change the domain support, you need to remove and re-add the domain.

 

Best Regards,

Aaron

armchair scouse say:

In trying to resolve your problem, one of things you may need to do is to determine whether you are actually dealing with Excel 2007 files or Excel 2003 files.

I believe that Excel 2007/2010 files of the format XLSX, XLSM or XLAM are, in effect, cabinet or ZIP files.  I have used this property in the past to determine if a file was really an Office 2007/2010 file, or if it was from an earlier version of Office, but with the extension modified to look like an Office 2007/2010 file.

To determine if the file is a valid XLSX, XLSM, XLAM file (Excel 2007/2010):

1.  MOST IMPORTANT!!  Take a copy of the file.  The next steps should be done on your copy, NOT the original.

2. Change the file extension to ZIP, e.g.  copy BOOK1.XLSX to BOOK1_a.ZIP.

3. Double-click on the file in Windows Explorer. 

If the file is a valid XLSX, XLSM or XLAM file, you should see an 'explorer' window open with three folders displayed:

a) _rels

b) docProps

c) xl

There will also be a file called [Content_Types].xml.

If those are not present, then I'd suggest that it is not a valid Excel 2007/2010 file.
