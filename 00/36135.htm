a. user say:

It probably downloaded just fine (if you have a valid executable file on your desktop).  It may just be a matter of permissions.  Right click on the executable icon and clidk run as Administrator (even if you already are an Administrator, this step is sometimes necessary).

If that doesn't work, try to boot into safe mode (repeatedly click the F8 key while booting and go to safe mode with networking).  Then let's check some of your system files:

Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator.

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.  If you have corrupt files, try to find their names from the log (it's a BIG log).

While in Command Prompt, type chkdsk /f /r and enter and let it run.  It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a potential cause.

If that doesn't work, then please post any repeating error messages from the Event Viewer concerning downloads or installs (Start / Control Panel / Administrative Tools / Event Viewer).  Here's how to use Event Viewer: http://www.bleepingcomputer.com/forums/topic40108.html.  Post them here unless there are a lot - then just give us a sampling of those you think are most relevant.

If that doesn't work, we'll need to do a system repair/upgrade using the genuine Windows Vista Installation Disk (one you own or one you can borrow from ANYONE).  Here's the procedure: http://www.vistax64.com/tutorials/88236-repair-install-vista.html along with the upgrade from an earlier version of Windows section of the following: http://support.microsoft.com/kb/918884.  Although this will not affect your data, settings, or programs, you should still backup your data before starting just to be on the safe side. If the other procedures don't work, THIS one is almost certain to work.  You may have a lot of updates to re-install (including any service packs you had to remove).  If the version on the system cane with SP1 or SP2 pre-installed and the disk is an earlier version, then you'll need to make a slipstream disk as follows: http://www.vistax64.com/tutorials/151606-vista-sp1-slipstream-installation-dvd.html.  

Hopefully one of these procedures will resolve your problem and it won't come to this last step.  I think click as Administrator will work (it has for others with similar problems though not with this particular program)..  If not, post back and we'll try something else.  Post back in any case so we'll know how it turned out.

Good luck!

Rather than creating a 2 hour movie in a single project, you might want to consider splitting the work into several smaller projects.

To do this, first create multiple copies of your original project file.  Keep the original as a backup.

In the first copy, go in and delete clips from the end.  I'd recommend making it only about 15-20 maximum.  Save this project file and also make sure you can save it as a movie (File > Save Movie).  That should create a .wmv file.  Call it something like Part1.wmv.

Now in the next copy, delete everything except the next 15-20 minutes.  And again make it into a movie (e.g. Part2.wmv).

Repeat with the third copy, 4th copy, etc. until you have it pared down to separate project and have created movie files from each of the parts.

In the end you can either just use a playlist to play the parts in order.  Or if you want, create one last project where you join together these individual parts and save this as the final movie.

DirectX Diagnostic Tool

  c. If your video card driver seems outdated please go to the video card driver manufacturer and see if a newer version is available.

http://support.amd.com/us/gpudownload/Pages/index.aspx
http://downloadcenter.intel.com/

b. Under Programs, click Uninstall a Program.

marilyn o say:

Hello lwgreen,

The eula.rtf files are End User License Agreement files. They are stored on your drive when you install applications. The reason you cannot remove the files is that you do not have the ownership rights to the files.
To change ownership:
Right click on the parent folder of these files and right click on this folder.
Choose Properties.
Click on Security tab.
Click on Advanced button.
You will need to change the ownership using Change Ownership to your username.
Once you are the Owner of the folder and all subfolders and files, you can delete them.

Hope this helps.
