suzanne s. barnhill say:

If you use Word's automatic feature to create a TOC, the entries will have period leaders by default. But I realize you were just giving this as an example (other applications are entries in a program for a play or concert), so here's how to create such a list manually:

  1. Select all the paragraphs you want to have this format (or just the first one if you haven't created them all yet).
  2. If the horizontal ruler is not displayed, select the View tab and, in the Show group, check the box for Ruler. 
  3. At the left end of the ruler is a box showing what kind of tab stop will be inserted when you click on the ruler. By default, this is a left tab stop; click twice to get a right tab stop.
  4. Click on the ruler where you want the right tab stop (for a TOC, for example, this would be at the right margin).
  5. Carefully double-click on the tab stop you just added, which will open the Tabs dialog. (If you have difficulty doing this, right-click in your text, choose Paragraph, and click the Tabs... button in the Paragraph dialog.)
  6. In the Tabs dialog, select the tab stop you just entered, and select the period leader.


  7. Click OK.

If you will be using this format for a number of paragraphs (in a program, for example), it would be a good idea to save it as a style so you can reapply it as needed.

max stewart say:

"upgrading to Outlook 2016 would disable EAS"...

Would CraigYoungOS care to elaborate please?

Firstly Outlook 2016 <i>does</i> support Exchange Server. What it doesn't like are Exchange Servers which don't publish Auto-discover records. And if an older Exchange Server won't, just use a client-based autodiscover XML file.

(I myself think MSoft is wrong in locking the door to manual configuration, but...)

Secondly, Outlook 2016 <i>does</i> support EAS, but apparently only for hotmail/outlook.com accounts (i.e. those which have not yet been converted to Outlook Mail). It uses EAS because it doesn't support the Deltasync protocol via the Hotmail/Office Connector. 

Thirdly, why would anyone running Outlook 2016 and a suitable Exchange Server ever want to use EAS when they could use the richer Exchange MAPI over HTTPS? EAS does not, for example, support contact lists, and its support of calendar syncing has some interesting 'side-effects' . Outlook.com accounts migrated to Outlook Mail do by default use EAS in Outlook 2016, but if the user removes and re-adds the account, Outlook 2016 autodiscovers Outlook Mail and uses MAPI over HTTPS. And MAPI over HTTPS replaces the older RPC over HTTP ('Outlook anywhere') which is, however, still supported.

EAS was intended for lightweight clients running over low bandwidths - mobiles for example, and Exchange Server does of course support them via EAS. (In passing, EAS is not a free open standard; the protocol - but not any code-base - is open, but its EAS is now licensed by Microsoft as a set of patents).

kergga say:

I had one other thought- you could do a stacked chart, and have the cells automatically vary the width of the two bars (to the total bar length is always the same). In cases where I've had to have bars "Start" at different places (not unlike a Gantt chart that will have different start dates) I've used a similar technique with the lowest data series formatted as no fill/ no line.

 

The only thing you give up is a little bit of pretty graphics, because instead of a fade from 'complete' to 'incomplete' you would just have a straight color change, the severity of which would depend on your choice of colors. You could even color the line (surround on the both series) the same color as the 'complete' color, so that it is visually clear where the incomplete color ends.

 

So for example (and this data doesn't match the sample below, this is just an explanation),

A1 = 10 = no fill/no line series, this 'deliverable' starts 10 spaces in (10 days)

B1 = 3 = completed 3 days

D1 = 7 = total task time allotted

C1 = D1-B1

Then just graph A:C, D is just a helper column.

 

Hope that helps.

 

Of course, if you went the VBA route you might also have a column for % complete, and vary the line color (Green/Yellow/Red) based on whether the item is on track, or at significant risk...
