treatyoak say:

This solved it for me, though I figured it out before I'd seen this. (Came back to share answer with others.)

So, to be more clear for those who need more help:

  • when you open a file, Excel creates a temporary autorecovery file IN THE SAME DIRECTORY. It's hidden by design, so you won't see it... but you CAN see this if you make an adjustment in Terminal.
  • To see hidden files on your Mac,
  1. Launch Terminal
  2. Type defaults write com.apple.Finder AppleShowAllFiles TRUE
  3. Relaunch Finder (Option + click on Finder icon, choose "relaunch")
  4. When you're done with whatever you're doing, if you don't want to see hidden files, repeat the process with FALSE instead of TRUE
  • If you're working on a file called "MyProject.xlsx" in a folder called "Red," when you open the file, Excel will create a temp file called "~$MyProject.xlsx" in "Red." This is a good thing, because some day when your laptop crashes, this will let you get your data back.
  • Unfortunately, sometimes files get corrupted. That's what's happening here. So, I was working on a file called "2014-02 JSA Review.xlsx" in a folder called "2014-02 Reports". Maybe it was corrupted when I shut the file down the last time I touched it two months ago? Who knows. Anyway, today, when I went to open it, I kept getting the "locked for editing" message.
  • I deleted the Excel preferences .plist, I rebooted, I tried opening the file "edit only" and saved it with a new name, I did some other things, but no luck.
  • THEN I looked at the "2014-02 Reports" folder and noticed that, even though Excel was closed, there was a hidden file there called "~$2014-02 JSA Review.xlsx" -- weird. Shouldn't be there, because the temp file should close when Excel is closed.
  • Fine. I erased that file, and that nailed it. Now I open the new file: all works as it should.

Good luck!

a. user say:

A couple of things to consider/look at:
 a. if your caption text comes before the label + number, ir will be treated as part of the label as far as the reference is concerned. (That's my best bet as to what is going wrong here).
 b. When you make a reference to a caption, what Word does is to "cover" the text it is going to display in the reference with an invisible bookmark, then creates a { REF } field to display the contents of that bookmark. Although it doesn't obviously get you very far, you can at least verify what the boomark is "covering" as follows...
  - use option-F9 or fn-option-F9 (depending on your keyboard) to reveal the { REF { field code, which will look something like
{ REF _Ref123456789 } 
(perhaps with an \h inside)

  - Select the _Ref123456789 part and copy it to the clipboard (e.g. cmd-C)
  - Click Edit->Find->Go To...
  - Under "Go to what", select "Bookmark"
  - Under "Enter bookmark name", paste (e.g. cmd-V) the bookmark name.
  - Click Go to

Word should then select the text that the bookmark "covers" Perhaps that will give you a clue as to what is going wrong.

If absolutely necessary, you could consider manually bookmarking each caption label + number and inserting your own REF fields to your own bookmarks. This shouldn't be necessary but it may at least be /a/ way to do what you need.
