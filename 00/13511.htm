Sheet Music Database in Access - Microsoft Community

docjrv say:

Sheet Music Database in Access

My wife is a music teacher and has over a hundred books of sheet music, some loose-leaf stuff, and some PDFs. I want to build a database to help her find songs. Here's the basic user experience I'm going for?

USER: What book is [song] in?

RESULT: [Song] is in [Book34], [Book 98], [PDF Hard Drive]

Or, when she's trying to a song to give to a student, she could search by voice-type (for example), and could filter results based on (for example) genre, character age, composer, or highest note.

Meanwhile, I'm JUST STARTING OUT with Access. I know this is a tall order, and a pretty complex database. I know it's going to be something I'm working on for a while. And I'm okay with that. I just have no idea what I'm doing. If anybody has done anything like this (not necessarily music), I'd really appreciate some guidance.

I thank you for your time and attention.

scottgem say:
Hi Doc,

I've actually done several apps that are similar. The key to a well working database is a properly normalized table design. So the first thing you should do is read up on normalization.

When I start a new database the first thing I do is make a list of all the data points I want to capture. I then organize those data points into entities that become my tables. Only after I have a good table design foundation do I start on building the rest of the application.

For your purpose you need to start off with a table for your songs. Attributes of a song are things like genre, composer, lyricist, etc. Most of those attributes will require lookup tables to make sure that answers are standardized. Another thing you have to deal with is that you appear to have many to many relationships. For example a song can be in multiple books and a book can have multiple songs. So you need a Books table and then you need what's called a Junction table to manage the relationships. such a table might look like this:

tblSongBook
SongBookID (Primary Key Autonumber)
SongID (Foreign Key)
BookID (FK)
PageNo

That should give you some background to get you started. If you have specific questions along the way, feel free to ask.
