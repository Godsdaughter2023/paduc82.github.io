a. user say:

Hi Shirley

Sorry for the late reply. I ended up fixing the issue myself. Basically the configuration file for RDS install should read like below. Perhaps you can share it with other users who face similar issue:

Configuration file should be :

<Configuration>

   <Add OfficeClientEdition="64" Branch="Current">

       <Product ID="O365ProPlusRetail">

           <Language ID="en-us"/>

       </Product>

   </Add>

   <Logging Level="Off"/>

   <Display Level="Full" AcceptEULA="TRUE"/>

   <Updates Enabled="TRUE" Branch="Current"/>

   <Property Name="SharedComputerLicensing" Value="1"/>

</Configuration>

Note - Property Name= "ShareComputerLicensing" Value="1" should be in the config file for a proper RDS install.

karengartner say:

Black Stationery with White Text becomes invisible in replies and forwards - Outlook 2013

My company stationery is black with white text and a signature block. Surface Pro 3 with Windows 8.1 running Exchange - IMAP email.

When a new email is created, the first line is black text (invisible on the black background) and not the font I saved the theme with. The proper font and color appear on the next line down.

For all recipients, forwarding and replying to the email turns the background white, but the original text of the message stays white and is therefore invisible - even highlighting the text does not show it in an inverted color value. I have set the reply and forward text value to be black text (but I guess that only applies to the replies and forwards I do to others' mail I receive).

When I create the email, the size of text changes from 12 (which is what I have saved with the theme) to 10 points at random intervals.

Am at a loss to find a resolution.

diane poremsky mvp (slipstick.com) say:

Outlook uses the original format on replies and forwards - it never uses your stationery. When other people reply, the reply will use the font setting in the original message. 

Is the font changing as you type? If you changed the default +Body format, it should apply throughout the message. 

Open Windows Explorer and type %appdata%\microsoft\stationery in the address bar (or browse to where your stationery is) and open the HTML file in notepad. After all the css is the message body and the first line will include the words "Color:black". If there is not a lot of customization in the file, it will be a few lines up from the bottom. Or search for color:black.  Either change it to white or delete that line.

Because white text on black background makes replies difficult and is hard to read, and print, you really should rethink using it as a company stationery. It doesn't make a good impression when the recipients are annoyed by it.
