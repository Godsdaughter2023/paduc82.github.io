Hi

If I understand you correctley you have made a tablestyle And in that table style you have formatted the header row and under table properties marked it to repeat as header row? If you have done that you can only have one row as repeating on top of new page in tables with that style applyed. To solve this: 

  1. Modify your tabel style and unclick "repeat as header row" under table properties in the table style dialog.
  2. Then go to the table in your document.
  3. Mark the two rows you want to repeat as headers.
  4. right click on them and in table properties mark them as "repeat as header row"
  5. The second row will automatically pick up the format you have set on header rows in your table style.

this way both rows stays as headers even when you Close the document and reopen it.

 

The drawback is that new tables will not automatically have the header row repeated. To make it easier for your users you can create a table with the settings above and save it as a buildingblock showed under the table gallery

robert aldwinckle on forums say:
the Google search page won't display the Google logo. I right click on the red X and clicked show picture/image and nothing happened.


That would be normal.   The purpose of that command is to use it on a image placeholder, different from a Red-X, to show a picture when it has been suppressed by having Show Pictures unchecked in Options, Advanced tab, Multi-media section.  

Instead what you could do with right-click is  E-mail picture...   You may also be able to use right-click, Properties to find the full URL for the image, copy that and try opening it by itself in its own tab.

However, a better diagnostic would be to use the Developer Tools, Image Report (F12, Alt-I,R) and see what it thinks about the missing images.   E.g. often it will call it a "broken image" which then explains why you are seeing it represented by a Red-X when the page is rendered.

herb tyson say:

For scenario 1, I suggest when pasting that you NOT include any section breaks in the material you're copying. This presumes that you do not want the footer from the source document.

For scenario 1, when you create the quick part, set Options: to Insert contents in its own paragraph. That way, it should bring its own paragraph formatting with it--at least that's the way it works here.

Note that I don't know which kinds of protection you're using, so I haven't tried any. I've experimented only in unprotected documents.
