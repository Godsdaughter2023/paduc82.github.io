e.j. gun say:

We should focus on why you can't "link and sync" between Access and Excel, because in general there should be no reason you should need VBA to do this.

 

How are you exporting the data from Access to Excel?  This is the first question.  The way you should be doing it is to:

1) Set up a query in Access that pulls all the data you want to export into a table.

2) In Excel, go to the Data tab, select "From Access", locate your Access database, select the query that pulls the data within Access, and then select a location in your Excel workbook to store the data.

3) Set the query to automatically refresh whenever you open the workbook to ensure the latest data, or you can click on a cell in the table, go to the Data tab, and select "Refresh All".

4) Go to one of your pivot tables.  Click on the "Options" tab, select "Refresh All" under "Refresh.

 

In this way the data table in Excel will mirror exactly what you have in access.

benjaminramsay say:

Actually we found a solution in our case. I don't know if it will help the OP, but for us the problem was that the "sender" address being used by our phone system differs from both the From address and the address/account being used to authenticate with 365 in the IIS relay. So while we had given the authenticating account proper Send As / On Behalf Of permissions to the From address account, we had not done so for the "sender" address account. We had to look at the actual email header to notice that From and Sender were not using the same address. From there just giving the "sender" account permissions on each From user did the trick.
