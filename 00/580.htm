hansv mvp say:

I think this is by design.

If you copy a range to the clipboard, then select a range that is an exact multiple of the copied range in size, pasting will fill the entire selection. If you select a range that is not an exact multiple of the copied range, pasting will affect a range of the same size as the copied range only.

For example, select a range of 2 rows by 3 columns and copy it.

Now select a range of 10 rows and 6 columns and paste - the entire range will be filled.

But if you select a range of 10 rows and 5 columns and paste - only the top 2 x 3 cells will be filled.

 

An entire column in an Excel 2007/2010 worksheet contains 1,048,576 = 2^20 rows. All powers of 2, i.e. 1, 2, 4, 8, 16, ..., up to 524,288 are evenly divisible into 1,048,576, so they will result in the behavior that you describe. Any other number is NOT evenly divisible into 1,048,576 so will not exhibit this behavior.
