reneyu_msft say:

Hi,

For your questions.

#1

No, we cannot use GPO to manage the settings of the OneDrive for Business, and, we can only change the settings for the whole My Site host, which would be all users' OneDrive for Business.

#2

The sharing approval request is not available for the external sharing. We can only either allow or disable it.

#3

From the angle of OneDrive for Business, the command of

Set-SpoSite -Identity https://organization-my.sharepoint.com/personal/user_domain_com -DisableSharingForNonOwners

would disable the ability of the members in the defined My Site inviting others into it.

If it successfully ran, then only the user who is the owner of his ODB library can share the contents with others.

Please confirm the command you run and further check the results.

#4

Basically, the permissions of Manage Permissions and Manage Personal Views control whether the granted user is able to share the contents with other users directly.

After revoking these two permissions, when the users,

a. try to internally share a document, an approval request will be generated for the owner at Site settings > Access requests and invitations,

b. want to externally share a file, he will be prompted "Your organization has disabled sharing with external users".

We can just use the Full Control permission level as the example to try to explain this simply.

To edit a permission level, please follow the below steps.

1. As the owner of the ODB library, go to the library > click the gear at the right top, Site settings.

2. Under Users and Permissions, Site permissions.

3. On the PERMISSIONS ribbon, Permission Levels. We can then either Add a Permission Level, or create a new level based on a existing one.

4. Click Full Control > scroll down the page, Copy Permission Level.

5. At Site Permissions section, untick Manage Permissions. At Personal Permissions, untick Manage Personal Views.

6. OK.

Then we can grant the new permission level to the user.

Rene

No, there is no way you can force Word to use a custom template for all new documents. You can add a switch to the shortcut used to start Word so that it will open with a document based on your custom template (see “How to use startup command line switches to start Word 2010, Word 2007, Word 2003, Word 2002, and Word 2000”), but once Word is open, Ctrl+N or the New button will still open a document based on Normal.dot.
