With Access 2010, you could to this with a Data macro that runs on the table level.  But I'm not as familiar with 2010 macros yet to be specific on how to do it. 

Since Daniel is 100% correct that all interaction should be done on the form level, you could do this int he After Insert event of your form. The code you want to run is fairly simple:

Dim strSQL As String
strSQL = "INSERT INTO tbl2 (ForeignKey)  VALUES(" & Me.keyfield & ");"
CurrentDB.Execute strSQL

This will insert a record into Tbl2 with the foreign key from the current record's primary key. Since that's all you need from the Tbl1 record. That will do what you want.
