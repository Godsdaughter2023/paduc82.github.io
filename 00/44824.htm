Uninstall the Device Drivers from Program List in Control Panel

  • Click on start, select Control Panel
  • Now click on Programs and select Programs and Features.
  • Find the Display card Driver.
  • Right click Display card Driver and select Uninstall.
  • Reboot the computer after the uninstall process has finished.

Download and install the latest drivers for the Display card from the manufacturer website

tryx3 say:

Trish,

 

Just to help bowl out some possible red herrings as I was not sure how you set up the new Admin account.

 

Log in with the new Admin account then open Windows explorer not Office.  Go to the normal My documents folder for your normal user account, i.e. the problem folder, and try to open that.  You should get challenged to confirm access ["Do you want to allow changes..." dialog demanding the Admin password].  Once this access is authorised once then it remains authorised in the future.  Now try to use the new account in Office to save to that My documents folder [being careful, of course, that you do not get mixed up with the new Admin account's own My Documents folder].

 

If that new Admin account can now save in My documents then it points to the original problem being corruption in the original [normal] account's user profile.  All the other symptoms you have mentioned are consistent with such user profile corruption.

 

Additionally, the Guest account should not be able to access your My Documents folder in any case.  A Guest has very limited permissions on the system and can normally only save in the Public folders as you discovered.

If the new Admin account seems to work correctly then even before posting back here to say so, create another new password-protected Admin account so that you have the confidence of knowing that you have two fully-functioning accounts that could be used to rescue the system.  I can explain all this but don't want to bore you with the significance of a test result that hasn't happened yet.
