As I started to say, Refresh is a good feature if you take full advantage of it.  The trick is that after you install "desktop" type applications (like Office) you have to create a new "image" to be used by Refresh the next by running the RECIMG.EXE program.  Here is a bunch of info about Win8 backup options, including Refresh and RecImg.EXE

Win 8 Total Backup System: File History + RecImg + Refresh + SlimImage

Win 8 has refined your backup and restore options. Before Win 8 you could create an Image Copy backup of the full drive (or selected directories), but it was hard to separate backups of the OS, Applications and personal Data unless you created a separate data partition.  I still think it is a good option.

 

Out of the box, MS has been advertising the “Refresh” and “Reset” options. Personally, I think they missed a valuable selling point by not mentioning RecImg.EXE in conjunction with Refresh. It removes my main concern/objection to Refresh.

 

Reset is equivalent to using the “Recovery Partition” on previous versions of Windows. The factory would create a “Recovery Partition” that you could use to “reset the computer to factory fresh”. It would typically include all of the factory “crapware” and it would wipe out all of your data files and installed applications.  A less than desirable outcome but an appropriate reaction to a bad malware infection.  The “recovery partition” type of restore typically had to be accessed at boot time while Reset can be accessed from within Windows 8.

 

Refresh is roughly equivalent to an image copy backup.  A Reset will restore a backup of the OS and Metro applications. It will wipe out “legacy” / x86 desktop type applications BUT it will not affect your personal data files, which is much better than a Reset.  But there is still that issue of wiping out your installed desktop applications. I do not have documented confirmation, but I’m guessing that Metro apps are treated like personal data and those directories/folders are simply not reset back to the old condition.

 

RecImg.EXE is a backup tool that ships with Windows 8. It creates the backup “image” that is used by the Refresh tool. This image that was created at the factory is why using Refresh will wipe out installed desktop programs in the “C:\Program files” and “C:\Program Files (x86)” folders because they haven’t been backed up. However, using RecImg.EXE you can create new images that will include recently installed desktop applications.

 

RecImg Manager is a third party tool the provides a GUI to access the RecImg.EXE command line tool and backup files.

 

File History is a new Win 8 feature. It is an extension to the “Previous Versions” and “Volume Shadow Copy” features. File History continuously monitors files stored in Libraries, Desktop, Contacts, and Favorites folders. It runs on a predetermined schedule, and when it detects changes in any file, it makes a backup of that file to an External drive. It isn’t enabled by default.

Windows 10.10162: What issues remain? - Microsoft Community

a. user say:

Windows 10.10162: What issues remain?

Greetings and salutations on this fine Independence Day.

Like (I assume) the rest of you, I've been playing with 10162 since it came out.  48 hours of use is enough to convince me that Windows 10 is just about there; every build that comes out feels less like a technical preview and more like a fully fleshed-out OS, so much so that I've actually made 10162 the only OS on one of my machines.  I'm particularly pleased to note that Windows at last has the ability to remove orphaned apps from the Start menu (those things that get uninstalled between builds, but still show up as list entries with their full app names, which until now were impossible to eliminate).

For me, that's a huge step, since up until now everything I use has dual-booted with something else...just in case.

That said, there are still some issues that I see, and I wanted to share them and get a feel for what everyone else was finding.  That said, here we go:

1)  Hardware issues.  That pesky problem with Realtek HD Audio drivers seems to have returned, and while it looks like it might've been resolved in 10162, thus far I've only used it in Stereo mode since I've got the computer in question (an Acer 8951G) connected to an external display--and its speakers only support stereo sound.  It would've been nice, however, to see a resolution by the time 10159 came out.  Also, on another machine--an MSI Titan--the integrated webcam not only doesn't work, it doesn't seem to be recognized by Windows 10 at all.

2)  Microsoft Edge.  Most of the updates for this build seem to be improvements, but curiously, starting in 10158, I can't do Web Notes onto PDFs.  I call that curious, because as of 10147, this function still worked perfectly.  I'd really like this function back!

3)  Start menu customization.  I've been saying it for months, but still, we have no fix for the width of the tile groups.  Up until 10041, the XAML Start menu used tile groups that were four columns wide (as wide as two Large/Wide tiles), but starting in 10049, that width was reduced to three columns (one Large/Wide and one Medium tile).  While I'm sure there was a reason for this change, I'm equally sure there are lots of people like me who created four-column-wide Start menu layouts and would really like to keep them.  The fix for this is obvious:  give users a way to customize the width of tile groups.  Some will prefer three columns, some will prefer four, and some will prefer one giant group like we used to have with Windows 8.

4)  Color selection.  Some things about 10162's color selections bother me, since my selections didn't seem to carry over from any of the builds I upgraded to it from (10041, 10130, 10134, 10147, 10158, each on a different machine).  I do love the Hero wallpaper being the background on the login screen, so let's keep that--but some of the other color touches now show an unappealing blue background (the logout/restart screen), an unappealing green color (the touch keyboard), or a plain gray background (on one machine, the logout/restart screen).  I actually preferred the dark purple from Windows 8, and with that in mind I've actually kept Windows 8 on one machine to ensure that I'd always have a way to keep that color synched to my Microsoft account, and thus synched to all my Windows devices.

This reveals two problems:  one, Windows 10's color selection palette is too limited (I actually changed the accent colors on one installation by accident, and I got stuck with a shade of blue I hated, and had to wipe it and reload it all the way from 10041 in order to get my colors back and keep my Start menu layout); and two, that control over UX colors is far too limited.  The fixes, fortunately, should be very simple:  one, give us back the millions-of-colors selection palette and related options from Windows 8; and two, give us that fine degree of control.

5)  Software glitches.  One unexpected issue came from Office 2016 Preview, and even then only from one installation of four, but it's there.  In Word 2016, when I try to open a document from the list (something from OneDrive that I started on another machine), Word immediately crashes...yet, when I go to my OneDrive folder and open it directly from there using File Explorer, it works just fine.  On every other installation, though, Word behaves exactly as it's supposed to.  Very strange...the only difference I can think of here is that on the machine that's misbehaving, I did an upgrade to 10162 directly from 10041, while everything else had at least one other build (10130, 10134, or 10147) in between.

What is everyone else seeing?
