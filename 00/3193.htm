jay freedman say:
Whenever you think "replace anything that matches this rule", you should immediately think "wildcard search". This topic is covered extensively at http://www.gmayor.com/replace_using_wildcards.htm.

You should need only one part of the macro out of the two nearly identical parts you posted, because you can do all the formatting in one step.

Sub MusicFormatting()
    With ActiveDocument.Range.Find
        .MatchWildcards = True
        .Format = True
        .Text = "\[[!\]]@\]"
        With .Replacement
            .Text = "^&"
            .Highlight = True
            .Font.Name = "Courier New"
            .Font.Bold = True
            .Font.ColorIndex = wdRed
        End With
        .Execute Replace:=wdReplaceAll
    End With
End Sub

The expression for .Text can be confusing, so let's take it apart:

   \[  [!  \]  ] @  \]

The innermost piece is the first \]  which says "match a right square bracket" -- the backslash says this is literally a square bracket and not part of a pair of [ ] that specify a range of characters.

To the left of that is an exclamation mark, the "not" operator, so now we're saying "match any character that is not a right square bracket". This inner expression  ! \]  is enclosed in a pair of square brackets (without backslashes) to say this is a range of characters, [!  \]  ].

To the right of that, we find an @ symbol, which says "match as many consecutive characters as possible that match the specified range". For example, each of the strings  C, C7, C7/G, and Dmaj7b5 will be matched, because none of them contain a right square bracket.

Finally, there are  \[  and \]  at the beginning and end of the expression to match the actual brackets in the text.

By using ActiveDocument.Range instead of Selection, you can omit setting any of the properties of .Find that should keep their default values. In contrast, the properties of Selection.Find are "sticky" -- they keep values that were assigned by any previous macro or manual (dialog-based) find/replace -- so your macro would need to set everything back to defaults before you can be sure it will operate as intended.

To set users’ passwords never expire, you can refer to the following steps.

Using CTRL+F9 to insert each pair of field delimiters { }, use the following field construction:
