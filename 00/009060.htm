amit tandon say:

Two options as below. Refer below image.

Option 1: Enter below formula in cell G1 for looking up value in F1:

=INDEX($A$1:$A$4,SUMPRODUCT((($B$2:$D$4)=$F$1)*ROW($B$2:$D$4))-ROW($A$1)+1)&INDEX($A$1:$D$1,SUMPRODUCT((($B$2:$D$4)=$F$1)*COLUMN($B$2:$D$4))-COLUMN($A$1)+1)

Option 2: Enter below formula in cell G1 as an array formula (CTRL+SHIFT+ENTER) for looking up value in G1:

=INDEX($A$1:$A$4,MAX(IF($B$2:$D$4=$F$1,ROW($B$2:$D$4)))-ROW($A$1)+1)&INDEX($A$1:$D$1,MAX(IF($B$2:$D$4=$F$1,COLUMN($B$2:$D$4)))-COLUMN($A$1)+1)

The above formulas will give a correct result even if you move your table up/down or to the right.

mat lcq say:

I found out what was causing the issue on my configuration (Windows 7 Pro with Outlook 2013 Pro and Outlook.com accounts). Posting answer here in case someone else ever encounter the same problem.

Since last upgrade of outlook.com, Outlook 2013 client stores encrypted Exchange account credentials under folder <user>\AppData\Roaming\Microsoft\Credential (system files).

CCleaner has an option to clean network passwords from windows explorer, which are located under the same folder as above. So simply uncheck that option and it should be fine.

I didn't have that problem before the upgrade and CCleaner had that option checked for a long time. So maybe it has changed since then... Dunno!
