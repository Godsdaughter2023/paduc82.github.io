john w. vinson say:

You certainly can do this with VBA.

However, you almost certainly SHOULD NOT do this, with VBA, with a data macro, or any other way!

Storing derived data like this in a table is incorrect design. If you store both the raw data and the derived (subtotaled) data in a Table, then either one could be edited at any time; you would then have a stored subtotal which is wrong, with no easy way to tell that it is wrong or why!

Just store the individual items and use a dynamic totals Query to calculate the subtotal whenever it's needed. That's what databases do - store data, and use queries to select, sort, and calculate it.

If you have a proven, demonstrated, and critical problem with performance doing it correctly, then you can use VBA (with update or append queries) to store this redundant data... but if you do, do it in private and wash your hands afterward <g>.

You are opening the file with this line of code

f actually holds the file name as follows.

Sub test()
   
    Dim wb As Workbook
    Dim f
   
    f = Application.GetOpenFilename()
    If f <> False Then
      MsgBox "File to open = " & f
    Else
      MsgBox "User did not select a file"
    End If
   
    'Following line opens the selected file
    'Set wb = Workbooks.Open(f)

End Sub

chip pearson say:

Code like the following should do what you want:

    Dim FName As Variant
    Dim WB As Workbook
    FName = Application.GetOpenFilename()
    If FName = False Then
        Debug.Print "user cancelled"
    Else
        Debug.Print "file selected: " & FName
        Set WB = Workbooks.Open(FName)
    End If

GetOpenFileName returns a Boolean False if the user cancels the open dialog or a String file name of the complete path to the selected file.

By any chance did you connect the phone to the computer before installing Zune? It seems that it is not uncommon for a device to become registered as an unrecognized device if it gets connected to the computer before the driver is installed. When this happens you usually need to removed the unrecognized registration and let the computer discover the device again. 

As far as I know Kin falls into the same category as Windows Phone and the Zune portable media players; you have to use the Zune desktop software to get media onto and off of the device from your computer. While people have found some hacks and workarounds for the Zune portable media player and Windows Phone these hacks still require the Zune software to be installed. 

Try the following. If this doesn't work then let us know what the error message is that the Zune desktop client gives you. 

 

  1. Connect the phone to the computer
  2. Click on start, type "Device Manager" and press [Enter]. 
  3. In the list of the devices look for your phone. If you see an icon for the phone with a yellow exclamation mark click on it and then press [Delete] to erase the entry. 
  4. disconnect the phone and then reconnect it
  5. Wait for the device manager to recognize the Kin
If all goes well after step 5 the Zune desktop client should be able to interact with the phone.
