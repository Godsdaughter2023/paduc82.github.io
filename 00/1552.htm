Change the margins (actual print area) on envelopes when - Microsoft Community

johnrainey say:

Change the margins (actual print area) on envelopes when printing addresses from a word document?

Some envelopes will have four lines (Company, Name, Street, City St Zip),  some will have only three lines (no Company name). When I make the font size large enough to be appropriate for the envelope size, the City St Zip line is cut off. Prints fine at 14pt, cuts off bottom line at 24pt. I cannot see a way to change the margins (increase the print area) on the envelope.

John Rainey

suzanne s. barnhill say:

The delivery address on an envelope uses a style called Envelope Address. This style includes a frame that is defined as Exactly 1.38" high. You can increase the size of the frame. The easiest way to do this is to deal with the envelope directly (as opposed to trying to work from within the Envelopes dialog).

  1. In the Envelopes dialog, click the button for Add to Document. This will give you a blank document (assuming that's what you started with) with an envelope attached as page 0, section 1.
  2. If you click in the address area of the envelope, you should see a dotted line indicating the frame.
  3. You can drag the top or bottom of the frame to resize it, or you can right-click on the frame border, choose Format Frame, and set different dimensions, but this will affect only the current document.
  4. Instead, right-click in the text area (if there is no text but you have nonprinting characters displayed, click by the paragraph mark), and choose Style... from the shortcut menu.
  5. This will open the Style dialog with Envelope Address selected. Click Modify...
  6. In the Modify Style dialog, click Format, then Frame.
  7. You can either change the height to a different exact amount or set it to Auto or an At Least amount. (You may need to experiment to see which works best.) Since the position is set as Bottom relative to Margin, the frame will expand upward.

A better approach for envelopes of less-common sizes is to create an envelope template. You can get a head start on this by attaching an envelope to a document as above, detaching the document (see http://sbarnhill.mvps.org/WordFAQs/DetachEnvelope.htm), and then saving the resulting envelope (with your desired modifications) as a template.

VBA to manipulate shapes embedded within charts in PowerPoint - Microsoft Community

rich007a say:

VBA to manipulate shapes embedded within charts in PowerPoint

Hi,
If I add a chart to my slide, then with the chart selected insert a shape, that shape is now "part of" the chart (it can not be dragged out of the chart area).

If I have that shape selected, is there anyway I can use VBA to change its colour for example?

I suspect this part of the OM is broken, but would like confirmation of this if I'm right.

Cheers
Rich

rich007a say:

OK, final (famous last words) update from me on this.  If you're interested, here is a method to return all selected shape objects in a chart on a slide, even if they are part of a group of shapes within the chart.

I got the grouped shapes thing to work first time (first time that has happened for a long time), and I thought I had it cracked until I found a nasty PowerPoint bug.  If you have a group of shapes, now (from 2007) you can select the shapes within the group individually.  If you select them all by clicking while holding down Ctrl and then delete (or Cut) them, in normal PowerPoint the grouped shape is automatically deleted... but if the group is in a chart, the now empty group-shape is NOT automatically deleted.  It's type remains as msoGroup, its groupitems.count returns zero and it loses its position and dimensions (since these are normally determined by the position and sizes of the grouped shapes within the group shape).  That loss of size and position causes problems when we cut and undo that cut. 

I got round this bug by searching for any empty group shapes and deleting them.  This fix allows us to cut the selected shapes so we can determine which shape on the slide contains the chart in which shapes are selected.

To determine which grouped shapes are selected, after cutting the selection (and identifying the active shape.chart) we build up a list of all the shape IDs of all the grouped shapes in all the group shapes in the chart.  We also record how many shapes are in each grouped shape in an array of variant arrays.

As I said in previous posts, the shaperange object is completely broken, so I could not get the function to return a shaperange object.  Instead, it now returns a variant array of shape objects.  Note even if only one shape is selected, the function will return that shape object in a variant array, so you would need to access it like   Set shp = vSelectedShapes(1)     if you had used    vSelectedShapes = GetSelectedShapesInChart().

I would love to hear if anyone uses this code in the future, so if you copy it from here please post a note to say if it was successful (or not).

Cheers
Rich

Here's the code:

_____________________________________________________________________

Sub TestGetSelectedShapesInChart()
' To test this, insert a chart on a slide.  Then with the chart selected, insert some shapes in
' the chart.  If the shapes are in the chart then it will not be possible to drag them outside
' of the chart area.  Select any number of those shapes and run Test().  This will work even if
' you select shapes that are part of a group.
'
    Dim shp As Shape
    Dim shprng As ShapeRange
    Dim vSelectedShapes As Variant, vShp As Variant
   
    With ActiveWindow.Selection
        If .Type = ppSelectionShapes Then
            If ActiveWindow.Selection.ShapeRange.Count = 0 Then
                 'Selected shapes are in a chart:
                vSelectedShapes = GetSelectedShapesInChart()
                For Each vShp In vSelectedShapes
                    Set shp = vShp
                    Call FormatShape(shp)
                Next vShp
               
            Else 'Selected shape(s) are on the slide:
                If .HasChildShapeRange Then 'Selected shapes are grouped:
                    Set shprng = .ChildShapeRange
                Else
                    Set shprng = .ShapeRange
                End If
               
                For Each shp In shprng
                    Call FormatShape(shp)
                Next shp
            End If
        End If
    End With
End Sub

Sub FormatShape(shp As Shape)
'Apply required formatting here:

     'THIS IS JUST AN EXAMPLE:
    shp.Line.Visible = msoTrue
    If shp.Line.Weight = 1 Then
        shp.Line.Weight = 10
    ElseIf shp.Line.Weight = 10 Then
        shp.Line.Weight = 5
    Else
        shp.Line.Weight = 1
    End If
End Sub

Function GetSelectedShapesInChart() As Variant
' Call this function if the following conditions are both met:
' 1) If ActiveWindow.Selection.Type = ppSelectionShapes
' 2) If ActiveWindow.Selection.ShapeRange.Count = 0
' This is indication that a shape within a chart is selected.
' The function will return a variant array containing shape
' objects, one for each shape that is selected.
'
    Dim sld As Slide
    Dim shp As Shape
    Dim cht As Chart
    Dim bSlideHasCharts As Boolean, bMatchFound As Boolean, bNeedToUndo As Boolean
    Dim i As Long, j As Long, k As Long
    Dim lShpId As Long, lChtShpsCount As Long, lChtShpsCountUncut As Long
    Dim arrChtShpsIds() As Long, arrGrpCount() As Long, lGrpItemsCount As Long
    Dim vTemp As Variant, vGrpIDs As Variant
    Dim arrGrpHolder() As Variant 'an array of variants
   
    On Error GoTo ExitPoint
   
    If ActiveWindow.Selection.Type = ppSelectionShapes And _
        ActiveWindow.Selection.ShapeRange.Count = 0 Then
        
         'A SHAPE IN A CHART MAY BE SELECTED
       
         'Check slide does have at least one chart:
        Set sld = ActiveWindow.View.Slide
        For Each shp In sld.Shapes
            If shp.HasChart Then
                bSlideHasCharts = True
                Exit For
            End If
        Next shp
        If Not bSlideHasCharts Then Exit Function
       
         'CAN ASSUME SHAPE IN A CHART IS SELECTED
       
         'Create a new undo stack entry:
        #If VBA7 Then
            Application.StartNewUndoEntry
        #Else
            Application.CommandBars.ExecuteMso ("ObjectFlipHorizontal")
            Application.CommandBars.ExecuteMso ("Undo")
        #End If
       
         'Cut the selected shape(s), the chart will become selected:
        ActiveWindow.Selection.Cut
        bNeedToUndo = True
       
         'Get shape containing the chart:
        Set shp = ActiveWindow.Selection.ShapeRange(1)
        If shp Is Nothing Then GoTo ExitPoint
       
        If shp.HasChart Then Set cht = shp.Chart
        If cht Is Nothing Then GoTo ExitPoint
       
        Set shp = Nothing
       
         'Count remaining shapes in the chart:
        lChtShpsCount = cht.Shapes.Count
       
         'Fix for when all shapes in a group are selected:
        On Error Resume Next
        For Each shp In cht.Shapes
            If shp.Type = msoGroup Then
                If shp.GroupItems.Count = 0 Then
                    shp.Delete
                    lChtShpsCount = cht.Shapes.Count
                End If
            End If
        Next shp
        On Error GoTo ExitPoint
       
        ReDim vTemp(0 To 0)
       
        If lChtShpsCount = 0 Then 'All chart shapes were selected:
           
             'Undo the Cut:
            Application.CommandBars.ExecuteMso ("Undo")
            DoEvents
            bNeedToUndo = False
            For Each shp In cht.Shapes
                ReDim Preserve vTemp(1 To UBound(vTemp) + 1)
                Set vTemp(UBound(vTemp)) = shp
            Next shp
           
        Else 'Not all shapes in the chart were selected:
           
             'Get array of remaining shapes' IDs in the chart:
            ReDim arrChtShpsIds(1 To lChtShpsCount)
           
             'Get array of variant arrays containing IDs of each
             'group item ID in a grouped shape (empty if not grouped):
            ReDim arrGrpHolder(1 To lChtShpsCount)
           
             'Get array of group item counts for each shape (-1 if not grouped):
            ReDim arrGrpCount(1 To lChtShpsCount)
           
            For i = 1 To cht.Shapes.Count
                arrChtShpsIds(i) = cht.Shapes(i).id
                If cht.Shapes(i).Type = msoGroup Then
                    lGrpItemsCount = cht.Shapes(i).GroupItems.Count
                    arrGrpCount(i) = lGrpItemsCount
                   
                    ReDim vGrpIDs(1 To lGrpItemsCount)
                    For j = 1 To lGrpItemsCount
                        vGrpIDs(j) = cht.Shapes(i).GroupItems(j).id
                    Next j
                    arrGrpHolder(i) = vGrpIDs
                Else
                    arrGrpCount(i) = -1 'for non-grouped shapes
                End If
            Next i
           
             'Undo the Cut:
            Application.CommandBars.ExecuteMso ("Undo")
            DoEvents
            bNeedToUndo = False
           
             'Count shapes.  Same as before?
            lChtShpsCountUncut = cht.Shapes.Count
            If lChtShpsCountUncut = lChtShpsCount Then
                 'Same number of shapes before and after cut, therefore
                 'the selected shape(s) are in a grouped shape.
               
                For i = 1 To UBound(arrGrpCount())
                    If arrGrpCount(i) > -1 Then
                         'Get the grouped shape from its ID:
                        For Each shp In cht.Shapes
                            If shp.id = arrChtShpsIds(i) Then
                                If shp.GroupItems.Count <> arrGrpCount(i) Then
                                      'This is the group that contained the selected shapes:
                                   
                                      'Compare ID of each grouped shape to array of ID's
                                      'If no match found, that's a slected shape:
                                     ReDim arrSelChtShpsIds(1 To 1)
                                     For k = 1 To shp.GroupItems.Count
                                         bMatchFound = False
                                         For j = 1 To UBound(arrGrpHolder(i))
                                             If arrGrpHolder(i)(j) = shp.GroupItems(k).id Then
                                                 bMatchFound = True
                                                 Exit For
                                             End If
                                         Next j
                                         If Not bMatchFound Then
                                             ReDim Preserve vTemp(1 To UBound(vTemp) + 1)
                                             Set vTemp(UBound(vTemp)) = shp.GroupItems(k)
                                         End If
                                     Next k
                                End If
                            End If
                        Next shp
                    End If
                Next i
               
            Else 'Selected shape(s) are not in a group:
           
                 'Compare ID of each shape to array of ID's
                 'If no match found, that's a slected shape:
                ReDim arrSelChtShpsIds(1 To 1)
                For i = 1 To cht.Shapes.Count
                    bMatchFound = False
                    For j = 1 To UBound(arrChtShpsIds())
                        If arrChtShpsIds(j) = cht.Shapes(i).id Then
                            bMatchFound = True
                            Exit For
                        End If
                    Next j
                    If Not bMatchFound Then
                        ReDim Preserve vTemp(1 To UBound(vTemp) + 1)
                        Set vTemp(UBound(vTemp)) = cht.Shapes(i)
                    End If
                Next i
            End If
        End If
       
         'Function Output:
        GetSelectedShapesInChart = vTemp
       
    End If
Exit Function
ExitPoint:
    On Error Resume Next
    If bNeedToUndo Then Application.CommandBars.ExecuteMso ("Undo")
End Function
