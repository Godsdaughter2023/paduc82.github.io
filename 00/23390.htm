What is it you're trying to "merge"? Are you certain that the different copies of the database have different data? Or are they just backups created when the conflict arose?

a. user say:

Hi,

Thank you for posting in Microsoft Community forum.

Task View is a way of expanding desktop real estate without adding an extra monitor.

  1. If you look in the lower left corner of the Task bar, you will see the Start Menu icon.
  2. Just to the right of this icon is the Search icon. The icon to the right of Search is the Task View icon.
  3. Clicking on the Task View icon reveals a prompt to add a desktop, to create a new desktop, simply click on the Add a Desktop link.
  4. Upon clicking the Add a Desktop link, your view changes to reveal two desktops, plus a blank desktop with a plus sign on it.
  5. The desktop shown on the left is the original desktop. The middle desktop is the one that was just created. The black desktop with the plus sign is really just an icon that you can click to create additional desktops. In other words, you aren't limited to only two desktops. Incidentally, it is also possible to create a new desktop by pressing WINKEY + Ctrl + D.
  6. To view a desktop full screen, just click on it. You can switch desktops at any time by clicking on the Task View icon and then clicking on your desktop of choice. You can also click on the Task View icon and then click the black icon with the plus sign to create a new desktop.
  7. You can use Alt + Tab to switch between open windows. This holds true regardless of which desktop a window might be open on. Pressing Alt + Tab gives you a consolidated view of all open windows across all desktops.

Hope this information is helpful, if not let us know with the updated status and we will be happy to assist you further.

The group's underlying site collection is hidden from the SPO admin center, but you can see the storage quota and usage via the SPO module for PowerShell or other APIs. Here's an example:

Get-SPOSite tenant.sharepoint.com/.../groupname| ft StorageUsageCurrent,StorageQuota

jerryduan_msft8021 say:

Hi Urfat,

The [tenant used storage] = [video site collections used storage] + [group site collections used storage] + [site collections in SPO admin site used storage].

But right now we are not able to see these “video site collections” and “group site collections” in the SPO admin site, so we don’t know how much storage has been consumed by these no-visible site collections.

The client side script attached is for getting all used storage for all these three type site collections mentioned above. You can run it and check SPOTenantStroageReport.csv to view the used storage for all site collections.

Here are steps for running the attached script:

1. Set up the SharePoint Online Management Shell Windows PowerShell environment by following https://technet.microsoft.com/en-us/library/fp161372.
2. Save the attach file to GetSPOTenantStroageReport.txt
3. Change the file extension from .txt to .ps1.
4. Start the SharePoint Online PowerShell as administrator and CD to the folder where the script is stored.
5. Run the command “Set-ExecutionPolicy Unrestricted” and input y for confirmation for allowing the computer run script
6. Run the script in the follow format
.\GetSPOTenantStroageReport.ps1 -searchCenter https://<contoso>.sharepoint.com/search -admin <username> -password <password> -adminSiteUrl https://<contoso>-admin.sharepoint.com
7. After the script completing, open the generated report SPOTenantStroageReport.csv, and check the sum of all site collections’ used storage.

Regarding “Is this the correct way of creating a collaboration folder”, group files is intended for the group users to share documents inside the group. For more advanced features in sharing files and team collaboration, we would recommend you use the SharePoint library and list.
