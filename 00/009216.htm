MyID: "CFL_" & [ID]

jay freedman say:

In the registry, the list of most recently used documents for Word 2013 is stored in the key

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Office\15.0\Word\User MRU\LiveId_<very long hex string>\File MRU

The <very long hex string> part is unique to each user.

Something (a misbehaving Word program or other program, malware?, a network script?) may be deleting that key or the values within it.

I don't have any great suggestions for tracking down what is responsible. As a safeguard, if you back up that key while it's valid by exporting it from regedit.exe into a .reg file, it can be restored by double-clicking the .reg file. At least that will be easier and quicker than restoring an entire profile.

jay freedman say:

Faced with backing up the MRU lists of 1000 clients, I would write a PowerShell script to do it. Rather than try to figure out the LiveId_ subkey, just back up the UserMRU key -- at most there should be only two LiveId keys under it, and they can be backed up together. The part of the script that does the local backup would be something like

$key = "HKCU\SOFTWARE\Microsoft\Office\15.0\Word\User MRU"

$path = [System.Environment]::ExpandEnvironmentVariables("%appdata%")

$file = $path + "\Microsoft\Office\MRU.reg"

reg export "$key" "$file"

You'd also want some logic there to avoid backing up an empty key. Otherwise, it would destroy any existing backup.

That would have to be wrapped in code that causes this subroutine to be executed on each client, either individually with a network signon script or as a centrally initiated task through Active Directory. (Sorry, it's been more than 5 years since I wrote that kind of script, so I'm no longer able to give you a working example.)

scott mcdaniel say:

MSysResources seems to store any resources used by your database, like Icons, Pictures, etc. It would be similar to the "Resources" folder on a .NET application.

MSysNameMap seems to store information associated with the "Track name AutoCorrect info" option. I assume it stores a "map" of the various controls and such on a Form/Report, and the AutoCorrect option would then use that table during the process. If you uncheck the "Track name AutoCorrect info" option, that table is removed from the database. If you then recheck that option, the table is added back.

As with your other question - these are internal system tables and can't be used for anything other than querying.
