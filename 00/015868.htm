norman david jones say:

Hi VisualWizzard,

Hi guys wondering if you can help me with some VBA code.

I have data in cells B3 to S47 [this range can grow or shrink] and I'd like to be able to transfer it to a separate sheet at the bottom of a table upon the press of a button called cmdTableTransfer.

So the formula needs to work out how big the range is, select it from the output sheet and move it to the last row on the Weekly Overview sheet.

In the code module of the data sheet with the CommandButton, try something like:

'=========>>
Option Explicit

'--------->>
Private Sub cmdTableTransfeR_Click()
    Dim WB As Workbook
    Dim srcSH As Worksheet, destSH As Worksheet
    Dim srcRng As Range, destRng As Range
    Dim LRow As Long

    Const sDestSheet As String = "Weekly Overview"
    Const iFirstDataRow As Long = 3

    Set WB = ThisWorkbook
    Set srcSH = Me
    Set destSH = WB.Sheets(sDestSheet)

    With srcSH
        LRow = LastRow(srcSH, .Columns("B:S"), 3)
        Set srcRng = .Range("B" & iFirstDataRow & ":S" & LRow)
    End With

    With destSH
        LRow = LastRow(srcSH, .Columns("B:S"))
        Set destRng = .Range("B" & LRow + 1)
    End With

    srcRng.Copy Destination:=destRng
End Sub

'--------->>
Public Function LastRow(SH As Worksheet, _
                 Optional Rng As Range, _
                 Optional minRow As Long = 1)
    If Rng Is Nothing Then
        Set Rng = SH.Cells
    End If

    On Error Resume Next
    LastRow = Rng.Find(What:="*", _
                       after:=Rng.Cells(1), _
                       Lookat:=xlPart, _
                       LookIn:=xlFormulas, _
                       SearchOrder:=xlByRows, _
                       SearchDirection:=xlPrevious, _
                       MatchCase:=False).Row
    On Error GoTo 0
    If LastRow < minRow Then
        LastRow = minRow
    End If
End Function
'<<=========

ken sheridan say:

As Tom says, it's better to store the PDFs as files rather than in the database.  You can then store the paths to the files in a text column in a table in the database.

The path can easily be entered into a control in a bound form by means of the FileDialog property of the Application object.  The following is a simple function to open the file dialogue, which will return the path of the selected file as a string expression after browsing to the file in the dialogue:

Function OpenFileDialog() As String

    With Application.FileDialog(1)
        If .Show Then
            OpenFileDialog = .SelectedItems(1)
        End If
    End With

End Function

So, after adding the above to a standard module in the database, the path can be assigned to a bound control in the form with:

    Me.PDFPath = OpenFileDialog()

The PDF file can be opened by calling the FollowHyperlink method, e.g. the following will open the file whose path is in the form's current record in the PDF file type's associated application:

    FollowHyperlink Me.PDFPath

Need help to hide upcoming months with blank data on a chart (Hide - Microsoft Community

justcallmeeesa2 say:

Need help to hide upcoming months with blank data on a chart (Hide until they are populated each month)

I have a spreadsheet in Excel 2013 that we manually enter data into each month, and I want it to automatically update on the chart each month. But I need to hide the future months on the chart until there is actual data to show. (I'm trying to eliminate having to reselect the data each month). Any help would be appreciated!

hansv mvp say:

You can define dynamic named ranges in Formulas > Define Name:

Name: Target

Refers to: =OFFSET($B$4,0,0,COUNTA($B$4:$B$15),1)

Name: Actual:

Refers to: =OFFSET($G4,0,0,COUNTA($B$4:$B$15),1)

Name: Cumulative

Refers to: =OFFSET($D$4,0,0,COUNTA($B$4:$B$15),1)

Name: Month

Refers to: =OFFSET($F$4,0,0,COUNTA($B$4:$B$15),1)

Now select the chart.

On the Design tab of the ribbon, click Select Data.

Select the Target Monthly Production series, and click Edit.

Clear what's in the Series values box, and replace it with

=Sheet1!Target

where Sheet1 is the name of the worksheet containing the data. Then click OK.

Do the same for the Actual Production series, using =Sheet1!Actual.

Also for the Cumulative % series, using =Sheet1!Cumulative.

Click Edit under Horizontal (Category) Axis Labels.

Clear the Axis label range box, and replace it with =Sheet1!Month, then click OK.

Finally click OK to close the Select Data Source dialog. The chart should now be dynamic.
