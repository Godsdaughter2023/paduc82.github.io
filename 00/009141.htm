a. user say:

Late response (and please see my earlier response to your other find/replace question), but as far as I know the count that you see in the search/replace pane doesn't reflect any property that the Word object returns through its find/replace object model, even in the Windows version of Word. Nor do I think that the code Word uses to populate the task pane has an equivalent in the published Word object model (i.e. I don't think there is a variation of the Find object or its Execute method in VBA or AS  that does what the pane does.)

So I suspect that the only way you could provide a count to the user would be to do a find/replace one-at-a-time, and count the number of replacements made.

In this particular case, you could use something like the following, but you would need to test it more than I have. The test for theEnd may not be necessary (and/or may not work properly if the whole document is selected. Doing this more generally would, I think, be significantly harder because you would have to cope with the fact that Word may be adding/removing a variable amount of content, changing the point at which the original Selection ends, and it might not be easy to determine exactly how much is being removed. 

tell application "Microsoft Word"
    -- this will only work when the replacement has the same number
    -- of characters as the find text (which it will in this case)
    set theSelection to the selection
    set theEnd to (the selection end of the selection)
    set theCount to 0
    set theFind to find object of selection
    tell theFind
        clear formatting
        set the content to "^~"
        tell its replacement
            clear formatting
            set the content to "-"
        end tell
        set myfound to true
        repeat while myfound
            set myfound to (execute find wrap find stop replace replace one with match forward without find format)
            if myfound then
                set theCount to theCount + 1
                if (the (selection start of theSelection) + 1) < theEnd then
                    set the selection start of theSelection to (the selection start of theSelection) + 1
                    set the selection end of theSelection to theEnd
                else
                    set myfound to false
                end if
            end if
        end repeat
    end tell
    display dialog (theCount as text)
end tell

In general, when working with AppleScript and Word, you have to bear in mind that objects can to be "dynamic" when their VBA equivalents are "static". e.g. if you set a range to be the range of the selection, then move the selection, the range will also move. But if you use "create range"to create a range, it does not. e.g. create a document with a small amount of text and run the following VBA...


Sub playWithRangesinVBAandAS()
Dim r As Word.Range
Dim rcopy As Word.Range
Dim rdupl As Word.Range
Dim s As String

' In VBA, a range set to the selection's range stays where it was...
ActiveDocument.Content.Select
Set r = Selection.Range
Set rcopy = r
Set rdupl = r.Duplicate
Debug.Print "Selection: ", Selection.Start, Selection.End
Debug.Print "Range r: ", r.Start, r.End
Debug.Print "Range rcopy: ", rcopy.Start, rcopy.End
Debug.Print "Range rdupl: ", rdupl.Start, rdupl.End
Debug.Print

' modify the selection
ActiveDocument.Words(3).Select
Debug.Print "Selection: ", Selection.Start, Selection.End
Debug.Print "Range r: ", r.Start, r.End
Debug.Print "Range rcopy: ", rcopy.Start, rcopy.End
Debug.Print "Range rdupl: ", rdupl.Start, rdupl.End
Debug.Print

' ...unless it's set to another range variable
' move the range
r.Start = Selection.Start
r.End = Selection.End
Debug.Print "Range r: ", r.Start, r.End
' the copy also moves
Debug.Print "Range rcopy: ", rcopy.Start, rcopy.End
' but the duplicate does not
Debug.Print "Range rdupl: ", rdupl.Start, rdupl.End
Debug.Print

Set rdupl = Nothing
Set rcopy = Nothing
Set r = Nothing

' In AS, a range set to the selection's range is "tied" to the selection
' A range created in the active document using the selection's start and end points is not "tied"
s = ""
s = s & "tell application ""Microsoft Word""" & vbCr
s = s & "    set msg to """"" & vbCr
s = s & "    select the active document" & vbCr
s = s & "    set r to the text object of the selection" & vbCr
s = s & "    set rnew to (create range the active document start (the selection start of the selection) end (the selection end of the selection))" & vbCr
s = s & "    set msg to msg & ""Selection: "" & the selection start of the selection & "" "" & the selection end of the selection" & vbCr
s = s & "    set msg to msg & return" & vbCr
s = s & "    set msg to msg & ""Range r: "" & the start of content of r & "" "" & the end of content of r" & vbCr
s = s & "    set msg to msg & return" & vbCr
s = s & "    set msg to msg & ""Range rnew: "" & the start of content of rnew & "" "" & the end of content of rnew" & vbCr
s = s & "    select the third word of the active document" & vbCr
s = s & "    set msg to msg & return" & vbCr
s = s & "    set msg to msg & ""Selection: "" & the selection start of the selection & "" "" & the selection end of the selection" & vbCr
s = s & "    set msg to msg & return" & vbCr
s = s & "    set msg to msg & ""Range r: "" & the start of content of r & "" "" & the end of content of r" & vbCr
s = s & "    set msg to msg & return" & vbCr
s = s & "    set msg to msg & ""Range rnew: "" & the start of content of rnew & "" "" & the end of content of rnew" & vbCr
s = s & "    set msg to msg & return" & vbCr
skip:
s = s & "    return msg" & vbCr
s = s & "end tell" & vbCr
'Debug.Print s
