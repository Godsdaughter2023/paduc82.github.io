I understand this is an old thread, but it is certainly a bug in Outlook Mac 2011, as the functionality is different in Windows versions of Outlook.  

But more importantly for me, it is creating a major compatibility issue with Office 365 - Even though my SMTP setting for my O365 account are correct, if I switch to reply via my O365 sender, O365 will reject the login with the error:

5.7.60 SMTP; Client does not have permissions to send as this sender 

This is because O365 is not only validating against the SMTP login information (which is correct), but it is requiring a matching "Sender:" header, NOT the "From:" header.  (Well, maybe it requires both)

The "Sender" header is inherited from the account that the email came in on, and cannot be changed when you set to send by another account.  This is NOT a case of "keeping headers", as by standard the "Sender:" header should be changed to match the actual sender (gasp!), not a previous recipient.

So, annoying bug when working with O365 accounts.

john p spencer say:

Try setting the criteria as follows.

>=[Forms]![QtrRpt]![StartDate] and < DateAdd("d",1,[forms]![QtrRpt]![EndDate])

That should return all records within the desired date range.  Your field probably contains a date and a time.  One way that happens is if you are using the NOW function to populate the field.  Now contains both a date and a time.   If you use the Date() function you get only the date (actually the date with a time of midnight).

Dates are stored as double numbers where the integer portion represents the date portion and the decimal portion represents the time.

barkoosh say: If all the numbers in your document (including the Table of Content) should be Hindi only, go to File > Options > Advanced. Under section "Show document content", choose "Hindi" for "Numeral". This way all the numbers in your document will appear Hindi. Please note that the "Numeral" setting depends on the Office on your computer not on the document itself, that is, if you copy your document to another computer, the display of the numbers could change depending on the settings of the Office on that computer.

However, if your document contains some numbers that you want to keep Arabic, you need to do the following:
- Choose "Context" for "Numeral". 
- Go to the Quick Access Toolbar at the top left of your Word document and click the down arrow. From the list, choose "more commands". Under "Choose commands from:" in the window that opens, choose "All Commands". From the column below look for "Rtl Run" and double-click it to add it to the column to the right, then click OK. You'll notice that a green circle appears in the Quick Access Toolbar. Now select the Table of Content and then click on that button. Numberes will then change from Arabic to Hindi. (Of course, you might need to press Right CTRL + Shift to make the TOC itself right-to-left).
