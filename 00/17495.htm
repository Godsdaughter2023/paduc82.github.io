bob jones aka: cybertaz say:
That indicates that you are (a) working in a Publishing Layout document & (b) have a Text Box containing more text than the box is large enough to display... The symbol is referred to as an overflow (or overset) type indicator.

You need to increase the size of the Text Box, reformat the text in the box (e.g., using a smaller font size), Link to another Text Box, or reduce the volume of text it contains.

NOTE: One of the most common causes is having unnecessarily pressed the return key at the end of the last line of text in the box which creates an empty paragraph [ ¶ ]. You may simply need to delete the empty paragraph.

john w. vinson say:

You can use External Data... Link... ODBC data sources to link to (not import) a SQL table, and base queries (such as append queries) on that linked table; or you can use a T-SQL Passthrough query to select a subset of the data; or you can create a SQL View and link to that.

Do note that Access SQL is a slightly different dialect than T-SQL (for example, you have to be careful about parentheses in JOIN clauses, can't use CASE, etc.) but it certainly supports JOIN and WHERE. Try building a query in the query grid (based on the ODBC link) rather than writing your own SQL and view the resulting SQL to get a feel for Access' peculiar dialect.

hannoverfist say:

Here's a sample connection string for the pass-thru query as Sean suggested since Microsoft's documentation sucks. Update the items in Bold with your info.

 

ODBC; Description=Main;DRIVER=SQL Server;

SERVER=YourServerName; DATABASE=YourDBName; Uid=YourSQLServerUserName;Pwd=YourSQLServerPassword ;

 

 

If you want to use the Windows users' credentials, use Trusted_Connection=yes instead of the UID and PWD on the last line. The users would need to have access to the db with their Windows username.

 

I've often found that my queries with JOINs on tables with millions of records returns in a couple of seconds.
