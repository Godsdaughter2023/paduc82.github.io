xps350 say: I took a closer look at your question (it was near bed time for me when I answered).

Understanding what you want now, I think we are on the wrong track. The Case/IIF stands for the (new) value of tblSpaceNumbers.Checked. But that value will always be True. What we need here is a WHERE-clause to select the right record in tblSpaceNumbers to be updated. Something like:
UPDATE tblSpaceNumbers SET Checked = True
WHERE "SpaceNumber = one of the selected space numbers"

My solution would be not use a button. You might forget that. And you don't need to update records explicit when tblPaymentInfo is the recordsource of your form.
Instead you can tell Access to update tblSpaceNumbers as soon as you picked a value in one of the combo's. You can do that by creating an 'after update event' on
************************
If not IsNull(Me.SpaceNumber1)
   DoCmd.RunSQL "UPDATE tblSpaceNumbers SET Checked = True " & _
          "WHERE SpaceNumber=" & Me.SpaceNumber1
Else
     DoCmd.RunSQL "UPDATE tblSpaceNumbers SET Checked = False " & _
          "WHERE SpaceNumber=" & Me.SpaceNumber1
End If

Me.SpaceNumber1.Requery
Me.SpaceNumber2.Requery
Me.SpaceNumber3.Requery
************************

So you have create 3 of those events.

On the other hand, you might not need the Checked field at all. You could alter the query for the SpaceNumber combo Boxes in such a way it only shows spacenumbers that are not used in tblPaymentInfo.
