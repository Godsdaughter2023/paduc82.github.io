bossdweeb say:

"i'm currectly running DirectX12 on my computer."

Yes, Win10 does have a couple of DX12 files, but it also contains many DX9,10 and 11 files as well.

Games all use different DX files and it is the game that selects which file/s is used, not Windows.

- Note that d3dx9_30 is not on the list in the dxdiag report, but it does specify that the files are already installed.

Your  problem may be due to using the often recommended old DX9.0c installer, which was superseded by

the DX Redist (June 2010) installer package, which contains all the files in DX9.0c as well as newer files

not found in DX9.0c.


- I don't know what would prevent the DX Redist (June 2010) (or the DX9.0c) package from installing, and

the DX Redist (June 2010)  is the best option when DX files have been corrupted and you don't get an error

that points to specific file names, but as you know the specific DX file try this...

- Open Windows > System32 and sysWOW64 and locate the d3dx9_30.dll .

Delete this file (or move it to the desktop as backup).

Then run the DirectX Web Installer below and it will replace the now 'missing' file.

Of course if this file is not already present it will also install this, and any other missing DX files.

Download DirectX End-User Runtime Web Installer from Official Microsoft Download Center

[EDIT: I also see that the files on the list are in your User\AppData\Temp folder.

I would either go to the Temp folder and manually delete all files (files in use cannot be deleted),

or run Windows Disk Cleanup and it will do the same, and also cleanup some other stuff.

- When using a DX Redist installer the best option is to unpack the files to a handy location of your choice

(I use a new folder on the desktop). You then have to open this folder and run the DXSETUP.exe to finalise the

reinstall of DX. This folder can be deleted after the process is finished ]

Just in case you are using DX9.0c. For future reference....

Download Details - Microsoft Download Center - DirectX Redist (June 2010)
