    Kamaci,

    Just to add a bit to Jose's information with specifics regarding KB307545...  Windows keeps regular system backups as part of the System Restore process which includes recent backups of your registry.  If your registry gets corrupted, the goal is to replace the corrupted registry with a  backup stored in the "System Volume Information" folder.  Since your computer can't boot, they set up a four-part process to get you going again.

    Part 1 Replaces your registry with a very old/basic -- but accessible --  registry so that you can boot to a primitive state.
    Part 2 The backup folder is "super hidden" and protected, so this step accesses the recent backup and copies it to a temp folder
    Part 3 Since you can't replace a registry in use, you boot to Restore Console and replace registry from Part 2
    Part 4 Reboots and does a complete System Restore to make sure everything matches.

    Part 1 is very susceptible to problems because you may have upgraded several services packs and IE versions since the registry repair files were created and it may not match your system -- plus the OEM warning.  This is most likely where Kamaci had the problem.  The workaround is rather simple. You need to boot your system from a Recovery-style CD.  Everyone has their favorite.  Jose prefers "HirensBootCD" (I've never tried that one).   I tend to prefer free Live Linux CDs such as Knoppix orUbuntu mainly because it is easy to build and Linux does not have any issues accessing files in the "super hidden" System Volume Information folder.  If you use the Ubuntu disk, recognize that it is an install disk, so be sure you use the "Try me" option and do not do an install.  To make the CD, you basically download a .iso file and use the procedure described by Jose to burn it to a CD then you boot your system with it.

    Once you have booted from the above CD, then you can pick up KB307545 at Part 2, Step 6 -- using regular GUI methods of copying/pasting the files specified in the KB article instead of doing it from a command prompt.  Both Part 2 and Part 3 can be done without rebooting using the Linux or Recovery CD.  After which you must reboot (fingers crossed) and do a regular System Restore as per Part 4.

    If everything else fails, the Linux [or Recovery] CD should allow you to plug in a USB drive into your computer and back up any of your critical data before taking the big step of reinstalling Windows.

    HTH,
      JW

    Being that the usual "temp" directory is different for each user, a possible work-around would be for you to create another user on your computer that has Administrator privilege.  Then logoff your normal user account and log into your computer as this new user and try to do some of your installs while logged in as the new administrator user.  This *should* use a brand-new clean "temp" directory to do the installation.  Hopefully this will work.

    Port Forwarding

    Netgear DGND3700

    Login Information:

    • Default Gateway - 192.168.0.1
    • Username – admin
    • Password – password

     

    Port Forwarding

     

    1. Login to the router by entering 192.168.0.1 in to the address bar.
    2. Enter the username “admin” and the password “password”.
    3. Select Port Forwarding/Port Triggering under the Content Filtering section.
    4. Click the Add Customer Service button.
    5. Type a Service Name that has something to do with Xbox.
    6. Set the Service Type to TCP/UDP.
    7. Set both the Starting Port and Ending Port to 3074.
    8. Enter the IP Address of your Xbox console in the Server IP Address space.
    9. Click the Apply button at the bottom of the page.

     

    Disable Port Forwarding

     

    1. Login to the router by entering 192.168.0.1 in to the address bar.
    2. Enter the username “admin” and the password “password”.
    3. Select Port Forwarding/Port Triggering under the Content Filtering section.
    4. Select the rule that refers to either the IP Address of the Xbox Console and/or contains the Port 3074.
    5. Click the Delete Service button at the bottom of the screen.
