  • In Outlook 2013 I deleted the entire Outlook.Com account to ensure it blew away the OST file.
  • On the server (Outlook.Com) I selected all the contacts and did a delete.
  • On Android (Lollipop), I did several things:
    • I deleted the exchange account
    • I deleted all the data associated with the exchange account
      • Settings > Apps > Contact Storage > Clear Data
      • Settings > Apps > Contacts > Clear Data
      • Settings > Apps > Exchange Services > Clear Data
    • Then I deleted the Cache on the phone (http://allaboutmotog.com/how-to-clear-cache-partition-on-moto-g-and-moto-x-after-android-lollipop-update/)

Now I was pretty sure I had no influence from anything strange so I started rebuilding the data.

  • I created the account for Outlook.Com in Outlook 2013.
  • I made sure the Outlook.Com data set was set as my default
    • Outlook 2013 > File > Account Settings > Data Files (Tab at top)
  • I opened the backup PST with the contacts and copied them to the Outlook.Com contacts in groups of 100.
  • I waited until I saw each group fully on the Outlook.Com server.

Amazingly,  the contacts started to duplicate on Outlook 2013 about 30 minutes later.  Then I noticed they were not duplicating on Outlook.com.  I downloaded the contacts on Outlook.Com to a CSV file to verify and it had the right count.  So .....

  • I deleted the account on Outlook 2013.
  • I let the server sit for several hours to make sure it had done all it's indexing.
  • I downloaded the contacts again to verify no duplication (very easy to do,  just check the total record count).
  • I then recreated the account in Outlook 2013 and let Outlook.Com populate Outlook 2013.

No Duplicates 24 hours later.

The final step was to recreate the account on my phone.

  • Settings > Accounts > Add Account > Exchange

The phone populated perfectly and still no duplicates.

This was a complete PITA but I got it working again.

While Doug's suggestion is probably the most logical, mail merging graphics is certainly possible, but you need to get the syntax right

{ INCLUDEPICTURE  { IF TRUE "https://upload.wikimedia.org/wikipedia/commons/thumb/b/b6/The_PyeongChang_2018_Paralympic_Winter_Games_Emblem.svg/263px-The_PyeongChang_2018_Paralympic_Winter_Games_Emblem.svg.png" } \d  \* MERGEFORMAT }

This certainly works using http://www.gmayor.com/ManyToOne.htm  to merge to e-mail, though the \* MERGEFORMAT part is superfluous.

Another way you could do it i to add a field (GRAPHIC) to your data source containing

https://upload.wikimedia.org/wikipedia/commons/thumb/b/b6/The_PyeongChang_2018_Paralympic_Winter_Games_Emblem.svg/263px-The_PyeongChang_2018_Paralympic_Winter_Games_Emblem.svg.png

for each record and then use

{ INCLUDEPICTURE { IF TRUE "{ Mergefield GRAPHIC }" } \d }

That too will work and if you are confused about the syntax, you could use http://www.gmayor.com/mail_merge_graphics_addin.htm  to insert it.
