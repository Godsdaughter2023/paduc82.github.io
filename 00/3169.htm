dale howard [mvp] say:

Clarkedi --

From your description, it sounds like the resource pool file has gotten corrupted, which they are prone to do by the way.  I would recommend you try the following:

  1. Open the shared resource pool file Read/Write.
  2. Open each of the sharer projects.
  3. Disconnect each sharer project from the shared resource pool file.
  4. Save and close each sharer file.
  5. Close and do not save the shared resource pool file.
  6. Delete the shared resource pool file.
  7. Create a new blank project and give it the name of the deleted shared resource pool file.
  8. Open each of the sharer projects and connect them to the new shared resource pool file.
  9. Save each of the sharer projects and the shared resource pool file as well.

Using the preceding process will actually create you a new shared resource pool file containing the names of all of the resources in all of the sharer projects.  Hope this helps.

bob jones aka: cybertaz say:

I don't know exactly what you may have been told, but no version of Word for Mac is "the same" as any version of Word for Windows... In fact, there are differences from one mac version to another just as there are differences from one Windows version to another.

Microsoft is attempting to make Office 2016 for Mac virtually identical to Office 2016 for Windows, but, in my estimation, it is a protracted process which will not be completed for quite some time. Further, IMO, the programs never will be exactly the same on both platforms simply because of the distinctions between the operating systems.

The term compatible pertains to common file types that can be used from one version to another. Many people seem to construe that to mean feature parity, which is a completely separate & distinct matter. IOW, If you're taking a version-specific class you will run into conflicts if you are using any other version.

As for your two specific points;

  1. In Word 2011 there is no Quick Access Toolbar. The QAT was introduced to Mac Office in the Office 2016 version. Even so, it is not customizable as of version 15.18 - it has been announced that customization will be added in the 15.19 update, but that has not yet been pushed out to the general population as of this writing. Having not seen it I can't be sure, but it very well may be handled in a manner that isn't exactly the same as what your lessons illustrate.
  2. Header/Footer creation can be done in either of several ways in Word 2011;
  • The tools for inserting one of the built-in H/F can be found on the Document Elements tab. The prefab H/F provided in Mac Word, however, are not identical to those provided in Win Word, nor are they named the same.
  • To create custom H/F, either go to View> Header and Footer in the main menu bar or simply double-click in the white space at the top/bottom of a page. When in the H/F 'layer' a contextual Header and Footer tab will appear on the Ribbon.
