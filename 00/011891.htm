andreas killer say:
There are 2 ways:

a) Use a primitive routine with the code and use the control as argument 

Private Sub Lab_1_Click()
  Lab_Click Controls("Lab_1")
End Sub
Private Sub Lab_2_Click()
  Lab_Click Controls("Lab_2")
End Sub
Private Sub Lab_3_Click()
  Lab_Click Controls("Lab_3")
End Sub
Private Sub Lab_4_Click()
  Lab_Click Controls("Lab_4")
End Sub
Private Sub Lab_5_Click()
  Lab_Click Controls("Lab_5")
End Sub
Private Sub Lab_6_Click()
  Lab_Click Controls("Lab_6")
End Sub
Private Sub Lab_7_Click()
  Lab_Click Controls("Lab_7")
End Sub
Private Sub Lab_Click(ByRef Ctl)
  If Ctl.Caption <> "" Then
    ActiveCell.Value = DateSerial(YearTextBox.Value, MonthComboBox.Value, Ctl.Caption)
  Else
    MsgBox "wrong, select a number"
  End If
End Sub


b) Create a class and take over the event

Create a class module, name it MyLabelClass and add this code 

Option Explicit
Public WithEvents MyLabel As MSForms.Label
Public YearTextBox, MonthComboBox
Private Sub MyLabel_Click()
  ActiveCell.Value = DateSerial( _
    YearTextBox.Value, MonthComboBox.Value, MyLabel.Caption)
End Sub


And in your Userform use the code below.

Andreas. 

Dim MyEvents As Collection
Private Sub UserForm_Initialize()
  Dim Ctl As MSForms.Control
  Dim MyClass As MyLabelClass
  'Check all controls
  For Each Ctl In Me.Controls
    'Is it a label
    If TypeOf Ctl Is MSForms.Label Then
      'Be sure we get only numeric labels
      If IsNumeric(Ctl.Caption) Then
        'Create a new object
        Set MyClass = New MyLabelClass
        'Assign the controls
        With MyClass
          Set .MyLabel = Ctl
          Set .YearTextBox = Me.YearTextBox
          Set .MonthComboBox = Me.MonthComboBox
        End With
        'Keep it alive in the collection
        MyEvents.Add MyClass
      End If
    End If
  Next
End Sub

Date parameter query - Microsoft Community

angie89 say:

Date parameter query

I want to post a query that will list everything base don today's date and tomorrow's date. It has to be a parameter query. I know how to list the 'today's date' as [Enter today's date] but how do I get tomorrow's date WITHOUT using another parameter for tomorrow's date. Can I do date + 1??

tom van stiphout say:

You don't need a parameter query. Today's date is whatever the Date function returns. Yes, Date()+1 will work for tomorrow, but I like to see you use the DateAdd function instead, because it is more explicit. +1, what does that mean? 1 day? 1 year?

ken sheridan say: I would do it like this:

SELECT *
FROM [YourTable]
WHERE [YourDate] >= DATE()
AND [YourDate] < DATE()+2;

The reason for defining the range in this way as 'on or after today and before 2 days following today' is that there is really no such thing in Access as a date value, only a date/time value, so unless you have taken specific steps to prohibit such values, i.e. an appropriate Validation Rule, the column could contain values where the time of day element is greater than zero. If you use a BETWEEN.....AND operation to define the date range any rows with such a value on tomorrow's date would not be returned.  The above approach caters for this.

You can of course base the query on more than one table, and if you wish to define the criteria in query design view you'd enter:

>= DATE() AND < DATE()+2

in the criteria row of the date column.

As Tom said, a parameter is unnecessary.  You can use the DateAdd function as Tom recommends, rather than simple date arithmetic, but personally I have no objection to the latter.

scottgem say:

Why does it have to be a parameter query? As Tom and Ken noted, if you always want today and tomorrow, the Date() function always returns today's date. No reason to prompt for it. 

However, if you want to filter for a specific date and the day after, then you would need to prompt for it. But i wouldn't use a parameter query. They are very limiting. Instead I would use a Form where you enter a Date (use a textbox named txtSearchDate). On that form you have a hidden control with a Controlsource of: =DateAdd("d",2,[txtSearchDate]).

Then, in your query use the criteria:

WHERE [YourDate] >= Forms!formname!txtSearchDate
