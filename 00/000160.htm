People contacts are very fussy about the format of data to import. If you have a CSV file that looks something like this when opened in a text editor:

FirstName,LastName,EmailAddress

then you will have to edit it so that the field names match those that Outlook.com uses, and enclose each data item in quotes:

"First Name","Last Name","E-mail Address"

The Birthday field seems to be mandatory, whether or not it contains data. The field-separating comma must be in place for any empty field (... Boyd",,"axel ...):

"First Name","Last Name","Birthday","E-mail Address"

You can add group affiliation data to each entry by adding a Company field and putting a group identifier into it for each member of the group. I suggest Company, because this is included in the Search function on the People page. I recommend starting group names with a number; this will ensure that they appear together the contacts list and makes referencing them much easier:

"First Name","Last Name","Company","Birthday","E-mail Address"

If you now use the Import contacts function at the bottom of the main People page to import this CSV file, you should find that typing 101 into the search box will find and list all contacts with 101 French in the Company field. Clicking the box next to Search results will select them all, at which point you can create a new group (Groups > New group) for your French 101 class.

If Axel is also a member of the German 102 class, add the group identifier to his Company field:

"First Name","Last Name","Company","Birthday","E-mail Address"

He will then appear in the results of a search for 102.

andreas killer say:
Do as follows:

- Download this file
  http://dl.dropbox.com/u/35239054/FileSearch.cls
- Open Excel, open the VBA editor
- Import that file
- Insert a regular module
- Copy all code below into that module
- Customize the path in the macro
  .LookIn = "F:\"
- Run the macro

I assume that your security settings allow access to the VB project.

Andreas.


Option Explicit
Option Compare Text

Sub ListMacros()
  Dim FS As New FileSearch
  Dim FName As Variant
  Dim wApp As Object 'Word.Application
  Dim wDoc As Object 'Word.Document
  Dim vbComp As Object 'VBComponent
  Dim i As Long, r As Long, t As Long
  Dim LastProc As String, ThisProc As String

  Set wApp = CreateObject("Word.Application")
 
  Range("A1:C1") = Array("File", "Module", "Macro")
  r = 1
 
  With FS
    .LookIn = "F:\"
    .FileName = "*.doc*"
    .Execute

    For Each FName In .FoundFiles
      'Ignore DOCX
      If Right(FName, 1) = "x" Then GoTo SkipFile
      Set wDoc = wApp.Documents.Open(FName, , True)
      For Each vbComp In wDoc.VBProject.VBComponents
        With vbComp.CodeModule
          For i = 1 To .CountOfLines
            ThisProc = .ProcOfLine(i, t)
            If ThisProc <> "" Then
              If LastProc <> ThisProc Then
                LastProc = ThisProc
                r = r + 1
                Cells(r, 1) = FName
                Cells(r, 2) = vbComp.Name
                Cells(r, 3) = ThisProc
              End If
            End If
          Next
        End With
      Next
      wDoc.Close False
SkipFile:
    Next
  End With
  wApp.Quit
End Sub
