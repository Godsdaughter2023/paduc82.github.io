ken sheridan say: As both your source and target tables appear to have a primary key column on which you can match the rows there is little point in trying to update one from the other by means of an update query.  It's simpler just to firstly delete all the rows in the target table which have a match in the source table, then append all rows from the source table to the target table; these will augment any rows in the target table which do not have a match in the source table, so still remain after deleting the matches.

If the structures of the two tables are identical then the queries would be along these lines:

DELETE *
FROM TargetTable
WHERE EXISTS
     (SELECT *
      FROM SourceTable
      WHERE SourceTable.ListID = TargetTable.ListID);

INSERT INTO TargetTable
SELECT * FROM SourceTable;

If the table structures differ, in that the target table has columns which are not also represented in the source table, then you need to preserve those column's values of course, so cannot undertake the delete and append operations without taking steps to do so.  This can be done quite easily by splitting the target table into two tables, both with a ListID primary key column, and one with those columns which are represented in the source table, the other with those columns which are not.  The ListID of the first of these can be an autonumber, but not of the second; that would be a straightforward long integer number data type.  Do not enforce referential integrity in the one-to-one relationship type between the tables, however, as that would prevent the delete query from executing.  The queries should now delete from and append to the first of these two tables.  The data in the other columns will be preserved in the second table, which can be joined in a query to the first table on List ID to return a result set with all columns.

It goes without saying that before attempting set operations of this nature it's imperative that the target table be backed up.

In VBA, in a class module named "EventClassModule":

Option Explicit
Public WithEvents App As Word.Application

Private Sub App_WindowSelectionChange(ByVal Sel As Selection)
    InvalRibbon   'Calls procedure in the code module to invalidate the ribbon when the selection changes
End Sub

And in a normal code module named "Module1" in VBA:

Option Explicit
Public MyRibbon As IRibbonUI
Dim X As New EventClassModule

'Procedure to invalidate the ribbon called from the event class
Sub InvalRibbon()
    MyRibbon.Invalidate
End Sub

'Callback for customUI.onLoad
Sub OnLoad(ribbon As IRibbonUI)
    Set MyRibbon = ribbon
    Set X.App = Word.Application
End Sub

'Function used by getPressed
Function getPressedOutput(strStyleIn As String) As String
    If Selection.Style = strStyleIn Then
        getPressedOutput = "true"
    Else
        getPressedOutput = "false"
    End If
End Function

'Callback for all Style Buttons getPressed
Sub getPressedStyleBtn(control As IRibbonControl, ByRef returnedVal)
    If X.App Is Nothing Then Set X.App = Word.Application

    If Not Selection.Style Is Nothing Then  'Selection contains only one style...
        Select Case control.ID
            Case "TogBtnBodyText"
                returnedVal = getPressedOutput("Body Text")
            Case "TogBtnH1"
                returnedVal = getPressedOutput("Heading 1")
            Case "TogBtnH2"
                returnedVal = getPressedOutput("Heading 2")
            Case "TogBtnH3"
                returnedVal = getPressedOutput("Heading 3")
            Case "TogBtnH4"
                returnedVal = getPressedOutput("Heading 4")
            Case Else
                returnedVal = "false"
        End Select
    Else                        'Case when selection covers multiple styles, unpress all style buttons
        returnedVal = "false"
    End If
End Sub

'Callback for all Style Buttons onAction
Sub onActionStyleBtn(control As IRibbonControl, pressed As Boolean)
    If pressed Then             'Button has just been changed from unpressed to pressed...
        Select Case control.ID
            Case "TogBtnBodyText"
                Selection.Style = ActiveDocument.Styles("Body Text")
            Case "TogBtnH1"
                Selection.Style = ActiveDocument.Styles("Heading 1")
            Case "TogBtnH2"
                Selection.Style = ActiveDocument.Styles("Heading 2")
            Case "TogBtnH3"
                Selection.Style = ActiveDocument.Styles("Heading 3")
            Case "TogBtnH4"
                Selection.Style = ActiveDocument.Styles("Heading 4")
            Case Else 'do nothing
        End Select
        MyRibbon.Invalidate
    Else                        'Button pressed when already pressed...
        MyRibbon.InvalidateControl (control.ID) 'invalidate only the button that has been (re)pressed
    End If
End Sub
