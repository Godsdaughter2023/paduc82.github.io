hal hostetler - mvp-outlook say:
You should always be able to send to the "domain" that hosts the SMTP server you're using.  The issue arises when you're connected to one ISP, but are attempting to use the SMTP server owned by another ISP. Let's say you're connected to ABC.com and you're attempting to use the SMTP server that XYZ.com owns. This SMTP server will let you send messages to anyone who has an email account with an "@XYZ.com" address, but will give you that 550 error message for any email address hosted by any other ISP.  This is actually hard to do these days because ABC.com should be blocking port 25 to everywhere but their own SMTP server (you shouldn't even be able to contact XYZ's SMTP server, when connected to ABC).  There are two other common causes:

1. An Anti Virus email scanner that's mangling your username, password, or other security info (a VERY common cause).
2. The ISP is having server issues.

Here's the function I'm using: 

=IFERROR(VLOOKUP(H5,'Inquiry Report'!A:H,8,FALSE),"8/1/2013")

Here's what I see when I sort showing the data error for "8/1/13":

Thanks for the help in sorting this out!

vijay a. verma say:

The reason is that your enclosing your date within double quotes which make it character. Dates are stored as numbers. Hence, you will have to use double unary operator before this to convert it to Numeric (There are various ways to convert this to number like you can add 0, multiply by 1 etc. I find double unary operator as most convenient).

Hence, your formula would be -

=IFERROR(VLOOKUP(H5,'Inquiry Report'!A:H,8,FALSE),--"8/1/2013")

Format your result cell as dates.
