jan de messemaeker - project mvp say:

Hi,

You will have to select the columns you want in the view before saving as pdf. Right-clicking on any column allows you to delete it (=hiding from view).

Saving as pdf will show the visible columns in the pdf as well.

Greetings,

dale howard [mvp] say:

TatiB --

If you regularly save a PDF to share with your team, I would recommend you create a new Table that contains just the columns you want to display.  This way you will not need to go through process of hiding and unhiding columns every time you want to save the PDF.  Then apply this Table in the Gantt Chart view and save the project to a PDF file.  Just an additional thought in addition to Jan's excellent answer.  Hope this helps.

john w. vinson say:

What real-life "entities" does this table represent? What is this one special field? Might the data be better modeled as a one (this field) to many (the other four fields) relationship with two tables?

That said - you can use the AfterUpdate event of the (unbound) textbox in the Header to set the Default Value property of a (bound, though it need not be visible) textbox in the form.
