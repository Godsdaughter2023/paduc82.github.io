karl timmermans [outlook mvp 2012-2018] say:
Just for peace of mind. Is this what I need to do to change my IMAP account to POP in Outlook 2013? I thought I had seen a menu option in setting. Thank you.

Do not delete the IMAP account until you have the new POP3 account configured in the same Outlook profile

OL'2013 IMAP accounts use an .OST data file using folders with the description of "This computer only" (calendar & Contacts). If you delete the IMAP account before you have copied the contents of these folders to other corresponding folders in a PST file (such as would be the case for a POP3 account) - you will lose all of that data since an OST file cannot be opened once the email account has been deleted.

Also - if you have created additional IMAP email folders - they will not be included in your POP3 account since POP3 only deals with the <Inbox>. You will have to create these folders in the POP3 folder tree and move any items over as required.

Once you have the POP3 account configured and you have moved all items over (as required) AND first ensure that these exist where you expect them to be after the move - THEN you can safely remove the IMAP account from the Outlook profile.

That is incorrect as written (underscores added).  Consider the following experiment:

A1:  ="0:00:01.234" + 0.0005/86400

It is correct (underscores added) that on
However, there is huge number of computable time values in between 0:00:02.123 and 0:00:02.124.  (Over 3.4 US trillion if the cell contains only time; only 1591 if the cell includes a current date. ;->)

That is why, for example, if you create a column of time starting with 0:00:02.123 and incrementing by "0:00:00.001", you will quickly find that some or all values do not exactly match the equivalent constant, depending on how you do the comparison.

This has been the bane of many Excel users who create columns of computed times, then use MATCH or similar lookup functions with those computed times.

joeu2004 say: Donna wrote:
I have a time of say, 00:02.1234567  (7 places past decimal)  and I can only get it to display 3 places using D:hh:mm:ss.000

As Rik explained, you cannot enter 00:02.1234567 and expect Excel to retain more than 1/1000-th of a second.  That is, the time
However, you might compute time with greater precision, either purposely or by accident due to the way that Excel stores time and does numerical computation (64-bit binary floating-point).

If that is your situation and you want to see the seconds to greater precision, you cannot use cell formatting directly.  It is limited to h:mm:ss.000 as you discovered.

However, you could write the following formula, assuming time is in A1:

=TEXT(A1,"[m]:") & TEXT(MOD(A1*1440,1)*60,"0.000000")

Note that that produces text, not a number.  Any attempt to change that to or use it as a number will result in time rounded to 1/1000-th of a second, just as if you had entered it manually.
