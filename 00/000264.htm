Font Compatibility Question - Microsoft Community

1smartcookie say:

Font Compatibility Question

I have Word 2010 and downloaded Cicle Font. How can I know if this font (or any other newer font) will be readable to someone using an earlier version of Word? Thanks!
jay freedman say:
All TrueType and OpenType fonts are "readable" by all versions of Windows and Office released in the last 12 to 15 years. As Cicle is a TrueType font, the version of the program isn't a concern.

What you do need to think about, though, is whether others reading your document have that font installed on their computers. If they don't, then Word will choose a substitute font, seemingly at random. (Actually it does try to match certain characteristics of the font, but that doesn't always work well.)

To make that font available with your document, you may be able to embed the font in the document file -- whether that's even possible depends on what "embeddability" setting the font designer specified in the font file. To try it, open the document, go to the Options dialog, click the Save tab, and check the box for "Embed fonts in the file". To minimize the size of the file, check the box for "Embed only the characters used in the document". Save the document and mail it to another computer where the font hasn't been installed, and open it in Word there. Go to the Options dialog, click Advanced, and click the Font Substitution button in the "Show document content" section. If it says there are no substitutions, you've successfully embedded the font.

jay freedman say:
TrueType fonts use a file extension of .ttf. OpenType fonts may use a .otf extension, or may also use .ttf because OpenType is an extension of TrueType. In a few cases you'll see .ttc, which is a TrueType Collection (multiple fonts in one file). In most cases it doesn't make any practical difference which of these classes any font belongs to.

In older versions of Windows and Office, there were some "bitmapped" fonts. With that kind, you needed a separate file for each size, and they were mainly useful only for screen display. Current versions of Word won't let you use them, or even show them in the font list.

Since you have Windows 7, the file manager (the Computer entry on the Start menu) will show you the font type information and more, for all the fonts you have installed. Go to the folder C:\Windows\Fonts. Set the display to Details. Click View in the menu and then click Choose Details. In that dialog, make sure at least these things are checked:
Name
Font embeddability
Font file names
Font type

Then look through the list of fonts.

The font embeddability tells you whether you can embed the font in a Word document -- "Installable" means the recipient can keep the font on their computer, while "Editable" means they can use it in your document but nowhere else. Avoid using fonts that are "Restricted license" in documents for other users. If there is no entry for a font, it may or may not be embeddable, and you'd have to experiment.

The font file names will show the file extension. The font type shows whether it's TrueType or OpenType.

jim liu msft say:
Hi Tech-Reader,

I notice the SR is closed. I'd share the summary here:

Issue Description
============
Use OneDrive to upload(sync) the files/documents to SharePoint Online/OneDrive site caused unexpected update for some files

Assessment
===========
Our product team confirmed these documents /files got updated even only upload them by OneDrive is expected behavior, as the sync engine will upload the file/document content first, then update the file/document metadata.
