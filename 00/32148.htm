lemp say:

What edition of Windows do you have and is it 32 bit or 64 bit?  If you're not sure, click the Start Orb, right-click "Computer" and select "Properties."

How is the printer connected to the computer?  It sounds as if it's connected by USB cable, but please confirm.

Do you get any error message associated with the printer? If so, please provide the complete text.

What happened to either your computer or your printer shortly before this problem started?  For example, installed some software (including Windows updates), had a malware attack, had a power failure, changed ink cartridges, etc.

Is there any common circumstance that you can associate with the "sometimes the printer would print what was in the queue"?  For example, does it print right after the computer is re-started?

Microsoft used to have a very useful automated "Fixit" tool that would reset the printing system to its original default state.  Unfortunately, in its zeal to force everyone to use Windows 10, Microsoft has removed that tool from its download servers.

You can try the Microsoft "troubleshooter" for devices and printers (5th item in the list), but I've never been able to get it to work:  https://support.microsoft.com/en-us/help/2970908/how-to-use-microsoft-easy-fix-solutions

If you can't get the "troubleshooter" to work or if it simply doesn't fix the problem, there are some manual things you can do.  Start with the following steps.  If they don't help, I can give you further instructions on how to accomplish what the now-retired "Fixit" used to do automatically.

First steps

  • Disconnect the printer from the computer
  • Go to Control Panel > Programs and Features and uninstall all HP software
  • Go to Devices and Printers, right-click on the icon(s) for your Photosmart 7520 and select "Remove device"
  • Go here and download the "HP Photosmart 7520 e-All-in-One Printer Full Feature Software and Drivers" being certain that the "Detected operating system" shown in the blue-green bar correctly identifies your installation of Windows (if it doesn't, click "Change" and fix it):  https://support.hp.com/us-en/drivers/selfservice/hp-photosmart-7520-e-all-in-one-printer-series/5199461/model/5199462

Clean out any stuck print jobs

  • Open an Elevated Command Prompt (click Start Orb,  type "cmd.exe" (without quotes); right-click and select "Run as administrator")
  • Type the following in the black Command Prompt window and press Enter after each line

                           net stop spooler
                           del /Q "%windir%\system32\spool\PRINTERS\*.*"
                           net start spooler
                          
Clean out possibly corrupt printer driver

  • If not still open from previous step, open an Elevated Command Prompt
  • Type the following in the black Command Prompt window and press Enter  
                            printui /s /t2
  • Select the HP driver in the list and click "Remove" -- be sure to remove both the driver and the driver package

Install the new driver

  • Double-click the file that you downloaded from HP (probably named PS7520_1315-1.exe) and follow the on-screen instructions.  Connect the printer when prompted to do so.

I built two win7/32 systems using the same model intel motherboard. The boards have two com ports, a db9 and a pin header. dicotyledon say:

I think I finally got it in Device Manager. The trick seems to be to do it in stages with a restart in between. Move Com2 to a whole other address/IRQ, restart, move Com1 to Com2's old address and restart. After the restart Com2 can be moved to Com1's old address and then the swapped assignments will stick without conflict.
