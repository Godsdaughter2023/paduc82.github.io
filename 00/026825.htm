don varnau say:

Hi,

If IE> Help> About says you're running IE11, you're running IE11.

Some sites have not yet been updated to properly detect and identify IE11. I have not seen this problem with Gmail or other Google sites.

On affected sites, Compatibility View may be the culprit.

Compatibility View tells websites that you're running an earlier version of Internet Explorer.

From Internet Explorer> Tools (on the Menu Bar) (or Alt + t)> Compatibility View Settings, verify that the site is not in the list, that "Use Microsoft compatibility lists" is unchecked and that "Display Intranet Sites in Compatibility View" is unchecked.

 If Compatibility View is not the issue, you can use Developer tool to emulate/spoof another version of Internet Explorer.

Use the F12 key to open Developer Tool and click on Emulation then select another version of Internet Explorer in the User Agent String box. Usually IE10 or IE9 will be a better choice than IE8 or IE7.

To get the IE window in front of Developer tool, click the unpin icon in the upper right of the Developer Tool window. Once it's unpinned you can click on the IE window to bring it to the front.

The emulation/User Agent spoof will only last while the Developer Tool window is open.

dianeoforegon say: Drag your Identity to the Shared folder.

Your Hard Drive/Users/Shared

http://www.flickr.com/photos/dianeross/7951190134/

When you "Switch Users", you can access the Identity in the shared folder.

CREATE A NEW USER

Go to System Preferences --> Create a New User in Accounts. Switch to the New User by logging out/in or use Fast User Switching.

Open Outlook. It won’t be in the Dock in the new User. Go to Applications to open.

Drag any data from Shared folder for testing to the new User. Use Database Utility to "Switch Identity".

Do you still see the issue?

If yes, then test in a new Identity. If the problem goes away then you need to move your data to a new Identity. (Any POP mail you download into the Identity in the test User will not be available unless you set to "leave on server"  in Account > Advanced settings/Server.)

http://www.officeformachelp.com/2012/05/how-to-move-your-data-to-a-new-identity/

If the problem does NOT go away then you need to look at your base files. OSX or Office 2011.
 
Remember to drag your Identity back from Shared when you log into your main User or Outlook will open in a new blank Identity.
