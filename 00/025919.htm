hansv mvp say:

Access has a very limited set of built-in mathematical functions. Here are some VBA functions, provided by Microsoft a long time ago (there are many more). You can copy them into a module in the Visual Basic Editor.

 

Function ArcCos(x As Double) As Double
  ' Inverse Cosine
  If x = 1 Then
    ArcCos = 0
  ElseIf x = -1 Then
    ArcCos = -Pi
  Else
    ArcCos = Pi / 2 - Atn(x / Sqr(-x * x + 1))
  End If
End Function
 
Function Deg2Rad(x As Double) As Double
  ' Degrees to radians
  Deg2Rad = x / 180 * Pi
End Function

 

Switch back to Access.

On the Design tab of the ribbon in Access, click Query Design.

Add both tables. For illustration purposes, I'll call them China and Capitals.

Add the City field from both tables to the query grid.

In the first empty column, enter the following expression in the Field: row

 

Distance: ArcCos(Cos(Deg2Rad(90-[China].[Latitude]))*Cos(Deg2Rad(90-[Capitals].[Latitude]))+Sin(Deg2Rad(90-[China].[Latitude]))*Sin(Deg2Rad(90-[Capitals].[Latitude]))*Cos(Deg2Rad([China].[Longitude]-[Capitals].[Longitude])))*6371

 

Replace the table and field names with the actual ones in your database.

hansv mvp say:

My apologies - I forgot one function. Please copy it into the same module where you have the ArcCos and Deg2Rad functions:

 

Function Pi() As Double
  Pi = Atn(1) * 4
End Function

 

I had used Capitals and China as table names. Since your tables are named CAPITALS and CITY, you should change China to CITY in the expression.

 

Finally, I'd name the calculated column Distance or something similar, instead of Module1.

 

Distance: ArcCos(Cos(Deg2Rad(90-[CITY].[LATITUDE]))*Cos(Deg2Rad(90-[CAPITALS].[LATITUDE]))+Sin(Deg2Rad(90-[CITY].[LATITUDE]))*Sin(Deg2Rad(90-[CAPITALS].[LATITUDE]))*Cos(Deg2Rad([CITY].[LONGITUDE]-[CAPITALS].[LONGITUDE])))*6371

When I upgrade a preinstalled (OEM) or retail version of Windows 7 or Windows 8/8.1 license to Windows 10, does that license remain OEM or become a retail license?

If you upgrade from a OEM or retail version of Windows 7 or Windows 8/8.1 to the free Windows 10 upgrade this summer, the license is consumed into it. Because the free upgrade is derived from the base qualifying license, Windows 10 will carry that licensing too.

If you upgrade from a retail version, it carries the rights of a retail version.

If you upgrade from a OEM version, it carries the rights of a OEM version.

Full version (Retail):

- Includes transfer rights to another computer.

- Doesn't require a previous qualifying version of Windows.

- Expensive

Upgrade version (Retail):

- Includes transfer rights to another computer.

- require a previous qualifying version of Windows.

- Expensive, but cheaper than full version

OEM :

OEM versions of Windows are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system

As it pertains to the OEM licenses this will invalidate the Windows 10 upgrade license because it will no longer have a previous base qualifying license which is required for the free upgrade. dhookom say:

All of the "documented" properties, values etc. are temporarily stored in a table in the wizard file in your profile folder. You can link to the table [doc_tblObjects] to create your own queries and reports. In my Access 2007 version the file is:

C:\Users\hookom\AppData\Roaming\Microsoft\Access\ACWZUSR12.ACCDU

The name changes based on your version of Access. 

The queries for table and fields with type and size is:

SELECT doc_tblObjects.Name AS TableName, doc_tblObjects_1.Name AS FieldName, 
doc_tblObjects_1.Extra2 AS FieldType, doc_tblObjects_1.Extra3 AS FieldSize
FROM doc_tblObjects AS doc_tblObjects_1 INNER JOIN 
     doc_tblObjects ON doc_tblObjects_1.ParentID = doc_tblObjects.ID
WHERE (((doc_tblObjects_1.TypeID)=11))
ORDER BY doc_tblObjects.Name, doc_tblObjects_1.Name;

You can add more copies of doc_tblObjects to drill down to additional properties. This table is refreshed when you run the documenter.

pamcaswell say: OpenType is a new version of, or rather it is based on, TrueType. It was developed by Microsoft and Adobe, and its on-screen rendering is tuned for LCD monitors. (Since TrueType was introduced in the nineties, I suspect it was tuned to CRT displays.)      The two companies each developed new OT fonts, with emphasis on their customer base's needs.  MS concentrated on providing support for world and special characters.  Adobe, on typographical features. 

W2010 supports limited typographic features and, according to several long posts here, has had trouble with (sometimes substituting another font for)  some Adobe-style OT fonts—although this may have been corrected with the release of the service pack. W2007&10 apparently have no problems with TT fonts.  If you are only using Word, check out how well the OT font works before deleting the TT.  If you use a publishing program (such and MS Publisher or InDesign) that uses OT fonts, you may want to keep both.
