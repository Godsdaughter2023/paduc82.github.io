There are several reasons that cause the file is locked, you could refer to this: "The document is locked for editing by another user" error message when you try to open a document in Word.
1, Word previously quit improperly and, therefore, did not delete the owner file.
2, A second instance of Word is running in the background with the document already open.

To make this better: delete the temporary folders or files here are few ways to do that:
1, quite the client completely when you close Office. 
2, manually delete the temporary files and folders:
    In Terminal, paste the following: defaults write com.apple.finder AppleShowAllFiles YES
    Press return
    Hold ‘alt’ on your keyboard, then right click on the Finder icon in the dock and click Relaunch.
    In finder, go to the folder where the problem doc exists.
    Delete: .~lock.(Doc Name).ods#(or ~lock.xxx.ods# )

bernie deitrick say:

Don't use hyperlinks - use the Selection_Change event of your worksheet to filter the way you want, like so, where FilterOne and FilterTwo are macros in a standard codemodule that apply the filters in the manner you want, and B2 and B3 are your 'specific cells'.

1) Copy this code.
2) Right-Click the sheet tab of interest.
3) Select "View Code"
4) Paste the code into the window that appears.
5) Save the file as a macro-enabled .xlsm file.
6) Make changes as needed

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    If Target.Address = "$B$2" Then FilterOne
    If Target.Address = "$B$3" Then FilterTwo
End Sub

Alternatively, you could insert buttons and assign those macros to the buttons - a little neater and more 'appliation-like'

rohn007 say:
Think of your document as a stream of text. Word applies the formatting to the text inserts graphics and what not into the text stream and when it calculates it has "filled" a page, inserts the logical page break.

Even when you insert a "page break", it is a logical break.

Compare this to the way PowerPoint handles pages. When you insert too much content, it falls off the bottom of the slide, it does not automagcially get transferred to the next slide.

In word there are no physical "pages". Pages are entirely a logical construct created by the program.
