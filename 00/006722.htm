jay freedman say:

Those are not "ghost copies", they're a type of temporary file known as an "owner file". Word uses the existence of the file to prevent other users (usually on a network) from opening the document while you have it open. This is explained in the "Owner File" section of http://support.microsoft.com/kb/211632.

By the way, the second character in the file name is a dollar sign, not an S.

If Word crashes, one or more owner files and other temporary files may be left orphaned on the drive. In that case, you can and should remove them -- see http://www.gmayor.com/what_to_do_when_word_crashes.htm.

stefan blom say:

An example of a seemingly blank page which isn't really blank is if you press Enter repeatedly until the insertion point moves onto the following page. The "blank" page is filled with paragraph marks (¶) which adds white space and an extra page to your document. To get rid of such a page, just select and delete the paragraph marks. The task will be easier if you first display nonprinting marks, which you can do by clicking the ¶ button on the Home tab.

Note that it is enough if a single paragraph mark gets pushed "off the page" for a new page to be created. This happens if a table is at the bottom of the page, and there isn't room on the page for the paragraph mark following the table (yes, a table, just like a paragraph of text, is always followed by a paragraph mark in Word). In such a case, you can format the paragraph mark as hidden.

True blank pages are added if your text ends on an odd page and you insert an Odd Page section break. Word adds a "filler" page to ensure that the new section starts on an odd-numbered page.

For more on all this, read MVP Suzanne S. Barnhill's excellent article at http://wordfaqs.mvps.org/BlankPage.htm.

1) and 2)  the attribute "dynamicanimation" is something FrontPage puts in that does nothing whenever you use one of the FrontPage DHTML effects. Delete the attribute and its value wherever you see it. (delete  dynamicanimation="whatever")

3) A properly formed script tab takes the form:
<script type="javascript">

</script>

or

<script type="javascript" src="url.js"></script> if the script is in an external file.

The language attribute is no longer required or used - one of the updates that browsers adopted. FireFox actually requires the type attribute, IE will work without it, and the standards say it must be there.
