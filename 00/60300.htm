¡firedog say:

This is by design. You can read about how Windows Live Mail decides to populate the To: line of an outgoing message here: Address messages.

If you want to assign nicknames to contacts but still have First name Last name in the outgoing message, put the nickname in the Notes field. That way, you can still easily find a contact by typing his nickname into the To... box, but the picker list will show his first and last names and those will be used in the outgoing message.

The way messages are displayed in the contacts window has no bearing on how they're used when addressing messages.


Repairing the program will of course not change any behaviour that's by design. It won't do any harm, though.
