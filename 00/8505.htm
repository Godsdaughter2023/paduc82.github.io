Locking a time/date from updating to current - Microsoft Community

andrewvolkenant say:

Locking a time/date from updating to current

How do put a time stamp in  a cell, based on input in another cell and lock it to that time data entered, with out it automatically updating when refresh

jeeped say:
The typical method of providing a cell with a date/time stamp is through an event Worksheet_Change macro procedure which puts the current date/time value into an accompanying cell when input is detected.

To put a timestamp into column C then a value is added, edited or deleted in column A, right-click the worksheet's name tab and choose View Code. When the VBE opens, paste the following into the pane titled something like Book1 - Sheet1 (Code),

Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Target, Columns("A")) Is Nothing Then
        Application.EnableEvents = False
        Dim a As Range
        For Each a In Intersect(Target, Columns(1))
            a.Offset(0, 2) = Now
        Next a
    End If
    Application.EnableEvents = True
End Sub

Tap Alt+Q to return to your worksheet once you have that pasted in. Anything typed into column A will result in a static date/time value being input into the corresponding cell in column C of the same row.

Post back if you have trouble modifying that for your own purposes.

The issue of a web browser being redirected to a fake "tech support" web page purporting to be Microsoft or any other well known and trusted business isn't typically able to be detected by antivirus or most other security applications, primarily because it's not actually malware.

In most cases these redirection popup pages are nothing more than a small script launched by an advertisement which was directed to display by the very same advertising networks that display any other ads, so there's really nothing there for an AV program to detect.

There's no real hacking going on here and the reason these can appear to be "locked" in place on the screen is due to flaws designed into the older web browsers (all of them), at a time when attacks like this weren't occurring.  In most cases these issues with the browsers can't be fixed on the older operating system platforms, since in order to fix them the changes would need to be so severe as to break many of the millions of web pages all over the Internet, leading to utter chaos.

Microsoft has already fixed many of these problems on the more modern Windows 10 S Mode operating system delivered with all Microsoft Surface tablet and laptop devices and many current 3rd-party computers as well.  However, these significant changes can't be made to the older browsers without breaking things as mentioned above, so the only way to get them is to choose to upgrade by buying a new device that ships with the Windows 10 S Mode installed.

All of the major web browser vendors have been working hard to move away from these outdated and flawed designs for a few years now and so these issues will mostly disappear as people replace their old hardware devices with new ones over the next few years.  For example, that Windows 7 system you have will officially end support from Microsoft on January 14th, 2020, so you should be planning what you'll purchase to replace that relatively ancient operating system that started sales a decade ago in 2009.

Actually, it isn't just Windows 10, since that doesn't stop the browser based ransomware, it's the very specific version of Windows 10 in "S" mode that doesn't contain the browser vulnerabilities which enable most of these popup exploits.

Windows 7 is ancient in computer terms and anyone with true security concerns stopped using it a few years ago, just as with Windows XP which was never a good operating system in terms of security and so was replaced by those with knowledge of security several years before it ended support by Microsoft itself.

Of course some will keep these outdated operating systems around, but that doesn't make it a good practice.  Unlike an old car where the issue is just old hardware that needs physical maintenance, an old PC that in today's world is Internet attached by nature is a risk to your financial health and safety.  Keeping it around for whatever reason is dangerous, so there's really no good excuse, ever.

Heartbleed was an issue with the OpenSSL cryptographic software library used as the core of most of the world's secure electronic communications, which had nothing to do with the CPU hardware.

Meltdown was the greatest immediate risk for Windows based systems, but the easiest to fix quickly in software.  Spectre was more technically difficult to fix, but also much more difficult to exploit, so though it affected nearly all currently popular processors it was never really exploited before there were fixes at least available from most major operating system developers.

These processor based hardware vulnerabilities really had little to do with the individual operating systems that run on them and in fact it was the browsers that were often updated most quickly and easily in order to insure that the most likely attacks were protected from before they ever became a true issue.

The hardware design issues that enabled these vulnerabilities are also understood and have been fixed already in most affected processors, so newer versions don't typically contain them.  Another reason to upgrade your physical devices, since the software fixes were really workarounds that required additional overhead and thus slowed the operation of the older affected systems that contained them.

The problems being discussed in this thread related to browser popups can't be easily fixed in older browser versions simply because the items being exploited are inherent to the designs of those older browsers.  Microsoft and most other browser developers have already designed out many of these issues, but they can't be redistributed in the older browsers and operating systems due to compatibility agreements Microsoft has with businesses relating to these, so the only way to resolve them completely is to move on from these old systems.

Though it's true there will always be new vulnerabilities and exploits, many of the easier to exploit items have already been discovered, with human social engineering the most prevalent remaining that will never truly be completely "fixed".  Whatever can be done to protect a person from himself aids in reducing these risks, but until everyone in the world grows up in an education system that teaches basic literacy relating to scams, whether technical in nature or not, there will always be fools available to be tricked (e.g. conned)

What is real is that most larger businesses have learned they need to keep ahead of the security curve, and that the cheapest way to do this is to purchase the latest equipment and software to go with it every 5 years or so.

Consumers and very small business are a different situation, since they're mostly cheap and uninformed, so the manufacturers have realized they need to deal with this problem in a different way.  Those like Apple began quite a while ago by making much simpler devices that many consumers wished to keep current, such as the iPhone and iPad, so rather than keep them for many years the consumer would tend to replace the entire device more often including the software that came with it.

However, that system has begun to break down as the devices became more technically capable and so expensive, resulting in similar problems to those the PC and other general purpose computers have had for decades.  Some like Google began to realize they'd need to maintain the devices they sold with patches or upgrades for at least a few (typically 3) years, expecting the consumer to keep them that long.

The other method is to make the devices themselves relatively cheap, which is where those like China's Huawei and other low cost manufacturers have mostly headed, though with less truly secure results.

Microsoft has remained in the better support group, switching their operating system support paradigm to one of software-as-a-service, or more specifically, Windows-as-a-service, where the significant upgrades come automatically as part of the security maintenance of the operating system.  This solves the historical problem that major upgrades were required regularly, but still  doesn't completely resolve the issue of an uneducated public making the wrong decisions, which has always been the true problem.

For those who care, there are 2 techniques I've successfully used throughout my computer career that over the long haul are usually the most cost effective, while still retaining the best security possible throughout the life of the hardware originally purchased.

One is to buy the latest and most technically capable hardware you can afford, seemingly overpaying for the hardware today, since that tends to future proof the ability for that same hardware to continue to operate effectively with future operating system upgrades, without the need to upgrade the hardware itself beyond more minor external devices.  This works most effectively for business and those with enough money to pay for the hardware in advance and was how I dealt with most of my purchases during my own computer career.

The other is to buy slightly less capable, but often much cheaper device models from last year, which won't last as long but save enough to allow replacing them more often.  This also has the advantage of replacing the operating system with the device, so rather than software upgrades you only need to install new (mostly free) app software and possibly migrate some data.

However, since most today such as banks, financial and other businesses I deal with store the data for me, I only need to maintain my own personal data like pictures and video, which is also much easier and more safely done in the cloud.  That means the hardware is even easier to replace, simply buying a new device and connecting it to my online identity when the old one either fails, is lost or damaged or simply gets too old to be useful.

The problem you bring up last of old donated hardware, stating "You have to work with what is real", isn't a problem the manufacturers can truly solve.  The financially limited person will always exist, which the cheaper models help with, but can't be solved except by giving away devices and in truth, free software has become common today with the ad supported model.

I no longer use the expensive device model, since as a consumer with plans to travel, a cheaper, lighter device makes more sense than the often larger, heavier business devices anyway.  However, I still bought a decent device in the Microsoft Surface Go tablet, since I want modern capabilities, albeit with smaller display and less of the high-end features I often got with business devices.

The Microsoft Windows-as-a-service model and other trends like their plans to replace the current Windows 10 Edge based on Mozilla open-source with a browser based on the Chromium open-source

model instead will only continue to make Windows better, so future support should be easier as well.

That's the more important thing for consumers to understand, that regardless of what small patches Microsoft tries to apply to outdated operating systems like Windows 7, these can never truly be "fixed".  Moving to Windows 10 and especially to modern devices that include the much more secure Windows 10 S mode is the best way to insure your personal data security far into the future regardless how it's acquired.

Rob
