jasonmercer say:

I know it's been a while since your post, however i would like to stipulate an

answer to this particular issue.

 

As with all issues they can be quite challenging and cause a bit of a headache.

 

Shaun is right that it is an external software entity that interupts the flow and

continuity of communication on a networked Client.

 

If it is not BackupExec thats causing the interruption, I have had the same issue

with regards to the client running Windows 7 and may apply to Vista, which I can't

comment on as we went from XP to Win7 without going through Vista phase.

 

If you have Active Sync enabled this can have the same effect in that it will interrupt

communication between the client and homefolders on the server.

 

A Word of warning you may experience some data loss contained within files this is because

you have offline files and online files because of the syncronisation issue.

 

This is were backups come into there own.

 

Here's what you need to do to disable it;

 

1. Click on the orb or start menu

2.Type in the dialogue box (Search Programs and files) and type active sync

3. Select 'Sync with other computers, mobile devices, or network folders.

4. This will bring up a dialogue box in the left hand pain select manage offline files

5. You need to select disable offline files

 

It will ask to be rebooted. Let it.  This should resolve your issue.

 

One tip that I have gave myself before joining any client to the domain is to disable

the active sync before hand and to configure the server accordingly.
