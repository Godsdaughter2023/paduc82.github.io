You can move a virtual machine to another computer as a virtual machine, of course, but the problem with using a v2p program to install the copy of Windows 10 now running in the vm to another computer as a native OS is that the hardware will be different.  By all accounts that would cause deactivation and you cannot use the dummy product key to activate.  Remember that Windows 10 uses device activation, not just software activation.  You cannot clean install to a device that is not already a registered Windows 10 device without a retail or programmatic (MSDN, volume license, etc.) product key.

rich007a say:

In VBA... in 2010, you can use Application.StartNewUndoEntry to start a new entry in the undo stack.  But in 2007 we have to use this work around:

If you do something using Application.Commandbars.ExceuteMso("...") then a new entry is created in the undo stack.  If you don't want that action to appear in the undo stack, you can undo it with

    Application.Commandbars.ExceuteMso("Undo")

A new stack entry will then be left for whatever the code does after calling the undo.

To get the string to include within the brackets, add a command to the QAT, then in the customize the QAT window, hover over it and you will see the string in brackets.

Which command to use will depend on what you are doing - obviously the command button has to be enabled to be able to press it.

So if you have a shape selected, you could use:

            Application.CommandBars.ExecuteMso ("ObjectFlipHorizontal")
            Application.CommandBars.ExecuteMso ("Undo")

but if you have text selected, you could use

            Application.CommandBars.ExecuteMso ("Bold")
            Application.CommandBars.ExecuteMso ("Undo")

You can test what is selected with If ActiveWindow.Selection.Type = .... Then

Alternatively, you can have your code create a shape and select it, flip and undo, then delete it.

Hope that makes sense.

ronrosenfeld say: If you should care to do this with a User Defined Function, which has some benefits in terms of being able to more easily adapt to changing requirements, I submit this:

To enter this User Defined Function (UDF), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this User Defined Function (UDF), enter a formula like

=ParseString(B365, Index, [NumberValues])

or, to return the 2nd instance of Text values:

=ParseString(B365,2, FALSE)

Index is a number from 1 to 3 indicating which sequence you want to process.
NumberValues if omitted or TRUE will return the number values as numbers;  if FALSE the preceding text values will be returned.

If you should want to return the number values as strings, a minor change in the UDF can accomplish that (delete or comment out the next to last line).

==============================================
Option Explicit
Function ParseString(s As String, Index As Long, _
        Optional NumberValues As Boolean = True) As Variant
 Dim re As Object, mc As Object
 Const NumPattern As String = "\b\d+(\.\d+)?\b"
 Const TextPattern As String = "[A-Z].+?[A-Z](?=\s\b\d+(?:\.\d+)?\b)"
Set re = CreateObject("vbscript.regexp")
With re
    .Global = True
    .ignorecase = True
    .Pattern = NumPattern
    If NumberValues = False Then .Pattern = TextPattern
  If .test(s) = True Then
    Set mc = .Execute(s)
    ParseString = mc(Index - 1)
  End If
End With
If NumberValues = True Then ParseString = CDbl(ParseString)
End Function

abi99 (2) say:

Your question is not that easy, because Nokia changed a lot in Wi-Fi auto-selection. Nokia changed a lot in the course of Symbian/S60. You use a quite early device, a Symbian/S60 3rd Edition Feature Pack 1 device from 2008. Furthermore, some apps (even some made by Nokia) did not use the latest features of the current platform release. This created a lot of confusion back in 2008. And makes your question even more complicated today: We have to double-check everything with a device from that time.

Actually, if your phone is configured correctly, your Nokia E63 is able to determine whether it is at home (in range of your Wi-Fi). Then, it uses mobile data. If your phone stays in mobile data while coming home, turn it off and on again. Alternatively (a bit faster), go for the profile Offline and then back to your normal profile. Consequently, if you still face problems with this, please, tell us which apps you use! Then, we double-check their settings together.

By the way: Removing the Access Point is not permanent. You are able to add it again at any time. With todays networks, all you need is the Access Point Name (APN) for Internet of your network operator in your country. Simply extract that name from the existing access point. However, the Internet moves to a concept called Encrypt All The Things which encrypts every connection via SSL/TLS. Furthermore, the Internet moves from legacy encryption to the latest standards – which are not backward compatible. Consequently, with a Symbian/S60 3rd Edition Feature Pack 1 device like yours, you face a lot of issues while surfing already today. Actually, a proxy-based browser like Opera Mini is your last resort.

Therefore, if you like Symbian that much, I recommend to buy a Nokia E72 or Nokia E6 now. Used phones just get worse when you wait any longer. Although I recommend a touch based phone because they matured a lot and are so easy to use today. Furthermore, you get help at every corner with those, for example with a Apple iPhone 4s.
