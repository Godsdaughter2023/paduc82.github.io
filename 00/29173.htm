ken blake say:

You can't do a system restore without restore points. Doing that would be like flying without an airplane (or similar device).

But see Alan Russell's reply. A restore point wouldn't help you, but an undelete program might.

"Deleting" a file doesn't actually delete it; it just marks the space as available to be used. There are third-party programs that can sometimes recover deleted files. The problem is that the
space used by the file is likely to become overwritten very quickly, and this makes the file unrecoverable.

So your chances of successfully recovering this file is decent if  you try recovering it immediately after deleting it, and rapidly go downhill from there. If you've been using the computer since then (for example to write this question and read this answer), your chances may be very poor by now.

But if the file is important enough, it's worth a try anyway. Stop using the computer in question immediately, if you haven't done so already. Download an undelete program (here's a good one:
http://www3.telus.net/mikebike/RESTORATION.html but there are several others to choose from; do a web search) on a friend's computer and bring it to yours on a CD or thumb drive to try.

If this fails, your only other recourse is to take or send the drive to a professional file recovery company. This kind of service is very expensive and may or may not work in your case.

gerry c j cornell say:

Select Start, All Programs, Accessories, System Tools, Disk CleanUp to empty your Recycle Bin, remove Temporary Internet Files and other selected options. I also recommend you click on the More Options tab, System Restore and remove all but the latest System Restore point. Run Disk Defragmenter.

Using the System restore option is most appropriate when your system is running slow and you need to increase free disk space on your C partition. If you have errors which might be resolved using System Restore to put your system settings back to before the error first occurred do not use this option.

If more than one person uses the computer and they have their own user Log on you need to run Disk CleanUp in each user profile to remove all unwanted files.

A better programme than Disk CleanUp is cCleaner but note that cCleaner will not handle System Restore points. Removing older restore points can be important if you have limited free disk space. cCleaner comes with a Registry tool. Do not use this tool as like any registry cleaner it can lead to the creation of very difficult to resolve system errors.

Hope this helps, Gerry Cornell

belalugosi say: There is another possibility that some people might find is "mysteriously eating space".

I set my Windows search (W7-X64) to look inside archives (zips, aka "compressed folders") etc, vastly improving my searching capabilities, BUT (there's always a BUT with Windows) what this means is that all the archives are opened one-by-one and searched. This "unzipping" is done in your Appdata/Local/Temp folder (which I think is a hidden folder too) and the files are not removed afterwards, so each search you do adds unzipped data into the location above (Appdata-temp folder). If you are searching a folder (or disk) containing thousands of files you might easily end up with thousands of unzipped files on your C-drive (which is where most people store their profiles) and find your system slowing to a crawl, or getting that annoying red bar that nags you that you are running low on disk space.

I compounded my problems because I had an explorer window opened in search mode that I left between sessions so that W7 reopened the search every time the window was restored after a reboot.
