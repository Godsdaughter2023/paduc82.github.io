This error message is due to a policy setting: HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\RestrictRun. A few options:

1. Boot to safe mode, login with build-in administrator account.
    If the registry restriction is only set for the current user, build-in administrator account may be able to use registry editor

If you can launch Registry Editor, Delete the following key:

HKey_Users\<Each_SID>\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\RestrictRun
HKey_Users\<Each_SID>\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\RestrictRun\*.*

2. Boot to Safe Mode with Command Promot
    You should be able to launch registry editor using Command Promot

When you can start Command Prompt, delete the following registry key and subkey
HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\RestrictRun
HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\RestrictRun\*.*
HKLM\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\RestrictRun
HKLM\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\RestrictRun\*.*

3. If you cannot boot in Safe Mode, boot to WinRE using your Vista installation disk. Use Reg.exe to delete the incorrect registry key
Reg Delete HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\ /v RestrictRun
Reg Delete HKLM\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\ /v RestrictRun

a. user say:

.opf files are associated with six programs that i know of (and probably more) http://www.fileinfo.com/extension/opf and http://filext.com/file-extension/OPF. I can  find nothing about .opef files - I don't think it exists.  I have neither of these file types in file associations on my 32-bit Vista Business SP2 system (with a lot of image software installed). You can check yours by going to Start / Control Panel / Default Programs / Associate a file or process with a program and scanning the list for .opf and .opef.  If you have them, see what program it uses to open the files.  If you don't have them then you have no program capable of opening the files (as they are named).

Are the files still on your usb drive with their original file extensions?   Are they still on the old computer?  You can try doing the process again to see if this strange occurence happens again (hopefully it won't and everyting will be OK).  Just copying files should not change their extenstions (even if the new machine doesn't have software to read the extensions and especially not if it does) - but you already know that and are looking for a solution.  I would try changing the extension of one of the files that was changed back to the original extension and see if you can open it with the appropriate software.  If that workd and you can't transfer the files without them changing extensions (and I don't understand at all why that's happening0 then your only choice may be to rename each of the files to give it the proper extension.  Either that or try downloading one of the programs in the first link of this post and see if you can open the .opf files with it.  If so you may need to learn to use that program for working with those transferred files.

You may be infected with malware. To fix this problem (if it is a problem) download,  install, and run the following two programs: http://www.malwarebytes.org/mbam.php and http://www.safer-networking.org/en/index.html.  You may also want to try a free trial of Pandasoft (www.pandasoft.com).  Before downloading pandasoft, uninstall whatever anti-virus program you are currently using (because installing and running two at the same time can cause conflicts, freezes, and all sorts of problems).  Once you've uninstalled your current AV program, download, install, update, and scan using Pandasoft.  I think you might be amazed at how much it finds that the other software missed.  Once complete, uninstall pandasoft and re-install your current AV program (unless you decide to switch which I did when I first tried it about 4 years ago).  You can also try the new, free Microsoft Security Essentials at: http://www.microsoft.com/security_essentials/default.aspx (with the same caveat that you can have only ONE anti-virus program installed and running on your system at any one time).  They say that MSE has the lowest impact on PC performance while i've found that Pandasoft finds the most malware.  You can decide if you want to keep your current AV program or switch to one of these others.  Then try again to transfer the files.

If this doesn't work (and please do try to do the scans with each of them), then let's check some of your system files:

Go to Start / All Programs / Accessories / Command Prompt and right click on command prompt and click run as Administrator.

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no errors it could not resolve (if it does have such errors, post back here or you can try to analyze them using http://support.microsoft.com/kb/928228).

That;s about it for now (that's certainly enough to keep you busy with things you'd rather not be doing  I realize - but if we're to recover these files we have to eliminate some potential problems that may be causing the issue *unless you want to change all the files one-by-one if that process even works).

Good luck and let us know how things are going.  We'll be here to offer more help if these proposed solutions don't do the trick.  One way or another, we'll get your files back somehow.
