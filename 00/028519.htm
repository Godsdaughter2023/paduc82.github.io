scottgem say:

Cascading comboboxes are a standard tool and they are well documented. Searching on the term will find several articles that detail how to do them.

The first step in the process is to create lookup tables for each of your lists. If your zones are only numbers then you can use a Value list for the zone combo. The lookup table for District should look like this:

tluDistrict

DistrictID (Primary Key Autonumber)

District

Zone

The lookup table for Block should look like:

tluBlock

BlockID (PK Autonumber)

Block

DistrictID (Foreign key)

Each Combobox has a RowSource property. The RowSource should be a query against the respective lookup table (again, if Zone is only numbers, you can use a Value list). The initial RowSource for the District should look like:

SELECT DistrictID, District FROM tluDistrict ORDER BY District;

Then, in the After Update event of cboZones you would enter code in the After Update event (using the Code Builder) like so:

Me.cboDistrict.RowSource = "SELECT DistrictID, District FROM tluDistrict WHERE Zone = " & me.cboZone & "ORDER BY District;"

This will restrict the Districts to only those in the Selected zone. You do pretty much the same thing with Blocks. 

A couple of other points. The only value you should be storing is the BlockID. You can get to the Zone and District by using Joins.

Note, if your subform is a continuous form, then this becomes much more complex. Ken Sheridan has an example of this in his sample apps.
