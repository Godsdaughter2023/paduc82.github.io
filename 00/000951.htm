jeeped say:

This sounds like a floating point error where the interval that is being added gets 'nudged' by floating point rounding and displays an extra second. The infinitesimal small difference between what Excel is adding eventually grows to the point where an additional second is added. For date and time progressions (and particularly time progressions) put the starting date/time in a cell (e.g. A1) then use something like the following in A2,

=$A$1+ROW(1:1)/24

Fill down as necessary. This always adds a certain number of hours to the original so any small difference will not stack up from line to line. In drafting, this process is called datum dimensioning although I'm not entirely sure that is an appropriate label here.

Hyperlink Description
TIME function Returns the serial number of a particular time
ROW function Returns the row number of a reference

Edit: while I didn't originally realize the scope of your data progression, I would still recommend adding a progressive amount to the original starting date/time instead of incrementing each new line. I've modified my original formula above to suit.

jay freedman say:
The *.qat files should be in the folder C:\Users\<your name>\AppData\Local\Microsoft\Office. Ordinarily, the AppData folder and everything under it is hidden, which probably explains why your search failed to find the files.

In Windows Explorer, click Organize > Folder and Search Options. In the dialog, click the View tab. Under Hidden Files and Folders, click the option for "Show hidden files, folders, and drives" and click OK. Now your search should find the files, or you can just go directly to the folder.
