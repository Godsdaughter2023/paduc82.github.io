As for your thinking that " At the early stage of installation, I noticed that Windows XP had wrongly detected the small system partition or drive as being the C:\ drive" that is an incorrect statement.  The Windows NT/2000/XP setup program has always assigned the C: letter to the first enumerated active partition, this was only changed with Vista and later operating systems where the setup program assigns the C: letter to the boot volume rather than the system volume.  The boot volume holds the Windows files, the system volume holds the system files necessary to boot the operating system, for Windows XP these are NTDETECT.COM, ntldr and the boot.ini files.

If you really insist or need to have Windows XP installed on a C: drive then flag the present G: drive as the active partition then proceed to reinstall Windows XP cleanly on the newly flagged active partition, the Windows XP setup program will assign letter C: to the active partition.  After Windows XP is installed move the active partition flag back to the present small C:drive and add the Windows XP installation to the Windows 7 boot configuration.

john john __ say: By "old software" I meant that you were maybe trying to install old user software on Windows XP, like trying to install old games or old accounting or spreadsheet programs from the Windows 95 era, some of these old software packages were sometimes hard coded to look for and install on a C: drive, I didn't mean that you were trying to install an old operating system.

As for the the instructions on the sevenforums all that these instructions really tell you is that when an operating system is installed it rewrites the boot sector of the active partition and that after the installation you must then restore the boot sector so that a common boot loader can be used to start all the operating systems.  When you install Windows XP after Windows 7 is installed the XP setup program will rewrite the boot sector and install the XP boot loader (ntldr).  Ntldr cannot boot Windows 7, the Windows 7 boot loader can boot Windows XP so in a multiboot environment you after Windows XP is installed you must then return the boot sector to the Windows 7 boot loader to be able to boot both operating systems.

As for the drive letters please keep in mind that drive letters are really almost completely irrelevant.  Drive letters are symbolic links assigned by the operating system and on NT operating systems it really doesn't make a hill of beans of difference which letters are being assigned to the drives, it is not a requisite that a "C:" drive exist on an NT system.  Also keep in mind that being that drive letters are a contrivance or creation of the operating system that the partition drive letters from one operating system to the other need not be and often are not the same.  You could have 3 partitions on a disk (partitions 1, 2 & 3) and one operating system could assign letters C:, D:, & E: to the partitions while another operating system on the same drive could assign them letters D:, E:, F: or other completely different letters and it wouldn't make a hill of beans to any of the operating systems, it would not cause one operating system to fail to boot or to not work properly.  Granted that this can be annoying and cause confusion for the user or that  it may cause problems if the user is using common scripts or batch files which refer to drive letters.

On NT operating systems the persistent drive letter assignments are recorded in the Mount Manager database at the HKEY_LOCAL_MACHINE\SYSTEM\MountedDevices key and as far as the operating system is concerned it is completely blind to what letters another system on the same computer assigns to the drives.
