The Scheduled Start field indicates the date when work on a task is scheduled to begin as calculated by Microsoft Project, based on dependencies, constraints, calendars, and other scheduling factors. john - project say:
Megan141,
The Scheduled Start field is actually the old original Start field. When the developers added manual scheduling mode in Project 2010, instead of creating a set of new fields for manual mode, they decided it was more appropriate to re-define existing fields. Not sure users would agree that was the best approach but that's what happened, so some confusion has arisen.

Now in Project 2010 and up, when in manual schedule mode the Start field will take on whatever value the user enters, including string values (e.g. "next friday"), whereas the Scheduled Start is automatically populated by Project with the Project Start Date, assuming there are no predecessors or start constraints. A similar thing applies to the other "Scheduled" fields.

In auto-scheduling mode both the Scheduled field and the non-scheduled fields, (i.e. Scheduled Start and Start), will track.

Hope this helps.

John
