bob jones aka: cybertaz say:

Well, I'm not sure what you've done in the past because Mac PPT has never supported linking to Excel workbooks or any other file types (other than media files). The only option is an embedded copy of a workbook. The chart on the slide can be updated based on changes made to the embedded copy, but there is no connection between either the chart or the embedded workbook & the source workbook on which it originally was based.

When working on a Mac, the only workaround I know of is to link the embedded workbook to the source workbook. That still requires that the embedded workbook be opened & updated in order for the chart to update. IOW, the chart doesn't update automatically. The method requires that the original workbook be available, but links are rather fragile when moving files from one location to another -- especially when different operating systems are involved. The best policy is to store the source workbook file in the same folder as the presentation & transport the entire folder, but even that can be dicey. At the very least it requires a high degree of coordination & cooperation between users.

I imagine you'll get input from others as well, so stay tuned :-)

pbezucha say:

Dear Astoria,

It really depends on the kind of interrogation you want to do. You have to really learn the principles. If you really mean it, you will see, however, that the normal ways that Excel offers are to no avail, if you apply them on S-shaped curves. Linear regressions of higher order are too swung, while none of other available regressions are able to keep pace with inflexion, which is the main feature of this type of curves.

Thus the only way is to select an equation of one of S-shaped curves (look up for sigmoid curves in Wikipedia) and optimize its constants by non-linear regression. Nothinq extraordinary complicated. For example, the arithmetic formula for so called logistic function is

y= (c3-c4)/(1+exp(-(x -c2) /c1))

Here x is an independent variable, c1 to c4 are regression constants. They can be obtained by means of Solver. If you happen to know lower limit c4 exactly (=0?) then you will have one unknown constant less, the same with c4.

You have your x and y data evidently arranged in column ranges. You must create another parallel range filled with formulas according to the equation. If you reserve the cells C1 to C4 for constants c1 to c4, and your x-values begin with A5, then in C5 you will write

= (C$3-C$4)/(1+EXP(-(A5-C$2)/C$1))

(I suppose you are acquainted with $ convention.)

You will obtain #VALUE as a result, as the division by zero takes place. Solver makes all the regression calculation (finding the minimum sum of squared differences between y data and y calculated for all x) for you, but you must provide proper first shot for each constant. Do it now: in C3 and C4 the limits, C1 be 1, C2 be x-value in the area where the data undulate (inflexion). Copy the formula downward along your data ranges.

The sum of squares to be minimized must be defined in a cell, say B4. Write the Excel function (from function list) SUMSQ into this cell, and define the difference of ranges as its arguments. It will have a look like

 =SUMSQ(A5:A50-C5:C50)

if you have 50 data points. When enter the formula, you must press Ctrl and Shift simultaneously, as, in this case, it is a matrix one.

Solver is an Excel add-in (learn on Help if not available); it is activated by Tools button. In the menu you must select B4 for Set Target Cells: Min for Eqal To: and all the range C1:C4 for By Changing Cells:. Now you can start Solve. If successful, Solver supplies optimum values in the range C1:C4, and you can use the y-calculated range for drawing regression line.   

Regards,

Way back in the days of Excel 2003, you could drag a point on the chart to a new location and Excel would change the value of the cell corresponding to that point.

This assumes that the points are joined by straight lines. Allows the user to place a vertical line at the x-axis value that they want, and returns the interpolated y-value. No VBA required.
Warning; only works in Excel 2003 and earlier.
