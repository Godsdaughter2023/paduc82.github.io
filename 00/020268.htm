rohn007 say:

Uninstall in this case is optional.  The uninstall process does not contact the MS activation system so it is not aware that you have done this.

There is no special procedure for re-installing. Just do a "normal" install.

Yes you can re-install on the same computer with a replaced HD. That is part of the license.   The activation process takes a "snapshot" of your hardware and software. The re-install compares the original snapshot to the new one. Since most elements, other than the HD, are the same the activation should be allowed.

In theory, you should be able to do an internet activation when you do the re-install.  If it does not work, you will have to do a phone activation.  You can try the phone activation process.  It may work, but probably will not. In that case, phone in again, but this time do not reply to the initial prompts.  Wait, a long ... time ... , until you are offered the option of talking to a live service rep.

The need for uninstall is to comply with the use terms of the license if you are transferring the license to a new computer.  In that case you do have to do an uninstall.  But that license term does not apply to your situation.

Amongst others the main benefits of a Table are:

  • As you add data the address of the data range is dynamically adjusted by Excel
  • The formulas, formats… are automatically copied on the new lines/rows
  • You can refer to the columns by there names (formulas easier to read)

here. I named the table containing the data tblData but you can always change it. If you look to the formula in F3 sheet Overview it now says:

Easy to understand it refers to tblData and to the columns by their names (Agent, Date…).

Now, the one thing I needed to explain is why Agent and Date are repeated between double brackets ( [[Agent]:[Agent]] ) but [Crit 1] isn't… If we enter the following formula:

=SUMPRODUCT(--(tblData[Agent]=$A$2); --(MONTH(tblData[Date])=$B$2); --(YEAR(tblData[Date])=$C$2); --(tblData[Crit 1]=1))

and copy it right it becomes (in the 1st next cell):

=SUMPRODUCT(--(tblData[Date]=$A$2); --(MONTH(tblData[Time])=$B$2); --(YEAR(tblData[Time])=$C$2); --(tblData[Crit 2]=1))

This means: [[Agent]:[Agent]] is an absolute ref. like $A2 while [Agent] is a relative ref. like A2

In your scenario the ref to the Agent and Date columns must be Absolute but the ref to the Crit column should be Relative
