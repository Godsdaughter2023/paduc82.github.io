jean-guy marcil say: Spambutnospam (I'll have eggs, bacon spam, spam and spam, it hasn't got too much spam...),

Just a heads up, in case you are not too familiar with frames (most people are not, a pity, really, they are so versatile and useful... as Jay's excellent solution demonstrates). Word considers that whatever content it finds inside the frame comes before the paragraph text to which the frame is anchored. In other words, frame content is part of the text flow, as opposed to Text Boxes which are on the drawing layer, apart from text flow.

Keep that in mind when moving frames around and when deleting paragraphs surrounding frames. For instance, a frame at the bottom of the page but anchored to the first paragraph in the document will be the starting editing point when you open the document, or the editing point you reach when doing CTRL-Home as opposed to the top of the first page...

Frames take some getting used to, but they are very handy.

jay freedman say:
You're correct that text wrapping inside a text box is not supported.

There is a solution that involves creating a custom style. It's a bit complicated to set up, but very easy to use once that's done.

  • Start with a document based on the template you're creating for this type of document.
  • Click the dialog launcher arrow in the bottom right corner of the Styles group on the Home ribbon, or press Ctrl+Alt+Shift+S, to open the Styles pane. Click the New Style button at the bottom left of the pane.
  • Name the new style (e.g., "InfoBox"). Make it a Paragraph style based on the List Bullet built-in style. Set the "style for following paragraph" according to your needs -- probably Normal or Body Text.
  • Near the bottom of the dialog, check "Add to Quick Style list" (or not, your choice). Be sure to clear the box for "Automatically update" -- that one's a disaster waiting to happen. Select "New documents based on this template".
  • Click the Format button and choose Frame. In the next dialog, set text wrapping to Around (this is for the box, not the graphic), set the width to Exactly and enter a size, and set the horizontal distance from text to something around 0.2". Leave the other settings at their defaults, at least for now. Click OK.
  • Click the Format button and choose Numbering. In the next dialog, click the Bullets tab, click the Define New Bullet button, click Picture, and select the file containing the graphic you want to use. Click OK until you get back to the New Style dialog.
  • Click the Format button and choose Border. Click the Box button in the left column. Modify the width and color if you want to. Click OK.

To use this, insert a paragraph in the regular text just before the paragraph to which it refers, and apply the InfoBox style to it. You'll get a box with the width defined in the Frame dialog, with a border, and with the graphic in the first line, followed by a tab character. Start typing, and the frame/border will expand downward as needed.

If you need more than one paragraph in a singe info box, you'll need another custom style (e.g., "InfoBox Continue") that is based on the InfoBox style, but with the Bullet set to "no bullet". Then modify the InfoBox style so its "style for following paragraph" is set to InfoBoxContinue.

Starting point is that "activation" is not permanent.  MS periodically goes back and validates all installations/activations. We have no way of knowing when this revalidation will happen.

Sorry, the keys are long enough (25 char) and "complex" (most alpha & numbers) enough that every product key is unique (NEVER "reused") and is 1-to-1 associated with a completely specific product.  That association is never changed!

How can keys become invalid/blocked ... lots of ways:

The simplest way is via stolen Product Key Cards.  Not all cards have to be "activated" at the store. If these already active PKC are reported stolen, the Product Keys are eventually identified to MS and marked as blocked/invalidated in the activation system

Next common way is that someone makes a "legal" purchase of a volume licenses.  These keys are intended to be used only within the company that bought them.  The person then illegally (contrary to license terms) resells these product keys to general public. Usually via a short term "discount" key web site.  Eventually, MS figures out that these keys are being used/activated in a pattern that indicates/suggests illegal use.  They block the keys.

Oops! Sometimes the message is simply wrong.  We have been seeing lots of that happening with the "upgrade" to Win 10 and the "upgrade" to Office 2016. In that case, when you absolutely know you have a valid license, you have to phone the activation center and talk to a "service" rep to (hopefully) clear up the problem.

"Invalid" vs "Blocked" is a matter of terminology. Yes I agree with your interpretation, but this is MS ... a global corporation with offices around the World. Lots of ESL workers who can have different understanding of words.

john sr wilson say:

Fairly easy to construct code to add the last saved date to the master

Something like:

Sub addLST()
Dim strLST As String
On Error Resume Next
ActivePresentation.Designs(1).SlideMaster.Shapes("LST").Delete
strLST = ActivePresentation.BuiltInDocumentProperties("Last Save Time")
With ActivePresentation.Designs(1).SlideMaster.Shapes.AddTextbox _
(msoTextOrientationHorizontal, 10, 10, 100, 10)
.TextFrame2.TextRange.Text = Format(strLST, "dd/mm/yyyy")
.Name = "LST"
End With
End Sub

Certainly possible to make an AddIn to do this automatically on File Open and File Save but not simple.

I think the text is being pasted all in one cell because it's not coming from a table. Try this:

  1. In a new document, paste the new text
  2. Select it and convert it to a table. Word should create one table cell per paragraph. (You may need to convert line breaks instead of paragraph breaks, so in the "Separate text at" box type Shift Return.)
  3. Make sure the number of cells is the same vertically. Otherwise, some content will repeat when pasting or blank cells will appear.
  4. Select the new table, including the weird marker to the right of the cells.
  5. Copy to the Clipboard.
  6. In the original document, select the column to replace.
  7. Paste.
