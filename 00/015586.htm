    joeu2004 say:

    First, XIRR does not return 0.00%. You see that simply because of the way you formatted the cell: presumably Percentage with 2 decimal places.

    If we format the cell as Scientific, we see that the result is about 2.98E-09.  In my experience, that is an error state. In this case, the interpretation is similar to when XIRR returns a #NUM or #DIV/0 error, namely: it indicates the need for a "guess", the 3rd parameter for XIRR.

    Indeed, =XIRR(B1:B32,A1:A32,-12.5%) returns about -13.4232640848495%.  And with that discount rate, the NPV is about

    2.66E-07, which is relatively close to zero.

    We cannot use XNPV because it does not permit negative discount rates, for no good reason. It is a design flaw.

    But with the XIRR formula in D2, we can use the following formula to calculate the NPV, which is derived from the math formula in the XNPV help page:

    =SUMPRODUCT($B$1:$B$32/(1+D2)^(($A$1:$A$32-$A$1)/365))

    You might wonder how I determined the "guess" value to use.  I look at the NPV curve for varying discount rates.  For example:

    The formula in G2 (copied down the column) is:

    =SUMPRODUCT($B$1:$B$32/(1+F2)^(($A$1:$A$32-$A$1)/365))

    Note the transition from positive to negative NPV between -15% and -10%.  Presumably, NPV is 0% in between. (But it might not be!) And -12.5% is halfway.

    Other "guesses" might work just as well; for example, -10%.  However, sometimes the NPV curve is so steep or variable that we need to choose a very close "guess".

    (And sometimes a "very close guess" does not work due to a flawed implementation design in Excel XIRR, IMHO.)
