It looks as if many of those who revert to Windows 7 from Windows 10 experience a corruption of the store folder (the folder where messages, calendars and account settings are stored). Depending on how complicated your store folder is (many accounts, many calendars, many messages in many folders), it may be simplest just to create a new store folder and repopulate it:

   

  1. While viewing your message list in Windows Live Mail, press Ctrl-Shift-O for Options. On the Advanced tab, click Maintenance and then Store folder. Copy the path shown for the location of the store folder.
     
  2. Close down Windows Live Mail and wait a few minutes (the program will continue to run in the background while it does its housekeeping).
     
  3. Press Windows key + E to open an Explorer window, then paste or type the path you copied at (1) into the address bar and press Enter. This should open your store folder.
     
  4. Right-click on the folder name in the navigation pane and select Rename. Call it Live Mail Backup.
     
  5. A repair should check that all the program files are present and correct and properly registered where necessary. To do this, close all running programs, then press Windows key + R, type wlarp and press Enter. In the window that opens, select Repair all Windows Essentials programs:



    When it's finished, reboot the computer.
     
  1. Launch Windows Live Mail and wait until the program settles down before doing anything. Then press Ctrl-Shift-T to start the Add your email accounts wizard and follow the prompts to add your email accounts again.
               
  2. Click once on Working online on the status bar and disconnect from the Internet to prevent interference from email downloads, then use File > Import messages (Alt-F I) and point the wizard at the Live Mail Backup folder. Don't interfere while the import is going on.
     
  3. Leave the program alone for some time (an hour or two, or even overnight) to allow it to re-index all your messages, then close it down and wait again for half an hour for the program to do its housekeeping.
     
  4. Re-launch the program and look for your messages under Imported folder in Storage folders. Taking it gently, drag folders out of Imported folder and drop them in the location you want them. Again, give the program plenty of time to re-index messages in their new location.

¡firedog say:

... the emails I had in my storage folders are all toast. They all state: Message cannot be found ...

When I open my mail I still get the box entitled "Windows Live Mail Storage Recovery" ...

There's something you've misunderstood. Both of those symptoms indicate that the message database index is either corrupted or incomplete. Since the procedure is intended to create a completely fresh store folder, it can't really have had time to be corrupted (e.g. by shutting down the computer while the program is still busy). So the index must be incomplete, because you didn't allow the program time to index all the messages in their new location.

When you select a folder to view, the program displays in the message list all the entries from its database that have that folder recorded as their location. If you have the reading pane visible, or you open one of those messages, the program will dig into the message store to retrieve the contents of the message from its .eml file and display them. If the indexing process is incomplete, the program will not be able to find the message content and will show either a blank message body with an error message or, in some circumstances, the body of some other message. 

I tried this more than once and it did not work.

This should be a one-time procedure, and it will take a long time if there are many messages and folders involved. If you did it more than once, what did you delete and where from before trying again? You must have deleted something, because you can't have more than one folder with the same name in the same place.  

One last thought: what security/privacy/'performance enhancing' software do you have running? Anti-virus or 'system care' programs will probably intercept every single message before it's written to disk, and this can of course interfere (unnecessarily) with the process. You can safely disable any such programs while you're not connected to the network.

    First, the zoom to center is gradual, taking the full duration of your clip.  So if you have a clip that is longer than a couple seconds, you probably won't notice the zoom in.  Second, since video has motion, it will often mask the fact that you are zooming in. 

    Try zoom in on a picture to confirm it is working.  Now go back to your video, remove the zoom in and look at the end of your clip.  Notice how much of the video is showing especially noting where the edge of the frame is.  Take a snapshot/screenshot if you want proof.  Now apply the zoom in effect and again look at the end of your clip.  You should see that the clip is zoomed in.
