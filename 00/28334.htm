dhookom say:                 

Function GetPCSerialNumber()
    Dim objWMIService As Object
    Dim colItems As Variant
    Dim objItem
    Dim strMsg As String

    Set objWMIService = GetObject("winmgmts://./root/cimv2")

    Set colItems = objWMIService.ExecQuery("Select * from Win32_BIOS where PrimaryBIOS = true", , 48)

    For Each objItem In colItems
        strMsg = objItem.SerialNumber
    Next

    GetPCSerialNumber = strMsg
End Function

petri j say:

Hi,

When just pushing the "Insert" button, PowerPoint embeds the image into your PowerPoint presentation file. I don't have the possibility to check now the behaviour in these earlier PowerPoint versions, but I recall it was similar.

Embedding the images into the PPT file may or may not be a good idea, depending on what you will be doing with the presentation. The problem is that the file may quickly become very large. You will also lose some image quality (so embedding your images into a PPT file is certainly not a way to store your original images). The good thing is you don't have to worry about breaking file links; so it seems this would not be the problem in your case.

To me your problem sounds like some kind of resource problem with PowerPoint, as it is apparently occasionally not able to show the pictures. It is relatively common to see this kind of behavior, usually connected to having a lot of and/or very large images. However to my experience (last 15 years) PowerPoint has become much better in managing large images compared to early versions, so you experiencing it only now seems strange. I have not noticed cropped images to cause special problems, but surely PowerPoint needs to do extra processing to show those.

You could use a trial version of Image Essentials for PowerPoint to analyze the images in your PowerPoint presentations and, in case of any external file links, to validate the links as well. You could also make a mass move from embedded to external files, although you will get better quality if you still have the original image files around to link to. (Disclaimer: I am the author)

My point is,

1) Make sure you understand where you images in the decks are actually stored and whether the file links are working

2) Try out if resources (memory etc.) are a problem, either by trying to use external file links, or by splitting the deck into smaller parts

With the earlier PowerPoint versions, I regularly had to split decks with 100+ pages with large images into several smaller files.

Hope any of this helps.

I have a procedure along the lines of (perhaps it can inspire you):

Sub ChangePrinterSettings(sReportName, Optional sWHERE)
    On Error GoTo Error_Handler
    Dim rpt                   As Access.Report
    Dim prtr                  As Access.Printer
    Dim sRptName              As String

    sRptName = sReportName
    If IsRptOpen(sRptName) = True Then DoCmd.Close acReport, sRptName

    Set application.Printer = Nothing
    Set prtr = application.Printer

    prtr.Orientation = acPRORLandscape
    prtr.PaperSize = acPRPS11x17

    If IsMissing(sWHERE) Then
        DoCmd.OpenReport sReportName, acViewPreview
    Else
        DoCmd.OpenReport sReportName, acViewPreview, , sWHERE
    End If
    Set rpt = Reports(sReportName)

    Set rpt.Printer = prtr

Error_Handler_Exit:
    On Error Resume Next
    Exit Sub

Error_Handler:
    MsgBox Err.Number, Err.Description, ChangePrinterSettings"
    Resume Error_Handler_Exit
End Sub
