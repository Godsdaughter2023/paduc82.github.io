stefan blom say:
Changing the page orientation requires section breaks; there is no way around it.

With regards to endnotes, what you must do is set endnote location to "End of section" and then suppress their display for all sections except for the one where you actually want them. The general principles are discussed at http://word.mvps.org/faqs/formatting/footnotefaqcontent.htm#TextAfterNotes.

Note that the article is not updated for recent versions of Word. In Word 2013 you can double-click the vertical ruler to display the Page Setup dialog box, or you can click the dialog launcher button in the Page Setup group on the Page Layout tab. To display the Footnote and Endnote dialog box, you can right-click endnote text and choose Note Options from the context menu.

suzanne s. barnhill say:

The workaround Stefan suggested works for Word's own endnote feature but possibly not for Endnote software, not to mention that the section breaks still cause problems with page numbering.

 

One way to rotate content without adding section breaks is to select the content on the page, copy it, then use Paste Special to paste it back in as a picture, which you can then rotate. The drawback, of course, is that the picture will not be editable, so this should not be done until editing is complete. Also, depending on picture resolution, it may not be as clear as the actual text.

chakraview say: This is how you can do it.

Excel has an inbuilt capability of transforming tab seperated values in different columns. You can simply try this by opening the txt file, copying the whole text and pasting it in a excel sheet. All the tab seperated values will automatically arrange in the coulmns. Since I am not sure, how many files are we talking of here, i cannot suggest you an optimal way to get it to a workbook.

Assuming that you figured out how to do this, try arranging the 16 rows from text1 file to first 16 rows, then followed by the next 16 rows from Text2 file. Thus every 17th row is the 1st row from your new text file. Let say, this is done in Sheet1

now go to sheet 2, in A1 cell, place the formula = sheet1!a1. Drag this formula horizantally to fill up as much columns you need to get all the columns in the row from 1st sheet.
now in A2, place the formula = sheet!a17. repeat the exercise as above.
Now drag vertically on column a, to cover the entire range. 

thus sheet 2 is filled with 1st row data from every text files. Repeat the exercise for getting the 2nd rows in the next sheet.

macropod say:

Hi Chris,

To do this, press Ctrl-F9 twice to create a pair of nested field (ie '{{}}'). Then fill in/around these so you end up with:

XXXX{=MOD({MERGEFIELD Account},10000) \# 0000}
where 'Account' is your mergefield's name.

When you're done, run your mailmerge.
