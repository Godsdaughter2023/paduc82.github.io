Hi Sonny,

The default view for Music (or other) search results can be modified, but only through a Registry edit. Have you ever used Regedit before? What is you general level of expertise/comfort with the computer? I'm happy to guide you through the process.

That being said, Windows does attempt to save modifications to Search Result views the same way it saves views for individual folders. But there is a limit to the number of views saved (5000). When you hit this limit, the OS has to delete a previously saved view every time it saves a new one. When a folder (or search) doesn't have a saved view, it uses the default view.

Clearing saved views would most likely make Search results behave better, but you would lose the individual views saved for regular folders as well. However, it's easy to set a custom view for any of the standard folder templates (General, Documents, Music, Photos, and  Videos). So if you use the same custom Detail view for all (or at least the vast majority) of your Music folders, this might prove useful as well.

A good starting point would be to see if you have maxed out on saved views. We can check this using PowerShell. Launch PowerShell using WinKey+X (or right-clicking the Sstart button) and selecting Windows PowerShell. Copy and paste the following command (all one line). It should return a numerical result. Please include that in your replay as well as answers to my earlier questions.

((gp "HKCU:\SOFTWARE\Classes\Local Settings\Software\Microsoft\Windows\Shell\BagMRU").Nodeslots).count

keith a. miller say:

OK. Trying to make a relatively quick post to get you started. I'll have more time later to elaborate if needed.

The Registry key that defines default views is:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\FolderTypes

So the first thing to do is right-click that key and Export to a file so you have a backup before making changes.

The subkey for Music Search Results is:

You want to modify the values found under:

TopViews\{0167146e-5395-4c48-9048-d584eeaca4f2}

To change the view to Details, change LogicalViewMode from 5 to 1.

To change to columns, you need to edit ColumnList. When trying to edit these long string values, I find it easier to copy the existing entry from the Edit dialog, paste into Notepad, edit, and then paste back into the Edit dialog. Default ColumnLIst is:

prop:0System.ItemNameDisplay;0System.Music.TrackNumber;0System.Title;0System.Music.Artist;0System.Music.AlbumTitle;0System.Rating;0System.ItemFolderPathDisplayNarrow;1System.Search.Rank;1System.ItemTypeText;1System.Size;1System.DateCreated;1System.DateModified;1System.Music.AlbumArtist;1System.Audio.EncodingBitrate;1System.Music.Genre;1System.Media.Duration;1System.DRM.IsProtected;1System.Media.Year;2System.Music.Conductor

Easier to read in a list:

prop:0System.ItemNameDisplay
0System.Music.TrackNumber
0System.Title
0System.Music.Artist
0System.Music.AlbumTitle
0System.Rating
0System.ItemFolderPathDisplayNarrow
1System.Search.Rank
1System.ItemTypeText
1System.Size
1System.DateCreated
1System.DateModified
1System.Music.AlbumArtist
1System.Audio.EncodingBitrate
1System.Music.Genre
1System.Media.Duration
1System.DRM.IsProtected
1System.Media.Year
2System.Music.Conductor

A property preceded by 0(zero) appears in the default view. A property preceded by 1 places it in the context menu presented when you right-click a column header.

So delete, add, and reorder those to your liking. Additionally, examining other ColumnLIst entries shows that column width can be specified as seen in this example:

prop:0(34)System.ItemNameDisplay;

I'm not sure what 'units' the 34 represents. You'll have to experiment.

You will need to log off & login for the changes to take effect.

Keith
