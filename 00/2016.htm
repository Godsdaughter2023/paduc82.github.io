Exchange Kiosk USD 2 (Web access, 2GB Mailbox)

Your challenge is that you are already on the lower cost SMB plan and prices are already special.

If you switch SOME of your users to Business, then THOSE users will lose email feature. The rest of the users will still have email and your primary domain will not be affected.

The <>"" portion simply means 'do not count blanks'. The &"" forces the value being examined to be considered text in order that you will not receive a #DIV/0! error. COUNTIF() will happily treat numbers and text that looks like the same number as the same.

The SUMPRODUCT() wrapper produces the array processing that is required to reiteratively examine each value for uniqueness while maintaining a count.

It may be best to explain how that formula actually works.

First, SUMPRODUCT() will take a series of formulas, calculate each in turn and produce the sum of the result.

Second, you have to realize that a boolean FALSE is mathematically 0 (e.g. zero) when used in a calculation with other numbers. For all intents and purposes (at least here) a TRUE is 1. So anytime that a cell in C13:C282 is blank (i.e. zero length string) then the division formula with have 0 as the numerator and we all know that no matter what the denominator is, the result will be 0.

Next, the COUNTIF() function is in an array format. It would not normally have ranges in both the range and criteria parameters; only the range. This is where the array processing comes in and converts each count into 1/count. BY converting the COUNT() into its own reciprocal and adding them together they will equal 1.

Consider the following,

                  

C2 has

=SUMPRODUCT((A1:A10<>"")/COUNTIF(A1:A10,A1:A10&""))

A1 is one of four blanks so this cycle will be ⁰⁄₄ or 0.

A2 has CAD and there are two CAD's in A1:A10 so the cyclic calculation of <not blank>⁄<count> will be ¹⁄₂.

A3's calculation result is ¹⁄₃

A4's is the same at ¹⁄₃

A5 is another CAD so it is ¹⁄₂,

A6 is the only USD so this is ¹⁄₁.

A7 is another EUR at  ¹⁄₃.

A8, A9 and A10 are all ⁰⁄₄.

SUMPRODUCT() then totals these cycled calculations.

⁰⁄₄ + ¹⁄₂ + ¹⁄₃  + ¹⁄₃  + ¹⁄₂ + ¹⁄₁ + ¹⁄₃ + ⁰⁄₄ + ⁰⁄₄ + ⁰⁄₄ = 3

So there are three unique values. Easy-peasy. Remember that the &"" is necessary to avoid the #DIV/0! error.

ronrosenfeld say:

Your request seems a bit odd as the formula in M5 will refer to $A4 -- in other words, the ID lookup will always be offset by one.

I also assume that you don't really want EVERY cell to refer to $A4 -- but the row reference should change for each step down.

In any event, to fill all the cells using VBA, again, entering and running the macro in the manner I describe above, try this:

Option Explicit
Sub FillWithFormula()
    Const sFormula As String = _
        "=IFERROR(VLOOKUP($A4,'Adhere time pivot'!$B:$CV,(MATCH(L$1,'Adhere time pivot'!$4:$4,0)),0)*(24*60),"""")"
    Dim R As Range
Set R = Range("M5:BX200")
R.Formula = sFormula
End Sub

Of course, if you want the formula to refer to the cell in Column A that is in the same row as the formula, then make one of these changes:

In your formula string, change $A4 to $A5

There is a similar issue with the Column information in your MATCH function.  It refers to a column that is offset by one compared with the location of the formula.

Windows Defender is a full anti-malware program that come built-in to Windows 8. so there is no need to install it.
