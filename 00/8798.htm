Hi Brian,

If the problem you are having is with the path coding to the file between PC and Mac, here is something that you can experiment with and hopefully will help you. Note that the path formatting between Office 2016 for the Mac and Office 2011 for the Mac are also different.

Sub Save2PDF()
' Save document as PDF
    Dim fPath As String
    #If Mac Then
        #If MAC_OFFICE_VERSION < 15 Then
            fPath = "~:Documents:Fool:"
        #Else
            fPath = "/Users/<UserName>/Documents/Fool/"
        #End If
    #Else
        fPath = "C:\Users\<UserName>\Documents\Fool\"
    #End If

    With Dialogs(wdDialogFileSaveAs)
        .Name = fPath & ActiveDocument.Name
        .Format = wdFormatPDF
        .Show
    End With
End Sub

This code uses object methods that have to be isolated from the VBA compiler (note the # sign that does the isolation). These special object methods determine if the machine is a "Mac" and if so the code proceeds accordingly.

With Office 2016 for the Mac, Microsoft also began officially using the Unix separator of "/" versus the former Linux separator of ":" that Office 2011 exclusively used. In addition, Office 2016 for the Mac does not recognize the former use of the tilde symbol "~" that formerly was a shortcut for "HD Name:Users:<UserHomeDir>"
