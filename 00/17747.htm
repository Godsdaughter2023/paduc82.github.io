    SolidWorks 2015 crashing when clicking 'Open file' or - Microsoft Community

 gazyg  say:


            SolidWorks 2015 crashing when clicking 'Open file' or 'Save As' on build 10240

                    
After trying a full advanced SolidWorks reinstall, rolling back to a previous graphics driver which I had been using with no problems for the last few months, to fix this problem, it turned out that this was easily fixed so I'll post it here in readiness
 for those SolidWorks users who may get this problem after the Windows 10 launch upgrade.




Go to the Start Menu folder, which is normally here: -




C:\ProgramData\Microsoft\Windows\Start Menu\Programs\SOLIDWORKS 2015




or right-click the icon on the Start menu and click 'Open file location'




Right-click the 'SolidWorks 2015 x64 Edition' icon and click 'Properties'




Go to the 'Compatibility' tab and click the tick box 'Run this program in compatibility mode for:' and in the drop-down box select 'Windows 8' and then 'OK'.




'Open file' and 'Save As' should now work as normal. 




Also note, if you launch SolidWorks from Windows Explorer, it will then use the main SLDWORKS.exe which you would also have to set to Windows 8 compatibility mode. This programme is normally located here: -




C:\Program Files\SOLIDWORKS Corp\SOLIDWORKS

Sub HorizontalScrollFullLeft()
