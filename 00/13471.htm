Synchronization Rule for Azure AD connect "What if .local objects - Microsoft Community

pasha... say:

Synchronization Rule for Azure AD connect "What if .local objects should't be synched?

Hello guys!

Recently, i was working on a Azure AD connect issue where azure AD connect was setup in a way to sync all AD objects to Azure AD. Having deep dive in, customer requirement was" I don't want my .local users to be moved over?"

Humm!!! really head scratching for many professionals. Let me help you out!!

You can check check domain partitions to exclude any container, if you want any container not to sync but here you need to pick .local objects from the whole domain.

So for all users who want their .local objects not to sync to azure AD I created a Inbound synchronization rule.Let me show you how you can do this.

1. Open Synchronization Rule Editor on your Directory synchronization server.

2.Here is a snap shot of Rule editor

3.Click Add new Rule tab on right hand side of the window

4.Give any Name to this rule.Right down any description if you want>Click Transformation drop down menu and select your on-premise AD connector as shown below.Select "User" for Connected system Object Type. Select Metaverse Object Type as "Person" Select Link type as "Join" and also select give the Precedence 50 or any other precedence order.Left other options as blank and click Next .

5.

Add Scoping Filters as shown below.

6. On Add Transformation page for Flow Type, select Constart, for Target Attribute ,select CloudFiltered and give source value as "True" as shown below.Click Save and Exit.

7. Now your Rule is saved and the last task is to run Azure AD Full Sync Manually to check if rule is working.

8. Now run miisclient.exe as Administrator  from the location shown below.

9.

Click Connectors on Synchronization Service Manager>Click Actions at the top ,Click Run and  select -AAD connector

Select Full Synchronization and Run the connector.

10. click Operations tab to see if the Synchronization is successful, you may also check event IDs 653,654 for full synchronization and 904 for import export.

Now go to office 365 portal and under user tab, see if .local objects are gone.

Tip:-If you have installed Azure AD connect and rule editor is not pulling default rules for you, please restart your server machine. It happens when you remove your old Azure AD or Dir Sync and installed a new one over it without deleting some reaming files and folders.

I hope this will help you to create a synchronization rule in azure AD Connect.

Alex St.

hansv mvp say:

I'd simply omit the fields from the WHERE clause if you don't need to impose a restriction on them. I often use code like this:

 

    Dim strWhere As String
    If Not IsNull(Me.txtCountry) Then
        strWhere = " AND Country='" & Me.txtCountry & "'"
    End If
    If Not IsNull(Me.txtStartYear) Then
        strWhere = strWhere & " AND StartYear=" & Me.txtStartYear
    End If
    ...
    If strWhere <> "" Then
        ' Remove the first " AND "

jan karel pieterse say:

With three helper columns (B, C and D):

In B1:

=SUBSTITUTE(A1,"(","|",LEN(A1)-LEN(SUBSTITUTE(A1,"(","")))

(replaces last opening bracket with pipe character)

In C1:

=TRIM(SUBSTITUTE(MID(B1,FIND("|",B1)+1,LEN(B1)),")",""))

(finds substring starting from pipe character, removes closing parentheses)

In D1:

=LEFT(B1,FIND("|",B1)-1)

Gets string up to pipe character

Copy B:D down to match data.
