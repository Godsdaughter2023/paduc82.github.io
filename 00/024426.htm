sweetsammy say:

john mcghie say:

There are three ways to do this :-)

The simplest is to use bookmarks and cross-references.  So:  You create the text for "Seller" then mark it with the bookmark (for example 'sellerName', no spaces in bookmark names).

Then in each other place in the document, you add a cross-reference to the bookmark "sellerName". Set the cross reference to just the text of the bookmark.

Now, to put in a new seller, you CAREFULLY type in the new seller's name within your original bookmark.  Turn on the display of bookmarks, so you can see what you are doing: if you damage the bookmark, all **** breaks loose.

When you have updated each of the bookmarks, you select the whole document and Update Fields (could be Command + Option + Shift + U or F9, depending on how you have customised your keyboard. All the cross references will update to their new names.

This method is quick and simple if you are the only user using the document and you know not to break the bookmarks.

A more robust method that will sometimes survive the hacking and chopping of unskilled users is to use Custom Document Properties and Fields. In Word, go to File>Properties and click the Custom tab.

You will find quite a list of Custom document properties that have been created, but they have no content (no "Value").  You can add a value to the ones you like, or you can insert properties with your own names and add values for those. It doesn't matter which you do, you can use some of the custom ones and add some, if you like.

To handle a problem you will get later, I would insert two properties for each name, "sellerNameFull" and "sellerNameShort". You can later choose to insert the full company name or the short name, as appropriate to the place in the text.

When you are adding the values, there is a checkbox to the right that says "Link to Content".  DO NOT check that!! It gives you the ability to link the custom document property to the contents of a defined bookmark.  But for unskilled users, you will get the same problem you have with any other bookmark: the users will type over it and break it. A broken bookmark can suddenly call in three or four pages of text, and you have no idea where it came from.  You do not want to go there...

Once you have set up the Custom Document Properties to contain the names you want, you then go through the document and add fields to insert the content of each property in the appropriate place.

To add the fields, go to Insert>Field.  In the left column, choose "All".  In the right column, choose "DocProperty".  When you choose DocProperty, click Options. You will see the Custom Document Properties that you have added in the list that appears.  Select one (only one!) and OK.

Go through your document and insert the DocProperty fields for each custom document property in the appropriate places.

To change a name, all you need to do is to go to File>Properties>Custom, find your custom property, change its text, and click Add.  Don't forget to click Add, or there will be no change :-)  Then: Select all the text and Update Fields.

You CAN set Word>Preferences>Print to "Update fields", which will perform the update each time you print. But that's extremely risky these days: many documents are emailed, not printed, and they would arrive with the previous customer's name displayed.  Not good for business!

Warn your users to set Word>Preferences>View>Field Shading to "Always", so they can SEE where the fields are and not type over them (which produces very peculiar results...). And adopt a rule where any user that breaks the fields in a document gets to stay after hours and fix the entire document themselves! If you do not adhere to this rule, your fields will last about a week.

While playing around with the fields, you will see there is also a mechanism in there named "DocVariable".  This works exactly the same way as custom document properties, except that it requires a VBA Macro to set the values of the variables.  Which makes the variables difficult to change inadvertently.  But the users will still cheerfully type over your fields, so generally these are not worth the trouble, unless you are building a fully-automated document.

Hope this helps

In your case you would setup a unique Style Names for the Buyer and Seller names that you want propagated throughout the document. These unique styles are only assigned to the first place you enter the names and you have to them keep pure from use with other text. One way to do this is to isolate the names in table cells and assign the "BuyersName" and "SellersName" style to each of those data entry cells.

In the example I've included below I turned the field toggle on so that it would show the coding of the field. BTW - for any PC enthusiast reading this the keyboard sequence on  a PC is CTRL+F9... on a Mac you must add the CMD key into the sequence.

suzanne s. barnhill say:

If you are creating a TOC, then what you are dealing with is not an index but a table of contents. How you proceed will depend on whether or not the TOC is based on TC fields or headings or outline levels. If you just want to omit the entries created as TC fields, it suffices to omit "Table entry fields." You can do that through the Table of Contents Options when you are creating the TOC. After it is created, you can edit the existing TOC field:

  1. Press Alt+F9 to display the field.
  2. Delete the \f switch.
  3. Press Alt+F9 to display the TOC, then F9 to update it.

VBA Name Columns based on names in first row - Microsoft Community

viktoria kiseleva say:

VBA Name Columns based on names in first row

I'm exporting data into excel with columns that have names in row A, i'd like to create a macro that will take the names in that row and actually name the whole column according to whatever it is. I'm thinking probably a loop?
mike h.. say:

Hi,

Try this. It creates a named range for each column using the name in row 1. The size of the range in each column is the number of rows with data in.

Sub Name_Ranges()
Dim LastColumn As Long, LastRow As Long
Dim x As Long, Rng As Range
LastColumn = Cells.Find(What:="*", After:=[A1], _
SearchOrder:=xlByColumns, _
SearchDirection:=xlPrevious).Column
For x = 1 To LastColumn
LastRow = Cells(Cells.Rows.Count, x).End(xlUp).Row
Set Rng = Cells(1, x).Resize(LastRow)
ActiveWorkbook.Names.Add Name:=Cells(1, x), RefersTo:=Rng
Set Rng = Nothing
Next

jay freedman say:

I don't know why Word would change existing table captions to figure captions -- I don't know of anything in the program that would do that if you wanted to. Changing it back is fairly easy, although tedious.

First, press Alt+F9 to display all the field codes in the document instead of their results. Each table caption should look like this:

Table { SEQ Table \* ARABIC } <caption text>

and each figure caption should look like this:

Figure { SEQ Figure \* ARABIC } <caption text>

The first word, Table or Figure, is plain text, and the field in brackets supplies the number within a sequence indicated by the word after SEQ. It's possible that only the word inside the SEQ field was changed, giving the incorrect hybrid

Table { SEQ Figure \* ARABIC } <caption text>

Once the field codes are displayed, you can use the Replace dialog to fix the incorrect captions:

  • Open the Replace dialog and click the More button to expand it.
  • In the Find What box, type Figure.
  • With the cursor still in the Find What box, click the Format button at the bottom left and click Style in the menu.
  • In the Find Style dialog box, click on the Caption style and click OK.
  • In the Replace With box, type Table.
  • Click the Find Next button as many times as necessary to get to a caption that's labeled Figure but belongs to a table. Click the Replace button. The word will change and the cursor will jump to the next occurrence.
  • Repeat the Replace (and Find Next when necessary to jump past real figures) until you've restored all the table captions to the correct format.
  • Press Alt+F9 to collapse all the field codes. Click Ctrl+A to select all of the document and press F9 to update the fields. It may take a few presses of F9 to update both the caption fields and the cross-references to them.

    macropod say:

    Your first post described hyperlinked content showin as:
    C:\Users\nsmr\desktop\Book\Book the New\8-30-16\BOOK PAGE BACKGROUNDS LIBBY\four upside d.jpg

    I see no evidence of that in your screenshots. Indeed, they indicate Word is behaving correctly. In that situation, the only reason for a hyperlink to display as you describe is because that's what you have for its display text.

    Of course, if the display changes after closing & reopening the document, there are two main possibilities:

    1. A faulty Office installation; and/or

    2. A corrupt Word document.

    For (1), you could try repairing the Office installation (via Start > Windows Control Panel > Programs > Programs & Features > Microsoft Office (version) > Change > Repair).

    For (2), corrupt documents can often be 'repaired' by inserting a new, empty, paragraph at the very end, copying everything except that new paragraph to a new document based on the same template (headers & footers may need to be copied separately), closing the old document and saving the new one over it.

    As for the Table of Contents, I have already advised the normal way of doing that. Your method is quite inflexible; it won't auto-update to reflect the insertion of new headings or the deletion of existing ones and, moreover, it won't reflect any edits to an existing heading. All of that is handled seamlessly if you use Word's Table of Contents tools. The only thing that might justify doing things differently is your apparent preference for having the page numbers on the left.
