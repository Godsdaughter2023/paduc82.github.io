Can WDO update over wi-fi? - Microsoft Community

markfilipak say:

Can WDO update over wi-fi?

When I boot to Windows Defender Offline, it can't update. I have ONLY wi-fi. How can I get WDO to update over wi-fi? Thanks in advance. - Mark.

SOLVED.
I'm adding this to help others with similar problems. I had 2 separate issues: 1, Windows Defender Offline (WDO) couldn't update in my laptop, and 2, WDO couldn't update when running in a virtual machine on that laptop. I'll address each solution separately.

Solution 1: WDO couldn't update in my laptop: I only have Wi-Fi. It uses an Atheros AR5001X Mini PCI Wireless Network Adapter. WDO apparently doesn't support that device. However, I can boot the laptop from USB, so that's what I do.

Solution 2: WDO couldn't update when running in a virtual machine: The virtual machine can only boot from HD & CD, not from USB. I chose to boot from an ISO image of WDO attached as a virtual CD because then, I don't have to burn a physical CD. I ran into two problems: 1, The particular version of WDO that I started with had a bug that left it 'blind' to its own virus definitions (which were current) and to therefore go to the net (which failed), and 2, The virtual network adapter in the virtual machine, a PCnet-FAST III (Am79C973-based) NIC, also is not supported by WDO - it's the gold standard NIC for the mid-2000s, so go figure. What I did was to upgrade the virtual machine to the Intel PRO/1000 NIC (for which WDO does have a driver) and attempt to run WDO. That didn't work because, though WDO was able to update using the Intel NIC, it ran out of (virtual) memory while scanning. I couldn't increase the amount of memory I gave to the virtual machine further because my laptop is thin on memory (1 GB) and WDO apparently needs almost all of it (leaving my real Windows host with none - impossible!). At that point (a week later), I downloaded the latest WDO ISO image and discovered that it was no longer 'blind' to its own virus definitions. Thus, I was able to download the ISO image, boot the virtual machine from it, and run the scan on the virtual machine's hard disk. Conclusion: If the virus definitions are on the CD (real or ISO image), then WDO uses them right off the CD. Otherwise, it downloads them into memory. It's the downloading-to-memory part that caused my virtual machine to run out of memory. A better solution for WDO would be for it to download virus definitions to the hard disk it's going to scan, but apparently the developers haven't thought of that.

Special Thanks to Steve Boots for his wonderful assistance.
