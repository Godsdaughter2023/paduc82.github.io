Your description suggests you used 'mmm' for the month paratmeter. Lower-case 'm's are for minutes. Use upper-case 'M's instead. For example:

{MERGEFIELD MyTime \@ "DD MMM YYYY"}

or:

{MERGEFIELD MyTime \@ "dd MMM yyyy"}

changing the format in the data source is of no consequence.

andreas killer say:

the Major and Minor arguments in the AddFromGuid command line? What do they do and what do they refer to?

Sorry, I thought my screenshot would be self-explanatory.

AddFromGuid searches the registry for the given GUID, e.g. {000204EF-0000-0000-C000-000000000046}

If the GUID is found, it looks further for existing version numbers. If there is just one, the available reference is added.

Otherwise it compares the Major and Minor numbers:

Excel 2016 uses this reference:

Visual Basic For Applications
{000204EF-0000-0000-C000-000000000046} 4.2
C:\Program Files (x86)\Common Files\Microsoft Shared\VBA\VBA7.1\VBE7.DLL

Excel 2010 uses this reference:

Visual Basic For Applications
{000204EF-0000-0000-C000-000000000046} 4.1
C:\Program Files (x86)\Common Files\Microsoft Shared\VBA7\VBE7.DLL

So in this example the GUID
  {000204EF-0000-0000-C000-000000000046}

refers always to
  Visual Basic For Applications
but which DLL is used depends on the (Major.Minor) version number.
