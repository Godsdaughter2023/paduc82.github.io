Perfect - that's exactly what you needed to do.

 

Would Microsoft have been able to do another update to remove the folder? Do I need to be concerned about other folders like this?

Long, involved story. Here's the condensed version:

Windows Update (AU) typically locates such "installer" files in

         C:\WINDOWS\SoftwareDistribution\Download <=this folder

Over time, the contents of that folder will "age out" and are deleted automagically by Windows routine maintenance.

But sometimes (e.g., "buggy" .NET Framework updates) and always when you resort to installing an update manually, they'll be placed on the root of the boot drive (i.e., C: ). Windows will not delete them so you just delete them manually.

One final comment: AU will always download "installer" files to the drive with the most available free space. If you leave an external storage drive (including a flash drive or SD Card) connected to the computer 24/7, AU might resort to saving them on the external drive. But when AU gets around to installing the update, it "expects" to find the installers where it put them - so if you've since disconnected the external drive, AU will basically "throw a hissy fit" & the install will fail. Then you'll have a major problem on your hands!

The moral of the story is: Don't leave any external drive connected to the computer 24/7.

jeeped say:

There is a registry tweak (hack?) that will hide drives from Computer. If you are comfortable with viewing/editing your registry, try and locate the DWORD registry key at:

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer\NoDrives

If the key is not there, you're done. This does NOT apply.

If it is there, you can delete it or set it to 0 to show all drives. If you have a legacy floppy or somesuch that you still want to have hidden, add the corresponding decimal values from this table to hide certain dirves and set the NoDrives DWORD value to the sum.

Drive
Letter Decimal Hex
 A        1     1
 B        2     2 
 C        4     4
 D        8     8
 E       16    10
 F       32    20
 G       64    40
 H      128    80

By the same token, if the NoDrives key exists, subtracting 4 from its current value should leave C drive unhidden.
