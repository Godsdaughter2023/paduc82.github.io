macropod say:

You could use an ordinary Find/Replace with, for example:

Find = ^w

Replace = ^32^32^32

to insert three spaces between all words, and before/after other words that have a space or tab before/after them. Do note that this will replace tabs as well.

Alternatively, you could use a wildcard Find/Replace with, for example:

Find = ([A-Za-z]>)([ ])

Replace = \1\2\2\2

to insert three spaces after all words, except those followed by a punctuation mark, and:

Find = ([A-Za-z]>)([.,:;\!])

Replace = \1^32^32^32\2

to insert the spaces between the words and their punctuation marks. To insert the spaces after the punctuation marks (except before paragraph breaks, etc.), you might use:

Find = ([A-Za-z]>)([,.] )

Replace = \1\2^32^32

macropod say:
Assuming the 'Article 1 - xxx', 'Article 2 - xxx', etc. use Word 'Heading 1' Style, simply insert a Table of Contents field referencing that Style. You can do that via Insert|Quick Parts|Field>TOC>Table of Contents, then set 'show levels' to 1, for you could press Ctrl-F9 wherever you want the Table of Contents to appear in the body of the document. Pressing Ctrl-F9 creates a pair of field braces (i.e. { }) that you would fill in with 'TOC \o "1-1" \h \z', so you end up with {TOC \o "1-1" \h \z}. Selecting the field and pressing F9 will update your Table of Contents, to display the various headings.

bwzzz say: When I try making Article 1, Article 2 "word heading 1 style", I lose my list numbering, so that the numbering under Article 2 become 1.3, 1.4, etc.  I need to maintain the list numbering so that everything under an article has the article's number, i.e., all items under Article 30 are 30.1 xxx, 30.2 xxx and so on.

suzanne s. barnhill say:

Start over. First read http://shaunakelly.com/word/numbering/numbering20072010.html for general principles. The important part is that each numbering level must be linked to a specific style. They don't have to be heading styles, but if you want them to be included in a TOC, using the built-in heading styles is the easiest way to start.

To construct your list:

  1. Apply Heading 1 to the first Article title.
  2. In the Paragraph group on the Home tab, click the button for Multilevel List.
  3. From the List Library, select the pane that shows 1 Heading 1, 1.1 Heading 2, 1.1.1 Heading 3, etc. This will add the number 1 to your article title.
  4. Click the Multilevel List button again and choose Define New Multilevel List.
  5. In the Define New Multilevel List dialog, Level 1 will be selected. Click More>> to expand the dialog. You'll see that Level 1 is selected and is linked to Heading 1.
  6. In the "Enter formatting for number" box, type "Article " before the number 1.
  7. Since it appears you want your subsequent levels indented, select Level 2 and adjust the indents as desired. Click OK when you're done. You can do this as often as you need to to fine-tune the formatting, but it's safest always to start with the insertion point in the first Level 1 heading.
  8. Apply Heading 2, Heading 3, etc., as needed for your lower levels.

I posted this in another thread but this helped fix it for me.

I ran into this issue with my Win7 Home Premium 64-bit system when I tried to Upgrade my MSE. It failed and from that point I couldn't Upgrade, uninstall, or reinstall the program with any of these methods or even using the One Care Removal Tool that I have seen suggested in other threads. It just kept asking for a location for a valid install package no matter where I pointed it. At this point I decided to start doing some digging and I found that in the install folder for MSE there is a backup folder that has the files that the uninstaller is looking for, so when I next ran the uninstaller and it prompted me for a location for the files I pointed it towards "C:\Program Files\Microsoft Security Essentials\Backup" and either msse.msi or mp_ambits.msi depending on which file the dialog box was looking for. Guess what...it worked perfectly! Hopefully this might help anyone else that is having this problem as well.
