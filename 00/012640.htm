rob koch say:

Generally, I'd assume that the reason you don't receive a malware alert is because there isn't any.

That may sound facetious, but it was intentional, since in many cases this is actually the truth.

Beyond the fact that you can't find an executable named G.exe to examine, the Internet is filled with invalid information regarding everything, especially items believed to be malware, whether they truly are or not.

Doing my own much more careful search and assuming only what we truly know, which is that a process named "G" (not necessarily a program executable) is preventing a restart, I found a couple clues in other forum articles elsewhere.  The first contained this tidbit that pointed towards a possible relationship to NVidia's Geforce video driver.

"I googled around a bit and some say its related to Geforce or utorrent? I do have an NVidia card so naturally Geforce comes with it. I also use Bitorrent but I only downloaded some music and shows. No programs or games. "

https://forums.malwarebytes.com/topic/217145-gexe/

Since that thread was dropped and never completed, I then discovered this similar one titled; "G.Exe Preventing Shutdown annoying.. - Malwarebytes Forums" which sounded promising. It included the following quote and reference by the forum helper.

"Whenever any C# application opens its first form that contains a control within a process, a hidden window named "G" is opened, whose class name is "GDI+ Hook Window Class", as shown by the application below:"

What is the hidden GDI+ Hook Window in C# - Stack Overflow

https://stackoverflow.com/questions/46155447/what-is-the-hidden-gdi-hook-window-in-c

Reading through the rest of the responses in that thread, they seemed to have difficulty explaining it until the last entry as follows.

"It is plumbing for gdiplus.dll, it provides synchronization for background threads used by GDI+. The docs for GdiplusStartup() talk about them, exactly what they do is unclear. Since System.Drawing relies on gdiplus.dll to get the job done, you'll always see this window back in a Winforms app. You need to ignore it. – Hans Passant Sep 11 '17 at 13:46"

What this is showing is something we commonly see here when people come in with unexplained issues and immediately assume malware.  This is that the assumption is probably wrong and it's something else entirely.

In this case, it appears to me that this is simply some sort of GDI+ graphical display issue that's causing a delay in the shutdown process (also seen in restarts) that's causing the error message indicating that the process named G is the cause.

Since the issue eventually resolves itself either due to closing correctly or simply timing out, the system eventually proceeds and shuts down on its own, albeit with some added delay.

If it were me, I'd pay attention to what's been running immediately before attempting to shut down, since it's likely some program such as a game or other notoriously badly written app that's not properly closing or even crashed earlier in the session that's causing the issue.  In fact, trying to shut down immediately after a clean startup should prove whether this issue is within the core operating system or due to something that happens or is run during a session.

preciousleb say:

Hi,

We appreciate the information you've provided. To locate the Group Policy Object (GPO) for Windows 10, we suggest that you follow these steps:

  1. Press the Windows logo key + R to open the RUN dialog box.
  2. Type gpmc.msc in the text box.
  3. Click OK or press ENTER.
  4. From the Group Policy Management Console, go to Computer Configuration > Administrative Templates > Windows Components > Data Collection and Preview Builds.
  5. Double-click Allow Telemetry.
  6. In the Options box, select the level that you want to configure, and then click OK.

Let us know how it goes and feel free to get back to us if you need further assistance.
