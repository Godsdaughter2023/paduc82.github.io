(without quotes, and without spaces before or after it) should work.

If you're using a different language version, you must use the local names of MOD and ROW.

If your system uses comma as decimal separator, you probably use semicolon as list separator, so in that case the formula should be

=MOD(ROW();2)=1

For highlighting every other block of 4 rows, you can use the formula

=MOD(ROW()-1,8)<=3

or if you use semi-colon as list separator,

=MOD(ROW()-1;8)<=3

Jim is right: The Default Language spec is stored in the Normal.dotm template. Note the confirmation prompt that I referred to in step #5 of my reply :-)

The reason it didn't "stick" before is that you were attempting to force a replacement of the "Normal.dotm template" of your own which Word will not use. As I explained in the last paragraph of that same reply, Word will ignore a Normal.dotm which it did not  generate.

bob jones aka: cybertaz say:

There is no reason to open the Normal.dotm template to specify a Default Language to be used for the creation of new documents.

When you initially launch the program;

  1. Go to Tools> Language
  2. Select the preferred Language
  3. Confirm that both check boxes are cleared
  4. Click the Default button
  5. Confirm updating the Normal template
  6. Click OK or Close

Each time you launch the program or create a new document the Language you chose should be in use. [Note that setting the Default Language has no impact whatsoever on existing documents.] If there is any problem with that procedure, please provide accurate details including which specific languages are involved.

Also, the user cannot create or "overwrite" a Normal.dotm template. That has to be done by the program as the template contains a great deal more than what a user has access to. The template generated by the program can be updated or customized in a variety of ways, but most of those changes do not require opening or replacing the template file, itself.
