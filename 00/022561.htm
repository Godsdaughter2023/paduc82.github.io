Using make table queries in a split database - Microsoft Community

terry carroll say:

Using make table queries in a split database

I was just wondering in a split database structure what the best practice is in terms of how to handle action based queries such as make table, append, and delete queries.  I have the queries stored in the front end but should I be pointing these action based queries to the back end of the database? 

 

Thanks,

 

Terry

hansv mvp say:

If you have created links in the frontend to the tables in the backend, update queries, append queries and delete queries based on linked tables will act upon the tables in the backend.

 

A make-table query will by default create a table in the frontend. It's possible to make it create a table in the backend, but keep in mind that this may cause problems if the database is used by several people simultaneously. The steps are as follows:

  • In the frontend, activate the Create tab of the ribbon.
  • Click Query Design.
  • Add the tables/queries that will provide the records. (Even though the tables are linked to the backend, you should be able to select them)
  • On the Design tab of the ribbon, click Make Table.
  • By default, the new table will be created in the Current Database, i.e. in the frontend.
  • Click Other Database, then Browse...
  • Locate and select the backend database, then click OK.
  • Specify a name for the new table, then click OK.
  • Now proceed as usual: add fields to the query grid, specify conditions etc.
  • Finally, click Run.
  • You will now have a new table in the backend, but you don't have a link to it in the frontend yet. You have to create the link explicitly, either in the External Data tab of the ribbon, or programmatically, using DoCmd.TransferDatabase with the acLink option.
