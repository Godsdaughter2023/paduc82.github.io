charles kenyon say:

I agree with everything Graham said.

If a global template is not in Word's Startup Folder it will not be automatically loaded upon starting Word. It will be manually loaded (or loaded through some macro in a loaded global). Once loaded, it will remain loaded (active) until you start a new Word session or otherwise unload it.

If you used Word's tools (interface) to make a change to the Ribbon, that change will last until you undo it. If you want ribbon changes to be related only to a certain template (or document) you have to do it through the XML. This is somewhat more tedious than doing it through Word, but is much more powerful and flexible as well. Greg Maxey's page is the best place to start that I know about.

In my previous post I said to make your modification and save it as a template. This could be a Word 2003 template modified with toolbars in Word 2003. In that case, the commands on the custom toolbar would show up under the Add-Ins tab. If you have Word 2003, this is easier than modifying the XML but much more limited.

If you are certain that the time value has no date part, then just multiply by the number.  For example,

   =Int((time * number) * 24) & Format(time * number, ":nn")

rich007a say:

Hi,

I didn't think this would be possible, but try this:

Sub ApplyLabelsToPptChart()
    Dim cht As Chart
    Dim ser As Series
    Dim pnt As Point
    Dim i As Long, lnSerIndx As Long
    Dim wb As Object  'Excel.Workbook
    Dim rng As Object 'Excel.Range
    Dim strMsg As String, strResp As String
   
    With ActiveWindow.Selection
        If .Type = ppSelectionShapes Then
            If .ShapeRange.Count = 1 And .ShapeRange(1).HasChart Then
                Set cht = .ShapeRange(1).Chart
               
                If cht.SeriesCollection.Count < 1 Then
                    MsgBox "No series in selected chart!": Exit Sub
                ElseIf cht.SeriesCollection.Count = 1 Then
                    Set ser = cht.SeriesCollection(1)
                Else
                    strMsg = "Please choose the series for applying Data Labels..."
                    For i = 1 To cht.SeriesCollection.Count
                        Set ser = cht.SeriesCollection(i)
                        strMsg = strMsg & vbCr & "   enter " & i & " for series '" & ser.Name & "'"
                    Next i
UserSelect:
                    strResp = InputBox(strMsg, "Apply Data Labels")
                    If strResp = "" Then Exit Sub 'User Cancelled
                    If Not IsNumeric(strResp) Then GoTo UserSelect
                    lnSerIndx = CLng(strResp)
                    If lnSerIndx > cht.SeriesCollection.Count Then GoTo UserSelect
                    If lnSerIndx <= 0 Then GoTo UserSelect
                    Set ser = cht.SeriesCollection(lnSerIndx)
                End If
               
                On Error Resume Next
                cht.ChartData.Activate
                Set wb = cht.ChartData.Workbook
                On Error GoTo 0
               
                If wb Is Nothing Then
                    MsgBox "Cannot access the chart's data."
                    Exit Sub
                End If
               
                wb.Activate
                On Error Resume Next
                Set rng = wb.Application.InputBox("Select range for Data Labels", Type:=8)
                On Error GoTo 0
                If Not rng Is Nothing Then 'user didn't cancel
                    ser.HasDataLabels = True
                    For i = 1 To ser.Points.Count
                        Set pnt = ser.Points(i)
                        pnt.DataLabel.Text = rng(i).Value
                    Next i
                End If
            Else
                MsgBox "Please select a chart to apply data labels to a series."
            End If
        Else
            MsgBox "Please select a chart to apply data labels to a series."
        End If
    End With
End Sub

An alternative approach would be to check the userID of each user, and match that up with sheets in your code.

I had one workbook that would check the userID and unhide only the sheets each user could edit, and kept the rest .veryhidden

The people with admin rights simply had IDs associated with all sheets.
