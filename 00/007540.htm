zoeyzan_msft6423 say:

Hi Michael,

You can set Address Book Policies to manage the Address Book in different domains only by PowerShell. Here are the steps:

I create the resource list named test-resource, address list named test, global address list named GAL-test, offline address book named OAB-test and Address Book Policy named ABP-test. And custom attribute is “test”.
1.Add a role to a role assignment policy
2.Connect to Exchange Online using remote PowerShell
3.Create the resource list
New-AddressList -name "test-resource" -RecipientFilter {(((RecipientDisplayType -eq 'ConferenceRoomMailbox') -or (RecipientDisplayType -eq 'SyncedConferenceRoomMailbox')) -and (CustomAttribute3 -eq "nullresource"))} -DisplayName "test Room List"
4.Create the address list
New-AddressList -name "test" -RecipientFilter {(RecipientType -eq 'UserMailbox') -and (CustomAttribute2 -eq "test")} –DisplayName “test Address List”
5.Create Global Address List
New-GlobalAddressList -name "GAL-test " -RecipientFilter {(CustomAttribute2 -eq "test")}
6.Create Offline Address Book
New-OfflineAddressBook -name "OAB-test " -AddressLists "GAL_test"
7.Create Address Book Policy
New-AddressBookPolicy -name "ABP-test" -AddressLists "test" -OfflineAddressBook "\OAB-test " -GlobalAddressList "\GAL-test" -RoomList "\ test-resource "
8.Assign the Address Book Policy to users
Get-Mailbox -resultsize unlimited | where {$_.CustomAttribute2 -eq "test"} | Set-Mailbox -AddressBookPolicy "ABP-test"
9.Hide test from address lists
Get-Mailbox -resultsize unlimited | where {$_.CustomAttribute2 -eq "test"} | Set-Mailbox -HiddenFromAddressListsEnabled $true

After doing these steps, login to the portal->People. As the user has been assigned this address book policy, he is unable to see other users who don’t have “test” attribute.

You can create other attributes for users who have the same domain.
 
If you meet any problems, please provide the error messages for troubleshooting.

Thanks,
Zoey

I know that Word is creating a big table to hold all the label data (in my case, 231 labels.)  But somehow it doesn't seem to be able to treat a subset of that table as a "page", despite showing the data as pages and displaying page numbers at the bottom of the Word window.  What I finally did was make a copy of my labels file, edit the Word table, delete all table rows after page 2, then delete all table rows on page 1.  I also got rid of section breaks to eliminate blank pages.  Since I had only page 2 left in this new file, I could finally print the 1 page that I wanted.

How should I have used the Print options to get just 1 page?  Specifying the page number in the Print dialog did not work.

Thanks!
jay freedman say:
The document created by the mail merge has a section break at the end of each page, and the page number restarts at 1 in each section. As a result, you had eight "page 1"s. To be able to print just the second of those pages, you have to tell the Print dialog the page and section number in the form p1s2. For a bit more about this, read http://www.word.mvps.org/FAQs/Formatting/PrintMultipleSections.htm.

The other way to do this is during the creation of the mail merge. Select the specific records from the data source, and print the result.
