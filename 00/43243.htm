frederik long say:

This is precisely the way things are designed: Accounts with administrative privileges (=members of the Administrator group) can view any folder unless you explicitly knock out this group. Have a look at permissions for the Administrator's profile folder. It will include the "Administrators" group. If you now knock out this group and give full access to "Administrator" only (note the missing trailing "s" - Administrator is an account, Administrators is a group!) then other accounts with administrative privileges will be kept out of this folder.

It makes no difference whether the Administrator account is active or not. Hiding or marking files as read-only makes no difference either.

If this makes no sense then click Start / Help and have a look at the various topics than come up when you look for help on "permissions".

frederik long say:

Please consider these points:

  • I never suggested you should remove the security group called "Administrators". I suggested you should remove it from the set of users+groups that have access to the profile folder c:\users\Administrator. This is a completely different thing.
  • The profile folder c:\Users\Administrator (which you call "Home Folder") was created automatically when you first logged on to Windows. It was not hidden before - it never existed until the first logon even. It will stay there until you delete it manually. Activating or deactivating the Administrator account has no effect on the existence or visibility of this folder.
  • You can completely erase an account in two ways:
    a) By manually deleting both the account and its profile folder.
    b) By running the utility delprof.exe
