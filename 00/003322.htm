Thank you for letting us know. You need to access security tab and change the owner. Please follow the steps provided by Anoop P K on this thread. If the same issue persist, please follow the steps below:

Method 1: Uninstall driver of USB

  1. Right click on Start and choose Device Manager.
  2. Go to View tab and click on Show hidden devices.
  3. Locate your USB flash drive and right click on it. Choose Uninstall and Ok.
  4. Restart computer.

Method 2: Use Command Prompt to change security permissions.

  1. Type in Command Prompt in Search bar.
  2. Once it comes up, right click on it and choose Ran as administrator.
  3. On Command Prompt, type in takeown /F <folder path> /R /D Y then hit Enter.
  4. After that, type in C:\>icacls <folder path> /grant administrators:F /T and hit Enter.
  5. Please make sure to replace <folder path> with the correct path of the folder that you can't access.
  6. Access the folder again.

Let us know if this resolves your concern.

Usually, we suggest customers stop DirSync in forest A and migrate users to forest B. During the progress, users in the cloud are still able to access Office 365 resources.

If migrated users in the forest B are the same users (same ObjectGuids), please use hard match to match accounts in the cloud.
