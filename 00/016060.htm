ken sheridan say: It depends how you are restricting FormB's recordset to the currently selected record in FormA.  If you are doing so via a parameter in its RecordSource which references the KeyField control in FormA, then you could include a chkShowAll checkbox in FormA, and reference this in FormB's query, e.g.

SELECT *
FROM YourTable
WHERE KeyField = Forms!FormA!KeyField
OR Forms!FormA!chkShowAll  = TRUE;

If, on the other hand, you are filtering FormB to the current record by means of the WhereCondition argument of the OpenForm method, then you could again include a chkShowAll checkbox in FormA, and use code like this to open FormB:

Dim strCriteria As String

If Nz(Me.chkShowAll,False)  = False Then
    strCriteria = "KeyField = " & Nz(Me.KeyField,0)
End If

DoCmd.OpenForm "FormB", WhereCondition:=strCriteria

ken sheridan say:
I do not need any record restrictions on formB's recordset. I just need to open formB to show all the records displayed in the subform which I call formA.

In that case you don't need a checkbox at all.  Simply put the button in the parent form and open FormB filtered on the value in the relevant foreign key column equalling the value of the parent form's recordset's key, which will be whatever column is cited as the subform control's LinkMasterFields property.  So if, as an example, we assume that the parent form is based on an Orders table with a numeric primary key OrderID, and the FormA subform and FormB are both based on an OrderDetails table with a foreign key OrderID then the code would be like this

Dim strCriteria As String

strCriteria = "[OrderID] = " & Me.[OrderID]

If Not IsNull(Me.OrderID) Then
    DoCmd.OpenForm "FormB", WhereCondition:=strCriteria
End If

Testing the OrderID for Null caters for the user inadvertently pressing the button while at an empty new record in FormA.  

scottgem say:

First, you need to understand that a form does not hold records it only displays them. It displays them by using a Recordsource. Further a subform has an additional filter beyond the recordsource that filters for the linking filed between the subform and the mainform. 

So, if you want to open FormB so it displays the same records as shown in the subform (FormA), you need to give FormB the same (or similiar) Recordsource as FormA. In addition, you need to open FormB using the WHERE clause of the OpenForm method to pass that filter on. 

To be specific in our advice we need to know what the Recordsource of FormA is and what the linking fields are between the mainform and the subform. The "Edit All Records" button needs only one line of code. 

P.S. I'm going to assume that your forms are not named FormA and FormB nor is your PK named keyfield. This would be very poor naming habits. Object names should be descriptive of what the object contains or does.
