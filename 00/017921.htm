john w. vinson say:

The Control Source of a textbox needs to be a single value of a single field. It can't be a Query: a Query could potentially have 255 fields and millions of rows, and Access can't tell.

Rather than using the Totals query, just use the builtin DCount() function as the control source:

=DCount("*", "[Actions Due Query]")

You may also want to use Date() - today's date - instead of Now(), the current date and time accurate to a few microseconds.

Formatting lost between Word for Mac 2011 & Word - Microsoft Community

thortellefsen say:

Formatting lost between Word for Mac 2011 & Word 2007(windows)

I am putting together my thesis both at home and work. I formatted the entire thesis at home using word for Mac 2011. When I went to work and opened up the thesis in word 2007 (windows), the formatting was all different. I lost page numbers at each section break and it appears my double spacing was different. The toal pages jumped from 89 to 138.

What is going on?

john mcghie say:
This can be a difficult problem to hunt down.

If you do ALL of your formatting using Styles, and do not do any manual overriding of styles, and do not use any hard page breaks or blank paragraphs, it won't happen.

When you transfer a document from one platform to the next, Word will recompute all the dimensions to fit the new environment.  The biggest influence is the printer driver.  It supplies all the measurements that Word uses.  So when you change to a different default printer, Word re-lays the document to make it display/print the way it will on the new printer.

Fonts also come into play: the way Windows draws a character is subtly different from the way Mac OS X does it.

Microsoft supplies a set of fonts with Office 2011 that are specifically engineered to match the same-named fonts on Windows as closely as possible.  One of the secrets to successful cross-platform work is to use the Microsoft fonts on both sides of the fence.

Word stores pointers to its formatting in each paragraph mark.  There's no such thing as a "blank line" in Word: each paragraph is a container containing up to 1,200 measurements.  One of the secrets to getting documents to be stable across platforms is to avoid all "blank paragraphs".  If you need spacing, add it to your paragraph styles, don't hit Enter twice between paragraphs.

For example: if you are using Body Text style for your normal text (and you should be) add 10 pts "Space below" to Body Text, and avoid hitting any blank lines.  You will get your spacing, but it will compensate correctly across platform.

Same with Page Breaks.  A Page Break doesn't contain any spacing, but the size of the print image will vary between one printer and the next.  If you avoid hard page breaks, Word will paginate the document automatically for you.  That's what it's designed to do.  Use the paragraph properties "Keep lines together" and "Keep with next" to tell Word where you do NOT want a page break, then let it lay the document out for you.

Format with styles, and only with styles, to make a document truly cross-platform.  Using directly-applied formatting is not only very laborious, but it is susceptible to variations in the underlying properties of the styles on each platform.  If you have set the styles in the document, that won't happen to you.

I've just skimmed the surface here: have a think about it, then come back with your further questions and we can talk you through how to get things flowing properly.

Unfortunately most of the articles I've seen on your question deal with Word rather than Excel.

This article provided the examples above and shows how to do some of the cleanup with pictures: http://lsntap.org/sites/all/files/metadata.pdf

Using the Document inspector is the obvious first step. 

Here are are few more ways you can look for metadata:

  • Enable “Track Changes”
  • View Comments
  • File –Properties and look behind the tabs
  • Text editors, "Unzip" the x-file format
  • Save as HTML and view source
  • Third party meta data viewing software

Did you know that the x-file formats introduced in 2007 are actually ZIP compressed files?  All you have to do to "unzip" them is add the .zip suffix after the existing file extension (xlsx, docx ... ).  Then in File Explorer, you can look at the underlying file structure.  Using this technique makes it easier to check the image files for metadata (see next item).

PS: make sure to do this on a copy of the file, not your "original"!

If you have any inserted images in the file, they may contain meta data you want to clean up:

https://davescomputertips.com/__trashed-2/

This PPT article talks about meta data in inserted objects:

https://support.microsoft.com/en-us/help/826825/how-to-minimize-metadata-in-powerpoint-presentations

This looks like a useful script

http://windowsir.blogspot.ca/2007/05/ppt-metadata.html
