You didn't mention your operating system. In Windows 7 (and possibly Vista and W8), each .eml file will have the salient message properties (From:, To:, Subject:, Date:) written to the file itself. To see the Date of the message, then, arrange the Explorer window to show this field in Details view.

Open the folder containing your .eml files. On the command bar, click Include in library and select Create new library. Windows will create a library including one location - the folder which contains your messages. Click on this folder to see the folders it contains.

On the View menu, click Details. Then right-click on a column heading and select More... Select at least Subject, From and Date. Deselect Date modified and Type and possibly Size. Click on column headings to sort, click and drag to rearrange the order of columns. Click the icon next to the Help question mark on the command bar to enable/disable the preview pane. You should be able to read messages in the preview pane, but pictures may be disabled as a security measure.

You should now have a view of your mail messages that approximates to that in Windows Live Mail. This same layout will be applied to all folders in the library, which is one of the (few) useful things about libraries. In normal circumstances, you would also be able to right-click on a message and invoke the Windows Live Mail functions Open, Forward, Reply or Reply all. This may or may not work for you.

 

You should be able to use the search box to find particular messages. Use the attribute:value syntax to find particular messages. For example, from:john should find all messages from anyone called John, subject:tennis any message with tennis in the subject, and christmas should find all messages with Christmas in the subject or body of the message. (There's a lot of should in there, because whether these work for you depends on the hundreds of settings for search and indexing.)

Hi Bret,

The error Windows can not access the specified device, path, or file. You may not have the appropriate permission to access the item can occur if you do not have permissions to the file or the file location, or the file is on a location that is not currently accessible, or that the file or shortcut is corrupt.

As our first recommendation, kindly check the permission of the file or folder. Permissions are rules associated with the files that determine if you can access the file. To check the permissions of the file or folder, kindly follow the steps below:

  1. Right-click the file or folder, and then select Properties.
  2. On the Security tab, under Group or user names, select the user name that you are using to see the permissions.
  3. Select Edit, then select to check the check boxes for the permissions that you need, and then select OK.

If setting the permissions doesn't help, it could be that the file is blocked by Windows. There are some cases when some files are blocked by Windows.
