stefan blom say:

There is no way to set up a default number format, unless you modify a paragraph style, as Doug suggested. You can use the built-in List Number style as a starting point. Apply it to a text paragraph. Click to select the actual paragraph number and then right-click; on the context menu, click Adjust List Indents. Set the indents the way you want them.

Keep in mind that "Number position" is the position of the list number. "Text indent" defines the position of the second and subsequent text lines in the paragraph. "Add tab stop at" determines where the text starts following the number on the first line of text. All measurements are from the left margin, meaning that if you want a hanging indent, you want the value for "Add tab stop at" to be identical to that of "Text indent" (and they should both be larger than "Number position").

After you have specified the new format, update the style to match selection via the Styles pane.

You may also want to copy the style to your template, which you can do by selecting "New documents based on this template" in the Modify Style dialog box.

suzanne s. barnhill say:

As I described, Update to Match Selection has uses beyond just updating the currently applied style (it can update another style to match the current selection). It requires action by the user, whereas "Automatically update" automatically (without intervention by the user) modifies the style every time the user changes the formatting of a paragraph in that style.

As Stefan mentioned, "Automatically update" is convenient for the TOC styles because they're otherwise rather difficult to update. Because they're hyperlinked by default, the style that is displayed is Hyperlink rather than TOC 1, TOC 2, etc. But in my experience, changes to paragraph formatting are more likely to be propagated than changes in font formatting. Sometimes it is necessary to get to the Modify Style dialog anyway.

I've just tested, and apparently just having the insertion point in a paragraph counts as a "selection" for the purpose of Update to Match Selection.

    What works is the following...

    Paste the following lines of code in a notepad file, save it as say notif.ps1 in a scratch folder.

    Start a PS or Command prompt set the current directory to the scratch folder.

    $toasttitle = "Refresh AC: " + [DateTime]::Now.ToShortTimeString()

    $toastXml.GetElementsByTagName("text").AppendChild($toastXml.CreateTextNode >$null($toasttitle))
    $imElem = $toastXml.getElementsByTagName("image")

    $xmldoc.LoadXml($toastXml.OuterXml)
    $toast = [Windows.UI.Notifications.ToastNotification]::new($xmldoc)
    # show the toast for 30 sec. Change the default (5sec.) in Setting Ease of access Other options to a value > or = than this!
    $toast.ExpirationTime = [DateTimeOffset]::Now.AddSeconds(5)

      ---

      6 Microsoft support agents were not able to tell me the solution. Maybe cloud-wisdom is better !?

      john korchok say:

      Normally, PowerPoint for Windows will automatically adjust the number format of charts depending on the language set in the operating system. Unfortunately, Windows 10 Settings panel does not appear to fix this problem. Instead you have to access the older Control Panel by searching for it. But how well this works can also depend on the format in which the chart was pasted. In Windows, please try this:

      1. Close the presentation and PowerPoint.
      2. Click on the Start button, then click in the Search field. Type control panel, then choose the Control Panel that displays under Programs.
      3. Choose Clock and Region.
      4. Choose Change date, time or number formats.
      5. Under Format, choose German (Germany). Click on OK.
      6. Re-open PowerPoint and the presentation. If the number format has not switched to European numbering formatting, please select all and use Review>Language>Set Proofing Language to set the language of the slide to German. You may have to close and reopen the presentation to see the correct number format.

      There are so many factors controlling number format display, it is sometimes necessary to spend a lot of time playing with settings to find a solution!
