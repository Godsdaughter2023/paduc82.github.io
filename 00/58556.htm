I've seen similar anomalities if a game uses OpenGL and the graphics cards driver doesn't support it well (known issue on older ATI/AMD cards). If you can exclude faulty or overheated hardware as the sources, you can always try fiddling a bit around with the game's or the driver's configuration settings.

For instance, the "seeing through walls" surprise is a clipping problem (wrong calculation of the collisions = the edges where the player can't pass and isn't supposed to see the other side). When caused by hardware, it should only appear at random instances or when too much must be calculated at the same time. When it's appearing continuously, I would rather suspect any software misconfiguration.
