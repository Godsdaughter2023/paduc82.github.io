ken sheridan say: If the subform is in single form view then you simply need to reference the current CustomerStatus control in the form to correlate the combo box.  So the RowSource property would be:

SELECT CustomerID, CustomerName
FROM Customers
WHERE CusomerStatus = Form!CustomerStatus
ORDER BY CustomerName;

In the subform's Current event procedure requery the combo box, e.g.

Me.cboCustomer.Requery

If the CustomerStatus control is updateable in the form also requery the combo box in the CustomerStatus control's AfterUpdate event procedure with:

Me.cboCustomer = Null
Me.cboCustomer.Requery

If the subform is in continuous forms view, however, it is not so straightforward.  In rows with a different status to the current row the combo box would appear blank.  No data would be lost, merely hidden.  To avoid this you would need to use a 'hybrid' control in which a read-only text box is superimposed on a combo box to give the appearance of a single combo box control.  The RowSource of the combo box would then be:

SELECT CustomerID, CustomerName
FROM Customers
WHERE CustomerStatus = Form!CustomerStatus
OR CustomerID = Form!CustomerID
ORDER BY CustomerName;

The form should be bound to a query which joins the referencing table into which data is being entered to the referenced Customers table, and in the latter situation above, the read-only text box would be bound to the CustomerName column from the Customers table.

You'll find an example using such a 'hybrid' control in ActiveEmployees.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

Finally sorted it... (thanks for all the help)

 

Private Sub Connect_Click()

    Dim FileName As String
    Dim Line1 As String
    Dim Line2 As String
    Dim Line3 As String
    Dim Line4 As String
    Dim Connect As String
    Dim Message As String
    Dim Whole As String

       
    ' Details
    FileName = "C:\Program Files\SIMS\SIMS .net\Connect.ini"
    Line1 = Forms![SchoolData]![Line1]
    Line2 = Forms![SchoolData]![Line2]
    Line3 = Forms![SchoolData]![Line3]
    Line4 = Forms![SchoolData]![Line4]
    Connect = [Line1] & vbCrLf & [Line2] & vbCrLf & [Line3] & vbCrLf & [Line4]
    Message = "The following CONNECT.INI has been created"
    Whole = [Message] & vbCrLf & [Connect]
    MsgBox Whole
    Set fs = CreateObject("Scripting.FileSystemObject")
    Set A = fs.CreateTextFile([FileName], True)
    A.WriteLine [Connect]
    A.Close
   
End Sub

Here is the best workaround I found so far:

1. Setup your Outlook.com account in Outlook Desktop as Exchange/Office365 account

2. Optional: Setup the same Outlook.com account in Outlook Desktop as IMAP/SMTP account (actually, SMTP part could be preferably from your non-Outlook.com account)

3. Setup your non-Outlook.com account in Outlook Desktop as POP3/SMTP account. Remember to disable "delete from server". The account should be disabled in the send/receive group settings, so POP3 is never used. New email delivery in Account settings should be set to the file for the account configured in "1.", the POP3 file can then be deleted in the Account settings dialog.

4. In Outlook desktop settings ("Options"), select "Always use default email address" and "Always start in Folder: IMAP Inbox".

5. Set the POP3/SMTP account as default account in Account settings.

This is a bit cumbersome to set up, but seems to work:

- New calendar entries are created with the non-Outlook.com address

- If setting up the account as IMAP, then in the IMAP inbox, "New mail" defaults to the proper non-Outlook.com address (in the Exchange account inbox, this does not work, as it does not follow the default account rule).

If you are fine with selecting the "From" for each new email, you can omit the IMAP inbox.

For replys, it has to be selected anyway.

The "from" selection could be possible automated via macros (and replacing the toolbar buttons with custom buttons).

=MAX(FREQUENCY(IF(Rng<0,ROW(Rng)),IF(Rng>0,IF(Rng<>0,ROW(Rng)))))

Positives:

=MAX(FREQUENCY(IF(Rng>0,ROW(Rng)),IF(Rng<0,IF(Rng<>0,ROW(Rng)))))

** array formulas need to be entered using the key combination of CTRL,SHIFT,ENTER (not just ENTER). Hold down both the CTRL key and the SHIFT key then hit ENTER.

doug robbins - mvp office apps & services (word) say:

This is the code that you would want to insert a Next Page Section Break, followed by a Heading 1 style paragraph and with the page setup in the new Section having a different first page header and footer and unlinking the headers and footers in that section from those in the previous section.

 

With Selection
    .InsertBreak wdSectionBreakNextPage
    .Paragraphs(1).Style = "Heading 1"
    With .Sections(1)
        .PageSetup.DifferentFirstPageHeaderFooter = True
        .Headers(wdHeaderFooterFirstPage).LinkToPrevious = False
        .Headers(wdHeaderFooterPrimary).LinkToPrevious = False
    End With
End With

For ribbon customization, see the following page of Greg Maxey's website:

 

http://gregmaxey.mvps.org/word_tip_pages/customize_ribbon_main.html

 

However, if this was the only customization that you were going to be using, assigning the macro to the keyboard or to the Quick Access Toolbar would probably be sufficient.
