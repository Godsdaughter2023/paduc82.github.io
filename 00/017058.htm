    Making query show an entire month of expired licenses? - Microsoft Community

    I got my previous question answered by myself. I had to create a field in where only dates would be entered of when a license would expire.     djsteele, mvp say:
    The most efficient method is to provide the dates for the first and last days of the month in your criteria:

    WHERE ExpiryDate BETWEEN #2013-06-01# AND #2013-06-30#

    It is possible to use a function on the ExpireDate, so that it can be simplified to

    WHERE Month([ExpiryDate]) = 6

    or even

    WHERE Format([ExpiryDate], "mmmm") = "June"

    but the fact that you need to execute the function call on each row makes that "more expensive".

    Note that it's possible to use a prompt in conjunction with my first suggestion:

    WHERE ExpiryDate BETWEEN DateSerial(Year(Date), [Enter Month Number], 1) AND DateSerial(Year(Date), [Enter Month Number] + 1, 0)

    djsteele, mvp say:

    In order to show 10 days in advance, you really should be using:

     

    Select[Annual CPOC].Client,[Annual CPOC].[CPOC Expired Date]

    From [Annual CPOC]

    Where [Annual CPOC].[CPOC Expired Date] >= (Date()+1)

    AND [Annual CPOC].[CPOC Expired Date]<= (Date()+10);

     

    What you had will only show those entries expiring tomorrow as well as those expiring 10 days from now: nothing in between. (and, in actual fact, if you CPOC Expired Date also contains time in addition to date, you won't get anything returned!)

     

    To be prompted to enter a month number and get everything for that particular month, you can use:

     

    Select[Annual CPOC].Client,[Annual CPOC].[CPOC Expired Date]

    From [Annual CPOC]

    Where [Annual CPOC].[CPOC Expired Date] BETWEEN DateSerial(Year(Date), [Enter Month Number], 1) AND DateSerial(Year(Date), [Enter Month Number] + 1, 0)

     

    or

     

    Select[Annual CPOC].Client,[Annual CPOC].[CPOC Expired Date]

    From [Annual CPOC]

    Where [Annual CPOC].[CPOC Expired Date] >= DateSerial(Year(Date), [Enter Month Number], 1)

    AND [Annual CPOC].[CPOC Expired Date] <= DateSerial(Year(Date), [Enter Month Number] + 1, 0)

    irgendwo say:

    After searching the web for quite some time without finding an answer but many other people with the same problem I decided to find the reason for this behavior myself.

    After some trial and error it turned out that this error was related to the current user profile. Once I realized this it was only a matter of endurance and time to find the fix.

    The solution is quite simple:

    1. quit IE
    2. delete the following file C:\Users\<username>\AppData\Roaming\Microsoft\Protect\CREDHIST
    3. restart IE

    Fixed it for me.
