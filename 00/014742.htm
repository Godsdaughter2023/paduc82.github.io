scottgem say:

I don't think I would do this with a Crosstab. Instead I would create 2 queries:

SELECT StudentID, Quarter, Period AS "Yes"

FROM Table

WHERE Yes/No = True;

SELECT StudentID, Quarter, Period AS "No"

FROM Table

WHERE Yes/No = False;

Then a third query that Joins them on StudentID

SELECT StudentID, Quarter, Yes, No, Yes+No AS PeriodTotal

INNER JOIN on q1.StudentID = q2.StudentID

FROM q1, q2;

Then use that third query for the report. 

Note: You can't use Yes and No as Column names so I'm not sure what you would use, but that's up to you.

ken sheridan say: You should be able to do this easily with a single query:

SELECT [Quarter], [Student ID#],
SUM([Period] * IIF([YourYesNoColum],1,0)) AS YesColumn,
SUM([Period] * IIF([YourYesNoColum],0,1)) AS NoColumn,
SUM([Period]) AS Total
FROM [YourTable]
GROUP BY [Quarter], [Student ID#];

The SUM operator is only actually aggregating values to return the Total column.  To return the YesColumn and NoColumn values it is merely a convenient way of returning each value in a single row. Any aggregation operator could in fact be used for these two columns.
