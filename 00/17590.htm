Stephens answer is closer, but I can't believe how many people were so off the mark with their responses.

Basically, Outlook 2010 on Windows 7. The default account is the mailbox on the corporate Exchange Server.

Every time Outlook is started, it pops up a Windows Security login prompt. Nothing to do with malware. This is "by design", but only in certain circumstances, such as "always prompt for credentials" being checked.

Another is that you have Outlook logins defined in the Credential Manager.

Close Outlook, goto Start>Control Panel>User Accounts, click "Manage your credentials", scroll down to "Generic Credentials" and remove from the vault any that start with "MS.Outlook:".

Now when you start Outlook it'll use your corporate domain login without prompting.

This situation occurs if you add multiple accounts to Outlook that have different logins, and then remove some of those accounts. The stored credentials get left behind and Outlook can't decide which one to use.

Regarding the deletion of the color box and having the whole background disappear, it is likely that you are dealing with a grouped object.  To delete just the colored box, you would need to ungroup your grouped object first, then delete the individual object (the colored box).  See this Microsoft article on grouping and ungrouping objects:

graham mayor say:
The entire background of the template is a single full page size graphic, so it is not possible to remove parts of it using Word. You would need to edit it in a graphics editor. The margins shown on the template are a function of the graphic size vis-a-vis the size of the page. To change the outer margins you would need to change the size of the graphic. The inner margins cannot be changed without editing the graphic.

The template background is not intended to be changed. You can edit the textual elements only.

The multicoloured fill in the first column is a separate graphic and can be both selected and deleted to leave the underlying colour of the background.
