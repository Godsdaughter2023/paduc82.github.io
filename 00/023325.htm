Hi John,

While you're waiting for the official Microsoft response I thought I'd offer some advice.

If you do a single user install of Office ProPlus, the licensed user activates Office and can then use it. Then every 30 days (I think, or is it every 4) it will do a license check to make sure that the license is still valid.

If the PC is used for shared access then the other users will see the Office apps and will be able to use them. What I am not sure on his for how long it will continue to work before it tried another license check. Office will not be signed in to Office 365 with the original users details.

But I believe that this will break the licensing agreement, as each user accessing the subscription based software is supposed to have their own license.

[This is what I have seen using Office 2016 on Windows 10]

The access to the OneDrive files is an interesting point as the security of the files will depend on to where on their PC the user has synchronised the files.

If they have used the default location under C:\Users\username then that will be secure, but if they decided to place the files somewhere else then they may become viewable. For example if they decided to place them on the PCs D drive.

If a computer is shared then I believe that you are expected to make use of the shared computer activation options: technet.microsoft.com/.../dn782860.aspx

Again this does require you to make sure that each user has their own individual pro plus license.

greginmich say:

I think the default scheduled time is actually 2:00 AM – but you certainly don’t need to leave your PC on all night to get Automatic Maintenance to run, and you don’t need to reschedule it for the daylight hours either (like you do with MSE’s scheduled scan). It’s all set up to run on restart if it can’t run at the scheduled time – so if you just leave the time-of-day setting alone, and turn off the PC at the end of the day, Automatic Maintenance gets triggered the next morning shortly after startup. But since it only runs when the system reaches an idle state, in order to prevent interfering with your work, it might get stopped and restarted any number of times before it finds a slot of idle time that’s long enough to finish the chores. So it might not finish until later in the day – and might not even finish at all if you don’t take at least one respectably long coffee break.

So this is definitely not your traditional “scheduled scan” – first of all because it doesn’t need any kind of setup, configuration or scheduling, and second because it will only run “on schedule” if you stop using the PC at the scheduled time and first wait for the system to enter an idle state – and then, once the maintenance gets started, you'll need to sit back and wait some more for the scheduled tasks to get done. If you interrupt this process at any point prior to the chores getting done, the whole cycle has to start over again with the next system idle.

Use below code. It will delete only those rows which are blank in cell D + one row below that 

Sub DeleteRows()
    Dim i As Long
