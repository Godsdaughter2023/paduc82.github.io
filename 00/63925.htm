wunders say:

By chance, do you mean "Winmail.dat" instead of Win Dat?

This problem can occur when the person sending the email uses Outlook as an email client (and you don't) and has his preferences set to "RTF" format for the body of the email.  This can cause attachments to be encoded in the "TNEF" format and results in a "Winmail.dat" file for the receiver.

"Email received from a sender using Outlook includes a Winmail.dat attachment"
   < http://support.microsoft.com/kb/278061 >

"How to Prevent the Winmail.dat File from Being Sent to Internet Users"
   < http://support.microsoft.com/kb/138053 >

There are utilities for decoding these attachments.  One such freeware is the following:

"Winmail Opener"
  < http://www.eolsoft.com/freeware/winmail_opener/ >

An article describing this problem and linking to several decoders comes from Wikipedia:

"Transport Neutral Encapsulation Format"
   < http://en.wikipedia.org/wiki/Transport_Neutral_Encapsulation_Format >

 

HTH
    JW
