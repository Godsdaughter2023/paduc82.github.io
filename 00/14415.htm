Real-time protection scans files as they‚Äôre accessed, so it has to move very quickly in order to keep from bottlenecking file system activity. Consequently it doesn‚Äôt take the time to unpack container (archived) files and scan the files inside the container. So the advantage of scanning archived files is that it will detect archived malware that real-time protection might have missed. Since any archived malware files would theoretically be detected and handled by real-time protection at the time they were extracted or executed, archived malware files shouldn‚Äôt pose a real threat. But on the other hand, reality doesn‚Äôt always seem to follow the prescribed theoretical path, and lots of people just don‚Äôt want any malware on their PC ‚Äì regardless of whether it poses a real threat or not. So it's really a question of personal preference.

On the vulnerability question, I don‚Äôt think it‚Äôs very likely that malware would target any of your personal archived files. So if you have a collection of large archived backup files, for instance, you can probably exclude those files without introducing too much of a risk. But use path exclusions on the folders where the archived files are located ‚Äì don‚Äôt use file-type exclusions.

When it says it is locked by another user, it should give you a user name. In this case it should be (annoyingly) your own name.

In Word go to File menu > Options command > General option > Personalize your copy of MS Office section.  In this section you should see your own USER Name.  If you don't, do enter it here, it will make that error message a little more clear.

After a crash, try using the File Open dialog. Look at the bottom of the pane, there should be a "Recover Unsaved Documents" option. Click on it to hopefully re-open the document(s) open at the time of the crash.

When you open a file in Word, Word creates a number of temporary "working" files, including the lock file that is giving you problems.  If the document is closed normally, and Word is closed normally, those temp files are automatically cleaned up by Word.  But when Word crashes, those temp files are left behind. With little luck you can use those temp files to recover at least some of your unsaved work.

The reason is simple. This "feature" was written some time in the late 1980's or early 1990's when networking was first introduced.
