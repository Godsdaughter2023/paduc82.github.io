craigmore say:
Any input would be much appreciated.
Here's how I tamed the beast...

1. It takes Win8 Pro. The functionality isn't there on Win8 "vanilla". As far as I can tell, this limitation is not documented anywhere. In fact, there's next to no documentation of any kind on Sync, from what I could discover.

2. The relevant directory(ies) must be explicitly shared as read/write for the Homegroup on a Home/SOHO network, otherwise you won't be able to write to them from the remote/client system whether you're online or offline. I found that not all sub-directories necessarily inherit the share, so there were a few I had to share individually.

3. A host folder set up as a drive on the remote won't show up in Sync Center as a partnership, nor as an available resource to sync. It will sync, however. I've yet to find anything that shows up as "available" for sync in the Sync Center.

4. Sync takes its own sweet time about keeping up to date. It's essential, if you need to work on stuff on the road, to manually initiate a sync and let it run to completion before you unplug from the network.

Hope that helps.
