You can use VBA code...

'---
Sub CustListRemoval()
 Dim N As Long
 Dim lngCount As Long

 lngCount = Application.CustomListCount
 For N = lngCount To 10 Step -1      'leaves a few in place
   Application.DeleteCustomList N
 Next
End Sub

Using Outlook on the web, perform these steps:

1. Open the mailbox in Outlook on the Web.

After this, create the list of mailboxes to migrate, for this it is necessary to have access to user mailboxes before being able to migrate them to Office 365.

The steps required to create the destination mailboxes depend on how the mailboxes are accessed:

• You can know the password of each user's mailbox or reset the passwords to the new passwords you know. Follow the steps described in creating the list of user mailboxes when knowing user passwords or resetting passwords.

• The source email system allows you to use the mailbox administration credentials to access users' mailboxes, which means that you do not need to know the passwords or redefine them. Follow the steps in Create a list of user mailboxes with administrative credentials to access them for information on how to access users' mailboxes.

Create the list of user mailboxes when the user's passwords are known or reset the passwords, a migration file containing a list of mailboxes is created to migrate to Office 365.

When you create the migration file, you must know the password of each mailbox that is migrated. We assume that you do not know user passwords, so you will probably need to assign temporary passwords (resetting passwords) to all mailboxes during the migration.

If you do not want to, you do not have to migrate all the mailboxes at once. You can do it in batches, according to your convenience. You can include up to 50,000 mailboxes (one row for each user) in your migration file, which can be up to 10 MB.
