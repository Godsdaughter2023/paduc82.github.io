steve rindsberg say:

If the presentation's not too large (number of slides, that is), you could make as many copies of it as there are slides, then for each copy, delete all but one slide (slide 1 from the first copy, slide 2 from the second and so on) to see what file's the biggest.

Or perhaps simpler, divide and conquer.  Save two copies, delete the first half of the slides from one copy, the second half from the other copy and then compare file sizes.  If one seems disproportionately large, give IT the divide and conquer treatment.  Repeat, Rinse, Repeat until you've worked out which slide is largest.

OR

Make a copy of the presentation, add .ZIP to the end of the file name (assuming you have file extensions turned on and can see the .PPTX that's there already).  Double click the ZIP file to open it, browse to the Media folder and have a look there to see what files are unusually large.  Your images, sounds and any other media will be there.

hansv mvp say:

Let's say the text box is named txtCity.

Create an event procedure for the After Update event of the text box:

  • Open the form in design view.
  • Select the text box.
  • Activate the Event tab of the Properties window.
  • Click in the After Update event.
  • Select [Event Procedure] from the dropdown in this event.
  • Click the builder buttons ... to the right of the dropdown arrow.
  • Make the code look like this:

 

Private Sub txtCity_AfterUpdate()
    If Not IsNull(Me.txtCity) Then
        Me.txtCity = StrConv(Me.txtCity, vbProperCase)
    End If
End Sub

 

  • Switch back to Access.
