CapitalOne  DNS records appear to have a very short Time To Live and the host name that you use is an alias, so if your DNS is too slow for IE's liking you could see your symptom with it.  Furthermore it doesn't allow ICMP so testing with a normal ping command would be too slow.

In order to let  IE use cached DNS records you might have to make a cmd script out of these 3 commands:

    ping -n 1 -w -1  www.capitalone.com
