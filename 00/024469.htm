    Dim prngFirstCat As Range
    Dim prngLastCat As Range
    Dim prngFullCatList As Range

    Set prngCatList = Range(prngFirstCat, prngLastCat)

End Function

Closing the other open workbooks and making an insignificant change in an empty cell causes this workbook to recalculate and the UDF returns the correct values/summary data & charted ranges now show the data they should.

Is there anything I can do to prevent the UDF returning the #VALUE! error with other workbooks open?

This file is not the final product, but it is an intermediate file in the process that does go out to clients.  It is not necessary that this UDF work with other workbooks open, but it is extremely desirable that it does work.

CTB

andreas killer say:

The problem is...anytime I have any other workbook open...even a blank workbook with some text entered into a single cell, this workbook will recalculate and return a #VALUE! error only in the single cell that should have a number in it (all cells that should appear blank do appear blank).  This causes summary cells/formulas somewhat dependent on this single cell to return #VALUE! errors...which causes several charted ranges dependent on these summary cells to return #VALUE! errors.

This is a symptom, because VBA "assumes" the named range that you call from your UDF in the active workbook.

The solution is not to redirect it to the workbook with the formula, what might be work...

A UDF is only calculated and called from Excel correctly when you pass the cells that you need inside as argument to the UDF.

When you access other cells in the UDF, the UDF must be volatile (means must be calculated every time when a formula anywhere is calculated, resp. a value is entered), otherwise the result can be wrong.

But when I look into your UDF, why not use a formula instead?

=COUNTA(OFFSET(Categories,0,0,MATCH("Exclude",Categories,0)-1,2))


That is what you want to accomplish, right?

Andreas.
