The suggestion re "do it in boxes" would be to use two pairs of linked textboxes - one set of linked textboxes for English, the other set of linked textboxes for German. doug robbins - mvp office apps & services (word) say:

A macro containing the following code will combine the two documents with the text from each on alternate pages

    Dim sourcea As Document, sourceb As Document, target As Document, Pages As Integer, Counter As Integer, targetrange As Range 'targetrange added

    Dim evenpage As Range

    Set sourcea = Documents.Open(FileName:="...")

    sourcea.Repaginate

    Pages = ActiveDocument.BuiltInDocumentProperties(wdPropertyPages)

    MsgBox Pages

    Set sourceb = Documents.Open(FileName:="...")

    Set target = Documents.Add

    target.PageSetup.LeftMargin = sourcea.PageSetup.LeftMargin

    target.PageSetup.RightMargin = sourcea.PageSetup.RightMargin

    target.PageSetup.TopMargin = sourcea.PageSetup.TopMargin

    target.PageSetup.BottomMargin = sourcea.PageSetup.BottomMargin

    target.AcceptAllRevisions

    Counter = 0

    While Counter < Pages

        sourcea.Activate

        ActiveDocument.Bookmarks("\page").Range.Copy

        Set targetrange = target.Range

        targetrange.Start = targetrange.End

        targetrange.Paste

        ActiveDocument.Bookmarks("\page").Range.Cut

        sourceb.Activate 'Assumed to be the document containing the even pages

        Selection.EndKey Unit:=wdStory 'Line of code added to start from the end of the document

        ActiveDocument.Bookmarks("\page").Range.Copy

        Set targetrange = target.Range

        targetrange.Start = targetrange.End

        targetrange.Paste

        targetrange.Start = targetrange.End

        targetrange.InsertBreak Type:=wdPageBreak

        Set evenpage = ActiveDocument.Bookmarks("\page").Range

        evenpage.Start = evenpage.Start - 1

        evenpage.Delete

        Counter = Counter + 1

    Wend

    sourcea.Close wdDoNotSaveChanges

    sourceb.Close wdDoNotSaveChanges

Before running the macro, you should probably adjust the page breaks in the individual documents so that the text on each page in one of them is synchronised with the text on the corresponding page in the other.

doug robbins - mvp office apps & services (word) say:

Here is a series of screen shots is the procedure for setting up the text boxes.  

Starting with a blank document, turn on the display of non-printing characters by clicking on the Show\Hide ¶ button on the home tab of the ribbon,

press CTRL+ENTER three times to add three pages

With the selection before the hard page break on the first page, on the Insert tab of the ribbon, expand the Text Box item and select Draw Text Box

Then draw a text box to fill the first page.

Don't worry about the Layout Options at this point, they will be dealt with later.  Rather, move the selection to before the hard page break on the second page and then press F4 (Repeat) to insert another text box (and repeat that process for the other two pages

Now select the first text box and on the Drawing Tools>Format tab of the ribbon, expand the Wrap Text dropdown and click on In Line with Text

Select each of the other text boxes in turn and press F4 each time to change their layout to in-line with text.

Then, with the selection in the first listbox, on the Drawing Tools>Format tab of the ribbon, click on Create Link

when you move the mouse back into the document window, the cursor will have changed to a "pouring jug".  Move the cursor into the third text box immediately before the paragraph mark (¶) and then left click, which will link it to the first text box.

Note that the paragraph mark will disappear from that textbox due to it being linked to the first text box. 

Repeat this process to link the second text box to the fourth text box.

At this point, move the cursor to the end of the document and press CTRL+a , then CTRL+c and thenCTRL+v, you can add another four textboxes to the document in which the fifth will be linked to the seventh and the sixth to the eight, so you will need to then link the 3rd to the 5th and the 4th to the 6th and you can repeat that process up to the maximum number of links that Paul mentions (two sets of 32)

If you need more that 64 pages, before doing anything else, save the document (probably as a template) so that you can use File New and create as many documents from that template as you need.

When you want to create your document with the text in the two languages, select and copy the first language and place the cursor into the first text box and press CTRL+v and the text will flow from that text box to the third, fifth, seventh, etc.  Then copy the second language and place the cursor into the second text box and press CTRL+v and the text in that language will flow from the second to the fourth, sixth, eighth, etc

Note, to synchronize the text between the different languages, you will need to use the ENTER key to force the text to move to the next text box as you cannot insert either a page break of column break inside a textbox.

macropod say:

As Doug's last post shows, creating the interspersed-linked textbox structure manually can be time-consuming. Here's a macro to automate it in a new, empty document.

Sub CreateLinkedTextBoxStructure()
Dim Rslt, i As Long, j As Long, sWdth As Single, sHght As Single, Rng As Range
Rslt = InputBox("How many source pages (max. 32) do you have in one document (choose the longer)?")
If Rslt = "" Then Exit Sub
If Not IsNumeric(Rslt) Then MsgBox "Numeric input required. Exiting": Exit Sub
j = CLng(Rslt)
If j < 2 Then: MsgBox "Nothing to do. Exiting.": Exit Sub
If j > 32 Then j = 32: j = j * 2
Application.ScreenUpdating = False
With ActiveDocument
  With .PageSetup
    sWdth = .PageWidth - .LeftMargin - .RightMargin
    sHght = .PageHeight - .TopMargin - .LeftMargin
  End With
  .Shapes.AddTextbox Orientation:=msoTextOrientationHorizontal, _
    Left:=.PageSetup.LeftMargin, Top:=.PageSetup.TopMargin, Width:=sWdth, Height:=sHght
  Set Rng = .Shapes(1).Anchor
  For i = 1 To j - 1
    .Range.InsertAfter Chr(12)
    .Range.Characters.Last.FormattedText = Rng.FormattedText
  Next
  For i = 1 To j - 2
    .Shapes(i).TextFrame.Next = .Shapes(i + 2).TextFrame
  Next
  .Shapes(1).TextFrame.TextRange.Select
End With
Application.ScreenUpdating = True
End Sub

For PC macro installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm
For Mac macro installation & usage instructions, see: https://wordmvp.com/Mac/InstallMacro.html

rich007a say:

Hi,

You're not going to be able to have a grid of cells in a textbox.  You can't even have tabstops.  The best you can do is something like this:

Sub TestIt()
    Dim str As String
   
    str = "Name : " & Range("A2").Value & vbTab
    str = str & "Gender : " & Range("B2").Value & vbCr
    str = str & Range("C1").Value & " : " & Range("C2").Value & vbTab
    str = str & Range("D1").Value & " : " & Range("D2").Value & vbCr
    str = str & Range("E1").Value & " : " & Range("E2").Value
   
    Call MakeTextBox(Range("B5"), str)
End Sub

Sub MakeTextBox(rngCell As Range, strText As String)
    Dim shp As Shape
   
    Set shp = rngCell.Parent.Shapes.AddLabel( _
        Orientation:=msoTextOrientationHorizontal, _
        Left:=rngCell.Left, _
        Top:=rngCell.Top, _
        Width:=1, _
        Height:=72)
   
    shp.TextFrame2.WordWrap = msoFalse
   
    shp.TextFrame2.TextRange.Characters.Text = strText
   
     'Optional formatting:
    With shp.Fill
        .Visible = msoTrue
        .ForeColor.ObjectThemeColor = msoThemeColorAccent3
        .ForeColor.Brightness = 0.6000000238
    End With
    With shp.Line
        .ForeColor.ObjectThemeColor = msoThemeColorAccent3
        .ForeColor.Brightness = -0.25
        .Visible = msoTrue
        .Weight = 1.5
    End With
    With shp.TextFrame2.TextRange.Font
        .Name = "Arial"
        .Size = 8
    End With
End Sub

Note that if the Names or any data gets too long, you'd have to start padding with extra tabs.  That is not easy to implement elegantly....

stefan blom say:

If you can settle for having the author names added to the text of the document for the printout, you can use the following macro, which adds the author names next to each revision, prints the document and then closes it without saving:

Sub PrintRevisionsWithAuthorNames()
'Macro created by Stefan Blom, MVP, June 2017
Dim c As Revision
Dim r As Range
Dim IsTrackingOn As Boolean
'First save the document
ActiveDocument.Save
'Find out if tracking is enabled
IsTrackingOn = ActiveDocument.TrackRevisions
If IsTrackingOn = True Then
ActiveDocument.TrackRevisions = False
End If
'Add the author names in the text
For Each c In ActiveDocument.Revisions
Set r = c.Range.Duplicate
r.InsertAfter " [" & c.Author & "]"

Next c

ActiveDocument.PrintOut
ActiveDocument.Saved = True
ActiveDocument.Close
End Sub

For installation instructions, see http://www.gmayor.com/installing_macro.htm.

cbarnhorst say:

I don;t mean "system image."  The "Restore from a system image" tool is on any Windows installation disc and is found by clicking the Repair metalink in the lower left of the initial splash screen when the computer is booted with the disc.  Call it a "repair" disc if you like.  A system image made with the tool in the Backup and Restore (Windows 7) section of the CP is always far too large to fit on a disc.

What is changing is Windows, so recently made images are going to be suspect since the tool is no longer fully compatible with evolving Windows.  The incompatibilities between the tool and Windows itself began to appear after the release of the Anniversary Update (1607), which appears to be when the development of SIB stopped.  If your earliest archived image is very recent I suggest you not trust it no matter what.  Currently, errors are showing up like the restore tool rejecting an image backup on grounds it is made from a non-UEFI device when in fact it is.  And so on.  Since you have to do a restore to test whether or not the image is usable, do not trust the image you have that was made at the time of the  Creators Update (1703).  It may or may not restore but you just don't know.  If that is the only image you have, IMHO you have to assume that you may not be able to restore it.

It is like trading in your Honda Civic for an SUV and discovering when you get home that the SUV doesn't fit in your garage.  You didn't find out until after the deal was done.
