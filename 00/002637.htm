scottgem say:

This would be unnecessary if you followed a better deployment model. I never have to worry about that within my company. 

The way we deploy apps is a) splitting the app between back end (tables) and front end (everything else), b) deploying the front end to the users local machines and c) using an automated front end updater like Auto FE Updater to update the local copy after changes are made. 

So the process works like this. I keep a copy of all the apps on my local PC (which are synced with the file server for nightly back up). When enhancements are requested I work with the dev copy of both FE and BE to test the changes. When I'm satisfied the change works, I then upload the FE to a folder on the server. The next time a user tries to run that app, it checks to see if a newer version is available. If there is, it is downloaded to the user's local machine transparent to them.

While the updater I linked to is commercial, it is well worth it. However there are many others some freely distributed that you can find.

Note also that sharing the front end (running from a shared folder) is NOT recommended as it is an invitation to corruption. And your apps NEED to split.

hansv mvp say:

You can't really do that with an input mask because you don't know where the last name will start.

You can use code in the After Update event of a text box bound to the field on a form:

 

Private Sub txtName_AfterUpdate()
    Me.txtName = StrConv(Me.txtName, vbProperCase)
End Sub

 

This is not "intelligent": paul mcmartney will become Paul Mccartney, not McCartney, and marc o'polo will become Marc O'polo, not O'Polo.
