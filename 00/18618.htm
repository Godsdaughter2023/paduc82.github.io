Populate a field on 1 table from another table - Microsoft Community

jrrock26 say:

Populate a field on 1 table from another table

I have 2 tables tblAssets & tblCal. In tblAssets I have a field "column" titled Calibration Due Date. In tblCal I also have the same field. tblCal contains multiple records for each asset and tblAsset is the unique asset list. What I would like to do is have the field Calibration Due Date automatically populated with the next due date after today or Date() from tblCal for each unique record in tblAsset. For instance...

 

tblCal:

Asset#                                Calibration Due Date

00001                                         7/20/2012

00001                                         7/20/2013

00002                                          5/10/2012

00002                                          5/10/2013

 

 

tblAsset:

Asset#                                Calibration Due Date

00001                                            7/20/2013

00002                                            5/10/2013      

 

Thank you                       

ken sheridan say: Don't include a Calibration Due Date column in the tblAsset table.  That introduces redundancy and the possibility of update anomalies.  The data will immediately be incorrect once the next Calibration Due Date for an asset is passed.  The value should be computed in a query, which guarantees that it is always up to date:

SELECT tblCal.[Asset#], MIN(tblCal.[Calibration Due Date]) AS NextDueDate
FROM tblCal INNER JOIN tblAsset ON tblCal.[Asset#] = tblAsset.[Asset#]
WHERE tblCal.[Calibration Due Date] > DATE()
GROUP BY tblCal.[Asset#];

In fact, with the columns you have shown, you don't need the JOIN, as you can simply use:

SELECT tblCal.[Asset#], MIN(tblCal.[Calibration Due Date]) AS NextDueDate
FROM tblCal
WHERE tblCal.[Calibration Due Date] > DATE()
GROUP BY tblCal.[Asset#];

However, if there are in reality other columns in tblAsset, e.g. AssetName etc, which you would also want returned, then a JOIN would be necessary.  Any additional columns from tblAsset should be included in both the SELECT and GROUP BY clauses of the query.

dani_wb say:

Hello, i've just came across the same issue.

The Problem is definitely that the Preview Pane is opening a Word.

Because Documents will get open in the same instance, the new Document opens in the Winword.exe from the preview pane.

That's the reason that closing the Preview Pane will not solve the problem, as the Winword.exe stays a while in the background. You can then just kill it with tskill winword (be aware that all winword will be killed!) and then the dotx will open a new Document.

Microsoft should fix this issue.

But in the meantime you can just delete the Preview Pane for DOTX Files, just delete the following key:

[HKEY_LOCAL_MACHINE\SOFTWARE\Classes\.dotx\ShellEx\{8895b1c6-b41f-4c1c-a562-0d564250836f}]

As Documented in the following site, this is the registry for the Preview Pane:

https://msdn.microsoft.com/de-de/library/windows/desktop/cc144144(v=vs.85).aspx

edit: Found a Solution!!!

(Please notice, that changing the registry on your own risk, at best, export before you modify it so you can revert the changes)

Under HKEY_CLASSES_ROOT\Word.Template.12\shell\New\command

You will find the opening Command for new Documents.

Just change the parameters /n /f "%1" with /n /w /f "%1" on both the entries:

(Default)

command


After this change, the Word Template opens in a new Process and opens visibly!
