bill manville say:

In the case that the data is on a different sheet, the code can be simplified significantly

Sub CopySheetWithPivotChart()
' assumptions the active sheet contains a single pivot table built on data in a different sheet and a single pivot chart built on the pivottable
  Dim WB As Workbook
  Dim stSource As String
  Dim stSheet As String
  Dim stNewSheet As String
  Dim I As Integer
  Set WB = ActiveWorkbook
  stSheet = ActiveSheet.Name
  ' make a name for the new sheet
  I = 1
  Do While IsIn(WB.Sheets, stSheet & I)
    I = I + 1
  Loop
  stNewSheet = stSheet & I
  ActiveSheet.Copy ' to new workbook
  ' change the name of the sheet
  ActiveSheet.Name = stNewSheet
  ' now move the sheet back to the original workbook
  ActiveSheet.Move before:=WB.Sheets(stSheet)
  ' change chart's source to its own pivottable
  ActiveSheet.ChartObjects(1).Chart.SetSourceData ActiveSheet.PivotTables(1).TableRange1
 
End Sub

Function IsIn(oCollection As Object, stName As String) As Boolean
  Dim O As Object
  On Error GoTo NotIn
  Set O = oCollection(stName)
  IsIn = True
NotIn:
End Function

How interesting. I believe I've fixed this problem.

It occurred to me that, just because you delete the whole profile on the machine (under c:\users), the registry is actually unaffected. So I removed my profile from the registry (while logged in with another admin account), logged back in and - problem solved.

Fair warning: Editing the registry can be detrimental to your computers health :)

***To be sure, I tested this a second time, to ensure I'd be aware of any detrimental effect to a working profile if I removed the registry profile. The computer makes a new profile for you. So your documents etc, would need to be copied from your old profile. A minor annoyance in comparison to Outlook not working.***

Article to copy files between user accounts:

http://answers.microsoft.com/en-us/windows/forum/windows_7-files/how-to-transfer-files-from-one-user-account-to/e262acd6-08f2-4a77-9cfb-1b670dd6706d

http://pcsupport.about.com/od/registry/ht/find-user-security-identifier.htm

S-1-5-21-3199535551-1522028377-790874951-3390

Open "regedit"

You'll want to look here:

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList

You'll see a list of GUIDs there, find the one that corresponds to your account.

Then simply right click and delete it.

Now log off the other account, log yourself back in.
