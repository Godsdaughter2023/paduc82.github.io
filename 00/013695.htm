Ah! I see it now. 

I was able to reproduce the difference in formatting between Excel 2010 and Excel 2011. Interesting difference. The difference shows up in the VBA code recorded between the two versions, as well.

I think the reason you get the result you want in Excel for Windows is that the Windows version is not properly maintaining the background setting. In other words, you’re able to get the result you want due to a bug in the Windows version. So don’t go complaining to Microsoft about this difference, because I think the proper fix would be to make the Windows version behave like the Mac version, which would not be the result I think you want to have.

However, the macro code made in Excel for Windows, when run on the Mac, produces the result you want. Here’s the code recorded in Windows that accomplishes what you want on the Mac:

Sub SetFormatting()
    Selection.FormatConditions.Add Type:=xlExpression, Formula1:= _
        "=IF($E$3=1,TRUE,FALSE)"
    Selection.FormatConditions(Selection.FormatConditions.Count).SetFirstPriority
    With Selection.FormatConditions(1).Font
        .Bold = False
        .Italic = True
        .TintAndShade = 0
    End With
    Selection.FormatConditions(1).StopIfTrue = False
End Sub

Obviously, you would substitute the formula you want as the condition to be met. This should solve the problem for you.

kosh vorlon say:

Hi Alex,

 

Try logging in as an Administrator if you are logged in as a standard user.

 

I'm not 100% sure about the attributes/permissions of these logs, so just to make sure it isn't a simple matter of how things are configured to show, in Windows Explorer / Tools / Folder Options / View, select the radio button to "show hidden files and folders", uncheck the box to "hide extensions of known file types" (not really critical or helpful here - but typically useful and while you're at it...), and uncheck the box to "hide protected operating system files" (but be careful ongoing that just because you see something doesn't mean you should do anything to it because this removes the minor protection of hiding them - but does help you in terms of seeing everything).  Click Apply and then OK to save the changes and then refresh the view of that folder and see if it helps.

 

If that doesn't help, then a simple search may not find these logs as I suspect they will not be part of the index (assuming you're using the indexing feature).  That means to locate them with a search, you'd need to do an Advanced Search and check the box to "include non-indexed, hidden and system files".  You could start with a more targeted location of say C:\Windows and if that fails, try going with Everything.  I'd also suggest trying just MPLog rather than anything more specific so you'll have a better chance of finding the general area in case, for some reason, that specific scan didn't create a log, then you may still find the location of others from the past (assuming this isn't the first time).

 

Next, try running WDO a few more times and repeat the above to verify it wasn't just a fluke that the log didn't get created (especially if it did find something and the process of removing what was found might somehow have wiped the log - which is so unlikely I hesitate to even bring it up, but this should clarify it for sure since subsequent scans should be clear if the first cleared your system).

 

Finally, if that too doesn't work and since whatever reason you had for running WDO rather than some other program in normal mode on the computer may have cleared some but not all infections, you may want to try some alternatives if WDO at least got you to be able to use the computer normally again (if that was a problem or even if not).  I'd suggest Malwarebytes - Free, ESET Online Scanner, TDSSKiller, and Hitman-Pro (trial) which if all clean would probably rule that out.

 

And to be honest, after that I'd be somewhat at a loss to explain and would need time to think about and research where to go from there.  But post back with details of what you did and what happened and we'll try to come up with something else to try.

 

Good luck!

 

Kosh
