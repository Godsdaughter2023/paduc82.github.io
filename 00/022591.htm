For example. AGGT of student in row should be 12 is 2+3+3+3+3+3+4= 21

Which formula will help to do that, and it should not return any value if student perfomed less than 7 subjects.

amit tandon say:

In case there are NO blank cells in any row then: Enter below formula as an array formula (CTRL+SHIFT+ENTER) in cell AM3 & copy down:

=IF(COUNT(E3,H3,K3,N3,Q3,T3,Y3,AD3,AI3,AL3)>=7,SUM(SMALL(CHOOSE(ROW($1:$10),E3,H3,K3,N3,Q3,T3,Y3,AD3,AI3,AL3),ROW($1:$7))),"")

In case there ARE blank cells in any row then: Enter below formula as an array formula (CTRL+SHIFT+ENTER) in cell AM3 & copy down:

=IF(COUNT(E3,H3,K3,N3,Q3,T3,Y3,AD3,AI3,AL3)>=7,SUM(SMALL(IF(CHOOSE(ROW($1:$10),E3,H3,K3,N3,Q3,T3,Y3,AD3,AI3,AL3)>0,CHOOSE(ROW($1:$10),E3,H3,K3,N3,Q3,T3,Y3,AD3,AI3,AL3)),ROW($1:$7))),"")

EXCEL FIND DUPLICATES IN TWO COLUMNS AND CREATE A LIST OF THEM IN A - Microsoft Community

seanrf say:

EXCEL FIND DUPLICATES IN TWO COLUMNS AND CREATE A LIST OF THEM IN A THIRD

I have an list of account numbers  in column 1 or A and another list of another list of account numbers in column J or 10 using an array formula i want to create a third column witch contains the account numbers which are in in both of the other lists any suuestions
bernie deitrick say:

Array-enter (enter using Ctrl-Shift-Enter) the formula

=IFERROR(INDEX(A:A,SMALL(IF(NOT(ISERROR(MATCH($A$1:$A$1000,$J$1:$J$200,FALSE))),ROW($A$1:$A$1000)),ROW(A1))),"")

Change the two $1000 to the last row filled in column A, and the $200 to the last row filled in column J. Then copy down until the formula returns blanks.

karl timmermans [outlook mvp 2012-2018] say:

How can I map (or get to) the above missing fields so that they are exported?


You can't change the fields that are available for either import or export. The button labeled as "Map Custom Fields" is a misnomer in that it has nothing to do with "custom defined fields" which cannot be mapped via the Outlook import/export wizard. In short, you can only map fields as made available in the Outlook field map.

 

In terms of your specific field set (Mailing Address), the MA is one of the 3 address field sets (Business, Home, Other) which can differ for each individual contact but unfortunately, you can also not export the value of the "Selected Mailing Address" to know which of the three is assigned as the Mailing Address for a contact.

 

You may find any of the ContactGenie programs with export functionality to be of interest (QuickPort/DataPort/MultiPort/Exporter) all of which provide the ability to specifically export the Mailing Address info (along with a host of other fields that are otherwise not available via the Outlook import/export wizard)
