john w. vinson say:

A Report object in an Access database doesn't contain any data at all. It's just a framework, a layout for organizing data from a Query (the report's Recordsource) onto a page. Saving a Report doesn't save any data anywhere; it just saves the structure of the report (which textboxes go where, the margins, labels, headers and so on).

So if you change the parameters of the Query upon which the report is based, you'll see the new data.

Windows 8 comes with YaHei, but not YaHei Light. I'm not sure where you got it from.

From your samples, it appears that YaHei Light only has 1 font, the Light. It is not a family, so applying the Bold and/or Italic properties makes Word fake the appearance. Since there is not corresponding Bold font, Word applies a stroke to the outline. This stroke does not print to a PDF. In your sample, the Italic has also been faked by Word by applying a fairly extreme slant to the Light font. This characteristic does print to PDF, but you don't actually have a YaHei Italic, just a slanted normal. This is normal operation for Word, not a bug.

To get a true Bold, use YaHei instead of YaHei Light. It has a corresponding Bold weight that will print correctly to PDF.
