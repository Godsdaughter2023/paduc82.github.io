Pop-up box in Windows Internet Explorer - Microsoft Community

jsw387 say:

Pop-up box in Windows Internet Explorer

Just recently I started getting a pop-up box in IE 11 when I try to change page, "Are you sure you want to leave this page?" then listed 2 options,  >Leave this page or  >Stay on this page  and the choice doesn't always work.  How do I get rid of this supposed warning? 
jenicel dri say:

Hi,

Here's to explain why some websites insist on double-checking your departure, and what you can do to stop them. There's a reason behind this feature. Consider a site where you’re typing in a large amount of text, such as an email or filling in a long form. It could be even more annoying to carelessly close the Window with a single click and lose all of your typing. In this case, the feature gives you a chance to go back and submit or save your work before losing it forever. For this reason, you should think carefully about disabling this feature. The pop-up message is generated by JavaScript code embedded in the web page, so if you’re sure you want to disable it just simply follow the steps bellow.

1. In Internet Explorer, click on the gear wheel icon at the top right.
2. Select Internet options from the menu.
3. Select the Security tab and click the Custom level button.
4. Scroll down to Scripting, then under Active Scripting, click disable to turn off JavaScript.
5. Then click OK.

Let us know if you need further assistance.

Here's are the few ways contacts typically disappear:
- a corruption (and you more or less ruled that out)
- the section that has the contacts is hidden (e.g. On My Computer contacts and the preference to hide On My Computer data is checked on, the address book is unchecked, etc)
- the identity folder has been moved and Outlook started as if the account never existed (move the identity back where it belongs in ~/Documents/Microsoft User Data/Office 2011 Identities — though it doesn't sound like that's what you saw).

Does any of this apply to you?
If you search for contacts making sure the scope is properly set to all itmes, do you find them?
