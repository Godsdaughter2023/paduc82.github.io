doug robbins - mvp office apps & services (word) say:

Two ways,

You could use an If  then  Else field to test for the result of the mergefield and if it returns 0, have it insert a space or nothing instead

{ IF { MERGEFIELD nameofmergefieldthatmightbeempty } = 0 " " { MERGEFIELD nameofmergefieldthatmightbeempty } }

instead of the " " which will insert a space, use "" if you do not want anything,

Note, you must use Ctrl+F9 to insert each pair of field delimiters { } and Alt+F9 to toggle off their display.

Alternatively, if you go into Word Options>Advanced and in the General section of the dialog, check the box for Confirm file format conversion on open, then when you attach the data source to the main document, a dialog box will appear in which you can select the method by which the data source is attached to the mail merge main document.  It will probably be necessary to check the "Show all" box in that dialog and then you will see a DDE method of connection (the one that was used by previous versions).  If you select that method, the behavior will probably be that which you are used to seeing.

Hope this helps.

alexsun_2012 say:

Hi Stephen,

After you resolve the initial problems, you don’t have to delete the old migration batch. You can pause it and then click Resume button to restart the migration batch.

When you restart the migration batch, the migration service connects to your on-premises Exchange server and queries the directory for the un-migrated mailboxes and complete the migration. During the process, the migration service skips the mailboxes, distribution groups, and contacts in your on-premises Exchange organization that have already been migrated and won’t affect the incremental sync of those mailboxes.
