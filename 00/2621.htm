Blank Picture Control Field print issue - Microsoft Community

ktug_914 say:

Blank Picture Control Field print issue

I have created a form with fillable fields.  I have been requested to add 3 Picture Control fields so that the user can click add pictures in these 3 fields with ease.  The only problem is they will not always have 3 pictures to add and when the control is empty, it still appears on printing.  How do I hide a blank/empty Picture Control field on print?
jay freedman say:

There is no "simple" way to do this. Anything you do is going to require a macro.

A partial solution is to install the following macro in the form's template (for instructions, see http://www.gmayor.com/installing_macro.htm, and note that the template will have to be saved as a Word Macro-Enabled Template, *.dotm).

Sub PrintPreviewAndPrint()
    Dim CC As ContentControl
    Dim rg As Range
    
    For Each CC In ActiveDocument.ContentControls
        If CC.Type = wdContentControlPicture And _
            CC.ShowingPlaceholderText Then
                Set rg = CC.Range.Paragraphs(1).Range
                rg.Font.Hidden = True
        End If
    Next
    
    Dialogs(wdDialogFilePrint).Show
    
    For Each CC In ActiveDocument.ContentControls
        If CC.Type = wdContentControlPicture And _
            CC.ShowingPlaceholderText Then
                Set rg = CC.Range.Paragraphs(1).Range
                rg.Font.Hidden = False
        End If
    Next
End Sub

Sub FilePrint()   ' used by Word 2007
    Call PrintPreviewAndPrint
End Sub

It's a "partial" solution in that the macro will run when the user press Ctrl+P to print, but it won't run if they click File and then click Print. Also, the Print dialog that appears is the old one that may be familiar from Word 2007 and earlier. 

A more complete solution is possible, but the programming will be more complicated. For a similar solution -- but one that works only with text content controls but not picture controls -- look at http://gregmaxey.mvps.org/word_tip_pages/content_controls_do_not_print_pht.html.
