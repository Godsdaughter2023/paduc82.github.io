john w. vinson say:

As a rule, it's best to do your database development in the LOWEST version that will be used. Access2010 can run Access2003 databases just fine (Access97 too, for that matter); but the reverse is NOT always true. New features introduced in 2007 or 2010 will cause errors if you try to use the database in 2003; and in practice even if you don't use an explicitly documented "new feature", it's risky! The code base is simply different, and developing in 2010 and running the database in 2003 or 2007 is a recipe for trouble.

 

Either persuade the customer to upgrade; give them a (free) copy of the 2010 runtime; or get a copy of the version they have installed, install it in a Virtual Machine, and do your development there.
