Writing Excel math functions (=SUM...) - Microsoft Community

smuga say:

Writing Excel math functions (=SUM...)

In an active sheet's cell I want to add a number of another cell to a value from previous sheet's cell. What's the magic?
gord dibben say:

If you don't know or don't want to hardcode the name of the previous sheet you can use a UDF

Function PrevSheet(Rg As Range)

'accounts for more than one workbook open
'and has hidden sheets
'Bob Phillips  October 4, 2009

Dim N As Variant
    With Application.Caller.Parent
        N = .Index
        Do
            If N = 1 Then
                PrevSheet = CVErr(xlErrRef)
                Exit Do
            ElseIf TypeName(.Parent.Sheets(N - 1)) <> "Chart" And _
                .Parent.Sheets(N - 1).Visible = xlSheetVisible Then
                PrevSheet = .Parent.Sheets(N - 1).Range(Rg.Address).Value
                Exit Do
            End If
            N = N - 1
        Loop
    End With
End Function

Usage in your case on activesheet enter in B1   =A1 + PrevSheet(A1)

Gord

amit tandon say:

Another option - UDF. Enter below code in a standard module:

Function PrevSheet(rCell As Range)
Application.Volatile
If rCell.Parent.Index > 1 Then PrevSheet = Sheets(rCell.Parent.Index - 1).Range(rCell.Address)
End Function

In worksheet, enter as below:

=A1+PrevSheet(A1)

This returns the SUM of current sheet A1 and previous sheet A1;

If you enter as below:

=PrevSheet(OFFSET(Sheet2!A1,2,))

This refers the previous sheet of "Sheet2" - cell A3

joeu2004 say:

The 0.00% that you see for May-21 through Sep-22 is not really zero.  If you format the cell as General, you will see 2.98E-09.

In my experience, that is just another error state, like #NUM and #DIV/0, that means that Excel XIRR is unable to determine a valid IRR for some reason.

And like #NUM and #DIV/0, sometimes it helps to provide a "guess" parameter.

Indeed, if we use a "guess" of -10%, Excel XIRR returns -98.92% through -6.69% for Jul-21 through Sep-22.  It returns #NUM for May-21 and Jun-21.

I can only presume that you used Excel XIRR, because you neglected to show us your formula(s).  (PS: But you did mention XIRR in the posting title.  Klunk!)

Also, I can only presume that the dates are the first of each month, because you neglected to show us your dates in a form that shows the day of the month.

Moreover, Excel IRR returns valid results for starting for Nov-21, even without a "guess" parameter, in contrast to your results.  My formula is:  =(1+IRR($B$3:B34))^12-1 .  We need to use a "guess" of -30%, not -10%, for Jul-21 through Oct-21.

IMHO, Excel IRR is better to use when the cash flows occur at "regular" intervals ("monthly").  Arguably, Excel XIRR introduces "too much" precision -- more precision than the original data -- because it uses the exact differences between dates.

Also, we can use Excel NPV to validate the calculated IRR, because Excel NPV accepts negative discount rates, as it should.

Excel XNPV does not; an inexplicable design flaw, IMHO.  Instead of XNPV, we can use the following formula:

=SUMPRODUCT($B$3:B30/(1+C30)^(($A$3:A30-$A$3)/365))
