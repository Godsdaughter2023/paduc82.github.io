An item marked as MISSING means that it was not installed on the machine or that the registry got screwed up. First, try running Excel to re-register itself. Close Excel and go to the Windows Start menu and choose Run. There, enter the following and click OK.

"C:\Program Files\Microsoft Office\Office12\Excel.exe"  /regserver

Of course, change the folder path to the appropriate location. The /regserver switch instructs Excel to start up and re-write its registry keys, file associations, and other stuff. graham mandeno say:

In 22 years of using Access I confess I have never used the wizard to split a database.  However, I just tried it now in Access 2013 (15.0.4663.1000) and it worked fine, so I can't say what the problem is on your computer.

However, it is easy to split a database manually - I have done it hundreds of times:

  1. Take a backup copy of the database.
  2. Take another copy of the database and rename it as the back-end (e.g. MyDatabase Data.accdb)
  3. Open the new back-end and delete all the objects EXCEPT the tables.
  4. Open the original database (which is now the front-end) and delete all the tables (except the system tables - MSys* - if they are visible)
  5. Use External data > Import & Link > Access to link all the tables from the back-end to the front-end.

Good luck!

How about creating and autocorrect entry for each - one that replaces em with an EM dash and one that replaces en with an EN dash.

 

Then when you type em and press the spacebar, it will be replaced by an EM dash and likewise en followed by the space bar will be replaced by an EN dash.

 

To create an autotext entry, insert the required symbol into the document, then select it and then use File>Options>Proofing>AutoCorrect options and you should see the symbol in the With: control.  Enter the text to be replaced by that symbol in the Replace: control and then click on OK.

suzanne s. barnhill say:

The shortcut Alt+Hyphen is not available, but you should be able to assign Alt+Ctrl+Hyphen. I would, however, suggest avoiding shortcuts that involve the hyphen, as Ctrl+Hyphen is already assigned to Optional Hyphen and Ctrl+Shift+Hyphen to Nonbreaking Hyphen. Most dangerously, in Word 2003 and earlier, Alt+Ctrl+Hyphen is assigned to ToolsCustomizeRemoveMenuShortcut, the "destructive hyphen" that removes any menu item you click on.

Assuming the table has Name in A1, enter the following formulas:

In D2:    =MIN(MAX(25-MAX(B2,18),0),C2)

In E2:    =MIN(MAX(33-MAX(B2,26),0),C2-D2)

jp ronse say:

Hi,

As an alternative on Hans' formulas and in case you want to change the age bands without adapting the formulas:

D2: =MIN(MID(D$1,FIND("-",D$1)+1,2)-MEDIAN(LEFT(D$1,FIND("-",D$1)-1),MID(D$1,FIND("-",D$1)+1,2),$B2),$C2)

E2: =MIN(MID(E$1,FIND("-",E$1)+1,2)-MEDIAN(LEFT(E$1,FIND("-",E$1)-1),MID(E$1,FIND("-",E$1)+1,2),$B2),$C2-SUM($D2:D2))

You can drag the formula in E2 to F2.

What it basically does is splitting the band, say 18-25 into 18 & 25. Now MEDIAN of 18,25,32 returns always the value in the middle. In this case 25 and 25 - 25 is 0. Of course it cannot be more then the years of service.

Another advantage of this approach is that you can easily add a band.
