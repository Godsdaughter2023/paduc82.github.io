linking excel tables in a word document - Microsoft Community

krisyen say:

linking excel tables in a word document

We use a word document that has several excel tables linked to it.  When the 2 documents are emailed, all the links are seem to get removed & we are unable to update the links to the excel document.  We want to be able to send the docs to another person, allow them to make changes in the excel file & then update all of the links throughout the word document.  How do you do this?
scjr say:

In order for Word to identify the Excel file, it must know the location of the Excel file.  The following should help resolve the problem. Since you are emailing both files to a new user, this user must first save both files to a common folder.  Now, with only the Word file open, the user should click on Office button > Prepare > Edit Links to File.  This will bring up the Links dialog box.  Now, select all the links (click on the first link, hold down the Shift key and then click on the last link, so that all the links are highlighted).  Now, click the Change Source button,  navigate to the folder where the Excel file is located, select the Excel file and click on "Open".  Repeat for each link.  After the source has been changed for all the links, select all the links again and click Update now.  Your links in Word should now point to the Excel file on the new user's computer.  After this user returns the file, you will need to repeat the process.

Unfortunately, sometimes when emailing files, the links do get broken, especially with *.dotx files.  Then you need to relink the tables. Best practice is always to open just the Word file when attempting the above steps, as at times Word will think you are trying to open two versions of the file and will not allow you to update the links or change sources.

ken sheridan say: You don't need correlated combo boxes for this, but a bound combo box and an unbound text box.  Set up the combo box as follows:

Name:                cboEmployee

ControlSource:    EmployeeID

RowSource:        SELECT E1.EmployeeID, E2.FirstName & " " & E2.LastName,
                          E1.FirstName & " " & E1.LastName,
                          FROM Employees AS E1 LEFT JOIN Employees AS E2
                          ON E1.SupervisorID = E2.EmployeeID
                          ORDER BY E1.LastName, E1.FirstName;

BoundColumn:    1
ColumnCount:    3
ColumnWidths:   0cm;0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert them to inches.  The important thing is that the first two dimensions is zero to hide the first two columns.

Set up the text box as follows:

Name:                   txtSupervisor
ControlSource:      =cboEmployee.Column(1)
Enabled:                No
Locked:                 Yes

the Column property is zero-based so Column(1) is the second column.

Not all phones restart this way, so check with your phone's manufacturer to find out how to force a restart.


If your phone has a replaceable battery (it's often under your phone's back panel), you can also restart your phone by removing and reinserting the battery, and then pressing the Power button to turn on your phone.
