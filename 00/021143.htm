MS Word Fields: IF with SEQ, conditional evaluation doesn't - Microsoft Community

cab555 say:

MS Word Fields: IF with SEQ, conditional evaluation doesn't work as expected

If have a document where I use fields to number steps in a multi-level fashion:

(Section.Step) -- Note, I'm not referring to Word's built in definition of a section in the document

1.1

1.2

1.3

I accomplish this by using { QUOTE "{ SEQ sec }.{ SEQ step }" }. This gives the two values separated by a period, and then lumps them together as one field. Some may ask why I don't use one of the other numbering fields like AutonumLgl or AutonumOut or ListNum. The reason for this is that SEQ allows me to reference the "current" number in the sequence without advancing the counter. For example

Step { QUOTE "{ SEQ sec }.{ SEQ step }" }

-> Step 1.1

This is some text in Step { QUOTE "{ SEQ sec \c }.{ SEQ step \c}" }

-> This is some text in Step 1.1

Step { QUOTE "{ SEQ sec \c }.{ SEQ step }" }

-> Step 1.2

This self reference can't be done with the other numbering fields without using bookmarks; not something I want to have to do unless referencing a step out of order.

My problem is now how to advance the list. If I use the current method, my numbers will increment after the period unless I explicitly tell it to increment the section. For example:

1.8

1.9

1.10

1.11

That's fine when I want a Section (1) to have as many Steps (11) as needed. But sometimes, I want the following numbering scheme:

1.8

1.9

2.0

2.1

This follows a decimal numbering system. I ALMOST figured it out using the IF field. Here's my code:

{ IF { SEQ step \c } = 9 { QUOTE "{ SEQ sec }.{ SEQ step \r 0 }" } { QUOTE "{ SEQ a \c }.{ SEQ b }" } }

Interpretation:

If the current Step is 9 (e.g. 1.9) then return the field of the next Section and the Step reset to zero (2.0).

If the current Step is not 9 (e.g. 1.5) then return the field of the current Section and the next Step (1.6).

This makes sense in the code. The problem is that the instructions to be executed when the condition is evaluated are evaluated every time! In other words, when { SEQ step \c } = 9, you'd expect the first response to be executed (the TrueText as Word calls it). But word actually executes both of them, but only displays the appropriate one.

Why is this a problem? Well, every time the field is executed { SEQ step } it increments the counter. The IF field should NOT BE executing the instruction that doesn't apply to the if statement. But since it does, instead of

1.8

1.9

2.1

2.2

I get:

1.8

2.1

3.1

4.1

This is because the IF field evaluates everything, not just the value that IF chooses. Let's take this example.

1.8 -> Section 1, Step 8

Is the current step = 9? IF evaluates the TrueText: { QUOTE "{ SEQ sec }.{ SEQ step \r 0 }" } and increments the Section sequence and resets the Step sequence, giving -->> 2.0

Then it evaluates the FalseText: { QUOTE "{ SEQ a \c }.{ SEQ b }" } and keeps the current Section sequence and increments the Step sequence, giving -->> 2.1.

The IF only returns 2.1, the False text, which is the correct field to return. But since the TrueText was evaluated anyway, the values in the sequences have changed, even if that arm of logic didn't apply. Can you tell I'm frustrated?

Is there any way around this?

a. user say:

> Can you tell I'm frustrated?

Yes, it's difficult.

If you can live with 

1.1
1.2
.
.
1.9

2.1
2.2


you can begin with 

{ SEQ X }.{ SEQ Y }

then continue with 

{ SEQ { IF { SEQ Y \c } = 9 X Z } \h }{ IF { SEQ Y \c } = 9 Y Z } \r \h }{ SEQ X \c }.{ SEQ Y }

The difficulty with adapting that so that Y restarts at 0 instead of 1 is that you can't reset a SEQ number to -1, so to output the "0" using something similar you would have either to use { SEQ Y \r{ something that sets the correct number, starting with 0 } or something like { SEQ Y \r0 \h }{ SEQ Y \c } and in either case you soon get back to the problem you have already described.

But I think this works:

Put the following at the beginning:

{ SEQ Y1 \r0 \h }{ SEQ Y2 \r1 \h }{ SEQ Y3 \r2 \h }{ SEQ Y4 \r3 \h }{ SEQ Y5 \r4 \h }{ SEQ Y6 \r5 \h }{ SEQ Y7 \r6 \h }{ SEQ Y8 \r7 \h }{ SEQ Y9 \r8 \h }{ SEQ Y10 \r9 \h }

Put this at the start of the numbering:

Use this for each number:

{ SEQ { IF SEQ Y \c } = 10 X Z } \h }{ SEQ { IF SEQ Y \c } = 10 Y Z } \r0 \h }{ SEQ X \c }.{ SEQ "Y{ SEQ Y }" \c }

I haven't spent much time on that so simplification may well be possible.

I doubt it makes any practical difference, but after looking at how Word actually updates its IF fields, I've come to the conclusion that it isn't the IF field execution that is doing the damage. I really don't think the IF field does execute branches that it doesn't need to. I think what happens is that Word has a completely separate algorithm for executing SEQ fields, that it does not take account of whether or not a field is nested in another field such as an IF, and that that's the real origin of this problem. You can test that using e.g.


{ SET A "before" }{ IF  1 = 1 { SEQ B } { SET A { SEQ B } } }


B is "2" at the end of the IF , but A still has its pre-IF value.

Incidentally, any attempt to make any of the numbers optional, so that you have something logically like

IF <some condition> <number> paragraph ELSE nothing


where you still end up with the continuous <number> sequence 1.0, 1.1 etc., exacerbates this problem.

How do I change the default font size in visio 2013 - Microsoft Community

meaniebrown say:

How do I change the default font size in visio 2013

How do I change the default font size in Visio 2013? Right now I have to go to each shape, highlight the text, and use the dropdown.
avej say: You can modify the text size in Style that you are using for the shapes.
You can see drawing explorer by selecting "Drawing Explorer" check box from DEVELOPER tab.

In drawing explorer you can see the Styles tab, and there you can modify the text size.

Detailed steps-
Setup:
------------------------------
1. Keep added "Style" command in ribbon (File -> Options -> Customize Ribbon -> Chose "Commands Not in Ribbon" -> Style) This will add a drop down for shape style as shown below-
2. Keep Drawing Explorer open (DEVELOPER -> check "Drawing Explorer" box)


Steps:
------------------------------
1. In Drawing Explorer, Expand Styles
2. Right click on Style that you are using for the shapes and select Define Style...
3. Click on Text to modify text property like size

Now all the shapes using the modified style, should respect the changes.

Workaround- Sometimes, if the changes made in style are not getting reflected, then you just select the shapes and reselect the style from "Style" command drop down box.

Hope this will help!
