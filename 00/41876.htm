considerate_guy say:

You were probably using a "classic" version of Movie Maker either on Windows XP or Windows Vista.  When Windows 7 was introduced, Microsoft created a new version of Movie Maker as part of Windows Essentials.  While the functionality is similar, there are some visual differences between the newer version and the earlier version.

In particular, there is no separate Collections area for previewing clips and making splits and trims.  Instead, that is all done directly on the storyboard/project area.

Another difference is the position of the preview window which is now on the left rather than the right.  A final difference is the storyboard/project which is not a traditional timeline anymore but more of a wrapping hybrid between a storyboard and a timeline.

The newer version has also added features for uploading directly to sharing sites, as well as saving in either .mp4 or .wmv formats where the old version only allowed .wmv.  It also has the ability to position the text for your title as needed, rather than being part of a pre-defined title animation.  There are also expanded options (themes) for creating an AutoMovie where the older version had only one AutoMovie option.

Again, in the latest version, you can achieve most (or more) of the things you could with the older version.  However, if you are truly wanting to use the older version, you can load it side-by-side with the new version.  They go in different folders, so you can actually use either one depending on the needs of your project.

suzanne s. barnhill say:

That is, in fact, the way Word saves files. According to https://support.microsoft.com/en-us/help/211632/description-of-how-word-creates-temporary-files:

A simplified view of the scheme used to save an edited file 


Create temp file                     Create ~wrdxxxx.tmp
Write temp file                      Save example data to ~wrdxxxx.tmp
Delete original file                 Delete EXAMPLE.DOC
Move temp to target name             Move ~wrdxxxx.tmp to Example.doc

Word gains significant performance speed by placing the temporary file in the same directory as the saved file. If Word placed the temporary file elsewhere, it would have to use the MS-DOS COPY command to move the temporary file from the other directory to the saved location. By leaving the temporary file in the same directory as the saved document file, Word can use the MS-DOS MOVE command to quickly designate the temporary file as the saved document.

The filenames given here don't match what you're seeing, but perhaps the nomenclature has changed since this article was written (though it was updated in April 2018).
