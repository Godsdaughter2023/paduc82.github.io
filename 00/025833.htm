karl timmermans [outlook mvp 2012-2018] say:
POP email issue fixed - see first post in the following thread

FIXED 3/4/16: Problems with POP accounts in Outlook 2016: Email is deleted from server or duplicated in Inbox
http://answers.microsoft.com/en-us/office/forum/office_2016-outlook/fixed-3416-problems-with-pop-accounts-in-outlook/79e8177f-b04e-491e-9a82-ae5b6565d3c0 

Pay special attention to the following in MS KB 3145116 - https://support.microsoft.com/en-us/kb/3145116

Note: After the fix is applied, Outlook may download all email one final time. If this occurs and email is duplicated in your Inbox, use the method described in the "More Information" section.

The operative word in the above is "may". If you have a lot of data in your <Inbox> and want to take some additional precautionary steps avoiding newly downloaded dupes as an alternative to what's suggested - the process is as follows <before> connecting to the POP account the first time with the updated build

#1 - Create a new mail folder
#2 - Highlight your Inbox
     --> move focus to <Inbox> items
     --> CTRL+A to highlight all items
     --> right-click 
     --> select <Move> 
     --> Other folder 
     --> select the newly created folder
#3 - If the update results in everything being downloaded again, you will have a clean inbox without dupes. If the items are not downloaded, then, you will need to reverse the process and copy everything back to the <Inbox> 

Also, for those using <rules> to move inbox items to other folders, you will probably want to turn those off during the first connection to your account in the event that everything gets downloaded again - else you will have dupes in all the other folders.

             j.e. mcgimpsey  say:
            
            
            
            
Which platform you're using doesn't matter - circular references work the same in both Win and Mac XL.





You don't give a lot of information here... does this happen in every workbook?




If not, what is the formula that gives you the circular reference, and what cell is it entered in?




Since cells that have values, rather than formulas, cannot cause a circular reference, if this only happens when you enter a negative number, I'd look for a IF() formula that only calculates one of the results if the test values is negative. For instance,
 if you enter this formula in cell A1





       =IF(B1 < 0, A1 + 2, A2 - 3)





XL won't give you an error as long as B1 >= 0, because it will only calculate the second (false) term, A2 - 3. But if B1 becomes < 0, XL will try to calculate the first term, A1 + 2, and won't be able to because it references the same cell as the one the
 formula is in.

Glad to hear the issue is resolved by unchecking the “Hide from address list” option.


Based on my experience, the reason why the issue is resolved by unchecking the “Hide from address list” option might be that the “Offline global address list” is not synchronized with the “Global address list” in time. As the shared mailbox was previously
 a user mailbox, after you converted it back to shared mailbox, the Offline global address list is still trying to email the user mailbox instead in the Outlook client. This can explain why the issue happened only in the Outlook client rather than Outlook on
 the web.
