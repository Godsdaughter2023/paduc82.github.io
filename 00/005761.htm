j.e. mcgimpsey say:
One way would be to create a template file with the 125% zoom setting, and always use the Template Gallery to start a new workbook using that file.  However, that won't change the file that opens if you start Excel from the dock.

You can also set your default workbook settings by establishing a default workbook...

Create a new workbook. Lay it out however you want: zoom, # of sheets, formatting, headers & footers, etc.

Save it as a template (.xltx) file named

  Workbook

 to the 

    HD:Applications:Microsoft Office 2011:Office:Startup:Excel:

folder. Navigate to that folder in Finder, select the file and type CMD-i to open the info dialog. Uncheck the Hide Extension checkbox, then delete the .xtlx extension in the Name & Extension input box. Click Remove when asked. What you want to end up with is a template named Workbook with no extension.

Close and reopen XL. Your new workbooks will now be based on Workbook.

Similarly, you can create a one-worksheet template named "Sheet" (no quotes, no extension) and put it in that folder, and whenever you insert a new worksheet, it will be based on that template.

aidan say:

Split a word file into separate files after every 10 pages

I'd appreciate it if someone can supply me with macro code that will split a long file in to separate files after every 10 pages. The first file would be called Contract 1, the second Contract 2 etc... The document has 2,260 pages so would ultimately need 226 separate files.

 

Thanks,

 

Aidan.

macropod say:

The following 'generic' macro splits a large document into multi-page blocks, the size of which are determined by the user.


Sub DocumentSplitter()
' Split a large document into multi-page blocks
Dim iSplit As Long, iCount As Long, iLast As Long
Dim RngSplit As Range, StrDocName As String, StrDocExt As String
With ActiveDocument
  iSplit = InputBox("The document contains " & .ComputeStatistics(wdStatisticPages) & " pages." _
    & vbCr & "What is the page block count for splitting?", "Document Splitter")
  StrDocName = .FullName
  StrDocExt = "." & Split(StrDocName, ".")(UBound(Split(StrDocName, ".")))
  StrDocName = Left(StrDocName, Len(StrDocName) - Len(StrDocExt)) & "_"
  For iCount = 0 To Int(.ComputeStatistics(wdStatisticPages) / iSplit)
    If .ComputeStatistics(wdStatisticPages) > iSplit Then
      iLast = iSplit
    Else
      iLast = .ComputeStatistics(wdStatisticPages)
    End If
    Set RngSplit = .GoTo(What:=wdGoToPage, Name:=iLast)
    Set RngSplit = RngSplit.GoTo(What:=wdGoToBookmark, Name:="\page")
    RngSplit.Start = .Range.Start
    RngSplit.Cut
    Documents.Add
    Selection.Paste
    ActiveDocument.SaveAs FileName:=StrDocName & iCount + 1 & StrDocExt, AddToRecentFiles:=False
    ActiveWindow.Close
  Next iCount
  Set RngSplit = Nothing
  '.Close Savechanges:=False
End With
End Sub

As coded, the output documents are given the same name as the input document, plus an index #, so the simplest way of getting all your output documents to have the 'Contract' name would be to save the input document with that name beforehand.
