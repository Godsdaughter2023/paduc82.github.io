I suspect you're misunderstanding what "pitch" is because 3.2mm is surely too small for a label. "Pitch" is not the distance between the labels but the distance from the top (or left side) of one label and the top (or left side) of the next. If there is no space between the labels, then the vertical pitch will be the same as the label height, and the horizontal pitch will be the same as the label width, though even 9.5mm (less than 1 cm) would be very small for either of those.

In general, even if Word appears to be rounding numbers, internally they are not rounded. My understanding is that, internally, all measurements are set in "twips." pcmk say:

2010 has an AfterSave event - and particularly useful it is too.

It's format is :

Private Sub Workbook_AfterSave(byVal Success as Boolean)

etc.

 

However, with thanks to ExcelWilbur on VBForums the following code will do the same job in 2007 (and presumably 2003, although I haven't tested that yet.

Private Sub Workbook_BeforeSave(ByVal SaveAsUI As Boolean, Cancel As Boolean)


' BeforeSave code here 

' Then this
          Cancel = True 'cancels the users original save command request
          Application.EnableEvents = False 'stops beforesave event from re-running 
          If SaveAsUI = True Then
                cFile = Application.GetSaveAsFilename(ActiveWorkbook.Name, "Excel 97-2003 Workbook (*.xls),*.xls")
                If cFile <> False Then
                       ActiveWorkbook.SaveAs Filename:=cFile
                End If
           Else
                ActiveWorkbook.Save 'saves workbook just like user wanted
           End If

          ActiveWorkbook.Save 'saves workbook just like user wanted
          Application.EnableEvents = True

' and finally .........
' After Save code here


End sub

I've added the bit to test whether it's Save or SaveAs, but otherwise as ExcelWilbur created/posted.
