john - project say:

Elasticia,

The only way I can see a way to do what I think you want is with a further extension of cross-project links. For example, create a milestone in the top level plan that you want to reference and then link that to the appropriate task in the subproject.

However, here's the issue you need to understand. Linked structures in Project are prone to corruption, whether it's cross-project links, master/subproject, or resource pool/sharer. None of the files must ever be moved, renamed, overwritten or "saved off" to another location. All files should reside in a single directory on a local drive and not be operated over a network.

Okay, so what are your other options. Assuming your cross-project links between team files are working and you obey the rules stated above, you could create a top level master that is unlinked but yet contains all the tasks for all subprojects. That master can then be filtered to show only the level 2 or 3 tasks. The downside is, you cannot simply create a dynamic master and then un-link each subproject because the cross-project links will not be preserved. In order to do that you need a macro. I do have a macro to do that but it is not freeware. If you are interested, I can give you more information.

Hope this helps.

john - project say:

Elasticia,

Something concerns me in your second paragraph. Are you saying that when you open the master you see ghost tasks (i.e. grey tasks that represent external dependencies)? If so, then it looks like your structure is corrupt. Why? When you open a dynamic master, Project will open each subproject in the background. In doing so, all cross-project links between subprojects are activated and therefore will show up as dependency link lines in the master. However, if ghost tasks are present in the master, it indicates there is a link fragment to a task in a file that was saved off to another location or improperly un-linked before deletion.

The only saving grace is if you have one or more subprojects that are NOT included in the master and they have cross-projects links to subprojects that ARE included in the master. Then you could have ghost tasks appearing in the master. This assumes you have the options checked to show external predecessors and successors (File > Options > Advanced group > Cross-project linking options).

That's the potential bad news. The good news is that you can easily show subproject unique IDs in the master by creating a custom field in each subproject (e.g. Number1). Simply copy the unique ID column into the custom field at subproject level. Then display that custom field at master level. And yes, if you add a task in a subproject, you will need to update the custom field to pick up the new unique ID for that task.

Hope this helps.

John
