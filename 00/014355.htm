The purpose of the regsvr32.exe program is to add entries to the Windows registry to tell programs where to find code stored in a dynamic link library or DLL (that is, to "register" it). In order to do its work, you have to tell regsvr32 the path and name of the DLL file.

Do you know which DLL you're trying to register, and what folder it's stored in? Why are you trying to do this?

Something in your Windows startup is trying to run regsvr32.exe, but (whatever it is) it isn't including the name of the thing it wants to register. Because the command is incomplete, regsvr32.exe is displaying the error message. The solution is (presumably) to get rid of whatever is trying to run regsvr32.exe. The "clean boot" article http://support.microsoft.com/kb/929135 shows how to start the msconfig.exe program. If you find regsvr32.exe there, clear the check box to the left of it, click OK, and reboot the computer (a regular start, not a clean boot).
