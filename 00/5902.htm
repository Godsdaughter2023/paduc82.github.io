cbarnhorst say:

No, a roll back restores your device to its previous state.  It is not a clean install.  However, you must backup your data first because this is pre-release (beta) software and anything can happen.  The roll back will take you back one step so if 16353 is the first preview installed after you changed your settings you should be back to the build you were on at that time, whatever that is.  

You should not be using preview builds of any type on your primary (every-day) device.  Only use previews on a test box or in a virtual machine (both require separate licenses).  Then you are not placing important data at the risk of bugs in a preview.  Some users recommend creating a second partition on your primary device and installing previews on the second partition (dual-booting) because that does not require a separate license.  I am very cool about that concept because some bugs can render the device unusable, not just the partition.  If you are not in a position to isolate previews to another device then you really aren't set up to participate in the program.

rich michaels - mvp (greatcirclelearning.com) say:

Hi Benj,

Below is VBA code that you can use with Word and control the zoom settings as you would like. The code also automatically sets a number of other parameters that I believe always should be enabled when you are writing.

Following the code is a screen clip showing you where you paste the code into your Normal template's "ThisDocument" class using the Visual Basic Editor, which you open from the Developer tab on Word's ribbon.

I hope it helps.

Option Explicit

'Created by Richard V. Michaels, Microsoft Word MVP

'These are VBA subroutines for you to add to the "ThisDocument" Class of Microsoft Word's

'Normal.dotm template. They automatically establish the Word preference settings for a

'professional writing environment. They function with Word 2011 and 2016 on the Mac

'and Word 2007, 2010, 2013, and 2016 on the PC.

'On a PC they work 100% of the time and on a Mac about 99% of the time. The one situation

'on a Mac where they do not work is if Word is open but no documents are open. In this

'scenario if you click on the Word icon in your dock, a new blank document is created by

'Word but the AutoNew, AutoOpen, and AutoExec routines do not execute. I don't know why

'and assume it is a shortcoming in the Word 2011 and 2016 Mac versions. If you use Word's

'menu bar instead and choose File > New, the AutoNew routine does execute and your

'special writing environment settings will be properly established.

'To obtain other products I have available for professional level writers and editors

'visit my publisher's website at: http://www.greatcirclelearning.com

Dim doc

Sub AutoExec()

    'what is contained in an AutoExec subroutine only executes from a globally loaded template

    'this means the template is either Word's Normal.dotm template or a custom template

    'of your own that you have placed in Word's Startup folder

    Application.OnTime Now + TimeValue("00:00:02"), "SetWritingEnvironment",

    'the above command will cause the SetWritingEnvironment subroutine to execute 2 seconds

    'after Word opens. Adjust the time if you find that your system is slower and more time is needed

End Sub

Sub AutoNew()

    'this only executes if you select from Word, "File > New" and start a new blank document

    'it does not execute if you start a new document from a template that also contains an

    'AutoNew routine. The other template's AutoNew routine executes instead

    Application.OnTime Now + TimeValue("00:00:02"), "SetWritingEnvironment",

End Sub

Sub AutoOpen()

    'this only executes if you open a new document either from Word > File > Open or

    'from File Finder on a Mac or File Explorer on a PC. There is no restriction if the other

    'document has an attached template with and AutoOpen routine... this one will work also.

    Application.OnTime Now + TimeValue("00:00:02"), "SetWritingEnvironment",

End Sub

Sub SetWritingEnvironment()

    On Error GoTo errHandler

    Set doc = ActiveDocument

    If doc.RemovePersonalInformation = True

        If MsgBox("The remove personal informartion is set on this document. Do you want to remove it?", vbYesNo, "Microsoft Word") = vbYes

            doc.RemovePersonalInformation =

        End If

    End If

    'add or edit any of the following settings to make the writing environment settings

    'satisfactory for your needs.

    

    If doc.TrackRevisions = True Then doc.TrackRevisions = False

    If doc.ShowRevisions = False Then doc.ShowRevisions = True

    

    If AutoCorrect.ReplaceText = False Then AutoCorrect.ReplaceText = True

    If Not Application.Browser.Target = wdBrowsePage

    If doc.ActiveWindow.DisplayRulers = False True

    If doc.ActiveWindow.DisplayVerticalRuler = False True

    With doc.ActiveWindow.View

        If .ShowBookmarks = False Then .ShowBookmarks = True

        If .ShowFieldCodes = True Then .ShowFieldCodes = False

        If .ShowAll = False Then .ShowAll = True

        If Not .FieldShading = wdFieldShadingAlways

        If .TableGridlines = False Then .TableGridlines = True

        If Not .Type = wdPageView Then .Type = wdPageView

        'Comment out one or the other of the next two lines. The first automatically adjusts the page zoom

        'so that it fills the open window space. The second alternative is to automatically adjust the zoom

        'to a fixed percentage.

        If Not .Zoom.PageFit = wdPageFitBestFit

        'If Not .Zoom.Percentage = 70 Then .Zoom.Percentage = 70

    End With

    'the reason for putting a check for opposite on all of the above settings is so the code will not

    'update preference files and the normal template unnecessarily. However, that is still not

    'enough for the new blank temporary document that Word first opens and so I've included the following:

    doc.Saved = True

    'it stops the save message prompt if you close the blank document and quit Word

' end default settings

errHandler:

    Select Case Err.Number

        Case 0

            'Once you know this routine is running (exactly when you want it to on your machine)

            'remove the following Message Box command

'            MsgBox "Writing Environment Now Established" & vbCr & _

                "http://www.greatcirclelearning.com", vbInformation, "Message from Rich Michaels, Word MVP"

            DoEvents

        Case Else

            MsgBox "Error Code: " & Err.Number & vbCr & _

                Err.Description, vbExclamation, "Set Writing Environment Failed"

            Err.Clear

    End Select

End Sub
