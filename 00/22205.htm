mailing labels: multiple names, one address, one label - Microsoft Community

diana_6871 say:

mailing labels: multiple names, one address, one label

Good morning.  I have a table of approximately 400 names (members) for a non-profit organization.  Three different member types.  Some members reside at the same address.  I send a newsletter monthly.  I need to make labels that print all names on the label- but just one label per address.  I know how to generate 'labels'- but, I can't figure out the rest.  Your efforts are appreciated.  Thank you.  Diana
ken sheridan say: You could use one of the many 'concatenation' functions available on-line to tack the names per address together, but a simple solution in this case would be to base the labels report on a query which returns distinct values of each address, e.g.

SELECT DISTINCT [AddressLine1], [AddressLine2], [City], [State], [Zip]
FROM [MembershipTable];

Then add a little subreport to it immediately above the AddressLine1 control.  Base the subreport on another query like this:

SELECT [FirstName] & " "  [LastName] AS Fullname, [AddressLine1], [AddressLine2], [City], [State], [Zip]
FROM [MembershipTable];

Make it one line deep, and include a text box bound to the FullName column.  Allow the subreport control in the parent report to grow.

Link the parent report and subreport by setting the LinkMasterFields and LinkChildFields properties each to:

[AddressLine1];[AddressLine2];[City];[State];[Zip]

This should give you a layout like this:

Joan Doe
John Doe
123 Main Street
Some Neighbourhood
Some City
Some State
12345

Each line of the address must be exactly the same for each occupant for this to work.  Typos will almost certainly occur (Murphy's Law dictates this!)  resulting in occupants being separated over two slightly differing addresses.  This really results from a design flaw in having the same addresses in the one table.  Ideally therefore the table should be decomposed into Members and Addresses tables (along with 'look-up' tables for Cities and States) with each row in the Members table including an AddressID foreign key column referencing the primary key of the Addresses table.

    Try following one by one and see after every step whether it resolves the issue-


    Try repairing your office installation (Quick repair is good enough). Try repairing your Office installation. First try Quick Repair (takes 5 - 10 mins) , if that doesn't help then try Online Repair (Takes more than 30 mins but less than 1 hour in general)

    https://support.office.com/en-us/article/Repair...

    If repair doesn't work -

    1. Sometimes, add-ins can cause problems. To determine, if add-in is a problem start your application in safe mode and see if problem has gone away.

    To start in Safe Mode -

    Hold CTRL key and click on application icon and don't release the CTRL key till it asks you for Safe mode confirmation.

    If yes, then start application normally and disable add-ins one by one and start application again every time you disable an add-in to determine the culprit add-in.
    To disable add-in > File > Options > Add-ins > Down below you have Manage: Com Add-in : Press Go > Untick the Add-ins to disable and tick to enable them

    2. Login to Windows with a new user profile and see if problem is resolved.

    3. Sometimes, this is caused due to application not being able to access the default printer. Close all office applications and change your default printer through Start > Devices and Printers to another printer. It may be some physical one or One Note or XPS or PDF or anything else. If this is the problem, then you will need to update your printer driver for default printer.

    4. Start your computer in Safe mode with networking and see whether the problem is solved. If yes, then some program is causing the problem. Many third party applications also cause the problem like Tuneup Utilities, Abby Finereader, Kaspersky / Avast Anti Virus etc. See if you have any of them installed. Close / Disable them one by one and see whether it helps.
    https://support.microsoft.com/en-us/help/12376/...
