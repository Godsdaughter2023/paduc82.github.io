Thank you for posting to the forum regarding error code 80070490.  This error indicates there is an issue with syncing your Microsoft account.  There are several reasons this may occur, a bad password, bad calendar entries, corrupted or oversized emails, account locks, etc...

 

The first thing I would do is to verify that the password being entered is correct.  We suggest you start this process by changing the current password associated with the Microsoft account that is not syncing.  You can do this by going to http://accounts.live.com and signing into your account.  When you go to the security info. section you will have the option to change the password.  Be sure that the password you create is no longer then 13 characters.  Once you have set up a new password, go to the phone and tap settings > emails + accounts > tap the account that is giving you trouble > tap the password field, enter the new password.  Remember that passwords are case sensitive.

 

If you continue to have trouble with this account syncing, check the Xbox account associated with the account on your phone.  Go to http://xbox.com and sign into the primary email on your phone.  Once you are signed in, click on MY ACCOUNT > billing and contact information.  Make sure that the details are complete and up to date.  If you are prompted to accept the terms of use at any point, do so.  After you have updated this, go back to the phone and tap settings > emails + accounts > tap and hold the primary account > select sync.

 

If the error continues to appear on the phone, the next thing to check is your calendar and your inbox.  Go to your PC and sign into http://calendars.live.com.  Once you are signed in, create a new calendar and make this your primary calendar.  Once you have done this, go back to the phone and sync your account.  If you receive the same error code, go back to the PC and sign into your email.  Go to your inbox and look for the largest emails over the past month.  Move these to other folders or delete them altogether.  Once this has been, go back to the phone and sync your account again.

 

If you have issues doing any of the above, detail for us exactly what you experienced when you tried.  Please let us know if you have any other questions or concerns.

dominik hoefling say:

Hi Timo,

if you use a cutover migration you cannot use AADConnect.

You can't get rid of your on-premises Exchange server unless you have no dependencies on your local Active Directory or domain controllers.

As Brook said, you cannot change any Exchange related attributes without Exchange on-premises. And it is not supported to "remove" Exchange hybrid. That is not Exchange but DirSync / AADConnect related. Microsoft is working on a solution to remove all on-premises Exchange server.

So: you can use a cutover migration if you have don't use directory synchronization and manage your users twice (on-prem and Office 365 with maybe different logon names and passwords).

Or: go hybrid, migrate your users, change DNS records to Office 365 / Exchange Online and use your on-premises Exchange only for user management.

How to specify a default (to display) heading level in the - Microsoft Community

lmerriam say:

How to specify a default (to display) heading level in the Document Map (Word 2003)?

I would like to specify the default level (e.g. Heading 3) to display for the document map (DM) when a given document is first opened.  Currently, because the DM defaults to all levels, I'm forced to change this manually.  It would save me a lot of time and effort if there was some way to default this setting.
TIA for any help!

stefan blom say:

You will have to intercept the View | Outline command with several macros:

 

Sub ViewOutline()
With ActiveWindow.View
.Type = 2
.Zoom = 100 'or the value you want
.ShowHeading 3 'or the level you want
End With

End Sub

Sub viewoutlinemaster()
With ActiveWindow.View
.Type = 2
.Zoom = 100 'or the value you want
.ShowHeading 3 'or the level you want
End With

End Sub

Sub ViewMasterDocument()
With ActiveWindow.View
.Type = 2
.Zoom = 100 'or the value you want
.ShowHeading 3 'or the level you want
End With

End Sub

 

Save the macros in the Normal template; for assistance, see http://www.gmayor.com/installing_macro.htm.

stefan blom say:

"Intercept" in this case simply means that the code will replace what Word otherwise does when executing these commands (no matter if you click the Outline button on the View tab, or if you click the Outline button on the status bar, or if you make use of a keyboard shortcut).

However, reading you original message again I realize that I haven't actually answered your question. You are referring to the Document Map, and the workaround I offered is for Outline view. As far as I know, you can't work with the Document Map in VBA.

john korchok say:

For your copy of PowerPoint to operate in Portuguese, you must set the language:

  1. In PowerPoint, open a new blank presentation. Click on a text box or text placeholder.
  2. From the top menu, choose Tools>Language.
  3. Select Portuguese.
  4. Click on the Set As Default button.
  5. PowerPoint asks if you want to set the default language. Click on Yes.
  6. Click on OK.

Now when you create a new presentation, the spellcheck language will be set to Portuguese.

Para a sua cópia do PowerPoint para operar em Português, você deve definir o idioma:

  1. No PowerPoint, abra uma nova apresentação em branco. Clique em uma caixa de texto ou texto espaço reservado.
  2. No menu superior, selecione Ferramentas> Idioma.
  3. Selecione Português.
  4. Clique no botão Definir como padrão.
  5. PowerPoint perguntará se você deseja configurar o idioma padrão. Clique em Sim.
  6. Clique em OK.
Agora, quando você criar uma nova apresentação, o idioma de verificação ortográfica será definido para o Português.
