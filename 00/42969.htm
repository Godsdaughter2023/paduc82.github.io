Network Credentials - Windows 10 - Microsoft Community

rjacobs77 say:

Network Credentials - Windows 10

We have just upgraded our six office PC's to Windows 10 - four from Windows 8.1 and two from Windows 7.  The network settings are the same on all six machines, particularly the advanced settings for "turn off password protected sharing".  Now none of the former Win 8.1 machines can access either of the former Win 7 machines on our network because of a "network credentials" popup which, when correctly filled out, says the wrong information has been provided.  Both former Win 7 machines can access each other without any credentials popup, and both former Win 7 machines can see and access all other machines on the network.  Everything worked fine on our network prior to the Windows 10 upgrades.

I talked several hours last week via the MS Chat help service with a MS tech who finally said he had no idea what the problem was after having access to several of our PC's, and now the scheduled "level 2" return call for today has not happened.  

Fix this, Microsoft.

rjacobs77 say:

Our final and very satisfactory solutions to the network credential popup networking issue with Windows 10 are twofold.  First, make sure that the user name of the machine you are trying to connect to is correct.  We are on a Workgroup.  Using the Microsoft ID account name (email address) associated with each machine seems to work fine for that, along with same password for that account as the password.  A former poster above very helpfully noted its good to go into the Credential Manager part of setup and make sure the information is correct there.  Deleting a wrong entry can help.

We also found that putting Windows 10 on some of our Windows 7 machines changed their IP addresses and so we had to change the firewall information for those machines on our Norton Security setups.  Having done all that, networking works great with Windows 10.
