hairyfool say:

Any client requires a CAL in order to access the server. It does not matter what service they are using, they need the CAL. Some versions of Windows Server include a minimum number of CALs, Essentials is limited to 25 clients and includes the 25 CALs in its price so there are no separate standard CALs for this or for Foundation.


In a client/server environment the normal process is for clients to store their files on the server and not on the client. This gives a centralised location to then back up. In the case of laptop users who may not always be connected to the server you can make a folder available "offline" to have the client hold a cache on their computer which then synchronises when reconnected.


Windows Server normally requires separate Terminal Services Client Access Licenses to allow clients to connect via RDP however by default a server license includes the ability for 2 clients to connect for server administration.

hairyfool say:

CALs are required on those systems but the purchase price includes the CALs. Therefore as I stated they cannot be obtained separately. CALs are still required because this is how the server licensing service manages the number of clients. If it was purely done on connected users/devices then you could have any number of clients if you managed the concurrence of the connections.

Once you have issued the limited number of CALs the Admin has to reclaim them in order to add another client whereas connections themselves time out.
