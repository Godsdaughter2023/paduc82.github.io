Word Printer Tray Selection

graham mayor say:
You can do it with a couple of macros. Put the documents to be printed in a folder and run the macro 'BatchProcess'.
or to just print one create a macro with the line and run that with the document in question displayed in Word.

PrintToTrays ActiveDocument

Note that you will have to establish the tray IDs for trays 3 and 4 for the particular printer. The link in the function header below demonstrates how to do that. Substitute the numbers 260 and 261 with the actual numbers.

Function PrintToTrays(oDoc As Document)
Dim oRng As Range
Dim sTray As Long
'Set the tray ID's for the particular printer
'See http://www.gmayor.com/fax_from_word.htm
Const Tray3ID As Long = 261
Const Tray4ID As Long = 260

sTray = Options.DefaultTrayID
Options.DefaultTrayID = Tray3ID
oDoc.PrintOut Range:=wdPrintRangeOfPages, _
              Copies:=1, _
              Pages:="1", _
              PageType:=wdPrintAllPages, _
              Collate:=True, _
              Background:=True, _
              PrintToFile:=False, _
              PrintZoomColumn:=0, _
              PrintZoomRow:=0, _
              PrintZoomPaperWidth:=0, _
              PrintZoomPaperHeight:=0
Set oRng = oDoc.Range
oRng.Start = oRng.End
If oRng.Information(wdActiveEndPageNumber) > 1 Then
    Options.DefaultTrayID = Tray4ID
    oDoc.PrintOut Range:=wdPrintRangeOfPages, _
                  Copies:=1, _
                  Pages:="2 - 999", _
                  PageType:=wdPrintAllPages, _
                  Collate:=True, _
                  Background:=True, _
                  PrintToFile:=False, _
                  PrintZoomColumn:=0, _
                  PrintZoomRow:=0, _
                  PrintZoomPaperWidth:=0, _
                  PrintZoomPaperHeight:=0
End If
Options.DefaultTrayID = sTray
Application.Run MacroName:="SaveReminder.SavRemMain.RunSave"
    Application.Run MacroName:="SaveReminder.SavRemMain.RunSave"
End Function



Sub BatchProcess()
Dim strFilename As String
Dim strPath As String
Dim oDoc As Document
Dim oLog As Document
Dim fDialog As FileDialog
Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)

With fDialog
    .Title = "Select folder and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , _
               "List Folder Contents"
        Exit Sub
    End If
    strPath = fDialog.SelectedItems.Item(1)
    If Right(strPath, 1) <> "\" _
       Then strPath = strPath + "\"
End With

If Documents.Count > 0 Then
    Documents.Close SaveChanges:=wdPromptToSaveChanges
End If
'Set oLog = Documents.Add
If Left(strPath, 1) = Chr(34) Then
    strPath = Mid(strPath, 2, Len(strPath) - 2)
End If
strFilename = Dir$(strPath & "*.doc?")

While Len(strFilename) <> 0
    WordBasic.DisableAutoMacros 1
    Set oDoc = Documents.Open(strPath & strFilename)
    '
    'Do what you want with oDoc here
    PrintToTrays oDoc
    'record the name of the document processed
   ' oLog.Range.InsertAfter oDoc.FullName & vbCr
    '
    oDoc.Close SaveChanges:=wdDoNotSaveChanges
    WordBasic.DisableAutoMacros 0
    strFilename = Dir$()
Wend
End Sub

Yes subforms are a solution. And the answer to your question is a Tab control. You can place each test's subform on a separate tab. So the user just selects the Tab and sees the subform. as soon as they enter data into the form, the FK is automatically filled.

You can do it with separate forms as well. The solution there is to run an Append query before opening the form. Something like:

Dim strSQL As String

strSQL = "INSERT INTO childtable (PatientID, TestDate) VALUES(" & Me.PatientID & ", #" & Me.TestDate & "#);"

CurrentDB.Execute strSQL, dbFailOnError

DoCmd.Openform "formname",,,"PatientID = " & Me.PatientID & " AND Testdate = #" & Me.TestDate & "#"

You should be careful about using spaces in Object names, they can come back to haunt you.

and the data from each test can span anything from 5-100 fields.

I also wanted to comment on that point. Relational Database tables should be tall and thin. Not wide. If you test table has fields for each test value, that is not proper design. You should probably have 2 tables. One for data specific to the test itself (patientID, date, location, tester, etc.) and a detail table with a field for the item tested and the results. For example, if Temperature, Blood Pressure, Pulse were taken as part of the test, there would be three detail records each recording the item being measured and the measurement result.

Yes that is possible.

First, when you open a Test form, you need to pass in the PatientID from the main form, via the OpenArgs argument of DoCmd.OpenForm:

DoCmd.OpenForm "FirstTest", OpenArgs:=Me.PatientID

(I am taking liberty with your object names; change as needed)

Then in the Test form's Form_BeforeInsert event set the FK value:

Me.PatientID = Me.OpenArgs

(this assumes a (potentially hidden) field named PatientID linked to the foreign key field)

Boot with Windows 10 install media and use SHIFT-F10 to get a Command Prompt. If you do not have it, you can run the Media Creation Tool to create it on a 8GB or larger USB stick (or DVD). Booting from install media gives adequate file rights and eliminates files that might otherwise be in-use.

rd /S /Q C:\Windows\SoftwareDistribution

rd /S /Q C:\$WINDOWS.~BT

You can also run Disk Cleanup by running cleanmgr /sageset followed by cleanmgr /sagerun at an Administrator Command Prompt. Select all options when running /sageset.

doug robbins - mvp office apps & services (word) say:
If typing a letter in Word 2010 when I hit return I get double spacing, how can I correct this to get single spacing?


If it is just the end of a line that you are hitting return, it is not necessary to do that as the words will automatically wrap to the next line when the line is full.

If you are only hitting the Enter key when you want to start a new paragraph and you do not want to have a space between those paragraphs, you can format the paragraph so that it does not have space before or after it.  You can get to the Format Paragraph dialog either by using of the dialog launcher at the right hand end of the Pagraph bar on the Home tab of the ribbon or by right clicking and the selecting Paragraph from the short cut menu that appears.

If you do not want to start a new paragraph, but just  a new line, you can use Shift+Enter.

I suggest that you also read up on the use of Styles to format your text.
