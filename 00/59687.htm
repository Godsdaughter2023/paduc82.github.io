helheimr say:

Hi all,

I have been able to resolve my problem.

I can confirm that manually forcing the installation of the plain Intel drivers from the Intel website works and I used that as a workaround for some time. But, then I tried to install the Anniversary Update and it failed, see this thread: http://answers.microsoft.com/en-us/surface/forum/surfbook-surfupdate/anniversary-update-fails-restoring-your-previous/55aaef02-84ee-4379-9bdc-312c491af7a9

I have now correlated the failed Anniversary Update to this issue Error 0x800705b3 in Windows Update for the graphics card.

My solution;

  1. Make sure your SB is undocked, not using any external screen.
  2. Uninstall the Intel Display Adapter completely, select "remove drivers"
  3. Your SB will now be using the Nvidia GPU or a generic driver (NOTE! I have an SB with an additional GPU. Not sure what happens if you do this on a device that does not have 2 GPU. I think it will switch to a generic driver and still work, i.e. you will see your screen....but I'm not certain. Try this at your own risk.)
  4. Open command prompt in admin mode
  5. do pnputil -e to get a list of all installed drivers
  6. look for "Intel Display Adapter" drivers and note the "oem###.inf" names of all of those
  7. do pnputil -f -d oem###.inf for each of those "oem###.inf" names to force removal of all of the drivers, including old ones, for the Intel Display Adapters from the windows/system32/driverstore (yes, all of them)
  8. reboot
  9. now, do Windows Update as usual and enjoy it finding and properly installing the latest MS approved Intel Drivers for your graphics card
  10. Now, the update to Anniversary Update will also be successful.

I'm at loss why this is necessary, but I'm just happy my SB now works OK and all drivers are updating as expected. Hope it works for you too.
