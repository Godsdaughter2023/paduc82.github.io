dirk goldgar say:

SELECT [SW21].patch, [SW22].patch, [SW23].patch, [SW28].patch, [SW29].patch
FROM [SW21], [SW22], [SW23], [SW28], [SW29]
WHERE Not IsNull(patch);

In your WHERE clause, in the expression "Not IsNull(patch)", you need to specify which of the [patch] fields in the various tables you are talking about; e.g., "Not IsNull(SW21.patch)" -- though incidentally it would be much more efficient to say "SW21.patch Is Not Null".  Maybe you want to only include records from any of the tables where that table's [patch] field is not Null.  In that case you would need to write (I think) this:

    SELECT [SW21].patch, [SW22].patch, [SW23].patch, [SW28].patch, [SW29].patch
    FROM [SW21], [SW22], [SW23], [SW28], [SW29]
    WHERE SW21.patch Is Not Null

           OR SW22.patch Is Not Null

           OR SW23.patch Is Not Null

           OR SW28.patch Is Not Null

           OR SW29.patch Is Not Null

Or maybe you'd need to use AND instead of OR.  It's hard for me to say, because this is an odd query, in that it is going to give you a cross-product of the [patch] fields in all 5 of the tables.  Are you sure that's what you want?  Maybe you want a union query instead:

    SELECT patch FROM SW21 WHERE patch Is Not Null

    UNION

    SELECT patch FROM SW22 WHERE patch Is Not Null

    UNION

    SELECT patch FROM SW23 WHERE patch Is Not Null

    UNION

    SELECT patch FROM SW28 WHERE patch Is Not Null

    UNION

    SELECT patch FROM SW29 WHERE patch Is Not Null

That will return all the unique [patch] values that appear in any of the 5 tables.

1.      Click on Start > Type Run. In the search bar and press Enter.

2.      Now in the open text box please type “Outlook /safe” without quotes (there is a space after Outlook) and hit Ok.

3.      Now check if Outlook is working fine.

 

If outlook is working in Application Safe mode rename the .dat files:

 

Outlook has the .dat files that it uses in order to keep Outlook operating smoothly. Try RENAMING these files (you might not find the entire files. launching outlook after renaming the new .dat files will be generated.)

 

1. Outcmd.dat:

Location:    C:\Users\<username>\AppData\Roaming\Microsoft\Outlook

 

2. Frmcache.dat

Location:  C:\Users\<username>\AppData\Local\Microsoft\Forms

 

3.  Extend.dat

Location: C:\Users\<username>\AppData\Local\Microsoft \Outlook

 

4. Views.dat

Location: C:\Users\<username>\AppData\Local\Microsoft\Outlook

After the renaming check if outlook is working.

 

 

Note: View hidden files feature has to be enabled in windows 7 to search these files.

1. Open a Windows Explorer window (click on start > Select Computer)

 

2. Click on the Tools menu item. (ALT+T)

 

3. Click on Folder Options > Click on View and select show hidden files and folders.
