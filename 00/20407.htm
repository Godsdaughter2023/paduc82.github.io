scott eastman - entatech uk ltd say:

Ok this issue is now resolved - turns out that the certificate generated when setting up ADFS for the "Token-decrypting" and Token-Signing" is set by default to - CertificateDuration: 36500, within the ADFS management console. So as a result the certificates it generated had an expiration date of 2116.

So we needed to set this back to only 3 years. 

Get-AdfsProperties

Set-AdfsProperties -CertificateDuration 1095

Once this was set i then logged back into the Office 365 using PS
Enable-PSRemoting 
Connect-MsolService
Update-MsolFederatedDomain -DomainName [domain name] -SupportMultipleDomain

Certificates now have an expiration date of 2019 and that should fix the issue.
