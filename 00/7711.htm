cpitchford say: You're right it is an issue, but it's usually caused by the charger..

The charger needs to have certain pins in the USB cable connected together, so the phone knows it is connected to a "dumb" charger

If the pins aren't shorted together, the phone expects to be able to talk USB protocol to the charger to negotiate power levels and stuff.. if the charger doesn't short pins AND doesn't speak proper USB then the phone will reject it as a charger.

It's a pain because some usb chargers are too dumb as it were.. that's why you need to special cable.. but that means you can't use the cable for data (as the pins are shorted)

a. user say:

If the numbers are always integers (i.e. it's ways ".00", then you can change your mergefield field to look like this:

{ MERGEFIELD id \#0 }

Otherwise, you might be able to get away with a nested field like this:

{ ={ MERGEFIELD id } }

(select your mergefield, press ctrl-F9 to add another pair of the special field braces { } around it, use Alt-F9 to toggle between field code view and field results view, and type the "=" manually.

Post back if neither of those is what you need.
