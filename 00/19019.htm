bob jones aka: cybertaz say:

I believe the confusion may stem from you expecting the Sort operation to have the same effect as the Insert Row operation. That just isn't the case... As you point out, when you Insert a row, A7 becomes A8, but when you Sort, A7 remains A7 but its content changes.

As for "...an instant real-time view" I can't think of anything fitting that description which provides faster, more accurate real-time response than right-clicking a cell containing an * & selecting Filter> By Cell Value from the contextual menu.

If you prefer something formulaic, you might take a look at those in the Lookup & Reference category rather than a formula link.

bob jones aka: cybertaz say:

The obstacle is that a formula link (e.g., =A3) links the cell containing the formula to the cell being referred to, not to the content of that cell. When you sort the list the records are rearranged causing the content of the referenced cells to be different. Long story short :-) this isn't the approach you need to take.

It appears that what you actually want is a restricted listing of those Items which have been starred (*) as Priority. There are a number of possible ways to go about it but unfortunately my time is quite restricted right now. Perhaps another responder will jump in to offer a usable suggestion. In the meanwhile, the simplest way to produce your focused list of Priority Items is to simply Filter the main list based on the criterion of whether the Priority field (column C) contains an asterisk. If you wish, you can then copy & paste the filtered list of Priority Items. For more on using Filters see topic #4 - Filter a List in the Excel 2011 Help article Sort and Filter Lists

14942 Start menu changes - Microsoft Community

ianrg say:

14942 Start menu changes

The new new Start menu has added the option to hide the All Apps column. This is a good change and in line with discussions here and with feedback.

However, it has a problem and is a missed opportunity:

The problem

The Start menu has 3 zones - Admin/quick access, All Apps and the Pin area. The width of these are 1 icon, 1 tile (XP style icon and name) and as many 6 or 8 icon wide columns as will fit and within this limit, you like respectively. If you don't use the Pin area and delete all pinned items you can resize this to zero width. This gives just the column of Admin/quick access icons and the All apps column. Now if you hide the All apps, instead of getting just the Admin/quick access column it reinstates an empty pin area column. Presumably this is because the Start menu cannot be just a column of icons

The missed opportunity

The resulting column of icons with no names and the empty pin area highlights what is missing - the option to display the names of the Admin/quick access icons permanently. We can only temporarily see these names with the hamburger menu. If the hamburger menu acted like is does in the Mail app (i.e. as a toggle between a column of icons or a column of icon and name tiles), then if you have nothing pinned and hide All apps the Start menu could default to displaying the admin/quick access items as icon and name tiles. Take this one step further and if you do not hide the All apps and choose to have admin/quick access display icon and name tiles, then the All apps could shrink to a column of jump letters (like the grid of jump letters it already):

suzanne s. barnhill say:

Are you using labels that are created as a half sheet or designed to be used as a half sheet? If so, you should be able to find a label description that matches the labels. If not, you may have to create your own, or create a custom template. I have done this for Avery 5164, which comes in full sheets with perforations down the center so that the sheets can be separated into half sheets. I started with a new document based on the full sheet and then deleted the unwanted column of labels, centered the remaining column, and saved the result as a template. I centered the labels because my printer feeds small custom paper sizes from the center of the Multipurpose tray, using centering paper guides, but on some printers, which have only a single paper guide, the paper would have to be aligned left instead. In either case, the printer doesn't know it's not printing a full Letter-sized page.

The bottom line is that the trick is to find out where your printer wants to print on a half sheet and then set up your document such that you can feed the label sheet there. Any template that works in Word 2003 should work equally well in Word 2010 provided you are using the same printer and printer driver.
