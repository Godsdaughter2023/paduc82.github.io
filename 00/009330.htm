Has Marshal said, typically, a combobox has a bound column that represents a code. But that column is hidden. So one only sees one column. What's important here is NOT the column that you see but the bound column. If the bound column is a numeric key, then that's what you have to set as the default value. 

It is very important, when dealing with list controls, that you understand what the bound column is and what datatype the bound column (and therefore the Controlsource) is.

marshall barton say:

Typically, a combo box's BoundColumn is set to the PK field and displays another column by setting the ColumnWidths property to zero for the PK column.  The DafaultValue property must be set to the PK value for the desired row, not to the text displayed in the combo box.  If the PK value for the Expired record is a number data type, use:

   Me.cboInActive.DefaultValue = "<pk value>"

or just

   Me.cboInActive.DefaultValue = <pk value>
