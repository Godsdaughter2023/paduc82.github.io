In Outlook

rob koch say:

Typically if you select either the Cancel or "X" option in the upper right-hand corner to exit that Windows Security dialog, an additional checkbox option to not allow the site to open more of these will be displayed.

However, in testing this doesn't always appear to work, so I've most often found that pressing Alt-F4 in quick succession or holding these keys continually eventually results in an offer to close either the current or all tabs.

Note that these browser hijacks are typically nothing more than an initial script executed by something within a website, which then copies a small html file containing the page[s] displayed into a local folder on the PC.  Since this file is local it opens very fast and so is difficult to abort when it loops back to itself, making it appear to be something more malicious than it really is, which is simply annoying.

Though these attacks most often seem to happen with Edge, that's more a function of the default browser settings in Windows 10 and how they are launched via advertising or other ancillary functions of the websites where they're encountered.  The attackers also target Edge since by default it restores the last page displayed after a crash if not cleared before opening.

Since these browser hijack attacks often come through advertising networks on popular sites now, I've chosen to block these using the Internet Explorer Tracking Protection with the Easy Privacy list which operates similar to the popular AdBlock extension.  I believe that with Edge the AdBlock, AdBlockPlus and possibly other extensions are available to deal with these as well.

However, if a website is actually directly affected by one of these, the only option may be to avoid that site.

Whether at some point Microsoft will find a way to disallow these popup attacks is uncertain, since for example the login dialog may be required by some commercial websites, it's not as easy as simply removing these completely.  And since when Microsoft initially added the site blocking checkbox to the earlier modal dialog (white announcement) popups which came first, the attackers just added the more difficult to close Windows Security dialog to the mix, they'd likely simply switch to something even worse.

The issue is that if you are using JRiver Media Center, one option you have is, if you are using WASAPI, is to have JRiver take exclusive control over the audio hardware.

What happens is if you hit Start and start typing, nothing happens. If you try to click into the Cortana search box, nothing happens.

If you close JRiver down, and click into the search box, nothing happens.

The only way to get Cortana back to a responsive state is to close JRiver and use the "Hey Cortana" question option, OR open Task Manager to kill the Cortana process.

It's "Easily" fixed by removing the exclusive control clause of JRiver's WASAPI properties, but I kind of like it as it makes it so OS sounds, and others, are blocked out of music playback.
