But you said, "The MDB is installed on a server in five locations."  That sounds suspicious.  For best results, each user's copy of the front-end should be installed on that user's PC, not on a network share.  Running Access across a network is more prone to database corruption than running it locally.  There are mechanisms you can put in place to let you easily distribute an updated copy of the front-end to every user.  (These mechanisms aren't built into Access, but are readily available or can be home-grown.)

Regarding the idea of a web-facing application, Access 2013 (now out in beta preview) supports the development of web applications, but I haven't played with that yet.  I don't expect the full capabilities of  the desktop client databases, and I know that VBA won't be supported.
