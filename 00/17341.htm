The reason Skype for Business wasn't connecting was because the computer didn't think it was connected to a network, even though it was.  In the Network & Sharing Center it said that "You're currently not connected to any network".

ken sheridan say: I can't explain why your are experiencing this behaviour, but the important point is that users should not be opening a single database file simultaneously.  That's a sure-fire recipe for corruption.  For users to simultaneously work with a database it should split into separate front and back end files, the latter containing the tables only, the former containing the forms, reports queries etc, along with the links to the back end file's tables.  The database splitter wizard will split the file for you, though it's very easy to do it manually.

Once you have the file split each user should be provided with a separate copy of the front end, which can be located on their local machine or in a 'personal' folder on the system.  Users will then be able to work with the same data, but without the risks associated with opening a single file.

Access itself should ideally be installed on each user's local machine, but a single installation of Access in a shared location on the system to which all users have access works quite well in my experience.
