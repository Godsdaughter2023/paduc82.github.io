Assuming that you are entering data via a FORM.

Use the form's Before UPDATE event to check for the fields being filled in.

Private Sub Form_BeforeUpdate(Cancel As Integer)

Dim strMessage as string

If Me.Step1CheckBox Then
   strMessage = strMessage & "Date 1 must be entered" & VBCRLF
   Me.Step1DateField.SetFocus
End If

If Me.Step2CheckBox Then
   strMessage = strMessage & "Date 2 must be entered" & VBCRLF
   Me.Step2DateField.SetFocus
End If

'(Repeat for each combination of checkbox and textbox)

If Len(strMessage) > 0 then
   Msgbox strMsg,,"Data needed"
   Cancel = True
End If

End Sub

albert d. kallal say:

You not only have to “cycle” though the forms collection, but you also have to open up such forms in “design mode”, make the change, and then save the form. So “just” opening the form and changing the property will not “persist” and be saved.

The VBA code (say pasted into a standard code module – not a forms module) would look thus like this.

   Dim r       As Object
  
   For Each r In CurrentProject.AllForms
      Debug.Print "working on form " & r.Name
      DoCmd.OpenForm r.Name, acViewDesign
      Forms(r.Name).Toolbar = ""
      DoCmd.Close acForm, r.Name, acSaveYes
   Next

Regards,
