Windows 10 - Initial Experience

Windows 10 Preview - Initial Experience 10/2/2014


Hardware


Dell Venue 11 Pro (7130/7139) tablet computer with:

Intel Core I3 4020Y processor (3MB Cache, 1.5GHz Dual-Core)
4GB 1600MHz memory.
128GB SSD.
Lexar 64GB MicroSD memory card.
Lexar 64GB USB memory card. (2 – used as needed, connected  via USB 3.0 port).
10.8 inch IPS Display with FHD (1920 x 1080) resolution with 10-pt capacitive touch.
Intel GT2 Graphics.
Dell Active Stylus.
Intel(R) 7260AC Dual-Band 2x2 802.11 ac WiFi + Bluetooth(R) 4.0
Dell Wireless 5808E (LTE/HSPA/EVDO) Mobile Broadband Card - Verizon
Factory installed OS: Microsoft Windows 8.1 (64 bit).


Download


Downloaded Windows 10 Technical Preview Enterprise Edition (64 bit) image file from: 

http://windows.microsoft.com/en-us/windows/preview.

Download time < 15 minutes. File size 3.888GB.

Download was smooth and apparently error free.


Transfer of OS to USB Memory Card

Rufus 1.4.10 application program was used to create a bootable USB Memory Card and to install
the OS on this card. (http://rufus.akeo.ie/). The process was easy and was error free.


Install Windows 10 on the Dell 11 Pro.


A Dell factory backup was done to a bootable USB memory card. (Dell does not ship the Venue with any OS or other software on external media).

The Rufus generated USB Memory Card was inserted into the Venue’s USB 3.0 port.
Windows Explorer was used to display the USB files and to select and execute “Setup”.
The Windows 10 installation asks if one wishes to keep the existing application programs, drivers and other current files. The recommended answer is YES. This was an unusual experience for me because the typical requirement is to do a clean install from a bootable device that is read on computer power up. It was very nice to be able to preserve the current application programs and drivers and to do the installation from “inside of” the old OS.

The installation process was quick ( < 15 minutes), there were no error messages or any obvious problems.
There was no requirement for user responses during the process.

The first tasks on initial start-up of the OS are to check for updates and to provide logon information.
The update requirement depends on an operational connection to the Internet. In our case, the connection is by WiFi and there would normally by no way to initiate this without having been logged on to a running version of the OS. It is a typical “cart before the horse” problem. Fortunately, it is possible to defer this step which is what we did.

The next problem is creating the logon. Dell’s default on display intensity is dimmer than is desirable and, at least on this tablet, it is difficult to see the contents of some of the logon fields. You cannot change the display intensity until after a successful logon. The “CAPTCHA” step is especially difficult and annoying. The dim display makes it tough to recognize the alphanumeric characters which may be on angles, overlapping, indistinct and of varying intensity. One has to guess about some of the characters, to change to different CAPTCHA selections and to repeatedly attempt a logon. My recommendation is not to use CAPTCHA in any part of the software.

Microsoft encourages the use of a “Microsoft system wide” user name and password.  There are some advantages to this, but, as others have pointed out, it has some risks.

Touch screen virtual keyboards are infamous for problems with accidental selections of adjacent keys and unintended character repetitions. Fortunately, the Venue 11 has a large sized virtual keyboard and there is less chance of key selection errors than is true with the small sized displays. The Venue does has problems with the unintended character repetitions. All of this causes difficulties with initial password setups in which the entered data are hidden with stars or other characters and one also usually has to enter the new password twice. A few software developers have started offering users a check box option for display of the entered password information. This is really helpful for reducing the number of times that one might need to enter a new password and then repeat it for confirmation.

Initial Experience with Windows 10

There have been no known problems with missing or incompatible drivers. I don’t know if any drivers were updated between 8.1 and 10 except that there was a somewhat lengthy automatic install of a new driver for the cellular modem. This device and its software have not been evaluated as yet. I do not know if the software enables regular cellular voice operations. It is unlikely that it enables the automatic transfer of voice communications between WiFi and Cellular networks, as is available from some vendors.

Access to the Control Panel, Command Prompt and other administrative functions is now possible via the “Start” icon display once one realizes that you need to scroll the display fully downward and to the right in order to see the “administrative” icons.

In spite of the relatively large Dell Venue display (10.8 inches), there are still many fields that are small and/or are too close together to be reliably selected by finger touch. Some of these fields cannot be selected at all by finger touch. The problem can be reduced by increasing character and display sizes and by the use of an Active Stylus. Use of a mouse plugged into the USB port is a temporary solution if one doesn’t have a stylus.

There have been many instances of failures to respond to icon selections by repeated finger presses. The reason for this behavior is unknown. The problem can be temporarily cleared by a soft restart.

In some fields, autofill correctly displays a list of choices, but the application program crashes with an unknown error message if you touch select one of the fields.

This isn't necessarily a Windows 10 issue, but there has been a recent marked increase in advertisements that are added to the results of Google and Yahoo searches, e-mail traffic and other Internet transactions. The latest twist is that unless one accepts an advertisement's initial suggestion - usually by clicking on a YES or OK icon - then the display is locked and you cannot close it short of ending the task or doing a soft reset. There is no choice to Cancel. Selecting the X close icon or close on a drop down box doesn't close the advertising Window display.

andreas killer say:

The Last Cell previously worked correctly.

That is not correct, the described behavior can be reproduced in all versions of Excel.

The Specialcells documenation say: xlCellTypeLastCell. The last cell in the used range

If you see it from the point that the used range depends only on the filled cells: you are right.

But that is not what Excel means with "used range".

Make a new file, write "x" into I14, press CTRL-END, press DEL, press CTRL-HOME.
At this point the sheet looks the same as you start, but if you press CTRL-END again, Excel jumps into I14 again.

The "used range" is more like what Excel has allocated in memory...
And it seems that a big border needs on row more... for whatever reason, I don't know.

Anyway, use Range.Find and search for the really last filled cell. See code below.

Andreas.

Option Explicit

Sub Test()
  MsgBox RangeLastCell.Address
End Sub

Private Function RangeLastCell(Optional ByVal Where As Range) As Range
  'Returns the last used cell in Where
  Dim R As Range, c As Range
  If Where Is Nothing Then Set Where = ActiveSheet.UsedRange
  Set R = Where.Cells(Where.Rows.Count, Where.Columns.Count)
  If IsEmpty(R) And Not R.Address = Where.Cells(1, 1).Address Then
    Set c = Where.Find("*", After:=R, SearchOrder:=xlByColumns, _
      SearchDirection:=xlPrevious)
    If c Is Nothing Then
      Set RangeLastCell = Where(1, 1)
    Else
      Set R = Where.Find("*", After:=R, SearchOrder:=xlByRows, _
        SearchDirection:=xlPrevious)
      Set RangeLastCell = Where.Cells(R.Row - Where.Row + 1, _
        c.Column - Where.Column + 1)
    End If
  Else
    Set RangeLastCell = R
  End If
End Function

doug robbins - mvp office apps & services (word) say:

If you assign a bookmark "Section3" to the text that you do not want to appear and then you use the following code in the ThisDocument object, the text that is within the range of that bookmark will be formatted as hidden and if the display of Hidden text is turn off, it will not be visible

Private Sub Document_ContentControlOnExit(ByVal CC As ContentControl, Cancel As Boolean)
If CC.Title = "Sec3" And CC.Checked = True Then
    ActiveDocument.Bookmarks("Section3").Range.Font.Hidden = True
ElseIf CC.Title = "Sec3" And CC.Checked = False Then
    ActiveDocument.Bookmarks("Section3").Range.Font.Hidden = False
End If
End Sub

bernd p. say: The "normal" procedure is:

  1. open the embedded file from Onenote
  2. make changes in the associated application
  3. save changes in the associated application
  4. close the associated application
  5. re-open the file from Onenote
  6. see the changes :-)

Is this your procedure or did you omit the close ?

Bernd

steven westby say: Our normal procedure is to Close application and when it asked do you want to save we say yes.  This covers your case but in a slightly different order (Which might be meaningful).

bernd p. say: I think that's equivalent. Important is the close as an action which tells Onenote to re-embed the file.

If you open an embedded file Onenote creates copies in a hidden cache folder, points the associated application to it for opening. After closing the application it should read in the changed copy.

Because you are in a "corporate server environment" there could be problem with redirections of user folders or access rights.

The mentioned important cache folder you should observe while changing an embedded file is:

C:\Users\<user>\AppData\Local\Microsoft\OneNote\14.0\OneNoteOfflineCache_Files

AppData is hidden.
