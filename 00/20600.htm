albert d. kallal say:

As a general rule if you just drag + drop a control like a textbox from the toolbox onto a form, then the wizard will not really do anything for you. And the resulting text box will NOT save data typed into that box.

this "concept" is called bound text boxes. (the above text box is UN-bound untill such time you tell that text box what column to save data into).

So the box THEN needs to be bound to a column that the forms data source is based on.

The most “easy” way to do the above is use the Add Existing Fields box from the design tab on the ribbon. This will result in creating a bound text box in one step that is correctly bound to the correct data column.

The wizard can be most helpful when creating combo boxes, as it lets you setup the data that will drive the combo box, and the “last” step in that combo box wizard does give you a choice as to what data column the form is based on save the selected value into.

However, if you JUST drop a text box onto the form, then it is called “un-bound” and will not save any data you type into that text box until such time you set the “control source” of the text box. In “most” cases the name of the textbox control is the same as the underlying data column the form is bound to, but this is not some requirement and often people will have different name for the text box as opposed to the underlying data column the text box saves into.

So for textboxes, either use Add Existing fields, or drop a text box from the ribbon. If you drop using the toolbox from the ribbon, then you need the extra step to set what column the textbox is to operate on. You do this by displaying the property sheet for the text box, and on the data tab you set what is called the control source.

Eg this:

So in  above the text box name is FIRSTNAME, and the data column (control source) into which the text box will save data is also FIRSTNAME. The textbox could have been named anything, but the control source is MOST important, since that setting determines what column the text box data will be saved into. Note that above setting does have a combo drop down that will ONLY show the "legal" list of data columns. So it is a VERY good idea to ALWAYS use the drop down of fields to choose the correct value. If a column does NOT appear in this drop down, then you have to add that column to the table, and if already in the table, the you have to change the forms data source BEFORE the above drop down will include that desired data column from the table.

Next you need to tell Windows Media Player where to look for the files so that it can monitor/manage them. To this:

1. Open Windows Media Player.

2. Select Organize then Music.

3. If the location that you put the music files in is not shown, select Add, browse to that location then select Include folder then OK. Select Music on the left to check that your music is now available from Windows Media Player.

Right-clicking on a folder in the manage library locations box will bring up other options.
