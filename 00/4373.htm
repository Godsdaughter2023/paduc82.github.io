How Do I Print Address and Information On A Postcard?

Windows 10 Pro x 64

Office 2010 Pro plus x32

I just want to print a single postcard with the address on the right and the message on the left.  How do I do that?

doug robbins - mvp office apps & services (word) say:

Go to the Mailings tab of the ribbon and click on the Labels button in the Create section and then select vendor\product number for the Post Card stock that you have, such as Avery US Letter, 8380 Postcard and then click on the New document button.  What you will then see is a page representing the post card with the selection inside the cell of a one cell table (Click on the Table Tools>Layout tab of the ribbon and then click on View Gridlines if you do not see the outline of the cell).

You may want to use the Split Cells facility on that tab of the ribbon to split the cell into two, so that you can get the text to word-wrap in the left hand cell and then, put the address in the right hand cell.  To get the address in the appropriate place, use the Align Centre Left button in the Alignment section of the Table Tools>Layout tab of the ribbon.

Option Group Default Value - Microsoft Community

questionboy say:

Option Group Default Value

I must be missing something really obvious, but I can't seem to figure it out.

 

I have a sub-form with a couple of Option Groups with 2 options each (Yes -1 and No 0).  In the table the default value is set to NULL.  Yet the minute the form is initialize by any other control, these Option Group always default to No.  I do not want a default choice selected.  I want the Option Group left blank until the user actually make the selection themselves.  What am I overlooking?

ken sheridan say: Karl is right in one way in saying that the value of an option group control cannot be Null, because Null is not a value; it's the absence of a value.   However, the Value property of an option group can be Null.

The problem in your case is almost certainly that the control is bound to a column of Boolean (Yes/No) data type.  a Boolean column can only be TRUE or FALSE, which in Access are implemented as -1 or 0, so can never be Null.  So in any row in a table a Boolean column can only be TRUE or FALSE, never NULL.  By default it is FALSE.  You can, however, paradoxically set the DefaultValue property of a Boolean column to Null, and similarly of a control bound to such a column, as you have done.  This is pointless, however, as the value will become FALSE as soon as you begin to enter any data in the row, which is what is happening in your case.

You'll be p[leased to know that there is a simple solution to this:  Change the data type of the column in table design view to Number and its Length property to Integer.  Make sure its Required property is False (No) so that it allows Nulls, and set its ValidationRule property to:

True Or False Or Is Null

You now have 3-way logic, TRUE, FALSE or MAYBE, which incidentally was one of Codd's original requirements when he introduced the database relational model back in 1971.
