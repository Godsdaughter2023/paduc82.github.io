run-time error '91' when closing ms word 2007 - Microsoft Community

cw00 say:

run-time error '91' when closing ms word 2007

Today, I experienced having the message "run-time error '91'" when I close ms word 2007. How can I get rid of this problem? 

jay freedman say:

Anything that says "run-time error" is coming from a macro. If the message appears every time you close Word, then the macro is in either the Normal.dotm template or an add-in that's stored in the folder %appdata%\Microsoft\Word\STARTUP.

 

First check out Normal.dotm:

  • Click View > Macros or press Alt+F8 to open the Macros dialog.
  • Open the Macros In dropdown and select Normal.dotm.
  • In the list of macros, look for one named AutoClose or AutoExit. If it's there, click on it and click the Edit button. Copy the code -- from the word "Sub" to the line "End Sub" -- and paste it into a reply to this thread so we can see what might be going wrong.

If you don't see either of those macros in Normal.dotm:

  • Close Word.
  • In Windows Explorer (the file manager program, which might be shown as "Computer"), paste the line %appdata%\Microsoft\Word\STARTUP into the address bar and press Enter.
  • Select all the files in that folder and cut/paste them to some other folder (for example, create a folder named C:\AddIns and paste the files there). The STARTUP folder should now be empty.
  • Restart Word and check whether the error occurs or not when you close Word again.
  • If moving all the add-in files stops the error, move one file back into the STARTUP folder and try starting and stopping Word.
  • Repeat the step of moving one file at a time back to STARTUP. When you find one file whose presence in the STARTUP folder causes the problem to start up, that's the one with the bad macro. You'll have to see whether that add-in has a new version that works better, or decide to do without the add-in.

jay freedman say:

A bit of explanation: The error number 91 corresponds to the description "Object variable or With block variable not set." In this macro, the only object variable is "Selection", which means any text that is selected, or the next character after the cursor if no characters are selected. The only way this variable can be "not set" is if no document is open when the macro runs. Unfortunately, that's exactly what happens when the macro is the special AutoExit macro (or, in this case, the MAIN macro within a module named AutoExit) -- any open documents are closed first, and then the macro runs just before Word shuts down.

To fix this, change the name of the module from AutoExit to AutoClose. Then the macro will run just before each document closes, and the Selection will still be available at that time.

If you delete the former user’s Office 365 account, his license will be freed up automatically. Then you can assign it to a new user.

To assign or remove Office 365 for Business licenses, please refer to the following article:

https://support.office.com/en-us/article/Assign-or-unassign-licenses-for-Office-365-for-business-997596b5-4173-4627-b915-36abac6786dc?ui=en-US&rs=en-US&ad=US

Moreover, to install Office applications, you can sign in to Office 365 for Business in your browser at http://portal.office.com/OLS/MySoftware.aspx. You doesn’t need to delete Office 365 applications from his computer either. Office 365 applications communicate with the Office Licensing Service and the Activation and Validation Service to obtain and activate a product key. If one user’s license is removed and Office 365 account is deleted, Office 365 applications will enter reduced functionality mode. You can refer to Overview of licensing and activation in Office 365 ProPlus for more information.
