ken sheridan say: Given the following tables:

Products
....ProductID  (PK)
....Product

StockTakes
....StockTakeDate
....ProductID  (FK)
....Quantity

The following query will return the stocktakes per product along with the date and quantity of the previous stocktake in each case:

SELECT Product, StockTakeDate, Quantity,
    (SELECT MAX(StockTakeDate)
     FROM StockTakes As ST2
     WHERE ST2.ProductID = ST1.ProductID
     AND ST2.StockTakeDate < ST1.StockTakeDate) AS PreviousStockTakeDate,
    (SELECT Quantity
     FROM StockTakes As ST3
     WHERE StockTakeDate =
        (SELECT MAX(StockTakeDate)
         FROM StockTakes As ST4
         WHERE ST4.ProductID = ST1.ProductID
         AND ST4.StockTakeDate < ST1.StockTakeDate)) As PreviousQuantity
FROM StockTakes As ST1 INNER JOIN Products
ON ST1.ProductID = Products.ProductID;

The above tables are taken from the simple Inventory demo file in my public databases folder at:

scottgem say:

How are you using stock takes? In an inventory application Quantity on Hand should be calculated in a query by adding incoming transactions and subtracting outgoing transactions. Stock takes can be used to adjust for physical counts on a periodic basis. The most recent stock take is then used as the base number for the additions and subtractions.

When I do inventory apps, I store the stock take and effective date in the Product record to use as the starting point. and then include transactions after the effective date. If I need to keep a record of the physical inventory at a point in time I wrote the previous values to a history table.

scott mcdaniel say:

You can refer to open forms in Access from your "Pop-Up Sub-Form", but this can be tricky (and especially with a Navigation form).

You refer to the contents of a Form like this:

Forms!YourFormName.SomeControl

If you want to refer to a Subform on that form:

Forms!YourFormName.NameOfYourSubformCONTROL.Form.SomeControl

Note that NameOfYourSubformCONTROL is the Name of the Subform Control object on the Form, and may or may not be the same as the Form you're using as a SourceObject.

If you have sub-subForms, you must drill deeper:

Forms!YourFormName.NameOfYourSubformCONTROL.Form.SubformControlOnTheFirstSubform.Form.SomeControl

One thing to remember in regard to Navigation Forms is that they use a single Subform Control, and change the SourceObject based on user selections - so if you're expecting an "Invoice" subform, but the user has selected a "Customer" view, then your code may break, or work with invalid data.

scott mcdaniel say:

You don't really need to pass in the value, but rather you can refer back to the Form using the methods suggested earlier and retrieve the value as needed.

Or if you prefer, you can pass in the value using the OpenArgs method. To do that, include the value of EnquiryID when using your OpenForm command:

DoCmd.OpenForm "EnquiryDetails", acViewNormal, , , , Me.EnquiryID

Now in the EnquiryDetails form, use the Form's Open event to get the value:

If Len(Me.OpenArgs) Then

   Dim EnquiryID As Integer

   EnquiryID = Me.OpenArgs

End If
