You need to verify any (sub)domains you want to use with aliases, and you need to set the domain intent to Exchange Online. If you see them under Accepted domains in EO, you are good to go.

There are no emailaddresspolicies you can use in EO, so any changes in aliases should be made on-prem, exactly as you describe them. If a particular alias in not being updated, make sure the sync process works correctly. A common problem is not having all the mandatory Exchange related attributes populated for the on-prem object. How to set up Information Rights Management (IRM) on your - Microsoft Community

elizabeth at ms say:

How to set up Information Rights Management (IRM) on your Windows phone

 

Before you can set up IRM on your phone, you’ll need to make sure that you have Windows Rights Management Services installed on your computer. If you’re running Windows Vista, Windows Rights Management Services is already installed. If you’re running Windows XP, you’ll need to install it through Add or Remove Programs in the Control Panel. (If you don’t see Windows Rights Management Services listed in Add or Remove Programs, you can also download it directly from the Download Center on Microsoft.com. ) You can tell if you have Windows Rights Management already installed if you’re able to open IRM-protected e-mail in Office Outlook.

Your computer must also be connected to a domain and on your corporate network.

1.       Install Windows Mobile Device Center

or ActiveSync if not already installed on your computer.

2.       Connect your phone to your computer through a USB cable.

3.       If you’re using ActiveSync:

1.       In ActiveSync, on the Tools menu, click Advanced Tools > Activate Information Rights Management.

2.       In the Information Rights Management Activation dialog box, click Yes, and ActiveSync will try to activate IRM automatically.

3.       If ActiveSync isn’t able to access your credentials, you’ll be asked to re-enter the information, including your user name, password, and domain.  Click Retry.

4.       After you’ve successfully activated Information Rights Management, click OK.

4.       If you’re using Windows Mobile Device Center:

1.       Click Mobile Device Settings > Activate Information Rights Management.

2.       Enter your credentials (user name, password, domain), and click Activate.

3.       After you’ve successfully activated Information Rights Management, click Done.

Report > layout view > col headings and contents not linked, cols don't move?

In Access 2007, when you created a plain tabular report and viewed in in Layout View, the column headings and fields were linked so that if you highlighted a column and widened or narrowed anything, the entire column changed width. Ah, ok. You have to use the "instant" report wizard.

 

Hightlight the table in the nav pane. Then then use on the ribbon create tab, just click on the report icon. You get a report with a control layout.

 

However, I just noticed if you use the report wizard, you will find that the report is not setup with a control layout (I believe that 2007 worked this way also). Remember, the one click report defaults to tabular anyway.

 

The simple solution when using the wizard here is to then apply a control layout. So, open up that report in design mode (if not already), click on the left ruler bar to select all OF THE text boxes in the detail section. Then shift click right above in the ruler on the page header to select all of the labels in the page header. Then on the arrange tab, hit the "tabular" format on in the table group. You just applied a control format and "connected" all of the labels to the text boxes.

 

There little advantage to using the wizard and tabular as opposed to the "one click" report option above since that DEFAULTS to a tabular report anyway. However, either of the above works fine in 2010.

 

So, if you use the step by step wizard in place of the one click, then the above tip will let you connect the lables back to the text boxes.

 

And, EVEN BETTER is the above tip allows you to take reports from 2003 and before or any reports that don't have this connection in place.

 

In fact, you have to learn how to apply/un-apply these control formats anway. Note that the above advice also works for forms.

This is a new feature, but one has to spend some time learning how to apply it when working with forms and reports. If you don't learn the feature, then you have to "hope" that it is applied by some wizard, and in some cases you want to de-tach this behavour.

Also, for web forms, you are forced to use layouts, but you can un-attach contorls from the layout.
