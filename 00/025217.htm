macropod say:
Word's Find/Replace behaviour has been known to return unexpected results when change-tracking is switched on for decades. The behaviour can be worked around with a macro if you're willing to go down that path; or you can disable change-tracking while doing the Find/Replace. 

macropod say:

As I said, a macro could be used. For example:

Sub Demo()
Application.ScreenUpdating = False
With ActiveDocument.Range
  With .Find
    .ClearFormatting
    .Replacement.ClearFormatting
    .Text = "[0-9]. "
    .Replacement.Text = ""
    .Forward = False
    .Wrap = wdFindStop
    .Format = False
    .MatchWildcards = True
    .Execute
  End With
  Do While .Find.Found
    .Characters.Last.Text = vbTab
    .Collapse wdCollapseStart
    .Find.Execute
  Loop
End With
Application.ScreenUpdating = True
End Sub

or, to use one of your own Find expressions:

Sub Demo()
Application.ScreenUpdating = False
With ActiveDocument.Range
  With .Find
    .ClearFormatting
    .Replacement.ClearFormatting
    .Text = "^13([0-9]@). ([a-z,A-Z])"
    .Replacement.Text = ""
    .Forward = False
    .Wrap = wdFindStop
    .Format = False
    .MatchWildcards = True
    .Execute
  End With
  Do While .Find.Found
    .MoveEndUntil " ", wdBackward
    .Characters.Last.Text = vbTab
    .Collapse wdCollapseStart
    .Find.Execute
  Loop
End With
Application.ScreenUpdating = True
End Sub

bob jones aka: cybertaz say:

Unless I'm completely misunderstanding what you've described wanting to do, my point was that the formula is unnecessary. Based on what I wrote before, you can just do F&R to remove the unwanted characters, although it does require 2 passes;

  1. Find what: ~*
  2. Replace with: <nothing>
  3. Replace All

  1. Find what: #
  2. Replace with: <nothing>
  3. Replace All

then do a simple Copy/Paste.

Since you had success with the formula, though, you might as well just continue from there. What you're now running into is that you're actually copying the formulas that are in the cells where the names are being extracted to, not the names themselves. The relative references in the formula(s) are automatically updating to refer to inappropriate cells when you paste to another location... That's why your getting the error messages.

Instead of just pasting, use Edit> Paste Special & select the Values option in order to paste the names rather than the formulas.
