I see two potential culprits:

If Error Resume Next

For error trappings in events you need to add:

If Error GoTo 0

at the ends of code, and one line above every place you have "Exit Sub" if trapings or Excel will continue to "go to next" whenever it encounters an error.

 

In addition, screenupdating was turned off, but was never turned back on.  At the end of that code, and one line above every "Exit Sub" add

Application.Screenupdating = True

nuculerman say:

Glad to help.  The problem arises when you change values of cells within change events.  You have to be very careful or you run into problems like this.  In general, I like to avoid error trapping at all for event macros, and put all my error trappings in the form of "Exit Sub" at the top of my macros.  So for your code, I'd write it like this:

Private Sub Worksheet_Change(ByVal Target As Range)
 
If Target.Column <> 4 Or Target.Cells.Count > 1 Then Exit Sub

Application.ScreenUpdating = False
Application.Calculation = xlCalculationManual
   
If Target.Row = 7 Then
    If Target.Value <> "" Then
        Call CUST_POP
        Call PN_POP
    Else
        Worksheets("DATABASE").Range("U2:U9999").ClearContents
        Worksheets("DATABASE").Range("AG2:AG9999").ClearContents
        With Worksheets("QUOTE FORM")
            .Range("D8").Value = ""
            .Range("D10").Value = ""
            .Range("D7").Select
        End With
    End If
ElseIf Target.Row = 10 Then
    If Target.Value <> "" Then
        Call PDID_POP
    Else
        Worksheets("DATABASE").Range("AE2:AE9999").ClearContents
        With Worksheets("QUOTE FORM")
            .Range("A12:A21").Value = ""
            .Range("D10").Select
        End With
    End If
End If

Worksheets("QUOTE FORM").Activate

Application.ScreenUpdating = True
Application.Calculation = xlCalculationAutomatic
End Sub

Setting default tab stops in custom templates - Microsoft Community

tomh64 say:

Setting default tab stops in custom templates

I use Word 2016 on Windows 10.  I've been trying to set the default tab stops in a custom template to 0.25".  When I look at the template in as .dotx, it shows that the default tab stop is .25, but when I try to apply the template to a Word document, the document's default tab stops remain the same.  All other aspects of the custom template are applied to the document, but not the default tab stops.  Is there a quick fix to the template, or should I just select the entire document and re-set the default tab stops?  Thank you.

jay freedman say:

When you say that you applied the custom template to a document, do you mean that you used the Attach button in the Templates and Add-Ins dialog box opened from Developer > Document Template, to change which template is attached to an existing document?

If so, that will never work. Quoting from http://shaunakelly.com/word/templates/attachtemplate.html,

So, what does happen when you attach a new template to a document?
Nothing.

Well, not quite. As the article goes on to explain in Section 2.7, you can cause styles from the template to be copied into the document by checking the box for Automatically update document styles in the Templates dialog (being careful to uncheck it after making the attachment).

To make this do what you want, first you must open the template, set the default tab stops in a paragraph, and then right-click the Normal style in the Home ribbon and click Update Normal to Match Selection. (For some reason, the default tab stop control doesn't exist in the Tabs dialog reached from a Modify Style dialog, only in the Tabs dialog reached from the Paragraph dialog or by double-clicking the ruler. That's what makes this little dance necessary.)

After saving and closing the template, you can attach the template to the document with the Automatically Update Styles checked, and the Normal style of the document will have the changed tab stops.

Actually, it will be much quicker to open the existing document, press Ctrl+A to select everything, open the Paragraph dialog, click the Tabs button, and set the default tab stops.

Either way, of course, new documents based on the template will inherit the template's tab setting.

tomh64 say:

Thank you, Jay.  I think I misunderstood which elements a template will apply to an existing document.

The way I understand it now - Applying a template to a document will update the styles of the document to the template's styles.  The same cannot be said of the template's default tab stops.  If I want to a template's default tab stops to be applied to an existing document, I would have to monkey around with the normal (aka default) template, something I do not want to do.

Looks like I'm going to have to do the Ctrl-A thing.  Maybe Microsoft will include default tab stops in the templates in a future version of Word.

jay freedman say:

That's not quite what I said. You can set the default tab stops in the Normal style of the template you will use to base new documents of this type. ("Normal style" is a paragraph style in every template and document, not the same thing as the Normal.dotm template.) Then updating an existing document's styles by attaching the template will apply the changed default tab stops to the document's Normal style. Further, since most other styles inherit from the Normal style, those styles will also get the new default tab stops.

Also, attaching a template to an existing document will update the document's styles waiting to happen.

As I said, though, doing "the Ctrl+A thing" in existing documents is quick and easy, and it avoids the risk that you may forget to turn off the "Automatically apply" option. You should still set the default tab stops in the template so that new documents from that template always have the right stops, but don't bother attaching the template to existing documents.
