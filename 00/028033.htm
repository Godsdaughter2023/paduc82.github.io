I'm not sure I understand what you're asking, and what exactly doesn't work.  If a folder doesn't exist, you can create it using the VBA MkDir statement.  If you need to create a subfolder, you need to create the parent folder first;  e.g., if you want to create "C:\MyFolder\SubFolder", and MyFolder doesn't exist yet, you would need to execute two statements:

    MkDir "C:\MyFolder"
    MkDir "C:\MyFolder\SubFolder"

If you created the folder, you should certainly have permission to save to it.  There are some folders that are protected, because they belong to either another user or to the operating system, and you would not normally have permission to write to them unless your application was running as an administrator.

The following works for me:

    DoCmd.OutputTo acOutputReport, "Report1", acFormatPDF, "C:\Users\Dirk\Documents\Report1.pdf"

This also works for me, if the folder Temp exists on my C: drive:

    DoCmd.OutputTo acOutputReport, "Report1", acFormatPDF, "C:\Temp\Report1.pdf"

The following does *not* work for me, because the folder "BadFolder" doesn't exist on my C: drive:

    DoCmd.OutputTo acOutputReport, "Report1", acFormatPDF, "C:\BadFolder\Report1.pdf"

When I try that, it shows the dialog "Now outputting page x of y", but fails at the end with run-time error 2501.

Are you working in a restricted business environment, where users don't have permissions on a lot of things?  If so, I could imagine that you might be running into permissions issues.  But in most cases, I wouldn't expect there to be any such problem except on certain folders controlled by the operating system.
