Could Microsoft please hire some competent UI and icon designers? - Microsoft Community

armandnic say:

Could Microsoft please hire some competent UI and icon designers? (And stop dumbing down the desktop)

I realize UI is largely a matter of getting used to and to some extent a matter of taste, but I believe the direction is wrong.

The trend to flatten everything  is a fad. It already shows its shortcomings as MS is slowly adding back features from 7 (shadows under windows, blur effects).

The problem is that the UI is oversimplified. Apple brought skeumorphism to extreme but that doesn't mean that the minimalist extreme is the right one.

The UI controls don't look like controls, there's zero depth. Look at OS X Yosemite, they still keep a bit of shading to emphasize the controls and depict their state.

The icons are also a problem. Simple, monochrome icons work best for small status icons, where simple lines are used to maximize contrast and thus legibility. Having the same icon style blown up (like in Settings) is just bad design. The icons are now so bland and generic it becoming harder and harder to tell them apart. I repeat: icons designed to be legible on 5" screens look crappy on 24" monitors.

Windows 7 had beautiful icons and due to their more 3D appearance, it was easier to convey context and variety:

Look above - on the right you can clearly see at the first glance the different printer states (default, offline) as well as the different models. On the left, it's much harder to understand. The same thing can be found all over the OS.

Then there's the whole mix of styles - old icons, new icons, Control Panel vs. Settings and so on. Some settings are accessible only from one area but not the other. There's only one word to describe it: a mess.

So while I like the idea of a unifying windows experience, the direction is wrong because it degrades the desktop experience.

Speaking of degrading experience, this is also true about apps. The new calculator starts much slower than the old one and you can have only one instance. The Mail app is still too simplistic for desktop use compared to the good old Windows Live Mail (I couldn't work without Message Rules or Quick Views).

It's a more significant problem than a display limit if we're truthful. Msft is always a bit evasive about this but if you go by user experiences by doing a google search on the issue, you'll find that most people experience  an almost total loss of functionality once they exceed 5k items. scott mcdaniel say:

This can be caused by missing references, among other things. To check for missing references on your  development machine, open the .accdb file, then open the VBA Editor and click Tools > References. If any are marked as MISSING, you must fix those (i.e. install the libraries, re-make the references, etc).

If you are deploying this to a different machine, then you have to make sure your references are available on that machine. If you've used anything other than the standard Access references, you'll have to check that.

This can also be caused by corruption. To fix that, first make a copy of your database, then do this:

1. Compact your database

2. Compile your code - from the VBA Editor click Debug - Compile. Continue doing this until the Compile menuitem is disabled.

3. Compact again

Sometimes it helps to decompile your code. To do that, create a shortcut with this as the Target:

"full path to msaccess.exe" /decompile

Run that, then open your database. Access will decompile it. After that, run the 3 steps again.

Also be sure all machines are fully up to date with Windows and Office.
