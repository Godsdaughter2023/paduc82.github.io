bob jones aka: cybertaz say:
Just a few points that may help clarify the issue :

Re: "If one creates an equation using the "Table - Formula" selection and use a bookmark 
such as =sum [bookmark] (a1+b1) you will get a  !Syntax Error"

Bookmarks are not the equivalent of Named Ranges in an Excel workbook. You can refer to a Bookmark in another cell [for example FirstNumber is a Bookmark name, perhaps for cell D11] but you can't use Bookmarks in formulas to identify a range or collection of cells.

In your example, the syntax is incorrect :-) That method is for using a formula outside a table. If there are values in Column B of a Table you'd first need to Bookmark the table [as described below] in order to give it a unique identity. Let's say you call it Fred... your formula would be:

=SUM(Fred B:B)

Or if the values are in cells A1 & B1 as per  your example:

=SUM(Fred A1:B1)

Re: "First issue

In Mac word, exactly how does one "Book Mark" a word table?"

You Bookmark exactly what you have selected when you create/name the Bookmark :-) So...

Select the entire table by clicking the Table Selector at the table's upper left corner or by clicking in the table & using Table> Select> Table.

Re: "2nd How does one "Book Mark" a cell in a mac word table?"

Accordingly, select just the cell - including the End of Content marker - before naming the Bookmark. If you don't see the markers (as shown in Rich's screen shot), display the non-printing characters [¶] by clicking the button on the Standard Toolbar or using the Command+8 keystroke. NOTE: This method will insert the entire cell - border & all - not just the calculated result.

Re: 3rd [unasked]: If you want to Bookmark only the result of the calculation, select only the calculation field in the table cell the ¶s displayed. When you reference a Bookmark created in this way only the calculated result will be inserted -
Re: "...you have to go back and convert that one cell reference else where from table to text."

In Word, only certain types of fields (footnotes, endnotes, page numbers & a few others) update automatically. All other fields must be updated manually - although not Update fields. If that preference is checked fields not only update when the document is printed but universal updating of all fields in the document can be 'forced' by switching to Outline View then switching back to Print Layout View.

I use the attached to produce a 3 million line nested xml in about five minutes.

There are two key items,

1) a simple piece of VB,


Public Function Export_ListingData()

    Dim objOtherTbls As AdditionalData

    On Error GoTo ErrorHandle
    Set objOtherTbls = Application.CreateAdditionalData
    objOtherTbls.Add "ro_address"
    objOtherTbls.Add "ro_buildingDetails"
    objOtherTbls.Add "ro_businessDetails"
    objOtherTbls.Add "ro_businessExtras"
    objOtherTbls.Add "ro_businessExtrasAccounts"
    objOtherTbls.Add "ro_businessExtrasAccom"
    objOtherTbls.Add "ro_businessExtrasAccom2"
    
    Application.ExportXML ObjectType:=acExportTable, _
                DataSource:="ro_business", _
                DataTarget:="C:\Users\Steve\Documents\Conversions\ListData.xml", _
                AdditionalData:=objOtherTbls
Exit_Here:
        MsgBox "Export_ListingData completed"
        Exit Function
ErrorHandle:
        MsgBox Err.Number & ": " & Err.Description
        Resume Exit_Here
End Function


2) Linking the tables in relationship manager using joins from primary to FOREIGN keys.

If there are no relationships the code will produce a sequential xml file, if there are
relationships between primary keys you will get a 31532 error and the data export will fail.
