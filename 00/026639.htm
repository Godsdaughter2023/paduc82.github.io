what is the difference between vb and vba

I have searched the same in several places but i didn't get the proper answer.

please help me out !!!

Regards
KarNee
hansv mvp say:

VBA is the programming language that can be used in the Office programs such as Word, Excel etc. to write code that is run from those applications. You can't use VBA to create a new stand-alone application.

 

VB is a programming language (or actually several) that comes with a compiler to create stand-alone applications (.exe).

The "classic" version is VB6; the syntax of VBA in the Office programs is almost the same as the syntax of VB6, so code written for VB6 can often be used with little or no modification in VBA.

There is a more modern version VB.NET that has a similar but slightly different syntax.

bob jones aka: cybertaz say:

I can appreciate your belief that it doesn't "work" but the truth is that the macro is doing exactly what it was recorded to do :-)

When recording a macro the navigational keystrokes, themselves, are not recorded, it's the reference of the selected cell where you make an entry or some type of change. IOW, if you type in cell A20 what is recorded by your next step effectively is "select cell B20", not "press right arrow once". The next step amounts to "select cell D20", not "press right arrow twice".

You can confirm this by viewing the recorded code in the VB Editor. Also, you can clear what you've entered then run your macro again. You'll see that the word test is written to whatever the current cell happens to be, the other entries are rewritten to B20 & D20, then the selector moves to A21.

In order to automate what you're trying to do I believe a macro would have to be written in VBA rather than recorded. I can't give you a solution because I really am not a proficient VBA guy.
