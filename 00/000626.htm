joeu2004 say: Nic wrote:

Hello,

I've been working on a formula that counts the number of times column D has the same number as column F. It also needs to only count the times this happens when columns C or G match a specific text and so I have been using a sumproduct formula.

My problem is the formula is counting blank cells in columns D and F as well, which is not what I want.

I've been playing with <>"" and counta and isnumber, but I can't get any of them to work.


Your syntax ($D$3:$D$8=$F$3:$F$8<>"") does not behave as intended.  Also, you should not count matches in column C separate from matches in column G unless they are mutually exclusive or you truly wish to double-count them.

If you do not want to double-count, write:

=SUMPRODUCT( (($C$3:$C$8=J4)+($G$3:$G$8=J4)>0) * ($D$3:$D$8=$F$3:$F$8) * ($D$3:$D$8<>"") )

It is sufficient to test only whether column D is empty or contains the null string ("").

Note that the test (($C$3:$C$8=J4)+($G$3:$G$8=J4)>0) returns TRUE if one or the other
On the other hand, if you do want to double-count in that case, simply remove the ">0" test, to wit:

=SUMPRODUCT( (($C$3:$C$8=J4)+($G$3:$G$8=J4)) * ($D$3:$D$8=$F$3:$F$8) * ($D$3:$D$8<>"") )

To learn about UserForms, see the following pages of Greg Maxey's website :

http://gregmaxey.mvps.org/Create_and_employ_a_UserForm.htm

http://gregmaxey.mvps.org/Populate_UserForm_ListBox.htm

End Sub

The listbox will be populated when you open the document (provided that macros are allowed to run).

jay freedman say:

Some further considerations:

In order to enable multiple selection, you have to change the list box's MultiSelect property from the default fmMultiSelectSingle to either fmMultiSelectMulti or fmMultiSelectExtended. (The latter is better, because it makes the selection behave like the ones people are familiar with in Windows Explorer and similar programs.) You can set this in the list box's Properties dialog, or add the line


        .MultiSelect = fmMultiSelectMulti   ' or fmMultiSelectExtended

to the macro that adds the values.

The point of Doug's suggestion to use the Document_Open macro is that ActiveX controls are always emptied by closing the document, and they must be reloaded on reopening the document. Besides that, if the control is to reflect selections made in the previous session, those settings must be saved before closing and restored on reopening. Typically that's done by storing the information in document variables (members of the ActiveDocument.Variables collection), which are saved in the document's file, and then reading the variables on reopening and selecting the corresponding entries in the list box.

I agree with Doug that a userform would be preferable to an ActiveX control -- it's both more stable and more flexible. The  list box that you would use in the userform is very similar to the ActiveX version, and the same considerations apply.

One other thing: When you send a macro-enabled document to another computer, the macros may be disabled unless (a) it's stored in a Trusted Location folder or (b) the security settings allow them to run from another location.

rob koch say:

So just because the initials happen to be the same as some government agency then it must be them adding this to a Microsoft product, with not even the slightest attempt to obfuscate (e.g. hide) it from anyone?

Time to remove your tinfoil hat...

The nis_svr.exe executable is the Network Inspection System (NIS) server which was first added to Microsoft Security Essentials at version 2.0 if I recall correctly and so was also included in Windows Defender for both Windows 8.x and 10.  This is a network monitoring module which watches for potential attacks via the network from things like web based exploits for vulnerabilities in either applications or portions of the Windows operating system networking components themselves.

NIS was added at a time when new network and application vulnerabilities were being discovered and exploited by malware creators almost monthly, so since it took significant time to create, test and distribute the critical Windows updates necessary to fix these, the NIS Signatures could detect and block these exploit attacks before they could truly breach the PC network.

The updates for the NIS signatures and how to view the About page where the information regarding these is displayed in either MSE or Defender are found near the bottom of the following page.

https://www.microsoft.com/en-us/security/portal/definitions/adl.aspx

It is illegal for either the NSA or the CIA to perform monitoring operations of US citizens unless they are communicating with known or suspected terrorists or other similar bad actors such as government agents from other countries.  However, there have been cases where this has been done as part of massive data sweeps or monitoring programs that have been questioned by US personal rights groups.

Personally though, I'm not worried about this possibility at all, since unless I'm doing something illicit myself, I should have nothing to worry about if my communications happen to be captured by such spying, intentional or not.  I would guess that the CIA and NSA have far better things to do than worry about what I have to say in my daily communications with friends or relatives.
