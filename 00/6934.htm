Hi,

 

Put a button on any of the workshets where you want this then attach this code. At the bottom of the code the underlined bit is the name of the worksheets that you want the data added to. Edit as required

 

 

Private Sub CommandButton1_Click()
Dim Response As String, NewName As String
Dim ws As Worksheet
Response = MsgBox("Have you selected the row below where you want to insert data?", vbYesNo)

r = ActiveCell.Row

For Each ws In Sheets(Array("Worksheet A", "Worksheet B", "Worksheet C"))
With ws
    .Rows(r).EntireRow.Insert
    .Cells(r, 1) = NewName
    .Cells(r - 1, 2).Resize(2, 256).FillDown
End With
Next
End Sub

Hi Arsh,

Question 1: We are planning to use a Dedicated Namespace for Hybrid Servers, are there any issues or challenges If we use a dedicate namespace?

Generally, there is a minimum of three domain namespace in Hybrid. The public domain namespace is contoso.com. Another two are contoso.onmicrosoft.com and mail.contoso.onmictosoft.com (Hybrid domain namespace). Therefore, if you mean you only want to add one custom domain name, it is fine.

Question 2: Assume our on premise environment is pointing to owa.contoso.com/autodiscover.contoso.com and we have installed Hybrid servers and I’ve pointed them to Hybrid.contoso.com namespace. Assume once we start Migration batch, how will Exchange Online make a connection based on the Migration endpoint created and how would Hybrid.contoso.com proxy or redirect the connection to on premise server? 

The endpoint is used to accept the requests for remote moves and it will proxy to the servers that have Mailbox Replication Service.

Question3: Do we need to enable to MRS Proxy on Internal CAS Servers also or is it fine if we enable on Only Hybrid Servers?

You need to enable the MRSProxy on each of your CAS servers.
