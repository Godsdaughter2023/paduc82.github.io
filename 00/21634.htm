First, if you have put any of your manuals into master documents you need to back them up. Now, from Windows, not Word!  Then read http://www.addbalance.com/word/masterdocuments.htm,

See Which Fields Update When You Open a Document (a knowledgebase article)
and Word Fields Part I : Automate Word Documents With Minimum Code

You can insert an AutoOpen macro in your compilation documents that updates IncludeText fields when the document is open. If you rename the following macro AutoOpen it will do this.

Sub IncludeTextFieldUpdateAllStory()
' Written by Charles Kyle Kenyon 15 November 2001
' repaired with help from Jezebel 10 December 2004
' Changed to IncludeText fields 30 January 2012
' All Story Field Updater - Ref fields
  Dim oField As Field
  Dim oStory As Range
  For Each oStory In ActiveDocument.StoryRanges
  ' This goes into headers and footers as well as the regular document
    Do
    For Each oField In oStory.Fields
      If oField.Type = wdFieldIncludeText Then
        oField.Update
      End If
    Next oField
    Set oStory = oStory.Next
    Loop Until oStory Is Nothing
  Next oStory
End Sub

ken sheridan say:

I have no occasion to use macros, but the VBA code for a button's Click event procedure would be like this:

    Const NO_NEXT_RECORD = 2105

    Me.AllowAdditions = False

    On Error Resume Next
    DoCmd.GoToRecord Record:=acNext

    Select Case Err.Number
        Case 0
        ' no error
        Case NO_NEXT_RECORD
        ' anticipated error - do nothing

        ' or you can call the MsgBox function here to inform the user of this if you wish

        Case Else
        'unanticipated error - inform user
        MsgBox Err.Description, vbExclamation, "Error"
    End Select

Setting the AllowAdditions property to False means the form cannot go to a new record,  The rest of the code handles the error which occurs if the user attempts to go beyond the end of the recordset by clicking the button while on the last record.

If you also have a mechanism for removing the filter, and would then wish to allow new records to be inserted via the form, you'd need to set the AllowAdditions property to True again to allow this.
