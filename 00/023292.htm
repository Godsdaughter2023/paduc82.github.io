Hi RK,

You are correct, there is very little information written about add-ins for the Mac. I think Jim Gordon pointed you to Ron De Bruin's website, and I agree, Ron has a lot of good information there.

You can also look at anything written about global VBA based templates for Windows. The same concepts will apply to the VBA code in general for the Mac... there are differences where these templates are stored on a Mac and unfortunately there are differences in some of the VBA coding, but again, none of this is formally documented by Microsoft. Only a few of us "non-experts in Microsoft eyes" really know how to make it work. In general however, if you can get the VBA coding to work in a Windows PC environment running Office 2013, you've got a pretty good shot at making it run on a Mac. Just stay aways from "Events" because they don't work on a Mac and use those native VBA routines that I mentioned.

Regarding your other question about user templates... what I was suggesting is that you could place the customized FileSaveAs routine in a separate template (dotm) file and then when your custom global template (a.k.a Word add-in) needs to invoke a special SaveAs function, it could run it from this separate template file. Keeping the customized FileSaveAs routine in a separate, non-globally active function, has the advantage of only impacting the documents you specify and not all other documents the end-user may have open and active.

I hope this helps
