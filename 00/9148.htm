Hi,

 

Risk is a subjective measure.... you must decide yourself what the risk is once you understand what Protected Mode is...

You can read about Protected Mode from

Help>Internet Explorer Help... type Protected Mode in the Help search box...

here is an extract...

Internet Explorer protected mode is a feature that makes it more difficult for malicious software to be installed on your computer.

In addition to helping protect your computer from malicious software, protected mode allows you to install wanted ActiveX controls or add-ons when you are logged in as an administrator.

Protected mode is turned on by default in the Internet, Local intranet, and Restricted sites zones.

 

What it does is to add another layer of protection....

when you download files or navigate to sites in different Security zones, actions that can potentially expose your computer to infection or exploits additional security restrictions are used....and you as the user will see more prompts like...

Downloading exe files can potentially harm your computer.... Are you sure you want to download this file.

.... which actually means.... are you realy, realy sure you want to continue and download this file.

this also stops attacks where a hacker has taken control of your computer and is attempting to install additional software to your machine.

Obviously there are a few more buttons to press (compared to when Protected Mode is off), but there is always a trade off between security and usability....

consider a secure computer = it is locked in a safe with no internet connection, usb drives or keyboard.... it is unusable but totally secure.

consider an insecure computer = it is in a public place on a wireless network with no password protection.... it is usable and accessable to anyone, but not very secure.

So with Protected mode on, you as a user will have to click or push more buttons to do the same thing compared to when Protected mode is off.....

Some ppl think nothing of this, some ppl think it is an annoyance....

I would suggest that you leave it turned on ALWAYS because we are human and we can make mistakes. With faster internet connections an accidental click on a link can have your computer infected in an eye blink...

Your anti=virus programs may not detect some threats because new vectors are continually being developed by the bad guys and you may not yet have updated your anti-virus definitions which has counter measures for the new threats.

With time you will get used to it and think nothing of it (the extra button pushing)....

Computers are not toys.... the consequences of infection or exploits can be severe.

 

HTH.

gord dibben say:

Try this macro after entering a number in A1 of Sheet1

Sub findbottom_paste()
    Dim rng1 As Range
    Dim n  As Long
    n = Sheet1.Range("A1").Value
    Set rng1 = Sheets("Sheet2").Cells(Rows.Count, 1).End(xlUp) _
            .Offset(1, 0)
    Sheets("Sheet1").Range("B1:G" & n).Copy _
            Destination:=rng1
End Sub

The error indicates that there is a limit in the total amount of connections that you can make with your Gmail account (across all devices at the same time). Please note that your mail client may actually open multiple connections in the background. This means it's possible to reach a connection threshold simply by using only two mail clients to access the same account at the same time. If you often use multiple clients or mobile devices with the same Gmail account, we recommend you close and/or signing out of any mail clients you're not actively using. graham mayor say:
Actually you can do this with merge fields using a field that will provide the font size and a series of conditional fields.

Let's assume that you want four font sizes 11, 12, 13, & 14 and you want to vary the size of the result of a field called Fieldname. Create a field FontSize in your data source and set the value for each record to 11, 12, 13 or 14 (or whatever)

Add a field construction - { IF { Mergefield FontSize } = "11" "{ Mergefield Fieldname }"  } for each font size that you wish to use, and format the bold part of the construction with the font size of 11 points

Add similar fields for each font size you require and format the fields accordingly. I have used four sizes, but you can use more or fewer as appropriate. The values don't matter as long as they are matched by the formatting.

{ IF { Mergefield FontSize } = "11" "{ Mergefield Fieldname }"  }{ IF { Mergefield FontSize } = "12" "{ Mergefield Fieldname }"  }{ IF { Mergefield FontSize } = "13" "{ Mergefield Fieldname }"  }{ IF { Mergefield FontSize } = "14" "{ Mergefield Fieldname }"  }

Alternatively if all the fields in the document are the same size you could use a bookmark to set the value rather than a mergefield thus

{ SET  FontSize "14" }{ IF { REF FontSize } = "11" "{ Mergefield Fieldname }"  }{ IF {REF FontSize } = "12" "{ Mergefield Fieldname }"  }{ IF {REF FontSize } = "13" "{ Mergefield Fieldname }"  }{ IF {REF FontSize } = "14" "{ Mergefield Fieldname }"  }

Change the value in the SET field (here 14) to whatever you need for that document (from 11, 12, 13 or 14) and merge. You should be able to adapt this to your requirements.

jay freedman say:

I'm sure that method worked when I posted in the cited thread a year ago -- I wouldn't have written a specific procedure if I hadn't tried it first to verify -- but now I can't get it to work in any version of Word. The Delete command is grayed out for any of the built-in heading styles.

To reset the style definition, you'll need to locate a document or template that contains the style definition you want. The style must be applied to at least one paragraph in that file. Then:

  • Open the document you want to fix.
  • Press Ctrl+Alt+Shift+S or click the dialog launcher (the little arrow in the bottom right corner of the Styles group on the Home tab) to open the Styles pane.
  • Click the Manage Styles button (third button from the left at the bottom of the Styles pane).
  • In the Manage Styles dialog, click the Import/Export button at the bottom to open the Organizer dialog.
  • The list of styles in the open document appears on the left, and the Normal.dotm template (or whatever template the document is based on) appears on the right.
  • If the style definition you want isn't in the template, click the Close File button under it; when it changes to an Open File button, click it again, and use the Open dialog to find and open the file that does contain the correct style.
  • In the list on the right, click the name of the style to fix. Check the Description box to verify that it's what you want.
  • Notice that the "arrow" on the Copy button now points right-to-left, indicating the direction of copying. Click that button, and answer the prompt.
  • Close the Organizer and check that the style in the open document now has the correct formatting. If so, save the file.

And yes, I've just done this procedure in Word 2013, and it works the same way in all versions from Word 2007 through 2016.
