jay freedman say:

For the captions, there is another way.

First, your page layout should be set to "Mirror margins" (in the Margins dialog, this is one of the choices in the Multiple Pages dropdown; and note that this is separate from the setting for
different odd and even headers). If you need extra room for the captions, increase the width of the "inside" margin -- this is the left margin on odd pages and the right margin on even pages.

Where you say "textframe", you need to be careful to use a "frame" (in the All Commands list of the Quick Access Toolbar Customization dialog, this is the tool labeled "Insert Horizontal Frame",
though I don't know what "horizontal" has to do with it) -- do NOT use a text box, which doesn't have the necessary positioning options.

After inserting a frame in the inside margin of whatever page you're on, right-click its edge and choose Format Frame. Set the horizontal position to "Inside relative to Page".

Or cell E3 ==INDIRECT("[Hours.xlsx]" &$G$1&"!V2")

 

Both of the above are absolute addressing and you cannot copy the formula down or across as relative addressing.  I am assuming that G1 is meant to be absolute but you want V2 to be relative on the row number. Is this assumption correct?

 

The thing to understand with Indirect function is that it only works with a full address. The full address can be made up with concatenation but the concatenated string must form the full address. Anything that is within double quotes cannot be relative. The following are valid:

 


A simple cell reference such as G1 if on the same worksheet as the formula.


A worksheet name and cell reference if on a different worksheet in same workbook.


A workbook name, worksheet name and cell address if in a different workbook. (This is what applies to your situation.)

 

The problem you are faced with is if you want the V2 to become relative then it has to be done in a round about way. The following example makes the row relative so that the formula can be copied down. (The column V remains absolute.)

 

In cell E3  =INDIRECT("[Hours.xlsx]" &$G$1&"!V"&ROW()-1)

 


The formula includes the V within the quotes and then it concatenates a calculated row number by taking the difference between the row number where the formula is entered and the row number of the cell being referenced in workbook Hours.xlsx.

 

If you want the column to also become relative then we have to use the second parameter of the Indirect function (See Help for second parameter info) and use R1C1 style referencing for the Indirect function. (Note that we still use $G$1 to reference the worksheet name which is on the same worksheet as the formula).

 

In cell E3  =INDIRECT("[Hours.xlsx]"&$G$1&"!R"&ROW()-1&"C"&COLUMN()+17,FALSE)

 


The formula places the R within the double quotes


Concatenates the row number as per the previous formula


Concatenates the C


Concatenates the column number by adding the difference between the column number where the formula is and the column number of the cell being referenced in workbook Hours.xlsx.
More examples for cell D3.

 

In cell D3 with absolute column T and relative row

 

=IF(INDIRECT("[Hours.xlsx]"&$G$1&"!T"&ROW()-1)>40,40,(INDIRECT("[Hours.xlsx]"&$G$1&"!T"&ROW()-1)))

 


In cell D3 with relative column and relative row

 

=IF(INDIRECT("[Hours.xlsx]"&$G$1&"!R"&ROW()-1&"C"&COLUMN()+16,FALSE)>40,40,(INDIRECT("[Hours.xlsx]"&$G$1&"!R"&ROW()-1&"C"&COLUMN()+16,FALSE)))

Yammer Login - Microsoft Community

alanr2012 say:

Yammer Login

Some users have been asked for their date of birth to log into Yammer. Please can you advise why?
sreejith_n say:

Hello,

Greetings,

Thank you for posting,

 

https://portal.office.com/AdminPortal/home?switchtomodern=true#/MessageCenter?id=MC135617

 

Introducing age gating in Yammer for non-Office 365 accounts

 

  • As per this message users without an Office 365 account  will need to verify that they are over the age of 16 or they will lose access without administrator intervention.

 

  • Yammer users who are under the age of 16 and are not logged on with an Office 365 identity could be impacted by this change.

 

  • Now users who log in to Yammer without an Office 365 identity will receive a prompt to indicate their age. If the user is under the age of 16, they will have their account suspended in Yammer. As an administrator, you will be notified via a message in Yammer and can reactivate the user’s account within 14 days. After 14 days, the user account will be deleted.

 

  • If your organization does not include users under the age of 16, then there's nothing you need to do to prepare for this change.

 

If any of your users are below the age of 16:

1. Ensure they have an Office 365 identity; or

2. If a user gets suspended, reactivate the account within 14 days to prevent account deletion.
