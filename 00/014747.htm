Actually, the combo would be a lot easier. In fact, NO programming is needed. I think the missing piece to this puzzle is the Column Widths property. A combobox displays the first non zero width column. So in your case, I suspect you have a lookup table for provinces that looks like this:
tluCanProvinces
Abbreviation
Province

So the RowSource of your combo would be: SELECT Abbreviation, Province FROM tluCanProvinces ORDER BY Province;

Now the key is to set the Bound Column to 1, the Column Count to 2 and the Column Widths to 0";2"

No extra text box is needed, nor is a list box (which takes up extra space on your form), is needed. A list box is valuable when you need to display several columns of information for a user to choose from and to see that info once a selection is made.

cursedwretch say:

Activesync - Exchange password issue

I have about 150 activesync users on Exchange.  Something changed around 11/9/08 that is a creating a lot of problems and we can't figure out what it was.
I have about 150 activesync users on Exchange.  Something changed around 11/9/08 that is a creating a lot of problems and we can't figure out what it was.

Before that time when someone updated their AD password, Activesync would notice at some point and prompt the user to "Please correct your exchange server password".  That did 2 things - 1) gave users a very simple way of updating the password on their mobile and 2) (and more importantly) stopped the activesync from trying to sync any more - stopped the schedule.

Now when the wrong password is stored in the mobile device, there is no prompt - if you drill into Activesync you can see the "View Status" link that provides the generic error of "Activesync encountered a problem on the server".  Activesync disastrously keeps trying to sync - with our 3 strikes policy and 5 minute sync schedule - people are getting locked out in 15 minutes without even having to have their device on them.

We are getting hammered with calls and the users are understandibly very unhappy.  If there is any other information you would need to get an idea of what the problem could be, I can provide it.  Security swears nothing was changed on the mobile policy, server team swears our OWA has not changed, and I know the devices haven't had updates.  But even after updating one, the result is the same and reproducable.  If I type the wrong password in, it doesn't prompt to correct it, just has the link to the generic error and keeps trying to sync.

Thanks greatly for any help.

chris de herrera say:

Hi,

It would help to get some background on your environment:

 

1. What version of Exchange and service pack are you using?

2. What version are the Windows Mobile devices?

3. Are the Windows Mobile devices using push or scheduled e-mail collection?

 

Did you look at the log for the Exchange server to see what was changed on 11/9?  Examples of items to look for would be:

1. Changes to Active Directory?  Is Active Directory in sync?

2. Patches to the OS and Exchange.

3. Updates to 3rd party applications that work with Exchange such as spam filtering.

 

Also, I would gather all the logs and review them for details including:

1. Logging on the device.

2. Logging in IIS for EAS

3. Logging in Exchange

 

With this information, you will be able to work on identifying possible causes.
