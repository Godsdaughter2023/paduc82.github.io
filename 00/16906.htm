steve rindsberg say:

This bit of VBA will delete all sounds on all slides in the presentation (a COPY of your original, please!)

Sub DeleteSounds()

    Dim oSh As Shape
    Dim oSl As Slide
    Dim x As Long
    
    For Each oSl In ActivePresentation.Slides
        For x = oSl.Shapes.Count To 1 Step -1
            Set oSh = oSl.Shapes(x)
            If oSh.Type = msoMedia Then
                If oSh.MediaType = ppMediaTypeSound Then
                    oSh.Delete
                End If
            End If
        Next
    Next

End Sub

rohn007 say:
Pivot tables, once you figure them out allow you to let Excel automate the formulas for you.

#1 Define your data as a table, (ie I started my test data on row 6)
. * leave several blank lines above the data (you will use them later)
. * I put insertion point inside the data and use <CTL><T>

#2 Define a Pivot table on the data
. * With insertion point inside table click on "Table Tools > Design contextual tab > Tools group > click on "Summarize with PivotTable
. * normally I would put the pivot table on a separate tab, but for this case I put it on same tab, so click on "Existing table" option , then click to right of your data table in a line above it, ie D1

#3 Create an Average calculation
. * in pivot table pane, drag column 2 into Sum values box
. * click on drop down in "Sum of Column 2"
. * select "Value field settings" command
. * select "Average"

#4 Add a "Slicer" (a fancy name for filter) to the Pivot table
. * click on the column heading in the pivot table, ie "Average of column 2"
. * PivotTable Tools > Analyze contextual tab > Filter group > click on Insert Slicer
. * click on "column 1" then OK
. * drag the slicer beside the pivot table field.

Now when you click on a value in column1, or <CTL><Click> on multiple values
The result, =AverageIf() in this case, is displayed

wayne-i-m say:

Hi, it can be done, but before I (or someone else) can give you an answer you need to supply some more details.

How do you identify the "previous record"

As an example, you say you have a receipt book ending with 828500 and the next one you'll be using will start 812013 - so you want the next record to have number 812014.

How do you (or access) know what the last record (in this case it would be 812012)

Do you have a date/time field or a Dmax+1 for records, etc

You can easily add 1 to a number but how do you tell what the previous number (to add to) is

I'm not sure what you're look for but a simple answer would be to add a button to your form like this

Private Sub MyButton_Click()
Dim Response As Integer
Dim lngReceiptNumber As Long
lngReceiptNumber = DMax("[ReceiptNumber]", "MyTable")
    Response = MsgBox("The last receipt number used was " & lngReceiptNumber & "." & vbCrLf _
    & "Do you want to use this to allocated the next available number?", _
    vbYesNo + vbQuestion, "Receipt Number")
If Response = vbYes Then
    Me.ReceiptNumber = lngReceiptNumber + 1
Else
    MsgBox "Please enter a receipt number into the box"
End If
End Sub


When you got a new book this would allow you to add the new receipt number into a box and then add to this.

But of course if you want this automating then you'll need to provide more information.
