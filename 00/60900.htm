graeme_bennett say: AT LAST, I'VE SOLVED THE PROBLEM! This probably won't help too many other people unless they, too, are booting multiple OSes and running VMware Tools, but I figured out how to fix things on my system without reinstalling Windows. Indeed, it was caused by an instance of VMware Tools. I removed that, rebooted and voila -- I was able to print again. In an effort to help a few other people, here's a summary of things I tried while working through the myriad possibilities. If you're having 'print spooler can't start' problems, any of these might help.

1) After unsuccessfully attempting to restart the Print Spooler service and checking all of its dependencies, I removed all printer drivers and registry entries as detailed here: http://support.microsoft.com/kb/324757.
2) I restarted in safe mode and checked the C:\Windows\System32\spool\ folder for any corrupt or leftover spool files.
3) I moved all of the printer drivers in C:\Windows\System32\spool\x64 (or \w32x86 if you're using a 32-bit OS) to a temp folder. While reviewing the DLLs, I became supicious of some "ThinPrint" drivers installed by VMware.
4) I tried replacing spoolsv.exe with a known good copy. (To do this, you'll need to take ownership of the file. Search engines are your friend.)
5) I tried using RegEdit to reconfigure the RPCSS http default shown in the "DependOnService" value. This, and all of the other steps I'm mentioning in this list, are described in plenty of detail in the links provided in my Nov 11th message.
6) I tried system restoring back as far as possible -- which in this case had the positive side effect of undoing all these unsuccessful efforts. Unfortunately, the problem persisted, due to the limited number of restore points available on my system.
7) Following up on my earlier suspicion, I removed VMware Workstation and rebooted. The problem persisted after a reboot.
8) I then booted into the other OS on my system and removed the VMware Tools it had installed on my boot drive, then rebooted. 

Bingo! The printers reappeared and all worked correctly. I was then able to reinstall VMware Workstation without breaking anything. Whew!

anannya podder say:

Hi,

 

Swapping a hard drive can cause severe damage to the computer you are doing this to.
But it can be avoided if done right.

The reason it's never recommended to swap hard drives (containing the Operating system) from one computer to another has to do with the data that is contained inside the hard drive being swapped.
Some of the system board drivers on this swapped drive can and will damage another computer's main-board if not formatted first.

Note:

You can swap but first you need to format that hard drive if it contains an operating system on it.
Otherwise you could damage your new computer.

 

If the old laptop came pre-installed with Windows, then it uses an OEM (Original Equipment Manufacturer) license, which means it is tied to the original hardware. You cannot legally use the OEM license on any other computer.

 

 

Hope the information helps.

If you have any further queries or suggestions please keep us posted.
