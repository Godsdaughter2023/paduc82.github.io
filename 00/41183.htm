¡firedog say:


clean - with, say, a week or two's mail in it. Older messages should be removed from the inbox and placed in user-created archive folders.

goes wrong with a download, the protocol just starts again from the beginning, to make sure that every message is downloaded.

Apart from that, and this seems likely in your case, there will be maintenance carried out at the server every so often. This might mean that the flag on a message that says it's been downloaded is reset - so it gets downloaded again. This doesn't matter much if there are only a few days' worth of messages in the inbox, but if there are thousands of messages there, you have a problem.

Because each downloaded message is stored as an individual file, it's not easy to determine whether any two messages are identical. They will have different file names and different Date created and Date modified, while many other properties will be the same. But messages with the same Subject:, From: and To: properties might well be different messages in the same conversation. It would take a reasonably complicated algorithm to determine whether two files in fact represented the same message.
