Sub SplitColumns_In_newShts()
'May 20, 2016
Dim r As Long, c As Long, x As Long
Dim ws As Worksheet, newws As Worksheet
Set ws = Sheets("Sheet1") ' << source sheet name
r = ws.Cells(Rows.Count, 1).End(xlUp).Row
c = ws.Cells(1, Columns.Count).End(xlToLeft).Column
Application.ScreenUpdating = False
Application.DisplayAlerts = False
For x = 3 To c
On Error Resume Next
Worksheets(CStr(ws.Cells(1, x).Value)).Delete
On Error GoTo 0
Next
Application.DisplayAlerts = True
For x = 3 To c
Set newws = Sheets.Add(after:=Sheets(Sheets.Count))
ActiveSheet.Name = ws.Cells(1, x).Value
Cells(1, 1).Value = ws.Cells(1, x).Value
ws.Cells(2, 1).Resize(r, 2).Copy Cells(2, 1)
ws.Cells(2, x).Resize(r).Copy Cells(2, 3)
Next
Application.ScreenUpdating = True
End Sub

steve rindsberg say:

PPT 2007 and previous always link to MP3s, never embed them, so the max size setting will have no effect.

And because they're linked, you need to send the mp3 files along with your presentation and ensure that they all end up in the same folder at the other end.

AND in order for even that to work, you have to save the PPT at your end at least once, put the mp3s in the same folder as the PPT, and only then insert them into your presentation.

AND back to the other end, the path to the mp3s can't be longer than 124 characters, including full path + filename + extension.  If the files land on a mapped network drive, you have to figure in the full UNC path ( ie,\\servername\sharename\folder\folder\file.ext) rather than the mapped drive.

What you haven't indicated is what your current Office version and update level are.

You can't install any update unless you have the version it applies to. IOW, the Office 2008 12.2.0 update is useless if you don't have Office 2008 installed.

Also, you can't apply an update which precedes the current build of what you already have installed. I.e., if you do have Office 2008 installed & updated to 12.2.0 or later you can't apply the 12.2.0 (or earlier) update.

Additionally, I see little reason for wanting to apply that specific update in the first place as it is far from current. The one scenario would be if the present installation of Office 2008 is lower than that level. SP2 (12.2.0) would have to be applied in order to update beyond that level.
