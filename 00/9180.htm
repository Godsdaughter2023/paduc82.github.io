Hi all,

This is a very common problem. We support customers with Office 365 / Office 2013 and have come across this a while ago. 1.   Shut down all Office applications

2.   Open cmd as administrator e.g.
    Click Start > Search and type cmd
    Right-click cmd and choose to “Run as Administrator”

3.   Browse to the Office installation folder, in 64BIT Windows for Office 2013 it is
   CD C:\Program Files (x86)\Microsoft Office\Office15\
if you are on 32BIT windows the path will be
    cd C:\Program Files\Microsoft Office\Office15
NOTE: you have to adjust the Office version number for your installed version:
Office 2010 = Office14
Office 2013 = Office15
Office 2016 = Office16

4.   type cscript ospp.vbs /rearm and press enter

5.   type cscript ospp.vbs /act and press enter

6.   Restart your office applications and if all has gone to plan, everything should now work again.

 

If you want more information on what the ospp.vbs can do for you, you can run it like so:

doug robbins - mvp office apps & services (word) say:

Consider this scenario

  1. You have a document with a header in the first Section.
  2. You insert a Section Break
  3. By default, the header in the second Section will be linked to that in the first Section

  4. If you click on the Link to Previous Button, the header in the second Section will be unlinked from that in the first Section, but it is otherwise unchanged.  {You could at this point, edit the text in the header without the text in the header of the first Section being affected.)
  5. If you now click again on the Link to Previous button, you will in effect be deleting the header in the second Section and replacing it with a link to the text of the header in the first Section and hence, you see the warning

What you are observing is the normal behaviour of the operation of linking or unlinking the header or footer in one Section of a document from that in the previous Section.

THE ADVANCED SHEETOFFSET FUNCTION - Microsoft Community

somnath chatterjee 6309 say:

THE ADVANCED SHEETOFFSET FUNCTION

The sheetOffset Function that enables the user to address worksheets in a relative manner. One can refer the cell A1 on the previous worksheet by using this formula:

=SHEETOFFSET (-1, A1)

The first argument represents the relative sheet, the second argument MUST BE A Reference to a single cell. The code is as follows:

Function SHEETOFFSET(Offset As Long, Optional Cell As Variant)

'   Returns cell contents at Ref, in sheet offset

    Dim WksIndex As Long, WksNum As Long

    Dim wks As Worksheet

    Application.Volatile

    If IsMissing(Cell) Then Set Cell = Application.Caller

    WksNum = 1

    For Each wks In Application.Caller.Parent.Parent.Worksheets

        If Application.Caller.Parent.Name = wks.Name Then

            SHEETOFFSET = Worksheets(WksNum + Offset).Range(Cell(1).Address)

            Exit Function

        Else

            WksNum = WksNum + 1

        End If

    Next wks

End Function

 

The limitation of the function procedure is that it can only refer a single cell at a time not an ARRY OF CELLS AT A TIME. When we use multi sheet Vlookup formula, we have to use the sheet name along with Indirect Function in the formula. In that case, if some only changes the sheet name, problem occurs. Moreover, if the workbook contains many worksheets and you have to encounter all the sheets through the vlookup formula, you have to get the list of sheet names through vba code.

The SHEETOFFSET Function encouraged me to have such a vba code ( either Sub or Function procedure ) that can refer ARRAY OF CELLS (having rows and columns) across all the sheets e.g. range A5:D10  across all the sheets or the “UsedRange” across all the sheets.

What kind of modifications can be made in the aforesaid code or a new code may be generated. Looking forward a helpful reply.

Like this:

Function SHEETOFFSET(Offset As Long, Optional Rng As Variant)
    Dim WksIndex As Long, WksNum As Long
    Dim wks As Worksheet
    Application.Volatile
    If IsMissing(Rng) Then Set Rng = Application.Caller
    WksNum = 1
    For Each wks In Application.Caller.Parent.Parent.Worksheets
        If Application.Caller.Parent.Name = wks.Name Then
            Select Case WksNum + Offset
                Case 1 To Application.Caller.Parent.Parent.Worksheets.Count
                    Set SHEETOFFSET = Worksheets(WksNum + Offset).Range(Rng.Address)
                    Exit Function
                Case Else
                    SHEETOFFSET = CVErr(xlErrRef)
            End Select
        Else
            WksNum = WksNum + 1
        End If
    Next wks
End Function

Example 1: select a range of 3 rows by 2 columns in a column.

Enter the following array formula, confirmed with Ctrl+Shift+Enter:

=SHEETOFFSET(1, A1:B3)

This will display the values of A1:B3 in the next worksheet.

Example 2: enter the following formula in a cell:

=SUM(SHEETOFFSET(-1, B2:B100))

This will return the sum of B1:B100 in the previous worksheet.
