k3n3 say:

how to display the versioning number of a office365 word document in the word document?

I'm using office 365 and office 2016 on OSX, and have setup a sharepoint library with versioning enabled.

john mcghie say:

You need to set Sharepoint up to generate a Label field, which it will then update with its version number on each check in.

You can then insert a DOCUMENTPROPERTY "Label" field in the Word document to display the content of the Label field.

The instructions are here:

http://www.bryansgeekspeak.com/2009/03/moss-2007-show-sharepoint-version.html

Word for Mac is a cut-down version that does not have the QuickParts controls shown in that article.

  1. However, it DOES have an "Insert Field" control on the Insert tab of the ribbon.
  2. Within the Insert Field dialog, you will find a field named DOCUMENTPROPERTY.
  3. If you click Options on the DOCUMENTPROPERTY field, it should enable you to select the "Label" document property.
  4. If you have set up Sharepoint correctly, that will contain SharePoint's Version number.

Notes:

  • Word's Version field has nothing to do with the Sharepoint Version column.  The two are unrelated: Word's Version field does not contain the Sharepoint version number.
  • The document must have been added to the Sharepoint library, checked in, and checked out again before the Label field will appear in the document.
  • The document must be in .docx format and remain in .docx format.  If you convert the document to any other format, the Label field will be removed.

The thing that usually catches people out is that the document must have already been checked in to Sharepoint, or the field you are looking for will not exist in the Word document.

There are also reports that a heavily-customised version of Sharepoint may not support this method.

Hope this helps

Microsoft Access Multiple Product Prices - Microsoft Community

kaseyjt say:

Microsoft Access Multiple Product Prices

I am working on building an order form for my work in Access and I can not find an answer to my problem if there even is one. I have an order form that has a subform that allows me to have multiple products in one order but we have multiple prices per product based on quantity. For example we have a 1-4 piece price and a 5+ piece price. Is there a way to make it where when we type in the part number and quantity access automatically bring up the price based on quantity? If not what would be the best way to go about making the order form work for us?
ken sheridan say: In fact you only need the one quantity value in a PartsPrices or similar table which models prices per part per quantity, the maximum quantity per price, e.g.

PartsPrices
PartNumber    MaxQty    Price
1                      5              £5.00
1                      10            £4.50
1                      20            £4.00
1                      1000        £3.50
2                      10            £8.00
2                      20            £7.00
2                     1000         £6.00

The values of 1000 in the MaxQty column are artificially high values which will never be reached in an order.  These can be as high as necessary.

You can then look the price up by calling the DMax function.  You can see how it works in the immediate window:

intQuantity = 3
lngPartNumber = 1

? DMax("Price", "PartsPrices","PartNumber = " & lngPartNumber & " And MaxQty >= " & intQuantity)
 5

If we increase the quantity to 12:

intQuantity = 12

We then get:

? DMax("Price", "PartsPrices","PartNumber = " & lngPartNumber & " And MaxQty >= " & intQuantity)
 4

If we change increase the quantity to 25 and change the PartNumber to 2:

intQuantity = 25
lngPartNumber = 2

We then get:

? DMax("Price", "PartsPrices","PartNumber = " & lngPartNumber & " And MaxQty >= " & intQuantity)
 6

So in the OrderItems table there would be a single Price column, and in the AfterUpdate event procedure of a combo box bound to the PartNumber column in an order items subform the price would be assigned to the Price column with:

Me.Price = DMax("Price", "PartsPrices","PartNumber = " & Me.PartNumber & " And MaxQty >= " & Me.Quantity)

You'll have noticed that I have not used spaces in the table or column names.  While Access unwisely allows this, it is bad practice to include spaces or other special characters in object names.  Either use CamelCase as I've done, or represent a space by an underscore character like_this.  If you do include spaces or special characters the object name must be enclosed in square brackets [like this] in VBA code or SQL.
