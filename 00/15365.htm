Public Function CreateDisplayEmail( _
    ToLine As String, _
    CcLine As String, _
    Subject As String, _
    MessageBody As String, _
    FileName As String)

    Dim olApp As New Outlook.Application
    Dim olItem As Outlook.MailItem

    Set olItem = olApp.CreateItem(olMailItem)

    With olItem
        .To = ToLine
        .CC = CcLine
        .Subject = Subject
        .BodyFormat = olFormatHTML
        .HTMLBody = MessageBody
        .Attachments.Add FileName
        .Display
    End With

    Set olItem = Nothing
    Set olApp = Nothing

End Function

Public Function CreateDisplayEmail( _
    Optional ByVal TOmail As String, _
    Optional ByVal CCmail As String, _
    Optional ByVal BCCmail As String, _
    Optional ByVal Subject As String, _
    Optional ByVal MessageBody As String, _
    Optional ByVal FileNames As String) As Boolean
  Const olMailItem = 0
  Dim olApp As Object 'Outlook.Application
  Dim olMail As Object 'Outlook.MailItem
  Dim fso As Object 'Scripting.FileSystemObject
  Dim Temp() As String
  Dim i As Long
 
  'Access Outlook
  On Error Resume Next
  Set olApp = GetObject(, "Outlook.Application")
  If olApp Is Nothing Then Set olApp = CreateObject("Outlook.Application")
  If olApp Is Nothing Then Exit Function
  On Error GoTo 0
 
  'Create the mail
  Set olMail = olApp.CreateItem(olMailItem)
  With olMail
    .To = TOmail
    If Len(CCmail) > 0 Then .CC = CCmail
    If Len(BCCmail) > 0 Then .BCC = BCCmail
    .Subject = Subject
    'The .HTMLBody did not contain the signature before you call .Display!
    .Display
    .HTMLBody = MessageBody & vbCrLf & .HTMLBody
    'Attach the files
    If Len(FileNames) > 0 Then
      Set fso = CreateObject("Scripting.FileSystemObject")
      Temp = Split(FileNames, ";")
      For i = 0 To UBound(Temp)
        If fso.FileExists(Temp(i)) Then .Attachments.Add Temp(i)
      Next
    End If
  End With
  CreateDisplayEmail = True
End Function

jp ronse say:

Hi,

On first sight, I think that RIK:UK's formula will work but maintaining a lot of nested IF's can be a tough 

job and is error prone.

A simpler solution is using the LOOKUP function.

  • The prices are in B13 to J13 (from 10 to 1)
  • The quantities are in B12 to J12 (from 0 to 500)
  • Because your formula refers always to > 48, > 95, I added 1 to the price points. 
  • The quantities can be put anywhere. No need to have them B12:J12 as your model is already using B12.
  • The quantities must be in ascending order.
  • =IF($B$10="","",LOOKUP(A16,$B$12:$J$12,$B$13:$J$13)*$B$10)
  • You have to adapt the formula: B10 => B12!
  • Very easy to maintain, just add a new price and quantity and extend the range in the LOOKUP function.

A few words about your formula. You didn't nest the IF's. The way to do this is:

IF( Condtion,

True part,

IF Condition,

True part,

IF Condition, 

True part,

But as said I do not recommend too many nested IF's.

john w. vinson say:

What's the context here? As written, the query will fail if PrintRep is a Yes/No field (as you suggested above); a yes/no (checkbox) field can have a value of TRUE (-1) or FALSE (0) only. Setting it to False rather than to "" will update EVERY SINGLE RECORD in TBL_CreditSales every time you click the button; this seems wierd, in that you would expect to update only records which have something to do with the transaction.

Could you step back a bit and describe the table, the meaning of PrintRep, and what real-life task you are trying to accomplish?

scottgem say:

Use

DoCmd.SetWarnings False

Before you run your query and

DoCmd.SetWarnings True

After. 

john w. vinson say:

I'd make two suggestions: change the query to

UPDATE [TBL_CreditSales] SET [TBL_CreditSales].PrintRep = False WHERE PrintRep = True;

so you don't waste time updating all the (possibly very numerous) records with FALSE to FALSE - that's just a no-op and a waste of time; and execute the query either with warnings turned off, as Scott suggested, or using the Execute method. It takes a couple more lines of code but it will trap errors if they occur, and supress the warning messaged:

Dim db As DAO.Database

Dim qd As Querydef

Set db = CurrentDb

Set qd = db.CreateQuerydef("UPDATE [TBL_CreditSales] SET [TBL_CreditSales].PrintRep = FALSE WHERE PrintRep=True;")

qd.Execute dbFailOnError

Set qd=Nothing;
