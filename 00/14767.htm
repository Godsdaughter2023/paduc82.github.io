john mcghie say:
This is so complex that you may decide that the effort is not worth it. There could be a couple of weeks of full-time coding in it.

First you need to test that your selection actually contains a picture, and then test to see whether it is inline or floating.  This area of VBA was not very well implemented: it is extremely difficult to do either.

The best way to access the images in the document is to iterate the collection.  But there are two: Inline shapes and ordinary shapes.  Having found one, there is no good way to find out which one it is, (because it is not in the text, so there is nothing to tell us where in the document it actually is).

What you can do is set a range equal to your selection, then iterate the pictures in the document and check to see if the picture range is inside your selection range.  If it is, you have found it.

Below are some pieces of code that contain things you may find helpful (or not!)

What's happening here is that we are trying to insert a photo into a document at a specific exact size, and rotate it if it's landscape.  We check that the graphic style is in the document, and create it if it isn't (because we're going to use that the position the photo) then we ask the user to select a photo and tell us whether to rotate it forward or backwards.  The only difference between the first three little macros is the number of degrees of rotation we pass: all of the work is being done in Function ProcessGraphic.

The rest of the logic inserts the photo, floats the graphic if it is inline, rotates it, unfloats it, then scales it to be an exact size.  At the end, "PicBorderColour" is a reference to a public constant that contains an RGB value, enabling me to change all the colours for a customer in one place.

Somewhere in this logic you will find the bits that you can use to answer your question :-)  I should also warn you that this logic is badly designed and thus a bit unstable.  I was prepared to put up with it blowing up a couple of times a day rather that spend the week or so it would take me to figure out what was wrong.

Hope this helps

Sub InsertGraphic()
    CheckGraphicStyle
    ProcessGraphic 0
End Sub
Sub RotateGraphic()
    CheckGraphicStyle
    ProcessGraphic 90
End Sub
Sub RotateGraphicBack()
    CheckGraphicStyle
    ProcessGraphic -90
End Sub

Function CheckGraphicStyle()
'
' Insert Graphic Macro
' Macro written 13/06/2004 by John McGhie

Dim styGraphic As Style
For Each styGraphic In ActiveDocument.Styles
    If UCase(styGraphic.NameLocal) = "GRAPHIC" Then Exit For
Next ' styGraphic

If styGraphic Is Nothing Then
    Set styGraphic = _
    ActiveDocument.Styles.Add(Name:="Graphic", Type:=wdStyleTypeParagraph)

    With styGraphic
        .AutomaticallyUpdate = False
        .LanguageID = CustLanguage
        .NoProofing = False
        .Frame.Delete
        With .ParagraphFormat
            .LeftIndent = 0
            .FirstLineIndent = 0
            .SpaceBefore = 4
            .SpaceAfter = 10
            .LineSpacingRule = wdLineSpaceSingle
            .Alignment = wdAlignParagraphCenter
            .WidowControl = False
            .KeepWithNext = True
            .KeepTogether = True
            .PageBreakBefore = False
            .NoLineNumber = True
            .Hyphenation = False
            .OutlineLevel = wdOutlineLevelBodyText
            .TabStops.ClearAll
        End With
    End With
End If
End Function

Function ProcessGraphic(Rotate As Long)
Dim ScaleFactor As Single
Dim aBorder As Long
Dim theRange As Range

Set theRange = Selection.Paragraphs(1).Range ' Mark it, we're going to need it

With Dialogs(wdDialogInsertPicture)
    If .Display = -1 Then ' if user said OK
        theRange.Style = "Graphic" ' Ensure placement style
        theRange.InlineShapes.AddPicture FileName:= _
            .Name, LinkToFile:=False, SaveWithDocument:=True ' insert the pic
    Else
        End ' Go home, user changed mind
    End If
End With

If Rotate <> 0 Then ' Rote the pic
    With theRange
        .InlineShapes(1).ConvertToShape ' Can't rotate if it's inline
        .ShapeRange(1).Rotation = Rotate ' Rotate is in degrees, - means anticlockwise
        .ShapeRange(1).ConvertToInlineShape ' Make it inline again
    End With
End If

With theRange.InlineShapes(1)
    .LockAspectRatio = False
    ScaleFactor = (.Height / .Width) ' derive aspect ratio
    .Width = CentimetersToPoints(5#) ' Set a required width
    .Height = .Width * ScaleFactor   ' Scale height in proportion
    For aBorder = -4 To -1 ' Borders all round
        With .Borders(aBorder)
            .Color = PicBorderColour
            .LineStyle = wdLineStyleSingle
            .LineWidth = wdLineWidth075pt
        End With
    Next
End With

End Function

rich michaels - mvp (greatcirclelearning.com) say:

Is there a reason you want to write your own code for this? For $20 U.S. there is a plug-in app that accomplishes it including inserting the pictures (one or by batch), provides a function to change/update link type. Pictures can be in any of the standard formats or even PowerPoint slides and you can have full control of the resizing of the images. The plug-in is called doc-Tools and you can click on the link to read about it. I am the author and it’s free to try.


Now assuming you want to go down this complex road yourself here are some tips.


There are three ways pictures can be added into a document. There is embedded, inked and third is a combo of the two, linked and saved with the document. The following is a code sample that uses a user form selection to determine the add type for the picture.


Dim rng As Word.Range

Set rng = Selection.Range

Dim Path as String

Path = “Enter your file path and file name”

If Me.EmbedButton Then

     rng.InlineShapes.AddPicture Path

Else

     If Me.LinkButton Then

            rng.InlineShapes.AddPicture Path, True, False

    Else

          rng.InlineShapes.AddPicture Path, True, True

   End If

End If


In the above code you see the three variations on the AddPicture command. To embed you only need the file name (and path). To link two Boolean values are added and the first must be True and second False. Finally for the linked and saved option both Booleans must be True.


Pictures in a document are part of the Shape collections and to locate them you have to look at both the inline shapes and shapes with wrapping text (aka floating shapes) collections. You declare them separately.


    Dim shp As Word.Shape

    Dim iShp As Word.InlineShape


As John has indicated, working with floating shapes can be very difficult, often you first have to convert them to inline “shp.ConvertToInlineShape” and then when done with your manipulations convert it back “ishp.ConvertToShape”.


These two shape type collections consists of more that just pictures so you have to interrogate each shape you find in the document to see if it is a picture and then is it a embedded or linked type of picture. Here is code that looks at the inline shapes found.


    For Each iShp In doc.InlineShapes

        Select Case iShp.Type

            Case Word.WdInlineShapeType.wdInlineShapePicture

                im1Count = im1Count + 1

            Case Word.WdInlineShapeType.wdInlineShapeLinkedPicture

                im2Count = im2Count + 1

            Case Else

                shpCount = shpCount + 1

        End Select

    Next


Finally, when you are working with pictures that are linked (remember there are two types, linked and linked/saved) you use the “iShp.LinkFormat.Update” command to refresh the link on these two types of pictures.


I hope this helps

myall_blues say:

OK. Given that information, and on reflection, I think it's a lot easier to do this with a user defined function that uses cells in column P as input to trigger the change.

So use the following code (note this code goes at the module level).:

Public Function TimeStamp(ByVal Data) As String

    TimeStamp = Format(Now(), "yyyy-mm-dd hh:mm:ss")

End Function

Change the format of the timestamp to how you want it to look. Then in column W (or whichever column you want the timestamp in), put the TimeStamp function with the relevant reference to column P as the input.

For example, cell W486: =TimeStamp(P486)

cell W515: =TimeStamp(P515)

Now whenever something changes that causes cells in column P to update, their corresponding cell in column W will update.

*** Edit 2016.10.20 - 6:10PM ***

THANK YOU to all the Windows Insiders who have provided us feedback on getting stuck at 0% while trying to download Build 14951 on their devices. We have narrowed this down to a new issue that only seems to affect devices on Windows 10 Mobile build 14946. We are making a change to how Build 14951 is published, so that Insiders can avoid hitting this issue. We will also make the same changes when we published the next build so Insiders do not hit this when upgrading to newer builds.

We appreciate all of the feedback that has helped us out.  We are still working on validating this resolves everyone’s issues; please keep filing feedback if you run into issues trying this out.

Here are 4 options we are validating to get out of this state: 

  • FASTEST  - You can force the update session to time out by:
    • Plug your device into a charger
    • Open Settings > Update & Security > Windows Insider Program -  Change your device to "Slow" and then back to "Fast"
    • Turn on airplane mode Settings > Network & wireless > Airplane Mode
    • Navigate to Settings > Time & Language > Date & Time -  If On, turn “Set date and time automatically” to the Off position
      1. Change the date on the device to 7 days in the future
    • Return to the home screen & Reboot the device
    • After the reboot, turn off airplane mode - Settings > Network & wireless > Airplane Mode
    • Check for updates - Settings > Update & Security > Phone update > Check for updates
    • This process will fail with an error code.  This is expected!
    • Change the device date back to the current date - Settings > Time & Language > Date & Time
    • Check for updates -  Settings > Update & Security > Phone update > Check for updates
    • This check may fail again, this is ok.
    • Reboot the device
    • After the reboot, check for updates - Settings > Update & Security > Phone update > Check for updates
    • The check for updates should succeed and begin downloading 14951

      *Note*  We have seen several instances of users reporting getting error code 80242006 after this last check for updates.  We will be doing some additional testing on our side for this, but in the mean time, Insiders are awesome and have noted these additional steps seem to bypass this final error message:

    • Switch Insider setting to Slow ring
    • Check for updates (will say you are up-to-date for the Slow ring)
    • Change back to Fast ring
    • Check for updates again and it should update


    And if you're *still* stuck?  Here's one additional set of steps to try:

  • EASIEST-LONGEST - If your device is already in this state, it will timeout after 5 days. Once the update times out, your device will get the updated metadata and should move forward as normal with download Build 14951.
  • MEDIUM WORK - If your factory reset your device on Build 14946 - Settings->System->About->Reset your phone – you will be able to download 14951 without issue now.
  • MOST WORK - Less optimal, but if you have already rolled back to an older build such as Windows 10 Version 1511 or Windows 10 Anniversary Update using WDRT, you can easily move forward avoiding this issue.
