American to european quotation marks.

After finishing a very long paper I realized that in Poland they placed quotation marks differently there. So instead of "He went" they have „He

P.S. Tried CTRL+H didn't change anything

graham mayor say:

The quotes that are applied when you convert to smart quotes are language dependent. Either you have used straight quotes, or you have a formatted the text with a language, other than Polish, which uses English style smart quotes.

Strictly speaking if the language of your text is English, then the quotes you have are correct. The fact that it is for a Polish audience, should be immaterial. If the language is Polish, then having the Polish proofing language applied to the text will automatically convert straight quotes to the type of Polish smart quotes you require.

If you want to convert the quotes from English smart quotes to Polish smart quotes, then the following macro should do the job. First convert smart quotes to straight quotes. Apply the Polish formatting language to the text and re-run the macro and convert straight quotes to smart quotes. (It will also convert English and French style smart quotes provided the appropriate language is applied to the text.).

Sub ReplaceQuotes()
Dim vFindText As Variant
Dim vReplText As Variant
Dim sFormat As Boolean
Dim sQuotes As String
Dim oRng As Range
Dim i As Long
    sQuotes = MsgBox("Click 'Yes' to convert smart quotes to straight quotes." & vbCr & _
                     "Click 'No' to convert straight quotes to smart quotes.", _
                     vbYesNo, "Convert quotes")
    sFormat = Options.AutoFormatAsYouTypeReplaceQuotes
    If sQuotes = vbYes Then
        vFindText = Array(Chr(132), Chr(147), Chr(148), ChrW(171) & ChrW(160), ChrW(160) & ChrW(187), Chr(145), Chr(146))
        vReplText = Array(Chr(34), Chr(34), Chr(34), Chr(34), Chr(34), Chr(39), Chr(39))
        Options.AutoFormatAsYouTypeReplaceQuotes = False
        For i = LBound(vFindText) To UBound(vFindText)
            Set oRng = ActiveDocument.Range
            With oRng.Find
                Do While .Execute(vFindText(i))
                    oRng.Text = vReplText(i)
                    oRng.Collapse wdCollapseEnd
                Loop
            End With
        Next i
    Else
        Options.AutoFormatReplaceQuotes = True
        ActiveDocument.Range.AutoFormat
    End If
    Options.AutoFormatAsYouTypeReplaceQuotes = sFormat
End Sub

Combo Boxes in Word 2010

jay freedman say:
In Word 2010 there are two kinds of combo boxes that can be on the document surface (not counting the kind that can be put into a UserForm through the VBA editor):

The one that you can see among the other buttons in the Controls group of the Developer tab is a "content control". There is no way to make the down arrow of this kind permanently visible -- the best you can do is to put a label next to it, or put it into a table so it has borders around it. (The design philosophy of all content controls includes making them look just like ordinary text when the document is read or printed.)

If you click the third button in the bottom row -- the folder-and-tools icon -- and look under the ActiveX heading, you'll find a combo box button there. I usually recommend avoiding ActiveX controls because in large numbers they tend to be ill-behaved, but a few per document should be OK. When you insert one of these, the Design Mode button becomes active and you can click the Properties button to open the Properties list. One property is named ShowDropButtonWhen, and its default value is fmShowDropButtonWhenAlways -- that is, the arrow will always be visible. (It will print, as will the borders of the box.)

The big drawback of the ActiveX combo box is that it doesn't store the values in its list, or the user's choice/entry in the box. You must write an AutoOpen or Document_Open macro to put the values into the list every time the document is reopened, and other macro code to save the user's choice so it can be shown on reopening. Then you get into the mess around macro security, because depending on the user's settings Word may prevent any macros from running, with or without notification to the user.

justin cavida msft say:

Hi KTB****,

We would like to inform you that protecting a SharePoint site with a password is not feasible. SharePoint uses permissions to restrict access to a site, list, or library.

As per your description, your requirement is to prevent users to see all the records on a list and allow specific users to have the right to access all records on the list. If so, you can do this by granting Full Control permission to certain users who can access all the records and modify Item-level Permissions to make the users only see their own records. To do that, please follow the steps below.

To grant Full Control permission to a user:
a. Go to List settings by clicking the gear icon on the upper-right of the screen

b. Select Permissions for this list under Permissions and Management

c. Click on Stop Inheriting Permissions to break the permissions inheritance for the list

d. Click Grant Permissions and specify the users who you want to have access to all records

e. Click SHOW OPTIONS at the bottom and in Select a permission level drop down, select Full Control

Next, to prevent users to access all records and access only their own:
a. Go to List settings again and select Advanced settings under General Settings
b. In Item-level Permissions section, choose Read items that were created by the user and Create items and edit items that were created by the user

Please let us know if you have any questions.

Regards,
Justin
