sylestra say:

I've got the same issue.  I opened up one of the .dmp files in VS and discovered that it's NahimicSvc64.exe.  It's just continuously crashing, so generating a .dmp file about every 1 to 5 seconds(depending on system speed) and then auto copying into the clipboard.  This overwrites anything you manually copied.

To get copy/paste functionality back, go to task manager and kill the NahimicSvc64.exe process (It's probably the highest CPU usage going on when it's crashing, so easy to find.)

Nahimic is what SonicStudio3 uses, which is part of the audio suite that Asus ships out for use with the ROG series systems.  
General guidance from Asus seems to be to make sure everything on your system is updated, sadly this doesn't seem to solve all the issues.
General guidance from the community seems to be to uninstall SonicStudio and just install RealTek drivers, but that does lose some of the functionality that SonicStudio has to offer.

Hopefully this helps.  If you are experiencing a similar issue but don't have NahamicSvc64.exe on your system, I'd recommend opening one of those dmp files up in a free version of Visual Studio Community and seeing what process it's showing as crashing.

FWIW I'm also using a ROG Strix Z390-H
