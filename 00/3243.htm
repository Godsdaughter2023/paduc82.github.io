richard xu msft say:

Hi Gary,

Welcome to community. I understand that you want to recover a mailbox you deleted.

About delete a mailbox, we have two kinds situation about it:

  • Soft-deleted user mailboxes

A soft-deleted user mailbox is a mailbox that has been deleted using the Office 365 admin center or the Remove-Mailbox cmdlet in the Exchange Management Shell, and has still been in the Azure active directory recycle bin for less than 30 days.

  • Hard-deleted user mailboxes

The user mailbox has been soft-deleted for more than 30 days, and the associated Azure active directory user has been hard-deleted. Check out the Remove-MsolUser cmdlet. All mailbox content such as emails, contacts and files will be permanently deleted. And The soft deleted mailbox has been deleted using the Remove-Mailbox -PermanentlyDelete cmdlet and parameter in Exchange Management Shell.

If a mailbox was deleted more than 30 days and isn't in any of the hold states, like Litigation hold or eDiscovery hold. This mailbox will delete permanently and can’t be recovered.

If a mailbox was deleted less than 30 days and you want to recover it, you can follow this reference.

Note: The method for restoring a mailbox depends on whether the mailbox was deleted by deleting the Office 365 user account or removing the Exchange Online license. When a user’s license is removed, all data that is associated with that user account is held for 30 days. After the 30 day grace period, the data is deleted and can’t be recovered, except for documents that are saved on SharePoint Online.

Karen

 

 

joeu2004 say: Karen wrote:

I am using Excel 2003.  The following SUMPRODUCT works fine if I manually enter the year "2011", but won't work when using a cell reference of D2.

=SUMPRODUCT((Data!$F$2:$F$4044=D2)*(Data!$J$2:$J$4044="Apr")*(Data!$A$2:$A$4044="01 Frozen")+(Data!$F$2:$F$4044=D2)*(Data!$J$2:$J$4044="May")*(Data!$A$2:$A$4044="01 Frozen"),(Data!$L$2:$L$4044)) 

 
That depends on how 2011 is put into D2, and how 2011 might appear in Data!$F$2:$F$4044.

If what works in the SUMPRODUCT is Data!$F$2:$F$4044="2011" with quotes, then you need to put text "2011" into D2, not a number.  Conversely, if Data!$F$2:$F$4044=2011 without quotes works in the SUMPRODUCT, then you need to put a numeric 2011 into D2, not text.

To understand better, enter the following formula:  =2011="2011".  Note that it returns FALSE.

If you need text "2011" in D2, there are several ways to accomplish that.  One is the formula ="2011".  Another is to set the cell format to Text can enter '2011 -- that is, prefix the text with an apostrophe (single-quote).

jay freedman say:

This sounds to me like VBA support in Office 2010 was omitted or removed during the update to Windows 10. That would be more likely if you did a clean installation of Windows 10 and then installed Office from scratch, but less likely if the Windows update was installed on top of a previous Windows version with all applications still in place.

What you can do:

  • Go to Control Panel > Programs and Features.
  • Click on Microsoft Office 2010 and then click the Change button on the toolbar. You may have to respond to a UAC prompt.
  • On the first page of the installer dialog, select the Add or Remove Features option button and click Continue.
  • On the second page, expand the Office Shared Features icon.
  • Click the Visual Basic for Applications icon, and choose the Run from My Computer item in the menu.
  • Click Continue. You may be prompted for the original installation disk, if the original installation wasn't cached on the hard drive. Wait for the operation to complete.
