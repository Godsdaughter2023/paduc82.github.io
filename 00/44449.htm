Search indexing appears to require SYSTEM permissions? - Microsoft Community

a. user say:

Search indexing appears to require SYSTEM permissions?

Windows 7 Search wasn't turning up results from certain folders, despite them being included in Libraries that should have resulted in them being indexed. I'd tried the usual solutions (Search troubleshooter (reported no problems), index rebuild (no change)).

I eventually found the problem was the Windows Search service (or perhaps a separate Indexing service?) was unable to access the libraries even though my user could -- the ACL only granted permissions specifically to me and to HomeGroup (probably irrelevant).

I solved the problem by granting read rights to the SYSTEM user, which appear to usually be inherited from the root of the volume, but I just granted them to the folders I wanted indexing. One rebuild later and search results are as expected once again.

So, if like me the usual index rebuild isn't working, try checking the Security tab and see if the SYSTEM user is being denied access. And a bug fix request for MS -- perhaps check that the indexer can see a folder when adding it to a Library or otherwise requesting indexing and warn / suggest amending permissions if it can't be done.
