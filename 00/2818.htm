In Outlook 2013 and Outlook 2016, Contact Linking option is gone. You  can still link Contacts by clicking File > Properties or Options dialog of an open item.

You can also enable the Contact Linking field by editing the registry. Please follow the steps below:

  • Press Windows key+R. Type regedit, then hit Enter.
  • Search for HKEY_CURRENT_USER\​Software\​Microsoft\​office\​16.0\​outlook\​preferences.
  • Right-click on the right-side, then point the mouse cursor to New.
  • Click on DWORD. Paste showcontactfieldobsolete as New Value#1, then hit Enter.

  • Right-click on showcontactfieldobsolete, then click Modify.
  • Put on Value data = 1 to enable, Value data = 0 to disable.

  • Click OK.
  • Restart your Outlook to save the changes.

It is not feasible to delete all responses at a time.

Can i have workstations with Access 97 and Access 2010 edit the same - Microsoft Community

joeor say:

Can i have workstations with Access 97 and Access 2010 edit the same database on my server?

When I try to open the old file with 2010 it asks to convert it. Will there be any problem if i do not convert? Will I be able to edit from both new and old Workstations?
john w. vinson say:

In this case you MUST split the database into a backend (containing just the tables, in A97 format, on the server) and one or more frontends (containing the forms, reports, queries and code with links to the backend). You can have different frontends in different versions, but the backend must be in the lowest version - you can't read an A2010 database using A97.

The two versions don't "play nice" together. If you convert it to an .accdb, or even to a 2010 format .mdb, A97 users will not be able to open or use the database at all.
