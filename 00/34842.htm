lemp say:

As Hairy Fool says, these are core Windows files.  If they genuinely are missing from your system, almost no printer will work, so returning the current printer for another different one is not the solution.

The first step would be to open an Elevated Command Prompt and run the System File Checker:

  • Click the Start Orb
  • Type "cmd.exe" (without quotes) in the Search box
  • Right-click on the link for cmd.exe in the search results and select "Run as administrator"
  • In the black Command Prompt window, type the following and press Enter
         sfc /scannow
  • What message do you get when the scan completes?

On a normal Windows 7 installation, there will be multiple copies of both files in various locations.  However, for the printing system to work, a copy of each file must be in the correct location, i.e., in C:\Windows\inf.

The System File checker should be able to get a copy of each file from one of the other locations and put it in the proper folder.


jake8610 say: The scan message is "Windows Resource Protection did not find any integrity violations" .  What is the next step?

lemp say:

Click the Start Orb, type the following into the Search box, and press Enter

         C:\Windows\inf

Scroll through the contents of the folder.  Are usbprint.inf and usbstor.inf now present?

If yes, then try to install the printer again; it should work now.

If no, navigate in the open file manager window to C:\Windows\System32\DriverStore\FileRepository

Type usbprint in the search box in the upper right corner

If usbprint.inf is found, right-click it and select Copy.  Then navigate to C:\Windows\inf and right-click > Paste the file.

Go back to C:\Windows\System32\DriverStore\FileRepository and do the same as above for usbstor (my system finds 2 copies of usbstor.inf this way; if you find multiple copies as well, copy and paste the one with the most recent date).

If you are still missing either of these files, let me know and I'll put copies of mine in my OneDrive for you ... but because these files should not be missing, your system may turn out to be compromised in additional ways.  You probably should do a scan with the ESET free Online Virus Scanner (http://www.eset.com/us/online-scanner/) and download, install, update, and run the free version of MalwareBytes AntiMalware (https://www.malwarebytes.org/antimalware/)

to apply a registry change by adding   TCPNoDelay to MSMQ folder.


Hi

OK. It is not uncommon for those Keys to be missing on some systems. Here is how to create the keys.

Open the registry editor and navigate to:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft

Right click the Microsoft key and select New /Key.

Name the new key  MSMQ  (case sensitive).

Next, right click the new MSMQ key and select New /Key again.

Name this key Parameters  (case sensitive).

From there, you can continue with the instructions in that other thread.

Here is the standard warning about editing the registry:

Warning: Serious problems might occur if you modify the registry incorrectly by using Registry Editor or by using another method. These problems might require that you reinstall the operating system. Microsoft cannot guarantee that these problems can be solved. Modify the registry at your own risk. SeeHow to back up the registry

Hope this helps.
