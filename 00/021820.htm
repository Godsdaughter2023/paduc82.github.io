doug robbins - mvp office apps & services (word) say:

You will have to use VBA to do this.

To get the number of Worksheets (other than those with a name that begins with Sheet), use a macro containing the following code:

Dim i As Long, j As Long
With ActiveWorkbook
    For i = 1 To .Sheets.Count
        If Left(.Sheets(i).Name, 5) <> "Sheet" Then
            j = j + 1
        End If
    Next i
End With
MsgBox "The workbook contains " &  j & " sheets."

The following code will add a sheet to the workbook that it names Shop List and it will then populate that sheet with the names of the shops (from the sheet names) and the contents of cell D16 on each sheet:

Dim i As Long, j As Long
Dim ShtShops As Worksheet
j = 1
With ActiveWorkbook
    Set ShtShops = .Sheets.Add
    With ShtShops.Range("A1")
        .Offset(0, 0).Value = "Shop"
        .Offset(0, 1).Value = "D16"
    End With
    For i = 1 To .Sheets.Count
        If Left(.Sheets(i).Name, 5) <> "Sheet" Then
            ShtShops.Range("A1").Offset(j, 0).Value = .Sheets(i).Name
            ShtShops.Range("A1").Offset(j, 1).Value = .Sheets(i).Range("D16").Value
            j = j + 1
        End If
    Next i
    ShtShops.Name = "Shop List"
End With

Resetting Windows Update Components will fix corrupt Windows Update Components and help you to install the Windows Updates. Please follow the below steps to reset the Windows Updates Components:

1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

5. Type Exit in the Command Prompt to close it.

Now you may try running the Windows Updates and check if the above steps resolve the issue.

breadbox88 say:
What I've found:

The issue appears to be associated with account "Signatures" that append to the end of a new email.  If I create a new email that has my default signature my typing is delayed by 1-5 seconds.  If I "select all" and delete the signature... my typing returns to normal.

The fix (which works but is bs that it has to be done this way) is to open a new email > click "Signatures: Edit" > click "Defaults" > click the drop-down associated with your email account > select "None".

Now when you open a new email there will be no default signature.  Typing should be fine.  When done with the email click "Signatures: YOUR SIGNATURE" and it will append to the end of the email.
