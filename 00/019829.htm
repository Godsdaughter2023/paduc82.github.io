Sub AnimateFont()
Dim sAnimation As String
If Len(Selection.Range) = 0 Then
MsgBox "Select text first!", vbCritical, "No Text Selected"
Exit Sub
End If
sAnimation = InputBox("Which animation? Enter the number: " & vbCr & _
" 1. Blinking Background" & vbCr & _
" 2. Las Vegas Lights" & vbCr & _
" 3. Marching Black Ants" & vbCr & _
" 4. Marching Red Ants" & vbCr & _

" 6. Sparkle Text" & vbCr & _

Select Case sAnimation
Case 1: Selection.Font.Animation = wdAnimationBlinkingBackground
Case 2: Selection.Font.Animation = wdAnimationLasVegasLights
Case 3: Selection.Font.Animation = wdAnimationMarchingBlackAnts
Case 4: Selection.Font.Animation = wdAnimationMarchingRedAnts
Case 5: Selection.Font.Animation = wdAnimationShimmer
Case 6: Selection.Font.Animation = wdAnimationSparkleText
Case 0: Selection.Font.Animation = wdAnimationNone
Case Else:
End Select
End Sub

How too set the font when using the mailto command in Outlook 2016 - Microsoft Community

anthonycook1 say:

How too set the font when using the mailto command in Outlook 2016

This is not a question, but a fix. As I had searched the internet for an answer and could find one and managed to solve the problem I thought I should share.

When using the mailto command from another program or website - that then contains some text for the body of the email message this text was not the default font that I had selected. So any extra text I was typing was in my default font, in my case Times New Roman (12) but the body of text that had come from the mailto command was in Calibri (11).

That was the problem and to fix the text font and size for the mailto command you need to (in Outlook 2016) click File, then select Options. From there select Mail and the click Stationary and Fonts. Once this opens make sure you are on the Personnel Stationary tab (but this seems to be the default tab that opens) and then from there you can see the default fonts. Now here is the strange part - you will see your normal email message font in the "New Mail Message" box (in my case Times New Roman 12) which is fine and correct. But in the "Composing and reading plain text messages" box is the font you need to change to set the font for the mailto command, as for my example this was set to Calibri (11) and once changed to the same as the font as the "New Mail Message" font, again in my case Time New Roman (12) the mailto command now uses the correct font in the body of the email message.

"New Mail Message" font, again in my case Time New Roman (12) the mailto command now uses the correct font in the body of the email message.

Problem solved.

john w. vinson say:

Don't confuse data display with data contents!

 

DateTime fields are not stored with either mm-dd-yyyy or yyyy-mm-dd format, or any other format. What's actually stored in the database is a double precision floating point number, a count of days (and fractions of a day, times) from midnight, December 30, 1899. The format is applied later, when the floating point number is pulled from the table and converted into a text string to display on the screen.

 

The format can be specified in the Form design, or a step back in the Query design, or yet further back in the Table design; but my guess is that this user has the format set yet further back, in the Windows setting. In Control Panel... Region and Language... Formats you can specify what Access means by "Short Date". If your database is set up to show dates as Short Date format, it will read this computer-wide setting. Try opening this user's computer's Region and Language setting and see what it's using for "Short Date"; either change it there, or (if the user has other applications which depend on the yyyy-mm-dd format) go into the design of the Report and change the Format property of these date fields from "Short Date" to "mm-dd-yyyy".
