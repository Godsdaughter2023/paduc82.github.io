Here's the code. But it has worked quite well in the past.

 

Option Explicit

Sub getTopRatedStocks()
Dim sht As Worksheet
Dim fso As New FileSystemObject
Dim txt As TextStream
Dim qry As QueryTable
Dim ie As InternetExplorer
Dim doc As New HTMLDocument
Dim tbl As HTMLTable
Dim strtxt As String
Dim msnfilename As String
Dim url As String
Dim i As Integer
Dim w As Integer

    w = 10
Do
  url = "http://moneycentral.msn.com/investor/StockRating/srstopstocksresults.aspx?"
  Set sht = Sheet1
 
  sht.UsedRange.EntireRow.Delete
 
  Do
    i = i + 1
    Set ie = New InternetExplorer
    sht.Cells(1, 1).Value = "Please wait. Retrieving data from " & url & "sco=" & w & "&page=" & i & "&col=" & 13
    ie.navigate url & "sco=" & w & "&page=" & i & "&col=" & 13
    Do Until ie.readyState = READYSTATE_COMPLETE
      DoEvents
    Loop
    Set doc = ie.document
    Set tbl = doc.getElementById("dataTable")
    If i > 1 Then
      tbl.deleteRow 0
    End If
    strtxt = strtxt & tbl.innerHTML
    If InStr(1, tbl.nextSibling.childNodes(0).innerHTML, "Next") = 0 Then
      i = 0
    End If
    ie.Quit
  Loop Until i = 0
 
  msnfilename = "msn.html"
  With fso.GetSpecialFolder(2).CreateTextFile(msnfilename, True)
    .Write "<table id=dataTable>" & strtxt & "</table>"
    .Close
  End With
 
  Set qry = sht.QueryTables.Add("URL;" & fso.GetSpecialFolder(2).Path & "\" & msnfilename, sht.Cells(1, 1))
  qry.WebTables = "dataTable"
  qry.Refresh
  qry.Delete
  If i = 0 Then
    w = w - 1
  End If
Loop Until w = 0

End Sub

greginmich say:

Automatic Maintenance runs a Windows Defender Quick Scan every day – provided that you leave a slot of idle time that’s long enough for the scan to complete. This scan doesn’t run like an old-fashioned scheduled scan does, because it gets cancelled (not paused) anytime you start using your PC and then it has to wait for the next idle state to restart. This prevents the scan from degrading system performance and interfering with your work (something that’s always been a major concern with the Microsoft Antimalware Engine; MsMpEng.exe) – but you do have to take a coffee break that’s long enough for the maintenance chores to get done:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-10-how-do-i-set-windows-defenders-scan/f399bb87-37c9-4a74-bb35-1d7e3d3c2436

Microsoft did manage to publish a method for scheduling a scan in Windows 10, but unfortunately that method just runs a clone of the Automatic Maintenance Quick Scan, which is already being run on a daily basis, and which can be “scheduled” in the Control Panel (Security and Maintenance > Maintenance > Automatic Maintenance > Change maintenance settings) – so that doesn’t make much sense:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-i-set-a-schedule-for-windows-defender-to/f54e5090-6dee-4cc4-ad44-40be810488ee  

If you’re having trouble with the Automatic Maintenance scan, though; or if you’d just prefer an old-fashioned kind of scheduled scan, then you do still have the option of using the Create Basic Task Wizard (Task Scheduler) to set up an independent scheduled task:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-i-set-windows-defender-to-a-custom-auto/ec19aab4-10ce-4f41-9990-343056fd104f

The Create Basic Task Wizard really is, and always has been, the key to scheduling anything in the Windows OS – including Windows Defender scans. This allows you to set up multiple scheduled scans of various types, and places the full complement of Task Scheduler options at your disposal:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02
