Windows Update Error - 0x80070005

 

This error code simply indicates a general “Access Denied” which does not help us in determining the source of this issue. This error can be occurring because of improper permissions (ACL) on a particular file or registry key. There are tools and troubleshooting to isolate the specific file or value that is returning the “Access Denied”. However, such troubleshooting is complex and falls outside the scope of these forums.

 

If this error occurs when attempting to download/install a specific update we may be able to locate troubleshooting specific to that update. I’d suggest that if you are attempting to install a group of updates that you attempt installing them one at a time. This will assist in isolating the failed update - as one update in a group will cause the entire group to fail. If you are simply attempting to install one update please try manually installing that update (see below). If the manual installation fails please provide any error codes or other information you receive when attempting the installation.

 

Manually locate and download Windows Updates…

1.     Click Start, click All Programs, and click Windows Update.

2.     Click on the View available updates link.

3.     Right-click the update you would like to locate a manual download for and select View details.

4.     In this detail window click on the link directly under More information.

5.     You will be redirected to a Microsoft Help and Support page directly related to this update.

6.     On this page you will typically find a manual download for the given update.

 

Should the above troubleshooting fail to resolve the issue I’ve included some additional information that will assist you with this error – from a general perspective. Troubleshooting includes verifying the DCOM settings, and resetting the security settings to their default values.

 

Verify the DCOM settings…

1.     Click Start, and type dcomcnfg in the Start Search bar, and press Enter.

2.     Double-click Component Services.

3.     Double-click Computers.

4.     Right Click on My Computer.

5.     Click on the Default Properties tab.

a.     Ensure that Enable Distributed COM on this computer is checked.

b.     Ensure that Default Authentication Level is set to Connect.

c.     Ensure that Default Impersonation Level is set to Identify.

d.     If you made any changes to correct settings click Apply.

6.     Now click on the COM Security tab.

7.     Under Access Permissions click Edit Default.

a.     Ensure that Self, System, and Administrator have Local Access set to Allow.

b.     Ensure that Self and Administrator have Remote Access set to Allow.

c.     If any of these users are missing please add them.

d.     If any of these settings are not checked please set them.

e.     Make necessary changes, and click OK to close this window.

This will help you restore the security settings to defaults on Windows XP and Windows Vista. This solution does not restore all security settings that are applied when you install Windows.
