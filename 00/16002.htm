When creating an account in Outlook 2013 that connects to the "Windows Live" mail server (that is, the mailbox found at http://mail.live.com/), if you allow Outlook 2013 to create the account automatically, it will choose EAS by default.  This will allow full sync between Outlook 2013 and your Outlook.com/Live.com/Hotmail.com mailbox, including contacts and calendar.  You can also use IMAP for all-folder sync, but it will exclude contacts and calendar because IMAP is mail-only.  POP is the only connection type that will limit "sync" to the Inbox, and it's not two-way; it's from the server to the client only.  For a POP account, you would have had to create the account manually.

suzanne s. barnhill say:

If it is the "Hide spelling errors in this document" setting that keeps rechecking itself (as opposed to "Do not check spelling or grammar"), then that is because there are "too many errors." Depending on your system resources, Word will shut down "Check spelling as you type" if there are "too many" words that it considers misspelled. This can happen if your document contains a lot of unusual names or terms, computer code, a foreign language (for which you don't have proofing tools), or the like. There are ways to get around this; see http://wordfaqs.mvps.org/MasterSpellCheck.htm#ExemptingText.

Usually (at least in my version of Word), Word does display a popup error message ("There are too many errors in this document...") before turning "Check spelling as you type" off.

jasles say:

This should work, but I had to create helper cells. For each row, I added 5 cells, each with their own formula that determines if there are 5-10 total rounds per player, then cell Y2 shows the lowest, cell Z2 shows the next lowest, and so on, from cell Y2 to AC2. If there are over 10 rounds per player, it shows the same, but only of the last 10 scores.

Then in cell AD2, the handicap is shown. If there are between 1 and 4 scores, it averages them. If over 5 total scores, it then averages the results from Y2 to AC2. If there are zero scores, the result is blank.

Here is a screen capture of it:

A close-up of the 5 helper cells and handicap cell:

This is the formula in cell Y2:

=IF(COUNT(B2:W2)>10,SMALL(INDIRECT(CELL("address",OFFSET(INDEX(B2:W2,MATCH(9.99999999999999E+307,B2:W2)),0,-9))):INDIRECT(CELL("address",INDEX(B2:W2,MATCH(9.99999999999999E+307,B2:W2)))),1),IF(AND(COUNT(B2:W2)>=5,COUNT(B2:W2)<=10),SMALL(B2:W2,1),""))

the formula in cell Z2:

=IF(COUNT(B2:W2)>10,SMALL(INDIRECT(CELL("address",OFFSET(INDEX(B2:W2,MATCH(9.99999999999999E+307,B2:W2)),0,-9))):INDIRECT(CELL("address",INDEX(B2:W2,MATCH(9.99999999999999E+307,B2:W2)))),2),IF(AND(COUNT(B2:W2)>=5,COUNT(B2:W2)<=10),SMALL(B2:W2,2),""))

the formula in AA2:

=IF(COUNT(B2:W2)>10,SMALL(INDIRECT(CELL("address",OFFSET(INDEX(B2:W2,MATCH(9.99999999999999E+307,B2:W2)),0,-9))):INDIRECT(CELL("address",INDEX(B2:W2,MATCH(9.99999999999999E+307,B2:W2)))),3),IF(AND(COUNT(B2:W2)>=5,COUNT(B2:W2)<=10),SMALL(B2:W2,3),""))

the formula in cell AC2:

=IF(COUNT(B2:W2)>10,SMALL(INDIRECT(CELL("address",OFFSET(INDEX(B2:W2,MATCH(9.99999999999999E+307,B2:W2)),0,-9))):INDIRECT(CELL("address",INDEX(B2:W2,MATCH(9.99999999999999E+307,B2:W2)))),5),IF(AND(COUNT(B2:W2)>=5,COUNT(B2:W2)<=10),SMALL(B2:W2,5),""))

I've had another go this morning, here's what it looks like.

It uses one range name in B2 and one little macro referenced in a formula.

The formulas are.

B2:  =COUNTA(2:2)

B3  =IFERROR(ROUND(D3-72,0),0)

The macro function is

=====================

Function columnName(Number) As String

Dim addr As String

    addr = Cells(1, Number).Address(False, False)

    columnName = Left(addr, Len(addr) - 1)

End Function

===============
