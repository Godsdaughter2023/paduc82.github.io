This will generate a list of lNumbers unique random numbers, each of a value no greater than iMaxVal (edit either/both to suit your needs, but with care:  obviously, you can't have 100 unique numbers each no greater than 20; less obviously, the more numbers you want to generate, the longer this will take to run, but for moderate numbers, it'll be almost instant).

When it's done, the array aNumbers will contain your numbers.  What you do with 'em is up to you.  Or us, if you need more help.

Option Explicit

' What's the maximum value we'll allow?
' Edit to suit:
Const iMaxVal As Integer = 20
' How many numbers should we come up with?
' MUST be equal to or less than iMaxVal
Const lNumbers As Long = 20
   
' we want to fill this array with random numbers
Public aNumbers(1 To lNumbers) As Integer

Sub Thing()
    Dim n As Long
    Dim i As Integer
    
    ' for each position in the array
    For n = 1 To UBound(aNumbers)
        
        ' get a random number to start:
        i = RandomNumber(iMaxVal)
        
        ' if it's already used, get another random number
        ' keep trying until we hit on one that's not used:
        While UsedNumber(i, aNumbers)
            i = RandomNumber(iMaxVal)
        Wend
        
        ' we've found an unused one
        ' add it to the array and carry on
        aNumbers(n) = i
        
    Next
    
    For n = 1 To UBound(aNumbers)
        Debug.Print n & vbTab & aNumbers(n)
    Next
    
End Sub

Function RandomNumber(iMaxVal As Integer) As Integer
    RandomNumber = Int(iMaxVal * Rnd) + 1
End Function

Function UsedNumber(i As Integer, aNumbers As Variant) As Boolean
    Dim x As Long
    For x = 1 To UBound(aNumbers)
        If aNumbers(x) = i Then
            UsedNumber = True
            Exit Function
        Else
            UsedNumber = False
            Exit Function
        End If
    Next
End Function

OK, if you're using the VBA UDF from joeu2004 from his post here: http://answers.microsoft.com/en-us/office/forum/office_2010-excel/how-to-convert-data-in-cells-with-a-specific/886e65f4-3ed0-4515-8fc3-09484d932a1e then you should be able to do the following:

2) In Module1, at the top, between the text "Option Explicit" and "Function myLogMARS(s As String) As String", paste the following as shown in bold here:

 

Option Explicit

 

Sub ConvertSelectedCellsToMyLogMAR()
    Dim c As Range
    For Each c In Selection
        c.Value = myLogMAR(c.Value)
    Next c
End Sub


Function myLogMAR(s As String) As String

rohn007 say:

Collizz, sorry I was not clear enough. 

I am NOT suggesting you change the video card.

I am suggesting you check to see if the "Video Driver" software specific for your video system is up to date.  If you have a separate video card, rather than video built in to the motherboard, then make sure you are using the most current video driver specific for that video card instead of a generic Windows video driver.  Microsoft produces generic video drivers that can run on most (NOT ALL) video systems. But these generic drivers cannot take full advantage of all of the features built into specialist video cards.

A more common version of this issue is printers.  The generic MS printer driver will often be usable on any printer, but it will usually not include features like booklet printing or duplexing or other special document handling. For that, you have to go to the printer maker.

On the other hand, sometimes you do have to fall back to the MS drivers if the hardware manufacturer does not create a driver that works on the current version of Windows (ie if MS changed the underlying system, older drivers from the manufacturer may no longer work. I am thinking specifically of the printer subsystem, but it also applies to a degree to video).
