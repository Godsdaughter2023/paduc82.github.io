Version 16.19 is the current general release build of Office. Your installation is up to date.

The trouble you described is new. The to-and-fro exchange is about the only way to troubleshoot this problem. Microsoft and community experts try to think "what could possibly cause this" and then offer troubleshooting suggestions one at a time until we hit on the solution.

What is Office Insider? - Office Support

If you use Antidote, Webex, Silverlight, EndNote, Zotero, Suitcase, 3rd party font management tools or other add-ins there is a potential for crashes and unwanted misbehavior unless you also update your add-ins to the latest update. Install updates for these programs if you have them.

SAFE BOOT

 

I think the next thing to do is another relatively simple procedure: Perform a Safe Boot by holding the SHIFT key down as your Mac starts up from a restart or shut down.

 

It will take a long time for your Mac to start up in Safe Mode. When you do a Safe Boot Mac OS fixes troubles with your Mac's directory and file system. The font cache is emptied. Only a limited number of fonts are enabled. Applications that run at startup don't run automatically. This allows you to test your app(s) by running them with a minimal set of fonts, an empty font cache, and without potentially conflicting apps running.

 

Once you reach the desktop in Safe Mode try running the app having a problem. Is the problem still there?

 

If the problem is gone: Performing a Safe Boot may have fixed the ailment. Or perhaps one of the fonts that wasn't loaded is bad. Or perhaps some other software utility is interfering. After testing in Safe Mode, do regular restart.

diane poremsky mvp (slipstick.com) say:

There are two ways outlook can complete the To field - you can type a name and Outlook will look for a match in Contacts. ie, you type Bob S and after you tab out of the field, outlook finds Bob Smith. This is autoresolution and works in all versions of outlook.  

The other way is with the autocomplete cache in newer versions of outlook. You start typing and Outlook displays a list of people you've previous sent messages to. The addresses are stored in a file on your hard drive. Unfortunately, its easily corrupted. The addresses are not written to the file permanently until Outlook closes - it something causes it to crash, the names are not saved.

See http://www.slipstick.com/problems/nicknamecache.aspfor more information and possible solutions.

albert d. kallal say:

Just as a heads up.

 

Access with such connections should work just fine with any recent version of SQL server.

 

In fact, since you using linked tables and ODBC then you can mix and match any version of SQL server with any version of access (even rather older versions).

 

So there is no restrictions here. You can use a 15+ year old version of Access such as old access 2000 with a brand new version of SQL server.

 

And you can use brand new Access with say a 10 or even older version of sql server (even older 2005 SQL server will work).

 

What this means is that some “other” reason is at play here as to why your setup does not work.

 

You do not need to upgrade Access to a newer version for this to work. I suspect you have to re-link the tables or some such, but Access and the version of SQL server you use is VERY flexible.

 

So changing access version should not be a problem.

 

And changing what version of SQL server you use ALSO should not be a problem at all.

 

And this issue also applies to VBA code. So some other issue is preventing this, as Access (any version) should work with sql server (any version).

 

No upgrade or change to access should cause this issue in regards to sql server. Perhaps there is some VBA code that is not working but this would be due to some reference issues or some such – not the fact of changing the version of sql server you are using.

 

You are free to use near any version of Access with near any version of SQL server – even if they are both more then 10+ years apart version wise.

karl timmermans [outlook mvp 2012-2018] say:

For info purposes, what kind of email account are you importing the contacts into (POP3, imap, hotmail etc)?

If your import process is still going by the time you read this, all I can say is that if it were my system, I would terminate Outlook, either gracefully if possible or forcibly if not. Importing 4700 contacts should <NEVER> take that long under any circumstances. Regardless of what the reason is for the "24hrs and counting"  - 4700 is simply not that big a number.

If you are importing into a folder contained in a PST file (pop account) there are two potential causes to this excessively slow processing

#1 - There is something seriously wrong with either the PST file you created via the export or the one you are importing into

OR

#2 - You have one of latest integrated Outlook anti-virus products that are nothing but a serious pain. One of the side-effects is an "extreme" slow down in processing - exactly what you are experiencing, Know that from first hand experience with our products being recently victimized by some of these new releases. Remove the integrated Outlook AV and the world returns to normal. So there is no mis-understanding, everyone should have A/V protection - just not <integrated> into Outlook. Some (not all) can cause nothing but problems.

If you need to terminate forcibly - start the task manager via Ctrl-Alt-Delete - end the Outlook application (if its not on the list - switch to the Processes tab and end the Outlook.exe process).

Restart Outlook and check the folder where you were importing the contacts into to see what (if anything) has been transferred.

Would also suggest running ScanPST on your primary PST file. If not familair with ScanPST - the the following article should provide the necessary information

How to find and run the Inbox Repair tool in Outlook

http://support.microsoft.com/kb/272227/en-us

Some other general comments

#1 - There was/is no need to "export" your contacts. All you need to do was copy your original PST file to the other computer, open the PST file in O'2010 and <move> the contacts from the old PST file to the folder in the O'2010 PST file (reason for using <move> is because this is a copy of your original PST so it can be disposed of completely when no longer required.

#2 - Since this involves 4700 contacts, would highlight several hundred contacts (highlight the first contact in the group to be copied, scroll down to the end contact in the group, hold <Shift> key down, highlight the contact, right-click on the highlighted contacts and select <Move to Folder>. Do this until all contacts are copied. Takes a few extra minutes to this in batches but avoids any potential "out of memory" messages.

Once all the contacts have been copied - close the old PST file and proceed normally.

Hello,

current text=> There is a place¹ in...

after apply style=> There is a place1 in...

stefan blom say:
By default, Word formats footnote references with the Footnote Reference character style, which should remain intact when you apply a different style to the paragraph. However, since it doesn't, you can use Find and Replace to reapply the Footnote Reference style. Press Ctrl+H. Specify ^f for "Find what" and then place the insertion point in the "Replace with" box; use Format, Style to select Footnote Reference. Click Replace All.
