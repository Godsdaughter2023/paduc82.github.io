1. EVERY table should have a primary key. The primary key doesn't need to be an autonumber or a single field. But there must be a way to uniquely identify a record.

2. Do you mean Cascade Updates? Cascade Updates will update a related Foreign Key if the PK changes. So lets say you use Social Security Number as a primary and foreign key. It is possible for a SSN to change. If it does all child records will be updated is Cascade Updates is set.

3. Again EVERY table needs a primary key.
