With that, a quick review of the difference between POP3 and IMAP. An email account configured using POP3, ONLY the <Inbox> gets downloaded in a one-time one-direction process. With IMAP, under normal circumstances. all email folders in the IMAP folder tree are sync'd bi-directionally which will happen each time Outlook (any email client to be accurate) connects to the account. This allows any device connected to the email account using IMAP to share the same data which
may not be something you're looking for.

The reason all folders are going thru the sync process now is that you copied the data from the PST file into the IMAP folder tree (an OST data file) .

The question becomes, what do you really want on the new computer - POP or IMAP?

<IF> you just want things to be the same as they were then

#1 - With Outlook closed, start the Control Panel <Mail> app
#2 - Create a <new> Outlook profile and <manually> configure the email account.

How to create profile and set up an e-mail account in Outlook
http://support.microsoft.com/kb/829918

Do this by clicking on <Advanced Options> and checking the box for <Let me setup up my account manually

#3 - During the process, make sure you select using an existing PST file instead of letting Outlook create a new one. Use the PST file where you archived everything to with the following caveat - <IF> in the archive process you did NOT delete anything from the source folders (i.e. delete anything over 1 yr old etc) - then make
sure you use the original PST file being used by the POP account in OL'2013

#4 - Start Outlook and you should be good to go subject to the things you'll need to check as to determine any additional steps to take.

ken sheridan say:

I suspect the link Tom has given you will do something similar as the methodology is pretty standard, but the following module allows you return the current time as UTC:

 

' basUTC

' returns current date/time as UTC (Coordinated Universal Time)

' by calling GetUTC() function

 

Option Compare Database

Option Explicit

 

Type SYSTEMTIME

  wYear As Integer

  wMonth As Integer

  wDayOfWeek As Integer

  wDay As Integer

  wHour As Integer

  wMinute As Integer

  wSecond As Integer

  wMilliseconds As Integer

End Type

 

Declare Sub GetSystemTime Lib "kernel32.dll" (lpSystemTime As SYSTEMTIME)

 

Public Function GetUTC() As Date

 

    Dim utctime As SYSTEMTIME

   

    GetSystemTime utctime  ' put the time and date in UTC time into utctime

    With utctime

        GetUTC = DateSerial(.wYear, .wMonth, .wDay) + TimeSerial(.wHour, .wMinute, .wSecond)

    End With

   

End Function

 

To convert a local time to UTC, allowing for daylight saving time or the equivalent, you just need to subtract the current local time, as returned by Now(), from the current UTC time and add the time to be converted, e.g. in the debug window:

 

TimeToConvert = #2016-08-15 06:30:15#

? GetUTC() - Now() + TimeToConvert

15/08/2016 05:30:15

 

The value is here returned in the UK format of dd/mm/yyyy hh:nn:ss to which my system is set.  BST time here is one hour later than UTC.

ken sheridan say: On second thoughts, while the GetUTC function will work universally to get the current UTC time, the expression to convert a date/time value would only do so correctly if the current date/time and the date/time to be converted are both subject to daylight saving time, or are both standard time.  The Windows API includes a GetTimeZoneInformation function which should be able to deal with that.  I'll have to take a look at Tom's link to see if it handles this.

scheduled clean up - Microsoft Community

vdervan say:

scheduled clean up


I just created my new email account in outlook.com (webmail) and would like to learn more about scheduled clean up features.  Here are my questions:
I just created my new email account in outlook.com (webmail) and would like to learn more about scheduled clean up features.  Here are my questions:

1) What is the default scheduled clean up on an email?

 

2) How do you look up what the scheduled clean up is on any email?

 

3) Can you customize scheduled clean up to automatically purge certain emails - every 90 days, every 6 months, every 12 months, and every 24 months?

 

4) How do you set ‘no scheduled clean up’ on certain emails?  Meaning any emails received from that sender will be kept until manually deleted. 


Thank you for your  help!



a. user say:

Hi,

 

Scheduled clean up is set up before it can take effect on your email.  By default it is set to delete after 10 days or move after 10 days but you have the option to make it 30,60 or 90 days.

 

You can check the scheduled cleanup for an email by opening the email then clicking the schedule cleanup on the Sweep tab.

 

By default it can only be set by 10,30,60 and 90 days allocation and deleted emails will go into your deleted folder. You can delete the email there to purge it out of the system after the 5 day recovery window.

 

You can leave the email as it is no need to set up a no scheduled clean up.

 

Thanks!

a. user say:

Hello Vdervan,

 

Please be informed that as long as you keep your account active, rest assured that your emails are safe with us. If you are using Outlook.com interface, we have a new feature where you can store desired emails on a separate folder called 'Archive'.

 

Follow the instructions below on how you can put emails on Archive folder:

 

  1. Sign in using your Microsoft account.
  2. Tick on the box of the message or messages that you wish to keep.
  3. From the ribbon above, choose Archive. (Note that you if this is the first time that you will be using this feature, you will be prompted to which folder you want to keep these archived messages.)
  4. From the menu, choose Create a folder called 'Archive' if you wish to have a separate folder.

Also I would like to inform you that this feature is not available on Hotmail interface.
