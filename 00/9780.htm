marshall barton say:

You can set the query's RecordLocks property to All Records, but that's supposed to lock the records so nobody can get to them while the query is open.

Because Access creates and runs a datasheet form to display a query's datasheet, I don't know of a query property you can set to make a query's datasheet, form, recordset or whatever readonly.  OTOH, it's easy to make a query non updatable because it's too complex for Access to figure out a way to store a change.  When I need to do that, I just UNION an empty set to the original query:

SELECT f1, f2, ... FROM table WHERE somthing
UNION
SELECT Null, Null, ,,, FROM [any small table] WHERE False

Note that Group By queries are non updatable so they are good to go as is.

Divide Work to complete a task as soon as possible - Microsoft Community

silburned say:

Divide Work to complete a task as soon as possible

I am using MS Project 2013 Standard.

I have a task that has a duration of 234 days.  I want to assign 8 resources to this task to complete it faster.  However two of the resources are on other tasks and won't start until later.  I know how to schedule the resources so they aren't over-allocated.  The problem is that each resource is given the same amount of work for the task (234hrs).  This makes some resources finish days or even weeks before others due to the resources not starting all at the same time.

I want project to divide the work so that all resources will finish at roughly the same time.  This means that some resources will have more work and some less.

How do I this?  Can it be automatic or do I need to manually adjust the end times for each resource?

julie sheets say:

I'm assuming you have the task set to effort driven.  For illustration, let's say R1, R2, R3, R4, R5, and R6 are the six resources who can start the task on the initial start date  (May 1) .Each resource has 312 hours of work and the task duration is 39 days - end date June 24.   The total work is 1872 hours.

R7 can join the task 20 days into the task and R8 can join 5 days later than R7. 

Insert the actual and remaining duration fields into your view.  Set the task to 19 days actual duration which will calculate 20 days remaining.  It will also calculate actual work - 912, remaining work 960.

Assign R7 to the task.  The remaining work (960) is now divided by 7 and R7 is assigned 137.15 hours of work.  The remaining duration drops to 17.14 days and the total duration drops to 36.14 days.  R7 doesn't start the work until May 28 but finishes with the other resources on June 22.

Increase the actual duration to 22 days. The remaining duration is 14.14 days and remaining work is 792 hours. Assign Resource 8.  Resource 8 is assigned 99 hours of the remaining work and the task duration drops to 34.38 days.

Set the % complete to 0.   Project will retain the delayed start dates for the resources, plus their portion of the work plus the calculated duration.

I hope this helps.

julie sheets say:

Unless told otherwise, Project will divide the work on the task based upon the assignment unit.  So yes, if  a resource has non-working time in his/her schedule they may finish the task later than others.

Project cannot possibly know that you think the resource should work 8 hours less because of non-working time.  You'll need to accommodate the changes manually.

sammy mah say:

I just fixed this.

User was changed from Exchange to POP3 email, using the same email address of the user.

Solution was to delete the AutoComplete list in Outlook 2013.

File, Options, Mail, (scroll down to find) Send Messages, then click the button Empty Auto-Complete List.

Outlook was trying to look for Autocomplete email contacts that were in the Exchange Autocomplete list.  So clearing it out, means Outlook will now look in the current .PST file for Autocomplete contacts.  That means the Autocomplete list has to be rebuilt slowly, by emailing contacts again.

Autocomplete is built into the Outlook .PST file, and there is not .nk2 file anymore. Only for Outlook 2007 and older.
