questmsg - mcp say: Hi Nancy R,

The difference between firewall and anti virus are :

A firewall program blocks intruders from accessing your PC over the Internet or a local network. The best firewalls also allow you, on a case-by-case basis, to stop programs that are already on your PC from connecting to the Internet. Firewalls may stop somebody from planting a virus, or worm, on your PC, and they may stop a malicious program already on your PC from using the Internet. But they don't detect or eliminate viruses.

An antivirus program monitors a PC for viruses that might have entered via paths a firewall doesn't usually block, such as an infected e-mail message, an illegal music download or even afloppy disk you borrowed. If it locates a virus, it tries to remove it, or to "quarantine" it so it can'tdo damage. The best antivirus programs watch over your PC all the time, scan incoming and outgoing e-mails, and then run complete scans every week or so

Antivirus programs associated with Microsoft can be verfied using the link - http://support.microsoft.com/kb/49500

lemp say:

the lady I spoke with said my Activation Code had been "pirated".  Strange since I have been using this copy of Win7 Pro for about three years now and it was Activated when I first obtained it. 

Not at all uncommon, although three years is a bit longer than it takes most improperly sold product keys to get blocked.

Many so-called "pirated" Microsoft product keys are legitimate volume license keys that are not intended for resale.  Because they are, in fact, legitimate keys, they very often will activate properly (especially if used soon after purchase).  They may continue to be perceived as valid keys until Microsoft "discovers" that a particular key has been misused, whereupon that key is blocked.

Subscribers to Microsoft's developer and technical programs have access to product keys for a wide variety of Microsoft products.  Such product keys often capable of multiple activations.  See, for example, several of the product key types listed here:  https://docs.microsoft.com/en-us/visualstudio/subscriptions/product-keys 

If an MSDN subscriber has several Windows product keys each of which permits, say, 10 activations, or if a volume licensing customer has a MAK key that is good for 50 installations of Windows 7 Pro, they can sell those keys (in violation of their agreement with Microsoft) and the purchasers will be fine ... until the activation limit for the key is reached. 

That may just result in the key not working for the unfortunate purchaser who bought a key that had reached its limit.  However, if too many such purchasers attempt activation after the limit has been reached, Microsoft may decide that the key was in fact sold in violation of the agreement and block all uses of the key, including those that previously had activated successfully.

This Ed Bott article from 2012 describes the situation:  http://www.zdnet.com/article/to-fight-piracy-microsoft-tightens-msdn-and-technet-terms-again/

The questions in my earlier post are intended to determine if you were caught in this net.

Hi Kyle,

Thank you for posting your query in Microsoft Community.

I understand that you are not able to delete a shared folder and getting the error "folder in use". I will be happy to answer your query.

If you don't have the proper rights (or permissions) for a file or folder, you can't delete it. If you didn't create the file, you might not be able to delete it, even if it is stored in the Public folder. If that's the case, you need to ask the owner of the file to delete it.

"Folder in use" means that the file is being used by a program. To delete the file, you need to find the program that the file is open in and close both the file and the program. If you close the program, but the file still won't delete, restart your computer. You can view all programs that are currently running on your computer in Task Manager.

You can open Task Manager by right-clicking the taskbar, and then clicking Start Task Manager.

Close all open programs and then try to delete the folder. If this doesn't work, restart your computer and try to delete the folder again. Most likely, the folder wasn't deleted because another program was using a file that was in the folder.

If the issue still persists, you can place the computer in clean boot state and check.

To help troubleshoot error messages and other issues, you can start Windows by using a minimal set of drivers and startup programs. This kind of startup is known as a "clean boot." A clean boot helps eliminate software conflicts.

Placing your system in Clean Boot state helps in identifying if any third party applications or startup items are causing the issue. You need to follow the steps from the article mentioned below to perform a Clean Boot.

How to perform a clean boot in Windows
