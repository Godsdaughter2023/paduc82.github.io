First we need to see if you have a TPM chip on your motherboard.
What is the make and model of your machine?
Go to Control Panel - System and Security - Bitlocker.
Down in the bottom left, click the TPM administration.
Does it see a TPM chip?
If not shut down your machine and boot it up and go into the BIOS.
Once in the BIOS see if you can find anything about TPM and enabling.
If you can not then you don't have the proper chip to encrypt your operating system drive so you will need a USB flash drive instead.

The problem with this option is if you lose that drive or it gets corrupt you will need to go into recovery mode to get into the drive.

So please make sure to store your 48 digit recovery key in a safe place.

Now to get your machine to use the USB flash drive instead of the TPM you will need to edit your policy.
Go to gpedit.msc as an administrator
Go to Computer Configuration - Administrative Templates - Windows Components - Bitlocker - Operating System Drives.

Reboot and try again.

teggertg say:

I finally found the problem.

My region uses Daylight Savings Time. Win7 and XP automatically set their clocks to "automatically adjust time for DST". The Win8.1 installation DID NOT DO THIS. It left that box UNCHECKED.

That means that there was confusion between the Win7 box and the Win8.1 box, since one was expecting a DST time and the other wasn't.

Why was Win7 smart enough to know that if I'm in the Eastern time zone for North America then DST is required, but Win8.1 isn't smart enough to do that? Needless confusion and wasted time!
