scottgem say:

Is there a way to remove someone so that they don't appear in future drop-down lookup fields AND keep the current data so that they continue to show up on what they worked on in the past? 

The answer is yes and no. Yes, it can be done, but not the way you are doing it. You should NOT be interfacing directly with tables. All interfacing with data should be done with Forms. It is also not recommended that you use lookup fields on the table level. Lookups should only be done on the form level.

So the way to do this takes a few steps:

1) Add a field to the table people that indicates whether they are active or not (if one doesn't already exist)
2) Create a form to work with the data. 
3) For the PersonID, create a Combobox that lookups againt the People table and shows all people.
4) In the Got Focus event of the combobox use code like the following:

Me.cboPeople.RowSource = "SELECT PersonID, Lastname & ", " & Firstname AS Fullname FROM tblPeople WHERE Active = True;"

So what this does is when someone uses the form and attempts to add a person to the record, they will only see Active people. However, when someone looks at the record, it will show the person assigned to the project, because the Rowsource is not being filtered.

I do this all the time.

 The Application.PrintCommunication property was introduced as part of VBA7 with the launch of Excel 2010. Consequently,  the instruction will fail with Excel 2003 and VBA6. In this connection, see:

    #If VBA7 Then
