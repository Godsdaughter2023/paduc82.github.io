Split vertically merged cells - Microsoft Community

a. user say:

Split vertically merged cells

Is there a way for VBA to detect and vertically split vertically merged cells in a Word table into the proper number of rows such that the table is left without any vertically merged cells?
andreas killer say:
Well, yes an no.

If you have only vertically merged cells we can detect them, but if there is just one horizontal merged cell you are lost.

Make a new file, create table with 5 columns and 3 rows and execute the macro below. You can see the row/column number of each cell in the MsgBox.

Now try to merge cells vertically and run the macro again. You can see a "X|X" at the position where a cell is missing. Theoretically we can use the cell above the "X|X" and split them into rows.

But now merge just 2 cells horizontally and run the macro again. No chance anymore.

Andreas.


Sub Test()
  Dim i As Long, j As Long
  Dim Data()
  Dim T As Table
  Dim C As Cell
  Dim S As String

  If Not Selection.Information(wdWithInTable) Then
    MsgBox "Selection is not within a table"
    Exit Sub
  End If
  Set T = Selection.Tables(1)
  With T
    'Build array for each cell
    ReDim Data(1 To .Rows.Count, 1 To .Columns.Count)
    'Start with the first cell
    Set C = .Cell(1, 1)
    'Visit all cells
    Do While Not C Is Nothing
      'Write the row and column into out array
      Data(C.RowIndex, C.ColumnIndex) = C.RowIndex & "," & C.ColumnIndex
      'Get the next cell
      Set C = C.Next
    Loop
    'Build a string with the row/columns from our array
    For i = 1 To UBound(Data)
      For j = 1 To UBound(Data, 2)
        S = S & IIf(IsEmpty(Data(i, j)), "X,X", Data(i, j)) & "|"
      Next
      S = S & vbCrLf
    Next
    MsgBox S
  End With
End Sub

dirk goldgar say:
I'm using VBA to create records in a table.  Two of the fields in the table are hyperlinks. In VBA, I've got some variables which contain addresses and links and am trying to combine them into link fields for the table.

That's actually easier than you think, and you don't even need a Hyperlink object to do it.  That's because a hyperlink field is nothing more than a text field that uses the '#' character to delimit the parts of the hyperlink, with the field type telling Access to interpret it as a hyperlink.  All you have to do is create a string value in the right format and insert it in the field programmatically.

For example, here's how you might create a record with a hyperlink field in a DAO recordset:

    Dim rs As DAO.Recordset
   
    Set rs = CurrentDb.OpenRecordset("tblData")
    With rs
        .AddNew
        !HyperlinkField = "test#http://www.cnn.com#"
        .Update
        .Close
    End With

Here's a way to do the same thing by executing an inline SQL statement:

    CurrentDb.Execute "INSERT INTO tblData (HyperlinkField) VALUES('test#http://www.cnn.com#')", dbFailOnError

If you look at the help-file entry for the Application.HyperlinkPart Method, you'll be able to see from their examples how to delimit the different parts of a hyperlink.

suzanne s. barnhill say:

This message isn't exactly bogus; it just isn't very communicative. Here's what will usually work: On the Page Layout tab, in the Page Setup group, click Columns and select More Columns...

In the Columns dialog, chances are you'll see that the box for "Equal column width" isn't checked. When this happens, the column width doesn't adjust automatically to the margin width but stays fixed at a certain amount. Where you can run into trouble (aside from increasing side margins) is when you switch from Landscape to Portrait orientation.

The fix is to click the preset picture for One (make sure that "Apply to" is set to "Whole document"). This checks the box (which you can't do manually if you have only one column). If your document has multiple columns in some sections, you can change the number of columns back after you get the error corrected.
