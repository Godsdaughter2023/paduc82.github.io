keep format from Access to Word mail merge - Microsoft Community

gustavo del valle say:

keep format from Access to Word mail merge

I have a database that contains some calculated fields that display in currency or with two decimal places.  When I do the mail merge the numbers come out of the merge with many decimal places and without the currency format.  How do I keep the format?
hansv mvp say:

Right-click one of the merge fields and select Toggle Field Codes from the context menu. You will see something like

 

{ MERGEFIELD Amount }

or

{ MERGEFIELD Amount \* MERGEFORMAT }

 

where Amount is the name of the merge field. Edit it to look like this:

 

{ MERGEFIELD Amount \# "$,0.00" }

 

You can replace $ with another currency suymbol, and if you use the comma as decimal separator, change "$,0.00" to "$.0,00".

 

With the insertion point between the field brackets { }, press F9 to hide the field code and to update the result.

Win10 - How to set Word 2013 as default rather than Word 2007 for - Microsoft Community

postparadoxical say:

Win10 - How to set Word 2013 as default rather than Word 2007 for doc/docx (two office versions installed)

I have two functioning versions of office running on my Win10, my own fully paid Office 2007 (Ultimate) and one supplied by my current employer/university, Office 2013 (academic), which they encouraged me to use for their support, and which I most willingly will use. My problem is that when I try to open any doc/docx file, the Win10 uses the Word 2007 version as default, even though the default settings on Win10 shows the icon of Word 2013.  Even when I use the 'Open With' possibility of Explorer, the only choice is 'Word' not  a version of word. I have eliminated all desktop links to Word2007, have opened things from within Word2013 repeatedly, etc.   Thus the question:

The Win10 default setting only shows 'Word (Desktop)' and does not allow me to choose which version of Word to use.  How can I set Word 2013 as default to open .doc/docx?

Background.  I don't want to eliminate the Word 2007 since I can use the Word 2013 only as long as I am fully employed by the employer (at least according to them), and my contract with them will end, since I only get 3 activations of Office 2007, and this was my third activation, since it is fully paid for (by me), since it serves my needs as a writer, since I prefer to back my material up on various external drives rather than the 'cloud,' etc.  On the other hand, I am only now getting used to the Navigation panel of 2013, am willing to see if i do want to upgrade to Office 2013 or beyond for collaboration possibilities, etc.  Obviously, the Outlook 2013 is the current email program since only one version of Outlook is possible.

suzanne s. barnhill say:

First, to address your concern about Office 2007. You can reinstall it as often as needed on the same machine, provided the motherboard has not changed.

However, it should be possible to make Word 2013 the default version for now by "reregistering" it (reestablishing its "ownership" of the Word file types in the Windows Registry). If you have a locally installed edition, this is relatively simple; if it is actually part of Office 365, then it is a click-to-run version, and it's a little harder. For the MSI version:

  1. In Windows Explorer, search for Winword.exe. You'll find more than one file, obviously, but the one you want will probably be located at C:\Program Files (x86)\Microsoft Office\Office15 (whatever the path, it will include Office15).
  2. Copy that path from the address bar of Windows Explorer.
  3. Press Windows Key+R to open the Run box in Windows.
  4. Paste the path you copied and add \Winword.exe.
  5. Enclose the path in quotes and add a space followed by /r
    "C:\Program Files (x86)\Microsoft Office\Office15\winword.exe" /r
  6. Press Enter.
  7. You'll get a message along the lines of "Please wait while Windows configures Microsoft Office 2013.

The result is to correct the file associations to make Word 2013 the default application for .docx and other Word file types.
