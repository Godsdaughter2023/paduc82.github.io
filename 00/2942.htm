Table 1 - Layer 1 = GL Group with fields that connect to Layer 2 and 3

The query I have now is based on Layer 1 or GL Group and it gives me the total of direct and indirect expenses.  For example:

Field 1                          Field 2
GL Group Name    Expense Total
GL Group 1              = $250,000
GL Group 2              = $250,000
GL Group 3              = $250,000
Grand Total              = $750,000 (This is the automatic "Totals" field you can add)

Now I want to have a column or field that takes each of the totals of each GL Group ($250,000) and divide it by the grand total of $750,000 to get a percent of the grand total.  For example:

Field 1                        Field 2                  Field 3
GL group Name   Expense Total         Percent of Total
GL Group 1            = $250,000                     33%
GL Group 2            = $250,000                     33%
GL Group 3            = $250,000                     33%
Grand Total            = $750,000                   100%

I am assuming there is some way to add an expression field like this "Percent of Total: [Expense Total]/Sum([Expense Total])" but this doesn't work.

Thank you for your help in advance.

MJ
ken sheridan say: As you are returning only one column of aggregated values, i.e. the data relates to only one budget year, why are you using a crosstab query?   You can do this with a simple aggregating query, e.g.

SELECT [GL Account Groups Name], [GL Account Catergory Name],
SUM([Expense Total]) AS [Total Expenditure]
FROM [GL Accounts]
GROUP BY [GL Account Groups Name], [GL Account Catergory Name];

You can then add in a subquery as I described to give:

SELECT [GL Account Groups Name], GL Account Category Name],
SUM([Expense Total]) AS [Total Category Expenditure],
SUM([Expense Total])/
     (SELECT SUM([Expense Total])
       FROM [GL Accounts]) * 100
AS [Percentage of Total Expenditure]
FROM [GL Accounts]
GROUP BY [GL Account Groups Name], [GL Account Catergory Name];

If, despite the fact that you haven't indicated this, you are in fact returning the values for multiple budget years then you could correlate the subquery with the outer query on budget year by giving the two instances of the table aliases, and concatenate the total category expenditure and percentage of total expenditure into a single column:

SELECT [Budget Year], [GL Account Groups Name], [GL Account Category Name],
FORMAT(SUM([Expense Total]),"Currency") & " (" &
FORMAT(SUM([Expense Total])/
     (SELECT SUM([Expense Total])
       FROM [GL Accounts] AS GLA2
       WHERE GLA2.[Budget Year] = GLA1.[Budget Year]), "Percent") & ")"
AS [Total Category Expenditure]
FROM [GL Accounts] AS GLA1
GROUP BY [Budget Year], [GL Account Groups Name], [GL Account Catergory Name];

This would return the [Total Category Expenditure] in a the format $250,000 (33.00%). You could then create a crosstab query on the result set of this query.  As there is only one value per GL Account Groups Name/GL Account Catergory Name per Budget Year you can TRANSFORM FIRST([Total Category Expenditure]).  The query would be GROUPed BY [GL Account Groups Name], [GL Account Catergory Name] and would PIVOT [Budget Year].

hansv mvp say:

No, VBA doesn't support that. You might use an array of collections:

    Dim clnTmpCollection As Collection
    Dim clnCollections(1 To 5) As Collection
    Set clnCollections(1) = New Collection
    clnCollections(1).Add "Alpha"
    clnCollections(1).Add "Beta"
    clnCollections(1).Add "Gamma"
    Set clnCollections(2) = New Collection
    clnCollections(2).Add "Amsterdam"
    clnCollections(2).Add "Brussels"
    clnCollections(2).Add "Copenhagen"
    ' ...
    ' ...
    Set clnTmpCollection = clnCollections(2)
    MsgBox clnTmpCollection(3)

Note the keyword Set. This is required when assigning a collection to a variable.
