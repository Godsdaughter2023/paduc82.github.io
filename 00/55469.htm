slance310 say:

Hi,

There is something unusual about how your SSD\HDD is set up...

The DATA partition is only 100MB... That is pretty much useless for any amount of DATA...

Normally a 100MB partition in that location would be a boot partition on a UEFI Bios machine... It would be FAT32 and not NTFS...

It would not be marked as "Active" either...

From what I could see on other images of the partitions on a setup similar to your's, your 100MB DATA partition is located where the System Reserved Partition should be, and it's properties are the same... The problem is that the System Reserved Partition needs to be in the area of 350-500MB...

The 100MB size indicates that your 100MB DATA was at sometime a "System Reserved" partition for Windows 7... That is the size Windows 7 used..

The fact that is listed as DATA may be a problem also...

You do not actually need the System Reserved partition... Your machine is probably not even using one , since the partition has the wrong file name...

What happens on my machine with a similar setup is that the partition you are having issues with is created at the end of the SSD\HDD, not the beginning... I normally delete it after the upgrade is complete...

After it is deleted you will need to make the remaining partition "Active"..

You need a bootable 3rd party tool to do this... You do not want to "Install" anything to deal with this...

After deleting the 100MB partition you could either leave the space unallocated, or expand your remaining partition into the unallocated space..

Partition Wizard 9 is a good tool for this, but it is getting hard to find a bootable ISO... Partition Wizard 10 is not bootable without installing it first to create the bootable media, but you may be able to find a bootable ISO that someone else has created...

Most of the links for PW9 actually try to download the version 10 that will not work...

There is other free stuff available, but a lot of it requires installation, which you do not want...

To change the administrator account, first create a new account.

1. Press the Windows key or click on Windows icon.
2. Click on Settings.

• Click on Settings.
• Click on Account.
• Now click on Family and other users.

jack mvp say:

Win7 when configured on peer-to-peer Network Win 7 has three types of sharing configurations.

HomeGroup Network = Works only between Win 7 computers. This type of configuration makes it very easy to Entry Level Users to start Network sharing.

 Work Network = Basically similar to the previous methods of sharing that let you control what, how, and to whom folders would be shared with.

 Public Sharing = Public Network (like Internet cafe) to reduce security risks.

 For best results log to each computer's System Screen and set all the computers to be on a Network with the same name while each computer has its own unique name.

http://www.ezlan.net/Win7/net_name.jpg


Make sure that the Software Firewall on each computer allows free local traffic. If you use 3rd party Firewall On, Vista/XP Native Firewall should be Off, and the active Firewall has to adjusted to your Network IP numbers on what is some time called the Trusted Zone (consult your 3rd Party Firewall instructions

General example,  http://www.ezlan.net/faq#trusted

Please Note that some 3rd party Software Firewalls keep blocking aspects of Local Traffic even it they are turned Off (disabled). If possible configure the Firewall correctly, or totally uninstall it to allow clean flow of local network traffic.

Since your computers are running Win 7 and Win XP you have to use the Work network option.

Win7 - http://windows.microsoft.com/en-us/windows7/Networking-home-computers-running-different-versions-of-Windows

 Win7  Work Network specific folder sharing - http://www.onecomputerguy.com/windows7/windows7_sharing.htm

 Printer Sharing XP - http://www.microsoft.com/windowsxp/using/networking/expert/honeycutt_july2.mspx

 Windows Native Firewall setting for Sharing XP - http://support.microsoft.com/kb/875357

 Windows XP patch for Sharing with Vista (Not need for XP-SP3) - http://support.microsoft.com/kb/922120

 When finished with the setting of the system it is advisable to Reboot all the hardware including the Router and all computers involved.

*Note. Some of the processes described above are done Not for Windows sake but to compensate for different Routers and the way their firmware works and stores the info about the computers that are Networked.
