john p spencer say:

Well if the values in the Drop and Input are not equal then this returns NO records.  If the values of Drop and Input are equal this returns EVERY record in the table. 

Assuming Drop has MyFieldName as a value and Input has Open as its value, your where clause once interpreted would be something like

    WHERE "MyFieldName" = "Open"

Normally you would see the where clause looking more like

WHERE dbo_Finance.[SomeField] = Forms![TestInput]![drop] 
     Or dbo_Finance.[SomeOtherField] = Forms![TestInput]![Input]

I guess what you are trying to do is test the value of a field that is named in DROP against the value in the textbox Input.  To do something like this you would have to build the query string in VBA and then assign the string to a queryDef or as the record source for a report or form.  The beginnings of the code would be something like the following.  You would need to know the datatype of the field so you could add the proper delimiters (quotes for string, # for dates, nothing for numbers).

Dim strSQL as String
Dim strWhere as String

StrSQL= "SELECT * FROM dbo_Finance"
strWhere = "[" & Me.Drop & "] = """ & Me.Input & """" 

strSQL = StrSQL & " WHERE " & strWhere

You might take a look at the BuildCriteria function to help you build the criteria string.
