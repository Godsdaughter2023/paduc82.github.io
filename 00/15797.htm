Security Essentials couldn't check for virus and spyware definition updates

I've been running Security Essentials for months now if not over a year.  I often review the Security Essentials to verify everything is working correctly.  Yesterday I noticed that the Definition last update was about 5 days old.  I clicked the Update button and received the error message below.  Internet connectivity is working fine.  IE is running fine.  I'm running XP Home, SP3, Internet Explorer 8.0.6001.18702

Error Message details:

Virus and spyware definitions update failed

Security Essentials couldn't check for virus and spyware definition updates.  Check your Internet or network connection and try again.

Error Code: 0x80070005

Security Essentials couldn't access the updates on the server because the authentication failed or the authentication method isn't supported.

 

I suspected malware or a virus so I rebooted into safe mode.  Installed Malwarebytes and ran a full scan.  There were only 2 registry data items detected:

HKLM\SOFTWARE\Microsoft\Security Center|AntiVirusDisableNotify (PUM.Disabled.SecurityCenter) -> Bad: (1) Good: (0) -> Quarantined and repaired successfully.
HKLM\SOFTWARE\Microsoft\Security Center|FirewallDisableNotify (PUM.Disabled.SecurityCenter) -> Bad: (1) Good: (0) -> Quarantined and repaired successfully.

I unstalled Malwarebytes and ran ccleaner to get rid of any residual Malwarebyte files/entries.  In addition to Malwarebytes I installed Stopzilla.  Ran a full scan.  I've found that Stopzilla finds some things that Malwarebytes does not and vice versa.  Stopzilla found nothing.  I unstalled Stopzilla, ran ccleaner and rebooted the machine in regular mode.  I unstalled Security Essentials and reinstalled it.  It reinstalled successfully and it even updated the definitions file.  I was hopeful the issue was resolved.  I opened Security Essentials and tried to Update...same error as above 

 

I've researched and reviewed a lot of cases.  I've tried a few more things but none of them worked.  Any ideas on what would be causing this issue?  Another interesting note is when I try to do a Windows Update I get an error that says to "Please change your Internet Explorer security settings....Under Userdata persistence, select enable (0x800A0046)".  I confirmed it's selected.  I even reset my internet options back to default.  Still no luck.  I suspect this is some how related to the Security Essentials issue?  Any help would be appreciated. 

 

 

robert.b say:

I wanted to provide an update for those who were generous enough to help me and to hopefully help others.  I did finally resolve the problem.  There were a couple issues in play.  The interesting part of this is that the root cause was with Windows Update (WU), not Security Essentials (SE).  I found out that SE is dependent on WU and if you do not have certain WUs (don't confuse this with updated Definition files) SE will not work correctly.

 

I tried a lot of things, I probably spent a combined total of 8-10 hours on this problem.  Here's what I believe resolved the problem.  Keep in mind I'm running Windows XP Home SP3 with IE 8. 

1.  I had an update that was pending but would not install when I rebooted my machine.  If you have a config.msi folder off your root directory then there's a pending update.  When you reboot the update should be installed and the folder deleted.  It did not for me.  I had to delete the folder and there was a registry entry PendingFileRenameOperations that had to be deleted.  There's a KB out there for this I just can't remember the #.  Search for PendingFileRenameOperations and I'm confident you'll find it.

 

2.  I had to update some of the windows system files.  It appeared that some were damaged due to WU or more likely malware or a virus.  If you need/want to check your sys files then at the command prompt if you run SFC /scannow (windows file check utility) it will check all of your windows system files and replace the ones that are damaged.  Have your Windows CD ready because it copies the files from the CD onto your workstation.  It doesn't tell you which files it replaces (I couldn't find a log file but maybe there's parm you can specify to create a fiel) but it did a good job for me 

 

3.  There was an issue with the files in the Windows\SoftwareDistribution folder.  We never found out the exact problem.  I had to rename the folder to resolve the issue. 

 

After these three things were completed I was able to get past the Userdata persistence  (0x800A0046) issue in windows update.  I had a lot of updates to process.  I had a few fail.  Note: if you have specific windows updates fail you can goto catalog.update.microsoft.com. and search for that specific update, download it and apply it manually.  Once I finished the WUs I then was able to successfully update SE.  

 

Thank you Stephen and Rick for your responses and help! 

Also, if you need help the Microsoft Answer Desk is a great resource.  They helped resolve this issue and currently support is free.
