daniel g. grau say:
You need to go to a header or footer (select View > Header and Footer in the main menu bar). Once you do that, you'll see a new purple tab named Header and footer appear in the Ribbon. Click there and you'll see a "Link to previous" option. Take the checkmark off. Once you do that, the header (or the footer, depending on where you had your cursor) will become independent.

The only use for continuous page breaks is to change the number of columns in the same page. Unless you are doing that, don't use them, as they won't have any effect and you'll be changing headers and footers in the wrong places. Instead, to change the looks of a header or footer, or to set a different page number, use a "Section break (Next Page)."

The header or footer that you see (and the effect of the "Link to previous" option) apply to the section where your cursor is.

stevennichols say: You can turn off Same as previous by unselecting Link to Previous in the Navigation group of the Header & Footer Design contextual tab. 

The trouble with the continuous section breaks is that it doesn't place the break at the very end of the page.  It's much easier to use "Next Page" Section breaks.  Use the View formatting button to see where your breaks are if you really need continuous.

On my blog there is a tutorial on Changing page numbering formatting in Word.  

I hope that helps.

hansv mvp say:

Press Alt+F11 to activate the Visual Basic Editor.

Select Insert > Module.

Copy the following code into the module:

 

Function ConcatenatePositive(CriteriaRange As Range, ConcatenateRange As Range, _
        Optional Separator As String = ",") As Variant
    Dim i As Long
    Dim strResult As String
    On Error GoTo ErrHandler
    If CriteriaRange.Count <> ConcatenateRange.Count Then
        ConcatenatePositive = CVErr(xlErrRef)
        Exit Function
    End If
    For i = 1 To CriteriaRange.Count
        If CriteriaRange.Cells(i).Value > 0 Then
            strResult = strResult & Separator & ConcatenateRange.Cells(i).Value
        End If
    Next i
    If strResult <> "" Then
        strResult = Mid(strResult, Len(Separator) + 1)
    End If
    ConcatenatePositive = strResult
    Exit Function
ErrHandler:
    ConcatenatePositive = CVErr(xlErrValue)
End Function

 

Switch back to Excel.

Select the cell where you want the formula.

Enter the formula

 

=ConcatenatePositive(H2:H10,G2:G10,", ")

 

where H2:H10 is the range with the amounts and G2:G10 the range with the item names.

 

Remarks:

- Save the workbook in a macro-enabled format: .xlsm, .xlsb or .xls but not .xls.

- Enable macros when you open the workbook.

Once you share a copy of a document, the content can be copied. It does not matter if it is a paper copy or an electronic copy! Just like a paper copy can be photo copied, the content of an electronic file can be copied too.

There are a number of things you can do to prevent the content in your FILE from being changed, but in the end you can't keep it from being copied.

Even if you manage to "lock" the file somehow (sharing a simple image based PDF format is probably the "best) I can always use a screen capture tool to get an image of the page then use any one of a number of OCR tools (including the one built into Office OneNote!) to extract the text in an editable form.

I guess the "best" you could do is pick a really funky font. A font that obscures the outline of the characters in such a way that it is still readable by people, but is hard to read for OCR tools. You see that sort of thing in the "captcha" images where you have to repeat the letters or numbers in an image. But the image is hard to read, it is low contrast or angled a funny way. 

In addition to a funky font, making the the background and font color similar, like blue on blue to make it harder for the OCR tool to differentiate will help you "protect" your content, to a degree.  Or making the background really "active" so that the lines and shapes of the characters fade into the background.

Don wrote:
Start with the time: hhhh:mm:ss which is in E2
Divide it up into it's parts:
Hours =LEFT(E2,4) and put that into I
Mins  =MID(E2,6,2) and put that into J
Secs  =RIGHT(E2,2) and put that into K

Then calculate the amount of the parts:
Hours remains the same
Mins =mm/60*100 and put that into L
Secs =ss/60*100 and put that into M

It is apparent now why you think 52:13:37 converts to 52.2262 and 33:18:35 converts to 33.3058.

You have a fundamental error in how you think seconds converts to decimal hours.  In particular, you think it should be 37/60/100 and 35/60/100.

It is difficult to explain why that is mathematically incorrect.

I hope it is sufficient to say that you convert seconds to decimal hours by dividing by 3600, not 6000, because there are 60 seconds in a minute and 60 minutes in an hour.

So ss/60 is the number of decimal minutes; and (ss/60)/60 is the number of decimal hours, just as you are doing with mm/60.  That simplifies to ss/3600.

But as I demonstrated, you do not need to do this at all.  It is sufficient to use =--E2*24.

The double negative solves that problem, as I mentioned previously.
