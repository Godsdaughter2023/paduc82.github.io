jacqui kaminer say:

Hi there

We were having the same issue and read this post and felt your frustration.

We noticed that we got the error only on profiles that were logged in to the domain.  When we set up a local computer account the IMAP configured fine.  As we needed this for a domain user we investigated further.   A rather long winded way around it but this is what worked for us:

Design flaw causing Microsoft outlook 2013 to not connect to imap accounts whilst joined on domain,

Here is the workaround.

 

Create a local profile with the same username and password, open outlook and set up the imap account as per usual.

When finished, and it is working, close outlook, log out of the profile, and log in on the domain profile.

 

For the next part I will be using "bob" as the local username, and "MYPC" as the local computer name.

 

Right click on the desktop, create new shortcut.

Pop-up window will appear, location of the item.

Type in the following,

Runas /savecred /user:MYPC\bob “C:\Program Files\Microsoft Office\Office15\OUTLOOK.EXE"

 

** you must make sure of the path of the OUTLOOK.EXE file **

 

Click Next

 

Type a name for this shortcut: I would type in My Mail, and remove the original outlook icon from desktop, start menu, start screen etc.

 

Click Finish

 

Right click on shortcut, properties, change icon, and browse to outlook exe

Choose it, and choose outlook icon.

 

Click ok, to close the properties box.

 

Double click on icon, it will prompt for password of local account, you will only need to input it once.

Outlook will now open with the profile that was created in the local account to loop around IMAP domain level issues.

 

PS: If it does not ask for a password, and cmd window just pops up and closes, right click the icon, and edit the location string, remove the /savecred part, this will unfortunately cause the user to enter his password every time he opens outlook, which is less of a headache than not having mail on domain level.

Hope this helps someone and full recognition to Jean Korster for resolving this issue for us and not myself!

I needed to add the following lines to the registry of the server:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Rpc\RpcProxy\ValidPorts_AutoConfig_Exchange

"casarray:6001-6004;casarray.localdomain.com:6001-6004;EXCHHTCA1:6001-6004;EXCHHTCA1.localdomain.com:6001-6004;EXCHHTCA2:6001-6004;EXCHHTCA2.localdomain.com:6001-6004;EXCHMBX1:6001-6004;EXCHMBX1.localdomain.com:6001-6004;EXCHMBX2:6001-6004;EXCHMBX2.localdomain.com:6001-6004"

judith_c. say:

Hi TimApple,

 

Welcome to Microsoft Community. It's my pleasure to explain to you the difference between events and task and how to use it.

 

Task, is formerly known as To-do list. This cannot be shared to other people. It is where you can create a calendar notification of plans or activities. To create a task, follow the steps below:

 

1. In your calendar page, click the arrow "v" down button beside New.

2. Select Task.

3. Enter the description of your task in the compose page.

4. On the right hand corner, select the name of the calendar where you want to add your task.

5. Select the Due date in the calendar and Time.

6. Select a Status of your Task, either In progress, Not started, and Done.

7. If you select the In Progress status Reminders list will appear below Priority box. In Reminders you have the option to select where the task notification will be sent to and occurrence before the created time.

8. Select the Priority of the task, either High, Normal and Low.

9. Click Save.

 

For event, it could be an occasion or meeting where you can set a specific time, date, and place. In event, you can share it by inviting other people. Here's how to create an event.

 

1. In your calendar page, click the arrow "v" down button beside New.

2. Select Event.

3. Enter the description of your event in the compose page.

4. On the right hand corner, select the date of the event and its start time.

5. Choose How long the event will be will.

6. Enter the place of the event in the text box provided under Where.

7. Select the name of the calendar where you want to add your event.

8. Select How often is the event, either Once, Daily, Weekly, Monthly, Yearly.

9. Select the Charm of your event.

10. Under Reminders select where the task notification will be sent to and occurrence before the created time.

11. Select a Status of your event, either Busy, Tentative, Free, Away.

12. Click Save.

 

Note: If you are sharing the calendar where you created an event, you have the option to make the event as Private by ticking on the check box provided so that other people won't be able to view it.

 

Hope this helps! Let us know how things work out for you.

 

Thank you.
