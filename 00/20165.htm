Hi,

 

It isn't ideal that first and last names are in different columns but it's still doable. Say we have first name in Col A and second name in col b and the hours in Col D, you can use this formula for the person whose first and last name is in G1.

 

 

=SUMPRODUCT(($A$2:$A$20&" "&$B$2:$B$20=G1)*($E$2:$E$20))

 

You must format the formula cell as [h]:mm to prevent the sum rolling over to zero if it exceeds 24 hours.

Add the code to the sheet module in the specific sheet.

Option Explicit
 
Private Sub Worksheet_Activate()
    Application.MoveAfterReturn = False

     With Application

             .MoveAfterReturnDirection = xlDown

allen browne say:

Suggestions:

  1. Run the query the form is based on. Does it give all 373 records? If not, the problem is probably the INNER JOIN as Marshall suggested.
  2. Double-check that the query's Filter property is cleared (in design view), and also the Filter property of each of your tables.
  3. In form design view, make sure the form's FilterOnLoad property is set to No.
  4. If you are still stuck after this, compact/repair the database in case it's a faulty index.

    http://allenbrowne.com/ser-47.html#NumberOfRecordsVaries
