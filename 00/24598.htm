    End If
    thisSaturday = savedSat
End Function

Function taskDueThisWeek(cellref As Range) As Boolean
    ' we must convert the string date in the cell to a date serial
    taskDueThisWeek = notSummary(cellref) And (DateValue(cellref.Value) <= thisSaturday())

The formatted value in which a value of currency data type is shown is not the actual value which will be passed into your function.  The currency data type is to a precision of four decimal places to combat cumulative rounding errors, so unless you round the value first your function is seeing the number at the second decimal place as that before rounding, hence the discrepancies in some cases.  You should be able to see the differences in the debug window (aka immediate window):

First assign a value to a variable:

MyAmount = 125.7891

Then call the Format function to show the value as currency:

? Format(MyAmount,"Currency")
£125.79

Yours will show as INR of course, not GBP as on my system.

Then call your function, passing in the variable:

? CurrencyToWord(MyAmount)
Rupees One Hundred Twenty Five and Seventy Eight Paisa Only

The value to its full precision of four decimal places has been used, resulting in 'Eight Paisa' being incorrectly returned.

Now call it with the value of the variable rounded as Daniel described earlier:

? CurrencyToWord(Round(MyAmount,2))
Rupees One Hundred Twenty Five and Seventy Nine Paisa Only

Because the value rounded to two decimal places has now been used it correctly returns 'Nine Paisa'.  Whenever calling the function you should do the same.

ken sheridan say: PS:  Alternatively you could amend the function so that it rounds the value to two decimal palces before processing it.  

 ' Convert MyNumber to a string, rounding to two decimal places

 MyNumber = str(Round(MyNumber, 2))
 '==========================Modified================================

You can then pass in the unrounded value as you are currently doing:

? CurrencyToWord(MyAmount)
Rupees One Hundred Twenty Five and Seventy Nine Paisa Only
