john w. vinson say:

Access cannot link across the Internet. If your backend is on your computer, and you email a copy of the frontend to a user in a different location, the frontend will try to link to C:\SomePath\MyBackend.accdb - ON THEIR COMPUTER - and of course that file won't be there, because it's not on their computer, it's on your computer!

Where are these users? Are they on the same, fast, stable, wired (not wireless! it's unreliable for Access) Local Area Network? Or somewhere else? Do the users have full read, write, update, delete permission to the folder containing the backend (that's required as well)?  What IS the path to the backend?

Note that the Forms, Forms, Queries, Reports, Macros, and links to the Tables ARE in the frontend. The linked tables are just pointers to the actual tables in the backend; and the users need to have a connection to the backend, using the same path. This is why it's preferable to use the network path (e.g. \\Servername\Path\mybackend.accdb rather than a mapped drive letter such as D:\MyDatabases\mybackend.accdb), because the folder that you call D: might be the K: drive on your user's computer.

scottgem say:

As John said, you left out some key info. Where did you put your back end? in Linked Table Manager what does it show as the location of the back end (we need to see the full path). Also where is the Front End If it is on your C drive, other people will not be able to open a link to your C drive.

The front ends should be stored on each individual PC and the back end should be stored on a shared network drive. Also, unless you have a networked file server that everyone is mapped to with the same drive letter, you need to use UNC coding for the back end. This is formatted like:

\\servername\sharename\foldername\filename

To do this you need to use the Network icon in the File Open dialog to find the server/computer name and then select the file.

You need to develop using the lowest version your users will use. If you have some 2007 users, then you need to do all your development in 2007.
