B,
re:  paste special

There are two copy methods involved... the copy method and the pastespecial method.
The Copy method uses one line of code (two sections separated by a space)...

  InputWorksheet.Range("B6").Copy GasSheet.Range("G11")
  -or-

  InputWorksheet.Range("B6").Copy Destination:=GasSheet.Range("G11")

'---

The PasteSpecial method requires two lines of code...
  InputWorksheet.Range("B6").Copy

  GasSheet.Range("G11").PasteSpecial  xlPasteValues

  -or-
  InputWorksheet.Range("B6").Copy  

  GasSheet.Range("G11").PasteSpecial  Paste:=xlPasteValues 
'---

The code above shows arguments by position and by name.
Using by name allows argument placement in any order.

      ken sheridan say: Your table is incorrectly designed.  Instead of having multiple Lot# columns, the table should be decomposed so that each lot per row in the current table is represented by a separate row in a related table with two columns, one being a foreign key referencing the primary key of the current table, the other being the lot number.  The primary key of this table is a composite of the two columns, which automatically precludes any duplication of lot number values per foreign key value.

      Also do not use special characters like the # character in object names.  Instead name the column LotNumber or similar
