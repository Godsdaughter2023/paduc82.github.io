davidf2 say:

In my experience the only way a link to a bookmark will work properly is if the bookmark is located near the top of the page. I don't know how to explain this well, but there must be enough page length for a 'full length view' below the bookmark in your PDF reader. It is as if the reader is set for single page view and not continuous view and once you scroll down so that you can read the bottom of the page you cannot scroll any further...you cannot move the last sentence to the top of the reader in single page view mode.

For example if you have 14 inch length Publisher pages and your PDF reader shows the top 8 inches of that document page when it is converted to a PDF and requires you to scroll down to see the bottom 6 inches, then the only way a link to a bookmark will take you to the top of the visible page is if the bookmark is inserted somewhere in the top 6 inches of the document page...there must be at least 8 inches of the page below the bookmark.

The bottom line is the only consistent link to bookmark is one to the top of a page...and that does not require a bookmark...just a link to the page. Sorry.

  2. Select the first cell on the worksheet (Cell A1 which should be the default selection anyway).
  3. Select Data ribbon.
  4. In the "Get External Data" block select "From Text"  icon.
  5. Navigate to the required .csv file and select it and click "Import" button (or just double click the file).
  6. In the first screen of the Text Import Wizard select "Delimited" (Probably is the default) and then click Next.
  7. On the second screen check the box against "Comma" and ensure all other boxes are unchecked and then click Next.
  8. On the third screen in the Data preview at the bottom click in the column that you require to remain as text and it should highlight.
  9. Then select the Text option button towards the top of the same screen.
  10. Click Finish.
  11. In the last screen it should have defaulted to Existing worksheet and =$A$1 for where to start inserting the import. You should be able to leave as is and click OK.

    A little extra information in case you are importing a file with dates some time in the future. When you select and highlight a column and tell the wizard it is a date column you will need to also select the date format. The format to select is the existing date format in the text file; not the date format you want to finish up with. Excel needs to know what date format is in the text file and it will convert that to your regional date format set for your computer.

    Date Mergefield - Microsoft Community

    cason2 say:

    Date Mergefield

    Hi,

    Is there a way for me to create a mergefield in my document that fills in the year only?? I'm aware of doing { DATE }, and then right clicking that field and selecting "edit field," and that shows me a list of options to choose from for the date formatting. But there isn't an option for the field to insert just the year only. All the options include the day, month, and year. Any help with this would be wonderful.

    Thanks,

    Cason

    macropod say:

    Select the DATE field in the mailmerge main document and press Shift-F9 to expose the field code, which might look something like:
    {DATE}

    or:
    {DATE \@ "dd MM yyyy"}

    and change the switch to:

    {DATE \@ "yyyy"}

    When you're done, press F9 to update the display.

    Note: If the merge is to produce new documents, you may want to prevent the DATE updating post-merge (e.g. if you open the document next year. To do that:

    1. Select the DATE field and press Ctrl-F9 to enclose it in another pair of field braces, thus:

    {{DATE \@ "yyyy"}}

    then type QUOTE between the first pair of field braces, thus:

    {QUOTE{DATE \@ "yyyy"}}

    This will cause the mailmerge to output the date as text instead of in a field.
