Update:  Got with my AD team and here's what happened to cause the orphan users in the SharePoint user profiles.  These are the same accounts ( *.onmicrosoft.com) that I need to clean out of SPO.

When the tenant was moved, we uninstalled AAD Connect and reinstalled. After the reinstall, we found that our UPN filter was not working as expected. We uninstalled and reinstalled once more and the filter started working but prior to doing that, we wanted to clean up Azure AD from all those @fisglobal.onmicrosoft.com accounts.

 

We ran the following commands to get a list of accounts that matched that UPN…

 

Get-MsolUser -ALL | Where-Object {$_.UserPrincipalName -like "*.onmicrosoft.com"} | Export-CSV onmicrosoft.csv

 

We then ran the following command to delete the accounts or rather move them to the recycle bin.

 

Import-CSV onmicrosoft.csv | Remove-MsolUser –Force

 

Once that was complete, we ran the following to delete them from the recycle bin.

 

Get-MsolUser -ReturnDeletedUsers -All | Remove -MsolUser -RemoveFromRecycleBin –Force

 

That’s all the steps we took.

andreas killer say:
We can use the WindowActivate event of the application, but unfortunately that event did not fire if we switch to the other workbook using the Excel UI (or Windows Taskbar) and the Userform is already shown.

The only workaround it to hide the form (e.g. if you click the form itself or on a button) and switch to the other workbook with code. After that, if the user switch back to your workbook, the event fires and you can show the form and continue the work.

The only other option (you can find this hints in the web) is to start a separate instance for each file. But that works only if you make profound changes in the PC ... I strongly advise against it, this has consequences that are in the end worse than the workaround.

Andreas. 

Option Explicit
Dim WithEvents App As Application
Private Sub App_WindowActivate(ByVal Wb As Workbook, ByVal Wn As Window)
  If Wb.Name = ThisWorkbook.Name Then
    ThisWorkbook.Activate
    Me.Show
  Else
    Me.Hide
  End If
End Sub
Private Sub UserForm_Click()
  Dim Wb As Workbook
  Me.Hide
  For Each Wb In Workbooks
    If Wb.Name = ThisWorkbook.Name Then GoTo Skip
    If Not Wb.Windows(1).Visible Then GoTo Skip
    Application.EnableEvents = False
    Wb.Activate
    Application.EnableEvents = True
    Exit For
Skip:
  Next
End Sub
Private Sub UserForm_Initialize()
  Set App = Application
End Sub

why not just rightclick the group, choose Format and add alt text.  The result is the same, isn't it?

That leads to an idea for a workaround.

Draw a small rectangle, any fill is fine but make the fill 100% transparent, and give it no line.

You now have alt text that'll appear in the PDF.

What happens to it from there is up to Acrobat.

john sr wilson say:

You should be able to automate the process with code. It would be easiest if all were in the same folder (make sure you use copies!) Obviously change the paths to suit your situation.

Sub EveryPresentationInFolder()
' most code taken from www.pptfaq.com
   Dim sFolder As String
   Dim sFileSpec As String
   Dim sFileName As String
   Dim oPres As Presentation
   ' Edit this:
   sFolder = "C:\Users\Optiplex\Desktop\ppt\"      ' must end with a \ character
   sFileSpec = "*.PPTX"
   ' Get the first filename that matches the spec:
   sFileName = Dir$(sFolder & sFileSpec)

   While sFileName <> ""
      Set oPres = Presentations.Open(sFolder & sFileName, msoFalse)
      Call oPres.ApplyTemplate(Environ("APPDATA") & "\Microsoft\Templates\testing.potx")
      oPres.Save
      oPres.Close

      Set oPres = Nothing
      ' Once done, see if there's another presentation that meets our spec
      ' then around the loop again
      sFileName = Dir()
   Wend
End Sub
