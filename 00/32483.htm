afzal taher say:

Hi,

By default, Remote Desktop uses port 3389 as the listening port for any incoming Remote Desktop connections. You can change the port to prevent any attacks.

Step to change the listening port:
a. Click on start and then type regedit in the start search box and press enter.
b. Expand the registry folders to:
HKEY_LOCAL_MACHINE > System > CurrentControlSet > Control > TerminalServer > WinStations > RDP-Tcp
c. Then locate the following registry sub key:
PortNumber
d. On the Edit menu, click Modify, and then on the Edit DWORD Value click Decimal which will show us the exactport number.
e. Type the new port number, and then click 'OK' button.
f. Quit Registry Editor and restart your computer.

Refer this link:
http://windows.microsoft.com/en-US/windows-vista/Remote-Desktop-Connection-frequently-asked-questions

DISCLAIMER: This method contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the
registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it.
Then, you can restore the registry if a problem occurs. For more information about how to back up and restore the registry, click the following link:
http://windows.microsoft.com/en-US/windows7/Back-up-the-registry

richard xu msft say:

Hi Divya,

 

As we know, both TCP and UDP are protocols used for sending bits of data — known as packets — over the Internet. They both build on top of the Internet protocol. In other words, whether you are sending a packet via TCP or UDP, that packet is sent to an IP address. These packets are treated similarly, as they are forwarded from your computer to intermediary routers and on to the destination.

 

TCP stands for Transmission Control Protocol. It is the most commonly used protocol on the Internet. UDP stands for User Datagram Protocol — a datagram is the same thing as a packet of information. The UDP protocol works similarly to TCP, but it throws all the error-checking stuff out. All the back-and-forth communication and deliverability guarantees slow things down.

 

For Skype for Business, TCP is more precise than UDP, Skype for Business will use TCP to transfer the IM, emoji, pictures and so on, since this data packet need high data integrity, if there is any data loss, the IM, emoji, pictures we sent will be unidentified, the recipients can’t recognize them. For UDP, this protocol always keep transfer the data packet, and it will be applied for audio and video, it can hold more data packet and it will also loss some packet when it transfer the data packet, but for audio and video, the imperceptible data loss will not affect the data integrity, the recipients still can understand and recognize it.

 

To summarize, in Skype for Business, TCP is for IM, emoji, pictures which need high data integrity, and the UDP is for audio and video which need high data packet but don’t need high

As you are unable to install the printer drivers on Windows 10, try few of the below steps and check.

Step 1: Run the Printer troubleshooter and check.

Printer troubleshooter find and fix problems with printing.

1.    Press Windows key from the keyboard, type Troubleshooting and select it.

2.    Click on View all option on the upper left corner of Troubleshooting window.

3.    Select the Printer options from the list.

4.    Click Next to run the Printer Troubleshooter.

Step 2: I suggest you to check the status of Print Spooler service and check.

Description: Print Spooler service, spools print service and handles interactions with the printer.

If you turn off this service you won’t be able to print or see your printer. 

Steps:

1.    Press Windows + R to open Run command, type Services.msc and press Enter.

2.    Search for Printer Spooler service, right click on it and Start the service.

3.    Also, set the Startup type to Automatic.

4.    Now check if you are able to install the driver.

Hope it helps. Reply to the post with an updated status of this issue, we will be happy to help you further.
