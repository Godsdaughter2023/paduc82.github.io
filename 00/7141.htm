Let’s try these methods below to check if that solves the problem.

 

Method 1:

 

Sometimes, this error appears when the CREDHIST file (The Credential history) gets corrupted. The CREDHIST file contains all of your previous password-linked master key hashes used by Microsoft's DPAPI - one key per previous password. When applications (such as Internet Explorer) use DPAPI calls to protect secrets on the device, they use these keys as part of the mechanism for protecting/unprotecting data.

 

When a user's password changes, or 90 days have passed (whichever is sooner) a new set of keys are generated. The old ones are kept in the CREDHIST file so that applications can still decrypt files that were encrypted using the older keys. When an application asks DPAPI to decrypt something, it will first try with the current master key (this is contained in the 'Preferred' file, which you'll find in the \..\protect\<User GUID> folder) - if that fails it will try each of the CREDHIST keys in turn until the data are successfully decrypted.

 

Let’s follow the steps in the article below to delete the CREDHIST file and check if that solves the problem.

 

Show hidden files and folders and Delete CREDHIST file

  1. Follow the steps in the article below to unhide hidden files
  2. Show hidden files
    http://windows.microsoft.com/en-US/windows7/Show-hidden-files
  • Close all Internet Explorer windows.
  • Once done, go to the following location and delete CREDHIST file C:\Users\<username>\AppData\Roaming\Microsoft\Protect
  • Restart Internet Explorer to check if the issue persists.

    If deleting CREDHIST file does not solves the problem, we can try changing the default search provider, or deleting the corrupted search provider and creating a new one.

    2. If I create a group under free network, I am the creator with admin rights. And seems I can also share / appoint this admin right to other group members. In this case, I am a bit confused about what do you mean for "admin rights" which cause difference between basic and enterprise network. 

    Looking forward to your reply. Thanks! 

    tonytony_123 say: Can anyone help? Thanks! 

    vincentchoy say:

    In the past, Yammer Basic and Yammer Enterprise differ by Administrator Feature.

    Over the years with deeper integration with office365, Yammer Enterprise has gone much further, whereas Yammer Basic has remained more or less what it was.

    Administrator -

    Yammer Enterprise has 2 levels of Admin. The Network Admin and the Verified Admin. None is found in Basic. The Verified Admin has the ability to control users, moderate posts, even in private groups. He also has the ability to make user changes in bulk, and also to export data from Yammer. 

    In Yammer Basic, the creator of a group becomes the group admin. However, there is no overall admin feature. So if a user in Yammer Basic creates a private group, and uses that group to plot the overthrow of the Managing Director - unless you are a member of the group, you have no access to it.

    Integration with office365

    With deeper integration with Office365, Some features of Yammer Enterprise can now be controlled via the Office365 admin dashboard, basically user licenses, groups integration, External user access...etc.

    Simply put, if you are using Yammer for some simple sharing within a smaller group, Yammer Basic might suffice. If the intention is to deploy it in a larger scale, then you would require the additional features that Enterprise provides.

    bandit101 say: I had a user call me with this exact problem yesterday afternoon, and I just now figure out how to turn the damn things off.

    symptom:   Paragraph symbols and other formatting symbols are visible when composing a message.   I can turn the symbols off by using the shortcut ctrl + shift + 8 , by clicking  on the Format Text tab of the ribbon, and then clicking the paragraph button.   However, as soon as I click  New Email the formatting symbols are back.

    Fix:       First,  create a new message by clicking  New  Email (Ctrl + N) on the far left of the ribbon. By default, your cursor will be in the  "TO:" field waiting for you to type an e-mail address.  You must move the cursor to the message body.  Next turn off the paragraph / formatting symbols by either going  to the "Format Text" tab on the ribbon and then clicking the paragraph button to turn them off, or by using the shortcut  Ctrl + Shift + 8.     Finally,   hit Save (floppy disk) icon at the top left of the window (Ctrl + S) and close the message window without actually typing anything in the message body.   The stubborn things should be gone when you compose a new message  :)
