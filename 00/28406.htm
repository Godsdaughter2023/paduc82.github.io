If you want consistency across various installations, then a folder under AppData\Roaming\Microsoft, parallel to the default Templates folder, is certainly a good choice. Presidents Day error in Outlook - Microsoft Community

josh and emilyunderhill say:

Presidents Day error in Outlook

In Outlook, if you add US holidays to the calendar, the 3rd Monday in February is incorrectly added as "Presidents' Day". 

 

There is no federal holiday called Presidents' Day.  The 3rd Monday in February is Washington's Birthday (observed).  Despite public misperception, it was never combined with Lincoln's Birthday (which was never a federal holiday) and was never renamed.  It was moved by the Uniform Monday Holiday Act from 22 Feb to the 3rd Monday in Feb, but that was all.  See federal law 5 US Code 6103 or do some simple research. 

 

Also, it would be nice if the calendar differentiated between federal holidays (days off) v.  national/religious/popular observances and events.  For example, Mothers' Day is not a federal holiday, it is a national observance.  See 36 US Code 101-144

phillip m jones say:

This might help: 

"Word for Windows supports controls [fields] that are not supported in the Mac version. There is nothing you can do to change that. In order for forms to work in both Mac & PC the only type of controls used must be what are referred to as Legacy Form Fields, so the document would have to be redesigned with cross-platform compatibility in mind. Apparently the document uses either Content Controls or ActiveX Controls.

Also, the Windows versions of Word support document protection features that the Mac versions lack. If those features have been used Mac Word will simply see the document as "protected" & not allow any changes whatsoever.

Additionally, not all macros which run on a Windows PC will run on a Mac & vice-versa. It depends on the commands written into the macro & what it is intended to do."

Courtesy of Bob Jones (Cyber Taz)"
