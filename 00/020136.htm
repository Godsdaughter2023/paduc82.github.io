large enhanced metafile - Microsoft Community

ethan t say:

large enhanced metafile

Split from original:

http://answers.microsoft.com/en-us/office/forum/office_2010-excel/large-enhanced-metafile/c8adf6b6-8f55-e011-8dfc-68b599b31bf5

I often create diagrams in PowerPoint, then paste them into Word using "Paste as Enhanced Metafile." This worked great until 2010, when PPT began bloating images containing gradients (and probably other effects). As I have learned, this bloating behavior happens because Office 2010 seems to create EMF data on the clipboard using tiny, tiny shapes to represent things like gradients. This means that a single gradient may create hundreds of thin rectangles, each of a different color. You can see this by opening an bloated EMF file saved within a .docx file in a vector graphics editor like Adobe Illustrator. This complex drawing method is not only huge in size, but it takes a lot of CPU power to render, which makes apps studder and slow down when they try to draw the EMF data on screen, or print it.

Office 2007, on the other hand, produced gradients by creating a bitmap texture that contained each of the colors in the gradient and setting that as the content of any gradient-filled shape. This was much more efficient both from a size standpoint and from a processing speed standpoint.

Basically, the bloat occurs when you hit "copy" in an Office 2010 app. Several versions of the copied data are created for the clipboard, including the bloated EMF data. When you "Paste as Enhanced Metafile" in the target app, whichever app that is, it's simply using the data created by the app that did the copy.

Fortunately, you can copy from 2007 and paste into 2010. When I finish my diagrams, my workaround for this problem is to open my completed diagram in PowerPoint 2007 (or wherever the graphical data is being produced) and copy the image from there. This creates old-style (non-bloated) EMF data on the clipboard, which you can either "Paste as Enhanced Metafile" directly into your Office 2010 document (if you're on the same machine or sharing a clipboard with a remote machine), or "Paste as Enhanced Metafile" into the same app and save, then open on another machine in Office 2010. I use an old copy of Office 2007 running on a virtual machine for this purpose. It's a big headache, but it does the trick.

An alternate workaround: simplify your graphics. Remove complex things like gradients (and possibly shadows?). This will vastly reduce the complexity that Office 2010 creates in the EMF data. Being very picky about my visual design, this isn't always an option to me.

Microsoft, please address this issue! What was wrong with the way Office 2007 created EMF versions of graphics? Can we please have that back? Thanks!

doug robbins - mvp office apps & services (word) say:

With the selection in the document where you want the date to appear, go to the Insert tab of the ribbon and then expand the Quick Parts pulldown and then click on field and select the CreateDate item.  Then select the required format for the date and then click on OK.

graham mayor say:
A CreateDate field is a field that displays the date the document was created, as opposed to a Date field that displays the system date of the PC.

Doug has explained how to insert such a field, but for your existing document issue press ALT+F9 - you will see something like { DATE } - perhaps with a formatting switch e.g. { DATE \@ "dd MMM yyyy" }.

Change DATE to CREATEDATE and press F9 to update the field, the ALT+F9 to display the document creation date. You can change the format of the date by changing the formatting switch - in the above example "dd MMM yyyy" to taste - see http://www.gmayor.com/formatting_word_fields.htm  for a description or the various switches.
