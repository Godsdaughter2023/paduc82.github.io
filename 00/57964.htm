Your question on Microsoft Community has received a reply from impinball

 

Are you wanting to empty it periodically (a.k.a. on a schedule)? If so, would setting a task at Administrators-level work (you get prompted once in this case)?

 

If so, then the following in the "run program" field of the task should work:

 

%comspec% /c YourBatchFile.bat

 

And your batch file this:

 

1. If you want to clear the recycle bin of every user

 

@echo off

setlocal enableextensions enabledelayedexpansion

 

set drivelist=""

for /F "usebackq skip=1 tokens=1,2,3,4 " %%i in (`wmic logicaldisk get caption^,providername^,drivetype^,volumename 2^>NUL`) do (

if [!drivelist!]==[]  (

set drivelist=%%i

) else (

set drivelist=!drivelist!;%%i

)

)

rem ends for loop

 

for "delims=;" %%i in (!drivelist!) do (

cd %%~i\$Recycle.Bin

for %%j in (".") do (

forfiles /m @ISDIR /c "rd @PATH /s /q"

for %%k in (".") do (

if /i "%%~nxk" neq "desktop.ini" del %%k /f /q

)

rem ends iterative check-and-delete

)

rem ends loop through user dir

)

rem ends loop through drives

 

2. If you want to clear the recycle bin of a specific user

 

@echo off

setlocal enableextensions enabledelayedexpansion

 

set drivelist=""

for /F "usebackq skip=1 tokens=1,2,3,4 " %%i in (`wmic logicaldisk get caption^,providername^,drivetype^,volumename 2^>NUL`) do (

if [!drivelist!]==[]  (

set drivelist=%%i

) else (

set drivelist=!drivelist!;%%i

)

)

rem ends for loop

 

for "delims=;" %%i in (!drivelist!) do (

cd $Recycle.Bin\%UserID%

for /d %%j in (".") do (

forfiles /m @ISDIR /c "rd @PATH /s /q"

for %%k in (".") do (

if /i "%%~nxk" neq "desktop.ini" del %%k /f /q

)

rem ends iterative check-and-delete

)

rem ends loop through user dir

)

rem ends loop through drives

 

Notes:

  1. I'm using the for loops to find a list of drives and recurse through the "$Recycle.Bin" directories here.
  2. You can find the value you need for the %UserID% by searching HKEY_USERS for the desired username and taking the portion I emphasized by underlining--the key's full path and name should look something like "HKEY_USERS\S-1-5-21-3529903732-1407875178-4072884794-500\Volatile Environment\USERNAME".
  3. The if-statement is required because deleting the desktop.ini in any of them will not be good for the user of that recycle bin.

A very common place for the recycle bin to show up is "C:\$Recycle.Bin". An example of a single user's own personal recycle bin directory is "C:\$Recycle.Bin\S-1-5-21-3529903732-1407875178-4072884794-500". If you look for it via Command Prompt, you will have to tack on "/ah" on your "dir" command in the "\$Recycle.Bin" directory and "/a-l" in the subdirectories (to my knowledge, there shouldn't be any reparse points in the subdirectories--correct me of I'm wrong). This also holds for their subdirectories. You may have to do a little finding for the exact path (i.e. -1000 instead of -1010 in the case of one of mine), but you shouldn't have to look too far.

One thing to keep in mind is that every drive used by Windows has its own version with the same structure on the first level of its directory (ex. "D:\$Recycle.Bin" and "D:\$Recycle.Bin\S-1-5-21-3529903732-1407875178-4072884794-500" as one of its subdirectories). I hope this helps.

vijay a. verma say:
You can execute following to gain space on your computer

1. Use Storage Sense to delete unnecessary files
Type Storage in Windows Start Search Box > Click Storage > Click Change how we free up space > Down below there is button Clean now under Free up space now which you should press

2. Use Disk Cleanup
Type Disk Cleanup in Windows Start Search box > Click Disk Cleanup > Check relevant options (you can also clean system files)

3. Analyze disk space
You should analyze your existing space with help of third party softwares and remove un-necessary files. TreeSize is most popular one but my personal favorite is WinDirStat.

TreeSize Freeware version - https://www.jam-software.com/treesize_free/

WinDirStat (Released under GPL license) - https://windirstat.net/download.html

4. Check for log files in CBS directory
Go to C:\Windows\Logs\CBS > Check the size of CBS.Log and if size is huge >
Type services in Windows Start Search box > Click Services > Scroll down to find the Windows Modules Installer service and double-click on it > Set its startup type to Manual > Reboot your computer

Go to C:\Windows\Logs\CBS > Delete all of the files in the CBS folder (or rename and move to a backup drive) > Reboot your computer > Then go back into the Services App and set Windows Modules Installer Service back to Automatic Startup (Note CBS log files contain information about windows update.

5. Microsoft has also some suggestions to improve disk space.
https://support.microsoft.com/en-us/help/12425/...
