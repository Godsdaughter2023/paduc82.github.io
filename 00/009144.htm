doug robbins - mvp office apps & services (word) say:
For the most part, you should not have a problem (of course, instead of using File New in VBA, you would use Documents.Add)  The only time where you may come up against something is that sometimes the decimal and thousands separators in Windows may be set to the reverse of the normal period and comma using in English and or a semi-colon may be used as a separator.

a. user say:

Two things that should make your testbed closer to (say) a French language installation of Office are:

 a. change the Display language (Word->File->Options->Language->Display language)

 b. change the regional options in Windows Control Panel to reflect the locale you want to test (e.g. French/France or whatever.

For (a), I would backup my normal.dotm template first. Office also usually requires that you restart Office programs before the change is made. You may need to get the relevant language packs to see menus and (for example) paragraph style names in the selected language. I think in Office 365 you can probably get the language packs you need from within the Word->File->Options->Language dialog box (e.g. by clicking on the "How do I get more Display and Help Languages from Office.com" but I haven't done it for a while so cannot remember whether you need a specific version of Word.

Sometimes it would probably also be necessary to install a suitable keyboard or IME (e.g. for Chinese), but I suspect there are few bits of VBA that would really require you to do that.

Other than the decimal point/comma type things Doug mentions, other things you have to look out for are pieces of code that use strings to identify certain things in your code. In the case of Word, I would say the main problem is code that uses strings to identify built-in style names such as "Heading 1". Sometimes you can replace them using locale-independent enums (such as wdStyleName)

Incidentally, it is not just VBA that can go wrong when you work with different language versions of Word. Some field types can also go wrong because the decimal point character, the thousands separator, the list separator and the currency symbol can be different in different locales.

alan wu msft say:

Hi Iris,

Office 365 ATP (Advanced Threat Protection) Subscription/License should be used combined with another Office 365 Subscription which has EOP license. For more information, you can refer to https://products.office.com/en-sg/exchange/online-email-threat-protection#howToBuy.

So, below are the answers to your 3 questions:
1.    I think EOP is required. But where I can check whether the school domain has EOP or not?
If your Office 365 Subscription includes Exchange Online license/service, it has the EOP features. The Subscription can be Office 365 Business Premium, Office 365 E3, Office 365 Education, etc. For more information, you can refer to Office 365 service description where you can find what Office 365 Subscriptions include EOP.

2.    If EOP is not ready, where I can get it? Does it free for school in Hong Kong?
If your school's Office 365 Subscription doesn't include EOP, it is recommended that you contact your school to buy an Office 365 EOP Standalone Subscription. Definitely, it is not free.

3.    In some set up guidance of EOP, it shows GoDaddy account is required when set up EOP. However, the school does not have a GoDaddy account, how can it be solved?
It is not necessary to acquire a GoDaddy account to set up the EOP. You only need to buy the Office 365 Subscription which includes the EOP.

Please feel free to let me know if anything is not clear.

Regards,
Alan
