ivan sanders say:

Hi All,

 

The issue usually the friendly name for the OWA Farm...

 

Ensure that the certificate meets the following requirements:

  • The certificate must come from a trusted Certificate Authority (CA) and include the fully qualified domain name (FQDN) of the Office Web Apps Server farm in the SAN (Subject Alternative Name) field (if the FQDN is not in the SAN when you try to use the certificate, the browser will either show security warnings or won’t process the response).
  • The certificate must have an exportable private key.
  • The Friendly name field must be unique within the Trusted Root Certificate Authorities store (OfficeWeb).
  • The FQDN in the SAN field must not begin with an asterisk (*).
  • The certificate properties and extensions do not matter. Office Web Apps Server requires no particular certificate property or extension
  • The FQDN of the certificate resolves to the IP address assigned to the load balancer for the Office Web Apps Server farm. 

In addition, I have noticed the Certificates I received from VeriSign don't have a friendly name. So after adding the certificate via MMC, Certificates, Right Click on the Certificate. click on properties, and enter the friendly name on the first Blank Line. The PoweShell cmdlet needs to have the friendly name.

 

However, what do you have listed in Subject Alternative Name, this either needs to be the FQDN of the load balancer or the FQDN of the servers and cannot start with an * and include the fully qualified domain name (FQDN) of the Office Web Apps Server farm in the SAN (Subject Alternative Name) field (if the FQDN is not in the SAN when you try to use the certificate, the browser will either show security warnings or won’t process the response).

span-unit say:

...  Also how do I find a recently installed exe file?

Open File Explorer - click Search Box to activate - Select Date Modified from tool bar - click time period - return to Search bar and type   *.exe    - Return

In Windows 8.1 , whatever icons appear, you can right-click each one and select Properties to view the exact date(s).

Note: You can Search by File Extension from the Tool Bar, but this will display EVERY .exe file in your system. You want the recent files displayed in the above procedure.
