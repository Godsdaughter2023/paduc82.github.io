macropod say:

What you're describing is quite possible if 'read-only' editing restrictions have been applied to the document and you're trying to edit a protected area. You can remove that restriction via Developer|Restrict Editing>Stop Protection.

It can also happen if the document has been marked as Final; in which case you should see a warning when you open the document. You can remove that restriction via File|Protect Document>Mark as Final.

Contrary to what you've been told, opening a read-only document as read-only (you get a choice) does not produce such a message - you can actually edit such a document but those edits can't be saved over the original. Likewise, repairing Office is not a remedy in this case.

If you do not have conditions to control in Range C16:C100, better:

Range("C16:C100").Interior.ColorIndex = 15

For example, this macro colors in gray the background of the cells C16: C100, Sheet1:

Public Sub m()
    Dim sh As Worksheet
    Set sh = ThisWorkbook.Worksheets("Sheet1")
    With sh
        .Range("C16:C100").Interior.ColorIndex = 15
    End With
    Set sh = Nothing
End Sub

Or, if you have conditions to control:

Public Sub m()
    Dim sh As Worksheet
    Dim lng As Long
    Set sh = ThisWorkbook.Worksheets("Sheet1")
    With sh
        For lng = 16 To 100
            'Code for your condition(If)
                .Range("C" & lng).Interior.ColorIndex = 15
            'Code end condition(End If)
        Next
    End With
    Set sh = Nothing
End Sub

Please note that it is clear to which sheet I refer

But, as written by Rick, you should post the whole macro to understand what triggers the error.
