Deleting fonts seldom, if ever, used - Microsoft Community

bob 1942 say:

Deleting fonts seldom, if ever, used

How are fonts safely deleted to save space and which ones shouldn't be touched because they are part of windows OS?  Also, what is "YAMMER"?
jay freedman say:

The link Joanna gave explains how to delete fonts, but it doesn't say anything about which ones you shouldn't delete.

First, be aware that deleting fonts -- even a lot of fonts -- isn't going to save all that much space. In Windows 10 with Office 365, the total size of the C:\Windows\Fonts folder is only about 400 MB. On most modern drives, that's a drop in the bucket. To some people, though, it's more important to reduce the length of the font list in programs like Word, to make it easier to find the desired fonts.

I haven't located a definitive list of required fonts. However, experience says these should not be touched:

  • the Segoe family, used for menus and various labels in Windows
  • the Windows "core fonts for the Web", listed at https://en.wikipedia.org/wiki/Core_fonts_for_the_Web#List_of_fonts_and_files
  • a few of the "dingbats" fonts such as Symbol, Wingdings, and Webdings
  • Cambria Math, used in Word's equation editor
  • any other fonts that you know you have used in documents that you may want to edit or print in the future

If you ever write or view documents in languages that use non-Roman characters (Asian languages, Hebrew, Arabic, etc.) keep the fonts that are commonly used in those languages.

In Word, if a document uses a font that is not installed (or has been uninstalled) on your PC, the program will substitute a font that it considers "similar" -- although it may not seem similar at all to you. To find the substitution and understand what Word is doing, see https://wordribbon.tips.net/T012657_Finding_Words_Font_Substitutes.html.

=ABS( IF(B13<0,B13-DataEntry!B$52,0))

Saving Styles - Microsoft Community

kataiae say:

Saving Styles

I have saved a formatted word document both as a template and as a quick style. My goal is to import a mindmap into word and then use quick styles to set the formatting to what I want. My problem with my current approach is 1. the template has words and it would require me to paste the imported document into the template and move the text around (surely there's an easier way?) and 2. saving as a quick style doesn't actually seem to save any of the formatting I've created in the document. For the second one, I've gone to the change style -> select style set -> and save it as a quick style. The formatting doesn't transfer (for some reason it's normal all over again).  Does anyone know a fix? 

charles kenyon say:
There are a number of very separate issues here.

Template is a special document type intended to hold resources for other documents.
Templates in Microsoft Word

A template can hold Styles. Some of those can be designated as Quick Styles. Designating something as a "Quick Style" controls its placement in the Quick Styles gallery. It does not change anything about the formatting of the style. A document cannot be saved as a Quick Style.
How to control the Quick Styles gallery on the Home tab in Word

If you have an existing template with styles you want to use, you can do a number of things.
  1. One of those is to attach the template to your existing document.  When you do that check the box to update styles from the template. (Go back later and uncheck this box, though.)
  2. You could create AutoText entries formatted in your style and save those in your template. Such an entry could contain a MacroButton Field prompt to mark the place to type text. Such a template could be saved as a Global Template in Word's Startup Folder. The AutoText would bring the style into your document.
  3. You could create a template stylesheet complete with macros to import styles and use that as a global template. A Global StyleSheet in Microsoft Word?

I'm sure this list is not complete, but it should give you some ideas.

I believe Charles is correct in pointing at the ActiveX controls -- especially when there are a lot of them in one document -- as the cause of the lag.

Please read at least the "Appropriateness for the Task" section of http://msdn2.microsoft.com/en-us/library/aa140269(office.10).aspx for a summary of the drawbacks of ActiveX objects. Although the article is old, nothing has changed in the meantime.

One issue you'll encounter in replacing those controls with content controls is that there is no exact equivalent of radio buttons. You can use dropdown list content controls instead.

charles kenyon say:

Unless you have users using Word 95, I urge you to use Content Controls rather than legacy Active X controls.

1. They do not work on macs or mobile devices.

2. Users have to permit macros to run if you put any commands in.

Content Controls have been around since Word 2007.

Create a Form Using Word Content Controls
