If you don't know where DMP files are located, here's how to get to them:

 1.    Navigate to the %systemroot%\Minidump folder.
 2.    Copy any and all DMP files in the Minidump folder to your Desktop and then zip up these files.
 3.    Upload the zip containing the DMP files to Skydrive or a hosting site of your choice and paste in your reply.

Please note that any "cleaner" programs such as TuneUp Utilities, CCleaner, etc, by default will delete DMP files upon use.

If your computer is not generating DMP files, please do the following:

1. Start > type %systemroot% which should show the Windows folder, click on it. Once inside that folder, ensure there is a Minidump folder created. If not, CTRL-SHIFT-N to make a New Folder and name it Minidump.

2. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Performance > Settings > Advanced > Ensure there's a check-mark for 'Automatically manage paging file size for all drives'.

3. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Startup and Recovery > Settings > System Failure > ensure there is a check mark next to 'Write an event to the system log'.

Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.

4. Double check that the WERS is ENABLED:

Start > Search > type services.msc > Under the name tab, find Windows Error Reporting Service > If the status of the service is not Started then right click it and select Start. Also ensure that under Startup Type it is set to Automatic rather than Manual. You can do this by right clicking it, selecting properties, and under General selecting startup type to 'Automatic', and then click Apply.

If you are having a problem with the upgrade process it may be due to something other than the driver version

you are using. 

What are the other system specifications for your machine ?

- If your motherboard is an 'older' model with an integrated graphics chip this may be what Win10 upgrade is

detecting. In this instance your motherboard is unlikely to have Wini10 specific drivers for the integrated graphics,

or other motherboard functions.

If you uninstall the integrated graphics it's possible the upgrade will proceed,  and it may run OK on generic

Win10 drivers, but personally, I wouldn't do it.

There are too may potential issues, which will only become worse over time if the motherboard cannot be

updated properly.

- Should you wish to proceed with the upgrade making a clone of your current Windows to another HDD

is the safest way to roll back to your current Windows version.

Or, if you have a Windows installation disk you can reinstall should a rollback be buggy.

- AOEMI  is a good free cloning (migrating) tool

Free Partition Manager - AOMEI Partition Assistant Standard

- I was still using 359.06 with a GTX 770 until recently, and I rather doubt the driver version is your issue.

Once you do manage to complete the Win10 upgrade if you want to use a different driver than the one

Win10 installs  you will need to block Win10 from reinstalling it's driver of choice when you reboot after

uninstalling the current driver.

You can use the Show/hide tool to do this.

How to temporarily prevent a Windows or driver update from reinstalling in Windows 10

btw - The GTX745 is not older than the GTX750. They are different models of the 700 series.

The GTX745 is an OEM model (used by HP, DELL, etc,).

- But just to make things a bit more complicated the GTX745, 750 & 750ti are all part of the 900 series family

Their gpu uses first generation Maxwell architecture that was later used with the 900 (2nd generation Maxwell)

series cards.

All the other 700 series use Kepler architecture gpu's .  More like  a latter day 600 series.

An i7 3770 would be suitable for Win10. I was able to get several Win10 specific drivers - IMF (chipset) ,

- Win10 will install a driver in the final stages of the upgrade, but you can install your driver of choice after that.

pauli o say:

Microsoft made changes to networking and removed Homegroup from Windows 1803 and that caused some problems.

To get you PC's to visible under File Explorer network section then most important part is to start some required services.

Push Win Key and type "Services", locate those services, start them and set Start type to Automatic. 

Services are:

   - Function Discovery Provider Host

   - Function Discovery Resource Publication

   - SSDP Discovery

   - UPnP Device Host

Look more detail from my example with screenshots how to make PC's to show in File Explorer network section, change network profile and setup file/folder shares.

https://answers.microsoft.com/en-us/windows/forum/windows_10-files/homegroup-removed-how-to-get-network-sharing-work/01277332-2916-4a68-853a-116696b20743
