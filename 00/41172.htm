If you're using Vista and working with the My Documents folder, then that is the problem.  In Vista, you use the Documents folder.  The My Documents folder is a junction point for backwards compatability (it was the main folder in XP but that's not true anymore) and should not be accessed or modified in any way by a user or even an administrator or owner (and has permissions set up to prevent this).  It has the everyone user set up as deny access and that overrides everything.  Don't try to change the permissions - just don't use this folder in Vista (use Documents instead which should work just fine).  Remember, permissions work on both the source file and the destination location - and you may be getting the problem with the destination location.

Cause
This issue occurs if User Access Control (UAC) does not let the activation processes run in a non-elevated command prompt.

Hi LD_407,

We currently don't have an option in Windows Live Messenger to know the time stamp on when you sent the video message if the save conversation history is disabled. What you can do is to enable it so it will record yout IM activity for future references.

Windows Live Messenger 2009:
1. Sign in to Messenger
2. Go to Tools (Shortcut is Alt+T), select Options
3. Click on Message on the left pane
4. Under Message History, check the Automatically keep a history of my conversations
5. The path under Save my conversations in this folder is where the conversation history is saved.

Windows Live Messenger 2011:
 1. Sign in to Messenger
 2. Go to Tools (Shortcut is Alt+T), select Options
 3. Click on History on the left pane 
 4. Under Conversations, check the Automatically save my conversation history.
 5. The path under Save my conversations in this folder is where the conversation history is saved.

Note: You can change the path under Save my conversations in this folder. The conversation history in Windows Live Messenger are saved as XML files, you can view them using a web browser.

As an alternative way in checking the conversation history, please follow the steps below:
 1. Open the Conversation window of a contact that you want to see your chat history.
 2. Press Alt + F, and then click on View Message History.

Other things you can try if the cumulative update does not resolve the problem:

Currently, a few steps have been outlined by the company to help users fix the bug:

1. Repair or Reset the missing apps:

Open Settings, and select Apps.
On the Apps & features tab, find the name of the missing app. Click the app and select Advanced options (if available).
If a Repair option is available, click Repair. If this option is not available, or if Repair does not resolve the issue, you may also try the Reset option, though you might lose any app data that was saved.
Once the repair or reset is completed, the app should again appear in the app list and can be pinned to the Start Menu.

2. Uninstall and reinstall the missing apps:

Open Settings, and select Apps.
On the Apps & features tab, find the name of the missing app. Click the app and select Uninstall.
Open Store and then reinstall the missing app. Once installed, the app should appear in the app list, and can be pinned to the Start Menu.

3. Re-register the missing apps using PowerShell – If you have a lot missing apps, advanced users can instead attempt to restore all of them at once using the following PowerShell commands. Note, however, that if Steps 1 and 2 were already attempted and did not restore your missing apps, this PowerShell solution will also likely not succeed.

In Cortana, type PowerShell. In the search results, right-click Windows PowerShell and select Run as administrator.
In the Powershell window type the following commands. These steps may take a few minutes to complete.

reg delete “HKCU\Software\Microsoft\Windows NT\CurrentVersion\TileDataModel\Migration\TileStore” /va /f

get-appxpackage -packageType bundle |% {add-appxpackage -register -disabledevelopmentmode ($_.installlocation + "\appxmetadata\appxbundlemanifest.xml")}

$bundlefamilies = (get-appxpackage -packagetype Bundle).packagefamilyname

get-appxpackage -packagetype main |? {-not ($bundlefamilies -contains $_.packagefamilyname)} |% {add-appxpackage -register -disabledevelopmentmode ($_.installlocation + "\appxmanifest.xml")}

Once the PowerShell commands are completed, the apps should appear in the app list, and can be pinned to the Start Menu.
