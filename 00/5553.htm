The "fat cross" has always been the default mouse pointer (cursor) in Excel when the mouse pointer is over a cell, since at least Excel 95. It changes to an arrow when the mouse pointer is over the ribbon, over the sheet tabs, over a scroll bar or over the status bar.

gord dibben say:

The default for years and many versions has been fat white cross.

You can change the behavior via VBA

Run these from Immediate Window to change the cursor.

Application.Cursor = xlNorthwestArrow

Application.Cursor = xlDefault

xLDefault enables the fat white cross and small black drag cross

also the larger 4-headed black move cross when working with cells.

xlNortwestArrow enables large white arrow and removes the three crosses

These do not retain settings when Excel is closed although you could use VBA to change the settings for any workbooks that are opened.

There may be a Registry setting you can change but nothing I can point to.

Are you using Microsoft's Switchboard Manager wizard? If so, you have several options of what can be done with a Switchboard Item. You can open a form in Add or Edit mode, Run a report, etc. You can also run a macro or a VBA Function.

For the justification you want, click in the paragraph and press Ctrl+J. What you're calling "fill justify" and Word calls "distributed" is a very rarely used alignment that's applied by Ctrl+Shift+J.

If you look in the Paragraph formatting dialog at the items in the Alignment dropdown, the "fill" alignment isn't there at all. The "Justify" choice is the one you're asking for.

charles kenyon say:
As has been pointed out, what you are using is what Word calls Distributed alignment. For the best look approximating what you want within Word, use Full Alignment (Ctrl+J) not (Ctrl+Shift+J) and turn on the Compatibility option to Do full justification the way WordPerfect 6.x for Windows does.



Text Alignment and Justification in Word

scottgem say:

First, to expand on the concept of bound and unbound forms. As noted a bound form has its Recordsource property set to a table or query. If you create a blank form, it will be, by default, be unbound. If you create a form using a wizard it will be bound. Since you didn't indicate what version of Access you are working with, we aren't sure what wizards you have available. With a bound form, Access handles the I/O between the form and the underlying table. As soon as you move focus from the current record, the record is committed to the table.  You can still have Submit and Cancel buttons that would save or not save the current data. These would have to be coded using VBA.

If you use an unbound form, then you have to handle the I/O between the form and the table yourself with VBA code. Appending new records is fairly easy, but retrieving existing records for editing is harder. Especially if you are not familiar with coding.

If you search on Access Unbound forms you should find code samples on how to work with them.

Hope this helps, Scott<> P.S. Please post a response to let us know whether our answer helped or not.
