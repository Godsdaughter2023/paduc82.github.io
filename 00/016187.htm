That won't happen.

You seem to be describing the completion milestone (0 duration?), rather than the task, as the FF predecessor.

If your milestone is assigned a date constraint which makes it finish later than scheduled finish date of the task successor, then that forces the task to finish no earlier than the finish of the milestone, ie it moves to the right. But it does not make the duration longer in order to retain the start date so the start date also moves to the right.

You could just as easily, and better, describe this with a FS0 predecessor link from the task to the milestone.

You could make a start milestone,

then turn the task into a hammock by copy/paste special, link from start date of start milestone to start of the task, and from finish date of the finish milestone to finish of the task.
