Hi,

While the connection is a single-step process, disconnection is a two-step process. The disconnection should be performed in this order particularly if the tasks have been tracked. Once the project is disconnected from the pool, the resources and tracking will remain in the project file.

dadboddan say:

Old/Terminated User Mailbox Archiving - PST Export

I work for a MSP with numerous clients. Many of them require local pst backups for archiving purposes. Many of them do not want to pay for a license for a user that is no longer with the company, which makes sense for small to mid size businesses that don't want added costs that aren't necessary.

My question is, why is there no possible way to export and archive a users mailbox directly to a pst from within the Office 365 admin or exchange portal? It is possible to import pst's INTO Office 365, but not export. The recommended way directly from Microsoft is to export through the Outlook desktop application. This can be quite unwieldy for companies that may do mass archiving a few times a year. So if I have to archive 20 users that's 20 times that I have to create a profile, which may include adding a license back to the user, enabling sign in, etc. Then I have to load up each of those users to export to a pst to be stored on the local or hosted server. I am aware of the shared mailbox solution, but many clients do not want to use that option and prefer to have their data.  There should be a better solution for this process. I'm sure I am not the only one that thinks this. The capability is there. Is this just a way for Microsoft to drum up more money through archive and ehold licenses? Thoughts?

iry chen msft say:

Hi DadBodDan,

For your first concern, generally all items (emails, calendars, contacts) can be exported out from a content search (when you run content search, you select Exchange email for selected location). And unsearchable items are mailbox items that can’t be indexed by Exchange Search or that have been only partially indexed. An unsearchable item typically contains a file, which can’t be indexed, attached to an email message. You can know more details for files the are returned as unindexed items when you run a search:

https://technet.microsoft.com/en-us/library/ms.exch.eac.unindexeditemslearnmore%28v=exchg.150%29.aspx?f=255&MSPPError=-2147217396

For your second concern, when you export result, you can select One PST file for each mailbox under  Export Exchange content as, then you can export them out with the MicrosoftOffice 365 eDiscovery Export Tool (when you clicked Download results, you will be prompted to install).

Please remember to copy your Export Key since you need to use it to run the export tool and also note you must use Microsoft Edge or Internet Explorer to download search results.

After you exported out to your local PC, you will see a folder named by your Search name. Open it, you will find all mailboxes will be divided to different pst files.

So, as my first reply "since there is no option to export an mailbox directly... or if you want to export all mailboxes, you just need to select Specific locations and Modify without any condition",if you want to export complete mailboxes, you should export all mailboxes in your organization and then select it's own pst file to import after exported to local.
