How to use Caption to Label Figure 1a, Figure 1b then Figure 2? - Microsoft Community

lisbri02 say:

How to use Caption to Label Figure 1a, Figure 1b then Figure 2?

This question was asked previously (https://answers.microsoft.com/en-us/msoffice/forum/msoffice_word-mso_other-mso_archive/how-to-use-caption-to-label-table-1a-1b-then-table/79d7624f-8390-4c60-ade0-650674425eb5) but is now closed. I have found a better answer than the suggested entering the letter manually.

I am using Word 2016, but this should work in any previous version of Word.

The following SEQuence will give the desired results as displayed in the title:

Figure: {SEQ Figure \* ARABIC \c }{SEQ -\* alphabetic \ * MERGEFORMAT }:

where:

{SEQ Figure \* ARABIC \c  } 

repeats the Previous SEQuence Number (e.g. Figure: 30), and

{SEQ -\* alphabetic \ * MERGEFORMAT }

creates a new Alpha SEQuence (e.g. Figure: 30a) while

\ * MERGEFORMAT } 

joins the two SEQs.

NOTE: A Dash (-) must be placed after the SEQ and before the

   \* alphabetic

   to prevent an error with the Alpha sequence.

To return to the next Figure Number, use:

Figure: {SEQ Figure \* ARABIC \n }

where

/n

continues from the previous number (e.g. Figure: 31).

Repeat as necessary.

I hope this helps someone else who is trying to figure this out. It is not clear anywhere in Help and an I suggest that Microsoft add it to the SEQ Help Info that appears within Word when you Search SEQ Field.

Merge table cells and font changes. - Microsoft Community

mattbranch say:

Merge table cells and font changes.

When I merge cells in a word table, the font changes to what appears to be bold.  All indications, other than the appearance, are that the font is exactly the same as before, however, upon printing out, the font is indeed in bold.  Can you explain how I can prevent this?

Word:Mac 2011, Version 14.0.0(100825)
iMac OSX Version 10.6.6
john mcghie say:
Aravinda:

A Support Engineer is supposed to know the answer to this one!

Matt:

When you MERGE two table cells, Word transfers the style of the second cell into the cell that remains.  (Basically it's an unanticipated side-effect of a horrible kludge they put in to handle deleting paragraphs -- a table is a special case of paragraph).

So let's assume you have column 2 with the style Table Body and Column 3 with the style Table Heading. If you merge cells 2 and 3, you will end up with a cell in column 2 that has the style Table Heading.

The same thing happens if you have applied a Table Style instead, but it's a bit harder to see, because a Table Style can apply different formatting to specified rows and columns.

In the example above, if you have specified a format of "bold" for the "Last Column" in your table style, when you merge cells 2 and 3, your cell will indeed get the Bold format, because it is the "Last Column".

Now for the complex bit: preventing it :-)

You have to be careful how you format in the first place.  Or put the formatting right after merging :-)

Or you can be a lazy person like me, and use Table Styles that are carefully set up.  I ensure that my table styles do not contain different formatting for the last column.  They do for the first column, but not the last. That enables me to merge cells without suffering the problem.

Since a Table Style reformats dynamically, the table formatting will update dynamically as you merge.  If you use Paragraph Styles to format your tables, then you will need to manage what happens when you merge cells, because the paragraph styles do not re-apply dynamically.

Hope this helps

a. user say:

Use ctrl-shift-spacebar to insert a "non-breaking" space between each pair of words in the name.

e.g. click immediately after "Mr.", press ctrl-shift-spacebar to insert the non-breaking space, then delete the old space after it. Then click immediately after "Jeff" and repeat.
