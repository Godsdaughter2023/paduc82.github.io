How to Calculate Calendar Duration in Project - Microsoft Community

tim_996_601 say:

How to Calculate Calendar Duration in Project

Hi, 

I'm using MS Project 2013 and need a column with calendar days from project start.  There was a prior posting on this site about it, but I've tried all the suggestions without success.  (ie; have used a custom field with formula as suggested)

Any suggestions would be much appreciated.

thank you,

Tim

john - project say:

Tim,

I'm not sure what all the suggestions were that you tried but this will work

Text1=DateDiff("d",[Project Start],[Start]) & " days"

Attempting to customize a formula for a duration field (i.e. Duration1) is more complicated because Duration fields are pre-formatted to display duration based on the user's options setting for the definition of a "day". By default, a "day" is 8 hours. For example, in the simple plan shown in the screen shot below the Project start date is 1/2/17. Both the Duration1 and Text1 fields have been customized to display the calendar days from the project start. The Text1 field uses the straighforward formula above. The Duration1 field needs the following formula (assuming a default definition for a "day"):

Duration1=DateDiff("d",[Project Start],[Start])*1440/3

Why the 1440/3 "fudge factor"? As noted above the Duration1 field is already pre-formatted so it must be re-formatted to display what we want. There are 1440 seconds in a 24 hour "day" so the existing Duration1 format needs to be first converted to minutes. Then there are three "8" hour periods in a 24 hour "day" so the resulting value needs to be divided by 3 such that the end result is in calendar "days".

Hope this helps.

John
