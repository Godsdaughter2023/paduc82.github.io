steffenlutz say:

"Click to call"-step needed when using the tel: protocol

Greetings,

I'm running Skype for Business 2015 (15.0.4849.1000) 32-bit on a Window 10 laptop (Office 365 for business). When placing a call via the tel-protocol (e.g. tel:+15141234567) the call is not placed automatically - instead the call has to be confirmed by clicking on a line in the call window (Click to call +1 (514) 123-4567). It's the same behavior on other users laptops.

On my previous laptop (Win 7) calls were directly placed when clicking on a tel-protocol link without the "Click to call"-line confirmation.

How can the "click to call"-confirmation be omitted in Skype for Business 2015?

Best regards,

Steffen 

allen z. msft say:
Hi Steffen,

We have tested on several versions, and the behavior is the same. It should be expected that the "Click to call" option will appear, and we cannot skip it.

Moreover, I’d like to explain that using the tel: protocol may not be the recommended way to start a Skype for Business call. Therefore, if you encounter anything wrong, we may not be able to provide support. The recommended ways are described in Make and receive calls using Skype for Business.

And if you have any concerns about using Skype for Business, it is suggested you submit feedback to Microsoft via https://www.skypefeedback.com/forums/299913-generally-available

Thanks,
Allen
