joshwright- [o365] say:

Thank you very much for putting me in touch with the Support Engineer Allen.

I'd Like to share the solution they had, which fixed my issue.

Troubleshooting:

===================

By running the recipient sync job in the Cloud, we find the errors:

“Failed to enable the new cloud archive zzzzzzzz-zzzz-zzzz-zzzz-zzzzzzzzzzzz of mailbox mmmmmmmm-mmmm-mmmm-mmmm-mmmmmmmmmmm because a different archive xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx exists.

To enable the new archive, first disable the archive on-premises. After the next Dirsync sync cycle, enable the archive on-premises again.”

A. Login the Local Exchange server:

Get-remotemailbox –identity <alias name> | fl *archive*

B. Find the archive guid doesn’t match.

Solutions

==============

Then we used the following method:

a. Backup the archived mails if the archived mails can be visible in the Cloud. You could try to use Outlook to export the archived mails to the PST file.

b. Then disable the DirSync temporarily.

c. In the Local exchange server, run the command:

set-remotemailbox –identity <alias name>  -archiveguid xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

#we replace the local archive guid with the online archive guid.

d. Run the command to make sure the archive guid is modified:

Get-remotemailbox -identity <user> | FL *archive*

e. If it is modified and matches the guid in the cloud, please enable the DirSync.

f. Please run the command:

Get-msoluser -UserPrincipalName UPN | FL

g. Please check whether the provision errors are cleaned up and make sure the title is synced or not.

Cause:

======

The archive guid doesn’t match the online guid, which caused the sync failure.

This was a great help to me. Basically, the online archive GUID did not match the on-premise GUID, so we manually updated the GUID via Powershell.

Well, I checked and indeed the blue wire was connected to the perm live and red wire to accessories. Now this happened not because I wired in icorrectly but  the red and blue wires were in the wrong places within the connector block. As far as I can remember the power cable supplied with the CK unit had the black connector block at one side for connection to the black box and had a white connector block at the opposite side which I connected to the SOT cableset. I bought the Accessory Interface Lead (SOT 043) cable set and all I had to do was to plug it into the stereo back and connect Volvo cables into the designated connectors. And the same with the CK-100. a. user say:

Yammer Notifications

I have set up a group in Yammer, but having issues with notifications. I've gone in and set up to get push notifications within the app, but it only sends me something when someone has started a new thread in the group. If I tag a name within that thread it doesn't get pushed to that person. 

Also, if I am in the desktop app, it doesn't send me push notifications to my phone at all. It would be nice to still receive them when I have walked away from my desk without having to shut it down every time.

Help please!

sreejith_n say:

Hello Jessica,

 

Email notifications and push notifications are sent when:

  • A new group announcement is posted.
  • Someone mentions you on Yammer.
  • Someone is added to a direct message.
  • A non-group participant is added to a private group conversation.
  • A message thread conversation in your Yammer Inbox goes from read to unread.
  • Someone replies directly to your message.
  • Someone sends you a new direct message.
  • The length of the notification delay interval (the time Yammer waits for a user to read new content before sending a notification) depends on whether the user accesses Yammer from a computer or a mobile device. If a user is on a computer and is active on the web, email notifications are delayed for 15 minutes. Notifications to users on mobile applications have a 20-minute delay.
  •  A push notification (subject to delay logic) goes out when there is activity on any Yammer Inbox conversation that makes the thread state change from read to unread AND is not a direct reply, an @mention, or a private message.

Please refer : https://support.office.com/en-us/article/Yammer-email-and-push-notifications-93e530e0-189f-4768-8f28-7683d48cc996
