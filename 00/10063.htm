don cuthbert say:
When an upgrade to a new build fails, these steps may enable it to succeed:
  • Completely uninstall any non-Microsoft anti-virus software.
  • Perform a clean boot before upgrading. https://support.microsoft.com/en-us/kb/929135 
  • Disconnect any unneeded peripherals, such as devices connected via USB and SD cards.
  • Disable all network adapters after the build has downloaded, but before it Restarts, or before running setup.exe, if you are upgrading via a mounted ISO or Windows 10 install media.
Sometimes disconnecting the Internet, such as unplugging the Ethernet cable, will enable a stalled upgrade to complete.

There should also be a log that recorded what happened. Check C:\Windows\Panther\setupact.log and cbd.log. There may be some details there.

The Reliability history should also indicate what error was encountered. Type reliability in Cortana search and choose View Reliability history.

doug robbins - mvp office apps & services (word) say:

If you send me a copy of the document, referencing this thread in the covering email message, I will investigate the issue.

This is the way in which they appear here in Word 2016:

Looking into this some more, in Word 2010, there was no facility to vary the layout of the footnotes and as you noted, they always appeared in the same column as the text in which the footnote reference was located.

 

In Word 2016 (maybe also in 2013), the facility was added for the user to vary the number of columns into which the footnotes are arranged, and it appears that the “Match section layout” refers to the number of columns across which the footnotes are arranged, and not to keeping the footnotes in the same column as that in which the footnote reference is located.  I guess the aim was to get a better balance of the text in the columns in the body of the document and you are not limited to the number of footnote columns being the same as the number of columns in the body of the document, which in the case of short footnotes does allow for a more efficient layout.

 

 

If you think that there should be an option to keep the footnotes under the column in which the footnote reference is located, you could raise the issue via http://word.uservoice.com/
