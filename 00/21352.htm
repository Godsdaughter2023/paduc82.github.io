aaronzhu_msft say:

Hi Fabio,

It’s not feasible to achieve your goal in Exchange Online services. Generally, if you flag an email in our primary mailbox, the email should appear in the To-Do List instead of your mailbox’s Tasks. If you flag an email in the shared mailbox, the email won’t appear in either To-Do List or shared mailbox’s Tasks. As a result, other members in the shared mailbox cannot see the flagged email. Given the situation, we recommend you submit feedback to our related team via: https://office365.uservoice.com/forums/273494-outlook-and-calendar. Providing your feedback is the best way to perfect our products and services.

However, based on my personal experience, if your goal is to let the 3 users see the same tasks, please try the following steps to see if it can be a workaround:

1. Create a security group and then put the 3 users into the created SG as members (this needs to be done by an Office 365 administrator).
2. Create a task under the shared mailbox’s Tasks.
3. Assign the created task to the security group above.
4. Then every member in the SG will receive an email, if they click Accept in the email, the task will appear in their own Tasks list.

Feel free to let us know if anything is unclear.

So for example, the second line in the data above represents a task needing 5 people to perform the work, so the task was duplicated five times in order to assign an individual to each task.  When the Number is 1, only one line in created; when 2, two lines; when 3, three lines  and so on.  Once all the duplications are completed, I then need to reset the Number (Col H) cell value to 1 on all lines.  All other data items remain the same.  The final result should show all lines with a 1 in the Number column.

Thank you for the help,

S. LeBeauf

eric lu msft say:

Hi Lebeauf,

You will get the data you want by running the following macros:

Private Sub CommandButton1_Click()

 Dim i As Long, totalRows As Long, lastRow As Long, Number As Long
    With ActiveSheet
        'for looping
        totalRows = .Cells(.Rows.Count, "A").End(xlUp).Row
        'index of row to add from
        lastRow = totalRows + 1 '<--| start pasting values one row below the last non empty one in column "A"
        'data starts at row #3
        For i = 1 To totalRows
            If .Cells(i, 8).Value > 0 Then
                Number = .Cells(i, 8).Value
                .Rows(lastRow).Resize(Number).Value = .Rows(i).Value
                lastRow = lastRow + Number
            End If
        Next i
       
    x = totalRows + 1
    Range(Cells(x, "H"), Cells(Rows.Count, "H").End(xlUp)) = 1
   
    End With
End Sub

Note: The processed data will show following your original data with number 1 in H column. I created a commandbutton to execute.

How do I delete files (trogans) after they have been - Microsoft Community

jazzmin790 say:

How do I delete files (trogans) after they have been quanteened?

Security Essentials quaranteened a trogan virus on my computer how do I delete it?

Thanks

Ellye

kosh vorlon say:

Hi Ellye,

 

You're perfectly safe with it in quarantine and personally I'd leave it there and let it be deleted automatically by the system in 30 days if you didn't change the default or whatever timeframe you set for retaining quarantined items (just in case); however, if you want to delete if from quarantine which is perfectly fine as long as you're certain it is something bad and not a "false positive", proceed as follows:

 

Open MSE.  Go to the History Tab.  Click on the Quarantine radio button.  Click on the quarantined file you wish to delete.  When you do three greyed buttons will activate at the bottom.  Restore, Remove, and Remove All.  If you want to delete just this one particular item in quarantine then while it is selected click Remove.  It will be deleted.  It will not show up in quarantine.

 

Note that in History if you click the detected items radio button it will still show it in quararantine, but that's just a list of what occurred, not what is presently the case.  It does not update the past with changes made now or in the future.  Because it still shows there does not mean it is on your system.  Again, it is only a list of past events and has no effect on files or anything and new events will not change what it shows occurred in the past.  If you removed it as noted above from quarantine, then it's gone.

 

I hope this helps.

 

Good luck!
