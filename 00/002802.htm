Assuming you're using Microsoft Security Essentials (MSE) on Windows 7...

If you're seeing an odd-additional extension - like .ccc or .vvv - appended to your original files names/extensions, you are most likely dealing with one (or more) of the latest variants (v2.2+) of one of the most prevalent crypto-malware in the ransomware threat landscape, which is known by the name of TeslaCrypt and/or AlphaCrypt (AKA Tescrypt - Microsoft's detection).
This infection has, unfortunately, encrypted your data.

These latest variants (v2.2+) of TeslaCrypt would leave ransom notes files (maybe already removed by MSE?) in each folder that a file has been encrypted, with names like: _how_recover_file_[3 random characters].txt/html, OR Howto_RESTORE_FILES.txt/html, the contents of which should be the same (ransom payment instructions).

For additional information please see:
- New variant of TeslaCrypt released with Minor Changes
- New TeslaCrypt version adds .VVV extension to encrypted filenames
- New TeslaCrypt version Released with Minimal Changes
- Latest TeslaCrypt Ransomware adds the .CCC extension to Encrypted Files

Please note that TeslaCrypt may (through their updates releases) rename their ransom notes files to add confusion to victims seeking assistance on the Web.
MSE's detection (Ransom:HTML/Tescrypt.D) may be a new/updated signature for Tesla's HTML file above. If possible/still available, can you please post here a screenshot of yours?

Also note that main executables responsible for the encryption should have been deleted after the encryption completed. When Tesla/AlphaCrypt is done encrypting the files on a system, it usually deletes itself and leave only the ransom notes behind.
Any remnants should have been cleaned off by your resident AV software and/or any other removal tools you may wish to use.

MSE (and other Microsoft malware removal tools) can detect and remove Tesla's main infection same as any other decent third-party anti-malware product, but they are unable to recover your already encrypted files.
Additional related information:
- Ransom:Win32/Tescrypt
- MSRT October 2015: Tescrypt
- MSRT November 2015: Detection updates

What should I do if files have been encrypted?

Your data cannot be recovered without Tesla's private key associated with your infection ID, which remains on a central server (C&C) maintained by the crooks and hence is not available for free; and unfortunately Microsoft cannot recover it for you in-house as it is not technically possible.

Unfortunately, at this time the only way to recover your files is through an external backup or by paying the ransom (which is, of course, NOT recommended) to obtain a decrypter tool and your private key from Tesla's C&C.
So once any malicious files have been removed, the encrypted files should be replaced from a recent backup as the first and best method to restore your data.

If you have been performing backups (as you should), then you should use your backups to restore your data.

Please note that earlier variants stored the private key as data files (key.dat, storage.bin) on the local disk which enabled victims to decrypt/recover their files with the locally stored private key (if that key was not destroyed or removed) using Cisco's Talos Group decryptor tool or BloodDolly's TeslaDecoder, but this is not the case anymore.

Since Tesla's v2.1+, their newer variants (including their v2.2's .ccc and .vvv extensions) no longer store any data files on the local disk and information stored in the registry as binary data only contains public keys and each shared secret.
Again, this info alone is not helpful without your private key.

You may wish to visit this BC's support and discussion topic on the TeslaDecoder tool, where you can read and learn more about specific scenarios and current state of things.

Also, this other ongoing discussion topic (Don't mind its original non-updated title ;) may be of help for you to ask general questions and seek further assistance about a TeslaCrypt infection.-

Finally, please note that the encrypted files themselves aren't a threat, since they do not hold the malicious encrypting executable in them. They are just encrypted.
If you have unsuccessfully tried to recover your files from a recent/prior to infection - external backup - as it's the first and best method to restore your data, or if there wasn't any; all you can do, at this point, is to back up the encrypted files 'as-they-are' on another storage media (like an external HDD) and leave it be until a solution is found in the near future - if one is ever found... There is always hope ;)

What usually happens is that if they paste into a block that already contains some formatting, the pasted data is going to inherit that formatting and loses its own formatting.

hansv mvp say:

When you copy a range of cells to the clipboard, Excel copies that range with all its properties (values, formulas, formatting).

If you paste into (the same instance of) Excel, it can paste some or all of those properties.

But if you paste into Notepad, only unformatted text is pasted, since Notepad is a plain text editor. All formulas and formatting is lost. So if you then copy the data from Notepad, only the unformatted text is copied. When you paste that into Excel, Excel has no way of knowing that the data was originally a formatted Excel range, it only sees plain text...
