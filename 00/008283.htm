toddalbertson say:

If I understand the problem correctly, your issue is that you have many footnotes and sometimes, the last footnote on the page will be broken and then continue on the next page.  If this is correct, you can fix the issue by editing the style for "Footnote Text."  Here's how you do that:

1.  Open the styles sidebar by hitting the arrow in the bottom-right corner of Home / Styles.

2.  It will look like the following.  Footnote Text may not show in this list, as it doesn't here.

3.  Click on Manage Styles.  The Manage Styles window will open.

4.  Ensure the Recommend tab is selected.  Scroll down until you see Footnote Text.  Click the Show button and then OK.

5.  The style will now show in the sidebar's list.  Now right-click Footnote Text and select Modify.  The Modify Style dialogue box will open.  Select Format / Paragraph.

6.  The Paragraph dialogue box opens.  Select the Line and Page Breaks tab.  Now, check the box that says Keep Lines Together.  Click OK to close the Paragraph window, and OK again to close the Modify Styles window.

Now, your footnotes will no longer break across pages (unless there's a paragraph break--in that case, ensure each footnote paragraph but the last is formatted with "Keep With Next").  One caution, though.  When footnotes are forced to not break across pages, this can create excessive whitespace in your document as Word adjusts pagination to keep footnotes in one piece.

    greginmich say:

    You’re missing the point that scanning with the MSRT is just a waste of time. Since Windows Defender uses the full set of Microsoft definitions, rather than just the small subset of definitions that the MSRT uses, you should just run occasional Full Scans with Windows Defender itself. A manual scan with Defender will find any malware that a Microsoft AV app can possibly detect – but since real-time protection is using the same definition set, a manual scan will normally only detect inactive and archived malware, or leftover malware fragments that were missed by Defender’s real-time protection. Real-time protection (on-access) will normally find any active malware when it automatically scans all files involved in file system read/write operations.

    Note that Windows Defender ignores Potentially Unwanted Applications by default, and that you need to enable PUA Protection in order to activate the complete Microsoft definition set:

    Right-click on the Start button and select Windows PowerShell (Admin). Then copy, paste, and enter this command:

    Set-MpPreference -PUAProtection 1

    In order to find active malware that Defender might have missed, you’ll have to scan with some third-party malware-removal apps, since these often have definitions that aren’t included in the full Microsoft definition set – which actually doesn’t even come close to covering the entire spectrum of online threats. And these third-party apps will consequently have the potential for detecting active malware that’s running on the system just because Defender wasn’t able to detect it. On the other hand, if you’re using a third-party AV solution for your real-time protection, then exactly the same principle applies – you should be using Windows Defender’s Limited Periodic Scanning feature; and also running scans with these third-party AV apps in order to see what’s being missed:
