Array formula or not ? - Microsoft Community

charabeuh say:

Array formula or not ?

Hello,

Someone told me that this formula is not an array formula:   =SUM({1;2;3}*{1;2;3})  (result = 14)

with A1=1, A2=2 and A3=3, I have noticed:

with Enter validation  =SUM({1;2;3}*{1;2;3})  gives 14

with Ctrl+Shift+Enter  validation  =SUM({1;2;3}*{1;2;3})  gives 14

with Enter validation  =SUM(A1:A3*A1:A3) gives 1

with Ctrl+Shift+Enter  validation =SUM(A1:A3*A1:A3) gives 14

with Enter validation  =SUM({1;2;3}*A1:A3) gives #VALUE!

with Ctrl+Shift+Enter  validation =SUM({1;2;3}*A1:A3) gives 14

If we replace anywhere {1;2;3} with A1:A2, the array validation is necessary to find the right result.

Is =SUM({1;2;3}*{1;2;3}) an array formula or not ? 

jllatham say:

I would consider it an array formula under the "if it walks like a duck..." rule.  You have declared two arrays of data to work with and so it does.

This definition: http://spreadsheets.about.com/od/a/g/2011-03-20-excel-array-formula-definition.htm would seem to share that opinion.

Perhaps the person who told you it is not an array formula only considers a formula to be an array formula when it is committed with the [Ctrl]+[Shift]+[Enter] key sequence.

You can do a web search for "Excel array formula definition" to view other references, including at least 2 Microsoft pages.

Also check out Charles Williams page on the subject: http://www.decisionmodels.com/optspeedj.htm which has links to two more pages by Excel MVPs (Bob Ulmas and Chip Pearson) who all share the opinion that if you reference an array within a formula, it is an array formula.

In summary, the general concensus among real-world users is that if you reference an array within an a formula, then it is an array formula regardless of whether or not you have to use the 3-key commit sequence to finalize the formula.

Some Observations:

I actually got a #Value error with Enter validation =SUM(A1:A3*A1:A3) [in Excel 2003] but got 9 in Excel 2010.  When entered with [Ctrl]+[Shift]+[Enter] the result was 14 in both versions.

If you try some experiments in VBA,

  Range("D4").Formula = "=SUM(A1:A3*A1:A3)"

then D4 displays #Value! and not the 9 that entering the same formula by hand did!!!

But if you code it as:

Range("D4").FormulaArray = "=SUM(A1:A3*A1:A3)"
then the anticipated result of 14 is obtained.

Then we find that if we code

Range("D4").Formula = "=SUM({1,2,3}*{1,2,3})"

The anticipated result of 14 is also obtained.

I can only surmise that with the curley braces used with parameters in a formula, then the Excel engine is smart enough to treat the whole thing as an array formula, but if no curley braces are part of the formula and you need it to act as an array formula, then you must use the 3-key commit sequence.

But there are exceptions to even that rule, because some formulas are, by definition, array formulas without including curley braces or committing them with the 3-key commit sequenct: SUMIFS(), SUMPRODUCT() and similar functions fall into that category.  Once again we can show that it does not always REQUIRE the 3-key commit sequence to create a formula that is an array formula, and thus walks like a duck, quacks like a duck ... :)


 

joeu2004 say: CharAbeuh wrote:

Is =SUM({1;2;3}*{1;2;3}) an array formula or not ? 


I would say it is not.  But this is really a terminology issue.  I would agree that Excel terminology is ambiguous.

If we look up "about array formulas" in Excel Help, we see two facts that distinguish an array formula:

  • "You create array formulas in the same way that you create other formulas, except you press CTRL+SHIFT+ENTER to enter the formula."

  • An array formula is displayed in the Formula Bar with curly braces around the entire formula, e.g. {=SUM(B2:D2*B3:D3)} [1].

Since neither is true for your formula, it is not an array formula per se.

Arguably, the Help page confuses the situation by stating:  " in an array formula you can [...] enter the array of values contained within the cells, also called an array constant".

But just because we can enter array constants in array formulas, that does not mean that all formulas with array constants are array formulas.  "All Greeks are men, but not all men are Greek".  (Apologies to Greek women ;->.)

Let's look at another example.  Suppose you have 1, 2 and 3 in B1:B3 and C1:C3.  =SUMPRODUCT(B1:B3*C1:C3) is not an array formula.  But =SUM(B1:B3*C1:C3) is an array formula; that is, it must be array-entered to work as intended.  In contrast, as you discovered, =SUM({1,2,3}*{1,2,3}) is not an array formula because it does not have to be array-entered.

-----
[1] [EDIT] Note that Excel documentation is not dispositive.  It contains errors; we're all human.  For example, the Excel 2003 Help page actually says:  "When you enter the formula ={SUM(B2:D2*B3:D3)} as an array formula ...".  But we cannot enter that literally as a formula.
