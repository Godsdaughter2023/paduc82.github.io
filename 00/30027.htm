See J W Stuart's reply, but let me add the following:

Most people with multiple "partitions"on a single physical drive
really have two partitions--one primary and one extended. The extended
partition then contains multiple logical drives.

Using a logical drive is almost exactly like using a partition, so the
logical drives are typically also called "partitions," even though it
is literally incorrect.

j w stuart say:

You are only allowed a maximum of four partitions. More than likely you have created four "Primary" partitions.

If you want to access that "Unallocated" space you have two choices:

1) Expand/Extend the last partition to include the unallocated space.

2) Delete the last "Primary" partition (save your data first) and create an "Extended" partition, which then can be used to create one or more logical drives.

    Hello Joachim Claus, welcome.

    This situation appears to be aimed more towards the manufacturer of the card itself. If the company is still around when Windows 7 is released, they will most likely push out a Win7 compatible driver.

    Microsoft will provide a very basic ( and I stress the word 'basic' ) driver for pieces of hardware. If the card requires a specific or more 'beefed up' driver to run, for lack of a better term, you would have to wait till the manufacturer releases an update. If you got an older generic driver to run the card properly, I would say you are lucky.

    Of course, the development team could try to mimic the older driver in future versions, however, with as many pieces of hardware that are present in today's market, they have to keep changing and updating the generic drivers which causes compatibility problems, such as yours.

    This situation is similar in nature with some versions of HP Multifunction printers that use the generic drivers from Microsoft on 64-bit systems. The generic drivers allow you to print but not to scan.

    In short, its up to the manufacturer. Microsoft dev teams will try everything they can to make one driver that works, but there are just too many different pieces of hardware out there to make one 'perfect' non-faulty generic driver.
