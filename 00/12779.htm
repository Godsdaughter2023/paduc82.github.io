greginmich say:

No, the interaction would be totally different if the SUPERAntiSpyware on-access scanner is loaded: The initiation of an on-demand SUPERAntiSpyware scan causes the Security Essentials on-access scanner to go into chase mode. You can see this in Task Manager or Process Monitor. That doesn’t create too much commotion. But if the SUPERAntiSpyware on-access scanner is also loaded, it might decide to go chasing after the Security Essentials on-access scanner. As soon as that happens, the Security Essentials on-access scanner will go chasing after the SUPERAntiSpyware on-access scanner. So if the SUPERAntiSpyware on-access scanner is loaded, the conflict can easily escalate to the point where the Security Essentials on-access scanner will go into a tail spin.

 

I don’t really know what’s happening here, I’m just pointing out why there is a rule against having two scanners loaded into memory. Because we don’t really know the reason for the crash, a bug report would be a good idea. But a conflict between on-access scanners is a known cause of such crashes, so it’s one possibility that you’ll need to take a look at.

bob jones aka: cybertaz say:

John's suggestion should work but may disrupt some other aspects of the document. Depending on how problematic that may be you might want to try this first...

I've created a similar scenario by using a small graphic formatted as Behind Text. That makes it virtually impossible to select the graphic with the I-Beam pointer. However, you can go to Tools> Customize Keyboard, then;

  1. Select All Commands in the Categories list
  2. Choose SelectDrawingObjects in the list of Commands
  3. Click in the Press new keyboard shortcut: field
  4. Press the key combination you want to assign, noting whether it is already assigned to another command
  5. Be sure to click the Assign button before clicking OK

The keystroke will cause the I-Beam to turn into an arrow. When you hover over the graphic the pointer will react & should enable you to click on the graphic to select it, then press delete to remove it. Use the keystroke to toggle the arrow pointer back to an I-Beam if necessary, but it should revert automatically when the graphic is deleted.
