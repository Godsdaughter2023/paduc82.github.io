entegy say:

Windows cannot mount the system partition from another disk. During boot, all relevant boot info and OS must be on the same volume because the disk manager is not yet loaded during boot.

Windows is not designed like Linux. It is designed to have all its system bits on the same volume. That will never change at this point in time.

The use for NTFS volume mounting to a folder is pretty much for if you run out of drive letters to mount to, drive letters being another thing from the past that is too late to change.

tomaustin say:

Found a solution that worked for me:

  1. Start regedit.exe
  2. Navigate to "Computer\HKEY_CURRENT_USER\Software\Microsoft\MediaPlayer\TunerConfig"
  3. Look for setting PREFERREDMETADATAPROVIDER -- it should have a value "pmpMusicMatch"
  4. Edit the setting PREFERREDMETADATAPROVIDER, and clear the value so you have a blank value
  5. Save / exit regedit.exe
  6. Restart Windows Media Player, you should be restored back to fai.music.metaservices.microsoft.com
