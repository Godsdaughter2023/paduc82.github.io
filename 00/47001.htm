considerate_guy say:

I hope you still have the clips because a Movie Maker project doesn't store a copy of the clips.  It just references them on your computer.  If you are seeing them previewing and playing back, then you do still have the original clips.

 

As for the saving problem, how long is your project timeline?  What I would suggest is making a copy of your project file (two copies actually).  Open up the first copy and try removing things from the end of the timeline.  Since you said it stops around 84%, try removing about 1/5 of the project from the end.

 

See if you can then save that as a movie.  If so, you at least have your 80% or so saved.  So save that project copy as "part 1" or whatever.  Now open up the second copy and try removing everything that was already turned into a movie in the first part.  In other words, get rid of everything except the end of the project that didn't save.

 

Now try saving this part as a movie.  If that works, save the project as "part 2" or whatever.  Finally open a new project, add the movies created from part 1 and part 2 and save them as a final joined movie.

 

If you aren't able to save the second part as a movie, that also tells you that the problem clip is somewhere in that section.  Go through and look to see if you have any different clip types or different songs in that last section.

filcard say:

Sadly, this is a problem affecting all users who have pen tablets - be they Wacom or any other brand. It is particularly troublesome for users of older model graphics tablets. For example, I am using a Graphire 4 model of Wacom tablet. The reason it is more troublesome for older models has nothing to do with the tablets or with Wacom... it is a Microsoft issue, verifiable by the thousands of users who are also experiencing the same issue. Microsoft is ignoring the problem, unfortunately.

Here's the rub. The problem was created by a Windows update. It is not a tablet driver issue or anything else. Basically in one of the semi-recent Creator's updates, Windows Ink Workspace became a heap of rubbish, forcing any device that is not a mouse to behave like a finger swiping on a screen. I know Microsoft's intent with this, but it was malconceived. They believe many of their users are using their fingers to swipe on a touchscreen, either with a touch-enabled monitor or with a Windows Phone. This might be true, but the changes made completely destroyed functionality for pens and tablets.

The number of issues is vast, and the list is large, so it's hard to describe or diagnose. Here are some of the issues created.

1) In most programs, you can no longer select text. This will affect nearly every program under the sun, including Microsoft products. Any notepad program, Microsoft Word, web browsers, etc, etc, etc. Basically, when you click and drag with your pen, gliding along the tablet's surface, Microsoft has changed the behavior to be as if you were dragging with your finger on a phone. Instead of highlighting text or doing normal application functions, it overrides normal behavior and forces a panning behavior within the window. Instead of selecting text, the window will scroll the direction you drag. Awful.

2) Clicking and dragging objects or painting brush strokes in many programs is overridden in the same way as above so that panning in the window is the preferred behavior. Some programs have become aware of this rubbish Windows change and have overridden this behavior to work properly... but it is an OS level issue, not the problem of various applications to solve. These responsible vendors are only solving it internally because they don't want to lose users who don't understand it's a Microsoft issue rather than theirs. No one wants to drag with their pen and have it scroll around! For fingers it makes sense, but NOT for pens! Artists want to drag objects, select text, paint brushstrokes, etc.... the same stuff people who use mice want to do. Graphics tablets should behave like mice, not fingers!

3) This #3 is what the original poster is talking about. There is a delay for ALL clicking and dragging in EVERY program and context... about 30 pixels effectively. Even if you try to click and drag a marquee on the desktop (with your pen, as all these problems relate to pen on tablet dragging), the dotted-line box will not appear until about 30 pixels later. Therefore you can know it's not a software issue. It's in every area of the OS. Scrollbars in Windows Explorer, everything. If you have a screen capture software that allows you to draw a box after pressing print-screen, the box will appear about 30 pixels after you drag. Drawing a box (or any other shape) in many graphics software will produce the same result. Clicking and dragging any scroll bar will result in a similar 30 pixel delay/lag. It's horrific functionality. Many paint programs have the same issue because the brush won't start painting until about 30 pixels after dragging has occurred. The only graphics software that doesn't do this delay are the ones smart enough to be aware of the recent issue and offer patches to fix the odd behavior.

It is a really terrible update that hundreds if not thousands of creative artists are affected by. Many are screaming about it online... but Microsoft is supposedly unaware of the problem, or is calling it a "feature" instead of a major bug.

I have been searching for weeks online for solutions, and I have tried all the suggestions. None of them seem to work fully. You can get rid of some of junk behavior with various windows registry hacks, but I can't seem to get rid of the delayed click-and-drag behavior. It makes most normal, everyday operations on a Windows 10 machine completely intolerable for anyone with a graphics tablet.

The only reason "newer" tablet models might work better is because Wacom is aware of this problem, and their newer driver has a checkbox to "disable Windows Ink," which might help some users. Older drivers don't have this checkbox because it was never a problem until this Windows Update which changed the underlying behavior of what a drag, click, and so on are... making them into finger motions instead. I hope Microsoft finally admits and addresses this issue. The solution is simple. There should be a checkbox to completely disable Windows Ink Workspace somewhere in the Settings dialog. A graphics tablet should function EXACTLY the same as a mouse except with a few additional features... 1) the screen should be configurable to be the fully size of the tablet instead of lifting a mouse or pen repeatedly to get across the screen and 2) pressure sensitivity. In all other ways, all operations should behave the same as a mouse, NOT as a finger on the screen. They have ignored this set of users who use graphics tablets to the frustration of all.

I believe this problem might relate to flick functionality in the Windows Ink Workspace. I have tried everything possible to disable and have messed with many registry settings. Nothing seems to get rid of the click and drag delay of about 30 pixels. Very annoying.

What I might end up having to do is buy a brand new tablet and hope it has the functionality to disable Windows Ink Workspace. Unfortunately many users who have new Wacoms and have this "uncheck the 'Use Windows Ink' button" option have reported that doing so also disables pressure sensitivity for their graphics tablet. So it seems no one has found a full solution. I'll keep hunting, and if I find something, I'll report back here.
