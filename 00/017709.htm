doug robbins - mvp office apps & services (word) say:
I am creating a template for the office, and we have mixed users of MS Word 2003 and 2007.  So I have to create .dot files.  Therefore, I am editing my templates using Word 2007 in Compatibility Mode.   

I used Legacy Forms (Not Content Control) to insert Text Form Field -> under properties, changed the default text to placeholder text such as "click here and type".  But I can't figure out how to set it up so that placeholder text don't get printed when it's not entered by the user.  


I would really really appreciate for your help!!


You would need to use a macro that iterates through the .Result of all of the formfields and for any of them that contain the default text, set the .Result to " "

Dim i As Long
With ActiveDocument
    For i = 1 To .FormFields.Count
        With .FormFields(i)
            If .Type = wdFieldFormTextInput And .result = "click here and type" Then
                .result = " "
            End If
        End With
    Next i
End With

If you put the code in two macros in the template that are name FilePrint and FilePrintDefault, The code will run when the user prints to document.

davidf2 say:

Yeah for some unknown reason MS has apparently taken away the option to insert page numbers on the 'inside' or 'outside' of a booklet page. This was changed as of Publisher 2010.

I found a less easy way to do this though. Assuming you have your publication in a booklet format go ahead and Insert > Page Number > Bottom left and uncheck the option to 'Show page number of the first page'.

Now go to Page Design tab > Master Page and click the down arrow > Edit Master Page and choose the two page master. (You can also just right click the first double page icon/thumbnail in the Navigation Pane to access the Master page if you prefer.)

Click in/on the footer section of the right hand page > Select the # character just as you would any text (You might want to zoom in to 100% so it is easier to see the # symbol.) > Format Text box tab > find the Alignment grouping and you will see nine small icons to the left of the Columns icon. Mouseover those alignment icons until you spot the Align Top Right, Align Middle Right, Align Bottom Right options. I would suggest you opt for the Align Top Right for now and your # symbol will shift to the top right side of the Footer text box or the outside edge of the page. Switch back to the Master page view and close that view. Your page numbers should be on the outside edge of the booklet pages now.

A Windows Defender Full Scan will be paused for the entire time that your PC sleeps, and then it will resume when the PC wakes up. The default sleep timeout setting for recent versions of Windows is 30 minutes (Control Panel > Power Options). So that’s definitely were some of those outrageously long scan times come from. The issue has been with us for many years now, and it’s well known to Microsoft developers. Let’s flashback to 2011 to make that point:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_scanning/when-we-run-a-full-scan/8ed1c27c-4ce0-44e5-a1a6-a2ed198061f3

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/porduct-improvement/d506ea23-afcd-4e3b-b847-bd26ea148343

I guess the “challenge” with laptops comes from manually initiated Full Scans, since the built-in Task Scheduler power conditions can prevent a scheduled task from running when a laptop is on battery power. Anyhow, in the meantime, I’ve spent many hours developing EPA-compliant workarounds that make changing and resetting the sleep timeout a little less tedious. But the majority of Windows users don’t want to be burdened with any technicalities, or with any energy-saving responsibilities, and I’m sure that most people simply set their PC’s sleep setting to “Never” and leave it at that – problem solved.

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender-scan-stops-when-windows-sleeps/f34df2d2-4245-4a24-906c-bada11b6d457

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02

And that's apparently  a perfectly acceptable solution for Microsoft – so good luck with getting this issue resolved via feedback.
