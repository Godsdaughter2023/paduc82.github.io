Concurrent Tasks in Project 2010 - Microsoft Community

kayeeee say:

Concurrent Tasks in Project 2010

Is there a way to have tow tasks linked together for both the start and the finish?

For example, it takes two people to operate a machine, one to feed raw materials into it and another to unload finished good out of it. Task A will be the feeding and Task B will be the unloading. Task B will begin about 15 minutes after Task A begins, and will end about 45 minutes after task B Begins.

In my first attempt to solve this issue, I've set Task B to be dependent of Task A, using FF with a lag. If Task A takes longer or shorter than scheduled will Project automatically adjust Task B to take longer. I know that this is not such an issue with two tasks, but is much more crucial with many tasks that overlap in some capacity. What are the best practices with this?

I'm using Project 2010.

john - project say:
Kayeeee,
To answer your first question, no and yes. No is the overwhelmingly preferred answer because a "yes" answer requires the use of paste links and those are notoriously corruption prone.

If the machine time is 15 minutes then why wouldn't you simply make Task B a successor of Task A with a normal finish-to-start  link with a 15 minute lag? Task B will automatically adjust depending on when Task A finishes, (i.e. loads the machine), and the machine time is accounted for in the lag.

John

kayeeee say: I think this is the way that I currently have it set up, but there is a problem.

Essentially, the two tasks are operating the machine. If there is variance in the productivity of the machine, ie. it takes more time to produce the same result, I need Task A and Task B to also expand with the increased duration of the machine. The idea is to capture the cost of the labor that is supporting/operating the machine.

I'm still new to MS Project. Are people using it for production planning and management?

john - project say:
Kayeeee,
If the machine has a variable operating time, then you will need to manually adjust the lag accordingly. An alternate approach is to list the machine as a separate task with a duration only (i.e. no resources). Then make Task A a predecessor for the machine task and let the machine task drive Task B. As machine time varies, you would adjust the duration of the machine task.

However, there must be something else going on here. In most cases machine operations (such as you describe) are well defined, that is, they take a very specific amount of time to do their process (i.e. numerical or time control machines). If the machine is used by an operator (i.e. non-NC operation), then the machine operator becomes the driving factor on how long it takes to do the operation.

To answer your question about using Project for production planning and management, I'll let someone else answer that question. I personally feel Project is better suited for development type programs where there is more-less a single output instead of a production environment where there are multiple variables feeding into multiple outputs.

John
