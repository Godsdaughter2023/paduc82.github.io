arthur.dent say:

This is documented behavior.  Check out this page:

http://answers.microsoft.com/en-us/insider/forum/insider_apps-insider_other/getting-the-windows-dvd-player-app/6c94b519-8b05-462e-a864-b7ad67d5c834

Specifically, FAQ #8.

If you reset your Windows 10 PC using Update And Recovery, then you will sever the link your PC used to have to its former installation of Windows 7/8/8.1.  After that, Windows Update will no longer be able to tell that you used to be eligible to receive the DVD Player app for free.

It won't be marked in the Windows Store as an app you own, because when it's purchased from the store, it will be installable on all copies of Windows that you own - but in reality, your entitlement to the DVD Player app is limited to only the individual PC which you upgraded from Windows 7/8/8.1.

The solution proposed is to re-install the edition of Windows 7/8/8.1 which originally entitled you to the DVD Player app, and then repeat the upgrade to Windows 10.  If you don't have recovery media for the older version of Windows 7/8/8.1, I'm afraid you may be out of luck.

However, there are 3rd party applications which you can use to play DVDs in Windows 10.  A popular, and 100% free option, is called VLC.  I haven't tried the Universal app available from the Windows Store so I cannot comment on it, but the standalone EXE installer you can obtain from VideoLAN's official website can certainly play DVDs.

nithyananda j say:

Hi,

1)      Since how long are you facing this issue?

2)      Did you make any changes prior to this issue?

 

 

Method 01:

On a working copy of a Windows system, set it to show hidden and system files. My guess is that your bootmgr is located on a drive that is not
C:\ and that drive is compressed. Even if compressed, Windows makes sure it creates an uncompressed bootmgr. However, subsequent
operations by a Windows 7 Loader do not, and may end up compressing it. This only happens if the root of bootmgr's location is set to use
NTFS compression, and the circumstances are rare.

The most common reason for this is the Windows boot drive being different from the boot drive in BIOS. When this happens, Windows will still install on the hard disk you've selected. However, the bootmgr file gets written to the hard drive defaulted in BIOS. While the Windows
partition is explicitly ensured by Windows to stay uncompressed, these checks don't apply to other hard drives.

Follow these steps to fix the problem:

1. Boot from your Windows 7 DVD and select Repair your Computer
2. Select Command Prompt.
3. in the command prompt, execute the following commands by typing them and pressing enter for each line:

C: (or the drive letter where your operating system is installed).

expand bootmgr temp

attrib bootmgr -s -r -h

del bootmgr

ren temp bootmgr

attrib bootmgr -a +s +r +h

1.      Restart the computer.
