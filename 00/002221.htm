karl timmermans [outlook mvp 2012-2018] say:

A little confused by your description and not sure what has or hasn't been done so in atttempt to simplify things - will summarize as follows:

#1 - On the basis that you are not using MS Exchange, then you options to publish any form is as follows:
    - in the Folder Forms lib (the folder to which it will be applied) OR
    - Personal Forms library

#2 - Where you are going to find a given form depends on what choice you made when it was published.

#3 - Deleting a form from any lib in O'2010 is done via the Forms Manager

File --> Options --> Advanced --> Custom Forms --> Manage Forms

#4 - Re-publishing a form using a different name is NOT "re-publishing" - it is the equivalent of publishing a new form and any contacts that reference the original form name need to be changed to reference the new form name. Additionally, the "default" form for the folder would equally need to be changed to reference the newly published form.

#5 - As for the best place to publish a given form in your circumstances, personally suggestion would be to publish a form to which it will be applied UNLESS the same form is going to be used in multiple folders within the same profile. In any case, some articles that discuss the topic are

How to determine where to publish a custom Outlook form
http://support.microsoft.com/kb/290802

Custom forms deployment : Where we can publish Outlook Custom forms?
http://blogs.msdn.com/b/deva/archive/2009/12/16/custom-forms-deployment-where-we-can-publish-outlook-custom-forms.aspx 

#6 - In reference to "User-defined fields in folder" - any user-defined field added to a custom form designed within a given folder will have those fields appear in the ":UDF in folder" group. However, if you publish the form to another folder, UDF's contained in the custom form will NOT appear in the "UDF in Folder" group but all fields will appear in the contact item's "UDF in this item" group (assuming that the form is the default for the folder). This does affect the data for individual contact items but result in those fields not being accessible in any "view"

To summarize

#7 - be clear as to what form name you want to use and ensure that it is published 
#8 - change the default form for a selected folder as per #7
#9 - just to ensure everything is set correctly - change the message class for all contact items for the folder in #8 to reflect #7

When your custom form issue is finally squared away - one final thing to take care of before it's forgotten, is to ensure that you have a backup of your custom form(s) if a future need arises

How to Back Up Outlook Custom Forms
http://www.contactgenie.info/how-to-back-up-outlook-custom-forms

There are minor variations to the steps below reducing some steps but these will ensure the form gets published where you want it.

From O'2010 menu bar --> click on Developer

   --> Choose Form

      --> Select "Look in" --> User templates in file System 

         --> Click <Browse> to folder containing the .oft file

            --> Select the .oft file you want --> Open

               --> With open custom form --> from menu bar --> Click Developer

                  --> Select Publish --> "Pulish Form As" 
                          (assumption here is that you don't want to make any changes to it)

                     --> Select "Look In" --> Outlook Contact Folders --> Browse
                               (or Personal Forms Library if that's what you want to do)

                         --> Select the specific folder you want to publish form to

                            --> Type in the unique part of the form name in <Display Name>
                                    (i.e. use "SomeForm" versUs "IPM.Contact.SomeForm)

                               --> Publish - done except for making sure you change the default
                                         for the folder where the form was published

For those using O'2007 - only difference is the starting point

     --> with O'2007 menu --> Tools

       --> Forms --> Choose Form

- If the folder where you are publishing the form already has contacts created with this custom form name - make sure the form name entered above is "identical" and to minimize any aggravation - make sure you publish it to the same library from which the contacts were originally created (i.e. eiter Folder or Personal Forms Lib)

Try this:

Sub SaveSheets()
    Worksheets(Array("Hide11", "Hide22", "Hide33", "Show44", "Show55", "Show66")).Copy

html5 for IE11? - Microsoft Community

joacchim say:

html5 for IE11?

Cannot launch videos on Win7 IE11.  Is there a way to add HTML5 to IE11??

a. user say:

Hello,

You might be experiencing this issue because one of the IE settings is preventing running HTML5 video formats. In line with this, we suggest that you check if alternative HTML5 codecs is enabled on your PC. Here's how:

  1. Click Start.  
  2. On the search bar, type Internet Options, and click on Internet Options from the results.
  3. Click on the Advanced tab.
  4. Under Multimedia, check Enable alternative codecs in HTML5 media elements.
  5. Restart your computer.

A third-party browser extension on Internet Explorer may be causing this issue. We recommend that you open Internet Explorer in no add-ons mode and try downloading again. Follow the steps below:

  1. Press the Windows Key + R on your keyboard to open the Run dialog.
  2. Type iexplore.exe -extoff and press Enter.

Let us know how it goes for us to assist you further.
