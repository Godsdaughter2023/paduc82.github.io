There appears to be an error in your post: EB0219 is NOT equal to 1EB021! Does the string start with the E or with the 1? 

Is there any consistency within each table? Guessing that the string starts with E, you could use expressions like

Mid([Table 1].[Fieldname], 5, 6)

Mid([Table2].[Fieldname], 6, 6)

to extract the string for comparison. If the string can occur anywhere, or variably, within the field then this becomes very tough and subject to false matches.

This is a great example of why the basic rules of relational design are important: one rule is that fields should be "atomic", each field containing just one chunk of information. Your difficulty arises because whoever entered this data did not follow this basic rule!

Create a new blank module and paste this function into it and save it as [modStringFunctions]

Public Function FindMatching(strOne As String, strTwo As String, intNumChars As Integer) As Boolean
    Dim intOne As Integer
    Dim intTwo As Integer
    Dim strFind As String
    If Len(strOne) < intNumChars Or Len(strTwo) < intNumChars Then
        FindMatching = False
     Else
        For intOne = 1 To Len(strOne) + 1 - intNumChars
            strFind = Mid(strOne, intOne, intNumChars)
            If InStr(1, strTwo, strFind) > 0 Then
                FindMatching = True
            End If
        Next
    End If
End Function

Then create a query with this SQL to return where "3" characters match. You can replace the 3 with any number you want to match.

SELECT tblStringTwo.FieldB, tblStringOne.FieldA, FindMatching([FieldA],[FieldB],3) AS Expr1
FROM tblStringTwo, tblStringOne
WHERE (((FindMatching([FieldA],[FieldB],3))=True));

How can I deal with part payments of invoices? - Microsoft Community

stonwardr say:

How can I deal with part payments of invoices?

I have what is essentially an accounting database under construction. Most customers naturally pay the amounts outstanding on their invoices, but some (damn them!) just 'drop in' an odd amount that is not correlated to any specific invoice. How can I deal with this?! Specifically I'm not sure how to relate the tables (do I have a 'part-payments' table & a 'paid-off' table? and if so how to relate them?

 

Thanks, Roy, WSM

ken sheridan say: If you have a single Payments table with columns such as CustomerID, PaymentDate and AmountPaid you can subtract the sum of payments per customer from the sum of all invoices per customer to give the balance due on each customer's account.  Here's an example of a query to do this, using the sample Northwind database:

SELECT Company,
SUM([Quantity]*[Unit Price]*(1-[Discount])) AS [Total Due],
SUM(NZ([AmountPaid],0)) AS [Amount Paid],
SUM([Quantity]*[Unit Price]*(1-[Discount]))-SUM(NZ([AmountPaid],0)) AS [Balance Due]
FROM ((Customers RIGHT JOIN Orders ON Customers.ID = Orders.[Customer ID])
LEFT JOIN [Order Details] ON Orders.[Order ID] = [Order Details].[Order ID])
LEFT JOIN Payments ON Customers.ID = Payments.[Customer ID]
GROUP BY Customers.Company;

I'm not sure whether Northwind already includes the Payments table or whether it's one I've added myself sometime in the past when dealing with a similar question.  I suspect I may have done so as it only includes payments for a couple of customers.

The other option is to have a table with credit and debit columns and compute the running balances per customer in a query like so:

SELECT T1.CustomerID, T1.TransactionDate, T1.Credit, T1.Debit,
SUM(T2.Credit-T2.Debit) AS Balance
FROM Transactions AS T1 INNER JOIN Transactions AS T2
ON (T2.CustomerID = T1.CustomerID)
AND (T2.TransactionDate<=T1.TransactionDate)
AND (T2.TransactionID<=T1.TransactionID Or T2.TransactionDate<>T1.TransactionDate)
GROUP BY  T1.CustomerID, T1.TransactionDate, T1.TransactionID, T1.Credit, T1.Debit;

scottgem say:
Like Ken I would use a Payments table. The main problem occurs if the customer has multiple invoices outstanding at a time and they just make payments without identifying the invoice they are paying on. In that case, you have to use a Foreign key for the account rather than relate a payment to an invoice. This also presents a problem in identifying when an invoice is paid. In that case I would use a FIFO method. payments are applied to the earliest invoice until paid in full. This would mean having a Paid in full Yes/No or date field in the Invoice record.

steve rindsberg say:
>> And believe me, "Be more careful inserting videos." has already crossed my mind.

;-)

Here's the fundamental problem:  without knowing where the original embedded video came from, there's no way to link to the source file.  We can't know what to link TO.

>> Plus, I'm not aware of an easy way to tell if a video is linked or embedded.

The free demo version of my FixLinks add-in includes a link reporting tool that'll give you this info.  It gives you the path to linked videos/sounds etc. and reports (granted, somewhat incorrectly) embedded vids as bad links.

BUT ... if you can move all the embedded movies into the same folder as the PPT itself, this VBA should get you most, if not all the way there.  Be sure to edit the path as directed in the comments, else it won't work.

PLEASE: run it only on a copy of your original file.  

And you might want to run a few quick experiments on putting the movies in the same folder as a test presentation and setting the path to "".  That might just give you pathless links, which'll work pretty much anywhere, so long as the movies and PPT/PPTX files stay in the same folder together.

Sub EmbeddedMoviesToLinkedMovies()

Dim oSl As Slide
Dim oSh As Shape
Dim x As Long
Dim sPath As String

' Edit this to the path where vids are stored,
' path must end in backslash
sPath = "C:\Stuff\MovieStuff\ThisProjectsMovieStuff\"

For Each oSl In ActivePresentation.Slides
    For x = oSl.Shapes.Count To 1 Step -1
        Set oSh = oSl.Shapes(x)
        If oSh.Type = msoMedia Then
            If oSh.MediaType = ppMediaTypeMovie Then
                If oSh.MediaFormat.IsEmbedded Then
                    Debug.Print oSh.Name
                    Call ConvertToLinked(oSh, sPath)
                End If
            End If
        End If
    Next    ' Shape
Next    ' Slide

End Sub

Sub ConvertToLinked(oSh As Shape, sPath As String)

Dim oSl As Slide
Dim oNewVid As Shape
Dim x As Long
Dim lZOrder As Long

Set oSl = oSh.Parent
lZOrder = oSh.ZOrderPosition

Set oNewVid = oSl.Shapes.AddMediaObject2(sPath & oSh.Name, msoTrue, msoFalse, _
    oSh.Left, oSh.Top, oSh.Width, oSh.Height)

Do Until oNewVid.ZOrderPosition = lZOrder
    oNewVid.ZOrder (msoSendBackward)
Loop

oSh.Delete
End Sub
