ashish mathur say:

Hi,

Array formulas are volatile formulas i.e. they recalculate for the slightest change made anywhere in the workbook.  So if you array enter most of your formulas, then a recalculation would trigger off for whatever change you make on any worksheet of the workbook.  Hence they should be used judiciously.

joeu2004 say:

Ashish wrote:

Array formulas are volatile formulas i.e. they recalculate for the slightest change made anywhere in the workbook. 

Absolutely wrong!  To demonstrate, enter the following into a normal VBA module:

Function myudf(x)
MsgBox "myudf " & x
myudf = x
End Function

Then array-enter the following formula (press ctrl+shift+Enter instead of just Enter) into A1:

=IF(myudf(TRUE), myudf(1), myudf(0))

After pressing ctrl+shift+Enter, we see two MsgBoxes.

Now enter 4 into any other cell.  Note that do not see any MsgBoxes popping up.  So the formula in A1 is not recalculated.  QED.

In contrast, replace myudf(TRUE) with RAND()<0.5, and press ctrl+shift+Enter.

After the initial MsgBox, enter 5 into any cell.  Notice that now we do see a MsgBox pop up.  That demonstrates that the formula in A1 is recalculated whenever any cell in the workbook is edited, the definition of a "volatile formula".

This is what Alex is asking about:  why not use ctrl+shift+Enter for all formulas, even if they are not "array formulas"?

Prevent losing email in Hotmail when you've set up POP3

Hotmail has a feature that prevents losing email messages if you accidentally set up POP3 to delete them after reading them on the server. Here’s how it works:

    The first time that Hotmail receives a POP3 instruction to delete email messages after reading from an email app, Hotmail won't delete the messages. Instead, Hotmail moves the email messages that were successfully downloaded by the email app into a POP folder and out of the inbox. The POP folder is available in the Folders list.

    Hotmail keeps the email messages in the POP folder until you sign in to your Hotmail account and verify whether Hotmail should respect or ignore the POP3 delete command.

To verify your POP3 settings:

        Do what my other program says—if it says to delete messages, then delete them.

    Click Save.

jay freedman say:

Greg Maxey's articles on ribbon customization all use the Custom UI editor.

If all you need to do is replace icons one-for-one, you can do this without the Custom UI editor:

  • Make a copy of the template that contains the custom tab, and change the extension of that copy's file to .zip.
  • Open the zip file, which contains several folders, and open the customUI folder. Within that folder, open the images folder. You'll see the image files for the custom controls. If you're lucky, the names will be informative and not just numbers or odd abbreviations. If necessary, extract the image files to another folder (outside the zip file) and open them to see what they contain. Keep notes!
  • In yet another folder outside the zip file, assemble the set of replacements icons. Rename those files to match the names of the corresponding original icons. This will avoid having to edit the xml that refers to those names.
  • Copy the replacement icon files into the customUI\images folder in the zip file, overwriting the old icons with the same names.
  • Close the zip file and rename the file from .zip back to its original extension.
  • Test the result by basing a document on the modified template. If all the new icons are correct and clicking them does the expected actions, (a) move the original template to safe storage and (b) rename the modified template to the name of the original template.

The alternative to this (admittedly work-intensive) effort is to use a computer that isn't subject to your company's restrictions, where you can download and install the Custom UI Editor.
