marshall barton say:

You could change the report's record source query to use parameter prompts as criteria by setting the  the date field's criteria to something like:
   Between [Enter Start date] And [Enter End Date]

That is a quick and dirty way of doing things and is not recommended for long term use.  The recommended approach is to create a form with text boxes (named txtStart ans txtEnd) for users to enter the start and end dates along with a command button to open the report with the date range filter.  The button's Click event procedure could look something like:
   DoCmd.OpenReport "name of report", acviewPreview, WhereCondition:= _

A text box's name is in its Name property, not its ControlSource nor its label's Caption.

Yes, the text boxes should be unbound.  Actually, for a form that is only used to supply the date range and open the report, the form should be unbound (ie. it's RecordSource should be empty).

It is a best practice to have:
   OPTION EXPLICIT
at the top of every module in your project.  This tells the VBA editor to warn you when you use an invalid name.  If you do not have that statement at the top of a module, the VBA editor will silenty create a Variant variable and use that, which makes it difficult to find the problem you are having and other simple problems such as simply misspelling a name.  There is a setting in VBA - Tools - Option to require variable declarations that you can set to have OPTION EXPLICIT automatically added to all NEW modules.  You should add it manually to all of your existing modules.

bernd p. say:

Onenote 2010: local notebooks or notebooks on Onedrive. Automatic local backup in both cases.

Free Onenote 2016: only notebooks on Onedrive, automatic local backup in the current version only after a manual workaround.

Paid Onenote 2016 (from Office 2016 or Office 365): local notebooks or notebooks on Onedrive. Automatic local backup in both cases, in the current version only after a manual workaround.

Concerning hard drive problems it's a theoretical pro for Onedrive that you have your notebook in the cloud. In practice the truth is that the current software for synchronising the changes made on your device(s) to the notebook in the cloud is buggy, unreliable and lame.

So my personal decision has been (and still is):

Use Onenote 2010 with local notebooks. And these notebooks are backed up using a reliable cloud service: Dropbox.

The consequence is that I use Onedrive for mobile clients on Android or iOS only with test notebooks.

doug robbins - mvp office apps & services (word) say:

The following macro will work in any version of Word from Word 97 onwards, splitting the document at each Section Break.

Sub splitter()
' splitter Macro
' Macro created by Doug Robbins to save each letter created by a mailmerge
' as a separate file, retaining the header and footer information.
Dim i As Long, Source As Document, Target As Document, Letter As Range
Set Source = ActiveDocument
For i = 1 To Source.Sections.Count
    Set Letter = Source.Sections(i).Range
    Set Target = Documents.Add
    Target.Range = Letter
    Target.Sections(2).PageSetup.SectionStart = wdSectionContinuous
    Target.SaveAs FileName:="Letter" & i
    Target.Close
Next i
End Sub

lopes - msft say: Hi Kopper1 .

If I understood your problem seems that you've hit the option " permanently delete all". This means that the items were already in delete items (recycle).

When you delete the items in your storage, you can go to deleted items and recover them to your storage. But once you hit permanently delete option in deleted items there is no way to recover those items.
