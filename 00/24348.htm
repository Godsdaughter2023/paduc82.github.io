Unless you want to modify and save a design change to the form each time the
form is closed (not recommended), you should have a table where you save
such info.  To be flexible, you'd want the records in this table to have
fields for form name and control name;  that way you could use it for
multiple controls on multiple forms.  Suppose you create a table called
"ControlDefaults:
 
ControlDefaults
---------------------
   FormName (text)
   ControlName (text)
   DefaultValue (text)
 
Make the combination of FormName and ControlName into the table's compound
primary key.
 
Create a function GetControlDefault to retrieve a value from this table, and
a function SaveControlDefault to save a value into the table.  Here are
quickie (untested) functions for that:
'
'------ start of (untested) code ------
Function GetControlDefault(FormName As String, ControlName As String) As
String
    Dim rs As DAO.Recordset
    Set rs = DBEngine(0)(0).OpenRecordset( _
       "SELECT DefaultValue FROM ControlDefaults WHERE " &
           "FormName = '" & FormName & "' AND ControlName = '" &
ControlName & "'', _
       dbOpenSnapshot)
    With rs
       If Not .EOF Then
           GetControlDefault = !DefaultValue
       End If
       .Close
   End With
    Set rs = Nothing
 
End Function
 Function SaveControlDefault( _
                   FormName As String, _
                   ControlName As String, _
                   DefaultValue As String)
    Dim rs As DAO.Recordset
    Set rs = DBEngine(0)(0).OpenRecordset( _
       "SELECT FormName, ControlName, DefaultValue " & _
           "FROM ControlDefaults WHERE " &
           "FormName = '" & FormName & "' AND ControlName = '" &
ControlName & "'', _
       dbOpenDynaset)
    With rs
       If .EOF Then
           .AddNew
           !FormName = FormName
           !ControlName = ControlName
       Else
           .Edit
       End If
       !DefaultValue = DefaultValue
       .Update
       .Close
   End With
    Set rs = Nothing
 
End Function
'------ end of code ------
 
Now, in your form's Load event, call GetControlDefault o assign the
.DefaultValue property of every control that should "remember" its last
value:
'
'------ start of code ------
Private Sub Form_Load()
    Me.DC_Estado.DefaultValue = GetControlDefault("YourFormName",
"DC_Estado")
 
End Sub
'------ end of code ------
'
And in your form's Unload event, call SaveControlDefault to save the
.DefaultValue of every such control:
'
'------ start of code ------
Private Sub Form_Unoad(Cancel As Integer)
    SaveControlDefault "YourFormName","DC_Estado", Me.DC_Estado.DefaultValue
 
End Sub
'------ end of code ------
'
Continue with the code you already have in the control's AfterUpdate, to set
the .DefaultValue whenever the value is changed, and that should take care
of it.

Well, I can't speak for Microsoft, but frankly, the Market for WinMo 6.x was sort of a disaster from the start.  Long before it existed, people using WinMo devices just bought or downloaded apps anywhere- from online stores, shareware sites, the developer's websites themselves, etc.  I've even bought WinMo apps on CDs in office supply stores way back when.

 

When Apple opened their exclusive App Store for iPhone, which made finding apps easier (and more profitable for the store- Apple takes a 30% cut of all sales) Microsoft scrambled to emulate that idea.  Unfortunately, only a fraction of the 20,000 or so existing WinMo apps ended up in the market, since many developers already successfully marketing their apps on their own saw little reason to pay to join MS' new Market and share their revenue with MS like iOS developers had to with Apple.

 

The Market was certainly a convenience, but its loss has in no way reduced the number of apps available for WinMo.

basilides say:

Parentheses Problem with Concatenate

Greetings!

When using concatenate a large number in parentheses were turned into a hyphen, as the examples below show:

JER 10:(8) became JER 10:-8

JER 33:(10) became JER 33:-10     ... as so on. Note that both parenthesis were dropped.

How can I (in Column K only) search for and replace items so that

JER 10:-8 becomes JER 10:(8) and

JER 33:-10 becomes JER 33:(10) ?

Also, I would like to make sure all these parentheses are pure text and not mathematical symbols.

Is there a way to do this because a simple search and replace does not work, plus even if it did it would not insert the second parenthesis?
hansv mvp say:

Try this macro:

 

Sub ReplaceHyphen()
    Dim cel As Range
    Dim arr As Variant
    Application.ScreenUpdating = False
    With Range("K:K")
        Set cel = .Find(What:="-", LookAt:=xlPart)
        Do While Not cel Is Nothing
            arr = Split(cel.Value, "-")
            cel.Value = arr(0) & "(" & arr(1) & ")"
            Set cel = .FindNext(After:=cel)
        Loop
    End With
    Application.ScreenUpdating = True
End Sub
