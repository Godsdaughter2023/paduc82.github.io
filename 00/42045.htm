greg carmack - mvp 2010-2020 say:

For a UEFI install the Windows Boot Manager should be set first to boot in firmware and remain there.  The hard drive is only set first to boot in a Legacy BIOS install to MBR.  Was your Windows installed in CSM or Legacy BIOS mode?

Please Post a screenshot of Disk Mgmt.

Another cause for Inaccessible Boot Device is that an Update loads a MSAHCI driver which toggles the SATA controller in BIOS setup/UEFI firmware to a setting other than the one it was installed in.  This should optimally be AHCI, but some have RAID or the older SATA set for this install.  Try the other setting - you can always change it back. 

Further problems compounded by non-AHCI might want to make you consider doing the Clean Install Windows 10 with SATA controller at AHCI (and latest BIOS/UEFI firmware).  Especially if you have a factory or Upgrade install, the difference is marked and there have been zero complaints from 100,000 happy users of that Wiki.

bossdweeb say:

I see you seem to have upgraded your laptop since 2016, but unfortunately it's still not really adequate for good performance with a

game such as GTA5, even though GTA5 does have a fairly low minimum system requirement relative to the system requirements

for maximum settings.

The lag probably comes from your machines inability to process the graphics adequately. Your Intel HD 610 graphics is better than

the old Intel GMA you had earlier, but your cpu is still a bit under the minimum specs for GTA5 @2.3GHz vs 2.4GHz.

This would be less of a problem if your machine had a dedicated graphics processor with at least 4GB dedicated VRAM.

Note that VRAM is only found on dedicated graphics cards. Integrated graphics uses you system RAM which is not as fast as real VRAM)

With games like GTA V where you have driving and foot action, the driving is going to lag if the frame rates on foot are low. Similarly

in big shoot-out with lots of enemy the frame rate will drop and lag will be introduced.

If you are already using all low settings, and a resolution no higher than 1366x768, about all you can do to improve performance

would be to lower the games resolution to 1024x768.

Some games are less prone to lag than others, but generally a game needs to run an average of at least 30/35 fps to avoid

occasional brief lag. 25 fps and lower and lag gets more serious.

The reason for the heat is you are stressing the cpu and graphics to the max. The lag likely gets worse because the machine

a game.

The best you can do to help keep a laptop reasonably cool is to use a cooling pad, but these still can't cool a laptop as well

as the cooling system in a desktop PC is capable of, largely because of the very cramped space within the laptop vs desktop case.

- The review for the HD 610 below shows benchmarks for a good number of games. About the only way you can determine

how well a game will run is to compare the system requirements for the benchmarked games with the system requirements of

games that are not in these benchmarks.

https://www.notebookcheck.net/Intel-HD-Graphics-610-GPU-Benchmarks-and-Specs.257794.0.html

frederik long say:

Please note that most replies in this forum come from volunteers in the general Windows community, not from Microsoft.

What you ask for can be done but is not trivial. Here are steps that are required:

  1. You instruct your BIOS to start the PC every night at 3am (for example)
  2. You create a scheduled task that runs at boot-up time under the System account. It invokes the batch file you see below. It passes a single number to the batch file:
    0 = default scan
    1 = quick scan
    2 = full scan
    3 = single file scan

The batch file would do this:
- Wait 5 minutes
- If someone is logged on, exit immediately
- Update the pattern file
- Perform the nominated scan
- Shut down if nobody is logged on.

@echo off
ping localhost -n 300 > nul
quser | find /i "console" && goto :eof
"C:\Program Files\Microsoft Security Client\Antimalware\MpCmdRun.exe" -scan -scantype %1
quser | find /i "console" || shutdown  /s  /f  /t:3

Reset IPv4 TCP/IP stack to installation defaults. netsh int ipv4 reset reset.log

Reset IPv6 TCP/IP stack to installation defaults. netsh int ipv6 reset reset.log

Reboot the machine.

In case it isn't clear, you need to type the bold commands, then press enter.

Hi Michael,

Thank you for writing to Microsoft Community Forums.

The UEFI Secure Boot may have been toggled off with a BIOS update. I'd suggest you to enable UEFI Secure Boot and check. To do that:

1. Power the device off and then back on.

2. Look for a message on the boot screen just before or after the manufacturer logo appears. You may need to press F1, F2, or Delete button, whatever key is indicated on the boot screen to enter the BIOS settings.

3. Inside BIOS, look for  a tab called BOOT and select that page. Depending on the BIOS manufacturer this page could be BOOT, ADVANCED, STARTUP, etc.
