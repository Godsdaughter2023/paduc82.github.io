albert d. kallal say:

First of all software don't work as above. If you going to make some calls to internet based services then guess what Mr. Sherlock? You are going to have to use some external libraries! (so asking for no external libraries is not only silly on your part, but is a pipe dream on your part).

 

Perhaps  you are looking for a solution that does not even require a computer? ;-)

 

Keep in mind the soap add-in tool kit used a reference to the Microsoft Office Soap Type Library.

 

And you can get that add-in code to work with Access 2010 with a bit of basic Access skills and competency (however I much suggest you avoid that add-in, and it is rather a messy approach to most soap services anyway).

 

However, while it is pipe dreaming on your part to avoid external libraries ("dependencies" to quote you), what you MOST certainly want to do is adopt a strategy in which you do not have to set any references inside of access (you thus will want to use late binding).

The next concept and strategy is of course to use an existing library that you can be darn sure will have been installed on the target computer. About the best bet is the MSXML library as it usually around if the computer in question has internet ability!

So, some code to hit yahoo web service would be:

Public Sub GetQuote2()

   Dim objXML           As Object
   Dim strSymbol        As String
   Dim strURL           As String
   Dim strWFormat       As String

   Set objXML = CreateObject("MSXML2.XMLHTTP")
           
   strURL = "http://ca.finance.yahoo.com/d/quotes.csv?s="
   strWFormat = "&f=sl1d1t1c1ohgv&e=.csv"
  
   strSymbol = "MSFT"
  
   objXML.Open "GET", strURL & strSymbol & strWFormat, False
   objXML.Send
  
   Debug.Print "Symbol = " & Split(objXML.ResponseText, ",")(0)
   Debug.Print "Trade  = " & Split(objXML.ResponseText, ",")(1)
   Debug.Print "Date   = " & Split(objXML.ResponseText, ",")(2)

End Sub

Now in the above soap call we asked for data back as csv, but most OFTEN the data return will be as XML, and thus it makes HEAPS of sense to use the xml library, since then you have all the bells and whistles to parse out the data returned from that web service. So you can use the above as a starting – Bing/Google MSXML for BUCKETS of examples on the web.
