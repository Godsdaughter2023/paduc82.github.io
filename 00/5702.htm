jay freedman say:
All default styles and margins, some settings, and any "formatted" AutoCorrect entries you created are stored in the Normal.dotm template in your Templates folder (by default, %appdata%\Microsoft\Templates). Copying that file from one computer to the same location on the other computer will bring along all that information.

For plain text AutoCorrect entries, see http://support.microsoft.com/kb/926927.

If you saved custom Quick Parts, they may also be in Normal.dotm or may be in %appdata%\Microsoft\Document Building Blocks\Building Blocks.dotx, depending on what you chose in the Create New Building Block dialog at the time. If you aren't sure, just copy that file anyway.

Customizations of the Quick Access Toolbar are stored in C:\Users\<your name>\AppData\Local\Microsoft\Office\*.qat (one for each Office program that has a customized toolbar).

The other customizations, such as the Ribbon color scheme, are stored in the registry and can't be copied directly from one computer to another.

You may want to read http://www.word.mvps.org/FAQs/General/FilesToBackup.htm and the pages it links to, but it hasn't been updated since Word 2003 and some things no longer apply. In particular, there is no Save My Settings wizard for 2007 or 2010 and the older one doesn't work with these versions.

Let me help you with the issue.

 

In Excel, leading spaces between the comma used for a field separator and the double quote will sometimes cause fields to be read in as unquoted fields, even though the first non-space character is a double quote. To avoid this, remove all leading spaces after the field-separator comma and before the double quote character in your CSV export file and then try again.

When importing CSV, do not try and use the quotes to impart type information to fields because the delimiters are always discarded.

duke carey say:

If you are open to a VBA solution, this code seems to avoid Excel's issues by loading the data into an ADODB Recordset object and then copying it in to the target range.  Embedded carriage returns and tabs APPEAR to be handled correctly.

WARNING: I've been unable to get this approach to bring in the first row as column headers.  It appears they are ignored

Option Explicit

Dim cn  As ADODB.Connection


' ********************************************************************
' a great source for getting the proper connection strings is
'     www.connectionstrings.com
' ********************************************************************
Private Function GetNewConnection(ByVal path As String) As ADODB.Connection
      Dim oCn As New ADODB.Connection
      Dim sCnStr As String
     
     
' It appears to make no difference whether you use HDR=YES or HDR=NO
'  can't figure out how to get column headers to be imported
'      sCnStr = "Driver={Microsoft Text Driver (*.txt; *.csv)};" & _
'               "Dbq=" & path & ";" & _
'               "Extensions=asc,csv,tab,txt;HDR=YES"

      sCnStr = "Driver={Microsoft Text Driver (*.txt; *.csv)};" & _
               "Dbq=" & path & ";" & _
               "Extensions=asc,csv,tab,txt;HDR=NO"

      oCn.Open sCnStr
     
      If oCn.State = adStateOpen Then
          Set GetNewConnection = oCn
      Else
         Set GetNewConnection = Nothing
      End If
     
      Set oCn = Nothing
     
End Function


Sub GetDataFromCSV_File()
   Dim objrs                  As New ADODB.Recordset
   Dim rng                    As Range
   Dim qryStr                 As String
   Dim path                   As String
   Dim csvFile                As String
  
   On Error GoTo CleanUp
  
   Application.Calculation = xlCalculationManual
   Application.ScreenUpdating = False
  
   '***********************************************************
   ' identify the PATH & FILENAME to be imported
   path = "C:\temp\"
   csvFile = "test.csv"
   '***********************************************************
  
   ' connect to the text file's folder
   Set cn = GetNewConnection(path)
  
   If cn Is Nothing Then
     MsgBox "Unable to establish connection to " & path, vbCritical, _
       "Critical Error"
     Exit Sub
   End If

   qryStr = "SELECT * FROM [" & csvFile & "]"
  
   objrs.CursorLocation = adUseClient
  
   ' ****************************************************************
   '  USE THE ADDRESS OF THE TOP LEFT CELL WHERE YOU WANT DATA
   Set rng = Range("A2")
   ' ****************************************************************
   rng.CurrentRegion.ClearContents
  
   objrs.Open qryStr, cn, adOpenStatic, adLockReadOnly, adCmdText
  
   rng.CopyFromRecordset objrs
  
CleanUp:

   If Err.Number <> 0 Then Debug.Print Err.Description
  
   On Error Resume Next
  
   objrs.Close
   Set objrs = Nothing
       
   cn.Close
   Set cn = Nothing

End Sub

peter thornton say:

Simple way is to trick Excel 2007 into showing the old drop down palette, eg

Sub ShowPopFill()
Dim cbr As CommandBar, cbp As CommandBarPopup
    DelPopFill
    Set cbr = Application.CommandBars.Add("PopFill", msoBarPopup, , True)
    Set cbp = cbr.Controls.Add(ID:=1691)

    cbr.ShowPopup ' 100, 300
    DelPopFill
End Sub

Sub DelPopFill()
    On Error Resume Next
    Application.CommandBars("PopFill").Delete
End Sub

You could add a button to the right click cell context menu to call ShowPopFill (in your Personal). Alternatively could simulate the palette and its actions with a Userform.

The Excel program or some component of it has crashed.

This shouldn't happen of course, but sometimes it does.

 

First see if it happens in Safe Mode.  Hold down the Ctrl key while launching Excel.

If it doesn't happen in Safe Mode, the problem is caused by some add-in or a workbook that is loaded on startup.  Disable all add-ins ( File > Options > Add-Ins ).  Move any files from your XLSTART folders.  Make sure File > Options > Advanced > General > At startup, open all files in: is blank.

You can then add things you need back in one at a time, restarting Excel and so discover which is causing the problem.

 

If it still happens in Safe Mode, try doing a repair from Control Panel > Programs and Features > Microsoft Office.

 

If it still happens after that,

Start button > Event Viewer (in the search box) > Windows Logs > Application > Look down the list for an Error : Application Error.
The faulting module name will give some clues as to what is crashing

two problems with m dashes used before close quotes - Microsoft Community

roxxsmom say:

two problems with m dashes used before close quotes

Say I want to write a sentence of dialog like this (I am a fiction writer).

He said, "I want to--"

Okay, now in word, this would be an m dash (not sure how to make one here in this forum). This is something you do when writing dialog (as in fiction) to indicate that the speaker was interrupted. There are two problems I've run into.

1. If you have curly quotes enabled (so-called smart quotes), the close  quote after the m dash will be written as an opening quote, not a closing quote, as it should be.

2. If the word with the m dash is at the end of a line, the close quotes will be printed on the next line without bringing the word down. It would end up looking like this.

 He said, "I want to--
"

It's clearly not recognizing the m dash as part of the previous word but as a space instead.

How can I turn this off and have it behave properly?

I'm sort of flabbergasted that this is an issue, since using m dashes in this way is correct as per Chicago manual of Style. I can't think of a situation where you'd want an m dash that is inside quotes to be treated as a space or not to register as the end of a sentence.
suzanne s. barnhill say:

Word allows a break after an em dash, which in most cases is desirable (sometimes I wish it allowed one before the dash as well); there's no way around that except to use another character as a "nonbreaking em dash." There was a thread here sometime back about this, and the conclusion was that, while the minus sign can be used as a nonbreaking en dash, there is not really a satisfactory em substitute. The workaround we came up with was two nonbreaking hyphens, condensed just enough to make them coalesce, then saved as an AutoText/AutoCorrect entry.

As for the quotes, since you know you're going to get the wrong kind, instead of proceeding and then going back to correct, insert the right kind to begin with. The keyboard shortcut for closing double quotes is Ctrl+', ". That is, press Ctrl and the apostrophe together, release, and type the " (shifted apostrophe). There are similar keyboard shortcuts for opening double quotes and opening/closing single quotes (handy when you need to force an apostrophe instead of an opening single quote). You can find these on the Special Characters tab of the Symbol dialog.

suzanne s. barnhill say:

You enter a nonbreaking hyphen with Ctrl+Shift+Hyphen. You then select the first hyphen, right-click, choose Font, select the Character Spacing tab, and choose Condensed. The default 1-point condensation will work for many text fonts at ordinary text sizes, but you can experiment to get the result you want. When you have formatted the result to your satisfaction, select the entire thing (both hyphens) and save as an AutoText or AutoCorrect entry.

Note also that it should not be necessary to have a macro to insert an ordinary em dash, as you can easily assign a keyboard shortcut (mine is Ctrl+M, though I would recommend Ctrl+Shift+M instead). I have shortcuts assigned to em and en dashes and em and en spaces.
