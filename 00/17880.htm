phillip m jones say:

Did this problem appear "after" updating system to Sierra? If so I am surprised it even works at all. The code has not been updated since version 12.3.6 probably in 2010. However, try this.

  1. First, open a blank document and, turn on Show Invisibles (¶ button). See if there are any Page or section breaks of any type or an excessive amount of Paragraph Marks (¶).

  2. Now remove all those items and then to dirty the document type a character then delete it.

  3. Now save the Document as a temporary document naming it "Test Document" to the desktop. 

  4. Quit Word, and remove "Test Document" to Trash. (This alters the Normal.dotm Template file).

  5. Now reopen Word and try creating a document. If works as should, you are done.

Back then I was not aware of Styles.  However, if you use Styles, it is possible you have altered one of your styles. Either open and edit that style, or Delete it all together.

Another way to accomplish the same thing but might take a few seconds longer. Use Lookup Utility such as Spotlight, Find File from the Finder ( File menu choose Find File), EasyFind or Tembo. Type "com.microsoft.word.plist" (without the quote marks). Each version of Office uses the same named file but placed in a slightly different location depending on what version of Office you have. Either rename  from .plist to .old. Or remove to Trash. Restart Office and the file will be rebuilt.

You can revert gently and not have to re-install Windows.  

Open Settings and choose Update and Security

Click on Windows Insider Program and then choose "Stop Insider Preview Builds."

You will get four choices.  Choose the circled one.  Whatever Insider plan your Windows is using, (there are three but it doesn't matter since opting out of Insider releases works the same for all) you will see the following.
