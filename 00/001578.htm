allen bauer say:

Final disposition... issue has been fixed... No, it wasn't "fixed" by a new build or anything; rather I took a drastic course of action. After staring at those two "offline" volumes for several months, and several new builds of Windows 10, I'd had enough.

I obtained two 1TB drives and used the Marvell PCIe card to create a new 2TB RAID 0 drive. Since the volumes could be read directly by various partition manager/copy/clone tools, I selected one of the "dead" volumes and cloned it to the newly created volume. New cloned volume appears and the file-system is fully intact. IOW, all the data was still there, just that Windows thought it was offline. New volume is now functioning with all my data.

I deleted the partition for the "offline" drive and then recreated it. As soon as it tried to format the drive, it failed and said the drive was "offline". The volume is still dead and inaccessible. Rebooted machine and entered the RAID BIOS manager.

Deleted the RAID array so the drives now appear as two separate distinct drives. Used Intel RST console from within Windows to create a new RAID volume from the drives. Went to Disk Manager and created a new partition and formatted it. Now it Worked! I repeated the above steps with the other "offline" volume and was able to recover all the data there too.

I suspect there was something about that RAID volume (which have existed since Windows Vista was on this machine) was fooling the driver or the current builds of Windows into thinking that the volumes were offline... I'm leaning toward a Windows issue because I was using the same hardware driver with the initial Windows 10 release and everything was fine. It was in subsequent builds that those volumes started appearing "offline".

How to Fix It

If you're using the new Mail experience in Outlook.com:

• On the toolbar, select the Settings icon, then select View all Outlook settings.

• Select Sync email to see your Connected accounts.

• Check the details for the connected account that's stopped working and correct the problem.

If you're using the Classic version of Outlook.com:

• On the toolbar, select the Settings icon > Mail > Connected accounts.

• Check the details for the connected account that's stopped working and correct the problem.

Note: To stop receiving these notifications, please remove or fix the connected account that's not syncing.
