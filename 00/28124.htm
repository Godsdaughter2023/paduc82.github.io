the 2 variables are from combo boxes
johnlv say: Dear Confused-

If I'm guessing your table relationships right, this query cannot be made updatable unless you remove either AreaScores or EmployeeNames.  A multi-table query is updatable only when you "walk" down a single one-to-many path.  It looks like EmployeeNames -> Valuations one-to-many.  But you also have AreaScores -> Valuations, you your relationship "tree" looks like:

EmployeeNames --> Valuations <-- AreaScores

The combination of EmployeeNames and Valuations would probably be updatable.  Ditto AreaScores and Valuations.  Technically, when you include all three tables, the query will return the Cartesian Product of all rows from EmployeeNames and AreaScores that match each Valuations record.

I see you have fields from all three tables in the SELECT clause.  Unless you can do without either the EmployeeNames or AreaScores fields (although that might be solved with a DLookup to create a non-updatable field), you're out of luck.

charles kenyon say:
You are correct, it cannot, because the header, technically is not on any page. It is in a section container surrounding one or more pages.

Generally, people put the information "Chapter 1 - Introduction" in a heading, often a Heading 1. Then a StyleRef field is used to reflect that in the Header. If you start each chapter with a new section, you can have a different first page header that does not have the StyleRef field.

If using automatic numbering in your document, you need to have it tied to styles using the steps outlined in Shauna Kelly's article: How to create numbered headings or outline numbering in Word 2007 and Word 2010

This has a couple of advantages, because a Table of Contents can't pick up information in a Header or Footer, either.

joesmith_961 say: Actually it turns out that I can reference information in the header but not in an obvious manner.

If (say) the header contained the words "Chapter 1 - Introduction", I can highlight the text within the header and press "Insert" and then "bookmark".  The trick in the next menu which I kept getting wrong is that the title of the bookmark must not contain spaces or several other "odd" characters i.e. if I called the bookmark "Chapter 1 - Intro", it wouldn't add it to the list of bookmarks.  If I called it "Chapter_1_Intro", then it would.  Then all I need to do is go into the main document, click "Insert", "Cross Reference", and then "bookmark" from the drop-down menu, and hey presto, there it is in the list...and the good thing is that it updates as other chapters are inserted.

So, thanks for the two replies - clearly not an easy MS Word function - but one that I'm glad to have worked around.
