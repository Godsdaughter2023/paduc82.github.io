MS Project 2013 Professional Planned hours changing...Help we migrate - Microsoft Community

peterbpgill say:

MS Project 2013 Professional Planned hours changing...Help we migrate on Friday!!

How do I stop the planned hours getting changed by actual hours exceeding them or clearing remaining work

Example.  A task with Planned(Work) 10 hours, If actual is 5, and I mark it 100%  the actual changes to 10 even though it has only taken 5.  If I change the remaining work to 0 it changes the planned to 5.  If I change the actual to 15 the planned changes to 15.

Can I stop it changing the plan??

Base lining doesn't help, as this just cements in the changes

rod gill say: Peter, what you are describing is desired behaviour. have you set a project baseline? Baseline hours are the hours that remain unchanged. Once a task is 100% complete, work and actual work are always the same.

trevor rabey say:

I think that the answer to your question is no, you can't. I am sure you won't like the answer because you think you should be able to, but the reason you can't is because that is not the way the software is designed to be used, and because it is subject to some very simple rules.

Consider the actual start date. A task has either a scheduled start date or an actual start date. It can't have both. Once it has an actual start date, its previously scheduled start date is irrelevant, and is superseded by the actual start date. Actuals trump scheduled. The only way that the previously scheduled start date can be available or visible is if it has been retained in the baseline (if there is one), or if it has been retained, because it could turn out to be of some interest, in a previous version of the file (if there is one).

Pretty basic concept, this one:

Work = Actual + Remaining

If you change the work, it must change the remaining, because actuals are actuals and don't change, ever.

If you change remaining, it must change work, again because actuals don't change.
