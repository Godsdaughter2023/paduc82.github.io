Here is a way to make sure your range is always in view using the following VBA event code. The way it works is the user can select the top pane and scroll it wherever they want, but as soon as they select any cell, the top pane will automatically scroll so that cell C1 is the top left cell in that pane. Here is the code to do that... 

 

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
  With ActiveWindow
    .ScrollColumn = 3
    .ScrollRow = 1
  End With
End Sub

 

HOW TO INSTALL Event Code
------------------------------------
If you are new to event code procedures, they are easy to install. To install it, right-click the name tab at the bottom of the worksheet that is to have the functionality to be provided by the event code and select "View Code" from the popup menu that appears. This will open up the code window for that worksheet. Copy/Paste the event code into that code window. That's it... the code will now operate automatically when its particular event procedure is raised by an action you take on the worksheet itself.

Exchange online archiving. Selecting what to archive. - Microsoft Community

fazerana say:

Exchange online archiving. Selecting what to archive.

I was wondering if there was a way to only have users email archived and not their tasks. We use tasks a lot, but do not want it to be archived online.

vanessa ouyang msft say:

Hi FazeRana,

The Tasks folder inherits the default archive policy for the mailbox which cannot be changed and overridden. To achieve your goal, I suggest that you set a default archiving policy “Never archive” and then override this for the folders that you want to archive. Here are the detailed steps:

     a.  Use an admin account to log into Exchange Admin Center;

     b.  Create a retention tag > select “applied automatically to the entire mailbox(default)”> Move to Archive> Never. This tag is for Task;

        c.  Create another retention tag > select “applied by users to items and folders(Personal)”> Move to Archive when the item reaches the following age;

       d.  Come to the Retention policies page and edit Default MRM Policy > add the tags created above to this policy;

In this way, this retention policy means items in the whole mailbox are by default not archived and emails in specific folders are moved to archive.

To assign this policy to all users, you can run PowerShell below:

Get-Mailbox | Set-mailbox -RetentionPolicy “Retention Policy name”.

To check whether it works, you can run: Get-Mailbox -ResultSize Unlimited | select Alias, RetentionPolicy.

Before that, you need to connect to Exchange Online with PowerShell referring to the article: Connect to Exchange Online PowerShell.

Best Regards,

Vanessa

asheryl,


Not sure what settings you are talking about, but since it appears that Chrome uses the same IAttachmentExecute Windows Shell interface for downloading files that Internet Explorer does, scanning of file downloads should be fully protected.


However, since the anti-phishing and other protections for a browser are typically operated as plug-ins, I'm not as certain how effective the SmartScreen Filter used with internet Explorer will be at protecting Chrome, though in Windows 8.x this is supposed to provide protection for all of Windows.


Other browser specific protections such as anti-script are also usually operated as plug-ins, so you may still need to add these separately in order to fully protect Chrome running on Windows.  Since as the various FAQs and other resources here indicate the features in Windows Defender are really tuned to be used with the various Microsoft IE browser and other applications, it's usually best to go to the support forums for the browser you wish to use and ask these questions there.  The people using these 3rd-party products on Windows systems are really far more likely to be able to answer such application specific questions.

bob jones aka: cybertaz say:
Not clear on this description: When selecting a cell within a row, the row number column is also "highlighted."

Do you mean just the Heading of the Row in which the selected cell is located? If so, that's perfectly normal -- The Row Heading & the Column Heading for the selected cell highlight to provide a clearer indication of the cell reference.

If you mean the entire gray column of Row Heading numbers [1-1,048,576] highlights when a single cell is selected, there is a problem with the sheet or the workbook. I've never seen that happen, so it will take a bit more exploration to determine what's going on. You may be dealing with a corrupt sheet. Please provide as much information about the origin, design & structure of the file as possible.

As for your first issue, it depends on whether you want to Clear the content/formatting/both of the Row (everything else stays where it is) or whether you want to Delete the Row (everything below it shifts up to fill the gap). Either way, start by clicking the Row Heading for the Row, then:

  • To Clear the content, the formatting or both, go to Edit> Clear to select whatever is preferred. Alternatively, use the Clear button in the Edit group on the Home tab. The keystroke for Clear All is the del key if you have a full-size keyboard. If you have a condensed keyboard [e.g., laptop] use the fn+delete key combination. (There is no built-in keystroke for Clear Contents or Clear Formatting.)
  • To Delete the Row go to Edit> Delete. The general built-in keystroke you can use for Deleting selected Rows, Columns or Cells is Control+-  or you can use the Delete button in the Cells group on the Home tab.
