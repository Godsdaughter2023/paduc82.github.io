Enter these:

 

Cell A1    "X Values"

Cell B1    "Y Values"

john - project say:

Sharon,

You mean like this?

There are basically three reasons why the dependency is not acting the way you expect. My first guess is because the Project Start date of the master is on a Monday whereas the Project Start date of the subproject is Wednesday. To avoid this apparent discrepancy, both master and subproject should have the same Project Start date and use the same project calendar.

If that's not the case (i.e. the Project Start date for both the master and subproject are the same), my next guess is that the task in the master either has a constraint date (e.g. Start-no-earlier-than), or has a date in the Actual Start field, either of which can lock it to the Monday start regardless of the predecessor.

A caution about using linked structures in Project. You must never rename, move, overwrite, or save off any of the files in the structure. Doing so will sow the seed for corruption. Also, all files should reside in the same directory on a local drive and not be operated over a network.
