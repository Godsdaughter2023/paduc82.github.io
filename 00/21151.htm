a. user say:

If you are merging to email, any page breaks you insert will disappear if you merge to plain text or HTML format - they will only be retained if you merge as an attachment.

If you are merging as an attachment, a new document, or to a printer, try the following instead:

{ IF CodeField <> "R" "" "<page break>All of the stuff on the second page" }

where you insert an actual page break where I have put  <page break>.


Notice that there are spaces around "<>" and that I have removed the two "," characters.

For example, in my test here I had

{ IF { MERGEFIELD myfield } <> "R" "" "<page break>All of the stuff on the second page" }

where all the {  } are the special field code braces that you can insert using cmd-F9 (or fn-cmd-F9, depending on your keyboard settings) on Mac.

For this issue, I suggest you first check whether the problematic users are listed in the member of the public folder.

If the users are listed in the list, try to re-add a problematic user and check if the issue persists.

 

Meanwhile, you can run the following PowerShell command to determine whether the public folder was deleted by someone:

Connect to Exchange Online PowerShell

Get-Mailbox -PublicFolder -SoftDeletedMailbox

In Outlook Press Ctrl + 6 > navigate to your public folder and select the shared calendar > right click on the shared calendar > permission

russ valentine say:

Well here are some general observations about what you've posted so far that might help:

1. The error message you are getting is not one you would get if you were opening a PST file from within a functioning Outlook profile. It is an error message you would get at launch from an Outlook profile that has not been configured correctly or that is corrupt. Accordingly, how you configured this profile and connected it to the PST file you want to use are very much in play.

2. Any error messages you get when opening your previous PST file _after_ you have opened a functioning Outlook profile (File > Open > Outlook Data File...) would be highly relevant.

3. If you discover that you created a corrupt Outlook profile by migrating your data improperly, your only option will be to create a new Outlook profile from scratch.http://support.microsoft.com/default.aspx?scid=kb;en-us;829918&Product=out2003  If you do, take care to migrate your data correctly using the guidelines we post here daily.
