SOLVED

Once I determined it was the Windows Driver Foundation (wudfhost.exe), I figured it had to be a driver leaking memory or behaving poorly.  It ended up being the "HID Sensor Collection V2" driver.  This driver apparently causes many issues in Windows 10 with tablets or laptops (Surface, Yoga, Twist) which sense their orientation to rotate the screen.

1)  Run the Device Manager.

2)  Right mouse click the HID Sensor Collection V2 driver under Sensors.

3)  Select Disable.

4)  Reboot.

You can still rotate the screen by going into Settings -> System -> Display -> Orientation or right mouse clicking your Desktop and selecting Display settings and selecting Orientation.
