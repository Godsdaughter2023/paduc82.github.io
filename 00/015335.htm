jeanette cunningham say:

Yes, you need to do a requery or sometimes a refresh to make access show the updated info that you have changed by code or entering info in a different form.

Just entering info into a form doesn't make access save the data just entered unless you force the save by moving to another record or closing the form. This is what happens when a user enters info into a form and then expects to see the updated data show in another form that was already open. If you close and reopen that form that was already open, it will then show the data entered in the other form (assuming that we are talking about a form that uses data entered in a different table form another form).

Closing and reopening the form, doing a requery or refresh makes access go back to the tables feeding the form and grab the latest data in the table.

You might be confusing the Modern (or 'Metro,' from the Start screen) version with the desktop version.  They're both IE 10 in Windows 8 and they share Favorites.  The Modern version doesn't show folders, just the long list.  Here's what I do to get to a Favorite:

 

Go to the Frequent and Favorites bar at the bottom.  All you have to do is start typing any part of your Favorite's name and you'll get a list of options to click on.  More letters make the list more specific.

 

The desktop version of IE 10 does allow folders, and any favorites you make there show up in the Modern version list.  That's also where you have access to Internet Options.
