Follow the steps mentioned below on how to enable the ‘Hidden Administrator account’ in windows 8 system:

1. Login to your computer with any account that has administrative privileges.

2. From the available options displayed on the screen click on Desktop.

3. Press Windows + R keys simultaneously in order to open the Run command box and in the available field type GPEDIT.MSC command and hit the enter key

4. On Local Group Policy Editor snap-in under computer Configuration from the left pane expand Windows Settings.

5. Expand the option Security Settings and then expand Local Policies.

6. From there, click on Security Options and from the right pane double-click on Accounts: Administrator account status option.

7. On the opened box click on Enabled radio button to select and click on Ok button to save changes.

8. Close Local Group Policy Editor snap-in and restart your computer to allow the changes to take effect.

I tried this process because the hard reset was not a favorable option, and it was the fix!

 

The steps are:

  1. Open Registry Editor (Click on Start, Click Run, type regedt32, click OK)
  2. Navigate to HKEYLOCALMACHINE\Software\Microsoft\Windows CE Services
  3. In the Windows CE Services folder, find the REG_DWORD value called GuestOnly
  4. Right click, click Modify and Set its value to 0 (zero)

how to maintain the correct color when importing a graphic - Microsoft Community

terencejl say:

how to maintain the correct color when importing a graphic

I imported a graphic created in Adobe Illustrator which was saved a TIFF. One element of the logotype is a spot color, PMS 5487. This color, however, changed when the graphic was placed in MSWord. Can someone tell me how to maintain the correct color?

Thank you.
bob jones aka: cybertaz say:
Hi John -

My understanding is the same as yours... I don't know it to be a fact but I believe The CMYK values are stored if -- and in today's world it's a big IF :-) -- the image is placed using the Insert menu method. Neither Copy/Paste nor dragon-drop are able to do so, AFAIK.

Even so, it's a matter of the Output device being able to produce the CMYK color. Most Local printers do not use CMYK because there is no 4 Color Separation. Even if CMYK values are sent to the printer its algorithms convert them. Spot color is no exception... if the color specified is outside the RGB gamut it cannot be reproduced accurately. Both for display as well as print the device comes as close as it can.

john mcghie say:
Hi Terence:

I am not at all good with colour.  On my computer PMS 5487 looks the same displayed in either Safari or Word or Chrome.  As it would, because I have no colour profiles applied to any device or application on this computer.

Bob is quite correct, Microsoft Office does everything in RGB at 24 bits depth.

However, I believe (someone correct me if I am wrong...) that Word stores the original image in the document, and sends that, instead of the RGB version, to the printer.  So it might be worth printing the image to see what you get.

For logotypes I always use a vector format: EPS, EMF or WMF.  EPS works best in Mac Word 2011, followed by WMF.  EMF works fine, but the on-screen display comes from a 72 dpi header thumbnail, which looks fairly disgusting.

Using a vector means that nothing is likely to "fiddle" with the colours on the way through -- especially if you use EPS where the original image is stored encapsulated in the document.

I think it's safe enough to send EPS to PCs these days: it used to be that you have to send WMF to be sure of getting printed on a PC, but I think they've grown up...

Oooh...  Make sure it "is" an EPS if you are going to use one.  I know that .AI is allegedly the same thing, but Word doesn't think so, and it probably won't work.
