joeu2004 say:

First, 39:45 is not 39.45 hours.  It is 39 hr 45 min, which is 39.75 hr.

Second, AD3*10.65 did not come out $423.20.  It came out 423:20, which is 423 hr 20 min.

But it appears that way only because Excel chose the time format [h]:mm because that is the format in A1.

If you change the format of column AE ("Gross Wage") to General, you will see 17.6390625 or some rounded form of it.

Third, the correct calculation is =24*AD3*10.65, which is some form of 423.3375 when you choose an appropriate format:  General, Number, Currency or Accounting.

Finally, note that the wages are 423.34 (rounded), not 420.14.

Similarly, 17:30 is not 17.30 hours.  It is 17 hrs 30 min, which is 17.5 hr.

The correct formula is =24*AD4*10.65.  And the result is some form of 186.375, not 184.25, when you choose an appropriate format:  General, Number, Currency or Accounting.

greginmich say:

Well… in the current version of Windows 10, Windows Defender should be completely disabled when you install a third-party AV app – but in many cases, Defender hasn’t been getting properly disabled in its registry key, and it's actually being put in a state of limbo (half-on/half-off) when another AV app is installed. When this happens, Defender is turned-on to the point where it shows its face (UI) when you try to start it, but turned-off to the point where the service won’t start (Error 8007139f) – and it’s turned-on to the point where it asks for definition updates, but turned-off to the point where it can’t install them (Error 0x80070643 in Windows Update). And its service executable also can’t be verified (Error 577 in services.msc).

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_updating/definition-update-for-windows-defender-kb2267602/ea933090-9301-4c27-b915-1d3ba92b301f

Microsoft has been made aware of the situation through this forum, and although there hasn’t been any indication of progress with this issue, it looks like the Anniversary Edition of Windows 10, which is scheduled for deployment starting August 2, will incorporate a mechanism that will resolve this issue. The new version of Windows Defender will never be completely turned off when you install a third-party AV app, and it will allow you to turn on its Limited Periodic Scanning feature in the UI with a third-party AV app installed. This new feature will allow you to run any kind of on-demand scan in the Defender UI; and it will also enable the Automatic Maintenance scan, as well as the Command Line Utility for scheduling and scripting Defender scans and updates. It just won't let you turn on Defender's real-time protection, because that's the component that tends to interact adversely with third-party AV apps:

http://answers.microsoft.com/en-us/protect/wiki/protect_defender-protect_scanning/limited-periodic-scanning-in-windows-10-to-provide/3dcfc8dd-36b4-43f2-9a0a-732e43191cf3

So with a little luck, the new edition of Windows Defender will both allow you to use Defender along with your third-party AV app, and also resolve any potential issue with it being trapped in a state of limbo. Since the interaction between Windows Defender and third-party AV apps will be changing substantially in just a week or two now, some folks might actually want to be thinking ahead when they’re "mulling their options".

GreginMich
