Work Pager --> Pager.

dan bielecki say:

The problem is not your Lenovo.  

The issue began about 2 updates ago from Windows Update.

Its a Permissions and Authority issue and internal to Windows.

Everyone always says its related to some 3rd party anti virus package, but even on machines where only Defender is running the issue occurs.   There is an temporary fix you can do to get Defender back up and running but at each reboot it will occur again, since it is the SYSTEM Permissions modifying the security on the Registry Values and setting the Defender Registry Keys to "1" (meaning a 3rd party package is installed even if there isnt one)

The temporary solution is this.

Open RegEdit with Administrator permissions (Run as Administrator box checked as YES)

Locate the Key:

HKEY_LOCAL_MACHINE/SOFTWARE/Microsoft/Windows Defender

Set the DisableAntiSpyware key to 0x00000000   (zero hex)

Set the DisableAntiVirus key to 0x00000000 (sero hex)

close RegEdit

Run SERVICES MMC from Administrator Tools (a Control Panel icon)

Locate and start the the two Windows Defender Services named:

WINDOWS DEFENDER SERVICE

WINDOWS DEFENDER NETWORK INSPECTION SERVICE 

Open Windows Defender and the if the Start/Stop box is red, (not started) then click it and start Defender.

*****************

There was also a Permissions and Authority issue right after the first Windows Update (3 updates ago) that caused this, some people experience total lock out from even the above steps, and others do not.   I did and had to set the \Programs Files\Windows Defender\ folder and all its subfiles and folders to allow Administrators to access the objects.   The issue was in the use of only "Administrator" (no S) to allow access to the folder and content.   I dont have an Administrator profile,   It was a profile from older Windows version.   Newer versions use Group Authority and the Administrators (with the S) is a Group Profile meaning anyone that is an Administrator Level user, not the older User Level Administrator profile that may not be present anymore.   That profile would be present on machines where an Upgrade was done to Win 10, but on a new machine where Win 10 is pre-installed, the Group Authority of administrators is use, of which the first person to ever to log in to the box when first powered up, is the administrator, not the ADMINISTRATOR profile of old.

So if you have a new Lenovo, you may have that same admin/admins issue too.   To resolve, change the Ownership of the folder to ADMINISTRATORS group.  This change would be a one time change.   The other temporary solution has to be repeated after each reboot, because the SYSTEM authority is resetting the Registrar Keys to 0x00000001 (hex 1) each reboot under the assumption that the box is under control of a 3rd Party virus scanner.

I only use Windows Defender, I have no 3rd Party application for that yet Windows will default it after each reboot because most users do use something other then the Microsoft Product.   If the Keys are ON...(set to 1) Windows Defender will not start the Services.  

After about short period of time based on Startup tasks loading and running, Security Center will fire up and start reporting and popping up message boxes that there is no Virus Protection running.   There isnt.   The box has no 3rd Party one and Defender cant start either due to the Keys saying one is installed.

Microsoft will have to correct their flaw in how they default machine key values.  So expect a Windows Update to resolve it in entirety.   They cant assume a user is using a 3rd Party package when the user is only using Defender.  The admin/admins group vs user profile needs to be addressed and corrected also.  There are was to know if a box is an update box or a preinstall box, they need to enhance the logic in that area to determine the default value for the Defender RegKeys because assuming "1" isnt cutting it.

I know what the issue is and how to resolve so I jsut do the temp solution after each weekly reboot until MS gets around to figuring out its actually their issue and not a user one involving 3rd party virus protection.   The keys related to the issue are pointing everyone in that path, but the issue is actually determining and setting default values and object authority of folders and keys.   That was introduced in a Win Update package 3 updates ago.  I wasnt using automatic updates so I updated manually.  I know it started 3 updates ago.
