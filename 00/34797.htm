Transferring files between drives: data loss or corruption? - Microsoft Community

hover cat dave say:

Transferring files between drives: data loss or corruption?

  1. Is there data loss or corruption while transferring files between drives in Windows 7?
  2. What about for very large files, gigabytes in size?
  3. Is there some way to check whether there is data loss or corruption?
lemp say:

Creating and verifying a checksum is a file operation.  It doesn't matter where the files are stored.

It doesn't matter whether the source is a CD, a USB flash drive, or a hard drive.  As far as the destination is concerned, if you're writing a large file to a CD, the chance of an error is greater than if copying to a USB flash drive or an external hard drive.  That's why you often see the advice to burn at the slowest speed available when writing large critical files such as an installation ISO file.

The only difference between "copy" and "move" is that move copies the file and automatically deletes the original.

You want to make sure that the file copied correctly before the original is gone.  The way to ensure that is to do a copy operation, verify the integrity of the copied file, and then delete the original.
