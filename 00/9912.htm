stephen boots say:

You cannot disable the restore point creation. System Restore will manage the space used and you can limit the percentage of the drive used by System Restore in the System Restore settings.

You cannot delete the updates. They are downloaded and merged into the database, then deleted. On the refresh, which usually happens monthly or so, the new database and engine is installed and the old is removed. MSE manages this process for you.

-steve

~ Microsoft MVP Windows Live ~ Windows Live OneCare| Live Mesh|MS Security Essentials Forums Moderator ~

rob koch say:

It's not MSE itself that's doing this, it's actually the Windows Update services that are used to perform the MSE updates and is a completely normal process of Windows.  Unless your PC hard drive is under about 15% available space, this shouldn't create any problems, since this is the recommended minimum available working space for the entire Windows NT based OS family.

To better understand how Windows manages System Restore disk space, see the following, though it mentions Windows 7 the process is basically the same on all currently supported versions.

http://windows.microsoft.com/en-US/windows7/How-much-disk-space-does-System-Restore-require

How the System Restore Tool Handles Hard-Disk Space Usage (Win XP) - http://support.microsoft.com/kb/300044

If you want to resolve the loss of older restore points, you should simply increase the percentage of disk alloted to System Restore.  If you're running out of disk space, delete some old programs or buy a bigger or second drive.  Obsessing over these things is counterproductive, let the PC do it for you, that's what it's for.

braintrove.com say:

I don't have IP2010 installed, but for IP2013, you do the following:

  1. Click the File tab.
  2. On the Info page, click Form Options.
  3. In the Form Options dialog box, select the Compatibility category.
  4. Choose InfoPath Filler Form from the Form Type drop-down.

Now, if you are already doing this and the drop-down is disabled, you can manually switch to it by editing the manifest.xsf file. First extract your form files:

  1. File tab.
  2. Public page.
  3. Export Source Files.
  4. Edit the manfiest.xsf file.
  5. Search for: <xsf2:solutionDefinition runtimeCompatibility="client server">
  6. Change to: <xsf2:solutionDefinition runtimeCompatibility="client">
  7. Save and exit.
  8. Right-click the manfiest.xsf file and choose Design.
  9. Save as an XSN again.

julie sheets say:

You can create a "top down" budget using both budget work resources and/or budget cost resources.

Create the budget resources (one cost, one work) and open the Resource Information dialog for each resource. (Double click on the resource name)  Click the Budget check box.

Assign the Budget Cost and/or Budget Work resources to the Project summary task.

In the Task Usage view, add the Budget Work and Budget Cost fields to the table (left side) to enter the values for each budget.

You can then view the Budget Cost and Budget Work reports (Visual reports) to compare Budget, Baseline, Actual data.  Having a budget will not prevent your from going over any budget, by the way - they are comparisons.
