greginmich say:

We usually see this issue with PCs that have a third-party AV app installed. Lots of people initially jumped to the conclusion that this was a Windows Update issue – but it turns out that in cases where another AV app has been installed, the Defender definition updates only seem to fail in cases where Windows Defender has been put into a state of limbo (an indeterminate state) because the third-party antivirus app is installed and registered with the Security Center, but Defender is still switched on in its registry key. So the definition updates are getting requested, but they can’t be installed because the Windows Defender Service can’t be started (“The group or resource is not in the correct state to perform the requested operation”).

That being said, Error 0x80070643 is a generic error code that will appear for any unknown error state – so people who don’t have a third-party AV app installed might get this code for an unrelated issue:

https://www.microsoft.com/en-us/security/portal/mmpc/help/errorcodes.aspx

In that scenario, your issue might be fixable with the Update Troubleshooter:

http://windows.microsoft.com/en-us/windows-10/windows-update-troubleshooter

The issue with the faulty entries in the Windows Defender registry key can be resolved by any method that completely disables Windows Defender: Repeating the installation of the third-party AV app should do the trick, but this sometimes fails for Norton users, and some other folks might not want to take the time to recycle the installation. So the quick fix is to just turn Defender off with a registry edit.

Since the entries that are responsible for the issue are in the Windows Defender registry key, which is owned by the System, we use Group Policy to override the faulty entries in the Windows Defender key. This can be done via the Group Policy Editor in the Professional edition of Windows 10, or with the Ten Forums Turn_Off_Windows_Defender.reg file in the Home edition. Both of these methods have been reported to resolve the definition update issue in other threads:

http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

But for Norton users, at least, it now appears that the relevant registry entries in the Windows Defender key can actually be toggled indirectly by just disabling Norton; turning on Windows Defender; and then turning Norton back on while Windows Defender is still turned on. This apparently works by mimicking the state changes involved in a reinstall:

 http://answers.microsoft.com/en-us/windows/forum/windows_10-update/definition-update-for-windows-defender-kb2267602/d4edd314-a438-4ac2-b7a9-dc1c45a1ef62
