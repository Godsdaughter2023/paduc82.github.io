Hi Luis,

Panning a bit in the Internet I found two possible ways to get the W8.1 Update 2 which is prerequisite for upgrading to W10. Once having W8.1 Update2 installed there should be no further problem to upgrade to W10 by deploying Windows Insiders.

Here is the first method:

Step-1: Go to Settings > Phone update. Uncheck the Automatically download update if my data setting allow option

Step-2: Download and sign up for the Windows Insider Program on your phone to get the 8.1 Update 2. Set it to Fast Track option

Step-3 Go to Settings > Phone update and you’ll see that there is an update available. Verify if it is the W8.1 Update2 / OS 8.10.14.219.341 which is the lowest for later W10 upgrade.

Step-4: Download the update

Step-5: Once the download starts, head over to the app list and uninstall Windows Insiders to prevent Windows Phone 10 update

And there is a second method:

Install Windows Device Recovery Tool which will reinstall your W8.1 so that you get the latest version, i.e. W8.1 Update2.

 

In my own case I followed the first method up to Step 3 and -voilá- the OS 8.10.14.219.341 is now available for download!!. As I am right now in the backlands without a decent Internet connection I did not download the update, but I do not see why it would not work.

Regarding the second way I got a feedback from a colleague stuck with the same problem and he told me the method deploying Windows Device Recovery Tool worked fine for him.

I hope this helps.

Hello

A pleasure to help you.

My recommendation is the following.
Run from the Power Shell module on the server where the synchronization tool is the next command to force the change of all passwords.

Important before executing the command update the names of the connectors.

$ adConnector = "name_conector.local"
$ aadConnector = "name_nube.onmicrosoft.com - AAD"
Import-Module adsync
$ c = Get-ADSyncConnector -Name $ adConnector
$ p = New-Object Microsoft.IdentityManagement.PowerShell.ObjectModel.ConfigurationParameter "Microsoft.Synchronize.ForceFullPasswordSync", String, ConnectorGlobal, $ null, $ null, $ null
$ p.Value = 1
$ c.GlobalParameters.Remove ($ p.Name)
$ c.GlobalParameters.Add ($ p)
$ c = Add-ADSyncConnector -Connector $ c
Set-ADSyncAADPasswordSyncConfiguration -SourceConnector $ adConnector -TargetConnector $ aadConnector -Enable $ false
Set-ADSyncAADPasswordSyncConfiguration -SourceConnector $ adConnector -TargetConnector $ aadConnector -Enable $ true


If this does not solve you, I suggest you follow the steps indicated in the following article specifically for errors in the credentials sync.
