For the question1 and 3, you can refer to Use sharing auditing in the Office 365 audit log. This article describes the external sharing process and the audit log admin can use to monitor the sharing history. However, notice that the audit log can only monitor the logs during a range of time that no more than 90 days. And there’s no way to get all links, but only can get the acting users/target users/shared items/date and time.

For the “Access Request Option” settings, this only works when the permission hasn’t assigned to the users (please check the Use sharing auditing in the Office 365 audit log and see the sharing process) and the target user is in the existing list in the SharePoint Online directory. You need to export the contacts as .vcf files to your memory card. Make sure they're all there because filename conflicts (there's one file per contact) will result in contacts being overwritten. Then delete all your contacts, re-import them from the memory card and then you'll be able to set the defaults.

The downside to this is that you'll lose the default numbers on all your contacts and you'll have to redo your speed dial settings and re-populate your contact groups.

Alternatively, you can save yourself a certain amount of trouble short-term by exporting, deleting and re-importing just the contact you're trying to edit with a default.
