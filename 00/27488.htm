bearishboy say: I had this same quandry.

After I had missed a very important text message because I did not hear the notification, I went searching to a solution for the lack of "repeat".  This is an option that every other phone I have had included, and other event types were capable of, so I knew their had to be a way to enable it for SMS messages.

After doing a little digging, I found this key in the registry:
  • HKEY_CURRENT_USER\ControlPanel\Notifications\
This key included a number of subkeys, each using a guid or guid-like identifier.  Each of these subkeys has a default value which is set to the text description of the notification type. I found one for a notification type that allowed repeat (reminder), and compared it to the one for New Text Messages, and the only real difference was the "Options" Value.

New Text Message Notification options value = 9
Reminder Notification options value = 1073741849

I changed the Options value from 9 to 1073741849 and voila, repeat is now an available option, and works.

Now just to be clear, I have no idea what all is controlled by the Options value, and do not have a breakdown of what the numbers mean.  My assumption is that it is a bit field.  The second number has 2 bits set differently that the first

0000000000000000000000000001001b  = 9
1000000000000000000000000011001b = 1073741849

If you are so inclined, you can play around and possibly figure out what means what.  I have been unable to find reference information for this, though, and haven't concerned myself since copying the value solved my problem.

Grab PHM Regedit and try it.

As a caveat, always always always make a backup before making changes.  While this worked fine for me on my Windows Mobile device, I can't guarentee it won't make yours send a signal to the battery to catch fire and signal a space cannon to target and fire upon your location.

 

It worked great for me, though.  I have also done this to the phones of several friends, and they have been pleased with it.

Good luck.

dale howard [mvp] say:

Matt --

No, there is no way to force Microsoft Project to create Friday or Monday exceptions automatically when a holiday occurs on a Saturday or Sunday.  This is something you must do manually.  The recurring holiday feature works great for holidays like Labor Day, which always occur on a Monday, but it does not work as well on holidays that occur on a specific day, such as Christmas Day which always occurs on December 25, and which can occur on a weekend, therefore.  So, if the Christmas Day holiday occurs on a Saturday, you would need to set up an additional nonworking time instance for that situation.  Hope this helps.

DAO and ADO are not incompatible, but if you have both references set, you have to explicitly reference objects, particularly recordset objects. Recordset objects are critical because both ADO and DAO have a recordset object, but they *are* incompatible and Access can get confused about which you mean. 

By explicitly referencing the appropriate model, you remove that confusion.

Dim rs as ADODB.Recordset
