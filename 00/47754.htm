Hi,

 

You could try these steps :

1. Click Start, and then click Run.
2. In the Run box, type regedit, and then click OK.
3. Locate the following registry key:
HKEY_Current_User\Software\Microsoft\Windows\CurrentVersion\Policies\Explor er
4. In the right pane of the Registry Editor window, locate the NoDriveTypeAutoRun entry.
5. Edit the entry to display the following correct binary value in the Value data box:
NoDriveTypeAutoRun 0000 95 00 00 00
To do this, right-click NoDriveTypeAutoRun, and then click Modify. Under Value data, type 95 00 00 00, and then click OK.
6. On the Registry menu, click Exit to quit Registry Editor.
7. Restart your computer.

hairyfool say:

You need a license for each installation of Windows not each partition.

If you have two partitions but only one of them has Windows on it you only need one license.

If however you should install a second copy of Windows on the second partition such as you can choose which copy of Windows to boot from (Multiboot) this then would require a second Windows license.

Modern UEFI based drives can have 6 partitions on them by default for a factory build but because only one version can run it only requires one license. Even when one of those partitions contain a backup copy of Windows to do a factory restore as is often the case because that copy will not boot independently only one license is still required. The Windows EULA allows a backup copy to be made.

hairyfool say:

No the license is valid for the computer "for the lifetime of the device". When we talk about installation we are referring to the completed installed image not the process of running setup.

You can reinstall Windows theoretically as many times as you wish on the same computer using the same license and activation will work. In practice excessive reinstalls flags up possible inappropriate activity and automatic activation fails but normally once you call Microsoft Activation they can manually authorise the installation.

The License allows a single copy installed at any one time which is where multiboot  or creating virtual machines using the same license is precluded. Whereas setup can be run multiple times you can only have one authorised copy able to be used. Some have tried to argue that with multiboot only one copy can be used at any one time but the premise is that there is another installed image available and that must have its own license

If you want to use multiboot on a separate partition or build a virtual machine you just buy another license for the product you want to install. There is nothing specific about the license you need buy so whatever purchase options you have that is specific to your region that's what you use.

hairyfool say:

Although Windows itself allows specific customisations on a per user basis how individual programs handle that will depend on the program developer. You can have as many user logins as you wish

To get technical for a moment the registry stores settings for programs but can be in the Local Machine hive or the Current User hive. The former affects the whole computer, the latter gives per user settings. There is normally no way to change how the application works

Some programs will allow Advanced/Custom installs during setup including a "for this user/for everyone" option but not all do. So having different programs available for different users/logins can be done but isn't straightforward.

Similarly configuring application preferences on a per login basis is entirely down to the application itself, some do not even recognise the fact that there can be a different profile, every user gets the same one.

So in order to be able to say that each user can definitely have their own setup they would need their own computer set up how they want it. That then leads to Multiboot when it's a single actual person needing two profiles.
