joel ivory johnson say:

I've got the explanation.

Version numbers reported by the Marketplace don't line up with version numbers reported by an application.  This can be said of applications both from Microsoft and from other parties.  When an application is being added into the marketplace the version numbering supported on an upload only allows for a major and minor version number (such as 2.3). However, a developer may have a version numbering system with more "points" than that. Many developers may use version umbers composed of 4 sections (such as 2.3.1233.332).

Let's say I make a new application an upload it to the marketplace and call the first version 1.0.0.0 inside the application but 1.0 in the marketplace.  Later on I decide to correct a minor spelling mistake within the application.  For such a minor change I may not want to increment the major or minor version number. Instead I might want to call this version 1.0.0.1. When uploading to the Marketplace since the version numbers only include one digit after the decimal point I would call this version 1.1 there.  If I made some other minor update I might make the internal version number 1.0.1.0 and on the marketplace this would be 1.2 and so on...

So as you can see the Marketplace version numbers and the internal version numbers don't have to be related.

song.usa say:

Rule to delete forever, bypass 'deleted item'

Deleted item folder count against my quota. I want to setup a rule based on word in subject or sender and delete forever (bypass deleted item). If it's not possible, what's the option? I have 30,000 messages to delete.
graham mayor say:

You could I guess use a macro - something along the lines of the following. However, be aware that if you make a mistake with your search string, or that search string is contained in messages you wish to retain, they will not be recoverable - so make sure you have a backup of your Outlook data file BEFORE running the macro.

There is no hard delete function in Outlook VBA, so the method I have selected is to delete any message in any mail folder that contains the indicated string in the message subject and then delete the same message from the deleted items folder. Previously deleted items shouldn't be affected. It will probably take a while to run with 30,000 messages to delete!

It would not be too difficult to add a further test against the message sender,

Sub DeleteMessages()
Dim olItems As Outlook.Items
Dim olItem As Outlook.MailItem
Dim objItem As Outlook.MailItem
Dim i As Long, j As Long
Dim iFolder As Folder
Dim subFolder As Folder
Dim strText As String

strText = "Text to find" 'in the subject line

On Error Resume Next
Set objDeletedFolder = Application.Session.GetDefaultFolder(olFolderDeletedItems)
For i = Application.Session.GetDefaultFolder(olFolderInbox).Folders.count To 1 Step -1
    Set iFolder = Application.Session.GetDefaultFolder(olFolderInbox).Folders(i)
    For Each olItem In iFolder.Items
    If InStr(1, olItem.Subject, strText) Then
        'MsgBox iFolder.Name & vbCr & _
        olItem.Subject
        olItem.Delete
        Set objItem = objDeletedFolder.Items(1)
        objItem.Delete
    End If
    Next olItem
    For j = iFolder.Folders.count To 1 Step -1
        Set subFolder = iFolder.Folders(j)
        For Each olItem In subFolder.Items
            If InStr(1, olItem.Subject, strText) Then
                'MsgBox subFolder.Name & vbCr & olItem.Subject
                olItem.Delete
