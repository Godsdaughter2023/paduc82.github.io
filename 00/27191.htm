jay freedman say:
  • Create the form as a macro-enabled document or template.
  • Open the macro editor (Alt+F11).
  • Select the name of the form/document in the Project pane. Click Insert > Module.
  • Click Tools > References. In the list of available items, locate "Microsoft Forms 2.0 Object Library" and check the box next to it. Click OK.
  • Paste in the code from the bottom of this post.
  • Change the value of the constant named pwd to the password you use for protecting the form.
  • Click Tools > Project Properties, go to the Protection tab of the dialog, check the box to lock the project, and enter a password that will be needed in order to view or edit the macro. That prevents unauthorized people from seeing the form password in the code.
  • Somewhere in the body of the form, insert a field (using Ctrl+F9 to insert the brackets) with the field code  {MACROBUTTON CopyForm Click here to copy the form} .
                                     Dim MyData As DataObject
    Const pwd As String = "foo"

    On Error GoTo ErrHdl
    With ActiveDocument
        If .ProtectionType <> wdNoProtection Then
            .Unprotect Password:=pwd
        End If

        Set MyData = New DataObject
        MyData.SetText Text:=.Range.Text
        MyData.PutInClipboard

        .Protect Type:=wdAllowOnlyFormFields, _

    End With

    Exit Sub

ErrHdl:
    If .ProtectionType <> wdNoProtection Then
        .Protect Type:=wdAllowOnlyFormFields, _

    End If

    MsgBox Err.Number & vbCr & Err.Description, , "Error"
End Sub

Setting Accurate RGB Colors in OS X - Microsoft Community

john korchok say:

Setting Accurate RGB Colors in OS X

There have been several posts lately about color values changing in Office programs in OS X. This affects all non-color-managed programs on Macs, but is most noticeable in PowerPoint, where accurate corporate color values are very important.

With the introduction of the Yosemite operating system, Apple introduced color management to the color picker. If you are using any of the slider panels (Grayscale, RGB, CMYK, HSB), you'll notice a gear icon to the right of the dropdown. Click on this, and you can select a color profile. Depending on your choice of color profile, values that you enter in the picker will be altered to match the color-managed value for that color. This is always different than what you entered and cannot be bypassed.

The workaround for this issue is to download an alternative color picker. There are several choices out there. One that I have installed and tested is the Developer Color Picker. Though it is targeted at web developers, it is useful for any artist who needs predictable RGB values. The download includes instructions, but installation only involves dragging the color picker file to your user Library>Color Pickers folder, then restarting your applications. The OS X color picker will then gain an additional panel that is not color managed and allows you to enter accurate values that will stick.

To make your user Library visible, hold down the Alt key while clicking on the Go menu, then select Library from the list.

If you Library folder doesn't already have a Color Pickers folders, you can create one manually and OS X will read the contents.
