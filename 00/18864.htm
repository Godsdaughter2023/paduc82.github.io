suzanne s. barnhill say:

You can do this with a multilevel list. Start with a Heading 1 paragraph. Click the Multilevel List button in the Paragraph dialog on the Home tab and choose the list style that has 1 Heading 1, 1.1 Heading 2, 1.1.1 Heading 3. That will apply the top-level numbering style to Heading 1. Then click the Multilevel List button again and choose Define New Multilevel List, which will open this dialog.

 

 

The screen shot above shows the result of customizing the dialog for the first three levels. Note that levels 1, 3, and 5 are assigned to Headings 1, 2, and 3, and levels 2, 4, and 6 are assigned to Body Text, Body Text 2, and Body Text 3. You'll need to modify the three Body Text styles (everything except the numbering and indents, which must be set in the Define New Multilevel List dialog) to meet your requirements, and you can modify Headings 1, 2, and 3 so that the "Style for following paragraph" is the corresponding Body Text style.

 

To make the changes shown in the above dialog, you'll need to remove higher levels from the numbering as needed (by carefully selecting the gray fields in the "Enter formatting for number" box and deleting) and, for the Body Text levels, change the number style from 1, 2, 3 to a, b, c. You'll also need to add a period after the last field (which I neglected to do) by typing it into the box. To see the entire dialog as shown, you need to click More >>. Be sure to change the style selected for "Link level to style" at each level.

 

It looks complicated, and it is, but if  you set about it methodically, it is not difficult (and it appears to work as expected).

I don't see any way to do that. Once a specific list level is linked to a style, that style is not available to be linked to another level. I noticed this when I linked Heading 2 to Level 3; when I then went back to Level 2, it was linked to "(no style)."

charles kenyon say:
Your numbering must be attached to styles if you want it to work through editing. I generally just use the built-in heading styles and perhaps format them to appear like ordinary body text if I want a number. See How to create numbered headings or outline numbering in Word 2007 and Word 2010

However, using the heading styles would cause other problems if you need to create a Table of Contents. Why use Microsoft Word’s built-in heading styles?

You could attach keyboard shortcuts to your styles if you are not using the built-in heading styles. You can even attach multiple styles to one key combination followed by a number.
So, you could have:
Alt-1,1 for main heading 1
Alt-1,a for numbered text under main heading 1
Alt-1,2 for heading level 2
Alt-1,b for numbered text under heading level 2
Alt-1,3 for heading level 3
Alt-1-c for numbered text under heading level 3

Alt-1,1 above indicates pressing the key combination of Alt key and 1 at the same time, followed by pressing 1. So, to number a paragraph at a particular level you first press Alt-1, followed by the level desired. This not only numbers the paragraph but formats it according to the other formatting set in the style. Changing the style will change not only the numbering, but, when appropriate, other formatting.

This two-step keyboard shortcuts is analogous to the way Ribbon versions of Word activate the ribbon shortcut keys.

Once learned, such a system is easy to implement.

Option Explicit
 
'This works only based that this style will always be named "Heading 1" to 9
'and expect the outline level to be the last character like in "Heading 1"
'else this function must be changed : Right(strStyle, 1) => Mid(strStyle,9,1) like in "Heading 1 Left 3" to return 1
 
Sub AddParaNumber()
 
Dim rng As Range
Dim strStyle As String
Selection.Paragraphs(1).Range.Select
Selection.HomeKey Unit:=wdLine
 
Set rng = Selection.Range
'get current heading outline level
    Do
        Selection.MoveUp Unit:=wdLine, Count:=1
        Selection.HomeKey Unit:=wdLine
        Selection.EndKey Unit:=wdLine, Extend:=wdExtend
 
        If ActiveDocument.ActiveWindow.Selection.Information(wdFirstCharacterLineNumber) = 1 Then Exit Do
    Loop Until LCase(Left(Selection.Style, 7)) = "heading"
   strStyle = Selection.Style
   If strStyle = "Normal" Then
      MsgBox "No Heading Found", vbCritical
      Exit Sub
   End If
   rng.Select
    rng.Fields.Add Range:=rng, Type:=wdFieldEmpty, Text:= _
        "STYLEREF ""Heading " & Right(strStyle, 1) & """ \n ", PreserveFormatting:=True
      Selection.TypeText Text:="."
    Selection.Fields.Add Range:=Selection.Range, Type:=wdFieldEmpty, Text:= _
        "SEQ h1", PreserveFormatting:=True
    Selection.TypeText Text:=vbTab
    With Selection.ParagraphFormat
        .LeftIndent = CentimetersToPoints(2)
        .RightIndent = CentimetersToPoints(0)
        .SpaceBefore = 0
        .SpaceBeforeAuto = False
        .SpaceAfter = 0
        .SpaceAfterAuto = False
        .LineSpacingRule = wdLineSpaceSingle
        .Alignment = wdAlignParagraphJustify
        .WidowControl = True
        .KeepWithNext = False
        .KeepTogether = False
        .PageBreakBefore = False
        .NoLineNumber = False
        .Hyphenation = True
        .FirstLineIndent = CentimetersToPoints(-2)
        .OutlineLevel = wdOutlineLevelBodyText
        .CharacterUnitLeftIndent = 0
        .CharacterUnitRightIndent = 0
        .CharacterUnitFirstLineIndent = 0
        .LineUnitBefore = 0
        .LineUnitAfter = 0
    End With
'ActiveDocument.Fields.Update
 
End Sub
