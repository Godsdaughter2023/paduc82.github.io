samhrutha g say:

Hi matty72,

 

1.     Check the following setting:

a.     Click Start -> Devices and Printers.

b.     Right click on the printer -> Select Printer properties.

c.      Under the Ports tab -> Clear the Enable bidirectional support check box.

 

2.     You may also try adding the printer by following the below steps:

a.        Click “Start”, “Control Panel”, “Hardware and Sound”.

b.         Click “Add a printer”.

c.         Choose “Add a local printer” and click on “Create a new port”.

d.         Select “Local Port” and click “Next”.

e.        Type in "[\\computer name\printer name]" (without quotation marks) for the Port Name and click “OK”.

 

Example: If the computer you wish to connect to is named Computer A and the printer is called Printer A then the path would be \\ComputerA\ Printer A

mike hall... always say:

This is the price you pay for believing the myth that registry cleaners speed up a PC..

Under any circumstances, registry cleaners will not improve performance, double the speed of a computer or any other spurious clams made.

Back in the days of Windows 9x, it was important to control the size of the registry, and for this purpose, Microsoft released their very own registry cleaner which only removed entries which were absolutely safe to remove.

BUT, that was back in the days of Windows 9x. Windows NT and all of the operating systems spawned from NT do not handle the registry in the same way as Windows 9x did. No user alive will ever install enough stuff to push the Windows NT registry over its maximum size, and a registry cleaner has LITTLE TO NO VALUE when run on Windows NT or highe because orphaned entries are ignored by Windows NT and higher.

 

Re low disk space.. How low? For a disk to perform at its best, you need to keep 25% of it as free space.

 

BUT, you don't want to do this at the expense of space used to keep stuff useful for recovery of your operating system. Some will get you to reduce the space allocated for thsi and that, but don't do it. It is easy to compromise the ability for Windows to fix itself..

 

If your C partition is less than 100gb and you have no way of making the size up to that, you probably need to get yourself a larger hard drive.

 

A 100gb partition will easily accommodate Windows 7, other failrly sizeable applications, the pagefile, system restore points, service packs, general updates, shadow copies and still have enough free space for a little bit more AND the 25% free space..
