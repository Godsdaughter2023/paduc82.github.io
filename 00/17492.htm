Peter --

Microsoft Project 2013 simply does not work the way you are assuming.  A custom view consists of four components, as follows:

View = Table + Filter + Group + Screen

The Expand/Collapse state of tasks in the view is NOT a part of the definition of a view, as you can see from above.  If you collapse a section in one task view, and then apply a different task view, you will see that the section is still collapsed.  The Expand/Collapse state of a summary task affects every task view, and you cannot specify this for individual views.  Sorry.  Hope this helps.
