greg maxey say:

Sub ScratchMacro()
'A basic Word macro coded by Greg Maxey
Dim oTmp As Template
For Each oTmp In Templates
    If UCase(oTmp.Name) = "FIRMBUILDINGBLOCKS.DOTM" Then Exit For
  Next oTmp
  If Not oTmp Is Nothing Then
    oTmp.BuildingBlockTypes(wdTypeQuickParts).Categories("General").BuildingBlocks("Terms").Insert Selection.Range, True
  Else
    MsgBox "The required building block template is not loaded."
  End If
lbl_Exit:
  Exit Sub

End Sub

Note:  Code assumes the building block is named "Terms" and saved in the template QuickPart galley, general category.

charles kenyon say:

Here is a sample of code that inserts the building block "z_HelpFileAlert" from the file "RandomText.dotm" which is located in the Word Startup Folder as designated in the Word Options for File Locations.

Sub InsertHelpFile()
 '  Will not work if there are multiple building blocks with the same name in the template!
 '
   Dim sStartUp As String
    sStartUp = Application.Options.DefaultFilePath(wdStartupPath)
    Application.Templates(sStartUp & "\RandomText.dotm").BuildingBlockEntries("z_HelpFileAlert").Insert Where:=Selection.Range, _
            RichText:=True
End Sub

There is a Default File Path constant in Word for the Startup folder, there is none for the building blocks folder that I know about.

If you are using a macro to do this, I would recommend changing your building blocks template to a .dotm and storing the macro in the template. That makes the code even simpler.

Sub InsertHelpFile()
 '  Will not work if there are multiple building blocks with the same name in the template!

jay freedman say:

Greg's macro needs a small tweak.

To shorten the startup time, Word doesn't load any building blocks until the user does something that requires it -- clicking a button to display one of the building block galleries such as Headers, Footers, Cover Pages, etc. or trying to insert an AutoText entry. If you run the macro before that happens, it will tell you the template isn't loaded.

The tweak is to add this statement before the For/Next loop:

   Templates.LoadBuildingBlocks

That forces any templates stored in the Building Blocks folder to be loaded immediately. If they're already loaded, the statement does nothing.

Sub ScratchMacro0()
'A basic Word macro coded by Greg Maxey
Dim oTmp As Template
Dim oBB As BuildingBlock
  Templates.LoadBuildingBlocks
  For Each oTmp In Templates
    If UCase(oTmp.Name) = "FIRMBUILDINGBLOCKS.DOTM" Then Exit For
  Next oTmp
  If Not oTmp Is Nothing Then
    On Error Resume Next
    Set oBB = oTmp.BuildingBlockTypes(wdTypeQuickParts).Categories("General").BuildingBlocks("Terms")
    If Err.Number = 0 Then
      oBB.Insert Selection.Range, True
    Else
      MsgBox "The BB that you specifically hardcoded in your macro which would lead one to assume that is exists does not exist."
    End If
  Else
    MsgBox "The required building block template is not loaded."
  End If
lbl_Exit:
  Exit Sub
End Sub
