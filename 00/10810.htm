chip pearson say:

You have two distinct objects: Range("A1") and Cells(1,1). While these refer to the same range, they are different entities. Under the covers, you are working with pointers, and the Is operator tests whether the pointer to two objects are the same.  You can see this with the unsupported ObjPtr function. ObjPtr returns the memory address of the start of the object passed to it:

    Dim R1 As Range
    Dim R2 As Range
   
    ' two separate entities that have a property in common
    Set R1 = Range("A1")
    Set R2 = Range("A2")
    Debug.Print R1 Is R2, R1.Address, R2.Address, ObjPtr(R1), ObjPtr(R2)
   
    ' R2 is made the same as R1
    Set R2 = R1
    Debug.Print R1 Is R2, R1.Address, R2.Address, ObjPtr(R1), ObjPtr(R2)
   

As another example, create a class called Class1 and insert:

Public Name As String
Public Color As String

Then in a regular module, use

Sub AAA()
    Dim C1 As New Class1
    Dim C2 As New Class1
   
    C1.Name = "chip"
    C1.Color = "blue"
   
    C2.Name = "cindy"
    C2.Color = "white"
   
    Debug.Print C1 Is C2, ObjPtr(C1), ObjPtr(C2), C1.Name, C2.Name
   
    Set C1 = C2
    Debug.Print C1 Is C2, ObjPtr(C1), ObjPtr(C2), C1.Name, C2.Name
       
    C2.Name = "cathy"
    Debug.Print C1 Is C2, ObjPtr(C1), ObjPtr(C2), C1.Name, C2.Name
End Sub

The first Debug statement shows that C1 and C2 are two distinct objects. The second Debug statement shows that the pointer C1 is now the same as C1, since the ObjPtr value is the same. Since C1 and C2 now point to the same object in memory, setting C2.Name = "cathy" changes the Name property of both C1 and C2.

In addition to ObjPtr, VBA also has StrPtr for string addresses and VarPtr for regular (non-object) variables. None of these are supported by Microsoft.
