Limitation on writing an array to a range - Microsoft Community

ronrosenfeld say:

Limitation on writing an array to a range

Most of us are aware of the process of reading/writing a VBA array to/from a worksheet range.  It is much faster than accessing each cell one at a time and I use it frequently with large sets of data.

However, in doing some testing this evening on a particular project, I ran across an interesting limitation in Excel 2007, and have not found this documented.

If the array to be written has two or more elements, and if one of those elements has more than 8,203 characters, an 1004 runtime error will occur.

Below is the simplest code I've used to reproduce the issue:

=======================

Dim I As Long
For I = 8203 To 8204
Cells.Clear
Dim V(1 To 1, 1 To 2) As Variant
    V(1, 1) = WorksheetFunction.Rept("a", I)
    Range("a1") = V
    Debug.Print "Last Good Contents Length: ", I
Next I

============================

If V has just a single element, the full 32,767 characters will be written to the cell (even if V contains more than that number of characters).

And, the problem can be avoided by writing the cells one at a time:

e.g. [a1] = v(1,1)

      [b1] = v(1,2)

and so forth.

Comments to explain this behavior would be welcomed.

EDIT:  Well I just found another observation of similar behavior by a blogger Dutch Gemini, but no explanation as to the why.  And it is also documented in the MSKB 832136 again with no explanations.

Guess it must be just something we have to live with.

Choose Modify then select the Quick repair option. Use the Quick Repair option first as it will fix many things and complete fairly quickly.

Yes. From the login screen, use SHIFT-Restart from the power icon at the bottom right. This will launch a boot menu, from which you can find Go back to the previous build.

If you still want to test Insider builds, install them separately from your daily use, public build. You can shrink drive C: by 64GB and install the Insider builds to that small partition. Then boot to Insiders or public build as you wish each time. This keeps the stable build working and you can wipe and reinstall Insider builds whenever you need to, or upgrade as new ones come out. Remember that Fast Ring builds are expected to not be stable and will have issues.

rich michaels - mvp (greatcirclelearning.com) say:

Because you are on a Mac, the preference setting that Stefan mentioned is not located where he indicates. It is in Word > Preferences > View.

If marking that preference does not clear up the situation for you, I suggest trying a few things:

  1. Close the document and quit Word completely. If you've left the document open and just closed the lid on your MacBook, the software might not be picking up cursor position properly anymore. With MacWord it is always best practice to save and close all open documents before closing the lid.
  2. Restart you MacBook. I recommend doing this at the end of every day. This clears used memory locations of leftover junk from the various running applications.
  3. Power down your MacBook at least weekly. This forces a refresh of the Operating Systems stored memory locations.
  4. Make sure you are running the most current version of the Office software. Click on Help and choosing the Check for Updates feature will tell you if the software is up-to-date. After applying any updates make sure you Restart your computer.

Hope this helps
