To fix the "Unmountable Book Volume" error:

1. Start your computer by inserting the Windows startup disc or the Windows installation disc if your computer can start from the CD drive.
2. When the Welcome to Setup screen appears, press R to select the repair option.
3. If you have a dual-boot or multiple-boot computer, select the Windows installation that you want to access from the Recovery Console.
4. Type the administrator password when you are prompted to do this. If no administrator password exists, press ENTER.
5. At the command prompt, on the drive where Windows is installed, type chkdsk /r, and then press ENTER.
6. At the command prompt, type exit, and then press ENTER to restart your computer.
7. After you repair the volume, check your hardware to isolate the cause of the file system damage.

If this procedure does not work, repeat it, but type fixboot instead of chkdsk /r in step 5.

IMPORTANT: Running Chkdsk /r can cause you to lose data. Back up your data first. If the above methods don't work, then your hard drive or motherboard may have failed and hardware troubleshooting is in order.

¡firedog say:

I need to get the contacts back from a POP email account that was in Windows Live Mail 2012. 

Contacts lists are not linked to any particular mail account. All mail accounts use the same contacts list. However, there may be different contacts lists available; this is determined by the account used to sign in with. If the user doesn't sign in, he sees the default contacts list.

Is there a straightforward way of doing this?

Short answer: no. Windows Live Mail makes it easy for users to secure their contacts list by signing in, which will maintain a copy of the list at contacts.live.com and keep it up-to-date. The program also offers a contacts backup method, involving exporting contacts to CSV or VCF. If the user didn't avail himself of one or other or both of these possibilities, there is no straightforward way of doing what you want.

Windows Live Contacts are stored in databases in a folder called Contacts. The location of this folder varies with the version of the program installed. In Windows Live Mail 2011 and 2012, it's

C:\Users\<user>\AppData\Local\Microsoft\Windows Live\Contacts
 

The database for the default contacts list is called Default\...\DBStore. Again, its location varies. For Windows Live Mail 2011 build 15.4.3555.0308 and later, it's

C:\Users\<user>\AppData\Local\Microsoft\Windows Live\Contacts\Default\15.5\DBStore
 

You may find other DBStore folders inside the Contacts folder, but they will not be much use. Because DBStore folders are marked as hidden protected operating system files, you will have to deselect Hide protected operating system files and select Show hidden files, folders and drives at Control Panel > Folder Options > View to be able to see their contents:

How to proceed depends what you intend to do with the retrieved contacts. If you need them in a format for transferring to a different program or service, you'd have to use a third-party utility to produce a CSV file. If you intend to use them in Windows Live Mail, you may be able to transfer the database as it is. This depends entirely on the build of the program that created it and that of the one you want to transfer to. If they're identical, you may be in luck. If they're not, the chances are slim.

Just to rub it in: you can avoid all this kind of hassle by the simple expedient of signing in.

ernie san say:

Hello,

The offline map is giving you a visibility to download specific region or location store to your phone or computer. Yes, it's taking a storage of your computer. To know more about maps on Windows 10, click here. To uninstall offline map, we recommend following the steps below:

1. Press Windows Key + Q.

2. Type PowerShell.

3. Right-click on Windows PowerShell.

4. Select Run as administrator option.

5. Type Get-AppxPackage and then press Enter.

6. Type Get-AppxPackage | Select Name, PackageFullName and then press Enter.

(This command will show only the name of the app and its PackageFullName information).

7. Select the PackageFullName information about the desired app, and copy it.

8. Type Get-AppxPackage PackageFullName | Remove-AppxPackage and press Enter.

(This command is to remove the app from the system, replace PackageFullName with the copied information with Step 7).

For example, if you want to remove Maps app, the command to remove it will be as follows:

Get-AppxPackage *Maps* | Remove-AppxPackage

9. Wait till completion of the command, and check if the app is uninstalled, if not restart the system and

check.

Note:

If you want to uninstall the app from all user accounts, use the following command format:

Get-AppxPackage -allusers PackageFullName | Remove-AppxPackage

If you want to remove the app from any particular user account, use following command:

Get-AppxPackage -user username PackageFullName | Remove-AppxPackage

(Replace username in the above command with the correct user account name).

Let us know how it goes.
