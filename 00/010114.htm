zorvek say:

The Excel Solver function operates on the principle of buckets and functions. In order to solve the problem represented by the functions (cajoling the function result to a target value,) Solver changes the values in the buckets (inputs to the functions) to do the adjustments. Because of the interdepencies between the buckets and the functions, Excel built Solver to rely on the worksheet model of cells and formulas.

Given this, you can still leverage the Solver function by using a hidden worksheet with it's "EnableCalculation" set to True and all other worksheet's "EnableCalculation" option set to False. In the UDF, load the hidden worksheet's buckets with the appropriate values and then run Solver against it. See this page for information about how to do this from VBA. Once Solver has arrived at a solution, return from the UDF with the answer and restore the "EnableCalculation" values. I have not done this myself but I have heard it can be done. I do know that a UDF can manipulate the "EnableCalculation" property.
