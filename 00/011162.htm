scottgem say:
A number datatype should only be used if the value will be used in a calculation. If you have to add leading zeros, you may not be using the value in a calc so can use a text datatype.

However, you can display a value with leading zeros using the Format function:

=format([fieldname,"00000000")

will display any number with leading zeros up to 8 characters.

marshall barton say: Generally, you don't care if a Number data type field has leading zeros or not.  If you are concerned about how people see the number, then set the Format property of the form/report text box or field that displays the number to 00000000 so it displays 8 digits.

When receiving a 12048 error, It is normal that the phone loses connection during the reboot phase.

jeeped say:
When cu8Cel is declared as a range, it can have a number of properties. Let's assume that Final3 is 10 and VlookColumn1 is 3.

Set cu8Cel = Cells(Final3 + 3, VlookColumn1)

This sets the cu8Cel variable to the cell C13 and it now has all of the properties of C13. Some of these would be .Row, .Column, .Text, .Address and .Value etc. The default is .Value and this is where your confusion is likely coming from. Once this is set, any number or text that you assign to cu8Cel will go into C13 which is what you do with your CountIf(...); e.g. you are putting a number into C13. It may be less confusing not to rely upon the .Value default and state it explicitly.

Set cu8Cel = Cells(Final3 + 3, VlookColumn1)
cu8Cel.Value = Application.WorksheetFunction.Count(Range(Cells(3, VlookColumn1), Cells(Final3, _
        VlookColumn1)))

During the next loop, VlookColumn1 would change to 4 and then you use

Set cu8Cel = Cells(Final3 + 3, VlookColumn1)

... again so now anything assigned to cu8Cel.Value will go into D13.
