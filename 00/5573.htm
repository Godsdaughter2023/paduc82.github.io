greginmich say:

Defender on Windows 10 is not done after 22 hours and 2.2 million items scanned.  Normally takes about 6 hours on less than 100,000 items.  What's up with that?  Keep it running or abort?

This is something that sometimes seems to happen for no apparent reason – so you should retry the scan a couple of times and see if things don't start to recover on their own. There are two known operational issues that can seriously affect the length of Windows Defender Full Scans:

1.) With the default power configuration, unattended scans are paused after 30 minutes of inactivity, when the PC idles to sleep, and are then resumed on wakeup. So many hours of scan time can easily be lost if you don’t prevent the system from idling to sleep – either by staying active on the PC, or by adjusting the sleep timeout in Control Panel > Power Options.

2.) Full Scans will always run for an excessive amount of time when Defender reverts to scanning all files after the cache is reset by an OS upgrade or update – and rechecking the reputation of all of your files can really add a huge amount of time to the first Full Scan that you run after the reset. Scan times should generally normalize after the first few scans build up the cache of trusted files.

Scan times are obviously dependent on the amount of ground that the scan has to cover – which depends on the number, size, and type of files that need to be scanned. So removing files with Disk Cleanup, uninstalling unwanted apps, and generally cleaning up the clutter will always reduce the workload and cut some time off of your Full Scans.

You can also shave off a significant amount of time simply by telling Windows Defender to stop unpacking and scanning the contents of archived files during the Full Scan – you can do this in the settings with MSE, but with Defender you have to use a PowerShell command:

-DisableArchiveScanning<Boolean>

Indicates whether to scan archive files, such as .zip and .cab files, for malicious and unwanted software. If you specify a value of $True or do not specify a value, Windows Defender scans archive files.

The TechNet documentation for the -DisableArchiveScanning parameter is wrong. This parameter is actually set to False (Disabled False) by default, which means that archive scanning is enabled. You can confirm this by running the Get-MpPreference command in PowerShell. So in order to disable archive scanning, you would run this command at the elevated PowerShell prompt:

Set-MpPreference -DisableArchiveScanning $True

https://technet.microsoft.com/en-us/library/dn433291.aspx?f=255&MSPPError=-2147217396

In addition to physically removing files, and to turning off archive scanning, you can further reduce the scan’s workload by excluding files in the Settings app. You can add your personal work files, media files, archived backups, disk images, etc., to the list of trusted files simply by creating either path or type exclusions for them:

https://support.microsoft.com/en-us/instantanswers/64495205-6ddb-4da1-8534-1aeaf64c0af8/add-an-exclusion-to-windows-defender

…and yes, this is something that Microsoft actually does recommend on occasion (with the proper caveats, of course) for the purpose of speeding up scans, e.g.:

https://technet.microsoft.com/en-us/library/ff823837.aspx?f=255&MSPPError=-2147217396

steven. s say:

Hello Ambarish Das,

 

When you are in an InPrivate Internet Explorer session you are operating in a sandbox. That means that everything you do is stored in a temporary session away from the normal operating files. That way when the session is closed the temporary files are removed and there is no history of what was done.

 

When you clear out the history, temporary internet files and cookies from within an InPrivate session the only files cleaned out are the files to the normal Internet Explorer. In effect your InPrivate session is unaffected.

 

For instance: I can open an InPrivate session, log into my Hotmail account,  and then delete all cookies, temporary internet files, and history. When I hit the refresh button from within my Hotmail account I’m still logged in.

 

This is because of the InPrivate session, everything I’m doing is in the sandbox so its unaffected by the delete browsing history. The only way to clear the InPrivate information is to exit the browsing session (window), thus emptying the temporary files.

 

 

In short, the only way to clean out the browsing history/temporary files of InPrivate Internet Explorer is to click File > New Session or close the session and open a new one.

 

Hope that helps.

ken sheridan say: To reference the combo box by name its bound column must be that containing the year value.  You refer to Column(1) in your post which suggests that the year is the second column, the Column property being zero-based.  If this is the case then the BoundColumn property of cboYear would have to be 2.  The reason for the apparent discrepancy is that the BoundColumn property, while also being zero-based, references the ListIndex of the control as 0, not the first column in the Rowsource, which is 1.

If cboYear is an unbound control then which column is defined as the BoundColumn is immaterial of course, but If it's a bound control then this does mean that the column to which it is bound would have years as its values.  If this is not the case, referencing a non-key column in the first combo box's RowSource as a parameter in the second combo box's RowSource would suggest a flaw in the underlying logical model, however.

You also need to requery the second combo box in the AfterUpdate event procedure of the first with something like this:

Me.cboCategory.Requery

Another point worth making is that where correlated combo boxes are both bound to columns in the underlying table, this can point to the table not being correctly normalized as it implies redundancy, e.g. a table of addresses might have columns for both CityID and State.  Such a table is not normalized to Third Normal Form as CityID determines State, so State is transitively dependent on the key of the table.  Some people are happy to accept this degree of non-normalization, though it does leave the table open to update anomalies.  For methods of using correlated combo boxes in this way with a table normalized to 3NF, however, you'll find a demo file, ComboDemo2K.zip, in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169
