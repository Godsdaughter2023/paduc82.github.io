    What is happening is that when you use the period as a "decimal" separator when the regional decimal separator is the comma, the period is being taken as the List separator, even though it is probably set as a semi-colon (;) under the Control Panel Regional Settings.

    If you have 40.00 + 50.01.01.01 you will get 93,00

    50.01.01.01 is functionally the same as 50.1.1.1 or with sum above it is taken as 50+1+1+1 and if you delete the forty, the sumabove returns 53.

    The 40.00 + 50.00 works of course because in effect it is 40+0+50+0 =90

    The lesson is "You cannot mix the separators"

    This is not unique to 2013.  Prior versions work the same way.
