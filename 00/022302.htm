philipradler say:

Try selecting your table, clicking "Layout," and then clicking the down-arrow under "AutoFit." That will reveal three choices, the last of which is "Fixed Column Width." Once you select that option for your table, you can set the width of each column as you wish, and it should stay put.

 

The behavior you describe sounds like your pesky column is set to "AutoFit Contents," in which case it will constrain how small you can make it to accommodate whatever is already there. Once you go "fixed," you can use the Height and Width windows (just to the right of the AutoFit icon) to set the conditions you need.

 

Incidentally, the "Distribute Columns" button is designed to make all the columns identical in width, so it probably isn't the choice you need.

 

Hope this helps!

philipradler say: In my first reply I assumed you wanted different widths because you referred to separate setting sizes for various columns. But if you actually do want identical widths (but just not the full width of the page), you can use Table>Properties to set the desired total table width (say 4 inches), then turn off AutoFit (go instead to that "Fixed Column Width" option). Then when you "Distribute Columns" you'll get a cheerful 1.33 inches per each of the three columns.

chris de herrera say: Hi,
There are multiple causes.

ynchronization Error 85010014

ActiveSync reports a Synchronization Error code 85010014 because of one of the following reasons:
  • Synchronization Fails because Application Data Folder is Redirected
  • Synchronization Fails because Outlook is in Offline Mode
  • Synchronization Fails because of Outlook script blocking
  • Synchronization Fails because ActiveSync is incorrectly registered with Outlook

See http://www.microsoft.com/windowsmobile/en-us/help/synchronize/activesync-usb.mspx for details.
