martzgfx say: I finally found out what was the problem that was plaguing me. Hope it may help others as well.

The issue I had was that there was a missing "Internet Plug-in" that do help make the connection from the Web browser (Safari in my case) to the Lync desktop client. I assume it may have been removed during the Mountain Lion upgrade, or I may have deleted it by accident at one time, I am not sure. Nevertheless, go check this directory:

[your hard disk]/Library/Internet Plug-Ins/

And see if you can find a plug-in called "MeetingJoinPlugin.plugin" – this is the one that Lync needs in order to make that bridge between itself and the Web browser from where the initial meeting activation came from.

Uninstalling and reinstalling Lync did the trick for me (in Mountain Lion).

Hope that helps!

    andreas killer say:
    Can anybody help as to how I declare a variable in an Excel 2000 workbook module so its value can then be passed to a Visual Basic 6 project (eventual .exe file) which is then called from the Excel module?

    You can not pass a variable to an external file, you can only pass the value using the commandline to that file.

    But the external module can access your variable if

    a.) your variable is declared as PUBLIC

    b.) your variable is in a class module (the code modules of sheets and "ThisWorkbook" are class modules too!)

    Your external project must access the excel instance with GETOBJECT and can retrieve the variable like

      if objExcel.Workbooks("MyFile.xls").Sheets(1).MyVar = 1 then

    The only other way is that you write a DLL and export functions with arguments and call them from excel.

    andreas killer say:

    Could you confirm that I can put an activeX scroll bar control onto the userform. Please note I need the flexibility of the activeX sroll bar to be able to increase the data points above the 30,000 limit 

    Sure, make a new file, open the VBA editor, add a Userform, add a Label and a ScrollBar to the Userform, open the code module of the form, paste in the code below and run it.

    Andreas.

    Option Explicit

    Private Const MaxLong = &H7FFFFFFF ' 2.147.483.647
    Private Const MinLong = &H80000000 '-2.147.483.648

    Private Sub ScrollBar1_Change()
      Me.Label1 = Me.ScrollBar1.Value
    End Sub

    Private Sub UserForm_Initialize()
      With Me.ScrollBar1
        'Limits
        .Min = MinLong
        .Max = MaxLong
        'Current
        .Value = 123
      End With
    End Sub

    roady [mvp] say:

    You should not have emptied the AppData directory of Outlook; Outlook 2010 would have automatically picked up your settings then. By deleting its contents you've corrupted your mail profile and you'll have to recreate it via the Mail applet in Control Panel. For details see;
    http://www.howto-outlook.com/faq/newprofile.htm

    Overwriting/replacing your pst-file like this will also corrupt your mail profile. The proper way is to reconnect to your pst-file via;
    File-> Open-> Open Outlook Data File

    Also, copy your pst-file to the "Outlook Files" directory in your My Documents folder (or create it if it doesn't exist). Leaving it in the AppData structure is asking for it to be forgotten someday. By keeping it in your My Documents folder, it'll be less likely that you'll forget it to include in your backups ;-)

    In addition, also see;
    http://www.howto-outlook.com/howto/backupandrestore.htm
