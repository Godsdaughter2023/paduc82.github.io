It sounds like you've applied protection to the document without making the checkboxes editable in the process. 

  1. With your form open in Word, click on DEVELOPER>Restrict Editing>Stop Protection.
  2. Select a checkbox.
  3. In the Restrict Editing task pane, check Allow only this type of editing in the document. Leave the dropdown at No Changes (Read only).
  4. Under Exceptions, check Everyone. 
  5. Click on Yes, Start Enforcing Protection. Do not use a password, click on OK.
  6. Click on Stop Protection.
  7. Repeat the above steps for each checkbox, but after the last one, leave the form protected. The checkboxes will now be checkable.

Sub CallResize()
    Dim strWs As String
    Dim strList As String
   
    strWs = "Sheet1"
    strList = "Table1"
   
    Call ResizeTables("Sheet1", "Table1", "OssieMac")       'Edit since posting: Hard coded instead of using variables.

End Sub

Sub ResizeTables(strSheet As String, strTable As String, strUser As String)
    Dim lstObj As ListObject
    Dim rngTable As Range
   
    Set lstObj = Worksheets(strSheet).ListObjects(strTable)
    lstObj.ShowTotals = False       'use when other tables on worksheet under table being resized
    Set rngTable = lstObj.Range
    With rngTable
        Set rngTable = .Resize(.Rows.Count + 1, .Columns.Count)
    End With
    lstObj.Resize Range(rngTable.Address)
    With lstObj.DataBodyRange
        .Cells(.Rows.Count, 1) = strUser
    End With
    lstObj.ShowTotals = True    'use when other tables on worksheet under table being resized

End Sub

The following code example for information. It loops through all tables on the worksheet and it is passing Objects in lieu of all strings.

Sub CallResize_2()

A Foreign Key is simply a field in a related table that contains the key value of a related parent record. A FK is defined by its contents. The only requirement of an FK is that it is the same datatype ad the corresponding key.
