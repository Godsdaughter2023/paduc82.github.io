arminatorx say:

Hallo Bozz Elle, 

was für ein USB Kabel verwendest Du, um den Controller an die Konsole zu verbinden? 

Ist das ein "echtes" USB Datenkabel, oder ist das ein Smartphone Ladekabel?

Bedenke, dass einige Smartphone Ladekabel nur die Stromleitungen belegt haben, aber die Datenleitungen eingespart haben. Der Controller bekommt dann zwar Strom, ist aber nicht über das Kabel verbunden. 

Prüfe also, ob Du mit dem Kabel beispielsweise einen Kartenleser oder eine Festplatte oder sonst ein Gerät am PC anschließen könntest, und der PC das Gerät auch erkennt. 

Wenn Du sagst die Controller blinken, aber die Konsole nicht: Drückst Du denn auch den Verbinden Knopf an der Konsole?

Der Knopf ist bei der alten Xbox One (mit externem Netzteil) an der linken Seite, neben dem Laufwerksschacht. 

Bei den neueren Konsolen (Xbox One S oder X) ist der Verbinden-Knopf unterhalb des Power Buttons. 

Wenn Du den drückst, sollte der Xbox Knopf der Konsole auch leuchten. 

[TL;DR Translation]:

Make sure the USB cord is a data cable, and not just a smartphone charging cable (those often have the data lines missing to save costs). 

As for the console not flashing rapidly: Make sure you press the bind button on the console as well. 

On older Xbox One models, that button is on the left side of the console next to the drive slot. 

On newer S and X models, the bind button is below the power button.

¡firedog say:

Two settings have to be in place for there to be an audible signal when new mail arrives:

  1. In Windows Live Mail, when viewing the message list press Ctrl-Shift-O for Options. On the General tab, ensure that Play sound when new messages arrive is selected.
  2. In Control Panel find the Sounds dialogue. On Windows 7 it's in the Sound applet.
    In the Windows section, find, enable and test the New Mail Notification. You can use any handy .wav file - here's mine: http://1drv.ms/1MfRDaQ

Please provide more information to enable your problem to be diagnosed.

 

Please restart your computer and allow 20 minutes for the system to run before uploading information. When examining Event Viewer log files many, not all, problems show in the period immediately after the computer has been booted.

 

Please provide a copy of your System Information file. Type System Information in the Search Box above the start Button and press the ENTER key (alternative is Select Start, All Programs, Accessories, System Tools, System Information). Select File, Export and give the file a name noting where it is located. Do not place the cursor within the body of the report before exporting the file. The system creates a new System Information file each time system information is accessed. You need to allow a minute or two for the file to be fully populated before exporting a copy. Please upload the file to your OneDrive, share with everyone and post a link here. If the report is in a language other than English, please state the language.

 

Please upload and share with everyone a fresh copy of your System log from your Event Viewer to your One Drive and post a link here. It helps to avoid confusion if you remove any earlier copies of the log files from your OneDrive.

 

To access the System log select Start, Control Panel, Administrative Tools, Event Viewer, from the list in the left side of the window expand Windows Logs and select System. Place the cursor on System, select Action from the Menu and Save All Events as (the default evtx file type) and give the file a name. Do not provide filtered files. Do not place the cursor in the list of reports before selecting Action from the menu. Do not clear logs whilst you have a continuing problem.
