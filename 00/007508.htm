joeu2004 say:

sharoraja wrote:

I want to filter data points against 0.02 step. I have applied Mod function. As you can see image attached. Answer for 0.06 reminder must be 0. But it is giving 3.47E-18

There are a couple things going on here (see below).

But generally, the remedy is:  do not use MOD with a non-integer second parameter (divisor).  It is more reliable to write:

=MOD(ROUND(B6*100,0), ROUND($C$1*100,0)) / 100

First, I suspect that what appears to be 0.06 is not the constant 0.06, but the formula =B6+0.01.  In that case, what appears to be 0.06 is actually about 0.06+6.94E-18.

(It is more reliable to write =ROUND(B6+0.01,2). )

The infinitesimal difference arises because 0.01 cannot be represented exactly in 64-binary floating-point, which Excel uses to represent numbers.

Likewise, 0.02 cannot be represented exactly.  So MOD(0.06+6.98E-18, 0.02) results in about 3.47E-18.

But secondly, even if you use the constant 0.06, the result will be surprising:  0.02(!) instead of 0.  Actually, what appears to be 0.02 is really about 0.02-3.47E-18.

Moreover, the result varies depending on the Excel release.  Microsoft made some arithmetic "improvements" in Excel 2010.

Sometimes, it seems to be more reliable to use the formula =B6-$C$1*INT(B6/$C$1).  But that is just a coincidence.

(And =B6-$C$1*INT(B6/$C$1) might result in exactly zero, but IF(B6-$C$1*INT(B6/$C$1)=0,TRUE) might result in FALSE(!).  This is due to a dubious heuristic that Excel applies inconsistently.)
