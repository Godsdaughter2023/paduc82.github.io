john korchok say:

When you select all, right-click and choose Save As Picture, the slide background is not saved, only the selected foreground objects. The picture will be smaller than the original slide and will have different proportions every time, depending on the size and arrangement of the selected objects.

Instead, use the File>Save As command, set the Save as type to your preferred graphic format, click on OK, and choose Just This One in the subsequent dialog. This will save the entire slide, not just the selected objects. The saved file will always be the full size and proportion or the original presentation and there will be no transparent areas.

Need help breaking down an Excel formula. - Microsoft Community

tj trujillo say:

Need help breaking down an Excel formula.

I am self taught when it comes to Excel.  Cand anyone break down the formula Barney style for me? Thank you for any help!!!

=SUMPRODUCT(SUM(INDIRECT("'"&Sheetlist&"'!"&CELL("address", Q10)))/MAX(1,SUMPRODUCT(COUNTIF(INDIRECT("'"&Sheetlist&"'!"&CELL("address",Q10)),">0"))))

bernie deitrick say:

It returns the sum of the values in cells Q10 from the sheets named in the range Sheetlist divided by the count of values in cells Q10 that are greater than zero.  Not sure if that is what you want, but that is what it gives.

This part returns the values from the sheets:

INDIRECT("'"&Sheetlist&"'!"&CELL("address",Q10))

Let's say, there are three sheets, with the values  3, 6, and -3 in Q10, so your formula reduces to

=SUMPRODUCT(SUM(Values)/MAX(1,SUMPRODUCT(COUNTIF(Values,">0"))))

SUM(Values) 

returns 6,  (the simple sum of those values) and 

COUNTIF(Values,">0")

returns 1,1,0

so

SUMPRODUCT(COUNTIF(Values,">0"))

becomes 2 and 

MAX(1,SUMPRODUCT(COUNTIF(Values,">0"))))

becomes 2

Then SUMPRODUCT(6/2) become 3, the overall return, which is the sum of the values in Q10 divided by the count of cells greater than zero - again, not a simple number like average or average of positive numbers.

jllatham say:

I finally have something that might work for you.  The screen capture shows entries and results from this process, so you can judge some from that.  Notice that it guessed wrong on row 7.

 

 

 

This is done with a User Defined Function (UDF) that can be used just like a regular Excel function in a formula.  It just needs the cell addresses of the two phrases to be compared.  For example, in C2, the formula is

=ApproximateEquivalence(A2,B2)

So your two columns don't always have to be A and B, and the result can go into any available column.  I think as long as a human is making the final judgment call on this, it may work for you.

 

To put the code to use:  Open your workbook.  Press [Alt]+[F11] to open the Visual Basic Editor (VBE).  In the VBE, choose Insert --> Module and then copy the code below and paste it into the module presented to you.  Be sure and save the workbook as type .xlsm (macro enabled).  Give it a trial run! 

 

Here is the code:

 

Function ApproximateEquivalence(entry1 As Range, entry2 As Range) As Integer
  Const removeList = ".,-_;><|\/?!" ' characters to remove and ignore
 
  Dim col_1_WorkingData As String
  Dim gp1 As Variant
  Dim gp1_Value As Long
 
  Dim col_2_WorkingData As String
  Dim gp2 As Variant
  Dim gp2_Value As Long
 
  Dim rmvLoop As Integer
  Dim LC As Integer
  Dim maxLength As Integer
 
  'remove leading/trailing white space & make UPPERCASE
  col_1_WorkingData = UCase(Trim(entry1.Value))
  'remove punctuation
  For rmvLoop = 1 To Len(removeList)
    col_1_WorkingData = Replace(col_1_WorkingData, Mid(removeList, rmvLoop, 1), "")
  Next
  Do While InStr(col_1_WorkingData, "  ") > 0
    'replace 2 spaces with just one
    col_1_WorkingData = Replace(col_1_WorkingData, "  ", " ")
  Loop
  'now break up the remaining phrase into 1-word groups
  gp1 = Split(col_1_WorkingData, " ")
  'find the shortest word of all
  'start by setting maxLength to value greater than any word should ever be
  maxLength = 32000
  For LC = LBound(gp1) To UBound(gp1)
    'don't use groups of less than 3 characters
    If Len(gp1(LC)) > 2 And Len(gp1(LC)) < maxLength Then
        maxLength = Len(gp1(LC))
    End If
  Next
  '
  'repeat the process for second column entry
  '
  col_2_WorkingData = UCase(Trim(entry2.Value))
  'remove punctuation
  For rmvLoop = 1 To Len(removeList)
    col_2_WorkingData = Replace(col_2_WorkingData, Mid(removeList, rmvLoop, 1), "")
  Next
  Do While InStr(col_2_WorkingData, "  ") > 0
    'replace 2 spaces with just one
    col_2_WorkingData = Replace(col_2_WorkingData, "  ", " ")
  Loop
  'now break up the remaining phrase into 1-word groups
  gp2 = Split(col_2_WorkingData, " ")
  'find the shortest word of all - keeping previous value of maxLength
  For LC = LBound(gp2) To UBound(gp2)
    'don't use groups of less than 3 characters
    If Len(gp2(LC)) > 2 And Len(gp2(LC)) < maxLength Then
      maxLength = Len(gp2(LC))
    End If
  Next
  'now we calculate values using the shortest maxLength from both groups
  For LC = LBound(gp1) To UBound(gp1)
    gp1_Value = gp1_Value + GetWordValue(Left(gp1(LC), maxLength))
  Next
  For LC = LBound(gp2) To UBound(gp2)
    gp2_Value = gp2_Value + GetWordValue(Left(gp2(LC), maxLength))
  Next
  'test the results
  ApproximateEquivalence = 0
  If gp1_Value = gp2_Value Then
    ApproximateEquivalence = 1
  End If
End Function

Private Function GetWordValue(ByVal myWord As String) As Long
  On Error Resume Next
  Dim charPos As Integer
  myWord = Trim(myWord)
  For charPos = 1 To Len(myWord)
    If Len(myWord) > 2 Then
      GetWordValue = GetWordValue + Asc(Mid(myWord, charPos, 1))
    End If
  Next
  If Err <> 0 Then
    Err.Clear
  End If
  On Error GoTo 0
End Function
