rodney zhang msft say:

Hi Anoop,

In your scenario, you may refer to this article: Managing Updates for Office 365 (focus on Step 4 and Step 2. You don’t need to copy the updates to the update folder.

Note: you may need to prepare two xml files. One is for downloading the update files to the path. The other is for configuring the update files.

The first script(configuration.xml):

<Configuration>

    <Add OfficeClientEdition="32" Channel="Current" SourcePath=" \\server\365\DC">

        <Product ID="O365BusinessRetail">

            <Language ID="en-us"/>

            <ExcludeApp ID="Publisher"/>

        </Product>

    </Add>

    <Display Level="Full" AcceptEULA="TRUE"/>

<Logging Name="OfficeSetup.txt" Path= "%temp%" />

</Configuration>

Run the command: ~~~~Setup.exe /download configuration.xml

Note: Make sure to delete the existing source files before you download or copy the newer source files to your source locations. Each released Office 365 ProPlus build creates a new folder (named by the build number) and will not overwrite existing files.

The second script(update.xml):

<Configuration>

    <Add OfficeClientEdition="32" Channel="Current" >

        <Product ID="O365BusinessRetail">

            <Language ID="en-us"/>

            <ExcludeApp ID="Publisher"/>

        </Product>

    </Add>

    <Updates Enabled="TRUE" Channel="Current" UpdatePath="\\server\365\DC" />

    <Display Level="Full" AcceptEULA="TRUE"/>

<Logging Name="OfficeSetup.txt" Path= "%temp%" />

</Configuration>

Run the command: ~~~~ Setup.exe /configure update.xml

If you still can’t update with ODT, please feel free to post back.

Best Regards,

Rodney

dale howard [mvp] say:

John --

One way to do this would be to specify multiple cost rates for each resource using the Cost Rate Tables A-E.  Your are limited to setting up to 5 cost rates for each resource, so I would recommend one cost rate for each phase of the project (assuming you have 5 or less phases in the project).

After setting the cost rates for each resource, you would need to assign one or more resources to each detailed task in the schedule.  Then, you would need to apply the Task Usage view and temporarily insert the Cost Rate Table column.  For each phase of the project, specify the appropriate Cost Rate Table value for every resource assigned to the tasks in that Phase.
