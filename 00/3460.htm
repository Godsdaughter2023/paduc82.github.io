allen browne say:
The simplest way to do this is with a report.
 
Create a main report that has the date and time on the left.
Then add a sub-report that shows the related records.
Design the subreport so its width shows only one name, but make it wide enough to show several (perhaps 4 across.) On the Columns tab of the Page Setup dialog, set up the subreport so it has 4 columns across, and its width there is wide enough to be 4 times the actual report width (setting column spacing to zero.)
 
If you don't want to do it that way, you could use a function that concatenates all the names together into one long string. There's an example in this article:
    Concatenate values from related records
at:
    http://allenbrowne.com/func-concat.html

jsssssssss say:

In case you're having the issue with MSE full scan:

Community Moderator  Stephen Boots posted on August 31, 2015 in another thread on same topic:

QUOTE

Microsoft is aware of the problem affecting a number of users running a full scan with MSE.

Most of the lockups are occurring in two folders:

\Windows\Winsxs\backup and/or \windows

Installer\$PatchCache$\Manager\

At this time, there are two, possibly three, workarounds while we await a solution from Microsoft which will probably come in the form of updated signatures or an update to the scanning engine.

  1. Exclude the above folders from being scanned in MSE settings
  2. Change your scheduled scan to a quick scan
  3. Boot the PC into Safe Mode and run a full scan - I've received a report that this worked for someone

As long as real time protection is enabled, a full scan on a frequent basis is usually overkill. A quick scan looks for malware in the most common locations. A full scan has to touch every file. Malware found in a full scan, not in a quick scan is typically not active. Personally, I only ever run a full scan if an infection has been detected or if one is suspected. And, if the machine is running MSE, I use a different scanner for the full scan (the malware was missed by MSE when it arrived, so why should I expect it to find it on a full scan?)

The following scans are good for a second opinion, even after the MSE freeze problem is resolved:

Try Malwarebytes Antimalware (free version, do not install the trial or Pro version) - http://www.malwarebytes.org/products/malwarebytes_free/

Try SuperAntispyware Free version - http://superantispyware.com/download.html

Try the ESET free online scan:  http://www.eset.com/us/online-scanner/

 

-steve
