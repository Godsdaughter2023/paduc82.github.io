How do I represent this data over the weight scale and show something that should be closer to 0.5.

The reason why I feel sumproduct is not working; if you put 100 in the first index (weight of 5%, the result exceeds 5.5 ... that value should not affect the total that much).

joeu2004 say:

"Chaos" wrote:

Ok I know sumproduct...  Is there something else to represent the data better. 0.8 does not logically seem correct when the majority of the index is closer to .4

This is a statistics question, not an Excel question, much less a question about SUMPRODUCT.

You started with the assumption that you want the (weighted) average.  SUMPRODUCT correctly calculates that, as you have used it.

But it is true that the average does not always represent the "centrality" of the data.

Consider the data 1, 2, 3, 4, 1000 in A1:A5.  =AVERAGE(A1:A5) returns 202.  The calculation is correct.  But you might argue that 202 does not adequately represent the "majority" of the data.

Perhaps the MEDIAN would be a better representation.  =MEDIAN(A1:A5) is 3.

If that appeals to you, you would want the "weighted median" in your case.  Unfortunately, that is difficult to calculate at all, much less in a single formula.  If you have the original data that you summarized by your table, it would be better -- more accurate as well as easier -- to use MEDIAN with the original data.

But MEDIAN is not the only alternative for calculating "centrality".  The "best" measure to use depends on the "distribution" of the data.

This is because POP accounts do not - cannot - sync. It's a one-way connection and once the desktop client has the message, it cannot tell what happens on the server. You must use a n IMAP account with gmail in order to have two-way synching.

suzanne s. barnhill say:

Unless you're actually using A4 paper, there's nothing to be gained from setting the paper size to A4. It may well be that you'd be better served by saving the full-size (Letter) document to PDF and then using Adobe Reader's booklet feature, but I think the aspect ratio problem would be the same. Consider the math: A Letter page is 8.5" x 11": an aspect ratio of 8.5/11 or 11/8.5, which is .77 or 1.29. Half-Letter (sometimes called Statement) is 5.5" x 8.5", a ratio of .64 or 1.54.

You can visualize the difference this way:

  1. On a Letter-sized page with 1" margins all around, draw a rectangle 6.5" x 9" (that is, filling the margins). Fill it with a bright color so you can see it easily.
  2. Change the paper size to 5.5" x 8.5" and the margins to 0.5" all around. Obviously, the shape is now too large for the page.
  3. I'm not sure what controls you have for shapes in MacWord, but in WinWord you would click the dialog launcher in the Size group on the Drawing Tools | Format tab. This opens the Layout dialog with the Size tab in front.
  4. In the Size dialog, check the box for "Lock aspect ratio."
  5. Change the width of the shape to 4.5" (the margin width of your resized document).
  6. Note that the height is just 6.23 instead of 7".

With some trial and error, you could figure out the balance of side vs. top/bottom margins that would create a document body the right shape to fill the Half-Letter page correctly, but it's unfortunate that MacWord doesn't have the "2 pages per sheet" or "Book fold" option.

What you can do that works quite well is to set the paper size to 5.5" x 8.5" to begin with, save it as a PDF, and then use the booklet feature in Adobe Reader. This produces a 5.5" x 8.5" booklet that can be printed on 8.5" x 11" paper (with pages in booklet order) automatically.
