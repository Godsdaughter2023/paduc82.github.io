Checking both the latest relevant ISO standard and the Microsoft summary of their support for the standard (MS-OI29500 - which does include some useful clarifications/confirmations) I can find no mention of the '\s' switch put into the STYLEREF field code by the Word interface when adding a caption and indicating that you want to include the chapter number.

Code placed by Word is: {STYLEREF 1 \s}

It seems fairly obvious that the switch returns just the calculated number for the last piece of text with Heading 1 style applied (ie. without the delimiter). I'd like to be able to rely on that switch when generating docx with numbered captions but concerned that it is an undocumented feature.

stefan blom say:

As you have noticed, \s is added by the "include chapter number" setting for caption numbering.

\s is equivalent to \n \t. Either of these variations will add the number (for the referenced style) and suppress delimiters.

doug robbins - mvp office apps & services (word) say:
I would have no qualms about relying upon it.  There would be hell to pay if Microsoft changed the behaviour as a result of which users' captions that included the heading number no longer worked.

pjd.au say:

Thanks Stefan for coming back so quickly...and please don't take this response the wrong way.

The spec and my own experimentation bear this out, 

\t  - when used with the \n, \r, or \w switch, causes non-delimiter and non-numerical text to be suppressed.  

So the delimiter still comes through, my assumption was that as captions give you the option of effectively redefining the way the number was delimited the switch was needed (just odd given its usefulness not to make it better known).

to pull data from the main form and just display it on the subform; these controls will not be editable, but in general you wouldn't want the same data being edited in two places anyhow.
