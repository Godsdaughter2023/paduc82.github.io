(Acknowledging that JP Ronse has offered a simple specific solution for you that avoids the international issue, I will just add this additional information from Stephen Bullen's site)

Unfortunately, I am based in the US and don't generally distribute to other regions so my knowledge of international issues is limited (I haven't really investigated the LCID either).  However, Stephen Bullen is from the UK and worked a lot in Norway.  He wrote a book on Excel 2002 VBA and had a chapter on international issues.   While most of the chapter is from the VBA perspective, he did address the formatting string in the Text function not being converted when the file is opened in a version of excel with a different region.    His solution was to use a cell format in the worksheet(cell formats will be converted).  This can be fetched using a old Excel4 Macro command (Get.Cell) used in a defined name (ex:  create a named range  DateFormat  defined as  =Get.Cell(7,$A$1)  ; the first argument 7 indicates to get the formatting string from the cell specified in the second argument   )   Then use this defined name/named range as the string argument to the Text worksheet function 

=Text(M9,DateFormat)

if A1 was formatted as   d/m/yyyy  then

=Text(M9,Right(DateFormat,4))

He has posted this chapter on his web site freely sharing this excellent information.
