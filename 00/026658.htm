john4742 say:
Here's one way!

Do the reverse: count cells with numbers.  Overwriting the numbers with text alters the total.

 Create a table e.g. 5 by 5 and insert an extra column of 6 cells on the left (add a row and delete the rightmost 5 cells).  Put a field in each of the 5 upper left cells to total each row (sum(right)), and in the bottom left put a sum(above) field. Then put (in this case) 25-  in front of the "sum(above).  Put 1 in all the remaining cells.  Select-All (Ctrl-A) and hit F9 to update.  Each row total will say 5 but the single cell at the bottom will say zero (25 minus 25).

Now type text into any single cell except the left column and Update again (Ctrl-A then F9) and the bottom left cell will read 1.

I'm sure Ken's demo will illustrate this, but the FileDialog property is what you use to open the Windows File Open dialog to select a file. I would not use a hyperlink field. Just store the location of the file as a text field. Then use

ken sheridan say: Take a look at Browse.zip in my public databases folder at:

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file illustrates how to browse to  file using the FileDialog property of the Application object.  The file can be opened by clicking the path, which is formatted as a hyperlink, or by clicking a button.  The file is opened by calling the Windows API ShellExecute function, which also allows it to be printed directly.

In my demo the path is assigned to an unbound text box control, but it could of course be assigned to a bound control in a form bound to a table.  Storing the paths in a text column like this would not bloat the database, so you could store as many as you wish in this way.
