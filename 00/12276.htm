kosh vorlon say:

Hi Dolores,

 

Assuming you're talking about browsers and using IE, there are several ways.

 

1.  Click on the Favorites in the Menu Bar while the page is open (Favorites and bookmarks for purposes of broswers are synonyms) and then choose to add to favorites or add to Favorites Toolbar if you are using that feature.

 

2. If using the Favorites Toolbar and you want to add it to that, while the page is open, click the yellow star on the far left and it will be added to the Toolbar.

 

3.  If using the Favorites Toolbar, you can use the left mouse button to click and grab the icon to the left of the address in the address bar and hold down the mouse button as you drag it to where you want it on the Favorites Bar (between two current items and don't release until you see a vertical bar indicating you are positioned between the two of them and not on one or the other) and then release the mouse button and the current page will show as a new item in the Favorites Bar wherever you placed it.

 

4. While in an open area on the page you want to save, right click and then click Save to Favorites and choose the name and destination (main folder or some sub-folder) and save it.

 

I think that should explain it and give you enough choices to choose whatever fits your fancy.

 

I hope this helps.

 

Good luck!

    bill manville say:

    A way in which that could go wrong would be if the workbook was not the active workbook at the time the Auto_Open procedure was run.  Sheets("Customer") has an assumed ActiveWorkbook. preceding it.

    Office 2013 changed the way in which Excel uses Windows, with each open workbook having a window of its own.  It is easy to accidentally change the active workbook while working, and I have some evidence that Excel sometimes changes the active workbook all by itself.  Either of these will cause macros to fail that expect their own workbook to be active.

    Why this is happening in the case where you send it to others is unclear, but I would suggest as a first attempted remedy you precede Sheets("...") or Worksheets("...") by ThisWorkbook. throughout the code (where the sheet is indeed within the same workbook), e.g.

      ThisWorkbook.Sheets("Customer").Select
