karl timmermans [outlook mvp 2012-2018] say:

By definition - Imap accounts are stored locally - there is no contacts folder on Imap servers so you have to have another account configured if there is another contacts folder so the question becomes

 

What exactly do you have configured in your profile by way of email accounts and data files (i.e. PST files). If you have a "contacts" folder - means <either> that you have another account setup OR you have added another data store (PST file) to your profile that was at one point a <default> store in another Outlook install.

 

<IF> you are trying to import your data into a <contacts> folder in a separate PST file, two things I'd suggest

 

#1 - Make sure that it's not an ANSI based PST file - click on the root of that store --> Data File Properties --> Advanced --> If format states <Outlook Data File (97-2002) it's an old format file.

 

#2 - Run ScanPST on that PST file to eliminate potential errors that may exist in the file

 

#3 - <IF> item #1 shows that this PST file is an ANSI PST file, export it, close your original PST file and open the new one and use the new one going forward.
