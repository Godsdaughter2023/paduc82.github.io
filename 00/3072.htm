    Ovi Maps vs. Google Maps - Microsoft Community

    pkreitler---01 say:

    Ovi Maps vs. Google Maps

    Let me begin by saying I am a long time user of Google Maps.  I have used Google Maps since its inception and know its ins and outs well.  I own an unlocked E71 with the most current available firmware for my model (v200.21.118 ) and use the most current version of Google Maps available for S60v3 (v3.2.1).  As Nokia has introduced its Maps products, I have paid attention.  I even tried Maps back at version 2, or something like that.  I wasn't impressed with the product then and went back to Google Maps.  Google hasn't updated their version of Google Maps since early 2009 because they are focused on their Android products (I consider this move to be evil, but that is another discussion), so I thought I'd take another look at Ovi Maps.

     

    Today I downloaded the most recent version available for my E71 (v3.0109wk44b01MW) and downloaded my current state here in the US. I am writing this as a complaint/review because I don't think that Ovi Maps 3.01 is up to par with Google Maps from a year ago.  It does have voice navigation, if you pay a fee, and it does have traffic, if you pay a fee, but other than that, I see nothing that Nokia's product does better than Google's.  Here are serveral short comings I've found in my brief exploration of the software.  I hope someone at Nokia is paying attention as their feedback form on their Maps page only asks what I think of their directions:

     

    1. When I go to search for a place or address, I have to use two steps until I can type.  I have to hit Options, then Right, and then the center button for Address or down and then center for places. 
    2. Searching for addresses doesn't work.  If I try a search for an address, I then have to go through another set of menus to choose "By Coordinates" which I doubt anyone ever knows, or  "From Contacts."  When I choose contacts, it takes me to my contacts as expected.  But, when I choose a contact, it highlights a single field within the contact like it wants to edit the contact.  I then have to press Option again and press Search.  But, it only searches the highlighted field, which always results in an unfound address.  So, essentially the address lookup is useless.  Contact look up used to work on Google Maps too, but they removed the function.
    3. Searching for a Place is very awkward: If I try to search for a place, I get a list of places to choose from or I can type in a place.  That is all well and good I suppose.  I like the way Google simply has a place to type and then uses predictive text to quicken the typing.  It is just smoother than navigating through more menus.  Though I like Google's approach better, Nokia's is acceptable.  Just clunkier.    Now to the heart of the matter.  The way Ovi Maps displays the search results in a list is ok, but I would prefer to see them all on a map relative to my position or the center of my map first and as a list second.  When I tell Ovi Maps to view the results on a map, it takes me to the item highlighted rather than showing me all the locations at once.  You can see all the locations at once if you zoom out, but again, this takes an additional step.  The other frustration is that it is difficult to jump from one search result to the next using the keypad.  Google Maps allows me to go forward with the space bar and backwards with delete, which makes navigation through my choices very easy.

    macropod say:

    The following macro will process your document's footnotes & endnotes, retaining their internal formatting (eg bold/italic/underline) and referencing style (eg superscripted roman numerals).

    Note the comments in the code about the limitations on footnote processing. As coded, theyâre effectively turned into a series of (unlinked) endnotes.

    Note also comments in the code regarding optionally enclosing the endnote and footnote references with { } and [ ], respectively, and for changing the reference colours.

    Sub UnLinkNotes()
    Application.ScreenUpdating = False
    Dim Fld As Field, nRng As Range, eNote As Endnote, fNote As Footnote, nRef As String
    With ActiveDocument
      'Convert any endnote/footnote cross-reference fields to text
      For Each Fld In .Fields
        If Fld.Type = wdFieldNoteRef Then Fld.Unlink
      Next
      'Delete the next two lines if you don't want to differentiate footnote and endnote references by colour
      .Styles("Footnote Reference").Font.Color = wdColorGreen
      .Styles("Endnote Reference").Font.Color = wdColorRed
      For Each eNote In .Endnotes
        With eNote
          With .Reference.Characters.First
            'The next two lines insert a brace after the endnote. Two lines
            'further down insert a brace before the endnote.
            'This is useful if you want to be able to re-create the endnotes later on.
            .InsertAfter "}"
            .Characters.Last.Font.Superscript = True
            .Collapse wdCollapseStart
            .InsertCrossReference wdRefTypeEndnote, wdEndnoteNumberFormatted, eNote.Index
            nRef = .Characters.First.Fields(1).Result
            .Characters.First.Fields(1).Unlink
            'The next two lines insert a brace before the footnote. Two lines
            'further up insert a brace after the footnote.
            'This is useful if you want to be able to re-create the footnotes later on.
            .InsertBefore "}"
            .Characters.First.Font.Superscript = True
          End With
          .Range.Cut
        End With
        'To position converted Endnotes at the end of the document, use:
        Set nRng = .Range
        'To position converted Endnotes at the end of the Section, use:
        'Set nRng = eNote.Range.Sections.First.Range
        'To let the code decide, based on whatever's in the document, use:
        'If .Range.EndnoteOptions.Location = wdEndOfSection Then
        '  Set nRng = eNote.Range.Sections.First.Range
        'Else
        '  Set nRng = .Range
        'End If
        With nRng
          .Collapse wdCollapseEnd
          .End = .End - 1
          If .Characters.Last <> Chr(12) Then .InsertAfter vbCr
          .InsertAfter nRef & " "
          With .Paragraphs.Last.Range
            .Style = "Endnote Text"
            .Words.First.Style = "Endnote Reference"
          End With
          .Collapse wdCollapseEnd
          .Paste
          If .Characters.Last.Next = Chr(12) Then .InsertAfter vbCr
        End With
      Next
      For Each eNote In .Endnotes
        eNote.Delete
      Next
      For Each fNote In .Footnotes
        With fNote
          With .Reference.Characters.First
            'The next two lines insert a square bracket after the footnote. Two lines
            'further down insert a square bracket before the footnote.
            'This is useful if you want to be able to re-create the footnotes later on.
            .InsertAfter "]"
            .Characters.Last.Font.Superscript = True
            .Collapse wdCollapseStart
            .InsertCrossReference wdRefTypeFootnote, wdFootnoteNumberFormatted, fNote.Index
            nRef = .Characters.First.Fields(1).Result
            .Characters.First.Fields(1).Unlink
            'The next two lines insert a square bracket before the footnote. Two lines
            'further up insert a square bracket after the footnote.
            'This is useful if you want to be able to re-create the footnotes later on.
            .InsertBefore "["
            .Characters.First.Font.Superscript = True
          End With
          .Range.Cut
        End With
        'The nearest a converted footnote can be positioned to its reference without
        'considerable extra work that can involve splitting paragraphs in the body of the
        'document and splitting footnote text across pages is the end of the same Section.
        'this can be done via:
        'Set nRng = fNote.Range.Sections.First.Range
        'The alternative is to position them at the end of the document, via:
        Set nRng = .Range
        With nRng
          .Collapse wdCollapseEnd
          .End = .End - 1
          If .Characters.Last <> Chr(12) Then .InsertAfter vbCr
          .InsertAfter nRef & " "
          With .Paragraphs.Last.Range
            .Style = "Footnote Text"
            .Words.First.Style = "Footnote Reference"
          End With
          .Collapse wdCollapseEnd
          .Paste
          If .Characters.Last = Chr(12) Then .InsertAfter vbCr
        End With
      Next
      For Each fNote In .Footnotes
        fNote.Delete
      Next
    End With
    Set nRng = Nothing
    Application.ScreenUpdating = True
    End Sub

    Sub ReformatFootnotes()
    Application.ScreenUpdating = False
    Dim FtNt As Footnote
    For Each FtNt In ActiveDocument.Footnotes
      With FtNt.Range.Characters.First.Previous
        If .Text = vbTab Then .Text = " "
      End With
    Next
    Application.ScreenUpdating = True
    End Sub

    doug robbins - mvp office apps & services (word) say:

    Use the following:

    ' Macro created 29/09/99 by Doug Robbins to replace endnotes with textnotes at end of document

    ' to replace the endnote reference in the body of the document with a superscript number.

    '

        Dim aendnote As Endnote

        For Each aendnote In ActiveDocument.Endnotes

            ActiveDocument.Range.InsertAfter vbCr & aendnote.Index & vbTab & aendnote.Range

            aendnote.Reference.InsertBefore "a" & aendnote.Index & "a"

        Next aendnote

        For Each aendnote In ActiveDocument.Endnotes

            aendnote.Reference.Delete

        Next aendnote

        Selection.Find.ClearFormatting

        Selection.Find.Replacement.ClearFormatting

        With Selection.Find.Replacement.Font

            .Superscript = True

        End With

        With Selection.Find

            .Text = "(a)([0-9]{1,})(a)"

            .Replacement.Text = "\2"

            .Forward = True

            .Wrap = wdFindContinue

            .Format = True

            .MatchWildcards = True

        End With

        Selection.Find.Execute Replace:=wdReplaceAll

    It will do all that you want in one operation, without the need to copy and paste.
