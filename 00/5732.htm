jay freedman say:
It isn't possible to print size A0 in any version of Word, even if you have a printer or plotter that is able to handle paper that large. There is a hard-coded limit of 22 inches (558.8 mm) in either direction.

You'll have to use another program that doesn't have this limitation. If you have Microsoft Publisher, that program can handle A0, although you'll have to define it as a custom size. You can import a Word document into Publisher, but for a single page it's probably easier to just paste in the text and set the proper font size.

You can create a .pdf file of the poster in Publisher, using the choice on the File menu. You can also use the Save As item on the File menu and change the type to Word (.doc) format, but then you'll get the large font on a standard (A4 or Letter) size page, and you'll have to reduce the font size to make it fit again.

I may just start using Gmail's filter feature in place of Outlook rules just to save myself the headache. Any ideas?
roady [mvp] say:

That is indeed a result of Gmail's take on IMAP.


They don't use folders but "labels" which are exposed as folders when you connect to Gmail over IMAP. As IMAP doesn't understand the concept of single storage and references, each item which has one or more labels will be stored in both Gmail's All Mail folder and in each label "folder" it has been assigned to. When a reference is changed, the item will become a duplicate.


As Gmail is server-side storage, it is best to do filtering as much as possible at server level as well rather than at client level. This will also make sure that you items are being sorted on other devices you may use even when you don't have Outlook running at that time.

A shared mailbox doesn't have credentials, so you cannot set it up as a standalone mailbox in Outlook. Once users have the permission, the shared mailbox folders will appear in Outlook automatically under their primary account.

Widow orphan control

How do I get to/use widow orphan control?  In Word for Office 2007 I would like to start a paragraph on the last line of a page whether 1 line or more and have it run over to the top of the next page if it is 2 lines or more and have a 2 line only paragraph still have a single line on the top of the next page with justification on all lines of the paragraph. RB
stefan blom say:

Make sure that the insertion point is in a paragraph to which the Normal style has been applied (unless you manually applied a different style, Normal is the style used for body text paragraphs in Word). Press Ctrl+Shift+S to display the Apply Styles pane. Next to the style name, click the Modify button. In the dialog box, click Format | Paragraph and go to the Line and Page Breaks tab. Clear the option that says "Widow/Orphan control" and click OK twice.

At this point, I think you can have one or the other, but not both at the same time.  The 7 style start is enabled by default, but you can switch it to the 8 style start.

1. Right-click on taskbar, select properties

2. Select "Start Menu" tab

3. Uncheck "Use the Start menu instead of the Start screen"

Doing this will prompt you to save your work, because you must log out and back in for the changes to take effect.  Because you need to log upon changing, I do not think the option for both is going to be easy for a user to do.
