a. user say:

It would help if you could be a bit more specific about what you are checking for, but let's say you have two columns in access called "col1" and "col2", and if the value of "col1" is blank you want to insert the value of "col2", otherwise insert "col1". Then you can insert a nested field that looks like this:

{ IF "{ MERGEFIELD col1 }" = "" "{ MERGEFIELD col2 }" "{ MERGEFIELD col1 }" }

Each pair of {} in there are "special field braces" that you can enter using ctrl-F9. You can't use the ordinary keys on the keyboard. You can also use Alt-F9 to toggle between "field codes" view and "field results" view. You may need to do that as you proceed, because Word can sometimes switch the view. So for example, to enter the above, you could
 a. click in the document where you want the result to appear
 b. click ctrl-F9 so you have
{  }
 c. click between the {  } and type IF "" = "" "" so you have
{ IF "" = "" "" }
 d. click between the first pair of ""
 e. click ctrl-F9, so you have
{ IF "{  }" = "" "" }
 f. click between the new { } and type MERGEFIELD col1, so you have
{ IF "{ MERGEFIELD col1 }" = "" "" }

...and I think you should be able to enter the other two MERGEFIELD fields. Please try to preserve the spacing as suggested - it doesn't have to be exactly as I have posted, but you usually need a space on at least one side of the "=", and I personally find it easier to remember the above pattern and apply it.

Custom Template location in PowerPoint on Office 2013 - Microsoft Community

keith- say:

Custom Template location in PowerPoint on Office 2013

Where can I find my custom templates in PowerPoint Office 2013? Where can I find my custom templates in PowerPoint Office 2013?
michael. o say:

In previous versions, when saving as a template from an Office application (e.g. - .dotx, .pptx, etc.), Office would automatically change the location to C:\Users\username\AppData\Roaming\Microsoft\Templates.

 

However, in Office 2013 this location is used to store templates from Office.com.  To separate personal templates from Office.com templates, the default personal location was moved to C:\Users\username\Documents\Custom Office Templates.  All Office applications will save templates to this folder by default.

 

You can choose to override this setting by specifying a Personal Templates Folder.  This can be done by going to File > Options > Save and manually entering a folder path.  This needs to be done for each application, and for any application that does not have a Personal Templates path manually set, Office will create the Custom Office Templates folder the first time you save a template, and will set the Personal Templates path to this location.

gord dibben say:

How about using a macro?

Copy/paste the macro to a General module then run it on the active worksheet.

Sub findavalue()
    findwhat = InputBox("Enter a value")
    With ActiveSheet.Range("A2:F49")
        Set c = .Find(findwhat, LookIn:=xlValues, Lookat:=xlPart)
        If Not c Is Nothing Then
            MsgBox c.Address
        End If
    End With
    If c Is Nothing Then
        MsgBox ("value not found")
    End If
End Sub

m h g say:

If there could be multiple instances of the number sought, selecting the match in the topmost row would be given by the array formula

=ADDRESS(MIN(IF(B2:G11=I1,ROW(B2:G11))),MIN(IF(INDEX(B2:G11,MIN(IF(B2:G11=I1,ROW(B2:G11)))-ROW(B2)+1,0)=I1,COLUMN(B2:G11))),4)

and selecting the match in the leftmost row would be given by the array formula

=ADDRESS(MIN(IF(INDEX(B2:G11,0,MIN(IF(B2:G11=I1,COLUMN(B2:G11)))-COLUMN(B2)+1)=I1,ROW(B2:G11))),MIN(IF(B2:G11=I1,COLUMN(B2:G11))),4)

2-cell approaches would be much more efficient. Topmost row,

X1:  =MIN(IF(B2:G11=I1,ROW(B2:G11)+COLUMN(B2:G11)/1024))

X2:  =ADDRESS(INT(X1),MOD(X1,1)*1024,4)

Leftmost column,

X1:  =MIN(IF(B2:G11=I1,ROW(B2:G11)/1024+COLUMN(B2:G11)))

X2:  =ADDRESS(MOD(X1,1)*1024,INT(X1),4)
