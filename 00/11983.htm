jay freedman say:
In the Customize Keyboard dialog, there's a dropdown near the bottom labeled "Save changes in". It defaults to "Normal.dotm", which is available to all documents. If that's what the dropdown is set to when you assign a shortcut, then the shortcut should be saved in the Normal.dotm template unless something is interfering with it.

One way to check on this is to go to the Options dialog, select the Advanced tab, scroll down to the Save group, and check the box for "Prompt before saving Normal template". Then assign a shortcut, and shut down Word. You should see a message asking whether to save changes to the template. If you don't see that message, there's a problem preventing Normal.dotm from being saved.

Such problems, when they occur, are usually caused by an add-in but could also be incorrect permissions on the template file or the Templates folder where it's stored.

Having ruled out Manual calculation mode, it sounds to me like the data had been entered or interpreted by Excel as text.  When you double-click on a cell, you are effectively re-entering the cell contents, which are then interpreted as numeric.

One way that happens is:  the cells were formatted as Text when data was entered, then the cell format was changed to General or a numeric format.  Simply changing the cell format does not immediately cause the cell contents to be re-interpreted as numeric.  We must re-enter them after changing the format.

Try the following.  Select the data in column C, select the General or a numeric format (even if you think it is already), then use the Text To Columns operation (click on Data, Text To Columns, then Finish).  Repeat each steop for the data in column D.  To select 16,000 records, enter C2:C16001 in the Name Box near the upper left.

If the cells are still not equal, the problem might be leading or trailing spaces or non-breaking spaces, especially if you cut-and-paste the data from a webpage.  Confirm or dismiss this conjecture by entering the formula =LEN(C2) into E2 and copy down appropriately.  Look for places where the length is not the same as the number of digits that you see.  Let us know if that's the case.

jferrone say: The text to columns did the trick.  This was an export from Access, and since some of the cells contain the letter N, it would be formatted as text, so that must have been the issue.  Anyway, glad to have a solution, thanks!
