racerex---01 say:

If you have a choice I would highly recommend that you install all your downloaded apps onto the E: drive for a few reasons...

 

1.  Some programs require installing on the C: drive so I always prefer to leave that space for those unique cases.

2.  In the case that anything becomes corrupted with your installed apps you can simply reformat the E: drive and then reload all your apps easily from the Ovi Store (Ovi remembers what apps you downloaded so reinstalling them is simple).  If you install apps onto C: and then something becomes corrupted you're in trouble because if you reformat the C: drive then you lose Qt and Ovi Store.  There is no official solution today for users to reinstall Qt and Ovi Store and they are NOT embeded in the current firmware.  So you are forced to go to a NOKIA care center where they will reinstall it for you.

3.  Finally, I don't think there's any performance difference between C: and E: because they are physically on the same piece of silicon.  They are simply partitioned that way from NOKIA.

formula to extract data from different sheets to a master sheet - Microsoft Community

purandarrao say:

formula to extract data from different sheets to a master sheet

I have 50 sheets of customer orders, each sheet has identical tabel headers and identical no of columns. However they have different no of rows. I wish to merge the all sheets in to one master sheet. The lookup column is Orderdate column. Fifty customers can have one or many orders in a perticular day. There are three products. Hence each customer can have one or many orders for each product on a perticular day. Hence I wish to arrange or extract all orders of all customers daywise in one master sheet. The design of the table is as given below. Please help me in this. Thanks for the same in advance.

orderdate   customerName   CityName   ProductName
rich007a say:

If all your sheets have the same sheet-name stub (e.g. "Sheet1", "Sheet2", "Sheet3", etc.
and
if all your sheets have the same column headings and are arranged identically,
then:

1) Insert a new worksheet, name it "Setup"
Put the sheet-name stub in cell B1 (e.g. "Sheet")
Put the number of sheets to be combined in cell B2
Put the number of header rows in B3


2) Insert a new sheet and name it "Summary" (or similar)


3) Copy the header row(s) from one of the sheets into the Summary sheet


4) In the Summary sheet, in the first column, in the first row under the column headers, paste this formula:

=IF(INDIRECT("'"&Setup!$B$1&(ROW()-(Setup!$B$2*((ROUNDUP((ROW()-Setup!$B$3)/Setup!$B$2,0))-1))-Setup!$B$3)&"'!R"&(ROUNDUP((ROW()-Setup!$B$3)/Setup!$B$2,0))+Setup!$B$3&"C"&COLUMN(),FALSE)="","",INDIRECT("'"&Setup!$B$1&(ROW()-(Setup!$B$2*((ROUNDUP((ROW()-Setup!$B$3)/Setup!$B$2,0))-1))-Setup!$B$3)&"'!R"&(ROUNDUP((ROW()-Setup!$B$3)/Setup!$B$2,0))+Setup!$B$3&"C"&COLUMN(),FALSE))


5) Drag that formula across to cover the full width of column headings.


6) Drag those cells down to cover the following number of rows:

[Max number of data rows on sheets to be combined] x [Number of sheets to be combined]

Note: "Data rows" do not include "column header rows".

For example, if you have 50 sheets, each with 2 column header rows, and with 20 rows of data, you would drag down to cover
20*50 = 1000, so drag down to row 1002.


7) You can now use auto filter on the Summary sheet, but if you want to sort you must first copy and paste as values.  First insert a new sheet, then copy all of the Summary sheet and paste into the new sheet using Paste Special > As Values...   You can now use Sort to sort the data.

Hope that helps.

Cheers
Rich

rich007a say:

Did you miss step 7?  Unfortunately I don’t think it is possible to do everything you need with a formula – and if it was, the formula would be huge and the load on the PC will probably crash it!

You could do the following though:
1) Complete steps 1-6 from my previous post.
2) Rename the Summary sheet “Summary Calcs”
3) Right-click on the tab of sheet “Summary Calcs” and click View Code
4) Paste the following code:

Private Sub Worksheet_Calculate()
    Dim ws As Worksheet
    Dim strWsStart As String
   
    Application.EnableEvents = False
    Application.ScreenUpdating = False
    Application.Calculation = xlCalculationManual
    Application.DisplayAlerts = False
   
    strWsStart = ActiveSheet.Name
   
    On Error Resume Next
    Sheets("Summary").Delete
    On Error GoTo ExitPoint
   
    Sheets("Summary Calcs").Copy Before:=Sheets(1)
    Set ws = Sheets(1)
    ws.Name = "Summary"
    ws.UsedRange.Value = ws.UsedRange.Value
   
    ws.Sort.SortFields.Clear
    ws.Sort.SortFields.Add Key:=Range("A:A"), _
        SortOn:=xlSortOnValues, _
        Order:=xlAscending, _
        DataOption:=xlSortNormal
    With ws.Sort
        .SetRange ws.UsedRange.Columns()
        .Header = xlYes
        .MatchCase = False
        .Orientation = xlTopToBottom
        .SortMethod = xlPinYin
        .Apply
    End With
   
    Sheets(strWsStart).Activate

ExitPoint:
    Application.DisplayAlerts = True
    Application.Calculation = xlCalculationAutomatic
    Application.ScreenUpdating = True
    Application.EnableEvents = True
End Sub


8) Go back to Excel and be sure to save your workbook as a Macro Enabled Workbook (.xlsm).

Now a new “Summary” sheet should be created each time the “Summary Calcs” sheet (the one that contains all the formulas) is recalculated, which should happen each time you make a change to one of the other sheets.  The Summary sheet will be sorted by Column A.

Hope that helps.
Cheers
Rich
