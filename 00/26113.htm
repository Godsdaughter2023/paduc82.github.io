Recovery Code in Outlook.com email app.

toby ornott say:

In Outlook Mail:

  1. on the far right of your screen locate your Profile .
  2. Click on Profile and select View Account.
  3. In the ensuing Microsoft webpage's blue ribbon heading, locate Security .
  4. In the next page...Update your security info. (*Note: Before you attempt to "update", you must have either an alternate email address, or an ID Verification, or a mobile phone number in order to receive a verification code.  This code is NOT the RECOVERY CODE; it is the "verification code".  If you do not have either of the three, then your best bet is to acquire one. See ID Verification Apps: FAQ)
  5. Once you have completed the "verification process", you will be redirected to your Microsoft Account Security page.
  6. Located on that page, under the following -- "Security info up to date? Explore more options to keep your account secure" -- will be a link to the Account recovery code.  It is a 25-digit, alpha-numeric code.  You can use your recovery code if you lose access to your security info. You need to print out your recovery code and keep it in a safe place.  This final point is extremely important.

The entire process of gaining access to your Microsoft Privacy & Security Settings can be a mind-boggling challenge if you do not prepare yourself with all the details as described above.

amit tandon say:

Please note the following for Ctrl+A in Excel to determine your action:

This shortcut behaves differently in different contexts. If the cursor is in an empty cell, Control + A selects the entire worksheet. But if the cursor is in a group of contiguous cells, Control + A will select the entire group of cells instead.

The behavior changes again when the cursor is in an Excel Table. The first time you use Control + A, the table data is selected. The second time, both the table data + table header are selected. Finally, the third time you use Control + A, the entire worksheet is selected.

Excel does its own calculations for how many columns it can show based on a variety of inputs - including if you have a high pixel count in your display setting, be it large or high DPI or both. The objective is to allow 8 zeros in each cell with sufficient space between the two to show the cell borders and a bit of a buffer - A normal screen requires more space so has a 8.43 characters default column width, but a high DPI scaled screen can have less space with no loss of clarity and uses about 8.1 characters default column width (@125 or 150%).
