cbarnhorst say:

I am certain that your Omen uses a gen 7 or gen 8 CPU.  Since your new Omen shipped with Windows 10, be aware that as of July of last year Microsoft no longer will supply you with updates to Windows 7 via Automatic Updates.  Windows 7 updates for your device will need to be applied manually.

Microsoft is saying that supporting Windows 7 and 8 on 6th generation or later hardware is extremely expensive because the newer CPUs and chipsets are simply too complex for the design of legacy operating systems so Windows 7 and 8 updates are not being tested on CPUs later than those for which Windows 7 and 8 were designed.

No doubt your Omen shipped with a Skylake (Intel 6th gen) or later.  You can tell by reading the CPU ID, which is listed on your System Properties page.  For Intel hardware, you will see "i7-6xxx" for a sixth gen CPU, "i5-7xxx" for seventh gen, and "i7-8xxx" for eighth gen."  The tags in quotes are just examples.  The i3, i5, i7, etc. don't matter.  It is the four-digit string afterward that tells you the hardware generation.  Just look at the first digit.  If you have an AMD CPU then you can look up the strings online to check the generation but it is the same idea.  Intel, AMD, and Qualcomm are all in sync on this.

You can still update Windows 7 manually, but you have to know what KB numbers you need and get the updates for Windows 7 from the Microsoft Update Catalog.  Of course, XP is well past end-of-life and there are no new updates for it.

As I said, I have a Dell Latitude D630 that I bought with XP for $75 on eBay, complete with battery and AC adapter.  As it happens, I also installed Windows 7 Pro on it (dual boot) and the Windows 7 installer found all drivers for it automatically.  You will find all of the XP drivers for it on the Dell Support website.  I used Dell's SupportAssist utility.  Other brands of computers should work the same way.

This is my legacy gaming machine.  I love it for that.

Stay away from the legacy operating systems on your Omen, if for no other reason you are under warranty.

Have you changed any fonts on your Mac lately?  The crash report hints that there's a problem with a font.

A simple troubleshooting step to perform is to do a Safe Boot. It will take a long time for your Mac to start up in Safe Mode. When you do a Safe Boot in El Capitan, Apple's Disk Utility runs and fixes any troubles with your Mac's directory and file system. Permissions are repaired. The font cache is emptied. Only a limited number of fonts are enabled. Applications that run at startup don't run automatically. 

 

Once you reach the desktop in Safe Mode try running Excel. Is the crash still there? If not, then maybe simply performing a Safe Boot fixed the ailment. Or perhaps one of the fonts that wasn't loaded is bad. Or perhaps some other software utility is interfering. After testing in Safe Mode, do regular restart. Try Excel again. Is the crash still there?

 

If the crash was gone in Safe Mode but returns during a regular boot then there are more things to try. The first would be to use a font utility to validate ALL your fonts. Mac OS comes with a free font utility called Font Book. Get rid of any font that doesn't pass as clean and green, restart your Mac and then empty the trash. If you've gotten this far and still have a crash, then there are more things to explore. But if you get this far you've probably gotten rid of the crash. Either way, please reply with your results.
