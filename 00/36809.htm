a. user say:

Hi Nicholas,

Thank you for posting your query in Microsoft Community.

I understand that your External Hard Disk Drive (HDD) is not recognized in Windows 10 and will surely assist you with the issue.

The issue may arise if the currently loaded USB driver has become unstable or corrupt or if your PC requires an update for issues that may conflict with a USB device and Windows. This may also occur if your USB controllers may have become unstable or corrupt. So, I would suggest you to try the following steps and check if the issue persists.

Method 1:

Run the Hardware and Devices troubleshooter and check if it helps.

a) Press Start button from the keyboard.

b) Type “Troubleshooting” in the search bar and press “Enter”.

c) In the “Troubleshooting” window, click on “View All” on the left pane.

d) Click on “Hardware and devices”.

e) Click on “Advanced” and then click on “Run as Administrator”.

f) Click “Next” and follow the on-screen instructions to complete the troubleshooting process.

If the issue still exists, try the next method.

Method 2: 

Remove and reinstall all USB controllers.

  1. Press Windows key + X, click Device Manager.
  2. In the list of hardware categories, find and expand Universal Serial Bus controllers.
  3. Right-click every device under the Universal Serial Bus controllers node, and then click Uninstall to remove them one at a time.
  4. Restart the computer, and let the USB controllers get reinstalled.

Hope this information is helpful. Do let us know if you need any further assistance, we will be glad to assist you.

Thank you.

Internet drpping out after installing Wondows 8.1 - Microsoft Community

shilohbob say:

Internet drpping out after installing Wondows 8.1

After updating several Desktops (3) & Laptops (2), I found that the Internet continually dropping out or pages timing out & being advised that the web page selected could not be accessed, which was naturally driving me right up the wall.


After investigating the problem, this what I found out:


The issue relates to the way W8.1 performs DNS lookups, which differs from all previous versions of Windows OS. This change exposed a timing /race condition bug within some router devices firmware, and results in some webpages failing to load or dropout, (in my case Technicolor Gateway) and some other brands, I understand some Netgear devices are not affected. Technicolor are in the process of issuing a patch which will most probably be a few days away yet. I am not aware of other device manufacturers intentions. If you have the problem contact your manufacture for information. I also understand that Microsoft are aware, but what the intend to do, I do not know.


A fix until modems are patched, below is a quick fix, which appears to work. I carried the out fix on all my machines, approx. 18hrs ago and at this stage, all are connecting OK and I have had no drop out whatsoever.


The quick fix is to change DNS Setting on your computers to a Google DNS which is:


PRIMARY DNS : 8.8.8.8

SECONDARY DNS: 8.8.4.4


There are other DNS (Domain Name Services) which you may be available for use.


The one above does work.


If you are not sure how to change your DNS, check out the link below.


https://portal.plus.net/support/software/dns/changing-dns-windows-8.shtml


You can also change you modem/router DNS, which will also fix the problem.


Suggest you contact your modem/router manufacturer or read your manual, personally I am going to wait for the patch to be released.


Others with more understanding, are welcome to add to above advice.

File History is not giving any notice of errors or warning via system tray notifications, etc. - but I found FH is now working after ensuring these items addressed by monitoring the Event Viewer "File History backup log" as linked from "Advanced settings" under Control Panel>File History: 1) If you see "MAX_PATH" in error: File path lengths need to be under 255 (might be 260)...while there are tweaks to the registry to allow Win10 to use files with paths' lengths greater than 255, FH is restricted. Google (don't BING - lol) "PathLengthChecker1.2.0" and change the "Root Directory" to whatever drive / folder path you're backing up and change "Min Path Length" to 255; click on 'Get Path Lengths' to scan for files that are too long.  Any software app that can do this will suffice really...typical problem files like urls, saved as shortcuts, are examples.  Rename these and it will remove the "MAX_PATH" errors.  2) FH cannot use "junction folders" so if you've done something like mklink /d "M:\symbolic" "S:\real_location" > creating a symbolic folder link this is failing on my system - remove these.   3) I've found using a .vhd as the target/destination for external drive is working (using a mapped drive to the .vhd file) but when I tried using .VHDs as the source folders this was failing; seems Win10's File History needs "physical folders" directly on local drives and not located in .VHD files.  Hope this helps, Rich
