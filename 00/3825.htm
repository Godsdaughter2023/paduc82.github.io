IF Statement with LEN function - Microsoft Community

teddy811 say:

IF Statement with LEN function

Hello,

Can someone please help me figure out what this formula represents? I'm having a difficult time deciphering it. Thanks in advance for your help.

=IF(LEN(C2)=4,(A2&"  "&B2&"         "&C2),IF(LEN(C2)=5,(A2&"  "&B2&"        "&C2),IF(LEN(C2)=6,(A2&"  "&B2&"       "&C2),IF(LEN(C2)=7,(A2&"  "&B2&"      "&C2),IF(LEN(C2)=8,(A2&"  "&B2&"     "&C2),IF(LEN(C2)=9,(A2&"  "&B2&"    "&C2),IF(LEN(C2)=10,(A2&"  "&B2&"   "&C2))))))))

hansv mvp say:

The formula returns the value of A2, followed by 2 spaces, followed by the value of B2, followed by a variable number of spaces (see below), followed by the value of C2.

The variable number is 9 if the length of C2 is 4.

It is 8 if the length of C2 is 5.

Etc. - in short, 13 - the length of C2.

This means that the formula can be simplified drastically to

=A2&"  "&B2&REPT(" ",13-LEN(C2))&C2

REPT(" ",13-LEN(C2)) returns a string of spaces of length 13-LEN(C2).

Directory Synchronization, part #3: AD-Connect- Installation, configuration and running a first sync cycle

Technical Level : Basic

Summary

Azure AD-Connect (or DirSync in it's former name) is an on-premise client that allows us directory synchronization.

Using this tool we allow centrally managed identities, by keeping our user accounts, groups and attributes in sync with Office 365 and Azure AD.

Centrally managed identities and groups are the base of user experience delivery in hybrid modes with SSO capabilities.

AD-Connect allows us the choice of three different sync types:

  • Password hash-sync, to allow sign-in to Office 365 services using full domain credentials.
  • Active-Directory Federation (AD FS) sign in, to allow SSO using domain managed federation services, and advanced logon rules (none password sync)
  • No password sync at all (use Office 365 based password).

In this guide we will discuss directory synchronization with password hash-sync.

Before proceeding, please make sure that you've enabled Office 365 for synchronization (part #1 of this article series) and remediated any possible issues using IdFix (part #2 of this article series).

PUPS are not classified as viruses, worms, Trojans, rootkits, botnets so not all security engines will detect and remove them. In many cases, including scan detection for PUPs is optional.

A Potentially Unwanted Program (PUP) is a very broad threat category which can encompass any number of different programs to include those which are benign as well as malicious. Thus, this type of detection does not always necessarily mean the file is malicious or a bad program. PUPs in and of themselves are not always bad...many are generally known, non-malicious but unwanted software usually bundled with other free software to include toolbars, add-ons/plug-ins and browser extensions. PUPs are considered unwanted because they can cause undesirable system performance or other problems and are sometimes installed without the user's consent since they are often included when downloading legitimate programs.

PUPs may also be defined somewhat differently by various security vendors and may or may not be detected/removed based on that definition.
* * * * * * * * *
Anti-virus and anti-malware programs each perform different tasks as it relates to computer security and threat detection. The scanning engines of anti-virus programs and anti-malware tools look for (detect) and remove different things based on the criteria used by the vendor. Anti-virus primarily protect against viruses, worms, Trojans, rootkits while anti-malware programs generally tend to focus more on adware, spyware, browser hijackers and PUPs. There can be some overlap in functionality and detection features between the two but the security vendor defines a specific threat and what naming standards are used.

5. In the Blocked senders list, add the name or domain (the part of an e-mail address that follows the @ sign, such as hotmail.com) that you don't want to receive mail from, and then click Add to list >>.

1. In the upper-right corner of the page, click the Cog icon, and then click onMore mail settings.

3. Click New and then under Step 1, click the drop down arrow on the first box and chooseSender's address.
4. Under Step 1, type in on the third box the email address.
5. Under Step 2, choose Delete these messages—don't even put them in my Deleted folder.
6. Click Save at the bottom of the page.
