ken sheridan say: Let's be sure that you really want a summation operation rather than an addition operation; the two are not the same.  Summation operates over an arbitrary number of values in a set; addition operates over a fixed number of distinct values.

The expression you are using at present returns the summation of result of the addition of two values, e.g. with rows:

x      y
2    -1
3    -4
4    -2

the addition of the values in each row gives us 1,-1 and 2 respectively, so the summation of  the result of the addition of the values, Sum(x+y) would therefore return 2.

Is this what you want, or do you simply wish to return the result of the addition of the two values in each row:

x      y    x+y
2    -1     1
3    -4    -1
4    -2     2

In the latter the Sum operator is not used at all; the operation is merely the addition of the values in each row.

If you are intending to sum the result of the addition of the values, I can see no reason why Sum([Profit/Loss]+[comm]) would return 360 where the summations of each of the values, Sum([Profit/Loss]) and Sum([comm]) return 250 and -110 respectively.  Are you absolutely sure that the actual value of the latter is a negative number, and not merely formatted as such.  If the actual values are both positive numbers then the operation should be subtraction rather than addition to return the result you expect.

ken sheridan say:

My understanding of =Sum([ProfitLoss]+[comm]) would be =sum([250.00] +[-110.00])

No, the expression would return the summation of the addition of the values in each row, not the addition of the summations of each column.  If there can be Nulls at either of the column positions in any rows then the addition in those rows would in each case return Null, as Null propagates in arithmetical operations.  The expression should be:

    =Sum(Nz([ProfitLoss],0)+Nz([comm],0))

Calling the Nz function is only necessary, however, if either of the columns can be completely empty, i.e. all Null.  If this is not possible you can use:

    =Sum([ProfitLoss])+Sum([comm])

This does return the addition of the summations of each column.

>>
I changed email providers... I'm using the same email address. Outlook 2016 will not let me edit the account or add a new account under the same email address.

All comes down to how your account is currently configured where how = POP3 or Imap?

Judging from your description that you exported a PST file, then it would mean that your account is configured as a POP3 account, in which case, all you need to is the following

#1 - Go to File --> Account Settings --> email tab
#2 - Highlight the existing PST account, go down to the <Change folder> button and take note of the folder path/file name of the existing PST file for the account
#3 - Remove the Email account from the profile - it will not result in the PST file for the account being removed from the profile
#4 - Add the email account back to the profile, manually configuring the POP3 account with the appropriate settings making sure to use the existing PST file (really important that you do that)
#5 - You're all done - no need to deal with the exported PST file

If your existing email account is configured as an IMAP account, then the steps you outlined apply but you really need to make sure to import your PST BUT have some other suggestions for you regarding your contact/calendar data. If this is the case, please let me know.

brenridley say: Ok.... I went ahead and deleted the account. Then when I tried to add a new account it did not let me choose what type (I want IMAP) It only adds a Microsoft exchange account. The .pst file did restore properly. It was and office 360 email account. Now what?

karl timmermans (outlook mvp 2012-2018) say:
>>
Then when I tried to add a new account it did not let me choose what type (I want IMAP) It only adds a Microsoft exchange account.
<<

#1 - The email account was configured as what originally?

#2 - Who is the email service provider for this account? (For it to default to an Exchange account, means that it is a hosted Exchange account and if that's the case, don't know why you would want it to be configured as an IMAP account)

a) if all you did was enter your email address/password and it configured itself as an Exchange account, then you have subscribed to a hosted Exchange account (far superior to configuring it as an IMAP account)

b) all you need to do is verify that you have a sufficiently large enough email space quota on the server for the items you want to copy - once that's out of the way then what you need to do is the following

#1 - Set Outlook to work <offline> --> go to Send/Receive tab on ribbon --> click on <Work Offline> which is an on/off toggle for online/offline. Reason for doing this is that you don't want Outlook to start sync'ing while you're copying the data from your PST file

#2 - Select the calendar folder in the PST file folder tree --> click on <View> --> <Change View> --> select a list type

#3 - Press CTRL+A to select all items
#4 - Right-click on selected items --> select <Move> --> select <Copy to folder> --> select the target calendar folder in the Exchange folder tree

#5 - Rinse and repeat steps #2-#4 for the contact and inbox folder with the only difference is that you don't need to <Change the view>

#6 - Right-click on each folder in the PST folder tree --> select <Copy Folder> --> select the location in the Exchange folder tree that you want to copy the data to

*** the reason you want to use <Copy> instead of <Move> in steps #4 & #6 is purely "precautionary" in the event that there are issues encountered during the copy phase. If you select <Move> the source data will get deleted from the source folder.

#7 - When you're comfortable that everything has been copied, set Outlook to work in <Online> mode

#8 - Start a manual Send/Receive on the account to initiate the sync process so that everything gets sync'd up to the server. How long this takes depends solely on the amount of data involved

#9 - Go to File --> Account Settings --Data Files tab --> select the .ost file for the Exchange account and make it the default data file for the profile

#9 - When you're happy that everything has been sync'd properly, you can remove the PST file from the profile --> right-click on the <Root> of the PST folder tree --> select <Close>

All done
