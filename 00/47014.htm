What is that update for and why do you need it?

A rhetorical question - perhaps the Microsoft engaged Support Engineer "expert" will tell us why you even need it.  Of course that will never happen.

That KB is one of several sometimes problematic installations that are part of the Microsoft "customer experience and diagnostic telemetry" which gather information about your system when you opt-in to participate in the Customer Experience Improvement Program (CEIP) and to help prepare your system for a future Windows 10 update.

Some of these updates will also cause the System File Checker (SFC) to generate errors that could make you think you have a problem when you really don't (since MS knows about the issue).

Some folks feel that these updates are an intrusion on their privacy since who knows that information they might be gathering about you and your system and sending to Microsoft and if you want to be a little more secret about what MS knows about you and your system and you aren't ready to transition to Windows 10 you really don't need that update - and there are a few more like it:

KB3022345
KB3068708
KB3035583
KB2952664

You can perform a Google search for that/those afflicted updates and read more about the problems with it.

You can uninstall these updates and then hide them so they are not offered again.  If it is your desire at some future date you can unhide and install them when you feel like it.

Here's how:

Go to Control Panel, Programs and Features, Installed updates (bottom left corner) and wait for the search to complete.

There are actually 4 updates to look for and consider uninstalling (then hide them later):

KB3022345
KB3068708
KB3035583
KB2952664

Enter the KB number in the Search for Installed Updates box to find each one and if you find it, right click and choose to Uninstall.

After uninstalling them, you need to hide them so they are not offered to you again.

From Control Panel, Windows Update, Check for updates and wait for it to finish (could take a long time).

For a few weeks reports have been seen of Windows checking for updates for a long, long, long time so you might want to schedule that accordingly.


You can always choose to unhide the update later if it is your desire.

Here is another list from Wilders Security Forum that you might also check out:

- Input “select disk n”, where is n is the number of the hard disk that the recovery partition on it.

- Type “list partition” to show all the partitions on the selected hard disk.

- Type “select partition m”, where m is the number of the recovery partition.

- Then type “assign”: the system will assign a drive letter automatically.

How do you stop the plague of desktop.ini files appearing - Microsoft Community

mikeslovakia say:

How do you stop the plague of desktop.ini files appearing

For the past month I have been plagued by these files appearing everywhere on my machines. They have appeared in the past -then disappeared and now they are back. Why does it keep happening and what can I do to stop it
bossdweeb say:

If you make any custom changes to Desktop, or any Windows settings and delete the relevant .ini files you will

loose your preferences, and as you have found, Windows will create new default one when you reboot.

They just sit there happily waiting for you to make a change to the function they are able to alter.

This happens with other Windows default folders such as the Default User folder as well.

It sounds like you do not have your Protected System Files hidden.
- Go to Control Panel > File Explorer options > View.

Tick the box next to  Hide protected operating system files (recommended)

The other section - Hidden Files and Folders will also clear up some visual space if you elect to hide them,

but some of these folders/sub-folders need to be accessed occasionally, particularly the  User\AppData folder.
