Hi,

If you receive Windows Update error 8024000b, it means that Windows Update can't read a file (Update Manifest) that is required for installation.

To fix this problem, follow these steps:

  1. Click Start, and then click Run. If you don't see Run in the Start menu, type Run into the Start search box, and then, in the results list, click Run.

  2. Copy the following text, and paste it into the Run text box:

    Cmd /c ren %systemroot%\System32\Spupdsvc.exe Spupdsvc.old

  3. Click OK.

  4. Run Windows Update again.

We appreciate the update, Marea. If the problem does not happen in Safe Mode, we could deduce that the problem is not caused by a component of Windows and may probably be triggered by a process or service that is running at startup. You may determine the process or service that is causing the problem by performing a clean boot of Windows. Here's how to perform a clean boot of Windows:

  1. Log on to the computer by using an account that has administrator rights.
  2. From Start, search for msconfig, then select msconfig or System Configuration from the search results.
  3. Click the Services tab, then click to select the Hide all Microsoft services check box, then click Disable all.
  4. On the Startup tab of the System Configuration dialog box, click Open Task Manager.
  5. On the Startup tab in Task Manager, for each startup item, select the item, and then click Disable.
  6. Close Task Manager.
  7. On the Startup tab of the System Configuration dialog box, click OK, and then restart your computer.

After restarting to a clean boot environment, observe if the issue persists. If the issue does not persists, it means that a process or service is triggering your task bar icons to overlay on each other. Here's how to determine which process or service is causing the problem while in a clean boot environment:

  1. Open System Configuration by following steps 1 to 3 from above.
  2. Click the Services tab, then click to select the Hide all Microsoft services check box.
  3. Click to clear the upper half of the check boxes in the Service list.
  4. Click OK, then click Restart.
  5. After the computer finishes restarting, determine whether the problem still occurs.
    • If the problem still occurs, repeat steps 1 to 4, but clear the lower half of the check boxes in the Service list that you originally selected.
    • If the problem does not occur, repeat steps 1 to 4, and select only the upper half of the remaining check boxes that are cleared in the Service list. Repeat these steps until you have selected all check boxes.
    • If you still experience the problem when only one service is selected in the Service list, this means that the selected service causes the problem, and you should go to step 9. If no service causes the problem, kindly go to step 6.
  6. Open System Configuration.
  7. Click the Startup tab, then click to select the upper half of the check boxes in the Startup Item list.
  8. Click OK, and then click Restart.
    • If the problem still occurs, repeat steps 6 and 7, but clear the lower half of the checked boxes in the Startup Item list that you originally selected.
    • If the problem does not occur, repeat steps 6 and 7, and select only the upper half of the remaining check boxes that are cleared in the Startup Item list. Repeat these steps until you have selected all the check boxes.
    • If you still experience the problem after only one Startup Item is selected in the Startup Item list, this means that the selected Startup Item causes this problem, and you should go to step 9. If no Startup Item causes this problem, a Microsoft service probably causes the problem, repeat steps 1 to 7 without selecting the Hide all Microsoft services check box in each step.
  9. After determining the startup item or the service that causes the problem, kindly contact the program manufacturer to determine whether the problem can be resolved. Or, run the System Configuration utility, and then click to clear the check box for the problem item.

If performing a clean boot does not resolve the issue, do not hesitate to post back to us.
