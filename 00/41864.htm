a. user say:

Hi Martin,

Windows Server 2008 R2 allows you to specify how and when computers are updated through the use of Group Policy. Follow the steps below to configure the GPO:

1. Open Group Policy Management Console

2. Create a new GPO, and name it whichever name makes sense to you for this case. Don't forget to add the delegation privileges for the object.

3. Open the GPO editor for the newly created object

4. Expand Computer Configuration, > Administrative Templates > Windows Components, and then click "Windows Update"

5. Find the policy "No auto-restart for scheduled Automatic Updates installations", and enable the restriction

6. To force group policy to update the policy, open a PowerShell or Command Prompt window with Administrative Privileges, and type "gpupdate /force", without the quotations. Wait a minute or so, and then reboot when the system is done updating the policy.

This problem can be caused by ‘malware’ but check this registry key before running your ‘anti‘ programs. If it isn‘t started, start Explorer via Task Manager by pressing Ctrl+Shift+Esc then on the Applications Tab click ‘New Task’ at the bottom, type explorer.exe and press Enter.
