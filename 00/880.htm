The backup feature represents the contents of the document "one save operation ago," so you would have to save twice to see a backup (*.wbk) file. But note that Word won't let you save a second time unless you first make a change in the document, for example if you add and delete a space.

Word creates a lot of temporary files when a document is open. For more information, see the KB article at http://support.microsoft.com/kb/211632 ("Description of how Word creates temporary files").

julie sheets say:
I would like to know how I am able to set up a budget resource in MS project 2010 ?

Budget resources are created the same way other resources are - enter the information in the Resource sheet.  After creating the resource, double click on the resource to show the Resource Information window.  Click the checkbox for Budget.  Budget resources can be either Cost or Work resources.

How to assign and manage them ?  and finally How to track this kind of costs ?

Budget resource can only be assigned to Project Summary tasks.  Display the Task Usage view and make sure the Project summary tasks is displayed (format tab).  Add the Budget cost and Budget work fields to the table.  Assign the budget resources and specify the budget cost and/or budget work values as needed.

The budget cost and budget resources are really for comparison purposes and provide a high level plan.

You should also assign your work and cost (non-budget) to tasks and then save a baseline to have a bottom up plan.  After saving the baseline, you may start to track work and costs and compare against the baseline data.

gooseberry say:

The answer was not correct, but given how hard it is to find this information, I cannot blame anyone.

The best reference I have found so far is at Fonts, sizes and color settings of the Outlook interface

The information there goes through Outlook 2013, but I found much of it useful for 2016 as well.

I have spent untold days off and on tweaking email programs for a friend with increasing visual problems. Microsoft really needs to address low vision better than they do.

Play with font sizes two ways:

1. View (tab)-> Settings, Other Settings (button), Row Font (button).

2. View (tab)-> Settings, Conditional Formatting (button).

The unresolved problem I have is that I want extra vertical spacing between lines. Simply making the font bigger can also make it look crowded. (I discovered by accident if you set the font bigger via No. 1 above than you do via No. 2 above, it can give you extra vertical spacing for SOME of your messages, but that is hardly a good solution.)

There is also a setting somewhere for changing the background to gray (a little darker than I would like but better than white).

To get the messages to have a larger font when you open them, try the ReliefJet add-in.

Good luck.

BUG: Reply-To is broken

Beginning with the May 2019 release, emails sent from an Outlook account that has a Reply-To are instantly rejected, with a message 550 Request failed; Mailbox unavailable. When the Reply-To is removed, or replaced with the real, physical email address, the test email sends correctly.

Prior to May 21, 2019, using an alias email (one that doesn't actually exist, but is redirected to a real email address) in the Reply-To worked as expected, for years. The only purpose of the Reply-To field is to handle this scenario.

I've tried every way I know to report this (sending "feedback", replying to other posts in this forum from the many who are having the same problem, contacting Microsoft support both by chat and by phone). The uservoice forum rejected my post because it wasn't a new feature request. It has not only not been fixed, but it hasn't even been posted as a known issue on the "Fixes or workarounds for recent issues in Outlook for PC" page. I'm hoping that somebody at Microsoft will see this post and take action.

In these days of constant spam, using an aliased Reply-To is the only safe way to send emails to un-trusted companies. Without the alias, the real email address will inevitably get sold to spammers. Therefore this bug has created a security/privacy risk.
