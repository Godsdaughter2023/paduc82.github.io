Hi,

 

you don't need to write code to do this (unless you want to become a programmer by all means ;-).

Instead you should either use a) the Format property of the control or b) Conditional Formatting.

 

a) In design view of the form/report open the property sheet for the text box. 0 [Black];-0 [Red]

 

b) In design view of the form/report click on the text box and from the Format menu choose Conditional Formatting... 

In the FC dialog choose/write:  Field Value   Less than   0

Then choose red as font color from the formatting icons.

hi ange,

when a user creates a recurrence event in a calendar, the calendar creates an item under the calendar list. if someone edit one of those events, a new item will be created in the calendar list.

if you don’t want the recurrence event to be edited by other users, you need to set up an isolated permission on the recurrence item. in the permission management page for the item, you need to first break the inheritance > remove the users who are not expected to have the edit right > re-grant them the read permission. then these users will not be able to edit the recurrence item.

jay freedman say:

Word really doesn't like to do that kind of thing. It considers the table to consist of rows; you can split between one row and the next, but not between columns.

If you have Excel available, the easiest method is to copy the table from Word and paste it into a blank Excel worksheet. Then select the first three columns, copy them, and paste them into the Word document after an empty paragraph. Press Enter after the three-column table so there's at least one empty paragraph there, too. Finally, copy the second set of three columns from Excel and paste them into Word after the empty paragraph (that empty paragraph makes sure that Word doesn't combine the first table and second table into one long table).

To place the two tables side by side, select the two tables (and the empty paragraphs above and between them) and click Page Layout > Columns > Two.

Word automatically inserts section breaks before and after the selected material (the one after the second table is invisible in this shot) so the rest of the document remains as one column.

Access Form Issue: "Control can't be edited; it is - Microsoft Community

intern2016 say:

Access Form Issue: "Control can't be edited; it is bound to an unknown field"

Hi everyone,

I would like your help with this Access survey/form I am working on.

I am designing a form and getting an error message when selecting a value from the Combo Box in Form View:

"Control can't be edited; it's bound to an unknown field 'Gender'.

Gender is a fieldname of the parent table I have. The same error persists with the other fields.

If it helps even a text box I have gives an error: "#Name?"

Can someone point me in the right direction? Thanks in advanced,

ken sheridan say:

There are a number of issues here:

 

1.  The 'lookup wizard' should be avoided like the plague.  For reasons why see:

 

http://www.mvps.org/access/lookupfields.htm

 

2.  Using a value list as the RowSourceType property of a control should only be done where the set of values is immutably fixed in the external world, e.g. days of the week, months of the year.  In your case the values are data and data must be stored in one way, and one way only, as explicit values at column positions in rows in a table.  This is known as The Information Rule.

 

3.  The error which you are experiencing suggest that the ControlSource property of the combo box is Gender, but there is no column (field) named Gender in the table on which the form is based. 

 

So, what you should have is a 'referenced' table named Genders with a short text column named Gender, defined as the table's primary key, and each of your four values in a separate row (record) in the table.  It should also have a GenderIndex column of number data type in which you enter the order in which you want the values to be listed in the combo box in your form.  This column should be indexed uniquely.  So Male would be numbered 10, Female 20. Transgender 30 and No Response 40. The reason for numbering in 10s is that you can later insert any new rows as intermediate numbers between two existing rows if necessary.

 

Your form should have as its RecordSource property the name of the 'referencing' table in which you wish to enter data, or better, a query based on that table, if only for the fact that the query would allow you to sort the rows on one or more columns to determine the order in which they are displayed in the form.  The 'referencing' table must include a foreign key column of short text data type named Gender.

 

You should then create a relationship between the two tables on Gender and enforce referential integrity.  This prevents invalid gender values being entered in the 'referencing' table.

 

In your form a combo box would have Gender as its ControlSource property (i.e. it's bound to the foreign key Gender column), Table/Query as its RowSourceType property, and the following as its RowSource property:

 

SELECT Gender FROM Genders ORDER BY GenderIndex;

 

You don't really need a 'Save Form' button as data in a bound form is automatically saved if you move to another record or close the form.  You can add a 'New Record' button if you wish; the button control wizard can easily set this up for you.
