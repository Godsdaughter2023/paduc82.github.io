It sounds like the "numbers" in column A are actually text.  You can verify that by using formulas of the form =ISNUMBER(A2).

The SUM function ignores cells with text.  But when referenced in arithmetic expressions of the form A2+A3+..., Excel will interpret text as numbers when possible.  For example, if A1 contains ="2" and A2 contains ="3", =A1+A2 returns 5, but =SUM(A1:A2) returns zero.

Exactly how to remedy all this depends on details that I am not clear about.  It might be as simple as changing your REPLACE formula to =--REPLACE(...), where "..." represents the REPLACE parameters that you use.

The double-negative converts text that looks like numbers into actual numeric values, just like any other arithmetic expression would.
