bernie deitrick say:

When finding blocks of data, it is almost always better for start from the bottom and go up: using .End(xlDown) will find a million rows if there is no data below the cell that you are starting with.

Here is how I would code your macro (there is rarely a need to select things in VBA)

Sub CopyANDPasteFillData2()
    
    With Sheets("Sheet1")
        .Range(.Range("A1"), .Cells(.Rows.Count, "E").End(xlUp)).Copy
    End With
    
    With Sheets("Sheet2")
        .Cells(.Rows.Count, "A").End(xlUp)(2).PasteSpecial Paste:=xlPasteValuesAndNumberFormats
    End With
    
    Application.CutCopyMode = False
    
End Sub

I don't have Excel 2003 available right now to confirm this, but the opening sort order in the File, Open dialog is taken from the order that is set within Explorer for Windows 7. This is retained from the time when Excel is initially opened.

Close Excel then open Explorer and set your Documents library to a Date descending sort order. Open Excel and File, Open should show a Date descending order as well. This will be retained as long as Excel is open, i.e. you can go back to Explorer and change the display sort and Excel will retain the original setting until; it is closed and reopened.
