Vlookup formula - Make column index number change with changes in the - Microsoft Community

bobmermans say:

Vlookup formula - Make column index number change with changes in the source table?

Hi there

I have an excel database for my own company which I use for calculating prices etc. For each project there's a new line with several columns and to make my offers or invoices, I use the vlookup function based on the project number.

But now if I want to add extra information or columns in the source table (for instance an extra column for measurements or...), I have to change the column index number in each formula in my offers or invoices in order to get the right information in the right cells there.

Is there an option to make the column index number vary with the source table? For example the column index number for a certain formula is 10, and in the source table I add a new column before this column, so in the formula it automatically changes to 11 without any action from me?

Thanks a lot!

Bob

bobgreenblatt say: You could compute the column index number by MATCH of the column name to the first row of you table. So, Instead of using 10 for the column index number in VLOOKUP, you could instead use:
MATCH("Price",rangeofthetableheaders,0)

j.e. mcgimpsey say: A minor extension of Bob's excellent suggestion:

Since you have a multiple formulas, you could use defined names to reduce the number of calculations. For instance:

  • Define your table's name by highlighting it (or its columns) and typing the name in the Name box (at the left of the formula bar), say, MyTable
  • Using Insert/Name/Define, define a name for your lookup column, using Bob's suggested MATCH() function:  
               Name in workbook:     Price_Column
               Refers to:                   =MATCH("Price", INDEX(MyTable, 1, ), 0)
 
  • Now you can use this formula:
            =VLOOKUP(A1, MyTable, Price_Column, FALSE)

and if you insert a column in MyTable to the left of your "Price" column, the formula(e) results will automatically adjust, and the lookup column will only be calculated once.

An additional advantage is that the formula is now more self-documenting, making it easier to understand when you go back six months from now to modify it...

Nick

john korchok say:

If you insert a Content placeholder on a layout, when a picture is inserted the placeholder will be resized  so the entire picture will display. This is the best choice if you want to ensure that all of the picture displays.

If you insert a Picture placeholder on a layout, the placeholder will always remain the same size. The smaller dimension of the picture will be enlarged and the larger dimension will be cropped. This is a better choice when you have a particular picture arrangement that you want maintained whatever content is used.
