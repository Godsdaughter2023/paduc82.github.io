Sharepoint Wiki page 'Edit Source' prettifies HTML incorrectly - Microsoft Community

samshannon say:

Sharepoint Wiki page 'Edit Source' prettifies HTML incorrectly

Just one example is the <span> tag.

The 'Edit Source' button on the Sharepoint Wiki Page Editor will prettify span tags to go on their own line, regardless of their previous/correct layout.

Because the span tag does not inherently imply or represent a type of layout or spacing (like 'div', and unlike 'p' or 'body'), it is commonly used in scenarios where it should not be split out onto it's own line (inline).

One example of this is something like http://hilite.me/which developers use to 'prettify' their code in online documentation.
If 'Edit Source' is used even on some unrelated section of that page, the inserted HTML is prettified incorrectly and it actually breaks the markup because those newlines are significant to the output.

Which is not good at all.

And if the 'Edit Source' button is clicked AGAIN...


As much as I appreciate the effort that has gone into making 'Edit Source' more friendly to use by trying to prettify it.
