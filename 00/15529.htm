One month and a day – finally figured this out – by pure serendipity!

 

Apologies if this explanation isn’t as techie accurate as others would like but this is how understand the situation.

 

Turning on DHCP Client is a facility within Windows to force automatic IP address generation. This forces recognition of your Windows Mobile device. And it works! Problem is if you have a peer to peer network using a "residential gateway" i.e. a wireless router and an ethernet hub, these pieces of kit don’t work that way.

 

The solution that I’ve found, and has worked for all of 6 hours, is to turn on DHCP Client and then specify the Internet Protocol (TCP/IP) DNS Server address of each computer on your network with the exception of the Windows Mobile device. In my case Preferred Address 194.117.200.10, Alternative Address 194.117.200.15. Working so far anyway. Obviously might be different for other kit.

 

My conclusion of the Microsoft fix is that it is assumes a single PC situation and takes no account of modern family

homes wired peer to peer. Also all of the foregoing doesn’t seem to have any bearing on the version of Outlook you are running, at least between 2002, 2003, & 2007 as has been suggested elsewhere.
