ken sheridan say: FWIW this is how I approach it:

The following function in the same module as the calling procedure returns a reference to the Word Application object:

Private Function GetWordApp() As Object

    ' if Word open return reference to it
    ' else establish reference to it
    On Error Resume Next
    Set GetWordApp = GetObject(, "Word.Application")
    If Err.Number = 429 Then
        Set GetWordApp = CreateObject("Word.Application")
    End If

End Function

Then in the calling procedure a reference to the Word document is returned like this:

    Dim wApp As Object
    Dim wDoc As Object
    Dim wActiveDoc As Word.Document

    Set wApp = GetWordApp()
    'open word merge document
    Set wDoc = wApp.Documents.Open("C:\Alaska\letters\Certificate Labels.doc")

In my case rather than the literal path I reference an argument by which the path is passed into the procedure, but that's not relevant to the current issue.

Then in the same procedure the merge is executed to the printer with:

    ' execute merge
    With wDoc.MailMerge
        .OpenDataSource strDataDoc
        .SuppressBlankLines = blnSuppressBlankLines
        .Destination = wdSendToNewDocument
        .Execute
    End With
    
    ' print the document
    wApp.ActiveDocument.PrintOut Background:=False
    
    ' close Word documents

Re: (a) - Outlook contact folders and AddressBook (singular on purpose)

The Outlook Address Book is a "specialized view" of the contact folder(s) that have been set to be a member of the AB. The default contact folder for the profile is automatically set to be a member of the AB.

Default contact/calendar folders are stored in the default data file for the profile which if you only had one IMAP email account configured, those folders would be stored in the OST file for the profile. Going to Outlook menu --> File --> Account Settings --> Data FIles tab --> shows the data file set as the default. There MUST always be a default data file set and with only one IMAP account with no other PST files configured, that would be an OST file.

Re: (b) - "This Computer Only"

Means exactly that. Here's the problem with those folders, <IF> Outlook cannot find/open the OST file associated with the account (for any reason) to which it belongs, the file is recreated and the email folders are sync'd with the server. Contact/Calendar folders would be empty. "This Computer Only" folders ONLY apply to IMAP accounts.

An OST file can only be opened by the Outlook profile/email account that originally created it. Backing up an .ost file separately would require a 3rd party OST2PST conversion utility.

Re: (c) - PST or OST

All depends on the configuration with particular emphasis on what's set as the "default data file" and the caveat for "This Computer Only" folders in a recreated OST file as noted in item (b).

If you only have one email account configured AND no other data files added then for

POP3 - located in a PST file
IMAP - located in an OST file with "This Computer Only" folders
Exchange - located in an OST file where everything including contacts/calendar items get sync'd with the server

bob jones aka: cybertaz say:

Master Pages [in this context only] simply are visual representation of Headers & Footers... They do not exist independently of the pages on which they appear, so...

  1. Make sure you're on the All Contents tab of the document window
  2. View the page associated with the unwanted Master Page
  3. In the Insert menu select Remove Page

The Master will be removed as well because it cannot exist by itself.
