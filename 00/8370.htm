Set the Force New Page property of the group header to None.

Set the Force New Page property of the group footer to After Section.

Add a new group in the Group, Sort and Total pane on the same field mbr_ID.

Specify that this group should have a group footer but not a group header.

Set the Force New Page of the new group footer to After Section.

Create an On Format event procedure for this group footer. Assuming it's named GroupFooter1:

 

Private Sub GroupFooter1_Format(Cancel As Integer, FormatCount As Integer)
    Dim bIsOdd As Boolean
    bIsOdd = (Me.Page Mod -2)   'Yields 0 for even, or -1 for odd.
    'Hide this Section (and its page break) if already at odd page.
    With Me.Section("GroupFooter1")
        If .Visible = bIsOdd Then
            .Visible = Not bIsOdd
        End If
    End With
End Sub

    Else
       GrpPage = 1
       GrpArrayPage(Me.Page) = GrpPage
       GrpArrayPages(Me.Page) = GrpPage
    End If
Else
  Me!ctlGrpPages = "Page " & GrpArrayPage(Me.Page) & " of " & GrpArrayPages(Me.Page)
End If

End Sub

a. user say:

Well...I did find out a couple of things that might help, hopefully:

  • Somehow, and don't ask me how, different portions of my table were somehow being subjected to overlapping conditional formatting rules (i.e. instead of having one series of rules covering the entire table, I'd have multiple instances of the rule series that applied to different parts of the table).
  • If I copied a table or part of a table (with formatting intact) to another spreadsheet, it didn't crash quite as frequently...combined with the first item, I did manage to mitigate the issue somewhat

What I'd recommend trying is this:

  1. First, create a duplicate of the XLSX file you're working on, name it something different, and remove *all* formatting from it.  Every single sheet.
  2. Now open your original spreadsheet, the one with formatting intact.  If you have the same formatting rules in place across your entire spreadsheet, copy one cell
  3. In the duplicate workbook you just created, select the cell range that you want to be affected by the formatting of the cell you just copied.
  4. Right-click and paste the formatting (and the formatting only, no content) from the single cell across your entire table.
  5. Save your spreadsheet immediately with a new name, and close everything.
  6. Re-open the spreadsheet you just pasted formatting into.
  7. Open the conditional formatting manager, and verify that the active range is correct and covers your entire table without any duplication.

You should have an easier time now, although I can't promise it won't continue to crash, so I advise to continue to save often, and have Excel save for you every couple of minutes.

I'll hasten to add that this solution assumes you have the exact same formatting rules in place across your entire table.  If there are a variety of different rules for different areas, you can still use this fix, you'll just have to repeat it for every effective range you have.
