Microsoft Office Word 2007 - Microsoft Community

jennawinterburn say:

Microsoft Office Word 2007

I have Microsoft office word 2007 and my documents are all in landscape mode. When I try to change to portrait it comes up with a message about margins. Also when I open a saved document - it only shows me the first page despite there been several pages?
suzanne s. barnhill say:

Both of these issues are symptoms of the same problem, which is a problem with the settings for columns. If the "Equal column width" box is not checked in the Columns dialog, then column width (even if there's just a single column) doesn't adjust automatically but is preserved at the current setting. If you have a Letter-sized document in Landscape orientation with 1" margins left and right, then the width between margins (your column width) is 9". When you try to change to Portrait orientation (that is, to a page width of 8.5"), obviously Word is going to balk.

 

At the same time, sometimes Word will get it into its head that you have more than one column, but the first column is set to the entire margin width, so of course there's nowhere to display the second column, and its content disappears.

 

The solution to both problems is the same. You can't check the "Equal column width" box yourself, but you can force Word to do it. In one of your problem documents, select the Page Layout tab and, in the Page Setup group, click on Columns and choose More Columns..., which opens the Columns dialog. In the dialog, even if the number of columns is already set to 1, click the preset picture for One. This will check the "Equal column width" box. Click OK.

 

If you find that this solves the problem in one document, you can use the same technique in other saved documents that exhibit the same problem. To prevent the problem in new documents, you will need to edit the Normal template. To do that, you have to open the template for editing. The easiest way to do that is as follows:

 

In Word, press Alt+F11; the Visual Basic Editor opens. Press Ctrl+G to activate the Immediate window. In that window, type

     NormalTemplate.OpenAsDocument

and press Enter. Normal opens, but note that it may not be on top of all windows. You can close the Visual Basic Editor. Make the desired changes to the Normal file, and then save and close it. If the changes you have made have not been recognized as changes (the Undo button is not activated), press Spacebar, then Backspace to "dirty" the file, thus ensuring that the template will actually be saved.

Internet Explorer (IE11) has fallen behind other browsers in terms of performance and this won't be changing. IE has more trouble, in particular, with JavaScript heavy pages. Here are some preventive maintenance tips which might help some.

1. Once per day...

 Internet Options> General tab> Browsing History> Delete> uncheck "Preserve Favorites website data" then delete Temporary Internet Files. No reason, yet, to delete cookies, history,  passwords, etc.

2. Internet Options> General tab> Browsing History> Settings>

Set cache size (Disk space to use) at 200-250 MB. A larger cache won't speed up browsing.

3. When the problem occurs or any time it's convenient, close all iexplore processes from Task Manager (Ctrl + Alt + Del> Task Manager) IE can use a lot of memory, especially after viewing JavaScript heavy pages. This will get IE's memory usage back to an acceptable level.

4. Shutdown at night or restart at least once per day. Some users use sleep or hibernate, instead of shutting down, but Windows still, after all these years, likes an occasional restart.

5. Other browsers handle these JavaScript heavy pages better than IE11. On pages that consistently cause this freezing, locking up problem, use another browser, Chrome, Firefox, Edge, Opera, or another.
