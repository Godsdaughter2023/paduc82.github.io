john - project say:

Michael,

Okay, this is how it plays out.

Ordinarily you should be able to modify the Bar Styles for the Resource Graph to only show for selected tasks and then apply that to your filtered view in the master file. However, Project 2010 has a bug wherein making that bar style change in a dynamic master will cause Project to crash. I confirmed this with a contact at Microsoft and unfortunately hot fixes for Project 2010 are no longer being made. This issue does not exist in Project 2013 or 2016.

So what can you do for Project 2010? Instead of creating a dynamic master, create a static master. Do that by unchecking the "link to project" option in the lower right corner of the Insert Project window. With a static master you will need to have a custom field to identify each subproject that makes up the master. Now you can create your filter and modify the Resource Graph Bar Styles to only show resources associated with the filtered project.

Hope this helps.

John

Hi Greg,

We can't directly sync the public folder calendar to the user's default calendar.

But, it comes to my mind that you may use a Windows Outlook client to export the calendar data of your public folder into .PST file firstly.
