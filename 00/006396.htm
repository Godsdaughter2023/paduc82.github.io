johnc87 say:

sum cells ignoring #N/A

I am trying to sum a range of 4 cells that sometimes will contain #N/A purposely. However to compute a meaningful total I need to be able to ignor the #N/A in the sum. I am trying the following formula however it produces the #VALUE! error.

=SUMIF((C10,D10,F10,G10),"<>#NA")

Can anyone see what I'm doing wrong?

jllatham say:

EDIT: See JohnC87's own solution below - the AGGREGATE() function handles this extremely well. His final formula:  =AGGREGATE(9,6,(C10,D10,F10,G10))

And he notes that you can find functional description and use of parameters here:

http://office.microsoft.com/en-us/excel-help/aggregate-function-HA010338704.aspx

=====================================

 

 

I don't think you're doing anything wrong other than asking Excel to do something that it apparently cannot.

If your range to SUMIF() was contiguous from C10 through G10, it would work:
=SUMIF(C10:G10,"<>#N/A")

actually works (note #N slash A, not #NA)

But it doesn't work for the physically separated cells.  So I think you're going to have to rewrite it to examine the cells individually.

The most primitive way would be:

=IF(ISNA(C10),0,C10)+IF(ISNA(D10),0,D10)+IF(ISNA(F10),0,F10)+IF(ISNA(G10),0,G10)

a little better, as far as length goes would be:

=SUMIF(C10:D10,"<>#N/A")+SUMIF(F10:G10,"<>#N/A")

or sum the sumifs:

=SUM(SUMIF(C10:D10,"<>#N/A"),SUMIF(F10:G10,"<>#N/A"))

but I really don't see a big gain using the last one when compared to the previous SUMIF()+SUMIF() version.

BUT!!

If E10 is not numeric, you could include it and use:

=SUMIF(C10:G10,"<>#N/A")

since SUM and SUMIF() ignore text and empty entries.

 

I could even think of another variation or two to deal with E10 being numeric:

=SUMIF(C10:G10,"<>#N/A")-E10

or if there's possibility of E10 holding some odd ball value (even #N/A), then perhaps

=SUMIF(C10:G10,"<>#N/A") - IF(ISNUMBER(E10),E10,0)

The variations of how to deal with E10 in this case really depends on the possible values for E10 including any error indications.  I actually think that
=SUMIF(C10:D10,"<>#N/A")+SUMIF(F10:G10,"<>#N/A")
is probably the best option of the ones I've offered.

This usually indicates that the document still contains some markup. Automatically update a picture - Microsoft Community

smurf06 say:

Automatically update a picture

I have a picture and i want it repeated throughout the document in certain places. Have used quick parts for words but do not know what to use for pictures.
doug robbins - mvp office apps & services (word) say:

To provide a place for the user to insert the picture, insert a single cell table into your document and set the row height and width to suit the picture, setting the Table options so that it does not automatically re-size to fit the contents, remove the borders (Ctrl+Alt+u) if desired, and then select the table and use the Insert>Bookmark facility to assign a bookmark to it.  Then at the other locations at which you want the picture to appear, insert a cross reference to the text of the bookmark.

Then when a picture is inserted into the original table and the cross references are updated, the picture will appear in all of them.

The reason for using the table is that if the whole table is bookmarked, the insertion of the picture into the table will not result in the bookmark being destroyed.

You could just insert the picture into a bookmark, but if that is not done correctly, the bookmark will be destroyed in the process and then the cross-references will not work.

-- Hope this helps.
