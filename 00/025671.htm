m4e_team_k say: I'm having trouble following your description, but it may help you to know the following:

MfE does not modify calendar items or create calendar items in this case. It only keeps items in sync. These items may be modified/created by the calendar app on the device, or by using an Exchange client like Outlook.

Dismissing the alarm will should not modify the calendar in my experience, but it's definitely not MfE that's modifying it.

It's a bad idea to use multiple applications to sync the same collection. You should not use MfE and PC Sync to keep the calendar in sync on the same device. This will result in duplicates. Is there any reason to do this?

It's ok to use PC Sync for contacts and MfE for Calendar, or other combinations like this, but please don't use multiple apps with the same collection.

Hi,

 

XSS Filter stands for Cross-Site Scripting (XSS). As both are the same you cannot disable cross-site scripting without using XSS Filter.

 

What is Cross-Site Scripting?

 

Cross-Site Scripting can help prevent one website from adding script code to another website. XSS Filter watches how websites interact, and when it recognizes a potential attack, it will automatically block script code from running. When this happens, you will see a message in the Information bar letting you know that the webpage was modified to help protect your privacy and security. This is like a security warning for you.

 

Note: It is not recommended to turn off the XSS Filter in Internet Explorer 9. Doing so will leave you vulnerable to cross-site scripting attacks as explained above.

 

If you still want to turn off the XSS Filter, refer to the below steps:

 

a.        Open Internet Explorer.

b.       Click Tools and then Internet Options.

c.        Select the Security Tab. Click Internet.

d.       Select Custom level. Scroll to the Bottom of the List.

e.        Click Disable under "Enable XSS Filter."
