Importing email folders from Webstarts webmail to Outlook 2010 - Microsoft Community

wkcz31 say:

Importing email folders from Webstarts webmail to Outlook 2010

I linked my Outlook 2010 with my Webstarts web-mail (POP3) (SMTP) and it is working great. Outlook imported my Inbox, Sent, Deleted,, Calendar.. etc. But I also have a few email folders that I created at the web-mail that didn't transfer over my Outlook. I thought they should be automatically imported... Any idea on how to import them over my 2010 Outlook?

Thanks. Really appreciate your help.

brian tillman [outlook mvp 2007-2019] say:

The POP protocol doesn't know anything about folders.  The only thing it can do is to download messages from the server-side Inbox.  While

This assumes you can use a web browser to access your online mailbox.

First, in Outlook, create a new folder.  You can name it anything you wish because it will be temporary.  Now, move the contents of the Outlook Inbox to that folder so that Outlook's Inbox is empty.

Next, log into your mailbox using a web browser.  Create a temporary folder there as well and move everything in the server's Inbox to that folder.

Now, for each folder you have on the server, copy the contents of the folder to the server's Inbox.  In Outlook, perform a send/receive, which should download the contents of the Inbox to Outlook's Inbox.  In Outlook, create a new folder whose name matches the server-side folder name and move the contents of Outlook's Inbox to that new folder.  Mark everything as read, if you wish.

Go back to the server, empty the Inbox and copy the contents of the next folder to the server Inbox and download it to Outlook as before, moving it to a matching folder there, just as you did for the first folder.

Do this for every mail folder you have on the server and eventually you'll have transferred everything.  When you have downloaded all of the messages, on the server move the contents of the temporary folder you created at the start of the process back to the server Inbox.  Outlook will download all of those messages one last time into the Inbox of Outlook.  Now you should be able to delete the temporary folder on the server and in Outlook because both folder sets will match.

Note that downloading messages from the server requires that the server and Outlook cooperate in deciding which messages are new and which have been downloaded before.  When you move messages into the server's Inbox from the other folders, some servers will consider them new (which is what you want) and when Outlook asks for new messages, the server will give them to Outlook.  Some servers, however, may remember that the messages you moved to other folders are no longer new and even if you move them back to the server Inbox, the server will not offer them to Outlook when it asks for new messages.  Usually, you can overcome this with a server-side setting, but sometimes not and if you can't, then this scheme simply won't work and IMAP is your only choice.
