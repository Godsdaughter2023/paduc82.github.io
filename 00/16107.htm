Lost Macros - Microsoft Community

teresae1035 say:

Lost Macros

When I opened Word this morning, all of my macros were gone.  We haven't changed systems or upgraded to a new version of Word.  Any suggestions on ways to recover them?  Thanks
jay freedman say:

I'll guess that, like most people, you never write macros from scratch but only use the macro recorder; and that you always accept the recorder's suggestion to save the macro in the Normal.dotm template.

If that's true, then the most likely cause of the macros' disappearance is that the Normal.dotm file that contains the macros was moved or deleted. When that happens, and you start Word when it can't find that file, Word creates a new Normal.dotm with no customizations.

To try to recover, close Word and then start Windows Explorer. In the address bar at the top, type or copy/paste the text %appdata%\Microsoft\Templates and press Enter. Look in that folder for a file named "Backup of Normal.wbk". If you find it, and if its modified date is yesterday or earlier, it will probably contain the macros.

In that case, look for the Normal.dotm file in the same folder, and delete it if you find it. Then rename "Backup of Normal.wbk" to "Normal.dotm". Restart Word, and see if the macros are back in place.

If you don't find the Backup file, or if you follow the procedure and it doesn't restore the macros, the only other suggestion is to try a program called Recuva to see if it can find an old deleted copy of Normal.dotm. Otherwise, you'll have to re-record your macros as best you can.

To avoid a repeat, it's best to store your macros in a custom template that you store in the folder %appdata%\Microsoft\Word\Startup. Templates in that location are automatically loaded every time Word starts, and macros in those templates are available in every document. They're also much less likely to be changed or deleted accidentally.

jllatham say:

What Jeeped has recommended is probably the typical way to go, although if your lists are long, then it can make for lots of entries and if they are not numeric and sequential, there's probably a good possibility of missing a combination somewhere.

An alternative is to set your data up on Sheet1 like an old fashioned multiplication table:

Then on the other sheet, you use cells A1:A11 as the source for data validation in cell A1 of that sheet.

You use cells A1:K1 as the source for data validation in cell B1 of that sheet.

In cell C1 on the second sheet (where you have data validation) you put a formula like this:

=INDEX(Sheet1!$A$1:$K$11,MATCH(A1,Sheet1!$A$1:$A$11,0),MATCH(B1,Sheet1!$A$1:$K$1,0))

The MATCH(A1,Sheet1!$A$1:$A$11,0), portion will get the row number for the table,

the MATCH(B1,Sheet1!$A$1:$K$1,0) portion gets the column number for the table, and

the INDEX(Sheet1!$A$1:$K$11  part of it is identifying the entire table to get the data from.

So this results in something like this on the data validated sheet:

Now, the obvious difficult part of this (or the tedious part) is filling in all of the values where your two lists intersect!  But it definitely covers all combinations, no matter how long or short either of the lists is.  Mine was easy to set up because of the math relationship between the values in the list:

I put this formula into B2  =$A2*B$1  and simply filled it down and over as needed.

jllatham say:

Thanks for your feedback.  Glad my suggestion helped.

As for the #N/A! issue, just wrap your formula in an 'error trap' like this:

=IFERROR(INDEX(Sheet3!$D$1:$J$6,MATCH(C3,Sheet3!$D$1:$D$6,0),MATCH(D3,Sheet3!$D$1:$J$1,0)),"")

or one of these variations of that:

=IFERROR(INDEX(Sheet3!$D$1:$J$6,MATCH(C3,Sheet3!$D$1:$D$6,0),MATCH(D3,Sheet3!$D$1:$J$1,0)),"No Match")

or  and since you are using the results in a SUM() formula, this is probably the best way to go

=IFERROR(INDEX(Sheet3!$D$1:$J$6,MATCH(C3,Sheet3!$D$1:$D$6,0),MATCH(D3,Sheet3!$D$1:$J$1,0)),0)

The question is, I suppose, why you are getting the #N/A! -- those will only come up if one or both of the MATCH() parts of the formula do not find a match.  If that's possible, then fine, otherwise not so fine.

tonyd123rx say:

Paragraph number converting auto number to fixed numbers

Is there a way of converting automatically generated numbers into fixed numbers that do not change again?

What I mean by way of example: I want to quote some text extracted from another document in my document.. The other document has automatically generated 

has automatically generated para numbers. The section which I quote starts with para 30. I want to keep that same para number in my new document. Apart from typing "30" is there a way of freezing that number 30 and the para numbers that follow in my quotation? 
graham mayor say:

Select the numbered section of text in the source document.

Run the following macro

Sub CopyWithFixedNumbers()
Dim oDoc As Document
Dim oRng As Range
    Set oDoc = ActiveDocument
    oDoc.ConvertNumbersToText
    Set oRng = Selection.Range
    oRng.Copy
    oDoc.Undo 1
End Sub

then paste the clipboard content to the target document.
