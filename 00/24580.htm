The Relationship should be gone!

 

I think what you are seeing is the Subdatasheet settings in the fields with the + signs.  Those are often generated automatically and, IMO, it's a real pain.  I recommend setting every field's SubdatasheetName property to [None] unless you really understand and need to use the feature.

ken sheridan say: To execute the query from multiple instances of the form you'll need to build the SQL statement in code in the form's module, concatenating the values of the controls into the string expression:

Dim strSQL As String

strSQL = "INSERT INTO tblEventDetail ( EntryID, Unit, Location, Room, Address, " & _
    "City, CallType, CaseNumber, Code, Description, Dispatcher, Source, [User], " & _
     "Disposition, PrimaryOfficer, Supervisor ) " & _
     "VALUES(" & Me.txtEventNumber  & ", """ & Me.txtUnit & """, " & _
      <and so on>                                                                                                          & ")"

CurrentDb.Execute strSQL, dbFailOnError

In the above I've assumed that EntryID is a number data type and Unit is a text data type in order to illustrate how the latter's value is delimited with literal quotes characters.

The Me keyword refers to the current instance of the class in which the code is running, so will refer to each instance of the form separately, unlike your reference to the form as a member of the Forms collection.

Here's an example of a module with code to open/close multiple instances of a form, in this case one per company, and stack the forms diagonally down the screen:

Option Compare Database
Option Explicit

Dim frm As Form
Dim colPopups As New Collection

Function CloseCompanyForms()

   Dim n As Long

   For n = 1 To colPopups.Count
       DoCmd.Close acForm, "frmCompanies"
   Next n

End Function

Function StackCompanyForms()

   Const conSQL = "SELECT * FROM Companies Order By Company"

   Dim dbs As DAO.Database, rst As DAO.Recordset
   Dim strSQL As String
   Dim n As Integer

   Set dbs = CurrentDb
   Set rst = dbs.OpenRecordset(conSQL)

   Do While Not rst.EOF
       n = n + 1
       Set frm = New Form_frmCompanies
       colPopups.Add frm, frm.hwnd & ""
       frm.Caption = rst.Fields("Company")
       frm.Filter = "CompanyID = " & rst.Fields("CompanyID")
       frm.FilterOn = True
       frm.Visible = True
       DoCmd.MoveSize n * 360, n * 360
       rst.MoveNext
   Loop

End Function

What should I  do.

jimgraync say:

Here is what worked for me (thanks to a reply from a forum person who gave me Excel 2010 instructions. Excel 2013 is a bit different, but I figured it out):

1. Open the workbook and enter the password to open the file.

2. Click the FILE tab and select INFO

3. Click the dropdown arrow in the PROTECT WORKBOOK box.

4. Click the ENCRYPT WITH PASSWORD option.

5. A box will pop up with the password shown by dots (........). Clear the password dots.

6. SAVE or SAVE AS the file and the password will be gone.

 

After that you should be able to open the file without entering a password.

JG
