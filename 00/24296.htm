FIX | Adobe Reader DC & Outlook 2016 | "No default - Microsoft Community

jayson hansen say:

FIX | Adobe Reader DC & Outlook 2016 | "No default client configured" when using "Send to Email" button

Hello all,

Just wanted to provide this fix to anyone that might be running into this issue:

SYMPTOM:

When viewing a PDF from within Adobe Reader DC, one tries to use the "Send to Mail" button (looks like an envelope) to easily attach the PDF to a newly-composed message (without having to manually perform this), but receives an error that states something along the lines of "no default client set" (even if Outlook 2016 has been set as the default mail client within Default Programs).

FIX:

Condition: One must be logged in with an account that has administrative privileges.

  1. Open 'regedit' (Windows key + R, then type in "regedit" and press the Enter key or OK).
  2. It is a good idea to perform a backup of your registry just in case you mess up or things go haywire for whatever reason.
    1. File -> Export -> Save as whatever file name you want, to wherever you want to back it up to.
  3. Travel to "HKLM\SOFTWARE\Clients\Mail\".
  4. Delete the folder entitled "Microsoft Outlook" from within the directory navigated to in step 2. Yes, the entire folder.
  5. Close regedit.
  6. Open up Control Panel.
  7. Go to Programs and Features.
  8. Repair your Office 2016/365 installation. Perform a quick repair, online repair is not necessary.
  9. Once complete, reboot your computer and test out PDFs sending to Outlook via the button.

Please provide feedback in the comments section to let me know how this works out for you.

If you need to revert any changes you made, you have the backup of your registry from step 2. I do not hold any responsibility to any potential damage that may occur from anyone performing these steps; you are following these at your own discretion and risk!

The fiscal month begins on 22nd of the month through the 21st of the following month. For example if my date is 8/21/13; my formula should return Aug-13; if my date is 8/22/13, my formula should return Sep-13 since 8/22 is already after the August cut off. More examples below

 

           Column A   Column B

8/29/2008      Sep-08 8/29 after the 21st cut off; already falls in the month of Sept
       5/22/2009        Jun-09 5/22 after the 21st cut off; already falls in the month of June
7/9/2009       Jul-09 within the July cut off
9/21/2009     Sep-09 within the Sept. cut off

JB

ronrosenfeld say: =DATE(YEAR(A1),MONTH(A1)+(DAY(A1)>21),1)

returns a date equal to the 1st of the fiscal month.  You can either custom format this as mmm-yy

or you could return the text string with:
=DATE(YEAR(A1),MONTH(A1)+(DAY(A1)>21),1)

ronrosenfeld say:

You're welcome.

Here are shorter versions, but requires the Analysis ToolPak to be installed in versions of Excel prior to 2007:

 

=EOMONTH(A1,--(DAY(A1)>21))

 

=TEXT(EOMONTH(A1,--(DAY(A1)>21)),"mmm-yy")
