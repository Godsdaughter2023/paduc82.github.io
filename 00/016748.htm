Saving a Power Point 2016 Custom Show in pptx format - Microsoft Community

r. keith howard say:

Saving a Power Point 2016 Custom Show in pptx format

Hello,

If I have a pptx file that has 2 custom shows:

Show 1: slides 1 and 3

Show 2: slides 2 and 3

How can I save Show 1 into a new pptx file that has slides 1 and 3 from the original file?

Many thanks.

Keith

steve rindsberg say:

Have a go with this.  It'll save the slides from each of your custom shows out to a separate presentation.  Watch out for linebreaks that the forum editor "helpfully" introduces.

Option Explicit

Sub WhoIsInSlideShow()
    
    Dim oPres As Presentation
    Dim oCopyPres As Presentation
    Dim lShowNum As Long
    Dim x As Long
    
    Set oPres = ActivePresentation
    
    With oPres
        ' Don't do anything if there are no custom shows!
        If .SlideShowSettings.NamedSlideShows.Count > 0 Then
        
            ' We have shows; work with each of them in turn:
            For lShowNum = 1 To .SlideShowSettings.NamedSlideShows.Count
                
                oPres.SaveCopyAs oPres.Path & "\" & "SHOW_" & CStr(lShowNum) & "_" & oPres.Name
                Set oCopyPres = Presentations.Open(oPres.Path & "\" & "SHOW_" & CStr(lShowNum) & "_" & oPres.Name)
        
                
                With oCopyPres
                    For x = .Slides.Count To 1 Step -1
                        ' is this slide part of the current custom show?
                        ' if not, delete it
                        If Not IsInCustomShow(.Slides(x).SlideID, lShowNum) Then
                            .Slides(x).Delete
                        End If
                    Next
                    .Save
                    '.Close
                End With    ' copy presentation
                
            Next    ' custom show
        End If  ' has custom shows
    End With   ' oPres, current presentation

End Sub

Function IsInCustomShow(lSlideID As Long, lShowNum As Long) As Boolean
    Dim x As Long
    For x = 1 To UBound(ActivePresentation.SlideShowSettings.NamedSlideShows(lShowNum).SlideIDs)
        If lSlideID = ActivePresentation.SlideShowSettings.NamedSlideShows(lShowNum).SlideIDs(x) Then
            IsInCustomShow = True
            Exit Function
        End If
    Next
End Function

excel auto day fill in - Microsoft Community

a. user say:

excel auto day fill in

Im trying to create the formula for when I type the date, in the next cell the day of the week will auto populate.

hansv mvp say:

Let's say you enter a date in A1.

You have several options for a formula in B1:

  1. Enter   =A1   in B1, and set the number format of B1 to the custom format dddd. With this option, the value of cell B1 will be the date, but it will be displayed as the day of the week.
  2. Enter   =TEXT(A1,"dddd")    in B1, and leave the number format of B1 set to General. With this option, the value of B1 will be the day of the week.

Remark: these options will result in the full day of the week (e.g. Wednesday). If you prefer abbreviations such as Mon and Tue, use ddd instead of dddd.

a. user say:

thank you for your help, I only have to type this in the first field and it will continue to do this function for the next cells down the line?

hansv mvp say:

You'll have to copy the formula down the first few times. From the fifth row with a date down, Excel should automatically create the formula in the cell to the right.

If you prefer, you can prefill the formulas as far down as you want. To avoid seeing a day in rows where you haven't entered a date yet, use

=IF(A1="","",A1)

or

=IF(A1="","",TEXT(A1,"dddd"))

john p spencer say:

Yeah.  You can get the actual SQL statement in the VBA window.  In the immediate window enter

? CurrentDb().QueryDefs("Name of the query").SQL

Once that prints in the immediate window, you can copy it and then open the query in design view and switch to SQL View.  Once there you can either fix the join expression and save the query in SQL view or paste the SQL string and save in SQL view.  NEVER go back to design view on this query.

In Excel 2010 and WinXP (my versions), the two return exactly the same, namely:

[Errata....  I can only assume the Calculator gets the same result to

Excel formats only up to the first 15 significant digits, rounding the 16th.  So we see 3.14159265358978.

Windows Calculator formats up to the first 32 significant digits, rounding the 33nd.  So we see 3.1415926535897824578569270670414.

(I might have changed some Calculator options in order to see so many digits.  I don't remember.)

In any case, infinitesimal differences are certainly possible, especially when there is more than one arithmetic operation.  There can be several reasons.  Among them:

1. Differences in binary representation of stored numbers.  64-bit floating-point is the most common; it is the native binary form for most computers.  But some computers support other binary forms.  And we can always implement different forms in software.

2. Differences in precision of binary computation and intermediate results of subexpressions.  In Intel-compatible CPUs, binary arithmetic is performed using an 80-bit floating-point form; intermediate results in that form are available to software.  That is why Excel and VBA sometimes have different results:  VBA tries to use the 80-bit intermediate results, whereas Excel rounds each subexpression to the 64-bit form.  And again, we can always do things differently in software.

Computer arithmetic differs from mathematics because mathmetical calculations have an unbounded number of digits, whereas computer arithmetic has a limited number of digits, even if it is extensible.  (The amount of memory is finite.)

In this case, the difference is only 1 in the least-significant bit.  But it could be a much greater difference, as demonstrated by the table below.

I suspect that EXACT compares numeric values rounded to 15 significant digits.

That makes sense since EXACT really "[c]ompares two text strings", not numeric values.

The proper way to compare two numeric representations is:  =A1-A2-0, formatted as Scientific.  [EDIT] Or MATCH(A1,A2,0) or =(A1-A2=0) instead of =(A1=A2).

The redundant -0 works around a dubious Excel heuristic that sometimes replaces the exact difference with exactly zero when the difference is "close enough". 

The operative word is "sometimes".  =SUM(A1,-A2) returns the same exact difference as =A1-A2-0.  But if A2 is =-PI(), =SUM(A1:A2) returns exactly zero.  Caveat lector!

The formula in column B is of the form =$B$13+A1*2^-51.

The formula in column C is of the form =EXACT(B1,$B$13)

The value in B13 is the constant 3.14159265358979.

Columns D and E display the exact representation, in binary (hex) and decimal, of the value in column B.
