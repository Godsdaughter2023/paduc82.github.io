Does MSE scan the whole drive and hidden directories and do I need to - Microsoft Community

tonysaprano say:

Does MSE scan the whole drive and hidden directories and do I need to execute it as Admin?

Does MSE scan the whole drive and hidden directories..Like the "System volume information" ????
Do you need to execute MSE as Admin to get the program to scan the whole drive? What about the MBR ? and
Memory ?
Thanks for listening  

Original title: MSE Scanning Questions

kosh vorlon say:
Hi Tony,
 
Manual and scheduled scans through MSE work the same way.
 
A Full Scan can scan all attached drives and partitions (and memory and hidden folders and rootkits).  It does scan for many rootkits, but frankly needs work or help removing some of them.  To get some parts (like Sys Vol) however, you may need to configure Settings / Advanced. Check the boxes there for Scan Archive Files and/or Scan Removable Drives (this is for flash drives and such - and attached external HDD will be included without needing to check this).
 
You do not need to be an Admin to run any scan and get what the scan covers, but you do if you need to change the Settings.  Also, if you want non-Admins to see all the scan history results, you need to configure Settings / Advanced and check the box to allow users to view the full history results.
 
 
A Quick scan will get the most likely locations for infection (and memory and rootkits), but not all drives and not all files on the drives (not even on the System partition).  I doubt it would scan any archive files as they are not likely sources of problems,
 
 
Other than as mentioned above (or removing drives before starting except of course the System Drive), you cannot control what is or isn't included in a Full Scan or a Quick Scan or Real-time protection (which is another reason why whether or not you're an Admin makes no difference and why hidden folders will be included).  The only exceptions would be excluded items and allowed items - but any of those you include open holes in your security and affect not only scans but also real-time protection so I STRONGLY recommend they be limited to ONLY files or folders or filetypes or processes that are causing you "false positive" detection problems and NOT to control scanning for the sake of convenience or timing. 
 
 
A Custom Scan will scan what you select (but not memory or rootkits).  You can choose specific drives, specific folders, and/or specific files as you wish.  Frankly I'm not sure if it will show hidden folders or not as showing hidden folders is one of the first things I do on any PC on which I'm working - but probably not.  The easiest way to check is find a folder using Custom Scan and then close it.  Then go to Windows Explorer and hide the folder.  Now go back to Custom Scan and see if it's still there.  Then you'll know how it works.
 
 
For MBR (if you have a specific reason for concern there), I'd personally recommend supplementing MSE with periodic checks with WDO http://www.sevenforums.com/tutorials/166445-windows-defender-offline.html as a better alternative.  It's not real-time and not MSE, but is more appropriate for that type of infection.
 
Now. all that said, you can do more with scheduled scans run through Task Scheduler (which you need to be an Admin to setup but not logged in as an admin to have the scan run).
 

There are some ways of doing scheduled scans not available in MSE by using Task Scheduler instead. See the later threads (10/11) from GreginMich here for one option (not exactly what you want, but it gives you an idea of what else is possible):http://answers.microsoft.com/en-us/protect/forum/protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02.

 

Also see the following thread which covers some of the same things and offers some other ideas as well:http://answers.microsoft.com/en-us/protect/forum/protect_start/scheduled-scans-not-working/fd769043-f67e-4990-8805-a02427f22dff.

 
I hope this helps.
 
Good luck!
 
Kosh

  7. Enter the command assign to give the thumb drive a letter designation.

3. click and select the security tab.
4. on the security tab, make sure the internet zone is highlighted in the box where you find local intranet, trusted sites and restricted sites.
5. click on custom level. if you cannot find the custom level option, see if you have already set the security for internet to custom level. if that is the case, you will have to click on default level button. once you do this, the custom level button would be available.
6. in the dialog box that comes up when you click custom level in step 5 above, check for allow meta refresh (allow automatic refresh in internet explorer version 7 and below) and click to disable it.
7. click on ok and click to confirm that you wish to change the default settings for internet zone.
8. click ok to close the internet options dialog box.
