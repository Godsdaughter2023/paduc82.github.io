I get a zero result when applying a COUNTIF formula instead of a count - Microsoft Community

stevekoschella say:

I get a zero result when applying a COUNTIF formula instead of a count result.

I am trying to calculate the results of categories calculated using an IF formula. The formula calculates brackets of time based on the time taken to complete a task. The results are shown in brackets of "<2", "2to4", "4to6" etc.

When I try to tabulate the bracket counts in a table, the COUNTIF formula returns a "0" result instead of counting the results.

I am wondering if the source range for the data being based on an IF formula is part of the problem. The COUNTIF formula works on other columns in the same spreadsheet.

elemantal say: Can you share a sample value range, worksheet structure and the formula you are using.

stevekoschella say:

I have managed to work out what the problem is. Thanks Google!

What the issue was, is that the COUNTIF function reads the "<" in the cell as an operator even if it is formatted as text, and try to calculate a result, which in my case returns a "0". This happens even if you specify that you are wanting to count on a text result by writing the formula as:

=COUNTIF(E:E, "<5") for example, it will return a "0" result no matter how many "<5" results there are in the range.

I found two solutions. One is to use asterixes to force the COUNTIF formula to read the contents of the source cell as text and to count it. The way to write the formula then is:

=COUNTIF(E:E, "*<5*")

This way it will read the "<" as a text symbol, not an operator.

The other way is to change the format of the result in the source cell, which in my case was derived from an "IF" function in the formula:

e.g. =IF($B$4<5, "less than 5", " ") instead of what I had: =IF($B$4<5, "<5", " ")

In this case I discovered the second option first, and opted to change the formula for the source cell to return a true text result, e.g. "less than 5". In future I will probably opt for the asterixes, as it applies for any symbol which may be encountered in the source cell, and the asterixes (which are used as wildcards) have a wide range of uses and can be used to select many more results to count on by changing what is between them.

Below is an example of what I am describing:

Source Result
<5 0 This is the result from the original formula which returned a "0" result.
less than 5 1 This is the corrected result obtained by changing the formula in the source cell to show a true text result.
<5 1 This is the corrected result obtained by using the asterixes in the COUNTIF formula to 'force' the recognition of the < as text.
