suzanne s. barnhill say:

Genevieve has answered your basic question (you can't set the document defaults in Word Online), but I would issue a caution about your changes on the desktop version. Changing the document default font (and/or the font for the Normal style) is unlikely to have unwanted repercussions, but changing paragraph formatting as you have done is not recommended because it affects every other document style. For example, if you create footnotes, you will find that they have 1.5 line spacing and a first-line indent. This may well also affect headers, footers, headings, and every other style because they are all (directly or indirectly) based on Normal. One thing many users don't realize or notice is that centered headings will never be properly centered because they're indented.

That is why I recommend leaving the paragraph formatting of Normal alone and instead using the built-in Body Text style (modified to your requirements) as your basic body style. In fact, if you're going to add a first-line indent, I recommend modifying the line spacing of Body Text and then adding the first-line indent to Body Text First Indent, which is based on Body Text. Then when you need an unindented paragraph, you can fall back on Body Text. See http://wordfaqs.ssbarnhill.com/NormalVsBodyStyles.htm for more.

Adding unique record identifier via query in Access - Microsoft Community

tstumph2016 say:

Adding unique record identifier via query in Access

I have a table that includes student ID and language information as well as modified and created dates. The way it is set up is if the same student speaks multiple languages there will be multiple records with the same student ID. What I need to do is run a query that will produce a single record with all the language information. The information is being pulled from a web application and in the past included a sequence ID for each record for each language e.g. student 1234 speaks three languages therefore there are three records for student 1234, each with its own sequence ID i.e. 1, 2, 3. What I would do is create three select queries with criteria set for each sequence ID and then combine using a fourth query. Now the sequence ID has been removed. Is there a way add a similar sequence number via query?
ken sheridan say: If each student speaks no more than two languages, then it can be done by joining an outer query which returns the MIN language to a subquery which returns the MAX language.  The following is an example using a table from my ParentActivities demo, to which I referred you earlier:

SELECT PA1.ParentID, MIN(ActivityID) AS Activity1, Activity2
FROM ParentActivities AS PA1 LEFT JOIN
    (SELECT ParentID, MAX(ActivityID) AS Activity2
     FROM ParentActivities
     GROUP BY ParentID
     HAVING COUNT(*) >1) AS PA2
ON PA1.ParentID = PA2.ParentID
GROUP BY PA1.ParentID,Activity2;

If a student speaks more than two languages it will return the MIN and MAX values of the set.

With an ActivityCounter column, as in my ParentActivities table, it can be done in a Crosstab query to return all activities per parent (languages per student in your case) with:

TRANSFORM MAX(ActivityID)
SELECT ParentID
FROM ParentActivities
GROUP BY ParentID
PIVOT "Activity" & ActivityCounter;

Or the first two activities only with:

TRANSFORM MAX(ActivityID)
SELECT ParentID
FROM ParentActivities
GROUP BY ParentID
PIVOT "Activity" & ActivityCounter IN("Activity1","Activity2");

You don't use Conditional Formatting for this. All you need is the following in the On click event of the object:

If Me.Item <> "Note" Then
