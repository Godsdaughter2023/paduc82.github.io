joeu2004 say:

Sklyn wrote:

What could be the cause and how can I fix this?

Typical rounding "error" due to binary arithmetic.

To explain....  Excel stores values internally at 64-bit binary floating-point.  Consequently, most decimal fractions cannot be represented exactly.  For that reason, for example, IF(10.1 - 10 = 0.1, TRUE) returns FALSE (!).

Generally, when you expect arithmetic to be accurate to some number of decimal places, you should explicitly round to that number of decimal places.  (Not to an arbitrary number of decimal places, as some people suggest.)

For example, B14 should be =ROUND(578358.34+582792.56, 2).

And B18 should be =ROUND(B12-B14-B15-B16+C14+C15+C16, 2).

Formatting alone is not sufficient.  Formatting affects only the appearance of the cell value.  It does not affect the underlying actual value.

trevor j lawrence say:

The People Hub What's New section aggregates updates from the social networking services that you have connected the phone to. e.g LinkedIn, Twitter, Facebook, Windows Live.  Since you have removed the connections to Twitter and LinkedIn, and you did not mention Facebook, all that is left is updates from Windows Live friends.  You mentioned, however, that you didn't really use Live previously, so I assume you have no Windows Live friends.  Hence there are no updates to show in What's New.

 

With regard to your duplicate contacts problem, the Phone can link these together so that they appear as a single contact on the phone, merging together attributes from the various sources.  Often the phone will recognise the connectio between contacts and link them automatically, but if not you can easily do it manually, although with 600+ contacts that may take a while.  Still, its a once off effort.
