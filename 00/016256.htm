sellarsj say:

Thanks for the reply. I discovered it was extremely important to keep track of the query fields that I had used when executing a previous query. On button-click I reset all these query fields to be blank as you described. However there is one special case. If you query on the ID field you have to set the value of that to zero, perform a query for data operation and then set the value of the query ID field to blank. If you just set the query ID field to blank like the other query fields the next query for data operation returns everything from your data source. If you don't change the query ID to blank after executing the query for data operation all future queries will return no data since the ID field will never have a value of zero.

 

It took me a while but I got there in the end. To summarise - in order to reset a query operation perform the following actions:

 

1. Reset all query fields used to blank

2. Reset the query ID field of the data connection used to zero

3. Perform a query for data operation against the data source

4. Reset the query ID field of the data connection to blank

 

You can now perform another query without fear that the previous query will interfere with your results. Hope this helps someone else. It took me two days to work out what was going on :-)

 

A "reset query" action in the next version of InfoPath perhaps?

zorvek say:

In a column adjacent to your data, enter this formula:

=COUNTIFS($A$2:$A$11,A2,$B$2:$B$11,B2)

and copy down the length of your table.

In your chart, select any point, right click, and select "Add data labels" and then "Add data labels". Select a label, right click, and select "Format data labels". Select "Value From Cells", enter the range containing the formulas entered above, and uncheck "Y Value".
