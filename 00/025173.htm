Reset Windows update components

Resetting Windows Update Components will fix corrupt Windows Update Components.

Please follow the below steps to reset the Windows Updates Components:

1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

                           net stop wuauserv

                           net stop cryptSvc

                           net stop bits

                           net stop msiserver

3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

                          ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

                          ren C:\Windows\System32\catroot2 Catroot2.old

4. Now, restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

                        net start wuauserv

                        net start cryptSvc

                        net start bits

                        net start msiserver

5. Type Exit in the Command Prompt to close it and then restart the computer.

6.After restarting the computer, now retry to install Windows Updates.

  I haven't seen any figures on the number of computer instructions in Windows, but my guess is around half a billion.  I came up with this guess based on the fact that most new builds we install are around 4 GB in compressed ISO format.  I have no good info on how much of that is code vs data, but let's say it's half.  That leaves 2 GB of code.  Now assume the code averages 4 bytes per instruction, so 1/4th of 2 GB would be half a GB or 500,000,000 instructions.  

 Now I wonder how much other software in current development and support is that large.  Maybe there are a few (e.g. maybe IBM's Watson and a few others), but I doubt if there are many.  So supporting 500 million instructions takes a lot of people.  I know that the number of communication paths among N people is N*(N-1), which is a parabolic curve which increases in proportion to the square of the number of people involved.  

  One other thing about the article is Onedrive.  Why does the home user need it?  Most new PCs these days have at least a TB of hard drive space, and if you make regular backups, there is no need for Onedrive.  The article only mentioned backups very briefly in passing.  If you rely on Onedrive to download your data "on the fly" when called for, that would cause significant unexpected performance degradation.

 So what does all this lead me to believe?  First, due to the size of Windows, it seems to me that we may well have to expect some bugs with these 6 month major updates.  Second I've always heard the old expression "the pioneers are the ones who get the arrows in the back", so if you plan to be one of the first to install major updates, you'd better make backups and be prepared for bugs.  Third, if you're the average home or small business user, be very careful about using new features like Onedrive and Edge unless you have an extremely good reason; I don't believe the great majority do.  Finally I'll just mention backup one more time.  If you have data you don't want to lose, don't fail to make frequent backups to an external hard drive.
