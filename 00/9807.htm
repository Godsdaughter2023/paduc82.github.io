You'll have a problem if you try to hide the field whose control has the focus on the subform. So you need to set focus to a control on the subform that won't be hidden. You could use an unbound text box for this - I'll name it txtDummy.

Private Sub cmdHide_Click()
    Dim itm As Variant
    Me.[Project List]!txtDummy.SetFocus
    For Each itm In Me.lstFields.ItemsSelected
        Me.[Project List].Controls(Me.lstFields.ItemData(itm)).Visible = False
    Next itm
End Sub

Private Sub cmdShow_Click()
    Dim itm As Variant
    Me.[Project List]!txtDummy.SetFocus
    For Each itm In Me.lstFields.ItemsSelected
        Me.[Project List].Controls(Me.lstFields.ItemData(itm)).Visible = True
    Next itm

hansv mvp say:

I'd create a table tbl_Dates with a single date/time field DateBooked (also the primary key).

Populate the DateBooked field with all dates you want to cover, e.g. from 1/1/2017 through 12/31/2020.

An easy way to do this is to create the series of dates in an Excel worksheet, and to copy/paste them into the Access table.

Next, click Query Design on the Create tab of the ribbon.

Add tbl_Dates and tbl_Room_Book_Content, then close the Show Table dialog.

Add DateBooked from tbl_Dates and RoomID from tbl_Room_Book_Content to the query grid.

Sort ascending on both fields.

Add the following criteria to the DateBooked column:

Between [DateFrom] And [DateTo]

Switch to Datasheet View to inspect the results.
