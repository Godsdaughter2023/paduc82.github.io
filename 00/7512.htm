Install this macro in your Normal.dotm template. It will cause all documents that you open (not new ones that you create) in Reading Mode with a single page view, the same as you would get by clicking View > Layout > Paper Layout.

Sub AutoOpen()
    ActiveWindow.View.ReadingLayout = True
    ActiveWindow.View.ReadingLayoutActualView = True
End Sub

How to create the Support Logs file for MSE/Defender - Microsoft Community

stephen boots say:

How to create the Support Logs file for MSE/Defender

Technical Level : Basic

Summary

Sometimes it is helpful to capture logs from Microsoft Security Essentials (Windows Vista/7) or Windows Defender (Windows 8.x/10).

This guide explains how to create the MPSupportFiles.cab file and where it can be found.

The screen captures were created on a Windows 10 machine. I've noted the difference for MSE vs. Defender.


Details

In the Search box, type command.exe (or just - command)

http://www.7tutorials.com/7-ways-launch-command-prompt-windows-7-windows-8

In the results, at the top, right click command.exe (command prompt) and select Run as Administrator.

Say yes to the UAC prompt, if prompted provide admin credentials.

http://www.bleepingcomputer.com/tutorials/open-an-elevated-command-prompt-in-windows-8/

http://www.bleepingcomputer.com/tutorials/windows-elevated-command-prompt/

In the resulting command.exe window, type the following:

cd\ <enter>

cd program files <enter>

cd Microsoft security client <enter> (Windows Vista/Windows 7) -or-

cd Windows Defender <enter> (Windows 8.x or Windows 10)
MpCmdRun -getfiles <enter>

You'll see the following as the logs are being captured:

When complete, you'll see the location of MPSupportFiles.cab

As shown in the above image, the file on my Windows 10 machine (would be the same for Windows 8.x)was created here:

C:\ProgramData\Microsoft\Windows Defender\Support\

On a Windows 7 machine, it would be here (same for Vista):

C:\ProgramData\Microsoft\Microsoft Antimalware\Support\

You need to enable view of Hidden Files in the Windows Explorer options.

http://windows.microsoft.com/en-us/windows/show-hidden-files#show-hidden-files=windows-7

http://www.howtogeek.com/howto/windows-vista/show-hidden-files-and-folders-in-windows-vista/

And you need to accept the UAC prompt to proceed or provide Administrator credentials:

Upload the resulting CAB on OneDrive and get a link to it. See this article for help:

Share OneDrive files and folders

john mcghie say:

I suspect the issue is that Word for Mac supports only the Legacy controls that it comes with on the Developer tab. It cannot use the current controls that PC Word provides.

So the bottom line may be "You can't".  If you re-make the form in Word for Mac using the legacy controls, it will work without issue in Word for the PC.

That said, I am a little suspicious that "none" of your macros work.  Some of them should at least try to run, to the point of issuing compile-time errors.

Go to Word>Preferences>Security and check your setting for "Warn before opening a file that contains macros".  If that is turned on, and you inadvertently clicked "No" to the prompt, the macro code would have been stripped.

You should also check that the document is on your local hard disk within your Documents folder.  In various other places, macros will be suppressed as a security hazard. Of course, it won't work if opened from within an email.

And open the thing in the Visual Basic Editor and ensure that your code is still there.  It may have been stripped.

Hope this helps
