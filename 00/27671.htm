bob jones aka: cybertaz say:

I don't fault you for being cautious :-) But I'm not sure your concerns are warranted... It depends on details not available to me. In general, though;

  • Macros that are document-specific typically are stored only in the document file, itself. It doesn't make sense to store macros in the Normal template if they aren't applicable to other documents. If not stored in the file they aren't available when the document is edited on a different computer. They may be, but aren't necessarily, stored in the [Normal.dotm] template as well. That's usually determined when the macro is created. Another possibility is if a custom template is used to create documents of a specific type. In that case, it's appropriate to store all required macros in the template.
  • Styles are handled in a similar fashion. By default, Custom Styles are stored only in the document where they are created or applied. In order for a Custom Style to be added to the Normal template [or the template on which the document is based] an extra step has to be taken... i.e., ticking the Add to template check box in the New/Modify Style dialog.
  • AutoText is stored only in the template, but once an instance is inserted into a document it will remain just as any other content, even if the AutoText Item is removed from the dialog afterward. The only concern is for AutoText stored in the template that has not yet been inserted into the document. Obviously, if you lose that it won't be available for later insertion.
