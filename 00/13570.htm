john - project say:
weishe,
There are some basic problems with the type of graphic you want to show. First, Gantt Bars are based on duration, not resources. If you have a single resource assigned to each task then you can customize gantt bars in unique colors for each resource. However, if more than one resource is assigned to any task, you're out of luck.

If the single resource assignment is applicable to your schedule, then you can also roll up those custom gantt bars to the various summary lines, however, it will get very busy if you try to do it at the highest level in the master.

Also be aware that formatting is unique to the master, that is, you would need to set up the custom gantt bars at master level. You will only be able to see that formatting at each subproject level if you duplicate the formatting.

John

john - project say:
weishe,
This method can handle up to 20 resources. You could do more if you customize each task bar style, but then you would have to go to each task and customize the bar for that task.

You didn't mention which version of Project you are using. I will assume Project 2010. Do the following at the master level.

First, add as many extra task flag fields (e.g. Flag1, Flag2, etc.) as you have resources. Second, assign a flag to each resource and set that flag in the Gantt Chart view for each resource. Third, go to Format/Bar Styles group/Format icon. Select Bar Styles from the drop down. At the bottom of the big humongous list of bar styles add a new bar style for each resource. Call the bar style whatever you like but the resource name seems like it might be appropriate for a bar name. In the Show For....Tasks, enter Normal, FlagX, where "X" corresponds to the flag assigned to that resource. Pick a bar color. That's it, you will now have task bars that are uniquely colored by resource.

Of course, you realize you could simply add the Resource Name as text next to or in each normal task bar and get the desired identification without the need to create a bunch of customized bar colors, that make the big humongous task bar styles list even more humongous.

Hope this helps.

Skype for Business is one of the applications that come bundled as a part of Office. If you do not have a Skype for Business item in those locations, it is contained in the Office Suite install. Removal of the suite or renaming/deletion of Lync.exe is generally the only official way of removing it from the suite on click-to-run installs. The alternative to that is to modify the click-to-run install using the Office Deployment Tool for your build of Office. With the deployment tool you can modify the click-to-run install to omit Skype for Business from the install. Office would still have to be removed beforehand.

In regards to stopping any program from launching on start-up, that can be controlled in the options for many applications, or by logging out prior to exiting most UWP Apps. If you do not want to go that route, you can use the start-up tab in the task manager to disable the item from starting up. On older OSes you would use msconfig to perform a similar function without having to mess with the registry.

mmahlatjie say:

Finding maximum values across fields

scottgem say:
I have a database and it manages student data. I would like to find  top most values across fields. Fields are subjects and I have to use two highest values to find a certain value. I have used a code to define maximum function but this works only in finding top most, how can i adjust it to find the next highest or next two highest.

This would be a lot easier if your database was properly normalized. Notice the bolded portion above. That is not the right way to design this. You should have at least three tables; 

Students: info identifying each student

Subjects: a listing of each subject

StudentSubjects: a record for each student and the scores for each subject. (If a student had 3 subjects there would be 3 records)

With this structure it would be very easy to sort scores within each student.

I suggest you look at redesigning your database.

mmahlatjie say:

Thanks for the reply.

Well I have student table with student details, exams-with scores and this is where the subjects are fields, then subject table which actually has no relationship with student table. Now with studentsubject table,won't it give a problem since that would mean more many fields(being students).

scottgem say:

and this is where the subjects are fields,

Now with studentsubject table,won't it give a problem since that would mean more many fields(being students).

No, you are missing the point here. A Relational database should be tall and thin, not wide. You define data with records not field names. So you should not have a table with field for each subject. You should have a record for each student/subject/score. So if John took Math and got a score of 75, that is ONE record defined by the studentID, the subjectID and the score. If John got scores for 5 subjects he would have 5 records. 

You may need to add another field to this table to define what term John received a score for in that subject. So we are talking about 4-5 fields:

StudentSubjectID

StudentID

SubjectID

TermID

Score
