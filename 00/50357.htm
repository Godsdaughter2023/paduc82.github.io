daavee say: Significantly improve performance? No.

All you need to do is determine which process is hogging memory and/or resources. To do this, open Task Manager and make sure the Processes tab is selected. Click on CPU. Then click on CPU again. This will cause the most resource-hungry processes to show at the top of the list. You can do the same with Mem Usage. In your next post, tell us what the worst offenders of both are.

Regarding memory, here is an effective way to determine if there is enough (that is, whether or not your system is overly relying on the pagefile):

Open Task Manager (Ctrl+Alt+Del) and click the Performance tab. Then note the three values under Commit Charge (K): in the lower left-hand corner: Total, Limit, and Peak.

The Total figure represents the amount of memory you are using at that very moment. The Peak figure represents the highest amount of memory you used since last bootup. If both these figures are below the value of Physical Memory (K) Total, then you probably have plenty of RAM. In case you want to explore this further, you may run Page File Monitor for Windows XP:
