print a single record from a subform - Microsoft Community

anneg3 say:

print a single record from a subform

I have a form with two pages. The first page is linked to table 1 and the second page has a subform which is linked to table 2.  Table 1 has details of a book collection and Table 2 has names and addresses of borrowers.  I would like end users to be able to print a report from a single selected record on this subform, in other words, the name and address of the current borrower with the title of the borrowed book.  Many thanks in advance.

ken sheridan say: Before looking at the question of printing a report, you say you have two tables, but for this situation you'd need three, e.g.

Books
....BookID (PK)
....Title
....etc

Borrowers
....BorrowerID (PK)
....FirstName
....LastName
....etc

and to model the many-to-many relationship type between Books and Borrowers

Withdrawals
....BookID
....BorrowerID
....DateBorrowed
....DateReturned

The primary key of the last table is a composite one made up of BookID, BorrowerID and DateBorrowed.  The Withdrawals table would be the RecordSource of your subform.

So, assuming the above model, firstly create a report based on a query which joins all three tables, Borrowers to Withdrawls on BorrowerID, and Withdrawals to Boooks on BookID.  Return the relevant columns from the tables which you want to see in the report, but be sure to include BorrowerID from Borrowers and BookID from Books, but not the columns of those names from Withdrawals. You don't need to show these two key columns in the report however, just include them in the query.  How you proceed then depend partly on how the subform is set up, so I'll assume that, in addition to controls bound to the two date columns, it contains a combo box in which to select the borrower, set up as follows:

Name:    cboBorrower

ControlSource:    BorrowerID

RowSource:     SELECT BorrowerID, FirstName & " " & LastName FROM Borrowers ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column and that the second is at least as wide as the combo box. Borrowers is the name of the table and BorrowerID its primary key.

You then have a couple of options as to how you open the report filtered to the current borrower: and add a button within the subform to print the subform:

1.  Add a button to the subform, which of course means it must be in continuous forms view rather than datasheet view.  This can be in its header or footer, or you could have a small button at the end of each row, e.g. with a printer image.  Either way the code in the button's Click event would be along these lines:

Const REPORT_NAME = "Your report name goes here"
Dim strMessage As String
Dim strCriteria As String

If Not IsNull(Me.BorrowerID) Then
   strMessage = "Print details of " & Me.cboBorrower.Column(1) & "'s withdawal?"
   If MsgBox(strMessage, vbQuestion + vbYesNo, "Confirm") = vbYes Then
       strCriteria = "BorrowerID = " & Me.cboBorrower & " And BookID = " & Me.BookID & _
            " And DateBorrowed = #" & Format(Me.DateBorrowed,"yyyy-mm-dd") & "#"
       Me.Dirty = False
       DoCmd.OpenReport REPORT_NAME
   End If
Else
    strMessage = "No borrower selected."
    MsgBox strMessage, vbExclamation, "Invalid Operation"
End If

2.  The alternative would be to have the button on the main parent form, in which case the code is very similar:

Const REPORT_NAME = "Your report name goes here"
Dim strMessage As String
Dim strCriteria As String

If Not IsNull(Me.BorrowerID) Then
   strMessage = "Print details of " & _
       Me.YourSubFormControl.Form.cboBorrower.Column(1) & "'s withdawal?"
   If MsgBox(strMessage, vbQuestion + vbYesNo, "Confirm") = vbYes Then
       strCriteria = "BorrowerID = " & Me.YourSubFormControl.Form.cboBorrower
       DoCmd.OpenReport REPORT_NAME
       strCriteria = "BorrowerID = " & Me.YourSubFormControl.Form.cboBorrower & _
            " And BookID = " & Me.BookID & " And DateBorrowed = #" &  _
            Format(Me.YourSubFormControl.Form.DateBorrowed,"yyyy-mm-dd") & "#"
   End If
Else
    strMessage = "No borrower selected."
    MsgBox strMessage, vbExclamation, "Invalid Operation"
End If

In this code YourSubFormControl is the name of the control in the parent form which houses the subform. Remember that if you've used spaces or other special characters in object names (not recommended) they must be wrapped in square brackets, e.g [Book ID].

ender315 say:

I am getting the same error. I looked at the HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows Search\ProtocolHandlers\Mapi16 registry key and found that it referred to ProgID Outlook.Search.MAPI16Handler.1. The error not be found'.

I attempted to find what module could not be found and found that this module belongs to the Outlook's Mapi16 Protocol Handler (see, HKEY_CLASSES_ROOT\CLSID\{F8E61EDD-EA25-484e-AC8A-7447F2AAE2A9}). This key HKEY_CLASSES_ROOT\CLSID\{F8E61EDD-EA25-484e-AC8A-7447F2AAE2A9}\InProcServer32 says that the protocol handler should be here C:\Program Files\Microsoft Office\root\VFS\ProgramFilesX64\Microsoft Office\Office16\Interceptor.dll, but there is no such file in my system.  Apparently it was not installed. The folder C:\Program Files\Microsoft Office\root\VFS\ProgramFilesX64\Microsoft Office is empty.

Now the question is how to install this Interceptor.dll to the system.

Found this file in this location "C:\Program Files\Microsoft Office\root\Office16\Interceptor.dll". Changed the (Default) value of the HKEY_CLASSES_ROOT\CLSID\{F8E61EDD-EA25-484e-AC8A-7447F2AAE2A9}\InProcServer32 key from  C:\Program Files\Microsoft Office\root\VFS\ProgramFilesX64\Microsoft Office\Office16\Interceptor.dll to C:\Program Files\Microsoft Office\root\Office16\Interceptor.dll. The error is no longer appearing in the Event Viewer.
