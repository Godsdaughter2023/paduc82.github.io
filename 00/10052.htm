CPU usage in Security Scan - Microsoft Community

zippyzippy say:

CPU usage in Security Scan

In the Scan Settings of my Security Essentials there is a box which gives me a choice of "limiting cpu usage during scan to:" a certain percentage, i.e. 10% to 100%. How do I know what percentage of cpu usage to set this to?? Thank you
kosh vorlon say:

Hi zippy,

 

It applies to the usage by MSE when it runs scans. The higher the number, the more CPU will be allocated to MSE during scans. So if you are running MSE scans in the background as you work (as I do) and your system response drops to a crawl or enough to be bothersome, lowering this number may help (I found that using 20% was best for me because I tend to do a lot of other things at the same time and lower just simply takes too long - it what I consider "optimal" for my primary computer), though it will then take longer for the scan to complete (and in my case and to me personally, that really doesn't make too much of a difference). On the other hand, if doing the MSE scan while the computer is not in use (like late at night when the computer is left on 24/7), raising the number may help it finish faster (I've seen some set it as high as 80-90% or even 100% when done in this manner) - but you do then need to remember to keep changing it (unless you use some tricks in the P.S. at the bottom).

 

Some/many suggest leaving it at the default of 50% unless you encounter response problems (in fact, that's the common "rule of thumb") - but I prefer to tweak it to the best possible setting in general for each computer and adjust rarely for special situations.

 

After doing all you can to generally improve your overall system performance (I can provide some suggestions if you request them), I suggest starting at the default and tweaking it by 10% increments (starting in the  direction which you think will best meet your personal goals) until you find the best number for your circumstances where it allows the scans to proceed as quickly as possible without adversely affecting your other activities (if that's the case). I also suggest running mostly quick scans and doing full scans only periodically or when warranted as quick scans tend to take far less time and use far less resources than full scans but provide good protection unless you have reason to believe you may be infected or a quick scan does detect an infection - then following with a manual full scan would be a good idea.

 

There are no set rules or standards. I generally recommend that you test various settings and see what works best for you and then use that for that computer unless or until you start having problems (say you start doing more things at the same time or start doing more things that are CPU-intensive or scans start taking too long to fit your schedule as more needs to be scanned) - then adjust it to fit the new situation. You may use a different number on each different computer as each situation may be entirely different. 

 

My way requires more time and effort than a standard "just use the the default unless you have trouble", but I think of it like mouse click settings.  They too can easily be left at the default and you'll deal with it and/or eventually adapt - but I like to go to settings and configure it to my natural clicking speed to set the mouse to my own personal preferences.  More work - yes.  Necessary - no.  Convenient in the long run if done right - yes.

 

Another thing to consider what it this MSE feature really means in terms of CPU (it isn't ALL of your CPU) as well as the real effect of this control compared to factors like memory and other seemingly unrelated options which may help far more.  Please see the post from Rob Koch here: http://answers.microsoft.com/en-us/protect/forum/protect_scanning/cpu-usage-during-scan/b2a45c4b-877f-e011-9b4b-68b599b31bf5.

 

I hope that helps.

 

Good luck!

 

Kosh

 

P.S. You may want to consider using Task Scheduler so you can awaken the computer from hibernation, run the scan, and put it back to sleep while you sleep or how you may otherwise be using hibernation. You can't do this with MSE directly (because the program-created MSE task in Task Scheduler is special and should not be modified), but it can be done with special tasks in Task Scheduler. You can even setup different schedules for different types of scans - as many as you want (which is what I use it for - using MSE for scheduled daily scans, but a special task in Task Scheduler for full scheduled scans every other week which I'd otherwise need to remember to run manually).

 

While I prefer to run them when I'm working, if other factors (such as Power Management or already heavy use of CPU for normal activities that you prefer not to change or reduce and can't tweak performance to improve has you running slow even without scans or travel where you need to use the battery {which I generally try to avoid due to the drain - but it depends on your battery [my older system only lasts about 3 hours or so without scans and even if I cut way back on open tasks], the type of scan, and whether or not it can handle it and complete without draining the battery}) are involved, then this may be one solution - and then you can set CPU usage high since it won't be bothering you or anything else while you are sleeping and it is otherwise not doing much.

 

It's a little advanced, but if interested, the following should assist you:

 

There are some ways of doing scheduled scans not available in MSE by using Task Scheduler instead. See the later threads (10/11) from GreginMich here for one option:http://answers.microsoft.com/en-us/protect/forum/protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02.

 

Also see the following thread which covers some of the same things and offers some other ideas as well:http://answers.microsoft.com/en-us/protect/forum/protect_start/scheduled-scans-not-working/fd769043-f67e-4990-8805-a02427f22dff.

 

NOTE about using Task Manager for scheduled MSE scans: The paths may have changed slightly with Version 4 - but I've seen no official update to the instructions by the author since then. AFAIK, the key difference is that there may no longer be an antimalware folder in the path in one of the command instructions. If trying, check the paths to be sure they are correct and post back if you have troubles and we'll try to find and help you correct the issue.  My own special task still runs fine after upgrading, but I don't use all the bells and whistles mentioned above (I simply created a task to schedule a full scan every two weeks to supplement the daily ones through MSE - nothing very special or complex) .
