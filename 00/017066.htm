albert d. kallal say:

Use the browseto command.

 

If you browseto a form that is included in the nav buttions, then that one becomes active.

 

eg:
 
DoCmd.BrowseTo acBrowseToForm, _
                      "Form3", _
                      "Navigation Form.NavigationSubForm", _
                      "where condition"
 
Note that if in say a choice of 5 nav buttons, you do NOT have form3 (as in the above example) then the current existing sub form will be replaced. However, if the 4th button is set to form3, then the above would display form 3, and the 4th button would highlight.
 
And, as the above shows, you can add in a where clause.
 
The above browse to command of course is available in both VBA as above, or as a macro command.

 

This navagation feature is for web forms, but is a great choice for desktop also.

ken sheridan say:

This is a basic one-to-many relationship type.  The 'new records' which you wish to add for each individual should be represented by rows in one or more related tables.  You'll find an example in FormsDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file There is a referenced table Clients, in which each row represents a client, with columns representing their attributes.  There are two related tables for Events and Visits in which multiple rows per client.  The model is as below:

Three means of representing this in forms are illustrated, but in your case a form with a tab control would probably be the most appropriate. The use of tab controls has largely replaced the old method of using a multi-page form.  In my demo the first page of the tab control has controls bound to the columns from the Clients table on which the form is based, while the second and third pages each have a subform, in continuous forms view, for events and visits respectively.  The subforms are linked to the parent form on Client.  The client's names are in controls on the parent form above the tab control, so these remain visible whichever page of the tab control is selected.

You could of course do the same thing with a multipage form, putting one or more subforms on the later pages, but a tab control tends to be used more commonly these days.  The choice is yours, however.

Below are general troubleshooting steps to solve various problems in Excel. Try them one by one and check the result after each step -

2. Perform Quick Repair of the office installation - https://support.office.com/en-NZ/Article/Repair-an-Office-application-7821d4b6-7c1d-4205-aa0e-a6b40c5bb88b

Move those files to some other location (i.e create a backup of those files / folders and delete all files / folders from here). Hence, make Excel folder blank.

Now open the file and see if the problem disappears or not. 

Note - to find Excel path, you need to find XLSTART path. Excel folder contains XLSTART folder in itself (In case, your XLSTART path is not in Appdata)

1. Open Excel
2. ALT+F11
3. If Immediate Window is not visible, type CTRL+G. Same can be also be accessed through View > Immediate Window

Close all office applications and change your default printer through Start > Devices and Printers to another printer. It may be some physical one or One Note or XPS or PDF or anything else.
