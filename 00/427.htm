jay freedman say:
When you click the Default button, the settings are supposed to be saved in the Normal.dotm template. For a number of possible reasons, though, the template may not be saved.

To force the changes to save:

  • Click the Office button and choose Open, or press Ctrl+O.
  • In the Name box, type or paste in   %appdata%\Microsoft\Templates\Normal.dotm   and click the Open button.
  • Set the margins in Page Layout as you want them.
  • Save the template.


Now try opening a new blank document.


If the margins are still wrong, there may be a problem with the Normal.dotm template itself. See http://word.mvps.org/FAQs/AppErrors/BlankDocNotBlank.htm for instructions on creating a new template.

Name:  cboSchool

BoundColumn:   1
ColumnCount:   3
ColumnWidths:  0cm;0cm; 8cm

teresa mc say:

using if formula to return the value of another cell in another worksheet

In cell A1 of worksheet 1, I need the following to happen.  If i14:i36 of worksheet 2 is not empty, I need the A cell of worksheet 2 that corresponds with the same row to be copied.

Can you help me?

chip pearson say:

It isn't clear what you are after. What does "corresponds with the same row to be copied" really mean? The following array formula may do what you want.

=IF(COUNTA(Sheet2!I4:I36)=0, "", OFFSET(Sheet2!A4,MAX((Sheet2!I4:I36<>"")*(ROW(Sheet2!I4:I36)))-ROW(Sheet2!I4:I36),0,1,1))

If Sheet2!I4:I36 is empty, it reutrns an empty (blank) string. Otherwise, it returns the value from Sheet 2 column A that correspondes to the last non-empty value in Sheet 2 I4:I36.

This is an array formula, so you MUST press CTRL SHIFT ENTER rather than just ENTER when you first enter the formula and whenever you edit it later. If you do this properly, Excel will display the formula in the formula bar enclosed in curly braces { }. See www.cpearson.com/Excel/ArrayFormulas.aspx for much more information about array formulas.

If this is not what you want, post back with considerably more detail.

Thanks - yup, that's the problem. We only support one copy of each Office 2016 application under the /Applications folder structure. Steps to resolve:

1. Close down all open Office applications

2. Delete either the apps directly under /Applications or /Applications/Office 2016 (it doesn't really matter which one you delete, just make sure you end up with one copy of each app)

3. Launch Word and choose Help -> Check for Updates

4. Open Terminal and type 'defaults read com.microsoft.autoupdate2' and verify that the duplicates have been removed
