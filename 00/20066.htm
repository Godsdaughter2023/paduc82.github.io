1. As far as I know, the file structure of a .pst file for Outlook 2010 and 2013 is the same.

 

2. The default location is

 

C:\Users\<username>\AppData\Local\Microsoft\Outlook

 

where <username> is your Windows login name. AppData is a hidden folder, so you must show hidden files and folders in Windows Explorer to be able to see it.

(If you don't see AppData: click Organize > Folder and Search Options in Windows Explorer. Activate the View tab, and select the option to show hidden files, folders and drives, then click OK.)

 

3. From http://support.microsoft.com/kb/291636/en-us, which applies to Outlook 2013 too:

  1. Open Outlook 2010 (read 2013).
  2. Click the File tab on the Ribbon, and then click the Info tab on the menu.
  3. Click the Accounts Settings tab, and then click Account Settings again.
  4. On the Data Files tab, click Add.
  5. Under Save as type, select Outlook Data File (*.pst).
  6. Find the new location for your .pst file, and then click OK.
  7. Select the .pst file, and then click Set as Default.
  8. If this is your default e-mail delivery location, you will receive the following message:
    You have changed the default deliver location for your e-mail. This will change the location of your Inbox, Calendar, and other folders. These changes will take effect the next time you start Outlook.

    Click OK.
  9. Click the .pst file that was identified in step 4 in the "How to Identify the Name and Location of Your Personal Folder File" section, and then click Remove to remove the local .pst file from your profile.
  10. Click Yes, click Close, and then click OK to close all dialog boxes.
  11. On the File menu, click Exit.
  12. Restart Outlook.

However, if I save and close the document (as a .docx file), when I reopen it I can no longer update the quick parts.

Thanks,

Ryan

stefan blom say:

Assuming that you are using the commands at Insert tab | Quick Parts | Document Property, you are inserting content controls which should update automatically. But note that content controls will be converted to static text if you save the document in a Word 97-2003 format.

On the other hand, if you are inserting DOCPROPERTY fields (Insert tab | Quick Parts | Field) you will have to force an update, for example by switching to the backstage view and then back to your current view (Ctrl+F2 followed by ESC does the trick).

Hi Michael RSP,

 

We appreciate your time posting in Microsoft Community. We understand that are getting error message "Pending DNS configuration." We also noted that you want know how to reset a user's password and add another email account. I would be happy to help you with that.

 

Regarding the error message "Pending DNS configuration." We suggest you change your MX record to a correct one so that your domain will be verified. Follow the steps below:

 

1. Delete all existing MX records configured in your registrar wait for an hour or two


2. Go to https://domains.live.com


3. Get the MX record from there and it should be in ###.pamx1.hotmail.com format if it’s in ###.msv1.invalid format, please replace it with .pamx1.hotmail.com

 

This should resolve the issue and email features will be enabled at the same time the domain will be verified.

 

Regarding on how to reset a user's password, please provide us your domain name and its natures and we will check it on our end.

 

After resolving the error message, kindly follow the steps below to create a new user account in Admin Center.

 

1. Sign in to Windows Live Admin Center with your Microsoft account ID (The user name and password that you use to sign in to any Windows Live, MSN, or Office Live sites and services. If you have a Passport Network, Outlook.com, or Messenger account, you can use it as your Microsoft account ID.).

2. Under Your domains, click the appropriate domain.

3. In the Member accounts section, click Add.

4. Type a user name and password for the new account.

5. Select the Require password change at first login check box, and then click OK.

6. Send the user the temporary password, and then inform them that they need to sign in to activate their new account.

 

Please take your time to try the suggestions and let me know the results at your earliest convenience. If anything is unclear or if there is anything I can do for you, please feel free to let me know.

 

Thank you.
