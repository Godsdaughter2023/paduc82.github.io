Option Explicit

Dim weThereYet As Integer, StopCount As Integer
Sub AutoNew()
    HavePatience

    HavePatience

Sub HavePatience()
    If StopCount > 9 Then
        MsgBox "Sorry, your document failed to open."
            StopCount = 0
            End
    End If

    StopCount = StopCount + 1

    weThereYet = Application.Documents.Count

    If weThereYet < 1 Then Application.OnTime _

dianeoforegon say: It sounds like you did a hard quit on your computer with Entourage open. When this happens, data can be lost. NEVER power down before you shut down (sleep is NOT enough). If it happens look for files in the invisible folder "lost+found".

1) Check the trash

2) Search for "Microsoft User Data"  or "Main Identity" or "Office 2008 Identities" (without the quotes)..  If you find it move it back.

~/Documents/Microsoft User Data/Office 2008 Identities/

3) Search for invisible files in 'lost+found' directory. This file is created by fsck if it finds orphaned files.

Select your Desktop so the Finder is showing in the Menu bar.
Under the 'Go' menu, select 'go to folder'.
Enter "/lost+found" (without the quotes).

If there is a lost+found folder it will open, if not, you will get a 'folder not found' error.

It's possible you have lost your Identity with no option to recover. Your only option is to create a new Identity, enter your account info so you can send & receive. Once you get access to your backups you can merge the new info with old Identity.

brian tillman [outlook mvp 2007-2019] say:

Aliases are not separate accounts. They are two addresses that both deliver to the same mailbox. So, any messages send to alias B will arrive in the same Inbox as messages sent to alias A, unless you have a rule, either server-side or client-side, that moves messages addressed to one of the aliases to another folder. Because you added the second alias to Outlook 2016 as a separate account, you are seeing the exact same mailbox in each set of folders.
