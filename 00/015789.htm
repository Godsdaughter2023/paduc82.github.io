robo ox say:

Ok, I'm using a Win7 Ultimate x64 OS with Office 2010 ProPlus x64.

+ Download ADM templates exe fromhttp://www.microsoft.com/downloads/en/details.aspx?FamilyID=64b837b6-0aa0-4c07-bc34-bec3990a7956&displaylang=en

+ Run the exe file. It will ask you where to extract the files. Extract to a folder e.g. C:\Office ADMs

+ Run gpedit.msc with administrative privileges (this will load up your local group policy editor).

+ Right Click one of the "Administrative Templates" folders in the left panel. (They both do the same).

+ Select "Add/Remove Templates". This will bring up a list of templates currently loaded.

+ Click the "add..." Button.

+ Browse to the extracted files and open the ADM folder, then your language. e.g. C:\Office ADMs\ADM\en-us

+ Select all the .adm files (or just the one for your particular program if you want). Click "Open". This will take you back to your list of loaded ADMs that now also lists the one you've just added.

+ Close the "Add/Remove Templates" dialog.

+ The gpeditor may freeze at this point - if it does just force close it and reload.

The settings should now be available to you in the gpeditor under, User Configuration\Administrative Templates\Classic Administrative Templates\

+ The original extracted folder can now be removed.

The particular setting mentioned in this thread is found at User Configuration\Administrative Templates\Classic Administrative Templates\Microsoft Outlook 2010\Outlook Options\Preferences\Calendar Options\Calendar item defaults

There are many other threads on how to modify settings using group policy so it's a bit pointless going into detail here.

A few pointers though,
+ edit your setting in group policy editor.
+ Then run the "gpupdate /force" command (without the quotes). This will force the computer to match its settings with the gpeditor.

+ The Calendar item defaults setting has to be set in minutes only in the gpeditor.
+ If the setting is successful, you should find that the drop down box that is in the outlook options dialog is greyed out.

mimmo57 (d. laurenza) say:

Hi,

you can create a function  (module)  to use into a query so as:

ConcField: fConcFields(CommonKeyField)

this code in the module:

Option Compare Database
Option Explicit

Function fConcFields(CommonKeyField)
    Dim myRs As Recordset
    
    Set myRs = CurrentDb.OpenRecordset("Select * From table2Name Where CommonKeyField= '" & CommonKeyField & "'")
    fConcFields= ""
    Do While Not myRs.EOF
        If Len(fConcFields) = 0 Then
           fConcFields= myRs("table2FieldName")
        Else
           fConcFields= fConcFields & ";" & myRs("table2FieldName")
        End If
        myRs.MoveNext
    Loop
    myRs.Close
    Set myRs = Nothing
End Function

When you put a formula in Excel this is stored in that cell itself. You can copy the same formula from Formula Bar and paste into any other cell and change the formula there.

You can also precede anything in Excel with apostrophe and store as a constant. So you can put '=SUM(A1:A21)*30

Anytime, you can copy this and remove apostrophe and use anywhere after changing the range.
