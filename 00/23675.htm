ken sheridan say: While I'm not generally in favour of using undocumented system objects, in this case it makes it very easy to do what you want.  Use the following query as the list box's RowSource property:

SELECT [Name]
FROM MSysObjects
WHERE [Type] = -32764
ORDER BY [Name];

This will of course list all report objects, including those used as subreports.  If you want to exclude those then use some form of naming convention.  I prefix all report names for those being used as subreports with 'rptSub', so in my case the query would be:

SELECT [Name]
FROM MSysObjects
WHERE [Type] = -32764
AND LEFT([Name],6) <> "rptSub"
ORDER BY [Name];

marshall barton say:

Just to re-enforce what others are telling you, stop trying to use a value list and use a table.

 

A good way to do is to have two (or more columns) in this table, one for the report's actual name and the other for a short user friendly report description.  Other columns in the table could be used to specify how the list of reports could be filtered or sorted.   You can use the combo or list box's ColumnWidths property to hide the columns you do not users to see (e.g. the report name) and still refer to the hidden columns by using Me.listboxname.Column(column number).

ericyang_msft6286 say:

Hi Asharaf,

One domain can only be added to a single tenant of Office 365 or other Microsoft cloud service. This behavior is intended to avoid various confliction that may happen. For example, if a same domain can be added to two Office 365 tenants, since each tenant has its own Exchange Online server and one domain can only has one active MX record, half of the users will lose their emails.

Eric Yang

vincentwu_msft say:

Hi Asharaf,

Yes, it is not feasible. In terms of client connectivity, Office 365 use a combination of factors to determine which datacenter is used.

For example, when you sign in to Office 365, you’re usually connected to the closest datacenter in your region. Then, say you open an email. To show you what’s in the email, a second connection might be opened between that datacenter and the datacenter where your tenant is. Datacenter-to-datacenter connections are very fast, so using that type of connection to get your email content from the Exchange server is quick.

For more details, please refer to the link below:
https://support.office.com/en-us/article/Client-connectivity-4232abcf-4ae5-43aa-bfa1-9a078a99c78b
