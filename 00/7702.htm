macropod say:

You can have an index display the song numbers instead of their page numbers via the XE field's \t switch in conjunction with a STYLEREF field. If, for example, you're using the 'Heading 2' Style for the song titles, the corresponding XE fields (which you'd insert into the heading paragraphs) might be coded as:

{XE "Yesterday" \t {STYLEREF "Heading 2" \w}}

{XE "Dancing Queen" \t {STYLEREF "Heading 2" \w}}

{XE "Always" \t {STYLEREF "Heading 2" \w}}

{XE "Goodbye Yellow Brick Road" \t {STYLEREF "Heading 2" \w}}

Even simpler, if you insert the XE fields into the paragraph immediately following the headings, you could use the one XE field code for all of them:

{XE {STYLEREF "Heading 2"} \t {STYLEREF "Heading 2" \w}}

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues.

For more on the XE field, see: https://support.office.com/en-us/article/field-codes-xe-index-entry-field-abaf7c78-6e21-418d-bf8b-f8186d2e4d08

bernie deitrick, excel mvp 2000-2010 say:

If they are making daily payments, you will have to accept that for whatever basis you use, for some months the daily payment will not add up to the monthly payment - unless you divide by 28 and then have a 2 or 3 day "Payment Holiday" for all the months other than February.

BUT, If the customer is making daily payments, the daily amount could reflect the actual cost of the loan amortization -

=PMT(AnnualRate/365,730,LoanAmount)

You should also note that if you have weekly payments, one month out of every three will have five payments, and the other two will have four, since each quarter is 13 weeks.

A better formula for that case would be:

=PMT(AnnualRate/52,104,LoanAmount)

though there is not an integer number of weeks in any year.

Bernie
