a. user say:

500 5.3.3 Unrecognized command on MAIL FROM: command if password length is 8 or 9 characters

I've encountered a problem sending email to Office 365, I get 500 5.3.3 Unrecognized command 'unknown' response after my client sends the MAIL FROM:<> command. This occurs if I've configured my email account to use a 8 (Office 365 minimum) or 9 character password, email sends without error if my email account has a 10 character password.

It would help me rule out client-side issues if someone else who has an Office 365 account tries using a 8 or 9 character password on their account.

a. user say:

My co-worker found the problem.

My email client was using strlen to calculate the password length, but the password wasn't null terminated. Thus depending on the length of my password, I would stumble across a null in my buffer after zero or more garbage characters. I'd send the password with these garbage characters, the authentication would pass I guess because the server's looking for \r\n as a password delimiter, ignoring those garbage characters which would be prepended to the next incoming command, MAIL FROM.

Calculating password length correctly fixed the problem.
