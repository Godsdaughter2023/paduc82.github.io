Data validation for a number to three decimal places

I need to use data validation to ensure two factors from a cell, one is that it must report three decimal places (without Excel auto-rounding)  and that the figure must fall between 39 and 41.

At present I can only get the Data validation to work on the second requirement, can anyone help?

Thank you

Rebecca

bernie deitrick say:

You can't require 3 decimal places using Data Validation- you can only limit to 3 decimal places.  If your uses enters 40, 40.0, 40.00, 40.000, etc. Excel will change the value to 40 before any check is performed.  If they enter 40.1234, then you can capture that as being too long.  If you truly want to require 3 decimal places, then you need to use an input box to capture a string instead of a number, and do the checks based on the string value rather than the numeric value.

If you want to use DV and limit to 3 decimal places, then try this custom DV, written for cell A2

=AND(A2>=39,A2<=41,OR(IF(ISERROR(FIND(".",A2)),LEN(A2)>0,LEN(MID(A2,FIND(".",A2)+1,25))<4)))

karl timmermans [outlook mvp 2012-2018] say:

I am an advanced Outlook user so know how to get to the Data and Email window, what I am struggling with is trying to find out how to actually link a data file to an email account.  

If your accounts are configured as "IMAP" - then you cannot alter what data file is associated with the account. Every IMAP account will have it's own <Inbox> folder in the Outlook <Nav> pane which shows the data files associated with the account.

If your accounts are POP accounts - the place to alter where email gets delivered is on the <Email Tab> page of Account settings - highlight the POP account and at the bottom of that window there will be a <Change Folder> button at the bottom of that screen.

For point of reference - outside of changing the delivery location for POP accounts, you do not/cannot "link a data file to an email account" - not under your control. as my previous reply of Nov 16, 2013 in this thread already indicated.
