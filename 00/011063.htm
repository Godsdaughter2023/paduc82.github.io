greg maxey say:

In earlier versions you could use Find:  ^p  with hidden text (as long as hidden text is displayed).  I think this somehow got broken after Word 2007 SP-2.  You can use a macro:

Private Sub cmdDeleteAll_Click()
Dim oRng As Word.Range
Set oRng = ActiveDocument.Range
ActiveWindow.View.ShowHiddenText = True
With oRng.Find
  .Text = Chr(13)
  While .Execute
    If oRng.Paragraphs(1).IsStyleSeparator = True Then
      oRng.Select
    End If
   Wend
End With

End Sub

You can modify the cell margins for the default table style. Create a new, blank document, for example by pressing Ctrl+N. In the Styles pane (Ctrl+Shift+Alt+S), click the Manage Styles button. On the Edit tab, locate and select the style. Click Modify. In the Modify Style dialog, click Format, Table Properties. On the Table tab, click Options. Set all margins to zero. Click OK. diane poremsky mvp (slipstick.com) say:

The mail server adds headers - so the message size will grow a little between when you send it and when it is delivered. It's never going to match exactly because of this. Other MAPI properties are also changed, so the messages won't be identical. This would account for a couple of KB.

Does the message have images in it? Are you using word or the outlook editor?

What type of email account? If you send from Exchange server mailbox and receive in an account which uses a PST, the message will be larger because of how the pst stores messages.

qa guy commelec say: One way is to move the data through an Excel file, add a column, and autofill with =Row() so as to number the rows.

john w. vinson say:

An Access Table has no (usable) order. Doesn't matter where it comes from or how it's entered; it should be viewed as an unordered "heap" of records. Access will present the records to the user in whatever order it finds convenient.

If the sequential order is important to you you must - no choice, no option! - have a field which defines its sort order.

What you could try is to create a local table with all the fields of your text file, and a sequential Autonumber field as a primary key (to provide the sort order). You may be able to import from the text file into this table, or (probably better) link to the text file and run an Append query to add the records to your table. The autonumber will increment (in the order that the records are added), and can be used as the sort field for a Query putting the records in order.
