scottgem say:
If you want to user to see the elapsed time after they enter Start and End times, then use a CONTROL on your form that performs the calc. You do this by setting the ControlSource of the control to an expression.

I.E. =DateDiff("n",[txtStart],[txtEnd])

If you need to include the results of the calculation on a report, then create a query and add a column with the expression to your query, then bind a control on the report to it. 

I haven't hear of any problems with using DateDiff in a calculated field (the new to 2010 calculation datatype), but I'm not a big fan of calculated fields in a table. If there is a restriction on what functions you can use in the expression, I'm even less enamored of it. You might be interested in my blog on the Calculated datatype.

scottgem say:
I agree with Graham, that using DateDiff and calculating the Minutes is the best option. You can then get the Hours using the expression:

Hours = Int(MinutesDiff/60)

You can get the minutes using the expression 

Minutes = MinutesDiff Mod 60

Another point to consider is if your open and close times span a calendar day. Access stores time as a double precision number where the Integer portions represents the number of days since 12/30/1899. The decimal portion represents a fraction of a day (ex 6 hours or 6Am = .25). 

So if the time period spans a calendar day and you don't store the date value, you can get unexpected results.

john korchok say:

Every box on a slide is a shape. This means that if you have a title on a slide with 2 charts, you have 3 shapes on the slide. So it's a good idea to test whether the shape has a chart is before applying the chart resizing:

      If .HasChart Then
      Perform Action
    End If
  End With

uses.

You're depending on the Selection object. This can be a good thing for the first selected chart that is used as the position model for the others. But to apply that position automatically to all the other charts, the selection object won't work. Instead you need to loop through each slide in the presentation. On each slide, you loop through all shapes, checking for whether a shape has a chart. Then you only apply the positioning to shapes that meet the criteria:

Sub ResizeCharts()
  Dim w As Single
  Dim h As Single
  Dim l As Single
  Dim t As Single
  With ActiveWindow.Selection.ShapeRange(1)
    If .HasChart = True Then
      w = .Width
      h = .Height
      l = .Left
      t = .Top
    Else
      MsgBox "Please select a chart with the correct position and size"
    End If
  End With
  Dim objSlide As Slide, objShape As Shape
  For Each objSlide In ActivePresentation.Slides
    For Each objShape In objSlide.Shapes
      If objShape.HasChart Then
        With objShape
          .Width = w
          .Height = h
          .Left = l
          .Top = t
        End With
      End If
    Next objShape
  Next objSlide
End Sub

Please note that if there are ever 2 charts per slide, they will end up on top of one another using this code.
