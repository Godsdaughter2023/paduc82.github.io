andreas killer say:
When the code is located in the code module of a sheet, a reference without a sheet statement refers always to the cell of the sheet where the code is located in.

Sounds complicated... so here is the explanation:

There are 4 points to understand:

a) If this code is located in Sheet1: 

Sub TestA()
  Dim rng As Range
  Set rng = Range(Cells(2, 9), Cells(100, 9))
  rng.Select
End Sub


The variable rng refers to I2:I100 of Sheet1, regardless which sheet is active.

b) You can only select cells in the active sheet. Means if you execute the code above when Sheet2 is active, you'll get the RTE.

The solution to this is to active the sheet before using the PARENT property to have an universal working code. 

Sub TestB()
  Dim rng As Range
  Set rng = Range(Cells(2, 9), Cells(100, 9))
  rng.Parent.Select
  rng.Select
End Sub


c) If the first code above is located in a regular module (within the menu click Insert then Module) then the rng variable refers always to the cells in the active sheet.

Means if Sheet2 is active, the cells I2:I100 in Sheet2 are selected afterwards.

d) If the you refer to a sheet using WITH (or if you write the SHEET statement before each RANGE resp. CELLS statement as suggested by other advisors) you refer to that sheet, regardless where the code is located in.

Note that there is a dot in front of each RANGE and CELLS: 

Sub TestC()
  Dim rng As Range
  With Sheets("Sheet2")
    Set rng = .Range(.Cells(2, 9), .Cells(100, 9))
  End With
  rng.Parent.Select
  rng.Select
End Sub


Sheet2 and the cells I2:I100 are selected afterwards.

That's it.

A final note:

Please never use SELECT, SELECTION, ACTIVECELL, it is slow and error prone. Always refer to the objects directly and call there methods and/or properties.

Andreas.

greginmich say:

The Windows Defender Scheduled Scan task is actually the system task for the Automatic Maintenance scan – although lots of “experts”, including Microsoft’s own documentation authors, have fallen into the trap of thinking that this is some kind of template that users need to add a trigger to in order to activate a scheduled scan (because no trigger appears in the task itself). In versions of Windows 10 prior to 1607, The Automatic Maintenance engine triggered this task on a daily basis, and a Defender Quick Scan would run on a daily basis if a slot of system-idle time long enough to run the scan was available. The Automatic Maintenance settings page is the only user interface for the Automatic Maintenance scheduled tasks, and the tasks themselves should normally be considered “off limits” because of their finely-tuned interaction with the Maintenance engine.

The current issue with the Automatic Maintenance scan seems to be either very widespread; or global, for systems that are running Version 1607 – so the chances of it being caused by malware would appear to be slim to none. Because no corrective action has been taken to date, however, we can’t rule out the possibility that the maintenance scan’s frequency is somehow actually being lowered intentionally, although I don’t think that’s very likely. But since people expect the Automatic Maintenance scan to run on a daily basis (or at least to be triggered on a daily basis), some users might well want to take the corrective action on their own, irrespective of the possibility that this erratic scanning behavior might conceivably be “by design”. None of the workarounds that I’ve suggested requires any special skills or programming ability, so there's no need to be put off by theatrical phantoms.  

Others, especially those who aren’t perfectly careful on the internet (or who share a PC with someone who isn’t perfectly careful), might simply want to replace Windows Defender with another free AV app that gets better ratings for effectiveness, and that might also suffer from fewer of these operational issues that seem to follow one-after-another with Microsoft AV apps. Even though Microsoft AV apps have officially been taken off of their previous “baseline” status at the testing labs, they’re still underperforming on real-world protection and file detection tests. And although I haven’t found a lab anywhere that actually compares AV apps in terms of their basic operational reliability (running, updating, and scanning consistently on typical PC systems), I certainly can’t imagine Microsoft AV apps winning any laurels in such a test.

Even before this reliability issue with the Automatic Maintenance scan cropped up, I’d already decided that the best security strategy for PCs that need to be considered “at-risk”, or for people who are just looking to maximize their AV/Antimalware coverage, is to run a higher-rated free AV app as the real-time AV provider, and then enable Defender’s new Limited Periodic Scanning mode (new in Version 1607):

http://www.tenforums.com/tutorials/51514-limited-periodic-scanning-turn-off-windows-10-a.html

Following my own guidelines, I’ve just started replacing Defender with other free apps on my own test machines, and then enabling Limited Periodic Scanning; and I’ve unfortunately noticed that the Automatic Maintenance scans also usually fail when Defender is running in its Limited Periodic Scanning mode. So this is really a spoiler for the Anniversary Edition of Windows Defender, and you’d think that Microsoft would be anxious to either explain the situation (if the lowered frequency is actually intentional), or address the issue (if it’s not) – but so far there’s been no acknowledgement either way.

GreginMich

As I recall, this method did originally schedule a Windows Defender scan that ran with normal priority in Windows 8; but when the Microsoft documentation authors tried to recycle this method for Windows 10, the scan would only start and run if the system remained in an idle state – exactly like the Automatic Maintenance scan. Since people expect a “scheduled scan” to actually start at the scheduled time and then run to completion without any interruptions, this “Microsoft” method for scheduling a scan has proved to be pretty useless:

https://support.microsoft.com/en-us/instantanswers/66eb0a31-2a59-4c90-ba5b-23bcdfb6f185/schedule-a-scan-in-windows-defender
