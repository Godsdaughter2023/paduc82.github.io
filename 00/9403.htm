mouran yan msft support say:

Hi Dave,

Yes, you are right. There is no such a file type to select when clicking Protection > Malware Filter > Settings.

As a workaround, we can build up a transport rule to block or quarantine the messages which have .dotm files as attachments. Below are the detailed steps:

  1. Go to the Office 365 Exchange Admin Center.
  2. Click mail flow > rules, then click the plus symbol to create a new transport rule.
  3. Under “Apply this rule if…”, select Any attachment… > file extension includes these words, input “dotm”.
  4. Under “Do the following”, choose the option as you like. For example, if you want to delete the message which contains the dotm file as the attachment directly, choose the option “Block the message” > delete the message without notifying anyone. If you want to quarantine the message, choose the option “Redirect the message to” > hosted quarantine.
  5. Click Save.

Note: If you are an Office 365 Small Business plan user, you may not be able to build up such a transport rule to block or quarantine these messages.

Try re-registering Word. You will have to adjust the path in the example to match your machine:

There are several different ways you can reset the proper file associations. Word itself provides a way to do it. Open a command prompt window and, at the command line, enter the following:

winword /r

Nothing seems to happen when you pressEnter, but Word will reregister itself back in Windows. In the process, it resets the file associations for the files it uses, such as DOC and DOT files. You can close the command prompt window and double-click on a document; Word should start right up.

If that doesn't work, try it this way

Re-register the application

Close the app. Open the Run Window: <WIN> R, use the following command:

"C:\Program Files\Microsoft Office\Office\winword.exe" /unregserver

The quotes are necessary, but you may need to change the path so it reflects the location of the winword.exe file on your system. (You can use Windows’ Search tool to locate the path to the winword.exe file.) The /unregserver switch “unhooks” all the references to Word in the Windows Registry.

You should then, immediately, use the following command:

"C:\Program Files\Microsoft Office\Office\winword.exe" /regserver

This command causes Word to rewrite all its Registry keys and to reassociate itself with document and template files. Now you can double-click a document file and see if Windows starts Word and opens the document.
