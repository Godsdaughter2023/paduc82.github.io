The message can come from several different reasons:

Hi Wolfgang,

I’d like to explain that creating a new batch is not the way the incremental sync works. The incremental sync automatically triggers every 24 hours for a migration batch. However, if you delete the migration batch, the incremental sync will be indefinitely ended. And that’s why we suggest migration administrators delete their migration batches only after they have verified that they no longer need any incremental sync.

For your scenario, creating the new batch should not be the option. Since some emails have not been synchronized yet, could you please see if they can be exported out as PST files, and then imported into the online users’ mailboxes? Detailed steps are introduced by the following articles:
Export or back up messages, calendar, tasks, and contacts
Import Outlook items from an Outlook Data File (.pst)

Windows Phone 7 Stuck on Syncing - Microsoft Community

davinci1983 say:

Windows Phone 7 Stuck on Syncing

We have had three users in our environment to who Samsung Focus phones have recently got stuck in "Syncing...".  This problem started surfacing on 6/20/2011.

 

Pulling the battery out does not resolve the problem. Browsing through IE and other data works correctly. 2/3 of the phones have the NoDo update and the 3rd does not. The users had a mix of Exchange, Hotmail, and Gmail accounts. All of the email accounts were stuck in "Syncing...".

 

A master reset resolves the problem (thus far), but this is pretty drastic and undesirable.

 

Are there any known issues?

davinci1983 say:

We found out the problem was caused on our hosted email provider's end. There was a corrupted calendar item on each affected user's mailbox. It generated hundreds on errors in the Exchange server application log. Once the item was deleted in non-cached mode, the phone started syncing and all of the other accounts started syncing as well.

The hosted provider is Intermedia and we are using Exchange 2007. It took several weeks for Intermedia to take ownership of the issue and see the errors in the application log. I hope this helps someone else out.
