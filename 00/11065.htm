SharePoint Online Modern Webpart "see all" link keeps webpart - Microsoft Community

davesnyder99 say:

SharePoint Online Modern Webpart "see all" link keeps webpart view - not desired

Historically, SharePoint list and library web parts could display a particular view (say "home page view", with an abbreviated number of columns or items) and the user could click on the heading to enter the list or library itself and access the "default view", which was different and typically more robust.

In the Modern Site Page experience, instead of clicking on the title of the web part to enter the list or library, you instead select "See All". The problem is that this link brings you to the library but maintains the web part view, instead of reverting to the default view. Now the only option you have to differentiate the web part from the library is to limit the number of items displayed on the web part. Of course the user could switch to a different view, but that is not intuitive.

Please return the option to maintain two different views between the web part and the list/library proper - this is a major change in functionality.
