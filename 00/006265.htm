ray gao msft say:

Hi YazQI and webbrewers3,

 

The View events are issued as you open the document / load the document from SharePoint. It records from both thin and thick clients (both needs to load the document from SharePoint). The value is recorded in the ViewsLifeTime managed property.

 

The Usage Analytics analysis (which does the actual counting) runs once a day, processing yesterday’s view events - and updates both the Analytics Reporting DB as well as Managed Properties in the Search Index  (these properties serve the counts in the search results).

 

NOTE:

It processes once a day. However, it would take 2 days to show up the results since the timer job processes are farm wide (if the usage of the system are high, the process may take longer in the cloud environment based on the previous observation).

 

Also, you can check the view number using the Most Popular Items feature in the library. You’ll see which items have the most views, either recently or ever (from the first time the item became available.)

For more information, you can refer to View Popularity Trends and Most Popular Items.

Ran

    Dim arrRestLoc(9) As String
    Dim bRestricted As Boolean

    Dim i As Integer
    Dim sProjName As String

            If Ans = vbYes Then

        End If

andreas killer say:

Am 09.12.2010 20:16, schrieb Neil Humphries II:

How do I access the user selected filename and path?

You can't do it with this dialog, use FileDialog instead.

Andreas.

Sub FileSaveAs()
  Dim sDest As String
  Do
    With Application.FileDialog(msoFileDialogSaveAs)
      If .Show = -1 Then
        sDest = .SelectedItems(1)
      Else
        Exit Sub
      End If
    End With
  Loop Until Not fRestrictedLocation(sDest)
  ActiveDocument.SaveAs sDest
End Sub
