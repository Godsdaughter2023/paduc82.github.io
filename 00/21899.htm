Although I'm not certain what you mean by the phrase"... ruins the formatting", one thing that concerns me is your statement;

"If I try to slide the right indent on the ruler, it completely ruins the formatting of the remainder of this question and all subsequent questions."

Indentation of any kind is a Paragraph Formatting attribute. Unless you actually select more than 1 paragraph changing the indentation directly should affect only the paragraph where the insertion point is located at the time. There should be no affect on other paragraphs.

The only exceptions I can think of are;

  1. If the Indentation is included in the Style being used & the Style is also specified to Automatically Update. In that case, directly changing the indent of a single paragraph would cause the Style to be updated accordingly. Ergo, all paragraphs to which that Style is applied would reflect the new indent.
  2. You may be dealing with a Table which has no Borders & may be set to not display Gridlines. Some ways to be sure;
  • With non-printing characters displayed do you see any that look like this  ⌑ ?

Sadly, there is a way to format a long document that makes it a terrible chore to do anything with, and it sounds like your predecessor has done exactly that.

It sounds as if the entire document has been formatted with a single style (Normal) and manually applied indents, tabs, and spaces.

What we should do is to re-set the entire document using styles.  You only need two.

I would use Body Text style for the question, and Body Text Indent for the potential answers.

Set the font and line formatting for the Body Text style to be correct.  Set no indents either end of the line.  Then apply Body Text style to each of the question lines. Ensure you have a paragraph mark at the end of each line.

Now set an appropriate left indent on the Body Text Indent style, and again set the font and line spacing appropriately. Apply Body Text Indent to each of the question lines.  Again, ensure you have a paragraph mark at the end of each line.

Now, you may find that you have to get rid of a whole lot of spaces and tabs before the styles can work correctly.

jay freedman say:

How to change it is "completely". On the assumption that the number of misspelled words is much smaller than the number of correctly spelled words, instead of deleting the correctly spelled ones, it will be much quicker to make copies of the misspelled ones in an initially blank document. That's done by this macro:

Sub CopySpellingErrors()
    Dim srcDoc As Document, destDoc As Document
    Dim errList As ProofreadingErrors
    Dim errSingle As Range
   
    Set srcDoc = ActiveDocument
    Set destDoc = Documents.Add
   
    Set errList = srcDoc.Range.SpellingErrors
    For Each errSingle In errList
        destDoc.Range.InsertAfter errSingle.Text & vbCr
    Next
End Sub

john korchok say:
  1. Open Applications>Utilities>Disk Utility.
  2. Select your hard disk in the left-hand pane.
  3. If you're on Yosemite or earlier, click on the Repair Disk Permissions button. If you're on El Capitan, click on the First Aid button. Allow the process to complete, restart your Mac and see if you can open Word.

You can also try restarting in Safe Mode to see if that will fix the issue: OS X Yosemite: Start up in safe modeOS X El Capitan: Start up in safe mode

If neither of those help, try creating a new account in System Preferences>Users & Groups, then log out of your account and into the new one. If Word starts successfully, your user account is corrupted.

 

bob jones aka: cybertaz say:

Since I have no way of knowing what those "responses provided previously" might include, please forgive me if these suggestions are redundant. Sometimes, though, it isn't just a matter of what you do, it's a matter of the order in which they're done :-)

  • Even though Office is current your Microsoft Error Reporting app isn't. Download & install from here: Download Microsoft Error Reporting for Mac 2.2.9 Update ...
  • Do as John suggested to repair disk permissions followed by a Restart
  • Do you have any add-ins or 'system helper' utilities, even though they may not be specifically for use with Word? A safe boot or a safe login can help determine whether something external is having an influence.
  • Following up on the above, launch Word while holding the Shift key to see if that avoids the crash
  • Testing in a New Standard User Account is another good troubleshooting measure

  • Crash on launch often is font related. Run the OS X Font Book app to validate your fonts & resolve duplicates. Delete, don't just disable, faulty fonts & older dupes. Shut down for a minute or two, don't just Restart afterward.
