joeu2004 say: Titus wrote:
When I copy information from one excel spreadsheet to a new one, the new spreadsheet gives me a yellow bar at the top and says automatic update of links has been disable. If I click on enable, my new spreadsheet become slow when I'm doing any editing (e.g. entering words in cell and press enter). I like to know how does this automatic update of links affect the old and new spreadsheet. 

The automatic update of links itself is not the cause of the slowness.  Granted, it might contribute to the slowness insofar as each external link must access another workbook.  But you might need the updates in order for the workbook to behave correctly.

There can be many reasons for the slowness.  The most likely is the overuse of "volatile" formulas (and dependencies on them) and/or the use of full-column ranges like A:A.

I am guessing the latter (A:A ranges) since you mention that you are dealing with an "old spreadsheet".  Although a combination of that in "volatile" formulas is a "perfect storm".

In Excel 2003, A:A references to "only" 65536 cells.  In Excel 2007 and later, it refers to 1+ million cells.  So if you have COUNTIF(A:A,...), instead of doing "just" 65536 comparisons, you are now doing 1+ million comparisons.  That is 16 times slower for one formula.  Multiply that by the number of such formulas and dependent formulas.

Certainly, it might help to enable Manual calculation mode.  But eventually, you will need to enable Automatic calculation or force a full-workbook recalculation (ctrl+alt+F9).  The recalculation might be very slow.

I think it would be better to identify the root cause of the slowness and fix that, if feasible.

I do not recommend that you set manual calculation mode, except perhaps as a diagnostic method; that is, just to "isolate" the problem.

I have little doubt that "manual" mode will mask the problem.  But I do not believe it will help you "isolate" (identify) it.

In any case, your perception is incorrect.

The calculation mode is determined by the state of the first file opened in an Excel instance.  (That is, by the current state, which you might have set manually.)

If the first file (or current state) is in "auto" mode, then yes, the "manual" file will be opened in "auto" mode.

But if the first file (or current state) is in "manual" mode, then the "manual" file will opened in "manual" mode.  In fact, if you open the "auto" file, it will be opened in "manual" mode.
