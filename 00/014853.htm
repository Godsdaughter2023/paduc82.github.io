Assume further that the Enterprise Resource was granted only on Mondays by the Resource Manager. Then the correct duration of the task would have to be 5 weeks. But this is not the case (due to the 100% availability).

I cannot change the availability of the Enterprise Resource in the details of the resource since this would affect other assignments of this resource.

My idea is to use a local resource which is created automatically (by VBA macro) from the Enterprise Resource and holds the project specific availability. The Enterprise Resource would be assigned to a single task just to visualize the commitment of the Resource Manager.

By this I can use all the levelling capabilities of MS Project while the Resource Manager can still see that the resource is assigned to my project.

What do you think? Is there another way than dealing with local resources?

Volker

dale howard [mvp] say:

Volker --

When the resource manager gives you this resource on your project for only one day per week, you should assign the resource to the task with a Units value of 20% and a Work value of 40h.  Assuming the task is either Fixed Units or Fixed Work, Microsoft Project will calculate the Duration of the task as 25 days.  This is the correct Duration under the circumstances you have entered for the resource.  You can then use this schedule for determining if you have resource overallocations anywhere in the project.  There is no need for attempting to use local resources in your situation.  Hope this helps.

volker_h say:

Hi Jan,

another point with task calendars arises when more than one resource works at a task. The manual updating of the task calendars can easily become a nightmare if you think of projects with 30+ resources.

My proposal is as mentioned in my initial post:

1) Have the Enterprise Resource assigned to a single task that represents the duration that this resource works in my project.

2) Exactly this task is then also visible for the Resource Manager and he can set the baseline work for the resource (which represents the availability of the Enterprise Resource in my project). Models like "only Mondays and half day on Friday" is no problem then.

3) My macro creates a local resource from the Enterprise Resource and sets the availability of the local resource according to the baseline work on a daily base for the whole allocation period of the Enterprise Resource.

A) Resource Management has the overview of Enterprise Resources and their project commitments

B) Project Manager has full flexibility by working with local resources which are "linked" to the Enterprise Resources
