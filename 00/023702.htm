rmikesmith say: Neal,

I'll assume your functions are contained in one or more standard code modules.  The simplest way to prevent these from appearing in the function dialog is to add the 'Option Private Module' statement to your code modules.  In that case, you do not need to add the Private keyword to each individual function.  Note, the statement should be placed at the top of the module before any procedures.  If you want to make certain functions available to the user, place them in a separate code module without the 'Option Private Module' statement.  Procedures in such a module can still call procedures within the private module, so no problems there.

Outlook 2007 to Hotmail.com Pop3 setup - Microsoft Community

de.o say:

Outlook 2007 to Hotmail.com Pop3 setup

This worked for me:

1.Leave Outlook off. 2. Log onto Hotmail.com with browser. Click the wheel icon (upper right) and go to Options. Go to Mail/accounts/Pop and Imap. Choose yes and let apps delete, etc.. You can leave the browser up if you want. 3. Go to Start screen and choose Start/Control panel. Choose Mail(32bit). 4. Choose Email Accs. Choose New. Choose Microsoft exchange, etc.. Choose manually configure/next. Choose internet email, pop, etc.. fill in information for Account type Pop3. Incoming mail server=pop3.live.com. Outgoing mail server= SMTP.live.com. Choose More settings. Outgoing=Outgoing same as incoming. Choose advance, Incoming=995 and check SSL, Outgoing, choose=587 and TLS. click next, click finish. Open Outlook and click Send/Receive.
