There is a page at the Word MVP site that shows a method that works for other versions of Word

http://word.mvps.org/FAQs/MacrosVBA/NumberDocs.htm

Unfortunately I don't know how to get this to work in Mac Word 2011 because the PrivateProfileString mechanism used in that article does not appear to work. Or at least, it is not clear what type of file the profile needs to be in order to make the mechanism work.

So here is an alternative method that uses VBA to access a .txt file that stores the last invoice number you used (and nothing else). If the file doesn't exist, the VBA routine should create it in a suitable folder and "seed" it with the first number you want to use.

The first Sub gets the next Invoice number, stores it in a Word Document Variable, and updates all of the DOCVARIABLE fields in the main body of the document. SO in your document, you need to insert a DOCVARIABLE field with the name "InvoiceNumber", e.g.
 a. at the point where you want the number, use cmd-F9 to insert a pair of the special "field braces" {  }
 b. between the { }, type "DOCVARIABLE InvoiceNumber" so you have

{ DOCVARIABLE InvoiceNumber }

If you want to left pad with zeroes, use a "numeric format switch", e.g.

{ DOCVARIABLE InvoiceNumber \#0000000000 }

You can toggle between "field codes" and "field results" view using Option-F9 and update the field result by selecting the field and pressing F9. Until the Document Variable has actually been created, the result will probably be an error message.

So, you would need to put these routines in your normal.dotm (see, e.g.http://word.mvps.org/Mac/InstallMacro.html for some instructions), and associate the macro with a menu option or keystroke so you can run it.

I've done this in a hurry so
 a. it's all quite crude - nothing to help you avoid gaps in the sequence, for example
 b. there's no error checking to speak of
 c. nothing to help you find the .txt file, although it will probably be in 

~/Library/Preferences/Microsoft/Office 2011

e.g.

Macintosh HD:Users:username:Library:Preferences:Microsoft:Office 2011

NB, there are many ways you could store this number but I'd rather not go into details at this point.

The macros...



Sub insertInvoiceNumber()
Dim f As Word.Field
Dim lngInvoiceNumber As Long
lngInvoiceNumber = getNextInvoiceNumber()
' With variables, Word creates them if they do not exist.
If lngInvoiceNumber < 0 Then
  ActiveDocument.Variables("InvoiceNumber").Value = "Error retrieving invoice number"
Else
  ActiveDocument.Variables("InvoiceNumber").Value = CStr(lngInvoiceNumber)
End If
For Each f In ActiveDocument.Fields
  If f.Type = wdFieldDocVariable Then
    f.Update
  End If
Next
End Sub

Function getNextInvoiceNumber() As Long
Const StartNumber As Long = 1000001
Dim lngFile As Long
Dim strInvoiceNumberFullName As String
Dim lngInvoiceNumber As String
strInvoiceNumberFullName = Word.Options.DefaultFilePath(wdUserOptionsPath) & ":InvoiceNumber.txt"
lngFile = FreeFile
On Error GoTo problem
Open strInvoiceNumberFullName For Input As lngFile
Input #lngFile, lngInvoiceNumber
Close #lngFile
lngInvoiceNumber = lngInvoiceNumber + 1
Open strInvoiceNumberFullName For Output As lngFile
Write #lngFile, lngInvoiceNumber
Close #lngFile
Debug.Print lngInvoiceNumber
