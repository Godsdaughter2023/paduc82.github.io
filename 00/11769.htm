ken sheridan say: It's very easy to change the control type as Gina has described, but the reason you have combo boxes may be that you have used the lookup field wizard when designing the table.  If so you might find that the values you see in the text box are not what you expect, but are a meaningless number.

If we assume your table is one of contacts, and take company name as an example, the usual way of setting this up in a relational database would be to have tables like this:

Companies
....CompanyID (autonumber primary key)
....CompanyName
....etc

Contacts
....ContactID (autonumber primary key)
....CompanyID (long integer number foreign key)
....FirstName
....LastName
....etc

So the Contacts table does not include the company name, but the numeric foreign key which references the primary key of the Companies table, which is the 'lookup table' or in the language of the database relational model the 'referenced table'.  If the lookup field wizard has been used when setting up the Contacts table the display control for the foreign key column in Contacts would be a combo box set up along these lines:

RowSource:     SELECT CompanyID, CompanyName FROM Companies ORDER BY CompanyName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

By setting the ColumnWidths property to zero the first column (the numeric CompanyID) is hidden and you see the referenced company name.  The value in the field will be the hidden number, however.

A number of points arise from this:

1.  Where you have a field in a table like Contacts which can include multiple instances of the same value, the company which a number contacts represent in this case, each company's data must be stored in a single row in separate table.  If you simply have a text field into which the name is entered mistakes will inevitably occur (Murphy's Law).  I once found three versions of my own name as the author of technical articles in my own field of work in a badly designed database.  As far as the database was concerned I'm three separate authors!

2.  The company names must not be stored as a value list as the combo box's RowSource.  These are data and a fundamental principle of the database relational model is the Information Principle (Codd's Rule #1).  This requires that all data be stored as values at column positions in rows in tables, and in no other way.

3.  The lookup field wizard is deprecated by experienced developers and should be avoided like the plague when designing a table.  For reasons why see http://www.mvps.org/access/lookupfields.htm.  In the table it does not matter (in fact it's preferable) that you see the actual numeric value rather than the referenced text value, as in the day to day use of the database you should never interface with the data in the raw datasheet view of a table, but only via forms, as you are doing.  In a form you can use a combo box control set up as described above, bound to the foreign key field.

4.  When a combo box in a form is used in this way you can still type the company name value into it, and it will progressively go to the first match as each character is entered.  Combo boxes now have a List Items Edit Form property which allows you to define a form which will open so that you can add a new item to the referenced table if it is not already represented in the list.  This avoids the need to write code in the combo box's NotInList event procedure as was previously necessary, though using the latter does still have advantages.  You'll find a examples in NotInList.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

The item list is the list of messages, contacts, tasks, or notes. By default, the item list is displayed vertically between the navigation pane and the reading pane.

Unfortunately, these type of malware attacks are difficult to keep up with because they trick you into letting them install. They usually come from an infected web site, and usually through an advertisement. You get a pop-up from the infection and you click it to close the pop-up - which allows the infection to install. They can also be delivered in a "drive-by" fashion with no action needed by the user due to the system being unpatched, no matter what security software is running.

When you encounter one of these fake virus pop-ups while browsing, immediately do the following:

 

-Do not touch any browser window to close it or browse further.

-Use the key combination <ALT>+<F4> to close all running programs, especially the web browser

--or--

-Immediately press Ctrl-Alt-Del and bring up Task Manager and forcibly end all instances of iexplore.exe, if using Internet Explorer, or the executable for your browser for any other web browser.

--or--

-Go to Start/Shut Down and restart the PC without touching any browser windows.

-If you used task manager to close browser instances, reboot the machine.

-Then go to Control Panel/Internet Options and delete all temporary Internet Files and cookies. If you are using an alternate web browser, open the browser settings to do the same - delete the local cached files and cookies.

-Perform a full scan with MSE.

 

The above steps should prevent the infection from taking hold.
