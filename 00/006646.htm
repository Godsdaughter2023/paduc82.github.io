suzanne s. barnhill say:

If you make changes in the original, then save it, then Save As to another name and Undo the changes and save, you'll have two different documents, but the second document will be the same as your original. Alternatively, if you meant to Save As before making the changes, so you need to retain the changes in a new document but save the original without the changes, you have three options:

  1. If you haven't saved the document with the changes, you can go ahead and Save As under a new name; the original document will be unaffected.
  2. Or you can Save As under the new name, with the changes, then Undo all the changes and use Save As to resave under the original name. Then close the original document and reopen the new document and continue with the desired changes.
  3. Or you can Undo all the changes and save the document under the original name (if you've actually used Undo to get back to the original state of the document when you opened it, it won't really be saved--this is just for your peace of mind). Then Save As under the new name and use Redo to repeat all the changes you had previously made. Continue with changes and Save before closing.

marshall barton say:

The conditions in a query's WHERE clause (criteria) are kind of like IIf that just returns True (accept record) or False (reject record).

 

I think you just want to use:

 

WHERE [Security Deposited] < [Security Deposit Req]

 

Or, if either of those fields might be Null and you want to see these records too:

 

WHERE Nz([Security Deposited],0) < Nz([Security Deposit Req],99999999)
