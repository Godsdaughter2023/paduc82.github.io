bob jones aka: cybertaz say:

I'm afraid I don't understand what you're attempting to describe...

The captions used in the Header Row of a Table appearing in place of the default Column Headings [A, B, C, etc.] is strictly for on screen viewing. It's merely a convenience to enable accurate ID of the column when the row containing the actual captions is not within the viewing area. It has no bearing on printed output. If the column is wide enough to display the full caption on screen in the Header Row of the Table it should print fully, as well.

One possible exception is due to a combination of font & printer metrics. If the column width is very tight it could cause the caption to be cut off when printed. Correcting it usually can be done by adjusting the Print Quality to a higher level in the Print dialog. In some cases a very minor widening of the column may be necessary.

As for your initial question, the answer is no, the program has not been modified to enable formatting of the default Column Headings & Row Numbers. Nor is there any likelihood that there will be any design changes due to the point in the life cycle of the version.

I don't claim to be any sort of model or example, but speaking from my own experience I have used literally every release of Mac Office on numerous Macs over the last 20 years including almost every version of Mac OS. I can honestly say that I do not recall ever having had an update fail to apply.

Additionally, reinstalling Office is useless [at best] unless a proper & complete removal is done first. It isn't the installed software that is the source of operational issues. I'm by no means saying that Mac Office is without bugs or design flaws, but any such shortcomings just get reinstalled along with everything else. Typically, it's the user-specific files generated & modified after installation or local factors [such as add-ins] that have a tendency to cause problems. Reinstalling Office does not remove or replace those user files because they are not in the installation package to begin with. Further, reinstalling without fully removing reverts the base installation to the update level of the source media. That can cause components to get out of sync causing more issues to arise & make re-applying updates a challenge.

Copy the IDs to column F, use Data\Remove Duplicates to get unique IDs

Write "[end]" after the last ID in column F and column A (we need an end marker for the last section)

Formulas:

G2:  =MATCH(F2,A:A,0)
H2:  =MATCH(F3,A:A,0)
I2:  =QUARTILE(OFFSET($D:$D,$G2-1,,$H2-$G2),1)
J2:  =QUARTILE(OFFSET($D:$D,$G2-1,,$H2-$G2),2)
