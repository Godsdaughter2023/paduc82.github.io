I'm still not sure I'm doing the same thing as you, but I'm not able to get my copy of Word 2010 to display the rectangle instead of the black circle.

Usually, the rectangle indicates that the font applied to the text doesn't include a character (technically, a "glyph") for the symbol. However, if you use the Insert > Symbol dialog to insert the symbol, Word does some internal magic to protect the symbol's display regardless of the font of the surrounding text. You can instead type the hexadecimal Unicode number for the black circle, 25CF, and press Alt+X to convert the number to the symbol.

Anyway, here are the three methods I tried, each of which successfully replaced the specified text with the [●].

  • In the Replace With box of the Replace dialog, type [25cf and press Alt+X; then type the closing ].
  • Somewhere in the document, use the Insert > Symbol dialog to put the black circle between the brackets, or type [25cf and press Alt+X, and type the closing ]. Cut the three characters to the clipboard. Open the Replace dialog and paste the three characters in the Replace With box.
  • Enter the characters in the document as before and cut them to the clipboard. Open the Replace dialog and type the code ^c (which means "the contents of the clipboard") in the Replace With box.

                        net start msiserver

Can Excel Display Nanosecond-stamped data? - Microsoft Community

judy peng say:

Can Excel Display Nanosecond-stamped data?

above is a snapshot of some trading data in sas format, where the time is show in hh:mm:ss.000000000 with nanosecond precision. But when I convert this sas file to an excel file, I couldn't get the numbers displayed right. How can I format my data? (using hh:mm:ss.000000000 does not solve this problem).

joeu2004 say:

Errata....  I wrote:

You can "extract" the number of seconds with nanosecond precision using the following formula:

=ROUND((C2 - TEXT(C2, "[h]:m"))*86400, 9)

Wrong!

Unfortunately, the format "[h]:m" rounds numeric time to the second. So if the original time is "9:01:59.512987654" (entered as text), the formula above would result in a negative value because TEXT(...,"[h]:m") would result in "9:02".

A more reliable way to "extract" the number of seconds with nanosecond precision is:

=ROUND(MOD(C2*1440, 1)*60, 9)

joeu2004 say:

Judy wrote:

Thanks for the reply, but I have tried this method and tried on different computers and still cannot get what I expect.

I want the data to be shown in the form hh:mm:ss.000000000 (as shown in the above graph) in NANOseconds.

Is that excel does not have this function?

That is correct. The idea that repairing Office is needed or might fix the problem is preposterous.

If you entered the data as shown, Excel recognizes it as numeric time. But the nanoseconds are rounded to milliseconds, so you lose precision.

For example, if you enter 9:01:26.274517892, Excel treats that as the numeric time 9:01:26.275. Not only is the format limited to milliseconds, but also the actual internal value is 0.375998553240741, the internal representation of 9:01:26.275.  (Internally, numeric time is represented as 1/24 of a day.)

So it is important that you import that column of time as text. If the source of the data is a text file (TXT or CSV, for example), use the Import Text File wizard to bring the data into Excel instead of opening the file directly. In Step 3, select Text for the Column Data Format.

You can convert the text to numeric time with nanosecond precision using the following formula (in C2, for example):

=LEFT(B2, FIND(".",B2)-4) + RIGHT(B2, 12)/86400

Unfortunately, that still displays 9:01:26.275 when formatted as Custom [h]:mm:ss.000 . But in this case, that is only an Excel format limitation. The actual internal value is 0.375998547660787, which is different from the internal representation of 9:01:26.275 above.

You can "extract" the number of seconds with nanosecond precision using the following formula:

=ROUND((C2 - TEXT(C2, "[h]:m"))*86400, 9)

It would be prudent to test these formulas with a large set of data to be sure that no errors arise due to binary arithmetic anomalies.

suzanne s. barnhill say:

Since your description doesn't actually mean anything in Microsoft terms, perhaps you can better explain what you're doing. You've used the ruler to set tabs and indents in TOC entries, perhaps? Because the TOC styles are set to update automatically, this will result in modification of those styles, and they will stay modified unless you modify them again.

But if you're asking how to prevent the TOC from updating and wiping out manual formatting you've applied, then the only way is to lock the TOC (Ctrl+F11). You will not be able to update the TOC unless you unlock it (Ctrl+Shift+F11), and unless you update only page numbers, your direct formatting will be removed.

Perhaps if you tell us what you're trying to achieve, we can help. You might also see http://wordfaqs.ssbarnhill.com/TOCTips.htm for information on formatting a TOC.
