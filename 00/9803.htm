marshall barton say:

Empty subreports have no content and will display nothing at all.  Even the controls in an empty subreport do not exist so you can not refer to them (thus an error).  Access provides the HasData report property so you can check if the (sub)report is empty and deal with the situation.

I expect your  =IIf(IsError(...  worked so I think the problem is elsewhere.  OTOH, IsError will pick up a multitude of different issues so problems other than no records in the subreport are covered up.  The best way to check if a subreport has no records is to use the HasData property:

   =IIf(rpt_WeeklySummary_DailyData_Wednesday].Report.HasData, rpt_WeeklySummary_DailyData_Wednesday].[Report]![Text9]), 0)

I suspect there is something about the Text9 text box or the running sum text box's setting that is not right.

The running sum text box needs to be in the main report detail section and its RunningSum property should be be set to Over All.  The main report grand total text box in the Report Footer (not page footer) needs to use an expression like  ={name of running sum text box}

I'm not clear about what you have in the subreport's Text9 text box, but it needs to be in the subreport's report footer section and use an expression like  =Sum({name of record source field to total}

Note:  the Sum function can only refer to fields in the subreport's record source table/query, it can NOT use the names of any controls in the subreport.

dhookom say:

Another possible solution is to pull the sums into the record source of the main report.

Let's assume the report is Orders for a particular Customer over a period of time. The main report record source would be the Orders table (no details). The subreport is based on the Order Details table. You want the sum of the order quantities from the detail section to appear in the main report. 

Create a totals query of Order Details table (subreport record source) by OrderID with a sum of the Quantity field. Then just add this totals query to your report's record source query and join on the OrderID fields. This will make the sum of Quantity available in the main report so you don't have to reference the subreport or set any running sums or other messy stuff.

anna ma msft say:

Hi Nathan Paul.

 

Thanks for your clarification. It is an expected behavior that tasks will automatically save into default Task fold when you create task via shortcut key and save it via clicking Save & Close button, and it is not feasible to change the default task folder to subfolder.

 

To save task to subfolder when not in Task view, you may click File-> Move to Folder as a workaround.

 

Meanwhile, if this feature is important for you, I suggest that you may submit a feedback via https://outlook.uservoice.com/forums/322590-outlook-2016-for-windows, your suggestions are valuable for the product improvements.
