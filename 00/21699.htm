Actually, I FOUND A WAY TO DO THIS!

1. Go to "Calendar" in Outlook (click the calendar icon at the bottom of the screen).

2. Click on "View" tab at the top of the page.

3.  Click on "Change View," then use your mouse to select "Manage Views" (you will see a screen pop up, titled "Manage All Views").

4.  Click "New" and give your new view a title, such as "Reminders" (don't worry about selecting a "type of view" and make sure the "All Calendar Folders" button is selected. Click ok.

5.  When the "Advanced View Settings" screen pops up, select "Columns" and add the "Reminder" field in the left column. Close that screen. You should see your new view (whatever you titled it) in the list of items on the "Manage All Views" screen.

6.  Save and close all the screens.

7. Now, go back to the "View" tab. When you select "Change View," you should see a new icon that is the title of your new view. When you select that icon, your reminders will appear in a list that is printable.  VOILA! :)

You can switch back to your normal calendar view anytime by clicking View, Change View, Calendar.

Email "send object" does not work between MS Access 2003 and - Microsoft Community

dave vancoevering say:

Email "send object" does not work between MS Access 2003 and Outlook 2013

Email "SendObject" code in MS Access 2003 no longer works with MS Office 2013 - Office 365 Pro Plus (Outlook 2013):

DoCmd.SendObject acSendNoObject, "No Title", acFormatHTML, Email_Address, Email_cc, , Email_label, Email_text, True

Returns:  Run-time error ‘2046’:  The command or action ‘SendObject’ isn’t available now.

Questions:

Is MS Access 2003 compatible with Outlook 2013 with regard to "sendobject" vb code?

Would like suggestions on how to fix this and get the email working again.....

Solution:
Looks like we found a fix for the issue.  It seems to be working good on all Windows XP and MS Office 2013 machines.

Was looking at the way some programmers were writing email code and found this.

Not sure why the new coding is recognizable to the old MS Access 2003 version but it accepted it and works.

Other coding was left out for clarity (just showing basic lines).

Replaced:

    DoCmd.SendObject acSendNoObject, "No Title", acFormatHTML, Email_Address, Email_cc, , Email_label, Email_text, True 

With:

    Dim OutApp As Object

    Dim OutMail As Object

    Set OutApp = CreateObject("Outlook.Application")

    Set OutMail = OutApp.CreateItem(0)         

    On Error Resume Next

    With OutMail

        .To = "Email address"

        .CC = "Email address"

        .BCC = "Email address"

        .Subject = "Example"

        .Body = "This is it.”

        '.Attachments.Add ActiveWorkbook.FullName   'I turned this off

        .Display                                    'Can use .Display to edit email or .Send to immediately send email without edit.

    End With

    On Error GoTo 0            

    Set OutMail = Nothing      

    Set OutApp = Nothing

End Sub

Very helpful thank you for the post I have taken your code made it it a utility sendEmail with some error checking for anyone else who may find it useful.

 '---------------------------------------------------------------------------------------
'Optional ByVal sendNow As boolan = False)

' Procedure : sendEmail
' Author    : John Moore
' Date      : 07/12/2015
' Purpose   : Replacement for DoCmd.SendObject for use with Outlook 2013 and 2016
'
' Ref       : https://msdn.microsoft.com/en-us/library/office/ff861252.aspx and
'           : http://answers.microsoft.com/en-us/office/forum/office_365hp-outlook/email-send-object-does-not-work-between-ms-access/e4a35c8d-a276-4ec9-8906-c01405a2c165
'           : http://www.rondebruin.nl/win/s1/outlook/mail.htm
'---------------------------------------------------------------------------------------
'
    Dim outlookApp As Object
    Dim outlookMail As Object
    
   On Error GoTo sendEmail_Error

    If Len(toField) = 0 Or Len(subject) = 0 Or Len(body) = 0 Then
        Call MsgBox("The To, Subject and Body cannot by empty ", vbExclamation, "Error in sendEmail sub")
        Exit Sub
    End If
    
    
    Set outlookApp = CreateObject("Outlook.Application")
    Set outlookMail = outlookApp.createitem(0)
    
    With outlookMail
        .to = toField
        .subject = subject
        .body = body
        .cc = cc
        .bcc = bcc
        If Len(attachFile) > 0 Then
            If FileThere(attachFile) Then
                .Attachments.Add (attachFile)
            Else
                Call MsgBox("Cannot find the attachment  ", vbExclamation, "Error in sendEmail sub")
            End If
        End If
        If sendNow Then
            .send
        Else
            .display
        End If
    End With
    
    Set outlookApp = Nothing
    Set outlookMail = Nothing

   On Error GoTo 0
   Exit Sub

sendEmail_Error:

    MsgBox "Error " & Err.Number & " (" & Err.Description & ") in procedure sendEmail"

End Sub

Function FileThere(FileName As String) As Boolean
     FileThere = (Dir(FileName) > "")
End Function
