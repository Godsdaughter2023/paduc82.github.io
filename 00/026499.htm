As others have noted, "and so on" doesn't cut it.  You need to enumerate the L-values for all 8 possibilities.

Nevertheless, we can propose a solution.  Think of columns A, B and C as binary digits:  column A is 4, column B is 2, and column C is 1, each multiplied by 0 (empty) or 1 (text).

If you are willing to relabel L-values in your example, the simplest solution is:

="L" & TEXT(4*ISTEXT(A1)+2*ISTEXT(B1)+ISTEXT(C1),"00")

That results in the following table:

A             B             C           D
empty    empty    empty    L00
empty    empty    text         L01
empty    text         empty    L02
empty    text         text         L03
text        empty     empty    L04
text        empty     text         L05
text        text          empty    L06
text        text          text         L07

Alternatively, if you would like the flexibility to assign arbitrary labels, you can use the following formula, following your example L-values ([EDIT] modified with the missing L-values that you included in a follow-up posting):

=CHOOSE(1+4*ISTEXT(A1)+2*ISTEXT(B1)+ISTEXT(C1),
"L00","L03","L01","L02","L06","L08","L04","L05")

PS:  If by "empty", you really mean "appears blank", i.e. empty or the null string (""), use the following expression instead of ISTEXT:  4*(A1="")+2*(B1="")+(C1="").

Required folder privileges
 
If you plan to share a database, the .mdb file should be located in a folder where users have read, write, create, and delete privileges. Even if you want users to have different file privileges (for example, some read-only and some read-write), all users sharing a database must have read, write, and create permissions to the folder.
