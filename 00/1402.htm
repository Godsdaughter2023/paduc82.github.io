ken sheridan say: You can concatenate the values into single string expression by means of a VBA concatenation function.  You'll find an example in Concat.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes a report which concatenates values from rows in a Northwind table using a function which calls the very efficient GetString method of the ADO recordset object.  The right hand column in the report does what you are aiming at, but because the values are much longer than yours it wrap's over multiple lines.

However, you can achieve a horizontal layout with no VBA whatsoever by returning the Results in a multi-column subreport with across-then-down column layout.  In your case I think this would also achieve a better standard of presentation as the columns of numbers would align vertically.  The ClubMembership demo in my same OneDrive folder is an example.  This little demo file contains a report rptAddresses_Sub which is based on an addresses table.  In the report is a subreport based on a People table.  This is a multi-column report with across-then-down column layout.  The columns in your subreport would be much narrower of course.

In the same file is another report rptAddresses_Code which produces the same results in a single report by manipulating the layout in code in the report's module at runtime.  Ignore this, however, other than as a technical curiosity.  I only produced it in response to a question by a reader of a databases column written by a contact of mine in a magazine.  For some reason the reader specifically wanted a code solution.  A code-free report/subreport solution is far simpler.
