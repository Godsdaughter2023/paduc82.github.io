adding commands to the the right click pop-up. - Microsoft Community

jennyl_73 say:

adding commands to the the right click pop-up.

Can you add additional commands to the the right click pop-up when you are in a cell?? using excel 2010.  thanks
hansv mvp say:

Added: my previous reply didn't appear, so I reposted it. Now there are two...

 

Create these two macros:

 

Sub CreateSumFormula()
    Dim rng1 As Range, rng2 As Range
    Set rng1 = ActiveCell.Offset(0, -1)
    If rng1.Offset(1, 0) = "" Then
        Set rng2 = rng1
    Else
        Set rng2 = rng1.End(xlDown)
    End If
    ActiveCell.Formula = "=SUM(" & Range(rng1, rng2).Address(False, False) & ")"
End Sub

Sub AddAutoSum()
    Dim cbr As CommandBar
    Dim cbb As CommandBarButton
    Set cbr = CommandBars("Cell")
    Set cbb = cbr.Controls.Add(Type:=msoControlButton)
    With cbb
        .Caption = "Auto&Sum"
        .Style = msoButtonCaption
        .OnAction = "CreateSumFormula"
    End With
End Sub

 

Run the AddAutoSum macro once. This will add the AutoSum item to the right-click menu.

To use it, right-click the cell in which you want to add the AutoSum formula, then select AutoSum from the context menu.

a. user say:

Unfortunately, the AddChart method is not available in Mac Word 2011 VBA. 

Also, the Chart class is not available in Mac Word 2011 VBA, which means that even when you insert a Chart object into a Word document, you cannot manipulate it in VBA in the same way that you can in Windows Word.

This also means that you cannot create your Chart in Windows Word, then open the Document in MAc Word and modify the Chart using VBA.

As far as I can tell, the only other way to insert something similar using VBA in Mac Word would be to insert an Excel Chart Object using AddOLEObject. But
 a. I do not think that works in Mac Word 2011 either. You just see an error.
 b. Although an Excel Chart object is very similar to a Word Chart Object, the user has to work with it in a different way from a Word Chart. 

Even though you cannot insert an Excel Chart Object using AddOLEObject, once you have an object like that in your document, you can open it for editing programmatically, using, e.g. 

Selection.Shaperange(1).OLEFOrmat.DoVerb VerbIndex:=wdOLEVerbPrimary

But I do not think that you will be able to work with that object programmatically, and even if you can, I do not think it will be easy.

Sorry!
