Outlook needs a data file to contain the non-mail default folders like Contacts, Calendar, Tasks, Journal, and Notes.  IMAP data files cannot contain non-mail folders.  Also, you cannot make an IMAP data file the default data file for this very reason.  Thus, when using an IMAP account, you must have at least two data files - one to be the default data file and hold the default folders, and one to hold the IMAP folders.

By its very nature, IMAP accounts leave messages on the server and the contents of the IMAP data file must match exactly the folder structure on the server.  Since every IMAP mailbox can have a separate folder structure, every IMAP account in Outlook must have its own data file.  There's no way to have multiple IMAP accounts share the same data file.

In a macro, you need to show "high-order Unicode" characters like that as character numbers.  The internal text strings in VBA will not go above ANSI (128).

That character is Unicode: U+0151, UTF-8: C5 91.

Which makes it a very simple macro:

Sub Macro1()
'
' Macro1 Macro
'
'
    Selection.InsertSymbol CharacterNumber:=337, Unicode:=True, Bias:=0
End Sub

Note that CharacterNumber expects a DECIMAL integer, so I converted it. In Word 2011 VBA the function to convert Unicode (Hexadecimal) to decimal does not exist, you need to convert the numbers using a calculator.

Explanation: by default, for 3 icons, Excel calculates the 67th percent and 33th percent. 67th percent = min + 0.67 * (max-min) = 2 + 0.67 * (95-2) = 64.31. 33th percent = min + 0.33 * (max-min) = 2 + 0.33 * (95-2) = 32.69.  A green arrow will show for values equal to or greater than 64.31.
