Assuming that there is no hardware problem with your SD card -

1. Right click on Windows Start > Device Manager > Click the topmost entry i.e. your device > Click Action in Menu > Click Scan for hardware changes
Now you should be able to see your SD card

2, Right click on Windows Start > Device Manager > Expand Disk Drive > Locate the SD card > Right click on this > Uninstall > Reboot your computer
Now it should be detected

3. Use the Hardware & Devices Troubleshooter
Type Troubleshoot in Windows Start Search box > Click Troubleshoot > Click Hardware & Devices > Run the Troubleshooter

4. Whether the Drive is assigned a Drive Letter or not
Right click on Windows Start > Click Disk Management > If you see your SD card is listed there but there is no Drive Letter > Right Click it > Click Change Drive Letter and Paths > Add > Click Assign the following drive Letter > Click in the Drive Letter List Box > Choose an appropriate Drive Letter > OK > OK

5: Fix USB Root Hub
Windows Key+X > Click Device Manager > Expand Universal Serial Bus controllers > Right-click USB Root Hub option and click Properties > Go to Power Management tab and uncheck the Allow the computer to turn off this device to save power. Click OK. > If you have more than one USB Root Hub, repeat this step for all

6. Update / Reinstall Generic USB Hub
Windows Key+X > Click Device Manager > Expand Universal Serial Bus controllers > Right click on USB Hub > Select Update Driver Software > Browse my computer for driver software > Click on Let me pick from a list of device drivers on my computer > From the list, select Generic USB Hub > Next and follow onscreen instructions > Reboot your Computer

If problem not solved, right-click USB Hub of your USB drive and select Uninstall
Reboot your Computer to install driver automatically

I have noticed something else that is very interesting; svchost.exe is using up an awful lot of your disk space.

svchost.exe is a Windows program that is used to handle various processes for other programs. These programs can be pre-installed with Windows 10, or something you have downloaded from the internet. In your case, it is probably a program that cannot run in tablet mode or a Windows service that has not been designed for your model. You can find out (apparently) what processes are using up svchost.exe by doing the following:

  1. Open Task Manager (as mentioned in my previous reply)
  2. Click 'Show processes from all users'.  If you're prompted for an administrator password or confirmation, type the password or provide confirmation.
  3. Right-click an instance of svchost.exe, and then click Go to Service(s). The services associated with the process are highlighted on the Services tab.

  1. Search your computer (using cortana) for 'Device Manager'.
  2. Open it.
  3. Find the 'Monitors' section and expand it.
  4. Right click on your monitor.
  5. Click 'Update Driver Software...'.

  7. Follow the steps it gives you.
  8. See if it works!
