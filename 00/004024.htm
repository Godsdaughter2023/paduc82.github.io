joeu2004 say:

Kristin wrote:

I have a formula, =IF(K57900<>"SALES",AND,IF(AJ57900="SIC","Review Plan Type"," "))

I'm trying to flag the data where the Job Function is not equal SALES, and the Bonus type is SIC, with the Text "Review Plan Type".    This formula is returning the flag where the Function is equal to Sales, instead of when Function does not equal sales.  

 

Your syntax for "not equal" is correct (<>).  It is you syntax for AND that is wrong.  Write:

=IF(AND(K57900<>"SALES",AJ57900="SIC"),"Review Plan Type","")

Note:  I changed " " (with a space in between) to "" (the null string).  It is a good practice.  It might make other dependent formulas easier to write.

don cuthbert say:

You can create, or recreate a dual, or multi-boot menu quite easily, using a simple command at an Administrator Command Prompt.  You just need a drive mapped (within one of the running Windows installs) to the other volume that you want to boot.  (This assumes that you have another Windows image on a volume and just need a boot menu item for it).

I install most Insider builds to a small 64GB partition, which I give the drive letter I:.  To create (or restore) a boot menu for Windows on the I: partition:

bcdboot I:\Windows

bcdedit set {default} description INSIDER

The above adds a boot menu item for Windows on the I: volume, makes it the default OS and then gives it the label INSIDER in the boot menu. Otherwise, both entries will say "Windows 10" and it can be confusing which OS you are booting to.  Use whatever drive letter and description you want. Put the description in "quotes" if it contains any spaces. The second command is optional, if you want to relabel the boot menu choices to tell them apart more easily.  Others here have already provided some other valid approaches.

Note that you must always set the Windows that you are attempting to upgrade as the Default OS prior to upgrade. Otherwise the required restarts end up booting to the wrong OS, unless you successfully catch it at every restart during the upgrade.

Usually, once I have the menu setup, I remove the drive letter mapping from my main Windows using diskpart or Drive Management, which prevents me from accidentally using that volume.  The boot menu does not require the volume to be be mapped, but it makes it easier to create the boot menu item.
