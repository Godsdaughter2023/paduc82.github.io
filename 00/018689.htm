Is this possible?

justin cavida msft say:

Hi Serena,

We would like to inform you that this is not possible by design. Since .docx and .dotx are different file types and SharePoint only keep version history to the same file types. As a possible workaround, you can sync your library in your local drive then change the file extension from .docx to .dotx. Make sure that the 'File name extensions' option is being shown.

After that, you can upload your .dotx file normally and SharePoint should ask you to replace the existing file. By doing this, the version history of the file should be retained.

UserID |First name  | Last Name | Username

 

I would like to add a password column as well and generate random passwords for the number of records found in my database , example if I have 50 records then I need to generate 50 password , 1 for every record

Any idea ?

john p spencer say:

Use an update query to populate all the records.

UPDATE YourTable
SET Password = fGenPassword(10,[UserName])
WHERE Password is Null

To do this is query design mode

Open up a query on your table

Set its type to UPDATE query

Select the Password field

Set the criteria to Is Null

Set the Update To: fGenPassword(10,[UserName])

 

If you want to generate different lengths of passwords change the expression to somethng like the following.

fGenPassword(Nz(Len([First Name]),10),[UserName])

john w. vinson say:

What are the better ways ?

 


As suggested upthread, include both a Password field and a date/time field PasswordExpires. When you create a new user give them a password that's really easy - "password" for example! - and set the PasswordExpires field to =Date().

Then in your logon process check the expiration date. If it's less than or equal to Date() make them assign a new password of their own choice. You can enforce "strong" passwords by (say) ensuring that their entry is of mixed case or contains at least one number:

Dim blnPwdOk As Boolean
If StrComp([txtPassword], LCase([txtPassword]), 0) = 0 Then
  MsgBox "Password must contain at least one uppercase letter", vbOKOnly
  blnPwdOk = False
Elseif StrComp([txtPassword], UCase([txtPassword]), 0) = 0 Then
  MsgBox "Password must contain at least one lowercase letter", vbOKOnly
  BlnPwdOk = False
< use Instr to check for digits and/or special characters >

You may also want to set the PasswordExpires field to some date (say three months in the future) after you update the password.
