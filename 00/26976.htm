veeya say:

Dear Jack:

I just wanted you to know that I resolved the myPhone issue 5/2/10.

You know how there are a lot of URL requests involved from the point of login to the myphone storage destination?
Well...somehow, the cookie of one of the websites (of one of the urls that the user would probably never have any knowledge of) got blocked
-->livefilestore.com

I discovered that [livefilestore.com was blocked] while perusing my registry when I saw there were a whole bunch of websites with the same data value as [livefilestore.com]. It occurred to me that I was looking at the list of websites to Always Block.

So, I went to Internet Options>Privacy-->Sites and sure enough...it was true: I found a number websites that were set to Always Block, that corrresponded to websites found in the registry having the same data value). And that is why--after logging on--the browser would never take me to my destination.

After I changed livefilestore.com's data value (in registry) to Always Allow...I've been able to get to my myPhone content successfully.

Don't worry...I'd never screw around in registry when you can use the graphic interface more safely. But, this was one of those instances where it was clear that changing the data value would be a safe action to recover from.
