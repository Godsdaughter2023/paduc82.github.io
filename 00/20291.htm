ken sheridan say: You don't say how the value is assigned to the variable X, but why use a variable at all?   The value appears to be in the fourth column of the RowSource, so you can use the following in form's BeforeInsert event procedure to assign the current date to a bound OpenDate control when a new record is entered in the form:

    Me.[OpenDate] = VBA.Date

and set the ControlSource property of an unbound ExpiryDate control to:

    =DateAdd("d",[cboChemical].Column(3),[OpenDate])

where cboChemical is the name of the combo box.  The Column property is zero based, so Column(3) is the fourth column.  There almost certainly should be no ExpiryDate column in the table of course, as it is determined by the OpenDate value, so to include such a column would introduce redundancy and the risk of update anomalies.  The exception would be if the Expiry Date can legitimately be edited independently of the value of the Open Date, in which case a column would be needed and the value would be assigned to a control bound to this column in the AfterUpdate event procedure of the cboChemical  control with:

    Me.[ExpiryDate] = DateAdd("d",Me.[cboChemical].Column(3),Me.[OpenDate])

amit tandon say:

Enter below formula as an array formula (Ctrl+Shift+Enter):

=SUM(--(FREQUENCY(IF(C7:L13<>"",ROW(C7:L13)),ROW(C7:L13))>0))

Counts non-blank rows.
