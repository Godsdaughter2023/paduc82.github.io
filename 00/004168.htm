oldoutboardjim say:

Converting Fractions To Decimals

Ah, the good old English fraction system!  Sheesh!  As an engineer, I frequently need to lookup or specify items as fractions, although I usually measure in decimal inches.

I know Excel can convert decimals into fractions, but which fractional value is my original value closest to? It would be handy for me if I could in-turn convert those fractional values into their true decimal equivalents. For example, if I convert .180” to fractions, the returns are 3/16 (‘as sixteenths’), 1/8 (‘as eights’), and 1/4 (‘as quarters’). In this case, the original value is closest to 3/16.  What I would like to do is take the returns (fractional value), automatically convert them to decimals, and then see which value is closest to the original value. For example, 3/16 = .1875, 1/8 = .125, and 1/5 = .250 In this case, 3/16 is closest.

Anyway, I cannot find a way to extract the fractional values as they appear in the cells. Excel treats all of them as the original value (.180). I would like to recognize “3/16”, “1/8”, and “1/4” as distinct values, so I can convert them to their true decimal values (.1875, .125, and .250, respectively).

How can I do this?

joeu2004 say: Jim wrote:
Allow me to re-state this:
Input [cell A2]: .180 formatted as ‘number, 3 decimal places’
Return [cell B2]: =$A$1 formatted ‘as sixteenths’, = 3/16
Return [cell C2]: =$A$1 formatted ‘as eights’, = 1/8
Return [cell D2]: =$A$1 formatted ‘as quarters’, = 1/4
Okay, I want to automatically calculate the actual, fractional values of the returns in B2, C2, & D2 – in other words, 3÷16=.1875, 1÷8=.125, and 1÷4=.250

Allow me to restate my suggestion.

B3:
=LEFT(TEXT($A$1,"0/16"),FIND("/",TEXT($A$1,"0/16"))-1)
/ MID(TEXT($A$1,"0/16"),FIND("/",TEXT($A$1,"0/16"))+1,99)

C3:
=LEFT(TEXT($A$1,"0/8"),FIND("/",TEXT($A$1,"0/8"))-1)
/ MID(TEXT($A$1,"0/8"),FIND("/",TEXT($A$1,"0/8"))+1,99)

D3:
=LEFT(TEXT($A$1,"0/4"),FIND("/",TEXT($A$1,"0/4"))-1)
/ MID(TEXT($A$1,"0/4"),FIND("/",TEXT($A$1,"0/4"))+1,99)

As noted previously, it would be better to put the TEXT() expressions into helper cells and to reference those helper cells repeatedly in the respective formulas above.

If you are saying that you want a formula that works regardless of how B2, C2 and D2 are formatted -- that is, that "detects" their formats automagically, I suggested the UDF myText.  See my previous posting for details.  Use myText(B2), myText(C2) and myText(D2) respectively in place of TEXT(...) the formulas in B3:D3 above.

If you want to use the UDF, but you don't know how, "restate" that ;-), and I will be happy to offer instructions.

bob jones aka: cybertaz say:
Although you selected Office for Mac 2011 [Version 14.x] when you posted the message you're seeing indicates that you actually have Office 2004 [Version 11.x] or prior. Due to decisions made by Apple, those earlier versions of Mac Office cannot be used on Macs running OS X 10.7.x or later. There is no way around it.

Double-check your source media or the actual program icons in the Applications folder to confirm your version of Mac Office. If it is earlier than Office 2008 [Version 12.x] you'll need to upgrade to a newer version of the software that will run on the current operating system.

I'm afraid I have absolutely no clue what you mean by "The format Mountain Lion  saves documents in is ( .otd )", though. AFAIK [and a quick web search confirms] there is no such file format. There is an .odt (OpenDocument Text) format which is an open source file format used almost exclusively by OpenOffice, not Microsoft Office for Mac & isn't what Mountain Lion "saves documents in" unless you're using the OpenOffice software. The operating system doesn't have a 'default' format for saving files -- file types are determined by the programs which run on any particular operating system.
