andreas killer say:
A simple example:

Make a new file
Open the VBA editor
Insert a regular module
Paste in this code 

Option Explicit
Public LastShapeName As String
Sub SetShapeMarker()
  Dim Sh As Shape
  'Errors off, maybe there is no previous shape
  On Error Resume Next
  'Refer to that shape
  Set Sh = ActiveSheet.Shapes(LastShapeName)
  'Color the border black
  Sh.Line.ForeColor.RGB = RGB(0, 0, 0)
  'Errors on, we want know if something went wrong during the development
  On Error GoTo 0
  'Get the name of the called shape
  LastShapeName = Application.Caller
  'Refer to that shape
  Set Sh = ActiveSheet.Shapes(LastShapeName)
  'Color the border red
  Sh.Line.ForeColor.RGB = RGB(255, 0, 0)
End Sub


Close the VBA editor
Insert at min. 2 shapes (e.g. rectangle) into the sheet
Right click each and assign the macro above
Select a cell
Save the file as XLSM
Done

Now click on the shapes, as you see the "active" shapes becomes a red border.

techfreak_ say:

That suite/integration/softphone software appears to be tied to the retired Desktop API (2013). All 3rd party developers were informed of the discontinued development and deprecation of support for that API years ago. They were to relay this to their customers. I have heard from other users that Jabra's first levels of support may be unaware if you call them for support. Their higher tier developers definitely know. No newer (8.x+) clients support that API. For this reason your headset cannot "see" Skype. They would have to develop a non-Desktop API solution to attempt to offer what no longer works.

Excerpt from the Skype blog in Nov 2013:

"...call recording and compatibility with hardware devices – until we determine alternative options or retire the current solution"

The only current solution I am aware of is running a classic version of Skype (7.x client). You would have to revert to a supported 7.x version (7.31 - 7.40), if you still wanted to attempt to use 3rd party software to interface with the Skype client. Compatibility may still be an issue as the API was never designed for the cloud.
