john w. vinson say:

If you want the InvoiceFirstName to be editable, then the textbox must be bound to a table field, e.g. to the InvoiceFirstName field in your table. Right now it's just an expression, and won't be stored anywhere. I'd suggest "pushing" the first name into the field in the AfterUpdate event of the custfirstname textbox:

Private Sub custfirstname_AfterUpdate()
If IsNull(Me!InvoiceFirstName) Then ' don't step on existing value
  If Me!sameascust Then ' only push if the checkbox is checked
    Me!InvoiceFirstName = Me!custfirstname
  End If
End If
End Sub

You'll want to put the same code in the checkbox's afterupdate.

sky pei msft say:

Hi Cathy_MT,

To answer your questions:

Question1:

There is no such a method. We appreciate your understanding.

Question2:

By default, folders will inherit permission from library, while library will inherit permission from the site.  If you change the permission at the site level, permission for folders will change as well. However, if you don't want to change the permission for one specific folder, you can stop inheriting permission by performing the following steps:

tick the checkmark on the left of the folder ->open the details pane ->manage access ->advanced on the bottom ->then you can stop inheriting permission and grant permission you want.

Regards,

Sky
