mbizup say:

Ok - the best I can do here is to suggest guidelines/approach plan. A few comments:

1.  Very important - make sure any data that will be used in mathematical calculations is defined as numeric in the underlying table.  This data can be formatted as percentages if needed.  (You currently have some percentages defined as text)

2. Determine what the various user inputs are going to be, and include them on your data entry form.  You can use lookup tables and combo boxes for users to select data from standard options (such as cylinder size)

3.  Rather than trying to reverse engineer the formulas from your excel spreadsheet (very complicated), obtain the actual mathematical formulas -- which you can include as textbox control sources or as columns in a query.

4.  Break larger formulas down into sub-calculations as needed,

For a simple example, the Required PPO2 is defined as Fill Pressure * Required Pressure

Assuming that Fill pressure and required pressure are user inputs, you can add a textbox to your form for Required PPO2 and set its Control Source property to (using whatever field names you have defined in the database):

          = FillPressure * RequiredPressure 

Alternatively you can include the calculation in a query, which can be used as your form's recordsource:

SELECT *,  FillPressure * RequiredPressure  AS RequiredPPO2 FROM YourTable

and then set the control source of a textbox to the RequiredPPO2

This textbox/ sub calculation can be used in larger calculations - similar to your excel spreadsheet...

It would be a good idea to read up on database basics - normalization, querying/ calculated values/etc.
