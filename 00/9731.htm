mark a walters say:

I got here asking the same problem, but the answer for me was a bit more subtle.  Although the above answers may help some, I'm including the following info in case others have the same problem I had.

 

First off, I need to state that a blank cell in the same column does not necessarily 'break' the flow (i.e. cause the list of auto-complete options to 'start over'/refresh).  In 2003 at least, you seem to need a blank row to cause the list to refresh.  (NB! if you have only populated a single column, or the column is separated from others either side of it by blank columns, then I expect that a blank cell would would be equivalent to a blank row, and the following may not be valid for you).

Secondly, Excel will only provide an autocomplete prompt when it has enough characters to identify a unique cell value, e.g.:

aaaaaaaaa
aaaaaaaaa
bbbbbbbbb
bbbbbbbbb
aaaa#bbbb

aaaa                this cell does not contain enough info yet to know which previously existing value you may be wanting to repeat, but the moment you enter the 5th character (assuming it will be either an 'a' or '#' per the example) it will autocomplete the rest of the cell.

Because I was under the impression that a blank cell had caused the autocomplete list to 'refresh', I was expecting the autocomplete to kick in after the first character, however, I would have needed to enter about 16 characters to get a unique value, so it looked like it was not working! 

The fix:

  1. Check that this is your problem too, use <Alt> and <down arrow> - this will show you all your autocomplete values (not just those which would match what you may have already entered in that cell, and you can select one as per usual drop down list fashion if you want).  From this you can quickly see, i) if autocomplete is active, and ii) why what you have entered is not unique.
  2. Insert a blank row to cause the autocomplete list to restart/reset at the point that you want/expect it to (e.g. following on from the above example, inserting a row after the last 'aaaaaaaa' row, would mean that autocomplete would work immediately an 'a' was entered on the new row).  If inserting a row is not an option, then you may have to use the  <Alt> and <down arrow> as a workaround.

I hope this makes sense, didn't have much time to tweak it.

Regards

Mark

 

mark a walters say:

A few blogs I've seen indicate that blank cells are the delimiting factor in determining what cells are used to populate the autocomplete list, however, others talk about a 'region', or similar term.  When I was trying to resolve my problem (see earlier input), I eventually established that it is in fact determined by the 'current region' as classically defined for Excel.  Select the cell you want autocompleted, then use <Edit> => <Go To> => <Special> => <Current Region>, to 'highlight' (i.e. select)  the block of cells that make up the region; you can then see what autocomplete uses when it creates its set of values per column (remember also that by design there are some types of data that are excluded from the autocomplete function, e.g. dates, time, numbers only; refer .

http://office.microsoft.com/en-us/excel-help/create-change-or-delete-a-custom-fill-series-HP005199498.aspx).

The problem with such windows is that you can't close them from the application that causes that this window is opened. If you try it anyway, the application crashs.

But we can use a trick if we use a second application and a windows timer, because the VBE applications are not connected in that way.

Copy this code into a normal module in Excel, save the file, but don‘t close it!

--- schnipp ---
Option Explicit

Private Declare Function GetForegroundWindow Lib "user32" () As Long
Private Declare Function GetClassName Lib "user32" Alias "GetClassNameA" ( _
    ByVal hWnd As Long, ByVal lpClassName As String, ByVal nMaxCount As Long) As Long
Private Declare Function GetDlgItem Lib "user32" ( _
    ByVal hDlg As Long, ByVal nIDDlgItem As Long) As Long

Private Const EM_REPLACESEL = &HC2
Private Const EM_SETSEL = &HB1
Private Const BM_CLICK = &HF5&
Private Declare Function SendMessage Lib "user32" Alias "SendMessageA" ( _
    ByVal hWnd As Long, ByVal uMsg As Long, ByVal wParam As Long, lParam As Any) As Long

Function EnterPassword(ByVal Password As String) As Boolean
  Dim hWnd As Long, hEdit As Long, hButton As Long, i As Long
  Dim ClassName As String
  'The current window should be the password window
  hWnd = GetForegroundWindow
  'Check the class
  ClassName = Space(256)
  i = GetClassName(hWnd, ClassName, 255)
  ClassName = Left$(ClassName, i)
  'If the class doesn't match, return false
  If ClassName <> "#32770" Then Exit Function
  hEdit = GetDlgItem(hWnd, 6)
  hButton = GetDlgItem(hWnd, 1)
  'Something went wrong, return false
  If (hEdit = 0) Or (hButton = 0) Then Exit Function
  SendMessage hEdit, EM_SETSEL, 0, ByVal -1&
  SendMessage hEdit, EM_REPLACESEL, 0, ByVal Password
  SendMessage hButton, BM_CLICK, 0, ByVal 0&
  'Return successful
  EnterPassword = True
End Function

Andreas.


Option Explicit

Private Declare Function SetTimer Lib "user32" ( _
    ByVal hWnd As Long, ByVal nIDEvent As Long, ByVal uElapse As Long, _
    ByVal lpTimerFunc As Long) As Long
Private Declare Function KillTimer Lib "user32" ( _
    ByVal hWnd As Long, ByVal nIDEvent As Long) As Long

Dim Password As String
Dim AppExcel As Object

Private Function TimerProc(ByVal hWnd As Long, ByVal uMsg As Long, ByVal idEvent As Long, _
    ByVal dwTime As Long) As Long
  If AppExcel.Run("EnterPassword", Password) Then KillTimer hWnd, idEvent
End Function

Sub Main()
  Dim Path As String, FName As String
  Dim Pres As Presentation

  Path = "G:\temp\"
  Password = "qwe"

  On Error Resume Next
  Set AppExcel = GetObject(, "Excel.Application")
  AppExcel.Run "EnterPassword", Password
  If Err Then
    MsgBox "Open Excel and load the file with the EnterPassword macro."
    Exit Sub
  End If
  On Error GoTo 0

  FName = Dir(Path & "*.ppt")
  Do While FName <> ""
    'Don't open ourself!
    If FName <> ActivePresentation.Name Then
      'Start the timer to enter the password
      If SetTimer(0, 0, 250, AddressOf TimerProc) = 0 Then Exit Sub
      'Open the file, this opens the #32770 password window too
      Set Pres = Presentations.Open(Path & FName)
      Pres.WritePassword = ""
      Pres.Save
      Pres.Close
    End If
    FName = Dir
  Loop
  'Kill the last timer if any
  KillTimer 0, 0
End Sub

Sub OpenMultiplePPTsWithSamePwd()
    ' Declare a variable as a FileDialog object.
    Dim fd As FileDialog
   
    ' Create a FileDialog object as a File Open dialog box.
    Set fd = Application.FileDialog(msoFileDialogOpen)
   
    ' Declare a variable to contain the path of each selected item.
    Dim vntSelectedItem As Variant
   
    '/* To reference the FileDialog object .*/
    With fd
       
        '/* The user pressed the button .*/
        If .Show = -1 Then
       
            ' Declare a variable as a Presentation object.
            Dim myPPT As Presentation
           
            Dim ws As Object
           
            Set ws = CreateObject("WScript.Shell")
           
            On Error Resume Next
           
            '/* Step through each string in the FileDialogSelectedItems collection. */
            For Each vntSelectedItem In .SelectedItems
                
                ws.Exec "%SystemRoot%\System32\WScript.exe D:\Sleep.vbs %1 %*"
                ws.SendKeys "123"  'Here puts your Password.
                ws.SendKeys "{ENTER}"
               
                ' To reference the opening presentation object.
                Call Presentations.Open(vntSelectedItem)
               
                ' To reference the opened presentation object.
                Set myPPT = GetObject(vntSelectedItem)
                               
                myPPT.WritePassword = ""
                myPPT.Save
                myPPT.Close
               
            Next
           
            Set ws = Nothing
        End If
       
    End With
   
    'Set the object variable to nothing.
    Set fd = Nothing
End Sub
