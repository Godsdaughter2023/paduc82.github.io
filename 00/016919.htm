Solution for SFC /Scannow not working in Build 14279

I have just found the reason why SFC /Scannow fails with a "Windows Resource Protection could not start the repair service" message.

The directory C:\Windows\WinSxS\amd64_microsoft-windows-servicingstack_31bf3856ad364e35_10.0.14279.1000_none_25a158fc7f85c69d is missing the wrpint.dll file. That file was not missing in the 14271 build. 

Microsoft Windows team. Can you please issue an update that replaces the missing DLL file so the insider community can use SFC and DISM again to resolve issues again.

To manually resolve the problem whilst logged in with an administrators account, click on the properties of the directory I mentioned above, Take ownership of the directory. when you apply, you will see some errors about not being able to enumerate the various files contained in the folder. Just click through and ignore the errors. 

After that has finished, copy the wrpint.dll file from the directory C:\Windows\WinSxS\amd64_microsoft-windows-servicingstack-onecore_31bf3856ad364e35_10.0.14279.1000_none_5a92ee0dd788e433 into the directory you just took ownership of. 

After the completion of the copy, Change the ownership of the directory back to the user NT SERVICE\TRUSTEDINSTALLER. SFC will then work to scan your system

WORD VBA: Change linkedstyle in a listtemplate, how to identify - Microsoft Community

astridz say:

WORD VBA: Change linkedstyle in a listtemplate, how to identify the correct listtemplate?

I have created a template with two styles for lists. One with and one without numbers. In the list style with the numbers, the listlevels are attached to heading styles.

I need to add functionality to have the users switch between headings with - and headings without numbers. My plan was to change the linkedstyle property for the chosen listtemplate.listlevel to heading1, heading2 etc.
This seems to do exactly what I want if I do it manually.

However, I can't seem to find a way to identify the correct listtemplate. It seems that every listtemplate.name gives me a blank string. Is there a way to get the correct listtemplate by using the name or in another way?

Or otherwise, is there a better way to archieve the same result?

astridz say:

Never mind, figured it out. I was a bit confused because most listtemplates have no name. Mine however, do.

I ended up with:

Sub ToggleHeadingNumbers(sListTemplateName As String)
Dim oListTemplate As ListTemplate
Dim oListLevel As ListLevel
 
  For Each oListTemplate In ActiveDocument.ListTemplates
 
    If StrComp(oListTemplate.Name, sListTemplateName, vbTextCompare) = 0 Then
   
      For Each oListLevel In oListTemplate.ListLevels
       
        If oListLevel.Index = 1 Then
          oListLevel.LinkedStyle = ActiveDocument.Styles(wdStyleHeading1)
        ElseIf oListLevel.Index = 2 Then
          oListLevel.LinkedStyle = ActiveDocument.Styles(wdStyleHeading2)
        ElseIf oListLevel.Index = 3 Then
          oListLevel.LinkedStyle = ActiveDocument.Styles(wdStyleHeading3)
        ElseIf oListLevel.Index = 4 Then
          oListLevel.LinkedStyle = ActiveDocument.Styles(wdStyleHeading4)
        Else
          Exit For
        End If
       
      Next
      Exit For
     
    End If
   
  Next

  Set oListTemplate = Nothing
 
End Sub
