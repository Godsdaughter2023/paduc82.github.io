=TRIM(MID(A2,SEARCH(",",A2)+2,255))&" "&LEFT(A2,SEARCH(",",A2)-1)

 

assuming the data is in A2. Your second example could be split using this formula:

 

=MID(A4,SEARCH(",",A4)+2,SEARCH(" ",SUBSTITUTE(A4,", ","|"))-SEARCH(" ",A4))&" "&LEFT(A4,SEARCH(",",A4)-1)&" "&MID(A4,SEARCH("&",A4),255)

 

assuming the data is in A4.

=IF(LEN(A2)-LEN(SUBSTITUTE(A2," ",""))=3,TRIM(MID(A2,SEARCH(",",A2)+2,255))&" "&LEFT(A2,SEARCH(",",A2)-1),MID(A2,SEARCH(",",A2)+2,SEARCH(" ",SUBSTITUTE(A2,", ","|"))-SEARCH(" ",A2))&" "&LEFT(A2,SEARCH(",",A2)-1)&" "&MID(A2,SEARCH("&",A2),255))

 

and then copied down as far as you need it.

No, SUMPRODUCT is a mathematical function, not a text function; but, since SUMPRODUCT is an array function underneath it all, your question doesn't make much sense to me. For your hoped for example function, namely...

=SUMPRODUCT(($A$2:$A$7="Admin")*($B$2:$B$7="A")*C2:C7)

the function works by evaluting each range one row at a time looking for those rows (notice that rows is plural) where the cell in A2:A7 equals "Admin" and the cell in the same row in B2:B7 "A" at the same time, but this place no restrictions on the values in C2:C7 because this third "term" is not a logical expression. So, if you had  more than one row where the cells in A2:A7 equalled "Admin"at the same time the cells in B2:B7 equalled "A" for those same rows, but in those rows the cells in C2:C7 were  different... then which of the cells in C2:C7 what would you expect the formula to return??? Visio has two ways of connecting, to the shape or to the connection point. When connected to a shape it uses the closest connection point to the other shape whenever the shapes are repositioned, but uses the same connection point if the connection is to a connection point.

When you are about to connect to the shape a red box shows up around the shape. When you attempt to connect to a connection point a red box appears around the connection point.

jay freedman say:

From the description, what you accidentally hit was probably the shortcut Ctrl+Shift+E, which turns Track Changes on or off. The red strikethrough is the default formatting for displaying deleted text. This is explained at http://shaunakelly.com/word/sharing/howtrackchangesworks.html.

To fix the document, first make a copy of the document for safekeeping. Next, make sure that Track Changes is turned off, by going to the Review tab of the ribbon and, if the Track Changes button is highlighted in orange, click it to remove the highlight. Now, on the same ribbon tab, click the down arrow next to the Reject button in the Changes group and click "Reject All Changes in Document". That will remove the strikethrough marking (as well as anything you typed since Track Changes was first turned on, which you can recover from the copy you made at the start).

The reason you couldn't find the Format menu or the other items mentioned in the post marked as Answer is that those things are only in Word 2003 and earlier, before the ribbon was introduced. That's one of the dangers of reading very old posts, and it's one reason why I split your question off to a separate thread. Another reason is that the original post was about a different kind of problem (although with a similar appearance).
