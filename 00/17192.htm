ossiemac say:

The following is the link to the example workbook. I suggest that you save the extracted file to the same folder as your main workbook to which you will copy the VBA code.

Example workbook with popup Userform

Ensure that you have macros enabled. See Help for how to do this. Setting option to "Disable all with notification" should be ok.

I don't know of any way to identify when a date changes so I have written the code so that it pops up a Userform with the message when the workbook is opened.

The code searches for all cells containing "Next review" and tests the Conditional Format color in the cell to the right.

The message displays all of the colors (except white) used in the Conditional format and lists the names from above the date with the matching Conditional Format. If more than one date matches then the names are separated with a comma. There is no message for white because I don't think that you need that.

I have changed some of the dates in the example workbook for testing purposes so they no longer have the formulas. You can still have the formulas in your main workbook to which you are going to copy the code.

You can close the Userform message by clicking the cross at the top right and if you want to see the message again while the workbook is open then Double Click on one of the colors in cells B1:B5.

Important: The RGB color code in the legend of colors in range A1:B6 must match the RGB color codes in the Conditional Format because the code uses this legend to compare the colors in the Conditional Formats. (In the workbook that you uploaded the RGB color code for cell B1 did not match the RGB color code that was used in the Conditional Format so in your main workbook you should check this.) 

Guidelines to copy the code to your main workbook.

  1. Open the example workbook.
  2. Close the Userform popup.
  3. Alt and F11 to open the VBA editor.
  4. Click once on Userform1 in the Project Explorer (left column). If you can't see Userform1 in the left column then you might need to click the plus sign adjacent to Forms to expand it. Userform should have blue background when selected.
  5. Click on menu item File - > Export file.
  6. In the dialog box navigate to where you want to save the exported file. I suggest the same folder where you have the workbook.
  7. Keep the default names for File name and Save as.
  8. Click Save to save the exported Userform.
  9. Alt and F11 to change back to the Worksheet.

10. Keep the example workbook open and open your main workbook to which you are going to import the Userform.

11. Alt and F11 to open the VBA editor.

12. You will see both workbooks in the Project explorer in the left column. Click and highlight the name of your main workbook.

13. Select File -> Import File.

14. Navigate to where you saved the exported Userform and you should see it in the Name column. Select the file and Open.

15. In the Project explorer double click the worksheet name Plan 1 (Plan 1) and it should display event code on the right side for that worksheet.

16. Copy all of the code.

17. In the Project explorer, Double click the equivalent worksheet name in you main workbook and paste the code into the VBA editor for the main workbook.

18. In the Project explorer under the Example workbook, Double click the ThisWorkbook.

19. Copy the VBA code.

20. In the Project Explorer under the main workbook Double click ThisWorkbook and paste the code into the VBA editor.

21. Close the Example workbook.

22. Important: Save the main workbook as Macro Enabled workbook (If you don't do this then you will lose all of the work you have just done to copy the code.)

23. Important: If the worksheet name (Plan1) or cells containing Next review are not the same in your main workbook because you have converted the example to English then you will need to edit 2 lines of code in the main workbook.

24. To edit the code.

25. Open the VBA editor (Alt and F11)

26. Double click Userform1 in Project Explorer and it will display the Userform in Design mode.

27. Right click Userform1 in the Project explorer and select View code.

28. Towards the top of the code you will see the following 2 lines:

                 Set ws = Worksheets("Plan1")
                 strToFind = "Next review"  'This is description to left of Conditional Formatted cells

      29. Edit Plan1 in the first line to match your worksheet name in your main workbook and edit Next review to match the equivalent cells in your main workbook.

      30. Close the VBA editor (Click the cross with the red background top right of editor.)

      31. Save the workbook

I hope that it does what you want and if you have any problems then please let me know.

  After this restart the windows update service and try again!!

Can someone help me lock cells that have a VBA code? - Microsoft Community

khalid abou kassem say:

Can someone help me lock cells that have a VBA code?

Hello all,

I am facing a problem while protecting some worksheet which have VBA codes encrypted in them.

I am locking all the cells in that worksheet but unlocking the ranges and cells that have the VBA codes encrypted in them, since these VBAs HIDE some rows when a specific criteria is selected or inserted. 

But when I run the protected sheet and try it, it gives me:

Run-Time error '1004':

Unable to set the Hidden property of the Range class

Could anyone help with this?

zorvek say:

You need to either unprotect the worksheet before make the changes, or set the worksheet's UserInterfaceIOnly property to True.

When a sheet is protected Excel not only prevents the user from changing locked worksheet elements, it also prevents VBA from changing them. This is circumvented by unprotecting the sheet, making the changes, and then protecting the worksheet again as illustrated below.

   Sheets("Sheet1").Unprotect Password:="password"
   ...
   Sheets("Sheet1").Protect Password:="password"

An alternative and more efficient technique is to use the UserInterfaceOnly option on the Protect method. This option protects the sheet from manual changes the same as without the option, but it allows VBA access to the sheet's elements as if the sheet was unprotected. The only drawback is that the option setting cannot be set from the Excel application user interface and it is not saved with the workbook. This means that it has to be reset every time the workbook opens. This is accomplished by placing the code to set the option in the Workbook_Open or similar routine.

Private Sub Workbook_Open()

   Sheets("Sheet1").Protect Password:="password", UserInterfaceOnly:=True
   Sheets("Sheet2").Protect Password:="password", UserInterfaceOnly:=True

End Sub

Note that protection does not need to be turned off to make the change. The above code can be placed anywhere in the project as long as the UserInterfaceOnly setting is adjusted before the first attempt is made to change a sheet element.

The UserInterfaceOnly option is available starting with Excel 2000.

Note that even with UserInterfaceOnly set to True, certain operations cannot be performed such as

- adding drawing objects and charts to the worksheet,
- turning AutoFilter on and off, and
- adding data validation to cells.

When these operations must be perfomed on a worksheet, the worksheet must be unprotected and protected again. Any operation attempted on a worksheet protected with the UserInterfaceOnly option that is not allowed will generate a 1004 error.
