Pivot tables lock Access database in 'ReadOnly' mode - Microsoft Community

itllfly say:

Pivot tables lock Access database in 'ReadOnly' mode

For years we have used processes that have Access queries extract data from SQL tables and populate data in Access tables.  We then have pivot tables that pull information from those tables as a means of 'feeding' information to those without ODBC rights to the SQL tables.  On converting to Office 2007 any pivot table user that refreshes their data will turn the target table's database into a 'ReadOnly' file thus locking the database from further updates.  This 'ReadOnly' state is only terminated when the offending Excel pivot table file is closed.  Access is set to a Default Open Mode of 'Shared' and Default Record Locking is 'No Locks'. 

This behavior does not occur when using Excel 2003 pivot tables to query these same database tables.

markntex say:

The Access table is locked when the PivotTable is created through PivotTable > Use an external data source > Choose Connection > Browse for More. This connection uses Provider=Microsoft.ACE.OLEDB.12.0 and sets the Mode = Share Deny Write. The table locking behavior can be eliminated by editing and changing the connection to Mode = Read.

 

Creating a PivotTable with Data > From Other Sources > From Microsoft Query does not lock the access table. But it is slower to refresh and takes more clicks to create the connection than browsing to a connection does. I created and refreshed two PivotTables with both methods using the same Access query. The Microsoft Query was 3 seconds slower on a 43 second refresh after three consecutive refreshes on each PivotTable.

 

Credit to Debra Dalgleish for the solution. I searched on “why does an excel pivot table lock an access database as read only” and read through a dozen answers before finding this solution on Debra’s blog.

bob jones aka: cybertaz say:
The short answer to your specific question is:

  1. Switch to Draft View
  2. Select View> Footnotes
  3. Open the list in the bar that appears in the lower part of the document window
  4. Select Footnote Separator
  5. Click Reset

The extension to that: Although there are certain modifications that can be made, length is not really one of them. Word (PC/Mac) employs a short separator (approx. 2") for footnotes & a longer separator for Footnote Continuations which extends from L to R margin. If you're seeing these 2 lengths, that's why. There are "workarounds" for make things look as though their length has been changed, but which can get also get you into a lot of trouble :-)

OTOH, if the lines are of various lengths that deviate from that, it is not due to the standard separator lines Word uses. Someone has either taken the liberty suggested above & replaced the standard separators in some way

Have a look at this article by MVP Suzanne Barnhill on this subject:

http://wordfaqs.mvps.org/NoteSeparators.htm


Don't worry about the fact that the article is PC Word based. The mechanism is exactly the same in the Mac version. Note that there is a special topic near the end for "Troubleshooting Note Separators".

Regards,

Bob J.
