ken sheridan say: A number of points here:

1.  The AfterUpdate event procedure of the first combo box should be used to requery the second combo  box, not it's Change event procedure.

2.  What do you mean by OnChange=True; combobox2.requery=true?   In the case of the first (though it should be the AfterUpdate event procedure as I said above) the event procedure should contain code to requery the second combo box, or the event property should call a macro to do so.  Requery is a method of the control so does not have a value of True or False, but is called by the code in the first box's event procedure with Me.[YourSecondComboBoxName].Requery, or by a macro as the event proiperty which does the same.  Alternatively the RecordSource property of the second combo box can be built in code in the AfterUpdate event procedure of the first combo box.

3.  The RecordSource property of the second combo box must be query, either the name of a saved querydef object or an SQL statement, if it is requeried on a selection being made in the first combo box.  If it is a value list then the string expression would need to be rebuilt in the AfterUpdate event procedure of the first combo box; this, while theoretically possible, would be an unusual approach and a bad one as it would involve encoding data in the object definition.

4.  The Records form must be open when the query opens in order to expose the control as a parameter, and the parameter must use the exact names of the form and the control.  Is the name of the form Records in the navigation pane, and is the Name property of the control Payee in the control's properties sheet?  Assuming both to be the case the parameter should be Forms!Records!Payee.

Hello milkZI,

 

Thank you for posting on Microsoft Community.

 

I understand that you can’t download apps from the Store and that you have an error message: set up My Family.

 

Please try the following steps:

 

  • Go on your phone to Settings > Email + accounts, and check the name of the first account you see in the list after “add an account”

 

  • Log into this account on a computer

 

https://login.live.com

 

  • Check your birthdate: I know you told us that it was correct, but did you check on the right Microsoft account ?

 

  • Fill all the personal information if you didn’t already, and check in Security & Password that the two steps verification is disabled (if not, please disable it).

If you find that your birth date was incorrect on the Microsoft account linked to the phone:

 

 

  • Try to change your birthdate, and to sync manually your account on your phone by keeping the finger on the first account listed in Settings > Email + Account. This is the easiest solution, but it is normally not working, as Microsoft set up a security to avoid children who change their birthdate by themselves.

 

  • If the first solution doesn’t work: save all the data on your phone:

 

http://www.windowsphone.com/en-gb/how-to/wp8/settings-and-personalization/back-up-my-stuff

 

Then, hard reset your phone:

 

http://www.windowsphone.com/en-GB/how-to/wp8/settings-and-personalization/reset-my-phone

 

And create another Microsoft account with all your personnel information, and the right birthdate from the beginning.

 

After this, you will be able to add your old account as a secondary account on your phone by going in Settings > Email + accounts > add an account. This way you will be able to recover all your datas.

 

  • If really you don’t want to do a hard reset: create another Microsoft account:

 

https://login.live.com/

 

Then log in with the new adult account here:

 

https://www.windowsphone.com/en-gb/family?signin=true

 

And click on Get started, then on Go. There you should be able to enter the credentials of your phone’s primary Microsoft account to add it as a child.

After that you should be able to set up the download and purchase authorizations for the child account.

 

 

If you did all these steps, and that your birth date was correct since the beginning on the right account:

 

 

 

  • Try a soft reset, and a hard reset and see if it solves the problem.
