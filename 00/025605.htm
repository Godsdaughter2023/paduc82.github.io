    Hi Muneeb,

    Try the following. It is based on Chip Pearson's code but has been modified so you can simply use an expiry date instead of using a specific number of days after the workbook is first opened.

    Make a backup of your workbook without the added VBA code before starting this process and if you do something wrong then you can get back to the original workbook.

    1. Alt and F11 to open the VBA editor.
    2. In the VBA editor select menu item Insert -> Module
    3. Copy the code below and paste into the editor.
    4. Edit the date constant at top of code to the required expiry date. (Note that this date is enclosed in # symbols and it must be in m/d/y format irrespective of you local date format.)
    5. Edit the password to your preferred password.
    6. Close the VBA editor (Cross with red background top right)
    7. Save the workbook as Macro Enabled.
    8. Ensure that Macros are enabled in Options. See help for how to do this. (Setting option to "Disable all with notification" should be OK.)
    9. You can re-open the VBA editor with Alt and F11.
    10. You can test by setting an Expiry Date in the VBA code to prior to today's date and then save and close the workbook and re-open. You will get the Message Box and will see the worksheets are protected if you answer OK to the message or the workbook closes if you Cancel.
    11. Finally set the Expiry date to the required expiry date and then protect the code as follows.
    12. In the Project Explorer (left column) Right click VBA Project.....
    13. Select VBA Project properties.
    14. Select Protection tab.
    15. Check the box against "Lock project for viewing"
    16. Enter your preferred password (It can be same or different to password used in step 5)
    17. Confirm the password.
    18. Click OK

    The VBA code is not hidden until after you save and close the workbook and then re-open it.

    If the Expiry date has not passed then all worksheets are unprotected when the workbook is opened.

    If the Expiry date has passed then the user gets a MsgBox with the option to either close or open in Read Only and Print mode or close the workbook.

    To get back to the VBA code after it has been protected open the VBA editor, double click the project in the project Explorer column and enter the password when prompted.

    If you save and close the workbook again then the VBA project is automatically protected again without going through the VBA protection process again.

    To remove the password from the VBA project, after double clicking the project in the Project Explorer and entering the password, start from step 12 above for setting the password and uncheck the box against "Lock project for viewing" and clear both the password and confirm password fields.

    Copy all of the following code into the VBA editor.

    Const ExpirationDate = #5/25/2015#      'Enter expiration date in m/d/yyyy format.
    Const strpassword = "ossiemac"          'Edit "ossiemac" to desired password


    Sub Auto_Open()
        Call ExpiredProcedure
    End Sub


    Sub ExpiredProcedure()
        'Adapted from chip Pearson's TimeBomb solution at following link
        'http://www.cpearson.com/excel/workbooktimebomb.aspx
        Dim msgResponse As Variant
        Dim ws As Worksheet
       
        'If today is past the expiration date, display the MsgBox
        If Date > ExpirationDate Then
            msgResponse = MsgBox("This workbook has expired." & vbCrLf & _
                                "Click OK to open in Read Only and Print mode" & vbCrLf & _
                                "or Cancel to close the workbook.", vbOKCancel)
                               
            If msgResponse = vbCancel Then
                ThisWorkbook.Close SaveChanges:=False
            Else
                For Each ws In ThisWorkbook.Worksheets
                    Call Protect(ws)        'Protects the worksheets
                Next ws
            End If
        Else
            For Each ws In ThisWorkbook.Worksheets
                Call Unprotect(ws)         'Unprotects the worksheets
            Next ws
        End If
       
    End Sub

       
    Sub Protect(wSht As Worksheet)
        wSht.Protect Password:=strpassword
    End Sub

       
    Sub Unprotect(wSht As Worksheet)
        wSht.Unprotect Password:=strpassword
    End Sub

    Hi AlvinTan1,

    According to your description, the signature picture in the email you received form the Mac user cannot be displayed. To confirm whether this issue is related to sender, we suggest you contact the sender to check if the signature picture can be displayed well in this sender’s Sent Items folder after sending.

    If the signature picture can be displayed well in sender’s Sent Items folder, it might be related to the recipients’ Outlook setting about pictures’ downloading automatically. Based on the screenshot you provided, you may use the Windows Outlook client. You can unblock picture downloads for all messages by following steps:

    1. Go to File > Options > Trust Center.
    2. Click Trust Center Settings under Microsoft outlook Trust Center.
    3. Uncheck the Don't download pictures automatically in HTML e-mail messages or RSS items check box.

    If you use Outlook 2011 for Mac, you can download images automatically by following steps below:

    1. Click Outlook > Preferences….
    2. Select Reading under E-mail.
    3. Select In all messages of Automatically download pictures from the Internet under Security.

    Meanwhile, if you use Outlook 2011 for Mac, as Outlook 2011 for Mac is no longer supported, it’s recommended for users to upgrade the Outlook client to Outlook 2016 for Mac for getting a better experience and more features (you can see how to upgrade to Office 2016 by referring to this article).

    If the issue persists, to further look into this issue, we need to confirm if you use the Outlook for Mac or for Windows. If you use the Windows Outlook client, you can try to fix the issue about picture cannot be displayed and are shown as red X in Outlook by referring to this article.

    It should be noted that if you use Office 2013, the location of its registry subkey in step 2 is “HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Outlook\Security” and if you use Office 2016, the location of its registry subkey in step 2 is “HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Outlook\Security”
