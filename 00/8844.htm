a. user say:

Hi Abhishek,

Thank you for posting the query on Microsoft Community. I am sorry to know that you are facing issues with Windows 10.

If you're having trouble playing a song or video, it might be because the file is protected with digital rights management (DRM). DRM is a technology that content providers sometimes use to make sure that the digital music and video files you get from them are used and distributed according to their media usage rights.

How does the Player use media usage rights?

When you try to play a protected file in the Player, the Player checks to see if you have valid media usage rights on your PC. If you do, the Player plays the file.

If the Player doesn't find valid media usage rights on your PC, or if the rights limit what you can do with the file, the Player can't play the file. In this case, you'll see an error message that tells you why the file couldn't be played.

How can I tell if a file is rights-protected?

  •  Swipe in from the right edge of the screen, and then tap Search. 

(If you're using a mouse, point to the lower-right corner of the screen, move the mouse pointer up, and then click Search.)

  • Enter Windows Media Player in the search box, and then tap or click Windows Media Player.

  • Swipe across or right-click a media file, tap or click Properties, and then tap or click the Media Usage Rights tab. If the file is rights-protected, the license terms appear in the Media usage rights box, and you'll see what usage rights you have. If nothing appears in the Media usage rights box, it might be because the file isn't protected, the usage rights aren't found on your PC, or you're playing a file that's being streamed to your PC (for example, you're playing subscription content directly from an online store's website).

I also suggest you to try the steps provided below and check.

  • Select Options under the "SYNC" button.

  • Choose "More Options"

  • Click the "Network Tab"

  • Only check box RTSP/TCP and HTTP and Multicast Streams Allow the player to receive multicast streams (which means I have to uncheck "RSTP/UDP")

  • Click Apply.

Hope this helps in resolving the issue. If the issue persists, do get back to us. We will be happy to assist you.

Since MS seems retarded on solving this issue completely, here comes the unofficial solution to the "right-click -> new" problem as described above.

  1. Press "Win + R", enter "regedit" and press Enter.
  2. In left panel, find "HKEY_CLASSES_ROOT -> .xlsx -> Excel.Sheet.12 -> ShellNew" (the path may vary, just find ShellNew in .xlsx).
  3. On right panel, you see a key named "FileName" and its data is path to a file. For Excel 2016, it is "C:\Program Files (x86)\Microsoft Office\Root\VFS\Windows\ShellNew\excel12.xlsx".
  4. Now open the above path in your File Explorer and locate that file. This file is the template used to create new file from right-click.
  5. Now open Excel application and create a new blank file and save it as EXCEL12.XLSX. Test if its external link works fine. If so, substitute the above file with this file. Done.

rohn007 say:

Your hardware sounds entirely capable to handle Pivot Tables.

One user reported that upgrading from 8 to 16GB of RAM removed hesitations from Office applications on his system with roughly similar specs.  I don't know if that will apply to your situtaion.

Go in to Task Manager and look at the GPU column. I don't expect it will be a limiting factor, but it is worth looking at.

How big are these Pivot tables?  File Size, RAM used when Pivot is eventually opened.

Is it 2 pivots in a single workbook or 2 workbooks?

Is it 2 pivots based on the same base data, or separate data sets?

If it is 2 pivots in same workbook, based on same data set, there is a way to share data.  Pivot tables create a hidden cache of data which is copied from the base data table.  If you create the second Pivot "properly", is shares the data cache. If you create it from scratch, you duplicate the data cache, increasing the stress on your computer.

In this article:

23 things you should know about Excel pivot tables

https://exceljet.net/things-to-know-about-excel-pivot-tables

item #17 Clone your pivot tables when you need another view

describes how to prevent creation of duplicate data cache
