Difference between Windows7 32-bit & 64 bit - Microsoft Community

cf2 say:

Difference between Windows7 32-bit & 64 bit

Hi, can any tell what the differance is between Windows7 32-bit and Windows 7 64 bit. Please. Thanks
ken blake say:

On Mon, 2 Aug 2010 16:49:51 +0000, cf2 wrote:

Hi, can any tell what the differance is between Windows7 32-bit and Windows 7 64 bit. Please. Thanks

Windows 7 or any other version of Windows, read the following:

The advantage of running a 64-bit version of Windows mostly exists
only if you also run 64-bit applications under it. Bear in mind that
there are very few such applications available yet. If you are
presently running 32-bit Windows, you don't have any 64-bit
applications, so to achieve any significant advantage, you not only
have to replace Windows, but also your applications, if (and that's
a big "if") 64-bit versions exist.
Also note that you will need 64-bit drivers for all your hardware. Those drivers may not all be available, especially if some of your hardware is a few years old. So it's possible that you might also have
to replace things like your printer, scanner, etc.
So the answer to your question is that it may not be a great idea
right now. That will undoubtedly change in the near future, as 64-bit
applications become more available, but for now, 64-bit Windows often
means some extra trouble and expense for little or no benefit.
On the other hand, installing 64-bit Windows instead of 32-bit Windows
makes you able to buy 64-bit software as it becomes available, instead
of the older 32-bit versions. That means that installing 64-bit
Windows--even though it may do very little for you at present--puts
you into a better position for the future.

One additional point: the 64-bit version lets you use more than the
approximately 3.1GB of RAM that the 32-bit version can use. Very few
people need or can make effective use of more than 3.1GB, but if you
are one of those who can, that's something else to consider.

Ken Blake

Phil,

I thought that dialog had a Replace all option. 

It is AppData and/or Application data generating all the Replace dialogs because default versions for many settings files will have been generated when you logged into the new user account. 

When you try again, I suggest you copy the potentially awkward folders first [AppData & Application data] and then copy the easy folders [Desktop, Downloads, Documents, Pictures, Music, Videos] only when you know it's worthwhile [i.e. only when you know the rest has copied okay].

If you want to do the job without the prompts, open an Admin command prompt [or Admin PowerShell window but I have not tested this specific command in one yet] and use RoboCopy for these two folders because this will avoid all prompts -

--{substitute the correct names for OldAccount & NewAccount}--

--{go to both C:\Users\OldAccount and C:\Users\NewAccount by browsing in File explorer first in the Admin account you will use just to make sure it has permission in the folders concerned--

ROBOCOPY "C:\Users\OldAccount\AppData" "C:\Users\NewAccount\AppData" /E  /MIR /DCOPY:T /MT /R:10 /W:5

then do

ROBOCOPY "C:\Users\OldAccount\Application data" "C:\Users\NewAccount\Application data" /E  /MIR /DCOPY:T /MT /R:10 /W:5

If you want to check the switches I have used, you can enter RoboCopy /? in a command prompt window.  As you will see -

/E                  Copy all subfolders including empty ones

/MIR             Make a mirror image copy i.e. copy everything in the source folder and overwrite / delete everything in the target folder

/DCOPY:T     Copy folder date-time properties {not essential for these two cases but possibly useful if you ever do any troubleshooting in the future}

/MT              Allow the processor to use multi-threading [this speeds it all up significantly]

/R:10            Reduce the number of retries if problems occur to 10 [instead of the default 1,000,000]

/W:5             Reduce the wait between retries if problems occur to 5 seconds [instead of the default 30 seconds]

You can also use RoboCopy to copy over the easy folders as well.  You will not need an Admin prompt for them but you could usefully add the extra switch

/XA:H           Skip hidden folders & files [Windows drops hidden desktop.ini files in various places but there is probably no need to complicate the copying operation by including them as they are so unpredictable.  I have never actually found an explanation of them - many people say that they record non-standard folder layouts in File explorer but they are not generated in all such cases so I do not know what purpose they really serve]
