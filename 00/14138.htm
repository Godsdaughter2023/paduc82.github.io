With Application.FileSearch
    .NewSearch
    .LookIn = "B:\Attachments\"
    .SearchSubFolders = False
    .FileType = msoFileTypeWordDocuments
    If Not .Execute( ) = 0 Then
        For i = 1 To .FoundFiles.Count

             Set rng = doc.Range

                .ClearFormatting

                .MatchSoundsLike = False

                .Execute Replace:=wdReplaceAll

FileSearch is not available in Word 2007 vba, however the following will work:

Sub BatchProcess()
Dim strFileName As String
Dim strPath As String
Dim oDoc As Document
Dim fDialog As FileDialog
Dim oStory As Range
Dim oRng As Range
Const strFind As String = "Andy"
Const strRepl As String = "Matt"
    Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)
    With fDialog
        .Title = "Select folder and click OK"
        .AllowMultiSelect = False
        .InitialView = msoFileDialogViewList
        If .Show <> -1 Then
            MsgBox "Cancelled By User", , _
                   "List Folder Contents"
            Exit Sub
        End If
        strPath = fDialog.SelectedItems.Item(1) & "\"
    End With
    strFileName = Dir$(strPath & "*.doc")
    While Len(strFileName) <> 0
        WordBasic.DisableAutoMacros 1
        Set oDoc = Documents.Open(strPath & strFileName)
        For Each oStory In ActiveDocument.StoryRanges
            Set oRng = oStory
            With oRng.Find
                Do While .Execute(FindText:=strFind)
                    oRng.Text = strRepl
                    oRng.Collapse wdCollapseEnd
                Loop
            End With
            If oStory.StoryType <> wdMainTextStory Then
                While Not (oStory.NextStoryRange Is Nothing)
                    Set oStory = oStory.NextStoryRange
                    Set oRng = oStory
                    With oRng.Find
                        Do While .Execute(FindText:=strFind)
                            oRng.Text = strRepl
                            oRng.Collapse wdCollapseEnd
                        Loop
                    End With
                Wend
            End If
        Next oStory
        oDoc.SaveAs Filename:=strPath & strFileName
        oDoc.Close SaveChanges:=wdDoNotSaveChanges
        strFileName = Dir$()
        WordBasic.DisableAutoMacros 0
    Wend
    Set oDoc = Nothing
    Set oStory = Nothing
    Set oRng = Nothing
End Sub

ossiemac say:

There has been a number of reports of the bloating in size of workbooks and I believe that it is due to corruption. I am not aware of any definitive answer to date but some of the following has worked in some cases.

 

Is the workbook shared? If so then see the following link.

 

http://support.microsoft.com/kb/930253

 

If not shared, do you have any workbook or worksheet events? If so, display the VBA immediate window. (to display Immediate window, in the VBA editor Ctrl/G.)

 

Enter Application.EnableEvents = False in the Immediate window and press Enter. This needs to be done before the next steps because even though the event might be controlled by whether the target is within the range etc, the event still gets called even though it might exit the sub early. If no worksheet or workbook events then go to next step.

 

Set worksheet calculation to Manual. (Formulas ribbon -> Calculation options in calculation block) This is to stop calculation slowing the next process.

 

Select all blank columns to the right of your data and Clear All. (Use the Clear button dropdown in the Editing block at far right of Home ribbon and select Clear all. Do not use the right click and clear because that only clears contents and formats etc can be left.)

 

Select all blank rows below your data and Clear All.

 

Repeat for each worksheet.

 

Turn on Automatic calculation.

 

Enter Application.EnableEvents = True

 

Save the workbook and check size and test dragging formulas etc.

 

If still a problem then while I know it is a pain, copy the data from each worksheet to a new workbook. Do not use worksheet copy method; just select the data only (not the entire worksheet) and copy and paste into the new workbook. Need to also copy the VBA code. You may need to check links to ensure that they are not pointing back to the original workbook. If you have any controls then re-create them in the new workbook.

 

I have personally fixed 2 workbooks; one with the method of clear all in the blank columns and rows and another by copying the data to a new workbook.

 

For future reference, I will really appreciate your response as to whether any of the suggestions worked.

Your table is clearly corrupted - either the Primary Key index is damaged, or you're getting the well known bug where autonumber values repeat.
