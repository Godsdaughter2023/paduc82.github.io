Here is a tip and macro I've collected that resets the flag:

MACRO to disable “Auto Update” ALL Styles

NOTE:     The Automatically update document styles under Tools/Templates & Add-ins means that no matter what changes you make to style definitions (NOT direct formatting changes) will NOT stick.  Every time you close & reopen the document (& possibly when you save, I can’t recall), the styles from the attached template are copied to the document & all your style changes are lost.

The automatically update under the Format/Style/Modify means that when you add any direct formatting to a charcter or paragraph, the style definition is changed to reflect your addition.  Therefore, all styles with that name in the document change.

By default in NORMAL.DOT MS creates some styles with “AUTO UPDATE STYLE” option turned on.  So whenever a change is made to format of text using that style, all uses of that style are changed also.  This macro will scan thru all styles and turn of the option.  Most useful when you have to recreate NORMAL.DOT.

Public Sub RemoveAutomaticallyUpdate()

' RemoveAutomaticallyUpdate Macro

' Created by Gary Frieder

'Dim aSty As Style

For Each aSty In ActiveDocument.Styles

    If aSty.Type = wdStyleTypeParagraph Then

    aSty.AutomaticallyUpdate = False

    End If

Next aSty

End Sub

 

'MACRO THAT CRAWLS ALL THE STYLES AND TURNS AUTO-UPDATE OFFPublic

 

Sub RemoveAutomaticallyUpdate()

' RemoveAutomaticallyUpdate Macro

' Created by Gary Frieder

 

Dim aSty As Style

For Each aSty In ActiveDocument.Styles

    If aSty.Type = wdStyleTypeParagraph Then

    aSty.AutomaticallyUpdate = False

    End If

Next aSty

 

End Sub

You can only do this if 0% is proportionately the same distance between the maximum and minimum values on both axes. Thanks for letting us know you're new to this.  We'll be gentle. ;-)

First off, let me explain a little bit about fonts in PowerPoint.  They're not really IN PowerPoint.  Actually, they're installed into Windows itself; once there, pretty much any Windows program (Word, PowerPoint, Excel, Notepad) can use the fonts.

Now Office installs quite a few fonts, so do many other programs that you may have installed.

I don't have Windows 8 handy at the moment, but all of the fonts you mention are included in my Windows 7 + Office 365 / 2013 setup.  And by the way, all these fonts did in fact ship with Office 2010.

The first thing I'd do is go to Control Panel | Fonts and verify whether these fonts are installed on your computer or not.  

Next, start PowerPoint, then on the HOME tab, click the downward pointing triangle next to Replace (in the Editing group) and choose Replace Fonts.  Open the upper of the two list boxes on the Replace Font dialog box and check to see if any have question marks next to them.  Those are fonts that the presentation needs but that PowerPoint doesn't think you have.  Verify that these fonts are not available on the lower of the two lost boxes.

You should also choose a default printer, one that has a Microsoft approved driver.  As a test, choose the XPS driver as your default printer, at least temporarily.  Then close/open PowerPoint again and check that Replace Fonts dialog box again.
