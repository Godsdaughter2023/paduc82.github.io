I guess another way of askiing this question is - does this dialogue box accept anything other worhsheet range coordinates such as a fourmula in place of "=$c#11:$h$23"

Sales for Jan

Date 06/01/2010
Hdr1 Hdr2 Hdr3 Hdr4 Hdr5 Hdr6

jon - vertex42 say:

You can do this using a PivotTable. Select your check register table (the entire table) and go to Insert > PivotTable and put the pivot table in a new worksheet.

Then, add the Date, Name, Debit, Credit, and Cleared fields to the Row Labels. If you want a Grand Total for the Withdrawal and Deposit columns, add those columns to the Values section of the PivotTable and change their field settings to Sum instead of Count.

For each of the row fields, edit the Field Settings to select "None" under Subtotals and "Show item labels in tabular form" in Layout&Print. Also check the "Repeat Item Labels" box.

In the Date column, you can click on the filter button and choose from a variety of Date filters such as "This Month" or "Last Month" or a custom date range.

Normally for a report like this you wouldn't want the Pivot Table to group transactions (for example if they have the same date, name, cleared status, etc.) so you could add the Balance field to the row labels section (choosing "none" for subtotals and "show item labels in tabular form") because the balance would not typically be identical for each row.

You could also add a calculated field to the Pivot Table report to add (or subtract) the Credit and Debit so that you have an "Amount" field.

Another handy feature would be to insert a Slicer for the Cleared column so that you could easily filter your table for just the "x" or "o" values.
