Unknown file formats

I'm a silver haired N70 owner trying to keep the grey matter active! I get a lot out of my N70 (I think) but one problem always bugs me. If I try to open a file in FExplorer I often get the message "Unknown File Format". Why?

petrib say: In your phone (in the Symbian operating system), a file type "handler" is registered as a special type of a program called a "recognizer", which knows what to do with a particular file.

If there is no recognizer for the file, then you get that error.

In other words, FExplorer asks the operating system to launch whatever program knows about this file, and the operating system is saying: "I don't have any registered programs that knows how to handle this type of files, so I don't know what to do with it".

Now, of course, the operating system is saying this through the error message "Unknown File Format", but what it means is what I wrote above. ;)

Of course, it doesn't mean that there are no programs that know what the file is for. Just that there is no system-level recognizer for it. The program that "owns" the file can still know what it is for, and do whatever it does with it.

There is also a chance that the file was, for example, downloaded from a PC, the Internet or wherever, and it contains something no program on the phone knows what to do with (e.g., some unknown video file format, or whatever).

greginmich say:

Many developers have routine issues with Windows Defender’s CPU consumption and have to resort to finagling the right combination of path and process exclusions for the main executable, modules, compiler, and project files in order to resolve the issue.

Turning off Cloud-Based Protection and Automatic sample submission might help. Turning off Behavior Monitoring might also help:

https://technet.microsoft.com/en-us/library/dn433291.aspx

The Technet documentation is still incorrect for the Set-MpPreference parameters with a Disable prefix. The -DisableBehaviorMonitoring parameter is actually set to False (Disable False) by default, which means that behavior monitoring is turned on. To turn it off, we would set the value to True (Disable True) – by running this command at the elevated PowerShell prompt:

Set-MpPreference -DisableBehaviorMonitoring $True

Then confirm the new setting by running the Get-MpPreference and Get-MpComputerStatus commands (the value of BehaviorMonitorEnabled should be set to False when the value of DisableBehaviorMonitoring is set to True).

Try using another AV app for real-time protection:

Other AV apps generally have fewer issues with CPU consumption; better malware file detection and real-world protection rates; and fewer of these acute issues with buggy updates. In the current version of Windows 10 (Version 1607), you can install a third-party AV app for real-time protection, and then enable Windows Defender’s Limited Periodic Scanning mode in order to supplement the protection provided by the third-party AV app:

https://www.tenforums.com/tutorials/51514-turn-off-limited-periodic-scanning-windows-10-a.html

https://answers.microsoft.com/en-us/protect/wiki/protect_defender-protect_scanning/limited-periodic-scanning-in-windows-10-to-provide/3dcfc8dd-36b4-43f2-9a0a-732e43191cf3

Isolate the PC and turn off real-time protection:

If relieving Windows Defender of its real-time responsibilities isn’t an acceptable option; or if it doesn’t resolve the issue, then isolating the PC might provide a workable solution:

greginmich say:

The Behavior Monitoring, Cloud-based Protection, and Automatic sample submission features are all part and parcel of the infrastructure for dealing with emergent threats; and of course an application under development can easily mimic some of the characteristics and behaviors of an unknown threat – so that’s the rationale for turning off these secondary components of real-time protection.

But, as you say, turning off the infrastructure for emergent threat detection introduces a significant element of risk, especially in an AV app that’s already weak in that area (although I fully expect Defender to do better in the next round of testing) :

https://www.av-test.org/en/antivirus/home-windows/windows-10/october-2016/microsoft-windows-defender-4.10-164047/

So this would only be recommended as a stopgap measure in a situation where there was a good chance that the underlying issue would be corrected in a timely fashion. Unfortunately, these “acute onset” issues can be amazingly persistent – so the best strategy is usually just to install a well-respected third-party AV solution for real-time protection.
