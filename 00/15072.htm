tdsutter say:

Format and Input Mask for Date in Access

In a table, I want to store a date in the format YYYYMM. I have tried several combinations but I can't seem to get the right one. My latest attempt looks like this:

 

     Field Name: TermDate

     Data Type:   Date/Time

     Format:        yyyymm

     Input Mask:  !999999;0;_

 

When I try to input data, I am confronted the the following error message:

     The value you entered does not match the Date/Time data type in this column.

 

Any help would be greatly appreciated.

ken sheridan say: You cannot do that.  A date/time value is stored as a 64 bit floating point number which represents an offset from 30 December 1899 00:00:00, so can only be a specific point in time, not a value to a precision of a month or any other temporal unit.  To do this the best option is to use two columns of integer data type, one for the year, the other for the month, formatting the latter as '00' to show a leading zero.

If you then need to do date arithmetic on the value you can do so by using the DateSerial function to return a date of the first of the month, passing the stored year and month value and a constant 1 into the function as its arguments.  The resulting date/time values could then be used with the DateDiff function for instance.

If for any reason it is necessary to use a date/time data type then you would have to enter the value as the first of the month, and format it as 'yyyymm' to show only the year and month.
