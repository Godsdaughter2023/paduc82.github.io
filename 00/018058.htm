ken sheridan say: You can use the Switch function for this, but first can the DateDue value be manually changed by the user after being computed?

If the answer is Yes then you do need a DateDue column in the table and you need to push the value into it, which you'd do by adding the extra code to each of the C controls AfterUpdate event procedures, and by putting the same code in the AfterUpdate event procedures of the Status control and the CalibrationDate control in the form, the latter two being for if these are edited manually.  The column in the table would need to be of text data type to accept the text values as well as the date.  The code would be:

Me.DateDue = _
    Switch([Status]="Current/In Service" Or [Status]= "See Certificate", _
    DateAdd("yyyy",1,[CalibrationDate]), _
    [Status]="Out of Calibration","Upon Correction", _
    [Status]="Missing","When found", _
    [Status]="Broken","When fixed", _
    [Status]="New","Before use", _
    [Status]="Sent For Repair","Upon return", _
    [Status]="Removed From Service","None")

If the answer is No then you don't need a DateDue column in the table and you can pull the value into an unbound control with an expression as ControlSource property of an unbound control in a form or report.  The expression would be like the above, but all as one line:

=Switch([Status]="Current/In Service" Or [Status]= "See Certificate", DateAdd("yyyy",1,[CalibrationDate]), [Status]="Out of Calibration","Upon Correction", [Status]="Missing","When found", [Status]="Broken","When fixed", [Status]="New","Before use", [Status]="Sent For Repair","Upon return", [Status]="Removed From Service","None")
