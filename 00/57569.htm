Phil,

About the System repair disk

You can boot from your system repair disk to restore a system image.  You can also boot from an installation disk to do the same [at about the third screen shown, you choose Repair your computer rather than Install now]. 

Windows also allows the restoration of a system image from within Windows itself but that necessarily assumes that Windows is working well enough for you to be able to do this.  I had this method fail on me once a few years ago so I realised that I would always need a system repair disk or installation disk handy to boot from.  Since one of them is therefore always handy, I can always restore Windows images by booting from one of them [this also acts as a test that they still work correctly but there is no particular reason to suppose otherwise].

You can see the contents of a vhd file within your system image by attaching it, as you already know.  You can also use this method to copy a file back onto your HDD [I have never had an reason to do this].  But you must always be careful to avoid changing an image [changing anything within the whole WindowsImageBackup folder] as that can make the whole image unusable.

- - - - - - - - - - - - - - - - -

So you are now ready to partition your primary HDD or to complete the reinstallation of all your applications.

You will want to make new system images afterwards so you might wish to create a library of images on your external HDD as you go along.  You can just create a folder on your external HDD with a suitable label such as the date and then move the whole WindowsImageBackup folder into it after it has been made.  You can only restore an image after it has been moved back to K:{etc}:\WindowsImageBackup but the images can be

When you changed folder permissions, all of its contents will also be affected. If you want to change permission on certain sub-folders, it is recommended to separate it up to one level and change only specific file or folder permissions which are applicable to you.
