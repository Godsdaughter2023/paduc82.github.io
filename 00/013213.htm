In addition, there are two methods to prevent the pushed Auto-upgrade and stay on Office 15 currently. Please check the information below:

1) Use Office Deployment Tool(ODT) for Office 2013 to deploy Office 365 for Office 2013 and disable the AutoUpgrade. Here is an example:

<Configuration>

<Add OfficeClientEdition="32” Branch="Business">

<Product ID="O365ProPlusRetail" >

<Language ID="en-us" />

</Product>

</Add> 

<Updates Enabled=”TRUE” AutoUpgrade=”0”/>

When the AutoUpgrade is set to 0, it will prevent eligible Office subscription installations from automatically upgrading to the next Office major version for a year. During this period, customers will also continue receiving security updates for 12 months.

[HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\office\15.0\common\officeupdate]
"enableautomaticupgrade"=dword:00000001

But it opens a new instance of Outlook.  Is there any way to make a toolbar button that jumps to the public folder within the same instance of Outlook?

rich007a say:

Are you running this from within Outlook?  In that case "Application" is... the outlook application!

In that case you just need:

    Dim myNameSpace As NameSpace
    Dim myFolder As Folder
    Set myNameSpace = Application.GetNamespace("MAPI")
    Set myFolder = myNameSpace.Folders("Public Folders").Folders("All Public folders").Folders("Test")
    myFolder.Display

 

But if (an only if) you are trying to automate Outlook from another application like Excel, then really you should test to see if Outlook is running already (and if it isn't then start it).  You can do that like this:

    Dim oOutlookApp As Outlook.Application
    Dim oNS As Outlook.NameSpace
    Dim bWeStartedOutlook As Boolean
    Dim strCurAddr As String
   
    On Error Resume Next
   
     'Get Outlook if it's running:
    Set oOutlookApp = GetObject(, "Outlook.Application")
    If Err <> 0 Then
        On Error GoTo CleanUp
         'Outlook wasn't running, start it from code:
        Set oOutlookApp = CreateObject("Outlook.Application")
        bWeStartedOutlook = True
    End If
    On Error GoTo CleanUp
   
   
     'DO STUFF WITH OUTLOOK HERE...(FOR EXAMPLE):
     'Get current user's e-mail address:
    Set oNS = oOutlookApp.GetNamespace("MAPI")
     'Need to logon if we started outlook:
    If bWeStartedOutlook Then oNS.Logon "", "", False, False
    strCurAddr = oNS.Session.CurrentUser.AddressEntry.GetExchangeUser.PrimarySmtpAddress
    
   
CleanUp:
    If bWeStartedOutlook Then
         'If we started Outlook from code, then close it:
        If Not oOutlookApp Is Nothing Then oOutlookApp.Quit
    End If
   
    Set oNS = Nothing
    Set oOutlookApp = Nothing
End Sub

 

Hope that helps.
