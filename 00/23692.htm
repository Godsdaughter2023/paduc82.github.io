Going to assume for the sake of this reply that you only have the one IMAP account configured in your profile AND your contact/calendar data is in folders with the added words "This Computer Only" - is that a correct assumption?

Given the answer being yes to the above - one process to accomplish what you want would be as follows:

#1 - Export the IMAP data to a PST file selecting the <Root> of the folder tree as the source along with making sure to "include all sub-folders"

How to Export to an Outlook PST File
https://www.contactgenie.info/how-to-export-out...

----------------------------------------------------------------------
Required Forum Disclaimer:

The above link is a non-Microsoft website. The page appears to be providing accurate, safe information. Watch out for ads on the site that may advertise products frequently classified as a PUP (Potentially Unwanted Products). Thoroughly research any product advertised on the site before you decide to download and install it.

*** the above links refer to pages that are mine and external links that appear (if any) can be trusted
--------------------------------------------------------------------------


#2 - Once everything is exported, add the PST file created by the export process to the existing Outlook profile via
Home ribbon --> New Items --> More items --> Outlook data file --> browse to the PST file

Reason for this is to
(a) rename any folders that have the words "This Computer Only" in the description to eliminate those words from the folder description (only from the PST folder tree)
(b) delete all items from the <Inbox> by
- selecting the <Inbox>
- press CTRL+A to select all items
- press the <delete> key on the keyboard
(make sure that you only delete the items from the <Inbox> in the PST file folder tree)

*** make sure that everything else has been properly exported to all other folders prior to proceeding with the next steps

#3 - Go to File --> Account Settings --> Data File tab --> select the PST file and set it as the default data file for the profile

*** I haven't gone thru this specific step and don't have time to test it right now so <IF> you end up having duplicate folders for those that were previously labeled as "This Computer Only", then will need to copy the items to the newly created folders that occurred when the PST file was set as the default data file. The default data file contains the default contact/calendar folders.

#4 - return to Outlook --> File --> Account Settings --> Manage Profiles --> select <Show Profiles> --> make a <copy> of the current profile and give it a name

#5 - Highlight the newly created profile and click on <Properties> --> click on <Email Accounts>

#6 - Highlight the IMAP account and remove it

#7 - Select <New> to add the POP3 account by manually configuring it AND making sure that you select the existing PST file in the profile for the POP3 account instead of letting Outlook create a new PST file (this step is really important to save a whole bunch of time)

#8 - Set the new profile as the default for the next time Outlook is opened. Close outlook an re-open it.

All done, when Outlook opens up all the <Inbox> data will be downloaded from the server.

After using Windows 10 - Microsoft Community

jean-françois peyroux say:

After using Windows 10

My beta user feedback.

----

After months of hesitations, I finally installed Windows 10 on one of my main computers. I have now a large array of windows versions on PCs I used every week. Win7, Win8.1, Win10, and Server 2012.

 Win8 on the UI front is a combination of two distinct worlds: Metro/Modern, and Windows 7. The gap between them has been in large part closed in Win10, with the Metro UI displacing the older Win7. It brings more unity, but not necessarily more appeal. For many it will be the opposite. The Windows Phone UI is invading the desktop. Aggressive color schemes, aggressive geometric shapes, no texture, no “glass”, sub optimal use of the real estate. It resembles the special Win7 UI mode for people with impaired vision, where contrast is paramount over aesthetic consideration! The grossly overpriced iPhone is selling like crazy because it is perceived as beautiful and cool. I doubt that Windows 10 will land in the same category. It might be perceived for most like Win8: something you have to have not something that you are pleased to have. The thousands of engineers at Microsoft working on well-known worthless projects, would be better off to be put on enabling multiple look and feel (aka skins), so that users have an array of choice.

The Start menu is back from Microsoft. It takes about 25% of the real estate on this laptop, which is way better than the whole screen as with Win8. Still it should take no more than 10%! It is divided in two parts: a list of apps, and large icons for the Metro apps taking most of the real estate. I don’t see the point of having any of the large Metro icons in the first place. It is a worthless annoyance. If “ live tiles” are a great feature then they should be located on the desktop, not on the start menu! (which I have not been able to do). The list of applications available is organized by places, most used, recent, and alphabetical. I would have liked to be able to perform some drag, and drop to place them according to my personal preferences. Applications can be pinned to the start menu, but then the icons are absurdly big! I miss the ability to select documents straight from the Win7 start menu. This would be perfectly fine if the “search” box would pull the name of a document instantaneously after typing a couple of letters, but it is not the case.

Metro app now runs in the desktop. For the first time I am going to consider looking for apps in the Windows App store. This is probably the biggest single feature for this release of the OS. (That feature has been available for the last 3 years from StartDoc as “Modern mix”, and I used to have it. However I had my Start8 button, which was making me never to see Metro in any shape or form, and so I was happily staying away from them.)

The Windows phone “notifications” feature has landed in Win10 and I like it. Search across everything would be great if the search engine could be specified. May be this will be settable in the near future, otherwise an outcry is unavoidable. “Bing” market share is in the low single digit in all the markets but the US; simply because it does not provide a competitive offer in all the markets but one!

I am going to continue using Win10 on this laptop, and will put it on my work computer as soon as released. It is a step forward compared to Win8, but some are going to argue that it is still a step backward compared to Win7. The call would be based on what matter most: the regressions in the UI, or the improvement in the feature set. All in all, I don’t see this version of the OS to reverse the decline of the Windows sales. Folks will continue to spend their limited budget on electronics they are more excited about.

ken sheridan say: The problem is not with the parameter, but with the expression Int(Log([Var_Unif_#])*1000)/1000.  I think this is because the [Var_Unif_#]  for April Diskey is Null.  You can correct this by calling the Nz function to return a zero in place of the Null, and calling the Val function to sort the values numerically rather than as string expressions.

PARAMETERS [Which season is this? 4 digits - hyphen - 2 digits] TEXT(7);
SELECT Roster_Info.[Var_Unif_#], [FirstName] & " " & [LastName] AS Name, Count(Game_Stats.Game_Played) AS CountOfGame_Played, Sum(Game_Stats.Baskets_Made) AS SumOfBaskets_Made, Sum(Game_Stats.Baskets_Attempted) AS SumOfBaskets_Attempted, Sum(Game_Stats.Field_Goals_Made) AS SumOfField_Goals_Made, Sum(Game_Stats.Field_Goals_Attempted) AS SumOfField_Goals_Attempted, Sum(Game_Stats.[3pt_Baskets]) AS SumOf3pt_Baskets, Sum(Game_Stats.[3pt_Attempts]) AS SumOf3pt_Attempts, Sum(Game_Stats.Free_Throws_Made) AS SumOfFree_Throws_Made, Sum(Game_Stats.Free_Throws_Attempted) AS SumOfFree_Throws_Attempted, Sum(Game_Stats.Points) AS SumOfPoints, Sum(Game_Stats.Cal_Pts_1) AS SumOfCal_Pts_1, Sum(Game_Stats.Cal_Pts_2) AS SumOfCal_Pts_2, Sum(Game_Stats.Offensive_Rebounds) AS SumOfOffensive_Rebounds, Sum(Game_Stats.Defensive_Rebounds) AS SumOfDefensive_Rebounds, Sum(Game_Stats.Unknown_Rebounds) AS SumOfUnknown_Rebounds, Sum(Game_Stats.Assists) AS SumOfAssists, Sum(Game_Stats.Steals) AS SumOfSteals, Sum(Game_Stats.Blocked_Shots) AS SumOfBlocked_Shots, Sum(Game_Stats.Turn_Overs) AS SumOfTurn_Overs, Sum(Game_Stats.Personal_Fouls) AS SumOfPersonal_Fouls, Sum([Fouled_Out]*-1) AS DQ
FROM (Players INNER JOIN Roster_Info ON Players.[Player_ID] = Roster_Info.[Player_ID]) INNER JOIN (Games INNER JOIN Game_Stats ON Games.[Game_ID] = Game_Stats.[Game_ID]) ON Roster_Info.[Roster_ID] = Game_Stats.[Roster_ID]
WHERE (((Roster_Info.Season_Play)=[Which season is this? 4 digits - hyphen - 2 digits]))
AND Games.Skill_Level="Varsity"
GROUP BY Roster_Info.[Var_Unif_#], [FirstName] & " " & [LastName], Players.Player_ID, Game_Stats.Roster_ID, Games.Skill_Level, Players.FirstName, Players.LastName
ORDER BY Val(Nz([Var_Unif_#],0));
