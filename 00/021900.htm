ossiemac say:

One of the problems with copying charts to another workbook is that the data for the chart still references the original workbook.

I have had a similar problem and the only way I could work around this was to use the Worksheet copy method and then delete any superfluous data and/or objects from the new workbook and use Copy -> Paste Special -> values if there were any references in formulas back to the original workbook.

Following guidelines to copy the worksheet just in case you are not familiar with it.

  1. If you want the VBA code then turn on the Macro Recorder first.
  2. Right click the worksheet tab
  3. Select Move or Copy
  4. Click the Drop down in the To book field and select New Book
  5. Check the Create Copy box. (Very important or you loose the original because it is moved to new book.)

Following is VBA code example to perform this action.

Sub Macro3()

    Sheets("Sheet1").Copy   'This line creates the new workbook
   
    'Save the new workbook and provide a workbook name
    ActiveWorkbook.SaveAs Filename:= _
        "C:\Users\User\Documents\Excel\Test Macros\NewBook.xlsx", _
        FileFormat:=xlOpenXMLWorkbook, _
        CreateBackup:=False
          
End Sub

How can I use a checkbox to Hide/Unhide a text box? - Microsoft Community

christopher hastings say:

How can I use a checkbox to Hide/Unhide a text box?

I need a way to connect a checkbox to a text box so that when the check box is selected, the text box appears. Otherwise I need the text box hidden. Is this Possible? Thanks for you help in advance!
jay freedman say:

Excellent!

Since you mention having "several places" with the same setup, if they'll all be in the same document, then you need to know a few more things.

First, each check box content control in the document must be given a different Tag value so the macro can identify which one triggered the event. Also, each bookmark that surrounds a text content control must have a different bookmark name. [Note: A Word document can't contain two or more bookmarks with the same name. If you try to add another bookmark with an existing name, the existing bookmark is automatically deleted. However, the Tag and Title values of content controls don't behave the same way, so it's up to you to make sure the values you give them are unique.]

Second, the document (or its template) can have only one event handler named ContentControlOnExit, which is called when the cursor exits any content control. The way you code it to handle multiple check boxes is to test the Tag (or the Title) for each possible value that needs to be used. There are several ways to do that:

  1. You can write a separate If...Then...End If structure for each value. Each time the handler runs, only one of the If conditions will be true, so only that one will execute.
  2. You can write a single structure using If...Then...ElseIf...Then...ElseIf...Then...etc....End If. This works exactly the same way as multiple If structures, just with fewer lines.
  3. For more than three possible values, the first two methods become cumbersome. Instead, you can use a Select Case structure, which executes exactly one of the Case groups within it.

Select Case ContentControl.Tag

   Case "ckSidejobs"

        ActiveDocument.Bookmarks("bkSidejobs").Range.Font.Hidden = _

            Not ContentControl.Checked

   Case "ckOtherCondition1"

        ActiveDocument.Bookmarks("bkOtherCondition1").Range.Font.Hidden = _

            Not ContentControl.Checked

   Case "ckOtherCondition2"

        ActiveDocument.Bookmarks("bkOtherCondition2").Range.Font.Hidden = _

            Not ContentControl.Checked

   ' maybe more cases

   Case Else

        ' do nothing

End Select


jay freedman say:

If you're satisfied with just hiding/displaying the tab and text content control based on the state of the check box, you don't need a building block at all, which eliminates any problems involved in inserting and deleting a building block.

Set up the arrangement shown in the second screen shot. Select the text content control and insert a bookmark around it, not including the tab character that precedes it. Let's say you name the bookmark bkSidejobs. While the text content control and its bookmark are selected, go to the Font dialog and turn on the Hidden formatting.

Next, click in the check box content control, click the Properties button on the Developer tab, and enter a name in the Tag box; let's call that ckSidejobs. (Actually, you could use the same name for the check box and the bookmark, but I recommend using the different prefixes to avoid confusion in the code.)

Finally, using the steps I outlined for you in this thread, create the ContentControlOnExit event handler with the following contents:

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
    If ContentControl.Tag = "ckSidejobs" Then
        ActiveDocument.Bookmarks("bkSidejobs").Range.Font.Hidden = _
            Not ContentControl.Checked
    End If
End Sub

What this says is "If the content control that was just exited is the one with the Tag value ckSidejobs, then make the range of the bookmark named bkSidejobs visible (= not hidden) if the check box is checked, or make it hidden if the check box is not checked. If the content control that was just exited is NOT the one with the Tag value ckSidejobs, then do nothing."

jllatham say:

Try using Page Setup to set really large top and bottom margins.  The exact numbers will depend on your row heights and actually will have something to do with the printer.

 

Taking a regular worksheet set up in Portrait layout on 8.5x11 inch paper, I found that if I set the top margin to 5.5 inches and the bottom margin to 5.25 inches, that I got just 1 row per page printed; effectively putting a page break after each row.

 

In the given example, the printout puts the row in the center (vertically) of the printed page.  You could play with the margin settings to get it to print elsewhere on the sheet.  With a top margin of 1 inch and a bottom margin of 9.75 inches, the single row gets printed 1 inch from the top of the sheet.

 

Hope this helps.
