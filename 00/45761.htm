cmdrkeene say:

Most of them get an external trigger. For example, System Restore will create a restore point automatically on a schedule.

The task is launched everyday to see if a restore point has been created within the last 7 days.  If no restore points have been created in 7 days, an automatic restore point is created then.  If there's already a recent one, it doesn't create another (to save disc space).

If you want to create them more often, check out here: http://www.sevenforums.com/tutorials/67114-system-restore-point-create-system-startup.html

But if you want more than one to be available later, make sure you go adjust the slider so it can use more space to store them, or each new one will delete the others.

chw5007 say:

The printer may have reset itself and changed its IP address in the process [which happened to me]. To fix this go to the Network & Sharing Center --> Full Map and click on your printer. This will tell you what your printer's current IP address is [the number that should be something like 192.168.1.xxx]. Next, go to Devices & Printers --> Printer Properties --> Ports --> Standard TCP/IP Port for your printer --> Configure Port. You then want to change the IP address where it says 'Printer name or IP address' to the new IP address that you found.

That's not the cause, then. but having both installed may be. This problem usually arises because Windows Live Mail only understands the Open command, while different file extensions may have several different commands associated with them.
