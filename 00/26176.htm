joeu2004 say:

Eric wrote:

If using round to make sure it only have two decimal point, when sum it up it still can't get zero.

You are rounding the wrong things.  In the second example, assuming that column B contains constants (e.g. you typed 2942.7; you did not calculate it), there is no reason to round them to the same or greater precision.

What you need to round is the SUM formula.  Write =ROUND(SUM(B2:B180),2).

In general, whenever you expect a

This has nothing to do with the version or update of Excel that you have.  It is an inherent side-effect of the way that Excel represents numeric values internally, namely 64-bit binary floating-point.  Most decimal fractions cannot be represented exactly in that binary form.  And the binary approximation of a particular decimal fraction varies depending on the magnitude of the number.

That is why, for example, IF(10.1 - 10 = 0.1, TRUE) returns FALSE(!).

However, there is an alternative to explicit rounding formulas:  setting the "Precision As Displayed" option.  That might explain why other users do not experience the problem.  Or they might simply have not stumbled across the problem yet.

I deprecate the use of PAD for many reasons.  Chief among them:  merely setting the PAD can alter some constants

For that reason, if you choose to experiment with PAD, be sure to make a back-up copy of the Excel first.

john korchok say:

There are several ways to do this, but a simple one is to create a style that is applied to only the table cell that is to be the data source. The style doesn't have to look any different, it just has to have a different name.

Then, at the spot where you want the table cell information to repeat, use Insert>Field..., select Links and References in the Categories list, then choose StyleRef from the bottom of the Field names list. Click on the Options button, then on the Styles tab and select the style you created for the table cell. OK out and the cell data will display in the StyleRef field.

The contributors to this forum are not Microsoft employees, do not speak for Microsoft and have little to no influence over Microsoft's design process. The reasons for the change to the ribbon have been well documented since its introduction, with the pre-release versions of Word 2007, at least 5 years ago and the arguments for and against have been waged First and foremost this was a business decision to stimulate sales.

Microsoft's response to the demand for more hands-on user configuration has been to add a ribbon editor to Office 2010. There will be no patch to make Word 2010 use the interface of Word 2003.

The idea of a third party add-in, which attempts, but ultimately fails, to recreate the menu structure of Word 2003 is a cul-de-sac that ultimately will inhibit your ability to adapt to the new menu ribbon driven system. Better to fine tune the ribbon to your own requirements.

How to set I can print full color for A4 paper size on PowerPoint - Microsoft Community

mark kwok say:

How to set I can print full color for A4 paper size on PowerPoint

I can not print color to fit whole A4 paper size on PowerPoint.
steve rindsberg say:

In the print dialog box, put a check next to "Scale to fit paper" then try printing again.

If that doesn't do it, read on ...

Not all printers are able to print clear to the edge of a page.  Most impose a 1/4" or so border all 'round.  If that's the problem, there's not much to be done about it (but check your printer properties dialog ... sometimes you can set something there to allow borderless printing).

If your presentation's Slide Size isn't A4 or proportional to it, PPT will make the slides as large as possible on the printed page w/o distorting them; if the paper and presentation sizes aren't proportional, you'll get larger borders.
