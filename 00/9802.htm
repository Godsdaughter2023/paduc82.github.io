Normally you would want to run your update routine only when the user has
committed the changes to the current record, in the form's AfterUpdate
event.  Otherwise, you run the risk of having the user make a change to one
of the fields, triggering your update routine, and then going back and
undoing that field change -- but your update procedure has already been run.
 
That said, it *is* possible to use a trick with conditional formatting to
call a user-written function that keeps watch on the value of a calculated
control, and runs some code whenever that calculated value changes.  That
would relieve you of the need to create an AfterUpdate event procedure for
each of the fields that contribute to the calculated field.  It's a neat
trick I just discovered, if I do say so myself, but I'm not sure that it's
really useful in this scenario, for the reason I mentioned above.

To investigate these possibilities, do the following steps until you find the solution or run out of steps.

  1. Turn on the display of nonprinting characters by clicking the ¶ button on the Home ribbon. If the missing text appears and has a dotted underline, select it and press Ctrl+Shift+H to remove the Hidden formatting.

The person did not access the account (WITH A BROWSER) within 270 days. The account will be deleted permanently in the system due to inactivity.

doug robbins - mvp office apps & services (word) say:

The way in which a Word document appears is dependant upon the active printer on the machine upon which it is opened.  The only way to ensure absolutely consistent appearance is to save the document in portable document format (pdf).

ashlillings say: Thank you for responding! Perhaps if I could understand why the printer affects the formatting and layout I could try to find a work-around. Could you provide more specifics?

charles kenyon say:
I agree with Doug, that is the only way.

Word prepares its on-screen display according to what the printer will do with it.

You can try using generous white space, wide margins and common fonts. Use page-break before formatting for paragraphs that you want to start a new page, not page or section breaks.

Are you using tables to organize information? If you are using the space bar or tabs, it can really make a difference when the document is formatted for a different printer. Also, make sure that your document is not set to automatically update its styles from the template.
See What happens when I send my document to someone else? Will Word mess up my formatting?

I regularly require a resume in Word format, because Word skills are essential in those working for me. I do it so that I can examine the formatting rather than just see what it looks like. The formatting tells me as much as the content.  Are there tables? Does the applicant use the space bar or tabs to position text horizontally? Are there extra empty paragraphs? Does the applicant use styles and use them with purpose?

Those are things that make a difference when you are trying to edit documents even though something that looks good can be done without knowing how to do them. My requirement still doesn't get around the possibility that someone else prepared the resume or they did a good job using a template, but it is a start. I would recommend sending both the pdf and the Word version.
