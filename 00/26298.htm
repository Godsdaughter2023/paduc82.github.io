rich michaels - mvp (greatcirclelearning.com) say:

Hi Bridget,

I've seen posts that said Word keeps a low res PNG version of graphics for quick display but I've never found one in the xml bundle so I'm not certain of the claim. Your size issue is definitely because of the EMF file type but there maybe other factors.

You mention that you use EMF because they are "searchable", can you say more about that because I'm not familiar with that feature versus the other formats?

In my experience the image quality and size issue with graphics in Word and PowerPoint documents are all about pixels. There's either too many or to few for how the document is being used.

The first thing I would check on your system is if Word is being allowed to "compress" graphics. It is an option setting under the Advanced category and it's in the Image Size and Quality group. You do not want Word compressing your graphics because it uses a "resampling" process that discards pixels in order to reach the specified PPI. Instead you should be concerning yourself with the original size and pixel density of the image being inserted. And if you do that well, you might be able to use one of the other file formats, which will result in a smaller file size and still retain image quality.

Assuming that your document is meant for print, then you optimally want your graphics to have a PPI density of around 220 and to get to that number you "scale" your image appropriate to the space it is being displayed on the page. If the width of the display space is 7 inches then the width of the original image should be 1540 pixels, which would result in roughly the 220 PPI density. If you insert into the same 7" wide space an image that is 3000 pixels in width then you are unnecessarily increasing file size without an equal benefit in print quality.

So the point is don't compress an image by resampling, which discards pixels, and instead scale your image to the appropriate PPI based on size and output requirements. 100 PPI is for screen/web viewing. 150-220 PPI is best for printed documents. And 300 PPI is for high definition viewing and printing in the largest sizes (22" width maximum for Word, 30" maximum for PowerPoint).

Hope this helps

It sounds as if you're trying to type a specific number into the header. You can't do that (or at least you can't do that and expect to have a different number on other pages). Instead, in each section where the pagination is incorrect, use Insert | Header & Footer | Page Number | Format Page Numbers... to set the page number format to "Continue from previous section."

If you have used an Odd Page section break (or have started a new section at 1, which will cause Word to force an Odd Page break), Word will insert a completely blank even page as needed. You won't see this page in your document or even in Print Preview (except under specific conditions), but it will "print." That blank page will be counted in the pagination, and you should not try to change that.

The same thing happens to me and I understand it.  It works like other Windows applications.  The window containing the message is just a display on the screen; a window independent of the other Outlook windows.  Once you paint something to the screen in its own window, it will remain there until you close that window.  For example, open a document in Notepad or Word so that you can see it on your screen.  Now, using Windows Explorer, delete the file.  The window you opened showing the file's contents will remain on the screen until you explicitly close it even though the file is gone.  You deleted it in a separate process.  The original process doesn't care.

The Delete button in the message window works because it also sends a close signal to the window manager.
