andreas killer say:

I want to be able to insert a row and the item numbers automatically renumber themselves based on the previous rows number + 1


I think this can not be done with formulas, because you must search for the last filled cell above the current cell.

Add the code below in a normal module and insert the formula

  =GetPreviousNumberedRow()

in every cell that should have a number.

Andreas.

Function GetPreviousNumberedRow() As Long
  'This function must be calculated every time
  Application.Volatile
  'Get cell where we are called from
  With Application.Caller
    'Get current row
    GetPreviousNumberedRow = .Row
    'If we are below the first row
    If GetPreviousNumberedRow > 1 Then
      'Is the cell above empty?
      If IsEmpty(.Offset(-1, 0)) Then
        'Yes, count on from last filled cell
        GetPreviousNumberedRow = .End(xlUp) + 1
      Else
        'Next number from cell above
        GetPreviousNumberedRow = .Offset(-1, 0) + 1
      End If
    End If
  End With
End Function

jllatham say:

Here is a possible solution that doesn't require a formula on your worksheet.  It works based on the fact that when you use Insert Row the Worksheet_Change event fires.  The indicated change column is always 1 (which would be perfect for your situation), but I've written the code so that any column can be "monitored" and set it up to monitor your A column.  It will not "see" rows inserted below the last used row in the column that is monitored, which could be an issue. 

However, you could overcome that by putting this formula in new rows at the bottom of the list:  =Max(A1:A99)+1 -- this assumes that the new row is at row 100, so the formula is referring to all cells ABOVE the new row you're using.  This will make sure that the entry is "max previously assigned + 1".  This adds a new problem: what if you sort the list later?  The cells with the formulas may change values!  So the actions for this situation might be: put in the formula as I showed earlier, and then simply overtype it with the value it returned; that will make it an unchanging number and sorting later won't affect it.  [Another alternative: put this formula in a cell not in column A: =MAX(A:A) which would always show the largest number in column A and then refer to that and enter a value 1 greater in your new cell].

To put the code to work, (work with a copy of your workbook for 'safety') - open the workbook, select the sheet this needs to function on and right-click the sheet's name tab and choose [View Code] from the list that appears.  Copy the code below and paste it into the code module presented to you.  Close the VB Editor and test it out by inserting rows between existing entries on the sheet.

Private Sub Worksheet_Change(ByVal Target As Range)
' change this Const as needed to
' refer to column with #s to monitor
  Const monitorCol = "A"
  Dim checkRange As Range
  Dim changedCell As Range
  Dim currentMax As Long ' assumes integers

  'Insert Row will trigger Change event but
  'will indicate the change was in column 1 (A)
  'so 2 tests are required, first test is
  'for change appears to be in column A
  If Target.Column <> 1 Then
    Exit Sub
  End If
  'second test, is specific column at change row
  'now empty.  If not, then wasn't an Insert Row action.
  If Not IsEmpty(Range(monitorCol & Target.Row)) Then
    'no new empty cell in monitored column, quit
    Exit Sub
  End If
  'a change was made in monitored column, see if
  'there is a blank cell in it
  Set changedCell = Range(monitorCol & Target.Row)
  'find current max in column that is monitored.
  Set checkRange = Range(monitorCol & "1:" _
   & Range(monitorCol & Rows.Count).End(xlUp).Address)
  currentMax = Application.WorksheetFunction.Max(checkRange)
  'put new high value into empty cell
  Application.EnableEvents = False
  changedCell = currentMax + 1
  Application.EnableEvents = True
  'housekeeping
  Set checkRange = Nothing
  Set changedCell = Nothing
End Sub

It seems like you could use something like (starting in cell A2)

That's because the 'relative' part of the address will continue to point to the row above the newly inserted row, not the newly inserted row just above itself. The longer formula I suggested should update correctly after inserting new rows, whether or not the formula has been copied into new cells or formula cells changed to blanks (or text).
