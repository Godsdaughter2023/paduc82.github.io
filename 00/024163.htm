Query to return highest value of multiple fields in the same record - Microsoft Community

sgeorg say:

Query to return highest value of multiple fields in the same record

I know how to return the maximum value in a particular field from a table with multiple records.  But in this case, I would like to make a query that will return the most recent date out of three date fields in each record.

Any help appreciated!

hansv mvp say:

You could copy the following function into a module in the Visual Basic Editor:

Function FieldMax(ParamArray arrFields())
    Dim dtmMax As Date
    Dim i As Long
    Dim f As Boolean
    For i = LBound(arrFields) To UBound(arrFields)
        If IsDate(arrFields(i)) Then
            If f Then
                If arrFields(i) > dtmMax Then
                    dtmMax = arrFields(i)
                End If
            Else
                dtmMax = arrFields(i)
                f = True
            End If
        End If
    Next i
    If f Then
        FieldMax = dtmMax
    Else
        FieldMax = Null
    End If
End Function

Use like this in a query:

MostRecentDate: FieldMax([DateField1], [DateField2], [DateField3])

where DateField1 etc. are the names of the date fields.

ken sheridan say: It's easily done with a little function such as that which Hans posted, but your question sets my relational antennae twitching.  What attributes do these three date columns represent?  If they are in fact three values of the same attribute then the table is not normalized to First Normal Form (1NF), whose definition is:

First Normal Form: A relvar is in 1NF if and only if, in every legal value of that relvar, every tuple contains exactly one value for each attribute.

Loosely speaking, in the language of the relational model, a relvar (relation variable) equates to a table, and a tuple to a row (record).

By normalizing such a table by decomposition into two related tables the latest date which relates to each  row in the referenced table can then be easily returned by means of the MAX aggregation operator.

sgeorg say:

Hi Ken,

I can't pretend I understand everything you say here (it's like another language!), but I can explain a little about how the three fields are used.  I am using Access to track an multi-step application review and approval process that involves multiple internal departments. The first date is the original application submisssion date, and every record has this field completed.  The second and third dates are resubmission dates if the application was denied at 1 or 2 points along the process, and then resubmitted.  Not all records have a date in these either or both of these fields.  I wanted to track the average processing time from the last date an application was submitted until it was approved, and to do so I needed a way to return the latest of all three dates.

ken sheridan say: Your description of the underlying reality confirms what I suspected to be the case.  A relational database models the reality in terms of its entity types and the relationships between them.  Each entity type is modelled by a table and the attributes of the entity types are modelled by columns in the tables.  The entity types need not be physical ones, and are in many cases abstract.  A relationship type is in fact an entity type, merely one of a special kind.

If this sounds like a foreign language, that's because it is in some respects.  Like all disciplines relational database design has a vocabulary of its own.  I try to use as plain English as possible here, but without dumbing it down to a level where its meaning is lost.  

What you have in essence is a many-to-many relationship type between applications and submissions, so this should be modelled by Applications and Submissions tables in which the latter has a foreign key column which references the primary key of the former.  The Submissions table will have columns which represent the attributes of each submission, the date being one, but which will probably also include a column to indicate the success or failure of the submission.  You will know better than I what other attributes each submission can have.

With a correctly normalized design like this aggregation operations become set operations by means of queries. In your case there is firstly the MIN and MAX operations over the subsets of submissions to get the first and latest dates per application, and then an AVG operation over the complete set of applications to get the average duration between the first and latest dates per application.

What you have at the moment is really more akin to a spreadsheet than a relational database table.  In fact some of my colleagues here have been known to refer to this type of design as 'committing spreadsheet'.  As you've seen you can often get the data to confess if you torture it for long enough, but that does not hide the fact that it's poorly structured data.

I hope I haven't left you in too much need of an interpreter, but good relational database design really is a question of following the advice given by Richard Feynman to his students in the quote below my signature.

ron sommer mvp 2009-2017 say:
Outlook 2010 To configure an outlook.com email account in Microsoft Outlook 2010, follow these steps:
  1. Install the Outlook Hotmail Connector, if you have not already installed it. You can install the Outlook Hotmail Connector from the following Microsoft website:
    http://office.microsoft.com/en-us/outlook-help/microsoft-office-outlook-hotmail-connector-overview-HA010222518.aspx
    (http://office.microsoft.com/en-us/outlook-help/microsoft-office-outlook-hotmail-connector-overview-HA010222518.aspx)

    Note Other Windows Live domains such as @hotmail.com and @live.com are automatically configured when you use the Outlook Hotmail Connector. However, you must manually configure an @outlook.com account.
  2. Manually configure your @outlook.com account. To do this, follow these steps:
    1. In Outlook, click Info on the File menu, and then click Add Account.
    2. In the Add New Account dialog box, click Manually configure server settings or additional server types, and then click Next.
    3. Click Other, click to select Microsoft Outlook Hotmail Connector, and then click Next.
    4. In the Windows Live Hotmail Settings dialog box, enter your name, email address, and password, and then click OK.
    5. When the confirmation dialog appears, click Finish.
