frederik long say:

To put it bluntly, putting a working disk into a different machine is a terrible idea. There are two reasons:

  • It is very rare that Windows will run on different hardware. In most cases it will crash.
  • During the first start-up, Windows will try to accommodate the changed hardware. During this process it will make a large number of changes in the registry. When you take the disk back to the old hardware then the registry is all wrong.

There is a small chance that you can fix the problem by restoring a backup copy of the registry that is hopefully intact. The process is involved and requires some careful typing. Ask a computer-savvy friend to assist you if you lack experience in this area. Note that it might reset your password to an earlier value.

1. Keep tapping F8 during the early boot phase.
2. Select Repair from the menu.
3. Log on as Administrator if prompted.
4. Select Repair when prompted.
5. Select the Command Prompt from the menu.
6. Identify the correct drive letter for your Windows installation. It could be a drive other than C: but it won't be X:. You do it by typing the commands and pressing Enter after each:
    dir C:\Win*
    dir D:\Win*
    dir E:\Win*
    until you find the Windows folder.
    I shall call that drive "Q:".
7. Type these commands and press Enter after each:
    cd  /d  Q:\windows\System32\config
    xcopy  *.*  Q:\MyRegBack\
    cd  RegBack
    dir
8. Examine the dates of the Software, System and SAM files. Were the files created before or after your problem became apparent?
9. If they were made before the problem became apparent then you can type these commands and press Enter after each:
    copy /y software  ..
    copy /y System  ..
    copy /y Sam ..
    (the two dots are part of each command)
10. Reboot normally. If things do not work out then you can backtrack by copying the original registry files from Q:\MyRegBack to Q:\Windows\System32\config.
