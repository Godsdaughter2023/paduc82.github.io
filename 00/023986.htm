januspu say:

Mailmerge

I have created a mailmerge document where a piece of information is used repeatedly, sometimes at the beginning of sentences. How can I get it to Capitalise in those instances only?

a. user say:

it depends on whether the situation is simple, or not so simple.

Simple = you know in advance which places you need to capitalise, and you know you will get the correct non-capitalisation if you either leave the text in your mailmerge data source alone, and the correct capitalisation if you capitalise the first letter of the first word (and there is one other SImple case**).

In that case, supposing your data source field is called mytext, you can use { MERGEFIELD mytext } for the uncapitalised version and { MERGEFIELD mytext \*Firstcap } for the capitalised version.

Not so simple = you do not know in advance where you do or do not need to capitalise, or to achieve the non-capitalised version you may need to *uncapitalise* the first letter of the first word.

e.g. "Simple" = (approximately) you know where all the sentences inyour mail merge main document end. So for any particular { MERGEFIELD mytext } you know whether it needs to be capitalisd or not. So you might have

...some text ending a sentence. { MERGEFIELD mytext \*Firstcap } is what you need here.

...some other text in the middle of a sentence, where you will need { MERGEFIELD mytext }

But if you are actually building your text from more than one field in the data source, e.g.

{ MERGEFIELD someotherfield }{ MERGEFIELD mytext }

then you may not know in advance whether you need to capitalise "mytext".

"Simple" = you do not have texts that are already capitalised (wholly or partially) and which contain words which should be capitalised, e.g. texts containing the English word "I" or people's names (or perhaps German texts where the nouns should all be capitalised).

If you have such texts, there is a difficulty, because for the uncapitalised version there is no "FirstUncap" switch, so you have to uncapitalise the whole text. So if your text could have

"It was in Canada"

for the uncapitalised version, to you would need something like

{ MERGEFIELD mytext \*Lower }

but that gives you "it was in canada", which you probably don't want.

In the not so simple case, the problem is that in order to get everything right, you would in fact have to have code (say VBA code) that could examine the entire text and get all the capitalisation completely right. Definitely not simple at all. IMO with mixed case texts the only way you could avoid that complication would be by ensuring the data source contains what you need to avoid the problem.

**The other simple case is where

{ MERGEFIELD mytext \*Lower } is always correct for the uncapitalised version and
{ QUOTE { MERGEFIELD mytext \*Lower } \*Firstcap } is always correct for the capitalised version, and you know which texts need to be uncapitalised/capitalised.
