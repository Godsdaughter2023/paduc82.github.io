Sub FindSymbols(FindChar As String, FindFont As String)

Dim FoundFont As String, OriginalRange As Range, strFound As Boolean
Application.ScreenUpdating = False

'set range to return to in case symbol not found
Set OriginalRange = Selection.Range

strFound = False
With Selection.Find
    .ClearFormatting
    .Text = FindChar
    .Replacement.Text = ""
    .Forward = True
    .Wrap = wdFindStop
    .Format = False
    .MatchCase = False
    .MatchWholeWord = False
    .MatchWildcards = False
    .MatchSoundsLike = False
    .MatchAllWordForms = False
  
    Do While .Execute    'Keep going until nothing found
        If Dialogs(wdDialogInsertSymbol).Font = FindFont Then
           'If the correct character was found, replace text
           strFound = True
            'OriginalRange.Select
            OriginalRange.Text = InputBox("Replace with: ", "Process", OriginalRange.Text)
            OriginalRange.Collapse Direction:=wdCollapseEnd
           Exit Do
        Else
           'Otherwise search again
           Selection.Collapse wdCollapseEnd
        End If
    Loop

    If Not strFound Then
        'if nothing found, search from the beginning of the document
        ActiveDocument.Range(0, 0).Select
        Do While .Execute
            If Dialogs(wdDialogInsertSymbol).Font = FindFont Then
                strFound = True
                Exit Do
            Else
                Selection.Collapse wdCollapseEnd
            End If
        Loop
    End If
 
End With

If Not strFound Then
    OriginalRange.Select
End If

Set OriginalRange = Nothing
Application.ScreenUpdating = True

End Sub

jay freedman say:
Verify that the certificate has actually been stored properly:
 
- Click Start > Run.
 
- Enter certmgr.msc and click OK.
 
- On the left side of the dialog, expand the Personal item and click Certificates.
 
- On the right side, look for the certificate. If it isn't there, the certificate wasn't properly placed in your computer's certificate store. You can try creating another one; the Office program
(selfcert.exe) should store it automatically.
 
Technically, the certificate created by selfcert.exe is for code signing, not for document signing, but you aren't prevented from using it that way. Of course, that certificate can be trusted only on
the computer where it was created, so it isn't much good for documents anyway. The intention is to let you sign macros that you write in the VBA editor, so the macros will run on your own computer
without requiring you to enable them manually each time.

steve rindsberg say:

John's correct.  The .Options object only has two properties in PPT.

 

You can verify the registry location by going to PowerPoint Options | Save | Save Presentations section | Default file location:

Plug in some unique value there, then have a look in the registry as John's mentioned.  Your unique value should appear there.

 

To have any effect, you'll have to make the registry changes before starting PPT.  If you change the reg while it's running, your changes will be ignored (while PPT is still running) then overwritten (when PPT quits).
