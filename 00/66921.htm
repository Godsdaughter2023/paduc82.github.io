ipoint.exe - No Disk - Microsoft Community

dwcol say:

ipoint.exe - No Disk

ipoint.exe - No Disk  \device\Harddisk3\DR3.
spl23 say: After 4 hours of tearing my hair out last night, I have finally found the cause and solution for this problem.

The Intellipoint mouse driver can have customised actions in different programs. To enable this, every time a program is run, the mouse driver makes a note in the registry of the path of the relevant EXE file. If a program is run from a removable disk (i.e. a card in a card reader), the path to the removable disk is added, and the mouse driver then searches that path when certain actions occur, including the startup of the mouse driver. If there is no disk in the relevant location, an error is generated. This is basically shoddy design, and also seems to be why pressing the secondary buttons has a noticeable lag before anything happens.

The fix is fairly easy as long as you are happy hacking the registry. Under \Software\Microsoft\Intellipoint\App Specific (sorry - can't remember if it is under HKCU or HKLM), there are a lot of folders with the names of exe files listed. The keys under each of these include the path to the exe - look through the list, and delete any folders for which the path isn't on a local drive.

Microsoft really need to fix this driver, because there is no reason for it to search through all paths like this, but until they do, this will fix the problem.
