analogrocks say:

In cases where AV conflicts and crashes occur, check likely and easiest to fix - probable - sources of problems first.

Disable "real-time" protection for Windows Defender by turning "off" that setting, found in "settings" for WD.

Or, disable and uninstall any other virus scanner - unless - you want that application to prevail. Pick one. (There is an exception to this procedure if you have another kind of scanner installed that doesn't run "real-time." Such a program rarely causes a problem; I use one as a "double-check" without incident.)

Such a potential conflict can occur in Windows 7, with the WD firewall installed along side of WSE, Windows Security Essentials. After WSE has been uninstalled, you are very likely to be unable to update Windows and/or Defender. My assumption would be MS's attempt to curtail the spread of malware on unprotected systems, thereby thwarting the update process via MS servers.

Now, reboot and see if you have been successful.

If not, consider this. It is possible that the update service is running prior to your attempt to update Windows or WD or WSE. This frequently happens "at boot", if a significant period has elapsed since the last update. In this case, wait as much as 30-45 minutes to see if that problem resolves. If your Internet connection is ADSL, for example, such a large update can require a fairly long time to download. A quick way to check what is happening is to look at CPU activity in Resource Monitor. Or, you can check the status of you Network Interface Card (NIC) to verify the number of packets received since logon. If that number is say, 20,000,000 or at similar scale for incoming packets, you can be reasonably certain that the system is likely to be downloading updates. More recent iterations of Windows are not particularly helpful in onscreen information regarding these behind-the-scenes processes. 

Failing these recommendations, then move on to treating for malware.

rajesh c. say:

There is no formula in Excel, which helps in naming/renaming a worksheet name.

Alternatively you will have to take help of a macro which can do this - but in  that case the

freedom of the team giving a name to the worksheet is lost.

 

However, there is a way out - but it is the reverse way.

 

Using the formula given below, you can fetch the worksheet name in a cell.

 

Copy this formula in (say) cell A1 and press ENTER.

The worksheet name (of that worksheet) will be fetched in the cell A1.

 

=MID(CELL("filename",A1),FIND("]",CELL("filename",A1))+1,255)<

 

If you do this for each of your team worksheet, then you can get the worksheet names in cell A1 of each worksheet, which, you can in turn use the way you like for automatic links.
