suzanne s. barnhill say:

Just as Doug suggested: You separate the chapters with section breaks. You can enable "Different First Page" for each section, which allows you to have a different header on the first page of the chapter (ordinarily, you would have no content in the First Page Header and a running head in the Header or Even/Odd Page Header on the remaining pages). If you unlink each header from the corresponding header in the previous section, you can insert different content, but, as Doug suggests, if you're wanting to pick up the chapter title in the running head, then, provided you use a specific style for the chapter titles, you can use a StyleRef field in the header that will change automatically.

OneDrive sync issues: affecting Office documents only - Microsoft Community

a. user say:

OneDrive sync issues: affecting Office documents only

Technical Level : Basic

Summary

Self-help steps to resolve OneDrive synchronization issues when issue affects only Office documents.

Microsoft Office is an Office suite of desktop applications such as Microsoft Word (.doc, .docx), Microsoft Excel (.xls, .xlsx), Microsoft PowerPoint (.ppt, .pptx, .pps, .ppsx) and others.


Details

If you are not able to either save Office documents in OneDrive folder, or open Office documents from OneDrive web or synced folder / app, or OneDrive creates duplicates of documents (e.g. Filename-PC-Name.docx), or any other issue in OneDrive where you see connection with Office Documents (basically all sync / save / open issues on Windows OS where only the Office documents are affected), please try these steps.

Steps for Windows

Delete Windows Credentials

First two steps are different for Windows 7 and 8:
  1. Windows 7
    1. Go to Start -> type Control Panel.
    2. Go to User Accounts and Family Safety -> Click on User Accounts.
  2. Windows 8
    1. Go to the Start Screen by pressing Windows key on your keyboard or on the tablet.
    2. Once on Start screen, type Credential Manager.
  3. In the left pane, click Manage your credentials
  4. Now look for the Live (Microsoft) account ID and password (for SkyDrive/OneDrive/Office) in the credential manager, remove it and restart the computer.

If you would like to know more about Credentials Manager, please review following articles:

  • What is Credential Manager?
  • Remove stored passwords, certificates, and other credentials

What credentials to delete?

The best solution is to delete all credentials, apart from:

  • virtualapp/didlogical

However, it will also work if you delete the specific sets of credentials (don't worry if some of these don't exist in your credentials store):

  • MS.Outlook.15:*** Email address is removed for privacy ***
  • MicrosoftAccount:*** Email address is removed for privacy ***
  • MicrosoftOffice15_Data:live:cid=XXXXXXX
  • MicrosoftOffice15_Data:orgid:*** Email address is removed for privacy ***
  • MicrosoftOffice15_Data:SSPI:*** Email address is removed for privacy ***

Rename csisyncclient.exe:

These steps should also fix OneDrive high CPU consumption issue (e.g. OneDrive.exe process uses 100% CPU):

  1. Open following location:
    • C:\Program Files\Microsoft Office 15\root\vfs\ProgramFilesCommonX86\Microsoft Shared\OFFICE15
  2. And rename csisyncclient.exe into csisyncclient_old.exe
  3. Reboot computer and see if it helps.

Note: csisyncclient.exe is the Microsoft Office Document Cache Sync Client Interface that belongs to Microsoft Office 2013 - the file which in one or another way affects synchronization between Office and OneDrive. This is one of the processes used by Office Upload Center.

Steps for MAC

If you have similar issues on MAC OS X, please try these steps:

  1. Shut down Office programs (Word, Excel, etc.)
  2. Open Keychain Access, located in the Utilities folder in the Applications folder.
  3. If keychains are hidden, choose View > Show Keychains.
  4. Select a keychain in the keychains list. Choose File > Delete Keychain (keychain name). Delete all Office and OneDrive related keychains.
  5. Restart MAC and try again.

Contact support

If none of the steps above help to resolve your issue, please contact OneDrive support by clicking this link. Please include as much information as possible in the very first contact; this will help to resolve your case quicker:

  • What operating system is affected?
  • Have you tried to perform all the troubleshooting steps available in this article? Please mention if you could not complete one or another troubleshooting step.
  • If it is 8.1, is OneDrive Sync Engine Host process running (see steps here)?
  • Does it affect another Microsoft account? (You will have to disconnect that account and re-connect with another, preferably new; see steps above)
  • Do you synchronize this account somewhere else? If yes, do you have similar issues on any of the other devices / computers?
  • Do you see error code or error text?
  • Does the sync issue affect all files or just some? Specific format, size?
  • Do you have any issues when using same account in OneDrive web version?
  • Do not hesitate to provide screenshots or even a screen capture file created by Problem steps recorder.
  • Other information that you find valuable.

Function COUNTIFCOLOR(CountRange As Range, CountColor As Range) As Long
Dim CountColorValue As Long
Dim rCell As Range

CountColorValue = CountColor.Interior.Color
For Each rCell In CountRange
If rCell.Interior.Color = CountColorValue Then
COUNTIFCOLOR = COUNTIFCOLOR + 1
End If
Next rCell
End Function

pucknstella say:

Hope your issue was resolved... I'm replying for others who encounter a similar problem. They key here is "the target principal name is incorrect"

I saw on other posts that it's a problem with POP and Outlook and users were recommended to delete the data file and use IMAP settings.

No! The problem is that the certificate is for one domain but the "target principal name" is for another domain.  This happens with email accounts that are setup on domains that are associated with a certain domain (usually via CPanel) that has a valid certificate but that the email domain that uses the same server does not. In other words, domain1.com has the certificate and is the server main account domain. Domain2 is the pop email account that uses the same server but does not have a certificate unto itself.

Just switch the pop outgoing/incoming domain to domain1 and your error messages will go away as Outlook will math pop service (domain1) to the certificate, and your email will work anyway, since domain2 uses domain1 as its server any way.

It's obvious once you realize it -- took me two weeks of annoying popup errors and wasting time reading forums advice to reset Outlook data files. I hope this helps someone else out there.
