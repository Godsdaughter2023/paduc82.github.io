How can I ignore overallocation in project past a certain - Microsoft Community

cole ferguson say:

How can I ignore overallocation in project past a certain deadline?

I want to use Microsoft Project to manage tasks and deadlines, but some days it might be necessary for someone to work overtime. This causes them to be overallocated on that project, which is fine: I want to be able to see that they've been overallocated.

What I don't like is that now and forever, that person is overallocated. For example, let's say that in a regular work week an employee is supposed to work 8 hours a day, but on Wednesday they work 15 hours. They're now overallocated, which is good: I can see that they've worked too many hours on that particular task.

However, let's say that for the next 5 months they work regular hours. In project, they're STILL overallocated despite the fact that the task they originally overallocated themselves on was resolved 5 months ago. How can I make it so that in the future they are not overallocated? Can I get overallocation to reset after a certain date, or after a task is completed?

john - project say:

Cole,

If you are looking solely at the overallocation indicator in the Info field (Gantt Chart or Task Usage views) or the fact the resource name is shown in red on the Resource Sheet or Resource Usage view, then no, you cannot "reset" that. However, if you look at the timescaled data on the Resource Usage view, it will clearly identify the specific period that resource was overallocated.

Probably not what you wanted to hear, but that's the way it works.

John

trevor rabey say:

You have correctly pointed out that it is not enough to know that a resource is overallocated, but that is a good starting point, but it is also necessary to know by how much, on which tasks and when.

Overallocation is overallocation, whether it is the future or in the past, and also whether the task is marked 100% complete or 0% complete or any % complete, so you can't "blame" the software for displaying it.

However, if you know how much of it there is and where the overallocation is, then you can choose to ignore it. If the task is entirely in the past and is 100% complete, which of course it should be, then the "red man" indicator will not be displayed in the indicators column, because the 100% tick is there instead.

I find the resource graph view very handy to dig a bit deeper and find out when overallocation is happening.

To do what you want to do, you would have to be able to have max units vary by task or different at different times, which isn't an available setting.
