.
The important thing to remember is that, underneath it all, what you are
duplicating is data.  Not forms or subforms, but the records that they
display.  So you can use an append query to generate a "duplicate" record in
Cutlist, and another append query to duplicate all the related records in
CutlistDetail.  We're going to need to know the names of all the fields in
each of these tables (unless we want to write much more complex code than we
have to for this one problem).  We also need to know the name(s) and type(s)
of the primary key field(s) in each table, and also the name of the field in
CutlistDetail that links it to Cutlist.
 
To make your duplicates, you are going to have to duplicate the parent (main
form) record first, and then the child (subform) records.  The trick here is
that you can't duplicate the parent form's primary key;  you have to
generate a new one for the duplicate record, and then use that newly
generated key in the foreign key (linking) field of the records we append to
CutlistDetail.  If the primary key field in Cutlist is an autonumber, we can
do that easily by creating the parent record, capturing the new record's
autonumber value, and then using that value for the foreign key(s) of the
child records we create.  If the parent record's primary key is not an
autonumber, though, we need some rule for creating a new key value.
