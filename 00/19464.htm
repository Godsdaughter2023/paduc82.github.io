evanus say:

Get the location by using Bing Map (or HERE map) in windowsphone 8.1 devices from the latitude and longitude value

Hi all

Actually some days ago I had a problem in my Lumia 535 windows phone maps (both Bing Map or HERE Map). The problem was I couldn't get the location while entering the latitude and longitude value (GPS Coordinate) as I did in the Bing Map desktop version previously. i.e There was no result while I enter -7.358440,109.903999 (this location is Indonesia) in my lumia 535, but I could get the result in the desktop version.

After searching in Microsoft community, I found this link :

http://answers.microsoft.com/en-us/winphone/forum/wp8-wpapps/longitude-latitude-in-maps-windows-phone-81/9ee51e88-436c-4770-9341-deafedcbeef7

with similar issue. Now my issue was solved, and I want to share with him/her, but unfortunately I can't reply it, since the thread was locked, so I start this discussion forum to give my opinion ( I hope he/she can read this too ).

Now my problem was solved (actually this is my friend suggestion, and it work, while the answer at that forum can't work for me).

Here are the steps to get the location from windows phone Bing Map (or HERE Map) :

1. Make sure you have download the appropriate map

2. Turn off the internet connection (cellular data or wifi must be turned off) or force the map to look for the location in its downloaded map. You can't get the result while the internet connection is on

3. Instead of entering -7.358440,109.903999 as in the desktop version, you have to enter : -7.358440,109.903999 as in the desktop version, you have to enter : -7.358440N,109.903999E

4. After you get the location, you can turn the internet connection on again

Note : since you can only search offline, you can only get the location while you enter the latitude and longitude if they are inside the downloaded map.Hope this can help other people that find similar issue with me.

Regards

evanus

    Regards,

    j.e. mcgimpsey say: Further to Bob's reply:

    XL stores date-times as numbers. 

    The integer part (to the left of the decimal separator) is interpreted as the number of whole days since the base date (either 31 Dec 1899 for the 1900 date system, or 1/1/1904 for the 1904 date system).

    The fractional part (to the right of the decimal separator) is the number of fractional days. So 0.25 = 06:00 (6;00 am), 0.75 = 18:00 (6:00 pm), etc. Times entered without dates (i.e., the number stored < 1) can be interpreted as stand-alone times, or as times on the base date.

    Of course, you can format date-times as dates, times, or any other numeric format.

    You can enter date-times in any non-ambiguous way consistent with your system date-time settings, e.g.:

       18 April 2012 4:00 pm

       4/18 16:00

       =TODAY() + 2/3

    will all display the same when formatted identically (for today only, in the third case).

    For now, here's what we're stuck with:

    You need two things for a slide number -- a placeholder, and to populate the placeholder.

    PowerPoint messes up the whole structure -- or seems to -- when you open 2003 and prior files in 2007 and 2010 (and 2013 and 2016). They really mucked stuff up! It's partly because the whole masters paradigm is different now, but they still blew it. (Trust me, I've complained mightily to the PPT developers about it. I will continue to do so.)

    Here's the most straightforward way to fix them that I know.

    Go to the slide master (the big thumbnail), select any footer / slide number boxes and delete them. Now look at the slide layouts (little thumbnails). Delete any footers / date / slide number boxes on each layout.

    You shouldn't have to do this part, I know. But because it's all messed up, this is the most surefire way I know to correct the issues.

    Now go back to the Slide Master (big thumbnail), click Master Layout on the Ribbon and tick the boxes for date, slide number, and footer. Format and position the placeholders as desired.

    Now select each slide layout and tick the Footers box on the Ribbon.

    Then close Master View.

    Now go to Insert | Header and Footer (or Insert | Slide Number is fine, too) and tick the box there for slide number. The slide number placeholder you created should be populated with the slide number.

    The key here is having the placeholder on all the slide layouts. When you add the placeholders to the slide master (big thumbnail), the associated slide layouts (little thumbnail) should inherit them. But part of the problem is that any "leftover" layouts from PPT 2003 don't always inherit the footer (and date and slide number) placeholders properly.

    And there's other weird stuff that goes on.

    The other thing is, if you paste or insert other existing slides, you may need to do this all over again. That's because you may end up with additional slide masters or layouts after pasting in slides. So I recommend you do it after you've added all the slides you're going to add.

    Hopefully this will get you going.
