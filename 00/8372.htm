Solution file @ https://1drv.ms/x/s!Akd5y6ruJhvhmyiQ0aFOQ9yF42Qg

1. Make a backup of your workbook.
2. Open your workbook and ALT+F11
3. Locate your Workbook name in Project Explorer Window
4. Right click on your workbook name > Insert > Module
5. Copy paste the Macro code given
6. Go back to your Workbook and ALT+F8 to display Macro Window
7. Run your Macro from here
8. Delete you Macro if the Macro was needed to be run only once.
9. Otherwise save your file as .xlsm if you intend to reuse Macro again. 

Sub InsertRows()
    Dim i As Long, Lr As Long
    Dim Ws As Worksheet
    
    Set Ws = ActiveSheet
    
    Lr = Ws.Range("A" & Rows.Count).End(xlUp).Row
    
    For i = Lr To 3 Step -1
        If Ws.Cells(i - 1, 1) <> Ws.Cells(i, 1) Then
            Ws.Cells(i, 1).EntireRow.Insert
        End If
    Next i
End Sub

Solution: Excel OART.DLL crashes when trying to format chart axis - Microsoft Community

robert stroess say:

Solution: Excel OART.DLL crashes when trying to format chart axis

FYI: there are quite a lot of queries on the internet about Excel crashes (also reported by the Event Viewer as caused by OART.DLL) when trying to format a chart's axis e.g. via context menu.

Here's a solution that worked for me (in my case: Excel2010 x64 on Windows10 Home x64 with a spreadsheet containing a xy scatter chart and x axis representing time/date values):

Ensure that Windows => PC Settings => Date & Time => Region => Languages **AND** Windows => Control Panel => Region => Formats **AND** Excel => Options => Language have consistent non-contradictory values when you have language packs installed. Ensure same uniqueness for time/date default formats.

When adapting any values to a consistent state, it's probably best to reboot Windows / restart Excel.

As I've spent hours in searching for a solution to those CDTs, and as Internet searches have not returned a functional fix, I hope this hint will help others searching in vain as I did before troubled by the same problem.
