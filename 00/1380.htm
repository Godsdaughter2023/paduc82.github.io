don cuthbert say:

When an upgrade to a new build fails, these steps may enable it to succeed:

  1. Completely uninstall any non-Microsoft anti-virus software.
  2. Perform a clean boot before upgrading.
  3. Disconnect any un-needed peripherals, such as devices connected via USB.
  4. Disable all network adapters after the build has downloaded but before it Restarts, or before running setup.exe, if you are upgrading via a mounted ISO or Windows 10 install media.

_________________________________________________________________________________

When upgrades fail, a clean install of the new build might work.  You can never be sure what issues the new build might inherit from previous ones.

Once Windows Update has downloaded the ESD file for a new build, you have everything that you need to clean install the new build to a small partition, to see if a clean install works on your hardware (as long as you have some room).  You can do this without formatting your drive and can easily delete it if you want afterwards.  If you are very short of space, this ESD file (in a hidden folder at C:\$WINDOWS.~BT\Sources\Install.esd) may get deleted after the PC restarts, so it's good to copy it somewhere to preserve it.  64GB is enough room, but the partition for the clean install could be as small as 20GB for 64-bit Windows 10.

If you shrink C: by 64GB with Disk Management and create a new partition you can use a DISM command to copy Windows to it and bcdboot to enable your PC to boot to it.

 

If we assume your new partition is F:, use a Command Prompt (Admin):

DISM /Apply-Image /ImageFile:C:\$Windows.~BT\Sources\Install.esd /Index:4 /ApplyDir:F:\ /Verify

When you restart, the latest build will proceed to clean install to the new partition that you created.
