Linking Cells Based on Cell Color - Microsoft Community

lainy13 say:

Linking Cells Based on Cell Color

I want to link certain cells to another workbook based on cell color. To be more specific, I have one document that has the different items I've spent money on color coded by category (grocery = red, clothing = green, etc.) In another document I have the sum for each month to sort the budget. I plan on adding things that I've missed. I want to be able to have the sum of the red cells on another sheet, so when I turn another cell red it will add to that total.

tasos_k say:

Hi,

sample:

colors and values in column A from row 2 up to last row

pic1

results in columns C-D

pic2

vba macro

Sum,

randomly colored cells (not conditional Format)

Sub Sum_Colored_Cells()

'### 02 Mar. 2019 oAnast ###

Const sCol As String = "C" '<< results in col C-D

Dim ws As Worksheet

Set ws = Sheets("Sheet1") '<< data in Sheet1

Dim L1 As Long, L2 As Long, t As Long, x As Long

Dim s As Double

L1 = ws.Cells(Rows.Count, "A").End(xlUp).Row

Dim v As Variant

v = Array("Colors", "Sum")

Application.ScreenUpdating = False

With ws.Cells(1, sCol).Resize(L1, 2)

.Interior.ColorIndex = xlNone

.ClearContents

End With

ws.Cells(1, sCol).Resize(, 2) = v

For t = 2 To L1

Cells(t, sCol).Value = ws.Cells(t, "A").Interior.Color

Next

Columns(sCol).RemoveDuplicates Columns:=1, Header:=xlNo

L2 = Cells(Rows.Count, sCol).End(xlUp).Row

ws.Cells(1, sCol).Resize(L2).Sort Key1:=ws.Cells(1, sCol), Order1:=xlDescending, Header:=xlYes

For x = 2 To L2

Cells(x, sCol).Interior.Color = Cells(x, sCol).Value

Cells(x, sCol).ClearContents

Next

For x = 2 To L2

s = 0

For t = 2 To L1

If ws.Cells(t, "A").Interior.Color = ws.Cells(x, sCol).Interior.Color Then

s = s + ws.Cells(t, "A").Value

End If

Next

ws.Cells(x, sCol).Offset(, 1).Value = s

Next

Application.ScreenUpdating = True

End Sub

john p spencer say:

Yes, the linking makes a big difference.

You need to base the report on a query that includes the current table and the table that contains the driver's name. 

  • Add both tables to a query
  • Drag from the driver field in your current table to the driver id field in the second table.
  • Select the fields you want in your report from your current table AND the driver's name field from the second table
  • Change your report's source to this query
  • In the report design view, select View: Sorting and Grouping from the menu and set the sorting by selecting the relevant fields (DriversName) to sort on.
