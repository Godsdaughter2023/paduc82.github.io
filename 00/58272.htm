frederik long say:

If the problem is caused by some bad registry data then restoring the registry files manually to an earlier point would fix the problem. The process is involved and requires some careful typing. Note that it might reset your password to an earlier value.
1. Boot the machine into Repair Mode.

2..4 (not relevant)

5. Select the Command Prompt from the menu.
6. Identify the correct drive letter for your Windows installation. It could be a drive other than C: but it won't be X:. You do it by typing the commands
    dir C:\Win*
    dir D:\Win*
    dir E:\Win*
    until you find the Windows folder.
    I shall call that drive "Q:".
7. Type these commands:
    cd  /d  Q:\windows\System32\config
    xcopy  *.*  Q:\MyRegBack\
    cd  RegBack
    dir
8. Examine the dates of the Software, System and SAM files. Were the files created before or after your problem became apparent?
9. If they were made before the problem became apparent then you can type these commands:
    copy /y software  ..
    copy /y System  ..
    copy /y Sam ..
    (the two dots are part of each command)
10. Reboot normally. If things do not work out then you can backtrack by copying the original registry files from Q:\MyRegBack to Q:\Windows\System32\config.

josé oliver-didier say:

Hi Donebb,

I have not used the apps you mentioned. Not sure they may work in this case.

Good thing about Windows Photo Gallery is that it stores much of the metadata back to the image files themselves, using industry accepted standards such as EXIF, and XMP. Captions, Ratings, Tags (Descriptive, People and Geotags). I say "much" because in certain situations information may not be written back to the file - for example if the file is read-only (quite rare), a format which does not support embedded metadata (such as PNG), or certain web service derived geotags. Anyways, for the most part you should be fine if you opt for the nuclear option.

It is important you ensure you have a complete backup of your photos in a separate media.

I am quite a fan for embedded metadata for the purpose of data preservation and is also one of the reasons which I like Windows Photo Gallery. I wrote a post (a bit technical) on how the information is stored - Accessing Windows Photo Gallery Metadata using Exiftool

[quote user="BengelAal174"]Jetzt meine Frage ist es für den Support möglich meine Email von der Xbox Live ID zu entknüpfen ohne das ich die 30 (Sicherheitszeit) warten muss ?[/quote]

Das kommt darauf an. Die 30 Tage Wartezeit waren als Sicherheitsmerkmal eingeführt worden, da in der Vergangenheit Account-Diebe recht schnell einen gestohlenen Account auf eine andere E-Mail umgelenkt haben, und dann schnell DLCs und FIFA Punkte mit dem gestohlenen Account gekauft haben, ohne dass der Ursprüngliche Besitzer Zeit hatte, den Diebstahl zu melden.

Daher ist eben seit ein paar Jahren eine Wartezeit von 30 Tagen vorgesehen, bevor ein Gamertag auf eine E-Mail umgezogen werden kann, die schon einmal mit einem Gamertag verknüpft war. So hat dann der rechtmäßige Inhaber Zeit, den Accountdiebstahl zu melden, und Xbox kann das Konto einfrieren, bevor es auf ein anderes Konto umgezogen wird.

Da einige Bösewichte es wohl auch geschafft haben, Support Personal mit Persönlichen Infos (z.B. Geburtsdatum und Wohnort von Facebook erfahren) überzeugen konnten, angelich rechtmäßiger Inhaber des Kontos zu sein, kann es sein, dass die Support Person wirklich nicht in der Lage war, Dir zu helfen, da Du nicht eindeutig beweisen konntest, Inhaber des Kontos zu sein.

Wenn Du dem Supporter eindeutig beweisen kannst, dass Du rechtmäßiger Inhaber der beiden Konten bist (beispielsweise dadurch, dass Du eine Kontoverbindung oder Kreditkarte dort hinterlegt hast), könnte der Supporter vielleicht helfen und vor Ablauf der 30 Tage den Umzug machen. Kommt eben darauf an, wie Microsoft die Sicherheitsvorschriften dafür intern weiter geändert hat.
