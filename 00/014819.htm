karl timmermans [outlook mvp 2012-2018] say:

I am unable to import contacts from a >csv file to outlook on my laptop as on completing the process I get an error message that the service provider does not support it. Any suggestions?

Means you're trying to import your contacts to a contact folder that's part of an Outlook.com account configured using Exchange ActiveSync - a limitation of the EAS protocol.

You have 2 choices

1) create a new contact folder outside of the Outlook.com folder tree (either in another existing data store or create a new PST file and a new contact folder in that newly created PST file

2) import your contacts into that contact folder 

3) when done, open the contact folder with the newly imported contacts and then --> Ctrl-A to highlight all the contacts --> right-click --> Move the contact items to the Outlook.com contact folder

OR

4) import your contacts directly into the Outlook.com account via webmail (browser access). If you follow this route, the field(column) names must be defined precisely to what the import is expecting

First, you don't need that link nor code nor article (that is just a wild goose and a big waste of your time).
 
I not aware that this has changed in 2007, or even 2010.
 
Just drop in a extra text box into the detail section of the report. Lets call the text box zoo1
 
For the data source of this text box, set it to the column that you want to total. Say AccountBalance.
 
Then, just set the running sum on this text box to over all, and then set the visible = false. Then place a text box in the page footer such as:
 
     =([Zoo1])
 
You now have the total for each page.

Do the above for net copies column. It it optional if  you want this column set a visible to display (you likely don't)
 
The groupings, and report total supports use of sum() and count(), but a page total does not (however the above suggestion works).
 
For the page number, just use
 
       ="Page " & [Page] & " of " & [Pages]
 
as the source of the text box in the page footer.
 
You don't need code here.
 
I am not aware of ANY changes as to how this works from previous versions.
 
However, do keep in mind for these totals (=(sum([Detail text box to sum])), you have to use the name of the text box that is placed on the detail section that is bound to the column. You can not use the name of the column.
