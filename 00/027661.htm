albert d. kallal say:

You can't cancel the after update event (it is too late).  Think of this much like the form update events. The forms after update event is also too late.

 

However, canceling the update in the before update does work. So your table validation code just like in a form goes in the before update event which has a cancel.  The same idea applies to a table event. So canceling updates only works in the before change table event.

 

So, do your checking and simply raise an error in the before change table event.

Note when you raise an error in the table event you will see that error message.  That error is trappable if your in a form.

nathan o. - microsoft support say:

Hi Brian,

 

Bre-x is correct.  This problem can usually be fixed by adding a timestamp field to your SQL table (or in your case PostgreSQL) and then after making this change, relink the tables within your Access database.  The problem tends to happen when one of the following is true:

 

Access is creating Null bit fields, which causes a type mismatch.

or

The linked table is using fields defined as floating point data type (i.e. Real). This data type allows for storing "floating point" number data. When the edited record is passed to SQL Server, if both engines see the data in these fields as having the identical value, there is no problem. However, because of the "rounding" algorithms used by JET and SQL Server, the actual value compared may be different. Since JET has to review each field and see if there are any concurrency issues, when JET compares the number it sees (in the "Real" data type field) to that value stored in SQL Server, if it does not match exactly the record is assumed to have been changed and a Write Conflict occurs.

 

 

This problem is outlined in KB 280730 and can also be found by doing a quick BING search with keywords Access Write Conflict J

 

280730 You may receive write conflict errors when you update records of a linked SQL Server table in Access 2000
http://support.microsoft.com/default.aspx?scid=kb;EN-US;280730

hansv mvp say:

It seems that YEARFRAC is only intended for dates that are at most one year apart.

With dates in A1 and B1, the following formula may be more in line with what you want:

=INT(YEARFRAC(A1,B1,1))+YEARFRAC(DATE(YEAR(A1)+INT(YEARFRAC(A1,B1,1)),MONTH(A1),DAY(A1)),B1,1)

amit tandon say:

You may try below formula:

=DATEDIF(A1,B1,"Y")+(B1-DATE(YEAR(B1)-(B1<DATE(YEAR(B1),MONTH(A1),DAY(A1))),MONTH(A1),DAY(A1)))/MAX(365,DATE(YEAR(B1)+1,1,1)-DATE(YEAR(B1),1,1)-AND(B1<DATE(YEAR(B1),2,29),YEAR(B1)<>YEAR(A1)))

For a leap year, if the B1 date is >= 29 Feb, the fraction will be calculated taking into account 366 days, and if the date <= 28 Feb, 365 days will be considered for the year to calculate fraction, unless its the same year in A1 and B1 in which case the leap year determines 366.

For the dates: Feb 27, 1959 and Feb 28, 2016, fraction will be =1/365 so that the formula returns 57.00273973, and

For the dates: Feb 28, 1959 and Feb 29, 2016, fraction will be =1/366 so that the returned value is 57.00273224;

For the dates: Feb 27, 2016 and Feb 28, 2016, fraction will be =1/366 so that the returned value is 0.00273224 because the year is the same and its a leap year even though B1 date <= 28 Feb,

and

For the dates: Feb 27, 2015 and Feb 28, 2015, fraction will be =1/365 so that the returned value is 0.002739726 because the year is the same but not a leap year;

In case the above is logical.

hansv mvp say:

INT(YEARFRAC(A1,B1,1)) is the whole number of years between A1 and B1.

For example, if A1 contains September 27, 1959 and B1 contains October 28, 2016, INT(YEARFRAC(A1,B1,1)) = 57.

DATE(YEAR(A1)+INT(YEARFRAC(A1,B1,1)),MONTH(A1),DAY(A1)) adds that number of years to the date in A1.

In the example, this would be September 27, 2016. This date is guaranteed to be less than one year before the date in B1.

YEARFRAC(DATE(YEAR(A1)+INT(YEARFRAC(A1,B1,1)),MONTH(A1),DAY(A1)),B1,1) calculates the year fraction between the new date and that in B2. Since these are less than a year apart, the calculation will be correct.

Finally, we add the whole number and the fraction.

Let's take the example A1 = September 27, 1959 and B1 = October 28, 2016 again.

Step 1: calculate the number of years between the dates. This is 57 plus a fraction of a year.
