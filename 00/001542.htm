What matters for the TOC is not the font formatting but the paragraph Style. If you over-ride a Style's formatting, that doesn't change the fact the paragraph still has the original Style definition. Access has its own meaning for the common terms "form" and "report".

An Access Form object doesn't itself have "pages", it's a tool for interacting with data in one or more Tables; you would typically have a Form for the "one" side table with one or more Subforms for the related "Many" side tables. john w. vinson say:

Scott's absolutely correct.

Paper forms and relational databases are both excellent and well-developed tools for managing data. But they are deeply and intrinsically different! A well-designed paper form is almost sure to be an utterly wretched blueprint for a database, and vice versa.

The database - properly constructed! - will end up being much easier to use, since there will be far less redundant entry of (e.g.) names and addresses, and the computer will be able to help the user just select an option from a list rather than having to type in data.

"New Is Evil" of course, but you really need to make the effort to get away from this century-old technology and take advantage of the benefits a database can provide.

suzanne s. barnhill say:

Assuming that Heading 1 is what is being used for "Chapter One," then the problem the original user probably encountered was that the style was not actually numbered; that is, "Chapter One" is being typed as plain text. The trick is to make this autonumbering instead. Here's what you have to do:

  1. Multilevel List gallery, select the list currently being used (or, perhaps better, the one that shows 1 Heading 1, 1.1 Heading 2, 1.1.1 Heading 3 since it already has the levels linked to the appropriate heading styles.
  2. Click Define New Multilevel List (confusingly, this is the command you have to use to modify an existing list as well as create a new one).
  3. In the Define New Multilevel List dialog, click More to expand the dialog.
  4. Make sure that the levels are linked to the heading styles (if you chose a list that didn't show the heading styles).
  5. At Level 1, in the "Number style" dropdown, choose One, Two, Three.
  6. Before "One" in the "Enter formatting" box, type "Chapter."
  7. Select Level 2, and check the box for "Legal style numbering" so that the previous level is displayed as "1" instead of "One."
  8. Repeat #7 for each lower level.

Looking again at your example, however, it appears that "Chapter One" is just dummy text and that that level is actually numbered with 1, 2, 3 numbering. If that's the case, then all you need to do is make sure you have a multilevel list applied in which each level is linked to the appropriate heading level and each lower level is set to restart after a higher one.
