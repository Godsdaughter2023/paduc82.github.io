john - project say:

Patrik,

To answer your question, yes, that functionality is available.

First, you should set a baseline as soon as your plan is ready for execution but before any actual work has been done.

In your example, let's say the task was scheduled to start on Monday, April 17 with 40h work. That would give it a finish date of Friday, April 21. Now, set the Status Date to Thursday, April 20 and then go to Project > Status group > Update Project. Since the task has not been started as of the status date, check the option to "reschedule uncompleted work to start after". That will shift the task to start on the status date with a new finish of Wednesday, April 26, assuming the default Standard calendar.

If you look at the Tracking Gantt view, you will see the task delay compared to the original plan (i.e. baseline).

Hope this helps.

John

The idea to add an automatic update is really not a good one. Although it may seem "neat", it makes the invalid assumption that everything is on track and in the real world that just isn't the norm.

And as a followup, this was on the Windows Mobile Help page:

Synchronization Error 85010017

In some cases, ActiveSync will fail to synchronize data with Microsoft Outlook with Synchronization Error code 85010017. This could be due to the presence of large volume of items on first sync or specific large items such as a calendar item with many meeting attendees. Some targeted instances of this have been fixed in Activesync 4.5.

If you are experiencing this on Windows 2000 or Windows XP SP1, please upgrade to Activesync 4.5. Otherwise you can try archiving your inbox to reduce the number of items in the mailbox for synchronization
