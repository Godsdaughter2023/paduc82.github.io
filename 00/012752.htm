For #1 & #2, yes, we are able to use content types to set up document templates. Content types can be created at each site level, so we set up multiple sites or libraries for different departments to work with, and have different templates.

As Office 365 is cloud-based services, the template would be stored in the online services, while we can still use the local Office or Office Online to work on them, and please note that some theme or formatting may not be available to Office Online.

Below is the general steps for adding content types:

 1. Unable Content Type management for a library.

  Go to the Library > LIBRARY ribbon, Library Settings > under General Settings, Advanced settings > set Allow management of content types? to Yes > OK.

 2. Create the new Content Type for the document templates

  a. Go to Site settings > under Web Designer Galleries, Site content types > Create.
       > Fill in the Name
       > Select parent content type from: Document Content Types
       > Parent Content Type: Document
     > OK.

  b. On the next Site Content Type page, under Settings, Advanced Settings > Document Template section, select Upload a new document template > upload a blank Word Document.docx > OK.

  c. Go back to the Library Settings, under Content Types section, Add from existing site content types > find the newly-created document content type to add > OK.

The template will be available at FILE ribbon > New Document.


#3, SharePoint Online has the defualt columns in document library. For your references:

 Name column for the file name;
 Created column for displaying the time when the item is created or uploaded;
 Created By column for showing who created the column.

#4, Macro scripting and Quick Parts continue to be available to the desktop version of Office provided by valid Office 365 subscriptions.

In Excel Online, we would be only able to open an VBA-enabled Excel files without removing or corrupting the VBA, though.

angel in need of help say:

Printing a #9 envelope on the HP Officejet Pro 8600 using Word 2010. Loads center - Prints Right

I set the envelope up using the mailing wizard in Word 2010.  The problem comes when the icon that it defaults to for printing is a right load, but the only way that envelopes can be loaded is in the center of the tray.  I have an HP Colorlaserjet 3600 and an HP Officejet Pro 8600 and both load this way.  I did not have this problem before Windows 7.  I have printed thousands of envelopes during my lifetime.  When I send an 8.5 X 11 sheet through rather than an envelope it prints to the right on the Officejet. 

The second problem that I have encountered is that I can select a #9 envelope in the wizard to set up envelopes, but when I go to print the #9 envelope it is not available in the print dialog and then I have to design a custom envelope in order to even get the printer to send the envelope through.  Many times it does not like the custom envelope.  If I set up a #10 envelope with the changed margins the printer tells me that I have a mismatch between what I am submitting as an envelope and what it is supposed to print and refuses to print.  This is such a frustrating problem to me who needs to print many envelopes every week.  I had a laserjet 1300 which use to print envelopes in a breeze, and now it won't do it either.  I like some others that I have seen comment about this do not understand why this problem can not be corrected.  Is it an HP problem, or a Microsoft problem????

angel in need of help say: I think that I just solved my own problem after much reading in this forum.  Here is what I did.  I changed the default print icon back to center, rather than right which is what comes up for this printer.  Then for the Pages I entered the 0 s1 comand.  I set the printer to print on one side, left collated on, and placed it in landscape orientation.  The next is the kicker, because I have tried this before but the envelope is set to the 10, but I did go in to the custom set up and provided the dimensions for this envelope and clicked OK.  The margins I set 0.65 for the left, and the top 0.5.  I tried 0.25, but that put the return address too close to the top of the envelope.  I just printed off 30 envelopes during the time I was composing this reply.  This had been a 2 year problem for me, and I feel so relieved that it is solved.  I would not have solved it without all of the wonderful replies that I read.  Thank you.
