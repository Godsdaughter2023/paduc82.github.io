If you are referring to the #VALUE errors in D18:Q18, the "right" correction depends on what you want when B18="".  That's entirely up to you.  Perhaps the following:

=IF($B18="","",IF(AND(D$4>=($B18-6),D$4<=$C18),"X",""))

That assumes that C18 is either empty or a constant, not a formula that might return the null string ("").  In the uploaded file, C18 is empty.  If C18 might be a formula that returns the null string when B18 does not return the null string, it would be prudent to write the following:

=IF(OR($B18="",$C18=""),"",IF(AND(D$4>=($B18-6),D$4<=$C18),"X",""))

The following would also work:

=IF(AND($B18<>"",$C18<>"",D$4>=N($B18)-6,D$4<=N($C18)),"X","")

The N() function returns zero when its parameter is a string (like the null string).  It is needed to avoid #VALUE errors even though one or both of the first two conditions (B18<>"" and C18<>"") is false because Excel evaluates all AND() function parameters, even if a parameter on the left is false.  In other words, it is not a left-to-right evaluation, unlike the AND operator of some computer languages.
