It is never a good idea to try to backup any file you actively have open, especially an Access database file.

Now, for the back-end, you could create code to ensure all the database objects are closed, ensuring it is not being used, and then create a copy, but the front-end is another matter as you would be in it.

Your best bet is to create a VBScript to launch your database and automatically back it up prior to actually launching it.

Another approach would be to create a VBScript and use a scheduled task to launch it on a planned basis.

In all cases, you should always ensure no lock file is present prior to performing the backup.

That all said, backing up the front-end does not need to be performed routinely (except when performing development work).  The critical piece is the back-end since it actually changes with use.

jllatham say:

What is the formula?

Typically, #VALUE! comes up when you try to perform math with one or more of the values being text.  A gross example might be:

= 3 + "jlatham"

or = A1 + A2

where A1 contains a 3 and A2 contains "jlatham".

So it might be that one or more of the values in cells referenced in your formula has been converted to text in the transition to the Android platform.  You may be able to deal with it by modifying the formula.

Let's say that it was a simple formula  = A1 + A2

and both cells should contain numbers, and when you look at them, they look like numbers, but you still get the #VALUE! error.  You might try changing the formula to something like this:

=(A1*1) + (A2*1)

That would (normally) get Excel to coerce a text numeric entry to actually become a number and work in the formula properly.

john w. vinson say:

I'm wondering if it's the (unwise) use of a hyphen in the table name. It's best to use just alphanumeric characters (no blanks, no punctuation) in table and field names; Access lets you use special characters but it can get confused!

 

If you can't (or won't) change the table name, try always using square brackets:

 

=DLookUp("[DFA Lates]","[score card – dfa table]","[primary acct]=" & [unique id])

 

As noted in the thread, if the datatype of Primary Acct is Text you must include either ' or " as delimiters:

 

=DLookUp("[DFA Lates]","[score card – dfa table]","[primary acct]='" & [unique id] & "'")

marshall barton say:

The issue is if primary acct is a Text or a number field in its table.  If it's a Text gield in the table, then use quotes as I demonstrated.  If it's a number data type in it's table then look at the spelling.
