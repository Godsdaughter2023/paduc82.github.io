Hi,

If you are not connected to the internet, or a web page is not reachable, IE (and other web browsers) will default to the about:blank page.

A web page may not be reachable if,

connection speeds may be slow (especially if you are using a mobile connection), or if you are using a public access wifi hotspot where you have exceeded your data or time limits.

or

there are server or dns errors.

I visited https://masterseo.ca/ and received a 403 error... indicating that there are possibly dns or htaccess errors on that host.

If you are the webmaster of this site please log in to Cpanel and check the Error Logs. You will find the exact reason for this error there.

Common reasons for this error are:

  • Incorrect file/directory permissions: Below 644.

    In order files to be read by the webserver, their permissions have to be equal or above 644. You can update file permissions with a FTP client or through cPanel's File Manager.

  • Restrictive Apache directives inside .htaccess file.

    There are two Apache directives which can cause this error - 'Deny from' and 'Options -Indexes'.

If you are the website owner/developer you should contact your hosting provider or a technical support forum for assistance with configuring your sites htaccess and dns records.

If you have a home user (no technical) consider changing your homepage setting in IE to always use about:blank. This will shorten the time that your browsers take to start up considerably.

IE has security zones... ensure that you are using the default IE security zone setting.

Tools>Internet Options>Security tab, click "Reset all zones to default".

Ensure that you have not placed the host in your trusted sites list or unchecked the option to use the secure protocol (https). The File>Properties menu in IE will tell you which IE security zone a host has been mapped to... Generally for developers or home users, you should not have a need to place websites in your IE Trusted sites list. (Internet zone has a medium high setting, while the Trusted zone has a Medium default setting)… That is the Trusted zone is LESS secure that the Internet zone.... Public access websites are designed to work with the default IE security zone settings of the Internet zone.

ken sheridan say: You could try calling the Windows API to check if the NumLock key is currently turned off.  Add this little module to the database, or incorporate the code in any existing module you have which calls the API:

Option Compare Database
Option Explicit

Public Declare Function GetKeyState Lib "user32" (ByVal nVirtKey As Long) As Integer
Public Const VK_NUMLOCK = &H90

Public Sub TurnOnNumLock()

    If GetKeyState(VK_NUMLOCK) <> 1 Then
        SendKeys "{Numlock}", True
    End If

End Sub

You then just have to put the line:

    TurnOnNumLock

at the end of the procedure before the End Sub line instead of conditionally calling the SendKeys statement within the If….End If constructs.  If NumLock is turned on it won't do anything. if it's turned off it will turn it back on again.

rohn007 say:

Yes, the "refresh" option does act as described.  It resets or refreshes Windows, it does not delete "Metro" applications or your personal data files but it DOES delete all "traditional" desktop applications like all versions of Office.

You must install all of the missing applications. That means you need the installation file/disk and the associated product key / registration number

In Win 8/8.1, but no longer in Win 10, there is a way to preserve desktop applications during a refresh. What you need to do is install the desktop application(s) (like Office), apply all required updates and do your personal customizations.  Once the application(s) is completely setup you run the RECIMGE.EXE tool.  This generates a new "image" file which is used by Refresh.  Unfortunately MS "new & improved" this refresh tool out of Windows 10 ... <sigh>.

a. user say:

Hi enraged_user,

 

I apologize for the delayed response. I will be happy to inform you why emails got lost or deleted from a Microsoft account. The possible reasons are as follows:

 

1. Emails got deleted due to Inactivity.

When users don't sign in for more than 270 days or within the first 10 days after signing up for an account, the system will flag it as Inactive. After an account becomes inactive, all messages, folders, and contacts are deleted. Incoming messages will be sent back to the sender as undeliverable.

 

2. When an account got compromised wherein a hacker gained access to the account and he deleted the emails in it.

The bar codes produced by Word are no longer compliant with USPS regulations (http://support.microsoft.com/kb/897290 ). They were only useful for bulk mail anyways.

If you still want a barcode on your enveloppe, you can enter a barcode field by hand. "Insert" tab => "Quick Parts" button => "Fields..." => "BarCode".

Alternatively, you can use one of the enveloppe templates provided by Graham Mayor athttp://www.gmayor.com/Zips/Envelope.zip
