What Dirk posted will get you the name of the form object displayed in the navigation form's subform control.

   Forms!NavigationForm!NavigationSubform.Form

And to refer to controls in the form, use:

   Forms!NavigationForm!NavigationSubform.Form.[control name]

   Forms!NavigationForm!NavigationSubform![control name]

Because there is a space between the words Program and Files, Windows is interpreting the first part as a program name and the second part as an input parameter (this has been going on since the days of DOS). To make Windows see the whole thing as a path, put double quotes before the C: and after the Startup\. (Yes, it is strange that the KB article omits the quotes, because the space is predictable.)

There is another possible gotcha. If you're running 64-bit Windows, the Startup folder is not in the location under %programfiles% but instead under %programfiles(x86)%.

When you get to step 7, about the Startup folder in your user profile, you may or may not need quotes around the path (but it won't hurt to put them in). There is no difference in this step between 32-bit and 64-bit Windows.
