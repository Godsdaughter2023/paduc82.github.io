a. user say:

I will save you some time.

Your reported PUM:

PUM.HiJack.Start Menu. HKCU\Software\Microsoft\Windows\Current Version\Explorer\AdvancedStart_ShowSearch


Registry Data Items Detected: 1
HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\Advanced|Start_ShowSearch (PUM.Hijack.StartMenu) -> Bad: (0) Good: (1) -> Quarantined and repaired successfully.

That has nothing to do with your slow startup issue.

That PUM is telling you about a change to your Start button menu (NOT the Classic Start menu) that tells your system to display for the currently logged in user the Search option or not to display the Search option (on the right hand side of your Start menu).

If it is set to 1 (the default value), the Search option is displayed.  If it is set to 0 (zero), the Search option it NOT displayed.

Somehow yours got changed from the default value of 1 to 0 and if you did not do it on purpose, some malware probably did it for you to make it harder for you to find and remove the malware.  If you are using the Classic Start menu, this modification has no effect.

MBAM noticed the change, offered it to you as a Potentially Unwanted Modification and allows you to fix it which I assume you did.  That is what some malware does - modifies your system to make it harder for you to find and fix it to try to fool you into thinking you have to perform a Repair install or even reinstall your XP from scratch when none of those things are necessary.

You can try a System Restore if you want, but if you restore your system to a date prior to MBAM fixing the PUM, you will restore the PUM too.

How do I know this?

I recreated the problem and tried it (even though I didn't need to to know that's what would happen).

Nothing to do with slow startup.
