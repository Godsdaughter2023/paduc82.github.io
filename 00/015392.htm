Steps to repro:
0. Fully patch Windows 10.
1. Set DPI for Windows 10 to 125%, sign out or reboot and log back in.
2. Open Powerpoint 2013 or 2016
3. Click "Insert" -> "Object" and select "Microsoft Equation 3.0" from the list.
4. Try to insert any type of equation. Clicking any box in the Equation Editor itself will show the drop-down so long as left-click is held down, but after letting go, the drop-down disappears and no equation notation is inserted. No selection box appears when the mouse cursor is moved over equation notation.

- This is speculation, but it seems that the equation editor is creating a "mask" onto which it will apply the typed in equation, but that the mask is not scaled correctly with the DPI settings increase. One can observe a hashed box that is not covered by the input box, whereas on a machine with DPI = 100%, the hashed box IS covered by the input box.

Fix:
5. Reset DPI to 100
6. Log out / reboot, log back in
7. Click "Insert" -> "Object" and select "Microsoft Equation 3.0" from the list.
8. Insert an equation. Note now that clicking and releasing on an equation box causes the drop-down to remain and that a selection box follows the mouse cursor as a user moves the mouse to select the piece of an equation.

Is there a way to show this to someone at Microsoft so it can be fixed? This was working fine until a patch recently.
