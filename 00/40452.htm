lemp say:

You have an "MAK" product key, not a "KMS" product key.  They are not the same thing and a system with an MAK license does NOT have to access a server.

A Microsoft MAK license is a type of volume license typically used by organizations that have relatively few computers.  If I recall correctly, the decision point is around 25.  MAK keys are also useful in situations where the client computer will not be connected to the organization's network for long periods of time.  A MAK key is good for a specific number of device activations.  Windows is installed, the MAK key is entered, the system is activated either online or by phone, just as with ordinary retail licenses, and that is the end of it.

A system with a KMS key, on the other hand, must be connected to a KMS server on the network of the organization that has purchased the volume licenses and get re-activated periodically (every 180 days).

Here's a reasonably easy to understand explanation:  https://www.reich-consulting.net/support/software/windows-software/the-difference-between-mak-and-kms-keys/

The problem with MAK keys is that unscrupulous people can sell the key, either as just a key or together with an installation of Windows, and it may work for a period of time.  Sooner or later, however, the number of devices authorized for that key will be exceeded, whereupon Microsoft blocks the key and you get the result you're seeing.
