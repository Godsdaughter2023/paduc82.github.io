palcouk say:

You are not understanding the concept, nor the function of the Outlook Data files, so here goes;

1) Any data file that is open within Outlook, has that data files archive bit set to modified.

2) It doesn't matter as to whether any data is added or not, the very fact that it has been opened in Outlook resets the files archive bit as modified.

3) Any backup program functions by detecting whether the archive bit has been set to modified, and if it has the file is backed up

3) To reduce the size of the primary data file (by default named outlook.pst) use the archive options, if necessary you can create a new Archive for each year, and name that archive as appropriate, ie Archive 2012.pst

4) Then ensure that that/those Archive(s) are not open within Outlook 

5) When your backup program then runs it will backup the Archives, but provided you haven't either opened that Archive within Outlook or added to that Archive the backup will not re-backup that Archive

6) Other than that, exclude the outlook data files from your on-line backup, and use an external drive backup for outlook.

There is no other option

SELECT AccuntNumber, LastName & ", " & Firstname AS Fullname FROM tablename ORDER BY Lastname, Firstname;

I would set the Bound Column to 1, The Column Count to 2 (since you are combining first and last names) and the Column Widths to 0";2". With these settings, when you pull down the list, you see just the names in alpha order, but when you select a name, the account number will be stored.

Tip: You should not have spaces in object (field) names, this can come back to haunt you.
