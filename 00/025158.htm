curiousabout life say:

Hello,

Unfortunately, with the madness which has become Outlook.com/Hotmail.com, one delete function which seems to be working is the deletion of email from the normal Inbox view.  Initially, before the team cleaned up the haphazard way it worked, you lost everything without wanting to do so! Currently, there are two (2) methods to delete email from the Inbox.

If you wish to remove one or more emails, but not all, from your Inbox, you would preform the following steps:

1.  Ensure you are in your Inbox view.

2.  Click the box to the left of the one ore more emails you wish to delete in this view.

3.  At the top of the view, click "Delete", next to the trashcan icon, to complete the email deletion.

If you wish to remove entire blocks of email, this function exists, but I caution you to be very aware to ensure you are deleting exactly what it is you wish to delete.  To remove blocks of emails means to delete all emails in the current Inbox view.  Outlook now deletes pages, or page views of emails, but if you wish to delete ALL emails in the Inbox, there is a small check to do this which was pretty much a very good and popular function in the previous version.  First, I will tell you how to remove pages of emails, followed by the removal of ALL emails (which is merely a small click in addition).

1.  Ensure you are in your Inbox view.

2.  Run your mouse at the top of the page, next to "Inbox" where you will see a hidden box appear.  Then click the box.  You will now see the entire page view of emails will be selected.  

NOTE:  You will  now see the "Inbox" label has disappeared, and now there is a message to the right of the clicked box saying "Select everything in  your Inbox folder".  DO NOT CLICK THIS MESSAGE!!!!!

3.  If all you want to do is to delete a single page of email, then click "Delete", or the trashcan icon.  You can use this same procedure to delete multiple pages at a time.  However, if you want to remove ALL email from your Inbox, then proceed to the next set of instructions.

To delete ALL emails from your Inbox:

1.  Ensure you are in your Inbox view.

2.  Run your mouse at the top of the page, next to "Inbox" where you will see a hidden box appear.  Then click the box.  You will now see the entire page view of emails will be selected. 

3.  You will also see the "Inbox" label has disappeared, and now there is a message to the right of the clicked box saying "Select everything in  your Inbox folder".    Click this message to remove all emails from your Inbox.

I hope this helps you with your needs.

As a sidebar, It would be great if the delete function worked when you wish to select specific emails for deletion by using, say, a search criteria.  This use to work, especially in the previous version, but now it does not.  While many examples, emails, and conversations have been sent to the team, so far there has been no feedback or response as to why this very useful tool was removed.

ken sheridan say: You can't hide controls selectively in different rows in continuous forms view; each row is showing an instance of the same control, so it's all or nothing.  Instead set the Enabled property of the control to False/True as appropriate in whatever event procedure you would use in single form view to set its Visible property to False/True to hide/show it.  Disabled controls are greyed out and unavailable to the user.  You will need to do this in the form's Current event procedure to cater for editing existing rows as well as in specific controls' AfterUpdate event procedures to disable/enable controls on the basis of values inserted by the user.

How to write a query to 'compare' values? - Microsoft Community

shannon9585 say:

How to write a query to 'compare' values?

I have a table that stores weekly status entries for projects managed by my office. Five columns in the table (named Scope, Schedule, Staffing, Finances, & Technology) have 3 valid values: "No Issues", "Minor Issues", "Critical Issues". I also have a report set up that shows the most current status entry for each project.

Here's what I want to do:

On the report, I'd like to include an indication of how the status of the project has changed from one week to the next. I need some sort of query that looks at the value of the previous week (for the columns listed above) and compares it to the current week. For example - if a project's current Scope value is "Minor Issues" and the previous week's value was "No Issues" I want to show a minus sign or indicator that says something like "declining" to indicate that the Scope status has declined from the previous week. A plus sign or "improving" message would be shown if the value has improved from one week to the next. One indicator value would be shown next to each of the values form the 5 columns listed above.

How do I set up a query to do this? For Scope, I want to say:

"Compare last week's value to this week's value.

Display 'No change' indicator if the values match.

Display 'Declining' indicator if this week's value is 'Critical Issues' and last week's value was not 'Critical Issues'.

Display 'Declining' indicator if this week's value is 'Minor Issues' and last week's value was 'No Issues'.

Display 'Improving' indicator if this week's value is 'No Issues' and last week's was not 'No Issues'.

Display 'Improving' indicator if this week's value is 'Minor Issues' and last week's value was 'Critical Issues'."

Then I'd have to repeat this for each of the other 4 fields.

I don't know the syntax for writing a query or macro like this.

hansv mvp say:

What kind of field do you use for the week? A date/time field, or a weeknumber field, or ...?

qa guy commelec say:

I would recommend using an Option Group for each on your form to select status and then storing 0, 1, or 2 for the status.  When comparing weeks the difference would be easy.

Use an IIF statement for each as below (You can do the rest) ---

SELECT tblProjStatus.Project, tblProjStatus.StatusDate, IIf([tblProjStatus_1].[Scope]=[tblProjStatus].[Scope],"No Change",IIf([tblProjStatus_1].[Scope]<[tblProjStatus].[Scope],"Improving","Declining")) AS Scope_, tblProjStatus.Schedule, tblProjStatus.Staffing, tblProjStatus.Finances, tblProjStatus.Technology
FROM tblProjStatus INNER JOIN tblProjStatus AS tblProjStatus_1 ON tblProjStatus.Project = tblProjStatus_1.Project
WHERE (((tblProjStatus_1.StatusDate) Between [tblProjStatus].[StatusDate]-1 And [tblProjStatus].[StatusDate]-8));

 

Build a little, test a little.
