jim gordon mvp say:
The table and the heading are the same on each page - the table content is different.

Ahhh.. That's a different situation.

It is pretty easy to accomplish what you are seeking.  In this case you have a single table that spans many pages. So what you want to do is tell word to repeat the same row at the top of each page that the table spans.

All you need to do is right-click into the first row of the table, which presumably has the headers and the rest of the rows have other information. From the pop-up menu choose Table Properties.

In the Table Properties dialog click the Row button at the top. In the Row options, click the check box for Repeat as header row at the top of each page. Then click the OK button.

All done!

ken sheridan say: You can do it with DAO.  The following is a function which creates a 'calendar' table for any date range, including whatever days of the week are required:

Public Function MakeCalendar_DAO(strTable As String, _
                            dtmStart As Date, _
                            dtmEnd As Date, _
                            ParamArray varDays() As Variant)
                            
    ' Accepts:  Name of calendar table to be created: String.
    '           Start date for calendar: DateTime.
    '           End date for calendar: DateTime.
    '           Days of week to be included in calendar
    '           as value list, e,g 2,3,4,5,6 for Mon-Fri
    '           (use 0 to include all days of week)

    Dim dbs As DAO.Database, tdf As DAO.TableDef
    Dim strSQL As String
    Dim dtmDate As Date
    Dim varDay As Variant
    Dim lngDayNum As Long
    
    Set dbs = CurrentDb
    
    ' does table exist? If so get user confirmation to delete it
    On Error Resume Next
    Set tdf = dbs.TableDefs(strTable)
    If Err = 0 Then
        If MsgBox("Replace existing table: " & _
            strTable & "?", vbYesNo + vbQuestion, _
            "Delete Table?") = vbYes Then
            strSQL = "DROP TABLE " & strTable
            dbs.Execute strSQL
        Else
            Exit Function
        End If
    End If
    On Error GoTo 0
    
    ' create new table
    strSQL = "CREATE TABLE " & strTable & _
        "(calDate DATETIME, " & _
        "CONSTRAINT PrimaryKey PRIMARY KEY (calDate))"
    dbs.Execute strSQL
    
    ' refresh database window
    Application.RefreshDatabaseWindow
       
    If varDays(0) = 0 Then
        ' fill table with all dates
        For dtmDate = dtmStart To dtmEnd
            lngDayNum = lngDayNum + 1
            strSQL = "INSERT INTO " & strTable & "(calDate) " & _
               "VALUES(#" & Format(dtmDate, "mm/dd/yyyy") & "#)"
               
            dbs.Execute strSQL
        Next dtmDate
    Else
        ' fill table with dates of selected days of week only
        For dtmDate = dtmStart To dtmEnd
            For Each varDay In varDays()
                If Weekday(dtmDate) = varDay Then
                    lngDayNum = lngDayNum + 1
                    strSQL = "INSERT INTO " & strTable & "(calDate) " & _
                        "VALUES(#" & Format(dtmDate, "mm/dd/yyyy") & "#)"
                    dbs.Execute strSQL
                End If
            Next varDay
        Next dtmDate
    End If
    
End Function

Rather than limiting the table to the dates of a specific range I'd suggest you build a general purpose calendar of all Mondays and Tuesdays over an extended range by calling the function as follows:

MakeCalendar_DAO "MonTuesCalendar",#2010-01-01#,#2020-12-31#, 2,3

You can then return a result table from this for whatever range you need , e.g. for the dates you cite, with a query:

SELECT *
FROM MonTuesCalendar
WHERE calDate BETWEEN #2012-06-25# AND #2012-07-15#;

It might help us to help you more categorically, however, if you were to explain in just what way you need to use this set of dates.

SELECT StartingDate + Choose( DatePart("w", StartingDate), 1, 0, 6, 5, 4, 3, 2) + 7* Iotas.iota
FROM Iotas
WHERE StartingDate + Choose( DatePart("w", StartingDate), 1, 0, 6, 5, 4, 3, 2) + 7* Iotas.iota
              BETWEEN StartingDate AND EndingDate

UNION ALL

SELECT StartingDate + Choose( DatePart("w", StartingDate), 2, 1 , 0, 6, 5, 4, 3 ) + 7* Iotas.iota
FROM Iotas
WHERE StartingDate + Choose( DatePart("w", StartingDate), 2, 1, 0, 6, 5, 4, 3) + 7* Iotas.iota
               BETWEEN StartingDate AND EndingDate

 

 

The first SELECT generates the Mondays while the second SELECT generates the Tuesdays.

 

NOTE: the query will be much faster if iota is indexed AND if you rewrite the WHERE clause like:

WHERE Iotas.iota <= ( EndingDate-StartingDate - Choose( ... ) ) / 7

 

(with ... replace by the DatePart( ) expression and the list of constants)  but that is far less "self evident". Indeed, the index can only be used if it is not part of an expression, in other words, SQL won't do the algebric manipulation for you. You can come with that simplification if you change "x  BETWEEN y AND z"    into    " x >= y   OR   x <= z".

yyyy Year

Note that if, as example, StartingDate is a Friday, DatePart(""w", dateIsAFriday)  returns 6, the Choose expression returns 3, and a Friday + 3 more days is a Monday, as required.

 

Iota * 7  just add 7 days, for all possible iota values, as long as the resulting date is less than or equal to EndingDate (a parameter)

Yes.  First, pause updates in the Insider page in Settings.  Then run Disk Cleanup to delete all traces of 17063.  Run it from a Command Prompt, using cleanmgr /sageset to select all options, followed by cleanmgr /sagerun to delete what was preselected (by /sageset).

Make sure that the hidden folder at C:\$WINDOWS.~BT is deleted.  Try to delete C:\Windows\SoftwareDistribution also.  I usually boot to Windows install media or use SHIFT-Restart to get a Command Prompt to delete these folders.  That way, they are not locked or in-use when you are trying to delete them.

Another way, if you have Windows 10 install media, is to begin an upgrade, even if it is to some older version, but then cancel it before proceeding.  One of the first things done in an upgrade is to clear away all traces of prior or pending upgrade files.  So this should eliminate the 17063 attempt and allow you to go directly to 17074.

Resume Insider updates to get 17074.

You don't need VB code try this. Put this in a cell in Col B and ARRAY enter it, see below on how to do that, and then drag down until it start returning blanks.
