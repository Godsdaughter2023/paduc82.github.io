dirk goldgar say:

I'd be inclined to do this with VBA code. 

I'd change the AttendeeFilter command button into a check box, "chkFilterAttendees" -- when checked, a filter on attendees < 7 is desired; unchecked = no filter on attendees.

A single command button, "cmdFilter", would be used to construct and apply a filter determined by the values of cboCourse, cboLocation, and chkFilterAttendees.  The command button's Click event procedure would be something like this:

'------ start of code ------

    Dim strFilter As String
   
    If Len(Me.cboCourse & "") <> 0 Then
        ' If Course is numeric, use this:
        strFilter = strFilter & " AND (Course = " & Me.cboCourse & ")"
        ' If Course is text, use this:
        'strFilter = strFilter & " AND (Course = """ & Me.cboCourse & """)"
    End If
   
    If Len(Me.cboLocation & "") <> 0 Then
        ' If Location is numeric, use this:
        strFilter = strFilter & " AND (Location = " & Me.cboLocation & ")"
        ' If Location is text, use this:
        'strFilter = strFilter & " AND (Location = """ & Me.cboLocation & """)"
    End If
   
    If Nz(Me.chkFilterAttendees, 0) <> 0 Then
        strFilter = strFilter & " AND (AttendeeCount < 7)"
    End If

        Me.Filter = Mid$(strFilter, 6)
        Me.FilterOn = True
    Else
        Me.FilterOn = False
        Me.Filter = ""
    End If

End Sub

Of course, you'd have to adjust field names and allow for text vs. numeric fields in the above code.

ken sheridan say: Personally I'd adopt a different approach, and base the form on a query which references the unbound controls as parameters in its WHERE clause:

WHERE (Course = Forms!YourForm!cboCourse
OR Forms!YourForm!cboCourse IS NULL)
AND (Location = Forms!YourForm!cboLocation
OR Forms!YourForm!cboLocation IS NULL)
AND ((AttendeeCount < 7 AND
Forms!YourForm!chkLocationchkFilterAttendees = TRUE)
OR Forms!YourForm!chkLocationchkFilterAttendees = FALSE)

The paraentheses are very important in the above to force the AND  and OR operations to evaluate independently.  In the form, to apply the restriction you just need to requery the form with:

   Me.Requery

which you can do in the Click event procedure of a separate button, or in the AfterUpdate event procedure of each of the three controls to apply the restriction progressively as soon as a value is selected in one of the controls or the control is cleared of its value (or unchecked in the case of the check box).

To clear the controls and show all rows you can have a 'Show All' button with the following code in its Click event procedure:

Me.cboCourse = Null
Me.cboLocation = Null
Me.chkLocationchkFilterAttendees = False
Me.Requery

With the above approach you do not have to concern yourself at all with the data types of the columns.

When a value is selected from List, UserName into B, Date into C

Private Sub Worksheet_Change(ByVal Target As Range)
'timestamp and username
    Dim cRow As Long
    Dim ws As Worksheet
    Set ws = ActiveSheet
    If Not Intersect(Target, Range("A1:A10")) Is Nothing Then
        cRow = Target.Row
        With ws
            .Range("B" & cRow).Value = Application.UserName
            .Range("C" & cRow).Value = Now
        End With
    End If
End Sub

     After going to the Answer Desk chat function and finding no help except being directed to forum links that suggested I go to the registry, back it up and delete files, (which scared the you know what out of me), I finally found the solution buried in one of the other forums.  The solution took all of 2 minutes once I discovered it.

Problem: I was using Hotmail as the source of my e-mails. Hotmail (and maybe others?) needs an add-in to function when you set it up in Outlook; it is the Outlook Connector. When converting Office (and thus Outlook) from 32 to 64 bit, the Outlook Connector did not work.

Solution: Download the Outlook Hotmail Connector 64-bit file (link below). If the link for some reason does not work, go to Microsoft.com then to Downloads and narrow it down to the Office suite and type in the search box Outlook Hotmail Connector 64-bit. That will take you right to it. Once you download it, Outlook should launch normally.

http://www.microsoft.com/en-us/download/details.aspx?id=21305

Background:

      My goal was to upgrade Excel to 64 bit because I was having problems with large complex files crashing. I was told that if you are running a 64 bit operating system (mine is Windows 8.1 64 bit), that by converting Excel to a 64 bit it would "unlock" the full potential of the memory that was available. After converting it did cut down the processing time and no crashes thus far. I went to the Answer Desk chat and, because I did not have my disk or product key, they were unable to assist and suggested I buy Office 365 or Office 2013. I dug in further into my files and found the product key. Went back to Answer Desk and they helped install the 64 bit (which was fantastic). The problem was, they/I did not test Outlook before they left. The next day when I tried to launch Outlook, I received the message: "The set of folders cannot be opened.  An unexpected error has occurred.  MAPI was unable to load the information service msncon.dll.  Be sure the service is correctly installed and configured."

     I then went back to Answer Desk and the tech immediately informed me that the install process was free but potentially any solution now would not be. This was reiterated a number of times throughout the process. I outlined the conversion of 32 to 64 bit and gave him the error message. He never once mentioned to just download the new Connector but sent me to a video that assumed I was able to launch and be in the program. He then sent me a link to a forum for messing with the registry. As I mentioned above, I eventually found the solution buried in another forum.

      I re-engaged the Answer Desk to give them the suggestion to inform their department: 1.  Have their techs doing an install of 32 to 64 bit conversion to ask about Outlook or test it out before leaving.  2. If someone is having the above error message, have them tell the customer to upgrade their connector from 32 bit to 64 bit. I have heard the most people install/use 32 bit. But as more people migrate to 64 bit, this issues may come up more often.

       I am starting this discussion in the hopes that someone else with this issue can find the solution faster than I did.

Hope it helped.
