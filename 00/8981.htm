Update Bookmarks - Microsoft Community

kseaver say:

Update Bookmarks

I have a form that requires information to be inputted into many different locations in the document. I am trying to figure out a macro that fires on exit so that when the first form field is filled in it auto comletes the other fields that require the same information. For example Bookmark 1, 5, and 15 should all read the same. Can I put an on exit macro on bookmark 1 that will fill in bookmark 5 and bookmark 15. All bookmarks are text form fields.
suzanne s. barnhill say:

You don't need a macro and you don't need multiple fields. Every form field includes a built-in bookmark, which you can see in the Form Field Options dialog. By default these are Text1, Text2, etc. for Text Form Fields, but you can modify the bookmark to something more meaningful. Where you want the data entered in that field repeated, you insert a REF (cross-reference) field referencing that bookmark. If you check the box for "Calculate on exit" in the Form Field Options dialog, all the REF fields will automatically be updated when you exit the text form field.

 

For more detail on how to do this, see http://gregmaxey.mvps.org/word_tip_pages/repeating_data.html.

Teams Files vs Sharepoint Sites for converting from a local File Share?

Hi,

Do we create a Team for each department and then dump all the files from their shared drive into the Files tab?

tim liu msft say:

Hi Kelemvor33,

I'd like to explain that Teams>Files is also related to a SharePoint Online site collection. When we create a team, it create a related site collection for it. Saving documents in Files means save them in that site collection. So, for saving files, Team>Files is the same as SharePoint Online team site. The difference is Teams has other functions. It's a services integrates several Office 365 for Business services. SharePoint Online team site is mainly used for file storing and sharing of the whole tenant.

Use Teams>Files

You can create different teams for different departments. Create private teams if you don't want users to access other departments. Team members can save files and user other teams services here. Also, each team has its separate site collection. You need to manage permissions for each of them.

User SharePoint team site

This is a default site collection in the tenant. It can be seen in the SharePoint admin center (the site collection which is related to a team won't disappear in the SharePoint admin center). So, we can easily configure its settings in admin center rather than use SharePoint Online PowerShell. You can choose to create different folders in one library for different departments. Or create different libraries for different departments in the same site collections. And then manage folder/library permissions to restrict users access. E.g. Department A can only access Folder A/Library A. This part is the same as Teams>Files. To learn more about permissions, see Understanding permission levels in SharePoint.

For the whole tenant, I will suggest you use SharePoint team site. As you can manage all things in one site collection and easier to configure other settings in SharePoint admin center. For more functions and some small project team, they can choose to use Teams. Below are some articles for your reference:

Microsoft Teams rolls out to Office 365 customers worldwide

What is a SharePoint team site?

Other community members are welcome to share your experience here.

macropod say:

Your =SUM(ABOVE)/2 formula won't be getting any data from the dropdown, but from other content in the cells above that add up to 24. If you need to reference the dropdown's result:

a) the dropdown must have the 'calculate on exit' property set; and

b) you need to use a formula field (not a formfield) coded as {={REF Dropdown1}/2}, where Dropdown1 is the dropdown's internal bookmark name and both pairs of field braces (i.e. { }) are created in the document via Ctrl-F9.
