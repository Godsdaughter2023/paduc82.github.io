ken sheridan say: A relational database management system like Access differs radically from a spreadsheet.  Data in a relational database table is not addressed by position, but by the value at one or more column positions in a row in a table, which can be a base table or the result table of a query.  Also Access uses expressions not formulas.

The nearest equivalent to the Excel VLOOKUP  function is the DLookup function  This takes three arguments, all as string expressions:

1.  The first argument is the name of the column (field) in which the value will be located.

2.  The second argument is the name of the table or query in which the column is located.

3.  The third argument is the criterion or criteria which identify the row from which the value will be returned.  This is a string expression which evaluates to TRUE or FALSE for each row in the table.  If it can evaluate to TRUE for more than one row the value from the first row located will be returned.

As an example, let's assume we have created a query named qryCustomerTotalSpend which returns the total value of all orders made per customer in a column named TotalSpend.  In a form which includes a combo box control named cboCustomer, bound to a CustomerID column, we want to return the total spend by the customer.  For this we could use the following expression as the ControlSource property of an unbound text box control in the form:

=DLookup("TotalSpend", "qryCustomerTotalSpend","CustomerID = " & [cboCustomer])

This assumes that the CustomerID column is a number data type, which it would usually be.  But if by any chance it were a text data type the expression would be:

=DLookup("TotalSpend", "qryCustomerTotalSpend","CustomerID = """ & [cboCustomer] & """")

In this case because the CustomerID value is a text data type it is necessary to wrap it in literal quotes characters as above.

How can I select and add a column of figures in a Word document using - Microsoft Community

michaelturton say:

How can I select and add a column of figures in a Word document using Office.Mac 2011 - as I could with earlier versions?

1  How do I select a column or selective text?
2  How can I add a column of numbers to derive a total?
amc-0 say: Answer to number 1: To select a column of text (that's not in a table), Option-drag diagonally through the column. That highlights a rectangular selection starting from your initial click. I usually start to the left of the first word/number I want to select, then option-drag down and to the right.

Answer to number 2: Assign a keyboard shortcut to ToolsCalculate, or add the ToolsCalculate command to the main Toolbar. I've found the latter is more reliable. This command is not on any menu I've ever been able to find, but you can use one of these 2 ways to make it do its thing, which is to add all of the numbers in a selection and put the results in your Clipboard. 

To assign a shortcut:
  1. Go to Tools > Customize Keyboard
  2. Choose All Commands from the list on the left.
  3. Scroll through the alphabetical list of all commands on the right until you find ToolsCalculate.
  4. Select that command, then click inside the field where you type a shortcut, and enter the shortcut you want to use (I use Control-C on my Mac for this command, but sometimes, nothing happens. I tend to use the Toolbar button instead).
  5. Don't forget to click the Assign button otherwise it won't remember
  6. Click OK
To add it as a button to the Toolbar:
  1. Go to View > Toolbars > Customize Toolbars and Menus
  2. Choose All Commands from the left
  3. Scroll through the list on the right until you find ToolsCalculate, and select it.
  4. Drag and drop the command from the dialog box to an empty area of your toolbar.

What's nice about it is that it works with free range selections as well as columns in a table (though you do have to select the cells in the column first) and the results are conveniently stored in your clipboard, ready for pasting anywhere in the document or in any file.

Also, it ignores text, just pays attention to numbers.
