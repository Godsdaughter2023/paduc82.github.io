OneNote sections/pages/comments merge - Microsoft Community

tatyana michnik say:

OneNote sections/pages/comments merge

One of my users had her Notebook copied from her C drive, to our network. She didn't realize that her OneNote was still linking to the old C drive copy and sometimes would make changes in there. We need to remove the reference to the old notebook but before we do that, we need to make sure that all the changes are on the network drive. 

Is there an easier way, besides manually, to merge the changes between two Notebooks? Or at least to see what's different between the two?

 

swischner say:
Although Microsoft isn't "officially supporting" sharing locally stored notebooks by simply copying or moving them to a network drive any more, this *usually* works. But best practice is to use the internal "File-Share" command to move the notebook to a supported cloud share (OneDrive, OneDrive for Business) or a Sharepoint library.

Now for your question: There is no automated way to merge changes of different notebooks/sections/pages together. It can't be because some changes could easily contradict each other and an algorithm cannot decide which change is correct. So it has to be done manually.

For this I' suggest the following method:

The user with the local notebook should open BOTH versions which can easily be done, even if they have the same name. Just use "File", "Open", "This PC" (also for the notebook on the network).

Open a second OneNote window with Ctrl-M, move both windows side by side and display (select from the list) each version of the notebook in one of the windows.

Now he can manually walk through all pages, check the differences and either update the correct changes by copying/pasting content between the windows or (if complete sections or pages are new / more recent) use the context menu in the corresponding list (section, page) to copy or move the whole section or page.

What may also help to find differences or recently made changes is the command "History", "Recent Edits" that lets you display a list of recent changed pages by a date range like "Last Week" or "Last Month".

After the network version is updated simply close the notebook (not just the OneNote window) which is stored locally and just keep working on the other (network) one. I'd keep that local version on the local disc for a while before deleting it (using Windows Explorer) until you are sure that no content/change has been left out. That way you can simply open it again in case to retreive the missing content like described above.

swischner say:
Additional suggestion that I forgot: Before closing the local (old) notebook at the enfd of the procedure I'd rename its "Display Name" (under "File", "Settings" beneath the notebook name in the big list, "Properties". Maybe add "(old)" to its name to avoid accidentally opening and working in the wrong version again.

a. user say:

Use columns on folders in sharepoint library

Hello,

I've created a column in a Sharepoint library I use to have project documentations.

I've got 1 folder per project, and I'd like to be able to assign a responsible person for these folders.

So I created a person picker column.

But it seems impossible to add any value to that column on folders... How can I fix this?

Thanks,

JG

cliff gu msft say:

Hi JG,

 

You need to create a custom folder type with the person picker column to do that. Please first go to Site settings>Site columns and Site content types under Web Designer Galleries to create a person picker site column and a folder site content type. In my environment, I create a person picker site column naming person picker 1 and a folder site content type naming folder type one. Then click on folder type 1>click Add from existing site columns>select person picker 1 to add the column to the folder content type. After completing the content type, navigate to the library>Library Settings>Advanced settings>select Yes in Allow management of content types?>click on Add from existing site content types to add the folder content type to the library. Now change the default view to add the person picker column and then you can add the person value to the folder item if you create a folder item by selecting the folder type 1 content type.

 

Thanks,

Cliff
