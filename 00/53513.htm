Updated (3/21/17)

Hi Jestoni,

We are currently investigating the issue and will update this thread when we have more information.

Advanced users can resolve the problem by trying one of the two options listed below:

Option 1:  Rename the WindowsApps folder in Safe Mode

1.    Boot into Safe Mode.
2.    Right click on Start and click on Command Prompt (admin).
3.    Type these commands below one by one.

             •    cd C:\Program Files
             •    takeown /f WindowsApps /r /d Y
             •    icacls WindowsApps /grant “%USERDOMAIN%\%USERNAME%”:(F) /t
             •    attrib WindowsApps -h
             •    rename WindowsApps WindowsApps.old
4.    Reboot back into Windows.
5.    Run System Restore.

Option 2:  Rename the WindowsApps folder from the Windows Recovery Environment (WinRE)

  1. Boot into WinRE.  To do this, click Start > Settings > Update & security > Recovery.  Then under Advanced startup, click Restart now.
  2. Click Troubleshoot > Advanced Options > Command Prompt.  Enter your administrator password when prompted.
  3. Type these commands, one by one:
    • cd C:\Program Files
    • attrib WindowsApps -h
    • rename WindowsApps WindowsAppsOld
  4. Reboot Windows.
  5. Run System Restore.

Do you have data on the computer you need to recover? You should do that first. The way I would do it is to remove the hard drive and to to place it in a spare drive slot in another desktop computer or if the second drive is a laptop place the drive in an external drive enclosure and connect the enclosure. Either way you most likely will be able to backup your data to another drive or removable media.

An external enclosure is illustrated in the next link:
http://www.amazon.co.uk/gp/product/B000V8KQE6/ref=pd_lpo_k2_dp_sr_2?pf_rd_p=103612307&pf_rd_s=lpo-top-stripe&pf_rd_t=201&pf_rd_i=B004ODU3WY&pf_rd_m=A3P5ROKL5A1OLE&pf_rd_r=1AE3HQ7S35N7VFJXSJ2K

You get variants for 2.5 inch drives from laptops. Other variants handle Ide rather than Sata drives.

Using Google on this error code "0x80070017" suggests the drive has failed a cyclic redundancy check. Run chkdsk and try to fix the problem.

Select Start, Run, type cmd and press ENTER. Type "chkdsk ?: /f /r" without the quotes and hit ENTER. Make sure you include the spaces indicated. Enter Y when asked whether you wish to run chkdsk on restarting the computer. Substitute drive letter in command for question mark. Exit and restart the computer.

Marking off bad sectors on a hard drive takes time so be patient. Marking off does not repair a bad sector. It places pointers on the drive telling the system not to read or write to those sectors which have been damaged.

If the number of bad sectors continues to increase after you have run the procedure above several times then you should replace the drive. If an important system file is written to a bad sector you can corrupt registry hives and lose the whole contents of the drive.

On the other hand if having run chkdsk you see no more new bad sectors then the drive can work for you for years.

You can then see if the drive will boot in the original computer but if the registry has corrupted you wil have to do a clean install of Windows with all that entails. Before doing that it would be worthwhile running the drive manufacturers test utilitity. What is the drive make and model? Is the drive still under warranty?
