marshall barton say:

Using auto prompting for query parameter is a verycrude and limiting mechanism that I only use for testing a concept, not ever for users.

The only thing that makes sense to me is for users to enter the parameters in text/combo boxes in the header section of a continuous form.  Then they can change one parameter and see the result set immediately.

This way, you can change your query criteria to this kind of where conditions:

    =Forms!theform.parametertextbox OR Forms!theform.parametertextbox Is Null

and use a Line of VBA code in each od the text/combo box's AfterUpdate event:

   Me.Requery

While that results in a greatly improved user interface, the query can become cumbersome and may take a bit of a performance hit.  IMO, the best way to achieve what I think you want to do is to get rid of all the criteria in the query and use a bunch of fairly straightforward VBA code to construct the form's Filter property.  The code is not much more that a few lines for each text/combo box:

Function BuildFilter()

Dim strFilter As String

   If Not IsNull(Me.textbox) Then

      strFilter = strFilter & " AND fieldname = " & Me.textbox            'when field is a Number field
  '    strFilter = strFilter & " AND fieldname = """ & Me.textbox & """ "   'when field is a Text field

  '    strFilter = strFilter & " AND fieldname = " & Format(Me.textbox, "\#yyyy-m-d\#")  ' Date/Time field

   End If

' repeat above If block for each text/combo box

   Me.Filter = Mid(strFilter, 6)
   Me.FilterON = True

End Function

Then set each text/combo box's AfterUpdate event property to   =BuildFilter()

This concept can be extended way beyond parameter prompts relatively easily to allow wildcards or combinations of criteria such as a date range.  There is a more in depth article of this approach at:  http://allenbrowne.com/ser-62.html

glenprouty say:

Hi Eli,

I believe you are right, Repair Upgrade is called for.

The Repair Upgrade procedure is very stable, but it doesn't hurt to take

precautions.  You should probably "create a recovery drive" before you 

begin, and if possible create a backup Image of your OS. Better safe than

sorry.   The only problem I have ever seen using this procedure came when

the OS was shared on an SSD and an HDD. If yours is only HDD you should

see no problem.

The ISO that Microsoft supplies is for the latest version of W10. At this point

in time that might be W10 v1903. If it is, you may see some differences on

your PC when finished.        Or it may still be v1809.

The procedure will take about an hour and a half. Your PC will restart

several times, and sometimes it appears to be doing nothing. Be patient

and do not interrupt it. When it finishes, the Sign In  screen will appear.

Begin by downloading the Media Creation Tool from here.

https://www.Microsoft.com/en-us/software-download/windows10

Once the tool has downloaded, right-click it and "Run as administrator"

Select the option to "Upgrade This PC Now". Accept the normal things

during the process.  When you are asked "what to keep", ensure that both

the "OS" and "Apps & Files" are selected.

They will recommend to include "updates" during the procedure, but I

prefer to decline that option. I believe that it is better to "Check for updates"

after the new operating system is in place.

When the process is finished, you have a brand new OS in place, and your

Files and Apps are included. The first thing to do is to go to Windows Update

and "Check for updates". Once all of the updates are installed, you should

find that Windows Defender is functional again.

If Windows Defender is not functional, that indicates your problem is caused

by one of your APPS. (Not likely in this case). But, since they were retained, a

slight chance exists. You will need to perform a "Clean Boot" procedure to

determine which one is the culprit, in that case.

Good Luck,  Glen
