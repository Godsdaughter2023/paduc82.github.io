Hi Likhitha,

Just saw your post. The bootrec /rebuildbcd was what was needed for the cloned drive. 

I didn't know that you could download an iso to create a Windows Recovery USB stick... that's rather convenient as I usually don't have another Windows 10 machine around (just a bunch of Macs) but... with the ISO (if any other Mac users find this) open up a terminal (command space bar and type terminal then hit return) and type...

diskutil list

If the command line whines about something, you may need admin privs. Try...

sudo diskutil list

If it says something like "username is not in the sudoers list, this will be reported" then you need to log in with an account that has administrative privs. Anyhow, that will give you a list of disks. If you have an internal drive and a USB stick mounted you'll probably see something like this:

/dev/disk0 (internal, physical):

   #:                       TYPE NAME                    SIZE       IDENTIFIER

   0:      GUID_partition_scheme                        *1.0 TB     disk0

   1:                        EFI EFI                     209.7 MB   disk0s1

   2:                  Apple_HFS Defiant                 999.2 GB   disk0s2

   3:                 Apple_Boot Recovery HD             650.0 MB   disk0s3

/dev/disk1 (external, physical):

   #:                       TYPE NAME                    SIZE       IDENTIFIER

   0:     Apple_partition_scheme                        *63.8 GB    disk1

   1:        Apple_partition_map                         4.1 KB     disk1s1

   2:                  Apple_HFS                         426.0 KB   disk1s2

You can identify what's what by the size... although disk0 will usually be your internal drive. You do NOT want to use that... be very careful with dd as it gives you many ways in which you can shoot yourself in the foot. Anyhow... in this example... /dev/disk1 (and you can see the note that it's external as well) with a size of 63.8 gigs, that's your USB stick. So... go to the site that Likhitha posted and download the iso. It will probably download to a path of...

/users/yourUserName/Downloads/

Look for the file name... the one I see is Win10_1511_English_x64.iso so... the full path to that is:

/users/yourUserName/Downloads/Win10_1511_English_x64.iso

Now you can clone it to your usb stick... first unmount the stick...

diskutil unmountDisk /dev/disk1 (DO NOT COPY AND PASTE... your disk number may be different) again, you may need to precede diskutil with the sudo command...

sudo diskutil unmountDisk.... blah blah

Now for the clone. Again.... DO NOT COPY ANY PASTE... DISK ASSIGNMENTS CHANGE! But... in the example above you would type...

dd if=/users/yourUserName/Downloads/Win10_1511_English_x64.iso of=/dev/disk1 bs=4m; sync

if is input file and of is output file. You don't need to reformat first... dd is going to completely blow over the volume with an image... so whatever format and partitioning schemes exist in that image... that's what you're going to get. This allows you to write bootable images, even if OS X does not support that file type (like ext4 in the case of Linux and although OS X does support reading NTFS volumes it can't write to them, but that's irrelevant with this tool).

Now... this is decidedly NOT the Macintosh way. I'm sure there's a GUI way to do this but... I've been spending a lot of time in the command line lately so, it's what I know.
