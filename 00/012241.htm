This line in the AddRevisionBar macro

    i = Selection.Information(wdVerticalPositionRelativeToPage)

should cause the variable i to contain the number of points from the top of the page to the current cursor location. Then the AddLine method uses that value as the vertical location of the top of the line (and i+j as the location of the bottom of the line).

To find out what value the macro is getting when you run it, add this line just after the "i = ..." statement above:

    Debug.Print i

from the top of the page). If the value you see is much larger than expected, that would push the line farther down the page. I'm not sure what would cause that.

stefan blom say:

Actually, the comment balloons will print just fine in black and white.

For comments to print "in line with text," you will have to make use of a macro such as the following:

Sub AddCommentsInlineWithText()
Dim c As Comment
Dim r As Range
For Each c In ActiveDocument.Comments
Set r = c.Reference.Duplicate
r.Collapse wdCollapseEnd
r.InsertAfter "[" & c.Range.Text & "]"
Next c
End Sub

Before running the macro, save the document. After you have run the macro, print (without markup) and close without saving.
