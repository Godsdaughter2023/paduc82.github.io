roady [mvp] say:

It all depends on which names are on the certificate, the DNS configuration for those names and what exactly is being queried. For using an Office 365 Exchange Online mailbox with Outlook 2016, there is no need to have a certificate of your own.

It sounds like your web server responds to any URL request and as the first URL that Outlook checks is https://domain.com/autodiscover/autodiscover.xml so your own certificate is being checked.

If this returns a proper (non-redirected) 404, then you shouldn't get this warning as Outlook will automatically start querying https://autodiscover/domain.com/autodiscover/autodiscover.xml next. For autodiscover.domain.com, you should have a CNAME entry in your DNS which points to an Office 365 server (it actually redirects via HTTP to another HTTPS location so you won't get a certificate mismatch error but that is another detail).

kevin d souza say:

Hi,

 

If you are referring to Outlook.com or hotmail account, then here's what you can do to block a contact.

 

1. Login to Hotmail or Outlook.com using your browser.

 

2. Click the "Options" button located to the top right of the page. Select "More Options" or "More mail settings" from the drop-down menu.  

 

3. Look to the "Junk e-mail" section when the Options page opens. Click "Safe and blocked senders." You can block a contact here.

 

4. Click "Blocked senders" when the "Safe and blocked senders" page opens.

5. Enter the address of a contact you want to block in the form next to "Blocked e-mail address or domain." After you add the e-mail address, click "Add to list." The contact will now be blocked from sending you e-mails.

 

Hope this information is helpful. If you have any concerns, please let us know.
