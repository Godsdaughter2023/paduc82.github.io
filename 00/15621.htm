Function Col2Num(ByVal ColName As String) As Long
  'Parse column identifier to column number => "ABC" to 731
  Dim j As Integer
  ColName = UCase(ColName)
  Col2Num = (Asc(Mid$(ColName, Len(ColName), 1)) - 64)
  For j = 1 To Len(ColName) - 1
    Col2Num = Col2Num + (Asc(Mid$(ColName, j, 1)) - 64) * 26 ^ (Len(ColName) - j)
  Next
End Function

Function Num2Col(ByVal ColNumber As Long) As String
  'Return column identifier for this column number => 731 to "ABC"
  Dim i As Integer
  Do While ColNumber > 0
    i = ColNumber Mod 26
    If i > 0 Then
      Num2Col = Chr(i + 64) & Num2Col
    Else
      Num2Col = "Z" & Num2Col
      ColNumber = ColNumber - 1
    End If
    ColNumber = ColNumber \ 26
  Loop
End Function

a. user say:

Now that you've got to the bottom of that, my previous message starting 
"Things can be confusing..."
may make a bit more sense!
 But just to try to fill in a few gaps for you...

1. The whole business of "properties" in Office documents etc. is historically, a mess. 

2. At least some of the built-in properties can indeed be inserted in several ways, e.g.

a. { AUTHOR }
b. { INFO AUTHOR }
c. { DOCPROPERTY AUTHOR }
d. The "Content control" version of { AUTHOR }, which is inserted via Insert->Quickparts->Document properties.
e. A custom content control, that points to the same XML content item as (d)


3. For 5 of the built-in properties, AUTHOR, COMMENTS, KEYWORDS, SUBJECT, and TITLE (I think!) you can also set a new value using the type (a) field or the type (b) field, e.g.

{ AUTHOR "new author" }

These 5 are special because
 a. they were there in the original versions of Word for Windows (whereas, for example "Category" was not)
 b. their values are not intended to be maintained by Word (cf. { NUMWORDS }) and therefor read-only.

4. INFO is, I suspect, little used, as all its values can be inserted using other field types. Further, it has been "deprecated" in the ISO standard for the OOXML format. I wouldn't use it.

3. I wouldn't personally choose to use DOCPROPERTY to insert the value of { AUTHOR } because
 a. the built-in property value would be overridden if you created a document property called AUTHOR, at which point things may become very confusing.
 b. the user-created properties that you can display using DOCPROPERTY (previously known as "Custom Document Properties," and now as "Advanced Properties", are also arguably in decline, because there is a new type of property called a "Custom Property" that is defined as part of a "Content Type" on a Sharepoint server. You can set the values of such properties independently of the document, and when you retrieve the document from Sharepoint, it creates custom XML containing the property definitions and values. Such properties can only be inserted as Content Controls. These are now the only type of user-defined properties that 
 - are connected to a custom data store in the document
 - appear automatically in the list in Insert->Quick Parts->Document Properties.

The other properties in that list are all content controls, and include
 - some of the built-in properties that can also be inserted as fields, e.g. AUTHOR
 - one or two of the older properties that never had a field version, e.g. CATEGORY
 - a set of new "built-in" properties called Cover Page Properties (Company Fax etc.) that have no field version. The Custom XML for Cover Page properties is created as required (i.e. as soon as you insert the property), but this handy mechanism is reserved for Word - you can't set up your own properties that work in this way.

Another significant difference between the "Advanced Properties" (DOCPROPERTY properties) and all the other property types, is that you cannot connect a content control to the value of a custom-defined "Advanced Property" in the XML store.

Recovering from a failed firmware update. - Microsoft Community

psychomania---01 say:

Recovering from a failed firmware update.

Technical Level : Intermediate

Summary

Details

Occasionally things go wrong when using the Nokia Software Updater (NSU). The exact cause is often unknown but it can be caused by the NSU, the phone or even the computer itself.

The aim of this guide is to offer users a few tips and tricks that may aid recovery. The guide applies to most nokia phones, except the hard reset which only applies to S60 smartphones.

So how do you know if an update has failed?

A failure can show itself in a number of ways, the most common one is where the NSU freezes. If it does this during the first part of the process where it downloads the firmware from nokia your phone will be ok. You will see that your phone is still switched on so it's safe to cancel the process or just remove the cable from the phone.

If it freezes during the second phase when the computer is transferring the firmware to the phone this can be much more serious. When this happens leave your phone for at least 15 minutes before taking any further action. If after that time nothing has happened all you can do is pull the cable.

Another failure is where your computer crashes during the second phase. This is usually the worst type of failure and often leaves the phone unrecoverable, but still it's worth trying.

What can I do to recover from this failure?

The first thing to try is to remove the phones battery and leave it out for at least five minutes. Whilst doing this take the opportunity to reboot your computer in case you need it to help with recovery.

Now re-insert the battery and try to switch your phone back on. If it works then all is good but before trying to update again there is a few things you need to check. Make sure that your computer is fully up to date including the latest service pack and disable all non-essential programs that are running. Also disable any anti virus and anti spyware software that you may have, now try again.

If removing the battery failed to make it work then prepare your computer as mentioned above and run the NSU again. Follow all instructions and prompts that it gives you carefully as it may offer to recover your phone. If it doesn't try various combinations of connecting your phone whilst it's switched on, off and also try removing and reinserting the battery whilst it's connected. Make sure you wait in between each attempt to give the NSU time to detect the phone. This classic thread is a good example of someone who succeeded in recovering their phone. It's a good idea to keep trying and trying.

What if that method doesn't work?

The next thing to try is a hard reset. This method only applies to S60 smartphones with keypads (not touchscreens):

1. Switch off the phone.

2. Hold down the following three buttons: Green (the call answer button), * button, and '3' button.

3. While holding these buttons, hold the power button for a second to switch on the phone.

4. Release when the phone shows the nokia hands logo or shows other signs of life like the language selection screen.

It may take a few attempts and very flexible fingers.

Notes: On the N78 use the silver multimedia key instead of the green call key. On the 5800XpressMusic (also for the X6 & 5230/5233/5235) see this post for instructions on how to hard reset it.  For an N97/mini see this link.

So none of that worked, what happens next?

Unfortunately you have now tried everything possible to recover from the failure yourself. The last thing for you to do now is to contact a nokia care point in your country for help. If your phone is under warranty they will repair it for free, possibly while you wait.

You can find your nearest care point by using these links:

Care points/service centres and repair info:

UK • Europe • Asia-Pacific • USA •
Canada • Middle East and Africa

Elsewhere: Click here, select your country, go to the support section, then select repair.

I hope you have found this guide useful and have managed to solve your problem.
