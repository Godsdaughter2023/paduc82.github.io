greginmich say:

External factors aren’t usually to blame for the fact that MSE’s scheduled scan doesn’t run reliably. The scan is actually unreliable by design. First of all because its scheduled task isn’t capable of waking up a sleeping PC in order to run the scan – so you not only have to reschedule the scan for the daylight hours, you also have to pick a time when the PC will actually be in use. Otherwise, the scheduled start will be missed because the PC will have idled to sleep.

Now, in the original design, the possibility of missing a scheduled start was compensated for with a “catch-up” scan that would run the next day on startup when the scan's scheduled start was missed – but the developers later decided that this catch-up scan was too intrusive – so they disabled it in the registry; leaving MSE users with an inherently unreliable scheduled scan:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/ms-security-essentials-unreliable-schedule-running/fc048489-a767-4881-8e06-2c5ab949bb9a
