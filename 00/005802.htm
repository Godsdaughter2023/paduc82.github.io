Private Sub Contact_Person_BeforeUpdate(Cancel As Integer)

ken sheridan say: 1.  You should do this in the form's BeforeUpdate event procedure, not the control's.
2.  You probably need to test for Null rather than a zero-length string, but you can cover both eventualities by calling the Nz function.
3.  You need to set the return value of the procedure's Cancel argument to True.  
4.  You are missing the closing line of the If.....End If construct.
5.   Finally, when calling the MsgBox function without returning a value the arguments should not be within parentheses.  So, try this:

    If Nz(Contact_Person,"") = "" Then
       MsgBox "Please enter Contact Person's Name", vbOKOnly, "Missing Data"
       Cancel = True
    End If

Note, however, that you will have a problem if the user ties to close the form while the record is incomplete.  In such a situation I would normally force the user to save a record via a command button before the form can be closed, again via a command button.  You'll find an example as SaveDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

This little demo file includes validation on the first and last name controls in the form.

In order for the time of an appointment or meeting invitation to be reflected properly, settings in Windows and in Outlook must be correct and they must match. There is no reason to suspect that microsoft servers have the wrong time settings since meetings and appointments time is always local to your mail client.

In Windows/Outlook:

Open Windows Control Panel. Change "View by" in the top right corner to Small icons. Click on Date and Time to open it.

In the Time Zone section, make sure that the time zone and Daylight Savings Time settings are correct for your region.
