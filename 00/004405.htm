john - project say:
Frustrated,
This usually happens when a user formats an individual field cell instead of taking advantage of group formatting. For example, say you want to change the font color on all summary line. One way to do that is to select each summary line and then change the font color for each line. Or, you could use the Text Styles function to format all summary lines at one time. The first method "eats up" the direct formatting limit because each format operation is treated as a separate unique action.

Another cause of reaching the limit is that some users really like to "dress up" the display and go overboard with formatting. I'm not saying this is the case in your file, but as a general rule, less customized formatting is better.

You can clear custom formatting by first selecting the whole file, or any one or more cells that have custom formatting and then go to Task/Editing group/Clear/Clear formatting.

However, before you do anything else, make sure you have SP1 installed. You can get it at, http://technet.microsoft.com/en-us/office/ee748587.

Enter below formula as an array formula (CTRL+SHIFT+ENTER in cell C2 & copy down:

=IFERROR(SMALL(IF(($I$3:$I$100<=DATE(2017,10,31))*($J$3:$J$100>=DATE(2017,10,1)),ROW($I$3:$I$100)),ROW(1:1)),"")

This returns multiple ROW NUMBERS where the Start Date-End Date include even one date in the month of October 2017 ie. any date between 1 Oct 17 to 31 Oct 17 date is covered between Start & End Dates in columns I / J. The texts syncs to the primary Microsoft account configured on the phone.

It is possible to have more than one phone on the same account, and have texts sync between all the phones, but it is a Microsoft account sync feature.

If texts are deleted from the phone, they should be deleted from the Microsoft account on the next sync, normally.

All I can suggest is make sure to keep the phone fully updated, delete the unwanted texts on the phone as they appear, and make sure the primary Microsoft account on the phone synchronizes without errors. Only the Microsoft Answer Desk supporters can escalate the issue to somebody who has access to your Microsoft account.

A quick and dirty workaround is to abandon the Microsoft account, hard reset the phone, and set it up on a new Microsoft account. john mcghie say:

Hi Gordon:

The "out of memory" error has not been updated since the most likely reason why things didn't work was that the computer had run out of memory: it's an old DOS-based error message.

What it normally means is "I was unable to write data where I expected to write it" or "I was unable to find the data I expected in the computer's memory".

In your case, I suspect what is happening is that you are opening a document from a website and Word is trying to write something back to the website, which will fail and produce the "Out of memory" error.  You would get the same error if you opened the document from a CD: Word would be unable to write back.

What I could suggest you try is to Save As the document to a new file name on your local drive.

It should then happily close.

Or, close Word and when prompted to save, say "No".

Either way, provided any save goes to a place you are allowed to write, the error won't occur.

If this DOESN'T solve your issue, please post back with an exact click-by-click description of how you are attempting to proceed.  I need to know the exact path you're taking through the various buttons and controls, and the exact text (including numbers) of any dialogs you see.
