stareg say:

Database backup Access 2010

Hello,

 

              I have finished my access 2010 database and used the run time application to generate an executable database to my users. Currently i have this exe file. My quiry is How can i enable my users to backup their database frequently from this exe file ?

 

thanks and waiting for your urgent reply,

djsteele, mvp say:

Recognize that you have NOT created an executable for your users. What you have is an exe file that will expand to install the Access runtime (msaccess.exe), all necessary supporting files and your application on the user's computer. Your application will not have been changed in any way, shape or form. Assuming you've split your application into a front-end (containing the queries, forms, reports, macros and modules), linked to a back-end (containing the tables and relations), yes, it's relatively straight-forward to add code to your application that will backup the back-end, and there should be no need to backup the front-end.

What I do is have a switchboard-type form that's unbound. As long as that's the only form that's open (so that there are no connections open to the back-end), I rename the existing back-end, then compact it to the proper file name (thus creating a back-up and a compacted database at once)

tom van stiphout say:

As Doug said, an Access application is not an exe. Rather it is an accde or mde processed by msaccess.exe.

Access applications should always be split between front-end (FE) and back-end (BE). We are assuming you did that. The BE would be located on a server accessible to all users. Each user has her own copy of the FE on her workstation.

Backing up the FE is not necessary: if it becomes corrupt, give her another one. There is no data in the FE.

Backing up the BE is the job of the system administrator, not the average user. Typically we would setup a job in the Windows Scheduler on the server to backup the database automatically every night. Or we would have specialized backup software (e.g. Windows Backup) that can be scheduled to do that as well or better.

Backups of the BE should be taken when all users are out of the application, e.g. overnight when all workstations are off.

There are exceptions to the above rules, but you did not give us any indication they apply in this case.

ken sheridan say: I would always recommend using a commercial back-up application for this, as this will include the means for scheduling back-ups, verifying a back-up etc.  But if you do want a cheap and cheerful home-grown solution you'll find an example as BackupDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

This backs up the back end file from within the front end, and includes methods for a single back-up or multiple dated back-ups; the latter is preferred.
