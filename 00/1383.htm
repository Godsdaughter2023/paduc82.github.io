The coupon, wherever it came from, is now saved in the template named Normal.dotm. To remove it, you need to open that template -- using the Open command on the File tab -- and delete the text and/or picture, then save and close the template.

When you click File and then Open, the left side of the dialog may contain a "Templates" item in the Libraries group; if it's there, click it to get to the correct folder. If you don't see "Templates" there, go up to the address bar at the top of the dialog and paste in

When the file opens in the Word window, select everything you see and press the Delete key.

Save and close the file. New blank documents should now really be blank.

dale howard [mvp] say:

Gabriele --

Good question, my friend.  A negative Total Slack value indicates that a task has missed an inflexible constraint date (such as a FNLT constraint) or a task has missed a Deadline date.  In this situation, Free Slack will always show a 0d value, even though the Total Slack field shows a negative value.  To detect missed constraints and Deadline dates, you could create a custom filter that shows tasks with a Total Slack value less than 0.  Then, as a part of your process of updating the project each week, apply this new filter at the end of the process to see if you have missed constraints or Deadline dates.  Hope this helps.

Jussissa, you can absolutely figure out the originating IP of the message. You have to look at the sources of the message for that. 

The easiest by far is to drag the message to the Finder and open up the resulting .eml message in a decent text editor (like BBEdit - even in the free version).

You'll see all the headers there, including the address of all the servers the message went through.

That being said, it won't help at all to corral spam since they usually don't all come from the same address. For these sorts of things, you're much much better of with a good third party spam filter.
