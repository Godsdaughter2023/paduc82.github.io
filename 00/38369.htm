Thanks
jeeped say:

The 'program' files on your desktop and menu items are actually shortcuts (e.g. .LNK) files that point to the actual programs or documents. The fact that they all 'look' like Adobe is because they have become inadvertently associated with the Adobe program.


I've seen this happen when upon opening a program through a shortcut (e.g. .LNK file), you are asked what program to open it with. After selecting Adobe (or some other program) via the Select a program from a list of installed programs, and checking the Always use the selected program to open this type of file (circled in red below), the document or program referenced by the original shortcut opens in Adobe as it should.


              


The trouble is that now all shortcuts (e.g. .LNK files) on the desktop or throughout Explorer will want to be opened by Adobe and use the Adobe icon to display the shortcut regardless of what the shortcut was originally intended to open. This is known as a file association and default icon. The same condition could occur if you performed those 'Open With' actions on a .JPG or even an .EXE, but the .LNK explanation is far more likely for items on the desktop or menu.


The good news is that a small free utility has been written to undo the .LNK file association that was created. There is also a registry hack to accomplish this, but I recommend the unAssoc_1_4 utility for its ease of use.


I would suggest using the UnAssoc utility to remove the errant file association. I used to send people to another site to download this utility, but that site has become confusing with all the ads and false download links. I've put a copy on my own SkyDrive here.


                                    

Just download, then use the utility to unassociate the .LNK files for the User or globally Delete the file type and let Windows rebuild a new one to defaults.
