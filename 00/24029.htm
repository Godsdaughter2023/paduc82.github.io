How to Download PDF email attachments on your Windows Phone 8.1 and - Microsoft Community

johan van mierlo say:

How to Download PDF email attachments on your Windows Phone 8.1 and attach them in new emails

Technical Level : Intermediate

Summary
Windows Phone has been great when it comes to using Office documents. However when it comes to PDF's there is another story. Most of the times when you open an attached PDF on your phone it is saved within the two PDF app readers(Adobe reader and PDF Reader) available and you can't really do anything with it besides reading. The only way to share these PDF's is by forwarding the original email it was attached to, but what if you didn't want to forward the whole email? There is way to work around it but it does get the job done for now. I had written this How to on my site but was asked to add to the Wiki here.
Details

Microsoft just released their own app "Files" to manage files on your Windows Phone internal and SD storage. You have to see it as the File explorer in Windows. Microsoft. Whit the app "Files" you can now share any file within the app as an attachment in your email. The easiest way to drop the files in these folders is by connecting your Windows Phone to your PC and and use "Copy and Paste" on your PC since you can see your Windows Phone as a drive.

But when you are on the road and you are getting a PDF as an attachment in an email you will find out that for now you can't "Tap and Hold" the pdf file in your email and select "save" because it is grayed out.

Opening the PDF file only allows you to read it in the PDF reader apps and is stored in their internal storage which you can't see in your "Files" app. Your only option at this point is to forward the whole email (since you can't edit the body of the forwarded email) with the attachment to someone else. But if you really want to save this file in your accessible storage and then attach it as a file (not a link) I have found just the way for you.

Steps how to download PDF email attachments on your Windows Phone 8.1 and attach these PDF's in new emails

Before you start make sure you download the following two apps in the Windows Phone Store:
-Files (developed by Microsoft)
-PDF Reader (developed by Microsoft) If you use Adobe Reader it can cost you some money to upload and share files.

Step 1: Open pdf file from email in “PDF reader”

Step 2: In “PDF reader” tap the screen and then the 3 dots and select “Share”

Step 3: Sign in to your Onedrive account to create a link and share it your own email.

  

Step 4: Open your email and click the link and select “Save”. This will download it your “downloads” folder on your phone or sd card.

 

Step 5: Open “Files” and find your document in the “Downloads” folder

  

Step 6: Tap and hold the file and select “Rename”. You just have to remove some numbers in front of the file to get the original file name back.

Step 7: Tap and Hold the file again and select “Move to” to move the file to your specific folder you want.

  

Step 8: To send this file as an attachment to some else you need to open the “Files” app and find the file first. Tap and Hold the file name and select “Share”. Select the email account you want and the PDF is an attachment in that email.

  

For now this will be the a cumbersome way to get your Pdf files from emails stored independently on your Windows Phone 8.1 and allow you to send them as file attachments in new emails.

john p spencer say:

I assume the C-Month is a string with the name of the month from your comment about alphabetic sorting.

Try using this expression to sort on.
     Month(Cdate([C-Month] & " 1"))

It will error on you if C-Month is null or not a valid month name.

SELECT [Time Recording].EM
, [Time Recording].[C-Year]
, [Time Recording].[C-Month]
, Sum([Time Recording].[C-Hours]) AS [Sum Of C-Hours]
FROM [Time Recording] INNER JOIN EM ON [Time Recording].EM = EM.ID
GROUP BY [Time Recording].EM, [Time Recording].[C-Year], [Time Recording].[C-Month]
ORDER BY [C-Year], Month(Cdate([C-Month] & " 1"))

qa guy commelec say:

If  [Time Recording].[C-Month] is a text field then you need to create a table of months with a number field for sorting.

tblMonthNUM –

Mon NUM

Jan   1

Feb   2

Mar   3

 

SELECT DISTINCTROW [Time Recording].EM, [Time Recording].[C-Year], [Time Recording].[C-Month], Sum([Time Recording].[C-Hours]) AS [Sum Of C-Hours]

FROM (tblMonthNUM INNER JOIN[Time Recording] ON tblMonthNUM.Mon = [Time Recording].[C-Month]) INNER JOIN EM ON [Time Recording].EM = EM.ID

GROUP BY [Time Recording].EM, [Time Recording].[C-Year], [Time Recording].[C-Month]

ORDER BY[Time Recording].[C-Year], tblMonthNUM.NUM;

marshall barton say:

The question of interpreting what users mean when they enter something like 9/10 is an open debate.  If the two numbers are acceptable month/day or day/month, then Access will add the current year.  If one of the numbers is >31, then Access will use that as the year and the other number as the month (unless the other number is > 12, then error).  Sooo, it all depends on what you enter, 9/31 = 9/31/2010, but 9/32 = 9/1/1932.

OTOH, you can do all kinds of things if you introduce some VBA code into the mix and just look at what they entered instead of what access thought was intended.  A simple example, without error checks, that allows users to type m/yy could be this kind of code in the text box's AfterUpdate event:

Dim pos As Integer
    pos = InStr(Me.Text0.Text, "/")
    Me.Text0 = CDate(Left(Me.Text0.Text, pos) & "1" & Mid(Me.Text0.Text, pos))

Then you can use the Format property as John said to display it however you like.

That is an Update error. During the creation process, the Wizard needs to connect to download the latest bits for the bootable media.

This may help: http://answers.microsoft.com/en-us/windows/forum/windows_other-windows_update/error-code-800700c1/1506421e-2d26-4da6-8914-8328f438315c

It sounds like you have experience with the WDO process already, so this may be an issue with connectivity from the machine you are on or there's a back end issue. Do note that the architecture of the infected machine and the architecture of the creation machine must match -- x86 or X64. The version of Windows is not an issue. If her machine is x86, your machine was also be x86 and the WDO install on your machine would be the x86/32 bit version, for example.

For the Java infection, though, I'm not sure that a rescue scan is needed.

Here's what I suggest for a Java infection:

Do the following:

1. Empty the Java cache

  • How do I clear the Java cache?

2. Update Java from www.java.com

3. In Control Panel/Programs remove any old versions of Java that might remain

4. Run a full scan with MSE
