graham mayor say:
No - for that you need a different approach. The following macro will open and close all the documents in a folder in turn. Add your code (or a call to your macro) where indicated and the document will be saved with the same name in a sub folder of the selected folder called "Modified"

Sub BatchProcess()
Dim strFilename As String
Dim strPath As String
Dim oDoc As Document
Dim fDialog As FileDialog
Dim icounter As Long
Dim opara As Paragraph
Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)

With fDialog
    .Title = "Select folder and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , _
               "List Folder Contents"
        Exit Sub
    End If
    strPath = fDialog.SelectedItems.Item(1)
    If Right(strPath, 1) <> "\" _
       Then strPath = strPath + "\"
End With

If Documents.Count > 0 Then
    Documents.Close SaveChanges:=wdPromptToSaveChanges
End If
If Left(strPath, 1) = Chr(34) Then
    strPath = Mid(strPath, 2, Len(strPath) - 2)
End If
strFilename = Dir$(strPath & "*.doc?")

While Len(strFilename) <> 0
    WordBasic.DisableAutoMacros 1
    Set oDoc = Documents.Open(strPath & strFilename)
    '
    'Do what you want with oDoc here
    '
    On Error Resume Next
SaveDoc:
    oDoc.SaveAs2 strPath & "Modified\" & strFilename
    If Err.Number = 5152 Then
        MkDir strPath & "Modified\"
        GoTo SaveDoc:
    End If
    oDoc.Close SaveChanges:=wdDoNotSaveChanges
    strFilename = Dir$()
    WordBasic.DisableAutoMacros 0
Wend
End Sub

graham mayor say:
The following variation will open a new document based on a template you select for each document in a folder, insert the document and save it as before. It is possible to create a function that will process all sub documents, but I would recommend that you don't.

Sub BatchProcess()
Dim strFilename As String
Dim strPath As String
Dim strTemplate As String
Dim oDoc As Document
Dim fDialog As FileDialog
Dim icounter As Long
Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)

With fDialog
    .Title = "Select folder and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , _
               "List Folder Contents"
        Exit Sub
    End If
    strPath = fDialog.SelectedItems.Item(1)
    If Right(strPath, 1) <> "\" _
       Then strPath = strPath + "\"
End With

Set fDialog = Application.FileDialog(msoFileDialogFilePicker)
With fDialog
    .Title = "Select Template"
    .AllowMultiSelect = False
    .InitialFileName = Options.DefaultFilePath(wdUserTemplatesPath)
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , _
               "List Folder Contents"
        Exit Sub
    End If
    strTemplate = fDialog.SelectedItems.Item(1)
End With

If Documents.Count > 0 Then
    Documents.Close SaveChanges:=wdPromptToSaveChanges
End If
If Left(strPath, 1) = Chr(34) Then
    strPath = Mid(strPath, 2, Len(strPath) - 2)
End If
strFilename = Dir$(strPath & "*.doc?")

While Len(strFilename) <> 0
    WordBasic.DisableAutoMacros 1
    Set oDoc = Documents.Add(strTemplate)
    oDoc.Range.InsertFile strPath & strFilename
    '
    'Do what you want with oDoc here
    '
    On Error Resume Next
SaveDoc:
    oDoc.SaveAs2 strPath & "Modified\" & strFilename
    If Err.Number = 5152 Then
        MkDir strPath & "Modified\"
        GoTo SaveDoc:
    End If
    oDoc.Close SaveChanges:=wdDoNotSaveChanges
    strFilename = Dir$()
    WordBasic.DisableAutoMacros 0
Wend
End Sub

If you want to set up your family, you will need to sign in using a parent or guardian's Microsoft account.   1. Sign in on this link with a parent or guardian's Microsoft account.

    rich007a say:

    Yes, I get exactly that too.  D'oh!

    Here's the best I can come up with as a work around.  Add this to the ThisWorkbook module:

    Private Sub Workbook_BeforeClose(Cancel As Boolean)
        Dim linestr As String
        On Error Resume Next
        Open "\\YOUR\NETWORK\PATH\HERE\FileInUseBy.txt" For Input As #1
        Input #1, linestr
        Close #1
        On Error GoTo 0
       
        Open "\\YOUR\NETWORK\PATH\HERE\FileInUseBy.txt" For Output As #1
        Print #1, linestr
        Print #1, "The workbook is now CLOSED"
        Close #1
    End Sub

    Private Sub Workbook_Open()
        Open "\\YOUR\NETWORK\PATH\HERE\FileInUseBy.txt" For Output As #1
        Print #1, "The workbook was last opened by " & Environ("username") & " at " & Now
        Print #1, "The workbook is now OPEN"
        Close #1
    End Sub

    Update the network path to match your file's path.

    As a work around, you'll be able to open the text file and see who has the Excel file open.

    Hope that helps.
