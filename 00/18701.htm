ken sheridan say: For this sort of thing I'd suggest writing a little function to return the amended street address.  For this I've assumed that the table of suffixes is named StreetSuffixes and is like this:

StreetSuffix    StreetType
Rd                  Road
Ave                Avenue
St                   Street
etc

Assuming that the suffix will always be the last part of the street address value, and preceded by a space, the function would be like this:

Public Function GetStreetAddress(strStreetAddress As String)

    Dim strSuffix As String
    Dim varStreetType As Variant
    
    strSuffix = Mid(strStreetAddress, InStrRev(strStreetAddress, " ") + 1)
    varStreetType = DLookup("StreetType", "StreetSuffixes", "StreetSuffix = """ & strSuffix & """")

    If Not IsNull(varStreetType) Then
        GetStreetAddress = Left(strStreetAddress, InStrRev(strStreetAddress, " ")) & varStreetType
    Else
        GetStreetAddress = strStreetAddress
    End If
    
End Function

You can the easily update the original table like this, for which I've assumed the table is named AddressList and the column in question is named StreetAddress:

UPDATE AddressList
SET StreetAddress = GetStreetAddress(StreetAddress)
WHERE StreetAddress  IS NOT NULL;

Because the function makes a number of assumptions, it might fail in some cases, e.g. street names like 'Salisbury Ave West', or if St. has been entered rather than St without a full stop.

If you really want to make a new table, rather than executing a Make table query, make a copy of the table and execute the query on the copy of the table.  That way you preserve the indexes etc.

Hi Steve,

 

That depends on when you do them, what else is happening at the time, whether or not you are actively using the computer when the scan runs, your available resources, and how much you really care about how long the scan takes (and when you think about it, how much does that REALLY matter).  There's no real "best recommendation" or "official guidelines" to set it.  It's more like something each person needs to determine for themselves (like the touch sensitivity or click speed of mouse buttons).  But if I read you correctly, you're interpreting the restriction backwards - though I may just not be undertanding you right.  Anyway, the following should help you understand and learn how to decide what's best for YOU (you can think of it like instructions on how to set the mouse button sensitivity or click speed settings).

If you prefer to run scans at night when nothing else is running and you aren't using the computer yourself, you can set the throttle higher to 50-60% or even more (80% is fine) since there will be nothing affected by it using that much of the CPU. Then the scans will go quite faster (to answer that specific question).

You can also tackle that problem by changing your startup programs to remove things you don't really need and closing programs you aren't using and that too may bring the system performance back in line too. It's a matter of finding the right balance for your situation. THAT is then what it should be set at for you.  What I'm really saying is there is no hard and fast set of rules and too many factors and differences to cover each possible situation and the desires or needs of the user.  It seems to me the two criteria to consider are system performance (can you do what you want to do without the system slowing down due to the scan) and scan completion time (which seems to me to matter only if you do it in the background during the day and want it to finish before putting it into sleep or hibernation or shutdown at night - otherwise, what real difference does an hour or two or even 5 either way make if all else is working fine and the scan isn't preventing you from doing some other CPU-intensive task you want to run after it is complete or you have need to logoff or reboot frequently for some reason).

 

Remember to consider your power saver settings as only a scan running will be perceived as the computer being idle and will activate set power saver actions like sleep or hibernation or shutdown unless you change them so the power saver action is longer than the scan so the scan has time to finish (and give it some good grace period just in case or the scan will cancel and everything already done will be as if it never happened - it does not pause to be restarted when the computer is restarted or awakened - I personally leave it on never (actuallly the High Performance plan) just to be sure unless running on battery and then I switch to the battery plan but I don't and suggest you don't run scans on the battery as they can often exceed the battery time, especially since they use above-average power as the system is running hard constantly during the scan, and since you don't want to cancel the battery shutdown settings and cause an unwanted and unnecessary hard crash which may otherwise occur, you don't want to risk the scan getting to 95% and then being cancelled because of the battery and have to start over again as it won't be saved and even if something was detected, no action will be taken and you may not even know).

 

I hope this helps.
