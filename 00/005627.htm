How does this formula for finding the last entry in a range of text - Microsoft Community

anomalous dna say:

How does this formula for finding the last entry in a range of text formatted data work (correctly)?

Can someone help explain why the following formula (in cell A1) returns the last value entered within the range A2:A11 when the reference cells are formatted as text?

=INDEX(A2:A11,MATCH(REPT("z",255),A2:A11))

When I've played around with this formula it seems that any value >255 for the number_times argument will error-out.  If you place 1 as the number_times argument it will bring a value that isn't the last in the range if a cell value of "a" preceeds "z".  If you change the number_times argument to 2, it will bring back the "z" (I get the logic going on here).  I also understand that the MATCH function doesn't have a match_type selected so it will follow a match type of 1.  However, regardless of the alpha/numerical sort, the formula will bring back the final entry, even if the final text is "a" and previous text values are >"a".  I'm stumped.

tom ogilvy say:

Cells were limited to 255 characters in Excel 95 and earlier.  In Excel 97 the limit was increased to ~31K

If you get one of the old developers references that explains how "records" are stored in the old binary file format, you find that for a text entry in a record for storing a cells information, there is only provision to store 255 characters.  A pointer was added in that record format that points into a string array type record where the remainder of the record's text is stored.  While I have not seen it documented anywhere, I am sure the match function will only look in the record associated with the cell.  This was probably retained for speed considerations.  Regardless of the new XML format, I would bet that in memory, excel still uses the old binary organization or something similar to it.  Thus the 255 character limit.

t. valko say:

Text:

Test

Test1

Test2

1Test

Alpha

Zelda

2Test

One of my main questions is why the REPT maxes out at 255 repetitions before erroring out at 256 and greater.

Ok, assuming that is all the data in the range then you can use this formula to return the last TEXT value in the range:

=LOOKUP("zzzzz",A2:A11)

With this version:

=INDEX(A2:A11,MATCH(REPT("z",255),A2:A11))

It isn't the REPT function that's causing the error. Try this, enter this formula in a cell:

=REPT("z",300)

Assume you enter that formula in cell A1 then enter this formula in A2:

=LEN(A1)

The result will be 300.

REPT will return an error if the number of characters in the string is >32767. See Excel help.

So, the error you're getting is coming from the "lookup" function, in this case, the MATCH function. All of the "lookup" functions will return the error when the lookup_value is >255 characters.

MATCH LOOKUP HLOOKUP VLOOKUP

That REPT("z",255) is just an arbitrary lookup_value that is assumed to be greater than any text value in the range. There is a very good chance that that is true so the formula works as expected.

However, I think that this is overkill. Take you posted data as an example:

Test
Test1
Test2
1Test
Alpha
Zelda
2Test

All you need is a lookup_value that is greater than any value in the data. Based on the sample data, the highest value entry is Zelda. To see which value Excel evaluates as the highest text value all you need to do is to sort the data in ascending order:

1Test
2Test
Alpha
Test
Test1
Test2
Zelda

So, you simply need a lookup_value that is >Zelda. Like "zz".

=LOOKUP("zz",A2:A11)

=INDEX(A2:A11,MATCH("zz",A2:A11))

The use of REPT has been around for a long time. I don't know who is responsible for introducing that method but it does work although, as I mentioned above, I think it's overkill. The best lookup_value to use depends on the data in the range.

The same basic technique also applies to return the last numeric value in the range. You simply need a lookup_value that is greater than the maximum numeric value in the range.

Instead of using the REPT function to generate a gigantic lookup_value, it's a lot easier when looking up the last numeric value. We can use scientific notation as shorthand to write very large numbers.

=LOOKUP(1E100,A2:A11)

1E100 is shorthand for 1 followed by 100 zeros. That's a huge number!

If you look around in Excel help forums you will find that formula written like:

=LOOKUP(9.99999999999999E+307,A2:A11)

9.99999999999999E+307 is largest numeric value that can be manually entered in Excel. This is assumed to be greater than any numeric value in the range. I also think this is overkill.

I use the shorter and easier to read 1E100. Both of these formulas will do the exact same thing:

=LOOKUP(1E+100,A2:A11)
=LOOKUP(9.99999999999999E+307,A2:A11)

Which one would you rather use? That's one of them there rhetorical questions! <g>
