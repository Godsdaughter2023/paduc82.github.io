***MICROSOFT***  if you are reading this, pay attention!

Microsoft offers 2 platform versions of its products:  32bit or 64bit.

Now, just because you have a 64bit operating system does not mean you must install the 64bit version of software. In fact, despite Microsoft Office being offered in 32bit and 64bit versions, the 32bit version is recommended... BUT NOT REQUIRED.  It is NOT explained prominently as to why you would want 32bit vs. 64bit (you have to dig to find it).  I have learned that the 64bit version is available for users who have very large working file requirements.  I'm not clear on what "very large" exactly is, but it seems to be when files are multi-hundred megabytes in size.  This probably also comes into play when you are using the collaborative features of Office on very large document authoring projects with multiple authors involved.

WHEN I FIRST INSTALLED OFFICE 365, I allowed the default settings to be used, which meant 32bit would be installed.  Everything was fine at first.  Where the problems arose was when I found myself MISSING PICTURE MANAGER, and wanted to install it.  I found that some people were enterprising enough to figure out that the PICTURE MANGER is available as a free option with Microsoft Sharepoint 2010 viewer.  So, I installed it.  I didn't realize that I had downloaded the 64bit version.  But Microsoft allowed me to install it.  And it worked.

"MICROSOFT OFFICE 365 64bit has ISSUES".  It apparently is an "undocumented" trade-off for being able to handle very large files.  And well... that's when I ran into these program start problems.  It was not a complete blockage of use, but the experience would be variable.  Sometimes a double-click would launch an Office program successfully, other times even 5 double-clicks in succession wouldn't get it to launch and I'd have to reboot.

So... the upshot?  Use Office 365 32bit, ALWAYS... unless you run into file size management issues.
