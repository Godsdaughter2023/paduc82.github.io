Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    If Target.Cells.Count > 1 Then Exit Sub
    If Not Intersect(Target, Range("A:A,B:B,D:D")) Is Nothing Then
        Calendar1.Top = Target.Top + Target.Height
        Calendar1.Left = Target.Left + Target.Width / 2 - Calendar1.Width / 2
        Calendar1.Visible = True

End Sub

    ActiveCell.Value = (Calendar1.Value)

jay freedman say:
The main difference is that "filling in forms" protection also disables a number of useful features such as spell-checking. If you use the ability to protect some sections but not others, you can't use any of the picture manipulation tools, even in the supposedly unprotected sections. There are some other odd inconveniences.

With the "no changes"/exceptions type of protection, you just indicate which areas should be editable, and they behave almost completely normally. The one feature that's disabled is Track Changes.

When "filling in forms" protection is used with content controls, there have been some odd behaviors, although they're inconsistent -- they may show up in one document and not another. I've found it easier just to avoid that combination.

The difference between Rich Text and Plain Text is that a Rich Text content control lets users format parts of their entries differently from other parts -- for example, one or two words can be made bold or italic or a different color. In a Plain Text control, all the text must be the same, such as all bold. Also, you can insert anything into a Rich Text control that you can into a regular document, including tables, pictures, hyperlinks, other content controls, etc.

One annoying difference is that, while you can use the Tab key to move from one Plain Text control to another control of any type, the Rich Text control accepts the Tab key for inserting a tab character. So you can't tab out of a Rich Text control.
