I actually came up with another workable solution, as noted below:

 

Private Sub CmdExport_Click()          'Export the CLP Format File -- using the dates input on the form

Dim FileExists As Boolean

    DoCmd.SetWarnings False
    FileExists = False
    
    DoCmd.TransferText acExportDelim, _
        "Export Spec", "Qry_FINAL", _
        "\\server\share$\directory\sub-dir\CLPFname.txt", True, ""
   
                                       ' Checks to see if properly named CLPFname already exists,
                                       '     if yes, MsgBox will tell User which file exists, and
                                       '     instructs them to go there & rename MANUALLY
                                       ' Existing file says previous file DID NOT properly process
                                       '     need to take care when manually renaming to meet
                                       '     filename specs for processing
    FileExists = Dir("\\server\share$\directory\sub-dir\CLPFname.txt") <> ""
       
    If FileExists Then
        MsgBox vbNewLine & vbNewLine & _
            "     CLPFname already exists." & vbNewLine & vbNewLine & _
            "     New CLP file was NOT renamed, please go to:" & vbNewLine & vbNewLine & _
            "     \\server\share$\directory\sub-dir\" & vbNewLine & vbNewLine & _
            "     and RENAME file with current date in name MANUALLY." & vbNewLine & vbNewLine, _
            vbCritical + vbOKOnly, "My DB: CLP File Rename Error"
   
    Else
                                       ' RENAMES file to meet CLP filename criteria
        Name "server\share$\directory\sub-dir\CLPFname.txt" As _
            "server\share$\directory\sub-dir\CLPFname"
   
    End If
   
End Sub

 

Decided to use "Name ... As ..." command instead, but before doing it wanted to check if file already exist to prevent overwriting an existing file.  If overwriting an existing file doesn't matter, then no need to do checking for it, and you can simply execute  "Name ... As ..." to strip off the file extention in my case.

Yes.  You can use VBA code (or macros) to set a check box control to any integer value as long as the check box, radio or toggle button is unbound or bound either kind of integer field.  Just remember that a checkbox displays as checked for any non-zero value, unchecked for zero or gray for Null.

   If Me.thecheckbox Then

ken sheridan say: It sounds like you have 13 columns (fields) in the table, one per category.  This is bad relational database design as it is 'encoding data as column headings'.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1).  This requires that all data be stored as values at column positions in rows in tables, and in no other way.

The table should be decomposed so that the many-to-many relationship type between whatever entity type your current table is modelling and the categories entity type is modelled by a table which resolves the relationship type into two one-to-many relationship types.  This is what Scott's tblCategorySelect does.  I'd point out that the ParentID and CategoryID columns in his table are a composite candidate key, so must be included in a unique index, or you can dispense with the CategorySelectID column and make the two columns the composite primary key.

The usual interface when inserting rows into a table which models a many-to-many relationship type is by means of a subform in the parent form, but there are other ways.  You'll find examples in StudentCourse.zip in my public databases folder at:

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.
