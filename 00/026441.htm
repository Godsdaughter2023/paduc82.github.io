Useful Debugging Technique - Microsoft Community

carlfalstrom say:

Useful Debugging Technique

Many times we run into problems that we blame on Windows and nothing suggested on the Internet seems to solve them. In many cases the problem is caused by a poorly written program, or interaction between programs that the developers did not anticipate.

Recently I solved a vexing problem that none of the suggestions on the Internet worked for: Edge refused to launch.

To solve the "Edge" problem, I did a clean boot (disable all non-Microsoft services with msconfig and reboot). Edge started working normally. Next, I did a binary chop (re-enable half the services and reboot). If Edge continues to work, the problem must be in the services that are still disabled. If not, the problem lies in the services that are enabled so disable half the services in that group. Again, if Edge works properly, the problem is in the group of services disabled, if not it lies in the other group. Continue in this manner until you have only one program left and that is the offender. Hopefully, that is a program you can do without or contact the vendor to report the behavior.
