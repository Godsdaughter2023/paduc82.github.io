Two reproducible bugs that can be triggered by just deleting files - Microsoft Community

trybird say:

Two reproducible bugs that can be triggered by just deleting files


Users have never been able to avoid doing many misoperations, always might inevitably delete some files or folders that should not be deleted.
So a stable operating system should not be leading to serious failures which are only caused by what as files are mistakenly deleted.
Up to now, Windows 10 still is clearly not such a stable operating system.
Let's take a look at the two reproducible bugs.

Bug A:
Deleting files causes that an error will be reporting after tapping or clicking on Windows PowerShell items of Windows 10 super menu.

Please follow the below steps to view the bug occurring.

Step 1:
Make the batch files for PowerShell shortcuts files' backing up, restoring, deleting.

Open Notepad and use it to make a batch file with the following contents:
robocopy "%AppData%\Microsoft\Windows\Start Menu\Programs\Windows PowerShell" "%UserProfile%\Documents\Windows PowerShell"
Save it as "%UserProfile%\Documents\BakPSShtcts.bat"

Use Notepad to make a batch file with the following contents:
robocopy "%UserProfile%\Documents\Windows PowerShell" "%AppData%\Microsoft\Windows\Start Menu\Programs\Windows PowerShell" /b
Save it as "%UserProfile%\Documents\RstPSShtcts.bat"

Use Notepad to make a batch file with the following contents:
del /a /f /q "%AppData%\Microsoft\Windows\Start Menu\Programs\Windows PowerShell\*.*"
Save it as "%UserProfile%\Documents\DelPSShtcts.bat"

Step 2:
Run BakPSShtcts.bat to back up PowerShell shortcuts files.

Open File Explorer, paste %UserProfile%\Documents in its address bar and then press Enter key to navigate to the Documents folder, then right-click on BakPSShtcts.bat, then click on "Run as administrator", then click on "Yes" button.
Then can be seen that the "Windows PowerShell" subfolder has been copied into the "Documents" folder.

Step 3:
Run DelPSShtcts.bat to delete PowerShell shortcuts files.

In File Explorer, navigate to the %UserProfile%\Documents folder, then right-click on DelPSShtcts.bat, then click on "Run as administrator", then click on "Yes" button.

Step 4:
View the bug occurring.

Right-click on Start button, then click on "Windows PowerShell (Admin)", do you see an error message popping up?

Right-click on Start button, then click on "Windows PowerShell", do you see an error message popping up?

Step 5:
Run RstPSShtcts.bat to restore PowerShell shortcuts files.

In File Explorer, navigate to the %UserProfile%\Documents folder, then right-click on RstPSShtcts.bat, then click on "Run as administrator", then click on "Yes" button.

Bug B:
Deleting files causes that after the first reboot some tiles of the Start menu will be partially lost and then after the second reboot the previous missing tiles will be automatically returned back.

Please follow the below steps to view the bug occurring.

Step 1:
Make the batch file for deleting all the files in the Windows Caches folder.

Open Notepad and use it to make a batch file, its contents are as follows:

for /f "delims=" %%f in ("%LocalAppData%\Microsoft\Windows\Caches\*.*") do (
takeown /F "%%f" /A
icacls "%%f" /grant Administrators:F
del /a /f /q "%%f"
)

Save it as "%UserProfile%\Documents\DelWinCaches.bat".

Step 2:
Run DelWinCaches.bat to delete all the files in the Caches folder.

Open File Explorer, paste %UserProfile%\Documents in its address bar and then press Enter key to navigate to the Documents folder, then right-click on DelWinCaches.bat, then click on "Run as administrator", then click on "Yes" button.

Step 3:
Remember the tiles layouts of Start menu.

Click on Start menu, then please firmly keep in mind the tiles layouts of Start menu.
If necessary, take the screenshots about them to save into the computer, please also save the screenshots into mind.

Step 4:
reboot and view the bug occurring.

Reboot the computer, click on Start menu,
please see the the tiles layouts of Start menu,
do the tiles before eyes become less?
squawk "Oh, my"?

Step 5:
reboot again to fix well the tiles.

Reboot the system again, will find that all the tiles work properly.

Hi,

Error messages which state that a given update isn't applicable to your computer can be caused by system time and date being incorrectly synchronized with the Windows time server.

To check system time and date do the following.

  1. Click the clock on the right hand side of the Taskbar and then click the link to "Change date and time settings".
  2. In the menu which appears, click the "Internet time" tab and then click the "Change settings" button.
  3. In the following menu, checkmark the box, "Synchronize with an Internet time server" and then change the Server to "time.windows.com".
  4. Finally, click "Update now" and then click OK.

If you get an error when you click the "Update now" button, please post that here without paraphasing.

If no errors appear but you're still unable to download updates run this Fixit to reset the Windows Update components: https://support.microsoft.com/en-us/kb/971058
