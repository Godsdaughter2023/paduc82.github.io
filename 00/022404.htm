john - project say:

Brijen,

Just because Project doesn't do something the way you expect or want doesn't mean it is a bug. And Project and Excel are not the same, they are "apples" and "oranges".

If you manually enter resource names and include a space before or after the comma, then yes, Project will strip those out, they are not actually part of the name. In order to create spaces you need to either insert the space before or after the name in the Name field on the Resource Sheet. I suggest you put it before the name as it will be more obvious and will give a better result when wrapped.

With regard to setting the view table, go to View > Data group > Tables icon. In the drop down the current view table is selected so click on "More tables" and then "Edit" to bring up the Table Definition window. Find the Resource Names field and set the Text Wrapping to "Yes".

Hope this helps.

John

brijendoshi,

While I will agree that it takes a little effort to do what you want, it is NOT a bug. A software bug is a failure to do something the software was designed to do and that's not the case here. As I said before, text wrapping is "triggered" by a space and the Resource Names field does wrap when spaces are present in the text string. If there are no spaces, it won't wrap. It is doing what it is supposed to do.
