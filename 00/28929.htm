dale howard [mvp] say:

Eric --

This is actually a fairly common situation.  In fact, I just worked with a client last week who has pretty much the same situation as you.  First of all, you need to know that it is NOT possible to create a master project using local projects that are stored in a SharePoint site.  So, that solution simply will not work.

The other option would be to store the projects in a network folder that all of your PMs have access to.  From there, you could create a master project.

Truth be known, I personally would not use either of these approaches.  Instead, I would simply create one or more milestone in my enterprise project that represent when work is expected to be completed in the vendor projects.  These milestones should have a Must Start On (MSO) constraint applied to them that represents the date when the vendor says the work will be completed for that part of the vendor's project.  Then I would like the milestone with Successor tasks in my own project so that if they vendor is late on delivering their work, my schedule will slip as a consequence.  If the vendor handoff date is going to slip, I would simply change the Constraint Date of the MSO constraint on the milestone.

Although it might seem like a lot of work, I think my solution is the simplest and most trouble-free solution to the problem.  Hope this helps.

Try the code below, customize the name of the sheet "Sheet 2" to your needs. 

Private Sub Worksheet_BeforeDoubleClick(ByVal Target As Range, Cancel As Boolean)
  'Write the value into the other sheet
  Sheets("Sheet 2").Range("A1").Value = Target.Value
  'Cancel the event, otherwise we edit the cell aftwards
  Cancel = True
End Sub


BTW, it is also possible to write the value into the same cell in the other sheet, in this case use this code: 

Private Sub Worksheet_BeforeDoubleClick(ByVal Target As Range, Cancel As Boolean)
  'Write the value into the other sheet
  Sheets("Sheet 2").Range(Target.Address).Value = Target.Value
  'Cancel the event, otherwise we edit the cell aftwards
  Cancel = True
End Sub

Here is some code that does approxiamtely what you want. I have 2 pictures on sheet1 called 'picture 1' and 'Picture 2'. On Sheet 2 if there is a yes in Cell A1 then it copies picture 1 else it copies picture 2.

 

Sub CopyPictures()
    Dim wks As Worksheet
    Dim shp As Shape
    Dim shpNew As Shape
   
    Set wks = Sheets("Sheet2") 
    
    For Each shp In wks.Shapes
        shp.Delete
    Next shp
   
    If LCase(wks.Range("A1").Value) = "yes" Then
        Set shp = Sheets("Sheet1").Shapes("Picture 1")
    Else
        Set shp = Sheets("Sheet1").Shapes("Picture 2")
    End If
   
    shp.Copy
    wks.Paste wks.Range("C10")
    wks.Paste wks.Range("D15")
    wks.Paste wks.Range("E20")
   
End Sub
