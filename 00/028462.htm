Multi Forest Address book - Microsoft Community

roger.a.dosunmu.avanade.com say:

Multi Forest Address book

Hi,

Do anyone know how I can deploy a multi forest address for to O365. We have for different Org. Do we merge all the Org address books or create a new address book for Office 365.

yangzheng_msft6300 say:

Hi Roger.a.dosunmu.avanade.com,

You can refer to the detailed steps below to create the address book policy.
1. Make sure that you admin account has the Address List role permission. You can check this in Office 365 EAC (Exchange Admin Center)->permissions->admin role, and add the Address List role to the admin role group where your account is a member.

2. Connect to Exchange Online with PowerShell.

3. Turn on address book policy routing.

4. Run the command below to create a new room list.

New-AddressList -name "Roomlist1" -RecipientFilter {(CustomAttribute1 -eq "domain1") or (CustomAttribute1 –eq “domain2”)}

5. Run the command below to create a new Address List which will contain the accounts in one domain.

New-AddressList -Name Addresslist1 -RecipientFilter{(CustomAttribute1 –eq “*** Email address is removed for privacy ***”)}

6. Run the command below to create a new Global Address List which will contain the accounts in one domain.

New-GlobalAddressList -name "GAL1" -RecipientFilter {(CustomAttribute1 -eq "*** Email address is removed for privacy ***")}

7. Run the command below to create a new Offline Address Book.

New-OfflineAddressBook -Name "OAB1" -AddressLists "GAL1"

9 Run the command below to create a new Address Book Policy.

New-AddressBookPolicy -name "ABP1" -AddressLists "Addresslist1" -OfflineAddressBook "\OAB1" -GlobalAddressList "\GAL1" -RoomList "\Roomlist1"

10. Then run the command below to assign the new policy to the accounts in one domain.

Get-Mailbox -Filter {(CustomAttribute1 -like "*** Email address is removed for privacy ***") or (CustomAttribute1 –like “*** Email address is removed for privacy ***)} | Set-Mailbox -AddressBookPolicy ABP ABP1

Regards,
Yang

brian tillman [outlook mvp 2007-2019] say:

If the messages are in the Sent Items fodler, then as far as Outlook is concerned, they were sent.  Outlook moves messages to Sent Items only after the outgoing server has accepted the message and has ackowledged that to Outlook.  If your recipients are not receiving the messages, it's not Outlook's fault.

Instead, whatever comes after Outlook is where you should be looking.  This is one of two things.  The first is that your mail server actually accepted the messages but is not forwarding them on.  This is the less likely of the two.  The most likely is that your antivirus program is interfering.  IN order to find out, completely uninstall your AV program and then reinstall it with a custom install, configuring it not to include the mail scanning feature.  You don't need it any way.  It doesn't keep you any safer.  After doing this, resend the messages that didn't go thorugh before.  You should be able to open them (one at a a time) from Sent Items and then click More Move Actions in the Move chunk of the ribbon and choose Resend this Message.
