greg maxey say:

The name of the variable in your document is WillShall.  The last macro that I posted was successfully converting the field value to text, but you or someone had applied manual shading to the field result.  This is why the document displayed “shall” but “shall” was still shaded like a field might be.

 

Here is the code I used to produce the result below:

Sub ScratchMacro()

 'A quick macro scratch pad created by Greg Maxey

 Dim oFld As Word.Field

 Dim oRng As Word.Range

 With ActiveDocument

   .Variables("WillShall").Value = "shall"

   .Fields.Update

 End With

 For Each oFld In ActiveDocument.Fields

   If oFld.Type = wdFieldDocVariable Then

   If InStr(oFld.Code.Text, "WillShall") > 1 Then

     oFld.ShowCodes = True

     Set oRng = oFld.Code

     oRng.Start = oRng.Start - 1

     oRng.End = oRng.End + 1

     oRng.Shading.BackgroundPatternColorIndex = wdAuto

     oFld.ShowCodes = False

     oFld.Unlink

   End If

   End If

 Next oFld

 End Sub

john w. vinson say:

I'm guessing that you don't have the DEFAULT value set to =Now() but the Control Source... am I right?

How are you entering data? Directly in the Table (not right) or on a Form (much better)? If it's a Form, what is in the underlying table, and what is the Control Source of the textbox?

Do note that Access Date/Time values are just that - date AND time, in one field. What's actually stored is a Double Float number, a count of days and fractions of a day (times) since midnight, December 30, 1899. The Format doesn't change what's stored, just how it's displayed. If you have a Medium Time format, what will actually be stored is  42632.587974537, which is equivalent to 9/19/2016 2:06:41 PM. If you set the Format to a time only format, you will SEE 2:06:41 PM, but today's date will be stored but invisible. I'd change the format to something like "hh:nn ampm mm/dd" so you can see 2:06 pm 09/19, and you can use the powerful builtin date functions to calculate how long the truck was out (including over one or more midnights).

Setting a Default will apply only when a new record is created. If you want to use a button (and/or the dblclick event of the textbox) you can put code like

Private Sub cmdTimestamp_Click()

Me!txtTimeOut = Now

End Sub

ken sheridan say: One thing I'd add to John's reply is that you should not be tempted to use the DefaultValue property of a column in a table or a control in a form to record the current date/time as that when the record is created.  It's often not realised that if you do this it assigns the date/time when the  record pointer first moves to a new empty record, not when the user begins to enter data.  Do as John describes and assign the current date/time to one bound control when the user clicks a 'Check In' button and to another bound control when they click a 'Check Out' button.

As regards computing the duration between two times provided that the duration is less than 24 hours you can simply subtract one from the other in an unbound text box and format the text box as hh:nn.  If the duration could be 24 hours or more then this won't work, however, so you have to process the result for it to appear as hours:minutes.  You'll find an example in TimeArithmetic.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes a TimeDuration function which formats the duration either as hh:nn:ss or as d:hh:nn:ss.
