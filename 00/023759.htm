a. user say:

This unfortunately is what happened:

When it was first installed, Windows 10 TP scanned your system, and based on the hardware present, created what's called a hash, a unique digital fingerprint that identifies your system for what it is.  As long as that hash doesn't change, Windows remains activated, since it knows it's running on an authorized system.

Purportedly, small changes of peripherals like the hard drives or the optical drives don't impact your system hash, and thus don't affect Windows 10's activation--but when a central component like the motherboard gets changed out, it reads as though it's an entirely new system.

You're not violating the EULA or any service terms, since you are still using it on the same system, so I suggest going through the activation process, calling by phone if you have to; it's a hassle, but it's how Microsoft is keeping the Insider Previews within the approved community only and not letting people have free software.

PMFJI, but the OP is correct about the behavior being different from earlier versions [although I'm not certain what is meant by "caption"].

In Word 2016 the dragged document is inserted as an embedded object as you say. As a result, only the first page displays. You can double-click the object to edit it in a separate window but still can't force the additional page(s) to be displayed in the host document.

In prior versions exactly the same steps did result in the text of the dragged document being appended to the text of the host. The content of the dragged document retained its formatting & displayed in its entirety in editable form rather than as an object.

suzanne s. barnhill say:

Open the footer pane and view the deviant footers. Press Ctrl+* or click the Show/Hide ¶ button to display nonprinting characters (formatting marks). Each ¶ you see represents a paragraph. Is there an extra, empty paragraph below the footer paragraph? If so, delete it.

The likely cause of this problem is frustratingly simple: When you paste content from one header/footer to another, no matter how carefully you exclude the paragraph mark when selecting the text to copy and even though you select the paragraph mark of the empty paragraph before pasting, you will still always get an extra, empty paragraph.
