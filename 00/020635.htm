suzanne s. barnhill say:

Normal.dotm is macro-enabled by default. It needs to be because it is the default save location for any macros you create. If some other template (or an add-in) makes changes to Normal.dot or Normal.dotm every time you start or quit Word, then you will get this prompt, and it's worth investigating what's causing the message. Sometimes (as in the case of the Acrobat add-in, if you want to keep the macro enabled, you just have to learn to live with the annoyance, but if you don't need the add-in, you can remove it.

When I said allowing Word to save the Normal template without prompting you is dangerous, yes, a macro virus could be a risk. And a lot of times you'll have made changes you don't necessarily want to save, especially if you've screwed something up. But Word also is kind of stupid about what kinds of changes you've made. For example, if you (in an older version) added a button to a toolbar and then removed it, you've made no net change, but Word still sees it as a change, so you'd be prompted to save the template. This happens to me a lot because I do a lot of testing to help answer users' questions.

Hi Marcos,

If you add the following code to your Normal templates macros, it will prompt you if you are opening a document where the Remove Personal Information has been set. Responding "Yes" to the prompt removes the setting.

Sub AutoOpen()
    Application.OnTime Now + TimeValue("00:00:02"), "SetWritingEnvironment", False
End Sub

Sub SetWritingEnvironment()
    Set doc = ActiveDocument
    If doc.RemovePersonalInformation = True Then
        If MsgBox("The remove personal informartion control is set on this document. " & _
            "Do you want to remove it?", vbYesNo, "Microsoft Word") = vbYes Then
            doc.RemovePersonalInformation = False
        End If
    End If
End Sub

Management of Office365 is via Office365 administrative console. As long as you have admin access, you can manage Office365. The management experience, whether you are the end user, the IT consultant, or the third party support, is similar.

What the third party has, is delegated admin access (can be removed at your request) to the tenant so that they can login and manage on-behalf. 

If their office365 was purchased under Cloud Service Provider (CSP) license, then they also do not have the ability to raise support tickets directly with Microsoft, this has to be done via the CSP (as part of their compulsory obligation as a CSP partner).

You should try to get Global Admin access from the current third party. Once in, you should have full admin access to self manage. Possibly, the only restriction is that you might still need to contact the third party to raise support tickets to Microsoft on your behalf - that you don't need to do very often. Anything else can be done via Admin console.

jay freedman say:

First, open a document that is based on the template that contains the macro code, if that isn't Normal.dotm.

Go to File > Options > Customize Ribbon and click the Keyboard shortcuts Customize button at the bottom left.

In the Customize Keyboard dialog, first look at the "Save changes in" dropdown and choose the template in which the macros are stored. Then scroll the Categories list and click on Macros near the bottom.

The right column, now relabeled "Macros", will list all the macros in the selected template. Click on each one in turn and note the content of the Current Keys box.
