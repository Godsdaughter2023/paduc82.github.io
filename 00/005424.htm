Please check the following settings on your Outlook.com:

6. Click Save.

1. Sign in to your Outlook.com account.

2. You’ll be directed to your Inbox.

3. In the upper right corner, please click the Gear icon and then choose More mail settings.

5. Click New.
6. Choose the condition that you want to apply to the rule in Step 1.
7. Choose the action in Step 2 that you want to take on the emails that meet the condition specified in Step 1.
8. Click Save.

stefan blom say:

The indentation (from the left) for numbered and bulleted items is controlled by the settings in the number or bullet format; you can't specify it in the paragraph (style) applied to text. For a single-level number (or bullet), click the actual number (or bullet) symbol to select them all, and then right-click and choose Adjust List Indents from the context menu. Using a single-level numbered list as an example, this is what the Adjust List Indents dialog box looks like:

"Number position" defines the location of the number. "Add tab stop at" defines where the text starts after the number. "Text indent" defines the starting position for the second and subsequent text lines in the numbered paragraph. All measurements are from the left margin. In other words, if you want a hanging indent, you should set "Text indent" equal to "Add tab stop at" and the value should be larger than "Number position." (For a multi-level list, the Adjust List Indents command will display the Multilevel dialog box, which is discussed in detail in the article at http://www.shaunakelly.com/word/numbering/numbering20072010.html.)

In addition, note that instead of applying a template to a document, you can create a new document based on the template and then bring in existing content into that new document via the Insert File dialog box.

To open the Insert File dialog box, click the arrow on the Object button (on the Insert tab) and then click Text from File.

I am assuming that the Userform is being Shown with Modal set to false otherwise you should not be able to navigate between worksheets while the userform is showing.

By the code sample you posted, the UserInterfaceOnly is locked and therefore access to the worksheets by VBA code should not be affected.

However, I have come across a problem similar to this before that a linked cell could not be updated in a worksheet where the Linked cell is locked and UserInterfaceOnly was protected. The workaround was to remove the linked cell property and use code to change the cell value in the Control's After Update event or Unlock the linked cell.

What I am getting at is it the cell that you are attempting to manually change that is causing the problem or is a linked cell's value changing due to the change you make manually and this is where the real problem is. Also note that linked cells can get updated when a userform opens.

Now the really strange thing is if the Worksheet is locked when saved and closed and re-opened then that is when the Linked cell gave problems. If it was the unlocked after opening and locked again then the problem disappeared.
