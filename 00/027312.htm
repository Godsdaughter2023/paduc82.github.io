Using a formula returning an array in data validation - Microsoft Community

dario de judicibus say:

Using a formula returning an array in data validation

Let's suppose that I have a functions that returns an array. For example

MYARRAY(5;4;2)

returns

{5,9,13}

and that I want to use that array in data validations (that is, WITHOUT using ranges). How can I do that?

hansv mvp say:

If you want to use this in data validation of type List, that is impossible. The source for data validation of type List can either be a fixed list of constant values or a range of cells. You can specify the latter in the form of a formula, but not the former: a list of constant values must be just that: a fixed list.

You could use VBA to remove the existing data validation, if any, then add it with the desired list of values.

Michael_80DJ

Short version. Read below for the explanation.

Always backup your registry before making any changes. 

Open Regedit

Browse to HKEY_CLASSES_ROOT\xmlfile\shell\Open\command

Change (Default) to 

"C:\Program Files\Microsoft Office\Root\VFS\ProgramFilesCommonX64\Microsoft Shared\Office16\MSOXMLED.EXE" /verb open "%1"

Delete

"DelegateExecute"="{17FE9752-0B5A-4665-84CD-569794602F5C}"

It took about 2 weeks to find the offending registry key, and the setting that was causing all the problems.

Always backup your registry before making any changes. 

Open regedit

Browse to HKEY_CLASSES_ROOT\xmlfile\shell\Open\command

This is the key that determines what program opens XML files.

Change the (Default) setting  to the path of the program you want to open XML files. In my case, I wanted the XML File handler included in Office 2016 Pro Plus, so I set it to

"C:\Program Files\Microsoft Office\Root\VFS\ProgramFilesCommonX64\Microsoft Shared\Office16\MSOXMLED.EXE" /verb open "%1"

The very important step is to delete the "DelegateExecute" setting.

This is what was causing me grief, as it was set to Internet Explorer on our systems, which caused IE to open XML files, but only if IE was already running.

The GUID for IE11 is {17FE9752-0B5A-4665-84CD-569794602F5C}

Your registry may have a different setting, if it does, you can search the registry for whatever value is in there, and eventually you'll discover what program it is pointing to.

Or you can just delete it and move on.

ron sommer mvp 2009-2017 say:

You can add a Gmail address as a send/receive account to an existing outlook.com account.

When you create an outlook.com account using a Gmail address, the Gmail address will only be able to send emails using the Gmail server.  Any emails sent to the Gmail address will go to the Gmail server. 

Enabling pop will not help.

To get emails to the outlook.com account, you must add an @outlook.com address to the outlook.com account.

Then login to the Gmail account and setup email forwarding to the @outlook.com address.

To add the Gmail address as a send/receive account, you would have to make the @outlook.com address the primary alias.
