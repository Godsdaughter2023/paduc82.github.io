I presume that you mistyped   .CVS  instead of .CSV

CSV file extension means Comma Separated Values but is is a normal text file with csv as extension. There is no Excel csv format.

Any basic text editor (eg notepad) can create, open, modify and save them.

csv file are very common as they can be opened by a lot of different programs.

When installing some programs, i.e. Excel, an association between the file extension .csv and the default program/application to open them is created, thus giving the false impression that these csv files are closely (but wrongly IMHO) connected to an Excel format .

If you right-click such a csv file, select open with/choose program , find notepad and tick the box "always use this program to open".

jeeped say:

I believe the term you are looking for is radio button. Grouped together, a number of radio button will only have one selected. Selecting another cancels the first. This is where the term radio button comes from because the behavior mimics an old style car radio where someone pushed one button of several to select a radio station and of course, only one radio station was selected at any one time, Put another way, you do not turn a radio button on and off; if a radio button is on, it is automatically turned off when you select another radio button in its group.

With that said, I'm unclear on whether you actually want radio buttons or checkboxes. If you have 50 radio buttons grouped together then you could only have one line visible at a time. If you had 50 independent checkboxes then you could have 1, 2, 50 or no lines visible as each checkbox works independently of the others.

when I open old Excel 97-2004 file with Excel for Mac 2011 I get a - Microsoft Community

paulmccusker say:

when I open old Excel 97-2004 file with Excel for Mac 2011 I get a window saying that there is a macro present and do I want to enable macros

I have a workbook originally created under a previous version of Excel 97-2004 (*.xls) when I open it with Excel for Mac 2011 I get a window saying that there is a macro present and do I want to enable macros, or disable macros or cancel. Enabling macros and opening with Excel 2011 I cannot find any trace of macro coding using the developer tab. Saving the file as an *.xlsx file gives no problems and the macro present window does not appear
j.e. mcgimpsey say: Couple of suggestions:

First, look for XL4M (old-style) macro sheets. They may be hidden (Choose Format/Sheet/Unhide - if the menu item is greyed out there are likely no hidden sheets).

Look in the Visual Basic Editor's Project window to see if the file has any code modules, user forms or class modules. Also check the ThisWorkbook and Worksheet code modules for macros).

Also, in XL, choose Insert/Name/Define to open the Define Name dialog, and check that there aren't any names in the workbook that have XL4M language in them.

If none of those show any macros, you may just have some corruption that XL can't parse. Saving as an .xlsx file is a good option  - you can then reconvert to a .xls if needed.
