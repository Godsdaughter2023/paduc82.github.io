Latest AutoUpdate (3.0.5) Bugged - Microsoft Community

johnedee say:

Latest AutoUpdate (3.0.5) Bugged

The latest version of AutoUpdate (v3.0.5) seems to be seriously bugged.  Either that, or maybe it's the servers it's talking to.

In our environment (we have many Macs), the previous version of AutoUpdate first updated itself to v3.0.5.  Then it found two updates to the Preview (Excel and Word 15 to v15.9), an update to Outlook 15 (to v15.9) and an update to Office 2011 (to v14.4.9).  In some cases it finds a Lync update too.

Selecting any one or all of them to Install almost always results in one of two error messages:

  • "Sorry, file not found on server.  Please try again later." (something along those lines...can't reproduce it now so that's just from memory).  The download doesn't even start:  it immediately returns you to the dialog with the available updates.
  • "Sorry, there's a problem with the downloaded file, so you can't update now. Please try again later." (happening now so was able to copy/paste the exact error).  This happens after downloading the entire update, which can be pretty lengthy.  After that, you're returned to the dialog with the available updates.

Either way, nothing is installed or updated.

It's been going on all day (Wednesday 4/16) and so far still on Thursday.

The workaround is a little bit of a pain but it works:  click once on each update to select it, then click the link in the description below the update.  On the resulting web page that opens in your default browser, there will be a link to download the installer directly, and that should download fine.  Run the downloaded installer and you're done.  Repeat for each available update.

AutoUpdate will thereafter see the updated products and will no longer offer the updates.

When you're done, be sure to delete those giant installers, as they take up quite a bit of space.  

Here's hoping that AutoUpdate (or the servers it talks to) get fixed before the next round of updates.  :-)
