john mcghie say:

My testing showed that:

  1. If you are in 10.9.4, the issue should occur only the first time you start for the day.
  2. When Word starts, it reads the Current Save Folder from its preferences.  If that entry is blank, it defaults to the User's Documents folder.
  3. If you then open a file using File>Open, Word receives the location of the file and sets that as current save folder.
  4. If you open a file using the Finder, Word does NOT receive the location of the file until you specify it during save.
  5. Each time you save, Word updates the current save directory (if it is different).
  6. If you leave Word running constantly, what happens is akin to a lottery.  Word doesn't get a chance to clear its memory and its behaviour will become increasingly strange.

I recommend logging OUT of your user session (Apple menu, Log Out...) every day, and Restarting the computer once a week.  This gives all the software, including OS X, an opportunity to do its house-keeping. You will get much more trouble-free operation.

rounding results

rounded results are:

44%  26%  15%  15%   FINAL RESULTS 101%  - SHOULD BE 100%

joeu2004 say: Jane wrote:
I hve the following %'s which add up to 100% but do not because of rounding.  What is the best you of =ROUND to solve my problem?
rounded results are: 44%  26%  15%  15%   FINAL RESULTS 101%  - SHOULD BE 100%

Assuming your cells are formatted as Percentage with 0 decimal places and, for example, the cell displaying 44% is actually a numeric value between about 0.435 and less than 0.445, the following sum the rounded displayed values:

=SUMPRODUCT(ROUND(A1:D1,2))

Actually, it would be prudent to write ROUND(SUMPRODUCT(ROUND(A1:D1,2),2), just in case you compare the results to other sums or the constant 100% (1).

The extra ROUND eliminates artifacts of computer binary arithmetic, which Excel uses.

PS:  It is not uncommon for a column of percentage not to sum to 100%.  This is why most professional accounting reports have a footnote that says something to the effect of:  "Due to rounding, percentages may not add up to exactly 100%".

For example, if you 3 percentages all computed by =3/9, they might each display as 33%, and the SUMPRODUCT(ROUND) formula above will result in 99%, not 100%.

There really are no "good" solutions to that problem.  In the example here, you would have to force one of the percentages to be 34%, which might misrepresent reality when the percentage is applied or compared to other parts of the report or worksheet.

ashish mathur say:

Hi,

 

Since you have only 3 options in the first drop down list, try this solution

 

1. In range D2:D4, E2:E6 and F2:F10, type the colour for T-shirt, Polo and Jackets respectively

2. Click on cell C2 and go to Data > Validation > Custom > List and enter this formula there

 

=IF(A2="T-shirt",D2:D4,IF(A2="Polo",E2:E6,F2:F10))

 

You may also want to refer to this link on my website - http://www.ashishmathur.com/secondary-validation-cell-entry-to-update-when-primary-validation-cell-changes/
