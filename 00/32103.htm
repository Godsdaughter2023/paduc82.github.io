Looking at my drives I decided to organize them differently, just to make sure ...

Drive 0 (SSD):

- System Reserved (System, Active, Primary Partition)

- OS (C:) (Boot, Crash Dump, Primary Partition)

Drive 1 (HHD):

- Data (E:) (Page File, Primary Partition)

- Bkp (G:) (Primary Partition)

Unfortunately I was still having the problem even with this configuration. I tried to change Page File location but it did not work. I eventually found out that one of the Windows Libraries (Documents) was pointing to my Data (E:) drive.

I removed the drive from my Documents library (right click on Document Library > Properties > Remove Data (E:) from Library Location) and I finally succeeded backing up my OS (C:) drive!.

It seems that Windows Backup do not work if it writes on a drive pointed to by a library...

I then resolved my problem - partially from Shalini that point out to potential disk problems, and by myself by looking at what could point to Data (E:) drive from OS (C:) drive.

Try following and check after each step (Note - You will not lose any mails because of below step) - -

1. Re-register All Store apps
Right click on Windows Start > Right click on Windows Powershell (Admin) > Copy the following from below and right click in Powershell to paste > Enter > Restart your computer

Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

Get-AppxPackage Microsoft.windowscommunicationsapps | Remove-AppxPackage

B. Reinstall

Get-AppxPackage -allusers Microsoft.windowscommunicationsapps | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}

Insert the Bootable Installation Media, then go into your BIOS and make the following changes:

2. Enable Legacy Boot
3. If Available enable CSM
4. If Required enable USB Boot
5. Move the device with the bootable disc to the top of the boot order
6. Save BIOS changes, restart your System and it should boot from the Installation Media
