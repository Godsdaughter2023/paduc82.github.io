Win 8 Total Backup System: File History + RecImg + Refresh + SlimImage

Win 8 has refined your backup and restore options. Before Win 8 you could create an Image Copy backup of the full drive (or selected directories), but it was hard to separate backups of the OS, Applications and personal Data unless you created a separate data partition.  I still think it is a good option.

 

Out of the box, MS has been advertising the “Refresh” and “Reset” options. Personally, I think they missed a valuable selling point by not mentioning RecImg.EXE in conjunction with Refresh. It removes my main concern/objection to Refresh.

 

Reset is equivalent to using the “Recovery Partition” on previous versions of Windows. The factory would create a “Recovery Partition” that you could use to “reset the computer to factory fresh”. It would typically include all of the factory “crapware” and it would wipe out all of your data files and installed applications.  A less than desirable outcome but an appropriate reaction to a bad malware infection.  The “recovery partition” type of restore typically had to be accessed at boot time while Reset can be accessed from within Windows 8.

 

Refresh is roughly equivalent to an image copy backup.  A Reset will restore a backup of the OS and Metro applications. It will wipe out “legacy” / x86 desktop type applications BUT it will not affect your personal data files, which is much better than a Reset.  But there is still that issue of wiping out your installed desktop applications. I do not have documented confirmation, but I’m guessing that Metro apps are treated like personal data and those directories/folders are simply not reset back to the old condition.

 

RecImg.EXE is a backup tool that ships with Windows 8. It creates the backup “image” that is used by the Refresh tool. This image that was created at the factory is why using Refresh will wipe out installed desktop programs in the “C:\Program files” and “C:\Program Files (x86)” folders because they haven’t been backed up. However, using RecImg.EXE you can create new images that will include recently installed desktop applications.

 

RecImg Manager is a third party tool the provides a GUI to access the RecImg.EXE command line tool and backup files.

 

File History is a new Win 8 feature. It is an extension to the “Previous Versions” and “Volume Shadow Copy” features. File History continuously monitors files stored in Libraries, Desktop, Contacts, and Favorites folders. It runs on a predetermined schedule, and when it detects changes in any file, it makes a backup of that file to an External drive. It isn’t enabled by default.

kris ver say:

Hi SpinyEchidna,

Usually, when you see the message "Continue typing the menu key sequence from an earlier version of Office or press ESC to cancel.", it is either the corresponding shortcut key is no a longer available or it should be part of a sequence. For example, the ALT + E shortcut key where it can be part of a sequence, ALT E+A+C. 

In the new ribbon interface of Office 365, the Edit menu was already redesigned and it was already removed. The functions under the Edit menu before can be seen on difference tabs, such as cut, copy and paste functions which can be seen under the Home tab, on the Clipboard group.

For a list of keyboard shortcuts in Excel for Windows, you may check this article: Keyboard shortcuts in Excel for Windows.

Regarding the issue of crashing, I tried replicating your issue here on our end by pressing ALT + A in Excel. However, I didn't experience crashing. In this case, we suggest we suggest that you install the latest updates for Windows because updates often fix problems that occur with software. Also, try installing updates for your Office, restarting your computer, and then launching Excel. The instructions are slightly different depending on version of the Office that you are using. 

If you continue to experience the same issue after updating Office, we suggest that you start Excel in Safe Mode because there is a possibility that third party add-ins might be causing the crashing. Safe mode lets you safely use Excel without encountering certain startup programs.

You can open Excel in safe mode by pressing and holding Ctrl while you start the program or by using the "/safe" (that is, excel /safe) option when you start the program from the command line. When you run Excel in safe mode, it bypasses functionality and settings such as alternative startup location, changed toolbars, and Excel add-ins. (However, COM add-ins are excluded.)

Note: To stop Office Safe Mode, exit and restart your Office application. 


If the issue doesn't occur in the Safe Mode as follows, enable your add-ins one at a time until you hit that one that is causing the conflict. Leave that one disabled or uninstall it. For more information about enabling and removing add-ins in Office, you can check this link and go to the sections: Enable an add-in and Permanently disable or remove an add-in.

Here's are the steps on how to disable add-ins one at a time:

  1. Click on the File tab and then click Options.

  2. When the Options menu appears, click on Add-Ins.

  3. Click Go next to the Manage list at the bottom of the menu.

  4. Uncheck one of the add-ins, and then click OK.

  5. Restart the application, but do not use Safe mode.

Best consolidated view using MS Project 2010

john - project say:
Jamie,
It's not obvious that a dynamic master is necessary for what you need. It all depends on your experience with Project and how your users interact with the files (i.e. to help gauge the probability of eventual file corruption).

With regard to visual presentation, you can set up Gantt views using font colors for text and bar styles, (i.e. colors and shapes), for the graphics. A version or two ago, there was a utility to create PowerPoint slides from Project data but I don't think it works with Project 2010. Project does have a Copy Picture feature found under Task/Clipboard group/Copy icon. Again, It would probably be your best option for use with PowerPoint or Word documents. Project 2010 has a Save & Send feature that allows you various options for distributing Project data to others (e.g. PDF). Keep in mind that if you send a project file itself (i.e. .mpp), the recipient will need to either have Project installed on their computer or have a third party Project viewer.

In some cases, it is better to export Project data to another application, such as Excel, which is more universally available for other people and Excel has much better charting and report formatting capability. Export can be accomplished with an export map (non-timescaled data only), visual reports (timescaled data), or to have the most flexibility, with VBA. Personally I use the latter.

Showing milestone, overallocations, etc. is a matter of which view you capture and how the view is structured (i.e. grouped, filtered, etc.).

Hope this helps.
