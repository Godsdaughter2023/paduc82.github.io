Edit scanned document - Microsoft Community

70bp2016 say:

Edit scanned document

I scanned a document from my printer into word.  In word, the scanned document will not allow me to edit that document.  How can I now allow this document to be edited?

[Original title: word]

suzanne s. barnhill say:

There are two ways to scan a document: as an image or as editable text. This is true even if you scan to PDF (it can be a static image or an editable PDF, though PDFs can't be edited in Word 2010).

To create editable text, the scanning software must incorporate OCR (optical character recognition) to convert images into text. If you have scanned the document as an image, you won't be able to edit it until you have OCR'd it. One (rather tedious) way to do this is through OneNote; this is suitable only for small amounts of text and requires a lot of cleanup. A better approach is to rescan the document, ensuring that the scanning options are set to scan as editable text.

jay freedman say:

The scan of the document is a picture of text. It isn't actually text, which is why you can't edit it. To convert the picture to editable text, you need to do Optical Character Recognition (OCR).

From the article https://support.microsoft.com/en-us/kb/982760,

To implement the functionality of this feature, use Microsoft OneNote 2010. To do this, follow these steps:

  1. In OneNote 2010, on the Insert tab, click Picture.
  2. Locate and then open the scanned file. The file is inserted into OneNote 2010 as a picture.
  3. Right-click the picture, click Copy Text from Picture, and then paste the information to another location.

ken sheridan say:

Take a look at RowNumbering.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169
If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes examples of queues for both numbering and ranking rows.  If you don't need an updatable result table the appropriate model for you to follow is:

SELECT T1.TransactionDate, T1.TransactionAmount, T1.CustomerID,
COUNT(T2.TransactionID)+1 AS Rank
FROM Transactions AS T1 LEFT JOIN Transactions AS T2
ON T2.TransactionAmount>T1.TransactionAmount
GROUP BY T1.TransactionAmount, T1.TransactionDate,
T1.CustomerID, T1.TransactionID
ORDER BY T1.TransactionAmount DESC;

which joins two instances of the table, retuning the details from one instance and the rank as COUNT(T2.TransactionID)+1 from the other.  In this the TransactionAmount column is analogous to the eighth column in your table.  If you do need an updatable result table, there is an example in the file for this, which calls the DCount function rather than using a join:

SELECT TransactionDate, TransactionAmount, CustomerID,

DCount("*","Transactions","TransactionAmount > " &
TransactionAmount])+1 AS Rank
FROM Transactions
ORDER BY TransactionAmount DESC;

ken sheridan say: The result table would not make a lot of sense to me as ranks, but you could use a subquery which counts the distinct rows returned by a further subquery.  With my RowNumbering demo to which I gave you a link the following query:

SELECT T1.TransactionDate, T1.TransactionAmount,
    (SELECT COUNT(*)+1
     FROM (SELECT  DISTINCT TransactionAmount
                  FROM Transactions) AS T2
                  WHERE T2.TransactionAmount > T1.TransactionAmount) AS Rank
FROM Transactions AS T1
ORDER BY T1.TransactionAmount DESC;
