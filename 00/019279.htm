    The Workbook Activate/Deactivate events trigger when the workbook is opened and when it is closed AND when, if you are working with multiple workbooks, you switch from using this one to another and back - so those routines ensure that should you switch to another workbook that the .Calculation state is as it was initially (assumed to be Automatic).

    The Worksheet_Change() event code I provided needs to be in that event or else a test of the contents of that cell needs to be tested specifically in any other private sub for the sheet if the actions to be taken need to take its contents into consideration.  But you may want to use a GoTo statement rather than exit the sub if it's an unknown.  Let's say your Worksheet_Change() routine is already doing checking for other cell's contents also, perhaps something like this:

    Private Sub Worksheet_Change(ByVal Target As Range)

     If Target.Address(False, False) <> "A1" Then
        'was not the Yes/No cell, go on down and do other checks.
         GoTo NextChangeTesting
       End If
      Select Case UCase(Trim(Target))
        Case Is = "YES"
          'turn on full automatic calculation
          'to recalculate data sheets and tables
          Application.Calculation = xlCalculationAutomatic
        Case Is = "NO"
          'set to Automatic Except for Data Tables
          Application.Calculation = xlCalculationSemiautomatic
        Case Else
          'not Yes and not No in the cell, do nothing
      End Select

      Exit Sub ' can exit here because only A1 was changed

    NextChangeTesting:

       'more code here to be done when the change was not to A1, but

       'perhaps to one or more other cells you are monitoring

       If Target = "Mikey will eat anything" Then

         ' buy more cereal

       End If

    End Sub

    but you can use formulas to flag rows which meet your criteria,
    then filter out & delete these flagged rows

    In "A",
    Insert a new row1 and a new col A (your previous col A is now col B)
