bob jones aka: cybertaz say:

Looking at your example document I'm assuming that the problem is with the Table at the top. I don't have a direct answer because I'm not exactly certain where the problem lies, but perhaps some general information will be useful.

If my assumption is correct, the key consideration is that the only size constraint you can impose directly on a Text Box Form Field is the number of characters it is allowed to contain. Neither the Height nor the Width of the Field are user-defined parameters.

The width of the Field in a table cell [or a Frame] is controlled by the width of the cell. If the allowed number of characters exceeds that width the content will wrap to additional lines to accommodate it. If the Field isn't in a cell or Frame its width typically is determined by the margin of the document just like body text.

Height is a different story altogether... You can specify the height dimension of a table Row or a Frame. That prevents the height of the cell/Frame from increasing & influences how many lines of the Field content will display. However, it does not restrict the amount of content allowed in the Field. IOW, if the cell/Frame height is set for a single line but the Field size property is left as Unlimited, an unlimited amount of text still can be entered, but the portion that wraps beyond the first line just will not be visible.
