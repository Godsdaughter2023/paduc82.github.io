dougeckert say: I started having this issue also after setting up 2-step authentication.  I use the K9 email app on my Android/GalaxyS3.

When you set up the 2-factor authentication from

   Account Settings > Security Info < Set up two-step verification

it provides a string to use as a password for incoming messages only.  The instructions leave you hanging about what to do for outgoing messages, however.

The solution is to create an "app password" from the same Security Info page

  Account Settings > Security Info > Create a new app password

Use the provided string in the password field for outgoing server settings.  No more SMTP 535 error!  This same method would work for other email clients or applications that do not support two-step verification.  

I don't know why two-step works for incoming, but not outgoing.
