mike hall... always say:

OEM Windows stays with the computer on which it is first installed.

It can be re-installed on the SAME computer as many times as you like, but if the period between installations is short, you may end up having to activate by telephone.

Any version of Windows can only be installed on one computer at any one time. The rule is that each computer has its own individual keycode.

Where a version is supplied with 32-bit and 64-bit DVD's, the keycode is good for both, BUT only one can be installed on the keycode at any one time. You can't install 32-bit on one machine and 64-bit on another.

 

In short, if you want to move Windows around, you need the full RETAIL version. This version also can't be installed on multiple computers with the same keycode.

I am currently connected through Ethernet between my Xbox and the router (Gigabit network).

You should not have to port forward anything in order to make your services run any faster. It is like saying I have to open port 80 for my local IP in order to make downloading a game on my pc faster. Networking doesn't work that way. Port forwarding means you are opening the router so connections externally can connect to your computer through that port. For a peer to peer network (eg. BitTorrent or Gnutella (RIP Limewire)), you should do this to make it run faster. However, on a web-browser and a gaming console, these are things you don't need to do because the systems are designed so you don't have to. Only port-forward when you are setting up for example a web-server or a game-server (Minecraft) and you want other computers to be able to connect to your specific IP-address. Normal users are not supposed to even know what "port-forwarding" means, and it would be dangerous to let them mess with their router settings.

That I am 1500 miles away from a xbox server does not really matter much. Microsoft is using something called a content delivery network (CDN) which means they have placed several servers to distribute content out to the users where the placement of the servers is based on where the user load comes from. This is my generic how to for proper driver updates :

This utility makes it easy to see which versions are loaded : Run DriverView - set
VIEW to Hide Microsoft drivers - update those without Dump in their names (and
BIOS and chipset drivers).

DriverView - Free - utility displays the list of all device drivers currently loaded
on your system. For each driver in the list, additional useful information is
displayed: load address of the driver, description, version, product name,
company that created the driver, and more.
http://www.nirsoft.net/utils/driverview.html
 
For Drivers check System Maker as fallbacks and Device Maker's which are the
most current. Control Panel - Device Manager - Display Adapter - write down
the make and complete model of your video adapter - double click - Driver's
tab - write down the version info. Now click UPdate Driver (this may not do
anything as MS is far behind certifying drivers) - then Right Click - Uninstall -
REBOOT this will refresh the driver stack.

Repeat that for Network - Network Card (NIC), Wifi, Sound, Mouse and Keyboard
if 3rd party with their own software and drivers and any other major device drivers
you have.

Now go to System Maker's site (Dell, HP, Toshiba as examples) (as rollback) and
then Device Maker's site (Realtek, Intel, Nvidia, ATI as examples) and get their
latest versions. (Look for BIOS, Chipset, and software updates at System Maker's
site while there.)

Download - SAVE - go to where you put them - Right Click - RUN AD ADMIN -
REBOOT after each installation.

Always check in Device Manager - Drivers tab to be sure the version you are
installing actually shows up. This is because some drivers rollback before the
latest is installed (sound drivers particularly do this) so install a driver - reboot
- check to be sure it is installed and repeat as needed.

Repeat at Device Makers - BTW at Device Makers DO NOT RUN THEIR SCANNER -
heck manually by model.

Manually look at manufacturer's sites for drivers - and Device Maker's sites.
http://pcsupport.about.com/od/driverssupport/ht/driverdlmfgr.htm

Hi,

Thank you for posting in Microsoft Community, we appreciate your interest in Windows 10. 

We understand how you feel about the situation. We’ll be glad to help you fix the issue.

Before we proceed, I need more information to assist you better.

1) What is the exact error message are you receiving?
2) What happens when you try to login with the wrong account?

Let’s start your computer in Safe mode from the Windows Recovery Environment. Follow the below step.

1. From the Lock screen, click on Power button.
2. Press and hold the Shift key from the Keyboard and Click on Restart and try to enter the Safe mode.

Then select Troubleshoot > Advanced options > Command Prompt.

In Command prompt type,

net user name password /add

Replace 'name' and 'password' with your new accounts name and password

After creating a new user account in command prompt, try logging in with the new account and see if it helps.

Hope this helps. If you have any queries, feel free to contact us.
