Word does not create charts. Charts are  Microsoft Office Objects which are created by a generator housed in the Excel program. That generator is used regardless of which program is being used to create the chart or what type of document it's displayed in. The sheet containing the chart & the data on which it is based is embedded within the document file.

I'm not clear on your objection or concern, but the alternatives would include;

1. Create the chart in Excel then;

  • Right-click the chart
  • Choose Save as Picture
  • Select preferred image file type (JPG, PNG, etc.)
  • Insert [Insert> Photo - Picture from File] into a Word document

2. Create the chart in Excel, then;

  • Select the chart
  • Hold the Shift key while opening the Edit menu
  • Select Copy Picture
  • Choose either option as preferred
  • Paste into the Word document

3. Create the chart in Excel then;

  • Copy the chart
  • In Word use Edit> Paste Special
  • Select either Picture or PDF [IMO, PDF usually renders more sharply]

detasseler say:

How do I print a simple list of Queries in my Access database?

I have hundreds of queries in my Access app and am constantly scrolling to find the one I want to work with.  How can I print a simple listing of all of my queries.  Just the name only and not one per page as the documentor does- 695 pages!  Thanks!
bill mosca say:
I have hundreds of queries in my Access app and am constantly scrolling to find the one I want to work with.  How can I print a simple listing of all of my queries.  Just the name only and not one per page as the documentor does- 695 pages!  Thanks!


Here is a function I use. It puts the list of names in a text file.


Function QueryList() As String
'Purpose  : put list of queries in a text file.
'DateTime : 5/9/2000
'Author   : Bill Mosca
'Requires :
    Dim db As DAO.Database
    Dim qdf As DAO.QueryDef
    Dim intF As Integer
    Dim strCurDir As String
    Const cstrFile As String = "\QueryList.txt"

    On Error GoTo err_PROC
   
    'Get a number for a new file object
    intF = FreeFile
   
    strCurDir = CurDir
   
    'Delete text file if it already exist
    If Dir(strCurDir & cstrFile) <> "" Then
        Kill strCurDir & cstrFile
    End If
   
    Open strCurDir & cstrFile For Output As intF

    Set db = CurrentDb
    For Each qdf In db.QueryDefs
        If qdf.Name Like "~*" Then
            'skip these.
        Else
            Write #intF, qdf.Name
        End If
    Next
   
    MsgBox "List completed. File located at " & strCurDir & cstrFile

exit_PROC:
    On Error Resume Next
    Close #intF
    Exit Function

err_PROC:
    Dim strErrMsg As String
    Dim lngIcon As Long
   
        strErrMsg = "Error " & Err.Number & " (" & Err.Description & ") " _
                & "occurred in procedure QueryList of " _
                    & "VBA Document Module1"
        lngIcon = vbOKOnly + vbInformation

        MsgBox strErrMsg, lngIcon, "Error"

        Resume exit_PROC


End Function

e.j. gun say:

VBA - Circular Dependencies Between Modules

I was trying to make a linked list using my own type definition in a VBA module.  The type definition was working well until I added a reference to the type itself, like:

Public Type My_Type

    My_Next_Type as My_Type

It's that last part that seems to cause the problem.  Is it not possible to have this sort of definition in VBA?  Are there any good workarounds that don't involve keeping track of things using longs to point to locations in an array?

Thanks,

Eric

andreas killer say:

I was trying to make a linked list using my own type definition in a VBA module. 

Use a class as Andy Pope said, but remove "My_Next_Type as My_Type" and store each class in a collection. A collection is much more faster and easier to handle then a linked list.

Andreas.

Sub Test()
  Dim clsA As My_Type
  Dim List As New Collection
   
  Set clsA = New My_Type
  clsA.My_String = "Hello"
  List.Add clsA
 
  Set clsA = New My_Type
  clsA.My_String = "World"
  List.Add clsA
 
  For Each clsA In List
    Debug.Print clsA.My_String
  Next
End Sub

 

andy pope say:

Try using a Class rather than a Type.

Class My_Type

Public My_Integer As Integer
Public My_String As String
Public My_Date As Date
Public My_Next_Type As My_Type

 

Example code use

Sub x()

    Dim clsA As My_Type
    Dim clsB As My_Type
   
    Set clsA = New My_Type
    Set clsB = New My_Type
   
    clsA.My_String = "Hello"
    clsB.My_String = "World"
   
    Set clsA.My_Next_Type = clsB
   
    Debug.Print clsA.My_String, clsA.My_Next_Type.My_String
   
End Sub
