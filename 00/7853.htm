Insert Card Suit Symbol Into a String - Microsoft Community

statmanhal say:

Insert Card Suit Symbol Into a String

I want to insert a card suit symbol into a string

Suppose cell A1 has a card designation such as As for Ace of Spades, which user entered

I have the following string:

Dim card as string

card = Left(A1,1) & (here I want the suit symbol)

I know spades is char(170) in the symbol font but how do I define that in code.

 I then want to replace the user entry with the correct suit symbol  I don't think I want the cell to be a symbol font for I use that cell for further entry

In other words, user enters As into the cell for Ace of spades. I then want to change that to A(spade symbol) and allow further entry later with normal font

jay freedman say:
You certainly shouldn't use Symbol font. In fact, you don't have to specify the font for the symbol at all, because Excel will figure it out for you. But you do have to use the Unicode character number for the symbol as it appears in one of the "large" fonts such as Arial Unicode or Lucida Sans Unicode, and you have to use the ChrW() function to convert that number to a character.

The numbers you need are

9824 for spade
9825 for heart
9826 for diamond
9827 for club

Try this as an example:

Sub x()
    Dim card As String
    card = "7" & ChrW(9827)
    Range("A1").Value = card
End Sub

One other thing: You're going to have to account for the possibility that the card is a 10 of some suit, so Left(A1,1) isn't going to be enough.

Document #1 is generated by Template #1, which has a macro that prompts the user for various data and puts it into the document.  Some of this data also applies to Document #2, which is generated by Template #2. Of course, since the templates are separate, they're regarded as separate projects.

I tried putting the appropriate variables in the General Declarations section of the macro attached to Template #1, but, of course, since the templates are in different projects, they don't seem to want to communicate.

fumei2 say:

Is there anyway to pass information from Document #1 to Document #2?

Yes, by making each a document object. This is NOT the same as: as the templates communicating. Once a document is created by a template (any template), it becomes an item in the documents collection. Make it an object and VBA can grab everything in it and communicate to any other document (object).

So. you put the "appropriate variables" in a routine in Normal (which communicates with everything); or a completely separate global template (which Normal also is) which can ALSO communicate with everything in the current Word session.
