rohn007 say:

I tested this is in Word 2007, but I suspect you have similar option. I think mac 2008 is not yet ribbonized so I'll include instructions for both.

2003 menu: Format / Paragraph / Paragraph dialog / Line and Page Breaks tab / turn off Keep lines together option.

2007 ribbon: Home tab / Paragraph group / dialog box launcher / Paragraph dialog / Line and Page Breaks tab / turn off Keep lines together option.

This option keeps all lines of a paragraph on the same page.

The option above it in the dialog box will keep multiple paragraphs on the same page.

You have to make this change the default in your Normal style, then you apply your normal style to all incoming text.

Date calculation: end of month - Microsoft Community

lupeant say:

Date calculation: end of month

Hi,

I have a database where we enter contract periods. The start date is entered and the end date is calculated based on the # of months.

However, we would like to have the have the "real" end of month as a result, when we the start date is the 1st of the month.

Example: start date = 1 October 2012 with 6 months end date should be 31st March 2013. For February (5 months) it works fine and gives me 28 Feb 2012.

For March it gives me 30 Mar 2012. It does this with the months with 31 days.

My formula is:

=IIf(IsNull([Forms]![Personnel Events Form]![Number of Months]),"",DateAdd("m",[Number of Months],[Effective Date]-1))

 

Thanks in advance for your valuable feedback.

Lupe

john w. vinson say:

You can use the DateSerial function to return the last day of the month. DateSerial takes three arguments, a year, a month and a day; the trick is to use a day value of 0 - the "zeroth" day of March is the last day of February!

So you could use

DateSerial(Year([StartDate]), Month([Effective Date]) + [Forms]![Personnel Events Form]![Number of Months] + 1, 0)

in place of your DateAdd.
