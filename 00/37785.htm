wunders say:

A lot depends on how you have your computer and router are set up.

 

Make sure that you change the default "Admin" password on your router to something other than what came with it.  This will discourage him changing the way your router works.

 

Routers/switches are good about only sending packets to devices that are being addressed, so it is unlikely he is snooping in on network traffic on your network.  These days, most password-related web sites encrypt the password anyway, so this should not be your major concern.

 

By far the biggest risk is that he can attach to your computer via File and Printer sharing and could potentially get access to any file on your computer.  The biggest determiner here is whether "Simple File Sharing" is enabled.  On XP Home, it is always enabled.  On XP Pro, it can be disabled.  If it is Enabled, then all connections to your computer will authenticate as "Guest" user... so any shared files and folders accessible to "Guest" user would be accessible to this user.  If Simple File Sharing has been disabled, then a connection is possible if he knows or guesses any valid user/password combo on your machine.  "Guest" with no password usually gets you in as "Guest" (with access to a limited number of files).  Windows comes as default with a built-in [often hidden] Administrator user with no password.  If you have not changed the "Administrator" user's password, then he could connect to your machine through your network and become authenticated as Administrator and have access to any file on your computer without you knowing.  I personally use the freeware "Truecrypt" on my computer to store all my sensitive information in a password-encrypted container so even if someone were to get whole access to my computer, the files are still protected.

 

If you are diligent about putting good passwords on all user accounts, then the risk has been reduced significantly.

 

HTH,

  JW

wunders say:

Without more details such as whether you have XP Pro or XP home or which router make/model you have, it is difficult to get more detailed....  but I'll try.

 

Changing the "Admin" router password:

Usually you do this by using a web browser to connect to your router.  The router will be on the same subnet as you are and usually has a ".1" as the last number.  The usual IP address to connect is:

  http://192.168.1.1/

Once you connect, you can change router settings, including changing the administrator password.  Go to your router manufacturer's website and download the manual for your make/model of router and follow the instructions there.

 

If you are really worried about security, then you do NOT use the same password for all your websites, banks, medical sites, etc.  You yourself admitted the consequences if it were known.  Every site needs a different password.  You won't be able to remember the password to them all, so write them down in an encrypted file and encrypt that file with a secure password that you will not forget.  That's what I do.  And just in case I forget that overall password, I went to my bookshelf and wrote it down in the inside margin of my favorite book on my favorite numbered page.  It is time to change all your passwords.

 

Yes, you do have a "Guest" user on your computer.  Windows comes with a built-in "Administrator" and "Guest" user.  You can't delete them (but you can disable them).  They are usually hidden.  Bring up a command prompt window (Start -> Run -> "cmd") and enter the command:

   net  user

This will give you a list of all registered local users on your machine.  Make sure they all have passwords.  The quick way is -- while you have the command prompt window open -- is to type the following command for each user (while logged in as an administrator user) to change their password.  If you have XP Pro, make sure the encrypting file system (EFS) is not is use when you do this or you will lose all encrypted data:

    net  user  <username>  <password>

where you replace "<username>" with a username (no brackets) and same for the <password> you assign.

 

I don't know what to say about your "Suspicious Behavior" message.  At this point in time, the only way to be absolutely sure you don't have malware on your machine is to re-install Windows.

The REFERENCE_BY_POINTER bug check has a value of 0x00000018. This indicates that the reference count of an object is illegal for the current state of the object.Important Information If You Have Received a STOP Code

If you have received a blue screen error, or stop code, the computer has shut down abruptly to protect itself from data loss. A hardware device, its driver, or related software might have caused this error. If your copy of Windows came with your computer, call the manufacturer of your computer. If you purchased Windows separately from your computer, Microsoft provides support. To find contact information for Microsoft or your computer manufacturer, Contact Support.

If you have experience with computers and want to try to recover from this error, follow the steps provided in the Microsoft article Resolving STOP (Blue Screen) Errors in Windows.

The following actions might prevent an error like this from happening again:

  1. Download and install updates and device drivers for your computer from Windows Update.
  2. Scan your computer for computer viruses.
  3. Check your hard disk for errors.

REFERENCE_BY_POINTER Parameters

The following parameters are displayed on the blue screen.

Parameter Description

1

Object type of the object whose reference count is being lowered.

2

Object whose reference count is being lowered.

3

Reserved

4

Reserved
Cause

The reference count of an object is illegal for the current state of the object. Each time a driver uses a pointer to an object, the driver calls a kernel routine to increase the reference count of the object by one. When the driver is done with the pointer, the driver calls another kernel routine to decrease the reference count by one.

Drivers must match calls to the routines that increase (reference) and decrease (dereference) the reference count. This bug check is caused by an inconsistency in the object's reference count. Typically, the inconsistency is caused by a driver that decreases the reference count of an object too many times, making extra calls that dereference the object. This bug check can occur because an object's reference count goes to zero while there are still open handles to the object. It might also occur when the object's reference count drops below zero, whether or not there are open handles to the object.

Resolution

Make sure that the driver matches calls to the routines that increase and decrease the reference count of the object. Make sure that your driver does not make extra calls to routines that dereference the object (see Parameter 2).

You can use a debugger to help analyze this problem. To find the handle and pointer count on the object, use the !object debugger command.

kd> !object address

Where address is the address of the object given in Parameter 2.

To delete a virtual drive, you'll need to click on the "Start" orb, and in the "Start/Search" box, type "Cmd".  When you see the "Cmd" program show up under "Programs", right-click on it and choose "Run as Administrator".  This will open up a command prompt.  Type: subst drive_letter: /path

(where drive_letter is the letter/name of the virtual drive, for example "X", and path is where it was created, for instance on a "D" drive.)  In this case, the command would look like this:
subst X: /D

omoikane1122 say:
Hello my name is Bradford, and I'm an Independent Advisor for the Microsoft Community, and I'll help you with maintenance timing.
SSD's undergo "Trimming" (more accurate to say the TRIM function is run) to remove excess and unused bits from their buffer.
Hard Drives, however, do not undergo flushing or trimming. Instead, Hard Drives experience "defragmenting."
Both the Trim Function and Defragmentation are performed when Windows does regular maintenance which by default is set to once every seven days from the first day and time post setup.
