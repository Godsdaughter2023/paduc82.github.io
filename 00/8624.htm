No, OWA mailbox policies apply to mailboxes (as their name suggests). But you can expand the membership of the security group and pipe it to the Set-CasMailbox cmdlets to set the policy. Here's an example:

Get-DistributionGroupMember DG | ? {$_.RecipientType -eq "UserMailbox"} | Set-CASMailbox -OwaMailboxPolicy newOWApolicy

kagger say:

Trying to create some tick boxes on a word document template

K
john mcghie say:
Start with View>Toolbars>Forms.

On the toolbar that appears, you will see a checkbox.

When you have inserted it, you then need to lock the document for "Forms" to get it to work.

To do that, use Tools>Protect Document and choose "Forms".  No need to bother with a password.

When a document is protected for forms, users cannot type in it.  To allow them to type, surround your checkboxes with section-breaks.  Normally, use "Continuous" section breaks, so you do not cause a page break in the document.

Once you have one or more section breaks in the document, the Tools>Protect command will enable you to select which sections to protect.  Protect only the ones containing the checkboxes.  You may need several section breaks in a paragraph to enable your users to click a checkbox or write an answer in each section.

Look up "Forms", "Fields", "Protect Document" and "Section Break" in the Word Help for more information.

If you need more information, just ask.
