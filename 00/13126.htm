You can disregard the 15.0 "display" version for OneNote; it is still a 16.0 version in the actual assembly.

No, the 2016 libraries are not backward compatible in the sense that if you use functionality that's not in a previous version then your solution will not work with that version.  The best practice is to develop against the oldest version you want to support and use Reflection if you need to use any members that are only in a newer version.
