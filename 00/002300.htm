tasos_k say: Hi,
try this code
export data in a new Sheet

Sub mySort()
Dim ws As Worksheet, newWS As Worksheet
Dim r As Long
Set ws = Sheets("Sheet1") '<< change name
r = ws.Cells(Rows.Count, "A").End(xlUp).Row
Application.ScreenUpdating = False
Set newWS = Sheets.Add
ws.Range("A1:B" & r).Copy Destination:=newWS.Range("A1")
For i = 2 To r - 1
If Left(newWS.Cells(i, 1), 1) = Left(newWS.Cells(i + 1, 1), 1) Then
newWS.Cells(i + 1, 2) = ws.Cells(i, 2)
newWS.Cells(i + 1, 2).Interior.ColorIndex = 6
End If
Next
newWS.Range("A2:B" & r).Sort key1:=newWS.Range("B2"), Header:=xlNo
For Each rr In newWS.Range("B2:B" & r)
If rr.Interior.ColorIndex = 6 Then
rr.Value = ""
rr.Interior.ColorIndex = 0
End If
Next
Application.ScreenUpdating = True
End Sub

When should I use define a new multi-level list and when should I - Microsoft Community

jblt83 say:

When should I use define a new multi-level list and when should I create a new list style?

I'm having a hard time understanding after researching the help options the differences between usinga  custom formatted multi-level list and defining a new List Style.  I'm writing an outlined publication that will have a table of contents based on headings within the outlined list titles and number paragraphs.  Is creating and using a multi-level list or a new List Style with the format that I want the numbering and level formats to be like?   Future publications i write will use the same leveling and numbered outline format so I almost want to have either or both of these included as part of a template to use later.  Any advice?  Which is the best option to use and when?
pamcaswell say:

The only difference between multilevel lists and list styles is that list styles can be named, shared, modified, and deleted. That  little description still leaves your question of when to use one or the other?

To help you decide whether to use a built-in multilevel lists or list styles in you documents,  choose one of Word's built-in lists from the List Library.  In W2010 (beta) there are four that are linked to heading styles.  If one works just fine without any changes, you surely don't need to define a list style.  If you are the only user, even several  small setting changes may not mean that you need a list style. By small, I mean, for example, adding  or changing a word before the number.  A multilevel list that is set up in a template will be in any document created from it.

I suspect that most people will never need to define a new multilevel list, let alone a list style.    So, when should  you use a list style?

These are just examples:

  • If you have unusual number schemes (First, Second or 00001, 00002--new to W2010), irregular indents for numbering levels,  numbering levels linked to user-defined styles, number font formatting that is different from the heading style it is linked to, or appendix numbering
  • If you want to use such a list in a variety of documents (different templates)
  • If you are using a such a list in documents with many writers or revisers--who will introduce multilevel lists from other documents (and mess up yours)
  • If you want other users to be able to choose a list by name

it would very likely be worth it to invest the time in defining a new list style.

HTH,

Pam
