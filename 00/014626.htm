mikeybrighton say:

I think I may have found a solution to this problem.  I found a System Preferences tool called Secrets (see http://secrets.en.softonic.com/mac) that lets you set the minimum size of font that will be "antialiased" (i.e. smoothed).   Changing the figure to 7 makes all text in Powerpoint over 7pt nice and smooth.  You have to close Powerpoint and reopen it for this change to be applied.

 

You can also change the settings via Terminal.  Looks a bit scary, but it's easy enough to revert to defaults.  (I found the commands from the Secrets web site at http://secrets.blacktree.com)

 

Open Terminal and type in (or copy and paste from here if you prefer):

 

defaults read -g AppleAntiAliasingThreshold

 

...and then hit the Return key.  This will give you the current minimum size.  On my Mac it was set at 8 (which seemed odd, since I was definitely getting the jaggies at 8pt and 8.5pt, but see below).  Note that -g is a shorthand for -globalDomain which is a setting for all applications.  It's apparently possible to set this antialiasing threshold for individual applications if you want.

 

Then type in the following to set the minimum to 7pt:

 

defaults write -g AppleAntiAliasingThreshold 7

 

... and then hit the Return key.  Then, repeat the first (read) line to show that the new threshold size is 7 (in Terminal use the up arrow on the keyboard to scroll back through commands until you find the "read" command.

 

Re-open Powerpoint and you should find that the jaggies have gone for all text below 8.  This seems a bit daft, but when I test this, setting the threshold actually means that antialiasing only applies to 8 and above (i.e. 7pt and 7.5pt are jaggy still).  This explains why, even though the default is 8, I was still getting the jaggies on 8pt and 8.5pt.


Of course you can play with this to make all your text jagged if you want.  Try setting it to 48 and see what happens!  Yeuch!


(Curiously Word in Office 2011 reacts differently.  If you set some text in 6pt and the Threshold size as 24 the 6pt text is jaggy, at 28 and 7pt is jaggy too.  Strange and inconsistent.  Maybe this is because it's more normal to have tiny text in Word than in Powerpoint?)

 

Well, problem solved for me.  I hope others find this useful.  Now all I've got to do is persuade my co-workers to stop using such stupidly small text in Powerpoint slides!

Assuming that the "Set left and first indent with tabs and backspaces" option is selected in the AutoCorrect dialog box (Office button | Word Options | Proofing | AutoCorrect Options | AutoFormat As You Type), you can demote to the next lower level of a multilevel list with the Tab key, and you can promote to a higher level with Shift+Tab.

Note that tabbing does not work for the first level 1 item of a list. However, you can always use Alt+Shift+Left arrow (for demoting) and Alt+Shift+Right arrow (for promoting).

Pressing the Enter key has changed for multilevel lists in Word 2007 and 2010; when you press the key, Word promotes the list, and it's only when you are back at level 1 that pressing Enter will clear the list formatting entirely. To suppress numbering (without changing list levels), you can place the insertion point at the beginning of the list item (immediately to the right of the number) and press Delete once.

To quickly clear the numbering for a list item, you can press Ctrl+Shift+N; this (re-)applies Normal style to text. Any indentation will be removed as well, unless Normal style includes indentation of course.

Note that the safest approach to multilevel lists is to carefully define it with a paragraph style associated with each level; see http://www.shaunakelly.com/word/numbering/numbering20072010.html and http://www.shaunakelly.com/word/bullets/controlbullets20072010.html.

Thanks for any help or guidance.

ericyang_msft6286 say:

Hi Mark,

Actually, this is not feasible, either in Office Online or Office desktop application. Actually the folder in your Shared with me section is not a real folder, it is just a mapping to a folder on the OneDrive for Business library of the user who shares the folder with you which is only accessible via the web interface. If you need to save files to the folder directly, you may need to first sync the library down using the OneDrive for Business sync client and then save files to the local synced folder.

Another way to share files with another user is to share a folder on your OneDrive for Business library with the user and then save files which you want to share into the shared folder. Then the folder will appear in the Shared with me section of that user.
