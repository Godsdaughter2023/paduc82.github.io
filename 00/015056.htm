scottgem say:

Your original post was very clear. If our answers don't help, then restate the question. 

But I stand by my answer, you don't store a "remaining balance". You calculate it by creating a query that totals the payments by loan:

SELECT LoanID, Sum(PrincipalPmt) As PmtTotal

FROM tblPayments

GROUP BY LoanID;

Then another query like:

SELECT LoanID, OrigLoanAmt, PmtTotal, OrigLoanAmt-PmtTotal AS Balance

FROM tblLoans, qryPayments INNER JOIN on tblLoans.LoanID = qryPayments.LoanID;

Will display the balances.

alexsun_2012 say:

Hi Kare,

Yes, you need to remove Azure AD connector.

In a Cutover migration, the directory synchronization tool needs to be disabled since Office 365 will provision the user accounts and mailboxes. If the Azure AD is enabled, local AD will sync local users to cloud again and lead to attribute conflicts like proxy addess.

In this situation, please disable your Azure AD connector and resume the migration batch. After the migration is completed, you can update your DNS records and remove the migration batch.

If you want to use directory synchronization tool to manage the user accounts in on-premises AD, you need to convert the on-premises mailboxes into mail enabled users in your Exchange server and then run the SMTP matching. You can refer to this article for SMTP matching: https://support.microsoft.com/en-us/kb/2641663.

If anything is unclear, please let us know.
