daniel pineault say:

First off, you should never name a field like 'Today's date'.  Do not use special characters '/"()!#...., Do not use spaces.  Do not use reserved words.  Instead it should be something more like:  CreationDate or TransactionDate, ... (you get the idea).

The behavior you describe is quite normal.  When you apply a rule for a field's default value, =Date(), it will only be applied moving forwards.  So existing records will not have a value.  Access has no way of know what date to input since it was never asked to track it previously.  You could create an Update query to update Null entries to =Date() just to have a value or you could leave them blank knowing that the blank one are of unknown date.

ken sheridan say: You would need to build a value list for an 'append' query which you'll execute in the code. The value list starts and ends with a parenthesis, separates each value with a comma, and wraps each value in the appropriate delimiter character.  So taking a simple example of three combo boxes whose values are to be inserted into columns of text, number and date data type:

Dim strSQL As String

strSQL = "INSERT INTO MyTable(MyTextColumn,MyNumberColumn, MyDateColumn) VALUES("

If Not IsNull(Me.cboText) Then
    strSQL = strSQL & """" & Me.cboText & ""","
Else
    strSQL = strSQL & "NULL,"
End If

If Not IsNull(Me.cboNumber) Then
    strSQL = strSQL & Me.cboNumber & ","
Else
    strSQL = strSQL & "NULL,"
End If

If Not IsNull(Me.cboDate) Then
    strSQL = strSQL & "#" & Format(Me.cboDate,"yyyy-mm-dd") & "#)"
Else
    strSQL = strSQL & "NULL)"
End If

CurrentDb.Execute strSQL, dbFailOnError

However, I see no reason why you can't use a bound form whose DataEntry property is set to true (Yes) to allow it to be used for inserting new rows only.  The form's RecordSource property would be the name of the table into which the row is to be inserted.  Where the value of the combo box is the value to be inserted into a column then the combo box can simply be bound to the column,  Where you are inserting values from multiple columns in a combo box then add hidden text boxes to the form, each bound to one of the columns into which the data is to be inserted, and in the AfterUpdate event procedure of the combo box assign the values for each of its columns to the appropriate hidden bound text box.

With a bound form the row will normally be saved to the table if you close the form, move to another record or otherwise explicitly save the record, but if you want to force the user to use a command button to save the record, there is a demo of how to do this as SaveDemo.zip in my public databases folder at:

To create an Internet access point for WLAN:


1. Select Desk > Tools > Control panel > Connections > Internet setup.

3. If Internet access points are already available, the setup asks if you want to use an existing access point as the basis for the new one. Using the correct network ID ensures that the data traffic is routed directly to the desired destination network. Especially virtual private network (VPN) software may restrict data traffic for a certain destination network. Network ID can be used to filter Internet access points when establishing an Internet connection.
- Internet settings type - Set the connection type to Wireless LAN.
5. Press Next.
6. Specify the following:
- Network mode - Select Infrastructure to allow devices to communicate with each other and with wired LAN devices through a WLAN access point. Select Adhoc to allow devices to send and receive data directly with each other; in this case, no WLAN access point is needed.

a. user say:

Hi Septimus98,

Thank you for contacting Bing Ads Support. Here’s how to do this:        

  1. Click Campaigns along the top of the page, then click the Ads tab.
  2. Click Create an Ad.
  3. Complete each of the fields. Above where you want the keyword to appear (for example, in the Ad title or in the Ad text), click dynamic text, and then click Placeholder {KeyWord}. If desired, you can add {KeyWord} to multiple places in your ad.
When implementing dynamic text in your ad copy you should provide a default string e.g. {Param1:default} that the system will use if Param1 for a keyword is null or empty, or if including the Param1 substitution value will cause the expanded string to exceed the element’s limit; otherwise the ad will not serve.

When you use any dynamic text, such as {param1} or {keyword}, make sure you don't end up with ads that exceed the character limits:

  • Ad titles: 25 characters
  • Ad text: 71 characters

Ads with dynamic text longer than the character limits won't display. Using default text in your ad title (or the ad text) can keep your ad text within the allowable character limit.

You define defaults for dynamic text by typing a colon and the default text after the placeholder name: {param1:default}. The default text appears in your ad whenever the ad would otherwise exceed character limits.

If any default value appears in your ads in this way, then default values for all parameters are displayed in the ad. The ad will not use default text for one parameter while using substituted text for another.
