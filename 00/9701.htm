Don't mind Arshad's response. All too often he's in a hurry.

I think that using bookmarks and { REF } fields might be a good choice.

1) We start by defining a bookmark in the Notes Section. Select the words "optimal experience" (today, my highlight color is green):

2) Select Insert > Bookmark in Word's main menu. You need to assign a name to the bookmark. Here I used part of the book title, plus a number.

3) The bookmark is now defined. You can put it in evidence by selecting Word > Preferences > View > Show > Bookmarks. It will appear surrounded by [large brackets]. Having defined the bookmark, we now go back to the reference at the top. Select that text, since we'll replace it by a cross reference to the text in the Notes Section. Then choose Insert > Cross Reference in Word's main menu and enable the indicated choices ("Bookmark" and "Bookmark text"):

4) Click Insert and the original text is replaced by text pulled from the bookmark (that is text from the Notes Section). It's a { REF } field, not text. To see that, right-click (or Control-click) on it and choose "Toggle field codes." Note the \h, which tells Word it's a hyperlink:

5) Check that the hyperlink works. To do that, hide the field code (again, using right-click) and click on the field result. It should take you to the Notes Section.

6) For convenience, you can tell Word to highlight fields with a gray shade, as in the last screenshot. To do that, select Word > Preferences > View > Field Shading and go for "Always" or "When selected." The gray highlight does not print and does not travel with the document, as it is a personal preference.

How do I calculate the average over time using SQL? - Microsoft Community

jon80 say:

How do I calculate the average over time using SQL?

Unfortunately, you cannot use the SQL AVG function to calculate the average over time. The SQL AVG function averages over all the rows received by the query, not just the number of dates. For example, if a query requested the average inventory for a cluster of three products in four stores across seven dates (that is, what is the average daily inventory of a brand in a geographic region during a given week), the SQL AVG function would divide the summed inventory value by
84 (3 products x 4 stores x 7 dates). Obviously, the correct answer is to divide the summed inventory value by 7, which is the number of daily time periods.
Because SQL has no standard functionality such as an AVG_DATE_SUM operator that would compute the average over just the date dimension, inventory
calculations are burdened with additional complexity. A proper inventory application must isolate the date constraint and retrieve its cardinality alone
(in this case, the 7 days comprising the requested week).
Then the application must divide the final summed inventory value by the date constraint cardinal-ity. This can be done with an embedded SQL call within the overall SQL state-ment or by querying the date dimension separately and then storing the resulting value in an application that is passed to the overall SQL statement.
Kimball (P.72)

SELECT "Week beginning " & #2012-08-19# AS WeekBeginning,

In reality you'd obviously want to use parameters for this.  While that's no problem with the dates the IN operator does not accept a parameter as its value list.  There are two ways of simulating this, however, published by Microsoft at:

http://support.microsoft.com/kb/100131/en-us

The second method, using the GetToken and InParam functions, is more reliable as it takes account of values which might be a substring of another value and give a false result with the first method.  The parameters should preferably be controls in a dialogue form of course.  Text boxes are fine for the dates (formatting them to short date so as to activate the date picker).  Multi-select list boxes can be used for the products and stores, looping through the ItemsSelected property of each to fill hidden text boxes on the form with the comma separated value lists.  The hidden text boxes would then be referenced by the InParam function calls in the query.

hansv mvp say:

This is not specific to dates; the same goes whenever you want to group data, whether by date, by product, by store, or whatever.

Let's say you have a table tblInventory with fields Product, Store, InvDate and Quantity. The following query will return the average quantity per date:

 

SELECT Avg(QuantityPerDate) AS AverageQuantity
FROM (SELECT InvDate, Sum(Quantity) AS QuantityPerDate
    FROM tblInventory
    GROUP BY InvDate)

 

Alternatively:

 

SELECT Sum(Quantity) AS TotalQuantity,
    (SELECT Count(*) FROM (SELECT DISTINCT InvDate FROM tblInventory)) AS NumberOfDates, 
    [TotalQuantity]/[NumberOfDates] AS AverageQuantity
FROM tblInventory

ChrisRaeburn,

I've explained this dozens of times here since earlier this year, but no one else seems to want to understand how these scam popups have changed over the last year or so.

Though all of the methods suggested to identify or remove malicious software from either the browser or Windows itself are valid if the popups are persistent and won't stop no matter what else you do, in most cases these are unnecessary if they can be stopped temporarily by either closing the browser process with Task Manger or shutting down and restarting Windows as you've done.

In 9 out of 10 cases today, these popups are just the result of advertisements displayed on the web page or pages being browsed, invoked based on the information that a particular advertiser has collected about your browsing habits over time.  In other words, the type of browsing activity they've collected from your PC fits the profile of others who've been known to call the scammers in the past and pay, so they've chosen you as a target to display their scam popup message and are thus invoking it at seemingly random (they aren't) times.

The difficulty here is that the advertising systems are now so large and complex, that determining where these are truly coming from is difficult as well, even if you're aware of which primary website might be displaying them.

Typically it's not possible to stop these with antimalware, since they aren't truly malicious code, but simply tiny scripts that launch what are otherwise nothing more than simple HTML web pages.  Though they're relatively easy for a human to recognize, the tools available within most antimalware don't see them either because they aren't malware or because they don't perfectly match earlier popups already identified.

That's why many here suggest using an advertising blocker to stop the ads that are initially used to redirect and launch the popups, which tends to work in some cases, but not always.

The other technique I personally prefer is to make relatively extreme privacy settings within Windows and the Microsoft browsers installed, since I almost never use 3rd-party browsers.  Configuring the browser to not display targeted advertising via setting; "Send Do Not Track requests" to sites you visit, or turning off the Windows 10/8 Privacy setting to "Let apps use my advertising ID" which also resets the ID, have the effect of suppressing most of these popups.

It's also best to delete and then turn off 3rd-party cookies as well as set 1st-party cookies to prompt and only allow those for websites you trust, like those for your bank, major commercial sites where you shop and possibly those where you read news or otherwise browse regularly.  This older tracking system is still used as well, so suppressing those you can will reduce the amount of information the advertising networks have about you.

Of course, the undesired side effect of these settings is that you won't see other targeted advertising specific to yourself, so this method only works if like me you don't care and are willing to see ads that often have nothing to do with your preferences.  In my case it doesn't matter, since the type of browsing I do often has little to do with my personal interests and more to the interests from research for others in this and other forums, so the targeted ads I receive would often be for items those others have discussed as well.

These popups are just showing one of the flaws in the online advertising systems and personal privacy controls that few understand.  That's why you'll keep hearing the same old, tired attempts to detect and remove malware that doesn't exist or block the advertisements after the fact, while the true problem is abuse of the tracking and advertising systems within the web browsing ecosystem.
