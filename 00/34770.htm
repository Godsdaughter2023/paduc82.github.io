Skype for business app instant message permanent delete

Hey,

I am developing a data security study and I would like to know if altough me and a colleague both erase our messages from the outlook they will still be saved anyway on the company server. Is this possible? we do not find the message history on outlook and on skype app.

Thank you. kind regards

rick song msft say:

Hi Jmfmy,

 

The Skype for Business conversation history is saved on Office 365 server. If an user has enabled the option “Save IM conversations in my email Conversation History folder”, then it will have a copy to this Outlook folder. If you and your colleague have both deleted the conversation history emails in this Outlook folder, it will still be saved on Skype for Business server.

 

You can see the history in the Conversations tab in Skype for Business.

 

If you and your colleague both deleted the conversation history in Skype for Business Conversation tab, they will be deleted from the server permanently.

 

If you want to make the conversation history can not be deleted permanently(or recoverable) from the server, you need to enable in-place or litigation hold for the user, then make the user enable “Save IM conversations in my email Conversation History folder”. Then if the user have deleted the history from his Outlook and Skype for Business, the conversation history emails can still be recovered by managing eDiscovery.

Rick

I resolved my issue by 1st uninstalling the VGA driver for SEE2 UV150 TRIUV150 via the "Uninstall or change a program". Then rebooting the PC's (had 2 PC's with the exact same issue), then allowing Windows 10 update to automatically find and install the correct driver. I checked the device manager and Windows installed the driver

 (Triger 1+ Graphics Card).

Apparently the old driver for the SEE2 UV150 TRIUV150 USB adapter which we had manually installed originally when we installed the USB adapter was no longer compatible with the latest version of Windows 10 (version 1703). The Intel HD Graphics 4000 driver would attempt to start but then it would unload and we would end up with the default Microsoft Basic Display driver and no 2nd monitor. Although the old original driver had been working perfectly fine with the previous Windows 10 release.

In any case that was what worked for me:

1. Manually uninstall the old driver via

     Control Panel -> Uninstall a program

2. Restart the PC

3. Let Windows do its thing and install the driver
