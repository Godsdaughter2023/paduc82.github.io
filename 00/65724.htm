a. user say:

It is possible that incorrect configurations, corrupt or broken-missing files in your version of windows could cause updating issues, if it has nothing to do with any installed software clashing with or preventing the update, such as a third party antivirus or firewall, which you may want to temporarily switch off whilst updating.

Try running the command prompt tool as administrator (in windows 8's metro screen, after right clicking and selecting the 'all apps' screen, it should be listed on the far right), and then use the system file checker function by entering

'sfc /scannow'

into the black box and hitting ENTER.

If some files cannot be fixed, try running the command prompt tool as administrator again (http://www.eightforums.com/tutorials/9564-run-administrator-windows-8-a.html), with the Deployment Image Servicing and Management function, which can restore system configurations, by entering

'dism /online /cleanup-image /scanhealth'

and then ENTER

(this will scan for issues)

'dism /online /cleanup-image /checkhealth'

and then ENTER

(this will scan for corruption)

and again with

'dism /online /cleanup-image /restorehealth'

and ENTER.

(fix corruption)

Note the spaces after certain text above for the functions to work.

You may also want to try a hard drive diagnostic tool to check your drives həalth before attempting another update.

If none of that helps then you may want to attempt an update to 8.1 after using windows System Restore, to take your installation back to a time when the update may possibly show no software exception error (depending on the size of your drive a restore can literally take hours), or if you have a windows 8 restoration disc, attempt an update to 8.1 after a fresh windows 8.0 install.

Make sure you always back up all of your important data before attempting any system update or restoration.

greg carmack - windows mvp 2010-2020 say:
Hi GN. I'm Greg, an installation specialist and 9 year Windows MVP here to help you.

Please post a screenshot of Disk Management, which I read like a doctor reads X-rays. Follow the steps here so I can see everything needed to advise you: http://answers.microsoft.com/en-us/windows/foru...

FYI if you have WIndows on an SSD and you install apps or programs on a hard drive you will lose the speed benefit; only moving files to them doesn't compromise SSD speed, so best to only move the User folders as shown here:
http://www.dummies.com/computers/operating-syst...

I also do not recommend moving apps or programs off of C because they write keys to the Windows registry on C and should remain a part of it's image. A better approach is to have a separate data partition with the User folders to keep the image smaller, so if Windows ever becomes irreparable you can restore C in 20 minutes from recovery media and your files will be current and safe in their own separate partition. If they are left on C they will be dated from when the image was taken.

The best free imaging app for saving the backup image externally is here:
https://www.howtogeek.com/howto/7363/macrium-re...

That said if you still want to move them then we can try to find a way. Changing App location only affects new ones, though, unless you want to uninstall and reinstall.

I hope this helps. Feel free to ask back any questions and keep me posted. If you'll wait to rate whether my post helped you, I will keep working with you until it's resolved.
