wunders say:

I am assuming that these files were encrypted using XP Pro and the Encrypting File System (EFS).  If this is the case, your only hope to recover these files is to use System Restore to restore your computer to a time when you did have access to these files -- even though your system might be slow.  Once you do this, you must backup the encryption certificate.  Consider backing up the files as well.  Once you have backed up the encryption certificate, you can use System Restore to go back to a time when your computer was running better then restoring the Encryption certificate from your backup.  Before using EFS, make sure you read thoroughly the following article:

"Best practices for the Encrypting File System"
   <http://support.microsoft.com/kb/223316 >

Pay particular attention to the paragraph entitled "Why you must back up your certificates".

My personal recommendation for encrypting data is to use freeware Truecrypt rather than EFS.  With EFS, Windows controls the encryption keys and a crash can cause loss of everything.  With Truecrypt, you maintain the encryption key yourself via a passphrase.

Truecrypt:  <http://www.truecrypt.org >

Hope this Helps,
  JW

Windows Update error 0xc1900107 occurs when you are trying to install a major update. This could be caused by broken components of Windows Update or due to a software conflict.

Running the System File Checker and DISM can help resolve your concern. The System File Checker (SFC) is a utility in Windows that scans for corrupted Windows system files. If it finds a problem, it will try to replace the problematic system files from a cached version that’s on your PC. If SFC can’t replace the files, the Deployment Image Service and Management Tool (DISM) can check the corrupted files then download and replace the files with Windows Update. To do this, refer to the steps below:

  1. On the Start menu, type Command Prompt. Right-click the result and then click Run as administrator.      
  2. Copy or type each command followed by hitting Enter:
      sfc /scannow
      dism /online /cleanup-image /restorehealth

You can also perform a clean boot. A clean boot will help eliminates software conflicts that occur when you install a program, update, or when you run a program in Windows 10. For instructions, refer to this article.
