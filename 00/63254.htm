I had the same problem and found a solution that worked for me.

First, start Windows and wait until touchpad smart gestures are working. Open the Task Manager (CTRL+SHIFT+ESC) and if you do not see multiple tabs, click "More Details" on the bottom left corner. Go to the Details tab and sort processes by Name or by Description in descending order.

Find the processes related to Asus Smart Gestures (in my case they were AsusTPCenter.exe, AsusTPHelper.exe and AsusTPLoader.exe). Right-click one of them and select "Open File Location". The program folder containing the executable files should open.

Open the Run dialog box (WIN+R), type shell:startup and confirm. A hidden (and probably empty) folder named Startup should open.

Now simply create shortcuts in the Startup folder to the executable files related to Asus Smart Gestures that you have previously identified - I did it by dragging them to the Startup folder while holding left-ALT.

Restart and voilà!

I have 100 processes on startup. Its eating my memory, and killing my - Microsoft Community

digitalfear say:

I have 100 processes on startup. Its eating my memory, and killing my fan.

On a fresh clean startup, i have about 100 processes running. its taking up 3 gb of memory when my laptop is idle, and my fan is always running on full speed. It says i have 24 running by me, the user, and the rest are from system. Is there anything i can do?
shenan stanley say:

The processes are controlled by whatever spawns them.  I have 84 processes right now using 2.15GB of memory (32 owned by me, the rest by SYSTEM, LOCAL SERVICE and NETWORK SERVICE) - so I guess yours isn't *too* out of bounds...

Now - I do have KeePass, Windows Live Messenger, two IE8 windows (4+ processes), one Explorer window (1 process + the 1 for the desktop), TeamViewer (3+ Processes), MalwareBytes (2 processes), PlayOn (2 processes), ATI Catalyst (3+ processes), one gadget (1 process), Virtual CloneDrive (1 process), eSet NOD32 AV (2 processes) and Windows Task Manager (of course 1 process) that I can *see* without task manager.  With Task manager I can note I also have "airprint" for faking out my iPad, two or three services for iTunes/Quicktime, one for Flash Player (likely because of an open IE tab), the Java update scheduler, a couple for a camera I have - waiting for me to hook it up and it can start the app for it, at least one printer status watcher (I have two network printers), a SlingBox Agent process, the printer spooler (that may be two processes), a Windows update process and several (3+?) Windows Live type processes.  Then everything that is getting hidden in the SVCHOST.EXE process and the various winlogon, system, wininit, etc.

 

I think the pure number of processes does not matter - what matters is how much memory each one is taking up individually and *if* that process should be taking up that much.  Let's face it - if you have 4GB of usable memory and you never use over 3GB - you are okay.  As long as you don't use 4GB - you are okay.  When you start using 4GB consistently - then you have something you might want to correct (buy more RAM.)

 

Looking at my list - and using Process Explorer to *REALLY* see what is going on - I don't see any issue.  Largest user of memory is 143,000K for a SVCHOST.EXE, next is 82,000 K for explorer.exe, MSNMSGR.exe is up there at 80,504 K, 61,884 K for eSet Kernel, MalwareBytes Service is taking up almost 51,000 K, but they all drop rather quickly after that, with only 16 more even above 10,000 K.

 

I could stop some of those from starting up - but everything seems to work fine.  I even played "Dungeon Seige III" last night at 2560x1600 resolution with my 4GB of memory and didn't seem to have any trouble.

 

Is your big concern the fan?  It's not really the amount of memory being used that is causing that to kick on - more likely the video display and processor.  Those are the two components that generally heat up fastes in a machine (not saying the memory does not - it can and does - but in general, not by much especially in comparison to the two mentioned pieces of hardware.)

 

What does all that mean to you?  Well - a couple of things.

1) Check your power settings and (if you can) when your fan is supposed to be on.  What are they now - can you be more conservative with them and put things to sleep/in low power to make things better when you truly are not using the system?  (The video is a no brainer - turn off the external display when not around after XX minutes.)  Some Performance Settings just leave your fans on full blast.  SOme people want that, others think it is too noisy.

2) What is using the processor and/or video when you are just sitting there, watching?  Process Explorer (mentioned already) might be a good thing to run and stare at - full screened and sorted by CPU.  My "System Idle Process" bounces between 95.5% and 97.8% when not using the machine.

 

Hope that helps, somewhat...

vijay a. verma say:
Hi redzone2003
I am Vijay, an Independent Advisor. I am here to work with you on this problem.

Please try these options one by one and check after each option.

1. If you are able to open MS Store, open MS Store > Click on your profile picture on top right and sign-out. Then sign-in again.

2. Windows Key+R > Type services.msc and press Enter > Locate Background Intelligence transfer and ensure it is set to Start and Automatic. Also do the same for Windows Update service as well

3. Run Windows Store Apps Troubleshooter
Windows Key+X > Click Settings > Click Update & security > Click Troubleshoot > Scroll down to the bottom > Click Windows Store Apps > Click Run the Troubleshooter

4. Reset Windows Store through Command Prompt
Type cmd in Windows Search box > Right click on Command Prompt > Run As Administrator > Type WSreset.exe and click Enter > Reboot your computer

5. Re-register All Store apps (You will get many Reds, ignore them)
Windows Key+X > Windows Powershell (Admin) > Copy the following from below and right click in Powershell to paste > Enter > Restart your computer

Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

6. Uninstall & Reinstall Store
Windows Key+X > Windows Powershell (Admin) > Copy the following from below and right click in Powershell to paste > Enter

Get-AppxPackage -allusers *WindowsStore* | Remove-AppxPackage

Copy the following from below and right click in Powershell to paste > Enter > Reboot your computer

Get-AppxPackage -allusers *WindowsStore* | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}

7. Check your Antivirus
Temporarily uninstall / disable AV

Do let me know if you require any further help on this. Will be glad to help you.

Source - https://answers.microsoft.com/thread/b8f36cad-9...
