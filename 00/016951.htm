http://answers.microsoft.com/en-us/office/forum/office_2013_release-excel/vlookup/50e05526-9bb3-4150-8c41-fe122ef00f7c

Let's say that the product type, model name and price are in the Product List worksheet like this:

In C7 you might have formulas that look like this; this one gets the Type:

=VLOOKUP($B$5,'Product List'!$A$1:$D$3,2,False)

in C8 it changes to pick up information from the 3rd column (Model) of the table instead of the 2nd

=VLOOKUP($B$5,'Product List'!$A$1:$D$3,3,False)

and finally another change in which column of info to return to get the price

=VLOOKUP($B$5,'Product List'!$A$1:$D$3,4,False)

Does that help any?  The $A$1:$D$4 should include all of the columns and rows of your table on the Product List worksheet that you need to pull information from.  It's usually easiest to set them up to reference all of them than to just 'minimize' to just the ones you need like this:

Doing it this way usually creates more work for you in the long run.

That is a classic sign that the document is corrupt.

It can be a lengthy chore to fix it: make sure you have your backup handy!

There are some things you can try:

1)  Create a new blank document and save it.

2)  Copy all of the text in the old document.

3)  PASTE into the new document.

This will accept all the changes and give you a clean document.  But it will probably crash trying to do this...

Another thing you can try is to use File>Save As to save the document in RTF (Rich Text) format.  If that succeeds, open the RTF version and re-save it as .DOCX.  You may lose some things from the document doing this, because RTF cannot describe all of the things that .docx can contain, but the losses should not be noticeable.

Chances are the cause of the problem is one table with merged rows and columns, or one floating graphic, of a bad set of numbered paragraphs somewhere in the document.
