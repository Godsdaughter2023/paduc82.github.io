The hard disk gets partitioned into a few small areas and one large one during Windows install. Sometimes during repair, the drive letters are assigned differently than you expect (for example C: might become D: or another letter). If you then insist on installing to C:, it might actually be pointing to one of the smaller partitions instead of to the Windows partition that you want. The letter used does not matter in this mode.

If you want to keep her files and they still exist, you can choose to do that, as it will keep the data only.

But if you are willing to delete all files and start over, boot the USB, choose Custom install and first delete all partitions. Install directly to the unallocated space of the hard disk. Windows setup will create the proper partitions.

james yan msft say:

Hi Here2share,

Generally, this error indicates that the group site has been deleted.

About how to solve this, first we need use PowerShell commands to check the status of the deleted site:

1. Download and install the SharePoint Online Management Shell from this article.

2. Launch SharePoint Online Management Shell (search and right click the "SharePoint Online Management Shell" and choose "Run as Administrator"), connect to SharePoint Online via the following commands using your Office 365 global admin account:

Connect-SPOService -Url https://contoso-admin.sharepoint.com

(replace contoso with your tenant domain)

To see all the deleted sites, you can run this command:

Get-SPODeletedSite | fl

If the deleted site is listed and DaysRemaining is NOT 0, we can restore the deleted site by running 

Restore-SPODeletedSite -Identity siteURL

If the deleted site is not listed, we can't restore that site.

James
