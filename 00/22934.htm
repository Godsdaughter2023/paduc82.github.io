Wups! That will get you in real trouble if the actual name happens to contain the text string PO. For example, "Walpole Community Schools PO#12343" will return "le Community Schools..."

Include the # sign if you can count on it reliably being included: InStr([customerName],"PO#"...

If you want to do on your own -

1. Make a backup of your workbook.
2. Open your workbook and ALT+F11
3. Locate your Workbook name in Project Explorer Window
4. Double click on your worksheet name
5. Copy paste the Macro code given below
6. Save your file as .xlsm 

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim r As Long
    Dim StrDOW As String
    Dim Cell As Range, Rng As Range
    Dim ArrDOW
    StrDOW = "Sun,Mon,Tue,Wed,Thu,Fri,Sat"
    ArrDOW = Split(StrDOW, ",")
    On Error GoTo ExitSub
    Application.EnableEvents = False
    Application.ScreenUpdating = False
    If Intersect(Target, Me.Columns(2)) Is Nothing Then GoTo ExitSub
    Set Rng = Range("B2:B" & Range("B" & Rows.Count).End(xlUp).Row)
    For Each Cell In Rng
        Cell.Offset(0, 1) = Application.Match(Left(Cell, 3), ArrDOW, 0)
    Next Cell
    Range("C1") = "Helper Column"
    UsedRange.Sort Key1:=Range("C1"), Header:=xlYes
    Columns(3).Clear
ExitSub:
    Application.EnableEvents = True
    Application.ScreenUpdating = True
End Sub

Some points to consider:

  1. The Microsoft how-to doesn’t schedule a Full Scan; it schedules a Quick Scan.

  2. No one needs to schedule a Quick Scan, because Automatic Maintenance is already running a daily Quick Scan. So the rationale for the Microsoft how-to is obviously flawed.

  3. The Quick Scan that’s scheduled by the how-to is also faulty, because it will only run during system idle states in Windows 10, exactly like the Automatic Maintenance Quick Scan.

  4. The Quick Scan scheduled by the how-to quite clearly conflicts with the Automatic Maintenance Quick Scan when the two scans are running concurrently.

  5. Setting up a Full Scan with the Windows Club method (by modifying the command line for the action) also schedules a scan that will only run during idle states in Windows 10. The scan will be cancelled with any user activity and then be restarted when the system returns to idle.

  6. Setting up a Full Scan with the Windows Club method will also convert the daily Automatic Maintenance scan into a daily Full Scan (that only runs during idle states).

  7. Now if you just sit back and give this dysfunctional Full Scan a chance to run, that probably won’t work either – because the scan will be terminated automatically in exactly 1 hour, (the default maintenance window for Automatic Maintenance, methinks).

  8. The aborted Full Scan will have a Last Run Result of (0x103) in Task Scheduler, and a 1002 Event ID in Event Viewer (cancelled), but it will nonetheless update the “Last scan” info in the Windows Defender interface, so the user will be none the wiser. greginmich say:

After upgrading to Windows 10 I can no longer schedule a full scan daily via Windows Defender. Why is that? I've searched in settings and there's no option to schedule a middle of the night full scan. I'm not sure why, but can someone answer that question for me please?

Why is there no option to schedule a Full Scan?

With Windows 8/8.1 the Windows Defender Scheduled Scan task was added to the package of tasks controlled by the new Automatic Maintenance engine – which is simply a higher-level scheduled task that triggers the tasks in the maintenance package. In its default mode, the Automatic Maintenance engine is charged with running multiple tasks within the confines of 1-hour “maintenance windows” that are opened when the system reaches an idle status and then closed again with any user activity. So the scan run by Automatic Maintenance will be cancelled (not paused) by any user activity and then have to wait for the next idle state to be restarted – and then, if it’s not interrupted by any user activity, it will be automatically terminated at the end of the 1-hour maintenance window. For busy users, it’s often hard enough to get the Automatic Maintenance Quick Scan finished under these conditions, and a Full Scan is pretty much out of the question. Consequently, there’s no longer a built-in option for a Full Scan.  

Do I still need to schedule a Full Scan?

The Windows 8/8.1/10 version of Windows Defender also has authentic real-time protection (on-access file scanning) which will automatically detect and clean any active malware that it encounters (and recognizes), and that doesn’t leave much for a Full Scan to find – so the need for regular Full Scans really just isn’t there anymore. Nevertheless, it’s not that difficult to schedule a middle-of-the-night Full Scan with the Create Basic Task Wizard:

How do I schedule a Full Scan?

Open Task Scheduler (Start button > All apps > Windows Administrative > Task scheduler)

1.) In the right-hand (Actions) panel, click on Create Basic Task and name the task “Weekly WD Full Scan” and click Next

2.) Click on the Weekly option button and click Next, and then pick a time and day of the week and click Next

3.) We’re using the default action (Start a program) so just click Next

4.) Browse to (or just enter) "C:\Program Files\Windows Defender\MSASCui.exe" in the Program/script box (with quotation marks)

5.) Enter -FullScan in the Add arguments box (no quotation marks) and click next

6.) Check the “Open the Properties dialog for this task…” box and click Finish

7.) Change the account to "system" and then click on the Conditions tab, and check the “Wake the computer to run this task” box (this is what wakes the PC up and prevents it from sleeping) and click OK
