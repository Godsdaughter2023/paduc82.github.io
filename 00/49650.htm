charitablecomputing.org say:

I think I may have finally resolved the issue.  Took some time, but finally have positive results... Let's see if this works for y'all.

First:  Make sure you have the correct services running.  Right click the start menu, click run and type in "services.msc"  Make sure all of the following services are set on automatic (delay start).  Note, if your system is fast enough, you may not have to put it on delay start.  I only did that not to bog down the boot time.

     DHCP Client

     Function Discovery Provider Host

     Function Discovery Resource Publication

     SSDP Discovery

     UPnP Device Host

Second: Make sure you have the correct Windows components running.  Open settings, click on Apps, and then in the upper right hand corner, you will find Programs and Features.  Now, on the upper left hand side of this new window, you will see Turn Windows Features On or Off, click that.  Make sure SMB 1.0/CIFS File Sharing Support and SMB Direct are both fully checked (not the black box, but a check mark.)  I also have Work Folders Client checked, but not sure if that will affect it.

Third:  Make sure you have Enable Remote Desktop enabled... even if you do not plan on accessing your computer remotely.  Always remember, if you don't have a password on your account, you can't access the computer remotely anyway.  This is found under Settings, then System, and finally the last choice on the bottom left.

Fourth and Final Step:  Make sure you are on a private network.  Click on the network icon on the toolbar, and select Network and Internet Settings.  If under your connection device, it says public, click on either the Ethernet or WiFi link, and change it to Private.

Now, reboot the computer, and see if this works.

andre da costa say:
Causes of this Error

This error has been known to occur as a result of one or more of the following:
Cause 1: BCD is missing or corrupt

One of the most common causes of this error is the BCD that became missing or corrupt. That could happen because of disk write errors, power outages, boot sector viruses, or errors made while configuring the BCD manually.

Cause 2: File system integrity compromised

For the same reasons, the file system itself may become corrupt or damaged. That happens even more often if one turns off a PC while it is in the middle of writing crucial data to the boot sector.

Cause 3: HDD data cable is faulty

Like all components of our PC’s, hard drive data cables tend to deteriorate as the time passes. If the same cable is used for several years(or even across several different computers), then it might be the cause of periodic read/write errors that can affect the loading process.
