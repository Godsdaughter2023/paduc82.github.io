This little demo file illustrates the use of TimeElapsed and TimeDurationAsDate functions to return a duration formatted in hours and minutes, or hours, minutes and seconds.  The examples in the form use relatively short durations, but you can see how it would work with longer durations by means of a little bit of code:

ken sheridan say: You cannot enter a value into a column of date/time data type in a format other than a recognized date/time format, so any value of 24 hours or more would not be accepted.  To do as you wish I would recommend that you store the values in two columns of integer data type, one for the hours, the other for the minutes, and enter them into separate controls in a form, using a label between the controls to show the colon.

To do time arithmetic on the values you can return a number in hours with the following expression:

    [txtHours]+[txtMinutes]/60

Or in days with:

    ([txtHours]+[txtMinutes]/60)/24

You could do arithmetic on the value returned by the last expression above, e.g. the summation of a set of times entered in rows in a table, returning the result of the summation in hours:minutes by calling the TimeElapsed function from my TimeArithmetic demo.

In a report to show the values in a single control you'd simply concatenate them with:

    =[txtHours] & ":" & Format([txtMinutes],"00")

as a control's ControlSource property.

ken sheridan say: I realise that you are not using a column of Date/Time data type.  I only mentioned it to make the point that an alternative methodology has to be used.  We try to direct our replies here not only at the OP but also at any lurkers.

The trouble with using a column of Short Text data type is that it makes aggregation of the values more difficult, as it is necessary to parse the single text value into its constituent elements and convert these to numeric values which can be aggregated.  Combining data from two columns on the other hand to return a single numeric value is a simple arithmetical operation.  Aggregation of the data then becomes a trivial task.  Also, by disallowing Nulls, setting the DefaultValue property of each column to zero, and constraining the value of the minutes column to less than 60 by means of the column's ValidationRule property, makes it easy to ensure that an invalid value cannot be entered.

You might not need to aggregate the values of course, but others reading this thread might want to.  It's a very common requirement with time durations in my experience.

EXCEL BECOMES UNRESPONSIVE WINDOWS 10 - Microsoft Community

johnmcastillo say:

EXCEL BECOMES UNRESPONSIVE WINDOWS 10

This is a work around / solution to an issue we have been working on for two weeks now.

Symptoms: Excel files become unresponsive causing restarts and lost work.

What we found: We conducted a Process Monitor from SysInternals Suite and discovered that DirectX is making a call to Microsoft with Telemetry data. It would not make the connection thus starting what we all know as a Buffer Overflow. At some point the Overflow takes up all available memory / processing power causing Excel to become unresponsive. Since we use Excel heavily this was a major issue.

Workaround: One of our Users by pure chance decided to open his Laptop lid, while docked, and the screen flashed and all his Excel files became responsive again. This led us to believe that this was a Display issue versus an Excel issue even though it affects Excel. That is when we ran the Process Monitor and discovered the Overflow.

The Fix: At this time we have gone into the Options Menu and Under the Advanced Tab in the Display settings and checked the "Disable Graphics Acceleration". So far this has resolved a majority of the issue. We are also making the same changes with Power Point and Word to ensure it does not happen on these applications. 

Potential Fix: We are researching to see if there is an update to DirectX that may resolve the issue permanently but still have not finished working on this.
