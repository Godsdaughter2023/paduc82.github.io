scottgem say:

When I enter values in this combo box, the employee information (Last name & First Name) prepopulates on the form and I want those values written back to the associated field in the bound table. 

Why? This violates one of the principle of a relational database. Data should exist in ONE place and ONE place only. By having fields in another table for name info you have redundant data. What happens if someone changes their name? Now your bound table is incorrect.

The proper way to handle this is to store only EmployeeID in your child tables. To display their names, you can use a variety of methods (see my blog on Displaying Data from Related Tables). You can use a query, subform, DLookup or other methods.


boblarson say:

The big answer is DO NOT DO THIS!!!.  You should be storing the ID and NOT the name again.  The name should ONLY be stored in one table, the employees table.  You can then bring those fields together again with a query and a query can be used in perhaps 99% of the places a table can be used. 

 

So, don't store that information as it violates the rules of Normalization and, unless this is for a data warehouse situation, it should not be stored in more than one table.
