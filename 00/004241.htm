steve rindsberg say:

Here's one possible approach:

Option Explicit

Declare Function ShellExecute Lib "shell32.dll" Alias "ShellExecuteA" _
                   (ByVal hwnd As Long, ByVal lpszOp As String, _
                    ByVal lpszFile As String, ByVal lpszParams As String, _
                    ByVal LpszDir As String, ByVal FsShowCmd As Long) _
                    As Long
                    
Sub OnSlideShowPageChange()
' This will run each time you move to a new slide
' It's not always reliable but it's easy to try out
    Dim sLaunchThis As String
    Dim lSlide
    
    ' EDIT THIS to the url or file to launch:
    sLaunchThis = "http://www.pptfaq.com/"
    ' EDIT THIS to point to the slide on which you want the launch to occur
    lSlide = 2
    
    If SlideShowWindows(1).View.Slide.SlideIndex = lSlide Then
        Debug.Print ShellExecute(0&, vbNullString, sLaunchThis, vbNullString, vbNullString, 0&)
    End If
    
    ' You may want to add a bit more code, for example to record whether the slide's been
    ' viewed already and if so, launch the url again or not
    
End Sub

Is this the message you see?

tom van stiphout say:

Hi Larry,

A barcode scanner is just a fancy keyboard, as far as Access is concerned. You set focus to a textbox, hit Scan, and the scanned value appears as if you had typed it yourself. Maybe it's a MemberID.

If it's a MemberID, and you want to show the details for that member, the simplest way I can think of is in the textbox' AfterUpdate event (which happens when you tab out of it):

In the textbox AfterUpdate event, write two lines of code:

Me.myPersonName = DLookup("PersonName", "tblMembers", "MemberID=" & Forms!myForm!myMemberID)

Me.myExpirationDate = DLookup("ExpirationDate", "tblMembers", "MemberID=" & Forms!myForm!myMemberID)

(of course I'm just guessing at your objectnames)
