The part about "the font doesn't even look like the font though it clearly states that it's the same font" sounds like what happens when the document is formatted with a font that isn't installed on that computer. Word silently substitutes some other font (according to undocumented rules about what makes fonts "similar"), but the Font box on the ribbon continues to show the name of the original font.

To see if that's the case, open the document on the problem computer; go to File > Options > Advanced; scroll down to the "Show document content" section, and click the Font Substitution button. You may see this dialog box:

If so, the solution is to install the missing font (copy the font file from one of the other computers and paste it into C:\Windows\Fonts).

The other problem is a new one to me. Does the problem computer actually show two identical copies of each page, or is it just that the page breaks are in different places than they should be and the number of pages is doubled?

suzanne s. barnhill say:
To add to what Jay has said, if the problem is just that the number of pages increases (not that actual content is duplicated), then the font problem, combined with manual page breaks, is likely the issue. If the font is just a little larger, so that lines don't break in the same place, then that will cause pages to break differently as well. If they were allowed to reflow freely, you might end up with just one extra page, but if there are manual page breaks, you might well get two pages for every one in the original document. jeeped say:

OK, if column B holds numbers then you can use,

=SUMPRODUCT((Sheet2!A2:A999="Customer A")*(Sheet2!B2:B999))+SUMPRODUCT((Sheet3!A2:A999="Customer A")*(Sheet3!B2:B999))

  ... or,

=SUMPRODUCT((Sheet2!A2:A999=F2)*(Sheet2!B2:B999))+SUMPRODUCT((Sheet3!A2:A999=F2)*(Sheet3!B2:B999))

Note that the ranges start in row 2 so that you are not trying to add a number to he word April. However, SUMPRODUCT() should be cast aside in favour of the more efficient SUMIF() function. With Customer A in F2 use one of these,

=SUMIF(Sheet2!A:A, F2, Sheet2!B:B)+SUMIF(Sheet3!A:A, F2, Sheet3!B:B)

  ... or,

=SUM(SUMIF(Sheet2!A:A, F2, Sheet2!B:B), SUMIF(Sheet3!A:A, F2, Sheet3!B:B))

Note that with SUMIF() you can efficiently use full column references.
