Hi Shiv,

In order to troubleshoot the errors on Reliability Monitor, we need to get the details behind those using System Stability report. The System Stability Report helps you identify changes that contribute to a lower Stability Index by identifying Reliability Events.

 
Reliability Events: These events are tracked in the following category:
a) System Clock Changes: Significant changes to the system time are tracked in this category.
b) Software (Un)Installs: Software installations and removals including operating system components, Windows updates, drivers, and applications are tracked in this category.
c) Application Failures: Application failures, including the termination of a non-responding application or an application that has stopped working, are tracked in this category.
d) Hardware Failures: Disk and Memory failures are tracked in this category.
e) Windows Failures: Operating System and Boot Failures are tracked in this category.
f) Miscellaneous Failures: Failures that impact stability and do not fall under previous categories are tracked in this category, including unexpected operating system shutdowns.

Basically, what we need to do so is use the data it provides to decide what steps you can take to improve your operating system stability.

b) Click on Action Center.
