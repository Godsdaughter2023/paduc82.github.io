albert d. kallal say:

You can read my following article on the runtime here:

 

http://www.kallal.ca/RunTime/Index.html

 

In a nutshell, there's no specific or technical linking between your ONE application and the runtime. In fact there is no  more "attchement" that occurs when using the full edition of MS access.

 

In other words if you install MS word on your computer, then you should be able to browse to any word file on your computer and then double click it and the application will be launched to open the particular file in question. 

 

The SAME goes when you install MS Access on your computer. And the SAME goes fro when you install the MS Access runtime. AFTER you install Access (full or runtime) then you can simply browse to any particular access database on your computer and then a simple double on that file in question will cause just like always windows to automatically launch the correct application in which to open that file with. 

So the runtime and full edition of access in this regards works the same way, and the only real special connection to the application in question is that of the file extension association that windows uses to launch the particular application in question when you clicked on the file.

 

About the only major consideration you need for the access runtime system is that you have to provide the "user interface" and system for the user to navigate around to use things like forms and reports. The reason for this is that the runtime does not provide the built in access menu system. 

 

So your application now has a bunch of nice menus and forms that allows the user to "run and use" your application then the amount of work to run your application under the runtime system is actually zero.

 

However if you have NOT provided startup forms and a user interface to launch reports and forms etc, then you have to spend some time building up these features into your application.

 

So the only real issue with runtime is that you cannot rely on the built in access user interface which does not SHOW or exist in the runtime edition of Access.

 

However at the end of the day, you're installing an access application on your computer, and as noted there's not some particular special association between that application and the many mdb, mde, accDB,accDE files that you copy to that computer.

So once the runtime has been installed on that computer, then you can simply copy any old access database to that computer, double click on it, and the runtime version of access will be launched.  As noted the suitability of your application will thus depend on if you provided menus and the user interface for that particular application.

 

The above concept is explained in the above article of mine.

Inserting a section break in a table

When I insert a continuous section break it splits the table so the "keep lines together" option does not work. Is there anyway I can accomplish this?

stefan blom say:

Adding a section break after a table row will inevitably break the table; there is no way around that.

What you can do is protect part of a table by applying "No changes (Read only)" document protection via the Restrict Formatting and Editing task pane (Review tab | Restrict Editing). Use the "Exceptions" box to choose which parts of the document should be editable. When you are done, don't forget to click Yes, Start Enforcing Protection.

jackie l say:

I have found that using the OLE objects tends to enlarge the data and can cause issues with viewing depending on the workstation and the software loaded.  I have found that saving the data as a text file with the location of the actual image file (.bmp, .jpg) in that field has been the best/easiest way to handle.

Below are some notes on how to view the image or put in a report.  I also have a function for browsing to the file location if you choose to go with this option.  Another advantage is that if the file location can be imported into the field (as any text could) if you have a large amount of records which need an image.

This procedure does not need document viewing software and does not enlarge the data due to OLE Object linking.  Can be used for linking/printing logos.

 

Step 1.  Create a text field in a table (255 characters) ex. FieldPhoto

 

Step 2.  On form or report:

Add FieldPhoto (hidden)

Add Image (Zoom - size mode, Center -  picture alignment).  You will have to browse to a file in order to create the image but you can delete it after the image frame is added

Name the Image (ex. FieldImage)

Step 3.  Code for the OnFormat of the section in the form or report where the image is to be shown:

 

For Forms:

 

Me.FieldImage.Properties("Picture") = Me.FieldPhoto

 

For Reports:  You will need to hide the image if there is no file attached (to avoid errors):

 

If Nz(Me.FieldPhoto)="" then

Me.FieldImage.Visible = False

Else

Me.FieldImage.Visible = True

Me.FieldImage.Properties("Picture") = Me.FieldPhoto

End IF

Set up My Family on the Windows Phone website

 

My Family on the Windows Phone website is the way to turn on app and game downloads for a child's phone. If the child has a Windows Phone 8, you'll also be able to manage their app and game download settings, including game ratings, on the My Family page.

To set up My Family, you'll need to sign in with both the child's and the parent's

If you want the child to be able to download Xbox games as well, you'll need to go to the Xbox website and accept the Xbox Terms of Use on their behalf. This is separate from the Windows Phone Store Terms of Use.

Running into issues? See

Note

My Family is only available for Windows Phone 8.

To set up My Family so a child can download apps and games

  1. On your computer, go to

  2. Sign in using the parent's

  3. On the My Family setup page, click Get started.

  4. To add the child to My Family, click Go. On the sign-in page, enter the child's Microsoft account email address and password. If the child doesn't have a Microsoft account, click Sign up to create one.

  5. On the My Family home page, next to the child's name, click Fix it and follow the instructions to accept the Windows Phone Store Terms of Use. Now the child can download apps from the Windows Phone Store.

  6. To enable the child to download Xbox games as well, go to Xbox.com, sign in with the child's account, and follow the instructions to accept the Xbox Terms of Use.

Tip

You can also add another parent to My Family so both parents can jointly manage kids' settings. Go to the My Family home page and click Add a parent. Parents won't be able to change each other's settings.

To change app download settings for a child

You can set whether or not a child can download apps and games from the Windows Phone Store and which game ratings you want the child to be able to view and download.

  1. On your computer, go to windowsphone.com.

  2. In the drop-down menu under Explore, click My Family.

  3. Sign in using the

    Note

    If you're already signed in to windowsphone.com, you'll see your name, not Explore, at the top of the drop-down menu, and this sign-in step will be skipped.

  4. On the My Family home page, next to the child's name, click Change settings, and then do any of the following:

    • To allow the child to purchase apps and games, under App+game downloads, select Allow free and paid.
    • To allow the child to download only free apps and games, under App+game downloads, select Allow free only. This will also prevent the child from making in-app purchases, even from within free apps.
    • To prevent the child from downloading any apps or games, under App+game downloads, select Don't allow.
    • To set the most mature game rating you want the child to be able to browse or download, turn on the Game rating filter. To go to the Microsoft Family Safety website and change the rating, click Change rating.
    • To allow the child to browse and download unrated games, select the check box next to Allow unrated games. (Some unrated games might have content you'd rather the child not see.)
    Note

    When the Game rating filter is turned off, the game rating settings are hidden. Turn it on to reveal them.
