IE9 BUG: Double URL Encoding in Form Element Name Attribute

There is a bug in IE 9's url-encoding of form elements. If a form element has a double quote in the name attribute, IE 9 encodes it twice.

 

For example,

    <input type="hidden" name="hello&quot;world" value="anything" />

All other browsers, including versions of IE up to IE 8 encode this as:

    hello%22world=anything

However, IE9 encodes it as:

    hello%2522world=anything

where "%2522" == urlencode("%22").

 

You can repro it for yourself here: https://defuse.ca/iebug.php (view source to see exactly what the form submits).

 

This is a serious compatibility problem because if one wants (or needs) to use double quote characters within the name attribute of a form elemement, they have to special case IE 9 based on the user-agent string in server-side code. What is the best course of action to get this fixed?

midlevel say:

Hanoze:

I know that it has been a while since you posted, but I wanted to put this out there for the people that are searching today.  If you wish to use .dot add-ins you have two options in 2010 you can turn them on in a single document or you can place them in the \startup\ location and they will be there every time you open MS Word.  In 2010 go to Options, Trust Center, Trusted Center Settings, Trusted Locations - look at the Word 2010 default location: Startup.  If you copy your .dot to that location your templates will be shown at the top of MS Word when you start MS Word each time.  If you want to just add them for a single event for a single document then you just need to turn on Developer in the Ribbon and then click on Add-ins and then select the add-ins you wish to use and click OK.  They will only stay there while you are in that document.  If you navigate away you will need to do the same steps to get them to be visable again.

Hello Phuong,

I realize the inconvenience you are experiencing with installing a game on your system. I will certainly help you.


To assist you better, please answer the below:
1. Is this issue specific to any particular game?
2. What is the make and model of your computer?
3. Which version of operating system is running on your computer?

    (To check the version, Press Windows + R keys to open Run command, type Winver and press Enter).
4. Is there any pending windows updates in your computer?

    If yes, I would suggest you to install all the windows updates by doing the below steps:
      a. On the search bar type Settings.
      b. Click on Update and Security.
      c. Click on Check for Updates. 
      d. Read and accept the license terms, and then tap or click Finish if the update requires it.


This issue might be with the security permissions on your profile temp folder. Follow the steps below to fix it: 
 
1. Navigate to %temp% or C:\Users\[Username]\AppData\Local and right click on the Temp folder
2. Choose properties, then click the security tab and click advanced.
3. On the permissions tab you should see the permissions that are there. There should be three which are:
      a. 'SYSTEM' with Full control which applies to 'This folder, subfolders and files'
      b. 'Administrators' with Full control which applies to 'This folder, subfolders and files'
      c. 'Your Username' with Full control which applies to 'This folder, subfolders and files'
All the three should be inherited from the C:\Users\[Username]\ folder.


If you don't have the Include inheritable permissions from this object's parent option ticked, then tick it and click continue if there are any problems, then remove the permissions that aren't inherited. Once you click 'Apply' and click 'OK' you should have the permissions to write to the directory.

Now try to run the setup.exe file and check if you can install the game.
