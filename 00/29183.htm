What is the "Date" column in Windows 7 Explorer? - Microsoft Community

astromere say:

What is the "Date" column in Windows 7 Explorer?

I am curious what data the "Date" column displays in Windows 7 Explorer.  Most of the headings are pretty clear, "Date Modified", "Date Created", etc.  But the "Date" column seems to almost randomly select what date value it shows.  I have a folder where half the "Date" values are the modified date and the other half are the created date.

the larch say:

I tried to figure it out a while ago and made notes.  In brief, it is still mysterious to me.  From my notes:

 

The date property is not an independent property, but is instead a copy of any of datelastsaved, datecreated, or some other date-related property, usually (but not always) depending on the file type.  Such variability in its meaning makes the date property risky to use if the underlying meaning is not known.  For most file types, the property matches datelastsaved (which is usually the same as datemodified).  On a Window 7 computer that I looked at, such file types included, but were not limited to:  adml, admx, ani, ax, avi, bat, bcm, bin, bmp, cat, cfg, chm, config, cpl, css, cur, db, dll, doc, dxt, etl, exe, exp, fon, gdl, gif, h1s, hlp, htm, html, icc, icm, ico, ini, inf, jar, gpd, lng, lue, man, manifest, mfl, mid, mof, mp4, mpg, mui, msc, mst, mum, nlp, nls, png, ppd, pdf, pps, ps1, psd1, reg, resx, rtf, sql, sys, tbl, theme,  tlb, ttc, ttf, txt, vbs, vdf, wav, wer, wma, wmv, wpl, xen, xrm-ms, xml, xsd, and xsl.  File types whose date property matched datecreated included, but were not limited to:  folders, cab, cdf-ms, cpl, crwl, evtx, idx, log, ocx, pf, pnf, properties, rar, regtrans-ms, sol, swf, and tmp.

I have found examples of files that had the date property matching datecreated, while others of the same file type had the date property matching datelastsaved instead.  These file types were:  shortcuts, internet shortcuts, extensionless files, dat, drv, js, mp3, and msi.  This peculiar variability is another reason to avoid relying on the date property.  (It is possible that the date property matched some other kind of date property that I did not notice, and only coincidentally matched datecreated or datelastsaved.) 

Image files made by a camera can have a datetaken property in their metadata, and the date property will match that property.  In absence of the datetaken property, datelastsaved (datemodified) is matched instead.

Some files may have their date property set to match any of a set of available date-related properties depending on the use of the file.  For example, email eml files’ date property can match datereceived or datesent.

I found some extensionless files in a Java cache with an empty date, matching an empty datelastsaved.  However, when I had the results resorted by date, mysteriously the date property was never empty.  Other files with an empty date property appeared to be fonts, Internet Explorer cache files, and cookies.

HiAmbermoon1,

 

The error message generally points to damaged file, folder or file system index (Master File Table [MFT] or File Allocation Table [FAT]).

To resolve this issue, run the Chkdsk utility.

 

WARNING: When Chkdsk detects problems with the file system and attempts to repair the damage, data loss can occur. Microsoft does not recommend running Chkdsk with the /f command-line switch unless you have a full backup of your hard disk.

 

Attempt to repair the damaged file, folder or file system index using Chkdsk. To do so, follow these steps:

 

1.    Click Start, and then click cmd.

2.    Right-Click on cmd and select “Run as administrator”.

3.    In the  command prompt window, type “chkdsk /f <drive>:” without the quotes, where <drive> is the letter of the drive on which the damaged file, folder or file system index exists.

4.    Click OK.

 

NOTE: If the hard disk is large, Chkdsk may take several minutes to several hours to complete.
