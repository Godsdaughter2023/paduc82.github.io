tryx3 say:

You could, instead, login to the desired account before you send the PC to sleep.  The Tasks you have created need to be checked to ensure they run correctly in the account you choose.

 

When you Sleep or Hibernate the PC, you remain in the same account that your were in immediately beforehand.  So no login would be required unless you had set the system to demand one [which can be unset* unless you are part of a network domain that retains that right for the domain admin only].

- - - - - - - - - - -

* Setting the need for login on wakeup

1 Check the screensaver’s login setting - Control panel, Personalise, Screensaver [in the bottom right-hand corner of the window].  You can alter the idle time for the screensaver to start and also clear the checkbox so you do not need to login again after it activates.

2  Clear this checkbox -  Control panel, Power options, [on the left-hand side of the window] Require a password on wakeup

3  Clear this checkbox - Control panel, Power options, {active power plan} Change plan settings, Change advanced plan settings, [you might need to click on Show settings currently hidden] {active power plan}, Require a password on wakeup

 

These multiple steps are just checking that any alterations have been set & saved consistently - there should be no problem under normal circumstances but some users have reported that setting this in all three locations was necessary for them.
