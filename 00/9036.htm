You can create a Color Theme that contains your color. Please follow these steps:

If you want to avoid using a Color Theme, you'll have to keep your color values in NotePad or other program, then manually enter the values when you need them.

Embedding font balloons file size on Windows 10 - Microsoft Community

drpowell say:

Embedding font balloons file size on Windows 10

I recently made an extremely simple PowerPoint with a single embedded font file. Without embedding the font the file's size is 63KB. The font is Truetype and is only a 26.8KB file. Embedding the font using PowerPoint 2013 on Windows 7 increases the file size to 2.2MB which is entirely acceptable. However, re-saving the same file with the font embedded in both PowerPoint 2013 and 2016 (Office 365) on Windows 10 increases the file size to 12MB on multiple computers. This is a file meant for sharing and downloading, so keeping the file size to a minimum and fonts embedded are essential.

Is this a known problem for others, or is this specific to me? Is there a way to remedy this problem?

Thanks in advance.

steve rindsberg say:

Off top of head, I can think of two possible reasons for this:

Suppose the font file in Win7 is Unicode but doesn't contain a huge variety of "extra" characters.  It'd be relatively petite.

If the same file in Win10 contains a larger collection of characters, it would be larger, so when the file is saved and the font (the current version in Win10) is embedded, boom ... fatfile.

OR

If you saved with fonts embedded from Win7 but opted to embed only the used characters, but somehow the options changed in Win10 to embed all of the characters, that could well account for the size increase, at least for fonts with thousands of characters.

To check on the first situation, in the Replace Fonts dialog box, look to see what fonts are listed, then look at the size of each font file (Control Panel | Fonts).  You might need to include the sizes of the font variants (Bold, Ital, BoldItal) for each font as well; not sure whether PPT will embed them on principle or not.

Then do the the same for the same fonts in Win10.  If the totals are more or less the same, we can rule out the first possibility.

To check on the second, open the file in Win10, do File | Save As, give it a new name and in the save options, choose to embed only used characters.

Do the same under Win7, then compare the resulting file sizes.

daniel g. grau say:
Word is trying to save a file into a folder, and can't. It's wrongfully assuming the reason is a full disk. Then it smacks you again with a second message telling you that it couldn't save the AutoRecover file.

See this post for a solution: Unable to save AutoRecover file.

If you want a backup, enable Word > Preferences > Save > Always create backup copy only or that document*. It's a lot safer than AutoRecover, in the sense that you get a fully-functioning version of the file every time you save (mind you: if you don't save, you'll have no backup). The backup copy is one version behind, which ensures that if the current version screws up, the file named "Backup of..." will contain the previous version, minus the steps that led to the screwup.

AutoRecover is not a backup, and neither is it an AutoSave function. Thus, it does not work if your file is closed normally, because Word will eliminate all its autorecovery files, since they are no longer needed. It only serves its purpose if Word freezes, which you never want to happen. In fact, it this occurs, the AutoRecover file may be in worse condition than the last-saved file.



* I say "only for that document" because if you leave it on for all files, pretty soon you'll find your disk littered with "Backup of..." files all over the place.
