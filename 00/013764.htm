krobichaux say:

I was able to "fix" this problem, against the advice of others in an Office365 forum.

  1. Closed Outlook 2016.
  2. Right-clicked on Outlook 2016 shortcut in Start; selected Properties.
  3. On Compatibility tab, clicked on Run compatibility troubleshooter, which yielded NO fix to the problem. But...
  4. Followed prompts in troubleshooter app to undo all changes and NOT to report issue to MS ("No. I am investigating...").
  5. Opened Outlook 2016, and emails in reading pane and as opened messages displayed properly.
  6. Rebooted Windows 10, and opened Outlook 2016, and emails still displayed properly in both environments.

I hope this will work for others. I figure my system had some wonky settings somewhere, and the run through the troubleshooter, though apparently undoing its changes, fixed what was wrong.

It can't hurt to try, I guess.

Kerry

scottb35e0 say:

Here's something that worked for me:

I have OneDrive and OneDrive for Business installed.  It seemed to me that they are conflicting so I started digging in the settings.  My hypothesis is that OneDrive is using the same document cache as OneDrive for Business and that is causing the errors we're seeing.  When I disabled that "feature," my Document Cache stopped getting corrupted. 

Here's some further reading on the feature you want to DISABLE:  http://windows.microsoft.com/en-us/onedrive/office-2013

To disable this in Windows 8, you need to install the desktop version of OneDrive (as referenced in the link above).
