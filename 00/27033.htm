proposing GetIf() a VlookUp - Index - Match - HLookup - Microsoft Community

apostolos55 say:

proposing GetIf() a VlookUp - Index - Match - HLookup alternative

syntax: GetIf ( Range1 , criteria , [range2] , [NoOfResults])

being tired of using V/Hlookup, Match, Index etc plus some things extremely hard to do (many results of same occurrence) , I have written GetIf()

feel free to use it, comment it etc, just keep me informed of any suggested changes-fixes

it is self explained and if I can post an archive somewhere I could demonstrate some usages.

For now:

' START OF CODE - GETIF()
'
' by Apostolos55
'

Public Function GetIf(A, B As String, Optional C, Optional ArraySize As Long = 1, Optional FillStr = 0)

Dim Counter As Long, Counter2 As Long, Counter3 As Long, dum1, matchFL As Boolean

Dim Counters() As Long, Holder()

 

' Fix ArraySize

If ArraySize = 0 Then ArraySize = Evaluate(Application.WorksheetFunction.CountIf(A, B))

'If ArraySize = 0 Then                                 ' Fix ArraySize

'     Counter = 0: For Each dum1 In A

'     If B = Trim(dum1) Then Counter = Counter + 1

'     Next: ArraySize = Counter: End If

 

If ArraySize < 1 Then ArraySize = 1     ' Restore to 1 even if evaluate finds 0 to avoid errors

 

' properly allocate arrays for VERTICAL results. Use TRANSPOSE() in excel fo HORIZONTAL

ReDim Counters(1 To ArraySize, 1 To 1), Holder(1 To ArraySize, 1 To 1)

 

Counter = 0: Counter2 = 0: Counter3 = 0

For Each dum1 In A                  ' Search for index

    Counter = Counter + 1

    matchFL = False: If B = Trim(dum1) Then matchFL = True

    If matchFL Then

        Counter3 = Counter3 + 1

        Counters(Counter3, 1) = Counter

        If Counter3 = ArraySize Then Exit For   'Search only as many as asked for

    End If

Next

 

If Counter3 > 0 Then                        ' if something was found

    If IsMissing(C) Then                    ' Edit:Jun2012 Ommitting range2 (c) returns Position within range

        GetIf = Counters

    Else

        Counter = 1

        For Each dum1 In C                  ' Use index to find equivalent

            Counter2 = Counter2 + 1

            If Counter2 = Counters(Counter, 1) Then

                Holder(Counter, 1) = dum1

                Counter = Counter + 1

                If Counter > Counter3 Then Exit For 'Search only as many as asked for

            End If

        Next

        For Counter = Counter3 + 1 To ArraySize

            Holder(Counter, 1) = FillStr

        Next

        GetIf = Holder

    End If

Else

    For Counter = 1 To ArraySize

        Holder(Counter, 1) = FillStr

    Next

    GetIf = Holder

End If

 

End Function


' END OF CODE

jim thomlinson say:

#Ref errors occur when a formula is refering to cells that no longer exist (or never existed). So if you deleted columns or cells in the lookup range then you will possibly get a #Ref error. #ref has nothing to do with formatting or such and it is not a matter of not finding a match. It is a reference to cells that do not exist.

Vlookup is a bit unique in that it can be a matter of the offset that causes the problem. For instance if my lookup range is 3 columns wide and I am looking at the 3rd column over then if I delete one of the columns the formula still referes to the thrid column despite the fact that there are not 3 columns in the data range.

If this post answers your question, please mark it as the Answer... Jim Thomlinson

j2kenned say: One possible issue is that the array you are using refers to the wrong column. Say you have a 4-column table, and you write your formula as 

=VLOOKUP(B3,'TABLE NAME'!B1:D2000,4,FALSE), intending to use column D as the result vector. This won't work because Excel will return #REF. The columns are relative, and even though column D is 4 over, if you only select  3-column range, the error will appear. The ways to correct this are to write 
=VLOOKUP(B3,'Esp-Eng'!A1:D2000,4,FALSE) 
or
=VLOOKUP(B3,'Esp-Eng'!B1:D2000,3,FALSE)
