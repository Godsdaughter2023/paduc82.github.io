It appears that once an alarm is set, it cannot be cancelled (unless it is turned off within a certain amount of time, maybe before midnight, or within 24 hours...I'm not sure.) The problem apparently is that turning off the alarm doesn't clear the alarm from the phone's notifications queue once the queue does its daily refresh for recurring items (which usually happens at midnight). The alarm feature is like a one-way valve. It can create an alarm in the notifications queue, but not cancel one. It can push but not pull. It's like a car without a reverse gear.

 

My wife has a WM6.0 Pro phone (a Wing) and doesn't seem to have these issues. From what you've reported, I may advise her not to upgrade to WM6.1 Pro.

I use SK Tools Lite (a free program for Smartphone) to erase the alarms from the phone's notifications queue. It works and is easy to use, but it's a bit annoying to have to do that every time you want to clear an alarm. At least it's better than doing a hard reset.

 

But now, instead of using the WM alarm, I use the free Snooze alarm program, which is a lot better except that it has the opposite problem...i.e., sometimes it doesn't go off at all!

 

There is a version of SK Tools for WM Pro, but it's not free. It probably has a feature to clear the alarms. Maybe you can use it during the free trial period to clear your alarms and avoid a hard reset. But for the long term you may want to look into installing another alarm program.

My ASUS laptop with build 10130 was having same problem. I used cmd that I got some good results from before. It resets your apps. After I ran cmd and rebooted I went into store (beta) and that update was gone. My Music app has version 3.6.1068.0

1. Press Ctrl+Alt+Del. Select Task Manager.
2. From the File menu, select Run new task.
3. Type powershell, put a check mark next to Create this task with administrative privileges and tap or click OK.
Paste or type the following at the Powershell prompt, and hit Enter:
Get-appxpackage -all *shellexperience* -packagetype bundle |% {add-appxpackage -register -disabledevelopmentmode ($_.installlocation + “\appxmetadata\appxbundlemanifest.xml”)}

Don't worry about any error messages. Reboot pc when it finished.

I had problems with an update to music that didn't complete in the store.   Fix, sort of, is on this page.

I did the powershell command

Get-AppxPackage  > C:\AppxPack.txt

Then the text file you just made will have a list of your apps... search Music because you need this;;;

PackageFullName   : Microsoft.ZuneMusic_3.6.10681.0_x64__8wekyb3d8bbwe

Remove-Appxpackage –Package Microsoft.ZuneMusic_3.6.10681.0_x64__8wekyb3d8bbwe

Be a good idea to restart the computer before going to the beta store and searching music to re-install it.

In general you don't. If you didn't save it it is gone.

Exception, if you have the "Autorecover" feature turned on AND word crashes or is not shut down normally (ie Windows is shutdown), then there should be an "autorecovery" file that loads the next time you open Word.

In Office 365 / Word 365, if you save documents to OneDrive there is an "autosave" feature. I don't known what they do with unnamed/saved files.
