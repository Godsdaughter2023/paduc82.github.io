If you can add another field to your Access data to return the character count of the Name field then you can do this in the merge itself by referencing the count. In the following example you can set three different font sizes based on the number of character is the field 'Length'


{ IF{ Mergefield Length } > 10 "{ IF { Mergefield Length } > "20" "{ Mergefield Name }" "{ Mergefield Name }"}" "{ Mergefield Name }" }

jllatham say:

A generic type of situation -

Just before executing a statement, or series of statements that may result in an error you would like to ignore:

 

On Error Resume Next ' will basically ignore any errors generated until reset

'code that may generate error you want to ignore in here

'after all possible error generating code has completed

If Err <> 0 Then
   Err.Clear ' just to keep a clean house
End If
On Error GoTo 0 ' resets error trapping back to system default

 

Another way is by using On Error GoTo with a label, kind of like this, which is actual code you can test with to get a better feel of things:

Sub ErrorHandlingExample()

  On Error GoTo MyERRHandler ' will be near bottom of the Sub/Function
'code here that may generate error(s)
  Error (13)
'at a point beyond the possible error generating code is another label
  MsgBox "Error wasn't generated"
ContinueAfterMyError:
  MsgBox "into ContinueAfterMyError area"
  'code to be executed regardless of whether errors took place or not here
  Exit Sub ' to keep from falling through to the error handler

MyERRHandler:
  MsgBox "Entered the error handler"
  Err.Clear
  'you could put up a message box here if you want, or not
  ' then you tell it where to go after having dealt with the error
  Resume ContinueAfterMyError
End Sub

'***********

One other way to deal with them, rather useful if there is a particular statement that you expect could generate an error and you'd like to deal with right then and there:

On Error Resume Next
'a statement that might generate an error (file not found or similiar in this example)
Workbooks.Open("C:\Users\Me\ExcelStuff\Dummy.xlsx")
If Err <> 0 Then
  MsgBox "An error took place:" & vbCRLF & _
   "Error Number: " & Err.Number & vbCRLF & _
   "Description: " & Err.Description, vbOKOnly+vbCritical,"OOPS!"
  Err.Clear ' clear the error
  On Error GoTo 0 ' let system handle any more errors
  GoTo SomePlaceElseMaybe
End If
'falls through to here if no error took place, so
On Error GoTo 0 ' we want to let system handle other errors now
'more code...
