Intel's driver is outdated could be a hardware or software issue. A game’s performance is associated with the graphics card (hardware) and its driver (software).

 

The Switchable Graphics feature allows you to switch between using the Intel graphics and a discrete graphics controller in a computer. For example, you can switch between the enhanced battery life of the Intel graphics while running on laptop battery power and the performance capabilities of the discrete graphics controller while the laptop is plugged into AC power.

 

Switchable graphics refers to mobile computers equipped with both a low-power imbedded graphics chip, and a high-power discreet graphics card, with the ability to switch between them. The imbedded chip does a fine job with everyday applications and video while reserving power, allowing longer battery-driven work periods. The discreet graphics adapter is capable of robust 3D rendering required for gaming, sacrificing power for performance.

 

A common imbedded graphics chip in many mobile computers is the Intel® Graphics Media Accelerator™ (GMA), also referred to as an integrated graphics processor (IGP). The chip is ideal for everyday use and will save juice when 3D rendering is not required.

 

A complimentary, discreet graphics card, often a Raedon™ or AMD® flavor, makes use of an internal PCI bus and comes with its own on-board memory, drivers and software interface. Such a card is referred to as a graphics processing unit (GPU) and does not overburden the computer processing unit (CPU) with tasks or overhead, or steal system resources like an integrated graphics chip. Instead it carries processing power and RAM on its back, delivering outstanding performance, but requiring more power.

 

You may update these drivers independently by downloading the drivers from their respective manufacturer’s website. Alternatively you may also update the drivers through Device Manager and Windows Updates.

 

For the game being slow, you may put the computer in a clean boot state and check if this helps.

 

When you start Microsoft Windows, typically there are several programs that start automatically and run in the background that may interfere with the game. These programs may include antivirus and system utility programs. When you perform a clean startup procedure, you prevent these programs from starting automatically.

Laptop wakes itself while inside carrying case - Microsoft Community

chameleon89 say:

Laptop wakes itself while inside carrying case

This is an infrequent but severe problem: my Windows 8 laptop will occasionally wake itself from sleep in its carrying case. The few times this has happened have been at night - I discover it in the morning, dangerously hot inside its unventilated bag, with the battery near-drained. Searching the internet reveals that this is actually a fairly common problem.

I believe I have found all the possible causes of the problem and disabled them.

1) Command prompt -> powercfg /devicequery wake_armed (get list of devices able to wake the computer)
2) Device manager -> network adaptors -> [e.g. Realtek PCIe GBE Family Controller] -> power management -> uncheck "allow this device to wake the computer"
3) Task scheduler -> task scheduler library -> click each task and check under "conditions" that "wake the computer to run this task" is unchecked
4) BIOS -> power management -> disable anything that wakes the computer without user input

For now the problem has not reoccurred.

However... it really should not be ABLE to occur in the first place. It strikes me as odd that Windows permits any programs/devices to wake the computer without some kind of "master check". Surely the operating system itself should not allow a wake under circumstances when the computer is potentially in a carrying case (or in another unknown situation).

It needn't be complicated:

*Power on master check*
if {lid = closed} and {power = battery}
then {NEVER TURN ON}

Perhaps have a checkbox on the power management screen, just in case some users want to disable this safety feature:

"Never wake my computer when the lid is closed and it is not plugged in."

(But the default should ALWAYS be "checked" for a laptop.)

In summary - a laptop, by default, should NEVER be allowed to turn on if its lid is closed and it is not plugged in.
- Waking a laptop while it is in a carrying case may cause physical damage from overheating (insulation and no ventilation).
- Waking a laptop while it is in transit may cause physical damage from shakes and jolts (e.g. if the unsuspecting owner is running or going up stairs).
There is NO program/update/check that is EVER worth waking up for if it risks physical damage to the computer. Surely this much is obvious - safety trumps everything else.

I hope the simple suggestion of a "master check" for powering on is taken seriously and can be passed on to someone with the power to implement it.

Kind regards,
John
