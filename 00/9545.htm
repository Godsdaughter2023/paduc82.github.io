I need a formula for each destination cells that will skip the used and empty cells and show the NEXT text.  If only A1 and A8 have text; the second cell of the form will have the text from A8 (after having found no text in A2-A7).  I'm thinking IF HLookup.  Any suggestions would be appreciated!

jeeped say:
You will have to increment the k (aka postion) value of the SMALL() function but this should get you what you want.

=IFERROR(INDEX($A$1:$A$8,SMALL(INDEX(ROW($1:$8)+(ISBLANK($A$1:$A$8)*1E+99),,),1)),"")

Note the bold 1. That will get you the first non-blank value. Manually edit and increment to 2, 3, etc for subsequent non-blank values.

INDEX function Uses an index to choose a value from a reference or array
SMALL function Returns the k-th smallest value in a data set
ROW function Returns the row number of a reference
ISBLANK function Returns TRUE if the value is blank
IFERROR function Returns a value you specify if a formula evaluates to an error; otherwise, returns the result of the formula

Note that ISBLANK() treats cells with a zero length string as FALSE. If you have formulas in these cells that may return a zero length string (e.g. "") then you should be checking for LEN() and not ISBLANK().

=ROUND(A2 * 1440, 0)

Is there a way I can use a radio button to filter the results of a combo box?  I have a combo box that lists various cities in Canada.  I would like to filter the combo box by use of a radio button - I have created a radio button for each province and I would like the combo box to filter based on the radio button selected.

doug robbins - mvp office apps & services (word) say:

I would suggest that instead of a whole series of radio buttons, it would be better to have a second combobox that is loaded with the names of the cities as it is then very easy to get the name of the city on which you want to filter (remove) the contents of the combobox that you now have.

If the combobox that contains the cities is cmbCities and the one from which you want to remove the items is cmbAddresses, you would use

Dim i As Long

Private Sub cmbCities_Exit(ByVal Cancel As MSForms.ReturnBoolean)

With cmbAddresses
     'include code to re-populate cmbAddresses with all of the records
    For i = .ListCount - 1 To 0 Step -1
        If .List(i, n) <> cmbCities.Value Then
            .RemoveItem (i)
        End If
    Next i
End With
End Sub

where n is the column index that contains the Cities in cmbAddresses

Note, where indicated by the comment, you should include code to re-populate the cmbAddresses combobox with all of the data, so that the user can change the city that they have selected.

dirk goldgar say:

Your radio buttons will be part of an option group, and each button will have a distinct OptionValue property, as specified on the Data tab of its property sheet.  Whichever button is selected, its OptionValue will be assigned as the value of the option group control, so it's the option group's value that you will use to filter your combo box.

If your provinces have been assigned numeric IDs as their primary keys, then you can set each radio button's OptionValue to the ID of the province it represents, and the combo box's RowSource query can refer to the option group's value directly as a criterion.  Then all you'd have to do is requery the combo box in the form's Current event and the option group's AfterUpdate event.

However, if your provinces don't have numeric primary keys, you can do it quite that way.  One way to do it would be to rebuilding the combo box's RowSource query every time the option group is changed.  You could create a Function in the form's module that looks at the combo box's value and sets the RowSource, similar to this:

    Private Function FilterCombo()

        Dim strProvince As String

        Select Case Me.grpProvince

            Case 1 : strProvince = 'ON'

            Case 2 : strProvince = 'QC'

            Case 3 : strProvince = 'NS'

            ' ... and so on for all provinces ...

        End Select

        If Len(strProvince) > 0 Then

            Me.cboMyCombo.RowSource = _

                "SELECT City FROM MyTable WHERE Province = '" & strProvince & "'"

        Else

            Me.cboMyCombo.RowSource = _

                "SELECT City FROM MyTable"

    End Function

You would need to call this function in the form's Current event and in the option group's AfterUpdate event.  Because it's defined as a Function, not a Sub, you can call it directly from the form's OnCurrent event property and the option group's AfterUpdate event property, setting each of those properties to this function expression:

    =FilterCombo()

And that ought to do it.

stephen boots say:
This error indicates a connection problem.
There can be many causes of this.
First, does Windows Update work?
In Internet Explorer, go to Tools/Internet Options/Connection and uncheck use proxy and automatically detect.
In the Start/Search box type Services.msc to open Services
In the list of services, stop and then start the Background Intelligent Transfer Service and Windows Update.
The Windows Update service should be startup type of Automatic (Delayed start)

Note that MSE cannot update via a proxy server. MSE cannot update via WSUS.
