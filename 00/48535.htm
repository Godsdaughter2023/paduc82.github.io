  1. Create the folder C:\Test.
  2. Put some text file inside, e.g. Test.txt.
  3. Open the folder's security tab.
  4. Click the Advanced button.
  5. Click Change Permissions.
  6. Untick the inheritance box.
  7. Click the Remove button when prompted.
  8. Click the Add button.
  9. Type System and click Check Names.
  10. Click OK.
  11. Tick "Full Control" in the Allow column.
  12. Click OK a few times.
  13. Click Start.
  14. Type the three letters cmd and press Enter.
  15. Type this command:
    cd  /  c:\Test
    You get a message "Access denied". Why? Because only the System account has access to this folder.
  16. Try to open the folder c:\Test in Windows Explorer. You will get a challenge. Why? Because Windows Explorer "knows" that you are an administrator and that you can override security restrictions.
  17. Log on under a non-admin account, then try to open c:\Test in Windows Explorer. You can't. Why? I think the answer is obvious.

If you now play a little with this folder then you will soon realise how it all works.

Updating: Run or Save? - Microsoft Community

daphnegoetz say:

Updating: Run or Save?

When installing software or updating, it often asks first whether we want to Run or Save.

What happens when you choose Run? What happens when you choose Save?

What am I choosing from?

gil deg say:

Hi Daphne,

The difference between Save and Run when installing a software or updating, is that follows:

When you select Run on a download, it begins a two-step process:

  • The file you’ve requested is downloaded to your machine and placed in a temporary location.
  • The file is then run exactly as if you had used Windows Explorer to navigate to the temporary location and double-clicked the file

The temporary location is either your browser’s cache or temporary folder, or one of your Windows temporary folders. In fact, it really only needs to be preserved long enough for the “Run” that happens next, and could be deleted at any time thereafter.

When you select Save (sometimes offered as Save As…) in response to a download, the browser simply downloads the file.

The file is copied to your computer, and it’s then up to you to do something with it.

The big difference is that it’s not downloaded into a temporary location. The file is downloaded into either:

  • A default folder, such as “My Documents” or “Downloads” (often the default when you select Save).
  • A folder that you specify in response to a “Save As…” dialog (the action when you specify Save As…).

Let us know if you need further assistance.
