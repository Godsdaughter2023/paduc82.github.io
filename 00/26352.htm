You can't change a setting to do this, unfortunately.  You can use Ctrl+Tab
to tab out of the subform to the next control on the main form, or you can
write code to allow the tabbing out of the last control on the subform.  For
example, you could trap the KeyDown event of the last control on the

control on the main form.
 
There are other ways to do this, too, but you do have to write a little code
to accomplish it.

Project 2010 Printing - Microsoft Community

markmware say:

Project 2010 Printing

We cannot consistently get Project 2010 to correctly print our gantt chart. Onscreen our project looks great, but trying all the options we know of still results in incorrect sizing or illegibility of parts of the chart. Where can I go to learn more about printing our work.

Thanks.

trevor rabey say:

It is a combination of how you set it up in the working screen and how you set it up in page setup.

In the working screen you have to set the timescale, and don't overlap columns with the chart if you want them printed. Font sizes and bar styles are set in the view.

You may need to custom design a particular view, not for working in but just for printing.

Each print has to be designed for the audience and for the size sheet of paper it is being printed on.

In page setup, the view tab is useful, to turn off "print blank pages" and "print first x columns".

Set margins to a minimum, say 1 cm. Don't try to scale 50 sheets onto one sheet. Choose portrait or landscape. Set your pdf writer as default printer, and print preview everything as PDF before sending it to a real printer. In page setup, start with a large format sheet of paper, metric A0, and work down to smaller sheets until it won't fit. Get a large format printer, because most printouts look terrible just because they are on A4.

julie sheets say:

Ensure that Project 2010 is updated with SP-2.  You have already discovered that what you see is NOT what you get when you print - for example, you may have columns to narrow to view the file correctly - but it prints fine.  You will want to set up custom views for printing that are not used for viewing.

See:

http://msdn.microsoft.com/en-us/library/ms244369(v=vs.90).aspx

for information on creating views.

Using scaling (fit to) often over compresses the view too much.  Zoom out of the Gantt chart (View ribbon, Zoom entire project) and then check print preview.

gloria gong msft say:

Hi Jennie,

I would like to confirm if the following two situations can meet your need:

Green: Conditional Formatting> New Rule...> Use a formula to determine which cells to format> Type in "=$C1>0"> Format...> Fill tab> Choose color> OK.
