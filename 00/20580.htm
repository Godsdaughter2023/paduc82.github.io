karl timmermans [outlook mvp 2012-2018] say:

There are 2 approaches to this (short of using a 3rd party conversion program) either export the existing PST file to a new file or make a copy of the existing PST file (with Outlook closed) and import that into a new PST file. Different steps involved but both will ultimately bring you to the same place.

 

Using the Export approach

 

From O'2010 menu

 

File --> Open --> Import
OR

File --> Options --> Advanced --> Export

(either of the above will bring you to the same Outlook import/export process)

 

Once you are in the import/export wizard - choices to make on the relevant screens

1) - Export to a file

2) - Outlook data file

 

*** this is the point where you have to play very close attention to what you choose

 

3) - For the "Select folder to export from" - be SURE to select the ROOT folder - the topmost folder where your Inbox / Contacts etc folders reside

4) - be SURE to check <Include SubFolders>

 

5) - click next - select a location for the exported PST file and give it a name - (remember both)

6) - choose your option for "duplicates" (for an <exact> match - allow duplicates to be created)

7) - Finish

 

When done (could take a bit depending on the size of your PST file being exported)

8) - close Outlook

9) - from Control Panel --> User Accounts & Family Safety --> start Mail applet

10) - click Data Files tab --> add the newly exported PST file --> make it the default

11) - click Email Tab

12) - for each email account that had its mail delivered to the old PST file - change the location to point to the same Inbox folder in the newly added PST file

13) - click on Data Files tab again --> remove old PST file from the profile
       (if you have defined custom contact fields - see note below regarding contact UDFs
        and complete those steps before removing the PST file)

14) - close Mail applet

 

15) - Open Outlook and you should be good to go

 

*** About contact UDFs 

 

If you created any user-defined fields in your contact folder, these will not appear in the "User-defined fields in Folder" list in the new PST file (the UDFs associated with individual contact items will exist).

- the above is resolved by completing the following steps

     a) open a new contact in the <OLD> PST file
     b) go to <All Fields> --> <User-defined fields in Folder> list

     c) record every UDF created in terms of <UDF Name> and <Field Type>

     d) close new contact - no need to save it

     e) open a new contact in the <NEW> PST file

     f) go to <All Fields>--> default list displayed is <User-defined fields in this item>

     g) create the identical UDFs as recorded in step (c). After the first UDF is created, the list will

         automatically change to <User-defined fields in folder>, continue creating the rest of the UDFs

         under that list

      h) when done, close the new contact - no need to save it

Have a look HERE, this post is a tweak of a great implementation of  Levenshtein string comparing algorithm, that returns % of match.

'----------START OF CODE------------

' Levenshtein3 tweaked for UTLIMATE speed and CORRECT results
' Solution based on Longs
' Intermediate arrays make difference, even Fixed length Arrays have impact on speed
' Levenshtein version 3 is the only one who will return correct percentage...
'
Function Levenshtein3(ByVal string1 As String, ByVal string2 As String) As Long

Dim i As Long, j As Long, distance(0 To 60, 0 To 50) As Long
Dim string1_length As Long, string2_length As Long, smStr1(1 To 60) As Long, smStr2(1 To 50) As Long
Dim min1 As Long, min2 As Long, min3 As Long, minmin As Long, MaxL As Long

string1_length = Len(string1): string2_length = Len(string2)

distance(0, 0) = 0
For i = 1 To string1_length: distance(i, 0) = i: smStr1(i) = Asc(LCase(Mid$(string1, i, 1))): Next
For j = 1 To string2_length: distance(0, j) = j: smStr2(j) = Asc(LCase(Mid$(string2, j, 1))): Next
For i = 1 To string1_length
For j = 1 To string2_length
If smStr1(i) = smStr2(j) Then
distance(i, j) = distance(i - 1, j - 1)
Else
min1 = distance(i - 1, j) + 1
min2 = distance(i, j - 1) + 1
min3 = distance(i - 1, j - 1) + 1
If min2 < min1 Then
If min2 < min3 Then minmin = min2 Else minmin = min3
Else
If min1 < min3 Then minmin = min1 Else minmin = min3
End If
distance(i, j) = minmin
End If
Next
Next

' Levenshtein3 will properly return a percent match (100%=exact) based on similarities and Lengths etc...
MaxL = string1_length: If string2_length > MaxL Then MaxL = string2_length
Levenshtein3 = 100 - CLng((distance(string1_length, string2_length) * 100) / MaxL)
End Function


'-----------END OF CODE---------
