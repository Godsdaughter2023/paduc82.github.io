andreas killer say:

In Ms Excel, sorting a list alphabetically in English is OK but is this facility available if the list is in a language other than English(e. g. Nepali/ Hindi)?

Not if you have an English system, the sort order is specified by the language (resp. settings) of the OS.

You can create your own sort order, but that is tricky and you need some expierience in VBA and sort in general.

The basic idea is to create a map table (any sort routine does that!) and encode a string into numbers and sort by this numbers instead of the original strings...

A sample: If you sort "a,b,c,A,B,C" in ascending order you get "a,A,b,B,c,C"

The ASCII value of A is 65 and a is 97, so if Excel would sort by the ASCII value the result would be "A,B,C,a,b,c".

The question is how does Excel the trick? The answer is simple (to simplify for this sample):

An "a" becomes 1, a "A" becomes 2 and so on:

So Excel maps each char by this table and finally sort by this numbers.

But this tables are much larger in reality, an unicode char table has 65.535 chars.

Back to the sample: The word "Abc" becomes 235 and "aBc" becomes 145 and if you sort this words ascending you end up with "aBc, Abc".

That's it. No easy to accomplish, really tricky, but possible.
