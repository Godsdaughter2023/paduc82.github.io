bmcdanel say: Sorry, but things are not in order.  The problem self-corrects after six hours of run time.  After booting, the problem is there again with the clock is running again for the next six hours before the problem goes away.

bmcdanel say: Finally found the problem.  As my original message stated, device manager behaved like there was an interrupt conflict, but did not indicate a problem.  I opened device manager when the problem was not occurring.  This time it showed a usb add-on card not working.  I got it working with an uninstall from device manager and reboot.  This morning, the problem was back. This time I disabled the usb card instead of uninstalling in device manager.  The buzz and hesitation stopped immediately.  Now, the problem has been identified, a hardware or driver problem associated with this card that for some reason did not occur until the June update, still something of a Windows problem too since it did not occur until the June update or perhaps you call it a refresh.  So, I will proceed from here and hope I don't have to replace the card.    Sometimes, the usual troubleshooting doesn't pick up an unusual problem like this. bmcdanel say: Problem solved.  The June update changed a device setting on the USB add-on card's properties.  The setting is to allow Windows to turn off the device to save energy.  It was unchecked and changed to checked.  With this setting checked, the card is constantly turning on and off as it is intended to remain on at all times and Windows wants it off.   Unchecking this setting solved the problem.  MS should never change a user set device setting.  This is a Windows problem that should have never occurred.
