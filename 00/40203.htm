This utility makes it easy to see which versions are loaded :  Run DriverView -

on your system. For each driver in the list, additional useful information is
displayed: load address of the driver, description, version, product name,
company that created the driver, and more.
http://www.nirsoft.net/utils/driverview.html

For Drivers check System Maker as fallbacks and Device Maker's which are the
most current. Control Panel - Device Manager - Display Adapter - write down
the make and complete model of your video adapter - double click - Driver's
tab - write down the version info. Now click Update Driver (this may not do
anything as MS is far behind certifying drivers) - then Right Click - Uninstall -
REBOOT this will refresh the driver stack.

Repeat that for Network - Network Card (NIC), Wifi, Sound, Mouse and
Keyboard if 3rd party with their own software and drivers and any other
major device drivers you have.

Now go to System Maker's site (Dell, HP, Toshiba as examples) (as rollback)
and then Device Maker's site (Realtek, Intel, Nvidia, ATI as examples) and get

roccat user say:

I had an identical error occur while running Windows 7, it is an issue with the peripheral and not with any specific version of Windows.


The error occurs when the on-board memory of the peripheral fails in some way; it occurs seemingly at random on a very rare occasion, but like any computer memory it isn't 100% stable 100% of the time. Most users will never experience this issue. The cause of the error is currently unknown (will update this post in the future if there is any new information), but it's effects are, from personal experience, quite frustrating:


The error causes the peripheral's Monitoring program to fail to maintain a connection to the device. You may or may not find that even after completely uninstalling the Drivers, the profiles you had saved to the on-board memory are still accessible (personally I could still swap between my old saved profiles using the 'Shift+' key, for example), but you are "EventType=APPCRASH" error after the software boots.


Standard users are only given one single access point to resetting the memory, via software: //DRIVER RESET, which allows you to "reset to factory default". It can only be accessed using the very software that fails to maintain a connection to the device in question.


After booting windows and experiencing the error, close the offending software, and then re-open the Driver settings. (Now this is where it gets unprofessional and therefore currently unofficial - personal results may vary:) You will have a very brief window of connectivity and, if you're lucky/quick to the draw, can hit the "RESET" button...!


For most Roccat Mice, you must first click the "ADVANCED CONTROL" tab, then the "RESET" button, then the pop-up "OK". (confirmed fix for Kone Pure)


For most Roccat Keyboards, you must click the "RESET" button, then the pop-up "OK".


You have to accomplish this before the error occurs and the software can no longer connect to the peripheral. I was allowed less than a second to hit all 3 options for resetting my mouse. I don't remember also having to hit "OK" for the Driver software application to close everything; once I successfully reset the drivers my mouse reverted back to default, so closing and re-opening the software afterwards caused no further errors to occur.

As it stands, the official response is to send it back to whoever you purchased it from and get a new one. Mine has been stable since the fix, but if it happens again before the warranty expires I will probably just send for a replacement and update this post accordingly.


Also: As far as I am aware, you can make a back-up of profiles and macros and such before attempting the factory reset - feel free to do a separate search regarding that. It was a new mouse for me, so I had very few changes to save and I didn't bother.


I wish you all the best of luck; if this works for anyone reading this, please let others know.

1. Press Windows key + X, click Command Prompt (Admin).
2. In the Command Prompt, type the following command, sfc /scannow and then press Enter.
3. Wait until the process completes. This usually doesn’t take longer than 20 minutes with a regular hard disk and

takes much shorter when you have a fast SSD drive.

4. Check to see if the issue is resolved. If not, restart the system.

Step 3: Run the DISM Tool

Run the DISM (Deployment Image Servicing and Management) Tool to fix any files that System File Checker can't repair and fix Windows corruption errors. Here's how:

1. Right click on Start, click on Command Prompt (Admin).

Type Dism /Online /Cleanup-Image /ScanHealth and click Enter.
