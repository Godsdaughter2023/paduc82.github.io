scottgem say:

Ambiguous name in query expression 

The error message is clear. It means that you are referencing a field in a query that appears in more than one table referenced in the query without identifying which table to pull the field from. Thats why the Z was recommended. it indicates the table. The Z is an alias that you assign in your FROM clause (i.e. FROM table1 X, table2, Y, table3 Z). 

You do not need to use an alias however, you can simply fully qualify the field reference using tablename.fieldname. 

But Ken's point is valid, you update tables, not queries. It sounds like you just need to add your Concatenate function into the other query.
