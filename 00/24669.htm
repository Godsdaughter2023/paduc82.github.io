macropod say:

For the most part, Word formfields do not survive a mailmerge (see https://support.microsoft.com/en-us/kb/286841). A document using forms protection also can't be used as a mailmerge main document.

Content controls, however, do survive a mailmerge and, since documents using them don't require forms protection, documents containing them in an active form can be used as a mailmerge main document. So, if those completing the forms use Word 2007 & later, you might do better to use content controls instead of formfields.

If that doesn't meet your needs, see: http://www.gmayor.com/Form_Fields_and_Mail_Merge.htm

mike h.. say:

Hi,

Every time a cell changes the workbook will re-calculate so that's probably what's slowing things down.

Try it like this. Changes in bold.

Sub RemovePercentage()
Dim Rng As Range, c As Range
Dim factor As Double
Dim cell As Range
With Application
    .Calculation = xlCalculationManual
    .ScreenUpdating = False
End With
Set Rng = Range("E16:E200")
If Not IsNumeric(Range("E215")) Or Range("E215") = 0 Then
    MsgBox "No value entered in E215"
    Exit Sub
Else
    factor = 1 - Range("E215") / 100
End If
For Each c In Rng
    c.Value = c.Value * factor
Next

Set Rng = Range("G16:Y200")
If Not IsNumeric(Range("E215")) Or Range("E215") = 0 Then
    MsgBox "No value entered in E215"
    Exit Sub
Else
    factor = 1 - Range("E215") / 100
End If
For Each c In Rng
    c.Value = c.Value * factor
Next

    
    
    For Each cell In Range("E16:E200")
        If cell <= 0.001 Then cell.ClearContents
    Next cell
  
    
    For Each cell In Range("G16:Y200")
        If cell <= 0.001 Then cell.ClearContents
    Next cell
With Application
    .Calculation = xlCalculationAutomatic
    .ScreenUpdating = True
End With
End Sub
