Try this, 

Private Sub Worksheet_calculate()
    Application.EnableEvents = False
    Set Rng = Intersect(UsedRange, Range("J:J")) - specify the column in which you have the If statement
    If Rng Is Nothing Then GoTo ExitHere
    For Each i In Rng
        If i.Value = 1 Then
            Rows(i.Row).EntireRow.Hidden = True
        Else
            Rows(i.Row).EntireRow.Hidden = False
        End If
    Next i
ExitHere:
    Application.EnableEvents = True

gord dibben say:

You can do it easily with a UDF

Usage is    =concatrange(A1:Z1)

Blanks will be ignored.

Function ConCatRange(CellBlock As Range) As String
'for non-contiguous cells  =ccr((a1:a10,c4,c6,e1:e5))
'note double parens
Dim cell As Range
Dim sbuf As String
    For Each cell In CellBlock
    On Error GoTo fred
    If Len(cell.text) > 0 Then sbuf = sbuf & cell.text & ","
        Next
    ConCatRange = Left(sbuf, Len(sbuf) - 1)   'edited from 2 to 1
fred:
End Function

This is a classic example of starting off from the wrong point. Your times are in fact text values and therefore have to be converted into times when it would have been much better to start off with times in the first place, but here we are!!

 

If we are using time then simply the formula is

 

Period Start time is in A2, period Stop Time is in B2, Shift Start Time is in C2, Shift Stop Time is in D2

and we get the formula to do this as


=MIN(B2,D2)-MAX(A2,C2)

 

If you're not too far down the line in developing this workbook then I would suggest that you change it so times are times and use the simpler formula above.

However as noted these aren't times so we need a more complicated formula in H5 of

 

=(MIN(TIMEVALUE(MID(H$4,8,2)&":" & RIGHT(H$4,2)),TIMEVALUE(MID($A5,8,2)&":" & RIGHT($A5,2)))-MAX(TIMEVALUE(LEFT(H$4,2)&":" & MID(H$4,3,2)),TIMEVALUE(LEFT($A5,2)&":" & MID($A5,3,2))))*24

Hi,

I've looked at the workbook, well done for taking the time to convert to using proper times, you'll benefit from this in the long term.

Looking at this latest question we have a work period of

10:00 to 18:00

and a task period of

05:00 to 13:00

The overlap is therefore 10:00 to 13:00 or 3 hours and the formula in the merged cell I18 calculates that correctly.

Now what you have is 4 hours overtime in P18 but there is no way Excel can know if this is 4 hours at the start or 4 hours at the end of the shift so I'm afrad you're going to have to change the workbook again with 1 column for overtime at the shift start and one for overtime at the shift end. In this proposal I'm assuming:-

Column P is overtime at shift start
Column Q is overtime at shift end

What we now do is subtract any overtime at the shift start to make the start time earlier and add shift end overtime to make the finish time later and the final formula for I18 becomes

=((MIN(J4,(B18+TIME(Q18,0,0)))-MAX(I4,(A18-TIME(P18,0,0))))*24)

and this evaluates to 6 hours

One last thing. You formula start in the merged cell I5 and because you've use relative referencing you can't drag the formula. Put this in I5 and you will be able to drag down and across.

=((MIN(J$4,($B5+TIME($Q5,0,0)))-MAX(I$4,($A5-TIME($P5,0,0))))*24)
