rich007a say:

OK, When I first read your original question, I thought you wanted something that would run as and when a live document has data added to it. 

It sounds more like you just need a macro that can be manually run that will check all cells in a column for an error and if one is found, show the pop-up and request input.  Is that right?

The reason the Worksheet_Change event didn't do anything with the first NA() cell when you dragged it down was because you weren't changing that cell.  Presumably you would have cancelled the input box when you first entered NA() in that first cell.  The Target range that is returned by the Worksheet_Change event covers only the cells that have been changed.

The immediate window is in the VB Editor.  If you can't see it, in the VBE go View > Immediate window.  Typing application.enableEvents=True in a cell will not have done anything.

The code you posted in your last post was not my latest code, did you miss a post?

Anyway, here is some code that you should run manually, either by clicking play inside the VBE, or via View tab > Macros... in Excel.
Hit Alt+F11, then do Insert > Module, then paste this code:

Sub CheckForErrors()
    Dim lastRow As Long
    Dim rng As Range, c As Range
    Dim str As String
   
    lastRow = Cells(Cells.Rows.Count, "O").End(xlUp).Row
    Set rng = Range("O1:O" & lastRow)
    For Each c In rng
        If IsError(c.Value) Then
            If c.Value = CVErr(xlErrNA) Then
                Application.EnableEvents = False
                c.Select
                str = InputBox("What is the Term?" & vbCr & vbCr _
                    & Cells(c.Row, "X").Value _
                    , "Your Title")
                If str <> "" Then c.Value = str
                Application.EnableEvents = True
            End If
        End If
    Next c
End Sub

You should remove any event code you have put in your sheet modules...

sky pei msft say:

Hi Sandy4856,

Yes, this is the a result of document sharing. In addition, we needn't do anything else, as it is an expected behavior. Here is the summary.

Symptom

When user navigates to the default documents library, on the subsite while on the classic UI on the SharePoint Online document library and selects a document and then clicks on the 'Get a link' option, an 'Edit link' is automatically created. If the user closes the 'Get a link' prompt and the navigates to the 'People and Groups' page, through Site Settings, a new group was created for that specific link associated with that document - the new group's name is 'SharingLinks.xxxxxxxxxx.OrganizationEdit.xxxxxxxxxxx'. 

Cause & resolution

An expected behavior – this type of groups are SharePoint Online groups internal to each specific site collection; this is designed this way so that we can allow administrators to directly track users of the shares, and so we can remove those used access when the share is removed; interfering with these groups has a negative consequence on sharing, so those should not be managed manually by removing these groups directly, as their intent is to only link the shared object and it’s children (if it is a shared folder); 

Thanks  for your understanding.

Sky Pei
