Transfer Data into Outlook 2002 from Outlook 2007 -- Not a Question - Microsoft Community

jcw2 say:

Transfer Data into Outlook 2002 from Outlook 2007 -- Not a Question

Here is a detailed procedure that might help anyone still using Outlook 2002, XP, or earlier.  (I have an old XP machine with Office XP and a newer one with Office 2007, hence the problem.)  It's based on an available Microsoft document which is, however, somewhat confusing and insufficiently detailed:

 

How to Transfer PST Data from Outlook 2007 to Outlook 2002
(Note that the transfer in the other direction is straightforward, since Outlook 2007 reads the older format.)


A) In Outlook 2007 do the following steps:  (NOTE that this approach will not work with the 'Contacts' folder because it no longer appears in the folder list in 2007.  See more cumbersome instructions in A' below.)

     1) Create an empty PST file in 'Outlook 97-2002 Personal Folders' format (File/New/Outlook Data File, select "Outlook 97-2002 Personal Folders (.pst)," select a file name and location, name the file).  This new file will immediately appear as 'open.'

     2) Copy whatever items or folder structures you want from the native Outlook 2007 PST into the newly opened 2002 one.

     3) Close the new 2002 file and close Outlook 2007 to release the file from use.

     4) Copy the latter from the computer that runs Outlook 2007 file to the one that runs Outlook 2002.

A') To transfer 'Contacts' from 2007 to 2002 a more complicated procedure seems necessary:

     1') exactly as in (1) above

     2') Export 'Contacts' to a new PST file in 'Outlook Personal Folders' format (File/Import and Export/Export to a file.../Personal Folder File (.pst), select the correct file to export FROM as the new 2002 one is also open, select the 'Contacts' 'folder' that DOES appear here, select a file name and location, name the file).  (This new file does NOT appear as 'open.')

     3') Import 'Contacts' from the new 2007 PST file into a new 2002 one (File/Import and Export/Import from another program or file/Personal Folder File (.pst), select the correct file to import FROM, select the 'Contacts' 'folder' that appears here, check "Import items into the same folder in:" and select the already-open 2002 PST file, click OK to the warning about different file formats, close the newly updated 2002 file).  (It is apparently not necessary to close Outlook 2007 this time.)

     4') As before, copy the new 2002 PST file from the computer that runs Outlook 2007 file to the one that runs Outlook 2002.

B) In Outlook 2002 do the following steps:  (NOTE that this procedure requires a small modification with the 'Contacts' folder because you cannot simply select it in the new PST file and copy it into the native file.  See somewhat more cumbersome instructions in B' below.)

     1) Open the newly created file (File/New/Outlook Data File, select the file).

     2) Copy the desired items/folders from the newly created file to the native PST file.

     3) When done, close the newly created file and delete if desired.

B') To transfer 'Contacts' from 2007 to 2002 this slightly modified procedure seems necessary:

     1') exactly as in (1) above

     2') Copy the desired 'Contacts' from the newly created file to the native PST file (select 'Contacts' in the new file, then select the individual contacts desired to be transferred -- "Select All" only works if the cursor is in the list of contacts -- and finally copy those contacts into the pre-defined 'Contacts' folder in the native Outlook 2002 PST).

     3') exactly as in (3) above

 

brian tillman [outlook mvp 2007-2019] say:

Some data may go missing, yes.  Contact Groups can't be exported and imported.  Connections between contacts and the calendar (like birthdays or anniversaries) may fail to transfer.  The Received date/time may become equal to the import date/time.  In general, importing and exporting is intended to transfer data between Outlook and some other non-Outlook program.


I find the best way to transfer a subset of your data is to create a new PST and copy to it the data you want to transfer, then transfer the PST itself, since it will contain only what you want to move to the other Outlook instance.

I found an article on Spiceworks by Bryce Katz about disabling hardware acceleration in Registry http://community.spiceworks.com/topic/456531-outlook-2013-hangs-loading-profile

I just tried his Fix #2 and it worked for me. 

Fix 2: Disable hardware acceleration 

  1. Run regedit 
  2. Browse to HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Common
  3. Create a New Key and name it "Graphics"
  4. Select Graphics, right-click on the right panel and create a New DWORD (32-bit) Value and name it DisableHardwareAcceleration.
  5. Enter Value data as 1
