ken sheridan say:

Do you want to set the values in the new columns to constants, i.e. for each row inserted will the values for each of the new columns be the same in each row.  If so you can include the constants as literal values in the SELECT clause, e.g.

 

INSERT Into x(a,b,c,d,e)

SELECT a,b,c,"foo","bar"

FROM y;

 

This will insert the varying values of a,b and c from table y into columns a, b and c in table x, and the constants "foo", and "bar" into columns d and e respectively in all rows in x.

boblarson say:

It is fairly easy to do - so you would create a recordset with just the departments and then iterate through and filter the data with another where clause. Download Access MVP Armen Stein's SQL Tools from here:

http://www.jstreettech.com/cartgenie/pg_developerDownloads.asp

And put them into a new module (name it basSQLTools) and then you can use the ReplaceWhereClause which is fantastic.  So then it would be something like this:

 

   Dim rst    As DAO.Recordset

    Dim strSQL As String
    Dim qdf    As DAO.QueryDef
    Dim strOldSQL As String

    strSQL = "SELECT DepartmentID, DepartmentName FROM Departments ORDER BY DepartmentName"

    Set qdf = CurrentDb.QueryDefs("YourQueryNameHere")
    ' save the original SQL statement to return to later
    strOldSQL = qdf.sql

    Set rst = CurrentDb.OpenRecordset(strSQL)

    With rst
        ' go through each department to send the query
        Do Until .EOF
            strSQL = qdf.sql
            strSQL = ReplaceWhereClause(qdf.sql, "WHERE DepartmentID = " & !DepartmentID)
            qdf.sql = strSQL
            DoCmd.TransferSpreadsheet acExport, acSpreadsheetTypeExcel9, qdf.Name, "C:\Temp\" & !DepartmentName & Format(Now, "yyyymmdd_hhnnss") & ".xls", True
            .MoveNext
        Loop
        qdf.sql = strOldSQL
        qdf.Close
        rst.Close

        Set qdf = Nothing
        Set rst = Nothing
