2.  Edge, Cortana, Windows update reported that there was no network.

So, some Windows 10 apps thought that there was no network while others (and legacy applications) did just fine.

Remediation Attempts without any change in symptoms:

1.   Removed Network adapters through Device Manager and allowed them to be rebuilt.

2.   Network Troubleshooter reported that it could not find any problem.

3.   Disabled IP6

I could not find an ISO version of a newer version to try.

richard_725 say:

Resolved.

Steps:

  1. Reverted back to Win7 installation using a image backup.
  2. Re-installed Win10 using the download (not ISO) option.   As you probably all know, it's not possible to know which release is being installed unless you use the ISO option.   As luck would have it, it was 10162 which was the original problem child.   

However, the network is working properly now.   And a couple of other problems which were present before are not there now (including one with audio adapters).

Moral of the story:  

  1. It looks like the network problems were not 10162 per se but rather the upgrade process from the previous version.
  2. Once the symptoms exhibited, the reset option didn't fix a darn thing.

I've seen it advocated that the best way to apply new revisions is to format the disk and apply the revision via iso.  It appears that there is a lot of validity to that approach.

dereknor say:

This thread is ancient, but in case anyone else encounters this sort of problem, I have an excellent solution.

My situation: I  have a Norwegian physical keyboard layout, Norwegian Microsoft Office, US English Microsoft Windows, and most of the time I type in English. However, there is more than one case where I must switch back and forth within a single document or OneNote Notebook. 


First, you need to let Windows know which input languages and what keyboard layout you use. In the Control Panel, find and open Region and Language (inside Clock, Language, and Region if your Control Panel is categorized). On the Keyboards and Languages tab, click Change keyboards... You can also reach these options  by right clicking on your Language Bar (if it is visible) and selecting Settings...  Now Text Services and Input Languages appears. 


There is a very specific, correct way to have only one keyboard layout but many languages! Click Add... Select your input language. Now I will revert to my situation, and you can extrapolate usage for your own situation. I ticked English (United States) > Keyboard > Show More... At that point, you are presented with the full list of keyboard layouts available on the system, not just US English layouts. I ticked Norwegian. Then, find your next language. For me, I ticked Norwegian, Bokmål (Norway) > Norwegian.

Then, Click OK. Now you have in your Installed services list whatever was installed previously and your new selections. You need to pick a Default input language above. This is the language Windows will assume you want to use when you have not specified otherwise. I chose English (United States) - Norwegian. Click Apply.

Next, remove the configurations you do not use. In the Installed services box, I selected my English (United States) > US and then clicked Remove. No more silly, limited, American keyboard!


That was the difficult part. The next bit is super easy!


Click on the Advanced Key Settings tab. The first option, "Between input languages" is what you want. By default, to switch between your installed input languages, you briefly tap Left Alt + Shift simultaneously. You can choose another key sequence to do that if you like. When you are done, click OK on Text Services and Input Languages to save and close. We are done with the Region and Language window as well as the Control Panel.


Finally, you can experience linguistic freedom in your programs. Open Word or OneNote. See the Language bar either at the top of the screen or docked on the taskbar near the clock? Since mine defaults to English, it says EN. When I want to start typing in Norwegian, I tap the Left Alt + Shift, and it switches to NO. Switching is that easy. And now, when I create a new tab or text area in OneNote, it uses whatever language I had selected when it was created. Moreover, if you want to type a mix of languages, you can use the same technique to switch back and forth.


Caveats:
You may need to add, change, or adjust defaults in the MS Office Language Settings.
Copying and pasting is not flawless. Usually it follows the pattern that, if I copy from the web browser with the language bar set to EN, then paste in OneNote, the language pasted will be EN regardless of which language was selected at that time in OneNote.


Hope this helps someone. Sorry to get long-winded.
