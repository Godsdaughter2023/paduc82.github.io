steve rindsberg say:

One possible explanation for this:

Years ago, Microsoft was smart.  If you copied charts or data from Excel and tried to Paste Link it into PowerPoint, PowerPoint simply wouldn't allow it if the XL file hadn't first been saved.

This is simple common sense.  Why permit the user to create a link to a file that doesn't exist, right?

Common sense seems to have flown the coop of whatever developer's in charge of linking nowadays.  For several versions now, you've been able to open Excel, add some data, copy it, Paste Link it into PowerPoint and close the Excel file w/o saving.

And get EXACTLY the result you've described. 

ALWAYS save the Excel file once before creating any links to it.

If you're SURE you saved the Excel file first, let us know exactly what steps you followed to create the link (there are several ways of doing it) and whether you moved the files (XL/PPT) to another folder/computer/whatever after creating the links.

ken sheridan say: You either enter the SQL statement directly as the RecordSource property of the order details subform, or you use it to create and save a query and use the query name as the RecordSource property of the order details subform.  To do the latter you open the query designed, but don't add any tables.  Switch to SQL view and paste in the SQL statement.  Then save the query.

You might not need to reference the InvID column, however.  The SQL statement as I posted it assumed a composite primary key for the PurchaseOrderDetails table, made up of the OrderID and ProductID, but it looks like you have a separate key for the table, OrderDetailID, so the following should suffice:

SELECT PurchaseOrderDetails.*,
DSum("QtyOrd", "Receipts", "OrderDetailID = "
& [OrderDetailID])
AS QuantityReceived
FROM PurchaseOrderDetails;

Similarly with the code to open the frmGoodsReceived form the criterion can probably be amended to:

strCriteria = "OrderDetailID = " & Me.OrderDetailID

In answer to your other post, opening this form from the order details subform was what I had in mind.  The frmGoodsReceived form would open filtered to the currently selected order detail item.
