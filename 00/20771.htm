dirk goldgar say:

Using Access 2010 I have created a form using tab control.

I renamed each tab using caption property and added existing fields to each tab.

The first tab shows the fields on both the first tab and the 2nd tab why ?


That's an indication that you inadvertently placed the controls, not on the tab page itself, but on the form's detail section *behind* the tab control.  It's possible that you actually placed all the controls on one page, but most commonly it's the first case I mentioned.  Do all the controls show on both pages?  If so, then all controls are really on the detail section, not on the tab pages themselves.

To add existing control to a tab page, you can't just drag them to the page.  You have to select them, cut them from the detail section, then click the tab for the page you want, and then paste them.  After you've done that, you can move them around on the page by dragging them, but to get them onto the page you have to cut and paste, and when doing so you have to select the tab page by clicking its tab before pasting.

swischner say:
@Vijay Verma: Sorry, but there is a lot of wrongness in your answer. Also the linked Zapier solutions have nothing to do with sharing whole notebooks on Google Drive! Let me try to clear things up:

To sync OneNote notebooks across several devices on an internal network has been an explicit feature that came with (as far as I remember) OneNote 2007 and it's still possible with OneNote 2013 or 2016. There are some prerequisites for it:

1. You absolutely need the OneNote version, that comes with MS Office and NOT the free OneNote 2016 (there has been a free OneNote 2013 before, introduced in March 2014). As only the "full" Office version of OneNote 2013 or 2016 has the ability to store notebooks locally as an alternative to the cloud. I assume, this is the case here.

2. The notebook(s) in question must be saved as local notebooks, not created on OneDrive. If you open the backstage view (click on "File") there is a list of open notebooks. Under the name you see the storage location. It may not start with https:// but show a "Normal" file path instead.

3. All devices that want to access this notebook, need the Office OneNote 2013. No chance with OneNote for Mac, mobile devices or the OneNote app from Windows 10.

4. The storage location on the server must be accessible from all devices that want to use the notebook and there have to be full read/write/create rights!

The easiest way to open a notebook shared on a network location is to use the Windows explorer. Navigate to the network folder containing the notebooks (those are folders, containing .ONE files) and double click on the open notebookonetoc2 file. This should open it in OneNote 2013. You can do this on all devices.
Usually you should not experience sync issues, except when you try to edit the same object (image, text paragraph) from both machines at about the same time. This is system immanent and normal for every OneNote (even cloud stored notebooks).

Microsoft is not officially supporting shared notebooks on a local network anymore. But this is mainly for product strategy and political reasons (and storing them on OneDrive does indeed have some advantages). MSFT even removed some hints and documentation regarding OneNote notebooks on network shares from the net.

Don't under any circumstances try to store local notebooks in your Google Drive or Dropbox folders. This will produce sync errors like you already experienced. If you want to use cloud storage, stick to OneDrive, as OneNote is using a special sync protocol called COBALT, which is only supported my OneDrive.
