MsgBox "File path and name is " & vbCrLf & "[" & MyFile & "]"

 

That MsgBox line will show you exactly what is in MyFile at that point before continuing.  If it is not what you expect, then we've more work to do.  If the file to be opened is not in the current directory, then you would need the full path to the file along with its name.

From the interface you can only rotate in one degree increments. However in the code object model you can use much smaller increments.

This would be simple code to rotate right by 0.1 deg.

Sub littleRot()
Dim oshp As Shape
On Error Resume Next
Set oshp = ActiveWindow.Selection.ShapeRange(1)
oshp.Rotation = oshp.Rotation + 0.1
End Sub

dirk goldgar say:
Why the push to upgrade the file format, since Even Access 2013 can run mdb files without issue?  What benefit are you expecting?

I can tell you from experience that this is not true.

Access 2013 will NOT convert MDB files to ACCDB format, nor will it import or link to MDB files.

Recent discussion

That's not correct, or only partially so.  Access 2013 can open, import from, and link to database in the Access 2002-2003 MDB format just fine.  However, it can't open databases in the Access 97 MDB format.  I don't know offhand if it can open databases in the Access 2000 MDB format.

Howver, your warning is certainly important in this case , because Scott Xe said he's been developing this database "since Access 97."  That doesn't necessarily mean that it's still in the Access 97 MDB format -- he could have upgraded it to Jet 4 format sometime along the way -- but if it's in Access 97 format he would have to convert it to a later format (probably .accdb) if he wanted to work with it in Access 2013.

Scott Xe only mentioned Access 2007 and Access 2010, so A3013 compatibility may not be an issue at the moment.  If it were me, though, and all users had been upgraded to A2007 or later, I'd go ahead and convert it to .accdb format if it's in the A97 .mdb format now, just to avoid issues further down the road.

Scott -

If the database is currently in Access 97 format, I would recommend upgrading it to .accdb, to avoid trouble late on when some user upgrades to Access 2013.  If the database is currently in Access 2002-2003 format, I'd leave it alone.
