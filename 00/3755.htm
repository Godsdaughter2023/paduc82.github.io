Excel error with ^ power operator with start minus - Microsoft Community

a. user say:

Excel error with ^ power operator with start minus

if you type:   =-(1+1)^2  the result is 4.  here I should have -4

if you type:   =5-(1+1)^2  the result is 1.  

if you type:   =-(1+1)^2+5  the result is 9.  here I should have 1

why does ^ operator not respect parenthesis order when formula star with minus?

iliru say:

This because Excel consider minus sign in front of formula as being part of other parenthesis who include minus sing and the actual formula given in parenthesis. So in this case =-(1+1)^2 Excel consider it as =(-(1+1))^2.

To avoid this, when a formula with power operator start with minus then you should put a zero in front of this formula. Only in this way you can have the correct results.

  • if you have =-(1+1)^2 you should apply this formula =0-(1+1)^2
  • if you have =5-(1+1)^2 you do not need to put nothing in front of this formula because everything is correct
  • if you have =-(1+1)^2+5 you should apply this formula =0-(1+1)^2+5

Or you can use POWER function. Respectively as above

  • if you have =-(1+1)^2 you should apply this formula =-POWER(1+1,2)
  • if you have =5-(1+1)^2 you should apply this formula =5-POWER(1+1,2)
  • if you have =-(1+1)^2+5 you should apply this formula =-POWER(1+1,2)+5

When you use POWER functions you do not need to put 0 in fornt of formula.

For sure you can use other parenthesis to avoid this error. For example when you have =-(1+1)^2 you can apply this =-((1+1)^2). Do the same thing for the other cases.

This type of error is shown only when even numbers are apply with power operator (^) as given above examples (so this sequence of numbers: ... -8, -6, -4, -2, 0, 2, 4, 6, 8 etc.).

Hope this helps,

IlirU

We've had several reports of this problem happening on both Defender and MSE.

Unfortunately, there is no option to disable scanning archive files in the Defender interface on Windows 10, so we can't make a quick change to confirm this as the issue.

You could exclude .zip, .cab, .rar, .iso file types, but I don't think this is a good solution.
