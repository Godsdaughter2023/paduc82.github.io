gary zhu msft support say:

Hi Cranberry,

Thanks for your quick reply. Regarding the "unlicensed mailbox ", what is the type of it, a shared mailbox or resource mailbox?

If it is a shared mailbox, please log into Exchange Admin Center (EAC: https://outlook.office365.com/ecp) with your administrator account > recipients > shared > double click the shared mailbox > mailbox features > Mail Flow > click View details > check "Enable forwarding" > "Browser" (If there are many recipients, please add them into a group first) > select the DG.

If it is a resource mailbox, there is no directly way to forwarding the emails in EAC. But, you can grand the full access permission to a licensed account firstly, and then let the licensed user open the resource mailbox via "Open another mailbox" in Outlook Web App (click the photo and you will find the option). After open the resource mailbox, please click the gear icon in the top right corner > Mail > Accounts > Forwarding > check "Start forwarding" option and input email address you want to forwarded > Save.

srikanth421 say:

You need Notepad++ to read the code well. At lines 23973 and 27284, you are missing end tag </mc:AlternateContent>.  

This is what I did:

1) Right click the file, then zip - extract files.

2) Navigate to Word-> document.xml

3) Open document.xml using a web browser. It will automatically tell you where the error is. 

4) Right click document.xml and select edit with Notepad++

5) In your case you are missing end tag for <mc:AlternateContent> at two places. Fix these and save.

6) Reopen document.xml in web browser and you will see no more errors(maybe some warnings).
