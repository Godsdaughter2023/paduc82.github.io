You can simulate a page header in a subreport by creating a top level group on a constant expression such as =1 or ="Faux page header".  Give the group its group header section and then Copy the controls from the real page header (that doesn't show) in this group header section and set the section's RepeatSection property to Yes.

boblarson say:

Remember, a subreport is actually a report that is displayed within a subreport control.  So, if it was opened as a regular report, you could use the page header/footer.  But when it is used within the subreport control it has limitations which preclude the use of the page header/footer.  That is why you can still get that tool tip because Access does not distinguish when a report is in a subreport control when you are in design view of a subreport because it can be used as a regular report just as easily.

 

EDIT:  And one more thing.  You can create a group on your report just by using =1 as the group field and then if you include the header for that, you can place the headings for your subreport there and then set the group header's REPEAT SECTION property to YES which will then display the headers on each page for the subreport.
