This is how I would set this up.  The approach takes advantage of the fact that when a form is opened in dialog mode -- a special mode that can be specified when it is opened via code or macro -- the code in the calling procedure halts until the form is either closed *or* hidden.

First, I would have a dialog form that is designed to let your user pick any of the records that is displayed on it, by clicking a button.  I'm going to assume that the items to be picked have an ID field -- the primary key -- and a display-text field (which could be text or memo).  You were talking about "rules", so let's suppose that every rule has a RuleID and a RuleText.

Create a form named "frmPickRule", set up for continuous forms view.  Initially, set its RecordSource to a query that returns all the rules from the Rules table, something like:

    SELECT RuleID, RuleText FROM Rules;

Set up the form's AllowEdits, AllowAdditions, and AllowDeletions properties all to No, because you don't intend that user should edit the rules themselves with this form.

On the form, have text boxes bound to the two fields, RuleID and RuleText.  They may as well be named to match the fields to which they are bound -- meaningless names like "Text0" and "Text4" should be avoided, as they are totally uninformative.  However, you don't need the RuleID to be visible, unless it matters to your users.  You can set that text box's Width to 0, set its Visible property to No, and move it out of the way.

Also on the form, have a command button in the Detail section, next to the RuleText, named "cmdPickThis".  The button's caption can be whatever you like, so long as it tells the use that by clicking the button, they are picking the rule that the button is next to.  The code for this button's Click event procedure will be very simple:

    Private Sub cmdPickThis_Click()

        Me.Visible = False
   
    End Sub

All that does is hide the form when the user clicks the button.  Note, though, that clicking the button on a particular row of the continuous form has the very important side effect of making that row the form's current record.

Also on the form, but in the Form Footer section, have a button named "cmdCancel".  This is the button the user would click if they decide not to pick a rule after all.  Set this button's Cancel property (on the Other tab of its property sheet) to Yes.  That will make it so the simple act of pressing the Escape key is the same as clicking the button.  The code for this button's Click event procedure should be as follows:

    Private Sub cmdCancel_Click()

        DoCmd.Close acForm, Me.Name, acSaveNo
   
    End Sub

As you see, this button closes the form, rather than hiding it.

Now, as I understand it, you want to be able to set the recordsource of this form on the fly, so that it can be used to pick from different sets of rules.  We'll set the form up so that you can pass it the desired recordsource when you open it.  Create an event procedure for the form's Open event, like this:

    Private Sub Form_Open(Cancel As Integer)

        Dim strOpenArgs As String
   
        strOpenArgs = Me.OpenArgs & vbNullString
   
        If Len(strOpenArgs) > 0 Then
            Me.RecordSource = strOpenArgs
        End If
   
    End Sub

I believe that's all the essential code for frmPickRule.

At last we can talk about how this form is used.  We go back to the form from which you want to pop up the pick list.  Let's suppose you pop it up from the Click event of a command button named "cmdGetRule", and you want to get the RuleID value of the rule that is chosen and put it in a control named "txtRuleID".  Also, you want to pass the name of a query that returns the specific set of rules you want to offer the user.  Let's suppose that query is named "qryRuleSet1".  Your code for the Click event of cmdGetRule might look like this:

    Private Sub cmdGetRule_Click()

        DoCmd.OpenForm "frmPickRule", _
                WindowMode:=acDialog, _
                OpenArgs:="qryRuleSet1"
   
        If CurrentProject.AllForms("frmPickRule").IsLoaded Then
            ' The user picked a rule, hiding the form but leaving it open.
            Me.txtRuleID = Forms!frmPickRule!RuleID
            DoCmd.Close acForm, "frmPickRule", acSaveNo
        Else
            ' The user chose to cancel, closing the form.
            Me.txtRuleID = Null
        End If
   
    End Sub

And that's about it.  You don't need to use a command button to pop up the dialog form -- I believe you mentioned using the AfterUpdate event of an option group -- but this example should show you how it can work, and I hope you can adapt it to your needs.

Good luck!
