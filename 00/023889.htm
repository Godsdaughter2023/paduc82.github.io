stephen boots say:

Quick scans will check for malware in the most common locations. A full scan checks every file. A found infected file does not mean that the infection is active -- it usually means that a saved file contains malware that might activate if you executed or opened that file. A quick scan would find any active malware that the definitions/signature/engine can detect. Note that no antimalware solution can detect and deal with 100% of the malware in existence.

In MSE, view the History and select details to see where the detections were found. I suspect that they existed in the Java cache or in temporary files someplace and neither were active infections.

A scan of any sort is "after the fact" to begin with. In theory, real time protection, which is always active, should be catching and dealing with infections attempting to act.

 

-steve

The setting state that allow content from My Computer, so it referring to local files and are not internet files. In this case, when you are on internet , you will still get Active X warning when needed for Internet websites and it applies to local files in your PC.
