¡firedog say:

Such surprises I encounter almost daily ... Just looking at a folder containing recently saved .eml files, automatically changes their Date modified to the present minute in time

When you point at a file in Explorer, Windows will look to see whether there is a property handler for the file type involved and, if there is, display an infotip giving some of those properties. For an .eml file, that would be From, Subject and DateReceived.

Those data are extracted from the headers of the message and written to an Alternate Data Stream (ADS) within the file so that Explorer can find them and display them. This would normally happen as soon as the file is written to disk if the location it's written to is indexed, so perhaps the folder you saved it to isn't indexed.

Writing to the ADS is a modification to the file, so its DateModified is updated to reflect this. The important file dates aren't affected - DateSent, DateReceived, DateCreated. Is this a problem?

This strange behavior is unique to .eml files;

I don't know whether that is true. .eml files are plain text files, so there's no provision for special properties within the file itself as there is for, say, .doc or .jpg files. The NTFS file system allows for a file to have many data streams, so that's where this kind of metadata can be stored. If you want to keep your .eml files inviolate, save them on a FAT drive.

debbie_h say:

Hi RVillarreal65,

Please see the following information regarding the Code 39 error:

Code 39

Windows cannot load the device driver for this hardware. The driver may be corrupted or missing. (Code 39)

Cause

There are several causes of Code 39 errors:

  • A required device driver is missing.
  • A required binary file is corrupted.
  • There is a problem with the file I/O process.
  • A driver that is referencing an entry point in another binary file is unable to load.

Recommended resolution

Reinstall the drivers for this device.

Uninstall and reinstall the driver

Uninstall the driver from Device Manager, and then scan for new hardware to install the driver again.

You may be prompted to provide the path of the driver. Windows may have the driver built-in, or may still have the driver files installed from the last time that you set up the device. However, sometimes, it will open the New Hardware Wizard which may ask for the driver. If you are asked for the driver and you do not have it, you can try to download the latest driver from the hardware vendor’s Web site.

  1. On the device Properties dialog box, click the Driver tab, and then click Uninstall. Follow the instructions.
  2. Restart your computer.
  3. Open Device Manager, click Action, and then click Scan for hardware changes. Follow the instructions.
