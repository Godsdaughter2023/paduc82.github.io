Hi, Bdog1,


What version of Windows are you using?

I/O errors when emailing from your software

NOTE:

Be aware that only AccountRight v19.9 (and later) and AccountRight 2013.2 (and later) are compatible with Microsoft Outlook 2013 (32-bit version). Other email programs will not be compatible unless they are MAPI compliant. Refer to your email program help and support documentation or contact the software vendor to determine if your email program is MAPI compliant.



I/O Error in Routine

The above error may be received when selecting the “Send To” > Email function from within sales, purchases or reports. This however may not affect all the “Send to” email functions.

This means that the error may occur in sales, and may not effect the purchases or reports “Send To” email function.

This is due to the “Send To” email function crashing or timing out at some point and leaving the ESale, EPurchase or EReport PDF document in a read only state in the users temp directory.

In other words, when the “Send To” email function is used, AccountEdge will generate the PDF document and store it in the current temp folder for the user logged into windows as ‘Read Only’ so it is not deleted or changed while the email is generated. When the email is finished generating in the email client (e.g. Outlook or Mail), the PDF document is deleted. In the event of a time-out or crash, AccountEdge does not get the chance to delete the pdf and it is left behind in a read-only state.

The ‘Read Only’ status will prevent Disk Clean-up from removing the file automatically. However, you can use the Windows Search utility, which is accessed from your computer Start menu, to locate the offending file on your Local Hard Drive C:/ and then delete it manually.

  • Select the All files and folders option.
  • Select More Advanced Options.
  • Tick the Search hidden files and folders option.
  • Click Search.

The PDF files are normally found here (replace username with the Windows login user name):

  • Windows XP - C:\Documents and Settings\Username\Local Settings\Temp
  • Windows Vista – C:\Users\username\AppData\Local\Temp
  • Mac OS X - /Applications/AccountEdge 20xx/Email attachments

Note: If you do not find the Email Attachment folder in your AccountEdge Folder, check to be sure that the permissions on the AccountEdge folder are set to read and write.

Once located, you can delete the offending PDF document depending on which area gives the error.

  • Sales = ESale.pdf
  • Purchases = EPurchase.pdf
  • Reports = EReport.pdf



I/O Error in Routine

In some cases, particularly in newer versions of Windows, this error can also be caused by the user's Windows permissions. Make sure you are logged into Windows as the Administrator, or as a user with Administrator permissions. Full access is required to the system drive (where Windows is installed, usually C:\), or at a minimum, full access to the following

Windows XP: C:\Documents and Settings\*Username*\Local Settings\Temp

Windows Vista: C:\Users\*username*\AppData\Local\Temp

Windows 7 and 8: C:\Users\*username*\AppData\Local\Temp

If required, you may need the assistance of an IT specialist regarding Windows permissions and access restrictions.
