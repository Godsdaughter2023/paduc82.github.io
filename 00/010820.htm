brian tillman [outlook mvp 2007-2019] say:

All that you describe in your first paragraph is normal for IMAP. Since all messages are kept on the server, Outlook provides a separate PST for IMAP accounts with folders that reflect what's on the server. IMAP folders can never be the default folders because IMAP servers do not support needed default folders like Calendar, Contacts, Tasks, Journal, and Notes and all of those folders are necessary to Outlook and all must be in the default folder set.


Deletion doesn't really take place at the time you click Delete. Instead, the messages are marked for deletion and then later deleted when you close Outlook or purge the folder. The IMAP standards say that's how IMAP works. If you'd like to stop seeing the strike-throughs, change your view to "Hide Messages Marked for Deletion" and then when you click Delete, those messages will disappear. They'll still be in the folder, but they'll be hidden and will be purged later.


Making an account the default and making a folder set (data file) the default are two completely different things. Accounts and data files are separate. You certainly can make the IMAP account the default. Click Tools>Account Settings. On the Email tab, select the IMAP account and choose Set as Default. That means that when you create a new message, the IMAP account will be chosen as the sending account. The default data file (or folder set) is chosen on the Data Files tab. As I said, because IMAP servers cannot support needed default fodlers, you will not be able to choose the IMAP data fle as the default.


Outlook 2010 improves how IMAP accounts are handled in that you can tell Outlook to treat deleted messages in a more traditional fashion by moving the messages to the Deleted Items folder on the server and by using the "Hide Messages Marked for Deletion" view by default so that it appears to behave more as you would expect. Outlook 2007 can't be made to do that, however.

williad(2) say:

What is the formula for a 12 month rolling total?

I can do a running total or running balance but I want one which only takes into account the preceding 12 months.  For VAT purposes I have a maximum amount I can earn over a 12 month period. As I have an hourly rate therefore also a maximum number of hours. This is a rolling 12 months, not a financial year. I can enter my hours and earnings for each month and total them but I want to show for each month the total for the preceding 12 months. If I could also see the balance remaining that would be a bonus.
Any ideas?
Dave
jllatham say:

Assumed:  Dates are full dates (with month, day and year) in column A

Assumed: Values you want the rolling total for are in column B

Assumed: First data entries are in row 2

 

This solution is for Excel 2007 or later - if you are using a version of Excel earlier than 2007, then we have to rewrite this formula because SUMIFS() doesn't exist in them.

 

This actually gives you a 13 month running total (12 months preceding plus current month, so for August 2013, it would give you total of August 2012 through August 2013)

Put this formula into a cell on row 2 and fill down to the end of your list:

=SUMIFS(B$2:B2,A$2:A2,">=" & DATE(YEAR(A2)-1,MONTH(A2),DAY(A2)),A$2:A2,"<=" & A2)

 

This one will give you a 12 month total (12 months preceding, but not current month, so for August 2013, it gives you total of August 2012 through July 2013)

=SUMIFS(B$2:B2,A$2:A2,">=" & DATE(YEAR(A2)-1,MONTH(A2),DAY(A2)),A$2:A2,"<=" & DATE(YEAR(A2),MONTH(A2)-1,DAY(A2)))

 

After entering the formula into a cell on row 2, simply fill it down to the end of your data entries.

 

Take note of the use of the $ symbol (and lack of it) in referring to the cells to be included in the formula - it's important.

The Border Painter definitely should be there.

If a repair of Office 2016 doesn't make it show up, you can either add a custom group to the Table Tools | Design ribbon and place the Border Painter tool in it, or you can add the Border Painter button to the Quick Access Toolbar.

Never manipulate a PST outside of Outlook or the Mail applet in Control Panel when it is part of an active mail profile.

PSTs don't "pull mail accounts".  Your mail profile controls what mailboxes your account(s) access and that is what you need to fix.  If you need the data in the PST, restore it from the Recycle Bin and Outlook will work again.  If you've emptied the Recycle Bin, you'll need to create a new mail profile using the Mail applet in Control Panel.  See http://www.howto-outlook.com/faq/newprofile.htm.  If you had only one account, I hope you realize that not only is your mail now gone, but so is your entire ccalendar, all of your contacts, and any notes or tasks you created for yourself.

atithd say:

Make sure delete all messages from your Trash mailbox before you begin the import process.

1.Launch Outlook.
2.Make sure that you have closed all the folders in the folder tree. Select File | Import and Export.
3.Select Import Internet Mail and Addresses.
4.Click Next.
5.Select Eudora (Pro and Light) 2.x, 3.x, 4.x.
6.Check the data you wish to import—Import Address Book and/or Import Mail and click Next.
7.Click Finish

In order for Outlook to locate and successfully import your Eudora mailboxes and address book(s),

The files MUST reside in the C:\Program Files\Qualcomm\Eudora folder. If you store your mailboxes, address book, etc. on a network drive (e.g., H:\eudora), then you need to copy the following files from H:\eudora to C:\Program Files\Qualcomm\Eudora before you begin the import process to Outlook:
•*.mbx - mailbox files
•*.toc - table of contents files
•nndbase.txt - nickname file
•*.fol folders - Eudora folders containing mailboxes
•Nickname folder (if it is not empty)
•Attach folder (if it is not empty)
•descmap.pce - keeps track of your mailbox and folder list


Note: If you are not going to import all of your Eudora mailboxes and folders, open the copied "descmap.pce" file with Notepad and remove any mailbox and folder names that you are not going to import. If you don't, you will get an Import Error (e.g., The mail folder could not be opened. If another application is using this file, please close it and try again).
