If you're programming, you can loop through all worksheets:

    Dim wsh As Worksheet

Yeah, sorry I didn't reply earlier there was more to this.

For us, after we did this, Project and Visio worked for one day and acted like they were activated, but then went back to throwing the license error.

We repaired the install and then immediately went in to change the activation. When you do this you have to look carefully for the little links about applying a key, the new default is using a login and that doesn't work.

What would have worked better and I recommend is to include the key in your XML configuration of the Office Deployment Tool. Otherwise, it will not ask you for the keys and you have to manually change it as we did.

Finally, double check the installation technology as it has to match. Ours both were Click to Run in the end and that wasn't the problem, but it won't work otherwise. The Office Deployment Tool is considered Click to Run and you will definitely need to use this to deploy Project and Visio. The installation of Office 365 where you use the link in the Office 365 portal also was Click to Run. You can tell if your current install is in Click to Run under the File > Account of a recent Office app and it will mention Click to Run.
