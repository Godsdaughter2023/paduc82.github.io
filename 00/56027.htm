After much digging I came across ASKVG's article and using that original article and reading through all the comments for that article came up with a solution.  I spent two nights with tier one support and never got a solution.  I was supposed to have gotten a call from tier 2 but have not received one.

 

Anyway, here is the solution:

 

If you read AskVG’s article and follow the solution it does not work.  The basis of the article is to create a configuration file and tell the installer what version of Windows you are installing.  As this was geared towards the retail version it did not work for me.  However, reading the comments from this doesn't work through all the variations I was able to put together this solution.

 

1. Create both an ei.cfg files and a pid.txt file.

2. Download Windows 10 ISO and mount it.  Or insert the installation DVD into a drive.  Copy either one onto another media where it will fit as you will need to add these files to it. (As I had a failed installation of Pro that would only install as Home I created a folder on my desktop called WinInstall and copied the entire contents into that folder).

3. With notepad create a file called ei.cfg with this info.  Make sure to match the case exactly and use brackets around the keywords:

 

[Edition]

Enterprise

[Channel]

OEM

[VL]

0

 

Save this file as ei.cfg (Use quotation marks around the name or the extension will be .txt.  At the save-as prompt type: "ei.cfg")

4.  With notepad create a file called pid.txt with this info in it:

 

[PID]

Value=VK7JG-NPHTM-C97JM-9MPGT-3V66T

 

Save this file as pid (Notepad will add the .txt extension).

5. Copy both of these files into the SOURCES sub-directory.  It will be found in the directory you created to copy over the ISO or DVD.  In my case it was in the WinInstall/Sources

6.  You can either run the setup.exe program from the root of the directory you created, in my case WinInstall, or create a bootable DVD or USB key with the contents of WinInstall in the root of that device.  I ran it direct from the folder I created on my desktop.

 

Once setup starts, it asks to get important updates, I chose "Not right now".  Accept the EULA, and it will do some checks like making sure you are ready and a disk space check.  Once that is done it should say, ready to install.  What's nice is it should say "Install Windows 10 Pro".  You can choose what to do with your personal files and apps.  Then start the install.  If it does not say Windows 10 Pro, stop the install and go back over the ei.cfg and the pid.txt files to make sure it matches exactly.  You are using the OEM version correct?  You could try the ei.cfg file from AskVG's article (https://www.askvg.com/fix-cant-select-windows-10-pro-edition-during-clean-installation/) if you have the retail version of Windows 10 Pro.

 

Once you complete the installation, you should see you have a non-activated copy of Windows 10 Pro.  Enter your OEM product key, it should now accept it and activate.
