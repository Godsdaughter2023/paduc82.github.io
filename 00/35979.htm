 - Hard reset the Xbox - Hard reset the Xbox by holding down the power button on the front of the Xbox for 15 seconds. After the 15 seconds let go of the power button then restart the Xbox.

 - Try another profile on your Xbox console

 - Remove your profile and add it back to the Xbox console

 - When its done it should state no updates needed

 - Choose Close

6) Pull the batteries out of the controller and leave them out until the Xbox has restarted from the step below

7) Hard reset the Xbox - Hard reset the Xbox by holding down the power button on the front of the Xbox for 15 seconds. After the 15 seconds let go of the power button then restart the Xbox.

8) Put batteries back in the controller

9) Re-sync your Xbox One controller to refresh the connection back to the Xbox.

 - If you're not sure how - https://support.xbox.com/en-US/xbox-one/accessories/connect-a-wireless-controller

tryx3 say:

Que Basil,


Many users have reported better results from SFC by running it from their Windows 7 installation disk.  SFC uses the installation disk as a source of uncorrupted replacement files so SFC detects problems and then copies files from there to your hard drive without any special action on your part.  This procedure just makes use of the installation disk as a repair tool and does not involve having to install anything.


Please do remember that SFC does one specific job and is not a universal system repair facility.  SFC detects problems with, then tries to repair, non-configurable protected system files.


You will normally use SFC by running it in an Admin Command prompt within Windows.  If, after at least three runs, SFC still reports that it cannot fix all the corrupted files it has detected then you can consider running it from an installation disk.


[Before starting this procedure, you might wish to look at the alternative recommendations in a WiKi article System file check (SFC) Scan and Repair System Files.  It recommends using SFC normally & then using DISM commands if SFC reports that it could not fix all the corrupted files.  I have not looked into this method at all but you might prefer to use it.]


If you have updated Windows 7 to SP1 then you can only use an installation disk that is at the SP1 state.  In case you do not have an SP1 installation disk, there’s an illustrated procedure for making & testing an SP1 installation disk [USB or DVD] at Precautionary measure 4 - Make & test a Windows 7 SP1 Installation Disk  [PDF version] 


The procedure & the command syntax you need to use to run SFC this way is a bit different so here's a blow by blow account:-


1  Before starting anything, open Windows explorer & check the path of your Windows folder.  It will normally be the Windows folder off the main root of your hard drive but there are sometimes variations [C:\Win, C:\Win7, ... - as far as I can gather these variations will normally only be found on systems that were upgraded from earlier Windows versions unless somebody took a deliberate decision to use a non-standard folder name during a standard installation].


2  Insert the Windows installation disk, restart the system.  When the PC maker’s logo appears, set the system to boot from the DVD / USB drive as appropriate by pressing repeatedly on the Boot options key or the BIOS key identified onscreen at this point [or identified in the PC manual]. 


3  After several minutes [that can include periods with a blank screen & no apparent activity], wait during the “Starting Windows” screen then, at the first Install Windows dialog, select the language the time-currency format & the keyboard.  Then click Next. 


4  At the next Install Windows dialog, select Repair your computer [not Install now], wait while it searches for windows installations. 


5  At the first System Recovery Options dialog, note the label being used, temporarily*, to identify your OS hard drive because it might not be called the C:\ drive then set the radio button for Use recovery tools … then click Next.

*  The boot-up disk sets its own temporary rules but this does not affect how the system will refer to the C:\ drive when you later reboot normally. 

 

6  At the System Recovery Options menu select Command prompt.


7  In the Command prompt, if your Windows installation has been identified as being on the C:\ drive and you had previously established that your Windows folder was actually in a folder called windows, enter


                    sfc /scannow /offbootdir=c:\ /offwindir=c:\windows


Run SFC this way at least three times to give it a good chance of detecting then repairing problems.  This is not a waste of time - several users have reported that SFC detected & resolved all problems only after its third run.  Personally, if SFC detects a problem, I give it three more runs after that to give it a good chance of repairing that problem and then detecting & repairing others.


8  If & only if SFC keeps reporting unfixable errors after multiple runs then create an output file manually as there is no CBS.log file created when SFC runs from the installation disk.  The resulting file does contain a lot of distracting rubbish as well as error detection reports but it does the job. 


8.1  To create an output file use this command


                    sfc /scannow /offbootdir=c:\ /offwindir=c:\windows >”C:\Users\YourUserName\Desktop\SFCResults.txt”


being very careful to replace YourUserName with your actual user name, to keep the spaces between elements of the commands, to keep each of the different parts of the command all in one line & to enclose the output file name-path in quotation marks in cases where your user name has any spaces in it -

sfcSpace/scannowSpace/offbootdir=c:\Space/offwindir=c:\windowsSpace>”C:\Users\YourUserName\Desktop\SFCResults.txt”


8.2  Review the output file in Notepad [which you can open from the Command prompt straightaway for an initial look] to see which files are at fault & then research these files in this forum & generally online.  If you cannot find anything to solve problems with these specific files then you’ll be forced to do an "In-Place Upgrade" [also called a "Repair Install"].  To run the Repair install, use the illustrated procedure* at Repair Install - Windows 7 Forums and also have a look at the MS summary at How to Perform an In-Place Upgrade. 


[*  But when the Repair install has all but finished & prompts you for your Product Key do not enter it, instead clear the box for Automatically activate Windows when I’m online then click on Next.]


8.3  After completing a Repair install, there is some tidying up to do.

8.3.1  Check your activation status at Control panel, [View by - Small icons], System.  It is shown as the second-to-last entry near the bottom of the System window.  You will probably find that its status has not been affected by the Repair install.   If it is not activated then use the Product Key on your CoA  / Proof of licence label [checking, of course, that the label matches the Edition of Windows 7 that you are using].  If it still cannot activate then phone the Activation Helpline - enter slui 4 in the Start menu search box to get the phone number.  When you call , press no buttons at all - stay on the line and a human operator will eventually answer so you can explain the situation.

8.3.2  Check drivers, applications & Windows settings as they might need to be re-installed or reset. 

8.3.3  Check & adjust settings for your networks.

8.3.4  Re-enable your malware protection if you had to disable it to run Repair install.

8.3.5  Make a new system image as a protection against future system problems.

jeffxs say:

Hello Singh Sarandeep,

thank you very much for your answer and hints. However ... I checked all those things before I put the question.

I have found the source of all those problems in registry.  There were deleted some sub-keys in  "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Print\Monitors".

I could not find any port in "print-processor manager" (USB nor TCP/IP or LPT) because port monitors specifications were missing in the registry.

All I had to do to repair whole functionality was adding to "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Print\Monitors" sub-key "Local Port" with item "Driver" of value "localspl.dll"; sub-key "USB Monitor" with item "Driver"
of value "usbmon.dll"; and sub-key "Standard TCP/IP Port" with item "Driver" of value "tcpmon.dll".

In sub-key "Standard TCP/IP Port" i created a sub-key named "Ports" with following items:

"LprAckTimeout" of decimal value 180

"StatusUpdateEnabled" of decimal value 1

"StatusUpdateInterval" of decimal valie 10.

... after that all ... did restart my computer and everything is ok.

It is risk to edit registry, but I hope this description will help to anyone else who faces a problem with the same description.
