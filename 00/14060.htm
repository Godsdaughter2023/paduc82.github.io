stevekra say:

Thanks Knut - I looked at your spreadsheet and I think a good workaround would be to stop using the TEXT function to format the values.  Instead, I think you can just calculate the actual date values and use Number Formatting to get the desired format regardless of whether you open on Mac or Windows and regardless of the computer's Location/Region setting.

It would take more detailed investigation, but I suspect that the problem here is just incompatibility between MacOS and Windows with regard to the date format codes for Norway regions. In the attached screen shots from Windows and Mac, you can see that the format code for year differs between MacOS and Windows. The format code is within the double-quotes in the TEXT function, which means it's not stored as one of Excel's internal format codes.  So I believe Excel needs to ask MacOS, "what is the code used to indicate the year (in the current region)?"  MacOS responds with ïïïï, and Excel checks to see if part of your format code matches.  Since Excel finds åååå rather than ïïïï in the format argument of the TEXT function, it treats it as text.

Instead of using the TEXT function, if you calculate the date, and then use Number Format to set the date format, Excel should always be able to show the proper format regardless of OS and regardless of Region.  So if you create the workbook in Norway and share it with someone in an English locale, it will still show the desired format, even if the characters used in that region are different.

To summarize, I recommend only using the TEXT function in cases where you need to concatenate some other text to the date. For example, if you want to say "Today is " & TEXT(NOW(), "dddd")), you would need to use the TEXT function. If the cell will only contain the date, then I would rely on cell number formatting to get the desired format.

Copy/paste this code in ThisWorkbook code module:

Private Sub Workbook_SheetActivate(ByVal Sh As Object)

    Dim v As Variant
    Dim bln As Boolean
   
    If Sh.Name = "Sheet1" Then Exit Sub
   
    v = Application.InputBox("Insert password")
   
    If Sh.Name = "Sheet2" And v = "abc" Then
        bln = True
    ElseIf Sh.Name = "Sheet3" And v = "xyz" Then
        bln = True
    Else
        bln = False
    End If
   
    If bln = False Then
        MsgBox "Operation is not allowed." & vbNewLine & _
            "Select " & Sh.Name & " and enter the correct password."
        ThisWorkbook.Worksheets("Sheet1").Select
    End If
   
End Sub

karl timmermans [outlook mvp 2012-2018] say:

Just for the record for anyone reading this.......

 

When moving backwards from O'2013 to an earlier version, suggestion is to create a new profile using the <Mail> applet OR use a profile that was not used by O'2013. Have gone through the process multiple times with no issues encountered (not to be construed as a statement of preference for or against O'2013 over earlier versions - just activities resulting for testing purposes).

 

Outlook profiles persist between installs and programs such as CCLeaner do nothing when it comes to Outlook profiles. Registry cleaners of any kind will similarly not remove existing Outlook profiles. A common misconception among many related to resolving Outlook profile related issues is to uninstall/re-install Outlook which is a complete and total waste of time and effort since that will never resolve a potential underlying profile related problem, whatever that may be (hence the misguided perception that only a "clean install of windows" is the solution since a clean install will result in earlier Outlook profiles no longer existing).

 

In short, if anyone wants to use Outlook as if it (it = any Outlook version) has never been installed on the computer before is to create a new profile and make it the new default.
