Loop through the section's Controls collection in its Format event procedure, e.g.

 

Private Sub Detail_Format(Cancel As Integer, FormatCount As Integer)

 

    Dim ctrl As Control

   

    For Each ctrl In Me.Section(acDetail).Controls

        If SomeControl = 1 Then

            ctrl.ForeColor = vbRed

        Else

            ctrl.ForeColor = vbBlack

        End If

    Next ctrl

       

End Sub

 

NB:  this won't work in Report view, only in Print Preview or if the report is sent to a printer.

If you haven't already set up your options to view anchors, please follow these steps:

  1. In Word, select Word>Preferences.
  2. Click on the View control panel.
  3. Check Object anchors and All. OK out.

Bob already mentioned that Picture Placeholders don't work in protected documents. This isn't a way to make Picture Placeholders work, this is a workaround to insert pictures in a form.

Here is a sample of a text box anchored to an unprotected section. It will hep to understand the concept if you download this: Text box anchor

If  you turned on All as recommended, opening will reveal 3 paragraph marks separated by 2 continuous section breaks on page 1. The first says End of Protected Section, while the second says Section Break (Continuous). This indicates the second section is unprotected.

If you click before the second paragraph mark, you can type text, but with the other 2 marks, you cannot. This is because the second section is unprotected.

Now click on the prompt in the frame Click here and Insert>Photo>Picture from file. Follow the prompt and insert a photo. 

Now click on the text box so it is activated. Use Insert>Photo>Picture from file to insert a photo.

Finally, use Tools>Unprotect Document to unlock the form. Click on the frame, then on the text box. If you set up the view to see object anchors, you'll see an anchor icon appear to the left of the second paragraph mark. This show the section that the object is anchored to. You can click on the anchor and drag it to a different paragraph mark, but that would make the object ineditable.

Now open and unprotect your document. Click on the text box where the picture should be placed. Its anchor lights up beside the line BLANKS ([   ], etc. This section contains form fields, so it must be protected. But since the text box is anchored to this section, it will become ineditable.

Somewhere in the document, you need to create at least a 1-line unprotected section. The best practice for your form would be to insert this just before the A. Identification heading, so if the form expands above it will push everything down and the photo will stay aligned. Then you need to drag the anchor to appear to the left of this unprotected line. After placing the anchor, choose Format>Shape>Layout>Advanced>Lock anchor. Now the anchor will remain fixed to the unprotected section.

As a finishing touch, you can select the paragraph mark of the unprotected section, format the font to white and 1 point. This ensures the final unprotected section is tiny and typing inside doesn't have any visible effect.

If you prefer the frame with the text prompt, you can copy and paste the example, or you can make a new one by making the Developer tab visible and choosing Insert Frame from that tab. Please write if you need help with this.

The text prompt is a MacroButton field. When the form is unprotected, you can right-click on it and choose Toggle field codes to edit the text.

john korchok say:

To make a frame, you have to make the Developer tab visible. Since you have already inserted form fields, this must be visible already, but I'm including it for the benefit of other readers:

  1. In Word, choose Word, then Preferences.
  2. Under Personal settings, click on Ribbon.
  3. Scroll down to Developer, check it and OK out.

Now you can insert a frame:

  1. On the Developer tab under Form Controls, click on Insert Frame.
  2. Draw the frame on the page.
  3. By default, the frame is drawn with a black border. To change this, click on the border of the frame. Corner and side handles will appear.
  4. Choose Format>Borders and Shading and set your preferred format. With ersatz picture placeholders, I usually eliminate the border and apply a 5% black shading.
  5. Using the Format>Frame command, you can set the size, position and wrapping. Once you get the anchor of the frame positioned next to the unprotected section, you should check the Lock Anchor option in this dialog to ensure the anchor sticks to the unprotected section even if you move the frame.

Now for the text prompt:

  1. Click inside the frame and choose Insert>Field.
  2. In the Categories section, leave the default choice at Document Automation.
  3. In the Field names section, click on MacroButton.
  4. In the Field codes field, type in a fake macro name like NoMacro.
  5. Type a space, then the prompt text you want to appear, then OK out.

This prompt can only be one line long, so if the prompt is lengthy and the box tiny, you'll have to make the point size smaller to make it fit.

Is this only for this message or any?

A "custom action" would suggest that you or trying to open a message based on a custom form.

Other possibilities are that the issue is caused by add-in that you have installed or a custom macro that you created.

To troubleshoot;
Can you repro this issue in Outlook Safe Mode?
Start-> Run; outlook.exe /safe
(note the space in the command)

If it works now, your issue is most likely caused by an add-in.
Which add-ins do you have installed?

Disable them all and see if the issue still occurs. If not, you can enable them one-by-one to find the culprit. For details see;
http://www.msoutlook.info/question/88

Created macros can be found via;
Tools-> Macro-> Macros
