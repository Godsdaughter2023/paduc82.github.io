ken sheridan say:

Hi Ken, the PK for tblWorkItem table is WorkItemID.

Your suggestion sounds interesting could you get me started on the coding side of things please.

The code for this would be:

Dim strSQL As String

' insert new row into tblReactiveWorkPoList
strSQL = _
    "INSERT INTO tblReactiveWorkPoList (ClientID, AssetNumber, " & _
    "Address, DESCRIPTIONOFWORKS, ProfessCode) " & _
    "SELECT tblWorkItem.ClientID, tblWorkItem.AssetNumber, " & _
     "tblWorkItem.Address, tblWorkItem.Description, tblWorkItem.ProfessCode " & _
     "FROM tblWorkItem " & _
     " WHERE WorkItemID = " & Me.WorkItemID

CurrentDb.Execute strSQL, dbFailOnError

One thing to note, however, is that this will insert a new row both when a new row is inserted into tblWorkItem, and when an existing row is edited in the form.  If in the event of an edit of an existing row you'd want the values in the corresponding row in tblReactiveWorkPoList replaced with the new values then you'd need to firstly delete that row by executing a DELETE statement in the code, but as there seems to be no way of relating the a row in tblReactiveWorkPoList  to the corresponding row in tblWorkItem I see no way currently of doing that.  It could be done if a WorkItemID column was also present in tblReactiveWorkPoList, and the latter contained only one row per WorkItemID  value.
