Having your email account configured as an IMAP account (or POP3 for that matter) is absolutely relevant.

First, folders for different item types (i.e. contacts/calendar etc) do not "work together" under any circumstance. Outlook sync's data to a server and what gets sync'd depends on how the account is configured - specifically...

POP3 - downloads <Inbox> only in a one-time one-direction process

IMAP - email folders are sync'd bi-directionally between the server and computer/device. Contact/calendar data is not sync'd with the server

Exchange - syncs all item types bi-directionally.

Going back to the opening sentence of your post

>>
I've recently moved my Outlook mail, calendar and contacts from a networked Win 7/Office 365 machine
<<

Outlook has never supported maintaining any data files on a network drive (or cloud location). "not supported" does not mean "not allowed". That configuration can lea to data corruption and if you've never encountered any kind of data problems - you have been very lucky.

So to get back to the prime question - " How can I change the location of my folders so that they work together?" - answer is "location of folders has nothing to do with anything in terms of <working together>, all comes down to how the email account is configured (POP3, Imap or Exchange)

Hi Irish,

 

It depends on what email account you are using. If it's an Exchange (for example Office 365) account, emails, calendars and contacts are all two-way sync with the server, so you could regard it as a sync between Outlook for PC/Mac and Outlook for phone.
