davidf2 say:

It would be easier to help if we could see the site and test it ourselves. Can you post a link to your site?

Publisher effectively produces two sets of code...one for IE and one for all other browsers. Assuming that the navbar does render but the links are dead in the second browser, my first guess is that you are using FireFox or some non IE browser on the second computer. That would suggest that the navbar is being converted to an image for FF which kills the links. You can test for that by trying to select the text of the navbar by left click drag to select the text of the navbar on the previewed web page. If you cannot select the text then it has been converted to an image. That happens under a couple circumstances.

If your navbar is grouped with other design elements or is composed of grouped elements, then this will be rendered as a single image in FF. The solution is to select all the design elements in your Pub doc and Arrange > Ungroup. This is a common reason for the bottom textual navbar produced by the Publisher navbar wizard not working in FF.

If your navbar is a single text box with hyperlinked text words that you manually built and if you are using either a fill color and/or a fancy border then Publisher will sometimes convert that text box to an image for FF, thereby killing the links. The workaround is to either remove the fill color and/or borders and just use the plain text box. Some fill colors won't do this and some of the basic borders also get by, so you can experiment. You can also duplicate your original effect by creating one plain navbar text box with the hyperlinked text but without any formatting and a second, identical sized text box with only the fill color and/or border and layer it on top the first plain text box and then Arrange > Send to the back.
