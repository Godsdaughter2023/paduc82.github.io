There are many differences in the UI, the biggest is that Excel up to 2010 is MDI and 2013 and above is SDI.

Your old codes are almost compatible, but there many parts that have changed, please read this article:
https://docs.microsoft.com/en-us/office/vba/exc...

And since 2007 we have a Ribbon instead of a CommandBar, the CommandBar object is still in Excel, but most code and/or the beahvior is incompatible/different.

If you look at your old codes, pay attenion on every word that is or includes "Select", in many cases such codes causes issues.

Avoid to use Protect / Unprotect on sheets, that needs now a long time.

I've made some tests and found that the grouping of the sheets is the problem, try the changed code below.
In my tests I found that your "old way" did work as before, even if DrawingObjects is now a hidden property.

You can use such properties, but the hidden state is like "this object is marked to be deleted in a future version", consider to rewrite the code. Therefore I've added a conditional define, so you have the codes side-by-side.

Andreas. 

Option Explicit
#Const Version = 1
Sub DeleteShapes()
  Dim SaveSelectedSheets
  Dim Ws As Worksheet
  Dim Sh As Shape
  Dim C As New Collection
  
  'Backup the current selected sheets
  Set SaveSelectedSheets = ActiveWindow.SelectedSheets
  'Store each sheet object in a collection
  For Each Ws In SaveSelectedSheets
    C.Add Ws
  Next
  'Ungroup the sheets
  ActiveSheet.Select
  'Get the sheet objects back from our collection
  For Each Ws In C
#If Version = 1 Then
    Ws.DrawingObjects.Delete
#Else
    'Delete the shapes
    For Each Sh In Ws.Shapes
      Select Case Sh.Type
        Case msoComment
          'Don't delete comments this way!
        Case Else
          Sh.Delete
      End Select
    Next
#End If
  Next
  'If necessary: Restore the selected sheets
  SaveSelectedSheets.Select
End Sub

Is 14393 the anniversary edition build - Microsoft Community

jim mccoll say:

Is 14393 the anniversary edition build

Or will Insiders receive a further update in the next day or two?
cbarnhorst say:

Nothing.  People are misunderstanding what MS really does. 

The Anniversary Update is 14393.0.  That is what went to the OEMs.

MS will download 14393.0 to users starting this weekend (timelocked so it cannot install until Tuesday and after) and will continue downloading for several days until 350,000,000 devices have their downloads.  Those that get the package Tuesday and later will experience the upgrade shortly after.  Everybody who gets the download over Sunday and Monday should experience the upgrade right away on Tuesday.  The main download is not the only load on the servers.  Remember that Setup search the web for updates during the installation process.

The part of the build string to the right of the decimal is called the minor build string.  14393.0 is what is downloaded but 14393.5 is what will be installed.  Sounds a little inconsistent but it is normal.  I'm sure you recall the stage in the installation of a build when Setup announces it is Searching For Updates, wheel spins, and the percentage counts up.  This is when updates, including the latest cumulative update are downloaded.  There is a later stage after the first reboot (maybe even after the second) when Setup announces that it is installing updates (same wheel, same percentage counting up).  That is when 14393.0 becomes 14393.5 and that is why there is no new cumulative update waiting for you after installation is complete.

You will notice that there are updates the first time you run WU.  But look closely at them.  They are for components that are not part of Windows.  Adobe Flash, Defender, maybe a Logitech camera or HP printer driver, etc.  Third-party updates like that do not affect the minor part of the build string.

Right now the official public release build is still 10586.  At 12:01 am Tuesday morning that rolls over to 14393.  Sometime during the day (Redmond time) MS will post build 14393 in all its editions to the Media Creation Tool site and those users who want to "jump the line" are welcome to download media and do the upgrade manually.  The Microsoft TechBench should change their .iso files that day also.  So when you are helping in the Forum you can direct users who want to get going to those websites and they can manually upgrade themselves. 

Insiders who want to get off of the preview build merry-go-round can do so as soon as the public release build is officially 14393 on Tuesday.  Right now the P.R. build is 10586 so anyone on 14393 and clicks the Stop Receiving Preview Builds button in Settings may be told they will have to reinstall Windows.  The P.R. Build number does not agree with the build they have installed!  On Tuesday when that user clicks the same button, Windows will just say "At Once, Master!" and that's it.  That's because the P.R. Build number and the installed build number agree (the minor string doesn't matter).  No reinstallation.  Perhaps a restart to reset the registry key, but that's all.
