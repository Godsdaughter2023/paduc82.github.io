sabretel say:

If you really want to keep an eye on how it is doing this at least lets you know how (and if) it is progressing.  It will not give you a % complete but with a bit of math you can figure it out.  Lame yes but at least a workaround until MS manages to move Windows 10 to bring themselves "a little" closer to the achievement they had already made in Windows 8.  Unless of course their definition of success is making what they are doing as obscure as possible.  In that case they are moving in the right direction, at least according to that playbook.

While the backup is running:

Open File Explorer

Click "This PC"

Double Click the Target Backup Drive

Right Click the folder "File History"

Click Properties

If the backup is active you should see a running tally of the size of the file.  You may have to leave and return to File Explorer to see it change but the size here compared to the size of the backup reported at

Backup Settings

More Options

should give an indication of progress.  When done, it should count up to the previously estimated backup size.

In my case the backup is about 50GB.  First pass took about an hour but after that it  was just incremental changes so it seems to run in a minute or two. 

Note: By default the system will retain backups forever but that is probably not a great idea for three reasons I can think of.

1. You tend to consume disk without limit

2. Because these are incremental backups, each one creates another potential point of failure that can invalidate the entire backup chain.

3. The more incremental backups that are performed, the slower the recovery process is likely to be.

Ideally I would set the backup frequency to a realistic number you can live with.  I chose daily since most of my key applications, Project, Excel, Word, Quickbooks, etc all have settings for auto recovery protection better suited to the unique  update frequencies inside those apps.  Most of those are set to every 10 minutes internally, which works well for the minor, system issues provided the OS and hardware don't fail completely.

For the full data backups I think the best approach may be using  two separate alternating drives, changing drives once a month.  Keeping in mind that the system is pushing the backup to the directory X:\File History where X is the backup drive, I would move the existing File History folder  on the incoming drive to a folder like July 2017 backups leaving the X:\File History folder empty, forcing the system to create a clean complete backup with no incremental file change segments.

It would be then be up to the user how many of  these monthly snapshot folders they might want to retain indefinitely.  I have found retaining the monthly snapshots for 6 months, and once a year setting aside an "annual snapshot" for 7 years seems to work fairly well.

Keep in mind that MS is not by default backing up all your programs since they will expect you to reinstall all those from your original media.  They also assume all your data is being stored where they expect you to put it.  C:\Documents, Pictures, etc.  If you keep something somewhere they are not anticipating (like in my case C:\scansoft documents\ and a Program path where I keep some handy old DOS programs and tools that work well and don't get tangled up in the registry ), don't forget to add such paths to the ones being backed up.

Hope this is of help to someone out there.

keyboard problem - Microsoft Community

ceejaydq say:

keyboard problem

Keyboard has lost English pound sign to a hash symbol and @ has changed over to " symbol on keyboard ,this happened after Microsoft 's updates.Would like to get back to original?

vijay a. verma say:
You should do following

Right click on Windows Start > Windows Powershell (Admin)
Copy and paste below and Enter in Powershell
Set-WinUserLanguageList en-GB

(If @ symbol was above 2, then en-US will be required. If @ symbol was above ' then en-GB will be required)

This will make other languages disappear. (In case, you want to add again or other languages, you can add again from language settings)

davem121 say:
Hi ceejayDQ, there is a known bug in the 1803 version of Windows 10.


Right click your Start Button, choose Powershell (admin)

Paste this command into Powershell and hit Enter:

$1 = New-WinUserLanguageList en-GB

Paste this command into Powershell and hit Enter:

Set-WinUserLanguageList $1

Close Powershell and restart your PC

That's It !
