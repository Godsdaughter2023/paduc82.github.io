doug robbins - mvp office apps & services (word) say:

I assume that oWord is declared as an object that is set to the merged document and that being the case, the update the table of contents in that document, you would just use:

oWord.TablesOfContents(1).Update

Note that the collection is TablesOfContents, not TableOfContents

However, your

With oWord

    .oWord

which is probably what is actually generating the error. if as I assume oWord is actually the merged document, instead of using a separate document with the TOC code in it, instead of the code that you now have, I would be inclined to use

With oWord
    Selection.HomeKey wdStory
    .Fields.Add Selection.Range, wdFieldEmpty, "TOC \o " & Chr(34) & "1-3" & Chr(34) & "\h \z \u", False
    .TablesOfContents(1).Update
End With

Modifying the "TOC \o " & Chr(34) & "1-3" & Chr(34) & "\h \z \u" if necessary to match what you have in the separate document.

rajesh c. say:

If the Due-Date cell is a cell of date keyed-in by the user then

in Excel the formula would be like =IF(COUNTA(A2)=0,do-ABC,do-PQR)

If Due-Date cell has a formula which gets/generates the Due-Date then

that formula should have a scenario built-in to put a "" when there is no Due-Date.

In such a situation, the excel formula would be like =IF(A2="",do-ABC,do-PQR)
