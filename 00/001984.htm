Checkbox on Sheet Checked / Unchecked - how to determine - Microsoft Community

bob98126 say:

Checkbox on Sheet Checked / Unchecked - how to determine

I have several checkbox on a excel spreadsheet.

I want to run a code based on if any of the checkbox is checked / unchecked.

But, it appears that neither the worksheetchange event nor the calculate event is triggered when a checkbox is checked / unchecked.

I know that we can set code for a checkbox to determine when it is checked / unchecked, but as there are many of them this way would not be ideal.

Any help on this will be appreciated.

bill manville say:

There are two types of checkbox you can use.

The simpler one is from the Forms controls set rather than the ActiveX set.

With the Forms controls you right-click > Assign Macro, and attach a macro that is in a standard module.  All the checkboxes can have the same macro attached and the macro can tell which box was clicked (if it is important) using Application.Caller.

You can also assign a macro to all the Forms checkboxes on the activesheet using a macro
Sub Assign()
  Dim CB As Checkbox
  For Each CB In ActiveSheet.CheckBoxes
    CB.OnAction = "CheckBoxChange"
  Next
End Sub

Sub CheckBoxChange()
' example
  Dim bVal As Boolean
  bVal =  ActiveSheet.CheckBoxes(Application.Caller) = xlOn
  MsgBox "You just changed checkbox " & Application.Caller & " to " & bVal
End Sub

--
If for some reason you need to use the ActiveX type then each one needs its own event procedure, though they can each call a common  procedure.

--

If the checkboxes are all linked to cells and you create a formula that references those cells then the Calculate event will be triggered when you change a checkbox, provided that calculation mode is automatic.

First of all I have a TyTN II and VAIO SZ460N running Vista Business with Bluetooth.

1.- On Control Panel/Device Manager...  Right click and Uninstall all your BT devices.

3.- Restart your PC

4.- Download from MS web the IntelliPoint 6.2 (32bit) for Windows Vista Drivers and install them..

( I know you don't have an intellipoint mouse, but this will install MS bluetooth drivers!!!)

5.- After install, do not configure any mouse!!

6.- Turn On your BT device

7.- Your Vista will tell you to automatic look for a driver online... SAY NO!!!!

8.- Go to Control Panel/Device Manager ...

You will see your BT device with and error (because it did not have a driver)

9.- Right click and install a driver (ALWAYS DO IT MANUALLY, NOT AUTO)

10.- click on BROWSE MY COMPUTER FOR A DRIVER

11.-  click on LET ME PICK FROM...

12.- choose GENERIC ADAPTER / GENERIC BT ADAPTER  and NEXT

Accept the warning message!!

 

13.- Now all your Toshiba BT Stack will dissapear, and on your Control panel you will see the BT Icon...

14.- Open it and check if you coud see the adapters on Hardware.

15.- Go to Devices and add your Smartphone/PDA...

16.- Go to your Smartphone/PDA and pair it to your laptop... you will have to see ActiveSync service...

from now you should not have any problem if you had lost a lot of time (like me) trying to connect in the past....

 

Well... enjoy your PDA and let me know that this helps you...

raoul_duke---01 say:

Hi Guys,

Thanks for your post.

When is comes to playing MKV container it really depends on how the actual file is encoded and what codec is being used. In almost all MKV files the codec H264 is being used as this codec is used for high definition content. There are many variables which could affect the playback such as audio and video bitrate, audio stream, resolution, frames per second and the AVC performance level. These settings are done when the file is encoded so if the file isn't playing some conversion maybe required. I haven't got any MKV files to test on hand the moment but will try some different ones once I get some. You can use a tool called Mediainfo which will give a complete breakdown on the encoding settings. When I get an MKV file to test I shall put some of my findings on this thread.

Most likely when an MKV doesn't work its normally down to the performance level profile used. The performance level profile for mobile devices should not be more than High@4.0, anything higher than you may experience choppy playback. Please be aware smooth playback also depends on the hardware being used as a mobile device may support the H264 codec but not at such high encoding settings and may need to be converted to use a lower bitrate, performance level etc. Also most audio used in high defination MKV files is DTS which most mobile devices do not support. If you do manage to get this type of file playing you wont hear no sound. This is needs to be converted AC3 normally 640kbs with a bit depth of 16bit and a sampling rate of 48.00 khz but this can be lower if you wish as it will reduce the size of the file and the above settings is normally for 6 channel surround so its not really necessary on mobile device but the above settings I have tested and they do work on the higher end mobile devices.

Please also be aware that the file size of the file should be no more than 4GB in size due to the file structure used on the 808.

 

For more reading on this please refer to the following link:

 

http://www.matroska.org/technical/guides/faq/index.html
