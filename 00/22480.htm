stefan blom say:
What kind of entries are these? Unformatted (= text only) AutoCorrect entries are saved in separate files with the acl filename extension, and these files are language-specific. In other words, if you intentionally, or unintentionally, have more than one language applied to text, that explains why you can't see all entries all the time, because an AutoCorrect entry available when a text range is formatted in one language may not be available when you move the insertion point into another text range.

As far as the AutoText entries are concerned, these are stored in the Normal template (or in the Building Blocks.dotx file if you are using Word 2007 or later). If you have deleted or renamed the Normal template recently, that may explain why entries were lost.

Just to be clear, have you lost AutoCorrect entries or AutoText entries?

stephen boots say:

Unfortunately, you will not be able to get your money back since you followed the scam's instructions to provide payment in a format that you are unable to dispute (like a credit card.)

 

As to how to stop it from happening again --

the following recommendations will assist in protecting the PC from infection:

-Make sure that the Windows Firewall is enabled.

-Make sure that all important/critical updates, including service packs for the operating system and programs are installed from Microsoft Update (Windows Update).

-Make sure Internet Explorer is at version 8, 9 or 10 and updated with all patches.

-In Internet Explorer 8, 9 or 10, use the SmartScreen Filter.

-Make sure that IE Internet Security settings are at least set to medium-high (default).

-Enable the pop-up blocker in IE.

-On Vista and Windows 7 make sure that User Account Control (UAC) ON  and not running with elevated privileges.

-Make sure that Windows Automatic Updates are set to at least notify, but the preferred setting is to download and install automatically. If you update manually, be sure to update as soon as possible after being notified of available updates.

-Make sure that installed applications, especially Adobe Acrobat, Adobe Flash, and Java are at their latest versions. Many vendors are regularly updating and patching for security holes.

See: https://blogs.technet.com/b/security/archive/2012/07/19/the-rise-of-the-black-hole-exploit-kit-the-importance-of-keeping-all-software-up-to-date.aspx?Redirected=true

-Never click through links from unknown sources and use caution even if they are from a "trusted" source.

-Never open unsolicited email attachments.

-Practice safe web browsing.

Unfortunately, I don't think there's any "royal road" to this, but here's how I'd do it:

  1. Start by adding the Insert Caption and Cross-reference commands to your Quick Access Toolbar. Do this by right-clicking on each button on the Ribbon and choosing Add to Quick Access Toolbar. That makes those commands available with one click regardless of which Ribbon tab is displayed.
  2. Set up the Figure (or Fig.) caption label, including the "chapter number."
  3. Select the text of the first caption (excluding the caption label) and cut (Ctrl+X).
  4. Select what's left of the caption, click your Insert Caption button, paste the text you cut, and click OK. The new caption should replace the old one.
  5. Repeat for each caption.
  6. Now you'll need to find the cross-references. The easiest way to do that is to display field codes (Alt+F9) and then search for ^d REF or ^19 REF. Each time you find one, recreate it using  your Cross-reference button.

It may well be that someone here can come up with a macro that would do this automatically, but I have no VBA expertise.
