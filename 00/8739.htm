When Word works okay in Safe Mode, this is an indication that there is a problem with one or more of the following:

  • Word settings in the Windows Registry
  • The Normal template
  • The printer driver
  • Add-ins

For a problem of this type, the most likely culprit is a poorly written add-in. Go to File | Options | Add-ins and see what add-ins are listed. Hi,

Try this ARRAY formula which returns the mode for text values and ignores blank cells.

=INDEX(A1:A100,MODE(IF(A1:A100<>"",MATCH(A1:A100,A1:A100,0))))

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

dianeoforegon say: Are you trying to sync to iCloud contacts? If yes, Outlook does not support sync to any CalDAV Address Book like iCloud and google contacts.

If this is regular sync, try this...

  1. Turn off Sync Services in this Identity. (Only one Outlook Identity can be set to sync at a time)
  2. Export your contacts as .olm file under File > Export.
  3. Create a new Identity. No need to do setup accounts for this test.
  4. Import your contact and move them into the local contacts.
  5. Turn on Sync Services. this could take a few minutes to sync.

Does the new Identity sync to the Address Book? If yes, you can try rebuilding the old Identity but you might have better results if you move to the new Identity. See Rebuild basics below.

If no.....

Try deleting these files in your User’s Library/Preferences folder:

com.microsoft.Outlook.SyncServicesPreferences.plist
com.microsoft.SyncservicesAgent.plist

Also in the User's Library/Preferences/Microsoft/Office 2011 folder delete OfficeSync Prefs

You'll need to restart after deleting prefs.


  • Do Not Rebuild Identities with Exchange and IMAP accounts. Delete the accounts (this deletes the cache files in Outlook). Do the rebuild then add back accounts.
  • Make sure you have plenty of free space on your drive. Make sure that you have at least three times as much hard disk space available as the current size of your Outlook identity PLUS 10% free for virtual memory.

I had the same issue like you. But the following steps (which was quite some effort) helped.
I assume that something went wrong with the (data) migration from the old to new outlook.com platform.

3) I changed all contacts, for which no first or last name was filled out (e.g. I took the company name "Restaurant Blah" and copied it to last name)
4) In case there was a link to another contact, I removed it (e.g. it had linked "John Doe" to "Mark Something" for whatever reason)
5) I deleted all duplicated

This took reaaaly some time. However, it was a good way to anyhow clean up the contacts.
After I cleaned up my contact list on outlook.com (there was not a single contact without first or last name), I did the following steps on my Android device:
1) Removed the Microsoft exchange account (just turning off sync does not help)
2) Re-added my account

Viola! This has been working for some time now. No "unknown name" contacts anymore! Let me know if it helped (and mark this as an answer, please, if so).
Maybe there is also a smarter way of doing it.
