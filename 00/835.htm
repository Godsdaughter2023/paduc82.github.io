hansv mvp say:

An apostrophe before a cell value forces Excel to interpret the value as text. This is mostly useful for values that look like a number or date.

For example, Dec-1 would ordinarily be converted to a date in December. If you want the literal text Dec-1, enter it as 'Dec-1.

In formulas that refer to another sheet whose name contains spaces, the sheet name is enclosed in apostrophes. For example:   ='Other Sheet'!A1

Equation editor

shashirekha say:

You cannot change the font in an equation editor. To accompany the new math feature, Microsoft created not only a new font, but a new class of Open Type math fonts. Only math fonts can be used inside equations in Word. Right now, the only font with these tables is Cambria Math, but very shortly the typography team will publish a spec on how to create new math fonts.

There is a workaround, however. Remember that an Equation Editor object is a graphic. Specifically, it is a metafile graphic. This means that you can copy it to a graphics program that allows you to modify vector artwork (such as CorelDraw, Illustrator, or Paint Shop Pro), and make changes to individual parts of the graphic. Once changed, you can paste the graphic back into Word.

ORA-01847 sounds like an error from Oracle, not from Access. Is the backend table in Oracle? And are you specifying this as a PassThrough query? If so it must follow Oracle's standards, not Access's.

I'd expect that (if you're doing this in Access and not as a passthrough) that a criterion of

DateInserted >= DateAdd("h", -18, Date()) AND DateInserted < DateAdd("h", 6, Date())

should work.

qa guy commelec say:

Try this --

SELECT EMS_VW_ODBC_DAILYVOUCHER.PTNID, EMS_VW_ODBC_DAILYVOUCHER.DTTYPID, Sum(EMS_VW_ODBC_DAILYVOUCHER.DEBITPOINT) AS [Points Redeemed1], Sum(IIf([dttypid]=81002,[MONEYVALUE]*-1,[MONEYVALUE])) AS [Points FSP Redeemed1], EMS_VW_ODBC_DAILYVOUCHER.DATEINSERTED, [dateinserted]-0.25 AS Rounded, DateSerial(Year([dateinserted]-0.25),Month([dateinserted]-0.25),Day([dateinserted]-0.25)) AS [Gaming Date]

FROM EMS_VW_ODBC_DAILYVOUCHER

WHERE (((EMS_VW_ODBC_DAILYVOUCHER.DATEINSERTED) Between Date()-0.75 And Date()+#12/30/1899 0:25:0#) AND ((EMS_VW_ODBC_DAILYVOUCHER.COMPID)=15))

GROUP BY EMS_VW_ODBC_DAILYVOUCHER.PTNID, EMS_VW_ODBC_DAILYVOUCHER.DTTYPID, EMS_VW_ODBC_DAILYVOUCHER.DATEINSERTED

ORDER BY EMS_VW_ODBC_DAILYVOUCHER.PTNID, EMS_VW_ODBC_DAILYVOUCHER.DATEINSERTED DESC;

 

If you still have the problem try it without the WHERE statement to test it.
