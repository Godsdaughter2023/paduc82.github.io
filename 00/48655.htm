BCCode: D1     0x000000D1

Cause

A driver tried to access an address that is pageable (or that is completely invalid) while the IRQL was too high.

This bug check is usually caused by drivers that have used improper addresses.

If the first parameter has the same value as the fourth parameter, and the third parameter indicates an execute operation, this bug check was likely caused by a driver that was trying to execute code when the code itself was paged out. Possible causes for the page fault include the following:

  • The function was marked as pageable and was running at an elevated IRQL (which includes obtaining a lock).
  • The function call was made to a function in another driver, and that driver was unloaded.
  • The function was called by using a function pointer that was an invalid pointer.

Is it possible to change the default download location for windows - Microsoft Community

papajer say:

Is it possible to change the default download location for windows updates?

I have a space problem on C drive and need to have windows updates download to another location is this doable?
mowgreen say: The updates are automatically downloaded to the WINDOWS\Software Distribution\Download subfolder.
They are then unpacked to the drive which has the most free space available and installed from there.
The contents of the Download subfolder are periodically flushed.

There's no way to redirect where the updates are downloaded to due to Security concerns.
Suggest you run Disk cleanup to free up HD space.

If the installed OS is Windows XP , you can also delete the older System restore points
by clicking the More Options tab, and then click the Clean up button under System Restore .
You can also resize the disk space that System Restore uses by opening System in Control Panel.
Click the System Restore tab.
Click on the drive that SR is monitoring, then click the Settings button.
Under Disk space usage , move the slider towards Min to reduce the amount of restore points retained.

In Windows Vista/ Windows 7 , no such option exists in Disk cleanup .
In Vista the only recourse for System Restore is to Disable it by unchecking the box next to the drive it monitors, click
Apply, then OK. That will flush all restore points.
Then reenable it and Create a new restore point.

In Windows 7 you can open System in the Control Panel.
Then click on the System Protection link in the left frame.
Click on the drive SR is monitoring and then click the Configure button.
Either move the slider under Disk Space Usage down to delete older restore points or click the Delete button to remove all restore points. Then create a new one by clicking the System Restore button in System Properties .
