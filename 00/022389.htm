stephen boots say:
It is actually cleaning out *old* information from the database, purging old records. If nothing has been detected, then you have no history and nothing is being deleted or purged.
There is no facility to change the retention of History, nor is there any way to see a record of scans that didn't detect anything unless you review the Event Log.

don cuthbert say:

To boot Windows 7 you need either UEFI with CSM compatibility mode enabled, installed to a GPT-initialized disk, or UEFI disabled,  and Windows 7 installed to an MBR-initialized disk.

Windows 10 is designed to have UEFI, GPT disk and Secure Boot enabled, but it can work with Legacy BIOS and an MBR disk.

If I want to boot Windows 7, I just disable Secure boot and have Windows 7 on a small partition (or other disk would work). On my system this enables CSM and I can boot Windows 7. But I keep all of my disks as GPT disks, not MBR.

I find it easiest to run one System partition that has the multi-boot menu appear at power-up. If I want Windows 7, it appears in that boot menu.

An easy fix might be to boot to your Windows 10 install media. Once Windows setup loads to the first screen, use SHIFT-F10 to get a Command Prompt. Run diskpart and use list volume to see what drive letters are assigned in this mode, as they may vary from what you would expect. Type exit to leave diskpart.

If Windows 7 is on drive D: try the command:

bcdboot D:\Windows

Then if Windows 10 is on C:

bcdboot C:\Windows

That will create a boot menu with Windows 10 as default and Windows 7 as optional OS (because you ran the bcdboot command for Windows 10 last). Change the drive letters to whatever they are in this mode. It doesn't matter what the drive letters are, as long as you figure out what letters to use with the bcdboot commands. Any time that you boot to the boot menu, you can change the timeout from 30 seconds to whatever you want or change the default OS.
