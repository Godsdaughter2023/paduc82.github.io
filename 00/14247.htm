jay freedman say:

This question can't be answered for certain unless you can tell us what settings in the original document cause the header on page 1 to be different from the header on page 2.

  • There could be one section (no section break) with the Different First Page option turned on in the Header & Footer Tools ribbon.
  • Or there could be two sections, with a section break between pages 1 and 2, with or without the Different First Page option.

Since the first one is more likely, here's some code to try. You'll have to supply the path of the new image's file.

Sub ChangeHeaders()
    Dim rg As Range
    Dim pic As Shape
    Dim lngTop As Long
    Dim lngLeft As Long
    Dim lngWidth As Long
    
    With ActiveDocument.Sections(1)
        .PageSetup.DifferentFirstPageHeaderFooter = False
        With .Headers(wdHeaderFooterPrimary).Shapes(1)
            Set rg = .Anchor
            lngTop = .Top
            lngLeft = .Left
            lngWidth = .Width
            .Delete
        End With
        
        Set pic = .Headers(wdHeaderFooterPrimary).Shapes.AddPicture( _
            FileName:="D:\Pictures\XYZ.jpg", _
            Left:=lngLeft, Top:=lngTop, Anchor:=rg)
        With pic
            .LockAspectRatio = msoTrue
            .Width = lngWidth
        End With
    End With
End Sub

Need an Excel Formula to Auto Insert Pay Roll Number - Microsoft Community

mlivingston say:

Need an Excel Formula to Auto Insert Pay Roll Number

I need a formula on employees timesheets that will automatically insert the pay roll number.  We have a total of 26 pay periods per year.  The next pay period will be number five (5).  I will need to do this manually.  Therefore, please explain how to get it started and how to reset it annually.

 

Thanks.

jllatham say:

Remember that any formula involving something like this solution is likely to use such as NOW() or TODAY() is going to update to the current date any time you look at it.

 

For example, lets say your first pay period ended Saturday, January 5, 2013.  And so you enter a formula that returns "1" as pay period #1.  But later you go back and look at that same worksheet - the pay period number is going to reflect the current pay period number based on the current date - so if you go back and look at it in late December, it may read as 25 or 26.

 

But with that in mind, explain to us a little about your pay periods: You say there are 26, and that implies a bi-weekly payroll (not twice monthly).  So we need to know a couple of things:

When did pay period #1 end, and

what days do your pay periods start and end on (as starts on Sunday, ends 2 Saturdays later).

 

Another thing that would probably help to know is does the Period Ending or Period Starting date appear on the timesheet?  If it does, which is it and what cell is it in?

jllatham say:

Let us assume that the Pay Period Ending date is in cell B2 on your timesheets.

 

For this year, things work out rather nicely, or can be made to work out easily enough.

 

Let's say that the pay period ends on 2/9/2013 (Saturday), then this formula will return an integer number:

=Weeknum(B2)/2

will tell you that it is the end of pay period #4.  You may need to adjust that by 1 by either adding or subtracting 1 from the result like this:

=(Weeknum(B2)/2)+1

would give a result of pay period #5.

 

Now, if the pay period ended on 2/16/2013, the result of Weeknum(B2)/2 would be 3.5 which could be rounded up to the next whole number using

=ROUND(WEEKNUM(B2)/2,0)

so 3.5 becomes pay period #4, and again you can adjust the result by plus or minus 1 as needed:

=(ROUND(WEEKNUM(B2)/2,0))+1  would show as pay period #5.

 

You can set this same thing up each year initially and determine by experiment and observation if you are going to need to add or subtract 1 from the result to get the proper pay period number displayed.
