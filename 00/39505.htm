There are a couple of things you may try one or both may work.  Go to command prompt type in ipconfig/release it will load all the information and spit out a summary of your network connection IP address, subnet mask, default gateway, etc..., after this (this releases all IP addresses connected to your "bridged" network).  After you type this into command prompt type ipconfig/renew (this assigns a new IP address to your XBOX and your laptop, each address to each system should be similar but different).  If that doesn't work go into your XBOX network configuration settinings, this gives you the option to manually enter your IP address, subnet mask, default gateway, and DNS servers.  By default these settings are automatically set in your XBOX to "detect automatic" or "automatically assign" something along those lines.  Go to manually enter settings and press "A" on your controller, go back to your laptop and go to command prompt.  Type in ipconfig/all the key item you are looking for is the "Default Gateway" IP address.  This IP will also look similar but will be different to that of your laptops, every system on your network needs its own unique IP address, the default gateway IP address is the address that connects all systems to the internet and may look like this 192.168.1.1  Your computers physical IP address may look like this 192.168.1.100 or 192.168.1.2.  In the manual network configuration settings of your XBOX go to IP address, see what your xbox is currently using as an IP address, if it looked something like 192.168.0.2 then use 192.168.0.200 if its 192.168.0.200 use something like 192.168.0.240 (only the last set of numbers at the end should change but make sure the IP address you enter IS NOT ALREADY ASSIGNED TO ANOTHER SYSTEM ON YOUR NETWORK, the last set of numbers can range from 2-254 so pick any number).  Click A and save it on your controller.  Next go to your Subnet Mask settings.  Almost every home or small business network uses a default subnet mask and it looks like this 255.255.255.0 enter this and click A and save it.  Next go to your Gateway settings.  Enter the IP Address that was pulled up in ipconfig under "Default Gateway" This is your router or internet connections source of the internet and giving the XBOX this address will obviously give it the address to the internet on your network.  Next test your XBOX Live connection, all addresses should be confirmed.  You may have to enter your DNS servers address in the DNS settings section right under the IP configuration option.  This is tricky, I've never had to do it, you can try entering the DNS Server information that pops up on the command prompt under ipconfig/all or you may have to call your ISP and request these two addresses, they may ask why and just tell them your XBOX is requesting it to connect to the internet.  Hope this helps!  If all works well and is confirmed your XBOX see's the internet.

The primary causes of this error are virus/malware, drivers, memory, and disk corruption. And there
can even be combinations of those so a thorough check of all is required. Added possible causes
are hardware errors such as damaged motherboard or other hardware.

BCCode: 7A    0x0000007A

Cause

Frequently, you can determine the cause of the KERNEL_DATA_INPAGE_ERROR bug check from the error status (Parameter 2). Some common status codes include the following:

  • 0xC000009A, or STATUS_INSUFFICIENT_RESOURCES, indicates a lack of nonpaged pool resources.

  • 0xC000009C, or STATUS_DEVICE_DATA_ERROR, typically indicates bad blocks (sectors) on the hard disk.

  • 0xC000009D, or STATUS_DEVICE_NOT_CONNECTED, indicates defective or loose cabling, termination, or that the controller does not see the hard disk.

  • 0xC000016A, or STATUS_DISK_OPERATION_FAILED, indicates bad blocks (sectors) on the hard disk.

  • 0xC0000185, or STATUS_IO_DEVICE_ERROR, indicates improper termination or defective cabling on SCSI devices or that two devices are trying to use the same IRQ.

These status codes are the most common ones that have specific causes. For more information about other possible status codes that can be returned, see theNtstatus.h file in the Microsoft Windows Driver Kit (WDK).

Another common cause of this error message is defective hardware or failing RAM.

A virus infection can also cause this bug check.


Resolution

Resolving a bad block problem: An I/O status code of 0xC000009C or 0xC000016A typically indicates that the data could not be read from the disk because of a bad block (sector). If you can restart the computer after the error, Autochk runs automatically and attempts to map the bad sector to prevent it from being used anymore.

If Autochk does not scan the hard disk for errors, you can manually start the disk scanner. RunChkdsk /f /r on the system partition. You must restart the computer before the disk scan begins. If you cannot start the computer because of the error, use the Recovery Console and runChkdsk /r.

Warning   If your system partition is formatted with the FAT file system, the long file names that the Windows operating system uses might be damaged if you use Scandisk or another MS-DOS-based hard disk tool to verify the integrity of your hard disk from MS-DOS. Always use the version of Chkdsk that matches your version of Windows.

Resolving a defective hardware problem: If the I/O status is C0000185 and the paging file is on an SCSI disk, check the disk cabling and SCSI termination for problems.

Resolving a failing RAM problem: Run the hardware diagnostics that the system manufacturer supplies, especially the memory scanner. For more information about these procedures, see the owner's manual for your computer.

Check that all the adapter cards in the computer are properly seated. Use an ink eraser or an electrical contact treatment, available at electronics supply stores, to ensure adapter card contacts are clean.

Check the System Log in Event Viewer for additional error messages that might help identify the device that is causing the error. You can also disable memory caching of the BIOS to try to resolve this error.

Make sure that the latest Windows Service Pack is installed.

If the preceding steps do not resolve the error, take the system motherboard to a repair facility for diagnostic testing. A crack, a scratched trace, or a defective component on the motherboard can cause this error.

Resolving a virus infection: Check your computer for viruses by using any up-to-date, commercial virus scanning software that examines the Master Boot Record of the hard disk. All Windows file systems can be infected by viruses.

Power on and off your computer three times

Fix 1 - Run Automatic Repair

1) In the menu, choose Troubleshoot > Advanced Options > Startup Repair.
2) Next, select the target operating system to fix. Windows will initiate the repair and then restart.
Fix 2 - Use SFC /SCANNOW to Repair System Files

1) In the menu, click Troubleshoot > Advanced Options > Command Prompt.
2) In the command line, type sfc /scannow and press Enter button.
3) Restart your computer when the process is finished.
Fix 3 - Run CHKDSK to Repair Boot Drive

1) Open Command Prompt using the steps in fix 2, type chkdsk /r c: and press Enter.
2) Type Y and hit Enter when you are prompted to schedule the check after a reboot.
3) Reboot and let the check proceed. It might take a while.
Fix 4 - Fix MBR

Open Command Prompt, type the following commands and press Enter after each move.

bootrec /rebuildbcd
bootrec /fixmbr
bootrec /fixboot
exit
