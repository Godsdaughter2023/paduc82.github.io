ken sheridan say: In doing this sort of thing I would recommend a 'no-loss' approach by first adding a new column of date/time data type to the table, and then executing an update query as follows:

UPDATE [YourTable]
SET [NewDateColumn] = CDATE([OldDateColumn])
WHERE [OldDateColumn] IS NOT NULL;

You can then check for any which have not updated with:

SELECT *
FROM [YourTable]
WHERE [NewDateColumn] IS NULL
AND [OldDateColumn] IS NOT NULL;

Once you've identified and, where appropriate, corrected the rows returned by the above query you can delete the original column (or rename it so that you keep the old data as a fallback) and rename the new column back to the name of the original column.

Hi ME,

The protection level is set to Low to catch only the most obvious spam emails. Low level is recommended if you receive quite few unsolicited emails.

Setting protection level to High is often considered the best practice to obtain the maximum protection. However, along with spam it may also misidentify legitimate messages and move them to Junk. So, if you opt for the High level, don't forget to periodically review your Junk mail folder.

Outlook Junk Email Filter protects against much of the spam sent to you, no filter is smart enough to catch every unsolicited email, even if set to the high level.
