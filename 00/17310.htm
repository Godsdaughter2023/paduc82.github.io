jim gordon mvp say:

Hi

Because you are using multiple cores, it sounds like you have a workbook that is either very large, has intensive calculations, or both. Calculation occurs automatically whenever a change is made to your worksheet.

There are various strategies you can use to control when Excel calculates. Excel preferences offers these options:

You could set Calculation to Manual, then press F9 whenever you want to tell Excel to calculate.

Keep in mind the scope when setting calculation to Manual. The setting affects all open workbooks and sticks when you Quit Excel. You have to remember to set calculation to Automatic if you want Excel to behave as usual with other workbooks.

A way around this problem is to use an AutoOpen() macro to turn calculation off when you open this particular workbook, and use AutoClose() to turn calculation back on when you close the workbook.

There's a good discussion on how to go about this here:

https://stackoverflow.com/questions/44782390/vba-turn-on-auto-calculation-after-workbook-is-closed

teylyn say:

Well, the code is a function and it will return the string of slicers to a cell. You enter the function into the cell and specify with the parameters which slicer you want to monitor.

A function should not be used to modify any other cells, so splitting the returned value into cells in column C is not what the function should do.

But you could use another VBA macro that splits the value that the function returns. If you have the original GetSelectedSlicerItems function in B1, then you could use code like this

Sub SplitSplicerResult()
   Dim ws As Worksheet
   Dim strTest As String
   Dim strArray() As String
   Dim intCount As Integer

   Set ws = ThisWorkbook.Worksheets("Sheet1")

   strTest = ws.Range("B1").Value
   strArray = Split(strTest, ",")
   ws.Range("C1", ws.Range("C1").End(xlDown)).ClearContents
   For intCount = LBound(strArray) To UBound(strArray)
      ws.Range("C" & intCount + 1) = Trim(strArray(intCount))
   Next
End Sub

Instead of passing cell B1 into strTest, you could use

strTest = GetSelectedSlicerItems(SlicerName)

Everytime you write to the workbook, you could be firing events or triggering a re-calc.  So try wrapping your code like so:

Sub SpeedCode()
    Dim xlCalc As XlCalculation
    With Application
        .ScreenUpdating = False
        xlCalc = .Calculation
        .Calculation = xlCalculationManual
        .EnableEvents = False
        .DisplayAlerts = False
    End With

    'Code to do your dashboard tasks - like multiple value entries

    With Application
        .ScreenUpdating = True
        .EnableEvents = True
        .DisplayAlerts = True
        .Calculation = xlCalc
    End With
