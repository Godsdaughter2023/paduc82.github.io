john - project say:

Peter,

Oops, must be a typo. The shift times you specify only give 9.5 hours shifts, not 10. But you can tweak the numbers to suit your needs.

Depending on your version of Project (ribbon/menu nomenclature may be different) here are the steps

1. Go to Project > Properties group > Change Working Time

2. Hit the "Create New Calendar" button

3. Since you are still using a 5 day workweek I'd just make a copy of the Standard calendar and give it an appropriate name (e.g. 1st Shift 10h)

4. Hit the "Work Weeks" tab in the lower part of the window and then hit "Details"

5. Select the option to "Set days to these specific Times"

6. Select Monday, and then set the hours as shown here

7. Repeat step 6 for for Tuesday through Friday

8. Then hit "OK" and "OK"

That gives you the first shift. Now repeat those steps for a second shift (e.g. "2nd Shift 10h). Because of the cross-over at midnight, you'll need to set the hours as shown below

Now that you have the shift calendars set you need to apply those to your plan

9. Go to Project > Properties group > Project Information

10. Set the start date for your plan to whatever date you want but also include the time (e.g. 11/10/16 6:00AM)

11. There are various ways to handle this but I suggest you set the Project Calendar to the built-in "24 hours" calendar. Then hit "OK"

12. Go to File > Options > Schedule group

13. Set the Default Start time to 6:00 AM and the Default End time to 2:00 AM

14. Set the "Hours per day" to 20. That will give you a correct reading in the Duration field for each "day".

15. Depending on whether your shifts are task based or resource based (I would expect resource based), on the Resource Sheet, set the Base Calendar for each resource according to the shift they will be working.

16. Finally, you might want to go to the Gantt Chart view timescale and double click to bring up the Timescale window. Hit the "non-working time" tab and select the "24 hours" calendar.

That should do it. Tasks will follow the Project calendar which allows working the two shifts and as long as tasks do not "ignore resource calendars", work will follow the assigned resource calendars.

Hope this helps.

John

john - project say:
