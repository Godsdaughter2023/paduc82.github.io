3) Now do Save As into Plain Text format. After clicking the Save button, you'll see a dialog where you need to enable "Insert line breaks." This option will add a ¶ mark at the end of every line.

jeeped say:
Neither of these dates is correct. Well, the second one might be correct, but it is more likely that the dd and the mm are swapped.

Excel takes the dd/mm/yy vs mm/dd/yy format from the system's regional settings. It would appear that your system is set for mm/dd/yy and you pasted in a number of dates from an external (web...?) source that come in the dd/mm/yy format. This has reduced the first date to a textual representation of a date and made the second a true date of 04-Jan-2013 when it should be 01-Apr-2013.

You could try changing your system settings in the Control Panel then re-pasting the dates or pasting them into a TXT file and importing them into Excel with Data, Get External Data, From Text where you can specify the dd/mm/yy format.

BTW, 39816 is actually 03-Jan-2009 so I'm guessing that you made some substitution errors in the sample data you typed...?

ashish mathur say:

Hi,

 

Try this process on a copy of your actual dataset

 

1. Select the data range (including the header row) and sort on the Date column in ascending order

2. Select the data range (including the header row) and in the auto filter drop down of the date column, uncheck the Years so that only individual dates are visible

3. Select the resultant data entries in the date column and go to Data > Text to columns > Delimited > Next > Next > Date > DMY > Finish

4. Clear the Filter

 

The process above should convert the "text Dates" to "actual dates".  The filter drop down for the date column should now only show all dates grouped by years.
