Hi Kay,

When you logon to the VLSC, you should choose to download the 'Office 2016 for Mac Standard' package. You'll want to set the operating system selector to MACOS so that it exposes an .ISO package download, which will be around 1.6GB in size. Once downloaded, you can double-click the ISO to mount it, and from there, you'll find two .pkg files inside. The larger PKG is the full Office install. The smaller PKG (which is known as the VL Serializer) can take an existing Office 2016 installation and convert it to Volume License. The latter approach is useful if you're deploying many machines, as you can download the SKUless Office install from https://go.microsoft.com/fwlink/?linkid=525133 (which always points to the very latest bits), and simply run the VL Serializer to make it VL.

ossiemac say:

If copying the formula down a column then the Table Array must be in absolute format (with the $ signs) as follows otherwise as the formula is copied down then the Table Array changes the range.

=VLOOKUP(CONCATENATE(B4,C4),Sheet1!$A$4:$D$2923,4,FALSE)

Further to above ensure that when you concatenate the values for the Table Array that you do not have any leading or trailing spaces in the concatenated data. To check this, select some of the cells and Copy -> Paste Special -> Values. Then select one of the concatenated values and check in the formula bar that there are no leading or trailing spaces. If you do have leading or trailing spaces then add a Trim function to the concatenation formula like the following.

TRIM(CONCATENATE(B4,C4))

Alternative concatenation formula including the Trim is as follows

TRIM(B4&C4)

The Trim function may be required in both the Table Array formulas and the Vlookup formulas. Note that you can use the Trim function irrespective of whether there are leading or trailing spaces.

If the data is downloaded from another source, there may also be other leading or trailing non printing characters and the Trim function does not remove them in which case you can try the Clean function like the following.

CLEAN(B4&C4)

Even the Clean function will sometimes not remove some non printable characters and it becomes a little more complex to remove them. See the following link for more information on this.

https://support.office.com/en-us/article/top-ten-ways-to-clean-your-data-2844b620-677c-47a7-ac3e-c2e157d1db19?ui=en-US&rs=en-AU&ad=AU
