rob koch say:

 There is no green check mark on the Device Performance & Health tile in the defender.

Can anyone provide me some solutions for this problem?

How recently did this Windows 10 operating system go into service?

If it's only been around a few days or weeks, there may not have been enough time for the Performance & Health system to have collected the information it requires to display the green indicators.  That because this isn't an indication of "protection" like most of the others, but rather that the automated tests that a system performs over time are displaying appropriate data that it's operating properly.

Doing things like resting the operating system or other maintenance tasks that clear these items from the logs in which they are stored will do nothing to fix this, since there's no indication that something's actually wrong.

If the system is new, just give it a few days or weeks and see whether the green check mark appears.  If there's not an error, yellow triangle with an exclamation mark or other negative indicator displayed, nothing's actually wrong, so just leave it alone to allow the system to catch up.

rohn007 say:

Sounds like you have a couple of problems:

First, trying to work on files on a CD is a problem for Word. When you open a file in Word, Word creates a "temporary" in the same directory by making a copy and changing its file name to start with ~$.... If Word cannot create this temp file, it gets 'cranky'. Simply copy the file from the CD to your HD (as Gordon implied in his answer).  The same problem used to happen on "Floppy disks" because they had such limited storage, and sometimes happens on USB drives that are full.

Second, when you get squares, questionmarks  and other nonsense charcters, the file you are opening is not the correct file type. So are you sure you were trying to open DOC or DOCX files, and not JPG or something else.

Third, you said this was a "backup". Is it possible that the files were placed on the CD using some type of Backup application rather than simply copied and "burned" to the CD? If they were "backed up" using a backup application, the files would still have their original extensions, DOC JPG etc, but not be in the correct format to be read by their original applications. You would have to use the backup application to copy them back to the HD for them to be readable.

When you get the unreadable text, do you first get any sort of warning message when opening the file?

If this proposed solution has resolved your issue(s), please mark it as  Propose as Answer  to let others know the solution that worked for you
