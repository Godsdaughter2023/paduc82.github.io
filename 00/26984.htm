dhookom say:

I would first normalize the data with a union query like the following which assumes the software fields are actually text fields containing Yes or No:

SELECT [NameField], "Excel" as SW
FROM tblSpreadsheet
WHERE Excel = "YES"
UNION ALL
SELECT [NameField], "Word"
FROM tblSpreadsheet
WHERE Word = "YES"
UNION ALL
SELECT [NameField], "PowerPoint"
FROM tblSpreadsheet
WHERE PowerPoint = "YES"
UNION ALL
SELECT [NameField], "Outlook"
FROM tblSpreadsheet
WHERE Outlook = "YES";

You can then create a report based on the union query or a main report based on tblSpreadsheet and a multicolumn subreport based on the union query.

scottgem say:

Well first you have a fundamental flaw in your table design. That is why you can't do what you want. 

You should have, at least, three tables here; An employees table with An EmployeeID, name and whatever other info you need about an employee. A Software table With a SoftwareID listing the names of all software someone might have access to. The third table is called a junction table. You see what you have is a many to many relationship. One employee can have access to multiple software titles and one software can be used by multiple employees. To model such a relationship you would use a junction table that has foreign keys for for the EmployeeID and SoftwareID. So this table would have one record for each combination of employee and software. 

From there its easy to create a query based on the Junction table which would just list the software accessible to an employee. You could create a query that produces one row for each employee with a comma separated list of software titles. or you could create a report with columns that would list the software  in columns. Something like this:

John

         Excel      Powerpoint

Your structure is fine for a spreadsheet, but Access is not a spreadsheet its a fully relational database. With your structure, you can only create a report that has columns for each field in the table. You can suppress displaying a value in the column if it is No, but you can't eliminate the column.
