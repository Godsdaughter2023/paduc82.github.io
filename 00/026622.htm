I checked the formula, and it had been changed to:

hansv mvp say:

I suspect that K18 was the active cell in the selection when you created the rule. When you edit the rule, the formula is displayed for the top left cell of the range it applies to (G6).

If K18 is the active cell, you should use =NOT(ISNUMBER(K18)). Alternatively, make sure that G6 is the active cell within the selection when you create the rule.

jeeped say:
... I checked the formula, and it had been changed to:
=NOT(ISNUMBER(C1048570))

Why does this happen, and how do I get it to stop?

This is occurring due the way your original selection is made. It looks like you are selecting G6:K18 by starting at K18 and dragging up and to the left. While this does make the G6:K18 range selected, the selected range has K18 as the active cell.

                  

So your formula should have been,

=NOT(ISNUMBER(K18))

When you used your formula and then returned to it with the G6:K18 range selected but with G6 as the Active Cell, Excel's CF used the relative G6 reference to start at G6 and go right 4 columns to column C and up 12 rows to row 1048570. It's row 1049570 because Excel wraps to the bottom and continues offsetting once you have reached row 1.

It may help to think if the formula in R1C1 terms. WIth K18 active using your formula becomes

=NOT(ISNUMBER(R[-12]C[-4]))

That's all pretty confusing. The solution is to make your initial range selection from top left to bottom right, not the other way around. In other words, always leave the top left cell as the active cell if you want predicable results.

You can select multiple sheets much like you can select multiple files in Windows Explorer:

  • Click the sheet tab of the first sheet you want to select, then hold down the Shift key while you click the sheet tab of the last sheet you want to select. All sheets in between will also be selected.
  • Hold down the Ctrl key to select or deselect individual sheets. This way you can select sheets that are not next to each other.

Warning: when multiple sheets are selected, changes you make to cells will apply to all selected sheets. So don't forget to select a single sheet again when you're done with editing/printing/exporting multiple sheets.

    Dim obj As Object

    Set objOL = CreateObject("Outlook.Application")
    Set objNS = objOL.GetNamespace("MAPI")
    Set objContactsFolder = objNS.GetDefaultFolder(olFolderContacts)
    Set objItems = objContactsFolder.Items
 
    For Each obj In objItems
        'Test for contact and not distribution list
        If obj.Class = olContact Then
            Set objContact = obj
                .Delete
            End With
        End If
 
        Err.Clear
    Next
 
    Set objOL = Nothing
    Set objNS = Nothing
    Set obj = Nothing
    Set objContact = Nothing
    Set objItems = Nothing
    Set objContactsFolder = Nothing
End Sub

Hi Mike,

 

Thank you for posting your query in Windows 10 Technical Preview.

 

It seems like you have selected an option to enable use of On-Screen Keyboard, hence starting On-Screen keyboard each time you sign in to the computer.

 

I suggest you follow the steps below to disable it and check if the issue persists:

 

  1. Press Windows key + R
  2. Type control
  3. Select Ease of Access Center
  4. Select Use the computer without a mouse or keyboard
  5. Uncheck the box which says Use On-Screen Keyboard
  6. Select Apply and Ok

 

Restart the device and check if the issue persists.

 

Hope it helps, do get back to us if you need further assistance with Windows.
