jay freedman say:

You can go either of two ways here.

The better way would be to leave all the section headers linked, and to put a StyleRef field in the header (once they're all linked, they behave like there's only one header for the whole document). Click in the header, click Insert > Quick Parts > Field, select StyleRef in the list of field names, and select Heading 1 in the list of field properties:

Click OK. The field will automatically repeat the text of the Heading 1 paragraph in each section. This has the advantage that if you add a new section, you don't have to think about how to get its heading into the header.

The other way, to continue what you're already doing, is that you must unlink the heading of every section from the one before it -- just unlinking section 2 from section 1 isn't enough.

ianrg say:

The scientific calculator using a more scientific order of operation and the basic calculator using a more basic order of operation is not new. It was the same with Windows 7.

Understanding how calculators use different orders of operations has been important since the first handheld calculators:

- Basic ones used simple left to right and display progressive results as entered (just like in Standard mode - in your example it shows an intermediate answer of 3 after the 5-2*),

- Medium ones do not display progressive results before you type "=". They wait until you enter "=" so they can wait to apply a more correct order,

- Advanced ones are like medium ones but also allow brackets to give you more control (just like in Scientific mode - it does not show the 3 because you have not told it the calculation is complete). This order is consistent with arithmetic in an Excel formula.

The answer is simple - don't use the standard mode and always, always, always check that your result on the calculator (any calculator) makes sense before you rely on it
