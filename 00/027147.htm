The NewMailEx event fires when a new message arrives in the Inbox and before client rule processing occurs. You can use the Entry ID returned in the EntryIDCollection array to call the NameSpace.GetItemFromID method and process the item. Use this method with caution to minimize the impact on Outlook performance. However, depending on the setup on the client computer, after a new message arrives in the Inbox, processes like spam filtering and client rules that move the new message from the Inbox to another folder can occur asynchronously. You should not assume that after these events fire, you will always get a one-item increase in the number of items in the Inbox.

 

For users with an Exchange Server account (non-Cached Exchange Mode or Cached Exchange Mode), the event will fire only for messages that arrive at the server after Outlook has started. The event will not fire for messages that are synchronized in Cached Exchange Mode immediately after Outlook starts, nor for messages that are already on the server when Outlook starts in non-Cached Exchange Mode.

 

For users using Cached Exchange Mode, the event will fire in all settings, provided that Outlook is running when the message is received: Download Full Items, Download Headers, and Download Headers and then Full Items.

 

Example

The following Microsoft Visual Basic for Applications (VBA) example demonstrates how the NewMailEx event passes an entry ID. The event will fire when an item is received in the Inbox. Note that for backward compatibility when running this code sample on versions of Outlook earlier than Microsoft Office Outlook 2007, this code sample also accepts as parameter a string that contains more than one comma-delimited Entry IDs for items that have been received in the Inbox since the last time the event was fired.


Private Sub Application_NewMailEx(ByVal EntryIDCollection As String)
    Dim varEntryIDs
    Dim objItem
    Dim i As Integer
    varEntryIDs = Split(EntryIDCollection, ",")
    For i = 0 To UBound(varEntryIDs)
        Set objItem = Application.Session.GetItemFromID(varEntryIDs(i))
        Debug.Print "NewMailEx " & objItem.Subject
    Next
End Sub

This event Occurs when one or more items are added to the specified collection. This event does not run when a large number of items are added to the folder at once. This event is not available in Microsoft Visual Basic Scripting Edition (VBScript).

Unfortunately, your presentation has been edited into corruption. You'll have to start over.

At first I thought it was the Preserve Master issue, so I opened a copy in PowerPoint 2010 for Windows and set the Master to be Preserved. But that made no difference, the layout would still disappear if the corresponding slide was deleted.

Then I noticed that everything on the Slide Master (the larger slide at the top in Slide Master view) had been deleted. All placeholders were gone. Someone didn't understand the role of the Master: it's the parent for all the slide layouts. I can't tell if it's because the master had all elements deleted or if the layouts were pasted in from another presentation, but the layouts no longer appear to have any connection to the Master Slide. As such, they act as independent slide masters that have the Preserve flag permanently turned off. Deleting the slide then causes them to disappear.

You should start over with a fresh blank presentation. Create a a custom color theme. You can also create a custom font theme using TextEdit to include your Helvetica Neue fonts, here's my tutorial on this subject: XML Hacking: Font Themes. Apply the theme to the presentation.

Then start with the slide master. Format this to look like a plain text slide with as much of the formatting that is found in the most common slides (usually a text with bullets design). Spec the colors from your color theme and the font choices should all have (Body) or (Headings) appended to the Helvetica Neue name.

Now create layouts. Make use of the existing layouts with their stock names: these are needed if someone tries to update an old presentation with the new theme. Then make new layouts for the designs you lack. Don't paste in layouts or copy your old ones, you'll just perpetuate your current problem.

When you find a change that should apply to all or most layouts, like a different bullet for level 1 text, make that change on the master. The change will automatically ripple through the layouts. Always do as much formatting as you can on the master rather than changing each layout separately.

Finally, create any sample slides needed. The layouts will not require a slide to keep them in existance.

If you follow this Theme>Master>Layouts>Slides workflow, your presentation will work as intended.

As an additional suggestion, you have multiple layouts that vary only with the background photo. The slides don't have any logos that might have to appear over top of the photo. Just to let you know, you could use 1 layout and place a Picture Placeholder in the background, then the user could choose a photo they want for the background. This would let you get by with fewer layouts.

So, is there a way to limit the box popping up to once per website?  Or is it set to pop up once per feed?


Do you want the prompt at all?   The default is not to have it (or any other prompt regarding scripting, etc.).   So, if that would be Ok, just doubleclick on the Security zone icon in the Status bar and click Default Level.   Or, if you do want prompts normally but not for this site, you could try adding this site into your Trusted Security zone and then make sure that it wasn't prompting (e.g. set Default Level for that too.)

To answer your question a prompt occurs as often as the feature it implies is encountered when Yes is used.   When No is used it tends to be encountered less frequently, so then there are less prompts.  The feature is mainly a diagnostic so there is really no compelling point to make it more usable IMO.
