Print Send to OneNote Select Location in OneNote print queue of death - Microsoft Community

onthemark say:

Print Send to OneNote Select Location in OneNote print queue of death

Using the Desktop OneNote 2013 print "Send to OneNote" if the "Select Location in OneNote" window is active any print spool jobs "sent to OneNote" will error with the following message "OneNote can't do this action while a dialog window is open.  We're sorry."

When the error message is displayed the print job has already been removed from the print queue and is lost.

I would expect OneNote to gracefully handle the print queue that if OneNote can't accept another print job while a dialog box is open then it should not accept the next print job until OneNote is ready.

This limitation with the print "Send to OneNote" and using the "Select Location in OneNote" for each print job makes it tedious, error prone, and unnecessarily long to perform many print jobs to OneNote that take time to spool and process in OneNote.

To print "Send to OneNote" many long documents you have to print each document one at a time.  This means you have to wait for the document to complete the print spooling.  Then wait for OneNote to handle the print job and display the "Select Location in OneNote".  Then after selecting the location you are able to continue with the next document.

During the time to wait for the spooling and oneNote processing you might multi-task and unless you use a method to help track the documents you have printed you may end up double printing or omitting a document by accident.  Oh, and if you happen to have the "Select Location in OneNote" dialog hidden behind a window you may find yourself with a wasted print job.

The always save might be a solution but isn't very good when the notebooks are synced with Cloud. Not all Send to OneNote print jobs will go to the same notebook and to move notes from one Notebook to another is another big hit on performance due to the unnecessary synchronization and increases synchronization failures between multiple computers.

I would like a better solution for faster print "Send to OneNote" tasks.  I might test out having a Notebook that is explicitly created on the local computer that after all documents have been printed they are then moved to the target Notebooks/sections.  This defeats some of the print and sync implicit features of OneNote because the local NoteBook will be the only copy of the data until the notes are moved into online notebooks.

-Mark
