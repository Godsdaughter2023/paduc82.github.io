jay freedman say:

There is nothing like that built into Word. There are only two ways to limit the text entry, and neither of them is really good.

The easier way is to put the control in a cell of a table, and use the Table Properties dialog to set the height of the table row to "Exactly" a desired size. 

Although the user can type more than will fit in the table cell, the overflow won't be visible or printable. This method isn't usable if the entries in the controls will be read by a macro, which can "see" everything in the control.

The other method requires macro programming. There are a couple of ways to do that. One is to check the number of characters in the control when the user moves the cursor out of the control, with code like this:

Private Sub Document_ContentControlOnExit( _
        ByVal CC As ContentControl, Cancel As Boolean)
    If CC.Type = wdContentControlRichText Then
        If CC.Range.Characters.Count > 250 Then
            MsgBox "Too much text. Limit is 250 characters."
            CC.Range.Select
            Selection.Collapse wdCollapseEnd
        End If
    End If
End Sub

However, if the user inputs too much text and then saves without exiting the control, this code won't run.

Another way takes more complex coding but can alert the user as soon as the text exceeds the limit. It involves setting up a "class module" with code that runs every time a character is added or removed.

Both of these programming methods suffer from the drawback that they can only count text characters. A rich text content control allows inserting graphics, tables, large font sizes, and anything else you could put into a document, but a macro wouldn't notice the change in the control's size. Unfortunately, VBA doesn't have any good way to directly measure the height of a content control.
