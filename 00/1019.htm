How to calculate a custom running sum field in a report - Microsoft Community

faradaynwonu say:

How to calculate a custom running sum field in a report

Hello,


In a report I have a field named "Previous Balance",
and another calculated field named "Running Balance" with running sum.

the running sum is working perfectly well by my problem is that I want the running sum of the "Running Balance" field to start from adding the value in the "Previous Balance" field first.

I am waiting hopefully for any solution to this please.

ken sheridan say: The poor performance of your query might be due to inadequate indexing of the columns in the base table, but it should be a simple task to compute the balances in the report.

1.  Assign the previous final balance per customer to a text box control in the customer group header section with code in the section's Format event procedure.

2.  Set the ControlSource property of a text box control in the detail section to [Credit]-[Debit], set its Visible property to False (No), and its RunningSum property to 'over group'.

3.  Add a text box control to the detail section and set its ControlSource property to an expression which adds the value of the previous final balance control in the group header to the value of the hidden control in the detail section.  This should give the current balance per transaction per customer.

ken sheridan say: The fact that you are getting the previous balance per customer from a 'separate table' causes me some concern.  One would normally expect the transactions data in the report and the computed balances, both current and previous, to be derived from the same table.  You can then return the balances per customer over all time in a query, such as the following:

SELECT T1.[CustomerID], T1.TransactionDate, T1.Credit, T1.Debit,
SUM(T2.Credit-T2.Debit) AS Balance
FROM TransactionsCD AS T1 INNER JOIN TransactionsCD AS T2
ON (T2.TransactionID<=T1.TransactionID Or T2.TransactionDate<>T1.TransactionDate)
AND (T2.TransactionDate<=T1.TransactionDate) AND (T2.[CustomerID]=T1.[CustomerID])
GROUP BY T1.[CustomerID], T1.TransactionDate, T1.TransactionID, T1.Credit, T1.Debit
ORDER BY T1.[CustomerID], T1.TransactionDate DESC , T1.TransactionID DESC;

This is taken from the file Balances.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

which illustrates ways to compute balances in various scenarios.

A report for a specific period of time and/or customer can be produced by basing it on a query such as the above (but omitting the ORDER BY clause and using the report's internal sorting and grouping mechanism) and filtering the report to a date range/and or customer.  The balances will then be returned correctly, taking account of all previous transactions, and the balance per customer immediately prior to the data range can easily be returned in the report by looking up the value from the query where the date is the MAX transaction date per customer prior to the start of the date range.

Insert WORD 2016 Document with TEXT and PICTURES into New WORD 2016 - Microsoft Community

jeroldaronson say:

Insert WORD 2016 Document with TEXT and PICTURES into New WORD 2016 Document using Insert OBJECT as TEXT

I have several, existing WORD 2016 documents containing Text and Pictures (jpg) that I want to aggregate into a Master WORD 2016 document with text and pictures. Do I use the Insert EMBED command or the Insert Text command to import existing documents. Once existing documents are inserted into Master Document, what controls the formatting of Master document. Current Microsoft videos are not specific for text and pictures in a previous document. Please advise.

jay freedman say:

Do I use the Insert EMBED command or the Insert Text command to import existing documents.

Neither. In the master document, go to the View tab of the ribbon and click Outline. The ribbon switches to the Outlining tab and the headings in the master (if any) appear as topics. Click the Show Document button on the ribbon. Position the cursor where you want to insert a subdocument, click the Insert button on the ribbon, and select the existing subdocument file in the dialog box.

 Once existing documents are inserted into Master Document, what controls the formatting of Master document.

If all the styles in the subdocuments are defined with the same properties as those with the same names in the master, there will be no change in formatting. Also, any styles in the subdocuments that don't exist in the master will remain as is. If there are styles with the same name but different formatting, you'll be prompted whether to rename the styles in the subdocument to something else.

The best way to avoid problems is to base all the subdocuments on the same template, but that isn't always possible when the subdocuments come from various sources.

The master document feature has had a very unhappy career in previous versions of Word; document corruption has been common, especially if you edit the subdocuments from the master view (http://www.addbalance.com/word/masterdocuments.htm). There's some evidence that Word 2016 is not much better. You'd be well advised to create a single document by copying the content of each "subdocument" except for its final paragraph mark (which contains the headers/footers, margins, and other document-wide properties) and pasting it into the "master" following a Next Page Section Break.
