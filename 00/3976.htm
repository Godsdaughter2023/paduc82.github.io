Function GetVIndex(Findme As Variant, vNamedrange As Variant) As Long

'To get the index of a named range

Dim row As Long
Dim iRows As Long
Dim Pos As Long

On Error GoTo ErrorHandler

vNamedrange = vNamedrange.Value2  'Convert Range to array
iRows = UBound(vNamedrange)
ReDim vNamedrange(1 To iRows, 1 To 1)

ReDim clears an array. To prevent that, use ReDim Preserve:

Function GetIndex(FindMe As Variant, SearchMe As Range) As Variant
  GetIndex = SearchMe.Find(FindMe, SearchMe(SearchMe.Count), xlValues, _

End Function

Note: The 2nd argument is a Range, not a String, so your formula would be written this way...

          =GetIndex(2, vNumlist)

rena yu msft say:

Hi Randy,

 

Nice to help you on this issue.

 

Based on your descriptions, please check the information below to see whether this is the cause of the issue:

1.      If the issue happens when you try to save the file to the OneDrive local sync folder, the error may be caused by the File Explorer path length 260-character limitation. That’s why when you change to save files to the parent folder, the issue disappears. Because the file path is shortened.

2.      When you directly upload file to the SharePoint Online location folder via Internet browser, the issue doesn’t happen. Because in this uploading way, it uses SharePoint URL length limitation which is 400-character.

3.      Please notice, although the limitation is 260, we don’t suggest you really make the path to 260. As the error message may appear when the path length is around 260. The best way is to shorten your path length as much as you can.

If the issue is caused by file path length limitation, you can try to save other file types such as Word document or Excel workbook with longer name to the location folder to see whether the issue can be reproduced.
