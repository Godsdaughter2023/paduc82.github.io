Okay, I've finally found an answer for this problem. There is a bug with storing attachments to e-mails on the storage card. To fix the problem, go to the Storage Card in file explorer, and delete the folder with "inbox" in the name. (Mine was called InboxXXXXXXX, where the X's were a string of numbers.) As soon as you delete that folder, your access to messaging will be restored, but you need to immediately turn off the option to store attachments to the storage card. So, go to the main messaging screen, click on Menu/Options, and go to the Storage tab. Uncheck the "When available, use this storage card to store attachments" option. Then do a soft reset, and everything should be fine again.

ken sheridan say: The Nz function will return a value in the event of the column being Null, or the value in the column otherwise, so:

Nz(Table1.[first],"") <> Nz( [Table2].[first],"")

The comparison is now between two string expressions rather than a string expression and NULL, or two NULLs.  Comparing NULL with anything, even another NULL, returns a NULL, neither TRUE nor FALSE.

karl timmermans [outlook mvp 2012-2018] say:
In the current version of Outlook, you cannot add the same email account to an existing profile using a different protocol. In other words, you can't add the same email account as both POP and IMAP in the same profile.

Just for the sake of accuracy, it has never been possible to "change" an email account from one protocol to another (i.e. changing POP to IMAP). The account always had to be configured again using the desired protocol.

Your options are (for sake of simplicity - the assumption is that only one account is configured in the profile as would be applicable to references to "default data file" in the description below)

#1 - Delete the POP3 account from the profile and add the account again as an IMAP account. The PST file associated with the POP3 account will not be removed from the profile and will remain the default data file for the profile (default data file contains the default contact/calendar folders).

One key advantage to this is that you don't need to worry about contact/calendar data retention and the IMAP folder tree will not contain any folders with the added description of "This Computer Only".

Essentially the above approach is the equivalent of protecting "This Computer Only" data as described in

How to Protect Outlook '2013/'2016 Imap Contact/Calendar Data
https://www.contactgenie.info/how-to-protect-ou...

#2 - With Outlook closed, using the Control Panel <Mail> app to create a new Outlook profile and configure the account using IMAP (default when entering only your email address/password for the account).

The disadvantage to this approach is that account will be created with "This Computer Only" folders (i.e.contact/calendar) which will be empty upon creation. You would then need to add the PST file from the POP3 account to the profile to add that data to the profile and make it the default data file for the profile. It's not recommended to maintain any data in "This Computer Only" folders since if the account is ever deleted or Outlook cannot find the underlying OST file for any reason, the underlying OST file will be recreated and the data in "This Computer Only" folders will be lost.
