daniel pineault say:

This is not surprising.  Access is backwards compatible, not forwards compatible (no software is).  So 2010 can handle older files (2007, 2003, ...), but 2007 can't handle newer version (2010, 2013, ...).  So it makes sense that trying to run a 2010 database in 2007 will fail.

You always have to do all your development using the oldest version of Access that will be used to run the database.  In your case, if your have Access 2010 and 2007, you need to do all the development in 2007.  Then you deploy an individual copy of the front-end to each user.  You never share a common copy of the front-end.  You never deploy an unsplit database for multiuser use.

At this point, you could try opening the db in 2007 and compiling it, check the references, ... and see if you can get it back in working order.  The other approach would be to create a new blank 2007 accdb and then import everything and go from there.

You may benefit from reviewing


  • Setting Up an MS Access Database
  • Access Best Practices and Troubleshooting Steps

greginmich say:

Windows Defender isn’t supposed to be turned off via Group Policy when a third-party AV app is installed, as I’ve explained in this thread:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-defender-is-off-i-cant-turn-it-on/c1ef0d9b-7940-4108-9578-de2c996b75ea

 

So on a home PC; we would normally assume that the Group Policy switch was thrown by malware that’s somehow managed to escalate its privilege-level to the rank of “administrator”. It’s also quite likely that the malware succeeded because Defender didn’t have a signature/behavioral detection for it – so I would recommend scanning with this set of malware-removal tools:

 

Kaspersky Virus Removal Tool:

http://support.kaspersky.com/viruses/kvrt2015

 

Emsisoft Emergency Kit:

http://www.emsisoft.com/en/software/eek/

https://www.malwarebytes.org/antimalware/

Then turn Defender back on with this REG command: Right-click on the Start button, select Command Prompt (Admin), and then copy, paste, and run (enter) this command line:

REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware
