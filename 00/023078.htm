Generally, it would not be recommended to use pop3 or imap to access your emails with the Office365 email system.

the below vba macro has been tested

using an excel  file with 503000 rows and 26 columns.

Convert data from active sheet to a .csv file

(at the end open .csv file via notepad)

csv file name is: ONE-CSV

Sub ADODB_Excel_to_CSV_OneFile()
'Jan 02, 2016
Dim ws As Worksheet
Set ws = ActiveSheet
Dim sPath As String
sPath = ThisWorkbook.Path & "\"
Dim sFile As String

r = ws.Cells.Find(What:="*", SearchOrder:=xlByRows, SearchDirection:=xlPrevious).Row
c = ws.Cells.Find(What:="*", SearchOrder:=xlByColumns, SearchDirection:=xlPrevious).Column
Dim obj As Object
Set obj = CreateObject("ADODB.Stream")
obj.Type = 2
obj.Charset = "utf-8"
'obj.Charset = "unicode"
obj.Open
Dim v() As Variant
ReDim v(1 To c)
For i = 1 To r
For j = 1 To c
v(j) = Cells(i, j).Value
Next
obj.WriteText Join(v, ","), 1 '<<comma
Next
obj.SaveToFile sFile, 2
Dim Npad
Npad = Shell("C:\WINDOWS\notepad.exe " & sFile, 1)
End Sub

NOTE

ignore the message on the ribbon (Not Responding)

ken sheridan say: If the label and text box controls are arranged vertically, then you might be able to set the CanShrink property of the bound controls to True (Yes), but if there are any non-shrinkable controls, including graphics, alongside the control then it won't shrink.  If you cannot use the CanShrink property then you can hide the control (if necessary) and its label with code in the detail section's Format event procedure, e.g.

If IsNull(Me.txtSomefield) Then
    Me.txtSomeField.Visible = False
    Me.lblSomeLabel.Visible = False
End If

Note that this won't work in report view, however, only in print preview or if the report is sent to a printer.

A common situation in which it is necessary to suppress a Null is when including a multi-line address in a report.  The best solution for this in my view is to use a single text box control for the address, with its CanGrow property set to True, and then concatenate the values of its component fields into the text box, separated by carriage return/line feed characters.  Microsoft has published a CanShrinkLines function which does this, suppressing Nulls so that you aren't left with a blank line in the address.  You'll find a slightly amended version of this function, along with other concatenation functions, in Concat.zip in my public databases folder at:
