Since the existing table is just a View, I suppose one way to get a random number is to generate an extra column containing random values in the query that creates the view.  However, the problem seems to be the same as in my original post -- to get an expression that results in a random value that varies by the record.

paul domaskis say:

This works:


   CREATE VIEW tBadPart AS
   SELECT TOP 100 * FROM ( SELECT Part FROM SourceTable ORDER BY getRndVal(Part) )


This does not:


   CREATE VIEW tBadPart AS
   SELECT TOP 100 Part FROM SourceTable ORDER BY getRndVal(Part)


The latter causes an error about only allow simple queries in creating views.


But for me, the source table was so big that it took a few seconds to create the random column and sort by it.  So I decided against a view and created a table instead.  It is, after all, a small table being created from a very big table.  This allowed to to avoid the bracketed nesting.

larryyu_msft3110 say:

Hi TT,

Below are the answers to your questions:

Q1: The Achieve feature is available for my package, don't you? How this Archieve feature work? Can you please tell me detail about this feature?

A: Yes, the Archive feature is available for Office 365 Business Essentials subscriptions. After you enable the Archive feature for a mailbox, there will be an archive mailbox linked to the primary mailbox. The archive mailbox also provides users with additional email storage (usually 50GB). Emails will be moved to the Archive mailbox if there are archive related retention policies or users can also move emails to the Archive mailbox manually. 

Q2: And the Retention policy, how it works? Where all deleted emails will be located or being deleted permanently?

A: By default, the Default MRM Policy retention policy is applied automatically to all users. Emails will be deleted or moved to archive when days meet the policies. Global admins can create custom retention policies and then apply them to users by following the steps in the article below:

https://technet.microsoft.com/en-us/library/jj150573(v=exchg.150).aspx

About the three tags, you can refer to the instructions in this thread.

The deleted emails will be moved to Archive or deleted permanently or deleted and able to be recovered. This depends on what you have chosen when create the retention tags. Note: since your subscription doesn’t have the In-Place hold feature, the deleted emails cannot be recovered after 30 days.

Q3: Except to use the retention policy, do you have any way to empty mailbox fastestly? 

A: If the email items are fewer than 10000, you can follow the steps below to empty the mailbox at once:

1. Use a global admin account to connect to the Exchange Online using remote PowerShell.

https://technet.microsoft.com/en-us/library/jj984289(v=exchg.160).aspx

2. Run the command below.

Search-Mailbox -Identity “*** Email address is removed for privacy ***” -DeleteContent

Note: Replace “*** Email address is removed for privacy ***” with your mailbox’s email address.

3. When it prompts the Confirm message, type “Y” and then press Enter.

It will take some time to empty your mailbox items. If your mailbox items are more than 10000, you need to use the retention policy or manually empty your mailbox. We appreciate your understanding.

Feel free to let us know if you have other concerns about this.

Regards,
Larry
