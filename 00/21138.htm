steven mulkerrins say:

Hi John,

The key difference is that Information Management Policies only apply to SharePoint content, but the Compliance Center will work across your whole Office 365 tenancy and will include content in Exchange Online, OneDrive, Skype for Business etc.

Also, as you suggest, Information Management Policies only work in the site collection they are created in, but Compliance Center policies will work across all site collections. 

There are some differences in the available functionality too.

Information Management Policies currently allow for more powerful and fine grained control, for example they can trigger SharePoint workflows, where the Compliance Center is limited to the templates it provides. To be fair, the Compliance Center templates probably meet most organisations' requirements.

The Compliance Center also offers more powerful auditing and reporting, eDiscovery, DLP, device management and more.

That's a symptom of a Word Data Key corruption - usually because Word has
been running when updates are installed.
 
Run Regedit and expand by double-clicking to the key below. With DATA
selected, press DELETE. Word will automatically recreate a new key when it
is restarted.
 
HKEY_CURRENT_USER\Software\Microsoft\Office\12.0\Word\Data
 
Symptoms of Data Key corruption include:
 
   •    The mouse does not work when you use Word.
   •    You cannot open a Word document from the Search window in Windows
Vista.
   •    You cannot open a Word document from Windows Desktop Search.
   •    Word crashes when you try to start or exit Word.
   •    Word crashes when you open the Open dialog box.
   •    Word crashes when you save a document.
   •    Word crashes when you close a document.
 
If you feel uncomfortable editing the registry yourself, Microsoft have a
fix that will do this for you.
http://support.microsoft.com/kb/940791 and press FixIt

rick rothstein, mvp - excel say:

First, two notes...

(1) You skipped Column H and included Column J twice. My code below assumes you
      meant those column widths to apply to Columns A thru AL contiguously in order.

(2) When using the Columns property to reference a single column, you do not have to
      repeat the column letter with a colon between it (you only have to do that with a call
      to the Range property), you can just use the single letter. For example, for Column A

      Columns("A").ColumnWidth = 14.14

Okay, now for the code (which replaces what you posted). The following assumes the column widths for Columns A through AL are set, in order with no column gaps, in the Array function call.

Sub FixColumnWidths()
  Dim X As Long, Widths As Variant
  Widths = Array(14.14, 7, 12.43, 14.86, 10.29, 3.29, 11.71, 6.57, 4.14, 5.14, 7.29, _
                 3.86, 2.43, 3.29, 3.71, 1.86, 12.57, 3.71, 3.57, 3.86, 3.71, 3.43, _
                 3.86, 5.71, 3.14, 3.29, 2, 5.86, 11.29, 4.71, 4.71, 4.57, 3, 4, 4.14)
  For X = LBound(Widths) To UBound(Widths)
    Columns(X - (LBound(Widths) = 0)).ColumnWidth = Widths(X)
  Next
End Sub
