      Feedback on case # 1426347851 re failing upgrade to Windows 10 - Microsoft Community

      grickm say:

      Feedback on case # 1426347851 re failing upgrade to Windows 10 1803 with Error 0xC1900101-0x20017

      On an ASUS P6T Deluxe V2 custom built system the upgrade from 1709 to 1803 failed repeatedly with Error 0xC1900101-0x20017. Even with help from Microsoft Level 2 we weren't able to resolve this problem. After a very detailed Internet search it turns out that this motherboard's BIOS setting of "enabled" for Intel Virtual Technology Directed I/O, aka vt-d, caused a hang after the first reboot during the upgrade. Disabling this item in BIOS let the upgrade proceed normally.

      I appreciate the attempts to help by Level 2 analysts Syed and Lakshman. These kinds of system-specific issues can be difficult to diagnose. Here, there were no installation log or event log entries that pointed to the cause of the hang.

      Strongly recommend having a recent system image backup of the OS drive, usually C:, for dealing with these types of situations. Level 2 and I made many changes trying to solve this problem. It was good to be able to restore the working system image before trying this latest fix.

      Update: re-enabling vt-d causes Windows 10 to 1803 hang on boot, just as it did during the upgrade. The feature must remain disabled after the upgrade.
