ericyang_msft6286 say:

Hi Samer,

Actually, the “sites/ImportantSite” part in the URL represents a directory on the host “ourdomain.sharepoint.com”, which indicates that the particular site collection is hosted under the particular directory of the server (or a server farm) whose IP address is associated with the domain “ourdomain.sharepoint.com”. Note that we can only associated an IP address with a domain (e.g. ourdomain.sharepoint.com) but not a URL (e.g. https://ourdomain.sharepoint.com/sites/ImportantSite). All the site collections on the same domain physically share a common IP address since they are hosted on a same server.

On the other hand, SharePoint server doesn’t support requesting a page using the IP address directly. Therefore, we you try to visit a IP address directly in a browser, no page will be returned by the server and you will see the message including some details about the HTTP connection.

jim gordon mvp say:
Safari and Outlook are not a combination that is supported for sending web pages. Apple supports only Apple Mail for this feature.

A work-around (not very good in most instances) is to use Microsoft Word. In Word use File > Open URL. Wait for the page to load (some won't). If the page loads and looks OK, then you can try File > Share > Email as HTML. If Outlook is your default email client, a new email message should open in Outlook and you can see how good or bad the web page looks. Word's HTML capabilities haven't been updated in like forever, so don't expect most pages to work using this method.
