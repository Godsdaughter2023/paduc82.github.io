davidf2 say:

Without seeing it I can only speculate about all the reasons. Generally a lot of print document formatting, fonts, and design techniques can't be converted to HTML. Sometimes there simply is no equivalent.

Do not use a Master Page with Publisher webs. It has always caused problems and should only be used with print media.

If you convert a text box into an image and kill the link you have to change the formatting.

A text box that is filled with color can kill links as it turns the text into a picture. Remove the fill color. One way to discover if the text has been converted to an image is to left click, drag and try to select the text in FF. If you can not then it has been converted to am image. Further you can save all the web files to your local hard drive and assuming you used the supporting folder option open the index_files folder and view the images in thumbnail view.

Make sure to ungroup everything on the page.

Use web "safe" fonts only. Publisher converts non web fonts to images.

Do not use anything but a single border on a text box and no drop shadows. You might have to drop the border altogether. There is a workaround I can provide if borders or fill colors are killing the links.

Run the Design Checker for Web site checks.

You might also want to reconsider using Publisher for this purpose. While I and others have used Publisher to build webs for years, MS decided to deprecate this functionality as of Pub 2010. I won't get into all the reasons but you might want to look to alternatives before you invest more time. Consider WordPress, MS Expression Web and there is a free version of Serif WebPlus that works much like Publisher and is where I wish MS had taken Publisher.
