john w. vinson say:

Hrm. Your table design isn't the best - you have a "person's name" attribute stored in two different fields, making this query more difficult!

 

I'd suggest a "normalizing union query" as the basis for a crosstab query. To get all the names in one column use a query like

 

SELECT Account, [Number], City, State, Male AS Person

FROM yourtable

WHERE Male IS NOT NULL

UNION ALL

SELECT Account, Number, City, State, Female

FROM yourtable

WHERE Female IS NOT NULL;

 

Save this query as uniAll, and then base a Crosstab query (using the Crosstab wizard) on uniAll, using Account, Number, City and State as row headers and Person as the column header.

dhookom say:

An Access crosstab query can have many row headings, a single column heading, and a single value.

 

Rather than using a single field for the column heading, use an expression like:

 

ColHead: Nz([Male],[Female])

 

This will generate column headings for every male and female. It won't work if there are possibly Male and Female values in the same record. If you have that possibility, you will need to use the union query as suggested by John "the Wizard" Vinson.

The problem occurs because the font characteristics in the list template have become corrupted.  It can be fixed by running a macro containing the following code:

For Each templ In ActiveDocument.ListTemplates

For Each lev In templ.ListLevels

lev.Font.Reset

Next lev

Next templ
