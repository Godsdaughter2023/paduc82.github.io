joeu2004 say:

When you have a question about a function, it would be helpful if you show us how you are using the function. A concrete example.

Jim wrote: "If I double-click in a specific "Table-array" cell, the corresponding VLOOKUP value evaluates correctly".

A couple thoughts....

1. The cells that you double-click and re-enter might contain values that are type text, even if they look like numbers. You can confirm by verifying that ISTEXT(A1) returns TRUE.

If that's is true, you should be able to correct the immediate problem by select the column of data, and using Data > Text To Columns to correct the values.

If that does not work, you might have to build a parallel column of formulas of the form =TRIM(SUBSTITUTE(A1,CHAR(160),"")), then copy the parallel column and paste-value over the original column.

2. The cells that you double-click and re-enter might contain the result of calculations that are infinitesimally different than the values in the lookup table. You can confirm by verifying that MATCH(A1,B10,0) returns #N/A, where B10 is presumed to be the lookup table value that should match.

If that is the case, the best solution is to explicitly round the original calculation to the precision that you require. For example, =ROUND(expression,2) for dollar amounts.

Alternatively, build a parallel column of formulas of the form =ROUND(A1,2), then copy the parallel column and paste-value over the original column. Change 2 to the precision that you require.

If neither works for you, I suggest that you upload an example Excel file that demonstrates the problem to a file-sharing website (e.g. box.net/files), and post the public/share URL in a response here. Test the download URL first, being careful to log out of the file-sharing website. (If you use box.net/files, ignore any preview errors, and just download.)
