Be advised that I just retested the “Microsoft” method for scheduling a scan and it still doesn’t work:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-10-automatic-scanning-in-defender/3b32ac97-7abe-4c53-96d4-d94f72b9c7af

As far as I know, the idea for scheduling a scan by hacking the Windows Defender Scheduled Scan task started with this tutorial:

http://www.eightforums.com/tutorials/14264-windows-defender-scheduled-scan-set-up-windows-8-a.html

 

At the time, the author believed that “For Windows Defender, Auto Maintenance will only check for updates. Only the task schedule you created will run the daily scan" (page 2). That’s not true – Automatic Maintenance runs the Windows Defender Scheduled Scan task on a daily basis along with the rest of its maintenance chores. So this scheduled task isn’t something that’s just sitting there waiting for you to come along and add a trigger to in order to get it started. It’s the system task for Automatic Maintenance, and any modification of this task will modify the scan that Automatic Maintenance runs daily. For example, if you schedule a weekly Full Scan here, as described in the Eight Forums tutorial, then Automatic Maintenance will attempt to run that Full Scan on a daily basis. That’s virtually impossible for most users, because Automatic Maintenance is strictly confined to idle time, and it cancels the scan (as opposed to pausing it) with any user activity. Furthermore, if you leave the PC unattended, your Full Scan will be automatically terminated at the end of the 1-hour maintenance window. So this method of scheduling a Full Scan can really foul up Automatic Maintenance, and I’ve been complaining about it ever since it was first published.

Now the “Microsoft” version of this hack only runs a Quick Scan, because it only adds a trigger to the existing task and doesn’t mess with the task’s action (command line). So this won’t foul up Automatic Maintenance, but on the other hand, it doesn’t really make any sense, because Automatic Maintenance is already running a Quick Scan every day. Why on earth would anyone want to schedule a daily Quick Scan when Automatic Maintenance is already running a daily Quick Scan? But wait, it gets worse – because the task that this method runs is just as flawed as the rationale for running it. In Windows 10, this hack doesn’t run the scan with normal priority like it did in Windows 8 – it just runs a clone of the Automatic Maintenance scan – which means that the scan waits for an idle state to get started and then gets cancelled whenever you start using your PC. So your scheduled “Quick Scan” can take hours to finish. That’s probably not what you had in mind. But so what if the “Microsoft” method doesn’t make any sense and doesn’t work worth a hoot – at least it provides us with a great example of what can happen when you hack a system task.

Since Automatic Maintenance is already running a daily Quick Scan, the only type of scan that anyone would normally want to consider scheduling would be a Full Scan, and for that I’m afraid you’ll have to use the proper method for scheduling any task in any version of Windows – running the Create Basic Task Wizard in Task Scheduler. This thread covers the topic of scheduling a Full Scan:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender/588b2978-dc88-432b-bc8e-066acf6b9258

Nevertheless, if you want a daily scheduled Quick Scan that runs with normal priority, and that runs on schedule, you can also use the standard scheduling method for that :

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender-in-windows-10/6979df16-e16e-4ec1-9170-7081f54c22df

An alternative method for that would be to use the shortcut method for getting the Windows Defender icon at startup and have the shortcut also run a Quick Scan:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/is-windows-defender-icon-supposed-to-be-in-system/6cfa9d1d-3db8-49a9-add7-280a284ccfdc

The full set of options and command line parameters for Windows Defender scheduled scans are covered here:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02?page=1

Well... bear in mind that a template is just that - someone ELSE'S idea of how YOU want to manage your data. No template is going to be written to exactly match how your business runs, especially since your business model isn't all that typical it would seem!

 

I would suggest that you think of Access as a tool chest. You wouldn't open your toolchest (or even a newly purchased toolchest) and expect to take out a finished kitchen cabinet; but you could use the tools in it to build a kitchen cabinet (after some learning and practice). Access is just the same: it's a very well stocked tool chest, and with some learning and practice, you can use it to build (from scratch, no template needed!) a database that fits just what YOU want it to do.

 

Or - just as I'd probably hire a professional carpenter to do my kitchen cabinets - you can hire a professional database developer to set up a database for your business.

The Office templates are all available from File > New.
