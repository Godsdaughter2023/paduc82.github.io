Open Outlook on the old computer and follow the steps below.

1.      In Outlook, on the File menu, click Import and Export.

2.      Click on File in the contacts window and click on Import and click on the last option Comma Separated Values (.CSV)

3.      Click on browse and go to the desktop and select the My Contacts.csv file saved on the desktop and click on OK.

4.      Click on Next and then make sure the fields required for the contacts in Windows Live Mail are properly mapped.

5.      Once the mapping is done, click on Finish to import the contacts from the .CSV file to Windows Live Mail.

beth melton say:

Try adding the Merge Record # field after UOFM000. This can be found on the Mailings tab, in the Write & Insert Fields group, under Rules.

However, if I suspect what you're really looking for is a 4 digit number with leading zeros, such as UOFM0042. If that's the case then you can still use the Merge Record # field but you'll need to add a numeric picture switch to add the approprate number of leading zeros:

  1. Type UOFM and add the Merge Record # field as described above.
  2. Press Alt + F9 to toggle the view of field codes.
  3. Modify the field and add \# 0000 at the end. Here's how it should look when you're finished:

UOFM {MEEGEREC \# 0000}

Note there is a space before the backslash (\) and after the pound (#) sign.

Then press Alt + F9 again to toggle off the display of field codes, select the field and press F9 to update it.

For more on using numeric picture switches see: http://office.microsoft.com/en-us/word/HP051862261033.aspx

Linked graphics in Word 2016 Mac seem very broken - Microsoft Community

steve kubis say:

Linked graphics in Word 2016 Mac seem very broken

Summary:  Linked graphics in Word 2016 Mac don't seem to work well any longer. After being added to a document, the graphics stop displaying and show the message "The linked image cannot be displayed. The file may have been moved, renamed, or deleted. Verify that the link points to the correct file and location."

Longer story...

I have a document that originated in Word 2011 Mac. It has about 50 referenced graphics. It had no problems converting to Word 2016 Mac, and was updated through April of this year. Beginning in June, the referenced graphics started displaying the message about not being able to be displayed.

Interestingly, the exact same document can be opened in Word 2011 Mac and view all the graphics with no problems. The file and graphics can be moved over to Windows, and both Word 2013 and 2016 can view the graphics with no problem. Only Word 2016 Mac has this issue.

This is the latest version of Word 2016. (15.24)

Some other notable behaviors with this problem:

The Links window in Word 2016 can't display any paths to the "missing" graphics for the document. The Links display in Word 2011 can properly display the Links window, but that would be expected since it can find the graphics.

We've done many experiments to try to find what might cause the problem. We can link to a graphic in Word 2016, and then purposely break the link by renaming the graphic. As expected, the graphic displays the 'missing graphic' message. When we change the name back to the original, one of two things happen. Either the graphic remains missing, or a message is displayed in Word 2016 Mac that says "Additional permissions are required to access the following files" with a list of the linked graphics. If we continue through the dialog to grant permission, then the graphic displays.

Word 2011 doesn't show this behavior with missing graphics. When we change a graphic name, the graphic show the 'missing graphic' message. When we change the name back, the graphic is always correctly displayed again. There is never any "permission" dialog displayed.

It seems like Time Machine backups may impact this process. Things seems to work normally for the referenced graphics in Word 2016 Mac, but after Time Machine has backed up files, we start to see these problems. We haven't been able to isolate the steps that cause this, but it's something we're suspicious about.


Work-around:

The current work-around is to switch from using linked graphics to using embedded graphics. Normally we avoid too many embedded graphics, because they traditionally have their own issues with document corruption. But, its our only option.

To embed the linked graphics, we must open the problematic documents in Word 2011 Mac, go to the Links window, select all of the links and choose the option to "break" them. That makes all of the graphics become embedded, rather than linked.

As an aside, we did some testing of the "break links" functionality in Word 2016, and it didn't seem to work properly. In most cases, it did nothing. The linked graphics didn't get embedded. That's why we had to do this work-around in Word 2011.

This has been a frustrating problem. Unfortunately, two support calls to Microsoft haven't resolved the issue. After the issue was reported, both of the cases were closed without explanation.

If you experience this problem, hopefully the "embedded" graphic work-around is an option for you. More importantly, we hope that Microsoft fixes this issue in an upcoming maintenance release.
