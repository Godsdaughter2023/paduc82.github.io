brian tillman [outlook mvp 2007-2019] say:

When you are in the Calendar view of the Navigation Pane, there is no way to suppress the Date Navigator (the little calendar) entirely as long as the Navigation Pane is open.  It will appear either there or in the To Do Bar.  You can minimize both bars if you don't want to see it, but if either is open, it will appear in one of them.  This is also true in the Folder List view.  As long as the Calendar is visible in the middle pane, the Date Navigator will be present.  Al least, I can find no way to suppress it.

In other views, use the To Do Bar, but minimize it.  Click the right arrow just to the left of the Date Navigator (the little month calendar) or right-click the To Do Bar and check "Minimized".  Alternatively, right-click the To Do Bar and uncheck Date Navigator to hide the small calendar but leave the To Do Bar open.

Select the text range, add a bookmark and then insert a cross-reference to the bookmark text.

If the style includes numbering, manual bookmarking shouldn't be necessary, because you should see the paragraph for which you are trying to create a reference in the "Numbered item" category in the Cross-reference dialog box.

stefan blom say:

What you wrote in a previous message puzzled me:

When going through my Word document (user manual) I select one of the customized heading styles...'FB_H2' and attempt to add a bookmark.  I can only add a bookmark based on the Hidden bookmarks (generated TOC selecting our custom FB_H1, FB_H2, etc.)  

It is true that an automatic table of contents will generate bookmarks (in order to create hyperlinks), but hidden bookmarks, especially those generated by a table of contents, are not relevant when you are creating a cross-reference.

"Move here/Copy here/Paste here" context menu (right click) What does it do? How can I kill it?

When I right click in Word/Excel/PPT 2013, I often get a short drop-down menu with Move here, Copy Here, Paste here commands, instead of the normal longer menu with Cut/Copy/Paste at the top. I've tried clicking the various commands, but as far as I can tell they don't do anything. Am I missing something? Is there any way to turn off this annoying alternative context menu? I've heard that the duration of the right mouse button click is what determines which menu will appear, but it seems to happen at random. I Just want the normal cut/copy/paste menu. steve rindsberg say:

It's not the duration of the rightclick, it's whether you move the mouse, even slightly, while rightclicking.  Rightclick + Drag is the way you copy/move/duplicate various things in Office, so if you move the mouse slightly, the program thinks that's what you're trying to do.

And it seems that some mice are a lot more sensitive (and more likely to provoke this when it's not really wanted) than others.

Sometimes concentrating on holding very still is enough.  If not, you might be able to sort it out via your mouse settings in Windows.

algebe say:

I've reduced the pointer speed, and the frequency of the rogue popup seems to have reduced. I'm also trying to avoid moving the mouse when clicking. neha singh msft say:

Hi JBM,

 

After you share a SharePoint site with a restricted domain, the user will get the email invitation to access the SharePoint site. And once the user click on the link in the email, it will ask the user to log in with Microsoft account or work or school account.

 

The external user with whom you wish to share the SharePoint site should create a Microsoft account using their favorite email address to access the site. To clarify this point, let’s take a scenario. From my Office 365 account, I added an email domain i.e. ‘Gmail.com’, as shown in the following screenshot.

 

 

And then I shared a SharePoint site to my Gmail account, I received an email invitation with the link to access the shared SharePoint site in my Gmail id. After clicking on the link, it asked me to sign in with Microsoft or organizational account. Since I received the email in my Gmail account, I must have a Microsoft account created by using my Gmail id. Once the account is created you’ll get the access to the site.

 

Restricted Domains sharing feature will allow end user to share SharePoint sites with only specific domains. If your end user attempts to share content with an external user whose email address domain violates your restricted domains settings, an error message will display and sharing will not be allowed:

 

 

 

Here’s an article with more details: Restricted Domains Sharing in Office 365 SharePoint Online and OneDrive for Business

 

Let me know if you have any doubt.
