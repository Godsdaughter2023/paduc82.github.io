The Count field determines how many duplicates of each original line to make in the appended data list and each time a line is created I want the Count field to be subtracted by 1 because the count turns into a Unique ID in the appended data set. How would I do thiis?

ken sheridan say: You don't need to append rows to a table for this; you can return the rows as the result table of a query.  First create a table named Counters with a single column of integer number data type named Counter as its primary key.  Fill the Counters table with values from 1 to at least the highest Count value anticipated in your existing table. You can then return the Cartesian product of the two tables, restricted to those rows where the  Counter value is less than or equal the Count value with:

SELECT Project, Mark, Counter, Length, Width
FROM YourCurrentTable, Counters
WHERE Counters.Counter <= YourCurrentTable.Count;

You can add an ORDER BY clause if you wish to return the rows in whatever order you want.

I would strongly recommend that you do not use Count as a column name, however.  It is an SQL keyword, and as such is a 'reserved word' which should not be used as an object name.

ken sheridan say: You are joining the tables.  Look at my example again.  You'll see that the tables are not joined.  This returns the Cartesian product of the two tables, in effect joining each row in one to each row in the other.  This is then restricted by the WHERE clause, so that the number of lines returned per row in the PieceTicketLines table is governed by the value in the mCount column for that row.

Also, if you wish to return sequential numbers per per row from PieceTicketLines, the Counter column from Counters needs to be included in the query's SELECT clause, not the mCount column from PieceTicketLines.
