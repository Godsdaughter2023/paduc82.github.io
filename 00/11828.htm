******************************************************************

 

If it really is occurring with downloading and not with installation (this would be unusual unless infected and then I wouldn't expect that error but some other error or problem), start with the following:

 

1.  Boot to Safe Mode with Networking and try downloading from there and see if that helps.

 

2.  If you have more than one computer and a way for them to talk (like a flash drive), try downloading to a different computer and then transferring the downloaded file to your desktop on the file where you can't download but on which you want to install MSE.

 

3.  If using IE, try opening it with no add-ons (Start / All Programs / Accessories / Internet Explorer (No Add-ons)) and see if the download will work that way.  If not or if using a different browser, then try another browser to see if that solves the problem.   Stick with one of the primaries like Firefox (this would be my first choice if choosing an alternate), Chrome, Opera, or Safari (or definitely IE if not using IE now).

 

I find it extemely difficult if it is a download and not an installation problem that the above would not resolve the problem.  If it is with downloading and none of those help, go directly to MS Support for assistance at: https://support.microsoftsecurityessentials.com/.

 

********************************************************************

 

If it is an upgrade or a first time installation, then please proceed as follows.  First, if it is an upgrade from Windows Update, forget that and do it manually as indicated below.  Some have troubles with the upgrade that are more related to update problems than with MSE problems - so let's just rule that part out entirely.

 

Here are general instructions for installing MSE (and please read and use them - don't just wing it because "you've done it before and know how to do it" - I know how it is and I do that myself way, way too often and usually it works out, but if it doesn't then I'm not sure exactly where I went wrong and have to start over again from scratch) if you get past that or if this helps you avoid that error:

 

To uninstall MSE (certainly if upgrading, but also who knows what's there after all these attempts if you got so far as installation rather than being stopped at downloading, so best to start from scratch), proceed as follows:http://answers.microsoft.com/en-us/protect/forum/protect_start/uninstalling-mse/a63b8c4b-58ed-437e-8086-fa08d80725a4 or try http://support.microsoft.com/kb/2483120.

 

To re-install, check out this article and follow ALL the procedures: http://answers.microsoft.com/en-us/protect/forum/protect_start/check-list-for-installing-microsoft-security/bf757e6a-e320-4a67-92bc-767e6acb26c4 - but also check out the next paragraph before you begin the download.

 

If you do or ever did have any other AV program installed on your computer (even if you didn't use it), you need to make sure they are all uninstalled and fully removed before you install. To fully remove them, check out this listing of removal toolshttp://social.answers.microsoft.com/Forums/en-US/msestart/thread/407bf6da-c05d-4546-8788-0aa4c25a1f91 or the listing in that article at the bottom under troubleshooting steps (why this link isn't at the top near things to do first where it discusses removing all the prior and current AV programs is beyond me) or, if not on the lists, go to the website of the AV software and get and use the removal tool for that product. You cannot just uninstall these programs – you MUST also use the removal tool – even if the uninstall was years ago but you didn’t use a removal tool at the time.  This means not only current ones but any EVER (including free trials you may never have used) or done years ago and possibly forgotten. See the following compliments of Dave__M: If you aren't sure of what may still be on your system as remnants, which could be interfering with the MSE install, try a scan of your system with this security application detection and removal specialist: http://www.appremover.com/.  Do not yet remove the one primary AV program you are using until you have downloaded the MSE file so you are protected while doing so – then uninstall and remove that program after the download but before beginning to install MSE.  This is the most likely cause (or doing this properly and completely is the most common "fix") and so pay particular attention to getting rid of every remnant of every previous installation of everything (except for the one until after downloading MSE and then be just as certain with that one as well). 

 

To install, go to http://www.microsoft.com/en-us/security_essentials/default.aspx and click on languages and locales and then choose your language and the appropriate version for your operating system. When you download, save it to the desktop (rather than running it). When finished, uninstall and remove your final AV program as indicated above, reboot, and then right click on the installation icon and click run as Administrator (even if you’re logged in as an administrator) as I’ve found this sometimes helps.

 

Then you need to configure your settings, set a scheduled scan, enable real-time protection, update MSE, and then run a quick scan (or a full scan if you have time).

 

I also recommend the following setup for MSE (though the decision is yours - I've just found this helpful and the most effective and best way to set it up and is how I have mine configured):

 

Make sure that ALL threat levels are sent to Quarantine where you can then decide whether or not to allow them or leave them there, or remove them. It is the safest solution for having the most options for dealing with the situation. Quarantine is as safe as removed, but in some cases you may want to remove them to be more certain - and with this one I would remove it (when the time comes). To do this, open MSE, click on the Settings Tab, and for each of the threat levels, set the action to be Quarantine and save.

 

Go to the History tab and click on allowed and unless you specifically placed something in allowed, remove everything else from the allowed category. Only put things there that you ABSOLUTELY know are not problems (are "false positives") and triple-check before you do it as doing this could allow a real threat to bypass security and infect you and it won't be detected or rather won't be reported or acted upon by MSE.

 

Go to the Settings Tab and click on Real-time protection and check all the boxes and save.

 

Go to the Settings Tab and click on SpyNet and click on Advanced and save. This is not at all dangerous to you and no information sent will be used by Microsoft to identify you even if some minor identification information is sent in terms of defining the infection, but it can sometimes help with new threats by initiating an immediate update to deal with the problem if it is in the SpyNet database (and it helps Microsoft update definitions to provide you and everyone else better protection).

 

Go to the Settings Tab and click on Advanced and check the box for save Quarantine items and change it from one month to three months so you have more time to decide if you want to restore something.

If that doesn't help, then there's a good chance you may be infected by malware. Please proceed as follows:

 

First, try the following programs if you haven't as they may help. Even if they or your current AV software seem to work or indicate you aren't infected, you shouldn't completely trust them and need to continue with the recommendations that follow (so it's up to you if you want to try them first or just skip over all of this and get the expert help you really need as described below).

Sub NumberColumn_R1()
    Dim ws As Worksheet
    Dim lastRow As Long
   
    With Selection.Font
        .Name = "Arial"
        .Size = 24
        .Color = 16750848 '-26368

        .Pattern = xlSolid
