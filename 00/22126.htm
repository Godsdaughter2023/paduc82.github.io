I'm not sure you can group and stack without using some code.

Some setup required:

- Assume fields Year, IncomeExp, Category, Amount all bound to text boxes in detail (can be hidden)

- IncomeExp contains either "i" or "e"

- No chart/graph control

- Detail section about 6" high

- Group footer section based on expression =1 and 6" hight

- Detail section has Back Color=#FFFFFF and Alternate Back Color: No Color

- Add a rectangle the size of the graph with transparent background

- Add rectangle named "bar" which will be every bar value displayed

Then add code to the On Format event of the detail section and General Declarations

    (just starter code ;-):

Option Compare Database
Option Explicit
Dim dblPreviousBottom As Double
Dim dblPreviousYear As Double
Dim strPrevIE As String

Private Sub Detail_Format(Cancel As Integer, FormatCount As Integer)
    Dim dblGraphFrameBottom As Double
    Dim dblGraphFrameHeight As Double
    Dim dblYScale As Double
    Dim dblMaxY As Double
    Dim dblBarHeight As Double
    Dim dblBarLeft As Double
    Dim dblGraphFrameLeft As Double
    Dim dblBarWidth As Double
    Dim dblBarGroupSpacing As Double
    Dim dblLeftMargin As Double
   
    If dblPreviousBottom = 0 Or dblPreviousYear <> Me.Year Or strPrevIE <> Me.IncomeExpense Then
        dblPreviousBottom = Me.GraphFrame.Top + Me.GraphFrame.Height
        strPrevIE = Me.IncomeExpense
        dblPreviousYear = Me.Year
     Else
       
    End If
    Me.MoveLayout = False
    dblLeftMargin = 300
    dblMaxY = 150000
    dblBarGroupSpacing = 2000
    dblBarWidth = Me.bar.Width
    dblGraphFrameHeight = Me.GraphFrame.Height
    dblGraphFrameLeft = Me.GraphFrame.Left
    dblGraphFrameBottom = Me.GraphFrame.Top + Me.GraphFrame.Height
    dblYScale = dblGraphFrameHeight / dblMaxY
   
    dblBarLeft = dblBarGroupSpacing * Abs(Me.Year - 2013) + dblGraphFrameLeft + dblLeftMargin + _
            IIf(Me.IncomeExpense = "I", 0, dblBarWidth + 30)
    dblBarHeight = Me.Amount * dblYScale
    Me.bar.Left = dblBarLeft
    Me.bar.Top = dblPreviousBottom - dblBarHeight  'dblGraphFrameBottom - dblBarHeight
    Me.bar.Height = dblBarHeight
    Me.bar.BorderColor = IIf(Me.IncomeExpense = "I", vbGreen, vbRed)
    dblPreviousBottom = Me.bar.Top
End Sub

prestonmitchell say:

Excel 2016 Error: "We couldn't free up space on the clipboard. Another program might be using it right now."

Excel 2010, 2013, & 2016 can all suffer from the error: "We couldn't free up space on the Clipboard. Another program might be using the clipboard."  

This Infamous Excel Error can prevent 'Cut/ Copy & Paste' and/ or Dragging & Dropping Cell Contents (to another cell).   It is one of the oldest and most misunderstood of all Excel Errors.   The average MSFT Office technician (Level 1 & 2) does not know how to fix the Excel Error.  They will perform a lot of frivolous 'repairs' that only provide a temporary fix at best.  That's why I decided to offer the ONLY solution.

The issue is caused by another software application installed on your PC that is incompatible with Excel.  Therefore, the Office 'repair' function won't fix the issue. Nor will uninstalling & reinstalling Excel.

Microsoft admitted that Zone Alarm and Bing Desktop Search will cause this Excel Error.  HOWEVER, the dirty Truth is that MANY software applications will cause the error.

There are only two 'fixes':

1. Uninstall Excel and find another spreadsheet program.

2. Discover what software program on your PC is incompatible with Excel. (That is a tedious ordeal in itself. And hopefully, you know how find the incompatible software.)

AFTER YOU DISCOVER THE SPECIFIC SOFTWARE CAUSING THE CONFLICT WITH EXCEL, you must do one or more of the following:

1.  Don't use the incompatible software while using Excel.  Simply close out the incompatible software before using Excel.  (That is the problem & fix for those who only suffer from the Excel Error infrequently.)

2.  Make sure that the incompatible software doesn't have a 'service' or 'process' running in the background (even though the software is not open).  Some software have background processes that start with Windows.   In this case, you must disable the background service or process.  (This is the likely problem for those who constantly suffer from the Excel issue.)

3.  Uninstall the incompatible software.

For me, the incompatible software was ToolWiz Time Freeze, which Excel 2016 apparently hates.  ToolWiz has a background service that starts up with Windows.  Disabling that background service solved the Excel Clipboard error.
