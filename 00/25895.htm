mattmees say:

Hopefully this will help - spent the last few hours trying to figure it out.

The problems in our case are related to an incorrectly configured Autodiscover service somewhere out there on the web.  We used to have our domain name hosted on an IMAP/SMTP server setup but moved it over to Office 365 about 3 months ago.   The problem only happens on clients running Outlook 2016 and the problems seem to be intermittent, disappearing for a short while on a clean profile build before eventually returning. 

After running several of the Microsoft troubleshooting tools I wasn't getting any great information but an OffCAT scan drew my attention to the Autodiscover.xml file that was created for the associated account under %localappdata%\Microsoft\Outlook\16

In my case, I have another Office 365 Exchange account (provided through GoDaddy) that does not exhibit the same behavior as the native Office 365 Exchange account that I was having trouble with.   Looking at the XML between the two AutoD documents it was obvious that the problem account was pulling incorrect settings.

To fix I did the following:  

Open Outlook with a working account or create a dummy POP profile with no information.   Whatever works to get Outlook open and running.   From there, locate the Outlook icon in the system status tray (small icons in the bottom right), hold down CTRL and Right Click.  Select "Test E-Mail AutoConfiguration"

Enter your email address and password and click the test button.  I had a popup asking to allow permission for AutoDiscover which I allowed, and then the test ran to completion.  If everything looks right (there are office365.com settings within the results) click over to the XML tab and CTRL+A, CTRL+C to select/copy everything in it.  

From there, go back to %localappdata%\Microsoft\Outlook\16 and open your problem account XML file in notepad or your favorite simple text editor (NOT Word).   Delete everything within and paste the new information you got from the AutoConfiguration Test.  Save and close.  From this point I was able to reopen Outlook with no problems.  I went ahead and made a backup copy of the XML configuration so I can easily replace it if Outlook decides to grab the wrong AutoDiscover information again.  

Hopefully I can track down where the bad information is coming from - if so I'll update this thread to hopefully help the next person.
