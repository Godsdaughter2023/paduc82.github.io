qa guy commelec say:

ken sheridan say: The possible return values of the IIf function are the Boolean logical expressions:

VIRAL_LOAD>69
VIRAL_LOAD>20 AND VIRAL_LOAD<>40
VIRAL_LOAD>25 AND VIRAL_LOAD<>50
VIRAL_LOAD>50

You can see this better if you structure the WHERE clause:

WHERE
VIRAL_LOAD=
     IIf([ASSAY_TEST_ID]="AB1",VIRAL_LOAD>69,
         IIf([ASSAY_TEST_ID]="OT1",VIRAL_LOAD>20 AND VIRAL_LOAD<>40,
             IIf([ASSAY_TEST_ID]="EQ",VIRAL_LOAD>25 AND VIRAL_LOAD<>50,VIRAL_LOAD>50)))
AND
(ASSAY_TEST_ID="AB1" OR ASSAY_TEST_ID="OT1" OR ASSAY_TEST_ID="EQ");

However, it appears that the VIRAL_LOAD column contains numeric values, so unless one of those values happens to be -1 or 0, which is Access's implementation of a Boolean TRUE or FALSE, then the WHERE clause will never evaluate to TRUE, so no rows will be returned, which is exactly what's happening. 

You need to rethink the logic of the WHERE clause.  I suspect all you need to do is remove the initial reference to the VIRAL_LOAD column like so:

WHERE
     IIf([ASSAY_TEST_ID]="AB1",VIRAL_LOAD>69,
         IIf([ASSAY_TEST_ID]="OT1",VIRAL_LOAD>20 AND VIRAL_LOAD<>40,
             IIf([ASSAY_TEST_ID]="EQ",VIRAL_LOAD>25 AND VIRAL_LOAD<>50,VIRAL_LOAD>50)))
AND
(ASSAY_TEST_ID="AB1" OR ASSAY_TEST_ID="OT1" OR ASSAY_TEST_ID="EQ");

In fact I think you can reduce it further as the final expression can never be returned by virtue of the restriction on the ASSAY_TEST_ID column in the last line, so:

WHERE
     IIf([ASSAY_TEST_ID]="AB1",VIRAL_LOAD>69,
         IIf([ASSAY_TEST_ID]="OT1",VIRAL_LOAD>20 AND VIRAL_LOAD<>40,VIRAL_LOAD>25 AND VIRAL_LOAD<>50))
AND
(ASSAY_TEST_ID="AB1" OR ASSAY_TEST_ID="OT1" OR ASSAY_TEST_ID="EQ");

I am making a bit of a leap in the dark here, so if not post back with more details of the underlying criterion in plain English rather than SQL.

How do I make the text in text box larger font in sway? - Microsoft Community

evelynvanderhoop53 say:

How do I make the text in text box larger font in sway?

How do I make the text in text box larger font in sway?
jason (office engineer) say:

Hi Evelyn,

Sway doesn't give you the ability to directly affect the size or positioning of content in Sway.  This is because that Sway generates a layout optimized for the screen/device that each individual user views it on and those types of settings wouldn't be able to be preserved across all layouts.  You can accent and/or emphasize pieces of text or entire text cards as way to hint that the text should be given more prominence in the layout and in some cases that will result in larger text.  One feature that will usually result in larger text is the button in the upper right of a text card to set its emphasis to moderate.  Definitely try out various combinations of these features though to get the output that you like best.

Hope this helps!
