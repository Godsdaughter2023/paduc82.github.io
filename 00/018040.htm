greginmich say:

Windows Defender is highly configurable in Windows 10, but the configuration can only be done via the PowerShell cmdlets or the Group Policy settings:

-DisableArchiveScanning<Boolean>

Indicates whether to scan archive files, such as .zip and .cab files, for malicious and unwanted software. If you specify a value of $True or do not specify a value, Windows Defender scans archive files.

The TechNet documentation for the -DisableArchiveScanning parameter is wrong. This parameter is actually set to False (Disable Get-MpPreference command in PowerShell. So in order to disable archive scanning, you would run this command at the elevated PowerShell prompt:

Set-MpPreference -DisableArchiveScanning $True

https://technet.microsoft.com/en-us/library/dn433291.aspx?f=255&MSPPError=-2147217396

Some of the Set-MpPreference settings appear to be either nonfunctional or dysfunctional, but this particular setting seems to work fine – and it had a very pronounced effect on the duration of my Full Scans when I tested it here:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender-taking-more-then-9-hours-and/c14bfaad-343d-40a9-95c9-5e4a00b363c2#LastReply
