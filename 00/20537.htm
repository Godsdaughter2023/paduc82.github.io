steve rindsberg say:
Thanks for sending the sample slides, Anne Marie.  

The problem is this:  current versions of PowerPoint include the ability to specify All Caps as part of the font formatting.  Earlier versions don't; and what they don't understand, they ignore, so when you open your file in an earlier version of PPT, the text reverts to whatever you originally typed.  In this case, that means lowercase text.

There's actually a bit of weirdness that makes the text lowercase even when you open the file in a version of PPT that supports All Caps formatting (2010, for example).

There's a little trick that might make it work better in earlier versions.  When you apply formatting that earlier versions don't support, such as beveled text, PPT turns the text into an image when you save it to an earlier format.  If you apply a very subtle bevel to your master text placeholders, the text (converted to an image) stays UPPERCASE when opened in e.g. PPT 2003.  Of course, because it's an image, it's no longer editable as text in earlier versions of PowerPoint.  That may or may not matter to you.  If not, it's a quick fix.

Otherwise, you'll need to retype everything as uppercase letters or use the text formatting button that lets you change between lowercase, UPPERCASE, Sentence case and so on.

Or perhaps we can write you a little macro that does the work for you.  Do you want to convert all of the text on every slide to uppercase, just the titles on each slide, or ???

How to filter subform from Main form combobox? - Microsoft Community

dale pittman say:

How to filter subform from Main form combobox?

I have seen countless posts on many forums.  But have not been able to figure it out yet.  I have a subform (ship_labor_data) embedded in a Main Form (Data Entry).  The Combobox on the main form (Emp_No) is supposed to filter the subform to employee numbers.  it works when it first loads.  but changing the data in the combobox effects no change.  I have record navigation buttons on the subform, and these work fine.  if I select next record it progresses through the filtered records based on the original combobox value.  The Master/Child links are set fine as the subform displays what it should.  I tried adding record nav buttons on the main form (Next & Prev).  When these are pressed the Emp_no combobox steps through the unfiltered fields and displays the coresponding Emp_No.  As the Emp_No changes the filltered subform changes, and if I press the subform nav buttons it steps through the filtered list. 

What I would like to have happen and what I thought does happen from reading posts and reference books is when I change the combobox value the subform would automatically filter.  It does not.  Any ideas on how I can get the subform to automatically filter when the combobox changes?

ken sheridan say: You can do both in one form, but you need to include an unbound combo box as well as the bound control, though if Emp_No is an autoumber in the parent form's underlying table there is no need for a bound control at all, just an unbound one.  What you need to do is navigate to the parent form's record with the Emp_No value which matches the value selected in the unbound combo box, which you do by putting the following in its AfterUpdate event procedure:

    Dim strCriteria As String
    Dim ctrl As Control
    
    Set ctrl = Me.ActiveControl
    
    strCriteria = "Emp_No = " & ctrl
    
    With Me.RecordsetClone
        .FindFirst strCriteria
        If Not .NoMatch Then
            Me.Bookmark = .Bookmark
        End If
    End With

Name the combo box cboEmp_No to distinguish it from the field name and in the form's Current event procedure, to keep the combo box in sync with the form's current record if you navigate other than with the combo box, put:

    Me.cboEmp_No = Me.Emp_No

The above assumes that Emp_No is a number data type.  If by any chance it is a text data type amend the code to:

    strCriteria = "Emp_No = """ & ctrl & """"

In the form's AfterInsert event procedure put:

    Me.cboEmp_No.Requery
    Me.cboEmp_No = Me.Emp_No

to add the Emp_No for a new record to its list and set its value to the new Emp_No.  Do the same in the form's AfterDelConfirm event procedure in case you delete a record in the parent form.

Assuming the subform is linked correctly to the parent form, as you navigate by means of the unbound combo box, or in any other way, it will be requeried to show only those rows which reference the parent form's current record.

You'll find an example of a form which does this as StudentLog.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

You might have to copy the text of the link into your browser's address bar (not the link location).  For some reason it doesn't always seem to work as a hyperlink.

The file is really intended to demonstrate how to build correlated subforms, but if you go to the Students form you'll see that it has an unbound  'Go to Student' combo box.  If you select a student in this the form will move to that student's record and its Registrations and Attendances subforms will be requeried to show the rows for the current student.  The form is more complex than you will need because it includes the two correlated subforms, but the combo box and the Registrations subform work in the same way as yours would.

BTW when you say 'Data Entry' I assume you mean for editing existing records as well as adding new ones, rather than the form's DataEntry property is set to True (Yes).  This property is rather confusingly named because setting it to True allows only new records to be entered, not navigation to or editing of an already existing record, so what I've described above would not work if the DataEntry property is True.  You'd need to set it back to False (No).

ken sheridan say: It's not very clear just what's being attempted here.  Is the Emp_No combo box in the parent form a bound control?  Your description tends to suggest that it is.  If so then changing its value will change the value in the underlying column in the table to which the parent form is bound, so you may be using a bound control inappropriately if you merely want to to restrict the rows returned in the subform.

If, on the other hand, the control is unbound and is intended to restrict the subform further within the set of rows already returned by virtue of the LinkMasterFields and LinkChildFields properties of the subform control, then if this restriction is by means of the subform's underlying query referencing the combo box control as a parameter, you will need to Requery the subform in the AfterUpdate event procedure of the combo box control for it to take effect.

If the later scenario is the case, i.e. the combo box control is unbound, you can achieve the same result by adding the name of the combo box to the subform control's LinkMasterFields property, separating it from whatever is already in the property with a semi-colon, and similarly adding the name of the Emp_No column in the subform's recordset to the subform control's LinkChildFields property.  The subform's query then no longer needs to reference the combo box control as a parameter.  

If you merely want to show rows in the subform which match the selected value in the combo box, and don't need to see any other data in the parent form, then you can use an unbound form as the parent form and simply set the LinkMasterFields property of the subform control to the name of the unbound combo box in the parent form, and its LinkChildFields property to the name of the Emp_No column in the subform's recordset.

In fact to do the latter you don't need to use a subform at all.  You can do it all in a bound form in continuous forms view, whose RecordSource is a query referencing an unbound combo box in the form's header, requerying the form in the combo box's AfterUpdate event procedure.

srinivasu guntimadugu say:

Hi Matt,

 

Thank you for choosing Microsoft Community. I will help you in providing the steps to save the excel file which is on a server or shared network.

 

Adding a download button in the e-mail is not possible. However we can insert a shape (E.g. a Box or a Circle) and create a hyperlink in order to open the Excel files from the Server/Network location. This will allow other users who have access to the server/Network to either open or save the Excel file.

 

Follow the steps:

1. Click on Insert tab -> Shapes in the body of the e-mail and create a desired shape.
2. Right click on the Shape -> Click on hyperlink.
3. Select Existing File or Webpage under Link to field.
4. Under Look In field select Current Folder -> Click on Browse for File below Text to Display.
5. Now browse to the Server or Network location -> Select the Excel file and click OK.

 

The recipients with access to the Server or Network location can click on the Shape from the e-mail received to Save or Open the file locally on the computer.

 

Note: The recipients whom the e-mail is sent should have the access to the Server or network location in order to open the file from the hyperlink.
