ken sheridan say: A multi-select list box has no value.  Instead it has an ItemsSelected collection, which is a collection of variants, each of which points to one of the items selected in the control.   It is necessary therefore to loop through this collection, and at each iteration of the loop, retrieve a value which can then be used to restrict the query's, and hence the report's output.  In my demo this code does this:

        With Me.lstInvoices
            If .ItemsSelected.Count > 0 Then
                For Each varItem In .ItemsSelected
                    Me.txtInvoiceList = .ItemData(varItem)
                    ' create folder if does not exist
                    varFolder_1 = varFolder & "\" & .Column(1, varItem)
                    MkDir varFolder
                    strFullPath = varFolder_1 & "\" & .Column(1, varItem) & " " & .ItemData(varItem) & ".pdf"
                    DoCmd.OutputTo acOutputReport, "rptInvoiceMultiple", acFormatPDF, strFullPath, True
                Next varItem
            Else
                MsgBox MESSAGE_TEXT_2, vbExclamation, "Invalid Operation"
            End If
        End With

What this does is assign each selected invoice number to the hidden txtInvoiceList  control in the form.  Because my demo allows for the report to include all selected invoices in one report, or to output each as a separate report, in both cases outputting the report as a PDF file, the report's query references the text box by means of Microsoft's InParam function.  This parses the text box's value into separate invoice numbers where it includes multiple invoice numbers as a value list.  As the above code is assigning each invoice number separately to the text box rather than all selected numbers  as a single value list, the use of the InParam function is not actually necessary in this context.  The txtInvoiceList  control could simply have been referenced as a parameter of the InvoiceNumber column in the query, in exactly the same way that you are referencing a conventional list box.  It's only because my demo includes the facility to output the invoices as a single or as separate reports that the InParam function is called.

So in your case you can use code like the above to write each EmployeeID to a hidden text box and simply reference that text box in your report's query rather than referencing the list box directly.
