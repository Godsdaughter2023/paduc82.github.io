Sub AutoSum(Cell As Range)
    Dim R As Long
    Dim C As Long
    Dim TopRow As Long
    Const Fx = "=Sum(#)"

'// Note:   The Behavior of End(xlUp) depends on if the starting cell has a value or not.
    With Cell
        R = .Row
        C = .Column
        .Value = "Dummy"
    End With
   
    If Cells(R - 1, C) = vbNullString Then
        TopRow = Cell.End(xlUp).End(xlUp).Row
    Else
        TopRow = Cell.End(xlUp).Row
    End If
   
    Cell.Formula = Replace(Fx, "#", Range(Cells(TopRow, C), Cells(R - 1, C)).Address(False, False))
End Sub

KRACK Wifi hack - Microsoft Community

elkhunter47 say:

KRACK Wifi hack

Does Window Defender provide VPN protection from WIFI hacks?
rob koch say:

As Solar-Eclipse mentioned, the Microsoft Wi-Fi clients had already been patched, even though current versions of the Windows operating system were already relatively safe from this particular attack.

The deeper question with routers though is whether any of these are using a bridge mode or other similar feature that uses the Wi-Fi client components, since this is that portion that's actually involved in the KRACK attack.

If your PC is only used at home, then this attack is unlikely, since it would require that someone be very close to your house with another Wi-Fi device and computer.  That means that only your next door neighbors (if houses are very close) or someone very nearby in a car could possibly attempt this attack.

If, though, your PC is a laptop or other mobile device, then the greater issue is when using Wi-Fi hotspots at restaurants, coffee shops or the airport, since you can't know whether these might by patched for this vulnerability nor whether they might be connected to each other via the Wi-Fi client, which is common when using the bridge ability to cover large areas like an airport or other large building.

Microsoft only provides a basic set of VPN client abilities in Windows itself, but doesn't specifically provide the public VPN relay server(s) that would be required to connect to the Internet from the other end of the VPN.  So you'd either need to use one of the (often questionable) public servers provided by others or look for other alternatives.

The best way to protect yourself beyond not using public Wi-Fi hotspots at all, is to confirm that anything you currently do when using public Wi-Fi is already encrypted, which is the safest in any case.  For example, your email should always be operating in the https secure SSL mode, since this provides encrypted security from your client PC all the way to the email server on the other end of the connection.  That should and typically already is true for anything like online banking or shopping apps like Amazon as well.

Only things like general browsing of websites, reading news or streaming music or movies are typically not encrypted today, though any critical authentication items like your login or other transactional items like buying a new song or movie, might best be done at home or in a known safe place if you aren't certain these portions are secured via https or similar encryption.

Everything I personally use is already encrypted, so this hack doesn't really concern me, but I understand how difficult it is for the typical consumer to know all of these things about their own email or other providers.  It would be worth your time, though, to learn whether all of these companies already perform such encryption of critical data like login or transaction (i.e. purchase) information or in the case of email or banking, everything transmitted, since that's the best policy to protect their customer's under any conditions.

Rob

scottb721---01 say:

I also had no sound when I first tried fitting the CA-160 manually to a single speaker. I tried FR+&- and FL+&- but got no phone audio. So I investigated how CA-160 operates. Hope this is helpful :-)

 

 

FOR AUDIO (all references to ISO wires are on the large black ISO connector)

 

Phone audio is ONLY output on TWO of the ISO speaker wires.

Phone audio +ve goes out the FR+ ISO (GREY) wire and phone audio -ve goes out the FL- (WHITE/BLACK) ISO wire.

 

The CA-160 also short circuits the FR- and FL+ ISO wires. What this does is place the FR and FL speakers in series so phone audio goes through both.

 

Both rear speakers get NO phone audio. The CA-160 simply open circuits them when it energises via the car kit mute wire.

 

If you are manually wiring to ONE dedicated phone speaker then use the FR+ and FL- wires.

 

FOR MUTE SWITCHING


The yellow mute wire going from the car kit to the CA-150 supplies an earth when the phone is being used. This earth goes through a 10K resistor and turns on a PNP transistor (2N3906). The transistor then supplies 12V from the fuse to three DPDT relays. These relays switch through phone audio and open circuits the rear speakers as described above.

 

The switching circuit would be very easy to fix if a component failed.

 

Looks much like this, http://www.rason.org/Projects/transwit/image1.gif

The +12v is from the fuse, Short to GND is the mute input and the 'Load' is the three relays.

jim gordon mvp say:

Hi

MERP is Microsoft Error Reporting.

If you choose Do Not Send when there is a crash, the crash report might be sent to the trash.

You can turn AutoRecover off for files saved locally in Word Preferences > Save

You can turn AutoRecover off for files saved to OneDrive or Sharepoint in the Ribbon

The database demon belongs to Microsoft Outlook. It performs functions such as storing your email messages, appointments, calendar invites, contacts, etc. It's a core part of Outlook.

The question arises - are you experiencing crashes? There are very few crashes these days, and most of them can be avoided by installing updates.

Make sure your Mac has a working internet connection before checking for updates. Install Mac OS X updates followed by Office updates.

 

Install Mac OS X updates first:

To get Mac OS updates, go to the App store on your dock, and then click the Updates button to reveal available updates.

 

Then Install Office Updates

To get the update:

  1. Open any Office application
  2. Go to the Help menu and choose Check for Updates
  3. Let AutoUpdate check for and install updates
  4. Restart your Mac

 

If you have any add-ins such as WebEx, Mendelay, EndNote, Zotero, TypeIt4me, etc it is essential that you install the latest updates for your add-ins.
