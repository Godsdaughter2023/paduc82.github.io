dhookom say:

200 fields kinda throws a wrench into most solutions. My typically method is to create two union queries (one from each table) and then join them in a third query. 

quniTable1

SELECT Item, "Color" AS [Column], Color AS [Value1]
FROM [Table 1]
UNION ALL SELECT Item, "Location",[Location]
FROM [table 1];

quniTable2

SELECT Item, "Color" AS [Column], Color AS [Value2]
FROM [Table 2]
UNION ALL SELECT Item, "Location",[Location]
FROM [table 2];

Then compare with a typical inner join:

SELECT quniTable1.Item, quniTable1.Column, quniTable1.Value1, quniTable2.Value2, [Value1]=[Value2] AS [Match]
FROM quniTable1 INNER JOIN quniTable2 ON (quniTable1.Column = quniTable2.Column) AND (quniTable1.Item = quniTable2.Item);

It might be best to write some vba with 2 DAO recordsets stepping through them and looping through fields. You could write the results to a tally table.

OneNote 2010: error code: 0x80070005 "A password is - Microsoft Community

boxerbenson say:

OneNote 2010: error code: 0x80070005 "A password is required to synchronise this notebook".

I have OneNote 2010 running on two PC's sharing the same SkyDrive notebooks. This has been the case for more than 6 months. Suddenly three notebooks on one PC will no longer sync due to error code: 0x80070005, "A password is required to synchronise this notebook".

All of the notebooks on both PC's are under the one Windows Live user account. Further more, some of the notebooks that do sync, are read-only on one of the PC's. There does not seem to be a fair dinkum answer on the internet so can a knowledgeable person from MS answer this cry for help?
nathaniel [msft] say:

I contacted boxerbenson directly and we were able to resolve the issue reported. The problem was two-fold. The first was resolved by our most recent update released on Nov 8, 2011.


The second issue was having more than one Windows Live ID credential saved in the Windows account that had access to the notebook. One of the Live IDs had read-only access (this is referred to as "Can view files" in the SkyDrive UI) and the other Live ID had read and write access. OneNote matches Live IDs with notebooks automatically but does not support having a mix of both read-only and read/write access Live IDs to the same notebook in the same Windows account. In that case we will choose the Live IDs based on the order they are given to us from Windows and on some machines we will use the Live ID with read-only access. The resolution is to either remove the Live ID with read-only access (through the Windows Credential Manager feature under the Control Panel) or change the sharing permissions on SkyDrive to give both full access to the notebook.
