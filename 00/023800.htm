jay freedman say:
First, the ability to change the language for each individual source was not removed in Word 2010. Here's what I see:
  • In the References tab, click Manage Sources.
  • Select a source in the Master List column.
  • Click the Edit button, which opens the Edit Source dialog.
  • Click the Language box at the top right of the dialog and choose the language.
  • Click OK.


There's a quicker way, though. In the Source Manager, click the Browse button above the Master List. That opens an Open Source List dialog pointing to the Bibliography folder within your profile, and there will be a file named Sources.xml.


Right-click the Sources.xml file and click Edit, which opens the file in Notepad. Immediately click File > Save As and save to a different name in the same folder -- I suggest Sources_English.xml. You must put quote marks around the file name to prevent Notepad from adding the .txt extension to the end of it.


Then use either the Find function or the Replace function... find each occurrence of the text

</b:Source>

and insert to the left of it the text

<b:LCID>1033</b:LCID>

the result should look like this:

<b:LCID>1033</b:LCID></b:Source>

Save the file and close Notepad. In the Open Source List dialog, select this new file and click OK. You should now find that all the sources are marked as English US.

In my previous post, I had assumed that your sources didn't already have a language assigned to them, and that the Language item in the Edit Source dialog showed "Default". I think the xml method didn't work because you probably now have entries that look like

with both languages specified, and it's only reading the first one. If you have the Sources.xml file open in Notepad, you should be able to use Find/Replace to replace

with nothing, all in one Replace All action. That should leave only

at the end of each source's entry, which would make the Language item say English (US).

vasil l. michev say:

Well it really depends on how easy it is to filter those users. The following cmdlet will return any user that has an UPN, SMTP or SIP address associated with the stu.school domain:

Get-MsolUser -DomainName  stu.school -All

Another possible filter is the -Department one

Get-MsolUser -Department Student -All

If this list includes all the users, you can easily block them via:

Get-MsolUser -DomainName  stu.school -All | Set-MsolUser -BlockCredential $true

But it's very likely that the list will include a bit more (or less), so you might have to use a more specific, client side filter. Such as:

Get-MsolUser -All | ? {$_.UserPrincipalName -like "*** Email address is removed for privacy ***" -and $_.Department -eq "Student"}
