In the line

 

        .ActiveSheet.Range(Cells(1, 16), Cells(999, 16)).Select

 

ActiveSheet refers back to appExcel through myWorkbook, but the two instances of Cells don't, so they will cause an extra instance of Excel to be created. Change this line to

 

        .ActiveSheet.Range(.ActiveSheet.Cells(1, 16), .ActiveSheet.Cells(999, 16)).Select

 

Similarly, in the following lines, Selection does not refer back to appExcel. Change ALL five occurrences of Selection to appExcel.Selection

#1 - EDA = Email DisplayAs

#2 - The external process in this scenario is the Outlook import process which is what is populating the DisplayAs (EDA) field. Unfortunately, it does not do it the same way as when the contact is created via the UI.

Given the above, knowing how the contacts are created, your options are as follows:

#1 - Using your existing data:

a) - Open your CSV file in Excel, add a column (i.e. DispAs), and populate it with the value that you want to see in the contact record
b) - Save your CSV file and when importing the file, specifically map the new field (DispAs) to the Outlook Email DisplayAs field

#2 - The Email DisplayAs standardization function in ContactGenie Toolkit (http://www.contactgenie.com/toolkit_4_outlook.htm ) may be of interest. It will enable you to set the EDA field to any of the formats provided

Whichever of the aboe two approaches you use - the DisplayAs field will contain the value you want and your Distribution Lists will also correctly reflect the field when those contacts are added.
