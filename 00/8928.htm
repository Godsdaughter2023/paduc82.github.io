bitflung say: [edit: i just re-read the orig. post and my response: sorry, didn't notice you're a programmer - most of this is common knowledge to programmers. i'm leaving the full response however, since non-techs may search, find, read, learn, act with this rather complete description of components and simple recommendation for starting to hack existing plugins - the iSync plugin maker another post refers to is an excellent point i failed to bring up as well. - bit]

iSync support has 2 components:
1. support in the phone
2. support on the mac

for part 1, look here:
http://www.forum.nokia.com/devices/7210_Supernova

search for "syncml" find it? yes you did. this means the phone has support for SyncML, tis is a standard for interfacing with synchronization servers, similar to HTML and XML, which you likely have heard of: this means the PHONE has support for SYNCML. this is all the phone needs to provide.

for part 2 look here:
http://www.apple.com/macosx/features/isync/
and here:
http://europe.nokia.com/mac/isync/

these are lists of phones with plugins available - a plugin for isync is basically (perhaps _actually_ by definition of the spec) a dictionary, providing the translation between the SyncML objects defined by the phone, and the SyncML objects defined by the iSync software. SyncML is based on XML, where the X stands for eXtensible - arbitrary objects can be defined as developers wish, but with a standard form. the plugin is required so the developers' potentialy arbitrary object types can be translated between the two devices (nokia --> mac).

the first link above shows that Apple did not ship a plugin for the 7210 SuperNova in MacOSX 10.5. the second link above shows that Nokia has not yet provided a plugin for the same.

in otherwords, if you owned the phone today and plugged it into your mac (or via bluetooth) iSync would NOT WORK. sorry. but, both the mac and the phone have built in support for this synchronization process. all you need is a plugin (as compared to phones without SyncML support, for which no plugin could every be written to provide direct iSync->Phone communication).

---

so, you need a plugin. searching around, i haven't seen anyone else providing 3rd party plugins for you (the architecture is a rather simple standard and companies like NovaMedia have been selling plugins for otherwise plugin-less phones for a long time now). this means there is no (immediately available, from what i can see) place for you to buy a plugin either.

you're left with 3 options:

1. buy the phone, wait (hope) for someone else to write a plugin and either download for free from nokia, or buy from novamedia, etc.

2. buy the phone and live without isync, or work on writing your own plugin. start by downloading all the plugins available from nokia, and look for the differences. when done, either sell the plugin or make it freely available for other SuperNova users (the later is recommended).

3. give up. don't buy this phone. look for another (nokia?) phone that has isync plugins available. buy that phone and sync to your heart's content.


goodluck,
-bit
