ken sheridan say: I think you want this:

DSum("[Amount]","Tbl1- JDEREPORT","[PrjNbr]='" & [PrjNbr] & "' And [g/l date]>= #1/1/2018#  And [g/l date]< #01/01/2019#")

Note how the date range is defined as on or later than the start date, and before the day following the end date.  This is safer than defining the end date inclusively as it allows for values in the column with a non-zero time of day element.  Unless you have specifically disallowed such values in the table definition you cannot confidently assume that no such values exist.

ken sheridan say: I think I see the problem.  There are unmatched quotes delimiters in the DSum function call's criteria expression.  The query also exhibits other issues:

1.  There is no need to include the expressions in the GROUP BY clause.

2.  The use of the NOT LIKE operator on integer number values is inappropriate.  It should only be used on string expressions where you wish to undertake pattern matching.  In this case the <> operator should be used.

3.  The unnecessary parentheses which the Access query designer puts in the WHERE clause can be discarded.  They are only necessary where some Boolean operations need to be evaluated independently of others.

Putting all this together gives us:

SELECT [Tbl1- JDEREPORT].PrjNbr,
Sum([Tbl1- JDEREPORT].Amount) AS SumOfAmount,
DSum("[Amount]","Tbl1- JDEREPORT","[PrjnBR]='" & [PrjNbr] & "'") AS Expr1,
DSum("[Amount]","Tbl1- JDEREPORT",
"[PrjnBR]='" & [PrjNbr] & "' And [g/l date]>=#1/1/2018# And [g/l date]<#11/1/2018#) AS Expr2
FROM [Tbl1- JDEREPORT] INNER JOIN [QryA1-CBRE-Projects]
ON [Tbl1- JDEREPORT].PrjNbr = [QryA1-CBRE-Projects].[Project Number]
WHERE [Tbl1- JDEREPORT].[Object Account] <> 3090
AND [Tbl1- JDEREPORT].[Object Account] <> 3020
AND [Tbl1- JDEREPORT].[Object Account] <> 1350
GROUP BY [Tbl1- JDEREPORT].PrjNbr;

I would also recommend that you do not accept default column headings like Expr1 and Expr2, but change them to something more meaningful.

mike h.. say:

Hi,

 

I ran this on 5000 rows and it was virtually instant.

 

Sub Hide_Dates()
 Dim MyRange As Range, c As Range
 Application.ScreenUpdating = False
 Application.Calculation = xlCalculationManual
 Set MyRange = Range("H2:H4436")
 MyRange.EntireRow.Hidden = False
 For Each c In MyRange
     If IsDate(c.Value) And c.Value < Date Then
         c.EntireRow.Hidden = True
     End If
 Next
 Application.ScreenUpdating = True
 Application.Calculation = xlCalculationAutomatic
 End Sub
