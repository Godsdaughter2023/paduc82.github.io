Method 1: Run the network troubleshooter and see if the issue persists.
http://windows.microsoft.com/en-US/windows7/Using-the-Network-troubleshooter-in-Windows-7
 
Method 2: Release and Renew the IP(Internet Protocol) Address and see if the issue persists.
a. Click Start. Type  cmd in the Start Search box, and then press ENTER.
b. At the command prompt, type the following command, and then press ENTER:
ipconfig /all (there is a space between ipconfig and /)
The ipconfig /all command displays Windows TCP/IP settings for all your network adapters.
 
If your IP address for your Local Area Connection is either 0.0.0.0 or 169.x.x.x (where x is any number), then your computer is not getting an IP address from the router. 
If that is the case, try this:
·        ipconfig /release in the command prompt and press ENTER
·        ipconfig /renew in the command prompt and press ENTER
 
Test it again by typing ipconfig /all to see what the address is?  (If you are on a home router, it should start with either 192.168.1.x or 192.168.0.x or 10.x.x.x)

You can synchronize your computer clock with an Internet time server. considerate_guy say:

Yes, this is expected.

When you add a media item to your storyboard in Movie Maker, you are just adding its filename and location into the project.

As you preview the project, Movie Maker dynamically loads that clip into memory and shows you the portion of the clip you are playing.

This is a common misconception that you are "importing" a copy of the media directly into the storyboard. You are not. Instead it is just a reference back to the original source file. So if you do anything to that original file (move, rename, delete), Movie Maker can no longer preview it when you press play.

I like to use an analogy of a project file being a recipe with a list of ingredients and their exact locations. For example it might say you need 2 cups of flour taken from the middle shelf of the pantry and 1 egg from the refrigerator.

Now let's say someone moved the flour into the garage and used up all the eggs. At this point, the recipe can't be used to make a cake anymore.

Thus you always want to maintain your media in the original location until you are completely done with your project. At that point you tell Movie Maker to actually pull all the parts together and to create a new movie from the individual ingredients. That's when it finally makes a copy of everything.

Yes, this is normal. A project file (.wlmp) contains no actual copies of the media. It's just a text file with a *list* of the media items and if those source files are modified, then Movie Maker can't preview the project anymore.

This is how all popular video editors work. By not making full copies of the original media each time, they are more efficient and don't consume lots of harddisk space as you are making changes in your project. The final rendering (saving of the *movie*) is when the copying happens. At that point the program only copies the relevant portions that you want to keep so again it is more efficient.

I hope that clarifies what is happening.

kenmwong say: Thanks a ton. This indeed helps a lot but if I may ask. How do I export/transfer/isolate the finished project so that I can finally clear up some space? In other words I am done my project but I want to move it somewhere else so I can delete everything and then transfer the final project back just for viewing purposes. 

considerate_guy say:

If you only want to view the final movie, then you just need the final .mp4 file that is created. Think of it like the cake that is created from all your ingredients and the recipe. You can share that with your friends as you like.

Assuming you then have no future need for the source media (photos, video, audio) you can take them off your hard disk. You can also get rid of your project file. But this should only be done if you never plan to make different edits and no longer want the full clips. This would be like getting rid of the ingredients in your kitchen and throwing away the recipe.

If you still think you could make further edits later on, but don't want them taking up space on your hard disk, archive everything off to an external hard drive or something else that can store the media and the project file(s).

To help with this process, I like to organize everything related to a project into a folder before I start. I put all the video clips, photos and audio files into that folder (or subfolders). I also create the project file and save it in this folder. Knowing that everything is self-contained in the single folder means that I can archive it and I'm not having to search in numerous places on my hard disk for everything. Similarly, if I want to work on the project again, I restore the subfolder contents and I have everything I need in one place.
