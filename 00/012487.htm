toby tu msft support say:

Hi Manhattan,

 

This is a server error, as the message says. The usual cause is not that the mailbox is undergoing maintenance (which might happen very occasionally, but only for a few minutes or an hour or two), but that some other device has an active connection to the mailbox. There can only be one POP3 connection at once. If you're accessing this same mailbox from a mobile device that's configured for push mail, it will monopolize the mailbox and other clients - like Windows Live Mail - won't get a look in.

 

Because the server response is interpreted as an authentication error, it is presented to the user as a fault with the username or password, but this isn't always the case.

The waiting period will not be waived.

Even when you choose to delete your out-of-date security info, MS don't fully remove it for 30 days.

So long as you have the data in a long list with a single row of column headings

 

Name, Address, City, State, Zipcode

 

and all the data underneath then you can mail merge to labels using Microsoft Word.  In Word 2003 click on Tools -> Letters & Mailings -> Mail Merge and select Labels as the type of document.  When you go to the next step you can click on Label Options to set the size of your labels and on the Select Recipients step you can click Browse to select the spreadsheet you have created.  When you have selecte the recipients you can click on the first label and insert an Address Block or a combination of individual field, then click the Update all labels button to copy this to the rest of the sheet.  The wizard should then take you through previewing the labels, completing the merge and then you can click the link to print them.
