robert aldwinckle on forums say:

tried connecting and the fonts turned yellow. Couldn't connect.

Thanks for those details.   We can try working from there.  I don't have any details about how it is supposed to work but can use my observations about how it seems to work to try to help you.

The first thing that I would do then is use that Find other...  link (at the bottom of the dialog).   It should open in Bluetooth & Other devices.  If you already have an instance of  that name MANDster_... in there remove it wherever you see it.  It could be under both Audio devices and Display devices.   Then try again.

Is that other device you are showing at all related?  E.g. a native port in the same TV?  Then I would wonder about its state and the state of the adapter's device in that box and would try making changes along that tack.   Remove it from the same Settings display if it is there too.

Next I would get into the Control Panel's Network adapters display.  One way to do that is to right-click on the Network icon (Wi-Fi or Ethernet) in the Taskbar and choose...  Ugh.  They have changed this and made it harder to describe.  Try using the Troubleshoot problems item first then.  Get into the list of Network adapters.  If you see something called Local Area Connection*... it may be the one that is being used to represent your MWDA.  Then either select that or diagnose them all.  I'm not particularly hopeful that this will do anything but it should be there.  More significantly if it isn't there it would indicate a problem in one of the undocumented state changes that we need to try to deal with to try to make this thing work.

Another way to check on the same thing but in your case it may not be there yet is to use PowerShell  Get-NetConnectionProfile.  For example, I can see my Local Area Connection*... thing listed there as an InterfaceAlias.  Then that way you would also know the SSID being used with your MWDA and you could also use another PowerShell command to get more details about your connection.  For example, here's how I would list mine 

Get-NetAdapter "Local Area Connection* 17" | fl

and then the clue that I would be on the right track is 

InterfaceDescription       : Microsoft Wi-Fi Direct Virtual Adapter #4

There is more but see what you can do with this.  If you're lucky the Remove device and the Troubleshooter steps may be enough to get it working.

 

HTH

Robert

malkeleah say: You don't need to run the XP Network Setup Wizard. Just allow File/Printer Sharing on the Windows Firewall Exceptions tab.

In Windows 7, go to Control Panel>All Control Panel Items>Network and Sharing Center. Click on "Change advanced sharing settings". You don't want to use Homegroup unless you have all Windows 7 machines. If you do and you want to use Homegroup, see Windows 7's Help & Support. Otherwise, in the Advanced Sharing:

Turn ON network discovery
Turn ON file and printer sharing
Turn ON sharing in the Public folder sharing section
Turn ON password protected sharing

A. Configure firewalls on all machines to allow the Local Area Network (LAN) traffic as trusted. With Windows Firewall, turning on Windows 7's File and Printer Sharing as above will take care of this for you. If you aren't running a third-party firewall or have an antivirus/security program with its own firewall component, then you're fine.  With third-party firewalls, I usually configure the LAN allowance with an IP range. Ex. would be 192.168.1.0-192.168.1.254. Obviously you would substitute your correct subnet. Refer to any third party security program's Help or user forums for how to properly configure its firewall. Do not run more than one firewall.

B. For ease of organization, put all computers in the same Workgroup. This is done from the System applet in Control Panel, Computer Name tab.

C. Create matching user accounts and passwords on all machines. You do not need to be logged into the same account on all machines and the passwords assigned to each user account can be different; the accounts/passwords just need to exist and match on all machines.
Start>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).

XP - Configure Windows to Automatically Login (MVP Ramesh) - http://windowsxp.mvps.org/Autologon.htm

D. If one or more of the computers on your network is XP Pro or Media Center, turn off Simple File Sharing (Folder Options>View tab).

E. Create shares as desired. In Windows 7 I usually share out the user's Desktop and the Public directory.

F. After you have file sharing working (and have tested this by exchanging a file between all machines), if you want to share a printer connected locally to one of your computers, share it out from that machine. Then go to the printer mftr.'s website and download the latest drivers for the correct operating system(s). Install them on the target machine(s). The printer should be seen during the installation routine. If it is not, install the drivers and then use the Add Printer Wizard.
