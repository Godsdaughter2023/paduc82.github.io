You will need to download the Windows 10 Pro 32 bit ISO, create a bootable copy, then boot from it then perform a custom install.

Step 2:

After you have downloaded the Windows 10 ISO, you need prepare it, this is especially important for Windows 7 users..

After obtaining the .iso file you can use the built in Disc Image burning tool or Microsoft .iso to USB/DVD tool to create a bootable DVD or USB (requires a blank DVD or USB flash stick of at least 4 GB).

After downloading the .ISO file, right click it and click Burn disc image

Insert a blank DVD

Click Burn

Windows 8/8.1 users can mount the .ISO file and start the upgrade automatically. Learn more here

Important:

If you are creating a bootable USB thumb drive, make sure the thumb drive is first formatted as FAT32.

Connect the thumb drive

Open Computer

Right click thumbdrive

Click Format

Make sure FAT32 is selected as the file system.

For UEFI based systems

If your computer is UEFI based, these are normally systems that come pre-loaded with Windows 8 or later, you will need to prepare the ISO file for such a configuration or you will receive an error message during setup. The thumbdrive needs to be formatted as FAT32 and use the GPT partitioning scheme. To do this, you need to use Rufus, a small tool you can download for free.

 

After you have installed Rufus:

  • Launch it
  • Select ISO Image
  • Point to the Windows 10 ISO file
  • Check off Create a bootable disk using
  • Select GPT partitioning for EUFI firmware as the Partition scheme
  • Choose FAT32 NOT NTFS as the File system
  • Make sure your USB thumbdrive in the Device list box
  • Click Start
  • Close when complete

Step 3: How to: Perform a custom installation of Windows

OEStandardProperty:$DATA - Microsoft Community

adb9122 say:

OEStandardProperty:$DATA

Hello,

 

I am successfully able to backup my email messages onto my Desktop in its own folder.  The problem comes when I try to copy/move the files or folders to a flash drive (I have tried two different flash drives).  The error message goes:

 

"The file [filename.eml] has extra information attached to it that might be lost if you continue copying.  The contents of the file will not be affected.  Information that might be lost includes ":OEStandardProperty:$DATA".

 

I've scoured the internet, but haven't found any information about this error.  If anyone can explain what this means and more importantly, how I can copy the messages without losing any data, I would greatly appreciate it.

 

Thank you for your time.  All the Best.

¡firedog say:

Your desktop hard disk drive is NTFS-formatted. This makes it possible to write extra data to a file, and Windows uses this function to write some file properties for .eml files to what's known as an Alternate Data Stream (ADS). These properties are among other things the date the message was sent, who sent it, who to and its subject. This makes it quicker and easier to use Windows Search to find the message. Each ADS has a name, and that's what you're seeing in the warning message.

Your flash drive is not NTFS-formatted, so it doesn't understand ADS. The extra data will be lost when you save the file to your flash drive. It just means that when you move that file back to your hard disk drive, Windows will index it again and write those properties back. You won't notice any difference at all, except that Windows Search may not find the file until some time (hours or days) after you moved it to the hard disk.

You can prevent the warning message from appearing by NTFS-formatting the flash drive.
