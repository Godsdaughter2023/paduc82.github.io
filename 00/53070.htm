mohammad _ imran say:

Hi Tia,

 

A Master Boot Record (MBR) is a special type of boot sector at the very beginning of partitioned computer mass storage devices like fixed disks or removable drives intended for use with IBM PC-compatible systems and beyond. The MBR holds the information on how the logical partitions, containing file systems, are organized on that medium. Besides that, the MBR functions as an operating system independent chain boot loader in conjunction with each partition's Volume Boot Record (VBR).

 

MBRs are not present on non-partitioned media like floppies, super floppies or other storage devices configured to behave as such.

 

The concept of MBRs with storage volumes now commonly exceeding 2 TB, it has become a limiting factor in the 2010s. The MBR partitioning scheme is therefore in the process of being superseded by the GUID Partition Table (GPT) scheme in new computers. A GPT can coexist with a MBR in order to provide some limited form of backward compatibility for older systems.

kevind79 say: I was able to find a solution thanks to "johnbl" on a tomshardware.com forum page: http://www.tomshardware.com/forum/1262-73-computer-stay-asleep

"it could be that it actually does not go to sleep because software services and refuse the sleep request. This happens when you run something that shares a video stream for other devices. Like Windows Media center. 
you can run powercfg.exe /Energy as a admin and look at the report in your c:\windows\system32\error-report.html file and it will list the service that told it not to shut down. Then you have to make configuration changes. (stop sharing your media for example)"

When I did this, the file it created was actually called energy-report.html. It wouldn't open it in that folder, so I copied the file to the desktop and opened that.

This is the relevant item I found in that report:
System Availability Requests:System Required Request
The program has made a request to prevent the system from automatically entering sleep.
Requesting Process \Device\HarddiskVolume2\Program Files\Windows Media Player\wmpnetwk.exe

An Internet search for that filename revealed that this is the Windows Media Player Network Sharing Service. I opened the Services tool under the Administrative Tools control panel and found this service, which was running and configured to start automatically. I stopped the service and changed the startup type to Disabled. I left my computer in sleep when I left for work in the morning and it was still asleep when I got home. This is the first time I have been able to do that since upgrading to Windows 8, so I am pretty confident that this fixed the problem.
