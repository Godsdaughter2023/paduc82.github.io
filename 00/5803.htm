First of all: this is incorrect design. You have created a "Repeating Group" which is a violation of proper relational design principles. I'm sure it's a gift that will keep on giving until you fix that.

But it's not the cause of your parameter prompt.

I think there may be confusion between the column name, and the caption. Design the table *in the back-end database*. Is the field name truly "ID" and the table name truly "tblUser"?

While you're in design view, also check the Validation Rule, Filter, and Order By properties. Set Subdatasheet Name to [None].

ken sheridan say: As Tom points out, this is a bad relational database design as data is being encoded as column headings. A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.

The following is the model form one of my online databases which shows how the many-to-many relationship type between users and groups should be modelled by a table which resolves it into two one-to-many relationship types:



You'll find the file as SecDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file shows how 'row level security' on the basis of user groups can be implemented in Access, though note the 'public health warning' on opening any of the files about the loose meaning of the word 'security' in this context.

A query to return group membership, and which might be used as the basis for a report would be:

SELECT  UserGroups.UserGroupNumber, UserGroups.UserGroupFlag, Users.UserName
FROM UserGroups INNER JOIN (Users INNER JOIN UserGroupsMembership
ON Users.UserName = UserGroupsMembership.UserName)
ON UserGroups.UserGroupNumber = UserGroupsMembership.UserGroupNumber
ORDER BY UserGroups.UserGroupNumber,Users.UserName;

Though, rather than a single report, I would use a user groups report within which is embedded a multi-column users subreport for better presentation.  The ClubMembership demo in my same OneDrive folder is an example of this.  In the parent report's detail section the subreport is positioned immediately to the right of a control with an expression as its ControlSource to show the address.  The subreport shows the residents at each address in a horizontal layout, wrapping to the next line if the page width is insufficient for all residents at the address.  Note that a report like this should be opened in print preview or sent to a printer, not opened in report view.

In the same file is another report rptAddresses_Code which produces the same results in a single report by manipulating the layout in code in the report's module at runtime.  Ignore this, however, other than as a technical curiosity.  I only produced it in response to a question by a reader of a databases column written by a contact of mine in a magazine.  For some reason the reader specifically wanted a code solution.  A code-free report/subreport solution is far simpler.

john korchok say:

The One Page setting puts you in Print Layout view with the entire page visible. Page Down then goes down precisely one page at a time. It's pretty reliable. If your text is "vanishing to the right", you did not choose One Page, or your computer has extremely serious video configuration problems.

Calibri 11 is only Word's default if you have not modified your Normal.dotm template. It can be anything you want, including Time New Roman 12. To set this, follow these steps:

  1. Open a new blank document in Word.
  2. Right-click on the Normal style in the Style Gallery and choose Modify.
  3. Set the font to Times New Roman and the size to 12 points.
  4. At the bottom of the dialog, check New documents based on this template. OK out.
  5. When closing Word, if it asks if you want to save Normal.dotm, choose Save.

By the same token, Word default page size is determined by the page size that Normal.dotm is formatted with. To set this, you actually have to open Normal.dotm. Follow these steps:

  1. In Word, choose FILE>Open>Computer>Browse. The File Open dialog appears.
  2. In the File name field, paste %userprofile%\Application Data\Microsoft\Templates
  3. Select Normal.dotm and open it.
  4. Use Page Layout>Page Setup>Size to set your preferred page size. Then set your preferred margins.
  5. Save Normal.dotm and close. All new documents based on Normal will be in your page size with your margins.
