joeu2004 say: Steve wrote:
I have a Monte Carlo workbook that was written for 2003, but when I use it in 2010 I get different answers each time I run it, even when using a fixed seed (ie. same sequence of pseudo random numbers). Is this a problem with the .calculate vba instruction, how can I force it to calculate the same order of cells each time it recalculates?

IMHO, your design should not depend on Excel's order of calculation of independent formulas.

(The order of calculation of interdependent formulas should be no different, unless you purposely have circular references and you depend on iterative calculations.  Not saying I know of any difference for the latter; but I know that iterative calculations are "eccentric" in general.)

If you do want to control the order of independent calculations, I would suggest that you use a sequence of Range(...).Calculate or Sheets(...).Calculate statements instead of Application.Calculate.

(I presume you have set manual calculation mode.)

You might also study http://www.decisionmodels.com/calcsecretsc.htm to see if offers any insights.  Notably, you might want to disable multithreading.

(I vaguely recall that there was some change in by-row v. by-column order of calculation of independent cells.  And I thought there is some way to control that order in VBA.  But I cannot find that information at the moment.)

-----

But I wonder if your assumptions about "a fixed seed" and "sequence of pseudo-random numbers" might be the culprit.

We cannot seed the PRNG used by the Excel RAND and RANDBETWEEN functions.

We can seed the VBA Rnd function.  But that would not affect the .Calculate method unless you are returning VBA Rnd results using a VBA function called from Excel formulas.

If that is the case, changes (corrections) in the Excel calculation engine have changed the number of times a formula is recalculated in some cases, ergo the number of times referenced VBA functions are called.  So that might alter the sequence of random numbers.

If you would like to discuss your assumptions about Excel/VBA PRNGs, please post some details about how you are generating the "same sequence of pseudo-random numbers".

ossiemac say:

Try the following:-

 

Sub Macro2()
    Dim rngConditFormat As Range
   
    With Sheets("Sheet1")
        Set rngConditFormat = .Range(.Cells(2, "A"), .Cells(.Rows.Count, "A"))
    End With
   
    With rngConditFormat
        .FormatConditions.Add Type:=xlExpression, Formula1:="=AND(A2<>"""",A2<B2)"
        .FormatConditions(rngConditFormat.FormatConditions.Count).SetFirstPriority
        With .FormatConditions(1).Interior
            .PatternColorIndex = xlAutomatic
            .Color = 255
            .TintAndShade = 0
        End With
        .FormatConditions(1).StopIfTrue = False
    End With
End Sub

 

Added with edit:

The formula for the conditional format ignores blanks in column A. If you do not want it to ignore the blanks and treat blank as less than any value in column B then change the formula expression line to  the following:

 .FormatConditions.Add Type:=xlExpression, Formula1:="=A2<B2"

Hi Jeremy,

Here is some quick background on how ONENOTE works. This is primarily a cloud based application. All your notebooks associated with your primary account (Microsoft account) gets stored in the cloud.  Once you install the app on your device the following happens.

  1. After sign in

  2. We talk to the cloud, download all the notebook information/ data down to your device. This is the initial syncing process. So if you have a notebook in the cloud which is about 100 MB, we download that data to your local device.

  3. With all your data now locally available on the device, we then sync (up to cloud storage or down to your device) only for those changes that we make. This process usually depends on what changes you make. If you insert a picture from your device which is about 10 MB then we sync that data to you cloud service

Opting into sending user data will not eat up your usage data. Like I mentioned the initial download would probably use most of your data usage, depending on the notebook size.

Start Button not working and ADSL modem/router frequently dropping - Microsoft Community

psybangas say:

Start Button not working and ADSL modem/router frequently dropping out after mid February update.

Okay so I've been running Windows 10 Technical Preview on 3 different devices (a HP laptop, desktop PC and HTPC), for a couple of months now with no real problems. Everything seems to run smoothly, and each device successfully upgraded to the latest build (that I'm aware of, being here in Australia) Build 9926.

Anyway, my troubles began around mid February when what seemed like a routine update suddenly began to cause problems with my home ADSL connection. I'm talking frequent drop outs, i.e. my Billion 7800N modem/ router began loosing connection with my ISP and reconnecting (only to disconnect soon after). It was very frustrating, and in particular I noticed that I'd lost Internet connectivity completely on the desktop PC that was connected via Ethernet cable.

Initially I thought it was trouble with my ISP and so began the tedious process of calling for technical support. Since I was travelling overseas in a couple of days, I didn't have much time to troubleshoot things apart from isolating the fact when I removed the 3 devices running Windows 10 Technical Preview Build 9926 the problem went away. I even changed hardware, using another modem/router I had lying around (Belkin F9J1102v1) just to be sure the problem wasn't my ISP.

So I returned from overseas and resumed using the devices only to discover now that the Start Button on the HP laptop was not working properly, and the drop outs were now occurring on both modem/ routers, with different filters tested, and even when the phone line was connected directly to the wall socket in a process of elimination. Alas at this point I was scratching my head wondering what the cause of all my problems were.

Then I started researching the problem on this forum and found two posts that were very useful in providing me with a solution to my problems, so in a effort to share what I have learned here is what I found worked for me.

To resolve the Internet connection drop outs, I simply reset the Winsock entries and TCP/IP stack by doing the following steps: 

  1. Press Windows Key + X together and select Command Prompt (run as Administrator),
  2. Then type the following commands and hit enter after each: 

netsh winsock reset catalog

netsh int ip reset reset.log hit

Then flush your computer's DNS cache.

ipconfig /flushdns ipconfig /registerdns

ipconfig /release

ipconfig /renew

Reboot the computer and check your Internet connection by doing a Speed Test. This worked for me, i.e. no more drop outs!

And now to fix the Start Button not working issue.

Follow following steps:

  1. Go to run (Windows + r)
  2. type regedit and press enter.
  3. click ok when notification window prompt.
  4. Go to HKEY_CURRENT_USER>Software>Microsoft>Windows>CurrentVersion>Explorer>Advanced
  5. Create a DWORD with name: EnableXamlStartMenu
  6. Give it value 0
  7. Log off and login again.

Your start menu should work now.

If not try this:

  1. Right click on your Start menu button.
  2. Click on search.
  3. type: power shell.
  4. right click on power shell from search results.
  5. click run as administrator.
  6. Execute code below:

Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

Again this worked for me, i.e. Start Button functionality returned!

Not sure why this happened, but hope this advice helps others out there (like myself), who were experiencing the same symptoms that I was. Any clues from Microsoft as to why this may have occurred would be greatly appreciated, as so far this is really the only issue I've had running Windows 10 Technical Preview.

stefan blom say:
Switching to Print Preview and back will update cross-reference fields as well. (I'm assuming that you inserted cross-references via References tab | Cross-reference.)

However, you must be careful when dealing with cross-referenced items. When you create a cross-reference, Word encloses the referenced item in a hidden bookmark. Later, if you add content at the beginning of a cross-referenced/bookmarked paragraph, the bookmark will expand and include the added content. To work around this issue, always add new paragraph at the end of a bookmarked paragraph (not at the beginning).

To fix an existing incorrect cross-reference, delete and re-insert it.
