To the second question: We can use Information Rights Management to prevent users from downloading files in SharePoint libraries. For your reference:
Set up Information Rights Management (IRM) in SharePoint admin center.

As already mentioned by Ryan, these problems are rarely malware and more often simply hardware issues with either the keyboard or other device.

The simple method to confirm this is to borrow a keyboard from another computer system to see whether the issue you're having goes away.  If it does you need to either find the paperclip or other metal floating around under the keys, sticking key(s) due to a liquid spill or finally just buy a new keyboard to permantly solve the problem.

doug robbins - mvp office apps & services (word) say:

Running a macro containing the following code when you have the selection in the table will assign the required bookmarks:

Dim BMName As Range, BMRange As Range
With Selection.Tables(1)
    For i = 2 To .Rows.count
        With .Rows(i)
            Set BMName = .Cells(1).Range
            BMName.End = BMName.End - 1
            Set BMRange = .Cells(2).Range
            BMRange.End = BMRange.End - 1
            ActiveDocument.Bookmarks.Add BMName.Text, BMRange
        End With
    Next i
End With
