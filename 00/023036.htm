Adding an author to my Sway

Dear Sir/Madam

I have made a number of Sway documents using an email from my college. I would like to add my new personal email onto these documents so that when I leave college I'll be able to still access these and edit them. I am not sure how to do this.

jason (office engineer) say:

There's two things you could consider here:

  • Make your personal account a coauthor on the Sways so that you can edit them - the downside with this approach is that the Sways will still belong to the college account and if/when that account is deleted, the Sways will go away.  The upside is that the URLs to the Sways won't change so everyone you've shared them with will still have access.
    • To do this, open the Share dialog, set the permission to anyone with a link, select to give someone edit access, and copy the edit URL from that dropdown.  Now open Sway and sign in as your personal account and open the URL you copied.  This will give you the option to edit it, the first time you do that you will be given edit access to the Sway and will be a coauthor from then on.  You will then open the Sway the normal URL for it.
  • Duplicate your Sways into your personal account - the upside to this is that you will have complete ownership and control over them in your personal account, but the URLs will change.  However, this will ensure you always have a copy of them even after you leave school.
    • To do this, open the Share dialog, set the permission to anyone with a link, and copy the URL that you opened the Sway with.  Now open Sway and sign in as your personal account and open the URL you copied.  From the ... menu, select Duplicate this Sway.  Now you will have a copy.

Hope this helps!

Jason

graham mayor say:

For the certificate to be trusted, it first has to be in the correct store location.

Use SelfCert.exe to create a certificate (SelfCert.exe is in the Office program folder). It will create a certificate in the Current User > Personal > Certificates store. It needs to go in the Local Computer > Trusted Root Certificate Authorities > Certificates store, so you need to export from the former and import to the latter.

1. From the Windows search type mmc and then run the associated command (or Winkey+R > mmc > enter)

2. Select File > Add/Remove Snap-in

3. Double Click 'Certificates'

4. Select 'My User Account' > Finish

5. Double Click Certificates again and select 'Computer Account' > Next > Local Computer > Finish

6. Click OK

7. In the Left Window > Certificates - Current User > Personal > Certificates. Your certificate should be listed in the main window.

8. Select the certificate > Right click > All Tasks > Export and export the certificate to a file. (I used .P7B format at the bottom of the options.)

9. In the left window select Certificates Local User > Trusted Root Certification Authorities > Certificates

10. Right click > All Tasks > Import and import the file you have just exported at '8'.

You should now be able to add your certificate to your project and it will be valid and you should be able to set the option to "Trust all documents from this publisher".

Update 15/03/2016: - I have added a, hopefully easy to follow, pictorial description of the process to my web site - http://www.gmayor.com/create_and_employ_a_digital_cert.htm
