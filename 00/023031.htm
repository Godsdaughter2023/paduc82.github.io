Solved Error while updating 0x80070003, 0x80070002, - Microsoft Community

maksym vakulenko say:

Solved Error while updating 0x80070003, 0x80070002, 0x80070003-0x2000c, 0x80246007

Hello friends!

I met the problem when I start updating Windows Technical Preview 9841 to Windows Technical Preview 9926.

All needed files were downloaded successfully and Windows ask me to reboot.

After reboot (for me it was about 20% of unpacking files) windows rebooted again. I receive "recovering old sys" message and 9841 was recovered.

I tried to stop/run again all process from the very beginning.

1) Stoped BITS, Event Log services to be able to delete all files from folders \Windows\SoftwareDistribution\Download and \Windows\SoftwareDistribution\DataStore.

2) Restarted services of Windows Update.

3) Deleted files from C:\$Windows.~BT and started from downloading.

Everything finished by Errors: 0x80070003, 0x80070003-0x2000c, 0x80246007.

Here are the solution worked for me.

In the thread http://answers.microsoft.com/en-us/insider/forum/insider_wintp-insider_install/windows-technical-preview-error-0x80070002-0x2000c/77595ef4-c88b-4ebd-9461-a4b23478b7f2 user Shilohbob said it's maybe because the partition doesn't have ACTIVE attribute.

My system consists of 3 HDD volumes.

1) Recovery (HDD manufacturers)

2) C: (Boot,Page file,Crash Dump,Primary)

3) System (System,Active,Primary Partition)

If you simply turn ACTIVE on the C: volume, windows will not start at all. (To solve this run from recovery media cmd and diskpart, "list volume", "select system",  "active").

To maked volume C: bootable we have to transfer system files and write MBR on this volume. Here we are.

1) Type CMD in the  Start search box. On the black box named "Command Prompt" press right mouse button and choose "Run as administrator".

2) Enter command "bcdboot c:\windows /s c:" (this will transfer system files to disk C:)

3) Enter command "bootsect  /nt:60 c: /mbr" (this will write Master Boot Record on C: making it bootable)

4) Enter command "diskpart" (running DISKPART utility)

5) "select volume c" (Making volume C: being in focus to operate with)

6) "active" (this will label disk C: as Active partition)

7) "list volume" (look the name or number of you volume with label System. If you already know this skip this step)

8) "select volume system" (Place focus to SYSTEM volume to work with. My volume with attribute system was named SYSTEM. It can be D: or E: or else.)

9) "inactive" (deactivating old system volume)

10) "exit" (will exit DISKPART)

11) "exit" (will exit Command Prompt)

Restart you computer and run update again. I even didn't download update files. Windows Technical Preview 9926 installed successfully.
