steve h [aka sjh_vt or smah] say: If your mail server is a POP3 server, by default Windows Mail will remove all messages from the server (so you won't see them online).

e.j. gun say:

Looks like you are getting the query results from the AS400 in a coding form known as EDCDIC.  This is an alternate way of coding numbers and letters frequently used on IBM mainframes.  The most common encoding is ASCII.

In EBCDIC, D6 is the hexadecimal value for the letter "O".  C4 is the hexadecimal value for the letter "D".

 

See this site for details of the two character sets:  http://www.dynamoo.com/technical/ascii-ebcdic.htm

 

Does the query produce a simple text file that you are importing into Excel?  How are you importing the data?  If it is a text file.

 

See this Microsoft knowledge base article for details on how to convert between the two character sets:

http://support.microsoft.com/kb/216399

 

You might have to resort to some coding to convert the text from EBCDIC to ASCII.

 

HTH,

Eric

chrispeng_msft6626 say:

Hi Guilherme,

I suggest you try clearing cached data and corrupted certificates in Lync to check if there is any improvement.
First, delete the following folders:

  • Users/Home Folder/Library/Caches/com.microsoft.Lync
  • Users/Home Folder/Documents/Microsoft User Data/Microsoft Lync History

Then, delete any corrupted or cached certificates. To do this, follow these steps:

  • Open the Keychain Access certificate management utility. To do this, in Finder, click Applications, click Utilities, and then click Keychain Access. Or, search for Keychain Access by using Spotlight.
  • In the left pane, click login, and then click Certificates.
  • In the right pane, find a certificate that's named Unknown or Communications Server, select it, and then delete it. Note: You may have to unlock your keychain by using your password.
  • Close Keychain Access.
  • Restart Lync for Mac.
