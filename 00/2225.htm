ossiemac say:

It took me a little while but I have identified the problem. Excel cannot handle the Table Reference for the ListBox RowSource when the table is resized.

The workaround is to code in the following order so the ListBox Rowsource is not affected when the row is added:

  1. Save the required data to the Array.
  2. Set the ListBox RowSource to a zero length string. (So it no longer has a RowSource)
  3. Add the row to the table.
  4. Copy the array to the new table row.
  5. Reset the Rowsource for the ListBox. (will then include the added row)

Sub addReceipt()
    Dim tgSheet As Worksheet
    Dim tbTransaction As ListObject
    Dim invoiceRow As ListRow
    Dim arr(0, 6) As Variant
   
    arr(0, 0) = Format(Time, "hh:mm")
    arr(0, 1) = SheetSettings.Range("B5").Text
    arr(0, 2) = Val(frmCashRegister.lblCardAmount.Caption)
    arr(0, 3) = Val(frmCashRegister.lblCashAmount.Caption)
    arr(0, 4) = Val(frmCashRegister.lblChequeAmount.Caption)
    arr(0, 5) = 0
    arr(0, 6) = Val(frmCashRegister.lblTotal.Caption)
           
    'Cancel the RowSource by setting to zero length string before adding the row to the table
    frmCashRegister.listBoxTodayTransactions.RowSource = ""
   
    Set tgSheet = sheetTodayTransactions
    Set tbTransaction = tgSheet.ListObjects(1)
    Set invoiceRow = tbTransaction.ListRows.Add
   
    invoiceRow.Range = arr
       
    'Reset the RowSource (Will now include the added row)
    frmCashRegister.listBoxTodayTransactions.RowSource = "tbInvoiceSummary[[#Data],[#Totals]]"
        
 End Sub

how to change the default font where you cross reference

How do I change to font when I want to insert a cross reference to a table, figure or appendix? The default is always 12 pt.

jay freedman say:

A cross-reference is just a field that points to a hidden bookmark at the place being referenced. It doesn't have any special style applied to it by the Insert Cross-Reference dialog, so it should appear in the same font, size, and formatting as the text surrounding it.

The next time you insert a cross-reference with unwanted formatting, open the Reveal Formatting pane by pressing Shift+F1. Check the box at the bottom of the pane for "Distinguish style source". Then click in the cross-reference text and look at the information in the pane about the font formatting. Is the 12 pt size listed under a style, or as direct formatting?

If it's part of a style, then you should modify the style's definition to change the size, or else apply a different style to the cross-reference.

If it's direct formatting, you can remove it by selecting the cross-reference and pressing the Ctrl key and the spacebar simultaneously.

ossiemac say:

You seem to be posting replies and then deleting them while I am attempting to answer and I am becoming confused about what I am answering.

The code below creates an array of the names and returns the results for each name.

You will see that VBA code has been used to identify the User's Reference Style and then process each of the names, one with A1 Style and one with R1C1 style (Both return the same result).

When it is finished, it restores the User's option to what it was before the code was run. This should always be done when changing options for a User. (ie. Identify the User's option first and when finished return the user to their pervious option.)

The On Error Goto ensures that if for some reason an error occurs for the user then their option is still reset but it does tell then an error has occurred and refer it to the project Administrator (whom in this case is you). To find the error you have to comment out the On Error Goto line and re-run the code.

Note that my example goes to Row 11.

Sub Macro2()
   
    Dim strRefStyle As String
    Dim arrNames() As Variant
    Dim Name As String
    Dim i As Long
    Dim t1 As Date
    Dim t2 As Date
   
    strRefStyle = Application.ReferenceStyle    'Save the User's current Reference Style
   
    On Error GoTo RestoreRefStyle
   
    arrNames() = Array("Owen Miller", "Sarah McCrae", "Yaoqing Ding", "John Green")
   
    For i = LBound(arrNames) To UBound(arrNames)
        Name = arrNames(i)
       
        Application.ReferenceStyle = xlA1
        t1 = Application.Evaluate("MAX(IF(Sheet1!$A$1:$A$11=""" & Name & """, Sheet1!$B$1:$B$11))")
       
        Application.ReferenceStyle = xlR1C1
        t2 = Application.Evaluate("MAX(IF(Sheet1!R1C1:R11C1=""" & Name & """, Sheet1!R1C2:R11C2))")
       
        MsgBox arrNames(i) & ": " & t1 & " (A1 Ref Style)" & vbCrLf & _
                arrNames(i) & ": " & t2 & " (R1C1 Ref Style)"
    Next i
   
RestoreRefStyle:
    If Err.Number <> 0 Then
        MsgBox "Error occurred in Sub Macro2" & vbCrLf & _
                "Comment out line On Error GoTo RestoreRefStyle" & vbCrLf & _
                "Report problem to project Administrator."
    End If

puasho---01 say:

N8: proximity sensor and left-handed people.

Well, after reading so many critical reviews about the effectiveness of the proximity sensor, and having experienced it myself (but just as occasional light up of the screen during cal, no auto hang or auto mute of the call)  I "investigated"  and noticed as follows:

The sensor is very sensitive but also very "localized". It seems to be  located between the "NOKIA" label and the ambient light sensor, on the top of the screen, and on my N8 a pencil, at 1 cm from the "hot spot", is enough to trigger the screen-lock.

The "problem" is that for left handed  people, during call the auricle is more likely to "uncover" the sensor, that is located in "upper" position, while holding the phone the  with the right hand on the right ear (of course..:smileyhappy:) gives more guarantee that the sensor is always covered, as in lower position,  by the back of the auricle .

I made some trials keeping the phone with left and right hand and noticed that in the first case the usual "adjustments" that occur during a (mainly  if long) call make from time to time light up the screen, keeping instead it with the right hand it's far more unlikely for this to happen.

 

So, in my humble experience....
