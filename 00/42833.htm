If it is not working in the Windows Camera App, it will not work in Skype. Nothing the VCamFix tool does should cause any actual permanent damage to your system. That tool performs several resets/repairs and unbinds any external 3rd party filters to leave you system close to an initial install of Windows. It does not attempt to modify or delete existing driver packages. Since your Intel camera drivers uses a combination of virtual devices to function normally (which is abnormal), it would be best to completely uninstall and reinstall your camera drivers. Avoid reinstalling additional virtual software like Cyberlink Webcamsplitter/YouCam, if possible. I recommend removing all traces of the existing drivers in Programs and Features and the Device Manager, then reinstall the Intel drivers. Depending on your OEM device you may need to re-download the latest firmware/system package update suite from your OEM to reapply those drivers.

Once it is operational in the Windows Camera App, test out the Skype App once again. If it works in the Windows Camera App but not the Skype App, then there may be a compatibility issue with the current Skype 12.x release. If it works in neither that generally points to a driver issue or compatibility with all Windows 10 Apps. Drivers that are incompatible with Windows 10 Apps or are no longer fully installed properly may still work with all desktop applications that access the camera. Technically this problem could also occur in the reverse, if the install is damaged.

samuel tai say: ClearQAM tool won't work if TV Pack is installed, since TV Pack natively can tune QAM.

What I tried in W7 was:
1. Delete all the devices associated with the M780 in Device Manager.
2. Rescan for hardware changes in Device Manager. It should detect the bridge device. Install the driver for the bridge device from Windows Update. This should be driver 2.5.64.29.
3. The bridge device should install the driver 2.5.64.29 for the tuner and capture devices.
4. Now run Windows Update. It should update the tuner and capture devices to 2.5.64.31.
5. Manually update the driver for the bridge. (I used Search automatically). This should bring the bridge up to 2.5.64.31.

Now, the analog coax input will only tune analog. The digital coax input will only tune ATSC or QAM. If you want to tune both from your cable source, you need a splitter to feed both inputs.
