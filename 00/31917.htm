    The issue was gone until yesterday, the issue returned.
    *The actions that trigger crash/restarts are as follows:
    -(might be relevant and might help if you have similar or identical issues)

    *create a new folder
    *open anything other than a folder

    *sometimes randomly some folder itself will trigger a crash/restart

    *programs that depend on windows select interface will crash that program
    -(for instance, loading or saving any project with autodesk 3ds max 2017 will crash it)

    I may have missed a few actions or triggers but thats essentially it.
    There is one solution and one dirty fix.

    The solution is to create a new user and delete the faulty one. There are enough tutorials how to do that.

    The dirty fix i found so far if you dont want or dont have time to make a new user and backup/transfer all the users data is to:
    *right click on empty space on toolbar and select settings;
    *select the "Start" tab on the left;

    *you might need to scroll down until you see "Show recently opened items in Jump Lists on Start or the taskbar" and turn that option off.

    I have observed that after crashes/restarts happen, any time you would navigate to:
    C:\users\username\AppData\Roaming\Microsoft\Windows\Recent Items
    and select that item, it would trigger a crash. I have tried to replace it, delete its contents but the issue still persist.
    I or the community may find a better solution but id suggest anyone that has more experience in Win 10 to take notice of that file folder.
