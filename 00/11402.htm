seattleexpat say:

We ended up using a low-speed hack to solve our problems. See the instructions below, modified with example IP address and DNS names:

1. Find the external IP address of the Exchange Server:

$ ping webmail.company.com

2. From the Terminal, locate and open the 'hosts' file for editing:

$ sudo vim /etc/hosts

3. Add a line to the file that maps the external IP address to the internal server DNS name:

10.10.10.10  internal-server342.internal-domain.company.com

3. Save the file, and restart Outlook. 

Been working for a couple days now with no problems.
