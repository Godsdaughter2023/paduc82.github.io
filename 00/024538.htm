Dim TopRow As Range
Private Sub Worksheet_SelectionChange(ByVal Target As Range)

e.j. gun say:

Why not just use VBA code to delete the sheets.  Add the code below to a standard code module.  If you want, assign it to a button on the QAT.

To use this code, you need to set a reference to the "Microsoft Scripting Runtime".  In the Visual Basic Editor, go to "Tools/References...", find "Microsoft Scripting Runtime", and make sure the box next to it is checked.

Then, select the sheets that you DON'T WANT TO DELETE.  The code will delete all sheets that are NOT SELECTED.

Run the macro - the "unselected" sheets are all gone!

HTH,

Eric

 

'====== BEGIN CODE ======

Option Explicit

'
' DeleteNonSelectSheets Macro
'
Sub DeleteNonSelectSheets()
    Dim i As Long
    Dim selSH As Scripting.Dictionary
'
' Create a dictionary that holds a list of all
' the currently selected sheets.
'
    Set selSH = New Scripting.Dictionary
'
    For i = 1 To ActiveWindow.SelectedSheets.Count
        selSH.Add Key:=ActiveWindow.SelectedSheets(i).Index, Item:=ActiveWindow.SelectedSheets(i).Index
    Next i
'
' Go through all sheets - if not selected, then delete it
'
Application.ScreenUpdating = False
Application.DisplayAlerts = False
    For i = ActiveWorkbook.Sheets.Count To 1 Step -1
        If (selSH.Exists(ActiveWorkbook.Sheets(i).Index)) Then
            Debug.Print "Sheet " & ActiveWorkbook.Sheets(i).Index & " is selected - DO NOT DELETE"
        Else
            ActiveWorkbook.Sheets(i).Delete
            Debug.Print "Sheet " & ActiveWorkbook.Sheets(i).Index & " is not selected - DELETE"
        End If
    Next i
Application.DisplayAlerts = True
Application.ScreenUpdating = True
'
End Sub

'====== END CODE ======

ian joseph thompson say:

I figured it out.  The behavior is different if your cursor is sitting in whitespace, or with text.

If I have the following in my Word doc (no bullets, no list):

HEADING

  • If I place my cursor at the beginning of "First Bullet", then I can press Tab as many times as I want to increase the indentation of that text.
  • If I start a new line after "First Bullet", then my indentation level starts at the same level as the previous line and I can press Tab as many times as I want to add tab characters, but pressing Shift+Tab ALSO adds a tab character.
  • If I start a new line after "First Bullet" and type a single non-whitespace character, then I can place my cursor in front of that character and press Tab as many times as I want to increase the indentation of that text.  I can also then use Shift+Tab to decrease the indentation as many times as I want.

Note:  It is really easy to think you are increasing the indentation level with Tab, but sometimes you are adding tab characters...  The presence of whitespace will prevent Tab from increasing the indentation level AND Shift+Tab from decreasing the indentation level - new tab characters will be added instead.

brian tillman [outlook mvp 2007-2019] say:
Exchange ActiveSync accounts don't support contact groups. Once your account has moved from the Outlook.com platform to the Office 365 back end, you will be able to change the account to full Exchange and then groups will work. EAS translates contact categories on the client side to contact groups on the server. So, if you assign a category to the contacts you want to mail at one time you can use that.

For the sake of "clarity"

#1 - Group will not work when the contact folder is contained in the data file associated with an Outlook.com account configured using Exchange ActiveSync and that data file is the "default data file" for the profile. As Brian mentioned, once your Outlook.com account is migrated - the issue disappears.

#2 - Groups <will> work for Imap <IF> the data file associated with the Imap account is set as the "default data file" for the profile. There can only be on "default data file" set for a profile and that data file contains your "default" contact/calendar folders

To see how your profile is configured in terms of data files - go to 

File --> Account Settings --> Data File tab

For anyone running across this thread - it is really not a good idea to store you contacts in an Imap contact folder (applicable to OL'2013/'2016 only) where you see the words "This Computer Only" unless you have the discipline to back that data up on a very regular basis - otherwise, you risk losing that data should you, for whatever reason, delete the email account or the associated .ost file gets separated from the Outlook profile/email account that originally created the .ost file.
