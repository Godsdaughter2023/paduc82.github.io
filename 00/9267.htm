A) Stop windows update from updating ALL hardware drivers

1. Right-click start, Control Panel

2. In the top menu click in  [Search Control Panel]
3. Type - Change device installation settings

B) Then uninstall Lenovo PM device

rohn007 say:
Windows Update is pretty smart. I think (actually, I hope) that you'll find if you look further, while the initial scan may offer you updates you don't require, when the actual update process runs it recognizes that you don't actually need the updates so it skips them.

Personally, I've given up trying to second guess Update. I let it install updates if it thinks it needs to. Frankly, I'd rather have unneeded small updates installed rather than skipping an update that I needed because I thought I didn't need it.

Update Offers Updates for Programs you do not have installed

http://support.microsoft.com/kb/830335

Windows Update often offers updates for programs that you think you do not have installed. The link above gives 3 reasons why this happens and some possible solutions. Another reason is that the updates are often not for the single product in the description.

Updates are often bundled for more than just the app identified in the label. More than one program or bundle may be using the same component.

For me, the short answer is, just let Update do “its thing” because it has better insight into the deep details of your system than you do. The only control I sometimes exert is limiting the number of update I let it apply at one time. For example, when installing Windows or an application Update may find very many updates so I limit it to 5-10 at a time.

andy31370 say: I had the same problem with my old 6100. I think the Nokia 6020 is only compatible with CA-42 and DKU-5 data cables. Also try connecting via Infrared. When I connected using the CA-42 cable the problem was solved.

bonxgz say: To resolve this behavior, add the %SystemRoot%\System32\WBEM folder to your PATH environment variable. To do so, follow these steps:
1. Right-click My Computer, and then click Properties.
2. On the Advanced tab, click Environment Variables.
3. Under System Variables double-click the PATH environment variable.
4. Type:
%SystemRoot%\system32;%SystemRoot%;%SystemRoot%\system32\WBEM
NOTE!! If other system variables appear, do not remove them.
5. Click OK, and then click OK.
