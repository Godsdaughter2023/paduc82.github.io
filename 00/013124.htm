daniel g. grau say:
Or you can create a master document, to visually combine all 10 documents, yet the documents will remain separate.

I found no tutorials for Word 2011 (nor any Help in Word itself!), so here's a crash course:

  • Create a new document and save it
  • In the main menu bar, select View > Master Document (you can type something here, like "Book Title" and perhaps a page break)
  • A new small toolbar will appear (called "Master Document")
  • Click on Insert subdocument (do for all 10 documents)
  • Save and close your master document

When reopening the Master document, you'll see links instead of file contents:

  • Do View > Master Document
  • The Master Document toolbar will appear
  • Click on the first icon, "Expand subdocuments"
  • Change the view to something else (Draft or Print Layout)

It is important to note that the documents will remain separate, yet any change you enter in them through the Master will affect them just as if you were editing them separately. Such changes will be committed when you save the Master.

If you want their respective TOCs to display page numbers as "1-1, 1-2... 2.1, 2-2..." etc., it can also be done.

Slack is the result of how you have linked the tasks and what the duration estimates are.

It is what it is.

It is not something that you can decide that you want, and how much of it you want and then type it in.

john - project say:

Anand,

First of all, early start, early finish, late start and late finish are all calculated by Project and not user editable. Even if the user could edit those dates, it would not impact the critical path.

If all your task inter-dependencies are valid, then you want the critical path to be visible. Adding a buffer to some tasks won't necessarily remove them from the critical path but it will push out the end date.

Trevor gave you a suggestion for adding buffer to all tasks in the plan and that may not be desirable. You indicate you want a buffer on only a few tasks. The following approach will do that by adding a dummy buffer "task" in parallel with those tasks you wish to add a buffer. Visibly the real tasks will not show as critical (the buffer task will) but keep in mind that the buffer "task" has no resources assigned and is a placeholder for viewing purposes only. The nice thing about this approach is that you do have visibility on how much buffer each selected task has both in the Gantt Chart graphic and in the Total Slack field.

In this screen shot, I've added a buffer task in parallel with task c. The buffer task's duration is set to be the same as the real task's duration plus a buffer, in this case, one day. That results in a positive slack of one day for task c.

As mentioned in the discussion, I too found using tags in Windoze rather flakey.  I never got it to work satisfactorily. As a result, I came up with an alternate method that works works well enough for me.

I start by using full LONG names.  I started computing in the says of the 8 character file name limit.  So I was over the moon when file name lengths were extended.  Now, the starting point of my search process is using the full title of the document as the file name.  I do use short forms of words in the file name ie Win8 vs Windows 8.

So my file naming scheme becomes:

tag-   one "primary" keyword tag

YYMMDD - numeric date

Long File name

-tag tag ...  secondary keyword tags

jay freedman say:
OK, now I see the cause.

Word 2010 offers three different kinds of checkbox objects. The one you used to replace the ActiveX objects is not the "legacy form field" checkbox that I thought it was. Instead, it's a content control checkbox, which is new in 2010 (meaning that no one using Word 2007 or earlier will be able to use it as a checkbox).

Every content control is surrounded by a bounding box, the blue box that appears when you select the control. That bounding box, which is larger than the control itself, is what is making the extra separations in the stack. You can make the checkboxes line up with the labels better by selecting each content control plus the paragraph mark to its right, and setting the point size to 9 pt -- the boxes will be slightly smaller than the ActiveX ones, but not too small.

The other kind of checkbox is the legacy form field one, in the top row of items on the Legacy Controls dropdown in the Controls group. This kind has no bounding box, so it obeys the font size and paragraph spacing just like text. The difference is that, unlike the other two, it requires you to protect the document for filling in forms. That's most easily done by adding the Lock icon to the Quick Access Toolbar (you'll find it in the All Commands category in the Customize dialog) and clicking that button to toggle protection on and off.
