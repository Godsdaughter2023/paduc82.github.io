In case of multiple nesting, it might be a good idea to use lookup - its a compact formula which enables easy increase in the options ... because you have indicated "26-35 units, 4% bonus, and so on".

=LOOKUP(R11,{0,15,16,26,36,46},{0,0.01,0.03,0.04,0.05,0.06})

less than 15 returns 0%

>= 15 and less than 16 returns 1%

>=16 and less than 26 returns 3%

>= 26 and less than 36 returns 4%

>= 36 and less than 46 returns 5%

46 and above returns 4%

Copy the code below into a regular module and this code into the code module ThisWorkbook:

Andreas.

Option Explicit

#If Win64 Then
Private Declare PtrSafe Sub keybd_event Lib "user32" ( _
    ByVal bVk As Byte, ByVal bScan As Byte, ByVal dwFlags As Long, _
    ByVal dwExtraInfo As LongPtr)
Private Declare PtrSafe Function GetKeyboardState Lib "user32" ( _
    pbKeyState As Byte) As Long
Private Declare PtrSafe Function SetKeyboardState Lib "user32" ( _
    lppbKeyState As Byte) As Long
#Else
Private Declare Sub keybd_event Lib "user32" ( _
    ByVal bVk As Byte, ByVal bScan As Byte, ByVal dwFlags As Long, ByVal dwExtraInfo As Long)
Private Declare Function GetKeyboardState Lib "user32" (pbKeyState As Byte) As Long
Private Declare Function SetKeyboardState Lib "user32" (lppbKeyState As Byte) As Long
#End If

Private Const VK_NUMLOCK = &H90
Private Const VK_SCROLL = &H91
Private Const VK_CAPITAL = &H14
Private Const KEYEVENTF_EXTENDEDKEY = &H1
Private Const KEYEVENTF_KEYUP = &H2

Property Get ScrollLock() As Boolean
  Dim Keys(0 To 255) As Byte
  GetKeyboardState Keys(0)
  ScrollLock = Keys(VK_SCROLL)
End Property

Property Let ScrollLock(Value As Boolean)
  Dim Keys(0 To 255) As Byte
  GetKeyboardState Keys(0)
  If Value <> CBool(Keys(VK_SCROLL)) Then
    keybd_event VK_SCROLL, &H45, KEYEVENTF_EXTENDEDKEY Or 0, 0
    keybd_event VK_SCROLL, &H45, KEYEVENTF_EXTENDEDKEY Or KEYEVENTF_KEYUP, 0
  End If
End Property

Sub DisableScrollLock()
  If ScrollLock Then Application.OnTime Now, "PressScrollLock"
End Sub

Sub PressScrollLock()
  ScrollLock = False
End Sub

janhec say: I'm sorry, but this is just one of those answers that will cost you time and not solve anything.
If you check this, you will find you cannot use excel 2013 as an "open with" program for ANYTHING.

It's not just text files.
Somebody else found out what the problem is and how to remedy it.
See http://superuser.com/questions/577109/why-wont-excel-2013-work-as-an-open-with-program-in-windows-7/594235#594235, page to techturtle.
The cause is that a registry entry is not updated during install of office 2013. The entry HKEY_CLASSES_ROOT\Applications\EXCEL.EXE\shell\open\command\(Default) will point to an old and usually invalid instance of excel.
Changing this value to point to the actual excel 2013 executable immediately brought back the usual functionality. The path named in the linked post may not be correct for everyone, so if you decide to change this registry key, check the path carefully.

Of course, changing the registry is dangerous, etc. etc., make sure you have a backup, etc., etc, if you do not know what I'm describing, don't try it, etc., etc..
But it did work for me,  and I suspect, it will work for lot's of others too.
