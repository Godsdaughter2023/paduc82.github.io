ziggyj say:

I just had this problem upgrading from Windows 7 to Windows 10. Any partition (drive) over 2 TB is not detected and instead is shown as unpartitioned space in Disk Management (search for 'partition' and select 'create and format disk partitions' to see this).

Fortunately this is easy to fix using the right third party recovery tools. It seems that partition recovery tools (rather than undelete tools) can find the lost partition(s). I used a free one called MiniTool Partition Wizard but I'm sure others will also work.

If you have tried to re-partition the lost drive you may have lost data but if it was only a quick partition (usual option) this may still work on its own or together with an undelete tool.

Download and install a trusted tool (specifics instructions relate to the tool names above) and select the partition recovery wizard on the menu on the left. Click next and select the physical disk you have lost a partition from and click next again. If you know which area of 'Unallocated' space select 'specified range' as the scanning range and select the correct area. Otherwise selecting 'Unallocated space' should work (I didn't use this). Click next again and selecting Quick scan worked for me.

Is should now find your missing drive(s) as Lost/Deleted. Select ALL the partitions you need, not just the Lost/Deleted one. And select Finish. You will now see the final result. Right click on any new drives and select 'Change letter' to connect them to windows. Now click Apply at the top for it to finish up.

dmcyx say:

Obviously, wasn't working on this all the time, but I tried to resolve this again today and finally got something to work.  What's ironic is that if there hadn't been all the "help" from Windows, this is probably what I would have done on my own.  Instead of letting the system continue to Troubleshoot, FixIT, debug, update drivers online, check the manufacturer, or check c:\windows\system32 for drivers locally; I forced a local search of C: (including subfolders) and it worked.  The fix caused several other related devices to error out, but just repeating this process for each of them resolved their errors, as well.  One additional odd note about the drivers that were finally located.  The drivers that were selected were all in either C:\windows\system32 or C:\windows\system32\drivers.  My original local search should have found them, but it didn't (I retried again today).  Only by the full search of C:\ worked. 

 

Solution for my system:

  • Go to Control Panel, Device Manager, find the device, right click and select Properties
  • Select the Driver tab and click the Update Driver button.
  • Select Browse my computer for driver software
  • Wipe out anything in the location and replace with C:\  (make sure that include subfolders is checked)
  • It will probably take a few minutes to run, but in my case it found a completely different driver and the Bluetooth was no longer identified as a Generic Bluetooth Device, but changed to an Atheros Bluetooth Device.
  • I did a full shutdown and restarted the system to make sure the change stuck and that the errors were resolved. 

 

When the system came backup, 6 new errors came up for Other Bluetooth devices.  The devices were not identified with any additional descriptive information and were listed as software uninstalled.  As I repeated the the process above for each of them, they appropriately identified as the real device again (for example Other Bluetooth device became Bluetooth Audio Sound).

Ignored that information and repeated the steps above one more time.  No more errors on full restart.

 

The problem now seems to be resolved.  There are no Generic or Other identifiers on any of the Bluetooth devices and everything seems to be coming up with no errors, when I restart the system.
