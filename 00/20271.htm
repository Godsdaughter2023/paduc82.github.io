It looks like you have a working HLOOKUP() and you just require some error control. Wrap the existing formula in IFERROR(), supplying a 0 as the default value.

=IFERROR(<existing formula>, 0)

daniel g. grau say:
The way a Master Document works, subdocuments are intended to be self-standing documents starting on their own page. You'll have to resort to combining documents so that page breaks are acceptable at each subdocument change, or use the { INCLUDETEXT "filename" } field. (See a nice explanation here; method 1 is the easiest.) 

Unfortunately, these fields require absolute paths in references to file names, so they will no longer update if you link them to the files (as opposed to inserting them permanently, in which case the would not update anyway) and then move the documents to another folder or transfer them to another computer. However, they will work fine as long as you don't move the files, or if you refrain from using links.

john mcghie say:
Hi MacEvans:

Good!  You have discovered two essential secrets of Master Documents :-)

1)  The section breaks are critical, DON'T touch them

2)  The file path names are critical, DON'T touch them either

{Grin}

The problem you are getting appears to be a "feature".  Because each subdocument is a separate document, it will always start on a new page.

You can "force" the section breaks to be "Continuous".  But if you do, chances are you will corrupt the master document.  It still won't enable you to print more than one subdocument on a page.

To force them, save, close, and re-open the master document.  Then use Format/Document to specify "Section Starts = Continuous" and specify "Whole document".  When I tried it, the change will save and hold, but the result can not then be read by other versions of Word.

To achieve what you want, use Insert>File not Insert Subdocument.  By inserting the document as a file, you will not add section breaks so you won't get extra page breaks.  You can also check the box that says "link to file" then disable "Embed in document" if you want to get the same behaviour as a master document.

The "Cannot open, file path is missing" is a bug.  The section break stores the path to the subdocument.  When you changed the section break, Word should copy the file path to the new section break.  That appears to be broken: Word is writing Mac file paths, it should be writing Unix file paths.

Just to explain: for every subdocument you add, Word inserts two section breaks to encapsulate the added subdocument.  These must be there, they act as a "fence" to ensure that all of the various properties of the subdocument are applied only to that subdocument.  If you delete the section break, you thus corrupt the master document, and your subdocuments will then all fail to open.

One thing you should remember is what Daniel is saying: subdocument paths are written as absolute paths (fully qualified and specific to your machine).  The only way to overcome that is to create a folder for your master document, and transfer all of the subdocument files into that folder before you begin to make your master document.  If you do that, you should then be able to move a master document without breaking it, provided you move the whole folder as a single unit.

Hope this helps
