jay freedman say:

I'll assume that you're telling Word to save the customizations in the Normal.dot template (that is, in the Customize dialog's Commands tab, the dropdown at the bottom is set to "Normal.dot"). It's possible that despite that setting, the changes aren't really being saved in the template's file. (This also happens sometimes with changes in the definitions of styles.)

As a workaround, open the Normal.dot template itself as a "document" in Word. The easiest way to do that is to open the macro editor (Tools > Macro > Visual Basic Editor), open the Immediate pane (View > Immediate Window), type or paste the following line and press Enter:

NormalTemplate.OpenAsDocument

Make the customizations there, and save and close the template. It may be necessary (and can't hurt) to make a small edit such as inserting and then deleting a space character in the body of the template to make Word recognize the need to save the file.

It should make no difference whether you're running Windows in a Parallels VM or as your main operating system.

macropod say:

The following macro will process all Word documents in a folder provided they:
a) are not password-protected;
b) are not mailmerge main documents; and
c) don't contain macros that require any form of interaction upon opening/closing.

Simply point the macro to the folder concerned. Except for the Word-specific code, the same basic code can be used with other applications.

Sub UpdateDocuments()
Application.ScreenUpdating = False
Dim strFolder As String, strFile As String, wdDoc As Document
Dim Scn As Section, HdFt As HeaderFooter, Rng As Range
strFolder = GetFolder
If strFolder = "" Then Exit Sub
strFile = Dir(strFolder & "\*.doc", vbNormal)
While strFile <> ""
  Set wdDoc = Documents.Open(FileName:=strFolder & "\" & strFile, AddToRecentFiles:=False, Visible:=False)
  With wdDoc
    For Each Scn In .Sections
      For Each HdFt In Scn.Footers
        If HdFt.LinkToPrevious = False Then
          Set Rng = HdFt.Range
          With Rng
            .ParagraphFormat.Alignment = wdAlignParagraphCenter
            With .Font
              .Size = 9
              .Name = "Arial"
              .Bold = True
            End With
            .Text = vbNullString
            .InsertBefore "ACME Proprietary (Internal Use Only)" & Chr(11)
            .Collapse wdCollapseEnd
            .InsertAfter "Not for use or disclosure outside the ACME companies" _
              & Chr(11) & "except under written agreement"
            .Font.Bold = False
          End With
        End If
      Next
    Next
    .Close SaveChanges:=True
  End With
  strFile = Dir()
Wend
Set wdDoc = Nothing
Application.ScreenUpdating = True
End Sub

Function GetFolder() As String
Dim oFolder As Object
GetFolder = ""
Set oFolder = CreateObject("Shell.Application").BrowseForFolder(0, "Choose a folder", 0)
If (Not oFolder Is Nothing) Then GetFolder = oFolder.Items.Item.Path
Set oFolder = Nothing
End Function

PS: To answer your question: No, updating footers isn't inherently easier than doing a Find/Replace. Every Section in a document has 3 footers any one of which may be linked to the corresponding footer in the previous/next Section.
