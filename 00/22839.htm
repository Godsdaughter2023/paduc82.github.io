graham mayor say:

You can do it with a simple macro on a PC e.g. as follows. I am not sure whether this will work on a Mac

Option Explicit

Sub SaveSelectionasPDF()
Dim oRng As Range
Dim strFileName As String
Const strPath As String = "C:\Path\" 'The path where you want to save the PDF
    Set oRng = Selection.Range
    strFileName = strPath & "MyPicture.pdf"
    oRng.ExportAsFixedFormat OutputFilename:=strFileName, _
                             ExportFormat:=wdExportFormatPDF, _
                             OpenAfterExport:=True
lbl_Exit:
    Exit Sub
End Sub

hansv mvp say:

Try this. Change the constant TargetCol to the column you want to look at:

Sub FindAndDelete()
    Const TargetCol = "C"
    Dim TargetCell As Range
    Set TargetCell = Columns(TargetCol).Find(What:=0, LookAt:=xlWhole)
    If TargetCell Is Nothing Then
        MsgBox "Target value 0 not found", vbExclamation
    Else
        Range(TargetCell, Range(TargetCol & Rows.Count)).EntireRow.Delete
    End If
End Sub

benoit barabe say:

You should NOT clear your cache with /safeboot for this problem.

We are internally converting the PDF to another vector format and with some PDF the rendering complexity can increase in the process. We are immediately displaying a low resolution version of the images while we asynchronously render the high-quality one. Admittedly, we found out that Atom-level processors were taking quite some time to render in high quality. The app is fully functional thanks to the background rendering, but it does create some annoyances.

One "weird" work-around is to create another notebook locally, right-click on the notebook, properties and "convert to 2007 format". Now do your printouts to the 2007 notebook then move the page/section to the 2010 notebook. The image quality will be slightly less, the file size will be somewhat larger, but it should be a lot more responsive. This will have the effect of caching the data as images, which are faster to display.

(Note: You should also update to SP1 before doing this which improve some conversions between 2007 and 2010)

Apologies for the trouble...
