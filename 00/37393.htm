a. user say:

Hi,

 

If you are not able to import all of your contacts or contacts’ data into Windows Live Mail using a CSV contacts file, this may be due to one of the following:

 

I. Incorrect field mapping: Contact fields in Windows Live Mail don’t map to the contact information present in the CSV file being imported

 

II. Data in CSV file is unreadable: Windows live Mail does not recognize text or related contacts data in the CSV file being imported.

 

I. Incorrect field mapping:

 

When importing contacts into Windows Live Mail, only those fields that are selected and mapped in the “Map the fields you want to import:” step will be imported. To ensure that all the data contained in your CSV contact file is imported, select each field in the CSV and provide a mapping for Windows Live Mail Fields by following these steps:

  1. Open Windows Live Mail and then select Contacts in the lower left navigation menu.
  2. Select Import from the Home tab ribbon menu.
  3. Select CSV as the format of the contacts that you are importing.
  4. Select the CSV file that you would like to import contacts from.
  5. When Prompted to Map the fields you wish to import, be sure to place a checkbox next to every field that contains data that you’d like imported.
  6. Upon checking a previously checked field you will be presented with a “Change Mapping” dialog. Select the appropriate field name from the drop down box to map the field.
  7. Click Ok in the “Change Mapping” dialog box and repeat steps 5 and 6 for any additional fields that contain data you’d like imported into Windows Live Mail.
  8. Click the Finish button to complete the importing process.

 

Note: Any fields that are not selected in the “Map the fields you wish to import” will not have its data imported.

 

II. Data in CSV file is unreadable:

 

Sometimes Windows Live Mail does not recognize information stored in one or more contact fields within the CSV file being imported. In such cases, only those contacts that contain recognizable recognized by Windows Live Mail. 

 

This issue can be addressed by removing that contact from the CSV file using a spread sheet editing software such as Microsoft Excel or a text editing software such as Notepad. The steps given below can help you determine which contact information in the CSV being imported is not recognized by Windows Live Mail.

 

Make a backup of existing contacts.

 

Note: If you’re experiencing errors in importing contacts, contact information may be lost or duplicate contacts may have been unintentionally added into Windows Live Mail. To prevent any loss or duplication of contact information, a backup of existing contacts within Windows Live Mail is strongly recommended before moving on to the next steps. If you don’t currently have any contacts in Windows Live Mail you can skip to step 2.

 

 Make a backup of existing contacts:

1.     Open Windows Live Mail and then select Contacts in the lower left navigation menu.

2.     Select Export from the Home tab ribbon menu.

3.     Select whether you would like to export contacts to Comma Separated Values of Business Cards.

a.     Comma Separated Values: All contacts will be exported to a single text file with a .csv extension.

b.     Business Card: Each contact will be individually saved into a separate file with a .vcf extension.

4.     Select the location that you would like to export the file(s) to and follow the steps in the Wizard to complete the export of the contacts.

 

Remove existing contacts:

 

After backing up existing contacts, we’ll remove the contacts from Windows Live Mail. Before we can do so, we need to correctly determine the number of contacts that are being imported to Windows Live Mail.

 

1.     From Windows Live Mail, select Contacts in the lower left navigation menu.

2.     In the left navigation menu under the Contacts heading select “All Contacts”

3.     In the list on Contacts select any of the contacts to highlight them and then press CTRL+A to select all contacts.

4.     Press the Delete key to delete all contacts.

NOTE: if you have Social Network contacts you may not be able to delete contacts in this manner. To work around this you may temporarily remove any linked connected services from your Windows Live home page.

 

Determine how many contacts are being imported:

 

1.     Complete the import process using the CSV contacts file (see steps in 1. Incorrect field mapping: provided above where not all contacts were being imported)

2.     Determine the total number of contacts that were imported. This is listed in the left navigation menu under the Contacts heading and is displayed next to “All Contacts”.

1.     Open the Contacts CSV in Microsoft Excel or Notepad.

2.     Determine the row that needs to be removed from the CSV contact file.

a.     Use the total number of imported contacts from step one as the base row.

b.     Add one to this total due to the Header/Title row at the top of the CSV file.

c.     Go to the Row/Line for the updated number (contacts imported from step 2  + 1).

d.     Confirm that the contact was successfully imported into Windows Live mail.

                                         ii.    Save the CSV file.

                                        iii.    Repeat the actions in Step 2 to remove existing contacts to prevent duplicates contacts from being added into Windows Live Mail.

                                        iv.    Import contacts from the newly updated CSV contacts file.

                                         v.    Confirm that all of the contacts have been imported from the CSV contacts file.

 

Note: If not all contacts have been imported complete the actions in c) and d) to remove any additional contacts that Windows Live Mail is unable to read.
