steve rindsberg say:

I think you can do what you want w/o any VBA.

Create a main presentation that users will open first.  In it you have nothing but buttons, text or whatever you like that you assign action settings to. The action setting is a hyperlink to another PPT file.  Each bit of content that you want to present as a "module" will be in its own PPT file, linked to the main file in this way.

As Shyam mentions, each file should be set to Kiosk mode in the Set Up Slide Show dialog, and each should include whatever navigation buttons users will need to move from slide to slide.  That will prevent the arrow keys, space bars and such from switching slides.

Each of the presentations should also, wherever appropriate, have an End Show button (any shape with an End Show action setting).

That will end the presentation and return to the main menu presentation.

Put all the presentations in the same folder before you start linking, make sure the users keep them all in the same folder at their end and you should be good to go. 

There may also be a way to do this with custom shows.  You might want to do some tests with all your slides in one presentation, each module's slides collected into a custom show, then instead of hyperlinks to external show, you can do hyperlinks to custom shows, each set to show and return.

That gets a bit weird in kiosk mode; NEXT SLIDE buttons will cycle from the end of a custom show to the beginning of the custom show instead of gonig back to the first slide in the presentation; you could get round that by using an action setting on a button/shape that returns the user to slide 1.

petrib say: Actually, it is a bit more complicated than that.

With Symbian operating system version 9 (and later), the installation file format was changed, as well as the actual executable/binary program file formats). And S60 3rd Edition based phones are based on Symbian OS v9.1 (and later).

Symbian also invented a new file extension for digitally signed installation files, .sisx.

S60 3rd Ed. also requires mandatory signing of the installation files.

However, the actual file extension does not determine whether a file is really signed or unsigned. And unfortunately Symbian's tools when a file is signed, they leave them with the .sis extension, even if the recommendation is that signed files use .sisx and that is why you see both.

Main thing, regardless of the extension being .sis or .sisx, you have to make sure that it is for S60 3rd Edition. As there are also earlier S60 versions using earlier Symbian OS versions, and those files are also .sis files, but will not install/work on S60 3rd Ed. devices.

As Symbian changed the file format, they really should have also changed the file extension and made it mandatory. That would have saved people from a lot of confusion.

How do I know how many browsers are OPEN and how do I close them? - Microsoft Community

bernardsaucier say:

How do I know how many browsers are OPEN and how do I close them?

Original Title:  closing open browsers
How do I know how many browsers are OPEN and how do I close them?
kosh vorlon say:

Hi Bernard,

 

By "browsers" I'm assuming you mean open windows (as opposed to open tabs within the same window).

 

It is not necessary to open the browsers to close them.  It can all be done from the taskbar faster and easier.

 

If you mean open IE instances (and not all tabs in each), then the taskbar will show you all that are open.  If many programs are open, they may be consolidated under one IE task item.  If this is the case, there should be an arrow to the right of the IE taskbar button which, if clicked, will show you how many (or rather, the names of each of the) instances of IE are concurrently open.  I'm not sure if this is true with all versions, but for me, if there is a consolidated grouping, besides the arrow to the right to see them, there's also a number to the left of the name (which changes from a file type icon to a folder type icon and changes the name from the individual items to the category under which those items are consolidated) which identifies how many separate entries are in that taskbar button (so you can get a quick count).

 

You can then click on the arrow and then right click on the name of any you want to close and click close, then that instance of IE will close (along with all the open tabs in it).  If you want to close all of them, then right click on the IE button itself and then click Close Group and all open IE windows will close (though it may take a few moments depending on how many are open and how many have tabs and whether or not you've configured IE to request confirmation before closing tabs which will mean doing it for every instance where more than one tab is open unless that isn't set and then they'll all simply close).  If they aren't grouped and are separately listed on the taskbar, then you just need to right click on each and click on close to close each of them in turn (and just manually count the number to see how many there are). 

 

Tabs are different and if you want to see how many of them are open, you actually need to visit each of the separately open windows and manually count them.  There's no place to see a grand total of all the tabs.

 

If you are using several different browsers at the same time, they will not group up together as I indicated though each will probably act separately as I just explained (all 6 I have installed work that way but I can't speak for every one in the world) and you'd need to do essentially the same thing for each separate browser.

 

I hope this helps.

 

Good luck!
