stefan blom say:

Unfortunately, you can't edit LISTNUM fields. (Well, in some versions it was possible to right-click the field and edit it, about the same way you can edit a multilevel list, but this doesn't seem to work anymore. It was never 100% reliable, anyway.)

The LISTNUM field is actually designed to pick up the numbering from a multilevel list. The exception is if you use it together with a "list name," such as the built-in names (LegalDefault, NumberDefault and OutlineDefault) that you see in the Field dialog box.

For complete control over field numbering, use SEQ fields. The downside is that those are single-level, of course, so you would have to use multiple fields if you want to fake a multilevel list. Another downside: the SEQ field isn't supported in all "stories" (such as headers and footers).

Alternatively, you may prefer to create a multilevel list associated with styles. See http://www.shaunakelly.com/word/numbering/numbering20072010.html.

davidf2 say:

To my knowledge the html coding engine in Pub 2010 is the same as 2007, but I also know that Outlook has changed since 2003, and I think again between 2007 and 2010. So it may be that the problem is not with Publisher as much as it is with how Outlook 2010 renders the html that Publisher generates. It is hard to say, but MS has made a lot of changes to make Outlook more secure.

About the only thing you can do is deconstruct your newsletter, stripping it down to just the plain text without any formatting, and start adding formatting and design elements one at a time testing as you go until you find what you are doing now that is skewing your layout. You will probably have to K.I.S.S.

I sent you some links to troubleshooting that tells you some of the things that can cause problems including print formating, non web fonts, etc. However there is no definitive list of what can cause the problems you are having and that is why I am suggesting starting with a plain text box, single space with no special paragraph formatting, and then building your newsletter one step at a time to find out what is breaking it...and then designing around it.

You might also consider sending the whole message as a single jpg. It is likely that the text will be a bit fuzzy but at least the layout should survive. In Pub 2010 under File > Options > Advanced > Web Options you will see the option to send as a single jpg. At least the layout would survive and you can send as the message body.

And seriously, if it is so important that you include the newsletter in the body of the email message then consider investing in constant contact or something similar. They have experts that test their html formatted messages agains all email clients and you will enjoy more consistent results.

And finally, you can increase the resolution of the pdf file if you want sharper text...

Good luck.

Conditional Formatting by Weekday - Microsoft Community

aimeezink say:

Conditional Formatting by Weekday

Hello,

I'm trying to make an excel sheet where each cell is a different color depending on what day of the week is in it. For example, I want all cells with Sunday dates to be orange, all cells with Monday dates to be yellow, etc. I want this to apply to columns D - R. But I just want each individual cell to change color, not the entire row or entire column or anything.

Thanks!

jasles say:
You would use conditional formatting with a formula rule in each cell to look at date in that cell and change the color.

Use the WEEKDAY() function as the formula in the rules

myall_blues say: Let's say your dates will be in D2:R20. Select that range D2:R20.
Conditional Formatting->New Rule->Use a formula...
Enter the formula =WEEKDAY(D2)=1
Set the fill color to what you want for Sunday.

Now repeat 6 more times, =WEEKDAY(D2)=2 for Monday, =3 for Tuesday etc. changing the fill colour each time.

aimeezink say:

This worked great! Thank you!

However, it's doing this weird thing where when I put in =WEEKDAY(D11)=7 for Saturdays, it turned all the blank cells that color as well. Any idea what that's all about?

myall_blues say: Sorry my mistake. =WEEKDAY returns a value even if the cell is blank, so you need to trap it.
Change the last one (for Saturday) to =AND(WEEKDAY(D11)=7,D11<>"")
