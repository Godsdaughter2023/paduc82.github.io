Multiple Desktops Tips - Microsoft Community

drew1903 say:

Multiple Desktops Tips

Once you have created a number of, any number of Desktops:

1. You can move across them & back again by  Ctrl+Win <--- OR --->

2. Hit the Task view icon; now you can move items, Universal APPs, browser windows, word docs, etc. (open windows) from one Desktop to another.  There are 2 ways... (1) Rt Clk on the item you want to Move & pick the new, different Desktop for it OR (2) Drag & Drop.  The Rt Clk, also, offers New which, starts (creates) that item in a new Desktop OR Drag & Drop on the New Desktop (+) icon (@ the right of the screen).

3. You can grab any browser, in any Desktop via the IE icon on the taskbar.

4. Alt+Tab will show all windows across all Desktops.

Personally, I recommend this Feature of Windows 10.  Can, really facilitate being nicely organised.  Can have common themes, groups or 'families', topics or areas of commonality in their own Desktop instead of having everything mixed up together in one.  I have used it daily since the onset of using Windows 10 some 5 months ago.

shultzcharles say:

If you select cells on both sides of the hidden area, then press Alt-O, R, U. This will unhide all rows between the selected cells.

If you hid all of the rows from, say, row 25 to row 1,048,576, then it's not possible to select cells on both sides of the hidden area. To do that, it's still not difficult. In the upper-left, just below the menu bar, is a field that displays what cell is currently selected. Enter into that field "A1048576". Then, while holding down the Shift key, press the Up arrow. This will select the range you need. Then, press Alt-O, R, U and it will unhide all of the selected, hidden rows.

How to import an Excel worksheet into Access table?

Hi all,

Excel 2003 | Access 2003 | WinXP

I have an Excel file with multiple tabs, but I only want to import a particular tab into an Access table.  I am not sure the exact vba syntax for doing that when using Transferspreadsheet.  Can you share with me a code snippet to import?

Gerry

 


hansv mvp say:

Use code like this:

DoCmd.TransferSpreadsheet acImport, acSpreadsheetTypeExcel8, "tblImport", "C:\Excel\MyWorkbook.xls", True, "MySheet$"

The last argument specifies the sheet name followed by $.
