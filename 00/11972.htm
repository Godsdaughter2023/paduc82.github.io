Function StringToArray(Query As String) As Variant
   Const StrLen = 127 ' Set the maximum string length for
                      ' each element in the array to return
                      ' to 127 characters.
   Dim NumElems As Integer
   Dim Temp() As String
   Dim i
   ' Divide the length of the string Query by StrLen and
   ' add 1 to determine how many elements the String array
   ' Temp should contain, and redimension the Temp array to
   ' contain this number of elements.
  
   NumElems = (Len(Query) / StrLen) + 1
   ReDim Temp(1 To NumElems) As String
  
   ' Build the Temp array by sequentially extracting 127
   ' segments of the Query string into each element of the
   ' Temp array.
  
   For i = 1 To NumElems
      Temp(i) = Mid(Query, ((i - 1) * StrLen) + 1, StrLen)
   Next i
  
   ' Set the function StringToArray to the Temp array so it
   ' can be returned to the calling procedure.
  
   StringToArray = Temp
End Function

In the module add following before the sub

Private Declare Function timeGetTime Lib "winmm.dll" () As Long

timeGetTime-t is the time that your sub has needed to run, expressed in milliseconds.

    Tackling Overflow Error For Numeric Data - Microsoft Community

    faraz a. qureshi say:

    Tackling Overflow Error For Numeric Data

    In one of the macros I need the user to provide a range of any number of cells to work upon. However, how to proceed for a large number?
    In manner of a simple example what could be the most appropriate way to achieve the number of cells present in a large range, let's say $B$2:$XFC$1048575 as the following leads to an Runtime OverFlow Error '6':

    Debug.Print Range("$B$2:$XFC$1048575").Cells.Count

    Even using the type double as in the following example doesn't cater the need in this regard:

    Sub test()
      Dim CellCounter As Double
      CellCounter = ActiveSheet.Cells.Count
      Debug.Print CellCounter
    End Sub

    Please see if you can help in this regard and explain the reason the data type like double, having a capacity of 1.79769313486232E+308 can't even handle a count of 17,179,869,184 cells?

    Thanx in advance.
    hansv mvp say:

    The Count property is of type Long, and a Long cannot be larger then 2,147,482,647. As a consequence, if the range contains more than 2,147,482,647 cells, referring to the Count property will raise an overflow error.

    As an alternative, assign the number of rows and the number of columns to separate variables of type Double, and multiply these:


    Sub CountLarge()
        Dim rng As Range
        Dim dblRows As Double
        Dim dblColumns As Double
        Dim dblCells As Double
        Set rng = Range("$B$2:$XFC$1048575")
        dblRows = rng.Rows.Count
        dblColumns = rng.Columns.Count
        dblCells = dblRows * dblColumns
        Debug.Print dblCells
    End Sub

    hansv mvp say:

    No, VBA does not consider every numeric data as Long. The Count property of a range is defined as a Long. The Excel VBA help says:


    Excel Developer Reference
    Range.Count Property

    Returns a Long value that represents the number of objects in the collection.


    So Count cannot handle counts larger than the largest Long: 2,147,482,647.


    If all your users will have Excel 2007 or later, you can use the CountLarge property that was introduced in Excel 2007. CountLarge returns a Variant and handles ranges of any size:


    Sub CountLarge()
        Dim rng As Range
        Dim varCount As Variant
        Set rng = Range("$B$2:$XFC$1048575")
        varCount = rng.Cells.CountLarge
        Debug.Print varCount
    End Sub

    I was looking for a way to create a list style in one document and reuse it in another document with copy/paste, without having to create a Word template.  This question has been asked in this forum here and here, but those questions are locked, so I couldn't answer there. Here are the steps that I used.

    In a plain document, I created the list style, using the technique shown at https://shaunakelly.com/word/numbering/numbering20072010.html . In my particular case, I wanted a bulleted list that looked like the default list, with the first bullet at 0.25", text position for first level at 0.4", and an additional indent at each level of 0.15".  Here is a picture of that particular layout.

    Once that was defined in my plain document, the new style appeared in the numbered bullets menu as a list style (highlighted in the menu, because I had selected the list I made):

    Now, here is the trick.  I opened a new document.  Then I copied the list from the first document to the new document.  To get the style applied to a brand new list in the second document, I just typed in the new list, using the default styles.  Once the new list existed, then the list style from the first list (the one that was copied from the first document) appears in the same menu in the second document for numbered lists as a list style.  I can then apply that style to the new list, and delete the copied list, and now the style has been copied from one document to another, and is available for new lists in that document.

    Hope this helps someone.

    stefan blom say:

    The recommended approach in Word is to set up a list style AND associate each level of the list with a paragraph style (see https://shaunakelly.com/word/numbering/numbering20072010.html).

    You can then copy both the list style and the nine paragraph styles to another document.

    However, the easiest and stablest approach is to create a template with your styles. If, for some reason, you want to add the formatting to an existing document, you can create a new document based on your template and then import the existing document.
