greginmich say:

Generally speaking, Windows 8 users should just leave things alone and let Automatic Maintenance do its daily Quick Scan. You’ll see the “Last scan” entry for the built-in scan on the Home tab when you open up Windows defender. But you might have problems with the automatic scan, or with leftover scan tasks, if you upgraded to Windows 8. I’ve also had the automatic scan lock up my own PC, which came with Windows 8 preinstalled. If the automatic scan doesn’t stop when you start using the computer, you can try rescheduling Automatic Maintenance for a time when your PC isn’t in use. Just open Action Center, expand “Maintenance”, click on “Change maintenance settings”, and then pick a time for the scan to run. Unlike the MSE scheduled scan, the Automatic Maintenance scan is very punctual. If that doesn’t help, you can disable the Windows Defender Scheduled Scan task in Task Scheduler by right-clicking on it and choosing “Disable”. Be careful not to disable any of the other tasks in the Windows Defender folder.

 

There’s normally no need to disable the built-in scan because it stops with any user activity when it’s working properly, and thus doesn’t have any impact at all on the PC’s usability, which is definitely an improvement. Under no circumstances should you try to fix problems yourself by modifying the Windows Defender Scheduled Scan task, because that could impair the ability of Automatic Maintenance to run its other tasks in its default “idle-time-only” mode. That mode has a very limited time window.

 

You should also specifically avoid the Eight Forums method for adding a scan by modifying the built-in task. The author is an MVP, so his hack gets a lot of copycat recommendations, but it’s based on the mistaken belief that Automatic Maintenance runs an update but not a scan.

 

“For Windows Defender, Auto Maintenance will only check for updates. Only the task schedule you created will run the daily scan.” (Page 2)

 

So the author isn’t aware that if you use his method to add a Full Scan it will also convert the Automatic Maintenance scan to a Full Scan and thus create a potential problem for the Automatic Maintenance workflow. The Windows 8 Task Scheduler isn’t as simple as previous versions, so it’s going to take some time for the “experts” to catch on. There generally isn’t any need for a scheduled Full Scan – but if you really want to add one, there’s only one right way to do it:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02

Sub RemoveLeadingSpace()
Dim R As Range
Application.EnableEvents = False
On Error GoTo ErrH:
If TypeOf Selection Is Excel.Range Then
For Each R In Selection.Cells

End If
Next R
End If

It is not quite as simple as that.

 

Phones always have to be below a particular SAR value, and depending on which part of the world you are in it is typically 2. That is already a very low level, so when you are comparing figures like 0.35 and 1.02 you are comparing two very small numbers below another small number that is already a threshold many times below which any harm would be caused.

 

You also should know that the figure quoted is the maximum for a range of different tests and conditions including against the head, body and hand and different channels and voice and data modes, also combined with WiFi etc..

 

All this means that in  a typical situation when you are just making a phone call the SAR value in most phones is below than the figure quoted - although again you are still only comparing two very small numbers....

 

and the reason why Nokia doesn't allow you to change the power levels is that those are very carefully calibrated to meet the GSM and WCDMA standards otherwise the device would not be certified and allowed for sale, and would most likely not work....
