Well... they're not much use for that purpose.

 

The names are misleading. The function will return the first or last record in disk storage order in a Table; and you have no control over where on the disk Access might choose to store your records.

as a criterion on the datefield, and include whatever other fields you want to see in the main query. Min() will return the earliest date, of course.

jan de messemaeker - project mvp say:

Hi,

I admit it is confusing, so let me try to clarify.

 

What you see under Change Working time is what drives the planning. You can have different calendars for different resources and even tasks. The project calendar you find here will only be applied for tasks without resources nor task calendar.

 

This being said, you have to be aware that Project calculates all schedules by the minute (not an option, that is how it works). Many users, however, like to use the day as a practical unit so Project wants you to define it. In your case when you enter a duration of say 2 days, Project will record that as 840 minutes. This is true for all tasks (conbtrary to calendars which may vary).

When (as is the case in your file) there is a contradiction between hours per day option and working time, this can lead to curious results. For instance if you enter a duration of 8 days this will translate to 56 hours - that will be scheduled via an 8 hr/day working time so the real duration of the task will be 7 days only (although the duration field will show 8 days..)

So you better synchronize the two indeed.

 

As for default start and end time, they are used by Project to complete dates that are "incompletely" entered, that is by day only. Again, if they are diffeent from the scheduling (working time) values curious results may be obtained. Again, it is best to synchronize the two.

Sub PrintDocuments()

'~~> Folder where the Excel Files are.
Const strSourcePath = "C:\Temp\"
'~~> Path for the log File
Const LogPath = "C:\"
'~~> Name of the Log File
Const strFile = "Log.txt"

Dim objFSO, objExcel, objWorkbook, objSheet
Dim objFolder, colFiles, objFile, a

Set objFSO = CreateObject("Scripting.FileSystemObject")
Set objExcel = CreateObject("Excel.Application")
Set objFolder = objFSO.GetFolder(strSourcePath)
Set colFiles = objFolder.Files
Set a = objFSO.CreateTextFile(LogPath & strFile,True)

For Each objFile In colFiles
    Set objWorkbook = objExcel.Workbooks.Open(objFile)
    Set objSheet = objExcel.Workbooks(1).Worksheets(1)
    a.WriteLine(objWorkbook.name & " Printed at " & time)
    '~~> Print to Default Printer
    objSheet.PrintOut
    objWorkbook.Close False
Next

a.Close

Set a = Nothing
Set objSheet = Nothing
Set objWorkbook = Nothing
objExcel.Quit
Set objExcel = Nothing

End Sub
