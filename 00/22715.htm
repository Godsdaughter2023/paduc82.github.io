joeu2004 say:

DD wrote:

just learning and struggling.  Problem:  create a formula in cell G4 to display the text "Yes" if the number of C3>1 and the D3>2.  Both conditions must test true to display "Yes", otherwise display "No"

To understand how to knit pieces together, it might help to remember that each piece is a separate function.

So work from the inside out.

To create an expression to determine if two conditions are true, we use the AND function.  The form of the AND function is AND(condition1, condition2).   (You can have more conditions.)

In your case, the two conditions are C3>1 and D3>2.  So the AND expression is AND(C3>1, D3>2).

Caveat:  In your region, you might use semicolon to separate function parameters.  So you might write

AND(C3>1; D3>2).  Mentally make that translation one way or the other for anything that anyone writes.

Similarly, we use the IF function to return one value or another value, based on a condition being true or false. The form of the IF function is IF(condition, value_if_true, value_if_false).

In your case, the condition is the AND expression we built above.  The value_if_true is "Yes"; and the value_if_false is "No".  Substituting:

=IF(AND(C3>1, D3>2),"Yes","No")

PS....  Note that a complete formula is introduced by the equal sign ("=").  But there is only one equal sign for the formula, not for each not write =IF(=AND(...),...).  That is a common mistake made by beginners.

ken sheridan say: If you must use datasheet view for the subform, then my solution using 'hybrid' controls (to which Duane referred you) won't help I'm afraid.  You would have to use continuous forms view, which allows you to create a 'hybrid' control by carefully superimposing a text box on a combo box to give the appearance of a single combo box control.  

There is a very simple solution, which is not to use 'surrogate' numeric keys but 'natural' (usually text) keys.  As the values of each text column in the referenced  tables are not usually distinct, however, this means that the keys will often need to be composite multi-column keys.

You'll find an example in ComboDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes an example using 'natural' composite keys.  The image below shows the conventional model using numeric single column keys, and below it, the alternative model using composite keys.

<<  Sorry, for some reason I can't upload the image.  See the relationships window of my demo file instead >>

The first thing to say here is that the alternative 'natural' model is perfectly valid in the database relational model, even though we would normally use surrogate keys.  As regards your current problem, it means that you can use simple single column queries as the RowSource property of a correlated combo box, without data apparently disappearing in rows other than the current one.  In my example, as it has three combo boxes, the lowest in the hierarchy, in which a parish is selected simply has to reference both the district and county combo boxes above it as parameters in its RowSource property, which is:

SELECT Parishes_NatKey.Parish
FROM Parishes_NatKey
WHERE Parishes_NatKey.County = Form!cboCounty
AND Parishes_NatKey.District = Form!cboDistrict
ORDER BY Parishes_NatKey.Parish;

BTW, note the use of the Form property here to reference the current form rather than referencing it by name as a member of the Forms collection.  This is possible because all the controls are in the same form.  In fact you can even omit the Form property completely and reference the controls solely by name.

Each combo box bar the highest in the hierarchy, is requeried in the usual way in the AfterUpdate event procedure of that above it in the hierarchy.

In my demo the form's AllowDatasheetView property is set to False (No), but if you change it to True (Yes), then you'll be able to switch the form to datasheet view, in which you'll see that it works just the same as it does in continuous forms view.

Hi,

1. Clearing the Internet Explorer History will reset the Popular Sites list so that blocked sites will appear on it after the site is visited a few times.

When clearing the History, you should uncheck (at the top) "Preserve Favorites website data."

Clearing History will clear the entire list of sites, so it will take a few days of browsing to rebuild the list.

2. Clearing the History is safer, but you can also edit the registry,

HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\TabbedBrowsing\NewTabPage\Exclude

Values in the right pane will be excluded sites and may be deleted. They're encrypted so you won't know which site you're removing from the list. I would clear them all.

- - - - - - - - -

Regedit details:

From Start> Run (or Windows logo key + r) type in REGEDIT and hit Enter.

In Registry Editor select Computer in the left pane then File> Export. Save that REG file in a convenient location. Double-clicking that REG file will restore the previous registry and undo any changes, if necessary.

Drill down, by clicking the + to

HKEY_CURRENT_USER

    Software

        Microsoft

            Internet Explorer

                TabbedBrowsing

                    NewTabPage

                        Exclude

Select Exclude and in the right pane, right-click and delete each value except default.

Exit Registry Editor and restart.
