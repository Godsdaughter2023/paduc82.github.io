daniel pineault say:

The simplest solution would be to use the .SendObject Method.

You build a string containing whatever message you want and then use it in the SendObject call.

The general idea would be something along the lines of (you need to use your control names obviously)

Dim sMsg as Sting

sMsg = "Record: " & Me.RecId & vbcrlf & "Comment: " & Me.Comments

DoCmd.SendObject acSendNoObject, ,,"*** Email address is removed for privacy ***;*** Email address is removed for privacy ***;",,,"Your Subject", sMsg,True

Once you get thing working the way you like, then you can switch the True parameter to False, so the e-mail is sent without any user intervention, or leave it as is if you want the user to always have a chance to review it prior to it being sent off.
