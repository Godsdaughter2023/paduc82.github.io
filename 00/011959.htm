As Bob was trying to explain, Word doesn't divide the content into pages in the way you are thinking. Everything in a Word document flows from the top to the bottom. To move content around, you use cut and paste and let Word figure out the page breaks, or manually add and remove breaks.

Limitations and issues when embedding fonts in PowerPoint - Microsoft Community

jaz isherwood say:

Limitations and issues when embedding fonts in PowerPoint

Looking for all potential issues when embedding fonts via PC using PowerPoint 365. 

We used to avoid embedding fonts as it would always be unreliable but through testing I've noticed more consistent results. Wondering what issues other people have faced when embedding fonts? Any limitations of doing this?

Hypothetical deck would be used on PC and/or Mac on any version of PowerPoint. 

Thanks in advance, look forward to reading.

steve rindsberg say:

Factors to consider:

PPT Version/Windows vs Mac PPT

Windows PPT has been able to embed fonts for 20 years or more.

Mac PPT just recently got the ability to embed fonts/use fonts that the Win version has embedded, so if you need to use embedded fonts on Mac PPT, you're limited to updated versions of Office 365. It won't work on PPT 2011 and earlier or (assuming it's available on Mac) PPT 2016.

Font embedding permissions

PowerPoint will only embed fonts that the font vendor has marked as embeddable.

There are several types of embeddability: not embeddable, view embeddable (you can view the font but not edit with it), edit embeddable (you can use the embedded font to make further edits to the presentation), and install embeddable (rare; when you open the presentation, the font gets installed on your computer and stays; you can use it to create other documents).

Font type

Windows PPT can no longer use PostScript/Type1 fonts (older Adobe fonts, for example). What it can't use, it can't embed.  Some OpenType fonts contain PostScript versions of the font; PPT doesn't seem to be able to use them either.

Font names

Differences between font naming conventions on Windows vs Mac can confuse things.

Best bet is to test.  And test.  And test.  Create the presentation with the fonts you want to use, save with fonts embedded then try to test the file on the oldest version of PPT you want to target, on each platform (Mac, Windows).

FirstName LastName (Company)

I tried few combinations but no luck so far.

Please help.

Public Sub ChangeFileAs()

    Dim objOL As Outlook.Application

    Dim objNS As Outlook.NameSpace

    Dim objContact As Outlook.ContactItem

    Dim objItems As Outlook.Items

    Dim objContactsFolder As Outlook.MAPIFolder

    Dim obj As Object

    Dim strFirstName As String

    Dim strLastName As String

    Dim strFileAs As String

 

    On Error Resume Next

 

    Set objOL = CreateObject("Outlook.Application")

    Set objNS = objOL.GetNamespace("MAPI")

    Set objContactsFolder = objNS.GetDefaultFolder(olFolderContacts)

    Set objItems = objContactsFolder.Items

 

    For Each obj In objItems

        'Test for contact and not distribution list

        If obj.Class = olContact Then

            Set objContact = obj

 

            With objContact

            ' Uncomment the  strFileAs line for the desired format

 

                'Lastname, Firstname (Company) format

                ' strFileAs = .FullNameAndCompany

               

                'Firstname Lastname format

                ` strFileAs = .FullName

              

                'Lastname, Firstname format

                ' strFileAs = .LastNameAndFirstName

              

                'Company name only

                ' strFileAs = .CompanyName

              

                'Companyname (Lastname, Firstname)

                ' strFileAs = .CompanyAndFullName

               

               .FileAs = strFileAs

 

                .Save

            End With

        End If

 

        Err.Clear

    Next

 

    Set objOL = Nothing

    Set objNS = Nothing

    Set obj = Nothing

    Set objContact = Nothing

    Set objItems = Nothing

    Set objContactsFolder = Nothing

End Sub End Sub
