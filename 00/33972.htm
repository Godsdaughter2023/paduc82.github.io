gemzon mar say:

Hi TalynX,

Based on the description of your concern, it seems like it is a hardware compatibility issue. This is a common issue to all devices manufactured prior to Windows 10. If there was a recent update done on your computer, then there is a possibility that an updated generic driver has been installed on your device or the latest update is not fully compatible with your current video card driver. Since you've mentioned that you already tried updating all your device drivers, many I know how and where did you get those drivers? Did you update your video card driver as well? If no, I highly suggest uninstalling the existing video card driver then install the latest version (Windows 10) directly from the manufacturer's website. You may also want to check if there is an available firmware update for your laptop. Most of the time, updating the BIOS firmware addresses hardware compatibility issues.

Keep us posted.

Situation has resolved itself.

I re-installed the Dell drivers (essentially rolling back the drivers since Dell don't update their drivers for this model anymore) and then reinstalled the most recent Windows default drivers.

nikatreus say:

Windows 7 Home Premium 64-bit refusing to recognize 1.5 TB external hard drive.

I have a problem with Windows 7 (64-bit) refusing to recognise an external hard disk drive (with the HDD plugged into a USB port and powered on, the drive does not appear in the 'My Computer' view, Device Manager or Disk Manager).

The basic facts:
PC: Acer Aspire 7740G laptop (Intel 2.27 GHz CPU, 4 GB RAM)
OS: Windows 7 Home Premium 64-bit
External HDD affected by HDD non-recognition bug: Toshiba Storeart 1.5 TB
External HDD unaffected by HDD non-recognition bug: LaCie 250 GB

The 1.5 TB drive worked without fault on my old laptop (unfortunately stolen): an Acer Aspire 5720Z (1.30 GHz CPU, 2 GB RAM) running Windows Vista Home Premium 32-bit.
Initially, it also worked perfectly on my new laptop (specs above).
A couple of weeks after I bought the laptop, the laptop only registered sporadically that the drive was plugged in.
After a while, it refused to register the drive at all.

All other external hard drives and USB flash drives work fine on my laptop. My 1.5 TB drive refuses to work on my laptop, but works just fine on 90% of all other machines running Windows 7 32- or 64-bit, Vista or Mac.
Taking the drive back to Hifi Corporation and Incredible Connection and testing it on some other machines SEEMED to yield the following results (I couldn't test on enough machines to discern a clearer pattern):
- Acer Aspire 7741G with Windows 7 Home Premium 64-bit: drive not recognized.
- Windows 7 32-bit machines (mostly Packard-Bell, Dell and Lenovo): drive recognized in all instances.
- Non-Acer makes of laptop running Windows 7 64-bit, Windows 7 32-bit and Windows Vista 32-bit: drive recognized in all instances (if my memory serves correctly).

The following results are also applicable:
- My ancient desktop system (AMD Sempron 1.10 GHz, 512 MB RAM, Windows XP Pro SP3): drive not recognized.
- A friend's basic-spec MacBook (more than five years old): drive recognized and accessible.
- Creating a Virtualbox virtual machine on my laptop's main (OS) partition and installing Windows XP Pro SP3 on that didn't work either (the drive is not registering on the VM's device manager).

Re-formatting my laptop's hard drive, re-installing the OS and plugging in the drive did not work.

Aside from a vague indication that Acer machines running Windows 7 64-bit seem prone to this problem, I can discern no other rhyme or reason.

It's also been suggested that the laptop's motherboard USB chipset drivers are at fault. Surely, if this were the case, the computer would register no USB devices at all? Every other USB device I own works perfectly with this PC. In any case, Acer upgraded to the latest mobo chipset drivers. Still no joy.

As far as I'm aware, a hard drive is a hard drive is a hard drive. A relatively simple plug-and-play device that a modern operating system should have no trouble registering and interfacing with by self-installing the appropriate driver(s).

Does Windows 7 64-bit have any known compatibility issue with Toshiba hard drives, or with any make of hard drive above a certain size? (My LaCie 250 GB external drive continues to work perfectly with this laptop, as it always has.)

Before using the Toshiba drive for the first time, I formatted it from FAT32 to NTFS. Could that be the cause of this problem?

Microsoft's help forums have given no joy.
As far as I can tell, this problem (or very similar) appears to have been part of Windows 7 more-or-less since the beta development stage, but it seems a dedicated patch to fix it has not yet been released.

The 'solution' presented by the MS help forums is to uninstall the affected device from Device Manager, then re-attach the device and allow the plug-and-play drivers to re-install.
This only works if the OS recognises the drive at that particular moment. If the non-recognition bug is currently in effect, the affected drive won't show up in device manager or disk manager, and thus can't be uninstalled.
In any case, uninstalling the drive then re-attaching the USB cable and allowing plug-and-play drivers to be installed by the OS is not a solution. The day after I tried this (during a momentary phase where the OS acknowledged the drive), the non-recognition issue was back.
According to the forums, it also seems that wherever two or more external drives connected via USB are used in tandem, the larger drive is usually the one that suffers non-recognition.

Standard troubleshooting procedures (plugging the drive into all the PC's different USB ports (four in total), swapping USB cables and power supplies, accessing the drive through other computers etc.) has revealed that the 1.5 TB drive is perfectly functional.

I have an unknown account and an unknown contact. How do I go about finding out who these belong to?

Original title: unknown account

marilyn o say: You can use the ProfileList in the registry to identify the Users SID.

To get to the registry, go to a Run command.
Press Windows key and R. Then type in: regedit.

1. Navigate to:HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList

2.Under the ProfileList key, you will see the SIDs. By selecting each one individually, you can look at the value entry and see what username is associated with that particular SID. This will be in the right pane.

Note: The SID to profile association is only for local machine based profiles. If a user is referencing a "server based" .usr or .man profile in user manager for domains they will not be listed in this area of the registry to associate the username to SID.


If the account is not being used, you can delete from the registry. Be sure to back up the registry before making any changes as they will be permanent.


Let us know if this helps.
