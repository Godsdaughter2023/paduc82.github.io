Access 2003 Conditional Formatting - Microsoft Community

karen_frp say:

Access 2003 Conditional Formatting

In Access 2003, can I use conditinal formatting to change the precision of a decimal number based on whether it is a pound price or a case price (i.e. case pricing should be 2 places past decimal point; pound pricing should be 4 places past decimal point)?  Thank you.
dirk goldgar say:
In Access 2003, can I use conditinal formatting to change the precision of a decimal number based on whether it is a pound price or a case price (i.e. case pricing should be 2 places past decimal point; pound pricing should be 4 places past decimal point)? 


No, conditional formatting only allows you to control a few specific format properties, and the DecimalPlaces property isn't one of them.  On a form in single-form view, you can use VBA code to control how many decimal places are displayed based on the type of pricing, but you can't do that if the form is in continuous-forms view.

If you need to, you can probably create and display a calculated control that shows the price formatted appropriately, using a controlsource expression similar to this:

    =IIf(PriceType="Case", Format(Price, "$0.00"), Format(Price, "$0.000")

 

You could have a normal text box bound to the Price field for data entry, sized to match the calculated control and placed behind it.  Then when the user clicks on (or tabs into) the calculated control, you can send the focus to the bound control, which will automatically bring it to the front.  When the focus leaves the bound control, it will go to the back again and be hidden by the calculated control.
