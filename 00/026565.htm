graham mayor say:

What I suspect that you want is to have the pasted text formatted to match the destination location. In that case what you want is

Sub PasteUnfText()
On Error GoTo oops
Selection.PasteSpecial _
    DataType:=wdPasteText, _
    Placement:=wdInLine
End
oops:
Beep
End Sub

If you want to then modify the pasted text alone, you need

Sub PasteAndAutoFormatText()
Dim oRng As Range, oStart As Range
    Set oRng = Selection.Range
    Set oStart = Selection.Range
    On Error GoTo oops
    With oRng
        .PasteSpecial _
        DataType:=wdPasteText, _
        Placement:=wdInLine
        .Start = oStart.Start
        'do whatever else you want with the inserted text e.g.
        .AutoFormat
    End With
    End
oops:
Beep
End Sub

'Autoformat' is a rather blunt weapon. You would probably be better formatting the inserted text range directly.

This issue cannot be solved because it's a feature on the Mac. Documents can only be SYNCed on CLOSE (see Microsoft technical document) and AutoSave does not change that fact; my autoSave was already on.

Read this Microsoft Technical document sent to me by an Microsoft engineer....

Restrictions and limitations when you sync files and folders

https://support.microsoft.com/en-ph/help/3125202/restrictions-and-limitations-when-you-sync-files-and-folders

"Open files can't be synced: A file that’s currently opened by an application can't be synced by OneDrive for Business. To sync the file, close any application where the file is currently being used, and then sync the file."


Furthermore, Macs cannot use co-authoring unless a user is a part of the Microsoft Insider program.

This feature cannot be used if the document is opened from the OneDrive folder on the desktop! Users have to save/open the document from Menu > File > Save/Open to the Sharepoint folder. Regardless of whether co-authoring is used the document will SYNC on SAVE if opened in this manner.

What a big mess of nonsense...

1. A PC will SYNC on SAVE regardless of how the document is opened, either from the Sharepoint folder via the File menu or the OneDrive folder on the desktop.

2. A Mac will only SYNC on SAVE by opening the document from the Sharepoint folder via the File menu. But if the same document is opened from from the OneDrive folder on the desktop, it will only SYNC on CLOSE.

3. Co-authoring works on a PC but co-authoring only works on a Mac if all the users are a part of the Microsoft Insider program.

4. Co-authoring is not required to SYNC on SAVE but the Mac cannot SYNC on SAVE if the document is opened from the OneDrive folder on the desktop (see point 1 and 2).

5. When Mac users open a document from the OneDrive folder on the desktop, save and forget to close the document, their document changes will be overwritten the next time they connect, if another user edits, saves and syncs a newer version of that document.

No wonder everyone is confused.
