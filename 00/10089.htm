bob jones aka: cybertaz say:
It isn't the Section Breaks that are creating the problem. The Document Map has no awareness of Section Breaks, its content is based strictly on paragraph formatting using Heading Styles or Styles that include the Outline Level 1-9 attribute.

Without seeing the document my first guess is that the return key has been used to manufacture 'blank space' in the document. If you'll display the non-printing characters (¶) you'll see that there is a vertical series of those symbols in the document, i.e.;
¶
¶
¶
¶
¶
¶

Those symbols represent empty paragraphs that correspond to the gaps in the Document Map Pane. Each one of those paragraphs carries the Style formatting that causes it to be recognized as a Heading. If you delete those meaningless empty paragraphs the gaps in the Document Map will be removed as well.

If there is a valid reason to have vertical space in the document there are more appropriate ways to do it rather than beating on the return key :-)

john mcghie say:
Yair Gidday Cobber:

In Word (even on the PC...) the "Language" is marked in the text.  The language controls the speller.  Language is a "property" of text, and can vary word-by-word in Word.

To begin, select the entire document, and go to Tools>Language> and set English (AUS) as the language.  Then click the "Default" button and say "Yes" to the prompt.

That does two things:

1)  It sets the whole of that document into English (AUS).

2)  It updates your Normal Template to set English (AUS) as the default language of the Normal style.  In a default Normal.dotm template, all styles are based on Normal style, so whatever language Normal has is inherited across the entire document.

The Normal.dotm template is the model from which all new documents are created.  For existing documents, you will need to use Tools>Language in each one.

Now, English (US) is poisonously pernicious.  Word's designers in Seattle cannot see a problem: I wonder why?  You may need to do the following to completely get rid of it:

1)  Close all documents.

2)  From within Word, use File>Open

3)  Navigate to /Users/~/Library/Application Support/Microsoft/Office/User Templates/My Templates (where ~ stands for your login-name)

4) OPEN Normal.dotm.  You must open it this way: if you double-click it, you will make a copy of it and your changes will be lost.

5)  Select all of the text (there apparently won't be any, so add some, so you can see what you are doing).

6)  Use Tools>Language to set English (AUS)

7)  Delete the text you added.

8)  Save and close the template, and quit Word (to ensure your changes are written back to disk).

You should be good to go from there.  Language is a propert of text.  It can be a property of a style (best if it is) and it can also be a property directly-applied to the text (poor practice, but it happens).  For whatever reason, Normal.dotm often picks up a vestigial Septic influence as a directly applied property, which you need to comprehensively clobber if you want to speak 'Strine.

You Custom Dictionary, BTW, should always remain set to "No Language".  That is so your speller added words remain available in all documents you handle.  Otherwise, if you open a document created in Pommie or Kiwi, your custom dictionary will be disabled if it's set to AUS.

Hope this is more than you need to know...
