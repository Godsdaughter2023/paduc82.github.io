diane poremsky mvp (slipstick.com) say:

If outlook crashes or windows forces a reboot and you had messages open, /restore should reopen them, but it won't work for normal shutdown when you closed outlook.  There will be occasions when it doesn't reopen the messages and windows though. For example, I use Outlook on monitor 2 and I know if it comes up on monitor 1, the messages won't be reopened - Outlook doesn't remember which monitor it was on, its not going to remember which messages. 

To see if it worked (both outlook and windows are insider builds), i opened a few items, waited a few minutes then opened Task Manager and End Task to force Outlook closed.  I used outlook.exe /restore in the run command reopened the message and reply I had open. (When you are composing a message, you may lose content if it wasn't auto-saved as a draft). 

FWIW, The restore switch should be automatic as part of the recovery process following a crash or windows forced restart. 

If you use multiple Outlook windows (for Calendar, contacts, mail etc) File > Exit will save the open windows and they will reopen the next time you open Outlook. It does not remember open messages though.
