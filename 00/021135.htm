Answer: Unlike for customized Skype name, a system-generated Skype name (with live:), cannot be used as a sign in credential to Skype. Issue 2: I can find the skype account "A"  (myself account) and message with her, is that normal ?

Answer: This is not normal as while signed in to a Skype account, you can neither search nor add the same Skype account. Furthermore, it is not possible to send messages to yourself.

To isolate the case, please try to view the account and check if having the same Skype name.

jllatham say:

You have declared it somewhere AFTER you have started a Sub or Function.  This kind of statement must go:

#1 - into a regular code module (not worksheet code, userform code or workbook code modules)

#2 - BEFORE any declaration of a Sub or Function.

Open up a regular code module in the VBA Editor.  Click anywhere in the code displayed then Press [Ctrl]+[Home]

If you see the words Option Explicit at the top of the module, then put your Public statement immediately below that statement, before anything else.

If you don't see the words Option Explicit at the top of the module, fine, just put your Public declaration there at the top.

john w. vinson say:

There's actually a logic to the color codes. Each code is a 24-bit binary number composed of three 8-bit colors: the first two digits are the amount of blue, the second two the amount of green, and the last two the amount of red. So &HFF0000 is pure blue ((255 units of blue, FF hexadecimal=255 hex), zero of green, zero of red; &H0000FF is pure red, &H000000 is black (no colors at all), &HFFFFFF is blazing white.

ken sheridan say: As Tom says, why do you need to loop?  The standard formula for returning the sum to which the principal will grow when interest is compounded is:

   S = P(1+i)n

So 100 GBP invested over 5 years at 5% compounded annually would be computed as follows:

? 100*((1+(0.05))^5)
 127.62815625

The same investment compounded monthly would be as follows:

? 100*((1+(0.05/12))^60)
 128.335867850351

If the requirement to loop is due to this being a school assignment as Tom surmises, i.e. it's an exercise in looping rather than a realistic solution, what you need to do is assign the principal to a variable, then loop for the number of terms, multiplying the value of the variable by 1+n, (where n is the interest rate expressed as a fractional value, e.g. 0.05 for 5%) at each iteration of the loop.  The final value of the variable would be the sum to which the principal would grow.
