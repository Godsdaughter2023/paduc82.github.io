macgyvernl say: I can confirm that this issue exists on multiple phones for multiple users. I recently noticed that several friends of mine using hotmail / outlook.com addresses were sometimes breaking the thread of messages. It turns out that the specific messages which break the thread are
-sent using the outlook.com android client
-missing in-reply-to and references headers.

Expected behaviour is that these headers contain the Message-ID of the message being replied to, to help mail clients group related messages in conversation threads. The Outlook.com android app therefore deviates from section 3.6.4 in RFC 5322, which states that `reply messages SHOULD have "In-Reply-To:" and "References:" fields as appropriate and as described below.' Section 3.6.4 of RFC 5322 also contains specific requirements for these headers. Sections 2.1.9 and 2.1.10 of RFC 4021 are also relevant in this context.

It should be noted that the stock android e-mail app has the same problem, which is listed on the android bugtracker as issue 2629 ( http://code.google.com/p/android/issues/detail?id=2629 ) which was merged into issue 28017 ( http://code.google.com/p/android/issues/detail?id=28017 ). It remains an unresolved bug there as well. Therefore, your advice of testing this with the stock email app might expose the same issue, but it is nonetheless a bug in both clients. Please forward this to your support specialists regardless of the findings using the stock android e-mail app.
