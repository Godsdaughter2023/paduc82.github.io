cindy meister say:

Hi Herb

If formatting is "in force" at the end of a line, and you press Enter, then the formatting will continue to the next line. That's by design, and there's no way to tell Word "don't do that".

The only excpetion is if you use styles. It sounds like you could be doing something like a Q & A: the one paragraph is an introduction to the following text. In that case, it would make sense to set up a style (or a pair of styles). The style definition for the "Q" paragraph would be bold AND you'd set the "style for the following paragraph" to the Normal style, or a style of your choice. In this case, when you press Enter a the end of the paragraph formatted as "Q", the other style would be applied automatically to the new paragraph.

Cindy Meister, VSTO/Word MVP

ken sheridan say: It's not a question of a row being referenced by more than one table, but by more than one row in a single table.  If a row is referenced by a one row in each of two tables, then only one row will be returned, but if it's referenced by two rows in one table then two rows will be returned.

As regards the error, is the ChooseRecords form a subform?  If so you need to reference it via the Form property of its parent form with the syntax:

Forms!ParentFormName!SubformControlName.Form!ControlName

Note that SubformControlName is the name of the control in the parent form's Controls collection which houses the subform.

If not, what happens if you open the ChooseRecords form and enter some values in the relevant controls, then open the query directly?
