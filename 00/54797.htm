jumpxav say: Hi Mouneshwar R,

Thank you for your help. I worked out that it is in fact a hardware issue in this particular make of Dell laptop. I did the things you suggested and used device manager to uninstall USB root hubs etc and in doing so I noticed an error in the MMC Disk section. The icon and alerts flashing up did momentarily say MMC device. 

I checked online and found that the MMC device or the memory card reader on this particular laptop is at the front beneath the touch pad, meaning the sensors for the memory card are inadvertantly set off by movement and pressure on the keyboard itself. The sensors were touching the "blank" card which is provided with the laptop to go in the gap when not using a memory card. Hence why the sensors appear to notice it and then immediately realise it is not an actual card.

Thanks for your help, and if you see anyone having this problem again, I would suggest fixing errors on the MMC Device or removing the blank card. I have removed it and I haven't had the problem since.

The NTVDM.EXE process in Windows is used to run legacy doc (16BIT) processes on the now 32BIT or 64BIT machines.  NTVDM has been around for a long time however the arcitecture of the operating system has changed, especially with Windows 7.

It may be that your application is trying to do something that Windows 7 simply does not allow any more - with certain changes things changed and some applications no longer work.
