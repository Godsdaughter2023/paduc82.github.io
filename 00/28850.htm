hi standaloneadmin,

yes, we can restrict specific users or groups from using their onedrive for business.

1. go to sharepoint online admin center > user profiles > manage user permissions under people.

1. go to sharepoint online admin center > user profiles > manage user profiles under people.
2. select the user which you want, click find, then click the downward arrow item menu to choose manage site collection owners.
 
3. change the site collection administrators to remove the user, click ok.
4. then you can choose manage personal site in the above picture to check the user’s permission to see whether the user still have some other permissions for his personal site(users and permissions > site permissions ). if there is, we need to remove it. after that, the user will not be able to access his onedrive.

daniel pineault say:

Before going down the automation route, why not simply apply an unique index that covers the fields that should not duplicate?

 

As for your code, try something more like:

 

Private Sub Form_BeforeUpdate(Cancel As Integer)
    Dim iAns            As Integer
    Dim rs              As DAO.Recordset
    Set rs = Me.RecordsetClone    ' get the list of records bound to the form
    rs.FindFirst "[LastName] = '" & Me![LastName] & "'" _
                 & " AND [FirstName] = '" & Me![FirstName] & "'"
    If rs.Match Then   ' There is a record for this combination already in
        iAns = MsgBox("This registration already exists! Click OK to jump to it, Cancel to skip", _
                      vbOKCancel)
        Cancel = True    ' cancel adding the new record in any case
        Me.Undo    ' erase what the user has entered up to now
        If iAns = vbOK Then
            Me.Bookmark = rs.Bookmark    ' jump to the found record
        End If
    End If
    Set rs = Nothing
End Sub

 

 

Lastly, having already developed such a system in the past, you cannot impose a uniqueness just based on First and Last names.  Many people have the same names, you will need to add a 3rd element to garantee the niqueness and not block valide 'dupliactes'.  You could use: SIN, Address, Phone, ...
