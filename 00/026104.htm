Hi S.N.N.

The 'suggested meeting' add-in is a default installed add-in Office 365 OWA (Outlook Web App). It would detects requested keywords or entities, such as meeting suggestions or addresses, that are in your message, and it underlines them. The add-in can run only after you have given your consent to activate it, and you've selected the text underlined by the add-in. Table = MHR

Field1= DocNumber

Field3 = STACD

SELECT DocNumber, Max(STADTTI) AS MaxOfSTADTTI
FROM MHR
GROUP BY DocNumber
ORDER BY DovNumber;

john w. vinson say:

You will need a Subquery to do this - using a query finding the latest date as a criterion for an "outer" query to pick up the code:

SELECT DocNumber, STACD, STADTTI

FROM MHR AS OuterQuery

WHERE OuterQuery.STADTTI =

   (SELECT Max(InnerQuery.STADTTI) FROM MHR AS InnerQuery

    WHERE InnerQuery.DocNumber = OuterQuery.DocNumber);

teylyn say:

I see what you mean.

But that is not a bug or a design flaw.  A clustered column chart with one series will have one slot for each data point on the X axis. If another series is on the secondary axis, the X axis slot is still only wide enough for one data point. In order to show data points side by side, a secondary data series is required on both axes.

That basic behavior of charts opens a treasure trove of possibilities. By arranging the data source it is possible to achieve visualizations that don't come with Excel out of the box, for example Gantt charts, clustered column charts, or Waterfall charts. These charting methods depend on the relationship that exists between the series and the axes.

You are showing one approach that is widely used to plot two columns side by side on two axes:  add a series with zero values to each axis, so now each axis shows two data series and there are two slots for each data point on the X axis. This will show the

Another technique that does not require a zero value series is to format the two data series with different settings for gap width: reduce gap width on the primary axis, increase gap width on the secondary axis. The data points will overlap, but their differing width will ensure all data points are visible and the column height can be related to their respective Y axis (if it is labeled accordingly).

So, if you want the columns side by side, then you need a helper series with zero values. If you don't want to use a helper series, the columns can be plotted with different width to make all data points visible.

macropod say:

Tracked changes involving fields are often tricky beasts, as can be tracked changes in tables. The following two macros can be used to handle the fields issue. The first one updates all fields, then calls the second to accept any tracked changes applying to them. The second macro (which uses the supplied WithinField function) can be used independently of the first.

Sub RefreshFields()
Application.ScreenUpdating = False
Dim TOC As TableOfContents  ' Table of Contents Object
Dim TOA As TableOfAuthorities ' Table of Authorities Object
Dim TOF As TableOfFigures ' Table of Figures Object
Dim pRange As Range ' Word Range Object

With ActiveDocument
  ' Loop through Story Ranges and update.
  ' Note that this may trigger interactive fields (eg ASK and FILLIN).
  For Each pRange In .StoryRanges
    Do
      pRange.Fields.Update
      Set pRange = pRange.NextStoryRange
    Loop Until pRange Is Nothing
  Next
' The following routines are necessary because the foregoing updates only page numbers
' in TOCs, TOAs and TOFs - field updating doesn't update TOC, TOA or TOF contents.
  ' Loop through Tables Of Contents and update
  For Each TOC In .TablesOfContents
    TOC.Update
  Next
  ' Loop through Tables Of Authorities and update
  For Each TOA In .TablesOfAuthorities
    TOA.Update
  Next
  ' Loop through Tables Of Figures and update
  For Each TOF In .TablesOfFigures
    TOF.Update
  Next
End With
Call AcceptTrackedFields
Application.ScreenUpdating = True
End Sub

Sub AcceptTrackedFields()
'Accepts all tracked field updates
Dim RngSel As Range, Story As Range, Rng As Range, Rvn As Revision, Fld As Field
With ActiveDocument
  Set RngSel = Selection.Range
  .ActiveWindow.View.ShowFieldCodes = False
  ' Loop through all range objects and accept tracked changes on fields
  For Each Story In .StoryRanges
    For Each Rvn In Story.Revisions
      With Rvn
        If .Type = wdRevisionDisplayField Then
          .Accept
        ElseIf .Range.Fields.Count = 0 Then
          If WithinField(.Range) = True Then
            Set Rng = .Range
            With Rng
              .Fields.ToggleShowCodes
              If InStr(.Text, Chr(21)) = 0 Then
                .MoveEndUntil cset:=Chr(21), Count:=wdForward
                .End = .End + 1
              Else
                .MoveEndUntil cset:=Chr(21), Count:=wdBackward
              End If
              .Revisions.AcceptAll
              .Fields.ToggleShowCodes
            End With
          End If
        End If
      End With
    Next
  Next
  RngSel.Select
End With
Set RngSel = Nothing: Set Rng = Nothing
End Sub

Function WithinField(Rng As Word.Range) As Boolean
Dim i As Long
WithinField = True
Rng.Select
i = Selection.Start
With Selection
  .Fields.ToggleShowCodes: .Fields.ToggleShowCodes
  If .Start = i Then
    .MoveRight
    .Fields.ToggleShowCodes: .Fields.ToggleShowCodes
    If .Start = i + 1 Then WithinField = False
  End If
End With
End Function

a. user say:

Windows 10 Pro x64 bit only supports 2 CPU's.

Only Windows 10 pro for Workstations supports 4 CPU's.

1. First install Windows 10 Pro x64 bit.

2. Then convert it to Windows 10 pro for Workstations by changing the product key

Click start, settings, Update & Security, Activation

Change Product key to professional workstation
