Function ExportRecordset2XLS2(ByVal rs As DAO.Recordset, strSheetName)
    Dim xls As Object
    Dim xlwb As Object
    Dim xlws As Object
    Dim fld As DAO.Field
    Dim strPath As String
    Const xlCenter As Long = -4108
    Const xlBottom As Long = -4107
Dim strTitleRange,strHeaderRange, strBodyRange as String

On Error GoTo err_handler
strPath = "C:\Sports\Roster.xlsx"
Set xls = CreateObject("Excel.Application")
    Set xlwb = xls.Workbooks.Open(strPath)

    xls.Visible = False
    xls.ScreenUpdating = False
    Set xlws = xlwb.Worksheets(strSheetName)
xlws.Activate

'Define ranges for formating
    intFields = rs.Fields.Count
    intRows = rs.RecordCount
    strTitleRange = "A1:" & Chr(64 + intFields) & "1"
    strHeaderRange = "A2:" & Chr(64 + intFields) & "2"
    strBodyRange = "A3:" & Chr(64 + intFields) & (intRows + 2)

'Build TITLE Row
xlws.Range("A1").Select
    xls.ActiveCell = Format(Now(), "YYYY") & " Roster (" & strSheetName & ")"

'Build HEADER Row
    xlws.Range("A2").Select

    For Each fld In rs.Fields
        xls.ActiveCell = fld.Name
        xls.ActiveCell.Offset(0, 1).Select
    Next
    rs.MoveFirst

 'Paste Recordset into Worksheet(strSheetName) starting in A3

On Error Resume Next

    Set rs = Nothing
    Set xlws = Nothing
    Set xlwb = Nothing
    xls.ScreenUpdating = True
    Set xls = Nothing
    xls.Quit

Exit Function

    DoCmd.SetWarnings True
    MsgBox Err.Description, vbExclamation, Err.Number
    Exit Function

Some conditions do turn gray in the Ribbon when choices are not applicable, such as attempting to insert a form into a text box through the Developer tab. Barring those instances, try the following standard fixes:

  1. Update Word – Make sure your version of Word is updated (Word > About Word should state 14.1.3). To update, select Check for updates in Word's Help menu. Run it more than once until you get a "No more updates" message.
  2. Bypass preferences and Normal – Quit Word and launch it while pressing the Shift key. This will temporarily (until the next normal launch) bypass your user preferences and your Normal.dotm template. Check the status of the fonts in the emerging Font Validation window, then select (place a checkmark) and remove any font that does not pass the test. Finally, if Font Book tells you there are duplicate fonts, go back to the previous window and select Edit > Resolve Duplicates (you may need to Select All; you can do this step duplicate elimination indepentently of the font validation step).

john korchok say:

Table text works the same way as text in placeholders: to change the style, you change the outline level by clicking on the Increase List Level or Decrease List Level buttons. Adding bullet characters to a line of text will not change the styling.

Table text must be formatting by hacking the XML. As with placeholders, you can create up to 9 levels of text, each with a different style. Since there is no program interface for setting levels of table text, the simplest approach is to format a text placeholder on a custom layout to look like the different styles you want. Then you can copy the XML formatting from the custom layout to the otherStyle section of the slide master XML. The syntax is exactly the same. Here's my article on the subject: XML Hacking: Default Table Text

I admit, it's not a simple technique, but it's currently the only way to get the job done.

mike h.. say:

Hi,

 

As a start I would remove the line

 

Application.Volatile True

What that line does is make the UDF recalculate every time the worksheet calculates and that isn't necessary and slows things down. Your UDF will still recaulate when the range referred to by the UDF changes even without this line.

 

One cautionary note is that the UDF will not recalulate if you change a colour of a cell in the range but removing the VOLATILE statement doesn't change that.
