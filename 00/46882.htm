limit reservable bandwidth - Microsoft Community

ugyen galey say:

limit reservable bandwidth

is it ok if i set limit reservable bandwidth to zero? Will it affect my computer updates? 
rajesh govind say:

 

Hi Ugyen,

 

The Windows Operating System reserves 20% of the total Internet bandwidth for the QOS or Quality of Service usage like Windows update, license renewal, etc. Thus, when you limit the Reservable Bandwidth of the operating system to 0, this will definitely affect the operating system activities like Automatic Windows Updates.

 

If a QoS-aware application reserves more bandwidth than it uses, then the unused, reserved bandwidth is available for use by other applications. The reservation does not ensure that the bandwidth will be available to the QoS-aware application because applications that are not QoS-aware might consume too much bandwidth.

 

For more information, you may refer to the TechNet article: “QoS Tools and Settings” from this link:

 

http://technet.microsoft.com/en-us/library/cc737728.aspx

juliannatramell say:

I had a similar problem.  The RDP audio driver was taking over even though I had selected "Play on remote computer".  Windows Media Player would then claim no audio device was available and I could get no sound from the remote computer.  A reboot of the remote computer would solve the problem.  BTW to do a remote reboot of the remote computer run "shutdown -r" since restart is not one of the options.  I disabled the RDP audio driver on the remote computer for remote desktop by editing the following key:

 

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Drivers32\Terminal Server\RDP

 

You should see the normal audio drivers under Drivers32.  Replace the TerminalServer\RDP audio drivers "rdpsnd.dll" with the normal drivers.  On my remote computer with a SoundMax the driver was "wdmaud.drv".

 

RDP audio will no longer be an option on the remote - the remote computer will only play sound remotely.

 

The original key:

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Drivers32\Terminal Server\RDP]
"wave"="rdpsnd.dll"
"mixer"="rdpsnd.dll"
"MaxBandwidth"=dword:000056b9
"wavemapper"="msacm32.drv"
"EnableMP3Codec"=dword:00000001
"midimapper"="midimap.dll"

 

The edited key:

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Drivers32\Terminal Server\RDP]
"wave"="wdmaud.drv"
"mixer"="wdmaud.drv"
"MaxBandwidth"=dword:000056b9
"wavemapper"="msacm32.drv"
"EnableMP3Codec"=dword:00000001
"midimapper"="midimap.dll"
