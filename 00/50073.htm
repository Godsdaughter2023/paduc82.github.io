¡firedog say:

1) You cannot drag a folder from Storage Folders>Imported Folders>local Folders>(FolderName) to a POP account's folder list.


No, you can't. Folders 'belong' to their parent, be it an account or Storage folders. To relocate messages from Storage folders to an account, create the folder under the account and then select the messages concerned, right-click and Move to folder.

In your case, it might make more sense not to bother. You can relocate folders to the top level under Storage folders with much less fuss, so you end up with a structure like this:
 
Account
Inbox
Drafts
Sent items
Junk email
Deleted items
Outbox
Storage folders
Drafts
Sent items
Deleted items
User folder 1
User folder 2
User folder 3
 
You can have the user folders in the order you want by giving them names starting with a prefix, like this:


This arrangement has the additional advantage that the user folders will not affected if ever you find it necessary to remove a POP3 account. In Windows Live Mail, removing an account permanently deletes all of the account folders and their contents, unlike some other programs.

3) There is no simple mechanism to rebuild the entire folder structure from scratch by using the files in the mail folder on the disk.


It's never a good idea to do any sort of file manipulation on files and folders in the store folder. Any changes made in the file system will not be visible to the message database, which will become confused when it cannot find files in the location it expected to find them. That's why you're seeing duplicated folders (one set of which is probably empty) as the database tries to recover from its confusion. The store folder is hidden by default for a reason: messing with it can have unfortunate consequences.

¡firedog say:

I googled "Check data integrity ...


We should bear in mind that this program is basically a simple mail client intended for the average home user. He wouldn't have a clue about data integrity, nor would he care about re-indexing and other more advanced topics - he just expects it to work.

Windows Live Mail makes extensive use of databases to store message data (but not the messages themselves), contacts and calendars. They are all of the same type as many other Windows databases such as those forming the meat of Windows Search, Windows Update and Event Viewer. The tools for managing this type of database (ESENT) have been included with Windows since NT days, so they are available to the advanced user. The utility esentutl.exe includes functions for defragmenting, recovering and repairing databases. Type esentutl /? in an administrator command window to see the options available to you.

If i browsed the web in private browsing, would the websites i view - Microsoft Community

boomsta say:

If i browsed the web in private browsing, would the websites i view appear in my activity reports?

I'm just curious because my computer administrator put activity reports on me, and i was just wondering if private browsing would prevent him from seeing what websites i go to.
myronh say: It depends on how you connect to the internet and how your computer admin is monitoring.

If you connect directly to the internet instead of going through company resources, then inprivate browsing should keep your browsing history private.

If you are using company resources to connect to the internet, then the admin can monitor traffic from/to a given computer from the web-server. If the admin is monitoring there, it doesn't matter what you do to hide web traffic on you computer, s/he is seeing what is going on at the outgoing end.

If using company resources but the admin is running reports from individual terminal to discover browsing history, then inprivate browsing will work.

Inprivate browsing doesn't leave tracks on your computer, but will still leave tracks of the IP calls on a web-server or router at your company.

antoneth ber say:

Hello CASHFLOW1,

To resolve your concern with System Restore Error 0x8007005, I suggest that you boot your PC in safe mode first before performing the System Restore. Safe mode starts Windows with a minimal set of drivers and services to help troubleshoot issues. To do so, please follow these steps:

  1. Open Settings.
  2. Select Update & security.
  3. Open Recovery.
  4. Under Advanced startup, select Restart now.
  5. After the PC restarts to the Choose an option screen, select Troubleshoot.
  6. Click Advanced options.
  7. Under Startup Settings, hit Restart.
  8. After the PC restarts, select 5 or F5 for Safe Mode with Networking.

Note: If a problem doesn't reappear when you start your PC in safe mode, you can eliminate the default settings, basic device drivers and services as possible causes.

Furthermore, you may also consider disabling your Anti-virus software temporarily before performing the System Restore.

Should you need further assistance, feel free to get back to us.
