bob jones aka: cybertaz say:

Contrary to the reply from Rosen Tong, the behavior in Mac Excel 2016 is no different than that in prior versions... The main ingredient is the manner in which you enter the data. For example, based on default settings, assume you are starting a new List in cell F5:

As you enter the data in that row use the Tab key (not the right arrow key) to move from one cell to the next. After entering data into the final cell, press the return key. [Note that full keyboards also have an enter key in the Numeric Keypad which does not operate in the same manner.]

When you do as described above the cell selector will go to F6. As you complete row 6, pressing return in the last cell will send the selector to F7, etc.

In an existing list, pressing return in the last cell in a row of data will cause the selector to return to the column where you started. IOW, if the data range is A1:H25;

  • If you select cell A1, then Tab to H1, then press return the selector will go to B1
  • If you start with cell D1, then Tab to H1 & press return the selector will move to D2

Behavior also can vary depending on whether you're working with a List Range or a Table.

If you're experiencing something different, please provide complete & accurate details describing the nature of the sheet content, the steps you're taking, & the response that occurs. Also, please indicate the present update level of your Office 2016 installation as well as the version of macOS.

stefan blom say:
Problems such as that one are usually caused by a poorly designed add-in interfering with saving changes to the Normal template. The problem must be troubleshot separately.

As a workaround, do the following to open the Normal template directly: From within Word, press Alt+F11 to display the Visual Basic Editor. Activate the Immediate window by pressing Ctrl+G. Type

NormalTemplate.OpenAsDocument

and press Enter. Normal opens in a separate window (see the file name on the title bar). Close the Visual Basic Editor. Change the defaults the way you tried to earlier, save and close Normal.

[Edit: corrected a typo]           

jay freedman say:
Try opening the Normal.dotm template itself, and making the change there. Verify that you can save the change and have it take effect in new documents.

Because the Templates folder is hidden by default, the Normal.dotm template can be hard to find. The simplest method is to press Alt+F11 to open the macro editor, press Ctrl+G to open the Immediate window, paste in the following command, and press Enter:

NormalTemplate.OpenAsDocument

Then you can close the macro editor and modify the spacing.

Instead of just modifying the spacing of the empty paragraph in the template, it's best to modify the default as follows:
  • Press Ctrl+Alt+Shift+S to open the Styles pane.
  • At the bottom of the pane, click the Manage Styles button (third from left).
  • In the Manage Styles dialog, click the Set Defaults tab.
  • Change the paragraph spacing (the "default of defaults" is supposed to be 8 pt After).
  • Because you're working directly in the template, it doesn't matter whether you choose "Only in this document" or "New documents based on this template".
  • Click OK.
  • Save and close the Normal.dotm template.
