Time format in Report adding hours that = more than 24 hr - Microsoft Community

peters_780 say:

Time format in Report adding hours that = more than 24 hr

I created a report that adds hours worked.  When the sum of the hours in the report goes beyond 24 hours its not displaying the sum of  hours worked properly.  In excel it is possible to format time to show hours created more that 24 hours.  Why it not happening in access report. 

dirk goldgar say:
I created a report that adds hours worked.  When the sum of the hours in the report goes beyond 24 hours its not displaying the sum of  hours worked properly.  In excel it is possible to format time to show hours created more that 24 hours.  Why it not happening in access report. 

I'm guessing that you are storing your hours in a Date/Time field, and that's not really designed to represent elapsed time; rather, it's meant to represent a moment in time.  And the date and time formats are meant to present such a moment in time.  You would do better to keep your time increments as, maybe, Integer or Long values representing minutes, or seconds -- whatever minimum unit of time you care about -- and then divide that into hours, minutes, and optionally seconds for presentation.

That said, here is a function that will convert cumulative time stored in a Date/Time field into a representation as H:MM:SS (allowing the H portion to grow to however many digits are needed).  I just threw it together now, so it may not be perfect.

Function fncFormatAsHMMSS(TimeAsDate As Date) As String

    ' into hours:minutes:seconds.
    '
    ' Will raise Overflow error if <TimeAsDate> exceeds ~69 years.
   
    Dim lngWork As Long
    Dim lngHours As Long
    Dim intMins As Integer
    Dim intSecs As Integer
   
    lngWork = CDbl(TimeAsDate) * 86400  ' convert to seconds

   
    lngWork = lngWork Mod 3600
    intMins = lngWork \ 60
   
    intSecs = lngWork Mod 60

scottgem say:
 In excel it is possible to format time to show hours created more that 24 hours.  Why it not happening in access report. 

Because Access is not a spreadsheet and its not Excel.

To expand a bit on Dirk's response. Access stores date/time values as a double precision number where the integer portion is the number of days since 12/30/1899 and the decimal portion is a fraction of a day (i.e. 6AM =.25). Therefore, more that 24 hours would spill into the integer portion and the time formats will ignore anything in the integer portion. So you need to use a function like the one Dirk listed.

stefan blom say:

You can use the Table of Contents Options dialog box to choose which styles should be used when Word creates a table of contents. See http://wordfaqs.mvps.org/TOCTips.htm#TOCOptions.

For an existing TOC, you can reveal the field codes and edit them directly in the document. Hold the Alt key and press F9 to show/hide field codes. For example, if your TOC field looks like this:

{ TOC \o "1-3" \h }

and you want to change it so that it only includes paragraphs formatted in the built-in Heading 1 paragraph, change it accordingly:

{ TOC \t "Heading 1,1" \h }

Press F9 to update the selected field. If prompted, choose to update the whole table. Press Alt+F9 again to hide field codes.

Microsoft offers two types of connectivity for ActiveSync or Windows Mobile Device Center.  By default Windows Mobile 5.0 or later devices connect using RNDIS.  Windows Mobile Devices prior to Windows Mobile 5.0 use a PPP connection.  The option to uncheck "enable advanced network functionality" switches the Pocket PC's connection between RNDIS and PPP.  This works for some devices because the software on the PC may block RNDIS while allowing PPP to connect.
