Facts about that symptom should be easily deduced by playing with the properties of the window, as you started to do below.

“Internet Options”in Control Panel is inoperative.

That symptom should be investigated.   E.g. start an elevated cmd window (aka Administrator Command Prompt) and enter:

control.exe  inetcpl.cpl,,6

What happens then?

Right click on mouse permits only Restore, Move, Minimize, Maximize, Close basic commands.

I think you may have to pay closer attention to which ones are operative (e.g. blue, not grey).   There is a set which is the new symptom indicating that the window involved is in Fullscreen mode.   E.g. instead of having Auto-hide checked or unchecked appropriately as was previously done, you may find (illogically) Move and Size enabled.  Pressing F11 should have changed the set of

If the window is in its Normal mode you will see (usually) that everything except Restore is active.  When that window is toggled into Fullscreen mode you will see only Move (as noted), Minimize, and Close are active.

If instead the window was originally in its Maximized state and then switched into Fullscreen you would see a different set and thus be able to deduce that case too.

In fact, another method of trying to get the same information is to press Alt-Space and then see which set of options are active in the window's window menu (instead of the Taskbar's thumbnail's impression of it).

Finally, another conventional method of changing window size and position is to use the Taskbar Cascade windows command.  That command also has some repair effect.  So, to use that, you would minimize all windows (e.g. press Win-d), restore the problem one, and then right-click on the Taskbar and use the Cascade windows item.

Sub ChangeSource()
