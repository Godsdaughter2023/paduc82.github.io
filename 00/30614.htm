adrianne buc say:

Thank you for posting back and apologies for the delay.

Your queries are more appropriate for our dedicated community in TechNet forums, we suggest posting it there.

To give you an idea for drive spanning, you can follow the steps below:

  1. Right-click on Start menu button and select Disk Management.
  2. Right-click the hard drive volume and select Delete volume.
  3. Click Yes to confirm the deletion of the current volume and all its content.
  4. Repeat steps 2 and 3 on the hard drives you want to combine.
  5. Right-click the unallocated space of the drive and select New Striped Volume (or New Spanned Volume).
  6. Click Next.
  7. Select the additional disks, one by one, and click Add.
  8. Click Next.
  9. Leave the default drive letter assignment and click Next.
  10. Make sure the file system is set to NTFS, allocation unit size is set to default, and choose a volume label.
  11. Check the Perform a quick format option.
  12. Click Next.
  13. Click Finish.
  14. You'll be prompted to convert the hard drive from basic to dynamic, click Yes to complete the task.

We hope this helps.

Hi Kibbeth. I'm Greg, an installation specialist and 9 year Windows MVP, here to help you.

When did this behavior begin? What was changed immediately beforehand? Is there a System Restore point to go back before then? http://home.bt.com/tech-gadgets/computing/how-t...

It is correct that Windows Boot Manager is set first in Boot Priority order, as this is invoked when you first install to UEFI and must stay set first, whereas in Legacy Mode the hard drive needs to be set first. Use only the BIOS Boot Menu to boot media, and only as a UEFI device.

It is strange that you can only start Windows from the BIOS Boot menu key and is likely a bug or anomaly with the firmware. So first thing to do is check for newer firmware on the PC or motherboard maker's Support Downloads web page, for your Dell Service Tag.

If no newer firmware then reset your firmware to optimal defaults. Back up Windows first as this may cause it to not boot. You may want to note the exact present settings for UEFI, CSM, Legacy BIOS or Boot, Secure Boot, Fast Boot and Boot Priority order so you can set them back.

You may even need to reinstall with the settings corrected by resetting to defaults, following these illustrated steps in this link which compile the best possible Clean Install of Windows which will stay that way as long as you stick with the tools and methods given, has zero reported problems, and is better than any amount of money could buy: http://answers.microsoft.com/en-us/windows/wiki...

I would run Dell hardware diagnostics as well from the F12 menu to see what it finds.

You can also Test your RAM overnight to stress it with a more reliable bootable test, following these steps which test both the sticks and the slots: http://www.sevenforums.com/tutorials/105647-ram...

I hope this helps. Feel free to ask back any questions and let us know how it goes. I will keep working with you until it's resolved.
