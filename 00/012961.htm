For completeness, I did want to post the 5 options we were considering.  

1. Account is deleted and Mailbox is stored in a recycle bin as an Inactive Mailbox for 30 days. To restore account must be restored (Using a license) and then Mailbox is restored and re-connected to the restored user. The User account can be locked out and the mailbox converted to a Shared Mailbox to grant another user access rights.

2. Create a .pst file from the Mailbox prior to the User being disabled then placed in Secure Storage Online using full security controls implemented via Rights Management. Note this must be done before the account I deleted in AD.

3. Create a shared mailbox using a restored AD profile in the cloud (Account is not disabled but is locked out). Note this must be done before the account I deleted in AD.

4. Create a “Hold” in Discovery Center that maintains the Mailbox in place hold as an inactive mailbox, using this method the mailbox data is searchable only via Discovery Center. Note this must be done before the account I deleted in AD.

5. Develop an offline storage system using an on premises exchange server, and connecting this to AD OU with non-synched User’s that are disabled.

Hi ToddMrskos,

Question 1： Are there options we can set in the tenant to extend the time past 30 days (i.e. 45 or 60 days)? 

The answer is no. By default, users cannot change it.

We recommend admins convert user mailboxes to shared mailboxes if related staffs leave. Admins can keep these mailboxes and convert it back at any time if they need (for my experience, staffs may come back to the original company in some cases).

Question 3: Any other options to keeping the data longer than 30 days for employees that leave and that are not on a Legal Hold?

Shared mailbox is our recommend method. Users can also use pst files to save data, but it may not be very convenient.

If you have any further concerns, please feel free to let us know. Meantime, any community members who are under the similar situation are welcome to share any experience.
