zack barresse say:

Great question! This is a somewhat confusing topic. As you found out, macros are not supported in Excel Online. Currently there is no way they can compile or run VBA code living in a spreadsheet. Could this be made to work? Yes, of course it can. The only thing limiting us and our technology is us. However, the time it would take to make that happen is severely offset by what we can do, and that is to use a web-based language to leverage the power of automation in a browser-based application, i.e. Excel Online.

All of the above info means Microsoft decided it would be a better direction to take Excel Online automation via Javascript instead of VBA. When you think about it objectively it's a smart move. Javascript is much more powerful and robust in a web-based app, and VBA is a very old COM tool and can't work across all endpoints. The transition between the two is proving difficult, as VBA is the industry standard in Excel automation. While it will be continued to be supported in Excel desktop, the same cannot be said for other Excel endpoints. There is a great many things you can do with Excel for iPad. Just look at the feature description in the iTunes store.

If you must have macros in your file, you basically have two options.

  1. Stay in desktop Excel
  2. Translate your VBA solution to a Javascript one

The latter is going to prove somewhat difficult, and in some situations impossible. The Javascript API isn't nearly as developed as VBA is. That being said, it's current release (v 1.4) is fairly robust, and Microsoft has been adding to it at a very regular cadence. Their commitment to expanding the API is inspiring towards future development. More information about the API can be found here.

I'm sure this isn't what you were looking for, but it is the current landscape for Excel and her automation story.
