Hi,

We also have another way in copying files to a disc.

Insert a burnable disc into the drive. Once the disc spins up, Windows will recognize it and ask what you want to do. The AutoPlay prompt looks different in different versions of Windows, but the options provided are similar.

Once you've selected "Burn files to disc", you'll be greeted by a dialog that gives you two options for burning your disc.

When you hit Next, Windows will take a some time to format the drive. This process can take a while.

Once the disc is formatted, you can use File/Windows Explorer to open the disc. When you look at its root location, you will see a message that says: Drag files to this folder to add them to the disc.

Locate the files or folders you wish to burn or you can drag and drop them to the disc or copy and paste them. When you add files, they will automatically be burned to the disc.

The Copying dialogue will appear and show you the status of the burning operation.

cmd

gpupdate /force

Bad number format of de-CH region in Windows 8.1 clean install - Microsoft Community

christophrüegg say:

Bad number format of de-CH region in Windows 8.1 clean install

It seems the number format of de-CH culture profile has been changed in Windows 8.1 and is now broken, although it seems to affect clean installs only (but not upgrades):
  • The decimal symbol should be a dot "." but now is a comma ",".
  • The digit grouping character should be a single quote "'" but now is a space " ".
The profile was fine up to Windows 8.0 but is clearly wrong now. Unfortunately this breaks all kind of Software that expects culture profiles to remain consistent, and also no longer represents what's common Switzerland (I'm Swiss myself).

See also http://www.localeplanet.com/icu/de-CH/index.html

Is this a known issue? Can we expect a patch to revert the de-CH profile back to what it was before?

Thanks,
Christoph Rüegg

christophrüegg say: Thanks for the reply, even though it is very surprising.

It is correct that some federal offices, among others the federal chancellery, do use the comma as decimal separator, but they are also the only ones that do so. The cited document is explicitly not a "standards document", but merely internal usage guidelines of some federal office and its publications. In fact, technically, there is no official standards document, but everyone (besides the mentioned offices) uses the dot as decimal separator in computing in Switzerland, making it at least the de facto standard.

Common repositories like unicode.org/CLDR and ICU all agree that the decimal separator in de-CH is a dot, not a comma. Also note that the decimal/number block on the Swiss keyboard does have a dot, not a comma. So do all pocket calculators.

I've talked to a few people about this, and everyone was very surprised and negative about the change (not representational though, of course). The German way of using a comma as decimal separator is generally frowned upon here in Switzerland and also considered as impractical (especially in lists of numbers, for example it breaks the common CSV format). 

I've noticed that in a similar thread in the German-speaking variant of this community, a local Microsoft Support Engineer confirmed that the new Win 8.1 behavior is indeed "wrong" and should be fixed.

Please consider to revert this change.

Thanks,
Christoph

Switching back and forth the decimal sign is really not an option, not for users and not for developers as me.

If a user has already existing documents (Excel or others) with the "." and create new ones with a ",", they would have to switch this option for each document.

Developers won't be able to do it right neither, even if we parse the numbers according to the Windows settings, it would fail with existing documents, otherwise it would fail with new documents. Currently we first try to parse with the system settings, then as a fallback try with a decimal point. This scheme includes guessing though and is not clear without ambiguity (other countries even use the comma as thousand separator).

I really hope this will be reverted as soon as possible.
