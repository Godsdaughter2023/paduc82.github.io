ken sheridan say:

The table isn't linked, but imported..............

That's the problem.  Importing data inserts values only into those columns corresponding to those in the source.  If you predefine the target table in Access to include a further column on the other hand, and then create a link to the source, an Excel worksheet in your case, you can then create an 'append' query which inserts not only the values from the source but a constant, "Toys" or "Games" in your case, into the further column in the target table.

You could import the data into the target table, having added the extra column to its design, but you would have to change the DefaultValue property of the extra column in the table design to "Toys" or "Games" as appropriate every time you import data.  It would be possible to amend the table definition in code to do this, but it would be far simpler to execute an append query based on a linked table, and include a parameter in the query so that the user can insert "Toys" or "Games" as appropriate at runtime.
