sazf say:

Hi Gloria

No it didn't because the data source I was using is the Apple Contacts, then selecting a group from that.  None of the screens offer an option to sort data and the Edit recipients field is grey out.

So I am on a MacBook Pro running Mojave, with Office 365 subscription and all are on latest versions of software.

In the end I made a work around detailed below.  The only annoying this is that of course it has to be done every time you want to print off because any updates to your contacts isn't included in the Excel spreadsheet you created.  But doing this is a lot less work than having to edit the whole massive merged document to put all the entries in alphabetical order when you are trying to create a directory.

1. In Contacts select all the members of the group you want to use in the merge.

2. Open a Numbers spreadsheet

Using Split screen is useful at this point.

3. Drag the selected members of the group into the Numbers Spreadsheet. Numbers creates a sheet with the field names as column headers.  You can close the Numbers spreadsheet now.

4. Export the Pages sheet to Excel

5. Save the Excel doc.

6. Open a new Word Doc, go to Merge, select Start mail merge, click on whichever document type you want, then click on Select Recipients and choose Existing List, then select the new Excel spreadsheet.  No need to enter a data range, just click on OK.

7. This is the fiddly Manual bit but it works and doesn't rely on lots of techie knowledge.  Click on Insert Merge field.  A drop down list appears that gives you all the field names as AutoMergeField and a number.  NB These do not obviously map onto the headers in your Excel table so at this point you need to add all these fields down to the number of headers (fields) you have in your  Contacts list.  I  use the standard Contact entry form so I have 15 fields, so I added Table1, AutoMergeField, AutoMergeField1, etc all the way down to AutoMergeField15.  

8. Then click on Preview Results and make sure you are on the first record.  This picks up the Field Names in the header row of your Excel spreadsheet so you can make sure you have got the fields you need in your document.  I wrote a list on a piece of paper so I could work with them (how clunky is that!).  But as long as you write it down once you can use the same list for doing this job every time!

9. Because you have got the data from an Excel Spreadsheet it automatically sorts it alphabetically!  (Come on Microsoft - you already make a version to work on Macs/OSX, surely you know we need to be able to sort data from an Apple Contacts file!)

After that it all works normally - when I came to finally merge and print out I selected a range from record 2 to the end to avoid having the record that just contains the Field names/Header titles in it.

If someone reading this can spot anything that would save any steps please add it in!.
