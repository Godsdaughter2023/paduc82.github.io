dale howard [mvp] say:

Oldmachead --

If your projects are small, in theory you could put all 40 of them together into a single project.  That would probably simplify things a great deal, especially with resource management and reporting.

If the projects are longer than a few dozen tasks, then you would probably want create a separate MPP project file for each of them.  This is where it gets more complicated.  You would need to create a shared resource pool file containing all of the resources to be used in all 40 projects.  Then you would need to link each project as a sharer file to the shared resource pool file.

To set cross-project links and to report on all of the projects, you would also want to create a master project, containing each of the 40 projects as a subproject.  Once you have created the master project, it is a fairly simple matter to set cross-project links.  To report on all 40 projects, you could display the Gantt Chart view and collapse each subproject.

Just some thoughts for you.  I will bet others in this forum will have some good ideas for you as well.  Hope this helps.

rod gill say:

As Dale said. I would try all projects in one file first but if the total task count approaches 2000 it is likely time for separate files for each project.

Note that master and resource pool files requires a higher level of understanding of project and careful use of them. Files mustn't be moved else file corruption will eventually occur.

Automatic requery Sub forms - Microsoft Community

ds at bell say:

Automatic requery Sub forms

I have a form that has 3 individual sub forms within it.  Each of the sub forms are based off a query.  I want all 3 sub forms to automatically re-query every 30 seconds.  For each sub form in the Properties settings I set the “Timer Interval” to 30,000 and the “On Time” to run a macro that has an action to Requery.  Individually when the sub forms are open they requery works fine.  It’s only when I insert the forms in another form is when they do not requery.  I did notices that if I click within the sub forms they do requery, but I do not want to have to click each sub form every 30 sec.  I there a way I can have all 3 sub form automatically requery at the same time?

ken sheridan say: You've misunderstood Scott's reply.  NameOfYourSubformCONTROL means you put the name of the subform control here, i.e. the control in the parent form's Controls collection which houses the subform.  This might or might not be the same name as the source form object.

I would usually tag subforms, i.e. the source form objects, as fsub and subform controls as sfc, so taking your Discrepancy subform as an example the subform would be named fsubDiscrepancy and the subform control would be named sfcDiscrepancy.  The syntax to requery it would thus be:

Me.sfcDiscrepancy.Form.Requery

scott mcdaniel say:

FWIW - many users will find this enormously distracting, especially if they are allowed to edit data in the subforms (since each time you requery they will lose their edits). Of course if you're building a "dashboard" type app, then you can ignore that little nugget!

To requery a Subform from the Main Form, you'd do this:

Me.NameOfYourSubformCONTROL.Form.Requery

Where "NameOfYourSubformCONTROL" is the name of the Subform Control object on the parent form. This may or may not be the same as the name of the Form you're using as a Subform.

Note this is for VBA, not Macros. You shouldn't be using Macros for things like this ...
