ken sheridan say: You seem to have the table name as the first column:

SELECT Mailings, [lastname] & " " & [firstname] & " " & [address] FROM Mailings ORDER BY Mailings.lastname, Mailings.firstname;

The first column should be the table's primary key, which would normally be an autonumber column with a name such as mailingID, in which case the RowSource property would be:

SELECT mailingID, lastname & " " & firstname & " " & address FROM Mailings ORDER BY lastname, firstname;

If your mailings table does not already have a numeric primary key column, then add an autonumber column to the table and save the amended table design.  The new column will automatically be filled with distinct values.

The other properties of the combo box would be like this:

BoundColumn:   1
ColumnCount:    3
ColumnWidths:  0cm;3cm;3cm
ListWidth:  6cm

If your units of measurement are imperial rather than metric Access will automatically convert them  to inches.  The important thing is that the first dimension of the ColumnWidths property is zero to hide the first column.  The ListWidth property should be the sum of the column widths.  Experiment with the dimensions to get the best fit.

If by 'subform' you mean a separate form being opened from the current form, as in my demo to which I referred you in another thread, then the code to go to open the form and go to the selected record would mirror mine, which is:

    Dim ctrl As Control
    Dim frm As Form
    
    Set ctrl = Me.cboGotoContact
    
    If Not IsNull(ctrl) Then
        DoCmd.OpenForm "frmContacts"
        Set frm = Forms("frmContacts")
        
        With frm.RecordsetClone
            .FindFirst "ContactID = " & ctrl
            If Not .NoMatch Then
                frm.Bookmark = .Bookmark
            End If
        End With
    End If

If you have a true subform, i.e. one embedded in the current form then it would be like this:

    Dim ctrl As Control
    Dim frm As Form
    
    Set ctrl = Me.cboGotoContact
    
    If Not IsNull(ctrl) Then
        Set frm = Me.YourSubformControlName.Form
        
        With frm.RecordsetClone
            .FindFirst "ContactID = " & ctrl
            If Not .NoMatch Then
                frm.Bookmark = .Bookmark
            End If
        End With
    End If

Note that YourSubformControlName would be the name of the control in the parent form which houses the subform, which might or might not be the same as its source form object.

However, to use a single form you don't need to embed a subform in it; just use a bound form in continuous forms view and put the unbound combo box in the form header, in which case the code would be:

    Dim ctrl As Control
    
    Set ctrl = Me.cboGotoContact
    
    If Not IsNull(ctrl) Then
        
        With Me.RecordsetClone
            .FindFirst "ContactID = " & ctrl
            If Not .NoMatch Then
                Me.Bookmark = .Bookmark
            End If
        End With
    End If
