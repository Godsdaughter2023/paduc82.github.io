suzanne s. barnhill say:

Generally speaking, this error message is misleading, and the usual solution is as follows. Go to Page Layout | Page Setup | Columns | More Columns... In the Columns dialog, click the preset picture for One (even though the document is already formatted for one column). This will check the "Equal column width" check box, which allows a single column to be resized to fit margin width as margins are changed. A bug in setting columns sometimes leaves this box unchecked (and there's no way to check it directly when you have only one column), resulting in a column that is too wide for the margins and doesn't adjust automatically.

Occasionally, the problem will be that there are two (or more) columns, one of them margin width, leaving no space for the second column, with the result that text at the bottom of the first column just disappears. The solution for that problem is the same.

BTW, it is usually not necessary to set a gutter when using "Book fold." Selecting "Book fold" automatically enables "Mirror margins," and you can just add the gutter amount to the Inside Margin.

john mcghie say:

Yes, it's certain :-)  Remember I said the built-in headings have "secret sauce" hard-coded into them that make numbering work?

When you copy them, you can't copy that, so the copies won't work for numbering.  You need to use the "originals" for the numbering and the copies for the un-numbered headings.

The secret sauce is actually the "Outline Level" property.  If you go into Format>Style>...>Modify>Paragraph, you will see the Outline Level property is set and greyed out in the built-in Heading nn series of styles.  It's actually that outline level property that controls the numbering, and controls whether the list styles that show "Heading..." in their samples will correctly attach or not.

On paragraph styles other than the built-in Heading series, that property is enabled and you can set it. You could use this to construct a List Style using styles other than the Heading series.  It's a lot of work, and sadly, it's not very stable. However, that's not the reason I don't recommend it: the real reason is that there can be only one  Outline List (which is what you are building) in a document.  So if you did build a second set of styles with their outline level properties carefully set from 1 to 9, you could then potentially get a "mixed" set of numbered lists in your document.  Which would be worse than you have now (certainly, less predictable).

So rather than go through the full agony of describing how to set up a set of nine styles, a List Template, and a List Style to lock the three together (and it's quite a bit of work...) I just say "Customise the built-in Heading styles to your taste and use them."  Saves all that effort and it "just works".

You can use Advanced Find Replace to search for the copy styles and apply the built-in styles.  Just take care that the "Find What" box and the "Replace With" box are truly empty, you don't want to change any text.

Hope this helps
