Import format for Project - Microsoft Community

Import format for Project

Hello,

What type of file should I be creating to import into Project?  Which can be exported from excel.

john - project say:
my_drive,
You should be able to import directly from an Excel Workbook, (no need for csv format), although a CSV format should also work.

However, be advised that importing from a non-schedule based application (e.g. Excel), to Project's schedule based format isn't necessarily a straightforward process. Some things to keep in mind.
1. Project is a scheduling program and calculates its schedule based on basic inputs. All you need from Excel is a task's start and duration, Project will calculate the finish date. Don't try to force Excel's data into Project's schedule. It may not "fit" like you think.
2. Project's dates are based on a working time calendar. Excel's dates are based on a 24 hour calendar. You may or may not get the start (i.e. 8:00 am) and finish (i.e. 5:00 pm) times you expect so it's good to check after the import.
3. Import map values are in text format. Make sure that's how they are expressed in Excel. For example, if you import percent complete, make sure it is shown as a text value in Excel (i.e. 100% should be 100)
4. If you are importing task links, I recommend you import them separately into an extra text field in Project. Your network structure in Excel may or may not be on track and attempting to import links directly into Project's Predecessor field could easily give unexpected results. After the import is complete and Project has calculated the schedule (i.e. start and finish dates), then you can copy the imported link data from the extra text field into the Predecessor field.

Hope this helps.

John

How do I create Macros in Excel spreadsheets - Microsoft Community

clicketysplit say:

How do I create Macros in Excel spreadsheets

After following the instructions in the Excel Help section it still will not create Macros.

 

Could anyone explain the correct way for creating Macros in Excel 2010?

 

 

jllatham say:

Make sure that when you save your file with created macros in it as a .xlsM (macro enabled) file type.  If you save it as the default .xlsX file type, any macros you did have in it will be deleted.

 

Try this test:

Open a new workbook and press [Alt]+[F11].  That will open the Visual Basic Editor (VBE).  From the VBE menu choose Insert --> Module

That should present a new clean code module to you.  Copy the code below and paste it into that code module:

 

Sub TestMacro()
  MsgBox "Good morning ClicketySplit, " _
   & "this message should NOT self-destruct."
End Sub

Put your cursor on somewhere between the Sub and End Sub lines and press [F5] - you should see the message displayed on your screen.  Click its [OK] to return to the VBE.

Now, close the VBE by pressing [Alt]+[Q]

Go to either the [Developer] or [View] tab and use the "Macros" drop-down to view the list of macros in the workbook.  Choose the TestMacro (highlight it) and click the [Run] button.  Once again you should see the message.

If all of that works, at least you know that macros are working in the workbook.

To RECORD A MACRO - go to the [View] tab, click on the "Macros" drop-down and choose "Record Macro"  That should bring up a dialog box where you can tell it to store the macro in the workbook and give it a unique name if you want to.  Take some actions such as choosing a different sheet and a new cell and maybe typing in some text or value on the sheet. 

You can stop recording the macro by clicking the "stop recording" button near the lower left part of the task bar or by choosing the "Macros" drop-down again and clicking "stop recording".

You can now test that macro from the list of macros again.  Save the workbook (remember to Save As .xlsm type).  Close the workbook and reopen it and see if the macro is still available for use.

---

If you cannot run the macro when you re-open the .xlsm file, then your Macro Security is probably set too high and preventing them from running.  The instructions for setting macro security on the page Mike H.. referenced are for Excel 2003 and earlier.

For Excel 2010,

Go to the [File] tab and select "Options"

Then choose "Trust Center" and click the "Trust Center Settings..." button.

The Trust Center dialog will open, choose Macro Settings from it.

In the top section, Macro Settings, I recommend choosing the 2nd option from the top: "Disable all macros with notification".

Click [OK] to close the Trust Center dialog, click [OK] again to close the Options dialog.

Close Excel and re-open it.  This must be done because the changes you made are not recognized by Excel until you close and open it again.

At this point, when you open a workbook with macros in it, you will see a dirty-yellow alert bar at the top of the worksheet display that says that contents is not enabled, with a button to be clicked to enable that content.  Click the Enable button to allow the macros to function if you trust the source of the workbook.

 

Other than the old instructions for setting macro security, the McRitchie tutorial is simply excellent.

lance cai msft say:

Hi Scott,

 

When you unsubscribe from a group, you are still a member but no longer receive messages and calendar event notifications in your personal inbox. You'll have to go to the group inbox to see them.

 

You'll still be a part of the group, but won't receive any conversations or invitations in your inbox, except in these two cases:

1.When someone replies to a message that was sent to the group by an unsubscribed user, the unsubscribed user receives those messages in his or her inbox.

2.When someone replies to a message that was replied to by the unsubscribed user, the unsubscribed user receives those messages in his or her inbox.

 

If you don’t want any emails from that group, we suggest you leave the group.

 

An article for your reference: Leave or unsubscribe from a group.

 

Thanks,

 

Lance
