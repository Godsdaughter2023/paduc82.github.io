Windows 10 UI/UX - a step in the wrong direction? - Microsoft Community

tamas deme say:

Windows 10 UI/UX - a step in the wrong direction?

Since the keynote I wanted to share my feelings about the new universal app design principles, both for phone and PC. Just now, I saw a blog post that basically said what I wanted to. Please read this first and than continue reading this letter: http://ignorethecode.net/blog/2015/01/31/windows_10_re_crappifying_windows_8/

These things mentioned in the article above are exactly my issues with the Windows 10 preview. I know it's a preview and not even a beta, but if this is the direction we are going forward I don't think it'll end up well.

Hamburger buttons:
This is something that I hate, and I don't know why is it now introduced to the official apps. Where are the original metro principles?
For example: the new universal maps app http://i.imgur.com/nhybNUy.jpg the new photos app http://i.imgur.com/rO9wwR3.jpg


Not using the app bar, but instead placing controls on the top of the screen:
Again something, that doesn't make sense. iOS does this, and now in iOS 8 Apple introduced a cumbersome way to move your whole screen down halfway, so you can reach these buttons easily with one hand. The appbar was a great solution but now there are examples everywhere that these commands are on the top of the screen. Check out outlook http://i.imgur.com/4Qw1uom.jpg

I know that I'm not alone with these opinions - just check out the WP uservice:

  • keep spartan address bar at the bottom https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/6993955-keep-the-spartan-address-bar-at-the-bottom
  • keep all the menu/nav buttons at the bottom https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/6997054-keep-all-the-menu-navigation-buttons-at-the-bottom
  • stick to modern, don't copy android https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/6623043-stick-to-modern-don-t-copy-android
  • buttons to down, no user "hamburger" https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/6993439-buttons-to-down-not-use-hamburger
  • move navigation bar back to the bottom https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/6996793-move-navigation-bar-back-to-the-bottom-in-windows
  • do not change photos hub style https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/5732950-do-not-change-photos-hub-style-in-windows-phone-8
  • windows10 should not copy android+ios https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/7005971-windows10-should-not-copy-android-ios-instead-ref
  • keep the ellipsis, remove the hamburger https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/6996104-keep-the-elipsis-remove-the-hamburger

And there are a lot more of these threads. Placing controls at the bottom and allowing users to swipe makes far more sense, as demonstrated by: https://www.youtube.com/watch?v=R00gUEdW8UY and https://www.youtube.com/watch?v=GNXS9Tc2SOQ .

Issues with the Feedback app

There are several things that I (and I presume others) don't like:

  • The feedback app is missing a category for general UI and UX things. I'm happy to report these issues there as well, but it makes zero sense to do it for every app, instead of having a general category for these.
  • It'd be good to see Microsoft moderate and respond to these: delete the ALLCAPS, shouting and nonsense posts BUT reply to or categorize the others. (Like on uservoice: planned / not planned / in backlog / under consideration / won't fix because XY)
  • It seems like the feedback app is country dependant OR some feedback goes missing. I live in Hungary, and when I open the app I basically can't see and of them. If I do a search for "appbar" it yields zero relevant results (see: http://imgur.com/a/LKrWb ), which I find hard to believe. In comparison the uservoice threads linked above have thousands of votes together.

I love seeing Microsoft doing these previews really early, and trust me I really do want for Windows 10 to succeed :) I hope there are others here who share my feelings about these issues.

doug robbins - mvp office apps & services (word) say:

But the requirement is to replace the "manual numbering" with multi-level list numbering where each numbering level is linked to a style.

Once the styles are setup with the required numbering, and given the variation in the ways that the manual numbering had been inserted, running a macro with the following code replaced the manually numbered of the paragraphs with numbering assigned by the relevant style

Dim arrFind As Variant
Dim arrStyle As Variant
Dim i As Long

arrFind = Split("PART [0-9]{1,}^t|PART [0-9 ]{1,}^t|[0-9]{1,}.[0-9 ]{1,}|[A-Z]{1,}.^t|[A-Z]{1,}. ^t|[A-Z]{1,}.[ ]{1,}|[0-9]{1,}.^t|[0-9]{1,}. ^t|[0-9]{1,}.[ ]{1,}", "|")
arrStyle = Split("Heading 1|Heading 1|Heading 2|Heading 3|Heading 3|Heading 3|Heading 4|Heading 4|Heading 4", "|")
For i = LBound(arrFind) To UBound(arrFind)
    Selection.HomeKey wdStory
    Selection.Find.ClearFormatting
    With Selection.Find
        Do While .Execute(FindText:=arrFind(i), MatchWildcards:=True, Forward:=True, Wrap:=wdFindStop) = True
            With Selection
                .Delete
                .Paragraphs(1).Style = arrStyle(i)
            End With
        Loop
    End With
Next i

This is indeed an Excel 2013 issue, it seems to be related to some ActiveX control temp files. Anyway, the fix for this and other ActiveX issues (like being unable to select from a combo box or a list box) is as follows:

  •          Quit Excel
  •          Start Windows Explorer
  •          Select the system drive (usually C:\)
  •          Use the Search box to search for *.exd
  •          Delete all found files

Narrowed down this bug a bit more:

  • If the input cells of the camera tool object (i.e. the cells being viewed/monitored) are on the same worksheet as the Data Validation enabled cells, the problem does NOT present.
  • If the input cells of the camera tool object (i.e. the cells being viewed/monitored) are on a different worksheet as the Data Validation enabled cells, the problem DOES present.
  • It does not seem to matter where the Camera Tool output object goes. That special graphic object could be on the same worksheet or a different worksheet as the Data Validated cells.
  • The bug is the same with different data validation list types. i.e. with a simple manual list (1,2,3,4,5) or a more complicated reference to a "Table[Column]" (=INDIRECT("Table1[Names]"). Either way, if the camera tool input cells are on the same worksheet as the data validated cells, there is not problem. But if the camera tool input is on a different worksheet, the drop-down-arrow goes into peek-a-boo mode.

Think about the implications: this means that using a tool on worksheet "A" will cause a bug to appear in worksheet "B". Which makes it pretty confusing.
