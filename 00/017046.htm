rich007a say:

Hi,

Something like this should work, it assumes you manually select one or more messages in Outlook before running the code:

 

 

Public Sub GetDetailsFromMsg()
    Dim Item As MailItem
    Dim Selection  As Selection
    Dim i As Long
    Dim strMsg As String, strDesc As String, strDate As String, strName As String, strColour As String
    Dim vMsg As Variant, vPara As Variant
   
    Set Selection = ActiveExplorer.Selection
    For i = 1 To Selection.Count 'loop through each msg in the current slection
        Set Item = Selection.Item(i)
        If Item.Class = olMail Then 'only work with mail items
            strMsg = Item.Body
            vMsg = Split(strMsg, vbCrLf, Compare:=vbTextCompare) 'split msg text into array of paragraphs
            For Each vPara In vMsg       'loop through each paragraph in the array
                If strDesc = vbNullString Then strDesc = TestPara("Description: ", vPara)
                If strDate = vbNullString Then strDate = TestPara("Date: ", vPara)
                If strName = vbNullString Then strName = TestPara("Name: ", vPara)
                If strColour = vbNullString Then strColour = TestPara("Colour: ", vPara)
            Next vPara
        End If
        
         '##########################################################
         'DO SOMETHING WITH THE DATA FROM THIS MSG ITEM HERE.... for example:
         'objDoc.CustomDocumentProperties("Desc") = strDesc
         '##########################################################
       

         'Set all data holders back to empty before next message:
        strDesc = vbNullString
        strDate = vbNullString
        strName = vbNullString
        strColour = vbNullString
    Next i      'loop to next msg in the selection
End Sub
 

Function TestPara(strTest As String, vPara As Variant) As String
    If Left(vPara, Len(strTest)) = strTest Then
        TestPara = Mid(vPara, Len(strTest) + 1)
    End If
End Function

 

 

Note how I added a few more categories as an example.  It's perhaps not 100% efficient as a piece of code, but if you've only got a few categories to extract it should be fine.  If you had many more categories, I'd perhaps think about storing them in an array...

 

Hope that helps.

Blank workbook when launching

When you launch earlier versions of Excel, it displays a blank workbook. Excel 2013 displays the start screen instead. You can disable this new default, as follows:

  1. Click the File tab and choose Options.
  2. Choose General in the left pane.
  3. In the Start Up Options section, uncheck the Show The Start Screen When This application Starts option.
  4. Click OK.

The next time you launch Excel 2013, it will bypass the start screen and open a blank workbook.


File Open

In older versions, you can display the File Open dialog by clicking the File tab and choosing Open or choosing Open from the QAT. Both routes in Excel 2013 display the Backstage View, which some users will appreciate. But users wanting to open files quickly will be annoyed. You can bypass the Backstage View as follows:

  1. Click the File tab and then choose Options.
  2. Choose Save in the left pane.
  3. Check the Don’t Show The Backstage When Opening Or Saving Files option.
  4. Click OK.
Remove SkyDrive Option from File Open and Save Pages

If you don't want to disable new file open and save pages but want to remove SkyDrive option from these pages, you can do this by disabling "Show additional places for saving, even if sign-in may be required" option (2013 File menu > Options command > Save option > Save Documents section). It'll remove SkyDrive option from file open and save pages.

You can also set your local computer as default file save location by enabling "Save to Computer by default" option.
