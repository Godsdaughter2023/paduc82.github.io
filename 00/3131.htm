Macro to compute Headings word counts - Microsoft Community

jph_b1 say:

Macro to compute Headings word counts

I have a document with type 1 Headings (Title1, Title2, etc.). Is there a way, presumably with a macro, to compute the number of words under each heading ?

So the result would look like this:

Title1 ...  590 words

Title2 ...  832 words

etc.

macropod say:

The simplest solution would be to change:

MsgBox

to:

ActiveDocument.Range.InsertAfter vbCr &

This will output the report at the end of the document. Do delete the report before re-running the macro, though; otherwise you're liable to have the previous report(s) included in the current report.

macropod say:

Try:

Sub CountHeadingSpanText()
Application.ScreenUpdating = False
Dim RngHd As Range, h As Long, strOut As String
h = CLng(InputBox("Input the Heading level (e.g. 1) for the heading spans to count", "Heading Span Word Counter", 1))
If (h < 1) Or (h > 9) Then Exit Sub
With ActiveDocument.Range
  With .Find
    .ClearFormatting
    .Replacement.ClearFormatting
    .Text = ""
    .Style = "Heading " & h
    .Replacement.Text = ""
    .Forward = True
    .Wrap = wdFindStop
    .Format = True
    .MatchCase = False
    .MatchWholeWord = False
    .MatchWildcards = False
    .MatchSoundsLike = False
    .MatchAllWordForms = False
    .Execute
  End With
  Do While .Find.Found
    Set RngHd = .Paragraphs(1).Range
    Set RngHd = RngHd.GoTo(What:=wdGoToBookmark, Name:="\HeadingLevel")
    With RngHd
      strOut = strOut & .ComputeStatistics(wdStatisticWords) - .Paragraphs.First.Range.ComputeStatistics(wdStatisticWords) & vbTab & .Paragraphs.First.Range.Text
    End With
    .Start = RngHd.End
    .Find.Execute
  Loop
End With
Set RngHd = Nothing
MsgBox "The following word counts are associated with each level " & h & " heading:" & vbCr & strOut
Application.ScreenUpdating = True
End Sub

The actual heading isn’t included in the word count. To include it, delete:
- .Paragraphs.First.Range.ComputeStatistics(wdStatisticWords)

Note: Enhanced code posted September 7, 2017.

jeeped say:
Word facilitates a mail merge by placing multiple items on a single sheet and using «next record» to pass along the next record set to your { MERGEFIELD } fields within that page, so no... you cannot halt the printing of background images and./or static text on the last page without removing it from all pages.

However, if you have access to Publisher, it produced mail merge results in quite the opposite way that Word does. In Publisher, you create a single label with image(s) and/or static text and tell it to Print Multiple Pages per Sheet. In this way, only the exact number of records will be printed and you will not have blanks printing on the last sheet of labels.

While there may be a short learning curve to achieve satisfactory results in Publisher, once you are familiar with the methods you can create better output and the one label printed multiple times is a real time-saver for creation and editing.
