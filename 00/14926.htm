davidf2 say:

What OS system are you running on the machine that has Pub 2016 installed?

I don't have Pub 2016 installed but I played with your scenario with Pub 2007 and 2010 on a WinXP machine and found that both versions adjust the length of the sentence and the internal character spacing in a similar way. I also found that if you changed the font from Arial to Calibri you get different results.

I can't give you every set of circumstances but in my test when I selected the first word (4 letter word) in a left justified paragraph, Calibri font, and toggle bold off and on then you get characters spacing shifting internally so that the added width of the bold first word does not change the length of the entire sentence. Change the font of the paragraph to Arial and you get a different result. The entire line of text shifts and is extended without any internal character spacing shift. This is consistent with both Pub 2007 and 2010.

If I select a single 4 character word somewhere in the middle of the sentence then again using Arial the text just shifts to accommodate the bold word size. Using Calibri the words following the bold word you again see an internal shift in the character spacing to maintain the sentence length.

If I run the same test but select the first 4 words totaling 16 characters and three spaces between the words, then both Calbri and Arial just shift the entire sentence to accommodate the added width of the bold characters.  The internal shifting and character spacing that I observed before using Calbri does not occur. Again this is consistent with both versions of Publisher.

My conclusions is that with some fonts such as Arial, Publisher will just shift the entire sentence. With some fonts such as Calibri, Publisher will try to adjust internal character spacing within the words in order to maintain the length of the sentence. At some point if the selected text exceeds a certain amount both Arial and Calibri will shift the entire sentence following the bold text without any internal character spacing. Again this was consistent with both versions of Publisher.

Again I don't have Pub 2016 installed but I would expect that Publisher is again trying to adjust for the added width of the bold word(s) and will treat different fonts in different ways. So I would expect that MS would say Pub 2016 is working 'by design'.

One additional way you can perhaps adjust for this is to use the Character Spacing tool under Home tab and within the font group as per the following:

Select your word(s) and change the option from Normal. You might be able to counter the shifting by changing the kerning of some words in the sentence.

You shouldn’t need to turn Windows Defender off – the OS should automatically turn Defender off when you install another AV app, and then automatically turn Defender back on when you uninstall the third-party AV app. This is how the Windows OS provides continuous real-time protection for itself and for you. greginmich say:

Windows Defender is no longer completely disabled when you install a third-party AV app. Both the Windows Defender UI and the Windows Defender Security Center now provide the option to run Windows Defender right alongside your third-party AV app – but only in a special Limited Periodic Scanning mode. This mode actually includes all of Windows Defender’s features and functions with the exception of real-time protection – which is the component that causes conflicts between AV apps.

But we do still have the perennial problem of third-party AV apps not always being properly registered and recognized by the OS, and this issue can allow Defender to keep right on running with its real-time protection enabled when you install another AV app. This can almost always be corrected by first uninstalling your third-party AV app and running its removal tool – and then also running the removal tools for any other AV apps that were previously installed, or that came preinstalled – and then reinstalling your third-party AV app.

https://answers.microsoft.com/en-us/protect/wiki/mse-protect_start/list-of-anti-malware-product-removal-tools/2bcb53f7-7ab4-4ef9-ab3a-6aebfa322f75

Once things are cleaned up, the AV registration process will generally succeed when you reinstall your third-party app. There are also some technical approaches to cleaning up the AV registrations – most notably, rebuilding the WMI Repository. We used to recommend this method routinely here in the forum – but I haven’t had a chance to review these methods lately, so any use will be at your own risk:

https://www.avast.com/en-us/faq.php?article=AVKB81

Also note that because rebuilding the WMI Repository can often effectively sort out the WMI AV provider registrations (in the root\SecurityCenter2 Namespace), this is quite often recommended in cases where Windows Defender refuses to turn on after a third-party AV app is uninstalled, e.g.:

http://www.thewindowsclub.com/windows-defender-is-turned-off
