Conditional Format - Highest/Lowest values ignore duplicates - Microsoft Community

lz. say:

Conditional Format - Highest/Lowest values ignore duplicates

Highlight Top/Bottom values but duplicates

Problem to solve

We have a range of values where there are duplicates and/or empty cells and/or non-numeric values. With Conditional Formatting we want to highlight the Top/Largest or Bottom/Smallest 3, 4, 5… values but the duplicates.

The proposed solution requires Excel Windows >/= 2007 or Excel Mac >/= 2011 as IFERROR is used. The same can be achieved with earlier versions of Excel with more complex formulas (not covered here)

NB: The proposed Conditional Formatting rules involve significant array calculations and are therefore not recommended on large ranges for performance reasons

Corresponding workbook with same explainations and formulas available here (includes examples with criteria)

Functions used - Other than LARGE & SMALL

ISNUMBER SIGN IFERROR
COUNTIF TRANSPOSE FREQUENCY

Horizontal range

(click on the picture to enlarge it)

How To reproduce

For Top/Largest 5
- 5 Conditional Formatting rules (Use a formula to determine which cells to format) are setup and apply to range =$H$9:$X$12
- For performance reasons tick the checkbox Stop If True for each rule

Formula for 1st rule highlithing the Top/Largest value

=AND(ISNUMBER(H9), COUNTIF($H9:H9,H9)=1, H9=LARGE(IFERROR(1/SIGN(TRANSPOSE(FREQUENCY(MATCH($H9:$X9&"",$H9:$X9&"",0),MATCH($H9:$X9&"",$H9:$X9&"",0)))*ISNUMBER($H9:$X9))*$H9:$X9,FALSE), 1))

Next rules to higlight the Top/Largest 2, 3… Use the same formula replacing the 1 at the end with 2, 3…

For Bottom/Smallest 5
- 5 Conditional Formatting rules (Use a formula to determine which cells to format) are setup and apply to range =$H$15:$X$18
- For performance reasons tick the checkbox Stop If True for each rule

Formula for 1st rule highlithing the Bottom/Smallest value

=AND(ISNUMBER(H15), COUNTIF($H15:H15,H15)=1, H15=SMALL(IFERROR(1/SIGN(TRANSPOSE(FREQUENCY(MATCH($H15:$X15&"",$H15:$X15&"",0),MATCH($H15:$X15&"",$H15:$X15&"",0)))*ISNUMBER($H15:$X15))*$H15:$X15,FALSE), 1))

Next rules to higlight the Bottom/Smallest 2, 3… Use the same formula replacing the 1 at the end with 2, 3…

Vertical range

(click on the picture to enlarge it)

How To reproduce

Same approach as with an horizontal range:

  • 5 Conditional Formatting rules (Use a formula to determine which cells to format)
  • Apply to range =$H$28:$L$48
  • For performance reasons tick the checkbox Stop If True for each rule

Formula for 1st rule highlithing the Top/Largest value

=AND(ISNUMBER(H28), COUNTIF(H$28:H28,H28)=1, H28=LARGE(IFERROR(1/SIGN(FREQUENCY(MATCH(H$28:H$48&"",H$28:H$48&"",0),MATCH(H$28:H$48&"",H$28:H$48&"",0))*ISNUMBER(H$28:H$48))*H$28:H$48,FALSE), 1))

Formula for 1st rule highlithing the Bottom/Smallest value

=AND(ISNUMBER(H28), COUNTIF(H$28:H28,H28)=1, H28=SMALL(IFERROR(1/SIGN(FREQUENCY(MATCH(H$28:H$48&"",H$28:H$48&"",0),MATCH(H$28:H$48&"",H$28:H$48&"",0))*ISNUMBER(H$28:H$48))*H$28:H$48,FALSE), 1))

Next rules to higlight the 2nd, 3rd… Use the same formulas replacing the 1 at the end with 2, 3…

Horizontal vs. Vertical ranges - Formula difference

There's an important difference in a formula that applies to an horizontal range vs. to a vertical range

The following applies to an horizontal range

range

=AND(ISNUMBER(H9), COUNTIF($H9:H9,H9)=1, H9=LARGE(IFERROR(1/SIGN(TRANSPOSE(FREQUENCY(MATCH($H9:$X9&"",$H9:$X9&"",0),MATCH($H9:$X9&"",$H9:$X9&"",0)))*ISNUMBER($H9:$X9))*$H9:$X9,FALSE), 1))

vs. the following that applies to a vertical range

=AND(ISNUMBER(H28), COUNTIF(H$28:H28,H28)=1, H28=LARGE(IFERROR(1/SIGN(FREQUENCY(MATCH(H$28:H$48&"",H$28:H$48&"",0),MATCH(H$28:H$48&"",H$28:H$48&"",0))*ISNUMBER(H$28:H$48))*H$28:H$48,FALSE), 1))

The FREQUENCY function returns a vertical array

When the range of values is horizontal that array must be TRANSPOSEd (to become an horizontal one) before being multiplied here by the array returned by ISNUMBER(range) and finally by the range of values

No need to TRANSPOSE with a vertical range of values

Howdy, 

I would also suggest another approach: 

During the 'rebuild' process, which may or may not have been needed for the original issue of 're-downloading' a message (as there are many possible reasons why this could happen), the Entourage database file get's duplicated, and the new one, get's the rebuild. The old one, which was perhaps still 'mostly' working from what you've said (other than the multiple re-downloading of the same e-mail) still exists, if it wasn't manually deleted. This should live in your User's account's home folder, within the Documents folder, within the Microsoft User Data folder, inside Office 2008 Identities folder. 

If you option-click upon Entourage to start it up, you get presented with the Entourage DataBase window, which should allow you to pick which database you wish to use, if I recall correctly, and there you should be able to select the older database that was at least still working.

Now, in regards to the 'stuck' repeating email, there are many possible issues that could cause this (i.e.: file attachment too large, corruption in the message, bad email headers, etc...), if indeed it's only one, or even a small # of messages, you can usually most easily resolve that issue by:

- First, see if you can open the message, and if it's anything important you want to save, do a 'save as' from file menu, and save it that way, or also you could try dragging & dropping to a 'Local' or 'On My Computer' folder, so that you save a copy.
- Then, simply removing that message(s) from the server should resolve the issue. You can usually do this most easily by using a web browser to access your webmail for that same account, if you can do so, then search for the subject of that offending email, find it, and delete it.

Of course, if you have backups on an external disk, you might want to also seek out the backed up database from prior to when you first noticed the issue. If all the emails are truly still on the server, then, you shouldn't loose anything at least from your inbox, although the 'sent' folder and potentially other folders might loose data since that older file possibly doesn't contain the newer sent items, etc...

Hope that can be helpful for you.

Online mail websites like gmail or Hotmail, although they use https (secure connections) work just as well if they are not placed in the Trusted Sites list...
