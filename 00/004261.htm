Would it be correct to assume that the "strange" date you are referring to has a year of "4501" either having the value of 1/1/4501 or something along the lines of 12/31/4501 h:mm:ss PM or other variation of the date format but with similar values?

In any case, some summary background about Outlook date fields:

#1 - There is no such thing as a truly "empty" date field in Outlook. An "empty" Outlook date field actually has a value of <1/1/4501>. When exporting any date field in Outlook - that value has to be checked so that this internal "empty" date value is not exported (or more specifically, the output field is correctly initialized to an empty value that corresponds to the output file/table/field type (i.e. text, Excel, Access etc)

#2 - There are 3 significant Outlook date values (for the sake of this reply - all date formats shown use mm/dd/yyyy format which should be adjusted accordingly for the regional date settings in place on a given system)

   a) Empty - 1/1/4501
   b) Lo/Min date - 4/1/1601
   c) Hi/Max date - 8/31/4500

#3 - Data values are stored internally as a "long" numeric value - ergo 1/1/4501 has an internal numeric value of 949998. How any given date is displayed in a user-interface is controlled by it's associated format setting (be that in Outlook or in any database) or however any program decides to show it as in the case of exporting a date field to a text file.

Sorry but this is one of the joys of Windows 8 and thus far there has been no fix forthcoming from MS. There has not been a good fix but some users have found that installing Windows 7 fonts on the Win8 machine solves the problem. Another user found that if they changed Windows 8 and ran it in windows compatibility mode for windows 7 that would solve the problem.

One other potential quick solution is to have the user export their Pub file as a PDF file and send that to you. The pdf should render the same on your Win8 machine. The built-in Office PDF tool can convert to a commercial grade quality pdf which might be adequate for your use.

hansv mvp say:

Here is a custom function you can use:

Function MyReplace(s As Variant) As Variant
    Dim i As Long
    Dim f As Boolean
    Dim r As String
    If IsNull(s) Then
        MyReplace = Null
    Else
        f = True
        For i = 1 To Len(s)
            If Mid(s, i, 1) = "{" Then
                f = True
            End If
            If f Then
                r = r & Mid(s, i, 1)
            End If
            If Mid(s, i, 1) = "}" Then
                f = False
            End If
        Next i
        MyReplace = r
    End If
End Function

On the Create tab of the ribbon, click Query Design.

Add the table, then close the Show Table dialog.

Add the relevant field of the table to the query grid.

In the Query Type group of the Design tab of the ribbon, click to highlight Update.

Enter the following in the Update To row:

MyReplace([FieldName])

substituting the actual name of the field.

Click the Run button (the red exclamation mark) and confirm that you want to update … rows.
