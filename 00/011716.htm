Need to Know How to Stop Word Macro at EOF - Microsoft Community

suemckinn say:

Need to Know How to Stop Word Macro at EOF

I have a macro that searches for something, backs up, inserts a word, then moves forward. This needs to be repeated throughout the document but stop when the document reaches the end of the file. The search information is not being deleted, so without a way to stop the macro, an endless loop will occur. I think something is needed in front of "With Selection.Find" and after "Wend" so the whole thing is captured, but I'm not sure how the EOF function is used.

Here's the macro:

Sub InsertBasic()
'
' InsertBasic Macro
' Insert Basic before each basic flow table
'
  Selection.Find.ClearFormatting
  With Selection.Find
        .Text = "basic flow"
        .Replacement.Text = ""
        .Forward = True
        .Wrap = wdFindContinue
        .Format = False
        .MatchCase = False
        .MatchWholeWord = False
        .MatchWildcards = False
        .MatchSoundsLike = False
        .MatchAllWordForms = False
    End With
    While Selection.Find.Found
        Selection.Find.Execute
        Selection.HomeKey Unit:=wdLine
        Selection.MoveLeft Unit:=wdCharacter, Count:=1
        Selection.TypeParagraph
        Selection.Font.Bold = wdToggle
        Selection.TypeText Text:="Basic"
        Selection.MoveDown Unit:=wdLine, Count:=3
    Wend

End Sub

jay freedman say:

Just change 

        .Wrap = wdFindContinue

to

        .Wrap = wdFindStop

jay freedman say:

That's exactly what the Wrap parameter is for. It determines whether Selection.Find.Execute stops when it reaches the end of the document, or "wraps around" and continues from the beginning of the document. It does not determine whether the While loop stops after finding one instance.

The way the macro is now, the loop repeats until it gets to the last instance of "basic flow" in the document, and on the next repetition the Selection.Find.Execute doesn't find another instance between the current location and the end of the document. Because of the .Wrap = wdFindContinue, though, the Execute starts looking at the beginning of the document, and it finds the first instance... and it'll keep doing that until you stop it with Ctrl+Break.

When you change to wdFindStop, the Execute that occurs after finding the last instance of "basic flow" in the document will set Selection.Find.Found = False (because there are no more instances between the current location and the end of the document) and the loop will stop.

The equivalent behavior happens when you're using the Find dialog and you keep clicking the Find Next button. After it finds the last instance in the document, if you click Find Next again you'll get a message asking whether you want to continue searching at the beginning of the document. (Actually, you can get this exact behavior if you set .Wrap = wdFindAsk.)

I asked that question based upon whatyou had indicated about connecting only as a guest.  There was a Microsoft

knowledgebase article that said the following:

 

SYMPTOMS

After you create a partnership and connect normally one or more times, your Pocket PC may suddenly begin connecting only as "Guest."

Instead of listing the Pocket PC device name (for example, "Pocket_PC"), the ActiveSync Status window lists "Guest." The device status displays "Connected," but none of the expected information types are listed. You can perform typical guest functions such as copying files, but you cannot synchronize the device.

 

CAUSE

This behavior can occur if you change the original device ID for the Pocket PC and then change it back to the original name.

The partnership information in the Pocket PC may not be properly updated if you do not "soft" reset the Pocket PC after you change the device ID. In this case, when you connect the device after changing the device ID, the device is unable to find the Repl.dat file for the correct partnership and defaults to a guest connection.

 

RESOLUTION

Perform a "soft" reset of the Pocket PC according the the manufacturer's instructions.

In the future, make sure to "soft" reset the Pocket PC if you change the device ID.
