On Wed, 2 Feb 2011 08:21:45 +0000, _Erik wrote:

I called my ISP again and they are now helping to?figure?out the problem in more detail, We have checked the handshake and serverlogs, it now seems that the problem is that the password is send encrypted in?NTLM or?NTLMv2 instead of the required MD5.

If the password being sent to your ISP is indeed using NTLM or NTLM2 and
your account is either a POP or IMAP account then it sounds like you've
configured the wrong type of account on the phone to begin with. The only

You should be using other account to setup your email, not Outlook or
advanced setup.

As an aside, as someone who works with cryptography and PKI for a living,
if your ISP is still using MD5 for security you should definitely be
looking for another ISP.

While you can relatively easily change the default system font size displayed in screens (using Start -> Settings -> System -> Screen -> "Text Size" tab), the pop up menu items are a different story. To the best of my knowledge, their is no utility that allows for an easy way to change these.

I do know that the settings for the menu are controlled in the registry of the device. To change these, you would need a registry editor installed on the device. Registry changes are (just like on a Windows PC) a rather risky proposition to change if you are not comfortable with this sort of thing. If you are, however, take a look at this forum entry at Modaco.com for more information.

Restoring a Windows 7 Backup and Restore Catalogue in Windows - Microsoft Community

shane steinmetz say:

Restoring a Windows 7 Backup and Restore Catalogue in Windows 10/TP

Now using:

- Windows 10 Technical Preview 32-bit (clean install)

Was using:

- Windows 7 Home Premium 64-bit

External 1 TB HDD

DESIRED RESULT

I would like to restore files created using the Backup and Restore feature from Windows 7 on to Windows 10/TP.

PROBLEM

The Windows 7 Backup and Restore functionality appears to have been replaced with a new function called "File History" in Windows 10/TP.  When attempting to choose the option to Restore Personal Files from this feature, I see that it is not able to find anything to restore.  I verified that the External 1 TB HDD containing the backed up data was connected and otherwise accessible from Explorer.

The Windows 7 Backup and Restore feature had created a folder at the root of my External 1 TB HDD with the same name as the computer's name in Windows 7.  The folder's icon is also set to the Windows Backup and Restore icon.  It appears to be full of folders named "Backup Set" with dates corresponding to the dates backups were created.  The folder itself - along with subfolders - are access restricted.

Windows 10/TP does not recognize these backup sets even when ownership of the top-level folder and all subfolders are changed to my current user account on Windows 10/TP.  I also attempted to adjust access permissions for 'Everyone' to 'full access' to eliminate access issues.

The backup sets seem to be a series of ZIP files.

INFERENCES

I am not certain whether this is a widely known issue or not; my attempt at searching for information on this forum has given me limited results.  I also attempted researching this issue with search terms that replaced "Windows 10/TP" with "Windows 8," since Windows 10/TP seems very similar to Windows 8.  I found complaints from other users who believed that Microsoft deprecated support for backup volumes created with Backup and Restore starting with Windows 8.  I was unable to find a clear resolution.

If I could somehow download the Backup and Restore functionality into Windows 10/TP, this problem would probably be solved.  I'm not even sure if this is possible, though.

DESIRED ASSISTANCE

Fortunately, I created a system image of my Windows 7 PC.  I can attempt to mount the VHD file and recover my files manually.  In fact, short of going back to Windows 7 and restoring the backup so that they can be backed up using a different method (or third party software), mounting the VHD file seems to be the primary solution recommended for this issue.  It is unsettling to know that I would have had to go back to Windows 7 and start over if not for the fact that I happened to create a system image during the last backup.

The reason I would prefer to use the "file for file" method offered by Backup and Restore is because of the interface and the ability to preserve file paths for certain restorations. 

For purposes of resolving this issue, assume that using a virtual machine and mounting a system image are not solutions.

Is there any way to get the Backup and Restore functionality on Windows 10/TP that was previously on Windows 7?
