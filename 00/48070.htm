jamoross say:

I think you didn't understand the problem.  Some bluetooth keyboard/mouse sets include 

a bluetooth adapter that is aware of the bootstrap problem and emulate a wired USB device

until the OS recognizes (and presumably initializes) it as a bluetooth adapter.  Consequently,

during a clean install of WinXP, the keyboard will work in the pre-boot environment. 

(Possibly with help from the BIOS itself which may be doing "legacy support" to make the

 "USB keyboard" look compatible all the way back to DOS!)

 

Anyway, once the install gets far enough along for some more advanced drivers to be loaded,

the BT adapter quits its compatibility mode and the keyboard stops working.  Alas, this is well

before the point where you can create BT connections.  It's at about the point where Windows

Setup asks you for locale settings, timezone, a *license key* etc. 

 

I haven't found any way around using another keyboard at that stage.  Another mouse is also

handy but not strictly required.  (You can do everything using <tab> <space> and <enter>.)

 

Hope this clears things up a bit.
