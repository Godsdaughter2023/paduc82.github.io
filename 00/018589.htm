If Report has No data, print Report with information.......... - Microsoft Community

ronnie ferns say:

If Report has No data, print Report with information..........

Hi

If a customer has no data,need to print the report with information " The above customer has no trades outstanding as on above Date"

How can we do that

 

Thanks

Rony

 

ken sheridan say: There is an inbuilt paradox here.  The customer ID number and name are data, so if the report shows them in bound controls the report has data.  There may be no data on the customer's trades, but there is data in the report in the form of the customer ID number and name.  However, there are two possible solutions:

1.  Use a query with a LEFT OUTER JOIN

1.1. For this forget about the No Data event procedure as that's not an appropriate solution to what you want.

1.2. To return the customer ID number and name base the report on a query which joins the customers table to the table containing the trades data, using a LEFT JOIN.  This will return the customer data regardless of any matches in the trades table.

1.3.  To show a 'no trades to report' label in the report header. To show the label you need to firstly identify a control in the report bound to a column from the trades table which will always have a value if there is any trades data, so let's assume for example that you have a control named TradeDate.  In the report header's Format event procedure you'd put:

IsNull(Me.Tradedate) Then
    Me.labelnodata.Visible = True
Else
    Me.labelnodata.Visible = False
End If

The label will show if TradeDate is Null, i.e. if there is no trade data.  If you also want to pop up a message box put the line:

MsgBox "No Trades Outstanding for this Customer", vbInformation, "No Records O/S"

immediately before the Else line.

1.4  But, there is a potential fly in the ointment with this solution.  If the query is restricting the report by criteria not only for the customer, but also on any columns from the trades table, e.g. if there is restriction to TradeDate values within a certain date range, then you cannot use this solution as, apart from testing for Null, a LEFT JOIN query can only be restricted on columns in the table on the left side of the join, i.e. customers, not on a column in the table on the right side, i.e. trades.  So, proceed to Plan B:

2.  Using unbound customer controls.

2.1  For Plan B, you'd use a query with an INNER JOIN, but instead of using bound controls in the report for the customer ID number and name (and any other data from the customers table) use unbound text box controls.  These would reference whatever parameter you are using to restrict the report to a specific customer.   Let's assume that you are selecting the customer from a dialogue form frmReportDlg, which includes a combo box cboCustomer, whose bound column is a hidden CustomerID and whose visible column is a text CustomerName number.  The ControlSource for a control in the report to show the CustomerIID would then be:

=Forms!frmReportDlg!cboCustomer

and that for a text box to show the customer name would be:

=Forms!frmReportDlg!cboCustomer.Column(1)

The Column property is zero-based so Column(1) is the second column, CustomerName.

2.2.  The cboCustomer combo box in the dialog form would be set up along these lines:

RowSource:     SELECT CustomerID, CustomerName FROM Customers ORDER BY CustomerName;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column and that the second is at least as wide as the combo box.

2.3  The dialogue form can include other controls referenced by the report's query, e.g. to restrict the results to a date range, and would also include a button to open the report.

2.4  To show a 'no trades to report' label in the report's header, and pop up a message box if desired, do exactly as in 1.3 above.
