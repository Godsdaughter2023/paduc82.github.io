andreas killer say:

Use a subroutine like this:

Sub FieldToSAP(ByVal From As Range, ByVal FieldName As String)
  If IsEmpty(From) Then Exit Sub
  SAP.Field(FieldName) = From.Value
End Sub

And if you have many non-contiguous cells something like this:

Sub Main()
  Dim C As New Collection
  Dim Item
 
  C.Add Array(Range("F5"), "Surname")
  C.Add Array(Range("H5"), "Forename")
  C.Add Array(Range("F6"), "Street")
 
  For Each Item In C
    FieldToSAP Item(0), Item(1)
  Next
End Sub

andreas killer say:

Thanks so much for your help... I resolved it

I'm pleased to hear that it works.

A few notes to optimize your code:

a) Please never use SELECT, SELECTION, ACTIVECELL, it is slow and error prone. Always refer to the objects directly.

b) You don't need to cache the items in a collection if you loop though that list.
c) If you span a range from A2 to the last cell, search from the bottom upwards.
  Otherwise you miss some data if there is a gap.
d) If you refer to a sheet, use the Name, not the Codename.

  Because if the user copy a sheet, rename the old to "backup" and the new to the old name, your code use the Backup sheet.

Well three reasons: First Both Recent Upgrades to MacOS and Office 2016 are tuned to use Solid State Drives as opposed to Hard drives. When using Hard Drives System and Software made heavy use of cache files Making systems and application load fast. The new systems don't depend on Cache files because SSD don't need them. 

Second, Apple has created a new security system installed in recent Operating system that places individual parts of the application in containers this is called sandboxing. This system slows down matters.

Third, as a result of Sandboxing has deemed it necessary to duplicate the Microsoft Fonts for each application. (Word, Excel, Outlook, PowerPoint, and One note  If you have Office 2011 There is an addition copy of the fonts. So you have 6 copies of  Microsoft fonts floating around. 

Office 2011 makes use of a font cache that each application, is allowed to read,  thus saving time loading on a Standard Hard Drive. Microsoft still could have loaded one copy in the group containers folder and allowed each application to read from and create a cache as needed. However, Microsoft didn't do that.It will be years before every Mac user switches to an SSD-equipped Mac. The cost of SSD is extreme (now) and the MTBF of SSD is still questionable.  Because of Mac's reliability, because they pay much attention to quality. Mac can go 6 or more years without an update. So fo many Mac user this slowness will just have to be put up with.
