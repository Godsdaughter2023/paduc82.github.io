amit tandon say:

If you want it do be done with excel formulas, one option is to enter each of the below formulas in separate cells, which will return the AVERAGE "Days" "Hours" & "Minutes" respectively:

All formulas are array - (CTRL+SHIFT+ENTER):

=ROUNDDOWN(AVERAGE(IF(A2:A100<>"",TRIM(LEFT(SUBSTITUTE(A2:A100," ",REPT(" ",255)),255))*24*60+TRIM(MID(A2:A100,FIND("Days",A2:A100)+LEN("Days"),FIND("Hour",A2:A100)-(FIND("Days",A2:A100)+LEN("Days"))))*60+TRIM(RIGHT(SUBSTITUTE(TRIM(SUBSTITUTE(A2:A100,"Minutes",""))," ",REPT(" ",255)),255))))/(24*60),0) & " Days"

=ROUNDDOWN(MOD(AVERAGE(IF(A2:A100<>"",TRIM(LEFT(SUBSTITUTE(A2:A100," ",REPT(" ",255)),255))*24*60+TRIM(MID(A2:A100,FIND("Days",A2:A100)+LEN("Days"),FIND("Hour",A2:A100)-(FIND("Days",A2:A100)+LEN("Days"))))*60+TRIM(RIGHT(SUBSTITUTE(TRIM(SUBSTITUTE(A2:A100,"Minutes",""))," ",REPT(" ",255)),255)))),1440)/60,0)&" Hour"

=ROUND(MOD(MOD(AVERAGE(IF(A2:A100<>"",TRIM(LEFT(SUBSTITUTE(A2:A100," ",REPT(" ",255)),255))*24*60+TRIM(MID(A2:A100,FIND("Days",A2:A100)+LEN("Days"),FIND("Hour",A2:A100)-(FIND("Days",A2:A100)+LEN("Days"))))*60+TRIM(RIGHT(SUBSTITUTE(TRIM(SUBSTITUTE(A2:A100,"Minutes",""))," ",REPT(" ",255)),255)))),1440),60),0)&" Minutes"

Obviously you may combine all the above in a single cell is you are fine with such a large formula.

In case you want the average in minutes only, just use below array formula:

=AVERAGE(IF(A2:A100<>"",TRIM(LEFT(SUBSTITUTE(A2:A100," ",REPT(" ",255)),255))*24*60+TRIM(MID(A2:A100,FIND("Days",A2:A100)+LEN("Days"),FIND("Hour",A2:A100)-(FIND("Days",A2:A100)+LEN("Days"))))*60+TRIM(RIGHT(SUBSTITUTE(TRIM(SUBSTITUTE(A2:A100,"Minutes",""))," ",REPT(" ",255)),255))))

rob koch say:

Since you didn't provide the reference whaere you read this it's difficult to be certain to what Microsoft may have been referring.  However, there are a couple possible reasons covered in both the following forum discussion as well as due to the potential use of the newer GPT (GUID Partition table) which was added as part of the UEFI initiative.

http://www.vistax64.com/vista-general/134441-vista-mbr-vs-xp-mbr.html

http://msdn.microsoft.com/en-us/windows/hardware/gg463525.aspx

Since the GPT spec includes a protective MBR as described in the following paragraph from the second link, this may be the reason behind the comment you found.

"The Protective MBR protects GPT disks from previously released MBR disk tools such as Microsoft MS-DOS FDISK or Microsoft Windows NT Disk Administrator. These tools are not aware of GPT and do not know how to properly access a GPT disk. Legacy software that does not know about GPT interprets only the Protected MBR when it accesses a GPT disk. These tools will view a GPT disk as having a single encompassing (possibly unrecognized) partition by interpreting the Protected MBR, rather than mistaking the disk for one that is unpartitioned."

In any case, you'd need to treat the case of Windows XP 32-bit completely separately from that of Windows XP 64-bit as well as Vista/7 and newer for all of the reasons mentioned.

Since Windows XP will end support from Microsoft as of April 8th, 2014 it seems pointless to do the work required for this archaic operating system at this point.
