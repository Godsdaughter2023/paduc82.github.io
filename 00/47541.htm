Maybe I am wrong that %AppData% is not derived from HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\Shell Folders/AppData.

I changed %APPDATA% in a command prompt. Perhaps this has no influence on the environment variables of the user session because the command prompt is like an own process?

Correct. Each process inherits its environmental variables from the process that invokes it. Inheritance is a one-way street: It always goes from parents to children, never from children to parents. You can easily test this by creating your own variable while in a Command Prompt. It gets lost the moment you close the Command Prompt.
