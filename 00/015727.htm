It will, if it the correct form of pdf.

A picture of a document also cannot be edited unless it is opened by another piece of software called optical character recognition (OCR). If the Malicious Software Removal Tool's mrt.log file shows clean (no detections found) then it's most likely that MSRT suspected a possible threat during the scan, displayed it as such (infected files) but determined the file(s) were not an actual confirmed threat(s) upon completion. rob koch say:

When I saw this post earlier this morning I went looking for a thread I found once in the past which stated something similar to your response, but I've been unable to find it since.

The only difference in that explanation was that heuristics weren't specifically mentioned.  Instead, the Microsoft employee indicated that the MSRT scanning process collects various unconfirmed detections and then once completed the check for false positives occurs, which is why some of the unconfirmed detections may be removed once the signature database is referenced.

There's little difference in either of these descriptions, though the Microsoft employee's description more closely matches the specific design of the other Microsoft security products which we've known about at least since the earliest versions of Microsoft Security Essentials were released in 2009.  The Dynamic Signature Service was often discussed back then as part of the SpyNet (now MAPS) system, with videos and other Microsoft provided information available at the time explaining how these functioned in highly technical detail.

The following excerpt and  web page describe the DSS and SpyNet, but are obviously dated with the video referenced no longer available as well, though the core functionality within the security apps hasn't changed.

"Additionally, when your FEP or MSE client reports new malware to the Microsoft SpyNet cloud service, the Dynamic Signature Service can recognize when a definition is available but not yet released, and deliver that definition for that specific threat in real-time from the cloud. Upon delivery of the dynamic signature, the threat will be detected and can be removed from the system."

https://blogs.technet.microsoft.com/clientsecurity/2011/02/22/microsoft-spynet/

 The renaming of SpyNet to MAPS occurred to deal with the tin foil hat, knee jerk response of the uninformed that associated this remnant of the original Giant AntiSpyware product with spyware and was part of the acquisition of GIANT Company Software Inc., which was Microsoft's entry point into that portion of the security industry.

As for the question of why this information isn't displayed within the MSRT client during scanning, the simple truth is it's never been in any of them, with the only such documentation found on the TechNet IT professional or MSDN developer resource sites.  This decision can be questioned, but the trend within the consumer clients over time has been the removal of any technical information or controls, since these have typically been found only to cause more confusion for the consumer user.

The truth is that only the obsessive compulsive among us typically ever run something like the MSRT manually, while most consumers leave this to the automation systems Microsoft has created for this purpose.  So the numbers of people coming here to question these visual anomalies are so tiny that the attempt to explain them within the interface is more likely to create added confusion than it is to remove it.  Most will simply accept the final results of "no infections found" at face value and in fact will typically never even see this in the first place.

To verify this, press Shift+F1 to open the Reveal Formatting pane and check the box at the bottom of the pane for "Distinguish style source". Then select a gray hyperlink. If you see "Direct formatting"

then use the Styles pane (Ctrl+Alt+Shift+S) or the Apply Styles dialog (Ctrl+Shift+S) to reapply the Hyperlink style to the selection.

If instead you see something like this,

but the color is still gray, I can't think of an explanation that allows other hyperlinks in the same document to be blue. The "Hyperlink" color mentioned in that screen shot comes from the document's current theme, and applies to the entire document.
