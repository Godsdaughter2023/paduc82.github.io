I suggest you to Manually reset the Windows Updates Components and check if it helps.

Resetting Windows Update Components will fix corrupt Windows Update Components and help you to install the Windows Updates. Please follow the below steps to reset the Windows Updates Components:

1. Press Windows Key + X on the keyboard and then select "Command Prompt (Admin)" from the menu.

2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the "ENTER" key after you type each command.

net stop wuauserv

net stop cryptSvc

net stop bits

net stop msiserver

3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the "ENTER" key after you type each command.

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

5. Type Exit in the Command Prompt to close it.

Now you may try running the Windows Updates and check if the above steps resolve the issue.

This organization has always had an Active Directory domain on-premises.  However, no password synchronization between any of these e-mail services.

Now, I've set up Azure AD Connect and have successfully synchronized some test users with a soft match on their UPN.  Everything seems fine.  Passwords are synchronized, licenses that had previously been applied to the cloud accounts are still applied to the synced accounts.

Here are my questions:

2.  Is it necessary to have Exchange on-premises?  What benefits do I gain?

3.  What would be the 'supported' configuration; according to Microsoft?

brook zhang msft say:

Hi Robert,

Office 365 contains Exchange Online. Usually, hosted exchange service and on-premise or on-site exchange server are the same concept.

In your scenario, I understand that you used the exchange server hosted in BitTian and use the MigrationWiz to migrate users to Office 365. I assume that you’re not using the hosted exchange service again. The Active Directory is on a new Windows server, which has no relationship with the former hosted exchange service.

If you install an exchange server, users in Active Drectory will have exchange attributes. But this is not enough. You still need to deploy Exchange hybrid so that you can manage them in local Exchange admin center.  

Without hybrid deployment, you can use ADSI Editor to manage local users. However, it is not officially supported in this sub-forum.

Yes, you're right. An Exchange server is needed to be installed on-premise to manage these users. Hybrid deployment is not required.

phillip m jones say: When Cutting and Pasting. Instead of ⌘-C then ⌘-V

Do this:
  1. use whatever method you do to select and copy the quote or whatever you want copied.
  2. Next instead of Straight Paste ⌘-V, instead go to edit menu and choose Paste Special.
  3. Choose Paste Special.
  4. Next choose Unformatted Text.
The allows the pasted material to take on the formatting of the Document.

As for the two spaces after a period (.) this is an old style that was used in the days of Typewriters. [Boy am I dating myself, there some here that probably would even know what a typewriter looked liked much less used one.] And some people still use a double space.

Anyway you just have to remove the double spaces manually.
