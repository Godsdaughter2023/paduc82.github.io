ramesh srinivasan say:

First one is probably due to LoadAppInit_DLLs set to 1. Malware usually enable this value and add their own DLL to the App_Init key, so that every process you launch in your system reference this DLL. Autoruns utility can be used to disable the AppInit DLL in the registry.

Download Autoruns from Microsoft, and run it as administrator. Save the log to a .ARN, zip it and upload it to your cloud account and share the link.

The 2nd error is due to missing file which the task was configured to run. In this case the task name is "ICRNZUSHUQ" (sounds like a malware), which you can delete it using Task Scheduler. How to Fix Event ID 414 Task Misconfiguration Errors in System Event Log?

Meanwhile, run a full system scan using Malwarebytes so that it eliminates most of the unwanted stuff in your system, which inturn may fix the above issues automatically.

davem121 say:
Hi Lax,

Click your Start Button, type regedit and hit Enter to open the Registry Editor

In the left pane, navigate to:
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\NetworkList\Profiles

Expand the Profiles Key (folder)

That will contain sub-keys with different GUID

Click each Sub-key

On the right, check the ProfileName value

When you find the sub-key with the correct ProfileName value

At the top you will see 'Category' - double click that

Change its value from 0 to 1

(0 = Public, 1 = Private)

Click OK

Close the Registry Editor and restart your PC

Hi Sgt. Wattson,

Changing passwords have specific requirements or parameters. This highly applies especially if you are under a corporate or business domain. This warning message "Unable to update the password. The value provided for the new password doesn't meet the length, complexity, or history requirement of the domain" means the following:

  • Your new password was already used by your account before. Meaning that you cannot use this password again.
  • Your new password has the same words and almost the same as your old passwords.
  • The password does not have the combination of special characters, lower case, or upper case.
  • The length or number of characters of your new password doesn’t meet with your domain requirements.
  • Your database of user accounts is not syncing with your computer.

had the same problem for a long while, but tonight I seem to have solved it. This is how I did it. Hope it works for you and many others. My network storage is Buffalo Linkstation LS-CHL (I will call it LS-CHL below).

Sign-in your network storage device manually as before.

Go to Credential Manager under Control Panel-User Accounts. Look for your network storage device under windows credential and open it. Then select Edit. Replece the domain name in front of your user name (LS-CHL, in my case). Save.

Log off from you Windows account and then log in again. Windows will again say it cannot connect network drive. Open Credential Manager again and you will notice that your device is not there under windows credential.


Now connect your device again manually. Then in Credential Manager click "Add a Windows Credential": enter device name (LS-CHL, in my case) as internet/network address, user name (without domain name) and password. Save.

That is it. Log off window and then log back on, your device should be connected automatically.

The sequence is important.
