doug robbins - mvp office apps & services (word) say:

The following assumes that the table has a header row,  and the you want the additional data to be inserted in the columns B, C, D, E and F of the row that contains an entry in Column A that matches the .Value selected in the ComboBox.

It also assumes that CBRegulatoryRMFDA, etc are the names of controls on your form.

If the data is to go in to columns other than B, C, D, E and F adjust the second index in the Offset  accordingly.

Private Sub CommandButton4_Click()
Dim i As Long, j As Long
With ActiveWOrksheet.Range("A1")
    For i = 1 To .currentregiona.Rows.Count - 1
        If .Offset(i, 0) = CBRegulatoryRM.Value Then
            .Offset(i, 1) = CBRegulatoryRMFDA
            .Offset(i, 2) = CBREgulatoryTSCA
            .Offset(i, 3) = CBRegulatoryBfR36
            .Offset(i, 4) = CBRegulatoryGB
            .Offset(i, 5) = CBRegulatoryFoodContact
            Exit For
        End If
    Next i
End With
End Sub

Longest path in MP 2010

Since critical path in based on default created under "Total slack is equal to or less than X", this does not reflect critical path. We need to show the longest path. tom_bpc say:

This is an old thread, still marked as unanswered.  I think it remains unanswered for good reason - MSP 2010 neither recognizes nor provides any treatment for a project Longest Path that is different from a "Critical Path" defined by Total Slack alone.

With great deference to John and Rod who have contributed quite a bit to this forum and to the overall community, I'm not sure you fully grasped what James was getting at here.   That is, that under certain conditions, the null value for Total Float (i.e. as classically defined: Late Finish minus Early Finish) does not adequately describe the driving path for project completion.  The most common instance of this seems to be when different task calendars are sprinkled through the project.  For example, if a task with a seven-day calendar finishes on a Friday and its Finish-to-Start successor (with a 5-day calendar) starts the following Monday, then the first task has 2 days of free float - two days of total float if the successor is "Critical".  So even though this task is the driving predecessor for a known critical path task, it is (and all its predecessors are) omitted from the TF=0 "Critical Path" filter.  Some tools (you mention P6, but I would start with P3) recognized this limitation many years ago and offered an alternate, "Longest Path" definition of the Critical Path.  "Longest Path" is supposed to catch all those tasks that drive the completion of the project, regardless of their float values.  While I do recognize many subtle differences between Total Float and Total Slack, I think they behave similarly here.

My firm recently ported some of our VBA code into an Add-In for Microsoft Project called BPC Logic Filter.  Among other things, it traces driving-path logic for the project's finishing task (i.e. the Longest Path) and for any other tasks in the project - all while ignoring total slack.  It's not easy to do, but it is possible.  You could start (like I did) by buying Rod's book on VBA Programming for MSP.

nitsuj5 say:

I figured out the answer to my own question:

Quote:

"If Excel is in automatic mode when the workbook is opened the calculation will take place before any event fires, so you cannot prevent it.
You can switch to manual calc in the workbook_open event

application.calculation=xlmanual

If the problem is caused by opening a workbook in a later version of excel than the version that last calculated it, you can apply a registry fix. "
