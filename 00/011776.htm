In PowerPoint 2007, video cannot be embedded. It can only be linked. So, you'll have to reconstruct the links. Please the video files in the same folder as the presentation, create the links and send the presentation as a folder.

When you're creating presentations for distribution, the best practice is to create it in the lowest version on which it will be displayed. Then it almost always works fine in later versions.

Rather than seek out codecs, convert your video to a format that the lowest version can use natively. I generally convert video to MPEG format. Even though it has a larger file size than some other formats, it plays well on Mac or Windows and old or new versions and doesn't require proprietary software (like Flash or QuickTime).

steve rindsberg say:

I don't have the time to test this at the moment, but my guess is that it's because PPT 2010 and later embed MP3 audio files when you record narration, but earlier versions understand embedded WAVs, not other audio file types.

Try this:

Make a copy of your presentation file, change the extension from PPTX to ZIP and open the resulting ZIP file.  In it, you'll find a media folder and in that, you'll likely find the embedded sound files.

Drag copies of them out of the zip and into the same folder as the PPTX renamed to ZIP.

Then rename the file from ZIP back to PPTX, open it and delete all the recorded sound files.

Finally, insert the sounds you copied out of the media folder in place of the previous sounds and be sure to link them.  Note that the audio files MUST be in the same folder as the PPTX at this point; moving them into the same folder later won't work.

Now, as long as the audio and PPTX files stay together in the same folder, they should work in previous versions of PPT.

kosh vorlon say:

Hi retred,

 

They don't stop MSE from starting but can cause it to show unprotected due to failure to run a recent scan because they clear the log used by MSE to determine that.  This is the fix for MSE problems caused by those two products:

 

If you are using CCleaner, please see the following compliments of Le Boule:

 

Piriform has programmed it for some reason to remove part of the MSE History files and it's deleting the files MSE checks to determine the last time a scan was conducted. This file removal has no effect on the antimalware protection provided by MSE.

 

Open CCleaner Applications Tab, then Under Windows uncheck MS Antimalware...that should prevent CCleaner from deleting the MS Antimalware scanning history files.

 

Per suggestion of MartinHampton you may also need to uncheck MS Management Console.

 

You may need to reboot the computer and run another quick MSE scan to create an entry in the MSE scanning history log (as this stops it from being deleted, but does not rebuild it).

 

**********************************************************

 

Do you use Advanced System Care? If so, see the following compliments of ahaap:

 

Update ASC to the most recent version which resolves most of the problems.  If not, continue with the following.

 

Open ASC, in the upper right corner, click More, Settings

 

Scroll to Quick Care, Privacy Sweep, go to Windows, uncheck MS Management Console.

 

Scroll to Deep Care, Privacy Sweep, go to Windows, uncheck MS Management Console.

 

If using ASC Pro, you may also need to revise the “Auto Scan” settings in a similar fashion. 

 

Run another Quick scan with MSE to repopulate the logs and that should do it.

 

If using ASC and convinced ASC is the issue but the above didn’t help, check with ASC support for help: http://www.iobit.com/support.html.

 

I hope this helps.

 

Good luck!
