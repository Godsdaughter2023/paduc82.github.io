ken sheridan say:

Selecting a value by means of the date picker does not cause the control's AfterUpdate event procedure to execute.  Putting code in the event procedure to move focus to another control won't help because, that code itself won't execute until focus is moved from the first control or it is otherwise explicitly updated.

 

The method I generally adopt to select a specific year/month is rather like those on-line forms which require you to enter the expiry month of your credit card, using two combo boxes, one which lists all years within an appropriate range, the other which lists all months of the year by setting its RowSourceType property to 'value list' and its RowSource property to:

 

1;January;2;February;3;March;4;April;5;May;6;June;7;July;8;August;9;September;10;October;11;November;12;December

 

its ColumnCount property to 2, and its ColumnWidths property to 0.

 

Set them by default to the current year/month by putting the following in the form's Open event procedure:

 

    Me.cboYear = Year(VBA.Date)

    Me.cboMonth = Month(VBA.Date)

 

In the AfterUpdate event procedure of each put:

 

    Dim strFilter As String

 

    If Not IsNull(Me.cboYears) And Not IsNull(Me.cboMonths) Then

        strFilter = "Year(MonthYear) = " & Me.cboYear & _

              " And Month(MonthYear) = " & Me.cboMonth

         Me.Filter = strFilter

         Me.FilterOn = True

    End If

    I only need two pieces of information in the footer -

    - On the left, I need "Page "n" of "x"

    - On the right, I need the word "Job" followed by the text from a textbox on the userform

     

    The code I was using (that worked perfectly in Word 2003) is below...

     

     

     

    Dim n as String

    Dim rG as Range

     

    'set footer
    n = "Job " & TextBox4.Text

    With ActiveDocument.Sections(1)

        .PageSetup.DifferentFirstPageHeaderFooter = False

        Set rG = .Footers(wdHeaderFooterPrimary).Range
        ActiveDocument.AttachedTemplate.AutoTextEntries("Page X of Y").Insert Where:=rG
        Set rG = .Footers(wdHeaderFooterPrimary).Range
        rG.Collapse wdCollapseEnd

        rG.Text = vbTab & vbTab & vbTab & vbTab & vbTab & n

       .Footers(wdHeaderFooterPrimary).Range.Fields.Update


    End With

     

     

    Can anyone offer a solution please?

    graham mayor say:
    The issue is probably that the autotext entry does not exist (or at least exist where the macro expects it to be). It would probably be better to simply abandon the autotext entry and build the footer as you want it e.g.

    Dim n As String
    Dim rG As Range
    Dim iRM As Long

    'set footer
    n = "Job " & TextBox4.Text

    With ActiveDocument.Sections(1)
        With .PageSetup
            iRM = .PageWidth - .RightMargin - .LeftMargin
            .DifferentFirstPageHeaderFooter = False
        End With
        Set rG = .Footers(wdHeaderFooterPrimary).Range
        rG.Text = "Page "
        rG.Collapse wdCollapseEnd
        ActiveDocument.Fields.Add rG, wdFieldPage, , False
        rG.Start = .Footers(wdHeaderFooterPrimary).Range.End
        rG.Text = " of "
        rG.Collapse wdCollapseEnd
        ActiveDocument.Fields.Add rG, wdFieldNumPages, , False
        rG.Start = .Footers(wdHeaderFooterPrimary).Range.End
        With rG.Paragraphs(1).TabStops
            .ClearAll
            .Add Position:=iRM, Alignment:=wdAlignTabRight
        End With
        rG.Text = vbTab & n
        .Footers(wdHeaderFooterPrimary).Range.Fields.Update
    End With
