john - project say:
Karen,
Effectively the milestones add nothing to the summary line actual duration or percent complete. Therefore your single performance task is the only data point that enters into the equation I noted earlier for summary line percent complete.

Bear in mind that the duration of the milestone is not contributing to the summary line duration, but rather the apparent lags in the predecessors between the several milestones. The duration of a summary line is the number of working days between the start of the earliest subtask to the finish of the latest subtask under that summary line.

But just to clarify one item. If you have a summary line with only milestones as subtasks, the formula for summary line percent complete isn't really a formula but rather an "if" statement. Until ALL of the milestones are 100% complete the summary line will remain at 0%. Keep in mind what a milestone really is. It is a point in time. It has zero duration. It is either complete or not complete. A milestone should never have resources assigned because there is no work or material associated with a milestone. A meeting or delivery is not a milestone. Those are performance tasks, albeit of very short duration.

Hi Mike,

 

I think the rule is that a negative difference from (the integer value of) the average takes priority over a positive difference of the same magnitude.

 

@ Sixthsense:

 

My offering, for what it's worth:

 

I used the sample data you provided above, rather than follow the links, so the cell references might be different than in the files. The data occupied A2:C10, with headings in row 1. Leave column D blank for the moment, and put this formula in E2:

 

=INT(SUMIF($B$2:$B$10,B2,$A$2:$A$10)/COUNTIF($B$2:$B$10,B2))

 

and copy down to row 10. This returns the appropriate average for the bunch. Put the following formula in F2 and copy down:

 

=1000*B2+ABS(E2-A2)-SIGN(E2-A2)*COUNTIF(B$2:B2,B2)/1000/(IF(A2<=E2,10,1))

 

This gives you the difference, but it is actually made up of three parts:-

 

the bunch number is multiplied by 1000 (as your largest data value is 359);

the magnitude of the difference between data value and the average is added on;

a small number is either subtracted or added depending on the sign of the difference.

 

Put the following formula in G2 and copy down:

 

=SMALL($F$2:$F$10,ROW(A1))

 

This sorts the numbers from column F, and this formula in H2:

 

=RANK(G2,$G$2:$G$10,1)-COUNTIF($B$2:$B$10,"<"&INT(G2/1000))

 

when copied down gives the rank of those numbers, but for each bunch number.

 

Finally, in D2 (or C2) you can have this formula:

 

=INDEX($H$2:$H$10,MATCH(F2,$G$2:$G$10,0))

 

which will give you the ranks as you require them. This is what it looks like:

anna ma msft say:

Hi Santiago N,

 

In Teams there isn’t any options to directly change the template name, you can only change the name for created PLC teams by “Edit team”.

 

If other 3 groups isn’t shown in OWA, may be they have been hidden from Outlook, you can connect to Exchange Online PowerShell, run the following command: Get-UnifiedGroup -identity "Staffteamtest" | FL HiddenFromExchangeClientsEnabled

 

If the result is True, you can then run: Set-UnifiedGroup -identity "Staffteamtest" -HiddenFromExchangeClientsEnabled:$false

 

After run the commands, you may wait a while to let PowerShell apply the changes, then go to Outlook Web App to see if you are able to see the groups. Here’s the relevant article: Get-UnifiedGroup, Set-UnifiedGroup, Connect to Exchange Online PowerShell.

 

About changing PLC notebook contents, as far as I know, you can directly modify it, like editing, deleting…etc., if you don’t like to template it provided, you may create a new sessions and new tabs in it.

There is an issue the Outlook Team is investigating where if you have the IMAP account as an Outlook.com alias and you then try to add that account to a new Outlook profile it will try to add the account as an Exchange account.  If you try creating another profile and try to add the IMAP account to the profile and it adds as Outlook.com Exchange account then that would be the root cause.  I'm not aware of a workaround yet aside from removing the IMAP account as an alias of your Outlook.com account.  You might have done that originally to use to purchase or sign into Office.  If so I would not remove the alias and wait on the bug fix.
