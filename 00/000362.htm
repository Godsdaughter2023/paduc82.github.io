scott .. b say:

MSCVR71.dll Is a Visual Studio C++ Runtime error indicating a missing file from the programming runtime libraries.  It is an important file used for the operation of devices, web browsing, and video accelerators.  There are many causes for MSCVR71 to be missing, one of which is improper installation of a program.  Since that is what you have recently done (Skype?), suspect that might be the problem.  You should correct it, even though it is not (yet) causing other problems.

 

Reinstall any recently installed applications. If you are experiencing the msvcr71.dll error after you have installed any new application or program on your computer, then uninstall and reinstall the application again to rectify the error. Incorrect installation often causes DLL errors. That is why before proceeding with a new installation, you should always ensure that installation setup files and the disk you are using for installation are not damaged.

 

Further, in case you are facing msvcr71.dll errors only when you try to open a particular program, then reinstall that program to replace the missing or corrupt files. This should fix the msvcr71.dll error.

 

Another reason for this error can be a corrupted file caused by malware, but I doubt it.  To play it safe, run a full virus scan as well.

jay freedman say:

Shouldn't I have a paste special option when I right click in word?  I pretty sure it used to be there....

No, it isn't and wasn't.

In Word 2003 and 2007, the right-click (context) menu had only a Paste item. Word 2007 had the small addition of a default keyboard shortcut, Ctrl+Alt+V, to open the Paste Special dialog. That shortcut is also in all later versions.

Starting in Word 2010 and continuing to the present, the context menu contains a Paste Options section with icons for the various types, depending on what's available on the clipboard,

but the Paste Special command is only in the dropdown from the Paste button on the Home ribbon (or from its shortcut).

ossiemac say:

If you want to make the combo box visible when you click in a particular cell (where the combo is located) then the following method.

Name the combobox so that the suffix of the name distinguishes the position of the combo. eg if the combobox is on row 3 then name it RentalBox3 and for the one of row 4 name it RentalBox4 etc.

Then when you click in the cell you need a selection event like the following that will unhide the required combobox, the name of the combobox is identified by the number suffix that should match the row number.

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Dim cboCtrl As OLEObject
    Dim lngRow As Long
    If Target.Cells.Count = 1 Then 'Only process if single cell selected
        'Only process if user clicks in specified range
        If Not Intersect(Target, Range("C3:C50")) Is Nothing Then
            lngRow = Target.Row
            Set cboCtrl = ActiveSheet.OLEObjects("RentalBox" & lngRow)
            cboCtrl.Visible = True
        End If
    End If
End Sub

The combobox change code you already have will hide the combo box after you make the selection.

Public Sub ShowAutocomplete(rngTarg As Range)
     Dim strVF As String
     Dim cboTemp As OLEObject
     Dim ws As Worksheet
    
    On Error GoTo errHandler
     Set ws = ActiveSheet
     Set cboTemp = ws.OLEObjects("TempCombo")
     
     With cboTemp
         ' Clear and hide the combo box
         .ListFillRange = ""
         .LinkedCell = ""
         .Visible = False
     End With
    
    If rngTarg.Validation.Type = 3 Then
        'The cell contains a data validation list
        Application.EnableEvents = False
        ' Get the data validation formula
        strVF = rngTarg.Validation.Formula1
        strVF = Right(strVF, Len(strVF) - 1)
        With cboTemp
            'Show the combobox with the list
            .Visible = True
            .Left = rngTarg.Left
            .Top = rngTarg.Top
            .Width = rngTarg.Width + 15
            .Height = rngTarg.Height + 5
            .ListFillRange = strVF
            .LinkedCell = rngTarg.Address
        End With
        cboTemp.Activate
        ' Open the drop down list automatically
        ActiveSheet.TempCombo.DropDown
    End If
     Application.EnableEvents = True
     On Error GoTo 0
     Exit Sub
errHandler:
    Application.EnableEvents = True
    
    'If it's 1004 there's no data validation in the cell
    If Err.Number <> 1004 Then
        MsgBox "Error " & Err.Number & " (" & Err.Description & ") in procedure ShowAutocomplete"
    End If
 End Sub
