rick rothstein, mvp - excel say:

Here is another way to write the code using a simpler temporary formula , a direct range assignment rather than a range Copy and using the first empty column after the last piece of data so that the worksheet is not "burdened" with inserting a column only to delete it afterwards...

Sub DeleteContiguousDuplicates()
  Dim LastRow As Long, UnusedCol As Long
  Const StartRow As Long = 2
  Const ColWithDups As Long = 7  'Column G
  LastRow = Cells(Rows.Count, ColWithDups).End(xlUp).Row
  UnusedCol = Cells.Find(What:="*", SearchOrder:=xlByColumns, _
              SearchDirection:=xlPrevious, LookIn:=xlFormulas).Column + 1
  Application.ScreenUpdating = False
  With Cells(StartRow, UnusedCol).Resize(LastRow - StartRow + 1)
    .FormulaR1C1 = "=IF(RC[" & (ColWithDups - UnusedCol) & "]=R[-1]C[" & (ColWithDups - UnusedCol) & "],1,"""")"
    .Value = .Value
    On Error Resume Next
    .SpecialCells(xlCellTypeConstants, xlNumbers).EntireRow.Delete
    On Error GoTo 0
    .Clear
  End With
  Application.ScreenUpdating = True
End Sub

Note: I provide two constants (the Const statements) allowing you to set the column with the data containing duplicates and the start row for that data.

oakparkme say:

This isn't fancy, but it will work...

 

Create a scratch column (say, Column Q). Q1=1. Q2 =EXACT(your content column row 1, your content column row 2). This tests whether Row 2 should be retained (i.e., is not the same as Row 1) or should be deleted (i.e., is the same as Row 1).

 

Create a second scratch column, say R. R1=1; R2=R1+1.

 

Copy the Q2 and R2 formulas down to your last row of data.

 

Select and copy Columns Q and R; paste-special-values.

 

Now sort your entire data by Column Q. All the rows with "TRUE" in Q will be grouped together; delete them with a single command.

 

Then, sort again by Column R; this returns your surviving data to its original row order.

 

Finally, delete Columns Q and R.

 

Ineligant, but quick and effective.

bob jones aka: cybertaz say:

Since it isn't specified, the first suggestion is to confirm that Office 2011 is fully updated -- 14.5.2 is the current version. Also confirm that your installation of OS X is current. Then run Disk Utility to repair disk permissions followed by a Restart. If that doesn't correct the issue...

Using the Shift key enables what's called a safe launch of the program. It is a troubleshooting technique which narrows the probable source the problem. When using that method any Add-ins & startup items, customizations [e.g., macros, etc.] as well as Excel's preferences are bypassed.

If you're certain that you have none of the external items or customizations, the issue is most likely a damaged preferences file. You can test it specifically by;

  1. Quit all Office programs
  2. Go to ~/UserHome/Library/Preferences
  3. Locate the com.microsoft.Excel.plist file
  4. Either move that file to the Desktop or change its name
  5. Launch Excel normally to see if the problem persists

If the problem is resolved you can delete the old preferences file - the program will create a new one when you next Quit. If you still have the problem you'll need to investigate further to determine what external forces are at fault.

Comparison of Visio 2002 and Visio 2013 - Microsoft Community

mr.piddybuddy say:

Comparison of Visio 2002 and Visio 2013

I very much hope this is somehow read by the Visio programmers and my words are heard.

Years ago I started with Visio 2002 as the most convenient drawing tool (CorelDraw is a pita compared to Visio), this year I was forced to switch to Visio 2013 (along with other Office tools) which - as we all knwo - are also a pain comparing to good, nice and comfortable old Office 2003. However, those times are gone and now it's harder to accomplish tasks with Office, but there are also some ups.

Let me list what I found:

Annoying things in Visio 2013 (partly compared to old Visio 2002)

Layers
- in the "Assign to layer" windows, layer names are not sorted alphabetically and you cannot sort them there
- in the "Layers" window:
- the layers are automatically sorted alphabetically, even if you don't want it
- there are columns with resizable column head, but nothing can be sorted by clicking them
- the column widths are always reset to a default value so you cannot see the full layer name and have to resize the column every time
- the "Layers" window cannot be resized to show all layers at once

Guides
- sometimes when you drag a guide out of the ruler, it does not work or at least the guide is not visible
- when dragging a new guide it is barely visible
- while dragging an already placed guide it becomes invisible
- when dragging a guide it changes colour and style
- when selecting multiple guides only the first guide is thick and the other guide(s) change from dashed to solid (for the time of being selected)
- guides cannot be locked without putting them on an extra layer

Note: the items about visibility seem to be related to the zoom level. It looks like Visio does not define a solid thickness for the guides, but calculates it.

Line properties
- in old Visio 2002 you could select a line thickness, then draw a line and click the "Line thickness" button again and again to apply the selected line thickness -> in the new Visio the button just opens a selection and you have to select the line thickness every time
- in old Visio 2002, if you wanted to apply corner rounding, you selected "Corner rouding" from menu and a window appeared to select a rounding -> in the new Visio the button "Corner rounding" only opens a side panel with a lot of other properties

Shapes
- in old Visio 2002, if you wanted to make a solid shape (rectangle) without outline but with corner rounding you could simply set "No outline" and then set the corner rounding value -> in the new Visio you can set "No outline", but then you cannot apply corner rounding; you have to set an outline with 0pt thickness, then you are allowed to apply corner rounding
- in the new Visio, shape handles are little white rectangles with a grey outline which has no contrast to the background grid -> in old Visio 2002 they were filled green and clearly visible
- in old and new Visio, when changing the reference point of a shape from like "Middle-middle" to "Lower-left", the shape is moved rather than to adapt the position values
- in old and new Visio you can select shapes on hidden layers and even move them
- when moving a shape at a certain zoom level, the shape can become invisible while moving -> when decreasing the zoom level, it becomes visible again
- after grouping shapes, the group frame should be the size of the sum of the shapes, but it is not -> instead it's a little bigger, which makes it harder to place the group correctly

What has gone wrong or worse in Visio 2013 compared to Visio 2002?
- when drawing a rectangular, grey filled shape with black outline and setting reference to "Lower-middle" and then set Y position to 1mm, after that creating a horizontal guide and also set the guide to Y = 1 mm, and then selecting both (the shape and the guide), the guide is congruent with the shape outline (should be in the center of the outline)
- when selecting a shape inside a group, the shape's position in relation to the reference point of the group is incorrectly shown in the window "Size & Position"
- when exporting pictures and switching PPI setting between "user defined" and "screen", it reverts setting "user defined" back to 96 when switching to "screen", instead of just keeping the previously entered value
- when exporting pictures, the resulting image dimensions in Visio 2002 only depended on the true size of the selected shape(s) and the PPI setting (which is logical) -> in Visio 2013, the scale setting of the sheet additionally affects the result (example: if you draw a square with 50x50 mm on two sheets, one with 1:1 scale and one with 1:2, then export both with 300 dpi/ppi -> one image has ~600 pixels, the other 300 pixels) --> this is a bug, because the image is still 50x50, so approx. 2 inches and should in both cases have ~600 pixels
- you cannot fill every shape with a gradient anymore, the option is greyed out if the shape is not rectangular, at least when grouped or loaded from a Visio 2002 document --> only via shapesheet it is possible to set foreground and background fill colour


Libraries
- after putting a shape into a libary and editing it, the shapes itself is far off the background sheet, because the relative coordinates have been copied from the sheet it came from instead of recalculating them according to size of the shape (middle-middle) or set to zero (lower-left) (this has been in Visio since, like, ever)

Save as...
- PDF: Visio resets the options for PDF in the export options window every time you start to export a sheet
- PNG: if you have several Visio files open at the same time and export a PNG in one file and change the settings to eg. 300dpi, those settings are not the same when later starting to export from another file

--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Positive, new features of Visio 2013
- the sheet can extend automatically when dragging outside of its former boundaries
- magnetic guides to help placing shapes aligned and positioned

Missing features
- there is no display to what layer a shaped is assigned to, except for when opening the "Assign to layer" windows (circumstancial)
- there is no quick way to lock a shape or the shape's layer
- there is no way to prevent grouped shapes from ungrouping or from selecting single shapes in the group, other than to open and modify the shape sheet
- Visio does not remember the last "Save as" file format
- shapes in libraries cannot be sorted or filtered
- shape libraries do not offer the use of grouping shapes into subfolders
- you cannot search shapes in shape libraries (like a filter) --> the shape search does not include currently open libs
- Visio does not adapt line thickness and character size when copying a shape from another sheet or from a library with different scale

charlesa22 say: In Outlook 2016 when you can't see the email correspondence history and the "social connector" option has been enabled, it's because the "People Pane" setting has defaulted to "Off".  Choose the "View" menu option and click on the "People Pane" tab which should be visible between the "Layout" group and the "Window" group.  The default setting is "Off", so select "Normal".  The email correspondence history should now appear at the bottom of the window when you open an Outlook contact and when you view individual correspondent's emails.
