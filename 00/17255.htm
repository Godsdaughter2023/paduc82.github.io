    Make sure that your "problem" file is the active document in Word. Click File | Options | Trust Center | Trust Center Settings | Privacy Options. Clear the check mark next to "Remove personal information from file properties on save" and click OK twice.

    Sub ReplaceQuotes()
    Dim vFindText As Variant
    Dim vReplText As Variant
    Dim sFormat As Boolean
    Dim sQuotes As String
    Dim oRng As Range
    Dim i As Long
        sQuotes = MsgBox("Click 'Yes' to convert smart quotes to straight quotes." & vbCr & _
                         "Click 'No' to convert straight quotes to smart quotes.", _
                         vbYesNo, "Convert quotes")
        sFormat = Options.AutoFormatAsYouTypeReplaceQuotes
        If sQuotes = vbYes Then
            vFindText = Array(Chr(147), Chr(148), Chr(145), Chr(146))
            vReplText = Array(Chr(34), Chr(34), Chr(39), Chr(39))
            Options.AutoFormatAsYouTypeReplaceQuotes = False
            For i = LBound(vFindText) To UBound(vFindText)
                Set oRng = ActiveDocument.Range
                With oRng.Find
                    Do While .Execute(vFindText(i))
                        oRng.Text = vReplText(i)
                        oRng.Collapse wdCollapseEnd
                    Loop
                End With
            Next i
        Else
            Options.AutoFormatReplaceQuotes = True
            ActiveDocument.Range.AutoFormat
        End If
        Options.AutoFormatAsYouTypeReplaceQuotes = sFormat
    End Sub

    You have a bad data design - you should use a database.

    Select the cells E5 to BX46 and run this macro to convert your cross tab table to a database. When you are done, select the new three column database, and use it as the source of a new pivot table. When you create your pivot table, put your "row" values in the column area, your "column" values in the row area, and your "value" values in the data area. Then use Grouping on both the row and column areas ot get your larger bin sizes.

    If you cannot get this to work, send me your file -  bdeitrick at alum dot mit dot edu   -  and I will set up the pivot table for you. 

    Sub MakeDataBaseTableFromCrossTab()
        Dim SummaryTableRange As Range
        Dim PivotTableSheet As Worksheet
        Set SummaryTableRange = Selection
        ActiveWorkbook.PivotCaches.Add _
            (SourceType:=xlConsolidation, _
            SourceData:=Array(SummaryTableRange.Address(True, True, xlR1C1, True))) _
            .CreatePivotTable TableDestination:="", _
            TableName:="PivotTable1"
        Set PivotTableSheet = ActiveSheet
        With PivotTableSheet
            .PivotTableWizard TableDestination:=ActiveSheet.Cells(3, 1)
            .PivotTables("PivotTable1").DataPivotField.PivotItems("Sum of Value").Position = 1
            .PivotTables("PivotTable1").PivotFields("Row").Orientation = xlHidden
            .PivotTables("PivotTable1").PivotFields("Column").Orientation = xlHidden
        End With
        Range("B4").ShowDetail = True
        Application.DisplayAlerts = False
        PivotTableSheet.Delete
        Application.DisplayAlerts = True
    End Sub

    An Excel error in a cell is the most probable thing that will stop the code on the line you have highlighted.

    Continue as Hans suggested with On Error Goto 0 (Disables the On Error) and run the code again.

    When the line is highlighted in the debugger, open the Immediate window while the line is still highlighted (Ctrl/G to open Immediate Window) and type the following in the Immediate window. (Note the dot between c and Address.)

    ? c.Address

    It will return the cell reference that has stopped the code and you can then look at the worksheet and see if you can work out what is the problem from there.
