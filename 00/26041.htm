ken sheridan say: All I can say is that it works for me.  Not Me.AllowEdits will evaluate to TRUE if the AllowEdits property is currently FALSE, and vice versa, so clicking the button will toggle between the two.  In cases like this I would usually change the button's Caption property in the event procedure too.

What type of button are you using?  It must be a conventional command button.  You cannot use a toggle button as, unlike a command button, the control is disabled if the AllowEdits property of the form is False.   This is because a toggle button has a Value property, which is either TRUE or FALSE, so could be used as a bound control.

david roodman say:

I got rid of this problem after making a few changes. I'm not sure which were crucial. The enemy appears to be the Microsoft Office Upload Center, which is a file synchronization program distinct from OneDrive. Its purpose appears to be to assure faster uploads of Microsoft Office files, which is good when two people are working on the same shared file. But it also appears to be a junky program. The underlying problem might be its reliance on Microsoft Access/Microsoft Jet, which is not a grown-up database program. "From a data access technology standpoint, Jet is considered a deprecated technology by Microsoft, but Microsoft continues to support [the latest version] as part of Microsoft Access."--Wikipedia article on Microsoft Jet

1. Right-click the OneDrive system tray icon (bottom right of screen, at least in Windows 7). Choose Settings. Un-select "Use Office to sync files faster and work on files with other people at the same time." Click OK. I found that this did not immediately solve the problem by itself, perhaps because the program still had pending uploads it was stuck on.

2. In c:\users\[Your user name]\AppData\local\Microsoft\Office\15.0, rename the "OfficeFileCache" folder. You might need to substitute another number for "15.0" depending on your Office version. And then if things still seem to be working OK for a while, you can probably just delete the folder. I did. I believe this is the structure the Upload Center uses to track what files it needs to sync, but if it loses the data base, it can rebuild it, and extricate itself from whatever buggy problems are causing it to get stuck in infinite loops. When I looked inside this folder, I found a multigigabyte Access/Jet database. I suspect this is the core issue: I have so many Office files that the database got huge and Jet's performance degraded.

3. I may have also run msconfig to disable the launch of the upload center at start-up. In Windows 7 at least, you do this by hitting the Windows icon button on the keyboard, typing "msconfig", hitting Enter, going to Startup, and un-selecting the item for MSOSYNC.EXE.
