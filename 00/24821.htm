bill manville say:

I would never personally recommend the use of shared workbooks in Excel.

This is partly due to a poor record of reliability in the past, but more because in situations such as the one you are describing conflicts can arise. 
Shared workbooks do limit the set of actions users can take but within the set of allowed actions there is clearly the possibility of conflicting actions between two users (e.g. both update the same cell).  Whilst the software may alert the user to the conflict when the second version is saved it then becomes difficult to know the correct resolution.

If having simultaneous update access is not a major requirement, opening for exclusive use and having the second user wait for the workbook to become available is preferable.

If simultaneous update access is a necessity then I would recommend re-thinking the application design and using a database as the data repository - as database systems are designed for such use.
