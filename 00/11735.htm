john p spencer say:

The problem here is one of design.  Those milestones should be a separate table and each milestone would be a separate record. 
Table:ProjectMileStones
ProjectID
MilestoneID
MilestoneDate

You might also want a simple lookup table:
Table:Milestones
MilestoneID
MileStoneName
MilestoneSequence (number field showing the order of the milestones)


With the present structure, you would probably be better off creating a function to examine the seven fields and return the needed data - a date or a field name.

 

Public Function fGetMilestone(idProjectID, strDataType As String _
    , Optional tfNextDue As Boolean)
Dim strSQL As String
Dim vReturn As Variant: vReturn = Null
Dim rst As DAO.Recordset
Dim I As Integer
Dim iFieldWithValue As Integer

   'Assumption ProjectId is primary key number field in the table.
   strSQL = "SELECT [Audit start date]" & _
   ", Design and testing of audit tool completed" & _
   ", Data collection completed" & _
   ", Data analysis completed" & _
   ", Proposed date for Audit Report Form to be sent to Clinical Audit" & _
   ", Action plan to be developed by" & _
   ", Approximate date for re-audit" & _
   ", 'All Done' as AllDone" & _
   " FROM [ProjectReportTable] WHERE ProjectID=" & idProjectID

   Set rst = CurrentDb().OpenRecordset(strSQL)
  
   'Step through the fields from back to front and find the first one with data
  
   For I = rst.Fields.Count - 1 To 0 Step -1
      If IsDate(rst.Fields(I)) Then
         iFieldWithValue = I
         Exit For
      End If
   Next I

   
   If tfNextDue = False Then
      If strDataType = "TheDate" Then
         vReturn = rst.Fields(I)
      Else
         vReturn = rst.Fields(I).Name
      End If
   Else 'tfNextDue = true
      If strDataType = "TheDate" Then
            vReturn = rst.Fields(I + 1)
      Else
            vReturn = rst.Fields(I).Name
      End If
