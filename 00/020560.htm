scottgem say:

A questionnaire or survey requires a very specific structure. Unfortunately that structure doesn't jive with paper forms. So you have to decide how to handle it.

The structure looks like this:

tblQuestion

QuestionID (PK Autonumber)

Question

tblRespondent

RespondetID (PK Autonumber)

Firtname

lastname

other info about respondent

tblResponse

ResponseID (PK Autonumber)

RespondentID (FK)

ResponseDate

tblAnswer

AnswerID (PK Autonumber)

ResponseID (FK)

QuestionID (FK)

Answer

You might need to expand your question table to include an Answer type (i.e. Text, Multiple choice, Yes/No, etc.) 

I just recently setup something like this for my HR people to do Exit Interviews. I use a subform bound to the Answer table. when the first is loaded the first question loads and the form displays the first question and the possible answers. they hit a Next button and the next question is displayed with the form set up for the answers. 

By the way many of their questions use a scale type answer like the first form. So I use Scale as an Answer type.
