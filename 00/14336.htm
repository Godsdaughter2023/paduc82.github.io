brian tillman [outlook mvp 2007-2019] say:

Then the damage is not being caused by Outlook, it's being done after the messages leave Outlook.

There are several possibilities.  The first is if you have your antivirus configured to scan mail.  That's a huge source of message interference and if multiple recpients see the damage, then this is a likely possibility.  Completely uninstall your AV program and reinstall it with a custom install, leaving out the mail scanning feature.  It will not make you less safe.  You can also test other add-ins on your PC by starting Outlook in safe mode.  Hold Ctrl when starting Outlook and it should ask you if you want to start in safe mode.  Safe mode disables Outlook add-ins.  If your messages get through OK, then you can investigate the add-ins you have.

The second is that something on the recipient's mail server is causing the problem.  This is a likely choice when multiple recipients who share a domain see the damage.  There's not too much that can be done about this, other than have them complain to their service provider.  You can tell if the problem lies here if the receipients see the damage when accessing the mailbox and reading the message via a web browser.

The third is that something on the recipients' PCs are cauing the problem, if your recipients use a mail client.  They can tell the difference between this and the last case in the same way, by using a web browser to read the message on the server.  If it's OK on the server but not on the PC, then it's more likely a problem on the PC.  The first place to look is, again, the antivirus program.  If it scans incoming mail, then the scanning should be removed and another test performed.  Testing within the mail client depends on the client, of course.  Since we don't have that information, there's nothing to suggest at this time.
