logiebard say:

Windows 10 Update kills HDMI out - Solved for NVIDIA Geforce 8200 M G for HP/Compaq Laptop

Windows Update may install the latest driver from NVIDIA even though an older driver available from HP (i.e. Windows 7) should be preferred. This is because HP may optimise the driver for their implementation of HDMI out. The general driver for the Geforce 8200 M may not support the proprietary HDMI out function, resulting in a black screen.

The solution is to download and install the latest HP Nvidia driver. However, Windows update may attempt almost immediately to re-install the "latest" non HP optimised, driver resulting in a black external HDMI screen again.

This behaviour can only be prevented by one of the following:

1) The Microsoft update engine learns heuristically that the W7 HP driver is best (is this possible ?)

2) The user is allowed to effectively turn-off windows update for that particular piece of hardware, until NVidia, Microsoft or HP catch on that this problem exists and steps are taken to prevent reoccurrence.

Until then, it may help to change the status of the WiFi connection to metered temporarily to prevent WU from sending updates, or to change the System setting for Hardware to "do not automatically download manufacturers' drivers that are available for your devices". However on my updated W10 Home, this has been changed from "drivers" to " apps and custom icons".

It has been possible in some versions of Windows to edit update policy so that specified hardware is not automatically served with driver updates using gpedit.msc. A blunter tool might be to use wushowhide.diagcab.

Comments welcome.......

UPDATE:

The disabling of the HDMI port appears to be by design. I have duplicated this on Windows 7 (not just w10) when the latest driver from NVIDIA is installed via windows update. If the hdmi port is working with the older driver the following problems existed:

1) If an HDMI monitor is left in a USB socket when powering down the PC might hang

2) With the older driver the GPU/CPU temperatures tended to be 10 degrees Celsius higher than with the latest driver that disables the hdmi port.

I have accepted that the latest driver is probably the best and I use the vga port for an external monitor with audio routed separately. Neither Nvidia nor the PC manufacturer has confirmed the issues I found however.

The issue you are having with McAfee is most likely that the Driver Verifier has been enabled and it is causing the mcsacore.exe (McAfee process) to crash with the mfesapsn.sys on login.

Disabling the Driver Verifier should stop this McAfee crash on login.

To disable the Driver Verifier do the following (either in Normal Mode or Safe Mode) :

Windows Key + R > type verifier.exe in the "Open" box > OK > Delete existing settings > Finish

Restart the computer.

Did someone enable the Driver Verifier because of a previous blue screen issue?

Some earlier minidump files showed a KMODE_EXCEPTION_NOT_HANDLED (1e) which appeared to be caused by the bwcW8x64.sys.

The bwcW8x64.sys is a driver for the Killer network adapter so I might suggest to update this driver - here is a link to the latest driver:

http://www.killernetworking.com/support/driver-downloads/standard-drivers

Or you could try MSI (On-board LAN) :

https://ca.msi.com/Motherboard/support/970-GAMING.html#down-driver&Win10 64

I might suggest to remedy a blue screen issue with the Killer card to uninstall the Killer Suite and just use the driver only package.

There are some other potentially problematic drivers so if you still have blue screen errors afterwards post back.

Hi,

After you remove the sidebar this is how to make Windows remember what size you made Internet Explorer last tine you used it.

Read Jack's answer:

http://answers.microsoft.com/en-us/ie/forum/ie8-windows_vista/how-do-i-make-the-default-window-size-open-to/a6261690-88e9-47ab-befb-f2cef12f2601

"Open the IE8 and size the window to your liking.  Then hold down the CTRL key while clicking the “X” to close the program.  This will set the IE window to open at this setting every time.  I hope this information is helpful."

 

You can resize the IE Window to your liking by running the mouse cursor over each edge or corner of IE until it becomes a double headed arrow > then hold left mouse button down > drag to required size.

 

If that does not fix it, here is some other advice about it:

  • Open a single Internet Explorer window to the smaller size.
  • Drag the corners of the window until it completely fills the screen. Do not use the maximize button to enlarge the window.
  • Hold down the Ctrl key, keep it depressed, and using the mouse click File and then Exit on the menu bar. Do not use the "X" in the upper right corner to close the window.
  • Internet Explorer should now open in a full window.
  • If it still opens to a smaller size repeat the above instructions, substituting the "Shift" key for the "Ctrl" key.
