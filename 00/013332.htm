jay freedman say:

The "brackets" are the way Word displays bookmarks, when this option is checked:

You're not supposed to be able to select them, and you can insert them only by using the Insert > Bookmark dialog.

The bookmark "brackets" are always paired, but it can get confusing if one or more bookmarks are inserted inside another bookmark. In your screen shot, it looks like the left bracket in row 2 column 2 pairs with the right bracket in row 4 column 4; the left bracket in row 2 column 3 pairs with the right bracket in row 4 column 3; and the left bracket in row 2 column 4 pairs with the right bracket in row 4 column 2. Only the ones in row 3 pair with their mates in the same cell.

You can click a bookmark's name in the Bookmark dialog and click the Go To button, and that bookmark will be shaded from end to end so you can see its extent. If you don't need the bookmark (for example, to use a macro that refers to the bookmark by name), you can click the Delete button in the dialog -- it deletes the brackets but not the contents of the bookmark.

By the way, if you send the document to another computer on which the "Show bookmarks" option is not checked, that user won't see the bookmarks. Since the option isn't checked by default, that includes most users.

Why would you expect to see tasks in the resource pool file if it only contains resource information?

The definition of a resource pool is a file that has data on the Resource Sheet only. The Resource Graph is a resource view and since the resource pool file does have resources but no tasks, the Resource Graph shows the resource data for all sharer files.
