Sorting by oldest to newest in query - Microsoft Community

hwm72 say:

Sorting by oldest to newest in query

How do I sort from oldest to newest in dates on a query? 

It is a query for Shoppers per Month.   In the date received column the dates are December 2013, February 2014, January 2014, November 2013, and October 2013.   The filter shows only the option of alphabetical order.  

Thank you for any help you can give a newbie.

john w. vinson say:

Date/time values are stored as numbers - larger numbers are later times. So if you sort a Date/Time value ascending, it will be sorted oldest first, newest last.

hwm72 say:  There is no offer of a filter ascending/descending, only A to Z or Z to A.  Do I have something set up wrong?       

john w. vinson say:

Aha. Your question makes more sense now!

The text string December 2013 looks (sort of) like a date to a human. But computers have no intelligence - a text field is just a text field, and "December 2013" will sort between "December Revolution" and "Decimal System". It has no way to know that you interpret it as a date and want to sort chronologically!

You'll need to convert it to a date with an expression such as

DateValue([Date Received])

which will give you a date/time value #12/01/2013 00:00:00# and sort by that. This expression will cause errors if there is a text value which isn't actually a date (e.g. "Late 2012" or "Sometime in 2012") or is null; to handle this you can use

IIF(IsDate([Date Received], DateValue([Date Received]), NULL)

scottgem say:
 There is no offer of a filter ascending/descending, only A to Z or Z to A.  Do I have something set up wrong?       

Yes, you did set something wrong. It appears that you have a TEXT datatype for your dates because you only need month. And so, its is sorting alphabetically. Fortunately for you Access is smart enough to recognize this text string as a date. So the Expression John gave you will work. It will return a date value for example:

=DateValue([DateReceived]) 

will return 1/1/14 for January 2014.

What I would suggest is that you add a Date/Time field to your table and run an Update query:

UPDATE table SET newfield = DateValue([oldfield]);

Do this on a copy of the table to test it. You can then delete the old field and replace the table with the copy once you have checked that it worked. If you only want to display month and year, you can FORMAT the field to display:

mmmm yyyy

john - project say:

Shellfish,

Each field in Project does have a "tool tip". If you hover your mouse over the column heading a brief explanation for the field will appear, and if you then hit F1, a more detailed description will appear.

No, standard fields cannot be customized, that's why they are "standard".

If the schedule is set up properly ALL tasks should have a successor and most, if not all, tasks should have a predecessor. It you want the users to see potential impacts, showing the Successors field would be very helpful but there is even a better way. Project has a feature that will show change highlighting so if a user changes the duration for example on one task, change highlighting will show how that change ripples through other tasks. To set up change highlighting you will need to activate it (I don't think it is on by default). To activate it, create a new ribbon tab. Then search the available commands to find "Change Highlighting" and add that to your new ribbon tab. Activate it. Then go to Format/Format group/Text Styles and select Changed Cells from the Item to Change list. Select an appropriate color (I chose red) and hit "OK". Now anytime a user makes a change the flowdown impact will be highlighted.

With regard to an explanation of Project's functions, in my opinion the Project Help file was a whole lot more helpful with older versions. Above I told you how to access an explanation about each of Project's fields. If you want to get more information on functions for custom fields, take a look at the following link: http://office.microsoft.com/en-us/project-help/project-functions-for-custom-fields-HP010080956.aspx.

Hope this helps and if it answers your questions, please mark as the answer. If not, how else may we help you?

Templates in Microsoft Word

doug robbins - mvp office apps & services (word) say:
  1. The normal template is Normal.dotm, not Normal.dotx
  2. It is normal for a blank document to open as Document# - Word
  3. You will find the Normal.dotm template in the following location:

    C:\Users\[User Name]\AppData\Roaming\Microsoft\Templates

    where [User Name] is the name with which you log onto Windows.

    If you do not see the AppData folder, in Windows Explorer, click on the Organize drop down and then on Folder and search options and in the Folder Options dialog, go to the View tab and select the item "Show hidden files, folders, and drives".  While there, it is a good idea to uncheck the box of "Hide extensions for known file types".

  4. If replacing the Normal.dotm template does not overcome your problem, you could start Word in Safe Mode by holding down the CTRL key until asked to confirm that you want to start it in Safe Mode and see if the problem still occurs.  If it does not, then happen, see the steps for disabling Add-ins in the Article “Why can’t I start my Office 2013 application at:

    http://office.microsoft.com/en-us/support/why-cant-i-start-my-office-2013-application-HA104011864.aspx

    If that does not help, then you might remove all traces of the Office installation by downloading the fixit from the following article:

    http://support.microsoft.com/kb/2739501
    saving it to your computer and then running it.

    Then reinstall Office by logging into     www.office.com/myaccount with your registered email address and click Install Office.
