Permantely disabling InPrivate Browsing? - Microsoft Community

lorileigh say:

Permantely disabling InPrivate Browsing?

How do I permantely disable InPrivate Browsing?
kosh vorlon say:

Hi Lori,

 

The best method depends on the version of Vista you are using.  The best method is to use Group Policy (if your version of Vista supports it).  Although written for Windows 7, Method Two in the following will do the trick: http://www.sevenforums.com/tutorials/62135-internet-explorer-inprivate-browsing-enable-disable.html.

 

Since I don't know, I will try to provide a registry solution that should work for all versions (though of course Administrators can override these or the other changes by undoing what you are about to do).  Keep in mind that this works with other versions of IE and versions of Windows that don't support Group Policy, so I'm hoping it will work here but I don't know for certain and there is a risk in doing this as I've seen no documentation about it for IE9 and many of the relevant entries don't exist (at least not on my system which is Vista using IE9) which usually isn't a good sign as the more you change, the greater the risk of possible problems.

 

Before you start, create a System Restore point AND do a full backup of the registry (just in case).  If anything at all goes wrong (if the changes cause any problems or issues or do anything other than disable InPrivate Browsing), then first do a System Restore to the point you created before the changes and see if things return to normal.  If not, replace the current registry with the backup copy and see if things return to normal.  If not, then go to the registry and undo the changes you made and see if things return to normal.  If so, this won't work and either it can't be done in IE9 or you need to find another solution.  If none of these options restore things back to normal, then I'm afraid this caused a serious registry or system problem and we'll have to see if we can recover using some other means - though what they might be depends on what problems are occurring that none of the above options resolved - so you'll need to explain that.  Again, this is NOT an officially approved procedure and there is risk involved.  Though I think it might work, I am not certain and have no documentation to backup this procedure.  I cannot recommend this given my level of uncertainty, but the decision is yours. 

 

How to Disable InPrivate Through the Registry

1. Go to Start / Search Box and type "regedit,exe" (without the quotes) and then double click on the program icon that appears.

2. The necessary setting is: HKey_Local_Machine\Software\Policies\Microsoft\Internet Explorer\Privacy\EnableInPrivateBrowsing

If that key doesn't exist (everything after Microsoft doesn't exist on my computer)...you may have to create them...do do this, navigate to the last available section of the key...mine stops at HKey_Local_Machine\Software\Policies\Microsoft\ so that's where the following instructions begin

Right click on Microsoft in the left pane and choose new, then key. Type "Internet Explorer" (no quotes) as the name of the key. Then right click on the new key you created and choose new, then key and type "Privacy" (no quotes) as the name of the key. Highlight the new privacy key, and then right click in the RIGHT pane, choose new, then dword value. Type "EnableInPrivateBrowsing" (no quotes) as the name of the item. The value will default to 0 (which is what you want - if it's something else change it to 0). Now restart your computer and you should no longer be able to enable in private browsing.

 

I hope this helps.

 

Good luck!

karl timmermans [outlook mvp 2012-2018] say:

Not entirely sure what you're asking in terms of a "list".

 

If by list you mean being able to create an external file with that information, you can export your inbox to any of the support file formats (csv, Excel xls, Access .mdb) selecting the desired fields. That will give you a complete list of every email contained in your inbox but will include "duplicate" <from> addresses assuming that you have received more than one email from the same sender which you'll need to clean out.

 

To export - from O'2010 menu (either one of the two will bring you to the identical import/export wizard)

 

File --> Open --> Import

 

or

 

File --> Options --> Advanced --> Export

 

When the window appears with the Map Custom Fields button, be sure to click on it and select the fields that you want to appear in the exported file.
