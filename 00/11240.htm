stefan blom say:
This kind of issues are usually caused by incorrect cross-references. These, in turn, may be the result of pressing Enter at the beginning of a cross-referenced paragraph, which expands the hidden bookmark to include the newly created paragraph. The workaround is to always create new paragraphs, or add content, at the end of a paragraph of text. For existing documents, recreating the cross-references will be the easiest solution.           

To clarify, a hidden bookmark is created by Word each time you create a cross-reference via the Cross-reference dialog box (unless of course you are creating a reference to an existing bookmark).

Help! I need a solution for a First/Middle/Last Name - Microsoft Community

dsmith44 say:

Help! I need a solution for a First/Middle/Last Name formula

I have a question - I am an IT Security Analyst and am responsible for creating IDs in our organization. All IDs are up to 8 characters in length, keeping within the Active Directory standards.  We have an InfoPath 2010 form with First Name, Middle Initial and Last Name fields to be filled out by the user.

My formula takes the first letter of the first name, the Middle Initial, and up to 6 characters of the last name to make up an 8-character user ID, and convert those in to uppercase beginning with the word "IDIR\":

translate(concat("IDIR\", substring(FirstName, 1, 1), substring(MiddleInitial, 1, 2), substring(LastName, 1, 6)), "abcdefghijklmnopqrstuvwxyz", "ABCDEFGHIJKLMNOPQRSTUVWXYZ")

It does not address the issue of if a person does not have a middle initial to make up 8 characters - it will appear as 7 characters only.  For example, if the user's name is Gerald Honeywell, this formula takes the "G" from the first name, and up to 6 characters from the last name, and puts it into uppercase for a user ID that looks like this:  GHONEYW -- when it should be GHONEYWE.

If Gerald Honeywell has a middle initial, such as the letter "P", it will appear correctly as:  GPHONEYW

If I set the Last name portion of the formula to substring(LastName, 1, 7) instead of substring(LastName, 1, 6) I get a 9-character user ID because it grabs an extra character of the user's last name if the user enters a middle initial.

How would you write a formula so that if the user does not have a middle initial, to take the first TWO (2) characters of the first name, and then up to SIX (6) characters of the last name to make up a total of 8 characters in the same formula?

Thanks to anyone who can help!

Dan

dsmith44 say: Anyone have a solution to this for me?  It would really help me out.  Thanks....

Dan

braintrove.com say:
See http://www.braintrove.com/id/125

There are two approaches you can take. It sounds like you could go for either one.

First: Concatenate the second letter of the first name with the middle initial. This will cause that second letter to be used in place of a middle initial if one doesn't exist.

Second: Use the whole last name, but then take a substring of the entire translated concatenation to get the full length.

However, this still doesn't account for short last names (i.e. "Wong Lu").

The following two updates should help all situations. Replace the numbering scheme with whatever you want:

substring(translate(concat("IDIR\", substring(FirstName, 1, 1), substring(concat(MiddleInitial, substring(FirstName, 2, 1)), 1, 1), LastName, "01234567"), "abcdefghijklmnopqrstuvwxyz", "ABCDEFGHIJKLMNOPQRSTUVWXYZ"), 1, 13)

substring(translate(concat("IDIR\", substring(FirstName, 1, 1), substring(MiddleInitial, 1, 1), LastName, "01234567"), "abcdefghijklmnopqrstuvwxyz", "ABCDEFGHIJKLMNOPQRSTUVWXYZ"), 1, 13)
