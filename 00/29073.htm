However, I did find a solution to the protable issue, so I thought I'd post it here in case someone else would need it. 

Apparently windows became confused when I connected a portable backup HDD, thinking it was on a portable unit itself.

A somehow changed value in the registry entry was what made the System Image Backup to fail as above.

The solution was;

To change the PortableOperatingSystem registry, follow these steps:

  1. Open Registry editor (regedit.exe)
  2. Locate and then select the following registry subkey: 

    HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control
  3. From the right-side pane double-click on PortableOperatingSystem Dword value
  4. Change the value data to 0 (The default value is 1)
  5. Close the registry editor
  6. Restart the computer.

gerdimae sal say:

Checking the integrity of EFI files seems to be too tricky. We suggest you to post this on TechNet forums for assistance from highly technical pool of users.

You should remove any doubts regarding your EFI files as we have what we call "Secure Boot". This Secure Boot feature, which can be configured in your BIOS, allows the BIOS to read through the Cryptographic Signature linked to your EFI files before it loads. If the EFI file has been compromised, this Cryptographic Signature gets removed, or changes. Secure Boot will then refuse to load the EFI file.
