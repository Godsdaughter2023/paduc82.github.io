jllatham say:

I'm actually not what the -4143 equates to.  I suspect it is saying "whatever default format has been set in Excel".

With Excel 2007/2010 the filename extension needs to match the specified FileFormat parameter (which is the position where your -4143 is showing up at).  If you don't specify a FileFormat parameter, it saves in the format setup in Excel as the default which can be .xls, .xlsx or .xlsm (and probably others such as .xlts and .xltm for templates).

Best to match explicitly using code like

ThisWorkbook.SaveAs  FilePath, fileformatParameter

Where fileformatParameter can be:

xlExcel8 (has numeric value of 56) to save with .xls file extension
xlOpenXMLWorkbookMacroEnabled (has numeric value of 52) (use with .xlsm files)
or
xlOpenXMLWorkbook (has numeric value of 51) (use with .xlsx files)

 

As for working with the template and saving it without triggering the Workbook_BeforeClose() event, here is one way:

Just before you get ready to save the file as a template file again, go into the VB Editor and type this into the Immediate window:

Application.EnableEvents=False [Enter]

That will prevent any events, including the _BeforeClose() event, from firing off.  You can then save the file without the annoyance of the forced save to a different file type.

The problem with the Application.EnableEvents=False statement is that it is persistent, so no other events will fire until it is re-enabled.  You could go back into the VB Editor and type this into the Immediate window:

Application.EnableEvents=True [Enter]

or just shut down Excel and re-open it, doing that will reset the .EnableEvents feature to =True.


 

lifescholar1 say: Even though you have specified an extension of .xlsm, your SaveAs command is still saving the file as non-macro enabled. Try the following:

Issue Description
============ 
Excel Online times out every 5 minutes when opening any Excel file in Excel Online without performing any actions.

Root Cause
=========
The value for IPAddressWACTokenLifetime is too small.

Solution
===========
1. Download and install SharePoint Online Management Shell.

2. Start the SharePoint Online Management Shell, run command

    Connect-SPOService https://YOURDOMAIN-admin.sharepoint.com

    to connect to SharePoint admin center. (Need to input admin username and password)

3. Increase the value of IPAddressWACTokenLifetime by running the command

   Set-SPOTenant -IPAddressWACTokenLifetime 600
4. Wait about one hour for the command to take effect.

hide and/or unhide columns and rows - support for DROPBOX - Microsoft Community

lowell zelinski say:

hide and/or unhide columns and rows - support for DROPBOX

with this version of excel for IPAD is it possible to hide and/or unhide columns and rows?

Also - does this version support saving to dropbox?

joeleblanc say:

There are two ways to hide columns:

1) Select the column(s) by tapping on its(their) header. You should see a black contextual menu pop up, with a 'hide' option

2) Select the column by tapping on the header. Drag the right edge of the column to resize it. If you drag this all the way to the left so that the width is 0, it becomes hidden.

To unhide column(s), select the columns adjacent to it(them). E.g. if C is hidden, select columns B and D. After selecting them, you should see the black contextual menu with an 'Unhide' option. 

This all applies to Rows in the same way. 

There is no support for Dropbox is this version of the Office for iPad apps.
