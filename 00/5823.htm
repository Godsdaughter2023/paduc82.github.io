<snip>

I had the same problem as described above.  The issue I found was the missing folders in C:\Windows\Installers.  The registry has references to these folders for the icons images and if you have any utilities that have deleted these files or folders, the icons will not appear correctly.  

I restored these folders (e.g. - C:\Windows\Installer\{91140000-0011-0000-0000-0000000FF1CE} ) and the icons displayed correctly again.

Jo --

Congratulations on saving the baseline.  That is key.  Here is what you can do:

  1. Apply the Tracking Gantt view.
  2. Make the changes being requested to the project, but DO NOT save the project.  The Tracking Gantt view will show you how the changes affect the project schedule compared to the baseline schedule.
  3. In the Data section of the View ribbon, click the Tables pick list and select the Variance table.  The Start Variance and Finish Variance columns will show you if the changes make the schedule slip.
  4. Click the Tables pick list again and select the Work table.  This table will show you how the changes impact the total amount of work in the project.
  5. Click the Tables pick list again and select the Cost table.  This table will show you how the changes impact the total cost for the project.

If the changes are approved by your stakeholders, save the project.  If not, close the project WITHOUT saving it.  Hope this helps.

    jllatham say:

    Try this macro - there's a link in the comments to a page that gives instructions on how to add it to your workbook for use.

     

    Sub SetPageFeatures()
    'sets page print area and
    'splits into multiple sheets
    'at date entries in column A
    '
    'works on the currently selected worksheet
    'code goes into a regular code module
    'see this webpage for instructions on adding
    'the code to a Regular Code Module:
    ' http://www.contextures.com/xlvba01.html#Regular

      'you can change any of these
      'Const values to allow for
      'changes in the worksheet layout
      'in the future
      'column with dates/products to
      'be broken up
      Const keyCol = "A"
      'first column in PrintArea
      'may be same as keyCol but by
      'declaring it separately it does
      'not have to be.
      Const firstPrintCol = "A"
      'last column in PrintArea
      Const lastPrintCol = "H"
     
      'working variables
      Dim lastRow As Long
      Dim myPrintArea As String
      Dim keyArea As Range ' all used cells in col keyCol
      Dim anyKeyCell As Range ' 1 cell in keyArea
     
      'find last cell in column A with entry in it
      Set keyArea = Range(keyCol & ":" & keyCol)
      On Error Resume Next
      lastRow = keyArea.Find(What:="*", _
       After:=keyArea.Cells(1), _
       Lookat:=xlPart, _
       LookIn:=xlFormulas, _
       SearchOrder:=xlByRows, _
       SearchDirection:=xlPrevious, _
       MatchCase:=False).Row
      If Err <> 0 Then
        'was an empty column!
        lastRow = 1
        Err.Clear
      End If
      On Error GoTo 0 ' reset error trapping to 'normal'
      'we have enough information to set the basic PrintArea
      ActiveSheet.PageSetup.PrintArea = _
       "$" & firstPrintCol & "$1:$" & lastPrintCol & lastRow
      'begin setting page breaks by removing any currently in place
      ActiveSheet.ResetAllPageBreaks
      'reset keyArea to exclude row 1 (with 1st date in it)
      Set keyArea = Range(keyCol & "2:" & keyCol & lastRow)
      'start looking for real date entries
      For Each anyKeyCell In keyArea
        If IsDate(anyKeyCell) Then
          ActiveSheet.HPageBreaks.Add Before:=anyKeyCell
        End If
      Next
      'release resources back to the system
      Set keyArea = Nothing
    End Sub
