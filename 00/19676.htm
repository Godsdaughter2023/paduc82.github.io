christinetaylor4 say:

Outlook Address book

I have 4 email accounts in OUtlook 2016. All are POP accounts except one. The IMAP account does not have a contacts folder. How do I create a contacts folder for the IMAP email address. Why is not there like the others.

karl timmermans (outlook mvp 2012-2018) say:
>>
#1 - I have 4 email accounts in OUtlook 2016. All are POP accounts except one.
#2 - How do I create a contacts folder for the IMAP email address.
#3 - Why is not there like the others.
<<

To answer the above out of order

Re: #1 - Going to assume that you have 4 POP accounts that are being delivered to the same <Inbox> which means that there would only be one folder tree for all accounts with only one contacts folder
OR
Each POP account has it's own PST file (folder tree) and only one would have a contact folder if Outlook was allowed to create a new PST file for each POP account.

Is the description above correct? For sake of this response, going to assume that the answer is <yes>

Re: #3 - Why the Imap account is different.....

Next assumption is that the Imap account was not the first email account configured in the profile which means that it would NOT have a contact folder in the IMAP folder tree. The reason being that it would default to using the default contact folder defined for the folder which is contained in the default data file where the "default" contact/calendar folders are stored.

So to answer the question as to "Why it's different" - it isn't. No different then how things would be created for POP accounts.

There is only one "default contact folder" for an Outlook profile and the data file for the first email account configured is the one which would be set as the "default data file" for the profile. The exception to this is for any Outlook.com/Exchange accounts configured in the profile - whether first or last. These account types would each have their own contact/calendar folders since that data is sync'd with the server for the email account in question.

Re: #2 - How to create a contact folder for an IMAP account

No different then any other account type. Simplest way is to show the full folder list in the Outlook Nav Pane by pressing CTRL+6 --> right-click on the root of the Imap folder tree --> select <New Folder> --> give it a name and then make sure to select <Contact Items> under <Folder Contains>.

There are two caveats to this

a) This will not create a contact folder in which any contacts iwill be automatically created unless the contact folder has been selected prior to a new contact being created

b) The more/most important second caveat is that maintaining a contact folder in an IMAP folder tree which means that it's stored in the underlying .ost file for the IMAP account and that in turn means that should the IMAP folder be removed from the profile, the ost file will be automatically deleted and any "local data" such as contact/calendar data will be lost. Similarly, if Outlook cannot find the underlying .ost file, it would recreate it and resync the email folders and all custom created folders would be lost.

If I may, I believe the OP was asking how to output Named Ranges that apply to a specific sheet.  So I modified your code thus:

Sub ListNames()
  Dim X As Long
  Const OutputSheet As String = "Sheet1"
  Dim OutputStartRow As Long
  Const OutputNameColumn As String = "A"
  Const OutputFormulaColumn As String = "B"
  Const NamesInSheet As String = "Sheet2"
  OutputStartRow =1
   For X = 1 To Names.Count
    If Mid(Names(X), 2, InStr(1, Names(X), "!") - 2) = NamesInSheet Then
        Worksheets(OutputSheet).Cells(OutputStartRow, OutputNameColumn).Value = Names(X).Name
        Worksheets(OutputSheet).Cells(OutputStartRow, OutputFormulaColumn).Value = "'" & Names(X).RefersTo
    OutputStartRow = OutputStartRow + 1
    End If
  Next
End Sub

    Any ideas?

    guruprasad ra say:

    Try the below mentioned steps.

    1. Open the word document.

    2. Insert a section break in the first page. To do this, place the cursor at the end of the 1<sup>st</sup> page > Click Page layout tab> Breaks > Section breaks > Next page.

    3. Now double click on the 2<sup>nd</sup> page header and click on the ‘Design’ tab > Click on Link to previous to disable it. (It would be enabled by default).

    Note:  We are performing the above steps as you need a right side page only for the 1<sup>st</sup> page. If the above steps are not performed, the changes done would affect the complete document.

     

    Once the above steps are performed, check the link below to on how to setup a right side page for page 1.

    http://office.microsoft.com/en-us/word-help/right-to-left-language-features-HA010351021.aspx?CTT=1#BM5
