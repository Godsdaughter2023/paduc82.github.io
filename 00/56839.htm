This is usually heat related, defective hardware, memory or even processor though it is "possible"

There is no technical reason what-so-ever for the machine to be physically dependent upon Microsoft in any way for the signing if anything, or even the OEM for that matter.
Asymmetric cryptography is extremely well understood, and any cryptographer will tell you that this concoction is entirely artificial.
All UEFI needs to do is allow the user to go into the UEFI boot screen, and specify, by typing the keys into a table, which public keys are acceptable public keys for that computer. That's it.
Microsoft would have the public-key that corresponds to its "Windows-boot-software-blob" stored in the EUFI public-key table. The OEM would put the key there on behalf of Microsoft.
But the table might have, say, 64-entries, for 64 different possible OS's [Yes, I know that's a lot]. In that case, the second entry would be for Linux, and the end user would manually enter the public-key for "Linux-boot-software-blob" into the second slot on the table.
For 2048-bit RSA modulus, the user would have to type 512 hexadecimal characters. Yes, that is a lot of letters to type, but there are three considerations:
1. The OEM could put three entries in the table: one for Microsoft, one for "Linux", and one for iOS

Hi there, had the same problem, no boot options in bios for built-in DVD player, but solved it through 'system recovery'.

My computer is not in english so I guessed the following option names. 

In the 'pc-settings', there's an option for 'system recovery', where you can restart the pc for troubleshooting. Doing this opens a startup menu.
