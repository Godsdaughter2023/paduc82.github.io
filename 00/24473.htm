Hi, Inmnmnen,

There is no specific maximum number of contacts because the actual limit is measured in kilobytes, not number of contacts. So if your contacts only consist of very short email addresses and names, you might be able to create distribution lists with more than 120 addresses. On the other hand, if your contacts contain large amounts of information, you might only be able to add 20 or so before you receive the error  "This distribution list has reached the maximum size for your network e-mail server". The 60 to 120 figure is simply a practical average that has been observed over the years. - Source

FYI: "Distribution Lists" are now called "Contact Groups".

doug robbins - mvp office apps & services (word) say:

Actually what you are taking as 1 is -1.  From that you can see that the left margin is set to 1" and the zero position on the ruler corresponds to the beginning of the text area.

The markings on the ruler are -1, 0, 1, 2, 3 etc.

There is no issue to be fixed.

If you want the zero mark of the ruler to align with the left hand edge of the page, set the margin to zero and define the style with an indent before text equal to the margin that you want and put up with Word complaining about the margin being too small.

Content Control within Header unrestricted editing - Microsoft Community

rick113 say:

Content Control within Header unrestricted editing

I have a report and placed the person's info in the header using RTF Content controls in Word 2013. When I turn on the Editing Restrictions and select Filling in Forms, I cannot access the content control fields within the header. 

***Post moved by the moderator to the appropriate forum category***

charles kenyon say:

That is because Filling in Forms protection does not give access to headers or footers. You can use "No Changes" with exceptions instead.

Consider placing your person's info on the title page, instead. Then you can use the StyleRef Field to reflect that information in the Header if you wish.

A somewhat more subtle way to do it is with mapped content controls.

Repeating Data Using Document Property Content Controls and Other Mapped Content Controls

rick113 say:

rick113 say:

I've read before the StyleRef Field before, but didn't really get it. If I add the fields that belong in the header to the body of the report, it changes the standardized look to the report.

Since the the first page of the header is slightly different from the next pages, I created a different section from the first page and mapped the controls within the headers.

charles kenyon say:

You have to mark items you want access to as exceptions. This can be done using grouping or individually or with a big selection of your document. The StyleRef field or mapped controls are probably easier.

If you are using Content Controls, there is no reason to apply protection to the document.  To prevent changes to the layout, using the Group facility in the Controls section of the Developer tab of the ribbon instead.
