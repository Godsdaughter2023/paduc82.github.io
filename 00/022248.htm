OR how to tell if the document created date properties matches the Word file?

macropod say:

Word's CREATEDATE field shows either: the date the document was created; or the date the document was last saved via Save As, whichever is the later (obviously). Therefore, Word cannot tell you directly whether the CREATEDATE field shows the actual date the document was created.

A giveaway that the CREATEDATE field isn't showing the actual date the document was created is when the PRINTDATE field returns a date earlier than the CREATEDATE. That won't work, though, if the document has never been printed or was saved via Save As before it was last printed, then saved normally.

If a proper backup regime is in place, a far more reliable approach is to go through the backups to identify when the file first appeared there.

hansv mvp say:

The NORM.S.INV function returns the inverse of the cumulative standard normal distribution.


The area below the entire curve from -infinity to +infinity is by definition 1. The NORM.S.INV function returns the z for which the area from -infinity to that z equals the input value. The z corresponding to an area of 0.95 is 1.64...

You were probably thinking of a two-sided picture:



But that's not what NORM.S.INV does...
