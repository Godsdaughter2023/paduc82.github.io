You'll need a macro to be able to specify the browser.

The following macro will open the MS support page in Internet Explorer, even if the default browser is something else:

Sub LaunchInIE()
  Shell _

    PathName:="""c:\program files\Internet Explorer\iexplore.exe"" http://support.microsoft.com", _
    WindowStyle:=vbNormalFocus
End Sub

(The seemingly extra double quotes are needed because the path to IE contains spaces.)

This macro can be invoked from a MacroButton field or from a button you place on the Quick Access Toolbar.

If there are multiple pages that need to be launched in the same browser, a slightly more complicated arrangement will be possible.

Word Document repaginating

jacques raubenheimer say:

This is actually pretty normal behaviour for Word. Remember that Word is not a page layout program, so it keeps "making up" the pages as you go along.

As you noticed, you can continue editing, and it normally is not disruptive.

So the big issue is to figure out why it is so slow on your pc.

It could be indicative of a very low-specification pc, but somehow I doubt that that is your problem. Or that you have large amounts of images in the document.

So possible fixes could be:

Switch to Normal (now called Draft) view--Ctrl+Alt+N. The repagination only takes place in Page Layout view, so in Normal view, you don't have that problem.

However, Normal view does not show you how your text will be located on the page, and it does not show images, which could be a problem for you.

The other thing I would try is tweaking the default printer on your pc. Since Word repaginates according to the currently selected printer, it could be a problem with the printer driver that is causing this slow behaviour. Switching to another printer driver could solve the problem.

So that is not a certain answer, but it is at least a handful of things you could look at.

I would actually start by looking at the printer driver, and then try some of the other alternatives if that doesn't fix it.
