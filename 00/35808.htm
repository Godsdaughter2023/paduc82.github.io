The cpu could well be a problem. @2.4 vs 2.6GHz minimum game requirement, you are cutting things close.

[EDIT: Is his Windows up to date ? I just found the Service Host: Web Account Manager is a feature introduced with the 1709 update which was causing some conflicts early on, but a subsequent cumulative update should have sorted it out.

You  could verify if the Service is the real issue by disabling it temporarily. Disabling is not a long term fix, as this Service is required

by Windows to manage any MS account (Store, etc.) Dad may use.

- Type -  services.msc  -  into the Run command or Search. In the Services Window scroll down to Web Account Manager.

Double click on Web Account Manager and in the General tab select Disabled from the Startup Type.

Reboot may be required after disabling this Service.   

You could also try Stopping the Service when launching the game.

- The default settings for this Service should be 'Running' and 'Manual'.    ]

- Thing is... Sandy Bridge Intel cpu's are getting on (2011), and if the machine is using an old integrated graphics processor it would

no longer be fully supported (Windows generic drivers only), so gaming graphics is likely going to start causing more

issues as Win10 'progresses' via the bi-annual version updates.

If an i5 were to fall into your lap cheap it may be worth the upgrade. More so if the machine already has a decent dedicated graphics card.
