Hi Tim,

 

You can’t get the view count like you get in a document library but there’s an alternate way to achieve this requirement. To get the view count in a list, you have to first enable a feature from site collection features and then you can get the count for most popular items in a list.

 

To do this, follow the steps below:

 

  1. Open SharePoint Site, click settings (gear icon at the top right corner) >Site Settings.
  2. Under ‘Site Collection Administration’, click Site collection features.
  3. Enable ‘Cross-Site Collection Publishing’

 

After enabling this feature-

 

  1. On your site, go to the relevant list.
  2. Click the List tab.
  3. On the ribbon, click Most Popular Items. 

As a result, you will get a list of items as shown below-

 

 

For reference: View Popularity Trends and Most Popular Items

 

Note: Steps mentioned in the article are for document library but can be applied to list as well.

You can try this registry hack to see if it fixes the problem. As always, back up the registry first if you're not sure what you're doing:

  1. Search for Regedit.exe, then open it.
  2. If User Account Control asks for an OK, choose Continue or provide an admin login.
  3. Look for HKEY_CURRENT_USER>Software>Microsoft>Windows>DWM.

  5. Set the Composition value to 0.
  6. Restart your computer, then see if PowerPoint behaves any better. If not, reset the value of Composition back to 1.

bob jones aka: cybertaz say:
Unless it can be done programmatically I don't see any way to accomplish this as a part of the Merge process. I still have no idea what the nature of the work is so I have no grasp of why the Labels approach needs to be used, but for something of this nature I believe I'd be more inclined to choose Catalog for the Main Document instead. I'd use a Table with one Row & set the row properties for no specified height & Allow to break, then do the merge.

If you're dealing with the end result of a document generated by a Label merge as a Table, though, you might see if this will help. It will still be cumbersome because of the Label layout which forces Section Breaks that can't be deleted easily, so you actually have multiple tables in the document (one per page). That means you may have to do it for each involved table because you can't modify Table Properties simultaneously for multiple tables:

  1. Select the Table
  2. Go to Table> Table Properties - Row
  3. Clear the check on Specify height
  4. Check the box for Allow row to break...

The height of the high volume rows will expand accordingly flowing onto the next page as necessary. The height of other rows will reduced to what is actually required by the record on that row. The down side is that you still have the Section Breaks to deal with because each page [i.e., Section] will be restricted to no more than 3 records/rows due to the specs of the Label template.

NOTE: If you were to use the Catalog merge described above rather than (3x5) Labels this would all happen automatically.
