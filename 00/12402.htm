scottgem say:

Ok first, I'm not clear if you are talking about sorting or filtering. You say sorting but it appears more that you want to filter. Sorting displays all your records in a specific order. Filtering restricts what records to display. 

In either case, is this an app your are building for someone else or just for your own use? Our suggestions will vary depending on the answer to that question. If this is only for you own use, you can use the datasheet filtering options. When you can create a form in datasheet mode. The ribbon will have options for sorting. The only caveat is that you can only sort left to right. So the columns that you want to sort by have to be in order, left to right. Datasheet view also gives you filtering options. In the heading for each column there is a pull down arrow. When you pull it down you get options that you can use to filter by that column. Filtering is cumulative so if you filter one column then go to another column it will filter within the the already filtered records. Allowing you to drill down. 

If you are building an app for others to use, this may help: http://www.allenbrowne.com/ser-62.html

One more point based on these statements: "I'm fairly new to Access...All of the information I need is on a single table...I have an Excel example "  Access is not Excel and modeling data like an Excel spreadsheet is generally a mistake. It is rare that a single table represents a proper design for an Access database.

Word's wildcard implementation has never had a full RegEx capacity - or purported to do so. Amongst other things, that has meant a lack of an 'or' test and a 'zero or one' variable.
