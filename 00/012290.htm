I have an excel sheet with data, and a check box in Column in E for each row
when I click a button all checked items to move to a sheet called check and unchecked items to a sheet called uncheck.
andreas killer say:
I already gave you some examples in the past in other threads how to copy a range, so I skip this part here. Also the question "How do I copy cells from one sheet to another" is a FAQ in this forum, you can find several examples if you use the search engine of this forum.

I assume that your checkbox is an ActiveX control, you can find these controls in the OLEobjects collection of every sheet.

The ProgID property lets you know which kind of control you've found the TopLeftCell and BottomRight property returns a range of the location. The Object property allows you to access the control itself.

Andreas.

Sub Test()
  Dim OO As OLEObject
  Dim R As Range
  For Each OO In ActiveSheet.OLEObjects
    Select Case OO.progID
      Case "Forms.CheckBox.1"
        If OO.Object.Value Then
          If R Is Nothing Then
            Set R = OO.TopLeftCell
          Else
            Set R = Union(R, OO.TopLeftCell)
          End If
        End If
    End Select
  Next
  If R Is Nothing Then
    MsgBox "No checkboxes are checked"
  Else
    MsgBox "Checkboxes in " & R.Address(0, 0) & " are checked"
  End If
End Sub

The Outlook Team has a bug for this and they have fixed it for the next update.  If you need to workaround it for now use the steps below to revert to an earlier build that does not have the issue.

1.  Click File, Office Account, Update Options and click Disable Updates

2.  Put a reminder on your calendar for 7/5 to remember to re-enable updates

3.  Open a Command Prompt 

4.  Enter the commands below one after the other

cd %programfiles%\Common Files\Microsoft Shared\ClickToRun

officec2rclient.exe /update user updatetoversion=16.0.8201.2075

or

officec2rclient.exe /update user updatetoversion=16.0.8201.2102

In my testing 16.0.8201.2102 worked for reverting Insider Slow and 16.0.8201.2075 worked for Insider Fast.

After you run the second command the revert will take a few minutes and will need you to close the Office Apps to finish.  After it finishes you will be on the older build that should not have the issue.
