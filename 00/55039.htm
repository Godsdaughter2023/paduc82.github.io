steve winograd say: Are the IP address and subnet mask 169.254.x.x and 255.255.0.0.?

If so, that indicates that:

1. The network connection is configured to obtain an IP address automatically.

and:

2. It can't communicate with a DHCP server to get an IP addrdess.

In that case, Windows assigns itself an APIPA (Automatic Private IP Address) in the 169.254.x.x range.

The problem could that the computer's network adapter is defective.  It could be a bad cable, switch port, or router port.  If you have a home router, something could be wrong with it.  The router's DHCP server configuration could be bad.

jasmin001 say:

How to repair the source for DISM when restorehealth fails.

The Windows 10 system files system for my desktop had become corrupted so I tried system file checker (sfc /scannow) in administrator cmd.exe

 Right click  Command Prompt (Admin).   Command Prompt (Admin). 

sfc /scannow – had previously been able to repair file corruption (see below for successful result)

but could no longer repair the corruption as it was extensive. (above from answers.microsoft.com and is not build 10.0.10568.0)

The following link

http://www.tenforums.com/tutorials/7808-dism-repair-windows-10-image.html  suggested that when a “Windows image becomes unserviceable, you can use the Deployment Imaging and Servicing Management (DISM) tool to update the files and correct the problem”.

Tenforums suggests using the following DISM statements

Dism /Online /Cleanup-Image /CheckHealth

Dism /Online /Cleanup-Image /ScanHealth

Dism /Online /Cleanup-Image /RestoreHealth

(Note that /Online means DISM targets the currently running operating system as compared to offline image held in a folder or Wim file).  However, these three commands did not work as the source files could not be found (see below).

(above image from superuser.com again not build 10.0.10568.0)

The above shows the feedback from a failed attempt to restorehealth.  (It should be noted that this is not my image version).  Error 0x800f081f means that a file needed by Windows Update is damaged or missing.  means that a file needed by Windows Update is damaged or missing. 

As restorehealth failed another source of the files are needed for the repair.  I then downloaded Win10 (version 10.0.10568.0 ) from technet.com. 

http://www.microsoft.com/en-us/software-download/windows10 

The downloaded file is an .iso image and as this is on a Windows 10 system that image can be easily mounted.  Right-click image and click mount.  See below the DVD Drive F.  

The files associated with Drive F are:

Open the file sources and scroll to find the Install.esd file copy this file and place in another folder.  I had read that the ESD file could be used as a source so I tried the following:

dism /online /cleanup-image /restorehealth /Source:ESD:F:\sources \Install.esd:1 / limitaccess where F is the drive letter. 

The /limitaccess is used to stop DISM using Windows updates as a source for the files.  Unfortunately, this did not work so I decided to convert the Install.esd to an Install.wim.  To convert the files several third parties software was tested.  These included ESDtoISO, Pismo File Mount Audit Package and WinReducerESWimConverter.  Only the WinReducerESWimConverter from WinReducer.Net worked easily and converted the Install.esd to an Install.wim.  http://www.winreducer.netInitially  http://www.winreducer.netInitially (see below) the three options are off and need to be clicked on and the install.esd is dragged to be in same folder.

Once the Wim file was available I made a copy and pasted it in the C drive. 

Then I used:

Dism /online /cleanup-image /restorehealth /Source:WIM:C:\Install.wim:1 /limitaccess

The component store corruption was repaired.  To check the sfc /scannow was run and it was able to

repair the system files.
