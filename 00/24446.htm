joeu2004 say:

Mr.Low wrote:

I am currently leaning how to use = XOR ( ) formula on Excel worksheet.  I find that it always returns with #NAME? error instead of TRUE or FALSE.  May I know if a Macro code need to be inserted to the module for this function to work ?

XOR(A,B) is the same as A<>B, where A and B are boolean expressions that return TRUE or FALSE.

More generally, XOR(A,B,C,...) is the same as ISODD(COUNT(A,B,C,...)).  Again, A, B and C are boolean expressions that return TRUE or FALSE.

(But beware:  those equivalences do not behave like Excel XOR if, for example, A is a reference to a range logical values, e.g. A1:A10.)

XOR(3>0,2<9) is the same as (3>0)<>(2<9).  They should return FALSE because both (3>0) and (2<9) are TRUE.

I do not have Excel 2013 to verify, but I believe the example in the online help page for Excel XOR (click here) [1] is incorrect.  It contradicts the (correct) explanation of the function behavior.  To wit....

Correct:  ``The result of XOR is TRUE when the number of TRUE inputs is odd and FALSE when the number of TRUE inputs is even.``

Incorrect:  ``=XOR(3>0,2<9)    Because one [sic!!] of the two tests evaluates to True, TRUE is returned.``  Since
