Based on the error message you've provided when signing on to Windows Live Mail can be broken down to 2 causes:

  1. Incorrect POP settings (usually the SSL option for SMTP) – check with provider if it is required or not
  2. Firewall blocking ports (usually port 25) – could be 3rd party since Windows Firewall generally allows this port through, unless specifically told not to do so

Please make sure that you are using the correct POP and SMTP settings for your account.

If you are using Hotmail/Outlook.com accounts, please visit this article for reference.

 

If not, please contact your domain administrator on how to.

 

Firewall or security software may be blocking Windows Live Mail or may be blocking a required port to send or receive mail

If your computer has any firewall programs, such as Norton, PC-cillin, MacAfee, Windows Live OneCare, Zone Alarm etc, Windows Live Mail or a port used to send or receive email may be blocked, preventing you from sending or receiving email.

The steps below are specific to Windows Firewall. If you are not using Windows Firewall, you will need to check the documentation or contact support for the software you are using.

  1. Disable Windows Firewall:
    An easy way to determine if your Firewall/security software is preventing you from sending or receiving email is to temporarily disable the software.

· Disable Windows Firewall in Windows XP

· Disable Windows Firewall in Windows Vista

· Disable Windows Firewall in Windows 7

· Disable Windows Firewall in Windows 8 and 8.1

  1. After disabling your firewall/security software, attempt to send and receive your email.Be sure to re-enable the firewall/security software after testing.
  2. If temporarily disabling Windows Firewall allowed you to send and/or receive through Windows Live Mail you may not be allowed Windows Firewall. Complete the steps provided in the following links to allow Windows Live Mail in Windows firewall.

· Allow Windows Live Mail in Windows Firewall in Windows XP

· Allow Windows Live Mail in Windows Firewall in Windows Vista

· Allow Windows Live Mail in Windows Firewall in Windows 7

· Allow Windows Live Mail in Windows Firewall in Windows 8 and 8.1

  1. If temporarily disabling Windows Firewall allowed Windows Live Mail to send and/or receive email, but Windows Live Mail is already enabled or enabling Windows Live Mail does not allow you to send/receive email you may need to open ports required by your email provider. You can determine the ports configured for your email in the account properties within Windows Live Mail. Follow the steps below to open an email ports in Windows Firewall:

Windows Live Mail 2009

· From the Menu Bar, click Tools

· Click Accounts, then select Mail Accounts, and then click Properties.

· The outgoing and incoming email server ports are listed in the Advanced tab.


Windows Live Mail 2011

· Select the account in the left navigation menu select your email account.

· Select the Accounts tab.

· Click on the Properties button in the Accounts ribbon.

· The outgoing and incoming email server ports are listed in the Advanced tab.

  1. After determining the required ports, follow the steps in the following links to enable the ports in Windows Firewall.

· Enable ports in Windows XP

· Enable Ports in Windows Vista

· Enable Ports in Windows 7

· Enable Ports in Windows 8 and 8.1

Hello Kala,

Thank you for posting on Microsoft Community. We regret the inconvenience caused and will assist you in resolving the issue.

Before assisting further, I would like to ask you few questions:

  1. Do you get any error message/error code when the action times out?
  2. Are you able to use the network connection for browsing online?

You may be facing the above issue due to some of the system components conflicting with the network connection on the PC. Running the troubleshooter will help in identifying and sometimes will fix the issue.

Method 1: Run the HomeGroup troubleshooter by following the steps below and check if you are able to connect to the network.

  1. Press Windows logo key on the keyboard, type Troubleshooting and select the top most search result.

  2. Click on View all from left side panel of the screen.

  3. Select HomeGroup, click on Advanced option and check Apply repairs automatically option.

  4. Follow the on screen instructions.

  5. After the troubleshooting is complete, check for the status.

  6. If the status shows no errors, close the window and check if the issue is resolved.

Method 2: If the issue persists even after running the troubleshooter, change the HomeGroup settings by following the steps below and check if the issue is resolved.

  1. Press Windows logo key on the keyboard, type Control Panel and select the top most search result.

  2. Click on View by located to the top right of the window and select Large option.

  3. Click on Network & Sharing center and select the Change advanced sharing settings from the left side panel of the window.

  4. Under Private network and Public network, check the boxes Turn on network discovery, Turn on file and printer sharing and Allow Windows to manage homegroup connections (recommended).

  5. Click on Save changes and close the window.

  6. Check if the changes are effective.
