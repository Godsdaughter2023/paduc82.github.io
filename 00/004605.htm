jay freedman say:
As Graham tried to explain, Word's Font dialog will report any font name that anyone types into the font box, whether that font exists or not. For example, I can type asdfg into the font box, and Word will tell me that a font named Asdfg is applied to the text. If I then press Enter in normal text or add a new row to a table where Asdfg is applied, then the new paragraph or row will continue to report that its font is Asdfg. It's nonsense, of course, but Word accepts it because you might later open the document on a computer that actually has a font named Asdfg.

But if I now go to Office button > Word Options > Advanced, scroll down to the "Show document content" section, and click the Font Substitution button, that dialog shows that the font Asdfg is missing, and Word has substituted Times New Roman for it. (There are some arcane rules for how Word picks which font to use as the substitute, but Times New Roman is common.)

Times is a font that's sometimes provided by a printer's read-only memory or packaged with the printer's drivers. The presence of both Times and New York fonts, though, indicates that the document may have originated on a Mac. In either case, those fonts don't normally exist in a Windows PC.  Times New Roman is equivalent to Times, and New York is somewhat similar (http://www.identifont.com/show?3IE).

Older versions of Word on Windows did not use Times or New York. Word on pre-OS9 Macs did (but current versions no longer use New York, which was a bitmapped font).

So the best advice is to use the Convert Permanently button in the Font Substitution dialog to change all of the text that "thinks" it's Times and New York to really use Times New Roman.
