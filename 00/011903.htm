The first is using Styles. Styles control the way text looks in your document. The trick is to apply styles rather than applying formatting directly, use Heading styles, use Strong or Emphasis instead of applying bold or italics.

So, for discussion lets assume you are going to start with then Manuscript Template.

Do you want a table of contents? Most books do. You can generate, and regenerate, one automatically, using Heading styles. In the home tab, look at the Styles gallery. There are heading styles. If you want the chapter name to appear more distinctive, larger font size, different font, bold, whatever, you would "Modify" the style itself. One of the problems with the template is that it uses Heading styles 1 and 2 for the title, and starts the chapters at heading 2. The problem with this approach is that the 3 lines of the title, show up as separate lines in the TOC (duh!). That is easy enough to modify the styles to fix.

For paragraphs, you should decide how you want to start paragraphs. If you want to use indents, then you can change the style to automatically do the indent for you, rather than using tabs, like they do (a left over from typewriters).

I noticed the same issue with my free memory slowy creeping down with use. As aspergerguy said, a factory reset will free up quite a lot of memory, at the loss of any customisation you may have done.

 

One thing to check is your message inbox and sent items. The amount of memory the phone reports as being used by messages is (in my experience) wrong. Backup all your sms messages and any recieved bluetooth files to ovi and delete them off the device, you may be shocked how much it frees up. I wiped the message history on my fiancé's phone last week, about 1400 messages in all including a few photos sent via bluetooth, and it freed up about 10mb!

 

I know you said you'd shifted the message repository from the internal to sd memory, but i'm not sure if that moves existing messages or just stores new ones there.

 

If you do a full phone backup and restore before and after the reset through ovi suite you may still find you have more memory  available after the restore. But if a previous hard reset didn't help, you might just have to put up with losing customisation and only restore contacts.

 

Another thing i did (don't know how necessary it is) was to rescale the images i used for contact pictures, since i didn't need a 2mp image for a tiny little thumbnail, i don't know if the handset makes a scaled copy or if it just scales the image on the screen, but it might help, especialy if you have a lot of contact images.

 

The other thing to look at is (as mentioned) the firmware version. V50 is the current version, but if you have a branded phone and your operator hasn't released their custom version, it's still worth checking if you can update to v40 because that uses about 10mb less than v20

Beware though, if you are on orange in the uk, and use the contacts bar it's broken in v40 orange.

dale howard [mvp] say:

Daryl --

 

What you are seeing is new functionality in Project Server 2013.  By the way, in the future, please mention that you are using Project Server 2013 in your questions about Microsoft Project.  When you go to open a project in Microsoft Project 2013 by clicking File > Open > Project Server > Browse, you will discover that the software preselects the Read Only option in the Open dialog.  If you want to open it Read/Write, you must select that option before you click the Open button.  Because Read Only is the default option, this means that if you open an enterprise project from the Local Cache, the software will automatically open it Read Only, not Read/Write.

 

An alternative to this would be to use the Project Center interface in PWA to open your enterprise projects Read-Write.  To do this, navigate to the Project Center page in PWA and then click the white row reader at the left end of the row for the project you want to open.  When you click the white row header, this selects the project for further action.  In the Project Section of the ribbon, click the Open pick list button and then choose the In Microsoft Project for Editing option.  The system will open the project Read/Write automatically.

 

Hope this helps.
