jay freedman say:
The standard way of using a protected form is that the first form field is selected when the document opens (because the selection can't be anywhere outside the form fields), and the user tabs from one field to the next. When you tab into a dropdown field, the button appears -- it's relatively rare to click into one with the mouse.

The button isn't shown all the time because you don't (usually) want the button to appear in a printout.

The "combo box" is called that because it's a combination of a dropdown list box and a text box. That is, you can make a selection from the dropdown list, or you can type an entry in the text box part that doesn't appear in the list. There's an option in the form designer to require the text to match one of the list entries, in which case the combo box functions the same as a dropdown list box. In that case, you don't generally see both a combo box and a dropdown list box offered in the form designer.

In a userform, you can insert a list box that isn't a dropdown but a permanently displayed large box.

macropod say:

You might add code like the following to the 'ThisDocument' code module of the document's template. The code will then run automatically when any document based on that template is opened. When the code is runn, a document variable named "TableShading" is added to the document. If the shading has already been done successfully by the code, the code will simply exit.

Private Sub Document_Open()
Application.ScreenUpdating = False
Dim Tbl As Table, r As Long, StrTxt As String
With ActiveDocument
  On Error Resume Next
  .Variables.Add "TableShading", 0
  On Error GoTo 0
  If .Variables("TableShading").Value = 1 Then GoTo ErrExit
  For Each Tbl In Tables
    With Tbl
      For r = 1 To .Rows.Count
        With .Cell(r, 3)
          Select Case Split(.Range.Text, vbCr)(0)
            Case "One": .Shading.BackgroundPatternColor = wdColorRed
            Case "Two": .Shading.BackgroundPatternColor = wdColorOrange
            Case "Three": .Shading.BackgroundPatternColor = wdColorYellow
            Case "Four": .Shading.BackgroundPatternColor = wdColorGreen
            Case "Five": .Shading.BackgroundPatternColor = wdColorBlue
          End Select
        End With
      Next
    End With
  Next
  .Variables("TableShading").Value = 1
End With
ErrExit:
Application.ScreenUpdating = True
End Sub

Simply change the One to Five to whatever the cell contents are.

macropod say:

Your original post implied only cells in column 4 had been merged. I see now there are other merged cells as well. The following rendition of the macro allows for that and adjusts the image sizes in column 4 as well. You might want to play with the InchesToPoints values to get exactly the results you want.

Note: Your template appears to be corrupt. I couldn't get the shading to take effect until I copied the tables from it into a new one and ran the macro from there.

Private Sub Document_Open()
Application.ScreenUpdating = False
Dim Tbl As Table, c As Long, StrTxt As String
With ActiveDocument
  On Error Resume Next
  .Variables.Add "TableShading", 0
  On Error GoTo 0
  If .Variables("TableShading").Value = 1 Then GoTo ErrExit
  For Each Tbl In Tables
    With Tbl.Range
      For c = 1 To .Cells.Count
        If .Cells(c).ColumnIndex = 3 Then
          With .Cells(c)
            Select Case Split(.Range.Text, vbCr)(0)
              Case "One": .Shading.BackgroundPatternColor = wdColorRed
              Case "Two": .Shading.BackgroundPatternColor = wdColorOrange
              Case "Three": .Shading.BackgroundPatternColor = wdColorYellow
              Case "Four": .Shading.BackgroundPatternColor = wdColorGreen
              Case "Five": .Shading.BackgroundPatternColor = wdColorBlue
            End Select
          End With
        End If
        If .Cells(c).ColumnIndex = 4 Then
          With .Cells(c).Range
            If .InlineShapes.Count = 1 Then
              With .InlineShapes(1)
                .LockAspectRatio = True
                .Height = InchesToPoints(1.8)
                If .Width > InchesToPoints(2.5) Then .Width = InchesToPoints(2.5)
              End With
            End If
          End With
        End If
      Next
    End With
  Next
  .Variables("TableShading").Value = 1
End With
ErrExit:
Application.ScreenUpdating = True
End Sub

jay freedman say:
When you close an unsaved document in Word 2013 (also in 2010), it does store a copy that can be recovered -- but it only keeps that copy for four days or until you start Word again, whichever is longer. Then Word deletes the copy so your drive won't fill up with useless files.

It's rare but sometimes possible to recover one of Word's temp files that contains some or all of the text from a document -- that's probably what the tech got for you.

The last hope is to use an "undelete" program to try to find the copy that was automatically saved and then deleted. It would have been a file with the .asd extension, in the folder C:\Users\<your name>\AppData\Local\Microsoft\Office\UnsavedFiles.

To delete files that have been checked out by another user, you can follow the steps below.

MS Word when Document Open clear text in bookmarks from UserForm

I have a template created in MS Word 2013. 

When I open the document I want the following to happen:

- clear text in a bookmark called "Check1" (but keep the bookmark)

- open UserForm2

I've been trying various suggestions I have found, but nothing seems to be working.

Additional question:  does the code go under "Module" or is it code specific to "This Document"?

My apologies - I'm quite new to this

Thank you in advance.

jay freedman say:

If you're really saving your form as a template (*.dotx or *.dotm) file, and users create new forms by choosing the template in the File > New page or by double-clicking the template in Windows Explorer, then the fields in the userform will automatically be empty and the bookmarks will be present (but empty, if that's the way you save the template).

On the other hand, if you're using a document (*.docx or *.docm) as the basis for new documents, that's a bad idea. Not only will the new document open with the field contents of the old document, but you run a strong risk that the user will Save (Ctrl+S) rather than Save As, and that will overwrite the old document.

To open a userform when a new document is created from a template, you have two choices:

  • In the ThisDocument module of the template, create a Document_New() procedure: click the left dropdown above the code area (which defaults to "(General)" and choose Document. You'll get the first and last lines of the procedure, with a blank line between them to insert code.
  • Or, in a (new or existing) module in the template, write a macro named AutoNew().

In either case, the minimum code to put inside the procedure is:

    Dim dlg As UserForm2
    Set dlg = New UserForm2
    dlg.Show
    Set dlg = Nothing

Optionally, you can set the values of members of the UserForm2 object before the .Show call, and/or read the values of members after the .Show call but before setting dlg = Nothing (which removes the userform from memory).

 

In order to save a template that contains a userform, you must choose the .dotm (macro-enabled) format.

Neeed to make UEFI bootable USB key for Win 10 installation - Microsoft Community

oherb40 say:

Neeed to make UEFI bootable USB key for Win 10 installation

I followed the link to instructions to creating a bootable USB key to do a clean install of a Win 10 recent build. The key successfully booted my 3-year-old HP laptop but not my 2-year-old Razer Edge tablet, even after I set the BIOS to give the USB key top boot priority. Razer tech support tells me the tablet requires a UEFI bootable USB key. Can you point me to a source for instructions on how to do this?
andre da costa say:

If your computer is UEFI based, these are normally systems that come pre-loaded with Windows 8 or later, you will need to prepare the ISO file for such a configuration or you will receive an error message during setup. The thumbdrive needs to be formatted as FAT32 and use the GPT partitioning scheme. To do this, you need to use Rufus, a small tool you can download for free.

 

After you have installed Rufus:

  • Launch it
  • Select ISO Image
  • Point to the Windows 10 ISO file
  • Check off Create a bootable disk using
  • Select GPT partitioning for EUFI firmware as the Partition scheme
  • Choose FAT32 NOT NTFS as the File system
  • Make sure your USB thumbdrive in the Device list box
  • Click Start
  • Close when complete
