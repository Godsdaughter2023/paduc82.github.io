don cuthbert say:

The simplest is to delete and recreate that partition. Then using a bootsect command and bcdboot command should rebuild it.  If your Windows partition is not right, this may not fix everything, but your System partition should then be good.

Run diskpart:

list volume <-- ensure that volume letters are as you said

select volume 1 <-- or whatever volume number is the FAT32 System one

delete partition override <--- deletes the partition for volume 1

create partition efi <--- recreates the proper partition type for System files

format quick fs=fat32 label=SYSTEM <-- quick formats the new partition

assign letter=S <--- gives it a drive letter for use with bootsect and bcdboot

list partition <--- shows the partition list and drive letters

exit

bootsect /nt60 S: /force  <--- places the System files on the FAT32 partition (letter S:)

bcdboot C:\Windows /s S: /f UEFI <--- creates the proper BCD to point to the Boot/Windows partition (this assumes that Windows is on C: -- if it is on another letter use that letter).

bcdedit <--- shows the main parts of the new BCD and should indicate drives C: and S:

Now try restarting and see if Windows 10 loads properly.  If everything looks good, you can also get a Command Prompt (Admin) and run sfc /scannow to ensure that all Windows files are OK.

Windows cannot control where a software decides to place it's 'extras', only where you

choose to install the main program. This is nothing new, or exclusive to Win10.

Many programs and games have folders\sub-folders somewhere in C:\ .

a corrupt driver install vs reinstalling.
