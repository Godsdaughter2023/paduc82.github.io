Hi,

I managed to tweak* a function that I found here: http://www.excelforum.com/excel-programming-vba-macros/472843-solved-arabic-characters-gives-ascii-code-63-a.html

so it will fix your data.

Press Alt+F11 to open the VBE, then do Insert > Module.

Paste the code below in the space on the right of the VBE:

Function FixIt(rng As Range) As String
    Dim i As Long, lnUnicodeChar As Long
    Dim strCellValue As String, strNewValue As String
   
    strCellValue = rng.Value
    For i = 1 To Len(strCellValue)
        lnUnicodeChar = AscW(Mid$(strCellValue, i, 1))
        strNewValue = strNewValue & Chr(Asc(ChrW(lnUnicodeChar)))
    Next i
    FixIt = strNewValue
End Function

Close the VBE.

In Excel, if your data is in A1, in another cell enter the formula     =FixIt(A1)

Drag that cell down and across and then you can then copy and paste special as values.

The issue is because you have some foreign (Arabic?) text which is not recognised properly by Excel.  If you test any of your original characters with the =CODE() function it returned 63!

Rich
