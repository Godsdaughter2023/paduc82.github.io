Backing up data on Windows Phone 8.1 + problems with backing up - Microsoft Community

tombrad95 say:

Backing up data on Windows Phone 8.1 + problems with backing up

With the introduction of saving your browser favourites, start screen layout, passwords and app data, the backups from before you updated to the 8.1 update won't have all of these saved, on the OneDrive website, in the options under device settings, it will tell you the last time your device was backed up but this date will be before you updated your phone, even though the backup is turned on it won't automatically backup daily, therefore, I recommend that you go to settings -> backup -> apps and settings and force a backup. Also to save time I suggest that you do this sooner rather than later before the app data gets higher.

Problems with backing up

I have read several issues with the backing up of phones since 8.1, these backups usually stop at 96-98% before the backup fails.

If this problem has arisen on your phone I'd suggest following the steps below,

First try backing up your phone while having it plugged into a constant power source, and stop the screen from going off, this can be done by changing the settings under lock screen then "screen times out after"

If this doesn't work then I'd suggest removing the previous backup and performing another one with the steps above, to remove the last backup go to backup in the settings and press apps+settings, then manage backups, then make sure that both backup options are ticked and press delete. It would be good to check to see if you have enough storage on your OneDrive to handle the backup, maybe if you've installed many apps between the backups there might not be enough storage on the account to do this and might require that you add more.

Thank you.

How do I change table direction (not text direction) in PowerPoint from left-to-right to right-to-left?

I have seen posts about changing 'text direction' in tables by clicking the paragraph text direction button. This does NOT solve the problem.

If I am translating a table that contains Latin text (left-to-right) in Word into a language that goes right-to-left, instead of re-creating the whole table, with all its varying width columns/varying colours in the target language, one can simply:

  • Click on the little '+' at the top-left corner of the table
  • right click the '+'
  • choose table properties
  • click right-to-left under 'table direction'.

This will 'flip' the whole table into right-to-left so that it is a 'mirror image' of the original table.

Is there a similar function in PowerPoint? I had, for now, to copy the table from PowerPoint into Word, flip it, type the translation then copy it back into PowerPoint.

Any help would be appreciated.

Thanks,

Manal

manal_nakli say:

Thank you, John, for the reply. I kept trying and now found out how to do it in PowerPoint. Here is how, for future reference:

  • click on the table in PowerPoint
  •  under 'Table Tools' select the 'Layout' tab
  • in 'Alignment' box click both of the following:
  1. the 'right-to-left' icon [when hovering the mouse it reads 'Set right-to-left table']
  2. the 'right-to-left' alignment icon [when hovering the mouse it reads 'Align text Right (Ctrl+R)']

This will produce a mirror image of the table, which can then be used to type in the text that reads right-to-left (Arabic, Hebrew, Farsi, Urdu,... etc.)

The same steps can be applied to flip a table from r-t-l to l-t-r by clicking the relevant buttons.

I hope this helps future enquirers.

I'd be willing to bet that the problem is that Cox doesn't let you send outbound messages from somebody else's network.  It's a common anti-spam technique - they only accept SMTP traffic from machines connected to their own network.  That means when you travel, and suddenly aren't connecting from a Cox-issued IP address, you can't send.

There may be a way to still use Cox when you're travelling.  What port do you have SMTP set to in Outlook?  (Account Settings | Advanced)  Is it 25?  or 465?

The other workaround is to get an account with a free provider (like GMail) who doesn't care what network you're on and, at least when you're travelling, send your outgoing mail through that server while still receiving on your normal address.
