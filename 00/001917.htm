umaitin say:
The host name that the on-premises organization will use to connect to Exchange Online is specified in the Send Connector that is created for hybrid mail flow by the Hybrid Configuration Wizard. Messages sent between Exchange Online and the onpremises organization are treated differently from messages that originate elsewhere. This allows Exchange Online to bypass anti-spam filtering  processing on messages that are received from on-premises servers. 
Every message in the hybrid mail flow is stamped with an additional SMTP X-MS-Exchange-Org-AuthAs header. The value for this header should always be “Internal” for messages flowing across a hybrid connection.

If Exchange Online Protection detects that the X-MS-Exchange-Organization-AuthAs header is set to Internal for a message, the Spam Confidence Level header (SCL) is automatically set to -1.This allows the message to be forwarded to the recipient without having to go through message hygiene processing. The authentication header also allows Outlook clients to recognize the sending partner as being part of the same organization, triggering Outlook to display a full contact card instead of a summarized version.

So, check these :

The following must be true to allow TLS mail flow:
1 The Subject Name or Subject Alternative Name on the certificate used to
negotiate TLS must match the FQDN used to connect to the remote
server. This allows the use of SAN or wildcard certificates.
2  The certificate must be issued by a trusted public Certificate Authority.
3  The certificate must be valid; it must not be expired or revoked.

4  The Certificate Revocation List (CRL) must be available.

ruchi bisht say:

Try the below mentioned steps:

1.    Open Outlook in the new profile.

2.    Click on File->Open->Outlook data file.

3.    Select the Outlook.pst (your old data file)->Open.

4.    Check if the personal folder added to you Outlook contains your old data.

5.    Now click on Tools->Account settings.

6.    Click on Data file tab->select the second personal folder.

7.    Click on set as default.

8.    Click ok.

9.    Close and Open Outlook.

10. Expand the first personal folder and check if it contains all of your data.

11. Right click on second personal folder (created due to new profile) and click close.

don cuthbert say:

Those are the same symptoms that I had on the Toshiba 32-bit laptop. I developed a workaround to get past the devices that show not enough resources. On my Toshiba, many devices were replicated, causing the not enough resources errors. I noticed that PCI.sys is the primary driver involved in all of the replicated devices. What I did was to replace PCI.sys with PCI.sys from Windows 10 Version 1603, build 14393 (it has to be the 32-bit file). This has to be done while Windows is not running.

I booted with Windows 10 install media for build 14393. I then copied PCI.sys to replace that file from build 15063, which was installed but had no networking, due to the replicated devices. This eliminated the replicated devices and enabled Ethernet to work. I still do not yet have working Wi-Fi. Later PCI.sys got replaced with the correct file either automatically or after one of the security updates, yet it did not cause the problem of replicated devices to come back.

EDIT: I found the 14393 PCI.sys on the X: drive of the Command Prompt after booting with Windows 10 install media. You can get a Command Prompt from the first setup screen by using SHIFT-F10.
