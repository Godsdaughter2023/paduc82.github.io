How do I get Outlook 2007 to run rules in the order shown instead of - Microsoft Community

bpellett say:

How do I get Outlook 2007 to run rules in the order shown instead of who-knows-what order?

The first rule at the top of the list is to look for a string "Run On Q/A" in the subject or body, and move it to the "Relax-JustTesting" folder.

 

Emails containing the above string but matching other rules (e.g. "Error occurred") will end up in other folders (e.g. "Yikes-RedHotTrouble").

 

Therefore, it logically follows that the rules are being run in some non-sequential order, (vs. the glaringly-obvious top-to-buttom order).  I have tested this theory by "run rules now" in the RedHotTrouble folder, and selecting the first rule, and having all the false-alarm emails moved to the JustTesting folder.

 

How do I enable sanity in Outlook 2007 rules, so that it runs them in top-to-bottom order for each email?

sanwin2 say: If emails match more than one rule, the last rule matched will take precidence as each rule in turn will match and move the email.  To stop this happening, use the 'stop processing more rules' action at the end of the most important rule.  You may have to reorder to catch the correct option.

e.g. If the email matches your first and second rules then the email will end up where the second rule sends it, as this rule will be run after the first rule.  If the first rule has 'stop processing ...' then the second rule will never be hit.  If the email doesn't match the first rule but matches the second, then the second will work.
