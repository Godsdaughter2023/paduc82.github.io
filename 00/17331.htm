julie sheets say:
The visual report is showing work values in hours.  As long as all of your resources are working the same number of hours per day and the same number of hours per week - you can take the total hours and divide by that value to calculate requirements.

The Resource graph shows Peak units by default.  The peak units are the highest assignment unit per period.  If you have a resource assigned at 60% for most of the week but at one minute have the resource assigned at 75% - the peak unit for that week will be 75%.  In your example that 3000% may be only for one day, one hour, or one minute.

Try looking at the Resource usage view and adding Peak Units to the right hand side.  That may help you discover where the unexpected rise in values is coming from.

scottgem say:

The problem is NOT with bound forms, in that case. The problem is with the code you are using to filter the form. I suspect that you are using a query that references the Combobox. In that case, as soon as you set the Combo, then the form is restricted to the record matching the value displayed in the combo. You have to clear the combo and Requery the form. 

What would be better is to use a Filter 

Me.Filter = "fieldname = " & Me.combobox

DoCmd.RunCommand acCmdApplyFilterSort

This lights up the Filtered indicator on the Navigation Bar. Clicking it will clear the filter or you can manually do it:

Me.Filter = ""

DoCmd.RunCommand acCmdApplyFilterSort

So your problem does not appear to be bound forms, but the code you are using to filter them. So instead of a simple 2 lines of code to create a filter, you would rather use many lines of code to populate controls on an unbound form!
