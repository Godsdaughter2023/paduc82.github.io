john mcghie say:

Andrew:

Word encodes its documents in XML, which is a more modern version of HTML.  Thus, Word creates real hyperlinks, with URLs, display text, and anchors; embedded right there in the document code.

For some reason, Adobe is choosing not to read that code in Acrobat.  What it does is cause Reader to look for the display text.  If it finds a pattern that Reader can guess to be a URL, Adobe re-creates that as a URL after formatting the text as a PDF.

The PDF writer provided free by the Mac is a fairly basic version of a PDF writer, it does not have the ability to write URLs.

Jim's idea of using Pages is a good one: Pages will read a Word document (within limits) and write a PDF that contains live URLs.  Or you can re-express the display text so that it looks like a URL, if you do, Adobe Reader will then guess that it is a URL and behave accordingly.

Hope this helps

graham mayor say:
There are potentially three header ranges and three footer ranges for each section of the document. the following macro will display the contents of the three footer ranges for each section, from which you should be able to determine where the problem lies.


Sub WhatsInTheFooters()
Dim oSection As Section
Dim strText As String
strText = ""
For Each oSection In ActiveDocument.Sections
    strText = "Primary Footer " & vbCr & oSection.Footers(wdHeaderFooterPrimary).Range.Text & vbCr + vbCr & _
              "First Page Footer " & vbCr & oSection.Footers(wdHeaderFooterFirstPage).Range.Text & vbCr + vbCr & _
              "Even Page Footer " & vbCr & oSection.Footers(wdHeaderFooterEvenPages).Range.Text

    MsgBox "Section Number " & oSection.Index & vbCr + vbCr & _
           strText
Next oSection
End Sub

stefan blom say:

The list names are saved to the individual list formats (so-called liste templates) stored in the document. To get rid of undesired lists, all you can do is copy document content, minus the final paragraph mark, into a newly created document.

To show/hide paragraph marks, ¶, as well as other nonprinting marks, click the ¶ button on the Standard toolbar.
