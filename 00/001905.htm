Some salient points about the differences between an PST2PST2 Import or Export versus importing/exporting to any other file type which could easily explain the difference in the results

---------------------------
PST2PST (PST in this case references an <infoStore> such as an Exch mailbox)

Items are copied in their entirety. In short, any item being exported/imported is being "copied" to the destination with no <field selection> (ergo - individual field values are not read - the complete item is simply read/written)

---------------------------

Folder level export

Each item within a given folder is traversed individually and the "values" of the selected fields being exported are extracted and included in the destination output file (CSV, Excel etc). In this scenario, if there are problem with an individual item and/or the value of some field is not valid in a given item (or items), an error could occur which would not otherwise adversely affect the operation of Outlook <unless> the specific item was referenced within the Outlook UI.

Going on the premise that there are no issues with your import/export filters then it sounds like the issue really is one of a corrupt calendar item that needs to be located and fixed.

One indicator of that is whether or not any records got exported at all into the selected destination flle type. If memory serves me correctly, Outlook records are output in the sequence that those were created and not based on any current view of a folder.

----------------------------
