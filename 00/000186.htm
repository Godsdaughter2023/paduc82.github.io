As Tom said, you can't just copy SQL files. SQL files are not like Access files. Access files contain the entire database and all its objects and metadata. SQL Server has separate metadata tables for each instance, not each database. 

It might be too late if you don't haven't backed up the database using SSMS. If you did back it up you can use SSMS to restore the database to the new instance of SQL Server.

It really bothers me that people haphazardly recommend using SQL Server databases as Access back ends without including the caveat that you REALLY need to learn at least the basics of SQL Server such as backup & restore, proper indexing, referential integrity, etc. While SSMS tools look a lot like Access it is not just a bigger Access database.
