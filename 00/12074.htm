bernie deitrick say:

Row heights are not sortable.  But you could use a macro that would store the row height in a column, sort your data, and then re-apply row heights based on the sorted values.  So try this:

Sub TestMacro()
    Dim rngC As Range
    Dim lngR As Long
    Dim lngC As Long
    Dim shtA As Worksheet
    
    Set shtA = ActiveSheet
    Set rngC = shtA.UsedRange
    
    lngC = rngC.Cells(rngC.Cells.Count).Column + 1
    For lngR = 1 To rngC.Rows.Count
        shtA.Cells(lngR, lngC).Value = shtA.Rows(lngR).RowHeight
    Next lngR
    
    Set rngC = shtA.UsedRange
    
    With shtA.Sort
        .SortFields.Clear
        .SortFields.Add Range("B1"), Order:=xlAscending
        .SetRange rngC
        .Apply
    End With
    
    For lngR = 1 To rngC.Rows.Count
        shtA.Rows(lngR).RowHeight = shtA.Cells(lngR, lngC).Value
    Next lngR
    
    shtA.Columns(lngC).Delete
End Sub

What have you tried so far?

Just that one set of options?

You need a strategy.

You need to find out how by much over-allocated the resources are before and after leveling.

You can do this by inserting the Peak field in the resource sheet, entry table.

While you are at it, you may as well also insert the work field.

The resources with the most work and/or the most over-allocation are the ones which are the main problem.

Once you find them, you can start to find out when they are over-allocated and on which tasks.

The resource graph view, the resource usage view and the task usage view are all useful for pinning down the problem tasks and resources.

When you level, try first of all leveling with all of the fine-tuning options turned off (unchecked).

If this produces a complete solution, ie no more over-allocation, then you at least know that a solution exists and you can start to see if the options, one at a time or in combination, will give a better one.

The objective is to get a project finish date as early as possible. If you can get an acceptable finish date without the fine tuning options then you have a feasible plan without having to squeeze too hard.

The more you allow leveling to adjust assignments and split tasks, the more complicated will be the plan.

Try for the least complicated plan which does the job.

After leveling, there may be some residual over-allocation on some resources on some tasks. Sometimes this may be acceptable and there is no point in pushing harder for a perfect solution. Let them work it out on site.

If there is even 1 minute of simultaneous assignment left over, then MSP will flag the over-allocation.

If you have tasks with various date constraints, you may be preventing MSP from finding a complete leveling solution.

I can't see your tasks so I don't know exactly what you are dealing with.

If you have predecessor/successor links on the summaries, you will almost certainly be preventing MSP from finding a complete leveling solution.

If you have weird predecessor links FF, SS, SF and = or - lag, you will almost certainly be preventing MSP from finding a complete leveling solution.

Having said all of that, I think it is almost certain that your week-by-week setting is too coarse.
