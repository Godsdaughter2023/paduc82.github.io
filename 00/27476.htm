jay freedman say:

If you insist on torturing yourself with Word, you can turn off or disable some features, but not all of them.

  • Keyboard shortcuts: Go to File > Options > Customize Ribbon > Keyboard shortcuts Customize (the button at the bottom left corner of the dialog). In the Customize Keyboard dialog, select All Commands in the Categories list. Then work your way down the Commands list; each time you see a shortcut in the Current Keys box, click on it and click the Remove button. This will take a long time, but needs to be done only once.
  • AutoCorrect and AutoFormat As You Type: Go to File > Options > Proofing > AutoCorrect Options. On the AutoCorrect tab of the dialog, uncheck all the boxes. Then go to the AutoFormat As You Type tab and uncheck all the boxes there. (The tab that says only AutoFormat can be ignored, because it takes effect only when you manually start an AutoFormat, and the button for that isn't on the ribbon or the Quick Access Toolbar by default.) More on this at http://wordfaqs.ssbarnhill.com/TameAutoFormat.htm.
  • Spelling and Grammar: In the File > Options > Proofing page, uncheck "Check spelling as you type" and "Mark grammar errors as you type".
  • To disable the Research pane: Follow the instructions in HansV's post marked as the Answer of November 16, 2013 at https://answers.microsoft.com/en-us/office/forum/office_2010-word/how-to-completely-remove-research-task-pane. (The same procedure works in Office 365 / Word 2016.) If you identify another pane that you want to disable, put its name in the command in place of "Research".
  • You can slow your mouse movements in Windows (not in Office alone) by going to the Control Panel, opening the Mouse settings dialog, and moving the setting in the Motion section of the Pointer Options tab.

As far as I know, your other requests are not available in Office or Windows, but third-party programs or add-ins that implement them may be available.

swischner say:
Unfortunately, it's quite normal that OCR is much faster on OneNote for Windows (at least the desktop version, eg. OneNote 2016) than on the other OneNote clients (MacOS, iOS, Android, Windows 10 UWP app). The reason for this is, that OneNote 2016 uses an OCR engine that is part of Windows and therefore instantaneous. MacOS, for example, does not offer an own OCR that OneNote could use. So all text recognition is done on Microsoft's servers which process all requests from a waiting queue. That process can take up to several minutes until the indexing is done (I've even experienced hours, but I guess that was a glitch in the service).

There is no way to speed this process up, except for at least temporarily opening the notebooks in OneNote 2016 for Windows, to let it's local OCR kick in,

dw1609 say:

I have found the root of this problem, but first some background. I was working with a document that had been created in a long-ago version of Word. So naturally I presumed that corruptions had occurred in this document through its various upgrades and conversions to the latest version of Word. Wrong! I remade the document from scratch in Word 2016, and still had the problem of my encrypted password vanishing into thin air. I easily replicated this problem with other Word documents.

The problem was never my document. Given the universality of this problem across all my documents, I got to wondering what common trait had occurred in each document before the password protection was stripped away. The common trait was time. It seemed like the longer I had an unsaved document open, the more likely I'd lose the password. So, I thought, what process occurs periodically in all my documents?

AutoRecover! That's the problem. Whenever an autorecovery backup occurred, the password disappeared. After going to File/Options/Save and turning off the AutoRecover feature, the problem was resolved. Next, thinking that my installation (on a Lenovo T420 with Win 7) was corrupt, I ran a Quick Repair of my Office 365 installation under "Programs and Features" in the Control Panel. Didn't resolve the problem. So I ran a full online repair, and that also didn't resolve the problem. For the record, my machine is fully protected by Norton Internet Security and Malwarebytes, and there's no indication of any viruses or malware. Everything else runs just fine.

That said, given the paucity of complaints about it, I'm assuming it affects a limited number of Office users. But still, it should not be happening. I've been a Microsoft loyalist for years. I started using Word in 1990! There are many things about Microsoft's software that I like, but this kind of thing drives me crazy! If you work in a corporate environment (I don't), then you likely have a legion of IT geeks to fix problems like this. But what if you're an individual on your own? Is it any wonder that so many of Microsoft's retail customers have fled to competing products from companies like Apple and Google? If I continue to experience more problems like this, I'll be joining them when I cycle through my next computer upgrade.

Cells with scroll bars - Microsoft Community

kim l say:

Cells with scroll bars

Is it possible to create a cell with a lot of text to have a scroll bar rather than increasing the row/column?
phil hageman say:

You're okay on this.  Try typing text letters along the left side of the textbox, make them vertical.  Letter/enter/letter/enter/letter/enter, etc.  If you set the vertical scrollbar to true, when the letters on the left side exceed the textbox, the vertical scrollbar will appear.  =EMBED("Forms.TextBox.1","") just shows what is been active.

In practice, as the user types in text, it will appear normal and when they exceed the limits of the textbox the scrollbar will appear.  Also try setting Textalign to left and wordwrap to true.  This should work just fine.
