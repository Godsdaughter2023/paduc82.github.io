ken sheridan say: The ability to select multiple values in a Multi-valued Field in a datasheet is controlled by the DisplayControl property of the column in table design view.  This should be a combo box whose RowSource is either a value list (not recommended) or an SQL statement which returns values from a referenced table.  This property is usually created automatically when you design the table.

You say that you are able to select multiple values in a form.  Do those values persist if you move off the record and then return to it, and when you close the form and reopen it?  If so then the data is clearly being saved, and what you see or don't see in the table's datasheet is not particularly important, as you should never interface with data in a raw datasheet, but only in forms.  A form can be in datasheet view of course if you so wish.

However, as Duane has pointed out, experienced Access developers by and large do not favour the use of multi-valued fields.  What such fields do in relational database terms is model a many-to-many relationship type between two entity types, in your case between the SurveyResponses and PossibleImprovements entity types.  The conventional way to model a many-to-many relationship type is by a third table which resolves it into two one-to-many relationship types.  The usual interface is then a subform in the parent form.  One or more rows can then be inserted in the subform.

You might like to take a look at DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169
