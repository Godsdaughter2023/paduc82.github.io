The error "Invalid Server Name" is not related to the phone itself. Well in most cases anyways. This is a router setting issues. Depending on which router you use, the Administrator of the router can put a cap to the number of users. For instance, if I (as Admin) set a cap on a router to 10 users, all 10 users can connect no problem. But the moment he tries to browse the web, he will the error "Invalid Server Name". Simply because the router cannot furnish him with an IP address to connect to the internet. A simple way to fix that is normally to reset the router or contact the ISP provider.

Same problem occurred for me with the E7-00 on WiFi via a DrayTek wired / wireless router. Important to note:

  • All other devices can use the wireless network without problems
  • The E7-00 could connect to other wireless networks

So, the "server not found" message reported by the E7-00 is a problem specific to the device / network combination.

 

Basics (like DHCP) of the (wireless) network were set up ok. Some relevant settings I made in the router:

  • Fixed DNP-server addresses for all DHCP-clients
  • Forced use of 11G wireless only
  • Wireless security: WPA2 with pre-shared key only
  • Longest possible key used

After many attempts, I found that the use of WPA2 is related to the problem. Solution: In the E7-00, you can set up the network destination for the wireless network by just searching for WLAN, selecting the right one, entering the pre-shared key. After that has been done, change the detailed network settings. Menu-Settings-Connectivity-(Settings)-Destinations-(Choose group in which your WLAN destination is)-Select your WLAN-Options-(Edit/Change). WLAN security should be on "WPA/WPA2". Select (WLAN security settings), then set (WPA2 only).

 

(in brackets are menu references for which I do not know the exact English text)

 

This fixed the problem for me. Wireless works without flaws now.

OK, the greatest fun fact is the confusion you have caused in my tiny mind :-)

It works here.

  1. I hit Command + p
  2. Then go to the "Copies and pages" drop-down
  3. Choose "Layout"
  4. On the Layout tab, turn ON "Two-sided".

Make sure you are using a recent version of the Preview.  It updates about once a month, and the menus and ribbon part is under active development.

My answer is for version 15.9.0, build 150408 (the build number is in YYMMDD format).

Note:  This is BETA software, we need to expect some "interesting" features.  There is an outside possibility that Word has cached the incorrect printer properties definition (PPD).

If it has, you can cure it by:

  1. Quit all software
  2. Unplug the printer cable so OS X can't see it
  3. From System Preferences, delete the printer
  4. Re-start the computer
  5. Restart Word so it notices that the printer has gone
  6. Quit Word again
  7. Plug the printer back in
  8. Allow OS X to re-install it.
  9. Restart the system once more.

This will cause Word to grab a new PPD from the OS.  I doubt this is the problem, so I probably would not do it.
