john w. vinson say:

Unlike Excel, Access makes a very clear distinction between Macros and VBA. You're not using a macro at all here so there's nothing to close!

Try using the MsgBox function (which lets you reply) rather than the MsgBox statement, which doesn't.

I'd also suggest that viewing the data in a table Datasheet, as you're doing, is very limited. Generally Tables should be kept "under the hood", and data should be presented to users with Forms instead. But I'll leave that part alone:

Function ImportRawData()

On Error GoTo ImportRawData_Err

Dim iAns As Integer

iAns= MsgBox("Which Cabinet would you like to import? - Please insert an exclamation mark after the cabinet ID.", "Netmap Raw Data Import", vbOKCancel)

Select Case iAns

    Case vbOK    DoCmd.TransferSpreadsheet acImport, 10, "Area", "G:\Location\INE Raw Data.xlsx", True

    Beep

    MsgBox "Raw Data has been imported.", vbInformation, "Netmap Raw Data Import"

    DoCmd.OpenTable "Area", acViewNormal, acEdit

    DoCmd.ShowAllRecords

    Case vbCancel

' do nothing, just exit

End Select 

 

ImportRawData_Exit:

    Exit Function

 

ImportRawData_Err:

    MsgBox "Error in Cabinet Import " & Error.Number & vbCrLf & Error.Description

    Resume ImportRawData_Exit

 

End Function

guruprasad ra say:

Hi Matt,

Thank you for posting in Microsoft Community.

The error message 0x800CCC92 is caused due to incorrect account settings.

Please provide us more information to help you better.

How did you configured the SMTP server settings (Is it according to the email service provider (ESP) or the Internet service provider (ISP))?

If your ISP and ESP are different try the steps mentioned below:

1. Open Outlook->File->Account settings.

2. Double click on your email account->more settings.

3. Click on Outgoing server tab select the option Logon using.

4. Enter the user name and password provided by ISP.

Let us know if these steps helped you resolving the issue.

Thank you.
