How do I archive data in my Access database? - Microsoft Community

maria elena diaz say:

How do I archive data in my Access database?

I am teaching myself Access  I need to know how to archive or mark as inactive items that have been completed.  Any help would be much appreciated.

ken sheridan say: Add a Boolean (Yes/No) column named Completed to the relevant table or tables.  You can then set it to TRUE for those rows which don't want to see in any forms or reports and base the forms or reports on a query which excludes the rows with a value of TRUE in the Completed column.

Note that TRUE is a Boolean constant; you don't enter the word 'True' in any column.  On a form for data entry you'd use a check box bound to the Completed column and check it for those rows you want to archive.

Let's assume you have a database with three tables, Employees, Projects and EmployeeProjects, the last modelling a many-to-many relationship type between Employees and Projects.  When a project is completed and you want to 'archive' it you can set the Completed column in Projects to True.  To show all the current projects and the employees engaged in them you'd join the three tables in a query.  In query design view you'd enter False in the 'criteria' row of the completed column and uncheck the 'show' checkbox in the same column.  This would be same as putting WHERE Projects.Completed = FALSE in the query in SQL view.

You can of course see the archived rows at any time in a query.  Not putting any criterion for the Completed column would show all projects, both current and completed, putting True in the 'criteria' row would show only the completed projects.

You could also have a Completed column in the Employees table which would be set to True when an employee leaves the organisation. This would allow you to show only current employees, but you'd need to be careful.  If, with a query joining the three tables you restricted it to where the Completed column in Employees is True you would not return any projects which, while still current, had no current employees engaged on them, which might not be what you had in mind.  It would be possible to return those current projects in which no current employees are engaged (so that you can assign employees to them), but the query would be a little more complex.  It might only confuse you to go into details of how to do this at this stage.

scottgem say:
Ken's method is very workable, but my personal preference is an alternative method. 

I've found, in most cases, when you close or complete something, you want to know WHEN that something was completed. So usually there is a complete date field in your table. If the complete date has been filled in, then the item is completed. Otherwise its not. Therefore no need to add another field that would have to be marked.

So if you want to filter out completed items you can set a filter to 
CompleteDate Is Null


ken sheridan say: Scott's approach is a good one, but one word of caution.  Whatever you do, don't combine the two approaches by having a Boolean Completed column and a CompleteDate column of date/time data type.  That would introduce redundancy as it is possible for the two to hold contradictory data, one column telling you the project is completed, the other telling you it is not.  This might sound obvious, but I've seen it done quite a few times.
