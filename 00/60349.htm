boydcarter say:

Problem with BITS service : The requested service has already been started. More help is available by typing NET HELPMSG 2182.

When I found that I could not complete an installation of new software after Windows 10 version 1809, I spent several hours trying to find solutions to this problem. 

The recurring theme that kept popping up was "Problem with BITS service : The requested service has already been started. More help is available by typing NET HELPMSG 2182."  I tried many proposed solutions but nothing worked.

I finally searched on "What is bits service" and found a simple solution that solved my problem.  The solution was found here.  https://helpdeskgeek.com/how-to/background-intelligent-transfer-service-will-not-start/  (give credit where credit is due) and I quote from that page:

"If BITS can’t be started for some reason, first make sure it is enabled in the Services control panel applet. Go to Start, Control Panel, Administrative Tools, and click on Services.

Make sure that Background Intelligent Transfer Service is set to Automatic or Manual as the startup type.

If it’s not set to Manual or Automatic, then double-click on it and change the startup type to either one of these. If the service is configured correctly, but BITS will still not startup properly, check to make sure none of the dependent services are disabled."

I double-clicked to edit the line, set it to Automatic, then manually started it.  I installed my new software and am writing this solution for the benefit of others who may have this problem.

I just went back to it and found that it had stopped again, so right before I shut down my computer, I am going to set it to automatic and hope that it stays that way.

Hope this helps

LEAVE THE $hf_mig FOLDER STRICTLY ALONE OR YOU WILL BREAK WINDOWS. Use CCleaner to remove Temporary Internet Files instead of doing it manually.

Your partitioning scheme is not as useful as you think it is.
