I finally managed to solve this on my own.  Here is how I did it, for anyone else having the same problem:

  1. From inside my own computer account, I went to Settings -> Accounts -> Your Info, and clicked "Sign in with a local account instead".  I followed all steps to make that account a local account rather than a Microsoft account.  (This was necessary for later steps.)

  2. From inside my daughter's computer account, I went to Settings -> Accounts -> Your info, and clicked "Sign in with a local account instead".  I followed all steps to make her account a local account.

  3. From inside my daughter's computer account, I went to Settings -> Accounts -> Your info, and clicked "Sign in with a Microsoft account instead".  I followed all steps to set it up with MY own Microsoft account.

  4. This done, I was now able to access the store as myself (from within my daugher's computer account) to download my purchased version of Minecraft for her.

    (Side note - at one point it prompted me for the pin for MY Microsoft account, but it actually required the pin from HER Microsoft account.  So if you are getting caught on PINS, try that of the second account instead of your own.)

  5. Once I had finished with the Microsoft Store and gotten software set up, I went back again (inside my daughter's account) to Settings -> Accounts -> Your info, and clicked "Sign in with a local account instead".  I followed all steps to remove MY Microsoft account from her computer account.

  6. Again, still inside my daughter's computer account, Settings -> Accounts -> Your info, and clicked "Sign in with a Microsoft account instead".  I followed all steps to set it up again with her own Microsoft account.

  7. Then I logged back into my own computer account, went to Settings -> Accounts -> Your info, and clicked "Sign in with a Microsoft account instead".  I followed all steps to set it back up again with my own Microsoft account.

Viola!

Both our accounts are set up again the way they should be again, and I managed to get into the Microsoft Store and set everything up on my daughter's computer account.

I have also verified that the installations still work after I set her computer account back up again with her Microsoft account.

    As per the issue description you have configured Windows 7 as NTP server but could not sync time with other computers. You are getting error message Access Denied, so the issue could be due to some permission. It could also be if service responsible for syncing the time is not running.

    You probably already know that having accurate time set on all clients and servers is important. Very bad things happen when computer clocks disagree. Things like:

    • Users cannot logon to a domain
    • Applications overwrite new data with older versions
    • Servers fail to synchronize data between each other, causing version conflicts
    • Financial transactions become subject to dispute
    • Regulatory requirements are not met

    Windows implements the W32Time service as both an NTP client and server. The service is off by default. So configuring a system as an NTP server requires both enabling the W32Time service and configuring it as a server. The process is very simple.

    • Press Windows key + R
    • Type services.msc in the run box
    • Now in the services list scroll down and search for Windows Time service.
    • Try to start the service and make sure the start type I set to be Automatic on both Server and clients computer.
