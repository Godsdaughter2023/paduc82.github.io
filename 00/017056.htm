tom van stiphout say:

The two properties exist to provide a link between the parent form and the subform. That link must be expressed by specifying the fields participating in the link. For example in a form that shows a customer with his orders:

Master: CustomerID

Child: CustomerID

The master typically has the PK, and the child has the FK. They don't need to have the same name.

 

Sometimes the relation is more complex and two or more fields are entered in this property, separated by semicolons. If it's not the same number of fields in both properties, your error will ensue.

 

ken sheridan say: I think you are referring to a table's subdatasheet.  If you specify a table for this by entering a value in the table's SubdatasheetName property values are entered into the table's LinkMasterFields and LinkChildFields properties.  This is generally done automatically on the basis of a relationship which has been created between the tables, but can be done manually in the absence of a relationship.  The properties entered are exactly the same as in a form/subform, as described by Tom.

Open the table in design view and see what is entered as these two properties in the properties sheet.  Amend them as necessary so that the LinkMasterFields  is the primary key of the current table and the LinkChildFields property is the corresponding foreign key in the referencing table.  If the keys are in each case made up of more than one column, then in each property the two column names are entered, separated by a semi-column, again as Tom described for a form/subform.

jon k. jr. say:

Presuming your scoring is assessed/recorded similar to ;

Total (100 max) = Accuracy (50 max) + Precision (30 max) + Center (20 max).

Value Tiebreak1 as .00 to .99  (=Accuracy / 100)

Value Tiebreak2 as .0000 to .0099 (=Precision / 10000)

Value Tiebreak3 as .000000 to 000099 (=Center / 1000000)

Total = (Accuracy+Precision+Center) + (Accuracy/100) + (Precision / 1000) + (Center / 1000000)

Thus, a real score of 48/28/18 -- > 94 nominal points tallied as 94.482818

 Score 45/25/15 --> 85.452515, etc.

If 'smaller is better' for any tiebreak factor, transform that tiebreak value as (100-score).

E.g. Total = (Accuracy+Precision+Center) + ((100-Accuracy)/100) + (Precision/10000)+(Center/1000000).   Score 48/28/18 would then --> 94.522818

Then, simply use the RANK function to find your winners.

Display INT(Total) as needed to show only the nominal point total.

Sub Note()

    orng.Select
