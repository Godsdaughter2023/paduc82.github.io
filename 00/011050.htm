·        Click Close

·        Surf a few pages

·        Open Internet Explorer

neha singh msft say:

Hi Ash8989,

You can run the following PowerShell command to set the password expiration date for a domain.

Set-MsolPasswordPolicy -DomainName <string> -NotificationDays <string> -ValidityPeriod <string>

  • ValidityPeriod - Specifies the length of time that a password is valid before it must be changed. A null value indicates the default value of 90 days will be used.
  • NotificationDays - Specifies the number of days before a user receives notification that their password will expire. A null value indicates the default of 14 days will be used.

After setting the password expiration date, run the following command to retrieve the values associated with the Password Expiry window or Password Expiry Notification window for a tenant or specified domain.

Get-MsolPasswordPolicy -DomainName <string>

Tanya,

Your issue as Steve mentioned isn't really  aren't due to an attack, they are simply misunderstanding the relatively complex design of a modern Windows operating system, its underlying filing system (files, folders and permissions) and the configuration and operation of various programs.

I know that much of this seems very confusing to most people, but that's simply a side effect of the increasingly complex operating system that Windows has become, partially in order to provide better security and partially to provide for the "extras" that you mentioned.  Though you may not want these yourself, the simple fact is that to allow others to have them, the support required to make them operate must exist within all systems of a specific Windows version.  This isn't really any different than in previous versions, there are simply more of them and some of them may be more visible to users than they were in the past.

In other cases such as your issues with email, I suspect these are simply changes to the interface that don't make sense to you.  For example, the statement that "my email account has a you and a me as me has sent emails to you that were sent by someone else first."  I suspect that you are using an email program provided by Yahoo! that recently changed?

The reason I ask is because one of my cousins' was confused by this same recent change to Yahoo! Mail, since rather than listing your own name or email address as it used to, this has been replaced with either "you" or sometimes "me" in the "from" or "to" columns.  I can see that the programmers thought this would be both easier to fit on screen as well as simpler to understand, but instead it has simply added confusion for those who don't catch the reasoning behind the change.

I could go nit-picking through everything else you've listed, but like Steve, I think it's better simply to state that nothing you've listed seems truly strange to me, just not obvious to someone without our technical background.

If you'll look at my first post in response to ilovemsft, that was intended to offer an alternative to someone who obviously doesn't understand the technical aspects of their computer and so might be better off using something else.  Whether or not any of that other person's issues were real, the confusion they have and thus the unease this creates in using their PC means that it can never be an effective tool, regardless of whether the current apparent problems are temporarily "fixed".

From your posts I feel you are experiencing the same basic issue, so my recommendation would be similar.  Rather than fighting with the complexity of the full Windows PC, you are likely to be far more comfortable working within the simpler operating environment of the newer tablet or convertible laptop like designs that have become available.  This will depend on whether you need any programs beyond a web browser, email, the most common Office apps of Word, Excel, Powerpoint and OneNote and others which are available from the Windows Store.

In fact, I see that you already mentioned a tablet above, so what doesn't that tablet do that you require and possibly we can guide you to a more capable convertible model that does do these things?

Rob

You say that saving to PDF comes up blank, which means Excel is not outputting anything.

Instead, try using File > Print. Click the PDF button in the Print dialog and save the PDF. Open that saved file to see exactly what Excel sent to the printer. If that file is empty, then something's wrong with Excel, not your printer driver.

It appears you have already checked the right things in Excel's interface, so my suggestion is to delete Excel's preferences and try again.
