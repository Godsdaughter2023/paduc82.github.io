rohn007 say:
Word not only "prefers" to use the NORMAL in USER TEMPLATES folder, that is the default one it is hard coded to use. 

Word will use its hardcoded defaults.  Word will create a new NORMAL.DOT(M) file whenever it detects a need for one. That need is defined as no NORMAL in the USER TEMPLATES folder and when any changes to the hardcoded default configuration is made.  WORKGROUP templates are intended as a place to store secondary templates.

In companies that I've worked for who required a corporate standard NORMAL.DOT(M) the solution was to store one in a place like WORKGROUP TEMPLATES then use a network logon script to download the standard every time you login to the network.  It could be handled in other ways.

graham mayor say:

If these are index entries  the following macro should do the job

http://www.gmayor.com/installing_macro.htm   

Sub Macro1()
Dim oFld As Field
Dim strFldText
Dim strAsk As String
Dim bHidden As Boolean
    With ActiveWindow.View
        bHidden = .ShowHiddenText
        .ShowHiddenText = True
    End With
    For Each oFld In ActiveDocument.Range.Fields
        If oFld.Type = wdFieldIndexEntry Then
            oFld.Select
            strFldText = Replace(oFld.Code, "XE ", "")
            strAsk = MsgBox("Delete " & strFldText, vbYesNoCancel)
            If strAsk = vbYes Then
                oFld.Delete
            ElseIf strAsk = vbCancel Then
                GoTo Finish
            End If
        End If
    Next oFld
Finish:
    With ActiveWindow.View
        .ShowHiddenText = bHidden
    End With
End Sub

roady [mvp] say:

That's because you are copying a recurring item, the behavior you are expecting only applies to single items. This is indeed a design change from Outlook 2007 as people were actually expecting to copy the recurrence pattern as well.

To create a single item copy of an appointment with a recurrence pattern, click the appointment with your right mouse button and hold down the mouse button while you drag and drop it on its new day/time. Now you can choose to Move or Copy the item. Choosing "Move" will create a one-off for the current series and choosing "Copy" will create a single item copy unrelated to the original series.
