All of the attached DMP files are of the KERNEL_DATA_INPAGE_ERROR (7a) bug check.

This bug check indicates that the requested page of kernel data from the paging file could not be read into memory.

In all of the dumps, we see - ERROR_CODE: (NTSTATUS) 0xc00000c0 - This device does not exist.

This indicates very few things and is fairly straight forward:

3. If it's an SSD you need to be 100% sure the SSD is at its latest firmware.

How do I put a picture over a video covering up the video but keeping the audio going?

I am using Movie Maker  2011 build 15.4.3508.1109. I am trying to overlay a picture over a video without stopping the audio. How do I do it?
considerate_guy say:

It really isn't possible in Movie Maker 2011.  If you were able to run Movie Maker 2012, I'd suggest you do it as follows:

  1. Add your video clip and set its volume to zero.
  2. Click to add Narration (new feature in 2012) and choose to use the same video clip as in step 1 for the audio (new feature in 2012).  The result is you now have the video and the audio running in parallel treated separately.
  3. At the point in the video where you want to add the picture, trim out the exact amount of video using the Split feature.  For example, to add a photo for 7 seconds, split out exactly 7 seconds of video.  You'll have to adjust this if you add transitions because you actually need to trim out less video.  In any case, you want to have the video start up again at the point that perfectly syncs with underlying "narration".

That's it!

 

In Movie Maker 2011, you don't have a separate narration track or the ability to use a video as your additional audio track.  The workaround to this is to use your music track.  Additionally, you'd have to convert your video to an audio-only file using something like Freemake Video Converter.  Select the clip and choose to convert it to .mp3.  That will give you a sound file that is just the audio.  Then add the .mp3 as your added "music".

wunders say: In a Domain environment, it is the Domain that authenticates users to a computer.  If permitted by Domain Group Policy, a limited number of user logons may be "cached" so that users can log on to a machine when the machine is not connected to the domain.  If for any reason this cache becomes invalidated, then the solution is to reconnect the computer to the domain and reestablish credentials.  The only other alternative is to log in using the local Administrator account (at login screen, click on "More..." and change from domain to local machine and log in as administrator).  By default, the Administrator user does not have a password (ie leave it blank), but in a Domain environment, the Domain administrator will most certainly put a password on the Administrator user.

Removing a computer from the Domain and changing to a Workgroup can cause essentially the same predicament.

"You cannot log on after you remove the computer from the domain"
  < http://support.microsoft.com/kb/317049 >
