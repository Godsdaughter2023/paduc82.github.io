Put this the Workbook code:

 

Private Sub Workbook_SheetActivate(ByVal Sh As Object)

    On Error Resume Next

    Dim ws As Worksheet

    Sh.Range("A1").Select

    Sh.ScrollArea = 1

    On Error GoTo 0

andreas killer say:

Am 14.08.2010 20:18, schrieb Dewayne66:

When I put the code in the module, it does not show up in the macros list.  Can you tell me what I am doing incorrectly?

The macro must be placed in the module named "ThisWorkbook".

But this macro does not exactly want you described.

It selects A1, but only and every time if the user change the sheet!

That means if a user save the file with i.E. C10 selected, and opens the file again, C10 is selected.

If the user switch to the second sheet, A1 is selected in this sheet. If he select i.E. B5, switch to some other sheet and back, A1 is selected.

The macro below does what you described. And it must be placed the the module "ThisWorkbook".

And if you like, you can also use Felipe's macro.

Andreas.

Private Sub Workbook_Open()
  Dim WS, SaveActiveSheet
  'Disable events, no screen output
  Application.EnableEvents = False
  Application.ScreenUpdating = False
  'Remember current sheet
  Set SaveActiveSheet = ActiveSheet
  'Go trough each sheet
  For Each WS In Sheets
    'Is it a sheet or a chart?
    If TypeOf WS Is Worksheet Then
      'Activate this sheet
      WS.Activate
      'Select A1
      WS.Range("A1").Select
    End If
  Next
  'Restore current sheet
  SaveActiveSheet.Activate
  'Events on, screen on
  Application.EnableEvents = True
  Application.ScreenUpdating = True
End Sub

dtinman say: I had the same problem saving 2010 32 bit powerpoint on windows 8 64 bit Dell laptop. After a few hours of searching the web I came across the below post and link. I had the Send to Bluetooth COM Add-in active and as soon as I deactivated the Add-in the problem seems to have been resolved.

Not sure I have a solution for you, but here goes:

PowerPoint is picky about a couple of things. One is printer drivers. It likes what it likes, and it likes 'em local. In other words, you'll want to try installing a known good printer driver locally (not as a network printer) and making sure it's set as the default printer. Here's specifics for you to follow: http://www.pptfaq.com/FAQ00952.htm

Next is add-ins. If you go to File | Options | Add-ins, what is listed? If you see especially a Send to Bluetooth (or some such) add-in, disable that and see if the problem doesn't resolve. If you don't see that, what do you see?

(To tell the truth, that's actually more of an issue with PPT 2007 than 2010, but it's still a good thing to do. )


http://office.microsoft.com/en-us/powerpoint-help/enable-or-disable-add-ins-in-office-programs-HA010034127.aspx#BM21
