slartibartfarst say:

The explanation for this would seem to be that WDS (Windows Desktop Search)  indexes/searches the Desktop perfectly well  - i.e., including everything on the Windows Client (a PC or laptop)  - but not what is on OneDrive.

Thus, if your OneNote Notebooks were created on OneDrive, then WDS cannot Search/Find anything in those Notebooks - they never get indexed by WDS, and thus they are not contained in the WDS Index cache. However, if you use WDS to search for something that is in OneNote, WDS will come up with a result displaying a blank icon alongside the referenced content you provide - it finds the latter in the .onecache files - but the Search Connector in the result bears no hyperlink to that actual content (WDS Index has no reference to its actual location) and so has no icon to relate to the source. All you know from that is that the referenced item exists in OneNote, but you can only find it by using the OneNote search on the open Notebook that contains it.

Similarly, if you have migrated your OneNote Notebooks from Desktop to OneDrive, then WDS might still be able to find what was cached in its index before the migration, so Search might appear to work OK, but it cannot find new/changed content in the now OneDrive-based Notebooks.  When you use WDS to search for something that is new/changed, WDS comes up with a result displaying a blank icon alongside the referenced content you provide - it finds the latter in the local .onecache files - but the Search Connector bears no hyperlink to that actual content (WDS Index has no reference to its actual location) and so has no icon to relate to the source. All you know from that is that the referenced item exists in OneNote, but you can only find it by using the OneNote search. I initially thought that this was a bug, but it seems such an egregiously stupid bug to leave in (i.e., not fix) that I now presume it is quite a deliberate feature of Microsoft OneNote's design. Maybe it has some marketing advantage, or something, but I don't know.

swischner say:
I am sorry to be the messenger, but this is not a bug or missing setting but a restriction of operating systems like iOS and MacOS.

On those, each application or app has its own directory where it has exclusive
write access. This also applies to OneNote. OneNote needs a cache folder on these systems as well, in which it keeps open notebook contents, including attached files like word documents. And only OneNote has writing rights here, while other applications, like Excel, may only read data from other than their own storage areas. There is no way though, that Word (or other applications) could write changed content back to that cache folder. That's why documents from the OneNote storage locations are opened as read-only in other apps.

It does not matter where the original document is stored as when you attach a file to a OneNote page, a copy of the original file is created and placed in OneNote's own data space (like when attaching files to an e-mail)

I talked to a OneNote engineer once and he confirmed this and said they are looking into a solution but as it is a restriction of the operating system it is not certain that this will change in an upcoming version.

It should be possible though to not attach the actual file into your notebook but a link to the original file on OneDrive instead. In that case, everybody that you might have shared your notebook with also needs full access to that OneDrive folder holding the linked document to open and edit it.
