Hmmm...  Sound files can be saved in Notebook View Word documents, but it is a seriously bad idea to do this.  The idea is that you can record lectures and take notes in Notebook Layout.

The net result is that re-starting a recording has a bad habit of wiping out the previous recording.  Not always, just when the recording is really valuable.

And the presence of a sound file in the document seems to make the document very unstable. By which I mean that instead of losing the recording, you lose the whole document.

I guess it seemed like a good idea at the time, but the feature was very buggy, and I am not aware of any fixes to it.

The bottom line is that Bob's idea is the best there is: place the sound file in the same folder as the document (NOT a subfolder within that) and link them from there.  The folder must be on your local drive, and the audio files must be at the same level as the .docx).

Before working on such a document, turn OFF Word>Preferences>General>"Update automatic links at Open".  If you forget this step, Word will re-write the links to explicit links when you open, so the file can't then be used on anyone else's computer.

Windows 10 file explorer needs queued transfers - Microsoft Community

erikwestdijk say:

Windows 10 file explorer needs queued transfers

Windows already stacks multiple transfer instances but starts them directly when added.

A new transfer should only start when both the source and destination device are not involved in already running transfers.

After a running transfer is done, a new transfer involving those devices can start.

This is especially useful when moving around data on local drives as well as to/from NAS devices.

cjlim say:

You won't be able to completely drain the battery in DC16 and transfer it all into the phone battery and here is the reason why. 

 

For the current to flow, there has to be potential difference between the 2 batteries. In simple terms, the DC16 starts with double the power of the Phone battery. After the first charge, both batteries will have roughly the same amount of power left eg around 1050. After the second charge equilibrium will be achieved at 525 at which stage charging will cease becase there is 525 in both batteries and there is no potential difference so no current flows. However if you run down the phone battery again and plug it in again, it will probably charge to around half that again. Only way to fully run down the DC16 is to keep it connected to the phone to power the phone till it runs down. 

 

I am using mah figures as a simplified way to compare the batteries but potential difference is measured in volts. Currently will only flow from higher voltage to lower. No current will flow when the voltage in both batteries is at equilibrium, hence no charging. The DC16 may not necessarily be empty when charging stops.
