macropod say:

You really wouldn't want to assign "Shift+i" to a keyboard shortcut. If you did so, how would you type 'I'?

To assign a keyboard shortcut for a macro:
1. Click on File|Options|Customize Ribbon>Customize.
2. In the Categories list, choose Macros.
3. In the Macros list, click the macro that you want to assign the shortcut to.
4. In the 'Press new shortcut key' box, type the key combination that you want to use. Note that shortcuts will require the use of the Ctrl and/or Alt keys.
5. Check the 'Current keys' box to make sure that you aren't assigning a key combination that already use/need to perform a different task.
6. In the 'Save changes in' list, click the option that matches where you want to run your macro
7. Click Close.

jay freedman say:

You really wouldn't want to assign "Shift+i" to a keyboard shortcut. If you did so, how would you type 'I'?

Going a step further, Office won't allow you to assign the combination of only the Shift key and any letter or number key to a shortcut. The only keys that can be combined with just the Shift key are the function keys F1 - F12. That's why the procedure "doesn't work" and the 'Press new shortcut key' box remains empty. 

All shortcuts (other than Shift+Fx) must include the Ctrl key, the Alt key, or both, with or without the Shift key.

hansv mvp say:

Let's say the January data are in A2:A10 and the February data in B2:B15.

To return the non-matching data from January:

In another column, in row 2, enter the formula =IF(ISERROR(MATCH(A2,$B$2:$B$15,0)),A2,"")

Fill down to row 10.

You can copy the results and paste them as values elsewhere, then sort them to get rid of the spaces.

Similar for the other way round: =IF(ISERROR(MATCH(B2,$A$2:$A$15,0)),B2,"")

Peter

Just to cover off a few things

#1a - If you copied the file to the new machine while Outlook was open on the old - could corrupt the file - in which case, redo the copy with Outlook closed and try again (but suggest doing the next step first just as a precaution)

#1b - If you ran ScanPST on the new machine, suggest running it on the old before doing a copy to the new machine (with Outlook closed)

#2 - As Brian mentioned, no need to use import on the new machine - just open the data file and if it's a PST file associated with a POP account, then select it as the file to use instead of letting Outlook create a new one

#3 - One thing to check on the old machine is the format of the PST file. Right click on the root of the folder tree --> Data File Properties --> Advanced --> if the format description includes (97-2002) - this is an ANSI based PST file which really should be converted to Unicode sooner then later.
