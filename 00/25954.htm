scottgem say:
The foundation of any database is a proper table structure. To build a proper data structure you need an understanding of the principles of relational databases. These principles are known as Normalization and there is a lot written about it that is available.

One of the concepts you need to understand is that of Entity/Attribute. An Entity Is the basic building block and attributes describe the entity. For example, Members are an Entity, the attributes of a member are things like names, addresses, contact points, and other demographic info.

So I see the need for the following entities; Members, Sessions, Payments and Attendance. The conundrum that is the base of your question is understanding the many to many relationship and how to model it. A member can attend multiple sessions and a session can be attended by multiple members. Many to Many. You use a Junction table to model this:

tblAttendance
AttendanceID (Primary Key Autonumber)
MemberID (Foreign Key)
SessionID (FK)

From there you can build queries to report on attendance per session.

This medium is not really suitable for a tutorial but rather asking specific questions. I've tried to give you some background for research along with a answer to the one specific you asked. Looking at models or templates for a school situation will probably help you.

First of all the text rendering 'bug' is associated with Windows 8, not with the specific version of Publisher. You can take a Publisher 2007 file produced on XP, Vista, Windows 7 and then open it using Publisher 2007 on Windows 8 and the text will render differently. It appears that the height of the font or the amount of line space it requires is different in Win8. Here is a good link to a thread where this issue is disscussed and documented. http://answers.microsoft.com/en-us/office/forum/office_2010-publish/documents-format-changes-when-working-on-the-same/892f528d-4750-47d6-a7da-8bb6e5273c0c  Interstingly one of the posters in that thread seemingly found a 'solution' by installing the Windows 7 version of the font on the Windows 8 machine.

 

Now, there is another part of the answer and that is that MS changed one of the default settings as of Publisher 2010. In Publisher 2007 if you right clicked text in a text box > Change text > Paragraph you will find the line spacing to be set at 1sp and 0pt after paragraphs. As of Pub 2010 MS changed that to 1.19sp and 6pt. I think they did this to match what the default had been in Word and wanted some consistency. Now if you open a Pub 2007 formatted publication in Pub 2013 then the original formatting will stick, but if you try to copy and paste from a Pub 2007 text box to a Pub 2013 text box then sometime the formatting changes, and this has created problems for users.

 

So, what you may be seeing may be the default line spacing that has been changed in Pub 2013, or it may be the different way that Windows 8 renders text in Publisher. If as you say you have both Pub 2007 and 2013 installed on the same Win8 machine you can open the same old Pub 2007 file built on an older operating system and I would expect the publication to look the same...and the formatting to be skewed a bit in both. You can save as a pdf from both and compare that way. The main culprit then is Win8 not Publisher.
