a. user say:

If it has to be ADO, I think the main problem is likely to be with your connection string, which probably needs to specify the correct provider. It looks as if you have been trying a number of different approaches, but 

(a) this is how I would typically do it (there are other ways):

Sub getExtDataADO()
Dim cn As ADODB.Connection
Dim rs As ADODB.Recordset
Set cn = New ADODB.Connection
cn.Open "Provider=Microsoft.ACE.OLEDB.12.0;Data source=C:\Working Files\Combined (Working)\Rays XYZ Data Entry v1_1.accdb;"
Set rs = New ADODB.Recordset
rs.Open "IR", cn

' do some stuff with the recordset
Debug.Print rs.Fields(1).Name, rs.Fields(1).Value

' close everything
rs.Close
Set rs = Nothing
cn.Close
Set cn = Nothing
End Sub

(b) i.e. you don't need both commands and recordsets, and you don't need ADOX for this. You do need a reference to the Microsoft ActiveX Data Objects 6.0 (or 6.1) Library.

6. Click Apply and Ok.
Close your browser and start a new session.
