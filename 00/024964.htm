A custom scan using MSE is a scan that will scan local drives, folders, and files that you select in the Windows Explorer like tree view with the check boxes.

Your email is stored by your email program and might be stored in a hidden folder as .eml files (Windows Live Mail does this) or in a .pst file in a different hidden folder (Outlook does this) or any number of other locations.

You are protected by Real Time protection, in theory, to *prevent* malware from getting to the machine. Of course, malware can be hidden in an email attachment and you wisely don't open that attachment, so real time protection does not catch it. A full scan will catch it, but even if not caught, it isn't active.

Protection while browsing the Internet is handled partially by security in the web browser (SmartScreen Filter in Internet Explorer) and partially by MSE (real time protection). No scan can protect you from infection while browsing.

Do note that the weakest link in security is what malware will exploit, and that is quite often the person using the PC.

Besides MSE, the following recommendations will assist in protecting the PC from infection:

-Make sure that the Windows Firewall is enabled.

-Make sure that all important/critical updates, including service packs for the operating system and programs are installed from Microsoft Update (Windows Update).

-Make sure Internet Explorer is at version 8 or above (latest released version for your OS) and updated with all patches.

-In Internet Explorer use the SmartScreen Filter.

-Make sure that IE Internet Security settings are at least set to medium-high (default).

-Enable the pop-up blocker in IE.

-On Vista and later make sure that User Account Control (UAC) ON  and not running with elevated privileges.

-Make sure that Windows Automatic Updates are set to at least notify, but the preferred setting is to download and install automatically. If you update manually, be sure to update as soon as possible after being notified of available updates.

-Make sure that installed applications, especially Adobe Acrobat, Adobe Flash, and Java are at their latest versions. Many vendors are regularly updating and patching for security holes.

See: https://blogs.technet.com/b/security/archive/2012/07/19/the-rise-of-the-black-hole-exploit-kit-the-importance-of-keeping-all-software-up-to-date.aspx?Redirected=true

-Never click through links from unknown sources and use caution even if they are from a "trusted" source.

-Never open unsolicited email attachments.

-Practice safe web browsing.

cindy meister say:

Hi allikatpur

When a document "stops responding" this is often a sign that the document's internal structures have been damaged. The common term is "the document has become corrupted".

Size is less often a factor than complexity, and how the document is managed.

The "short and simple" way to save a damaged document is to Ctrl+A (select all), Shift + Left Arrow (deselect the last paragraph mark), copy and paste into a brand new document. This often works because the most frequently damaged structures are stored in that last paragraph mark. Try that with a document you can't save and see if that helps (for the moment, at least).

In order to really fix the problem, it would help to know what kinds of things you have in these documents. Do you change the page orientation often, for example? Or use newspaper columns? Change headers and footers?

Are you using the "Master Document" feature?

cody c. say: Hey John,

Sorry to hear you're having a difficult time installing the 14.2.4 update. I split your post off into it's own thread so that we can better focus on your specific issue.

Without knowing which solutions you've tried, I'm afraid I might be recommending solutions you've already attempted. Sorry in advance if that happens.

The first issue (and most common) stems from re-installing on top of existing installations. In general, don’t do this. Re-installing on top of existing installations can re-introduce files that should not be present. If you need to re-install, please be sure to remove Office 2011 as described on the following links:

 

Video | Uninstall Office 2011

Step-by-Step Instructions | How to completely remove Office for Mac 2011

 

One workaround for the 14.2.4 update that has helped some users is to temporarily move or rename the /Library/Automator folder. You can do this by following the steps below:

 

  1. With Finder open, choose Go > Go to Folder...

  2. Type in the following path and then click Go:
    /Library

  3. Find the Automator folder, select it, and then click the name of the folder again to Rename the folder. Name the folder Automator_Old

    Note    :     OS X may prompt you for your password. This is normal.

  4. Attempt to install the 14.2.4 update again.

  5. If the update installs correctly, rename the Automator_Old folder back to its original name: Automator

 

If that solution does not work and you know you have re-installed Office 2011 on top of an existing Office 2011 installation, you’ll need to remove Office 2011 properly and re-install. 

 

The second issue was some users changed the size of the /tmp/ directory on their machine which was too small for the update to fully install. Re-sizing the /tmp/ directory resolved the install issue for these users.
