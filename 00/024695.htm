tom van stiphout say:

Too big of a topic to cover in one post, but a few keywords to help you focus your learning curve:

* SQL Server can (and should, in most cases) run with Windows Authentication where access is granted based on membership in certain Active Directory groups that you have given certain access to the server, the database, and the database objects.

* One advantage is that there is no username/password dialog: users are authenticated by logging into their Windows workstation and that authentication carries through to the SQL Server.

* To allow access to only certain columns (or certain rows, for that matter), you can link to a View rather than a Table, and this view only has the desired columns.

* Unless you are a very experienced programmer you don't want a form to give rights; rather you put people in certain Active Directory groups (e.g. Administrators, Assistants) and they get the rights given to those groups.
