  • Use Remote Interfaces  -  Use SOAP, Web DAV, the Client Object Model or SharePoint Designer interfaces to access the Web site.

  • Open  -  Allows users to open a Web site, list, or folder in order to access items inside that container

zhang_yi say:

hi cagan,

based on my test, only a user who has at least the read permission can sync the library to local.

in addition, sync a library to local is related to the open items permission. if you want to give a user the view only permission and let him be able to sync library to local, please follow the steps below:

1. click > site settings, in the user and permissions section, click site permissions.

2. click permission levels in the ribbon.

3. in the opened page, click view only.

4. click copy permission level in the bottom. this will create a copy of the view only permission level.

5. input the name of the copied permission level, such as sync library, check open items in the list permissions section and click create in the bottom.

6. then you can assign this group the sync library permission for the site. you can create a new group and assign the users in this group permission which you created

7. then the user can sync the library to local, but he won’t have as much permission as the read.

note: the sync button will not available in the page. to sync library to local, please right click the onedrive for business icon, click on sync a new library then paste the library url as the following shows.

meanwhile, you can edit permission for a library as well as edit it in the site level. a library permission can also be a standalone part. for your reference: edit permissions for a list, library, or individual item.
