Here's a link to a theme and template set up to use Times New Roman as the Normal style, with character styles in the Quick Style Gallery for Greek and Hebrew: Theme and Template Download

The theme should be copied to Library/Group Containers/UBF8T346G9.Office/User Content/Themes/Theme Fonts.

The template should be copied to Library/Group Containers/UBF8T346G9.Office/User Content/Templates. You can then create a new document from it in Word by choosing File>New from template...

john korchok say:

In this instance, create a character style with the font set to TekniaGreek and set the color to blue. The base font of your paragraph is Times New Roman. When you apply any kind of style, local formatting is often over-written, including your font choice, in this example. The font reverts to Times New Roman, as in the surrounding text.

From your sample, I can see that you're used to applying local formatting (selecting text, then using Home>Font or Home>Paragraph to change appearance). Style-based formatting works more from the bottom up: first you create or find a theme that has your two favorite fonts. Next, you create styles based on those fonts. Finally, you select text and apply styles to the selected text. Ultimately, one has to switch between the two types of formatting, because the work method is different. In the long run, using styles exclusively is less work and allows you to reformat documents much more quickly.

Small print output when printing .DOC (97-2004) within Word 2016 for Mac

Print output is very small when opening old .DOC (Word 97-2004) files - it’s as if the scale was reduced to ~10%.

We plan on deploying Office 2016 for Mac at our school but this could be a deal breaker as we have thousands of older DOC files.

These older .DOC files were printing fine on Microsoft Word for Mac 2011.

Has anyone else stumbled upon this issue?

Workarounds I have found below:

Workaround #1:
1. File -> Page setup.
2. Click OK to confirm existing settings (no changes required).
3. File -> Print
4. Print output is now be normal.

Workaround #2:
1. File -> Convert Document.
2. File -> Save As... and save as a .DOCX (now pre-selected)
3. Close the Word Document.
4. Open the new .DOCX file.
3. File -> Print
5. Print output is now normal.

Workaround #2 is probably the best long-term but I haven't been able to find any tools available to bulk convert .DOC to .DOCX.

Any help would greatly be appreciated!

john mcghie say:

I agree that converting to .docx is your best solution long-term.  The files are more robust, and take half to a quarter of the disk space.

There are no tools available to do a bulk conversion yet, simply because nobody has yet built one for Word 2016.

Part of the reason for that is that most users soon discover they do not really need to convert in "batch". If the file they open in Word 2016 can be upconverted, the label "[Compatibility Mode]" will appear in the title bar, and the command "Convert document" will appear in the File menu.

If the users click "Convert document" they will be prompted to save the result in .docx format.  You could simply tell your users to do this every time they use an old .doc

If you have access to a PC and a user with a little technical knowledge, Microsoft produced a batch converter: Instructions are here:

http://blogs.msdn.com/b/ericwhite/archive/2008/09/19/bulk-convert-doc-to-docx.aspx

You may have to hunt for the downloads you need.  One possible source is here:

https://technet.microsoft.com/en-us/library/cc179019%28v=office.14%29.aspx?f=255&MSPPError=-2147217396

Hope this helps

Function dayscount(Firstdate As Date, Lastdate As Date)

    Const lastWEday = 7 ' Saturday
    Dim x As Long
