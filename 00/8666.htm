don cuthbert say:

Windows 10 upgrade will not activate by Product Key.  Entering a Product Key at any time may actually prevent upgrade activation.  Auto-activation should happen when connecting to the internet after the upgrade, based solely on it being an upgrade from an activated, qualifying OS.

There are reports of Activation server issues also.  It may be that after some time it will auto-activate, since you did upgrade from a qualifying activated OS.

If you prefer not to wait, I would download and create the ISO using the Media Creation Tool.  I believe with the ISO, you can upgrade and activate from Windows 7, even if it is not upgraded at all.  Simply install Windows 7, activate it with your Product Key and then mount the ISO from a USB flash drive to run setup.exe from within Windows 7.  Do not enter any Product Key whenever asked for one.

After successful upgrade and activation, you can clean install on that same device any time in the future.

don cuthbert say:

The Product Keys are generic and the same for all Windows 10 Home and another generic one is used for Windows 10 Pro.  Activation is by upgrade, not by Product Key.

To successfully upgrade from Windows 7, the easiest way is to install and activate Windows 7 Home.  Then use a Windows 10 Home USB flash drive, created with the Media Creation Tool for Windows 10.  Run the Windows 10 setup.exe from within Windows 7.  Choose to keep files and settings to ensure an upgrade is performed and not a clean install.  Do not enter any Product Key during or after the upgrade when asked.  Skip these screens.  Windows 10 should auto-activate after the upgrade, as soon as it connects to the internet.  Now your device is registered as a Windows 10 device and clean installs of Windows 10 Home can be done if desired.

You could do this on a small dual-boot partition with an activated Windows 7 install if you wish to test it first.  The Media Creation Tool says that you need to choose Upgrade this PC now for it to upgrade properly.  This way would eliminate a lot of potential errors during the upgrade that would prevent activation, but it is not impossible to do it right by running setup.exe instead (as long as you have created the right ISO).

Importing multiple DAT files into access data base - Microsoft Community

jjrak say:

Importing multiple DAT files into access data base

I have 211 files  in separate folders that I want to import to access. These are DAT files that open with Note Pad.  I want to select the columns I want to be imported as separate fields in Access and I want to save the script so I don't have to redefine that process for each of the 211 folders. Can you show me how to do this?

anthony.b say:

One way to do this, is to save the import steps, like this:

[1]  Import one of the DAT files into Access, using "External Data -> Text File".

[2]  After the import is done, check the box for "Save import steps".

[3]  For additional imports, click on "External Data -> Saved Imports".

Another technique:

[a]  Create a new Macro.

[b]  In Macro Design View, click the "Show All Actions" button.

[c]  In the "Add New Action" box, select "ImportExportText"

[d]  Fill out the required fields.

Yet Another technique:

[i]  Create a new Macro.

[ii]  In Macro Design View, click the "Show All Actions" button.

[iii]  In the "Add New Action" box, select "RunSavedImportExport".

NOTE: An advantage of using a macro instead of "External Data -> Saved Imports" is that the macro uses one less step.  In other words, when you run your macro, it will start the import right away, instead of having to navigate through "External Data -> Saved Imports" each time.
