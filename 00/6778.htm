Unfortunately my original question was edited, so it no longer makes any sense.  I provided sample, completely fake telephone numbers and person names but they have all been removed.

 

To answer your question, if the number stored in the contact is eight (8) digits, and the network transmits eight (8) digits, then the phone will match the incomming call/SMS to the contact and display a name rather than a number.

 

Unfortunately, it is not practical to store two (2) numbers for every person.  This also creates an issue that certain applications only want to interact with the phone number listed as the 'Mobile' number so there would be issues with where to locate the two (2) different version so of the same number.

 

Again, the strange thing is, Nokia Lumia 900 handets, running what should be a similar OS, do not have difficulty with matching partial numbers, unless of course you have two (2) contacts with different country codes but the same last eight (8) digits, then the phone just randomly picks one contact, but this is a common behavior across all mobile Operating Systems.  Windows Phone on HTC is the only OS I've ever encountered to have this issue.  There was an issue with Windows Mobile 6.1 that we had to change the regional setting to Australia to make it work, but I've tried all different combinations of regions and that doesn't seem to change anything.

 

It is common for networks to transmit only 8 digit numbers for calls between users on the same network.  When a call/SMS is received from another user on my network I will receive 8 digits, and when a call/SMS comes from any other network I will receive the full E.164 international number beginning with the plus ('+') followed by the country code and, in Hong Kong where we have no area codes, the 8 digit local number.
