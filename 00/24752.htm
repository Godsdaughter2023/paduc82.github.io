You're not alone. There are many different views in Outlook and each sorts differently, so it's hard to keep track of them all. You used the term "OAB" in one of your posts. Other views sort by whatever field you specify for them if you customize the view. By default most of them sort by the File As... field. So for a view to sort "correctly" the end user must select both the correct field for sorting and the correct format for that field. 

To add to the confusion, the File As... field often gets changed when users migrate their Outlook data incorrectly from one version to the next or if they run syns with PDA's.

To do so within a forum, however, requires knowing the precise view in question, the field chosen for sorting, the format of that field in the user's existing data file, and whether any syncs or data migration has occurred. It is not "rude" to ask for that information. It is a necessity.

Normally it is not necessary to select more than one field by which to sort. Doing so will rarely if ever improve your results.

bob jones aka: cybertaz say:

Just to clarify, I believe you're referring to Cell Borders rather than gridlines... the color of gridlines isn't tied to Themes, but Borders are.

For better or worse, that's what Themes are all about. Each Theme uses a specific set of 8 colors along with 5 shades (for a total of 48) of each which categorically are assigned to fonts, borders, fills, etc. When you select one of them you aren't just picking the color, you're actually picking a number [1-48]. If you pick the 3rd color in the Theme, for example, switching to a different Theme automatically will apply that Theme's "color #3".

To apply a color that won't change use one of the Standard colors or a color/shade that isn't included in the current Theme Colors... That breaks the link between the element & the Theme.

manuelcapiro say:
Hello YIANISARCHONTIDES,

In the case of IMAP accounts, Outlook saves a sent message in the IMAP account's Sent message folder and your IMAP mail provider may also save a copy, which result in two messages in the Sent Items folder every time you send a message.

I will suggest you to change the IMAP Account's Sent Options

First, check your settings for saving Sent items. Open File, Account Settings. Double click on your IMAP account then click More Settings. In Outlook 2013 and 2016, look on the Advanced tab. After uncheck where it says "Do not save copies of sent items" (I will leave you attached an example).

Also Change your Outlooks global options

If you have only one email account in your profile, you could use the option to not save sent messages. This is in File, Options, Mail, in the "Save messages" section, uncheck the option to Save copies of messages in the Sent folder.

Note that this setting will apply to all accounts - if you have more than one email account configured, you will not save sent items for any account.

For this reason, you need to be careful using the global setting in File, Options as not all mail servers will save a sent item for you when sending. It's better to disable the option to save sent items on the More Settings, Sent folder tab.

Hope this helps you, have a good day
