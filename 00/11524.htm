Sure. You can group them together.

To select all the layered text boxes you can lasso them to select them all. Click above and outside of one of the top corners of the largest text boxes. Hold down your left mouse button and drag down to opposite lower corner and release the button. You have just lassoed/selected all the elements within. You can use this if you want to add another text box to an existing group of text boxes. Just drag the new text box into place, Ctrl key and left click the group and the new text box. and then group them.

One final tip: CTRL+A to select everything on a page.

daniel pineault say:

You're going to have to use some VBA.  Using the form's current event, as well as the click event for each of the Yes/No fields.

The basic syntax for controlling the visibility of an individual Tab Page is:

Me.YourTabPageName.visible = True 'Make it Visible
Me.YourTabPageName.visible = False 'Hide the tab page

 

So you'd have to construct a basic logic, maybe something like:

    'Start of by hiding each of the tabs
    Me.[course enrolment].visible = False
    Me.[attendance record].visible = False
    Me.[salary record].visible = False
    Me.[Qualification].visible = False

    'Check each criteria and make the appropriate tab pages visible
    'Students
    If Me.Student = True Then
        Me.[course enrolment].visible = True
        Me.[attendance record].visible = True
    End If
    'Tutors
    If Me.Tutor = True Then
        Me.[salary record].visible = True
        Me.[Qualification].visible = True
    End If

I hope this helps.

The bitness of your PC does not dictate the bitness of MS Office to install.  Thus just because you have Windows 64bit does not mean you should be installing MS Office 64bit.

Typically, it is a bad idea to install 64bit MS Office as it requires changes to any APIs you use in code and certain ActiveX controls are incompatible, ...  As such, it is recommended to install the 32 bit even on a 64 bit system and you will have no issues by doing so.  The only application that may benefit from the 64bit is Excel, and this is for very complex workbooks (not typical users).

That all said, if you don't use VBA code and/or ActiveX controls, there should be no difference to you one way or another.

You also have to keep in mind that the typical install is 32bit, so if you start working in 64bit and then e-mail a database to another person running 32bit they could encounter issues unless you code things carefully. 

Your IT department should be well aware of this.  See if they are will to install the 32 bit, I think it could avoid certain potential problems.

Things to consider when your Full Scans seem outrageously long:

Full Scans are paused when the System idles to sleep (after 30 minutes by default), and are not resumed until you wake up your PC:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/defender-full-scan-still-running-after-22-hours/cb69f242-a673-4dee-837d-317785a1b57a

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-can-i-run-windows-defender-walk-away-and-have/7b57798e-b2da-40a6-9c74-afd9ff2dff11

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-do-i-do-a-set-and-forget-full-scan-with/a1fb5e32-f227-42b6-b455-05ae41f40bf6

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/initial-scan-with-windows-defender-in-windows-10/eb9efd51-7a74-4f50-8a40-9cde2db5c25c

Running Quick Scans or Custom Scans will help build up Defender’s cache of trusted files and shorten up your Full Scans when the cache is reset by an OS Update:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-10-1511-10586-windows-defender-full-scan/436b7fd5-8e87-46c4-99fa-db4a0cf09fc1

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/wd-scan-still-running-after-more-than-10-hours/4d34fbbc-c459-46b4-8378-8bc9843420a7

Scans will be slowed (and CPU utilization will be elevated) by any other real-time security app, or remnants thereof. Make sure that you’ve run the cleanup utilities for any previously installed, or preinstalled, AV apps:

https://answers.microsoft.com/en-us/protect/forum/mse-protect_start/list-of-anti-malware-product-removal-tools/2bcb53f7-7ab4-4ef9-ab3a-6aebfa322f75

Uninstalling unused programs, deleting unused files, and using Disk Cleanup will reduce the workload for your scans and shorten their run times.

In addition to physically removing files, you can further reduce the scan’s workload by excluding files. You can add your personal work files, media files, archived backups, disk images, etc., to the list of trusted files simply by creating either path or type exclusions for them:

https://support.microsoft.com/en-us/instantanswers/64495205-6ddb-4da1-8534-1aeaf64c0af8/add-an-exclusion-to-windows-defender

Turning off Archive Scanning will exclude most types of archive files in one simple step, and it will have a pronounced effect on the run times for your Full Scans:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-can-i-exclude-archive-files-for-a-scan-in/2d515e9d-a0cb-4759-8cb5-9fd7f5cc84ca
