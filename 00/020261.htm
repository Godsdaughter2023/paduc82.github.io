jeeped say:

Here's another,

The standard formula in G6 is,

=INDEX($C$2:$C$99,MIN(INDEX(ROW($1:$98)+((($A$2:$A$99<>E6)+($B$2:$B$99<>F6))*1E+99),,)))

Note that the ROW(1:98) is offset from $C$2:$C$99. It represents the position within $C$2:$C$99 and not the row number on the worksheet.

In case you want to avoid error codes when there is no double match, wrap the formula in IFERROR() and supply and empty string (e.g. "") as the default.

=IFERROR(INDEX($C$2:$C$99,MIN(INDEX(ROW($1:$98)+((($A$2:$A$99<>E8)+($B$2:$B$99<>F8))*1E+99),,))),"")

a. user say:

Hi Bernard, What Chip Pearson tries to explain (and personally I believe he did a good job of it) is that if you try to count the number of cells that are a specific color when using the FormatConditions property, you are likely to get a false reading.  The VBA property for FormatConditions will count the conditional setting whether or not it has actually been activated.  So, what Chip suggested, and provided the code to do, was to test for the conditions that would trigger the conditional format.  See Gord's post for the link.  Basically, you cannot just count the colors set by Conditional Format.
