hansv mvp say:

You can use the INDIRECT function. Let's say you have a formula

=AVERAGE(Sheet2!C1:F1)

Change it to

=AVERAGE(INDIRECT("Sheet2!C1:F1"))

This will keep on referring to C1:F1 on Sheet2 even if you insert or delete columns.

One reason Internet Explorer opens in a small window is because it remembers the window size and position from the last time you closed it. So if you close it and it’s not maximized, the next time you launch it, it won’t open maximized.

Can't run 2 different documents in Word 2003 simultaneiously. - Microsoft Community

bob is interested say:

I'm pretty sure I used to be able to view different documents with Word 2003 at the same time.  Now I can't.  If I open a second one it opens on top of the first one and I can't view the first one.  I must have changed some setting but I have no idea which.  I looked at Options but didn't see anything relevant.  Running on XP SP3.
suzanne s. barnhill say:

Word has two different ways to behave. Earlier versions of Word opened all documents in a single window. This is referred to as a Multiple Document Interface (MDI). With Word 2000, Microsoft introduced the Single Document Interface (SDI), in which each document opens in its own window. In Word 2003, you have a choice, and the setting is on the View tab of Tools | Options: Windows in Taskbar. With this option checked, each document appears in its own window and consequently gets a separate Taskbar button.

 

That said, if you are running Word maximized, you will still be able to view only one document at a time because each window fills the screen. You can manually Restore the documents to a "Normal" window and then arrange them on the screen as desired. Alternatively, if you want to view two documents (or two windows containing the same document) side by side, you can open the two windows and then, on the Window menu in one of the windows, click Compare Side by Side With... and select the other window. Word will then allocate half of the entire window space (or the entire screen if you have Word maximized) for each document. If they're copies or versions of the same document, you can opt for synchronous scrolling or turn that off.

 

For ordinary documents that you just want easy access to, you can arrange the Normal windows on your screen as desired so that you can select the one you want; you can also click on the Taskbar button for a given document to bring it to the front.

lacey c. say:

Automatically Pull Data From Another Sheet - Excel 2016

On that list, there is the project number, name, contact info, and then a column for "active (A)" or "closed (C)". 

The second sheet of the workbook is a list dedicated to ONLY the active projects. Ideally I would like for the data to pull automatically to this second sheet any time I enter in a new project onto the master sheet with an "A" in the active/closed column.

So, for example - if I enter in a new project on the master sheet, and put an "A" in that column, the corresponding number, name, and contact info would automatically copy over to that second sheet.

I believe the formula I'm looking for would be similar to VLOOKUP, although since I am only using one identifier instead of unique identifiers per project, I don't think VLOOKUP would work for me. 

Does anyone know if what I'm looking for is possible, and if so, how to do it? 

jasles say:
You can do that with VBA. Here is some VBA that can work for you. You'll just need to change the sheet names and it looks for the value of A in column D so you may need to change that too. 

Sub copy()
Dim lastRow As Long
Dim counter1 As Long
Dim counter2 As Long
Dim cell As Range
counter2 = 1
Application.ScreenUpdating = False
Sheets("Sheet2").Select
    
Cells.Select
Application.CutCopyMode = False
Selection.Delete Shift:=xlUp
Sheets("Sheet1").Select
With Sheets("Sheet1")
    lastRow = .Cells(.rows.Count, "A").End(xlUp).Row
    For counter1 = 1 To lastRow
        If Cells(counter1, 4).Value = "A" Then
            counter2 = counter2 + 1
            rows("" & counter1 & ":" & counter1).copy
            Sheets("Sheet2").Select
            rows("" & counter2 & ":" & counter2).Select
            Selection.PasteSpecial Paste:=xlPasteValues, Operation:=xlNone, SkipBlanks _
            :=False, Transpose:=False
            Range("A1").Select
            Sheets("Sheet1").Select
        End If
    Next counter1
End With
Sheets("Sheet1").Select
Range("A1").Select
Application.ScreenUpdating = True
End Sub

albert d. kallal say:
It is not quite clear what you are looking to do.
 
You can have 1, 2, 4 or even 5 copies of Access, or word or excel launched, and you can simply switch between each copy of word, or each copy of access running by selecting each different copy running by using the windows task bar.
 
If you are talking about access 2007, or access 2010, then the default window mode is tabbed, but if you go file->options, then choose current database, you can then choose overlapping windows. This would allow you to view multiple tables and multiple forms within the CURRENT database.
 
I am suspecting you are asking about viewing and switching between multiple windows and multiple tables in a single database (and you are NOT asking how to view/use multiple copies of access at the same time). The basic general operation of windows has ALWAYS allowed one to have many copies of word + excel + access + web browsers open, and the means by which you switch between 3 copies of the browser, or 3 copies of Excel, or 3 copies of Access running is all the same process. 
 
Note that to launch multiple copies of the SAME database in access, you have to launch access first (and NOT click on the database in a folder). You then choose the file->open, or file->choose recent database, since clicking on the actual file will cause an existing copy of that database to be maximized, and you not see/get another copy of access running in that case.
