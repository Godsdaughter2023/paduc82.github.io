jim gordon mvp say:

Hi

Excel 14.7.7 is the final update for Office 2011. Moving it off the startup disk was a mistake. Office 2011 and 2016 do not interfere with each other. Office 2011 should be installed on the startup disc. The Microsoft Office 2011 folder belongs in the Applications folder, nowhere else.

Excel on line should have no trouble opening and displaying workbooks saved in .xlsx or .xlsb file format. Using any other format (such as older .xls) may cause difficulty.

I suggest removing the install of Office 2011 from the external drive, then reinstall with the final build.

You can download the latest full and final Office 2011 install (which is build 14.7.7) from https://go.microsoft.com/fwlink/?linkid=273962

Doing this is likely to reset the file associations from 2016 to 2011. You'll have to manually reset the file associations in Finder.

Make sure that the workbook files do not have special characters in the file names, nor in the names of the sheet tabs. Slashes, asterisks, + - and other symbols are operators that cause trouble if used in file names or sheet tab names.
