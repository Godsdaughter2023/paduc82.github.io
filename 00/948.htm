I think this has to do with the way Excel stores decimal numbers with a binary representation meaning that not all numbers are stored with their exact value.

In Excel 2007 part of your result column, column C, will also contain some non-zero numbers.
For example =MOD(0.15,0.05) is shown as -1.38778E-17, yes it is negative!
But =MOD(3*0.05,0.05) gives +1.38778E-17 as the result, so obviously 3*0.05 is not the same as 0.15 when caculated inside the MOD function.

My recommendation is to not use MOD with decimal numbers.
Transform the numbers to the integer domain while using MOD and then transform back, like this.

MOD(0.3,0.05)=MOD(ROUND(0.3*100),ROUND(0.05*100))/100
