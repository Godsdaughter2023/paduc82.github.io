If you don't need all the generalities Duane posted, you can get away with just this kind of thing.

In frmVendorInfo's button's click event, change the OpenForm line to something like:

    DoCmd OpenForm "frmPaymentInfo", _
                 WhereConditiom:= "VendorID=" & Me.cboVendorID, _
                 DataMode:= acFormAdd, _
                 OpenArgs:= Me.cboVendorID

acFormAdd indicates that new records can be added, but existing records can not be edited. If that's not what you want, use acFormEdit instead.

    Me.VendorID.DefaultValue = """" & Me.OpenArgs & """"

Does Microsoft Security Essentials replace Windows Defender? - Microsoft Community

abrahamchapman say:

Does Microsoft Security Essentials replace Windows Defender?

Windows Defender says that is defends against spyware.
Microsoft Security Essentials says that it defends against spyware and viruses?

Does this mean the Microsoft Security Essentials replaces Windows Defender?
Or should I run both?

Also, what would happen if I were to run both at the same time?

dr. strangelove say: Windows Defender is spyware protection only.

Security Essentials is full anti-malware protection.

On XP, Security Essentials will uninstall Defender.

On Vista and W7, Security Essentials will disable Defender.

No, you should not run both.

Also, if you have any previous anti-malware installed (norton, mcafee, etc), you must uninstall those products before installing Security Essentials.

Running more than one of any real-time product concurrently will cause performance issues and system instability.
