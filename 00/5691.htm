Ahhh... I misunderstood. Apparently you mean what appears to be a blank line separating the Rule 1 paragraph & the bulleted paragraph :-)

Whether that 'space' is selectable is determined by how it was created. In order to help make that determination it is immensely helpful to display the non-printing characters (¶) by clicking the button bearing that symbol on the Standard Toolbar or by using the Command+8 keystroke.

If the line is selectable there will be a mark on that line - either ¶ [indicating a paragraph break] or  ↵ [indicating a line break]. These can be deleted by the same methods as deleting any other characters.

OTOH, if no symbol appears it means that the spacing has been created as a paragraph formatting attribute. If that isn't where the problem lies move your insertion point into the bulleted para then check the settings for it  - only your concern will be for After. This also could be a matter of how the Bullet Style is defined.

hansv mvp say:

The following code is from Greg Maxey, I only changed it to take a document object as argument.

Sub myUpdateFields(doc As Word.Document)
Dim rngStory As Word.Range
Dim oShp As Word.Shape
For Each rngStory In doc.StoryRanges
    Do
        On Error Resume Next
        rngStory.Fields.Update
        Select Case rngStory.StoryType
            Case 6, 7, 8, 9, 10, 11
                If rngStory.ShapeRange.Count > 0 Then
                    For Each oShp In rngStory.ShapeRange
                        If oShp.TextFrame.HasText Then
                            oShp.TextFrame.TextRange.Fields.Update
                        End If
                    Next oShp
                End If
            Case Else
                'Do Nothing
        End Select
        On Error GoTo 0
        'Get next linked story (if any)
        Set rngStory = rngStory.NextStoryRange
    Loop Until rngStory Is Nothing
Next rngStory
End Sub

To update the fields in your document, replace the lines that update the fields and show/hide print preview with

Call myUpdateFields(objWord.Documents(1))
