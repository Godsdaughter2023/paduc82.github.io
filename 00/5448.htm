lowthers say:

Found the resolution for this. It is specific to MDaemon, Outlook Connector for MDaemon (MAPI) and upgrading from Outlook 2010 (or earlier) to Outlook 2013 or Outlook 2016.

Historically MDaemon had an auto-whitelisting feature where when you sent an email the recipient was added to your default contacts folder. MDaemon checked your contacts folder when an incoming email arrived for you and if the sender existed in it, that email was not passed through the Spam Filter since you’d sent to it in the past.

When mobile device support arrived, that function was moved to a dedicated “Whitelist” contacts folder as otherwise, you’d end up syncing 1000’s of contacts to your mobile device that you may have only sent to once.

To clean up the contacts in the original default contacts folder there is an option in "Accounts -> Account Manager -> {username} -> Whitelist -> Remove contacts which are missing name or phone data". Following that remove the individual .msg files from the affected users Contacts.IMAP folder, but leave the AddrBook.mrk and Imap.mrk files in place.

Following that re-open Outlook, which will regenerate the .msg files based on the contents of the updated AddrBook.mrk file. Once this is complete Outlook has run fine again.
