Mail merge does not give two hoots about the type of data that you are merging, or whether there are duplicate records.

By saying that you are able to get the page layout and the margins all set up, is it correct to assume that you are not starting with one of the label formats that can be selected in the Label Options dialog that appears when you go to Mailings>Start Mail Merge and select Labels...?

If that's the case, after using the Select Recipients facility to select the data source (your Excel Workbook, you will need to use the Insert Merge Field facility to insert the field that contains the numbers into the first cell of the table and then use the Update Labels facility to replicate the field set up to the other cells of the table which will also cause a «Next Record» field to be inserted before the merge field in all of the labels on the sheet except for the first label.

ossiemac say:

It appears that the Last 7 days, last 30 days etc is on another worksheet. I don't understand the 2 columns for each header. However, in cell B17 (Under first column of "Last 7 days") the following formula where "Sheet1" is the sheet depicted with the dates for the year and the "X"'s.

=COUNTIFS(Sheet1!$B$2:$NC$2,">="&TODAY()-7,Sheet1!$B5:$NC5,"=X")

Note the absolute referencing (with $ signs) for the first part of the formula for both column Id and row Id. For the second part of the formula only the column Id has the $ sign.

You should be able to copy the formula across in row 17 and then edit the formulas in each column and change the number of days to subtract from Today(). You can then select all of the formulas across row 17 and copy down.

sort by day of week in order - Microsoft Community

laurajane22 say:

sort by day of week in order

Hi:

Does anyone know how to sort by day of week in order? I have a small report that shows classes occurring on Tuesday, Wednesday and Thursday. When I sort, I can't get it to appear Tuesday, Wednesday and then Thursday (it puts Thursday, Tuesday and Wednesday because it's doing it in alphabetical order). Any ideas?

john p spencer say:

Several ways.  Use an expression for the sort.  In sorting and grouping dialog of the report set the sort as

     =Instr(1,"SuMoTuWeThFrSa",Left(DayName,2))  
which will return numbers from 1 to 13 and you can sort on that

Create a table with the names of weekdays and a second column for the sort order.  Add the table to the underlying query joining the table to the present weekday values and return the sort order column.

If you have a date field and are using format (or some other means) to get the day name, then either sort by the date field or get the weekday of the date and include it in the fields you return
