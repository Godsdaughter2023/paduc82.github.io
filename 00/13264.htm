porego say:

Hooray!!  I've solved my problem.  I'll describe what I did, in case my solutions is of any benefit to anyone else.

 

The solution has 3 parts: 1) Fixing the mapi32.dll file, 2)  Enabling PC Sync in the security agent installed in my PC, and 3) Re-configuring PCSync.

 

 PC Sync communicates with Outlook through a file in the Windows\system32 directory named "mapi32.dll".  Other programs use the same .dll file.  I have been using an e-mail program that has its own version of mapi32.dll which does not work with PC Sync, so I used to have to shut down my e-mail program in order to use PC Sync.  The e-mail program places it's own mapi32.dll file into the Windows\system32 directory when it starts, and then restores the original mapi32.dll file when it closes.  The e-mail program's .dll file got left in-place once when the e-mail program crashed.  The fix:  Go to Start -> Run, enter "fixmapi" (without quotation marks) into the Run window, and click "OK".

 

With the mapi32.dll problem fixed, a message started to appear saying  I was not authorized to access the outlook.ost file.  The fix was to tell the security agent that Nokia PC Sync is a "trusted application".  There was also an program in the "common" directory that is called by PC Sync that also had to be tagged as a "trusted application".  Also, disabling the security agent completely solved the problem, but only until the security agent re-enables itself.

 

The last step was to  go into PC Sync, click the Settings button (looks like a spanner wrench) and select the "Microsoft Outlook" tab.  Then click the "Select Folders" button.  Even though the "folder selection" window shows the correct folders for each component (Calendar, Tasks, Contacts and Notes) I had to click the"Browse" button (looks like a file folder) for each component and re-select the appropriate folders.

 

 Now, the added benefit of all this is that I've given my e-mail program a copy of the working mapi32.dll file, so PC Sync now runs whether or not the e-mail program is running.

Try formatting the cells as General. If you want the formula in A1 reference mode within your VBA try,
    ActiveCell.Formula = "=NETWORKDAYS(G2,H2,$S$1:$S$16)-1"
