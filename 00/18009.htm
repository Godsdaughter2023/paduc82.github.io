Actually, what you have should work.

 

Just keep in mind that a ribbon loaded for a form (or report) will NOT effect back stage. So, customizations will have to go into the system wide ribbon. So, this means the ribbon you specify in the file->options->current database->Ribbon and tool bar options.

 

It works for me. I also assume you have user interface errors on? And, remember, you have to use the new schema, but you would be seeing errors if you don't. So, don't use :

 

<customUI xmlns="http://schemas.microsoft.com/office/2006/01/customui">

 

It will not work, but using

 

<customUI xmlns="http://schemas.microsoft.com/office/2009/07/customui">

 

Will work.

http://www.accessribbon.de/en/?FAQ:FAQ_Access_2010:2

http://www.accessribbon.de/en/index.php?Access_-_Ribbons:Ribbon_XML___Controls:Backstage_View

   <buttonidMso="SaveObjectAs"visible="false"/>
   <buttonidMso="FileSaveAsCurrentFileFormat"visible="false"/>

   <tabidMso ="TabPrint"visible="false"/>
   <tabidMso ="TabShare"visible="false"/>
   <tabidMso ="TabHelp"visible="false"/>
   <buttonidMso="ApplicationOptionsDialog"visible="false"/>
   <button idMso="FileExit"visible="false"/>
  </backstage>
 
</customUI>

albert d. kallal say:

Actually, in the last 15 or so years, I can't recall one application that I have ever installed on my computer that did not have the file command. I guess what I'm saying is you're not going to shock or scare your users by having that file command. You can hide everything in backstage and RE purpose a exit command to do what you want (but, exit likely just fine). Or set it as close for when viewing reports.

 

This is really much like copy or paste, these are just features that people expect in software and your users will likely use some of these features without even giving it a second a thought. In fact in general the users mental model has become so accustomed to this feature they just click on it to exit your application without so much as giving this a second thought. I mean they do it all day and in all the other applications anyway, you might as well utilize they memory motor muscle they are building up by using these other applications all day long.

 

If you're looking to not really follow any kind of traditional user interface guidelines at all, then just execute one line of code and you'll hide the whole ribbon, the QAT and also the file button in one shot with the following one line of code:

 

DoCmd.ShowToolbar "Ribbon", acToolbarNo

 

So the above will hide everything for you. You can also use the reverse of the above command of

 

DoCmd.ShowToolbar "Ribbon", acToolbarYes

 

for the case in which if you say just wanted to display custom ribbons ONLY for reports (I am considering that for one application right now).

 

However the instant you start putting in custom menus, you might as well go with the flow and not fight this general UI that much. Perhaps it would be great if we could hide the file button, but giving the whole situation some thought I can't really think of any application I've ever installed on my computer that did not have a file option anyway, and I can see some pros here as much as there are some downsides. A bit of flexibility in ones design really goes a long way in these situations.

Please always give the make and model of your device and the build number of your preview when posting.  Help us help you.  

Your Windows is activated by a digital license, not by a product key.  It is based on the activation at the time your first copy of Windows was installed.  You do not get a product key when you update from one build to the next.  You don't need one for that.  If you are being asked for a product key then you have lost Windows activation.  The first step is to run the Activation troubleshooter at

You will only see the highlighted item if Windows is NOT activated.  If this does not resolve the issue try upgrading to a later preview, such as 16278, by downloading the ISO file from

https://www.microsoft.com/en-us/software-download/windowsinsiderpreviewadvanced

to your desktop.  Right-click the ISO icon and choose "mount."  A file explorer window will open.  Double-click the last item, "Setup.exe" to start the upgrade.
