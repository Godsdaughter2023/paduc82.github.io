suat.cini say:

Hi.

I used to increase idle core clock and memory clock speeds to avoid TDR issue in my early AMD gfx cards ( HD5770 and HD6850 ). Nothing else would eliminate the problem. In those years both AMD and Nvidia had the same issues.

To summarize, gfx cards came with variable speeds for both core and memory clocks to conserve energy ( idle speed, middle speed and high speed for different workloads ). The gfx cards could not sustain operation while switching from idle to other speeds to compensate for changes in workloads because idle and middle speeds were too low for some systems, causing TDR.

You may follow the guide outlined here < LiNK >

Hope this helps.

Note1: If you increase speed from, let's say from 150 MHz to 400 MHz, you must add 00 to 400 MHz to make it 40000.

Note2: Do it at your own risk. You may increase speeds by small values until you find your card's sweet spot.

Note3: You may also increase the speeds by flashing the vBiOS of the card, a method which I did for my HD6850.

nick andre say:

Engineer on OneNote here: we've identified the following issue and can provide a fix:

  • Imported using Evernote importer
  • Notebooks can be opened in OneNote Online (WAC) but sections don't have color associated with them, cannot be reordered
  • Notebooks cannot sync to desktop client (Typical error code 0xE0000B65)
  • Basic troubleshooting steps (removing and re-adding the notebook to the desktop client) fails to restart sync

Warning: this fix will reset metadata contained in the TOC file, particularly section ordering will be reset to the original creation order. This should not be an issue for notebooks converted from Evernote, as they will return to the order in which they were created.

About this issue

The cause is a corrupted Table of Contents file (TOC file) embedded within the OneNote notebook. This appears to have been caused by a bug in the OneNote API call which was fixed last week, but can still affect notebooks created before that date.

The fix is to delete the notebook's TOC file, and then reopen the notebook in either OneNote 2016 for Windows Desktop or OneNote Universal app(designed for touch-enabled Windows 10 devices, including tablets and phone). This will trigger regeneration of the TOC file.

Fixing issue on OneDrive Notebooks (consumer)

  1. Visit OneDrive.com.
  2. Find the name of the notebook affected by the sync issue above.
  3. In the search box, type "NotebookName *onetoc2" and search
  4. You should see a file called "Open Notebook.onetoc2" -- right click and view details, confirming that the file is beneath your affected notebook and that it is of type onetoc2 (OneNote Table of Contents file)
  5. Right click and delete the file from this page.
  6. Return to the notebook file location in OneDrive, right click and Open the notebook with OneNote (on your local machine)
  7. Sync will restart -- best to leave OneNote running until it's completely synced.

Fixing issue on SharePoint/O365 (business customers)

  1. Visit your "my site".sharepoint.com
  2. Locate the notebook by browsing to it, but do not open it.
  3. Create a folder with the same name as the notebook in the same location, but add an extra 'f' character to the end: "NickTestNotebookf"
  4. Click to open that folder.
  5. Alter the URL to remove the extra f and hit enter -- you should now see the contents of your notebook folder.
  6. Locate the .onetoc2 file and delete it.
  7. Browse up to the notebook and right click to open with OneNote 2016 Desktop app.

We don't believe this fix works with the older OneNote 2010 app -- you need OneNote 2016 for Windows Desktop or for touch-enabled Windows 10 devices, including tablets and phone. Reply to this post if you have any problems.
