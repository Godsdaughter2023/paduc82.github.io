graham mayor say:

Maybe http://gregmaxey.mvps.org/word_tip_pages/vba_find_and_replace.html would help? If not, you can roll your own with a macro and a table

Create a document with a two column table. Enter the words you want to find in the first column and the formatted versions you wish to replace then with in the second column.  Save the file and put the filename and path where indicated.

Sub ReplaceFromTableList()
Dim oChanges As Document, oDoc As Document
Dim oTable As Table
Dim oRng As Range
Dim rFindText As Range, rReplacement As Range
Dim i As Long
Dim sFname As String
Dim sAsk As String
    sFname = "D:\My Documents\Test\Changes.docx"
    Set oDoc = ActiveDocument
    Set oChanges = Documents.Open(Filename:=sFname, Visible:=False)
    Set oTable = oChanges.Tables(1)
    For i = 1 To oTable.Rows.Count
        Set oRng = oDoc.Range
        Set rFindText = oTable.Cell(i, 1).Range
        rFindText.End = rFindText.End - 1
        Set rReplacement = oTable.Cell(i, 2).Range
        rReplacement.End = rReplacement.End - 1
        With oRng.Find
            .ClearFormatting
            .Replacement.ClearFormatting
            Do While .Execute(findText:=rFindText, _
                              MatchWholeWord:=True, _
                              MatchWildcards:=False, _
                              Forward:=True, _
                              Wrap:=wdFindStop) = True
                oRng.Select
                sAsk = MsgBox("Replace - " & vbCr & oRng & vbCr + vbCr & _
                              "with - " & vbCr & rReplacement, vbYesNo, _
                              "Replace from Table")

                If sAsk = vbYes Then
                    oRng.FormattedText = rReplacement.FormattedText
                End If
                oRng.Collapse wdCollapseEnd
            Loop
        End With
    Next i
    oChanges.Close wdDoNotSaveChanges
End Sub

doug robbins - mvp office apps & services (word) say:

Starting with a blank document, with margins of say 1 cm all around, insert a 3 row 2 column table.  Adjust the column widths to the widths desired and merge the cells in the second row.  If you want space for two lines of text in the top and bottom rows, press the enter key in each of those rows to increase the cell height.  Then drag the border between the second and third rows down so that the table fills the page.   Then via the Table Tools>Layout tab of the ribbon, Select the Table and then copy it by using CTRL+C.

Then, in the document that you want to set up this way, access the Header and use Paste>Special and paste the table as a Picture (Enhanced Metafile).  Then Right click on it and then click on the Wrap Text item and then select Behind Text.  Then drag the table into the required position and size to fill the page, leaving an appropriate amount of white space on all sides so that it appears as a border.

Then, still in the header insert the required text into the first row.  Then go to the footer, and insert the required text into the bottom row and for the page numbers type the words Page; and of and then insert fields for the page number and number of pages.

joeu2004 say: Michelle Lynne wrote:
I have a simple spread sheet calculating rate per mile and miles per gallon calculated in columns.  I then have totals at the end.  Instead of adding the columns then dividing, I used the same formula for the totals as I did for the individual columns.  However, if I do add the columns and divide by the number of columns, it does not come to the same number.  Why does this happen

Because mathematically, the sum of ratios (averages) is not the same as the ratio (average) of the sums, in general.  That is, (x1/y1 + x2/y2 + x3/y3)/3 does not equal (x1+x2+x3)/(y1+y2+y3).

(Except when y1 = y2 = y3.)

But which one you should use is not clear from your description.  You say you have rate per mile.  Rate of what?  You say you are dividing something.  What?

Mathematically, rate per mile __times__ miles per gallon would give you rate per gallon.  Whether or not that makes sense depends on what it is you want to compute.

Thank you for this pointer. Seems that this does, indeed, fix the problem (for me, at least).

For my setup (Windows 8.1, Surface Pro 3, Office 365 - Office 15) the specific instructions are few and simple:

1) Go to C:\Program Files\Microsoft Office 15\root\office15

2) Right-click on powerpnt.exe.manifest and change the permission so you can modify the file

3) Right-click on powerpnt.exe.manifest and open with Wordpad

4) Change the line

    <dpiAware>True/PM</dpiAware>
    to

    <dpiAware>False/PM</dpiAware>
5) Save file

(at least with Office 365, you don’t need to do anything in the registry as it’s already looking at the external manifest file.)

Yes, line 4 seems counter-intuitive, but it’s correct. Once you change this to False, Powerpoint will scale moving from high to lower DPI monitors. I’ve not encountered any other issues, but I’ve not used it for very long, yet.

Thanks to the previous pointers/authors for this tip.

(Do read http://www.danantonielli.com/adobe-app-scaling-on-high-dpi-displays-fix/ if you want more details and the history of this.)
