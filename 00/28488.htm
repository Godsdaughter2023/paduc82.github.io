rohn007 say:

I would sugge you keep Starter for a little longer. You are having a common problem with Windows File associations. All too often, for some unknown reason, Windows does not properly redirect the file associations to your installed applications.

Here are some tips for redirecting the file associations

Try repairing your Office installation to fix your file associations:

Repair or remove Office 2010 - http://office.microsoft.com/en-us/excel-help/repair-or-remove-office-2010-HA010357402.aspx?CTT=1

 

If that doesn't work, try:

 

Re-register the application

There are several different ways you can reset the proper file associations. Word itself provides a way to do it. Open a command prompt window and, at the command line, enter the following:

winword /r

Nothing seems to happen when you press Enter, but Word will reregister itself back in Windows. In the process, it resets the file associations for the files it uses, such as DOC and DOT files. You can close the command prompt window and double-click on a document; Word should start right up.

 

If that doesn't work, try:

 

Alternate Re-register the application

Close the app. Open the Run Window: <WIN> R, use the following command (Modify the Path to match your specific installation):

"C:\Program Files\Microsoft Office\Office\winword.exe" /unregserver

The quotes are necessary, but you may need to change the path so it reflects the location of the winword.exe file on your system. (You can use Windows’ Search tool to locate the path to the winword.exe file.) The /unregserver switch “unhooks” all the references to Word in the Windows Registry.

You should then, immediately, use the following command:

"C:\Program Files\Microsoft Office\Office\winword.exe" /regserver

This command causes Word to rewrite all its Registry keys and to reassociate itself with document and template files. Now you can double-click a document file and see if Windows starts Word and opens the document.

 

If that doesn't work, try:

 

Need to associate the file type with program:

In Windows 7:

  • Right-click the file,
  • Select "Open With" select [Your Program Here]
  • Check “always use this program”.

  • Click Start
  • type cmd in the run dialog box then press ok.

  • In control panel > folder options > file types > new (button) in the new window click advanced.
  • In the file extension dialog box type the file extension to fix, ie: exe.
  • In the drop down menu locate and select application > click ok.

robin hqd say:

Printing Excel Document Does Not Print Border Lines

I have recently converted from Office 2010 to Online 365. Documents that previously printed fine using Office 2010 are not not printing border lines in Office 365 Excel. I'm not referring to the gridlines in Excel. I'm specifically speaking about cells that have borders around them. 

NAME  TYPE OF  WORK    AMOUNT OF CONTRACT  TOTAL WORK COMPLETED TO DATE
       
       
       

So for example, in the document above, only the text prints. None of the lines. 

How can I correct this so the borders print? I cannot submit the document without the borders, as it will not be accepted by the party that needs this form completed.

SOLVED

This is, by far, the most messed up answer I could have ever imagined. After doing 2 hours of research on this forum, I came across a post from someone who said that selecting the hi-contrast desktop mode for Windows could affect the printing of borders in excel documents. I almost laughed when I read it. How ridiculous. I had tried several other suggestions - unchecking draft quality, updating printer drivers, etc - to no avail. As soon as I got rid of my hi-contrast desktop option, the document printed correctly. And BTW, the Office 365 tech team could not solve this for me after an hour of trying their suggestions. They urged me to opt for a paid tech support option to the tune of $499 (this is not a typo) in order to fix this - with no guarantee of fixing it. This in addition to paying MS an added subscription price for upgrading from desktop to online and never having it work correctly. MS must think we were all born yesterday..................how sad. Remind me to go back to my Mac when I get home from work.
