I'm assuming that you mean the Markup area, that is, the area for comment and revision balloons. To get rid of that area permanently in a document, you have to accept all changes and also delete all comments. You can do that via the Review tab on the ribbon.

On the other hand, if you are saying that you just once want to print a version of the document without markup, do the following: Click File | Print and then click to expand the Settings drop down, where you must deselect the Print Markup option. Finally click the Print button.

Access 2010, project, how to modify subreport server filter in vba - Microsoft Community

jn2827 say:

Access 2010, project, how to modify subreport server filter in vba

Running a report, I need to modify the server filter of a subreport within that report based on operator selections.  I can do this successfully on a form with the following syntax:

Forms!frmMain.frmSub.Form.ServerFilter = "FilterText"
Forms!frmMain.Requery

But trying to do it in the report vba:

Reports!rptMain.rptSub.Report.ServerFilter = "FilterText"

gives error 2455 "You entered an expression that has an invalid reference to the property Form/Report"

That occurs when the code is in the report open event.  Interestingly when I move the same code to the Format event I get a completely different error, 2101 "The setting you entered isn't valid for this property"

This is in Access project (.adp file)

jn2827 say:

Well, found one way to do it, open it in design mode in vba, make the change, then close and save:

        DoCmd.OpenReport "rptSub", acViewDesign, , , acHidden
        Reports!rptSub.ServerFilter = "FilterText"
        DoCmd.Close acReport, "rptSub", acSaveYes

Surely there's a better way than this???
