bill manville say:

Try this, which attempts to put the hyperlink base in front of relative hyperlinks:

Sub ProcessHyperlinks()

Dim R As Range
Dim WB As Workbook
Dim stBase As String
Dim stAddr As String
stBase = ActiveWorkbook.BuiltinDocumentProperties("Hyperlink Base")
Set R = Cells(Activecell.Row, "t")
Do Until R.Row >= ActiveSheet.UsedRange.Row + ActiveSheet.UsedRange.Rows.Count
    If R.Hyperlinks.Count = 1 Then
      stAddr = R.Hyperlinks(1).Address
      If Left(stAddr, 2)<>"\\" And Mid(stAddr,2,1)<>":" Then stAddr = stBase & "\" & stAddr
      If Dir(stAddr)="" Then
        MsgBox stAddr & " is not found", vbExclamation
      Else
        Set WB = Workbooks.Open(stAddr)
        R.Offset(0, -8) = WB.Sheets("Summary").Range("F9")
        R.Offset(0, -7) = WB.Sheets("Summary").Range("G14")
        R.Offset(0, -5) = WB.Sheets("Summary").Range("F8")
        R.Offset(0, -4) = WB.Sheets("Summary").Range("G13")
        R.Offset(0, -2) = WB.Sheets("Summary").Range("F10")
        R.Offset(0, -1) = WB.Sheets("Summary").Range("G15")
        WB.Close saveChanges:=False
      End If
    End If
    Set R = R.Offset(1, 0)
  Loop
End Sub

ron coderre say:

MS Access stores data in tables.
The tables contain fields (columns) of data.
Each of those columns can contain only one kind of data (text, numbers). 

So, you need to ensure that all of the Excel worksheets have the same data structure:
• Same column headings (Name, Phone, Acct, etc) in the same order.
• Each of those columns contains only one kind of data
...Names are all text, no number (Access will ignore the numeric values and not import them)
...Numeric fields contain no text (Access will ignore the text values and not import them)

Once the data is scrubbed, import one of the sheets of data into Access (from the File menu).
Then, you can copy each of the other data ranges (skipping the headings) and Paste.Append to the table.

Does that get you heading in the right direction?

shane devenshire say:

1.  When you import data into Access, Access looks for data in the top left corner of a spreadsheet or for range names.  I strongly recommend you name each of the ranges you are going to import (in Excel) and use those rather than sheet choice.  If you don't use range names there are possiblities that you will have problems importing the correct data especially if there is other unrelated data to the left or above the desired range.  Technically you can drag a range from Excel and drop it in Access table object area - which avoids some problems with regard to your Excel data layout.

2. All text fields that come into Access, regardless of the method will result in text field sizes of 255 characters, you should reduce these in design mode after importing the data.

3.  If you import a date column which contains even one text item the entire column will be imported as text.

4.  If you import a numeric column that contains even one text item the entire column will be imported as text

No data is ignored, it just comes in in as an incorrect data type.

It makes no sense to consider a VBA- you don't want to automate a task you will only do once and getting the data into Access sounds like a one time process.  Besides, if you are not an Access expert you don't want to even consider programming in Access.

If you want to export data to an Access table from an Excel worksheet, the macro example below shows how this can be done:

Sub DAOFromExcelToAccess()

' exports data from the active worksheet to a table in an Access database

' this procedure must be edited before use

Dim db As Database, rs As Recordset, r As Long

   Set db = OpenDatabase("C:\FolderName\DataBaseName.mdb")

   ' open the database

   Set rs = db.OpenRecordset("TableName", dbOpenTable)

   ' get all records in a table

   r = 3 ' the start row in the worksheet

   Do While Len(Range("A" & r).Formula) > 0

   ' repeat until first empty cell in column A

       With rs

           .AddNew ' create a new record

           ' add values to each field in the record

           .Fields("FieldName1") = Range("A" & r).Value

           .Fields("FieldName2") = Range("B" & r).Value

           .Fields("FieldNameN") = Range("C" & r).Value

           ' add more fields if necessary...

           .Update ' stores the new record

       End With

       r = r + 1 ' next row

   Loop

   rs.Close

   Set rs = Nothing

   db.Close

   Set db = Nothing

End Sub

The macro example assumes that your VBA project has added a reference to the DAO object library.
You can do this from within the VBE by selecting the menu Tools, References and selecting Microsoft DAO x.xx Object Library.

Summary

Microsoft recently released an updated build of Windows 10 Technical Preview. Build 9860 introduces new features such as the Notification Center, improvements to navigating Virtual Desktops and smoother window effects to name a few. In this Wiki article, we take a look at downloading and installing build 9860 on Windows 10 Technical Preview which is build 9841.

Please note, build 9860 is not provided as a .ISO file, it can only be installed from within build 9841.

Details

As much there are improvements in build 9860, its quite unstable and does not feature the polish of a milestone build such as 9841. I personally have seen some regression in areas such as Networking, performance, application compatibility and even sleep. If you are not interested in experimenting with such a early build, it is probably best you wait until the next major milestone which will be available to consumers in the near future.

Lets get started.

The first thing I recommend you do is run Windows Update first and apply the latest Windows Updates.

Restart if prompted.

Click Start > PC Settings or Click Start, type: PC Settings then hit Enter on your keyboard

Click Update and Recovery in PC Settings

Click Preview builds then click Download now to begin downloading the latest preview.

When the download is completed, you will be prompted to Install Now, proceed to do so:

When complete, you will be prompted to Install Now or Later. If you choose to Restart, your computer will go through a couple restarts. 

After logging in, you should see the new build number on your desktop. 

If you don't see this, then build 9860 was not successfully installed. If it is, go ahead and run Windows Update and install the latest updates to resolve issues with text rendering.
