You should probably think about creating a query for each type of summary.  Then you can use Sum and Group By for each specific summary type with fairly simple calculated fields.  After you get the data for each summary calculating correctly in its specific query, you can then create a report for each type of summary (based on the summary's query).

I really believe that breaking each summary into its own smaller problem makes everything a lot simpler.  The final step of combining them as subreports on a blank main report should be a near trivial exercise.

rich007a say:

Here's a macro that will do what you want:

 

Sub CountWords()
    Dim rng As Range, c As Range
    Dim x As Variant, vArray As Variant
    Dim lnLastRow As Long
   
    Range("B:D").Clear 'CLEARS COLUMNS B:D AT START!
   
    Set rng = Range("A1:A" & Range("A" & Rows.Count).End(xlUp).Row)
    Range("B1").Value = "Word List"
    For Each c In rng
        vArray = Split(c.Value, " ")
        For Each x In vArray
            Range("B" & Rows.Count).End(xlUp).Offset(1, 0).Value = x
        Next x
    Next c
   
     'Now copy the list and remove duplicates:
    Range("B:B").Copy Destination:=Range("C:C")
    Range("C:C").RemoveDuplicates Columns:=1, Header:=xlYes
   
     'Count the occurances of each unique word:
    lnLastRow = Range("B" & Rows.Count).End(xlUp).Row
    Range("D1").Value = "Word Count"
    Range("D2").Formula = "=COUNTIF(B$2:B$" & lnLastRow & ",C2)"
   
    lnLastRow = Range("C" & Rows.Count).End(xlUp).Row
    Range("D2").AutoFill Destination:=Range("D2:D" & lnLastRow)
   
     'Freeze the data:
    Range("C1:D" & lnLastRow).Copy
    Range("C1").PasteSpecial xlPasteValues
    Range("B:B").Clear
    Range("C1:D1").Font.Bold = True
    Range("C:D").EntireColumn.AutoFit
End Sub

Have your data in column A, starting at row 1.

Press Alt+F11 to open the VBE, then do Insert > Module, then paste the above code.  Switch back to Excel and do View tab > Macros > CountWords > Run.

ronrosenfeld say: I believe that error is appearing because of a limitation in the worksheet function TRIM..  Although this is not documented, there seems to be a limitation for the TRIM worksheet function (and other worksheet string functions) in handling strings longer than about 32767 (2^15 - 1).  Since the limitation of string length in VBA is 2^31, there is an issue if your data will result in a longer string.

Does the above explanation fit with your data?

If so, we can use the regular expression engine to remove the extraneous spaces within the string "s", and use the VBA Trim function to remove the leading and trailing spaces.

If your data might be longer than 2^31 characters, a further modification to my solution below will be required.

Changes from the previous are bold'd.

==============================================
Option Explicit

Option Compare Text
Sub UniqueWordsAndCount()
    Dim vSrc As Variant
    Dim vRes() As Variant
    Dim rDest As Range
    Dim i As Long, j As Long
    Dim colWords As Collection
    Dim s As String, Words As Variant
    Dim re As Object, mc As Object
  
'Sentences Assumed to be in Column A including row 1
'This can be easily changed
vSrc = Range("A1", Cells(Rows.Count, "A").End(xlUp))

'first cell of results destination range
Set rDest = Range("C1")

'All the stuff into one string
For i = LBound(vSrc, 1) To UBound(vSrc, 1)
    s = s & " " & vSrc(i, 1)
Next i

'remove non "Word" characters
' i.e. everything except letters, digits,
'   underscores, hyphens and apostrophe's
'If you want to retain any other characters,
'  modify re.pattern appropriately
'replace with <space> to enable splitting
Set re = CreateObject("vbscript.regexp")
With re
    .Pattern = "[^-'\w\s]+"
    .Global = True
    .ignorecase = True
    s = .Replace(s, " ")
    .Pattern = "\s+"
    s = .Replace(s, " ")
End With
s = Trim(s)

'Generate list of unique words
Set colWords = New Collection
    Words = Split(s)
    On Error Resume Next
        For i = LBound(Words) To UBound(Words)
            colWords.Add Item:=CStr(Words(i)), Key:=CStr(Words(i))
        Next i
    On Error GoTo 0
  
'Generate results array
ReDim vRes(0 To colWords.Count, 1 To 2)
'Headers
    vRes(0, 1) = "Words"
    vRes(0, 2) = "Count"
'Words and counts
    For i = 1 To colWords.Count
        vRes(i, 1) = colWords(i)
            With re
                .Pattern = "(?:^|\s)" & vRes(i, 1) & "(?=\s|$)"
                Set mc = .Execute(s)
            End With
        vRes(i, 2) = mc.Count
    Next i
              
Application.ScreenUpdating = False
Set rDest = rDest.Resize(rowsize:=UBound(vRes, 1) + 1, columnsize:=2)
rDest.EntireColumn.Clear
rDest = vRes

'Sort by Count
rDest.Sort key1:=rDest.Columns(2), order1:=xlDescending, _
            key2:=rDest.Columns(1), order2:=xlAscending, _
            MatchCase:=False, Orientation:=xlSortColumns, _
            Header:=xlYes, dataoption1:=xlSortTextAsNumbers, _
            dataoption2:=xlSortNormal

rDest.Rows(1).Font.Bold = True
rDest.Columns(1).HorizontalAlignment = xlLeft
Application.ScreenUpdating = True
End Sub
