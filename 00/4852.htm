dale howard [mvp] say:

Noeneel --

Using the Delivery Dependencies feature between projects creates a "soft" external dependency that DOES NOT change the schedule of the Successor project is the Predecessor project slips.  To obtain the behavior you seek, you would need to set cross-project dependencies instead, where a task in a Predecessor project is linked with an actual dependency (FS, SS, FF, or SF) to a task in the Successor schedule.  If you set a cross-project dependency between tasks in two projects, if the task in the Predecessor project slips, the task in the Successor project will slip.  The PM of the Successor project will be warned about the slippage when he/she opens the Successor project, by the way.  Hope this helps.

I wanted to track how many were changed i.e. the partner's name is in the header, and how many were not.
Try this one.

Andreas.

Sub Test()
  Dim loDocument As Document
  Dim lnNumReplaced As Long
  Dim lnNumNotReplaced As Long
  Dim SaveChanges As Boolean
  For Each loDocument In Documents
    'Assume failure
    SaveChanges = False
    'Search the primary header
    With loDocument.Sections(1).Headers(wdHeaderFooterPrimary).Range.Find
      If .Execute("EMERITUS (RETIRED)", MatchWholeWord:=True) Then
        'Already replaced
      ElseIf .Execute("EMERITUS", MatchWholeWord:=True, _
          ReplaceWith:="EMERITUS (RETIRED)") Then
        lnNumReplaced = lnNumReplaced + 1
        SaveChanges = True
      Else
        lnNumNotReplaced = lnNumNotReplaced + 1
      End If
      loDocument.Close SaveChanges
    End With
  Next
End Sub
