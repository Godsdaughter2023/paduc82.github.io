    Windows 10 Insider Preview 15007 (rs_prerelease) Error - Microsoft Community

    michael.ellis say:

    Windows 10 Insider Preview 15007 (rs_prerelease) Error 0x80240013

    Keeps getting stuck on this update. Downloads, gets to 100% and then fails. Anyone got a fix? Thanks

    don cuthbert say:

    I made a public build (14393) Virtual Machine in Hyper-V and put it on the Fast Ring (it does not matter if Windows on the VM is activated). Because 14393 does not update via UUP, Windows Update downloaded a full ESD file for 15007, which I used to create a 15007 ISO. I then used that for other installs. The ESD file gets deleted after the first restart, so you have to copy it elsewhere before it restarts the VM. The ESD file is moved to C:\$WINDOWS.~BT\Sources\Install.esd before the upgrade restarts. With UUP upgrades we get a lot of smaller ESD files under SoftwareDistribution and there is no simple way to create install media from them.

    Maybe running the Windows Update Troubleshooter from Control Panel, and/or deleting the SoftwareDistribution folder under C:\Windows will enable it to upgrade too.

    jmarco0921 say:

    Don:

    Did you find ESD file in C:\$WINDOWS.~BT\Sources\ before the upgrade restarts when going from Build 15002 to Build 15007?

    I search both $WINDOWS.~BT or $WINDOWS.~WS folders and could not find it.

    don cuthbert say:

    No.  The full ESD file is only moved to there if the upgrade occurs from an older build that does not use UUP, which is why I upgraded from build 14393 (the public build) on a VM. During Windows Update download, the ESD file is downloaded to a subfolder under the SoftwareDistribution folder using its full filename. After the download completes, during the time it is preparing to install, the ESD file is moved and renamed to C:\$WINDOWS.~BT\Sources\Install.esd. If you allow the upgrade to Restart before making a copy of the ESD file, it will get deleted.

    If you upgrade from 15002 to 15007, it uses UUP and you get many small ESD files under SoftwareDistribution. There is no single full ESD file downloaded nor assembled under UUP. The small UUP ESD files end up under Windows.old\SoftwareDistribution after the upgrade completes and never get moved to $WINDOWS.~BT.

    don cuthbert say:

    I hope it is working out for you.

    Hyper-V is available in Windows 10 Pro, but requires a computer with Virtualization enabled via the BIOS settings. Some devices do not have all of the virtualization capabilities required to run Hyper-V. There is other software that is available too, such as VirtualBox and VMWare, which may work if Hyper-V does not.

But since I was only doing this to get the full ESD file for use elsewhere, using a VM was easiest for me.
