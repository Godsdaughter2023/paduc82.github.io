mjs1a say:

I think that this is the issue:

Security SPP: update Windows license and product key token failed with 0x80070005 - Event ID: 8211

The software protection service runs under the Network Service account.

0x8007005 is an "Access Denied" error.

Give the Network Service account "read and execute" "list folder contents" and "read" permissions to c:\windows\system32\spp and this error should go away.

Go to the folder.

right click on the folder

choose "properties"

choose the "Security" tab

choose the "edit" button

choose the "add" button

type "Network Service" in the search box

choose "OK"

pick the check boxes for the permissions that I describe above

choose "OK"

NOTE: this is not a default permission for this folder. All of its permissions are inherited from c:\windows\system32.

This worked for me. I had this error in my event log after updating Win 8.1 to Win 10, added the permission for Network Service, and the error message listed above went away.

Let's see if the updates will install now.

At least "Security SPP: update Windows license and product key token failed with 0x80070005 - Event ID: 8211" is gone from event viewer now.
