How to calculate salary based on several factors? - Microsoft Community

rockyflyer say:

How to calculate salary based on several factors?

Hoping someone can talk me through this. I have a salary database built but came to a stumbling block.

I have several employee types whose salary change depending on their type, grade, and location. What I was hoping to do was to create a "lookup" field that would look at the three fields and correlate with the appropriate table and salary.

1. Currently I have separate pay tables for each occupation, should I make just one table though some information may be repetitive? 

2. How do I make the field consider the three variables and lookup the correct salary?

Help!

ken sheridan say: Yes, you should have a single table.  In database terms the table would model a ternary many-to-many relationship type between employee types, grades, and locations, so would have a column for each, which would in each case be a foreign key referencing the primary key of the employee types, grades or locations table as appropriate.  The primary key of the table would be a composite of these three columns.  Additionally the table would have a column for the current salary, which is a non-key attribute of the relationship type.

For data entry of an employee's salary you'd thus need three unbound combo boxes in which employee types, grade, and location is selected.  Code in the AfterUpdate event procedure of each would assign a value to a salary text box, if the active control is not Null, by calling the DLookup function with an appropriate criteria expression to return the salary for the three selected values in the combo boxes.

You should be able to populate the single salaries table from your existing tables with a few simple 'append' queries.

To create a multilevel outline you can follow the steps given below:-

1.     Open a new document

2.     Click on make a multilevel list.

3.     A drop-down menu with different outline styles appears. Click on "Define New Multilevel List" to create an Multi level outline.

4.     On the left, where it says "Click level to modify," 1 should be highlighted.

6.     Highlight level 2 under "Click level to modify" select "A, B, C..." under "Number Style for this Level." Don't forget to change the ")" to a period! Repeat for level 3, selecting "1, 2, 3..." and repeat for level 4, selecting "a. b, c..." Click "OK".

jllatham say:

Are those the only html tags in the cells, or are there others such as <b></b>, etc?

Now, if each entry contains a single <p> and matching </p> tag, then you could get the text between them with this formula (assumes the full html structure is in A2)

=IFERROR(MID(A2,FIND("<p>",A2,1)+3,FIND("</p>",A2,1)-(FIND("<p>",A2,1)+3)),"")

That should show  Hello, world in the cell the formula is in.

If that works for you, fill the formula down to the end of the html entries.

jeeped say:

Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module (e.g. Alt+I, M). Paste the follow into the new pane titled something like Book1 - Module1 (Code),


Sub strip_html()
    With Selection
        .Replace What:="<p>", Replacement:=vbNullString, LookAt:=xlPart, MatchCase:=False
        .Replace What:="</p>", Replacement:=vbNullString, LookAt:=xlPart, MatchCase:=False
        .Replace What:="<Body>", Replacement:=vbNullString, LookAt:=xlPart, MatchCase:=False
        .Replace What:="</Body>", Replacement:=vbNullString, LookAt:=xlPart, MatchCase:=False
    End With
End Sub

Add more lines to remove additional opening and closing HTML tags. Tap Alt+Q when done to return to your worksheet. Make a selection of cells to run the sub against and then tap Alt+F8 to open the Macros dialog and Run the macro.
