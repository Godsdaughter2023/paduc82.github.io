macropod say:

For multiple content controls you could use something like:

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
Dim Val As Long
If ContentControl.Title = "Approver1" Then
  With ContentControl.Range
    Select Case Left(.Text, 4)
      Case "Pres": Val = 20
      Case "Prov": Val = 15
      Case "Dean": Val = 10
      Case "Dept": Val = 5
      Case Else: Val = 0
    End Select
  End With
  ActiveDocument.Tables(1).Cell(5, 3).Range.Text = Val
End If
If ContentControl.Title = "Approver2" Then
  With ContentControl.Range
    Select Case Left(.Text, 4)
      Case "Pres": Val = 20
      Case "Prov": Val = 15
      Case "Dean": Val = 10
      Case "Dept": Val = 5
      Case Else: Val = 0
    End Select
  End With
  ActiveDocument.Tables(2).Cell(5, 3).Range.Text = Val
End If
End Sub

In the above, the first control updates the first table and the second updates the second table. If the controls and the ranges they update are in the same table, a simpler approach might be possible.

Each folder will have less than 5,000 items.

Now to my questions:

(1) (1) To show you an example, I just created one folder in the list.  Let's say that the list has one folder, which is folder A, and the folder A has 4,900 files in it. Outside the folder, I added one more video to the list. Then, is the total number of items in this list is 4,901 or 2?

(3) (3)    Let's say that I create multiple folders, folder A, B, C, D, E... in the list and that each folder has less than 5,000 items. Does this affect the 5,000 item list view threshold?

tim liu msft say:

Hi Enju,

 

If we just say the scenario 5 folders with 4900 videos in each folder, this scenario doesn’t exceed the 5000 items list view threshold. However, as Webbrewers3 mentioned, SharePoint Online is related to SQL server, when we perform some operations in SharePoint Online, query will be send to SQL server, if the query causes more than 5000 rows, the table in SQL Server will be locked. For example, if you delete this library or if you share items with 5000 unique permissions, it will send a query which causes more than 5000 rows. These operations may cause some unexpected issues. For a better explanation, you could check the article mentioned in my first post.

 

Meanwhile, as Webbrewers3 suggested, you may put videos to different folders/libraries/sites. It should be better to put so many videos in one library.

 

Thanks for your understanding.

 

Tim

Paste in Power Point excel tables that can be refreshed - Microsoft Community

rodrigopomar say:

Paste in Power Point excel tables that can be refreshed

In Power Point for Windows there is an option (with paste special) to paste an excel's table that can be refreshed, but I can't find such an option in the Office 2016 Mac version. Is there any alternative for that?
rena yu msft say:

Hi Rodrigo,

 

You can refer to Use Excel data or formulas in PowerPoint 2016 for Mac, if you use Insert Object > From File, you will get the table linking to the copy of the Excel sheet. For example, if you Insert Object > From File > Select workbookA, you will get a table linking to the workbookB, this workbookB is the copy of the workbookA. Any changes in the workbookB can update data into your presentaion, but the changes in workbookA will not update data to your presentation. In Office for Windows, when you paste special> Paste > Microsoft Excel Worksheet Object, this feature works the same result.

However, In Office for Windows, when you paste special > Paste link > Microsoft Excel Worksheet Object, you will get a table linking data to the source workbook. For example, when you paste special > Paste link > Microsoft Excel Worksheet Object > select workbookA, you will get a table linking to workbookA directly. So all the changes in the workbookA will update directly to the table in your presentation. This feature is different. But this paste link feature is not available in PowerPoint 2016 for Mac. If you have the concern about this, we suggest you share your insights to our UserVoice.

 

Thanks,

Rena
