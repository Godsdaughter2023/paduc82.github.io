stephendawson say: (Okay, this is years later.)

I have no satisfactory solution, but a cheap work-around. First, plug the notebook directly into the router and check that you now have access to shared resources. In my case, it is only when connecting via WiFi to the router that I have problems.

Now here's the workaround. Buy a cheap WiFi access point or WiFi router. I picked up an N300 for $25. Set it up if it's a router so that it no longer provides DHCP support. Plug it into one of the ethernet ports of your Netgear modem. Use it as the Wireless access point. You can leave the Netgear WiFi access running if you like. Switch between them and you'll see no shared resources with the Netgear WiFi connection, and full shared resources with the other WiFi connection.

Note: that extends to a NAS plugged into the modem itself!

If you are using Internet Explorer you may try the steps and check if it helps.

Method 1:
I would suggest you to try adding those websites to security zone and check if it helps, here is the link:

To add a website to a security zone follow the steps mentioned below.

a) Open Internet Explorer.
b) Navigate to the website that you want to add to a specific security zone.
c) Click the Tools button, and then click Internet Options.
d) Click the Security tab, and then click a security zone (Local intranet, Trusted sites, or Restricted sites).
e) Click Sites.
f) If you clicked Local intranet in step d), click Advanced.
g) The website should be shown in the Add this website to the zone field. Click Add.
h) If the site is not a secure site (HTTPS), clear the Require server verification (https:) for all sites in this zone check box.
i) Click Close, and then click OK (or click OK twice if you clicked Local intranet in step 4).

Method 2:
If the issue is still not resolved I would suggest you to use Internet Explorer without add-ons and check.

To do this, click Start, point to All Programs, point to Accessories, point to System Tools, and then click Internet Explorer (No Add-ons).

If this resolves the issue, follow these steps to isolate the browser add-on that is causing the issue:

a) Click Tools, and then click Internet Options.
b) Click the Programs tab, and then click Manage add-ons.
c) Click an add-on in the Name list, and then click Disable.
d) Repeat step 3 until you identify the add-on that is causing the issue.

wunders say:

My guess is that the area of the disk where Restore Points are kept was scanned by your anti-spyware program and the program found and deleted traces of your malware files from the restore point area.  Touching any files in this area will contaminate a restore point(s).  Besides, if you had malware on your machine, the last thing you would want to do is perform a System Restore and risk getting them back.  Many Antivirus programs will instruct you to clear all restore points after removing malware to make sure your restore points going forward are coherent.  You do this by turning off and then on System Restore:

 

"How to turn off and turn on Sytem Restore in Windows XP"

  < http://support.microsoft.com/kb/310405 >
