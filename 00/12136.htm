tasos_k say:

Okay  MikeSe,

try this vba macro....

[edit2..]

Sub SplitCSVFiles_in_SeparateCSVFiles()
'June 20, 2016
Const lrow As Integer = 2 '<< data from row 2
Const N As Long = 100 '<<split data every 100 entries
Dim wb As Workbook, newwb As Workbook
Dim sPath As String, sFile As String
sPath = "E:\Folder A\Folder CSV\" '<< change Path
Dim r As Long, x As Long, t As Long
Dim v As Variant
Dim newName As String
sFile = Dir(sPath & "*.csv") '<< search for csv files
Application.ScreenUpdating = False
Do While sFile <> ""
Set wb = Workbooks.Open(sPath & sFile, xlDelimited, local:=True)
r = Sheets(1).UsedRange.Rows.Count
If r <= N + 1 Then GoTo nnext
v = Split(wb.Name, ".")
newName = Replace(wb.Name, "." & v(UBound(v)), "")
t = 1
For x = lrow To r Step N
Set newwb = Workbooks.Add
wb.Sheets(1).Rows(1).Copy newwb.Sheets(1).Cells(1, 1)
wb.Sheets(1).Rows(x & ":" & x + N - 1).Copy newwb.Sheets(1).Cells(2, 1)

Application.DisplayAlerts = False

newwb.SaveAs Filename:=sPath & newName & "-" & t & ".csv", FileFormat:=xlCSV, local:=True
Application.DisplayAlerts = True
newwb.Close False
t = t + 1
Next
nnext:
wb.Close False
sFile = Dir()
Loop
Application.ScreenUpdating = True
End Sub

You code works fine for me but it could fail for a different reason. When deleting rows you have to work from the bottom up because if there are 2 consecutive rows that meet the criteria then a row will be missed. Try it this way.

Sub DeleteRows()
Dim c As Long
For c = 400 To 2 Step -1
    If IsError(Cells(c, 3)) Then
        Rows(c).EntireRow.Delete
    End If
Next c
End Sub

Sub DeleteRow()
  Dim targetRange As Range
  Dim c           As Range
  Dim All As Range
  On Error Resume Next
  'Get all cells with any error
  Set targetRange = Range("C2:C400").SpecialCells(xlCellTypeFormulas, xlErrors)
  'Got any?
  If targetRange Is Nothing Then Exit Sub
  'If you want to delete all error cells, this is enough:
  'targetRange.EntireRow.Delete
  'Otherwise:
  'find the special #REF! error
  For Each c In targetRange
    If c = CVErr(xlErrRef) Then
      'Combine this cells
      If All Is Nothing Then
        Set All = c
      Else
        Set All = Union(All, c)
      End If
    End If
  Next
  'Found any?
  If All Is nohting Then Exit Sub
  'Delete this rows
  All.EntireRow.Delete
End Sub

Hi Farzam,

 

Thanks for choosing Office 365.

 

When you migrate an Office 365 subscription from GoDaddy to Microsoft, you need to back up all the data files in the original Office 365 tenant.

 

For email data, you need to export them as PST files and then import them into your new Office 365 tenant. Here are some articles for your reference:

Import email, contacts, and calendar from an Outlook .pst file

 

For the OneDrive for Business files, you also need to export them manually from the original Office 365 tenant and then import them into the new Office 365 tenant.

young yang msft say:

Hi Sounder,

If we just disable the AD account, the synced Office 365 account will not be deleted. Only the sign-in status is changed to “blocked”.

You’re correct. To delete the synced Office 365, you need to move the AD user to a non-synced OU.

Thanks,

Young 

muditha jayath chathuranga say:

Young's answer is correct.

Disabling an account from local Active Directory won't delete the user account from the Azure AD. You need to either move the account to a non-synchronizing OU or delete the object from the local AD. Otherwise only the sign in status in AAD will be changed to "Disabled".

Thank you.
