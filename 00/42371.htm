al jarvi say: In order to login to a remote computer with Remote Desktop [RDC] you need a userID that is a member of the Remote Desktop Users Group on the remote computer, ie. PC-A for example, or a userID that is an administrator on PC-A. The userID is local to PC-A. The userID must also have a pasword. On PC-A simply type the name of the standard user in the Enter the object names to select window. Click on Check Names to verify its a valid userID on PC-A.

Now when you open the RDC connection on remote computer PC-B use the local LAN IP or NetBIOS name of PC-A and the previously configured PC-A userID and password to login. If your calling over the public internet from a remote location use the public IP of PC-A or if PC-A is behind a router the public IP of the router. In this latter case you need to also forward/open TCP Port 3389 through the router to the static LAN IP of PC-A.

FWIW here are my small RDC help pages that you may, or may not, be interested in.
