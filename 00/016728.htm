Give an example.   In general you can not "paste onto a webpage".   If you are looking at the Edit menu, its Copy item may be available for use, e.g. not grey, as soon as some object it is associated with, e.g. anywhere on the page is selected (e.g. highlighted) and its Paste item may be available for use (not grey), when the pointer is somewhere in an input form but that would only be true if the ClipBoard happened to contain content that that input area would accept...  

Oh.  That's wrong?  Too bad.   That does make it unnecessarily confusing I agree.   E.g. using this thread as an example, I moved my pointer outside an input area and saw that the Edit menu showed neither Copy nor Paste were available, doubleclicked on a word and saw that Copy was now available but Paste was not.   Used the Copy (which left the word selected).  Moved the pointer to the input area and saw only the Copy item was available.  Clicked in the input area and noticed an insert cursor drawn there and the previously selected word stop being selected.  The menu then showed only Paste was available.  Used Paste and saw the copied word duplicated.   So all of that showed it working as expected with a simple text selection, copy and paste.

What was unexpected (for me) was this example:  click anywhere outside an input area and do a Select All, e.g. press Ctrl-a and Copy e.g. press Ctrl-c.  Click in the Search box and then check the menu.  See Paste available?  I don't think so.  (E.g. the search box would only want to receive text strings but what I have copied is not text, it's Bitmap.  So trying to do Ctrl-v  doesn't do anything (as expected but not for the reason I thought <w>).

So, I suspect you are trying to copy a data type which can not be received by wherever it is that you are trying to paste it.  The only possible way that you could have both the Copy and the Paste menu items active at the same time (assuming that is what you were describing) is if you had both copied and selected where you were going to be pasting.  (That's possible, e.g. I can see it in here by doing Ctrl-a, Ctrl-c.)

This sort of problem used to be much easier to sort out in XP using its ClipBrd.exe utility (ClipBoard Viewer).   In W7 the best we have AFAIK is WordPad or Paint and neither of them clarify which data type they are receiving (or what types are available which they are not receiving).   Similarly if there is any kind of text component in the captured selection you can often see it by pasting into Notepad.   Then if whatever it is that you were trying to paste is seen that way you can recapture it there and then paste it into a text input area with much less possibility of a problem.
