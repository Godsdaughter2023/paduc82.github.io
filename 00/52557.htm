Is a patch available to stop "DNS cache poisoning - Microsoft Community

chinchez say:

Is a patch available to stop "DNS cache poisoning attacks"?

I understand that a patch was issued to stop the problem I described above, where can I find it and is it easy to install?
kerry_brown say:

DNS cache poisoning affects DNS servers. There is a patch for Microsoft DNS servers that mitigates but does not totally eliminate the problem. DNSSEC which is in the process of being deployed at the DNS root level will mitigate the problem but it will take years before all the DNS servers on the Internet deploy DNSSEC. For DNSSEC to offer the best protection your DNS stub resolver and every DNS server in the chain between you and servers that answer your DNS query must be using DNSSEC. Currently Microsoft client stub resolvers ignore DNSSEC. Very few if any ISPs and Internet hosting companies have deployed DNSSEC other than as a test. Right now the DNS root is signed for DNSSEC and some Top Level Domains (like .org but not yet .com) are signed.

To answer your question - There is nothing you can do from your end to prevent DNS cache poisoning. To help ensure that it doesn't affect you make sure you use a DNS server that is patched to offer some protection against that DNS server being successfully attacked. If you are not sure about your ISP ask them.

 

Kerry Brown MS-MVP - Windows Desktop Experience

andre da costa say:

New computers that come preinstalled with Windows 7 often have what is called a recovery partition. This is used to reinstall the operating system in the event of a system crash. To access it, you will need to boot into when you start your computer by pressing a function key. This can be either F1, F2, F9, F10, F11, F12 or even DEL or Tab key. Consult the owners manual that came with your PC for instructions about how to reinstall Windows.

 

If your recovery partition is not available or damaged, you should contact the manufacturer of your computer and request a recovery disc set you can use to reinstall Windows 7. They might charge a small shipping and handling fee.

------------------------------------

This is how the recovery partition is accessed for the most popular makes

For Dell, press CTRL + F11 directly after powering up the unit

For HP, press F11 directly after powering up the unit

For Toshiba, press and hold '0' BEFORE and during the power up

For Acer, press and hold ALT + F10 as soon as you see the logo

For Asus, press F9 as soon as you see the Asus logo.

For Advent, restart your computer. Then Press F10 repeatedly until the message "Starting System Recovery" is displayed

I got a solution to this problem and everything is fine now. Do this right after you get an error of the kind described above.

  1. Open Event Viewer by typing Event Viewer in the start ment search or eventvwr.msc in run.
  2. Expand Windows Logs -> Application Events and select an error event.
  3.  See the General Tab below the event list and locate the "Faulting module path" property.

This Faulting module is the application that was causing problem. Check some more error events and if the same application is causing the problem, you might wanna remove that and if it is some active x, disable it.

In my case it was an Axis ActiveX I installed for development purposes and probably was trying to play something in the video players.

I uninstalled the application located at the path and it was fine and peaceful.

lizette ags say:

Hi,

Syskey is a Windows internal root encryption key that is used to encrypt other sensitive operating system state data, such as user account password hashes. The tool was designed to improve the security of the Windows boot process by adding another layer of protection to it. In this state, the operating system blocks the boot process and prompt users for the password.
 
In Windows 10 RS3 and Windows Server 2016 RS, the syskey encryption key and the use of syskey.exe are no longer considered secure. Syskey is based on weak cryptography that can easily be broken in modern times. Syskey utility has also been known to be used by hackers as part of ransomware scams.

You can disable the use of syskey.exe by following the steps below:

     1. Open the Command Prompt, type syskey, and then hit the Enter key.

     2. Select Update when the Securing the Windows Account Database window opens.

     3. Select Password Startup on the next screen.

     4. Select System Generated Password.

     5. Select Store Startup Key Locally. This saves the key to the hard disk of the local computer.

     6. Click Okay twice to complete the process.

Since syskey.exe utility is no longer supported in Windows 10, we recommend that you use BitLocker instead. For more information, visit this link.

Feel free to post back if you have any other questions and we'll be happy to assist you.
