suzanne s. barnhill say:

Assuming that the numbering is coming from the heading style itself and has not been added to the TOC 3 style:

  1. Place the insertion point in a Heading 1 paragraph.
  2. Click the Multilevel List button in the Paragraph group on the Home tab and select Define New Multilevel List. This should open the Define New Multilevel List dialog with your list (linked to your headings) selected.
  3. Select Level 3 and click Font...
  4. In the Font dialog, all the boxes (Font, Font style, and Size) should be empty. If anything is selected, then clear the boxes and click OK.
  5. I have found that if Bold has been selected, then, even if the boxes are cleared and OK clicked, the number will still be bold. So return to the Font dialog and select Regular, then click OK. If you then return to the dialog, you'll find that the boxes are empty, and this does remove the bolding from the number.

If the numbering has been applied to the TOC 3 style rather than Heading 3, then you'll need to use this procedure for that style instead.

If you need to refer to a control on a Subform you must use specific syntax:

Forms!YourParentForm.NameOfYourSubformCONTROL.Form.SomeControlName

NameOfYourSubformCONTROL is the part that trips up most people. It is the name of the Subform CONTROL on the parent form, and may or may not be named the same as the form you're using as a Subform.

scott mcdaniel say:

Are you using a Navigation form? If so, then you have to handle this somewhat differently. The Nav form uses a single Subform Control and toggles the SourceObject of that control according to the button you've clicked. So if you want to select a Record in one form, then click one of the Navigation Buttons to show details of that record in a second form, you must first store the ID value from that first form. You can do this in several ways - one is to use the TempVars collection, so in the Current event of your first form:

TempVars.Add("ID", Me.ID)

Now in the second form you can refer to that TempVars item, perhaps in the Load event of the Form:

Me.Filter = "ID=" & Tempvars("ID")

Me.FilterOn = True

That would show any records containing the ID value from the first form. Of course, if you need to do this in a Subform in that second form, you'd have to do things differently - but we would need to know more about that.

bob jones aka: cybertaz say:

Two things you haven't included in your description are the amount of space on the last page of each of those 3 chapters following the last line of text & the length of the first endnotes for those Sections.

In order for endnotes to begin on the same page there must be at least enough space for the Endnote Separator plus no less than the first endnote to be positioned above the Bottom Margin... if that first endnote comprises no more than 2 lines. Based on default parameters, that translates to roughly .5", but can vary depending on any modifications that may have been made to the formatting of the Endnote Text Style. If the first endnote comprises more than 2 lines...

Widow/Orphan Control applies to endnotes just as with any other paragraphs, so if the first endnote comprises 3 lines the entirety plus the separator will be moved to a new page even if there is space for 2 lines on the previous page. It's only if the endnote comprises 4 or more lines that it would be permitted to span a page break.
