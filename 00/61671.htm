To suspend BitLocker Drive Encryption on an operating system drive
1. Click Start, click Control Panel, click System and Security, and then click BitLocker Drive Encryption.
2. Click Suspend Protection for the operating system drive.
3. A message is displayed, informing you that your data will not be protected while BitLocker is suspended and asking if you want to suspend BitLocker Drive Encryption. Click Yes to continue and suspend BitLocker on the drive.
By completing this procedure, you have suspended BitLocker protection on the drive by changing the decryption key to a clear key. To read data from the drive, the clear key is used to access the files. When BitLocker is suspended, TPM validation does not occur and other authentication methods, such as the use of a PIN or USB key to unlock the operating system drive, are not enforced. This allows you to make system changes such as updating the BIOS or replacing a data drive. When you are finished making changes to the computer, click Resume Protection from the BitLocker Drive Encryption Control Panel item to start using BitLocker Drive Encryption again.
To turn off BitLocker Drive Encryption
1. Click Start, click Control Panel, click System and Security, and then click BitLocker Drive Encryption.
2. Find the drive on which you want BitLocker Drive Encryption turned off, and click Turn Off BitLocker.
3. A message is displayed, informing you that the drive will be decrypted and that decryption may take some time. Click Decrypt the drive to continue and turn off BitLocker on the drive.

greg carmack - mvp 2010-2020 say:

Windows 10 installed at factory has an embedded BIOS Product Key.  That will remain for the life of the motherboard, so if you do a free Upgrade to Pro you'll then own both Home (embedded) and Pro as an Entitlement - which is the keyless method by which the Upgrade is activated that records your hardware HASH in MS servers.

As Sumit mentioned you might be able to insert the Windows 8 Pro Product Key into Activation settings to both change the installed version to Windows 10 Pro and activate as a Free Upgrade.  This is because in Version 1511 MS allows using the old OS product Key to activate directly the free Upgrade.  So check now that you have Version 1511 at Settings>System>About and if so try the 8.1 Product Key at the link there.

If the key isn't accepted then you'll need to first change the Windows 10 version from Home to Pro, which can be done by inserting the Pro Generic key to install Windows 10.  This will change the version to Pro but not activate it.  Afterwards you can insert the 8.1 key to activate per the Version 1511 FAQ at Activation in Windows 10.

Keep in mind this will be the inferior Factory Install.  If you want a much better install then do the vastly superior Clean Install Windows 10 Upgrade.  Leave the Product Key blank, delete all partitions during the booted Custom install,  activate aftewards as per above.  You should get a menu to choose Pro during install, but if it forces Home version install due to embedded key, follow the procedure given above to convert to Pro and then activate with the 8.1 key.

Placing your system in Clean Boot state helps in identifying if any third party applications or startup items are causing the issue. Check this issue in clean boot state.

  1. Press Windows key + X key. And select Run.
  2. Type msconfig in the Run box and hit Enter.
  3. On the Services tab of the System Configuration dialog box, tap or click to select the Hide all Microsoft services check box, and then tap or click Disable all.
