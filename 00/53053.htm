a. user say:

The upgrade method has not "become corrupted." It works well if you know how to do it properly and if your computer's configuration is essentially the same as when you first brought it home, which is true for many people. However, most people don't have the technical background to perform an upgrade and know better than to try it.

Ironically, people who do have the technical background to perform an upgrade are more likely to do a clean install instead, which works much better when a computer's configuration has gone through lots of changes, which is something geeks people with technical backgrounds like to do. A clean install on supported hardware is virtually a can't miss.

The people who get themselves into trouble are the ones who don't understand technology, never really learned how to operate a computer correctly, don't know how to prepare for a new operating system and don't know how to do the deed correctly, and yet they go ahead and upgrade anyway. And of course, it's never their fault.

Microsoft's share of the blame is aggressively nagging people to upgrade. It's easy enough to avoid upgrading if you know how to use a search engine. However, many people are afraid to make the changes to their computer that avoid Windows 10, so they're stuck.

Utilities like System File Checker, DISM, Driver Verifier, Event Viewer and others are widely overused and misinterpreted. People imagine all kinds of amazing things from the output and worry about nothing. SFC is not the solution to a botched upgrade.

a. user say:

The first rule of upgrading is: Never upgrade over a problem. Fix the problem first, then upgrade. Upgrading never fixes things, it only makes them worse. (That's another lesson I learned the hard way.)

I think it's really unlikely that a factory image was corrupted.  If the image had a problem, the OEM would quickly find out because that image is downloaded to tens of thousands of computers. In that case the OEM would quickly push out a fix.

SFC tests for system files that are internally corrupt. It's a very old tool. System files are almost never corrupted now. Microsoft has gotten very good at protecting system files from corruption. There's nothing a user can do to corrupt a system file since users don't have access to the code.

For those reasons, I think that SFC is very unlikely to be of any real value. The last time I ran it was years ago with Windows 7.

People look for all kinds of excuses for why Windows 10 doesn't work. (I'm not talking about you personally.) The truth is, in the overwhelming majority of cases, the user messed up.

artiezan say:

And finally a working solution.  I went through hundreds of KBs, third party forums, thought about formatting my machine, deleting reparse points, but thank you Mr. Occam I have a solution for all of the tortured users out there.

You have the option to select the folders that you want to back-up by clicking on "Change Settings".  The issue, if you select a sub-folder, say Users in Windows 7, it automatically selects the root (usually C).  I found that if you deselect the root and only select the sub-folder the backup works.

Most likely cause of the problem, feel free to correct me, I am sure that you will anyways.  The root is a common place for log files of various programs, virus protection etc.  Many of these files are not visible by default in your file system and should not be to the everyday user.  Either these files are activly in use or there is, as many people seem to be running into, the reparse linking issue.
