1.  In that case set the property for each of the controls by looping through the section's Controls collection:

    Dim ctrl As Control
     
    For Each ctrl In Me.Section("GroupHeader1").Controls
        If Me.[Primary Status Date] = VBA.Date Then
            ctrl.ForeColor = vbRed
        Else
            ctrl.ForeColor = vbBlack
        End If
    Next ctrl

2.  The Cancel argument's return value can be set to True to conditionally prevent the section from printing; the FormatCount property can be used to determine how many times the OnFormat event property has been evaluated for the current section before it prints.  This can happen multiple times, e.g. if a section would span a page, but is 'kept together' by starting it on the second page.  It's not quite as straightforward as it might seem, however, and I've often found it difficult to predict the value I'd expect to be returned.

3.  I've seen countless explanations of this, but as a rough rule of thumb the dot operator is used for built in properties and the like, while the exclamation mark can be used for objects which you've created, though the dot operator can also be used for the latter.  I can remember reading years ago that the exclamation mark was then redundant in any context apart from when referencing a control as a parameter in a query, but the dot operator appears to have worked for that also for some time now.

ken sheridan say: In what way is it not working?  Are you getting an error; if so what?  Is it just not changing the colour of the controls in the section?  If so are you sure the data includes a row with today's date?  If not try substituting a date literal with a value you know exists, e.g. #2013-07-04# (4th July last year).  The date format is the ISO standard for date notation BTW, so is internationally unambiguous regardless of your system's regional date format.

One other possibility is that the 'dates' might include non-zero times of day without you realizing it.  This is not uncommon, and the usual culprit is the inappropriate use of the Now() function as a default value for a date/time column.  There is actually no such thing as a 'date value' in Access, only a 'date/time value'.  If you enter a date without a time you are actually entering the time at midnight at the start of the day, or more correctly speaking you are entering a 64 bit floating point number which represents the date/time as the duration since 30 December 1899 00:00:00.

Assuming that the Primary Status Date column should be dates only, try giving it a validation rule in the table design of:

Is Null Or DateValue([Primary Status Date])

If the field does not allow Nulls, i.e. its Required property is True, you can omit the first part of the expression.  If it won't allow you to save the amended table definition, it suggests you do have non-zero times of day.  You can correct these with a simple update query:

UPDATE [YourTableName]
SET [Primary Status Date] = DATEVALUE([Primary Status Date])
WHERE [Primary Status Date] IS NOT NULL;

Make sure the column does not have a DefaultValue property of Now(); if it does change it to Date().  Then apply the validation rule.  Be sure to correct anywhere else which is trying to assign the Now() function's return value to the column, e.g. the DefaultValue property of a bound control in a form, or code in a form's module.  If you don't an error will occur when this is attempted

fill in time gaps - Microsoft Community

sl1ph say:

fill in time gaps

i have a large spreadsheet that contains data downloaded from a datalogger. The logger was supposed to take readings every 15 minutes, but when i copied the data that was downloaded into my excell spreadsheet i realised that there were some time gaps where the information wasnt recorded (ie) data was recorded every 15mins for the majority of the time but once in a while there will be a half hour or more time gap in the rows where date and time was pasted.

I need to fill these gaps in with the missing 15 min intervals and need to find a way that is a little bit faster then scanning through the sheet and manually adding the times. an example is provided below.

4/9/09 19:00

4/9/09 19:15

4/9/09 19:30

4/9/09 20:00

I need to create a new row and insert the missing time (4/9/09 19:45) in between 19:30 and 20:00. Is there a formula i can use to fill in these gaps?

 

jeeped say:

I would leave the existing worksheet alone and consider it nothing more than a data source. Next I would start a new spreadsheet and populate the A column with a true series of time. This can be easily done with two good values (15 minutes apart) in A1 and A2, then highlight the A column and use Fill, Series, AutoFill.

Next I would use VLOOKUP in the B column of the new worksheet, using it to reference and pull the datalogger information from the original worksheet (the data source). With a modicum of error control to avoid error messages (and insert default

HOWEVER, There are a couple of problems that need to be addressed. When a timestamp is automagically generated to an accuracy of a minute it is rarely accurate beyond the displayed format. I found that copying your time data and then recreating my own manually to match yours produced differences of as little as 0.000001, but enough that Excel would not accept them as equal. To bypass this, I reformatted both time values to definitive text in the VLOOKUP and created an Array Formula for the match. This worked, but produced the second problem; the Array Formula was very CPU intensive, and not worth using over 40 or 50 rows or data. From your original inquiry, I've assumed you had many more.

The solution was to create (e.g. Insert) a new column in the original worksheet and let Excel perform half of the conversion to a time text format. I inserted a new coulumn (B) amd populated with the formula:

=TEXT(A1, "yyyymmddhhmm")

... and then Filled Down column B to the extent of the A column (e.g. timestamp). Now I had something to work with.

So, if the original worksheet (Sheet1) held the timestamp in column A , the new column with the new formula (e.g. =TEXT(A1, "yyyymmddhhmm")) in column B and the datalogger information in column C and the new worksheet has had a series of timestamps with 15 minute intervals Autofilled into column A, then B1 should have this in the formula bar:

=IF(ISNA(VLOOKUP(TEXT(A1,"YYYYMMDDHHMM"),Sheet1!B:C,2,FALSE)),"No event!",VLOOKUP(TEXT(A1,"YYYYMMDDHHMM"),Sheet1!B:C,2,FALSE))

... and Fill or Copy Down as needed to reference column A.

Additional information could be pulled by modifying the B:C and 2 attributes of that function.

Further visual reference of non-events (greyed text perhaps) could easily be accomplished with Conditional Formatting based upon the cell's content.
