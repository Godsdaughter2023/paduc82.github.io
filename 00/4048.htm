Sorry to know that the File Explorer crashes on Windows 10. I will certainly help you.

This issue might occur if the system files are corrupted.

I suggest you to follow the troubleshooting steps listed below and check if it resolves the issue.

Step 1: Run the SFC scan

I suggest you to run the Microsoft Windows Resource Checker tool, also know as "System File Checker" or "sfc".
This tool scans and verifies the integrity of all protected system files and replaces incorrect versions with correct versions. Here's how:
1. Press Windows key + X, click Command Prompt (Admin).
2. In the Command Prompt, type the following command, sfc /scannow and then press Enter.
3. Wait until the process completes. This usually doesn’t take longer than 20 minutes with a regular hard disk and takes much shorter when you have a fast SSD drive.

Step 2: Run the DISM Tool

Run the DISM (Deployment Image Servicing and Management) Tool to fix any files that System File Checker can't repair and fix Windows corruption errors. Here's how:

1. Right click on Start, click on Command Prompt (Admin).

Type Dism /Online /Cleanup-Image /ScanHealth and click Enter.
Type Dism /Online /Cleanup-Image /CheckHealth and click Enter.
Type Dism /Online /Cleanup-Image /RestoreHealth and click Enter.

2. Check to see if the issue is resolved. If not, restart the system.

scottgem say:

There is no max. You can put the tables in a separate accdb and link to them from your front end (which has all the forms, reports, queries etc.) 

There is a maximum size of 2GB for an accdb file. But you can get around this limit by 1) using multiple back ends for data or 2) using SQL Server Express or some other data store for your tables. 

2GB can cover a lot of data. it depends on your record size and how efficiently the data is designed.

andreas killer say:

You have to remember by yourself which subs are scheduled, so you have to write a manager that does the work for you. See the code below.

Andreas.

Option Explicit

Private Sub Example_Schedule()
  Debug.Print Now, "Example_Schedule"
  'Start 2 scheduled calls
  Schedule "Example_A_Schedule", Now + TimeSerial(0, 0, 1)
  Schedule "Example_B_Schedule", Now + TimeSerial(0, 0, 2)
  'This cancels only subroutine A:
  'Schedule "Example_A_Schedule", Schedule:=False
  'This cancels all scheduled calls:
  'Schedule ""
End Sub
Private Sub Example_A_Schedule()
  Debug.Print Now, "A_Schedule"
End Sub
Private Sub Example_B_Schedule()
  Debug.Print Now, "B_Schedule"
End Sub

Sub Schedule(ByVal Procedure As String, _
    Optional ByVal EarliestTime As Date, Optional ByVal LatestTime As Variant, _
    Optional ByVal Schedule As Boolean = True)
  'Manages scheduled macros
  Static Scheduler As Object
  Dim Items, Keys, i As Integer
  If Scheduler Is Nothing Then Set Scheduler = CreateObject("Scripting.Dictionary")
  If Procedure = "" Then
    Keys = Scheduler.Keys
    Items = Scheduler.Items
    For i = 0 To UBound(Items)
      Application.OnTime Items(i), Keys(i), Schedule:=False
    Next
    Scheduler.RemoveAll
    Exit Sub
  Else
    If Scheduler.Exists(Procedure) Then
      On Error Resume Next
      Application.OnTime Scheduler.Item(Procedure), Procedure, Schedule:=False
      On Error GoTo 0
      Scheduler.Remove Procedure
    End If
  End If
  If Not Schedule Then Exit Sub
  Application.OnTime EarliestTime, Procedure, LatestTime
  Scheduler.Add Procedure, EarliestTime
End Sub

jay freedman say:

You haven't done anything wrong. At the point where the field code disappears, Word has updated the field (collapsing the code to show the result), but the incomplete field code doesn't yet have a result -- so it's blank.

To show the field code again, press Alt+F9. It will look something like this, except with the name of your source column instead of my "Image":

{ INCLUDEPICTURE "{ MERGEFIELD Image } }

Finish it by adding the closing double quote between the two closing brackets, and press Alt+F9 again. Then click the Preview button on the ribbon. You should see at least the first image. To make sure the others update properly, use the IF field construction shown in the http://www.gmayor.com/mail_merge_graphics.htm article.
