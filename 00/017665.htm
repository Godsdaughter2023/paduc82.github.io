Not to be too sarcastic, but as seems to be the default position for users of Excel (etc.), I have solved the above problem myself. I outline the answer here in case anyone has this problem more widely.

Problem

A mysterious lack of option to change axis bounds, on some graphs, but not others - not restricted to date type axes.

Solution:

The solution is fairly simple but it would be better if the program warned you of this and proactively provided the option to change it: essentially if you have experienced the above problem but not in a "dates" context then you should check the type of chart you are plotting, and switch it from e.g. a 'Line' chart to e.g. a Scatter chart. Excel will now magically recognise that your horizontal axis is made of 'Values' whose range can be adjusted. Simples.

(The problem had nothing to do with the step size of the horizontal axis data, as I wondered in my earlier post above.)

Explanation:

Part 1: A long, long time ago, deep inside the mind of somebody at Microsoft (not far from a galaxy far away) there lurked the nefarious assumption that if you plot a 'Line' graph then your independent (horizontal) variable must be "Categorical" rather than made of "Values". (Categorical implies dates, or otherwise discrete, not strictly numerical variables.)  Quite why this assumption makes sense, I am not sure. Perhaps someone in the know can explain. Or at least institute a message warning unwary users.

Part 2: Even deeper (presumably) must lurk the assumption (presumably) that if you, the user, are dealing in Categorical variables, you couldn't possibly want to adjust the range of your variables. Again, quite why this assumption makes sense, I am not sure. Perhaps someone in the know can explain. Or perhaps it's just difficult to program in an option to change a display range of categorical rather than numerical variables. So Part 2 is more understandable than Part 1, which is, frankly, a mystery!

I hope this helps somebody.

about the clutter, firstly, i would like to explain when the feature is first enabled, it's not very accurate yet. it needs to process up to 5,000 email messages before it has a sufficient training model to take action on items that are delivered to the inbox. actually, clutter won’t start to work until it has processed at least 1000 messages. again, the more the better.

moreover, clutter analyzes more than 40 aspects of a message and actions that are taken upon a message when it's delivered to the inbox including sender, subject, content, and so on. clutter collects statistics about whether a person reads the message, replies to it, moves it, has a rule move it, and so on. if a user performs actions such as these to a message, it's less likely that clutter will mark a similar message as clutter the next time that it arrives. if a user reads and deletes all mails in their inbox regularly and keeps their inbox clean, clutter will not work.

inbox rules and folder structure also play a big role in the training of clutter, so if you have any rules already organizing mail for you and moving messages to folders, clutter will just ignore those messages. in fact, if a message is ever touched by an inbox rule clutter will never touch it.

The footnote separator has nothing to do with the footnote style :-) In order to modify the separator you first nee to switch to Draft View, then;
  1. Go to View> Footnotes
  2. In the Title Bar for the Footnotes Pane open the list & select Footnote Separator

One way...

Public Function CreateTestArray()
Dim A()
ReDim A(9, 1)
A(1, 1) = 1
A(2, 1) = 2

A(5, 1) = 5

Note that unless you set the option base to 1 then your arrays start at 0 meaning that you have an array of 10 element by 2 elements. For that reason the range is 10 rows by 2 columns...

jllatham say:

Unless you have previously assigned b to a specific range, you don't need it in the code.  Try this and see if you get what you need:

Cells(i,1) = a(i,1)

That should put the values into A1, A2, A3...A9

If you need to put it elsewhere on the sheet then start by setting b to a range like:

Set b = Range("C1:C9")

then your existing code would put the values from a() into C1 through C9 of the currently active sheet.

To pass the range to a function, set up your function to accept a range or an address referencing a range as a parameter:

Function DoSomethingElse(whatRange As Range) As Variant

And you would call it by passing a reference to the range.  Assuming you've Set b as shown above, you could call it like:

Dummy=DoSomethingElse(b)

 

You can also pass it to a Sub, with the Sub set up as:

Sub DoMoreWork(whatRange As Range)

and call it like this:

DoMoreWork b

or

Run "DoMoreWork", b
