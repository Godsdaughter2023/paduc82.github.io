rob koch say:

These types of occasional false positives for outdated software seem to have become more common since the release of Windows 10, possibly due to the fact that most current software is digitally signed now, while older programs rarely were.  In such cases it's likely that the programs being detected contain either portions of the same code or behavior that the Trojans do, so without the ability to verify that the software is from a specific vendor and has good integrity which a certificate provides, continued detections are likely.

In such a case if retaining the outdated software is required, then using Defender's Settings, Excluded files and locations and also possibly Excluded processes may be the only way to stop the detections.

This is just my own speculation from observation here in the forums, but rather than sit and wait for a fix that might never come, this may allow you to bypass the problem at least until a fix does occur.

Rob

a. user say:

Hi Blake

Thank you for your reply.

Let us try to reset the Windows Update Components and then try to install the build. If these components are corrupts, we would not be able to install the build.

Manually reset the Windows Updates Components.

a) Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

b) Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

net stop wuauserv

net stop cryptSvc

net stop bits

net stop msiserver

c) Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

d) Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

e) Type Exit in the Command Prompt to close it.

Now you may try running the Windows Updates and install the new build and check if the above steps resolve the issue.

For further reference, you can refer to the link:

How do I reset Windows Update components?

http://support.microsoft.com/kb/971058/en-us
