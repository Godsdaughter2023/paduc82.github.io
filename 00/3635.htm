redalert---01 say:

Sorry for the late reply...I didn't get an email notification that you replied back...

Anyways, what you will need to do is basically add Japanese glyphs to the following 3 fonts that the E72 uses and then store them onto the microSD card (E:/Resources/Fonts/):

> nosnr60.tff
> nssb60.ttf
> nstsb60.ttf

 

The E72 will then default to using these fonts and if there is anything wrong with the fonts all you do is remove the microSD card and restart your phone. The E72 will revert back to using the fonts stored on the phone. At any point, you should not delete or replace the fonts that are stored on the E72.

 

So the first thing you need to do is obtain these fonts from the E72 and there are two ways to do it...The first way is to use Y-Browser (http://www.drjukka.com/YBrowser.html) to locate the fonts on the E72 from  "Z:/System/Fonts/" or you can obtain the fonts from the Nokia S60 3rd Edition FP2 SDK pack (http://www.forum.nokia.com/info/sw.nokia.com/id/ec866fab-4b76-49f6-b5a5-af0631419e9c/S60_All_in_One_SDKs.html).

Once you have obtained these 3 files you will need to do the same to obtain the Nokia Japanese fonts which is called:

> HeiseiGothS60.ttf

It can be obtain in a similar way to the E72 fonts I described above.

What you should have now is the following 4 font files:  

> nosnr60.tff
> nssb60.ttf
> nstsb60.ttf
> HeiseiGothS60.ttf

What we want to do with these files is merge HeiseiGothS60.ttf into nosnr60.tff, nssb60.ttf, nstsb60.ttf.

So how to do that you can follow the instructions on the following link, which uses fontforge:

http://www.fwrnando.com/2009/06/merging-fonts-s60-specific-glyphs-and-other-stories/comment-page-1/

Once you have created the new merged font files you just place them onto you microSD card in E:/Resources/Fonts/. No other software like FontRouter is required, so just restart your E72 and you should now be able to read Japanese characters.

The same method I described will work for Chinese characters but during my testing, it does not work for Korean characters.

I hope the instructions helps...it took me a while to get the fonts working right on the E72. I mainly used the S60 3rd Edition FP2 SDK pack to test out the fonts before loading it onto the E72…so the SDK is a powerful tool to test if the fonts work before you load it onto the E72.

Anyways let me know if it all works out and if there are any questions.

 

Thanks

Excel formula to transfer data to Outlook - Microsoft Community

jcavallaro say:

Excel formula to transfer data to Outlook

I understand there is a formula or macro that I can enter so that when I input a date in an Excel Cell so that I can essentially click on that date and automatically maps to and creates an "appointment" in my outlook calendar (from the one cell on the excel worksheet). 

Does this exist and how can I find information to do it?

danielco say:

First you should go to Tools, References menus in the VBE window. Mark the "Microsoft Outlook xx.x Object Library" as selected. Paste the following macro in the sheet module :

Private Sub Worksheet_BeforeDoubleClick(ByVal Target As Range, Cancel As Boolean)

If Target.Address = "$B$1" Then

    Cancel = True

    Dim olApp As Object, myItem As Object

    Set olApp = CreateObject("Outlook.Application")

    Set myItem = olApp.CreateItem(olAppointmentItem)

    myItem.Start = Target.Value

    myItem.Display

End If

End Sub


The date must be in B1. When you double-click on B1, a new appoinment is displayed. I tested it with Office 2010 and it is possible that you have trouble with your version.
