jllatham say:

OK, I think I have it now, and depending on your needs it can either be almost instantaneous or take about 6-10 seconds to finish with existing data.

This moves the Activity entries from the Date sheet into the matrix on the Indexed Data sheet.  There is a button on the 'Date' sheet to do the job, or you can run the FillDataMatrix macro directly from the View Macros --> View Macros ribbon menu.  The code automatically figures out how many rows are on the Date sheet and how many rows/columns are used on the Indexed Data sheet.

Here are links to the file:

View link (you can download from there)

http://www.mediafire.com/view/yx3dn2j545w52n8/IndexVBA4.xlsm

or a direct download link to the same file

http://www.mediafire.com/download/yx3dn2j545w52n8/IndexVBA4.xlsm

Here is the code that does the work, and I'll have more to say about that after it:

Sub FillDataMatrix()
  Const dateSheetName = "Date"
  Const dateIDCol = 1         'column A
  Const dateDateCol = 2       'column B
  Const dateActivityCol = 3   'column C
  Const firstDSRow = 2        'first row with data on Date sheet

  Const IDSheetName = "Indexed Data"
  Const idLoginCol = 1  'column with login ids in it
  Const idDatesRow = 1  'row with dates in it

  Dim dateWS As Worksheet
  Dim matrixWS As Worksheet
  Dim idList As Object
  Dim dateList As Object
  Dim anyLastCell As Long
  Dim cellPtr As Long
  Dim loginToFind As String
  Dim dateToFind As String
  Dim anyTempEntry As String
  Dim lastMatrixRow As Long
  Dim lastMatrixCol As Long
Dim startTime As Date
Dim endTime As Date
startTime = Now()

  Set dateWS = ThisWorkbook.Worksheets(dateSheetName)
  Set matrixWS = ThisWorkbook.Worksheets(IDSheetName)
  Set idList = CreateObject("Scripting.Dictionary")
  Set dateList = CreateObject("Scripting.dictionary")

  With matrixWS
    anyLastCell = .Cells(Rows.Count, idLoginCol).End(xlUp).Row
    lastMatrixRow = anyLastCell
    'fill dictionary object idList with login ids & their row
    For cellPtr = 1 To anyLastCell ' work through rows
      anyTempEntry = CStr(.Cells(cellPtr, idLoginCol))
      If Not idList.exists(anyTempEntry) Then
        idList.Add anyTempEntry, cellPtr
      End If
    Next
    'fill dictionary object dateList with dates & their column
    anyLastCell = .Cells(idDatesRow, Columns.Count).End(xlToLeft).Column
    lastMatrixCol = anyLastCell
    For cellPtr = 1 To anyLastCell ' work through columns
      anyTempEntry = CStr(.Cells(idDatesRow, cellPtr))
      If Not dateList.exists(anyTempEntry) Then
        dateList.Add anyTempEntry, cellPtr
      End If
    Next
  End With
  Application.ScreenUpdating = False ' improves speed a bit
  'clear out old data, cell formatting, conditional formatting
  'within the matrix area beginning at cell B2
  With matrixWS.Range(Cells(2, 2), Cells(lastMatrixRow, lastMatrixCol))
    .ClearContents
    .FormatConditions.Delete
    .Interior.Color = xlNone
  End With
  'get last used cell on the Date sheet
  anyLastCell = dateWS.Cells(Rows.Count, dateIDCol).End(xlUp).Row
  'the actual data movement
  For cellPtr = 2 To anyLastCell
    loginToFind = CStr(dateWS.Cells(cellPtr, dateIDCol))
    dateToFind = CStr(dateWS.Cells(cellPtr, dateDateCol))
'*******************************************************************************
'Choose one of these 2 lines of code to fill the matrix with:
'This line will fill it almost instantly!!  But formats are not copied.
    matrixWS.Cells(idList.Item(loginToFind), dateList.Item(dateToFind)) = _
     dateWS.Cells(cellPtr, dateActivityCol)
'if you need the source cell formatting copied, use this line of code
'instead of the one above - time will be about 10 seconds or so to finish
'    dateWS.Cells(cellPtr, dateActivityCol).Copy _
     Destination:=matrixWS.Cells(idList.Item(loginToFind), dateList.Item(dateToFind))
'Activate the above line by removing the ' at the beginning of it
'*******************************************************************************
  Next
endTime = Now()
  MsgBox "Time to fill matrix: " & Format(endTime - startTime, "hh:mm:ss") & " (HH:MM:SS).", _
   vbOKOnly + vbInformation, "Task Completed"
End Sub

Notice the section between the lines of **********s

As implemented, this will do the job in the blink of an eye!  But if you need to have the formatting come over from the [Date] sheet Activity entries, then replace that section with this version of it:

'*******************************************************************************
'Choose one of these 2 lines of code to fill the matrix with:
'This line will fill it almost instantly!!  But formats are not copied.
'    matrixWS.Cells(idList.Item(loginToFind), dateList.Item(dateToFind)) = _
     dateWS.Cells(cellPtr, dateActivityCol)
'if you need the source cell formatting copied, use this line of code
'instead of the one above - time will be about 10 seconds or so to finish
    dateWS.Cells(cellPtr, dateActivityCol).Copy _
     Destination:=matrixWS.Cells(idList.Item(loginToFind), dateList.Item(dateToFind))
'Activate the above line by removing the ' at the beginning of it
'*******************************************************************************

That will slow it down considerably but the formatting will be copied over also.  It takes about 6-10 seconds this way using the current test data.

To test, simply go to the [Indexed Data] sheet and delete all the entries in the range B2:AE21 and then run the macro - either directly or with the button on the [Date] sheet.

Hope I got it right for you this time.

jllatham say:

There is one thing to remember if you modify the code for another project - when you "fill" the Dictionary object you have to place text values into the 'Key' and a number into the Item like this section of code did:

    For cellPtr = 1 To anyLastCell ' work through rows
      anyTempEntry = CStr(.Cells(cellPtr, idLoginCol))
      If Not idList.exists(anyTempEntry) Then
        idList.Add anyTempEntry, cellPtr
      End If
    Next

The first bold line above turned the Login IDs into text, then the second bold line added that text as the Key and the value of cellPtr (a row number in this case) as the new .Item in it.

The routine is already very adaptable - just by changing the values of the Const declarations you can pretty much adapt it to any similar situation, although the main, final loop is tailored to fill a matrix that has its upper-left corner at cell A1.

All the theme material is normally included in a PowerPoint template. You only need to send .thmx files if the client needs to update older PowerPoint, Word or Excel files to match the appearance of the presentation.

The best practice for sending files to a client is to find out the least common denominator: the lowest or least capable version they are using. Word templates are similar, no additional files, other than perhaps fonts, are necessary.
