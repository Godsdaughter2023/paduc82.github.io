andreas killer say:
We can subscript all numbers in cell easily with a macro. 

Sub SubscriptNumbersInActiveCell()
  Dim S As String, i As Long
  With ActiveCell
    For i = 1 To Len(.Value)
      If Mid$(.Value, i, 1) Like "#" Then
        .Characters(i, 1).Font.Subscript = True
      End If
    Next
  End With
End Sub


But a superscript O is not a degree sign, see screenshot below. If your keyboard doesn't provide this sign, you can create it this way:

Press the ALT key and hold it down
On the numerical keypad press the numbers 0176
Release the ALT key

Anyway, we can also superscript any o in a cell as well 

Sub SuperscriptOInActiveCell()
  Dim S As String, i As Long
  With ActiveCell
    For i = 1 To Len(.Value)
      If StrComp(Mid$(.Value, i, 1), "O", vbTextCompare) = 0 Then
        .Characters(i, 1).Font.Superscript = True
      End If
    Next
  End With
End Sub

hansv mvp say:

Hosting? If you mean displaying - yes.

Open the form in design view.

In the Controls group of the Design tab of the ribbon, click ActiveX Controls.

Select Microsoft Web Browser in the list, then click OK.

By default, the control will be named WebBrowser0 or similar, but you can change that of course.

To display a specific web page in the control, use code like this:

Me.WebBrowser0.Navigate2 "http://www.eileenslounge.com"

macropod say:
Without something to test every hyperlink to find out whether it's broken, it simply isn't possible to do as you ask. Moreover, implementing something such could dramatically increase the time it takes for Word to make a file available to the user when opening it. It was for that reason that, some years ago, Microsoft changed the update behaviour of a number of fields used by Word so they don't update automatically. Even if hyperlink fields (which often includes email addresses) updated automatically, that still wouldn't tell you whether what they're pointing to is available - and sometimes that may not even be of any consequence.

matts_22 say:

I'm a software developer and ran into a similar problem with emails that I was generating from code. I included user avatars and they displayed correctly in most clients, but not in Outlook for Mac.

The fix for me was to change the embeded image's Content-type from "image/jpg" to "image/png".


Because you are creating your email directly in Outlook, you probably don't have the ability to change the Content-type yourself. But you should try saving the screenshots you are adding as .png. Maybe that will fix the issue for you.
