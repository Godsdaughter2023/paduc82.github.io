joeu2004 say:

This is a common problem due to the way that Excel stores numbers, namely 64-bit binary floating-point.  Most decimal fractions cannot be represented exactly.  So anomalies arise with even simple binary arithmetic.

The remedy is:  whenever you expect a calculation to be accurate to n decimal places (2 or 3 in your case), explicitly round to that number of decimal places.  (Not to an arbitrary number of decimal places, as some people suggest.)

For example:  =ROUND(C21-C22, 2) if C21 is 468.3 and C22 is 466.5.

bernard liengme say:

Nothing strange here; it is a result of how Excel, and most numerical apps, store decimal numbers in binary.

Use a formula such as =ROUND(C20-C19,2) to avoid the problem

I would like to explain that, with only the Project Online Essentials license, the user can’t create a project and access the PWA site if the PWA admin doesn’t share the PWA site collection with her/him. A user is assigned with a Project Online Professional license and then he can click Project tile to provision the PWA site collection. After that, as a PWA admin, he/she can create a Project in PWA site and share it with other users who only have the Project Online Essentials license. The PWA admin add these users to the Team Members for Project Web App group when sharing the PWA site. The users can access and use Project Online in SharePoint.  

The PWA admin must share the root site in the PWA site collection with users, the user is given a specific set of permissions at the same time the root site is shared. When I click on IE explorer (Windows 7) the page pops up with the following error message "You are not connected to a network". However, there is no network. The desktop PC is wire connected to a Comcast modem.

That message may only be an indication that a DNS lookup failed.   Use a cmd  window to check that possibility.   E.g. use  tracert  with the problem host name.  I imagine that you would find that the Comcast modem is set up as a router and that you could get some connectivity diagnostics from it.   Don't worry if the tracert doesn't get to the final destination.  It uses ICMP and hence may be blocked by some hosts continuously but it should give you a clearer idea of the state of your networking than just using ping (another ICMP program).
