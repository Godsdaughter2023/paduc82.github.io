Check for existence, if there then do nothing, otherwise add controls...


Private Sub Workbook_Open()
    Dim subMenu                 As Object

    Set subMenuItem = subMenu.Controls(SUBMENU_BAR_NAME)

    subMenu.Controls(SUBMENU_BAR_NAME).OnAction = SUBMENU_BAR_EVENT

Delete then re-add.  This (as I do it) puts the following code as the Open/Close events...


Private Sub Workbook_Open()
    Call DeleteMenu
    Call CreateMenu

Private Sub Workbook_BeforeClose(Cancel As Boolean)
    Call DeleteMenu
End Sub

Sub CreateMenu()
    Dim subMenu                 As Object
    Dim subMenuItem             As Object
    On Error Resume Next
    Set subMenu = CommandBars(MAIN_MENU_BAR).Controls.Add(Type:=msoControlPopup, Before:=7)
    subMenu.Caption = MENU_BAR_NAME
    Set subMenuItem = subMenu.Controls(SUBMENU_BAR_NAME)

        subMenu.Controls.Add(Type:=msoControlButton).Caption = SUBMENU_BAR_NAME
        subMenu.Controls(subMenu.Controls.Count).OnAction = SUBMENU_BAR_EVENT

I wonder why it wouldn't explicitly reference the control it just named...

        subMenu.Controls.Add(Type:=msoControlButton).Caption = SUBMENU_BAR_NAME
        subMenu.Controls(SUBMENU_BAR_NAME).OnAction = SUBMENU_BAR_EVENT

In the first set it references the count of controls.  While it may be an assumption that the last control you added would go last, it's still an assumption.  Also, in the Close event, it is not as it should be.  Look at the If/Then structure.  The first time it is ran, the menu isn't deleted, just the controls.  It is needed to be run a second time to get the ElseIf portion here...

        ElseIf subMenu.Controls.Count = 0 Then

    On Local Error Resume Next
