IMAP Email Data Migration - Transient error - Microsoft Community

greg wickens say:

IMAP Email Data Migration - Transient error FolderHierarchyContainsMultipleRootsTransientException has occurred. The system will retry (231/600).

Thought I would share an issue that I found whilst I was doing an IMAP Data migration/synchronization and couldn't find information online.

The synchronization would begin and run for several hours but would eventually fail. When i checked the report in Exchange it was looping the same error message 600 times the error message is: 

Transient error FolderHierarchyContainsMultipleRootsTransientException has occurred. The system will retry (231/600).  

After digging into the error message and looking closely at the original mail server what this error was referring to was that there were embedded folders within the key folders like Junk, Trash and Sent. This obviously doesn't fit with the IMAP standards and the exchange server couldn't accept it as it was.

Once the sub-folders were moved out of the key folders I re-started the sync process and could see from the exchange process that data was transferring. The process completed once all data had been migrated correctly.

Hi Amanda

There will be some discrepancy between the folders in your original mailbox and the IMAP standards.

If you are using a email client such as outlook then you will be better off looking at the webmail as some of the folders may not be enabled for client software to subscribe to.

The problem could be that you have embedded folders in the key folders or you could have duplicate folders but the webmail is where you will get your best idea of this as that is essentially what exchange is looking at when syncing the data.

If you are unable to find the cause i would suggest doing a pst upload instead. I will give you a link to this but it's slightly more complicated than the IMAP migration process.

qing tang msft say:

Hi Brian,

 

I suggest you try the following steps to check the outcome:

1. Download the SharePoint Online Management Shell.

2. Run the SharePoint Online Management Shell as administrator.

3. Type the following commands:

a. Connect-SPOService  -url https://<yourtenancy>-admin.sharepoint.com/

b. Set-SPOSite -Identity https://<yourtenancy>.sharepoint.com/sites/<sitename> -LockState Unlock

c. Set-SPOSite -Identity https://<yourtenancy>.sharepoint.com/sites/<sitename> -owner admin@domain.onmicrosoft.com

d. Remove-SPOSite -Identity https://<yourtenancy>.sharepoint.com/sites/<sitename>

4. Go to SharePoint online to see the results.
