a. user say:

I suspect that the Globally Unique Identifier (GUID) use to start the MS Software Shadow Copy Service and the GUID for the Service in the registry have gotten mismatched (somehow) and they need to be the same.

Click Start, Control Panel, Administrative Tools, Component Services.

Select Component Services, expand Computers, (click the plus sign), expand My Computer, expand and select COM+ Applications.

In the top bar, click View, Status to see the Name and Application IDs for the COM+ applications and expand the window and columns so you can see the Application ID for the MS Software Shadow Copy Provider.

You will see the Application ID GUID (Globally Unique Identifier) for the MS Software Shadow Copy Service which is a string of numbers and letters enclosed in curly braces that looks something like this:

{0EA08B29-CD1C-4FA3-BD71-7647A2D2D6F2}

If the GUID that you see in Component Services does not match what is in the registry, the VSS Service will not start and you will have those kinds of error messages.

Now check the command line in the registry that is used to start the MS Software Shadow Copy Service.

Since there is no undo or quit without saving in the registry editor, you may want to backup your registry first using this popular free tool:

http://www.snapfiles.com/get/erunt.html

Click Start, Run and in the box enter:

regedit

Click OK to open the Registry Editor and navigate to the following key to see the settings for the MS Software Shadow Copy Service:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SwPrv

Take a look at ImagePath variable and it will/should look something like this:

C:\WINDOWS\system32\dllhost.exe /Processid:{0EA08B29-CD1C-4FA3-BD71-7647A2D2D6F2}

Notice that in this case, where things work properly. the GUID from Component Services matches the GUID for the MS Software Shadow Copy Service in the registry.  If they do not match, the VSS Service will not start and you will see those kinds of error messages.

Take a look at the GUID in the ImagePath value and compare it to the Application ID GUID noted earlier in Component Services.

If the GUID in the registry does not match the GUID in Component Services, double click the ImagePath value and edit it in the registry so it does match.

Close the registry editor and see if the MS Software Shadow Copy Service will start now.

If the Service starts, reboot the computer to be sure the changes "stick" and be sure the MS Software Shadow Copy Service starts automatically (if that is how you have it configured) or manually on demand.
