To get the header of the first occurrence of X, try this formula in cell M2 and copy down

=INDEX(A$1:L$1,MATCH(TRUE,INDEX(A2:L2="X",,),0),1)

To get the header of the last occurrence of X, try this formula in cell N2 and copy down

=LOOKUP(2,1/(A2:L2="X"),A$1:L$1)

john korchok say:

It's a bug, or at least a poor interface design choice. When you right-click on a brace, you see Set as  Default Shape. But PowerPoint considers it a line object instead of a shape.

Choose the line shape and draw a line, then set it to the color and weight you want to appear in the braces and brackets. Right-click and choose Set as Default Line. Now when you insert a brace, it will have your preferred settings.

I've created a new suggestion in the PowerPoint User Voice forum requesting that this be fixed. Please add your vote: Braces and Brackets: Right-click should display "Set as Default Line"

Recommended work-around for COUNTIF with big text

Problem What went wrong
Wrong value returned for long strings.

The COUNTIF function returns incorrect results when you use it to match strings longer than 255 characters.

Problem What went wrong

To match strings longer than 255 characters, use the CONCATENATE function or the concatenate operator &. For example, =COUNTIF(A2:A5,"long string"&"another long string").

Now that sounds like it would be suitable for my situation but I couldn't get it to work no-way-no-how. Trying to find COUNTIF(...)>1 to identify duplicates 600 characters long for a conditional formatting rule.

=COUNTIF(B:B, LEFT(B2, 255)&IFERROR(MID(B2, 256, 255), TEXT(,))&IFERROR(MID(B2, 512, 255), TEXT(,)))

Sample data/situation at: Identify Duplicates (long text)

jordan dom say:

Hi Jeeped,

We took a look at your concern and found a workaround using the SUMPRODUCT formula in Excel. The formula itself allows Excel to find duplicates longer than the 255 character string limit. Using conditional formatting, enter the formula below to highlight duplicates:

=SUMPRODUCT(--(B:B=B2))

See the sample image below to see the formula in use:



Feel free to reach out to us for any further concerns.

jeeped say:
  1. SUMPRODUCT should never be used with full column references.

  3. Will the 'rogue instructions' in support.office.com be addressed and corrected any time soon?

An alternate UDF based solution for COUNTIF criteria > 255 characters may be found here. And NO, that is not spam.
