shenan stanley say:

You can use built in methods to backup as well, even as simple as file copy+paste, maybe a scheduled task running a script even.

Unless you are rotating out external media (sometimes a full backup is disconnected and stored in a different, but secure, physical location) you still do not meet the criteria for a good backup.

Think of it this way: if your backup is continuously connected to your computer in the same room, same electrical outlets, etc. - whatever destroys the original could destroy the backup - so it is better than having no backups, but it is a 'backup methodology' that is easily affected by the same malady as the original given its proximity to said original.  Worthless, it is not.  It has a lot of utility and is a great first layer of protection that can (likely will) save your butt in normal circumstances.  If you get that ransomware or there is a fire however, the backups are likely as toasty as the original.
