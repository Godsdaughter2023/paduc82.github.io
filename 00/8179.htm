myall_blues say:

If you're doing this as conditional formatting then there is no real need to put a formula in the cell to make it evaluate to zero, and then conditional format on the basis of the zero value.

Conditional formatting works on a TRUE/FALSE basis - if the conditional format condition evaluates to TRUE, it does the formatting, if not, it doesn't.

So (without seeing your worksheet) it should be possible to just make the conditional format condition like this:

=IF(CC$2>=$F3,TRUE,FALSE)

Hope I haven't misunderstood what you want.

Murray

tom van stiphout say:
No, as soon as you type "Me.Forms" Intellisense stops responding and you're on the wrong track. If you want to update a subform from a parent form, the syntax is:
Me.mySubformControl.Form.myControlInSubform = somevalue

If you then want to call that control's AfterUpdate event, you have to do two things. First you need to make that event procedure Public so you can call it from other forms (i.e. its parent). Then you call it with very similar syntax to the above:
Me.mySubformControl.Form.txtActlength_AfterUpdate
