gary zhu msft support say:

Hi Stuart,

First, please check if the problematic user is assigned the mobile device mailbox policy like other normal users are.

To solve the issue, please try the following:

  1. Go to the Exchange Admin Center (https://outlook.office365.com/ecp).

  2. In the left navigate pane, select Mobile.

  3. Select the device that is quarantined in Quarantined Devices and check if the Allow button is available.

  4. Go to recipients > mailboxes > double click the problematic user’s account > mailbox features > scroll down and click View details in Mobile Devices > select the device and see if the Allow button is available.

  5. If the buttons in the two places are both greyed out, it is recommended to remove the device by clicking “” icon in the mobile device details window, and then re-configure the user’s Office 365 account on the Outlook for iOS/Android app to check if the Allow button is available.

Based on our test, you can add the task creator's email and the email address of the person who is assigned to the task to three-state workflow by the following steps.
