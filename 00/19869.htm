robert long [msft] say:

Yes, I think there may be.  You can use the "Open from URL" service a couple ways.  One way I will call 'normal OFU' and the other 'embedded OFU'.  In each case, you can use a query string parameter to 'allow typing', which is what I call 'scratchpad' mode.  And in each case you can also download the file locally.

 

In 'normal OFU' you would have a URL such as the following.You can add the  'allowtyping' query string parameter to allow users to type into the spreadsheet using scratchpad mode. Nothing is saved in this mode.  The view will include a Download button in the toolbar at top so users can download the file locally.

http://view.officeapps.live.com/op/view.aspx?src=<link to your file>&allowtyping=true

 

In 'embedded OFU' you would have a URL such as the following. You can add the  'allowtyping' query string parameter and the 'wdDownloadButton' query string parameter. As above the 'allowtyping' will allow users to type into the spreadsheet in scratchpad mode. If you add the 'wdDownloadButton' query string parameter the embedded view will include a button at lower right in the frame to allow users to download the file locally.

http://view.officeapps.live.com/op/embed.aspx?src=<link to your file>&allowtyping=true&wdDownloadButton=True

 

This page here gives you an overview of all the query string parameters you can use with embedding:

http://office.microsoft.com/en-us/web-apps-help/customize-how-your-excel-workbook-is-embedded-HA102004643.aspx

 

I will also mention we have other APIs you may find interesting and want to investigate.  This site (http://www.excelmashup.com/) provides more info about our APIs including Excel Interactive view, our JS API, our REST API and SOAP API.  (OFU is not covered at this site.)   

 

I hope I haven't overwehelmed you with links and info and that you find this helpful. Good luck and we'd love your feedback if there is something you're trying to do that we don't currently support or which isn't working well for you.  Thanks.

 

-Rob

Is there possibly a RUN CMD option to do this?  

Also, what is MpCmdRun.exe? My snooping in Program Files / Windows Defender spotted this.

Thanks for ANY ideas!

greginmich say:

You actually need to run a group of commands (batch file) in order to do this right:

  • The first command increments the sleep timeout to an interval that will prevent the system from idling to sleep, which would otherwise pause the scan.
  • The second command starts the scan.
  • The third command resets the sleep timeout to a value of your choice.
  • The last command shuts down the PC.

All you have to do is copy and paste the “code” below to a Notepad file, and then name the file Scan and Shutdown.cmd – and save it to your desktop. Then double-click to run the scan.

@echo off

powercfg -change -standby-timeout-ac 200

Cd C:\Program Files\Windows Defender

MpCmdRun.exe -Scan -ScanType 2

powercfg -change -standby-timeout-ac 20

shutdown -s -t 300

You can adjust the timeouts to suit your needs: The powercfg sleep timeouts are in minutes, and the shutdown timeout is in seconds. You also might want to add a Cancel Button batch file, just in case you change your mind at the last minute. This is all you need to stop the countdown:

@echo off

shutdown /a

If you'd rather run your Full Scan in the familiar Windows Defender UI, then you can use this “timed” batch file:

powercfg -change -standby-timeout-ac 50

"C:\Program Files\Windows Defender\MSASCui.exe" -FullScan

timeout /t 2400  /nobreak

powercfg -change -standby-timeout-ac 20

shutdown -s -t 300

For example, my Full Scans only take about 35 minutes to run, so my batch file sets the sleep timeout to 50 minutes; starts the scan; and then halts execution for 40 minutes (2400 seconds) in order to let the scan run. Then when the batch file resumes execution, the sleep timeout is reset to 20 minutes, and then the PC is shut down 5 minutes later.
