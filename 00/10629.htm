'  If Range("A2") <> "" Then GoTo ws_exit

Private Sub Worksheet_Change(ByVal Target As Range)
    On Error GoTo ws_exit:
    Application.EnableEvents = False
    If Not Intersect(Target, Me.Range("C7")) Is Nothing Then
    If Target.Value <> "" Then
'    If Range("A2") <> "" Then GoTo ws_exit
        Range("A2").Value = Format(Now, "dd mmm yyyy hh:mm:ss")
    End If
    End If
ws_exit:
    Application.EnableEvents = True
End Sub

jay freedman say:

I'm guessing here, but it sounds like the Next Page section break was inside the TOC field result. That would account for the section break and the roman numerals (which are stored inside the section break) disappearing on an update.

In the document, press Alt+F9 to toggle all fields to their field codes. The table of contents will collapse to a single field code like { TOC \o "1-3" \h \z \u }. Move the cursor to the next paragraph and insert the Next Page section break there. Press Alt+F9 again to return to field results. Change the page numbering in section 1 to roman numerals, unlink the Section 2 header and footer from those in section 1 if needed, and restart page numbering from 1 in section 2.

Your TOC should now be stable and updatable.

mike middleton say: I enter labels Data, Bins, and Frequency in A1:C1.

My fifty data values are in A2:A51.

In B2:B12 I enter -0.5, -0.4, ..., 0.5, respectively.

I select C2:C13 (one more cell than the bin range), type =FREQUENCY(A2:A51,B2:B12), hold down Ctrl and Shift, and press Enter.

Cell C2 is the count for data values less than or equal to -0.5.

Cell C13 is the count for data values strictly greater than 0.5.

Excel Help for "frequency function" says: The number of elements in the returned array is one more than the number of elements in bins_array. The extra element in the returned array returns the count of any values above the highest interval.
