Websites are reported via the Internet Explorer 8 SmartScreen Filter by selecting the Safety pull-down menu, SmartScreen Filter, Report Unsafe Website, which will submit the current web site URL and allow you to indicate that it is a ohishing site or contains malicious software.  To try it, just click those selections anytime, but don't actually submit the site if it's not 'bad' so you can see the submission form.

Don't worry about emails, only the bad URLs or malware attachments really matter, since most spam today is automatically changed too fast to be matched in a simple way.  If you can submit via an existing simple method like Hotmail, great, but otherwise the malicious payloads or links are the really dangerous parts that matter.  Most of the general 'email us your spam' addresses have been de-commissioned, since there's no one to actually investigate them and the automated 'Junk Mail' folder systems built-in to Hotmail and other web mail interfaces are much more responsive and easy to use.  There are still some places to report these, but most are now just big repositories of historical messages with little practical use, so more of a 'feel good' thing than anything truly useful.  See this link for some related info.

http://en.wikipedia.org/wiki/Anti-spam_techniques#Reporting_spam

Don't know about the large malware, but don't think I'd bother, since generally these would just be large packed files containing a group of smaller files some of which might be malware.  If you're uploading a large file to submit a single small file contained within, this is really not of much value anyway, since it's the small file that really matters.

cross reference, field: numeric format switch - Microsoft Community

robert from innsbruck say:

cross reference, field: numeric format switch

Hi everyone,

in Word 2010, I used the field code (Shift+F9) to get the number only from cross reverences e. g.

  • cross reference: Figure 2.1
  • original field code: { REF _Ref469400923 \h }
  • changed field code by adding the numeric format switch: { REF _Ref469400923 \h \# 0.0 }
  • resulting cross reference: 2.1

Now, in Microsoft Office Professional Plus 2016 the same field (updated) turns into 3,0  ??? (obviously 2 and 1 are summed up ?

What happened? For me, this is a serious deficiency of Word 2016! As a result, I am not able to print my "old" documents, because all the cross reference fields are updated then and turn into wrong numbers...

Thank you for any help

Robert

robert from innsbruck say:

Doug Robbins solved the problem!

The Windows system decimal separator (Control Panel>Clock, Language and Region>Formats>Additional Settings) was set to be a comma. If it is changed to a point, the numeric format switch works. Obviously, this is not a Word 2016 problem.

Thank you Doug, for solving this irritating issue!
