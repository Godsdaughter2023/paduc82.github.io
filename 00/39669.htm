arminatorx say:

[quote user="LostProdigy422"]Im staying at a Hampton Inn [hotel] ... my Xbox would connect to the wifi service but says there is no connection to the Internet. Any solutions?

[/quote]

Most hotels require you to sign in on a special welcome webpage before they unlock you for full web access. Maybe they ask for your room number or something else to prevent non-guests from using the WLAN.

Unfortunately, the previous post is of no use to you, as you won't be able to edit the hotel's settings to forward the proper ports.

Depending on how the hotel WLAN operates, one possible solution would be this: Find out the MAC hardware address of your computer's network card.
You do this on a Windows PC by pressing Windowskey + R, then typing "ncpa.cpl", and clicking on OK.

This will open up a window with your PCs network interfaces. Right click on your WLAN adapter and choose "Status". Click on the "Details" button and you should see a list with information about your WLAN hardware.


Find the Line labeled as "Physical Address", which should have six pairs of hexadecimal numbers. They usually look like this: 01-2A-C3-EF-6B-54

Make a note of this MAC address.

Now on your PC properly connect to the hotel WLAN, and make sure you can access the internet properly.

After your internet connection works, turn off the PC and turn on the Xbox.

On the console go all the way to the right to the settings hub, and there to the system tile in the upper left. Go to the network settings and try to connect to the WLAN. Then go to the network configuration, and go right to the additional network settings.

There you get the options to set PPPoE usernames and the likes.

What you need to set is to clone your PC's MAC address to the Xbox. There is an option to customize the WLAN Mac address of the console.

Input the MAC address you copied from your PC into the console, and test the network connection.

However: Some hotels block any network traffic that is not HTTP (website) or Email related. If that is the case, you won't be able to use the Xbox console online in that hotel.

rename zigxrtml.sys   zigxrtml.bad

Renaming or deleting the suspicious file to get the system running is not fixing the problem.  It is fixing the symptom of the problem.

The problem is your system is infected with malicious software and you must still follow up with some scans to remove the infection.

Once your system is running, follow up with MBAM and SAS to be sure the affliction has really been removed.

Reduce the chances of malicious software by running some scans.

Download, install, update and do a full scan with these free malware detection programs then resolve any remaining issues:

SUPERAntiSpyware: (SAS):  http://www.superantispyware.com/

If you have no bootable XP media (or are not sure what you have) create a bootable XP Recovery Console CD and be sure.

Press any key to boot from CD...

The Windows Setup... will proceed.

Press 'R' to enter the Recovery Console.

Select the installation you want to access (usually  1: C:\WINDOWS)

You may be asked to enter the Administrator password (usually empty).

You should be in the C:\WINDOWS folder.  This is the same as the

http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/recovery_console_cmds.mspx?mfr=true

From the command prompt window run the chkdsk command on the drive where Windows is installed to try to repair any problems on the afflicted drive.

Running chkdsk is fine even if it doesn't find any problems.  It will not hurt anything to run it.

Assuming your boot drive is C, run the following command:

chkdsk C: /r

Let chkdsk finish and correct any problems it might find.  

It may take a long time for chkdsk to complete or it may appear to be 'stuck'.  Be patient.  If the HDD light is still flashing, chkdsk is doing something.  Keep an eye on the percentage amount to be sure it is still making progress.  It may even appear to go backwards sometimes.

You should run chkdsk /r again until it finds no errors to correct.
