    How to import text data over the limit of 1,046,576 row in Excel 2007 - Microsoft Community

    alainbeaudoin say:

    How to import text data over the limit of 1,046,576 row in Excel 2007

    I have already tried to open another worksheet and start to import the text file at row 1,046,577 but only get 1 number on the 2nd worksheet
    jllatham say:

    Here is some VBA code you can put into a workbook and split your larget text file into 3 separate text files so that you can import each individually.  It's tailored to your setup, but can be extended if needed to write a 4th, 5th, etc sub-file.

    To put the code to work, open any workbook, even a new one, and press [Alt]+[F11] to open the VB Editor.

    In the VBE, choose  Insert --> Module

    Copy the code below and paste it into the module.

    At this point you can place the cursor anywhere within the code in the module (after Sub... and before End Sub) and press the [F5] key.  The code will start by asking you to browse and identify the source file.  Once you do that it will start reading the source file and splitting it into 3 files named Part1.txt, Part2.txt and Part3.txt {which you can change in the code if you want}.

    It's going to take a little time to run depending on the speed of your system, your network and your hard drives.  It takes at least 1 minute just to count to 1.04M 3 times.

    The code:

    Sub SplitTextFile()
      Dim mainFile As String
      Dim basicPath As String
      Dim recordCount As Long
      Dim inBuff As Integer
      Dim outBuff As Integer
      Dim rawData As String
     
      mainFile = Application.GetOpenFilename()
      If mainFile = "False" Then
        Exit Sub 'user cancelled
      End If
      basicPath = Left(mainFile, InStrRev(mainFile, Application.PathSeparator))
      inBuff = FreeFile()
      Open mainFile For Input As #inBuff
      outBuff = FreeFile()
      Open basicPath & "Part1.txt" For Output As #outBuff
      'first 1048575 rows/records (leaves 1 empty row)
      Do While Not EOF(inBuff) And recordCount < Rows.Count
        Line Input #inBuff, rawData
        Print #outBuff, rawData
        recordCount = recordCount + 1
      Loop
      Close #outBuff
      recordCount = 0 'reset
      outBuff = FreeFile()
      Open basicPath & "Part2.txt" For Output As #outBuff
      'next 1048575 rows (leaves 1 empty row)
      Do While Not EOF(inBuff) And recordCount < Rows.Count
        Line Input #inBuff, rawData
        Print #outBuff, rawData
        recordCount = recordCount + 1
      Loop
      Close #outBuff
      recordCount = 0 'reset
      outBuff = FreeFile()
      Open basicPath & "Part3.txt" For Output As #outBuff
      'remainder of records
      Do While Not EOF(inBuff) And recordCount < Rows.Count
        Line Input #inBuff, rawData
        Print #outBuff, rawData
        recordCount = recordCount + 1
      Loop
      Close #outBuff
      Close #inBuff
      MsgBox "File has been split " & recordCount & _
       " records written to the last file."
    End Sub

Disabling the card solved all issues, but I hope that there will be real fix rather soon...
