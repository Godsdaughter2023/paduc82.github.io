Hi Mike H. I'm using your formula to calculate SDLT. The Inland Revenue's calculator rounds the figure down to the nearest pound. Example, if I put £364,950.00 it comes out as £8,247.00. How should your formula be changed to make it round down?

=INT(SUMPRODUCT(--(A1>{125000,250000,925000,1500000}),

  (A1-{125000,250000,925000,1500000}), {0.02,0.03,0.05,0.02}))

I verified that these parameters still produce the same result as the Inland Revenue calculator for a purchase today, 08/04/2015 (dd/mm/yyyy format, like the SDLT calculator).

    bernie deitrick say:

    This macro will create a new sheet for every unique value in column D - it is hard to say how well it will work since you have hidden rows and you also don't show the values you want on the exported sheet.  But this will get you going.

    To start using macros in Excel, visit this page and follow the instructions:

    http://peltiertech.com/how-to-use-someone-elses-macro/

    but instead of the example macro on that page, use my code.

    Sub ExportDatabaseToSeparateSheets()
        'Export is based on the values in column D
        Dim rngCell As Range
        Dim shtTemp As Worksheet
        Dim strName As String
        Dim rngKeyValues As Range
        Dim strShtName As String
        Dim iKeyCol As Integer
        
        strShtName = ActiveSheet.Name
        
        iKeyCol = 4
        
        
        Set rngKeyValues = ActiveSheet.UsedRange.Columns(iKeyCol).Offset(1, 0).Cells
        
        Set rngKeyValues = rngKeyValues.Resize(rngKeyValues.Rows.Count - 1, 1)
        
        For Each rngCell In rngKeyValues
            On Error GoTo NoSheet
            strName = Worksheets(rngCell.Value).Name
            GoTo SheetExists:
    NoSheet:
            Set shtTemp = Worksheets.Add(After:=Worksheets(Worksheets.Count))
            shtTemp.Name = rngCell.Value
            With rngCell.CurrentRegion
                .AutoFilter Field:=iKeyCol, Criteria1:=rngCell.Value
                'These lines copy the filtered data, and also extra header rows
                ' and any SUBTOTAL formulas separated by blank row
                ' Comment them out to not use them
                rngCell.Parent.Cells.SpecialCells(xlCellTypeVisible).Copy
                shtTemp.Range("A1").PasteSpecial xlPasteValues
                shtTemp.Range("A1").PasteSpecial xlPasteFormats
                
                'These only copy the database values
                '.SpecialCells(xlCellTypeVisible).Copy shtTemp.Range("A1")
                shtTemp.Cells.EntireColumn.AutoFit
                .AutoFilter
            End With
            Resume
    SheetExists:
        Next rngCell
        
    End Sub

    POP email issue fixed - see first post in the following thread

    FIXED 3/4/16: Problems with POP accounts in Outlook 2016: Email is deleted from server or duplicated in Inbox
    http://answers.microsoft.com/en-us/office/forum/office_2016-outlook/fixed-3416-problems-with-pop-accounts-in-outlook/79e8177f-b04e-491e-9a82-ae5b6565d3c0 

    Pay special attention to the following in MS KB 3145116 - https://support.microsoft.com/en-us/kb/3145116

    Note: After the fix is applied, Outlook may download all email one final time. If this occurs and email is duplicated in your Inbox, use the method described in the "More Information" section.

    The operative word in the above is "may". If you have a lot of data in your <Inbox> and want to take some additional precautionary steps avoiding newly downloaded dupes as an alternative to what's suggested - the process is as follows <before> connecting to the POP account the first time with the updated build

    #1 - Create a new mail folder
    #2 - Highlight your Inbox
         --> move focus to <Inbox> items
         --> CTRL+A to highlight all items
         --> right-click 
         --> select <Move> 
         --> Other folder 
         --> select the newly created folder
    #3 - If the update results in everything being downloaded again, you will have a clean inbox without dupes. If the items are not downloaded, then, you will need to reverse the process and copy everything back to the <Inbox> 

    Also, for those using <rules> to move inbox items to other folders, you will probably want to turn those off during the first connection to your account in the event that everything gets downloaded again - else you will have dupes in all the other folders.
