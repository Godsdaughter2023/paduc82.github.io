hansv mvp say:

Let's say the home teams are entered in B2:B50 and the away teams in F2:F50.

Right-click the sheet tab.

Select 'View Code' from the context menu.

Copy the following code into the worksheet module:

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim rngCell As Range
    If Not Intersect(Range("B2:B50"), Target) Is Nothing Then
        Application.ScreenUpdating = False
        Application.EnableEvents = False
        For Each rngCell In Intersect(Range("B2:B50"), Target)
            If rngCell.Value = "NO SHOW" Then
                rngCell.Offset(0, 1).Value = 0
                rngCell.Offset(0, 3).Value = 1
            End If
        Next rngCell
        Application.EnableEvents = True
        Application.ScreenUpdating = True
    End If
    If Not Intersect(Range("F2:F50"), Target) Is Nothing Then
        Application.ScreenUpdating = False
        Application.EnableEvents = False
        For Each rngCell In Intersect(Range("F2:F50"), Target)
            If rngCell.Value = "NO SHOW" Then
                rngCell.Offset(0, -1).Value = 0
                rngCell.Offset(0, -3).Value = 1
            End If
        Next rngCell
        Application.EnableEvents = True
        Application.ScreenUpdating = True
    End If
End Sub

As you enter NO SHOW in column B or F, the score will automatically be filled in.

security flaw in .xlsm format - Microsoft Community

jerry w. lewis say:

security flaw in .xlsm format

An .xlsm file that contains VBA function definitions that are used in worksheet cells, may at run-time substitute (without warning or traceability) external VBA functions of the same name.  This would seem to make it impossible to consider as validated any template stored as an .xlsm file.

 

When you open an .xlsm (but not an .xls) workbook in Excel 2010, Excel re-determines how to interpret cell references to UDFs (user defined functions), and amazingly external Add-Ins take precedence over UDFs defined in the workbook itself!  This has been previously observed, e.g.

  http://stackoverflow.com/questions/5503526/xll-and-vba-function-name-collision-leads-to-permanent-name-error

but no one seems to have adequately thought through the implications.  My concern is that if an .xlsm uses an internally defined UDF, a subsequently installed Add-In that contains a function of the same name could hi-jack the calculation without any apparent changes to the .xlsm!

andreas killer say:
Well, you can prevent that the wrong UDF is called if you use the filename in your formula. Same way as you refer cells in other files.

Example:

- Make a new file.
- Copy this UDF into a normal module.

Function MyUDF() As String
  MyUDF = "MyUDF"
End Function

- Save the file as test.xlsm
- Create another new file.
- Copy this UDF into a normal module.

Function MyUDF() As String
  MyUDF = "HiJack"
End Function

- Switch to Excel, click the "fx" button.
- Choose Custom as category
- Select "test.xlsm!MyUDF"
- Click Ok, Ok

That leads to the formula
=test.xlsm!MyUDF()

- Copy that formula into test.xlsm
- Save and close test.xlsm

As you see the formula in the remaining file updates.

- Save the file as test2.xlsm
- Close that file.
- Open test.xlsm and change the UDF to

Function MyUDF() As String
  MyUDF = "Juhu"
End Function

- Save and close that file.
- Open both files.

The result of the UDF is different, because only test2.xlsm is calculated, but the Hi-Jack UDF is never called.

But if you installed malicious software on your PC, you did not need a XLL, even a simple VBScript can change the formula and e.g. add a random number to your calculation.

This is not a security problem.

Safety can not be delegated to others, if you install a software / add-in, you must decide for yourself whether this is safe.

Andreas.

jerry w. lewis say:

Your suggestion does not adequately solve the problem. 

 

While you are correct that using =test.xlsm!MyUDF() instead of =MyUDF() for all formulas in test.xlsm will prevent the formulas from being hijacked when test.xlsm is next opened, it will also prevent the formulas from working if test.xlsm is moved to a new location.  Standard business practice is to move a template to a production location only after it has been validated ...

 

Moreover, it would require that all templates that are to be saved in .xlsm format be developed using a style of function call that Microsoft has never recommended.

 

This is not just a problem of avoiding malicious code.  It sets up a situation where installing a new and tested AddIn can break an existing tested system that it has nothing to do with, though no fault of anyone but Excel.  Avoiding the problem in a business environment would require when installing a new Add-In, not only testing every existing production .xlsm, but checking with every employee to see if it might break any of their local .xlsm files.  As such, it most certainly is a security problem.

arthur.dent say:

Microsoft's own (and old) app is gone, this has been replaced by an app developed by Facebook.

It is true that the Microsoft-developed Facebook app is no longer available for new installs; it has been replaced by Facebook's in-house app.

Unfortunately, Facebook's app does not work correctly on the original poster's device (BLU Win HD LTE) after upgrading to the Windows 10 Mobile Anniversary Update.

Specifically, Facebook's in-house app fails to identify the BLU Win HD LTE as a handheld device; it attempts to render itself as though it was running on a desktop-class device instead. It also doesn't deal well with the fact that the display always has a zoom factor of at least 150%. These two problems combined lead to a user interface layout that is not well optimized for a small touchscreen, as well as a portion of the app's content being cut off on all four edges of the screen.

The final version of Microsoft's Facebook app does not have these problems, making it much more usable on these devices.

To the original poster: You can get back Microsoft's final version of the app by using the Windows Device Recovery Tool to wipe your phone and revert back to Windows Phone 8.1. Then, re-install the Windows 10 Mobile upgrade using the Upgrade Advisor app. (The Anniversary Update will automatically be downloaded and re-installed some time later.) Disable automatic updates inside the Windows Store app, and opt to install app updates manually instead.

You'll be offered to manually update the Facebook app. The update you download by this mechanism will be the final version of the Microsoft-developed Facebook app.

Note: I had trouble using a Windows 7 PC to run the Windows Device Recovery Tool for the initial downgrade. I was able to successfully downgrade my phone using a Windows 10 PC.

travis62 say:

Excel 2013

Is there a MAX and MIN function that works like AVERAGEIF?  Looking to find the max and min values in a database based on specific criteria.
teylyn say:

Hello,

not in Excel 2013 and earlier. The Office 365 edition of Excel 2016 has new functions MAXIFS and MINIFS, which also work in Excel online, but for Excel 2013 on the desktop you will need to use an array function that nests an IF into the MAX function.

Consider the following screenshot:

The formula in E2 is =MAX(IF($A$2:$A$13=D2,$B$2:$B$13)) and in F2 is =MIN(IF($A$2:$A$13=D2,$B$2:$B$13))

These are array formulas and must be confirmed with Ctrl-Shift-Enter. Excel will then put the curly braces in. You don't type these.
