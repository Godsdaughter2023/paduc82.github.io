rohn007 say:
The TOC is generated from a single field code. So if you place it inside of a table, it will only appear in a single cell of the table. You cannot spread the rows of the TOC over the table.

Thinking about it you can manually build a TOC the way you want.
Create a table, or format the row with tabs (old school 'table')
For EVERY heading entry setup cross reference fields, one reference for heading number, another for the heading text, and one for the page number, then you can enter revision number and date.

This way, if any of the heading info changes it will automatically update. you will have to manually sort the table if the headings are re-organized.

It is a cludge, but at least you would have the info you want and most of it will be automatically updated.
