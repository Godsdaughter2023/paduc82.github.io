If you don't know where DMP files are located, here's how to get to them:

 1.    Navigate to the %systemroot%\Minidump folder.
 2.    Copy any and all DMP files in the Minidump folder to your Desktop and then zip up these files.
 3.    Upload the zip containing the DMP files to Skydrive or a hosting site of your choice and paste in your reply.

http://www.wikihow.com/Use-SkyDrive

Please note that any "cleaner" programs such as TuneUp Utilities, CCleaner, etc, by default will delete DMP files upon use.

If your computer is not generating DMP files, please do the following:

1. Start > type %systemroot% which should show the Windows folder, click on it. Once inside that folder, ensure there is a Minidump folder created. If not, CTRL-SHIFT-N to make a New Folder and name it Minidump.

2. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Performance > Settings > Advanced > Ensure there's a check-mark for 'Automatically manage paging file size for all drives'.

3. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Startup and Recovery > Settings > System Failure > ensure there is a check mark next to 'Write an event to the system log'.

Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.

4. Double check that the WERS is ENABLED:

Start > Search > type services.msc > Under the name tab, find Windows Error Reporting Service > If the status of the service is not Started then right click it and select Start. miekiemoes say:
Hi,

From your post: "If I select an application the error appears"
So this means it wants to load this dll into most applications, so I can only suspect the Appinit_DLLs value in the registry being responsible for this. The Appinit_DLLs will be loaded into every process that links to user32.dll, so basically, this is almost every application, hence why you get that error message so often.
Since the usage of the Appinit_DLLs is not recommended (see http://support.microsoft.com/kb/197571/), I suggest you clear out the valuedata of it.
To do this, navigate in your registry to the following key: HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\Windows
On the right, you will find the Appinit_DLLs value.
Rightclick the Appinit_DLLs value and select to edit in order to edit the string.
You'll see, in the value data path, it will have the path that causes your error message. Clear out that path, leaving the value data field empty (as that's actually the default - empty valuedata there).
Hit OK and reboot your computer.

Note, be careful when you edit the registry - if you are not familiar with the registry at all, I suggest you backup your registry first: http://windows.microsoft.com/en-us/windows/back-up-registry

Let me know if that fixed the problem

jordana_s say:

Hey Wesley, 


There are a few ways to transfer files:

  • Move or copy them from a networked computer to Surface. This works if you have two or more computers using the same network at home  or work, and they are both set up for visibility and file sharing on that network.
  • Use a USB flash drive, external hard drive, or microSD, microSDHC, or microSDXC card to transfer files.
  • Store your files in the cloud and access them through SkyDrive.

For more info, see <Move files to your Surface>.


After your files are transferred to Surface, you can automatically back up your files using File History.

 

File History automatically backs up files that are in your libraries, contacts, favorites, Microsoft SkyDrive, and on your desktop. File History also lets you recover previous versions of files. Read <Backing up content> for more info.

 

<Surface storage options> explains options for where you can keep your files.

All hardware support (including device drivers and software to properly utilize said hardware in *any* operating system) comes from the hardware manufacturer of said product.
 
For the HP Deskjet 3820 Color Inkjet Printer, start here:
Software & Driver Downloads - HP Deskjet 3820 Color Inkjet Printer
 
Choose the proper operating system and architecture to see what drivers HP offers for the HP Deskjet 3820 Color Inkjet Printer.  For Windows 7, HP has chosen to actually *not* support this printer, but links you to this:
Installing the Printer with an Alternate Driver in Windows 7 for a USB Cable Connection

... and tell you to use the Windows Update option when trying to install the printer via the Add Printer wizard and allow the list of available printers to be downloaded and choose the "Deskjet 970cse" as the driver to utilize for the HP Deskjet 3820 Color Inkjet Printer.
 
For the other device, the HP Deskjet 3740 Color Inkjet Printer, you might start here:
Software & Driver Downloads - HP Deskjet 3740 Color Inkjet Printer
 
Choose the proper operating system and architecture to see what drivers HP offers for the HP Deskjet 3740 Color Inkjet Printer.   For Windows 7, HP has chosen to actually *not* support this printer and links you to this:
The Driver for Your Product Is Included in the Windows 7 Operating System

... which tells you that the driver you need is some generic one they gave to Microsoft when Windows 7 was released.  If I were you, I would not attempt to install this printer until after you get the other one installed.  Chances are it will not work (after all, HP pretty much abandoned it) - but it is a better chance after you have downloaded the updated set of drivers because of the other printer.
 
In the end - you may well be better off throwing out the printers and purchasing a new printer (or two - since you seem to have two now) and getting one that is Windows 8 compatible - all but insuring you three or so years of compatibility.
