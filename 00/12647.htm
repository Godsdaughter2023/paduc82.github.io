Now, the true fix to this problem and for anyone else who has the same issue in the future is for those who are running more than "one" hard drive such as SSD + HDD.

If you have changed the location of your Temp Folder to a different drive other than Local C:

Step 1: Go to Start > Computer (My Computer)
Step 2: Right click > Properties
Step 3: Click on Advance System Settings
Step 4: Click on Environment Variables
Step 5: Change Temp & Tmp Folder back to the default location (C:\Users\Name (PC Name)\AppData\Local\Temp)
Step 6: Restart PC and the setup installation should work

Email problems; Windows 7 OS; Avast Antivirus; Outlook 2010

They show being sent (in the sent folder) but they never arrive at the destination. I have spoken with my ISP and the problem isnt on their end. Please advise
brian tillman [outlook mvp 2007-2019] say:

If the message is in the Sent Items folder, then Outlook has sent the message; that is, Outlook has handed it off to what it believes is the outgoing mail server and that server has acknowledged the reception of the message.

Enable diagnostic logging in Outlook.  Here's an article that describes it for Outlook 2007.  It will be similar in Outlook 2010, but you'll need to click the orange File tab to find the Options selection. http://office.microsoft.com/en-us/outlook-help/what-is-the-enable-logging-troubleshooting-option-HA001230421.aspx.  The log that gets generated will contain the messages that Outlook and the server send back and forth to each other during the send/receive and it will give you ammunition that you can use when you contact your mail service provider to explain that the problem is, indeed, theirs because it will contain proof that their server accepted the message but did not forward it on.  A message being deliverable via a web mail interface does not prove anything about the SMTP server that the service provider runs.  The SMTP server can be broken even if the web interface allows message delivery because in the web case, the SMTP server isn't involved.

The log may also indicate that Outlook isn't really talking to the mail service provider's server, but to an add-in on your own computer and then you'll have some indication of what you should examine on your PC.
