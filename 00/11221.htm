A template is a file from which Word creates a document. See: http://shaunakelly.com/word/templates/templaterelations.html. Ordinary documents are usually created from templates via, File|New. Indeed, that would even apply to your original mailmerge main document.

qa guy commelec say: Use an IFF statement in your query.

john w. vinson say:

You can use a Query with a calculated field as the basis for a Form, Report, export, pretty much anything you can do with a table. Table calculated fields are rather limited (they can't call VBA functions for example).

In a Query, try using an expression like this UNTESTED AIR CODE:


TotalCost: Switch([CostPer] = "pound", [Length]*[Width]*[Thickness]*[Cost]*0.0975, [CostPer] = "sqft", [Length]*[Width]*[Cost], [CostPer] = "CuIn", [Length]*[Width]*[Thickness]*[Cost], True, Null)

The Switch function takes arguments in pairs, and evaluates them left to right; when it encounters a pair for which its first member is True it evaluates the second expression in the pair and quits. The last pair will return NULL if the CostPer field is empty or has an invalid value.

The corner indicators are really warnings, not errors.  In this case, if you hover the cursor over the cell, the warning explanation will probably say something about the fact that C448:E448 does not include all contiguous cells in the range, namely A448 and B448, as you discovered.

On rare occasion, the corner warnings identify a real mistake.  But usually, they are just a distraction; they identify differences in formulas that we write on purpose.  So I disable them.  Go to Excel Options (depends on your Excel version) and click Formulas.  Uncheckmark "Enable background error checking".  Also click "Reset Ignored Errors" to remove existing corner warnings.

doug robbins - mvp office apps & services (word) say:

You will need to use a TextInput type formfield, rather than a Date type formfield and then run a macro containing the following code on exit from the formfield:

Dim nummonth As Long
Dim strMonth As String
Dim ffname As String
Dim daynum As Long
Dim yearnum As Long
ffname = Selection.Bookmarks(Selection.Bookmarks.Count).Name
With ActiveDocument.FormFields(ffname)
    If IsDate(.Result) Then
        daynum = Format(.Result, "d")
        yearnum = Format(.Result, "yyyy")
        nummonth = Format(.Result, "m")
        Select Case nummonth
            Case 1
                strMonth = "enero"
            Case 2
                strMonth = "febrero"
            Case 3
                strMonth = "marzo"
            Case 4
                strMonth = "abril"
            Case 5
                trMonth = "mayo"
            Case 6
                strMonth = "junio"
            Case 7
                strMonth = "julio"
            Case 8
                strMonth = "agosto"
            Case 9
                strMonth = "setiembre"
            Case 10
                strMonth = "octubre"
            Case 11
                strMonth = "noviembre"
            Case 12
                strMonth = "diciembre"
        End Select
        .Result = daynum & " de " & strMonth & " de " & yearnum
    Else
        MsgBox "You did not enter a valid date."
    End If
End With

You must assign a bookmark name to the formfield.
