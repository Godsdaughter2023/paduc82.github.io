To eliminate the errors, you would have to use

{ IF { Mergefield FieldName}  <> "" { ={ Mergefield FieldName } * 100 \# "0%" } " " }

If you want 0% to appear where the field is blank, use

{ IF { Mergefield FieldName}  <> "" { ={ Mergefield FieldName } * 100 \# "0%" } "0%" }

Hope this helps.

johnsennekkah say:

Its been a while but think I've found the problem or at least a more elegant fix

Excel on MAC hyperlink security warning:

------------------------------------------------
Opening "file://localhost/<path><filename>"

Some files can contain viruses or otherwise
be harmful to your computer. It is important
to be certain that this file is from a
trustworthy source.

Would you like to open this file?
------------------------------------------------

SOFTWARE 

Microsoft Excel for Mac 2011
Version 14.0.0 (100825)
Latest Installed Update: 14.0.0


OS
ProductName: Mac OS X
ProductVersion: 10.11.4
BuildVersion: 15E65


This all started some years ago when JPG files, hyperlinked in an Excel spreadsheet, would bring up the above security warning each time they were referenced. I found that by loading the files into PS and doing a 'save as' and overwriting the original file, would cure the problem. It seemed that PS was adding/changing something to the file which made Excel less suspicious. At that stage I had neither the time nor inclination to find out the root cause.


Now, a few years down the track, I was attempting to reference .gpx files produced by a Garmin GPS which gave the same warning messages, it was obvious that the PS fix, which worked for .jpg's, was not going to do the job.
 
After a bit of delving, I found that each time a file is saved by PS an Extended Attribute file "com.apple.FinderInfo" is generated if it was not there already. If Excel finds this "com.apple.FinderInfo" in the Resource Fork it will NOT display the warning message. 




So came up with the following:



> mkfile ~/Documents/text_files/finder_info.txt   (adjust path and name to suit)
> xattr -p com.apple.FinderInfo <filename.jpg>  (any image file saved by PS)

this will produce the following Hexcode:

4A 50 45 47 38 42 49 4D 04 00 00 00 00 00 00 00
00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00

Copy/Paste or manually enter this code into the above ‘finder_info.txt' 





Now the following is the only command you really need:



> xattr -wx com.apple.FinderInfo "`cat ~/Documents/files_text/finder_info.txt`" ~/Documents/gps_docos/test.gpx




The “test.gpx” is not limited to one file at the time but can be expanded by wildcards.


Just to check:


> xattr -pl com.apple.FinderInfo ~/Documents/gps_docos/test.gpx

com.apple.FinderInfo:
00000000  4A 50 45 47 38 42 49 4D 04 00 00 00 00 00 00 00  |JPEG8BIM........|
00000010  00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00  |................|




test.gpx could now be hyperlinked from a cell in Excel without the security warning message appearing and does not seem to mind to being referenced as a JPEG in "com.apple.FinderInfo"



[The above might not be the actual reason for the warning messages nor the most elegant solution but it does fix the problem and so far have not experienced any gotta’s]
