Formatting changes from one computer to another are usually the result of one of two things:

- Your presentation uses fonts that aren't available on the other computer; Windows substitutes a different font that may or may not match up well, so the formatting gets thrown off.

- The PC doesn't have an appropriate printer driver installed and set as default.  PowerPoint shouldn't be so fussy about this but it is.

jay freedman say:

There could be a problem with the way the macros are recorded or where they're stored. More likely, there's a problem with the shortcuts.

First verify that the macros actually exist. On the View tab of the ribbon, click the Macros button (or press Alt+F8) to open the Macros dialog. In the dialog, change the "Macros in" dropdown to the Normal.dotm template. Do the names of all your macros appear in the list?

If they do, click on one of them and click the Edit button. That will open the macro editor in a separate window, with the selected macro's code displayed on the right. It should look like this:

If that's what you see, then it's OK. You can check the other macros the same way. If at any point in this process you don't get the same results, then there's some problem with how you're recording or saving (or not saving) the macros.

If the macros are all there, then the problem is in the shortcuts. To check that, open the Options dialog and click the Customize item in the left panel, and then click the Keyboard shortcuts Customize button in the bottom right corner. In the Customize Keyboard dialog, make sure the "Save changes in" dropdown is set to Normal.dotm. Scroll the Categories list to the bottom and click on Macros. On the right side, all your macros should be listed. Click on each one in turn and look at the Current Keys box. Your choice of shortcut should appear there.

If there is something else, or nothing, in that box, click in the "Press new shortcut key" box and press the shortcut you want. Click the Assign button. Repeat for the other macros.

I should tell you that using macros to enter fixed text in documents is a WordPerfect habit, but Word works better when you use either AutoCorrect or AutoText.

A website shows multiple times in history of visted sites - Microsoft Community

renxc say:

A website shows multiple times in history of visted sites

Original title: http://googleads.g.doubleclick.net/page

 

Why do I get http://googleads.g.doubleclick.net/page multiple times in history of visited sites so I can't go back to previous website by selecting back button.  It does not show up in history.

robert aldwinckle on forums say:

I'm referring to the list of last sites visited that shows when you right click on the back button in the upper left corner of the screen. When the problem occurs that list will be populated with many googleads entries and when I try to go back nothing happens.


If you mean that there are so many that you can't see the page you want to go back to, use the History, View By Order Visisted Today  (Ctrl-Shift-H)  instead.  

Otherwise, you should be able to keep picking the bottom entry and thus "go back" that many and then eventually find something which is not part of the apparent loop you discovered.

BTW the View, Go to...   (Alt-V,G)  menu item shows essentially the same presentation as the right-click Back button list but IMO using it makes it more apparent when there is actual movement back when you need to go back more than what is initially showing.
