jay freedman say:

To throw a little technical light on what may seem an arbitrary refusal:

The internal structure of Word documents is entirely unlike a plain text (*.txt) document. What you see in the Word editor does not start at the beginning of the disk-based file and does not necessarily end at the end of the file. There are extensive components at the beginning of each document file that describe how to interpret the rest of the contents.

The older *.doc files, because they're largely a proprietary binary format, are especially complex. The "file header" at the beginning consists of tables of pointers to structures later in the file. If you tried to jam two or more of those together, with the second document's file header somewhere in the middle of the file, Word would just go belly-up and refuse to load it.

The newer *.docx format is based on a specialized version of XML, with multiple XML files and other things held in subfolders, and the whole thing compressed into a zip file. (You can actually rename the file extension from .docx to .zip and open the archive in Windows Explorer to see the contents.) Again, if you mashed two or more document files together, the zip format wouldn't be usable and the document wouldn't open in Word or anywhere else.

The closest you'll get to the "chaining" of documents is to build a sort of container document in which you put INCLUDETEXT fields, one for each source document, with a section break between successive fields.

If your documents are so finicky that "even one micro-change in the format will render them unreadable", then Word is probably not the tool you should be using (see, for example, http://wordfaqs.ssbarnhill.com/TextReflow.htm). You may need a publishing program such as InDesign.
