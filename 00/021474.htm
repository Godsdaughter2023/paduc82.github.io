doug robbins - mvp office apps & services (word) say:

There are a number of issues with what you want to do.

  • For a start, the document would have to be saved as a Macro Enabled (*.docm) file and that is going to cause issues when the recipient opens the document as they may elect not to allow macros to run.
  • In addition, the scheme would have to rely upon the recipient using a version of Office that includes Outlook and also on the user having Outlook set as the default mail application.

Ignoring those potential obstacles, the thing to do would be to use a MacroButton field in the document that would run a macro containing the following code:

Dim oOutlookApp As Object 'Outlook.Application
Dim oItem As Object
Dim bStarted As Boolean
On Error Resume Next
Set oOutlookApp = GetObject(, "Outlook.Application")
If Err <> 0 Then
    Set oOutlookApp = CreateObject("Outlook.Application")
    bStarted = True
End If
Set oItem = oOutlookApp.CreateItem(0) 'olMailItem
With oItem
    .To = "email address"
    .Subject = "Subject Text"
    .BodyFormat = 2 'olFormatHTML
    .HTMLBody = "Email Message Text" 'put your message here
    .Send
End With
Set oItem = Nothing
Set oOutlookApp = Nothing
'  Close Outlook if it was started by this macro.
If bStarted Then
    oOutlookApp.Quit
End If

Why everyone continues to suggest creating a PDF in order to resolve the issue doesn't make much sense - You've already stated that you've done that :-) Apparently nobody seems to understand that the problem is occurring in PDFs as well as hard copy... and if the lines are generated in the PDF, they will show up regardless of which program the PDF is printed from.

Most printers [PDF is a print process] 'see' the cell walls & interpret the use of no color as white (i.e., apply no ink here). That often results in the faint hairline issue you're experiencing. The solution is to make the cells blend together so the printer sees them as one large area of a specific color.

The issue is rather common, though, & there are at least 2 ways to resolve it.

  1. Select the left column, then use the Merge button on the Table Layout tab of the Ribbon (IMO, this is the preferable solution)
  2. Select the left column, then apply cell Borders which are the same color as the Fill.

The simplest explanation, though it would be a very long shot in Office 2016, would be that you are opening in Draft view, with Courier selected as the draft font. Since it requires contortions to even switch to Draft view in Word 2016, much less open documents in this view, and since use of a draft font is disabled by default (File | Options | Advanced: Show document content), this is very unlikely.

A Fonts folder disaster would presumably wipe out all fonts, not just all but Courier.

What printer is selected as the Windows default/active printer in Word? Perhaps it is one that can print only Courier?

Beyond that I'm pretty much out of ideas. Have you restarted Word? Restarted Windows? If you've done both, then try repairing/reinstalling Word.
