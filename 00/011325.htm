jlh.2012 say: I know this is an old thread, but just had the same problem and took 4 hours to fix. Thought I'd share what worked for me in hopes it helps someone else. Am running Excel 2010 32-bit on Win 7 Ultimate 64-bit.

1) Launch Excel and disable the Solver add-in (File|Options|Add-Ins click Go, deselect "Solver Add-In", click OK).
2) Quit Excel.
3) Navigate to: C:\Program Files (x86)\Microsoft Office\Office14\Library\SOLVER. There will be 2 files: Solver.xlam and solver32.dll.
4) Create a new sub-folder: OLD in the folder above. Copy solver.xlam and solver32.dll to folder OLD.
5) Navigate to: C:\Windows\SysWOW64. Find the 2 files solver.xlam and solver32.dll in SysWOW64.
6) COPY (do NOT move) these 2 files into C:\Program Files (x86)\Microsoft Office\Office14\Library\SOLVER.
7) Launch Excel and re-enable Solver add-in.
8) If this works you can delete the directory OLD.

In my case the problem was a third party add-in, which had replaced solver with a different (older??) version. The steps above just restore the newer version.

I agree with Scott and Marshall.  It also depends on where the data is coming from.  Your application should be split into front-end and back-end.  The front-end should be on your own PC.  The back-end should be either on your own PC, or on a server to which you're connected with a fast LAN connection.  A wireless, VPN or WAN connection will slow things down a LOT and may cause corruption too.

scottgem say:

Amen,

What do you mean by front end vs back end?

EVERY Access app needs to be split between a front end and a back end. The back end only has the tables. The front end has everything else with LINKS to the tables.

A front end can have multiple back ends. Back ends can be Access native tables or a SQL Server database or Sharepoint lists, or a combination of data sources.

Front ends should be stored locally on each users's PC and linked to the back end(s) on shared resources.

reneyu_msft say:

Hi Jon,

Since we are not able to use URL tokens in SharePoint Online, as Sky said, we cannot insert the logo with a relative address for the site template or SharePoint Online site collection.

However, instead of try "inserting" the logo into the template, we can upload the picture to the front-end Web server, to have the logo be as one of the default logos, to make adding relative address doable.

Please note that, since we are try to directly add contents into the SharePoint Online default location which would be shared by all the site collections, we should do this at our own risk.

1. Using IE, go to site collection and access any one of the Libraries.

2. On the LIBRARY ribbon, Open with Explorer.

3. At the Exporer windows, press Alt+↑ to go up to the parent directory.

4. Find the "images" folder and upload, for example, our logo.gif into the folder.

5. Go back to the site collection > Site settings > under Look and Feel, access Title, description, and logo.

6. At Logo and Description, enter the relative address as: /_layouts/images/logo.gif

Then, we can proceed to save the site as template and create a new site collection.

Rene
