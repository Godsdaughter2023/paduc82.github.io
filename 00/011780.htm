v_gku say:

Think I've managed to sort this one out:

The values for metered or non-metered are stored in the registry keys (to be noted the solution was tested on W10 x64 Ent edition).

Logical explanation would be that when the drivers for the WWAN cards are installed, either when W10 is setup or afterwards (depends on what you chose), the drivers are installed under trustedinstall account, and that account remains the owner for the registry values and will block any changes to those keys regardless if you are the local admin or user accounts.

In order to make changes to those registry keys you should take a backup of your registry, change the owner of HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\NetworkList\DefaultMediaCost key to your account and set 3G and 4G values to the whished output (1 for non-metered, 2 for metered). Once done, reboot the machine and you will obtain the desired effect.

I started working on this after changing from the toggle button the WWAN connection to metered - off but Outlook 2013 when starting still saw the connection as metered.

In theory it's a registry key permission issue, if you would change the owner either to System or Users to allow changes to those registry values, when toggling the button in the GUI you would also see the change in the registry for the values between 1 and 2, but I do not know how that GUI management was designed by MS and I did not take the time to test it further, although MS should release probably a fix for it if it's a design issue.

Let me know if it works, although it did the trick in my case, the short steps would be:

1. take a backup of your registry;


2. change the owner to administrator/ users or any other account you are working with for HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\NetworkList\DefaultMediaCost

3. set the desired values for 3G and 4G keys to 1 for non-metered or 2 for metered

4. restart the machine.

jim gordon mvp say:
Another way...

1. Open a new, blank Word document
2. Save the document as a template (.dotx) somewhere handy, like the desktop
3. From the Format Menu choose Style
4. In the Style dialog click Organizer
5. In Organizer, click the AutoText tab
6. Select individual AutoText items and then click the Copy arrow to copy them to the template you saved
7. Click the Close button when done copying
8. Save the template again (File > Save not File > Save As)

Now you have a document that has just the AutoText entries you want to move. Open the document on another computer and you can use the Organizer to copy them to that computer's Normal document.

rich michaels - mvp (greatcirclelearning.com) say:

Yes you can.

For AutoText copy your Normal.dotm template from the source computer and put it into the same place on your other computer ... replacing the one that is there.  The path is ~/Library/Application Support/Microsoft/Office/User Templates/Normal.dotm

Now I am using the extension of "dotm" because that is what is on my system that has both Word 2008 and 2011. If your Normal template has an extension of "dot".  There maybe a difference on your system because 2011 allows macros and 2008 doesn't and the "m" on the dot extension means, with macros. Regardless, replacing the Normal template will carry over all AutoText from the source system.

Now you might not want to replace the Normal template because it will also change Paragraph style settings.

To do this selectively, copy the source Normal template to the other machine and give it a different name and then open it as a file, meaning, first open Word and then select the renamed template to open. With this opened as a document, go to the Organizer function. There you can move the individual AutoText entries over to your other computer's Normal template ... selectively.

As for AutoCorrect entries ...

On your source machine, go to Applications/Microsoft Office 2008/Office/Default ACL (Your Language) ... and copy it. Then go to the new machine in the same location and rename the existing ACL (Auto Correct Library). After renaming the existing, paste the library that you have on your clipboard.

Start Word, and you should be ready to rock.

Hope this helps!

suzanne s. barnhill say:

I inserted the page numbers this way:  Clicked on the footer area, opened up the footer, insert page number, start from, entered the number 1, OK, closed it out.

This suggests that you have only formatted the page number but not actually inserted the PAGE field. When you click on Page Number in the Header & Footer group of the contextual Header & Footer Tools | Design tab, you get a menu that includes Top of Page, Bottom of Page, Page Margins, Current Position, and then, below a separator, Format Page Numbers and Remove Page Numbers. The Page Number Format dialog is where you set numbering to restart from 1 (and choose i, ii, iii or 1, 2, 3), but that just determines what number will appear on the page if you display a page number. To actually make that page number display, you have to insert a PAGE field in one of the following ways:

  • Place the insertion point in the footer where you want the page number and press Alt+Shift+P. This is much the easiest way.
  • With the insertion point in the footer where you want the page number, select Current Position from the menu and select Plain Number (or any of the fancy ones, for that matter).
  • Choose Bottom of Page and select one of the offerings from the gallery.

A caveat about the last option: this inserts a footer building block that will be overwritten by any other footer you insert. The only way to combine a page number with a fancy footer is to insert the footer first, then place the insertion point where you want the page number and use Alt+Shift+P or Current Position.

Note that if you already have page numbers appearing in Section 1, you should be able to make them appear in Section 2 (but with the different format and restarting at 1) just by linking the footer in Section 2 to the footer in Section 1 (in Section 2, click the Link to Previous button). You can restart numbering or change number format without unlinking headers or footer; you need to unlink them only if you want to change the header/footer or omit it altogether.
