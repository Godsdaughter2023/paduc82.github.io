doing an if then statement on 2 different worksheets - Microsoft Community

denisecarter1 say:

doing an if then statement on 2 different worksheets

I'm trying to learn excel and am stumped right now.  I have 2 worksheets.  One is named activity the other is follow up.  there are company names on both spread sheets that will match.  the follow up spread sheet has 2 columns that I want to bring over to the activity spreadsheet .  So basically, If the name on spreadsheet Activity matched the name on spread sheet follow up, move column G and H to column I and J    Is this possible? 
hansv mvp say:

Let's say the company names are in column A on both sheets, starting in row 2.

In I2 on the Activity sheet, enter the following formula:

=IFERROR(VLOOKUP(A2,'Follow Up'!$A$2:$H$1000,7,FALSE),"")

And in J2:

=IFERROR(VLOOKUP(A2,'Follow Up'!$A$2:$H$1000,8,FALSE),"")

Adjust the range A$2:$H$1000 if you have more than 1000 rows with names on the Follow Up sheet (it doesn't matter if you have fewer rows), then fill or copy down from I2:J2.

jp ronse say:

Hi,

To use build-in functions you have to use worksheetfunction.function statement.

One way:

Function MaxHeader(HeadRow As Range, LookRow As Range) As Variant
   With WorksheetFunction
      MaxHeader = .Index(HeadRow, 0, .Match(.Max(LookRow), LookRow, 0))
   End With
End Function

Note the "." before each called function. It means that it is referring to the with statement.

You can of course precede each function with worksheetfunction, e.g. worksheefunction.index, worksheefunction.match, etc but the use of with is preferrable. It is faster because the application is called once and it is less typing.

A second way is to use to use the evaluate method. With this method you have to write the expression as you would have written it in a cell.

Function MaxHeader_Bis(HeadRow As Range, LookRow As Range) As Variant
    MaxHeader_Bis = ActiveSheet.Evaluate("Index(" & HeadRow.Address & ", 0, Match(Max(" & LookRow.Address & ")," & LookRow.Address & ", 0))")
End Function

The formula to evaluate is:

"Index(" & HeadRow.Address & ", 0, Match(Max(" & LookRow.Address & ")," & LookRow.Address & ", 0))

It like almost as you have written it in a cell, note the evaluate method requires a string.

To parse fi the HeadRow into this method you have to use the address of the range.

Let's say, you would put in a cell "Index(C1:L1 ....." The range arguments in your UDF refers to objects, the evaluate method requires an address, therefore range.address.

HTH, good luck.

psychomania---01 say: That is incorrect.

When a new firmware update is released nokia sends the generic unbranded firmware to the network, the network then has to add their branding and approve it.

Once this has been done the network then sends the branded firmware back to nokia for distribution on the NSU and at care points.

Delays are caused by the network when they don't approve the new firmware.

Vodafone and O2 are notorious for not approving new firmware.

petrib say: In reality, the network operator commissions Nokia to do specific changes/customization/branding to the firmware (and Nokia can further subcontract the work to another company, too).

Then when it is done, the network operator gets the modified firmware and they check whether they approve it or not.

If they approve it, then it is released.

And a network operator can skip entire versions of firmware (and decide they do not want to go through the effort of paying for the work and related testing, etc.).

They might even just order one version of customized firmware and decide to stick to that for the lifetime of the device in their portfolio (no updates at all, ever).
