greginmich say:

We have lots of similar reports, and while I certainly can’t offer an informed diagnosis, I can tell you that you’re wasting your time scanning with the MSRT. The MSRT is a relic scanner that can only detect a small subset of prevalent threats. So if you run your manual (on-demand) scans with this tool, you’re guaranteed to miss a lot of things that a Full Scan with MSE or Windows Defender Antivirus would catch. When the MSRT makes a large number of false detections like this, it's most likely caused by a bug in the program, but it's always possible that undetected malware could be involved. So it’s also important to scan with some third-party malware-removal tools. These tools routinely include signatures that are missing in Microsoft’s “comprehensive” signature set – which actually never comes close to covering the entire spectrum of online threats. So in a case like this its especially important to scan with Windows Defender Offline, and these malware-removal apps:

Kaspersky Virus Removal Tool:

scott mcdaniel say:

First - don't add Calculated Columns directly to the table. Do this in a query instead:

SELECT Field1, Field2, Field1 - Field2 AS MyCalcuatedField FROM SomeTable

Why are you using a Pivot table for this? I would think this would be a straight query, which could be used to create a Report.

ken sheridan say: You'll need a query along the following lines:

SELECT Vendor,
YEAR(OrderDate) AS OrderYear,
MONTH(OrderDate) AS OrderMonth,
SUM(IIF(ActualMileStone <= PlannedMilestone,1,0))/COUNT(*)*100
AS OnTimePercentage,
SUM(IIF(ActualMileStone > PlannedMilestone,1,0))/COUNT(*)*100
AS LatePercentage
FROM Orders
GROUP BY Vendor, YEAR(OrderDate), MONTH(OrderDate);

The on-time and late orders are counted by summing the return value of an expression which returns 1 or 0 in each case.  The resulting value is then divided by the total number of rows returned and multiplied by 100 to return the percentage of each per vendor per month per year.

rich michaels - mvp (greatcirclelearning.com) say:

Hi Kathrin,

Have you contacted Nuance the makers of Dragon to find out what they know about the crashing problem with Office 2016?

Regarding the "auto save" you mention. There really isn't an AutoSave function in Word, at least there isn't one operating like I believe you might expect it to be. The Word feature you are referring to is really AutoRecovery and it only works if something internal to Word crashes the program.

I don't know for certain but I believe Dragon is a separate application that passes digitized data to Word, probably by some global template add-in interface. However, Word does not treat template add-ins the same as it would if it were a component add-in. Component add-ins (a.k.a. Com Add-ins) are considered part of the host application and global template add-ins are external resource interfaces. An external resource might force Word to quit when something goes wrong but in my experience that doesn't necessarily trigger the auto recovery function.
