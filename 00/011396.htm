ken sheridan say:
I need the order to be exactly as I have entered them into the database

The fact that you can see no discernible pattern to the ordering is because there isn't one.  A table in a relational database represents a set.  By definition a set has no intrinsic order.  A query's results can be ordered, but reports do not respect the sort order of the underlying query.  A report must be sorted by means of its internal sorting and grouping mechanism.

If you want to sort by the order entered into the table the only really reliable way is to include a DateTimeStamp column in the relevant table, giving it a DefaultValue property of Now() so that the date/time when the row is inserted is automatically entered.  Strictly speaking it's the date/time when you navigate to the empty new row in a form or datasheet, but that's not pertinent to the issue here.

Using an autonumber column for the sort order of the report might at first sight seem a simple solution, but an autonumber guarantees one thing and one thing only, which is that the values are distinct.  You cannot confidently assume they will be sequential.

scottgem say:

First, you have to understand that reports have their OWN order, They do not, necessarily use the same order as the underlying tables or queries. So you have to use the Grouping and Sorting function in the report to manage the order.

If you have an Autonumber ID on your ingredients table and this number is the sequence you want to sort by, then, by all means, add it has a hidden control and sort by it. However, if you ever have to change the ingredients list, this will no longer work. So, I would suggest that you use a sequential number per recipe ID (see my blog on Sequential Numbering). You can set the increment to 5 instead of 1 to give you room to insert ingredients in order. Or you can manually adjust the sequential numbering if you change the the list. Then sort by that value.
