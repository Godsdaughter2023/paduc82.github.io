wunders say: This is expected when you chain NAT (Network Address Translation) Routers.  The subnets of both routers are probably 192.168.x.x which are normally non-routeable subnets -- and they are two distinct subnets.  You really want to have all the devices on the same subnet rather than creating two disparate subnets that can't communicate with each other.  To accomplish this, you should have an "access point" in place of the second "router" device (on the far end of the powerline adapter).  You can sometimes make a wireless router operate in "access point" mode by going into its configuration, disabling DHCP, and connecting your powerline adapter to a LAN port of the router instead of the WAN port (leaving the WAN port unconnected).  Try Googling "use wireless router as access point" on internet and you will get lots of instructions.

I fixed the problem.

I was in Hotmail.com's settings (for the fourth time), and opened the setting, "Rules for sorting messages." Hi

 

There are misconceptions about how ReadyBoost works to speed up the system. There are a lot of articles that say ReadyBoost equals more RAM. This is not exactly accurate.

 

What ReadyBoost actually does is supplement the Caching provided by the Hard Drive.  When you enable ReadyBoost it keeps a copy of the disk cache and monitors hard disk operations. It will go into action when reading and delivering files from its copy of the cache will boost performance. (This is why you do not see the extra memory in Task Manager)

 

There is an excellent article, written by Greg Shultz on TechRepublic about ReadyBoost. It includes step by step instructions about how to use the Performance Monitor to monitor how ReadyBoost is working. Here is a link to the article.

 

Keep tabs on ReadyBoost with Windows 7's Performance Monitor:

http://www.techrepublic.com/blog/window-on-windows/keep-tabs-on-readyboost-with-windows-7s-performance-monitor/2257?tag=content;siu-container
