First order of business would be to clarify the terminology to ensure there's a clear understanding on everyone's part about what exactly you are referring to in the various scenarios

#1 - There is only one Outlook '2013 desktop client whether it's referred to as part of Office 365 or Office 2013

#2 - Outlook.com and Outlook 2013 (desktop) client are two entirely separate things. The first is a web based email account which can be configured for use in Outlook '2013 (or any other email client)

#3 - There is no mandatory requirement to use an Outlook.com account with Outlook '2013

#4 - Always a good idea to include an explicit reference to the "instructions" you followed - no one to know which instructions you are referring to and whether or not you are referencing things related to the Outlook.com web interface or the Outlook '2013 desktop client

#5 - There is no question that both Outlook.com AND Outlook '2013 support contacts but how the two interact depends on how the Outlook.com was configured (i.e. EAS - Exchange ActiveSync, IMAP, POP3 etc)

#6 - There is a separate forum specific to Outlook.com for questions related to how to use the account from the web interface. For questions specifically related to contacts - the link to that forum is

A background article regarding the topic

Understanding Outlook.com Contact Groups/Distribution Lists
http://www.contactgenie.info/understanding-outlook-com-contact-groupsdistribution-lists

#8 - You cannot "import" contacts from an external data file in a an EAS based contact folder but you can move/copy contact data from an existing folder to an EAS based contact folder. Options are described at the bottom of the following article

How to Import Outlook.com contact data using Outlook '2013
http://www.contactgenie.info/how-to-import-outlook-com-contact-data-using-outlook-2013

#9 - You cannot move/copy email items directly into an EAS Outlook.com <Inbox>. You would need to configure the same Outlook.com account in O'2013 as an IMAP account and move/copy the email items into the IMAP <Inbox> (it's the same <Inbox> - just accessed using a different protocol.
