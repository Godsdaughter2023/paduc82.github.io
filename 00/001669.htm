Bug/Solution for Bibliography problems "Invalid source - Microsoft Community

luc_ae say:

Bug/Solution for Bibliography problems "Invalid source specified" and "There are no sources in the current document"

Hi

The Bibliography tools don't work anymore if the File Type "Web Pages" is blocked or will be opened in Protected View. The Manage Sources Dialog shows the two Errors "Invalid source specified" and "There are no sources in the current document" in the preview.

To reproduce the error/bug go to the Trust Center (Word Options -> Trust Center) and then select the entry "File Block Settings" on the left. Set the checkmark for "Open" for "Web Pages".

@ readers with the mentioned problem:

- Does it work in a new document? If so, some of your sources could be invalid (did you use < or > in a field?).

- Check if your Sources.xml is corrupted (eg. rename it to Sources.xml.old).

- Try different Styles (APA, MLA, Turbian).

- Go to Options -> Trust Center -> File Block Settings, Unckeck "Open" on the line "Web Pages".

Regards Luke

alistairlangford-wilson say:

Here's a little more info, discovered by trial and error.


If the Advanced option ("Insert long printouts on multiple pages" ) is unchecked, and you insert a document printout that has 5 or more pages, you will see a dialog giving you the option of inserting it as multiple pages or a single page. This second dialog pops up after you have selected the file that is to be inserted as a printout.


But if it is shorter than 5 pages, it will always be inserted as a single page, regardless of how your preferences are set.


This seems silly. The notion that 5+ pages constitutes a "long" document is purely arbitrary and completely indefensible (as is any other number apart from 5). A better way to do it would be:


* Remove the logic that looks at whether a document has 5+ pages and makes decisions based on that. That code can be thrown away -- one less thing to test, maintain and worry about in future.

* Change the Advanced preference so it just sets a default for all multi-page documents: "For multi-page documents, insert each page as a separate OneNote page"

* Have a simple checkbox in the Insert > File Printout dialog to override the default preference (it is checked or unchecked depending on the Advance preference, but can be changed on-the-fly by the user. if changed, it is not sticky: it reverts to the Preference each time).


It suits me to insert some documents as multiple pages. That way, I can easily rearrange the page order in OneNote which, for certain documents, is something I find useful. But for other documents, I don't need to rearrange the page order. Therefore, I like to make the decision on-the-fly, but I don't want to be stomped on by some program manager's funky notion that there should be a universal, unalterable override for all "short" documents, and that the definition of a "short" document is, and always will be, 5 pages.

amit tandon say:

You can do this using vba. Enter the following code in "ThisWorkbook" module - all cells in column A which are blank will be filled with 1 on opening the workbook:

Private Sub Workbook_Open()

Application.ScreenUpdating = False
Application.Calculation = xlCalculationManual

Sheets("Sheet1").Activate
Dim rng As Range, cell As Range

'Set the range
'Set rng = Range("A1:A100")
Set rng = Range("A:A")

For Each cell In rng
    If IsEmpty(cell) Then cell.Value = 1
Next

Application.ScreenUpdating = True
Application.Calculation = xlCalculationAutomatic

End Sub
