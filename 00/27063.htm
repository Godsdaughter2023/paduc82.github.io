scottgem say:

No it doesn't make sense. Your database is designed properly (at least what you've shown us). Your tblAgency has Agency info and your tblContact has Contact info. Assuming a contact can only be associated with one Agency, then using AgencyID as a foreign key in tblContact is the way to go. However, if a contact can be associated with multiple agencies you will need a third table. Let us know ans we can go into detail.

If you want to associate unassigned contacts from frmAgency, you CAN do that, but it will take some doing. What I would do is add an UNBOUND combobox to frmAgency. I would then set the RowSource of that combo as follows:

SELECT ContactID, ContactFullName

FROM tblContact

WHERE AgencyID Is Null

ORDER BY ContactLName, ContactFName;

In the After Update Event of that combo I would add code like:

Dim strSQL

strSQL = "UPDATE tblContact SET AgencyID = " & Me.AgencyID WHERE ContactID = " & Me.ContactID & ":"

CurrentDB.Execute strSQL, dbFailOnError

This code will set the AgencyID for the selected contact.

If you want to display the primary contact's info on the frmAgency, use a subform, linked on AgencyID and filter the Recordsource so it only shows primary contacts. Note you can't use the SAME control for this. The combobox needs to be filtered for unassigned contacts if you want to use it for assigning a contact to an agency. jay freedman say:
There isn't any command to jump to the endnotes, not even one that doesn't yet have an assigned shortcut (you can assign a shortcut to almost any command in Word).

There is a shortcut (Shift+F5) to go back to the most recent point of editing (not where you were just viewing, but actually inserting or changing text). Successive presses of Shift+F5 take you to the second and third most recent changes, and then back to the first one. You might be able to take advantage of this, if you can train your fingers: When you're about to make a jump from an endnote back to text, first insert a space and backspace to delete it, creating an edit point there. Do the same in the text. Then Shift+F5 will take you back and forth.

A better behaved solution might be accomplished with a macro, although I'd have to see what tools VBA provides for finding specific endnotes.
