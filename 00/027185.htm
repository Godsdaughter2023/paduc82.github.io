ken sheridan say: You need to concatenate a date literal, delimited with the # symbol, into the string expression. Date literals must be either in US short date format or an otherwise internationally unambiguous format.  The ISO standard of YYYY-MM-DD is a good choice:

"[HistoryDate] = #" & Format(Nz(Screen.ActiveControl,0),"yyyy-mm-dd") & "#"

If the control is Null the expression will return #1899-12-30#, which is day-zero in Access's implementation of the date/time data type.

ken sheridan say: I believe the US short date format is actually mm/dd/yyyy, not mm-dd-yyyy.  The ISO standard is YYYY-MM-DD.  The standard uses case to differentiate months and minutes, whereas Access does so with 'm' and 'n', hence the expression of the standard in upper case.  In Access it's not case sensitive, so you can use lower case.

You haven't said whether this is a web database or not, but if not, I'd suggest using code in the control's AfterUpdate event procedure:

    Const MESSAGETEXT = "No matching record"
    Dim ctrl As Control
    
    Set ctrl = Me.ActiveControl
    
    If Not IsNull(ctrl) Then
        With Me.RecordsetClone
            .FindFirst "HistoryDate = #" & Format(ctrl,"yyyy-mm-dd") & "#"
            If Not .NoMatch Then
                ' go to record by synchronizing bookmarks
                Me.Bookmark = .Bookmark
            Else
                MsgBox MESSAGETEXT, vbInformation, "Warning"
            End If
        End With
    End If

Ctrl + G to open the Immediate window

Enter the following in the Immediate window and press Enter.

Application.EnableEvents = False

Save the workbook

Test the Insert

Delete any worksheets not in use. (Right click the worksheet tab name and Delete)

Select the first worksheet

Are you being misled by the term "supported lifetime" [as in Win10 will continue to provide updates for the supported lifetime]?  "supported lifetime" is  not a properly defined term but, for all practicable purposes, it means for as long as you can get hold of device drivers that work. 

Device manufacturers will eventually not bother to continue modifying drivers for older devices so every device may eventually reach a stage where available device drivers are not compatible with the latest updates to Win10. 

When this happens, some users might figure out workarounds.  In the absence of any workarounds, users might well be faced with restoring their systems to the last Win10 update that their devices were compatible with and accepting the limitations of use that that entails [which could well include no longer connecting to the internet as the older drivers could be exploited by hackers].
