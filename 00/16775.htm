greg maxey say:

Dax,

 

1.  Insert the content control in a new paragraph.

2.  Apply Heading 1 to the paragraph.

3.  Type the FAQ title in the content control (e.g, How to use Content Controls for FAQ Listings)

4.  Click Developer>Controls>Properties and in the Content Control "Tag" property field enter the date of for the FAQ entry.  E.g., the FAQ created today would have 1/08/2012 for its tag.

 

Create the TOC.  The TOC will list the FAQ Titles as the appear in the document.

 

You could trigger the userform display from a ribbon control, the Quick Access Toolbar, or a macro button above or below the TOC.

 

So the TOC might look like this (less the tags):

All About Apples  (tagged 1/1/2012)

How to build a better mouse trap (tagged 6/1/2012)

What to do when pigs fly (tagged 1/1/2013)

 

When the userform displays the titles are displayed like this:

What to do when pigs fly

How to build ...

All About apples

 

When the user selects a title in the userform the title is selected in the document.

 

greg maxey say:

Jay,

 

I'm not saying this is best practice, but I like tinkering with content controls.  Here is a perhaps quirky idea ;-)

 

Put the FAQ headings in a Content Control formatte with Heading 1

Use the date the FAQ is added as the CC "Tag" property.

 

Build the TOC using Heading 1

 

Add a userform with a single listbox to the document.  List each FAQ (reverse date added order) in the listbox.  When the user clicks a listed FAQ, it takes them to the FAQ heading range.  This code might be a little rough, because I don't remember if I can skip the array and sort a colleciton or not:

Option Explicit

Private Sub UserForm_Initialize()
Dim arrCCs() As String
Dim oCol As New Collection
Dim i As Long
  Dim oCC As ContentControl
  ReDim arrCCs(ActiveDocument.ContentControls.Count)
  For i = 0 To ActiveDocument.ContentControls.Count - 1
   arrCCs(i) = ActiveDocument.ContentControls(i + 1).Tag
  Next i
  WordBasic.SortArray arrCCs, 0
  For i = 0 To UBound(arrCCs)
    If arrCCs(i) <> "" Then
      oCol.Add ActiveDocument.SelectContentControlsByTag(arrCCs(i)).Item(1)
    End If
  Next i
  Me.ListBox1.ColumnCount = 2
  For i = 1 To oCol.Count
    With Me.ListBox1
      .AddItem
      .List(i - 1, 0) = oCol(i).Range.Text
      .List(i - 1, 1) = oCol(i).Tag
    End With
  Next i
End Sub

Private Sub ListBox1_Click()
ActiveDocument.SelectContentControlsByTag(Me.ListBox1.Column(1)).Item(1).Range.Select

Unload Me
End Sub

If you have users who still have Access 2003 or earlier, stay with the Access 2000 format (.mdb).

You should also stay with this format if your database has user-level security.

On the other hand, if all users have Access 2010 or later, it is probably a good idea to convert both the backend and the frontend to the new .accdb format. It supports new features that aren't available in the older format, such as table-level triggers (data macros), and new types of fields such as Attachment fields, Calculated fields and Rich Text fields.
