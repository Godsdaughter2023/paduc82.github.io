Windows could not start the Background Intelligent Transfer service on the local computer. For more information, review the system event log. If this is a non-Microsoft service, contact the service vendor and refer to service-specific error code -2147467243. (0x80080005)
is error message is similar to the "Error 5: Access is Denied" error message. To resolve this problem, follow the steps described in the "Error 5: Access is Denied" section.

This error indicates that the access permissions for BITS have been modified. To repair these access permissions, use one of the following methods, depending on the operating system.

Windows XP

  1. Click Start, click Run, type cmd, and then clickOK.
  2. Type the following command, and then press ENTER:
    sc sdset bits D:(A;;CCLCSWRPWPDTLOCRRC;;;SY)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BA)(A;;CCLCSWLOCRRC;;;AU)(A;;CCLCSWRPWPDTLOCRRC;;;PU)
  3. Click Start, click Run, type services.msc, and then clickOK.
  4. Double-click Background Intelligent Transfer Service.
  5. In the General tab, click Start.

stephaniehaymore say:

Well, I figured out what my problem was. Thought I would post here in case it helps anyone else.

I had recently changed my password because my account had been hacked. I had also set up two-step authentification in order to make any changes to my account in the future. Apparently this process applies to everywhere you sign in to your account so any apps that can't confirm the two step process password will time out.

I read about it here:

App Passwords & Two-Step Verification

www.windows.microsoft.com/en-us/windows/app-passwords-two-step-verification

I had to make separate passwords for each of my email account that I had set up two-step verification for, using the password generator on my account settings. Voila, I can check my email again!

Hope this helps someone.
