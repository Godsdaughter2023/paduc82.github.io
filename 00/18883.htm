On the tab noted as Summary, right-click and select View Code. Paste this into the pane titled something like Book1 - Sheet1 (Code):

Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Target, Range("A:A")) Is Nothing And Application.ScreenUpdating Then
        Application.ScreenUpdating = False
        For Each cl In Sheet2.Range("A22:A190")
            cl.EntireRow.Hidden = Not cl.Value
        Next cl
        For Each cl In Sheet3.Range("A22:A190")

        Next cl
        Application.ScreenUpdating = True
    End If

There's not much point hiding/showing rows in a worksheet that you are not even viewing.  I would use the worksheet's Activate event to do the needful.

In the code module of the quote form and the order form worksheets, put the code below.  It assumes that the True/False values in column A are booleans (as Excel and VBA understand) and not the literal string "True" and "False"

Option Explicit

Private Sub Worksheet_Activate()
    Dim cl As Range
    Application.ScreenUpdating = False
    For Each cl In Range("A22:A190")
        cl.EntireRow.Hidden = Not cl.Value
        Next cl

henknz say:

Double sided printing using gutter does not give expected result

.docx in Mac Word 2011.  OSX 10.6.8, iMac 27"
Format -> Document -> Margins: top and bottom same, inside 2 cm, outside 1cm, gutter 1 cm, Mirror margins [x].   On tab --> Layout: Different First Page [x].  In document the page numbers are set on the outer edge of each page.  When printing to HP Color Laserjet 2605dtn: Layout: Pages per sheet 1, Two-sided Long-Edge binding selected, the document comes out with the gutter on the wrong side of each page and the page number on the inner edge of each page. 
In fact, in Layout View the first page has the gutter on the right and it is already clear that it cannot work.
Why is this happening? 
 
john mcghie say:
Henk:

You haven't said what your "Section starts with" property is set to, in your Format>Document>Layout properties.

It needs to be "Right Page".

The setting of the page NUMBER has no effect on the "sides": you can force even numbers to the right-hand side if you really want to.

It's the setting of "Section Starts With" that controls where the first impression of each section lands: either on a right page, or on a left page.  The Print subsystem will then continue this through to the end of the section.

So following sections should be set to "Next Page" or "Right Page" to continue getting the "sides" right.

Layout View does not show you the correct "sides": that's a weakness of Layout View.  You need to use Print>Preview to see the correct sides, with the added blank pages.  In Word 2011, the Preview command is on the Print dialog.

Hope this helps
