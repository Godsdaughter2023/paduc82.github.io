With the trends towards more targeted systems and apps, the interest in general web pages is declining, so this makes any additional effort to protect and maintain this avenue of less concern at all levels as well.  I think we're simply seeing a symptom of this decline as the more knowledgeable user moves away from the browser to devices with apps, which by their nature are less susceptible to such simplistic forms of attack.

neha singh msft say:

Hi Dvis182,

Unfortunately, we can’t restrict internal users from receiving emails containing a Microsoft Form link. If you restrict external sharing or block certain domain from SharePoint admin center, these restrictions will be applied to internal users, external user can still send emails with Microsoft Forms link.  

You can, however, block email messages that came from a specific mail server (specific IP) or based on the classification of Geographical location or Language.  For more information, see Spam email and Office 365 environment - connection and content filtering in EOP

Safe mode disables all fonts except for a system fonts and Office installed fonts. The behavior you describe points pretty much at there being a corrupt or duplicate font on your system.

Go back to Safe Mode and run Font Book or other font utility. Verify each and every font. Delete fonts that don't check out as perfect.
