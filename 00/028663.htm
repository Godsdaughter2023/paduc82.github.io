Cortana Speech Issues - PC Build 10122

In Build 10122, there is a bug that can cause issues when using Cortana’s speech capabilities. If you are affected by this bug, you may see the following behaviors:

  • The input box will not display the words that you speak
  • Some spoken commands will appear to be recognized  by Cortana, but may not behave properly (such as saying "Play music" will result in "Playing music" being displayed, but not music will start playing)
  • Other commands will cause an error to be displayed (for example, “can’t connect” or “something went wrong”)

If you are seeing these behaviors when using speech in Cortana, please try the following:

  1. Launch the task manager by hitting ctrl + shift + esc
  2. Click “More Details” at the bottom of the window
  3. Scroll down to Background Processes and find “Search” in the list.
  4. Right click Search and select End Process.
  5. After a few moments (the search process will take some time to restart), please try using Speech with Cortana again.

suzanne s. barnhill say:
Interestingly, this is true for Word 2007 and above but not for Word 2003 and earlier, which use a different Open dialog and can preview document contents.

jay freedman say:

None of the Office programs can preview their own documents in the Preview pane. This is described in the Support article https://support.microsoft.com/en-us/help/983097/this-file-can-t-be-previewed-error-in-word-excel-or-powerpoint. The suggested solution is to use Windows' File Explorer or a different Office program's Open dialog to see the preview.

The technical reason (poorly explained in the article) is that the Preview pane is created by opening an instance of the program that owns the selected document, using it to interpret and display the document file. However, a running copy of Word can't open another instance of Word, and similarly for Excel and PowerPoint.
