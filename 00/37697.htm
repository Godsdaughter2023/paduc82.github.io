alexander shakurin say:

I think I have finally managed to fix this issue. I checked many places and tried many approaches but finally I did the following:

1) Open regedit

2) Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Windows

3) Backup this registry hive

4) Check AppInit_DLLs key - if it's not empty then this may be a problem. Mine was "C:\\Windows\\system32\\nvinitx.dll,C:\\WINDOWS\\system32\\nvinitx.dll,C:\\PROGRA~1\\COMMON~1\\avest\\AVESTC~1\\AvSSPc.dll"

5)  One of dlls from this key is path to NVidia driver and second is 3rd party cryptography provider. The following steps both fixed my issue:

6.1) Remove redundant path from list

6.2) Move 3rd party crypt dll to another location (I just renamed original folder and renamed installation location of crypto provider so that it can't accessible).

I went with 6.1 and for now (1 day long) I have no more issue with Windows Firewall and RACM.

6.2 is quite dangerous - in my case I wouldn't be able to use software that is depend on this crypto provider. An option is to move everything back to original location when you need it.

I hope that this will help someone as I spent 4 days in frustration diving into this issue.

kavita m123 say:

Hi John,

From reading the entire thread, it sounds to me you have a bad NIC.  The 127.0.0.1 address is the test loop back address that exists on every computer.  

If you are having problems with your network, PING 127.0.0.1 is the first thing you do to prove the Network Card and the TCP/IP software is working correctly. You will get a response from 127.0.0.1 even of the computer is not connected to anything.  Since you are receiving a General failure from this test, I does look like you have a bad NIC.  If you’ve tried to re-install the drivers, tried pinging the loopback address (127.0.0.1) with no luck and re-installed NIC drivers and Winsock with no success, I would recommend getting a new NIC.  If you want to try a few more things, you can  try the following if you have not done so already

Click on Start button.
2. Type Cmd in the Start Search text box.
3. Press Ctrl-Shift-Enter keyboard shortcut to run Command Prompt as Administrator. Allow elevation request.
4. Type netsh int ip reset in the Command Prompt shell, and then press the Enter key.
5. Restart the computer.

You can also try to uninstall IPv6 Using the Network Connections Folder.  I personally don’t think there is a conflict between IPv4 and IPv6 but if you want to try this as well, you can.  Click Start.

1.     Either click Control Panel and then double-click Network Connections, or point to Settings, click Control Panel, and then double-click Network Connections.

2.     Right-click any local area connection, and then click Properties.

3.     Click Microsoft TCP/IP version 6 in the list of installed components, and then click Uninstall.

4.     Restart the computer.

Also, have you tried to reboot your computer in Safe Mode with networking and see if your NIC works in this mode? To do this:

1.

Remove all floppy disks, CDs, and DVDs from your computer, and then restart your computer.

Click the Start button , click the arrow next to the Lock button , and then click Restart.

2.

Do one of the following:

•

If your computer has a single operating system installed, press and hold the F8 key as your computer restarts. You need to press F8 before the Windows logo appears. If the Windows logo appears, you will need to try again by waiting until the Windows logon prompt appears, and then shutting down and restarting your computer.

•

If your computer has more than one operating system, use the arrow keys to highlight the operating system you want to start in safe mode, and then press F8.

3.

On the Advanced Boot Options screen, use the arrow keys to highlight the safe mode with networking option, and then press ENTER. For more information about options, see Advanced startup options (including safe mode).

4.

Log on to your computer with a user account that has administrator rights.

When your computer is in safe mode, you'll see the words Safe Mode in the corners of the display. To exit safe mode, restart your computer and let Windows start normally.  In safe mode, try seeing if your NIC is working.

If you’ve tried the above steps and still no luck, I would recommend you buy a plug-in NIC and see if this new NIC works on your pc.  If it does, then you know you have a bad NIC and not a configuration problem on your computer. 

Is your computer under warranty?  Perhaps the manufacturer of your laptop can run some extensive diagnostics that will check your NIC to see if it is a problem.  Some manufacturers can provide a boot up cd that can be run which  test your NIC and other components of your laptop to find the problem,

When you install a second operating system on your computer, Windows displays the Boot Manager screen on startup to allow you to boot from the second partition. We can configure Windows so that it skips the Boot Manager screen entirely and goes directly to your preferred operating system/drive.

To make your Windows start directly without prompting for Boot screen, follow the below steps:

Step 1:

  1. Click Start

  2. In the Search text box, type msconfig

  3. Launch msconfig by selecting it from the search results above the search text box.

  4. Go to the Boot tab

  6. Click the Delete button and then click OK.

  3. Click System and Security

  4. Click System

  5. Click Advanced System Settings (On left pane), then click on Advanced tab

  6. Under Startup and Recovery, click Settings

  7. Now, uncheck "Time to display list of Operating systems" then click on Apply and Ok.
