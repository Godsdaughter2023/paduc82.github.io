karl timmermans [outlook mvp 2012-2018] say:

Any advice would be appreciated.  Worst case scenario, can I reverse the order and have the "outlook' folder below the one I'm using?  

1) First to get something out of the way - uninstalling/reinstalling Outlook is a complete waste of time in this scenario. Outlook profiles "persist" any uninstall/reinstall.

2) Next, importing email data from an old computer is also unnecessary. All that was required would have been to copy the PST file from the old computer to your new computer and select that file when configuring a POP account as per

3) If you created an IMAP account in OL'2016 - it would have only created one data file (an .ost file) which would have been deleted if the IMAP account was deleted.

4) A PST file label "Outlook Data file" would have been created if an IMAP account was used on this computer using an earlier version of Outlook ('2010 or earlier) so there's something missing from the description in all of this. That would have been an empty data set since it would have been configured as the 'default data file" to contain the "default" contact/calendar folders.

5) So here's what I suspect has happened

- you created an IMAP account using a previous version of Outlook

- you installed OL'2016 and configured your POP account (deleting the IMAP account either before or after configuring the POP account)

6) And here's all that you need to do at the end of the day

a) in the NAV pane - show the full folder list via CTRL+6

b) ensure that all the folders are empty in the "Outlook" folder tree (if not and there's something you want, copy the data to the corresponding folders in the POP account folder tree, when done

c) go to File --> Account Settings --> Data file tab

d) be sure that the PST file for the POP account is set as the "default" for the profile

e) "Remove" the "Outlook" data file

Done - only one folder tree will now appear in the Nav Pane (assuming that you don't have other email accounts and/or haven't added other PST data files)
