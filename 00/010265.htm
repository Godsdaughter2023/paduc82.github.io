diane poremsky mvp (slipstick.com) say:
The easiest way would be a public folder, if your organization uses them.

If someone create a folder to share with the others, everyone could move mail into it. Instructions are here: https://www.slipstick.com/exchange/view-shared-...

Another way would be with a shared mailbox - the admin needs to create it and share it with the group members. You'd see the full mailbox in Outlook though - unless only the Inbox was shared with everyone.

jimmy t_626 say: On the target shared or user mailbox from where you're sharing data, right-click on the top level folder (usually has your name or email address), select properties, permission (tab), check the 'folder visible' box for the 'Default' user. If you don't allow this access from the top level folder, users wouldn't be able to browse into other subfolders even if they have permission. This is the secret ingredient to selectively share folders within a mailbox.

The folder you're sharing will need to be at the same level as other default folders (Inbox, Sent Items...). Adjust permission on the folder you're sharing as needed.

In an Exchange mailbox, each folder including the top level folder has its own permission set. Each time you have someone new on the team, you will need to re-visit permissions tab on each-and-every folders to make adjustments. (Unless you have a security-enabled DL group that can be used to grant permission.)

On each team member's mailbox, they will need to add the target mailbox by going into file, account settings, account settings (again), 'change' (in the email tab), more settings, advanced, and add the target mailbox there to list the target mailbox in Outlook's navigation pane. When users browse into the subfolders, they will only see subfolders where they are granted with permission.

Consult with your company's email admin if you have a large team, or if you need a more permanent solution.
