    DLookup fail in Access 365 - Microsoft Community

    peter_914_74 say:

    DLookup fail in Access 365

    I have a form in my production database with a Dlookup control.  As long as the calculation backing the control the Dlookup is based on less than 7 entries it produces a result no problem.  After there are 7 or more entries the Dlookup stops returning any results, no errors, nothing, just a blank control.

    Any thoughts?

    Thank you,

    Peter

    scottgem say:
    What is does the DLookup look like? show us the expression.,

    peter_914_74 say:

    =DLookUp("FeedName","tblFeed","tblFeed.[ChangeL]>=[EstL]")

    [EstL] is a calculated control =((1/[Sample])/0.0004)^0.333333

    [Sample] is a calculated control based on a summery = # fish/ TotalWt of fish

    Each [FeedName] has a [StartL] (min size of fish that can use the feed) and a [ChangeL] is the size in inches where we need to change to a different feed.

    Thank you

    peter_914_74 say: Anyone....anyone....Bueller.....Bueller.....

    scottgem say:

    Try:

    =DLookUp("FeedName","tblFeed","tblFeed.[ChangeL]>= " & [EstL])

    peter_914_74 say:

    All hail Scott the genius!!!!!!!!!!!

    That did it.  You are the man!

    OK now.. why.  What is the operational difference in the two dlookups?  Why was the problem tied to the number of data points behind the calculated control that fed the dlookup?

    1K thank you!

    Peter

    scottgem say:

    A Where clause generally works by concatenating a value rather then including it within the quotes. 

    In 

    "tblFeed.[ChangeL]>=[EstL]"

    Access is not sure where to find what value EstL represents.

    When you use:

    tblFeed.[ChangeL]>=" & [EstL] 

    you are telling Access to concatenate the value contained in the object name EstL.
