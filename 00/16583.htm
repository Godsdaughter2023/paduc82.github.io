nasim10 say:

Access table: wide to long

Hi,

I am a new to this. So, I appreciate any help I get.

 I have this table for Human Resources it has Employee Id and bunch of other variables. I have also 8 vaiables: HOL, HWD, ML, UAb, OT, SCK, VAC, REG which have hours worked. I need to create a case for every field that has a number bigger than 0. So, I will have a table that has as many rows for a person as many bigger than 0 variables they have. Then I will have two variables: Code and Hours- that will have the code(HOL, SCK, VAC, REG,...) and hours.  Other vaiables  (employee id, paydate) need to be copied to every row.

Thank you very much,

john w. vinson say:

Ah... I understand.

 

Yes; create the normalized table, empty. You can then create a "Normalizing Union Query" based on the wide-flat table. A UNION query needs to be created in SQL view - the query grid isn't up to the task!

 

In this case it might be something like

 

SELECT [EmployeeID], "HOL" As AbsenceTypeID, [HOL] AS AbsenceHours FROM [Human Resources] WHERE [HOL] > 0

UNION ALL

SELECT [EmployeeID], "VAC" As AbsenceTypeID, [VAC] AS AbsenceHours FROM [Human Resources] WHERE [VAC] > 0

UNION ALL

<and so on, and so on, through all the absence types>

 

You can and probably should also include other criteria in each WHERE clause to select which records in the wide-flat table to use (date range for instance).

 

You can then save this query and base an APPEND query on it to move the hours and absence types into the normalized table.

I will surely help you with this.

Cookies are a small text files that websites put on your computer to store information about you and your preferences.

I would suggest you to try the steps to enable Cookies.

Block or allow cookies from specific websites


a. Open Internet Explorer.

b. Click the Tools button, and then click Internet Options.

c. Click the Privacy tab, and then move the slider to a position between the top and bottom so you are not blocking or allowing all cookies.

d. Click Sites.

e. In the Address of website box, type a website address, and then click Block or Allow.

f. As you type, a list of webpages that you have already visited will be displayed. You can click an item in the list and it will be displayed in the Address of website box.

g. Repeat step e for each website you want to block or allow. When you are finished, click OK.

h. Move the slider back to the position it was originally in, and then click OK.
