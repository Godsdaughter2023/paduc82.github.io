Simple to export and import your set of default app associations - Microsoft Community

trybird say:

Simple to export and import your set of default app associations

There were many cases could cause your set of default app associations was lost, for example, after upgrading Windows 10 to Build 10586, someone found his or her set of default app associations was lost, for example like such:
Warning! The Windows 10 "Fall Update" resets ALL your default programs!

Imagine that you had been very hard to gradually set up your set of default app associations but then suddenly was lost, wouldn't it make you bitter? So, if there was a simple way to backup your set of default app associations at any time, and then could restore it easily and simply at any time, isn't that a good thing?
Here I would introduce my approach which only using two batch files to simply achieve this solution.


In here, "backup", namely export your set of default app associations; and "restore", namely import your set of default app associations. I named the data file as AppAssoc.xml that saved the set of default app associations, "AppAssoc" namely means "app associations", this data file will be automatically generated by the export batch file; I named the export batch file as ExAppAss.bat that achieved to export or back up the set of default app associations to AppAssoc.xml, "ExAppAss" namely means "export app associations"; I named the import batch file as ImAppAss.bat that achieved to import or restore the set of default app associations from AppAssoc.xml, "ImAppAss" namely means "import app associations".


The operations are very, very simple, please follow me.

First step, prepare your customized settings' backup folder.


You should prepare or already have a folder as your customized settings' backup folder which used to store many common system settings files such as batch files with .bat suffix, PowerShell script files with .ps1 suffix, registry export files with .reg suffix, and so on. For example I have such folder, its location is: "E:\Software\Backups\Settings".


Notice:
You'd better locate or put your customized settings' backup folder on a non-system hard drive; for example like me: Windows 10 installed on drive C: and Windows Server 2016 Technical Preview 4 installed on drive D:, so I putted my customized settings' backup folder on drive E: which is purely a programs data hard drive.


If you have not such folder, you could newly create it, just simply do as follows:


Please open a "Command Prompt (Admin)" window by one of the following three ways ('I. Keyboard' or 'II. Mouse' or 'III. Touch'):
I. Keyboard: Win+X, A, Alt+Y
II. Mouse: rightclick Start button, click "Command Prompt (Admin)", click "Yes" button
III. Touch: press-and-hold Start button, tap "Command Prompt (Admin)", tap "Yes" button


Then in the "Command Prompt (Admin)" window, please run the command below:
[Type or paste the command]
md "E:\Software\Backups\Settings"
[Hit or press the Enter key]


Note:
Replace the E:\Software\Backups\Settings placeholder with the location of your customized settings' backup folder.


Then please click the Close button on the upper right corner of the "Command Prompt (Admin)" window to close it.

Then please press Win+E combination key to open a File Explorer window, expand its tree and navigate to your customized settings' backup folder. For example in my case that is "E:\Software\Backups\Settings", as shown below:

Second step, create export batch file and import batch file.


The operations of creating the ExAppAss.bat export batch file are as follows:


In the File Explorer window that already located on your customized settings' backup folder, right-click on its right details pane, then a shortcut menu pops up, please point to "New" menu item, then its submenu expands out, then please click on the submenu's "Text Document" menu item,

and then name the new text file as "ExAppAss.bat",

then a rename warning dialog box pops up, please click the "Yes" button.

Then please right-click "ExAppAss.bat",

a shortcut menu pops up, then please click "Edit" menu item on the shortcut menu,

then the currently empty file is opened by Notepad.


Please use "hold down and drag" to select the following two commands text in here with your left mouse button:
if exist "E:\Software\Backups\Settings\AppAssoc.xml" del "E:\Software\Backups\Settings\AppAssoc.xml"
dism /Online /Export-DefaultAppAssociations:"E:\Software\Backups\Settings\AppAssoc.xml"


Note:
Replace the E:\Software\Backups\Settings placeholder with the location of your customized settings' backup folder.


And then right-click, then there pops up a shortcut menu, then please click "Copy" item on the shortcut menu.

Then go back the Notepad window's opened ExAppAss.bat file and right-click in it, a shortcut menu pops up, please click "Paste" menu item on the shortcut menu,

then the two commands are pasted in it, as shown below:


Please click the the Notepad main menu's "File" item, and then click "Save" item to save the contents of the file.

Then please click the Close button on the upper right corner of the Notepad window to close it.


The operations of creating the ImAppAss.bat import batch file are as follows:


In the File Explorer window that already located on your customized settings' backup folder, right-click on its right details pane, then a shortcut menu pops up, please point to "New" menu item, then its submenu expands out, then please click on the submenu's "Text Document" menu item, and then name the new text file as "ImAppAss.bat", then a rename warning dialog box pops up, please click the "Yes" button.
Then please right-click "ImAppAss.bat", a shortcut menu pops up, then please click "Edit" menu item on the shortcut menu, then the currently empty file is opened by Notepad.


Please use "hold down and drag" to select the following command text in here with your left mouse button:
dism /Online /Import-DefaultAppAssociations:"E:\Software\Backups\Settings\AppAssoc.xml"


Note:
Replace the E:\Software\Backups\Settings placeholder with the location of your customized settings' backup folder.


Tip:
Why should you make a batch file to execute it although only a command within it?
Because as such you can run it by only know its batch file name insteading you have to remember the full command.


And then right-click, then there pops up a shortcut menu, then please click "Copy" item on the shortcut menu.


Then go back the Notepad window's opened ImAppAss.bat file and right-click in it, a shortcut menu pops up, please click "Paste" menu item on the shortcut menu, then the command is pasted in it.


Please click the the Notepad main menu's "File" item, and then click "Save" item to save its contents.
Then please click the Close button on the upper right corner of the Notepad window to close it.

Now there are both of ExAppAss.bat and ImAppAss.bat in your customized settings' backup folder.

Third step, use ExAppAss.bat to backup to AppAssoc.xml and use ImAppAss.bat to restore from AppAssoc.xml.

You can use your ExAppAss.bat to backup your set of default app associations at any time, just right-click it in File Explorer, then a shortcut menu pops up, then please click "Run as administrator" menu item on the shortcut menu,

then User Account Control dialog box pops up, please click "Yes" button,
then it is running quickly within an opened "Command Prompt (Admin)" window, then its window closes automatically, means that the backup is complete;
if this is the first time running, you will find that the AppAssoc.xml file is generated after it running.

If you found that your set of default app associations was changed after upgrading your system, such as the default web browser was changed from Google Chrome back to Microsoft Edge, the default PDF viewer also was changed from Adobe Reader back to Microsoft Edge, and so on,
then you can use your ImAppAss.bat to restore your set of default app associations from the AppAssoc.xml file at any time, just please right-click ImAppAss.bat in File Explorer, then a shortcut menu pops up, then please click "Run as administrator" menu item on the shortcut menu,
then User Account Control dialog box pops up, please click "Yes" button,
then it is running quickly within an opened "Command Prompt (Admin)" window, then its window closes automatically, means that the restore is complete.


Look, this is the whole process by three steps, isn't it simple?

roady [mvp] say:

1) As mentioned in my first post, I ran Windows and Office with Outlook files saved in the "network" Dropbox folder on my Mac/OSX HDD for more than 3 years without any issues.  It is only since Thursday last week that Outlook suddenly cannot see or open the Outlook files in the "network" location.

2) I dont have any proof, but I have a very strong suspicion that there is/was an MS update on Windows 10 and/or Office 2016 last week (over which I have no control with MS's new update policy) that caused the problems.

1) It doesn't matter that it worked for you for more than 3 years, it isn't a supported configuration and you simply might have dodged the bullet for 3 years. It might have gone right as long as you didn't connect to the files from a different location so there weren't any changes to download (which means a full file sync). Additionally see:

In your case, the problem is even bigger since the location mapped as a network drive which results in a different I/O stream which also isn't supported for ost- and pst-files.

2) No updates were pushed last week that affected Outlook 2016 running on Windows 10.

You really need to get yourself into a supported state again first. That means that the pst-files and ost-files that you want to connect to need to be located on the local drive. In a virtualized environment this means the virtual local disk within the VM itself and not even on the host. Additionally, these files need to be located in a folder that doesn't sync with any server or cloud based storage service. You can of course create backups of these files to these storage locations as long as these are full file copies and are being copied while Outlook is closed.

roady [mvp] say:

It is hard to say why it used to work for you as there are so many variables at play. Most often it is I/O and latency related. Outlook is able to write much faster to the files and the Search Indexer of Windows also expects the data to be local; A network I/O stream has too much overhead.

Backups should of course at least be stored on a different physical disk. As these are mere file-copies, they can be located anywhere that is convenient to you. Cloud storage might not be optimal when you have a slow connection and large files though. A backup of the ost-file of an Exchange mailbox isn't needed since that file is only a local cache and not a "standalone" recoverable file.
