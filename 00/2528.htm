jay freedman say:

Which gallery and category were selected when you created the building block, and in which template was it saved? That information is available by clicking Insert > Quick Parts > Building Block Organizer and locating the name of the building block.

If the building block was saved in the AutoText gallery (with any category) in the Normal.dotm template, then the same code will work with it in 2013 as would have worked in earlier versions that didn't have building blocks, as in this example:

NormalTemplate.AutoTextEntries("foo").Insert Where:=Selection.Range, RichText:=False

If it was saved as AutoText in some other template whose name you know, you can use a function like the one below to get a reference to it. (For some reason, the string containing the name of a template won't function as the index into the Templates collection.) Also note that Word doesn't load building blocks other than those in Normal.dot at startup; it waits until the user opens a gallery or does something else that requires the building blocks. The LoadBuildingBlocks statement forces an immediate load if they aren't present yet.

Sub x()
    Dim BBTemplate As Template
    Templates.LoadBuildingBlocks
    Set BBTemplate = GetTemplate("Building Blocks.dotx")
    If Not BBTemplate Is Nothing Then
        BBTemplate.AutoTextEntries("foo").Insert where:=Selection.Range, RichText:=False
    End If
End Sub

Function GetTemplate(tmplName As String) As Template
    Dim T As Template
    For Each T In Templates
        If LCase(T.Name) = LCase(tmplName) Then Exit For
    Next
    Set GetTemplate = T
End Function

If the building block was saved in some other gallery than AutoText, then you have a more complex programming problem. You have to drill down through the building block type (= gallery) and category and name to get to the right one (look in the Organizer and you'll see that a lot of the blocks have the same name in different galleries, because they come from the same design theme).

Sub x1()
    Dim BBTemplate As Template
    Templates.LoadBuildingBlocks
    Set BBTemplate = GetTemplate("Built-in Building Blocks.dotx")
    If Not BBTemplate Is Nothing Then
        BBTemplate.BuildingBlockTypes(wdTypeCoverPage) _
            .Categories("Built-in").BuildingBlocks("Facet").Insert _
            Where:=Selection.Range, RichText:=True
    End If
End Sub

rob koch say:

If you aren't into yelling or blowing whistles into the phone (I wouldn't recommend it anyway, since they might just spoof your own phone number for their next batch of calls), you could try another common trick others use of just saying you don't own a computer and immediately hang up.

Though it's possible they have a list they acquired from some 3rd-party group that knew you have a PC, it's far more likely these are simply random calls to each number in order, so they know nothing about you.  That's why it's crucial to never provide any information, not even to correct your name or phone number, since that simply provides them with more to use the next time they call back if they retain it.

That's why I indicated that it's best for everyone to get a phone with CallerID and simply never answer callers you don't know.  Otherwise, the first call that's answered they now know is an active number and they can sell that and any other info they get to others as well.

If you think about this carefully, it's specifically designed to abuse the older population who long ago learned the then "proper" phone etiquette to be polite and helpful, filling in whatever information the caller might not have correct.  This plays right into the "phishing" trap these scammers use and is difficult to unlearn for seniors.  However, it's the most valid way to protect yourself in today's virtual world where anyone can reach you on your phone or access your PC if you allow it.

This is the difficult thing for most people to understand, since unlike the typical national and natural borders of the past, the electronic virtual world provides direct access to you and your devices from anywhere.  True Internet freedom comes with a price and that's security for the uninformed or feeble minded.

Rob

Most of these calls will simply terminate when the ring is blocked by the phone, while a few like those you are receiving may leave a voicemail.  I haven't received any of these in voicemail for a few years now, since that leaves recorded evidence which these callers rarely prefer to do.

https://www.donotcall.gov/
