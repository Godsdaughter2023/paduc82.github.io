Key Finder Programs, Belarc, MagicJellyBean, etc do not find a Product Key on a preinstalled OEM Licensed machine (Toshiba, Dell, etc) that is useable.

The Key that they find is the Key that was used on multiple machines during Manufacture at Toshiba's factory.

_________________________________________

It is often recommended in these Forums that you use a magnifying glass and torch to see if you can then read the Product Key digits on the sticker.

 

These are your reinstallation options:

 

You can contact your Computer Manufacturer and ask them to send you a set of Recovery Disks and reinstall the Operating System.

They should do that for a small cost.

 

And if you never received a Recovery Disk when you purchased your Computer there should be a Recovery Partition on the Hard Drive to reinstall Vista back to how you bought your Computer.

The Recovery process can be started by pressing a particular Key or Keys combination at Boot. (Power On / Startup)

With a Toshiba you press 0 (zero) at Boot to start the recovery process

_________________________________________

Or make your own Recovery disks:

Windows 7

1. Click Start

2. Select All Programs

3. Select Recovery Media Creator

4. Select the type of media to use under Disc Set (Single layer DVD, Dual Layer DVD or USB Flash Drive)

5. Click on Create then follow the prompts to insert disks when necessary

Please note: When creating a recovery partition to a USB Device, it must be a USB Flash Drive that is formatted under FAT32. An External Hard Drive will not work.


________________________________________

 

The above reinstallation methods do not normally require you to supply a Product Key during the Installation Process as the Recovery Process uses the Factory Installation Product Key to automatically Activate during the reinstallation process.

Here's what worked for me. Apparently the Windows 10 installation isn't as immune to errors as Windows 8.1 OS. I noticed that as I booted my 8.1, I got a very quickly flashed message on the screen, but it booted anyway. I had to take a video of the boot just to slow it down to see the message. It turned out to be an MBR Error 3 which means that the Master Boot Record was corrupted. A little research and I found a product I had that does modify this record, Acronis True Image 2015. I uninstalled this product, and then used a Windows 8.1 recovery disk to go into a command prompt where I was able to type bootrec /fixmbr to fix the MBR. You can also use bootrec /fixboot to fix the boot record.

Windows 10 installed without a hitch after that.

So advice is, even if your symptoms don't exactly match, use some tools to find out what is going on your 8.1 (or other) boot and if there is a problem (or not), use some technique, including a complete recovery of Windows with a repair option before you start the Windows 10 update. Do some research on products that might tamper with the boot process, and uninstall them so they can't redo what they did.
