ken sheridan say: What you have is a ternary many-to-many relationship type between schools, classes and teachers, which should be modelled by a table which resolves the relationship type into three many-to-many relationship types:

SchoolsClassesTeachers
....SchoolID  (FK)
....ClassID  (FK)
....TeacherID  (FK)

The primary key of this table is a composite one made up of the three foreign key columns, i.e. it is 'all key'.   This table on its own would assume that all schools can teach all classes, all classes can be taught by all teaches, and all teachers can teach at every school.  This would seem unlikely, so you'd model the relationship types between each pair of entity types with further tables.  To model which schools each teacher can teach at therefore would be modelled by a table as follows, which models the binary relationship type between schools and taechers:

SchoolsTeachers
....SchoolID  (FK)
....TeacherID  (FK)

Which classes each teacher can teach would be modelled  by a table as follows:

ClassesTeachers
....ClassID  (FK)
....TeacherID  (FK)

And to model which classes are taught by each school:

SchoolsClasses
....SchoolID  (FK)
....ClassID  (FK)

Again, in each case the table is 'all key'

The relationship types between each of these tables and SchoolsClassesTeachers will in each case be on the relevant pairs of columns, e.g. that between SchoolsClassesTeachers  and SchoolsTeachers will be on SchoolID and TeacherID.  In all of the relationships referential integrity must be enforced to ensure the integrity of the data.

stefan blom say:

The answer depends entirely on what you want these parts to "do." For example, if you want page numbering to restart in each part of the book, you'll have to separate them by section breaks.

If all you want is a page with "Part 1: Introduction" (and similarly for parts 2 and 3), you might use a manual page break (Ctrl+Enter).

suzanne s. barnhill say:

I imagine section breaks will still be required because I suspect Sudhir will not want headers/footers/page numbers on the part title pages, nor, probably, on the first page of a chapter. Therefore an arrangement such as the following will probably be required:

Section 1 (or more): Front matter (if any), perhaps numbered with roman numerals; more than one section will be required if some pages (other than the first) should not have a page  number or  header/footer.

Section 2 (or higher): Part 1 title page; Odd Page section start, header and footer unlinked from the previous and empty but numbering restarted at 1 if front matter uses roman numerals.

Section 3: Chapter 1; Odd Page section start, Different First Page enabled; First Page Header/Footer empty; Header and Footer unlinked from previous and containing desired content (Different Odd and Even enabled if desired).

Section 4: Chapter 2; same as Chapter 1, although header(s) and footer(s) can be linked if the content is the same or changed by using StyleRef fields.

And so on throughout the book. Reference to http://www.word.mvps.org/FAQs/Formatting/front_matter_2007.htm and http://sbarnhill.mvps.org/WordFAQs/HeaderFooter.htm may be helpful.

I had the same problem - looked everywhere and could not find where to revoke OAuth tokens for connected apps. I finally found it today though.

1. Sign in to Outlook.com

2. Click your user icon at top right

3. Click "View Account". This takes you to an account overview page on account.microsoft.com

4. Click "Security & Privacy" on the menu bar

5. There is an "Apps & Services" section there. Click the "Manage permissions" link in this section.

6. You'll see a list of connected apps, you can click Edit to view/remove their access to your account
