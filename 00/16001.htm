                          10

                     Example

In order to avoid this messing up the TOC, I insert a "soft enter" (Shift+Enter) between the chapter number and the chapter name in the heading. This works fine and my TOC entries are single-line.

However, it then messes up my page headers because the soft enter if not suppressed there. I have odd-page headers with right-justified chapter  name and page number, and even-page headers with left-justified page number and chapter name, e.g.

                                          Example  101

102 Example

                                           Example 103

I'd like these to be single-line but the soft enter, present at the start of the chapter name, causes a line-throw. In the even-page headers, especially, it splits the chapter name onto a different line from the page number.

Is there a way to fix this properly, or maybe a better way of achieving my original goal?

suzanne s. barnhill say:

If you could settle for having just the chapter name in the header (without the chapter number), or if you were willing to use two StyleRef fields to get both, you could use the solution described at http://wordfaqs.ssbarnhill.com/TOCTips.htm#UnnumberedHeadings, which uses separate styles for the chapter number and chapter title. One of these styles will be Heading 1 (or whatever level you're using for chapter headings). Whether it is the chapter number or the chapter title will depend on whether or not you're relying on the heading number for page or caption numbering.

Another approach that would work would be to define a character style (identical to Default Paragraph Font, so it adds no formatting) to apply to the title and use that style for the StyleRef field in the header. The problem with this is that you will have to take care not to apply the style to the paragraph mark (or to remove it if Word insists on adding it anyway), as the style applied to the paragraph mark is also applied to the auto number.
