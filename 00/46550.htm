"what is the problem? "

- That you don't understand Directx.  ;)   - How your 'loophole' would have made the game run, I don't understand :D

- The game decides which DX files it uses, not Windows. If any of the many DX files are missing from

System32 or sysWOW64,  or if a DX file is corrupt the game will not run.

- If your graphics processor is not suitable for the game Windows will also sometimes generate a DX error.

"get an DirectX 9.0c offline installer, install it to game dir. and extracted all .cab files,"

- Absolutely not necessary. An GTA-SA instillation does not normally have any DX files in the games directory,

and the .cab files are useless in that all the actual DX .dll files are contained within.

- You are supposed to unpack the DirectX Redist (June 2010 ) - which contains all DX9,10,11 files -

to a folder of your choice and then run the DXSETUP.exe that is in that folder to reinstall DX.

Once DX has been reinstalled you can delete this folder.

(Note that many people still recommend the older DX 9.0c installer but there is nothing special about this

out dated installer. The June 2010 installer  is the better option.)

"I didn't have administration permissions (my families PC) "

I've always run on an admin account so I'm not clear what you need Admin permissions for. The game

- I you are unable to use Compatibility settings you will need to log in on an Admin account.

(my DX cut and paste) -

- It's a common misconception that Win7/8/8.1 has only DX 11, and Win10 has only DX12.
Win7/8/8.1/10 all have  of DX - 9,10 and 11.

- The following is a list of the Direct X .dll's you will have when the files are up to date.
Go to the Windows / System 32 folder, (and SysWOW64 if you have 64bit).  
They are in alphabetical order and will start with d3dx9 - 24 > 43 . Then d3dx10 - 33 > 43 & finally d3dx11 - 42 > 43.
There should also be - d3d9, d3d10 and d3d11 (among others).

These come before the D3Dcompilers - 33>47.
- Win10 currently has only two d3d12.dll  files. One in System 32,and SysWOW64 .
- There are more DX files, but these are most of the graphics related Direct X  .dll's

-----

- A clean install of Windows may be missing some DX files. In this instance running the DirectX Web Installer

will install any 'missing' files. You normally only need to run the full Redist installer when there is a suspicion

of corrupt DX files.

Download DirectX End-User Runtime Web Installer from Official Microsoft Download Center

Download Details - Microsoft Download Center - DirectX Redist (June 2010)

-----

Win10 and DX12 -

- Win10 reports DirectX differently than previous versions of Windows.

MS has made this confusing because DX12 has what MS calls 'Feature Levels', so a graphics processor

that is not fully capable ( capable being the operative word) of using all  DX12 Feature Levels

will be reported as having DX 11.2 or 11.3, depending on which Feature levels it is capable of using.

- The Display tab is what matters. This shows the maximum Feature Level of  DirectX   the

graphics processor is capable of.

The version of DirectX shown on the System page is largely irrelevant as this is only a reflection of what

Win10 has (in addition to the previous versions of Windows and DirectX).

- It is the software/game that determines which version of DirectX that is used.  Not  Windows

There are only a handful of games that use DX12 at this point in time.

Registry Hack to Disable Writing to USB Drives - has Reg file for Enable and Disable
http://www.howtogeek.com/howto/windows-vista/registry-hack-to-disable-writing-to-usb-drives/

Enable USB Writes

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\StorageDevicePolicies]
"WriteProtect"=dword:00000000

Disable USB Writes

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\StorageDevicePolicies]
"WriteProtect"=dword:00000001

----------------------------------------------------------------------

The MyDIgital Life link works - to reverse it set it to 0.
http://www.mydigitallife.info/write-protect-and-disable-usb-drive/

Run Registry Editor (regedit).

Navigate to the following registry key:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control

Create a New Key named as StorageDevicePolicies.

Highlight StorageDevicePolicies, and then create a New DWORD (32-bit) Value named as WriteProtect.

Double click on WriteProtect, and set its value data to 0.

(Set to 1 to enable WriteProtect - makes it read only)

Hi srimanch,

 

Thanks for posting in the Microsoft Communities. If the update is not applicable to the Windows version, you can hide it. This is a pretty easy trick, but the option for hiding updates in the list is itself somewhat hidden and often goes overlooked.

 

In the list of available updates, right-click any update (KB number) that you don’t want to see again, and choose Hide Update. If you later you just want to see a list of the updates you’ve chosen to hide, on the main Windows Update page, click Restore Hidden Updates.
