    How to find the currency value of a cell... - Microsoft Community

    luckyita say:

    How to find the currency value of a cell...

    Hi guys.

     I need to write in a cell the currency value of data inserted in another cell.
     An example. If in A1 you have $23.99 I'd like to put in B1 "Dollar" (or $), and if in A2 you have €23.44 in B2 I'd like to find "Euro" (or €) , and so on.
     Do you have some tips? Actually I'm not able to do it.

    Thanks.
    jeeped say:
    If your monetary amounts are actual numbers (as they should be) with varying currency number formats, you will need VBA to determine the number format and parse the currency symbol from it. There are a variety of possible number formats that could produce the correct currency format, not all of them having the currency symbol as the first character and this must be compensated for when looking for the currency symbol that is applied to the cell.

    Tap Alt+F11 and when the VBA opens, immediately use the pull-down menus to Insert, Module. Paste the following code into the new pane titled something like Book1 - Module1 (Code),

    Function fcnWhatCurrency(rMoney As Range) As String
        Dim sFormat As String, sCurrency As String
        sFormat = rMoney.NumberFormat
        sCurrency = "none"
        If InStr(1, sFormat, "$", vbTextCompare) > 0 Then sCurrency = "Dollar"
        If InStr(1, sFormat, "£", vbTextCompare) > 0 Then sCurrency = "Pound"
        If InStr(1, sFormat, "¥", vbTextCompare) > 0 Then sCurrency = "Yen"
        If InStr(1, sFormat, "€", vbTextCompare) > 0 Then sCurrency = "Euro"
        fcnWhatCurrency = sCurrency
    End Function

    Tap Alt+Q to return to your worksheet. This User Defined Function (i.e. UDF) can be used in the same manner as any other built-in function. In the example from your post, you would use,

    =fcnWhatCurrency(A1)

    Additional currencies can be easily added by copying the If(Instr(… line and adjusting both the symbol and the resulting text.

    jay freedman say:
    When you protect a document for filling in forms, many of Word's functions are disabled,
    even in sections that remain unprotected. The building block gallery (a type of content
    control) is one of those functions.
     
    In some cases it's possible to work around the restriction by writing a macro that
    unprotects the document, lets the user choose something to insert or format, and
    reprotects the document; because the macro controls the actions, the user never gets the
    chance to change anything else in the document. It may be possible to write such a macro
    for this template, but it would require more information about the gallery (its title or
    tag value, and whether that is unique within the document).
     
    Basically, content controls and forms protection don't play nicely together. It would be
    best, if it's possible, to replace all the legacy form fields in the template with the
    equivalent content controls, so it wouldn't need forms protection to operate. To prevent
    users from changing anything outside the content controls, select the whole document and,
    in the Controls section of the Developer tab, click the Group button and choose Group from
    the menu.

    Hi,

    your data in columns A:H, on Sheet1

    (in row1 are headings)

    Group Reviewing in column E

    extract data in seperate sheets(GMO, GP&F ,..)

     

    try this code..

     

    Sub macro01()
    '12 Oct 2012//
    Dim ws1 As Worksheet, ws2 As Worksheet
    Dim r1 As Long, r2 As Long, X As Long
    Set ws1 = Sheets("Sheet1") ' <<< Source Sheet Name, change  as needed
    Application.ScreenUpdating = False
    ws1.Columns(1).Insert
    r1 = ws1.Cells(Rows.Count, "B").End(xlUp).Row
    ws1.Range("F1:F" & r1).AdvancedFilter Action:=xlFilterInPlace, Unique:=True
    ws1.Range("F2:F" & r1).Copy
    ws1.Range("A1").PasteSpecial Paste:=xlPasteValues
    Application.CutCopyMode = False
    ws1.Range("F1:F" & r1).AdvancedFilter Action:=xlFilterInPlace, Unique:=False
    r2 = ws1.Cells(Rows.Count, "A").End(xlUp).Row

    For X = 1 To r2
    For Each ws In Sheets
    Application.DisplayAlerts = False
    If ws.Name = ws1.Cells(X, 1) Then ws.Delete
    Application.DisplayAlerts = True
    Next
    Next

    For X = 1 To r2
    Set ws2 = Worksheets.Add(after:=Sheets(Sheets.Count))
    ws2.Name = ws1.Cells(X, 1)
    ws1.Range("F1").AutoFilter Field:=6, Criteria1:=ws1.Cells(X, 1)
    ws1.Range("B1:I" & r1).Copy
    ws2.Range("A1").PasteSpecial Paste:=xlPasteValues
    Application.CutCopyMode = False
    ws2.Columns("A:H").AutoFit
    Next X
    ws1.Range("A1").AutoFilter
    ws1.Columns(1).Delete
    Application.ScreenUpdating = True

    End Sub

    3) Press ALT+Q to Close Visual Basic

     

    To run the macro, press ALT+F8, 

    select <macro01> from the list and click the run button.

     

     

    note

    each time you run the macro, deletes previous entries (Sheets)

    You will need to make this change in the Normal template, Normal.dotm. Although you may be able to navigate to Normal.dotm using Mary's instructions, most users find it difficult to locate, especially since it is in a Hidden folder. Here's an easier way to open Normal.dotm for editing:

    In Word, press Alt+F11; the Visual Basic Editor opens. Press Ctrl+G to activate the Immediate window. In that window, type

         NormalTemplate.OpenAsDocument

    and press Enter. Normal opens, but note that it may not be on top of all windows. You can close the Visual Basic Editor.
