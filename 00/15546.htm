bernie deitrick say:

240 hours is 10 days - Excel stores times as decimal fractions of days.

So, it depends on if you are in time space or in hour count space. You could use a formula like:

=IF(A2>10,10,A2)

Formatted as [h]:mm

Or just this

=MIN(10,A2)

If the stored value is a string, you will need to convert to time first, using 

=MIN(10,(DATEVALUE(A2)+TIMEVALUE(A2)))

If you want the hours as a number instead of in time format, then you need to multiply by 24.

glenprouty say:

Hi Tim,

I do not know why Defender (Controlled Folder Access) would suddenly start to block software that it had previously passed.

You asked how you could "whitelist" your software. If your PC is at version 1809 that process is  fairly simple.

Click on the notification that it has been blocked, or on a notification on the taskbar concerning the same.

You will be taken directly to the "add an app" routine in CFA. Clicking on the + sign there, will present a list of "recently blocked apps". Selecting that list will display the apps that have been blocked by CFA. One click on any item in the list will "whitelist" it.

Caution, there may be items in the list that you do not want to pass through CFA.

If you are not at version 1809, in order to find the complete path for your app to pass, you must look in the event viewer.

Expand >> Application and Service Logs >Microsoft > Windows > Windows Defender and open Operational. Search for event ID 1123 or 1127. Displaying "properties" for any of these events will show the path that you will need to use to "whitelist" your app.

To allow your app to pass through CFA, under Virus & Threat Protection in Windows Defender,

find Ransomware Protection. Select "Manage ransomware protection". Under "Controlled folder

access" select "Allow an app through Controlled folder access". Yes to UAC.  Then select "Add an allowed app.

Using the PATH information that you obtained from the event viewer, navigate through the explorer screen. When you have reached your app, select "open". Your app is whitelisted.

I kept looking and found this slightly old similar question. Although it is "old", it does provide some interesting real world insights and comments.

Summary:

- optimize the spreadsheet to ensure that it is not forced to be single threaded

- build (rather than buy) a custom machine using the fastest CPU you can get,

- build with the most RAM you can put in it
