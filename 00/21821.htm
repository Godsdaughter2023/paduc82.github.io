ken sheridan say: It appears from your screenshot that each of the two subsets is ordered by Piece, in which case you do not need to, nor should you, store the sequential numbers in a column in a base table, but can return them in a computed column in a query.  The numbers would consequently be automatically recalculated as its are transferred from one subset to the other in aether direction.

The most efficient way to sequentially number rows is by a JOIN, but this returns a non-updatable result table, so would be unsuitable as the RecordSource of your subforms in which you need to update the Boolean column to which the check boxes are bound.  It would be necessary therefore to call the less efficient VBA DCount function.

You might like to take a look at RowNumbering.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

This little demo file illustrates a number of queries for numbering, ranking or partitioning rows.  The criteria for the DCount function call are a little more complex than you would need, as in the demo, the primary key is called into play as a tie breaker in the event of two rows having the same TransactionDate, whereas in your case the Piece column's values are distinct.  You would of course need to include the Boolean column in the criteria in your case to number each subset independently.

If we simplify the relevant query in my demo to remove the tie breaker, and assume that the TransactionDate column's values are distinct, and that the table incudes a Boolean column named Selected, the query would be:

SELECT DCOUNT("*","Transactions","TransactionDate  <=  #" &
FORMAT(TransactionDate,"yyyy-mm-dd") & "# And Selected = TRUE") AS RowNumber,
CustomerID, TransactionDate, TransactionAmount
FROM Transactions
WHERE Selected = TRUE
ORDER BY TransactionDate;

Yours would be similar to return the selected items sequentially numbered.  To return the unselected items would simply mean amending it to Selected = FALSE.

2. Remove files
3. Put your card back to phone
4. Reboot phone

DOWNSIDE AND WORKAROUND!!!
