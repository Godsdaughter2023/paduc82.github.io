Hello.

When you say "security scan", which anti-virus suite did you use?  Be sure you are not running any rogue anti-virus scans!  Continuing to your "script error message". Here are some possible reasons for the pop-up from http://www.technibble.com/how-to-stop-internet-explorer-script-error-messages/.


1. The author of the webpage has simply made some sort of programming error and this is typically the most common reason.

2. You are possibly running a pop-up killer of some kind which closes the pop-up too quickly which causes the script of the website to fail and thus giving you the error because it didnt complete.

3.  You (still) have some sort of spyware or adware infection that tries to launch something such as a popup but causes script errors.


#3 is the most likely issue in your situation. 

Solution:

1. Make sure your Anti-Virus software is up-to-date and then run a full system scan.

2. Try running a scan with other malware detection software. (Malwarebytes @ http://www.download.com)

3. Be sure your are not running Rogue Anti-Virus software ( Security Center, Anti-Virus 2009, and Anti-Virus 2010).

4. Take a look into the Event Viewer to see the path of where the script is coming from. (So far I see that it is coming from your documents and settings folder.  Some more detail may show up in the Event Viewer log)

5. Run windows updates and install any critical/important updates.

I hope this helps you with your problem.

john sr wilson say:

The Document Inspector never removed embedded objects (nor sadly tells you where to look)

This macro MIGHT find it.

Sub Chex()
Dim osld As Slide
Dim strReport As String
Dim oshp As Shape
For Each osld In ActivePresentation.Slides
For Each oshp In osld.Shapes
If oshp.Type = msoEmbeddedOLEObject Then
strReport = strReport & oshp.Name & " on slide " & osld.SlideIndex & " is an embedded object." & vbCrLf
End If
Next
Next
MsgBox strReport
End Sub

All versions of Excel do that. A formula in Excel cannot change the format of a cell in any way, all a formula can do is return a value.

mike h.. say:
That's what I thought. But Excel 2010 that I have tested on 4 different machines

Hi,

I've tested this in E2003, e2007 and e2013, I don't have E2010 to hand right now but believe it to be the same, and they all behave in the same way.

1. Put a shape on a worksheet.

2. Enter some text in (say) a1 and colour the font (say) Red.

3. select the shape

4. Click in the formula bar and type =A1

5. Tap enter and the shape is populated with the text in a1 and picks up the red colour.

6. Change the font colour of A1 and the colour of the text in the shape doesn't change.

7. Force Excel to calculate by tapping F9 or by entering another formula in a cell and the font colour still isn't picked up I the shape.

8. Select the shape and click in the formula bar and tap enter and the font colour in the shape updates to the same as the font colour in A1.

AFAIK there is no other way except by resorting to VB code.
