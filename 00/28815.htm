rob koch say:

Though Microsoft Security Essentials (MSE) does have some background capabilities relating to websites, it doesn't directly block them under normal circumstances, since they would need to contain malware or some dangerous script for it to do this.  For this reason it has no "preferences" or settings relating to websites or ports, which are instead typically controlled by your firewall.

In any case the MSE History logs should contain entries relating to the Google Earth website or Google.exe file if it were blocking anything for these, as well as displaying notifications on screen while you are attempting to perform this access.  If it hasn't done either of these then MSE is highly unlikely to be involved.

It's far more likely that some sort of Internet proxy or other website control application such as a firewall is involved in this type of blocking.  This could exist either within your PC or on the pathway to the Internet, such as the network in a business, a school or other small Internet Service Provider.  If you've confirmed that another PC can access Google Earth from within your same network then it's most likely something on your own PC.

Did you see this older thread on the Google Earth Help Forums that discusses the same problem?

https://productforums.google.com/forum/#!topic/earth/8X8vGO1Smd4

If the items in that specific thread don't help, you'd still be better off posting your problem there, since you'll find far more people using Google Earth in those forums than here in a Microsoft antimalware forum.

Rob

rob koch say:

System restore can often create other problems and so should only be used as a last resort if the entire system is not operating properly.  Since a system restore basically rips out portions of items that have changed back to the date you select, it can also leave other pieces, resulting in portions of programs out of sync with the rest of the system.  So no, don't try this again.

Yes, somewhat. Point of reference, Project Pro can either be used stand alone or with Project Server (I use it stand alone). Project Standard can only be used stand alone.

No tasks in a given project plan should have the exact same name. If they do, they are in fact the same task. That's why it is so important to create a project plan using action verb descriptions that are definitive of the work to be accomplished. But alas, it sounds like I may be preaching to the choir on that.

Normally at this point I would refer you to a Project FAQ on Project's three object types but something has happened to the webpage link to those FAQs. I'm trying to find out what. Meanwhile this is an excerpt from FAQ 51:

Project stores data as one of three individual types:  Assignment data bridges task and resource data by matching resources to the task. Each data type has several extra fields associated with it. There are extra text fields, cost fields, etc, and they are all independent, that is, Task Text1 is not the same as Resource Text1 nor is it the same as Assignment Text1. When using Project Professional with Project Server several more enterprise level extra fields are available.

When in a task view (e.g. Gantt Chart), only task data can be shown and in a resource view (e.g. Resource Sheet), only resource data can be shown. The Usage views (Task or Resource) are special combination views that display either task and assignment data or resource and assignment data. When extra fields are displayed in a Usage view they must be properly interpreted. If Text1 is shown in the Task Usage view, task rows will display Task Text1 while assignment rows will display Assignment Text1 - and they are completely different and independent.

Task and Resource extra fields are customizable by the user. They can be renaming and customized with formulae and value lists. Extra Assignment fields cannot be renamed or contain formulae or value lists but they can contain user data. However, in either Usage view Task or Resource value list data can be rolled down to Assignment  rows.

In Project versions up to 2003, there is only one set of assignment extra fields. Introduced in Project 2007, there are two types of assignment extra fields: those visible in the Task Usage view (Task Assignment fields) and those visible in the Resource Usage view (Resource Assignment fields), and they are completely different.


There are very few fields shared by more than one of Project's main object types (i.e. task, resource and assignment). So even if you use an extra field for a unique identifier, you will have to replicate that field for both the task and assignment objects.

My suggestion, press for modifying task description to be definitive and unique. Otherwise, get ready to do a lot of copy and paste or some VBA.
