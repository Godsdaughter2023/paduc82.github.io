john - project say:

Ronald,

It sounds like you have a intermediate milestone in the middle of your project. Something must be driving it.

If it's not a milestone but some other kind of task the needs to be completed before the rest of the project can keep moving, you can set a start-no-earlier-than constraint of the task. The easiest way to do that is to manually enter the start date. There is no built-in way to tie that start date to the current date. It could be done with a paste link from a custom date field that queues on the current date but paste links are prone to corruption. ¡firedog say:

I set up an IMAP account in Outlook 2010 and migrated the folders over to Outlook.com ... I was unable to move the emails in the folders between Outlook 2010 and Outlook .com. 


The whole point about IMAP is that folders on the client mirror those on the server. You can't have one set of messages in a folder on the server and a different set in the same folder on the client. If this is what you're trying to do, then it's doomed to failure.

You can have folders on the server that are not synchronized with the client, meaning that you can maintain an archive on the server whose contents are not replicated in the client. To do this, turn off synchronization in the client for the folder concerned.

You should be able to have folders on the client that are not synchronized with the server, so that you can maintain an archive locally of messages that are not on the server. Any such folder will have to be located outside the account in Outlook 2010. If you don't know how to arrange this, either consult Outlook 2010 Help or ask in the Outlook 2010 forum.

If you made it a bit clearer what you mean by move the emails in the folders between Outlook 2010 and Outlook .com, it would be easier to advise you. I can see no need to involve any other program (e.g. Windows Live Mail) in order to achieve what it looks like you're trying to do, and I don't really see where that misunderstanding arose.

There's only a couple of ways I can figure this would happen.  One is a rule you have created.  Visit http://mail.live.com/, log in with your Hotmail/Outlook.com address, click the Options gear, then "Manage rules".  Make sure you have no rules that would delete the messages.

The second thing that could be happening is you accessing the mailbox using a POP client.  It could be on a computer or it could be on a mobile device or it could be from another mail service like gmail.  If you pull messages via POP, the client side can then tell the server to delete the messages.  Outlook.com can be configured to disregard the deletions and, instead, move the messages to a folder named "POP", but it can also be configured to obey the deletions.

Only you can know if either of these situations is what is going on.
