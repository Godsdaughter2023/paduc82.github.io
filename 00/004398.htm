That would depend of exactly what was on slide 14 (or the current slide with the button)
IF eg the text was in shapes(2)

Sub sendMail()
    Dim OutApp As Object
    Dim OutMail As Object
    Dim osld As Slide
    Dim strmessage As String
 On Error Resume Next
 Set osld = SlideShowWindows(1).View.Slide
 If osld.Shapes(2).HasTextFrame Then
 If osld.Shapes(2).TextFrame.HasText Then
 strmessage = osld.Shapes(2).TextFrame.TextRange
 End If
 End If
 If strmessage = "" Then strmessage = "No Text"
 Err.Clear
 Set OutApp = GetObject(Class:="Outlook.Application")
 If Err <> 0 Then Set OutApp = CreateObject("Outlook.Application")
    OutApp.Session.Logon
    Set OutMail = OutApp.CreateItem(0)
   
    On Error Resume Next
    With OutMail
        .To = "*** Email address is removed for privacy ***"
        .CC = ""
        .BCC = ""
        .Subject = "Subject here"
        .Body = strmessage
        .Send
    End With
 
    Set OutMail = Nothing
    Set OutApp = Nothing
        
End Sub

john korchok say:

For a more complete range of symbols, try the Edit>Emojis & Symbols command provided by macOS. Initially, you'll just see Emojis, but if you click on the icon in the upper right corner of the dialog, it will expand to show categories of symbols. Perhaps the ones you need are in the Latin section.

If they're not there, it's worthwhile to enable the Unicode Hex Input:

  1. In macOS, open System Preferences>Keyboard and click on the Input Sources tab.
  2. Click on the plus + sign.
  3. Scroll down to the last category Other.
  4. Select Unicode Hex Input and click on the Add button.

Now you'll have a keyboard choice dropdown on the right side of the macOS menu. When you choose Unicode Hex Input, the effect of the Option key is altered. You can hold down Option and type a 4-digit number to get any Unicode character.

Excel versions 97 to 2003 used a file format called BIFF8. Later versions of Excel use a different XML-based format, but can also open older versions. Adding/modifying/deleting Office won't affect your personal files. The remain available for whatever version of Excel you are using. Excel 2010 has no knowledge of what version created it. To Excel, an xls file is just another xls file, regardless of how it was created.

 

With all Office applications closed, go to the Windows Start Menu, choose Run and enter the following and the press OK.

Excel.exe /regserver

 

THis will cause Excel to restore itself to "factory defaults".
