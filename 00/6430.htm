brian tillman [outlook mvp 2007-2019] say:

My question is...How can someone send an email from my own email account without being able to use my email account??? 


It's quite easy.  Everything you normally see on the screen when examining a mail message is part of the body of the message (even though it looks like the headers) and the From, To, Cc, Bcc, Subject, Date, and all those other items are totally unrelated to how the message is transferred between mail transports.  The actual sender and recipient are part of the protocol interchange in which the sending and receiving mail transports engage and that exchange (called the "envelope") isn't anything you see when examining the message under normal conditions.  So, the sender and recipient addresses are actually independent of whatever is in the From and To fields and the spammer can insert anything he wishes into those fields and it doesn't affect how or to whom the message is delivered.  So, the program being used by the spammer inserts your address (note that it has your address or you wouldn't be receiving the message) as the From field and when you view the message it looks like it comes from you because the program you use to view the message can't show the actual sender, only what's in the From field.  It doesn't really come from you, though.  It comes from the spambot computer which could be anywhere in the world.

jeeped say:
I am looking to a badge size of circa 2.5 x 2.0 inches, that is 15 per A4 sheet of paper. I could use mail merge to insert names for the initial run but subsequently I would need to insert individual names as newcomers joined the group.

It sounds like you have a limited number of badges to create and Word might be your best method of achieving the results. While both Word and Publisher can use Word Art and mail merge, the text in a Word Art object cannot be pulled from a mail merge. You would need to design the common badge background with the group's name, etc. and then use non-Word Art text for the names. Since you want to add to these later, I'd recommend abandoning the mail merge altogether and just creating a static set of badges that can be added or modified in the future.

With Publisher, you typically create a single badge and use the Print Setup command to print multiples per page. With Word, you would set up a table of badges to fill an entire page and could put the static text (e.g. group member names) into each one. Rows in the table could be copied and pasted to create new rows and a multi-page document would be your end result. Given your stated criteria, I believe the Word solution is a better fit for your immediate needs.

Performing a wildcard find and replace with Track Changes turned on. - Microsoft Community

therogueb97d say:

Performing a wildcard find and replace with Track Changes turned on.

Hi, all.  I have a problem with using Microsoft Word's find and replace functionality, using wildcards with Track Changes turned on.  The problem is an old one: I've seen it in many versions of Microsoft Word going back to at least Microsoft Word 2007, but it is really beginning to bug me.

In short, the problem is that wildcard find and replace does not work correctly when Track Changes are turned on: the replacement is not what it should be.  The problem is best explained with an example.  Start with the text "Now is the time for all good men 2 come 2 the aid of their country." and perform a wildcard find and replace as follows:

Find what: "(come) (<[0-9]>)"

Replace with: "\1 #D\2#"

(The double quotes are for clarity: they are NOT part of the find and replace entries.)

The idea is to replace only the second numeral "2" with "#D2#" (that is, to apply a coded prefix and suffix, for later processing: yes, ultimately this needs to work within VBA, but the problem exists even with the find and replace dialog box).  With Track Changes turned off, this works just fine.  But if Track Changes is turned on when the find and replace is run, the result is something like "Now is the time for all good men 2 come2 #D# aid of their country.  If you look at the text with markups, it is even uglier, as insertions and deletions are intermixed: "Now is the time for all good men 2 _come_-com-_2_-e 2-_#D#_ aid of their country. (underscores surround insertions; hyphens surround deletions).

Let me add that using the word "come" as a field that is reinserted (rather than straight text) changes the extent of the error, but not its existence.  If I perform a wildcard find and replace on the same text as follows:

Find what: "come (<[0-9]>)"

Replace with: "come #D\1#"

the numeral "2" now appears after the letter "c" in the inserted word "come".

Unlike some other posters with find and replace problems, who were advised to simply deactivate Track Changes, I need to see what has changed, so I need Track Changes to be active. 

I would prefer it if the find and replace kept the deletions and insertions as a block: for example, "-come 2-_come #D2#_".  But I can tolerate mixing them up.  What I cannot accept is that the result of the find and replace is wrong.

Can anyone help me get this find and replace to work?

Thanks in advance.

Ariel

doug robbins - mvp office apps & services (word) say:

If it needs to work with VBA, the following code will replace

with

when Change Tracking is in effect.

Here's the code:

Dim rngfound As Range
Selection.HomeKey wdStory
Selection.Find.ClearFormatting
With Selection.Find
    Do While .Execute(FindText:="come ", MatchWildcards:=False, Forward:=True, Wrap:=wdFindStop) = True
        Set rngfound = Selection.Range
        With rngfound

             .MoveEnd wdWord, 1
             If IsNumeric(Trim(.Words(2))) Then
                   .Words(2) = "#D" & Trim(.Words(2)) & "# "
             End If

        End With

        Selection.Collapse wdCollapseEnd
    Loop
End With

Or, if you want to end up with:

Use

Dim rngfound As Range
Selection.HomeKey wdStory
Selection.Find.ClearFormatting
With Selection.Find
    Do While .Execute(FindText:="come ", MatchWildcards:=False, Forward:=True, Wrap:=wdFindStop) = True
        Set rngfound = Selection.Range
        With rngfound
            .MoveEnd wdWord, 1
            If IsNumeric(Trim(.Words(2))) Then
                With .Words(2)
                    .InsertBefore "#D"
                    .MoveEnd wdCharacter, -1
                    .InsertAfter "#"
                End With
            End If
        End With
        Selection.Collapse wdCollapseEnd
    Loop
End With
