kergga say:

It looks like you have 4 SUMIFS, and the first and third are related, and second and fourth are related.

I'm not sure if this would be faster (SUMIFS I think is pretty fast) but you could combine the related statements in a sumproduct

(aircode)

=sumproduct('PO Tracker'!$Z:$Z,OR('PO Tracker'!$Z:$Z=1040, 'PO Tracker'!$Z:$Z,955)*1,( 'PO Tracker'!$V:$V<>””)*1, ('PO Tracker'!$AD:$AD<>”Transmit to Backend System")*1)

do the same for the other related pair and add them together.

Or you may even be able to shorten it further by just summing the two columns to sum, since the rest of the parameters I think are the same?

=sumproduct(('PO Tracker'!$Z:$Z+'PO Tracker'!$AN:$AN),OR('PO Tracker'!$Z:$Z=1040, 'PO Tracker'!$Z:$Z,955)*1,( 'PO Tracker'!$V:$V<>””)*1, ('PO Tracker'!$AD:$AD<>”Transmit to Backend System")*1)

Not sure that will work as is, but it will hopefully give you some ideas.

EDIT:

Actually, I just realized that your last two paired parameters are looking at different columns, so you probably will need to use two seperate sumproducts and add them together.

I also recommend that rather than using sumproduct against entire columns, that you limit the sumproduct range to the real range of your data, for example (if you have a header row in row 1, and data ends on row 150):

=sumproduct('PO Tracker'!$Z2:$Z150,OR('PO Tracker'!$Z1:$Z150=1040, 'PO Tracker'!$Z2:$Z150,955)*1,( 'PO Tracker'!$V2:$V150<>””)*1, ('PO Tracker'!$AD2:$AD150<>”Transmit to Backend System")*1)

    cbarnhorst say:

    There is no self-activation feature particular to previews.  There was before the original Windows 10 released in 2015, but that ended a couple of months prior to RTM of the final.  The vm on which you install a preview must already be a Registered Windows 10 Device resulting from previous activations of Windows, or you must have a license specifically for the vm (product key).  

    Unlike installation on a new partition, which activates automatically based on the device's registration as a Windows 10 device (free multi-booting), a virtual machine is a separate device and does not inherit the registration of the host.

     

    An alternative to using a vm:

    If you are already running an activated Windows 10, an alternative to creating a vm for checking out previews is to create a new partition on your present hard drive and simply install on that.  As long as you install the same edition that is already installed on the device, the new installation will activate automatically.

    While it is true that the free upgrade period has been over for nine months, persons who use assistive devices to operate their computers can still upgrade from Windows 7 or 8.1 for free.  Should you be such a person, the url for that is

    https://www.microsoft.com/en-us/accessibility/windows10upgrade

    If you have any uncommitted licenses for Windows 7 or 8.1 you can create a vm and upgrade from there via the above link should you be a person who uses assistive technologies with your computer.  (Remember, however, that Windows 8.0 cannot be upgraded to Windows 8.1 in a virtual machine.)

    If you are using Hyper V:

    I advise against upgrading a Windows 7 installation to Windows 10 in a Hyper-V vm because Windows 7 uses only Generation 1 vms and once created the generation of a vm cannot be changed, even as the result of an upgrade.  Windows 8.1 Hyper-V vms do support creation of the original vm as a Generation 2 vm and that is much more desirable when 8.1 is upgraded to 10.  For a TechNet article on the advantages of Generation 2 Hyper V vms go here:

    https://technet.microsoft.com/en-us/library/dn282285(v=ws.11).aspx
