If you want to filter, you need field names/column headings in row 1. So I'll assume the data start in row 2.

Try this in E2:

=SUMPRODUCT(($A$2:$A$100=$A2)*($D$2:$D$100=-$D2)*($D2<>0))>0

and fill down. It'll return TRUE if there is at least one match, FALSE otherwise. You can filter on TRUE.

jay freedman say:

Something -- maybe several somethings -- are very wrong here.

  • By design, a .dotx template cannot contain a macro. Only a .dotm template (or a .docm document) can contain a macro.
  • Word never "generates code" when you just create a template. It has to come from somewhere. More details in the next bullet point:
  • There are only three ways you could get a .dotm template (or a .docm document) to contain the code you showed in your first post: (1) It was copied from somewhere (such as a forum post) and pasted into the macro editor, then saved. (2) It was typed into the macro editor, then saved. (3) The template was created by copying another template that contained the macro, then saved to a new name/location. All of these are improbable, so let's go on to the next bullet point...
  • It's possible that the macro code you posted isn't in the template you created at all, but in the Normal.dotm template that Word loads whenever you start the program. In that case, the macro would run and display the same error every time you create a new document based on any template (except for the "new blank document" that appears if you start Word without any other document). Since most users rarely use any template other than Normal.dotm, the problem wouldn't have shown up until you took the course and created your own template.

The simplest way to tell whether my last guess is correct: When you click Debug on the error, and the macro editor opens, look at the macro editor's title bar. Does it contain the word "Normal"? (It may or may not say "NewMacros" after that.)

If so, the easy way to get rid of the problem takes three steps:

  • On the menu bar, click the Run menu and choose Reset. That stops the macro.
  • Select all of the code and then press the Delete key. (There's no earthly reason to keep this macro!)
  • Click the File menu and choose Save. That will save the Normal.dotm template without the macro, and now you can close the macro editor and forget about it.

You'll probably have to do the same thing on both your home and work computers, since the same problem appeared in both places.

I recommend that you talk to the instructor of the course that you mentioned, and find out what he or she thinks of this situation. As I said, the code you posted is certainly meant to be saved in and run from Excel, and it's important to figure out how it might have gotten into a Word template.
