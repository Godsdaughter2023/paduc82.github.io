Memory problems are not so simple.  A simple memory test can pass while a comprehensive memory test can fail because there are many factors that can cause it to fail.  If the CPU caches are enabled the CPU can write a value and then read it right back from its cache and see the same  value but when  that value eventually makes it back to the memory it could have dropped some bits.  It is best to run memory tests  with the  CPU caches turned off.  That causes the CPU to actually write all the way to memory and read those value back from memory.  Some memory will not fail unless certain patterns of reference occur so there are many types of memory tests that use different algorithms to try to detect memory failure.  It  could be that Windows 10 uses more memory than WIndows 8.1 did or references the memory differently and could experience memory problems that do not appear in Windows 8.1.  I do not see hardly any posts about file system problems so the first thing to eliminate is a memory issue by running a thorough memory  diagnostic.

The windows memory  diagnostic is found in Control Panel/Administrative tools.

You can choose to restart  immediately or schedule the  test and restart at a time of your choosing.

When  rebooting, the memory test  will start.  Press F1 to set options.

Choose Extended and tab to set Cache off.  Then press F10  to apply and start running the tests.

You can also do the extended test again with Cache turned on to see if any problems occur in the CPU caches.

jay freedman say:

No, Word is performing "as designed". When forms protection is turned on, the cursor/selection point cannot be placed anywhere in the document except within a form field. That's true whether you're using the mouse, the arrow keys, the navigation pane, TOC or other hyperlinks, the GoTo dialog, or any other means of moving the cursor.

If your document doesn't have to be shared with users of Word 2003 or earlier, or Word for Mac, then replace all the form fields with content controls. To provide protection against editing of the static text, select the "No changes (read only)" form of restriction, and then select each content control and check the "Everyone" box in the restriction pane. Alternatively, if you don't need to place a password on the protection, just select the whole document and click Developer > Group > Group. In either case, the TOC and navigation pane will operate normally.
