joeu2004 say: GWnAtl wrote:
Yes, before. However, when I opened the file in Notepad they 12 digit fields were in standard format, e.g. 123456789012 instead of 1.12345E+11. But when I go into Excel they are back to scientific notation and changed to General format. So, I think they are stored in non-scientific form but Excel just displays them in scientific. That is correct.

When Excel saves to a CSV file, numbers are written as they appear in cells.  So if the cell value is 123456789012, and it is formatted as Number with zero decimal places, that is what you will see in the CSV file.

But by definition, a CSV file is simply a text file.  It contains no format information.

So normally, when Excel opens a CSV file, it interprets the data as if we had entered them manually.  And the cell format is indeed General.

That is why you see 1.12345E+11, but the actual cell value is 123456789012.

It does not help to write numbers to the file surrounded by double-quotes.  Excel ignores the double-quotes and interprets the text between them as if we had entered them manually.

You can work around the original problem by starting Excel directly (do not click on the CSV file icon), format the cells as you require, then use Get External Data From Text to input the CSV file.

The CSV file can be saved normally beforehand, without putting double-quotes around long numbers, as long as the cell format displays the numbers with full precision.

Caveat:  If you write text that appears to be a number with more than 15 significant digits to the CSV file, even with double-quotes, you will lose precision when you reopen or re-input the CSV file because, again, Excel interprets the text between the double-quotes as a number just as if we entered it manually.  When we enter more than 15 significant digits manually, Excel truncates after the 15th digit and substitutes zero for the remaining digits.

To avoid that problem, you would need to format the entire column as Text when you do Get External Data From Text to input the CSV file.

FYI, if you were to resave to the same CSV file without changing the General format, Excel would write 1.12345E+11 into the CSV file.  If you subsequently open the resaved CSV file, you would lose precision because the actual value would be 1.12345E+11, i.e. 11234500000.

Sub OpenCSV_01()
Dim myPath
myPath = "C:\FILES_CSV\" '<< change path
Dim fName
fName = "abc.csv" '<< change file name
Dim mywb As Workbook, wb As Workbook
Set mywb = ThisWorkbook
Dim N As Long
Application.ScreenUpdating = False
Set wb = Workbooks.Add
wb.Sheets(1).Name = Split(fName, ".")(0)
For Each sh In wb.Sheets
If sh.Name <> Split(fName, ".")(0) Then sh.Delete
Next
Cells.NumberFormat = "@"
Open myPath & fName For Input As #1
N = 1
Do While Not EOF(1)
Input #1, Line$
Rows(N).Columns(1) = Line$
N = N + 1
Loop
Close #1
Columns(1).AutoFit

Application.DisplayAlerts = False
wb.SaveAs ThisWorkbook.Path & "\" & Split(fName, ".")(0) & ".xlsx", FileFormat:=xlOpenXMLWorkbook, CreateBackup:=False

Application.DisplayAlerts = True

mywb.Save
Application.ScreenUpdating = True
End Sub

In Excel 2007 I can right-click on a chart series (XYScatter) and click "Marker Line Style" on the left.  I can then set the marker's line Width, Compound Type, Dash type, Cap type, Join type, Arrow settings(!) and finally a Smooth line checkbox.

Obviously none of that gets recorded by the macro recorder....

Please can you tell me, how do I do all of that with VBA?

Is there anything else that's new from 2003 to 2007 that I should know about re. formatting data series?  All tips gratefully received.

Cheers
Rich

 

 

[UPDATE:

So I've done some more digging online and stumbled across a post by the great Andy Pope on another site.

With a bit of experiementing with what he put here, I've figured out that using:

    ActiveChart.SeriesCollection(1).Format.Line.Weight = 12

sets the weight (thickness) of the marker lines AND the series line (i.e. lines between each data point).

It appears you then have to reapply the desired formatting to the series line using:

    ActiveChart.SeriesCollection(1).Border.Weight = 2

Someone a Microsoft must be very proud of that day's work, forced to come in the day after the office Christmas party perhaps?

So, to set the mark line thickness, we use:

    Dim ser As Series
    Set ser = ActiveChart.SeriesCollection(1)
    ser.Format.Line.Weight = 12   'Sets both marker line and series line thickness
    ser.Border.Weight = 2               'Reapply series line thickness only

I also discovered that the Arrow settings are only applicable to the following marker styles: + x and *, or xlMarkerStylePlus, xlMarkerStyleX and xlMarkerStyleStar.

Still looking for how to set the other properties (Compound Type, Dash type, Cap type, Join type, Arrow settings and the Smooth line checkbox).  Any takers?]

andreas killer say:

Not all settings have a visible effect on the chart. And some settings to override other, as you've probably noticed.

 

The settings for the markers can be found in the points of the series, but this settings overwrides also some settings of the series line.

 

Andreas.

 

Sub Test()
  Dim S As Series
  Dim P As Point
  Set S = ActiveChart.SeriesCollection(1)
  For Each P In S.Points
    With P
      .MarkerStyle = xlMarkerStyleCircle
      .Format.Line.DashStyle = msoLineSysDot
      .Format.Line.Weight = 20
    End With
  Next
  S.Border.Weight = 2
End Sub
