Even if you get that code working (and it will be tricky if you don't know how to create and name Class Modules and declare them as WithEvents) It would only update when the slide show moves to a new slide - probably not what you need.

It will be a lot easier to put the code in Excel.

With your excel sheet open ALT f11

In 'Tools > References' add a reference to Microsoft PowerPoint

Open 'This Workbook' and paste in


Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
Dim pptApp As PowerPoint.Application
Dim osld As PowerPoint.Slide
Dim oshp As PowerPoint.Shape
Set pptApp = GetObject(Class:="PowerPoint.Application")
On Error Resume Next
Set osld = pptApp.ActivePresentation.Slides(63)
For Each oshp In osld.Shapes
oshp.LinkFormat.Update
Next oshp
End Sub

aravinda balakrishna say:

Hi,

 

You may assign range names to the cells to the source of the link and use the range names as the link instead of the cell address.

 

For example: name cell D4 MyRange, then the link will look like this:

=[Book1.xls]Sheet3!MyRange

Book1 refers to the workbook.

 

When rows or columns are inserted or deleted, the range name will travel with the cell and the link will not break.
