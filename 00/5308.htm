You can't be too paranoid when it comes to protecting your personal files.

The upgrade process is actually very simple and "user friendly".  You have to option of doing it either way you asked.  You can trigger the uninstall yourself before installing 2016, or you can let 2016 trigger the uninstall during the install process.

If you do the uninstall yourself do it from the Control Panel > Programs and Features. This is considered a "soft" uninstall. It will delete the program files and most of the registry entries.  It leaves behind files and registry entries that contain your personalizations / customizations.  That is files like Word's NORMAL.DOTM and the Outlook PST that contains your emails.  This soft uninstall was designed in to the applications from the very start, in the 1980's because Now as well as THEN uninstall was part of the normal general trouble shooting process (for problems with non-specific fixes).

There are version specific "Fix it" tools that do a more thorough uninstall. DO NOT use them for the upgrade!

How to disable Trust Center Message Bar (Office 2010 Professional - Microsoft Community

baneblade214 say:

How to disable Trust Center Message Bar (Office 2010 Professional +)

I work for an IT service desk at a university, and we have AD domain accounts for almost all users.  Because of this, settings made on one user generally don't roll over to every user. 

I need a way to disable the message bar in Office 2010 that will work for all users, not just a single user.  This message bar is popping up in Word whenever our users open a document from the network, but these particular users are children.  Children need as few distractions as possible, and it is an extra step for them that I'd rather them not have to do.

jay freedman say:
There is no way to do that through Word itself. However, there may be a way, by writing to the registry with a login script (probably written in PowerShell). It's definitely not "safe" in general, but might be acceptable given your circumstances.

The key you must modify is HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Word\Security\Trusted Locations and at least one of its subkeys.

Your script needs first to set the AllowNetworkLocations value to 1 (that's a REG_DWORD).

Then it has to create a subkey named LocationN where N is a number that isn't already in use (e.g., Location8). Within the subkey, you need to provide at least the value named Path, which is a REG_SZ in the UNC format  \\server\\share\\folder. Optional values are

AllowSubFolders   REG_DWORD    0x00000001    to trust subfolders of Path
Description            REG_SZ                                      to show a description in the Trusted Locations dialog
Date                        REG_SZ               current date & time

The script should first loop through all existing LocationN subkeys to make sure the desired path isn't already trusted. That loop can also determine the next usable number N.

This is all specific to each user (because it's in the HKCU hive), which is why it needs to be in a login script.

"Brad Sams at Petri speculates that the bugginess is a result of Microsoft's new development approach. The way Windows is developed has changed radically as a result of two things: laying off large numbers of dedicated software testers, and shipping regularly both to members of the Insider program using pre-releases, and to mainstream users with monthly updates and twice-yearly larger upgrades (of which the November release was the first).

This has moved the testing burden to developers and to Insiders, and Sams reports that this switch has proven difficult. There used to be a roughly one-to-one ratio of testers to developers, with developers left relatively unhindered by the need to test. Now they need to spend their own time on testing, while all the while having to produce software that's good enough to ship on shorter deadlines.

These changes are arguably necessary in the longer term, to give Microsoft the flexibility and responsiveness it needs, but the current state of Windows is vexing, to say the least. The software that ships needs to be reliably more reliable if Microsoft is to successfully persuade enterprises to make the switch. Right now, it's creating reasons for them to stick with Windows 7 or 8, or Windows 10 Long Term Servicing Branch. The latter avoids some of the bugs, by changing less quickly, but also avoids many of Windows 10's new features, too"
