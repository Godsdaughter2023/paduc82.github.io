ken sheridan say: If the values in the column are formatted as percentages the actual values will be 1 or a lesser fractional value, so you would need to divide each of the percentage values by 100.  The expressions are evaluated from top to bottom, so you don't need to define a range for the yellow values only the bottom of the range.  The expressions would thus be:

=1
>=0.85
<0.85

If the control is Null it will show in whatever is set as its BackColor property in its properties sheet.

If you want the entire row to change colour you would need to conditionally format all controls in the row on this basis, ensuring that the controls completely fill the entire detail section, with no unused 'white space'.  Alternatively add an unbound empty text box behind the bound controls and conditionally format this, ensuring that the bound controls' BackStyle property is Transaparant in each case.

You'll find an example of another way of colour coding like this as ColourCodes.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the colours are stored as plain bitmap images in a  ColourCodes table, which is joined to the referencing table on the column whose value determines the colour.  In my demo the join is a simple one of equality on a Boolean ColourCode column.  In your case you would need to specify the bottom and top of each range in two columns in the ColourCodes table, and join the tables on the value in the column in the referencing table falling within the range.  This can only be done in SQL, not design view.

My earlier post:

There is a workaround if you want to change the designated archive folder.  You will have to move the archived messages to the "new" archive folder, then delete the old archive folder (including from Deleted Items).  Outlook will then prompt you for the location of the archive folder next time you try to use archive.  You can then specify the "new" archive folder.
