Hi Melissa,

It looks like that we have a corrupted system file or user profile. What we can do is:

  1. Change the password of the problematic user profile.
  2. Run a System file check.
  3. Delete user profile from the registry.

However, we need a different admin account in performing these possible resolutions since you cannot log in using your user profile.

We can enable the built-in admin account by following the steps below:

  1. On the login screen, click on the power button on the lower right-hand side of the screen.
  2. Press and hold the Shift key and click on Restart.
  3. A command prompt will open at boot.
  4. Type net user administrator /active:yes and press Enter.
  5. Restart PC and you can login using the built-in admin account.
    • username: administrator
    • password: leave it blank

Once the built-in admin account is enabled, we can now perform the suggested resolutions.

  • Run a System file check. Once logged in using the built-in admin account, run the System File Checker (SFC) tool to repair missing or corrupted system files. To do this, click Start, type Command Prompt or cmd in the Search box, right-click Command Prompt, and then click Run as administrator. If you are prompted for an administrator password or for a confirmation, type the password, or click Allow. Restart your PC and try to log in using your account. If the issue is not resolved, proceed to the next suggestion.

  • Change the password of the problematic user profile.
    1. Log in using the built-in admin account.
    2. Type control panel on the Search box. Double-click the first result.
    3. Go to Manage user accounts > change the password for your account.
    4. Log out and log back in using your account. Use the new password that you just set.

If the suggestions did not work, the next step to do is to delete your user profile from the registry.

Important:  This section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs. For more information about how to back up and restore the registry, refer to the following Microsoft Knowledge Base article.

How to back up and restore the registry in Windows: https://support.microsoft.com/en-us/kb/322756

  • Delete user profile from the registry:
    1. Log in using the built-in admin account.
    2. Move the your corrupted user profile folder from C:users to c:Backup.
    3. Open the registry editor. In the search box, type in regedit and press Enter.
    4. Go to: HKEY_LOCAL_MACHINESOFTWAREMicrosoftWindows NTCurrentVersionProfileList.
    5. Locate the folder that has the ProfileImagePath that points to c:users. It usually ends in “.back”. Delete the entire folder.
    6. Restart your computer and log in with your user profile.

keith a. miller say:

Excellent. This is still your user folder.  It is displaying as 'Desktop' in Explorer because we haven't disabled the desktop.ini file (you should see it if you type:

>gci -force

To disable it, close all Explorer windows and type the following:

>ren desktop.ini desktop.sav -force

And then open Explorer and check the contents of C:\Users

The folder that was displaying as Desktop should now show as RobertEduard

If that's the case, go ahead & create a subfolder named Desktop and move the items that should be on your Desktop into that folder. Then we'll fix the Registry so Windows knows this new folder is your Desktop folder.

Keith

That's all a one-line command starting with 'gp'...

But the original got a <newline> Between 'User'  and 'Shell', so try to copy & paste this:

gp 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders'
