joeu2004 say: SavvyAl wrote:
Why do programmers call themselves engineers?  Do they have a degree in engineering?
Ken Blake wrote:
Some do and some don't.  And some have jobs for which their title is engineer.

This is a can of worms.  But I would also add:  not all software programmers are software engineers, notwithstanding job title.

(And yes, all software engineers are software programmers.  But not all software engineers are __application__ programmers.)

I would say that a software engineer is one who creates software "buildling blocks".  A software programmer is one who uses software "building blocks" to create end-user applications.

It's a fine-line distinction, I know; one which is blurred further in environments like UNIX that encourage the design of programs as "building blocks" to be used in pipelines.  Those are not the "building blocks" that I am talking about.

Some examples....

Software engineers develop operating systems, computer language translators ("compilers" and "interpreters"), and applications like Excel and Word.  Software engineers invented HTML, web servers that interpret HTML, and web browsers that communicate with web servers.

A software engineer might invent a new sorting algorithm.  A software programmer uses the new sorting algorithm in the development of an application.

A software engineer might develop a library of math functions.  A software programmer uses the math library to develop an application.
