ken sheridan say: It's a little surprising that the code worked in any circumstances as the Click event procedure does not have a Cancel argument.  The error is understandable therefore, as is the solution of declaring the variable   I can only assume that the class module of the form in which it is working is missing the Option Explicit line from its declarations area, removing the necessity to declare variables.

More to the point, however, is that it is not necessary to include the Cancel variable in the procedure at all as it does nothing.  You appear to be not actually deleting  a record here, but assigning a Null to a FIL_NME_FST control to remove one item of data only, so if the user responds No to the prompt a Null will not be assigned to the control, so in effect the 'deletion' will be cancelled without having do anything else.

So just remove any reference to the Cancel variable from the procedure and it should work as intended.

For an example of where you would set a return value for a Cancel argument, the following is some code from a form's module which deletes the current record at a button click.  Note how the return value of the form's BeforeDelConfirm is set to True if the deletion is not confirmed.

The button's event procedure firstly undoes any current edits if the record is unsaved.  It then attempts to delete the record, but this might not be possible if the record is a new one which has never been saved, so the error handling detects and ignores this.

One thing about this code which might not be apparent at first sight is that the BeforeDelConfirm event procedure executes immediately after the line RunCommand acCmdDeleteRecord in the button's event, and the code execution in the button's event procedure then resumes, so the DELETE_CANCELLED error is a result of the user not having confirmed the deletion in the BeforeDelConfirm event procedure, not the user's response to the earlier prompt in the button's event procedure.

Private Sub cmdDelete_Click()

    Const DELETE_CANCELLED = 2501
    Const CANT_DELETE = 2046
    Const MESSAGETEXT = "Are you sure you wish to delete the current record?"

    If Me.Dirty Then
        If MsgBox(MESSAGETEXT, vbYesNo + vbQuestion, "Confirm") = vbYes Then
            Me.Undo
        End If
    End If
    
    On Error Resume Next
    RunCommand acCmdDeleteRecord
    Select Case Err.Number
        Case 0
        ' no error
        Case DELETE_CANCELLED
        ' anticipated error, so ignore
        Case CANT_DELETE
        ' anticipated error, so ignore
        Case Else
        ' unknown error, so inform user
        MsgBox Err.Description & " (" & Err.Number & ")", vbExclamation, "Error"
    End Select
    
End Sub


Private Sub Form_BeforeDelConfirm(Cancel As Integer, Response As Integer)

    Const MESSAGETEXT = "Are you sure you wish to delete the current record?"
    
    If MsgBox(MESSAGETEXT, vbYesNo + vbQuestion, "Confirm") = vbNo Then
        Cancel = True
    End If
    
    Response = acDataErrContinue
    
End Sub

joeu2004 say: Haseeb wrote:
Enter the years in a range. eg: J2 to down, 2011, 2012, 2013 etc... then K2, copy down.
=COUNTIF(H:H,">="&DATE(J2,1,1))-COUNTIF(H:H,">"&DATE(J2,12,31))

See my comments (click here) in a discussion that reveals the dangers of using COUNTIF with indefinite ranges (H:H) unnecessarily.

Since Gareth has already indicated the willingness to restrict the range to H5:H200, I think Haseeb's suggestion would be better implemented as follows [1].

XL2003 and before:
=COUNTIF($H$5:$H$200,">="&DATE(J2,1,1))-COUNTIF($H$5:$H$200,">"&DATE(J2,12,31))

XL2007 and later:
=COUNTIFS($H$5:H$200,">="&DATE(J2,1,1),$H$5:H$200,">"&DATE(J2,12,31))

ok.

your confusion likely stems from looking at:
1. PC Suite
2. Nokia Multimedia Transfer
3. iSync plug-ins

none of these work with your phone - with one exception, isync can work if you're willing to get your hands dirty. note, however, that isync support does nothing regarding your goal, i just thought it worth mentioning since you'll likely want to be able to do this as well. the info for isync can be found here:
/discussions/board/message?board.id=pcsuite&message.id=16799


now, getting music onto your phone will require one of 3 methods:

0. (i added this after i posted everything else) i see that your phone has only 4MB of internal storage and that it supports microsd memory cards. you are certainly using the microsd card for music (4MB fits only 1 or 2 songs max). you can take the microsd card out, put music on it, then put the card back into the phone. this requires a microsd card reader, which will automatically act as your phone would if it were in "Mass Storage" mode - all other advice related to "Mass Storage" mode is valid for a card reader as well.
1. connecting to the phone in 'Mass Storage' mode and copy music to the proper folder in the phone.
2. if you have bluetooth on your phone as well as your mac, then you could send songs via bluetooth in an identical manner using Bluetooth File Exchange to 'Browse' the phone
3. you could use bluetooth to send individual mp3's to the phone by selecting them in Finder and clicking Finder->Services->Send_To_Bluetooth_Device. fiels sent this way appear on my s60 device just like incoming text messages - this might be different with your s40 device (the 6085 runs the s40 user interface)

you requested step-by-step direction for usb, so i'll try, but i dont have an s40 device so i dont know precisely what the screens will look like. ill start by stating that you don NOT NEED TO INSTALL ANY SOFTWARE, though after accomplishing this you might want to. after succeeding in this process, google for 'SyncTunes', an app i used to use to copy music to my unsupported sony ericsson phone a while back.



1. find the USB settings area of your phone - on my s60 device i find it here: Menu->Tools->Settings->Connectivity->USB

2. look through the usb settings, try to find something called USB Mode, or an analogous phrase. ensure that the mode is set to "Mass Storage", or (again) an analogous phrase (i'll stop bothering with that statement, but attach it to anything directly quoted text i talk about).

3. there may be a field titled, "Ask On Connection" - on my phone that simply prompts me to pick the USB Mode right after connecting the phone to my mac.

4. set the usb mode? good - connect the phone to your mac via usb.

--- NOTE: if you're using a card reader, then start here ---

5. your phone's memory will appear just a a USB Thumbdrive would appear - it's on your desktop as a Drive icon. open it up.

6. look in the drive contents - find a folder called 'Sounds' or 'Music'. look through the folder to see if it contains the same files as you already have available to play on your phone. (dont have any? try recording some audio if your phone supports it, or just use trial and error with one mp3 at a time)

7. put your music here. my n95 expects music to be distributed in subfolders, but i doubt it is required. **bleep** some mp3s into this location and make sure you UNMOUNT your phone before unplugging it (CTRL-CLICK the drive icon and choose unmount, then wait for the drive icon to disappear)

8. open the music player on your phone. you may need to refresh the music 'library' or 'database' on your phone (click on options->refresh, or something similar)

9. rinse, repeat, rejoice

goodluck!
