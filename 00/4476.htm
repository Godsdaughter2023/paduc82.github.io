I have a company wide instructional document however within that document are pages that have directives specific to location.  I have bookmarked each  of the local directives within the document with its location name. I am attempting to create a print macro where the end user can print the document with only their location specific directives.  How do I write the visual basic code so the macro will enable my user to select their own respective site and not the whole document?

jay freedman say:

The core of the macro you want is fairly simple: Given a particular choice of site, the code sets the Hidden property to True for the bookmarked ranges of all the other directives, and False for only the the directives for the chosen site. The document is then printed, and finally it's closed without saving changes.

The part that's not so straightforward is providing a way for the user to select a site. What I'd suggest is a "userform" (essentially a custom dialog box) containing a list box and a couple of command buttons. The list box will contain the list of site names. One button is the OK button that accepts the selected item in the list box, and then performs the rest of the work. The other button is the Cancel button that lets the user dismiss the userform without doing anything.

A tutorial on creating a userform is at http://gregmaxey.mvps.org/word_tip_pages/create_employ_userform.html, although much of the example material there won't be applicable to the one you need. If you want more hand-holding after looking at that article, post back here.

There are other ways to get the user's choice of the site, but if it involves something in the body of the document (e.g., a list content control) then the macro might have to hide that object as well as the directives.

First, does the query produce the duplicate rows? If so the problem is the query. A subreport shouldn't be needed.

ken sheridan say: To print a report with data from a single cheque you need to restrict the report on both the request number and the cheque number, or just the cheque number if each cheque relates to one request only.  If you restrict the report solely on the request number then it will print rows for all cheques related to that request number.  If the report is not grouped on the request number then all the request data will be repeated for each related cheque.

I would suggest that you group the report on the request number and return the data from the requests table in the group header.  Return the data from the cheques table in the detail section.  By default the report will print each request once, with multiple lines in each instance of the detail section, one per related cheque.

By restricting the report on the request number only, it will print the request data once, and the details of all cheques related to that request.

By restricting the report on the request number and the cheque number it will print one request and the data for the one cheque only.

You can restrict the report either by parameters in its query referencing controls on the form, or by means of the WhereCondition argument of the OpenReport method.  Whichever you use, when printing the report from the form, be sure that before doing so both the parent form's and the subform's current record are saved.  Otherwise the unsaved data won't be returned in the report.
