  Most likely your cooling fan intake/output is clogged with caked up dust,lint, pet hair ect. Note the air intake is usually located on the bottom of the laptop which usually sits on your knee, blocking the air intake. Also some cpus run hot normally, such as the i7.

Most new machines come with a recovery partition allowing you to go back to the time of purchase, you will have to check your user guides (or the manufacturer's Web Site) for instructions.  Choosing that path requires you to save all valuable data first. 

How is your machine when run from Safe Mode?  If it seems OK can you isolate any cause via a Clean Boot?

How to start Windows 8, or 10, in Safe Mode
http://www.bleepingcomputer.com/tutorials/start-windows-8-in-safe-mode/

If the error is bothering you then you have to use the Clean Boot procedure and isolate the cause by stopping half the Startup Programs and non-Microsoft Services and then checking if the fault is still there.  If it is still there then you stop half of the remaining non-Microsoft Services and Startup Programs until you isolate the cause.  If it is not there then you put the ticks back in half of the stopped non-Microsoft Services and Startup Programs.  It is a fiddley process of elimination until you find the cause there is no way around it.

How to perform a clean boot in Windows
http://support.microsoft.com/kb/929135/en-au

How to perform a clean boot to troubleshoot a problem in Windows Vista, Windows 7, Windows 8, Windows 10.

wunders say:

A quick explanation on how EFS works...  When you create your first EFS file or folder, Windows creates a "key" that is used to encrypt and decrypt all EFS files and folders from that point on.  It would not be very good security to just leave this key sitting on your drive, so your key itself is encrypted to your Secure ID (SID), your password, and only Microsoft knows what else.  If you lose this key, you will never be able to get to your data (otherwise, it wouldn't be secure).  This encrypted "key" is stored as a "certificate" in your certificate store which is located as part of your user profile, usually on the C: drive.  If you use EFS, it is extremely important that you back up this certificate -- (although Microsoft does not tell you this when you start using EFS).  They do, however, instruct you how to do it here:

    "Best Practices for the Encrypting File System"

    < http://support.microsoft.com/kb/223316/en-us >

(Check out the paragraph "Why you must back up your certificates").

 

If you would have used the Elcomsoft tool before your reformatted your C: drive (I'm assuming a full format here), then you stood a good chance of recovering your key.  When you reformatted your C: drive (and it wasn't a "quick" format), then all of your this reason.

 

My humble suggestion going forward:  If you need encryption, consider the [top-notch] freeware "Truecrypt" instead of EFS.  With it, you have control of the key (as opposed to letting the operating system manage the key for you -- and lose it when it crashes).  I have used it since Windows 2000 days (and before, with its predecessors) and have never lost any data despite crashing a few operating systems.  Since you manage your own key, it is not as transparent as EFS, but I consider that a plus.

    Truecrypt:  < http://www.truecrypt.org/ >

 

Sorry for your loss,

   JW
