bob jones aka: cybertaz say:
You won't find much reference to them unless you dig deeply into the archives stored in the basement :-)

Dialog Sheets are a workspace for creating Forms -- or Dialog windows -- that serve as VBA-driven User Interface tools. The controls that can be added to them facilitate/restrict/focus data entry & if properly used insure that entered data goes to the right places. The Forms can also include labels & Text Boxes that provide clarification or instruction specific to the task so that temporal information doesn't have to occupy or obstruct cells on a permanent basis.

Introduced in Excel 5.0 they're now virtually obsolete, having been replaced in the Windows version of Excel [starting with Office 97] by UserForms. They've been continued in the Mac version and [AFAIK] Forms designed on Dialog Sheets are still supported by the current PC versions.

When you save to your hard drive, are you saving to your user Documents folder (Users/YourUserName/Documents)? That is normally where the OneDrive folder is located. 

When you save to the OneDrive folder, that folder and its content exist on your hard disk. It syncs with the cloud version of your files, so by saving to the OneDrive folder you've already accomplished your goal of having a copy on your drive and a copy in the cloud.

This goes beyond what you asked, but here's my setup: I keep my data on a separate RAID drive. In both Windows and OS X, I've moved my user folders from the operating system disk to this RAID drive. It's RAID 1, so both disks are mirrored. If one disk goes down, I don't lose any data.

Then I have a NAS network share which is also on a mirrored RAID drive. As soon as I complete a job, it's archived to both the local and the networked NAS drives. This way I always have at least 2 copies of each file, plus another 2 when it's archived. I haven't lost a byte of data in 15 years.

I don't put much on OneDrive except what I am shared with others, but I also run my own WebDAV server, so I have my own private cloud. So far, not a single security issue with it.

slance310 say:

Hi,

You may want to make sure your BIOS is not set for RAID... a lot of times when the install is looking for drivers, that is the reason.

Renee is correct in that "seconds" is too fast... even with a SSD...

I have 16GB memory and a 512GB Crucial SSD... it took 4 minutes...

You may have the wrong type of USB Flash for the type of installation...

Check your BIOS setting for it's UEFI\Legacy setting...

If it is set for UEFI, your Flash drive needs to be formatted as FAT32, and the Microsoft tool does not do this... You need a utility like RUFUS to create a bootable ISO that will work with UEFI...

If your Bios is set for Legacy, the USB created with the Microsoft tool should work...

Make sure your Boot Order in BIOS is set to allow your SSD to be the first boot device...

Or 2nd... after the USB Flash... You can change things once everything is up and running...

BTW... make sure your RAM is in the proper slot(s) and properly seated..

doug robbins - mvp office apps & services (word) say:

If you insert a bookmark (Figure1, Figure2, Figure3, etc,) that encloses each of the figures and then where you want the list of figures to appear, you insert a table into the cells of which you insert smaller versions of each of the figures and then under each, you insert GoToButton for which you select the appropriate bookmark and use the Figure # as the text to be displayed, then, when you click on that text in the cell of the table, the selection will move to the relevant bookmark\Figure in the body of the document.

You insert the GoToButton via the Insert tab>Quick Parts>Fields dialog.
