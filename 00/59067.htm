jtkerry say:

I found my own answer...

The cause:

Windows 10 has the "Skype App" pre-installed.  When you go to download the Desktop version, the web page has a button for Windows 10 (a variant of the native Windows 10 App) - this is not the full Desktop version - I thought I would point that out for starters. When you install that version, it merges with the App already installed and causes a conflict, which throws up the error noted above.  

The solution:

Go to the Windows 10 Settings (the gear icon in the Start button, not the old Control Panel). Choose from the "Home" window - "Apps".   This will change to "Apps & features" - Windows developers have become consistent with the inconsistencies - find Skype click on it and uninstall.  This will remove both versions.  Go back to the Skype for Desktop web page, in the button to download "Get Skype for Windows 10" is a drop down arrow, click on it and choose "Get Skype for Windows" at the bottom of the list.  The installer will download and this installs the more robust consumer version of Skype for Desktop and not a Windows App.  And, Whala! the conflict is gone, and the 2 Skype accounts (one in the Desktop version and one in the Skype for Business version) can run together without conflict.

Specialist IT guy here.

Someone from my company had the same problem when they upgraded to windows 8.1 from windows 8 with their Toshiba Satellite L850 where the mouse and keyboard completely stopped working.

Go to the toshiba website and download the drivers for your laptop that there are for windows 8.1 (at this time, there was only a wifi one there, no other drivers under windows 8.1), the rest of the drivers, download the windows 8 ones. Just reinstall all the drivers and it should be ok.
