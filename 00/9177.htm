jay freedman say:

I can reproduce the problem in Word 2016 from MS Office 365 ProPlus, Version 1701 Build 7766.2076 (First Release for Deferred Channel), but it does not occur in Word 2016 from MS Office 365 ProPlus, Version 1704 Build 8067.2032 (First Release for Current Channel).

The Writing Style in the Options > Proofing dialog is represented in the registry by the WritingStyle value in the key HKEY_CURRENT_USER\SOFTWARE\Microsoft\Office\16.0\Word\Options. When it's working properly, the registry value is read only when Word starts, and the current value is written to the registry when Word closes. In the version where it doesn't work properly, it appears that the registry value is neither read nor written by the running program. I'll report this to the developers to be sure they're aware of it.

Until the program is patched, you can force the program to default to the Grammar setting by installing this little macro in your Normal.dotm template (see http://www.gmayor.com/installing_macro.htm if needed):

Sub AutoExec()
  Languages(1033).DefaultWritingStyle = "Grammar"

End Sub

The number 1033 represents English (US). If your default language is another one, look up the corresponding LocaleID value at http://msdn.microsoft.com/en-us/library/ms912047(WinEmbedded.10).aspx.

Sway

jason (office engineer) say:

You have two options: 1) you could upload an image that has the symbol you want or 2) you could find a text character that represents what you want and put that in your Sway.  Your best bet is probably a caption on the image so you should look for arrow characters on the web and then put the one you want into your caption.  With that said, Sway layouts are dependent upon the device and window size that they are viewed in and things can get positioned differently sometimes.  It's possible that whatever symbol you use could end up not pointing in the direction you expect.

Hope this helps!
