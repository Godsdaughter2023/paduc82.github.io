how do i retain bold formatting when concatenating in mac 2011 - Microsoft Community

ameliaray say:

how do i retain bold formatting when concatenating in mac 2011

I am trying to concatenate three columns of text: A1, B1, and C1. A1 needs to remain formatted bold, but concatenating removes the formatting. Is there a way to tell it to keep the bold font within the concatenation formula? Thanks!
j.e. mcgimpsey say: As Bob wrote, it can't be done with XL functions (including user defined functions), because functions by design can only return values to their calling cells. They can't change formatting.

You could use an Event Macro that would simulate a concatenation function when the worksheet was calculated. Here's a really quick and dirty one that concatenates A4:C4 and puts the results in J4. Change to suit. Put the macro in the worksheet code module:

    Private Sub Worksheet_Calculate()
        Const sDelimiter As String = " "
        Dim rTarget As Range
        Dim sTemp As String
        Dim i As Long
        Dim nStart As Long
        Dim nLen As Long
        
        Set rTarget = Range("J4")
        With Me.Range("A4:C4")
            For i = 1 To .Count
                sTemp = sTemp & sDelimiter & .Item(i).Text
            Next i
            rTarget.Value = Mid(sTemp, Len(sDelimiter) + 1)
            nStart = 1
            For i = 1 To .Count
                With .Item(i)
                    nLen = Len(.Text)
                    rTarget.Characters(nStart, nLen).Font.Bold = .Font.Bold
                    nStart = nStart + Len(sDelimiter) + nLen
                End With
            Next i
        End With
    End Sub

Note that if you apply Bold formatting to a cell, the target cell formatting won't change until a calculation is done.

daniel pineault say:

Your Query is doing exactly what you've asked of it by using the Like "*" ... criteria. 2,12 and 20 all are like *2*.

If you only want to display the selected cbo value then you need to remove the like part. Try:

Query Field Criteria = Nz([forms]![frmreports]![cboItem],"*")

No, the memory manager is not accessible.

Why is the memory not released... I can only guess... in old OS the heap fragmentation is a problem, which is similar to disk fragmentation. So it might be possible that the memory manager "thinks" it is better not to release the memory to avoid the fragmentation.

I'm not familiar with pivot tables, but didn't have pivot tables a pivot cache? Is it possible to delete the pivot caches (set it to Nothing) of the temporary pivot tables?

A workaround is to use a second file (or a second instance of Excel) for the temporary calculation. I'm not sure if the memory is released when you close the second file, but I'm 100% sure when you use a second instance.

Andreas.

Sub Test()
  Dim TempApp As Excel.Application
  Dim TempWb As Excel.Workbook
  Dim Data
 
  'Create a second instance
  Set TempApp = CreateObject("Excel.Application")
  Set TempWb = TempApp.Workbooks.Add
 
  'Copy the data from this sheet to the temp app
  Data = ActiveSheet.UsedRange.Value
  TempWb.ActiveSheet.Range("A1").Resize(UBound(Data), UBound(Data, 2)) = Data
 
  'etc.
 
  'Done
  TempWb.Close False
  TempApp.Quit
End Sub
