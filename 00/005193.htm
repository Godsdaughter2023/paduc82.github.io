frank fei msft say:

Hi Denise,

As Yoga said, a user needs at least contribute permission to approve an item, to be more specific, the approver needs contribute permission in the workflow associated task list to approve the item.

So for your situation, the workaround is setting unique permission for the approver in the workflow associated task list. Two steps here:

1) Find the associated task list of the workflow. Go to SharePoint Designer, locate the workflow, then you can find the associated task list in Settings box.

2) Navigate to the task list in SharePoint, Click List tab > List Settings > Permissions for the list, click Stop Inheriting Permissions so you will be able to assign unique permission to that approver.

After that, the approver should be able to approve/reject item so that the workflow can proceed.

Thanks,
Frank
