OTOH, if you look up RFC 5233 (http://tools.ietf.org/html/rfc5233) you'll learn that the "somename" part of an email address can absolutely be of the form that Spamgourmet uses.

 

The whole point of using Spamgourmet's "send using one of your disposable addresses" is actually a two-part story:

 

- Spamgourmet's addresses are useful because they have an automatic expiration based on a counter, which is decremented if and only if the sender is a "trusted" sender.
