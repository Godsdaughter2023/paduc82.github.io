From our logs we can see that the crashes/hangs are quite random and not predictable/repeatable in nature. The same lines of code can work perfectly in many cases whilst sometimes a crash/hang happens. We haven't been able to pinpoint any particular line of code that causes the crashes/hangs but we tend to see a pattern where the following operations are involved, or at least preceeding:

- copying calendars from one project file to another, ie Application.OrganizerMoveItem(...)

- reading resources in a project, ie  foreach (MSProject.Resource r in thisProject.Resources)

- editing tables in a project, Application.TableEdit(...)

- opening a project file or checking if the requested file exists.

We often encounter the exception "Exception from HRESULT (RPC_E_SERVERFAULT)", but have been able to work around this by running the procedure that caused the error again. The second time often without any exception.

Clews anyone?

doug robbins - mvp office apps & services (word) say:

If you really want to insert pictures with text wrapping, I would suggest that you go to File>Options>Advanced and in the Cut, copy and paste section of the dialog, from the "Insert/paste pcitures as:" dropdown, you select the type of text wrapping that you want to be used.

When you have inserted the picture, you will then need to select it so that you can drag it into the required position.

I would suggest that you regularly save your document using a revolving file naming system as from my experience of repairing documents that have become corrupted due to mis-matched xml tag errors, those errors have very often been associated with images having been inserted into a document with text wrapping, rather than in-line with text.

You should consider whether you can achieve the required layout by the use of a table with the image, inserted in-line with text, in one cell an text in another as that will generally result in a document that is less prone to corruption.
