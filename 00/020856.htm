one of many dans say:

SharePoint Bug: Publishing features makes List un-deletable.

I was experimenting with some Site Collection features, and now I'm stuck with an un-deletable List.

When I enabled "SharePoint Server Publishing Infrastructure", it created many Lists and Libraries. When I disabled it, I was able to delete most of those Lists and Libraries. However, some of them are stuck (in particular, "Workflow Tasks 1").

The stuck List is empty. It isn't used by any features (because the feature is disabled now). I can't delete it online, or in SharePoint Designer 2013, it complains: "The folder selected for deletion contains one or more lists, document libraries, or galleries that your Web site requires in order to functions correctly. Therefore, you cannot delete the folder."

Is there a way to get around this SharePoint bug so I can delete the List? I already have a normal "Workflow Tasks", and it's confusing to users to have an additional "Workflow Tasks 1".

Thanks for any help!

tina zhou msft say:

Hi Dan,

It is not workable to delete your Workflow Tasks1,  FormServer Templates, Long Running Operation Status , PublishedLinks, Sharing Links, Translation Packages and Translation Status folders either in your SharePoint Online site collection site contents page User Interface or in SharePoint Designer.

For Workflow Tasks1, if you haven’t created Workflow Tasks, when you activate SharePoint Server Publishing Infrastructure or activate and then deactivate it, Workflow Tasks folder will generate and display in your SharePoint Online site collection site contents page User Interface and SharePoint Designer All Files tab page. Since you have created Workflow Tasks, when you activated SharePoint Server Publishing Infrastructure, SharePoint Online adds suffix 1 to Workflow Tasks automatically. From the screenshot in my first reply, we can see there is no Remove option and it is designed to be unremovable. And as the error message says, the folder contains one or more lists, document libraries, or galleries that you Web site requires in order to function correctly and therefore, you cannot delete it (You may click into folder to check the affected files. ).

For Sharing Links, this folder is created automatically once you click Get a link on SharePoint Online library or list. It is not related to SharePoint Server Publishing Infrastructure and undeletable.

For FormServer Templates, Long Running Operation Status , PublishedLinks, Translation Packages and Translation Status, when you activate SharePoint Server Publishing Infrastructure, these folders generate and don’t display in your SharePoint Online site collection site  contents page User Interface, and are undeletable in your SharePoint Designer All Files tab page; when you activate SharePoint Server Publishing Infrastructure and then deactivate it, these folders still don’t display in your SharePoint Online site collection site  contents page User Interface and are undeletable in SharePoint Designer. As the error message says, these folders contain one or more lists, document libraries, or galleries that you Web site requires in order to function correctly respectively and therefore, you cannot delete them (You may click into each folder to check the affected files).

Thanks for your understanding.

Tina
