The simplest formula is just,

=A1

... and that will change both row and column if copied to another location. This is known as relative cell addressing. If you want to lock either the row or column or both, you need to use the absolute anchor symbol; e.g. $. So if you want to lock the row then,

=A$1

If you want to lock the column then,

=$A1

If you want to lock both the row and column then,

=$A$1

This is know as absolute cell addressing.

The names you've chosen for the worksheets makes things confusing. I've having a hard time believing that your worksheets are intentionally named Jan '13, Feb '13, etc. with oddballs like March '13 thrown in but I suppose it could be true. Life becomes easier when you do not intentionally complicate it so I would recommend that you lose the single quotes from the worksheet names and homogenize the month abbreviations. Put them right beside each other with no intervening worksheets and use,

=SUM('Jan 13:Dec 13'!H$65)

That will keep the row number as you copy to the right or left. The verbose method should be discarding the SUM() function or using commas rather than plus signs. Use SUM() or use plus signs; there is no need to use both.

=SUM('Jan 13'!H$65,'Feb 13'!H$65,'Mar 13'!H$65,'Apr 13'!H$65,'May 13'!H$65,'Jun 13'!H$65,'Jul 13'!H$65,'Aug 13'!H$65,'Sep 13'!H$65,'Oct 13'!H$65,'Nov 13'!H$65,'Dec 13'!H$65)

='Jan 13'!H$65+'Feb 13'!H$65+'Mar 13'!H$65+'Apr 13'!H$65+'May 13'!H$65+'Jun 13'!H$65+'Jul 13'!H$65+'Aug 13'!H$65+'Sep 13'!H$65+'Oct 13'!H$65+'Nov 13'!H$65+'Dec 13'!H$65

HOWEVER, I'm getting the distinct feeling that you want the column to change when you are copying down a row. This is not normal cell address behavior, but can be accomplished with OFFSET(), ADDRESS() or maybe TRANSPOSE(). INDIRECT() may be necessary to construct an address. Here are a couple of examples,

The OFFSET() function can be used to offset a cell address a number of rows. With this function, we can gauge the number of rows from the original position and use that to change the number of columns. So 'Jan 13'!H65 becomes,

OFFSET('Jan 13'!$G$64,1,ROWS($1:1))

... which is G64 offset 1 row and 1 column. As it is copied down to subsequent rows, each new row
offsets to a new column. Example:

=SUM(OFFSET('Jan 13'!$G$64,1,ROWS($1:1)), OFFSET('Feb 13'!$G$64,1,ROWS($1:1)), [...], OFFSET('Dec 13'!$G$64,1,ROWS($1:1)))

I'll let you fill in the worksheets from Mar 13 to Nov 13. Here's a pseudo-array formula using SUMPRODUCT() that cleans up the necessity of 12 separate entries,

=SUMPRODUCT(SUM(INDIRECT(ADDRESS(65,ROWS($1:8),4,1,TEXT(DATE(2013,ROW($1:$12),1),"mmm yy")))))
