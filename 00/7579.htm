skydrive word web app changing the format of my document - Microsoft Community

sasjames say:

skydrive word web app changing the format of my document

I uploaded several word documents to my Sky Drive.  When I open these documents using the word web app, the formatting has changed and has caused me hours of rework...

Why is the web app not able to open the document the way it was created in MS Word 14.2.5
john mcghie say:
This is a tough one to explain.

Basically, a word-processor document is designed to change its format depending on various things, the main one being the printer connected.  If the size of the paper in the printer, the width of the page or the size of the margins changes, a word-processor "flows" the text to accomodate the new dimensions.

In a Microsoft Word document, various things you may include in the document are anchored to the closest paragraph.  If the beginning of the paragraph they are anchored to moves, so do they.  Which moves the text surrounding them.  Which causes a flow-on effect throughout the document.

The SkyDrive app is displaying the document the way you coded it to display.  If you want it to display the same in both places, you need to intervene in the formatting to ensure that it can.

That means formatting the document so that it does not matter if things move around.  You can't stop things moving around, that's the way word-processors work.  But you can format your document so that it still looks good when they do.

First you need to do all of your formatting with Styles.  Mainly paragraph styles.  You can look these up in the help if you are not already using them.

Then you need to learn about the "Flow control" properties of styles.  These are the paragraph properties "Keep lines together", "Keep with Next" and "Page Break Before".

Keep Lines Together prevents Word splitting a paragraph at the bottom of a page.  "Keep with Next" causes Word to maintain a paragraph on the same page as the one following it.

Try to perform all your formatting in this manner.  Avoid using page breaks, anywhere.  The reason for that is that when things move around, fixed page breaks will end up in the wrong place.

Instead of using fixed page breaks to say where you "do" want a page break, use the style properties to tell Word where you "don't".  Then leave it up to Word to split the pages the way it is designed to do.  If you let Word do the work for you, you will get a good-looking document that will remain good-looking wherever you display it.

If you try to control the way a document paginates manually, you will do a lot of work, which will all have to be re-done each time the document goes somewhere else.

Hope this helps
