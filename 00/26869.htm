numeric keypad enter adds page break in word 2007

jay freedman say:
I would suspect an electrical fault in your keyboard that makes the numeric Enter key act like the Ctrl key was also pressed at the same time.

Word treats the Ctrl+Enter combination as the InsertPageBreak command, while Notepad and Wordpad (and browsers and most other software) treat Ctrl+Enter as the same as just Enter. So of the Ctrl key was somehow included with the numeric Enter key, you would see the result only in Word.

It's extremely unlikely that a Word macro is causing this behavior, because VBA has no "keycode" value for the numeric Enter key separate from that of the alpha Enter key. If I wanted to create such a macro, I'd have to use Windows API calls; it seems a lot of trouble to go to for a prank or something like that.

So, for a simple thing to try, borrow a keyboard from another computer and see if it behaves normally. If so, you'll find that replacement keyboards are very inexpensive.

jeeped say:

In Word 2007, whenever I hit the numeric key pad Enter key, the result is a page break added to the Word document.


The numpad Enter↵ is acting as if it is being pressed in combination with Ctrl, e.g.Enter↵ The Ctrl+Enter↵ keystroke combination is assigned to produce a manual page break in Word by default. The same thing is likely happening within Notepad or Wordpad, but you are not seeing any effect as that keystroke combination is not assigned within those programs.
Normally, I would advise you to examine your keyboard for a stuck Ctrl key, but you've stated that the standard Enter↵ key does not produce the page break. Word keystroke assignments do not normally discern between the standard and numpad Enter↵ keys although they send two distinct key codes.Look to any keyboard configuration software or possibly keyboard encryption software for an option to produce this behavior. It is possible that your numpad Enter↵ has been remapped with a utility program like KeyTweak.

FWIW, this subject has been previously discussed (but never resolved) in this thread.

With a number in A2, you can use this formula:

=IF(AND(MOD(10*A2,1)<=0.5,ISEVEN(10*A2)),ROUNDDOWN(A2,1),ROUND(A2,1))

Function MyRound(num As Double, digits As Long) As Double
    MyRound = Round(num, digits)

Use like this:

=MyRound(A2)

stefan blom say:

Word does not allow you to anchor text boxes (or any other objects) to a particular page. The best you can do is anchor the box to a paragraph somewhere in the middle of the page so that it is unlikely to move to the next page, even if you edit the surrounding text slightly (a text box will always display on the same page as its anchor).

To see anchors in Word, select "Object anchors" at File tab | Options | Display category. The anchor of an object will be visible when you select the object.

Note that if you make use of Word's built-in caption feature, you may want to insert the object "In line with text" and the caption in an ordinary text paragraph, and then wrap a frame around both of them. (The Insert Horizontal Frame command can be added to the Quick Access Toolbar.)

suzanne s. barnhill say:
To add to what Stefan has said, you can specify the position of a text box relative to the page, but it does have to be anchored to a paragraph on that page. If that paragraph moves to another page, the text box will move with it but will retain its position relative to the page.
