brian tillman [outlook mvp 2007-2019] say:

This is very common.  The only way to prevent it is to disable Internet access in the Reading Pane.  There are several ways.  One is in Outlook itself.  Click File>Options>Trust Center>Trust Center Settings>Automatic Download.  Check the box labeled "Don't download pictures automatically...".  Even if you can't see the image, there is still one embedded.  It's called a "web bug" or "web beacon"; a one pixel image too small to see.

Another way is to add the domain to the Restricted zone of your Internet.  Open Internet Explorer (if that's what you use), click Tools (on the menu if enabled or the gear at the top right)>Internet Options>Security>Restricted sites>Sites.  Add findall-newbalancealerts.rocks to the list.

A third is to change your hosts. file.  Open C:\WIndows\System32\drivers\etc.  Copy the file named "hosts" to, say, My Documents.  Open it in Notepad.  At the bottom, add a line that reads

127.0.0.1 image1.findall-newbalancealerts.rocks

then close the file.  Copy it back to the original folder.  You'll be prompted by UAC to grant admin permissions, so do that.  Instead of "127.0.0.1", you can also use "0.0.0.0".  There are web sites available that maintain hosts files containing lists of known tracking sites that you can load.  See, for example, http://winhelp2002.mvps.org/hosts.htm.  That page contains links to instructions for other ways to enable updating the hosts file as well.

rick rothstein, mvp - excel say:

I've modified Shane's code to do what you asked and made a couple of other small modifications along the way. For example, I changed the code to use the empty column after the last column containing either data or formula so that Excel doesn't have to shift everything to the right to accomodate the new inserted column only to move everything to the left when it is deleted; and I also change the square-bracket notation to normal Range notation (sorry Shane, but I hate the square bracket notation). Here is the modified routine (if this answers your question, please mark both it AND Shane's responses as the Answer)...

Sub InsertRows()
    Dim Bot As Long, MTcol As Long
    Bot = Cells.Find(What:="*", SearchOrder:=xlRows, _
          SearchDirection:=xlPrevious, LookIn:=xlFormulas).Row
    MTcol = Cells.Find(What:="*", SearchOrder:=xlByColumns, _
            SearchDirection:=xlPrevious, LookIn:=xlFormulas).Column + 1
    With Range(Cells(1, MTcol), Cells(Bot, MTcol))
      .Formula = "=IF(ROW()=19,TRUE,IF(AND(ROW()>19,MOD(ROW()-19,17)=0),TRUE,1))"
      .SpecialCells(xlCellTypeFormulas, 4).EntireRow.Insert
      .EntireColumn.Delete
    End With
End Sub

shane devenshire say:

Hi,

Assuming your data only goes down about 3000 rows:

Sub InsertRows()
    Dim bot As Long
    bot = [A5000].End(xlUp).Row
    [A1].EntireColumn.Insert
    Range("A1:A" & bot).Select
    Selection = "=IF(ROW()=18,TRUE,IF(AND(ROW()>18,MOD(ROW()-18,17)=0),TRUE,1))"
    Selection.SpecialCells(xlCellTypeFormulas, 4).EntireRow.Insert
    Selection.EntireColumn.Delete
End Sub

This is much faster than a loop.
