problem with regression line equation - Microsoft Community

georgemandersonatyale say:

problem with regression line equation

Who do I talk to about an apparent problem with the program used to compute a regression line in Excell? Briefly, if a set of x and y values are graphed and the line fitted and the x and y values then interchanged, the new fitted line does not have the expected inverse slope and the former y intercept is not now the x intercept. This problem did not happen with a small test set of 5 points, but does occur with a larger set of 20 real data points.

joeu2004 say: georgeMandersonATyale wrote:
if a set of x and y values are graphed and the line fitted and the x and y values then interchanged, the new fitted line does not have the expected inverse slope and the former y intercept is not now the x intercept. This problem did not happen with a small test set of 5 points, but does occur with a larger set of 20 real data points.

It is not difficult to see what George means.  And I think the fact that it did not happen with a small set is probably coincidence.

Consider the following example....  X1:X20 contains the series 1 through 20.  Y1:Y20 initially contains =RAND(), which I copy-and-paste-special-value and sort.  My numbers are in the table below [2].

In my case, using SLOPE(Y1:Y20,X1:X20) and INTERCEPT(Y1:Y20,X1:X20), the linear regression equation is about y = 0.043788x + 0.10547 [1].

When we reverse X and Y, I think George expects the linear regression equation to become about x = (y - 0.10547)/0.043788, or about x =  22.83745y - 2.40867 (using the actual SLOPE and INTERCEPT results, not the displayed approximations).

Instead, using SLOPE(X1:X20,Y1:Y20) and INTERCEPT(X1:X20,Y1:Y20), the linear regression equation is about x = 21.4913y - 1.64777.

Although I do not fault George for his expectation, it is easy to explain why the "reverse" linear regression equation is different if you look at how the slope and intercept of the linear regression line are determined.  For example, see the INTERCEPT help page.

The slope of the regression line is determined by:

Sum((x-xhat)(y-yhat))/Sum((x-xhat)^2)

where the "hats" are the averages of the respective data.

When we reverse X and Y, the slope of the regression line becomes:

Sum((x-xhat)(y-yhat))/Sum((y-yhat)^2)

keeping the original designations.  That is, the numerator is the same, but the denominator changes.

In order for George's expectation to be met, that would have to equal the inverse of the slope of the original linear regression, namely:

Sum((x-xhat)^2)/Sum((x-xhat)(y-yhat))

Thus, the following would need to be true:

( Sum((x-xhat)(y-yhat)) )^2 = Sum((x-xhat)^2)*Sum((y-yhat)^2)

I think it would be a rare set of data where that would be true mathematically, much less when we consider artifacts of computer binary arithmetic.

An explanation for the intercept formula for the linear regressions, original and "reverse", would be similar, albeit more difficult to talk through convincingly.  The formula is yhat - b*xhat, where b is the slope.  "The exercise is left to the student" ;-).

Obviously this is not a rigorous proof.  But I hope it sheds some light on the question for George.

A description of the method that Excel uses can be found at http://en.wikipedia.org/wiki/Simple_linear_regression.  I believe it is a common statistical method, although by no means the only method.

    Apps are crashing in lumia 720 - Microsoft Community

    sujiitt say:

    Apps are crashing in lumia 720

    Applications like Internet Explorer, youtube hd, UC browser are getting closed automatically and menu screen is getting displayed. This problem occurs often since 2 weeks. there was no problem for 4 months when i bought it
    jack cook - aka help_line say:

    OK, thanks for that info.


    Because it is happening with more than one app ( you indicated it occurs with Applications like Internet Explorer, youtube hd, UC browser) and since you indicated that it was recent issue (past 2 weeks), I believe that it is a software related issue. Possibly something you recently installed is running in the background and is "eating up memory". That is just a guess but I feel that may be the root cause.


    So if it were my device, I would Hard Reset and start with a fresh device. If the issue is gone after the hard reset then we know it was software related. If the issue returns BEFORE you reinstall any programs then it is a device issue (possibly bad memory) and you should return it for repair/replacement.


    You may not want to hard reset but it will be the easiest and fastest solution to find the root cause. You will get all your contacts, calendar back if you use the same live ID and if you have saved your pictures to Skydrive (or sync'd with your PC) they will not be lost. Reinstalling apps is relatively easy but I would do them just a few at a time then test for a day or so to make sure it is not an app causing the issue in case it returns.


    I WOULD NOT reinstall from a backup when you are given that choice when first setting up the device again just in case it was software related. That might bring the issue back.


    That is what I would do Sujiitt!
