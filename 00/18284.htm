betterworld? say:

You should understand that the title block and any frame used have a scale themselves, they are not parametric as they are in some other programmes.  They have a width, height and font size appropriate to a given page size and shape.  This is usually the scale of the front page.

Background pages can be added to the drawing on the basis of a single scale per background page. There is apparently no limit to the number of background pages that can be inserted into any foreground page within a drawing but I have never gone beyond four myself.  There is also no limit to the number of pages that can make up a drawing in Visio.  This is very different from most dedicated drawing systems like CAD and a legacy of the office systems environment of Miscrosoft Office.

When the "drawing" is sent to print in Visio all of the background and foreground pages are composited one on top of each other for each page in the drawing to produce the multiple scale drawing with its page-fitting title block and font sizes. It can be printed one sheet at a time or as a whole multi-page drawing.  The PDF can also be saved as a single page or as a whole document.  

People used to word processing or spread sheets will also find the Visio environment intuitive.  Indeed the copy and paste command which allows drawn information to be inserted directly into any MS document is a real benefit for speeding up report writing or sending parts of drawings an embeds within emails.

The Visio environment usually presents no problem for people who are used to the paper based drawing environment to whom it is intuitive.  However, it is confusing for people used to the virtual drawing environment in architecture and engineering where the legacy of CAD systems based on a native scale of 1:1 for everything automatically creates a difference between "model space" and "paper space".   Visio doesn't have this divergence unless you chose to create it deliberately.

However, if you are post-CAD person used to the BIM drawing environment, the differences are not as pronounced. In Revit, for example, the basic sheet set-up is not too different from Visio. In Revit, a printable page within a drawing is actually a predefined sheet with parametric data fields to fill in various boxes (job number, client, scale, etc) onto which you drop individual views, each of which can be assigned a unique scale.  In Visio the foreground page is the "sheet" and the background pages are the underlays onto which individual drawings are placed at a scale predefined by the background page scales.

When you come to print a drawing, it is important that the page size matches the page size and orientation in the printer.  There is a graphic which shows this in the print dialogue.  If you try to print an A1 drawing page onto an A2 sheet in your plotter you will end up printing the drawing across several pages.  We have all done it!

"Save as PDF" is essentially the same as printing. One important difference is that you can't have a different number of pages in the PDF from in the original drawing (background pages don't count). In effect the instruction to "save as PDF" produces a print file which is not allowed to print across several pages, so you only get the first page saved in the PDF and all the other pages that the printer would spit out, until the entire drawing has been revealed, are dumped.

This is necessary because, I guess, the PDF must have the same number of pages as the original drawing file.  (A PDF of an A4 report will always have the exact same number of pages as the original, otherwise the page numbering would not work).

I hope this helps you to visualise your issue.

I like the setup in Visio. I have a number of standard title blocks for different page sizes saved as templates and just open whichever one I need.  Its a bit like taking a preprinted sheet of paper and taping it to the drawing board before you start your drawing.  Those of you who started out in the virtual drawing environment will have missed out on that one.

deanh say:

For Autotext (and I assume QuickParts) you must include in the entry the last paragraph mark along with all the components in the Header or Footer.

To ensure you capture this last paragraph mark, turn on the view of the non-printing characters - Ctrl+ Shft+8 to toggle the view.

Be aware though, when pasting into Headings & Footers, and this applies to AutoText entries, Word for some unknown reason always added an extra paragraph mark after the pasted components. You will need to delete thsis to ensure your H&F's are as you want them.

All the best

DeanH

stefan blom say:

Also, unless direct formatting has been applied to the header paragraph(s) of the Quick Part, as you insert the Quick Part it will assume the formatting of the Header style in the target document.

entropist say:

Docs never saved or autosaved, now unable to find backups to autorecover HELP!

I have a 150-page 5MB report also using password protection I had been saving every 30mins. It was an old document converted from Word 2010. I had not been requested for my password the past couple of weeks, but every time would open and contain all work until updates were installed 5 May 2016. Now the last modified entry is 21 Apr 2016, and I'm unable to find autosaved docs in the file location. How can I find the autosaved files that are missing to recover the past two weeks of work?

We need to understand why docs are unable to save when clicking save or Ctrl+S, but still had current work until that horrible day when the doc reverted back to a 2-week old file. The laptop was also hibernated after saving, but may have interrupted the saving process because much time is needed to write a 150-page 5MB save file. Word 2013 also has been exited quite a few times, but commonly left open and only the file.docx closed. So the system state may have been reset, which removes autosave.asd files; but should still keep the saved.docx.

I found a way!

  1. File tab --> Options tab --> Save tab
  2. UNDER BOTH "Save recovery information every [specified time interval]" + "Keep the last autosaved version if I close without saving"; the file.asd is in "Autorecover file location"!!!!! 
  3. Return to File tab --> Open tab --> Recent Documents tab --> scroll to the bottom --> Recover Unsaved Documents. 
  4. Copy-paste the "Autorecover file location" into the Open file location directory bar to pinpoint the correct folder, and locate the correct file.asd to get your work back!! Yay!!
  5. Save as filename yyyymmdd.docx as a backup now! Do it now!
  6. Consider upload to OneDrive or save drafts as pdf as additional control measures.
  7. Have a 10-min timer nearby to ensure you manually SAVE, or Ctrl+S; and prevent this ever happening again!!
