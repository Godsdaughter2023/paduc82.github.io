lemp say:

I can't promise that the following will solve your problem, but it's my best guess based on reading other reports of the "unable to find a core driver package" error. 

The consensus seems to be that this error is caused by the remnants of a failed printer installation.  Microsoft used to have a handy automated "Fixit" tool to reset the Windows 7 printing subsystem but this has been removed (of course) now that Microsoft wants everyone to use Windows 10.  Fortunately, however, the guts of that tool have been preserved.

You can first try doing things using the normal GUI, but because your install never really worked, this procedure probably will not work either:

  1. Remove any HP printer-related programs using Control Panel > Programs and Features
  2. If there is an icon for the printer in Devices and Printers, right-click and "Remove device"
  3. Select any remaining icon in the "Printers and Faxes" section, then click "Print server properties" that appears in the tool bar at the top of the Devices and Printers window.  Then select the "Drivers" tab and "Remove" any HP drivers for the Laserjet 5000 (if any are present).  Remove both the driver and the "driver package"
  4. Clear out all files in C:\Windows\temp and also click Start, type %temp% in the Search box and press Enter and delete the files in that folder as well.  Just "skip" any files that Windows says can't be deleted for one reason or another
  5. Reboot and then attempt the printer installation

In the likely event that the above steps don't work, first read about the old Fixit tool here:  https://blogs.technet.microsoft.com/askperf/2012/02/24/microsoft-fixit-for-printing/

You will find that the two links (linkid 9829711 and linkid 9829710) no longer go to a download site.  Instead they go to a page that informs you that "the solution has been retired and is no longer supported."  That page suggests that you use a "troubleshooter" that you can download, put on a USB flash drive, and bring it to the problem computer.  You're welcome to try either the one for "devices and printers" or the one for "devices and hardware," but frankly, I've never been able to get either of these things to work.

Assuming that the "troubleshooters" didn't work either, the functionality of the old Fixit tool is provided by a PowerShell script that you can download from https://github.com/jdickson289/Print-Reset-Tool/

Instruction number 1 ("Start PowerShell in Admin mode") means

  • Click Start, type cmd.exe in the Search box, right-click the link to cmd.exe and select "Run as administrator" then respond to the UAC prompt as needed
  • In the Elevated Command Prompt window, type
         powershell

(PowerShell 2.0 is included with Windows 7)

There is a minor syntax error in the description of how to run the PowerShell script -- you need to type .\ before the commands shown in the article:

Following command resets print environment to out of box:
.\print-reset.ps1 -full

Following command corrects common printing issues:
.\print-reset.ps1 -light

Also note that when you start PowerShell, the system defaults to being in C:\Windows\system32.  If you saved the script (print-reset.ps1) in a different directory, you'll have to use the full path in the above commands.  Remember to surround the path and file with double quotes if there are any spaces in the path name.

If you've read the TechNet blog, you'll know the difference between "full" and "light" -- I suspect that you'll have to run the "full" mode, but be aware that (1) this will remove all printers other than the Microsoft XPS printer and Fax and (2) requires a reboot.

After you've cleaned out the printing subsystem, installing the printer using the properly-extracted contents of the cab file should (hopefully) work.

Good luck, and may the force be with you.

Running a Windows PowerShell Script

When you run a Windows PowerShell script, you must always indicate the full path with the name of the script even if you are working in the directory in which the script is located. You can use the following methods to run a Windows PowerShell script:

  • Use the dot and the backslash (.\) to indicate the local directory. For example:

    .\ <ScriptName>.ps1
  • Specify the full path of the script. For example:

    C:\Scripts\<ScriptName>.ps1
  • Specify the path of the script, but omit the extension. For example:

    C:\Scripts\<ScriptName>
  • Use the Invoke-Expression cmdlet to run a script. For example:

    Invoke-Expression C:\Scripts\<ScriptName>.ps1
  • Use double quotation marks for any paths that include spaces. For example:

    Invoke-Expression "C:\My Scripts\<ScriptName>.ps1"
  • Use the ampersand to run a script. For example:

    & C:\Scripts\<ScriptName>.ps1
