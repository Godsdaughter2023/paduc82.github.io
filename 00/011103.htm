hansv mvp say:

If you place all workbooks to be processed in a single folder, with no other workbooks, you can use a macro:

Sub ProcessFolder()
    Dim wbkS As Workbook
    Dim wshS As Worksheet
    Dim wbkT As Workbook
    Dim wshT As Worksheet
    Dim strFolder As String
    Dim strFile As String
    Dim blnSkipHeader As Boolean
    Dim lngFirstRow As Long
    Dim lngLastRow As Long
    Dim lngTargetRow As Long
    ' Prompt for folder
    With Application.FileDialog(4) ' msoFileDialogFolderPicker
        If .Show Then
            strFolder = .SelectedItems(1)
        Else
            MsgBox "No folder specified!", vbExclamation
            Exit Sub
        End If
    End With
    If Right(strFolder, 1) <> "\" Then
        strFolder = strFolder & "\"
    End If
    Application.ScreenUpdating = False
    ' Create new workbook with one sheet
    Set wbkT = Workbooks.Add(xlWBATWorksheet)
    Set wshT = wbkT.Worksheets(1)
    ' Initialize target row
    lngTargetRow = 1
    ' Get first filename
    strFile = Dir(strFolder & "*.xls*")
    ' Loop through folder
    Do While strFile <> ""
        ' Open workbook
        Set wbkT = Workbooks.Open(Filename:=strFolder & strFile)
        Set wshT = wbkT.Worksheets(1)
        If blnSkipHeader Then
            lngFirstRow = 2
        Else
            blnSkipHeader = 1
        End If
        ' Find last row
        lngLastRow = wshT.Cells.Find(What:="*", SearchOrder:=xlByRows, _
            SearchDirection:=xlPrevious).Row
        ' Copy rows
        wshS.Range(lngFirstRow & ":" & lngLastRow).Copy _
            Destination:=wshT.Range("A" & lngTargetRow)
        ' Close workbook
        wbkT.Close SaveChanges:=False
        ' Increase target row
        lngTargetRow = lngTargetRow + lngLastRow - lngFirstRow + 1
        ' Skip header row next time
        blnSkipHeader = True
        ' Get next filename
        strFile = Dir
    Loop
    Application.ScreenUpdating = True
End Sub

The space between lines of the same paragraph is determined by the 'Line spacing' setting in the Paragraph dialog.

Set it to Single if you don't want extra space between the lines.

suzanne s. barnhill say:

The space between lines of text represented by Single spacing is coded into each font. For example, if you are using 10-point Times New Roman, the Single line spacing will be 12 points (120% of nominal font size). Many fonts have unreasonable amounts of leading (line spacing) built in. Both Exactly and Multiple line spacing allow you to reduce the line spacing below Single/100%. So, if your text is, say, 16 points, you can try reducing line spacing to Exactly 16 points (or less), or you can select a Multiple value of .99 or less.

At some point, text may be cut off, but often it will print even though this is not evident from Print Layout view or even Print Preview.

p_369 say: After 6 long hours fiddling around, I finally figured it out! Instead of highlighting the actual text, I highlighted the white space only and used the "Exactly/Multiple" point feature to reduce the white spacing in my 11 point TNR text. As such, there were no issues with my text being chopped up. Doing this was very helpful and gave me more incremental control since I was working on my resume and trying to reduce the awkward/too big of space between my section headers and bullets that the "enter" button was automatically creating.

You have this line which is correct.

That defines the connection string to attach to the database. then you have this line

And that messes up your connection string. Delete that line...

Sub test()
    Dim strConnection As String
    Dim strSQL As String
   
    Dim cnt As ADODB.Connection
    Dim rst As ADODB.Recordset
   
    strConnection = "Provider=Microsoft.Jet.OLEDB.4.0;" & "Data Source=C:\Test.mdb;"
    strSQL = "Select * From tblTime"
    Set rst = New ADODB.Recordset
    Set cnt = New ADODB.Connection
   
    cnt.ConnectionString = strConnection
    cnt.Open
    With rst
        .CursorLocation = adUseClient
        .CursorType = adOpenStatic
        .LockType = adLockBatchOptimistic
    End With
    
    rst.Open strSQL, cnt, , , adCmdText

End Sub
