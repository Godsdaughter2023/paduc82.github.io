orchardsix say:

I had this problem. Took a while to figure it out through trial and error. This is what worked for me.

 

1. In Table Properties (right click after you have selected the table), go to the Row tab.

 

2. Check "Allow row to break across pages."

 

3. UNCHECK "Repeat as header row at the top of each page."

 

If you have "Repeat as header" checked, then the table won't break.

 

This may be a bug, but once I did that, the tables broke across pages as they should. No header repeat though.

 

It's non-intuitive, but it worked for me.

It could be undesirable to apply the Page Break before attribute to the caption as you could end up with a mostly empty page before the table if there are only a few lines of text on the page before the table.

If you format both the caption and the paragraph in the Repeat heading row so that they are kept with next:

  1. The caption and the Repeat heading row will NOT be orphaned at the bottom of the page.  (The whole table will moved to the next page if there is not room for the caption and the Repeat heading row AND the first row of the table after the Repeat heading row.
  2. The Repeat heading row will continue to function correctly.

The above is true in both 2010 and 2013.

Any suggestions would be appreciated.
john korchok say:

Unfortunately, Publisher links are always absolute links: they include the entire path. And they can't be edited after the fact to correct this.

The workaround is to place your Excel sheet at the root of your C:/ drive, or in a folder like C:/Source. Relink all the spreadsheets in your Publisher document. Then when you distribute the files to other computers, the Excel file goes in C:/ or C:/Source on everyone's computer.

Even better, if you have a network drive that everyone shares, you could also place the Excel  file there and relink. Everyone's computers should read that as the same location, plus you can do all updates to the one file on the network.
