Your 'NET' command format has errors - compare the following:
>  NET USER ADMINISTRATOR/ ACTIVE: YES  << two format errors ('/' and ':space').
>  NET USER ADMINISTRATOR /ACTIVE:YES   << correct
>  NET USER ADMINISTRATOR /ACTIVE:      << correct - 'yes' by default


This could be just a typo in your comments... but the command lines have to follow a formal format.

 

For assistance in the basic 'NET' command enter.
>  NET /HELP


For assistance in a compound 'NET {command}' enter.
>  NET {command} /HELP
>  NET USER /HELP   << Example of HELP for the NET USER command.


Example using the 'NET USER' command.
>  NET USER                 << List users of system (may also use plural USERS).
>  NET USER {user}          << List properties of a given User account.
>  NET USER ADMINISTRATOR   << List properties of the Admin account.


If you activated the Admin account, on completion of task, be sure to deactivate the Administrator account to return to normal status (unseen).
>  NET USER ADMINISTRATOR /ACTIVE:NO  << follow the same format at to activate the account with a 'NO' element.

 

To close out the Command Window enter:

>  EXIT


NOTE:  Most entries may be in lower case form. But if an error occurs, first check format, then try UPPER case. For assistance in common system commands enter 'HELP' at the line prompt or 'HELP {command}' for detailed format.  'NET' is not part of the library for common system commands, hence why you must use 'NET /HELP' as given above.
