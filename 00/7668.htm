Without actually spending time setting this up and debugging it, I suspect that what is going on is that the recordsetclone is not being repositioned by the requerying of the form.   It is completely possible for a form's recorsdsetclone to be positioned to a different place than the form's recordset.  For this reason, I wouldn't consider your current code safe in the first place.  I'd recommend you either just delete using the form's recordset itself:

 

    frm.Recordset.Delete
    frm.Requery

 

or else explicitly position the recordsetclone before deleting:

 

    With frm.RecordsetClone
        .Bookmark = frm.Bookmark
        .Delete
    End With
    frm.Requery

Thank you for reporting this issue.  The Custom Task Pane display issues may be related to some changes related to enabling Dynamic Dpi in this version of Office.  The painting issue can occur when there is a mismatch between the Dpi Awareness of the Custom Task Pane window and the host application, such as Excel.  There are some remedies for this situation:
