You cannot open a ZIP file with any version of Word whatever your operating system might be.

ZIP is a compressed file format. You can view the content of the zip and extract the files using Windows Explorer or one of the several ZIP file utilities.

However, some e-mail systems change the extension of DOCX format documents which are ZIP files in all but name, from DOCX to ZIP. If this is what has occurred then change the extension back to DOCX. Word will not open such a file if it does not contain the DOCX file structure.

I had the same problem upgrading my firmware. After trying lots of different things, I found something that worked:

After it downloads the 83MB file (it will say something like "15 minutes remaining")... the phone apparently shuts itself off. It really didn't. If you look really closely, the backlight just went off--but the Nokia Logo is still there. In fact, if you were patient and paid attention during the install, you might have heard the Windows "bee-bop" sound of something disconnecting a USB device...

So what you do is do exactly what the installer said NOT to do: unplug the USB cable from the phone. Wait a few seconds... and now plug the USB cable BACK into the phone, Windows will do the "bee-bop" re-detection of the phone. When it's finished redetecting, click on the "Retry" button on the installer... and boom, the installer continues fat dumb and happy.

At least it did for me. Your mileage may vary.

Good luck.

suzanne s. barnhill say:

In Word 2003, as you doubtless know, if you click the E-mail button, a mail header (for the default mail client) is added to the current document. This setting doesn't affect other documents you might open. It's possible the document was saved with this mail header, perhaps inadvertently added before the user figured out how to send the document as an attachment. Certainly it would not be usual, however, for the To: field to be already populated unless the user had typed something there.

If you remain curious about this, you could try opening a test file, adding the email header and typing something in it, then saving the document and closing it. If, when you reopen it, you get this effect, then that would be the explanation.

And if this ever happens again, you can remove the header by clicking the E-mail button to turn it off. Then make some other edit in the file and resave it, and you should be okay.

Serious bug with Subtotals in Excel 2011 (when compared to Excel - Microsoft Community

steve maser say:

Serious bug with Subtotals in Excel 2011 (when compared to Excel 2004 and 2010)

So, to explain the bug in words (which probably will be difficult, but…):    Doing a second Subtotal on a set of data where there is only one value of data in the column to So, to explain the bug in words (which probably will be difficult, but…):    Doing a second Subtotal on a set of data where there is only one value of data in the column to be subtotaled, creates the subtotal in the wrong row in the spreadsheet (compared to Excel 2004/2008/2010).

Specific steps to reproduce (which explains this bug better):

Create a new workbook with the following data:

Row 1 headers:   Amount, User, Period
Row 2 data:  1, bill, 1
Row 3 data:  2, bob, 1
Row 4 data:  3, bob, 2


The goal is to get subtotals for both the user and the period *for the user*  (not for all periods for all users.)

To do this:

1)  Select Rows 1-4

2)  Go to Data --> Subtotals.   In the pop-up box, select:

At each change in:  User

Use function:   Sum

Add subtotal to:   Amount (only -- deselect any other box)

Deselect "Replace Current subtotals"

Leave "Summary below data" checked.


This generates a (correct) result that looks like:

Amount User Period
1, bill, 1
1, bill total
2, bob, 1
3, bob, 2
5, bob total
6, Grand Total

Now, do another subtotal to get the *period for the user* totals

1)  Select rows 1-7

2)  Go to Data --> Subtotals.   In the popup box, change At Each change in: to "Period"

This generates a (correct) result that looks like this:

Amount User Period
1, bill, 1
1, , 1Total
1, bill total
2, bob, 1
2, , 1Total
3, bob, 2
3, , 2 Total
5, bob total
6, Grand Total

This is how Excel 2004/2008 and 2010 generate these results -- the "period" totals are before the "user" totals.

Now do the same thing in Excel 2011 (14.1.3 version):

1)  New spreadsheet.

2)  Same data:

Row 1 headers:   Amount, User, Period
Row 2 data:  1, bill, 1
Row 3 data:  2, bob, 1
Row 4 data:  3, bob, 2

Same first subtotal generation (which comes out correct):

To do this:

1)  Select Rows 1-4

2)  Go to Data --> Subtotals.   In the pop-up box, select:

At each change in:  User

Use function:   Sum

Add subtotal to:   Amount (only -- deselect any other box)

Deselect "Replace Current subtotals"

Leave "Summary below data" checked.


This generates a (correct) result that looks like:

Amount User Period
1, bill, 1
1, bill total
2, bob, 1
3, bob, 2
5, bob total
6, Grand Total


NOW THE BUG: 

Do the second subtotal generation:

Do another subtotal to get the *period for the user* totals

1)  Select rows 1-7

2)  Go to Data --> Subtotals.   In the popup box, change At Each change in: to "Period"

This generates an INCORRECT result that looks like this:

Amount User Period
1, bill, 1
1, bill total
1, , 1Total
2, bob, 1
2, , 1Total
3, bob, 2
3, , 2 Total
5, bob total
6, Grand Total


NOTE THE SWAPPING OF THE 3rd AND FOURTH ROWS.    This (simple) example does not show the cascading problem if you have multiple "single period" values for users.

WORKAROUND -- which is a non-starter, but shows the way it should work):   If I have a *second Period* row of data for "bill" -- like 0, bill, 2 -- then the order of the subtotals comes out correctly.


To reiterate:   Excel 2004/2008 and 2010 -- do this correctly.   Excel 2011 does not.
