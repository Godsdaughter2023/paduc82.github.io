doug robbins - mvp office apps & services (word) say:

You would be well advised NOT to store your documents on the Desktop.

Click on File>Options>Save and then use the Browse... button to select a folder on your system that you want to use as the Default file location.

charles kenyon say:

Adding to (and agreeing with) Doug Robbins' advice:

Store your documents in your Documents folder (or subfolders). This makes automatic backup by third-party software much more likely and easier.

If you need to access them from your desktop, put a Windows shortcut to the file on your desktop. This shortcut will not be moved by opening and editing the file using the shortcut.

Learn to use Word's recent documents feature. You can pin documents (and folders) in this feature. It under File>Open.

I've found it so useful that I've put a button for Recent Files on my QAT.

P.S. In case you really want to know why the file appears to move, it is because when you edit and save a file, Word converts the current file to a .bak backup file and creates a new file with the updated information. The backup file takes the spot the file was in on the desktop. I believe with certain file settings this file may not be visible.

=SUMPRODUCT(--(A1:C1))

This formula converts text values in right format and sums them up.

macropod say:

The simplest way of handling this is to not use a table, but use paragraphs with a tabular layout instead. Then you could use field coding like:

{IF{MERGEFIELD Monthly_Basic1}> 0 "Education Allowance→{MERGEFIELD Monthly_Basic1}→{MERGEFIELD Annual_Basic1}→{MERGEFIELD Monthly_Basic2}→{MERGEFIELD Annual_Basic2}¶

"}

or:

{IF«Monthly_Basic1»> 0 "Education Allowance→«Monthly_Basic1»→«Annual_Basic1»→«Monthly_Basic2»→«Annual_Basic2»¶
"}
for each entry.

You can do the same kind of thing using a one-row table for each item, but that will result in an empty paragraph being output between each of the data rows.

Note: The field brace pairs (i.e. '{ »') for the above examples are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. Likewise, the chevrons (i.e. '« »') are part of the actual mergefields - which you can insert from the 'Insert Merge Field' dropdown (i.e. you can't type or copy & paste them from this message, either). The spaces represented in the field constructions are all required. Instead of the → and ¶ symbols, you should use real tabs (with the appropriate tab-stops) and line/paragraph breaks, respectively.
