john w. vinson say:

STOP. You are going the wrong way!

Access works only over a fast, stable, preferably wired Local Area Network. You CANNOT use OneDrive for either the frontend or backend of a split database - Access is simply too demanding of fast, interactive, constant updating of the database file. Cloud file storage like OneDrive is designed to store and retrieve complete, intact files, not to keep files open and keep tweaking them.

If you need multiple users in different physical locations, you will need to use a Client/Server arrangement like SQL/Server, or a web application; or have the entire Access application (front and backend) on one stable LAN, and use some form of remote terminal connection such as Citrix to connect to it from the remote site.

Sorry, but Access just isn't designed for this task!

daniel pineault say:

You can't use Access using cloud shared drives (OneDrive, Dropbox, ...).  Look over http://www.devhut.net/2016/09/24/access-back-end-location-wan-online-server-onedrive-dropbox/

There are solutions to accessing information from multiple location, some covered in the above listed article, others include using proper web technologies (PHP, . net, ...) Or you could look into using Azure to host the data and then a conventional Access frontend thus creating what is referred to as a hybrid database.  The hybrid database needs to be design carefully or else you'll suffer from serious performance issues though!
