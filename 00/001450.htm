Are you on a Windows Vista machine?  If so, you could try going through the following steps.  The steps below involve making change to the registry, so if you are not comfortable with making these changes, you may want to open up a support case with Microsoft to have them go through these with you, since making an incorrect change could result in you not being able to use your computer.  In addition to this, I would recommend backing up the problematic key before making any changes.

 

+>Close Access.

+>Open the registry by clicking on the start menu, typing in regedit, and then launching regedit.exe.

+>Navigate to the following registry key on your Windows Vista machine (there are several keys that look similar to this, so be sure you are on the correct key):

+>After you select the 9.0 key, you will see several values listed on the right hand side.  One default value that is followed by a list of additional values that look like version numbers below it.

+>Change the default value from the version listed to the latest version number in the list below it.  When the problem occurs, the default value tends to be 9.0.30304.0. You would then change this default value to the latest version number in the list below it.  To change the default value, you would double click on the default value and in the dialog that opens, type in the new value.  For example, if the last value / most recent version in the list was 9.0.30729.4148, you would change the default key value to this value (make sure you change the default value to an actual most recent value that exists in the list below it).

+>After you make this change, you can close the registry and test Access again.  If Access works, you should be all set.  If it still fails, you can restore the registry key you changed by double clicking on the .reg file you exported above.
