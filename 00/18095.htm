>> I meant that "template" seems to have a far more restrictive meaning when it is a *design* template.  It's not the master slides or the layout.  It's only the backdrop.  But that was just my opener to complain about how the terminology can be ambiguous, which seems to be a large part of my confusion.

A template is a POT file either way.  The difference is in how/when it gets used.  When you apply a design, PPT shows you the available templates, you choose one, and only the design portion of the template gets applied to the existing slides in your presentation.  None of the slides, macros or other stuff in the template file is applied to your presentation.

And yes, it IS confusing.  You're not alone in the universe.  ;-)

>> If I “apply a design template” to a slide in Normal View, it only

This might be because the text boxes on your slides are not text placeholders.  The only text that's affected by the master is text in placeholder (the things that start out as "Click here to ..." blocks of text, or become that again if you delete all the text you've added).  Other text isn't controlled by the master/design/template.

>> Yes, I was picturing that more generic situation.  I have in fact moved text boxes in master slides, so I'm sure the layout on the master slide does not match any of the ones in the layout menu.

See above.  For example, if you have something that LOOKS like a title, check Outline view.  If it doesn't appear there, it's not really a title and won't be affected by the title text placeholder on the master.

You may also need to reapply the slide layout.  If you've moved e.g. the title text on a slide or changed its formatting, PowerPoint assumes that you did it for a reason and don't want it reset to the master formatting willynilly.  So when you change the master, PPT may not change the individual slides until you reapply the layout (sometimes several times).

jeanette cunningham say: If you still want to record both start and end odometer in the form, create a query you can use to look up the previous end odometer reading for that vehicle.

The new start will need to be calculated by finding the highest mileage recorded for that vehicle.

The query will have a field that correctly identifies the vehicle and a field for the end mileage.
Make it a totals query and use Max as the totals expression for the end mileage column. Save this query.

Use this query with a DLookup and a criteria for the vehicle field.

Something like below:
Dim lngMiles as long
lngMiles = NzDLookup("[Your end miles field]", "[The saved query]", "VehicleID= " & Me.VehicleID),0)
Debug.Print lngMiles

Note: replace my object names with your own.

You would use the DLookup expression on the form's current event to set the default value of the start miles control to the value of lngMiles for a new record.

By using the default value, you avoid dirtying the form if user changes their mind and wants to cancel the new record.
