hansv mvp say:

I see two problems:

  1. The word "PANT" in "LONG PANT"doesn't match the word "PANTS" in the list on Sheet2. If you change "LONG PANT" to "LONG PANTS", the word "PANTS" will be found.
  2. "DRAWSTRING SHORT DRESS" matches two items in the list: "DRESS" and "SHORT DRESS". Depending on the formula you use, one or the other will be found.

Option 1: enter the following array formula in B2, confirmed with Ctrl+Shift+Enter:

=INDEX(Sheet2!$A$1:$A$6,MATCH(1000000,SEARCH(Sheet2!$A$1:$A$6,A2)))

Modify the range $A$1:$A$6 as needed, then fill down to the end of the data.

In your example, it will return "SHORT DRESS" in B5.

Option 2: enter the following array formula in B2, confirmed with Ctrl+Shift+Enter:

=INDEX(Sheet2!$A$1:$A$6,MATCH(1,MATCH("*"&Sheet2!$A$1:$A$6&"*",A2,0),0))

Modify the range $A$1:$A$6 as needed, then fill down to the end of the data.

In your example, it will return "DRESS" in B5.

The one thing I can't seem to get through to you no matter how I say it is that the "Time Stamp" facility that exists within Code Signing certificates specifically overrides the fact that the certificate itself has expired.  The certificate expiration only affects the ability to use it to sign new code, not the validity of the code itself.

 

This Time Stamp facility does not exist or apply to SSL certificates used for securing transmission of data between servers and/or clients, since these must be valid at the time they are used and expiration is one criteria that makes these invalid.  Many who argue this point are confused by this apparent anomaly, which is actually a clear difference between certificate types and usage.

 

The problem I don't think you see is that if the expiration of code signing certificates wasn't modified by the Time Stamps, the result would be that all code created by them would need to be re-released and updated whenever the previously used certificate expired,  Meaning every single signed application on your PC, including all of the Windows components would require complete replacement.  Obviously this would lead to the use of much longer certificate validity terms of years or decades, which would create other far more difficult problems of high cost (CS Certs are $3-500/yr. now) and still higher risk that a certificate private key might somehow become compromised within that time.

 

So from a simple cost vs. true benefit analysis, it's clear that the current system is reasonably effective at true security, while still relatively cost effective.  As time moves on, the keys will get larger and more complex as the computers get faster and the costs will go up with inflation as well.

Every Code Signing Certificate is purchased with a specific validity period. The digital certificate can be used to sign code as frequently as needed during that validity period. Timestamping ensures that code will not expire when certificate expires. If your code is timestamped the digital signature is valid even though the certificate has expired. If you did not use the timestamping option during the signing, you must re-sign your code and re-send it out to your customers.

Moving marquee - How to Hide/Unhide - Microsoft Community

moniqueyaptenco say:

Moving marquee - How to Hide/Unhide

Microsoft Office for Mac 2011 tutorial: Excel basics tutorial has a hint/tip.

pg. 13

  • To hide the moving marquee around a copied selection, press ESC (located in the upper-left corner of your keyboard). 

Does anyone know how to UNDO this, and show the moving marquee?

Thank you

bob jones aka: cybertaz say:

Your screen shot doesn't show the 'moving marquee' referred to in the tutorial.

What your screen shot shows is only the range pasted to [D2:D4], the bold white + [which is the Excel select cells pointer], and the Paste Options button [clipboard icon]. The heavy border & shading in the screen shot is a selection marquee. There is no built-in feature for toggling the selection marquee on/off. Pressing esc will cause the Paste Options button to be removed because there no longer is anything on the Clipboard to be pasted, as Bob G explained. Neither the pointer nor the selection marquee are affected by pressing the esc key, though.

The moving [aka, dazzling] marquee mentioned in the tutorial is the animated dashed border that appears around the copied range as soon as you Copy or Cut. In the tutorial illustration that's B2:B4 in the North column. Pressing esc will remove the moving marquee immediately, but that marquee will go away of its own accord when just about any other action [e.g., applying any type of formatting to the selected cells, entering data in another cell, etc.] is taken. Once that marquee is removed there is no option for restoring it.

I believe that what may be deceiving you is the misuse of the term 'hide' in the tutorial. IOW, in reality the moving marquee isn't hidden when you press esc, it's removed or cleared along with the clipboard content.
