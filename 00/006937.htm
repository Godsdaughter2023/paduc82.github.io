sheen valentine ali msft say:

Hi Brie,

I would like to first explain that in SharePoint, there are many types of lists, all suited for different needs.

As a task list and a document library are separate lists, they do not share resources.

As a workaround, if you would like to use the document library to save blogs and would like to use the task list specifically for organizing tasks and not attaching documents, you may put a link to a document from the document library in the description of a task. This way, if someone opens the task and would like to look for the document, they can refer to the description and be directed to the document location.



Additionally, you may also use a three-state workflow to automatically / manually create a task for a document. You can refer to the article below on how to create a three-state workflow:
Use a Three-state workflow.

The workflow will create a task list similar to what we create manually, with the link to the document also in the description:

The benefit of using a workflow is automation. Every time you assign a new workflow, the system is able to send an e-mail to the person or group assigned to the item, and also create a task automatically, saving a lot of time in the process.

doug robbins - mvp office apps & services (word) say:

If you need to have the hyperlink functionality available in the target document, you will need to re-create the hyperlinks in that document as they are references to a hidden bookmark that is inserted into the original document by the process of creating the cross reference.

If you do not need that functionality, you could press CTRL+SHIFT+F9 to unlink the cross reference from its source.

stefan blom say:

As long as you make sure to copy both the cross-reference and its target, the cross-reference (and its hyperlink) should be left intact.

Cross-references make use of hidden bookmarks, and therefore the cross-reference may point to the wrong target if the document into which you are copying already has a bookmark with that name. I haven't tested this thoroughly.

john sr wilson say:

Looks like you might have to use a macro

Sub zapInk()
    Dim osld As Slide
    Dim L As Long
    For Each osld In ActivePresentation.Slides
        For L = osld.Shapes.Count To 1 Step -1
            If osld.Shapes(L).Type = 23 Then osld.Shapes(L).Delete
        Next L
    Next osld
End Sub

To turn off Link Preview for all users, try the steps below:

Note: Only admin have permission to perform the steps. If you're not an admin, please let admin do this.
