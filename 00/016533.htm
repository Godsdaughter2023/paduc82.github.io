thetao---01 say:

It's probably not email.  Maybe the other thread you mentioned was the one where I took part?  :)

 

I don't have this problem on my E73, but it bothered me because I like things to make sense.  My phone shows 149 MB free internally, and 67 MB of "Other Files".  I don't know many ways to get detailed information on the interal storage.  One is to install (if you have enough space?) Netqin Mobile Guard from http://www.netqin.com/en/guard/symbian/﻿ .  It's a free program of disk utilities.  One is called "OS Scan" which has an option to clear "System Junk Files".  I don't think this will fix your problem, but you can try.  Much more interesting is its File Manager, which unlike other products, lets you view and calculate sizes for the Symbian protected directories PRIVATE and SYS.  If you find that PRIVATE (think: the Windows Registry or the Unix \etc folder) has ballooned to hundreds of MB's, that's at least part of your answer.  You can't change these folders with Mobile Guard, only look.  A small warning: it also includes several helper programs like a download manager, but these can generally be turned off.  (Don't be surprised when you see the pop-ups.)  My PRIVATE folder is 23.2 MB.

 

So phone memory is somewhat off limits, but the memory card uses the FAT32 filing system and can be analyzed quite easily. :)  I have a couple cards that I use mostly for .mp3 files and the one inserted now is 8 GB.  Nokia's File Manager shows 317 MB free memory and 36 MB of "Other Files".  I put the phone in Mass Storage mode and was able to account for most of File Manager's categories:

 

1.  Calendar and Contacts are stored on the phone, so these are 0 KB.

2.  Messages (SMS + Email, I think) are also on the phone, but File Manager lists 34 bytes, which I tracked to two files in a "Mail2" folder in PRIVATE.

3.  Images was all my camera photos.

4.  Sound files was all my .mp3's, plus a separate podcast.

5.  Video clips showed 106 KB in use, but I hadn't recorded any videos.  I tracked it to an .mp4 sound file made with Recorder.

6.  Documents was three PDF's.

7.  MIDPApps was one .jar file I'd downloaded.

8.  SIS Applications was a problem.  I have five .sis/.sisx files on the card (none installed), but these total just 4165 KB, not the 6073 KB listed.  Seems this could be a bug.

 

But what about Other Files?  I'd looked everywhere, and there wasn't another 36 MB to be found.  Then it hit me.  Do you understand the use of clusters, blocks, etc. in file systems?  You write a 200 Byte text file to a disk, but if the cluster size is 32 KB, for reasons of efficiency, that's the amount of space actually allocated.  For example, my memory card's PRIVATE folder is only 3 MB, but the allocated space is 9.1 MB!  And this is for only 219 very small files.  If I total the difference between the cluster sizes and the file sizes for every file listed in 1-8 above, then add the cluster sizes for everything else not listed (like PRIVATE), I get 30.84 MB.  This is still off by ~5 MB, but maybe it also counts other disk space, like the FAT tables?

 

So "Other Files" seems a bit misleading.  Don't think of it as files you can delete, but "space allocated by the filing system or operating system, which you possibly can't reclaim".

 

The phone memory doesn't use FAT32, and a poster on the other thread suggested it uses either a network-based (maybe like SMB or NFS?) or a database-like system.  (I assume you've already moved Nokia Maps data to the memory card?  You could do the same for email and SMS, too, and also install programs there.)  Nobody here seems to know exactly how the phone memory allocates data, but short of you deleting a large, complex program, which in turn removes hundreds or thousands of tiny files (like the 219 files in my PRIVATE), I don't think you'll fix this without a hard reset. :(  It could be a bug, it could be a disk error, it could be from a program littering your phone with lots of little files, or it could be something completely different.

 

This answer probably isn't what you wanted, but is likely the best you'll get here...or from Nokia.  Good luck!

How do I view odd line numbers only in a database? - Microsoft Community

matthew thornton say:

How do I view odd line numbers only in a database?

How would I be able to just view the odd line numbers in the MS Office Access 2007 database?  I don't know how to just view those line numbers.
ken sheridan say: As Karl has pointed out, a relational database table, being a set, has no intrinsic order, so the concept of a line number is meaningless.  If you simply want alternate rows in an arbitrary order you can do as Karl suggests and add an autonumber column to enable the Mod operator to be used in a criterion.  You can, however, return alternate rows in a query's ordered result set by computing the position of each row in the sort order.  Here's an example using a Transactions table by date, with the primary key TransactionID column acting as the tie breaker in the event of more than one transaction falling on the same day:

SELECT COUNT(*) AS RowCounter, T1.TransactionDate, T1.TransactionAmount
FROM Transactions AS T1 INNER JOIN Transactions AS T2
ON (T2.TransactionID<=T1.TransactionID OR T2.TransactionDate<>T1.TransactionDate)
AND (T2.TransactionDate<=T1.TransactionDate)
GROUP BY T1.TransactionDate, T1.TransactionID, T1.TransactionAmount
HAVING COUNT(*) MOD 2 = 1;

The above method, using a JOIN, is the most efficient but the query is not updatable.  For an updatable query you can, albeit far less efficiently, call the VBA DCount function to get the same result:

SELECT DCOUNT("*","Transactions","TransactionDate  <=  #" &
Format(TransactionDate,"yyyy-mm-dd") &
"#  AND (TransactionID <= " & TransactionID & "  
OR TransactionDate <> #" & Format(TransactionDate,"yyyy-mm-dd") & "#)") AS RowCounter,
TransactionDate, TransactionAmount
FROM Transactions
WHERE DCOUNT("*","Transactions","TransactionDate  <=  #" &
Format(TransactionDate,"yyyy-mm-dd") &
"#  AND (TransactionID <= " & TransactionID & "  OR TransactionDate <> #" &
Format(TransactionDate,"yyyy-mm-dd") & "#)") MOD 2 = 1
ORDER BY TransactionDate, TransactionID;
