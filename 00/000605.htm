neha singh msft say:

Hi Duncan,

 

Shared with me is not a real library in OneDrive for Business. It’s similar as a view rather than a folder. In “Shared with me”, we will only have the files which are shared to us by others.

 

It is not feasible to move or copy a file from any outside folder to “Shared with me” view. This option is neither available in Windows system nor MAC.

 

If you try to move a file/folder from your local OneDrive for Business folder, you won’t see the option to move the files to Shared with me, you’ll see your local folders instead.

 

 

The reason behind this is the files in “Shared with me” are not stored in your OneDrive for Business library. The real locations of those files are other people’s OneDrive for Business libraries who shared the files with you.

 

If you want to share a file/folder with other user, you can simply select the folder and click on ‘Share’ button and enter the name of the user. Once the file is shared, now this file will be visible to other user in his “Shared with me” area.  

 

Let me know if you have any doubt.

Probably not, unless we get lucky with some wild-a.s.s guesses.  It is difficult enough to troubleshoot such problems while sitting shoulder-to-shoulder with the user, much less at arm's length.

Since you already took the draconian step of reinstalling Office 2010 to no avail, we almost rule out installation problems as the culprit.  (I say "almost" because we cannot know if the reinstallation was done correctly.)

First, define "intermittent".  Is the problem consistent for some files, but it does not happen for others?  Or is the problem inconsistent for the same file?  (Caveat:  In my experience, many users will insist that it is inconsistent for the same file when, in fact, they are not sure themselves.  In the final analysis, it is often consistent for the same file, be it always problematic or not.)

If the problem is truly inconsistent for the same file, look for a networked device as the culprit.  For example, a non-responding networked default printer or a slow-responding networked disk.  If not a networked device, then perhaps a non-responding (or slow-responding) website that is the target of a hyperlink in the Excel file.

If the problem is consistent for the same file (usually the case when push comes to shove), first determine if Excel is truly "frozen" or if it is simply very busy.  Being "very busy" might be expected for a complex workbook.  Try saving the file in Manual calculation mode with "Recalculate before save" unselected.

Try starting Excel in safe mode, open the file in Excel instead of clicking on the file icon, and perhaps save the file in Manual calculation with "Recalculate before save" unselected.  See safe mode instructions under "Let me start it myself" at http://support.microsoft.com/kb/280504.

I also like to restart the computer in safe mode without networking (or with networking if the workbook requires it) when I start Excel in safe mode.  The procedure might vary from computer to computer; you might need to consult instructions for your particular computer.  For my laptop, I press and hold F8 after the system has shut down and while it is restarting (black display).  That gets me into a BIOS menu with start-up options, which include safe mode with and without networking.

Hopefully one of those ideas will pan out for you.

PS:  All of the above assumes that the user is using a stand-alone desktop or laptop computer.  If he/she is using Excel across a network -- something like Citrix or Remote Terminal or "cloud computing" -- there are many other issues to consider.  If that is the case, it would be prudent to get IT staff involved in troubleshooting the problem.

Caveat:  In multicore CPUs with hyperthreading enabled, 100% CPU utilization is really 100/#cores or 100/#cores/2.  (The "/2" is for hyperthreading.  Current Intel CPUs are limited to 2 threads per core, AFAIK.)  For example, in a 4-core CPU with hyperthreading enabled, Task Manager will display 8 "CPUs" in the Performance tab; and a process on one "CPU" that consumes 12.5% of the system is in fact consuming 100% of that "CPU".

Equations automatically save in bold in Word 2012

Hello all,

I have been having the same problems in some word documents I use as others on this board have, and yet for me the answers have not been able to resolve my issues (possibly because there are many authors of the documents I use before they come to my desk). (See for example this thread on a similar issue: http://answers.microsoft.com/en-us/office/forum/office_2010-word/my-equations-get-bold-in-figure-captions-when-i/217af5e9-bcd2-4c37-af7a-e87406de38cc?msgId=27b492e6-c686-449d-8604-4909aafc3593

The problem lies in the equation fonts automatically saving as bold, even after I have changed them to normal. Even more infuriatingly, sometimes I manage to get half of the equations to save as normal (after having been bold), then once I have managed to get the other half saved as normal, the first half reverts to bold again.

Now in, for example, the answer linked above Stefan Blom talks about modifying something called "Caption Style" as a solution. Now I have no idea where I can find this caption style. I have tried searching both in Word help and on the internet without finding what this caption style is, or where I can find it.

But what I would really like to know is why Word does this. There must be some sort of a built in mechanic to word that looks for equations and turns them bold for some reason or another, and as a result, there must be some way to turn this mechanic off.

Is there anyone out there who can help me with this? It would end up saving me hours upon hours of work if this could be fixed.

markus wilde say: Dear Stefan,

I have finally figured this out (somewhat). At first I found out that there were two types of styles used for the equations. One was called Main Part Style, which was the same style used for the main body of the text. This style caused the equations to become bold, and even after manually changing them back to regular, to become bold again after closing and re-opening the document. I.e. this was the "bad" one.

The second one was called Table Grid Style (probably because all our equations are located in tables for editing purposes). This was the "good" style.

At first I tried to change the style of the "bad" ones to Table Grid Style by using the drop down menu in the Styles window (ctrl+shift+s), but for some reason Word would not let me change to this style (other styles worked).

At last I decided to try an experiment. Usually we use the Normal Style (specially formatted) for the main body of the text, so I thought, what happens if I change the text from this Main Part Style to Normal Style? The result was that this seemed to automatically change the tables that contained the equations to Table Grid Style - and while the bold font remained, by toggling bold on and then off again in the font bar the equations went back to regular - and have since then stayed that way.

So, again, the solution that worked for me: make sure the main body of the text (not just the equations) is in the correct Style.

Thank you Stefan for your help.
