macropod say:

Presumably you want to reduce wastage by having the label printing start with the unused labels from a previous merge. One way of doing that is to insert as many empty records as are needed in the data source to pad out the data, as Doug suggested.

There is an alternative, however. If you use the method described below, it becomes a simple matter to insert however many empty cells are needed into the mailmerge output document (e.g. via the insertion of column breaks and/or by however many cells you need for the blanks then choosing Table Tools | Layout | Insert Above). Do note that this won’t be especially useful for using multiple left-over labels from a conventional label merge in one go, but it should be useful for printing single labels on such stationery and for any new stationery you use.

The required setup entails change the Print Order for the merge. Normally, Word Label merges output the records using an across-then-down arrangement, but the following process converts it to a down-then-across arrangement:

1. In the Labels dialog, click New Document to create a new document based on the label definition you have chosen.
2. Display table gridlines (Table Tools | Layout | Table | View Gridlines) and nonprinting characters (Ctrl+* or Home | Paragraph | Show/Hide ¶) so you can see what you're doing.
3. Delete all columns except the first and all rows except the first. If there is a spacer column between the labels, note its width beforehand.
4. On the Page Layout tab, in the Page Setup group, click Columns and choose More Columns...
5. In the Columns dialog, select however many labels across your label stationery had
6. In the Spacing box, set the width to the original spacer column width or, if it had none, 0
7. If necessary, set the column width to your label width.
8. Change the mailmerge type to 'Directory'.
9. Run the mailmerge.

macropod say:

As Stefan mentioned, Word uses the currently-active printer's driver for layout optimisation. This can result in the layout changing from one computer to the next. So too can switching between Word versions (due to layout engine differences). Margins, indents, etc. will not be affected by those factors, however.

What can affect margins, indents, etc. is having Word's 'Automatically update document styles' option set (see under Developer|Document Template>Templates). With that option set, any Styles you modify in the document on one computer will revert to the definitions used on the other one when next opened there.

It's also possible the document has acquired some of corruption. Corrupt documents can often be 'repaired' by inserting a new, empty, paragraph at the very end, copying everything except that new paragraph to a new document based on the same template (headers & footers may need to be copied separately), closing the old document and saving the new one over it.
