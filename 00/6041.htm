rory archibald say:

You could use some code to reset them. Something like this (untested):

Option Explicit
Type dlFormatting
    AutoText                  As Boolean
    NumberFormat              As String
    Position                  As XlDataLabelPosition
    Separator                 As String
    ShowBubbleSize            As Boolean
    ShowCategoryName          As Boolean
    ShowLegendKey             As Boolean
    ShowPercentage            As Boolean
    ShowSeriesName            As Boolean
    ShowValue                 As Boolean
End Type
Sub ResetChartDataLabels()
    Dim ws                    As Worksheet
    Dim cht                   As Chart
    Dim oChart                As ChartObject

    For Each cht In ActiveWorkbook.Charts
        ResetDataLabels cht
    Next cht

    For Each ws In ActiveWorkbook.Worksheets
        For Each oChart In ws.ChartObjects
            ResetDataLabels oChart.Chart
        Next oChart
    Next ws
End Sub
Sub ResetDataLabels(cht As Chart)
    Dim ser As Series
    Dim dl As DataLabels
    Dim dlf As dlFormatting
    For Each ser In cht.SeriesCollection
        If ser.HasDataLabels Then
            Set dl = ser.DataLabels
            With dl
                dlf.AutoText = .AutoText
                dlf.NumberFormat = .NumberFormat
                dlf.Position = .Position
                dlf.Separator = .Separator
                dlf.ShowBubbleSize = .ShowBubbleSize
                dlf.ShowCategoryName = .ShowCategoryName
                dlf.ShowLegendKey = .ShowLegendKey
                dlf.ShowPercentage = .ShowPercentage
                dlf.ShowSeriesName = .ShowSeriesName
                dlf.ShowValue = .ShowValue
            End With
            ser.HasDataLabels = False
            ser.ApplyDataLabels
            Set dl = ser.DataLabels
            With dlf
                dl.AutoText = .AutoText
                dl.Position = .Position
                dl.Separator = .Separator
                dl.ShowBubbleSize = .ShowBubbleSize
                dl.ShowCategoryName = .ShowCategoryName
                dl.ShowLegendKey = .ShowLegendKey
                dl.ShowPercentage = .ShowPercentage
                dl.ShowSeriesName = .ShowSeriesName
                dl.ShowValue = .ShowValue
                dl.NumberFormat = .NumberFormat
            End With
        End If
    Next ser
End Sub

I confirm your findings. It takes increasingly long for the task window to appear as there are more Outlook tasks already on the page. The logic behind this escapes me.

You might ask yourself why you use Outlook tasks at all? Personally, I use OneNote tags for my GTD tasks. I have tags like @NextTask@Home, @NextTask@Work, and so on, each of them represented by a checkbox symbol. A OneNote GTD task gets "promoted" to an Outlook task only if and when it gets a place in the calendar, i.e. once a date gets associated with it and it thus becomes part of what David Allen calls the "hard landscape".

Doing it like this has some advantages:

= You can have as many tasks as you like without having the performance problems that you experience.

= You can use the tags summary to generate extremely useful taks lists on the fly, grouped by tag type (GTD context) or section (area of interest).

= When a task is completed, just check the checkbox and you can save as many of them as you want for as long as you like.

= You keep your GTD stuff basically in one place, or tool. Outlook is used only to actually schedule a Next Action, so for calendaring.

 

Does this make sense? To me, it just feels more consistent to do it like this.

Give it some thought and maybe a try!

Good luck.

Find next highest value in table and output the cell next to it - Microsoft Community

totalbs say:

Find next highest value in table and output the cell next to it

I have a table with two columns, the first column is what I want to output and the second column contains areas that I want to find a greater value compared to my input. What I am trying to do is obtain an area by dividing one cell by another, and using this area I want to look into my table, find an area that is greater than what I just calculated, and then output the cell next to this greater area. VLOOKUP is not going to work and I can't seem to get INDEX MATCH to work. My formula is =INDEX(L5:M12,MATCH(I5/C5,M5:M12,-1),1) and I included a snapshot below. I want this formula to return "#6" since the area I want to be greater than is 2.88/8=0.36. I would prefer it to not require additional helper columns with intermediate steps that I hide.

paulcordts say:

The issue is with your MATCH function.  The -1 type requires the list to be in descending order, your list is in ascending order.  If you reverse the order of your list your formula would work as is

If you don't want to (or can't) reverse your list use this

=INDEX(L5:M12,MATCH(I5/C5,M5:M12,1)+1,1)

This will work, since it finds the position of the largest value less that or equal to the lookup value, then adds 1

One thing to watch out for with either case is what happens if the value is a exact match to your criteria.   What if #6 was .36?  Do you want the result to be #6 or #7 (the next greater)?

The formula I gave you would return #7 in this case, if you want the result to be #6 you need to check for an exact match like this

=INDEX(L5:M12,MATCH(I5/C5,M5:M12,1)+IF(ISERROR(MATCH(I5/C5,M5:M12,0)),1,0),1)

If you resort the list and use the -1 type the result would be #6.  If your want #7 you also need to check for an exact match

=INDEX(L5:M12,MATCH(I5/C5,M5:M12,-1)+IF(ISERROR(MATCH(I5/C5,M5:M12,0)),0,-1),1)
