stefan blom say:
Make sure that the last two pages are in a separate section. With the insertion point in the header (or footer, depending on where the page numbers reside), on the Header & Footer Tools Design tab, click to deselect the Link to Previous option (in the Navigation group). Now you can delete the contents of the header (footer).
 
Note that a document can have up to three different headers (footers), which is the case if you have enabled "Different first page" as well as "Different odd and even" in Page Setup (or the similar commands on the Header & Footer Tools Design tab); each type of header (footer) within the section must be separately unlinked from its "neighbor" in the preceding section.

Remove carriage return after TOC. - Microsoft Community

lloyddh say:

Remove carriage return after TOC.

When I add a a table of contents to a document Word adds an additional carriage return after the last item on the TOC. How can I get it so word will not add the carriage return?
rohn007 say:
It appears Word has been written to assume there is a heading line before the TOC itself.  IF there is no leading heading, Word includes a blank line for it. For example, the "blank" line between the first and second TOCs I inserted "disappeared" when I inserted a Heading between the TOC .

So, a "fix" you can use is to use a heading level that is not used elsewhere in the doucment, ie Heading 8 or 9.  Define it to be VERY small, say 1 pt text. Put one of these heading lines between each TOC you insert.

It's nice to be able to beat Word into submission, or at least trick it into doing something "right".

jay freedman say:
In my copy of Word 2003, a table of contents does not add an extra paragraph mark after the TOC -- there is just a single paragraph mark at the end of the last entry, followed by whatever was present before the insertion. If I put the cursor in paragraph that contains nothing except the paragraph mark, then that paragraph mark remains in place and empty. But if I start with the cursor at the left end of whatever is visible in the first paragraph, there's no extra paragraph mark.

The situation is different in Word 2007 and 2010 if you use the building blocks in the TOC gallery, which place the TOC inside a content control that does have an empty paragraph mark at the end. However, if you click the Table of Contents button and choose Insert Table of Contents from the bottom of the gallery to open the TOC dialog, that behaves just like the one in Word 2003.
