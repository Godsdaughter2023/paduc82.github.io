I have an OnOpen command on a form that is =Hdr()

I use it for many forms' OnOpen property. It was doing this:

Public Function Hdr()

The basic problem here is probably the fact that in a form's Open event, it is not yet the active form.  If you called the function from the Load event, it would probably work, though I haven't tested that.

Failing that, and if you don't want to create a separate event procedure in every form, you could pass the function a reference to the actual form as from which it's invoked, and use that form reference to qualify the reference to the control, like this:

In the form's OnOpen property:

    =fncGoToHdr([Form])

Note: that's the exact text; you're not supposed to substitute anything for "[Form]".

Then the function would be defined like this:

Public Function fncGoToHdr(frm As Access.Form)

    On Error Resume Next  ' in case there's no "Hdr" control
    frm!Hdr.SetFocus

End Function

I agree with RM that "Hdr" is not a great name for a control, and probably should be changed.

dirk goldgar say:

This works great but I'm not sure why it works and screen.activeform.Hdr.SetFocus does not...

Screen.ActiveForm returns a reference to the form that Access thinks is currently active.  When you open a form, Access does a number of things before making it the active form, and it's only after it has become active (so far as Access is concerned) that Screen.ActiveForm will return a reference to that form.  Up until that point, Screen.ActiveForm will return a reference the form that was active before you opened this new form, or if no form was active, it will raise an error.

When you open a form, the form's Open event fires, but at that point the form has not yet been activated, so Screen.ActiveForm won't return a reference to it.  It'll either return a reference to the previously active form, or raise an error if there wasn't one.

A form as a "Form" property that always returns a reference to that form.  An expression like "=fncGoToHdr([Form])" passes the value of the [Form] property, so the function receives a direct pointer to the calling form, even though it isn't yet the "active form".

jeeped say:

This may seem like a bandaid solution, but you could remove the number formatting before the copy action from the original document.

  1. Triple click inside a numbered paragraph to Select Paragraph, then click the Numbered List toolbar button to remove it from the numbering sequence.
  2. Press CTRL+C (Copy), then CTRL+Z (undo Remove Numbering).
  3. Go to the Target Document and Paste Special, Unformatted Text.

Alternately, just remove all numbering from a copy of the source document during your Copy & Paste operations.

Can't search for three dots in Word - Microsoft Community

johnniemccoy say:

Can't search for three dots in Word

I have to seach for all instances of three dots in a document (...) and replace them with space + three dots ( ...). But the search function can't find any three dot instances in the document even though there are tons of them. I figure it's because Word autoformats them - every time you type three dots followed by a space word pulls the dots a bit apart making more space between them. So I tried to copy three dots from the document and paste them into the search field but that can't be done either. What to do?

It's Office for Mac by the way.

lene fredborg say:

The characters are most likely Horizontal Ellipsis characters. One of the standard AutoCorrect entries replaces three periods with that character.

 

The character is ASCII 133 on PC but as far as I can find out, ASCII 201 on Mac (you need to check that yourself).

 

The description below if for PC but most likely almost the same on Mac:

 

  1. Open the Find and Replace dialog box. Make sure to clear any formatting added to the fields.
  2. In the Find what field, type:
    ^0133
    or if the number on Mac is 201:
    ^0201
    (NOTE: a carat + a zero must be added in front of the ASCII number)

  3. In the Replace with field, type the same with a space in front:
    Alternatively, type a space followed by ^& (^& means “the same as the Find what text”)

  4. Click Replace All (you may want to test a single replacement first).

johnniemccoy say:

Thanks! It worked.

suzanne s. barnhill say:

By the way  - why can't you copy words, signs, characters into the "find" field?

You can. Copy any text in the document and use Ctrl+V to paste it into the "Find what" box.  This certainly works for the ellipsis and also for most font characters (they may not display correctly but will be found correctly).

The keyboard shortcuts Ctrl+C (copy), Ctrl+X (cut), and Ctrl+V (paste) are universal across all Windows applications.
