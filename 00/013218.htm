graham mayor say:

Hmmmm. An interesting dilemma. Do I take it that you are not going to keep a record of which random 'number' is sent to which respondent? In which case the following macro(s) will create a random 10 digit alphanumeric string. As the strings are not being saved it is not possible to ensure that the numbers are truly unique, but it should serve the purpose.

The main macro will populate a content control named "RandomID".

The supplementary macro InsertCC will insert such a content control at the cursor and populate it using the main macro. You should get a 10 digit aphanumeric string.

http://www.gmayor.com/installing_macro.htm   

If you are mail merging the surveys to the recipients, then add the content control to the merge document and use http://www.gmayor.com/individual_merge_letters.htm  to merge the document to separate files. This add-in allows the running of a macro on each merge document. Copy the bold line to the macro window then run the function. It will create as many documents as you have records each with a 10 character random string in the content control.

Of course if you are keeping a record of which ID went with which survey for admin purposes, then you could simply create random numbers in your merge data source.

Sub AddUniqueID(oDoc As Word.Document)
Dim myRg As Range
Dim idx As Integer
Dim oCC As ContentControl
Dim GetUnique As String
Const CharList = "ABCDEFGHIJKLKMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789"
    Randomize
    For idx = 1 To 10
        GetUnique = GetUnique & Mid(CharList, 62 * Rnd + 1, 1)
    Next idx
    For Each oCC In oDoc.ContentControls
        If oCC.Title = "RandomID" Then
            oCC.Range.Text = GetUnique
            Exit For
        End If
    Next oCC
End Sub

Sub InsertCC()
    With Selection
        .Range.ContentControls.Add (wdContentControlRichText)
        .ParentContentControl.Title = "RandomID"
        .ParentContentControl.Tag = "RandomID"
    End With
    AddUniqueID ActiveDocument
End Sub

Assuming legacy form fields - run the following macro

Dim oFld As FormFields
Dim i As Long
Set oFld = ActiveDocument.FormFields
If ActiveDocument.ProtectionType <> wdNoProtection Then
    ActiveDocument.Unprotect Password:=""
End If
For i = oFld.Count To 1 Step -1
    With oFld(i)
        Select Case .Type
            Case wdFieldFormTextInput, wdFieldFormDropDown
                oFld(i).Range = oFld(i).Result
            Case wdFieldFormCheckBox
                If oFld(i).CheckBox.Value = True Then
                    oFld(i).Range = "X"
                Else
                    oFld(i).Range = "-"

You will have to break the code into its component parts in separate columns to the right of your existing data and then sort on the components individually.

 

If the first component is always 6 characters and sorts to your liking then make that the first decomposed column say this is in column H and the original code is in column A starting at A2.  Then H2 would be =LEFT(A2,6)

 

If the second component is always one numeric character surrounded by "-" then you need to make the second column, I2 say, contain =MID(A2,8,1)

 

If the third component always starts at character position 10 and may be one or more numeric characters followed by one alphabetic character then

J2 =VALUE(MID(A2,10,LEN(A2)-10))

K2 =RIGHT(A2,1)

 

Copy these 4 formulas down the length of the data

 

Then you can sort your entire data including the new columns on column H ascending then column I ascending, then column J ascending and then column K ascending.
