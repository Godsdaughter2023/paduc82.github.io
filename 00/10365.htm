No, manual compacting is not required as Outlook will do that automatically when there is a lot of unused space in the pst-file. Having a lot of active background processes or having your pst-file located on a network share (which is not supported anyway) can interfere with this though and then manual compacting is required. The BIG problem with the Navigation form thingie is that it only has one, single, lonely subform control so only one tab's subform can be open at a time. That means it's tricky to link one eubform to another the parent subform closes as you open the other subform.

I strongly recommend that you forget about using the Navigation form and avoid all that gobbledegook code to make it work. IMO, new things are not always better and change for the sake of change is a waste of time. I think you can do better a lot more easily by sticking with your old approach.

Or, if you need to change things around for a good reason, use a normal main form for the parent table with a separate subform control for each if the child forms. You can position the subforms on top of each other. You can then use simple code behind command buttons to make one subform visible and the others invisible. This way the command buttons can be arranged similar to the Nav form's tabs, but all the subforms will stay open and their values can be used to open other forms.

Another, probably even easier way is to put each subform on a separate page of a Tab Control and let the tab control take care of all that.

Post back if you need to have one subform link to another subform in either of the previous two approaches. It only takes a hidden text box and a line of code to make it work.

=SUM(IF(ISNUMBER(-RIGHT(Rng,1)),--RIGHT(Rng,1),0))
