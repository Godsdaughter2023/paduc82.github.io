dax1792 say:

It looks like you've got 4GB of RAM. The paging file could be up to 12GB. So you could gain from moving the paging file to the D:/ volume.

The paging file settings can be found as described here:

http://windows.microsoft.com/en-US/windows7/Change-the-size-of-virtual-memory

 

The built-in Windows partitioning tool is almost certainly not going to work in this situation, unless you can completely empty the D:/ partition. There are third party programs which might be able amalgamate the partitions.

 

The Used and Free Space shown on the drive properties are accurate. The size of folders shown by Explorer are not accurate for System Files. They can either be double counted for hard links or be omitted because of access permission issues. For example, file in System32 are hard linked to files in WinSxS so there is only one copy on disk but Explorer will count them twice when calculating the size of the Windows folder.

12GB is around average for WinSxS. It's not recommended to modify this folder. At least, it could affect the ability to apply updates. If you've not already done so, use the 'Clean up System Files' option in Disk Cleanup and delete any previous Windows installation and Service Pack backups.

 

To dig down into what is using the space use Treesize Free. There may be applications using space in your AppData folder.

How do I delete old compressed files? I have over 200,000. - Microsoft Community

ciro chris maurici say:

How do I delete old compressed files? I have over 200,000.

After running disc clean-up, I show over 200.000 compressed old files.  How do I delete them?
lemp say:
You delete a compressed file just the way you delete any other file, BUT ...

you may not really want to delete those "compressed old files."  By default, the Disk Cleanup tool in Windows XP will compress files that have not been used for 50 days.  You may well want to keep files that you haven't accessed in the past two months.

You can change the period that Disk Cleanup uses to determine if a file is "old" --> http://www.howtogeek.com/howto/windows/make-disk-cleanup-compress-olderor-newer-files-on-xp/

However, many people (myself included) just disable the scan for "old files" because it makes Disk Cleanup run much faster:  see, e.g., --> http://www.pctools.com/forum/showthread.php?49060-Disable-Compress-Old-Files-in-Disk-cleanup  Or http://www.kellys-korner-xp.com/xp_tweaks.htm Line 48 right side (Note that there are two links, one to disable "compress old files" and one to re-enable it; these reg files also change the way Disk Cleanup treats temporary files -- by default, temporary files accessed within the past 7 days are not deleted; the first of Kelly's reg files changes this to 0 days and the "undo" file restores the 7 day default).
