cmdrkeene say:

It's probably not you, but the Taskbar will remain shown if any program icon sends a "needy flag".  For example, the icons near the right side of the taskbar (the notification area near the clock).  If your antivirus icon is there and has an alert to tell you, the taskbar will remain visible until you have dismissed the alert icon.  This applies to every program that is there on the notification side, and also all running programs on the left side too.  If any program broadcasts a notice that it "needs the user to see me", the Taskbar will stay visible.

One way to combat this is to exit and stop all non-essential background programs from running.  For example, run MSCONFIG and disable everything on the Startup tab.

If you're curious for more discussion on the topic, look online for needy program taskbar.

"ST String.dll not found" is caused by an application named Dell DataSafe Local Backup software. If you have a Dell computer and the software installed, I would suggest you to repair the particular program.

 

Repair Dell DataSafe Local Backup

 

Note: The following steps are applicable if your computer manufacturer is Dell and you have this software on your computer running.

 

a.    Click on Start; Go to Computer.

b.    Browse to the following location “C:\Program Files\Dell DataSafe Local Backup.

c.    Locate the file STLog.dll in this directory.

d.    Copy the file (do not move or cut the file, as this will cause many more issues)

e.    Find the Scheduler folder under: "C:\Program Files\Dell DataSafe Local Backup\Components\scheduler".

f.     Paste the file in to this directory.

 

Also copy all of the following DLLs (Dynamic Link Library) to the same folder.

 

·         CSTError.dll

·         STString.dll

·         STXml.dll

·         STStringArray.dll

·         LibXml2.dll

·         zlib1.dll

·         STFiles.dll

·         STRegistry.dll

·         STPE.dll

·         STNLS.dll

·         STSystems.dll

Uninstall or change a program
