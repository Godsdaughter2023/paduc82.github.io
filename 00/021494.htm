stefan blom say:

a. user say:

> workaround

Does the \*CardText switch work with any other field type?

If so, and you need an immediate fix and it's just the { PAGE } fields, then you could nest the PAGE field inside a QUOTE field or perhaps an { = } field, e.g.

{ QUOTE { PAGE } \* Cardtext }

However, that can only work if the PAGE field, without switches, displays a numeric result. If it is currently formatted to display chapter-page numbering, or (say) A, B, C, you would still have to include *a* formatting switch inside the field. If, e.g. \* Arabic doesn't work either, you are basically left with using a \#0...

{ QUOTE { PAGE \#0 } \* CardText }

Other possible workarounds are even more nasty. e.g. you could use IF fields ({ IF { PAGE } = 1 "one" }{ IF { PAGE = 2 "two" } but you would need a lot of them. You could use VBA to create a whole bunch of document variables named P1, P2, P3 with values "one", "two" "three" etc., then use

{ DOCVARIABLE "P{ PAGE }" }

9. Click Save at the bottom of the page.
