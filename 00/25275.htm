jay freedman say:
The important setting to turn on is the ‘Copy remotely stored files onto your computer, and update the remote file when saving’. That avoids sending everything to the server on every save (and possibly also temp files during editing -- I'm not sure whether they stay local anyway).

The ‘Allow Background Saves’ option can stay checked. The term 'background' is often misunderstood -- it means that Word uses a separate thread to do the save operation and lets you continue editing, instead of disabling editing until the save is complete. It should have no effect on whether there's a lag.

The ‘Always create backup copy’  option also has no effect on any lag, but it's a good idea to keep it turned on. All it does is rename the existing file on the disk to 'Backup of <filename>.wbk' and then save the updated copy. It isn't really a backup, because it doesn't contain the latest changes, but it's better than nothing if the updated copy becomes damaged.

tom van stiphout say:

Yes there is. Something like this:

(in the Declaration section of the form)
private m_OCP as Long

(in Form_Load)
m_OCP = Me.OCP

(in Form_BeforeInsert)
Me.OCP = m_OCP

Note that this last event fires AFTER the first character is typed in the new record, presumably in some other member field.

scottgem say:

This is one of the advantages of an embedded subform. An embedded subform is linked to the main form and automatically populates the FK when a new record is added. If you use a separate, synchronized form, you have to do this yourself through code.

Tom's code will work assuming the Comments form is loaded from the Site form. It also assumes there is already a comments record for that OCP. 

An alternative is to use the OpenArgs argument when you open the form. You pass the OCP as the OpenArgs argument simply by referencing the the control bound to it. 

You can then use the same Before Insert event and say:

Me.OCP = Me.OpenArgs

stefan blom say:

You can delete multiple styles at once in the Organizer dialog box. (There are several ways to display the dialog box. For example, in the Styles pane, which you can apply by pressing Ctrl+Alt+Shift+S, click the Manage Styles icon, and then click the Import/Export button.) But, as you have found, you can't delete the built-in ones. You may find the article at http://gregmaxey.mvps.org/word_tip_pages/style_report_addin.html useful.

stefan blom say:
You have to install the macro by placing it in the Startup folder for your Word installation. See http://www.gmayor.com/installing_macro.htm.

Note that an entry such as "Bold, Italic, Line spacing: single" isn't really a style but direct formatting displayed as a style in the Styles pane. If you look at the very right of the pane, you'll see that there is no symbol indicating the type of style. See example below.



However, you can point to the right of the entry and click to get the popup menu. In particular, you may find Select All X Instance(s) useful, because it lets you apply different formatting quickly.
