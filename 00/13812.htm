Your first version uses R1C1 notation for the formula and only works when Excel is in R1C1 reference style; the second uses A1 notation and only works when Excel is in A1 reference style.

This command Disable-RemoteMailbox is not a “soft-deleted” behavior.

Ian,

Go to File > Options > Save group. What is the file format? What is the location? Do you have auto-save activated or not? If it is, turn it off and then put yourself on a conscious plan do periodically Save As back to the same file name (that's how I have always saved my files).

You don't need a template file to create annual holidays. All you need is a custom calendar, based on the Standard if appropriate, with all holidays or other non-working periods (e.g. summer shut-down). Once that is created it will be part of your Global and you can select it as the Project calendar for any new project you create.

Sometimes strange behavior like you describe is due to corruption in a view. Try saving your file as XML. Then re-open the XML file in Project. Note, you will lose any custom formatting with this process.

Use the Organizer to move your saved attributes to the new Global.

Hope this helps.

John
