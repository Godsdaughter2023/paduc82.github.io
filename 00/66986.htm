duncan lawler say:

Assuming your system doesn't contain a GPS (and the Lenovo Ideapad series does not appear to from the documentation), Windows location services will fall back to using other signals like nearby visible WiFi access points and general known IP address locations. Its possible that one of these methods is throwing your system off (someone with a WiFi router that was seen somewhere else and is now near you, or an IP address got re-assigned to your area). It's odd that the browser is getting the right location because both the browser and Windows maps use the same location services from Windows. Google may be using a different database of WiFi signatures or IP address reverse lookups, but Bing and Windows maps should be getting the same location at least.

One thing you can do from within the maps app is to change the default location it uses when no precise location is available.

From the maps app:

  • Open the settings pane by clicking the … in the upper right and choosing settings
  • Scroll down to the "Default Location" section and click the "Change Default Location" button

This should allow you to change or clear the default location the system will use to wherever you like.

The WiFi and IP database is continually being updated, so the problem might fix itself over time.

The other responses here aren't terribly relevant - the behavior of the location service has nothing to do with the user account. Changing the default location for the bing web property also won't impact what Windows location services reports to the map application.
