The following macro may be helpful:

Sub ResetAllNumberedLists()
'Macro posted by MVP Stefan Blom, January 2019
'Based on a similar macro
'originally posted by MVP Doug Robbins
Dim t As ListTemplate
Dim i As ListLevel
For Each t In ActiveDocument.ListTemplates
For Each i In t.ListLevels
    If i.NumberStyle <> wdListNumberStyleBullet Then
    i.Font.Reset
    End If
Next i

End Sub

The macro resets the font formatting for each list level of each numbered list to that of the underlying paragraph style.

The error message refers to the font size (presumably in the definition of the numbered-list styles). Some corruption may have changed the document to set that value to zero or to a very large number.

If you're referring to the macro solution in http://www.eileenslounge.com/viewtopic.php?t=4563, here are the steps to use it:

  • Press Alt+F11 to open the macro editor.
  • Right-click the Normal folder in the Project box on the left side, and click Insert > Module. (If there were already some modules in the Modules folder, make a note of the name of the new one; see the last paragraph below.)
  • In the large blank area on the right side, type the line

Sub Fixit()

and press the Enter key. A blank line and the line End Sub are automatically added.

  • Copy the code from the macro in your browser and paste it into the blank line between those two lines.
  • Open the document that has the problem.
  • Click in the code in the macro editor and press the F5 key (or click Run > Run Sub/Userform in the menu).

Now check the document to see whether the problem has disappeared.

To remove the macro once it's no longer needed, right-click the module that was inserted in the Normal folder in the Project box and click Remove <module name>, where <module name> may be "Module 1" or something like that.

If you’ve lost some recent notes you were working on, you may still be in luck. OneNote keeps a limited set of automatic backups by default.

Open the notebook where you lost recent notes.

In the Open Backup dialog box, note the folder names that appear. Each folder represents a OneNote notebook that’s stored on your computer.

Double-click the notebook folder you want to open.

Select the notebook section backup file you want to open, and then choose Open.

If your deleted notes are included in the backup sections or pages, right-click the section tab or individual page tab that you'd like to restore, and then use the Move or Copy command to put the notes back in their original location.

2. If you’re working on notes that were stored in OneDrive, do the following:

Open the shared notebook where you lost recent notes.

Choose History > Notebook Recycle Bin > Notebook Recycle Bin.

If your deleted notes are found there, right-click the page tab that you want to restore, and then use the Move or Copy command to put the notes back in their original location.

Any notes that were deleted during the past 60 days should appear in the Notebook Recycle Bin, unless the Notebook Recycle Bin was emptied during that time.

darthvaderismyboss say:

Changed properties of a Word 2010 form field to number, but still able to enter characters in the fields after protecting the doc. What am I doing wrong?

In Word 2010 I've created a text form field and edited the properties to "number" and the "length" to 2 digits. However, when protecting, saving, and  then testing the form, letters (more than 2) can be entered in the space. How do I restrict the form field contents to 2 number characters only?
suzanne s. barnhill say:

Assuming that by "text form field" you mean a Text Form Field from the Legacy Forms tools palette on the Developer tab, then the Text Form Field Options dialog has four boxes:

Type: Regular text by default

Default text: Blank by default

Maximum length: Unlimited by default

Text format: Blank by default

When you change the "type" to "Number," then "Default text" changes to "Default number" and "Text format" changes to "Number format." In this last box, you specify the format you require for the number. If you use ##, then users can enter 0, 1, or 2 digits. If you specify 00, then users are required to enter 2 digits. Results are as follows:

  1. Enter "ab" and tab to the next field: 00.
  2. Enter nothing and tab to the next field: field is blank.
  3. Enter 0 or 1 and tab to the next field: 00 or 01
  4. Enter 11 and tab to the next field: 11
