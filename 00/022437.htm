Excel maintains Last Modified timestamp incorrectly - Microsoft Community

fthynne say:

Excel maintains Last Modified timestamp incorrectly

There are many threads with questions about modification and creation dates and what seems to be strange behaviour. There is one matter that I do not believe Microsoft has answered correctly and rationally. All the threads I have found so far, even recent ones, are locked which I find disturbing.

There have been questions about file creation dates showing as more recent than modification dates and I accept that as being correct behaviour when a file has been copied. What I do not accept is that in some cases modification dates are not changed when modifications have taken place.

Many questions arise in connection with Excel, and the error is clearly displayed when a new Excel file is created, populated with some data and saved. When its folder is opened and refreshed in File Explorer, the Excel file has no Last Modified date and it size is shown as 0 bytes.

This cannot possibly be correct! If the file is subsequently opened and closed, the Last Modified date is changed and the size is correctly shown.

This means that the treatment of the Last Modified does not mean what it says and means something else. This is a perversion that appears to have persistent for many years and cannot be excused. Furthermore, if an Excel file opened and closed without a change, the Date Last Accessed is not changed although it should have been. That makes no sense either!
