If you just want to share documents or folders to external users, it is no need to create another site. Just use your team site. If you want to share specific libraries to external users and don't want them to see other libraries, you can create a subsite or separate for external users access.

No, there is no need to create a page view. To present existing libraries to external users, you need to share the site to them. Hello, Trevor. I see the same problem you described.

This should not be happening in Publishing Layout, but can be made to happen if you switched to another view and accidentally deleted a page break. Go to Publishing View and enable the sidebar (View > Sidebar > Thumbnail…). Click on page 1. Pages 1 & 2 are selected! Same thing with pages 6-7-8 and 11-12.

graham mayor say:

Unless you can access the code that writes to the bookmarks, the short answer is no.

If you can access the code that writes to the bookmark then you need to add code to unlock the form, write to the bookmark, then lock the form again.e.g. as follows:

Dim oDoc As Document
Dim bProtected As Boolean
Set oDoc = ActiveDocument
If oDoc.ProtectionType <> wdNoProtection Then
    oDoc.Unprotect Password:=""
    bProtected = True
End If
'Do your stuff here
If bProtected = True Then
    oDoc.Protect _
    Type:=wdAllowOnlyFormFields, _
    NoReset:=True, _
    Password:=""
End If

However if you can access the code it would be better to replace the use of bookmarks with docvariable fields and write the data to the document variables, then the form could remain locked e.g. for one document variable

Dim oDoc As Document
Dim oVars As Variables
Set oDoc = ActiveDocument
Set oVars = oDoc.Variables
oVars("varData").Value = "The data you wish to include"
oDoc.Fields.Update

Depending on how your document is laid out you may be able to have separate unprotected sections with your bookmarked locations from the sections with the form fields. The form would not need to be unlocked to write to bookmarks in unprotected sections e.g.
 
Dim rText As Range
Set rText = ActiveDocument.Bookmarks("Bookmarkname").Range
rText.Text = "This is the text to insert"
With ActiveDocument
     .Bookmarks.Add "Bookmarkname", rText
     .Fields.Update
End With
 

Generally it is not advisable to have protected form fields and editable text in the same document. A more elegant solution would be to provide a userform in place of the form fields to collect the variable data and to write it to document variables.  For the basics, see Word MVP FAQ - Userforms
http://word.mvps.org/FAQs/Userforms.htm

for a more in depth explanation, see
http://gregmaxey.mvps.org/Create_and_employ_a_UserForm.htm

OK, percentages 101:

40% is 40/100.  So 40% of 10 apples is (40 * 10apples )/100, or   0.4 * 10 = 4.

So a test score in percent is 100 x Actual_Score / Max_pos_score

In your example, 29 out of 40 = 72.5%

But since the max possible marks for the three tests (class test, cont and H-work) add up to 100, the scores are effectively weighted 1:1:1.

29+35+18 = 82 !!

But if each test was out of 50 max possible marks, and each test was weighted 40:40:20, you would use

=( (40*29/50) + (40*35/50) + (20*18/50) ) /100

If you replace the three 50's here with the actual max pos marks (40, 40 and 20 respectively), they cancel out and you end up with (29+35+18) out of 100, or 82%

 

To attain the final score for the year (coursework and exam) , you said coursework is weighted 40% and exam 60%.  So if student scores "k" marks out of 100 for coursework, and "m" marks out of 100 for the exam, you would use

Total score = (40*k + 60*m)/100

Does this make sense?
