naman r say:

Hello Ken,

Are you working on Domain network?

When a C:\Users\<username>\<folder> subfolder is shared using the shell UI, the C:\Users folder and its contents are shared as well. Additionally, when sharing is revoked on C:\Users\<username>\<folder>, the C:\Users folder remains shared. This behavior occurs in both Windows Vista and Windows 7.

For more information please check:

The C:\Users folder is shared when any sub folders from C:\Users\<username>\ is shared

When you share a drive over the network, the subfolders are also shared but in few cases you might have to take permission of the sub folder to get access as certain sub folders within the C:\ have important files.

If you are working on Domain network, you can post your question on the Windows 7 IT Pro forums where we have a dedicated team which can help you better.

http://social.technet.microsoft.com/Forums/windows/en-US/home?category=w7itpro

Hope the information provided helps. Do let us know in case you have any queries/issues while working with Windows.

Thank you.

gts-nj say:

You are correct. When you set sharing on the root it does not share all folders. 

Note that the vendor is probably wrong in saying that will prevent their software from working. Once shared it can be accessed as [mapped drive letter] or \\compname\folder despite the fact that a separate share name exists. 

  I recently installed some tax software for a customer with a very similar issue. The program has to be run as [X:]\progfolder\executable where X is a letter mapped to the root of the host.
