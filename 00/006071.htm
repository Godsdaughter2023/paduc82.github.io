    Carriage return inserted with merge field text - Microsoft Community

    trish_c say:

    Carriage return inserted with merge field text

    When using "includetext" to insert language from another document, I keep getting extra carriage returns in my document in the middle of the paragraph.  How do I get it to merge in a paragraph without inserting that carriage return.
    doug robbins - mvp office apps & services (word) say:

    I just tested it here with a mail merge main document that included the following field construction:

     

    { INCLUDETEXT "{ MERGEFIELD text }" Test }

     

    where text is the fieldname in the data source that contains the path\filename of the file

     

    C:/Users/Doug/Downloads/AAA/Aveen/Test Doc.docx

     

    and Test is the name of the bookmark in the document Test Doc that defines the text that is to be inserted.

     

    If the Bookmark varied from record to record, you would use

     

    { INCLUDETEXT "{ MERGEFIELD text }" { MERGEFIELD Bookmark } }

     

    where Bookmark is the name of the field that contains the name of the bookmark applicable to each record.

     

    Both of the above work here.

    daniel pineault say:

    Bob,

    It has to do with the impersonationLevel ..., instead try:

    Function ProcessorID() As String
    On Error GoTo Error_Handler
        ' Using late binding to avoid a Reference problem
        Dim strComputer      As String
        Dim objWMIService    As Object
        Dim colItems         As Object
        Dim objItem          As Object
       
        strComputer = "." 'Get info for the current computer
        
        Set objWMIService = GetObject("winmgmts:\\" & strComputer & "\root\cimv2") 
        Set colItems = objWMIService.ExecQuery("SELECT * FROM Win32_Processor", , 48)
       
        For Each objItem In colItems
            ProcessorID = Nz(objItem.ProcessorID, 0)
        Next

    Error_Handler_Exit:
        On Error Resume Next
        Set objItem = Nothing
        Set colItems = Nothing
        Set objWMIService = Nothing
        Exit Function

    Error_Handler:
        MsgBox "MS Access has generated the following error" & vbCrLf & vbCrLf & "Error Number: " & _
        Err.Number & vbCrLf & "Error Source: ProcessorID" & vbCrLf & "Error Description: " & _
        Err.Description, vbCritical, "An Error has Occured!"
        Resume Error_Handler_Exit
    End Function

    Also, with this code what happens if the PC has multiple processors?  The value of the previous processor id will be overwritten.  You may wish to modify the 'For Each objItem In colItems' Loop a little.

    jeeped say:

    Dates and Times are kept as numbers by Excel with Dates being 1 for every day past Dec 31, 1899. Times are a decimal portion of a day. Today happens to be 41,705 and today at noon is 41,705.5.

    With a day composed of 24 hours, simply multiply your time value by 24 to receive the appropriate hours as an integer (with a possible decimal portion representing minutes).

    16:00:00 × 24 = 16
