I'm afraid you're probably out of luck. Up to and including Word 2007, AutoSave is intended only to be recoverable in case of a program crash. If you shut down the program normally, part of the shutdown process is to delete the AutoSave file.

Try this instead:

  9. Save each of the open projects.

john - project say:

Mary,

First of all, it is a bad idea to have any links to summary lines, particularly external links (Ref: http://social.technet.microsoft.com/wiki/contents/articles/29569.project-linking-andor-assigning-resources-to-summary-tasks.aspx).

Third, how did you create the external link?

There are multiple ways to create inter-project links but perhaps one of the best is the following:

1. Open a new blank project and insert each of the projects you wish to have linked as subprojects using Project/Insert/subproject.

2. Find the predecessor task and the successor task in the master. This may be facilitated by setting a flag and filtering so only those two tasks appear in the view.

3. Grab the center of the Gartt bar of the predecessor task and drag to the center of the Gantt bar of the successor task. This establishes the link between the two tasks.

4. Repeat for each desired external link pair.

5. When done, save each subproject but do not save the master

Bear in mind that creating a linked structure in Project opens the possibility of corruption. You must never rename, move, overwrite or save off any of the files in a linked structure. All files should be located in a single folder on a local drive and not operated over a network.
