Sub CopyPaste()
Dim ws As Worksheet
Dim r As Long
Set ws = ActiveSheet
r = ws.Cells(Rows.Count, "C").End(xlUp).Row
On Error Resume Next
ws.Range("A1").Copy
ws.Cells(r, "C").Resize(Range("B1").Value).Offset(1).PasteSpecial xlPasteValues
Application.CutCopyMode = False
If Cells(1, "C") = "" Then Cells(1, "C").Delete
On Error GoTo 0
End Sub

To run the code

1) press ALT+F8 to show the Macros window

2) select the macro "CopyPaste"

3) press Run

ken sheridan say:
Can I get the value from the report?

That would be a chicken and egg situation, surely?

 

If you are going to reference a control of a bound form with:

 

[Forms]![Part BC Label by Part Number].[OnHand]

 

then you would need to move the form's record pointer to the next record each time before referencing it, e.g.

 

    Dim intOnHand As Integer

 

    With Forms![Part BC Label by Part Number]

        DoCmd.GoToRecord acForm, .Name, acFirst

        Do While Not IsNull(.PartNo)

            intOnHand = .OnHand

            DoCmd.PrintOut , , , acHigh, intOnHand, 0

            DoCmd.GoToRecord acForm, .Name, acNext

        Loop

    End With

 

Note that the form would need to be able to go to an empty form once it had stepped through the existing records to allow the PartNo control to be Null.  Its recordset would therefore need to be updatable.  If not, you would need to trap the error when an attempt is made to move beyond the last record and terminate the procedure.

 

I would still be inclined to print the whole job as a single grouped report using Duane's approach, but I see no reason why the above should not work.
