srimadhwa b say:

Hi Wazniak,

Welcome to Microsoft Community and thank you for posting the question. I see that you want to know the effect of deleting Trusted installer Files. Let me help you with the issue.

 

I would like to let you know that TrustedInstaller.exe is a Windows Module Installer service which is part of Windows Resource Protection. Windows Resource Protection (WRP) is a technology that restricts access to certain core system files, folders, and registry keys that are part of the Windows 7 installation. WRP prevents files with .dll, .exe, .ocx, and .sys file extensions from being modified or replaced.

Protecting these key resources is important to overall system stability, and, as such, they can only be modified by the Windows Module Installer service (TrustedInstaller.exe). If someone with administrative rights attempts to modify or replace a file that is protected by WRP, he will be presented with the message "Access Denied".

If you change Trusted Installer settings, you put the system at risk and your system may not function properly. It’s not suggested to remove it.

krzemien say:

Just for the benefit of all: the culprit was/is HP Omen Control and/or HP System Event Utility software that come with my machine.

OMEN PC series 870 come with LED lights on the front panel that can be controlled via the above. Settings allow changing the colours and either have them static or dynamic - the latter can be just colourful show, or LEDs can be driven by CPU utilisation or temperature.

And that's exactly where this problem arises: when I used CPU utilisation or temperature, I did see increased MsMpEng.exe utilisation. I eventually decided to abandon such settings and chose static colour - and issue has gone to sleep ever since.

Furthermore and beside the above: when in dynamic mode, both the above apps do seem to be responsible for numerous and quite random BSODs (some of them do happen when users swap accounts). Again, once LEDs are set to static, PC seems definitely stable.

I sincerely believe that HP is aware of the above - you can see that front LEDs in the next family of OMEN PCs (series 880) are now static and cannot be controlled anymore...
