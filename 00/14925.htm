Option Explicit

Private Sub Worksheet_Change(ByVal Target As Range)
  Const Delimiter = ", "
  Dim ThisValue, LastValue, NewValue
  Dim i As Integer, k As Integer
  'Only if one cell is changed
  If Target.Count > 1 Then Exit Sub
  'Only if the cell has a data validation
  If Target.Validation Is Nothing Then Exit Sub
  'Only if the data validation is a list
  If Target.Validation.Type <> xlValidateList Then Exit Sub
 
  'Get the current value
  ThisValue = Target.Value
  'Do nothing when empty
  If IsEmpty(ThisValue) Then Exit Sub
 
  'Events off
  Application.EnableEvents = False
  'Undo the changes and get the last value
  Application.Undo
  LastValue = Target.Value
  If IsEmpty(LastValue) Then
    'Restore the current value
    Application.Undo
  Else
    'Split the values
    LastValue = Split(Target.Value, Delimiter)
    'Make a copy
    NewValue = LastValue
    'Search the entered value
    k = LBound(LastValue) - 1
    For i = k + 1 To UBound(LastValue)
      If StrComp(ThisValue, LastValue(i), vbTextCompare) = 0 Then
        'This item will be removed
      Else
        k = k + 1
        'Copy to the front if necessary
        If i > k Then NewValue(k) = LastValue(i)
      End If
    Next
    If k = UBound(LastValue) Then
      'Add the value
      k = k + 1
      ReDim Preserve NewValue(LBound(NewValue) To k)
      NewValue(k) = ThisValue
    Else
      If k < LBound(LastValue) Then
        'Remove all
        NewValue = Array()
      Else
        'Remove the value
        ReDim Preserve NewValue(LBound(NewValue) To k)
      End If
    End If
    'At this point you can sort the array if you like
    InsertionSort_Prim NewValue
    'Store the new values
    Target.Value = Join(NewValue, Delimiter)
  End If
  Application.EnableEvents = True
End Sub

Private Sub InsertionSort_Prim(ByRef Liste)
  Dim i As Long, j As Long, Temp
  For i = LBound(Liste) + 1 To UBound(Liste)
    Temp = Liste(i)
    For j = i - 1 To LBound(Liste) Step -1
      If Liste(j) <= Temp Then Exit For
      Liste(j + 1) = Liste(j)
    Next
    Liste(j + 1) = Temp
  Next
End Sub

Using CompactDatabase AND FileCopy to backup an .mdb file

I am working on an Access 2003 application not built by me. Would one do a CompactDatabase AND a FileCopy to backup a file to the same location? If yes, can you please explain the significance of doing both?  I have never used both in a module...only used one or the other.

djsteele, mvp say:

Despite what Paul says, I can see legitimate reasons for backing up to the same folder.

First, recognize that this will only work if:

  1. You've split the application into a front end (containing the queries, forms, reports, macros and modules) linked to a back end (containing the tables and relations)

  2. No one is currently connected to the back end database.

  3. All you want to back up is the back end database.

Rather than use FileCopy, use the Name statement to rename the existing back end to something else, then compact that renamed file to the "proper" name:

Dim strOriginalFile As String
Dim strBackupFile As String
Dim strLockingFile As String

  strOriginalFile = "E:\Folder\Backend.mdb"
  strLockingFile = "E:\Folder\Backend.ldb"
  strBackupFile = "E:\Folder\Backend_" & Format(Now(), "yyyymmddhhnnss") & ".bak"

  If Len(Dir(strLockingFile)) > 0 Then
    MsgBox "Sorry, " & strOriginalFile & " is in use."
  Else
    Name strOriginalFile As strBackupFile
    DBEngine.CompactDatabase strBackupFile, strOriginalFile
  End If

julie sheets say:
Sorry for the delay getting back to you.  Task Type on manually scheduled tasks is disabled because
Setting a task type to fixed duration may not mean what you think it does.  The effect of setting task type simply 'fixes' that particular value in the relationship between work, duration, and assignment units.  A Fixed Duration task type means that Project will alter assignment units if you change the Work assigned to a resource -- it will not change the duration. 

In essence, manually scheduled tasks work the same way.  For example, enter start date on a manually scheduled task of 27 June 2011 and an end date of 1 July 2011.  Project will show 5 days in duration.  Assign a single resource at 100% and project will calculate 40 hours of work.  If you now change the work to 32 hours, Project will maintain the same start and end dates and simply spread the 32 hours of work out over the 5 days.

So, let us know more what you expected and why you need to change manually scheduled tasks to Fixed Duration and we'll try to assist.
