I have used Publisher to produce HTML and websites over the years and in fact still produce some with Pub 2000. Among other things I have learned is that Publisher produces different code for different browsers and different browsers render the code in different ways. It was more compatible with IE than Firefox and the other browsers, though you could tweak the layout and design of the pages to make them more cross browser compatible. For example it used to be that if you grouped any design elements on a page, or used a border around a text box, those elements would be fine in IE but rendered as images in FF while killing any hyperlinks.

Publisher has never produced standards compliant code and instead produces code in quirks mode to maximize compatibility. Further more, different versions of Publisher use different coding engines. All versions of Publisher were designed to produce a static fixed width page. And in all cases Publisher would occasionally choke on trying to convert 'print' page formatting and layout to HTML. In many cases there simply was no HTML equivalent to the print formatting. As of Pub 2010 MSFT gave up and started deprecating the web building capability of Publisher and I don't know how much is left as of Pub 2016 ( I quit installing Pub after Pub 2013.)

The bottom line is that if you are using Publisher to generate or even to view HTML code you are going to get mixed results depending upon which browser you use as your default browser and which browser you use to view any code generated.
