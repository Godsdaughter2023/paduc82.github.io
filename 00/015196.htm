Shortcut keys changed in Office 2013?

I installed MS Office 2013 Pro Plus, and when I opened a document in word, to my surprise I found out that some key are changed, like save is CTR+F12 and Undo is ALT+BACKSPACE. How can I get the default key combinations back (the save should be CTR+S, and undo should be CTR+Z)

jay freedman say:
The default keyboard shortcuts in Word 2013 are no different from those in previous versions. The only things that should change that are (a) installing 2013 on a computer that had a previous version with customized shortcuts stored in Normal.dotm, which will be used to create the 2013 Normal.dotm; (b) installing an add-in that contains nonstandard shortcuts; or (c) manually changing the shortcuts in 2013.

Whatever the cause, you should be able to restore or change any shortcuts by going to File > Options > Customize Ribbon, clicking the Customize button at the bottom left of the dialog, and entering the correct shortcuts.

You'll see in that dialog that some commands have more than one shortcut. For example, Undo (the command for which has the full name EditUndo) has both Ctrl+Z and Alt+Backspace as default shortcuts; if Ctrl+Z has been reassigned to some other command, then Alt+Backsapce will still be a shortcut for Undo.

A tip: Press Ctrl+Alt+<number pad plus sign> so the cursor changes to a cloverleaf shape, and then press a shortcut such as Ctrl+Z. The Customize Keyboard dialog will open and show the command to which that shortcut is currently assigned. That might provide a clue to what caused the reassignment.

mariostoilov say: Well nothing shows up when I press CTR+Z. When I tried to press the reset button nothing changed. Since these setting are stored in a file, can I get a link to a default normal.dotm so I can switch it with mine?

jay freedman say:
There is no link to a default normal.dotm. Instead:
  • Locate the normal.dotm file: Press Alt+F11 to open the Visual Basic for Applications window, press Ctrl+G to open the Immediate window at the bottom right, type or paste the command Print NormalTemplate.FullName, and press Enter. Make a note of the folder path that appears. You can now close the Visual Basic window and close Word.
  • In the file explorer, go to the folder, select the Normal.dotm file, and rename it to something like OldNormal.dotm.
  • Start Word. Because there is currently no file named Normal.dotm, Word will create a new one from the defaults stored in the program code. (Technically, the new file won't exist until you make some change that needs to be stored in it.)

If you had styles or macros stored in the OldNormal.dotm, you can use the Organizer to copy them to the new Normal.dotm. To access the Organizer, press Alt+F8 to open the Macros dialog and click the Organizer button there.


If there was nothing in the OldNormal.dotm that you need to keep, you can delete the file.
