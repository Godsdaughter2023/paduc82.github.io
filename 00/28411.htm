john mcghie say:
Hi Gato Girl:

This takes a lot longer to describe than it does to do.  And you have to do it only once.  Make yourself a junk document you can practice in (never practice on the original!).

The first thing you need to know is that a Word document is a three-layer sandwich.  The layers are "The Text", "Behind the text" and "In front of the text".

Before you can begin, you need to get the screen dump "behind the text".

To do this, click the screen dump, then right-click and choose "Wrap Text".  You can choose any of the options EXCEPT "Inline with text".  For full-width screen dumps, I suggest you choose "Top and Bottom".  For pictures less than full width, choose "Tight".

The picture may move, if so, drag it back where you want it.

Now you can get the lines and circles to go in front of the picture.

On the Home tab, look for the Insert chunk, and in there find the "Shape" icon.  Click that.

In the Basic Shapes folder, choose the top left icon, which is a filled circle.  The cursor will turn to a cross.  Use the cross to drag diagonally to create a circle of appropriate size.  It doesn't metter "where", we will drag it into position later.

Go to the Format tab, and set the Fill to "None".  Then set the "Line" to the colour you want to use.  Set the line "Weight" to something appropriate (try 1 point).

Now go back to the Shape button, and this time choose "Lines and Connectors".  Choose a straight line or an arrow, and draw it to be the length you want.  Again, go to the Format tab and set the colour and weight.  You can use the Format Painter to get the properties of the circle and the line to be the same.

To get the line and the circle to join, on the Format tab, go to the Arrange chunk and click the Align button.  At the bottom, choose the Grid Options item.  In Grid Options, under Snap Objects at the top, enable "To other objects" and DISable "To grid..." (both of them).

Now if you drag the line to the circle, it will "snap" into place giving you a clean join.  Fiddle with this until the line and the circle are the correct size, shape, and angle.

Now select both the circle and the line, (click, then shift-click the other).

On the Format tab, click the Group button and choose "Group".  This welds them together so they behave as a single item.

Now drag the combination into place.  If it begins to disappear under the picture, stop dragging and right-click it.  Choose "Arrange" then "Bring In front of text".  There are actually lots and lots of "layers": about 99 levels behind the text, and the same number in front.  So when you get good at this, you can do tricky "This behind that infront of this under that" sorts of arrangements.  As soon as you "Group" two or more objects, the Group becomes a single layer.  The group will all be on one layer, which you can then bring forward or back as required.

Leave them selected, and go to the Insert menu (this one is not on the ribbon) and choose "AutoText>New".  Type a name -- e.g. "CircCallOut" (no spaces) and OK.  That has saved the whole construction so next time you want one, it's just Insert>Autotext and choose the name to get one exactly the same.  This way you only have to do all the work once.  Make the names short and easy to remember: you will end up with five or six different things in there: circles, arrows, boxes, call-outs, etc.

You can read the Word Help item "Have Word complete your typing" for more.

Each time you insert one, it will go into the position the original was in when you saved it.  Drag it where you want it.

Make sure your screen dump is "floating" (i.e. wrap is set to something other than "Inline with text") before you insert one, or it will replace the picture.

Hope this helps

How can I flip a worksheet horizontally. Left to Right?

lars-Åke aspelin say:
I want to flip a worksheet horizontally. Left to Right?

Try this macro:

Sub flip_worksheet()
  first_col = Selection.Column
  ncol = Selection.Columns.Count
  last_col = first_col + ncol - 1
  For i = 1 To ncol - 1
    Columns(last_col).Cut
    Columns(first_col + i - 1).Insert
  Next i
  If ncol < 2 Then MsgBox "Please select all columns you want to be included in the 'flip'!"
End Sub

Note: This macro will not work if you have formulas including ranges with more than one column, e.g. =SUM(A2:Z2).  Other formulas, like =SUM(A2:A99) will however be translated correctly.

joeu2004 say:

First, using VBA Rnd requires the use of VBA Randomize at some time (preferably not every call) in order to ensure a new

Also, VBA Rnd is not as good a randomizer as Excel RAND and RANDBETWEEN, at least since Excel 2010.

Alternative:

Function randOnce(r As Range) As Double
randOnce = [ rand() ]
End Function

[Rand()] is shorthand for Evaluate("rand()").

And for RANDBETWEEN:

Function randbetweenOnce(r As Range, lo As Double, hi As Double) As Double
randbetweenOnce = Evaluate("randbetween(" & lo & "," & hi & ")")
End Function

"Lo" and "hi" are type Double in order to ensure compatibility with Excel RANDBETWEEN; for example, when "lo" and "hi" are not integers, or they exceed 9,007,199,254,740,992(!).

I do not use WorksheetFunction.Randbetween because that seems to make the UDF "volatile"(!).

Keep in mind that calls to these functions might be recalculated (new random values) at other times; for example, whenever Excel decides to recalculate all formulas (e.g. pressing ctrl+alt+f9), and whenever VBA is "reset" implicitly as well as manually.

scottgem say:

So the relationship between the limit and usage is 1:many? 

Create Query:

SELECT ClientID, SUM(Usage)

FROM table

GROUP BY ClientID

Call it qryUsage

Then create a second query

SELECT ClientID, Limit, SumOfusage, Limit-SumofUsage As Balance

FROM limittable, qryusage

INNER JOIN ON Limittable.ClientID = qryUsage.clientID

WHERE Balance>0;

ken sheridan say: To return a running balance per usage transaction per client in date order with a closing balance per day do something like this:

SELECT [Client Limit].[Client Name], [Medical UsageDate], [Medical Usage],
[Medical Limit] -
    (SELECT SUM([Medical Usage])
     FROM [Client Usage] AS CU2
     WHERE CU2.[ClientID] = CU1.[ClientID]
     AND CU2.[Medical UsageDate] <= CU1.[Medical UsageDate])
AS [Balance]
FROM [Client Limit] INNER JOIN [Client Usage] AS CU1
ON [Client Limit],[ClientID] = CU1.[ClientID]
ORDER BY [Client Limit].[Client Name], [Medical UsageDate];

If you wish to return a running balance per usage transaction per client in date order with a closing balance per transaction where there might be more than one transaction per client per day, you'd need to bring the primary key of Client Usage into play as a tie breaker, as described in my reply to your earlier thread and illustrated in my Balances demo, to which I gave you a link.
