elhernan say: From the thread I linked to:

If you want the session to use the default keyboard layout stored in the user profile instead of the layout provided by the Terminal Server client after the user logs on to the Terminal Server computer, you can set the IgnoreRemoteKeyboardLayout registry value to 1. After users first connect to the Terminal Server computer, configure the default keyboard layout and input language that you want, and then log off. All later user logons will use the default keyboard layout and input language from the profile.

 

To do so, please perform the following steps: 

 

1.            On the terminal server, click Start, click Run, type regedit, and then click OK.

 

2.            Locate and then click the following registry subkey:

 

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Keyboard Layout

 

3.            On the Edit menu, click Add Value, and then add the following registry information:

 

Value name: IgnoreRemoteKeyboardLayout

Data type: REG_DWORD

Value data: 1

 

4. Quit Registry Editor.

 

After you first connect to the Terminal Server computer, configure the default keyboard layout and input language that you want, and then log off. All later user logons will use the default keyboard layout and input language from the profile.
