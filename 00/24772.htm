The thing to remember about Word documents is that they comprise a number of different story ranges that make up the document. Imagine a pile of transparencies, each with part of the document. You may be able to use a frame or a table cell, instead of a text box - both of which are in the text layer of the document.

hal hostetler - mvp-outlook say:

Norton, like most programs, used to offer a "Custom Install" where you could select what features you wanted to install.  In this case, you would select to install everything BUT the email scanning parts.  Disabling Norton's email scanner has historically not helped; in spite of the fact that it indicates it's not running, it still is. The rest of the article can be found here:

http://sitetechsupport.com/inbox-full.html

In a nutshell, email scanning is redundant because of the way email works.  ANYTHING and EVERYTHING that is sent through email MUST be composed of printable characters.  Any form of executable code MUST contain non-printable, binary characters; pictures work this way, too.  So we have a dilemma when it comes to attachments; we need to be able to send them, but we cannot when they're in their original, binary form.  The solution is to encode the binary characters with printable characters so we can send the attachment, then decode the printable characters back into the binary form they started with, since the printable characters cannot execute or be viewed as a picture.  This is what happens when we open an attachment - the email program grabs the block of printable characters, starts decoding them, and writing them to a temporary file on the hard disk.  The instant this process begins, the resident file system scanner becomes intimately involved and watches each and every character as it's being written.  If the attachment is malware, the instant the scanner identifies it, it grabs the file and either quarantines or deletes it.  In short, so long as you keep your virus definitions up to date, the resident file system scanner will FULLY protect your email, no email scanner required.
