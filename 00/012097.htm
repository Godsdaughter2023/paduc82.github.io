Here's my Summary:

(Yes -- I finally got it working on my Samsung i780).

The problem with all of this is there are many steps to take and many errors to make. But the program is working well, I assure you, probably with any Windows for Mobile phone.

If you want to remotely access your computer from your mobile phone, take the following steps in this precise order:

  1. Make sure you can connect over the internet to your PC or server from another PC, using the Remote Desktop Connection program on a Windows PC (in this way you know the correct settings for later use with your mobile phone)
  2. Download the Cab file to your PC (Searching for WM6RDP.cab in Google)
  3. Transfer it to your Mobile with some synch or file tranfer program
  4. Install it in your Phone Memory (do NOT install it into your Memory card - it will not work!)
  5. Connect your mobile phone to a WiFi
  6. Run Start > (...) >WM6 Remote Desktop on your Mobile
  7. Enter the same settings you were also using under Step 1
  8. Hit Connect ... YES!!!

I use WM6 Remote Desktop to be able to access my server (www.12manage.com, a free management encyclopedia) to be able to reset it if it goes down for some reason.

I will be eternally grateful if someone can help me out on this one? Of course, when you are on holiday for example there is no hot spot or WiFi available.

 

Anyway, I hope my little summary will help you to get started more quickly. Have fun with WM6RD!

[Coff, coff]

On 64-bit systems the pointers are double words, means in VBA LongPtr instead of Long.

And to determine if we run on 64-bit Office we must use Win64, not VBA7. Also 32-bit Office versions can run VBA7.

That is an important point, not for the declaration itself, because a LongPtr on 32-bit VBA7 is declared as Long (internally in VBA). But the code, especially when you deal with RtlMoveMemory, is different!

#If Win64 Then

Private Declare PtrSafe Function SetWindowText Lib "user32" Alias "SetWindowTextW" ( _
    ByVal hWnd As LongPtr, ByVal lpString As String) As Long
Private Declare PtrSafe Function MessageBoxU Lib "user32" Alias "MessageBoxW" ( _
    ByVal hWnd As LongPtr, ByVal lpText As String, ByVal lpCaption As String, _

stefan blom say:

You have to add a continuous section break at the beginning and end of the file you want to insert. (In Word 2007, use Page Layout | Breaks | Continuous to add a section break at the insertion point.) When you are done, save and close the file.

Open the target document, and insert two section breaks (next page): one at the beginning and one at the end of the range where you want to insert your file. Click where you want to insert the file, click Insert | Object | Text from File, locate your source file, and click Insert.

For "everything" on section breaks in Word, see http://word.mvps.org/faqs/formatting/workwithsections.htm. Note that the article hasn't been updated for the most recent versions of Word, but the general principles still apply.
