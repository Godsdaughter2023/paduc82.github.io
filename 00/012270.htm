charles kenyon say:

Basic level user word 2010


First, there is no basic  level user solution to this problem. You'll need to become an intermediate level user, at least in this area.

Second, unless you are concerned with document / template size, instead of includetext and other files, consider putting your content into AutoText - a Building Block. If you do, you could use an AutoText field instead, or simply actually insert the text with the AutoText command. AutoText has to be stored in a template, it can't be stored in a document. The resulting text, though, can end up in your document based upon your template.

Third, you'll need to write a number of macros. You could then create a macrobutton field to trigger the macro. For an example of macrobutton fields that work with AutoText and macros, see the Checkbox template on my site. This is writing macros, not recording them.

The basic macros are simple. You would need one pair for each option you want to insert, with corresponding AutoText entries.

Option Explicit
Sub CheckIt()
   oAutoTextChecked.Insert Where:=Selection.Range
End Sub
Sub UncheckIt()
   oAutoTextUnchecked.Insert Where:=Selection.Range
End Sub

The AutoText entries here actually are macrobutton fields. The text that can be included with these is limited to one line.

To include much more text would be easy, it would simply not all be in the macrobutton field. However for the uncheckit macro to work (deleting the text) the text would have to be marked with a bookmark or otherwise marked.

Possibly a better solution would be to create a UserForm (a vba construct) that queries your user and pulls in information to assemble your document.

As you can see, this isn't a simple process, but it can be done. If you want to attempt it, write back and I or someone with better skills will help you.

Word 2016 adds an _1 to each bookmark when it changes for all new references to that bookmark. Old references still use the original without the _1 added to the end.

Word doesn't come with a tool for reporting what all the cross-references to bookmarks, etc. are and I doubt that you need them; they're not the kind of thing you'd edit and they can all be updated by Ctrl-A, F9, for example, in each story range (document body, headers, footers, etc.). I don't use Word 2016, so I can't comment on why it might add _1 to edited bookmark ranges; doing so doesn't make sense since the original bookmark still applies to the range.

Option Explicit
Sub CalendarExceptions()
'Basic macro code created by Kiran.K and posted on MSDN Project
' customizing and programming forum Feb 7,2013
'Code streamlined and updated by John - Project June 2,2014

Dim MyXL As Object
Set MyXL = CreateObject("Excel.Application")
Dim i As Integer, j As Integer
Dim E As Exception
Dim r As Resource
Dim xlRng As Range

'open Excel, define workbook, and set column headers
MyXL.Workbooks.Add
MyXL.Visible = True
MyXL.ActiveWorkbook.Worksheets.Add.Name = "Exception Report"
MyXL.ActiveWorkbook.Worksheets("Exception Report").Activate
Set xlRng = MyXL.ActiveSheet.Range("A1")
xlRng.Range("A1") = "Proj Cal Holidays"
xlRng.Range("B1") = "Start Date"
xlRng.Range("C1") = "Finish Date"
xlRng.Range("E1") = "Res Name"
xlRng.Range("F1") = "Res Base Cal"
xlRng.Range("G1") = "Base Cal Excep"
xlRng.Range("H1") = "Start Date"
xlRng.Range("I1") = "Finish Date"
xlRng.Range("K1") = "Resource Name"
xlRng.Range("L1") = "Res Excep"
xlRng.Range("M1") = "Start Date"
xlRng.Range("N1") = "Finish Date"

'First gather and export Project calendar exceptions
i = 2
If ActiveProject.Calendar.Exceptions.Count > 0 Then
    For Each E In ActiveProject.Calendar.Exceptions
        xlRng.Range("A" & i) = E.Name
        xlRng.Range("B" & i) = E.Start
        xlRng.Range("C" & i) = E.Finish
        i = i + 1
    Next
End If

'Next, gather and export resource base calendar exceptions along with
'   resource calendar exceptions
i = 2
For Each r In ActiveProject.Resources
    If Not r Is Nothing Then
        j = i
        If r.Type = pjResourceTypeWork Then
                For Each E In r.Calendar.BaseCalendar.Exceptions
                    xlRng.Range("E" & i) = r.Name
                    xlRng.Range("F" & i) = r.Calendar.BaseCalendar.Name
                    xlRng.Range("G" & i) = E.Name
                    xlRng.Range("H" & i) = E.Start
                    xlRng.Range("I" & i) = E.Finish
                    i = i + 1
                Next E
                For Each E In r.Calendar.Exceptions
                    xlRng.Range("K" & j) = r.Name
                    xlRng.Range("L" & j) = E.Name
                    xlRng.Range("M" & j) = E.Start
                    xlRng.Range("N" & j) = E.Finish
                    j = j + 1
                Next E
        End If
    End If
Next r
MyXL.ActiveWorkbook.Worksheets("Exception Report").Columns("A:N").AutoFit
End Sub
