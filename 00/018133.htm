jperki say:

Comparing values in a form to set a pass/fail condition

I have a database that tracks calibration of torque drivers. There are 44 torque drivers split up by their torque values. The values of the various drivers range from 1.5 +/- .5 to 16.0 +/- 1.0.  The database requires that each driver be tested three times, the values of those tests summed and then averaged to come up with a pass or fail condition.

 

The problem I am having is writing an if statement that compares the results of the average to the individual specifications of each driver. In other words, if I am testing a 7.0 lbs. driver with a specification of 7.0 +/- 1.0 in. lbs. three times and the tests are: #1 - 7.3, #2 - 7.6, #3 - 7.1 then what I have now will sum these in a field and average them in another field. Now I need to compare the average of these tests (7.3) to the specification.

 

I currently have three tables with the associated columns:

tblEmployee

   IDNumber

   LastName

   FirstName

 

tblTools

    DriverNumber

    Specification

    Nominal

    IDNumber

 

tblReadings

   DriverNumber

   PerformedOn (Date)

   TestNumber1

   TestNumber2

   TestNumber3

   SumOfTests

   Average

   Result

 

I would like the "result" text box to show either a pass or fail condition depending on if the average is within the specification of the nominal value.

 

I have tried various forms of if statements and just can't seem to get it to work correctly.

 

Does anyone have any ideas on how to make this work?

 

Thank you in advance.

marshall barton say:

Try using:

 

In the average testbox:

 

=(Nz(TestNumber1,0) + Nz(TestNumber2,0) + Nz(TestNumber3,0)) / 3

 

and in the pass/fail text box:

 

=IIf(averagetextbox <= (Specification + Nominal) And averagetextbox  >= (Specification - Nominal), "Pass", "Fail")

 

(The Nz function is used to guard against missing test numbers.)

john w. vinson say:

If you are committed to this non-normalized, limited table design then Marshall's expression would certainly be the way to go. But I'd really, really suggest changing your table design! You're "committing spreadsheet": TestNumberx is a repeating group; and the derived fields SumOfTest, Average and Result should simply not exist in any table, but should instead be calculated on the fly.

 

Consider a Readings table like:

 

tblReadings

  ReadingID (autonumber primary key)

  SeqNo (Long Int, values 1, 2 or 3 - or maybe more if you need more replicates)

  DriverNumber (link to Drivers)

  PerformedOn

  Reading (what's now in your TestNumberx field)

 

a Totals query grouping by DriverNumber and summing and averaging the values in Reading will return the data that you need; and a comparison with the specification table will give you the pass or fail.

john mcghie say:
Bullets or numbering should be applied with a Style (they do not have to be, but if you do, this kind of thing becomes dramatically easier).

The easy way:

1)  Format>Style>Modify>Format>Numbering>Customise...

You can set the colour of the bullet or number in there, and that will change the whole document.

The hard way:

1)  Select the paragraph mark at the end of each bullet paragraph

2)  Apply a colour to just the paragraph mark.

That changes the colour of the bullet without affecting the text.

Personally, I would create a style with the colours and bullets and whatever you want.  I would probably modify the built-in style named List Bullet, since that's what it's for.

If you check "Add to Template" after you have finished making changes, it will be stored in your Normal template for use in other documents.

Hope this helps
