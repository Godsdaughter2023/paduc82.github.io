Excel for Mac 2016: How do you use Paste Link for numbers? - Microsoft Community

ronald schwartz say:

Excel for Mac 2016: How do you use Paste Link for numbers?

I created a workbook with five sheets. Sheet one is a summary of a cost estimate. Sheets two through five are the four components of the estimate. I want to link each of the totals for sheets two through five to its respective cell on sheet one. 

On the Home tab, I opened the Paste function and clicked Paste Special, which yields two options: Paste and Paste Link. Paste gives options PDF, Picture, HTML and Text, neither of which seem to apply to the number cells I want to link. However, Paste Link is grayed out.

How can I use Paste Link to link number cells from one sheet in a workbook to another?

bobgreenblatt say:

As usual, Phil's advice misses the point and is wrong. Do it this way:

Even though your summary sheet has a current value in it, you want to replace this value with a formula using values from the other sheet(s). Paste Link is not at all what you want to do.

- Select a cell on the summary sheet that needs values from the other sheets. In this cell, type =

- Then switch to the first sheet containing the component values, and click in the cell

- Then type + and switch to the next sheet, and click on the cell there.

- Continue the process until you have all the components, and then press ENTER.

Now, your summary sheet has a formula which grabs values from the other sheets. So, when they change, the value on your summary sheet well change automatically.

Whilst you could put your references into a DB, that would be duplicating the work of the Sources.xml file, since any citation you add to a document ends up there anyway. In any event, simply storing them in the DB doesn't address the core issue, which is that of getting the data into Word.

The mailmerge analogy doesn't really apply, as the logic to interpret a data source for mailmerge purposes is built into Word. There is no equivalent built-in functionality for bibliographic data. Accordingly, the 'tool' to do the import would have to be written from scratch and would require a detailed knowledge of the source data's structure as well.
