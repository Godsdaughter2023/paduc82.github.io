steve rindsberg say:

Ribbon buttons with macros assigned to them remember where the macro came from.  You'd need to have the original file open.

I'm not sure what it stores internally (you might be able to find out by exporting the customized settings of your tool bar and then examining the XML).  But conceptually, it seems to be something like:

When he clicks this button, launch MyFile.PPTM!MySubroutine

If MySubroutine is now in MyOtherFile.PPTM, it's not there as far as PPT and the button are concerned.

To make a macro available all the time, regardless of what files are open, you can save it in an add-in and add RibbonXML to invoke it; as long as the add-in is loaded, the macro will be available.
