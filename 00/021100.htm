alexr54 say:

I have found the reason for this issue as well as the solution. I will include details in my final response to this thread which I started.

Apparently the Link Preview is an option in OWA that is enabled by default. When a user in OWA is replying to an Email sent from Outlook that contains a URL link in the signature (or a URL link anywhere else in the Email body for that matter), and the Email being in HTML format; When replying to the Email using OWA, the link preview causes this issue if the user who is replying does not click on the Remove Link Preview button before sending the reply back.

This causes the URL in the signature, in my case (or anywhere else in the Email body) to automatically generate a small website preview which includes an image from the website URL and any text description on that website page. Similar to how it works in Facebook, when you post a link in a new topic or reply.

So what I needed to do was to find a way to disable this Link Preview option in OWA. Now, using OWA its simple to do this in options. There is a check box under Link Preview in the Options (Options-->Link Preview). Uncheck this box and the automatic Website Preview goes away for good.

Problem is, I am in an organization with lots of users to put in mildly, and we cannot tell everyone to go and uncheck this option in OWA. It is not very professional...

It took a good amount of digging since this is not a common complaint or issue for most users, but I did find a PowerShell command that disables this option. With this PowerShell command if you are proficient enough with PowerShell scripting it is simple enough to incorporate this into a script that will apply to all existing users and any future new users.

Here is the command for anyone else who may run into this particular issue:

âSet-MailboxMessageConfiguration -Identity âUser Name" -LinkPreviewEnabled $falseâ
