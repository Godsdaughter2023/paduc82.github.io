ronrosenfeld say:

Lots of ways to do this.  Here's one:

=CHOOSE(MATCH(Severity,{"Low","Medium","High"},0),1,5,9)*
CHOOSE(MATCH(Probability,{"Low","Medium","High"},0),1,5,9)

Severity and Probability are the two cells containing that data.

The array constant could be either entered directly, or as a cell reference containing those values (perhaps the same as you are using for your dropdown list).

Here's another method using VLOOKUP:

=VLOOKUP(Severity,{"Low",1;"Medium",5;"High",9},2,FALSE)*
VLOOKUP(Probability,{"Low",1;"Medium",5;"High",9},2,FALSE)

Again, the array can be entered directly, or as a cell reference to a two column range:

=VLOOKUP(Severity,$J$1:$K$3,2,FALSE)*
VLOOKUP(Probability,$J$1:$K$3,2,FALSE)

Hello seanlake1970

Re: #1 - Outlook doesn't sync with another copy of Outlook running on another machine. As an <email client>, it connects to and sync's folders according to the protocol used to configure the account. (the same explanation applies regardless of which <email client> is being used, of which Outlook is only one of many)

For POP accounts

- only the <Inbox> folder data is downloaded in a one-time one-way direction from server to local computer. The POP3 protocol allows downloaded data to remain on the server so that it can be downloaded by another <email client> on another machine if the appropriate setting is checked. No other email folders or contact/calendar folders are sync'd with the server)

For IMAP accounts

- email folders are sync'd with the server on a bi-directional basis. In other words, what gets added/deleted/changed on one side of the equation occurs on the other side. Contact/calendar folders are not sync'd with the server.

For Exchange Accounts

- all folders are sync'd with the server (email, contact, calendar)

The above describes how every email client functions on a native basis. There are 3rd party solutions that provide sync'ing of Outlook folders between machines.

Re: #2 - Yes you need to configure email accounts manually on each machine. There are (have been) 3rd party solutions claiming to migrate email account settings from one machine to another. Would strongly advise against using any of them, Have yet to find one that works in all scenarios and testing to see whether everything worked correctly can take longer then the few minutes it takes to do manually.

Re: #3 - You don't mention what version of Outlook is involved on the old machine so providing the following 2 articles on how to migrate rules

Import or export a set of rules
https://support.office.com/en-us/article/import...

How to migrate Outlook rules
https://www.codetwo.com/kb/migrating-outlook-ru...

hansv mvp say:

Try this version:

 

Sub ChangePivot()
    Dim pvt As PivotTable
    Dim pvi As PivotItem
    For Each pvt In ActiveSheet.PivotTables
        pvt.PivotFields("Module").ClearAllFilters
        pvt.PivotFields("Module").CurrentPage = Range("B4").Value
        ' Make all pivotitems visible
        For Each pvi In pvt.PivotFields("Tray").PivotItems
            pvi.Visible = True
        Next pvi
        ' Hide all except the one we want to display
        For Each pvi In pvt.PivotFields("Tray").PivotItems
            pvi.Visible = (pvi.Name = Range("C6"))
        Next pvi
    Next pvt
End Sub

joeu2004 say: Ryydo wrote:
I have a number of sheets of which a given cell = a cell on another sheet, but on one of the sheets it shows up as a zero. I've checked conditional formatting and cell formatting etc but they seem ok?? Anyone got any other ideas??

Excel treats empty cells as zero in "arithmetic expressions", which technically a formula of the form =A1 is.  Several work-arounds.  Two of them are:

  • Change =A1 to =IF(A1="","",A1)

  • Change the format of the cell with =A1 to one of the following Custom formats:

              General;-General;""

              or

              [=0]"";General

You can replace "General" with any other numeric format, for example 0.00 (Number with 2 decimal places).

Alternatively, you can unselect the View option "zero values" (under Tools > Options).  But keep in mind that that affects the entire workbook.
