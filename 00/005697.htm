These registry keys are known to be used by malware to disable installed AV/Antimalware apps, including Windows Defender:

  • HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\safer\codeidentifiers \0\Paths
  • HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\Current Version\Image File Execution Options
  • HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Policies\Explorer\DisallowRun

https://answers.microsoft.com/en-us/protect/forum/mse-protect_scanning/mse-disabled-after-malware-removal-regedit-advice/6a0c78a0-a74a-41c8-9d0e-a16db45628de

In most cases, however, malware just adds a DisableAntiSpyware DWORD to the Windows Defender subkey, and then sets its value to 1 (disable true):

  • HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows Defender

And this is normally done without removing the user’s administrator privileges – so in that case, we can just use conventional third-party malware-removal tools:

Kaspersky Virus Removal Tool:

http://support.kaspersky.com/viruses/kvrt2015

Emsisoft Emergency Kit:

http://www.emsisoft.com/en/software/eek/

Malwarebytes Anti-Malware (free version only):

https://www.malwarebytes.org/antimalware/

Eset Online Scanner:

http://www.eset.com/us/online-scanner/

Some other trusted third-party malware-removal tools are listed here:

https://answers.microsoft.com/en-us/protect/wiki/protect_other-protect_scanning/list-of-malware-removal-tools/d824b9af-ebd8-4c47-94e2-8ee6c544c100

And then turn Defender back on with a REG command: Right-click on the Start button, select Command Prompt (Admin), and then copy, paste, and run (enter) this command line:

REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender" /v DisableAntiSpyware 

As I’ve illustrated here:

Turn_On_Windows_Defender.reg is a trusted file that simply removes the DisableAntiSpyware  DWORD.

TTest in Excel across multiple columns - Microsoft Community

drerzaask say:

TTest in Excel across multiple columns

I have 56 columns that are of different lengths. I want to be a series of TTests between all of them. I know that the syntax in Excel is TTest(array1, array2, tail, type). Is there a fast way to do this with labeling? I know that manually there are 1540 combinations. I really do not want to type that many formulas out in Excel. So there are 56 columns. The first position in each column is a label for what data is in that column.

Thank you very much

bernie deitrick say: No, not labeling, but using INDEX.

I have assumed that your data table starts in cell A1.

Insert 99 rows above cells A1 (moving  the first numerical data point from A2 to A101)

Then in cells B1:BE1, enter the numbers 1 to 56, and in A2:A57, enter 1 to 56 also.

Then in B2, enter this formula (assuming you have fewer than 900 points at most in one column of data - you can increase or decrease the 1000 as needed) - change the Tail and Type to your desired values.

=IF($A2<B$1,TTEST(INDEX($A$101:$BD$1000,,B$1),INDEX($A$101:$BD$1000,,$A2),1,2),"")

and copy to B2:BE57. That will produce a half filled table giving you all the possible pairwise comparisons.
