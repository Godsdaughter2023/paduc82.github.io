butch pedorcik say:

I'm going to guess that part of the trouble you are experiencing is related to the field names in your Access table compared to the column names in your Excel sheet.  The data in the Excel columns that you wish to import must have a matching field within the Access table and the names need to be identical.

 

My assumption at the moment is perhaps you have some fields within your Access table that are named very similarly to each other which could easily lead to "crossed wires" or information ending up in a field in the Access table that you weren't intending.  It's very important that the names between the two are matched appropriately.  So, it's possible that the solution could be as simple as renaming a column or two in the Excel sheet. 

scottgem say:
I would do this in a 2 step process. First LINK to the Excel Spreadsheet, then run an Append query to append to your table. 

This way, it doesn't matter the order or data types of the Excel data. You can control what columns in Excel go to what fields in your Access table and control the format of the data during the Append.

Also, if this is existing data from an application in Excel that you are converting to Access, then a straight append is probably not a good idea. The structure of Excel tables is different then for Access. If you are converting, you should redesign your tables, to fit the Access relational model. So converting the data to Access should probably not be a straign Append to an Access table.

debbie_h say:

Hi additudeOBX,

Have you tried adding the site to trusted sites?

To add a website to a security zone

  1. Open Internet Explorer by clicking the Start button , and then clicking Internet Explorer.

  2. Navigate to the website that you want to add to a specific security zone.

  3. Click the Tools button, and then click Internet Options.

  4. Click the Security tab, and then click a security zone (Local intranet, Trusted sites, or Restricted sites).

  5. Click Sites.

  6. If you clicked Local intranet in the previous step, click Advanced.

  7. The website should be shown in the Add this website to the zone field. Click Add.

    If the site is not a secure site (HTTPS), clear the Require server verification (https:) for all sites in this zone check box.

  8. Click Close, and then click OK (or click OK twice if you clicked Local intranet in step 4).

From article:

http://windows.microsoft.com/en-US/windows-vista/Security-zones-adding-or-removing-websites
