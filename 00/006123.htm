jay freedman say:
To transfer styles from one document to another, use the Organizer dialog.

There are two ways to open the Organizer:

  • Press Ctrl+Alt+Shift+S to open the Styles pane, click the Manage Styles button at the bottom of the pane (the third of the three similar buttons), and click the Import/Export button in the Manage Styles dialog;  OR
  • On the View tab of the ribbon, click the Macros button (or press Alt+F8) and click the Organizer button in the Macros dialog; then click the Styles tab in the Organizer.


One side of the dialog shows the styles that are in whatever document was open at the time you started the Organizer. The other side shows the styles that are in a template, usually Normal.dotm.


On the template side, click the Close File button, which changes into an Open File button. Click it again and choose the other document involved in the copying.


Select the styles you want to copy (use the Ctrl or Shift key while clicking to select more than one simultaneously), and then click the Copy button in the center. The arrow on the button shows the direction the copying will use. If a style of the same name exists in the destination document, the copy will replace the old definition with the new one. You can continue copying until you have every style you need. Then click the Close button at the bottom of the dialog.


So far, all you've done is to copy the definitions of the styles from one document to the other. Now you have to apply the styles to the text in your document.

Copy Data from Multiple Work Book to One WorkBook using By VBA without - Microsoft Community

qasem abbasi say:

Copy Data from Multiple Work Book to One WorkBook using By VBA without opening destination Files

Entrance Data For Running Code is:

1-Directory File

2-Name Sheets (Name Sheet Should be the same in All excel Files)

3-Number Of Row

4-Start Number Of Row

5-Number Of column

6-Start Number Of Column

------------------------------------Sub ReadDataFromAllWorkbooksInFolder()
Dim FolderName As String, wbName As String, cValue, t As Variant
Dim wbList() As String, wbCount As Integer, i, j, r, s, k, l, p As Integer
Dim q As String

t = Application.InputBox("input Directory Files ")              ' input Directory Excel Files
  
    FolderName = t
 
      m = Application.InputBox("input Sheet Name Of Files ")    ' input Sheet Name
  
      k = InputBox("input number of row ")                      ' input number of row
      
      l = InputBox("input Start number of row")                 '  input Start number of row
     
     p = InputBox("input number of Column")                     ' input number of Column
     s = InputBox("input Start number of Column")               '  input Start number of Column
  
    ' create list of workbooks in foldername' --- Comment
    wbCount = 0
    wbName = Dir(FolderName & "\" & "*.xls")
    While wbName <> ""
        wbCount = wbCount + 1
        ReDim Preserve wbList(1 To wbCount)
        wbList(wbCount) = wbName
        wbName = Dir
    Wend
    If wbCount = 0 Then Exit Sub
    ' get values from each workbook' --- Comment
    For i = 0 To wbCount - 1
         For r = 1 To k
          For j = 1 To p
                q = Cells(r + l - 1, j + s - 1).Address(RowAbsolute:=False, ColumnAbsolute:=False)
                 cValue = GetInfoFromClosedFile(FolderName, wbList(i + 1), (m), (q))
                 Cells(r + i * k + 1, 1).Formula = wbList(i + 1)
                 Cells(r + i * k + 1, j + 1).Formula = cValue

        Next j
         Next r
          Next i
  End Sub

Private Function GetInfoFromClosedFile(ByVal wbPath As String, _
    wbName As String, wsName As String, cellRef As String) As Variant
Dim arg As String

    GetInfoFromClosedFile = ""
    If Right(wbPath, 1) <> "\" Then wbPath = wbPath & "\"
    If Dir(wbPath & "\" & wbName) = "" Then Exit Function
    arg = "'" & wbPath & "[" & wbName & "]" & _
        wsName & "'!" & Range(cellRef).Address(True, True, xlR1C1)
    On Error Resume Next
    GetInfoFromClosedFile = ExecuteExcel4Macro(arg)
   
End Function

stefan blom say:

Note that all cross-references in Word make use of bookmarks. It is just that most of these bookmarks are invisible, even if you enable the "Show bookmarks" option at File | Options | Advanced.

Without looking at an actual document it is difficult to tell exactly what has happened with your document, but you can quite easily "break" a bookmark in Word. In particular, if you add content (or press Enter) at the beginning of a cross-referenced (= bookmarked) paragraph, the content will be added inside the bookmark. You can test this yourself if you add a user-defined (visible) bookmark and experiment a bit.

The workaround is to always add content at the end of paragraphs (in a document where you make use of the cross-reference function).

suzanne s. barnhill say:
And the reason the problem may seem to surface when you save (if you don't notice it until you reopen the document) could be that "Update automatic links at open" is enabled at File | Options | Advanced: General.

Hi Richard

Mmm. I think there are two separate things, here.

The gray background: I'm guessing this is a setting in Tools/Options/View. Check the setting for "Field shading", I'm guessing it's at "always", rather than "when selected" or "never".

Excel opening in the background will happen if something is blocking in-place editing. having the field codes display rather than the field result, or being in "Normal" view rather than Page Layout can cause it. Lack of resources would be another possibility.

Given the problem with editing more than on embedded sheet, I'd say we're looking at something running on this system that's blocking resources OR interfering with OLE client/server.

So the next things I'd try are:

1. Start Word in SAFE MODE and see if the problem persists

2. Start WINDOWS in Safe Mode and see if the problem goes away.

My inclincation is that something is running (anti-virus, maybe) under windows that's interfering with OLE server/client.
