Instead, try creating a new, blank .ACCDB file and then "import" the objects from the corrupted file.

If/when you get this resolved, I'd seriously consider doing two things.

1.) Split the DB into FE/BE. As it's usually VBA that causes corruption, only the forms/reports in the FE would be at risk. The data in the BE would remain untouched.

2.) Create a new code module named "modBackup", and insert this code:

 

Public Sub BackUpAndCompactBE()
'Courtesy of datAdrenaline

On Error GoTo ErrHandler

    Dim oFSO As Object
    Dim strDestination As String
    Dim strSource As String

    'Get the source of your back end
    strSource = Split(Split(CurrentDb.TableDefs("one of your table names goes here").Connect, "Database=")(1), ";")(0)

    'Determine your destination
    strDestination = CurrentProject.path & "\chinook2013_be substitute the name of your BE (" & Format(Now, "yyyymmddhhnnss") & ").accdb"

    'Flush the cache of the current database
    DBEngine.Idle

    'Create a file scripting object that will backup the db
    Set oFSO = CreateObject("Scripting.FileSystemObject")
    oFSO.CopyFile strSource, strDestination
    Set oFSO = Nothing

    'Compact the new file, ...
    Name strDestination As strDestination & ".cpk"
    DBEngine.CompactDatabase strDestination & ".cpk", strDestination
    Kill strDestination & ".cpk"

    'Notify users
    MsgBox "Backup file '" & strDestination & "' has been created.", vbInformation, "Backup Completed!"

errExit:
    Exit Sub

ErrHandler:
    MsgBox Err.Number & ": " & Err.Description
    Resume errExit

End Sub

Private Sub CmdQuit_Click()

marshall barton say:

As Mimmo said, you can not use VBA to do that.  You must use Conditional Formatting to get different colors on different rows in a continuous or datasheet form.  But, you want the entire record to be colored, not just one field's text box.  It's a small bit of slight of hand to color the entire record and you must use a continuous form, datasheets just won't cut it.

First:  Add a large text box (without an attached label) to the form's detail section and size it to cover the entire detail section.  Set its BackColor to an agreeable shade of green.  Use conditional Formatting for this text box to change the back color to red using the Expression Is: option set to an expression like:

   [myField] = 5

(Unlike most everywhere else in Access, the [ ] are required here.)

Second:  Use the Send To Back on the ribbon's Arrange tab to put this text box behind all the other controls in the section.

Third:  Set all the other text box, combo box and label's BackStyle property to Tranparent.

If you insist on using a datasheet form, then the best you can do (just might be sufficient for your needs) is to use Conditional Formatting on every text box with the same Expression Is: settings.
