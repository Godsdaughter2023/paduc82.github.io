The subs I gave you are triggered on leaving and returning to the worksheet. If that wasn't the problem then try the following.

The Book1 - Sheet1 (Code) reference refers to your workbook's name (e.g. Book1) and the original name of the worksheet (e.g. Sheet1) before it was named or moved.

Try this: tap Alt+F11 and when the VBE opens, locate the code you pasted in and erase both subs completely. Next, close every code window you can see. Tap Alt+Q to close the VBE. Right-click the worksheet you want to suspend AutoComplete on and select View Code. When the VBE reopens, the only code window available should be the one related to the worksheet you want to affect. Paste the code into this window and tap Alt+Q to return to your worksheet. You will have to move off this sheet then back again to trigger the disabling code.

I have assumed that you do not already have Worksheet_Activate and Worksheet_Deactivate routines in the worksheet's code window. If they already exist, the new code line will have to be added to the existing routine as you can only have one of each of these triggered subs for each worksheet.

I haven't worked Worksheet_Activate and Worksheet_Deactivate routines in a while and I had thought that opening the workbook (with your worksheet as the active sheet) was enough to trigger the subs. I just tested it and it is not. You will need another sub like Workbook_Open to artificially force the disabling of the AutoComplete. With teh VBE open, right-click This Workbook from the list in the Project Explorer on the left and select View Code. Paste something like this into the Book1 - This Workbook (Code) window,

Private Sub Workbook_Open()
    If ActiveSheet.Name = "My Non-AutoCompleting Worksheet" Then _
        Application.EnableAutoComplete = False
End Sub

Change My Non-AutoCompleting Worksheet to the name of your own worksheet and tap Alt+Q to return to your workbook.

j.e. mcgimpsey say: I don't think that the number of SUMIFS() matters so much as the number of arguments and size of the arrays.

Without knowing more about your data set, it's hard to suggest specifics, but in general, efficiency comes down to reducing the number of independent repetitive calculations that XL has to make, especially when they involve large arrays.

Obviously, if any of your conditions duplicate any other condition (i.e., if column D=x whenever column J=y), then eliminating one or the other as a SUMIFS() condition will save time.

Likewise, if instead of evaluating a pair of conditions 46,000 times you can use a helper column to evaluate it once, you'll be more efficient. For instance, if you currently have in all your formulae:

   =SUMIFS($A$1:$A$4000, $B$1:$B$4000,TRUE,$C$1:$C$4000,TRUE,$D$1:$D$4000,">50000",$E$1:$E$400, "<0.5",...)

then creating a helper column Z:

  [=($B$1:$B$400=TRUE)*($C$1:$C4000=TRUE)*($D$1:$D$4000)]

(or, more efficiently:

  [=--($B$1:$B$400)*(--($C$1:$C$4000))*($D$1:$D$4000)

can reduce the evaluations in you SUMIFS() to 

  =SUMIFS($A$1:$A$4000, $Z$1:$Z$4000,">50000",$E$1:$E$400,"<0.5"...)

calculating that combination once, rather than 46,000 times.
