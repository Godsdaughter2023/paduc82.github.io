sablicious say: Update on my problem: SOLVED.
Cause: CORRUPT WINDOWS USER PROFILE.

My solution was to transfer my data to a new administrator profile ― i.e., make a new, clean profile.    This is done by creating TWO new user Windows profiles (though I only used one new one) and copy-pasting the entire contents of Local Disk C: / Users / [username] EXCEPT FOR ntuser.dat, ntuser.dat.LOG and ntuser.ini to the same location in the new profile.  To view all these "hidden"  files, you must remove the "Hide protected operating system files (Recommended)" parameter ― ALT > Tools > Folder options... > View > uncheck the applicable box > Apply (*in the aforementioned user directory).

The method advised here - http://windows.microsoft.com/en-US/windows7/fix-a-corrupted-user-profile - is to create THREE profiles, using one as an intermediary to transfer between the 'old' (corrupted) profile and the new, clean one.  I forgot and simply copied from old to new, but it's been fine ever since.
