scottgem say:
This is not proper design. If a value is based on another value, then this should be controlled by table design. 

You should have a table of values with corresponding labels. A user will then select $5000 from a combobox and the combobox will pull Gold from the table for display. You do not need to store Gold since it is a function of the amount selected. 

So the RowSource of your ammount combo would look like:

SELECT Amount, AmtLabel FROM table;

Then set the Controlsource of a textbox to:

=[cboAmount].Column(1)

Where cboAmount if the name of the amount combo.

winston m say: Hi,
 
 
Mixed content situation can happen on HTTPS secure websites such as online stores or financial sites that display images, banners, or scripts that are coming from a server that is not secured. IE has defaulted to prompt user for next course of action when encounter mixed content as displaying mixed content carries risk that non-secure web page or script will be able to access information from the secure content.
 
 
Having to click “No” (or “Yes” to block non-secure content) button for every visit to website with mixed content can be troublesome and annoying. Besides, its possible user accidentally clicks “Yes”, which prevents some insecure elements from loading and displaying. For user who confident that he or she does not visit possibly malicious site, it’s possible to instruct IE to display mixed content automatically, instead of asking for input.
 
 
How to Disable and Turn Off Do You Want To View Only the Webpage Content that Was Delivered Securely Security Warning

Follow the steps provided below and check if that helps.
a) Open Internet Explorer and click on “Tools” and click “Internet Options”.
b) Click on “Security” tab and click “Internet” under “Select a zone to view or change security settings”.
c) Click “Custom level” under “Security level for this zone”.
d) Scroll down to the "Miscellaneous" section.
e) Select “Enable” under “Display mixed content”.
f) Click “OK” and “Apply” to save the settings.
 
 
 
Hope this helps.

Winston,

The steps you outlined won't always solve the problem. That's because the "Display mixed content" setting has no effect if there's an IE add-on injecting content from a nonsecure web site. In my case, I solved the problem by scrutinizing the blocked content (i.e. the content that appeared only when I clicked "No" in the Security Warning dialog box). That content then gave me a clue as to which add-on was causing the problem. Once I disabled it (Tools> Internet Options> Programs> Manage add-ons) the annoying Security Warning went away. For me it turned out that Kaspersky Internet Security's Safe Web Page feature ("URLs check" add-on) was the culprit.
