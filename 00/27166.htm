Option Explicit

Sub Macro1()
Dim oRng As Range

    ThisDocument.Save

            FileFormat:=wdFormatXMLDocumentMacroEnabled, _
            AddToRecentFiles:=False
End Sub

john - project say:
Eric,
Transferring data always requires a fair amount of optimization and yes there are approaches that run faster than others. Probably the best way to gain transfer speed is to make sure you are doing operations using background processing. For example, code that operates directly on objects, (background processing), instead of selecting rows/cells to accomplish things, (foreground processing), will run faster because it isn't bogged down by screen processing. Recorded macros are the best example of macro that use foreground processing.

Something else than can slow down processing is to continuously switch between applications (e.g. Excel and Project) to preform various operations. When I have a large amount of data to transfer, (95% of my macro are from Project to Excel), I generally load the Project data into arrays, then I open Excel, turn off screen updating and keep Excel invisible while the code unloads the array data into Excel in the background. When everything is finish I make Excel visible. So yes I would probably load Excel's

Another method I've used is to create a text file of Project data and then open the text file in Excel. I'm not sure how that might work going the other way but it would be no different than opening a text file into Project.

You may find that an import map from Excel to Project will get the bulk of your data into Project very quickly. Be advised however that there are some quirks with import/export maps that seem to be associated with Excel's new xlsx format, so data imported from Excel needs to be audited once it's in Project.

Hope this helps.
