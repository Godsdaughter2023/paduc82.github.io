call divert - Microsoft Community

safir---01 say:

call divert

How I can divert my call if unreachable.. Nokia Lumia 820

paul van der heu say:

Call forward features are a function of the network and as such need to be supported by the network for your contract. In most cases call divert can be setup as follows:

 

Feature Mode Sequence
All calls Activate & register number ** 21 * Number # SND
Deactivate & deregister number ## 21 # SND
Activate to registered number * 21 # SND
Deactivate # 21 # SND
Verify *# 21 # SND
No answer Activate & register number ** 61 * Number # SND
Deactivate & deregister number ## 61 # SND
Activate * 61 * Number # SND
Deactivate # 61 #
Verify *# 61 # SND
Unreachable Activate ** 62 * Number # SND
Deactivate ## 62 # SND
Verify *# 62 # SND
Busy Activate ** 67 * Number # SND
Deactivate ## 67 # SND
Verify *# 67 # SND
All types Deactivate ## 002 # SND
All conditional types Deactivate ## 004 # SND

 

If you want to extend the time before a divert activates you can add **[time in seconds] after the number. So to activate divert for all calls to number 12345 after 30 seconds you would enter:

 

** 21 * 12345 ** 30 # SND

 

 

The codes mentioned are what most operators use, but they can be different so check with your operator. Also if these codes are not supported by your network you will get a 'Unknown request' or ' request denied' from the network as a response.

fb10 say:

Outlook 2010: change reply email address automatically depending on address sent to for IMAP (like POP)

If you have multiple email addresses set up in Outlook as aliases, and you are using a Outlook pst POP folder as your main folder, then when you hit reply to an email sent to *** Email address is removed for privacy *** it will default to sending from *** Email address is removed for privacy ***. Likewise, on another email sent to *** Email address is removed for privacy *** it will default to *** Email address is removed for privacy ***. That is, in a POP/PST style setup, Outlook is savvy enough to know which email address the email was sent to and defaults to that same email when you hit reply.

This is not the case if you have all of your emails going to an IMAP account. I've just transitioned to IMAP, and I have many aliases forwarding into my one IMAP inbox. Is there no way to have Outlook default to replying with the email address that the original email was sent to when using the IMAP setup?

Thanks!
brian tillman [outlook mvp 2007-2019] say:

Not for IMAP accounts.  Because IMAP items stay on the server, the folder set must match what's on the server.  Every IMAP server will have its own folder set, so if folder X resides on server A, you'll need a folder X in Outlook and if folder Y resides on server B, Outlook will need a folder Y.  Clearly these can't be in the same folder set because then you'll have the exact same folders and same items on both servers.  What would be the point of that?

Also, IMAP doesn't know anything about non-mail folders like Calendar, Contacts, Journal, Notes, and Tasks.  So, you need even another folder set to hold those.

POP accounts, however, know only about the folder Inbox and items flow only one way - from the server to the client, so it's fine to be able to use a single folder set on the client side.

monday win 600

i want excel to automatically calculate the win and loss ratio in percentage.

to work out loss: i would add 1700 + 500 = 2200, 500/2200*100 = 22% loss

to work out win: i would add 1700 + 500 = 2200, 1700/2200*100 = 77% win

vijay a. verma say:

Let's say that your values are in B2:B100.

Win % (Format result cell as %)

=SUMPRODUCT((B2:B100>0)*(B2:B100))/SUMPRODUCT(ABS(B2:B100))

Loss % (Format result cell as %)

=-SUMPRODUCT((B2:B100<0)*(B2:B100))/SUMPRODUCT(ABS(B2:B100))
