howardbone say:

I was able to get ths to work by doing two things:

I created a localhost entry in my host table that points to 127.0.0.1, and then changed the advanced setting in IE9 that allows running Active content in files on My Computer.

The problem had to do with the way IE9 determines what level of security to apply to the file being opened: local intranet or internet.  A TechNet artical on this topic, said that using the 127.0.0.1 address causes IE apply the Internet security, while the localhost applies the Intranet security.  However, on my system I couldn't use Intranet until I added the localhost entry to the host table.  Then, to get rid of the pesky warning about my file having Active content, I enabled the Active Content setting.

Of course doing this means that I need to be careful not to open files that I've saved to my local computer without knowing that they are safe, but I think I can handle that.
