Creating an input form with fields from two tables

The new ContributorID and Contributor Name didn't carry over to the appropriate fields on the Donation Form.

I suspected that might happen, but when I tried to just key them, they didn't match anything in the existing list so I was not allowed to continue entering data.

Is there a way to make this operation work or is it better to just let the staff select each of the two forms to complete for a new contributor with donation?

john w. vinson say:

By far the simplest way to do this is to make the Donation form a Subform of the Contributor form. The user would enter the data for a contributor, or navigate the form to the record for an existing contributor; the Subform's Master/Child Link Field would be ContributorID, ensuring that the donations will be attributed to the correct donor.

The Donations table should have the ContributorID as a foreign key, but shouldnot have any other donor-specific fields such as the contributor name.

If you're not familiar with Subforms, take a look at the Orders form in the Northwind sample database, or (actually) almost any of the sample databases - Subforms are very common and are just the ticket for what you're trying to do.

 

John W. Vinson/MVP

gina whipp say:

What is needed here is is two forms linked to each table.  Sounds like you tried to create one query with both tables as the RecordSource for one form.

Main form = Contributor

Subform = Donations

Linked by ContributorID.

In the above scenario you would use a Combo Box to look up the Contributor and then on the subform with is attached to the Donations table you would enter their donations.

You will probably also need a way to add new Contributors.  I would use a seperate form for that which you could put a button on the Main form and then requery the Combo Box upon filling in the new contributor.

Does MSE firewall is good enough to secure our computer from harmful - Microsoft Community

harsha rajapaksha say:

Does MSE firewall is good enough to secure our computer from harmful malwares?

Don't we want to use a another firewall software to protect the computer from harmful malware program instead of using MSE firewall alone?
kosh vorlon say:

Hi Harsha,

 

As far as the firewall goes, Windows Firewall is excellent and there's absolutely no need to buy anything to replace it with as it quite simply isn't necessary.  MSE does not include a firewall but it does work with Windows Firewall to cover other types of threats like malware, viruses, spyware and such things (a firewall merely blocks ports and access to your system - which is an entirely different function though clearly related to overall security).

 

Here's what I do (and it isn't 100% but nothing is, and in some cases only catches things after they've already gotten into the system, but it's a good process while remaining reasonable and they can all work together - as proven by the fact that I use them):

 

I've configured Windows Firewall to turn off any ports I don't actually use (by default, it enables quite a few which may not be required).  Every port you can close (without affecting how you work or communicate) is one less opening in your system's security.  I use some, but was able to disable almost half when I first did it and I've not needed to adjust it since then as nothing has changed that has required it.  In essence, once should do it unless/until you try to to something different with your networks and then you may need to do a litttle tweaking - but that's true in any case.

 

As my first line of defense, I use Microsoft Security Essentials (MSE) http://www.microsoft.com/en-us/security_essentials/default.aspx for real-time protection and run 2-3 quick scans a week with a scheduled weekly full scan (and confirm that the automatic updating schedule is working so it's never more than 3 days old - and I haven't needed to manually update since I started using MSE).  Here's how I setup MSE:

 

**********************************************************

 

Make sure that ALL threat levels are sent to Quarantine where you can then decide whether or not to allow them or leave them there, or remove them. It is the safest solution for having the most options for dealing with the situation. Quarantine is as safe as removed, but in some cases you may want to remove them to be more certain - and with this one I would remove it (when the time comes). To do this, open MSE, click on the Settings Tab, and for each of the threat levels, set the action to be Quarantine and save.

 

Go to the History tab and click on allowed and unless you specifically placed something in allowed, remove everything else from the allowed category. Only put things there that you ABSOLUTELY know are not problems (are "false positives") and triple-check before you do it as doing this could allow a real threat to bypass security and infect you and it won't be detected or rather won't be reported or acted upon by MSE.

 

Go to the Settings Tab and click on Real-time protection and check all the boxes and save.

 

Go to the Settings Tab and click on SpyNet and click on Advanced and save. This is not at all dangerous to you and no information sent will be used by Microsoft to identify you even if some minor identification information is sent in terms of defining the infection, but it can sometimes help with new threats by initiating an immediate update to deal with the problem if it is in the SpyNet database (and part of how MSE works is by depending on SpyNet for access to this larger set of definitions that isn't downloaded to the computers even with the updates) and it helps Microsoft update definitions to provide you and everyone else better protection as they can then recognize them as they are occurring and thus react faster with some type of solution.

 

Go to the Settings Tab and click on Advanced and check the box for save Quarantine items and change it from one month to three months so you have more time to decide if you want to restore something.

 

*****************************************************************

 

Technically, MSE with Windows Firewall is sufficient.  But I like some additional security (and it has saved me more than once).  It's all free (and here, unlike in many situations, free is perfectly fine and to be honest, I've found that some of the free products are superior to any of the "popular" ones people spend money on - not only for MSE but for anything else).  Don't be fooled by those "Security Suite" packages that offer all sorts of options (some acceptable, some worse than you already have, some of little or no value or potentially even harmful, and some that cause a variety of problems with many different programs you use - especially Norton and McAfee).  You can get everything you really need separately and for free as I've done (or already have just about everything required if you just use MSE and Windows Update and the recommendations below in terms of keeping your system updated and setting up various configurations).

 

To backup MSE in real-time, I also use Prevx. http://info.prevx.com/downloadcsi.asp. This is only a scanner (get only the free version - the paid version not only isn't worth the money, but includes things which would cause conflicts) - so if something is found, some other product or process must be used to remove it.  More than once it has warned me that a site I was about to visit isn't safe and so I backed away from it and quite possibly was saved from being infected.

 

I update and run run full weekly scans with Malwarebytes: http://www.malwarebytes.org/ and SuperAntiSpyware: http://superantispyware.com/ and ESET online scanner: http://www.eset.com/us/online-scanner. Get only the free versions. Any that also have paid versions include features like real-time protection that cannot be used along with MSE anyway (in these cases), so there's no point in spending the money anyway.

 

I use Secunia PSI http://secunia.com/vulnerability_scanning/personal/ to make sure all my programs are updated and current (since updates often plug security holes in the software and thus can be important as many people never check and/or update their software unless prompted - and this way I don't need to check everything since notifications and sometimes updates are automatic).

 

I always follow the following very important advice from Stephen Boots - MVP (not so much other products, but how to make sure the rest of your system besides the security-related products is maintained and configured properly and some hints on things to avoid and cautions to take):

 

-Make sure that the Windows Firewall is enabled. No other firewall truly offers better protection or is needed. This one is perfectly fine.

 

-Make sure that all important/critical updates, including service packs for the operating system and programs are installed from Microsoft Update (Windows Update).

 

-Make sure Internet Explorer is at version 8 or 9 and updated with all patches.

 

-In Internet Explorer 8 or 9, use the SmartScreen Filter.

 

-Make sure that IE Internet Security settings are at least set to medium-high (default).

 

-Enable the pop-up blocker in IE.

 

-On Vista and Windows 7 make sure that User Account Control (UAC) ON and not running with elevated privileges.

 

-Make sure that Windows Automatic Updates are set to at least notify, but the preferred setting is to download and install automatically. If you update manually, be sure to update as soon as possible after being notified of available updates.

 

-Make sure that installed applications, especially Adobe Acrobat, Adobe Flash, and Java are at their latest versions. Many vendors are regularly updating and patching for security holes.

 

-Never click through links from unknown sources and use caution even if they are from a "trusted" source.

 

-Never open unsolicited email attachments.

 

-Practice safe web browsing.

 

I hope this helps.

 

Good luck!
