Hi,

 

Thank you for posting in Microsoft Community.

 

I'd be happy to assist you regarding adding/changing of domain administrator in Windows Live Admin Center.

 

"Does "You will always use this Microsoft account to administer this domain", mean that if I click "I Accept" that I can never switch e-mail hosts for the domain name that I am about administer through Outlook.com?"
 
This means that you must use the Microsoft account in order for you to manage the Inbox of the said account that was registered or created in Windows Live Admin Center. Keep in mind that each administrator must have a Microsoft account.
 
"Also, by clicking "I Accept" this is not going to switch my domain's register from my current provider to Microsoft is it?"
 
Please be reminded that adding of domain administrator is different from changing it. You can have multiple administrator accounts for your custom domain. Each administrator must register the domain on the Windows Live Admin Center website and enter the MX record (an MX record specifies which server in your domain should receive incoming email) that's associated with that administrator at your registrar.

 

By changing domain administrator account, you must reregister the domain on the Admin Center website and enter the MX record that's associated with that administrator at your registrar (a company that is accredited to sell Internet domain names) or Domain Name System (DNS) provider's website. You must also delete the old MX record that was associated with the previous domain administrator account.

 

If you wish to add domain adminitrator account (Microsoft account), you can do so without changing your current provider.

 

"I would like to have my e-mail from sent to "*** Email address is removed for privacy ***" to be sent to my Outlook.com inbox, but I also want to be able to move it later if I choose to."

 

You can set up administrator accounts for your domain name (may it be your personal or business address on the Internet) or you can change administrators. However, if you wish to move to another provider at a later time, you will be deleting the old (ALL) MX record that you had with Microsoft. Therefore, we cannot guarantee that you will be able to bring them to where you are going to move.

 

If this will be the case, we recommend that you do a backup and restore. It's always a good idea to do a Windows Live Mail backup before you make necessary adjustment(s) or before you migrate.

Add or remove columns.

Tap the table, then do any of the following:

  • Add columns to the right side of the table: Drag to the right.

  • To add only one column, tap.

  • Delete columns from the right side of the table: Drag to the left. You can delete only empty columns.

  • Add a column anywhere in the table: Tap the bar above a column, then tap Insert. The new column is added to the left of the column you tapped.

  • Delete a column anywhere in the table: Tap the bar above the column, then tap Delete.

Add or remove rows

  • Tap the table, then do any of the following:

  • Add rows to the bottom of the table: Drag downward.

  • To add only one row, tap.

  • Delete rows from the bottom of the table: Drag upward. You can delete only empty rows.

  • Add a row anywhere in the table: Tap the bar to the left of a row, then tap Insert. The new row is added below the row you tapped.

  • Delete a row anywhere in the table: Tap the bar left of the row, then tap Delete.

(I have also reformatted all cells to Text, but am still getting some green corners letting me know that? Possibly related?)

tom ogilvy say:

formatting cells that already contain a value does not change the nature of that value.  If it is a numeric string (text that looks like a number), then formatting the cell as number would not change what is stored in the cell.  Or if a cell contained a number, then formatting that cell as text does not change the data stored in the cell to be a string/text.  

If you want to test what you think are matches, in a blank cell put in something like

=A1=Z31

to test if A1 matches Z31.  Of course you would only do this if you think they do match but Vlookup is having a problem.   If that test shows they do not, then you can use other functions like  

=IsNumber(A1)

=IsNumber(Z31)

to test if they are different.

And yes, if the error marks are indicating differences in form in a column or cells where there should not be differences then, yes, that would be indication of a problem from a lookup perspective..
