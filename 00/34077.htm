gemzon mar say:

When the power button on the computer is pushed and held for at least four seconds, it is noted in the event data as a PowerButtonTimestamp entry that has a value of something other than 0 (zero). Also, there are instances, such as power loss, hard-locked, or hard hang, where the system cannot write the PowerButtonTimestamp information before the computer restarts or shuts down. In this case, a value of zero is logged. However, since you have mentioned that there is an entry on PowerButtonTimestamp, it suggests that someone manually turned it off. You may also tell your friend to have his device checked by the device service center or a professional technician to know if cleaning is needed. Sometimes, dirt build-up can cause short circuit.

This worked for me:

A couple of runs of Windows Update Troubleshooter, "Try Troubleshooting as an Administrator". This cleaned up some other pending updates. I ended up with just the KB4022715 update failing repeatedly, using the Settings, Windows Update.

Tried a manual update of KB4022715 from the Microsoft Update Catalog. On the Package Details tab, on the KB4022715 page, it mentioned that KB4022715 had been replaced, by KB4022723(someone mentioned this in another reply).

Ran Windows Update Troubleshooter again, then did a manual install of KB4022723. This update was successful and Windows Update now thinks it is up to date.

Windows 10 Home, Version 1607 x64, Build 14393.1378

Don't stress too much about the choosing the exact Cumulative update(ie, x64, x86, Server). If you pick the incorrect version, it will just tell you it is incompatible during the install.
