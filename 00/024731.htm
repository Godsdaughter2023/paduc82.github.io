albert d. kallal say:

Just place a button on your AssetDetails form.

 

The command to print a label would look like this:

 

RunMenuCommand
   Command:    SaveRecord
SetLocalVar 
    Name:    lngID
   Expression:  =[ID]

OpenReport
    Report name:   AssetLabels
   Where Condition  = [ID] = [LocalVars]![lngID]
Window Mode  Normal

 

So in the above, we first save the record to the table. We have to do this since the report reads and pulls data from the table, not the current form.

After we save, we next set a handy variable with the current ID of the current record.

Next we launch a report were we simply restrict the report to the one current ID in the current form.

daniel g. grau say:
I'm guessing you are not looking at a real hyperlink, but at text formatted to look like one. The style Word applies to a hyperlink is the Hyperlink character style, which by default is underlined blue and turns purple for followed links. Being a character style, it only affects the looks of the text, not its paragraph indents, line spacing and such. Thus, the fact that some text looks like a hyperlink does not mean it is one.

So here's an easy solution: select the text and press the Reset Character combination Control + Space Bar.

If you don't want hyperlinks to look like that, redefine the style through the menu bar's Format > Styles (select "Hyperlink" on the list, then click on the "Modify" button).

The key combination you mentioned? You are only required to press the fn key if you have defined that behavior in System Preferences > Keyboard. Otherwise it's not needed and the shortcut should work without it.
