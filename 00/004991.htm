georgeogden say:

This sounds like you're possibly another victim of the failed 16288.1 update and is in need of a manual rest of the Windows Update process..... in my case the "NewOS" directory wasn't being created in the update directory C:\$WINDOWS.~BT which may have also corrupted the Windows Update process. 

If you've already tried running the Windows 10 Update Troubleshooter and that hasn't fixed the problem (and it quite often doesn't) then you'll need to take more drastic action and attempt to reset the Windows Update process manually.

A simple method is to copy the text between the ++++ lines and paste it into Notepad

++++++++++++++++++++++++++++++++++++++++++++++++

net stop wuauserv

net stop cryptSvc

net stop bits

net stop msiserver

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

+++++++++++++++++++++++++++++++++++++++++++++++++

Save as UpdateReset.BAT (note the .BAT and NOT .TXT) to your desktop and close it.

Go to the desktop and right click on the .BAT file and select "Run as Administrator"

That should flush out any corruption in the Windows update files and enable you to move on to download the latest build 16296.0. If you get a message saying that wuauserv can't be stopped then run the Update Troubleshooter and then try the batch file again.

Once done, go back and check for updates and also check to see if the C:\Windows\SoftwareDistribution.old and C:\Windows\System32\Catroot2.old directories were created and, if they exist, delete them in case you need to re-run the batch file again.

Finally, before trying to update again make sure you detach any non-essential peripherals, USB devices and completely remove any 3rd party anti-virus software as they may interfere with the update process,

Hope that helps and good luck.

To re-sync your Quick Notes to a folder, follow these steps:
  1. Open OneNote.
  2. Go to File > Options.
  3. Options >Save & Backup
  4. Under Save & Backup, under the Save heading, select the Quick Notes Section and click Modify.
  5. Choose a new notebook to sync your Quick Notes to (doing so will save your Quick Notes to this new notebook automatically*; ex.: If you create a new notebook titled "My Quick Notes", and you create a Quick Note and save it, clicking the Quick Notes button on the notebook list will take you to your "My Quick Notes" notebook, where your Quick Notes will be saved.).

*Note:  I suggest creating a new notebook for Quick Notes only so that regular notes will not get mixed up with Quick Notes. If you choose to do this, create this notebook first before attempting steps #2-5.


**Once these steps are completed, click Ok on the Options window to save your settings. When you check your OneNote Notebooks folder, look for your new notebook there. Click the notebook folder, and you will find that your Quick Notes have been saved there.

10100-67-123456-78

john w. vinson say:

If you want to keep the dashes in the record in the table, but just display the acctno without dashes in a query (or a form or report), you can put a calculated field in the query by typing

NewAcctno: Replace([Acctno], "-", "")

Acctno 10100-67-123456-78 will be displayed as 101006712345678.

If you actually want to permanently remove the dashed from the field, keeping just the numbers, just use the same expression in an Update query.
