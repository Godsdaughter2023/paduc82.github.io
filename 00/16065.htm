Yes, you can use INDEX() and MATCH() for this, but this is what VLOOKUP() is doing. Here are the two formulas so you can compare:

=INDEX(F:F,MATCH(K2,A:A,0)) 

Explanation:

Look for and exact match of K2 i column A. Use the position for the match as an index in column F (which is a one dimensional array)

Alternatively:

=VLOOKUP(K2,A:F,6,0)

Explanation: Look for an exact match of K2 in the first column of A:F (which is a  two dimensional array). Return the value in the 6th column (which is column F) on the row where you got the match on the first column.

Assuming you have no other shapes in the sheet:

Sub SizingCharts()
   Dim i As Integer

   With ActiveSheet
      .[A1].Select  'make sure that no shape is selected
      For i = 1 To .Shapes.Count
         .Shapes(i).Left = 1
         .Shapes(i).Height = 1 + (ActiveWindow.UsableHeight - IIf(ActiveWindow.DisplayHeadings, 20, 0)) / .Shapes.Count
         .Shapes(i).Top = 1 + (i - 1) * .Shapes(i).Height
         .Shapes(i).Width = ActiveWindow.UsableWidth - IIf(ActiveWindow.DisplayHeadings, 26, 0)
      Next
   End With
End Sub

  Dim S As Shape, R As Range
  'Get the shape by the name of the first chart
  Set S = ActiveSheet.Shapes(ActiveSheet.ChartObjects(1).Name)
  'Get visible range
  Set R = GetSaveVisibleRange
  'GetSaveVisibleRange may fail if no cell is selected
  If R Is Nothing Then Exit Sub
  'Adjust the shape
  ResizeShape S, R
End Sub

Private Sub ResizeShape(S As Shape, R As Range)
  'Resize and move a shape to fit a range

  'Exit if bad input
  If S Is Nothing Then Exit Sub
  If R Is Nothing Then Exit Sub
  'We can move a shape only in it's sheet
  If S.Parent.Name <> R.Parent.Name Then Exit Sub

  With S
    'Set position and size
    .Left = R.Left
    .Top = R.Top
    .Height = R.Height
    .Width = R.Width
    'Pictures may exceeds the range now
    If .Height > R.Height Then
      .Height = R.Height
      'Place in horizontal center
      .Left = .Left + (R.Width - .Width) / 2
    Else
      'Place in vertical center
      .Top = .Top + (R.Height - .Height) / 2
    End If
  End With
End Sub

Function GetSaveVisibleRange( _
    Optional WS As Worksheet = Nothing) As Range
  'Returns the visible range, cells at the bottom right of the _
    screen are excluded, they are possibly not entirely visible.
  Dim SaveScreenUpdating As Boolean, SaveEnableEvents As Boolean
  Dim SaveSheet As Worksheet
  If WS Is Nothing Then Set WS = ActiveSheet
  If WS.Name <> ActiveSheet.Name Then
    'We need to switch save to that sheet
    SaveEnableEvents = Application.EnableEvents
    SaveScreenUpdating = Application.ScreenUpdating
    Application.ScreenUpdating = False
    Application.EnableEvents = False
    Set SaveSheet = ActiveSheet
    WS.Activate
  End If
  'Get save visible range, may fail if ActiveCell is Nothing
  On Error Resume Next
  Set GetSaveVisibleRange = ActiveWindow.VisibleRange.Resize( _
    ActiveWindow.VisibleRange.Rows.Count - 1, _
    ActiveWindow.VisibleRange.Columns.Count - 1)
  If Not SaveSheet Is Nothing Then
    'Switch back
    SaveSheet.Activate
    Application.ScreenUpdating = SaveScreenUpdating
    Application.EnableEvents = SaveEnableEvents
  End If
End Function

Here are some other ideas, certainly not the best, except maybe #4:

1. If you don't mind the 2 in the name, say Month2, but you just don't want to see it.  You can highlight the 2 and change its color to match the background.

2. You could also name one of the fields "Mon th"

3.  And yet another hoky trick MONTH for one of the fields and M0NTH for the other. (one is using zero the other a capital O).

4.  You can put two tables directly next to each other and use the same name in both.  When you try to add a field next to an already existing table Excel wants to include it, but you can open the AutoCorrect Options dropdown and choose Undo TableAutoExpansion.  Then highlight the range for this new table and use the Format as Table command or the Insert, Table command.
