macropod say:

Try:

Sub BulkFindReplace()
Application.ScreenUpdating = False
Dim strFnd As String, strRep As String, i As Long, j As Long
Dim DocSrc As Document, DocTgt As Document, oTbl As Table, Rng As Range
Dim vFndULn As Variant, vFndBld As Variant, vRepULn As Variant, vRepBld As Variant
Set DocTgt = ActiveDocument
Set DocSrc = Documents.Open("Drive:\FilePath\FindReplaceTable.doc", _
  AddToRecentFiles:=False, Visible:=False)
With DocSrc.Tables(1)
  For i = 2 To .Rows.Count
    If Len(.Cell(i, 1).Range.Text) > 2 Then
      vFndULn = "": vFndBld = "": vRepULn = "": vRepBld = "": strRep = ""
      Set Rng = .Cell(i, 4).Range
      With Rng
        .End = .End - 1
        strFnd = Trim(.Text)
      End With
      Set Rng = .Cell(i, 7).Range
      With Rng
        .End = .End - 1
        If InStr(.Text, "same") > 0 Then strRep = "^&"
      End With
      Set Rng = .Cell(i, 5).Range
      With Rng
        .End = .End - 1
        For j = 0 To UBound(Split(.Text, ","))
          Select Case Trim(Split(.Text, ",")(j))
            Case "nu": vFndULn = False
            Case "u": vFndULn = True
            Case "nb": vFndBld = False
            Case "b": vFndBld = True
          End Select
        Next
      End With
      Set Rng = .Cell(i, 8).Range
      With Rng
        .End = .End - 1
        For j = 0 To UBound(Split(.Text, ","))
          Select Case Trim(Split(.Text, ",")(j))
            Case "nu": vRepULn = False
            Case "u": vRepULn = True
            Case "nb": vRepBld = False
            Case "b": vRepBld = True
          End Select
        Next
      End With
      With DocTgt.Range.Find
        .ClearFormatting
        .Text = strFnd
        If vFndBld <> "" Then .Font.Bold = vFndBld
        If vFndULn <> "" Then .Font.Underline = vFndULn
        With .Replacement
          .ClearFormatting
          .Text = strRep
          If vRepBld <> "" Then .Font.Bold = vRepBld
          If vRepULn <> "" Then .Font.Underline = vRepULn
        End With
        .Format = True
        .MatchWildcards = True
        .Execute Replace:=wdReplaceAll
      End With
    End If
  Next
End With
DocSrc.Close False
Set DocTgt = Nothing: Set DocSrc = Nothing: Set Rng = Nothing
Application.ScreenUpdating = True
End Sub

The macro will automatically process the open document. Simply change "Drive:\FilePath\FindReplaceTable.doc" to point to the document containing your table.

Note: Since your table doesn't actually have a flag to indicate whether wildcards should be used, you can't simply have "] (Wild Cards off)" for rule c3. The simplest solution, if you want to text for brackets, is to specify the ones you're interested in it terms that are acceptable for a wildcard Find. For example:

[\[\]\(\)]

  • Only one person is actively editing a single slide at one time.

arkengel say:

The question you are posing is not clear. It seems that there are different issues at play.

Your Hotmail account = Microsoft Account

 

Microsoft Account basically is the ID to log in to services such as "Xbox Music / Video" "Hotmail/Outlook" "Xbox Live" "Windows 8/Windows Phone 8" "Messenger/Skype".

 

1) Do you want to use same gamer tag with a different e-mail address?

If you want to keep the same gamer tag but associate a new e-mail you'll need to do it from an XBOX by going through Account Management. Instructions @ XBOX.Com (scroll down to "Can I switch my gamertag to a different Microsoft account?"

qa guy commelec say:

You are mixing terms -- how do I get those values that are stuck in my aggregate table to make them into values in my update table?

There is no such terms as aggregate table and update table. 

You have an aggregate query and table you want update. 

Create a second query containing the table you want to update and the aggregate query.  Join them on 'Method.'

Update the appropriate field using SumOfCostBymethod from the aggregate query.

Remember to backup the database before running the updates.
