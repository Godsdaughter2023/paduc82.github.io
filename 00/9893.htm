joeu2004 say: shnickins wrote:
testing the two dates... by AND or 1 cell minus the other (looking for 0 as the result)... both methods say the cells are the same.

That does not help me "see" the actual formula.

The point is:  due to a dubious trick in Excel [1], it is possible for IF(A1=A2,TRUE) to return TRUE, but IF(A1-A2=0,TRUE) to return FALSE (!).  Likewise, VLOOKUP(A1,A2,1,0) might return a #N/A error.

In other words, under some conditions, Excel treats two values as equal in some contexts, but not in other contexts :-(.

See the experiment in [3] below.

shnickins wrote:
The cells on intradaydata are not text but values according to ISTEXT.

I suspect this is a reaction to a misunderstanding of my suggestion that you try --TEXT(O$1,"m/d/yyyy hh:mm").

That returns a number, not text.  The double-negative (--) converts text to numeric.

The purpose is to ensure that a computed date/time (e.g. A1+TIME(0,30,0)) is rounded to exactly the same internal representation [2] as the equivalent constant, which I assume you have in your look-up tables in C2:LL1000.

shnickins wrote:
So right now... this returns an error entered in O4
=IF($A4="","",VLOOKUP(O$1,[....]))

and O1 =INDEX(IntradayData!$C$1:$LL$4,4,MATCH(D1,IntradayData!$C$1:$LL$1,FALSE))-TIME(1,0,0)
and the value is 9/7/2011 16:10

if I update the data and change the formula in O1 to reflect the same time as earlier that had the error 
=INDEX(IntradayData!$C$1:$LL$4,4,MATCH(D1,IntradayData!$C$1:$LL$1,FALSE))-TIME(1,15,0) 
value - 9/7/2011 16:10

This does sound like the rounding error that I allude to above.  Change all formulas of the form x+TIME(...) to --TEXT(x+TIME(...),"m/d/yyyy hh:mm").

(Note:  The correctness of "m/d/yyyy hh:mm" might depend on your Regional And Language settings in the Control Panel.  Use whatever date/time format is appropriate for your region.)

For example:

=--TEXT(INDEX(IntradayData!$C$1:$LL$4,4,MATCH(D1,IntradayData!$C$1:$LL$1,FALSE))
-TIME(0,30,0), "m/d/yyyy hh:mm")

=--TEXT(INDEX(IntradayData!$C$1:$LL$4,4,MATCH(D1,IntradayData!$C$1:$LL$1,FALSE))
-TIME(1,0,0), "m/d/yyyy hh:mm")

=--TEXT(INDEX(IntradayData!$C$1:$LL$4,4,MATCH(D1,IntradayData!$C$1:$LL$1,FALSE))
-TIME(1,15,0), "m/d/yyyy hh:mm")

-----
[1] See the poor description of the dubious heuristic under the misleading title "Example When a Value Reaches Zero" at http://support.microsoft.com/kb/78113.

[2] In XL2003, --TEXT(...,"m/d/yyyy hh:mm") sometimes does not return the exact internal representation as the equivalent constant (!).  But I have been told that this is fixed in XL2007 and later, based on empirical data.

[3] To demonstrate, try the following experiment.

A1:  9/7/2011 17:10
A2:  =A1-TIME(1,0,0)
A3:  9/7/2011 16:10
A4:  =IF(A2=A3,TRUE)
A5:  =IF(A2-A3=0,TRUE)

roeann (live.com say:

How do I import Contacts

I have just moved my work from MacBook Pro to a Dell with Outlook 365. How can I move all my contacts from the Mac to the Dell?
richard xu msft say:

Hi Roeann,

Thanks for your feedback. Let me help to explain all procedure for you: since the contacts in Mac OS default Contacts app is .vcf file (vCard), and Outlook for Windows do not suppor the .vcf file (only support .csv file), so you should convert .vcf file to .csv file: import .vcf file to Windows and then export them to a .csv file (you should export the VCF file from Mac OS default Contacts app and copy them to Windows OS):

1. On your PC, open File Explorer (or Windows Explorer if you are using an earlier version of Windows).

2. Go to your Contacts folder: C:\Users\<username>\Contacts

3. At the top of the page, choose Import.

4. Choose vCard (VCF file) > Import.

5. Navigate to and choose the .vcf file that you exported from iCloud, then choose Open.

6. For each contact you want to import to the .csv file, choose OK.

7. When you're finished, choose Close.

8. At the top of the File Explorer page, choose Export.

9. Choose CSV file > Export.

10. Choose Browse to name your csv file.

11. In the File name box, type a name for your .cvs file

12. Choose the fields you want to export into the .csv file and then choose Finish. It's recommended that you accept the defaults for now. You can always create another csv file if you want other fields.

13. You'll get this final message that your contacts have been exported to a .csv file.

Then, you can import the CSV file to Outlook for Windows: https://support.office.com/en-us/article/import-contacts-to-outlook-bb796340-b58a-46c1-90c7-b549b8f3c5f8?ui=en-US&rs=en-US&ad=US#bkmk_import
