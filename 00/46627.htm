¡firedog say:

That is a DNS error (the computer was unable to translate pop-server.triad.rr.com into 71.74.56.73, which is the sort of language that computers understand. There have been a few cases of similar problems recently, so perhaps something has gone wrong with the RR DNS servers. If that's the case, it will usually come right in a few hours or days.

You can harmlessly test the DNS server you're using like this:

  1. Type cmd into the start search box and press Enter.
  2. In the window that opens, type
    nslookup pop-server.triad.rr.com

This should show you first the name and address of your own DNS server, followed by the name and address of the mail server you were trying to contact. If this is something like

Non-authoritative answer:
Name:    pop-server.triad.rr.com
Address:  71.74.56.73 


then the DNS server is working properly.  If it isn't, then there's something wrong either with your DNS server or with the RR DNS record and we'll have to try something else. Close the Command window.

 

If the response was like the one I got, you may have a bad local copy of this information in your DNS cache, so we could try refreshing that:

  1. Type   cmd   into the start search box and press Ctrl-Shift-Enter (three keys at once).
  2. Give permission when you're asked for it.
  3. In the window that opens, type   ipconfig /flushdns   (with a space before the '/') and press Enter.
  4. Close the Administrator window.

Now try Send/Receive again in WLMail. Post any new error message in your reply.

    • net start msiserver
