THANKS!!

davidf2 say:

I suspect that at least the first problem is associated with the text formatting and "style" you are using. Some print publication text formatting just does not translate to html very well.

As I look at your page: http://www.leopardrocksporthorseappaloosas.com/index_files/Page778.htm I see that you are trying to justify the text and have mixed in some tabbed paragraphs. I also see from the source code that you are using: "text-justify:newspaper;  text-kashida-space:50%;text-align:justify;text-justify:newspaper;text-kashida-space:" formatting some places. Newspaper style or formatting does not translate well. Did you copy and paste from Word perhaps? Also if you compare the rendered html page with your Publisher page you might see some hyphenated words that are not hyphenated in the web page. All these things can add up to the text box not rendering the same.

I would try removing the indents on the paragraphs, perhaps adjust spacing you have built into and between the paragraphs and/or try to format the whole text box in the same style... You might have to select all the text > format > styles > clear formatting. Then go back and adjust your layout. You can try to apply a more appropriate style such as one of the "body text" options or just manually edit the layout.

Sometimes it is difficult to find what formatting is causing the problem with the spacing and all you can do is minimize it. One technique I have used is to break up my text boxes into shorter amounts of text. It seems that some of these formatting issues become more noticeable if I have long text boxes and the spacing issues become less noticeable if I break up the text boxes.

The second issue with the added space at the bottom of the page could also be attributed to the formatting or style, but I would first look closely to see if I had anything on my page below the textual menu...perhaps overhanging into the scratch area. Run the design checker under tools and see if that spots anything.

Ultimately it will boil down to playing with your formatting and layouts and adjust what you get when you render your page. I can't tell you specifically, you will just have to experiment until you get closer to what you want. This is one of the challenges of using a DTP designed primarily for print documents to produce web documents...the two are different animals and some print formatting just won't convert correctly. And when that happens you just have to change the formatting.

DavidF

Hi David,

The KB Cliff provided also applies to Windows 10. Please refer to the info he provided to make sure the value of the registry key SignInOptions is 0.

If the issue persists, please try the following method to check the outcome.
1.Exit all Office applications including OneDrive for Business.
2.Clear Windows credential cache: Control Panel ->User Accounts->Credential Manager ->Windows Credentials -> clear all credentials related to Microsoft.
3.Clear the Browser cache: Ctrl+Shift+Delete -> Delete Browser History
4) From Registry Editor, browse to: HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Common\Identity\Identities, and then delete the identity folder. If you are using Office 2016, change 15.0 to 16.0
5) Go to Control Panel to do an Online Repair 
6) Restart the computer and go to the OneDrive for Business library in the browser to sync it again.
WARNING: Using Registry Editor incorrectly can cause serious problems that may require you to reinstall your operating system. Microsoft cannot guarantee that problems resulting from the incorrect use of Registry Editor can be solved. Use Registry Edit at your own risk.
