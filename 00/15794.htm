joeu2004 say: Marie wrote:
One cell within the subtotal formula in G23 has a formula of its own that causes it to be rounded. Here is the information for that formula:
G2: 240
G3: 25.39
G4: 139.82
G22=SUM(G2:G4)*.2 = 81.04
Technically if I were to add G2, G3, and G4 and mulitply them by .2 I would get 81.042, which is rounded to 81.04 in Excel (which is the number that I want in that space).

Normally (when "Precision as displayed" is not set), Excel only rounds the displayed value, not the actual value.  So what might appear to be 81.04 is actually 81.042.

To round the actual value (without relying on PAD), change the formula in G22 to:

=ROUND(SUM(G2:G4)*0.2,2)

See if that corrects all of your problems.  I have not worked through all the other details in your posting.

Marie wrote:
I have already tried checking the "precision as displayed" box in the advanced settings but it only worked for the formulas that display a result that is .01 over what the correct result should be. Those that are .01 under remain incorrect.

I am not quite sure what led you to this incorrect conclusion.

If a cell is formatted as Number with 2 decimal places (or the equivalent), PAD will round the result to exactly what is displayed.  With your example, SUM(G2:G4)*0.2 would result in "exactly" 81.04 [1] instead of 81.042, if the cell is formatted as Number with 2 decimal places.

In any case, I do not recommend the use of PAD in general.  If you experiment with it, be sure to make a copy of the Excel file first because PAD can change some constants irreversibly.
