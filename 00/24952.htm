rich michaels - mvp (greatcirclelearning.com) say:

In Word, list numbering is contextual to the list(s) in the document. This means that other numbered lists that exist, or not, effect the numbering. If you copy and paste a partial numbered list from one document to another, the destination numbering will be influenced by any numbering that already exists in the destination document. For example ...

If you copy list items numbered 3, 4, and 5 from a source document, and paste them into a new destination document that does not have a list, the pasted items will be numbered 1, 2, and 3. If the destination document already contains a numbered list, the pasted items will automatically convert, dependent on location, to a numbering scheme relative to the current numbered list. If the pasted location is within a table, the results can get very strange because the numbering is determined by the last list item entered, which might not be in the same table row or column.

Document text boxes and programmatically generated user form text boxes also work independently of the source document. This means that if you copy and paste items from a numbered list in a base source document into a text box, the resultant numbering will convert just as if you were pasting the items into a new document. In other words, what I have told you about the list numbering behavior when pasting into a new document, also applies to pasting into a Text Box.

With all of the above as background, your question concerns programmatically controlling or probably better said, adapting to this behavior.

There are two VBA methods to look into because they should help you adapt your plug-in code to handle the list item data the way you need it.

Selection.Range.ListFormat.ListValue - this will return the list number value of the selection.

Selection.Range.ListFormat.ConvertNumbersToText - this will, as the method name suggests, convert the list numbering scheme to regular text and that is probably what you want to use in your Text Box.

Experiment with these two methods and see if they help to resolve the issue you are having.

jllatham say:

I wasn't sure what you need to do after it is initially created and you'd said "left click on the sheet" to have it move to the final destination.  Gord has provided one way of halting things and then making the final move.

I actually had envisioned that all that code near the end that calculates the final destination would quite possibly reside (along with proper declarations) in another routine entirely such as the Worksheet_SelectionChange() or Worksheet_Activate() processes - and I said so in the comments.

In this case, I'm a little like JP Ronse - curious as to the WHY you would want to wait for the move to be made unless there is some specific actions you need to make dealing with the textbox itself before making the move.  And with the use of the InputBox() like Gord has shown, there really isn't anything that you could do on the sheet or text box before the move is made: the whole show stops until you do hit [OK] to acknowledge it.

Now, here are two routines that would do that kind of thing: the first one, which is what you already have with some stuff removed will create and initially position the TextBox.  The TextBox will still be selected and you could type some text into it.  The second piece of code, which is worksheet event code for that worksheet will move the box to its final resting place as soon as you click a cell in the worksheet.

Sub AddTextBox()
  Const UpperLeftCell = "P23"
  Const LowerRightCell = "Y34"
  Const myBoxName = "MyTextBox"
  Dim P23_Left As Double
  Dim P23_Top As Double
  Dim P23_Height As Double
  Dim P23_Width As Double
  Dim Y34_Left As Double
  Dim Y34_Top As Double
  Dim Y34_Height As Double
  Dim Y34_Width As Double
 
  Dim upperLeftX As Double
  Dim upperLeftY As Double
  Dim lowerRightX As Double
  Dim lowerRightY As Double
 
  Dim boxHeight As Double
  Dim boxWidth As Double
 
  With Range(UpperLeftCell) ' P23
    P23_Left = .Left ' P23 left edge
    P23_Top = .Top ' P23 top edge
    P23_Height = .Height ' height of P23
    P23_Width = .Width ' width of P23
  End With
  upperLeftX = P23_Left + (P23_Width / 2)
  upperLeftY = P23_Top + (P23_Height / 2)
 
  With Range(LowerRightCell) ' Y34
    Y34_Left = .Left ' Y34 left edge
    Y34_Top = .Top ' Y34 top edge
    Y34_Height = .Height ' height of Y34
    Y34_Width = .Width ' width of Y34
  End With
  lowerRightX = Y34_Left + (Y34_Width / 2)
  lowerRightY = Y34_Top + (Y34_Height / 2)
 
  boxWidth = lowerRightX - upperLeftX
  boxHeight = lowerRightY - upperLeftY
  'before adding this delete any with same name!
  'this prevents piling them up on top of one another
  On Error Resume Next
  ActiveSheet.Shapes(myBoxName).Delete
  On Error GoTo 0
  ActiveSheet.Shapes.AddTextBox(msoTextOrientationHorizontal, _
  upperLeftX, upperLeftY, _
  boxWidth, boxHeight).Select
  Selection.Name = myBoxName ' so you can find it when you need to
End Sub

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
  Const DestinationCell = "P34"
  'this constant must have same name as used to create the box
  Const shapeToMoveName = "MyTextBox"
  Dim anyShape As Shape
  Dim myBox As Shape
 
  'is there a shape with the correct name on the sheet?
  For Each anyShape In Me.Shapes
    If anyShape.Name = shapeToMoveName Then
      Set myBox = anyShape
      Exit For
    End If
  Next
  If Not myBox Is Nothing Then
    'the shape to be moved does exist
    'calculate where to move it TO
    'and just put it there
    'if it's already there, then so what?
    With myBox
      .Left = _
       Range(DestinationCell).Left + _
       (Range(DestinationCell).Width / 2)
      .Top = _
       Range(DestinationCell).Top + _
       (Range(DestinationCell).Height / 2)
    End With
  End If
  'housekeeping
  Set myBox = Nothing
End Sub
