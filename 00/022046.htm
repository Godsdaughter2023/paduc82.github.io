Does Windows Defender autorun on startup in Windows 10... - Microsoft Community

{blurgh} say:

Does Windows Defender autorun on startup in Windows 10...

...even though it does not show in the icon bar until after I've manually started the program?
greginmich say:

The core of Windows Defender is its Antimalware Service Executable, aka the Antimalware Engine (MsMpEng.exe), and this file runs automatically at startup – but the Windows Defender user interface (MSASCui.exe) doesn’t run automatically. So in order to get the icon at startup, you have to run the user interface at startup. Windows includes a Startup folder just so users can run programs automatically at startup (without editing the registry), so you can get your Defender icon by just dropping a shortcut for MSASCui.exe into the Startup folder:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/is-windows-defender-icon-supposed-to-be-in-system/6cfa9d1d-3db8-49a9-add7-280a284ccfdc

The Startup shortcut method also gives you the option for an automatic update and/or Quick Scan along with your icon. You just enter the run line for the desired option as the location (target) of the shortcut:

"C:\Program Files\Windows Defender\MSASCui.exe" -hide

"C:\Program Files\Windows Defender\MSASCui.exe" -Update -hide

"C:\Program Files\Windows Defender\MSASCui.exe" -QuickScan -hide

"C:\Program Files\Windows Defender\MSASCui.exe" -UpdateandQuickScan -hide

We can use these predefined startup lines to make the icon/task setup a piece of cake:

1.)  Select and copy the line for the option you want.

2.)  Right-click on the desktop and select New > Shortcut and then paste the run line as the location (target), and name the shortcut.

3.)  Open File Explorer: Start Menu > File Explorer; and type (or copy and paste) shell:startup (user Startup folder) or shell:common startup (common Startup folder) into the Quick access box to open the Startup folder.

4.)  Then drag and drop the shortcut into the Startup folder. All done!

In case anyone was wondering how Brink’s reg file works, it actually just starts the Windows Defender user interface by adding exactly the same startup line to the Run key for the current user. Here’s the entire contents of the file:

Windows Registry Editor Version 5.00

; Created by: Shawn Brink

; http://www.tenforums.com

; Tutorial: http://www.tenforums.com/tutorials/11974-windows-defender-notification-area-icon-hide-show-windows-10-a.html

[HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run]

"Windows Defender"="\"C:\\Program Files\\Windows Defender\\MSASCui.exe\" -hide"
