Well, now that I had the weekend to think this over, I have a solution ... this is too simple!!!

There's more than one way to skin a cat ... (no, I don't hate cats :)

I already have an SQL Pass-Through query that retrieves the desired records.  (Or, if I didn't have this query,  I could use the Microsoft kb article 304323 to create the query from a recordset.)

(On the Title slide I have Presented: <manually type date> - slides show last saved date.  I use the autodate to keep track of the age of the printed version of the slides.)

=====
The Slide Master is a hierarchy.  The Slide Master at the top of the tree influences the formatting of the Slide Layouts in the tree.  The Slide Layouts influence the slides.  So set the formatting in the Slide Master and it will be applied to the Slide Layouts - if not follow the actions below. 

If a Slide layout is not following the Slide Master you can select the offending placeholder box and press Ctrl-Space to clear all local formatting.   You can also do that to highlighted text.

If that doesn't work, delete the placeholder and insert a new one (from the Master Layout tab).  Choose the placeholder and drag out the area it has to occupy and click.

(The Content place holder gives you text and the multi-type of auto-media insert selector)

You can have multiple Slide Masters in a template and each Slide Master can have its own set of Slide Layouts.

You can create extra layouts and given their own descriptive names.  Right click on the layout and select rename.

When slide masters are all done, and include date (fixed or automatic) and page number in the Slide Master - don't forget: Apply to all.  Check the slides after closing Slide Master.  If the date and page number are not visible on the slides: Insert > Header & Footer > select date and page number and tick don't show on title page.  Don't forget: Apply to all.
