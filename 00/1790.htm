hansv mvp say:

In my humble opinion, that goes against the grain of how Access works. If Limit to List = Yes, users can only select/enter values from the list, anything else will be rejected. But if you really want, you could create an event procedure for the On Key Down event:

Private Sub ComboBoxName_KeyDown(KeyCode As Integer, Shift As Integer)
  Select Case KeyCode
    Case vbKeyTab, vbKeyReturn, vbKeyUp, vbKeyDown
      ' OK
    Case Else
      KeyCode = 0
  End Select
End Sub

The user can still use the up and down arrow keys in the list, and press Enter or Tab to move to the next control, but cannot type in the text box part of the combo box.

Copy/paste and the like isn't possible either.

marshall barton say:

Don't forget that you can drop the list automatically by using a line of code in the GotFocus event:

   Me.[the combo box].DropDown

That and the LimitToList should be all you need without abandoning the standard user interface design guidelines.  Preventing the use of the keyboard here is only a short way from disconnecting the keyboard.
