The easiest way to remove an entry in the Windows Vista Boot Configuration is to use msconfig.

ahicks66 say:

I may have posted prematurely... I have just fixed the Java issue and both the x86 and x64 versions have installed successfully. I will post my solution below in the hopes that others can benefit from what I discovered!

  1. Perform a Clean Boot to an account with Administrator privileges.
  2. Download Revo Uninstaller and uninstall each program that uses Java. Delete all files and registry keys associated with the programs you are uninstalling.
  3. Download Piriform CCleaner and delete all remaining registry keys that are associated with Java and Java-based programs (including all Minecraft-related programs like World Painter, etc).
  4. Finally, download McAfee Consumer Product Removal Tool 9.1.142.0 and uninstall any failed McAfee installations that were botched - apparently, in my case, un-checking the bundled McAfee junk in the online Java installer was ignored as McAfee was partially installed (I have never used McAfee on this machine and it was not bundled/pre-installed from Sony).
  5. Reboot (Clean Boot)
  6. Download the Java Offline Installer Packages from Oracle's website. In my case, since my PC is 64-bit, I downloaded the 32-bit version AND 64-bit versions.
  7. Install the 32-bit version by right-clicking the executable (EXE) file and selecting "Install with Revo Uninstaller Pro" from the context menu. (Revo makes a log file of installations using this method, I used it merely for diagnostic purposes if it didn't work.)
  8. After installation is complete and you have verified installation on Java's test page, install the 64-bit version the same way you did in #7 (only applies for 64-bit installations and even so, is optional).
  9. Double-check Java is in the Control Panel, re-enable your normal startup programs, and enjoy!

Hope this helps everyone out. Some of the steps may have been unnecessary, but it worked for me. For all I know, it could have been the botched unwanted McAfee install that was the whole problem. I just tried a handful of possible solutions all at once! :-)
