I've been at this the whole night and I think I finally figured it out.

The reason why Microsoft Family cannot be accessed is because the signed in account (the account you signed into when booting your PC) is a local account and not an administrator account. Check if this is the same in your situation. If it is, then do the following:

Click windows key.

Click Settings.

Click Accounts.

Click "Sign in with a Microsoft Account instead"

Follow prompts logging into your microsoft email account (hotmail/outlook), and set up a PIN (I think this is optional).

Make sure that when you get back into "Settings/Accounts/Your Email and Accounts" you will see your Username, Microsoft Email, and Administrator on the first three lines.

There will also be a gold-colored text that says "Verify", click this and it will prompt you for the last 4 digits of your contact number.

A verification code will be sent to your phone and use this to verify your account.

Once verified, the child account that was previously created will now be able to sign in, or if that has not been created yet, you can now add a family member. It will be much easier if the family member (child/adult) also uses a microsoft email account. I was not so lucky setting up a family member using any other email account.

Hope this helps.

-Registry hack for DNS query

lincmerclover say:
FINALLY found local group policies that produce the desired network behavior in Windows 10...

Computer Config>Administrative Templates>Network>
     WWAN Service>WWAN Media Cost>
          Set 3G Cost - Enabled/Unrestricted
          Set 4G Cost - Enabled/Unrestricted
     WLAN Service>WLAN Media Cost>
          Set Cost - Enabled/Fixed (Might not need this policy)
     Windows Connection Manager>
          Minimize the number of simultaneous connections to the Internet or a Windows Domain - Disabled
     Network Connectivity Status Indicator>
          Specify global DNS - Enabled/Use global DNS
