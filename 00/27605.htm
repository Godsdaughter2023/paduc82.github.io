I have the same problem. Based on another thread, Microsoft has reproduced the problem and is working on a fix. https://answers.microsoft.com/en-us/msoffice/forum/msoffice_excel-mso_win10-mso_365hp/lost-formatting-in-excel-2016/87ca9032-aba6-4a03-86bb-a46a875f12be

I guess for the time being, I found a workaround for the issue that I posted on an earlier thread:  https://answers.microsoft.com/en-us/msoffice/forum/msoffice_excel-mso_win10-mso_2016/excel-2016-keeps-on-changing-fonts/9097f1a2-dff3-42a5-bb65-cc55125aa4ae

1) The font change applies to text after subscripts and superscripts and changes in font size, and basically any change in font properties within the same cell. For instance, ex: F_d = m*a; the font would change starting right before "d".

2) Excel changes the font to whatever properties you set in your "Normal" style font. (In styles, right-click the style and click "Modify" to see the properties.)

3) If you open the file through Excel (rather than directly through the desktop or folder), the font issue does not occur. If you have Excel open already (i.e. a new workbook), the font issue does not occur in other files when you open them directly through the desktop or folder. So definitely a software startup issue. 

This has worked for me and several other people.

j.e. mcgimpsey say: One way:

Select the rows you want to CF (or the entire sheet):

Formula is: =$A1<TODAY()
Format: <fill as desired>

Or, if you want to leave blank rows unformatted:

Formula is: =AND($A1<>"", $A1<TODAY())

This assumes your dates are actually dates, rather than Text (to convert text to dates, copy a blank cell, select your 'dates', and choose Edit/Paste Special, selecting the Add and Values radio buttons. You may then need to reformat the cell as a date).

ken sheridan say: You can use the built in Database Documenter from the Analyze area of the Database Tools ribbon to output the code for all standard and custom class modules, and for all forms' and reports' class modules.

For standard and custom class modules, in the modules tab of the documenter dialogue Select All, and then select Options, and in the next dialogue select Code only from the available options.

For forms and reports in the forms and reports tabs Select All and then select Options, and in the next dialogue select only the Code option in the top part of the dialogue, and select Nothing in the lower part.

The documenter will output the code as a report in print preview.  Select Text File from the Data area of the Print Preview ribbon to output the results to a text file from which you can copy and paste any of the text in the usual way.  Note that the documenter will show some lines which are normally hidden when the code is viewed in the VBA editor.
