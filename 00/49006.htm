dr.slinky.mcvelvet say:

I think I found the solution, you need to open Windows Defender and click the second item in the side menu 'Virus & Threat Protection' then click 'Ransomeware Protection' --> 'Protected Folders' and then 'Add a Protected Folder', this can be a folder of your choosing.

I noticed having added additional protected folders stopped the warnings about setting up OneDrive.

One thing you will notice though is that Windows will start alerting you whenever a program tries writing files to a protected folder which in my case meant having to also 'add an allowed app' to the exceptions list (shown below) to allow certain programs to even work properly, clearly this feature does work but it's not user friendly and might even dissuade some users from actually using it.

So far I've added the following apps to the exceptions list. It seems necessary to at least add your browser because you won't be able to download anything otherwise, especially if your download folder happens to be a protected folder.

 

In addition to the above you should also note that in order to add a program to the exceptions list that program must be closed when doing so otherwise it will keep alerting you. Once the exception has been added you can then open the program and use it normally.

Of course one observation which Microsoft seemed to have overlooked is that when installing programs many will try adding a program shortcut link to your desktop in order to launch the program after installation, however the issue arises whereby the users desktop is added to the protected folders list by default and it cannot be removed. Basically meaning that you won't be able to install new programs or updates unless you choose not to add desktop shortcuts. You could however add those shortcuts manually if needed but it's more work and seems counterproductive.
