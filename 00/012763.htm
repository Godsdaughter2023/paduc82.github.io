Sub Back_Vessel_Button_BrazilSW()

I think best I can do is offer some debugging help for you to use.  What you could do is set up to create a log file of the processing done within the application, and examine that log file when it suddenly fails.  This might help narrow things down for you.

 

Here is how I would do it:

 

First a little code in the Workbook_Open() event processing like this which will create/overwrite a file named MyLogFile.txt in the same folder with the Excel file.  This ensures that when you view the log file contents you're not looking at a lot of old

 

Private Sub Workbook_Open()
  'clear my "event" log text file
  Dim logFileName As String
  Dim logBuffer As Integer
 
  logFileName = ThisWorkbook.Path & Application.PathSeparator & "MyLogFile.txt"
  logBuffer = FreeFile()
  'this simply clears out the text file, so don't open this
  'file after a failure until you've examined MyLogFile.txt
  'and possibly even saved it with a different name.
  Open logFileName For Output As #logBuffer
  Print #logBuffer, "File Opened for Use at " & Format(Now(), "dd-mmm-yyyy hh:mm:ss")
  Close #logBuffer
End Sub


Now add a code module and place this code in it:

 

Sub MakeLogEntry(logEntry As String)
  Dim logFileName As String
  Dim logBuffer As Integer
 
  logFileName = ThisWorkbook.Path & Application.PathSeparator & "MyLogFile.txt"
  logBuffer = FreeFile()
  'this simply adds entries to the logfile
  Open logFileName For Append As #logBuffer
  Print #logBuffer, logEntry
  Close #logBuffer
End Sub

Next, be generous with calls to this so that you can track the path your code is taking towards its doom.  Simple example using the UnprotectSheet() code:

 

 

Sub UnprotectSheet()
Call MakeLogEntry("Entered UnprotectSheet()"
Sheets("2. Inst. Cost").Select
    Call MakeLogEntry("Sheet 2. Inst. Cost selected")


    ActiveSheet.Protect Password:="XXXX", Contents:=False, AllowFormattingCells:=True, AllowFormattingColumns:=True, AllowFormattingRows:=True, AllowInsertingColumns:=True, AllowInsertingRows:=True, AllowDeletingColumns:=True, AllowDeletingRows:=True

   Call MakeLogEntry("Sheet 2. Inst. Cost protected - Exiting UnprotectSheet()")

End Sub

 

Like I said, be generous with the calls to MakeLogEntry.  Then when you have a failure you'll be able to examine the file to find what the last successful action was.  If things seem to be going along fine again at some point (you find and fix a problem) you can leave the MakeLogEntry process in the workbook, but take it out of the picture by putting a simple EXIT SUB statement as the first executable statement in it.  Then if things get problematic again, just remove that and once again you have a way of getting an inside picture of what's happening even when Excel crashes without giving you any real clue as to why it died.

a. user say:

Hello Jon,

 

Thank you for getting back to us. It is possible that your sender has mistyped the email address when he was trying to send you a message. If your sender is using Outlook.com to send the message, I suggest that you have him try the steps below:

 

1. Go to Inbox.

2. Click on the options icon located on the upper right part of the screen. Choose More mail settings from the dropdown (it could be More email settings for Firefox and Chrome browsers).

3. Under Customising Outlook, choose Advanced privacy settings.

4. A new window will appear. Under Auto-complete suggestions, you will be given 2 options.

5. Choose Only suggest people in my contact list.

6. Click Save.
