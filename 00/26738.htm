If your site doesn't require Limited-access user permission lockdown mode, deactivate this site collection feature.

Other features such as publishing may require this feature to work correctly.

When you share a folder with a user who can't access the parent folder or site, SharePoint assigns the user limited access to the parent items. Specifically, SharePoint lets the user access the folder without obtaining permission to access the parent folder and other items (other than limited access). However, after Limited-access user permission lockdown mode is enabled, the user doesn't have access to the folder because the necessary limited access permission on other items no longer works correctly.

The Limited Access permission level is unusual. It lets a user or group browse to a site page or library to access a specific content item without seeing the whole list. For example, when you share a single item in a list or library with a user who doesn't have permission to open or edit any other items in the library, SharePoint automatically grants limited access to the parent list. This lets the user see the specific item that you shared. In other words, the Limited Access permission level includes all the permissions that the user must have to access the required item.

paul vg say:

Thanks for your reply - I've just used the fixit program to uninstall 2003, and then just to be sure I used the fixits intended for ininstalling 2007 and 2010.  Then I ran setup.exe and unfortunately it hung again.  This error message showed up: "Error 1935. an error occurred during the installation of assembly component {A75F2217-AD54-3EA6-AE14-F255F8660531}. HRESULT: 0x800736FD

I googled "Office 2010 Error 1935" and found this:

http://windows7forums.com/windows-7-software/41887-office-2010-error-1935-error-occured-during-installation-assembly-component.html

It appears that there is a known bug in the installer that prevents a successful setup when there are older versions of Office installed.  The recommendation is to completely uninstall Office first - which I have now done.  This didn't completely do the trick though - but HERE'S THE GOOD NEWS (yay!)

It turns out that there is a second potential difficulty with the  install.  When I googled:

HRESULT: 0x800736FD

I found this:

Solution: http://fiestachapina.net/informatica/2010/06/25/error-1935-0x800736fd-al-instalar-office-2010/

It's a problem on registry

 

It's in spanish, which I don't know, but one can tease out what's going on.  run 'regedit' and go to:

HKEY_LOCAL_MACHINE

  SYSTEM

    CurrentControlSet

      Control

        RegistrySizeLimit

double click on RegistrySize Limit and set it to FFFFFFFF (eight f's)

 

After doing this, and with older Office versions removed, I tried once again to install 2010 and it worked!
