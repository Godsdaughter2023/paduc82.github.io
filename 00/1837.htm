brian tillman [outlook mvp 2007-2019] say:

If you connect Outlook 2013 to Outlook.com allowing Outlook to configure the account automatically, your Contacts and Calendar should sync.  No export/import needed.

You can add as many accounts on Outlook.com as you wish or you can add extra addresses to your single account as aliases.  With the former (i.e., each address is a separate account) you can send/receive in Outlook 2013 and people to whom you send mail will see the separate addresses.  If you use aliases (multiple addresses but one account), you can send/receive in Outlook 2013 as well. Incoming mail can be delivered all to one Inbox or you can use rules to separate the incoming mail into separate folders (either on the server side or the client side), but it will still be one mailbox. If you send, the recipients will see only your primary address.  If you send from the outlook.com web site, however, your recipients will see the sending address, which can be any of the aliases.

You can set up a connection to your Outlook.com mailbox on the iPhone and it will sync the calendar and contacts to the phone as well so your phone and Outlook 2013 will contain the same data.  If you use separate accounts, you can see multiple calendars (one per account).  Outlook 2013 will show multiple Contacts folders as well, but I don't know what the phone will do with multiple contacts folders.

rich007a say:

I don't think Image controls on userforms have scroll bars.  To do this, I would create a Frame control.  Set the ScrollHeight and ScrollWidth of the frame to equal the dimenssions of the image (see here: http://www.xtremevbtalk.com/showthread.php?t=292134 to get the dims of the image).

In the frame, add an image control, and make the size of that image control the same as the picture.  Set the Left and Top properties of the Image control to zero (in the frame).

User should then be able to scroll around the image using the scrollbars of the frame.  You could add other buttons to enable zooming, but you'd need to keep the frame's scroll hight&width the same as the image control's height and width.

You also need to set the frame's ScrollBars property to frmScrollBarsBoth.

Make sense?
