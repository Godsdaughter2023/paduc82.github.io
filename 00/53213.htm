As far as I know, the former just asks you go to your legacy application that is already installed in Windows 7, get its property and check the Compatibilit Mode check box and select your old Windows OS (e.g. Window XP,SP2) - that seems no any extra HW/SW requirement beyond your Windows 7 OS; But the latter asks for extra 1 GB RAM and also installing Virture PC Mode before you can run the XP Mode. Are the two modes the same thing or different ones? It seems to me that the Compatibility Mode is to individual legacy Windows application, so no extra HW/SW requirement, and the XP Mode is to the entire OS enviroment. Is that right? Appreciate if anyone can clerify this.
john_barnett say: Hi,

No they are not the same.

Compatibility mode enables you to run legacy software simply by, as you say, selecting properties and then checking compatibility mode and selecting the operating system suitable for running that application. As you say 'no extra SW/HW requirements.'

XP Mode is, to all intent and purpose, a virtual machine mode. You first need the Virtual PC software installed and then, after you create a virtual machine, you install your software and it runs as if it is running on a seperate PC - hence the need for an 1GB additional RAM. The 1GB of RAM is allocated to the virtual machine. If you have say 4GB RAM then your PC would use 3GB and your 'virtual' PC would use the other 1GB.

It is also worth pointing out that your PC also has so support XP Mode, if it doesn't then XP Mode cannot be installed.
