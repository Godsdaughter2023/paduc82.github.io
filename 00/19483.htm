Sum of Yes/No control returning negative number - Microsoft Community

stevenmoss say:

Sum of Yes/No control returning negative number

I have a Form which has a nominal list of my students with a Yes/No control "Attended", selected Yes if they attended the Class that day.

In the Form Footer I have a textbox with the formula:

=Sum([Attended])


This is returning the correct number; only in negative form. What am I doing wrong. My idea is to give a total of all the Yes and then I will add a control to give a total of No for this Form.

Steve

ken sheridan say: You need to use an expression which sums the value of the Pay Awarded column by the return value of an expression which returns 1 or 0 on the basis of the value of the value in the Relationship column:

=Sum([Pay Awarded]*IIf([Relationship] = "Staff",1,0))

Note that you can only use aggregation operators in group or report footers, not in a page footer.

You refer to 'other controls within the footer'.  Are these referencing the above control?  If so you should not do so, but repeat the above expression as part of the expressions used as the ControlSource properties of the other controls.

ken sheridan say:
..............and then I will add a control to give a total of No for this Form.

That you will have to do differently as the summation of a set of zeros is zero.  This will return the count of False values:

=Sum(IIf(Attended,0,1))

and of course:

=Sum(IIf(Attended,1,0))

will return the count of True values.  The head of a software company of my acquaintance always insisted on the latter approach in this sort of context rather than summing the absolute values of the column.  To use his words, he regarded the latter as being 'unduly chummy with the implementation'.
