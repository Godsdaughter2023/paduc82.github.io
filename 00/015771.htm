Hi...this one's easy...ready?

 

Click the File tab, and select the "Info" option, which displays permission and restriction settings....er, *scanning the rest of your question as I type, and backspaced my "easy answer out*.

 

OK, I've had MASSIVE issues exactly as you describe, and I have figured out why this happens, but not the trigger that changes the UAC permissions I had just the day before.

 

Now, I'm assuming your document is in your user folder, and not in a protected Windows folder (ie., users do not have permission (even as admins) to save, edit, or delete a Word doc in the Program Files(86) Microsoft Office folder; so it shouldn't have allowed you to save it there (or on system protected folder) in the first place. But a personal computer's PRIMARY function is to frustrate the user until all hair has been ripped out. Strange things happen. :-)

 

Moving on...

 

If the folder your word doc is saved in has a "lock" symbol on it, it means it's protected. I won't go into how this happens to a user's personal folder(s), but it is constantly happening to me, and I have to go reclaim ownership of the folder. Try the following:

 

1.        Right-click on the folder your word doc is on, go to properties, and on the general tab, is the Read Only box filled in or checked?

 

2.        If yes, click on the security tab, and click on your user name in the Group or user names box.  In the box below, it will list the permissions you have.  If you have checked to read & execute, list folder contents, and read, this is the problem.  Continue to step 3.  If no, jump out of this list and refer to the bullet, below the closing paragraph.

 

3.        Don’t waste time trying to “edit” your permissions, just go straight to the Advanced button and click.  This opens the Permissions options.  Click on the Owner tab.  Whoever is listed as the current owner needs to be replaced.  In the Change owner to: section, you are probably listed.  Click Edit, which will enable you to select your user name among any others listed here, then click the box “Replace owner on subcontainers and object”, then hit Apply, then OK.

 

4.        NOW, you will be back at the main security tab, be able to click your user name, and if you don’t have full “Full Control”, you can click edit, click on full control, “apply”, then  “OK.”  Click OK all the way back to the General tab, and close the properties window.

 

5.        Re-open the properties for the folder, (or doc), and uncheck “Read only.”  Apply, OK, close.

 

 

This should solve ALL permission issues for you, in THAT folder.  (Personally, I just check my User folder, and follow the steps above, so I reclaim all rightful ownership permissions of my personal user files.  But, I am not a certified Windows technician, so I recommend just following these steps for the folders YOU created, in your user folder, as needed.)

 

  • Note:  you MAY just need to “share” this folder, with yourself (depending on your share settings.)  Try clicking on the Sharing tab, in the Properties view, and if it indicates this is not shared, change that.  I’m sure you know these steps.

 

I hope this helps!  Let me know how it goes, and please, follow my advice, as follows:

 

 

Do NOT attempt to change ownership on folders you did not personally create.  The UAC (User Account Control) is a very complex “protocol” that maintains the security and permissions on ALL objects, classes, groups, users, etc, for the entire OS system.  You must have admin rights to proceed making any changes like I describe above.  If you are an admin, it will prompt you that only an Administrator can proceed, and you’ll have to click “Yes” (or Proceed…I forget) to continue.  If you are denied access, you either are not signed on as the Admin, or you are attempting to modify a file or folder that is defined, by UAC settings, as System protected.

 

Good luck!
