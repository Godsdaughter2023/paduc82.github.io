marc.kupper say:

Here is how I was able to get rid of conime.exe.

First of all - conime.exe appears when you first run "Command Prompt" (cmd.exe) and does not go away when you close the prompt.  If you run two or more copies of Command Prompt you usually only get one copy of conime.exe running. Some applications also run Command Prompt as part of what they do. So, even if you never started a Command Prompt it's still possible for you to see conime.exe in the process list.

As others have said, conime.exe is the Console Input Method Editor which is sometimes shortened to "Console IME" or "conime." Its purpose is to allow you to type Asian languages within Command Prompt boxes.

I tested and found that enabling keyboard input support for Hebrew, Arabic, and Hindi all cause conime.exe to be started when I start a command prompt. I would not be surprised if conime is also needed for languages from Africa or other areas of the world.

To prevent conime.exe from starting you need to get rid of Asian, and possibly more, language support for the keyboard.

  1. Run Control Panel
  2. Run Regional and Language Options.
  3. Select the "Keyboards and Languages" tab.
  4. Click the "Change Keyboards" button.
  5. On my systems I was seeing "English (United States)" at the top of the installed services list followed by "Japanese (Japan)".
  6. Remove all of the Asian languages from the list. and then click [Ok], [Ok], [Ok], etc. to close the windows.
  7. Log off and back on.  You don't need to do a full reboot but I found that I needed to log off and back on to get the changes to the keyboard input methods to fully take effect.

A shortcut for steps 1 and 2 above is to run intl.cpl.  For testing I was running intl.cpl directly rather than pulling up Control Panel and selecting Regional and Language Options each time.

On step 6 I noticed under Japanese a line with "Microsoft IME" and tested with removing just that to disable the IME.  This did not work and conime.exe kept reappearing. If I had either "Japanese" or "Microsoft IME" underneath "Japanese (Japan)" then conime.exe would start when I started a command prompt. The fix was to remove both of them or to remove "Japanese (Japan)" which deletes everything underneath it.

Step 7 (log off and back on) is critical. It's possible you can get the settings to take by killing and restarting explorer.exe.  You don't need to do a full reboot though.

There is a registry setting at [HKEY_CURRENT_USER\Console], "LoadConIme"=dword:00000001 but setting LoadConIme to 0 did not help. conime.exe was still starting when I started a Console Prompt.

janetstaub say:

angelcarreno say: Hi,

My name is Angel. I am an Independent Advisor. Thank you for posting in Microsoft Community.

Check your privacy settings:

If you recently updated Windows 10, you may need to give apps permission to use the camera. This is because in newer versions of Windows 10, some apps don’t have default access to the camera.

To let apps access the camera, select the Start button, and then select Settings > Privacy > Camera. Then turn on Allow apps to access your camera.

After allowing access to the camera, you can choose which apps can access it under Choose which apps can access your camera.

Check your antivirus software:

In your antivirus app, look for settings related to blocking access or permission to use your webcam. For help, visit your antivirus software company’s website.

Check for an outdated webcam driver
1. Select the Start button, type Device Manager, and then select Device Manager from the search results.
2. Find your webcam under Imaging devices or Sound, video and game controllers.
3. Press and hold (or right-click) the name of your webcam, and then select Properties.
4. Select the Driver tab, select the Driver Details button, and look for a file name that includes stream.sys. If it’s there, your webcam was designed before Windows 7 and you'll have to replace it with a newer webcam.
5. If you don't find a file name that includes stream.sys, try rolling back your webcam driver.


Roll back your webcam driver
1. In Device Manager, press and hold (or right-click) your webcam, and then select Properties.
2. Select the Driver tab, select Roll Back Driver, and then choose Yes. (Note that some drivers don’t provide a rollback option.)
3. After the rollback is complete, restart your PC, and then try opening the Camera app again.
4. If rolling back didn’t work or isn’t available, uninstall your webcam driver and scan for hardware changes.
Uninstall your webcam driver and scan for hardware changes
1. In Device Manager, press and hold (or right-click) your webcam, and then select Properties.
2. Select the Driver tab, select Uninstall > Delete the driver software for this device, and then select OK.
3. In Device Manager, on the Action menu, select Scan for hardware changes. Wait for it to scan and reinstall updated drivers, restart your PC, and then try opening the Camera app again.
4. If your webcam still doesn’t work, try looking for help on the webcam manufacturer’s website.
