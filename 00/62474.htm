Trusted PC:

A trusted PC is a computer that you've added to the password reset information for your Microsoft account. This information helps Microsoft verify the identity of the person using the trusted computer.

Note: You should only designate a computer as a trusted PC if you're using a private computer. If you share your computer with others, anyone who uses the computer can reset your password without your knowledge.

What is a trusted PC?:

Trusted Device:

If you ever forget your password, or if someone gains unauthorized access to your account, it's easier to reset your password from your trusted PC. When you request to have your password reset from a trusted PC, Microsoft trusts that you are who you say you are and doesn't require you to provide any additional information to verify your account.

When you try to view or edit sensitive info related to your Microsoft account—like your credit card details—we might ask you for a security code first, to make sure that only you can get in to your account. But if you use a device often you can designate it as a trusted device. On trusted devices, you don't need to enter a security code each time you try to access sensitive info.

You can mark a device as trusted just by selecting a check box. When you're prompted to enter a security code to verify your identity, select the “I sign in frequently on this device. Don't ask me for a code” check box, and we'll no longer ask you for a code when you sign in on that device.

Trusted devices: FAQ: http://windows.microsoft.com/en-in/windows/what-is-trusted-device

Re-build Windows 7

If you are going to do a re-install, you may be able to take your data off first.  Remove the hard drive and put it into a USB external drive housing.  Connect to a working PC, copy the data off.

Always check to ensure the hard drive you are about to rebuild is actually sound from a hardware perspective.  Download and run your hard drive manufacturer’s drive testing software.

First off, if you have not already tried and failed, you want to try a System Repair disk to boot the PC.  It may be more successful in fixing and offer other alternatives.  If you do not have one, go to any other PC that has windows 7 with the same bitness (32/64).  Start globe, All Programs, Maintenance.  You need just a blank CD.  When you boot, ensure that you are in fact booting on that CD and not the hard drive.  Usually pressing F12 will lead you to the option to do so.

If that does not work, you are likely into a re-install.  Takes a lot of time, but not all attended time.

  1. Factory restore — replaces everything on the hard drive with exactly what it had on it when you bought it.  You access this before Windows starts using a special key that can varies by brand.  F11 is the most common
  2. “Upgrade” install using the Win7 install disk.  You start as if you are re-installing from scratch but when it gets to the point where it offers the Upgrade choice you choose that.
  3. Full Clean install.  The only absolutely sure way to have a perfectly clean system.  If you are not using a brand new disk, when it shows the screen to select where you want to put Windows, click on the Advanced link and delete all partitions.  Do not install any software until after ALL Windows Updates are installed and Device Manager shows all devices installed.

You will need a Win7 install disk for 2 or 3.  You will need a legal product code for 3 and the win7 disk must match the edition of the code and its bitness (32 or 64).  If you do not have the original Windows 7 install disk, borrow one from a friend.

cruis3 say:

Least Privilege, Windows 10, and Microsoft Accounts

We are supposed to run with all users as standard, non-admin accounts. However, the first account used to log onto the computer is by default the administrator account. This would seem to violate the policy of least privilege, especially if one considers that most people are really not that savvy.

I understand that UAC keeps accounts running as a standard user and prompts one if administrative privilege is required; but, if the only impediment to that is clicking "Continue", which most people are going to do without thinking, because if they do not, then the action they are attempting will not work, then the system is literally no better than just running as an administrator all the time.

Am I the only one bothered by this? At a minimum, a better prompt, indicating what app needs what permission, or what application is trying to install itself, as well as the requirement to re-enter a password or PIN might make people a bit more cautious.

The other issue is that disk encryption will not work if you do not use a Microsoft Account for that first account which is the default administrator account. So if you are trying to keep things safe with a separate local admin account, you lose disk encryption.

It seems that for all the talk about things being secure, nothing has changed, everyone is an admin, and machines will be pwned like always.

Regards,

   John

a. user say:

Every Windows installation needs an Administrator account in order to perform tasks that require Administrator permissions. For that reason, when you clean install Windows 10, the first account is automatically an Administrator account. All subsequent accounts are Standard users.

Many users are unaware that they should be using a Standard account for their day-to-day computer use. I agree that it would be a good idea for Microsoft to include a prompt for this when installing Windows.

To see what applications or tasks require Administrator permissions, look for the shield symbol on the icon.

Disk encryption works fine in a Local account.

a_user_here say:

Even though I see your point, I think that there is little chance that Microsoft would change this, because you are basically asking them to go back to what they had.  In past Windows versions they asked for the Administrator account password and then for the user accounts that the person wanted to setup.

So you have to ask why did they drop it?

And I think the most likely answer is that based on the feedback they got, people fall into two categories.

The ones that already know all of this and can set it up for themselves, and the ones that won't have a clue no matter how much text you put on the screen.  So instead of pushing away the novices, they just dropped the Administrator account in favor of the idea that UAC is "good enough".

Yeah the security model stinks, where you ask a novice if you should do something, but inconveniencing them by forcing them to log into another account, really doesn't add any more security.

BTW I really get your comments.  You see the shield on the operations you need permissions on, and prompts for things like installs or running a program in compatibility mode already tell what program is requesting the permission.
