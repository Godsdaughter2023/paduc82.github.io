alex wang msft say:
Hi Alan,

Lookup column is limited to specific match instead of matching a range.

If your 2 list is like this, maybe you could use a calculated column to store a week number in SharePoint list and match it as a workaround:

List 1: User Name | Reserve Date | Parking Code Returned | Week Number
List 2: Parking Code | Start Date | End Date | Week Number

When your users submit a reservation, the list 1 calculate the week number, then the lookup column will fetch the Parking Code base on the Week Number.

Here are two articles for your reference:
Create list relationships by using unique and lookup columns
Calculate data in lists or libraries

Alex

I notice that I missed one thing earlier: a column cannot be a calculated column and a lookup column at the same time.

That means we need another Week Number column in the first list which is set as the lookup column like this:
User Name | Reserve Date | Parking Code Returned | Week Number Calculated | WeekNumber input
