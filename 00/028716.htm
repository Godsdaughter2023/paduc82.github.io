scottgem say:

Also note that when you link the tables on your machine with a full license, you need to use UNC coding since the mappings may be different on the second machine. The format for UNC coding is:

\\servername\sharename\foldername\filename

As I understand you, the data is currently stored in a split backend on PC#1. So you need to make the folder on PC #1 a Shared folder giving it a share name. You might want to make the share one level up so you can have other folders to share with PC #2.

Then, using Linked Table Manager on the full license machine, check off the tables and check the always prompt box. when it prompts you to select the new Backend scroll down to Network, select the name of the PC #1, then selected the share name, the folder path and finally the name of the back end. 

Now you can copy the FE to PC #2 and run it using the runtime version.

If you need the in separate instances of Excel, you need to set Excel not to accept OPEN messages from Windows.

1.     In Excel, click on the Office Button

2.     choose Excel Options at the bottom of that window.

3.     Choose the Advanced button

4.     scroll down to the General section.

5.     In that section check the setting for "Ignore other applications that use DDE". It should be unchecked. This will cause Windows to open a new session of Excel, with your second workbook.
