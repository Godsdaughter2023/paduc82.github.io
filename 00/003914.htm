andreas killer say:

if B4's value was POTATO, then the hash would be 0718efaf8b438e839194f164a83581519f3de4022d355f124b15b5f1aede6b25854c498f55776eca930647f1a82707f4782f2aa6538e25e07a3b9b8a0a9f17d5
Option Explicit

Private Sub Example_SHA512()
  Debug.Print SHA512("POTATO")
End Sub

Function SHA512(ByVal S As String) As String
  'https://msdn.microsoft.com/en-us/library/system.security.cryptography.sha512managed.aspx
  Static UTF8 As Object, SHA As Object
  Dim Data, Temp, i As Long
  If SHA Is Nothing Then
    Set UTF8 = CreateObject("System.Text.UTF8Encoding")
    Set SHA = CreateObject("System.Security.Cryptography.SHA512Managed")
  End If
  Data = SHA.ComputeHash_2(UTF8.GetBytes_4(S))
  ReDim Temp(LBound(Data) To UBound(Data)) As String
  For i = LBound(Data) To UBound(Data)
    Temp(i) = Right$("0" & Hex(Data(i)), 2)
  Next
  SHA512 = Join(Temp, "")
End Function

    e.j. gun say:

    Since you do not have oVars explicitly declared (i.e. Dim oVars as Object), VBA automatically uses the Variant type, which means that oVars can be any type (integer, string, date, etc).

    The "Set oVars =" part tells VBA that you want oVars to be an Object type.  The type of the object is "Variables", which is a collection of all the variables defined in the active document.  The "Set" keyword is used to establish an object by "setting" (or pointing) it to some existing object.  In this case, oVars is set to represent the collection object ActiveDocument.Variables.

    The references to oVars("Something") are setting the various Variables defined in the document to the values entered into the user form.  You can also refer to elements within oVars as "oVars(1)", "oVars(2)", etc.  Using the label, such as "App_No", is more intuitive than using an index number.

    HTH,

    Eric

    P.S. - the ActiveDocument.Fields.Update line takes the information entered into the document's Variables (through oVars) and sets the value of any field in the document that is linked to one of the variables.
