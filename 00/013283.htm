hannoverfist say:

OK - this is a little complicated but I think I can get you going in the right direction.

 

I think you want to loop through your tblMandatory table and get the Program Key value to use in the other queries.

 

I'm a little fuzzy as to what you're doing after that - are queries 1 and 2 part of three? Then you use q3 to populate the NonAttendance table?

 

 

You'll need a VBA module to go through your table.

 

 

Public Sub PopulateNonAttendance


    On Error GoTo ErrorSub

    Dim strSQL As String
    Dim ProgramKey As Integer
    Dim rsTemp As Object

    Dim Counter As Integer

 


    Set rsTemp = CurrentDb.OpenRecordset("Select * FROM tblMandatory ", dbOpenDynaset, dbSeeChanges)

 

    rsTemp.MoveFirst

    DoCmd.SetWarnings False


    Do

        Counter = Counter + 1

        Call SysCmd(acSysCmdSetStatus, "Checking " & Counter)

 

        ProgramKey  = rsTemp("("Program Key") 'get the key from the tblMandatory

 

 

        strSQL = "SELECT NAMES.[LAST NAME], NAMES.[FIRST NAME], CEU.[UNIV ID], CEU.[PROGRAM KEY], CEU.[DATE VIEWED], CEU.[CEU TOTAL], CEU.GRADES INTO tblStaffAttend

FROM CEU INNER JOIN [NAMES] ON CEU.[UNIV ID] = NAMES.[UNIV ID]

WHERE (CEU.[PROGRAM KEY] =" & ProgramKey   & " )

ORDER BY NAMES.[LAST NAME], NAMES.[FIRST NAME];"

        CurrentDb.QueryDefs("QUERY1").SQL = strSQL   'Update query with Program Key

 

        strSQL = "SELECT " & ProgramKey   & " AS [PROGRAM KEY], [STAFF Q].[UNIV ID], [STAFF Q].[LAST NAME], [STAFF Q].[FIRST NAME], [STAFF Q].Assignment, [STAFF Q].SECTION, [STAFF Q].OWNERSHIP, [STAFF Q].[IMMEDIATE SUPERVISOR NAME]

FROM [STAFF Q] LEFT JOIN tblStaffAttend ON [STAFF Q].[UNIV ID] = tblStaffAttend.[UNIV ID]

GROUP BY [STAFF Q].[UNIV ID], [STAFF Q].[LAST NAME], [STAFF Q].[FIRST NAME], [STAFF Q].Assignment, [STAFF Q].SECTION, [STAFF Q].OWNERSHIP, [STAFF Q].[IMMEDIATE SUPERVISOR NAME], tblStaffAttend.[UNIV ID]

HAVING (((tblStaffAttend.[UNIV ID]) Is Null));

        CurrentDb.QueryDefs("QUERY2").SQL = strSQL   'Update query with Program Key

 

 

        DoCmd.OpenQuery "QUERY3" 'Run the append query

 

        rsTemp.MoveNext

 

    Loop Until rsTemp.EOF

    rsTemp.Close
   
    Set rsTemp = Nothing

 

ExitSub:

    Call SysCmd(acSysCmdClearStatus)

Exit Sub


ErrorSub:

    MsgBox "Error: " & Err.Number & vbNewLine & Err.Description, vbCritical + vbOKOnly, "Program Key Update Error"

Resume ExitSub


End Sub

 

 

This will go through your table one record at a time, update queries 1 and 2 to use the program key and run query 3. Then it starts over with the next till it goes through the table.

SELECT Fa, Fb, ..., tblMandatory.ProgramKey
