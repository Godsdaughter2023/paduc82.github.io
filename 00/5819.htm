webbrewers3 say:

I don't advocate syncing anything because the clients are known to be unreliable and the main reason we use a cloud storage solution is to get away from storing files locally. Being in the same network is not an issue because all the content is stored in the cloud.

Here's what we do:

User A and B both have shortcuts (aka network locations) set up on their pc's to a SP site that both can access.

User A uploads a new document to a library that both users have rights to (you can drag and drop directly to the shortcut if you prefer not to open Sharepoint)

User A drops User B a note about the new doc (email can be used but a chat app like communicator is preferred)

User B navigates to the library using windows explorer and opens/edits/views the new doc. 

It doesn't matter where the users are geographically or what network they are on - that's why SP is used!

It did take me a while to undo the bad habits of emailing attachments, saving files locally, overuse of syncing etc., - and the job is not finished - but those users who "get it" are much happier now they don't have to deal with Sharepoint's slow, unintuitive interface. hi ed,

i have gone through all the posts and done some related research. to my understanding, both statements in official articles should be correct. as you can see in this table below:



the purge action in in-place hold placed on an inactive mailbox merely aims at items in the recoverable items folder. it just keeps these items in this folder from being deleted for a specific duration or indefinitely. and if it’s just retained for a certain duration (not indefinitely), it’s calculated from the original date the mailbox item was received or created. some other proofs:



so in your example, as the retention policy is moving items in all folders to the archive mailbox after 1 year and the email was received on january 1, 2010, it would be moved to the archive mailbox on january 1,2011 if it’s not deleted within that year. then as no other deletion policies are set up, it will be retained in the archive mailbox until the mailbox is deleted (with in-place hold enabled) on january 1,2016. because this item is not in recoverable items folder when deleting, it will be kept until january 1,2018 and later. and about the second question, it’s the same one logically. the deleted data will be protected from being deleted in recoverable items folder for two years as it’s placed on a 2-year in-place hold. and then it will be purged when the hold duration expires.

macropod say:

Word's field numeric picture switches won't do as you ask, so you need a formula field for it. Such a field could be coded along the lines of:

{QUOTE{SET Val {MERGEFIELD Amount}}{=INT(Val/100000) \# "#',';;"}{=MOD(INT(Val/1000),100) \# "##',';;"}{=MOD(Val,1000) \# "###.00"}}

where 'Amount' is your field name.

Note: The field brace pairs (i.e. '{ }') for the above example are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac or, if you’re using a laptop, you might need to use Ctrl-Fn-F9); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues. The spaces represented in the field constructions are all required.
