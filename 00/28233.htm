Don't add the quotes yourself. This is not a defect; and it does not matter what version of Excel you have, AFAIK.

This is just an unfortunate consequence of the way that Excel writes to CSV files.  Poor design, IMHO; but intentional, so it is not a defect.

Excel writes the data as it appears, not the actual value.

Apparently, when you saved as a CSV file, the cell was formatted as General (or Scientific, albeit unlikely).

The General format always displays numbers with more than 11 digits in Scientific form, even if the column is wide enough to display the number in Number form.  The number of fraction digits depends on the column width.

(For numbers with 11 digits or less, the General format chooses the Number or Scientific form based on the column width.)

Consequently, yes, the value might be changed if it has more precision than is displayed.

The remedy is to ensure that the cells are formatted to display values with as much precision as possible.

In this example, you could get away with formatting the cell (or column) as Number.

But to ensure the most precision (15 significant digits) in general, you might format all numeric cells as Scientific with 14 decimal places.

Caveat:  Even then, some values might lose precision because Excel can represent more precision internally than it will display.  But that is the best we can do with the CSV format.

When Word starts a new blank document in Compatibility Mode, it means that your Normal.dotm template is from an older version of Word. The fix is to open the Normal.dotm file itself "as a document" and convert it to 2016. An alternative method is to rename the Normal.dotm template (so it's no longer recognized by the program) and restart Word, which will build a new Normal.dotm template from the internal defaults.
