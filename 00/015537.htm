ken sheridan say: A very simple way is to add an unbound text box to the form and set its ControlSource property to:

=Count(*)

This will return the number of rows in the form's current recordset, so when its filtered it will show the number of records returned by the filter.  If you don't want this to be permanently visible, you can set the control's Visible property to False, and pop up a message box after filtering the form, referencing the control in the message box. e.g.

Dim strMessage As String

' code to filter form goes here

Me.Recalc

If Me.txtHiddenCount > 1 Then
    strMessage = "There are " & Me.txtHiddenCount  & _
        " records with the same fragrance number. " & _
        "Use the navigation controls to view each record."

    MsgBox strMessage, vbInformation, "Search Result"
End If

doug robbins - mvp office apps & services (word) say:
Take a look at the GetCrossReferenceItems Method.  It returns and array of items of the ReferenceType specified that you could then use the Ubound() of the array to get the last item. 

sarahbbo say:

Thanks Doug!  That was exactly what I was looking for! 

Here's an example of the code that ended up working for me in case anyone has a similar question.  Since I have my Appendices set up as a Numbered List, I'm using wdRefTypeNumberedItem, but I assume others would need to use wdRefTypeHeading instead, depending on needs.

Sub InsertRefToLast()
'
'
'
'Insert preliminary text

Selection.TypeText Text:="Additional information can be found in "

'Quantify last numbered item, assign to a variable

Dim MNH As Integer

myNumberedItems = ActiveDocument.GetCrossReferenceItems(wdRefTypeNumberedItem)

MNH = UBound(myNumberedItems)
   
'Insert Numbered Item Cross Reference
 Selection.InsertCrossReference _
 ReferenceType:=wdRefTypeNumberedItem, _
 ReferenceKind:=wdNumberFullContext, _
 ReferenceItem:=MNH, _
 InsertAsHyperlink:=True

'space first
 Selection.TypeText (" ")
 
'Insert Numbered Item Cross Reference Text
 Selection.InsertCrossReference _
 ReferenceType:=wdRefTypeNumberedItem, _
 ReferenceKind:=wdContentText, _
 ReferenceItem:=MNH, _
 InsertAsHyperlink:=True

'Finish the sentence

Selection.TypeText Text:="."

End Sub

vijay a. verma say:

CTRL+A is designed to select the content of an object in Excel. Hence, if you press CTRL+A in a worksheet, it will select entire worksheet. In a Text Box, it will select entire Text box..

Cell is just an element of a larger object which is worksheet. Hence, it will not select entire cell.

If you go in a blank cell and in edit mode, if you press CTRL+A, it introduces = sign.

Now, suppose you have text in a cell and wants to select within cell-

1. If you are in the beginning of the cell, SHIFT+END.

2. If you are at the end of the cell, SHIFT+HOME

3. SHIFT+Right (Left Keys) will help you selective copying (I find selection through mouse easier though)

Whatever, I have written above for Cells equally applies FIND and Replace Box also.

Your observations regarding CTRL to select and not possible to deselect is correct. There is no shortcut to deselect, AFAIK.

ken sheridan say: One thing I'd add is that you should ensure the form's current record is saved before opening the report.  Otherwise any unsaved changes which might have been made to the record won't be reflected in the report, and if it's a new record nothing will be reported.  Doing it in the button's Click event procedure rather than via a macro the code would be like this:

    Dim strCriteria As String
    strCriteria = "[primary key field name] = " & Me.[primary key field name]

    ' ensure current record is saved
    Me.Dirty = False

    ' print report filtered to current record
    DoCmd.OpenReport "Price Tags", WhereCondition:=strCriteria

ken sheridan say:

access puts [] around the Me automatically


Where are you entering the code?

The Me keyword can only be used in VBA and refers to the current instance of the class in which the code is running.  In plain English think of it as referring to the form or report in whose module the code is located.  It should not automatically bracket the Me key word in this context.

If you are unfamiliar with entering code in event procedures, this is how it's done in form design view:

1.  Select the button and open its properties sheet if it's not already open.

2.  Select the On Click event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new lines between these.
