Although you're undoubtedly doing everything right, here are a couple of cautions regarding mistakes that are commonly made:

  1. You mention that you are checking the box for "Repeat as header row at the top of each page" on the Row tab of Table Properties. Make sure that, when you do this, you have only the top row(s) selected. Also make sure that it is actually the top row (if there's a blank row above it, this won't work).
  2. There are two controls in the contextual Table Tools that relate to header rows. The Header Row checkbox in the Table Style Options group on the Design tab just determines whether special formatting is applied to rows designated as header rows. The control that actually designates a row as a header row is the Repeat Header Rows button in the Data group on the Layout tab. Again, make sure you have only one row (or contiguous rows) at the top of the table selected when you click this button.

vinay, india say:
A normal VLOOKUP only retrieves the value of a cell. It does not retrieve the Hyperlink of the cell. 

To lookup and reestablish the hyperlink, you may use a VBA function and INDEX and match functions.

OPEN VBA using ALT+F11, Right click on the VBAProject in the Project Explorer and INSERT a new module. Paste the following function in the code area on the right side.

Function getlink(HyperlinkCell As Range)
    If HyperlinkCell.Hyperlinks.Count > 0 Then
        getlink = HyperlinkCell.Hyperlinks(1).Address
   Else
        getlink = ""
   End If
End Function

Now on the sheet try using the following formula instead of a normal VLOOKUP.

Suppose the data is on Sheet 1, cells A1:B3 as follows
G          Google
Y           Yahoo
D          D_Drive

The second column values are respectively hyperlinked to:

http://www.google.com
http://www.yahoo.com 
D:\ 

Enter your search criteria in A6 and result in B6 as follows:

A6: G
B6: =HYPERLINK(getlink(INDEX($B$1:$B$3,MATCH(A6,$A$1:$A$3,0))),VLOOKUP(A6,$A$1:$B$3,2,0))


Try changing A6 to D or Y and see the hyperlink changing.

It's true that Mac Word does not provide style separators. It's a bit of a chore, but it can be simulated by making the caption into 2 separate paragraphs. For example:

Perhaps the caption reads:

Figure 1- Teamwork: Sharing knowledge with others is a beautiful thing!

but you want to include only

Figure 1-Teamwork

  1. Display the nonprinting characters
  2. Place the insertion point between the k & the colon
  3. Press return
  4. Select only the ¶ that now follows the k
  5. Go to Format> Font, check Hidden

The caption will display as 2 separate paragraphs while the nonprinting characters are displayed but will appear as a single paragraph when they're turned off. The 2nd paragraph will not be included when you generate the Table of Figures.

How to create a combo box/list box that allows a multi-selection - Microsoft Community

runlolarun75 say:

How to create a combo box/list box that allows a multi-selection user response

I am creating a form for a medical group. There are questions on the form that will have more than one possible answer and check boxes will not suffice as the potential responses number more than 750. I want the users to be able to select from one drop down list/combo box more than one response. (I would user either a list box or a combo box, whichever serves the need better, I do want the potential of the users' replies to be added to the existing range of responses)

An example would be,  the user needs to select from a list of diagnoses to rule out and the response is both pneumonia and lung ca..

Many thanks for all help.

jay freedman say:
Of the list box or combo box controls that you can put into the body of a document (including legacy form fields, content controls, or ActiveX controls), only the ActiveX list box is capable of multiple selection. While it might be possible to use this, ActiveX controls in general aren't well suited for Word (for discussion, see http://msdn2.microsoft.com/en-us/library/aa140269(office.10).aspx).

The preferred way is to create a userform (http://gregmaxey.mvps.org/Create_and_employ_a_UserForm.htm) and place a list box on the userform. That control has a MultiSelect property that defaults to fmMultiSelectSingle but can be set to fmMultiSelectMulti or fmMultiSelectExtended. (Read the help topic on MultiSelect to understand the difference.)

In the Click procedure for the userform's OK button, you'll need to write code to test the Selected property of the list box for the index of each item in the list to determine which ones were selected, and use that information to write text into the body of the document.

The big advantage of the userform over ActiveX controls is that you can have a button (usually a MacroButton field) that calls the same userform for each question. With ActiveX controls you'd have to put a separate copy of the list box into each question.

If you want to make it possible to reopen an answered form and change the selections, you'll need a mechanism to save the lists of selections in the document file. Each time the userform is called, it should check the list that was saved for that question (if any) and initialize the list box with those selections. Document variables are a good candidate for saving the data. (The same sort of mechanism can be used with ActiveX controls, if you decide to go that way.)
