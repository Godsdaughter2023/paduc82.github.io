a. Open Control panel

Let's look at that because it could just be a problem with DNS.

You can both look at and change your DNS using  netsh

netsh  in  ip  sh  dn

(sh for show and se for set).

So, just as important as testing a browser you can test your DNS, e.g. using nslookup or ping or tracert.  Also notice that nslookup allows us to test an alternate DNS, bypassing the configured one.   E.g. this would be a test for seeing a lookup for this site using a Google public DNS server

nslookup  answers.microsoft.com  8.8.8.8

Then if that seemed better you may decide to add it as an override to the above list (using the set dns command).  Or you could also do it with a GUI, e.g. Properties, TCP IPv4 etc.

Through Firmware

  1. Plug the computer directly into the devices LAN port via an Ethernet cable.
  2. Open a web browser.
  3. Log into the router by entering the default gateway (192.168.1.1) into the address bar of the internet browser.

  6. Select "Settings" from the sub menu, then "Restore Default" below that.
  7. Select "Restore Default Settings"

 

Through Hardware

  1. Plug the computer directly into the devices LAN port via an Ethernet cable.
  2. Locate the Reset button on the device.
  3. Using a toothpick/pen/paper clip, press and hold the button for 10 seconds.
