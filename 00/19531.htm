Sub FileDialog_SaveAs_XLSX()
'Nov 07, 2016
Const sName$ = "Sheet1" '<< source sheet

Dim wb1 As Workbook, wb2 As Workbook
Set wb1 = ThisWorkbook
Dim r As Long, c As Long
r = wb1.Sheets(sName).UsedRange.Rows.Count
c = wb1.Sheets(sName).UsedRange.Columns.Count
Dim sPath As String
Application.ScreenUpdating = False
Set wb2 = Workbooks.Add
wb1.Sheets(sName).Range("A1").Resize(r, c).Copy
wb2.Sheets(1).Range("A1").PasteSpecial xlPasteValues
wb2.Sheets(1).Range("A1").PasteSpecial xlPasteFormats
Application.CutCopyMode = False
Application.ScreenUpdating = True
With Application.FileDialog(msoFileDialogSaveAs)
If .Show = True Then
sPath = .SelectedItems(1)
Application.DisplayAlerts = False
wb2.SaveAs Filename:=sPath, FileFormat:=xlOpenXMLWorkbook
Application.DisplayAlerts = True
Else
MsgBox "cancel"
End If
End With
wb2.Close False
End Sub

candacemacfarlane say:

How to re-open a Userform (created in VB) in Word

I have created a userform title "frmDetails" in VB, containing text boxes (for data input) and commands (to update fields within the document).

 

I have input codes so that when I create a new document from a template the the userform automatically appears. I then place the information into the userform and once I hit OK it automatically updates the Field codes ( {DOCPROPERTY "RevisionDate" \* MERGEFORMAT}) I have in the actual document.

Can anyone advise me on how I can than reopen this Userform in the word document so I can modify the information in the future.

 

Thanks

 

hansv mvp say:

Use the UserForm_Initialize event to populate the text boxes with the values of the document properties if they exist:

 

Private Sub UserForm_Initialize()
    ' Load properties into text boxes
    On Error Resume Next
    Me.txtRevisionDate = ActiveDocument.CustomDocumentProperties("RevisionDate")
    Me.txtInfo = ActiveDocument.CustomDocumentProperties("Info")
    ...
End Sub

 

Insert a standard code module (Insert | Module in the Visual Basic Editor) and create a macro that opens the userform:

 

Sub StartForm()
    frmDetails.Show
End Sub

 

You can assign this macro to a keyboard shortcut, for example.
