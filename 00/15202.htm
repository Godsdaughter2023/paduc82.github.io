Your post and the error message suggest to me that you have both a Module and a Sub with the same name like the following screen snippet.

If you have a module and sub both by the same name then to call the sub both the Module and the Sub need to be included in the call like the following.

Call MyModule.MyModule

However, if the calling sub and the called sub are both in the same module (ie. MyModule) then the module name does not need to be included.

Personally I avoid having Subs with the same as Modules.
