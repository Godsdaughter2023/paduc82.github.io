robert aldwinckle on forums say:

All desktop applications (not talking about modern or metro) start full screen.

How big are you trying to make them?  I used to have this problem on W7 before I got a widescreen monitor.  Windows such as WLMail and IE that I did not want to be either maximized or maximally sized would be both if I tried to make them too big.  A workaround for IE is use its Fullscreen mode (F11) with a normal mode window which is much smaller.  You can also turn off Fullscreen's default of Auto-hide the toolbars and have a usable window that way.  Then if you need to work with other windows you can just press F11 to toggle back to its normal mode size.  BTW the Fullscreen mode is available to other Desktop windows too, in particular File Explorer.  So again, you wouldn't have to try to make a window so big that it would be usable that it would become maximized by whatever quirk it is that we have seen happening, just use the F11 Fullscreen toggle as needed.

A related issue is keeping your Title bar away from the top of the screen otherwise there is a tendency for the mouse pointer to slip up beyond (perhaps with a drag but I'm not convinced that that is a requirement for this quirk) and be interpreted as a Snap request.  Disabling Snap (especially the one at the top, if it is possible to differentiate) would be a good tack for you to try I imagine but I don't know if that is possible.  The best workaround I suspect is just being aware of it and taking care to avoid its effects.
