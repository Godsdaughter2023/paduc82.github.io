  2. Choose an option
  3. Troubleshoot
  4. Refresh your PC
  5. Click 'Next' button

  13. Choose an operating system (Windows 8 or Windows 7)
  14. Click 'Windows 8'
  15. Preparing (the OS) goes on to finish and you can then login with the PC refresh successfully completed

The Underlying Issue:

The underlying issue appears to be on my 8 month old hard ware from Hewlett Packard the firmware version 7.13 does not natively support UEFI-mode (Unified Extensible Firmware Interface) and secure boot.  When I setup the dual hard drive, dual boot system, Windows 8 install did not automatically pick up on the dual boot arrangement causing me to need to modify the MBR (Master Boot Record) via the 'EasyBCD' utility to get the dual-boot option.

 

Using the EasyBCD utility saved the dual-boot record in 'Windows Legacy Mode' where it gives you a text choice on a black screen to pick your OS, not the new blue background, Windows 8 UEFI graphical boot choice screens.

 

Windows Legacy Mode seems to cause problems in a variety of places inside of Windows 8 OS. 

In Modern Mode:

  • Settings > Change PC Settings > General > Refresh Your PC Without Affecting Your Files (does not work)
  • Settings > Change PC Settings > General > Advanced startup (various options do not work)

For example: 

Advanced startup > Troubleshoot > Advanced Option > Command Prompt > (command prompt will not appear)

Giving the error message "There was a problem. Restart your PC to try again. It looks like something didn't load correctly. Restarting might fix the problem. If this happens more than once, your might also be able to find help by searching online for the specific error code. Error code: 80070490"

 

Only by using the "Repair Disc Windows 8 64-bit" can you do these options.

 

Additional Troubleshooting Attempted - Modifying the MBR (Master Boot Record):

During the trouble shooting I attempted to modify the MBR in three ways to fix the problem but without success:

  1. bootrec /fixMBR
  2. bootrec /fixBoot
  3. bootrec /rebuildBCD

Reference Microsoft's knowledge base article:  http://support.microsoft.com/kb/927392 

 

Rebooted between each variation and attempted the PC refresh all without success.  These three mods still left the boot record in 'Windows Legacy Mode' so the problem unresolved.

 

Only by refreshing the PC from the "Repair Disk Windows 8 64-bit" DVD was the UEFI-mode boot correctly established and picked up the dual disk, dual boot arrangement.  After that then all in built Window 8 functions worked correctly.

 

Hope this information helps others in this situation.  You appear to need the UEFI-mode for Windows 8 to offer all functionality and Windows Legacy Mode boot record mods break things.

zeuscalypso say:

Continued to have issues so dug deeper into this problem.

 

Correction:

The Hewlett Packard BIOS v.7.13 on confirmation with HP does actually does support both UEFI-mode and legacy BIOS boot modes, with both modes appearing automatically active without BIOS setting changes being needed.

 

BCD (Boot Configuration Data Store):

The underlying issue actually turns out to be a problem with the BCD (Boot Configuration Data Store) after the third party EasyBCD v2.2 utility modifies the boot record.  Once touched & saved in any way by EasyBCD utility, the boot record will only support 'Windows Legacy Mode' and not the new Windows 8 UEFI graphical boot choice.  To date I haven't figured out what the incompatible data change is being made.

 

BootMenuPolicy Setting:

The switch / parameter setting in the BCD store, that typically controls which mode is booted into Standard (Win8 Graphical) or Legacy (Windows Legacy Mode) DOES work if only modified using the built in operating system BCDEdit (Win7 or Win8 OS) from a command prompt launched with administrative privileges or powershell.

Technet BCDEdit reference:  http://technet.microsoft.com/en-us/library/cc709667(v=ws.10).aspx

Walk Through:  http://www.eightforums.com/tutorials/5245-startup-options-enable-disable-windows-8-a.html

 

By using the command:

bcdedit /set {default} bootmenupolicy Standard

Where 'default' defines which of the dual-boot OS is being referenced (Win 7/8) inside of the BCD record.  You tell the BIOS to use the Windows 8 UEFI graphical boot choice

 

bcdedit /set {current} bootmenupolicy Legacy

Where 'current' defines which of the dual-boot OS is being referenced (Win 7/8) inside of the BCD record. You tell the BIOS to use the Legacy Windows Boot choice.

 

Either {identifier number} will also likely be a hexidecimal value needing to be used,  depending on your system's BCD record so you need to check your system by first listing the BCD record.

 

Problem solved:

Avoid using EasyBCD v2.2, hopefully they will update the utility to work correctly with Windows 8 BCD store.  Just use the built in BCDEdit feature of either Windows 7 or 8 to switch between graphical or legacy boot modes.  I caused my own problem by using a third party utility.

  3. On the run window, type in %appdata%\Skype and hit enter. Look for a file named shared or shared.xml and delete it.

  4. Open the folder that has your Skype name.

  5. Locate and delete the config or config.xml

  6. Open Control Panel and look for Internet Options.

  7. Go to the advanced tab and select Reset.

  8. Put a check mark under "Delete Personal Settings", then hit okay and enter.

  9. Close it when prompted.

  10. Restart Skype.

I hope this helps. Should you have further queries, please do not hesitate to reply to this post.
