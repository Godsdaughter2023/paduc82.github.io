Hi Alicia, I had the exact same issue with my HP laptop, Panda Global Security caused my problem even though it is their Windows 10 compatible version, try doing this it fixed it for me.

1) Reboot your machine, you can do this by holding its power button down until its light goes off.

2) Now power it up and at the same time hold down your ESC key.

3) From the menu select F11

4) Click the Troubleshoot icon, next click the Advanced options icon.

5) Now click on System Restore.

6) When prompted always click on OK or Continue, pick a restore point prior to the Windows 10 Cumulative Update.

7) When the process has finished, your machine should restart and boot into life.

8) This is the most important bit, if you have any third party Anti Spam software like Panda or Norton, uninstall it.

9) When uninstalled, reboot your machine, at this point you may consider creating a new System Restore point.

10) Now go to Settings, Update & Security and check for Updates.

11) Cumulative Update for Windows 10 (KB3081424) should now install.

12) At the prompt click on Restart now, your machine should now boot up OK.

hi mika,

if you aren’t a site administrator, you will not have the full control permission in the site collection. if you haven’t the full control permission, the move button may disappear in content and structure. please make sure if you have the full control permission in the site, check your site permission and provide us with a screenshot showing your site permission for our analyzing.
note: the content and structure feature can only move files between libraries in one site collection.

alternatively, if you want to move all files in one library to another library across different site collections, you may try running the script below.
https://gallery.technet.microsoft.com/powershell-bulk-upload-b9e9d600
disclaimer: microsoft provides no assurances and/or warranties, implied or otherwise, and is not responsible for the download you receive from the third-party linked sites or any support related to the download or the downloaded technology. if you need support relating to the third party technology, please contact the manufacturer directly. 

here are the steps:
1. download and install sharepoint online client components sdk from https://www.microsoft.com/en-us/download/details.aspx?id=42038.
2. download the script above from https://gallery.technet.microsoft.com/powershell-bulk-upload-b9e9d600.
3. map a sharepoint site library which is the destination library as a network drive via the steps in https://support.microsoft.com/en-us/kb/2616712. in my environment, i mapped it as y:\.
4. unzip the script file. in my environment, it is in c:\ bulkuploadsharepointcsom.
5. open powershell with administrative rights.
6. change the path. in my environment, i change it as cd c:\ bulkuploadsharepointcsom.
7. run the command.\bulkuploadsharepointcsom.ps1  -username "*** Email address is removed for privacy ***" -password "xxx" -siteurl "https://contoso.sharepoint.com/" -doclibname "xxx" -folder "y:\" -o365 $true.
note: https://cotoso.sharepoint.com/ is the site where the library you want to move the files points and xxx is the name of the library where you want to move the files. y:\ is the destination library in my environment.

 

you can follow the steps to have a shot and feel free to post back if you need further help.

thanks,
cliff

jeeped say:
MID() will always return a text representation of a number, not an actual number. It is like the difference between ="2" and =2. The two are not the same. If you want to compare them, change them to be both numbers or both text, not a mix.

=VALUE(MID(G1,1,1)
... or,
=--MID(G1,1,1)
... or,
=MID(G1,1,1)*1

... will be the number 2, not text that looks like 2. For best results, wrap it in an IFERROR() for cases where the text cannot be converted to a number.

=IFERROR(VALUE(MID(G1,1,1), "<not a number>")

Addendum: btw, text defaults to a left alignment and true numbers to a right alignment. If you have not changed the default cell alignment, then this is a good indication that the TYPE() is different. You can also use the TYPE() function to return which is which.

chip pearson say: You need to force Excel to convert the text character "2" to the actual number 2. Your MID function returns a text character "2" rather than a number 2. Try something like

=IF(--G2=A5,"yes","no")

The double negation (--) before G2 forces Excel to convert the character "2" (in G2) to a numeric value 2, and then multiplies the numeric value 2 by -1 twice, returning a numeric value 2.

The problem is that you are comparing the actual number value 2 with a text string containing the text character "2".  Behind the scenes, Excel is comparing a numeric value of 2 with the the numeric value 50, which is the ASCII code number use to represent the text character "2". When Excel performs arithmetic on text representations of numeric values, it normally converts the text to its numeric value before calculation. For example, ="2"+1 will cause Excel to convert the text character "2" to a numeric value of 2, and return a numeric result of 3. The formula =2+1 seems at first glance to do the same thing, but in this formula, the 2 is not enclosed in quotes so Excel treats it as a numeric value, not a text character, and no conversion from text to number is required.

If you attempt to perform arithmetic on a text character that is not numeric, you'll get a #VALUE error. For example,  ="A"+1 . Here, Excel cannot convert "A" to a number and returns a #VALUE error.

When doing a simple comparison operation, Excel does not convert a numeric text character to its numeric value. For example, =IF("2"=2,"yes","no") will return "no" because Excel is not converting the text character "2" to the numeric value 2. The formula ="2"=CHAR(50) will return TRUE because the CHAR function takes the ASCII code number (50 in this example) of a character and returns the character represented by that number. Since the text character "2" is represented internally by the number 50, the formula returns TRUE since "2"="2".  The formula ="2"=2 returns FALSE because the text character "2" is not the same as the number 2.  The formula ="A"=CHAR(65) return TRUE because internally, the upper case character "A" is represented by the number 65, and the formula is then comparing "A" to "A", which are equal.

You can use double negation to force Excel to convert the text character to a real number. The double negation does two things. First, it causes Excel to convert the text character "2" to a numeric 2 and then multiplies this by -1, giving a numeric value of -2. The second negation multiplies the number -2 by -1 again, which returns the positive number 2. 

The differences between an actual number and a textual representation of a number can be confusing at first because on the screen a text character "2" appears exactly the same as a numeric 2.  If you don't realize the difference, you will get confusing results, as you have found.
