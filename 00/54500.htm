bossdweeb say:

The three main issues with old games are Direct Play, a 'security feature' MS built into Win10 - DRM Blocking,

and Screen Scaling.

DRM Blocking affects many older disc based games. I have more details in my post at  --- this thread ---

- To circumvent DRM Blocking, you will need to use a noCD fix/hack with any disc based game that uses SafeDisk

copy protection, and both of these games use SafeDisk.

Latest Game Fixes | MegaGames

- Verify Direct Play is enable by going to....

Programs and Features > Turn Windows Features On or Off

Scroll to an tic the box next to Legacy Components and

- I don't know if it's still necessary if you have updated to 1803, but you used to have to

Sign Out and back in again to save the setting. I have not needed to sign out since updating to 1803 though.

Note that when Direct Play is disabled Windows will often pop up a Window that offers to 'install' Direct Play

with many games that require it, but I believe using this option has caused problems for many users saving the setting.

I would be inclined to close this pop-up, should you see one, and enable Direct Play manually as the safer option.

-  Screen Scaling is often fixed automatically by Windows, but interestingly Windows often fails

to fix screen scaling on games that actually need it.

I don't have these games, but if they don't fit the screen correctly (usually low and to the right) your will need to...

- R/click on the games shortcut or  .exe file (the file you launch the game with) in the folder the game is installed in.

- Click on Properties and in the Properties Window click on the Compatibility tab.

- Click on the Change High DPI Settings button, and in the next Windows tic the box next to

'Override High DPI Scaling Behaviour'

- Occasionally using XP sp2/3, Vista sp1, and more rarely Win95/98  Compatibility will help with old games, but using

Compatibility with an older version of Windows is not often effective.

However, it seems that some games are coded to look for certain versions of Windows, so they react poorly when

confronting a version of Windows they aren't programmed to recognise, so they generally respond well to Compatibility with

an older version of Windows.

One good example is GTA4, which even on Win7 required using Compatibility for Vista sp1 or XP sp3.

For the most part, any Windows Compatibility that was required on Win7 will work on Win10 too, but I

have two games that required Win98 on Win10, but only prior to Win10 v1703 for some reason.

i. Uncheck the Read-only property

Method 3: To Manually Rebuild the Icon Cache using the Command Prompt.

 

a. Close and save anything that you are working on. This command will kill explorer and restart the computer when completed.

b. Open the Start Menu, and type cmd in the search box and press enter.

c. In the command prompt, Copy and Paste each command line below and press enter after each command.

taskkill /IM explorer.exe /F

CD /d %userprofile%\AppData\Local

DEL IconCache.db /a

shutdown /r

d. The IconCache.db file has now been rebuilt.
