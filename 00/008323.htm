You are right Times New Roman is not included with Office 2013, but it IS included with Windows 7 and 8 and just about every other version of Windows since time immemorial, so no-one, least of all Microsoft has got rid of it.

If you have the font installed on your system and the currently active printer driver is capable of printing it, then it will be displayed in the Word font list. If you don't have a printer driver installed, it won't and you will see only a very limited set of fonts.

Microsoft made a design choice to switch to a new family of fonts back in 2007 and Calibri is the default. However it is a simple matter to change the default font to whatever you like, including Times New Roman. That default is stored in the normal template, which for Word 2013 is normal.dotm.

When you paste text into an existing document, and use the unformatted option as Suzanne suggested, then the inserted text adopts the style at the cursor, whatever that is. That's why it is called 'unformatted'.

john - project say:

NaeemAslam1,

Sorry but there is no automated way to have Project multiply assigned resources whether work type or material type. You could however, create a fairly simply macro to take an input from the user and make the appropriate changes.

However, if you have a template for a single home and then need to re-create that for "x" number of homes, are you going to create "x" separate project files, each for a given home or do you plan to replicate all the tasks under separate summary lines where each summary line represents one home? Either way there won't be any need to multiply resources. It doesn't make any sense to take each construction task in the template and multiply it directly by "x" since each home will in fact have it's own schedule. For example, you're probably not going to pour "x" foundations all in a continuous sequence unless you just happen to have "x" dedicated crews. The same with the other trades.

John
