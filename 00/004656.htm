It looks like your code was adapted from an earlier version of Excel; if you were to use that approach when there is more than 65,536 rows of data you may get unexpected results. Below is a standalone function you can use to identify the last used row. Note that your results may vary if you delete rows, until you save the file (manually or via VBA) to reset the last used row stored by Excel.

Usage (aircode, apologies for any mistakes):

   BlankRow= LastRow(Sheet8)+2

   Sheet1.Rows("13:1500").Copy destination:= Sheet8.Range("A" & BlankRow)

(My examples use Sheet1 and Sheet8 without quotes because I always refer to sheet codenames, but if that is the worksheetname in the UI, it may be different from the codename, so use what you feel comfortable with)

Function LastRow(sh As Worksheet)    'Finds last used row.
   On Error Resume Next
   LastRow = sh.Cells.Find(What:="*", After:=sh.Range("A1"), _
   Lookat:=xlPart, LookIn:=xlFormulas, SearchOrder:=xlByRows, _
   SearchDirection:=xlPrevious, MatchCase:=False).Row
    On Error GoTo 0
End Function

This is a nasty bug because when you invoke the ImportFromFile method, it will delete all animations associated with that shape on the slide and then add an appear animation.

Possible workarounds:

A) Re-assign the animation after invoking the method. This does not solve the problem of the animations being deleted though.

B) Insert the audio as a shape on the slide and assign a withprevious animation on the audio file to play in combination with the original animation.

C)Insert a dummy shape and assign the animation to that shape including the associated audio track and then change the shape reference for that animation to the original shape. Delete the dummy shape.
