cristin yan say:

Hi,

 

Slide.NotesPage Property Returns a SlideRange object that represents the notes pages for the specified slide or range of slides (Read-only).

 

Note

The following properties and methods will fail if applied to a SlideRange object that represents a notes page: Copy method, Cut method, Delete method, Duplicate method, HeadersFooters property, Hyperlinks property, Layout property, PrintSteps property, SlideShowTransition property.

 

And SlideRange object in PowerPoint is a collection that represents a notes page or a slide range, which is a set of slides that can contain as little as a single slide or as much as all the slides in a presentation.

 

That is to say, SlideRange object contains Slide(s) object, so when you use:

ActivePresentation.Slides(i).NotesPage.HeadersFooters.Header.Visible = False

 

You will get the error message (Invalid request. This action is not supported for notes pages.)

 

If you want to remove slide numbers form all slides in PPT, please use the following code:

Sub RemoveHeadersFooters()
    Dim sld As Slide
   
    For Each sld In ActivePresentation.Slides
        sld.HeadersFooters.Clear
    Next
End Sub

 

PS:

The macro RemoveHeadersFooters will remove all headers and footers including slide numbers after running.

Changing an existing .ost file to .pst - Microsoft Community

mike_canada say:

Changing an existing .ost file to .pst

I want to move by e-mial account which is using Exchange now (.ost data files) to be a non-exchange e-mial account i.e to use .pst file, can anyone help please
brian tillman [outlook mvp 2007-2019] say:

Mail profiles care about data files.  Mail accounts less so, as long as its in the profile.  By definition, if it's in the profile, it will appear in the Navigation Pane, unless it's an archive data file that you've told Outlok to hide.

If you create the new data file and copy the data from the Exchange account to it, it will be accessible from within Outlook.  You will already have opened it when you created it and you won't need to add it to the profile again.  You can then access the messages you copied after you delete the Exchange account.  If you don't copy before deleting the Exchange account, you'll lose access to the OST and the data it contains.

You can specify the data file you add as the default data file and then when yo add the non-Exchange account, you can tell Outlook to allow you to configure the account manually, which will allow you to tell the account to use the data file you created as its delivery location.  Alternatively, you can allow Outlook create a second new data file for the default delivery location and keep the one containing the Exchange data as a secondary data file.

Keep in mind that the messages you've copied to the data file will still be tagged as having arrived via the Exchange account which you will no longer have and you will probably need to manually choose the sending account if you reply to any of the messages, since Outlook always replies or forwards using the account that received the original message.

It's quite common for people to delete an Exchange account because they no longer have access to the server and then discover they've lost all the mail they had.

brian tillman [outlook mvp 2007-2019] say:

To move the data, simply right-click each folder, drag the folder to the root of the new data file as it appears in the Navigation Pane, release the mouse button, and choose Copy.

Yes, the subfolders will be intact because you'll be copying all of the top level folders.

You could, as you say, export the Exchange folders to a PST as well, but I prefer copying.

neha singh msft say:

Hi Mark,

 

PowerPoint will show you measurements in either inches or cm, depending on your regional settings. To change the ruler to centimeter, follow the steps below:

 

  1. Quit PowerPoint
  2. Go to System Preferences in your Mac and choose Language & Region preferences.
  3. Click Advanced button and then the General Tab.
  4. Click the Measurements unit and change it from US to Metric.

 

Now, open PowerPoint and the unit of measurement should be changed.

Neha
