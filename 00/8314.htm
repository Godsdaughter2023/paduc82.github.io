Finding Potential Duplicates - Microsoft Community

self untaught say:

Finding Potential Duplicates

Trying to find potential duplicate rolodex cards in my table Rolodex.  Finding exact matches is easy. Trying to find records where the first 3 characters each of company, LName and FName are a match.

 

1. Made query "Rolodex 1st 3 Characters" :

 

SELECT (Left([co],3)) AS [Company], (Left([FirstName],3)) AS FName, (Left([LastName],3)) AS LName, Rolodex.*
FROM Rolodex;

 

2. Used query wizard to find duplicates

    a. source: query "Rolodex 1st 3 Characters"

    b. field selection: company, LName, Fname

 

SELECT [Rolodex 1st 3 Characters].[Company], [Rolodex 1st 3 Characters].[FName], [Rolodex 1st 3 Characters].[LName], [Rolodex 1st 3 Characters].[id], [Rolodex 1st 3 Characters].[Co], [Rolodex 1st 3 Characters].[FirstName], [Rolodex 1st 3 Characters].[LastName]

FROM [Rolodex 1st 3 Characters]

WHERE ((([Rolodex 1st 3 Characters].[Company]) In (SELECT [Company] FROM [Rolodex 1st 3 Characters] As Tmp GROUP BY [Company],[FName],[LName] HAVING Count(*)>1  And [FName] = [Rolodex 1st 3 Characters].[FName] And [LName] = [Rolodex 1st 3 Characters].[LName])))

ORDER BY [Rolodex 1st 3 Characters].[Company], [Rolodex 1st 3 Characters].[FName], [Rolodex 1st 3 Characters].[LName];

 

When I try to run it, I get error:

 

You tried to execute a query that does not include the specific expression 'Count(*)>1 And [FName]=[Rolodex 1st 3 characters].[FName] And [LName]=[Rolodex 1st 3 characters].[LName]  And [Company]=' as part of an aggragate function.

 

ken sheridan say:

What is the "AS R1" and "AS R2" all about?

They are aliases for the two instances of the Rolodex table.  This is what enables the subquery to be correlated with the outer query.

ken sheridan say: You should be able to do this with a single query.  Try this:

SELECT *
FROM Rolodex AS R1
WHERE EXISTS
    (SELECT LEFT(R2.Company,3)
     FROM Rolodex AS R2
     WHERE LEFT(R2.LastName,3) = LEFT(R1.LastName,3)
     AND  LEFT(R2.FirstName,3) = LEFT(R1.FirstName,3)
     AND  LEFT(R2.Company,3) = LEFT(R1.Company,3)
     GROUP BY LEFT(R2.Company,3)
     HAVING COUNT(*) > 1);

The subquery can return any of the columns; it's the criteria by which the subquery is correlated with the outer query which matter.

gibbsyns3 say:

Thanks Rohn007, I really appreciate the time you have taken, I had searched before I posted and couldn't find anything. 

Oddly it seems that I am doing better than the mac user and onedrive for business user, as I am having no difficulty mailmerging using onedrive. My problem is limited to the need to reattach the data source each time of using, not inability to use the data source as seems to be the case in the other threads.  

I think I have discovered a workaround - so if anybody else comes across this thread and their need for mailmerge using onedrive is worth these extra steps, here goes. 

This will only work for setups with one user per one workstation, you won't be able to use it for multiple users on each workstation.

1. On initially installing onedrive it will select the location as c:\users\your-username\onedrive - but you are given the option to choose an alternative location. Select a location for onedrive that allows you to mirror the folder path on other workstations.  An easy option that you already have access to providing you are aware and happy with the security implications is to use c:\users\public\onedrive 

2. Set onedrive with the sync all files option, not download on demand. 

3. Set up your mailmerge templates connecting to the local onedrive data - in my example c:\users\public\onedrive\data.xlsx

4. Now providing onedrive is installed  by other users on their workstations in the same way, you can share your data files and word merge templates with them, and when they run them the mailmerge template will be able to locate the data source as it is connecting to it using the same path. 

It is not an elegant solution - but I have it working.
