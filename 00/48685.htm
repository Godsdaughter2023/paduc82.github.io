What is the D partition on Disk1?  Unless it is an extension of C (doubtful since that would require Dynamic rather than Primary drives) then it should not have an Active flag which is only meant to point to the System partition that is booting Windows, or other bootable media which N appears to be (if so unplug it).  Follow the same steps here:  Partition - Mark as Inactive

You can also Extend D in Disk Management to pick up the unused 45gb to its right.  Likewise all of the other wasted Unallocated space which can be added to preceding partition or create a new partition there. 

Unfortunately the RAW partitions are likely ruined unless they have some sort of encryption that gives the RAW label.  There are some resources for trying to repair RAW drives here:

How to Recover a Drive with a RAW File System

Raw Partition Recovery | How to Recover Raw Partition - EaseUSChange file type RAW to NTFS - Storage - Tom's Hardware

Recover a drive turned RAW – Partition Wizard and TestDisk

Because of the stray Active partition and the RAW disks it's possible one or more of these could cause the lag.  The way to determine this with certainty would be to power down and unplug all of the drives except Windows 10, including any racks or caddies. You will have to move the Page File back to C first:  How to Move the Windows Page File   Make sure then that the Win10 drive is plugged into the fastest optimal port.  Try powering up to see if the problem eases.  If so I'd add back each corrected drive to see which is the culprit and then run it's maker's HD Diagnostic followed by a full Disk Check to know it's condition for certain.  If nothing else works then move the data off and wipe the drive with Diskpart Clean Command to repartition it in Disk Management.

Having to mark a Bitlocker key flash stick Active to have to recognized is suspect, unless BitLocker says otherwise somewhere.  Only a System partition booting Windows or bootable media should be marked Active.  It causes the BIOS to pause to query the drive with sometimes unpredictable results. While this is normally at boot any anomaly on a drive can also affect performance in general. 

A way to test the browser theory is to uninstall the browsers and then add them back one by one over time to test performance.  IE can be uninstalled in Programs and Features>Features.  Save Edge for last because if necessary that must be done using Powershell:  How To Reinstall Microsoft Edge In Windows 10.   Hopefully it will be a third-party browser you suspect to make it easier.

If none of these help and you pass all steps in the checklist, plus issue persists after unplugging all other drives after moving Page File to C,  then I'd probably do the Windows 10 Repair Upgrade

with all of the other drives unplugged because it is easy and resolves almost all problems by reinstalling the OS keeping files, programs and settings.
