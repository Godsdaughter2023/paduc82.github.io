How do I know where the page breaks are in the Word Web App? - Microsoft Community

mikeyg79 say:

How do I know where the page breaks are in the Word Web App?

How do I know where each page starts and ends on Word Web App?

I can't tell when editing text and have to keep checking in the reading view?

Word Web App would be far better if I could edit in the print format view.
rob rolnick say: Hi mikeyg79,

Word can be thought of as having two forms of page breaks.  The first is caused by the type of paper specified for a section.  For example A4 in portrait orientation.  The second is caused by pagination within a section.  Pagination within a section is purely affected by Word's layout code, which would be virutally impossible to mirror perfectly in the browser.  Especially given how the Word Web App does not support all of Microsoft Word's rich features.  (E.g. footnotes, wrapping text around pictures, etc.)

As such, any pagination provided by the Word Web App editor would be "best guess" and would likely not mirror how the page would actually print.  For this reason the Word Web App viewer is used to provide pixel perfect representations of a Word document including the layout of pages.  It admittedly isn't the perfect solution, but I believe it to be the right choice.  (It would be worse if faux pagination suggested that a document would fit in one page and then you printed it and it was two.)

marshall barton say: Since the combine year-month is a text value , that is the expected sorting.  You can avoid that be making the month part two characters:
   [Year] & "-" & Format([Month], "00")

john p spencer say:

Try changing your base query to use
  Format([Month],"00")  so that the leading zero is present in the month.  If

The current layout of the columns is an alphabetic sort.

You could also use something like the following if the Year and Month fields are number fields.
[Year] *100 + [Month]

By the way Year and Month are both reserved words in Access since they are both VBA functions.  It is a good idea to avoid using reserved words as field or table names.  I would suggest you rename the fields to ReadingYear and ReadingMonth.
