how can I merge columns so that the numbers for the same names are automatically consolidated

I have two columns- one with names and one with numbers corresponding to them, if there are duplicate names how can I merge them so that the numbers for the same names are automatically consolidated?
jaeson cardaño say:

Okay, let's assume, A Column is the List of Names and B Column is the List of Numbers....

 

First you need to insert UDF (User Defined Function)...

  • Open up your workbook.
  • Get into VB Editor (Press Alt+F11)
  • In the Project window, Insert a new module (Insert > Module)
  • Copy and Paste the code (given below)
  • Once you pasted the Codes (Press Alt+Q) to return to your Workbook
  • Save your Workbook as Excel Macro-Enabled workbook.

 

Here is the code...

'==================================================================

Function ConcatIf(ByVal compareRange As Range, ByVal xCriteria As Variant, Optional ByVal stringsRange As Range, _
            Optional Delimiter As String, Optional NoDuplicates As Boolean) As String
    ' code base by Mike Rickson, MrExcel MVP
    ' used as exactly like COUNTIF() with two additional parameters
    ' of delimiter and "no duplicates" as TRUE/FALSE if concatenated values
    ' might include duplicates  ex. =ConcatIf($A$1:$A$10,C1,$B$1:$B$10,", ",True)

Dim i As Long, j As Long

With compareRange.Parent
    Set compareRange = Application.Intersect(compareRange, Range(.UsedRange, .Range("a1")))
End With

If compareRange Is Nothing Then Exit Function
If stringsRange Is Nothing Then Set stringsRange = compareRange
Set stringsRange = compareRange.Offset(stringsRange.Row - compareRange.Row, _
                                    stringsRange.Column - compareRange.Column)
   
    For i = 1 To compareRange.Rows.Count
        For j = 1 To compareRange.Columns.Count
            If (Application.CountIf(compareRange.Cells(i, j), xCriteria) = 1) Then
                If InStr(ConcatIf, Delimiter & CStr(stringsRange.Cells(i, j))) <> 0 Imp Not (NoDuplicates) Then
                    ConcatIf = ConcatIf & Delimiter & CStr(stringsRange.Cells(i, j))
                End If
            End If
        Next j
    Next i
    ConcatIf = Mid(ConcatIf, Len(Delimiter) + 1)
End Function

'==================================================================

 

Enter the names you want to see the numbers under for them in C Column starting at C1, then enter this in D1...

=ConcatIf($A$1:$A$100000,$C1,$B$1:$B$100000,", ",TRUE)

 

Copy C1 paste downward.

 

UDF is from this link, thanks to them...

http://www.excelforum.com/excel-general/680160-returning-a-multiple-result-lookup-to-a-single-cell.html

 

Hope it helps,

Excel moves the rows with #DIV/0! in the Range % column to the top, but leaves their relative position unchanged. So if the row with MM07/80 /58 FT BRN in the Grade column was above the row with ECH-65 3515 HNY before sorting on Range %, it will still be above it after sorting.

vijaykumar shetye say:

Dear JayantDubey,

(1) Your statement "there can't be one more infinite and one less infinite.", is wrong. There can be one more number within infinity. The total count of whole numbers is 1 more than the count of natural numbers.

The total number of whole numbers starting from a particular number 5, will be 5 less than all whole numbers.

The total count of odd or even numbers is infinite, but less than that of all natural numbers.

The total count of all numbers between 1 and 2 is also infinite. There are infinite such cases.

To avoid such issues, errors are always treated differently by many software applications.

(2) In many cases, the result of #DIV/0! error is not infinity, but rather it is infinite number of values.

(3) Generally functions like IFERROR(), IF(ISERROR()), ISNA() and others are used by users to avoid such situations.
