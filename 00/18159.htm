In SharePoint Online, it is not feasible to give users the Add Items permission without giving them the View permission.
It's an required permission for Add Items to work.

As a workaround, you can try creating an permission level with Add Items, View Items permissions on the top-level site of the site collection.

The users who are granted this permission level can create a new file and upload files to a document library under that SharePoint site collection.

You can follow the steps below to do this.

First, create a permission level.
1. On the top-level site of the site collection, click Settings>Site Settings.
2. On the Site Settings page, in the Users and Permissions section, choose Site permissions.
3. On the Permissions tab, in the Manage group, choose Permission Levels.

4. On the Permission Levels page, choose Add a Permission Level.
5. On the Add a Permission Level page provide the name and the description.
6. In the Permissions area, select the check boxes as the following picture shows:


7. Click Create.

Next, on the Site Permissions page, click Create Group, and give the above permission level to this group.
Then click the created group, click New and choose Add users to this group, you can add users to this group to give them this permission.

You can also apply this permission level to a SharePoint list or library under this site collection, you can refer to Edit and manage permissions for a SharePoint list or library.
