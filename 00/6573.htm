mstaessen say:

From what I have seen, there seem to be two HTTP engines in Windows. The one in IE, which is used by most applications and which is configured from the Settings app/Control Panel, and another one you can configure with the command netsh winhttp.

If you open a command prompt with administrative privileges and issue the command

    netsh winhttp import proxy source=ie

it will update the settings of the latter to be the same as the IE http engine.

Doing so will enable the use of the store, but will also enable OneDrive Sync, Calendar, Mail and probably others...

But you need to remember to set this manually, because it cannot be set via the Settings UI.

Resetting can be done by issuing

    netsh winhttp reset proxy

Hope this helps.

lzandman say:

I think the problem is that the Windows 10 Store app does certificate pinning. That means it requires an SSL encrypted connected to its back-end and they don't want anyone snooping in on that connection. Most companies install proxy servers that basically perform a man-in-the-middle (MITM) attack on the connection. They'll act like they are Microsoft by presenting a fake Microsoft SSL certificate to the app. This enables them to decrypt the connection and let their security software do deep packet inspection / logging.

By use of certificate pinning the Store app checks if the connection is actually protected by an official Microsoft certificate, instead of the fake company-provided one. And it will just present an error when it's not an official certificate.

This indeed prevent the use of the Store app in most company scenarios. But that's not a bad thing, perse. Companies would like to control what apps are installed on users computers and they have other means to enable this to work. If your company doesn't have this set up, it means they probably don't want you to install apps and you shouldn't be doing that.

If your system allows it, you could always connect to a different network (i.e. your home network) and disable the proxy server (or create exceptions for the microsoft domains).
