smithl4819 say:

Disk Management doesn't allow any actions on the EFI/recovery partitions.  Need to use Diskpart run as administrator.

1. Open a command prompt as administrator.

2. Run Diskpart application by typing Diskpart in the command prompt.

3. In the “Diskpart” prompt, enter rescan command and press Enter key to re-scan all partitions, volumes and drives available.

4. Then type in list disk and press Enter key to show all hard disk drive available.

5. Select the disk that contains the partition you want to remove. Normally, with just 1 hard disk, it will be disk 0. So the command will be:

Select disk 0

Finish by Enter key.

6. Type list partition and press Enter key to show all available and created partition in the disk selected.

7. Select the partition that wanted to be deleted by using the following command, followed by Enter key:

Select partition x

Where x is the number of the recovery partition to be removed and unlocked its space. Be careful with the number of this partition, as wrong number may get data wiped off.

8. Finally, type in delete partition override and press Enter key.

Be really careful that you are not deleting the wrong partitions!

how to remove a shortcut virus on my harddrive - Microsoft Community

garthgovender say:

how to remove a shortcut virus on my harddrive

please assist i have a short cut virus on my hard drive how can i remove it

aerdian say:

How to remove a shortcut virus:

  1. Open File Explorer.
  2. Find the infected drive. If it's an external drive like a USB, you'll need to connect it.
  3. Press shift and right-click on the drive.
  4. Select Open command window here.
  5. Type in attrib -h -s -r -a /s /d X:\*.* and press enter. Replace "X" with your drive letter.
  6. Type in del *.lnk and press enter.

suzanne s. barnhill say:

I'm going to jump in and assume you probably mean "footer" rather than "footnote" (refer to Jay's explanation). In that case, you need to be aware that there are settings in Word that can give you up to three different footers per section.

When you have the header and footer open for editing, Word will display the contextual Header & Footer Tools | Design tab. In the Options group on that tab are check boxes for Different First Page and Different Odd & Even Pages. You can enable Different First Page selectively for one or more sections; this will add a separate First Page Header and First Page Footer, to which you can add different content. The Different Odd & Even Pages setting applies to the entire document (it can't be enabled or disabled for a single section) and gives you a separate Odd Page Header/Footer (which contain the content of the original primary Header and Footer) and Even Page Header/Footer.

Those options can take you most of the way you want to go. You can put some of the footer content in all the footers, some of it just in specific ones. For footer content that appears only on the last page, you will need a field:

{ IF { PAGE } = { NUMPAGES } "Content you want only on the last page" }

The braces in the above formula can be inserted only with Ctrl+F9; you can't type them from the keyboard. Pressing Ctrl+F9 gives you the pair of braces with two spaces between and the insertion point between the spaces. You then type the text between them. A shortcut to insert the { PAGE } field is to press Alt+Shift+P.

If you are restarting numbering in each section and want content only on the last page of the section, you can try this field:

{ IF { PAGE } = { SECTIONPAGES } "Content" }

jay freedman say:

> Is the apparent absence of WOW64 my problem or do I need to install the 64-Biot version of Office?

Neither. The problem lies elsewhere.

The code for WOW64 functionality is (mostly) in the folder C:\Windows\SysWOW64 and its subfolders. You should be able to see it in File Explorer, but don't mess with it. If there are no problems with other 32-bit programs outside of Office, then WOW64 is working as it should.

The 64-bit version of Office is not likely to behave any better than the 32-bit version, and it has compatibility issues that make it not advisable unless you really need to use the huge memory addressing scheme that it offers. The proper approach is to determine what causes the crashes and to treat the cause, not the symptoms.

According to this article, "If you use Avast Antivirus version 10.3.2225, and you upgrade to Windows 10, your Office apps may crash when you try to open them." The recommended solution is to upgrade Avast AV to the R4 release.

If you upgraded an existing system, you may have Office add-ins that don't play nicely on Windows 10 (although in my experience everything that ran on Windows 7 will run correctly on Windows 10).

Use the Windows Event Viewer to examine the reports from Office crashes to see if there's a common item in them, particularly the "Faulting Module" item. Also, make notes of exactly what you were doing in the few seconds before each crash -- for instance, if the fault occurs in the spooler when you print, you can try reinstalling or updating your printer driver.

Use the Office Configuration Analyzer Tool (OffCAT) to look for configuration errors. Instructions for using it are at https://support.microsoft.com/en-us/kb/2812744.

For example, the data could be collected via a form rather than on a PPT slide, then code could generate the slides from the collected data.  You might do this in such a way that the users never even SEE the final slides, or you could hide the slides but show them a PDF generated from the slides.

Under code control, it'd also be possible to pull Excel data into a table and break it across multiple slides.  It's not something I'd consider a beginner project though.
