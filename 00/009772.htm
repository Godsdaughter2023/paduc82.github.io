john - project say:

dictatorbob211,

You're missing a basic principle of tracking in Project. Unless you tell it different, as soon as you enter a non-zero value into the % Complete field, Project assumes the task started as planned (i.e. on the Start date). Similarly if you declare a task complete, Project assumes the task finished as planned, even though that date may be in the future.

It is quite common to have tasks that start earlier or later than their planned start date and also that finish earlier or later than their planned finish date. When a task starts, if it did not start as planned, enter the date into the Actual Start field. That's why that field exists. You will still see the Start field change to agree with the actual start and that's how Project adjusts the plan to match reality. Project will automatically hold the duration and move the finish date accordingly.

Similarly if a task finishes earlier than planned, enter the actual finish date into the Actual Finish field. Again Project will adjust the Finish field to agree with the real finish date AND will also now adjust the duration to match the difference between actual start and actual finish.

But before you do any statusing of your plan make sure you first set a baseline. That preserves several fields of the original plan which can then be used to compare the original plan to the current plan.

And yes, managing a project may seem "cumbersome" but Project is just a tool, it has zero intelligence. It can't possibly know all the variables that impact a project, but it can perform the basic scheduling operations that free up the project manager to make the softer decisions.

Hope this helps.

John

Sorry, there is no shortcut to managing a project. trevor rabey say:

Just a couple of observations here, and probably also repeating what John has already said.

15th of October is Sunday, a nonworking day in the standard calendar.

Without changing the project calendar, the task will be scheduled to start on Monday the 16th, and 20 working days duration takes the finish to 10th of November, not the 15th as in your example. Ignoring that and pressing on...

At your second paragraph, you say you want the actual start to be set to the current date (or the status date) if the % complete is input as anything other than 0. I don't see how a task that starts now can be, say, 80% complete.

The MSP default when you input a % complete is precisely what should happen, what must happen You have basically told it that the task has actually started, but provided no information about when it actually started, so MSP has no choice but to assume that it actually started as scheduled. And it will do this even if that means that the actual start is in the future (ie impossible), and without any warning. That's fine as long as you then overwrite the default and input what the actual start actually was.

Duration should always be an input. That's why the duration column is right next to the task name column in the entry table.

Regarding your last paragraph, it is not too slow and cumbersome if you have the tracking table in front of you rather than displaying all of the columns you need in the entry table. View, tables, tracking.

Two things:

When doing tracking, do it in the tracking table.

Give up typing in % complete. Stick to inputting the facts in the order in which they occur - actual start, actual duration, actual finish, then the actual work and actual cost. Let the software calculate the % complete.

tom_bpc say:

dictatorbob311,

John an Trevor have given you excellent advice, which I would repeat.  In particular, updating task progress using %Complete (other than 100%) is not good practice for a number of reasons.  Allowing the program to enter or modify actual dates is also not advised.

Nevertheless, Project provides a setting that will lead to a result that is close to what you desire in your second paragraph.  Specifically, go to the bottom of the Advanced Project Options and check the boxes for "Move end of completed parts after status date back to the status date," and "And move start of remaining parts back to the status date."  (You could also check the two boxes for "Move start of remaining parts before status date forward to status date," and "And move end of completed parts forward to status date", although these aren't required in your example.)  These options do exactly what they say they will do: In response to a %Complete or Actual Duration entry, Project will adjust the Actual Start and Resume dates to ensure that all completed work is in the past (first box) and all incomplete work is in the future (third box).  The second and fourth boxes essentially remove splits that are created by the other two.  These options don't change the Duration or the Remaining Duration.  That's for you to do.

I don't use these options because they over-write actual start dates that have been manually entered - such dates should be sacrosanct.

Good luck, tom

Hello,

When you use the newest version of Skype for Outlook.com, you will have the option to automatically add people from your address book who are also on Skype. If you chose to automatically add friends, you might notice that when you visit your People list, you'll see suggested contacts that weren’t already in your Skype contact list. These suggested contacts are people who are in your other address books who are also on Skype. You can remove them if you'd like by deleting individual contacts or turning the feature off in your Skype settings.

To turn this feature off:

  1. Click Settings on the sidebar of the Skype plugin.
  2. Click Contacts.
  3. Under How people find me, toggle the Automatically add friends option off.

Hi Ozone,

A.   Is this issue with a single user account?

B.   Are you having trouble accessing secured websites?

C.   Are you facing this issue with all the websites?

Step 1

Make sure that any third-party firewall or antivirus programs that are installed on your computer are configured correctly and are not preventing you from connecting to Web sites. Many at times a problem with your antivirus or firewall can cause this issue.

Configure the settings to default and restart your Computer. Check if you are able to view the secured websites after restart.

If this doesnt fix it, temporarily disable your antivirus or firewall and try the site again. If you can access the site after temporarily disabling your antivirus or firewall, then you'll want to repair or reinstall your firewall or antivirus to fix this.

If, however, this doesnt fix the problem, proceed to the next section.

Step 2

To find out if your router is causing the problem, unplug the network cable that runs from your cable modem to the router, and plug it directly into the network (ethernet) jack on your computer temporarily cutting the router out of the loop. If that solves the problem, check your router's manual to see if it can block or unblock certain ports. It's unlikely, but it may be configured to block port 443, which is used to make secure connections.

You may also contact your system administrator regarding this issue.
