vasil shterev say:

I figured it out so I will post my final version. It probably is not the prettiest but it gets the job done

Sub NewWordDocument()

Dim wrdApp As Word.Application
Dim wrdDoc As Word.Document
Dim i As Integer
Dim j As Integer
Dim LineCount As Integer
Dim DocumentCount As Integer

LineCount = Application.CountA(Range("A:A")) ' To see how many lines should be inputed
DocumentCount = Application.CountA(Range("B2:AZ2")) 'To see how many documents should be created

For j = 1 To DocumentCount
    Set wrdApp = CreateObject("Word.Application")
    wrdApp.Visible = True
    Set wrdDoc = wrdApp.Documents.Add ' or 'Set wrdDoc = wrdApp.Documents.Open("C:\Foldername\Filename.doc") 'sample word operations
    
    wrdApp.Selection.Font.Name = "Calibri"
    wrdApp.Selection.Font.Size = 18
    wrdApp.Selection.Font.Allcaps = True
    wrdApp.Selection.Font.Bold = True
    wrdApp.Selection.TypeText Text:="Title"
        
    With wrdDoc
    .Sections(1).Footers(wdHeaderFooterPrimary).PageNumbers.Add _
        PageNumberAlignment:=wdAlignPageNumberRight, _
        FirstPage:=True

    .Sections(1).Headers(wdHeaderFooterPrimary).Range.Text = "Header text" 'Add text in the header
    .Content.InsertParagraphAfter
    
    For i = 1 To LineCount
        If Cells(i + 2, j + 1).Value = "Yes" Then
             Range("A" & i + 2).Copy
             wrdApp.Selection.GoTo What:=wdGoToBookmark, Name:="\Page"
             wrdApp.Selection.MoveRight Unit:=wdCharacter, Count:=1
             wrdApp.Selection.PasteSpecial
            .Content.InsertParagraphAfter
        End If
    Next i
            
            wrdApp.Selection.Font.Name = "Calibri"
            wrdApp.Selection.Font.Size = 11
            wrdApp.Selection.Font.Allcaps = False
            wrdApp.Selection.Font.Bold = False
    wrdApp.Selection.TypeText Text:="Ending Text"
    

    If Dir("D:\" & Cells(2, j + 1).Value & ".docx") <> "" Then
    Kill "D:\" & Cells(2, j + 1).Value & ".docx"
    End If
    
    .SaveAs ("D:\" & Cells(2, j + 1).Value & ".docx")
    .Close ' close the document
    End With
    wrdApp.Quit ' close the Word application
    Set wrdDoc = Nothing
    Set wrdApp = Nothing
 
Next j

End Sub

Using the Windows File Explorer, paste the expression %appdata%\Microsoft\Office into the address bar and press Enter. In that folder, look at the Modified date of the file MSO1033.acl. If it's from before today but more recent than 2009, it should be intact. (If it's either today or a date in February 2009, then the crash removed the file and replaced it with a copy of the original file from the installation.) It's also possible that the file is missing.

If you have a valid backup, either from Windows 10's File History or from a full backup, copy that into the Office folder and restart Word.

If you don't have a copy of the file with your additions, it's time to put them back manually. It will help to use the AutoCorrect utility from my website.

  • Download the AutoCorrect2007.zip file from https://jay-freedman.info/ and extract the AutoCorrect.dotm template from it. Right-click the file and click the Unblock check box, click OK, and move the file to the %appdata%\Microsoft\Templates folder. 
  • In Word, start a new document based on the AutoCorrect.dotm template.
  • In the box that appears on the document, click the Backup button. 
  • When the processing is done, the Save As dialog opens. Save the document in a convenient folder. The backup will close.
  • Click the Close button.

Reopen the backup document, which will contain the AutoCorrect entries that come with Word. The start of the document looks like this:

You can add more rows to the table, one per new entry. Put the name of the entry in the first column and the replacement value in the second column. In the third column, type the word True if the replacement is a "formatted" value or False if it's a "plain text" value (in the same sense as in the AutoCorrect dialog).

You can use the Restore button in a document based on the AutoCorrect.dotm template to load the backup (with the new values) into Word. Also, the backup document itself is literally a backup of all your AutoCorrect entries, in case something goes wrong again.

As a final suggestion, if you haven't set up File History or a strict regimen of backups to an external drive, it's worth doing it now. Note that AutoText is different from AutoCorrect (and the latter is the topic of this thread).

Indeed, for specific AutoText entries (Building Block entries in the more general term) to be loaded into memory, the template containing those entries must be open.

What you should do is put your template in Word's Startup folder. By default, the location for that folder is C:\Users\YourWindowsUserNameHere\AppData\Roaming\Microsoft\Word\STARTUP. To find the exact path, look in the File Locations dialog box (in Word, click the File Locations button at File | Options | Advanced).
