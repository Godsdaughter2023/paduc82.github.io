Please see the following instructions between the two colorful parts and then you might understand what you should do.

 

Option Explicit

 

Public Const strConfigFileName = "RibbonSettings.cfg"

 

'###############################
'Convert a general path string.
'###############################
Public Function CGPath(ByVal strGPath As String) As String
    If Right(strGPath, 1) <> "\" Then strGPath = strGPath & "\"
    CGPath = strGPath
End Function

 

'################################
'Get the content of a text file.
'################################
Public Function GetFileContent(ByVal strPath As String) As String
    Dim FileNum As Integer
   
    '/* To obtain the next available file number. */
    FileNum = FreeFile(0)
    '/* Set the default return value of empty. */
    GetFileContent = ""
   
    On Error Resume Next
   
    '/* If the file exists. */
    If Dir(strPath) <> "" Then
        Dim strTemp As String
        Dim strResult As String
       
        '/* Open the file for reading. */
        Open strPath For Input Access Read As #FileNum
            Do While Not EOF(FileNum)
                Line Input #FileNum, strTemp
                strResult = strResult & strTemp & vbCrLf
            Loop
        Close #FileNum

        GetFileContent = strResult
    End If
End Function

 

'################################
'Get the content of a text file.
'################################
Public Function SetFileContent(ByVal strPath As String, ByVal strWrite As String) As Boolean
    Dim FileNum As Integer
    On Error GoTo errLine
   
    '/* To obtain the next available file number. */
    FileNum = FreeFile(0)
    SetFileContent = True
   
    '/* Write to file. */
    Open strPath For Output Access Write As #FileNum
        Print #FileNum, strWrite
    Close #FileNum
   
    Exit Function
errLine:
    SetFileContent = False
End Function

marshall barton say:

I think you're asking about the Subdataseet feature.  I don't use that  because creating forms with subforms provides more control and options in how to display the data.  But, to try to answer your question, open the table in design view and view the table's property sheet.  look down the properties to find the SubdatasheetName property.  That's where you specify the other table you want to see when you click the + sign.  If you were hoping to be able to select a table on-the-fly, forget it.  There is only one Subdatasheet property for per table.

 

As I said earlier, I think this is a crude, quick and dirty feature that I avoid.  First users should never see a table's datasheet because it is trivially easy to make a mistake that can have catastrophic consequences to the data in the main table and any related tables.  It is much better to create a form for your main table and use subforms for the related tables.

This little demo file concatenates the names in the opening form's combo box with address data, using the following function:

Public Function ConcatValues(strSeparator As String, ParamArray arrVals())
 
    ' Pass this function a character or characters
    ' to be used as the separator followed by the values to be combined
    ' For example: strFullName =
    ' ConcatValues(" ",FirstName, MiddleName, LastName)
    
    Dim X As Integer, strLine As String
    
    For X = 0 To UBound(arrVals)
        If Not IsNull(arrVals(X)) And Trim(arrVals(X)) <> "" Then
          strLine = strLine & strSeparator & arrVals(X)
        End If
    Next
      
    ' remove leading separator character(s)
    ConcatValues = Mid(strLine, Len(strSeparator) + 1)
 
End Function
