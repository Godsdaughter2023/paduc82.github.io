Order of operations:

First negation, then exponentiation: -2^2 = (-2)^2 = -2*-2 = 4
First exponentiation, then multiplication: -1*2^2 = -1*(2*2) = -4

The first results in -2^2 = 4 and the latter -2*2 = -4.
Parentheses > Negation > Exponentiation > Multiplication.

a. user say:

1. If it's today's date you need, you can try using a date field to insert it

{ DATE } (use ctrl-F9 to insert the { }).

Use a nested field

{ QUOTE { DATE } }

if you want Word to replace the date field by its result when you merge.

2. There can be some disadvantages to using .dot format with mail merge main documents, but the main advantage of saving it as a .dot is that the user /should/ use File->New to create a new document based on the template, and that prevents them from overwriting the template when they modify the mail merge main document, which they will do as soon as they do something like "preview". Certain things behave differently when you use .dot rather than .doc - as examples

 a. if you want your macro to run automatically when the user does File->New, you need an AutoNew macro. If the user starts by opening a .doc, you need an AutoOpen macro

 b. If your .dot  contains any { CREATEDATE } fields, the resulting merge document will have today's date. If you start with a .doc, the { CREATEDATE } field retains the creation date of the .doc you started with.

3.

> How can I page through all pages, make idividual changes, if necessary and print all letters?

Merge to a new document (you can do it from the Mailmerge Wizard or Toolbar, which you can enable via View->Toolbars). Then save and/or modify that document, then print it.

At some point my guess is that you may want to start merging to individual letter documents - if so, you should visit Graham Mayor's page at

http://www.gmayor.com/individual_merge_letters.htm

Peter Jamieson
