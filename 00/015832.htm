Query/SQL Help - how to combine multiple queries into one - Microsoft Community

thekingelk say:

Query/SQL Help - how to combine multiple queries into one

The only way I could figure out how to get the data I needed in a single query, was by creating four separate queries, then a fifth to combine them all. I have a feeling this is not the most efficient way to query this data. Could someone please help me combine these queries more efficiently? e.g. into a single statement. I've listed my queries below. THANK YOU!!  -Mark

Queries to combine the four queries:

SELECT GraphFYavoidApprove.FY, Sum(GraphFYavoidApprove.[Approved Cost Avoidance]) AS [SumOfApproved Cost Avoidance], Sum(GraphFYavoidPend.[Pending Cost Avoidance]) AS [SumOfPending Cost Avoidance], Sum(GraphFYavoidtargets.Target) AS SumOfTarget, GraphFYavoidApprove.catShort, Sum(GraphFYavoidEx.[Expected Cost Avoidance]) AS [SumOfExpected Cost Avoidance]
FROM ((GraphFYavoidApprove LEFT JOIN GraphFYavoidPend ON (GraphFYavoidApprove.catShort = GraphFYavoidPend.catShort) AND (GraphFYavoidApprove.FY = GraphFYavoidPend.FY)) LEFT JOIN GraphFYavoidtargets ON (GraphFYavoidPend.catShort = GraphFYavoidtargets.catShort) AND (GraphFYavoidPend.FY = GraphFYavoidtargets.FY)) LEFT JOIN GraphFYavoidEx ON (GraphFYavoidtargets.catShort = GraphFYavoidEx.catShort) AND (GraphFYavoidtargets.FY = GraphFYavoidEx.FY)
GROUP BY GraphFYavoidApprove.FY, GraphFYavoidApprove.catShort
ORDER BY GraphFYavoidApprove.FY;

Query 1: GraphFYavoidApprove

SELECT Right([register].[FY],4) AS FY, Left([register].[trkID],3) AS catShort, Sum(register.avoidApprove) AS [Approved Cost Avoidance]
FROM register
GROUP BY Right([register].[FY],4), Left([register].[trkID],3)
HAVING (((Right([register].[FY],4)) In ('2013','2014','2015','2016','2017','2018')));

Query 2: GraphFYavoidPend

SELECT Right([register].[FY],4) AS FY, Left([register].[trkID],3) AS catShort, Sum(register.avoidPend) AS [Pending Cost Avoidance]
FROM register
GROUP BY Right([register].[FY],4), Left([register].[trkID],3)
HAVING (((Right([register].[FY],4)) In ('2013','2014','2015','2016','2017','2018')));

Query 3: GraphFYavoidTargets

SELECT Right([targets].[FY],4) AS FY, targets.catShort, Sum(targets.avoidTarget) AS Target
FROM targets
GROUP BY Right([targets].[FY],4), targets.catShort
HAVING (((Right([targets].[FY],4)) In ('2013','2014','2015','2016','2017','2018')));

Query 4: GraphFYavoidEx

SELECT Right([register].[FY],4) AS FY, Left([register].[trkID],3) AS catShort, Sum(register.avoidEx) AS [Expected Cost Avoidance]
FROM initiatives LEFT JOIN register ON initiatives.trkID = register.trkID
GROUP BY Right([register].[FY],4), Left([register].[trkID],3), initiatives.initiativeStatus
HAVING (((Right([register].[FY],4)) In ('2013','2014','2015','2016','2017','2018')) AND ((initiatives.initiativeStatus)="In Process"));
