If that is the only cell that shows the formula instead of the result, the likely cause is that the cell format was Text when you entered the formula.

In that case, simply changing the cell format to General afterwards does not change the type of the cell contents.

In the future, be sure cells are not formatted as Text before entering a formula.  The formula can return text, even if the cell format is General or some numeric format.

cynergy say:

How do I convert a code to month & year?

Not all of the manufacturers do this, just this one. So, i wanted to evaluate the date codes that correspond to this mfg's model# and write a formula that would give me the year (in a four-digit format). I don't want to use "Text to Columns" because the other models use a conventional date code (mm-yy or mmyy). If I use =IF(LEFT(D2,4)="1813",LEFT(H2,2),RIGHT(H2,2)), (D2 is the model#, H2 is the date code), it works but only gives me a two digit value even if I have the cell formatted to a four-digit year (yyyy) or number (0000). The model# is formatted as text (contains numbers & text) and the date code is formatted as text (so I don't lose the leading zeros. I'm not sure how to get the results I want; a four digit year or number. Thanks for any help you can give. Cyd.

mike h.. say:

Hi,

 

If you have a date code of 05nn then it's impossible to know if it's 1905 or 2005 so Ron has had to be pragmatic in deciding whether this 2 digit year is in the 1900's or 2000's and he has made an educated guess that none of your date are prior to 1930.

 

the part formula (--LEFT(H1,2)<30) will return either TRUE or FALSE. if the number is <30 it will return TRUE otherwise it will return FALSE. An example is appropriate.

 

let's say we have a date code of 8509.

 

The first part of Rons formula LEFT(H1,2)+1900 simply takes the 85 and adds 1900 to it giving 1985. Note here that 85 is text but in the same way as -- can coerce text to a number so can an arithmetic operator so text 85 + 1900 works and gives 1985. Now we have to decide if it's 1985 or 2085 and the second part of the formula does that.

 

this bit 100*(--LEFT(H1,2)<30) evaluates as FALSE because the leftmost 2 digits aren't less than 30 so 100 * false =0. that may seem a bit odd but to Excel TRUE=1 and FALSE=0 and if you want to verify this enter TRUE in 2 seperate cells and in another cell add then together =A1+B1 and if a1 and B1 contain TRUE you get 2.

 

Now if the leftmost 2 digits had been <30 then 100*(--left(H1,2)<30) would have evaluated as 1 or 100*1=100 and added 100 years and got us into the 21st century.

 

So to repeat Ron has guessed; safely in my view, that none of your dates are prior to 1930.

ronrosenfeld say: To get a four digit number representing the year, try:

=LEFT(H1,2)+1900+100*(--LEFT(H1,2)<30)

or, if you know all the years will be after the year 2000:

=LEFT(H1,2)+2000

Format as General or as Number or as "0000"

jmh3143 say:
The error "Microsoft Account does not exist" usually means:

The person is logging in using an Alias.
If the user renamed the affected email address into a new account, the original account will become an Alias.
Thus logging in using the original account (Alias) will trigger the system to prompt the user that it does not exist.
The solution is to log in using the new account.

The person mistyped the email address.
Retyped the correct email address.

The person did not access the account (WITH A BROWSER) within 365 days.
The account will be deleted permanently in the system due to inactivity.
You may recreate the account as a new account if it is still available.
