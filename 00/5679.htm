You can use my RowCounter function. See the in-line documentation for typical usage:

Public Function RowCounter( _
  ByVal strKey As String, _
  ByVal booReset As Boolean, _
  Optional ByVal strGroupKey As String) _
  As Long
 
' Builds consecutive RowIDs in select, append or create query
' with the possibility of automatic reset.
' Optionally a grouping key can be passed to reset the row count
' for every group key.
'
' Usage (typical select query):
'   SELECT RowCounter(CStr([ID]),False) AS RowID, *
'   FROM tblSomeTable
'   WHERE (RowCounter(CStr([ID]),False) <> RowCounter("",True));
'
' Usage (with group key):
'   SELECT RowCounter(CStr([ID]),False,CStr[GroupID])) AS RowID, *
'   FROM tblSomeTable
'   WHERE (RowCounter(CStr([ID]),False) <> RowCounter("",True));
'
' The Where statement resets the counter when the query is run
' and is needed for browsing a select query.
'
' Usage (typical append query, manual reset):
' 1. Reset counter manually:
'   Call RowCounter(vbNullString, False)
' 2. Run query:
'   INSERT INTO tblTemp ( RowID )
'   SELECT RowCounter(CStr([ID]),False) AS RowID, *
'   FROM tblSomeTable;
'
' Usage (typical append query, automatic reset):
'   INSERT INTO tblTemp ( RowID )
'   SELECT RowCounter(CStr([ID]),False) AS RowID, *
'   FROM tblSomeTable
'   WHERE (RowCounter("",True)=0);
'
' 2002-04-13. Cactus Data ApS. CPH
' 2002-09-09. Str() sometimes fails. Replaced with CStr().
' 2005-10-21. Str(col.Count + 1) reduced to col.Count + 1.
' 2008-02-27. Optional group parameter added.
' 2010-08-04. Corrected that group key missed first row in group.

  Static strGroup As String
 
  On Error GoTo Err_RowCounter
 
  If booReset = True Then
    Set col = Nothing
  ElseIf strGroup <> strGroupKey Then
    Set col = Nothing
    strGroup = strGroupKey
    col.Add 1, strKey
  Else
    col.Add col.Count + 1, strKey
  End If
 
  RowCounter = col(strKey)
 
Exit_RowCounter:
  Exit Function
 
Err_RowCounter:
  Select Case Err
    Case 457
      ' Key is present.
      Resume Next
    Case Else
      ' Some other error.
      Resume Exit_RowCounter
  End Select

ken sheridan say:

The real time apply of this concept is

A query fetch records of 100 employees based on their seniority (1 to 100) holding the same post.

And there are 3 vacancies only available  to be filled up from the above employees by promotion.


What would be the basis for selecting a limited number of employees from those equally qualified 100 employees?  Is it entirely arbitrary? If so one of the standard SQL methodologies for numbering rows is to use a subquery, in which the primary key is used as the tie-breaker in the event of more than one row having the same value (of seniority in your case) in the column by which the set is ordered.  A subquery can thus be used to restrict the result table on the basis of a parameter.

The following is an example, using Northwind data.  This returns the top n rows from the Customers table sorted in descending order by Job Title.  In this example, this does not return the rows in order of seniority of course as the order is alphabetical by job title, but in reality a column in which the higher values mean greater seniority would be employed.

If, with this example 8 is entered as the parameter value, 6 employees with the job title Purchasing Representative and 4 with the job title Purchasing Manager are returned.  However, there are a further 9 employees with the job title Purchasing Manager who are excluded from the result table.  These are arbitrarily excluded on the basis of their lower autonumber ID column values.

PARAMETERS [Enter number:] SHORT;
SELECT T1.*
FROM Customers AS T1
WHERE
  (SELECT COUNT(*)
    FROM Customers AS T2
    WHERE T2.[Job Title] >=  T1.[Job Title]
    AND ( T2.ID >= T1.ID
    OR T2.[Job Title] <> T1.[Job Title])) <= [Enter number:]
ORDER BY [Job Title] DESC;

NB:  The declaration of the parameter in a PARAMETERS cluse is essential with this type of query.  It will not return the correct results otherwise.

One possibility would be to launch PPT every so often using the Windows scheduler.  Here's one link:

http://www.tushar-mehta.com/powerpoint/auto_schedule/

It describes older versions of Windows/PPT but should be easily adaptible.  Or google for eg.

Powerpoint task schedule

Another approach:

I have an add-in that converts PPT to HTML, called PPTools PPT2HTML.  Some of our customers use it in situations like yours.

They'll set up a looping presentation, convert it to HTML, then play it back in a browser on another PC. 

PC-A has PowerPoint and does the editing/conversions, saving them to a shared drive.

PC-B plays the conversion back in a browser from files on the shared drive.

The presentation is a series of HTML and image files; if PC-A makes changes to the PPT and exports new HTML/Images of the changed slides, PC-B picks up the new versions next time through the loop.

This also got round the problem that some older versions of PPT had on some computers ... it'd run for hours or days but then lock up solid. 

Take PPT out of the loop, it can't lock up in the middle of the loop.  ;-)

If animation is important to the presentation, however, this wouldn't be a good choice.
