¡firedog say:

The emails appear to be listed randomly.

It may appear so, but I'm sure it's not. I think you'll find that a message list sorted by To will show messages in order of Subject, remembering that subject prefixes like Re: and Fw: will be ignored for sorting purposes. The listing for messages with the same subject will in my experience be ordered by date, something like this:

Messages 3-7 have the same addressee, so they are ordered alphabetically by Subject. Messages 4-6 have the same subject, so they are additionally ordered by Date.

I think all the bases are covered so there won't ever be any 'random' ordering.

Entering a name in the Search for a message box will find all messages sent by Jim, because it first consults the message database index for matches, and this will be pretty instantaneous. Only after this phase is complete will the search continue using the Windows Search index to find matches in the message body, which you don't want anyway. Find is a real-time search trawling through all the message headers looking for matches, and this will often be a slow process.

shawn - support engineer say:

Hi AppleWilliam,

Thanks for posting. The first thing we need to do is identify where the issue actually is. This could be an issue with a piece of hardware, Internet Explorer, Windows, or your ISP.

First when the issue happens again do the following.

Type the following command:

IPCONFIG /ALL [Note that there is no space between the slash and ALL.]

Right click in the command window and choose Select All, then hit Enter.
Paste the results in a message here.

In the same command prompt window identify the DNS and Gateway (it will look something like          Default Gateway . . . . . . . . . : 192.168.1.1 and DNS Servers . . . . . . . . . . . : 192.168.100.54 )

Now in the black comand prompt type the following command:

PING 192.168.1.1 (replacing your gateway IP here) [Note that there is no space between the slash and ALL]
Let us know if you get reply or timeouts.

Also again in the same window type the following command

PING 192.168.100.54 (replacing your DNS IP here) [Note that there is no space between the slash and ALL]
Let us know if you get reply or timeouts.

If both reply then you may have an issue with your web browser. If you use and other Web Browsers test with those and see if you have the same result.

If you do not get a reply, test with another computer if possible to see if the issue persists there. If the issue does not occur on the other PC, then Check your network cable connecting to the PC and see if there is a link light. If there is a link light then try to replace the cable and see if the issue persists.  If the issue persists after replacing cable with a known good working one. If the issue still occurs with a known good working cable then try a different port on the router or switch.

Hope this helps. Please let us know if any of these steps did or did not resolve your issue.

Exit Device Manager
Restart
