Stefan asked me to take an independent look at your question. I recorded the same macro as you showed, and ran it in a document (based on my English (US) Normal template) that has never had any contact with anything containing Dutch proofing language. The macro repeatedly and consistently displayed "NOT found".

As experiments, I tried (a) applying Dutch proofing language to some text in the document and then reapplying English proofing language to it, and (b) including Dutch proofing language as part of Body Text style but not applying that style to any text in the document. In both cases, both the manual search and the recorded macro continued to find nothing.

Although I don't have an explanation for the behavior you're seeing, I agree with Stefan that using a Range instead of the Selection is a better option. Unfortunately, the macro recorder doesn't know anything about Range objects and always uses the Selection. One advantage of the Range.Find method is that it always starts with the default set of properties, whereas a Selection.Find starts with whatever properties were used in the most recent search, either manually or by a macro. That means your code doesn't have to specify every property, such as .MatchCase, except where you want to use a non-default value. 

Sub FindDutchText()
    Dim rg As Range
    Set rg = ActiveDocument.Range
    With rg.Find
        .Format = True
        .LanguageID = wdDutch
        .NoProofing = False
        .Wrap = wdFindStop
        .Execute
        If .Found Then
            MsgBox rg.Text
        Else
            MsgBox "Not found"
        End If
    End With
End Sub

jay freedman say:

I ran a series of experiments of changing the Dutch language setting to English or to German, manually or by your SpellCheckerChangeDutchToEnglishUS macro, and then examining the document.xml for its language settings.

The conversions of Dutch to English completely removed all language properties from document.xml. That's probably because the default language for Windows and Office on my computer is English (US).

The conversions of Dutch to German simply replaced nl-NL with de-DE in all locations. There was no Dutch formatting left behind as in the document you posted.

I conclude that using the macro isn't the cause of the initial problem, and I still don't know how the document acquired its mixed formatting.

What I did find is that, starting with your posted document, I could replace all of the nl-NL instances with en-US by using Ctrl+A to select the whole document and then selecting English (United States) in the Review > Set Proofing Language dialog. To do this in a macro, use this code:

Sub SpellCheckerChangeDutchToEnglishUS_2()
    ActiveDocument.Content.Select
    With Dialogs(wdDialogToolsLanguage)
        .Language = "English (United States)"
        .NoProof = False
        .Execute
    End With
    Selection.HomeKey wdStory, False

End Sub

Replace "English (United States)" with any other language (exactly as it's shown in the Proofing Language dialog's list) to convert the document completely to that language.

If the entire document is already formatted as the single language you want, running this macro to "convert" it to the same language won't do anything -- but it won't hurt -- so you don't really need to detect whether the language settings are mixed.

mitch h. say:

Hi Ayman,


To address your concern, we suggest that you use the cascading dropdown in Excel. Here's how:
1. Type the entries that you want to appear in the dropdown lists, each in a separate column. Ex. Products (A, B, C, etc.)
2. Create names for your main list and for each of the dependent lists. You can do this by typing the name directly in the Name Box or by using Name Manager from Formulas tab.
Notes:

  • You should not include the title of the list in the named range.
  • The items to appear in the first drop-down list must be one-word entries only.
  • The names of the dependent lists must be the same as the matching entry in the main list.

3. Select the cell where you want the dropdown list to appear.
4. Click Data tab, then click Data Validation.
5. Select List under Allow then enter the range name in the Source field. Ex. =Products
6. Create the dependent dropdown list by selecting another cell and by applying the Excel Data Validation again, but this time, instead of the range's name, enter the "=INDIRECT(cell where the first dropdown appears)" formula in the Source field. Ex. =INDIRECT(A2)
7. Click "Yes" when this message appears: "The Source currently evaluates to an error. Do you want to continue?"

Let us know how it goes.

greg maxey say:

Sub ScratchMacro()
'A basic Word macro coded by Greg Maxey
  With Selection.InlineShapes(1)
    .LockAspectRatio = False
    .Height = InchesToPoints(3.2)
    .Width = InchesToPoints(6.15)
  End With
End Sub
