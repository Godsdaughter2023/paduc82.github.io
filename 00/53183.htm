Below is the guide my boss made to fix the issue:

After certain Windows updates, standard users (users who are not members of Elevated Group) cannot log into machines that they have not previously logged into. They input their credentials properly but are immediately signed out.

The cause for this issue is that the Default User profile is corrupted by the update, and the new user cannot create a profile, because the system cannot use the Default profile template to properly create a proper registry key. One will be created, but it is corrupted.

To fix this issue follow these instructions:

  1. Log into the affected machine as either local or domain admin
  2. Go to C:\Users
  3. If you cannot see the Default user folder click “View” and check the box next to “Hidden files”.  Be sure to uncheck this box when you are finished with all other steps.
  4. Rename the Default folder to “Default.bak”
  5. Go to C:\Windows.old\Users (If there’s no Windows.old folder use the backup Default folder in this share to complete the following steps  ****see below****)
  6. Copy the Default user folder.
  7. Go back to C:\Users and paste the Default folder there.
  8. If a folder has been created for the affected user, delete it.
  9. Deselect the check box next to “hidden files”.
  10. Right click over the Windows icon and select Run.

    Hi Cassie, 

    Thank you for writing to Microsoft Community Forums.

    If you want to see the underlying script commands, then you can Right-Click on the ASPX file and select Edit. The file should open in Notepad and you should be able to read the code. 

    If you want to convert that ASPX file to PDF file to just view the content of the file, then you can simply rename the ASPX file to whatever you expect it to be. For example, if you expected a PDF version and instead got an ASPX file, just rename the file as ‘filename.pdf’ and then open the file.

    Note: In order to rename the file, your computer has to be set up to show you the file extension and to do this, press the ‘Windows’ key and the ‘R’ key from your keyboard and type ‘Control folders’ without quotes and hit ‘Enter’. Then, go to View tab and scroll down the option ‘Hide extensions for known file types’ and uncheck that option. Then, click on Apply and OK.

    Follow the below steps to rename the ASPX file to PDF.

    Step 1: Right-Click on the ASPX file and click on Properties.

    Step 2: Then, you will see the name of the ASPX file with the extension .ASPX, so you can delete that .ASPX and change it to .PDF.

    Step 3: If prompted, ‘If you change a file name extension, the file might become unusable’, click on ‘Yes’.

    Step 4: Then, click on Apply and OK.

    Now, once you double click on that file, which you changed the extension, then that file will be opened in PDF format.

    In the same way, if you want that file to be opened with Word application, then you can rename .ASPX to .DOCX.

    Hope it helps.
