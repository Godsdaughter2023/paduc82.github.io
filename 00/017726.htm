Sort Merged Cells ( error message: merged cells must be same - Microsoft Community

i4b say:

Sort Merged Cells ( error message: merged cells must be same size )

Just trying to help you out, for me this worked. Hopefully it does for you as well...

I concluded that I cannot not work without Merged Cells because I also give them Conditional Formats and Validation Lists (Yep, I like to 'control the options' for the user).

I also encountered the issue that I could not sort rows with Merged Cells manually, even if there were no 'different sized Merges Cells' on any of the rows. (I checked this with a bit of code.)

Of course the rows did intentionally contain merged cells, however, they were intentionally the same size in the same columns.

Also merging was done only on the same row (horizontally) and not over different rows (vertically).

Anyway, this little VBA code below helped me out. Hopefully it helps somebody else out as well. And maybe Microsoft gets inspired...

First you must create an AutoFilter on the specific worksheet (I assumed the 'Activesheet') that contains the merged cells that you wish to sort.

Then set the variables:

        Set FilterSortSheet = Activesheet '(or any other sheet you wish to sort with)

        SortColumn1 =  2 '(2 is an example, I wish to sort on column nr. 2. you can change it in the column you wish to sort on)

        SortColumn2 =  8 '(8 is another example, just to show that you can sort on more than one criteria)

        FirstDataRow = FilterSortSheet.Autofilter.Range.Rows(2).Row

        LastDataRow = FilterSortSheet.Autofilter.Range.Rows(FilterSortSheet.Autofilter.Range.Rows.Count).Row

Then this code actually does the job:

        With FilterSortSheet.AutoFilter.Sort
            .SortFields.Clear
            .SortFields.Add Key:=Range(FilterSortSheet.Cells(FirstDataRow, SortColumn1),FilterSortSheet.Cells(LastDataRow, SortColumn1)), SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal

            .SortFields.Add Key:=Range(FilterSortSheet.Cells(FirstDataRow, SortColumn2),FilterSortSheet.Cells(LastDataRow, SortColumn2)), SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal

            .Header = xlYes
            .MatchCase = False
            .Orientation = xlTopToBottom
            .SortMethod = xlPinYin
            .Apply
        End With

The only remark I have (for Microsoft) is that they may have been 'overprotective' by disallowing any sorting on a Range that contains Merged Cells. It makes sense that you cannot sort rows with Merged Cells if some of the Merged Cells merge over rows (vertically). As long as this is not the case it should be completely possible as the above approach shows. So please Microsoft, first check if there are any Merged Cells that include more than one row, then show us your message. If there are no Merged Cells that include more than one row, let us sort our Merged Cells happily.
