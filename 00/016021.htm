Preview: Windows 10 build 10130 - Microsoft Community

andre da costa say:

Preview: Windows 10 build 10130

Technical Level : Basic

Summary

With two major conferences (BUILD and Ignite) now in the rear-view, Microsoft is ramping up its development process as it races towards its July 29th 2015 launch for Windows 10. At the end of April, the company delivered a new build of Windows 10 to Windows Insiders, dubbed the Windows 10 Insider Preview, build 10074. In addition to the new build, Microsoft has finally revealed the official name of its next generation web browser, now called Microsoft Edge. I have been forced to revise this preview multiple times now because the rapid delivery of newer builds since then, 10122 and 10130 the most recent.


Details

Depending on how you install the latest build you will receive a different experience. If you download through Windows Update, you will have booted into the streamlined setup experience which features a spherical design broken down into 3 phases: copying files, applying Windows and configuring settings. If you boot from setup media such as the .ISO file or bootable DVD or USB thumbdrive, it’s pretty much identical to Windows 8. I find it interesting Microsoft does not apply the end to end fit and finish to these parts of the operating system. OOBE hanged a few times and I had to restart a few times and patiently leave until I could log in and complete setup. Microsoft has done some work to speed up the Out of Box Experience, in particular the Installing Apps phase has been cut short; I still wish it was not there. Microsoft managed to this by having certain apps install in the background after reaching the desktop (smart).

There is still a lot of rough edges in this build and I would like to receive a build for a change that actually fix these problems once and for all and at least let me see some attention of them. The glaring ones are reliability of Wireless, the Network Flyout is just plain limited, it does not support the option to set a connection as a metered. Performance is an issue, I notice Universal apps are excessively CPU hungry. For instance, the Photos app from time to time, will pop up in the background running some unknown task or it will crash unexpectedly when I open it. I notice, if I leave my laptop running idle, the cooling fan will just rev up excessively requiring that I put the computer to sleep. I also notice some universal apps, even when they are closed will be running in the background. So, these are just some of the issue that I think need to be ironed out.

I initially started out reviewing build 10074, but this build was soon superseded by builds 10122 and subsequently build 10130. With the launch date for Windows 10 officially revealed (July 29th 2015), Microsoft is racing to finish up in time to push it to customers. The following I would like to see some focus on:

  • Fit and Finish – the drab white of the user interface is rough on the eyes. Its hard to look at your screen for long periods. So contrast is certainly overdue.

  • Excessive CPU usage – Internet Explorer and Microsoft Edge are the biggest culprits. I notice if I have 9 or more tabs open in IE the CPU fan just goes into overdrive. Memory usage in Edge is out of control. I opened a mashable web page and memory usage went up to 600 MBs just for that one web page alone.

  • System hang – critical_process_died – get this often when resuming from sleep or hibernation. I have reported it; I hope it doesn’t pop in the next build or even the final release or else I will be gladly sticking with Windows 7 Professional on this machine.

  • Fingerprint reader not supported – I guess I will have to depend on HP for an updated driver. I really do hope it gets updated. Can’t stand to see that exclamation sign in device manager.

  • Performance of the UI – I suspect optimizations will be done closer to the final build such as accessing jump list context menu faster.

Apart from the above issues, Windows 10 is shaping up to be a release I believe that does provide a decent balance between the old and new. Universal apps feel more cohesive and integrated with the desktop although I miss some of the old features such as Live Thumbnail previews and jump list. Microsoft has made many UI adjustments since build 10074 for example, the expand Start control which makes the Start screen a full screen now only available through Settings if you need it. The Power Options have also been relocated to its rightful place providing quicker access. You still can’t set default options for it though (which to be honest with you is not a big deal). I wish there was an option for the Start screen settings to hide all Universal apps. If you want to remove all the modern apps displayed on the Start you will have do that one by one (unfortunately). Resizing the Start menu which can still be done using the anchors is not fluid enough, it doesn’t feel native.

This build also features a revised icon set although many icons still remain the Vista style, but I really don’t see that as a big deal. Persons who derided the bright yellow folders (which I actually liked) will be pleased to see a return of something more akin to those found in Windows 10. Users will notice new subtle animations throughout the UI, progress bars for instance use a fill up effect.

Windows Update is probably the most controversial aspect of Windows 10. With this release Microsoft will be requiring Windows 10 Home users to automatically install Windows Updates. Users of the Pro edition will have limited but more flexible options. Users are starting to complain and I would not be surprised if Microsoft ultimately relents and give users the choice of managing Windows Updates they have since Windows Update was introduced.

Since we are in Settings; users will notice many Control Panel items have been modernized limiting the need to access it. There are still some areas of the system that has to be configured in Control Panel such as Themes and Power Options, but considering that these are rarely accessed areas of the system it is not really a big deal. Other areas of the system where you can see Microsoft eschewing the past include the new Date and Time applet from the system tray which features a straightforward date and calendar.

Conclusion

Windows 10 is certainly ramping up to be an exciting release for many. As evidence by the strong reception from those reserving for the free upgrade, its evidence that many are ready to leave Windows 8.1 and Windows 7 behind. Windows 10 will not be the jarring experience that Windows 8.0 was in 2012. It does have many differences, but they are improved and welcome and will make users engage more with Universal apps which have been through a rough time these past three years. There are a lot of areas I did not cover, but it’s kind of getting tiresome for me. Two weeks before Windows 10 launches I plan to make a short return to Windows 7 one last time to take advantage of the free upgrade offer. I have been dual booting Windows 10 since October and I have spent more time in it than my commercial installations of Windows. Which is probably a sign of approval in itself that this update is certainly a welcome upgrade.

joanne li msft say:

Hi Denise,

 

I'd like to explain that we cannot change the approve order type when we are trying to add a new approver to the SharePoint 2010 approval workflow (out of box).

We can only select the order type when we are starting the approval workflow, One at a time (Serial) or All at once (Parallel). After that, every time the existing approvers add a new approver to the workflow, the approve order type will inherit the order type how the prior approver was added, we can't make changes to it.

 

For example, the order type was selected to Serial when A was added as an approver. Now A wants to add B as an approver to the workflow, then the order type of B will be set to Serial.

 

Given this situation, you may try selecting the approve order type to All at once (Parallel) when starting an workflow.

 

Furthermore, after one person is added as an approver, he/she can go to Workflow Tasks directly to check the tasks that need to be approved.

Hello there, AndrewGFarmer! Thanks for posting your query on Microsoft Community.

About your concern, if your phone stops working or responding like it used to, restart it to get back on track.

To restart, press and hold the Power button until a Slide down to power off message appears on-screen, then swipe down. (It usually takes about three seconds for the message to appear.) To turn your phone back on, press the Power button.

If that doesn't work, press and hold the Volume down and Power buttons at the same time until you feel a vibration (about 10 to 15 seconds), then release the buttons. Your phone will restart automatically.


Please be reminded that:

Your phone may vibrate as soon as you press the Volume down and Power buttons because it's changing your ringtone to vibrate. Wait to release the buttons until you feel the second vibration 10 to 15 seconds later.

If you don't let go of the buttons immediately after the vibration at 10 to 15 seconds, your phone may show a black startup screen instead of restarting normally. If that happens, try pressing the button combo again.

Not all phones restart this way, so check with your phone's manufacturer to find out how to force a restart.


If your phone has a replaceable battery (it's often under your phone's back panel), you can also restart your phone by removing and reinserting the battery, and then pressing the Power button to turn on your phone. The date and time may be off after you restart this way, but you can fix that by swiping over to the All apps list from Start, then selecting Settings > Time & language > Date & time.
