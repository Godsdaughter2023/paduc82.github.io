marshall barton say:

As it say somewhere in the fine print, this is by "design".  The RecordCount property only tells you how many recors have been "accessed" by some internal record manager process.  For example. if you scroll the records to the last one, click on it, the scoll back to the first on and click on it, the number should be correct.  That's not useful, it's just to test the point.

To solve the problem, add a line of code somewhere to force the last record to be "accessed":

   Me.RecordsetClone.MoveLast

If you are setting the filter programatically, do it right after you set the filter.  Otherwise, you can do it in the Current event before the If statement.

Hi,

I modified a bit your workbook, you can download my version here:

VacationTracking-share

To share a bit my approach, correct me if I'm wrong.

In D2 you want the vacation start date based on the day+month of the hire date.

=IF(DATEDIF(C2;TODAY();"y");DATE(YEAR(TODAY());MONTH(C2);DAY(C2));"")

By using DATE(YEAR(TODAY());MONTH(C2);DAY(C2)), it will automatically adapt the date to 9/2/2015, 9/2/2016, 9/2/2017, ...

The end date of the vacation is very similar, just add 1 to the year (bold).

=IF(DATEDIF(C2;TODAY();"y");DATE(YEAR(TODAY()+1);MONTH(C2);DAY(C2));"")

But IMO the end date is the day before the new vacation period starts, maybe better to use:

=IF(DATEDIF(C2;TODAY();"y");DATE(YEAR(TODAY()+1);MONTH(C2);DAY(C2)-1);"")

The tricky part is the DATEDIF function (undocumented in Excel but a lot of hits to find)

You calculated the years of service by subtracting the hire year from the current year. This is not a correct approach, say I started 1 February 2010 and today we are 1 January 2015, your formula will return 5 which is not correct because my seniority is still 4, it will become 5 on the first of February! That is the reason why I use the DATEDIF function.

Instead of using a complicated IF function, you can use the VLOOKUP to get the number of vacation hours.

See the help for more details on VLOOKUP.

Finally, to calculate the number of vacation hours taken, I've added a calendar sheet for employee "name 1". You can duplicate the sheet for each employee and change the name of the sheet. Enter in A1 the current year.  The weekend days (Saturday/Sunday) are filled gray, non-existing days are black. Select the day and fill the number of hours taken.

In H2: =IFERROR(SUM(INDIRECT("'"&B2&"'!$B$2:$AF$13"))+SUM(INDIRECT("'"&B2&"'!$B$17:$AF$28"));"")

Herein is the INDIRECT function the most important part, it allows to refer text to a sheet and a range.

SUM(INDIRECT("'"&B2&"'!$B$2:$AF$13")) is equal to SUM('name 1'!$B$2:$AF$13) but by using INDIRECT it will automatically adapt to name 2, name 3, ...

What is BitGuard

roggen say: BitGuard, by PerformerSoft LLC, is a software component which register the bitguard.dll file under the AppInit_DLLs in the Windows Registry. Thus bitguard.dll will be loaded by each Microsoft Windows-based program that is running on the computer.

BitGuard also register its bitguard.exe file as a Windows Service. This means that bitguard.exe will start before users log on to the machine.
When BitGuard is running, it intercepts and blocks attempts to change some settings in the web browser, such as the home page, search settings and changes to the web browser's toolbar area. BitGuard pops up an alert window where the user is informed about the blocking.

If you have PerformerSoft's BitGuard on your system, you will typically see two "BitGuard.exe" processes running in the Windows Task Manager.

If you want to remove BitGuard, you can do so from the Windows Control Panel. If that failed, here's an alternative approach to remove it:
