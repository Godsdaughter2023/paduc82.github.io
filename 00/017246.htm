    Phone screen – Accessibility support

    The Your Phone app The Your Phone app supports screen reading technology for users.  When using the phone screen feature, screen reader spoken feedback will play from the PC.  To enable this accessibility service, there are a few necessary steps to get started.  

    Prerequisites: 

    1. Download the Android Accessibility Suite from the Google Play store on your Android phone.
    2. Enable the TalkBack accessibility service on your Android phone.  To do so:
      • Open the Settings app
      • Tap Accessibility
      • Under Services heading, Tap TalkBack
      • Turn switch for TalkBack “On” to enable and Click “OK” to accept service permissions 

    Note: If you have Secure Startup enabled, you will need to enter your PIN after accepting permissions. 

    Note: If you have Secure Startup enabled, you will need to enter your PIN after accepting permissions. 

    To enableYour Phone Companion accessibility: 

    1. Make sure you have the Your Phone app open and click to phone screen. Your Phone won't be able to configure accessibility with your PC correctly if you're not running the app.  The accessibility service requires you initiate a phone screen session before you can enable the service.
    2. Once you begin the phone screen session, the application will be able to recognize that TalkBack is enabled.  This allows the Your Phone Companion accessibility service to connect to your phone.  You can also enable the service using the following steps:
      • Open the Settings app
      • Tap Accessibility
      • Under Services heading, Tap Your Phone Companion
      • Turn switch for Your Phone Companion “On” to enable and Click “OK” to accept service permissions
    3. Once the Your Phone Companion accessibility service is enabled, the spoken feedback will play through the PC speakersor headphones plugged into the PC.  You can navigate through your Phone Screen or your PC by using your keyboard and mouse. 

    Navigating while service is enabled: 

    1. When the Your Phone Companion accessibility service is enabled, you will navigate your PC and Phone Screen with the keyboard using default Android accessibility keys.

    • To move to previous itemhold Alt + Left arrow

    VBA to Paste Data to a Word Doc without Activating/Selecting the - Microsoft Community

    ossiemac say:

    VBA to Paste Data to a Word Doc without Activating/Selecting the Document

    I have the code below to copy data from an Excel worksheet and paste into a word document.

    The code all works but I would like to be able to paste into the Word doc without Selecting/Activating the Word Doc so that I do not need to make Word Visible.

    Although it is only the code between the asterisk lines that requires modifying, I have included all of the code to make it easy for others to test. Just, copy the code into an Excel workbook and place a small amount of data on Sheet1.

    Note the comments at the top of the code re the Reference. When I get it working properly I will re-code with Late binding but for development purposes it is easier with Early binding so that I get the intellisense dropdowns.

    All help will be greatly appreciated.

    Sub ExportExcelToWord()
        'EARLY BINDING METHOD.
        'Go to VBA menu item Tools -> References and scroll down to _
         Microsoft Word nn.n Object library (where nn.n is version) _
         and Check the box and click OK. (Ensure you check the box _
         and not just select the line before clicking OK.)
        'The reference is set for this project only. Does not remain set for other projects.

        Dim objWord As Word.Application
        Dim objDoc As Word.Document
        Dim wsSource As Worksheet
        Dim lngLastRow As Long
        Dim lngLastCol As Long
        Dim rngToCopy As Range
       
        Set wsSource = ThisWorkbook.Worksheets("Sheet1")
       
        lngLastRow = LastRowOrCol(wsSource, True)   'True for Last Row
        lngLastCol = LastRowOrCol(wsSource, False)  'False for Last column
       
        With wsSource
            Set rngToCopy = .Range(.Cells(1, 1), .Cells(lngLastRow, lngLastCol))
        End With
       
        'Try GetObject first in case Word Application is already open.
        Set objWord = Nothing
        On Error Resume Next
        Set objWord = GetObject(, "Word.Application")
        On Error GoTo 0
       
        If objWord Is Nothing Then  'Word not open so create object
            Set objWord = CreateObject("Word.Application")
        End If
       
       '********************************************************************
        With objWord
            'Create a new Word document
            .Visible = True

    Sub References_Unlink()

    Dim Fld As Field, FtNt As Footnote, EndNt As Endnote, Rng As Range

    SDD compatibility - the real reason why some do work and some do - Microsoft Community

    twain---01 say:

    SDD compatibility - the real reason why some do work and some do not.....

    Hi

     

    The last days I played around quite a bit  to get my Runcore SSD running again with Windows 7 after I upgraded the SSD's firmware. After some investigations I can now state following:

     

    The SATA controller in the US15W chipset is not capable to support properly the Windows 7 trim feature together with Indilinx Barefoot controlled SSD's. If the firmware of the SSD has this feature enabled, then the SSD won't be detected anymore by the OS. I upgraded my Indilinx controller from firmware V1571 (no trim support) to the last version V1916 (trim support and higher performance) and got the described behaviour.

     

    Workaround: install a clean Windows 7 to the normal HD. Open the command prompt as administrator and put in: fsutil behavior set disabledeletenotify 1 . This will disable the trim function within the OS. Then clone the installation to the SSD. Voilà.

     

    These circumstances might be the reason why some of the SSD drives worked fine here with othe forum members and some not.

     

    Cheers

    twain

    • Bart PE boot CD or USB key
