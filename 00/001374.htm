You could use the following code in a standard Module and run it manually

Dim strName As String
Dim i As Long, j As Long, k As Long
With Sheets("List").Range("B1")
    strName = .Value
    For i = .CurrentRegion.Rows.Count - 1 To 1 Step -1
        .Offset(i, 0).Value = " "
    Next i
End With
With Sheets("Data").Range("A1")
    For i = 0 To .CurrentRegion.Columns.Count - 1
        If .Offset(0, i) = strName Then
            k = 1
            For j = 1 To .CurrentRegion.Rows.Count - 1
                If .Offset(j, i) <> "" Then
                    Sheets("List").Range("A1").Offset(k, 1).Value = .Offset(j, i).Value
                    k = k + 1
                Else
                    Exit For
                End If
            Next j
            Exit For
        End If
    Next i
End With

                        Exit For
                    End If
                Next j

            End If
        Next i
    End With

End Sub

joeu2004 say: SBrown wrote:

These are two separate problems.

If you want large numbers (more than 10 or 11 digits) to not be displayed with Scientific format, you need to change the General format to the Number format, perhaps with 0 decimal places.

However, when entering numbers, Excel will only interpret up to the first 15 significant digits.  It replaces any digits after that with zero.  So number 9405510200882825259216 will be interpreted as 9405510200882820000000.

There is no work-around for the latter.  Well, usually you can get a little more precision by entering the formula =9405510200882820000000+5259216.  But usually the result is not exactly what you intended; for example, it is 9405510200882825242880.

The issue is:  Excel relies on the native computer 64-bit binary floating-point form for representing numbers and performing arithmetic [1].  That limits precision, typically (but not always) to 15 or 16 significant digits.  The largest integer that we can enter without loss of precision is 9,007,199,254,740,992.

That is, indeed, the way to handle IDs that appear to be large numbers.  Format the cells text first, or prefix the IDs with a single-quote (apostrophe).  Or as you did, enter a formula of the form ="...".

But you need to be consistent:  even smaller IDs must be entered as text, even though Excel might handle them as numbers without problem.

One caveat:  if you use a text file (e.g. a CSV file) to save the data, you need to use the External Data Import Text feature to input the data into an Excel file, instead of opening the text file directly using Excel.

A clean install is when the computer is booted to the install media, and everything that was in the OS partition is removed. When the install media is run within the existing OS, it does an upgrade (or, if it's of the same build, a repair install) and all installed programs and data files remain, unless the installer is instructed to remove them.
