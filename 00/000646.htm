    CPU usage - Microsoft Community

    sharkly say:

    CPU usage

    What does it mean to limit the usage of the CPU while scanning? If it is 100% or 90%, does it mean that the scanning takes all the CPU ussage or that I can use it 100% for other porpuses? If it is 90%, the scanning is using 90% or I have the 90% of the CPU for other porpuses? Thank you
    greginmich say:
    What does it mean to limit the usage of the CPU while scanning? If it is 100% or 90%, does it mean that the scanning takes all the CPU ussage or that I can use it 100% for other porpuses? If it is 90%, the scanning is using 90% or I have the 90% of the CPU for other porpuses? Thank you

    You’re setting the limit for the amount of available CPU that’s going to be devoted to your scheduled scan (the maximum CPU load factor) – so setting the value at 100% means that you’re not explicitly reserving any resources at all for your own work, while setting it at 10% would mean that you’re reserving 90% of the CPU resources for yourself. The CPU throttle was originally added to MSE because of the trouble that people were experiencing with the high CPU utilization of the OneCare antimalware engine (MsMpEng.exe) on low-resource PCs. It was nearly impossible to do anything when a scheduled scan was in progress because the antimalware engine would hog almost all of the available CPU.

    The best value for this setting depends on the hardware resources that you have available, and on the workload that you’re actually placing on those resources while the scheduled scan is running. You can adjust this setting “on the fly”; so it’s relatively easy to find a suitable value for your particular circumstances, and then readjust as your workload changes. Generally speaking, things have changed quite dramatically since this feature was added, and in many cases, people with newer PCs should now be able to just turn off CPU throttling. If you monitor the CPU usage of MsMpEng.exe (in Task Manager) on a newer machine, you’ll probably find that it doesn’t use anywhere near 100% of the available CPU when a scheduled scan is running unbridled (with the CPU limit set to 100%; or with the “Limit CPU usage” option unchecked). People who are scheduling Full Scans should especially consider setting a higher CPU limit or turning off the CPU-limiting feature altogether, because the run time for a scan is inversely proportional to the throttle setting – so an unthrottled scheduled scan (100% CPU limit) will finish in about half the time of a scan with the default 50% CPU throttle setting.
