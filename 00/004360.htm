lizhughes say:

Data Validation in VBA

Can anyone suggest any code that will enable the data validation to be retained on the new copied sheet.

 

The original sheet that the data is copied from is called "Data Entry", the sheet that is created does not have a static name and instead it takes its name from the client reference number inputted in cell D3 of the Data Entry sheet.  The data that populates the data validation can be found on a separate sheet called "Lists" and is in cells A98:A108

 

Can anyone help?

jllatham say:

Jeeped, LizHughes,

After getting a copy of the workbook to examine I discovered the problem:

 

The Data Validation source is over at Lists!$A$98:$A$108.  However, each of those cells contains about 7 levels of nested IF statements!  Each one is dependent on the entry in $I$9 of the [Data Entry] sheet! 

 

So what happens is that a copy of the [Data Entry] sheet is made, but then it is cleared for more data entry actions - and all of the copies previously made take their data validation for cell C14 (where the problem is/was) from the current setup on the List sheet - which is still controlled by the content of 'Data Entry'!$I$9, not the copy.

 

What I did was to copy the entire setup from rows 98:108 of the [List] sheet and place it into columns beginning at AH1 on the [Data Entry] sheet and set it all up to reference appropriate cells on that sheet.  Then I hid the columns and made sure the Print Area didn't include these newly used columns.

 

Now the Data Validation setup and list is part of the worksheet, so when copies of it are made, they will be independent of the original [Data Entry] sheet.

 

jllatham say:

That should work.  When you Copy the sheet, the new copy becomes the active sheet, regardless of it's name  Notice that you have just referenced it as ActiveSheet.Name = Val & "-" & num  So it is still the active sheet, even though you just changed the name.  BTW: You could then reference it as:

Worksheets( Val & "-" & num)  at that point.

There are many ways to change the background slide color.

This is will change the background color of the slides that are currently selected.

With ActiveWindow.Selection.SlideRange
    .FollowMasterBackground = False
    .Background.Fill.Solid
    .Background.Fill.ForeColor.RGB = RGB(0, 45, 155)
End With

You could also change the colors of the custom layout or slide master itself thereby changing the colors of the slides following that master/layout too.

The only way that you can do that is when you have finished editing the document, select the underlined numbers (which, if you are careful, you can use the CTRL key to make multiple selections) and then toggle off the underlining by clicking on the

You can update just the page numbers in the table of contents without the underlining of the numbers being re-instated, but if you update the whole table (as you would need to do it additional headings have been added to the body of the document) the underlining will be re-instated.
