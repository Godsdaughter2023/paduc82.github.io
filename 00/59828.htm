Hello,

I have had the same problem and following solution worked for me.

Uninstall the Windows Calculator with the following Windows Powershell command:

Get-AppxPackage *windowscalculator* | Remove-AppxPackage

After this run the following command to re-install all default Apps:

Get-AppxPackage -AllUsers| Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}

The command may take some time and will also show some errors but after a reboot the machine should show the Apps again correct.

You have adequate specs, and lowering in game settings too much can actually move the

load more to the cpu causing an artificial bottleneck. You should be able to run this game close to maxed out.

- Software never uses cpu cores evenly, and an 80% peak cpu load on some cores should not generally be a problem,

but your gpu should run at a higher percentage than the cpu in the vast majority, if not all games.

Some say the gpu can run at 100% capacity, but I've found that TDR crash is somewhat more likely if the gpu

goes over 99% use.

- Because you mention 60fps, I assume you are using V-sync (and 1920x1080 res). When a system struggles

to produce steady 60 + fps  you can see big fps drops with V-sysnc enabled, but is should stay within a couple

of frames when V-sync is enabled if your system can push a steady 80+ fps in a particular game. 

However, V-sysnc affects games differently and it can have a negative impact on performance. I generally

only enable V-sync if the game has bad tearing without it, but I do use a 120/144Hz monitor, so tearing

is not so common as when using a 60Hz monitor.

- There could be another software that is conflicting.  MSI Afterburner, among others, has been known to

conflict with some games.

- Running in Clean Boot may help troubleshoot other potential software or Services conflicts.

How to troubleshoot a problem by performing a clean boot in Windows Vista or in Windows 7 > 10

"Im just wonder why there are people making "lets play" videos on YouTube that are able to sustain a constant 60 fps with the same GPU with no issues at all?"

Apples and Oranges.  It's the small differences in machine configuration and software that bring the issues that

make troubleshooting tricky. They have to also be playing the same map with the same amount of action, and on the same server if multi-player, as you are when they are testing.

However, your gpu use should be higher than cpu. This low use would, I believe be the root of your issue.

I would be leaving V-sync disabled, at least while troubleshooting the low gpu use.

I just fired up COD 2 (i7 4790 w/ GTX980) . With V-sync off  and fps ranging up to 300+
