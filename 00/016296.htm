ossiemac say:

I have not been able to replicate the problem but when you run code to show a userform with the Form's ShowModal property set to true, any code in the sub after the show command does not run until you close the Userform. I am not sure but I am suspecting that this is what is causing the problem with the remainder of the code running after the form closes.

You can establish the above by removing the comment character on the MsgBox line of code immediately after the line with UserForm.Show in my example below. The message is not displayed until the form closes.

Try inserting Exit Sub immediately after each line of code with the Show. You will need to use separate lines of code for the If/then code so the Exit Sub goes in the correct place. That should prevent the remainder of the code in the sub from running when the form closes.

Rick's suggestion to set the rng object to nothing when finished with it is also good programming.

Also in VBA Tools -> Options check the box "Require variable declaration" so that Option Explicit is inserted at top of each module and that forces the issue to declare all variables.

Example:

Private Sub Worksheet_SelectionChange(ByVal Target As Excel.Range)
Dim rng As Range
Set rng = Cells(ActiveCell.Row, 1)
 
  'Backup Board
    If rng(1, 2).Value > 0 Then
        If Not Application.Intersect(Target, Range("D5:W70")) Is Nothing Then
          UserForm2.Show
          'MsgBox "UserForm2 (First instance) closed"   'For testing only
          Exit Sub
        End If
    Else
        'MsgBox "Sorry but you must enter a Name in Column B before proceeding"
        End
    End If
   
  'Part Time Available
    If rng(1, 2).Value > 0 Then
         If Not Application.Intersect(Target, Range("D72:W97")) Is Nothing Then
          UserForm2.Show
          'MsgBox "UserForm2 (Second instance) closed"   'For testing only
          Exit Sub
        End If
    Else
        'MsgBox "Sorry but you must enter a Name in Column B before proceeding"
    End If
  
  'Routes To Cover
    If rng(1, 2).Value > 0 Then
        If Not Application.Intersect(Target, Range("D100:X126")) Is Nothing Then
          UserForm1.Show
          'MsgBox "Userform1 closed"     'For testing only
          Exit Sub
        End If
    Else
        'MsgBox "Sorry but you must enter a Name in Column B before proceeding"
    End If
 
End Sub

Starting point when Word does something unexpected

As a general rule, when something seems to be going wrong, or at least not the way you want it to go, in Word, any version, go to Tools and then Options and study all the settings on all the tabs. You may even find some settings that you want to try that have nothing to do with the current problem.

When I was teaching Word and other Microsoft Office programs at a private business college, I had three rules:

1.         Always check Tools Options

2.         When in doubt, right-click on something

3.         If you're wondering how to set up a special kind of material in a document, consider whether Word tables is suitable. The placement officer of the school came to me several times and asked how I thought she should set something up. Each time the answer was, "I'd use a Word table." By the fifth time she had stopped asking--she got the point.
