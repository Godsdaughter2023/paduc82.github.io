In versions of Excel prior to 16.9, when you made a chart from a PivotTable it was a regular chart.

Beginning with 16.9, when you make a chart from a PivotTable it is an actual PivotChart - a different kind of chart.

This is because 16.9 introduces an entirely new code base to Office for Mac. Now we have the same code base as Office for Windows.

This is a "check or egg" problem. Microsoft could not build out the interface until the underlying object model was updated. Now that the object model has been updated, Microsoft can start updating the user interface.

It's a work in progress and it will take some time to complete. 

To get at the formatting options for the new PivotCharts, right-click the border of the entire chart. Choose Format Chart Area from the pop-up menu and then Format Chart Area panel will display. joeu2004 say: Shawn wrote:
I've got a tab for each month with totals, one of which being a margin percentage.  For each transaction I have column F saying, "=IF(D25=0,"",ABS(E25/D25-1))" with cell F28 saying "=SUM(F2:F25)/COUNTIF(F2:F25,">0")" for the total.  I'd like to get rid of the #div/0! for F28, and take the result of F28 on each page (12, one per month), copy it to the final page and average it on the final page to get an annual average margin.

To avoid the #DIV/0 error in Excel 2003, which does not have IFERROR, you might use the following formula in F28:

=IF(COUNTIF(F2:F25,">0")=0,"",SUM(F2:F25)/COUNTIF(F2:F25,">0"))

But it is unclear why you are ignoring legitimate margins of zero (e.g. E25=D25) in the first place.

If you really just want to exclude cells with the null string ("") because D25=0, the formula can be:

=IF(COUNT(F2:F25)=0,"",SUM(F2:F25)/COUNT(F2:F25))

[EDIT] But of course that can be written simply:  =IF(COUNT(F2:F25)=0,"",AVERAGE(F2:F25)).  Klunk!

In either case, it is unclear if the null string ("") should or can be zero in F28.  That depends on your design goals.

Moreover, it is unclear if your formula in F28 calculates the average that you expect.

Even if it does, it is unclear if the requested formula in the summary worksheet ("final page"), as you described it, would calculate the average that you expect.

-----

First, do you really need to "take the result of each page" to the summary worksheet?  Or are you doing that just because you think it makes it easier to calculate the average?

Second, note that in general, the average of averages (or ratios) is not the average (or ratio) of the whole.

What you calculate in F28 is the average transaction margin for the month.

But if you want to calculate the monthly margin, the formula in F28 should be:

=IF(SUM(D2:D25)=0,"",ABS(SUM(E2:E25)/SUM(D2:D25)-1))

And again, it is unclear if the null string ("") should be zero.

[EDIT] Or even one.  Some people treat anything/0 - 1 as 100% gain, just as 0/anything - 1 is 100% loss (-100%).

-----

The formula you request calculates the average of the monthly average transaction margin (unusual!), or the average monthly margin if you use my last formula above.  They can be written:

=IF(COUNT(Sheet1:Sheet12!F28)=0,"",SUM(Sheet1:Sheet12!F28)/COUNT(Sheet1:Sheet12!F28))

Note that SUM and COUNT are two of the few functions that allow "3D" references (e.g. Sheet1:Sheet12!...).  Replace "Sheet1" and "Sheet12" with the actual names of your 1st and 12th worksheets.

Also note that IF(COUNT(Sheet1:Sheet12!F28)=0,"",...) can be avoided if F28 returns zero instead of the null string ("") when the divisor is zero.

But if you want to calculate the average transaction margin for the

=IF(COUNT(Sheet1:Sheet12!F30)=0,"",

  SUM(Sheet1:Sheet12!F29)/COUNT(Sheet1:Sheet12!F30))

where in each worksheet, F29 is =SUM(F2:F25) and F30 is =COUNT(F2:F25).  (See "Design Note" below.)

And if you want to calculate the annual margin, the formula should be:

=IF(SUM(Sheet1:Sheet12!F30)=0,"",

  ABS(SUM(Sheet1:Sheet12!F29)/SUM(Sheet1:Sheet12!F30)-1))

where in each worksheet, F29 is =SUM(E2:E25) and F30 is =SUM(D2:D25).  (See "Design Note" below.)

-----

Design Note....

Instead of "3D" references like Sheet1:Sheet12!D2:D25, I would be inclined to calculate the partial sums in cells in each worksheet (e.g. F29 and F30), as you did using F28, then use a "3D" reference to sum all of the partial sums as demonstrated above.

That gives you the flexibility to change the number of transactions per month (e.g. D2:D50), and even to have a different number of transactions in each month, without having to change the summary worksheet.

d r h say:

In the screenshot, I have clicked on Pictures in the Menu. File Explorer opens showing the Pictures folders and files but all of those are on the 'J' drive as shown by right clicking on Pictures in FE and selecting properties.

First, create a new folder for each folder that you want to move on another drive(s) as required and suitably named.

Open C:/ Users/ Username in FE (since these are the actual folders being moved) and right click on a folder e.g. Documents and select properties. Click on the Location tab and then 'Move'. Navigate to the newly created folder on the other drive and click 'Select folder'.  If asked whether the folders should be merged - say yes. This is in case some programs insist on a fixed path for its files.

Repeat for other folders you want to move.
