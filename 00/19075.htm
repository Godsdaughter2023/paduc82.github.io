graham mayor say:
The short answer is no. Word is not a page layout application and the viewable 'pages' are a combination of different layers - imagine a stack of transparencies each with part of the document. Furthermore a displayed 'page' is a volatile entity that is determined by a variety of variable factors including the distance between the margins and the size of the text.

Documents are formatted with styles. You can only have one style definition per style name, so there is the potential also for conflicts between styles.

The answer to your question is therefore determined by what is actually on the 'pages' and the formatting both of the source and target documents.

Auto Populate body of Meeting Email when Resource:Room is selected

Is it possible to auto populate the body of Meeting Email when a room is selected?

What I'm trying to accomplish; when a meeting email is first created and then a room is select, I would like the body of the meeting email automatically populate with a brief description of the room and URL to a website that show either a Floor Plan to locate the room or map on the internet showing the location of the building .

larryjones- [o365] say:

Hey Rick,

After a lot of trial & errors and research I was able to successfully add text with a URL in the message of a meeting request and have that message with URL appear in the calendar.  What made this work was identifying the correct message header (X-MS-Exchange-Organization-BCC) being used by the room's alias for sending meeting request.

To accomplish this you need to do the following

  1. Log into the Online Exchange Admin Center
  2. Select Rules (under Mail flow) from the Dashboard
  3. Select "+" sign and select Create New Rule
  4. Give the rule a  Name
  5. Select "More Options"
  6. Underneath "*Apply this rule if..." select "A message header..." and then select "matches these text pattern"
  7. Add the following text to "*Enter test..." "X-MS-Exchange-Organization-BCC"
  8. Add the Room alias in the "*Enter test patterns..."
  9. Add a new condition by selecting "add condition"
  10. Underneath "*Apply this rule if..." select "The message properties...." then select "include message type"
  11. Click "*select one..." a  pop out box apper. On the "select message type" select "Calendaring"
  12. Under "*Do the following...." select "Applied a disclaimer to message with..." and then select "append a disclaimer"
  13. Add your information you wan tot appear in the meeting message to "*Enter test..." and select "Wrap" in "*Select one..." To add a new lines to the information you enter, just add a "<br>" at the end of each sentence.

Note: The sender of the meeting request will not see the message with text and/URL only the recipients. Maybe further down the road I might take a crack at adding the text to the sender calendar.

If you find a error in these instructions please update this post. Thank You.

Hope this help others.

Larry
