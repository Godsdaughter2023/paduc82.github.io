jay freedman say:

The only problem with the macro you posted is that the Do...Loop to count the files leaves the variable named image equal to "". That's why the Add command complains about an invalid filename. To fix that, you just need to run the image = Dir() statement again between the If count statement and the Add command.

You don't need to select the range rng; you can just  write rng.InlineShapes.AddPicture instead of using the Selection there. This isn't an error and won't change the result of the macro, but avoiding the Selection (and possibly redrawing the screen) can speed up the macro.

The following code shows how to incorporate a file picker dialog in your macro:

Sub AddPicture()
    Dim dlg As FileDialog
    Dim rng As Range
    Dim vShape As InlineShape
    Dim imgpath As String
    Dim FolderPath As String
    Dim count As Integer
    Dim image As String

    FolderPath = ActiveDocument.Path

    imgpath = FolderPath & "\*.jpg"

    image = Dir(imgpath)
     

    '// Count the .jpg files in folder

    Do While image <> ""
       count = count + 1
        image = Dir()
    Loop


    If count = 0 Then
        MsgBox "No jpg files found in " & imgpath, vbExclamation
        Exit Sub
    ElseIf count = 1 Then
        image = Dir(imgpath)
    Else
        Set dlg = Application.FileDialog(msoFileDialogFilePicker)
        With dlg
            .AllowMultiSelect = False ' only one choice
            .Title = "Select one file"
            .Filters.Clear
            .Filters.Add "JPG files (*.jpg)", "*.JPG"
            ' you can add more file types with more .Add statements
            
            If .Show = True Then
                image = .SelectedItems(1)
            Else
                MsgBox "File selection canceled", vbInformation
                Exit Sub
            End If
        End With
        
        ' release the dialog's memory
        Set dlg = Nothing
    End If
    
    If image <> "" Then
        On Error GoTo ErrHandler
        
        '// Range where to add the picture
        Set rng = ActiveDocument.Tables(1).Cell(4, 1).Range
    
        '// Add the picture
        Set vShape = rng.InlineShapes.AddPicture(FileName:=image, LinkToFile:=False, SaveWithDocument:=True)
    
        '// Resize. LockAspectRation maintains the aspect
        With vShape
            .LockAspectRatio = True
            .Height = CentimetersToPoints(4)
        End With
    End If

    Exit Sub

ErrHandler:
    MsgBox Err.Description, vbExclamation

End Sub

Hi Tomasz. Yes it is possible for your OneDrive files to be encrypted and then synced to Office 365. This can happen via the sync tool or via a mapped network drive.

I believe that you can put in a request to Microsoft via your Office 365 management portal to have a document library restored from backups. suzanne s. barnhill say:

You should never turn off the option to "Prompt before saving Normal template." This is like taking the battery out of your smoke detector because the alarm keeps going off. Turning off the prompt doesn't prevent the template from being saved; it just allows Word to save it without asking for permission.

 

That said, it is not normal to be prompted to save a template that hasn't changed, and the fact that you're getting the prompt even with the option disabled is an indication that there is something wrong. There are an old KB article and an old assistance article that address the problem with the Normal template (there is a current version at http://support.microsoft.com/kb/918064), and, although asking about every template is a somewhat different (and much more annoying) problem, the advice in these articles is about the best you will find. Over the course of many years and many Word versions, the reported solution has almost invariably turned out to be finding and disabling an add-in of some kind that is causing the problem.

stefan blom say:
The option to "Prompt before saving Normal template" applies specifically to that template; you can't prevent Word from prompting you when changes have been made to a custom template.

Think of the prompt as a warning that something has changed—something that you may or may not want to keep (in other words, don't turn it off).

You should expect to be prompted if you are saving Building Blocks, key assignments, styles, or macros to a template. If you are not doing any of that, you should troubleshoot for (poorly designed) add-ins. In the Knowledge Base article at http://support.microsoft.com/kb/921541/en-us, take a look at the sections that refer to add-ins.
