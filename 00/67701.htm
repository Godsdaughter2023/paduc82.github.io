a. user say:

Hello,

As per the description, you are experiencing issue with Feed Formulation software PRO4L1 not working and giving error message: “comdlg32.ocx is missing” on the system.

I certainly understand your concern.

I would like to provide you with the information that Comdlg32.ocx is a type of OCX file associated with MSDN Development Platform U.S. developed by Microsoft for the Windows Operating System.

OCX files, such as comdlg32.ocx, are COM (Component Object Model) interface technologies often referred to as an ActiveX control. ActiveX controls are tiny programs, sometimes called "add-ons” that are used on websites and in software applications.

ActiveX controls are often encountered when using a web browser such as Internet Explorer, Firefox, or Chrome. If you open a web page embedded with a video in the Window Media (.WMV) file format, your web browser uses a built-in ActiveX control that allows you to view the video directly on the web page, rather than having to load that file separately in Windows Media Player.

Because comdlg32.ocx is an external file, it presents a big opportunity for errors or malicious file manipulation to occur. Improperly shutting down your PC or getting a virus infection could corrupt the comdlg32.ocx, which could create lead to ActiveX errors. When comdlg32.ocx becomes corrupt, it cannot be loaded properly by your application, and will result in an error message.

Other times, comdlg32.ocx file errors could be related to issues in the Windows registry. Broken DLL file references can prevent your OCX file from registering properly, giving you a comdlg32.ocx error. These broken registry keys can be as a result of a missing DLL file, moved DLL file, or an leftover DLL file reference in your Windows registry from an unsuccessful software installation or uninstallation.

More specifically, these comdlg32.ocx errors can be caused by:

  • Invalid or corrupt comdlg32.ocx-related registry entry.
  • Virus or malware infection which has corrupted the comdlg32.ocx file.
  • Microsoft hardware failure, such as a bad hard drive, which has corrupted the comdlg32.ocx file.
  • Another program overwrote the required version of comdlg32.ocx.
  • Another program maliciously or mistakenly deleted the comdlg32.ocx file.
  • Another program uninstalled the comdlg32.ocx file.

Manually Register Comdlg32.ocx Using Microsoft Register Server

When you install software that uses the comdlg32.ocx dependency, the software should automatically register the file for you. In some cases your OCX file may not register properly, and as a result, will provide a "comdlg32.ocx not registered" error. Fortunately, you can use a built-in utility called "Microsoft Register Server" (regsvr32.exe) to re-register your comdlg32.ocx file.

How to re-register comdlg32.ocx from an elevated command prompt (Windows 7, 8, and 10):

  1. Click the Start button.
  2. Type "command" in the search box... DO NOT hit ENTER yet!
  3. While holding CTRL-Shift on your keyboard, hit ENTER.
  4. You will be prompted with a permission dialog box.
  5. Click Yes.
  6. Type the following command: regsvr32 /u comdlg32.ocx.
  7. Hit ENTER. This will UN-REGISTER your file.
  8. Type the following command: regsvr32 /i comdlg32.ocx.
  9. Hit ENTER. This will RE-REGISTER your file.
  10. Close the command prompt window.
  11.  Re-start the program associated with comdlg32.ocx error.

kevin d souza say:

Hi,

 

 

This could be an user account corruption problem.  I would suggest you to create a new user for test purpose and install an app and then restart the computer and check if the settings and apps remain in the new user account.

 

http://windows.microsoft.com/en-in/windows/create-user-account#create-user-account=windows-8

 

If the issue does not appear in the new user account, then I would suggest you to move the data from the old user account to new user account.

 

Here's the procedure.

 

Note: you need to create a new user account first.

 

Enable Built Admin account.

a) Type Command prompt in the Search Charms.
b) Right click on Command prompt and select Run as Administrator
c) Type: net user administrator /active: yes

Note: Please follow the same steps and type net user administrator /active: no to disable admin account after the troubleshooting is done
After enabling Built-in admin, login to the admin account and  please try these steps to move the data from old user account to the new user account you have created:
 
a) Press Windows and x together and select Windows Explorer.
b) Click on View tab and Click on Options at the top right corner.
c) Click the View tab, and then click Show hidden files, folders, and drives.
d) Uncheck the box which states Hide protected operating system files.
e) Click Yes to confirm, and then click OK.
f) Locate the C:\Users\Old_Username folder, where C is the drive that Windows is installed on, and Old Username is the name of the profile you want to copy files from.
g) Select all of the files and folders in this folder, except the following files:
 
•Ntuser.dat
•Ntuser.dat.log
•Ntuser.ini
 
h) Right click and Select Copy.
i) Locate the C:\Users\New_Username folder, where C is the drive that Windows is installed on, and New Username is the name of the new user profile you created.
j) Right click inside the folder and select Paste.
k) Restart your PC and login to the new user account.
 
Once done troubleshooting, disable the Default Administrator account.
 
Hope this information is helpful. If you have any concerns, please let us know.
