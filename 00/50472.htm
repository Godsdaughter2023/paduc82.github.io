bill.p. say: Hi Stephen, I'm Graeme, an independent adviser here to help.

Your router will have assigned the drive an IP address, so you'll need to determine what that IP address is by accessing your router's administration panel.

Once you've found your external drive's IP address, you can type it into the address bar of File Explorer on your Windows 10 PC to access it.

In File Explorer, find This PC, and right-click somewhere in the window and you should see an option to "Add a network location". You can enter the IP address for your USB drive here, and it'll add it to This PC as an accessible drive.

papajohn... say:

What's Wrong with Movie Maker 2012?

In 2015 I'm hoping to spark more discussions about Movie Maker 2012... here on the Microsoft Community forum.  I'm kicking it off with two discussion threads, one about the things that need fixing and the other about the good features.

I'll updale this summary list as items are identifield.... post if you have something to add or to just comment. Check the other list of good stuff to see the flip side.

  1. It doesn't de-interlace video clips on Windows 8.1 as previous versions of Movie Maker do >> the work-arounds include setting the compatibility to Windows 7 or de-interlace them someplace else or with other software. >> resolved by updating the AMD graphics driver
  2. When heading to DVD Maker for standard video discs, it doesn't pass the widescreen tag along... as a result widescreen videos on the discs play as standard 4:3 >> use other DVD making software or other versions of movie Maker.
  3. It doesn't handle MPEG2 files without first converting >> use other software to convert
  4. Tech support is minimal or less...  >> there are bugs but you're on your own... bugs include the software corrupting the project (.wlmp) file.  These corruptions seem to fall into 3 categories:
    • Movie Maker writes deleted captions and source files into the file but that are still referenced in the saved project file.
    • Movie Maker incorrectly writes an outTime that occurs before the inTime resulting in a negative duration on a clip.
    • Movie Maker writes an empty file consisting of nothing but zero hex bytes (0x00)
  5. Text clips are buggy.... to the extent they can corrupt a project file >> with no tool to fix it that is available to users
  6. Preparing source files and saving movies is so CPU intensive that a computer can overheat >> limit multi-core CPUs by changing the affinity setting as needed
  7. Adds an unwanted black/gray outline on the final saved video (considerate guy)
  8. Audio transitions such as a fade out of a song as another fades in can't be easily added as they could in classic versions of Movie Maker
  9. All audio on the computer suddenly stops >> turn the Windows Audio service back on
  10. Help info specific to the software used to be included. The combination of more general help info, tool tips and external help resources could use a good review and change. 
  11. coming soon....  comment if you have one to suggest...

My hope is that listing and discussing issues in an open forum such as this will help users by giving them links to work-arounds and resolutions, along with suggestions about third party software and hardware needed to compliment Movie Maker.

devlin sco say:

Hi,

You may setup User Account Control to ask for the username and password though it will still depend on the user account type.

To change these settings, follow these steps:

To change the elevation prompt behavior for administrators

  1. Click Start, click Accessories, click Run, type secpol.msc in the Open box, and then click OK.

  2. From the Local Security Settings console tree, click Local Policies, and then click Security Options.

  3. Scroll down to and double-click User Account Control: Behavior of the elevation prompt for administrators.

  4. From the drop-down menu, select one of the following settings:

    • Elevate without prompting (tasks that request elevation will automatically run as elevated without prompting the administrator)
    • Prompt for credentials (this setting requires user name and password input before an application or task will run as elevated)
    • Prompt for consent (default setting for administrators)

  5. Click OK.

  6. Close the Local Security Settings window.

To change the elevation prompt behavior for standard users

  1. Click Start, click Accessories, click Run, type secpol.msc in the Open box, and then click OK.

  2. From the Local Security Settings console tree, click Local Policies, and then Security Options.

  3. Scroll down to and double-click User Account Control: Behavior of the elevation prompt for standard users.

  4. From the drop-down menu, select one of the following settings:

    • Automatically deny elevation requests (standard users will not be able to run programs that require elevation, and they will not be prompted)
    • Prompt for credentials (this setting requires user name and password input before an application or task will run as elevated, and it is the default for standard users)

  5. Click OK.

  6. Close the Local Security Settings window.

bootrec /FixMbr

bootrec /FixBoot

bootrec /ScanOs
bootrec /RebuildBcd

If you are able to get to a cmd window, and the above doesn't work,  you can also edit the bootloader using bcdedit - find the how to for that via Dr Google, it's simple enough to do, it's editing one line.....
