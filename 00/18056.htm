How do I convert a Access2010 32bit application to 64bit - Microsoft Community

wonderbarr say:

How do I convert a Access2010 32bit application to 64bit

I have created an Access 2010 application on a 32bit machine and saved it as an executable file but when I try to use it on a 64bit machine it won't open. How do I get it to work on a 64bit machine?
albert d. kallal say:

When you say executable, then this usually suggests you created a accDE.

Such an accDE should run + work just fine on BOTH an x32 and an x64 bit machine.

In fact, few machines today are x32.

However, the accDE will NOT work and NOT run if the user installed the x64 bit version of office.

The solution(s) in the above case are:

  • Don't use a compiled executable (accDE), but give the user an accDB. The accDB has source code, and Access x64 is thus smart enough to re-compile the code to x64 when you run/use that accDB file on Access x64

 

  • Don't use the x64 bit version of Access. For the most part, it is NOT recommend installing the x64 bit version of office/Access. The recommend install is to use office/Access x32 EVEN on an x64 box. The vast majority of machines are setup this way.

 

  • Last but not least: If they are committed to office x64, then you have to use the first suggestion above (use accDB). If you cannot use a accDB, then YOU MUST re-compile the accDB to an accDE WITH an x64 bit version of Access. So the OS choice of x32 or x64 does NOT matter here in a round about way. However, the choice of x32 or x64 Access MOST certainly matters. The x32 and x64 OS does not have to match office x32, but the version of Access and a accDE MUST match. If they are going to run x64 Access and you want to use an executable accDE, then you MUST re-compile your accDB with an x64 bit version of Access.

 

The best approach is to suggest using x32 office in all cases.  However, as noted, if such a choice is beyond your control, then you can use a accDB on EITHER Access (x32/x65).

 

However, to use a accDE withh Access x64, you must compile the accDE with Access x64 and there no other choice when using a accDE.

2. The "standard" TOC Field code looks something like this:

{ TOC \o "1-3" \h \z \u }

This part:

 \o "1-3" 

means "create entries for paragraphs with styles that are associated with outline levels 1 to 3"

But you can also specify a list of paragraph style names and "ToC levels" using \t. e.g.

{ TOC \o "1-3" \h \z \u \t "mystyle1,1,mystyle2,2mystyle3,3,mystyleA,1" }

Or you can just use \t and leave out \o
