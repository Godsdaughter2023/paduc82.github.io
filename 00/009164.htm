licensedengineer say:

Having searched for the problem throughout the Web, I applied the following fixes. I have not seen the problem since applying these fixes:

1. Turn off fast startup (Power-Options->Choose what the power button does).

2. Check the Advanced Settings on the power plan. PCI Express->Link State Management should be set to moderate power savings (my machines are all desktop workstations). The upgrade to Windows 10, for some reason, changed some of the machines to maximum power sacings.

3. Turn off Enhanced Pointer Precision under the mouse settings.

You mayneed to reboot after making the first two changes.  I did to be on the safe side.

Fixing the first two significantly reduced, but did not solve, the jumpy mouse. I needed to do all three to fix the problem.  My machines were very difficult to use.  They have run problem-free for almost a week now.

A few suggestions/points:

1.  If you are working in a corporate environment, please check with your IT department before making and changes.

2.  Make sure that your drivers are up-to-date.  I found some drivers that were not updated properly. Although it didn't solve the problem, there is no way to know if the problem would have been solved if I kept running with the old drivers. 

3.  If you don't know where the advanced power options are, you probably shouldn't be changing them. And if you don't know what the advanced power options are, you absolutely should not change any other options.
4.  Also, all of my machines are high-end machines with ample memory, good disk drives, gigabit NICs, etc.  I have no idea if these fixes will work on other machines.

anula d'souza say:

Hi,

 

As per the issue description you want to synchronize the contacts and calendar data from Outlook to AOL web-mail.

 

Outlook 2013 will synchronize all the folders\emails, contacts and calendar data only if you have an Exchange Active Sync account or Exchange account. There is no other option in Outlook to synchronize contacts and calendars from Outlook to webmail if the email account is configured as POP or IMAP.

 

Regarding synchronizing Outlook contacts and calendar items with web-mail using AOL sync software, Outlook plays a passive role in synchronization process. However, ensure that all the contacts and calendar items are present in Default data file present in Outlook and also create a new profile, open the PST file in new profile and then try to synchronize and check.

To create new profile:

Refer to the link mentioned which explains how to create new profile in Outlook:

http://support.microsoft.com/kb/829918

 

To open PST file in Outlook:

Open Outlook. Click on File< Open< Outlook Data File< browse to the location where the PST file is saved and click on open.

 

Set the data file as default:

Open Outlook, Click on File< Account< Data Files tab< highlight the data file which you want to sync with AOL web-mail and click on Set as Default. Close and open Outlook.

 

If the above steps do not fix the issue you may also get in touch with AOL technical support team for better suggestions on AOL sync software issues.

 

I hope the above information helps. Let us know if you need any further assistance.

Thank you.

kim stedt say:

This might be an old answer, but since I had the same problem just now I figured I could give you my solution.

In my case, another user had previously used the client machine with his credentials (and older version of Office). Somehow the registry locked permission to the previous user.

To get Office 365 installed (or, first installed with errors, then uninstalled and lastly installed successfully) I had to start regedit with psexec from sysinternal. Then change permissions in the registry tree from the old user to the new one and searched the registry for "ClickAndRun" and removed everything connected to Office.

After that my installationen went successfully through.
