don cuthbert say:

In Legacy mode, the disk is probably initialized as an MBR disk.  If the laptop has UEFI BIOS mode available, you could switch to that at some point, but it would mean wiping the hard disk.

Run diskpart at a Command Prompt.

diskpart

list disk

  • a gpt disk will show an asterisk * beside it
  --------  -------------  -------  -------  ---  ---
  Disk 0    Online          119 GB    11 GB        *
  Disk 1    Online         2794 GB     9 GB        *
  Disk 2    Online          931 GB      0 B        *

  • list volume
    • this will show all volumes and assigned drive letters
    • if any show with no letter, you can assign them a letter
      • select volume 1
      • assign
      • select volume 2
      • assign
      • etc

Your disk will probably just show a Recovery/System partition and drive C: partition (which might have letter D: but that does not matter, as long as you know what letter it is.  If C: is the Recovery partition and D: is where Windows is, the commands to fix would look like this:

bootsect /nt60 C: /force /mbr

bcdboot D:\Windows /s C: /f ALL

You might not even need to do the bcdboot command.  BootRec can add the Windows menu entry if it is missing, but it might also find things like Windows.old and offer to add that (which would not be useful).
