toddallcock say:

Windows CE is a different OS, running on different processors that Intel-based Windows PCs. Any virus that targets CE-based devices has to be written specifically for Windows CE; the malware that targets Windows PCs can't affect a CE-based device. 

 

Theoretically, CE is potentionally vulnerable, but pragmatically the number of Win CE devices in use is so insignificant it has never become a target. A few proof-of-concept viruses were created in the Windows Mobile days, but AFAIK, none ever made it "into the wild".

 

Typically today, use of Windows CE devices has been relegated to industrial/vertical market handheld equipment and thin client network terminals, neither of which are generally used to download executable software from unknown sources, so even if some nostagic virus writer decided to release a Windows CE-specific virus, I'm not sure how you'd stumble across it. These days it's hard enough to find legitimate Windows CE software! ;)

rohn007 say:
The number of times you can "re-install" on the same device is not limited.

The number of machines you can install it on concurrently, now that is a good question that I have not yet seen answered.  It depends entirely on the type of license you bought, either a "PKC" or "FPP" type license. 

PKC can only be re-installed on the first machine it was installed on. No transfer to other devices.

FPP may be installed on up to 2 machines at the same time, and those installations may be transferred to other devices as required.

We have no way of telling what license type you got.  If you still have the original purchase confirmation email it may be mentioned there.  If you bought an online download, there should be an online account setup for you and the license type should be listed there along with the product key.

You could also take a look at the Microsoft Office License Terms in your package. Go into the File menu > Help command. On the far right of the "backstage pane", under the office logo, near the very bottom there is a "Microsoft Software License Terms" hyperlink. Click on it and see if you can find Office University named explicitly anywhere.  I have not seen it mentioned in any of the online copies I could see, but the previous package called "Office Academic" was in the "Retail License Terms" section (making it a FPP license).  If you are lucky University will be there too. Please let us know if what you find.

The only other thing you could try is to do a phone activation.  When you call in, don't enter any info, just wait (a long time) until you are offered the option of talking to a "service" rep. Explain the problem. They should be able to tell you the license type.

If you do want to use a single cell for entry you must use VBA event code.

When you enter 500 or any number in a cell it will be multiplied by .5

Private Sub Worksheet_Change(ByVal Target As Excel.Range)
    If Target.Count > 1 Then Exit Sub
    If Not IsNumeric(Target.Value) _
            Or Target.Value = "" Then Exit Sub
    Application.EnableEvents = False
    With Target
        .Value = .Value * 0.5
    End With
    Application.EnableEvents = True
End Sub

neha singh msft say:

Hi Joseph_N_

SharePoint groups enable you to control access for sets of users instead of individual users. SharePoint groups are usually composed of many individual users. They can also hold Windows Active Directory security groups (created in Office 365) or can be a combination of individual users and security groups.

Users added to a Security Group do not automatically gets the permission to a SharePoint site. You still must add the Security Group to a group in People and Groups in SharePoint online site to give the permission to the site.

You can either create a Security Group and then add this group to People and Groups in SharePoint online site or you can add the users directly to group created in People and Groups.

Please check this article about SharePoint Groups for more information: Default SharePoint Groups
