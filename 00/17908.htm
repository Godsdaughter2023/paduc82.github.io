How do i copy multiple excel files using VBA ?

What i need is;

-Open all excel files in a certain folder.

-Select sheet 2 A3:T12 just copy values. Paste to my main workbook, to say A2.

-Select the next free cell which would be A12 and so on until all files are copied.

-Also close the copied excel files down afterwards.

 

All help is greatly appreciated.

hansv mvp say:

Try this macro. The destination sheet should be the active sheet when you run the macro.

 

Sub ProcessFiles()
    Dim strPath As String
    Dim strFile As String
    Dim wbkSrc As Workbook
    Dim wshSrc As Worksheet
    Dim wshTrg As Worksheet
    Dim lngRow As Long
    With Application.FileDialog(4) ' msoFileDialogFolderPicker
        If .Show Then
            strPath = .SelectedItems(1)
        Else
            MsgBox "You didn't select a folder", vbExclamation
            Exit Sub
        End If
    End With
    Application.ScreenUpdating = False
    If Right(strPath, 1) <> "\" Then
        strPath = strPath & "\"
    End If
    Set wshTrg = ActiveSheet
    lngRow = 2
    strFile = Dir(strPath & "*.xls*")
    Do While strFile <> ""
        Set wbkSrc = Workbooks.Open(strPath & strFile)
        Set wshSrc = wbkSrc.Worksheets(2)
        wshSrc.Range("A3:T12").Copy
        wshTrg.Range("A" & lngRow).PasteSpecial xlPasteValues
        lngRow = lngRow + 10
        wbkSrc.Close SaveChanges:=False
        strFile = Dir
    Loop
    Application.CutCopyMode = False
    Application.ScreenUpdating = True
End Sub

Ctrl + V not working within Word 2013 - Microsoft Community

margokw say:

Ctrl + V not working within Word 2013

Recently (as of Monday), I have noticed that all Word documents that I open (including new, blank documents) now display a notification that says macros have been disabled. I know these documents do not have macros because they never displayed this notification before and I have not changed my Trust Center settings (it's set to not allow macros and show a notification).

If I ignore the notification and keep macros disabled, I cannot use Ctrl + V to paste within the document (though I can use the normal Paste button). If I enable macros by clicking the "Enable Content" button, I can then use Ctrl + V to paste. Ctrl + V seems to be the only keyboard shortcut that is affected, though I don't use many of these. So far I can confirm that Ctrl + C and Ctrl + Z work.

This is the testing/troubleshooting I have done so far, before you ask:

  • Ctrl + V works outside of Word, including in PowerPoint and Excel (neither of those programs has the macros alert, either).
  • Ctrl + C has been confirmed to work as I can copy within Word and then use the paste button to paste it or I can use Ctrl + V in another program and see what I copied.
  • Changing my Trust Center settings to allow macros DOES solve this issue, but that's a security risk and I've never had this problem previously so I don't consider this to be an actual fix for the issue and would appreciate other suggestions for solving it.
  • I have checked for and applied all current updates, including a restart to ensure they are fully applied.
  • I'm using Windows 7 Professional with Office 2013 (for Business, I think)

stefan blom say:
Exit Word and rename the Normal.dotm template. The next time you restart Word, see if the message has gone away. Chances are that this will make the paste command work properly as well.

margokw say: Do I need to rename it anything in particular, or do I just keep the same name?

stefan blom say:
A useful name is OldNormal.dotm. That way, you will remember what it is if you need to restore it in the future. Of course, if renaming it fixes the problem you can just delete OldNormal.dotm after a day or two.

margokw say:

I ended up trying both (renaming it the same, as well as renaming it something different) and that didn't change either issue.

Any other ideas?

I really appreciate your reply and trying to help!

stefan blom say:
OK, the next step is to troubleshoot COM and Startup add-ins. See https://support.microsoft.com/en-us/help/921541/how-to-troubleshoot-problems-that-occur-when-you-start-or-use-word.
