cliffgu_msft5984 say:

hi bruce,

no, the amount of personal site storage is counted separately, and does not add to or subtract the overall storage allocation for a tenant. it will not affect the amount of all site collection’s storages.

if you set the storage management option to "automatic”, it is not necessary for you to manage every site collection’s storage and they are all allocated by office 365 manually. the maximum storage of every site collection is up to 1tb (but it is limited by the overall storage in your tenant. if the overall storage in your tenant is lower than1tb, the maximum storage of every site collection can’t reach 1 tb. if you need more storages, you need to buy them.). it is obvious that the storage quota button disappears when you do that and you can’t set a new site collection’s storage when you create it.

 

if you set the storage management option to "manual”, you can set every site collection’s storage limit and manage them by yourself. but the maximum storage of every site collections is also up to 1tb. it is obvious that the storage quota button appears when you do that and you can set a new site collection’s storage when you create it.

maor bracha say:

Hi Samir,

Yes, you can use "Set-AdUser <SamAccountName> -Add @{ProxyAddresses=<New Address>}"

Keep in mind that the proxy addresses attribute has prefixes- smtp, x500, etc... And it's case sensitive- capital SMTP: for primary, lowercase smtp: for secondary.

So, for instance if you would want to add a secondary email address you should add "smtp:*** Email address is removed for privacy ***" .

If you want to change the primary as well, you would have to change the current primary to lowercase "smtp:" and add a new primary with the capital "SMTP;" prefix.

Note: if changing primary, I would usually change the mail attribute to match.

Cheers,

Maor

The error "Microsoft Account does not exist" usually means:

The person is logging in using an Alias. If the user renamed the affected email address into a new account, the original account will become an Alias. Thus logging in using the original account (Alias) will trigger the system to prompt the user that it does not exist. The solution is to log in using the new account.

 The person mistyped the email address. Retyped the correct email address.

 The person did not access the account (WITH A BROWSER) within 365 days. The account will be deleted permanently in the system due to inactivity. You may recreate the account as a new account if it is still available.
