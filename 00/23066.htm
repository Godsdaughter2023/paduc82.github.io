joeu2004 say:

Chris wrote:

One of my cells contains the number "11982:46:07". This cell is formatted as [h]:mm:ss, but it is being ignored by my sum formula. The number is left-aligned, indicating it is formatted as text and not a number, though it explicitly is formatted as such. Any idea there?

The maximum data entry in h:m:s form is 9999:59:59, an arbitrary limitation.  Anything larger is not recognized as time.  So it is interpreted as text.

But Excel can calculate and display time in h:m:s form up to 71003183:59:59 because 12/31/9999 23:59:59 is the maximum date/time.

So you can enter time greater than 9999 hours as a date (e.g. 11982:46:07 is 5/13/1901 6:46:07) or as a formula (e.g =11982/24 + "0:46:07").

andreas killer say:
You can convert the text values to times using this formula
=IF(ISTEXT(A1),LEFT(A1,SEARCH(":",A1)-1)/24+TIMEVALUE("0"&MID(A1,SEARCH(":",A1),99)),A1)

Where A1 contains a text like "11982:46:07" or a real time like 1503:58:58

Andreas.

FYI, no need to use TIMEVALUE.  The following work just as well:

pc-learner say:

I don't mean to be picky, but this thread is getting kind of hard to follow because you keep responding to your own posts, and they're only identified by date. Since you're not quoting what you're replying to, it can be difficult to figure out what matches with what.

To clarify terminology a little, a disk usually means a disk drive. A disk drive can be divided into a number of partitions. Each partition can be formatted differently and used for different purposes by different operating systems. Unless a disk drive has only one partition that spans the whole disk, disk does not equal partition.

A basic disk is limited in the number of partitions it can have. It can have up to 4 primary partitions (each may have its own drive letter in Windows), or 3 primary partitions (each may have its own drive letter in Windows) and one extended partition divided into up to as many logical NTFS drives (each with its own drive letter) as you have space for. I'm pretty sure only a primary partition can be used for booting (i.e. be an active partition).

A dynamic disk is much more flexible than a basic disk and supports many more features than a basic disk, including removing the limit on partitions.

The whole topic can get quite complex - see https://msdn.microsoft.com/en-us/library/windows/desktop/aa363785%28v=vs.85%29.aspx?f=255&MSPPError=-2147217396

It would, with some trouble, be possible for you to change your disk back to a basic disk with 3 primary partitions (System Reserved, C:, and whichever one of the others you want) and one extended partition with 3 logical drives for the remaining ones.

That's only worth it if using a dynamic disk really is causing you insurmountable problems.
