VARIOUS UNINSTALL METHODS FOR OFFICE VIA: CONTROL PANEL / FIX IT / TROUBLE SHOOTER / MANUALLY

SIMPLE/ “SOFT” UNINSTALL SAVES CUSTOMIZATION SETTINGS, IN THEORY!
Since day one, what became Office applications have had quality problems. So MS designed them to be “user friendly”. You can do a simple uninstall, from the Control Panel and the process leaves behind your configuration files, like Normal.DOTM for Word, and Registry Entries. The developers came up with a 4 step “fast and dirty” trouble shooting process for themselves and for users:
. Step 1, start the application in Safe Mode to disable addins.
. Step 2, do an Office Repair
. Step 3, un-install through the control panel then re-install The re-install picks up the exiting configuration information to save the user the effort of having to re-setup their installation
. Step 4, do a full uninstall-reinstall using the appropriate “fix-it” script. This wipes all configuration information, on the chance the problem in is in the configuration files/registry.

This “soft” uninstall also plays a part in “upgrades”. During an upgrade do a simple unstall either manually or through the upgrade installer and the upgraded installation will pick up the existing configuration information making for a “painless” User Experience.
