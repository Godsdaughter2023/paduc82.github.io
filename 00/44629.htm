ronnie vernon say:

Hi Elcron

 

This is a subject that requires a long explanation to really understand what is happening.

 

The issue is focused on the use of Hard Links in Windows. Hard links were developed and first used in UNIX, but were first implemented in windows on Windows NT4. However they were not used extensively until Windows Vista and continuing in Windows 7.

 

Every physical file on the Windows System is called a file object. The file object occupies a certain amount of space, in a certain location on the hard drive. Each file also has a Reference Count that is stored in the File Object. Each file has a reference count of 1 (one) when it is created.

 

A hard link is a way that you can create another name (alias) or the same name for the same file object. For each hard link that is created for the file object, the Reference count is increased by 1. So a file object that has 5 hard links will have a Reference Count of 6 (the original file name + 5 hard links). Another way to explain this would be that every name for a file object is a Hard Link, including the original name that was used when the file object was created.

 

When a file object is opened and changed from any Hard Link to that file, that change is made to the file object and will be reflected regardless of which hard link is used to open the file object.

 

A file object cannot be deleted until the Reference Count reaches 0 (zero).

 

Since all hard links refer to the same file object, when you look at the Properties of a hard link, it will show the Size of the file and other properties of the file object.

 

The main function of Winsxs is the Servicing Component for windows. If any of the thousands of windows system and runtime files need to be updated this will be done to the file object from within the Winsxs folder in one single operation. Since all of the hard links use that same file object they are all updated. In previous versions of windows, if a file was needed in different locations, when that file was updated, it required searching for every instance of that file and each instance found required a separate operation for the update.

 

In previous versions of Windows, when you installed a software program, the install included a copy of all of the many windows Runtime, and other files (.DLL, etc.) that were required for the program to work with the operating system. In Windows 7, the software installation can simply create a hard link to the latest version of the .dll file that is already in the Winsxs folder.

 

On the subject of disk space:

 

Remember that every file on the entire Windows 7 original installation disk is contained in the Winsxs folder. During the installation, any system files that are needed in other locations, including the Windows, System32, and other locations are projected as Hard Links to those locations. This is why the Winsxs folder has an initial size of 3 to 5GB.

 

When you look at the Properties of any hard link, you will see the size of the file object.

 

There is not any file manager anywhere, including Windows Explorer, that I'm aware of, that can enumerate hard links and show how much disk space would be available, if all of the hard links were ignored. Most of the mainline operating systems use hard links including Windows, MAC, and Linux. They all treat hard links the same way.

 

Even if a system were able to enumerate all of the hard links, it would simply be a redundant operation.

 

The main reason is that each hard link to a system file is important to the operation of some component or program on the operating system. If you do delete a hard link, it will free up the space for a single instance of the linked file object, but it will also break something.

 

Also, disk space IS cheap. It wasn't too long ago that the 64GB SSD you are using would have cost you around 1500+ USD. It also will not be too much longer when you will be able to purchase a 160GB SSD for less than 100 USD.

 

As for your 17GB Winsxs folder, I have no idea why some systems have this problem? I have 3 Windows systems running. 2 have had Windows 7 installed since August 2009 and the other since January 2011. None of them have a Winsxs folder that has ever gone over 7GB.

 

OK, if you read this far, you deserve a real answer. It is No. :)
