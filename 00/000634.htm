ken sheridan say: This is a road to Dublin question.  "In Progress, "On Hold", "Completed", and "Cancelled" are data.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1).  This requires that all data be stored as values at column positions in rows in tables, and in no other way.  By including these values in a value list as the RowSource property of a control data is, on the contrary, being encoded as part of the object definition.

The correct model for this is to store the values in a table with the values 1 to 4 in the primary key column, and the text values in a text column.  An enforced relationship should be created between the form's underlying table and this new table.  You can then simply include the new table, along with the current table, in a query as the form's RecordSource, ordering the query by the text column in the new table.
