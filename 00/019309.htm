creating a print icon on a Word Document - Microsoft Community

michaelbenson42 say:

creating a print icon on a Word Document

I have created a Word document that requires the user to complete several fields, print then fax only one page.  I would like to create a PRINT icon on that one page that will automatically print that page only.
doug robbins - mvp office apps & services (word) say:

With the selection at the location in the document where you want the "icon", go to the Developer tab and then click on the Legacy Tools drop down in the Controls section of the ribbon and then click on the Command Button (ActiveX Control) which will cause a command button to be inserted into your document (and the Design Mode will have been invoked).  Right click on the Command Button and then select View Code and in the Visual Basic Editor that opens, insert the following command between the Private Sub CommandButton#_Click() and the End Sub (There will be a number appear inplace of the #)

Private Sub CommandButton#_Click()
ActiveDocument.PrintOut Range:=wdPrintCurrentPage
End Sub

Go back to the document and, while still in Design Mode, right click on the button and select Properties and in the Properties dialog, change the Caption to "Print" (or, if you want to get fancy and you have an image of a Printer, you can use the Picture attribute to insert a picture onto the Command Button.

After exiting from the Properties dialog, click on Design Mode in the Controls section of the Developer tab to exit that mode.

jay freedman say:
There are two kinds of AutoCorrect entries, "Plain text" and "Formatted text", indicated by the option buttons in the dialog:



The formatted ones are stored in the Normal.dotm template (regardless of what template you have open). The plain text ones are stored in MSO*.ACL files, where the MSO is followed by a numerical language identifier, depending on the proofing language applied to the text when the entry was created. In neither case are the entries stored in your custom template.

There's more detailed information in http://www.word.mvps.org/FAQs/Customization/ExportAutocorrect.htm. Although the article is very old, most of it is still accurate.

The sort of classification you want might be available with the AutoCorrect manager offered at http://www.clockworksoftware.com.au/Products/AutoCorrectManager.aspx. I haven't used it, so I don't know how easy it might be.
