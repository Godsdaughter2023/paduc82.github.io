rob koch say: Unfortunately, once malware has embedded itself deep into a system it is sometimes nearly impossible for any anti-malware program to remove them.  This is why there are so many 'Malware Removal' forums and specialty removal tools that require skilled help to perform removal.

In fact, most true security professionals will tell you that once a PC is as badly infected as you are implying, it can never be trusted without a complete wipe of the hard drive and re-installation of the operating system and programs from the ground up.  Even some Malware Removal experts come to this conclusion in cases where the specific malware are either deeply embedded or known to be difficult to remove completely.

Programs such as Microsoft Security Essentails are really designed primarily for prevention and so must be installed before an attack takes place to perform at their best.  This is partially because some malware are designed with the ability to disable many of the most common anti-malware so performing a successful installation and clean-up afteer the fact may be nearly impossible.  This is why the special tools exist and even these sometimes require using 'clean boot' techniques, where a special operating system disk (CD) is used to boot the PC and then the manual removal tools are used to remove the most difficult infections.

Even many of the specialty manual tools can be defeated if the malware is already present and is aware of the particular tools that are being used.  They can simply kill the installer or the main executable process of the anti-malware program before it can ever detect and remove the malware itself.

Rob

rob koch say: Though virtual machines are great for testing of normal client and even some server software, they are not exactly the same as a normal PC, especially at the deep levels that an anti-malware application such as Microsoft Security Essentials must operate.  In fact, I'd expect that many of the virtualization tricks used to redirect requests from the client VM to the Host OS might appear to be malware like activity similar to a rootkit operating behind the OS, and thus triggering the anti-malware detections or heuristics modules.

Also, since a virtual machine doesn't actually have control of the physical PC, which is what it expects, it will obviously not operate at full performance.  This is especially true due to the rootkit capabilities within MSE, since these are likely to be in overdrive as I mentioned above.

Many Malware Removal forum helpers with experience removing malware have often mentioned that even some malware doesn't operate as expected on a virtual machine, again because the software isn't actually getting the low level access it expects.  For this reason it's not possible to guarantee the validity of any results in a virtual environment and as I recall, someone from Microsoft has already mentioned here that operation in a VM is not officially supported.

So though your testing of previous security software may have succeeded in a virtual environment, this type of testing is invalid for MSE.

Rob

New ways to share presentations in 2010

PowerPoint 2010 has two new ways to share your presentations with others remotely: You can broadcast them in real time over the Web or share them on a CD or DVD. Click the File tab and select Save & Send in the left pane to get to these options.

To share a live presentation over the Web, click Broadcast Slide Show and then click the Broadcast Slide Show button that appears. (Yes, it's redundant, but that's the way it works.) When you do this, you're connected to Microsoft's free PowerPoint Broadcast Service. (You'll need a Windows Live ID to make it work.)

With PowerPoint 2010, you can share presentations in real time over the Web.
Click to view larger image.

Click the Start Broadcast button and a link is created to the presentation; you can send that link to other people who can then click it and watch your live presentation streamed over the Internet in any Web browser. If you already use a different broadcast service, such as your company's internal SharePoint site, you can switch to that one instead by clicking the Change Broadcast Service button.

You can also turn your presentation into a video that you can distribute on CD, DVD or via the Web. To do that, go to the Backstage Save & Send screen and click Create a Video in the File Types area. You'll see options for having the video formatted optimally for different devices -- for example, computer monitors and high-definition displays, DVDs and the Internet, or on portable devices such as the Zune and smartphones.

When you've made your selections, click the Create Video button, choose a location to save the video to and click the Save button. The video will be saved in the Windows Media Video (.wmv) format and will be playable on any device that can play .wmv files. You can copy the .wmv file to a CD or DVD, send it via email, save it on the Web and so on.

If you prefer instead to distribute the presentation as a self-running CD, click Package Presentation for CD in the File Types area. Then click the Package for CD button that appears on the right, give the CD a name, click Copy to CD and follow the instructions for burning it to a CD.
