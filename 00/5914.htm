ken sheridan say: What you have to do in a case like this is filter the parent form on all values of its primary key which are represented in the corresponding foreign key column in rows in the referencing table on which the subform is based where the relevant column in the referencing table also contains the value or values on which you wish the results to be restricted.

You'll find an example in the file StudentLogzip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you open the Students form in this demo you'll see it includes a 'Filter by Course' button.  This opens a dialogue form containing a multi-select list box from which one or more courses can be selected.  Clicking the 'Confirm' button in the dialogue form filters Students form with the following code:

Private Sub cmdConfirm_Click()

    Const MESSAGETEXT = "No courses selected."
    Dim strCourseList As String
    Dim varItem As Variant
    
    With Me.lstCourses
        If .ItemsSelected.Count > 0 Then
            For Each varItem In .ItemsSelected
                strCourseList = strCourseList & "," & .ItemData(varItem)
            Next varItem
            strCourseList = Mid(strCourseList, 2)
            Forms("frmStudents").Filter = "StudentID In" & _
                "(Select StudentID From CourseRegistrations " & _
                "WHERE CourseID In(" & strCourseList & "))"
            Forms("frmStudents").FilterOn = True
        Else
            MsgBox MESSAGETEXT, vbExclamation, "Warning"
        End If
    End With
    
    DoCmd.Close acForm, Me.Name
    
End Sub

jimbergquist say:

Limits on Setting Excel Histogram Intervals

In a Excel histogram chart, overflow and underflow bin settings are ignored if they exceed the range of the data series. This makes it impossible to clean up the bounds for the intervals. For example, if the minimum value for the data is -15.97 one cannot change the lower limit to -16.00. In such cases the user loses control over specification of the intervals. This makes it difficult to compare observed counts with the expected values for a desired set of intervals, especially if one of the end intervals is empty. The chart becomes unbalanced and Excel's utility is impaired. Would it be possible to fix this in a future upgrade?
