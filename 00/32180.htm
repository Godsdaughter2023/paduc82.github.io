a_user_here say:

The "administrator" (or root on Unix/Linux) is often misunderstood.  People talk about it being "god" and that you are suppose to be able to do whatever you like with that account.  This is not true and never has been.  The administrator is just restricted by permissions as any other account.  The main difference is that an administrator can do things like take ownership and change permissions.  If I have a file and the permissions are set so that the administrators don't have access to it, that is what happens, up until the Administrator changes the permissions.

You ask why such as system should exist that the Administrators group doesn't have access to change anything they want.  Well there are many accounts/groups used on a machine, and they all have their purposes, and as such there might be cases where they need certain permissions and the Administrators don't, and this is all about security.  For instance Windows purposely sets up permissions where the Administrators can't change things in Program Files.  Instead a "Trusted Installer" is what does most of the changes here.  This is to prevent viruses from using your account to install things.

Now on to why you being in the Administrators group doesn't always give you these permissions.

You can sort of chock this up to the abuse of such privileges by the users and Microsoft for that matter.

If you were to look at the recommendations for security on Unix/Linux it would say that you should only use the root (which is the same as the administrators) account when the extra permissions are necessary.  And the same should have applied to Windows, but from the get go, the one and only user was put in the Administrators group and it was used for everything.  Clearly a security risk and that was proven when the viruses took advantage of it.

So Microsoft needed a change in policy to combat this bad behavior/policy.  So starting in Vista they came up with UAC.  And what that means is that by default you are running with your normal non administrator permissions.  It isn't until you do something and causes the program to ask for administrator privileges, that you actually get those permissions.  And I might add it is not an all or none thing.  A program can ask for only one type of administrator privilege that it needs.

This can be illustrated like this.

Create a folder.  Put some files in it.  Go in to the permissions for the folder and change the owner to Administrators, and remove every other user/group other than Administrators from that folder and below.

Now do Win Key + X -> Command Prompt

Now do: dir c:\FOLDER_YOU_CREATED

It will come back with File Not Found

Now do: Win Key + X -> Command Prompt (Admin)

Now do: dir c:\FOLDER_YOU_CREATED

It will show the files in the folder.

In the first case you were running with the permissions that your user account has outside of the permissions it would have in the Administrators group.

So the bottom line is that there is no way for you to just blank change the permissions on all the files on your computer and then just have access to them based on that.  It is up to the programs you are using to switch to ask for these privileges, in some form.  Like when you see the shield icon or if you use Run as Administrator.  BTW a program can purposely turn off Administrator privileges.  Explorer seems to do that when it first starts.  So if for instance started an administrator command prompt and typed: Explorer

The Explorer that came up would not be in administrator mode.

There are a lot of programs that will not ask for elevated permissions, for instances Notepad.  So if you want it to have those permissions you would have to start it with Run As Administrator, or set it in the Compatibility tab of the permissions.  Of course what you would be doing is trying to defeat the very changes Microsoft put in to protect you from threats like viruses.
