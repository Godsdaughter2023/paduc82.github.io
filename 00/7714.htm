john w. vinson say:

Scott's suggestion to use IN() is right on the mark; do so!

But to explain why your current query isn't working, you need to see how OR and AND work in criteria expressions. It's sort of like how + and * work in arithmetic: just as 2 + 3 * 5 is 17 (calculate the 3*5 first, then add 2 to the result), so AND is applied first, and that result is used in the OR expression. To force the order of operation you must use parentheses on the <<<< lines:

PARAMETERS [Enter Start Date (mm/dd/yyyy):] DateTime, [Enter End date (mm/dd/yyyy):] DateTime;

SELECT COUNT(CustomerName)/

    (SELECT COUNT(CustomerName)       

     FROM Untitled

     WHERE (CustomerName = ('MERCURY PLASTICS OF' OR 'MERCURY PLASTICS, IN')   <<<<<

     AND RptDateStart = [Enter Start Date (mm/dd/yyyy):]

     AND RptDateEnd = [Enter End date (mm/dd/yyyy):])*100 AS Percentage

FROM Untitled

WHERE CustomerName = ('MERCURY PLASTICS OF' OR 'MERCURY PLASTICS, IN')    <<<<<

AND [Profitability$]< 0

AND RptDateStart = [Enter Start Date (mm/dd/yyyy):] 

AND RptDateEnd = [Enter End date (mm/dd/yyyy):]

and similarly for the UNION. But the IN() clause is clearer and probably more efficient so go with it.

Depending on the data you might also be able to use

WHERE CustomerName LIKE "MERCURY PLASTICS*"

to pick up both names.

Security Essentials

In another thread,  truespace  reported that removing all registry keys related to Microsoft Security Essentials manually and disabling the startup service from services. While this worked, I suggest that you back up your data files and do a reset of Windows 10 which will turn your upgrade into a clean install. a. do a reset of Windows 10, which will remove all installed programs and data and will leave you with what will in essence be a clean Windows 10 installation with no programs installed.

b. roll back to Windows 7, remove MSE, perform the upgrade to Windows 10 a second time

c.  manually rip out MSE by editing the registry, stopping services while running in Safe Mode, and possibly removing leftover program files manually.

d. not really advice -- wait for Microsoft to provide a fix and risk that your PC is unprotected for a very long time while waiting for a fix that may never appear.

I think that the most palatable solution is b.

julie fka profjulie say:
Windows Phone does not sync Outlook or Office related data directly between a computer.  It syncs that data from your online Microsoft account, and you'll set up your computer to sync this data to the same Microsoft account.  The Microsoft account could be considered to be the "server" of your data, with the computer and phone "clients" that receive data from the "server."

Microsoft Office Professional Plus 2013 has options that allow you to set up a direct sync account within Outlook that will be used to sync your Contacts, Calendar and Tasks automatically to your Microsoft account - You'll set this up under Outlook's Accounts options.  You can review this article I wrote to learn how to set up and Sync Outlook 2013 to a Microsoft Account.

There is a SkyDrive app that you can install to your computer that will sync your Office related files automatically between your computer and your Microsoft-related SkyDrive account. (If your computer is running Windows 8.1 - you don't need this app since it is incorporated into the OS).

Email that you currently receive via a POP3 account (even though you may have used this account as a Windows Live login and use it as the Microsoft account on your phone) will need to be set up on the phone as a POP3 account and not as a Microsoft account.  If you are using a POP3 account as your Microsoft login, you can set your Microsoft account up to retrieve your email from the POP3 servers and then the email will sync via the Microsoft account settings and you won't need to use a separate POP3 account on the phone.
