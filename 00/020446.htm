There is no "VBA voodoo". That attitude will not help you. VBA is a very logical language you just haven't studied / mastered yet.

Creating an Index of names with sub-entries - Microsoft Community

rony2rf say:

Creating an Index of names with sub-entries

I want to create an index that has the surname followed with the given names with page numbers. To do this I must mark the name then mark the given name and move it to the Subentry field. Does anyone know of a macro to mark and move the given name to the subentry field?

jay freedman say:
This is not a trivial problem, because there is no way for Word, or a macro in Word, or any automated process to determine which part of a highlighted name is the surname and which part is the given name. Consider these names from history:

Susan B. Anthony
Ludwig van Beethoven
Johannes Diderik van der Waals
Charles-Camille Saint-Saëns
Mao Zedong

Any rule you might define to separate the names correctly will fail for at least one of them. The only way you can work around that is to manually replace all the spaces with nonbreaking spaces except for the one between that person's given name and surname, using whatever knowledge you have about how that person's family and culture define them.

Once you've done that, you can write a macro -- or ask someone else to write a macro -- to insert the proper XE fields in the document. The field code should look like this:

{ XE "Flintstone:Fred T." }

where the surname is first, then a colon, and then the given name. The macro doesn't have to use the Mark Entry dialog box (doing so would make the macro unnecessarily complicated).

You cannot run a macro from inside the dialog box -- Word just doesn't work that way.
