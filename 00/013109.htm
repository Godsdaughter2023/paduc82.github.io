karl timmermans [outlook mvp 2012-2018] say:

First, the answer/recommendation that you are going to get from many is that you shouldn't really do a PST2PST import. The suggestion is going to be to copy the PST file from each of your respective machines (where the PST files originated from) - copy them to your 2010 system, open the PST file in O'2010 and copy or move the contacts to the folder where you want those contacts to reside.

The description of your problem is somewhat fuzzy (at least for me at the moment) so this response will treat the export and import separately and you can extract the portion(s) that apply to your scenario.

In terms of "exporting" - the PST file name that will be used to create the output PST file is set by you but Outlook will use the last path/filename used as a default. Don't pay attention to that and you're not going to know where the data went. If that is indeed what happened, then re-run the export and take note of the path/file name that is shown for the output pst file - it should be the same as the last time it was run.

In terms of PST2PST contact importing, if the source PST file indeed had the contacts you required from the PST2PST export, then any issues you encountered during the PST2PST import is likely simply related to the wrong options having been selected (it doesn't take much to fly by these things resulting with incorrect settings).

When importing from a PST file - you have one of 2 options (after your source PST file and corresponding source folder have been selected)
1) "Import items into current folder"
      *** that means that you need to have the target contact folder selected before you start
            the import process
2) "Import items into the same folder in..." --> The PST file you select right underneath that option

Given your description, one of the above options was not set in a way that would result in the contact items being imported to where you expect(ed) them to be (again assuming that the exported contacts existed to begin with).
