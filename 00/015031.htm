WORD 2013 Rounding Top and Bottom margin settings up - Microsoft Community

denisevelarde say:

WORD 2013 Rounding Top and Bottom margin settings up

I am trying to set a WORD document top and bottom margin to .4375 and Word rounds it up to .44 - which is causing my labels to print incorrectly - any way to stop this?

Denise

jay freedman say:

All measurements in Word dialogs are rounded to at most two decimal places for display purposes, but the entire entry is retained internally. You can see this by changing the option (in the File > Options > Advanced page) to display measurements in points.

There are 72 points per inch, so 0.4375 inch is the same as 31.5 points, while 0.44 inch is 31.68 points.

When the measurement display is set to points, you can enter inch measurements by typing a double-quote mark after the number. Go to the custom margins dialog and enter 0.4375" in the top margin box, and click OK. Reopen the dialog, and you'll see 31.5 pt in the box, not 31.7 pt (again rounded by Word).

Even if Word was rounding up the internal measurement, I doubt that a difference of two-and-a-half thousands of an inch (or 0.2 points) would be visible without a magnifier.

The implication of all that is that Word's treatment of your measurement isn't the cause of the label misalignment. It may be slippage of the paper-moving rollers in the printer, or possibly some error in the printer driver. Can you send the print job to a different printer?

Macro for separate widow/orphan control? - Microsoft Community

blakers_644 say:

Macro for separate widow/orphan control?

Unlike some word processors, Word offers no way to control widows and orphans separately. What I would like to do is prevent widows (paragraph ending line on new page), but not orphans (paragraph opening line). Has anybody created, or does anybody know of, a macro which will accomplish this?
macropod say:

Try:

Sub Demo()
Application.ScreenUpdating = False
Dim Rng As Range, i As Long
With ActiveDocument
  For i = 2 To .ComputeStatistics(wdStatisticPages)
    Set Rng = .GoTo(What:=wdGoToPage, Name:=i)
    With Rng
      .Paragraphs.First.WidowControl = False
      If .Characters.First.Information(wdHorizontalPositionRelativeToPage) = _
        .Paragraphs.First.Range.Characters.Last.Information(wdVerticalPositionRelativeToPage) Then
        .Paragraphs.First.WidowControl = True
      End If
    End With
  Next
End With
Set Rng = Nothing
Application.ScreenUpdating = True
End Sub

Do be aware that opening a document on a computer using a different active printer driver may cause Word to change the document layout, in which case the macro would need to be re-run to get the correct result for that printer.
