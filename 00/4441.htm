markragan say:

Hi Lawrence

Didn't solve this but I have a work-a-round.

It seems that Windows 10 is re-starting outlook whenever it is running at shutdown (Which in my case it always is).

You can ....

1) Remove the shortcut in "startup" and let Windows manage the startup. The only issue here is if outlook is NOT active at shutdown, then Windows will not re-start it.

2) you can change the shortcut to add the "/recycle" switch 

    "C:\Program Files (x86)\.......\OUTLOOK.EXE""  /recycle"

   This will make sure that if outlook is Active BEFORE this instance, then it will open in the SAME instance.

   The issues with this are that you will need to insert a sleep for a few seconds to make sure that this is the 2nd instance because if this starts before the Windows restarted one, then you will still get two.

   The 2nd issue is that even when this is working the 2nd instance will bring outlook to the front which is a bit annoying.

3) MY final solution was to let Windows start this itself, and I have created a VB script which will fire off 30 seconds after boot, it will check if outlook is running, and if NOT it will then start an instance.

This works a treat for me.

Happy to share the VB script if it helps.

jay freedman say:

Yes, you can do a wildcard search for a missing full stop at the end of a paragraph. Use the search expression

    [!.]^13

and either Find Next or Reading Highlight. This search will match a paragraph mark preceded by any character other than a full stop. It will, of course, match a paragraph mark preceded by a space character, a question mark or exclamation mark, or a footnote/endnote reference, so you'll have to examine each match.
