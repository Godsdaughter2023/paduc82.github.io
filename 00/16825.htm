shane devenshire say:

Record a macro which applies the four conditonal formats to a cell.  Turn the recorder off and modify it to something like this with 4 rather than 2 conditions:

Sub Recolor()
    For Each cell In Selection
    If cell.FormatConditions(1).Interior.Color = 13551615 Then
        cell.Interior.Color = 13551615
    ElseIf cell.FormatConditions(1).Interior.Color = 13551000 Then
        cell.Interior.Color = 13551000
    End If
    Next cell
End Sub

You are recording it so you can determine the color numbers.

Tomo,

As with anything when troubleshooting computers, the first thing that's required is context.

What exactly were the users doing when this message pops up?  Working with email, an Office document, browsing Internet or something else?

It doesn't matter which portion of the operating system appears to be reporting such items any longer, since the Windows Defender portions included in Windows 10 are now often integrated with other applications such as Office or the Edge browser and so the notifications can appear to come from the Defender apps which are modularized within the Windows Defender Security Center.

Are your operating systems the Enterprise or Pro version or are you aware that you've got Windows Defender ATP installed?

Note that the "Your IT administrator requires...". message is generated due to administrative policies in place on the PC and not because any particular individual configured them.  This is merely displayed in order to indicate that the computer user isn't allowed to change these policies.

The original poster in other thread that was referenced above had specifically indicated that Office 365 was involved, so that's why I referenced the information relating to that particular set of optional protection.

In your case the notifications may be coming from a different source, but the core reasons are likely to be either due to Office apps including email or possibly the browser, since these are where most of the Windows Defender ATP related items have been integrated at present.  It's always possible they've been added to other areas of the operating system or apps, but I'd start with trying to understand what might be triggering them in order to find the actual origin.

Since Word works correctly in Safe mode, the cause is a damaged registry key, a damaged Normal template, or a poorly designed add-in. Did you install something recently (that may have included an add-in), or was Word running when Microsoft updates were installed?

If you can't identify a possible culprit directly, start with renaming Normal.dotm and see if that makes a difference. If the problem still remains, delete the Data key in the registry. Finally, look for add-ins in Word. See http://support.microsoft.com/kb/921541 ("How to troubleshoot problems that occur when you start or use Word 2010, Word 2007, Word 2003, or Word 2002"). You may also want to read http://word.mvps.org/faqs/customization/CheckIfAddinsInstalled.htm.

Note that reinstalling Office rarely fixes any issues, because registry entries and some files may be retained.
