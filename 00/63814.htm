Transferring folders between internal hard drives

Can I transfer folders between two internal hard drives by dragging and dropping or is dragging and dropping only good for files? What if the folder to be transferred has a lot of subfolders such that the main folder is on the order of 10 GB? Aside from the time it would take to transfer would this work? This is Windows 7.
ramesh srinivasan say:

Yes. that would work the same way as files. When you drag and drop a file/folder to a location in another partition or drive, the default drop effect is "Copy". When dragged and dropped to a location within that partition, the default drop effect is "Move".

So, to move file/folder to another partition, drag and drop by holding the SHIFT key down (so that the item is "moved" instead of being "copied".

>>What if the folder to be transferred has a lot of subfolders such that the main folder is on the order of 10 GB? 

The entire thing is transferred, including the subfolders, files etc that are present in that folder.

Also, see the excellent post by Daniel Martin here.

Yes, you can set a drop default action “per file type”, too | The Hows and Whys of Windows : http://blogs.msmvps.com/dmartin/blog/2012/01/01/yes-you-can-set-a-drop-default-action-per-file-type-too/

If your data is organised well, copy and paste is the simplest backup solution. Of course, if your data is distributed in lots of folders that do not have a common 'parent' this backup solution can be tiresome.

NTbackup can be used to back up the entire contents (or specified sections) of your disk drive(s)/Partition(s) into one file. This is installed by default in XP Pro but you have to do a manual install in XP Home from the XP installation CD (the installer is in <opticaldriveletter>:\VALUEADD\MSFT\NTBACKUP). The program is accessed by Start-All Programs-Accessories-System tools.

http://support.microsoft.com/kb/308422

 

I would only recommend NTBackup as a means of backing up and restoring data. Do not rely on it as a means of backing up and restoring the whole Windows system including Windows itself and installed programs.

 

Or you could research your own backup solution. But here is the most important thing, especially if you are new to any form of backup/restore...

 

Test the restore process, before re-installing windows, to make sure that your data is recoverable after you have wiped it by a clean Windows installation.

 

It is also a good idea to do two distinct backups to separate media.

a. user say:

Your system is afflicted with malicious software.

The malicious software knows the tools you are going to use to try to find and get rid of it, so it disables (using Group Policy) things like System Restore, Task Manager, Command Prompt, Regedit, etc.  The malicious software wants to fool you into thinking you have to perform drastic measures like System Repair or reinstall your XP when none of that is necessary.

You don't need to be shocked when you don't see things in Group Policy Editor - that just means that you need to add a few templates to GPE, but you can do that later.  And, if you were running XP Home, there is not GPE in XP Home so what would you do then?

You need something that is going to work under all circumstances.

You can run malware scanners all day and they may not fix this problem (or the ones you don't know about yet), since as far as they can tell, these things being disabled on your system could be legitimate so they will/may not report them as "issues" for you to fix.

First get your TM working, then you can use a registry import to undo the Group Policy changes, then you can scan your system for malicious software and remove it:

Here's how:

Before making any changes to your registry, back up the registry with this popular free tool:

http://www.snapfiles.com/get/erunt.html

You may also want to manually create a System Restore point before making these adjustments.

Click Start, Run and in the box enter (copy/paste the command into the Run window would be safer) the following text:

REG add HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\System /v DisableRegistryTools /t REG_DWORD /d 0

Click OK and respond in the affirmative if prompted about an overwrite message.

Click Start, Run and in the box enter (copy/paste the command into the Run window would be safer) the following text:

REG add HKLM\Software\Microsoft\Windows\CurrentVersion\Policies\System /v DisableRegistryTools /t REG_DWORD /d 0

Click OK and respond in the affirmative if prompted about an overwrite message.

Now see if you can run regedit.exe and if the Registry Editor has been enabled, you can run this enableit.reg file to fix the rest of the problems (and some things that are probably broken that you don't know about yet):

Open notepad to create a new text file:

Click Start, Run and in the box enter:

notepad

Click OK to open a new notepad file.

Copy and paste the following lines of bold text into the new notepad file.


Windows Registry Editor Version 5.00

Save the new text file with a .reg extension to your desktop or someplace you can remember with a name you can remember, something like:

enableit.reg

After saving the file, close notepad.

debleena s say:

Hi,

Which antivirus software are you trying to install?

Side by Side architecture uses a trick in a file called a 'manifest' to call a special system file instead of the regular Vista system file (a .dll). The error is saying that finding that special dll written for that application is not working somehow.

Let’s follow these steps which should resolve the issue.

Step 1

Run a System File Checker [SFC] scan on the computer which will replace the missing or corrupt files.

To do this, follow the steps given below:

1. Click the Start button.

2. From the Start Menu, Click All programs followed by Accessories.

3. In the Accessories menu, Right Click on the Command Prompt option.

4. From the drop down menu that appears, Click on the 'Run as administrator' option.

5. If you have the User Account Control (UAC) enabled, you will be asked for authorization prior to the command prompt opening. You may simply need to press the Continue button if you are the administrator or insert the administrator password etc.

6. In the Command Prompt window, type: sfc /scannow and then press Enter.

7. A message will appear stating that 'The system scan will begin'.

8. Be patient because the scan may take some time.

9. If any files require replacing SFC will replace them. You may be asked to insert your Vista DVD for this process to continue.

10. If everything is okay you should, after the scan, see the following message "Windows resource protection did not find any integrity violations".

11. After the scan has completed, close the command prompt window, restart the computer and check.

For further reference check the link given below:

How to repair the operating system and how to restore the operating system configuration to an earlier point in time in Windows Vista

http://support.microsoft.com/kb/936212

Step 2

Download, install and run the windows cleanup utility to remove any remaining files from the previous installation of the antivirus software.
Link to download cleanup utility: http://support.microsoft.com/kb/290301/

Then try installing the antivirus software and check if it’s gets installed.

 

Hope the information helps. Please post back and let us know.
