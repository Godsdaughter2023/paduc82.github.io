almagrood say:

PRC server

lemp say:
i have been insteld aprinter in my labtop but finly not insteld because there is no PRC server in my labtop

Do you mean that you were trying to install a printer and could not, and that the error message you got was "Operation could not be completed. The RPC server is unavailable"?

What version of Windows, including service pack, do you have?

Has anything unusual or out of the ordinary happened to your computer recently?  If so, what?

Do you have a current antivirus application with an unexpired subscription? Which one?

Have you scanned for malware using MalwareBytes Anti-Malware orSUPERAntiSpyware ?  If not, it would be a good idea to do so.

Do you have any other printers installed?  If so, can you print from them?  Is any printer a Lexmark printer?

A.  Make sure that your system is not in "diagnostic mode."

1.  Click Start, then click Run, then type the following into the Run box and click OK

     msconfig

2.  Make sure that the radio button next to "Normal Startup" is selected.  If it is, click Cancel.  If it isn't, click the button next to "Normal Startup" then click OK and reboot the computer.  Can you install the printer now?

There are many reasons why the RPC server may be "unavailable," so please provide diagnostic information as follows:

B.  Check the status of the RPC Service

1.  Click Start, then click Run, then type the following into the Run box and click OK

     services.msc

2.  Scroll down to "Remote Procedure Call (RPC)."  Note that this is NOT THE SAME as "Remote Procedure Call (RPC) Locator"

3.  What is in the "Status" column for this entry?

4.  What is in the "Startup Type" for this entry?

5.  Double click on the line for "Remote Procedure Call (RPC)" and click the "Dependencies" tab.  What is listed in the top box "This service depends on the following system components"?

6.  Click Cancel and then close the Services window.

C.  Check for system errors

1.  Open the Event Viewer: click Start, then click Run, then type the following into the Run box and click OK

     eventvwr

2.  In the left pane, click on "Application" and then look at the right pane.  Start at the top of the list and see if there are any entries with either an X in a red circle or an exclamation point in a yellow triangle.  If so, double click on the entry and then click on the two-page icon (below the up/down arrows).  This copies the information so that you can paste it into your response here (if there are several entries, you may want to open a Notepad document and paste the entries into there before copying and pasting the entire collection into your response).  Don't scroll too far down the list looking for errors.

3.  Now click on "System" in the left pane and do the same as above.

D.  Check for missing files

Open Windows Explorer and confirm that all of the following files are in C:\Windows\system32

     Spoolsv.exe
     Spoolss.dll
     Rpcss.dll

E.  Check the Registry (but do not make any changes at this time)

1.  Open the Registry Editor: click Start, then click Run, then type the following into the Run box and click OK

     regedit

2.  In the left pane, navigate to

     HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\RpcSs

3.  Click the + to the left of "Services" to expand the entry and confirm that there is an entry named "RpcSs"

4.  Confirm that there is an entry in the right pane named "ImagePath" and that the value under "Data" is
     %SystemRoot%\system32\svchost -k rpcss

5.  The complete right pane should look like this http://cid-18b259158254905b.office.live.com/self.aspx/.Public/rpcss.jpg

6.  Click File > Exit (or the red X at the upper right corner) to exit the Registry Editor without making any changes.
