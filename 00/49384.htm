drifterx22 say:

A latency test is run to a specific server (s) that Microsoft has.  I do not know that MS has published where these server(s) may be located; most likely in different data centers depending on region.

When you run a speed test from a PC or tablet it is by default looking for the closest server for that speed test to give you the lowest possible result.  

Bandwidth versus latency.  You overall bandwidth is the size of the pipe and has little to do with the actual latency.  Bandwidth has to do with how much can I pull down; higher bandwidth pipe do tend to have lower latency because they are of a higher quality.  

When you say "I have checked with internet provider and they are getting same as me 15-20ms. Only the Xbox one X gets 150-170ms?" does this mean from you to provider is 15-20ms or from provider to MS is 15-20ms?  If your latency to your provider is 15-20ms that is terrible as that's your first hop essentially.

Some things you can do to try to see where a problem may be is to ping some websites.  You can also run a command called trace route to a site or IP (tracert) is the command.  You would run these command from a command prompt on a PC/laptop.

Please keep in mind you reference MW2 lobbies and those servers are likely in a different location than the MS latency test servers so your ping to those will be different.

A ping is simply how long does it take a packet of data to travel from your console to a server and return to your console.

Figuring out network latency is extremely difficult as a home user.  You have zero access to 98% of what you need to see to know where a problem may be.  It could very well be your ISP is not providing you the level of service you pay for.  Some do and just hope you don't notice or if you do complain they bump you up because most people have no idea.

Hi GaryDrabwell,

 

1) Were you able to connect to the Internet earlier?

This issue can occur, if-


a) The security passkey has been entered incorrectly repeatedly.
b) The Network adapter drivers are not updated.
c) The security passkey has been changed recently.

 


Method-


As mentioned in the thread by Halima S, there are two security settings for the network.
One being WPA-PSK (Wi-Fi Protected Access) and the other being WEP (Wired Equivalent Privacy).
These security settings can be accessed on the Router Status Page (RSP).


The RSP is generally accessed with an Internet Protocol (IP) address. The details will be provided in the manual that you might have received with the Router.


Try logging on to the page and change the Security type to either WEP or WPA and generate a new password.


Save the changes made and try to connect with the new password.
Check if it helps.

Also try updating the Wireless Internet drivers.
To do so, you need to follow the steps mentioned below –

 

a) Click on “Start, and Right Click on “Computer” and select “Manage”.
b) Now select “Device manager”.
c) Now expand “Network Adapters” and Right Click on your Network Adapter drivers and select “Update driver Software”.
d) Now select “Search automatically for updated driver software”.
e) Wait for it to Update the driver and restart the computer.
f) Check if the issue persists.

 

Additionally, you may refer the article mentioned below for further assistance –

 

Update drivers: recommended links

http://windows.microsoft.com/en-US/windows7/Update-drivers-recommended-links

 

If none of the steps help fix the issue then you need to contact the Internet Service Provider (ISP) for further assistance.
