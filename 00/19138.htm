Is there a way for me to set up the command button so that the data is saved, yet returns to the previously viewed record while saving the new record and including that new record in the query as well.  There are several subforms tied to this main form.

Any information would be helpful....

Sammie

scottgem say:

How does the user get to a New record? What I would do is set a TempVar before you move to a new record with the PK and the last record viewed. Then after the Requery, filter for that record.

ken sheridan say: You'd need to use command buttons to go to a new record and to requery the form/navigate back to the previous record.  Here's an example of the code for the module of a form whose recordset's primary key is ContactID:

Option Compare Database
Option Explicit

Private lngContactID As Long

Private Sub Form_Current()

    Me.cmdSave.Enabled = Me.NewRecord
     
End Sub

Private Sub cmdNewRecord_Click()

    lngContactID = Me.ContactID
    DoCmd.GoToRecord acDataForm, Me.Name, acNewRec
    
End Sub

Private Sub cmdSave_Click()

    Dim strCriteria As String
    
    strCriteria = "ContactID = " & lngContactID
    
    Me.Requery
    
    With Me.RecordsetClone
        .FindFirst strCriteria
        If Not .NoMatch Then
            Me.Bookmark = .Bookmark
        End If
    End With
    
End Sub

The value of the primary key of the current record when the cmdNewRecord button is clicked is grabbed to the module level variable, lngContactID.  After requerying the form, which inherently saves the new record, the code in the cmdSave button's event procedure navigates back to the previous record by synchronizing bookmarks in the usual way.  In the above example the cmdSave button immediately returns the record pointer to the previous record, but you could easily separate the code to navigate back to the previous record out into the Click event button of a third button if you'd prefer.

sillbit say:

Memory full don't have anything to do with mass storage (neither C: nor E: ).

 

Think of phone memory as ram (same as in your computer) and C: and E: as hard disks, and you have a good analogy.

If you run out of ram, freeing space on the hard disk isn't going to help. You can delete any number of files, and your running programs will still not be able to allocate any more ram.

The only solution is to kill some of the running applications, so hopefully the ones left can get some more ram. However, depending on how the system works, and the amount of ram requested, it might still not work.

 

Turning off the phone, and on again should help. Just as removing the battery and reinserting it. You basically start the phone up from scratch, and unless you have too many programs starting at power on, yor situation will be improved.

 

However, all this said, the N97 have unfortunately, a little bit too little ram as well, so it is not likely the situation will ever become perfect. Learn to live with it, just as you'll have to live with a C: which is annoyingly small.

When you paste a slide of one aspect ratio into a presentation with a different aspect ratio, many objects, including the motion paths are resized to fit the new aspect ratio. This can distort objects or paths. So you should create this on a slide that is already the correct size.

After creating the oval motion path, create a rectangle and resize it to be a square as large as you want the circular path to be. Move the square behind your motion path and resize the path until it just touches the square on both sides, top and bottom. Now it's a circle. Repeat for the other 4 pictures.

john korchok say:

Thanks for uploading those, that clarifies the problem (but doesn't solve it). As you've noticed, PowerPoint behaves in the following unfriendly ways:

  • Inserting a Circle path only creates a circle if the aspect ratio of the slide is square.
  • If the slide aspect ratio is 16:9, inserting a circle gets you an oval in 16:9 ratio width to height. At the same time, the speed is hard-coded to the same ratio as the oval. As long as you leave the path oval, the picture moves around it at a constant speed.
  • Changing the oval to a circle makes the picture speed up at the points of the oval that were pointy, because it now has more distance to cover. This is unconnected with the ease in/ease out effect, it's a whole separate issue.
  • Because your presentation is actually wider than 16:9, it makes the effect even worse.
  • Because you have to rotate the "circle" paths to line up with each photo, the spots where the pictures speed up is different for each photo.

To be honest, I wasn't able to find a way to fix this. Pasting the version from the square presentation automatically stretches the paths to wide ovals that can't be made circular using PowerPoint's limited tools. However, it does have a cool look, like planets orbiting the sun. You might have to settle for that unless someone else has a brain wave.

Having similar tools to the drawing tools to manipulate and constrain motion paths would be mighty handy. I've posted a suggestion to this effect at the PowerPoint User Voice forum: Motion paths need measurements and constraints If you can add your vote, it will help motivate Microsoft to add this.
