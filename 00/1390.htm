If you are using parameter prompts in the report's record source query, then a text box in the report can display the range by using an expression like:

If the query gets the range from text boxes on a form, then use this kind of expression:

Create a COUNTER field (NOT COUNT) in a query - Microsoft Community

christw1 say:

Create a COUNTER field (NOT COUNT) in a query

How do I genrate a COUNTER field (NOT COUNT)

What I want to do is run a query with a particular sort order, and have a field with a counter in it that will count records 1 to (max value) based on the sort order.

qa guy commelec say:
This will do it --
SELECT Score.Player, Score.TotalScores, Sum(IIf([XX].[TotalScores]<=[Score].[TotalScores],1,0)) AS Rank
FROM Score, Score AS XX
WHERE (((XX.TotalScores)<=[Score].[TotalScores]))
GROUP BY Score.Player, Score.TotalScores
ORDER BY Sum(IIf([XX].[TotalScores]<=[Score].[TotalScores],1,0));

If you cannot follow then post your select query SQL with sort.

ken sheridan say:

One problem you might encounter in doing this is if there are two or more rows with the same value in the column or columns by which the query's result table is ordered.  You can overcome this by bringing a column with distinct value, e.g. an autonumber primary key column, into play as the tie-breaker.  This is an example with query on a table of financial transactions ordered by date in which there can be multiple transactions per date:

 

SELECT COUNT(*) AS RowCounter, T1.TransactionDate, T1.TransactionAmount

FROM Transactions AS T1 INNER JOIN Transactions AS T2

ON (T2.TransactionID<=T1.TransactionID

OR T2.TransactionDate<>T1.TransactionDate)

AND (T2.TransactionDate<=T1.TransactionDate)

GROUP BY T1.TransactionDate, T1.TransactionID, T1.TransactionAmount;

 

Don't be confused by the fact that the COUNT operator is used.  In this case it is merely used as the mechanism to return sequential row numbers in the result set.

 

An alternative, but less efficient method, using a subquery:

 

SELECT

   (SELECT COUNT(*)

    FROM Transactions AS T2

    WHERE T2.TransactionDate  <=  T1.TransactionDate

    AND ( T2.TransactionID <= T1.TransactionID

    OR T2.TransactionDate <> T1.TransactionDate))

AS RowCounter, T1.TransactionDate, T1.TransactionAmount

FROM Transactions AS T1

ORDER BY T1.TransactionDate, T1.TransactionID;

 

Or, for an updatable result table, by calling the VBA DCount function:

 

SELECT DCOUNT("*","Transactions","TransactionDate 

<=  #" & Format(TransactionDate,"yyyy-mm-dd") & "# 

AND (TransactionID <= " & TransactionID & " 

OR TransactionDate <> #" & Format(TransactionDate,"yyyy-mm-dd") & "#)")

AS RowCounter, TransactionDate, TransactionAmount

FROM Transactions

ORDER BY TransactionDate, TransactionID;
