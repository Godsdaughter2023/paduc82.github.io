REG DELETE "HKLM\SOFTWARE\Policies\Microsoft\Windows Defender\Real-Time Protection" /v DisableRealtimeMonitoring

a. user say:

Can I delete Outlook.pst?

Outlook recently stopped working every time it received emails, and the only way I could fix it was by switching from POP3 to IMAP.  I deleted the dysfunctional POP3 E-mail account.  However, in Account Settings/Data Files, a file currently called Outlook.pst, in Documents/Outlook psts, is Set as Default, and the Set as Default button is greyed out for the IMAP Data File.  When I launch Outlook, it opens to the old POP3 Inbox, despite the fact that the POP3 E-mail account has been deleted.

karl timmermans [outlook mvp 2012-2018] say:

Question: if I move all my important subfolders from the old POP3 folder into my IMAP folder, can I just change the name of Outlook.pst to oldoutlook.pst, and delete it from Account Settings/Data files?  Will the Set as Default button then not be greyed out for my IMAP Data File, and will Outlook open to the IMAP Inbox when launched?  I don't want to do this if it will create new problems.

First thing that needs to be understood is the significance of the "default data file" which is the data store where the "default" contact/calendar folders for the profile are maintained.

Just going by your description - 

1 - You deleted the POP3 email account either before or after configuring the same account using IMAP

2 - Removing a POP3 account doesn't delete the associated PST file from the profile 

3 - You can't delete a data file from the profile if it's designated as the "default data file" for the profile for reasons noted above

4 - Your Imap folder tree has no contact/calendar folders. Those folders, if existed, would have also contained the words "This Computer Only" in the folder name

You could make the underlying .OST file associated with the IMAP account be the default for the profile but wouldn't suggest it. If you ever delete the IMAP account all data in any folder labeled with "This Computer Only" will be lost. So in your case, strongly suggest keeping the status quo for the purposes of contact/calendar data.

As for the question about moving your important folders to the IMAP account - short answer is: Sure. Just make sure that your email service provider gives you enough space on the server to store it. Also, suggest setting Outlook to work in off-line mode while copying the folders to the IMAP account so that sync'ing doesn't start until after everything is completed. When done - go back to "online" mode.

Finally, in the above description I explicitly used the word "copy" instead of "move" for a very good reason. If you "copy" - you won't lose any data in the source data file is something happens to go sideways during the process. "Moving" something is in effect a copy "to destination" & delete "from source" process.

    Any assistance would be appreciated.

    jay freedman say:

    The extra paragraph mark is part of the footnote separator (the horizontal line). To remove it:

    • Click View > Draft.
    • Double-click any footnote number in the text to open the footnote pane at the bottom of the window.
    • Change the dropdown from "All Footnotes" to "Footnote Separator".
    • Delete the extra paragraph mark.
    • Go back to Print Layout. The extra space should be gone.
