macropod say:

Further to Graham's post, you may need to suppress any spaces and other characters associated with the Unit field when it's empty. The simplest way of handling this is to add the \b or \f switch to the mergefield and insert whatever separates the two fields into that. For example, suppose you have:
«Unit»/«StreetNum»
and the «Unit» field is sometimes empty. Select and field and press Shift-F9. You should now see:
{ MERGEFIELD Unit }/«StreetNum»
Edit the Unit mergefield and delete the '/' after it, so you get:
{ MERGEFIELD Unit \f "/"}«StreetNum»
Then select the field and press F9 to update it. Word will now insert the Unit content and the '/' whenever the Unit field is not blank.

Similarly, suppose you want to have:
Unit «Unit», «StreetNum»
Naturally, you won't want 'Unit' appearing when there's no unit! In this case, you'd use both switches, as in:
{ MERGEFIELD Unit \b "Unit " \f ", "}«StreetNum»

a. user say:

The following code broke it down per your example.

               

'// VBA Reference set to Tools / Reference...
'// Microsoft VBScript Regular Expressions 5.5

    Dim RE As RegExp
    Set RE = New RegExp
    Dim Match
    Dim R, C
    Dim Sp
    Dim Result
    Dim Cell
    Dim S
    Dim P(1 To 13)

    Sp = "[ ]?"             '0 or more spaces

    P(1) = "^"              'Start
    P(2) = "([A-Z .]+)"     'Address w/ spaces & .
    P(3) = Sp
    P(4) = "(\d*)?"         'Digits 1 or more
    P(5) = Sp
    P(6) = "([A|B])?"       ' A or B   Problem if next is "st"
    P(7) = Sp
    P(8) = "(\d|st)?"       'Digit or "st"
    P(9) = Sp
    P(10) = "(th|tv|[A-Z ])*"  'th or tv, or text
    P(11) = Sp
    P(12) = "([-\d]*)?"     'ending digits
    P(13) = "$"             'End

    RE.Global = True
    RE.MultiLine = True
    RE.IgnoreCase = True
    RE.Pattern = Join(P, vbNullString)
    Debug.Print RE.Pattern ' Just to see

    For R = 1 To 13
        S = Cells(R, 1).Text
        S = Replace(S, ",", " ")
        S = Replace(S, ".", " ")
        S = WorksheetFunction.Clean(S)
        S = WorksheetFunction.Trim(S)

        If RE.Test(S) Then
            Set Result = RE.Execute(S)
            Set Match = Result.Item(0).SubMatches
            For C = 3 To 8
                Cells(R, C) = Match(C - 3) ' First Index is 0
            Next C
        End If
    Next
End Sub

ken sheridan say: Not in a single query, but you can execute the two queries in a single function, e.g.

Public Function RunMyQueries()

    Dim dbs As DAO.Database
    Dim strSQL As String

    Set dbs = CurrentDb

    strSQL = _
    "UPDATE dbo_WipJobAllLab INNER JOIN [SCHEDULE FOR ORDERS TO PRINT] " & _
    "ON dbo_WipJobAllLab.Job = [SCHEDULE FOR ORDERS TO PRINT].Job " & _
    "SET dbo_WipJobAllLab.WorkCentre =  " & _
    "[SCHEDULE FOR ORDERS TO PRINT]![WorkCentre2],  " & _
    "dbo_WipJobAllLab.PlannedQueueDate =  " & _
    "[SCHEDULE FOR ORDERS TO PRINT]![JobStartDate],  " & _
    "dbo_WipJobAllLab.PlannedStartDate =  " & _
    "[SCHEDULE FOR ORDERS TO PRINT]![JobStartDate],  " & _
    "dbo_WipJobAllLab.PlannedEndDate =  " & _
    "[SCHEDULE FOR ORDERS TO PRINT]![JobStartDate]  " & _
    "WHERE dbo_WipJobAllLab.Operation = 2"

    dbs.Execute strSQL,dbFailOnError

    strSQL = _
    "INSERT INTO dbo_RTE_JobTracking ( Job, Status, [Time] ) " & _
    "SELECT [Schedule For Orders to Print].Job,  " & _
    "[Schedule For Orders to Print].JobClassification,  " & _
     "#" & Format(Now(),"yyyy-mm-dd hh:nn:ss") & "#  " & _
     "FROM [Schedule For Orders to Print]"

    dbs.Execute strSQL,dbFailOnError

End Function

You can call the function from a button on a form for instance by setting its On Click event property to:

=RunMyQueries()

BTW, I'd recommend against using Time as a column name as it's the name of a built in function.  Something like TimeEntered would be better.

suzanne s. barnhill say:

Don't uninstall the update. We're expecting that Microsoft will issue a patch soon. For the time being, if you are able to open Word documents from within Word Starter, then you should save attachments to your HD, open Word Starter, and use its Open dialog to open the documents. If you are able to open documents this way, then you can probably also solve the problem by manually repairing the damaged file associations:

  1. In Windows Explorer, right-click on a Word document.
  2. Choose Open With, then Choose Default Program.
  3. If you see "Microsoft Office Client Virtualization Handler," choose that.
  4. Making sure that the box for "Always use the selected program to open this kind of file" is checked, click OK.

If this solves the problem, you're home free. If it does not, do nothing more. Do not follow any repair or especially uninstall instructions you may find in these forums or elsewhere online, as they currently do not work and will just make the situation worse. Instead, continue to open files from within Word Starter until MS issues a fix.
