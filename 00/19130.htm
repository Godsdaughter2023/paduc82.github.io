It’s not feasible to enable the “$bcs.RevertToSelfAllowed = $true” in SharePoint Online via SharePoint Online Management shell. I guess you have searched some articles and found the following command maybe able to enable authentication mode as BDC identity.
$bcsServiceApp = Get-SPServiceApplication | where {$_ -match "Business Data Connectivity Service"
$bcsServiceApp.RevertToSelfAllowed = $true;
$bcsServiceApp.Update();

steven. s say:

Hey Loni8217,

 

You can change the default download directory by editing the registry.Modifying REGISTRY settings incorrectly can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the configuring of REGISTRY settings can be solved. Modifications of these settings are at your own risk.

·        Press the Windows Key &  R keys on your keyboard

·        Type regedit into the Open: field

·        Navigate to the Computer\HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer

·        Select the Internet Explorer “folder” in the path above

·        Right click Download Directory in the right column

·        Click Modify…

·        Change the path to the path you would like to be the default directory

·        Click OK

·        Close the Registry Editor

 for office 2016: http://www.microsoft.com/en-us/download/details.aspx?id=49117

 <configuration>
 <add sourcepath="c:\odt" officeclientedition="32">
     <product id="o365proplusretail" >

       <excludeapp id="onenote" />
     </product>
 </add>
 </configuration>
