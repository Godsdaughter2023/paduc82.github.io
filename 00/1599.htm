scottgem say:

First, mdb is the old extension used for Access files prior to Office 2007. Since then its been accdb, though mdb is still supported.

The accde format takes an accdb format and "compiles" it so that the user can't get into Design mode for forms, reports, macros and VBA.

So it sounds like someone updated the front end to the newer format but left the back end as mdb. All of that is, however, immaterial to your problem. What it also sounds like is that, when the front end was upgraded, the developer decided not to use certain fields that were in the tables. So he just left them off the forms he created. I can't tell you why that was done. But if you need to have the data, then controls need to be added to the form in the accdb. 

Finally this does not make sense:

Someone wanted to have the db "more secure" and converted it to .accde with an .accdb for use by data entry folks.

If one creates an accde, then that is what is deployed to users. Creating an accde but deploying the accdb makes no sense.

Hi Nightrider1981,

Please try the following script:

================

$Username = "username"

$Password = ConvertTo-SecureString “password” -AsPlainText -Force

$LiveCred = New-Object System.Management.Automation.PSCredential $Username, $Password

Connect-MsolService -Credential $LiveCred

$AccountSkuId = "contoso:ENTERPRISEPREMIUM"

$UsageLocation = "US"

$LicenseOptions = New-MsolLicenseOptions -AccountSkuId $AccountSkuId

$Users = Get-MSOLUser -All -UnlicensedUsersOnly

$Users | ForEach-Object {

             Set-MsolUser -UserPrincipalName $_.UserPrincipalName -UsageLocation $UsageLocation

             Set-MsolUserLicense -UserPrincipalName $_.UserPrincipalName -AddLicenses $AccountSkuId -LicenseOptions $LicenseOptions

}

================

Note: Please change the username, password and contoso:ENTERPRISEPREMIUM based on your scenario.
