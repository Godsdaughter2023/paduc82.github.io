Files with .cab file extensions, formerly known as the Diamond File, are compressed files that store data related to Windows installations. It usually involves device drivers and system files.

karl timmermans [outlook mvp 2012-2018] say:

Let's start by being very clear with the terminology used - specifically in terms of "not visible in the Contact list". Are you referring to the contents of a specific "contact folder" or the list of contacts displayed when you open the AddressBook?

#1 - Any contact without a valid email address or fax number will not appear in the AddressBook but will still be "visible" in the contact folder where it resides which is the only way it would be included in the export

#2 - As for the error encountered during import - just means you tried to import to a contact folder associated with an Outlook.com account configured using Exchange ActiveSync. You can't import into that contact folder - contacts need to be "copied/moved into it" - ergo if you have contacts in a csv file not already in Outlook - import them first into another contact folder and then copy them to the Outlook.com contact folder

So abbreviate this - if you're trying to get the contacts in the folder you tried to export into Outlook.com then open the contact folder with the contacts --> use CTRL-A to highlight tem all --> right click --> Move --> Copy to folder (suggest using copy so that you don't lost anything in the event of a problem - you can always delete the original items after everything is successfully copied to your Outlook.com folder)
