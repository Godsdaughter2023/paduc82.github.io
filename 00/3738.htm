IE9 displays oversaturated images on wide gamut monitors - - Microsoft Community

rishithukral say:

IE9 displays oversaturated images on wide gamut monitors - WHY????

I use IE9 on a Dell U2711 wide gamut monitor. All websites look oversaturated. Firefox, Chrome and Safar show colors correctly though. Why?

My minitor is color calibrated and the correct profile is being used as the default in the color management settings.

Does anyone know why oversaturation is visible only in IE and not other browsers? How do i fix this?

radeldudel say:

There probably could be disagreement if this is a bug or by design.

The problem is color management, of course. For all images on websites which do not contain any color profile IE9 or the OS has to decide: how am I going to interpret this colors?

On wide gamut displays IE9 goes like "hey, this image on the web has no color profile. I'm just going to assume the profile which the OS is using to show it". Of course, since the OS is using adobeRGB but the image usually is in sRGB this leads to oversaturated display of the images.

The other browsers go like "hey, this image does not contain a color profile, I'm going to assume sRGB for them".

Or, in a nutshell: The IE team did not have anyone with a wide gamut display on the team and did not stumble across this. The other browsers teams had someone. And entering a fix for this to a huge dev project like IE is not something done on the side, so the issue still is unresolved (and probably there are heated discussions by some devs/designers if it wouldn't be the better choice not to assume sRGB but the OS color space anyway, which is stupid but hey, it's live).

 

So in short: Color management in IE9 sucks. The only workaround is to switch your OS color space to sRGB, but this way you'll forfeit the additional color room of the wide gamut display, which would be a real pity - you did buy this expensive for the wide gamut, so switching it off usually is no option.

 

My Eizo CG has a button to switch between different color spaces quickly. So for surfing I'm switching to sRGB, for image work back to adobeRGB.

 

Hope this did explain - and I really hope they fix this in IE10.

Sam
