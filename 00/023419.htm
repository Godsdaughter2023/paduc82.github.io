============


When you opens an Excel file directly from a SharePoint document library by Excel for Mac, it shows an older version of the file. While it shows the correct version when open the same file by Excel Online.


The Office Document Cache is corrupted.


Open Finder, go to Library > Containers > com.microsoft.Excel > Data > Library > Application Support > Microsoft > AppData > Microsoft > Office > 16.0 and delete the OfficeFileCache folder.


The OfficeFileCache folder contains the Office Document Cache. When files opened directly from SharePoint by Office applications. Storage Folder in Outlook

How do I turn on the "Storage Folder" for Outlook in Office365?  I have searched nearly everywhere (obviously not Everywhere since I haven't found the option yet).  I ran across the option just a few months back on my other computer, but don't remember where it is.  Please tell me.  *Note--The folder in question is called "Storage Folder" and sits below all the accounts' folders in the left hand side pane that holds all the inboxes, outboxes, sent folders and all the other folders.  It is a standalone folder that you can add emails to or add other folders and then emails to those.  Please help me find the option.

jasenewalt say:

Hi Alan, You're right about one thing, it's just called "Storage", not "Storage Folder".  That was my fault.  But I have Office 365 on both of my pc computers.  On my other computer I have 3 email accounts that are controlled by outlook.  It has the standard "inbox", "drafts", "Sent Items", etc...1 for each of the 3 accounts.  Underneath all three accounts is a 4th entry that is simply called Storage...

Scratch all that.  I just found a way to do it.  On Outlook 2016 or Outlook 365 there is an option hidden somewhere as I have found it on 2 separate computers, just not this one.  In the past it has taken literally weeks to locate the option.  But here's a way to do it.

Select File-->Account Settings-->Account Settings-->Highlight the Data Files button-->Click Add and then provide a file name, in this case I typed in "Storage", and click OK.  Now a separate entry is created in the folders pane on the left side of the screen.  

In the past I have found an option somewhere within the program that simply required checking a box.  But this works just as well or maybe even better.  Thank you, Alan, for researching this and getting back to me so soon.  I think by going by things this way I should never forget how to do it in the future.

Accde form update without db loss - Microsoft Community

alesalomao say:

Accde form update without db loss

Hi,

  In my company we use an access 2013 form and distribute it as accde, is there any way to copy it to users without losing already entered data?? Since it replaces the previous accde form in Windows explorer and most of the users use only access runtime.

scottgem say:

EVERY Multi-user application NEEDS to be split between a back end (the data tables) and a front end (everything else). The back end is stored on a network shared drive and the front ends installed to each user's PC.

I'm not sure why you think you would lose data. 

daniel pineault say:

As long as your database is properly split, which it should be one way or another, then you can safely replace the FE with updates anytime without any risk of loosing data since the data itself is store in the BE.

To be 100% safe, you can always make a backup copy of the FE and BE prior to pushing out any updates.  This is a good idea to do anyways, just in case you make some critical error in one of your updates.

john w. vinson say: Data is NOT stored in Forms - only in Tables. A form is just a window, a way to see and manage data stored in a Table. As Scott and Daniel say, you need to split the database into a shared "Backend" containing the tables (and the data!), and individual copies of a "Frontend" containing the forms, reports, queries and code, with links to the tables. Note that these links are designed to work only over a fast, stable, wired Local Area Network; if your users are spread over the internet you'll need to consider a Web application instead of a desktop database.
