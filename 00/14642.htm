trevor rabey say:

I don't see it it as a big problem.

It seems to me that your main issue is with the rounding off to 2 decimal places when using decimal days as durations.

The rounding off only occurs in the display, but internally is stored in minutes. So it is just accepted that is the way it is.

Because you can't express all decimal days numbers as exact hours or minutes, there has to be some rounding off in a display and it may as well be 2 decimal points instead of 3. The issue only arises because "day" is inherently ambiguous even though it is a commonly used duration. We all know that "1 day" means whatever number of hours it is set to mean in file, options, schedule. It is a conversion factor and you get precisely one/project.

The obvious way to deal with it is to be prepared to input durations in hours, and not decimal hours except where it rounds out to exact whole minutes, so 1.1 hours = 66 minutes etc.

If necessary, input durations in both hours and days, but being sure that "days" is really a number of hours anyway.

The rounded off display also occurs with the dollars and cents in the costs.

The precision in MSP is more than adequate for any real world project.

  1. Get the Username that is being used to login to the service listed in Services.msc under Windows Azure Active Directory Sync Service
  2. Add that user to the local "Administrators" user group
  3. Open the Default domain GPO and navigate to Computer Configuration > Policies > Windows Settings > Security Settings > Local Policies > User Rights Assignment
  4. Open "Log on as a service" and add the computer's local "Administrators" user group
  5. Go back to the DirSync Server and perform the "gpupdate /force"
  6. Then perform "Start-onlineCoexistanceSync" in the DirSyncPowershell

roger on excel say:

link shape color to cell value

I would like the color of a shape (for example a circle) to be linked to a cell value in A1

 

The shapes name is CIRCLE1

 

For example if 1 is in A1, the shape is red, 2 will give blue and 3 will give white.

jeeped say:

I would like the color of a shape (for example a circle) to be linked to a cell value in A1

 

The shapes name is CIRCLE1

 

For example if 1 is in A1, the shape is red, 2 will give blue and 3 will give white.

 

Can anyone help?


I hope that name is correct. I believe that Excel assign a shape's name and number with a space in between like Circle 1 by default. In any event, right click the worksheet's name tab and select View Code. When the VBE opens, paste the following into the pane titled something like Book1 -Sheet1 (Code),

Private Sub Worksheet_Change(ByVal Target As Range)
    If Not Intersect(Target, Range("A1")) Is Nothing Then
        For Each c In Target
            If Not Intersect(c, Range("A1")) Is Nothing Then
                lngColor = ActiveSheet.Shapes("CIRCLE1").Fill.ForeColor.RGB
                Select Case c.Value
                    Case 1
                        lngColor = RGB(255, 0, 0)
                    Case 2
                        lngColor = RGB(0, 0, 255)
                    Case 3
                        lngColor = RGB(255, 255, 255)
                End Select
                ActiveSheet.Shapes("CIRCLE1").Fill.ForeColor.RGB = lngColor
            End If
        Next c
    End If
End Sub

Press Alt+Q when done to return to your worksheet.
