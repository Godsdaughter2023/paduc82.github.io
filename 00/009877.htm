Private Sub Workbook_Open()
    Dim rngC As Range

jay freedman say:
There are two factors in enabling macros. First, the template containing the macros must be stored in a folder that Word has been told is a "trusted location". Second, there are settings in Word for what kinds of macros can be allowed to run.

First, the location: In Word, go to Office button > Word Options > Trust Center > Trust Center Options > Trusted Locations. The usual installation makes the Office templates folder trusted, at
   C:\Users\<your name>\AppData\Roaming\Microsoft\Templates
You can store the template there*, or you can store it in another folder and use the Add New Location button in the Trusted Locations dialog to make that folder trusted.

* The AppData folder and everything under it is marked as Hidden, but you can get there by putting   %appdata%\Microsoft\Templates   into the address bar of the Computer program and pressing Enter.

Second, the macro settings: In the Trust Center, go to the Macro Settings tab. Select the option "Disable all macros except digitally signed macros". This is misleading; it really should say "Disable all macros except digitally signed macros or those in templates in trusted locations".

After changing these settings, you'll have to close any document that warned about disabled macros, and reopen them to make the settings take effect.

You also say that the referencing table is 'purchase detail', but the SQL statement refers to a table PDetails.  Moreover, you say that the tables are related on the PO number columns, but the SQL statement refers to an ID column.

If you want to delete all matching rows in the referencing table when a row in the referenced table is deleted then you can simply enforce cascade deletes in the relationship between the purchase orders table and purchase detail table.  No code is required.

If you want to delete a record in a form via a command button first create the following standard module:

' basDeleteRecord
Option Compare Database
Option Explicit

Public Function DeleteRecord(frm As Form)

    Const DELETE_UNAVAILABLE = 2046
    Const DELETE_CANCELLED = 2501
    On Error Resume Next
   
    If frm.Dirty Then
        frm.Undo
    End If
   
    RunCommand acCmdDeleteRecord

    Select Case Err.Number
        Case 0
        ' no error
        Case DELETE_UNAVAILABLE
        ' anticipated error, do nothing
         Case DELETE_CANCELLED
        ' anticipated error, do nothing
       Case Else
        ' unknown error
        MsgBox Err.Number, vbExclamation, "Error"
    End Select
   
End Function


Public Function ConfirmDeletion() As Boolean

    Const MESSAGETEXT = "Are you sure you wish to delete the current record?"
   
    If MsgBox(MESSAGETEXT, vbYesNo + vbQuestion, "Confirm") = vbYes Then
        ConfirmDeletion = True
    End If

End Function
' module ends

Then as the On Click event property of the button enter:

    =DeleteRecord([Form])

In the BeforeDelConfirm event of the form enter:

    Cancel = Not ConfirmDeletion()
    Response = acDataErrContinue

and if you wish the deletion to be confirmed enter the following in the AfterDelConfirm event procedure of the form:

    Const MESSAGETEXT = "Record deleted"
   
    If Status = acDeleteOK Then
         MsgBox MESSAGETEXT, vbInformation, "Information"
    End If

No you cannot delete it, but there is another solution:

1) Open a new text message.
2) Type the problematic word (in your case "vn" ).
3) Press the '*' button until the word you want is shown (in your case "to" ).
4) Press the "0" for a space.
5) Repeat steps 2, 3 & 4 until the word you want is shown immediately. This is usually 4 or 5 times, but it may be more.
6) Delete the message, open a new message and type in the problem word again. It should now be back to normal.

Repeat all of the above for each problem word.

The reason is that the phone gives each word a priority, so if you type in the word you want more often it will eventually become the default.

dianeoforegon say: If this is Exchange I suspect it's your Inbox. See info below for Outlook. It's basically the same for Entourage.


The Inbox syncs more frequently than any other and should be kept as clean as possible. (this is the number one cause of problems with sync). Do NOT make subfolders under the Inbox.

Any folder with more than a few thousand messages is going to take some time to fully come down when you first sync your account. Due to the design of Outlook's sync engine (and how exchange works), you will see the 512 newest messages in a folder at the initial start of sync then the rest of the folder's contents will be back filled before you see any mail that arrives after this point. This is why it will look like you are not getting new mail when a folder is still going through initial sync. This is also further exacerbated by an Exchange issue where it gets unnecessary change events before the newer mail arrives. If you carefully watch the progress during the sync of a folder with greater than a couple thousand messages, you will see "Updating local..." a lot before the newer mail starts arriving (newer than the initial 512 that come down).

Outlook checks the server for updates every minute, any folder that has updates will subsequently be synced. Since Outlook has a limit on how many folders can be synced at a time, there can be a queue of folders waiting. The Inbox does get high priority so it will generally sync before other folders that also need to sync.
