As JP notes, you can't do it in 2 (not without some jiggery-pokery) because as soon as you enter a value in one of them it will overwrite the existing formula that is there.

One option if you don't mind using a bit of VBA is to set up a user-defined function - code below. Then you can have one cell in which you enter a number (doesn't matter if it's decimal or binary) and one that provides the converted answer (as per image). Caution - this code is pretty rough and ready as there is no error checking or such.

If this would suit your purpose but you need help to add the code to your worksheet let us know.

Regards

Murray

Function MYDECBINCONV(InVal As Variant, Optional ConvDir As Boolean = True)
    '
    ' Convert a decimal/binary value to binary/decimal
    '
    ' Usage: =MYDECBINCONV(InVal, [ConvDir])
    '   where:
    '       InVal = decimal/binary value
    '       ConvDir = optional conversion direction indicator. TRUE=decimal to binary,
    '                   FALSE=binary to decimal. Default value if omitted is TRUE
    ' Examples:
    '   =MYDECBINCONV(15) convert 15 to binary
    '   =MYDECBINCONV(B12 TRUE)  Convert the decimal value in B12 to binary
    '   =MYDECBINCONV(B12, FALSE) Convert the binary value in B12 to decimal
    '
    If ConvDir Then
        MYDECBINCONV = Application.WorksheetFunction.Dec2Bin(InVal)
    Else
        MYDECBINCONV = Application.WorksheetFunction.Bin2Dec(InVal)
    End If
End Function

I agree with John and Karl, I see no reason for separate tables or to remove records from a copy of the master. This is a simple process, you append each day's record to the table with a timestamp indicating when the reading was taken. You then run an Unmatched query between the list of vehicles and the sensor reading. Any vehicle that didn't have a sensor reading shows up. You are making a fairly simple process more complex.

qa guy commelec say:

The best and simplest is to have only 2 tables, MasterList with an Autonumber as primary key field and 2nd table to record when vehicles pass the sensor, with a one-to-many relationship.

Then you use an 'Unmatched' query as mentioned.  The SQL for the query would look like this --
SELECT Table1.*
FROM Table1 LEFT JOIN Table2 ON Table1.AutoNumber = Table2.ForeignKey
WHERE Table2.ForeignKey Is Null  AND (Table2.SensorDate Between [Enter period start] AND [Enter period end];
