michaelharleman say:

I think I may have something that will help here.  If you are getting error code 0xc0000185 that can mean one of two things.  The first possibility is that you have been infected with Malware and may be able to fix it with Malwarebytes or some other anti-malware product, or by running a virus scan with your virus scanner.  

There is, however, another possibility that shouldn't be overlooked as in my case.  This error code could be an indication that some piece of hardware attached to your machine, whether internal or external, is failing to start when Windows boots.  I am currently running Windows 8.1 with all the necessary updates.  I have reinstalled the operating system 3 times and still get the same error code.  So in an effort to resolve the issue I called Geek Squad who informed me that if I have a USB device attached to my machine such as a backup drive that is failing to start and was attached when I shutdown Windows, Windows looks for it when it tries to start back up.  If it issues a command to the device to start and it is faulty (I had a drive that was at least 6 years old), then you can get Windows error code 0xc0000185 and the computer will tell you you must reset the operating system.  It's a hardware error not a software error in this case, so it can be either.

So try disconnecting any backup drives or other USB devices you have attached before shutting Windows down if you can get it to come back up which may be possible if you just try booting normally a couple of times such as in my case.  Often on my second or third attempt to reboot it would come up normally but then crash again on shutdown because I hadn't disconnected the device.  I hope this helps anybody else out there that is getting this error code.

gerry c j cornell say:

I cannot see any problem with your WGA Report.


Select Start, Run, type cmd and press ENTER. Type "chkdsk c: /f /r" without the quotes and hit ENTER. Make sure you include the spaces indicated. Enter Y when asked whether you wish to run chkdsk on restarting the computer. Exit and restart the computer.


Marking off bad sectors on a hard drive takes time so be patient. Marking off does not repair a bad sector. It places pointers on the drive telling the system not to read or write to those sectors which have been damaged.


If the number of bad sectors continues to increase after you have run the procedure above several times then you should replace the drive. If an important system file is written to a bad sector you can corrupt registry hives and lose the whole contents of the drive.


On the other hand if having run chkdsk you see no more new bad sectors then the drive can work for you for years.


In Windows 7 you may need to use an Elevated Command Prompt;
