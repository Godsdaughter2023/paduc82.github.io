You can achieve this by enabling the default administrator on your PC. However, we cannot link it with another user account. Default administrator has permissions to access all the settings and features without any UAC prompts. Please follow the steps to enable the default administrator on the PC.

 

  1. Start Menu and type CMD.

  2. Run as administrator.

  3. net user administrator /active:yes

  4. Administrator account.

a. user say:

The Administrators account in Windows 10 is the user account with highest permissions. There are other accounts with higher permissions, but they are not user accounts - they are system accounts.

As a system account, they do not work the same way that a user account works, and so you won't be able to use them as you would normally use a user account.

In general, it's a big mistake to interfere with the industry-standard system of permissions that Microsoft has incorporated into Windows. Unfortunately, some people don't realize that until it's too late to fix the mess they have made. You don't want to be one of those people.

a. user say:

Was the issue preceded by a power interruption, aborted restart, or improper shutdown?  (this includes plug pulling, power buttons, removing the battery, etc.)

Okay - you have two immediate problems...

You need to first run chkdsk /r from the XP Recovery Console.   The chances of the file really being missing or corrupt are slim.  Always start with chkdsk /r from the Recovery Console when you see this message.

Since you have a SATA HDD, when you boot into the XP Recovery Console from a CD, it may not understand that configuration and either hang (trying to figure it out), or if you do get into RC, you will not be able to run chkdsk /r since it does not understand the SATA HDD configuration.

You need to get into your system BIOS (that is before XP starts to load), sometimes by pressing the 'Del' key on your keyboard and temporarily change your HDD settings to some kind of IDE or IDE compatibility mode, then boot into Recovery Console, then run chkdsk /r (repeatedly until it runs clean), then put your BIOS back to the way it was before you started.

That means, when you change the BIOS, you need to make a note of how it is set up NOW, so you can change it back when you are done (or if things get worse).

I have accumulated the following which may help you with adjusting your BIOS, but you may just have to figure it out with a little experimenting - being sure to make a note of the current settings of anything you change so you can change it back.

The adjustment is made in the BIOS and could be under Integrated Peripherals, SATA Device Configuration, SATA Mode.  After you make the changes, save them and see how your Recovery Console works now.

If you find the mode is set to RAID/SATA mode, change the mode to IDE.

If you find the mode is set to SATA, change the mode to IDE.

If you find the mode is set to SATA, disable SATA mode.

If you find the mode is set to AHCI, change the mode to ATA.

If you find the mode is set to AHCI Emulation Mode, change the mode to IDE Emulation.

If you find the mode is set to SATA, disable SATA mode and/or change the mode to ATA.

If you find the mode is set to SATA Native Mode, disable SATA Native Mode.

If you find the mode is set to RAID/Autodetect AHCI, change the mode to Combination.

After making the adjustments, ACCEPT and SAVE the changes, and restart the system and see if it will now boot on the Recovery Console CD.

sumit (independent advisor) say:
Hi,
Two things:
1. Originally OEM licenses are non transferable and are tied to the first motherboard and you cannot use it on another computer. Windows 10 inherits the rights from the qualifying license hence you have to buy a new license.

2. During a recent conversation with Microsoft staff, MVPs were told that there are exceptions always and here too, and if the license isn't misused, activation troubleshooter might activate your machine(The things are very unclear on misuse, but I think this is case by case).

So it is just your luck if you successfully transfer it or not. Otherwise, new license, sorry.
https://support.microsoft.com/en-in/help/12440/...

I hope that helps.
