daniel g. grau say:
On the PC, individual TOC-field entries can be a hyperlink if you add the \h switch. On the Mac, this switch does nothing (although it's tolerated for compatibility) and only the page numbers are links, with or without the \h. You cannot link to a file through a TOC, unless you are using a Master document. Is that your case?

As for the cross-reference error message, as it's a link to a hidden bookmark and again, unless you are using a Master document, the link should be to the current document. Somehow Word thinks it's referencing an external file.

You've probably already tried updating fields (Select All = Command A, followed by Update Selected Fields = F9*) reinserting the TOC and the cross-reference and still it does not work, so try to rebuild your document.

1) Save As in RTF format, close, open the RTF and test. If it works, re-save into the original format.

AND/OR

2) Select the complete document minus the last ¶ (the last character in the document) and paste into a blank document. The last ¶ mark contains a lot of information that may have become corrupted, so that's why you need to leave it out.

AND/OR

3) Although unlikely, corrupt Word Preferences and Normal.dotm could be causing the misbehavior. Hold Shift while you open Word (this will temporarily bypass those two things) and test. It things work, post back to continue solving your problem.


* As for Option F9 not working, I bet none of the F keys work within Word. System Preferences > Keyboard [icon] > Keyboard [tab at the top] has a setting that lets you control whether the F keys do System functions (like volume up, contrast, eject...) by themselves or by pressing the fn key, and vice versa for applications. Your setting must be requiring you to also press this key within programs. Thus, either change that setting or use fn Option F9 to view codes and fn F9 to update fields.

Try to run the Windows update troubleshooter first to see if it fixes the issue:

 

1 Press Windows Key then type ‘troubleshoot’ and hit Enter

2 Select “View all” on the left panel and run “Windows update” troubleshooter from the list

3 Click Advanced, ensure to check Apply repairs automatically and also select Run as administrator

4 Click Next and follow the on-screen instructions to complete the troubleshooting.

 

Once done, check if it allows you install Windows Updates without any errors. Renaming of the Windows Update folders, and then running Windows Update again.

 

Step :1

 

a. Press Windows Key + X.

b. Click on ‘Command Prompt (Admin) Note: Click ‘Yes’ if it prompts for UAC (User Account Control).

c. Type following commands into the command prompt one after the other

net stop wuauserv

net stop bits

net stop cryptsvc

 

Keep Command Prompt window opened. Rename Windows Update folders

At the command prompt, type the following commands, and then press Enter after each command:

 

ren %systemroot%\System32\Catroot2 Catroot2.old

 

ren %systemroot%\SoftwareDistribution SoftwareDistribution.old

 

Then type the following, commands and then press ENTER after each command:

net start wuauserv

net start bits

net start cryptsvc

 

Close the Command Prompt window and check if you can install the Build update.
