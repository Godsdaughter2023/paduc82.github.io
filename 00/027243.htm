The behavior you are describing may occur if there are remnants of a third party program interfering with Windows Defender.  

I presume you did a restart of the computer?  

Go to Security and Maintenance, look in 'Security' and see what program is listed under Virus Protection.  Windows Defender will not be enabled and working properly if there is a program other than Windows Defender listed.

Think back to what other antivirus programs have been previously installed. Do you remember which AV product came pre-installed or was previously used on the computer? Maybe McAfee or Norton?  Try running the removal tools for any AV program that has ever been on the computer and then reboot and see if WD is working okay.  See List of anti-malware product removal tools

For further information see the reply of quietman7 in this thread: http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/windows-defender/e994f94f-92d5-4d44-9721-79c2a686ce24.

If the issue continues try the steps recommended in Problems starting Windows Defender in Windows 8/8.1/10.

Alternatively you can consider a reset which will essentially be a “clean install” of Win 10 but you’ll lose installed programs and data/pics/music that is on the computer. https://support.microsoft.com/en-us/help/12415/windows-10-recovery-options

Or - install a third party antimalware program (doing so should disable the real time protection of Windows Defender) and use the third party program for your antivirus protection.  There are a number of viable alternatives to Windows Defender - paid or free - take your pick.

The below example will give you the method to insert 'blank columns in between the columns multiple columns  totally 1000 columns'. First test it on a blank file.

Be careful with your formulas, when you are working on your main file. Preferably make a copy of the file.

tobias scheer say:

There is a workaround for that to make it run again w/ an existing OneDrive folder located somewhere else:

  1. Stop the application by right clicking the symbol in the taskbar and select "Close OneDrive"
  2. Navigate to the folder where the OneDrive folder is located in
  3. Back up your files contained in OneDrive if needed
  4. Delete the folder
  5. Click in the address bar of your explorer window
  6. Type "cmd" and press Enter/Return to open the Console
  7. Type "mklink /J OneDrive %SOURCE%" where %SOURCE% is the path which holds your correct OneDrive folder
  8. Start OneDrive

Good luck.
