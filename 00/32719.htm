bossdweeb say:

Because the one Page File is adequate . If your C:\ partition is short on space you could use one of the other partitions.

However, if C:\ is an SSD and the other partitions are on an HDD, C:\ would be the faster Page File.

Then again, putting a Page File on an HDD protects the SSD from unnecessary writes, which is proving to be maybe less of

a worry than the theories a few years ago that suggested SSD's would wear out quickly if too many writes are done to them.

If we do have to worry about writes to and SSD Win10's twice yearly version update overwrites will go a long way toward proving or

disproving that theory.

Experiment, and find what works for you. Chances are you will not notice a difference with any of the options.

The Page File is not nearly as important as it was in the day of 512MB RAM and less.

It's not much good setting aside good SSD/HDD space if the Page File does not make use of that space.

The 'best' Page File settings have long been debated. There was a time (XP) when 1.5X the installed System RAM for both

min and max was all the rage, and I rather doubt 2X would benefit over 1.5X or just 3GB for that matter.

Many of the people here suggest the default System Managed, which is as good as any on Win10.

System Managed is dynamic, so the Page File cannot run out of space (assuming the SSD\HDD has adequate reserves).

Having said that, I use 2GB min/max with 8GB (or more) System Ram, just for old times sake. :)
