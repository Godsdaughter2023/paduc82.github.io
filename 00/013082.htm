The error trap for this circumstance needs to be set at the form level as follows:

Option Compare Database
Option Explicit

Private Sub Form_Error(DataErr As Integer, Response As Integer)
   
    Dim strMsg As String
   
    Select Case DataErr
    Case 2113 'Invalid value entered
        Select Case Me.ActiveControl.Name
        Case "txtFullness"
            Response = acDataErrContinue
            strMsg = "You entered an invalid value for this field." & vbCrLf & vbCrLf _
            & "A decimal value is required."
            MsgBox strMsg, vbExclamation, "Error - Invalid Entry"
        End Select
    End Select
   
End Sub

quietman7 say:
I have not seen any ransomware with a specific .PMGTPSM extension. It could be something new or one of several variants which leave randomly named extensions. CTB-Locker, Crypt0L0cker, Maktub Locker, Alma Locker, Princess Locker, Locked-In, Mischa, Goldeneye and some Xorist variants all append a random 4, 5, 6, 7, or 8 character extension to the end of all affected filenames.

Uploading both encrypted files and ransom notes together at ID Ransomware provides a more positive match and helps to avoid false detections.

Samples of any encrypted files, ransom notes or suspicious executable's (installer, malicious files, attachments) that you suspect were involved in causing the infection can be submitted here with a link to this topic. There is a "Link to topic where this file was requested" box under the Browse... button. Doing that will be helpful with analyzing and investigating by our crypto malware experts.

When you discover that your computer is being infected with ransomware you should if possible, create a copy or image of the entire hard drive. Doing that allows you to save the complete state of your system (and all encrypted data) in the event that a free decryption solution is developed in the future. In some cases, there may be decryption tools available but there is no guarantee they will work properly since the malware writers keep releasing new variants in order to defeat the efforts of security researchers.

Imaging the drive backs up everything related to the infection including encrypted files, ransom notes and registry entries containing possible information which may be needed if a solution is ever discovered. The encrypted files do not contain malicious code so they are safe. Even if a decryption tool is available, they do not always work correctly so keeping a backup of the original encrypted files and related information is a good practice.
