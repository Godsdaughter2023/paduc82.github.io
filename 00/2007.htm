Two columns page set up is messing up Page numbers - Microsoft Community

orlando andrea say:

Two columns page set up is messing up Page numbers

Hello,

I wanted to use two columns approach for a business document that I am writing. I thought it would be fancier this way.

But it is messing up my page numbers. I thought it was about the sections. but when I delete sections it loses the two columns. Can anybody help?

simplest solution is to go back to one column per page.

Also the document has headings. Shall the two column approach be applied to headings as well? or just to text?

suzanne s. barnhill say:

Well, it is about the sections, but you can't avoid section breaks if you are using a mixture of multiple and single columns (if the entire document is in two columns, no section breaks would be needed).

The issue is that every section has a header and a footer (more than one each, if you've enabled Different First Page or Different Odd and Even), and page numbering can be restarted in any section, but you will see the page numbers only in the headers/footers that are visible in the document. If a section is "landlocked" (in the middle of a page), you won't be able to access the header or footer to deal with the page numbers; fortunately, however, you can access them directly.

Start by adding the section number to your status bar if it is not already displayed (right-click on the status bar and choose Section). This will permit you to see the section number at the insertion point. If you also add Formatted Page Number, you will see the number that would be printed on the page if there were a header/footer for that section.

You can then scroll from Section 1 to Section 2 and proceed forward until you find the section where the page numbering is messed up. Then use Insert | Header & Footer | Page Number | Format Page Numbers... to open the Page Number Format dialog, where you can reset the number from "Start at" to "Continue from previous section."

Public Function MicroTimer() As Double
Dim cyTicks1 As Currency
Static cyFrequency As Currency

If cyFrequency = 0 Then getFrequency cyFrequency ' Get frequency.
getTickCount cyTicks1 ' Get ticks.
'Result in Seconds
If cyFrequency Then MicroTimer = cyTicks1 / cyFrequency Else MicroTimer = 0
End Function

' Ticks are translated to MiliSeconds
'
Public Sub DelayMicro(Ticks As Long)
Dim StartTick As Double, lclMilis As Double

If Not DMinFl Then
DMinFl = True
lclMilis = Ticks / 1000
StartTick = MicroTimer
Do: DoEvents: Loop Until StartTick + lclMilis <= MicroTimer
DMinFl = False
End If

End Sub
