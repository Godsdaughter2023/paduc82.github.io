I have a Word document and a Visio document, and I would like to combine them both into one file.  The problem is that both files have internal links and links to the other.  The Word document needs to link to Visio pages, and the Visio document needs to link to Word bookmarks.

I was able to accomplish this by converting both files to html and combining them into a pdf, but this loses the Word formatting, which I would like to maintain.  I tried exporting the Word document to a pdf file directly and combining them as before, but the links from Visio to Word will only bring up the title page even after I was able to export the bookmarks as named destinations.

I also tried copying the Visio objects into Word, though the Visio links (which are bound to shapes and text boxes) no longer work at all.

barb way say:

Hi Winchella,

This is not a simple task, as you have already found out.  I can help you with a link method from Visio that will go to the specific Word document page.  That is accomplished by using Word Bookmarks.  In the Word doc, add Bookmarks (on the Insert tab) in the places in your doc where the Visio links should connect to.  Then, in Visio, edit the Hyperlinks by adding the Bookmark name in the 'subaddress' field.  This should cause Visio to open the doc directly to a specific location.  (You may also want to manually rename the Hyperlink to designate it better for identification - up to you.)

You can use Insert Object / From File, and then choose Link to File, to embed a linked Visio drawing in Word, which allows you to dbl-click on the drawing image in Word and open the Visio drawing in a Visio edit window.  Note, however, that the links will only be active in edit mode.  When you are in Word and simply viewing the Visio drawing, there will be no evidence of Hyperlinks.

Unfortunately, I don't know of a way to force the Word link to do page/shape selection, because it is aware of Word Bookmarks, but not Visio Pages/Shapes.  So Hyperlinks from Word simply open the Visio drawing with no finer selection options.

I have successfully created a linked document / drawing pair, using the steps above to select Word bookmarks, and the test works well for me.  Notice, of course, that going back and forth necessitates that each application is still responsible for editing it's objects contents. If you truly want to have only a single object as the outcome, and don't need to edit the contents, then the PDF method you already have may be better for you.

I hope that helps!

Barb

marshall barton say:

IIf is a Function, not a statement so it can be used in a query's calculated field.  Since it can be calculated from other values in the same record, it should NOT be saved in a table (so an UPDATE query is not helpful).

Whenever you need the x/y, just use a calculated field as Mimo suggested OR you can leave the query alone and display the x/y in a form/report text box by using the expression:

     =IIf(Column1= Column2, "x", "y")

scottgem say:

Dave,

First, you need to get away from the spreadsheet mentality. Access is not a spreadsheet, its a relational database and data structures are very different. 

An IF function in Excel is closer to the IIF (Immediate IF) function in Access, but still different. An IF statement in Excel is a VBA statement and is the same as in Access.

Second, as a general rule you don't store calculated values in Access tables. So using an Update query is the wrong approach. You do calculations in queries or on forms and reports, not in tables. The expression Mimmo gave you will work as a column in a query and can be used anyplace you want to display that result.
