>=67 percent means that the values must be at least 67% of the largest value, so with the above data 100 is the largest value and 67%*100 = 67 so if the value is >67 it will get a green ball (for example).  In the above example that means the last 6 entries.

<67 and >= 33% means that the value must live between 33 and 67% of the largest value, in the case where the largest value is 100 this translates nicely into values between 33 and 67.  In the above example no items will get a yellow ball.

<33 percent means the values will be less than 33% of the largest value, in this example 33%*100 = 33 so the first four entries get red balls.

From some preliminary tests it appears the thresholds are calcuated as % * (max - min) + min

Suppose you have a 3 icon set (red, yellow, green), 2 thresholds at 33% and 67%, and the data range from 1 to 14.  Then, the yellow icon will show for a value that is at least 33% *(14 - 1) +1 or 5.29 (and of course, less than the green threshold).  Similarly, the green icon shows for a value that is at least 67% * (14 - 1) +1 or 9.71

bernie deitrick say:
Then VBA is the way to go. Insert a standard code module into your workbook and copy the code below into it. then use it like so:

=ABCDSum(A1:A3)

Function ABCDSum(r As Range) As String
    Dim s As String
    Dim c As Range
    Dim i As Integer
    Dim j As Integer
    Dim k As Integer
    Dim v As Variant

    s = "ABCD"

    For i = 1 To Len(s)
        j = 0
        For Each c In r
            v = Split(c.Value, " ")
            For k = LBound(v) To UBound(v)
                If Right(v(k), 1) = Mid(s, i, 1) Then
                    j = j + Val(Left(v(k), Len(v(k)) - 1))
                End If
            Next k
        Next c
        ABCDSum = ABCDSum & " " & j & Mid(s, i, 1)
    Next i
End Function

If you have other letters of interest, then you can use this version, used like

=LSum(A1:A3,"ABCDF")

Function LSum(r As Range, s As String) As String
    Dim c As Range
    Dim i As Integer
    Dim j As Integer
    Dim k As Integer
    Dim v As Variant

    For i = 1 To Len(s)
        j = 0
        For Each c In r
            v = Split(c.Value, " ")
            For k = LBound(v) To UBound(v)
                If Right(v(k), 1) = Mid(s, i, 1) Then
                    j = j + Val(Left(v(k), Len(v(k)) - 1))
                End If
            Next k
        Next c
        LSum = LSum & " " & j & Mid(s, i, 1)
    Next i
End Function

2) When you configured you email account - you used "auto-discover" which defaults to IMAP instead of checking the box at the bottom of the first config window for "Manual setup or additional server types"

Does that sound about right? Suggest you have Outlook setup correctly before configuring Mailwasher to use the account as POP3. You can either add the same account as a POP3 in the same Outlook profile and once done - delete the IMAP account. When configuring the account as POP3, you will also have the opportunity to use your old PST file from your '2010 config instead of creating a new one.

Aside from all of the above - if you are going thru the MW <wash> process and the emails aren't getting deleted from the server, you should be posing the question to the MW (Firetrust) support folks since Outlook is only going to download what it finds in the <Inbox>.
