karl timmermans (outlook mvp 2012-2018) say:

First and most important thing to understand the word "difference" when it comes to <email clients> and <email accounts>.

Win10 Mail and Outlook (desktop) are two completely separate and distinct <email clients> and neither interacts with the other. HOWEVER...........................

An <email client> connects to an <email account> and if both <email clients> are configured to use the same <email account> then BOTH email clients will reflect the same email account data the next time the <email client> connects to the <email account> as follows

1) Exchange account - all items are sync'd with the server and in turn with every email client where that account is configured as an Exchange account.

2) IMAP account - only email items are sync'd with the server and in turn with every email client where that account is configured as an IMAP account.

3) POP3 account - the <Inbox> data is downloaded on a one-time one-way basis where Inbox items are deleted from the Inbox UNLESS the "Leave messages on server" option has been set (an option that only applies to a POP3 configured account

*** note - there is always only one email account on the server but it can be configured differently in different email clients. In other words, one email client may have it configured as an IMAP account and another as a POP3 account etc etc

how to add th st or rd to dates in excel - Microsoft Community

catherine215 say:

how to add th st or rd to dates in excel

I want to know how to format dates so that Monday, 1, October, 2012 ready Monday, 1st, October, 2012

 

Many thanks.

e.j. gun say:

If your actual date value is in cell A1, then the formula below (the main part of which comes from Chip Pearson's site) can be used to create the string you desire.  Note that A1 must contain a DATE value, and not just the string "Monday, 1, October, 2012".

 

=TEXT(A1,"mmmm") & ", "&DAY(A1)&IF(AND(MOD(DAY(A1),100)>=10,MOD(DAY(A1),100)<=14),"th",CHOOSE(MOD(DAY(A1),10)+1,"th","st","nd","rd","th","th","th","th","th","th"))&", "&YEAR(A1)

 

If A1 contains the literal string "Monday, 1, October, 2012", then use the formula below (a little more complex):

 

=LEFT(A1,FIND(",",A1,FIND(",",A1,1)+1)-1)&IF(AND(MOD(INT(MID(A1,FIND(",",A1,1)+2,FIND(",",A1,FIND(",",A1,1)+2)-FIND(",",A1,1)-2)),100)>=10,MOD(INT(MID(A1,FIND(",",A1,1)+2,FIND(",",A1,FIND(",",A1,1)+2)-FIND(",",A1,1)-2)),100)<=14),"th",CHOOSE(MOD(INT(MID(A1,FIND(",",A1,1)+2,FIND(",",A1,FIND(",",A1,1)+2)-FIND(",",A1,1)-2)),10)+1,"th","st","nd","rd","th","th","th","th","th","th"))&", "&RIGHT(A1,LEN(A1)-FIND(",",A1,FIND(",",A1,1)+1))

 

HTH,

Eric

jay freedman say:
When you download a file from the Internet or from some other computer and save it locally, Windows marks it as a "foreign" file. That is what triggers Protected View (unless you're careless enough to turn off a carefully hidden option).

In Windows Explorer, go to the folder containing the downloaded file. Right-click the file and click Properties. If you trust the source of the file, click the Unblock button in the Properties dialog and click OK. The next time you open the document, it should not be in Protected View (unless there's something else wrong with it).
