scottgem say:

Ok, So a booked performance consists of a Client and a Performance, correct? If so, looks like you are close to having a handle on it. 

I think your problem is the Booked checkbox. I'm guessing you don't clear it out after creating a booking record. So it appends any records marked booked. I would do two things. First, I would base the Subform on a Query. I would set criteria for PerformanceID column to:

NOT IN(SELECT [Performance ID] FROM Bookings)

This will filter out any performances already booked from the subform.

Next I would use the Double Click event of the form to run code like:

Dim strSQL As String

strSQL = "INSERT INTO Bookings ([Performance ID], [Client ID]) " & _

               "VALUES(" & Me.[Performance ID] & ", " & Forms!mainformname!controlname & ");"

CurrentDB.Execute strSQL, dbFailOnError

You need to substitute the name of the mainform and the name of control bound to the Client ID for controlname

The Double Click event can be triggered by clicking on the Record Selector just to the left of the record.

graham mayor say:
The macro recorder has many limitations. This is one of them.

Insert your watermark manually. Open the header view. Select the watermark (which is simply a graphic) and press ALT+F3. Save the autotext entry with the name CopyWatermark. (You can call it what you like, but you will have to change the macro to match).

Delete the watermark and close the header. Next create the following macro

Sub InsertCopyWatermark()
Dim oSection As Section
Dim oHeader As HeaderFooter
Dim oRng As Range
Set oSection = ActiveDocument.Sections(1)
For Each oHeader In oSection.Headers
    Set oRng = oHeader.Range
    oRng.Collapse wdCollapseEnd
               Application.Templates(ActiveDocument.AttachedTemplate.FullName). _
                    BuildingBlockEntries("CopyWatermark").Insert Where:=oRng, _
                                                              RichText:=True
Next oHeader
End Sub

which will insert the autotext entry in the headers of Section 1 of the document. If you want it in all headers of all sections then you will also have to loop through the sections

Dim oSection As Section
Dim oHeader As HeaderFooter
Dim oRng As Range
For Each oSection In ActiveDocument.Sections
    For Each oHeader In oSection.Headers
        Set oRng = oHeader.Range
        oRng.Collapse wdCollapseEnd
        Application.Templates(ActiveDocument.AttachedTemplate.FullName). _
                BuildingBlockEntries("CopyWatermark").Insert Where:=oRng, _
                                                             RichText:=True
    Next oHeader
Next oSection

doug robbins - mvp office apps & services (word) say:

I have a Microsoft Word 2010 file with links to several charts an Excel 2010 file.  When I open the Word document it opens and closes the document once for each chart.  I've tried to stop this by unchecking the "Update automatic links at open" option in the Advanced Options menu, but it still updates the links.  I also selected all of the linked charts and set link updating to Manual, and also locked them.  Word disregarded all of these settings and continued to update all the links the next time I opened the file.  Are there any other settings I should be aware of?


This is a problem that I do not think has ever been attended to by Microsoft (It has definitely been brought to their attention).

For a particular application that I developed that contained hundreds of links, it was necessary to replace links with DOCVARIABLE fields and then use VBA to update the values of the variables (for text type links)

Charts were inserted into a document by copying them to the clipboard and then using the following routine:

Dim Msg, Style, Title, Response

Dim iShape, LastShape As InlineShape

Msg = "Have you copied the chart that you want to insert to the Clipboard?"

Style = vbYesNo + vbCritical + vbDefaultButton2    ' Define buttons.

Title = "Insert Portrait Chart"    ' Define title.

Response = MsgBox(Msg, Style, Title)

If Response = vbYes Then    ' User chose Yes.

    Selection.PasteSpecial Link:=True, DataType:=wdPasteOLEObject, Placement:= _

        wdInLine, DisplayAsIcon:=False

    Set LastShape = Nothing

    For Each iShape In ActiveDocument.InlineShapes

        If iShape.Type = wdInlineShapeLinkedOLEObject And iShape.Range.End = Selection.Start Then

            Set LastShape = iShape

        End If

    Next iShape

    If Not LastShape Is Nothing Then

        LastShape.LinkFormat.AutoUpdate = False

        If ActiveDocument.Variables("chkColor").Value = True Then

            LastShape.PictureFormat.ColorType = msoPictureBlackAndWhite

        End If

    End If

Else    ' User chose No.

    Exit Sub

End If

 

And then when charts needed to be updated the following code was run:

 

DataSource = ActiveDocument.Variables("DataSource").Value

For Each alink In ActiveDocument.Fields

    If alink.Type = wdFieldLink Then

        Set linkcode = alink.Code

        i = InStr(linkcode, Chr(34))

        Set linktype = alink.Code

        linktype.End = linktype.Start + i

        j = InStr(Mid(linkcode, i + 1), Chr(34))

        Set linklocation = alink.Code

        linklocation.Start = linklocation.Start + i + j - 1

        linkcode.Text = linktype & Replace(DataSource, "\", "\\") & linklocation

        alink.Update

    End If

Next alink
