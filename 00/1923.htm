rich007a say:

So, to make the stacked column (you want a single column right?), you will have to add each row individually, so if your data is in cols A, B and C, and the headings are in row 1, data starting in row 2...

Set the Axis Label Range to B1  (Time)

Set the first series' Series name to be A2   (AVB)
Set the first series' Series values to be B2  (85)

Set the second series' Series name to be A3   (BDS)
Set the second series' Series values to be B3  (48)

etc.

[Edit: to do that the quick way: select cells A2:B10, Insert tab > Charts group > Column > Stack Column...  then right-click on the chart > Select Data...  and click the Switch Row/Column button.]

Now add another series using C1 (Avail Hrs) as the Series name, and C2 (1000) as the Series value.

Right-click on that 1000 block in the chart and Change Series Chart Type...

Set it to be Line.  It looks like it's not there, but because there is only one point it doesn't show up with a line... yet.

So next, select the Avail Hrs series (if you can't find it, use the dropdown in Chart Tools > Layout Tab > Current Selection to select it) and then Format Series/Selection.

Set it to Plot Series on the Secondary Axis.

Now go Layout tab > Axes group > Axes > Secondary Horizontal Axis > Show axis without labeling.

Then Layout tab > Axes group > Axes > Secondary Vertical Axis > None.

Job done?

ken sheridan say: Take a look at the file TimeSheet.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

This little demo file includes a number of functions for calculating time durations.  An important point to note when aggregating date/time values is that you should always aggregate the actual values first, and then format the result as Hours:Minutes, or whatever you want, as the final stage of the process.  

Under the skin a date/time value is no more than a 64 bit floating point number which counts days.  When you aggregate them the result will by default be expressed as a date/time value, so rather than getting a result in hours and minutes, if the total duration is 24 hours or more you'll get a date and time, which will be around the start of the 20th century because Access uses 30 December 1899 00:00:00 as the origin for its implementation of the date/time data type.  It is this which my functions cater for, allowing you to return a value as hours:minutes:seconds or optionally, days:hours:minutes:seconds.

Another point about my functions, and where they differ from some others, is that they allow for times which span midnight to be entered as time values rather than full date time values.  The code assumes that a start time which is later than an end time is on the previous day.
