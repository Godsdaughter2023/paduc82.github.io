All Outlook data is in the PST file. Outlook writes to the file whenever you make a change to an item or receive a new item. Take a look at these pages for info on Outlook data backup or transfer:
http://www.slipstick.com/config/move_outlook.asp
http://www.howto-outlook.com/Howto/backupandrestore.htm
http://office.microsoft.com/en-us/assistance/HA010771141033.aspx

Note that some of the information on the Microsoft page is outdated and contains errors, particularly when applied to Outlook 2003, 2007 or 2010. Specifically:

2. Ignore any advice that tells you to use export or import to transfer an entire Outlook data file. That has never been sound advice, but the process has become too deeply flawed to be trusted.

dirk goldgar say:
Ok so after checking the name of the textbox it was not the same name as the code. However, once I fixed this I then got a Error 2135 Property is read only and can't be set. This is in reference to the second line I mentioned in my first post the line Me![ONEPATH].Text=fileName so now I am confused on why I am getting this error.

The .Text property of a text box can't be set unless the control has the focus.  That's the purpose behind the code you posted:

    Me![THIRDPATH].Visible = True

    Me![THIRDPATH].Text = fileName

    Me![THIRDPATH].Visible = False

The code attempts to make the text box visible, set the focus to it, set its .Text property, and then set the focus elsewhere and make the text box invisible again.  The error message you just now got, about the property not being able to be set, suggests that for some reason the call to [your textbox name].SetFocus didn't work.  I don't know why that would be.

HOWEVER, there is no need to set the .Text property of the text box in this case.  You can set its .Value property (which is its default property) without ever setting the focus to it.  All of the code above can happily be replaced by just one line:

    Me![your textbox name] = fileName

Since you mentioned that "THIRDPATH" isn't the correct name of the text box, I didn't use that name in the above code, but you should replace "your textbox name" with the name of the text box.

abulkashim say:

how to highlight similar cells which contains same values on mouse over or mouse click on Excel?

how to highlight similar cells which contains same values on mouse over or mouse click on Excel?

jeeped say:
If you have a) cells with current highlighting and you want to b) override that highlighting in favour of a new selection-based highlighting, then c) revert to the original then that sub event macro is inadequate in its current form.

You will need a Worksheet_SelectionChange routine that is filtered to the D6:I30 range using Intersect and constructs a conditional formatting rule based upon the currently selected cell if the cell is within the D6:I30 range. If the currently selected cell is not within the D6:I30 range, then I'm guessing you want the highlighting removed. The conditional formatting rule will override any existing formatting and auto-restore the original formatting when removed of modified.

Try this,

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
    Me.Range("D6:I30").FormatConditions.Delete
    If Target.Count = 1 And Not Intersect(Target, Me.Range("D6:I30")) Is Nothing Then
        If Not IsEmpty(Target) Then
            With Me.Range("D6:I30")
                If IsNumeric(Target) Then
                    .FormatConditions.Add Type:=xlExpression, _
                      Formula1:="=D6=" & Target.Value
                Else
                    .FormatConditions.Add Type:=xlExpression, _
                      Formula1:="=D6=" & Chr(34) & Target.Value & Chr(34)
                End If
                .FormatConditions(.FormatConditions.Count).SetFirstPriority
                .FormatConditions(1).Interior.ColorIndex = 3
            End With
        End If
    End If
End Sub

r. h. wilkinson say:
I recently encountered the same issue noted in this question, and I may have a process which helps to resolve the issue. I only spent a few minutes working on the issue myself, so there is a high probability that this process could be refined to fewer steps. (I am using OneNote for Windows 10.)

Steps:
  • Select the title text which is wrapping incorrectly, and copy that text to the clipboard.
  • Paste that text into a Notepad window.
    • Since Notepad is a plain text editor which does not support formatting markup, this serves two purposes: it is a temporary repository for the original title, and it strips any formatting markup from that text.
  • Select and copy the title text of another Note which does not exhibit the undesired behavior.
    • Make sure that you include the invisible "carriage return" marker (at the end of the title) in your selection. It can be observed when it is highlighted, but it is not normally visible otherwise.
  • Select the title (including carriage return) of the Note with the issue, and paste.
    • At this point, the issue should be absent.
  • Return to your Notepad entry to retrieve your original title text, and paste it where it belongs.

Also note that just selecting the carriage return by itself for copy/pasting does not appear to resolve the issue.

If you've encountered this issue, please try these steps, and feel free to report back if you are able to refine the process.
