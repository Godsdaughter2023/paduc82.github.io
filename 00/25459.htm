jay freedman say:
In the Insert Symbol dialog, leave the Symbols tab selected (not the Special Characters tab).

In the top left, set the Font dropdown to "(normal text)".

Here's the important part: In the lower right, if the "from" dropdown says "ASCII (decimal)" or "ASCII (hex)", change it to "Unicode (hex)".

Then the Subset dropdown will appear in the top right corner of the dialog. Set that one to "Greek and Coptic". Then you'll find the entire Greek alphabet in the character area.

Word supports a number of link formats with Excel workbooks, including:

a. user say:

If I have understood correctly,

 a. you have an Excel cell that contains a hyperlink

 b. you want to make an updatable Word LINK to that cell's contents

If so, I think what you will need is to 

 a. make a copy of the Hyperlink *in Excel* using a suitable EXCEL formula. Just as an example, if you have a hyperlink in cell B6 you could put the following in B7:

=HYPERLINK(CONCATENATE(LEFT(B6,10),UNICHAR(8204),RIGHT(B6,LEN(B6)-10)))

Then copy B7 and paste *that* link into the Word document - the No-Width Optional break (character 8204) will still be there and the link still appears to work.

a. user say:

Still not sure I have understood, because your original message mentioned adding No-Width spaces to solve a URL wrap problem, but your later post suggests it's the additional paragraph marks that are causing problems.

To summarise: I do not think there is any really simple way to do what I think you are trying to do.

However, if you are able to take the general approach I suggested on splitting the URL, the question is how to eliminate paragraph marks created by the LINK field.

The only way I can see that might do it and that preserves a clickable hyperlink is a set of fields along the following lines, assuming your workbook is at c:\mywbs\mywb.xlsx, and your *split* link is in cell B2 in sheet Sheetx

{ SET thelink "{ LINK Excel.Sheet.12 "c:\\mywbs\\mywb.xlsx" Sheet1!R2C2 \a \u }" }{ HYPERLINK "{ thelink }{ SET "keepme{ SEQ keepseq }" 1}" }

The LINK field brings in the data from sheet1, cell B2 in the workbook. The \u switch specifies Unicode data. This link on its own would not generate an extra paragraph mark, but would not be clickable either. 

Ideally we could just nest it inside a HYPERLINK field, but that doesn't work either - you just get blank. So we have to set a bookmark to the result of the LINK field, then nest that inside a HYPERLINK field. Finally, we have to insert a { SET } field to create a unique bookmark withint the hyperlink field to stop Word deleting or resolving the content of { thelink" field when you close and reopen the document.

It is difficult to imagine that Microsoft could not make this easier, but I think they need to be persuaded by someone with some commercial clout.

michaels---01 say:

As a guide for the future, if you want to remove a photos from a particular album, find it in the album concerned, press and hold the image then select Remove from album, do not select Delete.﻿ The delete option is always acting on the original copy of the image and removes it from the device altogether.

 

Albums in Symbian phones are not folders, they are databases. This allows the possibility for photos to be included ion more than one album without having multiple copies of the same file occupying space in the device memory. For example, if you create an album called "Family" and add all photos of your family to it, you are not actually moving the images to a different folder, merely telling the album database that these photos should be shown when you select the family album. It is possible that you have another album called "Holiday 2011" which also contains some of the same images of your family members, but you only ever have one copy of the actual file stored on your phone.

 

You cannot remove images from the "All photos" view, this feature will always display all compatible image files wherever they are stored in the phone (all folders of the phone memory, mass memory and memory card) - it too is just calling image data from a database.
