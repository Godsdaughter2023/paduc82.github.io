COnverting multiple pages in a single Word document to individual - Microsoft Community

j taylor 500 say:

COnverting multiple pages in a single Word document to individual pages

I have a Word document that contains 12 seperate job descriptions, each is a seperate page within the document.  Is there a quick way to save each page as it's own document??
suzanne s. barnhill say:
Word MVP Doug Robbins has frequently posted the following "splitter" macros for this purpose, as follows:
 
Sub splitter()
'
' splitter Macro
' Macro created 16-08-98 by Doug Robbins to save each page of a document
' as a separate file with the name Page#.DOC
'
Selection.HomeKey Unit:=wdStory
Pages = ActiveDocument.BuiltInDocumentProperties(wdPropertyPages)
Counter = 0
While Counter < Pages
    Counter = Counter + 1
    DocName = "Page" & Format(Counter)
    ActiveDocument.Bookmarks("\Page").Range.Cut
    Documents.Add
    Selection.Paste
    ActiveDocument.SaveAs FileName:=DocName, FileFormat:= _
        wdFormatDocument, LockComments:=False, Password:="",
AddToRecentFiles:= _
        True, WritePassword:="", ReadOnlyRecommended:=False,
EmbedTrueTypeFonts:= _
        False, SaveNativePictureFormat:=False, SaveFormsData:=False, _
        SaveAsAOCELetter:=False
    ActiveWindow.Close
Wend

End Sub

or to split into [documents] by sections rather than pages, use:

Sub splitter()

' splitter Macro

' Macro created by Doug Robbins to save each letter created by a mailmerge
as a separate file.

Dim i As Long, Source as Document, Target as Document, Letter as Range
Set Source = ActiveDocument
For i = 1 to Source.Sections.Count
    Set Letter = Source.Sections(i).Range
    Letter.End=Letter.End-1
    Set Target = Documents.Add
    Target.Range=Letter
    Target.SaveAs FileName:="Letter" & i
    Target.Close
Next i

End Sub

If you do not know what to do with these macros, see ttp://www.gmayor.com/installing_macro.htm

bob jones aka: cybertaz say:

First let me emphasize that there are never any iron-clad guarantees... There are a number of variables that could be involved any time older files are opened by newer releases of the software -- even on the same operating system.

That point having been made, in general Office 2011 should have no problem with files created by Office 2003 apps. The default file types used by Win Office 2003 are the same as those used by Mac Office 2004 & are still supported by the newer releases of Office.

What you will find is that the newer versions offer different feature sets not supported by the old binary file types. In most cases it will be preferable to save new versions of the files in the current OOXML file formats (e.g., .docx rather than .doc, .xlsx rather than .xls) in order to take full advantage of what the new version has to offer. The OOXML file types also provide other significant advantages over the former binary formats.

phillip m jones say:

There may be differences in VBA and Macro here is a quote from a fellow I am well acquainted with. This is an exact copy of a portion of a previous answer this person gave to another person. Although it applies to Word, I suppose some could apply to Excel as well.

If this doesn't apply don't worry:

"Word for Windows supports controls [fields] that are not supported in the Mac version. There is nothing you can do to change that. In order for forms to work in both Mac & PC the only type of controls used must be what are referred to as Legacy Form Fields, so the document would have to be redesigned with cross-platform compatibility in mind. Apparently the document uses either Content Controls or ActiveX Controls.

Also, the Windows versions of Word support document protection features that the Mac versions lack. If those features have been used Mac Word will simply see the document as "protected" & not allow any changes whatsoever.

Additionally, not all macros which run on a Windows PC will run on a Mac & vice-versa. It depends on the commands written into the macro & what it is intended to do."

Courtesy of Bob Jones (Cyber Taz)"

daniel g. grau say:
I would say that if some language choices in the Tools menu are grayed out, Word actually cannot find some components it needs. Some of them should be present in the following folder:

  • Applications > Microsoft Office 2011 > Office > Shared Applications > Proofing Tools
Verify that you have a whole bunch of files named "LANGUAGE_NAME.proofingtool" in there. If you do not, you will need to reinstall (or restore them by copying the folder from another computer). Also check carefully that the folders have not been renamed (or moved, if you cannot find them).
