Microsoft Word 2003 - Programming for Different Languages - Microsoft Community

debraann say:

Microsoft Word 2003 - Programming for Different Languages

I have created a template with macros and special toolbars.  One of the macros requests the user (in a popup) to list the section their document is supposed to start with (e.g., 3.2.5). The program then takes the three digits and creates the outline by assigning each heading as follows: 

Heading 1 = 3; Heading 2 = 2; Heading 3 = 5

This issue I am having is that both English PCs and German PCs are using the template.  The German PC apparently has a different name for Heading 1, Heading 2, etc.; therefore, they receive an error when the outline tries to link to the headings.  If they change their PC temporarily to English, it works fine.  They would like me to fix it so it will work no matter what the language the PC is assigned to.  Is there a way to do this? 

jay freedman say:

On Mon, 23 Aug 2010 20:51:18 +0000, DebraAnn wrote:

I have created a template with macros and special toolbars.  One of the macros requests the user (in a popup) to list the section their document is supposed to start with (e.g., 3.2.5). The program then takes the three digits and creates the outline by assigning each heading as follows: 

Heading 1 = 3; Heading 2 = 2; Heading 3 = 5

This issue I am having is that both English PCs and German PCs are using the template.  The German PC apparently has a different name for Heading 1, Heading 2, etc.; therefore, they receive an error when the outline tries to link to the headings.  If they change their PC temporarily to English, it works fine.  They would like me to fix it so it will work no matter what the language the PC is assigned to.  Is there a way to do this? 

If you're using an expression in your macro like

   ActiveDocument.Styles("Heading 1")

then change it to

   ActiveDocument.Styles(wdStyleHeading1)

and similarly for the other two styles. The wdStyle values are numeric constants that are independent of the language in use.
