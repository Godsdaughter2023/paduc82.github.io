john korchok say:

Normally TMP files save to C:\Users\YourUserName\AppData\Local\Temp. They do not delete themselves. In this location they also build up until there are thousands of them, but you don't see it, so it's not so much of an issue. Since you have other machines that do this, I would suspect an errant group policy resulting in bad registry settings.

PowerPoint does create work files that delete themselves. These are always in the same folder as the actual file and are named the same as the file, but preceded with ~$ before the file name. They are always hidden files, so you wouldn't see them unless you modified your Windows view settings.

Assuming that the first filter drop-down needs to be hidden:

ActiveSheet.Range("A1").AutoFilter Field:=1, Criteria1:="FilterValue", VisibleDropDown:=False

In case you want to hide drop-downs for more than one column then make change the field (column) numbers in succession.

john sr wilson say:
I misunderstood the problem. You need to use the TextFrame2 object but in 2007 this is not fully implimented in the object model. You would need to come at it by checking if the paragraph was selected I think.

What about:

Sub RemoveIndents()
    Dim oTR    As TextRange
    Dim oshp   As Shape
    Dim x      As Long
    Dim i      As Integer
    Dim otxF2  As TextFrame2
    Set oTR = ActiveWindow.Selection.TextRange
    Set oshp = oTR.Parent.Parent
    oTR.ParagraphFormat.Bullet.Type = ppBulletNone
    Set otxF2 = oshp.TextFrame2
    For i = 1 To otxF2.TextRange.Paragraphs.Count
        If isSelected(oTR, oshp.TextFrame.TextRange.Paragraphs(i)) Then
            otxF2.TextRange.Paragraphs(i).ParagraphFormat.FirstLineIndent = 0
            otxF2.TextRange.Paragraphs(i).ParagraphFormat.LeftIndent = 0
        End If
    Next i
End Sub

Function isSelected(oTR As TextRange, opara As TextRange) As Boolean
    If opara.Start < oTR.Start + oTR.Length _
       And oTR.Start < opara.Start + opara.Length _
       Then isSelected = msoTrue
End Function

dhookom say:

You shouldn't need to use DLookup() since the Row Source already contains all the values you need. I created a combo box (cboEmpID) bound to the Employees table in Northwind and added a text box to enter the last name. The after update event loops through the combo box items looking for a match in the second column. This code took the last name like "Fuller" and correctly updated the combo box value.

NOTE: This could/should be encapsulated in a more generic public function for use everywhere.

Private Sub txtLastName_AfterUpdate()
    Dim cbo As ComboBox
    Dim intRowCount As Integer
    Set cbo = Me.cboEmpID
    cbo = Null   'set to no value
    For intRowCount = 0 To cbo.ListCount - 1
        If Me.txtLastName = cbo.Column(1, intRowCount) Then
            Me.cboEmpID = cbo.Column(0, intRowCount)
        End If
    Next
End Sub
