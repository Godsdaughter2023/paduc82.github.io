Transferring data from Outlook Express to Windows Live Mail:

For messages:

Copy the *ENTIRE* OE message store folder to a flash drive. (Folders.dbx must be included). Place this on the Desktop or other location on the machine using WLMail. Open WLMail and: File | Import | Messages | Microsoft Outlook Express 6 and point to where you saved it.

OE Message Store Location:

Open the Address Book in OE and File | Export | Address Book (wab) and save it to the Desktop. Copy to a flash drive. Place this on the Desktop or other location on the machine using WLMail.

Open the Contacts list in WLMail, (Go | Contacts on the Menu Bar), and File | Import | Windows Address Book (wab) and point to where you saved it.

franktekstar say:

Joe,

There are two types of backup that are built-in to Windows 10. One type is called "Backup and Restore (Windows 7)" and the other type is called "File History". Both of these are accessed through Control Panel > System and Security.

It sounds like you are trying to do the "Backup and Restore" type of backup. This backup type allows you to backup all of your data files and also gives you the option of creating a System Image backup (all of the files and programs on your drive). If you are trying to use a DVD for this type of backup, you will probably need a LOT of DVDs, because each DVD can only hold about 4 GB and a full backup of data files plus system image is likely going to be 100 GB (or more). So the first piece of advice I would give you is to use an external hard drive instead of DVD. In my system, I have an internal 500 GB hard drive and I separately purchased a 1 TB USB 3.0 hard drive for backups. Recently, I replaced the 1 TB drive with a 4 TB NAS (network accessible storage) that is part of my home network and so I am using the NAS for backups. This frees my PC (a Dell XPS18 all-in-one) from having to hang the external drive off of a USB port and lets me use my machine as the portable device it is.

On the Backup and Restore screen there is also an option to "Create a system repair disk". This particular backup should be made onto a DVD and you only need to do it one time. The system repair disk will give you a DVD that you can boot from if your system's internal hard drive gets so trashed that it won't boot.

The other backup type (File History), is only needed if you are really paranoid about losing your data files (like me) and you want to be able to recover a specific version of a file. The "Backup and Restore" backup only backs up my data files once a week. The File History backup allows you to backup any data files that have changed in the last 10, 15, 20, 30 minutes or 1, 3, 6, 12, 24 hours. My strategy is to backup files that have changed in the last day. I use my computer for work and want to be able to recover all of my files back to (worst case) the state they were in yesterday. Doing both types of backups will accomplish this for me. But if you can live with losing no more than the last 6 days for your data files, then you do not need to use the File History backup type - the weekly Backup and Restore backup with system image will do the trick.

Hope this helps.

kgladstone say:

I think I finally solved this!!!!!

(At least with my setup: Surface Pro 3, running Windows 10, in a docking station, connected to three external Dell monitors, all daisy chained via DisplayPort / Mini DisplayPort).

This Reddit thread suggested changing monitor refresh from 59 to 60 Hz, and that worked for me!

https://www.reddit.com/r/Surface/comments/3ror4p/surface_book_dock_issues_monitors_flickering/

To do this on Windows 10:

Right-Click Desktop | Display Settings.
Click one external monitor in the diagram.
Click advanced display settings.
Click Display adapter properties.
Click Monitor tab.
Change refresh rate from 59 to 60 Hz and click OK.
Repeat with other external monitors.
