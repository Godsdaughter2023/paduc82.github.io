Because you used the disk version of Windows and not the recovery!  Like I said before, these tablets do not have the chipsets on the motherboard.  The disk version has generic drivers for "function" of the system and tend to cause problems when attempting to getting the factory drivers that are needed (example: The Intel graphics driver may not install because the Windows Display Adapter "WDA" installed instead).

Now, sometime in the future, and OEM disk of Windows will install and all drivers will install correctly once the driver repository base has been completely updated.

Normally, you should have been able to install the OEM version and relied completely on Windows Update for all the drivers.  To get a failure means the driver base is not complete yet in the Windows Update repository.

 If the laptop is powering off, it may be overheating. Consider checking the event logs (Event Viewer in Administrative Tools) to see if there are any noteworthy events.

Consider updating your BIOS, as well, if possible.

Also, check the behavior with a clean boot.

Can you check that any fans are working, and are clear of dust?   Heat sinks should be clear of dust as well.  You aren't using the laptop on a soft surface, are you?  Try using it on top of something like a cooling pad, if you have one, or something that can keep it elevated a few inches above the surface of a table (but still relatively "open" underneath).   If you keep the laptop cool(er than "normal"), does that affect the problem?

Some laptops also have a removable filter that traps dust, and should be periodically checked and cleaned.

Usually the error code 0xC004F213 comes when there is a hardware change done on the computer (like Motherboard or Hard drive). The Windows license is associated with the device hardware. If there is any significant hardware changes, such as replacing motherboard or hard drive, Windows won’t be able to recognize the license.

I suggest you to try the troubleshooting steps from the article

If you are using an OEM computer that came with Windows 7, and the motherboard has not been changed, then you can follow the steps to retrieve the product key that came with your OEM computer. To retrieve the product key follow the steps mentioned below:

  1. Type Command prompt in the search bar, that is on the taskbar.

  2. Right click on Command prompt icon and select Run as administrator.

  3. Type the command and press on Enter: wmic path SoftwareLicensingService get OA3xOriginalProductKey

  4. You will find the product key.

Hope it helps.

Hi,


We have three bug checks:


INTERNAL_POWER_ERROR (a0)


This bug check indicates that the power policy manager experienced a fatal error.


BugCheck A0, {c, ffffffffc000000e, ffffe000053ea580, 0}


The 1st parameter of the bug check = c which the power policy manager experienced a fatal error.


  1. Click the Start button.
  2. In the search box, type command prompt.
  3. In the list of results, right-click Command Prompt, and then click Run as administrator.
  4. In the Command Prompt window type:

powercfg /hibernate /size 100


Restart afterwards.


DRIVER_POWER_STATE_FAILURE (9f)

This bug check indicates that the driver is in an inconsistent or invalid power state.

BugCheck 9F, {4, 12c, ffffe0000678a880, fffff801bfb94860}

1st parameter = 0x4, which implies that a power IRP has failed to synchronize with the PnP Manager. We cannot debug this without a kernel-dump.

KERNEL_DATA_INPAGE_ERROR (7a)

This bug check indicates that the requested page of kernel data from the paging file could not be read into memory.

BugCheck 7A, {4, 0, ffffe00005c73390, 889fbe1bd0}
