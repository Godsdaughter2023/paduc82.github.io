Hi Yleclerc,

Reinstallation of the Office desktop applications is required for after switching to another subscription.

However, apart from asking the users to do it manually, you can also use the Office Deployment Tool (ODT) to deploy Office for your users in an easier way. Here are the general steps to do the deployment:

1. Download the Office Deployment Tool and configure the deployconfigfile.xml file. Note: in the XML file, you can use the <Remove> tag to remove the current installation and use the <Add> tag to add a new installation.
2. Place the files of the Office Deployment Tool (including a setup.exe file and the deployconfigfile.xml) on a shared network place which is reachable for the users.
3. Running a command on the user’s computer to start the installation. You can write the command in a script file ask the users to run the script. If you have Active Directory on your local environment, you can also run the command via Group Policy.

1. Deploy Click-to-Run for Office 365 products by using the Office Deployment Tool
