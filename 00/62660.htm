a. user say:

The only way I can think of that Windows Update history would be empty (and the other described symptoms) is if you ever followed the popular advice to "Reset Windows Update Components" (or words to that effect) perhaps while troubleshooting a problem with Windows Update.

That process involves stopping a couple Windows Services and renaming/deleting the %SystemRoot%\SoftwareDistribution folder then restarting the Services or the system which rebuilds the content of the folder which will not have the WU history.

Does any of that sound familiar?

One of the consequences of following that advice is that the WU history will be wiped out but the list of installed updates remains intact which you have already noticed.

It is too bad that when that idea is suggested most folks will not mention the consequence - perhaps because they have never followed the advice themselves to see what happens.

Short of performing a System Restore to a date/time prior to the renaming/deleting of the SoftwareDistribution folder I can't think of a way to restore or retrieve the WU history.

     In certain editions the default language packs will be listed in Windows Update again

ramesh.kumar say:

Hi,

If it is getting detected by other computer as storage device and you still unable to access it. This means you have encrypted your data using TPM.

A Trusted Platform Module (TPM) is a microchip that is built into a computer. It is used to store cryptographic information, such as encryption keys. Information stored on the TPM can be more secure from external software attacks and physical theft. As there is a dedicated microchip already available in the computer to safeguard the encryption keys, so the BIOS battery does not store any password.

During the startup process, the TPM releases the key that unlocks the encrypted partition only after comparing a hash of important operating system configuration values with a snapshot taken earlier. This verifies the integrity of the Windows startup process. The key is not released if the TPM detects that your Windows installation has been tampered with.

Your data is protected by encrypting the entire Windows operating system volume. BitLocker uses the TPM to lock the encryption keys that protect the data. As a result, the keys cannot be accessed until the TPM has verified the state of the computer. Because the keys needed to decrypt data remain locked by the TPM, an attacker cannot read the data just by removing your hard disk and installing it in another computer. And, so if you remove your Hard Disk Drive (HDD) from the system, you may still not be able to use it without the key (stored in TPM).

Additional information:

BitLocker Drive Encryption Overview
