Text with highlight color: selecting and copying - Microsoft Community

glyn parry say:

Text with highlight color: selecting and copying

Hi,

In a Word 16 document, I have highlighted in color, particular sections of text.

Can i select for 'highlight' to extract just those highlighted words and phrases ie copy and paste?

thanks

*****Post moved by the moderator to the appropriate forum category.*****

suzanne s. barnhill say:

Yes, you can do this:

  1. Press Ctrl+H to open the Replace dialog.
  2. Select the Find tab.
  3. Click More>> to expand the dialog.
  4. Leaving the "Find what" box empty, click Format and select Highlight. You'll see "Format: Highlight" below the "Find what" box.
  5. Click Find In and select Main Document.
  6. All the highlighted text will be selected.
  7. Click on the title bar of the document to return focus to it.
  8. Press Ctrl+C to copy the selected text, then Ctrl+V to paste it where needed. Each selection will be in a separate paragraph.

Note that this will copy all highlighted text, not distinguishing highlight color, and that the text will be pasted with the highlight.

Excel autofill giving wrong numbers. - Microsoft Community

garyschwartz66 say:

Excel autofill giving wrong numbers.

Enter the number 4 in cell A1

Enter the number 4.1 in cell A2

Select A1 and A2, and drag the handle down to autofill 4.2, 4.3, etc.

Cell A16 is 5.49999999999999 instead of 5.5

This turns autofill into autofail. What's up with that?

joeu2004 say:

This is a common problem, due to the way that Excel represents numbers internally (64-bit binary floating point).   Work-arounds.....

1. Enter 4 into A1, then enter the following formula into A2:

=ROUND(A1+0.1,1)

Copy A2 down the column.  If you wish, copy A2:A1000 (whatever you choose for the last row of column A), then paste-special-value into A2:A1000.

2. Use the Fill Series feature instead of dragging down the column.  Enter 4 into A1, select A1, click Fill > Series > Columns, set Step Value to 0.1 and Stop Value to 100 (or whatever you want), then click OK.

PS....  Technically, method #1 is more reliable if you are building a "table" for comparison with calculated values later.  For example, with method #2, MATCH(ROUND(A24,1),A24,0) returns #N/A (no match!) even though we cannot see any difference due to Excel formatting limitations.
