julie fka profjulie say: Select "Do Nothing"  Then manually address the issue on each device.  Doing nothing will do exactly that...leave things just they way they are so you can go figure out how you wish to proceed.

To avoid conflicts, I tend to only make changes on one device at a time...For instance, I do not make changes to the same contact/task/appointment on both my computer and my windows mobile device.  Changes made to the same item on both the computer and the windows mobile device do not get combined automatically and having to go back and figure out what I did is too confusing for me.
Prof Julie, Microsoft MVP Mobile Devices | http://pocketprimer.com

staceyag say: Do nothing is not an option - I have to select either keep the record from the phone or keep the record from the computer.  Any other ideas?  Otherwise, I may look at returning the phone!  I cannot risk losing data.

julie fka profjulie say: If you've made a change only on the computer, the changed record will be synced to your windows mobile device (overwriting the record that had previously been there); if you've made a change to a record only on the windows mobile device the changed record will be synced to your computer (overwriting the record that had previously been there).  You will only encounter a syncing conflict when you've made changes to the same record on both your computer and your device.  For instance, if you add a phone number to John Doe's contact record on the windows mobile device and then add a fax number to John Doe's record on the computer.  Because the same record was changed on both devices, WMDC will experience a conflict and that's when it will either replace the item on the device or on the computer based on how you set up the option with WMDC.  This really shouldn't happen very much.

I went back and reviewed and compared the syncing options between ActiveSync (used with XP computers) and WMDC (used with Vista computers) and ActiveSync includes an option to "Do Nothing," along with the "Replace on Computer" & "Replace on Device" options, but WMDC does not include the "Do Nothing" option.

Confusion between Incremental Sync and Initial - Microsoft Community

ashishsrkhan say:

Confusion between Incremental Sync and Initial

I have got a confusion between Incremental sync and Initial one

Suppose I have got 100 mailboxes and I create an imap batch. Does that mean in Initial sync it will migrate max 20 mailboxes and thereafter once every 24 hours it will pickup other 10.

If that's true is it going to take 8 days to migrate all my 100 mailboxes.  

Also at what speed does it upload the data on Microsoft Exchange online

yang zheng msft support say: Hi Ashishsrkhan,

The answer to your first question ‘does that mean in Initial sync it will migrate max 20 mailboxes and thereafter once every 24 hours it will pick up other 10’ is no. Your understanding about initial sync and incremental sync in IMAP migration is not correct. By default, initial synchronization is performed on 20 mailboxes at a time until all mailboxes in the migration batch have been migrated. There is no 24 hours’ time span between the first 20 mailboxes and the second 20 mailboxes in the initial synchronization.

Until all the mailboxes are migrated into Office 365, the incremental sync starts. By default, incremental synchronization is performed on 10 mailboxes at a time until all mailboxes in the migration batch are synchronized. And the incremental sync process will repeat every 24 hours.

Then about the second question ‘what speed does it upload the data on Microsoft Exchange online’, since there multiple factors that will affect the IMAP migration such as the network, the mailbox size, there is no exact uploading speed for IMAP migration.

Regards,
Yang

zorvek say:

Place this macro in a general code module in the workbook, select the cells to clean, and run the macro "RemoveInitials". 

Public Sub RemoveInitials()

    Dim Cell As Range
    Dim Tokens As Variant
    Dim Index As Long

    For Each Cell In Selection
        Tokens = Split(Cell.Value)
        For Index = LBound(Tokens) To UBound(Tokens)
            If Tokens(Index) Like "?." Then
                Tokens(Index) = Empty
            End If
        Next Index
        Cell.Value = Trim(Replace(Join(Tokens), Space(2), Space(1)))
    Next Cell

End Sub

To add VBA code to a regular or general module in an Excel workbook, press ALT+F11 to open the VBA development environment (VBE). Select the menu command Insert->Module to create a new VBA module. Paste the code into the document window that appears. Press ALT+F11 to return to the Excel workbook.

To run a macro from Excel, press ALT+F8. A dialog box appears listing all available macros. Find the desired macro and select it. Click the Run command button to start the macro.
