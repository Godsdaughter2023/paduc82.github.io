I need assistance with archiving. pst/data file? Beginner - Microsoft Community

don krogstad say:

I need assistance with archiving. pst/data file? Beginner needs help

 I am using Outlook 2010 and am trying to archive to a home network drive. After numberouse attempts (and numerous created data files) I see my inbox is approaching its 1gig limit. What do I need to do differently? Thanks in advance!
brian tillman [outlook mvp 2007-2019] say:

Click File>Account Settings>Account Settings, then select the Data Files tab.  There you will see all of the data files corrently connected with your mail profile.

brian tillman [outlook mvp 2007-2019] say:

The agent was trying to get you to uncheck which option?

If you could archive, yes it would relieve the "pressure" on your mailbox.  Outlook's archive function doesn't work with IMAP however.

You have a couple of choices.  Since an IMAP data file cannot be the default data file, you will already have a local data file containing your Calendar, Contacts, Tasks, Notes, and other default folders.  You can copy your Charter data to that data file or you can create a new one specifically for the purpose of containing the older Charter messages.

To make the new data file, click New Items>More Items>Outlook Data File.  You can allow the new data file to reside in the folder where Outlook creates data files by default (Documents\Outlook Files) or you can change its location to any folder you want.  I'd use the default folder.  You can accept the file name Outlook wants to choose or you can pick another.  I'd pick one that reflects the purpose of the data file, like "Charter Archive.pst".  Click OK and the data file will be added to your Navigation Pane.

Now, whether you want to use your existing non-IMAP data file or the new one, in that data file, create a folder whose name matches the name of one of your folders that also appears in your Charter folder tree.  RIght-click the root of the data file and choose New Folder.  Specify the name that matches the IMAP folder you want to save and press Enter.  Now, click the IMAP folder you want to save so that it opens and you see all the messages it contains.  Select the messages that you want to move off the server.  Ctrl+A will select them all.  Now, drag that selection to the corresponding folder you created in the data file and release the mouse button.  Outlook will copy the selected messages from the IMAP server to that folder and then delete them from the server.  If you wish, hold Ctrl while you drag the selection to make it a copy instead of a move so you can verify all of the messages have been copied prior to deleting the messages from the server.

Continue in this fashion until you've moved all of the messages you wish to keep but clear from the server.  You'll have your messages, but they won't take up space on the server.  You'll be able to see them only in Outlook and not if you log into the mailbox using a web browser.  Since you're running out of space, this is what you want anyway.

An alternative to all of this is to create yourself a Google Mail IMAP or Hotmail account using the Outlook Hotmail Connector.  Then, instead of moving your messages to a local data file, you can store them in the cloud in either Hotmail or Google Mail.  The transfer process is nearly the same.

Office 365 installed - Open Word 2010 by default - Microsoft Community

larry harper jr say:

Office 365 installed - Open Word 2010 by default

We are running both Office 2010 and 365 on our workstations and wish for Word 2010 to be the default programs for the traditional Microsoft Word extensions.  

Changing the 'default program' in the right click menu and in the Control Panel simply returns the default to 365 even after clicking the correct application.  

Is there a way to change this default association with 365 installed?  

suzanne s. barnhill say:

You'll need to reregister Word 2010 explicitly, using the full file path. By default (for 32-bit Office 2010), this will be C:\Program Files (x86)\Microsoft Office\OFFICE14\WINWORD.EXE. To do this:

  1. Press Windows Key + R to open the Run box.
  2. In the box, paste the full path to the Office 14 version of Winword.exe (as above).
  3. Type quotes around the path and add the /r switch, resulting in
    "C:\Program Files (x86)\Microsoft Office\OFFICE14\WINWORD.EXE" /r
  4. Press Enter. You'll get a message that Windows is reconfiguring.

If you will ever be opening Word 2013, you need to edit the Registry to keep the two programs from reconfiguring every time and grabbing back the file associations. See the "Hacking the Registry" section of the article at http://www.gmayor.com/Toolbars_in_word_2007.htm.

suzanne s. barnhill say:

Reregistering Word should have no effect on Excel. You can reregister Excel similarly, however; in that case the command is "C:\Program Files (x86)\Microsoft Office\OFFICE14\EXCEL.EXE" /regserver.

If that doesn't do the trick, you'll probably be better off posting in the Excel forum.
