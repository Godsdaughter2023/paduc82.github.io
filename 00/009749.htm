Weighted average of what?

I suppose you mean:  the weighted average % on-time deliveries.

I assume that "% of on-time delivery" is relative to the number of deliveries for each department, not to the total deliveries of 20 departments.  For example, if a department made 100 deliveries and 20 were on-time, the statistic would be 20%.

In that case, I don't believe you have the correct information to compute the weighted average %on-time delievers correctly -- unless "%volume" is really "%deliveries".  (I interpret it as %units or %dollars sold.)

Assuming that the sum of %volumes is 100% (i.e. the 20 departments represent the total volume), you might want to compute the following weighted average:

=SUMPRODUCT(B2:B21,C2:C21)

formatted as Percentage, where column B is %volume for each department and column C is %on-time deliveries of the deliveries for each department.

However, I believe the following example demonstrates the result is erroneous.

Consider that just 3 departments comprise the total volume.  We might have:



C6 is computed by =SUMPRODUCT(B2:B4,C2:C4), per above.

But E6 is the correct overall %on-time deliveries.

Column F represents %total deliveries for each department.  Then the overall %on-time can be computed by the following weighted average in F6:

=SUMPRODUCT(F2:F4,C2:C4)

[EDIT] The difference in weighted averages arises because, for example, deptA has many small deliveries (e.g. online orders to individuals) whereas deptC has a few large deliveries (e.g. corporate orders).

Matthew

jeeped say:
Because your desired returned value is in a column to the left of the lookup column, you will not be able to use VLOOKUP(). Use an INDEX(MATCH()) pair of functions instead.

=IFERROR(INDEX(T:T, MATCH(C1, U:U, 0)),"not found")

You will get speedier results if you define the number of rows to look through like this,

=IFERROR(INDEX($T$1:$T$400, MATCH(C1, $U$1:$U$400, 0)),"not found")

Those formula will look in column U for a matching value from C1 and return the corresponding value from column T.

I suspect there hasn't been anybody chiming in because there aren't a lot of other options that people have identified. However, you also haven't given very much information:

  • What font is it, what kind of font is it, and where did you get it from?
  • Where did you install it?
  • Did validating/weeding find any damaged fonts?
  • Did you fully update Office after you reinstalled it?
  • Does font book give that font a clean bill of health?
FWIW, reinstalling applications on a Mac is almost always a complete waste of time. Applications just don't corrupt. Reinstalling is a Windows technique that often works because the installer often rewrites the Registry, where most corruptions occur. On the Mac platform, it's usually preference files that get corrupted, and those are usually not rewritten by installers.

If you don't find any damaged or duplicate fonts, you should try running Word in a new user account - if you don't see a problem in the new account, you've likely got a preference or Normal template problem.
