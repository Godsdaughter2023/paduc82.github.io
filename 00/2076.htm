hansv mvp say:

The line

afolder = SelectedItems(1)

should be

afolder = .SelectedItems(1)

and the line

Set ofolder = fso.GetFolder(ofolder)

should be

Set ofolder = fso.GetFolder(afolder)

Sample macro:

Sub GetWorkbooks()
    Dim fso As Object
    Dim ofolder As Object
    Dim ofile As Object
    Dim afolder As String
    Dim wb As Workbook
    With Application.FileDialog(4) ' msoFileDialogFolderPicker
        If .Show Then
            afolder = .SelectedItems(1)
        Else
            MsgBox "No folder specified!", vbExclamation
            Exit Sub
        End If
    End With
    Set fso = CreateObject("scripting.filesystemobject")
    Set ofolder = fso.GetFolder(afolder)
    For Each ofile In ofolder.Files
        If LCase(Right(ofile.Name, 4)) = "xlsx" Then
            Set wb = Workbooks.Open(Filename:=ofile.Path)
            ' do something with the workbook
            ' ...
            wb.Close SaveChanges:=True ' or False if you don't want to save
        End If
    Next ofile
End Sub

No it does not matter that LeaseID (personally I would call it VendorID) does not show on the form. It doesn't even have to be hidden on the form.

A blank vendor record is hopefully not really blank: at least the LeaseID would be populated as soon as one character is entered into the parent form (at least if LeaseID is an autonumber). Temporarily add the field to see this in action.

If you then click in the subform, the parent record will be saved. You can now enter data in the subform. As soon as you do, the linked field from the parent form (LeaseID) will be populated. If the Locations table has an autonumber, it will also be populated. Continue to enter other fields and click the record selector or click outside of the record to save it.

If at that time you receive the error you reported, carefully read the error message and interpret it. "...they would create duplicate values in the index, primary key, or relationship..."

Design the table and click the Indexes button. Check the unique indexes on the Locations table. Check the primary key. Perhaps you have an index over the wrong combination of fields. For example if I had a unique index on LocationName, there can be only one "Phoenix" in the entire database, while you probably meant one Phoenix per LeaseID. If so, change the index to be over two fields.
