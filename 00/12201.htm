patrickfreer say: After a day of working this out I have a solution.

The problem was I was using a self-signed certificate from SBS2003. The problem being that my E50 phone doesn't support security certificates stores, only CA Certificates (and SBS doesn't like to give one).

You can tell by looking at your exported certificate (make it the DER format with .cer extension). If it has the fields "Basic Constraints" and "Key Usage" then the phone will import it. If not, you can't.

My solution was this:

Deleted the SBS certificate from all stores.

Installed Certificate Authority Services on SBS

Generated a new Cert Request from ISS (direct to new CA)

Used this certificate for the OWA listener

Exported THE CA CERTIFICATE (i.e. the one called SERVERNAME not www.yourcompany.com) from the CA Root folder in certificates

This one shows the "Basic Constraints" and "Key Usage" fields. This DER format .cer extension file CAN be installed onto the phone via file transfer then running it.

Hope this helps,
Patrick, MiDSYS (UK)
