Hello!

Is your Gateway the Technicolor TC8717T ? Or perhaps the TC8715D? No worries it does all this to protect the console from harm and there is a fix. To be more precise there is 3 known fixes for this gate way and always the option to swap it out. Your console is not broken. You can go wired, Change the encryption or have TWC roll back the last firmware update.

How to Change Wireless Security

1.Open a web browser.
2.Log into the router by entering the default gateway into the address bar of the internet browser. (Example http://192.168.0.1/ )
3.The gateway will then prompt for a user name and password.  The default is the user name “admin” and the password is “password”.
4.From the menu on the left select Gateway.
5.From the submenu on the left select Connection.
6.From the next submenu select WiFi.
7.Under Private WiFi Network select Edit on the Network you wan to use.
8.Select the "Show More Security Options"  (allow the pop up)
9.Select WPA TKIP (about the 6th option from the bottom)
10. Press Apply. (confirm the prompt about WPS no one used that and it is kind of insecure)
11. Select Save.

Give this a go and tell me how it goes.

Sorry for the issue with your headsets. Here is a list of troubleshooting to try out. Some you may have done already and some you may not have. Number 9 seems to be a temporary solution to this ongoing issue.

7.) Make sure the controller is not full of dust in the headset port (you can use compressed air to clear the dust). 

8..) Make sure controller is fully charged/ change battery type. 

9.) Use a headset adapter or standard headphones (Main fix) 

10) Soft or full console reset. (may not work all the time)

dax1792 say:

Administrator accounts on Windows 7 run with the same privileges as Standard User Accounts until they are elevated, usually by means of a UAC prompt.

Correctly set up programs should contain a manifest which notifies Windows if they require elevation. This will cause a UAC prompt to appear when the program is launched. Older programs can be started with Administrator privileges by right clicking them and selecting 'Run as administrator'.

 

There a lot more on this in http://technet.microsoft.com/en-us/magazine/2009.07.uac.aspx
