Why is the saved emails stored in the drafts disappearing?

¡firedog say:

The "Drafts" and subfolders underneath have disappeared 3 times.

I can only think of one reason for this: the account(s) concerned are Microsoft Email accounts (e.g. @outlook, @hotmail, @msn, @live) set up to use HTTP, and your Windows Live Mail has been updated to build 16.4.3563.0918 or 16.4.3564.1216.

In these circumstances, the accounts will not have a drafts folder. This is because the update enables support for Exchange ActiveSync (EAS), and the version of EAS in use for some of these accounts does not allow for synchronization of the drafts folder. Saving a draft will now place the saved message in the Drafts folder under Storage folders.

You can check which build of the program you have by pressing Alt-F B when viewing your message list.
