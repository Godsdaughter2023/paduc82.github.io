If each subfield ends in a "]", then I suggest you create a Public Function to return the individual parts:

 

Public Function GetPart( N As Integer, S As String) As String

   GetPart = Trim(Split(S, "]")(N)) & "]"
End Function

 

Then you can use a query like:

 

SELECT field, GetPart(0. field) As Part1, GetPart(1. field) As Part2, GetPart(2. field) As Part3, ...
FROM table

tomnation say:

Original title: text formatting

When breaking a hyphenated word at the end of a line, the hyphen should stay on the first line.  It should not lead off the new line, as I am seeing happen with IE9.

In fact this is the more proper way of handling a word (or compound word, name, or number) with a natural hyphen within, and the hyphenation at the end of the line occurs at that internal hyphen. Consider the compound word editor-in-chief. The rule to hyphenation is to break at the end of a syllable, add a hyphen, then continue on the next line. So to break at the end of "in" one would type "editor-in" then ADD a hyphen "editor-in-" then continue on the next line, which would START with a hyphen "-chief."
