a. user say:

I don't think there is a good way built in to XP so some manufacturers have programs you can use to make some sort of recovery CDs.  Many manufacturers don't even ship you an XP installation CD or your system make not even have a CD/DVD drive.  So what are you going to do?

If you have a genuine bootable XP installation CD that is not the same Service Pack as your installed SP (most people don't), you will not be doing a Repair install of your XP either.  if you have an older XP CD, you can make a SP3 installation CD through a process called slipstreaming  and that's a good thing to have (but won't help you all of the time).

I used to like making bootable XP Recovery Console CDs (no XP media required) but sometimes you just can't do what you want or need to do and I started liking Hiren's Boot CD better since it seems to work all the time and not just some of the time.

One day I wrote down some of the reasons why I switched:

The bootable XP Recovery Console CD made from xp_rec_con.iso is okay for some things but it may not work well on some systems - especially those with SATA hard drives.  That is because it will not contain any SATA drivers and you might be able to boot on it, but you may not be able to maneuver and run even simple commands. 

A good sign is that when you boot on the CD instead of being in the C:\WINDOWS folder where you need to be, you will see that you are in the C:\ folder (wrong) and when you enter commands you may see things like:

enumeration error
Access denied (or something similar)

If you try to run bootcfg /rebuild you might see:

Error:  Failed to successfully scan disks for Windows installations.
        This error may be caused by a corrupt file system, which would prevent Bootcfg from   successfully scanning.  Use chkdsk to detect any disk errors.
Note:     This operation must complete successfully in order for the /add or /rebuild commands to be utilized.

These are all signs of a system with a SATA drive and the Recovery Console won't help you much since there are no native SATA drivers built in so it can't understand the HDD interface.

You might be able to get it to behave by fidgeting with the system BIOS to temporarily change the HDD interface to something other than SATA, do what you have to do and then remember to change it back when you are done.  Even if you use a genuine bootable XP installation CD to get into the Recovery Console, you still might run into these problems (no natively built in SATA drivers) so it just can't understand the HDD.

Another problem is that unless you have previously tweaked your system registry to automatically log you in to the Recovery Console, sometimes folks can't get past the prompt for Administrator password when using Recovery Console no matter how hard they try.   That can be a problem with or without SATA drives.

Unless you have previously tweaked your registry, you will not be able to access a floppy, CD, USB thumb drive or any of your personal folders from the RC either and you will not be able to get into your personal folders like My Documents, etc.  You can only get into the Windows folders that are allowed and that ain't many - mostly just the stuff under the C:\WINDOWS folder and C:\

With those variables in the equation, I started to like the Hiren's Boot CD better since is handles SATA drives just fine and doesn't prompt for a password.  It also brings up a Windowsy looking desktop that people feel more comfortable using instead of the RC command line interface.  They are already used to "exploring" folders and files and copying and pasting things.

Hiren's has many good built in tools too - like a Registry Restore Wizard that lets you roll back your registry (just the registry) to a previous date based on the System Restore points already on your system.  Just a few clicks and you're done.  It is much simpler than trying to do that from the Recovery Console command line interface where you can easily make mistakes.

If you need to edit your boot.ini file, there is a text editor built in, etc.  Supposedly, you can even get on the Internet if you need to but so far when fixing things, I never need to so I haven't tried that part out.

For most problems, people just need to run a chkdsk with error correction, maybe replace a file they have accidentally deleted or perhaps they want to copy all their personal files off to some external USB drive and all that is easy using Hiren's.  There are lots of other tools on that CD too and I have not used them all but they are fun to look at.

So if somebody has a problem using the Recovery Console or it doesn't like their commands, or they can't get past the Admin password prompt or they just need to replace a file or two, I like Hiren's better since it always seems to work no matter what the environment.  Fewer questions to ask, fewer 'I'm not sure' replies to wonder about, not so many suggestions that "might work" or suggestions that begin with the word 'try' and less back and forth messaging.

For systems that have no CD/DVD drive, a Recovery Console CD is not going to do you much good either.  Some folks will say to go out and buy an external CD/DVD drive, but you can take the Hiren's and make a bootable USB thumb drive too.  I could never get that to work well with xp_rec_con.iso so I  gave up trying to figure it out - too many steps.

I say:  You might be able to do what you need to do with the Recovery Console, but you will always be able to do what you need to do with Hiren's.  I do like to have both though (old fashioned I guess).

Go ahead and make one and check it out - you don't have to "do" anything...  just boot on it and poke around.  If you build the Hiren's CD and check it out you will probably find it very useful. 

So if somebody just needs to run a chkdsk with error correction, here is what I tell them... 


If you are not sure what you have for bootable media, from a working system, make yourself a Hiren's Boot CD, boot on that into the Mini XP mode and run the chkdsk from there. 

Here's how:

Make yourself a Hiren's Boot CD which you can download from here:

http://www.hirensbootcd.org/download/

The ZIP file is large, so the download will probably take a little while to complete bit it is worth it.  Then unzip the download to extract the Hirens.BootCD.ISO file that will be used to create your new bootable CD.

Creating a bootable CD from a .ISO file is not the same as just copying the .ISO file to a blank CD.  You have to use software that understands how to burn a .ISO file to a CD to create a bootable CD.

In the Hiren's ZIP file are the BurnToCD.cmd file that you can double click to launch it, but I have never used it and prefer to use ImgBurn.

If you need a free and easy CD burning software package, here is a popular free program:

http://www.imgburn.com/

When installing ImgBurn, DO NOT install anything from UniBlue or the Ask toolbar (never install the ASK toolbar for anything).

Here are some instructions for ImgBurn:

http://forum.imgburn.com/index.php?showtopic=61

It would be a good idea to test your new bootable CD on a computer that is working.

You may need to adjust the computer BIOS settings to use the CD ROM drive as the first boot device instead of the hard disk. 

These adjustments are  made before Windows tries to load.  If you miss it, you will have to reboot the system again.

When booting on the Hiren's CD you will see a menu of options.  Choose the Mini XP option.  Then it will appear that Windows is being loaded and you will be presented with a desktop that has the look and feel of the Windows Explorer interface you are already used to using.

Remember:  You did not boot on your hard disk - you booted into the Hiren's desktop.

  1. Press Windows logo key + X, select Command Prompt(Admin).
  2. Run the "sfc /scannow" command by copy/pasting it into the Command prompt(Admin) and hit Enter key.
  3. Once it complete, try to install updates again to see if they work correctly

For more information about the System File Checker tool, see

Step 2: Run the DISM tool

If the problem continues, use the Deployment Image Servicing and Management (DISM) tool to fix corruption errors. To do this, follow these steps:

  1. Press Windows logo key + X, select Command Prompt(Admin).
  2. Run the "DISM.exe /Online /Cleanup-image /Restorehealth" command by copy/pasting it into the Command prompt(Admin) and hit Enter key
  3. Once it complete, try to install updates again to see if they work correctly.

Important: When you run this command, DISM uses Windows Update to provide the files that are required to fix corruptions. However, if your Windows Update client is already broken, use a running Windows installation as the repair source, or use a Windows side-by-side folder from a network share or from a removable media, such as the Windows DVD, as the source of the files. To do this, run the following command instead:

DISM.exe /Online /Cleanup-Image /RestoreHealth /Source:C:\RepairSource\Windows /LimitAccess

Note: Replace the C:\RepairSource\Windows placeholder with the location of your repair source. For more information about using the DISM tool to repair Windows, reference
