phillip m jones say:

Yes. For 2011 and lower they are located in the Computer Main Library → Fonts in a Subfolder called Microsoft  Ofice Fonts.  Or just the initials MS Fonts. 

2016 and above they Are located Home → User → Library either Container or Group Containers for each application.  In other words, there is a separate copy of fonts for each office application. So for 201l, there is one copy of fonts that work with Word, Powerpoint, Excel about 60 fonts. In 2016 there 3 copies times 60 or 180 fonts.

In 2016 and above You would have to hunt down each copy and remove it in order not to use it in any of Office Applications.

italophile say:

The gallery only shows List Styles not ListTemplates. A List Style sits above and is linked to a ListTemplate. The code below gives an example of how you can do this.

Public Sub CreateListHeadings(ByRef docTarget As Document)
'*******************************************************************************
'*  Sub:    CreateListHeadings
'*  Purpose:
'*
'*  Date         Ver   Author          Detail
'*  -----------  ----  --------------  -----------------------------------------
'*  05-Jul-2015     1  Timothy Rylatt  Created baseline
'*******************************************************************************
    Dim styNewList                  As Style
    Dim ltNewList                   As ListTemplate
    Dim lngLL                       As Long

Set styNewList = docTarget.Styles.Add(Name:="ListHeadings", Type:=wdStyleTypeList)

  .Name = "ListHeadings"
  'remove existing numbering
  For lngLL = 1 To 9
      With .ListLevels(lngLL)
          .NumberStyle = wdListNumberStyleNone
          .NumberFormat = ""
          .TrailingCharacter = wdTrailingNone
      End With
  Next
  For lngLL = 1 To 4
      With .ListLevels(lngLL)
          .NumberStyle = Word.WdListNumberStyle.wdListNumberStyleArabic
          .TrailingCharacter = Word.WdTrailingCharacter.wdTrailingTab
          If lngLL > 1 Then
              .NumberPosition = (lngLL - 1) * CentimetersToPoints(0.6)
              .TextPosition = lngLL * CentimetersToPoints(0.6)
              .TabPosition = lngLL * CentimetersToPoints(0.6)
          Else
              .NumberPosition = 0
              .TextPosition = CentimetersToPoints(0.6)
              .TabPosition = CentimetersToPoints(0.6)
          End If
      End With
  Next
  With .ListLevels(1)
      .LinkedStyle = docTarget.Styles(wdStyleHeading1).NameLocal
      .NumberFormat = "%1"
  End With
  With .ListLevels(2)
      .LinkedStyle = docTarget.Styles(wdStyleHeading2).NameLocal
      .NumberFormat = "%1.%2"
  End With
  With .ListLevels(3)
      .LinkedStyle = docTarget.Styles(wdStyleHeading3).NameLocal
      .NumberFormat = "%1.%2.%3"
  End With
  With .ListLevels(4)
      .LinkedStyle = docTarget.Styles(wdStyleHeading4).NameLocal
      .NumberFormat = "%1.%2.%3.%4"
  End With
  .ListLevels(5).LinkedStyle = docTarget.Styles(wdStyleHeading5).NameLocal
  .ListLevels(6).LinkedStyle = docTarget.Styles(wdStyleHeading6).NameLocal
  .ListLevels(7).LinkedStyle = docTarget.Styles(wdStyleHeading7).NameLocal
  .ListLevels(8).LinkedStyle = docTarget.Styles(wdStyleHeading8).NameLocal
  .ListLevels(9).LinkedStyle = docTarget.Styles(wdStyleHeading9).NameLocal
End With

ProcExit:
Set styNewList = Nothing
Set ltNewList = Nothing

suzanne s. barnhill say:

In addition to what #aakkam has said, there are several other steps you can take:

  1. At File | Options | Save, check the box for "Save to Computer by default."
  2. At File | Options | Save, check the box for "Don't show the Backstage when opening or saving files." With this option enabled, pressing Ctrl+S or clicking the Save button (in a new, unsaved document) will take you directly to the classic Save As dialog.
  3. At File | Options | Save, you can also specify the default local file location. When you first save a document, Word will default to this folder (unless you have selected a different folder during the current Word session).

bill manville say:

I would create an additional column to aid with the sort.

 

If the first project number is in H2 then the formula in row 2 would be
=IF(ISERROR(VALUE(LEFT(H2,4))),VALUE(LEFT(H2,3)),VALUE(LEFT(H2,4)))

 

That will give the project number alone.

You can then sort by this new column.

 

You could hide the column after doing the sort

Or you could do the sort using a macro that creates the additional column, does the sort and then clears the additional column
