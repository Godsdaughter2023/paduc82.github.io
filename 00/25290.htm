Sub CheckFolder(objFolder As Object)
  Dim objFile As Object
  Dim objSubfolder As Object
  Dim strFolderPath As String
  strFolderPath = objFolder.Path
  If Not Right(strFolderPath, 1) = "\" Then
    strFolderPath = strFolderPath & "\"
  End If
  ' *** New code ***
  r = r + 1
  wshOut.Range("A" & r) = strFolderPath
  ' *** End of new code ***
  ' Loop through files
  For Each objFile In objFolder.Files
    r = r + 1
    wshOut.Range("A" & r) = objFile.Path
  Next objFile

  For Each objSubfolder In objFolder.SubFolders

bob jones aka: cybertaz say:
Just as clarification -- When you AutoMark the index entries it really doesn't matter where the cursor is... Cursor position only matters when you actually generate the index, itself.

Now -- There are 2 primary causes for that message to appear:

  • Everything in the Concordance file has already been marked in the document. Make sure you have the non-printing characters (¶) displayed. With them revealed you'll see that { XE } codes have been inserted in the document following each item to be included in the index. E.g., if the term investing is to be included you'll see;
Infrastructure We are investing{ XE "investing" }in technology systems designed to keep the Company competitive in future years.

If this is the case you need only place the cursor in the right location then go to Insert> Index & Tables - Index - OK.

  • The 2nd possibility is that the Concordance file does not comply with the required form. In that case Word doesn't know what to mark so no { XE } fields will be inserted. Verify that the Concordance file consists strictly of a single 2-Column Table & that the terms are correctly entered, no blank cells, etc.
