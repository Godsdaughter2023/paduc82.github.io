Thank you. As it turns out outlook.com has been going through an update. We received an e-mail noting that this was causing "synch" problems. Since others may find this thread, I thought I would add a few more things to do.

To correct the synching problems you had to reconnect with the outlook, which is in itself very simple. Please follow the "official Microsoft" guidelines for doing this which can be found at

Remove an account: https://support.office.com/en-us/article/Remove-an-email-account-from-Outlook-1fa900ae-6dc8-468c-b754-10292f7ee47a

Add an account: https://support.office.com/en-US/article/Add-your-Outlook-com-account-in-Outlook-for-Windows-642c1902-bdd9-4dc3-abe7-76d60b148b23

I would always recommend you save a copy of your ost file. it can be found in

C:\Users\username\AppData\Local\Microsoft\Outlook

One note If you delete and read your account- and you rely on your autofill addresses you will lose these as well. To prevent this before you delete the account go to

C:\Users\username\AppData\Local\Microsoft\Outlook\RoamCache

and copy and rename the file that looks like this :

Stream_Autocomplete_0_6DD74B923F25D64FA4C27CAA06509D8B.dat

After you have added back your account send at least one email to recreate the file. Then exit outlook.

go back to C:\Users\username\AppData\Local\Microsoft\Outlook\RoamCache and find the Stream_Autocomplete file. Copy exactly the file name (Include the .dat) Then rename it. Go to your original file you renamed (your old one) and change its name to the new one you just copied. You can restart outlook and you should have all your old autofills!

Please note I am an enthusiast not an expert, so all I know is what works for me!

jllatham say:

You didn't answer HansV MVP's question about which cell needs the date.

Below is some code that you can adapt that should help you get started and may even provide a "final" solution once you make a couple of changes to it.  This is worksheet event code and it assumes that for a given invoice, there is at least one cell you MUST ALWAYS type an entry into when starting a new invoice.  It also assumes that there is NO FORMULA or other entry in the cell that holds the date for the invoice when you start a new one; this means delete the formula in that cell in your template file if you're using a template file, otherwise it means delete an existing date before making the mandatory entry if you are creating new invoices from old invoice files.

As I said, this is worksheet event code, to put it into the workbook in the proper place,:
Open your template workbook,
Select the invoice sheet
Right-click on the invoice sheet's name tab and choose [View Code] from the popup list
Copy the code below and paste it into the code module presented to you
Change the addresses for the 2 cells defined by Const values to fit the layout of your invoice sheet.
Close the VBA editor and try it out.

Private Sub Worksheet_Change(ByVal Target As Range)
  'set this cell address to one that a person would
  'type new information into when first starting a
  'new invoice
  'Be sure to use the $ symbols in it
  Const monitoredCellAddress = "$A$1"
  'set this cell address to the one that should
  'hold the date of the creation of the invoice
  Const invoiceDateCellAddress = "F3"
  'this tests to see if a change has been made
  'in the monitored cell
  'and checks to make sure no date is already in
  'the invoice date cell
  If Target.Address = monitoredCellAddress And _
   IsEmpty(Range(invoiceDateCellAddress)) Then
    Range(invoiceDateCellAddress) = Now()
  End If
End Sub

jeeped say:

In addition to the excellent suggestions from ringerxyz you could clear your %TEMP% folder.

Tap the Windows key+R to bring up the Run dialog. Paste in %TEMP% and click OK. This should open whatever folder is specified by your TEMP environment variable. With the folder options allowing Show hidden and system files, permanently delete (Shift+Delete, not just Delete) all the files you can in that folder. There may be several that are in use and cannot be removed but it is very likely that you have thousands of old temp files comprising a few gigabytes.

Checking your available drive space is another good suggestion.

stefan blom say:

To restore missing items in a gallery, such as the TOC gallery, try deleting the Built-In Building Blocks.dotx file in your user profile. Exit Word before doing that.

Alternatively, just make use of the Table of Contents dialog box (click References | Table of Contents | Custom Table of Contents). This gives you more control in general.

Also, using the Table of Contents dialog box inserts the TOC field directly without putting it inside a content control, which often means improved performance.
