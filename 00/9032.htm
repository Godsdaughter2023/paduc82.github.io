donleverton say:

Tom has a very good point when he asks if the Form's Recordsource (This query as per the SQL you have posted) has been changed since you first created the form.

I know it's common practice to create a query, and then use that query as the Recordsource for the form. In fact almost every book out there suggests that we do this.

I believe the issue arises when you start changing the query (adding fields, changing criteria, sort order, etc.) AFTER the form has been created. My theory is reinforced by your "but as of yesterday" statement in your original post.

In order to avoid this from happening, I have gotten into the habit of creating the query, making sure that it returns the expected results ... and then copying/pasting the query's SQL string directly into the Form's Recordsource in the property window. From that point forward, the form will always show the desired results if you modify or even delete the query.

I have also acquired the habit of making a copy of the original object (table, query, form, report, etc) and then using that to experiment with.

That way, if thing should go horribly wrong (and they have) ... I have the original working version to go back to / compare with.

I also see that you're using Access 2013.

I am NOT a fan of the fact that the combo-box wizard now uses MACROS instead of VBA Event Procedures. I quite often create my own Event Procedures and delete the macro. This allows me to set breakpoints and debug all code in the Immediate Window.

HTH,

Don

charles kenyon say:

The MacroButton Field prompt  cannot handle more than one line of text. This is true in all versions of Word since Word 97.

Workarounds I can think of:

  • Shrinking the font size of the prompt text (which can have different formatting from the field itself). If you do this, text typed in will not have the smaller size.
  • Taking a screenshot or other graphic image of your multi-line text and using that image as the MacroButton prompt. Using an image for the button is an idea as old as the field itself, I believe.

See also MacroButton Prompt Add-In which demonstrates using prompt text which is formatted differently.

as good as the MacroButton field for your purposes but may meet your needs.

A further method would be to have the macrobutton field actually call a macro that would display your instructions in a message box.

{ MacroButton HelpTextMacro [Click here for instructions] }

Then have a macro along these lines.

Private Sub HelpTextMacro()
' help text called by macrobutton field
'
MsgBox Prompt:="Help text paragraph 1" & vbCr & vbCr & _
        "Help text paragraph 2" & vbCr & vbCr & _
        " Help text paragraph 3", Title:="Instructions"
End Sub

See Installing Macros for information about how to insert this.

Of course this requires that the document be macro enabled and used in an environment where the macros can be run.
