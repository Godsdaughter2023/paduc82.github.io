andreas killer say:

I'm sure that the load instructions for the variables X and Y are the same in both lines and the maschine code has finally a assembler CMP (compare) instruction.

 

The NOT from VBA is the same instruction in assembler.

 

The compare operators are JE (jump if equal) or JNE (jump if not equal) in assembler.

 

So theoretically the IF NOT THEN has one more instruction in the maschine code (and need some CPU-ticks longer, around 0.005 seconds for one million NOT codes).

 

Andreas.

 

Option Explicit

'The QueryPerformanceFrequency function retrieves the frequency of the high-resolution performance counter, if one exists.
Private Declare Function QueryPerformanceFrequency Lib "kernel32" (cyFrequency As Currency) As Long
'The QueryPerformanceCounter function retrieves the current value of the high-resolution performance counter, if one exists.
Private Declare Function QueryPerformanceCounter Lib "kernel32" (cyTickCount As Currency) As Long

Function MicroTimer() As Double
  'return containing seconds uses Windows API calls to the high resolution timer
  Dim Ticks As Currency
  Static Frequency As Currency
  MicroTimer = 0
  ' get frequency
  If Frequency = 0 Then
    QueryPerformanceFrequency Frequency
    If Frequency = 0 Then Exit Function
  End If
  ' get ticks
  QueryPerformanceCounter Ticks
  ' calc seconds
  MicroTimer = Ticks / Frequency
End Function

Sub Test()
  Dim i As Long
  Dim x As Integer, y As Integer
  Dim T As Double
  T = MicroTimer
  For i = 1 To 1000000
    If Not x = y Then Stop
  Next
  Debug.Print MicroTimer - T
  T = MicroTimer
  For i = 1 To 1000000
    If x <> y Then Stop
  Next
  Debug.Print MicroTimer - T
End Sub

Hi

Thank you for your interest in Windows 10 Technical Preview.

I can understand that you are facing issue installing KB3016725. I suugets you the following methods to fix this:

Method 1

Using the SFC /SCANNOW Method

System File Checker is a utility in Microsoft Windows that allows users to scan for and restore corruptions in Windows system files. Perform System File Check (SFC), and then check if this fixes the issue

1. Press the Windows button + X on your keyboard, a mini dialogue box will appear
2. Choose Command Prompt (Admin). You must run the command prompt in Administrator level
3. On the command prompt type sfc /scannow and hit enter
4. If everything went okay and no errors where found, restart your computer and check if the issue reoccurs.

Method 2

Manually reset the Windows Updates Components.

a) Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

b) Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

net stop wuauserv

net stop cryptSvc

 net stop bits

 net stop msiserver

c) Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

d) Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

e) Type Exit in the Command Prompt to close it.

Now you may try running the Windows Updates and install the new build and check if the above steps resolve the issue.
