john - project say:

Karen IBS365,

Like Trevor I cannot replicate your re-calculated work value most likely because you are using something other than the Standard calendar.

However, I think your first mistake is in using manual scheduling. Had you used auto-scheduling, your fixed work task would have kept the 120hrs of work even after you corrected the errant duration.

Whether manually or auto scheduled, Project first spreads the work over the entire 120d duration resulting in 1hr per day for the resource. The difference is that when you correct the 120h duration to the intended 20h, for a manually scheduled task Project will keep the 1hr per day and truncate the extra 100 hrs work whereas for an auto-scheduled task, Project will honor the fixed work parameter and keep the 120 hrs of work while re-spreading that work over the 20d duration. If you have your options set for scheduling alerts, you will also see an alert that allows you to confirm the change (see screen shot).

Hope this helps.

John

bob jones aka: cybertaz say:
I'm afraid what Cody suggested has no bearing at all on the default formatting of Charts. The thread he linked to pertains only to making a global change to the default font used in the text of new documents.

The worse news is that there really is no provision for doing what you would like... at least not within Word/Office 2008 or 2011. As you're probably aware, you can specify the default font for documents & styles in a custom template, but not for objects such as Charts & SmartArt.

The default fonts & colors for those objects is determined by the Document Theme -- Each Theme uses a specific Font Scheme & Color Scheme. You can select from the available Font Schemes, pick a built-in Color Scheme, then save that combination as a Custom Theme. Using PowerPoint you can create a Custom Color Scheme, but there is no facility for creating a Custom Font Scheme -- You're still restricted to using one of the built-in Font Schemes which dictates the Theme Headings & Theme Body fonts.

Although there are some possible workarounds, most [IMHO] are cumbersome & highly impractical. AFAIK, there are only 2 realistic ways to go about it;

  • Have someone who is expert in the XML language create the customized Theme Fonts (.xml), Theme Colors (.xml) & Office Theme (.thmx) files, or
  • Use Win Office 2007 or later to create the components. Those versions do have the tools for doing so & the files should be usable on a Mac running Office 2008 or later.

Saving the template with any particular Document Theme applied causes that Theme to be the default for new documents generated from the template.

brian tillman [outlook mvp 2007-2019] say:

One likely cause of this is on the recipient's end. Your video arrived in the recipient's mailbox. If they use POP3 to access that mailbox, what could be happening is that their mail client starts to download the message. Since it's a video, it's big (videos should never be sent via email; email's not intended for that) and the mail client takes time to download it. If the mail client's send/receive interval is too short, while it's trying to download the first copy of the video, their mail client will start a new send/receive and start to download the video again. This will continue, producing multiple copies of the message. This does not mean that your mailbox is sending multiple copies. It would mean they're trying to download the same message over and over.

Suggest to them that they log into their mailbox using a web browser, download the video once with the web interface, then delete it. It shouldn't bother them again.

As I said, I think this is the most likely situation and the first thing to check. If it doesn't pan out, post back.
