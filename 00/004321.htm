roady [mvp] say:

No, not without custom VBA. What you can do by a rule is to delay the delivery with X minutes so you still have a chance to get it out of you Outbox folder. You can combine this with the rule looking for certain words in the message body or subject

 

This code should give you a warning upon sending and allows you to cancel it:

 

Private Sub Application_ItemSend(ByVal Item As Object, Cancel As Boolean)

    If TypeOf Item Is Outlook.MailItem Then
        Dim olMsg As MailItem
        Set olMsg = Item
        Dim Prompt As String

        Prompt = "***WARNING***" & vbNewLine & vbNewLine & _
                  "Are you sure you wish to send this message?"

        If MsgBox(Prompt, vbYesNo + vbQuestion, _
                    "Check before Sending") = vbNo Then
            Cancel = True
        End If
    End If
End Sub

 

Place it in the ThisOutlookSession of the VBA Editor (ALT+F11).

Then, sign your code to prevent any security warnings or the code from not running at all.
See: http://www.howto-outlook.com/howto/selfcert.htm

teylyn say:
Hello,

you may not want to hear this, but the Excel error message is actually a blessing, rather than a curse and may save you a lot of time and hassle down the line. Regardless of what's keeping the workbook from being a "shared workbook", be grateful for it.

Even if I knew what kept your workbook from getting shared, I would not want to tell you.

Sharing workbooks to be edited by multiple users at the same time is, although possible and available in many Excel versions, a very bad idea. Shared workbooks have a limited life span. The question is not "if" but "when" they will become corrupt and start to act "funny". Lose data, don't save changes, show different things to different users, and so on. 

Excel is not - and it was never intended to be - a multi-user application. If you need multiple users to edit a data set simultaneously, then Excel is the wrong tool. Consider a database instead. Chances are that if you have Excel, you also have Access. Now, Access has been DESIGNED for multi-user access and is the perfect tool for these scenarios. You can have your database in Access and design a user interface for data entry and manipulation and you can still use Excel to report on the data held in Access.

Other alternatives include SQL Server, which has a free option, or MySQL which is completely free.
