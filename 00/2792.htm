Windows Defender - Microsoft Community

reality1963 say:

Windows Defender

I have Windows Defender installed on my computer but it only scans when I manually select it to scan. The last time it was actually scanned was on 03/31 which is almost a month ago. Isnt there a feature that will automatically scan my computer without me having to manually select it to do it each and every time? McAfee and Norton had that feature, why doesnt Defender? 
greginmich say:

I have Windows Defender installed on my computer but it only scans when I manually select it to scan. The last time it was actually scanned was on 03/31 which is almost a month ago. Isnt there a feature that will automatically scan my computer without me having to manually select it to do it each and every time? McAfee and Norton had that feature, why doesnt Defender? 

Windows Defender does have that feature. Automatic Maintenance should be running a Windows Defender scan every day, provided that you take a break that’s long enough for the system to first enter an idle state and then complete the Quick Scan. Any activity on your part will cancel Automatic Maintenance, and it will then have to be restarted again with the next idle state, and so on. If the scan consistently fails to run for about 2 weeks, then you should be prompted to run it.

Some things to try first if the scan isn’t running:

  • You can run the Automatic Maintenance tasks with normal priority to see if the Quick Scan completes successfully: Right-click on the Start button > Control Panel > Maintenance > Automatic Maintenance > Start maintenance.
  • There’s also a very basic diagnostic tool for Automatic Maintenance: Control Panel > Troubleshooting > System and Security > Run maintenance tasks.
  • You can enable logging for, and then review the history of, the Windows Defender Scheduled Scan task in Task Scheduler. Type “Task Scheduler” in the search bar and click on its entry. Then navigate to Task Scheduler Library > Microsoft > Windows > Windows Defender.
  • You can also look for clues in the Windows Defender Operational log (in Event Viewer). You can filter this log for the relevant event IDs:

Right-click on the Start button and choose Event Viewer. Then navigate to Applications and Services Logs > Microsoft > Windows > Windows Defender > Operational:

Then click on Filter Current Log in the Actions pane and enter “1000, 1001, 1002, 1005” for the Event IDs.

1000 – Scan started 

1001 – Scan completed

1002 – Scan stopped (scan canceled by user activity)

1005 – Scan terminated due to error

https://technet.microsoft.com/en-us/library/mt622090%28v=vs.85%29.aspx?f=255&MSPPError=-2147217396

Beyond that, I can only recommend this general troubleshooting protocol.

1. Remove any external sources of error:

  • Remove registry/system cleaning utilities. Revert any recent changes made by registry/system cleaning utilities, and then disable/remove those utilities. That would include CCleaner, which is known to remove Windows Defender’s Last scan data with its default settings
  • Remove previously installed third-party antivirus apps completely. Download and run the cleanup tool for every AV app that was ever installed on the PC or that came preinstalled:

http://answers.microsoft.com/en-us/protect/wiki/mse-protect_start/list-of-anti-malware-product-removal-tools/2bcb53f7-7ab4-4ef9-ab3a-6aebfa322f75

  • Remove undetected malware. Run some malware-removal tools to make sure that the system is clean:

Kaspersky Virus Removal Tool:

http://www.kaspersky.com/antivirus-removal-tool?form=1

Emsisoft Emergency Kit:

http://www.emsisoft.com/en/software/eek/

Malwarebytes Anti-Malware (free version only):

https://www.malwarebytes.org/antimalware/

Eset Online Scanner:

http://www.eset.com/us/online-scanner/

  • Repair the operating system. Run both of these system diagnostic/repair tools to confirm the integrity of your Windows 10 installation:

http://answers.microsoft.com/en-us/windows/wiki/windows_10-update/system-file-check-sfc-scan-and-repair-system-files/bc609315-da1f-4775-812c-695b60477a93

2. Repair Windows Defender’s internal sources of error or replace it with a free third-party AV app:

Windows Defender does seem to have quite a few error states that can’t be resolved by removing external sources of error. The error messages associated with some of those error conditions are undecipherable; and some of the error conditions themselves appear to be intractable. So when you run into serious issues with scanning, updating, CPU usage, etc., and there isn’t any clear-cut solution in sight, then that might be the perfect time to try out one of the free third-party AV apps:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning/how-to-stop-antimalware-service-executable-from/a3a57d31-4687-43c0-b274-261da7d89245

3. Recovery options:

If you’d rather stick with Windows Defender and you just can’t seem to get it working properly, then you might want to consider one of the system-recovery options:

http://windows.microsoft.com/en-us/windows-10/windows-10-recovery-options
