Set date on OWA email rules - Microsoft Community

ripthorn say:

Set date on OWA email rules

We have 365 OWA users that don't work every day. I can't see a way to set up a rule to forward any received emails to another user on those days. What have I missed?

Thank you.

diane poremsky mvp say:

Since its in OWA, your options are limited (you could use VBA to update the dates in Outlook on windows desktop.) 

However, the admin can use powershell to set the rules. 

https://docs.microsoft.com/en-us/powershell/module/exchange/mailboxes/set-inboxrule?view=exchange-ps

ETA: example, after the rule exists (my attempts at creating a rule using New-Inbox rule didn't work so i manually created a rule to forward all mail, then ran this to update it)    That was user error :)  (I tried using the same format to identify the mailbox as Set uses). 

Create a rule: 

New-InboxRule -mailbox *** Email address is removed for privacy *** -name ForwardTo2 -ReceivedAfterDate 10/15/2018 -ReceivedBeforeDate 10/16/2018 -ForwardTo *** Email address is removed for privacy ***

Set-InboxRule *** Email address is removed for privacy ***\ForwardTo -ReceivedAfterDate 10/9/2018 -ReceivedBeforeDate 10/10/2018 -ForwardTo *** Email address is removed for privacy ***

I'm not sure why the dates are goofy... but it definitely is simple to update the rule using powershell.

tomthetech say:

I was recently searching for this answer myself as I had a client with Windows 7 and Office 2010. The Microsoft sites are so often visited but the answers seem to be somewhere else. 'HelpMeRic' had the answer...

 

The solution is to remove the 'Protected View' settings in Word (maybe in Excel too or Powerpoint if they pose problems).

 

Close Outlook. Open Word. In Word (2010) Go to the File menu then...Options>Trust Center>Trust Center Settings>Protected View and there, remove the checkmark from the three 'Enable Protected View for...blah, blah, blah' boxes. Reopen Outlook and the 'protection' is gone.
