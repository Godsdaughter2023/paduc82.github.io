andreas killer say:

- Right click the sheet tab and select "View code" to open the VBA editor.

- Copy all code below my name and paste it into the VBA editor.

- Click anywhere inside Sub Init and press F5 to run it.

- Close the VBA editor

From now on if you enter the value 15 into column E, the row is grayed out and the cells are locked, except the cell in column E. If you change the value, the row is unlocked.

Andreas.

Option Explicit

Private Sub Init()
  'Unlock all cells
  Me.Unprotect
  Me.UsedRange.Locked = False
  Me.Protect Contents:=True
End Sub

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim C As Range, R As Range
  'Changes in column E?
  Set Target = Intersect(Me.Columns("E"), Me.UsedRange, Target)
  If Target Is Nothing Then Exit Sub
  'Remove the protection
  Me.Unprotect
  'Check each cell
  For Each C In Target
    'Get the row
    Set R = Intersect(Me.UsedRange, C.EntireRow)
    'If the value is 15
    If C = 15 Then
      'Grey background
      R.Interior.ColorIndex = 15
      'Lock the cells
      R.Locked = True
      'But not in column E
      C.Locked = False
    Else
      'Remove the background
      R.Interior.ColorIndex = xlColorIndexNone
      'Unlock the cells
      R.Locked = False
    End If
  Next
  Me.Protect Contents:=True
End Sub

Try this, which should work with strings of any legth:

    Dim i As Integer

    Different versions of Facebook for Windows Mobile - Microsoft Community

    xxsweetangelxx say:

    Different versions of Facebook for Windows Mobile

    I was checking the Facebook app on the Marketplace for Mobile and it has to download the latest version 1.2, but when I downloaded it gave me a different version 1.1.014.

    Why can't I get the 1.2 version? I also checked the screenshots for the app and they're a bit different than what I have. 

    joel ivory johnson say:

    It's not unusual for the version number as reported by the application and the version number as reported by the marketplace to differ. When ever an update is made to an application the Marketplace must always increment the version number by 0.1 intervals. However many software developeers  use version numbers composed of 4 numbers and may increment one of the smaller sets of numbers for a minor update. 

    So If I made an application that had version 1.0.0.0 and uploaded it to the marketplace and later realize I made a spelling mistake in the app I might consider that a minor update. So I make the version number up the update 1.0.0.1 and upload it to the Marketplace. The Marketplace will increment the version number that it reports to 1.1. If I made another minor update my applications version number may be 1.0.1.0 but the marketplace would at this point report 1.2.
