    PC Build Install: "Some Update Were Cancelled" [Workaround Included]

    Hello Insiders!

    We're seeing some reports of the following error message displaying on the Update screen in settings:

    "Some updates were cancelled. We'll keep trying in case new updates become available."

    We've identified the bug already and a fix is in the works.  In the mean time, the workaround is to delete one registry entry:

    • Click Start
    • Type Regedit
    • Launch "regedit (run command)"
    • Navigate to and select the following registry key:

      HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update\RequestedAppCategories\8b24b027-1dee-babb-9a95-3517dfb9c552


    • Delete the selected registry key
    • Reboot your PC
    • Re-scan for updates

    This should resolve the issue.  Post up if you have any questions!

    Thanks,
    Jason

    The following formula returns the lowest local minimum strictly "inside" the data range, i.e. possible minimas at the ends are not counted as minimas. This would correspond to what you call "the minimum at the biggest trough", I guess.

    =MIN(
      IF(OFFSET(my_data,,,ROWS(my_data)-2)>OFFSET(my_data,1,0,ROWS(my_data)-2),
      IF(OFFSET(my_data,2,,ROWS(my_data)-2)>OFFSET(my_data,1,0,ROWS(my_data)-2),
      OFFSET(my_data,1,,ROWS(my_data)-2),1E+99),1E+99))

    Note: This is an array formula that has to be confirmed with CTRL+SHIFT+ENTER rather than just ENTER.

    my_data is a named range of your data. It is assumed to be a multiple row, single column range.
    If you don't want to use named range you can replace my_data with e.g. $A$1:$A$250 in all places in the formula, but I do strongly recommend that you name the range. That can done by selecting the range and the just write my_data in the address box at the top left where you normally see the cell addresses.

    The "index", data point number, for this minimum is given by the following formula:
    =MIN(
      IF(OFFSET(my_data,,,ROWS(my_data)-2)>OFFSET(my_data,1,0,ROWS(my_data)-2),
      IF(OFFSET(my_data,2,,ROWS(my_data)-2)>OFFSET(my_data,1,0,ROWS(my_data)-2),
      IF(OFFSET(my_data,1,,ROWS(my_data)-2)=MIN(
      IF(OFFSET(my_data,,,ROWS(my_data)-2)>OFFSET(my_data,1,0,ROWS(my_data)-2),
      IF(OFFSET(my_data,2,,ROWS(my_data)-2)>OFFSET(my_data,1,0,ROWS(my_data)-2),
      OFFSET(my_data,1,,ROWS(my_data)-2),1E+99),1E+99)),
      ROW(OFFSET(my_data,-ROW(my_data)+2,,ROWS(my_data)-2)),1E+99),1E+99),1E+99))

    Now that you have found the position for "the biggest trough", the next step is to find the "following peek" for the"blue data". In order to design a formula for that we have to get a more strict definiton of "peak". The example you gave shows that there are sometimes "a double peak". What it the criteria to choose the right peak?

    - The first local maximum (after the biggest trough)? Strictly, even if there is another higher peek just after, as in you picture?
    - The second local maximum (after the biggest trough)?
    - The biggest of the n local maximas (after the biggest trough)?
    - The biggest local maximum within a n interval with values all bigger than that local maximum minus x %?

    Refering to your image:
    - If the local peek at 11000 would have been at 16000, would that have been the peek you are looking for, although it would be the third peak after the biggest trough?
    - If the dip at 14500 would have been as low as 8000, would the second peek still be the one you are looking for?
