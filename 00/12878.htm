Change the second line to read:

client_id: WM5_1.1

and then save the file with this amended line back to its original position on the SD card. The Map Loader application will now recognise the memory card for download of new map data.

I noticed a secondary problem with an offset between the world bit map file putting 'sea' and 'land' shading in an offset position. I solved this by

(1) downloading the new map data as above, making a temporary store for the newly downloaded data on the PC by back transfer to the mother PC;
(2) Erasing ALL data S2G data on the SD card.
(3) Uninstalling the S2G app on the WM5 mobile device;
(4) Reinstalling S2G application on WM5, running once on-line to set up a new file structure on the SD card, etc.
(5) finally replacing the SDcard diskcache data contents (the 1, 2, 3...e, f folders) with the large volume map data downloaded and transferred to the mother PC WHILE LEAVING THE OTHER DISKCACHE files as per the the newly reinstalled mobile Smart2Go application.

Finally changing the QF cookie as outlined at the start.

A bit of a chore but everything then works as it should including voices, navigation and all the rest!

psychomania---01 say:

If you put aside the conspiracy theories there's a far simpler explanation why Nokia UK has stopped selling it: supply & demand.

In the UK 90% of Nokia's are sold subsidised, either on contract or as PAYG handsets. If the networks and other major retailers stop buying a model then demand greatly drops. When the demand falls below a certain level production is stopped and existing stock is sold off (or shifted to another country). So the nokia store has just run out of stock and it won't be replenished due to lack of demand from the networks/major retailers.

Many of the UK networks abandoned the standard N97 and sold the mini instead. This may have been due to issues or due to better profit margins on the mini.

As for the 5800 still being sold this is because it is still an in demand model with orders being placed by networks that are still selling it. It will likely disappear shortly after the networks stop selling it (which will probably be very soon, if not already happened recently).

So it's not a case of quietly being withdrawn at all.

stefan blom say:

The message suggests that the Building Blocks.dotx file is corrupt. Exit Word and then rename or delete that file from your user profile. Be sure to rename or delete the file in your user profile, not the one in the Program Files folder.

For Word 2010 on Windows 7, the location is a subfolder of C:\Users\your_user_name_hereAppData\Roaming\Microsoft\Document Building Blocks\, but you can search for the file name in Windows.

When you restart Word, the file should be recreated with the factory default settings. If it doesn't, repair Office from Control Panel.

Added clarification (January 2012): In Word 2010, there is also a Built-In Building Blocks.dotx file (for the built-in entries). Try deleting that one as well.
