That's really useful. So for the record, the final formula is as follows.

=IF(ISTEXT(A19),"100%",IF(YEAR(A19)=2012,VLOOKUP(A19,$O$10:$Q$30,3,TRUE),IF(YEAR(A19)=2013,VLOOKUP(A19,$R$10:$T$30,3,TRUE),"WRONG")))

The formula is fine.

For the record. the first thing the formula does is check if there is text (a day) in A19. If so, it returns 100% (that's what you asked for).

If not, then it tries to find the date entered in A19. The year of that date will determine in which of the 2 tables to look. Now it will return "WRONG" if the year is not 2012 or 2013. A completely wrong date like 32/32/2012 will be considered text and result in 100%.

If the exact date is not found, because of the TRUE statement, it will go to the next largest value that is less than A19. This way the appropriate date range is found. Then it returns the value in the 3rd column (percentage) on the same row as that date.

david a matson say: Greetings frank!   Welcome to the Windows Mobile Total Access forums!

I believe there is no way to deactivate your WiFi card while it sits in your SD slot, unless the manufacturer created one.  

I would be willing to guess that the error that you receive is due to your using the WiFi card and then shutting-down your device (putting it asleep, etc) before turning-off the WiFi card via the Wireless Manager.  After starting the device back up, the WiFi configuration info is missing, causing WM to attempt to rescan (re-enumerate) the device, but for whatever reason it fails.  If this sounds like it might be true in your case, you could try this obvious experiment: with the unit working and connected to the 'net via WiFi, turn-off the WiFi via Wireless Manager and turn off your device and see if you get the same error when you restart (re-awake) the device.

Alternately, if you know you forgot to turn-off the WiFi prior to turning off the device, then you ought to be able to extract the card from the slot and then power up the device.   This should cause the rescan attempt by the system to correctly identify that you no longer have a card in the slot, which should theoretically prevent this error message from appearing.  However, it's also likely that the WM will instead give you a new error message "Missing Card in Slot 1".

Let us know if this helped...

danida_u say:

Hi SharynAdelman and welcome to the Skype Community! 

If you still remember the email address associated with your Skype account, you may try to use the Microsoft Accounts Diagnostic link to determine the Skype Name of the account you wish to log in. Follow the suggested steps below. 

  • Go to this link. 
  • Enter the email address associated with your Skype account and click Continue.
  • You'll be given the list of the accounts that are associated with the email address, click the account you want to have access with and choose Reset password. 
  • Choose I forgot my password on the list of the reasons for password reset and click Next. 
  • Enter the captcha code. 
  • You'll be given the list of the associated security info of your account, choose one and click Send code. 
  • Confirm the code that was sent and proceed with password reset. 

Important note: If you do not have access to the associated security info of the account, click I don't have any of these and you'll be directed to the account recovery page. 

Hope this helps. Let us know if you need further assistance.
