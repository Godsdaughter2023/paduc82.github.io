Data Execution Prevention , is a Microsoft technique for preventing the execution of presumably unauthorized software in RAM memory. DEP protects against a common technique used by malware writers: the exploitation of memory bugs in Windows and other software applications. A buffer overflow creates an opportunity for malware to insert itself into parts of RAM that are not normally used for running programs, and then run itself from there. DEP blocks execution of programs from areas of RAM marked "non-executable". Malware based upon buffer overflow exploitation is dead as a brick, it can't do anything.

    l4uren say:

    How to create a link between salutation and gender in microsoft acess 2010

    Hi,

    If I have a list of names and their gender,  how do i link this to the salutation column so that when it is input that a person is male, then the salutation will automatically become 'Mr' and for females 'Ms'?

    hansv mvp say:

    You don't need to have a Salutation column (field) in the table with the names and genders.

    Create a query based on the table, add the fields from the table to the query grid, and create a calculated field in the first empty column:

    Salutation: IIf([Gender]="Male","Mr",IIf([Gender]="Female","Ms",Null))

    If you have coded the gender as a number, for example 1 = Male and 2 = Gender:

    Salutation: IIf([Gender]=1,"Mr",IIf([Gender=2,"Ms",Null))

    Another option would be to create a table with two columns: Gender and Salutation.

    You can then add this table to the query, join it to the names table on the Gender field, and add the Salutation field to the query grid.

    Let's start from the beginning.

    Activate the Create tab of the ribbon.

    Drag the fields that you want to include from the field list in the upper pane to the query grid in the upper pane.

    Salutation: IIf([Gender]="Male","Mr",IIf([Gender]="Female","Ms",Null))

    Replace Gender with the actual name of the gender field in your table, and "Male" and "Female" with the actual way gender is stored in your table. For example, if you have "M" for male and "F" for female:

    Salutation: IIf([Gender]="M","Mr",IIf([Gender]="F","Ms",Null))

    amit tandon say:

    Refer below image:

    Duplicates in column A & B are extracted in Column C;

    The addresses of occurrence of the duplicate - "bb" (cell D2) in column A is given in column E;

    The addresses of occurrence of the duplicate - "bb" (cell D2) in column B is given in column F;

    The addresses of occurrence of the duplicate - "cc" (cell G2) in column A is given in column H;

    The addresses of occurrence of the duplicate - "cc" (cell G2) in column B is given in column I;

    All formulas below to be entered as array formulas - (CTRL+SHIFT+ENTER).

    Enter below formula as an array formula in cell C2 & copy down:

    =IFERROR(IFERROR(INDEX($A$2:$A$100, MATCH(0, COUNTIF(C$1:C1,$A$2:$A$100)+IF(COUNTIF($B$2:$B$100,$A$2:$A$100)>=1, 0, 1), 0)), INDEX($B$2:$B$100, MATCH(0, COUNTIF(C$1:C1,$B$2:$B$100)+IF(COUNTIF($A$2:$A$100,$B$2:$B$100)>=1, 0, 1), 0))), "")

    Enter below formula as an array formula in cell E2 & copy down & to column F:

    =IFERROR(ADDRESS(SMALL(IF(A$2:A$10=$D$2,ROW(A$2:A$10)-ROW($A$2)+1),ROWS($A$1:$A1)),COLUMN(A$1)),"")

    Enter below formula as an array formula in cell H2 & copy down & to column I:

    =IFERROR(ADDRESS(SMALL(IF(A$2:A$10=$G$2,ROW(A$2:A$10)-ROW($A$2)+1),ROWS($A$1:$A1)),COLUMN(A$1)),"")

    You may update the range / sheet per your requirement. The above illustrates how to extract duplicates in 2 columns & then return their address of occurrence. Hopw your query has been correctly understood.
