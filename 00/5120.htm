gary's student say: If you are streaming data into A1, we can use the Calculate Event to capture changes.  In B1 enter:

=A1

In the worksheet code area enter the following event macro:

Private Sub Worksheet_Calculate()
If [A1] > [A2] Then
    [A2] = [A1]
End If
End Sub

  Because it is worksheet code, it is very easy to install and use: 
1. right-click the tab name near the bottom of the window
2. select View Code - this brings up a VBE window
3. paste the stuff in and close the VBE window  

If you save the workbook, the macro will be saved with it.   To remove the macro:
1. bring up the VBE windows as above
2. clear the code out
3. close the VBE window

To learn more about macros in general, see:   http://www.mvps.org/dmcritchie/excel/getstarted.htm   To learn more about Event Macros (worksheet code), see:   http://www.mvps.org/dmcritchie/excel/event.htm

 


Whenever the stream refreshes A1, B1 should get calculated.  This triggers the macro.  The macro checks this new value in A1 against the value in A2.  If A1 is larger, the larger value gets stored in A2.

dan_hill2802 say: You can use the LARGE excel formula to return the highest value in a given range.

You can also do this in VBA like this:

highestValue = WorksheetFunction.Max(range("A1:A25"))

I asume your stream comes in at regular intervals, so you should be able to figure out how to calculate the range to be equal to the previous 25 minutes.

Hope one of those methods answers it for you.

error message 0X80072EE7 - Microsoft Community

rapxm say:

error message 0X80072EE7

I get error message 0X80072EE7 when I try to activate office hom and student 2010
jay freedman say:
According to the article at http://support.microsoft.com/kb/883821, which is about errors during attempts to use Windows Update,

"The 0x80072EE7 error code may occur if the client computer cannot find the correct IP address when it tries to resolve a URL for the Windows Update Web site or for the Microsoft Update Web site. This error may occur if the Hosts file contains a static IP address."

The same error number might occur when the Office installer tries to reach the activation server, if there is a problem with that address (not the Windows Update site). It's worth checking the reason given in the article (a static IP address in the Hosts file). There are instructions in the Resolution section, under the 0x80072EE7 error code heading, for how to find and read the file. Except in very unusual circumstances, every line in the file can be commented out by putting a # character at the left margin.

A static IP address isn't the only possibility. It's also possible that during the installation/activation you don't have a working Internet connection, or that you don't have access to a Domain Name Service (DNS) server, or (unlikely) that the DNS has a bad address for Microsoft.
