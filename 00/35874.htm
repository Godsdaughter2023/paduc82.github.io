HDMI is not just a cable, it is an authentication cable originally designed to keep recording equipment from being added from your cable providers cable box (STB) (among other things). The reason why HDCP (in part the authentication layer) is denied, is a request being accepted or denied by the STB. This causation could be either a. The HDMI cable is not layered to the correct format (there are tons of different 'types' of HDMI the wiki article has a good run-down on a few of these versions: en.wikipedia.org/.../HDMI).


Using the super awesome HDMI cable that came with the Xbox One (it is a crazy good HDMI cable) between the Xbox One and the STB will eliminate this variable. b. The STB 'knows' that there is a device between the TV and STB therefore denying the request (as it thinks there is recording equipment in the chain). c. Xbox is sending the reply request in the wrong format to match the firmware version of the STB (as each STB has its own 'OS' that tells it how to function and what to do). It is important to note about 'c' that every single STB must have its own software layer written: take for example the Motorola DCT3416 there are 4 versions that I know about, I have rumors of a 5th. Even the model numbers of this TV equipment is misleading: take for example Scientific Atlanta, as it is marked Cisco on the sticker, in this way Pace also has matching model numbers match Motorola devices.

  • Click Start.

  • Type cmd, and then press CTRL+SHIFT+ENTER to open elevated command prompt(I.e. Command prompt with admin rights).

  • If the User Account Control dialog box appears, confirm that the action it displays is what you want, and then click Continue.

  • Type or copy paste the below command lines into it and hit Enter:

DISM.exe /Online /Cleanup-image /Scanhealth

DISM.exe /Online /Cleanup-image /Restorehealth

sfc /scannow

To close the Administrator: Command prompt window, type Exit, and then press Enter. Reboot your system to see if problem still occurs.

You could try this:

Right-click Command Prompt and Run as Administrator. Type the following command:

winmgmt /verifyrepository

If the system returns "WMI repository is not consistent", run this command:

winmgmt /salvagerepository

 

The first time you run this it will fail. It will issue stop commands to the services causing it to fail. It might take a couple minutes for the services to shut down. Run the command again. You actually may have to run it 3 times before it finally runs and completes on its own.
