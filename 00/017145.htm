jllatham say:

The basic problem is caused by the way that VBA handles the " symbol.  To get your formula to be built properly, it needs to be written this way (I changed the sheet name for my testing)

Sheets("Sheet1").[B1].Formula = "=REPLACE(RIGHT(A1,5),1,SEARCH(""."",RIGHT(A1,5)),"""")"

or you could write it this way

Sheets("Sheet1").[B1].Formula = "=REPLACE(RIGHT(A1,5),1,SEARCH(" & Chr$(34) & "." & Chr$(34) & ",RIGHT(A1,5))," & Chr$(34) & Chr$(34) & ")"

Chr$(34) is the code for the " symbol.  In the second example, picture it this way: Excel VBA is going to make use of all of the actual " symbols itself to determine where strings start and stop, so the ones you see in that line of code for the formula portion just simply disappear.  But you still need some " symbols to appear in the final built up formula.  That is where the "" pairing of the " symbols or the use of & Chr$(34) & comes in - those tell VBA to include a " within the built up formula itself.

OTHER Notes:

The 'standard' way of refering to a cell in VBA is  Range("B1") so the left portion of your formula could have been written as 

Sheets("Sheet1").Range("B1").Formula = "=REPLACE(RIGHT(A1,5),1,SEARCH(""."",RIGHT(A1,5)),"""")"

although the way you wrote it works just fine and there's nothing especially 'wrong' about it.

Copying the formula down.  There's a quick way to do the equivalent of filling a formula if you use a formula in R1C1 format instead of using a standard notation for cell/range addresses in the built up formula.  If you look at the formula in cell B1 in R1C1 format after it is in the cell, it looks like this:

=REPLACE(RIGHT(RC[-1],5),1,SEARCH(".",RIGHT(RC[-1],5)),"")

(Go to the [File] tab, Options, Formulas and check the "R1C1 reference style" option)

If you were to fill that formula down the sheet and then look at it in normal reference style you'd see it had changed to A2, A3, A4, etc. while in R1C1 form, they all look exactly the same!

Given all of that, here is some code that would 'fill' the formula down to the last row used in column A in a single step:

Sub FillWithR1C1Formula()
  Const firstRow = 1
  Dim fillRange As Range
  Dim lastRow As Long
 
  'find last used row in column A
  lastRow = Sheets("Sheet1").Range("A" & Rows.Count).End(xlUp).Row
  'set a reference to associated cells in column B
  Set fillRange = Sheets("Sheet1").Range("B" & firstRow & ":" & "B" & lastRow)
  'put the formula into every cell in fillRange
  'you don't have to be in R1C1 format in the workbook to use this
  fillRange.FormulaR1C1 = _
   "=REPLACE(RIGHT(RC[-1],5),1,SEARCH(" & Chr$(34) & "." & Chr$(34) & ",RIGHT(RC[-1],5))," _
   & Chr$(34) & Chr$(34) & ")"
  
'or   "=REPLACE(RIGHT(RC[-1],5),1,SEARCH(""."",RIGHT(RC[-1],5)),"""")"
 
  'just good housekeeping practice:
  'release resources assigned with Set back to the system
  'for reuse to prevent potential "memory leaks"
  Set fillRange = Nothing

End Sub

rohn007 say:
Yes you can run both 2007 and 2010 on same machine.

I currently have 2003, 2007, 2010 AND 2013 all running in Win 7.

There are a few considerations:
  1. You can have only one version of Outlook installed, your choice
  2. You have to decide which version of the app, 2007 or 2010, is defined as the Windows default
  3. You have to apply a registry hack to make it work

Office is written to assume that all "reasonable" / "average" users will ONLY ever have one version of Office installed, "the newest".  When you have more than one version installed, and you start an app from the "other" version, Office will assume you just installed it and trigger a "configuration process" that sets up registry entries, even if they are already there.  That annoyance can be suppressed by adding "NoReReg" registry keys.

Manual method:

Before you close up shop and start working on this reg fix, open the version of Word that you will want to be your default. In other words, say you have two versions of Word on your system. You want to use 2007, but you want 2003 to be the version that opens by default when you click on a DOC file. Open Word 2003 before you insert the fix below. This will ensure that Word 2003 is the last version opened and it will, if necessary, reset the registry to that version. (More on getting around this default in the next section.)

·         Now make sure Word (office apps) is closed

·         Start regedit  <Win  R> and type REGEDIT

·         Note that if this doesn't do anything, you're using a system where the administrator has taken away your ability to edit the registry so you don't mess it up!

·         Once within the registry, click the + (plus sign) to expand the /HKEY_CURRENT_USER/ folder.

·         Scroll down and locate the /Software/ folder and expand that one. We have a long journey ahead, so hang in there!

·         Once within the Software folder, scroll down that branch of the folder tree until you find the folder for /Microsoft/. Click the + to expand that one.

·         Now scroll down to locate the /Office/ folder.

·         There you'll see the numbered versions of the Office versions you have:

·         2013 is 15.0

·         2010 is 14.0

·         2007 is 12.0

·         2003 is 11.0

·         2002 is 10.0e.

·         This fix can be used for all recent versions. Or at least, I know it works as far back as 2000. I can't confirm anything earlier. But if you're still using a version older than 2000, speed is obviously not an issue for you!<grin> Expand the folder for the version you want to speed up. (Note, this speed up fix will only help you if you are running multiple versions. So if you only have one version of Office on your system. ·         Move to the /Word/ folder and expand that. Almost there!

·         Okay, locate the /Options/ folder and select that one. This is the folder where you'll be adding a new registry key.

·         With the Options folder selected in the route we took above, click Edit > New > DWORD Value

·         This will insert a new registry key into the Options folder. It will be highlighted in edit mode so you can go right ahead and start typing the name of this key: NoReReg

·         Hit the Enter key after typing the name to have the name accepted.

·         Now double click this new key to open it. The Value data will read 0 (zero) by default. Change that setting to the number 1

·         click OK to accept that value.

Then close the registry. You did it!

So what is it that you've done? Well, you went into the Word registry settings for the version of Word you selected and you set a new registry key. The key is an option that stops Word from reregistering itself if another version is the current default. The value of 1 means Yes. So the NoRereg (no reregistering) is set to Yes...i.e., don't do it! This will cause Word to ignore the default version and just open, without taking several minutes to go through the reregistering hassle.

Hi Gene_,

 

Generally, for Skype user to add Skype for Business contact, in Skype, type the contact address you want to add, it will appear in the list of Public Directory of Result, you may see below:

For Skype for Business to allow external communication, some settings need to be done:

 

  1. Go to Office 365 admin center->Settings->Service & add-ins.
  2. Under External sharing, check the box ”Let people use Skype for Business to communicate with Skype users”
  3. Go to Office 365 admin center->Skype for Business center.
  4. In Skype for Business admin center, navigate to organization-> external organization, select the check box ”Let people use Skype for Business to communicate with Skype users outside your organization”
  5. To know more about communication between Skype and Skype for Business, you may refer to: https://docs.microsoft.com/en-us/SkypeForBusiness/set-up-skype-for-business-online/let-skype-for-business-users-add-skype-contacts .
  6. After all have been done, it may take up to 24 hours to take into effective.
  7. If you use Skype for Business not Skype, and want to communicate with external Skype for Business user, both organizations need to allow external communications, you may refer to the settings steps in:  https://docs.microsoft.com/en-us/SkypeForBusiness/set-up-skype-for-business-online/allow-users-to-contact-external-skype-for-business-users .
