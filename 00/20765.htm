rena yu msft say:

Hi BF654,

Although the lookup column can’t help you match the person column, but I find a way you can do with SharePoint Designer workflow. For example, I want to get person from list 2 to list 1.

First, create a person and group column in list 1. For example, name it “TestPerson”.

Go to SharePoint Designer, create a SharePoint Designer workflow for list 1.

Add the Action command: Set field for current item

Click the “field” and Set the “field” to “TestPerson”.

Click the “value” to get the “Select User” window. Select “Workflow Lookup for a User…” and click “add”.


You will get a “Lookup for Person or Group” window. Set “Data source” to “list2”. Set “Field from source” to “person” (this is the person and group column name I created in list2). Set “Return field as” to “Display Name”.


In the “Find the List Item”, set “Field” to “ID”. Because I set lookup column in my last reply. We can use this to match the value. This “ID” means the Item ID of list 2.


Click the “fx” button to select “Value” to the lookupID column in list1.


After the steps above, the settings should like this. Click ok.


Save the workflow, go to workflow settings to set it “Start workflow automatically…” when item is created or changed.


Then Save and Publish the workflow.

When you modify the existing items or create new items, the workflow will run automatically. See the screenshot below. “Test1” is the workflow status, it shows that the workflow has run completely. The workflow set the value to “TestPerson” column (person and group type) automatically from another list. The value is from “Lookup:ID” matching.

Word Document compatibility mode

suzanne s. barnhill say:

I think you've nailed it. As long as your Normal.dotm template is the one created in a previous version, the documents based on it will be in Compatibility Mode. I believe the correct procedure is to open Normal.dotm and resave it in Word 2013 format.

This problem is discussed (though wrt Word 2007 vs. 2010) in http://answers.microsoft.com/en-us/office/forum/office_2010-word/how-do-i-keep-word-2010-from-opening-in/b1e41e45-fafa-4d5a-a771-7fb56ee775ad. Jay Freedman gives these instructions:

What you're seeing is that the Normal.dotm template, which is used to create new blank documents, is the one you had in Word 2007. It was kept intact during the upgrade, but Word 2010 knows that it isn't "2010 native". Every document based on it will be in Compatibility Mode until you up-convert the template itself.

To fix it, use the File > Open dialog to locate and open Normal.dotm (not a blank document based on it). I expect it will have "Compatibility Mode" in the title bar. Click the File tab and then click the Convert button on the "Information about Normal" page. Save and close, and shut down Word. When you restart Word, the new blank document should no longer be in Compatibility Mode.

I think you can probably use the same procedure to up-convert your old template. Many users find it difficult to locate Normal.dotm. If that is a problem for you, here's a way to open it without having to find it:

In Word, press Alt+F11; the Visual Basic Editor opens. Press Ctrl+G to activate the Immediate window. In that window, type

     NormalTemplate.OpenAsDocument

and press Enter. Normal opens, but note that it may not be on top of all windows. You can close the Visual Basic Editor. Make the desired changes to the Normal file, and then save and close it. If the changes you have made have not been recognized as changes (the Undo button is not activated), press Spacebar, then Backspace to "dirty" the file, thus ensuring that the template will actually be saved.

graham mayor say:
Whichever document template you are using, you can open it in Word 2013 and then use SaveAs to save it with the same name and it will be updated to the new format and thus lose the compatibility mode. You can do the same with any old template add-ins that you may be using.

%appdata%\Microsoft\Templates
to the address window of Windows Explorer and pressing Enter.

The default startup folder it can be located by entering or copying

ito the address window of Windows Explorer and pressing Enter.

If your device is running Home then the Insider build will be Home.  You don't get to choose Home or Pro.  It is an upgrade and upgrades are always like for like.

The Microsoft Software License Terms, in section 13. d. (iv), state:

(iv) Preview. You may choose to use preview, insider, beta or other pre-release versions of the software (“previews”) that Microsoft may make available. You may use previews only up to the software’s expiration date and so long as you comply with all the terms of this agreement.
