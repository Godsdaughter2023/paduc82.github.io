    If Not Intersect(Target, Range("A7")) Is Nothing Then
        Range("B21:B44").ClearContents
    End If
End Sub

So, then you would just right-click, Copy Shortcut on that, then paste it into your Address bar, (Alt-d, Ctrl-v), then replace (conveniently) that last parameter in the URL with the one you want, e.g. press End, Ctrl-Backspace,  and type  2936068 (unless you want to paste it too but then you would have to build your URL somewhere else than the Address bar.)  Then press Enter.   Instead of giving you that derived link, I'll just leave it as an exercise.  Note that the number given to type is just an example, which happens to be current now.  You derive the actual number by following the previous steps.  (On second thought, this is making the "detour" through the Downloads site simpler.  I'll let you try to find your way through that route to the KB article.  The current KB article is your goal.)

Whew!  So now you would have the current KB article for the latest cumulative update for IE.  

(If you think you have current maintenance another way to find it would be using a link in IE's Help About dialog, e.g. press Alt-H A and click on the link in there.   Hmm... now would be a bad example for using that because the current update level for IE is (atypically) not a cumulative update.  Worse, there doesn't seem to be any easy way for using it to find the relevant KB article for the relevant cumulative update for that update (let alone the latest cumulative update).  E.g. use the link in the Help About dialog and try to do a Ctrl-f for cumulative in that page and see "Not found".  Too bad.  Sorry for this digression but it might save some steps in a more typical case.)

Now you need to find the  File Information  in it which is relevant to your version of IE and your OS level and expand it.   For example, for the OP of this thread that would be: 

+ Windows 7 and Windows Server 2008 R2 file information

Unfortunately, the various IE and OS level combinations are all listed together in there, so you have to scroll and scroll to find the applicable group label and then scroll and scroll some more to find the module name that you are interested in.  If you're using IE don't try to use the Find tool to avoid this scrolling. especially if you have already found your desired group.   You can use the Find tool to help only with highlighting.  In fact, it is probably a good idea to do the Find first to make sure that the hightlighting assist is in place because apparently the modules are not listed in alphabetical order, in case you would have made a silly assumption like that about how to find one.

When you Copy, it's the source program in combination with the OS that determines what variants are placed on the Clipboard [a service of the OS].

When you go to Paste, the program being used assesses what's being presented on the Clipboard & in which ways it can interpret that information.

The 'change' you're experiencing could be due to a difference in either or all of the 3 participants -- Source data format, Target program interpretation, OS Clipboard management.
