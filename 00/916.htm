jay freedman say:

Yes, it's possible with the use of an ActiveX textbox. Insert the textbox in the document from the Developer ribbon:

Only the ActiveX textbox will work for this task.

When the textbox fist appears, the Design Mode button in the ribbon is turned on. Click the Properties button just below it, which opens a list of properties. Scroll down and click in the cell to the right of "PasswordChar" and type in the character you want to use for the mask, such as an asterisk.

Close the Properties pane and turn off the Design Mode button. The textbox will now show an asterisk (or whatever password character) in place of all text typed into it.

To see the text that was typed, turn on Design Mode and open the Properties dialog. Delete whatever character is in the PasswordChar property and press Enter.

greginmich say:

In order to make it more likely that you’ll see detections, you can uncheck the “Apply recommended actions: Help protect your PC by applying these actions after Security Essentials detects potential threats.” option in the “Default actions” settings. If you uncheck this option, you’ll get a more persistent popup: “Security Essentials detected a potential threat and suspended it. Click Clean PC to remove this threat”. This popup only has one button, but it really gives you two choices; “Clean computer” or “Show details”. If you click on the “Show details” link, you’ll see the information on the threat, and also be presented with the options for alternative actions.

 

I have to agree that MSE is pretty buggy when it comes to record keeping and error reporting. Here’s a hypothetical reconstruction that might help explain what happened: The file was probably blocked by real-time protection and never actually downloaded, so the “quarantined” status was incorrect (there might be an error event that explains this in Event Viewer). Hence the “quarantined” file couldn’t be restored (once again, check Event Viewer for an error event). When you restored the quarantined items, the “temporary path” exclusions were created. This type of exclusion is only created when quarantined items are restored, and it’s pretty confusing because it doesn’t show up anywhere in the main interface, and it automatically expires after a few days without any notification. So the threat was probably never actually “allowed”, because it wasn’t actually quarantined in the first place; and the subsequent download probably succeeded only because the temporary path exclusion was in place. Unfortunately, MSE is in a perpetual state of “arrested development”, and the problematic elements of the user interface aren’t very likely to ever get a makeover.

 

 

GreginMich

gregsmac say:

daniel g. grau say:
Could the answer be that Word offered spell checking long before it was available systemwide on the Mac? Then they just carried it on.

A custom dictionary is a list of additional words that Word will look at, and suggest from. It works like an expansion of the built-in dictionary. What I meant to say was, you would not have to enter the words twice, but could just copy them all at once in one single operation, into a custom dictionary. That would make Word ignore/suggest terms from that list. Of course, it would not keep updated.

But I found a solution for you. I did a short test and it worked, but I don't know if it will hold out in a longer work session. At any rate, it's worth a try:

  1. Do a spell check operation in Word.
  2. Once it stops at an unknown word, click on the Options button.
  3. You will see the "Spelling and Grammar" dialog box that you can also access directly through Word > Preferences > Spelling and Grammar.
  4. There will be a Dictionaries button, that lets you enable/disable your custom spelling dictionaries. Click on it.
  5. Now click on Add.
  6. Navigate to the following folder:  YourUserName > Library > Spelling.
  7. In there, you should see a file called "LocalDictionary" that contains all the terms added to Apple's dictionary (if you see other files, one of them should be what you need, anyway).
  8. Enable that file as a Word custom dictionary. You will need to select All Files in the drop down menu at the bottom.

Let me know how it works out.

If would be advisable to Option drag Apple's file to the Desktop, to keep a copy handy in case you observe some corruption once you start using it in Word.

suzanne s. barnhill say:
If the PDF was created (by scanning) as a graphic, it will not be edited unless you process it with OCR software; it is just a picture.

charles kenyon say:

Many pdf files are simply images.

In other words, just as you can't take a picture of a car, get in it, and drive off... you can't get in a picture of a document and edit it.

OCR software converts images of documents into words (more or less). It is not perfect. A scanned document put through OCR will not create the same pdf as a Word Document saved as pdf. Even with the latter, when you convert it back to Word, chances are that you will have a number of formatting anomalies.

If you have Office OneNote, that has OCR capabilities. Adobe Acrobat (not Acrobat Reader) can do this also.

There are also a number of freeware OCR programs out there.
