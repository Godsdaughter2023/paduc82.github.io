craigmore say:

The 8.1 upgrade process is inexplicably defeated by really simple things (that have been reported since 8.1 Preview).


1. If you added a second drive to your system (e.g., you installed an SSD as the new boot drive), the upgrade will fail if the second (original) drive is not disconnected. In the same way, any external drive must be disconnected.


2. It's necessary to run msconfig and disable all non-Microsoft services, reboot, then initiate the upgrade (or re-initiate the upgrade, since you've already seen it fail by this point).


These two points seem to resolve many instances of failed upgrade.

ruqing say:
I had previously migrated from HDD to SDD. After migration, I did not cleaned up my HDD, all the system files etc were left in the HDD. I cleaned up my HDD and retried Windows 8.1 installation. It succeeded immediately.

Those recoverable partitions on my HDD is not removable using Windows 8 Disk Management. Do the following if you want to remove them.
- Backup your full HDD because the following actions will wipe out your whole HDD (I've learnt the hard way !)
- Open a command prompt.
- Enter diskpart
- Select [disk number of your HDD]
- Clean
** Note that Clean command basically wipe out the whole HDD and give you one empty partition.

After this, install Windows 8.1 update. After that, then restore your HDD from your backup.

Also had a fresh OS install.
