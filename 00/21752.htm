macropod say:

Open your document, scroll to where a page # appears. Does the page # look greyed-out compared to the other content on the page? If so, it's in the header footer area. Next, press Alt-F9. Does the page # change to something like { PAGE }? If so you're using a PAGE field. Press Alt-F9 again to restore the display.

 

If the page # is in the header/footer area, double-click on the top page margin. This will access the page header. Now scroll to where the page # appears. If you're using a PAGE field, press Alt-F9 again to toggle the display, then select the entire field and delete it.

 

If the page #s are not in the header/footer area or are not created via a PAGE field, each one will probably have to be deleted individually.

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file is primarily intended to illustrate how to output a report as a PDF file, but it serves to illustrate the sort of model you need.  The fact that it deals with invoices rather than orders is immaterial.

To decompose the tables you'd firstly set up the new tables and create enforced relationships between them.  By using append queries you'd firstly insert all DISTINCT customer data into a Customers table, all DISTINCT product data into products, all DISTINCT orders data into Orders and finally all order details data into OrderDetails.  The order in which the queries are executed is very important; data must be appended to a referencing table before data can be appended to a corresponding referencing table.  The DecomposerDemo file in my same OneDrive folder illustrates this, using data imported from Excel as an example

For outputting the data with a concatenated list of products see the Concat file in my same OneDrive folder.  This demo uses the highly efficient GetString method of the ADO recordset object to concatenate values form a related table by means of a GetList function, using data from Northwind as an example.  The left hand column of the report in my file concatenates the values into a list, terminating each row with a carriage return/line feed.  While my file outputs this in a report it could also be done in a computed column in a query in which the GetList function is called in the same way.

When he tries to sync ODFB on his machine, it shows synced but the folder name is still of his old tenant which should not be the case. This issue is not occurring on any other machine.

tina chen msft say:

Hi Prabhat,

 

You may let the user try following on the affected machine to check the outcome:

1.Quit OneDrive sync client.

2.Press Win + R to open the Regedit to open Registry Editor.

3.Go to HKEY_CURRENT_USER > Software > Microsoft > OneDrive > Accounts.

4.In Accounts folder, click Business one by one to find the account using the old tenant and delete it.

 

5.Then try syncing OneDrive for business again to check the outcome.

 

Note: Before you modify the registry, back up the registry for restoration  in case problems occur.

WARNING: Using Registry Editor incorrectly can cause serious problems that may require you to reinstall your operating system. Microsoft cannot guarantee that problems resulting from the incorrect use of Registry Editor can be solved. Use Registry Edit at your own risk.
