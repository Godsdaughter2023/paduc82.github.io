Sub AddChangeDelete()

To do this visually, you have the right idea of adjusting all the years to a common 'base.'  However, instead of changing the type of only series 2 to XY Scatter, change all the series to XY Scatter!  Essentially, create a XY Scatter chart instead of a Line chart.

Finally, you may want to look at a regression analysis to check for patterns.  For your data, create 2 columns (Year and Month or Year and Week) and do a regression of the prices as the dependent variable and the 2 new columns as 2 independent variables.

In all of the above, you should also account for 'unusal events' such as a abundant harvest in a particular year or a drought or whatever such event(s) that might affect food prices.

How can I transfer contacts in "This Computer Contacts" to the "Contacts" area?

First, going to assume that you're referring to a contact folder marked as "This Computer Only" vs "This Computer Contacts"

Next, what exactly do you mean by "....to the "Contacts" area"? Specifically, are you referring to another Contact folder configured in your Outlook profile? How many contacts folders do you have in this profile?

>>
I'm not too sure what are the benefits with either "This Computer Contacts" (seems restrictive) vs the "Contacts" area.
<<

#1 - Contacts are not sync'd to the server and hence across devices with either a POP3 or Imap account. Doesn't matter what folder is involved.

#2 - Aside from item #1 above, really not recommended to keep data in any folders marked as "This Computer Only". These are folders that are stored in the underlying OST data file associated with the Imap account which gets summarily deleted should the Imap account get removed from the profile. Additionally, should Outlook not find the .OST file for any reason, it will recreate the OST file and re-sync the email folders - the "This Computer Only" folders (i.e. contact/calendar) will be empty. In either instance, you will lose the data in the "This Computer Only" folders unless you have explicitly backed up those folders by exporting the data to a PST file.

#3 - If by "...contacts area" you mean a contact folder contained in a PST file, then the benefit is that you will not lose the data. If you don't need to share your contact/calendar data across devices, then this would be the suggested approach as per the following article

Just to make sure that we're on the same page - when you use the words "Contacts group" - you are referring to an individual Contacts folder - correct?

Assuming the answer to the above is yes - process to copy everything over is as follows

1) Select the "This Computer Only" contacts folder
2) Press CTRL+!A to select all contact items
3) right-click on the selected contact items --> Select <Move> --> select <Copy to folder> --> select the destination folder
4) check to make sure everything is copied over as expected

*** reason for selecting <Copy to Folder> in step #3 instead of <Other Folder> is purely as a precaution in the event something goes awry during the items being moved to the destination folder. The <Other Folder> option is a <Move> process where the source items will be automatically deleted from the source folder. When you're done with step #3 and everything is OK, you can manually delete everything from the "This Computer Only" folder using the same process - press CTRL+A followed by the Delete key

john w. vinson say:

What's the structure of your CaseManagers table? Typically there would be a Primary Key CaseManagerID field (autonumber or employee ID for example), name fields, other biographical data, etc.; and there would be a CaseManagerID field (NOT the manager's name!) in the Cases table.

The combo box can be set up so the numeric ID is the bound column, which gets stored in the Cases table, and the manager's name is the first nonzero width column - that way the computer sees a meaningless numeric ID, the user sees "Hernandez, Sophia", and they both go away satisfied.

ken sheridan say: Here's an example of how a combo box would be set up to select an employee:

ControlSource:   EmployeeID (for a bound control; leave blank if the combo box is an unbound 'navigational' control)

RowSource:     SELECT EmployeeID, FirstName & " " & LastName FROM Employees ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

Alternatively, you can concatenate the names so that the last name is first, which would be better with a large list of names as it allows the user to enter the intial characters of the last name and progressively go to the first match as each character is entered:

RowSource:     SELECT EmployeeID, LastName & ", " & FirstName FROM Employees ORDER BY LastName, FirstName;
