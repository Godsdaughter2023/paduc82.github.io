ken sheridan say: It's not clear from your description whether you want to return only the year and count of NOTELIGIBLE rows, or whether you want to combine this with data from other columns in the table.

If you want only the year and counts in the report then you can simply group the query, e.g.

SELECT YEAR(field_date) AS field_year, nreg_stt_d,
COUNT(*) As StatusCount
FROM YourTable
GROUP BY YEAR(field_date), nreg_stt_d;

This will returns the counts per year per nreg_stt_d value.  If you only want the ineligible count add a WHERE clause:

SELECT YEAR(field_date) AS field_year, nreg_stt_d,
COUNT(*) As StatusCount
FROM YourTable
WHERE nreg_stt_d = "NOTELIGIBLE"
GROUP BY YEAR(field_date), nreg_stt_d;

If you want to return other columns from the table as well as the year and nreg_stt_d, then you can't group the query as the counts would be on the grouping by the other columns as well.  You can do the cont in a subquery which is correlated with the outer query on the nreg_stt_d colum:

SELECT *, YEAR(field_date) AS field_year,
    (SELECT COUNT(*)
      FROM YourTable AS T2
      WHERE T2.nreg_stt_d = T1.nreg_stt_d
       AND YEAR(T2.field_date) = YEAR(T1.field_date)) AS StatusCount
FROM Yourtable As T1;

This will return all columns from the table, plus a count of the rows with the same year and nreg_stt_d as the current row. Note how the two instances of the table are differentiated AS T1 and T2 to enable the subquery to be correlated with the outer query.    A report based on this query can  then simply bind a control to the StatusCount column.  Again you can restrict the results to the ineligible rows if you wish by adding a WHERE clause:

SELECT *, YEAR(field_date) AS field_year,
    (SELECT COUNT(*)
      FROM YourTable AS T2
      WHERE T2.nreg_stt_d = T1.nreg_stt_d
       AND YEAR(T2.field_date) = YEAR(T1.field_date)) AS StatusCount
FROM Yourtable As T1
WHERE nreg_stt_d = "NOTELIGIBLE";

The other way would be to do the counting in the report.  You could either do this by grouping the report on YEAR(field_date) and then on nreg_stt_d and putting a text box control in the nreg_stt_d group footer with a ControlSource property of:

= Count(*)

or you could group the report solely by YEAR(field_date), in which case the rows for each year would be grouped together regardless of thenreg_stt_d value, and in the group footer add a text box control  with a ControlSource property of:

=Sum(IIf([nreg_stt_d] = "NOTELIGIBLE",1,0))

The IIf function will return 1 or 0 depending on the value in the nreg_stt_d column.  Summing the ones is the same as counting the rows with a "NOTELIGIBLE" value.

ken sheridan say: PS:  You could also do it with the DCount function, either in a computed column in the query or in a computed control in the report:

DCount("*","terrest_archeological_point","nreg_stt_d = 'NOTELIGIBLE' AND YEAR(field_date) = " & YEAR([field_date]))
