Change 'line numbering' in margin from numeric to alphabetical - Microsoft Community

heathergreig say:

Change 'line numbering' in margin from numeric to alphabetical

I have created a Word 2013 document which includes 'line numbering' in the margin on every 10th line, however I need to change this from numeric 1, 2, 3 to alphabetic A, B, C

If I try to use 'Define new number format' on the Paragraph, numbering tab, it does not keep the 10th line spacing and enters a letter at every hard return instead.

Any suggestions please.

jay freedman say:

Word cannot automatically "number" lines with letters. The Line Numbering feature has no option to change the format of the numbers; and the paragraph numbering feature is exactly that, the sequential numbering of paragraphs rather than lines. (In Word, a paragraph is any length of text that ends with a paragraph mark, what you called a hard return.)

There is a way to fake it. This is best done only after all other editing has been completed, or else you'll have to go through the whole document to make adjustments.

  • Click in the first paragraph of text. Turn on nonprinting characters by clicking the ¶ button on the Home tab so you can see paragraph marks.
  • Click Insert > Text Box > Draw Text Box.
  • Draw a box in the left margin, from the bottom of the header area to the top of the footer area.
  • In the text box, insert nine line breaks (Shift+Enter).
  • After the last line break, press Ctrl+F9 to insert a field (it looks like a pair of braces, { } ).
  • Between the field braces, type this: SEQ a \*Alphabetic
  • Press F9 to update the field, which will now appear as the letter A.
  • Copy the set of nine line breaks, the field, and the paragraph mark at the end.
  • Put the cursor between the A and the paragraph mark and paste three times. You now see four As, one at each 10th line.
  • Press Ctrl+A to select everything in the text box, and press F9 to update all the fields. You should now have A through D on the first page.
  • If necessary, mo

    (Note to other readers: Don't suggest anchoring the text box in a header or footer. SEQ fields don't work anywhere but in the main text story.)

    a. user say:

    Hi Gustavo,

    Shared mailbox in Exchange Online allows a group of users to view and send email from a common mailbox. With regard to your questions, here are some understanding about shared Mailbox limitations in Office 365:

    • With Office 365 there's no limit on how many user-mailboxes a user can access.
    • It is free and do not require a license, but every user that accesses the Shared Mailbox must be assigned an Office 365 license.
    • It can't be accessed by users with Exchange Online Kiosk license.
    • It can be used to store emails sent to and received by the Shared Mailbox.
    • It can be used to store data migrated from on-premises Public Folders.
    • Each shared mailbox can be a maximum size of 50GB but shared mailboxes over 50GB in size need to be licensed.
    • A Shared mailbox doesn’t have a username and password and users cannot log into it directly. A user must sign in to his/her own mailbox and then open the shared mailbox using permissions.

    mouran yan msft support say:

    Hi Charlie,

     

    No, you don’t need to put any entry into NDS to have internal clients access the ADFS server because internal clients don’t need to access the ADFS server via the proxy.

     

    What you need to do is to create an A record to analyze the internal domain corp.domain.com as an IP address which belongs to the ADFS server.
