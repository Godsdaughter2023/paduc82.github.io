I have been experiencing the same issue.  Tonight, it finally got the better of me.  I waited more than two minutes after right-clicking a downloaded .exe and clicking Run As Administrator.
Found a fix or work-around for it. 

I right-clicked the .exe file and clicked Properties.

Noticed a Digital Signatures tab, so clicked that.

Highlighted the digital signature (in this case, QNAP), and clicked Details.

On the Digital Signature Details page, General Tab, clicked View Certificate.

On the Certificate page, General Tab, clicked Install Certificate.

On the Certificate Import Wizard Tab, chose Local Machine (instead of Current User), and clicked Next.

In order to make this change, was prompted to enter admin credentials; this worked flawlessly.

Chose "Automatically select the certificate store based on the type of certificate" and clicked Next.

Windows informed me that the certificate would be placed in the Content store.

Clicked Finish.

Right-clicked the original .exe file and clicked Run As Administrator - was Immediately prompted for credentials instead of being forced to wait several minutes.
