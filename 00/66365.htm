ntkrnlpa.exe: It is a kernel image file, located in system32 folder for Windows systems supporting PAE (Physical Address Extension). Hi helpjean,

Error seems to be pointing at a non-Hotmail or non-Live email account configured with Windows Live Mail which attempts to send an email with pictures or photo albums.

When Windows Live Mail sends pictures and albums, it automatically generates a photo album in Windows Live SkyDrive which is another service/feature available to those who signed up an account with Windows Live. This becomes a problem for non-Hotmail or non-Live accounts configured with Live Mail because their account can't connect to SkyDrive for the albums to be created.

I'd recommend that you attach the files using Insert > Single Photo in the ribbon menu. You still select multiple photos using that command but it won't be sent as an album, or an album won't be created in Windows Live SkyDrive.

Another way is to attach the photos. It won't display the photos in the email, but it will be sent as attachments to the email.

I also have additional questions for us to better settle your issue:

  • When you select all pictures in the folder, and then you click on Email button in the menu bar, does the "Attach Files" dialog box appear?
  • If yes, and you clicked on Attach, does the Live Mail client automatically open a compose box which contains the pictures to be sent?
  • Does the compose box contain any indication that the pictures will be sent as an album or will be have an album created for the pictures?
I'd recommend you try the workaround first. Most of the time it works.

rebekah001 say:

I finally found a workaround, and it was a bit of a pain but not awful. It took awhile, but it all works now.

I had to create two new user profiles for my kids in the accounts settings area (windows key+"i" > accounts > family and other users > add a family member). Even though each kid already had a "local" account.

Then we had each kid sign in to their newly-created profile with their Microsoft account one time. That automatically creates all the basic folders, settings, etc. for the profile.

Then, going back into my own administrative profile account, I went to windows menu > this PC > C drive > users, and for each kid's "old" profile there (the old local ones), I double-clicked the profile folder, and when it opened, I highlighted all the folders/files that were inside it (documents, pictures, etc.), clicked control-c to copy them, went back a level to the users again, single-clicked the "new" profile folder to highlight it (but not open it), and typed control-v to paste all of those files from the old profile folder into the new profile folder. I said "yes" to replace what was already in there when it asked, since the ones I was putting there were the legit ones replacing the stock empty ones that had just been created a few minutes before.

This basically created a duplicate new user profile for each kid, with all their photos, etc. inside. Of course I then had to tweak the settings by logging in as each kid (re-assign a desktop photo or theme, profile photo, relink google and gmail by logging into chrome to sync, etc.), so that was tedious.

But then when I was finally sure the new profiles looked and behaved just like the old -- and, most importantly -- we could now sign my kids into app store, cortana, all the expected things, and I doublechecked everything to be sure (!!), I got rid of the old profiles by going into the same accounts > family and other users settings area and deleting them.

Now it all works!

But be careful. When you've copied all the folders into the new profiles and etc., check and doublecheck all the data in the new ones before you delete the old. It's not as scary as it sounds, but you don't want to miss anything.
