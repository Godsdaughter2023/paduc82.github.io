ron sommer mvp 2009-2017 say:

This is from an unknown source>

In Windows Live Mail: All messages are stored as individual files. The database is an index containing the headers of each message along with information about the message's location and status. Occasionally, that information gets out of date in respect of some messages. The database will usually sort itself out on program shutdown and start-up, but once in a while it may be necessary to conduct a complete re-indexation to make sure that the information about each file is correct. This re-indexation is what is referred to - for historical reasons - as 'compaction'.

 

When a folder index gets out of date, the same message content shows for every message in a folder.

 

Alt + F, Options, Mail, Advanced, Maintenance, make sure that Compact the database is checked.

Change the number of shutdowns to 1.

Save.

Close Windows Live Mail.

Do not use the computer while the compaction is occurring.

After the compaction is finished, change the number of shutdowns to a higher number.

Edit text file in OneDrive - Microsoft Community

thomasrosendal say:

Edit text file in OneDrive

Hi,

We are multiple users that share a OneDrive directory and edit a text file. We recently encountered a bug that causes the last editor to save to overwrite the changes of the other simultaneous editors. For example, it Bob and Alice both open foo.txt at the same time and Bob adds the line 'abc' to the end of the file and clicks SAVE. Then Alice doesn't see that this has happened and adds something else to the file and clicks SAVE. foo.txt now doesn't contain the line abc. It exists in the version history but not in the latest version as it should. The problem is transparency, as the users think everything is working but really work is being lost. Three possible solutions:

1. Lock the file for editing

2. Enable simultaneous cursors for realtime collaboration as other tools do.

3. Every SAVE click is a git commit, if you have changes that don't conflict you move on and see the other editors changes. Conflict results in the last person to SAVE(commit) must resolve the conflicts.

diane poremsky mvp (slipstick.com) say:
Unless the text editor is multi-user-aware, the last save will update the file to what the users see. Text editors don't generally lock a file when it is opened - older versions of word will, but the change needs to sync to the other users before they open it. If its locked when they save, the file will be saved as 'filename-computername"

thomasrosendal say: In the scenario described above, both Bob and Alice are using the web-based OneDrive text editor and are modifying a file that exists on OneDrive. If Bob and Alice were responsible for the file system, ie working on a local file, they would also accept the responsibility of providing version control or file locking. Since they are working in the cloud this is out of their control and a feature that allows them both to silently overwrite each other's changes is counterintuitive.

In general the links you have on your home page that are failing, are links that have been converted to an image, thus killing the links.

An example is the link to 'Patrice's Horse blanket...". Here is a link to that image:

http://www.arborinasnursery.com/index_files/image342.gif

When Publisher attempts to convert some print formatting options to HTML or web format it will choke when there is no HTML equivalent and do the best it can by producing an image. One way you can tell if text has been converted to an image is to do a web page preview and then try to left click drag to select the text. If you can't select the text it has been converted to an image.

My guess is that you have grouped the 'Patrice's Horse blanket..." text box/link with the rectangle shape or bullet you put in front of that text box/link. Select that group > Arrange > Ungroup and do a web page preview (File > Web page preview) to see if that fixes the link.

If that fixes the problem then I suspect the other dead links on the home page are killed for the same reason. Another example:

http://www.arborinasnursery.com/index_files/image346.gif

Try ungrouping the links to repair. If that does not fix the link then there is some other formatting issue. If you have other specific links that are failing and can not figure out why, post back with the specifics and I will try to help you.

Though you did not ask I would also suggest that you go to Tools > Web Page options > and change the home page title to something like 'Arborinas Nursery' instead of 'index'.  Or something like 'Arborinas Nursery - (some sort of short summary of your specialization)'. This title is most important for google indexing and page ranking. Give the web crawling bots something more descriptive to read and save for your site other than 'index'. Here is an example of a Pub 2007 generated site with a much more descriptive and web search friendly title 'Devon View Caravans, privately owned caravans at Devon Cliffs HolidayPark' : http://devonviewcaravans.co.uk/

I would also suggest an alternative way of generating your web files. Instead of doing a File > Save As a web page-filtered, after you have finished editing your publication, saved the changes and are ready to update you site, then go to File > Publish to the Web and direct your output to a folder you can easily find with FileZilla. This will produce an index.htm file (your home page) and an index_files folder that contains your other web pages and all your supporting images and graphics. Then use FileZilla, find that default folder, select the index.htm file and the index_files folder within that folder and upload to your web host.

Distortion from a headphone or audio device usually happens when the sound within the source is driven to a higher output level than what the hardware can handle. These devices are designed to handle a certain level of volume.  Once they pass a certain level, they will distort the sound.

  6. Go to ‘Exclusive Mode’ section in advanced tab. Uncheck the option ‘Allow applications to take exclusive control of this device’.
  7. Click on Apply and Ok. Restart the computer and check for the issue.
