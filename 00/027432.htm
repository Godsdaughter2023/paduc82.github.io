a. user say:

How to do this depends partly on how you data is set up, but the simplest way is probably to do a 2-step merge, where you
 a. create a letter from your data that contains all the recipient names and the food items for each one.
 b. either...
 b. If you are emailing, you could then 
    - if you are emailing, consider sending that document to all the recipients - the usual way to do it is to put your own address in the Cc (Copy) field and everyone else's in the Bcc (Blind copy) field. They will all get the same letter. Or
    - if you want to personalise the letters/emails, you use the document created in (a) as your mailmerge main document, and add whatever MERGEFIELDS you want (e.g. recipient name). Connect that to a data source that has one recpient per record, and perform the merge.

For example, if your data is in a table like this (in Excel, Word, whatever):

Recipient name Recipient email Food items
Zetland z@abc item1; item2; item3
Ystad y@def item4
Xhoffraix x@ghi item5; item6
.
.

then you could
1. copy the data into a Word document.
2. If it isn't inserted as a Word table of some kind, you can select the data and use Table->COnvert text to table to get a table
3. Delete any columns you do not need
4. Format the table and the rest of the letter

If you want to personalise, it, add other text and Mergefields, e.g.

Dear { MERGEFIELD "Recipient name" }

(where the { MERGEFIELD } inserted from the Mailings tab in the usual way

If you want to pick out the food items for that recipient, you could use something like

As discussed, can you please bring { MERGEFIELD "Food items" }

(again, it depends on the layout of your data).

Then perform the merge.

If your data layout is different, it will help if you can describe what you have (a word file? An excel sheet? What's in each row? Etc.)
