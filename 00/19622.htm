doug robbins - mvp office apps & services (word) say:

We really need more information on how the location of the split is to be determined.

To split a document into individual pages:

Sub splitter()
'
' splitter Macro
' Macro created 16-08-98 by Doug Robbins to save each page of a document
' as a separate file with the name Page#.DOC
'
Dim Counter As Long, Source As Document, Target As Document
Set Source = ActiveDocument
Selection.HomeKey Unit:=wdStory
Pages = Source.BuiltInDocumentProperties(wdPropertyPages)
Counter = 0
While Counter < Pages
    Counter = Counter + 1
    DocName = "Page" & Format(Counter)
    Source.Bookmarks("\Page").Range.Cut
    Set Target = Documents.Add
    Target.Range.Paste
    Target.SaveAs FileName:=DocName
    Target.Close
Wend
End Sub

If there is a Section break between each letter, then use:

Sub splitter()
' splitter Macro
' Macro created by Doug Robbins to save each letter created by a mailmerge
' as a separate file, retaining the header and footer information.
Dim i As Long, Source As Document, Target As Document, Letter As Range
Set Source = ActiveDocument
For i = 1 To Source.Sections.Count
    Set Letter = Source.Sections(i).Range
    Set Target = Documents.Add
    Target.Range = Letter
    Target.Sections(2).PageSetup.SectionStart = wdSectionContinuous
    Target.SaveAs FileName:="Letter" & i
    Target.Close
Next i
End Sub

If neither of those do what you want, provide some more information on how the location of the split can be identified and it will be possible to devise the necessary code to do it.

Simple fix: Turn off the Wildcard setting and click Find before closing the dialog.

steve rindsberg say:

Ben, what you're missing is the very useful old recolor feature from earlier versions of PPT.  The one that got tossed out when all of the new fancy effects got added. 

Most image editors will let you do color replacements (you may need to convert to 8-bit or lower color to make it work satisfactorily).  You can probably copy/paste from PPT into an editor, save as a file, edit, then insert the file back into PPT. 

In some cases, you can safely copy/paste from the editor back into PPT; in other cases, that'll give you an OLE object, which will tend to be heavier (though it'll let you doubleclick to edit it further, so might be useful.)

As to your other question, try copying the graphics in AI then Paste Special | As EMF into PPT.

rainald say:

What's the purpose of "Unpack" process?

Easy enough:
"pkg" stands for "package". This indicates that the file contains a
package. All parts of a notebook - folders ad subfolders for section
groups ad ".one"-files for sections "packed" into one file (like folders
in file packed into a ZIP-file).
To use the content of a .onepkg-file the file has to be "unpacked" (just
like a ZIP-file) in order to create a notebook from it.

So in your laptop a notebook will be created if you unpack the file.
If you stop the process by clicking the Cancel-button no notebook will
be created.

Before returning my laptop for repair, I saved my OneNote notebook as
a onepkg file, then copied it onto my desktop PC. When I double click
on this file on my desktop PC, it opens the file and at the same time
prompts the "Unpack notebook" window, there's a "Create" button which
unpacks the file to the "C:\Users\ID\Documents\OneNote Notebooks", if
click "Cancel", nothing happens, I can still work on it properly.

Obviously you have the notebook in question already open in ON in your
desktop.
If you cancel the unpacking nothing happens. If you click "Create" and
the notebook already exist in the target folder you'll receive a
warning.

I copied that onepkg file from desktop to laptop, then double click
it, the "Unpack notebook" window still pops up, but if click "Cancel",
it won't open the file!

That's absolutely correct. If you want to create the notebook you have
to use the "Create" button.

Why it's different?

The sub-folder with the notebook does ´not yet exist.

Rainald
P.S. As you seem to store the same notebooks in your desktop and your
laptop a warning:
Don't do that. The way to work with ON in multiple computers is storing
the notebook-files only in one computer and opening them "shared" on
the other(s). If any questions in so far, pls ask in a new thread.
