cortig say:

You can copy it manually and restore it manually, but you’re heading for major disasters if you do so. The database would then be disconnected from changes in the individual files that compose the identity and there is a serious risk of data loss.
That’s why the file is flagged (by Outlook) not to be backed-up by Time Machine! The backup and restore restores all the data that compose the identity and forces Outlook to rebuild this "index".
I’m not sure I see what’s wrong with that, especially since it ensures that a fresh new optimized version of the index is being recreated.
Sure it’s a bit more time consuming, but it’s well worth it from what I can see.

If the rebuild fails, it’s because you have some sort of major corruption. Restoring the old database file would not correct that issue: you would simply not be aware of it…
