rich007a say:

Maybe you could use the WindowSelectionChange Application event...

See http://word.mvps.org/faqs/macrosvba/appclassevents.htm for instructions on how to set up Application Events in Word.  You would then be able to type normally, but as soon as you click on a word or move the cursor with the arrow keys over a word the Thes procedure would run.  Follow the instructions in the above link.  Then:

in the normal module:

Option Explicit

Dim oAppClass As New ThisApplication

Public Sub AutoExec()
     Set oAppClass.oApp = Word.Application
End Sub

Sub Thes()
    Dim rng As Range
   
    Set rng = Selection.Range
    Application.CommandBars.ExecuteMso ("Thesaurus")
    rng.Select
End Sub

in the ThisApplication class module:

Option Explicit

Public WithEvents oApp As Word.Application

Private Sub oApp_WindowSelectionChange(ByVal Sel As Selection)
    Call Thes
End Sub

Cheers
Rich

PS. And you could modify the oApp_WindowSelectionChange procedure so it only calls Thes when the Thesaurus has been manually opened by you (to allow you to close it and it will stay closed) like this:

Private Sub oApp_WindowSelectionChange(ByVal Sel As Selection)
    If Application.CommandBars.GetPressedMso("ResearchPane") Then Call Thes
End Sub

mathildelundsberg say:

Don't know if you're still looking for a solution but thought I would share anyway as I spent so long looking for something like this. It works for me so hopefully does for other people too. I posted this as a possible workaround on the UserVoice thread but thought I would post it somewhere on Microsoft's own site too. FYI I have Outlook for Mac v16.14 on MacOS High Sierra v10.13.5.

I’ve been looking for a solution to this for months and I finally figured out a workaround

Even-though Outlook for Mac doesn't directly support Aliases, you can set up a new account for an alias as if it were a separate e-mail account, however your emails and other items will all appear twice. 

To avoid this, you want to set up a new account that doesn’t connect to the incoming server for your email, so that it won’t receive e-mail (so you don't get duplicates of everything) but is still able to send from the correct outgoing server - so that emails can be sent from the alias address rather than the primary address.

To do this, first, set up a new IMAP account for the alias email address. (Preferences -> Accounts; “+” in the bottom left corner -> New account. Enter your alias email address; click continue; click "Choose the provider" or "Not Outlook.com?" in the top right corner if choose provider isn't an option; then select IMAP/POP at the bottom).

Check and correct the following to:

 

Type: IMAP

 

Email Address: Alias email address

Username: Alias email address

 

Incoming server: imap-mail.outlook.com

(Incoming server) port: 993

 

Outgoing server: smtp-mail.outlook.com
(Outgoing server) port: 587

 

(Both “Use SSL to connect (recommended)" are ticked).

 

When the account has been added, go to the account’s preferences and for the alias account:

  • Tick "Override default port" underneath incoming server
  • Change the incoming server to smtp-mail.outlook.com (same as the outgoing server)
  • Change the incoming server port to 587 (same as the outgoing server)
  • Click "More Options..." underneath outgoing server
  • Change "Authentication" to "User Name and Password"
  • Enter your alias email address and account password and click OK
  • Check that the email address under "Personal Information" is the alias - this is the address used to determine where the emails was sent from

 

When you click away from the text boxes, the new account should disappear from the mail sidebar as it doesn’t receive anything, but you will see the alias as an option everywhere else that you can select an email address, including in the from drop-down menu when composing an email. You can check that the IMAP account folder settings (Advanced -> Folders) are all set to server folders, so that drafts and emails sent from the alias show up in your normal account folders on the desktop outlook app as they are put in the sent folder on the server.

(Extra info: An incoming server receives data but as the SMTP server is an outgoing server, it doesn’t do this and can't receive email (but it is still recognised by the outlook app) so the account isn’t connect to your real incoming email server, or any other incoming server, and won’t receive anything – a.k.a a send only account. Changing the outgoing authentication allows the app to “sign in” and send emails from the account even though it can’t connect to the incoming server.)

Troubleshooting: For some unknown reason, the first time I added an alias account it stayed in the sidebar and I got the “Mail could not be received at this time” message. Every time I repeated the process, with the same alias, to check it or to add other aliases, the account went away from the sidebar immediately and I haven’t got any other error messages. 

If these happen then:

  • Try repeating the steps to add a new account for the same alias and then delete the first attempt afterwards (minus sign in the accounts preferences). 

Otherwise: 

  • Empty the account’s folders so that the emails don’t show up in the all accounts inbox, by right-clicking on each of the folders and selecting repair folder (this replaces the items in the folder with the items from the server - as the account can't connect to the server, the folder will becomes empty). I’ve tried loads of ways and haven’t been able to make the new account stay in the sidebar since the first time I tried this so I can’t say what made the difference.)
  • Click “No” on messages asking you to try re-entering your password for the alias account - this will stop it from trying to connect and you won’t get any more error messages. (The message comes up because the server thinks the username or password is wrong when it can’t connect.)
