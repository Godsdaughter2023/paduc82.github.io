rex zhang say:

Hi,

 

 

First, Close Access and reboot the computer, go to the folder holding the database files and see if there are any file names with a .ldb suffix, Delete all the .ldb files.

 

There are several possible causes:

 

1.There is a corruption in the database application.

2.The record has been locked by a user and Microsoft Access or another user is attempting to modify the record.

3.VBA code has opened a recordset and has a lock on a record.

4.Two or more instances of the Microsoft Access application is running on the same PC.

5.The application is using page record locking and has exceeded the 4K limit (2K limit for Jet database engine version 3.5x and earlier). 

6.The back-end application resides on a Novell Server and the maximum record locks has been exceeded.

    Add the gmail account as "Other email accounts". Enter any nonsense address and password you want, then click Next. When the connection fails you'll get a new dialog allowing you to choose all the settings for username/address, password, POP or IMAP, server, ports, and encryption. You can then add the gmail information. Specify a nonsense POP server, but specify the correct SMTP server. You should be able to send through the account. If you have two-step authentication enabled for the gmail account, be sure to generate an app password on the gmail end to use when specifying the password when setting up the connected account on the Outlook.com side.
