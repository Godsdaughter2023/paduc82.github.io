Macro for creating hyperlink to MS Project tasks - Microsoft Community

paulkuznetc say:

Macro for creating hyperlink to MS Project tasks

Hello! I have writteh this code for Outlook macro to creating hyperlinks to different Outlook objects. So, is it possible to make something like this for MS Project? I want to create hyperlinks to tasks and other Project objekts and add them to clipboard

Sub CopyLink() 
    Dim olTemp As Outlook.MailItem 
    Set olTemp = CreateItem(olMailItem) 
     
    Dim olInsp As Outlook.Inspector 
    Dim wdDoc As Object 
    Dim oLink As Object 
    Dim oRng As Object 
     
    Dim linkName 
    Dim linkID 
     
    Dim doClipboard As DataObject 
    Set doClipboard = New DataObject 
     
     'One and ONLY one message muse be selected
    If Application.ActiveExplorer.Selection.Count <> 1 Then 
        MsgBox ("One and ONLY one message muse be selected") 
        Exit Sub 
    End If 
     
    Dim SelectedItem 
    SelectedItem = TypeName(Application.ActiveExplorer.Selection.Item(1)) 
     
    If SelectedItem = "MailItem" Then 
        Dim objMail As Outlook.MailItem 
        Set objMail = Application.ActiveExplorer.Selection.Item(1) 
        linkName = objMail.Subject 
        linkID = objMail.EntryID 
    Else 
        If SelectedItem = "TaskItem" Then 
            Dim objTask As Outlook.TaskItem 
            Set objTask = Application.ActiveExplorer.Selection.Item(1) 
            linkName = objTask.Subject 
            linkID = objTask.EntryID 
        Else 
            If SelectedItem = "ContactItem" Then 
                Dim objContact As Outlook.ContactItem 
                Set objContact = Application.ActiveExplorer.Selection.Item(1) 
                linkName = objContact.FullName 
                linkID = objContact.EntryID 
            Else 
                If SelectedItem = "NoteItem" Then 
                    Dim objNote As Outlook.NoteItem 
                    Set objNote = Application.ActiveExplorer.Selection.Item(1) 
                    linkName = objNote.Subject 
                    linkID = objNote.EntryID 
                Else 
                    If SelectedItem = "AppointmentItem" Then 
                        Dim objAppoint As Outlook.AppointmentItem 
                        Set objAppoint = Application.ActiveExplorer.Selection.Item(1) 
                        linkName = objAppoint.Subject 
                        linkID = objAppoint.EntryID 
                    Else 
                        MsgBox ("Invalid object selected") 
                        Exit Sub 
                    End If 
                End If 
            End If 
        End If 
    End If 
     
     'Create temp message to copy hyperlink to clipboard
    With olTemp 
        .BodyFormat = olFormatHTML 
        Set olInsp = .GetInspector 
        Set wdDoc = olInsp.WordEditor 
        Set oRng = wdDoc.Range 
        .Display 
        oRng.Text = "" 
        Set oLink = wdDoc.Hyperlinks.Add(Anchor:=oRng, _ 
        Address:="outlook:" + linkID, _ 
        SubAddress:="", _ 
        ScreenTip:="", _ 
        TextToDisplay:=linkName) 
         
         
         'font correction
        oLink.Range.Font.Name = "Adobe Fan Heiti Std B" 
        oLink.Range.Font.Size = 10 
         'oLink.Range.Font.Color = vbRed
         
         
        Set oRng = oLink.Range 
        oRng.Copy 
        olTemp.Close olDiscard 
    End With 
     
End Sub

    albert d. kallal say:

    I done this for many years now and it works fine. The only real issue here is if your web provider allows external connections to their database or not.

    It varies quite a bit from provider to provider if they allow external database connections are not.

    Often, some providers don't allow external connections and this result in a more secure system as then ONLY THEIR internal web servers are allowed to pull data from the database server (and not any desktop client programs).

    However a good number of providers do allow those external connections since often you need to upload data, make changes to the database table(s), and even send data up from some kind of "data collection" device and then make such data available in a web site. In fact, for those that do allow connections, then I find that SSMS (Sql Server Management Studio) works just fine.

    And if you optimize your application for performance, then the setup works really quite well. So
    just keep in mind the typical optimizing is required for good performance.

    So opening up a form bound to a large table is a bad idea, but if you execute an open form with a "where" clause to restrict records loaded, then even bound forms will only pull down the one record – it works well indeed.

    Also, keep in mind that for Access 2010, you can also place your data tables on office 365. This setup can often work better then SQL server, and it even works if your internet connection breaks (the data is entered local – when you get a connection, it starts to sync).

    So you can deploy a front end to different locations, and use some type of SQL server, and you can also consider using office 365 for the back end, as this can work well with many types of applications.
