karl timmermans [outlook mvp 2012-2018] say:

Every item in Outlook is assigned a unique "EntryID" whenever it is "created". Importing anything will result in a new "EntryID" for what you "think" is the same contact item. In this case -  "the same = visible pieces of information" but to Outlook a newly imported item also becomes a "new item" rendering all "links" to the original useless.

Move an item to a "different" information store (i.e. PST file) or otherwise change anything significant and the results will be the same - EntryIDs change - links are broken.

Your option - will have to go through each contact with broken links and re-link them to the correct contact item. There is no self-correcting function to go through contact "link collections" in bulk which will figure out the new links for broken links.

scottgem say:
Hi Hound, I'm an independent advisor and will try to help.


The first part would be to use the SendObject command. This access command can take a query, and output it to an Excel file as an attachment to an e-mail. You can specify a recipient, subject and text for the e-mail. Just search on SendObject for more info on how to use it.

The second part is automating this on a daily basis. Access can only send something if its open. So you have to use Task Scheduler to open your Access app daily. Now you don't want to open your main app, so you need to create a new front end for this task. You want to setup an autoexec macro that will process the SendObject command and then Quit the Access app.

If you need further clarification on this, feel free to ask specific questions.
