ken sheridan say:

How do I get the on NoData to go ahead and continue this output?

You can't per se.  A possible solution would be to create a separate dummy report with a RecordSource which will only return one row, and whose controls in the detail section are empty unbound controls, or possibly with values such as 'N/A' or similar, but which otherwise mirrors the layout of the main report.  You could then include code in the main report's NoData event procedure which both cancels the main report and opens the dummy report, which can then be exported to Excel.

The RecordSource for the dummy report need have no connexion with the main report's data at all.  All that matters is that it returns a single row, so that a single detail will print.  So, it could be based on any table, e.g.

SELECT MIN(EmployeeID) AS Dummy
FROM Employees;
