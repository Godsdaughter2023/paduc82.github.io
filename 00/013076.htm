Re:  copy some rows to another sheet

I changed your code to loop thru each cell in the selected column.
Also, the Instr function was used to determine if the particular text occurred anywhere within each cell.
To use the code you must select the data before running the code.
Only the first column of the selection is acted upon...
'---

Sub MoveEmOut()
 Dim LCopyToRow As Long
 Dim rng As Excel.Range
 Dim rngCell As Excel.Range
 Const strBoot As String = "reboot"
 Const strIPL As String = "IPL"
 
 On Error GoTo Err_Execute
 
 Set rng = Selection.Columns(1).Cells
 LCopyToRow = 2
 For Each rngCell In rng
    If VBA.InStr(1, rngCell.Value, strBoot, vbTextCompare) > 0 Or _
       VBA.InStr(1, rngCell.Value, strIPL, vbTextCompare) > 0 Then
       Worksheets("Sheet2").Rows(LCopyToRow).Value = rngCell.EntireRow.Value
       LCopyToRow = LCopyToRow + 1
    End If
 Next 'rngCell
 
 MsgBox "All matching data has been copied."
 Set rngCell = Nothing
 Set rng = Nothing
 Exit Sub
 
Err_Execute:
 MsgBox "Error " & Err.Number & " occurred.  "
End Sub

from a win7 .bat file how can I tell if a MSE scan is finished so the - Microsoft Community

gregory luznicky say:

from a win7 .bat file how can I tell if a MSE scan is finished so the pc can be put back to sleep

In my win 7 .bat fle i scheduel a .bat task to wake the pc and put it in the no standby power option. Mcrosoft Scurity essentials does full scan. I need to know when it is finished so I can put the PC back on stndby.
greginmich say:

It’s actually a little more complicated than the prior answer here indicated, especially for Windows7 users:

 

Batch file commands will be processed sequentially, so there’s no need to determine the status of an MSE scan in a batch file. There’s also no need to change the sleep timeout setting (in order to keep the computer from idling to sleep), or to put the machine back to sleep, when you’re running a Full Scan in Task Scheduler – because Task Scheduler will automatically do these things if you just check the “Wake the computer to run this task” option as described here:

 

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02

 

The above thread also shows you how to add a shutdown.exe command to your scan task without using a batch file.

 

The complication comes in when we want to run an unattended Full Scan directly with a batch file, because in that case we actually do need to change the power settings with powercfg.exe in order to keep the PC from idling to sleep (In Windows 7).

 

 

This example changes the sleep timeout to allow for an unattended scan of the entire system drive, and then resets the sleep timeout when the scan is finished:

 

@echo off

 

powercfg -change -standby-timeout-ac 200

 

cd C:\Program Files\Microsoft Security Client

 

MpCmdRun.exe -SignatureUpdate

 

MpCmdRun.exe -Scan -ScanType 3 -File C:\

 

powercfg -change -standby-timeout-ac 20

 

 

This example runs an unattended Full Scan (of all internal drives) and then shuts the computer down:

 

@echo off

 

powercfg -change -standby-timeout-ac 200

 

Cd C:\Program Files\Microsoft Security Client

 

MpCmdRun.exe -Scan -ScanType 2

 

powercfg -change -standby-timeout-ac 20

 

shutdown -s -t 300

 

 

We can also modify SpiritX’s command line slightly so that it copies the MpCmdRun.exe syntax directly to the desktop:

 

"C:\Program Files\Microsoft Security Client\MpCmdRun.exe" -h >Desktop\MSEcmd.txt

 

 

 

GreginMich
