a. user say:

This has long been a nasty area within Word, but I believe that the origin of this problem is described in the following article:

http://support.microsoft.com/kb/286880

and that it will occur as soon as you have an OLE DB connection to a data source and specify "is blank" or perhaps "is not blank" as one of the criteria in your "Advanced criteria". 

It is possible that this problem has been sorted out in one of the Word 2007 SPs, but I don't think so.

As far as I know, your main options are:

 a. revert to using DDE. (click Word Office button->Word Options->Advanced->General and check "Confirm file format conversion on open", then go through the connection process again and select the DDE option in the "Confirm Data Source" dialog (check "Show All" if you don't see the DDE option, e.g. with a .xlsx data source, and use the Excel via DDE option even though it doesn't mention .xlsx). However, DDE connections to Excel sources can be unreliable in Word 2007.

 b. If you are setting up merges where the filter criteria do not need to change, you can consider modifying the SQL Query string directly in Word VBA. As long as you do not need to go back into the Query options dialog, the SQL that you specify should "stick". 

[[some additional info

NB, the article states that 

" 
The SQL string for the expression "MyField Is blank" is: 
(MyField IS NULL) OR (MyField IS '') 
"

which is 
  a. syntactically incorrect - it should say 
(MyField IS NULL) OR (MyField = '') 
  b. arguable. It all depends on what they mean by "blank". Do they mean 
     1 has one or more space characters (specifically, ASCII/Unicode 
character 32) 
     2 has one or more "white space characters" (spaces, tabs etc.) 
     3 has a zero-length string (which in Excel you could insert using 
the formula ="") 
     4 is null (which in Excel appears to correspond to a cell with no 
text or formula in it 
     5 something else? 
     6 some combination of the above?

The Jet SQL that is used to get data from Excel actually retrieves both (1) and (3) using Myfield = '', and (4) using myfield IS NULL. So I guess to cover the ground, Microsoft more or less has to have both expressions in its SQL. The problem is that ODSO takes a simplistic view of the data source, and Word+ODSO do not work together properly to maintain the overall expression correctly. 

FWIW, ODSO appears to me to have been written by someone who knows little about SQL as it makes quite a mess in other areas too, but I live in hope that some resource might one day go into fixing it.

Unhiding rows in Excel - Microsoft Community

lady di 50 say:

Unhiding rows in Excel

Can you unhide multiple rows at one time?
lars-Åke aspelin say:

Yes, that can be done.

For each row that is hided and you want to unihide do as follows:

While pushing the CTRL button select the row immediately above a hidden row
While pushing the SHIFT button select the row immediately below a hidden row.
Right click on any of the selected rows and choose Unhide.

Repeat the above steps for all hidden rows that you want to unihide.

If there are no hidden rows that you would like to keep hidden between a pair of hidden lines that you want to unhide you can simplify this to the following:

While pushing the CTRL button,  select any row  above the first row that you want to unihide (without having any row that you want to keep hidden inbetween),
While pushinf the SHIFT button. select any row below the last row that you want to unhide (without having any row that you want to keep hidden inbetween).
Right click on any of the selected rows and choose Unhide.

If you want to unhide only a subset of a number of adjacent rows that are hidden, I think you have to do this via a macro.

scott .. b say:

Unfortunately, there is no way to make the IE8 browser selection in f12 Developer tools a default.  That is because it only deals with the design of the current (open) page.  As you discovered, when you move to the next page, it has to be set again.

 

There is one thing that can help some, and that is to "pin" the f12 ribbon onto the bottom of Internet Explorer, so that it is always accessible (sort of like a toolbar) and when you change pages, you can just reach down to the ribbon and change the browser to IE8 with one click.

 

To do that, drag the window closed until it is just the top strip (with "Browser", Quirks, et) and then select the "Pin" icon on the far right or press Ctrl + P.  Cumbersome to be sure, but it does offer at least some help until the website developers can catch up with IE10 standards.
