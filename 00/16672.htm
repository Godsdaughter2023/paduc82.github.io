ken sheridan say:

The Column property is zero based, so the third column is, as Tom pointed out, Column(2).  The ControlSource property for an unbound text box in the form would therefore be:

=[cboEmployer].Column(2)

Change cboEmployer to the actual name of the combo box.  You'll probably find Access will throw a few unnecessary square brackets into the mix.  They are harmless.

Another way, particularly where you want to show multiple values from a referenced table is to base the form on a query which joins the referenced table (Employers in your case) to the form's current table.  You can then simply bind controls to the columns in the referenced table, setting the Locked property of each control to True (Yes), and the Enabled property to False (No) to make the controls read-only.  You'll find an example in InvoicePDF.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

tom van stiphout say:

Why are you copying the PhoneNumber from Employers to Contacts? That is a violation of proper database design. Rather, any time you're working with a Contact and you want the employer's phone (or any other field in tblEmployers), you simply create a query that joins the two tables and picks up the employer fields. The only employer field in the contacts table should be EmployerID.

That said, on your Contact form you can certainly DISPLAY the employer's phone. The simplest technique is to add PhoneNumber to the query for the Employer dropdown. Let's say it's the 3rd column. Then you can have an unbound textbox and set the ControlSource to:

=myEmployerCombobox.Column(2)

rich michaels - mvp (greatcirclelearning.com) say:

You are trying to work outside the bounds of how a list works and that is what is causing the problems you see.

Lists are a series of paragraphs linked by a List Style. The list can have up to 9 levels and the formatting of each, including indentation, symbol preceding the text, spacing between symbol and text, etc are all predefined. Word gives you a number of these pre-defined list setups that you can use. There is a set for numbered lists and another for bulleted lists.

However, what is sounds like you are attempting to do is combine some pieces from various list sets and you are doing that by direct formatting of the paragraph once you've entered it into the document. That does not work... and you are seeing the consequences.

I suggest you use only the level 1 number list item. When you need to insert a level 2 item, instead of a list paragraph, use a custom paragraph style that is formatted as you would like it with indentation, a bullet or number, etc. Setup multiple styles like this, don't direct format! So you will have a level 2 looking style with a bullet symbol and another level 2 looking style with a number.

Hope this helps
