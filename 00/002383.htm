Excel 2013 conditional formatting not working with imported SharePoint data

I have exported data from a SharePoint List into Excel 2013 (List -> Export to Excel) and set the conditional formatting of a single cell to change background color when its value equals 1.

The conditional formatting *does not* work when the data is refreshed from SharePoint.

The conditional formatting *does* work when I manually type in the value 1.

Is it possible to use conditional formatting with data refreshed from a SharePoint List?

tom ogilvy say:

test the cell.

=isnumber(A1)

as an example.  If that says false, then the "number" in the cell is actually the text character 1.  So adjust your conditional formatting to recognize both "1" and 1 as cause to act.

one way might be

=Countif(A1,1)

If countif doesn't return a zero then you know the cell contains either a "1" or a 1 and the conditional formatting will act.  (any nonzero value will be considered TRUE).

How to rerun a query in a form, after entering a new record? - Microsoft Community

nursejohn say:

How to rerun a query in a form, after entering a new record?

Hey all.  I'm a total rookie when it comes to Access.

I am a school nurse and I need to create a form to enter records in a TBLVisitLog, linked by a StudentID to a TBLStudentRecords.  First I want the user to enter the student's name to pull up his/her unique Student ID, and then enter the new record in the TBLVisitLog.  Once the new record is entered, I want the "student name" query to automatically run again.  Is that possible?

ken sheridan say: You can easily do this with a form bound to a query on the TBLVisitLog table which references an unbound combo box in the same form's header.    Create a form frmVisitLog say, with an unbound combo box in its header set up as follows:

Name:  cboStudent

RowSource:     SELECT StudentID, FirstName & " " & LastName FROM TBLStudentRecords ORDER BY LastName, FirstName;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column.

The query for the form's RecordSource property will be along these lines:

PARAMETERS Forms!frmVisitLog!cboStudent LONG;
SELECT *
FROM TBLVisitLog
WHERE StudentID = Forms!frmVisitLog!cboStudent
ORDER BY VisitDate;

In the AfterUpdate event procedure of the combo box put the following line of code:

Me.Requery

When a student is selected by name the form will be restricted to the records with a StudentID value which matches that of the hidden column in the combo box.  The form can be in single form view or continuous forms view.  The ORDER BY clause in the query will cause the records to be shown in visit date order (change the column name to your actual date column's name, and remember that if a column name includes spaces or other special characters it must be wrapped in square brackets in the query, e.g. [Visit Date].
