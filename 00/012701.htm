john mcghie say:
If you have nuked your font cache, you must re-start the computer.  The Office font cache is rebuilt from the System Font Cache which is rebuilt when the computer restarts.

You also need to visit OpenType fonts to learn what kind of image they contain: TrueType images are fine, PostScript images are not.  An OpenType font can contain either.

And some fonts are simply too old, or encoded to the wrong format.  You will have a much better chance with Unicode fonts than with any other system.

Finally, Word's font list often sorts the font names out of sequence.  It's a bug: it's sorting on the wrong name field.  But that normally doesn't matter, because most Wordies never use the font list.  If you define the fonts you want to use into the styles you normally use, then you never have to look at the font list :-)

dirk goldgar say:

I'm still not sure I understand what you're really looking for.  But if you want to find all the Prefixes that are unique to each table -- all that appear only in Table1 and not in Table2 or Table3, all that appear only in Table2, and all that appear only in Table3 -- then you can forget about the union query we've been talking about and run three separate queries, one for each table.

For Table1, you'd have a query with this SQL:

SELECT DISTINCT Table1.Prefix
FROM (Table1 LEFT JOIN Table2 ON Table1.Prefix = Table2.Prefix)

WHERE Table2.Prefix Is Null AND Table3.Prefix Is Null;

For Table2, you'd have a query with this SQL:

SELECT DISTINCT Table2.Prefix
FROM (Table2 LEFT JOIN Table1 ON Table2.Prefix = Table1.Prefix)

LEFT JOIN Table3 ON Table1.Prefix = Table3.Prefix
WHERE Table1.Prefix Is Null AND Table3.Prefix Is Null;

And for Table3, you'd have a query with this SQL:

SELECT DISTINCT Table3A.Prefix
FROM (Table3A LEFT JOIN Table1A ON Table3A.Prefix = Table1A.Prefix)
LEFT JOIN Table2A ON Table3A.Prefix = Table2A.Prefix
WHERE Table1A.Prefix Is Null AND Table2A.Prefix Is Null;

run macro when cell becomes 'dirty' - Microsoft Community

wwk say:

run macro when cell becomes 'dirty'

Could anyone can help to solve my below questions?!

How could I run my own macro automatically (i.e. run_advFilter) once the value of cell A10 change (get dirty)  either by typing something / by paste the value from other worksheet ..)

Many thanks!!

 

hansv mvp say:

Right-click the sheet tab.

Select View Code from the context menu.

This activates the worksheet module.

Create code like this:

 

Private Sub Worksheet_Change(ByVal Target As Range)
    On Error GoTo ErrorHandler
    ' Has A10 changed?
    If Not Intersect(Range("A10"), Target) Is Nothing Then
        ' Prevent a cascade of events
        Application.EnableEvents = False
        ' Call macro - change name to that of your macro
        Call YourMacro
        ' Turn on event handling again
        Application.EnableEvents = True
    End If
    ' Done
    Exit Sub
ErrorHandler:
    ' Report error
    MsgBox Err.Description, vbExclamation
    ' Make sure that event handling is turned on
    Application.EnableEvents = True
End Sub

ben m. schorr say:

Actually what you do is not write the ink in the cell but rather you open the pop up keyboard and switch it to Ink Input. Tap the keyboard icon at the bottom right corner of the pop-up keyboard and select the button that looks like a stylus on a slate. Then you just tap the cell you want to input to and write your number on the line in the pop up at the bottom and it will insert those numbers (or words or whatever) right into the cell.

Tap the next cell and write on the line. Rinse, repeat. It's pretty slick.
