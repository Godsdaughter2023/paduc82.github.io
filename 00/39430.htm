1. Right click on Windows Start > Windows Powershell (Admin)
Type following (or copy and paste following) one by one and Enter

net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
Ren C:\Windows\SoftwareDistribution SoftwareDistribution.old
Ren C:\Windows\System32\catroot2 Catroot2.old
net start wuauserv
net start cryptSvc
net start bits
net start msiserver

2. Type following in Windows Start Search box and hit Enter %systemroot%\Logs\CBS > Rename CBS.Log to something else.
If it doesn't allow you to rename - Type services in Windows Start Search box > Click Services > Scroll down to find the Windows Modules Installer service and double-click on it > Set its startup type to Manual > Reboot your computer
Go to C:\Windows\Logs\CBS > Rename CBS.Log > Reboot your computer > Then go back into the Services App and set Windows Modules Installer Service back to Automatic Startup (Note CBS log files contain information about windows update, SFC etc)

(Source - https://partnersupport.microsoft.com/en-us/par_... )

How do I find a deleted storage folder in my windows live - Microsoft Community

hunter2411 say:

How do I find a deleted storage folder in my windows live

I was cleaning up my email storage folders and accidently deleted one folder that ended up deleting all the folders linked to it. I can see them in the find option but can't seem to work out how I can get them back. Help please

a. user say:

Hi,

 

If you’re using Windows Live Mail, you can selectively recover the deleted storage folders using the Previous Versions component. You will need to know the name of the folders that were deleted in order for this to work. Follow the steps below on how to do this.

 

1. You will first need to select the Show hidden files option.

    a. Open Windows Explorer and click Organize / Folder and search options. Click the View Tab.

    b. Place a check mark in the Show Hidden files, folders and drives option. Click Apply/OK.

    c. Close and restart Windows Explorer.

 

 2. Open the WLM Store folder.

    a. Navigate to the following folder.

    C:\Users\your name\ AppData\ Local\ Microsoft\ Windows Live Mail

     (Where 'your name' is the name of the account that you use to logon to Windows 7)

    b. In this folder you will see all of the WLM storage folders. Each email account will have its own group of folders.

    c. Any general storage folders that you created will under the main Storage Folder item.

 

3. Make a copy of the Windows Live Mail folders previous version.

    a. Right click the Windows Live Mail folder and select Properties / Previous Versions Tab.

    b. After the list populates, select a version of the folder with a date and time prior to when the folders were deleted.

    c. Click the Copy Button. In the Copy dialog box, select the Desktop as the location to place the copied folder.

 

 4. Move the previous versions of the deleted folders to the WLM store folder.

    a. Go to the Desktop and open the copied WLM folder. Locate the folders that were deleted.

    b. Right click the folder and select Copy. Go to the current WLM storage folders and paste the folder into the same location where it was deleted from.

 

After you have retrieved all of the deleted folders, you can delete the copied WLM previous version folder from the Desktop.

 

Thanks!

cw93z say: I recently had this issue and took ownership of the SystemData folder, which gave me access to the folder and erased the lock screen but nothing more.  I then noticed that the Administrators group had been removed from the permissions on the folder - it is this group that I believe is used to authenticate when setting the screen.

Added the Administrators group back in, giving it Modify rights i.e. not Full Control and voila it all worked.  I can now set the lock screen to any wallpaper I like via the standard route of PC Settings.

Hope this works for you.

cw93z say:
What name did you give and which category did you choose?

did you just type administrators and it all worked?


Check the Security tab on the Properties window for the SystemData folder, if the Administrators group is listed then it should have Modify access allowed, plus your local user account should have Full Control - this is what worked for me.

If any of the above are not set then:
a. Ensure that your local user account has ownership of the folder by clicking Advanced
b. Once you have full control of the folder then back on the Security tab click Edit followed by Add.  Just type Administrators (notice the plural so that the group is selected rather than the local Administrator user account).  Once added give the Administrators group access to Modify the folder.

As I said above, this worked for me after a several failed attempts to fix the problem.  I realised from the outset that it was probably a permissions issue but tracking it down took quite a while.  I hope this works for you.
