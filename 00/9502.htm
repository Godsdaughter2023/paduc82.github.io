I would first explore the settings in your printer setup to see if there is a way to turnoff greyscale.

 

Failing that, one workaround that seems to work for me is to File > Publish as a PDF which does 'print' the background, and then print the pdf file on your HP.

 

The second fix:

1. Select all the design elements including the text boxes etc, group together and drag off the publication into the scratch area. To make this easier go to View and click Boundaries and Guides so you can see the different elements. Then hold down you Ctrl key and click on each text box and picture and then group them together and drag off. This will make it easier to move them back on later and keep the layout intact.

 

2. After you move off all the text etc you should have only 'background' left. This is actually composed of two elements. If you move your cursor over the boundary you can drag that 'rectangle' element to the side to expose a second 'rectangle' element that has a solid green fill. Tip...if you hold down the shift key when you drag it off, it will stay in line and then hold the shift key again when you drag it back.

 

3. Now that you have just the 'empty' rectangle on the page, select it and go to the Fill icon on the menu and click the down arrow (the bucket pouring) and choose Fill Effects. This will open the Fill Effects dialog.

 

4. In the Fill Effects dialog > gradient tab change colors from two colors to one color, 'color one' click the down arrow and choose the Accent 3 green color (102,153,153), move the dark to light slider all the way to light, choose the variant with the color at the bottom of the box, OK out. You should have a rectangle object that looks the same as it did before. Now go to File > Print preview and you should see the gradient background instead of an empy rectangle and it should be in black and white.

 

5. Move all your text elements etc. back onto your page on top of the rectangle (remember the hold shift key as you drag tip), do a print preview and you should be good to go.

 

Once again do this on a copy of your orginal in case something goes wrong.

john - project say:
Nathalie,
There is nothing inherently wrong with your syntax
ActiveProject.Tasks(1).Assignments.Add ResourceID:=9, Units:=0.6

However, if you attempt to add that same resource as another assignment to the same task, you will get a runtime error, because that resource is already assigned to the task.

I'm just guessing here but what I think you may be trying to do is to change the resource assignment, not add the assignment. If you want to change an existing assignment, you first must delete the assignment and then use the add method. Or, if you just want to change the units assignment of an existing assignment, the following syntax will work.
ActiveProject.Tasks(1).Assignments(1).Units=0.6
Note: you would need a loop of all that tasks assignments to determine which assignment index to update.

Hope this helps.

John

ActiveProject.Tasks(1).Assignments.Add ResourceID:=9, Units:=0.6

Which is the same as what you had, but no brackets. John actually did the same as I just did, be he forgot to mention it!

Some ideas:

Sub Demo()
    Dim M
    Dim J
    Const D = 40
   
    M = [A1].Resize(D, D)
   
    With WorksheetFunction
        For J = 1 To 4
            M = .MMult(M, M)
        Next J
    End With
   
    [A50].Resize(D, D) = M
End Sub
