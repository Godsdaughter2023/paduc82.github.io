hansv mvp say:

This is tricky, since the Record Source of a form or report could be

  • The name of a table
  • The name of a query
  • A SQL statement

Although it's unusual, it is permissible to give a table or query a name that looks like a SQL statement. So in theory, it's difficult to distinguish between the types.

 

But if you're sure that the record source is either a table or a query, you can create the following custom VBA function:

 

Function ObjectType(Source As String) As String
    Dim varType As Variant
    varType = DLookup("Type", "MSysObjects", "Name='" & Source & "'")
    If IsNull(varType) Then
        ObjectType = "Error!"
    Else
        Select Case varType
            Case 1, 4, 6
                ObjectType = "Table"
            Case 5
                ObjectType = "Query"
            Case Else
                ObjectType = "Other"
        End Select
    End If
End Function

 

You can then use

 

="Based on " & ObjectType([RecordSource]) & ": " & [RecordSource]

For 5.4.1 error, it will occur when Exchange Online Protection rejects the coming messages because the Directory Based Edge Blocking (DBEB) doesn't recognize the public folder as a valid recipient in Exchange Online. A workaround for this issue is to covert the SMTP domain from Anthoritative to Internal Relay.
