dhl2060 say:

i have a multi-value combo box so that employees can pick all of the reasons a member has come to the credit union for.  We have been doing this on paper for each branch - which is ridiculous.

Now that I have this multi-value combo box on a form, and it works great, I would like to be able to pull out all of those reasons, sort them by reason and add them up to see what type and how many of each inquiry we are getting at each branch.

Make sense?

So far, I'm not having much luck with this and would really appreciate a helping hand.

ken sheridan say: Unless your application is interfacing with SharePoint there is no reason for using a multi-valued field, and you are only causing problems for yourself by doing so.  Use a conventional model in which a table represents the many-to-many relationship between members and Reasons, so the tables would be along these lines:

Members
....MemberID (PK)
....FirstName
....LastName
....BranchID (FK)
....etc

Reasons
....ReasonID (PK)

and to model the relationship between them:

MemberReasons
....MemberID (FK)
....ReasonID

There would also be a Branches table referenced by the Members table on BranchID.

The MemberReasons table models the many-to-many relationship type by resolving it into two one-to-many relationship types.  If the same reason can be recorded only once per member, then the primary key of this table is a composite one made up of the two columns.  If the same reason can be recorded more than once per member, e.g. on different occasions, then the table would have another column to represent this other attribute, and this would also be part of the key.

For data entry the appropriate interface for this would be a Members form, in single form view, within which would be a subform, in continuous forms view based on the MemberReasons table.  The parent form and subform would be linked on MemberID.  To select a reason the control in the subform would be a combo box set up as follows:

ControlSource:    ReasonID

RowSource:     SELECT ReasonID, Reason FROM Reasons ORDER BY Reason;

BoundColumn:   1
ColumnCount:   2
ColumnWidths:  0cm;8cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the first dimension is zero to hide the first column.  As few or as many reasons per member can be entered by inserting a row for each in the subform, selecting the reason from the combo box in each case.

To count the reasons per branch you now just need a simple query which groups by branch and reason and returns the count of rows per group, so the query would join Branches to Members on BranchID, Members to MemberReasons on MemberID, MemberReasons to Reasons on ReasonID.  Return the branch name and reason in two columns and group by these columns.  Add ReasonID from Reasons to a third column and select Count in the 'total' row for that column.

johnlv say: I fundamentally agree with Ken, but if you want to generate the report without having to make these changes, you must include the underlying list of values in the query you're using to do the analysis.  Rather than include the Reason multi-value field in your query, ask for all the values instead by including Reason.Value on the query grid.  This will generate one output row per customer per reason to allow you to do the analysis.

Multi-value fields are one of the worst ideas ever implemented in Access.  They seem to be perfect for situations like this on the surface, but they simply hide what should be the correct relational structure.  Thus you end up with complete confusion when trying to solve problems like this.

To vlookup two criteria you need to create a new first column containing the combined criteria as a new criterion. For example, it only works when the first column in your table array contains "B7&C4".

In your case, you have to insert a new column before Col A in tab CW_8.22, which will contain the the two requirements (use formula such as "=B7&C4").

=INDEX('CW_8.22 '!D3:D$24764,MATCH(1,(B7='CW_8.22 '!A3:A$24764)*(C4='CW_8.22 '!B3:B24764),0))

=INDEX('CW_8.22 '!D3:D$24764,MATCH(1,INDEX((B7='CW_8.22 '!A3:A$24764)*(C4='CW_8.22 '!B3:B24764),0,1),0))

increase the bass level (causes clipping in loud passages - I'm guessing this is what you tried and didn't like)

compress the bass (bass will be loud enough in quiet passages, but won't be louder in the loud passages - I think this is a bit more processor intensive and complex and doubt there's any software that does it)

reduce the treble and midrange (you will have to turn the volume up, but if you can that's the best option as it should be lower in distortion). I would try this last idea and see how that does.

stephen boots say:

You managed to elude a fake antivirus infection.

 

Unfortunately, these type of malware attacks are difficult to keep up with because they trick you into letting them install. They usually come from an infected web site, and usually through an advertisement. You get a pop-up from the infection and you click it to close the pop-up - which allows the infection to install.  They can also be delivered in a "drive-by" fashion with no action needed by the user due to the system being unpatched, no matter what security software is running.

 

When you encounter one of these fake virus pop-ups while browsing, immediately do the following:

-Do not touch any browser window to close it or browse further.

-Use the key combination <ALT>+<F4> to close all running programs, especially the web browser

--or--

-Immediately press Ctrl-Alt-Del and bring up Task Manager and forcibly end all instances of iexplore.exe, if using Internet Explorer, or the executable for your browser for any other web browser.
--or--
-Go to Start/Shut Down and restart the PC without touching any browser windows.
-If you used task manager to close browser instances, reboot the machine.
-Then go to Control Panel/Internet Options and delete all temporary Internet Files and cookies. If you are using an alternate web browser, open the browser settings to do the same - delete the local cached files and cookies.
-Perform a full scan with MSE.

The above steps should prevent the infection from taking hold.

Start here - https://support.microsoftsecurityessentials.com/

and select the link that says - I think my computer is infected. Options will vary by region, but phone support leads you to Microsoft Answer Desk (http://www.answerdesk.com/) in the US at this time. After an initial free consultation, a fee will be charged for assistance, based on the details of the case.

 

This web site - http://www.bleepingcomputer.com -  contains details for many of these common infections, often immediately after they began to appear in the wild, and instructions are provided for how to remove the infections using their malware removal guides. They also have forums where you can seek help from people who specialize in malware removal.

 

Besides MSE, the following recommendations will assist in protecting the PC from infection:

-Make sure that the Windows Firewall is enabled.

-Make sure that all important/critical updates, including service packs for the operating system and programs are installed from Microsoft Update (Windows Update).

-Make sure Internet Explorer is at the latest version for your operating system and updated with all patches.

-In Internet Explorer 8, 9, or 10, use the SmartScreen Filter.

-Make sure that IE Internet Security settings are at least set to medium-high (default).

-Enable the pop-up blocker in IE.

-On Vista and Windows 7 make sure that User Account Control (UAC) ON  and not running with elevated privileges.

-Make sure that Windows Automatic Updates are set to at least notify, but the preferred setting is to download and install automatically. If you update manually, be sure to update as soon as possible after being notified of available updates.

-Make sure that installed applications, especially Adobe Acrobat, Adobe Flash, and Java are at their latest versions. Many vendors are regularly updating and patching for security holes.

See: https://blogs.technet.com/b/security/archive/2012/07/19/the-rise-of-the-black-hole-exploit-kit-the-importance-of-keeping-all-software-up-to-date.aspx?Redirected=true

-Never click through links from unknown sources and use caution even if they are from a "trusted" source.

-Never open unsolicited email attachments.

-Practice safe web browsing.
