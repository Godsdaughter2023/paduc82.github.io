You can reference any value on an open form using the syntax:

So if you need to filter by the selected contact, you would place that as the criteria in your query for the contact column. Use your actual form and control names.

ken sheridan say: As Scott has pointed out, to select a contact from a combo box's list you will have to include the combo box in a form.  Moreover, the combo box's value should be a distinct numeric ContactID value or similar.  Personal names can legitimately be duplicated, so are unsuitable as keys.

You might like to take a look at FindRecord.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes an option to filter by a foreign key.  In this case the unbound combo box is in the header of a bound form, which is filtered by code in the control's AfterUpdate event procedure, but it could equally well be in an unbound dialogue form in which a query is opened with a button on the form.  The query would reference the combo box as a parameter on the ContactID column using the syntax:

    Forms![NameOfForm]![NameOfComboBox]

The demo lists contacts solely by name in the combo box's list, so there would be no way of distinguishing two or more contacts of the same name.  For an example where the combo box lists other data to differentiate between people of the same name take a look at NotInList.zip in the same OneDrive folder.  The opening form of this demo includes concatenated address data in the combo box's list to differentiate between contacts of the same name.

vincentchoy say:

There are two kinds of networks in Yammer. Internal and External.

An Internal network is meant for company staff, and as such is highly restrictive against outsiders from joining.

However, you may invite outsiders in via guests (usually temp staff, board of directors), or via external groups.

The method has been posted in the previous reply.

The difference between a guest in the network vs external groups, is that a guest in an internal network would be treated with the same rights as an employee, whereas if you invite an external party in a external group then the person can only go to that group.

Alternatively you can create an external network, and invite any number of outsiders to discuss matters in a totally separate network.
