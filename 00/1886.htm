jay freedman say:
I think that because you're running this in VBScript rather than inside Word's VBA environment, the named constant wdReplaceAll is locally undefined. Like most Basic variants, VBScript then assumes that the symbol wdReplaceAll is just a variable that hasn't been initialized, so it's implicitly given the value 0.

In VBA, a Replace parameter with a value of 0 corresponds to the named constant wdReplaceNone. You didn't say what error is displayed when you run the macro, and I wouldn't expect an error message, just that it wouldn't replace anything.

Two things you can try: In the VBScript, initialize the value
   wdReplaceAll = 2

or simply remove wdReplaceAll from the Execute statement and insert the number 2 there. (That's the value of the named constant in VBA.)

Also, it looks right in your post, but make certain that there are 10 commas in the Execute statement. (Within VBA you could use named arguments, but VBScript supports only positional arguments.)
