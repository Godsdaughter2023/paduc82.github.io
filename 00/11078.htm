yoga yao msft say:
Hi Clynk,

Each site collection has its unique metadata, while subsites inherit all these properties from the top-level site in a site collection. So in other words, if you setup metadata or navigation or security groups or adjust site colors and logos on Site Collection A, you have to manually repeat all of it on Site Collection B, should you decide to proceed that route.

When you are in the following scenarios, you can create subsites.
    1. Create a site with same metadata, security or navigation as the existing sites you already have.
    2. You are a small organization.
    3. Keep same look and feel as the template site.
    4. Have a handful of SharePoint sites in your organization.

When you are in the following scenarios, you can create site collections.
    1. You are a large organization with multiple business units and each business unit has independent and separate business functions.
    2. You are a large organizations and each of your departments has a large number of people and each department is expected to create lots and lots of sites.
    3. You are a large organization and want to separate your SharePoint administrative functions. (Separate SharePoint administrators for each site collection)
    4. Your organization has lots of content. (Server resources and storage)
    5. Share content externally.

You can decide which one is better according to your scenario. You can refer to this article for more details about sites and site collections.

pete_uk say:

You can do this by formula and a bit of manual intervention. Insert a new column A in both sheets, and put this formula in A2 of both:

 

=TRIM(B2)&","&TRIM(C2)

 

copy down as far as is necessary. This will combine last name with first name and put a comma between them. Then in Sheet1 cell E2 you can use this formula:

 

=IF(ISNA(MATCH($A2,Sheet2!$A:$A,0)),"",VLOOKUP($A2,Sheet2!$A:$E,COLUMN(D2),0))

 

Copy this into F2, and then copy E2:F2 down as far as you need.

 

Now you can fix the values in those two columns, by selecting columns E and F, clicking <copy>, right-clicking and selecting Paste Special, clicking on <Values> then <OK>, and then pressing the <Esc> key.

 

Finally, you can delete both the inserted columns A in both sheets.

 

Hope this helps.

 

Bye the way, I don't know if Smith,John in Sheet1 of your example is meant to be the same as Smith,Jonathan in Sheet2, but as these are not exact matches they will not be considered as the same by that formula.

 

Pete

kenwhelp say:

I would like to add the following to this. I had the same error "Microsoft Security Client OOBE stopped due to following error 0xC000000D" in my Event Log. It seems that Microsoft Security Essentials and Microsoft Security Client each have a different file causing a similar error. First you have to set  the Folder and Search options to "Show hidden files, folders and drives", or you won't be able to see the ProgramData folder.

Now, for MS Security Essentials you locate and delete the C:\ProgramData\Microsoft\Microsoft Security Essentials\Support\MSSEOOBE.etl file and reboot. After reboot this file should be recreated and the error should be gone.

For MS Security Client you have to locate and delete the C:\ProgramData\Microsoft\Microsoft Security Client\Support\EppOobe.etl file and reboot. After reboot this file should be recreated and the error should be gone. It worked fine for me and got rid of the error after the next reboot. Hope that helps.
