This is a very loose giude:

Windows must first start by loading a mini OS with a miniumum set of device drivers to comunicate with the motherboard chipset and grahics card.

After you choose the type of installation then Windows loads and expands any uncompressed files specific to the version you are installing.

A script is followed which directs Windows what to perform including a set of chipset and device drivers to allow communication with hard drive, LAN, Etc.

Windows the starts the actual installation process inclusing building the registry, starting a bare mininum of Services.

On each reboot more Windows components and services are made available and for all practical purposed you are running a subset of Windows 7.

By the time you enter your name and password (which creates a user profile) most of Windows is installed.

a. user say:

Method 1:
1. Open Windows Explorer
2. Right-click the encrypted file or folder, and then click Properties.
3. On the General tab, click Advanced.
4. Clear the Encrypt contents to secure data check box.

Method 2: Do you have a Recovery Agent?
1. Make a copy of the file in case of loss or damage.
2. Send the original encrypted file to the designated recovery agent.
3. Have the recovery agent use their recovery certificate and private key to decrypt the file.
4. Have the recovery agent send the decrypted file back to you, using any file transfer method that is desired.
Source - EaseUs.com

Method 3:
Import the recovery agent certificate with private key to the server stored the encrypted files, and then open the files as the user himself.
If you can open the encrypted files, I suggest you use recovery agent to decrypt the files and then re-encrypt the files as user's certificate.

Method 4:
The EFS certificate files can be found in "C:\Documents and Settings\<username>\Application Data\Microsoft\SystemCertificates\My\Certificates".
They are stored each in one file, named by thumbprint, with no extension.

If by some miracle you do manage to locate them then try this: http://technet.microsoft.com/en-us/library/cc72...

If you are unable to import them you will be forever unable to access the contents of the encrypted files. - Spiceworks

robert aldwinckle on forums say:

after user login there are no icons - only a grey taskbar at the bottom of the blue screen with the Windows button on the left, keyboard button, and Upload Centre button on the taskbar.


Sounds like the monitor that you are looking at is being treated as your secondary in an Extended Display--with the Taskbar enabled on both.   Try pressing Win-P, CursorUp, Enter  (as if you could see the Project aka Second Screen dialog) and use it to switch to a mode which will show something on the monitor that you are looking at.   BTW most users having this symptom seem to have nVidia video support.

robert aldwinckle on forums say:
             This is a tablet (no keyboard).

You have the On-Screen Keyboard (aka OSK) --  an option in the Accessibility menu or start it in the Task Manager as osk.exe but you would have to have the Touch Keyboard (TK) to do that.  It is possible to enable a layout called the Standard keyboard for the TK, which has a Win- key too, but that is not a default and needs to be enabled using PC Settings (under Devices, Typing apparently--I searched for it and then went backwards to find out how you could have navigated there).





seandonovan say:

On the Iconia you get to the bios settings at startup by holding the volume-down button while simultaneously pressing the power button.


I went into the bios setting screen and, without changing any settings, saved the settings.


Upon reboot the Windows initially flashed to the blue-grey window and then flicked over to the normal Windows Start screen.


Subsequent reboots now go to the Windows Start screen correctly.


(Note: It appears you do need to save the settings without any changes. I initially went into the bios settings but did not save, and that didn't fix the problem)
