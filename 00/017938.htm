Dim prm As DAO.Parameter

Change this snippet of code

  Select Case Left(qryX.Name, 7)
    Case "qryErro"
      Set rstQryX = qryX.OpenRecordset(dbOpenDynaset)
      If Not (rstQryX.BOF And rstQryX.EOF) Then

      End If
  End Select

to

  Select Case Left(qryX.Name, 7)
    Case "qryErro"
      If qryX.Parameters.Count > 0 Then
        For Each prm In qryX.Parameters
          prm.Value = Eval(prm.Name)

Another option that won't impact existing formats or format conditions is to use "Find":

  1. Use keyboard short cut Alt-E F to bring up the "Find and Replace" dialog. 
  2. Click "Options>>" if you don't see a "Format..." button.
  3. Click "Format...". 
  4. Click "Protection" tab. 
  5. Check "Locked" and uncheck "Hidden"
  6. Click "OK"
  7. Click "Find All"

Notes!

  • This only works on used cells.  Cells that are locked that have not been used will not display.

  • When designing workbooks, I highly recommend setting the INPUT style to Unlocked and changing its color to whatever seems best to you. Then, with the INPUT style set to Unlocked and the NORMAL style set by default to Locked apply INPUT to all cells where user entry is allowed to show users which cells are open for entry.

brian tillman [outlook mvp 2007-2019] say:

Don't restart your computer, start Outlook, click the Send/Receive tab, and select "Work Offline", then close Outlook, wait a just a bit, then start Outlook again.  You should be able to delete the message.  Alternatively, unplug your PC from the network, start it, then start Outlook.  When the send/receive fails you should be able to delete the message.
